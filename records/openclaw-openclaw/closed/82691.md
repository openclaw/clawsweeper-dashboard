## Summary

The PR adds a `messages.toolMessageLogging` boolean across config schema/help/types/docs and uses it in reply dispatch to override intermediate tool-summary/status delivery, with focused dispatch tests.

## Best Possible Solution

Rework the PR around the canonical trusted-room verbose-progress policy: preserve Slack non-DM defaults, apply one effective gate to summaries, statuses, and plan updates, document the active config page/security model, and provide redacted real chat proof.

## Reproduction Assessment

Yes, source-reproducible: current main suppresses Slack non-direct progress at the shared dispatch gate, while the PR default drops that condition and leaves plan updates on the old gate. I did not run tests because this review is read-only.

## Solution Assessment

No. A config override may be a maintainable direction, but this implementation should preserve existing channel defaults, use the override consistently, update the active docs page, and align with the trusted-room policy before merge.
