## Summary

Keep open: current main still excludes allow-only plugin ids from both the one-time release repair set and the normal missing-install repair, while stale cleanup can remove those ids afterward. The closed related PR #77193 matches the narrow fix direction but was not merged into current main.

## What This Changes

The issue asks doctor migration repair to preserve and install allow-only official external plugins such as `lobster` instead of removing them from config.

## Best Possible Solution

Add an official-catalog-gated pass over `plugins.allow` to the one-time release repair set, preserving arbitrary allow-only exclusion while ensuring known externalized plugins are installed before stale cleanup.

## Reproduction Assessment

Yes, by source inspection: current main still omits allow-only ids from the release repair set, normal missing-install repair also excludes allow-only ids, and stale repair removes unknown `plugins.allow` entries. I did not execute a live doctor run in this read-only review.

## Solution Assessment

Yes, the best fix is the narrow path described by the issue and closed PR #77193: include only allow-only ids that resolve to official external catalog entries in the release repair set, without changing normal arbitrary allowlist behavior.
