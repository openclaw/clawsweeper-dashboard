## Summary

Current main already implements this request: custom-provider onboarding now defaults to the 16,000-token hard minimum and normalizes older lower values such as 4096 upward.

## Evidence

- **16k hard minimum is the source of truth:** `CONTEXT_WINDOW_HARD_MIN_TOKENS` is defined as `16_000`, and the guard message uses that same minimum.
  - file: [src/agents/context-window-guard.ts:5](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/src/agents/context-window-guard.ts#L5)

- **Custom-provider onboarding uses the hard minimum by default:** `DEFAULT_CONTEXT_WINDOW` is set from `CONTEXT_WINDOW_HARD_MIN_TOKENS`, so new custom OpenAI-compatible providers no longer default to 4096.
  - file: [src/commands/onboard-custom-config.ts:16](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/src/commands/onboard-custom-config.ts#L16)
