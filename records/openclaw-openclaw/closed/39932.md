## Summary

Current `main` already serves the root-mounted Control UI when enabled and contains the package-root/global-install resolution needed for pnpm-style installs. The same implementation is present in release `v2026.4.22`, so this report appears fixed on current OpenClaw.

## Evidence

- **Root path is explicitly served for root-mounted Control UI:** `classifyControlUiRequest()` returns `{ kind: "serve" }` for `GET /` when `gateway.controlUi.basePath` is unset, which is the exact path from the report.
  - file: [src/gateway/control-ui-routing.ts:18](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/gateway/control-ui-routing.ts#L18)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Gateway defaults Control UI to enabled and wires the Control UI HTTP handler:** Runtime config resolves `controlUiEnabled` from `gateway.controlUi.enabled ?? true`, and the HTTP server registers the `control-ui-http` stage when enabled.
  - file: [src/gateway/server-runtime-config.ts:83](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/gateway/server-runtime-config.ts#L83)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Control UI handler serves `/` from `index.html` instead of 404ing:** `handleControlUiHttpRequest()` resolves the UI root, maps `/` to `index.html`, and serves it; only missing/unsafe assets fall through to 404.
  - file: [src/gateway/control-ui.ts:778](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/gateway/control-ui.ts#L778)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Global/symlinked install root resolution is covered:** `resolveControlUiRootSync()` checks realpath-resolved `argv1`, package root, and `dist/control-ui`, which is the path shape expected for pnpm/global installs.
  - file: [src/infra/control-ui-assets.ts:188](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/infra/control-ui-assets.ts#L188)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Tests cover the previously reported shapes:** Tests assert root-mounted `/` is served, symlinked/global `argv1` resolves to the package root, and pnpm-style hardlinked bundled assets are treated as bundled roots rather than rejected.
  - file: [src/gateway/control-ui-routing.test.ts:5](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/gateway/control-ui-routing.test.ts#L5)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
