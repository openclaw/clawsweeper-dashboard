## Summary

Current `main` already contains the isolated-cron timeout mitigations described in the report: nested embedded agent work is remapped off the single-concurrency `cron` lane, and descendant follow-up reads are scoped to the current run window. The shipped `v2026.4.22` tag contains the same implementation and regression tests, so this issue is safe to close as already fixed.
