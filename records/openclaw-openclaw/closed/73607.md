## Summary

Keep this issue open. The reported fields remain documented, schema-supported, and consumed by the runtime, but current main still lacks a targeted persistence fix or regression test proving that gateway/config write-back preserves `agents.defaults.params` and `agents.defaults.models[...].params` across the reported reload/write path.

## Best Possible Solution

Keep this issue open and add a focused regression around `replaceConfigFile`/`writeConfigFile` or `resolvePersistCandidateForWrite`: start with source config containing both `agents.defaults.params` and `agents.defaults.models["openai/gpt-5.4"].params`, simulate the gateway/runtime write that changes `agents.list` or bindings, and assert the persisted source config retains both params objects. If it fails, fix the lossy runtime-to-source projection or mutator path while leaving the existing extra-params runtime behavior intact.
