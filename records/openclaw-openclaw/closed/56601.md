## Summary

Current main no longer exposes the stale Control UI per-message browser-native speaker/read-aloud button that ignored configured TTS voices. The v2026.4.26 changelog explicitly records that fix, and current regression coverage asserts the old `.chat-tts-btn` / `Read aloud` action is absent.

## Best Possible Solution

Keep the shipped v2026.4.26 behavior: do not restore the browser-native per-message speaker, and direct configured voice output through Talk or Gateway-backed TTS paths such as auto-TTS and `/tts audio`.
