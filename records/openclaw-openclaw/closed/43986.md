## Summary

Current main already forwards `stream: "tool"` events over Gateway WebSocket surfaces, including run-scoped WS recipients and session-scoped `session.tool` mirrors for late-joining observers, so the feature request is already implemented well enough to close.

## Close Comment

Closing as implemented. Codex review of current `main` found that Gateway WebSocket delivery already forwards `stream: "tool"` events: `src/gateway/server-chat.ts:930` broadcasts tool events to registered WS recipients and `src/gateway/server-chat.ts:954` mirrors them onto `session.tool` for session subscribers, which covers observers attaching after a run was started through `/hooks/agent`. Clients opt in via the `tool-events` capability in `src/gateway/server-methods/agent.ts:743`, and the behavior is covered by tests in `src/gateway/server-chat.agent-events.test.ts:730`. This is also present in the shipped release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
