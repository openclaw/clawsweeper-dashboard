## Summary

Current main already covers the reported failure mode well enough to close: malformed Anthropic SSE JSON is converted into a stable transport error, chat.send broadcasts a terminal error and cleans the active run, and the Control UI clears chatRunId on error/abort so the input and stop state do not remain stuck. The proof commit is on main but is not contained in the latest tagged release v2026.4.26.

## What This Changes

The issue asks for webchat/edit-tool recovery when non-ASCII edit oldText/newText triggers an Anthropic JSON parse failure and leaves the chat UI frozen.

## Best Possible Solution

Keep the current main behavior, ship it in the next release, and use the broader Anthropic streaming issue for any remaining provider-stream malformed-fragment cases that reproduce after this fix.
