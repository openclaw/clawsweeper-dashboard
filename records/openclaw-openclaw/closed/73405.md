## Summary

Current main implements the WebChat duplicate-message fixes that cover this v2026.4.24 report: unowned async/inject/announce/side-result final events no longer clear or replay the active run, final payloads with visible assistant text no longer force redundant history reloads, stale history reconciliation avoids optimistic/canonical duplicates, and the older delivery-mirror transcript family is filtered/deduped.

## Best Possible Solution

Close this issue as implemented on main. Keep the current run-ownership-aware WebChat event handling, final-event reload gating, history reconciliation, and transcript-only filtering; if duplicate messages still reproduce on a newer build, the next issue should include fresh steps, logs, session history excerpts, and the affected channel/plugin path.
