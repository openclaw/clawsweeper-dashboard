## Summary

Current `main` already handles this in the core Pi path: model switches clear stale cached `contextTokens`, each run resolves the active model's effective context budget before starting, and preflight overflow logic compacts instead of submitting an oversized prompt to the newly selected smaller-window model.

## Evidence

- **Model switch clears stale cached context budget:** `applyModelOverrideToSessionEntry()` deletes `entry.contextTokens` when the selected model changes or the runtime model is stale, so a previous larger-window model does not pin the session budget after `/model` changes.
