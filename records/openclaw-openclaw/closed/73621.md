## Summary

Current main still matches the reported Control UI loss path: busy follow-ups are kept only in `host.chatQueue`, session switching clears that array before the gateway receives the queued message, and an open contributor PR already targets this issue with closing syntax. Per policy, this should stay open until that PR is reviewed and merged or maintainers choose another fix.

## Best Possible Solution

Keep this issue open and review/land PR #73679 as the immediate focused fix. If that PR is not accepted, implement session-scoped preservation for queued Control UI messages, restore them when switching back, and add regression coverage for switch-away/switch-back with a pending follow-up. Broader refresh and reconnect durability can remain tracked by #45952 and #51549.
