## Solution Assessment

Yes, this is the best narrow fix for the self-open-id portion: it filters at the receive boundary using the existing bot identity source and avoids dropping all app-authored receive events. Queue timeout, backpressure, and session-lock fixes belong under https://github.com/openclaw/openclaw/issues/90559 instead of this PR.
