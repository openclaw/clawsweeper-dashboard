## Summary

This PR adds a third-party Higgsfield remote MCP bundle, docs, and packaging support, but current OpenClaw policy routes optional commercial service integrations and new skills/bundle plugins through ClawHub unless there is a clear maintainer-owned product or security reason to bundle them in core. Existing bundle-plugin support can already install Claude/Codex/Cursor-style bundles and expose HTTP MCP servers outside core, so no missing OpenClaw API blocks an external Higgsfield bundle.

## Best Possible Solution

Close this core PR and route Higgsfield to a ClawHub bundle/plugin package using the existing bundle MCP support. If OpenClaw later wants an official bundled Higgsfield integration, that should come back as a maintainer-scoped product/security decision with labeler/docs coverage.
