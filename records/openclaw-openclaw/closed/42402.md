## Summary

Keep this PR open. Current `main` still lacks the requested runtime guard for malformed non-string `env.vars` values, the crash path is still reachable before schema validation, and the PR remains a focused implementation candidate with regression coverage and a changelog entry.

## Best Possible Solution

Keep this PR open and finish it as the focused fix: add the runtime string/trim guard in `collectConfigEnvVarsByTarget`, preserve the string-only schema contract, keep the mixed-type regression test, decide warning versus silent skip behavior, then run the targeted config tests and the appropriate changed gate.
