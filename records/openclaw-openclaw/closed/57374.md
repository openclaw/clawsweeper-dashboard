## Summary

Keep open. Current main still does not implement the PR's central behavior: restart-health probes read only process env credentials, and daemon restart, update restart, and status restart-inspection paths do not pass config-resolved probe auth into those checks. The PR remains a plausible focused fix, but its current head also has an unresolved review concern around ensuring local restart probes do not fall back to remote credentials.

## Best Possible Solution

Keep the PR open and ask for a final revision or maintainer pass. The best implementation is a restart-health auth resolver that reads configured local gateway auth, preserves explicit/env fallback where intended, does not fall back to `gateway.remote.*` for local `127.0.0.1` probes, and threads that auth through daemon restart, unmanaged restart, update restart verification, and status restart inspection with focused regression tests for config-only local auth and remote-mode credential separation.
