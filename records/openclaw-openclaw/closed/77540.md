## Summary

Keep this PR open: the performance work is still useful and current main does not already implement the session-result, node-pairing, or cron-preview caches, but the current head still has merge-blocking correctness issues around configured-agent session store targeting and config-sensitive cron preview caching.

## What This Changes

The PR adds short-lived in-process caching/coalescing for Gateway session list results, node pairing list reads, cron delivery previews, and manifest model-id normalization lookups, with invalidation tests and changelog text.

## Best Possible Solution

Land the scoped performance caches after rebasing them onto current Gateway diagnostics/configured-store targeting and making cron preview caching config-sensitive; keep broader orphaned-agent cleanup work tracked separately.

## Reproduction Assessment

Yes for source-level review: current main still performs direct request-time session, node, and cron preview reads, and PR head shows the two remaining stale/regressive paths. I did not run live latency reproduction because this cleanup review is read-only.

## Solution Assessment

No, not yet: short TTL caches with explicit invalidation are a reasonable narrow fix, but the PR head must preserve configured-agent store targeting and include runtime config state or config-write invalidation for cron preview cache entries.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve configured-only store targeting:** `src/gateway/server-methods/sessions.ts:699-701`
  - body: Current main passes `configuredAgentsOnly` into `loadCombinedSessionStoreForGateway`, and the existing regression test asserts that configured-only session lists do not even read disk-only unregistered agent stores. This branch wraps caching around the older call shape, so `configuredAgentsOnly=true` still loads broad store targets before filtering and can reintroduce the latency class this PR is trying to reduce.
  - confidence: 0.88
- **[P3] Include config state in cron preview cache keys:** `src/cron/delivery-preview.ts:128-131`
  - body: The new preview cache key is built only from `defaultAgentId` and job fields, but preview resolution passes `params.cfg` into delivery target selection, session-store lookup, account binding, allowlist, and outbound target resolution. A config or routing change that leaves the job list unchanged can return the previous preview for the cache TTL; key this cache with `resolveRuntimeConfigCacheKey(params.cfg)` or clear it on runtime config writes.
  - confidence: 0.87
