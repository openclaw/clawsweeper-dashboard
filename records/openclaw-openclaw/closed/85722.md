## Summary

Keep open: this is a useful-looking new plugin/session workflow primitive, but it should not merge as-is because the same-session contract still accepts a separate agent route, the test helper returns the wrong result discriminator, and the supplied proof does not exercise the production scheduler path.

## What This Changes

The PR adds a bundled-plugin workflow API and SDK facade to request or clear one same-session continuation lease backed by Cron scheduled turns, with focused contract coverage.

## Solution Assessment

No as-is: the narrow same-session lease shape is plausible, but the safest solution is to derive routing from trusted session context only, align helper/docs with the public contract, and require production-path proof or review alongside the /goal PR.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Derive continuation routing from the trusted session:** `src/plugins/host-hooks.ts:304`
  - body: `PluginSessionContinuationLeaseParams` exposes `agentId` separately from `session`, and the implementation passes it through to `scheduleSessionTurn`. That lets a same-session continuation lease carry a mismatched agent route even though the API is advertised as host-owned same-session routing. Please derive the agent from the trusted session context, or reject mismatches with coverage.
  - confidence: 0.86
- **[P2] Return the documented lease result shape:** `src/plugin-sdk/plugin-test-api.ts:80`
  - body: The test API default returns `{ enqueued: false }`, but the new public result type and production implementation use the `scheduled` discriminator. Because `createTestPluginApi()` backs `api.session.workflow.requestSessionContinuationLease`, downstream plugin tests will see `result.scheduled === undefined`; return `{ scheduled: false, reason: ... }` and cover that facade.
  - confidence: 0.94
- **[P3] Document the new workflow API before exposing it:** `src/plugins/types.ts:2539-2545`
  - body: This adds public SDK methods and exported types, but the SDK overview still only documents `scheduleSessionTurn` and `unscheduleSessionTurnsByTag`. Plugin authors need the bundled-only restriction, trusted session-context expectation, failure reasons, and clear semantics documented before this becomes part of the SDK surface.
  - confidence: 0.82
