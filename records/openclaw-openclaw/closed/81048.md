## Summary

Keep open: the PR targets a real Claude CLI resume bug and the latest proof is materially stronger, but the current diff still computes the deterministic transcript directory with a narrower encoding rule than the canonical resolver already on current main, which can turn valid resume sessions into false `transcript-missing` resets.

## What This Changes

The PR threads `workspaceDir` into Claude CLI transcript checks, probes a deterministic session JSONL path with a 250 ms grace scan and warning, and makes `parseCliJsonl` keep the last Claude `session_id`.

## Best Possible Solution

Share or move the existing canonical Claude project-directory resolver into a common module, use it for this transcript probe, keep the last-wins JSONL parser fix, and preserve focused regression coverage for both call sites.

## Reproduction Assessment

Yes, at source level: current main returns false for a missing or header-only Claude project transcript, and both reuse paths turn that false result into `transcript-missing`. The PR discussion also includes live logs for the user-visible context-loss symptom and a v5 after-fix soak.

## Solution Assessment
