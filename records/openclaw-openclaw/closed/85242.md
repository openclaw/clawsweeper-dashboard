## Summary

Keep open: current main still has the 30-minute Codex app-server terminal-idle ceiling and generic timeout guidance, and the open linked implementation PR has not merged yet.

## What This Changes

The issue requests that the Codex app-server terminal-idle watchdog stop hardcoding a 30-minute ceiling and report diagnostics that name the actual limiting timeout.

## Best Possible Solution

Review and land or revise https://github.com/openclaw/openclaw/pull/85296 so the watchdog follows the effective run timeout and terminal-idle failures get accurate guidance, then close this issue after the fix merges.

## Reproduction Assessment

Yes, at source level: a Codex app-server turn with params.timeoutMs above 1800000 ms can still be aborted by the 30-minute terminal-idle constant when no terminal event arrives. I did not live-wait a 30-minute scheduled job.

## Solution Assessment

Yes: deriving the terminal-idle ceiling from the existing effective run timeout is the narrow maintainable fix, with a small follow-through needed for terminal-idle-specific guidance.
