## Summary

Keep open: the branch addresses the main Telegram SecretRef prompt-discovery crash and includes terminal proof, but the latest diff still advertises Telegram actions for token-file accounts that the inspector marks configured_unavailable, which can lead to unusable message actions.

## What This Changes

The PR rewrites Telegram prompt-time action, inline-button, and reaction discovery to use the read-only account inspector for unresolved SecretRef-backed credentials and adds focused Telegram regression tests.

## Best Possible Solution

Land the read-only inspector approach after narrowing discovery to include unresolved SecretRef-backed config while excluding unavailable tokenFile/env credentials, with a regression test for missing or unsafe token files.

## Reproduction Assessment

Yes. The linked issue provides sanitized failing logs for unresolved SecretRefs, and the remaining PR defect is source-reproducible through inspectTelegramAccount returning configured_unavailable tokenFile accounts that channel-actions still advertises.

## Solution Assessment

No, not quite. The read-only inspector is the right direction, but the discovery predicate should distinguish tolerated unresolved SecretRefs from unavailable token-file/env credentials before advertising actions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Skip unavailable token-file accounts during discovery:** `extensions/telegram/src/channel-actions.ts:59`
  - body: `inspectTelegramAccount` preserves `tokenSource: "tokenFile"` even when the file is missing or rejected as unsafe and marks it `configured_unavailable`; this new filter only checks `configured`, and `listTokenSourcedAccounts` then keeps any non-`none` token source. That makes `describeMessageTool` advertise Telegram actions for an account the runtime token path cannot use, so missing or symlinked token files can prompt failed Telegram sends. Keep the SecretRef tolerance, but exclude unavailable token-file accounts or require an available token status for this credential class.
