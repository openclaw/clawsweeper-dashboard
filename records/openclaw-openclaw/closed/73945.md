Full review comments:

- **[P2] Keep startup failures on the startup retry path:** `extensions/feishu/src/monitor.transport.ts:220-221`
  - body: After one SDK reconnect handoff, `recoveringFromDisconnect` remains true, so later `createFeishuWSClient` or `start` failures that are not reconnect-required errors still use the long 120s+ reconnect delay. That makes ordinary startup/factory failures wait minutes instead of following the intended 1s-to-30s startup retry policy.
  - confidence: 0.78
- **[P3] Remove unreachable terminal cleanup:** `extensions/feishu/src/monitor.transport.ts:209-213`
  - body: `raceWithTimeoutAndAbort` is called without `timeoutMs`, so this terminal wait can only resolve, abort, or throw through the input promise. The cleanup and return after the resolved throw are unreachable, which makes the reconnect path harder to reason about.
  - confidence: 0.74
