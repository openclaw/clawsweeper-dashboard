## Summary

Current `main` already treats `plugins.entries.<id>.enabled=false` as authoritative disablement, and plugin config validation only runs for enabled plugins or entries with an explicit `config` object. For the reported shape (`memory-lancedb` disabled, no `config.embedding`, `mem9` selected), the issue no longer matches current behavior; the latest release `v2026.4.23` also contains the same guards.

## Evidence

- **Explicit disablement wins:** Plugin activation resolves `plugins.entries.<id>.enabled === false` to a disabled state with reason `disabled in config`, including bundled plugins and memory-slot candidates.
  - file: [src/plugins/config-activation-shared.ts:178](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/src/plugins/config-activation-shared.ts#L178)
