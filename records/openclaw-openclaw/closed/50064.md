## Summary

Current main no longer matches the reported root cause. The Telegram probe uses `getMe`/`getWebhookInfo` instead of `getUpdates`, the channel wires that probe through status inspection rather than the polling transport, and polling 409 recovery now rebuilds the transport on conflict. The same implementation is already present in release `v2026.4.22`.

## Evidence

- **Probe no longer calls getUpdates:** `probeTelegram` performs `getMe` and optional `getWebhookInfo`; there is no probe-side `getUpdates` call in the implementation described by the issue.
  - file: [extensions/telegram/src/probe.ts:96](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/extensions/telegram/src/probe.ts#L96)
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **Channel uses probe for status inspection:** The Telegram channel registers `probeAccount` through the computed status adapter, so the probe path is health/status inspection rather than the polling runner itself.
  - file: [extensions/telegram/src/channel.ts:762](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/extensions/telegram/src/channel.ts#L762)
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **Polling 409s trigger transport rebuild:** On `getUpdates` 409 conflicts, the polling session resets webhook cleanup state, marks the transport dirty, and restarts so the next cycle uses a fresh transport instead of reusing the stale keep-alive socket.
  - file: [extensions/telegram/src/polling-session.ts:366](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/extensions/telegram/src/polling-session.ts#L366)
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
- **Regression tests cover 409 recovery:** Tests assert that a 409 conflict causes `deleteWebhook` to rerun and the Telegram transport to be rebuilt on the next cycle.
  - file: [extensions/telegram/src/monitor.test.ts:790](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/extensions/telegram/src/monitor.test.ts#L790)
  - sha: [45146913007d](https://github.com/openclaw/openclaw/commit/45146913007da723b4f06aba6a66cf4a695974c0)
