## Summary

This PR should close as superseded, not as implemented. Current main still lacks user-role `<relevant-memories>` stripping in the Control UI/WebChat extraction path and in the server-side `chat.history` display projection path, but the same remaining work is already tracked by broader overlapping PRs, especially #59697, which covers both UI extraction and `chat.history` with regression tests. This PR is a narrower two-file patch with no regression test and no server-side history coverage.

## Best Possible Solution

Close this narrower duplicate and consolidate maintainer review on a single canonical fix for the memory-context display leak, preferably a tested change that covers both the Control UI extraction path and the gateway `chat.history` display projection path without unintended user-text trimming.
