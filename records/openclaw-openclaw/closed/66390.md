## Summary

Current main moved bundled plugin runtime-dependency repair for packaged/global installs out of the installed package tree and into external staged roots, and the package-update flow now reruns doctor in update mode from the updated install. That directly addresses the package-tree update race the issue describes, even though the fix is not a pre-update `launchctl unload` step.

## Evidence
