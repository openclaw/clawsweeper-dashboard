## Summary

Keep open. The PR fixes the selected-media no-provider path, but it still misses the linked bug's no-attachment reproductions because provider resolution happens after attachment filtering.

## What This Changes

The PR propagates media-understanding runtime decisions into file results, maps no-provider skipped image/audio infer results to clearer CLI errors, adds regression tests, and updates the changelog.

## Best Possible Solution

Land a narrow fix that reports missing image/audio provider configuration before attachment filtering, or carries an explicit no-provider signal through no-attachment outcomes, with regression tests for both selected-media and directory/unsupported-path cases.

## Reproduction Assessment

Yes. The linked bug gives deterministic no-provider CLI commands, and source inspection shows directory/no-media paths return `no-attachment` before provider resolution, leaving the generic error path reachable in this PR.

## Solution Assessment

No. Preserving the runner decision is useful, but the proposed helper is too narrow; the missing-provider check needs to happen before selection or survive no-attachment outcomes.
