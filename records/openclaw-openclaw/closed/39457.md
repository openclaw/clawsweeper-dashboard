## Summary

Current main does not add a generic `~/.openclaw/hooks/post-update.sh` hook, but it does solve the reported update failure mode with a supported `--wrapper` / `OPENCLAW_WRAPPER` service contract that persists wrapper-first `ProgramArguments` across forced reinstalls, updates, and doctor repairs. That shipped in `v2026.4.26` via the wrapper persistence work that closed #69400 and replaced #69425.

## Best Possible Solution

Close this as implemented by the shipped wrapper persistence contract. Keep the supported `--wrapper` / `OPENCLAW_WRAPPER` path as the maintainer-blessed solution for durable LaunchAgent wrapper customization across updates; open a separate design issue only if arbitrary post-update hook execution is still desired beyond wrapper persistence.
