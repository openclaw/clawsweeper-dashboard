## Summary

Current `main` now covers the reported Ollama failure modes: native Ollama runs forward `thinking off`, local/default Ollama routing uses `127.0.0.1`, transport `fetch failed` is surfaced as a network error, and embedded-run HTTP dispatchers now inherit the run timeout instead of dying at Undici's default 60s body timeout.

## Evidence

- **Native Ollama now forwards `thinking off`:** `extensions/ollama/src/stream.ts:199` wraps native Ollama requests with `think: false` when `ctx.thinkingLevel === "off"`. The shipped docs also call this out at `docs/providers/ollama.md:464`, and `CHANGELOG.md:257` records the fix for Ollama watchdog/idling behavior.
