## Summary

Current `main` already supports MCP tool-level filtering through the normal tool policy system: bundle/configured MCP tools are exposed as individual tool IDs in `serverName__toolName` form, and the existing `tools.allow` / `tools.deny` pipeline supports wildcard matching on those names. That delivers the requested allow/deny behavior, even though it uses the general tool-policy surface rather than a nested `mcp.servers.<name>.tools` block.

## Evidence

- **Docs say bundle MCP tools can be filtered by individual exposed tool names:** `docs/plugins/bundles.md` documents that bundle MCP tools are registered as `serverName__toolName`, and that profile allowlists/deny lists can include either individual exposed tool names or the `bundle-mcp` plugin key.
