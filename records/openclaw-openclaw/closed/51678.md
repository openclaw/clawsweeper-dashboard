## Summary

Current `main` already implements a global config switch for this request. `messages.suppressToolErrors` is part of the config surface, the embedded-run payload builder reads it before emitting `⚠️ … failed` chat warnings, and tests cover suppression for non-mutating tool failures plus the existing default suppression of normal `exec` errors.

## Risks / Open Questions

- The implementation is global under `messages.suppressToolErrors`; the issue also mentioned a possible per-channel variant, which is not present.
- Mutating tool failures are intentionally still surfaced unless the run-level `suppressToolErrorWarnings` path is used, so this does not hide every possible tool failure class.
