## Summary

Keep #69970 open. Current main still generates the Windows auto-update restart batch file with an unconditional `schtasks /End` before any state check or timeout, and the tests still assert the same `schtasks /End` plus `netstat | findstr` polling behavior reported by the issue. The related #71909 PR is open and described as addressing only the stale-listener/findstr portion of #69970, so it does not make this issue obsolete on current main.

## Best Possible Solution

Keep #69970 open as the canonical bug for the Windows auto-update restart helper hang. The best fix belongs in `src/cli/update-cli/restart-helper.ts`: avoid or bound `schtasks /End` when the Scheduled Task is not running, remove the visible `findstr` pipeline or move it into hidden PowerShell/cmd-safe logic, and add Windows restart-helper tests that assert the generated helper cannot hang before self-cleanup. #71909 can be expanded to cover the `schtasks /End` hang, or a separate PR should link and close this issue after landing.
