## Summary

Keep open. Current main still defaults/falls back to WhatsApp `pairing` and can send pairing-code replies for live unknown DMs; the guided personal-number setup flow is only a mitigation for users who run it, so changing the remaining default/fallback needs maintainer security/product policy direction.

## What This Changes

The issue requests safer WhatsApp personal-account DM defaults so unknown contacts do not receive pairing-code replies from the user's own number.

## Best Possible Solution

Define the supported WhatsApp account-intent policy so personal-number accounts get durable owner-only allowlist behavior while dedicated/shared bot accounts explicitly opt into pairing.

## Reproduction Assessment

Yes. Source inspection on current main shows unset WhatsApp policy resolves to `pairing`, and the existing access-control test path asserts that a recent unknown live DM sends a pairing reply.

## Solution Assessment

Unclear as an implementation choice. The setup prompt/workaround mitigates configured personal-number setups, but the remaining fallback/default behavior needs maintainer policy for personal versus dedicated WhatsApp accounts before a narrow fix is safe.
