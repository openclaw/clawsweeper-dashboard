## Summary

Current `main` does not match the reported routing bug. The image-generation tool passes `agents.defaults.imageGenerationModel` into the shared resolver, the resolver returns explicit `primary`/`fallbacks` before provider inference, and the runtime tries `imageGenerationModel.primary` before auto-detected dialog-provider defaults. The reported `resolveDefaultModelRef` ordering path is only used when no explicit image-generation model is configured.

## Best Possible Solution

Close this as not reproducible against current `main`. Keep the existing explicit-config-first behavior. If someone still sees Google win with `imageGenerationModel.primary: "openai/gpt-image-2"`, the next actionable report should include the loaded config, `image_generate action=list` output, and attempt/error details to show whether OpenAI was skipped or tried first and failed before auto-fallback.
