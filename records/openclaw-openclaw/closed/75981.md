## Summary

Keep open: the startup-performance goal is valid, but the patch currently disables provider normalization hooks that can change context metadata, so it can regress context-window reporting for models such as Anthropic Opus 4.7; it also needs the required changelog entry for a user-facing fix.

## What This Changes

The PR changes the agent context-window warmup to call `discoverModels` with `normalizeModels: false` and adds comments explaining the TUI/chat startup performance motivation.

## Best Possible Solution

Add a context-window warmup path that avoids expensive per-model startup hooks while preserving provider-owned `contextWindow`/`contextTokens` semantics, with regression coverage and an `Unreleased` changelog entry.

## Reproduction Assessment

Unclear for the 30-60 second cold-start report because I did not run a live TUI with a 100+ model registry. Source inspection does verify the current warmup path invokes normalizing discovery by default, which matches the reported hot path.

## Solution Assessment

No. Passing `normalizeModels: false` is too broad unless the patch preserves context metadata changes from provider normalization or replaces them with a narrower context-only seam.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve context metadata when skipping normalization:** `src/agents/context.ts:246`
  - body: This disables every `normalizeDiscoveredPiModel` hook for the warmup path, but those hooks are not limited to compat or transport fields. For example, Anthropic's provider hook raises Opus 4.7 `contextWindow` and `contextTokens` to 1,048,576; with raw registry rows, this cache can keep the stale 200k value for model-only context lookups and warm status/session paths. Please keep a context-metadata-preserving path before turning normalization off here.
  - confidence: 0.89
- **[P3] Add the changelog entry for this user-facing fix:** `src/agents/context.ts:245-246`
  - body: This changes TUI/chat startup behavior for users with large model registries, but the PR does not update `CHANGELOG.md`. OpenClaw policy requires user-facing fix/perf changes to be listed under `Unreleased`.
  - confidence: 0.86
