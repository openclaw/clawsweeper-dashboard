## Summary

This PR should stay open: it is a plausible narrow fix for https://github.com/openclaw/openclaw/issues/78893, but the new QQBot parser can mask mismatched provider prefixes before the shared fail-closed guard runs, and the external PR still lacks real behavior proof.

## What This Changes

The PR adds QQBot target parsing and outbound normalization for cron delivery, rejects repeated `qqbot:` prefixes, and updates cron preview formatting/tests to avoid duplicated provider labels.

## Best Possible Solution

Land a narrow QQBot-owned target normalizer that preserves provider-prefix fail-closed validation, adds regression coverage and a changelog entry, and includes redacted real cron/QQBot proof before merge.

## Reproduction Assessment

Yes, at source level: current main can duplicate `qqbot:` in cron previews and can treat `qqbot:qqbot:c2c:...` as a malformed bare C2C id. I did not run a live QQBot cron delivery path in this read-only review.

## Solution Assessment

No. The overall direction is narrow, but the PR should preserve shared provider-prefix rejection before QQBot-local parsing, and it still needs real behavior proof plus the required changelog entry.
