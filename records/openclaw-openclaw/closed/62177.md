## Summary

Current main already addresses this failure mode: pairing/auth reconnect failures are treated as non-recoverable, the iOS companion explicitly stops its reconnect/watchdog loops on pairing-required errors, retry behavior is bounded, and the shipped changelog shows the pairing/reconnect guidance landed in the 2026.4.20 release line.

## Evidence

- **iOS companion disables reconnect and records pairing pause state:** Gateway problems with `pauseReconnect` disable auto-reconnect, and pairing-required problems also set `gatewayPairingPaused` plus the pending request id. Both reconnect loops check that paused state before attempting another connection.
  - file: [apps/ios/Sources/Model/NodeAppModel.swift:1894](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/apps/ios/Sources/Model/NodeAppModel.swift#L1894)
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Pairing-required errors hard-stop the reconnect/watchdog path:** On connect failure, when the mapped problem needs pairing approval, the iOS node loop cancels the operator reconnect task, disconnects both sockets, and returns early so it does not keep generating repeated pending requests while waiting for approval.
  - file: [apps/ios/Sources/Model/NodeAppModel.swift:2329](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/apps/ios/Sources/Model/NodeAppModel.swift#L2329)
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Gateway auth error mapping marks these failures as reconnect-pausing:** `GatewayConnectionProblemMapper` maps pairing-required, missing/invalid token/password, bootstrap-token invalid, and non-retryable mismatch cases to `pauseReconnect: true`, turning them into operator-action problems instead of infinite reconnect churn.
  - file: [apps/shared/OpenClawKit/Sources/OpenClawKit/GatewayConnectionProblem.swift:168](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/apps/shared/OpenClawKit/Sources/OpenClawKit/GatewayConnectionProblem.swift#L168)
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Regression tests cover paused pairing/auth reconnect behavior:** OpenClawKit tests assert that pairing-required problems preserve request metadata and set `pauseReconnect == true`, so the paused-reconnect behavior is intentional and tested.
  - file: [apps/shared/OpenClawKit/Tests/OpenClawKitTests/GatewayErrorsTests.swift:42](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/apps/shared/OpenClawKit/Tests/OpenClawKitTests/GatewayErrorsTests.swift#L42)
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Gateway client logic treats pairing/auth failures as reconnect-stopping conditions:** Current gateway client logic pauses reconnect after `PAIRING_REQUIRED`, missing/invalid auth, rate limiting, and after the one-shot trusted device-token retry budget is exhausted for token mismatch.
  - file: [src/gateway/client.ts:608](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/gateway/client.ts#L608)
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Docs and changelog show the behavior shipped:** The gateway protocol docs say clients should stop automatic reconnect loops after a failed bounded retry, and the 2026.4.20 changelog entries call out pairing/auth reconnect guidance and actionable `PAIRING_REQUIRED` surfacing in the shipped release line.
  - file: [CHANGELOG.md:479](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/CHANGELOG.md#L479)
