## What This Changes

The issue requests a session-bound IM voice handoff that moves an existing Telegram/Discord/Slack-style OpenClaw thread into a live voice room while preserving the original session context.

## Best Possible Solution

Build this as a ClawHub plugin using the existing command, HTTP route, realtime voice, TTS/STT, and session runtime APIs; open a narrower core issue only if that implementation exposes a specific missing SDK contract.
