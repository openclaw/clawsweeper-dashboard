## Summary

Current `main` has Windows-specific npm shimming, but I did not find strong enough proof that issue #37563 is fixed or obsolete. The plugin install path still reaches `runCommandWithTimeout(["npm","pack",...])`, and the current Windows branch still spawns `node.exe` plus `npm-cli.js` directly with `windowsVerbatimArguments` unset. There is no direct regression test in this checkout for `openclaw plugins install` with a spaced Node path such as `C:\Program Files\nodejs\node.exe`, so a conservative cleanup review should leave this open.
