## Summary

Keep open because current main and the latest shipped release still have the source-visible onboarding mismatch, and https://github.com/openclaw/openclaw/pull/74891 is the open canonical closing PR.

## What This Changes

`openclaw onboard` should not ask for unrelated OpenAI Whisper API or ElevenLabs keys after presenting the selected local `openai-whisper` dependency as requiring no API key.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/74891 with a narrow `setupSkills` fix that gates credential prompts to selected/configured skills while preserving required prompts for non-presented skills.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with missing local `openai-whisper` plus missing `openai-whisper-api` or `sag` credentials, `openclaw onboard` can show the keyless install option and then prompt for a separate keyed skill because the credential loop scans all missing skills.

## Solution Assessment

Yes for issue handling. Keeping this issue open until the linked canonical fix lands is the narrowest maintainable path; the code fix belongs in `setupSkills`, not in individual skill manifests.
