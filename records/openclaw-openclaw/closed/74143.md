## Summary

Keep open. Current main still routes Feishu block streaming payloads through delta append while partial snapshots use overlap-aware snapshot merging, and the open same-author PR is the appropriate fix path rather than a reason to close before merge.

## What This Changes

The issue reports duplicated Feishu streaming card content when overlapping block payloads are appended after partial snapshot replies.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/74146, or replace it with an equivalent Feishu-local dispatcher and regression-test fix, while leaving broader final/stale delivery variants on the separate tracker.

## Reproduction Assessment

Yes, at source level. Current main has a focused dispatcher path where `onPartialReply("hello")` followed by a `kind: "block"` payload of `"lo world"` still produces the duplicated `hellolo world` expectation.

## Solution Assessment

Yes. Switching only the Feishu block fallback update to the existing snapshot merge path is the narrow maintainable fix because it reuses `mergeStreamingText` and stays inside the Feishu plugin boundary.
