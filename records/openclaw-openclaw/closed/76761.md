## Summary

Close as implemented on main. The reported v2026.5.2 behavior is addressed in current main: interactive doctor now routes auth health through the external CLI overlay with keychain prompting allowed, and stale stored Claude CLI profiles are overlaid with fresher external credentials.

## What This Changes

The issue requests that `openclaw doctor` stop reporting stale `auth-profiles.json` Claude CLI OAuth expiry as expired when the live Claude CLI keychain credential is fresh.

## Best Possible Solution

Keep the current main behavior: interactive doctor should evaluate Claude CLI auth through the live external CLI overlay, while non-interactive/status paths keep the documented file-only keychain policy.

## Reproduction Assessment

No against current main: source inspection shows the interactive doctor path now passes keychain prompting through to the auth store and overlays fresh Claude CLI credentials before health evaluation. The v2026.5.2 dist path described in the report explains the released behavior, but it no longer matches current main.

## Solution Assessment

Yes. The current implementation uses the existing external CLI credential overlay and keychain policy instead of adding a new config knob or broad startup migration, which is the narrow maintainable fix for the false doctor warning.
