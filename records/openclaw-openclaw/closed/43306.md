  - confidence: 0.93
- **[P2] Deep-merge importanceBoost overrides:** `src/agents/memory-search.ts:363-368`
  - body: This chooses either the whole override importanceBoost object or the whole defaults object. A partial override such as only boostFactor can drop inherited fields like enabled or patterns, silently changing behavior.
  - confidence: 0.88
- **[P2] Reject blank importance patterns:** `src/config/zod-schema.agent-runtime.ts:722-723`
  - body: The schema accepts empty strings, but the implementation uses substring matching. Since includes("") is always true, one blank pattern makes every result important and effectively suppresses normal decay.
  - confidence: 0.9
- **[P2] Do not accept unwired periodicExtraction config:** `src/config/zod-schema.agent-defaults.ts:126-134`
  - body: This schema validates periodicExtraction, but the runtime config type and memory-flush planner do not carry or read it. Users could configure a setting that passes validation but has no effect.
  - confidence: 0.86
- **[P2] Regenerate config help artifacts:** `src/config/zod-schema.agent-runtime.ts:716-729`
  - body: Adding public config fields only to Zod leaves schema/help lookup surfaces stale; generated config artifacts still expose only the existing temporal decay keys.
  - confidence: 0.84
- **[P3] Align boostFactor validation with runtime behavior:** `src/config/zod-schema.agent-runtime.ts:719`
  - body: z.number().positive() accepts values between 0 and 1, but the implementation only applies a boost when boostFactor is greater than 1, so accepted fractional values become silent no-ops.
  - confidence: 0.84
