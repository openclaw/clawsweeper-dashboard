## Summary

Keep open: current main has explicit kill cascades and generic stale-run cleanup, but it still does not provide a parent-death lifecycle path that recursively terminates, reattaches, or marks depth-2 descendants when an intermediate subagent dies. Related open work covers stale task maintenance and false lost-context handling, but neither is a merged replacement for this parent-death descendant reconciliation.

## What This Changes

When an intermediate subagent dies mid-run, its active descendant sessions should be terminated, recovered, or made eligible for automatic cleanup instead of remaining stale running work.

## Best Possible Solution

Add focused parent-death descendant reconciliation in the subagent lifecycle layer: when a parent run becomes terminal or unsupervised, recursively cancel, reattach, or mark descendants through the existing subagent registry and task lifecycle, with depth-2 regression coverage.

## Reproduction Assessment

Yes, source-level. Current main shows explicit kill cascades for controlled kill paths, but restart/orphan recovery is marker-gated and the stale sweep is generic rather than driven by parent terminal lineage.

## Solution Assessment

No. The current generic stale/lost cleanup is not the best complete solution because it is not parent-aware and overlaps with the open false lost-context report; the maintainable fix is explicit descendant reconciliation coordinated with the existing registry/task lifecycle.
