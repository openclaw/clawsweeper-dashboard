## Summary

Keep open: current main still has a documented behavior mismatch where Codex OAuth/token-backed usage is supposed to hide dollar cost, but OpenAI Codex pricing metadata and session/transcript aggregation paths can still produce nonzero local cost estimates. The only linked fix PR is closed and unmerged.

## What This Changes

The issue asks OpenClaw to stop recording OpenAI Codex OAuth usage as authoritative per-token dollar cost, or to clearly mark/document it as non-authoritative subscription-style usage.

## Best Possible Solution

Make usage cost reporting auth/provider-aware for OpenAI Codex OAuth across transcript persistence, session summaries, and `/usage cost`, while preserving token counts and metered API-key cost estimates.

## Reproduction Assessment

Yes, there is a high-confidence source reproduction path: run a current-main `openai-codex/gpt-5.4` OAuth turn that returns usage metadata, then inspect assistant JSONL and session cost summaries. I did not run the live OAuth turn during this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is to suppress or explicitly mark non-authoritative dollar cost for Codex OAuth while retaining token accounting. A transport-only guard is likely incomplete unless session-store and usage-cost recomputation paths are covered too.
