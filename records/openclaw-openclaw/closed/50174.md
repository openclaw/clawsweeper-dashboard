## Summary

Current main addresses both observable parts of the report: Telegram polling now has bounded getUpdates request timeouts, liveness-based stall detection, HTTP/1.1-only bounded undici dispatchers, sticky IPv4 fallback, and transport disposal/rebuild on stalls; Discord gateway reconnects now set a 30s WebSocket handshake timeout so stalled TLS/network reconnects do not leave the bot silent indefinitely. The Telegram hardening is shipped in v2026.4.20 and later, while the Discord handshake timeout is on current main after v2026.4.26, so the full issue is implemented on main but not yet fully in a tagged release.

## Best Possible Solution

Close as implemented on main. Keep the current Telegram polling transport hardening and Discord gateway handshake-timeout behavior; if a user can reproduce the same Discord or Telegram instability after a release that includes 0a2d635e6849a856e1f53be6d5f3c5d6f8a8af2e, they should open a fresh, narrower report with current logs and channel status output.
