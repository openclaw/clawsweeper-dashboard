## Summary

Keep open. Current main intentionally rewrites legacy `claude-cli/*` model refs to canonical `anthropic/*` refs and records execution intent separately in `agents.defaults.agentRuntime.id: "claude-cli"`. That means the requested prefix preservation would conflict with current docs and tests, but the reporter’s user-visible concern is still valid: the Gateway `agents.list` schema and Agents UI still expose only the model, not the effective runtime, so users cannot distinguish direct Anthropic API routing from Claude CLI routing in that page.

## Best Possible Solution

Keep the issue open and fix the actionable part by surfacing the effective runtime beside the canonical model in the Agents page. The likely path is to add an additive `agentRuntime` or `runtime` field to the Gateway `agents.list` row/schema/shared type, resolve it in `listAgentsForGateway()` using the same precedence as the existing `agents_list` tool, and render it in the Agents overview/status panels with focused protocol and UI tests. Do not reintroduce `claude-cli/*` as the canonical stored model ref unless maintainers decide to reverse the documented provider/runtime design.
