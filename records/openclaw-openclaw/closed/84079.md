## Summary

Keep open: current main still normalizes target/channel aliases but not text-body aliases, so a `SendMessage`-only send reaches canonical `message` validation and fails before outbound delivery.

## What This Changes

The issue asks OpenClaw to canonicalize model-generated `SendMessage`, `content`, or `text` send bodies to `message` before validating the core `message` tool.

## Best Possible Solution

Add narrow pre-validation send-body alias normalization in the core message action path, keep canonical `message` authoritative, and add regression tests for alias recovery plus the existing no-message rejection.

## Reproduction Assessment

Yes by source inspection, though not by live Discord run: a send payload with only `SendMessage` bypasses current normalization and reaches the required canonical `message` read as missing.

## Solution Assessment

Yes: central fallback normalization before send validation is the narrow maintainable fix, provided canonical `message` remains authoritative and aliases are only used when it is absent.
