## Summary

Keep open: this focused PR still adds useful behavior on top of current main by keeping soft convergence warnings nonfatal while restoring a hard error for blocking failed plugin repairs. I found no blocking correctness finding, but the restart-gating change remains availability-sensitive and needs maintainer acceptance or optional live upgrade proof before landing.

## What This Changes

The PR updates post-core plugin convergence to treat repair warnings as visible warnings while blocking restart on scoped failed repair IDs or payload smoke failures, with regression tests.

## Best Possible Solution

Land the scoped convergence gate if maintainers accept failedPluginIds plus payload smoke failures as the hard-failure boundary, with optional live update and restart proof if they want end-to-end upgrade confidence.

## Reproduction Assessment

Yes, there is a high-confidence source-level reproduction: current main ignores repair.failedPluginIds when smoke failures are empty, and the caller maps convergence.errored directly to post-update error status. I did not run a live package-manager update in this read-only review.

## Solution Assessment

Yes, this is the best scoped fix I found: it uses the existing failedPluginIds contract and the existing active/trusted-official plugin filtering instead of parsing warning text or making all warnings fatal. The safer alternative to require a live update smoke is validation, not a different code shape.
