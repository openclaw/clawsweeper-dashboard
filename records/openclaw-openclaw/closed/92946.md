The branch adds Telegram fallback from rich send/edit/preview paths to legacy sendMessage/editMessageText with legacy chunking and focused tests.

## Best Possible Solution

Land one Telegram plugin fallback that only retries before visible rich delivery, narrowly classifies missing or unsupported rich API errors, keeps preview state aligned with visible text, defines long edit behavior, and includes current-head Telegram proof.
