## Summary

Current `main` already prevents overlapping executions of the same cron job by marking jobs as running before execution, rejecting due/manual runs while that marker is set, and recomputing the next schedule after completion so missed intervals are skipped rather than run concurrently.

## Evidence

- **Due check blocks overlapping runs:** `src/cron/service/jobs.ts:870` returns `false` from `isJobDue` whenever `job.state.runningAtMs` is set, so a still-running job is not considered runnable again.
  - file: [src/cron/service/jobs.ts:870](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/cron/service/jobs.ts#L870)
  - command: `nl -ba src/cron/service/jobs.ts | sed -n '868,882p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Scheduler persists running marker before execution:** `src/cron/service/timer.ts:707` collects due jobs, sets `job.state.runningAtMs`, and persists that state before any job body runs, which prevents a second scheduler pass from starting the same job concurrently.
  - file: [src/cron/service/timer.ts:707](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/cron/service/timer.ts#L707)
  - command: `nl -ba src/cron/service/timer.ts | sed -n '700,760p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Next run skips past overlapping interval:** `src/cron/service/jobs.ts:247` computes `every` schedules from `lastRunAtMs + everyMs` when that next slot is still in the future, otherwise falls back to the next anchored future slot. That means a long run does not spawn a second overlapping instance for the missed interval.
  - file: [src/cron/service/jobs.ts:247](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/cron/service/jobs.ts#L247)

- **Regression tests cover no-double-run behavior:** `src/cron/service/ops.regression.test.ts:30` asserts a forced manual run is skipped while a timer-triggered run is active, and `src/cron/service/ops.regression.test.ts:89` asserts overlapping `cron.run` plus a due timer tick does not double-run the job.
  - file: [src/cron/service/ops.regression.test.ts:30](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/cron/service/ops.regression.test.ts#L30)
