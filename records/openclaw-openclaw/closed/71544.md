## Summary

Current `main` already fixes the reported session-pinning bug: it now honors the configured embedded harness before applying the legacy PI history fallback, and current tests cover the exact Codex-history regression. The latest release `v2026.4.23` still had the old `"pi"` fallback logic, so this appears fixed on `main` but not yet released.

## Evidence

- **Current main resolves configured harness before PI history fallback:** `resolveSessionPinnedAgentHarnessId()` now returns `resolveConfiguredAgentHarnessId(...)` before falling back to `"pi"` for historical sessions, so `embeddedHarness.runtime: "codex"` sessions no longer get implicitly pinned to PI.
  - file: [src/agents/command/attempt-execution.ts:469](https://github.com/openclaw/openclaw/blob/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5/src/agents/command/attempt-execution.ts#L469)

  - file: [src/agents/command/attempt-execution.cli.test.ts:529](https://github.com/openclaw/openclaw/blob/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5/src/agents/command/attempt-execution.cli.test.ts#L529)
