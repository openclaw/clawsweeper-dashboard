## Summary

Keep open: current `main` still misses `cliBackends` when resolving bundled provider-policy ownership for `claude-cli`, and the linked closing PR is open rather than merged.

## What This Changes

The issue reports that `claude-cli` subagents cannot use the Anthropic max-capable thinking profile because provider-policy ownership is resolved from `providers` and `providerAuthAliases` but not `cliBackends`.

## Best Possible Solution

Land a narrow fix that treats manifest-declared `cliBackends` as provider-policy-owned refs, with the linked regression test, then close this issue through the merged PR.

## Reproduction Assessment

Yes at source level: current `main` resolves provider-policy ownership from providers/auth aliases but Anthropic owns `claude-cli` through `cliBackends`, so the Claude max-capable profile is skipped. I did not execute a live subagent run in this read-only review.

## Solution Assessment

Yes, the linked PR’s generic `cliBackends` ownership fix is the narrow maintainable direction because it uses the existing manifest contract instead of special-casing `claude-cli`. The issue should remain open until that PR lands or maintainers choose a different resolver boundary.
