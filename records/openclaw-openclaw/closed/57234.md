## Summary

Current `main` no longer leaves ordinary CDP/Playwright downloads in Playwright's ephemeral artifacts directory. It now eagerly saves unmanaged browser downloads into OpenClaw's managed downloads directory, with matching tests and docs.

## Evidence

- **CDP connections are observed and wired for download handling:** `connectBrowser` uses `chromium.connectOverCDP(...)` and immediately calls `observeBrowser(browser)`, so attach-only/existing-session CDP pages get OpenClaw's page event handlers.
  - file: [extensions/browser/src/browser/pw-session.ts:505](https://github.com/openclaw/openclaw/blob/924271385ba905c523b263f32dbcdb8ba373fc87/extensions/browser/src/browser/pw-session.ts#L505)
  - command: `nl -ba extensions/browser/src/browser/pw-session.ts | sed -n '490,525p'`
  - sha: [924271385ba9](https://github.com/openclaw/openclaw/commit/924271385ba905c523b263f32dbcdb8ba373fc87)
- **Unmanaged downloads are redirected into OpenClaw's download directory:** `ensurePageState` installs a `page.on("download")` handler that creates `DEFAULT_DOWNLOAD_DIR`, calls `download.saveAs(managedPath)`, and overrides `download.path()` to return that managed path instead of Playwright's temp artifact path.
  - file: [extensions/browser/src/browser/pw-session.ts:418](https://github.com/openclaw/openclaw/blob/924271385ba905c523b263f32dbcdb8ba373fc87/extensions/browser/src/browser/pw-session.ts#L418)
  - command: `nl -ba extensions/browser/src/browser/pw-session.ts | sed -n '410,445p'`
  - sha: [924271385ba9](https://github.com/openclaw/openclaw/commit/924271385ba905c523b263f32dbcdb8ba373fc87)
- **New pages in observed contexts also inherit the handler:** Observed browser contexts call `ensurePageState` for existing pages and on each future `page` event, so later tabs in the connected browser also use the managed download path behavior.
  - file: [extensions/browser/src/browser/pw-session.ts:458](https://github.com/openclaw/openclaw/blob/924271385ba905c523b263f32dbcdb8ba373fc87/extensions/browser/src/browser/pw-session.ts#L458)
  - command: `nl -ba extensions/browser/src/browser/pw-session.ts | sed -n '452,468p'`
  - sha: [924271385ba9](https://github.com/openclaw/openclaw/commit/924271385ba905c523b263f32dbcdb8ba373fc87)
- **Managed download root is OpenClaw-owned, not Playwright artifacts:** `DEFAULT_DOWNLOAD_DIR` resolves to `<preferred OpenClaw tmp root>/downloads`; the preferred temp root is `/tmp/openclaw` when available, with a secure fallback if needed.
  - file: [extensions/browser/src/browser/paths.ts:25](https://github.com/openclaw/openclaw/blob/924271385ba905c523b263f32dbcdb8ba373fc87/extensions/browser/src/browser/paths.ts#L25)
  - command: `nl -ba extensions/browser/src/browser/paths.ts | sed -n '20,32p'`
  - sha: [924271385ba9](https://github.com/openclaw/openclaw/commit/924271385ba905c523b263f32dbcdb8ba373fc87)
- **Tests and docs match the new behavior:** Tests assert unmanaged downloads are saved under `DEFAULT_DOWNLOAD_DIR`, and the CLI docs now state that ordinary click-triggered downloads go to the OpenClaw downloads directory.
  - file: [extensions/browser/src/browser/pw-session.test.ts:139](https://github.com/openclaw/openclaw/blob/924271385ba905c523b263f32dbcdb8ba373fc87/extensions/browser/src/browser/pw-session.test.ts#L139)

## Risks / Open Questions
