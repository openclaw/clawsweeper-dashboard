## Summary

Keep open. Current main still resolves aliases only for model strings without `/`, so a configured slash-form alias still falls through to provider/model parsing. The linked PR #73575 is an open, unmerged implementation candidate from the same author and uses closing syntax, so the issue should remain open until that PR is reviewed/landed or replaced.

## Best Possible Solution

Review and land or replace PR #73575. The code change should move alias lookup ahead of provider/model parsing in `resolveModelRefFromString()` for all inputs, then add focused regression coverage for direct resolver calls and `resolveAllowedModelRef()`/isolated cron `payload.model` with a slash-form alias.
