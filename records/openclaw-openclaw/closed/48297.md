## Summary

Current `main` already contains the CLI↔Gateway handshake handling and timeout hardening that match this report: the client consumes `connect.challenge`, signs the nonce into the `device` payload, yields an event-loop turn before starting CLI gateway requests, and uses a 10s pre-auth handshake timeout that the changelog says shipped in 2026.3.22 for busy local gateways dropping healthy CLI connections at 3s.
