## Summary

Keep this PR open. Current main still preserves config diagnostics flags when `OPENCLAW_DIAGNOSTICS` is set to a false-like value, which conflicts with the documented disable-all contract; the PR is a narrow resolver-level fix with matching tests and no blocking review finding.

## What This Changes

The PR changes diagnostics flag parsing so false-like `OPENCLAW_DIAGNOSTICS` values override config diagnostics flags, and adds resolver and timeline regression tests.

## Best Possible Solution

Land a resolver-level explicit disable override for false-like diagnostics env values, preserve blank env values as no-ops, keep normal config-plus-env union semantics for enabled flags, and keep the added regression coverage for both the resolver and timeline gate.
