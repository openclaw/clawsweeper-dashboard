## Summary

Current main already carries the multi-provider Control UI model-switch fix that matches this report: the chat dropdown now emits provider-qualified model refs instead of letting the active default provider leak into the selected model, and regression coverage exists for stale-provider and qualified-model cases. The changelog says this shipped in 2026.3.22.

## Evidence

- **Shipped fix noted in changelog:** CHANGELOG documents: "Control UI/model switching: preserve the selected provider prefix when switching models from the chat dropdown, so multi-provider setups no longer send anthropic/gpt-5.2-style mismatches when the user picked openai/gpt-5.2." This is the same failure class as switching from a MiniMax default to an Ollama model from the chat picker.
  - file: [CHANGELOG.md:2342](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/CHANGELOG.md#L2342)

  - file: [ui/src/ui/chat-model-ref.ts:266](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/ui/src/ui/chat-model-ref.ts#L266)

  - file: [ui/src/ui/chat/session-controls.ts:95](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/ui/src/ui/chat/session-controls.ts#L95)
