## Summary

Current main already treats heartbeat turns as internal system events and prevents their synthetic heartbeat origin/session from replacing the visible Main Session label in Control UI/WebChat.

## Evidence

- **System event classification:** Heartbeat, cron-event, and exec-event providers are classified as system events that must not trigger session resets or conversation-binding retargeting.
  - file: [src/auto-reply/reply/session.ts:235](https://github.com/openclaw/openclaw/blob/b1b29a8fc210a0772eb3728e03783890586e07f8/src/auto-reply/reply/session.ts#L235)
  - sha: [b1b29a8fc210](https://github.com/openclaw/openclaw/commit/b1b29a8fc210a0772eb3728e03783890586e07f8)
- **Heartbeat origin is skipped:** When applying session metadata, system events pass skipSystemEventOrigin, and the metadata helper returns no origin for heartbeat providers under that flag, preventing a synthetic 'heartbeat' origin from becoming the session label/source metadata.
  - file: [src/config/sessions/metadata.ts:54](https://github.com/openclaw/openclaw/blob/b1b29a8fc210a0772eb3728e03783890586e07f8/src/config/sessions/metadata.ts#L54)
  - sha: [b1b29a8fc210](https://github.com/openclaw/openclaw/commit/b1b29a8fc210a0772eb3728e03783890586e07f8)
- **Synthetic heartbeat leak explicitly guarded:** The session code explicitly preserves the prior route for system events so a heartbeat target or synthetic sender named 'heartbeat' does not leak into the shared session.
  - file: [src/auto-reply/reply/session.ts:538](https://github.com/openclaw/openclaw/blob/b1b29a8fc210a0772eb3728e03783890586e07f8/src/auto-reply/reply/session.ts#L538)
  - sha: [b1b29a8fc210](https://github.com/openclaw/openclaw/commit/b1b29a8fc210a0772eb3728e03783890586e07f8)
- **Control UI label is tied to selected session key:** The chat session selector derives the visible selected label from state.sessionKey and the matching session option; main session keys resolve to 'Main Session'.
  - file: [ui/src/ui/chat/session-controls.ts:28](https://github.com/openclaw/openclaw/blob/b1b29a8fc210a0772eb3728e03783890586e07f8/ui/src/ui/chat/session-controls.ts#L28)
  - sha: [b1b29a8fc210](https://github.com/openclaw/openclaw/commit/b1b29a8fc210a0772eb3728e03783890586e07f8)
- **Other-session heartbeat events do not apply to active chat UI:** Control UI chat event handling returns immediately when the event sessionKey differs from the active state.sessionKey, so an isolated ':heartbeat' run cannot mutate the Main Session chat surface.
  - file: [ui/src/ui/controllers/chat.ts:657](https://github.com/openclaw/openclaw/blob/b1b29a8fc210a0772eb3728e03783890586e07f8/ui/src/ui/controllers/chat.ts#L657)
  - sha: [b1b29a8fc210](https://github.com/openclaw/openclaw/commit/b1b29a8fc210a0772eb3728e03783890586e07f8)

  - file: [src/gateway/server-chat.ts:50](https://github.com/openclaw/openclaw/blob/b1b29a8fc210a0772eb3728e03783890586e07f8/src/gateway/server-chat.ts#L50)
