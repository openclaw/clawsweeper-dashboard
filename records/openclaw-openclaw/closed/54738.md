## Summary

Current `main` already ships the requested Claude Code integration as the bundled `claude-cli` backend: OpenClaw can route agent runs to `claude -p`, reuse Claude CLI auth, keep sessions warm, map noninteractive permission mode, and expose gateway tools through the loopback MCP bridge. The shipped config surface is `claude-cli/<model>` plus `agents.defaults.cliBackends`, not the proposed `model.backend` JSON shape.

## Risks / Open Questions

- The implemented config surface is different from the issue sketch: users configure `claude-cli/<model>` and `agents.defaults.cliBackends.claude-cli`, not `model.backend: "claude-code"`.
- CLI backends are documented as a text-first runtime rather than ACP; if the reporter wanted ACP-style persistent external coding-session semantics beyond the existing Claude CLI backend, that would need a narrower follow-up issue.
