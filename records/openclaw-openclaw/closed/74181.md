## Summary

Current main does not contain the reported lock sequence. `sessions_spawn` defaults to isolated context, so the default path does not fork or open the parent transcript; the optional `context="fork"` path rejects near-full parent sessions before attempting the fork; and the pinned Pi `SessionManager.open()` implementation reads the file synchronously without acquiring a read lock. The linked compaction-timeout PR #9855 is relevant background, but this issue’s specific parent-write-lock/child-read-lock deadlock does not match current source or the v2026.4.26 release surface.

## Best Possible Solution

Close this issue as not reproducible against current main/current release behavior. Keep the isolated default, explicit fork mode, large-parent fork guard, and compaction wait hardening. If someone can reproduce a current-build subagent timeout near compaction, it should be filed as a fresh issue with exact OpenClaw version, `sessions_spawn` parameters including `context` and `lightContext`, parent/child session keys, and logs showing where the child blocks.
