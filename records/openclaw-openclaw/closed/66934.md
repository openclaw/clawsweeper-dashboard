## Summary

Current main resolves the reported WebChat/Control UI behavior. New `chat.send` user transcript updates persist the visible user message as raw user content without a `senderLabel`, the runtime-only inbound metadata is split away from the transcript-visible prompt, and the UI falls back to the local user label (`You`) when no sender label is present. The input-field disappearance/desync half is also covered by optimistic-tail history preservation code and regression tests, with release-note evidence in 2026.4.15 and the runtime-context transcript fix shipped by v2026.4.24.

## Best Possible Solution

Close this as implemented and keep the current main behavior: WebChat should persist/display the operatorâ€™s visible user turn without leaking `openclaw-control-ui` sender metadata, while preserving optimistic local messages during stale or empty history reloads. Any future recurrence should be reopened as a fresh regression with current-version reproduction steps and a captured `chat.history` payload.
