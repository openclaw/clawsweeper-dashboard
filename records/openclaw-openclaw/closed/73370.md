## Summary

Keep open. Current main still writes LINE inbound media to the preferred OpenClaw temp directory and returns that temporary path; the LINE plugin does not call the shared media-store persistence helper. WhatsApp already uses `saveMediaBuffer(..., "inbound", ...)`, and the SDK exposes that helper, so this is a concrete LINE plugin bug with an available owner-module fix path.

## Best Possible Solution

Fix this in the LINE plugin owner module. Update `extensions/line/src/download.ts` to persist downloaded buffers through `openclaw/plugin-sdk/media-store.saveMediaBuffer(buffer, contentType, "inbound", maxBytes)` and return the saved path, cleaning up any temporary file if one remains. Update LINE download/context tests to assert managed inbound persistence and retained size/content-type behavior. No new core API appears necessary because the existing SDK helper already exposes the needed contract.
