## Summary

Current `main` already supports same-message model switching through inline `/model ...` directives and configurable `/<alias>` shortcuts in chat surfaces, which covers the requested CLI/TUI and WebChat workflow even though the exact `*XAI*` token syntax is not used.

## Risks / Open Questions

- The implementation uses slash directives/aliases such as `/model grok ...` or `/<alias> ...`, not the exact `*XAI*` token syntax proposed in the issue.
- The current code path persists inline model switches for the session; if the requester specifically wanted one-turn-only switching, that narrower behavior is not what current main implements.
