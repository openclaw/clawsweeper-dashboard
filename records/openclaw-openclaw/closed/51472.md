## Summary

This PR should close as ClawHub/plugin-owned work rather than adding a Feishu-branded core SDK subpath. Current main does not implement the requested `openclaw/plugin-sdk/feishu` export, but the documented SDK direction now steers channel-specific helpers into plugin-local barrels or generic SDK contracts, and the existing public SDK already has the primitives needed for an external Feishu plugin to build workspace-safe artifact handling without a new branded core subpath.

## Best Possible Solution

Close this core PR and keep Feishu-specific artifact handling in the Feishu plugin or a standalone ClawHub/npm plugin using generic SDK primitives. If the need proves broadly cross-channel, open a new plugin-neutral SDK proposal with root-scoped write semantics, portable filename normalization, docs, and contract tests.
