## Summary

This report is a low-detail instance of the already-open remote WebSocket token-auth scope-clearing problem tracked by #51396. Current main still clears self-declared scopes for ordinary device-less shared-token/password WebSocket operator clients, and write RPCs then fail with `missing scope: operator.write`; the canonical issue is the right place to decide whether to support that client shape or document the required device-identity/device-token flow.

## Best Possible Solution

Close this issue as a duplicate and keep #51396 as the canonical open tracker for the remote WebSocket shared-token/device-less operator-scope regression. Maintainers should resolve that canonical issue either by adding a narrow, tested gateway auth path for trusted external WebSocket clients or by documenting that remote clients must use device identity/device tokens and approved scopes.
