## Summary

Keep open: current main can already render explicit Telegram inline links through the Markdown-to-HTML send path, but it does not provide the requested automatic @username-to-ID rewrite, Telegram mention helper, or Telegram mention-alias contract.

## What This Changes

The issue asks the Telegram channel to support highlighted clickable user mentions using tg://user?id= links instead of raw @username text.

## Best Possible Solution

Add a Telegram-owned explicit mention helper or per-account alias mapping from known handles to numeric user IDs, document Telegram privacy constraints, and avoid implicit raw @username rewriting without an explicit mapping.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken documented behavior. Source inspection shows explicit inline links can already be rendered, while the requested helper or @username-to-ID mapping is absent.

## Solution Assessment

Partly: tg://user?id= inline links are the right Telegram primitive, but automatically rewriting raw @username text is not the best default. A helper or explicit alias mapping is the safer maintainable direction.
