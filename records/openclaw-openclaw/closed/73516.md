## Summary

Current main already addresses the reported crash-loop path. Package-manager updates stop the managed Gateway before replacing the package, package updates do not restart through the pre-update process, npm installs are staged and verified before swapping into the global package root, and bundled runtime deps resolve to the current versioned package root instead of scanning sibling old `openclaw-*` directories. Old runtime-deps directories may still exist as disk artifacts, but they are no longer part of the startup lookup path that would crash the Gateway.

## Best Possible Solution

Close as implemented on main. Keep the current update hardening and runtime-deps versioned-root isolation; if maintainers still want automatic pruning of old runtime-deps directories for disk hygiene, track that separately from the crash-loop bug.
