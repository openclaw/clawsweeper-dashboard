## Summary

PR #49854 targets a real remaining `NO_REPLY` preamble leak, but it is now superseded by open PR #69394, which tracks the same bug with a narrower guard that addresses the main review concern on this PR. Current main is not fixed: it still strips trailing `NO_REPLY` and delivers the remaining text, and tests still encode that behavior.

## Best Possible Solution

Close #49854 as superseded and continue the fix on #69394 or an equivalent narrow PR that suppresses only actual trailing silent-token payloads, preserves media/channelData delivery, and updates the old #30916 expectations with explicit reasoning-leak regression coverage.
