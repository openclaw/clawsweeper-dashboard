## Summary

Current `main` already contains the subagent/Discord hardening that addresses the reported symptom cluster: subagent announce calls use a configurable 120s timeout, transient gateway timeouts are retried, bare `NO_REPLY` completion chatter is suppressed, and the channel health monitor now avoids runaway restart behavior and caps restarts/hour. Those paths are already present in release `v2026.4.22`.

## Evidence

- **Subagent announce timeout is configurable and defaults to 120s:** `resolveSubagentAnnounceTimeoutMs()` reads `agents.defaults.subagents.announceTimeoutMs` and defaults to `120000`, and direct announce agent calls use that timeout.
  - file: [src/agents/subagent-announce-delivery.ts:97](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/agents/subagent-announce-delivery.ts#L97)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Transient announce delivery failures are retried:** Gateway timeout / unavailable / disconnect errors are classified as transient and retried with backoff before failing.
  - file: [src/agents/subagent-announce-delivery.ts:126](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/agents/subagent-announce-delivery.ts#L126)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Timeout behavior is covered by regression tests:** Tests verify the 120s default/configured timeout and that externally delivered completion announces retry repeated gateway timeouts before giving up.
  - file: [src/agents/subagent-announce.timeout.test.ts:281](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/agents/subagent-announce.timeout.test.ts#L281)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Bare `NO_REPLY` is intentionally suppressed:** Completion announce flow now preserves/suppresses silent `NO_REPLY` states instead of surfacing stale fallback chatter; tests cover timeout histories ending in `NO_REPLY`.
  - file: [src/agents/subagent-announce.ts:373](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/agents/subagent-announce.ts#L373)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)

  - file: [src/gateway/channel-health-monitor.ts:149](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/gateway/channel-health-monitor.ts#L149)
