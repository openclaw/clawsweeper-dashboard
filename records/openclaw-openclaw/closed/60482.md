## Summary

Current main no longer matches the reported restart repro. Gateway Tailscale startup enables only the OpenClaw port with `tailscale serve --bg --yes <port>`, and the cleanup that calls `tailscale serve reset` is only registered when `gateway.tailscale.resetOnExit` is explicitly enabled. The default restart path therefore does not clear other Tailscale Serve bindings, and the behavior is present in the v2026.4.26 release tag.

## Best Possible Solution

Close this issue as fixed for the reported default gateway restart path, keep the current opt-in reset behavior documented, and open a narrower follow-up only if maintainers want `resetOnExit` to remove only OpenClaw's own Serve/Funnel binding instead of using Tailscale's global reset.
