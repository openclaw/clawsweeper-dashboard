## What This Changes

Restore automatic compaction memory flush so the daily memory summary can be written before transcript compaction lowers or rotates the active session state.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/84792 after review confirms memory flush always runs before preflight compaction, while leaving the separate in-turn compaction and reset/sandbox policy issues to their own tracks.

## Solution Assessment

Yes, the linked fix direction is the narrow maintainable path: run memory flush before preflight compaction and cover that ordering with a regression test. Broader in-turn compaction, reset flush, and write-sandbox behavior should remain separate from this fix.
