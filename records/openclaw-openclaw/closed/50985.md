## Summary

Close #50985 as superseded by the already-open canonical tracker #50989. Current main partially suppresses phase-marked `commentary` output, but the broader requested behavior here, dropping unphased pre-tool narration in Slack/group-channel multi-agent deployments, is still not implemented and is the same remaining work tracked by #50989.

## Best Possible Solution

Close #50985 as duplicate/superseded and keep #50989 open for the actual product work: a documented per-agent or per-channel suppression mode that drops unphased text before tool calls while preserving normal final text replies, with regression coverage for Slack/group-channel delivery and unphased transports such as Anthropic.
