## Summary

Current `main` already addresses this Feishu false-positive in two places: duplicate-ID handling now suppresses warnings when both candidates resolve to the same physical plugin root, and doctor repair rewrites legacy bundled `plugins.load.paths` entries to the packaged bundled path and dedupes the rewritten list. The latest tagged release `v2026.4.22` still lacks the same-path duplicate guard, so this appears fixed on `main` but not yet shipped in a release.
