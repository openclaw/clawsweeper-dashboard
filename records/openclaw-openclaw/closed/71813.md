## Summary

Close #71813 as superseded by the broader open #65919 active-run presence issue. Current main already has the Telegram-specific short-TTL typing refresh path: Telegram passes sendChatAction("typing") into the shared reply pipeline, and the shared typing callback sends immediately plus runs a keepalive loop. The remaining concern, keeping presence leased through all long quiet phases until run completion plus dispatch idle, is already tracked in #65919 with prior related PR #66365.

## Best Possible Solution

Close this Telegram-specific item as superseded. Keep the already-shipped Telegram/shared typing keepalive path, and centralize any remaining long-quiet/run-complete presence lease work in #65919 so maintainers can fix and test it once across Telegram, Discord, Slack, and other typing-capable channels.
