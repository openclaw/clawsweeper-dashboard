## Summary

Current `main` intentionally keeps background execs unbounded when no explicit timeout is provided, and both runtime code and tests confirm that `null` is the expected value rather than a bug. PR #58997's `maxBackgroundTimeoutSec` change no longer matches `src/agents/bash-tools.exec.ts:1677` / `src/agents/bash-tools.exec-runtime.ts:623` behavior on `main`.
