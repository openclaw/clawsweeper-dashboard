## Summary

Current `main` already implements the requested configurability for the channel health monitor stale-event threshold. The gateway now accepts `gateway.channelStaleEventThresholdMinutes`, validates it against the health-check interval, passes it into `startChannelHealthMonitor`, documents it, and covers it with tests. The implementation landed after the latest tagged release, so it appears fixed on `main` but not yet shipped in `v2026.4.22`.

## Risks / Open Questions

- The shipped config key is `gateway.channelStaleEventThresholdMinutes`, not the issue's proposed nested `gateway.channelHealthMonitor.staleEventThresholdMs` shape.
- The default remains 30 minutes on current `main`; only configurability was implemented, not the suggested 2-hour default.
- `git tag --contains 38caa6832d4e1d344736d635b0c5fdad37e4430b` returned no release tag, so this appears unshipped as of latest release `v2026.4.22`.
