## Summary

Current main already supports the requested isolation through account-scoped channel routing to separate agents, where each agent has its own workspace, memory files, and session store. For `openclaw-weixin`, the supported path is one account per bot plus one agent per bot, not a shared global workspace.

## Risks / Open Questions

- This close treats the requested outcome as satisfied by the existing explicit `agent + binding` model. Current main does not appear to auto-create new agents/workspaces for each newly added WeChat account.
- The `openclaw-weixin` runtime itself is external to core, so any account onboarding UX gaps may still need to be handled in that plugin or its docs.
