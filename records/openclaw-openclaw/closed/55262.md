## Summary

Current `main` already provides this capability through the public Plugin SDK hook surface: `before_tool_call` and `after_tool_call` are defined, executed in the tool pipeline, documented for automation use, and present in the shipped `v2026.4.22` release.

## Evidence

- **Public hook names include tool-call hooks:** `before_tool_call` and `after_tool_call` are part of the public plugin hook enum, and the event/result payloads include tool name, params, result, error, and duration.
  - file: [src/plugins/hook-types.ts:57](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/plugins/hook-types.ts#L57)
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Tool-call hook payload types:** The hook event types explicitly expose the fields requested by the issue: tool name and params before execution, then params/result/error/duration after execution.
  - file: [src/plugins/hook-types.ts:318](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/plugins/hook-types.ts#L318)
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Hook runner executes both hooks:** The hook runner has dedicated `runBeforeToolCall(...)` and `runAfterToolCall(...)` paths, with `before_tool_call` able to modify/block calls and `after_tool_call` dispatched as a normal plugin hook.
  - file: [src/plugins/hooks.ts:781](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/plugins/hooks.ts#L781)
