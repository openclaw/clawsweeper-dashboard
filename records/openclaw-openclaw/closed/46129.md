## Summary

Current main already ships the fix: `cron` is a built-in core tool in the `coding` profile, the false bundled-pass "unknown entries" warning is explicitly suppressed, and the cron tool still routes `list`/`add` through the gateway with regression coverage. The same implementation is present in release `v2026.4.22`, so the 2026.3.11 report is obsolete.
