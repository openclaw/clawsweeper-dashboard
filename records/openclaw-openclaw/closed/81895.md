## Summary

Keep open: current main still lacks `config unset` dry-run support, and this active protected-label PR is the implementation path, but the latest branch still misses a narrow SecretRef fallout case for specific `secrets.defaults.<source>` unsets.

## What This Changes

This PR adds `--dry-run`, `--json`, and `--allow-exec` support to `openclaw config unset`, wires fast-route parsing, reuses shared config mutation dry-run validation, updates dry-run JSON docs/types, and adds focused CLI/router tests.

## Best Possible Solution

Land one canonical `config unset` dry-run implementation that preserves existing write behavior, returns structured JSON for every dry-run outcome, validates both broad and specific SecretRef provider/default fallout, and keeps focused tests/docs aligned.

## Reproduction Assessment

Yes. Source inspection of current main shows `config unset` has no dry-run-related options, while PR-head source shows the remaining specific `secrets.defaults.<source>` fallout gap without needing a live config write.

## Solution Assessment

No. Reusing shared config mutation dry-run validation is the right direction, but `secrets.defaults.env`/`file`/`exec` child unsets need to mark affected refs for revalidation before the patch is the narrow maintainable fix.
