## Summary

Current main already addresses the reported Bedrock Converse replay failure mode. The issue describes session-specific, zero-token Bedrock crashes after prior error turns; current code now persists stream errors with a non-empty assistant content sentinel, repairs older empty assistant error turns on disk, and normalizes empty assistant error replay before the next provider request. The fix shipped in v2026.4.26 via the related Bedrock replay fix noted in the changelog.

## Best Possible Solution

Close this as implemented by the shipped Bedrock Converse replay repair. Keep the current non-empty fallback, replay normalization, and on-disk session repair. If a current build still reports a .replace crash, it should be filed as a fresh issue with current version, sanitized logs, session shape, and whether the transcript contains empty assistant error turns; a broader stack-trace logging enhancement can be tracked separately if maintainers want that observability change.
