## Summary

The request is a valid capability, but current main’s documented scope and extension surfaces place prompt-injection scanning in the plugin/ClawHub layer rather than OpenClaw core.

## Evidence

- **Vision scope is plugin-first:** OpenClaw’s vision says core stays lean, optional capability should usually ship as plugins, and the bar for adding optional plugins to core is intentionally high.
  - file: [VISION.md:55](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/VISION.md#L55)
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Hook-only plugins are an intended pattern:** The plugin architecture explicitly supports hook-only plugins and says prompt mutation work should prefer before_prompt_build.
  - file: [docs/plugins/architecture.md:83](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/docs/plugins/architecture.md#L83)
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Per-plugin config already exists:** The configuration reference documents plugin-scoped config objects and prompt-injection hook policy under plugins.entries.<id>, which is the intended place for optional scanning configuration.
  - file: [docs/gateway/configuration-reference.md:2916](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/docs/gateway/configuration-reference.md#L2916)
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Core already exposes the needed interception points:** The agent loop documents hooks for before_prompt_build, before_tool_call, tool_result_persist, and inbound/outbound message handling, which are the relevant seams for scanning untrusted inputs and tool outputs.
  - file: [docs/concepts/agent-loop.md:94](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/docs/concepts/agent-loop.md#L94)
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Tool-result middleware already lives in plugin space:** Tokenjuice is documented as an optional plugin that transforms tool_result content after execution, showing that this class of prompt/tool-output middleware is already modeled as a plugin concern.
  - file: [docs/tools/tokenjuice.md:10](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/docs/tools/tokenjuice.md#L10)
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **No native top-level promptInjection config on main:** A repo search found prompt-injection config only in plugin hook policy (`plugins.entries.*.hooks.allowPromptInjection`), not as a core `security.promptInjection` schema key.
  - command: `rg -n 'security\.promptInjection|"promptInjection"\s*:' src/config docs`
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
