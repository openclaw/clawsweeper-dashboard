## Summary

Current `main` already has the Telegram inbound-ordering and polling-liveness fixes that cover this report: it preserves same-chat turn order while a prior turn is still active, bounds persisted Telegram update offsets by pending updates so queued inbound messages are not skipped, and prevents the polling watchdog from aborting long-running outbound API activity such as media sends.

## Evidence

- **Shipped fix for long-running Telegram reply delivery:** `CHANGELOG.md` records: "Telegram/polling: keep the watchdog from aborting long-running reply delivery by treating recent non-polling API activity as bounded liveness instead of a hard stall." under release `2026.4.5`.
  - file: [CHANGELOG.md:1462](https://github.com/openclaw/openclaw/blob/bfa6708c03f24c7caa5d0a96a298c292be0d85c1/CHANGELOG.md#L1462)
- **Current polling code treats outbound sends as liveness:** `TelegramPollingSession` marks every non-`getUpdates` Telegram API call as active liveness before the stall detector runs, which is the guard needed while media/file sends are in flight.
  - file: [extensions/telegram/src/polling-session.ts:249](https://github.com/openclaw/openclaw/blob/bfa6708c03f24c7caa5d0a96a298c292be0d85c1/extensions/telegram/src/polling-session.ts#L249)
- **Regression test covers in-flight outbound send during polling:** The Telegram polling test starts an in-flight `sendMessage`, fires the watchdog while that send is still pending, and asserts that neither the runner nor bot is restarted.
  - file: [extensions/telegram/src/polling-session.test.ts:763](https://github.com/openclaw/openclaw/blob/bfa6708c03f24c7caa5d0a96a298c292be0d85c1/extensions/telegram/src/polling-session.test.ts#L763)
- **Current bot core preserves pending inbound updates:** `bot-core.ts` tracks `pendingUpdateIds` for updates already in the middleware pipeline and persists only a safe watermark below the smallest pending update, preventing queued inbound updates from being skipped.
  - file: [extensions/telegram/src/bot-core.ts:287](https://github.com/openclaw/openclaw/blob/bfa6708c03f24c7caa5d0a96a298c292be0d85c1/extensions/telegram/src/bot-core.ts#L287)
- **Regression tests cover same-chat ordering and pending-update safety:** Telegram tests assert both same-chat reply ordering while an earlier run is still active and that update offsets are not persisted past pending updates.
  - file: [extensions/telegram/src/bot.create-telegram-bot.test.ts:382](https://github.com/openclaw/openclaw/blob/bfa6708c03f24c7caa5d0a96a298c292be0d85c1/extensions/telegram/src/bot.create-telegram-bot.test.ts#L382)
