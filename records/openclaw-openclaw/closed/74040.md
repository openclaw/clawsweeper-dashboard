## Summary

Close as implemented on main. A newer merged change replaces the old length-only prompt-content check with a centralized skip-reason helper, and current main now skips blank visible prompt submissions before provider dispatch while preserving runtime-only turns.

## What This Changes

The PR adds an inline secondary guard in `runEmbeddedAttempt` that detects blank prompts with no images and no non-empty user message, then replaces the prompt with a safe greeting before provider dispatch.

## Best Possible Solution

Keep the merged main implementation from #74168 as the canonical guard. If maintainers want `/new` or `/reset` to synthesize a greeting instead of skipping a blank submission, that should be a separate behavior change with explicit runtime-only and active-memory coverage.

## Reproduction Assessment

Yes. The linked report provides the active-memory race path, and current main has a focused runner test proving a blank prompt with replay history is skipped before provider submission.

## Solution Assessment

No for this PR branch. The current main helper is the narrower maintainable solution because it blocks empty provider dispatch centrally and preserves `runtimeOnly` semantics, while the PR’s inline greeting injection would need additional exclusions and tests.
