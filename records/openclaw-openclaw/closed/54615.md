## Summary

Current `main` already contains a targeted OpenAI Codex OAuth stale-profile repair that matches the report’s core symptom: repeated re-auth still using a stale `openai-codex:default` identity/profile. The runtime now replaces stale per-agent default Codex OAuth profiles with the newer identity-scoped main-agent profile, `openclaw doctor` can migrate legacy config references, the behavior is regression-tested, the fix is called out in the shipped `v2026.4.22` changelog, and the reporter later said reinstalling helped.
