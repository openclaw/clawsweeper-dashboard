## Summary

Keep open. Current main still has one global subagent lane plus reject-only requester active-child caps; it does not implement queued per-parent spawn admission, and the adjacent open concurrency issues cover different scheduler scopes rather than superseding this request.

## What This Changes

Request a configurable parent/requester-scoped concurrency cap that queues excess subagent spawns to reduce gateway event-loop saturation during large fan-out.

## Best Possible Solution

Add a documented requester/parent-keyed queued spawn admission layer for native and ACP spawns, preserving current defaults while defining fairness, cancellation, status, cleanup, and compatibility with adjacent concurrency controls.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this requests new scheduler and config behavior. Source inspection gives high-confidence proof that current main still lacks queued per-parent throttling, but no live 30-agent Slack reproduction was attempted.

## Solution Assessment

Unclear. A queued per-parent cap plausibly targets same-parent fan-out pressure, but maintainers need to choose the parent key, defaults, fairness model, queue UX, and interaction with related open concurrency work.
