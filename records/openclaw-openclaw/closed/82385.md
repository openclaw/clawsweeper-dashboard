## Summary

Close as implemented on current main: the Xiaomi MiMo replay path now preserves or backfills `reasoning_content` for thinking-mode assistant replay, matching the reported 400 failure, though the fix is not in v2026.5.12.

## What This Changes

The issue reports Xiaomi MiMo replay failures after tool calls because prior assistant tool-call messages omit MiMo-required `reasoning_content` in thinking mode.

## Best Possible Solution

Keep the current provider-boundary fix and ship it in the next release so MiMo tool-call follow-up turns work without requiring user config changes.

## Reproduction Assessment

Yes from source and reporter proof, though I did not run a live Xiaomi credentialed call. The reported replay shape maps directly to current tests that preserve or backfill `reasoning_content` on MiMo assistant tool-call replay.

## Solution Assessment

Yes. The current implementation fixes the provider-owned replay/stream payload boundary and keeps stock OpenAI replay stripping intact, which is narrower than adding a user-facing workaround or global replay exception.
