## Summary

Current `main` already preserves Gemini `thought_signature` values for OpenAI-compatible tool calls, re-injects them on tool-call replay, and includes regression coverage for the multi-turn case described in #58235. The same implementation is present in the shipped `v2026.4.23` release.

## Evidence

- **Stream parser captures Gemini thought signatures:** `processOpenAICompletionsStream` extracts `extra_content.google.thought_signature` from streamed OpenAI-compatible `tool_calls` and stores it on the assistant tool-call block as `thoughtSignature`.
