## Summary

Current `main` loads and activates the plugin registry before the gateway starts listening, so the PR’s claimed cold-start race in `deliverOutboundPayloadsCore()` is not supported by the present startup path.

## Evidence

- **Plugin bootstrap runs before HTTP bind:** Gateway startup awaits `prepareGatewayPluginBootstrap()` during `plugins.bootstrap`, well before the later `startListening()` call.
  - file: [src/gateway/server.impl.ts:348](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/gateway/server.impl.ts#L348)
  - command: `nl -ba src/gateway/server.impl.ts | sed -n '348,355p'; nl -ba src/gateway/server.impl.ts | sed -n '805,817p'`
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Gateway startup plugin load is eager:** `prepareGatewayPluginBootstrap()` calls `loadGatewayStartupPlugins()` for non-test startup, so gateway plugins are loaded during bootstrap rather than lazily on first delivery.
  - file: [src/gateway/server-startup-plugins.ts:91](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/gateway/server-startup-plugins.ts#L91)

- **Plugin activation initializes the global hook runner synchronously:** `loadOpenClawPlugins()` activates the registry, and `activatePluginRegistry()` immediately calls `initializeGlobalHookRunner(registry)`.
  - file: [src/plugins/loader.ts:1831](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/plugins/loader.ts#L1831)

  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Outbound delivery still reads the hook runner directly:** `deliverOutboundPayloadsCore()` still does a single `getGlobalHookRunner()` lookup, but by the startup ordering above that lookup should already see the initialized runner in normal gateway startup.
  - file: [src/infra/outbound/deliver.ts:821](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/infra/outbound/deliver.ts#L821)
