## Summary

Current `main` does not expose the exact requested `ModelProviderConfig.timeoutMs` field, but the timeout failure described in the issue is obsolete: guarded model transports now use explicit attempt timeouts when provided and otherwise inherit a 30-minute Undici stream timeout, so the reported 10-30s local-provider cold-start case no longer matches current behavior.
