## Summary

Current main already guards persisted Gemini tool results, repairs replayed tool-result names from matching tool calls, and ships regression coverage for this failure mode. The same implementation is present in release v2026.4.22.

## Evidence

- **Persisted tool results are backfilled before session write:** `normalizePersistedToolResultName` trims/repairs `toolResult.toolName`, using the pending tool-call name as fallback before persistence; `guardedAppend` passes that fallback from the pending state.
