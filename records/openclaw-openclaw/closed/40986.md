## Summary

Current main already supports the requested same-agent cross-session recall through opt-in session transcript memory indexing, session visibility scoping, and Active Memory chat-type controls. It does not use the proposed `memorySharingScope` field name, but the DM-to-group same-agent continuity problem is covered without collapsing session identities.

## Risks / Open Questions

- The implemented controls are split across memory search, session visibility, and Active Memory configuration rather than a single `memorySharingScope` key.
- Session transcript indexing is opt-in and asynchronous, so results can be stale; sandboxed sessions may also be clamped to tree visibility unless sandbox visibility is configured deliberately.
