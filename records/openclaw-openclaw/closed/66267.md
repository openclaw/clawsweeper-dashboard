This should close as superseded by the maintainer-owned i18n/localization policy thread #3460, not as implemented. Current main has a native Control UI i18n pipeline and generated locale bundles, but still contains hard-coded Control UI strings, so a standalone translate.js integration proposal should not remain as the tracking item for remaining localization work.

## Best Possible Solution
