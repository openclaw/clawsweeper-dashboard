## Summary

Keep open: current main preserves the supported `mcp.servers` shape, but the remaining `defaultModel` and `agents.list[].description` behavior is still source-reproducible and needs a maintainer config-contract decision rather than an automatic cleanup close.

## What This Changes

The issue asks `openclaw doctor --fix` to preserve user config fields such as `mcp`, root `defaultModel`, and agent descriptions instead of pruning them as unknown keys.

## Best Possible Solution

Keep strict unknown-key cleanup and supported `mcp.servers`, then explicitly add or migrate `defaultModel` and agent descriptions only if maintainers decide those are valid OpenClaw config fields.

## Reproduction Assessment

Partly: current main source gives a high-confidence path for pruning root `defaultModel` and `agents.list[].description` via strict schemas and `stripUnknownConfigKeys`. The `mcp.servers` portion is not reproducible from current source because `mcp` is recognized.

## Solution Assessment

No, not as broad unknown-key preservation. The safer solution is a maintainer-approved schema or migration change for any fields that should be valid config while keeping documented cleanup for true unknown keys.
