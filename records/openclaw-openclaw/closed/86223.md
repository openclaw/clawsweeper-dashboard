## Summary

Keep open: current main and the latest release still point `openclaw docs` at the `/mcp` MCP endpoint while the docs source redirects `/mcp` to the human MCP docs page, and the live route currently returns HTML instead of SSE.

## What This Changes

The issue reports that `openclaw docs <query>` fails because `https://docs.openclaw.ai/mcp` serves `text/html` instead of the `text/event-stream` MCP SSE endpoint expected by mcporter.

## Best Possible Solution

Keep the documented `/mcp.search_open_claw` CLI endpoint stable, free `/mcp` for the MCP SSE server, and add a focused guard so docs redirects cannot shadow the search endpoint again.

## Reproduction Assessment

Yes. Current source shows the CLI calls `https://docs.openclaw.ai/mcp.search_open_claw`, and a live header/body check shows `/mcp` returning an HTML redirect to `/cli/mcp` even when `Accept: text/event-stream` is sent.

## Solution Assessment

Yes. The narrowest maintainable fix is to restore `/mcp` as the MCP endpoint by moving or removing the human-doc redirect; changing only the CLI URL would require a new release and leave the shipped `v2026.5.22` command broken until users upgrade.
