## Summary

Keep open. Current main still has the reported UI/backend divergence: the Control UI label formats the `thinkingDefault` exposed by `sessions.list`, but the gateway session rows/defaults populate that field with provider/model fallback logic instead of the config-aware `resolveThinkingDefault` path that honors per-model `params.thinking` and `agents.defaults.thinkingDefault`.

## Best Possible Solution

Keep the issue open and fix the gateway session-list payload, not just the browser label. `getSessionDefaults` and `buildGatewaySessionRow` should use the same config-aware thinking default resolution as runtime/status for the selected session model, including per-model `agents.defaults.models[provider/model].params.thinking`, global `agents.defaults.thinkingDefault`, and per-agent defaults where applicable. Add focused regression coverage for `sessions.list` plus the Control UI selector showing `Default (adaptive)` for a custom Ollama/non-Claude model configured with per-model or global adaptive thinking.
