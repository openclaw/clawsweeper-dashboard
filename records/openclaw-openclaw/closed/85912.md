## Label Justifications

- `P1`: The report describes automatic compaction making an active Anthropic agent session unusable for every later turn.
- `impact:data-loss`: The persisted session JSONL can be left in a corrupt state where recovery requires reset or manual transcript surgery.
- `impact:message-loss`: After the failure, every user message receives only the provider rejection instead of a usable assistant reply.
- `impact:session-state`: The broken state lives in replayed session history and survives subsequent messages in the same session.

## Summary

Close as implemented on main: the merged replay fix broadens Anthropic thinking-signature recovery, preserves latest assistant thinking by default, opts compaction replay out of that preservation so bad compacted history can be stripped, and live main contains the fix. The fix is current-main-only and not in v2026.5.22.

## What This Changes

Reports that embedded auto-compaction with Anthropic thinking can persist invalid thinking-block signatures, causing every later turn in the session to fail until manual transcript surgery.

## Best Possible Solution

Keep the merged replay and compaction recovery path, then ship it in the next release so affected users no longer need manual JSONL surgery.

## Reproduction Assessment

Yes at source level: before the merged fix, the exact invalid-signature provider error was not matched by recovery and compaction did not opt out of latest-assistant preservation. I did not run a live Anthropic compaction session in this read-only review.

## Solution Assessment

Yes. The merged fix is the narrowest maintainable path: broaden the existing recovery matcher, preserve latest thinking for normal replay, and let compaction strip invalid stored thinking after appending its prompt instead of adding a new operator knob.
