## Summary

Keep open: current `main` and `v2026.5.12` still prompt for Gateway run mode before applying the model section, so an unresponsive Gateway selection blocks the documented configure/config auth path before `promptAuthConfig` can run. The direct `models auth setup-token` and `paste-token` commands are useful workarounds, but they do not fix the `--section model` wizard contract.

## What This Changes

The issue asks to stop the configure wizard from blocking Anthropic auth at the Gateway selection prompt, especially for `configure --section model`.

## Best Possible Solution

Make section-filtered model/auth setup reach `promptAuthConfig` without an unrelated Gateway run-mode dependency, while preserving Gateway setup for the full wizard and gateway-related sections.

## Reproduction Assessment

Yes, by source inspection: `configure --section model` and `config --section model` both enter `runConfigureWizard`, which awaits the Gateway selection before the model branch calls `promptAuthConfig`. The exact Docker TTY keypress failure was not live-run in this read-only pass.

## Solution Assessment

Yes. The narrowest maintainable fix is to honor section scope before Gateway run-mode selection for model-only auth, rather than adding a new bypass flag or relying on separate auth commands.
