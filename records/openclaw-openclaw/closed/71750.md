## Summary

Keep open. Current main does not implement the PR's requested `bindings[].session.dmScope` override: route bindings still have no `session` field in the config type/schema, and routing still builds session keys from only the global `cfg.session.dmScope`. The PR discussion is useful and positive, including Greptile's review saying the patch is narrow and well-tested, but the capability remains missing from main.
