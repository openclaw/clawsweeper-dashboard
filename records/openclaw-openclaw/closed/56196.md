## Summary

Keep this PR open. Current main and the latest tagged release still gate skill-command discovery on configured model aliases, so the reported `/skill` and custom workspace slash-skill behavior remains broken; the PR targets the right runtime path, but the branch is stale/unmergeable and should be refreshed before landing.

## What This Changes

This PR changes auto-reply directive and inline-action skill command discovery so slash skill commands can be loaded without configured model aliases, and adds regression coverage for an empty upstream `skillCommands` list.

## Best Possible Solution

Land a rebased current-main fix that loads workspace skill commands for authorized slash-command handling without depending on model aliases, preserves a distinct not-loaded versus loaded-empty signal to avoid duplicate scans, and adds focused regression coverage for both `/skill <name>` and custom tool-dispatch skills when `agents.defaults.models` has no aliases.

- **[P3] Avoid rescanning after an already-loaded empty command list:** `src/auto-reply/reply/get-reply-inline-actions.ts:189`
