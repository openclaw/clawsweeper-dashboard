## Summary

Keep open: current main and the latest stable release still withhold Codex app-server assistant deltas from `onPartialReply`, while Telegram partial mode is wired and documented as a live answer preview path. The adjacent interleaved-progress PR is useful context, but it intentionally leaves final answer prose out of its progress lane and does not resolve this missing assistant-delta path.

## What This Changes

This issue reports that Telegram `streaming.mode: "partial"` with `chunkMode: "newline"` no longer shows live assistant answer updates for `openai-codex/gpt-5.5`, delivering only the completed answer near the end.

## Best Possible Solution

Restore partial preview updates by emitting `onPartialReply` only for Codex app-server assistant deltas that are known to be terminal answer text, keep commentary as progress, and document any remaining provider limitation if final-answer deltas cannot be safely identified.

## Reproduction Assessment

Yes by source inspection, but not by a live run in this review. Current main has a direct path where Codex app-server assistant deltas are accumulated and explicitly not forwarded to `onPartialReply`, and the current test suite asserts that behavior.

## Solution Assessment

Yes, the narrowest maintainable fix is at the Codex app-server event bridge/projector layer because Telegram already consumes partial reply callbacks. The safer approach is phase-gated final-answer streaming plus regression tests, not a Telegram-only workaround.
