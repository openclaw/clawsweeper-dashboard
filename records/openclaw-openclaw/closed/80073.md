## Summary

Current main already fixes the false degraded event-loop health shape by requiring delay co-evidence before ELU/CPU can mark the gateway degraded; the fix is also in the published beta release v2026.5.9-beta.1, while v2026.5.7 matches the reported pre-fix behavior.

## What This Changes

The issue reports a possible false-positive gateway health degradation where event loop utilization is flagged despite zero event-loop latency and low observed CPU.

## Best Possible Solution

Keep the current main co-evidence heuristic and let the next stable release carry it forward from the published beta.

## Reproduction Assessment

No, not against current main: the classifier now returns no ELU/CPU reasons for the zero-delay sample shape. The historical v2026.5.7 behavior is well supported by this issue and the related report at https://github.com/openclaw/openclaw/issues/79017.

## Solution Assessment

Yes, the current main fix is the narrow maintainable solution: preserve the metrics and output shape, but require delay co-evidence before load counters can mark gateway health degraded.
