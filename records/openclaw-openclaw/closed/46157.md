## Summary

Current `main` already has explicit live-session model-switch reconciliation for `/model` changes during retry/fallback loops, plus queued-followup retargeting and regression tests covering the stale-provider failure mode.

Thanks for the report. Codex review against current `main` (`c1383680`) found this behavior implemented in the current session-switch path. User-driven `/model` changes now persist the new selection, clear stale runtime model state, mark `liveModelSwitchPending`, and retarget queued followups (`src/sessions/model-overrides.ts:56`, `src/auto-reply/reply/directive-handling.impl.ts:414`). Current regression coverage also exercises the restart, bounded-retry, and queued-followup cases (`src/auto-reply/reply/agent-runner-execution.test.ts:2155`, `src/auto-reply/reply/directive-handling.model.test.ts:729`, `src/agents/live-model-switch.test.ts:399`).
