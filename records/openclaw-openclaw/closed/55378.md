## Summary

#55378 is a specific Telegram forum topic / Claude Opus 4.6 reproduction of the broader open #25592 problem: ordinary assistant text emitted between tool calls can be routed to messaging channels. Current main has commentary-phase suppression, but the non-commentary/unphased path remains the same class of work tracked by #25592, so this should close as a duplicate rather than as fixed.

## Best Possible Solution

Close #55378 as a duplicate of #25592 and keep #25592 as the canonical implementation discussion for suppressing or deferring normal, unphased text emitted before or around tool calls across messaging channels. A future fix should cover the shared embedded delivery boundary and add Telegram/Anthropic regression coverage for unphased text-plus-tool-call turns.
