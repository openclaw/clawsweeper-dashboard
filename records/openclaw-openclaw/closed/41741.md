## Summary

#41741 is superseded. The PR targets the old Discord-specific direct GatewayClient path under src/discord, but current main has Discord in extensions/discord and routes approval button decisions through resolveApprovalOverGateway and the shared operator approvals client. The local self-signed TLS bug is still real, but the remaining current-path work is already tracked by #41740 and #62532.

## Best Possible Solution

Close #41741 as superseded by the current shared operator-approvals path. Continue the actual fix in #62532 or #41740 by adding TLS fingerprint resolution to src/gateway/operator-approvals-client.ts or src/gateway/client-bootstrap.ts, with tests for local loopback wss, remote mode, configured remote fingerprints, and URL overrides.
