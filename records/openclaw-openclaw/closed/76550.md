## Summary

Keep open: current main still has the doctor recovery gap. The stale-plugin repair scans and mutates allowlist, entries, dangling channel config, heartbeat targets, and channel model overrides, but it does not scan or repair `plugins.slots.*`, while an explicit missing memory slot remains a hard config validation error.

## What This Changes

The issue asks `openclaw doctor --fix` to repair or warn about stale `plugins.slots.*` references when a configured plugin is missing.

## Best Possible Solution

Extend doctor stale-plugin cleanup to handle slot selections atomically with allow/entries cleanup, using existing slot default semantics and regression coverage for the missing-plugin slot case.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes from source inspection: an explicit `plugins.slots.memory` value for a missing plugin fails config validation, and current doctor stale cleanup never scans or repairs that slot reference.

## Solution Assessment

Is this the best way to solve the issue? Yes: the narrow maintainable fix belongs in `stale-plugin-config`/doctor repair with regression tests, rather than weakening explicit slot validation or adding a new config option.
