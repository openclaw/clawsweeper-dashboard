## Summary

Current `main` already classifies HTTP 529/high-demand/capacity overload responses as `overloaded`, and existing end-to-end coverage verifies overloaded primaries fall through to configured fallback providers. The same logic is present in the shipped `v2026.4.23` release.

## Evidence

- **HTTP 529 now maps to failover reason `overloaded`:** Current main classifies HTTP 529 as `overloaded`, and raw transient HTTP text with status 529 also resolves to `overloaded`, which is the failover reason the fallback chain uses.
