## Summary

Keep open: this is a useful, source-backed fix for the nodes CLI startup regression, but the current guard changes a compatibility-sensitive plugin nested-command overlap path before maintainers have confirmed that built-in `nodes` command names are reserved.

## What This Changes

The PR adds an argv-aware guard in `registerNodesCli` and updates tests so built-in `nodes` subcommands skip plugin CLI/runtime registration while plugin-provided node commands still load lazily for non-built-in paths.

## Best Possible Solution

Land a guard that avoids full plugin runtime for normal built-in nodes commands while preserving documented `registerNodeCliFeature` nested-command behavior, or explicitly reserve built-in node names with maintainer approval and focused docs/tests.

## Reproduction Assessment

Yes. Current main and `v2026.6.10` still unconditionally enter plugin CLI registration from `registerNodesCli`, and the PR body/comment include after-fix terminal output showing built-in `nodes status` avoids plugin output while `nodes canvas --help` still loads plugin commands.

## Solution Assessment

No, not merge-ready as-is. The fix is at the right registration layer for the latency regression, but it needs either overlap-preserving behavior or a maintainer-approved reserved-name rule before it is the best solution.
