## Summary

Keep open: the PR addresses a real CLI startup regression, but the new guard can suppress existing plugin-owned nested `nodes` commands when they overlap built-in names, so it needs a compatibility-preserving change or an explicit maintainer API decision before merge.

## What This Changes

The branch adds an argv-aware guard in `registerNodesCli` plus focused tests so built-in `nodes` subcommands skip plugin CLI/runtime registration while plugin-provided non-built-in node commands still load lazily.

## Best Possible Solution

Land a guard that avoids full plugin runtime for normal built-in nodes commands while preserving documented `registerNodeCliFeature` nested-command behavior, or explicitly reserve built-in names with maintainer-approved docs and tests.

## Reproduction Assessment

Yes. Current main and `v2026.6.10` still unconditionally enter plugin CLI registration from `registerNodesCli`, and the PR body/comment include after-fix terminal output showing built-in `nodes status` avoids plugin output while `nodes canvas --help` still loads plugin commands.

## Solution Assessment

No, not merge-ready as-is. The fix is at the right registration layer for the latency regression, but it needs either overlap-preserving behavior or a maintainer-approved reserved-name rule before it is the best solution.
