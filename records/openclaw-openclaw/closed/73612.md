## Summary

Current main already resolves `voyage` as a bundled memory embedding provider through the generic memory provider capability path. The Voyage plugin declares `contracts.memoryEmbeddingProviders`, exports an adapter with id `voyage`, and the memory embedding runtime falls back to bundled capability lookup when the process-global registry is cold, which is the same class of failure covered by the related prior memory-provider reports.

## Best Possible Solution

Close as implemented on main and let the next release carry the current bundled capability-provider behavior; users still seeing it on v2026.4.26 should upgrade to a build containing current main.
