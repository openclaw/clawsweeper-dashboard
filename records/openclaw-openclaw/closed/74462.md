## What This Changes

The issue requests durable config support for `session.contextSafety` or top-level `contextSafety` so long-running Telegram/direct sessions can configure context safety and rollover behavior.

## Best Possible Solution

Track the durable context rollover design in https://github.com/openclaw/openclaw/issues/75699 and keep the current schema strict until runtime, config, metadata, docs, commands, privacy rules, and tests are added together.

## Reproduction Assessment

Yes for the schema rejection path: source inspection shows strict session and root schemas plus raw validation through `OpenClawSchema.safeParse`. No for the runtime-support premise: the named runtime and commands are absent from current source.

## Solution Assessment

No, not as a schema-only fix. Accepting `contextSafety` without the runtime rollover, handoff, authorization, Telegram/direct delivery, docs, and tests would create durable no-op configuration; the broader feature issue is the safer path.
