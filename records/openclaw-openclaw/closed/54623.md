## Summary

Current `main` already surfaces bootstrap truncation instead of leaving it silent: the runtime builds a `[Bootstrap truncation warning]` block with per-file raw vs injected counts, CLI runs prepend that warning to the prompt body, `/context` reports truncation and causes, and `openclaw doctor` warns with sizing/tuning guidance. Docs and changelog also describe the behavior.
