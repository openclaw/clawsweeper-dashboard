## Summary

Current `main` already mitigates the reported infinite `ws-stream` 500 reconnect loop by degrading the WebSocket session after failure, skipping reconnect attempts during a 60s cooldown, and falling back to HTTP/SSE; regression coverage and docs are present.

## Evidence

- **WebSocket cooldown/backoff implemented:** `openai-ws-stream` marks a session degraded for 60s (`DEFAULT_WS_DEGRADE_COOLDOWN_MS`), skips reconnects while degraded, and falls back to HTTP when connect/runtime failures happen before output.
