## Summary

Close as superseded: the reported doctor warning is real, but this doctor-only suppression is blocked by a correctness issue and the open runtime-registration PR at https://github.com/openclaw/openclaw/pull/80211 is the better canonical fix path for doctor/status.

## What This Changes

The PR suppresses the doctor missing-memory-plugin note for non-default memory slot owners, adds two doctor-memory-search tests, and adds a changelog entry.

## Best Possible Solution

Use https://github.com/openclaw/openclaw/pull/80211 as the canonical path and make memory-lancedb visible through the memory capability/runtime contract, rather than merging a doctor-only suppression.

## Reproduction Assessment

Yes, at source level. Current main warns when resolveActiveMemoryBackendConfig returns null, and memory-lancedb is documented as an active slot plugin while current main does not register the memory runtime this doctor path expects.

## Solution Assessment

No. The suppression target is valid, but this implementation treats slot text as enough proof of an active alternate memory plugin; the canonical runtime-registration approach in https://github.com/openclaw/openclaw/pull/80211 is narrower and covers doctor/status through the existing contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Require active ownership before suppressing:** `src/commands/doctor-memory-search.ts:323`
  - body: This treats a slot-only alternate memory selection as healthy because `plugins.entries[memorySlot]?.enabled` is `undefined` when the entry is missing, so `undefined !== false` returns true. With `plugins.slots.memory = "memory-lancedb"` but no valid configured entry, memory-lancedb registers only its warning service and no memory tools/hooks, and doctor would hide the missing-runtime note for an unavailable memory plugin.
  - confidence: 0.87
