## Summary

Current `main` already contains the Discord component wildcard-matching implementation the issue asks for, and the same code is present in release `v2026.4.23`: Discord registers the interaction handlers, maps both sentinel registration IDs and runtime `occomp`/`ocmodal` IDs to Carbon wildcard key `"*"`, and has a regression test covering the exact wildcard-registration behavior.
