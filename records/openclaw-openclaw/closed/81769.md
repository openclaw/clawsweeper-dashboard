## Summary

Close as implemented on current main: the CLI compaction path now initializes built-in context engines before resolving the default legacy engine, and the merged canonical PR includes regression coverage. The fix is not in v2026.5.7, so users on that release still need a post-fix build or the next release.

## What This Changes

The issue reports that CLI transcript persistence for Discord-backed claude-cli agents fails on v2026.5.7 because the default legacy context engine registry is empty.

## Best Possible Solution

Keep the current-main initializer ordering, ship it in the next release, and direct v2026.5.7 users to upgrade to a post-fix build once available.

## Reproduction Assessment

Yes for the original released bug: the resolver throws the reported default legacy unregistered error when the registry is empty, and the canonical PR body includes fresh-process before/after output. Current main no longer has a failing source path because CLI compaction initializes before resolving.

## Solution Assessment

Yes: calling the existing idempotent initializer at the CLI compaction boundary is the narrowest maintainable fix and matches adjacent context-engine paths without adding configuration or policy.
