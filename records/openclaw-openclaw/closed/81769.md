## Summary

Current main already fixes the reported CLI transcript persistence failure by initializing the built-in context-engine registry before CLI compaction resolves the default `legacy` engine; the fix is current-main-only and not present in the v2026.5.7 release snapshot cited by the report.

## What This Changes

The issue reports that CLI transcript persistence for `claude-cli` agents can warn that the default `legacy` context engine is not registered after a turn.

## Best Possible Solution

Keep the current main fix and make sure it is included in the next shipped release so v2026.5.7 users receive it by upgrading.

## Reproduction Assessment

Yes. The report’s log matches the v2026.5.7 source path, where CLI compaction resolved the default context engine without first registering built-ins; the merged fix PR also reproduced the clean-process failure before the lifecycle and successful resolution after it.

## Solution Assessment

Yes. Initializing built-in context engines immediately before CLI compaction resolves the active engine is the narrow maintainable fix and matches the existing embedded/subagent initialization contract.
