## Summary

Close as implemented_on_main. Current main, and the shipped v2026.4.22 release, already load `<workspace>/skills`, classify an override of a bundled skill like `weather` as a workspace skill instead of a built-in skill, surface that source as `Workspace Skills` in the UI, and refresh the skills list after a ClawHub install.

## Evidence

- **Workspace skills are loaded from `<workspace>/skills` with workspace precedence:** The workspace loader reads `workspaceDir/skills` as source `openclaw-workspace` and merges it last, so it overrides bundled or managed copies.
  - file: [src/agents/skills/workspace.ts:574](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/agents/skills/workspace.ts#L574)
  - command: `git show v2026.4.22:src/agents/skills/workspace.ts | sed -n '548,610p'`
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Workspace overrides are not marked as bundled:** `buildSkillStatus` only marks a skill as bundled when its source is `openclaw-bundled` or an unknown entry matches a bundled name; an explicit `openclaw-workspace` source is not treated as bundled.
  - file: [src/agents/skills-status.ts:190](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/agents/skills-status.ts#L190)
  - command: `nl -ba src/agents/skills-status.ts | sed -n '170,225p'`
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Regression test covers same-name workspace override vs bundled skill:** There is a targeted test asserting that an overridden workspace skill with the same name as a bundled skill keeps source `openclaw-workspace`, `bundled=false`, and `eligible=true`. That is the same failure mode reported for `weather`.

  - file: [ui/src/ui/views/skills-grouping.ts:10](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/ui/src/ui/views/skills-grouping.ts#L10)
