## Summary

Keep this PR open, but it is not merge-ready: the linked Feishu abort-race bug is real at source level, while the PR detaches identity recovery from the channel lifecycle and still lacks real Feishu/Lark restart proof.

## What This Changes

The PR changes Feishu monitor startup to start bot identity recovery when bot identity is missing, including when the startup lifecycle signal is already aborted.

## Best Possible Solution

Keep the Feishu-local race fix, but make recovery owned by an active account lifecycle or guarded against stale shared-state writes before merge.

## Reproduction Assessment

Yes, at source level: current main skips bot identity recovery when identity is missing and the lifecycle signal is already aborted. I did not run a live Feishu tenant restart.

## Solution Assessment

No. Scheduling recovery is the right Feishu-local area, but dropping the abort signal creates unowned retry work; a lifecycle-owned retry or stale-write guard is safer.
