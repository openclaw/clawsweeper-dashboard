## Summary

Current main already implements the reported behavior: the channel catalog now reads the persisted plugin install ledger and forwards those install records into plugin discovery, so npm-installed channel plugins can be resolved by `channels add`. The fix is on main only; I did not find a shipped release tag containing it yet.

## What This Changes

The report asks for `openclaw channels add --channel <id>` to discover channel plugins installed from npm via `openclaw plugins install`.

## Best Possible Solution

Keep the current install-record-aware catalog path and ship it in the next release; reopen only with a fresh valid install-record example if a newer build still cannot resolve an npm-installed channel plugin.

## Reproduction Assessment

Yes for the released source path, not as a current-main failure. v2026.5.4's catalog registry called discovery without install records, while current main forwards the ledger and has focused regression coverage.

## Solution Assessment

Yes. Loading the persisted install ledger in `listChannelCatalogEntries` is the narrow shared fix because `channels add` already flows through that catalog before loading the scoped channel plugin.
