## Summary

Keep open. Current main does not implement this PR’s intended change: short-term promotion still gates and scores against aggregate signalCount, and the PR is an active proposed fix for linked issue #72021. The change is small, but it conflicts with current tested/documented daily and grounded promotion semantics and already has review feedback about an avgScore numerator/denominator mismatch, so this needs maintainer review rather than conservative cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review against #72021. If maintainers want recall-only gate semantics, revise the implementation so numerator and denominator semantics stay consistent, update the daily/grounded promotion tests and CLI/docs to match the chosen product behavior, then run the targeted memory-core promotion tests. If maintainers want daily and grounded signals to remain promotion evidence, close this PR with that product decision and leave any narrower scoring recalibration to a follow-up.
