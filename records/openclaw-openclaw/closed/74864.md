## Summary

Keep open. Current main still resumes any active subagent run whose child session has `abortedLastRun: true`, and the only bounded retry path covers immediate resume-call failures rather than the accepted-then-rewedged restart-loop described in the report. The issue is concrete, recent, and complementary to the older age-bound recovery request rather than a duplicate of it.

## What This Changes

The issue asks OpenClaw to stop automatically re-resuming orphaned subagent sessions that were already wedged or repeatedly re-orphaned across gateway restarts.

## Best Possible Solution

Add a persisted subagent orphan-recovery health guard: record repeated stuck or recently recovered-again state on the child session or task run, skip automatic recovery after a small threshold, transition the run to `lost` or a clear maintenance-required state, and surface cleanup through `openclaw doctor` or task maintenance while leaving the separate age-bound recovery policy to #69778.

## Reproduction Assessment

Yes at the source level: an active subagent run with a child session entry marked `abortedLastRun: true` will be selected for synthetic resume by `recoverOrphanedSubagentSessions`. A full live reproduction still needs a controlled wedged model turn that becomes orphaned again shortly after accepted recovery.

## Solution Assessment

Yes, the report's narrow direction is the best maintainable fix: persist a recovery health or attempt signal around subagent orphan recovery instead of relying on in-process retry state. The current implementation is not sufficient because accepted resumes that later re-wedge are not bounded across gateway restarts.
