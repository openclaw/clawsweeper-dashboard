## Summary

Current main already reframes heartbeat as periodic awareness/main-session agent turns rather than uptime monitoring, and it ships a due-only/task-gated heartbeat mode plus push-driven task completion wakes that address the requested workflow gap well enough.

- There is no literal task-lifecycle toggle named 'auto-enable/disable'; the shipped shape is due-only heartbeat tasks plus push-driven task wakes. This is close enough to the request, but it is an alternative implementation.
