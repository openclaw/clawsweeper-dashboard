## Summary

Keep open. Current main still has a source-backed path where a trailing `openclaw.cache-ttl` custom row prevents embedded CLI gap-fill from seeing the canonical assistant turn, so it can persist a duplicate `api: "cli"` assistant message into the same session transcript.

## What This Changes

The issue reports duplicate assistant transcript rows after cache-ttl metadata in CLI-driven multi-turn sessions.

## Best Possible Solution

Teach embedded assistant gap-fill to dedupe across trailing cache-ttl/custom metadata while stopping at real user or assistant turns, with focused regression coverage.

## Reproduction Assessment

Yes, at source level: cache-ttl appends a custom row after the canonical embedded assistant turn, then gap-fill reads only the physical tail and appends `api: "cli"` when that tail is not an assistant. I did not execute the live multi-turn model flow in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to make embedded assistant gap-fill skip trailing metadata such as `openclaw.cache-ttl` without broadly filtering all CLI transcript rows from model snapshots.
