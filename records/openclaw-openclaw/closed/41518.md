## Summary

Current main already exposes native session lifecycle callbacks: `session_end`/`session_start` hooks, `session_end` reasons including `idle` and `compaction`, and separate compaction hooks. That covers the requested post-session callback surface, even though it ships as lifecycle hooks rather than a single `sessionEndHook` config key.

## Evidence

- **Session-end hook surface exists:** `session_end` is a first-class plugin hook, and its payload includes `reason`, `sessionFile`, and next-session linkage. The allowed reasons include `new`, `reset`, `idle`, `daily`, `compaction`, and `deleted`.
  - file: [src/plugins/hook-types.ts:391](https://github.com/openclaw/openclaw/blob/a50aef82ba326f183efef140deaa91534d11f30b/src/plugins/hook-types.ts#L391)
  - sha: [a50aef82ba32](https://github.com/openclaw/openclaw/commit/a50aef82ba326f183efef140deaa91534d11f30b)
- **Compaction emits session_end:** When compaction replaces a session, OpenClaw emits `session_end` with `reason: "compaction"` and then emits `session_start` for the replacement session.
  - file: [src/auto-reply/reply/session-updates.ts:62](https://github.com/openclaw/openclaw/blob/a50aef82ba326f183efef140deaa91534d11f30b/src/auto-reply/reply/session-updates.ts#L62)
  - sha: [a50aef82ba32](https://github.com/openclaw/openclaw/commit/a50aef82ba326f183efef140deaa91534d11f30b)
- **Idle expiry maps to session_end reason:** Session rollover logic resolves stale idle sessions to `reason: "idle"`, which is then used for the prior session's `session_end` emission.
  - file: [src/auto-reply/reply/session.ts:122](https://github.com/openclaw/openclaw/blob/a50aef82ba326f183efef140deaa91534d11f30b/src/auto-reply/reply/session.ts#L122)
  - sha: [a50aef82ba32](https://github.com/openclaw/openclaw/commit/a50aef82ba326f183efef140deaa91534d11f30b)
- **Idle path is covered by tests:** Tests explicitly assert that an idle-expired session produces a `session_end` event with `reason: "idle"`.
  - file: [src/auto-reply/reply/session-hooks-context.test.ts:243](https://github.com/openclaw/openclaw/blob/a50aef82ba326f183efef140deaa91534d11f30b/src/auto-reply/reply/session-hooks-context.test.ts#L243)
  - sha: [a50aef82ba32](https://github.com/openclaw/openclaw/commit/a50aef82ba326f183efef140deaa91534d11f30b)
- **Docs now describe the lifecycle hooks:** The agent loop docs list `before_compaction` / `after_compaction` and `session_start` / `session_end` as supported plugin lifecycle hooks.
  - file: [docs/concepts/agent-loop.md:97](https://github.com/openclaw/openclaw/blob/a50aef82ba326f183efef140deaa91534d11f30b/docs/concepts/agent-loop.md#L97)
