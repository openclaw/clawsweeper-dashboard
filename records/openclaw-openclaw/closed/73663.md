## Summary

Keep open. Current main still lacks the Bedrock Opus 4.7 temperature omission, and the Bedrock failover classifier still does not recognize the reported nested Anthropic invalid_request_error shape. Open PR #73689 is the right implementation candidate for the primary payload bug, but it has not landed and it intentionally leaves the failover-classification bug for follow-up.

## Best Possible Solution

Keep this issue open. Review and land #73689 or an equivalent amazon-bedrock provider fix that strips `inferenceConfig.temperature` only for the Bedrock Opus 4.7 family, with coverage for bare/system profile IDs and application inference profile paths. Then add a focused follow-up in the failover classifier so Bedrock `ValidationException` messages containing nested Anthropic `invalid_request_error` surface as a non-retryable schema/format failure and advance the configured fallback chain.
