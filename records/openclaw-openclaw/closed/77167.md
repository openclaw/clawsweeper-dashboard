## Summary

Current `main` and the shipped `v2026.5.3` release already route NVIDIA's vendor-prefixed Nemotron id through manifest normalization, with tests and docs covering the expected single runtime model id despite the literal `nvidia/nvidia/...` display/config form.

## What This Changes

The issue asks for cross-provider NVIDIA fallbacks to avoid producing an unusable doubled provider prefix and to provide a working fallback syntax for `nvidia/nemotron-3-super-120b-a12b`.

## Best Possible Solution

Keep the shipped manifest-owned NVIDIA normalization and documented literal ref form; if this still fails on `v2026.5.3` or later, a new report should include fresh fallback logs and the effective model catalog from that version.

## Reproduction Assessment

No; I do not have a high-confidence current-main failing path. Source inspection shows the current fallback parser and manifest normalizer resolve the NVIDIA fallback to the single runtime model id covered by regression tests.

## Solution Assessment

Yes; the best current solution is the manifest-owned provider normalization plus docs that distinguish the literal `provider/model-id` ref from the model id sent to NVIDIA, without adding a core NVIDIA special case.
