## Summary

Close as duplicate/superseded by open #12831. Both reports ask to separate tool/runtime activity from the main chat conversation; current main still renders tool activity inside the single Control UI chat thread, so the remaining work should stay consolidated in the broader canonical tracker.

## Best Possible Solution

Close this duplicate and keep #12831 as the canonical product/design tracker for separating conversation from tool/runtime activity. When implemented, the Control UI should route tool activity into an independently scrolling, collapsible activity surface while preserving the main communication thread's scroll and selection behavior.
