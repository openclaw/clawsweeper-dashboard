## Summary

Current `main` already supports ACP bindings on plugin channels. The old Discord/Telegram-only restriction is gone from config/type validation, ACP binding resolution now goes through the generic configured-binding registry, and multiple plugin channels on `main` wire that binding interface and route inbound conversations through it.

## Evidence

- **No Discord/Telegram-only ACP type or validator remains:** `ConfiguredAcpBindingChannel` is now `ChannelId`, and the ACP binding schema only requires a concrete `match.peer.id`; it no longer whitelists `discord`/`telegram`.

## Risks / Open Questions

- Support is plugin-provided rather than automatic for every possible channel id: a plugin channel must expose the configured-binding adapter for ACP bindings to work.
