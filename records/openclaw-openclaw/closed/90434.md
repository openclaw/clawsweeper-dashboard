## Summary

This is a valid Copilot provider regression, but it is the same `tools[].eager_input_streaming` failure already tracked by the older open canonical issue and an open fix PR that closes that canonical thread.

## What This Changes

Reports that GitHub Copilot Claude models fail on OpenClaw 2026.6.1 because tool-enabled Anthropic payloads include `eager_input_streaming`, which the Copilot proxy rejects.

## Best Possible Solution

Keep the older canonical issue open and review, land, or replace its open fix PR so Copilot Anthropic model definitions consistently set the eager-streaming compat flag with regression coverage.

## Reproduction Assessment

Yes. The issue body gives concrete v2026.6.1 logs, and current source shows Copilot Claude definitions still omit the compat opt-out while the Anthropic transport adds `eager_input_streaming` by default.

## Solution Assessment

No separate tracking item is needed. The maintainable fix belongs in the existing Copilot provider/model compat path tracked by https://github.com/openclaw/openclaw/issues/75348 and its open fix candidate.
