## Summary

Current main still computes the cron direct-delivery idempotency key before the local `try/catch`, but supported cron and session inputs are normalized to strings-or-undefined before they can reach that path. I could not find a supported runtime path on `main` where non-string `channel` or `to` values reach `normalizeDeliveryTarget()`.

## Evidence

- **Cron job input normalization strips non-string delivery fields before runtime:** `normalizeCronJobInput()` runs `coerceDelivery()`, which only keeps `mode`, `channel`, `to`, `threadId`, and `accountId` when `parseDeliveryInput()` accepts them. `parseDeliveryInput()` accepts `channel`, `to`, and `accountId` only as trimmed strings, so malformed persisted/API job data is dropped during normalization.
  - file: [src/cron/normalize.ts:260](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/cron/normalize.ts#L260)
  - command: `nl -ba src/cron/normalize.ts | sed -n '260,288p'`
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Persisted cron jobs are normalized on load:** `ensureLoaded()` reloads jobs from disk and immediately passes each raw job through `normalizeCronJobInput()` before using it in memory, so malformed persisted `delivery.channel`/`delivery.to` values do not flow unchanged into cron execution.
  - file: [src/cron/service/store.ts:37](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/cron/service/store.ts#L37)
  - command: `nl -ba src/cron/service/store.ts | sed -n '37,57p'`
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Session-derived delivery targets are normalized before resolution:** Session store loading normalizes `lastChannel`/`lastTo`/`lastAccountId` via `normalizeSessionDeliveryFields()`, and `normalizeDeliveryContext()` only retains string `channel`/`to` values. `resolveSessionDeliveryTarget()` then reads from that normalized context.
  - file: [src/config/sessions/store-load.ts:31](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/config/sessions/store-load.ts#L31)
  - command: `nl -ba src/config/sessions/store-load.ts | sed -n '31,61p'`
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **The reported throw site is still present but depends on already-normalized inputs:** `normalizeDeliveryTarget()` still calls `to.trim()`, and `buildDirectCronDeliveryIdempotencyKey()` is still outside the local `try/catch`, but the surrounding code only consumes `SuccessfulDeliveryTarget` values produced from the normalized cron/session paths above.
  - file: [src/cron/isolated-agent/delivery-dispatch.ts:34](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/cron/isolated-agent/delivery-dispatch.ts#L34)
