## Summary

Current main already implements WhatsApp env-proxy support for the Baileys socket path and its upload fetch path, and the same behavior is present in the latest shipped release `v2026.4.22`. The issue's requested behavior is therefore already implemented.

## Evidence

- **WhatsApp socket now wires env proxy agents into Baileys:** `createWaSocket()` resolves an ambient Node proxy agent from standard env vars, resolves an undici proxy dispatcher for fetch uploads, and passes both into `makeWASocket` as `agent` and `fetchAgent`. That addresses the reported direct-DNS WebSocket path.
