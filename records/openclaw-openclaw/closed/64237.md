## Summary

Current `main` already implements the requested runtime safety: oversized tool results are capped before transcript persistence, the live request path truncates oversized tool results before the next model call, and `exec` output is separately bounded.

## Evidence

- **Pre-persistence tool-result cap:** `installSessionToolResultGuard()` truncates tool-result messages and explicitly applies a hard size cap before persistence: the write path comments 'Apply hard size cap before persistence' and caps both normal and synthetic tool results before `originalAppend(...)`.
  - file: [src/agents/session-tool-result-guard.ts:201](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/agents/session-tool-result-guard.ts#L201)
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **Guard wired into embedded agent sessions:** Embedded Pi attempts open the session through `guardSessionManager(...)`, so live agent transcripts go through the persistence guard. The wrapper resolves `maxToolResultChars` from `agents.*.contextLimits.toolResultMaxChars`.
  - file: [src/agents/pi-embedded-runner/run/attempt.ts:1076](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/agents/pi-embedded-runner/run/attempt.ts#L1076)
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **Same-turn pre-prompt protection:** `installToolResultContextGuard()` wraps `transformContext`, truncates oversized tool results before the next model call, and throws a preemptive overflow error if the assembled context still exceeds the safety threshold.
  - file: [src/agents/pi-embedded-runner/tool-result-context-guard.ts:316](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/agents/pi-embedded-runner/tool-result-context-guard.ts#L316)
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **Regression coverage for persistence-stage expansion:** Tests verify the cap is re-applied even if `tool_result_persist` or `before_message_write` hooks expand a tool result after initial truncation, which is the exact pre-session-write safety boundary the issue asks for.
  - file: [src/agents/session-tool-result-guard.tool-result-persist-hook.test.ts:167](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/agents/session-tool-result-guard.tool-result-persist-hook.test.ts#L167)
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **Exec output is not unbounded:** The exec runtime already enforces a native output ceiling via `DEFAULT_MAX_OUTPUT`, clamped to 200,000 chars from `PI_BASH_MAX_OUTPUT_CHARS`.
  - file: [src/agents/bash-tools.exec-runtime.ts:106](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/agents/bash-tools.exec-runtime.ts#L106)
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **User-facing config/docs describe persisted-result cap:** Agent config docs describe `toolResultMaxChars` as a live tool-result cap used for persisted results and overflow recovery, matching the implemented generic session-write cap.
  - file: [docs/gateway/config-agents.md:187](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/docs/gateway/config-agents.md#L187)
