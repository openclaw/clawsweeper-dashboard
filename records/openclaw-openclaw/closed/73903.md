## Summary

Current main still has the reported behavior: the Claude CLI doctor note is global, detects only defaults-level Claude CLI/config/auth signals, and resolves workspace/project-dir health through the default agent instead of agents whose effective runtime is `claude-cli`. The issue is valid, focused, and suitable for one fix PR.

## Best Possible Solution

Keep this issue open and fix the Claude CLI doctor note to be runtime-policy aware: detect agents whose effective runtime is `claude-cli`, skip the section when no Claude CLI runtime or legacy/default signal applies, and report workspace/project-dir health for the relevant agent or agents instead of always using the default agent. Add focused tests for a default Codex/OpenAI agent plus a non-default Claude CLI agent, while preserving existing legacy defaults-model behavior.

Add focused tests in `src/commands/doctor-claude-cli.test.ts` for default OpenAI/Codex agent plus non-default Claude CLI agent and for legacy defaults-level behavior.
