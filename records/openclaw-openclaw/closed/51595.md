## Summary

Current `main` no longer routes `sessions_send` through the single bare `nested` lane targeted by this PR. The live paths now use per-session nested lanes (`nested:<sessionKey>`), so the reported cross-agent serialization no longer matches current behavior, and the proposed `setCommandLaneConcurrency(CommandLane.Nested, ...)` change would not affect those paths.

## Evidence

- **`sessions_send` now uses a per-session nested lane:** The tool starts agent runs with `lane: resolveNestedAgentLaneForSession(resolvedKey)`, so sends to different target sessions do not share one global `nested` lane.
  - file: [src/agents/tools/sessions-send-tool.ts:281](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/src/agents/tools/sessions-send-tool.ts#L281)

  - file: [src/agents/tools/sessions-send-tool.a2a.ts:95](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/src/agents/tools/sessions-send-tool.a2a.ts#L95)
