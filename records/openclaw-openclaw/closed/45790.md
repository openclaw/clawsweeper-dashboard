## Summary

Current main already retries ordinary WhatsApp disconnects instead of treating them as fatal relink events. Inbound close handling only stops for logged-out sessions or explicit non-retryable conflict status 440; an ENOTFOUND-style disconnect with no logged-out status falls into the reconnect/backoff path. The latest release tag `v2026.4.22` also contains this behavior and the related reconnect tests.

## Evidence
