## Summary

Issue #54501 asks OpenClaw core for agent-scoped configuration or multiple instances of the external `modelstudio-memory-for-openclaw` memory plugin. Current main intentionally keeps memory as a single active plugin slot, while the existing plugin API already passes `agentId` into plugin hooks and memory-runtime calls, so per-agent isolation belongs in the plugin implementation rather than new core support.

## Evidence

- **Vision keeps optional capability in plugins and memory in one slot:** The project vision says core should stay lean, optional capability should usually ship as plugins maintained in their own repository, and memory is a special plugin slot where only one memory plugin can be active at a time.
  - file: [VISION.md:55](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/VISION.md#L55)

- **Core config only models one active memory plugin:** `PluginSlotsConfig.memory` is a single string selector, not a per-agent or multi-instance map.
  - file: [src/config/types.plugins.ts:24](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/config/types.plugins.ts#L24)

- **Memory runtime contract is agent-scoped already:** The active memory runtime interface requires `agentId` for `getMemorySearchManager` and `resolveMemoryBackendConfig`, which is the exact seam a memory plugin would use for per-agent isolation.
  - file: [src/plugins/memory-state.ts:96](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/plugins/memory-state.ts#L96)

  - file: [extensions/memory-core/src/memory/manager.ts:160](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/memory-core/src/memory/manager.ts#L160)

## Risks / Open Questions

- The external `modelstudio-memory-for-openclaw` plugin still needs to implement agent-derived namespacing itself; closing this core issue does not add that behavior automatically.
- If maintainers later choose to redesign core plugin instancing, this conclusion could change, but current main and the documented vision do not support that direction.
