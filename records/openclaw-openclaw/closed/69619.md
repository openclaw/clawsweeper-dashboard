## Summary

Current main already supports the Docker/headless-node workflow without requiring systemd: run the node host in the foreground with `openclaw node run` under Docker/Compose, then approve its device pairing through `openclaw devices list` and `openclaw devices approve <requestId>`. `openclaw node install` remains intentionally scoped to host service managers, so Docker should provide process supervision instead.

## Best Possible Solution

Keep the shipped path: Docker or Compose should supervise a foreground `openclaw node run` process, and pairing should be completed through `openclaw devices` or the explicit trusted-CIDR auto-approval policy. A future docs-only sample for a node-host Compose service could be opened separately if maintainers want a turnkey example.
