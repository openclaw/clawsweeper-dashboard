## Summary

Keep open: the linked display/session-state bug is valid, but this PR changes the shared fresh-token helper in a way that would feed stale totals to runtime callers that intentionally require fresh values.

## What This Changes

The PR removes the stale-freshness guard from `resolveFreshSessionTotalTokens`, updates `isSessionTotalTokensFresh`, and rewrites part of the active changelog.

## Best Possible Solution

Split preserved display totals from fresh-only runtime totals, then update `/context`, session-list/status, and Control UI display consumers with regression tests that keep compaction, memory, fork, and compact paths fresh-only.

## Reproduction Assessment

Yes, at source level: a finite `totalTokens` plus `totalTokensFresh: false` is filtered to `undefined` for display callers on current main. I did not run a live patched UI or `/context` session in this read-only review.

## Solution Assessment

No. The desired display behavior is narrow and valid, but changing the shared fresh-only helper globally is not the safest fix because runtime callers still depend on stale totals forcing transcript fallback.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Keep fresh-only callers from using stale token totals:** `src/config/sessions/types.ts:553-556`
  - body: Removing the freshness guard makes `resolveFreshSessionTotalTokens` return stale preserved values to runtime callers. `resolveParentForkTokenCountRuntime` exits early on that helper, so a stale parent row like the existing fixture with `totalTokens: 1` and `totalTokensFresh: false` would return `1` instead of the transcript-derived count; preflight compaction similarly skips transcript estimation when the helper returns a number. Keep this helper fresh-only and add a separate preserved/display-total path for UI and command display callers.
  - confidence: 0.94

  - body: This patch deletes most of the active `### Fixes` section while adding the new session-token entry. Merging it as-is would drop unrelated release notes, so the branch should keep the existing entries and add only the intended line.
  - confidence: 0.91
