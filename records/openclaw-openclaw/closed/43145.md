## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Authorize timeline reads before returning spans:** `src/gateway/server-methods/agent.ts:715-724`
  - body: The new handler looks up a caller-supplied `runId` and returns `sessionKey` plus raw trace spans without checking that the requesting operator owns or is allowed to inspect that run. Any read-scoped paired operator that learns a run id could read another run's trace details and error text.
  - confidence: 0.9
- **[P1] Classify run errors before critic replan returns:** `src/cron/isolated-agent/run.ts:797-805`
  - body: The `needs_replan` early return runs before `hasFatalErrorPayload` and `runLevelError` are computed. With the critic loop enabled, an embedded run that only produced error payloads can be reported as `status: "ok"`, suppressing normal failure handling and downstream error signals.
  - confidence: 0.86
- **[P1] Block dynamic Python imports in sandbox validation:** `src/agents/sandbox/execution-policy.ts:170-171`
  - body: The validator only checks modules returned by static import parsing. Code such as `__import__("subprocess")` can bypass the sandbox execution deny lists, so the new policy can approve generated Python that imports blocked modules at runtime.
  - confidence: 0.88
- **[P2] Strip inline comments from parsed import segments:** `src/agents/sandbox/execution-policy.ts:54-59`
  - body: `parseImportSegment` removes aliases but not trailing comments, so valid Python like `import math  # comment` is classified with the comment in the module root and can be rejected as an unallowlisted dependency. Strip inline comments before classification.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The diff introduces concrete security-sensitive trace and sandbox surfaces that are not safe to merge as written.

- **[high] Unauthorized trace disclosure:** `src/gateway/server-methods/agent.ts:715`

  - body: The branch adds a process-global timeline map with terminal count pruning but no hard cap for running timelines or spans per timeline, allowing high memory growth during the running retention window.
  - confidence: 0.78
