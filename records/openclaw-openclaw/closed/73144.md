## Summary

Keep open. Current main has partial documentation and model catalog improvements, but the reported model-switch workflow still has unresolved provider-write policy, allowlist/diagnostic UX, managed-service restart expectations, and a stale-session subcase tracked by an open issue and PR.

## What This Changes

The issue requests a smoother model-provider switch workflow covering provider creation, dual config/models.json diagnostics, allowlist handling, gateway restart UX, and stale active-session model selection.

## Best Possible Solution

Adopt an owner-approved model setup path that separates explicit provider/auth registration from agent-facing config.patch, adds model status or doctor diagnostics for provider and allowlist mismatches, documents managed-restart expectations, and lands https://github.com/openclaw/openclaw/pull/77339 or an equivalent narrow stale-session fix.

## Reproduction Assessment

No, not for the whole five-part workflow as reported. Several subpaths are source-reproducible on current main, and the stale-session subcase has a clearer reproduction path in https://github.com/openclaw/openclaw/issues/77322.

## Solution Assessment

Unclear for the broad requested direction. Broad provider writes through config.patch and automatic allowlist/session remapping are not clearly the safest solution; explicit setup, targeted diagnostics, and the narrow stale-session fix are more maintainable.
