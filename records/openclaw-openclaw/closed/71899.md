## What This Changes

The branch changes agent workspace resolution so setup-created main workspace defaults fall back to peer-level non-default workspaces, adds resolver regression tests, updates multi-agent docs, and adds a changelog entry.

## Best Possible Solution

Keep one canonical implementation in https://github.com/openclaw/openclaw/pull/71941 or an equivalent narrow wizard-prefill fix that preserves explicit workspaces and the shared-base resolver contract.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure agents.defaults.workspace as <stateDir>/workspace-dev from a profiled setup, then resolve a new non-default agent without that profile; PR head still falls through to <stateDir>/workspace-dev/<agentId>.

## Solution Assessment

No. This branch improves the common default case, but the runtime guard is incomplete across profile transitions and broader than the safer wizard-prefill boundary used by the canonical PR.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Recognize profiled setup workspaces before nesting:** `src/agents/agent-scope-config.ts:72-73`
  - body: The guard only recognizes `<stateDir>/workspace` and the current profile's default workspace. If setup persisted `<stateDir>/workspace-dev` under `OPENCLAW_PROFILE=dev` and the wizard later runs without that profile, this check returns false and the resolver still produces `<stateDir>/workspace-dev/<agentId>`, preserving the nested layout the PR is meant to prevent.
  - confidence: 0.84
