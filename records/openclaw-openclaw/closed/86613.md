## Summary

Keep open: the deterministic follow-up now points to a current memory_search gateway path, and current main still constructs a long-lived chokidar watcher for memory roots when memory sync watching is enabled. The earlier watcher polling fix removed awaitWriteFinish pressure, but it does not address this retained watcher fan-out/default behavior.

## What This Changes

The issue reports that an authorized gateway memory_search call can retain thousands of read-only file descriptors on workspace memory Markdown files.

## Best Possible Solution

Keep this open for a compatibility-safe memory-core/gateway watcher policy and add a deterministic large-tree /tools/invoke regression that proves FDs are released or never fan out.

Yes. The issue now includes a deterministic clean-main curl repro through authorized POST /tools/invoke memory_search against a 12,391-file memory tree, and source inspection confirms that route constructs the default memory manager and watcher; this review did not rerun the large-file live repro.

## Solution Assessment

Unclear. The previous awaitWriteFinish fix is not sufficient, and the better fix likely needs a maintainer policy choice between watcher default, cap, and lifecycle changes because each affects live memory freshness.
