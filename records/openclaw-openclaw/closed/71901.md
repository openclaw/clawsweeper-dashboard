## Summary

Current main already implements the ACP completion-announce behavior requested by PR #71901: ACP-ready sessions receive plain completion prompts, OpenClaw internal runtime-context envelopes are stripped from ACP prompt/transcript text, and reply instructions are preserved. The linked #71811 user-facing bug is therefore solved on main, making this PR obsolete.

## Best Possible Solution

Close PR #71901 as obsolete because current main already has the ACP completion-announce fix with source, tests, docs, and changelog coverage. Keep the main implementation; if desired, track the PR's extra untrusted-child-result delimiter escaping as a separate narrow hardening issue or PR.
