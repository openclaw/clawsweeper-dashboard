## Summary

Keep this PR open: it addresses a real CI/i18n failure path, but the patch introduces a security-sensitive regression by moving cross-repo publish tokens into job-wide environment variables, and the branch is currently dirty/unmergeable.

## What This Changes

The PR updates scheduled live-check permissions, skips docs/website publish sync when cross-repo tokens are missing, and moves two Control UI Quick Settings labels into i18n locale outputs.

## Best Possible Solution

Keep the useful CI permission and i18n changes, but use non-secret presence flags for skip conditions and pass actual publish tokens only to the checkout/clone/push steps that need them.

## Reproduction Assessment

Yes, by source inspection: the scheduled caller does not grant actions: read while the reusable Docker prep job requests it, and the publish workflows currently assume cross-repo tokens are present. I did not run the expensive scheduled live workflow.

## Solution Assessment

No. The actions: read and i18n parts are directionally correct, but job-scoping the real publish tokens is not the narrowest maintainable fix; use a non-secret boolean for gating and step-scoped secrets for token use.
