## Summary

Keep open: the PR addresses a real CI permission mismatch and useful i18n/type drift, but the publish-token skip implementation exposes cross-repo tokens job-wide and the supplied proof does not show the changed GitHub Actions behavior.

## What This Changes

Adds scheduled live workflow `actions: read`, skips docs/website publish sync when cross-repo tokens are absent, moves two Quick Settings labels into i18n, aligns Telegram grammY type imports, and adjusts related tests/type config.

## Best Possible Solution

Keep the permission, i18n, and type-import fixes, but gate skip logic with non-secret presence flags and scope publish tokens only to checkout/clone/push steps.

## Reproduction Assessment

Yes, by source inspection: the scheduled caller lacks `actions: read` while the reusable Docker image job requests it, and the publish workflows currently assume cross-repo secrets are present. I did not run the expensive scheduled workflow.

## Solution Assessment

No. The workflow permission and i18n/type changes are directionally correct, but job-scoping the real publish tokens is not the narrowest maintainable fix.
