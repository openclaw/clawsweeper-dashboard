## Summary

This is a valid source-backed bug: the portable presentation contract already includes `disabled`, but current `main` does not expose it in the message tool schema and Discord does not preserve it through capability adaptation or presentation component mapping.

## What This Changes

The issue requests preserving `disabled: true` on portable presentation buttons through the message tool schema and Discord component rendering.

## Best Possible Solution

Land a narrow fix that treats `disabled` as an existing presentation-button field, advertises Discord disabled-button support, preserves it in Discord component specs, and adds focused regression tests.

## Reproduction Assessment

Yes, at source level: a `presentation.blocks[].buttons[]` item with `disabled: true` is documented and typed, but the tool schema omits the field and Discord adaptation/mapping will not preserve it. I did not run a live Discord send because this review is read-only.

## Solution Assessment
