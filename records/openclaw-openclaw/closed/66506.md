## Summary

Current `main` already routes `openclaw infer audio transcribe` through the OpenAI batch transcription path that parses `payload.text`, and the same implementation is present in the released `v2026.4.23` tree with regression coverage for both transcript extraction and missing-text failures.

## Evidence

- **OpenAI plugin registers batch audio transcription:** The bundled `openai` plugin declares `audio` capability and wires `transcribeAudio` to `transcribeOpenAiAudio`, so OpenAI STT runs through the media-understanding audio path instead of a separate realtime-only path.
  - file: [extensions/openai/media-understanding-provider.ts:21](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/extensions/openai/media-understanding-provider.ts#L21)
  - command: `nl -ba extensions/openai/media-understanding-provider.ts | sed -n '1,80p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **OpenAI transcription parser reads `payload.text`:** `transcribeOpenAiCompatibleAudio(...)` posts to `/audio/transcriptions`, parses `await res.json()` as `{ text?: string }`, and returns the transcript text after `requireTranscriptionText(...)` validation.
  - file: [src/media-understanding/openai-compatible-audio.ts:39](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/media-understanding/openai-compatible-audio.ts#L39)
  - command: `nl -ba src/media-understanding/openai-compatible-audio.ts | sed -n '1,120p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Infer CLI uses transcription runtime:** `runAudioTranscribe(...)` in the `infer`/`capability` CLI calls `transcribeAudioFile(...)` and only throws `No transcript returned for audio` when the returned `text` is empty, so a successful provider transcript is not intentionally discarded on current main.
  - file: [src/cli/capability-cli.ts:796](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/cli/capability-cli.ts#L796)
  - command: `nl -ba src/cli/capability-cli.ts | sed -n '796,820p'`

- **Provider regression test covers successful `{text: ...}` extraction:** The OpenAI provider test expects a mocked OpenAI-style JSON response `{ text: "hello" }` to come back as `result.text === "hello"`, matching the issue's reported successful provider payload shape.
  - file: [extensions/openai/media-understanding-provider.test.ts:28](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/extensions/openai/media-understanding-provider.test.ts#L28)
