## Summary

Keep open: the fix is focused and now has adequate after-fix gateway proof, but the new resolved-error dedupe entry does not include an ErrorShape, so idempotent cached replays can still surface only `unknown error` to gateway clients.

## What This Changes

The PR updates gateway chat completion handling to broadcast terminal `state: "error"` and cache an error status when delivered agent replies resolve with `isError: true`, with regression tests for single/multiple payloads and transcript ordering.

## Best Possible Solution

Land this gateway fix after caching a real ErrorShape for delivered-error dedupe replays and covering both the live broadcast and cached replay paths.

## Reproduction Assessment

Yes by source inspection. Current main collects delivered replies, then the `agentRunStarted` branch only emits the transcript update and writes an ok chat dedupe entry; the linked issue includes matching JSONL prompt-error evidence, though this read-only review did not execute tests.

## Solution Assessment

Mostly yes. The gateway post-dispatch branch is the right ownership boundary and the tests cover the main delivery path, but the cached replay path needs the same ErrorShape semantics as thrown errors before this is the best fix.
