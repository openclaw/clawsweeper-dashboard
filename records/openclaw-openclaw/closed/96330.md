## Label Justifications

- `P2`: A missing WebChat approval prompt would block a normal Skill Workshop workflow but is bounded to one UI/channel path.
- `impact:message-loss`: The report is about an approval request event not being surfaced to the WebChat user, causing the visible prompt to be lost.

## Summary

Current main and released source already implement the reported WebChat plugin approval prompt path: Control UI receives `plugin.approval.requested`, parses it as a plugin approval, renders the shared approval modal, and resolves through `plugin.approval.resolve`.

## What This Changes

The issue reports that WebChat does not surface Skill Workshop approval prompts, causing `skill_workshop(action=apply)` to time out under the default pending approval policy.

## Best Possible Solution

Keep the shipped Control UI plugin approval modal path; if a packaged Windows build still lacks it, track that as a packaging/version-specific regression with the exact version and asset hash.

## Reproduction Assessment

No for current main: the reported missing WebChat plugin-approval UI path is present in source and covered by focused UI tests. The report can be understood from older or mismatched built artifacts, but the current checkout no longer has the missing-handler failure path.

## Solution Assessment

Yes. Reusing the existing Control UI approval queue and `plugin.approval.resolve` method is the narrow maintainable solution, and current source/tests show that is already the implementation.
