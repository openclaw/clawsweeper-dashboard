Keep open: the report matches a credible runtime dependency-boundary failure, because a generated facade runtime imports `json5` while the plugin/package dependency surfaces I inspected do not clearly prove that isolated runtime chunks always resolve it. Current source also treats `plugin-runtime-deps` as legacy cleanup, so the exact remaining fix point needs packaged current-main confirmation before closing.

## What This Changes

The issue reports that `memory_search` can fail when an isolated generated plugin runtime dependency sandbox omits `json5` even though generated runtime files import it.

## Best Possible Solution

Make packaged/bundled runtime dependency resolution guarantee `json5` is available to generated runtime chunks used by `memory_search`, or remove the stale `plugin-runtime-deps` path from that flow, with package-level regression coverage.

## Reproduction Assessment

No. The reporter supplied a concrete released-version failure and source inspection explains how a missing `json5` package would break the generated runtime import, but I did not establish a failing packaged current-main run.

## Solution Assessment
