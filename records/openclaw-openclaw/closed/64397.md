## Summary

Keep open. Current `main` still routes TTS through `speech-core/runtime-api.js`, and while the repo now has build tests that assert that file is packed, I did not find strong evidence that the specific installed/subagent resolution failure from #64397 has been fixed. The relevant resolver/runtime files are also unchanged between `v2026.4.23` and `HEAD`.

## Evidence

- **TTS still depends on the speech-core bundled runtime surface:** `src/tts/tts.ts` re-exports the TTS API from `../plugin-sdk/tts-runtime.js`, and `src/plugin-sdk/tts-runtime.ts` still resolves `speech-core/runtime-api.js` through `loadActivatedBundledPluginPublicSurfaceModuleSync(...)`. That means the same bundled-public-surface path remains on the hot path for voice-call TTS.
  - file: [src/plugin-sdk/tts-runtime.ts:21](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/plugin-sdk/tts-runtime.ts#L21)
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **Current tests prove packing, not this installed subagent path:** `test/scripts/bundled-plugin-build-entries.test.ts` asserts that `dist/extensions/speech-core/runtime-api.js` is included in packaged artifacts. That is useful, but it does not prove that an installed VPS build can resolve that surface during `sessions_spawn` subagent startup.
  - file: [test/scripts/bundled-plugin-build-entries.test.ts:56](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/test/scripts/bundled-plugin-build-entries.test.ts#L56)
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **Postpublish verification does not explicitly cover speech-core as a required runtime sidecar:** The installed-package verifier checks only `PUBLISHED_BUNDLED_RUNTIME_SIDECAR_PATHS`, which is derived from `scripts/lib/bundled-runtime-sidecar-paths.json`. That baseline includes many `runtime-api.js` sidecars such as `voice-call`, but it does not list `speech-core/runtime-api.js`, so I did not find an explicit shipped-install guard for this exact surface.
  - file: [scripts/openclaw-npm-postpublish-verify.ts:50](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/scripts/openclaw-npm-postpublish-verify.ts#L50)

I did not find a post-release code change that clearly closes #64397.
