## Summary

Current main and the shipped prompt-cache stack already keep Tooling and skills in the cache-stable system-prompt prefix and split Anthropic-family payloads so only the stable prefix receives cache control; the report has placeholder reproduction fields and no cache trace that contradicts that implementation.

## What This Changes

The issue asks for tools and skills to be included in the prompt-cacheable stable prefix instead of only a tiny core system prompt.

## Best Possible Solution

Keep the shipped prompt-cache boundary implementation and ask for a fresh report only if a current release or current-main cache trace still shows tools or skills excluded from the stable prefix.

## Reproduction Assessment

No. The report uses placeholder steps/model/provider fields and has no logs or cache trace, while current source, docs, and regression coverage show tools and skills in the cache-stable path.

## Solution Assessment

Yes. The current boundary-and-shaping design is the maintainable fix because stable prompt content is cached while volatile runtime/channel content remains below the boundary.
