## What This Changes

This PR changes the Feishu WebSocket monitor to supervise Lark WSClient lifetimes, detect stalled reconnect state, recreate dead clients with backoff, preserve bot identity across cycles, and add cleanup regression tests.

## Best Possible Solution

Use #52618 and #73739 as the canonical path: land a current-main Feishu WebSocket supervisor that preserves heartbeat/proxy/async client behavior, uses public SDK lifecycle callbacks where possible, keeps redacted logging, and closes older duplicate PRs after the replacement is resolved.

## Reproduction Assessment

Yes. The PR body and #52618 provide concrete reproduction steps, and current main source confirms the failure mode: after `wsClient.start()` returns, OpenClaw waits only for abort and has no post-start signal to recreate a dead SDK client.

## Solution Assessment

No. The desired product direction is right, but this branch is no longer the best merge path because #73739 tracks the same fix against the newer Feishu transport and preserves current redaction/lifecycle behavior.
