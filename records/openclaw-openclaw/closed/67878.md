## Summary

Current main fixes the specific Active Memory timeout overshoot: the plugin now races the recall subagent against the abort signal and returns a timeout result at the configured deadline instead of awaiting the embedded runner cleanup path. Regression coverage simulates a subagent that never observes abort and asserts the hook returns near timeoutMs rather than 30s.

## Best Possible Solution

Close #67878 as implemented on current main. Keep the shipped hard-deadline behavior in Active Memory; leave broader model/backend recall failures or runtime rewrite questions to the already separate related tracking items rather than reopening this narrow elapsedMs overshoot report.
