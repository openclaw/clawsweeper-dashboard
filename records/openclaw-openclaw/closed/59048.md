## Summary

Keep #59048 open. Current main still omits `encoding_format: "float"` from the `memory-lancedb` OpenAI embeddings request even though the plugin depends on OpenAI SDK v6.34.0, whose embeddings client defaults omitted encoding_format to base64 and decodes the response. The PR is a narrow, relevant fix with a matching test update, and no merged/current implementation supersedes it.

## Best Possible Solution

Keep #59048 open as a plausible focused bug-fix PR. The best path is to review or land a narrow change that explicitly sets `encoding_format: "float"` in `extensions/memory-lancedb/index.ts` and updates the colocated test, then close the remaining open reports such as #45982/#63655 only after that fix ships or a canonical replacement PR lands.
