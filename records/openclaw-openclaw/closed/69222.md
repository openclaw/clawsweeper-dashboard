## Summary

Keep #69222 open. Current main only contains the narrower #71741-style behavior: Ollama Web Search now calls `/api/web_search` and honors an explicit Ollama Cloud base URL, but it does not implement #69222's compatibility chain, legacy endpoint fallback, conditional Ollama Cloud retry after local 404s, or documented auth scoping. The linked bug #69132 is also still open in the provided context, and #69178 was closed unmerged, so there is no canonical merged replacement for the remaining work.

## Best Possible Solution

Keep #69222 open or ask the author to rebase it onto current main, preserve the already-shipped `/api/web_search` and explicit `https://ollama.com` behavior, add the missing local 404 fallback/cloud retry/auth-scoping tests, and address the mixed-auth review comment before merge. If maintainers prefer not to add automatic cloud fallback, close only after documenting that product decision on #69132/#69222.
