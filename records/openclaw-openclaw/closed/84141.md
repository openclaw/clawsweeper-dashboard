## Summary

Keep open: current main still has the Codex isolated-cron tool-hydration gap, with live reporter evidence on v2026.5.20 and source evidence showing restricted `toolsAllow` disables native Codex shell while the dynamic fallback filters out `exec` before allowlist application.

## What This Changes

The issue reports that isolated cron `agentTurn` jobs requesting `exec` through `toolsAllow` can start without executable tools and still produce success-shaped cron outcomes.

## Best Possible Solution

Land a narrow maintainer-approved fix that keeps restricted native Codex shell/file mode fail-closed, but either hydrates the explicit OpenClaw dynamic `exec`/`process` fallback or fails cron preflight before model invocation when required tools are unavailable.

## Reproduction Assessment

Yes at source level: current main forwards cron `toolsAllow`, then the Codex path disables native tools for finite allowlists and filters out dynamic `exec` before the allowlist can select it. Reporter comments also give live v2026.5.18 and v2026.5.20 reproductions.

## Solution Assessment

No for current main: it preserves the right fail-closed native-shell rule but leaves explicit `exec` cron requests without a usable fallback or loud preflight failure. The best fix should be the linked narrow Codex fallback or equivalent preflight path with security-boundary tests and live proof.
