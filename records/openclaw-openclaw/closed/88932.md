## Summary

Current main now detects the reported Anthropic invalid-signature thinking-block rejection and retries the provider call with thinking blocks stripped, so the session no longer hard-loops; the proof is current-main-only and is not in v2026.5.28.

## What This Changes

The issue asks OpenClaw to prevent expired Anthropic extended-thinking replay signatures from bricking long-running sessions.

## Best Possible Solution

Keep the current main preserve-then-recover implementation and ship it in the next release; reopen only if a current-main build still loops after replay-invalid recovery.

## Reproduction Assessment

Source-reproducible: v2026.5.28 lacks the replay-invalid thinking-signature classifier, while current main has classifier and recovery tests for the exact error shape. I did not run a live Anthropic session in this read-only review.

## Solution Assessment

Yes. Current main preserves signed thinking where Anthropic requires it for tool-use/current-turn continuity, then recovers opaque invalid-signature rejections by retrying once without thinking blocks, which is narrower than removing all Claude 4.6+ historical thinking up front.
