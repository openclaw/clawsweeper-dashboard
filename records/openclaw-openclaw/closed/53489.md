## Summary

Current main already solves the user-visible thread/topic context-bleed problem by making parent transcript inheritance opt-in at the channel boundary. The PR branch is now stale because it removes generic session branching in core, which would also disable the documented Slack/Discord `thread.inheritParent` opt-in.

## What This Changes

The PR changes `src/auto-reply/reply/session-fork.runtime.ts` so thread/topic forks create a clean JSONL session header with `parentSession` lineage instead of calling `createBranchedSession()`.

## Best Possible Solution

Keep the current channel-level contract: default thread/topic sessions use isolated session keys and optional thread starter/history context, while only callers that deliberately want transcript seeding set `ParentSessionKey`. Preserve `ModelParentSessionKey` as the model-selection fallback so parent model choices can flow without copying transcripts.

## Reproduction Assessment

Yes. The original #758-style reproduction is concrete, and current main can be checked deterministically by the routing/session tests that show default Slack and Discord thread sessions omit `ParentSessionKey` while preserving explicit opt-in inheritance.

## Solution Assessment

No for this PR diff. The current main approach is narrower: it prevents default context bleed at each channel caller while preserving the documented `thread.inheritParent` transcript-copy opt-in; removing `createBranchedSession()` from the generic runtime would break that contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve explicit parent transcript inheritance:** `src/auto-reply/reply/session-fork.runtime.ts:62-71`
  - body: This change always creates a clean child session whenever `ParentSessionKey` is set. Current Slack and Discord code use `ParentSessionKey` as the documented `thread.inheritParent` opt-in, with `ModelParentSessionKey` handling model-only fallback, so this would silently disable the explicit transcript-inheritance path. Keep the opt-in branch or move clean-session behavior to callers that should not request inheritance.
  - confidence: 0.9
