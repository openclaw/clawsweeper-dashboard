## Summary

Keep open. Current main moved PDF parsing behind the bundled document-extract plugin, but the active pdf.js call still omits standardFontDataUrl. The PR may need adaptation to the current plugin-owned file layout, but the reported bug is not implemented on main and the PR discussion contains useful reproduction, review, and regression-test evidence.

## Best Possible Solution

Keep this open until the PDF standard-font fix is either adapted here or consolidated into a canonical related item. The concrete implementation should live in extensions/document-extract/document-extractor.ts: resolve the installed pdfjs-dist package root, pass a trailing-separator filesystem standardFontDataUrl into getDocument, widen the local PdfJsModule test/type shim, and add plugin-level regression coverage in extensions/document-extract/document-extractor.test.ts.
