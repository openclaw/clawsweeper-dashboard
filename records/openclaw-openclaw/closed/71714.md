## Summary

Current main already implements Enter-to-send for the Control UI chat textarea, and the same behavior is present in the verified v2026.4.24 release tag. The textarea binds a keydown handler, unshifted Enter prevents the default newline, and it calls the same send path as the Send button.

## Best Possible Solution

Keep the shipped Control UI behavior: Enter sends, Shift+Enter inserts a newline, IME composition is respected, and the Send button/keyboard path share handleSendChat. If a user can still reproduce on a newer build, the useful follow-up would be a fresh issue with exact OpenClaw version, browser, IME/composition state, and a short recording or console trace.
