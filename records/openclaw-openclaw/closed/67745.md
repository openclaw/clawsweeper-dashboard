## Summary

Current main implements the reported custom OpenAI-compatible tool-call path: proxy-like `openai-completions` endpoints with tools now get `tool_choice: "auto"`, streamed `tool_calls` are parsed into `toolCall` blocks with `toolUse` stop reasons, and those parsed calls feed the embedded runner's normal OpenClaw custom-tool execution path.

## Best Possible Solution

Keep the current main implementation and ship it in the next release. If TimeWeb still fails after that, the follow-up should be a new, narrower provider-specific report with OpenClaw's streamed request/response logs, because the generic OpenAI-compatible request shaping and tool-call parsing path is now implemented.
