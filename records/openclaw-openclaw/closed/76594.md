## Summary

## What This Changes

The PR proposes config-scoped caching for effective tool-policy resolution and provider hook resolution, plus minor whitespace/comment edits in the tools planner.

## Best Possible Solution

Close this branch as superseded, continue provider-resolution caching in https://github.com/openclaw/openclaw/pull/77848, and revisit tool-policy caching only as a separate benchmarked change with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows the PR caches the result of a helper that can return [] only as an in-flight reentry guard, and current provider-runtime tests cover the nested-load behavior that would be poisoned by that cached miss.

## Solution Assessment

No. A config-scoped provider-hook cache is not the narrow safe fix; the operation-scoped approach in https://github.com/openclaw/openclaw/pull/77848 avoids stale provider state and does not cache the in-flight guard result.
