## Summary

Current `main` already ships the underlying request in a stronger form: plugin installs and gateway-backed skill dependency installs now fail closed on built-in dangerous-code `critical` findings by default, with a break-glass `--dangerously-force-unsafe-install` override instead of the PR's proposed opt-in `--code-safety block-critical` mode. That makes this PR obsolete on current `main`.
