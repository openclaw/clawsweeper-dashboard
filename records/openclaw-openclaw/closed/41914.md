## Summary

Current `main` already reinstalls and restarts a loaded gateway service during `openclaw update`, and the Linux systemd install path recreates the unit file if it is missing. The updater code and tests on `main` match the issue’s reported workaround (`gateway install --force` + restart), so this is implemented on `main` even though it does not appear to be shipped in the latest tagged release yet.

## Evidence

## Close Comment

Closing from a Codex review: current `main` already implements the recovery path this issue asked for. `openclaw update` now detects a loaded gateway service and refreshes it from the updated install root by running the equivalent of `openclaw gateway install --force` before restart (`src/cli/update-cli/update-command.ts:256`, `src/cli/update-cli/update-command.ts:739`, `src/cli/update-cli/update-command.ts:1395`). On Linux, that install path recreates the systemd unit file if it is missing and then runs `systemctl --user daemon-reload`, `enable`, and `restart` (`src/daemon/systemd.ts:441`, `src/daemon/systemd.ts:489`, `src/daemon/systemd.ts:541`). There is also updater coverage for this path on `main` (`src/cli/update-cli.test.ts:1515`). I could not confirm a released tag containing this yet, so `main` is fixed even if older releases are not.
