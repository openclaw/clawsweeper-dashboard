## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/94730. Current main preserves copy-sensitive paths in `wrapNoteMessage`, then passes the text to `@clack/prompts.note`, whose pinned `1.4.0` implementation hard-wraps via `wrapAnsi({ hard: true })`; `src/commands/doctor-session-locks.ts` therefore can show `.jsonl.lock` paths split inside the final note box. Fix this in the shared terminal-core note renderer, preferably by owning the final note-box rendering or otherwise preventing Clack's second hard wrap from breaking copy-sensitive tokens. Do not add config flags, do not special-case only session-lock text, and do not edit `CHANGELOG.md`. Add regression coverage that captures final rendered `note()` output at 80 columns and proves long paths/URLs remain unbroken while existing normal wrapping still works.
