## Summary

Issue #51168 is about the external `openclaw-agentmail` channel plugin's WebSocket handler and imports, not an AgentMail implementation in OpenClaw core. Current main has generic channel plugin APIs and inbound reply helpers, and the project scope directs optional service-specific channels to ClawHub/npm unless a concrete missing core plugin API is identified.

## Best Possible Solution

Close this OpenClaw core issue as ClawHub/third-party plugin scope. The practical fix is to update `wko/openclaw-agentmail` to use the current focused SDK imports and wrap/log its WebSocket inbound handler with the existing channel inbound dispatch helpers; open a new OpenClaw issue only if that work exposes a concrete missing generic plugin API.
