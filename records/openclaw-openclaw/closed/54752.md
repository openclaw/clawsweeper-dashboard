## Summary

Current main already addresses the reported Discord gateway crash modes: the gateway plugin now uses a nonzero reconnect budget, gateway `error` events are supervised through startup/teardown/dispose instead of surfacing as uncaught exceptions, and regression tests cover the late `Max reconnect attempts ...` shutdown path without throwing.
