## Summary

Close as duplicate: this report and the latest 2026.5.7 confirmation describe the same remaining false-success subagent completion path already tracked by the open canonical report at https://github.com/openclaw/openclaw/issues/74448.

## What This Changes

The issue reports that subagent sessions can emit a `completed successfully` handoff even when the child task is interrupted, compacted, or still running.

## Best Possible Solution

Keep one canonical tracker at https://github.com/openclaw/openclaw/issues/74448 and fix `agent.wait` plus subagent outcome mapping so incomplete lifecycle states cannot be announced as successful completion.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: emit a lifecycle `end` with paused/abandoned/replay-invalid liveness but without `yielded` or `aborted`, and current `agent.wait` can still return `ok`; I did not run a live multi-agent compaction repro.

## Solution Assessment

Is this the best way to solve the issue? Yes: the narrow maintainable path is to fix the canonical runtime wait/outcome contract, not to infer completion quality from child progress text in a second issue.
