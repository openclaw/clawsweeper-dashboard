## Summary

Current main no longer uses the older direct tar extraction/write path that produced the reported SafeOpenError during npm plugin installs. Registry plugin installs now pack to a temp .tgz, extract via the shared archive flow, stage tar extraction under the canonical destination, preflight entries, reject link/device entries, and merge only regular files/directories through the safe copy boundary. The reported npm tarballs also have normal package-root layouts with openclaw.plugin.json, and current tests cover npm-spec install success plus tar preflight behavior.

## Best Possible Solution

Close this issue against the shipped staged archive extraction path. Keep the current npm-pack-to-archive installer and staged tar merge implementation; if someone can still reproduce on the current release, ask for a new issue with the current OpenClaw version, Node/npm versions, exact plugin spec, full stack trace, and whether the downloaded tarball contains links or path conflicts.
