The issue asks for managed default-line npm plugins such as `@openclaw/codex` to sync during `openclaw update` or clearly warn when stale relative to the updated core.

## Best Possible Solution

Keep the current main behavior and ship it in the next release so package-manager updates also synchronize tracked managed plugins and report scoped plugin failures clearly.
