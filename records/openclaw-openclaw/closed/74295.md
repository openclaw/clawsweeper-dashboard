## Summary

Keep this PR open for maintainer review. Current main still omits the LID fallback in the WhatsApp ack-reaction path, while the inbound identity pipeline can produce LID-only group senders and the existing send layer already accepts LID participants.

## What This Changes

This PR updates WhatsApp automatic ack reaction participant construction to prefer `sender.jid` with `sender.lid` as a fallback, and adds ack-reaction regression tests for LID-only and JID-precedence group senders.

## Best Possible Solution

Land the narrow WhatsApp plugin fix after normal maintainer review and green targeted checks, keeping the fallback local to the ack-reaction monitor and preserving direct-message behavior plus JID precedence.
