## Summary

Keep #64108 open. Current main still has the root hardcoded DEFAULT_AGENT_ID behavior in resolveSessionKey(), and the PR is a focused attempt to fix a real #63992 regression rather than obsolete or out of scope. The PR also has a latest unresolved review concern about preserving legacy main-key resume for plain --to requests, so it should be revised or replaced, not auto-closed.

## Best Possible Solution

Keep the item open and ask for a revised patch, or have a maintainer take it over. The fix should make resolveSessionKey cfg-aware or otherwise canonicalize every remaining raw caller, update Slack and agent-command paths, derive/load stores from the final resolved agent key, preserve legacy agent:main main-key lookup for both sessionId and plain --to resumes, and add regression coverage for a non-main configured default agent.
