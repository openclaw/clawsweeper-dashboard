## Summary

Current main already fixes the reported implicit OpenAI-to-Codex crash path: implicit Codex selection falls back to PI when the Codex harness is not registered, while explicit `agentRuntime.id: "codex"` remains fail-closed by design. The fix is current-main-only after v2026.5.12.

## What This Changes

Report asks OpenAI official-provider models to stop failing with an unregistered Codex harness when Codex was not explicitly selected.

## Best Possible Solution

Keep the current main fallback semantics and ship them in the next release; users who intentionally want direct OpenAI API traffic should set provider/model `agentRuntime.id: "pi"`.

## Reproduction Assessment

No high-confidence current-main reproduction remains. The v2026.5.12 tag shows the reported throw path, but current main has a source-level fallback for implicit Codex selection when the harness is unavailable.

## Solution Assessment

Yes. The current implementation fixes the narrow crash by falling back only for implicit Codex default selection, while preserving documented fail-closed behavior for explicit provider/model Codex runtime config.
