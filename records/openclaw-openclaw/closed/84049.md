## Summary

Keep open: current main and the latest release still show the Slack lifecycle gap, and an open linked PR explicitly targets this issue, so the issue should remain open until that PR lands or is closed.

## What This Changes

Slack reply dispatch should signal run completion before dispatch idle so Slack typing/status cleanup can use the configured completion delay instead of the fallback TTL.

## Best Possible Solution

Land or replace the linked Slack fix with a narrow change that calls markRunComplete before markDispatchIdle and covers both normal completion and pre-dispatch failure paths with focused regression tests.

## Reproduction Assessment

Yes, at the source level: current Slack dispatch only calls markDispatchIdle, while shared typing cleanup requires both runComplete and dispatchIdle before normal cleanup. I did not run a live Slack session in this read-only review.

## Solution Assessment

Yes, the linked direction is the narrow maintainable fix: align Slack with the shared lifecycle contract and Discord ordering, with regression coverage for both relevant settle paths.
