## Summary

Keep open: current main still has the Slack lifecycle gap, and the linked fix at https://github.com/openclaw/openclaw/pull/84061 is open but not merged. This issue should close only after that PR, or an equivalent replacement, lands.

## What This Changes

The issue asks Slack reply dispatch to call `markRunComplete()` before `markDispatchIdle()` so typing/status cleanup happens after completion instead of waiting for the TTL fallback.

## Best Possible Solution

Land or replace the linked narrow Slack fix that marks run complete before dispatch idle in both affected cleanup paths, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes at source level: current main Slack dispatch omits `markRunComplete()` while the shared typing controller requires both run-complete and dispatch-idle before cleanup. I did not establish a live Slack run in this read-only review.

## Solution Assessment

Yes: aligning Slack with the shared and Discord lifecycle marker order is the narrow maintainable fix. The current open PR is the implementation candidate, so this issue should wait for that PR to land or be replaced.
