## Summary

Keep open: current main and `v2026.5.2` still generate `mdl_list_hf.co_1` for dotted provider IDs while the Telegram model callback parser rejects dots, and the direct fix PRs #40251 and #46035 are closed unmerged.

## What This Changes

The issue asks Telegram `/models` to render model lists as inline keyboard buttons for provider IDs containing dots such as `hf.co`.

## Best Possible Solution

Land a narrow Telegram plugin fix that accepts dots in `mdl_list_{provider}_{page}` provider IDs, adds dotted-provider parser and keyboard regression coverage, and records a user-facing changelog fix entry.

## Reproduction Assessment

Yes. Source inspection plus a direct regex check show current main accepts `mdl_list_open-ai_1` but rejects `mdl_list_hf.co_1`, while the provider keyboard emits exactly that rejected callback for a dotted provider id.

## Solution Assessment

Yes. The narrowest maintainable fix is to expand only the model-list provider capture to include dots and add regression tests; no core model catalog or provider configuration behavior needs to change.
