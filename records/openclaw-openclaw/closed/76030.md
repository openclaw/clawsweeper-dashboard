## Summary

Close as a duplicate of the existing skills symlink containment work: current main still has the reported extraDirs cross-root warning path, but #41651 is the open canonical tracker and #45912 is the open configured-root implementation candidate covering this class of bug.

## What This Changes

The issue requests allowing symlinked skills under one `skills.load.extraDirs` root when the symlink target resolves under another configured extraDirs root, without repeated escape warnings.

## Best Possible Solution

Keep #41651 as the canonical issue and land one source-preserving configured-root symlink policy through #45912 or an equivalent reviewed PR, with extraDirs-to-extraDirs coverage added there.

## Reproduction Assessment

Yes. A static reproduction is high-confidence: with extraDirs `[A, B]`, a symlink `A/name -> B/name` is evaluated against root `A`, its realpath is outside `A`, and current main logs `reason=symlink-escape` before scanning `B` separately.
