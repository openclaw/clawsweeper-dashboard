## Summary

Current `main` no longer matches the reported behavior. Gateway startup now computes a narrow startup plugin set, loads only those plugin IDs, explicitly keeps unrelated provider plugins out of idle startup, and suppresses the plugin-auto-enable startup write from retriggering the config reloader.

## Evidence

- **Gateway bootstrap loads only the startup-scoped plugin set:** `prepareGatewayPluginBootstrap` resolves `pluginIds` with `resolveGatewayStartupPluginIds(...)` and passes them to `loadOpenClawPlugins(... onlyPluginIds: pluginIds)`, so gateway boot is no longer a blanket load of every bundled plugin.
