Yes. This is the best narrow fix shape: direct `readResponseWithLimit` keeps the existing malformed-body catch/status fallback, while the broader `readProviderJsonResponse` helper would convert malformed JSON into a new labeled error instead of preserving this probe's current fallback semantics.
