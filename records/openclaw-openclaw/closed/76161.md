## Summary

The gateway listener detection this PR was opened for is already implemented on current main and shipped in v2026.5.4 via 9c37cfcbdbf7d49ebee84ae2d60dca3b1f66602b. The open branch is now obsolete for that fix and also carries unrelated memory-core watcher changes with a correctness issue, so it should not be merged as-is.

## What This Changes

The PR branch adds argv-based Unix stale gateway listener detection for node/nodejs/bun lsof entries and later adds memory-core/QMD watcher file-count caps.

## Best Possible Solution

Keep the shipped gateway implementation; if memory watcher caps are still desired, re-submit them as a focused memory-core PR with glob-aware counting and real behavior proof.

## Reproduction Assessment

Yes for the original PR-base bug: source inspection shows service installs can run `node <entrypoint> gateway`, while the old lsof filter only accepted command names containing openclaw. Current main no longer reproduces that failure because argv verification with `ps -ww` is implemented and covered by regression tests.

## Solution Assessment

Yes for current main, not for the PR head. The shipped fix preserves the lsof fast path and adds untruncated argv verification, while the PR branch remains obsolete and mixes in unrelated memory watcher changes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Request wide ps output for argv detection:** `src/infra/restart-stale-pids.ts:120-123`
  - body: On non-Linux platforms, this reads argv with `ps -p <pid> -o command=`. Long Homebrew or LaunchAgent command lines can be truncated without wide mode, losing the entrypoint or `gateway` token and causing the stale gateway PID to be missed in the exact scenario this PR targets.
  - confidence: 0.91
- **[P2] Honor collection globs when pre-counting QMD files:** `extensions/memory-core/src/memory/qmd-manager.ts:1631`
  - body: For glob collections, the pre-count walks the whole collection root and only checks Markdown suffixes, so unrelated Markdown outside the configured glob can disable the watcher even though chokidar would watch a smaller set.
  - confidence: 0.82
