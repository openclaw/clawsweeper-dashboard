## Summary

Keep open: the issue remains valid against current main because Feishu is still excluded from packaged core artifacts and the Docker release workflow still builds only `diagnostics-otel` as an opted-in extension, while current channel docs still describe Feishu as bundled.

## What This Changes

The issue asks for the Feishu channel plugin runtime JavaScript to be included in the official Docker image instead of shipping only metadata files.

## Best Possible Solution

Decide the official Docker/package contract for externalized Feishu, then align image contents, dependency installation, docs, and diagnostics so a configured Feishu channel either loads out of the box or points operators to the exact external plugin install path.

## Reproduction Assessment

No live high-confidence reproduction was run here. Source inspection plus the reporter's Docker listing gives a clear reproduction path: inspect `ghcr.io/openclaw/openclaw:2026.5.3-amd64` for `/app/dist/extensions/feishu/*.js`, then compare `openclaw plugins list` before and after installing `@openclaw/feishu`.

## Solution Assessment

Unclear until maintainers choose the distribution contract. Adding Feishu to the Docker extension build args is narrow if Docker should bundle it, but the safer policy alternative may be to keep Feishu external and fix migration/install guidance around that model.
