## Summary

Keep open. Current main and the latest shipped release still have the reported Gateway observability gap, while the requested marker would add a protocol-visible event payload contract that needs maintainer confirmation before a fix PR.

## What This Changes

Add an explicit suppressed marker to the existing `sessions.changed` message-phase payload when a silent/control assistant reply is filtered before `session.message` broadcast.

## Best Possible Solution

Add an additive, typed Gateway event payload contract for suppressed transcript updates, document it, and cover the text-only silent-token path with a focused session-event regression test.

## Reproduction Assessment

Yes, by source inspection: a text-only assistant `NO_REPLY` update is filtered by display projection, causing no `session.message`, while the following `sessions.changed` payload has no suppression marker. I did not run a live gateway timing scenario in this read-only review.

## Solution Assessment

Unclear. Reusing the existing `sessions.changed` message-phase broadcast looks like the narrowest implementation point, but the protocol-visible marker and reason taxonomy should be decided before implementation.
