## Summary

Current `main` already auto-cleans successful one-shot cron jobs. One-shot `at` jobs default `deleteAfterRun=true`, the scheduler removes them from the store after a successful run, tests cover the removal path, and the CLI docs document the default behavior.

## Evidence
