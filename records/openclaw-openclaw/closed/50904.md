## Summary

The request mixes two things: core model fallback, which OpenClaw already has, and prompt/task-based model routing, which current project scope pushes into plugins/skills or external routing layers rather than new core orchestration.

## Evidence

- **Vision keeps optional orchestration out of core:** `VISION.md` says optional capability should usually ship as plugins and lists `heavy orchestration layers that duplicate existing agent and tool infrastructure` under what will not merge for now.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/VISION.md#L54)
  - sha: [6944d7025d9c](https://github.com/openclaw/openclaw/commit/6944d7025d9c0ba4891cbe646ab5dab039d504ca)
- **Plugin hook already supports per-prompt model selection:** `before_model_resolve` receives the user prompt plus attachment metadata and can return `modelOverride` / `providerOverride`, which is the exact seam a routing plugin would use for task-type or complexity-based selection.
  - file: [src/plugins/hook-before-agent-start.types.ts:7](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/plugins/hook-before-agent-start.types.ts#L7)
  - sha: [6944d7025d9c](https://github.com/openclaw/openclaw/commit/6944d7025d9c0ba4891cbe646ab5dab039d504ca)
- **Core fallback is already implemented:** `docs/concepts/model-failover.md` documents auth-profile rotation plus fallback across `agents.defaults.model.fallbacks`, including ordered candidate evaluation and retry behavior.
  - file: [docs/concepts/model-failover.md:10](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/docs/concepts/model-failover.md#L10)
  - sha: [6944d7025d9c](https://github.com/openclaw/openclaw/commit/6944d7025d9c0ba4891cbe646ab5dab039d504ca)

  - file: [docs/providers/litellm.md:9](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/docs/providers/litellm.md#L9)
  - sha: [6944d7025d9c](https://github.com/openclaw/openclaw/commit/6944d7025d9c0ba4891cbe646ab5dab039d504ca)

## Risks / Open Questions

- There is no bundled ClawHub/plugin implementation in this checkout, so closing on scope may disappoint users who wanted a first-party declarative `openclaw.json` router.
- If maintainers later decide task-based routing is core product direction, this classification would need to be revisited.
