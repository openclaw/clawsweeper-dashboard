## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P1] Render timestamps outside assistant Markdown:** `src/tui/components/assistant-message.ts:12`
  - body: Prefixing the timestamp inside `HyperlinkMarkdown` mutates the assistant message before Markdown parsing, so replies that start with fences, headings, or list markers no longer begin at column zero and can render incorrectly. Keep the timestamp as a separate rendered element or wrapper metadata instead of changing the assistant Markdown payload.
  - confidence: 0.9
- **[P1] Match session events by full agent key:** `src/tui/tui-session-actions.ts:137-145`
  - body: Comparing only `parseAgentSessionKey(...).rest` makes `agent:ops:main` look like the current `agent:main:main` session. When such an event is applied, TUI state can move to the wrong agent/session and route later sends incorrectly, so the matcher needs exact or agent-qualified identity.
  - confidence: 0.91
- **[P2] Preserve configured TUI session defaults:** `src/tui/tui.ts:224-225`
  - body: When `--session` is omitted, this now injects a generated project key before the existing session resolver can honor `session.scope=global` or configured main-key defaults. That changes established default routing for existing users unless they pass `--session`, so the project key should be opt-in or applied only after preserving configured defaults.
  - confidence: 0.88
- **[P2] Keep pending user timestamps on updates:** `src/tui/components/user-message.ts:6`
  - body: The constructor prefixes the first user message, but pending rows are later updated through the inherited `setText(text)` path, which replaces the rendered text without the timestamp. Override `setText` or keep the timestamp out-of-band so updated pending messages remain consistently timestamped.
  - confidence: 0.82
- **[P3] Format timestamps deterministically:** `src/tui/components/timestamp.ts:2-6`
  - body: `toLocaleTimeString` inherits the host hour cycle, so an en-US machine can produce values such as `02:30:45 PM` instead of the intended fixed-width `HH:MM:SS`. Add an explicit 24-hour format or avoid locale formatting for this TUI prefix.
  - confidence: 0.78
- **[P3] Avoid duplicate waiting-status wording:** `src/tui/tui-waiting.ts:45-46`
  - body: The status template always adds `working`, so the `working on the request` phrase renders as repeated wording like `working on the request... • working 3s`. Use a neutral elapsed label that does not duplicate one of the rotating phrases.
  - confidence: 0.74
