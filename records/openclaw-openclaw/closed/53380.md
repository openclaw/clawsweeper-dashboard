## Summary

Current main already implements the requested Control UI avatar/emoji support well enough: agent identity emoji/avatar are resolved into the assistant identity, exposed over `agent.identity.get`, rendered in chat avatars, and user avatars are supported as browser-local text/emoji or image identity. The user-avatar portion also overlaps the previously reviewed #48064 implementation.

## Best Possible Solution

Close as implemented. Keep the shipped model: agent avatars come from gateway-resolved assistant identity (`ui.assistant`, agent identity, or `IDENTITY.md`), while the human user's Control UI avatar remains browser-local and documented. Any future request for richer session-picker avatar decoration should be filed as a narrower UI enhancement, not keep this broad issue open.
