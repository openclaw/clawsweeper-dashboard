## Summary

Current main fixes the reported WhatsApp channel reappearance. The configure wizard still leaves credentials on disk, but persisted WhatsApp auth state alone is no longer treated as configured channel state, so a leftover `credentials/whatsapp/<account>/creds.json` does not regenerate `channels.whatsapp` during gateway startup.

## Best Possible Solution

Close as implemented on current main. Keep the current boundary where saved channel auth can be probed for status/logout flows but cannot auto-enable the channel or rewrite config; handle any future credential-purge UX as a separate, explicit logout/removal enhancement.
