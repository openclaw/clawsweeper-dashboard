## Summary

Current main already fixes the reported cross-session Feishu identity drift by routing cross-agent runs through the target agent's bound account/session context and by having Feishu tools prefer contextual agent account state over the configured default.

## Evidence

- **Changelog documents the shipped cross-agent account-routing fix:** The changelog entry says cross-agent subagent spawns now route through the target agent's bound channel account instead of inheriting the caller's account.
  - file: [CHANGELOG.md:468](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/CHANGELOG.md#L468)
- **Cross-agent spawns resolve the target agent's bound account:** `resolveRequesterOriginForChild` re-resolves `accountId` from the target agent's bindings when `targetAgentId !== requesterAgentId`, then returns that account in the child delivery context.
  - file: [src/agents/spawn-requester-origin.ts:113](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/agents/spawn-requester-origin.ts#L113)
- **Gateway preserves target-session account context for nested agent runs:** When an existing session is targeted, the gateway merges/persists delivery context and carries `lastAccountId`; later it dispatches the run with `accountId` and `runContext.accountId` set from that resolved session/account context.
  - file: [src/gateway/server-methods/agent.ts:661](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/gateway/server-methods/agent.ts#L661)
- **Gateway passes resolved account into the active run context:** `dispatchAgentRunFromGateway` sends `accountId: resolvedAccountId` and `runContext.accountId: resolvedAccountId`, which is the account the tool runtime sees.
  - file: [src/gateway/server-methods/agent.ts:940](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/gateway/server-methods/agent.ts#L940)
- **Feishu tools prefer contextual agent account over default account:** Feishu tool account resolution checks explicit `accountId`, then contextual `defaultAccountId`/`agentAccountId`, and only then falls back to the configured default account.
  - file: [extensions/feishu/src/tool-account.ts:15](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/extensions/feishu/src/tool-account.ts#L15)
- **Feishu regression coverage preserves sender-scoped/session identity:** Tests cover sender-scoped Feishu topic bindings and verify the child session keeps the bound sender conversation instead of drifting to another identity.
  - file: [extensions/feishu/src/subagent-hooks.test.ts:164](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/extensions/feishu/src/subagent-hooks.test.ts#L164)
- **Feishu tool routing coverage uses contextual account selection:** Tests verify Feishu wiki/drive/perm tools execute against the contextual `agentAccountId` rather than whichever account is configured as default.
  - file: [extensions/feishu/src/tool-account-routing.test.ts:69](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/extensions/feishu/src/tool-account-routing.test.ts#L69)
