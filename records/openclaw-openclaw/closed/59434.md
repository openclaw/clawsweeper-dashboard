## Summary

Keep open. Current main still has the exact heartbeat guard this PR removes, and the existing regression test still expects the reasoning payload to be delivered when the final heartbeat reply is `HEARTBEAT_OK`. The PR is a focused implementation candidate for the linked bug, not obsolete or already implemented.

## Best Possible Solution

Review and land this heartbeat-runner fix, ideally with the small follow-up cleanup Greptile identified: return early whenever the normalized main reply is a HEARTBEAT_OK ack, regardless of reasoning payloads; update the existing reasoning + HEARTBEAT_OK test to expect no sends; and keep the added edge case where the reasoning text itself contains HEARTBEAT_OK.
