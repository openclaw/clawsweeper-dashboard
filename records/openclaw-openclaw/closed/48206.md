## Summary

Current main already addresses this `/status` mismatch: the status path enables transcript-usage fallback by default, the status card backfills context/token values from transcript-derived usage when live session metadata is sparse, and session persistence now keeps `totalTokens` from `promptTokens` even when providers omit structured usage payloads.

## Evidence

- **`/status` enables transcript fallback by default:** `buildStatusText` passes `includeTranscriptUsage: true` unless explicitly overridden, so normal `/status` calls use transcript-derived recovery when session metadata is sparse.
  - file: [src/status/status-text.ts:343](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/src/status/status-text.ts#L343)
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **Status card reads transcript-derived usage and uses it for the Context line:** `buildStatusMessage` reads the latest transcript usage snapshot, backfills `totalTokens`/token counters/model from it, and then renders `Context: <total>/<window>` from that recovered value.
  - file: [src/status/status-message.ts:541](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/src/status/status-message.ts#L541)
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **Session accounting persists prompt-token totals even without provider usage payloads:** `persistSessionUsageUpdate` derives and stores `totalTokens` from `promptTokens` whenever there is a fresh context snapshot, covering providers that omit structured usage.
  - file: [src/auto-reply/reply/session-usage.ts:100](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/src/auto-reply/reply/session-usage.ts#L100)
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **Regression coverage exists for `/status` transcript fallback:** The test writes transcript usage with a sparse session entry and asserts `/status` renders `Context: 1.0k/32k` instead of the stale stored value.
  - file: [src/auto-reply/reply/commands-status.test.ts:439](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/src/auto-reply/reply/commands-status.test.ts#L439)
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
