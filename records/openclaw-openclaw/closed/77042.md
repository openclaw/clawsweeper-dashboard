## Summary

Keep open: the PR addresses a real current-main bug path, but the latest diff over-narrows known-plugin detection and would regress restrictive allowlist preservation for registry-known channel plugin IDs that are not marked bundled.

## What This Changes

The PR changes plugin auto-enable to avoid adding unclaimed configured channel IDs to `plugins.allow`, adds a Feishu regression test, and adds a changelog entry.

## Best Possible Solution

Keep the no-claim channel guard, but treat plugin IDs present in the manifest registry as known regardless of origin before suppressing bare channel-ID fallbacks.

## Reproduction Assessment

Yes. Source inspection reproduces the linked current-main bug path through the no-claim channel fallback and also reproduces the PR-head regression against the existing configured-entry allowlist contract.

## Solution Assessment

No. The requested direction is narrow and appropriate, but the current PR branch requires a bundled plugin for channel-shaped IDs instead of first honoring registry-known plugin IDs.
