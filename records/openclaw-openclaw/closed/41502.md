## Summary

Keep this PR open. Current main still hashes generic tool outcomes as `{ details, text }`, so repeated `exec` calls with volatile `durationMs`, `sessionId`, `pid`, or `startedAt` can still defeat the no-progress/global circuit-breaker path that this PR targets. The linked bug remains open, and the current PR scope is a plausible focused fix candidate with regression tests rather than an obsolete or superseded change.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land a focused version that adds exec-specific stable outcome hashing, including completed-output hashing that uses `details.aggregated` and running-output hashing that avoids volatile session/pid text, plus regression tests for warning and global circuit-breaker behavior. Then close the linked bug through the merge.
