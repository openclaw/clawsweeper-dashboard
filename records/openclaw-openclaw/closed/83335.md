## Summary

Current main and the latest release already use the raw/resolved config tree for `config set` mutations and preserve explicit/default-equal values during persistence, so the reported sibling-drop behavior from 2026.5.7 is already fixed.

## What This Changes

The issue reports that deep `openclaw config set` writes under `channels.discord` can drop sibling fields such as `groupPolicy`.

## Best Possible Solution

Keep the shipped raw/resolved config mutation and explicit-path persistence behavior; new reports should target v2026.5.12 or newer with exact before/after config evidence if a different path still fails.

## Reproduction Assessment

No current-main failure was reproduced; source inspection shows the reported deep-write path now mutates a cloned resolved config tree and preserves explicit sibling values through persistence. The issue describes an older 2026.5.7 install, while v2026.5.12 already contains the fixed path.

## Solution Assessment

Yes. The current implementation fixes this at the config write boundary by preserving source config and explicit set paths, which is narrower and safer than adding Discord-specific shims.
