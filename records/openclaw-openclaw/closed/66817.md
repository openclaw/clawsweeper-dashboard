## Summary

Current `main` already prevents metadata-only route updates from refreshing `updatedAt` on existing session entries, which is the behavior that made stale `agent:main:main` rows appear active in Control UI under `dmScope=per-channel-peer`.

## Evidence

- **Route bookkeeping no longer bumps activity:** `updateLastRoute()` now preserves the existing activity timestamp by calling `mergeSessionEntryPreserveActivity(...)` for existing entries, while still updating route metadata fields.
  - file: [src/config/sessions/store.ts:785](https://github.com/openclaw/openclaw/blob/55d3e7bc473e34883a2d2cce51ce904b87c32eed/src/config/sessions/store.ts#L785)
  - command: `nl -ba src/config/sessions/store.ts | sed -n '778,790p'`
  - sha: [55d3e7bc473e](https://github.com/openclaw/openclaw/commit/55d3e7bc473e34883a2d2cce51ce904b87c32eed)
- **Inbound metadata refresh also preserves activity:** `recordSessionMetaFromInbound()` uses the same preserve-activity merge path for existing sessions, so metadata refreshes do not make stale sessions look freshly active.
  - file: [src/config/sessions/store.ts:703](https://github.com/openclaw/openclaw/blob/55d3e7bc473e34883a2d2cce51ce904b87c32eed/src/config/sessions/store.ts#L703)
