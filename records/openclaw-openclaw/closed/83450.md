Full review comments:

- **[P1] Preserve symlink-parent rejection during rotation:** `src/agents/queued-file-writer.ts:74-87`
  - body: The existing writer only appends through appendRegularFile with rejectSymlinkParents, but rotation now runs raw fs.rm/fs.rename first. With a symlinked parent, those operations can rename or remove files through the symlink before the append guard rejects the path, so rotation needs the same safe regular-file boundary and a symlink-parent regression test.
  - confidence: 0.91
- **[P2] Remove the zero queue cap from the sample config:** `docs/reference/prompt-caching.md:317`
  - body: maxQueuedBytes: 0 is not the unlimited default in the implementation; it is a defined zero-byte queue cap, so every positive trace line is dropped. The sample should omit the key or show a positive value so copied config does not silently disable cache tracing.
  - confidence: 0.94
