Full review comments:

- **[P1] Keep rotation inside the fs-safe path checks:** `src/agents/queued-file-writer.ts:78`
  - body: The new rotation step runs raw `fs.rename`/`fs.rm` on the configured path before `appendRegularFile(... rejectSymlinkParents: true)` runs. Current tests require symlink files and symlinked parent directories to be rejected, but once a trace file crosses the cap this code can mutate through those paths instead of preserving the existing fs-safe boundary.
  - confidence: 0.88
- **[P2] Clamp maxFiles before the archive loop:** `src/agents/queued-file-writer.ts:76`
  - body: `maxFiles` comes from config/env as any nonnegative integer and the rotation loop attempts a rename for each slot. A large value can make a single cap-triggered diagnostic write perform an excessive number of filesystem operations, so archive retention needs a small enforced maximum and config validation.
  - confidence: 0.9
