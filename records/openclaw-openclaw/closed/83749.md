## Summary

Current main and the v2026.5.18 release already handle the reported disconnect: dreaming session corpus files remain `.txt`, but they are recorded directly into the short-term dreaming store and deep ranking now gates on total signals rather than raw `recallCount` alone.

## What This Changes

The issue reports that `.txt` dreaming session corpus files are not indexed by `memory_search`, leaving `recallCount` at 0 and preventing deep promotion.

## Best Possible Solution

Keep the shipped design: memory search stays Markdown-focused, while dreaming feeds `.txt` session corpus snippets into the short-term promotion store directly and uses total signal counts for deep promotion.

## Reproduction Assessment

No; I did not run mutating tests in this read-only checkout, but current source and shipped tests show the reported `.txt` corpus gap no longer blocks promotion on current main.

## Solution Assessment

Yes; the current implementation preserves Markdown-only memory search while routing dreaming corpus snippets through the short-term promotion store, which is a narrower fix than indexing private `.txt` machine-state files through general `memory_search`.
