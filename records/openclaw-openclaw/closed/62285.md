## Summary

Current `main` already contains the plugin-loader reentry guards and regression coverage that match the reported 2026.4.5 gateway/CLI hang, and `CHANGELOG.md` records the stack-overflow fix as shipped in 2026.4.7.

## Evidence

- **Shipped fix in changelog:** `CHANGELOG.md` under `2026.4.7` includes: `Plugins/provider hooks: stop recursive provider snapshot loads from overflowing the stack during plugin initialization...`, which matches the linked follow-up about stack overflow during plugin loading.
  - file: [CHANGELOG.md:1183](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/CHANGELOG.md#L1183)

  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Direct reentry guard in plugin loader:** `loadOpenClawPlugins()` now rejects same-cache-key in-flight recursive loads by throwing `PluginLoadReentryError` instead of recursing indefinitely.
  - file: [src/plugins/loader.ts:1893](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/plugins/loader.ts#L1893)

  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Runtime snapshot short-circuit:** `resolveRuntimePluginRegistry()` now returns `undefined` when a compatible plugin-registry load is already in flight, preventing helper/runtime callers from recursing back into the same snapshot load.
  - file: [src/plugins/loader.ts:1069](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/plugins/loader.ts#L1069)
  - command: `nl -ba src/plugins/loader.ts | sed -n '1068,1076p'`

- **Regression coverage for the failure mode:** `src/plugins/loader.test.ts` has a regression case asserting that a reentrant snapshot load surfaces `PluginLoadReentryError` and records an error on the plugin instead of overflowing the stack.
  - file: [src/plugins/loader.test.ts:2396](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/plugins/loader.test.ts#L2396)

- **Gateway startup still uses this loader path:** Gateway startup in current `main` still routes through `loadOpenClawPlugins()`, so the loader fix is directly relevant to the reported `openclaw gateway restart` hang.
  - file: [src/gateway/server-plugins.ts:493](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/gateway/server-plugins.ts#L493)
