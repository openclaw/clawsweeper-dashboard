## Summary

Keep open: current main still has the fragile `mod.r` diagnostic-events alias fallback, and the same reporter has an open linked fix PR that should own the repair until it merges or is closed.

## What This Changes

The issue reports that `normalizeDiagnosticEventsModule` maps a minified diagnostic-events export alias to `onDiagnosticEvent` using a stale hardcoded `mod.r` fallback, causing plugin SDK diagnostic-event unsubscribe failures during gateway stop/restart.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/87084 if it proves the packaged diagnostic-events alias behavior and adds a regression case for shuffled minified exports; otherwise apply an equivalent narrow fix in `src/plugin-sdk/root-alias.cjs`.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not as a live current-main gateway run in this review; the source path is clear, but the exact failing alias mapping depends on packaged beta dist output that is not present in this checkout.

## Solution Assessment

Is this the best way to solve the issue? Likely yes if packaged output proves function names are stable: the maintainable boundary is a narrow root-alias lookup fix plus regression coverage for a shuffled alias map, not a broader SDK API change.
