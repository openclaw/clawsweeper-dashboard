## Summary

Current main and the latest release now cover the reported oversized-context path: the embedded runner resolves the effective context budget and runs a pre-provider prompt-pressure precheck that skips model submission when recovery is needed.

## What This Changes

The issue reports that an Ollama Cloud turn could receive an oversized prompt before OpenClaw enforced the configured context window.

## Best Possible Solution

Keep the shipped generic runner-side precheck and ask for a fresh report only if Ollama Cloud still shows oversized dispatch after `v2026.6.6` or a current-main build.

## Reproduction Assessment

Yes from source for the oversized-context path: current main computes a context budget and runs a pre-provider LLM-boundary precheck before prompt submission. I did not run a live Ollama Cloud repro.

## Solution Assessment

Yes. The shipped fix is the maintainable generic boundary because it prevents provider dispatch before transport instead of adding Ollama-specific policy.
