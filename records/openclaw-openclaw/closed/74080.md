## Summary

Current main and release v2026.4.26 already preserve the parent/requester delivery account for subagent completion announce-back. The child session may still be considered for same-account thread routing, but cross-account child bindings are not selected because the stored completion origin uses the requester account and the bound-delivery router requires a channel+account match.

## Best Possible Solution

Close as implemented and direct affected users on 2026.4.24 to upgrade to v2026.4.26 or newer. Keep the current split between parent completion origin and child thread-delivery origin, plus the regression test that proves mismatched child account bindings do not take over completion announce delivery.
