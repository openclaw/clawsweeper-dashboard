## Summary

Keep open. Current `main` still derives browser control/CDP defaults directly from `resolveGatewayPort(rootConfig)` with no `gateway.mode="remote"` exception, so a remote ingress port like `443` would still shift local browser ports. PR #37949 addresses that gap, but the proposed implementation described in the PR context would also discard `OPENCLAW_GATEWAY_PORT` in remote mode, so there is not strong evidence to close this as already implemented or obsolete.
