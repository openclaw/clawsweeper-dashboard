## Summary

Current main already cleans up session transcripts during session removal: subagent sweep routes through `sessions.delete`, deletes no longer leave raw orphan `.jsonl` files, stale/capped store maintenance archives removed transcripts, and released tests cover the behavior. The same implementation is present in release `v2026.4.23` (`a9797214338ba31c52c796adbb75afb16e0684a9`).

## Evidence
