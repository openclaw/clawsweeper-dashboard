## Summary

Current main contains the memory-lancedb auto-capture starvation fix from 69c30e37. The plugin now keeps per-session auto-capture cursors, only scans new messages after the cursor, retries failed message processing, handles compacted histories, and clears cursor state on session end. The unreleased changelog explicitly records the fix for this issue and carries forward the earlier #42083 approach. The adjacent doctor/status runtime-recognition problem remains separate and is already represented by #65066/#60177.

## Best Possible Solution

Close this issue as implemented by the cursor-based auto-capture fix on main. Keep the shipped heuristic capture behavior, and handle the separate doctor/status runtime-recognition work through #65066/#60177.
