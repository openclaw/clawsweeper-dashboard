## Label Justifications

- `P2`: The request addresses real multi-agent reliability pain, but as a core item it is a bounded feature/product-scope question rather than an urgent runtime outage.
- `impact:message-loss`: The issue is about inter-agent messages timing out or failing without guaranteed target delivery or retry.

## Summary

Close as ClawHub/community-plugin scope. Current core has baseline `sessions_send` and partial timeout/delivery recovery, but the requested durable A2A message bus is an optional orchestration layer, and the repository already has a maintained plugin path plus an existing SQLite message-bus proposal for this exact need.

## What This Changes

The issue requests a durable agent-to-agent inbox or queue so `sessions_send` failures and timeouts cannot silently lose messages.

## Best Possible Solution

Keep core on the existing `sessions_send` contract and route durable A2A bus semantics through ClawHub/plugin work unless a future narrow issue identifies a missing plugin API.

## Reproduction Assessment

Yes, at source level: current `sessions_send` has accepted/delayed A2A paths but no durable target-agent inbox with ack/retry state. I did not run a live timeout/restart scenario in this read-only review.

## Solution Assessment

No as a core JSONL/BOOT.md queue proposal; the maintainable path is a SQLite-backed ClawHub/plugin bus or a narrow plugin API issue if the plugin surface proves insufficient.
