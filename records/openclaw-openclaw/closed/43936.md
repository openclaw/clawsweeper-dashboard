## Summary

Keep open. Current main still has the reported behavior: OpenAI TTS instructions are only preserved for `gpt-4o-mini-tts`, even when the resolved `baseUrl` points to a custom OpenAI-compatible endpoint. The PR is useful but stale against the current plugin-owned OpenAI speech files and needs rebase from the old `src/tts` path to `extensions/openai`.

## Best Possible Solution

Keep the PR open and ask for a rebase onto the current OpenAI speech plugin. The best patch is the maintainer-suggested narrow change: add `baseUrl?: string` to `resolveOpenAITtsInstructions`, preserve trimmed instructions for explicit custom OpenAI-compatible endpoints, keep the default OpenAI endpoint gated to `gpt-4o-mini-tts`, call the resolver with `baseUrl` from `openaiTTS`, and add focused tests in `extensions/openai/tts.test.ts` for default endpoint omission, custom endpoint preservation, and blank-instructions omission.
