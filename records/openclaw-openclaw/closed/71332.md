## Summary

Keep open. Current main still builds `openclaw models status` OAuth/token health from the persisted auth store, so a stale stored `anthropic:claude-cli` profile can still drive the reported status even when native Claude CLI credentials are fresh. The PR is an active, unmerged bug-fix attempt for open issue #71256 and now has security-review concerns that need maintainer/author resolution rather than conservative cleanup closure.

## Best Possible Solution

Keep the PR open and require a focused maintainer review/update: solve #71256 by making `models status` reflect native Claude CLI credential freshness for `anthropic:claude-cli` without putting raw Claude CLI credentials into serializable status output, widening status-command plugin execution unexpectedly, or changing persistence/runtime auth semantics. The regression should live around `src/agents/auth-health.ts` and `src/commands/models/list.status-command.ts`, with tests for stale stored profile plus fresh native Claude CLI credentials and for JSON output sanitization if synthetic auth is involved.
