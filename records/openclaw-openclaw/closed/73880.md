## Summary

Current main still appears to have the reported bug: post-update plugin sync targets tracked install records without checking whether the plugin entry is explicitly disabled, and plugin update errors still make the overall update fail. An open fix PR, #73902, already references this issue with closing syntax, so the issue should remain open until that PR lands or is replaced.

## Best Possible Solution

Keep this issue open and review/land #73902, or replace it with an equivalent narrow fix in the plugin updater. The fix should skip explicitly disabled tracked installs before ClawHub/npm/marketplace network calls, keep enabled plugin update failures fatal, preserve install metadata and id migration behavior, and add regression coverage in `src/plugins/update.test.ts`; docs only need an update if the public failure policy wording changes.
