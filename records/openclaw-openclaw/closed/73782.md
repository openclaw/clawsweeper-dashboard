Close as superseded by #73685. The current report's observed failure is the same external `@openclaw/discord@2026.3.13` compatibility problem already tracked there, with #73703 open as the implementation candidate. The `openclaw@2026.4.26` npm tarball does include the bundled Discord sidecar, and current main routes bare `discord` installs to the bundled plugin before npm.

## Best Possible Solution

Close this as a duplicate of #73685 and keep the fix discussion on #73685/#73703. The maintainer path is to land an intentional compatibility/export fix or publish a compatible `@openclaw/discord` package, then verify both bare bundled install and scoped/npm external install behavior.
