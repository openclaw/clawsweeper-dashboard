## Summary

Keep this issue open. Current main still has the reported host-side behavior, and the issue is paired with open, unmerged closing PR #74255, so cleanup should wait for that PR or an equivalent fix to land.

## What This Changes

The issue asks OpenClaw to make context-engine assembled messages authoritative for prompt admission instead of hard-gating the turn on larger pre-assembly history.

## Best Possible Solution

Land or adapt #74255 so successful context-engine assembly is prompt-authoritative by default, raw pre-assembly history is explicit opt-in or separate maintenance/debt telemetry, and the tests/docs reflect that contract.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to pass a small assembled `messages` array and a much larger `unwindowedMessages` array into the current precheck helper, then observe that the larger unwindowed estimate drives compaction.

## Solution Assessment

Yes. The narrow maintainable fix is a generic context-engine prompt-authority contract or default, not a lossless-claw-specific bypass and not removal of overflow protection entirely.
