## Label Justifications

- `P1`: The original bug broke multi-agent completion delivery by letting parent sessions trust incomplete child output.
- `impact:message-loss`: The reported failure delivered the wrong subagent completion payload and suppressed the real final handoff.
- `impact:session-state`: The bug depended on compaction/resume lifecycle state being mistaken for terminal session state.

## Summary

Close as implemented. Current main and the latest release route subagent run completion through a non-terminal `finishing` lifecycle phase while post-turn compaction and delivery finish, and `agent.wait` plus the subagent registry only treat `end`/`error` as terminal.

## What This Changes

The issue reports that subagent completion was announced during compaction before the child produced its real final handoff.

## Best Possible Solution

Keep the shipped terminal lifecycle split: compaction and post-turn maintenance remain internal, and parent sessions only receive subagent completion after the final lifecycle `end`.

## Reproduction Assessment

No. The original source path no longer matches current main because intermediate completion now uses `finishing`, while `agent.wait` and the subagent registry ignore non-terminal lifecycle phases.

## Solution Assessment

Yes. Deferring the terminal `end` until after compaction and delivery is the right boundary because it fixes the lifecycle contract without adding polling or heuristic result-text checks.
