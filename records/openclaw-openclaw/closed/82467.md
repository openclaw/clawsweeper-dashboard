## Summary

Current main and the shipped v2026.5.12 package already solve the runtime auth chunk resolution problem by preserving the stable runtime-model-auth alias; the proposed fallback is redundant and also broadens the public SDK surface with test-only resolver helpers.

## What This Changes

The branch scans for `runtime-model-auth.runtime-*` files when the stable runtime auth module alias is absent and adds direct unit coverage through a test-only helper.

## Best Possible Solution

Keep the shipped stable-alias build fix and only revisit a hashed-chunk fallback if a fresh packaged build shows `dist/runtime-model-auth.runtime.js` missing after runtime postbuild.

## Reproduction Assessment

No; the no-stable-alias condition is not present in current main's built/released package because runtime postbuild writes `runtime-model-auth.runtime.js`. The old failure mode is covered by source and release proof rather than a current-main failing runtime path.

## Solution Assessment

No; this PR is not the best merge path because the shipped build alias fix already solves the observable runtime problem, and this branch exposes private resolver helpers from a public SDK subpath. The safer direction is to keep alias generation covered and require a concrete new package failure before adding another fallback.
