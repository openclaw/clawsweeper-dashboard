## Summary

## Risks / Open Questions

- Slack `message.send` plus auto-delivery can still duplicate replies when suppression depends on channel-aware target parsing rather than raw string equality.
- Slack native streaming can still leak `isReasoning` payloads because the dedicated streaming path lacks the generic dispatch guard.
- Landing this PR as-is may still need the review-thread test fixes before merge, but that is a reason to keep it open for follow-up rather than close it.
