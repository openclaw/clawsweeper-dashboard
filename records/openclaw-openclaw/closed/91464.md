## Summary

Keep this PR open, but it is not merge-ready: the useful bug target is real from source inspection, while the patch currently overrides an explicit no-recall answer and only has mocked-test proof.

## What This Changes

The PR adds active-memory transcript scanning so `memory_search`/`memory_recall` tool-result evidence can be used when the recall subagent returns no usable final summary.

## Best Possible Solution

Revise the fallback to preserve evidence only for empty or failed-to-summarize outputs while respecting explicit no-recall answers, then add real active-memory proof plus focused regression coverage.

## Reproduction Assessment

Yes, source-level: current `runRecallSubagent` returns `rawReply || "NONE"`, so an empty final payload can discard earlier transcript tool evidence. I did not run the mocked test because this review is required to keep the checkout read-only.

## Solution Assessment

No, not yet. The default `memory_search` evidence fallback is plausible for empty output, but overriding explicit `NONE` conflicts with the documented prompt semantics and needs a narrower fix.
