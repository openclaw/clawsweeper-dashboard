## Summary

## What This Changes

The issue requests a public plugin hook that can inspect, replace, or block tool results after execution but before same-turn model consumption.

## Best Possible Solution

Consolidate the design under https://github.com/openclaw/openclaw/issues/29132, using https://github.com/openclaw/openclaw/issues/62123 for async semantics, then either ship a reviewed trusted Plugin SDK seam or document that pre-model result rewriting remains bundled-only.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that the public hook is absent and the only awaited pre-model result middleware remains bundled-only.

## Solution Assessment

Yes for triage: consolidating this under the existing open canonical issue is the narrowest maintainable cleanup. Implementing the hook itself still needs maintainer Plugin SDK and security-policy design before a fix can be scoped.
