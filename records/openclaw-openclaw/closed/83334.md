## Summary

Current main and the latest shipped release already validate `config patch` after applying the patch to the existing config snapshot, so the reported 2026.5.7 fragment-only validation behavior is obsolete.

## What This Changes

The issue requests that `openclaw config patch` validate the merged post-patch configuration instead of validating the patch fragment by itself.

## Best Possible Solution

Keep the shipped merged-result validation behavior and ask for a new report only if a schema-valid patch still fails on v2026.5.12 or newer.

## Reproduction Assessment

No. I did not run the mutating CLI repro in this read-only review, but current main and v2026.5.12 source show validation runs after merging the patch into the existing config snapshot.

## Solution Assessment

Yes. Validating the post-merge `nextConfig` is the narrow maintainable solution and matches the documented recursive patch contract.
