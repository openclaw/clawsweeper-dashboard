## Summary

Keep open. Current main has Telegram silent-message plumbing and an error-only silent-reply option, but it does not implement the requested Telegram config that auto-silences only verbose/streaming messages while preserving notifications for normal final replies.

## Best Possible Solution

Keep the issue open and implement this in the Telegram plugin surface: add a default-off `channels.telegram.silentVerbose` config key (and account-level behavior if that matches existing Telegram account inheritance), define which verbose/streaming payloads it covers, apply `silent` only to those sends, and update Telegram config types, schema, generated metadata, docs, and targeted tests proving normal replies still notify.
