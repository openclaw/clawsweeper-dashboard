## Summary

Current main still matches the reported CLI alias persistence and fallback asymmetry. The CLI runner reports the raw CLI model alias as `agentMeta.model`, session usage persists that value, and model fallback adds the primary candidate before using the alias resolver that configured fallbacks use. This should remain open for a targeted fix and regression test.

## Best Possible Solution

Keep this issue open and fix the current code path. The narrow fix should resolve the primary candidate in `src/agents/model-fallback.ts` through `resolveModelRefFromString` using the same alias index used for fallbacks, with a regression test for a persisted bare alias such as `sonnet` resolving to the configured `anthropic/claude-*` model. A complementary improvement is to have the CLI runner report canonical model identity instead of the raw CLI alias.
