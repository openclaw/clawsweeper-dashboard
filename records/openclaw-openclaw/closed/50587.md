## Summary

Current main already solves the PR's central user problem: inbound channel MediaPaths are rendered into the agent prompt, detected as image refs, loaded through the shared media loader with safety limits, and passed to the model as images. Telegram and Signal both populate the MediaPath/MediaPaths fields, and regression tests cover the prompt plumbing and native image loading path.

## Best Possible Solution

Close this PR as implemented by the current native prompt image pipeline. Keep the shipped shared-loader implementation rather than merging the older dispatch-level fs.readFile approach; any future changes should extend the shared media loader or add focused regression coverage around a remaining channel-specific gap.
