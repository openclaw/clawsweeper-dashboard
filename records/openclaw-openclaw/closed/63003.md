## Summary

Keep this PR open. Current main still does not implement the requested `sandbox.mounts` JSON field or `Effective mounts:` text section, and the linked issue #62988 is already closed as superseded by this PR. The PR remains the active implementation candidate, but its mount reporting needs backend-aware and canonical-runtime cleanup before merge because the output describes filesystem security boundaries.

## Best Possible Solution

Keep this PR open as the active implementation candidate. Revise it to compute sandboxed status through `resolveSandboxRuntimeStatus`, derive mounts from the selected sandbox backend or a shared backend-aware mount contract, preserve both JSON and text output, add focused coverage for JSON, text, Docker binds, direct runtime, session aliases, agent workspace overrides, and remote roots, and remove unrelated test-file churn before review or landing.
