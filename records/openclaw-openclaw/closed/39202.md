## Summary

PR #39202 is superseded by current-main tokenjuice support: OpenClaw now ships an opt-in bundled plugin and documented tool-result middleware seam for compacting noisy exec/bash output without rewriting the approved command. The RTK-specific core rewrite is still absent and still has the approval-plan trust concern raised in PR review, so this PR should close in favor of the shipped tokenjuice/plugin path rather than land as-is.

## Best Possible Solution

Close this PR as superseded by #69946/current tokenjuice support. Keep exec-output compaction on the documented tool-result middleware/plugin path, where output can be reduced without allowing an external binary to replace the command that passed approval. Any future RTK-specific integration should be a fresh security-reviewed proposal rather than landing this core rewrite PR.
