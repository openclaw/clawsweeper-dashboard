## Summary

Keep open. Current main still has the source-level startup/load-plan gap, and the open linked PR #76636 is the right implementation candidate to review rather than closing this issue before it merges.

## What This Changes

Ensure a configured and enabled external context-engine plugin such as `lossless-claw` is registered and used during agent/QQBot turns instead of falling back to `legacy`.

## Best Possible Solution

Review and land or replace PR #76636 so the generic gateway startup/runtime plan always loads the selected non-legacy context-engine plugin before any turn or compaction resolves it.

## Reproduction Assessment

Yes. Source inspection shows a selected context-engine plugin without `activation.onStartup` can be absent from `startup.pluginIds`, and the resolver then emits the exact fallback warning when the registry lacks that engine id.

## Solution Assessment

Yes. The narrow maintainable fix is to include the selected non-legacy `plugins.slots.contextEngine` plugin in the generic startup/runtime load plan, not to add QQBot-specific behavior or a `lossless-claw` special case.
