## Summary

Current main and the shipped v2026.4.24 docs implement the issue's acceptable resolution: ACPX harness permissions remain separate from `approvals.exec`, but the user-facing docs now explicitly explain that separation, show the correct `plugins.entries.acpx.config.*` keys, and map the exact non-interactive permission error to the `permissionMode=approve-all` fix. The remaining per-agent/per-binding override request is already tracked separately in #40332.

## Best Possible Solution

Keep the shipped ACPX separation and user-facing documentation as the resolution for this bug. Any further work should stay scoped to the already-open per-agent/per-binding permission override request in #40332 rather than keeping this duplicate operational confusion report open.
