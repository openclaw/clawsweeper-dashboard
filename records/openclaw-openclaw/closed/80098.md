## Summary

Keep this PR open: it targets a valid linked bug, but the patch is not merge-ready because the new generic hint can preempt existing provider-specific recovery hints and the external PR has no after-fix real behavior proof.

## What This Changes

The PR adds a targeted unknown-model hint and regression test for models configured only under `agents.defaults.models` without a matching `models.providers[provider].models[]` entry.

## Best Possible Solution

Keep the diagnostic fix, but make provider-specific unknown-model hooks win or scope the new hint to the missing-provider-registration case, then add real CLI/gateway proof and focused regression coverage.

## Reproduction Assessment

Yes, at source level: current main has a clear resolver path where an `agents.defaults.models` entry alone is not a provider registration source and the error builder falls back to generic `Unknown model`. I did not live-run the gateway in this read-only review, so this is `source_reproducible` rather than a live reproduction.

## Solution Assessment

No, not as written. The narrow diagnostic is the right direction, but this implementation should not run before provider-specific unknown-model hints, and the PR still needs real behavior proof before merge.
