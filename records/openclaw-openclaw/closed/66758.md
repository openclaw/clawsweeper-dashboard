## Summary

Keep open. Current `main` still sends `image` tool requests through `describeImageWithModel`, and that runtime still does a single `modelRegistry.find(provider, model)` lookup before throwing `Unknown model`. The only post-`v2026.4.23` lookup hardening I found is in `src/agents/tools/media-tool-shared.ts` with new tests, but the image-tool runtime does not use that helper. The related `#69171` behavior is fixed separately in media-understanding model selection, so that cross-reference is not enough to close this issue.
