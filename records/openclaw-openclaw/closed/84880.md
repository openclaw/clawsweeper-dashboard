## What This Changes

The issue asks OpenClaw to restore non-off `sessions_spawn(...thinking)` support for OpenAI/Codex GPT-5 subagents and correct the prior doctor-migration close rationale.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84902 or a successor with shared resolver coverage, runtime-probe or live subagent proof, and separate provider follow-up if the Ollama Cloud report remains distinct.

## Reproduction Assessment

Yes, at source level. A catalog entry with `reasoning: false` reaches an off-only resolver before provider profiles are consulted, while native subagent spawn forwards explicit `thinking` into the reply/run validation path.

## Solution Assessment

Yes, the best direction is a shared resolver fix that lets authoritative provider thinking profiles override stale catalog false for OpenAI/Codex while preserving off-only behavior for genuinely non-reasoning models.
