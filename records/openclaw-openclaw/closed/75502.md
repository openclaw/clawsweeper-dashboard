Keep open. Current main still has a credible installed-plugin-index recovery gap for this downgrade path: disabled persisted records can bypass stale-source detection and then be rehydrated by metadata/config validation with a generic plugin config error.

## What This Changes

The issue asks OpenClaw to tolerate or repair stale disabled bundled-plugin records in `plugins/installs.json` after downgrading, and to report that state file instead of blaming `openclaw.json`.

## Best Possible Solution

Add generic installed-index validation and doctor cleanup that prunes or ignores missing disabled bundled records before metadata/config validation, while keeping enabled missing-plugin failures strict and naming `plugins/installs.json` in diagnostics.

Yes. The narrow maintainable fix is generic installed-index freshness validation plus doctor repair with clearer diagnostics, not a `file-transfer` special case or relaxed validation for enabled plugins.
