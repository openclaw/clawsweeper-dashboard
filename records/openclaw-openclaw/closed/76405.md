## Summary

Keep open: current main still has the Discord release-artifact mismatch that explains the 2026.5.2 boot failure, and no close reason applies. The root package excludes Discord compiled sidecars while shared sidecar metadata still lists them, and the external Discord package metadata still points at TypeScript source entries.

## What This Changes

The issue reports that openclaw@2026.5.2 cannot start the Discord channel because bundled sidecar metadata references missing Discord dist files and the @openclaw/discord npm fallback is source-only.

## Best Possible Solution

Make the release artifact contract consistent by shipping @openclaw/discord with compiled loadable JS entries and package metadata/tests that prove startup can load it, or intentionally restore Discord compiled artifacts to the root package if maintainers reverse the externalization for this release line.

## Reproduction Assessment

Yes. The high-confidence path is upgrading a Discord-enabled install to openclaw@2026.5.2 and restarting the gateway; two issue comments report that path, and current source inspection still shows the package exclusion plus TS-only fallback metadata that explains it.

## Solution Assessment

Yes, with a preference for the external package path. Given current main’s npm-first externalized plugin metadata, the narrow maintainable fix is compiled @openclaw/discord package entries plus release checks, unless maintainers deliberately decide to put Discord back into the root npm package.
