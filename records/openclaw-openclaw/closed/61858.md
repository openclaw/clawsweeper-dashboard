## Summary

Keep open. Current main does not implement per-agent TTS configuration: agent schema/types reject or omit `agents.list[].tts`, resolved agent config does not carry a TTS override, and the TTS runtime resolves only `cfg.messages.tts`. The related PR #66049 was closed unmerged, and the related open issue #56231 is broader/provider-specific rather than a clear canonical replacement for this general per-agent TTS request.

## Evidence

- **Agent schema has no tts field:** `AgentEntrySchema` is strict and lists allowed agent fields from `id` through `runtime`; there is no `tts` entry, so `agents.list[].tts` is not accepted by the runtime schema.
