## Summary

Current `main` already has TUI/gateway-client duplicate-suppression in the relevant path, and the latest shipped release `v2026.4.22` contains the same guards. The client drops repeated `delta`/`final` events for an already-finalized run, reuses a single assistant component per `runId`, and rebuilds history from scratch on reload instead of appending, which matches the reported duplicate-identical-reply symptom from `2026.3.2`.

## Evidence

- **TUI ignores repeated finals for the same run:** `handleChatEvent` returns early for `delta` and `final` events once a run is in `finalizedRuns`, preventing the same gateway run from rendering multiple assistant replies.
  - file: [src/tui/tui-event-handlers.ts:297](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/src/tui/tui-event-handlers.ts#L297)
