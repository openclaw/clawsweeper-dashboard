## Summary

Current `main` does not show a `resolveAgentRoute` bug here. OpenClaw’s router already supports per-sender DM isolation through `session.dmScope`, documents that `agent:main:main` is the shared-DM default, and newer onboarding defaults unset configs to isolated DM sessions. The remaining plausible bug surface is the external WeCom community plugin, not OpenClaw core.

## Evidence

- **Core router honors DM isolation config:** `resolveAgentRoute` reads `cfg.session?.dmScope` and passes it into `buildAgentSessionKey`, so DM routing is config-driven rather than hardcoded to `agent:main:main`.
  - file: [src/routing/resolve-route.ts:623](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/routing/resolve-route.ts#L623)
  - command: `nl -ba src/routing/resolve-route.ts | sed -n '618,670p'`
- **Regression test covers isolated DM session keys:** Routing tests assert that `dmScope="per-channel-peer"` yields a per-sender key (`agent:main:whatsapp:direct:+15551234567`) instead of the shared main session.
  - file: [src/routing/resolve-route.test.ts:118](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/routing/resolve-route.test.ts#L118)
  - command: `nl -ba src/routing/resolve-route.test.ts | sed -n '112,140p'`
- **Shared DM main session is documented behavior:** Docs explicitly say direct chats collapse to the main session by default, and security docs describe `session.dmScope: "per-channel-peer"` as the secure isolated mode.
  - file: [docs/help/faq.md:1339](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/docs/help/faq.md#L1339)
  - command: `nl -ba docs/help/faq.md | sed -n '1332,1348p'`
- **Isolation shipped before this report:** Changelog records `session.dmScope` for multi-user DM isolation in release `2026.1.15`, so per-user DM isolation was already available in OpenClaw before the March 2026 report.
  - file: [CHANGELOG.md:6276](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/CHANGELOG.md#L6276)
  - command: `nl -ba CHANGELOG.md | sed -n '6258,6278p'`
- **New setups now default to isolated DMs:** Changelog and onboarding code show local onboarding now writes `session.dmScope = "per-channel-peer"` when unset, reducing the chance of reproducing the shared-DM setup on current main.
  - file: [src/commands/onboard-config.ts:5](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/commands/onboard-config.ts#L5)
  - command: `nl -ba src/commands/onboard-config.ts | sed -n '1,120p'`
- **WeCom is an external community plugin:** The repo docs list WeCom as `@wecom/wecom-openclaw-plugin` in a separate repository, so a plugin-specific failure would live outside OpenClaw core.
  - file: [docs/plugins/community.md:115](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/docs/plugins/community.md#L115)
