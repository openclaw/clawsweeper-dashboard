## Summary

This PR addresses a real current-main bug, but it is superseded by the open #74258, which fixes the same OpenAI Codex SSH device-code issue while preserving the existing boundary that the short-lived user code is shown in the prompter note, not emitted through `runtime.log`. The current PR's added `runtime.log` user-code output is the unresolved security/logging concern.

## What This Changes

This PR changes the OpenAI Codex device-code auth flow to always show the user code in the prompter note and remote runtime log, and updates the provider test to expect the code in remote mode.

## Best Possible Solution

Close this PR as superseded by #74258, then review and land #74258 or an equivalent scoped patch that shows `Code: <userCode>` in the remote prompter note while preserving the existing test boundary that `runtime.log` does not contain the device user code.
