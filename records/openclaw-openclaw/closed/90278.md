## Summary

Keep open. The SDK provider conversion and regression test are useful, but current embedded/default Anthropic runs use the boundary-aware transport, whose sibling converter still reads tool schema fields without the same guard, so the fix is one-sided.

## What This Changes

The PR changes Anthropic provider tool conversion to skip tools whose schema accessors throw, preserve healthy tools, move cache control to the last converted tool, and add a regression test.

## Best Possible Solution

Apply the unreadable-schema filtering and regression coverage to the active boundary-aware Anthropic transport as well, or narrow the PR to a documented legacy provider-only fix.

## Reproduction Assessment

Yes for the legacy SDK provider path by source inspection and the PR's red/green test: current conversion reads schema accessors while building the payload. The active transport sibling remains source-reproducible rather than fixed because it still performs the same unguarded reads.

## Solution Assessment

No. Provider-local filtering is the right behavior, but this PR applies it only to `src/llm/providers/anthropic.ts` while current embedded Anthropic runs also use `src/agents/anthropic-transport-stream.ts`.
