## Summary

Keep open. Current main still lets Control UI consume broad Gateway session discovery, and the issue is now paired with open draft PR #77175 using closing syntax, so it should remain open until that PR lands or is replaced.

## What This Changes

Control UI should display sessions only for configured agents and avoid restoring stale deleted-agent session keys.

## Best Possible Solution

Land a reviewed version of #77175 that preserves broad Gateway/ACP discovery by default while making Control UI request configured-agent rows and recover stale restored agent keys to the configured main session.

## Reproduction Assessment

Yes. Source inspection shows Control UI calls broad `sessions.list`, the Gateway combined store includes disk-discovered agent stores, and existing tests document disk-only ACP visibility.

## Solution Assessment

Yes, conceptually. The additive `configuredAgentsOnly` direction in #77175 is the narrowest maintainable boundary because it lets Control UI opt in without removing broad Gateway recovery/ACP discovery.
