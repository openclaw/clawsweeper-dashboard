## Summary

This PR adds a new optional bundled memory-recall plugin, but the current OpenClaw product boundary sends optional plugin capabilities to external plugin distribution/ClawHub unless there is a missing core API. The PR branch itself demonstrates the needed public hook surface already exists (`before_prompt_build` returning `prependContext`), while current main already ships automatic recall options through `active-memory` and `memory-lancedb`. The bot-review defects in the script backend were addressed in later PR commits, but the remaining question is product scope, not a missing implementation seam.

## Best Possible Solution

Close this PR from the core repo and encourage publishing the auto-recall plugin through ClawHub/npm using the existing native plugin hooks. Keep core focused on the existing Active Memory and memory-slot paths; open a narrower core issue only if external publication exposes a concrete missing plugin API.
