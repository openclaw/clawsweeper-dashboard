## Summary

## What This Changes

The PR moves target-kind inference into outbound target normalization, passes the inferred kind into outbound and cron id-like resolution, adds focused tests, and updates the changelog.

## Best Possible Solution

Keep the preferredKind propagation, reuse the existing loaded-or-catalog target-read helper for inference, add unloaded catalog direct-target coverage, and merge only after inspectable redacted Discord proof is provided.

## Reproduction Assessment

Yes, at source level: current main can infer a user/direct target kind, but outbound and cron id-like resolution do not carry that inferred kind into maybeResolveIdLikeTarget. I did not run live Discord delivery during this read-only review.

## Solution Assessment

No, not as written. Forwarding inferred preferredKind is the right narrow fix, but the new helper must preserve the existing loaded-or-catalog target-read fallback before this is merge-ready.
