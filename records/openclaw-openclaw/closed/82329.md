## Summary

Keep open: current main still has a source-level path where Telegram partial preview streaming can leave multiple final payloads for later delivery, and the issue is now paired with an open fixing PR that uses closing syntax but has not merged.

## What This Changes

Telegram partial preview streaming should produce one final visible reply when block streaming is off, instead of sending each assistant block as a duplicate final message.

## Best Possible Solution

Revise and land the linked Telegram/finalizable-preview fix so partial previews suppress duplicate final text while preserving media, errors, long-message continuation, and other channel finalization semantics; close this issue after that PR merges.

## Reproduction Assessment

Yes for a source-level reproduction: current main routes partial preview separately from final payload suppression, and the linked PR demonstrates duplicate final payloads before the patch. No live Telegram turn was run in this read-only review.

## Solution Assessment

Yes to keeping this issue open behind the linked fix path. The proposed repair should be scoped to Telegram or an explicit finalizable-preview contract before merge so it does not break other preview-finalization channels.
