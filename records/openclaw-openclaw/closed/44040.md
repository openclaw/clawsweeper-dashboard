## Summary

Current `main` already falls back to durable exact-command `allow-always` approvals when it cannot safely persist an executable allowlist pattern, and the current test suite covers reruns without re-prompting. That makes the PR’s reported same-command repro obsolete on `main`, while the proposed PATH fallback is no longer necessary for that user-visible behavior.
