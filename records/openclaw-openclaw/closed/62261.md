## Summary

Current main and v2026.5.7 implement the central fix: dev update preflight lint is now opt-in and constrained, so the reported default `openclaw update --channel dev` path no longer runs `pnpm lint` or walks back otherwise-good commits because lint was OOM-killed.

## What This Changes

The issue reports that switching to the dev update channel fails when preflight lint is SIGKILLed and an earlier failed lint candidate is reported after fallback work appears to succeed.

## Best Possible Solution

Keep the shipped opt-in lint behavior for dev updates, and handle any future failures in explicit opt-in lint mode or dirty preflight worktrees through narrower follow-up issues.

## Reproduction Assessment

No. Current main shows the reported default `openclaw update --channel dev` path does not run `pnpm lint` unless `OPENCLAW_UPDATE_PREFLIGHT_LINT=1` is explicitly set, so the old default lint/SIGKILL path is no longer present.

## Solution Assessment

Yes. Making lint opt-in while preserving the dev preflight build gate is the narrow maintainable fix for update hosts that OOM-killed parallel lint; separate preflight walkback dirtiness remains tracked at https://github.com/openclaw/openclaw/issues/64818.
