## Summary

Current `main` already mitigates the reported fallback/compaction death spiral: small-context fallback models no longer inherit an impossible compaction reserve floor, context-overflow errors do not keep walking into smaller fallback models, cooldowned primaries are probed on a throttle instead of being hammered every turn, and unrecoverable compaction/session-corruption paths now reset the session instead of requiring manual transcript deletion.

## Evidence

- **Small-context compaction floor cap:** `applyPiCompactionSettingsFromConfig()` now caps `reserveTokensFloor` against the active context window, with an inline comment explicitly calling out small-context models and the prior infinite compaction loop risk when the default 20k floor exceeded the model window.
