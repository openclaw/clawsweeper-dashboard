## Summary

Current main fixes the reported Telegram `/status` rich-message email failure: OAuth/profile email text now causes Telegram rich sends to set `skip_entity_detection`, covering normal sends, draft previews, and reply delivery before Telegram can reject the payload.

## What This Changes

Ensure Telegram `/status` replies with `richMessages=true` are delivered when OAuth/profile email text would otherwise trigger `RICH_MESSAGE_EMAIL_INVALID`.

## Best Possible Solution

Keep the current shared Telegram rich-message builder guard and close this report as fixed on main; any broader plain-text fallback policy should be handled separately from this exact email-entity failure.

## Reproduction Assessment

No for current main: the v2026.6.10 failure is well explained by the logs and shipped source, but current main now sets `skip_entity_detection` for email-like rich text before `sendRichMessage`.

## Solution Assessment

Yes. A central guard in `extensions/telegram/src/rich-message.ts` is the narrow maintainable fix because normal sends, draft previews, and reply delivery already share that rich-message builder.
