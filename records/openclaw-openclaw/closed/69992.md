## Summary

FriendliAI is a useful third-party provider integration request, but current main already has the public plugin, manifest, provider-auth, catalog, onboarding, and dynamic-model seams needed to ship it outside OpenClaw core. The linked PR #69174 demonstrates the same integration shape as `extensions/friendliai/`, and was already reviewed as ClawHub/community-plugin scope rather than a core-bundled provider.

## Best Possible Solution

Close the core issue as ClawHub/provider-plugin scope. The best path is for FriendliAI or the community to publish an external FriendliAI provider plugin on ClawHub/npm using the existing provider SDK and manifest metadata; reopen or file a narrower core issue only if that work exposes a missing generic plugin SDK seam.
