## Summary

Current main separates queued system-event text from the persisted transcript and rewrites the submitted embedded prompt after the run, so raw async exec completion notifications should no longer remain in WebChat user bubbles.

## Evidence

- **Prompt split keeps system events out of transcript prompt:** `runPreparedReply` still drains queued system events into the live command prompt, but the paired test now asserts they are excluded from `transcriptCommandBody` and `followupRun.transcriptPrompt`, which is the path that feeds session history.
  - file: [src/auto-reply/reply/get-reply-run.media-only.test.ts:835](https://github.com/openclaw/openclaw/blob/355c92d69b3803644a6a56247e5e9273bc1d17b9/src/auto-reply/reply/get-reply-run.media-only.test.ts#L835)
  - sha: [355c92d69b38](https://github.com/openclaw/openclaw/commit/355c92d69b3803644a6a56247e5e9273bc1d17b9)
- **Embedded runner rewrites stored user prompt:** After prompting the embedded session, the runner calls `rewriteSubmittedPromptTranscript(...)` with `transcriptPrompt`, replacing the submitted prompt in the transcript with the sanitized version used for history.
  - file: [src/agents/pi-embedded-runner/run/attempt.ts:2561](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/src/agents/pi-embedded-runner/run/attempt.ts#L2561)
  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **Rewrite helper targets user transcript entries:** `rewriteSubmittedPromptTranscript` finds the just-submitted user message and rewrites its text to `transcriptPrompt`, emitting a transcript update only when the stored prompt actually changes.
  - file: [src/agents/pi-embedded-runner/run/transcript-prompt-rewrite.ts:56](https://github.com/openclaw/openclaw/blob/355c92d69b3803644a6a56247e5e9273bc1d17b9/src/agents/pi-embedded-runner/run/transcript-prompt-rewrite.ts#L56)
  - sha: [355c92d69b38](https://github.com/openclaw/openclaw/commit/355c92d69b3803644a6a56247e5e9273bc1d17b9)
- **Regression test proves sanitized transcript content:** The dedicated test appends a user message containing embedded runtime context, runs the rewrite, then reopens the session and verifies the stored user text is reduced to `visible ask` rather than the embedded prompt payload.
  - file: [src/agents/pi-embedded-runner/run/transcript-prompt-rewrite.test.ts:56](https://github.com/openclaw/openclaw/blob/355c92d69b3803644a6a56247e5e9273bc1d17b9/src/agents/pi-embedded-runner/run/transcript-prompt-rewrite.test.ts#L56)
