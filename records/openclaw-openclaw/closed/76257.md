## Summary

Close as implemented: current main and release v2026.4.29 route native Ollama requests through the active custom provider's configured baseUrl, with tests and docs covering multi-host custom provider IDs; the reported 2026.4.24 behavior is obsolete.

## What This Changes

Honor custom Ollama provider baseUrl values so multi-host ollama-* fallback model refs route to the declared daemon instead of localhost.

## Best Possible Solution

Keep the shipped custom-provider routing and guide affected 2026.4.24 users to upgrade to v2026.4.29 or newer.

## Reproduction Assessment

Yes. The report maps to focused source and test coverage: custom Ollama provider IDs resolve through the Ollama hook, pass the provider baseUrl into native dispatch, and strip the provider prefix from the wire model id.

## Solution Assessment

Yes. The implemented extension-owned fix is the narrow maintainable solution because it honors models.providers.<id>.baseUrl per active provider without adding separate plugin instances or core special cases.
