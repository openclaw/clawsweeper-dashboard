## Summary

Keep open: current main still uses the older managed-npm rollback path, while this PR adds the missing per-plugin project snapshot/restore and current-head terminal proof for blocked installs. I found no blocking code-review findings, but the full node_modules snapshot is a real installer availability tradeoff that maintainers should explicitly accept before merge.

## What This Changes

This PR snapshots and restores the per-plugin managed npm project root, including package files, node_modules, and staged npm-pack artifacts, when post-install validation blocks a plugin install or update.

## Best Possible Solution

Land this rollback only after maintainers accept the exact-tree snapshot cost as the right reliability tradeoff for blocked managed npm installs.

## Reproduction Assessment

Yes, with high-confidence source and supplied terminal proof rather than a local rerun. Current main lacks a whole-project snapshot, npm install mutates package files and node_modules, and the PR body shows the before/after blocked-update scenario restoring the tree only with this patch.

## Solution Assessment

Yes, this appears to be the best practical fix if maintainers accept the availability tradeoff. Reinstalling or uninstalling after failure cannot reliably reconstruct npm's hoisted mutations, while the PR snapshots the exact per-plugin project and keeps the legacy shared root boundary explicit.
