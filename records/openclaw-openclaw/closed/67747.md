## Summary

Issue #67747 does not describe a current-main gateway serializer bug. Current code, tests, and docs all treat `askFallback` as an enum limited to `deny | allowlist | full`, so the reported `askFallback: "allow"` is normalized away on load and then written back during gateway startup. The other requested behavior change in the issue body also conflicts with the documented/current contract: requested `tools.exec.*` policy and host approvals are merged conservatively, with stricter security and more aggressive ask winning.

## Evidence

- **`askFallback` is a security enum, not a free-form string:** Current exec approval types define `askFallback` as `ExecSecurity`, and the normalizer only preserves `deny`, `allowlist`, or `full`; any other string becomes `undefined`. That matches the behavior the commenter described and means `"allow"` is unsupported, not silently omitted by a serializer bug.
