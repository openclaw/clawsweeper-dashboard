## Summary

Close because this is a contributor-local Docker Compose sync, not a repo-wide OpenClaw change; it conflicts with the existing generic Docker configuration knobs and has no real behavior proof.

## What This Changes

The PR hard-codes Docker Compose container names, Asia/Singapore timezone, loopback port mappings, and a read-only Obsidian host bind mount in `docker-compose.yml`.

## Best Possible Solution

Keep the root Docker Compose file generic and put personal timezone, port, and host-folder needs in `OPENCLAW_TZ`, `OPENCLAW_GATEWAY_PORT`, `OPENCLAW_BRIDGE_PORT`, `OPENCLAW_EXTRA_MOUNTS`, or a local compose overlay.

## Reproduction Assessment

Yes, at source level: applying the diff would replace configurable Docker defaults and add a contributor-local Obsidian bind mount to the shared Compose file. I did not need a live Docker run because the conflicting env contracts are visible in current main and the Docker docs.

## Solution Assessment

No. The maintainable solution is to keep repo defaults portable and use the existing Docker environment variables or local overlay path for user-specific mounts and locale choices.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.96

Full review comments:

- **[P2] Keep personal host mounts out of the shared compose file:** `docker-compose.yml:42`
  - body: This adds `~/MyDocs/Obsidian/Primary` to the gateway volumes. The root Compose file is used by all Docker installs, so merging this would either rely on a contributor-specific path or expose a private notes vault to the gateway; use `OPENCLAW_EXTRA_MOUNTS` or a local overlay instead.
  - confidence: 0.97
- **[P2] Preserve configurable Docker timezone defaults:** `docker-compose.yml:38`
  - body: Changing `TZ` from `${OPENCLAW_TZ:-UTC}` to `Asia/Singapore` overrides the documented env setting and changes timestamps for every Docker user regardless of locale. The same issue appears in the CLI service.
  - confidence: 0.96
- **[P2] Preserve env-driven gateway port mappings:** `docker-compose.yml:64-65`
  - body: Replacing `${OPENCLAW_GATEWAY_PORT}` and `${OPENCLAW_BRIDGE_PORT}` with fixed `127.0.0.1` mappings ignores documented custom port config and breaks users running multiple gateways or non-default ports.
  - confidence: 0.95
- **[P3] Avoid fixed container names in the base compose file:** `docker-compose.yml:4`
  - body: Hard-coded `container_name` values make separate checkouts or Compose projects contend for `openclaw-gateway` and `openclaw-cli`; Compose's project-scoped default names avoid that collision.
  - confidence: 0.9
