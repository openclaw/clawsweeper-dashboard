## Summary

Keep this issue open: current main and the latest release still use plain lexicographic `localeCompare` in model alias resolution, and the issue is now paired with open closing PRs that should be reviewed or landed before the issue closes.

## What This Changes

The issue asks for model alias resolution to sort version-like model ids numerically so aliases such as `opus` pick `claude-opus-4-10` over `claude-opus-4-9`.

## Best Possible Solution

Land one focused comparator fix with regression coverage, preferably the strongest open candidate or an equivalent sibling, then close this issue after the merge.

## Reproduction Assessment

Yes. A high-confidence source reproduction is resolving `parseModelPattern("opus", [claude-opus-4-9, claude-opus-4-10])`; the current comparator orders `4-9` ahead of `4-10`.

## Solution Assessment

Yes. Numeric-aware collation at the existing `tryMatchModel` sort points is the narrowest maintainable fix, with a regression test for the double-digit alias rollover.
