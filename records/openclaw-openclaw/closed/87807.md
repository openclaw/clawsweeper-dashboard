## Label Justifications

- `P1`: The report describes a recent core embedded-runtime regression that changes live channel model behavior by replacing the intended OpenClaw prompt.
- `impact:session-state`: The bug is about session prompt state drifting after tool activation, causing later model turns to run with the wrong system prompt.

## Summary

Keep open. Current main still has the reported source-level prompt clobber path, and the maintainer label plus MEMBER author association make this explicit maintainer-owned work rather than cleanup-close material.

## What This Changes

The issue reports that active tool selection in the embedded agent runtime replaces OpenClaw's assembled system prompt with the generic embedded coding-agent prompt.

## Best Possible Solution

Add a real session-owned prompt override path that survives active tool changes, migrate the embedded runner and compaction callers to it, and cover both direct session behavior and embedded-runner prompt preservation with focused regression tests.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: apply a system prompt override to an `AgentSession`, then call `setActiveToolsByName`; the current method rebuilds and writes a different system prompt. I did not run a live or unit repro because this review is read-only.

## Solution Assessment

Yes, the suggested direction is the maintainable fix shape: make prompt override state an explicit `AgentSession` responsibility and preserve it when rebuilding tool-aware prompt options. The repair should also cover the compaction sibling path and avoid relying on stale underscored internals.
