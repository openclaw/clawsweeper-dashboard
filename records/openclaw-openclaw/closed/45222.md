## Summary

Current main already fixes the central loopback WebSocket handshake problem reported against 2026.3.12/2026.3.13. Gateway preauth and client connect-challenge timeouts now default to 10 seconds, production can override the server timeout with OPENCLAW_HANDSHAKE_TIMEOUT_MS, the server and client both use the shared timeout helper, and cron list still goes through the shared Gateway RPC path. The issue thread also separated the remaining AntFarm failure as a caller-side --timeout 120ms bug, not an OpenClaw core defect.
