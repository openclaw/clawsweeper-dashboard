## Summary

Keep open. Current main still has the Docker channel mismatch: the release workflow does not run on normal git-main pushes, docs still list the floating `main` image tag, and live GHCR metadata shows `ghcr.io/openclaw/openclaw:main` still points to the March 26 image revision rather than current main.

## What This Changes

The issue asks OpenClaw to either rebuild/publish the GHCR Docker `:main` tag from every git-main push or stop directing users to that tag as current main.

## Best Possible Solution

Define the Docker image channel contract and align workflow, docs, and maintainer guidance around it: either guarded `main`/`main-slim` publishing from main pushes, or explicit deprecation of Docker `:main` as a current-main test path.

## Reproduction Assessment

Yes. `skopeo inspect` against both amd64 and arm64 `ghcr.io/openclaw/openclaw:main` shows March 26 image revisions, while current main is `947c9512c30b09d23090d0f49b9bd685f27e2001` and the workflow source has no normal main-branch trigger.

## Solution Assessment

Unclear because no patch is proposed and both plausible fixes are policy choices. Re-enabling guarded main-push publishing is the direct behavioral fix, while deprecating or documenting Docker `:main` may be safer if releases are intended to be the only published Docker artifacts.
