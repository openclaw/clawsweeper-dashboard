## Summary

Close this PR as superseded. The linked bug is already fixed on current main by merged PR #74188 with source and regression-test coverage, and the extra URL-surfacing change in this branch should not be merged without a separate safety design.

## What This Changes

The branch updates assistant error formatting to parse top-level string error envelopes, adds snake_case billing-code regexes, and appends upgradeUrl-style fields to formatted error text.

## Best Possible Solution

Keep the merged #74188 implementation as the canonical fix for the raw JSON leak. If billing-link surfacing is still wanted, handle it as a separate validated URL feature with explicit rendering rules and tests.

## Reproduction Assessment

Yes. The linked report gives a concrete openclaw 2026.4.26 payload, and current main now has focused tests for the same flat insufficient_balance JSON through both the assistant formatter and billing classifier.

## Solution Assessment

Yes. The superseding PR is the narrower maintainable fix: it parses flat envelopes, classifies the reported billing payload, adds tests, and avoids the unsafe provider-controlled URL display introduced by this branch.
