## Summary

Keep open: current main and the latest release still let the dream diary fallback persist the first raw snippet or promotion into `DREAMS.md`, and the existing unit tests encode that behavior.

## What This Changes

The issue asks for dream diary failure fallback to avoid writing raw memory snippets or promotion fragments into `DREAMS.md`.

## Best Possible Solution

Replace the fallback with a safe generic placeholder or skip, and update timeout, empty-output, and request-scoped fallback tests to assert snippets and promotions never appear in `DREAMS.md`.

## Reproduction Assessment

Yes, at source level: the current fallback returns raw snippets/promotions and the timeout test asserts that raw text is written to `DREAMS.md`. I did not run the test because this review was read-only.

## Solution Assessment

Yes, the narrow maintainable fix is to stop fallback from using raw input fragments and keep successful generated narratives unchanged. The existing placeholder string gives a low-risk repair path without adding config or new product behavior.
