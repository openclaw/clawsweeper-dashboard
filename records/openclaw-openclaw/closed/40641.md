## Summary

Keep open: current main and v2026.5.20 still document unquoted bracket-notation config paths that can fail in shells before OpenClaw receives argv, and https://github.com/openclaw/openclaw/pull/83058 is an open draft PR with closing syntax for this issue.

## What This Changes

The issue reports that the documented `openclaw config set agents.list[0].tools.exec.node ...` command fails because the unquoted bracket path is treated as a shell glob.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/83058 or an equivalent docs/test change that quotes bracket-notation config paths or uses dot-index notation, then let the docs i18n pipeline update localized pages.

## Reproduction Assessment

Yes. Current main still documents unquoted bracket-path examples, and `bash -O failglob` rejects `agents.list[0].tools.exec.node` before OpenClaw receives argv while quoted bracket notation and dot-index notation pass through.

## Solution Assessment

Yes. A docs/test fix is the narrow maintainable solution because the CLI parser already accepts bracket notation after the shell passes the path through.
