## Summary

Current main already has the shared command-queue singleton, wires `agents.defaults.maxConcurrent` into the main lane, and includes regression coverage that distinct module instances share one queue state. The latest release `v2026.4.22` also contains that implementation, so the reported Rolldown split-chunk `lanes` duplication issue appears already fixed.

## Evidence
