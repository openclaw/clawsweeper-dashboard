## Summary

Current main and the v2026.5.19 tag already build and pass a cron `SourceDeliveryPlan` for isolated `agentTurn` runs, including `delivery.mode: "none"`, so the executor no longer reaches `sourceReplyDeliveryMode` through an undefined plan.

## What This Changes

The issue reports that isolated cron jobs with an `agentTurn` payload fail before agent execution with a `sourceReplyDeliveryMode` TypeError.

## Best Possible Solution

Keep the shipped SourceDeliveryPlan-based cron delivery path and direct users still on v2026.5.7 to update to v2026.5.19 or newer.

## Reproduction Assessment

No live current-main cron run was executed in this read-only review. The reported v2026.5.7 failure path is source-auditable, and current main plus v2026.5.19 now construct and pass a SourceDeliveryPlan before the executor reads `sourceReplyDeliveryMode`.

## Solution Assessment

Yes. Centralizing cron delivery into `SourceDeliveryPlan` is the maintainable fix because it gives announce, webhook, and no-delivery modes one explicit contract before agent execution starts.
