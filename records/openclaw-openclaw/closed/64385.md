## Summary

Current `main` already protects against the exact failure mode from #64385: suspicious truncated configs are auto-restored from backup, destructive OpenClaw-owned writes are rejected before replacing the live file, and the shipped tests cover missing `meta`, missing `gateway.mode`, and large size-drop cases. The same implementation is present in release `v2026.4.23`, so this is strong enough to close as fixed on main.

## Evidence
