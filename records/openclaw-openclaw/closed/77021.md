## Summary

Keep open: the PR is a narrow CLI feature with green CI, but it introduces a concrete terminal/log-safety bug by printing raw session labels in dry-run text output.

## What This Changes

The PR adds a text-only per-label kept/pruned summary to `openclaw sessions cleanup --dry-run`, covers labeled and unlabeled dry-run stores in tests, and adds an Unreleased changelog entry.

## Best Possible Solution

Keep the text-mode summary, but render a sanitized display label and add a regression test for newline/ANSI labels without changing JSON, Gateway, or applied cleanup behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Sanitize labels before logging the summary:** `src/commands/sessions-cleanup.ts:102`
  - body: `renderLabelSummary` writes the raw session label into `runtime.log`. Session labels can be supplied as ordinary strings and the label parser only trims/enforces length, so a label containing `\n` or ANSI escape codes can forge extra dry-run lines or affect the terminal when an operator runs cleanup. Please sanitize the displayed label, e.g. with `sanitizeTerminalText`, and add a regression test for control characters.
  - confidence: 0.9
