## Summary

This issue should stay open. Current main still has the default bootstrap identity path and a separate session-specific identity RPC path that can write the same Control UI assistant identity state, and there is an open implementation PR that explicitly says it closes this issue. Per the review rule, an issue with an open closing PR should remain open until that PR merges or is closed.

## Best Possible Solution

Keep this issue open while maintainers review PR #72579. If that PR is accepted, merge it and let GitHub close this issue automatically; if it is rejected, keep this issue as the active tracker for a narrower fix that ensures session-specific `agent.identity.get` results take precedence over default bootstrap identity in Control UI Chat.
