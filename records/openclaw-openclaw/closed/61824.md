## Summary

Current main already fixes PR #61824's user-visible bug: logging config is read directly from the active OpenClaw config path and logger settings use logging.file before falling back to /tmp/openclaw. Regression tests cover both reading the active config file and writing logs to the configured path, and the unreleased changelog records the #61295 fix.

## Best Possible Solution

Close PR #61824 as implemented on main. Keep the shipped main approach that reads logging settings from the active config file directly and has regression coverage, rather than merging the older direct loadConfig import approach that review comments flagged for logger re-entry risk.
