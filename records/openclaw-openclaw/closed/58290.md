## Summary

Current `main` already contains explicit Discord gateway recovery for the reported stuck-READY/offline loop, and the latest shipped release includes the same lifecycle code unchanged.

## Evidence

- **Startup stall recovery is implemented:** `waitForGatewayReady` now gives startup 15s to reach READY, logs `startup-not-ready`, restarts the gateway once, and throws a clear error if READY still never arrives after restart.
