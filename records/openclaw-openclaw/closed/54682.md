## Summary

Current `main` already implements Discord gateway recovery/status handling for this failure mode: reconnect/close events mark the channel disconnected, startup-stuck gateways are restarted, reconnects that never become READY are force-failed instead of hanging silently, and startup status no longer reports Discord as logged in before gateway readiness.
