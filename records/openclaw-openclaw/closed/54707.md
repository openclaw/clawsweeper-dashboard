## Summary

Current main already supports per-agent skill filtering through `agents.list[].skills`, and that filter applies to bundled skills as part of the unified skill set. The exact proposed nested field (`agents.list[].skills.allowBundled`) does not exist, but the shipped behavior already covers the request well enough with a broader per-agent final skill allowlist.

## Evidence
