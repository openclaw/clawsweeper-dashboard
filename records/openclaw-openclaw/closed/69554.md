## Summary

Current `main` already implements `skills.update` as a direct `skills.entries.<skillKey>` config write and includes regression coverage around unrelated runtime-derived config projection, which addresses the ghost `tools.web.search` validation failure described in #69554.

## Evidence

- **Current `skills.update` path only patches skill config:** The handler loads config, clones `skills.entries`, applies `enabled`/`apiKey`/`env`, and writes the result with `writeConfigFile(nextConfig)`. There is no `tools.web.search` mutation path in the handler.
