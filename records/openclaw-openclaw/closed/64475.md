## Summary

Current `main` does not implement the requested behavior. OpenClaw still intentionally preserves explicit user/session model overrides across `/new`, `/reset`, and gateway-side `sessions.reset`; only auto-sourced fallback overrides are cleared.

## Evidence

- **Reset preservation helper keeps user-set overrides:** `resolveResetPreservedSelection` explicitly preserves `providerOverride` and `modelOverride` when `modelOverrideSource === "user"`, and also treats legacy override entries without a source as user-driven.
  - file: [src/config/sessions/reset-preserved-selection.ts:14](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/config/sessions/reset-preserved-selection.ts#L14)
  - command: `nl -ba src/config/sessions/reset-preserved-selection.ts | sed -n '1,120p'`
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **Auto-reply reset path carries preserved selection into new sessions:** In the `/new` and `/reset` path, `initSessionState` calls `resolveResetPreservedSelection` and reassigns the preserved `modelOverride`/`providerOverride` onto the new session entry instead of clearing them.
  - file: [src/auto-reply/reply/session.ts:502](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/auto-reply/reply/session.ts#L502)

  - file: [src/auto-reply/reply/session.test.ts:1921](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/auto-reply/reply/session.test.ts#L1921)
