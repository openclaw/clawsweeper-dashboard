## Summary

Close as implemented on main. The observable Telegram reaction warning problem is already covered by the current released path: Telegram reaction soft failures are returned as `jsonResult({ ok: false, warning: ... })`, `jsonResult()` does not add an error status, and the embedded handler only treats tool results as errors for explicit error events or `details.status` values of `error` or `timeout`. The linked issue was already closed after a Codex review, and the same behavior is present in `v2026.4.27`.

## What This Changes

The PR adds a warning-only tool-result classifier, uses it when handling embedded tool execution end events, and adds regression coverage for message reaction warning results.

## Best Possible Solution

Keep the shipped soft-result Telegram reaction behavior and close this branch rather than adding another generic warning-only override in the embedded subscriber.
