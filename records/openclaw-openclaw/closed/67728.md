## Summary

Keep open. Current main has useful remote-CDP hardening, but it does not implement the requested post-open auto-reconnect: raw CDP retries only the WebSocket handshake before commands run, and current tests still assert that an in-flight socket close fails with one callback and one connection. The related implementation PR #67748 was closed unmerged, so there is still actionable browser-plugin work.

## What This Changes

The issue requests automatic reconnect/retry behavior for remote browser CDP WebSocket drops so remote browser actions can recover without restarting the gateway.

## Best Possible Solution

Keep the shipped handshake retry and safe-read recovery, then add bounded recovery for recoverable post-open transport closes in the browser plugin. The end state should evict stale cached CDP/Playwright connections for later actions, retry only operations that are safe or explicitly idempotent, leave ambiguous mutating commands single-shot, and cover mid-command close, next-action recovery, retry exhaustion, and unsafe non-replay with targeted tests.

## Repair Work Prompt

Fix the browser plugin's remote CDP post-open transport recovery for safe/idempotent operations. Current `withCdpSocket` only retries WebSocket handshake/open failures before any command is sent, while `extensions/browser/src/browser/cdp.helpers.internal.test.ts` still asserts that a socket close during an in-flight command fails with one connection. Preserve the existing handshake retry behavior and the current safety boundary that ambiguous mutating operations are not replayed. Add bounded recovery for recoverable post-open transport closes by evicting stale cached CDP/Playwright connections for later actions and retrying only safe/read/idempotent paths such as tab listing and snapshots. Do not replay tab creation, navigation, uploads, permission grants, or other mutating browser actions unless the operation has explicit idempotence or dedupe. Use #67748 only as background because it was closed unmerged. Add focused tests for close-after-open, mid-command close, next-action recovery, retry exhaustion, and non-replay of unsafe mutations. Add a single user-facing changelog fix entry crediting the reporter `@KouJun01` and credit `@Phymath` too if code or design from #67748 is reused. Keep the work in the browser plugin; do not move owner-specific recovery policy into core.
