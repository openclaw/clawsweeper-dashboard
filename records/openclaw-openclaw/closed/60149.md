## Summary

Current main already has explicit support for Feishu named-account setup plus account-scoped agent bindings, and the same code paths are already present in the latest shipped release. The current wizard/setup code writes named Feishu accounts into `channels.feishu.accounts[accountId]`, the binding schema accepts `bindings[].match.accountId`, and routing upgrades channel-only bindings to account-scoped bindings instead of leaving an invalid shape.

## Evidence

- **Feishu wizard writes named-account config at the account-scoped path:** `patchFeishuConfig` documents and implements named-account writes under `channels.feishu.accounts[accountId]`, with `enabled: true`, instead of mutating the top-level Feishu section into an invalid mixed shape.
  - file: [extensions/feishu/src/setup-surface.ts:75](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/extensions/feishu/src/setup-surface.ts#L75)
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Feishu setup adapter enables named accounts explicitly:** The setup adapter resolves non-default Feishu account ids and writes them through `accounts[accountId].enabled`, which matches the multi-account schema rather than a legacy single-account layout.
  - file: [extensions/feishu/src/setup-core.ts:33](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/extensions/feishu/src/setup-core.ts#L33)
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)

  - file: [src/config/zod-schema.agents.ts:15](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/config/zod-schema.agents.ts#L15)

  - file: [src/commands/agents.bindings.ts:34](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/commands/agents.bindings.ts#L34)
