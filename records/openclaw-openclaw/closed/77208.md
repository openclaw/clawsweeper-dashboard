## What This Changes

Ensure Slack DM normal replies and message-tool sends stay attributed to the same direct Slack session after the v2026.5.2 direct-user send change.

## Best Possible Solution

Keep the current Slack-owned live-reply and outbound mirror canonicalization, and ask for a fresh issue with redacted live Slack logs only if a post-v2026.5.10-beta.3 setup still drops DM replies.

## Reproduction Assessment

Yes for the source-backed attribution bug: before the fix, `message.send` with a native Slack `D...` target could create a `slack:channel:d...` mirror route while inbound DMs used `slack:direct:u...`. I did not run a fresh live Slack workspace repro in this read-only review.

## Solution Assessment

Yes. The narrow maintainable solution is the current Slack plugin behavior: live replies use the concrete `D...` channel, while outbound mirror reconstruction canonicalizes native IM targets to the peer user session without reverting text-only direct user posts.
