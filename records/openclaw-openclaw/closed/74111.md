## Summary

Current main routes built bundled public artifacts such as media-understanding-core/image-ops.js through the bundled runtime-deps preparation path before loading them. That mirrors the artifact under the staged install root, so ESM resolution can find staged node_modules/sharp without requiring a package-local dist/extensions/media-understanding-core/node_modules directory. This is fixed on main after v2026.4.26 and is not yet proven shipped in a release.

## Best Possible Solution

Close this issue as implemented on current main, ship the runtime public-artifact mirror fix in the next release, and keep separate platform-specific sharp issues open where the native package itself is unavailable.
