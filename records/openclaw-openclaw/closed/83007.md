## What This Changes

The issue reports intermittent Read-tool `ENOENT` and false sandbox-escape errors for existing files inside the configured workspace on OpenClaw 2026.4.22.

## Best Possible Solution

Fix the sandbox Read/stat contract so real sandbox rejections and ambiguous remote stat output do not masquerade as missing files, while preserving existing containment checks.

## Reproduction Assessment

Yes, at source level: remote `stat()` can return null after unexpected or empty exists output, and Read then reports `ENOENT`. No high-confidence live reproduction of the intermittent timing or false escape was established in this read-only pass.

## Solution Assessment

Unclear as written: normalizing roots at boot appears already covered in the current sandbox context, but separating sandbox rejection from not-found and rejecting ambiguous remote exists output is a narrow maintainable direction.
