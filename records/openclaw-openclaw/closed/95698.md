Full review comments:

- **[P1] Preserve configured cron fallbacks:** `src/cron/isolated-agent/run-fallback-policy.ts:27-29`
  - body: Returning `[]` for every cron payload model override makes all such jobs strict before `resolveEffectiveModelFallbacks` can return configured fallbacks. Current docs and tests say cron job model overrides remain job primaries with configured fallbacks, and only payload `fallbacks: []` disables fallback, so existing scheduled jobs with configured fallbacks would stop using them on provider errors.
  - confidence: 0.93
- **[P1] Do not trust unvalidated compaction fingerprints:** `src/agents/embedded-agent-runner/run/attempt.session-lock.ts:1017-1021`
  - body: This helper reads whatever fingerprint exists after compaction and records it as owned outside the controller's validated write-lock paths. If another process edits the session file while compaction is running, the next fence can accept that external edit as an owned write instead of reporting takeover; the existing fence invariant only trusts fingerprints produced while OpenClaw held the write lock.
  - confidence: 0.88
- **[P2] Use the real-conversation classifier:** `packages/agent-core/src/harness/compaction/compaction.ts:762-769`
  - body: The new compaction guard only checks for `role === "user"`, but this harness converts `bashExecution` messages to LLM user messages and OpenClaw's existing compaction classifier treats meaningful custom, bashExecution, branchSummary, assistant, and anchored tool results as real conversation. A transcript with those real entries but no user role would now be summarized as `No prior history`, losing context.
  - confidence: 0.72
