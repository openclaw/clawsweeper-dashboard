## What This Changes

The issue reports that subagent completion announce/wake runs on Anthropic Opus reasoning models inherit a deprecated thinking payload, causing the direct completion ping to fail and leaving orphan recovery to deliver the notice.

## Best Possible Solution

Keep the current main provider-level adaptive thinking mapping and Opus 4.8 default behavior, and let the next release carry the fix instead of adding a subagent-only workaround.

## Reproduction Assessment

No current-main failing reproduction is established. The report has strong 2026.5.12 log/source evidence, and current source/test evidence shows the Opus 4.8 payload path now uses adaptive thinking instead of the deprecated enabled schema.

## Solution Assessment

Yes. Fixing the Anthropic transport/default-thinking path is the narrower durable solution because it covers direct announce and any other Opus 4.8 agent run without adding a special subagent-only branch.
