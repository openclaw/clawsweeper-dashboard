## Summary

Current main already supports channel-scoped skill filtering and dispatches those filters into reply execution, which covers the issue’s requested outcome of activating/prioritizing skills per channel without relying on prompt hints.

## Evidence

- **Slack channel config has per-channel skill filter:** `SlackChannelConfig` exposes `skills?: string[]` as an optional skill filter for a channel.

## Risks / Open Questions

- The current surface is an allowlist-style `skills` filter, not the exact proposed tri-state `autoload` / `preferred` / `restricted` schema. A softer 'preferred but not exclusive' mode is not evidenced here.
