## Summary

Current `main` already contains a targeted fix for this exact crash: `src/agents/bash-tools.exec-runtime.ts` suppresses late exec `onUpdate` callbacks once the run exits or aborts, `src/agents/bash-tools.exec.ts` disables updates immediately on abort before leaving background sessions alive, and `CHANGELOG.md` records the shipped fix for the same `"Agent listener invoked outside active run"` failure in release `v2026.4.10`.

## Evidence

- **Shipped release note matches the reported crash:** The changelog entry under the `2026.4.10` release says: `Agents/exec: prevent gateway crash ("Agent listener invoked outside active run") when a subagent exec tool produces stdout/stderr after the agent run has ended or been aborted. (#62821)`
  - file: [CHANGELOG.md:1011](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/CHANGELOG.md#L1011)
  - command: `nl -ba CHANGELOG.md | sed -n '996,1020p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Runtime now suppresses late exec updates after exit/failure:** `runExecProcess` tracks `updatesDisabled`, skips `emitUpdate()` when that flag is set, and flips it before marking the session exited or on runtime failure so queued stdout/stderr cannot call back into a disposed run.
  - file: [src/agents/bash-tools.exec-runtime.ts:561](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/agents/bash-tools.exec-runtime.ts#L561)
  - command: `nl -ba src/agents/bash-tools.exec-runtime.ts | sed -n '561,585p' && nl -ba src/agents/bash-tools.exec-runtime.ts | sed -n '770,823p'`
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Abort path disables updates before preserving background sessions:** The exec tool's abort handler now calls `run.disableUpdates()` before the `yielded || run.session.backgrounded` early return, so backgrounded/orphaned processes can keep running without sending `tool_execution_update` events into an ended run.
  - file: [src/agents/bash-tools.exec.ts:1715](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/agents/bash-tools.exec.ts#L1715)
  - command: `nl -ba src/agents/bash-tools.exec.ts | sed -n '1715,1725p'`
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Regression coverage keeps background exec alive across aborts:** Tests cover background and PTY background exec surviving tool-signal aborts, plus timeout behavior after abort, which is the edge case described in the report.
  - file: [src/agents/bash-tools.exec.background-abort.test.ts:200](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/agents/bash-tools.exec.background-abort.test.ts#L200)
  - command: `nl -ba src/agents/bash-tools.exec.background-abort.test.ts | sed -n '200,227p'`
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Gateway restart path still aborts active embedded runs on drain timeout:** The gateway run loop performs a final best-effort `abortEmbeddedPiRun(undefined, { mode: "all" })` before restart when drain times out, which aligns the restart lifecycle with the new exec-side late-update guard.

## Risks / Open Questions

- The report describes one specific orphaned-process restart path. If it still reproduces on `v2026.4.23` or newer, that would likely be a new regression or a separate late-callback path outside the exec guard added here.
