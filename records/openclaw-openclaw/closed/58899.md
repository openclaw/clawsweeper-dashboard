## Summary

Current `main` intentionally defaults single-agent gateways to `agentId = "main"`, and session storage is isolated by `OPENCLAW_STATE_DIR`/profile, not by `OPENCLAW_CONFIG_PATH`. The reported collision only occurs when multiple gateways share one state dir against the documented multi-gateway setup, so this does not reproduce as a current-main bug.

## Evidence

- **Single-agent default is intentionally `main`:** The multi-agent docs explicitly say single-agent mode defaults to `agentId` `main`, with state under `~/.openclaw/agents/main/...`.
  - file: [docs/concepts/multi-agent.md:65](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/docs/concepts/multi-agent.md#L65)
  - command: `nl -ba docs/concepts/multi-agent.md | sed -n '61,69p'`
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **Session directory is derived from state dir plus agent id:** Session transcript paths resolve as `<stateDir>/agents/<agentId>/sessions`; there is no config-path-based isolation here.
  - file: [src/config/sessions/paths.ts:14](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/config/sessions/paths.ts#L14)
  - command: `nl -ba src/config/sessions/paths.ts | sed -n '1,40p'`
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **`OPENCLAW_CONFIG_PATH` does not change the state dir:** `resolveStateDir()` only honors `OPENCLAW_STATE_DIR`; `resolveCanonicalConfigPath()` separately honors `OPENCLAW_CONFIG_PATH`. Different config files alone do not produce different session roots.
  - file: [src/config/paths.ts:60](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/config/paths.ts#L60)

  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **Current multi-gateway docs already require unique state dirs:** The multiple-gateways guide says each gateway instance must keep `OPENCLAW_STATE_DIR` unique because it owns sessions, creds, and caches.
  - file: [docs/gateway/multiple-gateways.md:121](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/docs/gateway/multiple-gateways.md#L121)
  - command: `nl -ba docs/gateway/multiple-gateways.md | sed -n '118,140p'`
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **Supported isolation flow is `--profile` or unique state dir:** The gateway runbook and CLI docs both point users to per-profile/per-state-dir isolation for multiple gateways.
  - file: [docs/gateway/index.md:161](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/docs/gateway/index.md#L161)

## Risks / Open Questions

- There is still no runtime warning when operators start multiple gateways with different `OPENCLAW_CONFIG_PATH` values but the same `OPENCLAW_STATE_DIR`, so this misconfiguration remains possible outside the documented profile/state-dir flow.
