## Summary

Current main already ships the Windows exec mojibake fix via codepage-aware decoding of command stdout/stderr, with regression coverage and a v2026.4.26 changelog entry. The PR's PowerShell preamble approach is no longer the best path because it changes command parsing semantics that the shipped decoder avoids.

## Best Possible Solution

Close this PR and keep the shipped codepage-aware Windows output decoder. If a remaining Windows PTY/profile edge case is reproduced on latest OpenClaw, handle it in a narrower follow-up rather than landing the PowerShell preamble wrapper.
