## Summary

Keep open for maintainer review. The patch is focused and I found no blocking correctness findings, but it intentionally makes post-core plugin convergence fail closed on bad managed npm `openclaw` peer links, so the upgrade availability tradeoff still needs maintainer acceptance before merge.

## What This Changes

The branch recovers stale managed npm install-record metadata, relinks managed npm `openclaw` peer dependencies during post-core convergence, and makes payload smoke validation reject missing, stale, or wrong-target peer links.

## Best Possible Solution

Land this after maintainers explicitly accept the fail-closed peer-link guard, or adjust it to a softer warning or opt-in strict mode while preserving the stale-record and peer-audit regressions.
