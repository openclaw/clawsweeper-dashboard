## Summary

Keep open: current main still drops or ignores the reasoning marker before cron announce selection, and the latest release has the same helper gap. The open fix PRs already target this exact bug, including https://github.com/openclaw/openclaw/pull/73363 with closing syntax, so this issue should stay open until one canonical fix lands.

## What This Changes

Reported Matrix/Feishu cron announce deliveries can expose model reasoning text and duplicate content when reasoning payloads lose their `isReasoning` marker.

## Best Possible Solution

Land one canonical fix that preserves `isReasoning` through embedded payload construction and treats reasoning as non-user-visible across cron summary, output, delivery, synthesized fallback, and post-error recovery, then close this issue after merge.

## Reproduction Assessment

Yes. Source inspection on current main shows reasoning blocks become internal payload items, the returned payload drops `isReasoning`, and cron announce selectors choose text using content and `isError` checks only.

## Solution Assessment

Yes. The shared embedded-payload and cron outcome helpers are the right fix boundary; channel-only Matrix or Feishu filtering would leave other cron announce selectors exposed.
