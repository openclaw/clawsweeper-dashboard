## Summary

Close as implemented on current main. WhatsApp now has a channel/account-scoped `exposeErrorText` option; when it is set to `false`, `isError` reply payloads are filtered before `deliverWebReply`, and embedded-runner tool-error warnings are emitted as `isError` payloads. The proof commit is on main after `v2026.4.26`, so this appears fixed on main but not yet in a tagged release.

## What This Changes

The issue requests a config-level policy to keep tool failure warning payloads from being delivered as WhatsApp replies to external recipients.

## Best Possible Solution

Keep the channel-scoped WhatsApp error visibility option and document it as the supported privacy control; separate work such as #46548 can continue to improve operator-facing tool-error wording where errors are intentionally visible.
