## What This Changes

The issue reports that plugin-skill publishing calls `fs.readlinkSync` on managed entries without confirming they are symlinks, causing `EINVAL` warnings and leaving stale real directories in place on Windows-style setups.

## Best Possible Solution

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable fix: inspect the existing entry with `lstat`, replace generated managed directory entries, and keep non-generated entries untouched.
