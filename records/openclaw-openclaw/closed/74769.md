## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Define artifact locator params as one closed object:** `src/gateway/protocol/schema/artifacts.ts:13-21`
  - body: `Type.Intersect` emits an `allOf`, and both operands here set `additionalProperties: false`. Ajv will reject real `artifacts.get` and `artifacts.download` params because `artifactId` is additional to the query schema and `sessionKey`/`runId`/`taskId` are additional to the id schema. Build one closed object for these params.
  - confidence: 0.96
- **[P2] Include image_url transcript blocks:** `src/gateway/server-methods/artifacts.ts:155-160`
  - body: Current WebChat media code can store generated images as `{ type: "input_image", image_url: "data:..." }`, but this detector ignores `image_url`. Those transcript images will be absent from `artifacts.list` and unavailable to clients through the new API.
  - confidence: 0.9
- **[P2] Regenerate tracked protocol clients:** `src/gateway/protocol/schema/protocol-schemas.ts:345-351`
  - body: Adding these entries changes the protocol schema set, but the tracked Swift `GatewayModels.swift` outputs are not updated. The repo protocol check regenerates those files, so native protocol clients and the protocol gate will be stale until the generated models are included.
  - confidence: 0.88
- **[P2] Use real RequestFrame shapes in handler tests:** `src/gateway/server-methods/artifacts.test.ts:67`
  - body: The new handler tests pass request frames without `type: "req"` and with numeric ids, while `RequestFrame` requires `type: "req"` and a non-empty string id. This makes the test fixtures diverge from the Gateway contract and should fail the test typecheck lane.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The PR adds a read-scoped artifact download surface and currently treats any non-data URL-like transcript value as downloadable.

Concerns:

- **[medium] Validate artifact URL download sources:** `src/gateway/server-methods/artifacts.ts:129`
  - body: `resolveBlockDownload` classifies any non-data `url`, `openUrl`, or `source.url` as a URL download. Since this reads raw transcript blocks, unsupported schemes or local-path-like values could be surfaced to read-scoped clients despite the stated boundary that unsupported local-only sources return typed unsupported errors.
  - confidence: 0.82
