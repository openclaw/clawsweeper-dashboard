## Summary

Current `main` already removes the blocking Discord listener path that would stall slash/button interactions behind long-running inbound message work, and it adds regression tests for the non-blocking behavior.

## Evidence

- **Message listener is now non-blocking:** `DiscordMessageListener.handle()` fire-and-forgets the handler and explicitly documents that the Carbon listener no longer serializes or applies its own timeout.
