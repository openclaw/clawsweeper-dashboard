## Summary

Current `main` already addresses this Linux/systemd-user-session failure mode by detecting `XDG_RUNTIME_DIR`/user-bus errors, retrying via machine user scope when possible, and emitting explicit remediation hints when systemd user services are still unavailable.

## Evidence

- **`XDG_RUNTIME_DIR` failures are classified explicitly:** The systemd-unavailable classifier treats `dbus_session_bus_address` / `xdg_runtime_dir` / user-bus connection failures as `user_bus_unavailable`, which is the exact failure family from the report.
  - file: [src/daemon/systemd-unavailable.ts:23](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/daemon/systemd-unavailable.ts#L23)
  - command: `nl -ba src/daemon/systemd-unavailable.ts | sed -n '1,220p'`
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Gateway install/status code now retries through machine user scope:** `execSystemctlUser` first tries `systemctl --user`, then falls back to `systemctl --machine <user>@ --user` for user-bus-unavailable errors. That avoids depending solely on the current shell's `XDG_RUNTIME_DIR` and prevents the later misleading `enable` failure path when machine scope is reachable.
  - file: [src/daemon/systemd.ts:357](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/daemon/systemd.ts#L357)

- **User-facing remediation now includes the exact missing-session fixes:** Headless Linux hints now tell users to run `sudo loginctl enable-linger $(whoami)` and `export XDG_RUNTIME_DIR=/run/user/$(id -u)`, matching the issue's requested remediation.
  - file: [src/daemon/systemd-hints.ts:17](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/daemon/systemd-hints.ts#L17)
  - command: `nl -ba src/daemon/systemd-hints.ts | sed -n '1,220p'`

- **The improved hints are surfaced in CLI status/doctor flows:** Daemon status printing and doctor formatting both route Linux systemd-unavailable runtime details through the new hint renderer, so users now get actionable guidance instead of only a cryptic systemctl failure.
  - file: [src/cli/daemon-cli/status.print.ts:231](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/cli/daemon-cli/status.print.ts#L231)
