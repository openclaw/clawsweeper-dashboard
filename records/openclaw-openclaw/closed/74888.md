## What This Changes

The PR updates capability provider fallback load options so a source config with `plugins.enabled: false` passes `installBundledRuntimeDeps: false`, and adds regression tests for list and targeted provider fallback paths.

## Best Possible Solution

Keep the narrow guard that prevents bundled runtime-dependency staging when the original config globally disables plugins, with regression coverage for both capability-provider list and targeted lookup paths; decide separately whether compatibility fallback should also avoid runtime module imports under global disablement.

## Reproduction Assessment

## Solution Assessment

Yes for the dependency-repair regression as scoped. The PR preserves existing compatibility fallback behavior while preventing the package-manager/runtime-dependency repair side effect under explicit global disablement; a stricter no-load policy would be a broader follow-up.
