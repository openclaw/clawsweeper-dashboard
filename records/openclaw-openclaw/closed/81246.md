## What This Changes

The PR updates the WhatsApp Web inbound listener to drain queued debounced messages and in-flight upsert work before socket close, with regression coverage for replies/media sends before `sock.end()`.

## Best Possible Solution

Land the scoped WhatsApp listener drain after normal maintainer CI/review; if it lands, use it to supersede the older broad `flushAll()` proposal at https://github.com/openclaw/openclaw/pull/63038.

## Reproduction Assessment

## Solution Assessment

Yes. The listener-local drain is narrower than adding a shared debouncer `flushAll()` API and addresses the known late-enqueue/active-flush gap from https://github.com/openclaw/openclaw/pull/63038 by tracking pending handlers, pending keys, and active flushes.
