## Summary

Close as a duplicate of #73421. Both reports describe the same OpenClaw 2026.4.26 gateway/control-ui stall pattern: WebSocket reconnects repeatedly hit `handshake-timeout` / `closed before connect` before eventually recovering. Current main still has the same Gateway pre-auth timeout and Control UI reconnect surfaces, and I did not find a separate nginx-specific merged fix, so the remaining investigation should stay centralized in #73421.
