## Summary

Keep this PR open. Current main still does not implement the PR’s intended Swabble features: logging is text-only, status/health are local/simple outputs, config lacks the redaction field, and `serve` has no control socket or in-process config reload. The PR remains an implementation candidate, but it is not ready to treat as obsolete or already handled because the proposed control socket/reload work has unresolved correctness and security review findings.

## Best Possible Solution

Keep this PR open as the active implementation candidate, but require a rebase onto current main, resolution of the control socket security/correctness review comments, targeted Swabble tests for logging/status/health/reload behavior, and real macOS validation. If maintainers prefer smaller changes, split this into focused PRs for structured logging/redaction, status-health output, and the local control socket/reload path.

## Risks / Open Questions

- The PR adds a local control plane that can expose recent transcripts and mutate runtime behavior, so owner-only socket permissions, descriptor hygiene, SIGPIPE/EAGAIN handling, and timeout behavior need to be resolved before merge.
- The reload-config feature currently appears semantically broad; review comments indicate some speech pipeline settings would not actually update in-process, so the command may report success while leaving runtime behavior stale.
- The PR is XL and touches logging, config, transcript persistence, status/health output, and socket control together; maintainers may want it split into smaller, validated Swabble slices with targeted Swift tests and macOS runtime proof.
