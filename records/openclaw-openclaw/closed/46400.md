## Summary

Current main already implements the reported timeout inheritance path: embedded LLM idle timeouts fall back to `agents.defaults.timeoutSeconds`, and subagent run timeout is a separate `agents.defaults.subagents.runTimeoutSeconds` control. This behavior is covered by tests, documented, and recorded in the 2026.4.9 changelog.

## Evidence

- **Idle-timeout resolver inherits agent default timeout:** `resolveLlmIdleTimeoutMs` checks `cfg.agents.defaults.llm.idleTimeoutSeconds`, then `runTimeoutMs`, then falls back to `cfg.agents.defaults.timeoutSeconds`, matching the issue's requested behavior for slow subagent model calls.
