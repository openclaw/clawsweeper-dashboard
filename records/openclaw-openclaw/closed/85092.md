## What This Changes

The issue reports that env-scoped `openclaw infer model auth logout --provider` returns `removedProfiles` for non-default agents without persisting the scoped `auth-profiles.json`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/85129 or an equivalent fix that selects the env-scoped agent directory, mutates the locked local store, and reports only profiles actually removed from that store.

## Reproduction Assessment

Yes. The report gives concrete env-scoped logout steps, and current main source shows logout resolving the configured default agent dir while runtime profile loading can include merged profiles.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable fix: select the env-scoped agent dir before logout, mutate the locked local store, and report only profiles removed from that store.
