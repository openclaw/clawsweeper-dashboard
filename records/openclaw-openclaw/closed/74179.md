## Summary

Current main and v2026.4.26 already solve the reported multi-Feishu completion announce account problem by preserving the parent/requester origin separately from the child-session delivery origin and by requiring bound completion routes to match the requester channel/account. This PR’s intended observable fix is therefore already shipped, and the PR branch is not safe to merge as-is because it removes the local `router` declaration before calling `router.resolveDestination`.

## What This Changes

This PR changes `resolveSubagentCompletionOrigin` to resolve the requester/parent session binding before falling back to the child session binding for subagent completion announce routing.

## Best Possible Solution

Close this PR as already implemented by the shipped v2026.4.26 requester-origin fix and do not merge the branch. If maintainers later want an exact parent-first lookup refactor, it should be a separate tested change that keeps the router declaration and proves behavior beyond the already-fixed cross-account bug.
