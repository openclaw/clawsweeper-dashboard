## Solution Assessment

Yes as an acceptable runtime mitigation if maintainers accept the diagnostic tradeoff. Once an old process has loaded chunks from a replaced dist tree, build-only hardening cannot repair that in-memory module graph, so a scoped clean restart is the maintainable response.
