## Summary

The report is superseded by the shipped group-visible-reply policy work: normal Telegram group/channel requests now default to automatic final delivery, while explicit `message_tool` and ambient `room_event` modes intentionally keep normal finals private and log diagnostics instead of using the rejected fallback.

## What This Changes

The issue asks OpenClaw to deliver Telegram group assistant text when a model replies without calling the `message` tool.

## Best Possible Solution

Keep the shipped automatic default for normal group/channel requests and the strict opt-in `message_tool`/ambient-room contract; operators who want visible plain finals should use `messages.groupChat.visibleReplies: "automatic"`.

## Reproduction Assessment

Yes for the old/tool-only path from source inspection: unmarked finals are suppressed in `message_tool_only` mode. The default-settings reproduction no longer matches current main because normal group/channel requests now resolve to automatic delivery outside ambient room events.

## Solution Assessment

Yes. The maintainable shipped resolution is the default reversal plus explicit strict tool-only policy; the same-turn fallback proposed in the linked PR was closed unmerged and is not the accepted direction.
