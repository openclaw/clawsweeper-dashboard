## Summary

Current main and the latest shipped release already align doctor validation with the public config schema for the reported compaction fields, so the older 2026.4.x stripping behavior is obsolete.

## What This Changes

The issue asks for `openclaw doctor --non-interactive` to preserve valid `agents.defaults.compaction` fields instead of stripping them from `openclaw.json`.

## Best Possible Solution

Keep the current v2026.5.28 schema contract and direct affected users on older 2026.4.x/2026.3.x builds to update instead of adding another doctor compatibility path.

## Reproduction Assessment

No current-main repro remains. Source and shipped-release inspection show the reported fields are now accepted by the same Zod schema doctor uses for unknown-key stripping.

## Solution Assessment

Yes. The best fix is the current single canonical schema path: `config.schema.lookup` is built from `OpenClawSchema`, and doctor strips only keys that fail that schema rather than maintaining a separate compaction allowlist.
