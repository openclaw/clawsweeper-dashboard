## Summary

Current main fixes the reported destructive global-npm path by moving Matrix out of the root package and through the official external plugin install flow, which uses OpenClaw-managed plugin roots instead of the global OpenClaw node_modules tree.

## What This Changes

The issue reports that the Matrix dependency bootstrap in OpenClaw 2026.5.7 can run npm install from a global node_modules directory and delete the global OpenClaw install.

## Best Possible Solution

Keep Matrix on the official external plugin path and carry that fix into the next stable release so root OpenClaw installs no longer repair Matrix dependencies from the global package tree.

## Reproduction Assessment

Yes. The v2026.5.7 package contents and deps.ts bootstrap path make the reported failure source-reproducible, but I did not execute the destructive global npm-prefix scenario.

## Solution Assessment

Yes. Externalizing Matrix and using OpenClaw-managed plugin install roots is the narrow maintainable fix for the reported global node_modules deletion path.
