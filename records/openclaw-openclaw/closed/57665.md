## Summary

Current `main` already routes CLI/runtime version reporting through the shared version resolver and has regression tests covering stale env-version fallbacks, so this PR’s one-line `__OPENCLAW_VERSION__ !== "0.0.0"` guard is no longer needed to address the reported behavior.
