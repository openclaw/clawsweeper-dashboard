## Summary

Keep open: the reported defect is credible and the proof is sufficient, but this patch changes every transient WebSocket close into cancellation even though current chat and agent paths explicitly support same-session late joining after refresh or reconnect.

## What This Changes

The PR aborts chat abort-controller entries owned by a closing Gateway WebSocket connection and adds a focused unit test plus an operator log line.

## Best Possible Solution

Define disconnect ownership semantics, then implement a grace or rebind-aware cleanup path that cancels truly abandoned runs without breaking reconnectable active runs.

## Reproduction Assessment

Yes. Current source shows owned abort-controller entries exist and the close handler does not consume them, while the PR body supplies production-bundle before/after log proof for the disconnect path.

## Solution Assessment

No, not as written. The existing abort helper is the right primitive, but firing it on every socket close should be replaced with explicit cancellation semantics or a grace/rebind-aware path.
