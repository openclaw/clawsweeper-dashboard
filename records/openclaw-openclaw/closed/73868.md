## Summary

This issue is a duplicate of the already-open stuck-session recovery thread. Current main has diagnostic stuck-session warnings and a conservative stale lane-release helper, but it still does not implement the requested configurable server-side auto-fail/active abort behavior; that remaining work is already tracked by #71127, with #73243 open as the active implementation candidate.

## Best Possible Solution

Close this issue as a duplicate of #71127 and keep review/implementation focused on #73243 or a replacement PR that adds a conservative configurable stuck-session abort policy, telemetry, docs/schema updates, and regression coverage.
