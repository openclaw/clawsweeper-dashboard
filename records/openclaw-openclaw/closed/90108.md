## Summary

Keep open: current main still preserves nonblank provider thinking signatures through compaction successor transcript construction and outbound Anthropic replay, and the latest release has the same behavior reported by the user.

## What This Changes

The issue requests that compaction stop replaying stale Anthropic extended-thinking signatures after preserved messages move into a new post-compaction context.

## Best Possible Solution

Strip or downgrade provider-bound thinking signatures when compaction writes preserved assistant messages into a successor context, while preserving visible assistant text and adding a focused regression test for signed Anthropic thinking after compaction.

## Reproduction Assessment

Yes, at source level: a compacted session whose retained tail includes an Anthropic assistant `thinking` block with a nonblank `thinkingSignature` will keep that signature through successor transcript construction and outbound replay. I did not run a live Anthropic compaction session in this read-only pass.

## Solution Assessment

Yes, the narrowest maintainable fix is in the compaction successor/replay boundary, not a new config knob. The existing generic recovery wrapper is useful hardening, but it reacts after an API rejection and leaves the bad replay state intact.
