## Summary

Keep open: current main still forwards `temperature` to the direct Anthropic Messages payload for adaptive-thinking Claude models when reasoning is absent, while the linked implementation PR #76871 is closed unmerged. Anthropic's migration guide says Opus 4.7 requests should omit `temperature`, `top_p`, and `top_k` because non-default sampling parameters return 400. ([platform.claude.com](https://platform.claude.com/docs/en/docs/agents-and-tools/tool-use/token-efficient-tool-use))

## What This Changes

The issue asks the gateway to omit unsupported sampling parameters when routing Anthropic reasoning/adaptive models such as Claude Opus 4.7 and Claude Sonnet 4.6.

## Best Possible Solution

Land a narrow direct Anthropic transport fix with regression coverage and a changelog entry, preserving temperature for models that still support it and only changing OpenRouter if current wrapper evidence shows OpenClaw must sanitize it.

## Reproduction Assessment

Yes at source level. A configured `params.temperature` flows into the Anthropic transport and is emitted for adaptive Claude models whenever no reasoning option is requested; I did not run a live Anthropic request in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to gate sampling-param forwarding on the existing adaptive/model capability path for the direct Anthropic Messages payload, with tests proving adaptive models drop `temperature` and non-adaptive models keep it.
