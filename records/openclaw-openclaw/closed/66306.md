## Summary

Current `main` contains a new macOS health-state implementation that consumes gateway health responses, derives non-pending UI states, and adds focused regression coverage for health snapshot decoding and degraded/linked state handling. That matches the reported stuck-`Health check pending` symptom closely enough to close as already implemented on `main`, but it does not appear to be in the latest tagged release yet.

## Evidence

- **Health refresh now stores decoded gateway health snapshots:** `HealthStore.refresh` calls `ControlChannel.shared.health`, decodes the payload, updates `snapshot`, clears `lastError`, and logs recovery/failure transitions. This is the core path that moves the app out of a pending state when the gateway is healthy.
  - file: [apps/macos/Sources/OpenClaw/HealthStore.swift:115](https://github.com/openclaw/openclaw/blob/30aa7e0d4d225650b58c2fba79e71bffa5627b08/apps/macos/Sources/OpenClaw/HealthStore.swift#L115)
