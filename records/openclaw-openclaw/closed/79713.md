## Summary

Keep open: current main still returns no reseed messages for uncompacted CLI sessions, and the issue is paired with an open closing PR plus maintainer guidance that the fix should preserve the no-raw default or make raw replay explicit.

## What This Changes

The issue asks for invalidated CLI sessions without compaction to provide a bounded historyPrompt from previous turns instead of restarting with no prior context.

## Best Possible Solution

Keep this issue open while maintainers review https://github.com/openclaw/openclaw/pull/79764 or a safer default-off/schema-documented opt-in, with tests that preserve the no-raw default and prove the chosen recovery path.

## Reproduction Assessment

Yes from source: current main returns no reseed messages for uncompacted sessions, so `buildCliSessionHistoryPrompt` has no rendered history and fresh invalidated CLI runs log `historyPrompt=none`. I did not live-run the idle-reset flow in this read-only pass.

## Solution Assessment

No for the unconditional raw-tail fallback: it is narrow, but it conflicts with the existing no-raw-pre-compaction safety test. The safer maintainable path is an explicit backend opt-in with schema/docs/tests, or a summary-on-invalidation design chosen by maintainers.
