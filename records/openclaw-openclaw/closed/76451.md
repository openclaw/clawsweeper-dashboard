## Summary

Current main already implements the missing external channel secret-contract discovery path, and the fix directly covers SecretRef-backed external channel credentials after the plugin split. The fix is on main only in this checkout, not in the latest provided v2026.5.2 release.

## What This Changes

The issue reports that SecretRef values under `channels.feishu.*` stopped resolving after Feishu moved from bundled channel code to the external `@openclaw/feishu` plugin.

## Best Possible Solution

Keep the generic external channel secret-contract loader on main and ship it in the next OpenClaw release so Feishu and other externalized channels resolve SecretRefs consistently.

## Reproduction Assessment

No current-main failing reproduction was established because the checkout already contains the fix. The 2026.5.2 failure path is source-backed by the parent collector, and current main has targeted source and regression-test coverage for the external channel SecretRef path.

## Solution Assessment

Yes. The current implementation solves this at the generic channel secret-contract boundary instead of special-casing Feishu, and it preserves bundled/bootstrap fallback behavior.
