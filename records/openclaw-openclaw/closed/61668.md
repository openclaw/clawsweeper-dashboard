## Summary

Close as duplicate/superseded. The requested fail-closed structured-output gate is a specific instance of the older, still-open outbound policy boundary tracked in https://github.com/openclaw/openclaw/issues/56349, which already covers validation, modification, and blocking across agent replies, auto-replies, tool sends, and channel adapters.

## What This Changes

The issue requests a universal fail-closed validation gate for structured operator-facing replies before native dispatch reaches channel delivery.

## Best Possible Solution

Track the remaining validation contract under https://github.com/openclaw/openclaw/issues/56349 and use this issue as supporting evidence for structured operator-facing payloads.

## Reproduction Assessment

Yes, at source level. A ReplyDispatcher delivery spy or direct message(action=send)/deliverOutboundPayloads path can reach normalization, hooks, and delivery without a shared fail-closed structured-payload validator.

## Solution Assessment

No, a dispatcher-only validator is too narrow because direct outbound and message-tool paths bypass that seam. The maintainable solution is the canonical outbound policy boundary tracked in https://github.com/openclaw/openclaw/issues/56349.
