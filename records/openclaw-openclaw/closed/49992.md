## Summary

Current main already covers this well enough: provider-scoped model setup preserves unrelated allowlist entries, the UI model picker is sourced from the allowlisted model catalog, and `sessions.patch` accepts explicit provider/model overrides. The reported `model not allowed` failure now matches documented allowlist behavior rather than a UI switching bug.

## Evidence

- **Configure flow preserves other provider models in the allowlist:** Provider-scoped auth/setup applies `applyModelAllowlist(..., { scopeKeys })`, so adding or updating one provider no longer drops unrelated model entries from `agents.defaults.models`. That addresses the reported "only the primary method is allowed" behavior.
