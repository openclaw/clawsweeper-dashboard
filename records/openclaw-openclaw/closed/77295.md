## Summary

Keep open: current main still has the Fal provider behavior reported here, and #77388 is an open implementation PR with closing syntax that should be reviewed or landed before this issue closes.

## What This Changes

The issue requests routing Fal GPT Image 2 and Nano Banana image edits to `/edit`, using `image_urls`, and allowing the vendor-supported edit capabilities instead of the Flux-only edit contract.

## Best Possible Solution

Review and land #77388 or an equivalent Fal plugin-local fix with regression coverage, docs, and a changelog entry, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes. Source inspection on current main traces `fal/openai/gpt-image-2` to the Fal provider, where edit requests still append `/image-to-image`, send `image_url`, and reject multi-image/aspect-ratio edit paths; I did not run a live Fal request.

## Solution Assessment

Yes. A provider-local, model-family-specific Fal fix is the narrowest maintainable direction because GPT Image 2/Nano Banana edit contracts differ from Flux while the shared image runtime can stay generic.
