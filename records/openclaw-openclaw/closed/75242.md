## Summary

Close as ClawHub/plugin scope. The PR adds a new bundled Vesicle iMessage channel, while current main has no Vesicle implementation, current project docs direct optional channel integrations to external plugins, and the existing channel-plugin SDK covers the config, outbound, webhook, secret, SSRF, and runtime seams used by this branch.

## What This Changes

The PR adds a bundled `@openclaw/vesicle` channel plugin for Vesicle native iMessage health probes, chat-GUID text sends, and signed inbound webhooks, plus channel docs, secret metadata, generated config metadata, lockfile entries, and an install smoke script.

## Best Possible Solution

Keep OpenClaw core unchanged for Vesicle and publish the integration as an external channel plugin through ClawHub or npm. If external publication exposes a missing generic channel-plugin API, bring back that narrow SDK gap as a plugin-neutral core PR.
