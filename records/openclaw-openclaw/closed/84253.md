## Summary

Keep this open: current main still has the permissive debug proxy numeric parser and SQLite-bound runtime path, and the linked fixing PR is open rather than merged. The issue should close only after that PR, or an equivalent parser-boundary fix, lands.

## What This Changes

Reject invalid debug proxy numeric CLI options like `proxy sessions --limit abc` at parse time instead of passing them through to SQLite.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84260, or an equivalent parser-boundary fix with focused regression coverage, then close this issue as implemented after merge.

## Reproduction Assessment

Yes. Source inspection shows invalid `--limit` can become a non-nullish value that bypasses the default and is bound into SQLite `LIMIT ?`; the reporter and linked PR also include terminal before-fix proof, though this read-only review did not execute the CLI.

## Solution Assessment

Yes. Rejecting malformed numeric values in `src/cli/proxy-cli.ts` before runtime dispatch is the narrowest maintainable fix, with https://github.com/openclaw/openclaw/pull/84260 already covering that path.
