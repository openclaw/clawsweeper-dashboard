## What This Changes

The PR adds execution-identity hashing and a scoped running-session lookup so repeated identical background/yield `exec` calls reuse a matching in-flight process session instead of spawning another child process.

## Best Possible Solution

Land this runtime-level coalescing after maintainers accept the compatibility tradeoff, or add an explicit duplicate-launch escape hatch if independent identical background runs must remain supported.

## Reproduction Assessment

Yes, as a source-level reproduction: current main sends repeated background-capable `exec` calls through `runExecProcess` without checking for an equivalent running session first. I did not run a live Xiaomi/MiMo replay in this read-only review.

## Solution Assessment

Yes, this is the right code layer because it fixes the generic exec/process runtime boundary with exact resolved execution identity instead of provider-specific policy or fuzzy text matching. The remaining question is maintainer acceptance of the compatibility tradeoff.
