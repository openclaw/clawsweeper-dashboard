## Summary

Close as superseded by the all-channel group visible-reply cleanup tracked in #74876. The latest discussion confirms the shipped `messages.groupChat.visibleReplies: "automatic"` setting fixes the visible Telegram group reply symptom on 2026.4.29+, while current Telegram directory behavior is intentionally config-backed rather than automatic observed-group enrollment.

## What This Changes

The report asks for Telegram group chats to be discovered, ingested, and visibly answered like direct messages, including group listing through the directory CLI.

## Best Possible Solution

Keep Telegram group authorization explicit through `channels.telegram.groups`, use `messages.groupChat.visibleReplies: "automatic"` for operators who want legacy visible group replies, and track any future automatic observed-group discovery as a separate product/security decision rather than reopening this support-style report.

## Reproduction Assessment

Yes. The code-level path is high confidence: group/channel turns default to message-tool-only visible delivery unless `messages.groupChat.visibleReplies` is `automatic`, and the Telegram directory lister only reads configured groups. The latest live comments confirm the same config fixes the user-visible Telegram group reply symptom.

## Solution Assessment

Yes. Superseding this through the shipped all-channel visible-reply configuration is the narrowest maintainable outcome; automatically discovering or authorizing observed Telegram groups would widen the group access boundary and should be handled separately if maintainers want that feature.
