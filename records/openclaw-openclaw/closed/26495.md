## Summary

#26495 is a valid integration idea, but current main explicitly routes optional integrations like this to external ClawHub/community plugins rather than OpenClaw core. The repo vision says optional capability should usually ship as plugins, new skills should go to ClawHub first, and new core skills that can live on ClawHub are out of scope for core.

## Close Comment

Closing after a Codex review because this request fits the ClawHub/community plugin path rather than OpenClaw core. Current main is explicit that optional capability should usually ship as plugins, that new skills should be published to ClawHub first, and that new core skills that can live on ClawHub are not on the core merge path (`VISION.md:54`, `VISION.md:69`, `VISION.md:101`). The current docs and runtime also already support plugins that ship their own `skills` directories (`docs/tools/skills.md:80`, `docs/plugins/building-plugins.md:16`, `src/plugins/bundle-manifest.ts:138`). For an Intel AI Super Builder bridge, the intended path is a ClawHub/npm plugin or skill package rather than a bundled OpenClaw core feature. If a concrete SDK seam is missing for that external implementation, please open a focused follow-up issue for the missing seam.
