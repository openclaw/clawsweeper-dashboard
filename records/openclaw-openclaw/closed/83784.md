## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Route typing keepalives through the Telegram handler:** `extensions/telegram/src/native-tool-progress-draft.ts:183-189`
  - body: This new keepalive calls `bot.api.sendChatAction` directly, while the existing Telegram path injects `sendChatActionHandler` to coalesce typing and suspend repeated 401s. With an invalid or revoked token, native draft keepalives can bypass that circuit breaker and keep calling Telegram every few seconds, so pass the guarded typing seam into this helper or disable the keepalive until it can use that path.
  - confidence: 0.93
- **[P2] Preserve fallback when the first native draft send fails:** `extensions/telegram/src/native-tool-progress-draft.ts:383-385`
  - body: `update()` returns `true` immediately after scheduling the first `sendMessageDraft`, so `pushStreamToolProgress` treats progress as delivered and skips the edited-preview fallback. If Telegram rejects that first native draft, the user gets no progress fallback until final delivery; make the initial success path await acceptance or otherwise report failure back to the caller.
  - confidence: 0.91
