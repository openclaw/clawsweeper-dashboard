## Summary

## What This Changes

The issue requests Android recovery from a stored Gateway TLS fingerprint mismatch after reverse-proxy certificate renewal without clearing app data.

## Best Possible Solution

Keep the shipped Android user-confirmed TLS thumbprint replacement flow and handle any future UX refinements as new, narrower follow-ups.

## Reproduction Assessment

No current-main failure remains: the pre-fix path is high-confidence from source and tests, but current main now turns a stored old fingerprint plus presented new fingerprint into an explicit trust prompt instead of a terminal mismatch.

## Solution Assessment

Yes. The shipped fix is the narrow maintainable path because it preserves fail-closed pinning until the user explicitly accepts the probed replacement fingerprint, rather than silently updating or disabling pinning.
