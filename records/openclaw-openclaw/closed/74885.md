## Summary

Keep open. Current main still has the native Slack block/presentation-control collision shape described by this PR, and the proposed patch is a narrow Slack plugin fix with matching regression coverage and no blocking review finding.

## What This Changes

This PR threads Slack interactive block offset options into presentation block rendering, offsets presentation controls against native Slack blocks in the outbound adapter, and adds an outbound payload regression test.

## Best Possible Solution

Review and land the narrow Slack plugin fix after normal CI/changed-gate validation, keeping the offset behavior in Slack block rendering rather than moving Slack-specific block-id policy into core.

## Solution Assessment

Yes. The PR uses the existing Slack offset helper and applies it at the two render entry points that can see native blocks, which is the narrowest maintainable fix for this Slack-specific renderer collision.
