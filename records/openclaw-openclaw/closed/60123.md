## Summary

Current `main` already implements configurable embedded Pi LLM idle timeout handling. The runtime now reads `agents.defaults.llm.idleTimeoutSeconds`, falls back to `agents.defaults.timeoutSeconds`, and disables the default idle watchdog for cron runs when no timeout is configured, which covers the reported cron/embedded-run timeout path.

## Evidence
