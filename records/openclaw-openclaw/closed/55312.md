## Summary

Current `main` no longer shows the reported configure/runtime mismatch. The model picker now writes the exact `provider/model` key from the catalog, and the OpenAI Codex runtime’s canonical legacy GPT-5.3 id is `openai-codex/gpt-5.3-codex`, not `openai-codex/gpt-5.3`.

## Evidence

- **Model picker emits exact provider/model keys:** `addModelSelectOption` builds `key = modelKey(provider, id)` and uses that exact string for both `value` and `label`, so the picker surface now exposes the runtime key directly rather than a prettified alternate id.

  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)

  - file: [src/flows/model-picker.ts:538](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/flows/model-picker.ts#L538)
