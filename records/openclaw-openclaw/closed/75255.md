Xiaomi native web_search is a real upstream capability, and the reporter added useful working details, but current OpenClaw policy and SDK docs route optional vendor search/provider behavior through plugins unless a missing core API is identified.

## What This Changes

Request support for Xiaomi MiMo’s native provider-level web_search tool in OpenClaw’s Xiaomi provider.

## Best Possible Solution

Move this to a ClawHub or npm Xiaomi plugin that either registers a Xiaomi-backed web_search provider or an alternate Xiaomi model provider with a provider-owned wrapStreamFn; open a narrower SDK issue only if that implementation exposes a missing plugin API.

## Solution Assessment
