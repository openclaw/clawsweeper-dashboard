## Summary

Current main still reserves `mode:"session"` for thread-bound conversations, but the reported headless orchestration blocker no longer matches behavior: spawned sessions default to `cleanup:"keep"`, remain available until auto-archive, and can be re-entered with `sessions_send` against the existing `sessionKey`. That covers the WhatsApp follow-up/session-retention use case without thread binding.

## Evidence

- **Spawned sessions default to keep:** `sessions_spawn` defaults `cleanup` to `"keep"` when the caller does not specify cleanup.
  - file: [src/agents/tools/sessions-spawn-tool.ts:192](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/agents/tools/sessions-spawn-tool.ts#L192)

  - file: [src/agents/subagent-registry-run-manager.ts:304](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/agents/subagent-registry-run-manager.ts#L304)
