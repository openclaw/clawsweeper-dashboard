## Summary

Keep open: current main still lacks session-store recovery, but this PR remains incorrect because it can restore stale artifacts for missing or valid empty primaries, reads backup artifacts before validation, accepts fresh temp files, and has fs-safe recovery tests using the wrong temp prefix.

## What This Changes

The PR adds load-time session-store recovery from `.bak` and temp artifacts, self-healing writeback, recovery tests, and a changelog entry.

## Best Possible Solution

Keep recovery at the session-loader boundary, but trigger it only for an existing corrupt primary store, validate every candidate before reading, reject fresh temps, and cover the real fs-safe temp prefix.

## Reproduction Assessment

Yes. Source inspection shows current main returns the initialized empty store after a zero-byte or malformed `sessions.json`, while the PR’s own recovery tests describe the failing path with sibling `.bak` or temp artifacts.

## Solution Assessment

No. The loader boundary is the right place, but this PR head is not the best fix until recovery is gated to corrupt existing primaries and all recovery artifacts use the same trust and freshness checks.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P2] Gate recovery to corrupt existing primary stores:** `src/config/sessions/store-load.ts:176`
  - body: The gate still includes `Object.keys(store).length === 0`, and missing files also set `primaryReadFailed`, so a valid empty `{}` store or absent primary can restore stale `.bak` state. Split this into an explicit existing-corrupt/zero-byte/non-object primary signal before trying recovery.
  - confidence: 0.98
- **[P2] Validate backup artifacts before parsing:** `src/config/sessions/store-load.ts:182-183`
  - body: `sessions.json.bak` is read and parsed before any `lstat`, regular-file, symlink, or hardlink validation. Since this imports private session metadata as trusted recovery state, apply the same candidate validation before reading the backup file.
  - confidence: 0.93
- **[P2] Reject fresh temp recovery candidates:** `src/config/sessions/store-load.ts:218-221`
  - body: Temp candidates are accepted solely by name and file shape, then sorted oldest-first. If the only matching artifact is from an active writer, a transient primary read failure can import an in-flight temp despite the PR’s stated freshness mitigation.
  - confidence: 0.88
- **[P2] Use the actual fs-safe temp prefix in recovery tests:** `src/config/sessions/store-load.recovery.test.ts:87-88`
  - body: These fixtures use `.sessions.json.<pid>.<uuid>.tmp`, but the pinned fs-safe atomic helper creates `.fs-safe-replace.<pid>.<uuid>.tmp` and the loader regex matches that prefix. The advertised fs-safe recovery tests currently miss the intended branch.
  - confidence: 0.96
