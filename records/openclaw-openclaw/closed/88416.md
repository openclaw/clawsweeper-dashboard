## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Resolve omitted accounts through channel defaults:** `src/agents/subagent-spawn.ts:782-786`
  - body: When `requester.accountId` is omitted, this path resolves only `channels.<id>.defaultAccount ?? "default"` before probing the binding adapter. Discord and Matrix already support channel-owned default selection, including a single configured named account, so configs like `channels.discord.accounts.work` without `channels.discord.defaultAccount` can register the adapter under `work` while core rejects `thread: true` against `default`. Use the channel plugin default-account contract instead of the raw config fallback.
  - confidence: 0.9
- **[P1] Preserve the bound Discord child thread:** `src/agents/subagent-spawn.ts:956-959`
  - body: For Discord, the generic binding projection produces `to: channel:<child-thread>` but no child `threadId` because the Discord plugin does not expose `messaging.resolveDeliveryTarget`. The later same-channel merge keeps the requester thread id from `childSessionOrigin`, so a child session spawned from an existing Discord thread can deliver its first run back to the requester thread instead of the newly bound child thread. Add a Discord route projection or otherwise set the bound conversation id as the delivery `threadId`.
  - confidence: 0.88
- **[P1] Preserve the deprecated spawning fallback:** `src/agents/subagent-spawn.ts:905-909`
  - body: This PR records `subagent_spawning` as deprecated and available through 2026-08-30, but the new spawn path returns core-binding errors without invoking registered `subagent_spawning` hooks. Installed third-party channel plugins that still rely on the documented hook lose `thread: true` support immediately during the deprecation window. Keep the hook runner as a fallback when core binding cannot prepare the target, or make the breaking removal an explicit maintainer-approved migration.
  - confidence: 0.87
- **[P2] Reject Feishu group chats without a topic:** `src/agents/subagent-spawn.ts:838-844`
  - body: A Feishu request with `to: "chat:..."` and no `threadId` now flows through generic conversation resolution and can bind the whole group chat as the subagent conversation. The removed Feishu spawning handler intentionally rejected chat/group targets without a topic, allowing only DMs or topic conversations; preserve that restriction so a group-level spawn does not capture all chat messages for the child session.
  - confidence: 0.82
