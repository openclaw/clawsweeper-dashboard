## Summary

Current `main` already hardens edit-tool conflict recovery for stale content by re-reading the file on failure, surfacing current contents for exact-text mismatches, and converting post-write false failures back to success when the requested edit already landed.

## Evidence

- **Recovery wrapper handles stale-content edit failures:** `wrapEditToolWithRecovery` snapshots the original file, re-reads after failures, returns success if the requested edits already appear in the current file, and appends `Current file contents:` when an exact-text mismatch occurs.

- **Both host and sandbox edit tools use the recovery wrapper:** The host and sandbox edit-tool constructors both wrap the base edit tool with `wrapEditToolWithRecovery`, so the hardened behavior is active in the real edit paths.
  - file: [src/agents/pi-tools.read.ts:644](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/agents/pi-tools.read.ts#L644)
