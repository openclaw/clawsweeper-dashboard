## Summary

Keep this PR open for maintainer review: current main has not implemented the Liquid Glass bottom-bar change, and the PR still has blocking layout findings plus an unrelated provider-registry change that needs to be split or removed before merge.

## What This Changes

The PR rebuilds the iOS home bottom toolbar as floating Liquid Glass-style status/action capsules, moves it from a bottom safe-area inset to an overlay, switches startup through RootView, extends ScreenTab into the bottom safe area, and adds a DeepSeek provider contract registry entry.

## Best Possible Solution

Land a narrow iOS-only bottom-bar redesign after rebase that uses supported iOS 26 glass surfaces, preserves bottom content reachability and compact-layout accessibility, removes the DeepSeek/provider registry change, and coordinates with #40874 if maintainers want one broader Liquid Glass direction.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Restore content avoidance for the floating toolbar:** `apps/ios/Sources/RootCanvas.swift:478`
  - body: Changing the toolbar host from a bottom safe-area inset to a plain overlay removes the vertical space that kept canvas content clear of the control. Because ScreenWebView disables UIKit inset adjustment and the scaffold only pads for the OS safe area, bottom web content can render under the toolbar and become hidden or hard to tap.
  - confidence: 0.88
- **[P2] Let the status title shrink in compact layouts:** `apps/ios/Sources/HomeToolbar.swift:90`
  - body: The new fixedSize(horizontal: true) makes the status title non-compressible while the trailing action cluster keeps fixed-width buttons. With states like Connecting plus an activity icon, compact widths or larger Dynamic Type can overflow and clip the action buttons that need to stay reachable.
  - confidence: 0.82
