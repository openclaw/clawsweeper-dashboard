## Summary

Current `main` no longer allows arbitrary workspace reads through `agents.files.get`: the handler now rejects any filename outside a fixed allowlist before opening the file, and that implementation is already present in the shipped `v2026.4.23` release.

## Evidence

- **Allowlist gate before any read:** `resolveAgentWorkspaceFileOrRespondError` restricts `params.name` to `ALLOWED_FILE_NAMES` and returns `invalid_request` for anything else (`unsupported file "..."`). That blocks `.env`, `secrets.txt`, `config/private.json`, and similar arbitrary workspace paths.
