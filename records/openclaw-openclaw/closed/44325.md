## Summary

PR #44325 is a valid fix for the Telegram DM-thread cron inference bug, but it is now superseded by the open canonical follow-up PR #44412. Current main still has the bug, so this should not close as implemented; the remaining work should continue on #44412, which carries the same #44270 fix plus hardening for optional Telegram account IDs and using the thread-suffix chat ID.

## Best Possible Solution

Close #44325 as superseded and continue the Telegram DM-thread cron inference fix through #44412. The final landed patch should preserve the direct-thread regression test from #44325, prefer the thread-suffix chat ID, support optional Telegram accountId session keys, and keep non-Telegram thread inference unchanged.
