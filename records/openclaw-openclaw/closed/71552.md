## Summary

Keep open. Current `main` still strips the leading `nvidia/` segment from static refs like `nvidia/nemotron-3-super-120b-a12b`, and that normalized value is then used in provider model resolution. The NVIDIA bundled catalog still defines the model id as `nvidia/nemotron-3-super-120b-a12b`, so the report matches a real code/catalog mismatch rather than an obsolete issue.

## Evidence

- **Static ref normalization lacks an NVIDIA re-prefix case:** `normalizeStaticProviderModelId()` special-cases `openrouter` and `vercel-ai-gateway`, but not `nvidia`. `parseStaticModelRef()` splits on the first slash, so `nvidia/nemotron-3-super-120b-a12b` becomes provider `nvidia` plus model `nemotron-3-super-120b-a12b`.
