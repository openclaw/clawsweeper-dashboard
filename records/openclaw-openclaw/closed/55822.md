## Summary

Current `main` already supports non-plaintext credential references in `openclaw.json` via SecretRefs, including `${ENV_VAR}` interpolation and external secret stores through `file`/`exec` providers. The supported surface explicitly includes `models.providers.*.apiKey`, `gateway.auth.token`, and channel secrets such as `channels.feishu.appSecret`, so the requested capability is implemented well enough.
