## Summary

Current `main` already gates live session model switches behind an explicit `liveModelSwitchPending` flag, which removes the persisted-session override path described in #56958 and preserves fallback execution unless the user explicitly requested a live `/model` switch.

## Evidence

- **Live switch now requires explicit pending flag:** `shouldSwitchToLiveModel` returns a switched model only when `entry.liveModelSwitchPending` is true, and its docstring says this replaced the previous in-memory approach that could not distinguish user `/model` switches from fallback rotations.
