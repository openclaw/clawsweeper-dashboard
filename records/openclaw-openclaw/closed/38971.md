## Summary

Issue #38971 is already addressed on current `main`, and the same protections are present in the shipped `v2026.4.22` release: the cron store loader backfills missing legacy timestamp fields before `cron.list` returns jobs, and the macOS app now decodes `cron.list` lossily so a malformed job no longer breaks the whole Cron tab.

## Evidence

- **Cron store backfills legacy runtime timestamps:** `loadCronStore()` repairs missing/invalid runtime fields by populating `updatedAtMs` from `createdAtMs` or `Date.now()`, which covers the legacy-store shape described in the issue.
  - file: [src/cron/store.ts:134](https://github.com/openclaw/openclaw/blob/4d7c4b3298920c0e6e2c5cc8bd7a1546e8122abe/src/cron/store.ts#L134)
