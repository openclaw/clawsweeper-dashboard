## Summary

Keep open: current main does not have ledger-backed ACP session replay, and this active maintainer PR still needs review because the proposed ledger can remain authoritative after a Gateway stream gap and cause `loadSession` to skip the safer transcript fallback.

## What This Changes

The PR adds in-memory and file-backed ACP event-ledger storage, uses it for `loadSession` replay before Gateway transcript fallback, and updates ACP tests, docs, and changelog.

## Best Possible Solution

Keep the ledger-backed replay direction, but make the ledger incomplete whenever the bridge may have missed Gateway stream events so `loadSession` can fall back to the Gateway transcript.

## Reproduction Assessment

Not applicable to the feature request as a user bug, but yes for the review finding from source inspection. A send-accepted prompt can complete through disconnect reconciliation without all chat/tool events recorded, then a later `loadSession` trusts the complete ledger and skips transcript fallback.

## Solution Assessment

No. Ledger-backed replay is a maintainable direction, but the PR should first invalidate or bypass complete ledger replay after stream gaps so it does not regress the existing transcript recovery path.

## Review Findings
