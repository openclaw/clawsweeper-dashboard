## Summary

The report no longer matches the supported current dependency path: OpenClaw still passes `--skip-trust`, but current Gemini CLI and Homebrew stable define that flag, so the old v0.38.x parser failure is obsolete rather than a current OpenClaw bug.

## What This Changes

The issue asks OpenClaw to stop passing `--skip-trust` to the `google-gemini-cli` backend because Gemini CLI v0.38.x rejected it.

## Best Possible Solution

Keep the current trust-bypass flag for supported Gemini CLI releases; reopen or file a new issue only if a current stable Gemini CLI again rejects the flag with logs and versions.

## Reproduction Assessment

No. The historical v0.38.x failure is credible from the report, but current Gemini CLI v0.46.0 and the current Homebrew formula define `--skip-trust`, so the reported current install path no longer reproduces the parser error.

## Solution Assessment

Yes. Closing as obsolete is safer than removing the flag, because current Gemini CLI documents `--skip-trust` as the headless trust bypass and the removal PRs would risk regressing supported users.
