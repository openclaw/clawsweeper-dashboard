## Summary

Keep open. Current main does not implement the PR's proposed `params.context1m` override for `claude-cli`; it only grants 1M accounting for `claude-cli` Opus 4.7 variants. The broader PR remains a product/runtime decision because current Anthropic transport code strips the `context-1m` beta for Claude CLI/OAuth token auth, so merging the PR as-is could overstate usable context for those sessions.

## Best Possible Solution

Keep the PR open for maintainer decision. The likely path is either to rework it so Claude CLI context accounting only advertises 1M when the runtime can actually send an accepted long-context request, or to close/re-scope it in favor of the current main behavior: `context1m` remains Anthropic API-key gated, while `claude-cli` gets the shipped Opus 4.7-specific 1M handling.
