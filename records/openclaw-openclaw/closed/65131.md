## Summary

Close as implemented on current main. The heartbeat and empty runtime-only rows are filtered from chat.history, queued restart/system events are excluded from the visible transcript prompt, and the previously missing pre-compaction memory flush path now uses an empty transcriptPrompt with test coverage. This fix is on main after v2026.4.26 and is not proven shipped in a release yet.

## What This Changes

Add or use Control UI/WebChat filtering so internal system messages such as heartbeat prompts, gateway restart notices, and pre-compaction memory flush prompts do not appear as normal chat messages.

## Best Possible Solution

Keep the structural main-branch behavior: internal runtime/system turns should remain available to the model and raw audit/replay paths, while chat.history and Control UI display only user-visible conversation rows. Separate product work like #43567 can decide how to render intentionally visible system notices without reopening this hidden-internal-message cleanup.
