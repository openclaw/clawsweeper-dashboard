## Summary

Current main already provides native gateway-side user-turn persistence via session transcripts, plus built-in cross-session recall/search by indexing those transcripts through the memory system. The request's underlying capability is implemented, even though the storage shape is session `.jsonl` files rather than `memory/user-prompts-YYYY-MM-DD.md`.

## Risks / Open Questions

- The implementation uses per-session `.jsonl` transcripts plus optional transcript indexing, not the exact daily Markdown file/path proposed in the issue.
- Transcript indexing is opt-in/experimental, so users must enable it to get semantic cross-session search rather than raw transcript files only.
- `pnpm docs:list` could not be run in this checkout because `pnpm` was unavailable in the environment, so doc discovery relied on direct file inspection instead.

## Close Comment

Closing from a Codex review: current OpenClaw already has native gateway-side prompt capture and transcript-backed cross-session recall. The gateway eagerly emits/persists accepted user turns in `src/gateway/server-methods/chat.ts:2185`, with coverage for both agent-run and no-run paths in `src/gateway/server-methods/chat.directive-tags.test.ts:1663` and `src/gateway/server-methods/chat.directive-tags.test.ts:2426`. Session transcripts are a documented built-in surface in `docs/concepts/agent-workspace.md:135`, and transcript indexing/search is documented in `docs/concepts/memory-search.md:125` plus `docs/reference/memory-config.md:340`. This was already shipped in `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). The exact `memory/user-prompts-YYYY-MM-DD.md` format from this issue is not how it is stored, but the built-in automatic logging and cross-session recall requested here are already implemented.
