## Summary

Keep open: current main still drops ordinary ACP background-task progress from terminal delivery and still caps progress at 240 characters, so this PR remains the open implementation candidate for https://github.com/openclaw/openclaw/issues/74070. The branch is not merge-ready because the fallback can surface silent control tokens and the external contributor has not supplied after-fix real behavior proof; the merged chunk-boundary fix at https://github.com/openclaw/openclaw/pull/78383 is related but does not cover this fallback/cap request.

## What This Changes

The PR raises the ACP background-task progress cap from 240 to 1200 characters and returns normalized progress as `terminalSummary` for non-blocked completions.

## Best Possible Solution
