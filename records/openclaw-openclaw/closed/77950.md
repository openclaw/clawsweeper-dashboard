## Summary

Keep open: current main still has a source-reproducible path where a target agent's `systemPromptOverride` can drop the sub-agent task because the task lives only in `extraSystemPrompt` while the first user message points back to the system prompt.

## What This Changes

Request that `sessions_spawn(mode="run")` inject the `task` into the child context even when the spawned agent has a custom `systemPromptOverride`.

## Best Possible Solution

Append or otherwise preserve runtime `extraSystemPrompt` when composing override-based agent prompts, with a regression test proving sub-agent task visibility under `systemPromptOverride`.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: configure a spawned target agent with `systemPromptOverride`, then `sessions_spawn(mode="run", task="unique marker")`; the task is only in `extraSystemPrompt`, while the override prompt path omits that field.

## Solution Assessment

Yes, the narrow maintainable fix is to preserve `extraSystemPrompt` in the `systemPromptOverride` branch and add focused regression coverage; exposing a new prompt mode or duplicating the task in every initial user message is broader than needed.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/77950. Current main puts native `sessions_spawn(mode="run")` task text only in the child `extraSystemPrompt`, while the kickoff user message intentionally says the task is under `## Your Role`; when the target agent has `systemPromptOverride`, `buildAttemptSystemPrompt` uses the override/bootstrap branch and drops `embeddedSystemPrompt.extraSystemPrompt`, so the child never sees the task. Preserve runtime `extraSystemPrompt` in override-based prompt composition, preferably with the same minimal-mode `## Subagent Context`/runtime-context semantics used by `buildEmbeddedSystemPrompt`. Add a regression test that constructs an override prompt plus `extraSystemPrompt` containing a unique sub-agent task marker and asserts the final prompt includes both the override and the task. Do not globally re-add task text to `buildSubagentInitialUserMessage`; preserve the #72019 single-source/no-duplicate behavior. Add a user-facing changelog fix entry because this changes user-visible sub-agent behavior.
