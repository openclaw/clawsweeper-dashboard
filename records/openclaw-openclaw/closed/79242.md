## Summary

Keep open: current main still hardcodes `/usr/bin/scp`, and the linked open PR is the active fix path with closing syntax, so this issue should close only after that PR lands.

## What This Changes

Resolve sandbox media staging `scp` execution from `PATH` instead of hardcoding `/usr/bin/scp`.

## Best Possible Solution

Review and land the linked PR if maintainers accept PATH-based `scp` lookup, keeping the change local to auto-reply media staging and preserving the existing host/path validation.

## Reproduction Assessment

Yes. Current main unconditionally spawns `/usr/bin/scp`, so a host where `scp` exists on `PATH` but not at that absolute path has a high-confidence failure before SSH connection setup.

## Solution Assessment

Yes. Spawning `scp` by command name while preserving no-shell execution, existing arguments, and existing host/path validation is the narrowest maintainable fix for the reported portability bug.
