## Summary

Keep open. Current main still treats any inline `[[tts:...]]` and `[[tts:text]]...[[/tts:text]]` sequence as an active directive, and the speech-core TTS path still uses that parse result to trigger tagged auto-TTS. I found no main-branch test coverage for preserving literal inline examples, and the latest release tag ships the same parser implementation.

## Evidence

- **Parser still strips any inline TTS tags and marks them active:** `parseTtsDirectives()` returns early only when `[[tts:` is absent, then unconditionally replaces both `[[tts:text]]...[[/tts:text]]` blocks and `[[tts:...]]` directives anywhere in the text while setting `hasDirective = true`. There is no literal-example or code-span/code-block guard in this function.
  - file: [src/tts/directives.ts:65](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/tts/directives.ts#L65)
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Tagged auto-TTS still triggers from that parser result on outbound replies:** `maybeApplyTtsToPayload()` passes every reply text through `parseTtsDirectives()`, rewrites the visible text from `directives.cleanedText`, and in `tagged` mode proceeds whenever `directives.hasDirective` is true.
  - file: [extensions/speech-core/src/tts.ts:1056](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/extensions/speech-core/src/tts.ts#L1056)
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Current tests only cover active-directive parsing paths:** The direct parser tests exercise provider-routing and override extraction for active directives, but there is no test asserting that inline literal examples like `[[tts:voice=...]]` or `[[tts:text]]...[[/tts:text]]` remain plain text.
  - file: [src/tts/directives.test.ts:58](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/tts/directives.test.ts#L58)
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Bundled speech-core contract tests also assume inline tags are active:** The shared TTS contract suite verifies that inline tags are stripped and converted into overrides/ttsText, again with no coverage for literal-example preservation.
  - file: [test/helpers/plugins/tts-contract-suites.ts:555](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/test/helpers/plugins/tts-contract-suites.ts#L555)
