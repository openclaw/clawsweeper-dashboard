## Summary

Current main and the latest release tag no longer contain the reported `beforeExit` listener path, and the session write-lock cleanup now has process-scoped listener de-duplication plus regression coverage.

## What This Changes

The issue reports a regression where long-running gateway churn in OpenClaw 2026.5.2 again accumulates `process` `beforeExit` listeners and RSS despite the earlier #65391 fix.

## Best Possible Solution

Keep the shipped v2026.5.3/current-main cleanup for the `beforeExit`/session-lock listener path, while leaving the broader plugin restart signal-handler leak pattern to #73655.

## Reproduction Assessment

No. I did not reproduce the reporter's long-running 2026.5.2 workload on current main, and current source no longer contains any `beforeExit` registration to exercise.

## Solution Assessment

Yes. The current implementation is the narrow maintainable fix for this path: one process-scoped cleanup state, bounded registration, unregister coverage in drain/reset, and a regression test for listener accumulation.
