## Summary

Keep open. This PR is the active successor for the intermediate task-state work and the paired operational-summary request, but the current patch still has source-backed correctness blockers, failing live checks, and no after-fix real behavior proof from the external contributor.

## What This Changes

This PR adds `awaiting_approval` and `waiting_external` task states, propagates active-state handling across task consumers, and adds lifecycle/operational formatting for CLI, auto-reply, and task tests.

## Best Possible Solution

Land or replace this with an additive task-lifecycle feature that preserves status persistence, redaction, the TaskFlow revision contract, public docs, and real behavior proof.

## Reproduction Assessment

Yes for the PR blockers: source inspection and live CI show the normalization, redaction, and TaskFlow API failures on the PR head. This is not a current-main bug reproduction; it is review proof for a proposed feature branch.

## Solution Assessment

No, not as currently proposed. The direction is plausible, but the patch needs to preserve new statuses through normalized writes, suppress sanitized-empty updates, satisfy the current TaskFlow API, document the public statuses, and provide real behavior proof before merge.

  - confidence: 0.84
