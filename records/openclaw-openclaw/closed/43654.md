## Summary

Current main implements the likely root fix for this report: local managed browser profiles on Linux now automatically fall back to headless mode when no display server is present, matching the reporters' Ubuntu/headless-host symptom where manual headless Chromium worked. Main also removes the permanent no-retry hint from transient browser timeouts.
