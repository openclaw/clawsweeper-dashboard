## Summary

Keep open: the reported Feishu card-thread reply path is plausible on current main because inbound text events are normalized from only top-level `text`, so a nested `content[].body` text payload would become empty and be dropped before reaching the agent.

## What This Changes

Fix Feishu DM thread replies to bot card messages so nested reply text is extracted and passed to the agent.

## Best Possible Solution

Teach the Feishu inbound content normalizer to extract nested card-thread reply text before debounce and empty-message filtering, with regression coverage and no routing or config changes.

## Reproduction Assessment

Yes, at source level: a Feishu text event whose JSON lacks top-level `text` but carries nested body text will normalize to an empty string and hit the existing empty-message skip. I did not run a live Feishu reproduction in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to extend the Feishu inbound content parser and cover this payload shape, while preserving the existing blank-message guard and session-routing behavior.
