## Summary

Keep this issue open: current main and the latest release still create an originless automatic subagent model override that the legacy cleanup path can discard, and an open linked implementation PR already closes this issue if it lands.

## What This Changes

The issue reports that config-selected subagent models from `agents.defaults.subagents.model` are resolved at spawn time but discarded before the child run, causing the child to use the agent default model instead.

## Best Possible Solution

Land one focused provenance fix that sets and preserves origin metadata for config-selected subagent model overrides, then close this issue from the merged PR.

## Reproduction Assessment

Yes, at source level. Configure `agents.defaults.subagents.model`, spawn without an explicit model, and current main writes an originless automatic override that the child run cleanup path treats as legacy and discards.

## Solution Assessment

Yes. The narrow maintainable fix is to complete provenance where the config-selected override is created and preserve it when writing the child session, rather than weakening cleanup for genuinely stale legacy fallback entries.
