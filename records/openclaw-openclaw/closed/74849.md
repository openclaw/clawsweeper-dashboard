## What This Changes

The PR changes SDK run-event chat projection normalization and Gateway abort/wait dedupe handling so app clients receive incremental assistant deltas and cancelled runs cannot later appear completed or failed.

## Best Possible Solution

Keep the implementation already on main and close this duplicate draft branch; any remaining broader SDK stabilization work should continue under the maintainer-owned tracking issue rather than by merging this PR.

## Reproduction Assessment

Yes. The reproduction path is high-confidence from the PR body and current regression tests: chat-only projection frames with accumulated text would duplicate appended `data.delta`, and a late run completion after `sessions.abort` would previously overwrite the cancel snapshot.

## Solution Assessment

Yes. The current-main implementation is the narrow maintainable fix: it normalizes only chat-projection fallback deltas in the SDK and makes the existing Gateway dedupe/cancel contract authoritative without adding a new API or broad migration.
