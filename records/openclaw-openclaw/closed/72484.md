## Summary

Keep this issue open. Current main still uses serial per-collection QMD CLI invocations for non-mcporter multi-collection search, and open same-author PR #72485 is an implementation candidate with closing syntax for this issue. That PR should be reviewed or closed before the tracker is resolved.

## Best Possible Solution

Keep this issue open while maintainers review PR #72485. If the approach is correct, land that PR or an equivalent source-grouped direct-QMD implementation that preserves mixed memory/session diversification and handles QMD result collection hints; otherwise close or revise the PR and keep this issue as the tracker. Reconcile with #69583 before landing if both remain viable optimization candidates.
