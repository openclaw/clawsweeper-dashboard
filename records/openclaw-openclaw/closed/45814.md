## Summary

Current `main` already implements the Telegram explicit-proxy media-download fix. The SSRF guard now has a trusted explicit-proxy mode, `fetchRemoteMedia` opts into it for Telegram media downloads, and tests cover the proxy-preserving path. The fix is present on `main` but not in the latest tagged release `v2026.4.22`, so `fixedRelease` is unknown/unshipped here.

## Evidence

  - file: [src/infra/net/fetch-guard.ts:348](https://github.com/openclaw/openclaw/blob/37c37eecfbe4bef413c33b8606db62cf51c2d1d4/src/infra/net/fetch-guard.ts#L348)
