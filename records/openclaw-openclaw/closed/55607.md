## Summary

PR #55607 is internally contradictory: its title/body ask to enforce `hooks.allowedAgentIds` for implicit routing, but current OpenClaw docs/code/tests define that setting as an explicit-routing control, and the PR’s later review history switched to preserving implicit default routing instead.

## Evidence

- **Docs define `allowedAgentIds` as explicit-routing only:** The Gateway configuration reference says `allowedAgentIds` "restricts explicit routing" and documents `*`/omitted/`[]` semantics accordingly.
  - file: [docs/gateway/configuration-reference.md:481](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/docs/gateway/configuration-reference.md#L481)

  - file: [src/gateway/hooks.ts:300](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/gateway/hooks.ts#L300)
