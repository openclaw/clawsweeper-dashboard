## Summary

Current main already addresses this LINE failure mode by rejecting `dmPolicy: "open"` without wildcard `allowFrom` during config validation, so normal gateway startup no longer reaches the webhook path that silently drops DMs.

## What This Changes

The issue requests that LINE `dmPolicy: "open"` allow inbound direct messages instead of returning a webhook 200 OK without delivering messages to the agent.

## Best Possible Solution

Keep the fail-closed validation behavior and require users who intentionally want public LINE DMs to configure `allowFrom: ["*"]`.

## Reproduction Assessment

No for current main. The reported invalid config is rejected by LINE schema validation and gateway startup before the webhook handler can acknowledge and drop messages.

## Solution Assessment

Yes. Failing config validation is the narrowest maintainable fix because OpenClaw's documented access-control model requires an explicit wildcard before `dmPolicy: "open"` becomes public.
