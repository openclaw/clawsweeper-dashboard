## Summary

## What This Changes

The PR adds NVIDIA NIM baseUrl detection to OpenAI-completions compatibility defaults and tests that NIM endpoints use `max_tokens`.

## Best Possible Solution

Move the NIM `max_tokens` requirement into NVIDIA-owned compatibility metadata or a small manifest/providerRequest extension, with focused regression coverage and redacted live NIM proof.

## Reproduction Assessment

Yes for the current-main source path: NVIDIA routes through OpenAI-completions, the payload builder follows `maxTokensField`, current defaults emit `max_completion_tokens`, and NVIDIA documents `max_tokens` for the endpoint. A live NIM request still needs contributor proof.

## Solution Assessment

No: the PR fixes field selection but does so by hard-coding a NVIDIA hostname in core agent compatibility logic. The safer fix is provider-owned compat metadata or a generic request metadata seam.
