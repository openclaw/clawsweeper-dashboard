## What This Changes

This PR changes Telegram DM authorization so transient pairing-store read failures do not become empty allowlists, skips unnecessary store reads for configured/open DMs, preserves per-DM text-fragment authorization, and adds focused tests plus a changelog entry.

## Best Possible Solution

Land the focused fix after a Telegram owner adds redacted live Telegram proof for the store-failure path or explicitly accepts mocked-only proof for this maintainer PR.

## Reproduction Assessment

Yes. Source inspection shows current main turns pairing-store read failures into an empty allowlist before DM pairing authorization, and the PR adds mocked regressions for that path; I did not run a live Telegram repro in this read-only review.

## Solution Assessment

Yes, the proposed direction is a narrow maintainable fix: stop treating a store failure as an empty allowlist while preserving configured allowFrom and per-DM policy resolution. The remaining blocker is proof/owner acceptance, not an alternate code shape.
