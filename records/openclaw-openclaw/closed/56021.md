## Summary

Current `main` already implements the reported replay/tool-result repair: it drops orphan `toolResult` entries instead of rewriting them into invalid IDs, repairs missing OpenAI Responses outputs before ID downgrades, and adds regression plus live-model coverage across both OpenAI Responses and strict non-OpenAI replay paths.

## Evidence

- **Main replay path now repairs OpenAI Responses transcripts before ID rewriting:** `sanitizeSessionHistory(...)` treats `openai-responses`, `openai-codex-responses`, and `azure-openai-responses` as the shared Responses family, then runs `sanitizeToolUseResultPairing(...)` with Codex-style `"aborted"` repair before the OpenAI pairing downgrade. That directly addresses the issue’s reported orphan/invalid `function_call_output` replay path.
