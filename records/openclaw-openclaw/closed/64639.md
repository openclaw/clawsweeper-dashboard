## What This Changes

The issue requests a configuration option to disable hardcoded skill source directories, especially the personal `~/.agents/skills` source, to avoid prompt bloat when skills are routed externally.

## Best Possible Solution

Track the short-term config change in #56366 as the canonical issue: add a backwards-compatible setting that skips disabled built-in skill roots before discovery and watcher registration, preserving default precedence when unset. Treat native on-demand skill selection as a separate design item rather than bundling it into this cleanup.
