## Summary

Keep this PR open: the shared helper direction is plausible and CI is currently green, but the PR introduces a QQBot STT regression by moving a user-configurable OpenAI-compatible endpoint behind strict guarded fetch without a policy for configured local/private hosts.

## What This Changes

This PR adds shared plugin SDK runtime helpers for JSON Schema and MIME handling, migrates bundled plugins to those helpers, updates docs/changelog/API metadata, and changes QQBot STT uploads to use guarded fetch.

## Best Possible Solution

Keep the helper consolidation direction, but make QQBot STT guarded fetch trust only the configured base URL and cover local/private endpoint behavior before merge.

## Reproduction Assessment

Yes. By source inspection, the PR routes QQBot STT through fetchWithSsrFGuard without policy while the configured baseUrl can be loopback/private; the guard blocks those targets before request delivery.

## Solution Assessment

No as patched. The maintainable fix is to pass a configured-base-url SSRF policy, similar to existing OpenAI TTS/LMStudio/Ollama paths, and update the STT test seam for guarded fetch.
