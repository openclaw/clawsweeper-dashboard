## Summary

Current main already covers the PR's underlying security goals: Feishu webhook mode now fails validation/runtime startup without both event secrets, Feishu event secrets are part of the maintained SecretRef/plaintext audit surface, and reverse-proxy trust is handled by generic gateway security checks and docs rather than a China-specific core collector.

## Best Possible Solution

Keep the current main implementation: Feishu-owned webhook and secret behavior stays in the Feishu plugin, plaintext secret detection stays in `openclaw secrets audit`, and reverse-proxy hardening stays in generic gateway security audit/docs. If maintainers want more Feishu-specific operator guidance later, add it to `extensions/feishu/src/security-audit-shared.ts` with source-config-based tests.
