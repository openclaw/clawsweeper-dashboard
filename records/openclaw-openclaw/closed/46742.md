## Summary

Current `main` already implements the requested behavior: snapshot/read-only plugin loads no longer replace active runtime state, and the gateway config schema path now uses manifest metadata instead of booting plugin runtime.

## Evidence

- **Snapshot loads are explicitly non-activating and cannot cache:** `src/plugins/loader.ts:1830` rejects `activate:false` unless `cache:false`, and `src/plugins/loader.ts:1890` only clears global plugin state when `shouldActivate` is true, so read-only loads do not replace the active runtime registry.
  - file: [src/plugins/loader.ts:1830](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/src/plugins/loader.ts#L1830)

  - file: [src/plugins/registry.ts:1165](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/src/plugins/registry.ts#L1165)
