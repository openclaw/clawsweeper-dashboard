## Summary

Close as duplicate/superseded by the already-open #55334 session-store bloat/performance tracker. Current main still has the same unresolved whole-file session-store update path and inline static payloads, while #55334 is the broader canonical issue for duplicated `skillsSnapshot`/`systemPromptReport` data and session-store growth; this issue's run-budget logs are valuable evidence but do not require a separate tracker.

## What This Changes

The issue reports large per-agent `sessions.json` files with repeated inline `skillsSnapshot` data causing full-file parse/stringify/write latency to exhaust the agent wait budget and return `No reply from agent`.

## Best Possible Solution

Keep #55334 open as the canonical session-store structural bloat tracker, and solve the shared root cause there with a backward-compatible design that avoids full-store parse/stringify/write work for one active session and removes repeated large static payloads from retained rows.

## Reproduction Assessment

Yes. The issue provides production file sizes, affected agents, and gateway log timing, and current main confirms a focused source-level reproduction: synthesize many recent entries with repeated large `skillsSnapshot` payloads and time one `persistSessionEntry` or `updateSessionStore` call.
