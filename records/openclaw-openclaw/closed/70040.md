## Summary

Keep open. Current main still lacks Chinese/CJK trigger support in `memory-lancedb`, and this PR's regex-only change would not fully satisfy the stated behavior because `shouldCapture` rejects common short Chinese memory statements before it checks trigger patterns. Related CJK PRs are useful prior context, but they are closed unmerged and do not supersede the work on main.

## Best Possible Solution

Keep this PR open and request an amended implementation: add focused CJK trigger coverage, account for short high-signal Chinese inputs before or within the length gate, add Chinese `shouldCapture` tests, and add Chinese category-detection tests if category quality is in scope. Reuse #16392, #16672, and #20241 as prior context, but do not treat them as merged fixes.
