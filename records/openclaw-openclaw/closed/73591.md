## Summary

Close as duplicate/superseded by #73424 and its implementation candidate #73451. This WhatsApp report is useful additional evidence, but current main shows the same root cause is the shared `loadWebMedia` image optimization path rather than a separate WhatsApp-only failure.

## Best Possible Solution

Close this issue as a duplicate and consolidate the WhatsApp evidence onto #73424/#73451. The fix should live in the shared `src/media/web-media.ts` optimization path, preserve MIME and byte-cap behavior when falling back, and include regression coverage for WhatsApp outbound media loading.
