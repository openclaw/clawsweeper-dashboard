## Summary

The shipped solution uses an explicit `requiresInteractiveConfirmation` marker for orphan transcript archival, which is narrower than the PR's global `initialValue: false` guard and avoids changing other doctor repair defaults.

## What This Changes

The PR adds an `initialValue !== false` guard to doctor runtime repair auto-approval and a regression test asserting orphan transcripts are preserved under a `--fix`-style prompter.

## Best Possible Solution

Keep the shipped explicit interactive-confirmation contract for transcript archival and handle any broader orphan-transcript classification concerns separately in the already-open follow-up discussion rather than merging this obsolete branch.

## Reproduction Assessment

Yes. The original bug is reproducible through the doctor state-integrity orphan-transcript path under repair mode; current main has focused tests showing interactive-only transcript archival is not accepted in non-interactive or `--yes` repair flows.

## Solution Assessment

No for the PR diff as submitted. The current-main solution is better because it marks the destructive transcript archive prompt explicitly instead of treating every `initialValue: false` runtime repair as unsafe.
