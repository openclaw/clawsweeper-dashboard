## Summary

Keep this PR open. Current main still rejects and strips the `explicit` Active Memory chat type, resolves webchat-style explicit sessions as `direct`, and has no regression coverage for `agent:...:explicit:...` session keys. The supplied PR diff remains a focused implementation candidate for linked bug #65775.

## Best Possible Solution

Keep this PR open as the active implementation candidate for linked bug #65775. Review and land the focused manifest, runtime normalization, resolver ordering, changelog, docs, and regression-test changes after normal CI, or replace it with an equivalent main-branch fix that supports explicit Active Memory sessions.
