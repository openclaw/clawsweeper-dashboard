## Summary

This PR should stay open rather than be closed as implemented: current main still does not deliver raw ACP child final text back to direct chats, but this patch adds a separate direct-send path that conflicts with the current parent-owned task completion model and has concrete review blockers.

## What This Changes

The PR adds ACP parent-stream buffering and `routeReply`-based final/error delivery for non-threaded direct-chat parent sessions, plus relay tests.

## Best Possible Solution

Integrate final ACP result text into the parent-owned task-completion delivery path so requester routing, idempotency, direct/threaded behavior, whitespace preservation, and tests are handled in one model.

## Reproduction Assessment

Yes for the PR defects by source inspection: the optional `cfg` reaches a required `routeReply` parameter, whitespace-only deltas are dropped before direct buffering, and current main already has terminal task delivery that this patch does not coordinate with. I did not run a live WhatsApp/ACP smoke for the original report.

## Solution Assessment

No, not as-is. The direct-reply idea addresses the visible gap, but the maintainable fix should feed final child output into the existing task-completion delivery model instead of adding an independent terminal send path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Require cfg before direct replies:** `src/agents/acp-spawn-parent-stream.ts:302`
  - body: `cfg` is declared as possibly `undefined` after the best-effort session lookup, but this call passes it to `routeReply`, whose params require `cfg: OpenClawConfig`. Keep the config and target in one narrowed object, or skip direct delivery unless both are present, so the PR typechecks.
  - confidence: 0.92
- **[P2] Append whitespace-only deltas before filtering:** `src/agents/acp-spawn-parent-stream.ts:363-367`
  - body: The direct-reply buffer is appended only after the `!delta.trim()` return, so standalone space/newline chunks are dropped. A stream split as `"hello"`, `" "`, `"world"` would be delivered as `helloworld`; buffer raw final-output deltas before the progress-only trim filter.
  - confidence: 0.9
- **[P2] Avoid double-sending terminal completion:** `src/agents/acp-spawn-parent-stream.ts:395-400`
  - body: Current main already creates a parent-owned ACP task and delivers a terminal requester notification. This new lifecycle-end direct reply sends raw child output without marking that task delivery handled or sharing its idempotency path, so non-threaded chats can receive both the raw answer and the task completion message.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `src/agents/acp-spawn-parent-stream.ts:296-302`
  - body: This is a user-facing ACP direct-chat delivery fix, but the PR only changes code and tests. Add a `CHANGELOG.md` entry for the behavior change before merge.
  - confidence: 0.86
