## Summary

Keep open: this is a maintainer-labeled core updater architecture follow-up, and current main still runs package replacement orchestration from the existing `openclaw update` path rather than a separate external updater.

## What This Changes

Request to move `openclaw update` package replacement into a small updater process outside the package tree being replaced.

## Best Possible Solution

Define and implement a core updater handoff that writes a durable update plan, exits the current package process before replacement, performs verified staged swap and rollback outside the install root, and updates the updater docs and regression coverage.

Not applicable: this is an architectural follow-up request, not a bug with a failing current-main reproduction path. Source inspection confirms the requested external updater boundary is not present today.
