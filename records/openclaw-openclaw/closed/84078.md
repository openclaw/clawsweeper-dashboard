## Label Justifications

- `P2`: This is a normal-priority Slack delivery reliability bug with visible reply loss on a limited configuration and tool-argument path.
- `impact:message-loss`: The reported failure prevents visible Slack assistant replies from being posted when source replies are routed through the message tool with required durability.

## Summary

Keep this open: current main still has the Slack/source-reply/durable-send shape described in the report, and the related Slack issues do not fully supersede this narrower `bestEffort:false` failure path.

## What This Changes

Slack `message_tool_only` source replies can fail when a message-tool send requests required durability because Slack lacks `reconcileUnknownSend`.

## Best Possible Solution

Add Slack unknown-send reconciliation if feasible; otherwise make Slack `message_tool_only` replies choose best-effort or fail validation based on adapter capability while preserving strict required-durable failures for callers that explicitly need them.

## Reproduction Assessment

Partly. Source inspection gives a clear reproduction path for Slack required sends with `bestEffort:false`, but this review did not run a live Slack room to confirm the full session/tool-argument path on current main.

## Solution Assessment

Unclear. The maintainable fix depends on a maintainer decision between Slack reconciliation, targeted best-effort downgrade/capability negotiation, or stronger validation and tool guidance.
