## What This Changes

Slack visible source replies routed through `message_tool_only` can fail before posting when a Slack `message` tool send requests required durability but the Slack adapter lacks `reconcileUnknownSend`.

## Best Possible Solution

Review and land the targeted source-reply best-effort PR if maintainers accept that contract; otherwise replace it with Slack `reconcileUnknownSend` support while keeping required durable failures for explicit external sends.

## Reproduction Assessment

Yes at source level: current main maps `bestEffort:false` sends to required durable delivery, required delivery requires `reconcileUnknownSend`, and Slack does not declare that capability. I did not run a live Slack current-main reproduction in this read-only pass.

## Solution Assessment

Unclear until maintainers accept the delivery contract. The linked PR is a narrow and well-proven source-reply best-effort fix, while Slack reconciliation remains the stricter durability alternative.
