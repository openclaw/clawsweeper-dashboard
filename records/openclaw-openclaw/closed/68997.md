## Summary

Current main already fixes the reported npm packaging omission. The task-registry control runtime is now an explicit tsdown build entry, release packaging requires and smoke-imports `dist/task-registry-control.runtime.js`, and the changelog records the fix for this issue under Unreleased. The latest release in the provided context is still affected, so no shipped fixed release is recorded.

## Best Possible Solution

Keep the current main implementation: emit `task-registry-control.runtime` as a stable packaged runtime sidecar, retain the packed-release import smoke and runtime-sidecar guard, and let the next OpenClaw release carry the fix to npm users.
