## Summary

Keep open. Current main still lacks an iMessage-owned per-turn watchdog or one-shot progress ping for tool activity during agent silence; the shared progress callback seam exists and Telegram consumes it, but iMessage only wires typing lifecycle options into its dispatch path.

## What This Changes

Request to add a one-shot iMessage monitor watchdog that sends a progress ping when a turn is silent past a threshold after tool activity.

## Best Possible Solution

Add an iMessage-owned one-shot progress watchdog on the existing GetReplyOptions callbacks, reset it per inbound turn and visible output, and coordinate Claude CLI event coverage through the linked bridge issue.

## Reproduction Assessment

Yes, at source level. Current main's iMessage dispatch passes only typing lifecycle options into dispatchInboundMessage, while the shared runner only calls channel progress callbacks when the channel registers them.

## Solution Assessment

Yes, pending maintainer UX sequencing. A channel-local watchdog on the existing callback seam is the narrow maintainable direction, but timeout/copy/spam policy and Claude CLI tool-event coverage should be reviewed together.
