## Summary

Close as implemented on current main. The reported v2026.4.26 release had the unsafe materializing fallback, but current main now streams fallback chunk scoring through `iterate()` and keeps only top results in memory, with regression coverage for the sqlite-vec-unavailable branch.

## What This Changes

The issue requests that `memory_search` avoid heap-loading the full memory database when sqlite-vec/vector search is unavailable on large memory stores.

## Best Possible Solution

Keep the current streaming fallback and its regression test, then ship it in the next OpenClaw release while leaving separate sqlite-vec availability issues to their own tracking threads.
