## Summary

Close as superseded. Current main still lacks the Mattermost draft-preview opt-out, but open PR #73252 now tracks the same bug with the canonical `streaming.mode` config path, type/schema/test coverage, and a runtime bypass; this PR also has a blocking type-surface issue around `account.config.streaming`.

## What This Changes

The branch adds a Mattermost `streaming.draftPreview` schema option, a no-op draft stream factory, a runtime gate that skips draft-preview posts when disabled, and a draft-stream unit test.

## Best Possible Solution

Keep the existing draft-preview behavior as the default, but implement the opt-out through the shared Mattermost `streaming.mode` contract with schema, exported types, account resolution, runtime delivery bypass, docs/changelog, and focused Mattermost tests aligned.

## Reproduction Assessment

Yes. A static current-main check is enough: the strict Mattermost schema lacks `streaming`, the monitor always creates `createMattermostDraftStream`, and the docs/shared config contract already describe `streaming.mode: "off"` as the opt-out.

## Solution Assessment

No. This PR is not the best implementation as-is because it introduces a parallel `draftPreview` boolean and reads an undeclared `MattermostAccountConfig.streaming` property; the safer path is the canonical `streaming.mode` implementation now tracked by #73252.
