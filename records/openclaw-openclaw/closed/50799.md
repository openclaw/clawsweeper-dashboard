## Summary

Current main and the latest stable release already implement the requested document attachment path: the chat picker accepts PDF/TXT/DOCX-style files, the UI serializes non-image files as file attachments, and the gateway persists non-image uploads instead of dropping them.

## What This Changes

The issue reports that the macOS GUI paperclip attachment picker restricts selection to image files and disables documents such as PDF, TXT, and DOCX.

## Best Possible Solution

Keep the shipped WebChat/Control UI non-video attachment implementation and close this report as implemented.

## Reproduction Assessment

No, not against current main: the file input now uses a broad accept list and focused tests cover PDF selection. The original failure is source-backed by the merged fix diff, which changed the accept list from image-only to common non-video file types.

## Solution Assessment

Yes. The merged solution fixes the picker, local preview/serialization, and gateway persistence path, which is more complete than only loosening the macOS/browser accept filter.
