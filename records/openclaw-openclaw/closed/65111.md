## Summary

Close as superseded. Current main still has the underlying CLI diagnostic bug, but this PR's branch is not safe to merge because its `plugins.entries` heuristic would drop the existing allowlist diagnostic for real bundled plugin commands; the same remaining work is already tracked in the broader open CLI diagnostic PR #64112.

## What This Changes

The PR adds a guard in `src/cli/run-main.ts` that returns null from `resolveMissingPluginCommandMessage` when an excluded command has no explicit `plugins.entries` entry and no manifest alias, and it changes the allowlist error wording.

## Best Possible Solution

Consolidate on the broader registry-backed CLI diagnostic fix in #64112, or the overlapping protected maintainer thread if maintainers prefer that route: the final implementation should only emit plugin allowlist/disabled diagnostics for real bundled plugin CLI roots or known command aliases, return null for built-in chat and unknown roots such as `commands` or `run`, and keep focused tests for both `commands` and excluded real plugin commands like `browser`.
