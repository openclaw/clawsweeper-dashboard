## Summary

Current main resolves plugin-bundled skill directories from the installed plugin index, applies plugin enablement, and feeds those entries into the same skills snapshot prompt used by live agent sessions. The memory-wiki plugin declares `./skills`, and `wiki-maintainer` is prompt-visible because it has no `disable-model-invocation` gate. This was implemented by `bc24b547d0888fb716505b07e3f2dceeb8abd203`; no release tag contains that commit yet.

## Best Possible Solution

Keep the installed-index based plugin skill discovery now on main, ship it in the next release, and rely on the existing skills snapshot path so CLI-visible enabled plugin skills and live session `<available_skills>` are derived from the same source.
