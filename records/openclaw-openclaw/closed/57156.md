## Summary

Current `main` already supports Feishu local-path media sends through the shared agent-scoped media allowlist pipeline, which covers `~/.openclaw/media/tool-image-generation/...` via the default `stateDir/media` root. The issue’s proposed Feishu-specific schema key is no longer required for the reported use case.

## Evidence

- **Default allowlist already covers `~/.openclaw/media`:** `buildMediaLocalRoots()` includes `path.join(resolveStateDir(), "media")`, and `resolveStateDir()` defaults to `~/.openclaw`, so generated images under `~/.openclaw/media/tool-image-generation/...` are inside the default outbound local-media roots.
