## Summary

Keep open. Current main still aggregates `sessions.usage` only over the `limit` slice, while this PR is a narrow, source-backed fix with matching regression coverage; the remaining blocker is ordinary maintainer review plus the provided `mergeable=false` branch status.

## What This Changes

This PR changes `sessions.usage` to aggregate totals over all merged session entries while returning only the limited page, and adds a regression test plus changelog entry.

## Best Possible Solution

Land this PR or an equivalent gateway fix that computes aggregates over all matching sessions while keeping the returned `sessions` list paginated.

## Reproduction Assessment

Yes, from source inspection. Current main applies `limit` before the aggregation loop, so any range with more matching sessions than `limit` will undercount totals and aggregate breakdowns.

## Solution Assessment

Yes. The PR separates aggregation scope from page membership with a small local change and a focused regression test; maintainers should still resolve mergeability and run the targeted gateway test before landing.
