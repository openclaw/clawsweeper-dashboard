## Summary

Current main and the latest release already cover the central failure: memory-core now waits briefly for post-run assistant text before falling back, and the gateway session API returns normalized flat messages rather than raw transcript envelopes.

## What This Changes

The issue reports memory-core writing fallback Dream Diary entries even though the narrative subagent generated usable assistant text.

## Best Possible Solution

Keep the released settled-read implementation and normalized `sessions.get` contract; broader memory-core no-text/archive variants should remain tracked in their separate canonical issues.

## Reproduction Assessment

Yes. The June 10 live repro gives a concrete timing race where `agent.wait` returned before assistant text was visible, and the current source has a matching delayed-read retry path that covers that failure mode.

## Solution Assessment

Yes. Retrying the memory-core read after a successful run is the narrow owner-local fix for delayed transcript persistence, while `sessions.get` already unwraps transcript envelopes before returning plugin messages.
