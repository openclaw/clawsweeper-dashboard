## What This Changes

## Best Possible Solution

## Reproduction Assessment

Yes from source inspection, though I did not execute a failing local repro: current main reads pricing bodies with `response.arrayBuffer()` before enforcing the 5 MiB cap, so a no-Content-Length stream can exceed the bound before rejection.

## Solution Assessment

Yes; replacing the local full-body read with the existing shared bounded response helper is the narrowest maintainable fix, and it preserves declared Content-Length rejection, malformed JSON handling, and source-failure fallback behavior.
