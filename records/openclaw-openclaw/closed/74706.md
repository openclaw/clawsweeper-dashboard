## Summary

Keep open. This is a maintainer-authored public Gateway/App SDK API request, and current main explicitly leaves the artifacts SDK namespace unsupported while the Gateway protocol, advertised method list, and handler map do not define `artifacts.*` RPCs.

## What This Changes

Add SDK-facing Gateway artifact RPCs for listing, getting, and downloading rich run/session/task outputs such as files, diffs, logs, media, screenshots, trajectories, and PR links.

## Best Possible Solution

Define an additive Gateway artifact resource with typed protocol schemas, read-scoped handlers, stable summary metadata, typed not-found/unsupported errors, and bounded download behavior; then wire `@openclaw/sdk` to those RPCs and update the App SDK and Gateway protocol docs.
