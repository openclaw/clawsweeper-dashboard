## Summary

This PR is now superseded by current main. The merge-side `input` preservation is implemented and covered, configured provider models now feed the gateway catalog, and current gateway tests intentionally keep image capability provider-scoped instead of borrowing image support from another provider as this PR proposes. The linked bug was closed, and related follow-up PRs were closed after the catalog-level implementation landed.

## Best Possible Solution

Close this PR as superseded. Keep the current main implementation: preserve configured `models[].input`, surface configured provider models through the catalog, and keep gateway capability checks provider-scoped. Any remaining desire to route image requests away from a text-only selected provider should be opened as a separate routing proposal with explicit product review and tests.
