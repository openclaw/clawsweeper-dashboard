## Summary

Keep open: current main still drops the gateway freshness signal before the TUI footer, while this focused PR preserves it so fresh missing totals render as `0` and stale or unknown totals remain `?`. I found no blocking review findings and the supplied terminal proof exercises the changed TUI path.

## What This Changes

The PR adds `totalTokensFresh` to TUI session metadata, forwards it in session lists, updates the session-info merge to map fresh missing totals to `0`, and adds a focused regression test.

## Best Possible Solution

Land the narrow TUI propagation of the existing gateway freshness signal so fresh missing totals render as `0` while stale or unknown totals remain `?`.

## Reproduction Assessment

Yes at source level: current main loads `chat.history` session info with `clearMissingUsage` and clears a missing `totalTokens` to `null`, which the footer renders as `?`. I did not run a live TUI repro in this read-only review, but the PR body supplies after-fix terminal proof for fresh and stale payloads.

## Solution Assessment

Yes: this fixes the TUI-owned session-info merge boundary using the existing gateway `totalTokensFresh` contract. A shared formatter fallback would be broader and could hide stale or genuinely unknown totals.
