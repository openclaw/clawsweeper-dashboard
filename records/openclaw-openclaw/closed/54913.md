## Summary

Current `main` already implements the requested generic pre-dispatch message gate via the new `before_dispatch` plugin hook, so non-channel plugins can now claim/drop inbound messages before the model runs.

## Evidence

- **Generic plugin hook surface:** Non-channel plugins use `definePluginEntry(...)`, and the shared plugin API exposes typed `api.on(...)` hook registration for all `PluginHookName` values, including `before_dispatch`.
  - file: [src/plugin-sdk/plugin-entry.ts:194](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/plugin-sdk/plugin-entry.ts#L194)

  - file: [src/plugins/hook-types.ts:247](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/plugins/hook-types.ts#L247)

## Risks / Open Questions

- The original request specifically named `inbound_claim`; current main solves the capability with `before_dispatch` instead. If maintainers still want this exact behavior on `inbound_claim`, that would be follow-up design work rather than an unresolved bug in the reported use case.
- The implementing commit is newer than the latest tagged release `v2026.4.22`, so this is fixed on `main` but not yet in a released tag.
