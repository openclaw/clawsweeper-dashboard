## Summary

Current `main` already supports custom per-session names in Control UI: the Sessions page exposes an editable `Label` field backed by `sessions.patch`, the gateway persists that label and surfaces it as session display data, and the chat session picker prefers the label/display name over the raw session key.

## Evidence

- **Control UI exposes editable session labels:** The Sessions view includes a `Label` column with an input bound to `row.label`; changing it calls `props.onPatch(row.key, { label: value })`, which is the Control UI rename path.
  - file: [ui/src/ui/views/sessions.ts:381](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/ui/src/ui/views/sessions.ts#L381)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Gateway persists session labels:** `sessions.patch` accepts `label`, validates it, rejects duplicates, and stores it on the session entry.
  - file: [src/gateway/sessions-patch.ts:219](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/gateway/sessions-patch.ts#L219)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Session rows and chat picker render the custom name:** Gateway session rows derive `displayName` from `entry.displayName`, group metadata, `entry.label`, or origin label; the chat session selector then prefers `label`/`displayName` over the raw key.
  - file: [src/gateway/session-utils.ts:1256](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/gateway/session-utils.ts#L1256)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Control UI selection logic prefers labels:** The chat session controls compute the selected label from grouped session options, and `resolveSessionDisplayName()` returns `row.label` before falling back to `row.displayName` or the raw key-derived fallback.
  - file: [ui/src/ui/chat/session-controls.ts:30](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/ui/src/ui/chat/session-controls.ts#L30)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Behavior is covered by tests:** Gateway tests verify `sessions.patch` stores `label` and `sessions.list` returns both `label` and `displayName`; UI tests verify grouped session labels are shown instead of raw session keys/display names.
  - file: [src/gateway/server.sessions.gateway-server-sessions-a.test.ts:1192](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/gateway/server.sessions.gateway-server-sessions-a.test.ts#L1192)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Shipped in latest release:** Verified the same label-editing and label-display logic exists in release commit `a9797214338ba31c52c796adbb75afb16e0684a9` (`v2026.4.23`).
  - command: `git show a9797214338ba31c52c796adbb75afb16e0684a9:ui/src/ui/views/sessions.ts | rg -n "<th>Label</th>|props\.onPatch\(row\.key, \{ label: value \}\)"; git show a9797214338ba31c52c796adbb75afb16e0684a9:ui/src/ui/chat/session-controls.ts | rg -n "resolveSessionDisplayName|label && label !== key|displayName && displayName !== key"`
