## Summary

## What This Changes

The PR changes OpenAI-compatible streaming so `delta.reasoning_content` is deleted before stream processing when the model has reasoning disabled.

## Best Possible Solution

Handle disabled-reasoning suppression at the normalized reasoning-delta boundary, preserve the existing MiMo final-answer promotion path, cover all supported reasoning carriers with regression tests, and require redacted after-fix curl/log proof.

## Reproduction Assessment

Yes, source-reproducible: a current-main `processOpenAICompletionsStream` mock chunk with `model.reasoning === false` and `delta.reasoning_content` still becomes a thinking block. The proposed patch can also be source-reproduced as lossy for MiMo-style reasoning-only final answers because it deletes the field before the existing promotion wrapper can see it.

## Solution Assessment

No. The current patch is too narrow because it deletes one raw field at the wrong layer instead of applying a normalized disabled-reasoning policy that covers all reasoning carriers and preserves known provider fallback behavior.
