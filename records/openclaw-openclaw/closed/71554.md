## Summary

Keep open. The proposed Telegram recovery change has a concrete correctness hole for archived previews, and the PR also mixes in unrelated CLI loader changes outside the reported Telegram fix.

## Evidence

- **Archived preview edits already use a different snapshot than the live lane state:** `consumeArchivedAnswerPreviewForFinal` calls `tryUpdatePreviewForLane` with `previewMessageId: archivedPreview.messageId` and `previewTextSnapshot: archivedPreview.textSnapshot`, so the edit target can differ from `lane.lastPartialText`. Any ambiguous-error fallback must key off the edited preview snapshot, not the active lane snapshot.
  - file: [extensions/telegram/src/lane-delivery-text-deliverer.ts:431](https://github.com/openclaw/openclaw/blob/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5/extensions/telegram/src/lane-delivery-text-deliverer.ts#L431)
  - command: `nl -ba extensions/telegram/src/lane-delivery-text-deliverer.ts | sed -n '330,455p'`
  - sha: [84dc9f12f1b9](https://github.com/openclaw/openclaw/commit/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5)
- **Ambiguous final-edit handling currently lives inside `tryEditPreviewMessage` without the archived snapshot:** `tryEditPreviewMessage` receives `lane`, `messageId`, and final-edit flags, but not `previewTextSnapshot`; current ambiguous errors retain the preview there. A PR change that checks `args.lane.lastPartialText` in this function would read the wrong text when finalizing an archived preview target.
  - file: [extensions/telegram/src/lane-delivery-text-deliverer.ts:225](https://github.com/openclaw/openclaw/blob/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5/extensions/telegram/src/lane-delivery-text-deliverer.ts#L225)
  - command: `nl -ba extensions/telegram/src/lane-delivery-text-deliverer.ts | sed -n '225,310p'`
  - sha: [84dc9f12f1b9](https://github.com/openclaw/openclaw/commit/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5)
- **Tests already prove active and archived previews can diverge:** Current Telegram tests cover archived-preview behavior separately from the active lane: one test keeps the active preview when an archived final edit target is missing, and another keeps the archived preview when the final text regresses. That makes the snapshot/source mismatch a real path, not a hypothetical edge case.
  - file: [extensions/telegram/src/lane-delivery.test.ts:481](https://github.com/openclaw/openclaw/blob/84dc9f12f1b9e36f7bfc9bea438bce3852ed9dd5/extensions/telegram/src/lane-delivery.test.ts#L481)

That is unrelated to issue `#71525`'s Telegram preview-finalization bug and makes this harder to merge conservatively as-is.
