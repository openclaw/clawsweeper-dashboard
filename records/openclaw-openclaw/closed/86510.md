## Summary

Current main already has the requested native HTTP callback surface: Gateway hooks expose authenticated POST endpoints for wake events, isolated agent turns, and mapped custom payloads, and the same surface is present in the latest release.

## What This Changes

Request to add a native local HTTP callback endpoint so external scripts can notify or trigger OpenClaw agent work without polling or a separate bridge.

## Best Possible Solution

Use the shipped Gateway webhooks surface (`/hooks/wake`, `/hooks/agent`, or mapped hooks) with a dedicated hook token instead of adding a second `_internal/callback` endpoint.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. Source, docs, and tests show the requested callback-style external trigger path already exists on current main.

## Solution Assessment

Yes: the existing Gateway webhooks are the maintainable solution because they already provide authenticated local HTTP POST triggers, agent-turn dispatch, custom mappings, and safety controls without adding a parallel callback API.
