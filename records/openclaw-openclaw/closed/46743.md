## Summary

Current `main` already supports disabling Ollama ambient model discovery. The implemented switch is `plugins.entries.ollama.config.discovery.enabled: false` (with legacy code support for `models.ollamaDiscovery.enabled`), the discovery path exits early when disabled, there is test coverage for the disabled case, and the change is listed in the `v2026.4.22` changelog.

## Evidence

- **Ollama plugin exposes a discovery disable flag:** The bundled Ollama plugin manifest defines `configSchema.discovery.enabled` and UI help text saying that `false` skips implicit startup discovery of ambient local or remote Ollama models.
  - file: [extensions/ollama/openclaw.plugin.json:27](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/extensions/ollama/openclaw.plugin.json#L27)
  - command: `nl -ba extensions/ollama/openclaw.plugin.json | sed -n '1,120p'`
- **Discovery code honors the disable flag:** `resolveOllamaDiscoveryResult(...)` reads `pluginConfig.discovery?.enabled` (and legacy `models.ollamaDiscovery?.enabled`) and returns `null` before ambient discovery when the flag is `false` and there are no explicit models.
  - file: [extensions/ollama/src/discovery-shared.ts:102](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/extensions/ollama/src/discovery-shared.ts#L102)
  - command: `nl -ba extensions/ollama/src/discovery-shared.ts | sed -n '94,109p'`
- **Regression test covers disabled discovery:** Unit test `skips ambient discovery when plugin discovery is disabled` asserts the provider returns `null` and does not call the discovery builder when `plugins.entries.ollama.config.discovery.enabled` is `false`.
  - file: [extensions/ollama/index.test.ts:184](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/extensions/ollama/index.test.ts#L184)
  - command: `nl -ba extensions/ollama/index.test.ts | sed -n '180,205p'`
- **Shipped in release notes:** The changelog for `2026.4.22` explicitly says toggling `plugins.entries.ollama.config.discovery.enabled` takes effect without a restart.
  - file: [CHANGELOG.md:235](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/CHANGELOG.md#L235)
  - command: `nl -ba CHANGELOG.md | sed -n '228,238p'`
