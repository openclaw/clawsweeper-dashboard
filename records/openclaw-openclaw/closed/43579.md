## Summary

Deepgram Aura TTS is not implemented on current main, but the remaining request is a vendor speech-provider implementation that fits the existing plugin/ClawHub path. Current main already has registry-driven TTS provider selection, config, Gateway RPCs, slash commands, SecretRef handling, normal audio output, voice-note output, and telephony synthesis hooks; the existing Deepgram bundled plugin is STT/media-understanding only, so Aura TTS does not reveal a missing core API.
