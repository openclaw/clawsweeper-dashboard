## Summary

Keep open: this is an active, contributor-authored Codex migration bug-fix PR with the protected `maintainer` label, and the latest diff still has two merge-blocking correctness issues in the migration apply path.

## What This Changes

The branch prewarms the target Codex app-server during post-install migration, polls target `plugin/list` for `openai-curated`, adds warning-status migration output and next steps, and extends focused migration/output tests.

## Best Possible Solution

Keep the prewarm and bounded polling, but make migration apply own shared-client cleanup on every path and preserve `plugin_missing` as an error once the target marketplace has loaded.

## Reproduction Assessment

No reviewer-run current-main live reproduction is present. Source inspection shows current main asks the target app-server for `plugin/list` without a readiness wait, and the PR includes terminal screenshots for after-fix success and timeout-warning paths.

## Solution Assessment

No as written. Polling/prewarming is a narrow maintainable fix, but the branch must not leave shared app-server clients undisposed on direct CLI runs or downgrade a loaded-marketplace `plugin_missing` failure to a generic warning.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Dispose the shared app-server for direct migrations:** `extensions/codex/src/migration/apply.ts:177`
  - body: Changing plugin activation from `isolated: true` to `isolated: false` routes ordinary `openclaw migrate codex` through the shared app-server client, but only the post-install onboarding path calls `prepareApply` and disposes that client. Because the stdio transport is spawned with piped streams and is only unref'd during explicit close, the direct CLI migration and the new "run `openclaw migrate codex` after onboarding" retry path can leave the Codex app-server attached or hang instead of exiting; either keep non-onboarding apply isolated or make every apply path own and dispose the shared client.
  - confidence: 0.86
- **[P2] Preserve real missing-plugin failures:** `extensions/codex/src/migration/apply.ts:206-217`
  - body: After the retry returns a response containing `openai-curated`, `plugin_missing` means the selected plugin is absent from the loaded marketplace, not that marketplace discovery is still warming up. This branch converts that case to a warning, and warnings are not counted as errors, so migrations can succeed while silently leaving a renamed or removed plugin unresolved; gate the fallback on the final response still missing the marketplace or keep `plugin_missing` as an error.
  - confidence: 0.88
