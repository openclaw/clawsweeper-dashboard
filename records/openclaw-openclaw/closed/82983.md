## What This Changes

The issue asks OpenClaw to preserve or backfill Xiaomi MiMo `reasoning_content` during multi-turn tool-call replay so follow-up turns do not fail with `400` errors.

## Best Possible Solution

Keep the current provider-owned Xiaomi replay fix; any new failure on `v2026.5.16-beta.1` or later should be reported with the exact model ref, provider config, and gateway logs.

## Reproduction Assessment

Yes. The original failure has high-confidence reproduction evidence from Xiaomi's documented 400 behavior, the earlier live logs on the canonical report, and the pre-fix source gap; I did not re-run a live Xiaomi request during this read-only pass.

## Solution Assessment

Yes. The current implementation keeps Xiaomi-owned behavior in the Xiaomi plugin and preserves MiMo replay fields in the OpenAI-compatible transport/proxy paths without adding a user-facing config knob.
