## Summary

Keep open: current main and the latest release still lack the persisted paused-session route binding needed for yielded subagent completions to resume the waiting parent instead of falling through to direct handoff.

## What This Changes

The issue reports that subagent completion announcements to a yielded external-route parent create a new route session instead of resuming the paused parent session.

## Best Possible Solution

Persist and resolve a bounded paused-session binding for yielded route entries so completion handoffs wake the existing parent session before direct fallback can allocate a fresh run.

## Reproduction Assessment

Yes, at source level and with reporter-provided live evidence. Current main has no paused-session route marker, yielded runs leave active-run state, and inactive queue attempts can fall through to direct announce handoff.

## Solution Assessment

Yes. The narrow maintainable direction is a bounded paused-session binding or equivalent resolver in the existing session store and announce delivery path.
