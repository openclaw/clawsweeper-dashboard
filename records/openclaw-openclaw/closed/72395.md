## Summary

Current main already fixes the Windows raw drive-letter ESM loader failure reported for Discord delivery and the browser plugin in v2026.4.24. The shared plugin loader now disables native Jiti on Windows, converts Windows absolute import paths to file:// URLs where needed, and has regression coverage for channel, browser/public-surface, and bundled sidecar plugin load paths.

## Best Possible Solution

Close this as implemented on main, keep the shared Windows-safe plugin loader behavior, and ship it in the next release. If the problem recurs after a release containing this code, open a fresh issue with current OpenClaw and Node versions, full gateway logs, and the exact bundled plugin ids still receiving raw Windows paths.
