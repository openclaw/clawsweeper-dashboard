## Summary

Current `main` already parses Telegram forum-topic session keys into numeric chat id plus `threadId`, and the async `sessions_send` announce path is covered by a regression test asserting the correct gateway send payload.

## Evidence

- **Announce target resolution uses parsed session id plus separate thread id:** `resolveAnnounceTargetFromKey` calls `resolveSessionConversationRef(sessionKey)`, builds `group:${parsed.id}`, normalizes it through the channel plugin, and returns `threadId: parsed.threadId`. That means Telegram topic session keys no longer pass `group:<chatId>` through as the final recipient.
