## Summary

Keep open: the bug is real and the PR is focused, but the patch is not merge-ready because stripping `XDG_CACHE_HOME` from mcporter also changes the environment inherited by local `qmd mcp` servers, which can make searches read the wrong SQLite memory index.

## What This Changes

The branch adds a separate `mcporterEnv` that removes `XDG_CONFIG_HOME` and `XDG_CACHE_HOME` from mcporter spawns and updates qmd-manager env test expectations.

## Best Possible Solution

Fix mcporter config discovery without dropping QMD runtime env from the qmd MCP server path, then prove both direct qmd and mcporter-backed search use the intended per-agent index.

## Reproduction Assessment

Yes, source inspection reproduces the original issue and the likely patch regression: current main leaks `XDG_CONFIG_HOME` to mcporter, while the patch would also remove `XDG_CACHE_HOME` from the qmd MCP server inheritance path. I did not run a live qmd/mcporter daemon in this read-only review.

## Solution Assessment

No, not as submitted. The right fix needs separate mcporter config-discovery handling and qmd server runtime env handling instead of dropping all QMD XDG state from the mcporter-launched qmd path.
