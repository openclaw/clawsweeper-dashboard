## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Wire the audit RPC into gateway handlers:** `src/gateway/server-methods-list.ts:44`
  - body: Adding `exec.approval.auditLog` to the public method list is not enough to make it callable. `handleGatewayRequest` dispatches aux methods only through `createGatewayAuxHandlers().extraHandlers`, and that map still lacks this method, so clients would get `unknown method` for the new RPC.
  - confidence: 0.9
- **[P1] Gate audit-log reads with approval scope:** `src/gateway/method-scopes.ts:109`
  - body: This puts `exec.approval.auditLog` under `operator.read`, but the returned records contain command text, args, agent/session identifiers, and resolver identity. Existing live exec approval methods require `operator.approvals`, so read-scoped tokens would gain historical approval data they cannot access live.
  - confidence: 0.91
- **[P1] Reuse shared argv redaction before persistence:** `src/infra/exec-approval-audit.ts:36-43`
  - body: The array redactor only masks uppercase `KEY=value` entries. Common argv forms such as `--token secret`, `--password=secret`, lowercase `token=secret`, bearer headers, and raw token-shaped values can still be written to `approvals.jsonl`; use the shared argv/tool redaction path or equivalent coverage before persisting.
  - confidence: 0.88
- **[P2] Avoid synchronous full-log reads:** `src/infra/exec-approval-audit.ts:90`
  - body: `readFileSync` loads and parses the complete append-only audit file before applying `limit`. A long-lived gateway can accumulate a large file, so one audit-log request can block the Node event loop; use an async bounded or streaming read path.
  - confidence: 0.84
- **[P2] Register the audit RPC protocol contract:** `src/gateway/protocol/schema/exec-approvals.ts:156-169`
  - body: Adding this params schema and validator is not enough for a public gateway RPC. The method also needs a result schema plus protocol registry, generated schema/client artifacts, docs, and focused protocol tests so clients learn the new contract.
  - confidence: 0.78
