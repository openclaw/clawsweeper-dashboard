## What This Changes

## Best Possible Solution

Keep #68214 as the canonical tracker and implement the feature there as an additive, privacy-minimal channel access-control design: a blocked/unauthorized event or hook with constrained metadata, plus any explicit pairing-reply suppression config maintainers choose to support, with docs and channel tests updated together.
