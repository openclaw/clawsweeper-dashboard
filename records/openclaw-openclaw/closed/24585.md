## Summary

Current `main` already gives plugin hooks enough signal to distinguish heartbeat runs: `PluginHookAgentContext` now carries `trigger`, and heartbeat-driven hook invocations pass `"heartbeat"` there. That makes the requested heartbeat detection available on `main`, albeit via `ctx.trigger` rather than a separate `isHeartbeat` boolean.

## Risks / Open Questions

- The proving commit appears newer than the latest tagged release `v2026.4.22`, so this looks resolved on `main` but not yet released.

## Close Comment
