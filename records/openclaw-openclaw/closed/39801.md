## Summary

Keep open. Current `main` still routes isolated cron agent turns through the embedded runner, and the Kimi provider path still preserves native Anthropic-style tool payloads instead of rewriting them to OpenAI-style `{type:"function", function:{...}}`. A current test explicitly asserts that Kimi tool payloads stay in the native `{name, description, input_schema}` shape, which matches the reported failing shape rather than the requested fix.

## Evidence

- **Isolated cron jobs still use the embedded runner path:** `executeCronRun()` calls `runEmbeddedPiAgent(...)` for non-CLI providers and forwards `bootstrapContextRunKind: "cron"` plus `toolsAllow`, so isolated cron agent turns still use the embedded request path the report points at.
