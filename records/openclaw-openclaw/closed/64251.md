## Summary

Current `main` already routes CLI-backed follow-up runs through the dedicated CLI runner instead of the embedded model-resolution path, and it includes targeted `codex-cli/gpt-5.4` follow-up regression coverage. The fix is present on `main` but I could not tie it to a published release tag from this checkout, so `fixedRelease` remains null.

## Evidence

- **Current follow-up path bypasses embedded model resolution for CLI providers:** `runAgentTurnWithFallback` checks `isCliProvider(provider, runtimeConfig)` and, when true, executes follow-up turns via `runCliAgent(...)` instead of `runEmbeddedPiAgent(...)`. That matches the reporter's traced root cause and avoids the embedded `resolveModelAsync(...)` failure path.
