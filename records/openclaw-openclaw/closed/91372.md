## Summary

Keep this issue open: current main and the latest release still lack a reserved-literal guard, and https://github.com/openclaw/openclaw/pull/94107 is the active open closing PR rather than merged code.

## What This Changes

The issue requests fail-closed gateway/outbound handling for reserved bare target strings such as `current`, `self`, `this`, and `me` when they are not explicitly bound to a real Telegram/session destination.

## Best Possible Solution

Keep this canonical issue open until maintainers land or replace the open candidate with a directory-aware fail-closed resolver that blocks reserved-word misses before literal Telegram fallback.

## Reproduction Assessment

Yes. Source inspection shows current main still has no reserved-literal miss guard, Telegram accepts username-shaped strings like `current`, and delivery planning can carry the explicit target forward; I did not run a live Telegram send in this read-only review.

## Solution Assessment

No for a blanket reserved-word rejection; the safer likely fix is the linked directory-first fail-closed resolver path, because real configured Telegram destinations named `current`, `self`, `this`, or `me` may need to keep working.
