## Summary

Current `main` already implements the requested Google Chat fallback for invalid thread replies by sending threaded messages with Google Chat's `REPLY_MESSAGE_FALLBACK_TO_NEW_THREAD` option, and the same logic is present in release `v2026.4.23` with test coverage.

## Evidence

- **Threaded Google Chat sends use API fallback:** `sendGoogleChatMessage()` adds `messageReplyOption=REPLY_MESSAGE_FALLBACK_TO_NEW_THREAD` whenever a `thread` is supplied, which is the API-side fallback requested by the issue instead of repeatedly retrying the same invalid thread name.
