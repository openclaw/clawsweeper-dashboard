## Summary

Current `main` already resets the WhatsApp watchdog on reconnect and includes a regression test for the cascading-reconnect case described here.

## Evidence

- **New connections start with a fresh idle baseline:** `createLiveConnection` initializes each reopened connection with a new `startedAt` timestamp and `lastInboundAt: null`, so the previous idle timestamp is not carried across reconnects.
  - file: [extensions/whatsapp/src/connection-controller.ts:103](https://github.com/openclaw/openclaw/blob/6a61cb73c5622749d7940fd8d8b90c93adbb5081/extensions/whatsapp/src/connection-controller.ts#L103)

  - sha: [6a61cb73c562](https://github.com/openclaw/openclaw/commit/6a61cb73c5622749d7940fd8d8b90c93adbb5081)
- **Watchdog timeout is computed from the current connection only:** The watchdog baseline is `connection.lastInboundAt ?? connection.startedAt`, which resets to the new connection start time after reconnect instead of continuing to count from an old inbound message.
  - file: [extensions/whatsapp/src/connection-controller.ts:563](https://github.com/openclaw/openclaw/blob/6a61cb73c5622749d7940fd8d8b90c93adbb5081/extensions/whatsapp/src/connection-controller.ts#L563)

- **Old timers are cleared before retrying:** Reconnect flow clears the prior heartbeat and watchdog intervals in `closeCurrentConnection`, and `monitorWebChannel` calls that cleanup before sleeping and opening the next connection, which prevents the old watchdog from cascading into the replacement connection.
  - file: [extensions/whatsapp/src/connection-controller.ts:505](https://github.com/openclaw/openclaw/blob/6a61cb73c5622749d7940fd8d8b90c93adbb5081/extensions/whatsapp/src/connection-controller.ts#L505)

  - file: [extensions/whatsapp/src/auto-reply/monitor.ts:496](https://github.com/openclaw/openclaw/blob/6a61cb73c5622749d7940fd8d8b90c93adbb5081/extensions/whatsapp/src/auto-reply/monitor.ts#L496)

The WhatsApp reconnect path now creates each replacement connection with a fresh `startedAt` and `lastInboundAt: null` in `extensions/whatsapp/src/connection-controller.ts:103`, computes watchdog staleness from `lastInboundAt ?? startedAt` in `extensions/whatsapp/src/connection-controller.ts:563`, and clears the old heartbeat/watchdog timers before retrying in `extensions/whatsapp/src/connection-controller.ts:505` and `extensions/whatsapp/src/auto-reply/monitor.ts:515`.
