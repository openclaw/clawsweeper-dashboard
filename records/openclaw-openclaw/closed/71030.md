## Summary

Current main already handles the reported QMD diff-format promotion path. QMD snippets may still be stored in diff form, but promotion rehydration no longer requires exact stored-snippet equality: it scans the live daily note, accepts a live source window contained inside the stored candidate snippet, and writes the live snippet/range to MEMORY.md. The shipped v2026.4.24 release contains this behavior.

## Best Possible Solution

Keep the shipped live-file rehydration behavior. It solves the observable zero-promotion failure without changing the recall-store format; a narrow regression test combining QMD diff snippets with promotion rehydration would be the only useful follow-up.
