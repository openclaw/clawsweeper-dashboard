## Summary

Keep open. Current `main` still lacks the silent-cron/node-host exec notification suppression this PR proposes, and the PR discussion points to a targeted, coherent fix rather than something already shipped or out of scope.

## Evidence

- **Silent cron runs on main do not pass exec notification overrides:** `createCronPromptExecutor` forwards delivery target and message-tool policy into `runEmbeddedPiAgent`, but on current `main` it does not pass any `execOverrides` keyed off `deliveryRequested`, so `delivery.mode=none` has no path here to disable automatic exec completion notifications.
