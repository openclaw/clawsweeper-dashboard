## Summary

Keep open: current main still allows scoped plugin-tool registry reuse when any requested plugin id has tools, and open PR #76781 is already the linked implementation path.

## What This Changes

Require scoped plugin tool registry reuse to cover every selected plugin id before skipping cold-load of missing requested plugin tools.

## Best Possible Solution

Review and land #76781, or an equivalent narrow fix with regression coverage, then let this issue close after the implementation merges.

## Reproduction Assessment

Yes, at source level: a registry whose selected plugin set includes A and B but whose `tools` entries only include A passes the current `some(...)` check and skips cold-loading B's tools. I did not run a live Feishu/Lark scenario.

## Solution Assessment

Yes: requiring full requested plugin-id tool coverage before registry reuse is the narrow generic fix and preserves the existing cold-load path for missing selected tools.
