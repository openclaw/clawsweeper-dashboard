## Summary

Close as superseded by open #41548. Current main has Gateway transcript notification plumbing, but the bundled TUI still does not subscribe to or handle live `session.message` transcript events, so the broad Telegram/TUI cross-surface sync ask is not implemented. Open PR #73402 covers the narrow TUI live transcript gap and explicitly treats this as broader related context; the remaining unified inbox/sync work is better tracked on #41548.

## Best Possible Solution

Consolidate the product request under #41548 as the canonical unified cross-channel sync tracker, and review #73402 for the narrower TUI live transcript subscription gap. Preserve the KeepMyClaw and `sessions_send` discussion as design input on the canonical thread.
