## Summary

Current `main` does not have a separate dashboard/node offline-alert state that could stay active after recovery. The nodes surface is driven by a single `connected` boolean from `node.list`, and the dashboard renders only that live status, so the reported `Online` plus active `Gateway Offline` contradiction does not match the current implementation.

## Evidence

- **Node list payload has no alert/incidents field:** `NodeListNode` exposes pairing/connectivity metadata only (`paired`, `connected`, timestamps, caps, commands). There is no persisted alert or incident field in the node/dashboard payload.
  - file: [src/shared/node-list-types.ts:1](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/shared/node-list-types.ts#L1)
  - command: `nl -ba src/shared/node-list-types.ts | sed -n '1,24p'`
  - sha: [6944d7025d9c](https://github.com/openclaw/openclaw/commit/6944d7025d9c0ba4891cbe646ab5dab039d504ca)
- **Gateway builds node state from live connection presence:** `buildEffectiveKnownNode` derives `connected` directly from whether a live node session exists (`connected: Boolean(live)`), with no separate alert lifecycle to reconcile.
  - file: [src/gateway/node-catalog.ts:91](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/gateway/node-catalog.ts#L91)
  - command: `nl -ba src/gateway/node-catalog.ts | sed -n '88,131p'`
  - sha: [6944d7025d9c](https://github.com/openclaw/openclaw/commit/6944d7025d9c0ba4891cbe646ab5dab039d504ca)
- **`node.list` returns merged live node records only:** The RPC handler assembles the catalog from paired devices, paired nodes, and `context.nodeRegistry.listConnected()`, then responds with `nodes`; it does not load or emit any active alert set.
  - file: [src/gateway/server-methods/nodes.ts:697](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/gateway/server-methods/nodes.ts#L697)
  - command: `nl -ba src/gateway/server-methods/nodes.ts | sed -n '697,716p'`
  - sha: [6944d7025d9c](https://github.com/openclaw/openclaw/commit/6944d7025d9c0ba4891cbe646ab5dab039d504ca)
- **Nodes UI renders only the live connected/offline chip:** `renderNode` reads `node.connected` and shows either `connected` or `offline`. There is no second alert badge/list on this screen that could remain stale after reconnection.
  - file: [ui/src/ui/views/nodes.ts:439](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/ui/src/ui/views/nodes.ts#L439)
  - command: `nl -ba ui/src/ui/views/nodes.ts | sed -n '436,468p'`
  - sha: [6944d7025d9c](https://github.com/openclaw/openclaw/commit/6944d7025d9c0ba4891cbe646ab5dab039d504ca)
- **Overview attention items do not include node-offline alerts:** Current dashboard attention items are limited to gateway error, missing operator scope, missing skill deps, blocked skills, failed/overdue cron, and model-auth status. No node/gateway-offline alert history exists here either.
  - file: [ui/src/ui/app-settings.ts:658](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/ui/src/ui/app-settings.ts#L658)
