## What This Changes

Adds an opt-in Claude CLI tmux execution mode while also changing broad CLI runner, plugin SDK, Feishu/Telegram, lesson-engine, scripts, docs, dependency, and patch-artifact surfaces.

## Best Possible Solution

Split this into a focused Claude CLI tmux PR that preserves current child-mode and image behavior, uses a non-argv secret handoff, keeps plugin code behind SDK seams, and includes redacted live tmux/Claude proof before rollout.

## Reproduction Assessment

There is no original bug to reproduce because this is a feature PR. The main blockers are source-reproducible: image paths are produced but not appended for Claude's `@` stdin contract, and auth env values are placed into tmux argv.

## Solution Assessment

No. The tmux direction may be useful, but this branch is too broad and currently regresses image prompts, exposes auth env values in process arguments, and crosses plugin boundaries.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Pass tmux auth without putting values in argv:** `src/agents/cli-runner/tmux/manager.ts:144`
  - body: `envEntries` includes the auth-bearing runtime env, then expands each value into `tmux new-session -e KEY=value`. That puts tokens such as Anthropic/OpenAI credentials in the tmux client argv during startup, which local process listings can expose. Use a 0600 env file, fd, or another non-argv handoff for the launcher.
  - confidence: 0.91
- **[P2] Keep @ image refs in the prompt before CLI execution:** `src/agents/cli-runner/execute.ts:626-628`
  - body: Claude's backend uses stdin with `imageArg: "@"`; current main appends `@<path>` refs into the prompt for that case. This branch only appends image paths when there is no `imageArg`, and the tmux call only receives `prompt`, so Claude image turns lose their images. Preserve the existing helper behavior and cover both child and tmux paths.
  - confidence: 0.88
- **[P2] Keep Feishu production code behind SDK seams:** `extensions/feishu/src/bot.ts:20`
  - body: This imports a core auto-reply queue type by walking from the Feishu plugin into `src/**`. Bundled plugins are treated like third-party packages, so production code should use an SDK/exported contract or a local structural type instead of depending on a private core path.
  - confidence: 0.84
- **[P3] Define the detect failure helper before calling it:** `scripts/tools/request-merge:49`
  - body: `request-merge --detect` calls `fail` when PR detection fails, but this script does not define a `fail` function. That error path will report `fail: command not found` instead of the intended diagnostic.
  - confidence: 0.82
