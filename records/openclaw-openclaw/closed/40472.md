## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Clear pending chat runs on external reset:** `src/tui/tui-event-handlers.ts:566-568`
  - body: Current main tracks `state.pendingChatRunId` after `chat.send` returns and before the first chat event arrives. This reset handler suppresses active/session/finalized runs, then clears `activeChatRunId`, but it never suppresses or nulls the pending run id, so a reset in that gap can still let the old run become active when its first event arrives and repaint the reset session. Add `pendingChatRunId` to the suppressed set, clear it on reset, and cover that pre-event gap in the handler tests.
  - confidence: 0.87
