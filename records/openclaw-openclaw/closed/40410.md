## Summary

Current `main` already contains explicit protections for this failure mode: destructive config writes that would drop `gateway.mode` or sharply shrink `openclaw.json` are rejected, and suspicious clobbered configs are auto-restored from backup/last-known-good snapshots. The behavior is covered by regression tests and documented as shipped in `v2026.4.22`.
