## Summary

Keep open. The PR addresses a valid current-main regression, but the submitted schema shape is not ready because transform branches are unsafe for the generated JSON schema path and the required generated config artifacts are missing.

## What This Changes

The PR adds a shared `VisibleRepliesSchema` that coerces boolean `visibleReplies` values to existing enum strings, applies it to global and group message config, adds focused tests, and updates the changelog.

## Best Possible Solution

Land a narrow config-boundary compatibility fix that accepts booleans at both visible-reply paths, preserves normalized runtime enum output, and keeps generated schema/docs constrained and current.

## Reproduction Assessment

Yes. Set `messages.groupChat.visibleReplies: true` or `messages.visibleReplies: true`; current main's enum-only Zod fields reject the value, and invalid snapshots are skipped by the gateway reload path.

## Solution Assessment

No, not as submitted. Coercing at the schema boundary is the right direction, but the implementation needs a JSON-schema-safe representation plus regenerated config schema/docs artifacts before merge.
