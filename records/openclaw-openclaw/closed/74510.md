## Summary

Current main and the latest release snapshot now cover the reported Control UI symptom: provider conversion and WebSocket streaming preserve assistant phase metadata, completed responses drop commentary-only text while keeping tool calls, and the shared Gateway/UI projection prefers final_answer text and hides explicit commentary. The broader #74928 audience-boundary report remains separate, but this narrower same-level commentary/final-answer rendering bug is implemented well enough to close.

## What This Changes

The issue asks for Control UI chat history to avoid rendering intermediate assistant commentary/progress as normal same-level replies alongside the final answer for one user turn.

## Best Possible Solution

Keep the shipped phase-aware provider conversion, history projection, and Control UI extraction path as the default behavior. If old phaseless transcripts remain painful, handle them with a separately scoped legacy repair that does not hide legitimate unphased assistant replies.

## Reproduction Assessment

Yes. The current source-level fixtures exercise the original forward path with commentary-phase tool-use responses and WebSocket partials, and they now verify that commentary-only text is not surfaced as the visible assistant reply.

## Solution Assessment

Yes. Preserving phase metadata at provider conversion/streaming boundaries and filtering explicit commentary in shared projection is the narrow maintainable fix; broadly hiding every unphased toolUse assistant message would risk suppressing valid legacy replies.
