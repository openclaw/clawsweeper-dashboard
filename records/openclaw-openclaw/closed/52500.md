## Summary

Keep open. Current main still lacks centralized numeric min/max bound enrichment in config validation errors, while the field and docs expose real numeric ceilings that operators should be able to recover from diagnostics without reading source.

## What This Changes

The issue requests explicit maximum-value guidance when schema-validated numeric config values exceed configured ceilings, across all ceiling-constrained config fields.

## Best Possible Solution

Add a central numeric-bound diagnostic helper in the config validation mapper that appends accurate max/min hints for Zod numeric bound issues, preserves existing allowed-values behavior, and covers inclusive/exclusive bounds with focused regression tests.

## Reproduction Assessment

Yes by source inspection. A value above `session.agentToAgent.maxPingPongTurns`'s `.max(20)` ceiling reaches `mapZodIssueToConfigIssue()`, which currently has no normalized numeric-bound hint path.

## Solution Assessment

Yes. The central validation mapper is the maintainable fix point because it can handle all schema-generated numeric ceiling and floor diagnostics without adding per-field messages.
