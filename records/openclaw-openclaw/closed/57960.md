## Summary

Current `main` already resolves Discord thread routing through the parent channel binding, and the shipped `v2026.4.23` release also contains that behavior plus parent-session model override fallback for thread contexts.

## Evidence

## Risks / Open Questions

- The exec-permission/tool-allowlist part is not implemented as a separate thread-specific permission path; the conclusion there is by design inference: once the thread resolves to the same parent-bound agent/session instead of the default agent, agent-scoped exec/tool policy follows that route.
