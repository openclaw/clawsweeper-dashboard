## Summary

Current main already routes Control UI updates from a symlinked/local OpenClaw checkout through the git/source update path, not the global npm install path that would replace an npm link. The issue has no comments, protected labels, or maintainer author association; the related title-search matches are unrelated to this npm-link update path.

## Best Possible Solution

Close this issue as already implemented: keep the current source-root detection and git/source update path for linked checkouts, and point developers at the documented `pnpm` source workflow or `openclaw update --channel dev` rather than adding npm-link-specific relinking logic.
