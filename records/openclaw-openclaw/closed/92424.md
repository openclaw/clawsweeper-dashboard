## Summary

This PR targets a real model-switch regression, but its registry lookup is keyed by the stale pre-switch model identity, so it cannot select a newly chosen provider or model ID. The added tests duplicate the helper instead of exercising AgentSession, and the supplied proof does not include an after-fix model switch.

## What This Changes

The PR changes eight AgentSession post-turn model reads to prefer a session-registry lookup keyed by the current model snapshot and adds seven tests for a standalone copy of that helper.

## Best Possible Solution

Synchronize AgentSession with the canonical selected provider/model at the switch boundary, or make post-turn work consume that canonical identity directly, then cover a different-provider and different-model-ID switch through the real AgentSession lifecycle.

## Reproduction Assessment

Yes. Current source separates persisted override identity from AgentSession's concrete model object, and the linked live observation shows the previous context window after selecting a different provider/model.

## Solution Assessment

No. Looking up the registry with the stale model's provider and ID can only refresh that old identity; the narrow maintainable fix must adopt the newly selected identity at the canonical state boundary.
