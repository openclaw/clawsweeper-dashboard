## Summary

Current `main` supersedes this PR with a different bundled-runtime-deps implementation: it stages from the trusted root workspace graph when possible, otherwise falls back to installs pinned to exact already-installed versions, invalidates on `pnpm-lock.yaml` changes, and verifies the packed activation path in release checks. The per-plugin `package-lock.json` approach in PR #58291 is no longer needed on `main`.
