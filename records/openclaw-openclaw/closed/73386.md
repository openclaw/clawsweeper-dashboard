## Summary

Keep this PR open. Current main still has the Ollama thinking-profile behavior the PR is trying to fix: discovered `/api/show` thinking capability is converted into catalog `reasoning`, but the runtime resolver and several `/think`/session surfaces can run without that catalog and then only advertise `off`. The linked regression report #73366 is still open, and this PR is an active implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Keep the PR open for normal review. The best path is to land a focused Ollama fix, likely with the proposed no-catalog regression coverage, while tightening the implementation if needed so discovered thinking support is keyed and invalidated in a way that matches provider/base URL and model refresh behavior.
