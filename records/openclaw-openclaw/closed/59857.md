## Summary

Current `main` already implements the underlying websocket cool-down and HTTP fallback behavior, with focused regression coverage. The open PR's model-scoped shared cooldown is superseded by a safer session-scoped implementation on `main`.

## Evidence

## Risks / Open Questions

- Current `main` uses a session-scoped cooldown, not the PR's proposed model-scoped cooldown. If there is still a cross-session handshake-storm case worth solving, that should be a fresh follow-up against the current implementation rather than keeping this PR open.
