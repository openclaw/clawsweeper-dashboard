## Summary

Current main already routes session-list previews through phase-aware assistant-text extraction, and the shipped v2026.4.22 release contains the same logic and tests. The preview path now prefers `final_answer` text and suppresses commentary-only assistant blocks, which matches this subagent session preview regression.

## Evidence

- **Gateway session preview uses transcript preview items:** `sessions.preview` resolves each session transcript and returns preview items from `readSessionPreviewItemsFromTranscript`, which is the gateway path session-list UIs consume.
  - file: [src/gateway/server-methods/sessions.ts:654](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/gateway/server-methods/sessions.ts#L654)
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Assistant preview text is phase-aware:** Transcript preview extraction calls `extractAssistantVisibleText(message)` for assistant rows, so previews use the assistant-visible text path instead of raw block concatenation.
  - file: [src/gateway/session-utils.fs.ts:634](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/gateway/session-utils.fs.ts#L634)
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Visible assistant text prefers final answers:** `extractAssistantVisibleText` explicitly prefers `final_answer` blocks and otherwise falls back to unphased visible text, preventing commentary-only content from becoming the session preview.
  - file: [src/shared/chat-message-content.ts:182](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/shared/chat-message-content.ts#L182)
