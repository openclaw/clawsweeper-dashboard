## Summary

Current main already covers the two repository-owned failures in this report: packaged bundled channel startup now prestages and aliases channel runtime dependencies before verify-only loading, and DeepInfra is now a bundled provider with API-key onboarding, OpenAI-compatible transport metadata, static plus refreshable catalog support, and regression tests. The implementation is on main only; no release tag contains the relevant runtime-dependency fix yet.

## What This Changes

The issue asks OpenClaw to make DeepInfra model discovery/routing work and to restore bundled Telegram, Discord, Slack, and Nostr channel module loading in packaged releases.

## Best Possible Solution

Keep the current main implementation and ship it in the next release; users needing the full DeepInfra catalog should use the documented provider-catalog path rather than a plain configured-model list.

## Reproduction Assessment

Yes for the released versions from the reporter's logs: v2026.4.29 fails with missing packaged channel modules and v2026.4.23 shows no configured DeepInfra rows. Current-main verification is source/test-backed and shows the relevant runtime dependency staging plus bundled DeepInfra provider paths are now present.

## Solution Assessment

Yes. The current solution keeps provider-specific DeepInfra behavior in the DeepInfra plugin and fixes packaged channel dependency loading through the generic bundled runtime dependency staging path, which is the maintainable boundary for these failures.
