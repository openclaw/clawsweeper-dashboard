## Summary

Current main no longer has the reported silent drop: reset/deleted usage-counted session archives are included in the session file scan and `buildSessionEntry` now returns their message content, with focused tests covering the archived transcript path.

## What This Changes

The issue asks for archived `.jsonl.reset.*` and `.jsonl.deleted.*` session transcripts to remain searchable by `memory_search` after session reset or deletion.

## Best Possible Solution

Keep the current main behavior: index usage-counted reset/deleted archives for durable recall while continuing to exclude bak and checkpoint artifacts from active session indexing.

## Reproduction Assessment

No for current main: source inspection shows the reported short-circuit no longer applies to usage-counted reset/deleted archives. The focused test path constructs those archive filenames and expects real content and line maps rather than zero chunks.

## Solution Assessment

Yes. The current implementation is the narrow maintainable fix because it keeps recall for reset/deleted session history while preserving the guard against backup/checkpoint double-indexing and internal cron/dreaming transcripts.
