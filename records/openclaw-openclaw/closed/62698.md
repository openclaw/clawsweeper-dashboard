## Summary

Current main already supports agent/file media sends to Telegram through the shared message tool and Telegram outbound adapter, including audio via `sendAudio`/`sendVoice`, video, and generic documents.

## Evidence

- **Shared message tool already accepts media attachments and voice hint:** The shared `message` tool schema includes `media`, `path`, `filePath`, `contentType`, and `asVoice`, so agents can send attached files through the normal outbound path instead of text-only replies.
  - file: [src/agents/tools/message-tool.ts:101](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/src/agents/tools/message-tool.ts#L101)

  - file: [src/infra/outbound/message-action-runner.ts:552](https://github.com/openclaw/openclaw/blob/c997a9f9788e0a96e8eab05e7ae94959e76cdb4a/src/infra/outbound/message-action-runner.ts#L552)
