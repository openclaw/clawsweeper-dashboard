## Summary

Keep open: the CSP warning is source-plausible, but this PR fixes it by globally permitting eval in the Control UI, lacks after-fix browser proof, and would close a related issue that also mentions a separate config-fetch 401.

## What This Changes

The PR adds `'unsafe-eval'` to the Gateway Control UI CSP `script-src` directive and updates the CSP unit tests to expect it.

## Best Possible Solution

Preserve the stricter CSP and disable Zod JIT for the Control UI bundle, then add focused tests, changelog coverage, and separate handling for the config-fetch 401 if it remains actionable.

## Reproduction Assessment

No live high-confidence reproduction was established in this read-only pass. Source inspection makes the CSP warning plausible because the Control UI startup path can invoke Zod parsing, and Zod 4.4.3 probes/uses `Function` unless configured `jitless`.

## Solution Assessment

No. Adding `'unsafe-eval'` is broader than needed when Zod provides a `jitless` switch specifically for environments that disallow eval.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88
