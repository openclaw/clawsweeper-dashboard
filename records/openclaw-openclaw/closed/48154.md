## Summary

Current main already changes Feishu DM authorization so allowlist mode does not depend on pairing-state persistence and accepts either Feishu open_id or user_id for sender matching. The current code and tests cover the reported symptom of allowlisted users being re-challenged after upgrade.

## Evidence

- **Allowlist mode bypasses pairing-store state:** Feishu DM handling now skips pairing-store allowFrom reads when `dmPolicy === "allowlist"` and authorizes directly from config before any pairing challenge path is considered.
  - file: [extensions/feishu/src/bot.ts:509](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/extensions/feishu/src/bot.ts#L509)
  - command: `nl -ba extensions/feishu/src/bot.ts | sed -n '470,580p'`
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **Sender matching accepts both open_id and user_id:** `resolveFeishuAllowlistMatch` checks the primary sender id plus additional immutable sender ids, so a configured allowlist entry can match either Feishu `open_id` or `user_id`.
  - file: [extensions/feishu/src/policy.ts:98](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/extensions/feishu/src/policy.ts#L98)
  - command: `nl -ba extensions/feishu/src/policy.ts | sed -n '1,260p'`
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **Comment flow preserves user_id for downstream authorization:** Drive comment events now carry `senderUserId`, and comment authorization passes that through the same allowlist matcher instead of relying only on pairing/open_id state.
  - file: [extensions/feishu/src/monitor.comment.ts:1239](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/extensions/feishu/src/monitor.comment.ts#L1239)
  - command: `nl -ba extensions/feishu/src/monitor.comment.ts | sed -n '1230,1295p'`
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **Regression tests cover the reported class of failure:** Tests explicitly cover matching a Feishu allowlist entry by `user_id`, preserving comment `user_id` for authorization, and allowing comment senders in `dmPolicy: "allowlist"` mode without issuing a pairing reply.
  - file: [extensions/feishu/src/policy.test.ts:218](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/extensions/feishu/src/policy.test.ts#L218)
