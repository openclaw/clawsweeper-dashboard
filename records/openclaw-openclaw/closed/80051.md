## Summary

Keep open: the PR targets a real ACP timeout/error propagation bug, but the current patch still forwards every late ACP event through the visible `onEvent` path and only supplies mocked unit-test proof.

## What This Changes

This PR changes `consumeAcpTurnStream` to invoke `onEvent` and capture `error` events after `eventGate` closes, and adds focused unit tests.

## Best Possible Solution

Preserve late error and terminal diagnostics after timeout through a filtered or non-visible path, with helper plus dispatch/projector coverage proving visible output stays suppressed.

## Reproduction Assessment

Yes. Source inspection shows current main closes `eventGate` in the timeout path while `consumeAcpTurnStream` keeps draining, and any later event hits the closed-gate `continue` before terminal tracking or error conversion; I did not run a live ACP timeout in this read-only review.

## Solution Assessment

No. Capturing late errors is the right direction, but forwarding every closed-gate event through `onEvent` is not the narrowest maintainable fix because ACP dispatch uses that path for visible projector output.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
