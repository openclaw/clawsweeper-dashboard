## Summary

Current `main` already defaults dreaming storage to `separate`, keeps the runtime fallback aligned, ships tests for the default, and the fix is present in the `v2026.4.23` release snapshot. The user-proposed PR #66948 was closed unmerged, but equivalent core behavior landed separately in commit `8c392f0019b0c2666cd3bcf739c3cd990a79383c`.

## Risks / Open Questions

- Historical daily files that were already polluted before the default changed are not migrated automatically by this fix.
- The issue body also mentions dreaming theme quality/stop-word output; that is a separate concern from the daily-file pollution bug and is not resolved by the storage default change alone.
