## Summary

Current main and the shipped v2026.6.8 release already solve the central Anthropic Enterprise token-profile header-routing problem; this PR repeats https://github.com/openclaw/openclaw/pull/94196 and adds an unnecessary runtime credential rewrite.

Is this the best way to solve the issue? No; the shipped transport-level split is narrower and already released, while this PR adds a duplicate runtime config rewrite path.
