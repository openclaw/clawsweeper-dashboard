## Summary

Keep open: current `main` still has a source-proven gap where provider output text transforms do not restore tool-call argument data before tool execution.

## What This Changes

The issue requests applying provider text transform output replacements to `toolcall_delta` and `toolcall_end` assistant events so transformed tool-call arguments are restored before execution.

## Best Possible Solution

Add generic output replacement support for tool-call argument data in the shared provider/CLI stream transform path, with regression coverage proving final executable `toolCall.arguments` are restored.

## Reproduction Assessment

Yes from source, though not from a live run: current `main` transforms ordinary assistant text output but not `toolCall.arguments`, and the agent loop executes those final arguments.

## Solution Assessment

Yes; the shared stream text-transform helper is the narrow owner-boundary fix because provider and CLI plugins already declare the replacement contract there.
