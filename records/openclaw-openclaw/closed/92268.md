## What This Changes

The issue asks for a supported OpenClaw way to allow outbound network access for Codex app-server workspace-write execution without using danger-full-access.

## Best Possible Solution

Keep OpenClaw sandbox egress plus sandbox exec-server as the supported policy path, and continue leaving raw Codex config.toml unable to override OpenClaw sandbox policy.

## Reproduction Assessment

Yes. The old raw native workspace-write policy still denies network by source inspection, but current main also shows the supported OpenClaw sandbox exec-server path supplies an egress-derived policy instead.

## Solution Assessment

Yes. The implemented approach is the maintainable fix because it keeps OpenClaw as the sandbox policy owner and mirrors explicit sandbox egress rather than letting raw Codex config bypass that boundary.
