## Summary

Current main already handles the reported MiMo regression: `mimo-v2.5-pro` is in the reasoning-content replay allowlist, the Xiaomi provider opts out of dropping reasoning history, and regression tests cover both Xiaomi and custom MiMo proxy replay. This is fixed on current main but not included in `v2026.5.12`.

## What This Changes

The issue asks for MiMo `mimo-v2.5-pro` OpenAI-completions tool-call replay to retain assistant `reasoning_content`, likely through a per-model compat option, instead of inheriting the default stripping used for other backends.

## Best Possible Solution

Keep the current model-aware replay preservation for known MiMo reasoning models and ship it in the next release; add a user-facing compat knob only if future unknown models need manual override.

## Reproduction Assessment

No: there is no current-main failing reproduction path from source inspection. The reported 5.12 release behavior is addressed on main by preserving or backfilling `reasoning_content` for the reported MiMo model family.

## Solution Assessment

Yes: the current main implementation is narrower than a broad global opt-out because it preserves valid `reasoning_content` only for known MiMo/DeepSeek-style replay contracts while keeping the default strip path for other OpenAI-compatible backends.
