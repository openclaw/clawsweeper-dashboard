## Summary

## What This Changes

The issue reports a Telegram topic async exec/heartbeat/context-overflow incident where internal HEARTBEAT_OK or followup text leaked to chat and the session became stuck.

## Best Possible Solution

Keep the shipped heartbeat/exec-event filtering, topic routing, stale-final replay, and compaction retry safeguards; any broader heartbeat preemption or stuck-session work should stay on the remaining canonical reliability issues.

## Reproduction Assessment

No current-main failure reproduction remains; the original 2026.4.26 incident is source-reproducible from the exec-event, heartbeat, pending-final, and compaction retry paths now covered by regression tests.

## Solution Assessment

Yes. The current solution is the maintainable boundary fix: classify relayable exec completions, suppress heartbeat acknowledgements before delivery, preserve Telegram topic routing, and resume compaction retries from transcript state instead of adding a Telegram-only text filter.
