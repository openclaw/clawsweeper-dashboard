## Summary

Current `main` already handles the missing-user-bus case without requiring `DBUS_SESSION_BUS_ADDRESS` in the generated unit: systemd probes fall back to `systemctl --machine <user>@ --user`, install flows treat user-bus probe failures as non-fatal, and regression tests cover both `Failed to connect to bus` and `No medium found` paths.

## Evidence

- **systemd user-bus fallback exists on main:** `execSystemctlUser` retries bus-unavailable `systemctl --user` calls through `--machine <user>@ --user`, which avoids relying on `DBUS_SESSION_BUS_ADDRESS` inside the service environment.
