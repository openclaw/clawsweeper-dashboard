## Summary

Current `main` already drains queued followups after reply-run completion and after escaped run-path errors, which covers the timeout/surface-error failure mode described in #67173. The issue timeline also points to closed fix PRs for this exact path. The fix appears present on `main` after the latest tagged release, so it is implemented but likely not yet shipped in `v2026.4.23`.
