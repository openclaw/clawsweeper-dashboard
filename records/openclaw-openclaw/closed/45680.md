## Summary

Current main already contains shipped fixes for the three bundled regressions in this issue: the CLI loopback handshake/startup failure, the Control UI device-identity/shared-auth regression, and the Anthropic alias initialization crash.

## Evidence

- **CLI regression was called out as fixed in a shipped release:** The changelog for v2026.3.13 says: "CLI/startup: stop `openclaw devices list` and similar loopback gateway commands from failing during startup by isolating heavy import-time side effects from the normal CLI path."
  - file: [CHANGELOG.md:2564](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/CHANGELOG.md#L2564)
- **Current main still contains explicit CLI handshake hardening with regression coverage:** `src/gateway/call.ts` yields one event-loop turn before starting the websocket connect so `connect.challenge` is not starved into a handshake timeout, and `src/gateway/call.test.ts` asserts that yield happens before CLI pairing requests start.
  - file: [src/gateway/call.ts:479](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/gateway/call.ts#L479)
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Control UI auth/device-identity regressions were shipped as fixes:** The v2026.3.13 changelog includes both "preserve explicit shared token and password auth on plain-HTTP Control UI connects" and "restore the operator-only device-auth bypass and classify browser connect failures".
  - file: [CHANGELOG.md:2538](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/CHANGELOG.md#L2538)
- **Current main Control UI code covers the reported webchat/device-identity path:** `ui/src/ui/gateway.ts` now keeps explicit shared auth for insecure contexts, retries a bounded reconnect on token mismatch, and closes with structured connect errors; node tests cover retry/reconnect after the first rejected connect and app tests map generic fetch failures to actionable device-identity guidance.
  - file: [ui/src/ui/gateway.ts:403](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/ui/src/ui/gateway.ts#L403)
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Anthropic alias init crash was already fixed and is reflected in current code/tests:** The v2026.3.12 changelog says Anthropic alias normalization was inlined during config load (#45520). Current main has direct `normalizeAnthropicModelId()` normalization in `src/agents/model-ref-shared.ts`, and `src/config/config.pruning-defaults.test.ts` covers dated Anthropic refs like `anthropic/claude-sonnet-4-20250514` without the old init-order crash path.
  - file: [CHANGELOG.md:2664](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/CHANGELOG.md#L2664)
