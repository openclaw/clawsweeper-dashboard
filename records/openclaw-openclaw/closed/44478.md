## Summary

Current main already implements the shared-session and origin-aware routing behavior this issue asked for: main-scoped direct DMs collapse to the agent main session, Telegram uses that resolver, WebChat/TUI read and stream by session key, and reply routing avoids cross-surface duplication while preserving origin delivery. The implementation is present in the v2026.4.24 release tag.
