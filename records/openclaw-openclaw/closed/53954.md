## Summary

Current main already treats the manifest id as authoritative for npm-installed plugins and has regression coverage for the exact class of mismatch described here. The same behavior is present in the shipped `v2026.4.22` release.

## Evidence

- **Install path prefers manifest id over npm package name:** `installPluginFromPackageDir` reads `openclaw.plugin.json`, sets `pluginId = manifestPluginId ?? npmPluginId`, and only logs an informational note when the npm package name differs. That matches the requested behavior and avoids deriving the config key from the package basename.
