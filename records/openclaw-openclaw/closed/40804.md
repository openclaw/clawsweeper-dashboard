## Summary

PR #40804 is an older implementation of the backup exclude-pattern feature. Current main still does not ship generic `--exclude` / `--exclude-file` support, but the same remaining work is now centralized in open issue #40786 and successor PR #44288. PR #40804 targets the older inline `src/commands/backup.ts` implementation and still has matcher/negation/workspace-ignore defects at its final head, while #44288 targets the current `src/infra/backup-create.ts` architecture with a broader, safer implementation.

## Best Possible Solution

Close #40804 as superseded and keep the remaining product work consolidated in #40786 and successor PR #44288. The eventual implementation should target `src/infra/backup-create.ts`, use a dependency-backed gitignore matcher, preserve restorable paths by default, and include archive-level tests rather than reviving the older inline `src/commands/backup.ts` patch.
