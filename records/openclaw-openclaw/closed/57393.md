## Summary

Current `main` already provides the requested pre-model-selection plugin seam under a different public hook name: `before_model_resolve`. It is part of the SDK hook surface, returns per-turn `modelOverride`/`providerOverride`, is wired into agent setup before model resolution, is documented as the preferred model-switching hook, and has regression tests. The exact proposed `before_agent_turn` name and an explicit `thinkingLevel` override field were not added, but the reported missing hook for model routing is implemented on `main`.

## Risks / Open Questions

- The exact API shape differs from the issue proposal: `before_model_resolve` exists instead of `before_agent_turn`/`model_select`.
- The current hook covers provider/model override; an explicit `thinkingLevel` override field was not found in this seam.
- The implementation appears to be newer than release `v2026.4.23`, so users on the latest release may not have it until the next ship.
