## Summary

Close as ClawHub/plugin scope. This is optional session-greeting personalization, and current main already exposes hook and plugin surfaces that can replace the bare `/new` acknowledgement without adding another core config/message template; broader core reset-prompt configurability is separately tracked at https://github.com/openclaw/openclaw/issues/45501.

## What This Changes

The issue requests adding the agent name and current model to the generic `New session started` acknowledgement, optionally via a new per-agent `sessionGreeting` template.

## Best Possible Solution

Keep the core acknowledgement generic, let custom templated greetings ship as a ClawHub code plugin, and use https://github.com/openclaw/openclaw/issues/45501 for any core reset-prompt config decision.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing contract. Source inspection confirms the current generic acknowledgement and the existing hook path that can replace it.

## Solution Assessment

Yes. A ClawHub code plugin using `command:new` and `command:reset` hooks is the narrowest maintainable home for optional personalization; adding a new core template config would overlap an existing reset-prompt product discussion.
