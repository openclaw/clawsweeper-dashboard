## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use raw capture length for strict recovery:** `src/shared/text/reasoning-tags.ts:91`
  - body: The strict fallback checks `!result.trim()`, so an input such as `"\n<think>secret"` treats the whitespace prefix as no captured output and exposes the unclosed reasoning tail. Strict callers use this path as a safety boundary; key the recovery on raw captured length instead.
  - confidence: 0.92
- **[P2] Emit sessions.changed after reset:** `src/gateway/server-methods/sessions.ts:978`
  - body: `sessions.reset` now discards `context` and returns success without emitting `sessions.changed`. Other subscribed clients rely on that event to refresh the new session ID and metadata after a reset, so they can remain stale until another mutation occurs.
  - confidence: 0.87
- **[P2] Bound awaited transcribed hooks:** `src/auto-reply/reply/message-preprocess-hooks.ts:31-38`
  - body: `message:transcribed` is awaited before the agent reply starts, but this await has no timeout. A slow or stuck hook handler can now block the inbound reply path indefinitely for that message.
  - confidence: 0.84
- **[P1] Use execFile for memory extraction:** `src/hooks/bundled/session-memory/handler.ts:186-188`
  - body: The new memory extraction path passes a shell string containing `extractScript` and `sessionId` to `execAsync`. If either value contains shell metacharacters, the shell can run more than the intended Python command; use `execFile("python3", [extractScript, "--session", sessionId], ...)`.
  - confidence: 0.86

## Security Review
