## Summary

Close #71974 as a duplicate of open canonical issue #42538. Current main still has the same WhatsApp health/runtime mismatch: health snapshots are built without gateway runtime state, while channels.status has the runtime snapshot path, and WhatsApp defaults missing running/connected fields to false. Open PR #42586 already targets #42538 with the expected fix direction.

## Best Possible Solution

Keep the remaining product bug tracked in #42538 and resolve it through #42586 or an equivalent maintainer-reviewed change that passes gateway runtime snapshots into health snapshot generation, then updates or adds regression coverage for WhatsApp running/connected parity.
