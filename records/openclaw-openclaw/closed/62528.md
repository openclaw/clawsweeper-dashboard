## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve mapped hook behavior before reserving message:** `src/gateway/server-http.ts:674-675`
  - body: Current docs/config support arbitrary `hooks.mappings[].match.path` subpaths, but this branch handles `subPath === "message"` before `applyHookMappings`. Any existing `/hooks/message` mapping will stop executing and be validated as the new `{ message, requestId }` contract instead, so preserve mapped-hook precedence or add a tested migration path before reserving this subpath.
  - confidence: 0.9
- **[P2] Preserve retryable session creation errors:** `src/gateway/server/hooks.ts:388`
  - body: When `sessions.create` fails, `createResult.error` may carry a structured `UNAVAILABLE` code, but this branch throws a plain `Error`. The HTTP layer then returns a generic 500 instead of the 503 path it already handles, which gives webhook senders the wrong retry semantics.
  - confidence: 0.84
- **[P3] Update the shared sessionKey policy message:** `src/gateway/server-http.ts:681-684`
  - body: The new message branch calls `resolveHookSessionKey` for `/hooks/message`, but the shared policy error still names `/hooks/agent`. Callers using `/hooks/message` with request session keys disabled will get the wrong endpoint in the error text.
  - confidence: 0.82
