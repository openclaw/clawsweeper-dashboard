## Summary

Current main already restores the compatible Telegram text path by keeping Bot API rich messages behind explicit `richMessages === true`, which addresses the v2026.6.8 iOS font-size regression reported here; the fix is current-main-only and not in `v2026.6.8`.

## What This Changes

The issue reports a Telegram iOS regression where bot replies sent through v2026.6.8 rich-message delivery ignore the user's app font-size setting.

## Best Possible Solution

Keep standard Telegram text delivery as the default and leave Bot API 10.1 rich messages as an explicit opt-in for deployments whose clients support them.

## Reproduction Assessment

Yes, source-level: `v2026.6.8` sends Telegram text through rich-message delivery by default, while current main only does so when `richMessages === true`. I did not run a live Telegram iOS font-size reproduction.

## Solution Assessment

Yes. Restoring standard text delivery as the default is the narrowest maintainable fix for users who never opted into Bot API 10.1 rich rendering, while preserving explicit rich-message support.
