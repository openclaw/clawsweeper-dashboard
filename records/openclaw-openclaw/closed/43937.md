## Summary

Current `main` already contains a cron timer re-arm fix for the exact silent-stall failure mode described here: when a timer tick fires while the scheduler is still marked `running`, `onTimer()` now arms a 60s watchdog instead of returning with no timer, and a regression test documents that this previously 'silently kill[ed] the scheduler' until restart.

## Evidence

- **Scheduler now re-arms when a tick arrives during an active run:** `onTimer()` checks `state.running` and immediately calls `armRunningRecheckTimer(state)` before returning. The inline comment says the old behavior left no timer set and 'silently kill[ed] the scheduler until the next gateway restart.'
