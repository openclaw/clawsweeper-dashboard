## Summary

This should close as external-plugin-owned rather than an OpenClaw core bug. Current main intentionally keeps `sessionKey` as the stable conversation bucket, rotates `sessionId` on `/new`, and passes both identifiers plus lifecycle/reset hooks to context-engine plugins; the reported stale-summary behavior matches lossless-claw's own current `/new` retention semantics rather than a missing core reset boundary. ([github.com](https://github.com/martian-engineering/lossless-claw?utm_source=openai))

## What This Changes

The issue reports that a Discord channel `/new` session using the external `lossless-claw` context engine can receive retained summaries from the previous logical channel conversation.

## Best Possible Solution

Leave the OpenClaw core session contract unchanged and handle any change to `/new` retention, default `newSessionRetainDepth`, or stale-summary filtering in the lossless-claw plugin or its configuration/docs.

## Reproduction Assessment

Yes. The report gives concrete steps and transcript excerpts, and the current lossless-claw documentation describes `/new` as retaining summary structure while `/reset` creates a clean LCM conversation; I did not run a live Discord/LCM deployment in this read-only review.

## Solution Assessment

No core fix is indicated. The narrow maintainable solution is a lossless-claw-side behavior or configuration change because OpenClaw already rotates `sessionId` and exposes reset lifecycle hooks to plugins.
