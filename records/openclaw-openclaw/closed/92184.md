## Label Justifications

- `P1`: The originally reported shipped behavior silently lost concurrent group messages in a real channel workflow.

## Summary

The reported 15-second admission drop no longer exists: both current main and the latest release let visible inbound turns wait for the active reply run, reserving the short settle timeout for queued followups. The proposed timeout configuration is unnecessary because the central message-loss bug is already resolved.

## What This Changes

Prevent concurrent group messages from being silently discarded when a reply run remains active beyond 15 seconds.

## Best Possible Solution

Keep the shipped unbounded wait for visible inbound turns and its focused regression coverage; track the distinct foreground-fence delivery loss separately in https://github.com/openclaw/openclaw/issues/92186.

## Reproduction Assessment

No on current main or v2026.6.5: visible admission now waits without the former 15-second timeout, although the report remains source-credible for OpenClaw 2026.5.26.

## Solution Assessment

Yes—the shipped canonical fix removes the lossy visible-turn timeout instead of adding a new configuration knob, which is narrower and consistent with repository configuration policy.
