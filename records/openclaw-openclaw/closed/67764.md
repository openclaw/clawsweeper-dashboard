## Summary

Current main already solves the central operator need with trajectory capture and `/export-trajectory`: it records per-run compiled context, submitted prompt, system prompt, active history/messages, tools, model/runtime metadata, outputs/artifacts, and packages them into a local debug bundle. It is not a Sessions-tab drilldown, but the issue framed that UI as ideal rather than mandatory, and the shipped trajectory surface covers the requested local/debug-first inspection workflow well enough.

## Best Possible Solution

Keep the shipped trajectory capture/export path as the implemented prompt/context inspector. If maintainers want a richer UI later, track a narrower follow-up to render existing trajectory sidecars from the Sessions tab rather than keeping this broad feature request open.
