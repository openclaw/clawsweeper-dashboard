## Summary

Current main, and the v2026.4.24 release, already address the reported duplicate startup-context path. The generated workspace AGENTS.md now tells agents to use runtime-provided startup context first and not manually reread AGENTS.md/SOUL.md/USER.md/MEMORY.md unless needed, while the runtime startup prelude explicitly says those bootstrap files are already provided separately and only adds recent daily memory for the first /new or /reset turn.
