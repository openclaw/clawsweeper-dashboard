## Summary

Keep open. Current `main` still has the reported Slack monitor gap: Slack accepts and documents `dmHistoryLimit`, but the Slack monitor derives and carries only `historyLimit`, while prompt-history recording, injection, structured `InboundHistory`, and cleanup remain tied to the roomish path that excludes 1:1 DMs. The closed unmerged #67054 is useful prior work, and open #64427 overlaps on Slack DM history but is broader cross-session hydration rather than a clean supersession of this narrower monitor-wiring bug.

## What This Changes

The issue requests wiring Slack `dmHistoryLimit` into the 1:1 DM prompt-history path so non-threaded DM replies can include prior DM context.

## Best Possible Solution

Land a focused Slack plugin fix that derives an effective DM history limit from Slack account config, carries or computes it in the monitor path, records pending 1:1 DM history when enabled, uses the effective room-vs-DM limit for prompt body history, structured `InboundHistory`, and dispatch cleanup, and adds regression coverage plus a user-facing changelog entry. Leave cross-session Slack API history hydration and cross-channel parity as follow-up scope.
