## Summary

Close as implemented: current main and the shipped v2026.5.3 tag preserve commentary suppression while preventing silent no-final turns and routing Telegram topic finals through observed delivery/fallback paths.

## What This Changes

The issue reports that Telegram-backed agent turns can persist assistant text but deliver nothing when final-only or topic final-delivery paths suppress the output.

## Best Possible Solution

Keep the shipped behavior: suppress commentary leaks, surface terminal no-visible-output diagnostics, and require observed Telegram send/edit/fallback before treating topic finals as delivered.

## Reproduction Assessment

Yes for the reported 2026.5.2 failure shape: the transcript evidence plus current source make the old commentary-only and topic-final suppression paths clear. I did not rerun a live Telegram turn because the current shipped source now routes those cases to diagnostics or observed final delivery.

## Solution Assessment

Yes. The implemented path is safer than auto-promoting arbitrary commentary because it preserves final-only/commentary boundaries while ensuring a visible diagnostic or standard final send when delivery would otherwise be silent.
