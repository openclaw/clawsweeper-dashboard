## Summary

Current main already removes the session-store rotation path that matches the reported v2026.4.26 log spam. The user-linked logs show repeated `sessions/store` `sessions.json.bak.*` backup/cleanup messages around a 15 MB store, and current main commit 490e6d6 removes `rotateSessionFile`, ignores legacy `session.maintenance.rotateBytes`, documents that behavior, and adds regression tests that hot oversized writes no longer create rotation backups. The fix is on main only; no release tag contains it yet.

## Best Possible Solution

Close this as implemented on current main, keep the 490e6d6 session-store maintenance behavior, and ship it in the next release. If a build containing that commit still hangs without `sessions.json.bak.*` spam, open a new focused bug with fresh logs and the remaining gateway timeout/provider/channel symptoms.
