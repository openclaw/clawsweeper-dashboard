## Summary

Keep PR #69910 open. Current main still leaves the native Ollama inbound `buildAssistantMessage()` path unnormalized: `response.message.tool_calls[].function.arguments` is copied directly into the assistant tool call. Adjacent replay/OpenAI-compatible paths already use the unsafe-integer-preserving helper, and related PR #72102 was closed as a duplicate of this earlier canonical PR, so #69910 remains the best open implementation path for #69735.

## Best Possible Solution

Keep #69910 open or land an equivalent narrow patch on current main: normalize `toolCall.function.arguments` in `buildAssistantMessage()` with the existing `ensureArgsObject()`/unsafe-integer-preserving parser, keep object pass-through, fall back safely for malformed or non-object strings, widen the local Ollama response type to reflect runtime inputs, and retain focused regression coverage for JSON strings, unsafe integers, malformed strings, and object arguments.
