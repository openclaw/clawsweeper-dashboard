## Summary

Current main implements the requested guard for #71912. QQBot outbound sends are recorded as bot-authored refs, inbound QQBot events whose current msgIdx matches that bot ref are blocked before agent dispatch, user quotes of bot messages are still allowed, and transcript-only delivery-mirror rows are filtered from model replay/embedded re-delivery. The linked PR #72032 is relevant, but its antml thinking-tag concern is already present on main and the direct #71912 fix is the QQBot self-echo suppression recorded in the changelog.

## Best Possible Solution

Close #71912 as implemented on main. Keep the QQBot self-echo suppression and transcript-only replay filters as the core fix; if contributors want additional generic throttling or rate-limit behavior, that should be a narrower follow-up rather than keeping this resolved QQBot feedback-loop bug open. The linked PR #72032 can be handled separately as redundant or rebased against the already-present main behavior.
