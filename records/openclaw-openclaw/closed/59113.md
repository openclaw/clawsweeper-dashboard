## Summary

Current `main` already addresses this Telegram retry/duplication report with two layers: inbound Telegram update dedupe in the bot pipeline and a 5-second webhook callback timeout that returns `200` before long-running work triggers Telegram retries. The shipped `v2026.4.23` release already contains the webhook timeout change and the corresponding Telegram dedupe/test coverage is present on the tagged release as well.

## Evidence
