## Summary

PR #57885 is a service-specific Graphiti/Cognee memory gateway proxy with hard-coded local services, agent/dataset ACLs, audit logging, and anomaly rules. Current project scope and plugin APIs make this a better fit for an external ClawHub/npm memory plugin than a one-off core gateway feature, and the PR review thread already identified unresolved security/integrity problems in the proposed core implementation.

## Best Possible Solution

Close this PR and continue the Graphiti/Cognee memory proxy as an external ClawHub/npm plugin using `api.registerHttpRoute`, memory SDK helpers, routing helpers, and plugin-owned audit/ACL config. If implementation proves a specific missing core extension point, open a narrow plugin API issue or PR for that seam only.

## Risks / Open Questions

- This does not reject Graphiti/Cognee memory integration as an idea; it rejects this large, service-specific core gateway implementation as the wrong home under the current project scope.
- If an external plugin cannot safely derive the trusted session/agent context it needs, the right follow-up is a focused plugin API proposal rather than a hard-coded core proxy.
