## Summary

Current main already suppresses WhatsApp reasoning/thinking payloads before delivery, and the same suppression logic is present in the shipped v2026.4.23 release.

## Evidence

## Risks / Open Questions

- If a provider/plugin were to emit hidden reasoning as ordinary visible text without `isReasoning: true` and without the `Reasoning:` prefix, that would bypass the current guards; I did not find evidence of that contract in the current WhatsApp path.
