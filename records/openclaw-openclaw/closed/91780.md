I found no blocking correctness or security findings in this read-only review.

## What This Changes

The branch exports the existing Control UI idle queue reconciliation helper, invokes it after session switches, and adds a regression test for restored queued messages after switching back.

## Best Possible Solution

Land the focused UI fix after normal CI so session switching uses the same restored-queue idle reconciliation as chat refresh.

## Reproduction Assessment

Yes, at source level: current main restores queued messages on session switch but does not call the idle reconciliation helper from the switch path, so the restored queue can remain stuck after off-screen completion until another refresh. I did not execute tests in this read-only sweep.

## Solution Assessment

Yes, this is the narrow maintainable fix: it reuses the existing app-chat idle proof and drain helper instead of duplicating queue-drain rules in app-render.helpers.ts.
