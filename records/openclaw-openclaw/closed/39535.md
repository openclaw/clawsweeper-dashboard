## Summary

Close as `cannot_reproduce`. Current `main` still models plugin-owned settings under `plugins.entries.<pluginId>.config.*`, not directly on `plugins.entries.<pluginId>`, and gateway validation explicitly validates `entry.config` against each plugin's `configSchema`. The original report path is not the supported config shape.

## Evidence

- **Gateway patch API only accepts full patch payload fields, not a `path` parameter:** `ConfigPatchParamsSchema` allows `raw`, `baseHash`, `sessionKey`, `deliveryContext`, `note`, and `restartDelayMs`, with `additionalProperties: false`. There is no `path` field on the RPC.
  - file: [src/gateway/protocol/schema/config.ts:30](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/gateway/protocol/schema/config.ts#L30)

- **Gateway `config.patch` validates the merged config through plugin-aware validation:** `config.patch` parses the patch object, merge-patches it onto the current config, then calls `validateConfigObjectWithPlugins(restoredMerge.result)` before accepting it.
  - file: [src/gateway/server-methods/config.ts:471](https://github.com/openclaw/openclaw/blob/cb2c36b04975e34ab7014a11575d6589aa96ae32/src/gateway/server-methods/config.ts#L471)
