## Summary

The reported long-running 2026.5.27 process was missing restart hardening that is now present in current main and shipped in v2026.6.8; the v2026.6.8 tag contains the eager lifecycle-runtime SIGUSR1 handling and token cleanup that prevents silent permanent restart coalescing after an in-place package update.

## What This Changes

The issue reports that internal gateway restart requests can hang silently after long macOS LaunchAgent uptime, without sending SIGTERM or returning an error.

## Best Possible Solution

Keep the shipped v2026.6.8 restart hardening; if a fresh 2026.6.8-or-newer process still shows a no-SIGUSR1/no-shutdown restart hang, open a new report with logs from that running version.

## Reproduction Assessment

Yes for the source-level failure class: the prior issue logs map to the token guard and SIGUSR1 listener path, and the current shipped code now hardens that path. I did not live-reproduce a 20-day macOS LaunchAgent run in this read-only review.

## Solution Assessment

Yes. The narrow shipped fix keeps the lifecycle-runtime restart dependencies loaded before disk chunk rotation and clears failed SIGUSR1 handler state, which is the right owner boundary for the old-process-after-update internal restart hang.
