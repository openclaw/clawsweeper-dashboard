## Summary

Current main already provides browser network-request inspection and response-body capture on the supported browser control path, and the legacy Chrome extension relay named in the issue was removed in v2026.3.22.

## Close Comment

Closing from a Codex review because current `main` already covers the requested network-capture capability on the supported browser control path, and the specific Chrome extension relay named here no longer exists. Current code records request/response/failure events in `extensions/browser/src/browser/pw-session.ts:350`, exposes `GET /requests` in `extensions/browser/src/browser/routes/agent.debug.ts:69`, exposes `POST /response/body` in `extensions/browser/src/browser/routes/agent.act.ts:627`, and documents the corresponding `openclaw browser requests` / `openclaw browser responsebody` flows in `docs/tools/browser-control.md:25`. Separately, the old Chrome extension relay path was removed in `v2026.3.22` (`CHANGELOG.md:2191`; `extensions/browser/src/browser/server-lifecycle.ts:13`). If someone still wants equivalent capture specifically for the newer `existing-session` / Chrome MCP path, that should be tracked as a separate request against the current browser architecture.
