## Summary

Current main already solves the reported WhatsApp/Baileys log-noise problem: normal WhatsApp gateway sockets pass a pino-like Baileys logger at `silent` level, with `info` only when verbose/debug logging is enabled. The same behavior is present in the v2026.4.24 release, while the report was from 2026.4.5.

## Best Possible Solution

Close as implemented. Keep the shipped silent-by-default Baileys logger for normal WhatsApp gateway operation and use existing global verbose/debug logging when maintainers need Baileys diagnostics; a future narrower issue can request per-channel diagnostic verbosity if that becomes necessary.
