## Summary

Keep open: this PR addresses a real current-main gateway TLS SAN bug, but the latest head still lets the runtime replace explicit operator-provided cert/key files when a SAN-less certificate has `CN=openclaw-gateway`.

## What This Changes

The PR adds subjectAltName entries to auto-generated gateway TLS certificates and adds runtime regeneration for older CN-only generated certificates, with regression tests.

## Best Possible Solution

Ship SAN-bearing generated certs with runtime repair limited to OpenClaw-owned generated files, or move old-certificate repair into a guarded doctor migration.

## Solution Assessment

No, not yet. Adding SANs in the generator is the right layer, but the automatic upgrade path is too broad for explicit operator certificate paths, and the final SAN identity set still needs maintainer confirmation.
