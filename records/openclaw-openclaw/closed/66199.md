## Summary

Keep this PR open, but do not merge it as-is. The iMessage hardening goal is still relevant on current main, but the patch wraps only the legacy `formatInboundEnvelope(...)` body while the active prompt path still uses raw `BodyForAgent`; the existing review discussion correctly identifies this as a blocking security defect.

## What This Changes

The PR imports `wrapExternalContent` into iMessage inbound processing and wraps current-message and group-history strings passed as the `body` argument to `formatInboundEnvelope(...)`.

## Best Possible Solution

Revise this PR or replace it with a maintainer-reviewed iMessage hardening patch that protects the actual model-bound path, likely through `UntrustedContext` or another prompt-bound structured field, while preserving raw command fields and adding focused tests for direct messages, group history, and reply-context behavior.
