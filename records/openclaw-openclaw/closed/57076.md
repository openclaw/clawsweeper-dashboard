## Summary

Current `main` already implements the reported `agents.defaults.heartbeat.model` behavior through a different path: the heartbeat runner resolves merged heartbeat config, forwards `heartbeatModelOverride` into reply generation, and model selection now preserves that override instead of falling back to the session model. The PR's exact patch shape is not present, but the user-facing bug it targets is covered on `main` with tests.
