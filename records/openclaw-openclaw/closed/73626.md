## Summary

Keep open. Current main still has a plausible stale Discord binding path: Discord preflight resolves the current configured route, then lets a persisted current-conversation binding rewrite the effective route/session key and agent id from its stored targetSessionKey. I did not find a post-review fix, canonical superseding issue/PR, or freshness check that compares the stored binding against current Discord routing/default-account state.

## Best Possible Solution

Keep this issue open and fix it in the Discord ingress/current-conversation binding path. Add a route freshness check or route fingerprint for Discord current-conversation bindings, compare stored target agent/session against the current effective route when account/defaultAccount/agent binding config changes, and ignore, unbind, or clearly warn on stale normal Discord channel bindings while preserving intentional ACP and plugin-owned bindings. Add focused regression coverage for a stale agent:<old>:discord:channel:<id> binding when current routing resolves to a different agent/account.
