## Summary

Current `main` no longer matches the reported failure mode: config reads now auto-apply runtime legacy migrations, configured bundled channels are preserved in gateway/read-only startup planning, and Discord startup now fails loudly on missing credentials instead of silently skipping initialization. The changelog also records follow-up fixes shipped after `v2026.3.28` for bundled startup selection and `3.28` schema compatibility.

## Evidence
