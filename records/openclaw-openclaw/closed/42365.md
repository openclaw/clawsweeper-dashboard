## Summary

Current `main` already handles `node.event` and supports the issue’s agent-routed alternative for proactive node-to-channel delivery. Nodes can send `event: "agent.request"` with `deliver: true`, optional attachments, and optional explicit/implicit routing, and the gateway forwards that into the normal agent/channel delivery path.

## Risks / Open Questions

- There is no special direct `event: "alert"` channel fan-out path on `main`; the implemented solution is the issue’s own agent-routed alternative via `event: "agent.request"`, which is sufficient for proactive alerts and attachments.

## Close Comment

Codex review: this looks implemented on current `main` through the agent-routed path. The gateway already handles `node.event` in `src/gateway/server-methods/nodes.ts:1102`, and `src/gateway/server-node-events.ts:380` supports `event: "agent.request"` with `deliver`, `channel`, `to`, attachment parsing, and fallback to the session’s last delivery route before handing off to the normal agent/channel pipeline. Because the node-to-channel proactive delivery path already exists on `main`, I’d close this as implemented.
