## Summary

Close as implemented. Current main and v2026.4.26 solve the reported low-resource BM25-only QMD case: lexical `searchMode: "search"` now skips QMD vector probes and `qmd embed` maintenance across forced, first-run, and periodic paths, with docs and focused tests covering that behavior.

## What This Changes

The issue asks for QMD embed scheduling to respect disable/large interval settings, or to provide and document a way for BM25-only QMD deployments to prevent `qmd embed` from spawning.

## Best Possible Solution

Keep the shipped BM25-only contract: `searchMode: "search"` should remain the documented way to use QMD without vector probes or `qmd embed`, while `query` and `vsearch` can continue requiring embeddings unless maintainers choose a separate product-level opt-out later.
