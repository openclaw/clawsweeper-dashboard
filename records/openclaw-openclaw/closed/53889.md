## Summary

Current main now fixes the session-deadlock path reported here by bounding the post-run pending tool-result drain; a never-settling delivery task can no longer keep the reply operation active forever. The fix is on main only at this point, not in the latest published v2026.4.26 release.

## What This Changes

The issue asks OpenClaw to recover from dangling tool calls or lost tool results so a session cannot remain permanently stuck in the agent loop.

## Best Possible Solution

Keep the focused main-branch bounded-drain fix for this issue, and track any broader lane-watchdog or all-provider tool-timeout hardening separately from this already-fixed deadlock path.
