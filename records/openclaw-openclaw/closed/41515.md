## Summary

Current `main` already contains shipped fixes for the reported false embedded-runner rate-limit path: same-provider cooldown probing is capped to one transient probe per provider per fallback run, and cooldown windows without recorded failure history no longer default to `rate_limit`.

## Evidence

- **Main fallback logic caps duplicate same-provider cooldown probes:** `runWithModelFallback` now skips additional transient cooldown probes for the same provider within one fallback run, preventing repeated same-provider stalls before cross-provider fallback.
