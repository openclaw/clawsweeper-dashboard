## Summary

Keep open: the branch addresses the missing `agent_model_catalogs` producer for the linked slow `models list` bug, but one provider-catalog cache path can persist degraded fallback rows after a transient live-catalog failure.

## What This Changes

Adds a SQLite-backed persisted cache for agent model catalog and `models list` provider-catalog results, with fingerprinted cache keys and focused tests.

## Best Possible Solution

Keep the SQLite cache, but make provider-catalog writes conditional on full successful discovery while preserving fingerprint invalidation for config, auth, environment, and plugin metadata changes.

## Reproduction Assessment

Yes, source-level. Current main defines `agent_model_catalogs` but has no producer path, and the linked issue gives concrete 2026.6.1 commands showing repeated `models list` calls leave the table empty and slow.

## Solution Assessment

No, not quite. The SQLite-backed cache is the right product boundary for the reported issue, but the provider-catalog path should not persist rows produced only after a transient live-catalog failure.
