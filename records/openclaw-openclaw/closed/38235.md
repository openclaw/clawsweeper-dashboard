## Summary

Keep this open: current main still resolves Slack agent identity but drops it before the draft preview send, and the native streaming helper also has no identity input. An open PR already references this issue with closing syntax, but it is stale against current plugin paths, so the issue should wait for that PR to be rebased/replaced or closed.

## What This Changes

The issue asks Slack streamed replies to preserve configured agent identity instead of falling back to the Slack app identity.

## Best Possible Solution

Land or replace a Slack plugin fix that threads SlackSendIdentity through createSlackDraftStream, adds focused regression coverage, and separately verifies the native streaming identity contract before closing the full report.

## Reproduction Assessment

Yes, at source level. Current main resolves slackIdentity and normal sends use it, while the draft stream constructor and initial send have no identity field; I did not run a live Slack workspace test.

## Solution Assessment

Yes for the draft path: passing the existing SlackSendIdentity through the Slack plugin draft stream is the narrow maintainable fix. Native streaming should be fixed or explicitly scoped out only after checking the current Slack API/SDK contract.
