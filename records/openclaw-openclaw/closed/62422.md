## Summary

Current main and the shipped v2026.4.24 release already cover the reported macOS-to-local-Docker token-auth path: the app has a remote/direct gateway token field, persists it to `gateway.remote.token`, resolves that token for remote-mode WebSocket connections, sends it as `auth.token` during the gateway handshake, and surfaces structured token-mismatch guidance instead of leaving only `Schema unavailable`. The later linked #64973 is a separate Talk Mode/config-write race report and should remain the tracker for that narrower failure mode.

## Best Possible Solution

Close this specific Docker/local-gateway schema-unavailable report as shipped. Keep the current macOS Remote direct credential flow (`gateway.remote.url` + `gateway.remote.token`) and leave the later #64973 open for the distinct Talk Mode stale `config.set`/gateway restart/token-mismatch sequence.
