## Summary

Current main does not support the reported causal path. Docker sandbox containers are intentionally persistent per sandbox scope and pruned on idle/age, while subagent concurrency limits are enforced by the in-process subagent lane and active-run registry rather than by counting running Docker containers.

## Evidence

- **Sandbox containers are persistent by scope:** Docs define sandbox scope as controlling how many containers exist (`agent`, `session`, `shared`), and `ensureSandboxContainer()` reuses an existing named container instead of removing it after each run.
  - file: [src/agents/sandbox/docker.ts:524](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/agents/sandbox/docker.ts#L524)

  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)
- **Persistence is documented, not accidental:** Sandbox CLI docs explicitly say existing runtimes keep running and are normally pruned only after inactivity; defaults shown are `idleHours: 24` and `maxAgeDays: 7`.
  - file: [docs/cli/sandbox.md:151](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/docs/cli/sandbox.md#L151)

- **Prune logic removes old runtimes by idle/age:** `maybePruneSandboxes()` checks `idleHours`/`maxAgeDays` and removes registered runtimes only when those thresholds are exceeded.
  - file: [src/agents/sandbox/prune.ts:24](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/agents/sandbox/prune.ts#L24)

  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)
- **Subagent maxConcurrent is a queue lane limit, not a container-slot limit:** Docs define `agents.defaults.subagents.maxConcurrent` as the dedicated in-process `subagent` queue lane concurrency, and the queue implementation stores only active task ids per lane.
  - file: [src/process/command-queue.ts:235](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/process/command-queue.ts#L235)
