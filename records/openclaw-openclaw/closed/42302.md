## Summary

Current `main` already supports the requested "one agent" cross-channel behavior: multiple channels can bind to the same agent, direct chats can share that agent’s main session, WebChat uses the same session/routing model, and there is regression coverage for preserving shared main-session routing across WebChat and external channels.

## Risks / Open Questions

- Some installs now opt into more isolated DM scopes during onboarding, so the capability exists on main but may require `session.dmScope: "main"` plus routing those channels to the same agent to match the exact requested behavior.

## Close Comment

Closing this as implemented on current OpenClaw. This Codex review checked current `main` and the latest release tag, and OpenClaw already supports the "one brain" setup by routing multiple channels to the same agent and letting direct chats share that agent’s main session. The core behavior is in `src/config/sessions/session-key.ts:25` and `src/routing/session-key.ts:142`, the docs describe it in `docs/channels/channel-routing.md:24`, `docs/web/webchat.md:12`, and `docs/cli/agents.md:33`, and there is regression coverage for WebChat sharing/preserving the external main-session route in `src/auto-reply/reply/session.test.ts:3355`. If you still want this behavior on an install that was onboarded with isolated DM scope, the supported path is binding those channels to the same agent and using `session.dmScope: "main"`.
