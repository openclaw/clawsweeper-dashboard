## What This Changes

The issue asks OpenClaw to prevent subagent announce-delivery transcript entries from entering Anthropic replay with provider/model metadata that can poison thinking-block turns after a gateway restart.

## Best Possible Solution

Add a focused replay-safety fix that preserves current delivery routing while ensuring announce-delivery artifacts cannot enter Anthropic replay as a separate provider-owned assistant turn after restart.

## Reproduction Assessment

No live high-confidence reproduction was performed in this read-only review. Source inspection supports the reported chain through dormant announce delivery, provider-owned assistant replay, Anthropic turn validation, and request-local thinking recovery.

## Solution Assessment

Unclear that retagging all direct announce output is the best fix, because the dormant requester-agent handoff is an existing behavior. The safer repair is to add a regression test for the restart/dormant completion case and then make the replay path resilient without changing delivery semantics broadly.
