## Summary

Current `main` already fixes the sticky auto-fallback override problem through broader model-selection logic: direct `modelOverrideSource: "auto"` session overrides are cleared before each turn, persisted back to the session store, and the request is forced back to the configured primary model. That makes auth-triggered fallback self-heal on the next request without needing this PR's `previousFailureReasons` plumbing.

## Evidence
