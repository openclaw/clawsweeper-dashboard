## Summary

The resolver inconsistency is real and the PR now updates both core and memory-host consumers, but it replaces a shipped persistent workspace layout without migrating existing agent files. The PR should remain open pending an explicit canonical-layout decision, upgrade-safe migration, and refreshed proof of the final branch.

## What This Changes

The PR changes non-default agent workspace fallback paths from nested directories to hyphenated siblings in core and memory-host resolution, updates related tests, and adds a standalone reproduction script.

## Best Possible Solution

Choose the canonical layout explicitly; if hyphenation wins, add a focused `openclaw doctor --fix` migration that detects, safely moves or reconciles, verifies, and then retires shipped nested workspaces before runtime uses only the new path.

## Reproduction Assessment

Yes. Current main deterministically resolves this configuration to `<defaults.workspace>/<agentId>`, while the PR resolves it to `<defaults.workspace>-<agentId>`; source inspection also identifies the persistent consumers affected by that change.

## Solution Assessment

No. Updating both resolvers is necessary for internal consistency, but replacing a shipped persistent path without a doctor migration and upgrade proof is not the safest complete solution.
