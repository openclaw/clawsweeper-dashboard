## Summary

WeChat group chat support is a valid channel-plugin feature request, but the current OpenClaw repository does not own the WeChat runtime. Current main documents `openclaw-weixin` as an external `@tencent-weixin/openclaw-weixin` channel plugin, explicitly notes that group chats are not advertised by that plugin, and keeps WeChat iLink/API/message handling outside core. The generic channel plugin contract already supports group chat capability metadata, group mention policy, session routing, and outbound delivery hooks, so the best path is implementation in the external WeChat plugin/ClawHub package, with a narrower OpenClaw SDK issue only if that plugin hits a missing core seam.

## Best Possible Solution

Keep OpenClaw core channel-agnostic and leave this feature to the external `@tencent-weixin/openclaw-weixin` / ClawHub plugin. The plugin should add WeChat group capability metadata, group @ parsing, group allowlist/invite policy, and group outbound delivery there; if implementation exposes a concrete missing OpenClaw SDK seam, open a narrow core SDK issue or PR for that seam.
