## Summary

Current main already supports this as an opt-in configuration path: depth-1 subagents can be given `sessions_send`, and same-agent session visibility can include the parent session, which makes parent-directed progress sends possible without new core code.

## Evidence

## Risks / Open Questions

- There is no single end-to-end test in this checkout that sends an actual `sessions_send` from a depth-1 subagent to its parent; the conclusion comes from composing the tested policy and visibility paths.
- This is not the default setup: sandbox clamps or leaving `tools.sessions.visibility` at `tree` will still block parent targeting.
