## Summary

Keep open. Current main still lacks catalog metadata in the shared command-choice context, so Ollama models whose reasoning support comes from catalog metadata can still collapse to `off` in native `/think` menus. Also, open PR #73568 explicitly says it fixes this issue and is still unmerged in the provided GitHub context, so the issue should stay open until that PR or a broader replacement lands.

## Best Possible Solution

Keep the issue open and review/land or supersede PR #73568 with a shared fix: add catalog-aware data to the command argument choice/menu context, pass the runtime model catalog from native menu/autocomplete surfaces, and add regression coverage for an Ollama catalog entry with `reasoning: true`. If Ollama Cloud still lacks reliable runtime capability metadata, consider a provider-owned fallback that respects configured catalog `reasoning` without hardcoding Ollama behavior in core.
