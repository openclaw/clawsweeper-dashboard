## Summary

## What This Changes

The issue reports Telegram Bot API fetch timeouts for getMe, setMyCommands, and sendChatAction during multi-account gateway startup on v2026.4.29 despite healthy host connectivity.

## Best Possible Solution

Keep the main-branch fixes and ship them in the next release; persistent failures after that should be reported with fresh current-version logs.

## Reproduction Assessment

Yes for the shipped behavior: v2026.4.29 source has the reported short startup/control request budgets, and the issue provides six-account timeout logs. I did not run live Telegram, but current-main tests directly verify the replacement timeout, retry, and startup-fanout behavior.

## Solution Assessment

Yes. The current fix is narrowly placed in the Telegram plugin timeout/fallback paths plus a generic channel startup fanout cap, avoiding risky duplicate-prone sendMessage retries or Telegram-specific core policy.
