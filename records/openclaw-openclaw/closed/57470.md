## Summary

Close as superseded. Current main intentionally rejects an explicit cron payload.model when it is not allowed, fixing the silent fallback problem through fail-closed validation rather than bypassing the allowlist. The source, tests, docs, and changelog now all codify that behavior, so this PR would undo the current policy.

## Best Possible Solution

Close this PR as superseded by the current fail-closed cron model validation path. Keep the shipped source/tests/docs policy from current main; any future change to allow non-allowlisted explicit cron models should be a new, deliberate design discussion that updates allowlist semantics across code, tests, and docs.
