## Summary

Current main already fixes the likely v2026.4.29 regression: shared channel typing startup no longer blocks the reply path, and Slack plus Telegram both use that shared typing callback for inbound replies. The fix is on main only; no release tag contains it yet.

## What This Changes

Fix a v2026.4.29 regression where Slack app mentions and Telegram messages stopped producing agent responses while WebChat still worked.

## Best Possible Solution

Keep the generic shared typing callback fix on main and let it ship in the next release; reopen only with fresh logs if Slack or Telegram still fail after running a build containing 45b86450795d8bd3d1e548c8815cd97d6583199d.

## Reproduction Assessment

Yes. The high-confidence reproduction is a pending or hung channel typing start in v2026.4.29: onReplyStart awaited it before reply processing, and Slack/Telegram both wire inbound replies through that shared callback.

## Solution Assessment

Yes. Fixing the shared channel typing callback is the narrowest maintainable solution because it covers both Slack and Telegram without adding channel-specific bypasses or changing provider/runtime behavior.
