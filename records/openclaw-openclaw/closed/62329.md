## Summary

Current `main` already routes assistant PNG/data-URL history through managed media records instead of leaving raw base64 image payloads in `chat.history`, and the shipped regression test covers that exact reload path.

## Evidence

- **Assistant image replies are converted to managed image blocks before history/display reuse:** `buildWebchatAssistantMessageFromReplyPayloads` collects reply `mediaUrl`/`mediaUrls` and calls `createManagedOutgoingImageBlocks(...)`, so assistant image payloads are persisted as managed media instead of remaining inline base64 in chat history.
  - file: [src/gateway/server-methods/chat.ts:340](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/gateway/server-methods/chat.ts#L340)
  - command: `nl -ba src/gateway/server-methods/chat.ts | sed -n '330,380p'`
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Managed-image path persists image data URLs to stored media records:** `createManagedOutgoingImageBlocks(...)` parses `data:image/...;base64,...`, saves the image via `saveMediaBuffer(...)`, and writes a managed record keyed by `attachmentId`, so the expensive image handling happens once on ingest rather than on every later history read.
  - file: [src/gateway/managed-image-attachments.ts:793](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/gateway/managed-image-attachments.ts#L793)

  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **History reads use stored transcript messages, not raw image reprocessing:** `chat.history` reads messages with `readSessionMessages(...)`, sanitizes/bounds them, and schedules managed-image cleanup separately; it does not re-run assistant history through the image preparation pipeline on each reload.
  - file: [src/gateway/server-methods/chat.ts:1874](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/gateway/server-methods/chat.ts#L1874)

- **Regression test covers assistant PNG history reload behavior:** The gateway test sends an assistant `data:image/png;base64,...` reply, reloads `chat.history`, and asserts the assistant message contains a managed `/api/chat/media/outgoing/...` image block while the serialized history no longer contains the raw PNG data URL or base64 payload.
  - file: [src/gateway/server.chat.gateway-server-chat.test.ts:793](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/gateway/server.chat.gateway-server-chat.test.ts#L793)
