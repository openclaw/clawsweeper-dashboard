## Summary

Keep open. Current main contains useful mitigations for Bonjour churn and startup runtime-dependency repair, but it still default-starts Bonjour plus other bundled sidecars from empty config, and maintainer discussion explicitly left the broader default-plugin/minimal-profile upgrade policy unresolved.

## What This Changes

The issue asks OpenClaw to prevent default bundled plugins, especially Bonjour, from making upgraded gateways unusable and to provide a safer minimal or configured-only startup path.

## Best Possible Solution

Define and document a maintainer-owned startup and upgrade policy that preserves explicit small plugin sets, keeps optional discovery/channel/provider plugins non-blocking, and gives operators a minimal recovery path.

## Reproduction Assessment

Unclear for the full Debian/npm-global upgrade failure because I did not replay that host path. The remaining policy surface is source-reproducible: current main still default-starts Bonjour and other bundled sidecars for empty config, and tests assert that behavior.

## Solution Assessment

No, current main is only a partial solution. The Bonjour timeout/restart caps and runtime-deps changes are good mitigations, but the remaining default-plugin/minimal-profile behavior needs a maintainer policy decision before implementation.
