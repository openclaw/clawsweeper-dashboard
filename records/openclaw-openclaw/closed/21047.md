## Close Comment

Closing as implemented on current `main` after a Codex review. Discord inbound handling already forwards the sender's Discord user ID into `SenderId` in `extensions/discord/src/monitor/message-handler.process.ts:501`, and the shared inbound metadata formatter includes that value both as `sender_id` in conversation info and as `id` in the sender metadata block in `src/auto-reply/reply/inbound-meta.ts:206` and `src/auto-reply/reply/inbound-meta.ts:250`. The existing tests cover both fields in `src/auto-reply/reply/inbound-meta.test.ts:358` and `src/auto-reply/reply/inbound-meta.test.ts:464`. This behavior is also present in shipped release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`), so there is no remaining maintainer cleanup item here.
