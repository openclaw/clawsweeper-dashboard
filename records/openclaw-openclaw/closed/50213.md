## Summary

Close as duplicate_or_superseded. Current main still lacks Gemini web_search baseUrl support, but open PR #57496 is the narrower active implementation thread for the same remaining work: it adds `webSearch.baseUrl`, provider-level Google baseUrl/API-key fallback, docs, metadata, and regression tests.

## Best Possible Solution

Use #57496 as the canonical implementation thread. The best fix belongs in the bundled Google plugin: add `plugins.entries.google.config.webSearch.baseUrl`, optionally fall back to `models.providers.google.baseUrl`, preserve credential/SecretRef precedence, include the resolved baseUrl in the cache key, and land matching docs, metadata, and tests.
