## Summary

Current main already addresses the reported failure path: blank prompt submissions are skipped before a model call, and the Anthropic transport also prevents an empty `messages: []` payload if sanitized history reaches the transport boundary.

## What This Changes

The issue reports that session bootstrap can persist an empty user message, which is later sanitized to an empty provider payload and causes an Anthropic 400 error.

## Best Possible Solution

Keep the current main behavior; if new artifacts show bootstrap noise after commit 365c986a5b2a823feb9f303d3b8dc4ac332825f9, track that as a narrower current-main regression with a fresh trajectory.
