## Summary

Current `main` no longer matches the reported failure mode. The shared message-tool schema accepts local attachment inputs via `media`, `path`, and `filePath`, Telegram exposes shared `presentation` capability rather than a required provider-specific `buttons` field, and the send runner maps `path`/`filePath` into attachment media before Telegram delivery. The same implementation is already present in release `v2026.4.22`.

## Evidence

- **Message-tool send schema accepts local file inputs and does not require buttons:** `src/agents/tools/message-tool.ts` defines optional `media`, `path`, and `filePath` fields for send actions. Rich UI is exposed through optional shared `presentation.blocks`, where `buttons` is itself optional on a block, not a required top-level send field.
