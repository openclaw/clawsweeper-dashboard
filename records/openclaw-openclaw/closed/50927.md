## Summary

Issue #50927 is a useful idea, but the requested Pulse service-health dashboard fits OpenClaw's plugin/ClawHub extension model rather than OpenClaw core. Current main already covers OpenClaw-specific gateway/channel health and OTLP metrics export; the proposal's broader monitoring of arbitrary workspace services and infra is outside the core scope described in VISION.md.

## Evidence

- **Vision says optional capability should stay out of core:** `VISION.md:52-70` says OpenClaw has an extensive plugin API, core stays lean, optional capability should usually ship as plugins, and new skills should go to ClawHub first.
