## Summary

Close: the Gmail watcher startup blocker identified by the reporter is fixed on current main and shipped in v2026.5.18; current code schedules the watcher as a cancellable post-ready sidecar, and focused tests assert it no longer delays sidecar readiness.

## What This Changes

Request that Gateway hook startup, especially Gmail watcher and model-pricing network work, stop blocking Gateway readiness.

## Best Possible Solution

Keep the shipped post-ready Gmail watcher path and use the canonical core-ready tracker for any remaining channel/plugin sidecar startup contract work.

## Reproduction Assessment

Yes for the affected 2026.4.x path: reporter Windows logs and source inspection localized the delay to awaited Gmail watcher startup. Current main no longer reproduces that path because Gmail watcher startup is scheduled after sidecar readiness and covered by regression tests.

## Solution Assessment

Yes. Moving only Gmail watcher bootstrap to a cancellable post-ready sidecar is narrower than adding a generic hook timeout/config framework and preserves the broader Gateway sidecar readiness contract.
