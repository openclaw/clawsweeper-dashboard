## Summary

Current `main` already contains the handshake-timeout fix this issue asked for, and the changelog shows it shipped in `v2026.3.22`. The gateway now uses a 10 second pre-auth handshake timeout with a runtime override instead of the old 3 second default that was dropping slow CLI/browser startup flows.
