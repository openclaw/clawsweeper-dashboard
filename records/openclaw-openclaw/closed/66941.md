## Summary

The report targets a QClaw/`wechat-access` surface that is not implemented in the OpenClaw core repo. Current main documents WeChat as an external plugin (`@tencent-weixin/openclaw-weixin`), keeps WeChat-specific runtime ownership outside core, and does not contain the reported `QCLAW_*`/`wechat-access` identifiers. This is best handled in the plugin/QClaw ecosystem rather than as an OpenClaw core issue.

## Evidence

- **WeChat support is external to core:** Current docs state that OpenClaw connects to WeChat through Tencent's external `@tencent-weixin/openclaw-weixin` channel plugin, and that the WeChat code does not live in the OpenClaw core repo.
  - file: [docs/channels/wechat.md:10](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/docs/channels/wechat.md#L10)

- **WeChat-specific runtime is plugin-owned:** The same doc says WeChat login, Tencent iLink API calls, media handling, context tokens, and account monitoring are owned by the external plugin, while core stays channel-agnostic.
  - file: [docs/channels/wechat.md:27](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/docs/channels/wechat.md#L27)
