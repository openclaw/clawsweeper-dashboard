## Summary

Close as implemented: current main stages the backup manifest outside selected backup source paths and regression tests verify exactly one root manifest; the fix shipped in v2026.5.9-beta.1, while the reported Homebrew v2026.5.7 build predates it.

## What This Changes

The issue reports that `openclaw backup create --verify` creates an archive with two root `manifest.json` entries and fails verification.

## Best Possible Solution

Keep the containment-aware temp-manifest staging on main and direct v2026.5.7 users to a release containing v2026.5.9-beta.1 or later; track any remaining broken-output-link symptom as a separate focused report if it still reproduces.

## Reproduction Assessment

Yes. The v2026.5.7 source path stages `manifest.json` under ambient `os.tmpdir()` and passes it plus recursive backup sources to `tar.c`; current main has targeted tests for TMPDIR inside the state dir and verifies one root manifest.

## Solution Assessment

Yes. Moving the temporary manifest to a root proven outside all selected backup sources is the narrow maintainable fix and preserves the verifier's exactly-one-root-manifest invariant.
