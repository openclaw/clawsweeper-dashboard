## Summary

This is actionable for the external WeCom plugin package rather than OpenClaw core: the reported package registers `wecom_mcp` at runtime but does not declare it in the manifest contract that current OpenClaw uses for plugin-tool discovery.

## What This Changes

Allow plugin-registered tools listed in agent `tools.alsoAllow` to remain available in isolated cron sessions.

## Best Possible Solution

The WeCom plugin should publish a release that declares `contracts.tools: ["wecom_mcp"]`; OpenClaw should only reopen this if a contract-compliant plugin is still filtered from isolated cron sessions.

## Reproduction Assessment

Yes by source and package inspection, but not by a live cron run. The specified WeCom package registers `wecom_mcp` while omitting the manifest `contracts.tools` entry that current OpenClaw uses to select plugin-tool runtimes.

## Solution Assessment

Yes, the narrow maintainable fix belongs in the external plugin manifest. Changing OpenClaw to load manifestless plugin runtimes for isolated cron would bypass the documented manifest discovery contract and reopen hot-path runtime loading.
