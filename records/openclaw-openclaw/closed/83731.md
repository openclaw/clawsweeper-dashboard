## Summary

Close as duplicate/superseded: the same remaining configurable no-persist/per-turn model fallback policy is already tracked in https://github.com/openclaw/openclaw/issues/80502, while current main still intentionally persists automatic fallback selections and the related open PRs cover narrower cleanup paths rather than the config/API decision.

## What This Changes

Requested a `gateway.routing.persist_auto_fallback` option so automatic model fallback can be used for the current turn without persisting `modelOverrideSource: "auto"` into session state.

## Best Possible Solution

Track the configurable no-persist fallback policy in https://github.com/openclaw/openclaw/issues/80502, use this issue's outage data as supporting evidence, and choose one config contract before implementation.

## Reproduction Assessment

Yes at source level for current behavior: current main still writes and preserves `modelOverrideSource: "auto"`, and docs/tests codify sticky auto fallback. I did not run a live timeout-plus-unreachable-fallback scenario, so the production outage path remains source-supported rather than end-to-end reproduced here.

## Solution Assessment

Not as a separate tracker. The requested flag is a plausible config shape, but the maintainable path is to consolidate the fallback persistence API decision under https://github.com/openclaw/openclaw/issues/80502 rather than keep competing config proposals open.
