## Summary

Keep open. Current main still turns Feishu message-tool presentation button blocks into markdown fallback text instead of native Feishu card actions, and the same-author PR at https://github.com/openclaw/openclaw/pull/76055 remains the active implementation candidate rather than a reason to close this issue early.

## What This Changes

The issue asks for Feishu `message` tool presentations with button blocks to render as interactive Feishu cards instead of plain text.

## Best Possible Solution

Revise and land https://github.com/openclaw/openclaw/pull/76055, or a narrow replacement, that shares a lightweight Feishu native presentation-card renderer between message actions and outbound payloads while preserving text, threading/media behavior, fallback text, and lazy runtime boundaries.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: a Feishu `message` tool send with `presentation.blocks[].type === "buttons"` enters `buildFeishuPresentationCard()`, and the current test asserts markdown fallback text instead of a native action element.

## Solution Assessment

Yes at the issue level. The maintainable direction is to reuse or extract the existing Feishu native card block renderer for message-action presentations; the current open PR needs revision and real Feishu proof before it is the best mergeable implementation.
