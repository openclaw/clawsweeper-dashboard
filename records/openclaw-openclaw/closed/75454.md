## Summary

Keep open. Current main still has a source-reproducible path where a persisted or caller-supplied 272k context cap can win before explicit provider/model `contextWindow` or `contextTokens` config is consulted; the broader Copilot 1M auto-promotion question is related but separate.

## What This Changes

The issue asks OpenClaw to report/use 1M context for `github-copilot/claude-opus-4.x-1m` models by honoring explicit context config, explicit `context1m`, or a 1M model-id suffix.

## Best Possible Solution

Honor explicit provider/model `contextWindow` and `contextTokens` before stale persisted or discovered caps across resolver, status, and session metadata surfaces; keep proxied-provider `context1m` policy review on the related PR.

## Reproduction Assessment

Yes, source-reproducible. A session whose stored `entry.contextTokens` is 272000 and whose `github-copilot` model config sets `contextTokens` or `contextWindow` near 1M will pass the stored value through `contextTokensOverride`, and current main returns it before checking explicit config.

## Solution Assessment

Yes for the narrow explicit-config precedence repair. The maintainable fix is to make explicit provider/model context config beat stale persisted or discovered caps, while leaving Copilot-wide 1M auto-promotion to the related provider-policy review.
