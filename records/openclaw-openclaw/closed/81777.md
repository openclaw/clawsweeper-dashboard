## Summary

Keep open: the PR addresses a real Codex app-server isolation gap, but it is not merge-ready because bound conversation turns still queue by transcript file before using the new isolation key. GitHub also reports the draft branch as conflicting with current main.

## What This Changes

The PR adds opt-in Codex `appServer.clientIsolation=session`, related app-server config fields, runtime-policy key plumbing, binding/client cleanup, and tests across Codex, gateway, commands, and session reset flows.

## Best Possible Solution

Keep the opt-in Codex session-isolation direction, but make every shared state and bound-turn queue use the same runtime-policy isolation key while preserving current main app-server lifecycle and auth behavior.

## Reproduction Assessment

No local high-confidence runtime reproduction was established. Source inspection shows current main shares one Codex app-server client per agent, and the PR body supplies real setup proof, but the cascade stall depends on a wedged Codex worker that was not reproduced here.

## Solution Assessment

No, not yet. The opt-in session-isolation design is a reasonable narrow direction, but bound conversation turn queueing must be keyed by the same isolation key before the patch satisfies its contract.
