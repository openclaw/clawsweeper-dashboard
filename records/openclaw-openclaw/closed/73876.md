## Summary

Keep open. Current main still leaves direct `moonshotai/kimi-k2.6` on provider id `moonshotai`, while the bundled Moonshot provider and catalog remain keyed as `moonshot`; open PR #74946 is the active closing candidate but needs runtime-resolution coverage before this issue can close.

## What This Changes

Accept OpenRouter-style `moonshotai/<model>` direct Moonshot model refs while preserving existing `moonshot/<model>` configs.

## Best Possible Solution

Land a reviewed backward-compatible provider alias that maps `moonshotai` and likely `moonshot-ai` to canonical `moonshot` at runtime, with direct model-resolution regression coverage and a changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows `moonshotai/kimi-k2.6` is parsed through provider normalization, but no `moonshotai` alias exists and the Moonshot catalog only exposes `moonshot/kimi-k2.6`.

## Solution Assessment

Yes. Keeping this issue open while the linked PR is reviewed or updated is the right path; a runtime alias is narrower and safer than renaming the canonical provider, while a catalog-only alias is insufficient.
