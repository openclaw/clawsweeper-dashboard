## What This Changes

## Best Possible Solution

Keep #74876/#75099 as the canonical resolution: preserve the safer private-by-default group/channel behavior, use the warning and `config.patch`/docs path to restore legacy automatic replies when desired, and open a new POPO-specific report only if POPO still fails after `messages.groupChat.visibleReplies` is set to `automatic`.
