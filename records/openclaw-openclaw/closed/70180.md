## Summary

Keep open. Current `main` still has the broader media-understanding lookup gap described in the issue: the image-understanding path does a single direct registry lookup and still throws `Unknown model` on miss, while the newer provider-prefixed custom-model fallback was added only in a separate shared helper/test path. The latest maintainer comment also says the canonical fix is PR #71458 and explicitly leaves this issue open until that lands.

## Evidence
