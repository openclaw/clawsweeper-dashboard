## Summary

Current main and the latest release already route normal compaction through safeguard mode, where prior summaries are re-distilled instead of passed to the upstream update prompt that preserves prior summary text verbatim.

## What This Changes

The issue asks for compaction summaries to re-compress prior summaries, cap runaway sections, and avoid re-emitting sticky static instructions on every compaction.

## Best Possible Solution

Keep the shipped safeguard redistill path as the default compaction behavior; track the separate proactive scheduler problem in the already-open related issue.

## Reproduction Assessment

No current-main failing repro for the default path: source and tests show default compaction uses safeguard mode, re-distills prior summaries as input, and does not pass `previousSummary` into the upstream preserving updater.

## Solution Assessment

Yes. The current default safeguard path is the narrow maintainable fix because it avoids the dependency's concatenating update prompt while preserving recent context and capping summary size.
