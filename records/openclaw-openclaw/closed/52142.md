## Summary

Current main already solves the linked loopback WebSocket handshake timeout problem with a shipped shared 10s preauth/connect-challenge timeout, production override via OPENCLAW_HANDSHAKE_TIMEOUT_MS, server/client usage of the shared helper, docs/tests coverage, and release-note evidence in 2026.3.22. The PR’s proposed 30s loopback-only branch is obsolete and had review feedback about misclassifying reverse-proxied external traffic when using the raw socket remote address.

## Best Possible Solution

Close this PR as obsolete. Keep the shipped shared 10s gateway handshake budget plus OPENCLAW_HANDSHAKE_TIMEOUT_MS override on main; do not merge the PR’s raw-socket loopback special case. If maintainers later want a different local-only timeout policy, it should be a new trusted-proxy-aware design with focused gateway tests, not this stale branch.
