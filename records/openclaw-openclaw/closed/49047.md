## Summary

Current main already treats silent subagent completions as successful and the gateway wait/CLI path now reports a generic success summary instead of falling back to `No reply from agent.` for empty normalized payloads. The same behavior is present in release `v2026.4.22`.

## Evidence

- **NO_REPLY subagent completion is an expected silent-success path:** Regression test asserts that a subagent completion with `NO_REPLY` suppresses delivery and does not call send/agent, which matches the issue’s expected behavior for silent helper completions.
  - file: [src/agents/subagent-announce.format.e2e.test.ts:714](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/agents/subagent-announce.format.e2e.test.ts#L714)
  - command: `nl -ba src/agents/subagent-announce.format.e2e.test.ts | sed -n '660,745p'`
- **Successful gateway agent runs carry an explicit success summary:** `agent` responses are wrapped with `status: "ok"` and `summary: "completed"` before being returned to wait/CLI callers, even when the normalized result payload list is empty.
  - file: [src/gateway/server-methods/agent.ts:264](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/gateway/server-methods/agent.ts#L264)
  - command: `nl -ba src/gateway/server-methods/agent.ts | sed -n '220,330p'`
- **Gateway CLI/log path prefers the success summary over the old fallback text:** When the returned payload list is empty, the CLI/gateway path logs `response.summary` first and only falls back to `No reply from agent.` if no summary exists. On the shipped release tag, that summary is already `completed` for successful runs.
  - file: [src/commands/agent-via-gateway.ts:165](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/commands/agent-via-gateway.ts#L165)
