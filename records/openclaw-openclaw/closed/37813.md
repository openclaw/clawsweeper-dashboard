## Summary

Current main no longer silently falls back from an explicit user-selected unknown or unavailable model to the primary/default model. User session model selections are treated as exact, the fallback runner honors an empty fallback override without appending the configured primary, xAI now has forward-compatible Grok resolution for the reported Grok 4.1/4.20 family, and status rendering can show the same active fallback line through the shared status text path. I could prove this on current main, but not in the latest tagged release v2026.4.26.

## Best Possible Solution

Close this as implemented on current main. Keep the exact user-selection policy so explicit model choices fail visibly instead of silently falling through to a primary/default model with different permissions; track any separate request to support additional provider-specific model families as a narrower provider issue.
