Overall confidence: 0.95

Full review comments:

- **[P1] Preserve runExec default maxBuffer capacity:** `src/process/exec.ts:115`
  - body: When callers pass an options object without `maxBuffer`, this now sets `maxBuffer` to 64 KiB on non-ARM64 or 128 KiB on ARM64. Node's execFile default is 1 MiB, so existing commands that previously succeeded can now fail with max-buffer errors.
  - confidence: 0.96
- **[P1] Keep the trusted cmd.exe wrapper path:** `src/process/exec.ts:146-148`
  - body: The current code resolves the System32 cmd.exe path before wrapping .cmd/.bat commands, but this branch goes back to `process.env.ComSpec ?? "cmd.exe"`. That lets the environment choose the command wrapper in a security-sensitive execution path.
  - confidence: 0.93
- **[P1] Do not detach without killing the process group:** `src/process/exec.ts:251-254`
  - body: Setting `detached = true` puts the child in its own process group, but the timeout handlers below still use `child.kill("SIGKILL")`, which only targets the direct PID. Shell grandchildren can survive timeout or no-output-timeout cleanup on Apple Silicon.
  - confidence: 0.94
- **[P1] Preserve kill-tree group cleanup semantics:** `src/process/kill-tree.ts:52-54`
  - body: `useProcessGroup` becomes false for every non-Apple-Silicon Unix host, so the delayed SIGKILL path no longer checks or kills `-pid`. That regresses process-tree cleanup on Linux and Intel macOS.
  - confidence: 0.95
- **[P1] Define the thread-pool sizing helpers before including this file:** `src/utils/thread-pool.ts:24-29`
  - body: This new included `src` file calls `getPhysicalCpuCount()` and `getPerformanceCoreCount()` without importing or defining either symbol. Since `tsconfig.json` includes `src/**/*`, this is not a landable module.
  - confidence: 0.92
- **[P2] Make the platform tests compile:** `src/utils/platform.test.ts:44-56`
  - body: Several test callbacks use `await import()` inside non-async functions; this is a TypeScript syntax error before the tests can exercise any behavior.
  - confidence: 0.97
- **[P2] Remove CommonJS require from the ESM utility module:** `src/utils/platform.ts:25-26`
  - body: This repo uses NodeNext/ESM TypeScript, but the new platform utility calls `require()` inside exported functions and even imports `./shell-utils.js` from the wrong directory. Those paths can throw at runtime instead of returning platform data.
  - confidence: 0.88
- **[P2] Stop calling missing thread-pool exports:** `src/utils/tensor-ops.ts:395-405`
  - body: The tensor helpers destructure `getPoolParallelFactor` and `getPoolBatchSize` from `./thread-pool.js`, but the new thread-pool module exports neither name. Any caller of these helpers will fail when it calls `undefined`.
  - confidence: 0.91
