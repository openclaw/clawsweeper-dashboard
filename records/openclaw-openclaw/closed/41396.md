## Summary

Current `main` already contains a Discord gateway readiness/reconnect fix that matches this report's failure mode: startup now waits for READY, restarts once if the gateway never becomes ready, drains stale sockets before reconnecting, and surfaces a hard error if READY still never arrives. That behavior is regression-tested and was shipped in `v2026.3.28`.

## Evidence
