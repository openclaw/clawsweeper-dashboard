## Summary

Keep open: current main does not appear to show a session id or compact cwd/path in Telegram verbose/progress output, and the request is a small but product-sensitive Telegram UX improvement rather than stale or already implemented work.

## What This Changes

Request to add lightweight session identity metadata, especially session id and compact cwd/path, to Telegram verbose mode without adding noise to non-verbose output.

## Best Possible Solution

Add a Telegram-only verbose/progress identity line or header with a stable session id and redacted compact cwd/path, with tests and docs proving non-verbose output stays unchanged.

## Reproduction Assessment

Not applicable as a bug repro; this is a feature request. Source and docs show the current Telegram verbose/progress path exposes tool/progress details but not session id or compact cwd/path metadata.

## Solution Assessment

Unclear until maintainers choose the UX shape. The direction is plausible, but the best fix should decide whether identity belongs in durable verbose bubbles, the progress draft heading, or another Telegram-scoped status surface, and how paths are redacted.
