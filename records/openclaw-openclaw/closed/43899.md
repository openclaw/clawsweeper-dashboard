## Summary

Keep open. Current main still routes Telegram ACP prompts from `BodyForAgent`, which is populated from the inbound Telegram text/caption path, but I did not find a fix commit, release note, or regression test that proves the reported forum-topic permalink/pending-identity failure is resolved. Existing Telegram ACP topic tests mostly verify routing/session binding, not the actual prompt payload handed to ACP on fresh topic turns.
