## Summary

Keep this PR open: current main still lacks the central Telegram future-thread model default behavior, but the PR head is not safe to merge as-is because it has concrete runtime blockers and mixes unrelated gateway, memory, recovery-script, and live E2E tooling into the feature branch.

## What This Changes

The PR adds Telegram parent-chat future-thread `/model` defaults, DM-topic thread-id routing, memory embedding diagnostics/retry changes, gateway probe/recovery changes, and optional Telegram live E2E scripts.

## Best Possible Solution

Land a narrow Telegram-only implementation that persists same-chat parent future-thread model defaults from all `/model` paths, seeds only genuinely new Telegram forum-topic and DM-topic sessions, avoids cross-chat leakage, handles metadata pre-created rows and `direct_messages_topic`, and adds focused regression coverage.

## Reproduction Assessment

Yes, at source level. Current main has no parent future-thread default fields, no parent write from `/model`, and no parent-default lookup when new Telegram thread sessions are initialized.

## Solution Assessment

No. The feature direction is reasonable, but this PR is not the best way to solve it because it is broad and still has concrete runtime and supply-chain blockers; a smaller Telegram-focused revision or replacement is the maintainable path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Scope main-thread defaults by Telegram chat:** `src/sessions/session-key-utils.ts:173`
  - body: For `agent:...:main:thread:<chatId>:<topicId>` keys, this helper collapses the parent root to shared `main`. A `/model` set in one Telegram DM chat can then seed future threads in another chat, so the parent key must retain chat scope or skip this key shape.
  - confidence: 0.91
- **[P2] Seed defaults before metadata rows block inheritance:** `src/auto-reply/reply/session.ts:474`
  - body: Telegram inbound metadata can create the session row before `initSessionState` runs. Because seeding requires `!hasExistingSessionEntry`, the first real turn for a brand-new topic can skip the parent default and start without the inherited model.
  - confidence: 0.86
- **[P2] Gate parent default writes on actual model changes:** `src/auto-reply/reply/directive-handling.impl.ts:427-429`
  - body: The directive-only store callback writes the parent entry whenever it exists, even for non-model directives. A concurrent `/queue`, `/thinking`, or similar directive can write a stale parent snapshot and revert a `/model` future-thread default.
  - confidence: 0.84
- **[P2] Gate inline parent writes on actual model changes:** `src/auto-reply/reply/directive-handling.persist.ts:235-237`
  - body: The inline directive path has the same stale-parent overwrite window: any updated inline directive writes the loaded parent entry back even when this request did not modify the parent default.
  - confidence: 0.82
- **[P2] Use live Telegram context for directive defaults:** `src/auto-reply/reply/directive-handling.impl.ts:321`
  - body: Directive-only `/model` derives the Telegram hint only from persisted session fields. Older main-scoped Telegram thread entries without `channel` or `lastChannel` skip parent default persistence even though the live context is Telegram.
  - confidence: 0.78
- **[P2] Use live Telegram context for inline defaults:** `src/auto-reply/reply/directive-handling.persist.ts:88`
  - body: Inline `/model` also uses only persisted session channel hints. Migrated main-scoped Telegram threads without those fields update the current session but never persist the future-thread default for new Telegram threads.
  - confidence: 0.77
- **[P2] Preserve remote device identity for authenticated probes:** `src/gateway/probe.ts:77`
  - body: For non-loopback authenticated URLs, this branch passes `deviceIdentity: null`, which suppresses the signed device identity already used by current main. Remote paired-device token/probe flows that require identity can fail even with valid credentials.
  - confidence: 0.8
