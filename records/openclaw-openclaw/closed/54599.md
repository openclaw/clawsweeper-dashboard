## Summary

Current `main` has a targeted fix for this class of registry churn: channel resolution now uses a separately pinned channel registry instead of trusting every later `setActivePluginRegistry()` swap, and outbound bootstrap keys/retries off that pinned channel surface. The current code and regression tests match the failure mode in #54599 closely enough to treat it as implemented on `main`.

## Evidence

- **Pinned channel registry added to runtime state:** `setActivePluginRegistry()` now updates a separate channel surface, and `pinActivePluginChannelRegistry()` / `getActivePluginChannelRegistry()` preserve the startup channel snapshot across later active-registry replacements.
