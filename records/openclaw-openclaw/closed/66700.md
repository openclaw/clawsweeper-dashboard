## Summary

Keep this open: current main and the latest release still treat the Nextcloud Talk webhook body as raw text, so structured `{mention-user1}` placeholders are not resolved before mention gating. The linked implementation PR at https://github.com/openclaw/openclaw/pull/79397 appears to cover the exact bug and should be reviewed, landed, or replaced rather than closing this issue now.

## What This Changes

Fix Nextcloud Talk mention gating so structured webhook placeholders like `{mention-user1}` are resolved against the bot identity instead of being matched as plain text.

## Best Possible Solution

Land or replace the focused structured-mention fix in the Nextcloud Talk plugin, preserving plain-text behavior and shared mention policy, then close this issue from the merged PR.

## Reproduction Assessment

Yes, source-reproducible: a group-room `NextcloudTalkInboundMessage.text` containing a JSON body with `{mention-user1}` is checked by regex as `rawBody`, then dropped when `requireMention` is true. I did not run a live Nextcloud Talk server in this read-only review.

## Solution Assessment

Yes, the maintainable fix is plugin-local parsing of Nextcloud Talk structured mention evidence before shared mention-policy evaluation. The existing linked PR or an equivalent replacement is the right path; changing global regex mention helpers would be the wrong layer.
