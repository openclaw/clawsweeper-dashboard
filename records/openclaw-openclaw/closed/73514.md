## Summary

Keep this PR open. Current main still does not preserve or dispatch Feishu `card.action.trigger` sibling fields such as `action.form_value`; the parser narrows callbacks to `action.value`/`tag`, the decoder/fallback only consume `action.value`, and the handler dispatches only the existing structured command text. The PR is an active implementation candidate, but it overlaps #42754, #42908, #43202, and #43953 and needs maintainer selection of the canonical payload encoding.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to choose one canonical Feishu card callback-field fix across #42754, #42908, #43202, #43953, and this PR: preserve missing fields in `parseFeishuCardActionEventPayload`, expose them through the decoder/handler consistently, decide whether synthetic command text or structured JSON is the supported payload format, and add parser plus handler tests for real `card.action.trigger` payloads.
