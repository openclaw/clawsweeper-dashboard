## Summary

Keep this issue open because current main still lacks llama.cpp runtime `n_ctx` discovery, and #74057 is an open, unmerged closing PR for the reported behavior.

## What This Changes

The issue asks OpenClaw to show llama.cpp's loaded runtime context window in session context usage instead of the model's trained context ceiling.

## Best Possible Solution

Review and land #74057, or an equivalent narrow provider-discovery fix that records llama.cpp `/props.n_ctx` as `contextTokens` while preserving native `contextWindow` metadata and explicit configuration.

## Reproduction Assessment

Yes, by source inspection and upstream endpoint docs: current main does not query `/props.n_ctx`, while sessions consume persisted/configured `contextTokens` for their denominator. I did not run a live llama.cpp server in this read-only review.

## Solution Assessment

Yes. Capturing the runtime cap during self-hosted provider discovery is the narrow maintainable fix; probing llama.cpp from the sessions display path would spread provider-specific HTTP behavior into a command surface.
