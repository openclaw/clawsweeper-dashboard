## Summary

Current main and the latest release already cover the reported stale config overwrite path: config writes now refresh the canonical source/runtime snapshots and the gateway reloader consumes in-process write notifications before SIGUSR1 restart handling can act on stale state.

## What This Changes

The issue reports that gateway config.patch updates to plugins.load.paths and plugins.entries are written to openclaw.json, then lost when a SIGUSR1 restart overwrites the file from a stale in-memory config snapshot.

## Best Possible Solution

Keep the shipped snapshot-refresh and in-process write hash guard; users seeing this on an older d587705 build should update to v2026.5.22 or newer.

## Reproduction Assessment

No current-main failing reproduction was established in this read-only review. Source and release inspection show the current write/reload path now refreshes the persisted snapshot and suppresses duplicate stale watcher handling.

## Solution Assessment

Yes. The current solution is the narrow maintainable fix: update the canonical source/runtime snapshots after the write and feed that committed write directly into the gateway reloader before restart decisions.
