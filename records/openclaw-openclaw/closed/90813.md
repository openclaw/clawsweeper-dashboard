## Summary

Keep open for normal maintainer merge review: the final head centralizes cwd and ancestor-of-cwd refusal in the shared cleanup helper, adds regression coverage, supplies terminal command proof, and I found no blocking patch defects.

## What This Changes

This PR adds a shared cleanup guard that refuses to remove `process.cwd()` or an ancestor, covers it with cleanup utility tests, and updates uninstall docs to recommend dry-run before non-interactive removal.

## Best Possible Solution

Land this narrow shared cleanup guard after normal CI and maintainer checks, then close the linked data-loss issue after merge.

## Reproduction Assessment

Yes. Current main's `removePath` rejects only empty/root/home before `fs.rm`, and `openclaw uninstall --all` sends configured workspace dirs into that helper; the final-head proof exercises the nested-cwd case.

## Solution Assessment

Yes. Putting the guard in `removePath` is the right layer because both uninstall and reset reach the same destructive helper, and the tests cover both equal-cwd and ancestor-of-cwd cases.
