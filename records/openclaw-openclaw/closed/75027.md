## Summary

Current main already addresses this packaged status regression: plain text `openclaw status` no longer preloads channel setup runtime fallback, and the fallback used by deeper status/channel/doctor paths now resolves loader candidates inside the packaged `dist/` root. The fix is main-only in the available release evidence; `v2026.4.27` still contains the old escaping `../../plugins/loader.*` candidates.

## What This Changes

The issue reports that `openclaw status` crashes in the npm-packaged 2026.4.27 release because the read-only channel/plugin status scan cannot resolve the plugin runtime loader fallback path.

## Best Possible Solution

Keep the current main behavior and let the next release carry it to npm; no additional core issue is needed for this loader-path crash.

## Reproduction Assessment

Yes. The issue gives a concrete packaged npm reproduction and exact URL-resolution evidence; current-main tests directly model the old packaged importer path and assert the corrected candidates.

## Solution Assessment

Yes. Avoiding setup-runtime fallback for plain status and resolving deeper fallback candidates from the detected packaged `dist/` root is the narrow maintainable fix, because source-mode loading remains unchanged while packaged chunks stay package-relative.
