## Summary

Current `main` already contains the missing end-to-end path for TTS tool audio generation and WhatsApp voice-note delivery. The reported 2026.4.1/2026.4.2 behavior does not match the current code: TTS now writes a temp audio file, the `tts` tool returns that file as `details.media` for reply delivery, and the WhatsApp send path transmits audio with `ptt: true`.

## Evidence

- **TTS tool now returns deliverable media, not just a success string:** `createTtsTool()` calls `textToSpeech()` and, on success, returns `details.audioPath` plus `details.media.mediaUrl` with `trustedLocalMedia` and `audioAsVoice` for downstream delivery.
  - file: [src/agents/tools/tts-tool.ts:57](https://github.com/openclaw/openclaw/blob/4d1ee3a73ed19063d16ac627d3071b7637fda82d/src/agents/tools/tts-tool.ts#L57)

  - sha: [4d1ee3a73ed1](https://github.com/openclaw/openclaw/commit/4d1ee3a73ed19063d16ac627d3071b7637fda82d)
- **Speech core now writes an on-disk temp audio file:** `textToSpeech()` creates `tts-*` under the preferred temp root, writes `voice-<timestamp><ext>`, and returns that `audioPath`. This directly addresses the report's 'no file is written to disk' symptom.
  - file: [extensions/speech-core/src/tts.ts:768](https://github.com/openclaw/openclaw/blob/4d1ee3a73ed19063d16ac627d3071b7637fda82d/extensions/speech-core/src/tts.ts#L768)
