## Summary

Current `main` already contains the routing guards for this report, and the latest shipped release includes them as well. The code now preserves bound thread/conversation delivery targets for announce/session-fallback paths and seeds missing `deliveryContext.to/threadId` on spawned sessions, which matches the failure mode described in #41562.

## Evidence

- **Spawned sessions no longer keep channel-only delivery context:** `src/gateway/server-methods/agent.ts:645` seeds missing `deliveryContext` fields from the request (`channel`/`to`/`accountId`/`threadId`) and explicitly documents that the old failure mode was a channel-only context causing announce delivery to hit the wrong channel when the parent `lastTo` drifted.
  - file: [src/gateway/server-methods/agent.ts:645](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/gateway/server-methods/agent.ts#L645)

- **Completion routing prefers the bound conversation destination:** `src/agents/subagent-announce-delivery.ts:239` resolves a bound delivery route for task completions and returns the bound conversation target before any fallback merge.
