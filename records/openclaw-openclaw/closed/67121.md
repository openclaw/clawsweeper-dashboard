## Summary

Current `main` already persists MiniMax OAuth through the shared auth-profile store and includes the `minimax-portal` config patch fields (`api: "anthropic-messages"`, `authHeader: true`) needed for Bearer auth to work. The report's expected `~/.openclaw/credentials/...` path is obsolete in current docs and code; auth is stored in `auth-profiles.json`, and the relevant MiniMax OAuth fix is recorded in the `v2026.4.11` changelog with regression coverage on `main`.
