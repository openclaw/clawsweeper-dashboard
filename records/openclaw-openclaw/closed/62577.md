## Summary

Current main implements the requested recovery path: stale ACP-backed tasks whose ACP session metadata is gone are reconciled to terminal `lost` state instead of remaining zombie `running` tasks. Direct ACP cancellation still requires ACP metadata, but the issue explicitly accepted auto-recovery by marking the task failed/lost, and that behavior is now present and shipped in v2026.4.24.

## Evidence
