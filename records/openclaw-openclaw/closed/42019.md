## Summary

Close #42019 as a duplicate of the broader open tracker #48360. Current main has related probe/status improvements, but the specific accepted-WebSocket/read-timeout false-negative class is still represented in code and tests, and #48360 is already tracking that remaining work.

## Best Possible Solution

Use #48360 as the canonical open tracker. The best product/code direction is to align `openclaw gateway probe` with the documented split state: treat accepted WebSocket connect as reachable, surface detail/read RPC timeout as degraded or read-not-ready, and keep startup/readiness/channel readiness separate in the output and JSON fields.
