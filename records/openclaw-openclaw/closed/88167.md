The issue reports that strict inline-eval approval timeouts with exec ask off are delivered through a followup channel, leaving the calling agent with a generic missing-tool-result transcript repair placeholder instead of the real denial reason.

## Best Possible Solution

Keep strictInlineEval denial semantics, but make approval-timeout denials reach the same calling session as a specific denied tool result or equivalent same-session denial signal instead of generic transcript repair noise.
