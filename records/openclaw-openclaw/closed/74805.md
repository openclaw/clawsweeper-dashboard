## Summary

Keep this PR open: current main still has the production/test-runtime mismatch, and the branch applies a narrow kernel fix with regression coverage. I found one low-priority docs follow-up for the public `runPrepared` observe-only behavior, but no blocking correctness or security issue in the patch.

## What This Changes

The PR refactors channel-turn dispatch through a private core helper so direct `runPrepared` observe-only admissions return `observeOnlyDispatchResult` without invoking `runDispatch`, while preserving assembled-turn dispatch behavior and adding a regression test.

## Best Possible Solution

Merge the narrow kernel and regression-test fix after updating the public channel-turn SDK docs so third-party plugin authors understand direct prepared observe-only turns and `observeOnlyDispatchResult`.

## Reproduction Assessment

Yes. On current main, call `runtime.channel.turn.runPrepared` with `admission: { kind: 'observeOnly' }` and a spy `runDispatch`; the production function still invokes `runDispatch`, while the SDK mock and full `run(...)` path suppress it.

## Solution Assessment

Yes, with the docs clarification. Moving suppression into the exported prepared-turn wrapper while keeping assembled turns on an unsuppressed core helper is the narrow maintainable fix because it aligns production with SDK tests without changing assembled no-op delivery semantics.
