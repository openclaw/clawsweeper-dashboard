## Summary

Close as superseded by the open canonical status-surface request in #52640. Current main already has event-driven liveness surfaces for long chat turns, but not the strict fixed-interval heartbeat lifecycle proposed here; the remaining product work is better tracked in the newer durable task-status thread that explicitly references this request.

## What This Changes

Request to add optional global and per-channel fixed-interval progress heartbeat updates for long-running chat turns, with started/running/done/failed lifecycle messages and configurable cadence/templates.

## Best Possible Solution

Keep #52640 as the canonical design and implementation thread for long-running channel status. If maintainers still want fixed cadence, capture it there as an optional fallback policy for the durable status surface rather than as a separate global ticker.

## Reproduction Assessment

Not applicable as a feature request. The observable gap is source-checkable: current main has preview streaming, tool-progress, and typing refreshes, but no dedicated fixed-interval progress heartbeat lifecycle config.

## Solution Assessment

Yes. Superseding this with #52640 is the narrower maintainer cleanup because it preserves the same user problem while routing implementation toward a single durable status contract instead of a potentially noisy standalone heartbeat stream.
