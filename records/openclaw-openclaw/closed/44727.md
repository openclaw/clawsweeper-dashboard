## Summary

Current main no longer takes the plugin-runtime startup path for `openclaw secrets configure`; the command is manifest/public-artifact based, so the Feishu runtime registration logs from the report are not on the interactive configure path anymore.

## Evidence

- **CLI startup defaults to no plugin preload, and `secrets` is a bypassed command path:** The command policy defaults `loadPlugins` to `"never"`, prefix-matches command paths, and includes `commandPath: ["secrets"]`, so `secrets configure` inherits the no-plugin-preload startup policy.
  - file: [src/cli/command-path-policy.ts:5](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/cli/command-path-policy.ts#L5)
  - command: `nl -ba src/cli/command-path-policy.ts | sed -n '1,80p'; nl -ba src/cli/command-catalog.ts | sed -n '30,115p'; nl -ba src/cli/command-path-matches.ts | sed -n '26,35p'`
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Preaction exits before bootstrap/plugin loading for bypassed paths:** `registerPreActionHooks` returns immediately when `shouldBypassConfigGuardForCommandPath(commandPath)` is true, so the normal bootstrap that would load the plugin registry is skipped for `secrets ...` commands.
  - file: [src/cli/program/preaction.ts:64](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/cli/program/preaction.ts#L64)
  - command: `nl -ba src/cli/program/preaction.ts | sed -n '64,95p'; nl -ba src/cli/command-startup-policy.ts | sed -n '1,70p'; nl -ba src/cli/command-execution-startup.ts | sed -n '50,70p'`
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **`secrets configure` now builds from config/target registry data, not plugin runtime init:** The interactive action calls `runSecretsConfigureInteractive`, which reads config snapshots and builds candidates via `buildConfigureCandidatesForScope`. That in turn discovers targets from the secret target registry. The registry enumerates bundled plugins from manifest data and bundled channel public artifacts, not runtime registration hooks.
