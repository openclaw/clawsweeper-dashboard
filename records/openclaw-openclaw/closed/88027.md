## Summary

Current main now fixes the central Feishu post-mode mention bug: structured mention targets are serialized as native post `at` elements, while arbitrary body text remains literal to avoid unintended pings. The fix is merged on main but not included in the latest stable release yet.

## What This Changes

The issue requests fixing Feishu post-mode replies so OpenClaw-generated @ mentions render as real Feishu mentions instead of plain `<at user_id="...">name</at>` text, with a secondary cache-collision concern.

## Best Possible Solution

Keep the merged structured-mention native-at implementation on main, and leave arbitrary body `<at ...>` text literal unless a separate maintainer-approved ping-safety change is opened.

## Reproduction Assessment

Yes at source level: the latest release-era code still wraps generated mention text into one post `md` element, while current main emits native `at` elements for structured mentions. No live Feishu tenant reproduction was run in this read-only review.

## Solution Assessment

Yes: the merged fix uses structured mention targets rather than reparsing arbitrary assistant/body text, which is the safer maintainable boundary for avoiding unintended live pings. The cache symbols named in the report are not present on current main, so no speculative cache change is needed for this close.
