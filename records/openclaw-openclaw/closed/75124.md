## Summary

Keep open: current main still bypasses the effective tool-policy pipeline for `command-dispatch: tool`, and the remaining fix work is already represented by open PRs rather than a completed main-branch implementation.

## What This Changes

The issue reports that tool-dispatched skill slash commands construct raw tools and skip profile, group, sandbox, and subagent policy filtering.

## Best Possible Solution

Land or replace one canonical inline dispatch fix so skill tool commands resolve through the same effective policy surface as normal agent turns, with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows the inline skill dispatch branch still constructs raw tools and applies only owner-only filtering, so a restricted `tools.deny` or leaf-subagent skill command would exercise the bug.

## Solution Assessment

Yes. The requested direction is the narrow maintainable fix: route inline skill tool dispatch through the shared effective-tool policy pipeline or a small helper around it, rather than layering another ad hoc filter onto raw tool construction.
