## Best Possible Solution

Land a narrower classifier that preserves the read-only probe cases while keeping flag or subcommand variants with side effects fail-closed and covered by regression tests.

No, not as submitted. The right fix is the same classifier boundary, but with command-specific flag handling so mutating git and OpenClaw doctor variants stay fail-closed.

- **[P2] Keep mutating git variants fail-closed:** `src/agents/tool-mutation.ts:196`
  - body: This only checks whether the first git subcommand is in the read-only set, so `git branch -D feature` and `git remote add origin ...` become non-mutating because `branch` and `remote` are allow-listed.   - confidence: 0.93

  - body: Allow-listing bare `doctor` makes `openclaw doctor --fix` read-only because `isReadOnlyOpenClawCommand` accepts the single subcommand before looking at flags. `--fix` repairs config or state, so failed repair attempts would be hidden like harmless probes instead of going through the mutating failure path.
  - confidence: 0.92
