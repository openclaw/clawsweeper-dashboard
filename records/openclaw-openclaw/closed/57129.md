## Summary

Close as implemented. Current main routes actionable Slack top-level mentions into thread-scoped session keys derived from the root Slack timestamp, keeps same-thread follow-ups on that same key, and shipped that behavior in v2026.4.27.

## What This Changes

Slack should start fresh isolated context for unrelated new top-level questions while preserving same-thread context for follow-ups.

## Best Possible Solution

Keep the current Slack thread-routing behavior: actionable top-level room questions get their own thread session, later Slack thread replies reuse that session, and ordinary unmentioned channel chatter is not forced into separate sessions.

## Reproduction Assessment

Yes. The deterministic routing path is `resolveSlackRoutingContext` with `replyToMode: "all"` and `seedTopLevelRoomThread: true`; the existing regression test covers a top-level app mention and later Slack thread reply using the same root-thread session key.

## Solution Assessment

Yes. The current implementation is the narrow maintainable fix because it isolates actionable top-level Slack questions without reverting to making every channel message its own thread session.
