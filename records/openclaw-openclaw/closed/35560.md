## Summary

Current main no longer runs Doctor repairs from the materialized/defaulted runtime config, so Telegram schema defaults are not injected into a valid multi-account `channels.telegram.accounts.default` config. The same relevant implementation is present in `v2026.4.24`.

## Evidence

- **Doctor uses authored source config:** Doctor preflight returns `baseConfig: snapshot.sourceConfig ?? snapshot.config ?? {}`, and `loadAndMaybeMigrateDoctorConfig` clones that base into the candidate it repairs. This keeps Doctor off the runtime/materialized config that contains schema defaults.
