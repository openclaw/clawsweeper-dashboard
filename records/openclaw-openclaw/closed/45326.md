## Summary

Keep open: current main and the latest release still clear TUI input before any busy-run decision and still allow normal chat sends to reach the gateway without an active/pending-run guard; the linked fix path is the open PR at https://github.com/openclaw/openclaw/pull/82774, so this issue should remain open until that PR or a replacement lands.

## What This Changes

The issue reports that normal TUI text submitted during an active model run is cleared or later replayed instead of being rejected, preserved, or treated as an explicit abort.

## Best Possible Solution

Land a focused TUI fix that gates normal chat submission before draft loss, preserves or restores blocked drafts, keeps Esc and explicit abort commands as the interrupt path, guards the send path against active/pending runs, and adds regression coverage for busy normal submits.

## Reproduction Assessment

Yes, at source level. Current main clears the TUI editor before any active-run decision and then sends normal chat without checking active or pending run state, which matches the reported swallowed-input path.

## Solution Assessment

Yes, the best fix shape is narrow and local to the TUI submit/send boundary. The linked PR is the right implementation path to review, but it must address the real editor-clearing boundary and proof requirements before merge.
