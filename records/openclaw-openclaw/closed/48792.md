## Summary

Keep open: the linked bug is still valid on current main, but this PR needs rework because it patches a stale normalization path, strips all string parameters including write/edit payloads, and lacks real after-fix behavior proof.

## What This Changes

The PR adds an XML suffix-stripping helper for tool arguments, applies it to the exec command string, and adds unit tests around the helper and legacy parameter normalization.

## Best Possible Solution

Land a rebased, field-scoped sanitizer at the active tool-argument boundaries that covers command/path-like exec and read inputs, preserves legitimate write/edit XML-like content, and adds positive and negative regression coverage.

## Reproduction Assessment

Yes for source-level reproduction: if current main receives command or path strings ending in the reported suffix, the active exec and read paths forward those values unchanged. I did not rerun the live Windows DashScope/Qwen provider path.

## Solution Assessment

No. The defensive direction is right, but this PR patches an inactive legacy normalization path on current main, mutates all string params including payload text, and only sanitizes exec command rather than the full affected string-option set.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Move stripping to the active parameter boundary:** `src/agents/pi-tools.params.ts:140-142`
  - body: `normalizeToolParams()` is no longer the active read/write/edit boundary on current main; those tools validate and forward params through `wrapToolParamValidation()`. Rebase the sanitizer onto the current wrappers so the fix covers the runtime path that actually handles read/write/edit calls.
  - confidence: 0.9
- **[P1] Restrict suffix stripping to command and path fields:** `src/agents/pi-tools.params.ts:140-142`
  - body: The record-wide sanitizer runs after `content`, `oldText`, and `newText` normalization, so valid write/edit payload text ending with `</arg_value>` can be silently truncated. Scope stripping to affected command/path-like fields and add negative tests that preserve payload content.
  - confidence: 0.91
- **[P2] Sanitize all exec string options consistently:** `src/agents/bash-tools.exec.ts:230`
  - body: Exec only strips `command`, but `workdir`, `host`, `security`, `ask`, and `node` are also string params consumed later. If the provider suffix lands in one of those values, exec can still fail or choose the wrong option.
  - confidence: 0.82
- **[P3] Allow any trailing suffix chevrons:** `src/agents/pi-tools.params.ts:96`
  - body: The regex caps the malformed closing tag at three total trailing `>` characters, so a more malformed value such as `</arg_value>>>>` leaves residual characters. Use an unbounded trailing-chevron matcher for this exact suffix shape.
  - confidence: 0.8
