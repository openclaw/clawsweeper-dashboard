## Summary

Close as configuration/support rather than a Telegram routing bug. Current main shows `groups:unmentioned` is emitted when any configured Telegram group has `requireMention:false`, and group/channel turns intentionally suppress normal final replies by default unless the agent uses the message tool or `messages.groupChat.visibleReplies` is set to `automatic`; the follow-up Discord data point fits the same visible-reply symptom but does not identify a missing source fix.

## What This Changes

The issue reports Telegram supergroup messages being received and processed without visible group replies despite `requireMention:false`, and asks whether extra routing, migration, or diagnostic config is required.

## Best Possible Solution

Keep the current message-tool-first group safety default and route affected users to `messages.groupChat.visibleReplies: "automatic"` when they want DM-like visible group finals; a new source bug should be opened only if Telegram still fails with that setting and a sanitized single-turn log.

## Reproduction Assessment

Yes. Current code and tests provide a high-confidence reproduction path for the visible-reply symptom: a group turn can process, show typing, and suppress normal final delivery under the default `message_tool_only` policy.

## Solution Assessment

Yes. Configuration guidance is the narrowest maintainable solution here because changing `requireMention` or Telegram routing would not address the documented visible-reply policy and would weaken the group safety default.
