## Summary

Current main implements the requested structured cron introspection through the built-in agent `cron` tool and Gateway `cron.*` RPCs. Agents with owner authorization can call `cron` actions `status`, `list`, and `runs`; the returned job objects include schedule, enabled state, agent/session binding, next run, last run, and last-run status fields.

## Best Possible Solution

Close this broad feature request as implemented on main. Keep the shipped `cron` agent tool and Gateway `cron.list`/`cron.status`/`cron.runs` APIs as the supported structured introspection path; any remaining convenience request, such as a per-agent filter or mirroring cron summaries into `session_status`, should be tracked as a narrower follow-up.
