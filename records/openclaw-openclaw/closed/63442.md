## Summary

Close as implemented on current `main`: isolated cron agent-turn runs now reuse an explicit job `sessionKey` instead of always forcing `agent:<agentId>:cron:<jobId>:run:<sessionId>`, and expired cron `:run:` receipt entries are now explicitly reaped by `cron.sessionRetention`.

## Evidence

- **Store writer only creates separate run receipts when needed:** `createPersistCronSessionEntry` writes both `agentSessionKey` and `runSessionKey` only when they differ. That means an explicit shared session key no longer creates extra per-run `:run:` entries.
