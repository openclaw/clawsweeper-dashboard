## Summary

Current main already implements the reporter's preferred Bedrock credential probe in the Amazon Bedrock plugin, so auto-selection can skip Bedrock before the first embed call when AWS credentials are unavailable.

## What This Changes

The issue requests that the Bedrock memory-embedding adapter probe AWS credentials during provider creation so `auto` can fall through instead of crashing on first embed.

## Best Possible Solution

Keep the provider-owned credential gate in the Amazon Bedrock plugin and let users consume a build after b9d6bb91ec034159d4c2ae2e569ab4c54ff46651 or the next release that contains it.

## Reproduction Assessment

No current-main repro remains: source inspection shows Bedrock `create()` now probes AWS credentials and throws a skippable missing-key sentinel before provider construction. I did not run the CLI because this review was read-only, but the current source and regression tests cover the reported path.

## Solution Assessment

Yes. The current implementation is the narrow owner-plugin fix the reporter suggested, avoids a new adapter contract, and reuses the existing auto-selection sentinel behavior.
