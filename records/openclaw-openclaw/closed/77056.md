## Summary

Keep open: current main still has the broad chat-completion session refresh and uncached CLI-provider classification path, and open PR #77057 is the linked implementation candidate rather than a reason to close before merge.

## What This Changes

Request to reduce Gateway stalls from large session stores by caching CLI-provider classification and bounding Control UI chat-turn `sessions.list` refreshes.

## Best Possible Solution

Review and land #77057 or an equivalent narrow change that caps only the chat-turn session refresh and caches CLI-provider classification while preserving explicit Sessions page and search behavior.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows the current-main path that can issue a broad chat-completion `sessions.list` request, but the large session store and latency measurements were not recreated locally.

## Solution Assessment

Yes. The linked approach is narrow because it uses the existing `sessions.list` limit contract for routine chat-turn refreshes and localizes the CLI-provider cache to the repeated classification helper.
