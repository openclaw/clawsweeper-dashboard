## Summary

Current main already solves this PR's central gateway/webchat stream-merge problem without the capped suffix-overlap approach. The old `appendUniqueSuffix`/`MAX_DELTA_OVERLAP` path is gone from the gateway code; live assistant events now merge through `live-chat-projector`, appending real deltas directly and accepting cumulative prefix snapshots. Tests cover repeated boundary characters, and the changelog records the repeated-boundary merge fix on main.
