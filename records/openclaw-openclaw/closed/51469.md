## Summary

Current main still does not implement the remaining request. The shipped gateway pre-auth/connect-challenge timeout is 10 seconds with an environment override, while the issue asks for a 15 second default plus a persistent `gateway.handshakeTimeoutMs` config field for cold-start CLI cases observed at 12 seconds or more. The paired implementation PR #51503 is closed unmerged, and the related closed reports only cover the earlier 3s-to-10s/env mitigation, not this remaining configurable timeout.

## Best Possible Solution

Keep this issue open. Maintainers should either land a scoped version of the #51503 direction, with bounded validation and docs for `gateway.handshakeTimeoutMs`, or explicitly close it only after current-main timing evidence shows the existing 10s default plus `OPENCLAW_HANDSHAKE_TIMEOUT_MS` is sufficient for the reported cold-start CLI path.
