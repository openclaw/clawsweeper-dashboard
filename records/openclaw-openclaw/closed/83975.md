## Summary

This issue should stay open: current main still drops a configured ClawHub base URL pathname when building API requests, and the linked open fix PR has not merged yet.

## What This Changes

Preserve non-root path prefixes from `OPENCLAW_CLAWHUB_URL` or `CLAWHUB_URL` when OpenClaw builds ClawHub API request URLs.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83982 if its URL join and regression test remain correct; otherwise apply the same focused fix to the shared ClawHub URL builder and tests.

## Reproduction Assessment

Yes. Source inspection plus Node's URL resolution contract show that current main drops `/clawhub` from a prefixed base URL when the request path starts with `/api/v1/...`.

## Solution Assessment

Yes. Fixing the shared `src/infra/clawhub.ts` URL builder and adding a focused regression test is the narrowest maintainable solution because all affected ClawHub endpoints flow through that helper.
