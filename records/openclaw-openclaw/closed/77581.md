## Summary

Current main and shipped builds after the reported 2026.4.24 package no longer label an inherited configured thinking level as `Default (off)`; the gateway now exposes the effective configured default and the Control UI consumes it for the chat and sessions selectors.

## What This Changes

The issue asks the Control UI thinking selectors to show the effective inherited thinking level separately from explicit per-session overrides.

## Best Possible Solution

Keep the shipped gateway/UI default-label contract and ask for a new report only if a current `v2026.4.26` or newer build still displays `Default (off)` for an effective non-off default.

## Reproduction Assessment

Yes. The source-level reproduction path is a session with no `thinkingLevel` override and an effective configured `thinkingDefault`; current main renders `Default (high)` or the resolved level from row/default metadata instead of `Default (off)`.

## Solution Assessment

Yes. The current shipped fix preserves `value=""` as clear-inherit behavior while displaying the effective inherited level, which addresses the misleading off label without changing runtime thinking behavior.
