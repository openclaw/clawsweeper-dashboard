## Summary

This issue is still necessary: current main and the latest release still resolve the Z.ai GLM-5 path without a manifest `baseUrl` fallback, and the open linked PR is the active fix candidate rather than a merged implementation.

## What This Changes

The issue requests that bundled Z.ai static/GLM-5 model resolution inherit the provider manifest `baseUrl` so Z.ai requests do not fall through to the OpenAI API host.

## Best Possible Solution
