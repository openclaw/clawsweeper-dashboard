## What This Changes

The PR changes task-session maintenance to normalize running cron job ids with `normalizeCronLaneSegment`, updates the maintenance contract comment, and adds a regression test for a non-slug cron job id.

## Best Possible Solution

Broaden the preservation set to every cron-run key segment current producers can create, then cover both the main-session slugged key and default isolated raw non-slug key in focused tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence repro path: seed a stale running row `agent:main:cron:daily report:run:old-run` plus a running default isolated cron job id `Daily Report`; this PR's slug-only set would miss and prune that row.

## Solution Assessment

No. Matching the slugged main-session producer is necessary, but replacing the raw lowercase key is too narrow because default isolated cron-run rows are still produced from raw `cron:${job.id}`.
