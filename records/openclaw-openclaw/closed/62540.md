## Summary

Close as duplicate. The older open tracker #49251 covers the same remaining behavior: preserve API-limit-blocked work and resume it automatically when capacity is available. Current main has immediate mitigations for 429s through auth-profile/key rotation, cooldowns, and model fallback, but not the durable paused-rate-limited run state requested here, so the implementation should be consolidated on the canonical open issue.

## What This Changes

Request OpenClaw to treat provider 429/rate-limit failures as resumable pauses, parse provider retry hints, schedule automatic retry, and continue the pending model call without a new user prompt.

## Best Possible Solution

Use #49251 as the single implementation thread for capacity-wait/resume behavior, carrying over this issue's concrete retry-delay parsing and run-state acceptance criteria into that design.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main handles 429s through immediate cooldown/profile/model fallback paths, while the remaining durable pause-and-auto-resume behavior is tracked by the older open issue.

## Solution Assessment

Yes. Closing this as a duplicate is the best cleanup path because the remaining product change should be designed once around the run/session scheduler and wake primitives rather than split across two overlapping trackers.
