## Summary

Keep this PR open. Current main has partially superseded the original core allowlist approach by moving thinking support to provider-owned profiles, and direct OpenAI/OpenAI Codex/GitHub Copilot xhigh support now lives there. But the PR’s proxy-ref use case is still not implemented for Vercel AI Gateway, and the current PR branch also carries stale/unrelated plugin-discovery churn with unresolved review concerns.

## Best Possible Solution

Rework the PR around current main’s provider-profile architecture: add a Vercel AI Gateway `resolveThinkingProfile` that recognizes trusted OpenAI/Codex upstream refs, add focused regression coverage for accepted Vercel OpenAI xhigh refs and rejected unsupported or untrusted namespaced refs, and drop or split the unrelated web-search/plugin-loader/channel test churn before maintainer review.
