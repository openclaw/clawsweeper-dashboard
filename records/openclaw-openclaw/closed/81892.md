## Summary

Keep open: current `main` still has a source-level path where Matrix acknowledges `/reasoning on` but suppresses the explicit reasoning payload before delivery, while the docs define enabled reasoning as a separate message.

## What This Changes

The issue requests Matrix delivery of reasoning messages when `/reasoning on` is enabled, without showing reasoning when it is off.

## Best Possible Solution

Add a narrow Matrix delivery path for explicit `isReasoning` payloads emitted by enabled reasoning, while continuing to suppress raw reasoning-tag leaks and off/default reasoning payloads.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: enabled reasoning emits an `isReasoning` block reply, and current generic dispatch plus Matrix delivery suppress that payload before it reaches Matrix.

## Solution Assessment

Yes. The maintainable fix is a focused Matrix-aware exception for explicit reasoning payloads, not deleting the generic suppression that protects other channels and the prior inverse-leak case.
