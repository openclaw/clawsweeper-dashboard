## Summary

Current main already contains a targeted fix for main-session `systemEvent` cron jobs: cron now forces heartbeat delivery via `target: "last"` for these jobs and the gateway cron adapter preserves that override when invoking the heartbeat runner. New regression tests cover both the cron timer path and the gateway adapter path.

## Evidence

- **Cron timer now forces last-route delivery for main-session jobs:** `executeMainSessionCronJob` enqueues the system event, then passes `heartbeat: { target: "last" }` to both `runHeartbeatOnce` and `requestHeartbeatNow` for `sessionTarget: "main"` jobs.
  - file: [src/cron/service/timer.ts:1205](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/cron/service/timer.ts#L1205)
