## Summary

Issue #41589 asks for a core `skills.entries.<skill>.model` field. Current main does not implement that top-level field; the maintained contract keeps per-skill core config narrow and provides plugin/skill surfaces for model-backed workflows. Given OpenClaw’s stated scope of keeping optional capability in plugins/ClawHub, this is better closed as ClawHub/plugin scope rather than adding another core skill routing control.
