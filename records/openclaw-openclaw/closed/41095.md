## Summary

Current `main` already preserves single-file top-level `plugins` `$include` layouts by writing plugin mutations through to the included file instead of flattening `openclaw.json`. That behavior is wired into the plugin install path, covered by tests, documented in shipped docs, and makes this PR's broader `restoreIncludes()` approach unnecessary. The PR branch also has unresolved review findings showing it would still inline included secrets and could resurrect removed keys.

## Evidence
