## Summary

Keep open. Current `main` now exposes and documents `channels.feishu.streaming`, but the Feishu reply path still enables streaming by default, still throws on Card Kit HTTP 400 create failures, and still resets state so the next message retries streaming again. That means the issue's main latency complaint is not clearly fixed on `main`, even though the older "no way to disable streaming" detail is now outdated.

## Evidence

- **Streaming still defaults on unless explicitly disabled:** The reply dispatcher enables Feishu streaming whenever the reply is not a thread reply, `account.config?.streaming !== false`, and `renderMode !== "raw"`.
