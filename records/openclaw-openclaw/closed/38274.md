## Summary

Current `main` already implements `openclaw plugins update <npm-spec>` for tracked npm installs, with shipped docs/changelog coverage and regression tests. This PR is superseded by the implementation on `main`.

## Evidence

- **CLI routes update args through npm-spec selection:** `runPluginUpdateCommand` resolves plugin update targets with `resolvePluginUpdateSelection(...)` and forwards `specOverrides` into `updateNpmInstalledPlugins(...)`, so `plugins update` already accepts spec-based targeting on main.
  - file: [src/cli/plugins-update-command.ts:22](https://github.com/openclaw/openclaw/blob/627c19c5cbbe8856a11effe1871c442ac87d56c9/src/cli/plugins-update-command.ts#L22)
  - command: `nl -ba src/cli/plugins-update-command.ts | sed -n '1,120p'`
  - sha: [627c19c5cbbe](https://github.com/openclaw/openclaw/commit/627c19c5cbbe8856a11effe1871c442ac87d56c9)
- **Selection helper maps `<npm-spec>` back to tracked plugin ids:** `resolvePluginUpdateSelection` parses a raw npm spec, matches it against tracked install package names, and returns the tracked plugin id plus a `specOverrides` entry. This covers unscoped, scoped, dist-tag, and exact-version inputs.
  - file: [src/cli/plugins-update-selection.ts:9](https://github.com/openclaw/openclaw/blob/627c19c5cbbe8856a11effe1871c442ac87d56c9/src/cli/plugins-update-selection.ts#L9)
