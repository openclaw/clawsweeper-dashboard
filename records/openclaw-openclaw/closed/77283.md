## Summary

Keep open: current main still leaves the Discord model picker component path without an early deferred acknowledgement while it performs async routing, model catalog, and recent-preference work before the first component response. The reporter’s 2026.5.3 failure matches the current source path, and no merged main change in the affected files fixes it.

## What This Changes

Request to make Discord `/model` picker component selections acknowledge promptly instead of failing with Discord’s interaction timeout after the 2026.5.3 upgrade.

## Best Possible Solution

Acknowledge Discord model picker component interactions before route/model/preference I/O, then refresh the picker by editing the original response through the existing reply/edit semantics with regression coverage.

## Reproduction Assessment

Yes, source-reproducible: current main dispatches the picker fallback components with `defer=false`, then the handler awaits route, model-data, and preference work before the first update. I did not mark this live reproduced because I did not run a Discord bot session.

## Solution Assessment

Yes, the requested early acknowledgement is the right direction, but the one-line `deferUpdate` suggestion needs adaptation to this wrapper. After `DeferredMessageUpdate`, the picker should edit the original response via `reply()`/`editReply()` semantics rather than call `interaction.update()` as a second callback.
