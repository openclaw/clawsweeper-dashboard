## Summary

Current `main` already replaces the old numeric-only announce-target parsing with channel-aware session parsing, and the Feishu parser plus regression tests cover non-numeric `topicId` values. The same implementation is present in the tagged release `v2026.4.22`, so this bug is already fixed and shipped.

## Evidence

- **Announce target resolution no longer uses numeric regex parsing:** `resolveAnnounceTargetFromKey()` now delegates to `resolveSessionConversationRef(sessionKey)` and returns `parsed.threadId`; the old `:topic:(\\d+)` / `:thread:(\\d+)` extraction path is gone.
