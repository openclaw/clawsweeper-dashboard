## Summary

Keep open. The TUI startup-summary idea is useful, but the current PR still cannot preserve selected-agent scope for Gateway `global` sessions, the latest-head real behavior proof is insufficient, and the branch carries unrelated support/script test drift.

## What This Changes

Adds a TUI startup summary fetched on initial non-local connection, extends TUI backends with `describeSession`, and includes adjacent support/script test stabilizations.

## Best Possible Solution

Land a narrow TUI/Gateway implementation where `sessions.describe` preserves selected-agent scope, global/per-agent cases have focused coverage, unrelated drift is removed, and latest-head real Gateway TUI proof is attached.

## Reproduction Assessment

Yes from source, but not from a live run: a non-local TUI using `global` with a non-default `currentAgentId` passes `agentId` into `describeSession`, while the Gateway client and handler drop that scope and resolve through the default global store.

## Solution Assessment

No. The current implementation is a plausible start, but the best fix is to make the Gateway `sessions.describe` contract explicitly agent-scoped, or avoid global summaries until that contract exists.
