## Summary

Keep open. Current main includes the narrower agents-list JSON startup fix from related #71739, but this issue’s core tasks/status behavior is still only partially addressed: tasks list/audit have no route-first lightweight JSON path and status --json still explicitly disables channelSummary.

## Best Possible Solution

Keep this issue open. The best fix is to add route-first lightweight handling for `openclaw tasks list --json` and `openclaw tasks audit --json`, or split the tasks registrar/modules so these JSON paths do not import status, health, sessions, TaskFlow audit, and maintenance work unnecessarily. Separately, implement a lightweight `status --json` channelSummary path that preserves the documented/status JSON surface without full plugin/runtime preload, with regression coverage for the tasks JSON load path and configured channel-summary behavior.
