## Summary

Keep open. The prior ClawSweeper finding still matches current main: the incomplete-turn classifier has no structured accepted-`sessions_spawn` success fact to distinguish a pure relay from a generic zero-payload stop, so the false warning can still fire.

## What This Changes

Treat a zero-payload stop after a successful `sessions_spawn` child-run acceptance as a completed relay turn instead of an incomplete response.

## Best Possible Solution

Preserve structured accepted-`sessions_spawn` child-run facts through tool completion into attempt metadata, then suppress the incomplete-turn warning only for that proven terminal-success case.

## Reproduction Assessment

Yes, at source level. A current-main attempt with `payloadCount: 0`, no assistant text, `stopReason: "stop"`, no `lastToolError`, and a successful accepted `sessions_spawn` still reaches the generic empty-response warning because accepted child-run fields are not available to the classifier.

## Solution Assessment

Yes. The narrow maintainable solution is to carry accepted child-run metadata and special-case only `status: "accepted"` with non-empty `runId` and `childSessionKey`, rather than adding a broad config bypass or treating every tool call as a completed response.
