## Summary

Keep open. The issue is paired with open closing PR #77240, and source inspection still shows the providerless nested OpenRouter resolver case is not fully covered on current main before that PR lands or is rejected.

## What This Changes

Allow cron model overrides whose stored model id lacks the provider prefix to match provider-prefixed entries in `agents.defaults.models`, especially nested OpenRouter IDs.

## Best Possible Solution

Review and land or reject #77240 with a focused regression for a nested OpenRouter providerless raw model against a provider-prefixed configured allowlist.

## Reproduction Assessment

Not end to end. Source inspection gives a clear boundary reproduction once the cron model reaches the resolver as `arcee-ai/trinity-large-preview:free`, but the current-main path that strips `openrouter/` still needs a stored job or run log.

## Solution Assessment

Yes, the narrow solution is to patch shared model resolution after the initial allowlist miss and cover it with a resolver regression; broad cron storage migration should wait unless the stripping boundary is confirmed.
