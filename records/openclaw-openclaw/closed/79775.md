## Summary

Close as implemented: this was reported against 2026.4.29, and current main plus released 2026.5.3+ CLI now requests admin scope for admin or inherited-admin device approvals while preserving the Gateway caller-scope guard.

## What This Changes

Report requests that a CLI device showing operator.admin be able to approve a pending browser or Control UI pairing request behind a Synology DSM reverse proxy.

## Best Possible Solution

Keep the current scoped CLI approval behavior, and treat any recurrence on v2026.5.3 or newer as a new token-drift or reverse-proxy reproduction with redacted diagnostics.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No current-main failing reproduction: the historical 2026.4.29/5.2 path is source-backed by the fix diff and related report, and current main has regression coverage for requesting admin scope during admin approvals.

## Solution Assessment

Is this the best way to solve the issue? Yes: the shipped fix asks for admin only when the exact pending request would mint or inherit admin, while leaving the Gateway's fail-closed caller-scope boundary intact.
