Keep open. Current main still exposes the requested bootstrap/profile knobs only as global defaults, while the narrower related reports were closed back to this issue as the canonical per-agent bootstrap/profile API decision.

## What This Changes

Request per-agent bootstrap profile support for `contextInjection`, `bootstrapMaxChars`, and `bootstrapTotalMaxChars` instead of only global `agents.defaults` and Control UI presets.

## Best Possible Solution

Define one additive public contract for effective per-agent bootstrap/profile settings, then wire it through types, schema, runtime resolution, config patching, Control UI, docs, metadata, and focused tests.

## Reproduction Assessment

Not applicable as a feature/config request. Source inspection gives a high-confidence verification path for the gap: current main keeps the requested knobs under `agents.defaults`, not per-agent schema or runtime resolution.

## Solution Assessment

Unclear until maintainers choose the API.
