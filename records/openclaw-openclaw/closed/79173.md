## Summary

The PR addresses a valid current-main bug: disabled skill configs are skipped during fresh skill discovery and runtime secret collection, but persisted snapshot env override application can still reach disabled `apiKey` SecretRefs. The patch is narrow, aligned with the existing contract, and has supplied terminal proof, so it should stay open for maintainer validation rather than be closed or replaced.

## What This Changes

The PR adds `enabled:false` guards before applying skill env overrides from live entries or persisted skill snapshots, plus a regression test and changelog entry.

## Best Possible Solution

Land this focused guard, or an equivalent replacement, so disabled skill entries stay inert in both fresh and persisted snapshot env override paths.

## Reproduction Assessment

Yes. Source inspection shows a persisted snapshot skill whose current config has `enabled:false` can still reach `applySkillConfigEnvOverrides` on current main, while that disabled skill's raw `apiKey` SecretRef is allowed to remain unresolved elsewhere.

## Solution Assessment

Yes. The PR fixes the shared env-injection boundary instead of adding channel-specific behavior or a migration, and the regression test targets the stale persisted snapshot case that fresh discovery cannot filter.
