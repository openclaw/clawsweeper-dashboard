## Summary

Keep this PR open. Current main still does not implement the PR's core interrupt scheduling race fix: there is no force-detach path, no retry/force-supersede path for lingering ReplyOperation state, and no two-phase wait after an embedded run clears. The PR is also a dirty/large branch and needs rebase/rescope before maintainer review; the newer maintainer PR #73365 covers the adjacent steer-gate work but explicitly leaves this interrupt scheduling race as follow-up.

## Best Possible Solution

Keep the work open but ask for a narrow rebase or maintainer replacement focused only on the core interrupt scheduling race: two-phase embedded/reply wait, a conservative force-detach/recovery path, and identity-safe ReplyOperation supersede tests. Split or drop the unrelated release/docs/generated/Discord/QA/script churn, and keep #65010/#73365 and #65222 as adjacent but separate steer/Telegram delivery work.
