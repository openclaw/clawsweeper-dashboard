## Summary

Current main has moved the relevant bundled-plugin startup path away from the source/Jiti cold path that produced `register/babel.cjs`, and channel/account startup is now bounded; the linked #76100 context points to the same staging/startup race and is already closed.

## What This Changes

Avoid Telegram startup `register/babel.cjs` cold-start errors when multiple accounts start against a fresh bundled dependency tree.

## Best Possible Solution

Keep the current built-artifact bundled-plugin startup path and channel fanout guard; if the log recurs after the next release, reopen with fresh packaged multi-account logs.

## Reproduction Assessment

Yes. The issue provides concrete fresh-dist, multi-account startup steps, and current-main source/test checks cover the relevant bundled-plugin and channel startup path; I did not run a live Telegram gateway.

## Solution Assessment

Yes. Avoiding the shared source/Jiti cold path is narrower than serializing all Telegram account startup, while the channel fanout cap reduces the remaining startup burst without changing Telegram semantics.
