## Summary

Current main still does not forward Discord reactions into plugin `message_received` handlers, but the same remaining hook/plugin-surface gap is already tracked by open issue #38714. This item can close as a duplicate of that canonical open issue.

## Evidence

- **Discord reaction listeners are separate from message listener:** `registerDiscordMonitorListeners` registers the normal `DiscordMessageListener` first, then separate add/remove reaction listeners, so reactions do not enter the message listener by registration.
  - file: [extensions/discord/src/monitor/provider.startup.ts:257](https://github.com/openclaw/openclaw/blob/8e12c24d17218dc50023aaf5543df1b7c36520aa/extensions/discord/src/monitor/provider.startup.ts#L257)

  - file: [extensions/discord/src/monitor/listeners.ts:631](https://github.com/openclaw/openclaw/blob/8e12c24d17218dc50023aaf5543df1b7c36520aa/extensions/discord/src/monitor/listeners.ts#L631)
