Full review comments:

- **[P1] Export the new command-index helper before importing it:** `extensions/telegram/src/bot-native-commands.ts:12`
  - body: `resolveValidatedCustomCommandIndices` is imported from `openclaw/plugin-sdk/config-runtime`, but the PR only adds the helper to the old config module and does not add it to that public barrel. On the shown base, `config-runtime` exports only the existing Telegram command helpers, so this import is unresolved and the Telegram build is not merge-ready.
  - confidence: 0.91
- **[P1] Recheck custom-command auth on menu callback fallback:** `extensions/telegram/src/bot-handlers.runtime.ts:1583-1585`
  - body: When a custom menu button is not in `routes`, this branch forwards its `callback_data` to the normal callback-to-agent path after only generic callback authorization. In shared groups, a user who was not allowed to invoke the restricted custom command can still press a menu posted by someone else and drive that command's fallback action.
  - confidence: 0.88
- **[P2] Validate custom-menu callback payloads during config parsing:** `src/config/zod-schema.providers-core.ts:130`
  - body: The new schema accepts any string for `menus.buttons[*].callback_data`, then sends those buttons directly to Telegram. Existing Telegram button construction sanitizes callback payloads against the Bot API limit, so malformed config should be rejected or normalized before a menu command turns into a runtime API failure.
  - confidence: 0.84
