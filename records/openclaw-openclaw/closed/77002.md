## Summary

Current main already implements the timeout path needed for this report: provider-scoped Ollama timeouts are converted into model request timeout metadata and forwarded into the guarded native Ollama fetch, including Undici body timeout handling.

## What This Changes

The issue asks for slow Ollama/local model runs to honor configured timeouts instead of aborting with `Body Timeout Error`.

## Best Possible Solution

Keep the current provider-scoped timeout implementation and guide affected users to `models.providers.ollama.timeoutSeconds` until the next release includes the main-branch fix.

## Reproduction Assessment

No live slow-model reproduction was run. The issue has clear logs and steps, and the current source/tests provide a high-confidence source-level path for the reported Ollama body timeout behavior.

## Solution Assessment

Yes. Forwarding provider-scoped timeout settings into the Ollama HTTP fetch is the narrowest maintainable fix because the failure is a provider request/body timeout, not the whole agent runtime timeout or legacy `agents.defaults.llm` config.
