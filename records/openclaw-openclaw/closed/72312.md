## Summary

Current main and the shipped v2026.4.24 tag already use the valid fal endpoint hosts. The bundled fal image provider posts to `https://fal.run/{model}`, the video provider maps queued jobs to `https://queue.fal.run/{model}`, tests assert those URLs, and the fal plugin has no `queue.fal.ai` or `api.fal.ai` references. Local DNS/reachability checks also confirm the obsolete `queue.fal.ai` host is NXDOMAIN while the hosts used by OpenClaw resolve.

## Best Possible Solution

Close this DNS outage issue as already handled by the current bundled fal provider. Keep the existing `fal.run`/`queue.fal.run` implementation; if maintainers want a broader fal image model catalog, track that separately as a catalog enhancement rather than this broken-endpoint bug.
