## Summary

Current main already implements opt-in session transcript indexing, and the reported health data showing only the memory source matches a configuration that has not enabled session indexing.

## What This Changes

The issue reports that memory_search with corpus=sessions returns no results because session transcript JSONL files are not indexed.

## Best Possible Solution

Keep session transcript indexing opt-in for storage, cost, and privacy, and direct users to enable sessionMemory plus the sessions source before expecting corpus=sessions hits.

## Reproduction Assessment

No for the documented configured behavior: current source and tests show session transcript hits work when experimental.sessionMemory is enabled and sources includes sessions. The reported index-health output showing only memory indicates the sessions source was not enabled or indexed.

## Solution Assessment

Yes; the maintainable solution is the existing opt-in indexing path because session transcripts can be large and sensitive. The supported fix for this setup is configuration plus reindexing, not adding sessions to the default corpus.
