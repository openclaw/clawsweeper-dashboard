## Summary

Keep open. Current main still has bundle-configured stdio LSP startup, hover/definition/references tools, plugin inspect server listing, and generic gateway diagnostics/log views, but it does not collect LSP publishDiagnostics notifications or expose LSP server health, latency, history, smart repair, or an editor diagnostics dashboard.

## What This Changes

The issue requests an in-editor real-time LSP diagnostics dashboard with server health, diagnostic feeds, troubleshooting suggestions, history, and reporting.

## Best Possible Solution

Define a versioned LSP lifecycle and diagnostics event contract with clear plugin/core ownership and trust boundaries, then expose scoped status and diagnostic views before adding smart repair or CI integrations.

## Reproduction Assessment

Not applicable; this is a feature proposal rather than a bug report. Source, docs, UI, and related-item checks show the requested LSP diagnostics dashboard is not implemented on current main.

## Solution Assessment

No; the all-in-one dashboard and smart-repair suite is not the narrowest maintainable first step. A focused LSP diagnostics/lifecycle contract with scoped UI or plugin-served views is the safer direction.
