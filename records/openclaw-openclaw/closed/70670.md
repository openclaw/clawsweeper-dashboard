## Summary

Current main already pins workspace bootstrap context files by session key, so edits to injected files such as AGENTS.md, SOUL.md, USER.md, and MEMORY.md do not cause those files to be reloaded into the same active session/turn. The snapshot is intentionally cleared on reset/session rollover so the next session can pick up updated workspace context.

## Best Possible Solution

Keep the shipped session-scoped bootstrap snapshot behavior: workspace context files should reload on explicit reset/session rollover, not immediately after in-session edits. If a future report still sees looping, it should be opened as a narrower bug with the exact session key mode, context engine or hook involved, model/provider logs, and the specific tool sequence because the core bootstrap reload path is already guarded.
