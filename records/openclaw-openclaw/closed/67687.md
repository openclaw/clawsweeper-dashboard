## Summary

Keep open. Current main still separates Feishu group admission from mention gating: the reporter's config shape can preserve `groups.<chat_id>.requireMention=false`, but validated Feishu config defaults `groupPolicy` to `allowlist`, inbound group handling checks `groupAllowFrom` first, and an empty allowlist returns before the per-group `requireMention` override is evaluated.

## Best Possible Solution

Keep this issue open. The conservative fix is to either treat explicit `channels.feishu.groups.<chat_id>` entries as admitted groups under `groupPolicy: "allowlist"`, or keep the current admission model but add a targeted doctor/config warning and clearer Feishu docs that `groups.<chat_id>.requireMention=false` only controls mention gating and still requires `groupAllowFrom` or `groupPolicy: "open"`. Add a regression test that validates the reporter's config shape through plugin-aware config validation and verifies whether a Feishu group text event reaches route/session dispatch.
