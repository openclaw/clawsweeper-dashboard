## Summary

Current main already covers the central failure mode with a durable pending/suspended subagent delivery record and an agent-steering queue that injects preserved results into a later parent turn, so a separate file-based fallback issue is no longer needed.

## What This Changes

The issue requests a durable file-backed fallback for subagent completion announcements when direct delivery and queue delivery both fail.

## Best Possible Solution

Keep the current main implementation: durable subagent delivery should remain in OpenClaw-managed runtime state and the parent-turn steering queue, not a new JSON sidecar fallback.

## Reproduction Assessment

Yes. Source inspection shows the exact direct-primary plus steer-fallback failure path, retry-limit handling, and the current durable recovery path that preserves and re-injects pending/suspended completion payloads.

## Solution Assessment

Yes. The current state-backed agent-steering queue is the better OpenClaw fix than the proposed file sidecar because repository policy keeps runtime state in canonical stores rather than JSON fallback files.
