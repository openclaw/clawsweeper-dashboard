## Summary

Current main already covers the substance of this request: `allow-always` approvals are persisted and reused, broad approvals are stored as executable-path allowlist entries rather than exact shell text when safe, exact-command durable approvals exist as a fallback, unattended approval timeouts can fall back to allowlist/full behavior, and OpenClaw now ships a simple preset CLI for common exec-security postures.

## Risks / Open Questions

- Preset names and scope differ from the exact proposal: current main uses local `exec-policy preset yolo|cautious|deny-all`, not `approvals set-profile developer|automation|strict|learning`.
- Remote gateway/node preset syncing still uses `openclaw approvals set --gateway/--node ...`; the one-step preset helper is local-only.
