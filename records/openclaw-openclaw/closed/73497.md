## Summary

Current main already fixes the staged-wrapper self-import failure by preserving the dist-runtime mirror depth and separately mirroring the canonical dist implementation tree. The PR's regeneration pass is therefore redundant for the reported Docker/non-root symptom.

## Best Possible Solution

Close this PR as implemented by the current main staged-runtime mirror behavior. Keep the main implementation that stages dist-runtime wrappers under installRoot/dist-runtime and mirrors canonical dist beside it; reopen or file a new issue only if a current-main Docker/non-root repro still shows the bundled-channel-entry contract warning.
