## Summary

Current main already solves the active transcript growth problem through the maintainer-owned compaction byte guard and successor transcript rotation path. This PR's rolling-window rewrite is now a competing stale implementation with unresolved correctness review concerns and an unrelated package-export test change.

## Best Possible Solution

Close this PR and keep the current mainline design: configurable active transcript byte guarding plus normal semantic compaction and successor transcript rotation. If maintainers later want raw rolling-window pruning independent of compaction, it should be reopened as a fresh design issue against the current rotation architecture and review concerns, not merged from this stale branch.
