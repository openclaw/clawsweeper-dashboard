The issue asks for Telegram `/status` to show the actual custom-provider auth source from `models.json` after a custom provider is selected.

## Best Possible Solution

Make `/status` and `session_status` use the same selected-provider auth-label source for target sessions, then add regression coverage for native Telegram `/status` with a selected custom provider and an unrelated built-in env key.
