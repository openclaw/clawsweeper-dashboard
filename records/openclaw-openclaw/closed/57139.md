## Summary

This should close as a duplicate of the open canonical archive-recovery thread #56131, with #45003 tracking the broader reset-archive cleanup area. Current main still does not implement archive listing or archive selection in `sessions_history`; the value in this report is preserved by keeping the canonical issue open.

## Best Possible Solution

Keep #56131 as the canonical product/API thread for making reset archives discoverable and readable through supported session tools, and use #45003 for the broader reset-archive cleanup plan. Review #60409 only as a partial fallback, not as the complete solution for this request.

## Evidence

- **Canonical duplicate is open:** #56131 is open and tracks the same remaining product gap: reset archives are not discoverable through `sessions_history`/`sessions_list`, so agents cannot recover reset context through supported tools. #45003 also lists this issue and #56131 as part of the broader reset-archive access gap.

  - file: [src/gateway/session-transcript-files.fs.ts:70](https://github.com/openclaw/openclaw/blob/f256eeba431b5f59e6085b7c11c9c0a309e408b7/src/gateway/session-transcript-files.fs.ts#L70)
