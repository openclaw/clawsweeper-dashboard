## Summary

Keep open. Current `main` still gates WhatsApp read receipts on the account-level `isSelfChat` flag, so when the bot's own number is present in `allowFrom`, allowed inbound messages from other people can still skip blue-tick read receipts. The PR addresses behavior that is not yet implemented on `main`.

## Evidence

- **Current read-receipt gate still uses account-wide self-chat mode:** `maybeMarkInboundAsRead` only calls `sock.readMessages()` when `!access.isSelfChat`; otherwise it logs `Self-chat mode: skipping read receipt`. That means the gate is still tied to the account policy, not to whether the specific inbound message is `fromMe`.

That is broader than 'this message was sent by me'.
