## Summary

Keep open. Current `main` still stores Slack HTTP routes in a module-local `Map` in `extensions/slack/src/http/registry.ts`, while registration and request dispatch are reached from different Slack entry surfaces. The codebase already treats split-chunk mutable state as a real risk elsewhere via `globalThis[Symbol.for(...)]` stores and duplicate-module tests, but the Slack HTTP registry has not been migrated to that pattern and has no equivalent cross-module coverage.
