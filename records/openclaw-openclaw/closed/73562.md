## Summary

Keep open: current main still awaits the shared readline confirmation directly, and the active closing PR is unmerged, so the CLI crash remains until that PR or an equivalent fix lands.

## What This Changes

Make `openclaw plugins uninstall <id>` fail cleanly when confirmation stdin closes, while preserving `--force` and piped `y`/`n` confirmations.

## Best Possible Solution

Land PR #73566, or an equivalent narrow fix, so the shared prompt helper settles on EOF/close and plugin uninstall reports actionable interactive-input or `--force` guidance.

## Reproduction Assessment

Yes. The current source path still reaches the direct readline await, and a local Node readline/promises probe with stdin closed reproduces exit 13 with the unsettled top-level-await warning; I did not mutate local plugin config to run the full uninstall command.

## Solution Assessment

Yes. An EOF-aware prompt primitive plus uninstall-specific handling is the narrow maintainable fix; a blanket non-TTY gate would be broader and would break valid piped confirmations.
