## Summary

Keep this issue open. Current main still allows `openclaw cron add --message ...` without `--agent`, forwards `agentId: undefined`, and later resolves the run through the default-agent path. The provided context shows open PR #42245 is an unmerged implementation candidate with closing syntax for this issue, so the issue should stay open until that PR or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review/refresh PR #42245, or land an equivalent focused fix. The implementation should either require `--agent` or warn on stderr for agent-turn `openclaw cron add` jobs that omit it, keep system-event jobs quiet, preserve explicit `--agent` and intentional session routing behavior, and add regression coverage in `src/cli/cron-cli.test.ts`.
