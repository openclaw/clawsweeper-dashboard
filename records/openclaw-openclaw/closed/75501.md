## Summary

Keep open. Current main and the latest release still pass whole skill roots to Chokidar, and the linked open pull request is an implementation candidate rather than a merged fix.

## What This Changes

The issue reports a macOS regression where the skills watcher can exhaust file descriptors by recursively watching very large workspace skill trees.

## Best Possible Solution

Keep this issue open while maintainers revise, land, or replace the linked pull request with a watcher that mirrors bounded skill discovery or adds a documented ignore mechanism with regression coverage.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection gives a plausible current-main path when large trees are under a watched skills root or configured `extraDirs`, because Chokidar recursively watches directory targets and the skills watcher still uses those targets.

## Solution Assessment

No, current main is not enough and the linked pull request at https://github.com/openclaw/openclaw/pull/75518 is not merged. The safer fix is to bound watcher targets or depth to the supported skill layout, or add a documented ignore setting if maintainers choose that direction.
