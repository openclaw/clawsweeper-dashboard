## Summary

Current `main` does not show the reported route-matching mismatch: Signal group IDs are passed into routing without lowercasing, binding peer IDs are normalized by trimming only, and session-key lowercasing happens later in persistence key construction rather than before binding resolution.

## Evidence

- **Signal inbound routing preserves raw group ID for binding resolution:** `resolveSignalInboundRoute()` passes `groupId` directly as `peer.id` when routing Signal group messages, and the caller feeds it from `dataMessage.groupInfo?.groupId` without lowercasing.
  - file: [extensions/signal/src/monitor/event-handler.ts:88](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/extensions/signal/src/monitor/event-handler.ts#L88)
  - command: `nl -ba extensions/signal/src/monitor/event-handler.ts | sed -n '80,220p'`
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Route binding peer IDs are not lowercased before match:** `normalizeRouteBindingId()` trims strings and numbers only, and `matchesBindingScope()` compares `scope.peer.id` to `match.peer.id` directly. `resolveAgentRoute()` also normalizes inbound peer IDs with the same trim-only helper.
  - file: [src/routing/resolve-route.ts:483](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/routing/resolve-route.ts#L483)
  - command: `nl -ba src/routing/resolve-route.ts | sed -n '480,620p'`
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Session-key lowercasing is a later persistence step:** For non-direct peers, `buildAgentPeerSessionKey()` lowercases `channel` and `peerId` only when constructing the stored session key; this happens after route selection, so it does not affect binding lookup.
  - file: [src/routing/session-key.ts:174](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/routing/session-key.ts#L174)
  - command: `nl -ba src/routing/session-key.ts | sed -n '121,210p'`
  - sha: [028b6c9b13c9](https://github.com/openclaw/openclaw/commit/028b6c9b13c960b6f59a7643aff4f6de04e07f71)
- **Signal target helpers also preserve group ID case:** Signal normalization keeps `group:<id>` payloads with the original group ID text instead of lowercasing them, which is consistent with the inbound routing path.
  - file: [extensions/signal/src/normalize.ts:15](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/extensions/signal/src/normalize.ts#L15)
  - command: `nl -ba extensions/signal/src/normalize.ts | sed -n '1,260p'`
