## Summary

Keep open. Current main still has the packaging gap: builtin memory reaches a dynamic `sqlite-vec` import, but `sqlite-vec` is only a root dependency and is absent from the root runtime-dependency mirror list used for packaged bundled-plugin snapshots. The linked #74711 PR is an open implementation candidate with closing syntax, so this issue should remain open until that PR is reviewed, revised, or merged.

## What This Changes

Ensure packaged builtin memory runtime snapshots include `sqlite-vec` so vector memory can initialize and search after an OpenClaw upgrade.

## Best Possible Solution

Review and land or revise #74711 so packaged runtime-deps stage `sqlite-vec` for builtin memory, add regression coverage for the memory-core runtime plan, and either fix or explicitly track the `extensionPath` short-circuit behavior raised in the discussion.
