## Summary

Current `main` already covers this startup-activation bug: the bundled Codex plugin declares `activation.onAgentHarnesses = ["codex"]`, gateway startup explicitly includes required agent-harness owner plugins for configured runtimes, and current tests cover default-runtime, per-agent, and env-forced Codex activation. Legacy `codex/*` model refs are also normalized on config read into canonical provider refs plus `embeddedHarness.runtime = "codex"`, so the model-ref path now feeds the same startup activation logic.
