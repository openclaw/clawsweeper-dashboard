## Summary

Keep open: the refreshed PR is on the right installer surface and has sufficient Crabbox E2E proof, but it still auto-selects older npm fallback candidates without checking npm deprecation metadata. ## Best Possible Solution

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Skip deprecated compatible npm fallbacks:** `src/plugins/install.ts:457-467`
  - body: When `latest` is incompatible, this fallback returns the first older stable version whose OpenClaw compatibility metadata passes. The candidate list comes from `npm view <pkg> versions --json`, and `resolveNpmSpecMetadata` does not request or expose npm's `deprecated` field, so an unpinned install can automatically choose an older release that package maintainers deprecated for a security or breakage reason. Please include deprecation metadata in candidate inspection and continue to the next older compatible version instead of selecting a deprecated fallback.
  - confidence: 0.88
