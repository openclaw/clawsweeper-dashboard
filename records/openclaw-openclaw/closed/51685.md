## Summary

Current `main` already contains a targeted fix for stale `chat.history` results during session switching in Control UI, plus a regression test that exercises switching from one session to another and verifies the old response is ignored.

## Evidence

- **Session switch path reloads chat history through the shared helper:** `switchChatSession` resets chat state for the new session and immediately calls `loadChatHistory`, which is the dropdown-switch path reported in the issue.
  - file: [ui/src/ui/app-render.helpers.ts:509](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/ui/src/ui/app-render.helpers.ts#L509)
