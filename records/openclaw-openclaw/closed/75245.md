## Summary

Keep this issue open: current main and the latest release still contain the shallow-copy `registerHook` wrapper, but https://github.com/openclaw/openclaw/pull/97281 is an open clean fix candidate with closing syntax for this issue.

## What This Changes

The issue asks OpenClaw to preserve mutable `event.context` changes made through plugin `api.registerHook()`, especially `agent:bootstrap` `bootstrapFiles` replacements before prompt context is built.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/97281 after maintainer acceptance of the mutable hook-context contract, preserving per-handler `pluginConfig` cleanup.

## Reproduction Assessment

Yes. Source inspection shows the plugin wrapper gives handlers a cloned context while the bootstrap hook consumer reads the original event context after dispatch.

## Solution Assessment

Yes. The focused registry-wrapper repair in https://github.com/openclaw/openclaw/pull/97281 is better than a bootstrap-only workaround because the bug sits at the plugin hook API boundary.
