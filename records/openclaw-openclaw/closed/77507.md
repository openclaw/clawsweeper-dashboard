## Summary

Keep open. The read-through direction addresses a real source-reproducible gap, but this PR still lacks after-fix real behavior proof and has blocking parser/path-safety issues before it is mergeable.

## What This Changes

Adds a Claude CLI transcript read-through resolver and threads it through CLI hook history, gateway session history, and the sessions_list transcriptPath projection.

## Best Possible Solution

Revise this PR to use one validated Claude transcript resolver/parser for read-through, keep archive and maintenance paths on the OpenClaw store, and leave the separate writer-stall investigation tracked by https://github.com/openclaw/openclaw/issues/78273.

## Reproduction Assessment

Yes, source-reproducible: current main reads sessions_list and HTTP history from OpenClaw transcript candidates only, while Claude CLI transcripts are parsed separately under the Claude project store. I did not run a live claude-cli install in this read-only review.

## Solution Assessment

No. Read-through is a reasonable direction, but this branch should reuse the existing Claude JSONL parser and add fail-closed session-id plus realpath validation before merge.
