## Summary

Keep open: current main still maps normal Active Memory no-summary/`NONE` recall and terminal zero-hit search paths to `status=empty`, and several open PRs already reference this issue with closing syntax but have not merged.

## What This Changes

The issue asks Active Memory to stop showing normal no-useful-memory recall outcomes as user-visible `status=empty`.

## Best Possible Solution

Land one canonical Active Memory fix, likely https://github.com/openclaw/openclaw/pull/80015 or a replacement, that classifies `NONE` as successful no-relevant-memory, avoids caching that result, separates unavailable/failed/timeout paths, and updates tests/docs.

## Reproduction Assessment

Yes, from source inspection: make the embedded recall subagent return `NONE` and current main persists an Active Memory status line containing `status=empty`. Current tests assert that behavior, though I did not live-run the reported Telegram setup.

## Solution Assessment

Yes, the requested direction is the narrow maintainable fix: keep hidden no-injection behavior unchanged while using a distinct successful no-relevant-memory status and reserving failure-like statuses for real unavailable, failed, or timeout paths.
