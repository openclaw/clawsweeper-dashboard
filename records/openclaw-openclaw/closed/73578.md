## Summary

Keep this issue open. Current main still renders the status Context numerator from stored/session usage without a proven current-working-set snapshot for the claude-cli path, and the relevant status/session usage files are unchanged since the prior Codex review comment. The related stale-zero status fixes do not cover this cumulative-versus-window case.

## Best Possible Solution

Add a CLI/status contract that carries a current working-set prompt snapshot separately from provider/session usage, such as agentMeta.promptTokens or agentMeta.lastCallUsage for CLI runners. Stop treating raw claude-cli usage as a context snapshot unless that backend explicitly guarantees it, render Context from the fresh snapshot or unknown, and add a claude-cli /status or session_status regression where cumulative usage exceeds the context window but the latest prompt snapshot does not.
