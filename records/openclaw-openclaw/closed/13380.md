## Close Comment

Thanks for the write-up. This Codex review is closing the issue because the proposal fits ClawHub/community-plugin scope rather than OpenClaw core.

Current project direction is explicit that core stays lean, optional capability should usually ship as plugins, and new skills should be published to ClawHub first instead of being added to core by default (`VISION.md:54`, `VISION.md:66`). The repo also documents community-owned extensions as third-party plugins/skills published on ClawHub or npm (`docs/plugins/community.md:9`, `docs/tools/clawhub.md:10`).

This issue is also framed as a partnership pitch for a separate repository/product, not a concrete OpenClaw core change to implement or track. If you want to integrate Water with OpenClaw, the maintainable path is an external plugin or skill published through ClawHub rather than an issue against the core repo.
