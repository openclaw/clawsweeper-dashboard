## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Split provider size failures instead of capping at 2000 items:** `extensions/memory-core/src/memory/manager-embedding-ops.ts:54`
  - body: The reported failure is a provider batch-size/length rejection, but this cap only limits the number of inputs. A batch with two large chunks can still exceed a 2000 byte/token combined limit, while 2000 inputs can exceed providers with lower item caps. Please use adaptive splitting or a real provider size limit instead of this fixed item-count cap.
  - confidence: 0.9
- **[P2] Accept the actual Responses text format shape:** `src/gateway/open-responses.schema.ts:223-225`
  - body: For Responses API structured output, `text.format` carries `type`, `name`, `schema`, and `strict` on the format object itself. This nested `json_schema` object still rejects SDK payloads that include `schema` or `strict`, so the claimed SDK parity is not achieved. ([platform.openai.com](https://platform.openai.com/docs/api-reference/responses/compact?api-mode=responses&utm_source=openai))
  - confidence: 0.88
