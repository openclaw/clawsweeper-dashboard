## Summary

Close as duplicate/superseded: the zh-CN Windows Hub WSL preflight failure is the same localized `wsl.exe --version` parser bug now tracked by the canonical OpenClaw release-side issue and the upstream Windows Hub issue, with the parser fix released in Windows Hub v0.6.3 and the OpenClaw docs pointing users at fixed v2026.6.5 installer assets.

## What This Changes

Windows Hub setup should parse localized and NUL-interleaved `wsl.exe --version` output during the `preflight-wsl` check on zh-CN Windows.

## Best Possible Solution

Use the fixed Windows Hub v0.6.3 or newer assets and keep the remaining release drift/prevention discussion on the canonical OpenClaw and upstream Windows Hub issues rather than tracking this duplicate separately.

## Reproduction Assessment

Yes, from source and reporter logs: the shipped v0.6.0 parser only matched English `WSL version:` while the logs contain localized zh-CN/NUL-interleaved WSL output that reaches the same terminal failure. I did not rerun the Windows installer in this review.

## Solution Assessment
