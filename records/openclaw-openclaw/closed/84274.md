- `P2`: This is a normal-priority gateway bug fix with limited blast radius but real session-state impact when abort misses an active send.

## Summary

Keep open: the patch targets a real source-level `chat.abort` race and the implementation shape looks sound, but this external PR only provides Vitest/mock-harness proof and the Real behavior proof check failed, so it needs real gateway/UI evidence or a maintainer proof override before merge.

## What This Changes

Moves `chat.send` abort-controller registration before attachment/model preparation, cleans up pre-dispatch abort paths, and updates gateway chat tests for duplicate sends and abort-during-attachment preparation.

## Best Possible Solution

Land this shape after real OpenClaw proof confirms immediate abort of an image-backed `chat.send` returns `aborted: true` and does not dispatch the run; otherwise use a maintainer proof override if the focused harness is accepted.

## Reproduction Assessment

Yes at source level: current main and `v2026.5.18` await attachment preparation before inserting the run into `chatAbortControllers`, while `chat.abort` returns `aborted:false` when that entry is missing. I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

Yes: registering the chat-send abort controller after validation/policy but before awaited attachment/model preparation is the narrow maintainable fix and mirrors the earlier agent-run registration pattern. The missing piece is real behavior proof, not a different code direction.
