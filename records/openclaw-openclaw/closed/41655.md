## Summary

This PR is superseded by merged maintainer PR #72308, which replaced the older depth-cap approach with the current watcher repair on main. The original depth-2 memory proposal also had review feedback about silently missing deep memory updates, while current main has moved the memory watcher to `extensions/memory-core` and records the #41606 fix in Unreleased changelog.

## Best Possible Solution

Keep the merged #72308 watcher repair on main as the canonical implementation. If maintainers want additional FD-hardening, it should be handled as a focused follow-up against the current `src/agents/skills/refresh.ts` and `extensions/memory-core/src/memory/manager-sync-ops.ts` implementations, preserving deep memory coverage.
