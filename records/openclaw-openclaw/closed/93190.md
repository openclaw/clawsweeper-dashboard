## Summary

Keep this PR open, but it is not merge-ready yet. The follow-up commit fixed the log-sanitization issue, but the warning-only implementation now drops blocked env keys silently for already-saved or hand-edited configs because those records never pass through the new config-write warning path.

## What This Changes

The PR exports the MCP stdio env blocklist helper, removes per-resolution runtime warnings for dropped blocked env keys, adds a sanitized config-write-time warning in `setConfiguredMcpServer`, and updates focused tests.

## Best Possible Solution

Either extend this warning-only approach with one-time legacy diagnostics plus matching docs, or select https://github.com/openclaw/openclaw/pull/92484 as the canonical validation/doctor-recovery fix and close the overlapping branches after a viable path lands.

## Reproduction Assessment

Yes. Current main writes blocked stdio env keys and logs them during each transport resolution; PR head removes that runtime callback and only warns from the MCP config writer.

## Solution Assessment

No. The save-time warning is plausible for newly written configs, but it is not the best complete fix until existing or hand-edited configs get a one-time diagnostic path and the docs match the selected contract.
