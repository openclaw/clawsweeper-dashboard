## Summary

Current `main` no longer reads Telegram/Slack legacy config rules through the full bundled channel runtime path. Config-read and validation flows now use narrow doctor/public contract artifacts (`doctor-contract-api` / `contract-api`) plus dedicated Telegram/Slack exports, with tests covering the fast path and fallback behavior.

## Evidence
