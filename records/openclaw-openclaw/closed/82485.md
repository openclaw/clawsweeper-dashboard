## Summary

Keep open: current main still has the source-visible approval delivery race, and the linked unmerged PR is the active fix path for this exact bug. This issue should close only after a reviewed fix lands and proves the in-flight native binding cleanup behavior.

## What This Changes

The issue reports a race in `approval-handler-runtime` where `onStopped` can clear active approval entries while `deliverTarget` is between awaits, allowing a later wrapped entry insertion that never receives `unbindPending`.

## Best Possible Solution

Land a reviewed lifecycle-safe fix for in-flight native approval delivery cleanup with deterministic regression coverage, then close this issue with the merge commit as proof.

## Reproduction Assessment

Yes. Source inspection on current main shows the reported await/cleanup window, and the issue supplies deterministic stub-plugin steps; I did not run that setup, so this is source-reproducible rather than locally reproduced.

Yes. Keeping this issue open while the linked focused PR is reviewed is the narrowest maintainable path; the implementation should add a lifecycle guard and parked-delivery regression coverage without changing native runtime contracts.
