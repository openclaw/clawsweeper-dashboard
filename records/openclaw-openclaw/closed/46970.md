## Summary

Current `main` already implements MCP/plugin tool approval gating through the existing plugin-approval flow, including gateway request/wait/resolve RPCs, chat/native approval resolution, MCP bridge enforcement, and Codex harness routing. This PR's separate `approvals.toolPolicy` / `tool.approval.*` stack is redundant against the implementation already present on `main`.
