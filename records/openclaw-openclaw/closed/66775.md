## Summary

Current `main` contains an idle-aware cleanup fix that directly prevents synthetic `missing tool result in session history` failures from being inserted while long-running tool work is still finishing, which matches this report's core failure mode.

## Evidence

- **Idle-aware cleanup added on main:** The embedded runner now explicitly waits for the agent to become idle before flushing pending tool results during attempt cleanup; the inline bug note says the previous behavior could insert synthetic `missing tool result` errors while tools were still executing, causing silent failures.
  - file: [src/agents/pi-embedded-runner/run/attempt.ts:3076](https://github.com/openclaw/openclaw/blob/55d3e7bc473e34883a2d2cce51ce904b87c32eed/src/agents/pi-embedded-runner/run/attempt.ts#L3076)
  - command: `nl -ba src/agents/pi-embedded-runner/run/attempt.ts | sed -n '3040,3115p'`
  - sha: [55d3e7bc473e](https://github.com/openclaw/openclaw/commit/55d3e7bc473e34883a2d2cce51ce904b87c32eed)
- **Flush helper waits instead of racing tool completion:** `flushPendingToolResultsAfterIdle()` now races `waitForIdle()` against a timeout and only flushes after idle; on timeout it can clear pending tool results instead of synthesizing a false missing-result entry.
  - file: [src/agents/pi-embedded-runner/wait-for-idle-before-flush.ts:43](https://github.com/openclaw/openclaw/blob/55d3e7bc473e34883a2d2cce51ce904b87c32eed/src/agents/pi-embedded-runner/wait-for-idle-before-flush.ts#L43)

  - sha: [55d3e7bc473e](https://github.com/openclaw/openclaw/commit/55d3e7bc473e34883a2d2cce51ce904b87c32eed)
- **Cleanup path uses non-corrupting timeout behavior:** Attempt teardown now calls the idle-aware helper with `clearPendingOnTimeout: true`, so cleanup no longer has to write a synthetic missing tool result when shutdown happens before the tool finishes.
  - file: [src/agents/pi-embedded-runner/run/attempt.subscription-cleanup.ts:40](https://github.com/openclaw/openclaw/blob/55d3e7bc473e34883a2d2cce51ce904b87c32eed/src/agents/pi-embedded-runner/run/attempt.subscription-cleanup.ts#L40)

- **Regression tests cover the reported symptom:** The new guard test proves that when a tool result arrives before idle, the transcript keeps the real tool result and does not inject the synthetic `missing tool result`; a second test covers the timeout cleanup path that clears pending state without corrupting history.
