## Summary

The Typecast work is useful and the author has actively maintained it, but current main already exposes the generic speech-provider plugin seam needed for this as an external provider plugin. The PR is a bundled vendor integration, not a missing OpenClaw core/API fix, so the conservative maintainer outcome is to close it as ClawHub/community-plugin scope.

## Best Possible Solution

Close this PR as ClawHub scope and encourage the author or Typecast/Neosapience to publish it as an external speech-provider plugin. If publishing exposes a missing SDK/runtime hook, the best OpenClaw core contribution would be a narrow plugin-API PR for that seam rather than a bundled vendor provider.
