## Summary

This PR should close as ClawHub/plugin-scoped: current main already exposes model-selection hooks and model fallback, while the PR adds a new core router with unresolved gateway semantics and unrelated policy/docs churn.

## Best Possible Solution

Close this core PR and route the feature to a ClawHub plugin that uses before_model_resolve for prompt/attachment-aware tier selection, model_call_* and tool hooks for telemetry, and existing configured model fallbacks for failure recovery. If same-turn escalation needs new host support, open a focused plugin API design instead of landing a one-off core Gateway router.
