## Summary

PR #44665 is superseded by merged PR #69285, which shipped the same cron `delivery.mode="none"` status fix. Current `main` already reports non-requested cron delivery as `not-requested` before preserving real requested-delivery failures as `not-delivered`, with regression coverage for `delivered=false` plus a delivery error.

## Best Possible Solution

Close this PR as superseded by the already-merged and shipped #69285 implementation. Keep the current `main` behavior and regression tests; any future webhook-delivery-status semantics should be handled as a separate product/behavior issue because webhook delivery is currently posted from the gateway finished-event path, not by this PR's timer-only patch.
