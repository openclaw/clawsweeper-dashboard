## Summary

Current main already solves the reported Discord multi-account startup burst with a plugin-owned startup stagger. The exact `channels.discord.connectStagger` knob was not added, but the observable problem is handled: later configured Discord accounts sleep before probing/monitoring, and the behavior is covered by a regression test and present in the latest local release tag.
