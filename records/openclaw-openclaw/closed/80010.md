Close: current main and v2026.6.6 already split the Discord voice surfaces into `/voice` for TTS audio and `/talkvoice` for Talk voice selection, so the reported command collision is resolved without adding `/vc list/chat`.

## Best Possible Solution

Keep the shipped `/voice` TTS plus `/talkvoice` Talk voice split, and direct Discord users to `/voice audio <text>` for one-off outbound audio.

## Solution Assessment

Yes. The current split is the narrowest maintainable solution because `/vc` remains voice-channel session control while Talk voice selection and TTS audio each have their own command surface.
