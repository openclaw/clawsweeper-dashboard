## Summary

Keep open. Current main has partial runtime-backed health work, but the Discord health/status mismatch class is still present: default `health --json` remains non-probe/cache-eligible, and health builds summaries from runtime-projected fields instead of the full per-account status snapshot path that carries Discord credential metadata into `channels.status`.

## What This Changes

The issue requests consistent Discord state between `openclaw health --json` and `openclaw channels status --probe --json` for running/connected state and `tokenSource`.

## Best Possible Solution

Health should use the same safe per-account snapshot contract as `channels.status` for Discord and other token-backed channels, carrying runtime state plus non-secret credential-source metadata before summaries are built, while preserving cache behavior only when it cannot mislead fresh channel state.

## Repair Work Prompt

Fix the health/status snapshot mismatch for Discord and the related token-backed channel health cluster. Current `openclaw health --json` builds summaries from projected runtime fields in `src/commands/health.ts`, while `channels.status` uses `buildChannelAccountSnapshot` and therefore carries plugin-owned credential metadata such as Discord `tokenSource`. Update health to reuse `buildChannelAccountSnapshot` or an equivalent shared helper so runtime state, probe/audit data, and non-secret credential-source/status fields are present before `buildChannelSummary` runs. Preserve non-sensitive filtering: do not expose raw tokens, raw SecretRef values, or raw probe payloads when `includeSensitive` is false. Avoid core owner-specific Discord special cases if a generic channel status helper works. Add a regression covering configured/running Discord health output with `running: true`, `connected: true`, and `tokenSource: "config"`, and keep existing cache/runtime behavior tests meaningful. Coordinate with #46494, #59287, #49758, #46527, and #72417. Add a user-facing changelog fix entry with reporter credit, using `Thanks @ferenc-acs` and any related reporter credits that fit the final scope.
