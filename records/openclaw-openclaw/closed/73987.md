## Summary

Keep open. Current main still scopes Control UI token storage by exact gateway URL, and this replacement PR is focused useful work, but its test proof relies on one mocked sessionStorage across different page origins and the loopback bearer-token reuse boundary still needs maintainer/security approval.

## What This Changes

The PR normalizes Control UI gateway storage/token scope for loopback aliases, adds storage coverage, and adds a changelog entry.

## Best Possible Solution

Define and approve the local Control UI credential boundary, then land focused same-origin storage-key normalization with browser-origin-accurate tests and wording that does not promise cross-origin sessionStorage handoff.

## Reproduction Assessment

Yes for the source-level exact-key fragmentation: current main keys tokens by parsed protocol, host, and path. No for cross-origin dashboard-link reuse, because real sessionStorage is partitioned by origin and tab.

## Solution Assessment

No as-is. The PR touches the right module, but the accepted solution should keep proof within one browser origin or add an explicit handoff design, and maintainer/security approval is needed before broadening token reuse.
