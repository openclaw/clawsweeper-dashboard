## Summary

Current `main` already implements the macOS LaunchAgent restart fix: restart now uses `launchctl kickstart -k` instead of unloading the service, self-restarts are handed off to a detached helper, and the CLI has explicit recovery paths for unloaded LaunchAgents. The fix was shipped in `v2026.3.11`, with later hardening also present on `main`.

## Evidence

- **Current launchd restart path keeps the service registered:** `src/daemon/launchd.ts:656` enables the LaunchAgent and restarts it with `launchctl kickstart -k`; `src/daemon/launchd.ts:671` only falls back to `bootstrap` when the service is actually not loaded, which directly addresses the old unload/reinstall failure mode.
  - file: [src/daemon/launchd.ts:656](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/daemon/launchd.ts#L656)
  - sha: [e94c0bf51542](https://github.com/openclaw/openclaw/commit/e94c0bf51542fe518e49ad4e2ee29882891b5603)
- **Self-restarts are handed off out of process:** `src/daemon/launchd.ts:635` routes restarts initiated from inside the managed LaunchAgent through a detached helper, and `src/daemon/launchd-restart-handoff.ts:95` runs `launchctl kickstart -k` after the caller exits so the initiating process is not killed mid-command.
  - file: [src/daemon/launchd-restart-handoff.ts:95](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/daemon/launchd-restart-handoff.ts#L95)

  - file: [CHANGELOG.md:2672](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/CHANGELOG.md#L2672)
