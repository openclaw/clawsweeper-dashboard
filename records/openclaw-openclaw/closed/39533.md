## Summary

Current `main` no longer has a dedicated `/api/stats/subagents` handler. The equivalent capability is implemented in the Gateway session API that shipped in `v2026.4.22`: `sessions.list` accepts a caller-provided `limit`, supports `spawnedBy` filtering for subagent trees, and applies the requested limit instead of a hardcoded 20.
