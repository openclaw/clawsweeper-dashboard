## Summary

Current `main` already routes `anthropic-messages` runs through OpenClaw's own per-request SSE transport instead of the Anthropic SDK `MessageStream`, and the embedded runner rebuilds the stream function on each tool-use turn. That removes the cross-request snapshot state described in the report, and the same transport path is already present in release `v2026.4.23`.

## Evidence

- **Anthropic messages always use the boundary-aware transport path:** `createSupportedTransportStreamFn()` maps `api: "anthropic-messages"` to `createAnthropicMessagesTransportStreamFn()`, and `resolveEmbeddedAgentStreamFn()` selects that boundary-aware transport whenever the session is on the default/simple stream path.
