## Summary

Current main implements the Discord stale-socket restart-loop fix by separating transport liveness from app-message activity and recording Discord WebSocket frame activity into channel health state. This addresses the quiet-Discord false restart feedback loop described in #41354 and the related #47535 discussion; the useful #45712 reconnect-grace PR is superseded for this observable symptom by the transport-liveness implementation.

## Evidence
