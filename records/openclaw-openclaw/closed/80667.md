## What This Changes

Pure claude-cli sessions are reported to write a trajectory pointer but never create or populate the runtime trajectory JSONL because the CLI executor does not emit recorder events.

## Best Possible Solution

Land a narrow fix through https://github.com/openclaw/openclaw/pull/81039 that wires the existing trajectory recorder into the CLI executor path, or explicitly document a maintainer-approved CLI trajectory exemption.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main documents default trajectory capture and records it for embedded runs, while the pure claude-cli executor path has no recorder calls and only fallback emits model.fallback_step. I did not run a live claude-cli session in this read-only review.

## Solution Assessment

Yes, the best fix is the narrow existing-recorder integration in the CLI executor path, with regression coverage and real pure claude-cli proof before merge. A docs-only clarification would only be appropriate if maintainers intentionally exempt CLI runs from the documented trajectory contract.
