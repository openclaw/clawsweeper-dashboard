## Summary

Current main does not support the report’s root-cause theory. Feishu is now loaded as a single bundled channel entry, and duplicate-id warnings are emitted from manifest discovery when the same plugin id is discovered from multiple install roots, not when a channel plugin shares an id with its owning entry.

## Evidence

- **Feishu is a single bundled channel entry:** The current Feishu entry exports `defineBundledChannelEntry({ id: "feishu", ... })`, not a separate top-level plugin object plus channel plugin object as described in the issue.
  - file: [extensions/feishu/index.ts:56](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/extensions/feishu/index.ts#L56)
  - command: `nl -ba extensions/feishu/index.ts | sed -n '1,120p'`
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Bundled channel entries register a channel, not a second plugin record:** `defineBundledChannelEntry` loads the channel plugin and calls `api.registerChannel({ plugin: channelPlugin })`; there is no duplicate top-level plugin registration path here.
  - file: [src/plugin-sdk/channel-entry-contract.ts:476](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/plugin-sdk/channel-entry-contract.ts#L476)

  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Duplicate-id warning is tied to multi-root manifest discovery:** The duplicate diagnostic is produced when two discovered manifests share the same plugin id and one origin overrides another; the warning text names overridden and winning origins/sources.
  - file: [src/plugins/manifest-registry.ts:668](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/plugins/manifest-registry.ts#L668)

  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Doctor/config warnings are surfaced from manifest-registry diagnostics:** Validation maps manifest-registry diagnostics into `plugins.entries.<id>` warnings, which matches the kind of warning path shown in the report.
  - file: [src/config/validation.ts:784](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/config/validation.ts#L784)

  - file: [docs/plugins/manifest.md:664](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/docs/plugins/manifest.md#L664)

## Risks / Open Questions

- A mixed install with both bundled Feishu and an installed `~/.openclaw/extensions/feishu` copy will still warn, but that is a real duplicate-install condition rather than the false-positive channel-registration theory in the issue.
