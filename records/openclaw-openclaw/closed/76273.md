## Summary

Keep open. Current main still has a real Codex-vs-PI split: PI/CLI paths feed workspace bootstrap context into the OpenClaw system prompt, while the native Codex app-server path builds developer instructions and turn input without loading workspace bootstrap files; the docs still describe SOUL.md as injected workspace context.

## What This Changes

Clarify or align Codex app-server handling of workspace bootstrap files so SOUL.md persona behavior matches PI behavior or is documented as AGENTS.md-first.

## Best Possible Solution

Pick one public contract and make it executable: either project workspace bootstrap into Codex in a cache-aware way, or update Codex-specific docs/templates/onboarding so users know durable persona belongs in AGENTS.md for that runtime.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to compare the Codex app-server path, which has no bootstrap resolver call sites, with the PI path that resolves and passes `contextFiles` into `buildEmbeddedSystemPrompt`; a focused Codex unit test with a workspace SOUL.md should currently expose the gap.

## Solution Assessment

Unclear. This issue is asking for the policy decision rather than proposing a patch; existing docs favor alignment, but maintainers should confirm before automation chooses between code injection and AGENTS.md-first documentation.
