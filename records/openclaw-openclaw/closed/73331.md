## Summary

Current main already addresses the reported v2026.4.26 runtime-deps/mirror busy-loop path: bundled runtime mirrors are now fingerprinted and skipped when unchanged, and staged npm installs keep retained dependencies in the generated manifest while installing only newly missing specs. The matching tests and changelog entries are present on current main.

## Best Possible Solution

Keep the current main runtime-deps staging and mirror-freshness implementation, ship it in the next release, and reopen only if the reporter can reproduce the same openai/resources CPU loop on a build from current main with fresh startup trace or strace evidence.
