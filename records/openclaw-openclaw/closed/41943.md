## Summary

Close this PR as superseded, not fixed: the Feishu HTTP proxy bug remains tracked by open canonical issues, while this older branch is conflicting, lacks real proxy proof, and no longer matches the current Proxyline-backed proxy surface.

## What This Changes

The PR changes the Feishu plugin HTTP client wrapper to add an HTTPS proxy agent from proxy environment variables and set Axios `proxy: false` on wrapped Lark SDK HTTP calls.

## Best Possible Solution

Close this stale branch and keep the canonical Feishu proxy issues open for a target-aware replacement PR that preserves Proxyline policy, timeout/User-Agent behavior, tests, docs, and real proxy proof.

## Reproduction Assessment

Yes, at source level: Feishu probing reaches `probeFeishu` -> `createFeishuClient` -> the Lark Axios-backed HTTP instance while proxy env or managed proxy mode can be active. I did not run live Feishu credentials through a local proxy in this read-only review.

## Solution Assessment

No. The PR targets the right seam, but forcing raw-env `httpsAgent` plus `proxy: false` is outdated against current Proxyline helpers, crosses the Lark option type, and lacks the egress-policy decision needed for a safe fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Honor proxy eligibility before forcing the agent:** `extensions/feishu/src/client.ts:57-60`
  - body: When any proxy env var exists, every wrapped Lark HTTP request receives `httpsAgent` and `proxy: false`. That bypasses Axios/proxy-from-env target eligibility and can ignore `NO_PROXY` or managed-proxy policy, so resolve the effective Feishu/Lark URL and chosen egress mode before setting these options.
  - confidence: 0.93
- **[P2] Keep Axios-only fields behind a typed adapter:** `extensions/feishu/src/client.ts:58-60`
  - body: `Lark.HttpRequestOptions` does not declare `httpsAgent` or `proxy`, so assigning those fields directly crosses the SDK type contract. Keep the Lark request options typed and add a narrow Axios-backed handoff type or adapter for Axios-only fields.
  - confidence: 0.9
- **[P2] Use the current proxy helper surface:** `extensions/feishu/src/client.ts:10-17`
  - body: The branch constructs proxy behavior from env directly, but current main routes plugin proxy construction through the shared Proxyline-backed helper and `@openclaw/feishu` no longer owns `https-proxy-agent` in package metadata. Rebase onto the current helper seam or add valid plugin-owned dependency metadata.
  - confidence: 0.84
- **[P3] Refresh proxy routing when reusing cached clients:** `extensions/feishu/src/client.ts:52-60`
  - body: `proxyAgent` is captured when the HTTP instance is constructed, while cached clients are reused based on credentials, domain, and timeout only. If proxy env changes during the process lifetime, cached clients keep stale routing; resolve eligibility per request or include proxy state in the cache comparison.
  - confidence: 0.76
