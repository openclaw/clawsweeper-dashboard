## Summary

#68965 is the same remaining models.mode="replace" bug already tracked by open issue #66957. Current main still has the bug: provider planning unconditionally runs implicit plugin provider discovery and merges implicit providers before mode handling, and Codex still contributes a catalog hook that can be injected. PR #68978 attempted the exact fix but is closed unmerged, so the remaining work should stay consolidated under #66957 rather than keeping this duplicate open.

## Best Possible Solution

Close #68965 as a duplicate of the older open canonical issue #66957, while preserving this report's useful Codex-specific analysis and the closed PR #68978 as implementation references. The actual fix should branch in `src/agents/models-config.plan.ts` before implicit provider discovery when `cfg.models?.mode === "replace"`, then add a focused regression test proving replace mode does not call the implicit resolver while default/merge still do.
