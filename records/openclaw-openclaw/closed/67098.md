## Summary

Current `main` no longer routes `sessions_send` work through a single shared `CommandLane.Nested` lane. Nested agent sends are now sharded onto per-target-session `nested:<sessionKey>` lanes, and the queue runs each lane independently, so the PR's claimed global serialization bottleneck no longer matches shipped behavior on `main`.
