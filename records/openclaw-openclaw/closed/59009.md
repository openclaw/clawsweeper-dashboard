## Summary

Current `main` does not match the reported misrouting bug. Feishu DM routing is keyed from `senderOpenId` when `chat_type` is not `group`, so an `oc_`-prefixed `chat_id` on a p2p reaction does not route into a group session. The remaining no-`chat_type` drop is still present, but current code and changelog show that is intentional fail-closed security behavior rather than the reported `oc_` group misclassification.

## Evidence

- **DM routing ignores `chat_id`:** For non-group Feishu events, `isGroup` is false and routing uses `peer: { kind: "direct", id: ctx.senderOpenId }`. The synthetic/event `chat_id` is not used to choose the session, so an `oc_...` DM id does not become a group route on current main.
