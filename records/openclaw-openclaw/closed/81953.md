## Summary

Current main already fixes the reported Telegram forum-topic serialization by draining isolated polling spool updates per Telegram lane instead of awaiting every spooled update serially; the fix is also present in prerelease v2026.5.14-beta.1.

## What This Changes

The issue reports that Telegram forum-topic messages in different topics of one group appear to serialize before reaching the agent pipeline.

## Best Possible Solution

Keep the merged lane-keyed isolated ingress drain and direct affected stable users to upgrade to v2026.5.14-beta.1 or the next stable release containing it.

## Reproduction Assessment

No, not as a current-main failure. The old v2026.5.12 failure is source-traceable to the pre-fix isolated-ingress serial drain, and current main has regression coverage for interleaving different Telegram topic lanes.

## Solution Assessment

Yes. Reusing Telegram's existing sequential key inside the isolated spool drain is the narrow maintainable fix because it preserves same-topic ordering while allowing different topics, DMs, and control lanes to progress independently.
