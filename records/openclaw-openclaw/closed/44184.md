## Summary

Current `main` and the latest tagged release already contain two concrete fixes for this symptom: OpenAI-compatible streaming requests now always ask providers to include usage metadata, and `/status`/`session_status` now fall back to transcript usage when the live session snapshot is sparse or stale.
