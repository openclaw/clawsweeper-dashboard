## Summary

Keep open: current main's recall hook honors the active-memory agent allowlist, but the session-scoped `/active-memory status` command can still report `on` for a non-targeted agent because it only checks the per-session toggle state.

## What This Changes

The issue asks for `/active-memory status` to respect `plugins.entries.active-memory.config.agents` instead of reporting active memory as on in agents outside the allowlist.

## Best Possible Solution

Status and session-toggle responses should report effective Active Memory availability for the current agent/session, including when the agent is not targeted, while leaving the existing recall gate intact.

## Reproduction Assessment

Yes. From source inspection, configure Active Memory for `sandbox`, run the session-scoped status command from another agent, and the handler can reach the `on for this session` branch because it only checks the session toggle.

## Solution Assessment

Yes. The narrow maintainable fix is in the Active Memory command/status path with regression coverage for an untargeted agent; the recall runtime should not be broadened because it already checks the configured agent list.
