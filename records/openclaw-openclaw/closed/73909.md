## Summary

Keep open. Current main has native Ollama cloud/local routing and generic capability discovery, but it still does not prove or specifically fix `ollama/gemini-3-flash-preview:cloud` or `ollama/deepseek-v4-pro:cloud` through the full OpenClaw agent path. The reporter's successful `/api/generate` curl is materially different from OpenClaw's streamed `/api/chat` path with converted messages and possible tools, and current code/tests only show an explicit cloud compatibility branch for Kimi-style models, not the named Gemini or DeepSeek cloud models.

## Best Possible Solution

Keep this open and reproduce both named Ollama Cloud models through OpenClaw's native Ollama `/api/chat` agent path with tools enabled and disabled. If the failure is payload-specific, add the smallest Ollama-plugin compatibility path in `extensions/ollama/src/stream.ts` or model capability handling in `extensions/ollama/src/provider-models.ts`, plus regression tests for the observed request/response shape. Add a docs workaround only if the limitation remains model-specific.
