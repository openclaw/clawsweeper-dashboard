## Summary

Current `main` already supports Junie-style ACP harnesses through ACPX custom agent command overrides, so the requested capability exists without a new OpenClaw core feature.

## Evidence

- **ACPX plugin exposes per-agent command overrides:** The ACPX plugin manifest defines an `agents` config surface described as optional per-agent command overrides for the embedded ACP runtime.
  - file: [extensions/acpx/openclaw.plugin.json:144](https://github.com/openclaw/openclaw/blob/4f4288e3b5be2e55abfce5213f0bcf182a252f5e/extensions/acpx/openclaw.plugin.json#L144)
  - sha: [4f4288e3b5be](https://github.com/openclaw/openclaw/commit/4f4288e3b5be2e55abfce5213f0bcf182a252f5e)
- **Custom agent commands are part of the validated config contract:** `AcpxPluginConfig` includes `agents?: Record<string, { command: string }>` and the schema accepts arbitrary agent ids with required non-empty commands.
  - file: [extensions/acpx/src/config-schema.ts:38](https://github.com/openclaw/openclaw/blob/4f4288e3b5be2e55abfce5213f0bcf182a252f5e/extensions/acpx/src/config-schema.ts#L38)
  - sha: [4f4288e3b5be](https://github.com/openclaw/openclaw/commit/4f4288e3b5be2e55abfce5213f0bcf182a252f5e)
- **Runtime uses configured agent overrides:** The ACPX runtime service passes `pluginConfig.agents` into `createAgentRegistry({ overrides: ... })`, so configured ids become usable ACP agents at runtime.
  - file: [extensions/acpx/src/service.ts:49](https://github.com/openclaw/openclaw/blob/4f4288e3b5be2e55abfce5213f0bcf182a252f5e/extensions/acpx/src/service.ts#L49)
  - sha: [4f4288e3b5be](https://github.com/openclaw/openclaw/commit/4f4288e3b5be2e55abfce5213f0bcf182a252f5e)
- **ACP agent ids are not hardcoded in core config:** Core ACP config accepts `defaultAgent` as a string and `allowedAgents` as `string[]`, which allows a custom id such as `junie` once configured in ACPX.
  - file: [src/config/zod-schema.ts:471](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/config/zod-schema.ts#L471)
- **Docs already describe custom ACPX aliases:** The setup docs list built-in aliases, then explicitly say to prefer them unless the user's acpx config defines custom agent aliases.
  - file: [docs/tools/acp-agents-setup.md:16](https://github.com/openclaw/openclaw/blob/4f4288e3b5be2e55abfce5213f0bcf182a252f5e/docs/tools/acp-agents-setup.md#L16)
  - sha: [4f4288e3b5be](https://github.com/openclaw/openclaw/commit/4f4288e3b5be2e55abfce5213f0bcf182a252f5e)
- **Junie exposes an ACP mode compatible with this path:** The issue-linked Junie documentation says ACP mode is enabled with `junie --acp true`, which fits OpenClaw's configurable ACPX command-override model rather than requiring a new core protocol surface.

## Risks / Open Questions

- There is no first-class built-in `junie` alias or Junie-specific docs example yet, so discoverability is weaker than for the documented built-in harnesses.
