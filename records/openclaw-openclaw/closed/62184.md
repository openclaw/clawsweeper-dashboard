## Summary

Close as `clawhub`: current `main` already documents Dreaming's artifact and promotion boundaries, while the remaining request is an optional memory-hardening workflow that project vision explicitly prefers to keep in community skills/plugins. The reporter's own `memory-governor` implementation is positive evidence that this belongs on the extension path rather than in OpenClaw core.

## Evidence

- **Reporter already linked a working community implementation:** The issue comment explicitly points to `https://github.com/SakenW/memory-governor` and says it is not a request to absorb the whole skill into core, only to discuss narrower staging ideas. That is direct evidence that an external skill/plugin path already exists for this workflow.
- **Project vision prefers optional capability as plugins/skills, not core:** `VISION.md` says core should stay lean, optional capability should usually ship as plugins, memory is a plugin slot, ClawHub is the discovery/publish surface, and new skills should be published through ClawHub first rather than added to core by default.
