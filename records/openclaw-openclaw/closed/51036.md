## Summary

Current `main` no longer silently returns a successful agent result with `payloads: []` for empty terminal turns. The embedded agent runner now classifies zero-payload/empty-response turns as incomplete, retries GPT-5-family empty replies once, and then surfaces a visible error payload instead of returning an empty success result.

## Evidence

- **Provider-agnostic empty-response detection now maps zero-payload terminal turns to a visible incomplete-turn error:** `resolveIncompleteTurnPayloadText` returns `"⚠️ Agent couldn't generate a response. Please try again."` whenever a turn has zero payloads and the assistant ended as an empty response, unless it was a deliberate silent reply or already delivered via messaging tool.
  - file: [src/agents/pi-embedded-runner/run/incomplete-turn.ts:160](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/agents/pi-embedded-runner/run/incomplete-turn.ts#L160)
  - command: `nl -ba src/agents/pi-embedded-runner/run/incomplete-turn.ts | sed -n '150,235p'`
  - sha: [6944d7025d9c](https://github.com/openclaw/openclaw/commit/6944d7025d9c0ba4891cbe646ab5dab039d504ca)
- **Empty terminal turns are no longer returned as silent success from the embedded runner:** `runEmbeddedPiAgent` retries GPT-5-family empty responses once, then logs `empty response retries exhausted` and returns an error payload from the incomplete-turn path instead of propagating an empty payload array.
  - file: [src/agents/pi-embedded-runner/run.ts:1910](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/agents/pi-embedded-runner/run.ts#L1910)

  - file: [src/agents/pi-embedded-runner/run.incomplete-turn.test.ts:438](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/agents/pi-embedded-runner/run.incomplete-turn.test.ts#L438)
