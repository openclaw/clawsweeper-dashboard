## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Keep pump errors out of the initial-rejection retry:** `src/agents/embedded-agent-runner/thinking.ts:708`
  - body: The new unified `try` catches rejections from `pumpStreamWithRecovery()`, not only the initial `streamFn` promise. Current `pumpStreamWithRecovery` deliberately throws recoverable thinking errors after `yieldedOutput` is true so the wrapper does not retry after chunks were already pushed; this catch treats that same error as an initial rejection and starts a stripped-thinking retry, which can duplicate assistant output in the same stream.
  - confidence: 0.9
- **[P1] Do not replace an active agent database during memory import:** `src/infra/state-migrations.ts:4534-4536`
  - body: This archives any existing target database whenever `memory_index_chunks` is empty. The per-agent database also owns auth profile, cache, schema, and other agent-scoped state, so a valid upgraded database with zero memory chunks would be moved aside and replaced by the legacy memory file, breaking existing auth/session state.
  - confidence: 0.91
- **[P1] Preserve SQLite sidecars when importing legacy memory:** `src/infra/state-migrations.ts:4539`
  - body: Copying only `main.sqlite` ignores `main.sqlite-wal` and `main.sqlite-shm`. A WAL-mode SQLite database can have committed rows in the WAL; copying the base file and then backing up the legacy base can silently omit indexed memory rows from the migrated store.
  - confidence: 0.83
