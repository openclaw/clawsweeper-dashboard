## Summary

Keep open. The layered BOOT.md wrapper, echo guard, and message-tool sanitizer are the right fix direction, but the current PR head still leaves JSON-string `presentation` payloads unsanitized before they are parsed into visible message content.

## What This Changes

The PR wraps BOOT.md content as internal runtime context, strips boot/runtime-context echoes from message-tool send fields, adds a session-scoped boot echo guard, and adds regression coverage plus a changelog entry.

## Best Possible Solution

Keep the layered boot wrapper and echo guard, but parse and sanitize or reject JSON-string presentation payloads before `runMessageAction` can deliver them, then land this PR as the canonical fix for https://github.com/openclaw/openclaw/issues/53732.

## Reproduction Assessment

Yes. Source inspection shows the current-main BOOT.md fallback leak path, and PR-head source inspection shows a remaining JSON-string `presentation` bypass through the message-tool sanitizer into `runMessageAction` parsing.

## Solution Assessment

No. The overall approach is the narrow maintainable fix, but it is incomplete until stringified presentation payloads are parsed and sanitized, or rejected, before they can become channel-visible content.
