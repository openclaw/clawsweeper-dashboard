## Summary

Keep open: current main only partially addresses the disk-bloat problem. `openclaw sessions cleanup` now prunes old unreferenced primary transcripts, checkpoints, and trajectory sidecars, but it still has no dedicated dry-run/text/JSON planning for `.deleted` and `.reset` archive files, and https://github.com/openclaw/openclaw/pull/75672 is still the open closing implementation candidate.

## What This Changes

Have `openclaw sessions cleanup` preview and remove physical `.deleted`/`.reset` archive files and orphan checkpoints, not only prune `sessions.json` entries.

## Best Possible Solution

Review and land, reject, or replace https://github.com/openclaw/openclaw/pull/75672 while preserving retention semantics and adding dry-run/text/JSON archive cleanup reporting.

## Reproduction Assessment

Yes. Source inspection on current main shows the cleanup preview and summary lack an archive-file plan, while `.deleted` and `.reset` files are excluded from the unreferenced-artifact sweep unless optional disk-budget pressure is configured.

## Solution Assessment

Yes on direction, but the best path is to finish the linked implementation PR or a replacement rather than close this issue or queue a parallel repair. The fix should integrate with existing retention, warn/enforce, and JSON/text summary contracts.
