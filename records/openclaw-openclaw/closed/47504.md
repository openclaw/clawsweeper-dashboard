## Summary

Current main fixes the reported standalone `/new` channel-freeze path by persisting the reset session and then returning a local acknowledgement without starting a model run. The behavior is covered by command-level and end-to-end regression tests and documented. The fix is on current main after the latest shipped release, so no fixed release tag is known yet.

## Best Possible Solution

Close this issue as implemented on current main for the standalone `/new` channel-freeze path. Keep the no-model local acknowledgement behavior, and leave narrower remaining session-state cases to the existing focused reports such as #47561 and #47335.
