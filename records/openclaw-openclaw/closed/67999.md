## Summary

The reported Control UI chat-rendering race is already fixed on current `main` and the fix is present in the shipped `v2026.4.23` release.

## Evidence

- **Optimistic user-message append on send:** Control UI now appends the user's message to `chatMessages` immediately before awaiting `chat.send`, so the sent message renders locally without waiting for a history reload.
  - file: [ui/src/ui/controllers/chat.ts:325](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/ui/src/ui/controllers/chat.ts#L325)
  - command: `nl -ba ui/src/ui/controllers/chat.ts | sed -n '320,339p'`
  - sha: [fa976e5b93b3](https://github.com/openclaw/openclaw/commit/fa976e5b93b3819f9e137fe08635764edee38290)
- **Mid-run session.message reload guard:** The gateway handler now skips `loadChatHistory` while `chatRunId` is active and explicitly documents the prior race: reloading history mid-run reset streaming state and delayed the user message card until later.
  - file: [ui/src/ui/app-gateway.ts:496](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/ui/src/ui/app-gateway.ts#L496)

  - file: [ui/src/ui/app-gateway.node.test.ts:725](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/ui/src/ui/app-gateway.node.test.ts#L725)
