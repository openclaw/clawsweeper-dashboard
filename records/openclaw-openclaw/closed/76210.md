## What This Changes

The issue reports Docker/VPS deployments timing out API calls and saturating the event loop at idle because plugin dependency installation and session processing block the Gateway.

## Best Possible Solution

Keep the current main behavior: plugin dependency convergence stays in install/update/doctor paths, Gateway startup remains verify-only, and session transcript hydration yields so RPC and channel traffic stay responsive.

## Reproduction Assessment

Unclear for the exact VPS deployment. The affected-release reproduction steps are concrete, but this cleanup is based on current-main source, docs, changelog, and targeted regression coverage rather than a live Vultr run.

## Solution Assessment

Yes. Moving dependency repair out of startup/hot paths and making transcript-heavy session listing yield is the narrow maintainable fix for this report, rather than masking stuck-session or timeout symptoms.
