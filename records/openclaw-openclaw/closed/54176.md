## Summary

Keep open. Current main still lacks the legacy Word/OLE MIME guard in the media-understanding file-embedding path, and the issue is already paired with open implementation PR #73799 using closing syntax, so it should stay open until that PR or an equivalent fix lands.

## What This Changes

Treat legacy `.doc`/OLE attachments with MIME `application/x-cfb` or `application/msword` as binary so raw bytes are not embedded into prompts.

## Best Possible Solution

Land #73799 or an equivalent narrow patch that marks `application/x-cfb` and `application/msword` as binary in `src/media-understanding/apply.ts`, preserves vendor `+json`/`+xml` text behavior, and adds regression coverage in `src/media-understanding/apply.test.ts`.
