## Summary

Current main already resolves the linked legacy SQLite crash path: requester session keys are restored from `requester_session_key` with an `owner_key` fallback, legacy rows are normalized before writes, `markTaskLostById` persists a merged task record, and a regression test covers a legacy `requester_session_key TEXT NOT NULL` table. The linked bug #61097 was already closed as implemented, and this PR no longer has a necessary merge delta.

## Best Possible Solution

Close this PR as already implemented. Keep the shipped restore/migration/update behavior and existing regression test; if maintainers still want an additional defensive SQLite binder guard for malformed in-memory `TaskRecord` objects, that should be a fresh narrow hardening PR against current main rather than this stale branch.
