## Summary

Current OpenClaw main already contains the cron scheduler guard for the timezone/year-rollback path, current UI code renders `nextRunAtMs` as milliseconds directly, overdue cron execution is covered by regression tests, and the reported bare `sessionTarget: "investment"` shape is not a supported contract. The scheduler fix is also present in the tagged `v2026.4.22` release.

## Evidence
