The PR adds Google Chat schema validation, docs, generated baseline hashes, and unit tests to require `appPrincipal` when `audienceType` is `app-url`.

Full review comments:

- **[P1] Do not require appPrincipal for every app-url config:** `src/config/zod-schema.providers-core.ts:902`
  - body: This changes every `audienceType: "app-url"` config into a hard `appPrincipal` requirement, but current Google Chat verification accepts normal Chat OIDC tokens from `chat@system.gserviceaccount.com` without `expectedAddOnPrincipal`; only Workspace add-on issuer tokens need the extra principal binding. As written, standard HTTP endpoint URL configs that work today would fail schema validation on upgrade, so this needs to be narrowed to the add-on-token case or kept as a warning/setup diagnostic.
  - confidence: 0.92
- **[P1] Preserve inherited appPrincipal values for accounts:** `src/config/zod-schema.providers-core.ts:923`
  - body: The account loop only checks `account.appPrincipal`, but Google Chat account resolution inherits top-level fields and shared `accounts.default` values into named accounts. A config with root `audienceType`/`appPrincipal` and `accounts.work: {}` currently resolves with that principal, while this validation rejects it, so the effective value needs to follow the same inheritance model before failing validation.
  - confidence: 0.9
- **[P2] Move the test to the current GoogleChat schema module:** `src/config/config.googlechat-appprincipal.test.ts:2`
  - body: Current `main` no longer exports `GoogleChatConfigSchema` from `zod-schema.providers-core.ts`; the GoogleChat schema lives in `src/config/zod-schema.providers-googlechat.ts` and is re-exported from the bundled-channel schema barrel. This new test imports the stale module path, so the branch needs to rebase and exercise the current schema owner.
  - confidence: 0.86
