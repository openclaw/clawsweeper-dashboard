Full review comments:

- **[P1] Gate eviction to unexpected transport closes:** `src/agents/pi-bundle-mcp-runtime.ts:277-280`
  - body: This handler runs for every `Transport.onclose`, but the MCP SDK contract says `onclose` should also fire when `close()` is called. The catch path below still calls `await disposeSession(session)` before removing the session, so an intentional cleanup for one failed server can invoke `onTransportClosed` and retire the whole cached runtime, leaving healthy MCP sessions disposed instead of returning a degraded catalog. Mark intentional closes before disposal or no-op here unless the session is still registered as live.
