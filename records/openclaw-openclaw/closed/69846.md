## Summary

Current `main` already fixes the reported retry/crash-loop behavior by treating failed streamed assistant turns as non-replayable and by skipping synthetic tool-result repair for `error`/`aborted` turns; unit and live regression coverage were added on 2026-04-25.

## Evidence

- **Replay filter on current main:** `transformTransportMessages` classifies assistant turns with `stopReason === "error" || "aborted"` as failed and filters them out before replaying history to the next model call.
  - file: [src/agents/transport-message-transform.ts:33](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/agents/transport-message-transform.ts#L33)
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **Anthropic retry path uses the replay filter:** Anthropic transport message conversion runs all prior messages through `transformTransportMessages(...)` before building the outbound request, so the reported Anthropic/Slack retry path no longer replays the failed partial tool-call turn.
  - file: [src/agents/anthropic-transport-stream.ts:257](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/agents/anthropic-transport-stream.ts#L257)
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **Persistence guard skips failed-turn tool repair:** The session append guard now skips tool-call extraction for assistant messages whose `stopReason` is `error` or `aborted`, preventing follow-on synthetic tool-result repair on incomplete tool calls.
  - file: [src/agents/session-tool-result-guard.ts:223](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/agents/session-tool-result-guard.ts#L223)
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **Unit regression coverage:** Tests assert that aborted and errored transport assistant turns are dropped before replay and do not keep their tool-call ids in the transformed transcript.
  - file: [src/agents/transport-message-transform.test.ts:128](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/agents/transport-message-transform.test.ts#L128)
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)

## Risks / Open Questions

- Current `main` fixes the retry/replay failure path by dropping failed turns rather than rewriting old `partialJson` bytes out of existing JSONL entries; if a future code path reparses `partialJson` directly, that separate path would still need hardening.
