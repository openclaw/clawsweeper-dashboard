## Summary

Close as duplicate/superseded. #73911 is already open for the same #73758 TTS regression and its latest diff includes the post-strip empty-text guard and regression coverage that this PR is missing; this branch would otherwise let markdown-only explicit tagged TTS reach synthesis as an empty string.

## What This Changes

The PR changes speech-core TTS auto-apply logic, a shared TTS contract test, and the changelog so explicit `[[tts:text]]...[[/tts:text]]` content shorter than the auto-TTS short-skip threshold is synthesized.

## Best Possible Solution

Consolidate the fix through #73911 or one equivalent single PR: explicit tagged TTS text should bypass the untagged short-reply suppression, but empty text after markdown stripping must still skip synthesis, with regression coverage and the changelog entry preserved as appropriate.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep empty tagged text out of synthesis:** `extensions/speech-core/src/tts.ts:1598`
  - body: The new explicit-tag bypass also disables the post-`stripMarkdown()` short guard. A payload like `[[tts:text]]***[[/tts:text]]` has non-empty tagged text before stripping, then becomes empty after markdown removal and would call `textToSpeech` with an empty string. Add an empty-text guard after `stripMarkdown()` before bypassing only the length threshold.
  - confidence: 0.92
