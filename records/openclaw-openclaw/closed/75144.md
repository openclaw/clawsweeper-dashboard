## Summary

Current main already implements the requested systemd EADDRINUSE restart-loop fix in a narrower form than this PR: systemd-supervised duplicate starts exit 78, while unmanaged duplicate starts keep the prior exit-success behavior. The fix is present on main with tests and docs, but it is not contained in any tag in this checkout, so I would close this PR as already implemented on main rather than merge the broader branch diff.

## What This Changes

The PR changes `openclaw gateway` to exit with code 78 instead of 0 for healthy `GatewayLockError`/EADDRINUSE conflicts and updates the gateway CLI coverage test expectation.

## Best Possible Solution

Keep the current-main supervisor-aware implementation and close this PR as already handled; it solves the linked systemd loop without changing the non-systemd duplicate-start contract.

## Reproduction Assessment

Yes. The current-main reproduction is represented by the supervised-lock test path: systemd supervisor, `GatewayLockError` for an already-listening gateway, and a healthy `/healthz` responder produce exit-78 semantics.

## Solution Assessment

No, this PR is not the best remaining way to solve the issue. Current main has the safer fix because it applies exit 78 only to systemd-supervised lock conflicts and preserves existing behavior elsewhere.
