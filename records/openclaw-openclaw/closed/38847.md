## Summary

Current main already implements this feature request via the opt-in `active-memory` plugin: it runs a bounded memory sub-agent before eligible conversational replies, uses `memory_search`/`memory_get`, and prepends the resulting memory context to the prompt. The shipped surface is `plugins.entries.active-memory` rather than `agents.defaults.memorySearch.autoInject`.

## Risks / Open Questions

- The implementation is not a literal `memorySearch.autoInject` field; it is a plugin-owned feature under `plugins.entries.active-memory`.
- Scope is intentionally narrower than 'every inbound message': it runs for eligible interactive persistent chat sessions, not all possible triggers/surfaces.
