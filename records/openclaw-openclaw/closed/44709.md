## Summary

Current main already handles the reported Linux/systemd status path: `gateway status --deep` still reads the primary service runtime normally, `--deep` only adds extra-service scanning, and the systemd runtime parser maps `ActiveState=active` to `running`. The changelog also records a shipped daemon/systemd fix in version 2026.4.7 for sudo/user-scope handling in this area.
