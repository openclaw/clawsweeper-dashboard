## Summary

Keep open. Current main still lacks a Poppler fallback, and this PR is a useful owner-module implementation candidate, but it still needs the required changelog entry plus focused docs/test coverage before merge.

## What This Changes

The PR adds a `pdftoppm` image-rendering fallback in the bundled `document-extract` PDF extractor when `@napi-rs/canvas` cannot be loaded.

## Best Possible Solution

Land the plugin-local Poppler fallback after adding a single-line changelog entry, a docs note for the optional system renderer, and focused regression tests around fallback rendering and cleanup.

## Reproduction Assessment

Yes. Source inspection on current main shows the low-text PDF path catches a missing `@napi-rs/canvas` import and returns text with no images, and the linked issue provides matching Ubuntu 24.04 setup evidence.

## Solution Assessment

Yes, directionally. Keeping the fallback inside the bundled `document-extract` plugin is the narrow owner-module solution, provided the merge adds docs, changelog, and regression coverage for the external renderer path.
