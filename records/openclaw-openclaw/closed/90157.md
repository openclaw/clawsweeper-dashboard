## Summary

Keep open: current main and the latest release still build the Windows dashboard config opener with `Start-Process -LiteralPath`, while the supported PowerShell syntax uses `-FilePath`. The affected path is narrow and should be fixed in the Gateway opener plus its Windows command test.

## What This Changes

The issue asks to fix the dashboard config opener on Windows by changing the PowerShell `Start-Process` argument from `-LiteralPath` to `-FilePath`.

## Best Possible Solution

Use `Start-Process -FilePath` while preserving the existing single-quoted escaping, update the Windows unit expectation, and collect a Windows dashboard or PowerShell proof before landing.

## Reproduction Assessment

Yes for source reproduction: current main emits the exact Windows PowerShell command reported, and the PowerShell docs check shows `Start-Process` supports `-FilePath`, not `-LiteralPath`. I did not run a live Windows dashboard session in this read-only review.

## Solution Assessment

Yes: changing the existing Windows branch to `-FilePath` and updating the focused test is the narrowest maintainable fix. The existing escaping should stay because it protects the path from shell interpolation.
