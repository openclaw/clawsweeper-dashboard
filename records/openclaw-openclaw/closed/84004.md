## Label Justifications

- `P2`: The report describes a medium-severity cross-channel session visibility bug with limited blast radius.
- `impact:message-loss`: The issue is about Feishu messages not appearing in WebChat despite a shared-session setting.
- `impact:session-state`: The reported failure mode is session separation where the user expected a shared main session.

## Summary

Close as implemented: current main and v2026.5.18 route Feishu DMs through the generic direct-message session key, which collapses to `agent:main:main` when `session.dmScope` is `main`, and WebChat normalizes/reloads that same session.

## What This Changes

The issue reports that Feishu p2p messages with `session.dmScope: "main"` stay separate from WebChat instead of sharing the main session.

## Best Possible Solution

Keep the shipped generic routing/session-event path; users on 2026.5.12 should upgrade to v2026.5.18 or later and file a fresh report with redacted logs if the split still reproduces.

## Reproduction Assessment

No live Feishu/WebChat run was performed, but current source and shipped regression coverage show the expected path: Feishu p2p DM routes to `agent:main:main`, and WebChat loads/reloads that same session.

## Solution Assessment

Yes: the maintainable solution is the current generic routing/session-event implementation, not a Feishu-specific WebChat bridge.
