## Summary

Close this PR as superseded by #74002. Current main still has the exec-approvals `OPENCLAW_STATE_DIR` bug, but #74002 is already the open canonical fix for the same JSON/socket/default-host-source behavior and includes the docs, changelog, and broader regression coverage this PR does not carry; this PR also repeats the direct `resolveStateDir()` fallback problem found in prior duplicate reviews.

## What This Changes

This PR changes exec approval JSON/socket path resolution to use `resolveStateDir(env)` and adds a store-helper regression test plus `OPENCLAW_STATE_DIR` environment cleanup.

## Best Possible Solution

Consolidate review on #74002 or an explicit maintainer replacement that honors `OPENCLAW_STATE_DIR` for approvals JSON, default socket, and effective-policy display paths while preserving the unset `~/.openclaw` default, defining compatibility for existing approval policy/socket state, and updating docs, changelog, and focused tests together.

## Reproduction Assessment

Yes. The underlying bug is reproducible from source by setting `OPENCLAW_STATE_DIR` and observing current main still resolves approvals through `~/.openclaw`; this PR's regression is reproducible when the override is unset and only legacy `.clawdbot` exists, because `resolveStateDir()` can select that legacy root.

## Solution Assessment

No. The direction is valid, but this is not the best landing path because it uses the generic state-dir resolver directly, omits docs/changelog/effective-policy coverage, and is already superseded by the broader canonical #74002 approach.
