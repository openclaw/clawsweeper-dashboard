## Summary

The reported Matrix plain-text mention case is already implemented on current main and in shipped Matrix plugin code: plain MXID mentions are rendered through the mention-aware Matrix Markdown path, producing both `m.mentions` metadata and `formatted_body` Matrix HTML links.

## What This Changes

Ensure Matrix outbound plain-text messages containing `@user:server` mentions include `format` and `formatted_body` with clickable `matrix.to` links.

## Best Possible Solution

Keep the shipped mention-aware Matrix send path; if this still reproduces for a user, collect the raw Matrix event plus exact installed plugin bundle/version in a new report.

## Reproduction Assessment

No, not as a current-main bug: the existing Matrix send regression test covers a plain `hello @alice:example.org` message and expects both `m.mentions` and a `formatted_body` `matrix.to` link. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

Yes: the current implementation centralizes Matrix mention metadata and HTML link rendering in the send formatter instead of special-casing the formatted-body deletion branch.
