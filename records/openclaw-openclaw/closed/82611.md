## Summary

Keep open. Current main still marks Feishu-originated runs as hidden from Control UI live event paths, so the selected Control UI session cannot receive assistant/tool progress for that external-channel run until history is reloaded.

## What This Changes

The issue asks for Control UI WebChat to live-update when observing the selected session while a Feishu DM starts the agent run.

## Best Possible Solution

Define a session-scoped external-run observer contract that streams assistant/tool progress only to clients subscribed to the exact selected session, with tests proving no unrelated live-event leakage.

## Reproduction Assessment

Yes at source level: a Feishu DM enters as channel "feishu", is registered as not Control-UI-visible, and then fails the gateway live broadcast gates for the selected session. I did not execute a live Feishu run in this read-only pass.

## Solution Assessment

No, not as a broad visibility flip. The maintainable solution is a selected-session observer path that preserves the hidden external-run policy for unrelated Control UI clients.
