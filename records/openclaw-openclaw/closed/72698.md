## Summary

This should close as ClawHub/plugin-shaped work rather than OpenClaw core. Current main already exposes prompt-build and bootstrap hook surfaces that can inject shared, stable system-prompt content for all agents, while VISION.md says optional capability should usually live in plugins unless a missing extension API blocks it.

## What This Changes

Requested support for one centrally maintained global bootstrap prompt file that is appended to every agent session without editing each agent workspace.

## Best Possible Solution

Keep OpenClaw core on the existing generic prompt-hook and bootstrap-hook contracts, and implement a ClawHub plugin or managed hook that reads a configured central markdown file and returns it as appendSystemContext for every eligible agent run.

## Reproduction Assessment

Not applicable; this is a feature request rather than a bug report. The relevant verification path is source/docs inspection of prompt assembly, bootstrap hooks, and plugin hook contracts on current main.

## Solution Assessment

Yes, the plugin/hook path is the narrowest maintainable direction. It satisfies the central-file use case without adding a new core-wide prompt policy knob, and current APIs already provide stable system-prompt insertion points.
