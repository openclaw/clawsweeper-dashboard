## What This Changes

Request that Discord multi-account startup start the configured default/primary account before specialist accounts while preserving the existing stagger.

## Best Possible Solution

Keep https://github.com/openclaw/openclaw/issues/77429 as the canonical issue and implement defaultAccount-first Discord startup ordering there with focused regression coverage.

## Reproduction Assessment

Yes: current source gives a high-confidence reproduction by configuring enabled Discord accounts such as `billy`, `farber`, and `main` with `channels.discord.defaultAccount: "main"`; the sorted account list gives `main` a nonzero startup delay. ## Solution Assessment
