## Summary

Current `main` already carries the Slack context plumbing this issue asked for, and the same implementation is present in release `v2026.4.22`: inbound Slack messages preserve `MessageThreadId`/`ReplyToId`, set `NativeChannelId` from the Slack event payload, and Slack's tool-context builder converts that into `currentChannelId`/`currentThreadTs` for message actions like reactions and uploads.

## Evidence

- **Inbound Slack context now preserves channel and thread ids:** `prepareSlackMessage` finalizes Slack inbound context with `ReplyToId`, `MessageThreadId`, and `NativeChannelId: message.channel`, which is the exact missing data described in the report.
  - file: [extensions/slack/src/monitor/message-handler/prepare.ts:684](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/extensions/slack/src/monitor/message-handler/prepare.ts#L684)
  - command: `nl -ba extensions/slack/src/monitor/message-handler/prepare.ts | sed -n '680,740p'`
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Slack tool context uses DM native channel id fallback:** `buildSlackThreadingToolContext` falls back to `NativeChannelId` when `To` is `user:...`, so Slack DMs/Socket Mode contexts still get a concrete `currentChannelId`; it also carries `MessageThreadId ?? ReplyToId` into `currentThreadTs`.
  - file: [extensions/slack/src/threading-tool-context.ts:22](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/extensions/slack/src/threading-tool-context.ts#L22)
  - command: `nl -ba extensions/slack/src/threading-tool-context.ts | sed -n '1,80p'`
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Slack channel plugin wires the tool-context builder into runtime threading:** The Slack channel plugin registers `buildToolContext: (params) => buildSlackThreadingToolContext(params)` in its threading adapter, so the fixed context is actually used by Slack message actions.
  - file: [extensions/slack/src/channel.ts:603](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/extensions/slack/src/channel.ts#L603)

- **Slack action runtime consumes currentChannelId/currentThreadTs for outbound actions:** `handleSlackAction` only auto-injects thread context when `currentChannelId` and `currentThreadTs` are present and match the Slack target, which is the path used by sends/uploads after the tool context is built.
  - file: [extensions/slack/src/action-runtime.ts:94](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/extensions/slack/src/action-runtime.ts#L94)
