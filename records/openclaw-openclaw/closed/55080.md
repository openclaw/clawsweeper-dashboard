## Summary

Keep open. Current main still exposes `openclaw/plugin-sdk` as a deliberately tiny root entry, and it does not re-export `stringEnum`, `optionalStringEnum`, `jsonResult`, or `formatErrorMessage`. Those helpers remain available from focused subpaths such as `openclaw/plugin-sdk/core`, `channel-actions`, `agent-runtime`, and `error-runtime`, and docs now recommend subpath imports, but that does not resolve the reported backwards-compatibility break for external plugins importing from the root SDK barrel. The linked broader compatibility issue #55533 closed after documenting policy, and the external `stepandel/openclaw-linear` PR was a workaround/migration signal, not an OpenClaw core fix.

## Best Possible Solution

Keep this issue open until OpenClaw either restores a tested backwards-compatible root import path for the affected legacy helpers, with deprecation guidance to focused subpaths, or records an explicit maintainer decision that root `openclaw/plugin-sdk` helper imports are unsupported and closes with a migration path. The likely code surface is `src/plugin-sdk/index.ts` plus the plugin-sdk export/contract tests and migration docs.
