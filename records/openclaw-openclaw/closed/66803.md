## Summary

Keep open. Current `main` still hardcodes the two local browser startup/CDP readiness windows the issue asks to make configurable, and the public config schema/docs only expose remote CDP timeout knobs, not local launch-time readiness settings.

## Evidence

- **Issue request matches current hardcoded local CDP wait:** The issue asks for configurable CDP launch/readiness timeouts because low-end hardware hits `Chrome CDP websocket not reachable after start`. Current code still throws that exact failure after waiting on a hardcoded post-launch window.
  - file: [extensions/browser/src/browser/server-context.availability.ts:164](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/extensions/browser/src/browser/server-context.availability.ts#L164)
