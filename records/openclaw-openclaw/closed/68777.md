## Summary

Current `main` already covers the reported missing-Control-UI path: packaging now requires built dashboard assets, npm release validation rejects tarballs without `dist/control-ui` payloads, and gateway startup/source flows auto-build or clearly warn when assets are absent. The latest provided release tag appears to predate these changes, so I could not tie the fix to a shipped release.

## Evidence

  - file: [scripts/openclaw-prepack.ts:71](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/scripts/openclaw-prepack.ts#L71)
