## Summary

Keep open: current main and the latest release still have the Control UI hidden-subagent pagination/count path, and https://github.com/openclaw/openclaw/pull/89323 is the active candidate fix. The issue remains necessary until that PR lands, is replaced, or maintainers reject the direction.

## What This Changes

The issue reports that the Control UI chat session picker paginates raw session rows including spawned subagent sessions, leaving real chat sessions buried and Load more ineffective.

## Best Possible Solution

Land or explicitly reject the bounded UI pagination/count fix in https://github.com/openclaw/openclaw/pull/89323, while keeping CLI nextOffset parity and optional subagent-reveal controls separate.

## Reproduction Assessment

Yes, at source level: current main filters inactive spawned/subagent rows after raw sessions.list pagination and loads only one raw page per Load More click, so a hidden-only page can leave the visible picker unchanged. I did not run a live browser reproduction because this cleanup review is read-only.

## Solution Assessment

Yes, the active UI PR path is the best current fix shape: advance through hidden-only pages and show visible-row counts without adding Gateway, config, CLI, or plugin API surface. The CLI cursor PR is useful parity work but not a standalone resolution for this Control UI bug.
