## Summary

Keep open: this is a focused security hardening and the patch shape looks correct, but it intentionally removes a shipped write-scoped HTTP kill path and still needs maintainer acceptance plus real Gateway proof or override before merge.

## What This Changes

The branch removes requester-header write-scoped HTTP session kills, requires `sessions.delete`/admin scope, routes through `killSubagentRunAdmin`, and updates the HTTP handler tests.

## Best Possible Solution

Land the admin-only HTTP kill path after maintainer acceptance of the compatibility break and either redacted live Gateway proof or an explicit proof override.

## Reproduction Assessment

Yes. Current main and `v2026.6.6` show the requester header selecting the write-scoped `sessions.abort` path, and the removed test case encoded that success path.

## Solution Assessment

Yes, with maintainer acceptance. Because HTTP auth has no authenticated binding between the caller and the requester session key, requiring the existing admin `sessions.delete` scope is the narrowest safe fix; preserving non-admin kills would need a new proof mechanism.
