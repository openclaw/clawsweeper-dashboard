## What This Changes

The issue reports that Docker system context advertises `/root/.openclaw/workspace` instead of the configured `OPENCLAW_WORKSPACE_DIR=/home/node/.openclaw/workspace`.

## Best Possible Solution

Track and fix the remaining env-to-workspace prompt mismatch in https://github.com/openclaw/openclaw/issues/66786 with a narrow workspace resolver/startup contract change and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: set `OPENCLAW_WORKSPACE_DIR` without `agents.defaults.workspace`, then `resolveAgentWorkspaceDir` still falls through to config/default home paths and the system prompt renders that fallback.

## Solution Assessment
