## Summary

Current main already solves the media-preview auth problem this PR targets. The shipped path resolves Control UI media credentials from the hello-issued paired device token first, then falls back to the configured token/password, and assistant-media preview URLs include that resolved token. Server-side assistant-media auth accepts query tokens and paired operator device tokens, with checked-in regression coverage and release evidence in v2026.4.24.

## Best Possible Solution

Close this PR as already implemented. Keep the shipped shared Control UI auth resolver plus hello-issued paired-device token media path; if a fresh failure exists, it should be reported as a new narrow bug against current v2026.4.24-or-newer behavior with exact auth mode, browser context, generated-media source, and request/response details.
