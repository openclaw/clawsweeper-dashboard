## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Gate Codex picker callbacks with command auth:** `extensions/telegram/src/bot-handlers.runtime.ts:2620-2621`
  - body: The new cdx callback path renders and edits Codex picker messages after the coarse inline-button callback gate, but it never performs the command-level authorization check used by the existing model callback path. In a group or DM configuration where inline buttons are allowed more broadly than `/codex` commands, a user who should not be able to invoke the Codex command surface can still tap cdx buttons and render permission/plugin inventory menus. Please run the same command-style authorization preflight before handling cdx callbacks, and add a regression test for an unauthorized callback sender.
  - confidence: 0.88
- **[P2] Include the global plugin enable flag in picker state:** `extensions/telegram/src/codex-picker-buttons.ts:204-209`
  - body: The dynamic enable/disable picker only checks each entry's `enabled` value, but `/codex plugins list` treats a plugin as ON only when `codexPlugins.enabled` is true and the entry is not false. When the global flag is off, the disable picker will still list entries as enabled and the enable picker can omit the entries needed to turn the policy back on. Pass the global flag into the picker and filter/label from the same effective state as `formatPluginList`.
  - confidence: 0.9
- **[P2] Make advertised plugin menu commands real:** `extensions/codex/src/command-handlers.ts:248-249`
  - body: The top-level picker fallback advertises `/codex plugins menu`, but the plugins branch still delegates directly to `handleCodexPluginsSubcommand`, which treats `menu` and the button's `/codex plugins help` leaf as unknown subcommands. That breaks typed users and non-Telegram/presentation fallback users who follow the advertised command. Add explicit `menu` and `help` handling, or change the fallback/buttons to only point at supported commands.
  - confidence: 0.86
