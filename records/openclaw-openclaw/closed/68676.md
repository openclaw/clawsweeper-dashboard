## Summary

Current `main` no longer has the `madge@8` dependency conflict this PR was working around. The repo still uses TypeScript 6, but the import-cycle check now uses the TypeScript compiler directly, `madge` is absent from the current manifest and lockfile, and the latest shipped release `v2026.4.23` already reflects that state without a `legacy-peer-deps` workaround.
