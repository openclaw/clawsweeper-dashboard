## What This Changes

Fix Discord reply/reference turns so a restart or missing referenced message cannot cause the assistant to answer from unrelated stale session context.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82801 for the LLM-boundary context preservation, then coordinate any remaining missing-reference hydration or fail-closed behavior with https://github.com/openclaw/openclaw/pull/81226 or a narrow follow-up.

## Solution Assessment

Yes for the immediate fix path: preserving active reply metadata at the LLM boundary is narrow and matches the observed stale-context answer. The broader missing-reference fail-closed policy should remain coordinated with the Discord hydration/backfill work rather than folded into this issue cleanup.
