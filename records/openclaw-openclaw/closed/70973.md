## Summary

Keep open. Current `main` does not show a landed Gemini CLI restart/session-persistence fix: the Google CLI backend still lacks the Claude/Codex-style auth-epoch ownership hooks, local CLI credential fingerprinting still excludes `google-gemini-cli`, the docs only promise restart resume behavior for `claude-cli`, and the issue already has an open linked PR (#71076), which is strong evidence the maintainers still consider this unresolved.
