## Summary

Keep this PR open. Current main has a bundled Microsoft Foundry provider implementation, but it does not have the Azure AI Foundry/Microsoft Foundry provider guide or docs navigation entry that this PR is trying to add. The contribution is still potentially useful, but it needs maintainer review and likely rework against the newer `microsoft-foundry` provider instead of the older manual adapter-focused setup.

## Best Possible Solution

Keep the PR open and ask for a rebase/rework into the current provider docs shape: add a discoverable Foundry provider guide and docs navigation entry, align setup with the bundled `microsoft-foundry` provider and current auth-profile semantics, and explicitly validate whether Claude on Azure AI Foundry belongs in this guide via generic `models.providers`/`anthropic-messages` config or a separate note.
