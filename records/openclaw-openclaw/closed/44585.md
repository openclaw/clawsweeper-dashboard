## Summary

Current main already resolves `/status` and `session_status` context limits with the active provider plus model, not a bare model id, and the changelog says that fix shipped in 2026.4.7.

## Evidence

  - file: [src/status/status-message.ts:594](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/status/status-message.ts#L594)
  - command: `nl -ba src/status/status-message.ts | sed -n '592,675p'`
- **Context resolver prefers explicit provider config:** `resolveConfiguredProviderContextTokens` reads `models.providers.<id>.models[].contextWindow/contextTokens`, and `resolveContextTokensForModel` returns that explicit provider/model override before falling back to shared cache entries. That prevents duplicate bare model ids like `glm-5` from inheriting another provider's 128k window.
  - file: [src/agents/context.ts:323](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/agents/context.ts#L323)
