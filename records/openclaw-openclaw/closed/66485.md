## Summary

Current main already ships the useful Control UI context-usage fix, and the remaining branch diff is superseded by the shipped implementation or would regress current behavior if merged as-is.

## What This Changes

The PR changes Control UI `/usage` to compute context percent from `totalTokens` and reloads chat history when an agent compaction event completes.

## Best Possible Solution

Keep the shipped canonical usage snapshot and compaction counter reset behavior; any new post-compaction transcript-refresh bug should be handled as a narrower, session-scoped follow-up rather than merging this stale branch.

## Reproduction Assessment

Yes. Source inspection reproduces the branch regressions: valid rows with `totalTokens` but missing `totalTokensFresh` lose `/usage` context, and sessionless compaction events would reload the currently visible chat without proving ownership.

## Solution Assessment

No. Current main's shipped `/usage` snapshot semantics are safer, and any compaction-history refresh should be scoped to the active session/run instead of reloading on all sessionless compaction events.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Treat missing freshness as usable in /usage:** `ui/src/ui/chat/slash-command-executor.ts:384-387`
  - body: The new guard requires `totalTokensFresh === true`, but current UI semantics treat only explicit `false` as stale. Older or compatibility-shaped session rows with valid `totalTokens` and no freshness flag would lose the context percentage even though current main and the chat notice display it.
  - confidence: 0.92
- **[P2] Scope compaction reloads to the active session:** `ui/src/ui/app-gateway.ts:451-453`
  - body: The `!sessionKey` branch reloads the visible chat for any sessionless completed compaction event. Because `loadChatHistory` mutates active chat state, an unrelated compaction can interrupt the active session; reload only when the event is tied to the active session or active run.
  - confidence: 0.86
