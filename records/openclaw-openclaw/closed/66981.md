## Summary

Current main contains targeted existing-session browser-tool changes that match this Windows `profile="user"` timeout report: the tool now warns agents not to send unsupported per-call `timeoutMs` overrides for Chrome MCP actions, the route rejects that bad shape explicitly, existing-session manage actions get a longer default timeout, and a new browser-automation skill/doctor flow was added for multi-step recovery.

## Evidence

- **Browser tool now warns agents about existing-session timeout misuse:** The current browser tool description explicitly tells agents using `profile="user"` / other existing-session profiles to omit per-call `timeoutMs` on `act:type`, `evaluate`, `hover`, `scrollIntoView`, `drag`, `select`, and `fill`, and points multi-step flows at the bundled browser-automation skill.
  - file: [extensions/browser/src/browser-tool.ts:428](https://github.com/openclaw/openclaw/blob/6f72b74cecf8d1fec5c147ae932161bf0f37a68b/extensions/browser/src/browser-tool.ts#L428)

The v2026.4.23 code did not have this logic.
