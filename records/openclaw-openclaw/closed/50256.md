## What This Changes

The PR filters `runDaemonRestart` stale cleanup candidates so the active runtime PID is not passed to `terminateStaleGatewayPids`, with regression tests for mixed and self-only stale PID lists.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Yes. Solving this at restart-health and Windows stale-PID discovery is broader and more maintainable than adding a second lifecycle-only guard, and that path shipped in v2026.4.5.
