## Summary

Current main does not implement the proposed fix. The OpenAI-compatible `/v1/chat/completions` SSE path still forwards `delta` from assistant events and skips empty content, while the upstream embedded runner can emit `replace: true` events with `delta: ""` and the corrected full `text`. The two related implementation attempts, #63635 and #63894, were closed unmerged, so they do not supersede or resolve the remaining bug.

## Best Possible Solution

Keep this issue open and implement the focused gateway fix: in `src/gateway/openai-http.ts`, track the assistant text already sent to the SSE client and, when assistant events carry authoritative `text` or `replace: true`, derive the next OpenAI delta from that cleaned full text instead of blindly forwarding `delta`. Add a regression test in `src/gateway/openai-http.test.ts` that simulates the upstream `replace: true`, `delta: ""`, `text: <correct full content>` path caused by split `<final>` stripping, verifying no leaked tag remnants, no missing lines, and no duplicated content. The closed PRs #63635 and #63894 are good implementation references but should not close this issue unless equivalent code lands on main.
