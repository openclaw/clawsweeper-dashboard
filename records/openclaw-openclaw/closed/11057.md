## Summary

Keep this issue open. Current main still treats Anthropic Opus/Sonnet 4.x 1M context as an explicit `params.context1m: true` beta path, and setup-token-style `sk-ant-oat-*` browser/session auth still strips the 1M beta header and falls back. The custom-provider workaround in the discussion is useful, but it does not resolve the built-in Claude Max/browser-session request; adjacent #66766/#66862 cover no-flag context reporting and do not supersede this auth/support boundary.

## Best Possible Solution

Keep this issue open until maintainers either implement and test native Opus 4.6 1M support for Claude Max/browser-session auth in the built-in Anthropic/Claude CLI path, or explicitly document that Opus 4.6 1M is API-key/custom-provider/eligible-Claude-login-only and that setup-token-style browser tokens fall back. Coordinate with #66766/#66862 so no-flag context reporting and actual request-header/auth behavior do not drift.
