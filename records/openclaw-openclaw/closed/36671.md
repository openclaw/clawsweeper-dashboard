## Summary

Current main has landed several related hook surfaces, but the umbrella request is not fully implemented. The public hook contract still does not expose the requested post-LLM/final-response interception hooks, and `PluginHookAgentContext` still lacks the richer sender/account/owner/provenance identity context the issue asks for.

## Evidence

- **Public hook list lacks the requested interception hooks:** `src/plugins/hook-types.ts:56`-`85` defines the exported hook names. It includes `before_model_resolve`, `before_prompt_build`, `before_tool_call`, `after_tool_call`, `before_message_write`, `before_dispatch`, and `reply_dispatch`, but there is no `after_llm_call` or `before_response_emit` hook in the public contract.
  - file: [src/plugins/hook-types.ts:56](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/src/plugins/hook-types.ts#L56)

  - sha: [7d4718373698](https://github.com/openclaw/openclaw/commit/7d471837369885c12d540d6bafa319ff41129b89)

It does not include sender identity, account/conversation ids, owner-vs-non-owner flags, group/DM state, or provenance-chain data.
