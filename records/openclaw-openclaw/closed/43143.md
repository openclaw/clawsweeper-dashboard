## Summary

Current main now explicitly documents and enforces the 10-open-PR policy that this issue asked about. The limit is a hard contributor policy, docs/size carveouts are not implemented, and the maintained exception path is coordination with maintainers plus the existing override label.

## Best Possible Solution

Keep the current documented hard limit and maintainer-coordination path. If maintainers later want docs-only or XS/S exceptions, update `CONTRIBUTING.md`, `.github/workflows/labeler.yml`, and `scripts/github/barnacle-auto-response.mjs` together so the policy, labels, and automation remain consistent.
