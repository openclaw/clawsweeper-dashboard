## Summary

This proposal fits OpenClaw's plugin/skill layer rather than core. Current main already provides the core multi-agent primitives (`sessions_send`, `sessions_spawn`, `subagents`, `session_status`) plus background-task state tracking and notifications, while the project vision explicitly keeps optional capability out of core, sends new skills to ClawHub first, and rejects heavy orchestration layers that duplicate existing agent/tool infrastructure.

## Evidence

- **Vision keeps optional capability out of core:** `VISION.md` says core should stay lean, optional capability should usually ship as plugins, new skills should go to ClawHub first, and OpenClaw will not merge heavy orchestration layers or default agent-hierarchy frameworks.

- **Plugin SDK already exposes the right extension seams:** The public SDK supports hooks, HTTP routes, gateway methods, CLI commands, and background services, which is enough surface for a signal bus, status dashboard, or observability layer outside core.
  - file: [docs/plugins/sdk-overview.md:92](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/docs/plugins/sdk-overview.md#L92)
