## Summary

## What This Changes

Adds a command cron payload that spawns an executable without an agent turn and wires it through CLI, agent-tool schema, gateway protocol validation, persistence, timer execution, output delivery, tests, and changelog.

## Best Possible Solution

Land only after command payloads are first-class across backend, CLI, Control UI, docs, and real privileged-command proof, with explicit maintainer/security acceptance for scheduled local process execution.

## Solution Assessment

No. The runner shape is plausible, but the proposed merge path is incomplete until UI compatibility, command edit parity, docs/proof, and command-execution review are handled together.
