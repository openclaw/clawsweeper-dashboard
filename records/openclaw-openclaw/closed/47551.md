## Summary

Current `main` already preserves the parent logger threshold when a child logger is created without an explicit level, and it has regression tests covering that behavior. This PR is redundant against current `main`.

## Evidence

- **Current implementation keeps parent minLevel:** `getChildLogger()` now computes `minLevel` as `opts?.level ? levelToMinLevel(opts.level) : base.settings.minLevel`, then passes that to `getSubLogger()`, so an unset child level inherits the parent threshold instead of overriding it with `undefined`.
