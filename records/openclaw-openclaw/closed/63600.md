## Summary

Close as ClawHub/community plugin docs work. The PR is a LocalAI-specific provider guide for a provider not bundled on current main; the related LocalAI provider plugin PR #63613 was already closed as the proper ClawHub/community plugin path. Current OpenClaw docs and SDK already support external provider plugins and explicit OpenAI-compatible provider config without a missing core API.

## Best Possible Solution

Move the LocalAI provider guide and any auto-discovery/onboarding implementation into an external LocalAI provider plugin published through ClawHub/npm, reusing the existing provider plugin API and self-hosted OpenAI-compatible setup helpers. Keep OpenClaw core docs focused on generic provider configuration and plugin extension points; open a targeted SDK issue only if the external plugin uncovers a missing generic API.
