## Summary

Current main already fixes the reported OpenAI Realtime GA breakage in the bundled OpenAI realtime voice provider, and the fix is traceable to the merged GA bridge PR. The stable v2026.5.7 release predates the fix, but I could prove it is present on main and first shipped in the v2026.5.9-beta.1 prerelease.

## What This Changes

The issue reports that Voice Call's OpenAI realtime bridge in openclaw@2026.5.7 still uses the deprecated beta WebSocket header, beta session payload, old model default, and old event names against the GA Realtime API.

## Best Possible Solution

Keep the provider-local GA Realtime bridge implementation on main and direct any remaining failures to a new report with logs from current main or a release containing the fix.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the old v2026.5.7 release path from the issue logs and the canonical fixed report, but no failing current-main reproduction remains because current main now sends the GA session payload and the merged fix includes live OpenAI Realtime smoke proof.

## Solution Assessment

Is this the best way to solve the issue? Yes: the fix stays inside the OpenAI provider, removes the beta-only native OpenAI path, preserves Azure deployment compatibility separately, and adds regression plus live smoke coverage.
