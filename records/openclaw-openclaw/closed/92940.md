## What This Changes

Land a candidate-scoped, auth-focused coalescer that preserves first-occurrence structured logs and onFallbackStep behavior, with log-capture tests and redacted real expired-token proof.

## Reproduction Assessment

Yes. By source inspection, expired-token text classifies as auth, current main calls the warn-level fallback decision logger per candidate, and the default auth skip cache remains disabled.

## Solution Assessment

No. A process-global key of decision/provider/reason is a plausible mitigation but too broad; the maintainable fix should coalesce only true duplicate auth-class candidate observations while preserving candidate-model diagnostics.
