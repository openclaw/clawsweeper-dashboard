## Summary

Current `main` already serves the Control UI at `/` or the configured `gateway.controlUi.basePath`, and it includes explicit installed/global-package asset resolution for `dist/control-ui`, so the March 2026 localhost 404 report is already addressed.

## Evidence

- **Root-mounted Control UI now claims `/`:** `classifyControlUiRequest` returns `{ kind: "serve" }` for read requests when `basePath` is empty, so `GET /` is treated as the Control UI entrypoint instead of falling through to a 404. It also redirects configured base paths like `/openclaw` to `/openclaw/`. See `src/gateway/control-ui-routing.ts:18-49`.
  - file: [src/gateway/control-ui-routing.ts:18](https://github.com/openclaw/openclaw/blob/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b/src/gateway/control-ui-routing.ts#L18)
  - sha: [b98a6a46fb54](https://github.com/openclaw/openclaw/commit/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b)
- **HTTP handler serves `index.html` and SPA fallback:** `handleControlUiHttpRequest` uses that route classification, resolves the UI root, and serves `index.html` for `/` plus SPA fallback for unknown read-only routes; it only returns 404 when assets are truly unavailable or the path is unsafe. See `src/gateway/control-ui.ts:684-883`.
  - file: [src/gateway/control-ui.ts:684](https://github.com/openclaw/openclaw/blob/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b/src/gateway/control-ui.ts#L684)

See `src/infra/control-ui-assets.ts:101-122` and `src/infra/control-ui-assets.ts:188-247`, used by `src/gateway/server-control-ui-root.ts:32-61`.
  - file: [src/infra/control-ui-assets.ts:101](https://github.com/openclaw/openclaw/blob/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b/src/infra/control-ui-assets.ts#L101)
