## Summary

Current `main` hardens Telegram command-menu sync so `deleteMyCommands` failures no longer escape as unhandled rejections, which matches the restart-loop symptom reported in #66880.

## Evidence

- **Delete failure no longer escapes the sync helper:** `syncTelegramMenuCommands` now turns `deleteMyCommands` errors into `false` for the empty-menu path, logs `telegram: deleteMyCommands failed; skipping empty-menu hash cache write`, and returns without rethrowing.
  - file: [extensions/telegram/src/bot-native-command-menu.ts:257](https://github.com/openclaw/openclaw/blob/55d3e7bc473e34883a2d2cce51ce904b87c32eed/extensions/telegram/src/bot-native-command-menu.ts#L257)

  - file: [extensions/telegram/src/bot-native-command-menu.ts:317](https://github.com/openclaw/openclaw/blob/55d3e7bc473e34883a2d2cce51ce904b87c32eed/extensions/telegram/src/bot-native-command-menu.ts#L317)
