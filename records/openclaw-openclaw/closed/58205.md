## Summary

Current `main` already handles completed-subagent follow-up reactivation in the gateway send path, with explicit ended-run gating and tests for both `agent` and `sessions.send`. That makes this PR's extra `sendControlledSubagentMessage` replacement redundant, and the review discussion correctly identified that doing it unconditionally in this helper would be riskier for still-running subagents.

## Evidence
