## Summary

Keep open: current `main` now includes a bundled MiniMax speech provider, but it still sends raw float `voice_setting` values and keeps overlapping generic directive keys, so the PR's core fix is not already implemented on `main`.

## Evidence

- **MiniMax speech provider is present on current main:** `extensions/minimax/index.ts` registers `buildMinimaxSpeechProvider()`, and `extensions/minimax/openclaw.plugin.json` declares `contracts.speechProviders: ["minimax"]`, so the old "no provider registered" part is already covered on `main`.
  - file: [extensions/minimax/index.ts:28](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/extensions/minimax/index.ts#L28)
  - command: `nl -ba extensions/minimax/index.ts | sed -n '1,80p'; nl -ba extensions/minimax/openclaw.plugin.json | sed -n '1,120p'`
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Current main still sends floats through to MiniMax:** `extensions/minimax/tts.ts` serializes `speed`, `vol`, and `pitch` directly into `voice_setting` without integer coercion. That means the PR's advertised fix for MiniMax's int64 mismatch is not present on `main`.
  - file: [extensions/minimax/tts.ts:58](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/extensions/minimax/tts.ts#L58)
  - command: `nl -ba extensions/minimax/tts.ts | sed -n '1,260p'`
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Current tests codify float behavior rather than integer coercion:** `extensions/minimax/speech-provider.test.ts` expects `speed` directive parsing to return `1.5`, and later expects the outgoing request body to contain `voice_setting.speed === 1.5`. That is the opposite of this PR's integer-coercion behavior.
  - file: [extensions/minimax/speech-provider.test.ts:138](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/extensions/minimax/speech-provider.test.ts#L138)
  - command: `nl -ba extensions/minimax/speech-provider.test.ts | sed -n '1,360p'`
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Generic directive collisions still exist on main:** MiniMax still claims bare `voice`, `model`, and `speed` tokens in `extensions/minimax/speech-provider.ts`, while OpenAI claims bare `voice`/`model` with `autoSelectOrder: 10` and ElevenLabs claims bare `model`/`speed` with `autoSelectOrder: 20`. MiniMax is `autoSelectOrder: 40`, so the PR's namespacing follow-up remains relevant.
  - file: [extensions/minimax/speech-provider.ts:97](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/extensions/minimax/speech-provider.ts#L97)

  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)

Current `main` still shows the older float/generic behavior, so the PR is not stale noise.
