## Summary

Keep open: the schema rejection is real from source inspection, but current main does not define `directLike` as a Discord runtime/docs/config contract, so this is a new Discord channel mode/config request rather than a schema-only drift bug.

## What This Changes

The issue asks for the Discord bundled plugin/config schema to accept a `directLike` guild-channel override consistently with claimed runtime support.

## Best Possible Solution

If maintainers want this mode, define the Discord direct-like guild-channel semantics first, then implement it across config types/schema/generated metadata, channel-config resolution, direct-vs-channel route/context decisions, docs, and regression tests.

## Reproduction Assessment

Yes for the schema rejection by source inspection: the Discord guild-channel Zod schema and generated metadata are strict and omit `directLike`. No high-confidence path exists for the claimed existing runtime support because current main has no source or docs contract for that key.

## Solution Assessment

No. A manifest/schema-only addition is not the best fix unless maintainers confirm the intended behavior; the maintainable path needs the runtime semantics wired at the same time.
