## Summary

Current main contains targeted fixes for the reported post-update Gateway/TUI slow paths: external CLI auth discovery is scoped to configured providers, `chat.history` no longer waits on model catalog discovery, local TUI clients wait for Gateway event-loop readiness within the configured handshake budget, and restart deferral is bounded. These fixes are on `main` only; no release tag contains the proof commit yet.

## What This Changes

Fix the regression where updating OpenClaw made the Gateway disconnect, TUI history load slowly, and restart/status commands hang or respond very slowly.

## Best Possible Solution

Keep the current main fix set, ship it in the next stable release, and ask for a fresh diagnostics bundle only if the same symptoms reproduce on a version containing the proof commit.

## Reproduction Assessment

Yes for the main slow paths: current tests cover opencode-only configs avoiding unrelated external CLI credential probes and `chat.history` returning without model catalog discovery. The exact reporter machines were not live-reproduced, but the code-level regressions have focused coverage.

## Solution Assessment

Yes. The current main fixes are narrow and maintainable: they scope expensive auth discovery, remove model-catalog work from history reads, align client handshake readiness, and bound restart drain behavior instead of adding broad startup migrations or release-specific workarounds.
