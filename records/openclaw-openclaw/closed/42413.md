## Summary

Current `main` already auto-watches the active Gateway config file and applies hot reloads or restart-required reloads for the running Gateway, including the launchd/LaunchAgent service path on macOS.

## Evidence

  - file: [src/gateway/config-reload.ts:271](https://github.com/openclaw/openclaw/blob/7d9172d5e093435d37ed3e00e476a4eaf84a252b/src/gateway/config-reload.ts#L271)

## Close Comment

Closing as implemented on main. Codex review found that the running Gateway already watches the active config file and automatically evaluates config changes for hot reload vs restart (`src/gateway/server.impl.ts:859`, `src/gateway/config-reload.ts:271`, `src/gateway/config-reload.ts:382`). For service-managed installs, restart-required changes already flow through the Gateway restart path, and the run loop has explicit `launchd` handling for LaunchAgent-supervised restarts (`src/cli/gateway-cli/run-loop.ts:82`, `src/cli/gateway-cli/run-loop.ts:230`). The docs also describe this behavior: the Gateway watches the active config path with default `gateway.reload.mode="hybrid"`, and the macOS Gateway runs as a LaunchAgent service (`docs/gateway/index.md:66`, `docs/platforms/mac/bundled-gateway.md:27`). This behavior is present in the released `v2026.4.22` snapshot, so there is not an open maintainer cleanup item left to land here.
