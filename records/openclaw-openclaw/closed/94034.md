## Summary

Keep open: this is a focused bug-fix PR, but the patch only checks `stdout`, so a stdin-piped invocation can still enter the interactive Clack wizard and leave the central reported failure unresolved.

## What This Changes

The PR adds a non-TTY guard to the shared configure command helper so bare `openclaw config` and `openclaw configure` print config subcommand guidance and exit before running the wizard.

## Best Possible Solution

Land a shared configure-wizard TTY guard that requires both stdin and stdout, preserves interactive TTY behavior, points to non-interactive config subcommands, and includes regression tests plus real command proof.

## Reproduction Assessment

Yes. Source inspection shows the issue's stdin-pipe repro can still bypass this PR's guard when stdout is a terminal, and Clack's prompt runtime reads from stdin.

## Solution Assessment

No. The shared helper is the right layer, but the guard must require both stdin and stdout TTYs and be backed by focused regression coverage and real command proof.
