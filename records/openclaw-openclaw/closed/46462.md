## Summary

Close as cannot_reproduce. The reporter followed up that the problem was caused by an `.env` path/mount configuration mistake, and current `main` still explicitly supports Docker LAN binding rather than forcing loopback.

## Evidence

- **Reporter self-resolved:** The issue comment says the final cause was an `.env` file path configuration problem affecting container file mounts, and that the problem is now solved.
- **Current bind logic maps `lan` to all interfaces:** `resolveGatewayBindHost()` returns `0.0.0.0` for `bind="lan"`, and `auto` also returns `0.0.0.0` inside containers.

- **Docker Compose defaults the gateway to `lan`:** The checked-in Compose command passes `--bind ${OPENCLAW_GATEWAY_BIND:-lan}`, so the standard Docker path is configured for host-reachable binding unless overridden.
  - file: [docker-compose.yml:29](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/docker-compose.yml#L29)
