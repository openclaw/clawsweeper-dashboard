## Summary

Current main already solves the central local/OpenAI-compatible provider need with an explicit `params.extra_body` pass-through surface, backed by source, docs, and regression coverage; the supported shape is namespaced rather than forwarding every flat agent param.

## What This Changes

Request to allow custom OpenAI-compatible local LLM providers to send arbitrary model request parameters beyond temperature and max tokens.

## Best Possible Solution

Keep the explicit `params.extra_body` pass-through as the supported API so provider request fields do not get confused with OpenClaw runtime params.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing behavior report. Source, docs, and tests on current main show the pass-through path exists via `params.extra_body`.

## Solution Assessment

Yes; the current explicit `extra_body` surface is the narrowest maintainable solution because it permits arbitrary provider request fields for OpenAI-compatible completions without leaking unrelated OpenClaw params into every request.
