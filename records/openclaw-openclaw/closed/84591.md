## Summary

Current main and shipped v2026.5.19 already handle this class of reasoning-only final turn by detecting signed thinking with no visible text and retrying with a visible-answer continuation, so the reported blank channel reply from v2026.5.18 can be closed as implemented.

## What This Changes

The issue reports DeepSeek V4 high-thinking final turns producing only `reasoning_content`/thinking blocks, causing no visible Feishu reply.

## Best Possible Solution

Keep the shipped retry-and-visible-answer continuation behavior rather than exposing raw reasoning content directly to channels.

## Reproduction Assessment

Yes. The issue includes sanitized session evidence, and current source shows the exact class of signed thinking-only assistant turn that used to have no visible text is now detected and retried before normal final delivery.

## Solution Assessment

Yes. Retrying with an explicit visible-answer continuation is safer than promoting raw `reasoning_content` into channel text, because it preserves the reasoning/privacy boundary while preventing a silent blank final reply.
