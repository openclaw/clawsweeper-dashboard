## Summary

Keep open: current main still has narrow final-tag matchers in the user-facing sanitizer and embedded streaming path, so self-closing and attributed Gemini/Gemma `<final>` variants can still leak. The issue also has open implementation PRs linked to it, which should be reviewed or replaced rather than closing the report before a fix lands.

## What This Changes

The issue reports Gemini/Gemma raw `<final>` wrapper tags leaking into user-visible WebUI/Control UI messages after an OpenClaw update.

## Best Possible Solution

Land or replace the open PRs with one constrained shared final/reasoning-tag parser across streaming, sanitizer, persistence/replay, and Control UI rendering, backed by focused regression tests and redacted real Gemini proof.

## Reproduction Assessment

Yes, at source level. Current main's sanitizer and streaming regexes leave `<final/>Hello` and attributed final tags visible, matching the reported Gemini/Gemma leak class; I did not run a live WebUI session.

## Solution Assessment

Yes for the triage outcome: keeping this issue open is the narrowest maintainable path while the linked fix PRs are reviewed, reconciled, landed, or replaced. Closing now would be premature because current main and the latest release still show the source-level gap.
