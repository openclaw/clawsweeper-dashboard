- `merge-risk: 🚨 compatibility`: The diff changes public cron payload normalization and currently risks rejecting create payloads that current main treats as an omitted optional model override.

## Summary

Keep this PR open: it targets a valid, source-proven cron update bug, but the current patch is not merge-ready because the shared normalizer change can make cron.add reject create payloads that previously omitted an empty model override, and the PR has no after-fix real behavior proof.

## What This Changes

The PR makes cron update patches accept `payload.model: null`, preserves null/empty model clears through normalization, deletes stored model overrides during payload merge, and adds normalize/applyJobPatch tests.

## Best Possible Solution

Land a patch-scoped clear implementation: cron.update accepts null/empty model to remove the stored override, cron.add continues to omit empty or invalid optional model values, and tests plus redacted terminal proof cover both paths.

## Reproduction Assessment

Yes for source reproduction: current main drops `payload.model: null` during update normalization and then ignores non-string model patches during merge. I did not run a live gateway repro because this was a read-only review.

## Solution Assessment

No: the merge-layer clear is the right direction, but applying null preservation inside the shared create/patch normalizer is too broad. The safer fix is to make null/empty model a patch-only clear signal while preserving create-time omission behavior.
