## Summary

Current `main` and the shipped `v2026.5.2` tree already keep QMD cold-start lazy: default memory remains builtin, QMD gateway startup is opt-in, and even opt-in startup uses a one-shot CLI-mode manager that returns before creating the long-lived watcher.

## What This Changes

The issue reports that TUI startup eagerly starts a QMD chokidar watcher whose default collection path can resolve to the filesystem root, freezing the terminal during a recursive scan.

## Best Possible Solution

Keep the shipped lazy QMD startup behavior; any additional root-path hardening can be tracked separately as defense-in-depth rather than keeping this startup-freeze report open.

## Reproduction Assessment

No high-confidence current-main failing reproduction remains. Source and shipped-release inspection show TUI/gateway cold start no longer initializes QMD by default, and opt-in startup uses CLI mode before watcher creation.

## Solution Assessment

Yes. The current implementation fixes the reported startup freeze at the narrow lifecycle boundary by keeping QMD lazy and using a one-shot startup path instead of adding another startup-time watcher mode.
