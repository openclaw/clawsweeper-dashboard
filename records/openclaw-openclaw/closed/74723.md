## Summary

Keep this issue open. Current main still has the reported metadata gap: the browser plugin declares runtime dependencies including `playwright-core`, but its `openclaw` package block does not opt into bundled runtime dependency staging, while the build and postbuild scripts only create per-plugin staged output and `node_modules` for plugins with `openclaw.bundle.stageRuntimeDependencies === true`. The prior Playwright runtime-shim work is present, so this is a narrower packaging follow-up rather than a duplicate of the older Playwright resolution issue.

## What This Changes

The issue requests adding browser plugin runtime dependency staging so production builds emit and stage the Playwright-backed browser runtime files and dependencies correctly.

## Best Possible Solution

Add `openclaw.bundle.stageRuntimeDependencies: true` to the browser plugin package metadata, add a small browser manifest/package regression test or extend the existing bundled-runtime smoke so this cannot regress, then validate the packaged build/runtime-deps smoke before release.
