## Summary

Keep open. The PR is aimed at a real recovery problem, but the current patch changes a helper that current main only exercises in tests, and the proposed drop-all behavior would lose valid assistant content in one `incomplete-text` case if wired into production.

## What This Changes

The PR changes `sanitizeThinkingForRecovery` so `incomplete-text` removes the last assistant message with `prefill: false` and updates the focused unit test.

## Best Possible Solution

Retarget the fix to the active outbound replay/history sanitation path, avoid assistant prefill for unfinished turns, preserve completed text blocks, and keep the broader #62322 recovery-state semantics tracked separately.

## Reproduction Assessment

Yes for the PR defects: source search reproduces that the changed helper is test-only, and the classifier/drop-all edge case follows directly from `assessLastAssistantMessage`. The broader aborted-run inconsistency is supported by the PR discussion but still needs focused live or integration reproduction after a runtime fix.

## Solution Assessment

No. The current PR is not the best fix because it changes an inactive helper and would be too broad if wired in as-is; the safer path is to update the active replay sanitizer and preserve valid assistant content.
