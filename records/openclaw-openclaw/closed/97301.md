## Label Justifications

- `P2`: The issue describes a normal-priority embedded runtime transcript consistency bug with limited but real plugin/session impact.
- `impact:session-state`: The affected behavior is whether context-engine lifecycle hooks can reconcile against the durable session transcript for the current turn.

## Summary

Keep open: current main still calls embedded context-engine `afterTurn` from an in-memory snapshot before the later session-event drain, while the Codex app-server path mirrors the transcript before its finalizer.

## What This Changes

The issue reports that the embedded context-engine host calls `afterTurn` before the session transcript jsonl is guaranteed to be flushed, unlike the Codex app-server host.

## Best Possible Solution

Give the embedded context-engine host a durable transcript-before-afterTurn contract, matching the Codex app-server behavior where feasible, with regression coverage for suppressed room-event turns.

## Reproduction Assessment

Yes from source, but not from a live run: current main still calls embedded `afterTurn` without a pre-hook transcript mirror/flush, and room-event suppression can skip the durable user-message write.

## Solution Assessment

Yes: the narrow maintainable fix is to materialize or await the current turn transcript before embedded `afterTurn`, rather than requiring every reconciling plugin to special-case a host timing gap.
