## Summary

This is a useful macOS host-integration idea, but it fits OpenClaw's external plugin/ClawHub boundary rather than core. Current main already has the needed native plugin surfaces for agent tools, optional side-effectful tools, approval hooks, and ClawHub distribution; the existing launchd code is narrowly scoped to OpenClaw's own Gateway service and does not indicate a missing generic core API.

## What This Changes

Request adding a macOS-only host-integration plugin that exposes launchd install, uninstall, list, kickstart, and logs actions for user-domain LaunchAgents.

## Best Possible Solution

Let this be built and published as a ClawHub native tool plugin, using optional tool registration and plugin approvals for the launchctl side effects. Core should only grow a new SDK/API seam later if an external launchd plugin hits a concrete missing capability that cannot be solved through the current plugin tool and hook APIs.
