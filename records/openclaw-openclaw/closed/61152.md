## Summary

Close as `clawhub`: current main does not add a core `localCommand` cron payload, but the existing plugin surface already supports provider-free cron interception before any model/backend work and exposes public process helpers for plugin-owned local command execution. Per `VISION.md`, that capability belongs in a plugin/ClawHub path rather than a new core cron payload kind.

## Evidence

- **Core cron payloads are still limited to two kinds:** The cron tool schema and core cron types only accept `systemEvent` and `agentTurn`; isolated/current/session jobs are explicitly restricted to `agentTurn` and main-session jobs to `systemEvent`.
  - file: [src/agents/tools/cron-tool.ts:28](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/agents/tools/cron-tool.ts#L28)
  - command: `nl -ba src/agents/tools/cron-tool.ts | sed -n '21,115p'`
  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **Core enforcement rejects a third payload style:** `assertSupportedJobSpec` enforces `main` => `systemEvent` and `isolated/current/session:*` => `agentTurn`, confirming there is no current core slot for `localCommand`/`hostCommand`.
  - file: [src/cron/service/jobs.ts:156](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/cron/service/jobs.ts#L156)
  - command: `nl -ba src/cron/service/jobs.ts | sed -n '156,169p'`
  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **Cron plugin hooks can short-circuit before model execution:** Both embedded and CLI cron runners call `before_agent_reply` when `trigger === "cron"` and return early if a plugin handles the turn, before the model attempt or CLI backend execution starts.
  - file: [src/agents/pi-embedded-runner/run.ts:338](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/agents/pi-embedded-runner/run.ts#L338)

- **Tests prove claimed cron runs skip the backend entirely:** Regression tests assert that a claimed cron run does not call the embedded attempt and does not invoke the CLI subprocess/preparation path.
  - file: [src/agents/pi-embedded-runner/run.before-agent-reply-cron.test.ts:23](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/agents/pi-embedded-runner/run.before-agent-reply-cron.test.ts#L23)
