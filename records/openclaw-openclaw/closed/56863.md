## Summary

Keep open. Current main does not implement the PR's central WhatsApp changes: outbound WhatsApp text/caption payloads are still sent without Baileys mention metadata, group metadata is not cached as a phone-to-JID map, quoted reply media is not downloaded into model context, and the group prompt has no WhatsApp outbound mention guidance. The PR remains a plausible implementation candidate for the open WhatsApp mentions work, especially given the discussion that the auto-reply mention path was manually confirmed while the proactive send path still needs the participant-map callback refinement.

## Best Possible Solution

Keep this PR open for review/revision. The best path is a WhatsApp-owned implementation that shares outbound mention extraction across auto-reply and message-tool/CLI sends, wires participant JID resolution through the existing group metadata/lidMapping cache, adds quoted-media context with correct MIME/size/timeout handling, updates the WhatsApp prompt/docs, and lands targeted tests. Fold in fridayck's proactive-send callback suggestion or split quoted-media into a separate lower-risk PR if maintainers want a narrower landing path.
