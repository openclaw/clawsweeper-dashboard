## Summary

Current main already implements the PR’s systemd unit-name fix. The node service sets `OPENCLAW_SYSTEMD_UNIT=openclaw-node`, and `installSystemdService` activation plus `uninstallSystemdService` now resolve their systemctl target through `resolveSystemdServiceName(env)` instead of the gateway-only helper. Regression tests cover install and uninstall targeting `openclaw-node.service`, and the linked bug was maintainer-closed as fixed on main by commit `eca9f468246e7451ab676f844818ea43da414d2a`.

## Best Possible Solution

Close this PR as implemented on main, keep the landed implementation and regression tests from `eca9f468246e7451ab676f844818ea43da414d2a`, and let the next normal release carry the fix.
