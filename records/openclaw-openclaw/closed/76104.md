## Summary

Keep open: current main still has the reported route-precedence gap, and recent comments corroborate the Feishu-plus-webchat failure on newer releases. The duplicate cluster points back to this issue as the canonical repair thread, with no open or merged PR found for the remaining work.

## What This Changes

The issue reports that `sessions_send` replies from a Feishu-originated session are delivered to webchat because stored session route fields can outrank the explicit Feishu `deliveryContext`.

## Best Possible Solution

Make explicit persisted external `deliveryContext` authoritative for sessions_send announce/direct delivery when stored session route fields point at stale or internal webchat, with regressions for Feishu DM, Feishu group, and control-ui-triggered routes.

## Reproduction Assessment

Yes. By source inspection, seed a `sessions.list` row with a Feishu session key, stored `channel: "webchat"`, and `deliveryContext: { channel: "feishu", to: "user:ou_x" }`; the announce target path normalizes stored route fields before the explicit Feishu deliveryContext.

## Solution Assessment

Yes. The narrow maintainable fix is a generic deliveryContext precedence repair for stale/internal stored routes, not a Feishu-specific branch or a new config flag.
