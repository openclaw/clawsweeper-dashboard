## Summary

Issue #41924 proposes optional ops-style health monitoring and auto-healing for agents/gateway. Current main already provides health/diagnostic primitives and limited built-in recovery hooks, while the repo vision explicitly says optional capability should usually ship as plugins and avoids heavy orchestration layers in core. Plugin APIs already support background services, hooks, gateway methods, and HTTP routes, so this fits better as a ClawHub/community plugin or external monitor built on those surfaces than as new OpenClaw core behavior.
