## Summary

Keep this PR open. The prompt-boundary hardening goal is still relevant for BlueBubbles, but the submitted patch is not merge-ready because it wraps a legacy envelope field while the active reply path still prefers raw `BodyForAgent`; it also diverges from the Discord guard for empty bodies.

## What This Changes

The PR imports `wrapExternalContent` into BlueBubbles monitor processing and wraps `baseBody` when passing the legacy `body` field to `formatInboundEnvelope(...)`.

## Best Possible Solution

Revise or replace the patch so BlueBubbles hardens the actual prompt-bound field, likely with a guarded wrapped `BodyForAgent` or structured `UntrustedContext`, while preserving raw command/dedupe fields and adding targeted monitor and reply-context tests.
