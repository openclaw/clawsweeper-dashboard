## Summary

Current main already routes gateway reload planning through source-authored config snapshots instead of runtime-materialized config, which prevents absent/default-injected fields from surfacing as false `models` / `agents.defaults.*` changes that restart the gateway.

## Evidence

- **Reload diffs compare source snapshots:** `startGatewayConfigReloader` keeps a separate compare snapshot and computes changed paths from `currentCompareConfig` to `nextCompareConfig`, not just the runtime config, which is the core fix for false diffs from materialized defaults.
  - file: [src/gateway/config-reload.ts:245](https://github.com/openclaw/openclaw/blob/4e4aeacae4a87f2fe09125d462d88cd71a18cb66/src/gateway/config-reload.ts#L245)

- **Watcher/in-process reloads use `sourceConfig`:** External snapshot reloads call `applySnapshot(snapshot.config, snapshot.sourceConfig)`, and in-process write notifications pass `compareConfig: event.sourceConfig`, so reload planning is based on authored config content instead of runtime-expanded fields.
  - file: [src/gateway/config-reload.ts:369](https://github.com/openclaw/openclaw/blob/4e4aeacae4a87f2fe09125d462d88cd71a18cb66/src/gateway/config-reload.ts#L369)

## Close Comment

Closing as implemented on main after Codex review. The current gateway reload path now plans diffs from source-authored config snapshots instead of runtime-materialized config (`src/gateway/config-reload.ts:245`, `src/gateway/config-reload.ts:369`, `src/gateway/server.impl.ts:860`), which prevents absent/default-injected fields from showing up as false reload changes. The current reload rules also classify `models` as a hot heartbeat reload and leave the broader `agents` subtree as non-restart/no-op for gateway restart planning (`src/gateway/config-reload-plan.ts:63`, `src/gateway/config-reload-plan.ts:96`), so the reported `models` / `agents.defaults.contextPruning` behavior is covered. There is explicit regression coverage in `src/gateway/config-reload.test.ts:839`, and this shipped in `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`), whose changelog notes: `Config/reload: plan gateway reloads from source-authored config instead of runtime-materialized snapshots` (`CHANGELOG.md:178`). If you still hit a fresh crash on a newer build, please open a new issue with current logs and the exact changed paths.
