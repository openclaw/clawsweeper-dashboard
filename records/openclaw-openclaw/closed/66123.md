Full review comments:

- **[P1] Preserve explicit port when normalizing ws transport URLs:** `apps/android/app/src/main/java/ai/openclaw/app/ui/GatewayConfigResolver.kt:167-173`
  - body: This omits `:80` from `transportUrl`, but the Android parser defaults a bare `ws://host` back to OpenClaw port 18789. Since this PR stores `transportUrl` and reparses it before connecting, a valid manual `ws://example.com:80` endpoint is rewritten to the wrong port and will fail to reach the intended gateway.
  - confidence: 0.89
- **[P2] Consume pairing edge only after retry actually runs:** `apps/android/app/src/main/java/ai/openclaw/app/ui/GatewayPairingRetry.kt:35-48`
  - body: `previousEnabled` becomes true before the 6s delayed retry callback fires, so if the app backgrounds during that delay the coroutine is cancelled and the edge is lost. On resume `enabled` is still true, `shouldTriggerPairingRetry(previousEnabled, enabled)` is false, and the promised one retry for the pairing-required transition never happens.
  - confidence: 0.85
