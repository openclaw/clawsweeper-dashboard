## Summary

This PR should stay open: it targets a real current-main installer bug, but the patch still drops the new boolean `cask` metadata before command generation, carries a now-stale SDK subpath reclassification, and lacks after-fix real install proof.

## What This Changes

The PR adds `cask` metadata support for skill installs, marks CodexBar as a Homebrew cask, switches the bundled summarize installer to `@steipete/summarize`, and reclassifies Matrix/Mattermost SDK subpaths.

## Best Possible Solution

Revise this PR so `cask` is parsed and propagated end to end, remove the stale SDK subpath reclassification, align hook metadata, add focused regression coverage, and land it as the fix for the linked skill installer bugs.

## Reproduction Assessment

Yes, from source inspection plus the linked issue logs. Current main still parses `raw.cask` only as a string fallback, installs brew specs as formulae, and ships manifests pointing at the reported CodexBar and summarize install sources; I did not run a live Homebrew install.

## Solution Assessment

No. The cask/npm direction is narrow and plausible, but this patch is not the best current form until parser normalization, stale SDK cleanup, hook metadata alignment, focused tests, and real install proof are addressed.

  - confidence: 0.94

  - confidence: 0.83
