## Summary

Current main already implements the requested decoupled/asynchronous agent handoff: gateway `agent` calls return an accepted `runId` immediately, callers can wait separately through `agent.wait`, and `sessions_spawn` starts child work on the `subagent` lane while returning `{ status: "accepted", runId, childSessionKey }`. The remaining hung-task timeout concern is separate and already tracked by #48488/#48690.

## Best Possible Solution

Close this broad feature request as implemented on main and keep the shipped gateway `agent` accepted-ack plus `agent.wait` design, non-blocking `sessions_spawn`, and configurable main/subagent lane concurrency. Leave the narrower hung-promise timeout work to #48488/#48690 rather than keeping this umbrella item open.
