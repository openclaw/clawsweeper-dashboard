## Summary

Current main already routes Google Gemini model requests through the guarded provider fetch path, includes the dispatcher-aware Undici fallback that matches the proxy-only/bundled-undici discussion on this issue, and the same transport hardening appears in the shipped `v2026.4.23` release.

## Evidence

- **Issue discussion matches the fixed path:** Comment `#4298109342` describes proxy-only failures after 4.14+ and specifically suspects an additional bundled-undici fetch path that was not honoring the intended proxy/dispatcher behavior. That aligns with the guarded-fetch logic and regression coverage now present.
- **Google transport uses guarded model fetch:** The Google Gemini streaming transport builds a guarded fetch before sending `streamGenerateContent`, so Google model traffic now goes through the shared dispatcher-aware transport path instead of a raw fetch call.
  - file: [extensions/google/transport-stream.ts:621](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/extensions/google/transport-stream.ts#L621)

- **Provider transport is routed through fetchWithSsrFGuard:** `buildGuardedModelFetch` wraps provider requests with `fetchWithSsrFGuard(...)` and carries the resolved dispatcher policy for model traffic, which is the transport seam Google uses.
  - file: [src/agents/provider-transport-fetch.ts:177](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/agents/provider-transport-fetch.ts#L177)
