## Close Comment

Thanks for the PR. I did a Codex review against current `main`, and this behavior is already implemented in the current browser screenshot path. The old target file from this PR, `src/browser/cdp.ts`, no longer exists; the active implementation is in `extensions/browser/src/browser/cdp.ts:83`, where full-page capture is handled by temporarily expanding the viewport only when `opts.fullPage` is true, before the screenshot call at `extensions/browser/src/browser/cdp.ts:134`. Current `main` also has regression coverage for both modes: viewport screenshots without the full-page expansion path at `extensions/browser/src/browser/cdp.screenshot-params.test.ts:90`, and full-page expansion/restoration behavior at `extensions/browser/src/browser/cdp.screenshot-params.test.ts:108`. Because `main` already implements the requested behavior through the newer browser-plugin codepath, I’m closing this as implemented on main rather than merging the older one-line patch as-is.
