## Summary

Current main and the latest release already address the reported 2026.4.27 packaged memory runtime-deps gap: `sqlite-vec` is present as a root runtime dependency, the memory vector loader still imports that module, and v2026.4.29 release notes explicitly shipped the `sqlite-vec` bundled runtime-deps fix for the same root cause tracked in #74692.

## What This Changes

Report that OpenClaw 2026.4.27 npm installs omitted `sqlite-vec` from packaged memory runtime dependencies, leaving vector memory unavailable.

## Best Possible Solution

Keep the shipped v2026.4.29/current-main dependency handling and have affected 2026.4.27 npm users update rather than manually editing legacy runtime-deps directories.

## Reproduction Assessment

Yes. The old failure path is high-confidence from the report and source: memory dynamically imports `sqlite-vec`, so a packaged install missing that dependency would report vector unavailable; current main and v2026.4.29 contain the dependency fix.

## Solution Assessment

Yes. Mirroring/providing `sqlite-vec` through the packaged runtime dependency path and keeping it root-owned is the narrow maintainable fix; adding startup-time package repair would fight the documented dependency model.
