## Summary

OfoxAI is a third-party OpenAI-compatible gateway provider request with no protected label or maintainer author signal, and current main does not already bundle it. Current OpenClaw scope and docs route optional provider integrations to external plugins on ClawHub/npm, and the current provider SDK already supports API-key auth, catalogs, onboarding config, dynamic model resolution, runtime hooks, and manifest env-var lookup without a missing core API. The PR discussion is useful as a starting point, but it also identified unresolved model-id, capability, metadata, and test gaps that should be fixed in an external OfoxAI provider plugin rather than kept as a bundled-core PR.

## Best Possible Solution

Close this bundled-provider PR and move OfoxAI to an external ClawHub/npm provider plugin using the current provider SDK. Reuse the PR as a starting point, but fix the namespaced OfoxAI model IDs, proxy compatibility hooks, dynamic model metadata, and registration tests before publishing. Open a focused OpenClaw SDK/API issue only if that external plugin hits a concrete missing core seam.
