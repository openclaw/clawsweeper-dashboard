## Summary

Current `main` has explicit live-history reload handling for both Control UI and TUI around final chat events and transcript append events, which matches this report’s missing-live-update symptom for approval-gated exec completions.

## Evidence

- **Gateway now emits transcript append events to live subscribers:** `createTranscriptUpdateBroadcastHandler` broadcasts `session.message` whenever a transcript update is persisted, giving live UIs a push signal to reload without manual refresh.
  - file: [src/gateway/server-session-events.ts:83](https://github.com/openclaw/openclaw/blob/86fa8eeb68c64eb0ae25e1eb8c6196061862a010/src/gateway/server-session-events.ts#L83)

  - file: [ui/src/ui/app-gateway.ts:456](https://github.com/openclaw/openclaw/blob/86fa8eeb68c64eb0ae25e1eb8c6196061862a010/ui/src/ui/app-gateway.ts#L456)
