## Summary

Current main already avoids the reported forced-kickstart startup path, prefers attaching to an existing healthy local gateway, and clears stale timeout/failure state after later health success. The current macOS startup path is materially different from the March 2026 report.

## Evidence

- **macOS app now attaches before touching launchd:** `startIfNeeded()` launches an async task that first calls `attachExistingGatewayIfAvailable()` and returns early if a healthy gateway is already running, instead of immediately forcing a launchd restart.
