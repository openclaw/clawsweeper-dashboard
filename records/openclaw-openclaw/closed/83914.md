## What This Changes

The issue asks the diffs viewer to isolate a single card hydration failure so later diff cards on the same page still hydrate.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84775, or a replacement, with per-card error isolation, updated served runtime asset, focused DOM coverage, and served-viewer proof.

## Reproduction Assessment

Yes, source-level: with two `.oc-diff-card` hosts, forcing the first card's `FileDiff` construction, `hydrate()`, or `applyState()` to throw exits `hydrateViewer()` before the second card and reaches the page-level `openclawDiffsError` catch. I did not execute a DOM/browser repro in this read-only review.

## Solution Assessment

Yes on direction: a per-card guard around the hydration body is the narrow maintainable fix. The mergeable fix should also keep controller state clean, update the served runtime bundle, and prove the actual browser asset path.
