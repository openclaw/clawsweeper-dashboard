## Summary

PR #54527 targets a real stale dated-file bug, and its discussion/review feedback was useful. Current main now solves the same observable problem: subsystem file writes no longer use a forever-cached child logger, the rolling file transport follows the current local date, and regression coverage verifies a long-lived subsystem logger moves from one daily log file to the next.

## Best Possible Solution

Close PR #54527 as implemented on current main. Keep the broader main implementation, which avoids stale subsystem child loggers and also makes the rolling file transport current-date aware; any remaining performance preference about per-emit child logger creation should be filed as a separate targeted optimization only if maintainers see real cost.
