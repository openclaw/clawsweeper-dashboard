## Summary

Keep open: current main still has the active-requester fallback dead end, and PR #75669 is the open implementation candidate with closing syntax.

## What This Changes

Subagent completion announcements should fall through to direct gateway delivery when active-session steering and completion fallback fail, so Telegram users receive background results without sending another message.

## Best Possible Solution

Land PR #75669 or an equivalent narrow patch that preserves successful steering and direct fallback behavior while letting fallback miss or recoverable fallback failure continue into the existing gateway delivery path with regression coverage.

## Reproduction Assessment

Yes. A focused source-level reproduction can set `expectsCompletionMessage` true, an active requester session id, `queueEmbeddedPiMessage` returning false, and missing or throwing fallback delivery; current main returns undelivered before `callGateway` is invoked.

## Solution Assessment

Yes. Falling through to the existing gateway delivery call is the narrowest maintainable fix because the delivery mechanism already exists and no new channel, config option, or product policy is needed.
