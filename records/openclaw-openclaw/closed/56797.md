## Summary

Current `main` no longer has the reported immediate-`requestPairingCode` flow. WhatsApp login is QR-based and waits for a QR/connection event before returning the login result, and the same implementation is present in the shipped `v2026.4.23` tag.

## Evidence

- **Current login flow waits on QR before succeeding:** `startWebLoginWithQr()` creates a `qrPromise`, passes an `onQr` callback into `createWaSocket()`, and then waits via `waitForQrOrRecoveredLogin()` before returning a QR login result. There is no immediate `requestPairingCode()` call in this path.
