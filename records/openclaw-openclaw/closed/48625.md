## Summary

PR #48625 is superseded by the current sandbox media policy. Current main deliberately does not trust broad host `/tmp` as a media root: it allows only OpenClaw-managed temp roots, applies alias-escape checks, and has regression coverage for traversal, symlink, and hardlink cases. The PR’s direct `/tmp` bypass also matches the review comment’s symlink-escape concern, so merging this old approach would regress the shipped security model.

## Best Possible Solution

Close this PR as superseded. Keep the current OpenClaw-managed temp-root policy and alias checks; any remaining current-main image-tool gap should be filed with a fresh repro and fixed by staging/managed-temp handling, not by allowing arbitrary `/tmp` paths.
