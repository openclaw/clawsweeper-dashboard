## Summary

Keep open: source inspection shows the patch can still replay the `msg_*` output item id that the reported OpenAI error says requires the omitted `rs_*` reasoning item, and the PR only supplies unit/type/build proof rather than a real model-switch run.

## What This Changes

The PR gates OpenAI Responses reasoning replay on `model.reasoning`, adds a regression test for switching to a non-reasoning model, and adds a changelog fix entry.

## Best Possible Solution

Sanitize OpenAI Responses replay as a coupled unit: when reasoning is not replay-safe for the target route, omit dependent Responses item IDs while preserving visible text and add both regression coverage and live model-switch proof.

## Reproduction Assessment

Yes, at source level: a prior assistant turn with `thinkingSignature` `rs_*` plus text `textSignature` `msg_*` reproduces the incompatible payload shape when reasoning is omitted but item IDs remain. I did not establish a live OpenAI/Venice run in this read-only review.

## Solution Assessment

No, not as written. The narrow fix needs to strip or suppress the dependent Responses item IDs at the same time as the reasoning items, rather than gating only `replayReasoningItems`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Drop dependent Responses item ids with reasoning:** `src/agents/openai-transport-stream.ts:997`

The fix needs to omit those replay item ids, and cover that in the test, whenever reasoning is not replayed.
