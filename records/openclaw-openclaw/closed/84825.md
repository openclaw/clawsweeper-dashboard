## Summary

Current main and the latest release now preserve heartbeat-origin state through queued follow-up execution, so the reported model override leak is already fixed and shipped.

## What This Changes

The issue reports that `agents.defaults.heartbeat.model` is honored for the initial heartbeat turn but lost by follow-up runs in the same heartbeat session, causing those runs to use the default primary model.

## Best Possible Solution

Keep the shipped v2026.6.6 behavior that carries heartbeat origin through follow-up runs and preserves normal user-turn model persistence.

## Reproduction Assessment

Yes. The issue's live trace maps to a source-level path where follow-up runs needed heartbeat trigger propagation, and current/release source now carries `isHeartbeat` through that path.

## Solution Assessment

Yes. Preserving heartbeat trigger state through the existing follow-up runner and model-selection contract is the narrow fix; it avoids a new config option and does not make heartbeat models sticky for ordinary user turns.
