## Summary

Keep open. The PR targets a real direct-Gemini `cachedContent` conflict, but the proposed managed-cache change would regress tool-calling runs: OpenClaw currently creates Gemini cached-content resources with only the system prompt, and its managed prompt-cache wrapper intentionally preserves `tools`/`toolConfig` on the live request path.

## Evidence
