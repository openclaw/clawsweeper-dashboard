## Summary

Keep open. Current main and the latest tagged release still use the legacy disabled-delivery exec-completion prompt that asks for `HEARTBEAT_OK only` without `NO_REPLY` or `<final>` guidance, and the adjacent sentinel-output PR does not cover this prompt-layer zero-payload path.

## What This Changes

The issue reports that disabled-delivery async exec completion prompts can steer models into bare silent replies that strict final-tag handling treats as empty turns.

## Best Possible Solution

Update the legacy no-output and disabled-delivery exec-completion prompts to state the existing silent terminal contract unambiguously, with focused regression coverage for strict final-tag sessions and no behavior change to structured `heartbeat_respond` mode.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: `deliverToUser=false`, legacy non-tool exec prompt, strict final-tag enforcement, and an unwrapped silent reply can produce zero visible payloads and enter the empty-response/incomplete-turn path.

## Solution Assessment

Yes. The narrowest maintainable fix is to align this prompt with the existing `NO_REPLY` final contract; the adjacent sentinel-output filtering PR is useful but not sufficient for the zero-payload prompt-layer failure.
