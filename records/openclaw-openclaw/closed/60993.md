## Summary

Current `main` already routes WhatsApp TTS through native voice-note mode, requests OpenAI `opus` for that path, preserves the voice-note flag through reply delivery, and sends WhatsApp audio as OGG Opus voice notes. The shipped `v2026.4.23` release contains the same OpenAI/voice-note routing.

## Evidence

  - file: [extensions/speech-core/src/tts.ts:810](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/extensions/speech-core/src/tts.ts#L810)

  - file: [extensions/openai/speech-provider.ts:83](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/extensions/openai/speech-provider.ts#L83)

## Risks / Open Questions

- An explicit `messages.tts.providers.openai.responseFormat` override can still force a non-Opus format, so user config can bypass the default WhatsApp-safe path.
