## Summary

The reported TLS handshake failure no longer reproduces against the current hosted docs endpoint used by `openclaw docs`. Current main still routes docs queries through the hosted MCP search tool, and live Node/curl/OpenSSL/MCP probes now complete TLS and return docs-search results, so the observable failure appears to have been a transient hosted endpoint problem rather than a remaining source-tree bug.

## What This Changes

The issue reports that `openclaw docs <query>` on macOS cannot connect to `docs.openclaw.ai` because the docs MCP TLS handshake fails in Node.js and LibreSSL.

## Best Possible Solution

Close this report as no longer reproducible against the current hosted docs MCP endpoint. Any future hardening should be tracked separately as a focused improvement, such as using the canonical `search_open_claw` selector directly or adding a docs-MCP smoke check.
