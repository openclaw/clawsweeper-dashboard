## Summary

## What This Changes

Report asks Telegram streaming to keep assistant fragments chronological when intermediate tool/block messages and final replies are delivered in one turn.

## Best Possible Solution

Keep the current main implementation and let the next release carry the Telegram ordering fix; no further source change is needed for this report.

## Reproduction Assessment

No for current main. Source inspection found a regression test that simulates the reported Telegram maxDBdays-to-Done sequence and asserts the final answer is sent after the intermediate block, so there is no current-main failure path from this review.

## Solution Assessment

Yes. The current implementation is narrow: it keeps preview streaming, but switches to a fresh final only when a later visible non-preview message would otherwise make an edited final appear above it.
