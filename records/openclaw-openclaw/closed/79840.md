## Summary

Keep open: current main still drops explicit `cwd` on the native subagent path, and https://github.com/openclaw/openclaw/pull/81896 is an open, unmerged implementation PR with closing syntax for this issue, so the issue should stay open until that PR is landed or rejected.

## What This Changes

The issue requests that native `sessions_spawn({ runtime: "subagent", cwd })` run delegated subagents from the explicit `cwd` for same-workspace handoff flows.

## Best Possible Solution

Review and land or reject the linked implementation PR; the final fix should thread `cwd` as internal native spawn state, resolve one child workspace for metadata and attachments, and keep Gateway agent params schema-clean.

## Reproduction Assessment

Yes, at source level: call `sessions_spawn` with `runtime="subagent"`, a different `agentId`, and explicit `cwd`; current main reads `cwd` but never forwards it into native spawn params, so it cannot affect workspace inheritance. I did not run the live macOS/Telegram canary in this read-only review.

## Solution Assessment

Yes: the narrow maintainable direction is to carry `cwd` through `SpawnSubagentParams` as explicit native spawn state and reuse the resolved workspace for metadata and attachments. The open linked PR is the current implementation path and should be reviewed rather than closing this issue first.
