## Summary

Keep this issue open. Current main still accepts config-mode `skills.update` payloads containing `apiKey` and `env`, writes them to `skills.entries.<skillKey>`, and returns `config: current` directly without response redaction. The provided GitHub context also shows open unmerged PR #69998 with closing syntax for this issue, so the issue should stay open until that implementation is reviewed and merged or closed.

## Best Possible Solution

Review and land #69998 if the diff is clean, or apply the same narrow fix directly in `src/gateway/server-methods/skills.ts`: pass the config-mode `skills.update` response object through `redactConfigObject()` and add a regression test that proves `apiKey` and secret-like `env` values are redacted in the response while full values are still written to config.
