## Summary

Keep open: the report overlaps earlier Telegram streaming-loss work, but current main still intentionally uses a single edited progress draft with bounded rolling lines. Preserving all progress as durable chat history needs maintainer product direction rather than a cleanup close.

## What This Changes

Preserve Telegram agent progress history instead of replacing prior progress steps in the edited preview message.

## Best Possible Solution

Maintainers should decide whether Telegram needs durable progress history beyond the existing /verbose path, then implement it in the Telegram plugin with explicit UX scope, docs, regression tests, and live Telegram proof.

## Reproduction Assessment

Yes for the source-level behavior: current Telegram draft streaming edits one message and keeps only a bounded rolling progress list. No live Telegram reproduction was run, and current main may preserve more recent lines than the issue's npm bundle observation suggests.

## Solution Assessment

Unclear: appending or paging every progress update is plausible, but it is not obviously the best default because Telegram maintainers have intentionally kept one edited preview and moved durable work logs to /verbose.
