## Summary

#53330 is a duplicate of #53321. Current main still does not implement proactive auto session splitting, but the same remaining feature request is already tracked by the earlier open issue #53321, so this later duplicate can close while keeping the canonical work item open.

## Best Possible Solution

Close #53330 as a duplicate and keep #53321 as the canonical open feature request. The best product path is a core session lifecycle design that adds a token-threshold rollover policy, memory flush before rollover, summary/recent carry-over into the new session, channel binding preservation, user notification options, and tests covering active Discord/channel sessions and compaction failure boundaries.
