## Summary

Current `main` already applies `before_model_resolve` on each run before model resolution, and the `LiveSessionModelSwitchError` path is now limited to explicit user-driven session model changes rather than hook-driven overrides. Claude persistent sessions also restart when the model fingerprint changes, so the reported per-message switching failure no longer matches current behavior.
