## Summary

Current `main` already resolves spawned subagent tool policy from the child agent/session plus `tools.subagents.tools`, and checked-in regression tests cover both default coding-tool availability and explicit subagent allowlist enforcement. The same implementation is present in release `v2026.4.22`.

## Evidence

- **Subagent policy reads `tools.subagents.tools` and preserves explicit allow/alsoAllow entries:** `resolveSubagentToolPolicy()` and `resolveSubagentToolPolicyForSession()` build the subagent deny list, then remove any entries explicitly allowed by `tools.subagents.tools.allow` or `alsoAllow` before returning the final policy.
  - file: [src/agents/pi-tools.policy.ts:81](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/agents/pi-tools.policy.ts#L81)
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Child sessions resolve the child agent's own tool config, not the parent's resolved tool set:** `resolveEffectiveToolPolicy()` derives `agentId` from the explicit target or the child `sessionKey`, then loads that agent's config and tool policy from `resolveAgentConfig(...)`.
  - file: [src/agents/pi-tools.policy.ts:321](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/agents/pi-tools.policy.ts#L321)
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Regression test shows subagent sessions keep coding tools like `exec` instead of collapsing to session-only tools:** The checked-in test for `sessionKey: "agent:main:subagent:test"` expects `read`, `exec`, and `process` to remain available while session-control tools are filtered out.
  - file: [src/agents/pi-tools.create-openclaw-coding-tools.test.ts:216](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/agents/pi-tools.create-openclaw-coding-tools.test.ts#L216)
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Regression test shows explicit subagent allowlists are honored:** The checked-in test with `tools.subagents.tools.allow: ["read"]` expects the spawned subagent tool inventory to be exactly `["read"]`.
  - file: [src/agents/pi-tools.create-openclaw-coding-tools.test.ts:328](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/agents/pi-tools.create-openclaw-coding-tools.test.ts#L328)
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Regression tests cover `tools.subagents.tools.allow` and `alsoAllow` semantics directly:** The policy tests assert that `tools.subagents.tools.allow` and `alsoAllow` re-enable otherwise default-denied subagent tools, including merged behavior.
  - file: [src/agents/pi-tools.policy.test.ts:55](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/agents/pi-tools.policy.test.ts#L55)
