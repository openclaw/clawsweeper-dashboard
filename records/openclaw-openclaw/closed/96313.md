## Summary

Current main and the v2026.6.10 release already merge top-level Feishu channel config into named account config before reply dispatch, so `renderMode` is inherited unless the account overrides it. The remaining confusion appears to be that the supported non-card value is `raw`, not `markdown`.

## What This Changes

The issue requests that Feishu named accounts inherit top-level `channels.feishu.renderMode` so sub-account replies do not unexpectedly use the card-template path.

## Best Possible Solution

Keep the existing merged-account configuration path; users who want non-card Feishu replies should use the supported `renderMode: "raw"` value and report a fresh failure if `raw` still routes through cards.

## Reproduction Assessment

No. I did not run a live Feishu reproduction in this read-only cleanup pass, and source plus release inspection shows the requested inheritance path is already present; the report uses `markdown`, which is not a supported `renderMode` value.

## Solution Assessment

Yes. The current centralized account resolver is the best shape because it merges top-level Feishu config once and all reply paths read the merged account config instead of duplicating inheritance logic.
