## Summary

Keep open: current main still violates the documented subagent completion contract because restored ended runs with missing child session entries are pruned before announce cleanup, and the wait/result path still lacks direct final reply text. The issue is narrow and source-backed, so it should be fixed at source level rather than closed.

## What This Changes

The issue requests that native subagent completions deliver the child output to the parent and move task delivery from pending to delivered instead of timing out with empty output.

## Best Possible Solution

Apply a source-level registry/wait fix that treats missing child session rows as expected for already-ended runs, preserves frozen child final text through cleanup and wait/delivery contracts, and adds focused regression plus live subagent proof.

## Reproduction Assessment

Yes, by source inspection: an ended restored subagent run with a missing child session row still resolves to missing-session-entry and is pruned before announce cleanup. A live parent-output smoke is still needed after a patch to prove the full delivery_status and parent-output symptom.

## Solution Assessment

Yes, the maintainable fix is source-level, not dist patching: add the completed-run guard, preserve frozen child output through the durable wait/delivery path, and update the regression that currently expects pruning.
