## Summary

Current main addresses the reported packaged memory runtime-deps failure: the default memory slot now selects `memory-core` runtime dependencies, root-level runtime mirrors include both reported missing packages, and the staged runtime-deps install plan covers `chokidar`, `sqlite-vec`, and `typebox` for packaged memory loads. The proof commit is on main only; I found no release tag containing it yet.

## What This Changes

Ensure packaged Linux installs include the runtime dependencies needed for `openclaw memory status --deep`, specifically `chokidar` and `sqlite-vec`, after updating to v2026.4.27.

## Best Possible Solution

Keep the current main runtime-deps selection and root mirror behavior, then ship it in the next OpenClaw package so Linux users no longer need to manually install memory runtime dependencies.

## Reproduction Assessment

Yes. The report gives a concrete packaged v2026.4.27 Linux path: update, run `openclaw memory status --deep`, and observe missing `chokidar` followed by unavailable `sqlite-vec`; the release-vs-main metadata diff matches that failure mode.

## Solution Assessment

Yes. The current main fix is at the right packaging/runtime-deps boundary: memory-core remains plugin-owned, while the core staging layer installs active plugin deps plus declared root mirrors before loading built memory facades.
