## Summary

Current main already implements the documented secondary-agent read-through path for Codex auth profiles, including empty child stores, non-cloning reads, Codex app-server profile login/refresh, and ordered profile rotation before external CLI fallback.

## What This Changes

The issue asks OpenClaw to make Codex harness auth-profile inheritance and rotation work for secondary agents instead of treating empty or populated local stores as unusable.

## Best Possible Solution

Keep the current main implementation: secondary agents read through to main auth profiles without cloning OAuth refresh tokens, while Codex app-server uses the resolved profile order and owner-aware refresh path.

## Reproduction Assessment

No failing current-main repro was established. The reporter supplied concrete logs, and current source/tests now exercise the empty-child-store, inherited-main-store, Codex app-server login, and inherited OAuth refresh paths that match the reported failure shape.

## Solution Assessment

Yes. The current implementation is the maintainable fix path because it centralizes inheritance in the auth-profile store, keeps OAuth refresh-token ownership with the persisted owner store, and leaves external Codex CLI bootstrap as a scoped fallback rather than the primary profile source.
