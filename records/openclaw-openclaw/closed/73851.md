## Summary

Keep this PR open. Current main at 955b4df0939f90b4b6de5e7e6f3149dd125e1e50 still has the older exec-event heartbeat routing shape: no scopedExecEventWakeOptions helper, exec completions still call the generic scopedHeartbeatWakeOptions, wake overrides only model target, cron still canonicalizes forced session keys and pre-merges heartbeat config, and the combined Telegram silent/thread-retry regression coverage is absent. The PR is draft and appears to continue the related active implementation path from #71582 rather than being obsolete.

## Best Possible Solution

Keep the PR open and review it as the active implementation candidate. The best path is to finish the draft investigation, decide whether the scheduled-heartbeat edge belongs in this PR or a follow-up, then land a scoped implementation with the exec-event wake helper, structured heartbeat route-clear overrides, cron forced-session handling, wake coalescing coverage, and the Telegram disable_notification retry regression. Before merge, run the targeted tests already listed by the author plus the appropriate changed gate in Testbox.
