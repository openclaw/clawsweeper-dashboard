## Summary

Keep open. Current main already preserves native Ollama thinking/reasoning stream chunks, which likely resolves the linked #61223 webchat empty-response/queue-lockup repro, but it does not implement this PR’s explicit incomplete-turn retry behavior for unsigned-thinking-only or generic empty Ollama turns. Since the remaining PR-specific hardening is plausible and not fully present on main, conservative cleanup should not close it automatically.

## Best Possible Solution

Keep this PR open for maintainer review. If the desired scope is only #61223’s native Ollama thinking-token stream drop, the best outcome is a maintainer close as superseded by the shipped v2026.4.24 stream preservation. If the desired scope includes retrying final unsigned-thinking-only or empty Ollama turns, the best implementation is a narrow change in `src/agents/pi-embedded-runner/run/incomplete-turn.ts` with focused Ollama retry tests, kept separate from unrelated provider timeout, plugin auto-enable, or workspace-bootstrap changes.
