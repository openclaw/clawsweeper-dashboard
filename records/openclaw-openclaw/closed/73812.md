## Summary

Keep open: current main does not have the PR's intended restart-continuation fixes, but the branch still has a blocking scheduler gap where a later explicit restart timeout is not propagated into an already-pending or preparing restart. The automatic group-topic delivery override looks like the right narrow direction; the coalesced timeout state, shared constant, and changelog entry need repair before merge.

## What This Changes

The PR adds automatic source reply delivery for queued restart continuations, passes a 30-second per-request deferral timeout from explicit restart entry points, and adds regression coverage for group-topic continuation delivery and timeout behavior.

## Best Possible Solution

Keep the automatic-delivery override, store/update explicit restart deferral timeout in scheduler state for pending and preparing coalesced restarts, share one exported timeout constant, add coalescing regression coverage, and add a single active Fixes changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction is to set gateway.reload.deferralTimeoutMs to 0, keep pre-restart pending count nonzero, schedule a no-timeout restart, then coalesce an explicit restart with deferralTimeoutMs and observe that the original timer or active deferral poll still controls SIGUSR1 emission.

## Solution Assessment

No as submitted. The source-delivery override is the right narrow fix, but the timeout guarantee must live in pending/preparing restart state rather than only in the first scheduled call's captured opts.
