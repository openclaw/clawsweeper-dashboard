## Summary

Close as clawhub. The PR is a workspace/operator-specific variant of the bundled coding-agent skill, not a core OpenClaw docs fix. Current main already provides local skill override, ClawHub skill, and bundle-plugin paths for this kind of policy, while the proposed patch would make dangerous `--yolo` examples appear unconditional in a public bundled skill.

## Best Possible Solution

Do not merge this PR into core. Keep the bundled coding-agent skill generic and safety-conscious, and put yolo-required Codex examples in a workspace/personal same-name skill override or a ClawHub/bundle-style skill package. If OpenClaw wants a global bundled policy change, open a maintainer-owned docs/security PR that updates all examples consistently and resolves the existing unsafe-example concerns.
