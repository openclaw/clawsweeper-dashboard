## Summary

The reported Telegram group behavior matches the current documented group-room visibility policy: group/channel turns default to message-tool-only visible output, so a normal assistant final answer can remain private unless `messages.groupChat.visibleReplies` is set to `automatic` or the agent calls `message.send`.

## What This Changes

The issue asks OpenClaw to restore automatic delivery of normal Telegram group final replies after upgrading to v2026.5.12.

## Best Possible Solution

Keep the current message-tool default and use `messages.groupChat.visibleReplies: "automatic"` only for deployments that intentionally want legacy group-room final replies.

## Reproduction Assessment

Yes, from source inspection: a Telegram group turn without explicit `messages.groupChat.visibleReplies: "automatic"` follows the message-tool-only visible-reply path, so a normal assistant final answer is not automatically posted to the room. I did not live-run Telegram because this read-only review could not use credentials or mutate the checkout.

## Solution Assessment

No repository code change is the best fix for the report as written; the maintained path is to set the existing visible-reply config to `automatic` for legacy behavior, or have the agent use `message.send` in group rooms.
