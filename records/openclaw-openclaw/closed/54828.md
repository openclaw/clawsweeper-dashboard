## Summary

Current main already fixes #54816 by preventing `config.schema`/`config.get` from re-activating the plugin runtime registry on each MCP reconnect. The fix landed in `src/config/runtime-schema.ts` and is covered by an explicit regression test in `src/config/runtime-schema.test.ts`, so PR #54828's `src/plugins/registry.ts` dedupe is no longer the right path.

## Evidence

- **Current main uses manifest metadata, not runtime activation, for gateway schema loads:** `loadGatewayRuntimeConfigSchema()` calls `loadPluginManifestRegistry({ config, cache: false, env, workspaceDir })` and builds schema metadata from that manifest registry, instead of booting/replacing the active runtime plugin registry. See `src/config/runtime-schema.ts:11` and `src/config/runtime-schema.ts:21`.
