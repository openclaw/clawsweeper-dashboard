## Summary

Current main still has the reported doctor warning path. `openclaw doctor` requests cached `doctor.memory.status` with `probe: false`, the gateway can return the skipped `embedding.checked === false` payload, and the CLI currently coerces any successful payload to `checked: true` before `noteMemorySearchHealth` warns for key-optional providers such as `ollama`. This is a valid narrow bug, not implemented or stale.

## Best Possible Solution

Keep this open and fix the doctor memory probe contract narrowly: preserve `embedding.checked === false` from `doctor.memory.status`, treat skipped or inconclusive cached readiness as non-actionable for key-optional providers such as `ollama`, and continue warning for explicit checked failures or credential problems. Add a regression test covering an `ollama` config plus the skipped `memory embedding readiness not checked` payload.

## Clownfish Work Prompt

Fix the doctor memory-readiness false warning reported in https://github.com/openclaw/openclaw/issues/73882. On current main, `openclaw doctor` calls `doctor.memory.status` with `{ probe: false }`; the gateway may return `embedding: { ok: false, checked: false, error: "memory embedding readiness not checked; run `openclaw memory status --deep` to probe" }`, but `probeGatewayMemoryStatus` currently turns any successful payload into `checked: true`, causing `noteMemorySearchHealth` to warn for key-optional providers like `ollama`. Continue warning for explicit checked failures, missing remote credentials, and true gateway/provider failures. Add focused regression coverage for the skipped payload flowing through `probeGatewayMemoryStatus` and for an `ollama` memory-search config not emitting the warning when the gateway probe is unchecked/skipped. Existing adjacent tests are in `src/commands/doctor-gateway-health.test.ts`, `src/commands/doctor-memory-search.test.ts`, and `src/gateway/server-methods/doctor.test.ts`.
