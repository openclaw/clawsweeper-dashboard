## Summary

Current `main` already prevents transient auto-fallback model overrides from affecting later turns by clearing direct `modelOverrideSource: "auto"` session overrides during model selection and resetting the run back to the configured primary model. That makes this PR redundant even though it uses a different implementation strategy.

## Evidence

- **Current main clears direct auto-fallback overrides before next-turn selection:** `createModelSelectionState` detects a direct session override with `modelOverrideSource === "auto"`, rewrites the session entry back to the default selection, persists that change when `storePath` is present, and resets in-memory `provider`/`model` to the configured primary for the turn.

## Risks / Open Questions
