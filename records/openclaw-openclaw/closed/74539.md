## Summary

Close as implemented on current main. The reported v2026.4.26 Telegram bare `/new` fall-through is fixed by the shared reset command handler: bare `/new` now emits a local acknowledgement and returns before the reply pipeline can start an agent/model request. The latest release tag still shows the old fall-through behavior, so this is fixed on main but not proven shipped in a release yet.

## What This Changes

A bare Telegram `/new` command should reset the session without sending an empty model request.

## Best Possible Solution

Keep the current shared reset-command behavior: bare `/new` and `/reset` acknowledge locally without model execution, while reset commands with trailing user text continue into the agent path. Ship this in the next release so users on v2026.4.26 receive the fix.
