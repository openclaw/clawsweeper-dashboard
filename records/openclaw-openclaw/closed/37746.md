## Summary

Keep open. Current main has prompt-based skills and heartbeats, plus untrusted-content wrapping and deterministic slash-command tool dispatch, but it does not have a declared sandboxed heartbeat function contract; the requested capability changes public skill metadata, runtime enforcement, and ClawHub trust policy, so it needs maintainer security/product design rather than cleanup closure.

## What This Changes

The issue requests schema-declared sandboxed skill functions so remote heartbeat endpoints can invoke bounded actions instead of sending free-form natural-language instructions.

## Best Possible Solution

Design a narrow public contract that lets skills or plugins declare schema-bound callable actions for remote heartbeat-like data, with runtime validation, tests, and docs; ClawHub can then surface that contract as a trust signal.
