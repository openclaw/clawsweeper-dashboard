## Label Justifications

- `P1`: The report describes a broken group-channel agent workflow where Claude CLI loses multi-turn session continuity on each mention.
- `impact:session-state`: The affected behavior is session reuse and conversation state continuity for Claude CLI group chat turns.

## Summary

Current main already separates volatile group/chat history from the CLI session reuse hash and carries it as current-turn user context, so the reported Claude CLI group-session reset path is addressed on main; the fix is not yet in the latest tagged release.

## What This Changes

The issue asks to keep Feishu group recent context out of the stable Claude CLI system prompt while still showing that context to the model each turn.

## Best Possible Solution

Keep the current main implementation and ship it in the next release so Claude CLI group sessions use a stable reuse hash while still receiving per-turn group context.

## Reproduction Assessment

No, I do not have a high-confidence live Feishu plus Claude CLI reproduction on current main. Source inspection shows current main no longer has the reported failure path because group history is threaded through currentInboundContext and CLI reuse hashing uses extraSystemPromptStatic.

## Solution Assessment

Yes, the current main approach is the narrow maintainable fix: static prompt material is hashed for session reuse, while volatile group history remains model-visible as user-role current-turn context.
