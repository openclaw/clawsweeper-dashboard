## Summary

Current main contains post-2026.4.24 fixes for the slow onboarding/CLI paths described in #71938: QuickStart channel setup no longer eagerly loads setup/status plugins before channel selection, gateway/status-style commands skip the eager context-window warmup, provider startup scans are cached/guarded, and the plugin runtime-dependency scan now caches bundled manifest reads. The Bonjour-related comment also matches a current main Bonjour hardening change, though the primary close reason is that the onboarding/startup slow paths are already fixed on main.

## Best Possible Solution

Close this issue as implemented on main and keep the post-2026.4.24 cold-start/onboarding fixes. If users still see multi-minute steps on a newer build, they should open a fresh report with timing logs for the exact step and channel/plugin set.
