## Summary

The request is coherent, but current main treats Tavily-specific capabilities as plugin-owned scope, and OpenClaw's published direction is to keep optional/vendor-specific integrations out of core when they can ship as plugins. Tavily already exists as a bundled plugin with plugin-owned tools, and the public SDK/docs explicitly support publishing additional agent tools through ClawHub or npm without core changes.

## Evidence

- **Vision scope: optional capability belongs in plugins, not core:** OpenClaw's vision says core should stay lean, optional capability should usually ship as plugins, bundle-style plugins are preferred when possible, plugin promotion belongs in ClawHub, and commercial service integrations outside the model-provider category are generally not merged into core.

## Risks / Open Questions

- Maintainers could still decide to expand the official bundled Tavily plugin later, but current published scope points away from growing first-party vendor-specific tool surface in core.
- There was no linked external Tavily crawl/map/research plugin in the issue thread, so this closure is based on scope and existing plugin seams rather than a ready-made replacement package.
