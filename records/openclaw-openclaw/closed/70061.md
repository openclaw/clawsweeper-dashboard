## Summary

Keep open. Current main still has the Active Memory chat-type classifier gap, but an open closing PR already targets this exact issue, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Active Memory should classify Telegram DM topic session keys like `agent:main:main:thread:{chatId}:{topicId}` as direct chats instead of skipping recall and transcript persistence.

## Best Possible Solution

Review and land #75533 or an equivalent narrow fix that strips the canonical thread suffix before Active Memory chat-type classification and keeps unknown session shapes fail-closed.

## Reproduction Assessment

Yes. Static reproduction is high confidence: current Telegram tests prove the `agent:main:main:thread:1234:42` shape, and the Active Memory classifier can be traced to `undefined` for that key before the hook returns early.

## Solution Assessment

Yes. Reusing the existing thread-suffix parser before the current exact classifier is narrower and safer than falling back every unknown provider/channel key to `direct`.
