## Summary

Current main implements the useful part of this PR with a safer, explicit `gateway.auth.trustedProxy.allowLoopback` opt-in instead of unconditionally removing the loopback denial. The config type/schema, gateway auth path, regression tests, docs, and security audit warning are all present on main. The latest release tag checked does not include this yet, so no fixed release is set.

## Best Possible Solution

Close this PR rather than merge it. Keep the current main implementation: an explicit, documented `allowLoopback` opt-in with tests and an audit warning, which addresses the local reverse-proxy use case without weakening the default trusted-proxy boundary.
