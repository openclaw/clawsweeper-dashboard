## Summary

Current main already handles non-persistable `allow-always` approvals by storing a durable exact-command approval, so shell builtins like `cd` no longer re-prompt on rerun.

## Evidence

- **Durable exact-command approvals are recognized:** `hasDurableExecApproval` now accepts hashed exact-command approvals (`=command:<digest>`) even when normal allowlist analysis does not produce executable matches.
