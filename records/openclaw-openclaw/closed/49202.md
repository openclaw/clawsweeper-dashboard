## Summary

Current main does not yet implement the UUID-preservation fix, but the issue is directly superseded by open PR #55351, which is the focused implementation for the same Open WebUI bare-UUID session split and explicitly says it fixes #49202.

There is no bare UUID/session-id exception here.
