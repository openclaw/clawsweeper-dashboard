## Summary

Close as implemented on current main. The Telegram quote-error classifier now matches both `QUOTE_TEXT_INVALID` and normalized `quote text invalid`, and the existing native-quote retry path removes `reply_parameters.quote` and resends as a legacy reply. Regression coverage was added for the reported variants; the fix is on `main` only and is not contained in the latest release tag.

## What This Changes

Handle Telegram Bot API `QUOTE_TEXT_INVALID` native quote send failures by retrying without `reply_parameters.quote` instead of dropping the reply.

## Best Possible Solution

Keep the current main implementation and let the next release carry it to users; no additional core or plugin API work is needed for the reported Telegram fallback gap.
