## Summary

Current main and the shipped v2026.4.23 release already handle Ollama reasoning/thinking chunks as stream activity, preserve reasoning-only outputs, and default discovered Ollama model definitions toward reasoning-enabled metadata. That matches the requested fix closely enough to close as implemented.

## Evidence

- **Ollama stream now emits thinking activity:** The Ollama stream parser reads `chunk.message.thinking ?? chunk.message.reasoning` and pushes `thinking_start` / `thinking_delta` events before any visible text, so iterator activity continues during reasoning phases instead of looking silent to the idle-timeout wrapper.
