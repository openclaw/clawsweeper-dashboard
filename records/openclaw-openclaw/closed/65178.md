## Summary

Keep this PR open. Current main still uses exact case-sensitive provider/model matching in the gateway image-support resolver, so the PR's intended bug fix is not implemented on main. The latest PR context says the contributor addressed the earlier fail-closed concern, and the diff is limited to gateway resolver code plus tests, so the right maintainer path is review and validation rather than cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to land a narrow gateway fix that uses `findModelInCatalog` for provider-scoped lookups, allows model-only fallback only when no provider was supplied, preserves explicit-provider fail-closed behavior, derives any legacy-shim provider from the resolved entry when needed, and adds regression tests for mixed-case provider/model IDs, provider aliases, providerless fallback, and provider mismatch. If the linked user bug also depends on configured-provider models being absent from the gateway catalog, track or fold in that catalog fix explicitly.
