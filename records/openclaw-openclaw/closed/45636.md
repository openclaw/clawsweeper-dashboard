## Summary

Current main already contains the Control UI history-refresh handling that was missing in the reported 2026.3.13 build, and the same behavior is present in release v2026.4.12 with regression coverage.

## Evidence

- **Current main reloads history on transcript updates:** `handleSessionMessageGatewayEvent` reloads `chat.history` for the active session, defers reload during an active run, and `handleGatewayEventUnsafe` wires both `session.message` and `sessions.changed` gateway events into UI refresh paths.
  - file: [ui/src/ui/app-gateway.ts:481](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/ui/src/ui/app-gateway.ts#L481)

- **Regression test covers the behavior:** Node test asserts that `session.message` reloads history for the active session and ignores other sessions; the same test file also covers `sessions.changed` reloading sessions.
  - file: [ui/src/ui/app-gateway.sessions.node.test.ts:112](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/ui/src/ui/app-gateway.sessions.node.test.ts#L112)
