## Summary

Issue #45644 asks for conditional routing and multi-model orchestration in core, but current main intentionally keeps agent model config static and exposes plugin hooks for request-time model/provider routing. The project vision also says optional capability should usually ship as plugins and that heavy orchestration layers should not land in core, so this belongs in ClawHub/plugin space rather than OpenClaw core.

## Risks / Open Questions

- A future docs-first effort could still standardize a recommended plugin pattern for intent-based routing without moving the feature into core.
- The existing hook seam covers deterministic per-run routing; full parallel multi-model collaboration would still require plugin-specific orchestration design.
