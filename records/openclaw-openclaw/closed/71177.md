## Summary

Current main already implements the heartbeat-specific fix through `agents.*.heartbeat.isolatedSession`. When enabled, heartbeat turns are routed to a canonical `:heartbeat` sibling and `resolveCronSession(... forceNew: true)` creates a fresh session ID for every heartbeat run, with stale transcript/session state cleared. The config field is in schema/types, documented publicly, covered by regression tests, and present in release v2026.4.24.

## Best Possible Solution

Close this heartbeat-specific request as implemented by the shipped `agents.defaults.heartbeat.isolatedSession` / `agents.list[].heartbeat.isolatedSession` mode. Keep the current opt-in behavior for deployments that want heartbeat runs without prior transcript context, and track any broader all-session TTL or default policy separately rather than adding another heartbeat-only rotation matrix.
