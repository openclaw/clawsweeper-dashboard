## Summary

Current main resolves the reported release failure path: Telegram and Slack runtime packages are root package dependencies for the core install, while Discord and Nostr are treated as installable plugin packages rather than postinstall-staged bundled dependency trees. The old postinstall/runtime-deps staging contract is no longer a supported path on main.

## What This Changes

Ensure default global installs do not lose Telegram, Discord, Slack, or Nostr channel runtime dependencies or silently hide channel load failures.

## Best Possible Solution

Keep the current packaging split: core-shipped channels must have their runtime deps in the root install, and heavier optional channels should install through the normal plugin package flow.

## Reproduction Assessment

Yes for the affected v2026.4.29 release: the issue gives a clean global npm install plus `openclaw status`/`doctor` path with specific missing module errors. Against current main, the source/package metadata shows that exact postinstall-staging dependency path is no longer used.

## Solution Assessment

Yes. The current-main approach is more maintainable than eager postinstall package-manager side effects because dependency ownership is either the root OpenClaw package for core channels or the plugin package install flow for externalized channels.
