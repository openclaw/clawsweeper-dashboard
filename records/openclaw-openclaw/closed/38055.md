## Summary

Current `main` no longer treats subagent completion announce delivery failures as terminal for the parent-session announce path. Completion announces now run through best-effort delivery/session-only fallback, and outbound channel send failures do not fail the agent run itself.

## Evidence

- **Completion announce uses best-effort agent delivery:** `sendSubagentAnnounceDirectly()` forwards completion announces through `callGateway({ method: "agent" ... })` with `bestEffortDeliver`, preserving the parent-session agent call while still attaching the internal event payload.
  - file: [src/agents/subagent-announce-delivery.ts:517](https://github.com/openclaw/openclaw/blob/627c19c5cbbe8856a11effe1871c442ac87d56c9/src/agents/subagent-announce-delivery.ts#L517)
  - command: `nl -ba src/agents/subagent-announce-delivery.ts | sed -n '432,558p'`
  - sha: [627c19c5cbbe](https://github.com/openclaw/openclaw/commit/627c19c5cbbe8856a11effe1871c442ac87d56c9)
- **Gateway defaults main-session agent calls to best-effort and can downgrade to session-only:** The gateway agent handler auto-enables `bestEffortDeliver` for main/global sessions when the caller does not force strict delivery, and downgrades `deliver=true` requests to session-only instead of rejecting when no deliverable external route exists.
  - file: [src/gateway/server-methods/agent.ts:729](https://github.com/openclaw/openclaw/blob/627c19c5cbbe8856a11effe1871c442ac87d56c9/src/gateway/server-methods/agent.ts#L729)

  - sha: [627c19c5cbbe](https://github.com/openclaw/openclaw/commit/627c19c5cbbe8856a11effe1871c442ac87d56c9)
- **Outbound delivery errors no longer abort best-effort runs:** Outbound send failures are swallowed under `bestEffort` mode: delivery logs the error and continues instead of throwing, which prevents a downstream channel outage from failing the parent-session agent run.
  - file: [src/infra/outbound/deliver.ts:992](https://github.com/openclaw/openclaw/blob/627c19c5cbbe8856a11effe1871c442ac87d56c9/src/infra/outbound/deliver.ts#L992)
