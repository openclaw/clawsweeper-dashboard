## Summary

Current main already covers the report: Telegram routes voice-compatible TTS media through `sendVoice`, Microsoft TTS marks compatible outputs as voice-note-capable, the TTS tool explicitly tells agents to return `NO_REPLY` after successful auto-delivery to avoid duplicate narrative replies, and docs now clearly scope `~/.openclaw/settings/tts.json` to local overrides rather than provider voice/output settings.

## Evidence
