## Summary

Keep this issue open. Current main still converts blank iMessage media-only sends into `<media:image>` / `<media:${kind}>`, then sends that value as the JSON-RPC `text` field alongside `file`. The latest release tag has the same implementation, and the two focused fix PRs in the discussion were closed unmerged. This is a narrow, valid fix-PR candidate.

## Best Possible Solution

Keep the issue open and fix it in the iMessage plugin. Remove outbound placeholder substitution from `extensions/imessage/src/send.ts` for blank text plus media, preserve real captions, reply metadata, markdown/directive stripping, and the no-text/no-media guard, then add focused regression coverage for media-only iMessage sends using `text: ""` with `file`. Treat #15840 as the broader cross-channel placeholder policy request, not as a replacement for this specific iMessage bug.

Current `extensions/imessage/src/send.ts` resolves a blank text + media attachment into `<media:image>` / `<media:${kind}>` and sends that as the JSON-RPC `text` field, which is visible to recipients.
