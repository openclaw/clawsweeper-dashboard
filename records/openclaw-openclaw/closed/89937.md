## Summary

Close as not reproducible: current source and the published v2026.5.28 package both invoke `before_prompt_build` on the embedded attempt path, and the report's verification only greps a chunk that does not own prompt-build execution.

## What This Changes

The issue reports that chat-channel agent-runner/embedded-agent user messages do not invoke the existing `before_prompt_build` plugin hook, leaving hook-dependent plugins inert on that path.

## Best Possible Solution

Keep the existing late prompt-build hook path and treat future reports as deployment-specific until a trace shows a concrete branch skipping the verified helper.

## Reproduction Assessment

No; the cited missing-call path does not reproduce in current source or in the published v2026.5.28 tarball. The hook is in the split `selection-*` and `attempt.prompt-helpers-*` chunks, not necessarily in `embedded-agent-*`.

## Solution Assessment

No code change is the best answer for the cited root cause. Moving `before_prompt_build` into `resolveHookModelSelection` would be the wrong phase because the documented hook receives loaded session messages later in the attempt.
