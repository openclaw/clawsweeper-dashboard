## Summary

Current main already accepts `channels.telegram.richMessages` in the strict Telegram config schema and covers the top-level plus account-level paths; the fix is on main via the merged Telegram rich-message opt-in work, but it is not in the latest stable release `v2026.6.8`.

## What This Changes

The issue requests that the documented Telegram `richMessages` config key be accepted so operators can disable or opt into rich Telegram output without gateway startup failure.

## Best Possible Solution

Keep the current main implementation and close this issue as fixed-on-main; users on `v2026.6.8` still need the next stable release or a main build for the schema fix.

## Reproduction Assessment

Yes, at source level: `v2026.6.8` lacks `richMessages` in the strict Telegram schema, while current main includes it. I did not run a gateway restart because this was a read-only review.

## Solution Assessment

Yes. Adding the missing key to `TelegramAccountSchemaBase` is the narrowest maintainable fix because both the top-level Telegram config and per-account config derive from that schema base.
