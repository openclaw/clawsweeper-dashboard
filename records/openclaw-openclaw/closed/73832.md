## Summary

Current main already handles the reported Ollama/OpenAI-compatible usage path: Ollama `/v1` models are opted into streaming usage, OpenAI chat-completions chunks parse `usage.prompt_tokens`, `usage.completion_tokens`, and `usage.total_tokens`, session status derives prompt/context tokens from that normalized usage, and configured per-model `contextWindow` values are honored. The implementation is on current main after the latest shipped release, so released 2026.4.23 users would need a newer build/release.

## Best Possible Solution

Close this issue as implemented on current main. Keep the current generic OpenAI-compatible usage parsing plus the Ollama plugin usage opt-in, and let the next release carry the fix to users on 2026.4.23.
