## Summary

Current `main` already preserves session metadata for heartbeat/cron/exec turns, so heartbeat polls no longer clobber the main session label or routing state that drives the control UI selector.

## Evidence

## Close Comment

Closing as a Codex review because current `main` already implements this. The session init path now preserves existing session identity and routing for heartbeat/cron/exec turns instead of rewriting them from the heartbeat context (`src/auto-reply/reply/session.ts:557` and `src/auto-reply/reply/session.ts:656`), and session-origin derivation explicitly skips heartbeat-style providers (`src/config/sessions/metadata.ts:54`). There are also targeted regression tests for this exact class of bug: no synthetic heartbeat route on an unrouted session (`src/auto-reply/reply/session.test.ts:3092`) and preserving the existing user route on the shared main session during heartbeat (`src/auto-reply/reply/session.test.ts:3121`). The implementation is on `main` in commit `4737a86071e35c409b0af9d7849bad05d7cb387f`. I could not tie it to a shipped release tag, so this appears fixed on `main` but not yet in a tagged release.
