## Summary

Keep open: the bundled-plugin direction fits OpenClaw scope and the author of the prior core implementation agreed with that shape, but current head still has a source-reproducible JSONC insertion bug and is not implemented on main.

## What This Changes

This PR replaces the prior core oc-path shape with an optional bundled `oc-path` plugin that registers `openclaw path` via plugin CLI metadata and adds docs, tests, package metadata, labeler, lockfile, and one gateway test-mock adjustment.

## Best Possible Solution

Land the plugin-owned CLI after normalizing JSONC insertion parent paths through the same concrete path resolution used by lookup, with regression coverage and corrected JSONC edit-preservation docs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `oc://config/items.$last/tags/+` can resolve as a JSONC insertion point, then `mutateAt` replays `$last` through `Number(seg)` and returns `unresolved`.

## Solution Assessment

No. The bundled-plugin ownership is the right direction, but the current patch should not merge until JSONC insertion mutation consumes a concrete resolved parent path and the docs stop over-promising comment preservation for insertions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Normalize JSONC insertion parent segments:** `extensions/oc-path/src/oc-path/universal.ts:454`
  - body: `resolveJsoncInsertion` can accept an insertion parent reached through positional segments like `items.$last.tags`, but mutation then replays the raw parent path and array descent uses `Number(seg)`. That lets a path report a valid insertion point and then fail during `set`; pass a concrete resolved parent path through or resolve positional segments inside `mutateJsoncContainer`.
  - confidence: 0.91
- **[P3] Correct JSONC insertion preservation docs:** `docs/cli/path.md:326-327`
  - body: The CLI docs say JSONC edits preserve comments and whitespace through `jsonc-parser`, but insertion paths rebuild via `emitJsonc(..., { mode: "render" })`, which drops comments and formatting. Either make insertions use the parser edit path or carve out insertion behavior in the docs.
  - confidence: 0.78
