## Summary

Current main no longer swallows Shift+Space in the TUI input path. OpenClaw delegates unhandled editor input to the pinned pi-tui editor, and the shipped pi-tui version explicitly maps Shift+Space to insertion of a regular space, including Kitty/CSI-u and modifyOtherKeys paths relevant to tmux-style terminal forwarding.

## Best Possible Solution

Close this issue and keep the shipped pi-tui-based behavior. If this regresses in a specific terminal again, the follow-up should add a focused TUI editor regression test for the reported escape sequence rather than reopening this already-implemented Shift+Space report.
