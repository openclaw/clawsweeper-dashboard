## Summary

Keep open. Current `main` still does not justify closing this issue: the `diagnostics-otel` plugin only turns `model.usage` diagnostic events into `openclaw.model.usage` spans, and the only non-test `model.usage` emitter still lives in the auto-reply channel path. That means non-channel agent entrypoints can still miss LLM OTEL spans entirely. Separately, prompt/completion content is still not part of the `DiagnosticUsageEvent` schema, so the original input/output-content expectation is still unimplemented.
