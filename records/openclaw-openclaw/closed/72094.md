## Summary

Current main already implements the requested Codex Desktop Computer Use routing path through the bundled Codex app-server harness. It is separate from PeekabooBridge: OpenClaw exposes `plugins.entries.codex.config.computerUse`, can status/install the Codex `computer-use` plugin through app-server, checks the MCP server before Codex-mode turns start, and leaves the actual native desktop-control tool loop to Codex during Codex-mode turns. The implementation landed after the latest listed release, so no shipped release tag was found yet.

## Best Possible Solution

Close #72094 as implemented on main and keep the shipped Codex app-server Computer Use path as the supported solution: users should enable the bundled `codex` plugin, select `agentRuntime.id: "codex"` for Codex-mode turns, configure `plugins.entries.codex.config.computerUse`, and use `/codex computer-use status/install` for setup. Any remaining refinements should be tracked as narrower follow-ups, such as docs polish, release-note coverage, or a future generic desktop-control abstraction if maintainers explicitly want one.
