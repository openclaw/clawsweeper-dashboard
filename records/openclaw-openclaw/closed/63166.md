## Summary

Keep open. Current `main` still unconditionally reinitializes the global hook runner on every activating plugin load, so a later default-mode load can replace the gateway-bindable runner installed at gateway startup. The existing compatibility logic only reuses the active gateway-bindable registry when the later load resolves to the same cache key; it does not protect the hook runner across different activating loads.

## Evidence

- **Activating loads always replace the global hook runner:** `activatePluginRegistry()` calls `setActivePluginRegistry(...)` and then immediately `initializeGlobalHookRunner(registry)` with no guard for an already-active `gateway-bindable` mode.
  - file: [src/plugins/loader.ts:1831](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/plugins/loader.ts#L1831)

  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Cached activating loads also overwrite it:** When a cached registry is reused and `shouldActivate` is true, `loadOpenClawPlugins()` still calls `activatePluginRegistry(...)`, so cached later loads reinitialize the hook runner too.
  - file: [src/plugins/loader.ts:1885](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/plugins/loader.ts#L1885)
