## Summary

Keep open. The PR is the right narrow direction for the linked runtime-provider allowlist bug, but the new config surface still needs generated schema/doc metadata, public docs/changelog coverage, and the maintainer-requested contract note before it is ready to merge.

## What This Changes

The PR adds an opt-in `plugins.bundledMode: "respect-allow"` setting, schema/help wiring, runtime provider discovery and bundled-compat gating, and focused tests so bundled providers can honor `plugins.allow` while defaulting to legacy compat behavior.

## Best Possible Solution

Land the default-preserving `respect-allow` implementation after adding generated config metadata, public plugin/config docs, an Unreleased changelog entry, and the small contract note without changing the runtime semantics.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: use a restrictive `plugins.allow`, then bundled provider discovery returns non-workspace provider plugins before allowlist checks and compat activation can widen the effective config.

## Solution Assessment

Yes as the product direction, but not yet as a shippable patch. The opt-in mode is the narrow maintainable fix because it preserves legacy defaults; the remaining blockers are docs/generated metadata/changelog and the requested contract note.
