## Summary

Keep open. The change is relevant to the linked session-title UX problem, but it currently replaces greeting-only titles with the opaque session-id/date fallback rather than conversation content, and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR adds generic greeting-only filtering to `deriveSessionTitle()` and tests that `hi`, `hello`, and `hey` fall back to the session id/date title.

## Best Possible Solution

Define title derivation to prefer existing metadata, then a bounded meaningful source such as a stored compaction summary or later meaningful user prompt, with regression coverage for gateway listing, `sessions_list`, and the TUI picker.

## Reproduction Assessment

Not applicable as a strict current-main bug reproduction because this is a feature/UX request. Source inspection does show the PR gap: the branch skips generic first messages but callers still provide only `fields.firstUserMessage`, so the title falls through to the session id/date fallback.

## Solution Assessment

No. The greeting filter is a plausible narrow heuristic, but it is not the best complete fix while it still falls back to opaque session-id/date titles instead of using a bounded meaningful title source or narrowing the closing claim.
