## Summary

Current main already handles the reported Gemini empty-output failure path by treating non-visible terminal results as failover-worthy and advancing through the configured model fallback chain. The implementation is present on main only; I could not prove it is included in a shipped release tag.

## What This Changes

The issue asks for Gemini Flash empty/null text responses to trigger the configured fallback model instead of ending the agent turn silently.

## Best Possible Solution

Keep the current main behavior that treats empty Gemini turns as incomplete, surfaces them to the fallback classifier, and advances through configured fallback models.

## Reproduction Assessment

No live Gemini API reproduction was run because the provider symptom is intermittent. Source inspection and regression tests cover the current-main empty Gemini/non-visible result path and the model fallback handoff.

## Solution Assessment

Yes. The current implementation keeps the generic fallback loop and classifies empty successful runs as failover-worthy, which is the narrow maintainable fix rather than adding Google-specific fallback policy in core.
