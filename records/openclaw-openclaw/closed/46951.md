## Summary

Current `main` already implements the reported fix: eager context-window warmup is skipped for `openclaw logs` and other non-model CLI commands, and regression tests cover that behavior. The latest shipped release `v2026.4.22` also contains the same implementation.

## Evidence

- **Non-model CLI commands now skip eager warmup:** `src/agents/context.ts` defines `SKIP_EAGER_WARMUP_PRIMARY_COMMANDS` including `logs`, `doctor`, `health`, `plugins`, `gateway`, and `status`, and `shouldEagerWarmContextWindowCache()` returns false for those commands before the import-time `ensureContextWindowCacheLoaded()` call.
