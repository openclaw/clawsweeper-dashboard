## Summary

Keep open. Current `main` partially addresses the request by exposing `runId` on tool hook context and actually invoking `after_tool_call`, but the core observability piece is still missing: the diagnostics OTel path does not create `openclaw.run` or `openclaw.tool.call` spans, `model.usage` diagnostic events still omit `runId`, and the OTel service starts standalone spans without any parent trace context propagation.

## Evidence
