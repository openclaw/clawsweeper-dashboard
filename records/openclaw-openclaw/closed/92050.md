## Summary

The reported beta-era deadlock is resolved: merged PR https://github.com/openclaw/openclaw/pull/87442 capped recall growth and added oversized-store repair, and v2026.6.5 moved dreaming state to the canonical SQLite-backed plugin store with doctor migration for legacy JSON. Current promotion no longer reads the legacy 16 MiB-limited file in steady state.

## What This Changes

Prevent unbounded short-term recall storage from permanently blocking dreaming promotion after the buffer exceeds its read-size limit.

## Best Possible Solution

Keep the shipped SQLite-backed capped recall store and its doctor migration as the single canonical path; no additional JSON rotation or runtime fallback should be added.

## Reproduction Assessment

No on current main or v2026.6.5: recall data is stored in capped SQLite-backed plugin state, while legacy JSON is handled by doctor migration rather than the former bounded promotion reader.

## Solution Assessment

Yes; the merged bounded canonical store plus one-time doctor migration is more maintainable than adding steady-state JSON rotation, retry, or fallback paths.
