- `impact:session-state`: The affected persisted recall state controls memory continuity across agent sessions.

## Summary

Current main and the latest release eliminate the reported deadlock: memory-core now stores short-term recall in bounded plugin state and retains at most 512 entries, so promotion no longer depends on reading an ever-growing 16 MiB JSON file.

## What This Changes

Prevent the memory-core short-term recall buffer from growing past the read ceiling and permanently blocking dreaming promotion.

## Best Possible Solution

Keep short-term recall in the bounded SQLite-backed plugin-state store and retain focused audit and repair coverage rather than restoring runtime JSON rotation or fallback reads.

## Reproduction Assessment

Yes for the reported older build: the production trace and legacy design establish that an oversized JSON read could fail before promotion. Current main no longer follows that runtime path, so the failure is not reproducible there.

## Solution Assessment

Yes. Using the existing bounded plugin-state store is narrower and more maintainable than adding rotation and retry behavior to the retired JSON runtime path.
