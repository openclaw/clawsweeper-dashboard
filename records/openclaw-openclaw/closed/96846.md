## Summary

Keep open: current main and v2026.6.10 still show the source-level Telegram album prompt-context gap, and https://github.com/openclaw/openclaw/pull/97045 is an open closing PR that should be reviewed or landed before this issue closes.

## What This Changes

The issue asks OpenClaw to pass every Telegram album photo to the model as usable media instead of leaving sibling photos as unreadable `telegram:file/...` references.

## Best Possible Solution

Review and land the focused Telegram-plugin fix in https://github.com/openclaw/openclaw/pull/97045 if maintainers accept its prompt-context semantics, then let the merged PR close this issue.

## Reproduction Assessment

Yes. Source inspection shows current main collects all album media but hydrates only the first current-turn media item into prompt context, while sibling cached media can still fall back to `telegram:file/...` refs.

## Solution Assessment

Yes. The narrow maintainable fix is the Telegram plugin's media-group to prompt-context handoff, reusing `media://inbound/...` projection instead of adding gateway batching or a new config option.
