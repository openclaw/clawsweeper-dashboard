## Summary

Current main still uses the memory runtime contract for doctor/status, and the SDK already exposes the required memory plugin runtime registration API. The concrete plugin named in the report, `memory-lancedb-pro`, is not part of this repository, so the fix belongs in that external plugin package. The linked #65066 is useful related work for bundled `memory-lancedb`, but it is separate from shipping a fix for `memory-lancedb-pro`.

## Best Possible Solution

Keep the OpenClaw core doctor/status runtime contract as-is. `memory-lancedb-pro` should add and release plugin-side `registerMemoryCapability({ runtime })` or `registerMemoryRuntime(...)` support in its own repository, while #65066 can proceed separately for the bundled `memory-lancedb` plugin if maintainers want that fix.
