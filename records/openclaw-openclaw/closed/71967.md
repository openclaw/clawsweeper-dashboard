## Summary

Close as ClawHub/plugin-scoped. The PR adds a self-contained process supervisor for an external Mission Control dashboard, and current main already has the plugin install/ClawHub path plus `api.registerCli` for plugin-owned CLI commands. It should not become a bundled core plugin, especially because `openclaw dashboard` is already a core command for opening the Control UI and plugin CLI registration skips roots that are already registered.

## Best Possible Solution

Do not merge `extensions/dashboard-launcher` as a bundled core plugin. Publish the Mission Control supervisor as an external ClawHub/npm plugin in its own repository with a non-conflicting CLI root, and open a separate maintainer design issue only if core should add a generic status-row contribution API or intentionally repurpose the existing `dashboard` command.
