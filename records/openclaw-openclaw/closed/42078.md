## Summary

mem9 is a useful external memory/context-engine plugin, but current main's documented scope and plugin architecture place this kind of integration in the community/ClawHub plugin path rather than the OpenClaw core issue tracker.

## Close Comment

Closing this as a ClawHub/community-plugin item after a Codex review of current `main`. The current project direction is that optional capability should usually ship as plugins, be hosted in the plugin author's own repo, and only rarely be added to core (`VISION.md:54`). The community-plugin docs also make ClawHub/npm the canonical path for third-party plugins (`docs/plugins/community.md:9`). Current `main` already supports the integration surface this proposal targets: `kind: "memory"` and `kind: "context-engine"` slots are documented in `docs/plugins/manifest.md:705`, the public SDK exports `registerContextEngine()` and `delegateCompactionToRuntime()` in `src/plugin-sdk/index.ts:116`, and the compaction helper is documented for third-party engines in `src/context-engine/delegate.ts:21`. There is also explicit test coverage for one plugin owning both slots (`src/plugins/slots.test.ts:205`). This makes mem9 a good fit for the external plugin ecosystem, but not a core-tracker item for OpenClaw itself. If you want broader discoverability, the right path is publishing and maintaining it as a community plugin on ClawHub or npm.
