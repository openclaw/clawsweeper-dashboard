## Summary

This is a useful Anthropic-specific memory integration request, but it fits OpenClaw's plugin/ClawHub boundary rather than core. Current main does not natively implement `memory_20250818`, but the existing plugin surfaces cover the necessary pieces: agent tools, per-agent/workspace context, provider hooks, and request-payload/header wrapping. The discussion already points to external memory work, including Kelisi808's PitStop skill link and KinthAI's persistent-memory architecture notes.

## Best Possible Solution

Move this to a ClawHub plugin/provider-wrapper path: register a plugin-owned `memory` tool with per-agent storage, use provider hooks to add Anthropic beta headers and patch the outbound Anthropic tool definition to `memory_20250818`, and document isolation/security behavior. Keep generic Anthropic context-management passthrough questions in #54041 if a concrete core API gap appears.
