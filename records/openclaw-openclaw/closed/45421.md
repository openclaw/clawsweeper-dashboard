## Summary

Keep open: current main still lacks Slack `ackReactionTiming`, and the PR is useful source work, but the patch is not ready because the deferred path does not integrate with the current Slack status-reaction lifecycle and still needs test, generated metadata, and changelog fixes.

## What This Changes

The PR adds Slack `ackReactionTiming` config, docs, schema/help labels, context plumbing, and tests to optionally defer ack reactions until `onAgentRunStart`.

## Best Possible Solution

Land a rebased Slack implementation that keeps default receipt-time behavior, accepts `received | run-start` through strict Slack config/schema/docs and generated metadata, starts the existing ack/status reaction lifecycle at run start when configured, and preserves scope gates, tool-only behavior, terminal status reactions, silent-success restore, and `removeAckAfterReply`.

## Reproduction Assessment

Yes. Static inspection of current main shows the requested config key is absent, and the PR defects are visible by comparing its diff to the current Slack status reaction lifecycle and reply runtime import seam.

## Solution Assessment

No, not as currently patched. The `onAgentRunStart` hook point is reasonable, but the implementation should defer the existing status reaction lifecycle instead of sending only a raw reaction, then refresh tests, metadata, and changelog.
