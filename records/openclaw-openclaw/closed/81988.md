The issue reports that opencode-go OpenAI-compatible completions models replay historical assistant `reasoning` fields and fail on multi-turn requests.

## Best Possible Solution

Keep the current OpenAI-completions replay sanitizer as the shipped direction; any later passthrough-policy cleanup can be a separate code-quality follow-up.

## Solution Assessment

Yes, the current-main sanitizer solves the observable provider rejection by removing invalid replay fields while preserving valid `reasoning_content` for Kimi-style models. A replay-policy-level cleanup would be narrower-looking, but it is no longer necessary to keep this issue open.
