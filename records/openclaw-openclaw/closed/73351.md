## Summary

Keep this PR open. Current main still has the targeted bug surface: Claude live-session mode transfers only the skills-plugin cleanup to the live-session lifecycle, while prepared bundle-MCP cleanup remains attached to the per-turn runner finally. The PR is still an active implementation candidate for #73244 Bug 1, not obsolete or implemented on main.

## Best Possible Solution

Keep this PR open and review it as the active fix candidate for #73244 Bug 1. The best path is a narrow ownership transfer in `src/agents/cli-runner/execute.ts`: capture `context.preparedBackend.cleanup` in the Claude live-session branch, clear it before the outer runner finally can delete the tempDir, wrap it with the existing skills-plugin cleanup passed to `runClaudeLiveSessionTurn`, and keep regression coverage proving fresh-session cleanup is deferred while close/restart/abort cleanup runs exactly once.
