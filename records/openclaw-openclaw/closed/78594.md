## What This Changes

Request to let OpenClaw recover orphaned transcript files by registering them back into the session store so they appear as known sessions again.

## Best Possible Solution

Consolidate recovery work into https://github.com/openclaw/openclaw/issues/73471, preserving valid historical transcripts and adding deliberate restore/rebuild semantics instead of blindly registering every unindexed file.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence verification path showing current main warns and archives unreferenced primary transcripts but does not recover or re-register them.

## Solution Assessment

No. Blindly re-adding every unreferenced .jsonl file into sessions.json conflicts with documented current-session pointer semantics; the safer direction is the semantic classifier and recovery path tracked in https://github.com/openclaw/openclaw/issues/73471.
