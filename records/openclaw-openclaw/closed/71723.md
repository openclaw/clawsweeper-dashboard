## Summary

Keep open. Current main still does not thread `models.providers.google.request` into the Google TTS HTTP request path, while the shared Google helper and sibling image-generation path already support `allowPrivateNetwork` when a sanitized request override is passed. The PR addresses a real remaining gap and adds focused regression coverage.

## Evidence

- **Current main lacks TTS request threading:** `synthesizeGoogleTtsPcm` accepts `baseUrl` but no request override, and calls `resolveGoogleGenerativeAiHttpRequestConfig` without `request`, so `allowPrivateNetwork` cannot reflect `models.providers.google.request.allowPrivateNetwork` for TTS.
