## Summary

Keep this PR open. Current main still has the Hetzner-specific docs gap this PR targets: the Hetzner guide does not define an `openclaw-cli` Compose service and does not document the Docker dashboard token or device-pairing approval flow. The PR needs a refresh for current main because startup now auto-seeds local Control UI origins, and the added CLI service should align with the current root Docker Compose hardening.

## Best Possible Solution

Keep the PR open and ask for a small rebase/update: add the Hetzner `openclaw-cli` service using the current root Docker Compose security/runtime pattern, keep the dashboard token and device approval instructions, and revise the `allowedOrigins` wording to match current automatic localhost/127.0.0.1 seeding plus manual custom-origin cases. Then rerun the docs check.
