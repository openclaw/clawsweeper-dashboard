## Summary

Current main, and the v2026.4.24 tag, address the actionable failure mode: Google Chat webhook auth rejections now log account-specific reasons when all candidates fail, and app-url startup warns when appPrincipal is missing or shaped like an email. The verifier still enforces the add-on token sub binding, with tests covering missing, matching, and mismatched principals plus the new logging and warning behavior. The workaround comments in the issue match the shipped behavior; remaining 401s with appPrincipal already configured are better tracked by the separate open Google Chat add-on auth report #58541.

## Best Possible Solution

Close this broad silent-failure/missing-appPrincipal item as implemented. Keep the explicit appPrincipal binding and shipped diagnostics; use #58541 or a focused docs-only follow-up for any remaining Google Chat add-on 401s or documentation polish.
