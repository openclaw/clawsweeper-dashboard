## Summary

Current `main` no longer requires `OLLAMA_API_KEY` for the ambient/local Ollama discovery path that would have caused this startup blocker. The Ollama provider now falls back to the non-secret `ollama-local` marker, the behavior is covered by tests, documented, and the changelog shows it shipped in `v2026.4.15`.

## Evidence
