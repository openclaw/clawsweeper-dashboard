## Summary

The reported docs search failure no longer reproduces against the current hosted docs MCP search path used by `openclaw docs`. The live MCP endpoint now exposes the docs search tool and returns relevant results for both reporter queries, while current MCPorter documents auto-correction for the legacy case/punctuation-only tool name used in `src/commands/docs.ts`.

## Best Possible Solution

Close this issue as no longer reproducible against the current docs MCP search path. A small hardening follow-up could rename the selector to the canonical `search_open_claw` tool and add smoke coverage, but the user-visible failure reported here is not present now.

## Risks / Open Questions

- `src/commands/docs.ts` still uses the legacy `SearchOpenClaw` selector and relies on current MCPorter auto-correction; an older globally installed `mcporter` without that heuristic could still fail until the selector is hardened to `search_open_claw`.
- The docs search backend is hosted and can change independently of this repo, so future regressions should be guarded by a small smoke check for representative queries like `Skills` and `browser extension`.
