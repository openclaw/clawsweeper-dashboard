## Summary

Close as duplicate of the already-open canonical report #75546. Current main still has the same narrow source-level CLI compaction initialization gap, and #75546 is already the focused open tracking item for the remaining fix.

## What This Changes

The issue reports that CLI transcript persistence/compaction can fail because the CLI compaction path resolves the default legacy context engine before built-in context engines are initialized.

## Best Possible Solution

Keep #75546 as the canonical open bug, preserve this issue's detailed reproduction by linking it there if needed, and fix the shared root cause by initializing built-in context engines at the CLI compaction boundary before resolving the active engine.

## Reproduction Assessment

Yes. A cold CLI-backed turn with a persisted session file and positive context token budget reaches `runCliTurnCompactionLifecycle()`, which currently resolves the default `legacy` engine without first calling the initializer that registers it.

## Solution Assessment

Yes for issue cleanup: closing this duplicate while keeping #75546 open is the best maintenance path. The eventual code fix should mirror the embedded compaction ordering with focused regression coverage, not change the registry fallback semantics.
