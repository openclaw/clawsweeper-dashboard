## Summary

Current `main` already includes a bundled Google Gemini TTS speech provider, tests for it, and user-facing docs/config examples, and the same implementation is present in release `v2026.4.22`.

## Evidence

- **Bundled Google speech provider implements Gemini TTS:** `buildGoogleSpeechProvider()` defaults to `gemini-3.1-flash-tts-preview`, resolves API keys from provider config, model-provider config, `GEMINI_API_KEY`, and `GOOGLE_API_KEY`, and posts to the Gemini `:generateContent` TTS endpoint.
  - file: [extensions/google/speech-provider.ts:14](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/extensions/google/speech-provider.ts#L14)
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Provider is registered in the bundled Google plugin:** The Google plugin registers the speech provider on startup, so this is part of the built-in product surface rather than an unmerged draft.
  - file: [extensions/google/index.ts:152](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/extensions/google/index.ts#L152)
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Contract metadata exposes Google as a speech provider:** The plugin manifest declares `speechProviders: ["google"]` and authenticates with `GEMINI_API_KEY` / `GOOGLE_API_KEY`.
  - file: [extensions/google/openclaw.plugin.json:7](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/extensions/google/openclaw.plugin.json#L7)
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)

  - file: [extensions/google/speech-provider.test.ts:35](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/extensions/google/speech-provider.test.ts#L35)
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
