## Summary

Keep open. Current main and the latest release still lack the requested provider-backed per-keyword FTS fallback, but this PR head shares the capped helper with the no-provider FTS-only path and can reduce existing fallback recall.

## What This Changes

The PR adds a capped per-keyword FTS fallback for provider-backed hybrid memory search, updates memory-core tests, and adds a changelog entry.

## Best Possible Solution

Refresh or replace the branch with a bounded provider-backed per-keyword FTS fallback while preserving the uncapped no-provider fallback, then land it with focused regression coverage and the existing contributor credit.

## Reproduction Assessment

Yes, by source inspection. Current main searches every extracted keyword in no-provider FTS-only mode, while PR head routes that path through a helper that slices fallback terms to six, so a long query whose only indexed hit is later than the cap can be lost.

## Solution Assessment

No, not as-is. The provider-backed bounded fallback is the right narrow direction, but the no-provider FTS-only path should keep its current uncapped recall or receive separate coverage for any intentional change.
