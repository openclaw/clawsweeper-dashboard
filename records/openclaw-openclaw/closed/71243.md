## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Guard the redeploy target before deleting it:** `scripts/redeploy-control-ui.sh:28`
  - body: The script lets OPENCLAW_INSTALLED_DIR point anywhere and then runs rm -rf before any sanity check. A typo can delete an arbitrary current-user directory, so validate the resolved target or require an explicit force mode before removing it.
  - confidence: 0.91
- **[P2] Fail verification when deployed assets do not match:** `scripts/redeploy-control-ui.sh:45-54`
  - body: The Python block only prints existence and same_index, so missing files or a false comparison still fall through to Done with exit 0. Make the verification exit non-zero when the deployed UI does not match the build.
  - confidence: 0.94
- **[P2] Remove the personal install path from the docs:** `docs/web/control-ui-redeploy.md:29-31`
  - body: The runbook includes the author's local install path, which is not reusable and conflicts with the docs rule to keep local paths generic. Use a command-derived path or placeholder instead.
  - confidence: 0.96
- **[P2] Localize the static shell copy:** `ui/index.html:270-286`
  - body: The added index.html shell renders English onboarding headings and notes outside the Control UI i18n system, so non-English users still see English guidance copy. Move this copy into the app/i18n path or remove the static shell text.
  - confidence: 0.84
- **[P3] Use python3 in the manual redeploy check:** `docs/web/control-ui-redeploy.md:69`
  - body: The docs' quick manual check still invokes python while the script uses python3; hosts without a python shim will fail this verification snippet. Use python3 consistently.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The PR adds a local redeploy script with destructive filesystem mutation and a predictable PATH shim, so it should not merge as-is.

Concerns:

- **[medium] Unbounded target deletion:** `scripts/redeploy-control-ui.sh:28`
  - body: rm -rf "$INSTALLED_DIR" runs on an overrideable path without checking that it is the intended OpenClaw Control UI install directory, so a mistaken environment value can delete unrelated files as the current user.

  - body: The script writes a pnpm shim under a fixed /tmp/openclaw-pnpm-shim directory and prepends it to PATH for the build, which can cause shared-system clobbering or unexpected executable resolution.
