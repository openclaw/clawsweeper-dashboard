## Summary

Current main already exposes the channel health-monitor polling interval through the gateway-level `gateway.channelHealthCheckMinutes` config key and threads it to `checkIntervalMs`. The exact proposed per-channel `healthMonitor.checkIntervalMs` field is still not accepted, but the operator problem of tuning the hardcoded 5-minute poll cadence is implemented.
