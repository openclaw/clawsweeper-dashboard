## Summary

Current `main` no longer routes `POST /hooks/agent` through a live-session input queue. It now runs the hook as an isolated agent turn, which makes the reported ~120s delay mechanism obsolete for this endpoint.

## Evidence

- **Hook handler now forces isolated execution:** The gateway hook handler builds a one-shot `CronJob` with `sessionTarget: "isolated"` and immediately calls `runCronIsolatedAgentTurn(...)` for `/hooks/agent`, instead of injecting into an already-live session transcript.

- **User-facing docs describe isolated behavior:** The public docs for `/hooks/agent` now explicitly say `Run an isolated agent turn`, which matches the current implementation and no longer promises live-session injection.
  - file: [docs/automation/cron-jobs.md:228](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/docs/automation/cron-jobs.md#L228)
