## Summary

Keep open: the branch uses the existing Gateway derived-title contract, but the actual picker option-label path still ignores rows that only have `derivedTitle`, so the dropdown can continue showing the parsed session key. The supplied proof is structured, but it is tests/build/grep only and does not show the live Control UI behavior.

## What This Changes

The PR adds `derivedTitle` to the Control UI session row type, requests `includeDerivedTitles` for chat session picker `sessions.list` calls, and adds display-helper/request expectation tests.

## Best Possible Solution

Wire `derivedTitle` into `resolveSessionScopedOptionLabel` with the same label/displayName precedence, add a render-level picker test for rows with only `derivedTitle`, and provide redacted UI proof.

## Reproduction Assessment

Yes, from source inspection. After the patch, picker options still flow through `resolveSessionScopedOptionLabel`, which only delegates to `resolveSessionDisplayName` for `label` or `displayName`, so rows with only `derivedTitle` still render the parsed session key rest.
