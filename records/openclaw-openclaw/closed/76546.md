## Summary

Current main already addresses the reported v2026.5.2 failure: npm plugin installs now maintain a shared managed npm-root manifest and merge persisted install records, so installing one external plugin should not prune or hide another external plugin. The proof is on current main only; no release tag in this checkout contains the proof SHA yet.

## What This Changes

The issue reports that sequential `openclaw plugins install` runs remove previously installed external npm plugins from the shared plugin root.

## Best Possible Solution

Keep the managed npm-root manifest and merged install-record behavior on main, then ship it in the next release for users affected by v2026.5.2.

## Reproduction Assessment

Yes, source-reproducible for the released failure: the report matches the shared npm-root pruning path fixed by adding a managed package.json. I did not run live plugin installs because this review was read-only, and current main now has source and regression-test coverage for preserving sibling packages.

## Solution Assessment

Yes. Maintaining the shared npm root dependency manifest and merging install records is the narrowest maintainable fix; it avoids plugin-specific special cases and covers arbitrary external npm plugin pairs.
