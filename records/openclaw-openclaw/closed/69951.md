## Summary

Current `main` already preserves explicit `/model` session overrides across fallback handling and status/session surfaces now prefer the selected override over stale runtime model fields, which addresses the MiniMax reversion described in #69951.

## Evidence

- **`/model` persistence clears stale runtime model fields:** `applyModelOverrideToSessionEntry` writes `providerOverride`/`modelOverride`, records `modelOverrideSource`, sets `liveModelSwitchPending`, and deletes stale `model`/`modelProvider` when the selected model changes so the persisted session row does not stay pinned to the old runtime model.
