## Summary

Close #69107 as duplicate/superseded by open PR #64044. The issue requests a newer Docker glibc baseline for derived images after `gws` failed on GLIBC_2.39; #64044 explicitly tracks the same migration by moving the Docker image family from `node:24-bookworm` to `node:24-trixie` and calls out `gws`/GLIBC 2.39+. Current main and the latest release still use Bookworm, so this is not fixed yet, but the remaining work has an active exact implementation PR.

## Best Possible Solution

Close #69107 as a duplicate/superseded tracker and keep the Docker base/glibc decision concentrated in #64044. The best product path is to either land a validated Debian 13/Trixie migration with Docker docs, tests, and release workflow alignment, or explicitly document that newer glibc requirements in derived third-party images are outside the supported OpenClaw Docker image boundary.
