## Summary

Current `main` does not match the report’s stale-provider-adapter model. Hot reload activates a fresh runtime config snapshot, `loadConfig()` serves that snapshot process-wide, and embedded runs resolve provider transport from the current config on each run rather than from a startup-only adapter. The latest stable `v2026.4.22` appears to share the same reload/runtime-snapshot code paths.
