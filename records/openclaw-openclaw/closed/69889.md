## Summary

Current `main` already converts empty `stopReason:"error"` assistant turns into error payloads or incomplete-turn errors, and cron isolated runs persist those as `status:"error"`. The shipped `v2026.4.23` release already contains that behavior.

## Evidence

- **Assistant stopReason=error becomes an error payload:** The embedded runner checks `lastAssistant.stopReason === "error"`, formats the assistant error, and pushes `{ text: errorText, isError: true }` into the reply payloads. That means downstream cron code sees an error payload instead of a silent successful run.
  - file: [src/agents/pi-embedded-runner/run/payloads.ts:162](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/agents/pi-embedded-runner/run/payloads.ts#L162)

If retries exhaust, it returns an `isError: true` incomplete-turn payload instead of leaving the run silent.
