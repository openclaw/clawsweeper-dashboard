## Summary

Keep this PR open. Current main still has the Docker Compose mount bug the PR targets: `docker-compose.yml` still bind-mounts `${OPENCLAW_CONFIG_DIR}` and `${OPENCLAW_WORKSPACE_DIR}` without defaults, and `docker compose config` fails when those variables are unset. The PR is a focused implementation candidate, though it still needs maintainer review for the final default path semantics, verification coverage, and changelog attribution.

## Best Possible Solution

Keep this PR open and either land it after review or replace it with an equivalent maintainer patch. The final change should add safe host-side defaults for both compose services, keep in-container config resolution aligned with `/home/node/.openclaw`, add or document a `docker compose config` regression check for rendered mount sources, and fix the changelog attribution before merge.
