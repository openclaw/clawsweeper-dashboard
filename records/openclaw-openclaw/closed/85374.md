## Reproduction Assessment

Yes, at source level: a terminal session patch for the active run can clear `chatRunId`, clear `chatStream`, and set `chatRunStatus.phase` to `done` before the async `loadChatHistory` path finishes rendering final content. I did not run a live browser reproduction in this read-only pass.

## Solution Assessment

Yes for triage: keeping this issue open while https://github.com/openclaw/openclaw/pull/85576 is reviewed is the right path. The maintainable fix is to separate local stale-run cleanup from publishing the visible terminal status until final content or history replay is complete.
