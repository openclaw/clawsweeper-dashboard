## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Avoid full-cache pruning once saturated:** `src/infra/dedupe.ts:60-61`
  - body: `check()` still calls the full `prune(now)` path whenever a new key pushes the map over `maxSize`. With Discord's five-minute, 5000-entry replay guard, a busy server can keep the cache saturated, so each new unique delivery still scans roughly the whole map and the targeted delay can continue.
  - confidence: 0.91
- **[P2] Preserve invalid-limit pruning behavior:** `src/infra/map-size.ts:10-14`
  - body: When `maxSize` is `NaN`, `deleteCount` becomes `NaN` and `i++ >= deleteCount` never becomes true, so the loop deletes every key. Current behavior is a no-op for `NaN`, and webhook guard callers can normalize malformed numeric config to `NaN` before pruning.
  - confidence: 0.88
