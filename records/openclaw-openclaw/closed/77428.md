## Summary

Keep open. Current main still routes an already-loaded `launchctl bootstrap` result directly to `launchctl kickstart`, and the existing unit tests still assert that behavior instead of checking whether the loaded service is already running.

## What This Changes

The issue asks `repairLaunchAgentBootstrap` to return `already-loaded` without kickstart when the LaunchAgent is already loaded and running.

## Best Possible Solution

Gate the already-loaded branch on `readLaunchAgentRuntime`: return `already-loaded` when the service is running, preserve kickstart for stopped or unknown states, and add regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. Source inspection shows the already-loaded bootstrap branch always calls `kickstart`, and the current unit tests assert that path; I did not run a live macOS launchd scenario.

## Solution Assessment

Yes. The suggested runtime check is the narrowest maintainable fix because it uses the existing `readLaunchAgentRuntime` helper and keeps the current kickstart fallback for loaded-but-not-running services.
