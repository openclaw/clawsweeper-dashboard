## Summary

Close as implemented. The reported v2026.4.26 Feishu bare `/new` path is covered by the shipped shared reset-command short-circuit: Feishu text messages preserve `/new` as `CommandBody`, command handling runs reset before other handlers, and bare `/new` returns a local acknowledgement before the model runner can submit an empty request. Related blank-message work such as #74717 remains separate from this exact `/new` reproduction.

## What This Changes

The issue reports that sending bare `/new` in a Feishu private chat falls through into an empty LLM request instead of starting a fresh session.

## Best Possible Solution

Keep the shipped shared behavior: Feishu bare `/new` should be treated as a local reset command and acknowledged without model execution, while `/new <text>` can continue through the agent path with non-empty user text. General Feishu blank-text ingress hardening should remain tracked separately through its own Feishu issue or PR.
