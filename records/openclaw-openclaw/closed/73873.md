## Summary

Current main and the latest release already provide the requested config controls. Telemetry/export can be disabled with `diagnostics.enabled` and `diagnostics.otel.enabled` (OTel export defaults off), and cloud/model providers are plugin-owned surfaces that can be blocked with `plugins.entries.<id>.enabled: false`, `plugins.deny`, `plugins.allow`, or the global `plugins.enabled: false` switch. The issue body has no usable repro beyond the repeated complaint, and the provided related-title matches are unrelated to this config surface.

## Best Possible Solution

Close this as already implemented and keep the existing granular config controls. A separate focused feature request would be appropriate only if maintainers want a new single privacy preset that composes these existing knobs across named provider/plugin surfaces.
