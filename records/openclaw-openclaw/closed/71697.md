## Summary

Keep #71697 open. Current main still documents and implements native Ollama Cloud setup, but the native `api: "ollama"` agent/Gateway path still forwards replay history to `/api/chat` without normalizing strict user/assistant alternation; tests explicitly expect no native Ollama replay policy.

## Best Possible Solution

Keep the issue open and fix the native Ollama provider path. The likely maintainer path is an Ollama-owned replay policy or converter-level normalization for `api: "ollama"` that merges or reshapes non-alternating user/assistant turns while preserving native Ollama tool-call behavior, plus regression tests that simulate the agent/Gateway replay history that currently reaches `/api/chat` as non-alternating roles.
