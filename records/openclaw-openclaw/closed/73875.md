## Summary

Keep open. Current main still forwards generic ACP runtime config keys when a backend exposes `session/set_config_option` without strict advertised keys, and the reporter’s latest v2026.5.4 data narrows the remaining Claude ACP failures to unsupported `timeout` and `thinking`/`effort` handling while PR #74048 remains open.

## What This Changes

OpenClaw should only send ACP session config options that the target adapter advertises, including avoiding `timeout` and mapping generic `thinking` to Claude ACP `effort` when supported.

## Best Possible Solution

Review and land PR #74048 or a replacement that filters `session/set_config_option` against adapter-advertised options, handles Claude’s `effort` key for generic thinking, and keeps timeout as an internal runtime control unless advertised.

## Reproduction Assessment

Yes. Source inspection on current main shows generic `timeout` and `thinking` keys are still sent when acpx advertises `set_config_option` without strict keys, and the published Claude ACP adapter rejects unknown config IDs; reporter live data confirms the failure on v2026.5.4.

## Solution Assessment

Yes. Capability-aware filtering plus adapter-key mapping inside the ACP/acpx boundary is the narrow maintainable direction; the open PR should be reviewed against the newer `timeout` and `thinking`/`effort` evidence before landing.
