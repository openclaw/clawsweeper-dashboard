## Summary

Current main already covers the PR’s intended safe-delivery behavior through the shipped source-delivery/send-policy path instead of a new `agents.defaults.replyMode` field. Direct/shared-session setups can suppress automatic source delivery while the agent still processes the turn and explicit `message` tool sends remain available; group/channel rooms also default to message-tool-only visible replies.

## Best Possible Solution

Close this PR as implemented by the existing source-delivery controls. Keep the current `session.sendPolicy` and `sourceReplyDeliveryMode` path as the single core mechanism, and handle any remaining UX/docs polish as targeted follow-up rather than adding a parallel `agents.defaults.replyMode` config surface.
