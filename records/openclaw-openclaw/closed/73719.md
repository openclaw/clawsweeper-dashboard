## Summary

Current main addresses the reported v2026.4.24 Bonjour failure path: ciao `CIAO PROBING CANCELLED` is classified and handled before the fatal unhandled-rejection path, stuck probing/announcing advertisers are bounded and self-disable for the current process, and `openclaw.json` now has `discovery.mdns.mode: "off"` so Bonjour is no longer env-var-only. Release/appcast entries show the direct crash-loop fix in v2026.4.25 and additional handler-across-restart hardening in v2026.4.26.

## Best Possible Solution

Keep the shipped/default LAN discovery behavior with bounded self-disable and the `discovery.mdns.mode: "off"` config path. Users on 2026.4.24 should update to a release containing the v2026.4.25/v2026.4.26 Bonjour fixes, or use the documented config/env/plugin disable path for deployments where mDNS is unavailable.
