## Summary

## What This Changes

This PR adds an `arrayToolParameterItems` model compat mode, wires it through tool-schema/model-catalog normalization and SDK exports, and defaults Xiaomi OpenAI-compatible token-plan endpoints to string array item schemas.

## Best Possible Solution

Keep the merged Xiaomi provider-local omission fix for the linked regression and handle broader custom-provider schema/config stability in the canonical open follow-up rather than adding this unproven public compat knob.

## Reproduction Assessment

Yes for the patch defect: source inspection shows the PR adds a config-facing compat key but leaves the strict config schema without that key, so normal config validation would reject it. The linked Xiaomi regression itself has prior live proof in the superseding merged PR.

## Solution Assessment

No. The narrower provider-local Xiaomi fix is already on current main, while this broader config/SDK surface would need schema, docs/product follow-through, and real behavior proof before it could be considered.
