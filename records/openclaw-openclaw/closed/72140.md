## Summary

Keep open. Current main no longer renders stale `totalTokens` as current context, but the shared status formatter still does not fall back to the latest compaction checkpoint `tokensAfter`, so it can show unknown context instead of the requested post-compaction size; open PR #72285 remains the active implementation path.

## What This Changes

The issue asks `/status` and shared status cards to report current post-compaction context size rather than stale pre-compaction totals.

## Best Possible Solution

Land or replace PR #72285 with a narrow shared formatter fix that keeps fresh totals authoritative and uses the newest valid checkpoint `tokensAfter` only when current totals are stale or unavailable.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `buildStatusMessage()` with `totalTokensFresh: false`, stale `totalTokens`, and a latest compaction checkpoint containing `tokensAfter`; current main renders unknown context instead of the post-compaction size.

## Solution Assessment

Yes in direction, but not complete on current main. The narrow maintainable solution belongs in the shared status formatter using existing freshness and checkpoint state, with focused regression coverage.
