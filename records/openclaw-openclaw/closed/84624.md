## Summary

## What This Changes

This PR adds a WhatsApp `dmPolicy: "open-except"`, a `manualFrom` skip list, schema/types/tests, and a self-chat vCard command handler to add or remove manual contacts.

## Best Possible Solution

Land only after maintainer approval of the WhatsApp access-policy shape, shared config-write authorization is used, vCard mutations are limited to `open-except`, public docs/config surfaces align, and redacted real WhatsApp proof shows add/rm plus manualFrom suppression.

## Reproduction Assessment

No high-confidence real WhatsApp reproduction is present; I did not run a live WhatsApp setup. The main blocker is source-reproducible: the PR passes a raw Boolean configWrites value where current main’s shared resolver defaults to enabled and honors account overrides.

## Solution Assessment

No. The feature may belong in the WhatsApp plugin, but the current implementation should use the shared config-write policy, gate self-chat vCard mutations on `open-except`, and provide real transport proof before merge.
