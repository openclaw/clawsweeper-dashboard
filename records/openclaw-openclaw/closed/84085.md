## Summary

Keep open: the linked bug is valid and the PR edits the right UI path, but the patch is not safe as-is because it removes the explicit Off override for non-off inherited defaults and only includes unit-test-level proof.

## What This Changes

The PR filters explicit off/none thinking options from the chat picker, disables the picker when no selectable levels remain, and adds helper-level unit tests.

## Best Possible Solution

Keep the duplicate-Off fix, but make Off filtering conditional on the inherited/default row actually representing Off, then update both helper and chat render tests for default-off and default-on models.

## Reproduction Assessment

Yes from source inspection: current main renders the inherited row plus normalized dynamic options, and existing tests show the duplicate Off shape when thinkingLevels include off. I did not run the UI because this review is read-only.

## Solution Assessment

No; the PR fixes the default-off duplicate by filtering Off unconditionally, but the narrower maintainable fix is to suppress Off only when the inherited row already means Off.
