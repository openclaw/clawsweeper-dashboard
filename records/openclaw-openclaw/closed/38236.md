## Summary

Keep open. Current main still does not implement the key cleanup requested for orphaned Chrome renderer/GPU/utility processes: managed Chrome is spawned as a normal child and stop only signals the tracked parent process. The related PR #38247 proposed process-group cleanup, but the provided GitHub context shows it was closed unmerged, so it is not a superseding implemented fix.

## Evidence

- **Current stop path only kills tracked parent process:** stopOpenClawChrome sends SIGTERM to running.proc, polls CDP reachability, then sends SIGKILL to the same proc. There is no process-tree or process-group cleanup here.
  - file: [extensions/browser/src/browser/chrome.ts:589](https://github.com/openclaw/openclaw/blob/81a41fe5bee350c6a035bfcc458eb8be380afe2c/extensions/browser/src/browser/chrome.ts#L589)
