## Summary

Keep open: the prompt surface is useful, but the branch's unconditional standard-Markdown hint is unsafe for Slack block and presentation paths that bypass conversion, and the PR lacks the required changelog entry.

## What This Changes

The PR appends Slack formatting guidance to `messageToolHints` and updates Slack prompt tests to expect the added hints.

## Best Possible Solution

Scope Slack prompt guidance so plain `message` text is described as standard Markdown that OpenClaw converts, while native Slack blocks and presentation text are described as direct Slack mrkdwn, with focused tests and an active changelog entry.

## Reproduction Assessment

Yes. Source inspection shows plain Slack sends call `markdownToSlackMrkdwnChunks`, while blocks and presentation text are forwarded or rendered as Slack `mrkdwn` without conversion, which gives a high-confidence reproduction of the PR-introduced mismatch.

## Solution Assessment

No. The prompt surface is right, but one global Markdown-only rule is not the narrowest maintainable fix because Slack plain text and Block Kit paths have different formatting contracts.
