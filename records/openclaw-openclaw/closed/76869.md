## Summary

Keep open: current main still leaves CLI backend assistant deltas on the agent-event bus while Telegram preview streaming consumes `onPartialReply`, and open PR #76914 is the linked closing implementation candidate.

## What This Changes

Bridge CLI backend assistant text deltas into Telegram preview streaming so draft updates appear before final delivery.

## Best Possible Solution

Land a narrow shared reply-runner bridge that forwards active CLI assistant deltas into the existing partial-preview and typing-signal path, with regression coverage for the CLI-to-Telegram callback path.

## Reproduction Assessment

Yes, source-reproducible: configure a JSONL CLI backend and Telegram preview streaming, then CLI deltas emit as assistant agent events while Telegram listens to `onPartialReply`. I did not run the external Telegram/Claude CLI setup, but the code path and independent reproduction are consistent.

## Solution Assessment

Yes, the best fix is the narrow shared reply-runner bridge rather than a Telegram-only subscription or new config. PR #76914 is the current implementation candidate, so this issue should stay open until that PR merges or is closed.
