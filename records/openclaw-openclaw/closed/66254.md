## Summary

Current `main` already routes configured third-party channel status through the read-only external-plugin path, with regression coverage for configured setup entries, env-only discovery, and install-path provenance handling. That addresses the reported `SETUP | no token` symptom for external channel plugins like WeCom on cold `status` paths.
