## Summary

Current main already has a native post-compaction continuity path: OpenClaw injects a post-compaction system refresh for the next turn, re-injects configured AGENTS.md sections, and runs an automatic pre-compaction memory flush to preserve durable context.

## Evidence

- **Core enqueues a post-compaction system event:** After compaction, the reply runner calls readPostCompactionContext(...) and enqueues the returned content as a system event for the session.
  - file: [src/auto-reply/reply/agent-runner.ts:1572](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/auto-reply/reply/agent-runner.ts#L1572)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
