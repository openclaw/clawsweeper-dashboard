Keep open. The patch is a focused and likely correct fix for a source-reproducible agent transport error-message loss, but it still needs redacted real runtime proof and maintainer review of the downstream failover/auth-profile classification effect before merge.

## What This Changes

The PR changes three agent transport stream error throw sites to prefer an existing provider `output.errorMessage` over the generic fallback and adds shared-helper regression tests.

## Best Possible Solution

Land this PR or an equivalent complete fix after redacted runtime proof is added and maintainers either accept the classifier behavior change or add coverage for provider finish reasons that should still rotate fallback/auth profiles; then close the narrower overlap at https://github.com/openclaw/openclaw/pull/84146 as superseded if this branch lands.

## Reproduction Assessment

Yes, source-level. Current main can map a provider finish reason into `output.errorMessage`, enter the `stopReason === "error"` finalization path, throw the generic wrapper, and then write that generic message back into the output.

## Solution Assessment

Yes, with a merge caveat. Using `output.errorMessage || "An unknown error occurred"` at the three throw sites is the narrowest diagnostic fix, but maintainers should explicitly account for the resulting failover/auth-profile classification behavior.
