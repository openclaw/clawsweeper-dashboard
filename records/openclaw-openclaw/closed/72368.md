## Summary

Keep #72368 open. Current main still only strips the literal `ollama/` prefix in both the Ollama chat and embedding paths, while #72353 reports the same failure for custom Ollama-compatible provider keys such as `ollama-spark/`. The PR is a focused, plausible fix with regression coverage and it also addresses the bot review’s duplicate-regex concern in its second commit.

## Best Possible Solution

Keep the PR open and either merge/adopt its small normalization after CI or request any final polish. The best code path is a single shared Ollama provider-prefix regex reused by chat and embedding normalization, with regression tests for custom `ollama-<key>/...` prefixes and a counter-test that preserves real registry namespaces such as `library/llama3`. Also fix the changelog attribution before landing.
