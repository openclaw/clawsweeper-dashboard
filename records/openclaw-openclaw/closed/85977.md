## Best Possible Solution

Keep the shipped usage surfaces and update the external/local cron to call `openclaw gateway usage-cost --days 1 --json` or the `usage.cost`/`sessions.usage` Gateway RPC; native budget enforcement should stay with the broader product discussion at https://github.com/openclaw/openclaw/issues/42475.

Yes. Source inspection and the issue output both show that root `openclaw usage` is not registered, while current code and docs expose `openclaw gateway usage-cost`, `usage.cost`, `sessions.usage`, and `openclaw status --usage`.

## Solution Assessment

Yes. Updating the repo-external cron to the existing Gateway usage/cost surface is narrower and less duplicative than adding a new root command for one local automation job.
