## Summary

Keep open: the requested per-agent memory backend override is still valid and was positively confirmed by the requester, but this PR is not merge-ready against current main because it misses the active package-owned resolver/config mirror and generated schema surfaces.

## What This Changes

The PR adds optional `agents.list[].memory` backend/QMD overrides, docs, and schema/runtime tests while keeping `memory.citations` global.

## Best Possible Solution

Land this PR or an equivalent focused patch after refreshing it onto current main, implementing the override in the package host SDK resolver/config mirror, updating source and generated config surfaces, documenting the boundary, adding a changelog entry, and preserving regressions for backend selection, QMD overlay, agent isolation, and citations rejection.

## Reproduction Assessment

Yes. Static inspection of current main is high-confidence: the strict agent schema has no `memory` key and the active resolver reads only root `cfg.memory`, so a config with `agents.list[].memory` is still outside the accepted/runtime surface.

## Solution Assessment

No, not as written. The proposed product shape is the narrow maintainable solution, but the diff must be ported to the current package-owned resolver/config mirror and regenerated schema/docs/changelog surfaces before merge.
