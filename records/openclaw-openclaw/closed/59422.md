## Summary

Current main already solves the Feishu/Lark @all problem in a different shape: broadcast-only `@all`/`@_all` mentions no longer count as bot mentions by default, while messages that include both broadcast and a direct bot mention still count. The PR’s exact `ignoreAtAll` config field is not present, but the observable user problem from the linked Feishu request is implemented, documented, covered by regression tests, and shipped in `v2026.4.26`.

## Best Possible Solution

Close this PR as implemented by current main and keep the shipped default behavior: Feishu/Lark broadcast-only mentions do not trigger the bot, while direct bot mentions still do. If maintainers later want an explicit opt-out or compatibility knob, that should be a separate Feishu product/config discussion with schema, docs, and regression tests.
