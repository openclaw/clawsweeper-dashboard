## Summary

Current main routes the implicit main-agent directory through `OPENCLAW_STATE_DIR`, and the reported repro steps do not match current code paths unless an explicit `OPENCLAW_AGENT_DIR`/`PI_CODING_AGENT_DIR` override is also present.

## Evidence

- **Default main agent dir uses state dir:** `src/agents/agent-paths.ts:6-12` resolves the default agent dir as `path.join(resolveStateDir(env), "agents", DEFAULT_AGENT_ID, "agent")`, so the implicit `main` agent follows `OPENCLAW_STATE_DIR` when no explicit agent-dir override is set.
  - file: [src/agents/agent-paths.ts:6](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/src/agents/agent-paths.ts#L6)
  - command: `nl -ba src/agents/agent-paths.ts | sed -n '1,30p'`
  - sha: [7d4718373698](https://github.com/openclaw/openclaw/commit/7d471837369885c12d540d6bafa319ff41129b89)
- **Auto-created models.json writes through that resolver:** `src/agents/models-config.ts:138-145` uses `resolveOpenClawAgentDir()` when no explicit `agentDirOverride` is provided, so the auto-created `models.json` path inherits the state-dir-aware resolver.
  - file: [src/agents/models-config.ts:138](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/src/agents/models-config.ts#L138)
  - command: `nl -ba src/agents/models-config.ts | sed -n '138,152p'`
  - sha: [7d4718373698](https://github.com/openclaw/openclaw/commit/7d471837369885c12d540d6bafa319ff41129b89)
- **Current service installer does not inject OPENCLAW_AGENT_DIR:** `src/daemon/service-env.ts:319-326` writes `OPENCLAW_STATE_DIR` and `OPENCLAW_CONFIG_PATH` into the managed service environment, but not `OPENCLAW_AGENT_DIR` or `PI_CODING_AGENT_DIR`, so the issue's listed repro steps do not explain a fallback to `~/.openclaw/agents/main/agent` on current main.
  - file: [src/daemon/service-env.ts:319](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/src/daemon/service-env.ts#L319)
  - command: `nl -ba src/daemon/service-env.ts | sed -n '319,327p'`
  - sha: [7d4718373698](https://github.com/openclaw/openclaw/commit/7d471837369885c12d540d6bafa319ff41129b89)
- **Regression test covers state-dir default:** `src/agents/agent-paths.test.ts:18-29` asserts that with `OPENCLAW_STATE_DIR` set and no agent-dir overrides, `resolveOpenClawAgentDir()` returns `<stateDir>/agents/main/agent`.
  - file: [src/agents/agent-paths.test.ts:18](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/src/agents/agent-paths.test.ts#L18)
  - command: `nl -ba src/agents/agent-paths.test.ts | sed -n '16,83p'`

## Risks / Open Questions

- A user who still has an explicit `OPENCLAW_AGENT_DIR` or `PI_CODING_AGENT_DIR` in their service environment will continue to override `OPENCLAW_STATE_DIR` by design.
