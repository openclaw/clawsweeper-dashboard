## Summary

Current main no longer has the reported default-heartbeat token leak: the runtime HEARTBEAT.md template is separate from the docs page, remains effectively empty, and the fix shipped in v2026.5.27.

## What This Changes

The issue asks OpenClaw to stop fresh default HEARTBEAT.md files from triggering recurring model API calls when they contain only scaffold or docs-template content.

## Best Possible Solution

Keep the current split runtime template plus doctor repair path; reopen only with evidence that a fresh v2026.5.27-or-newer workspace still sends model heartbeat calls for an untouched template.

## Reproduction Assessment

No on current main: the fresh runtime template is comments-only, the bootstrap writes that template, and current tests assert it is effectively empty. The old affected versions are source-explained by the docs Related link being copied into HEARTBEAT.md.

## Solution Assessment

Yes: separating the runtime template from docs content and adding doctor repair for old docs-wrapped templates is narrower and safer than ignoring arbitrary Markdown links in user-authored heartbeat tasks.
