## Summary

Current main fixes the reported sandbox config fallback: the Codex app-server bridge now keeps per-agent CODEX_HOME isolation without rewriting HOME by default, so child OpenClaw CLI commands inherit the normal user home and resolve the canonical config instead of Signal's 8080 default. The fix is merged on main but not included in v2026.5.7.

## What This Changes

Ensure OpenClaw CLI commands launched inside a Codex/OpenClaw agent sandbox use the user's canonical OpenClaw config instead of sandbox-local defaults such as Signal port 8080.

## Best Possible Solution

Keep the merged HOME-inheritance fix on main and let it ship in the next release; no further core change is needed for the reported Signal config fallback.

## Reproduction Assessment

No current-main failure path remains. The old behavior is supported by the reporter logs and the merged PR's before evidence, while current source and tests show HOME is no longer rewritten by default.

## Solution Assessment

Yes. Preserving inherited HOME at the Codex app-server bridge is narrower than changing Signal defaults, global config fallback behavior, or relying on sandbox symlinks, while still keeping CODEX_HOME isolated.
