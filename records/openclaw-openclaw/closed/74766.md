## Summary

Close as implemented on current main: group/channel turns now suppress automatic final delivery without removing the message tool, the prompt directs agents to use message actions for visible group output and polls, current-channel targeting defaults are wired through the message action runner, and WhatsApp action discovery exposes poll creation.

## What This Changes

WhatsApp group auto-reply sessions should keep an agent-callable poll-capable message surface instead of losing the message tool when same-chat duplicate-send protections apply.

## Best Possible Solution

Keep the current main behavior: suppress automatic group final delivery, keep the message tool available when tool policy allows it, expose WhatsApp poll through the shared message action surface, and only consider a dedicated poll tool if a future narrower API/design issue asks for it.
