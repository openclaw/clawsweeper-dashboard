## Summary

Current main already implements the isolated-cron stale transcript fix: fresh cron sessions are sanitized so they do not inherit `sessionFile`, a fresh transcript path is assigned before the pre-run session-store persist, and the executor uses that stored path for the run. The shipped changelog also records the broader cron isolated-session cleanup.

## Best Possible Solution

Close this PR as implemented by current main’s shipped cron isolated-session cleanup, and keep the broader mainline sanitization plus regression coverage rather than merging the older branch.
