## What This Changes

The issue requests automatic WebM/Opus to OGG/Opus conversion before WhatsApp PTT dispatch when Microsoft Edge TTS emits WebM audio.

## Best Possible Solution

Keep the current main WhatsApp media normalization and let it ship in the next release; reopen only if the same WebM PTT failure reproduces on a build containing `756379b11ddfafb5f9c597889a26a9c0b4fb2f39`.

## Solution Assessment

Is this the best way to solve the issue? Yes: fixing this in WhatsApp outbound media keeps the provider generic, preserves the Baileys PTT contract, and handles Microsoft WebM plus other non-Ogg audio without provider-specific special cases.
