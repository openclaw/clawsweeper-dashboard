## Summary

Current main already fixes the reported v2026.6.8 regression surface by making Bot API rich messages opt-in and returning default Telegram text delivery to standard HTML `sendMessage`; the fix is not in the latest release yet.

## What This Changes

The issue reports that Telegram iOS bot replies became larger after upgrading to v2026.6.8 and stopped respecting the client font-size setting.

## Best Possible Solution

Keep current main's standard Telegram HTML default with Bot API 10.1 rich messages behind explicit `channels.telegram.richMessages: true`; leave the broader parse-mode product choice to the existing open parse-mode request.

## Reproduction Assessment

Yes, source-level for the shipped regression surface: v2026.6.8 used rich-message sends by default, while current main no longer does unless `richMessages` is enabled. I did not run a live Telegram iOS font-size repro.

## Solution Assessment

Yes. Making rich messages opt-in is the narrowest current-main fix for a client-rendering regression, and it avoids adding a new parse-mode config surface in this cleanup path.
