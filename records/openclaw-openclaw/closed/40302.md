## Summary

Current main already strips `[[reply_to_current]]` / `[[reply_to:<id>]]` from iMessage outbound text before delivery, and the same behavior is present in the shipped `v2026.4.22` release.

## Evidence

- **iMessage send path strips directive tags before RPC send:** `sendMessageIMessage` converts markdown tables, then calls `stripInlineDirectiveTagsForDelivery(message).text` immediately before building the JSON-RPC send payload.
  - file: [extensions/imessage/src/send.ts:147](https://github.com/openclaw/openclaw/blob/9f4ef6162dbde384797abc002cd4241080eb65e3/extensions/imessage/src/send.ts#L147)
