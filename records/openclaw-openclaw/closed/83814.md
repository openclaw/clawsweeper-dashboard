The PR changes the Codex harness source visible-reply default from `message_tool` to `automatic`, updates Codex tests/docs/changelog, and adds coverage for explicit `messages.visibleReplies: "message_tool"` overrides.

## Best Possible Solution

Align prompt snapshot fixtures with opt-in message-tool coverage, then land this only if maintainers accept automatic visible replies as the new Codex direct/source default while preserving the explicit `message_tool` override.

## Reproduction Assessment

Yes, at source level: current main sets the Codex harness default to `message_tool`, the resolver maps that to `message_tool_only`, and dispatch suppresses automatic final delivery. I did not run live Telegram in this read-only review.

## Solution Assessment

Mostly yes if maintainers accept the default flip: the runtime change is narrow and preserves an explicit `message_tool` override. The safer alternative is to keep the recent tool-gated default and document `messages.visibleReplies: "automatic"` for users who want automatic direct replies.
