## Summary

Current main no longer routes the normal `pnpm check` typecheck through declaration emit, which is the path that produced the reported TS2883 inferred-type portability errors. The same wrapper behavior is present in the shipped `v2026.4.29` tag, so the reported `pnpm check` failure from 2026.4.6 is resolved for the current check path.

## What This Changes

Fix `pnpm check` so `tsgo` no longer fails with TS2883 inferred-type portability errors in xAI and bash exec source files.

## Best Possible Solution

Keep the shipped check-path behavior: normal `pnpm check` should typecheck production core and extension code without declaration emit, while explicit declaration builds remain opt-in for package-boundary surfaces.

## Reproduction Assessment

Yes for the historical report: the 2026.4.6 log shows TS2883 during `pnpm tsgo`. Current-main source verification shows that reproduction path no longer matches the normal `pnpm check` route because the wrapper passes `--declaration false` by default.

## Solution Assessment

Yes. For the reported `pnpm check` failure, disabling declaration emit in the normal tsgo check path is a narrow maintainable fix; explicit type annotations can still be added later if these xAI helpers become part of an emitted declaration surface.
