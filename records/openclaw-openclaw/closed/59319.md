## Summary

Current main already fixes the reported duplicate bootstrap injection. `resolveBootstrapFilesForRun` now sanitizes hook-adjusted bootstrap files by resolving each path against the workspace and dropping later entries with the same workspace-relative normalized path before building injected context. The repo also has a regression test for duplicate relative/absolute hook paths, docs clarify that `memory/*.md` daily files are not ordinary bootstrap Project Context, and the v2026.4.26 changelog explicitly lists the fix as carrying linked PR #59344 and fixing this issue.

## Best Possible Solution

Keep the shipped path-based dedupe in core bootstrap context assembly and treat any remaining daily-memory/startup-context surprises as new, narrower reports with reproduction against current releases.
