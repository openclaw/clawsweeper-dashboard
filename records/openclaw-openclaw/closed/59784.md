## Summary

Current main no longer follows the reported bad path where a Claude subscription limit disables the auth profile as billing and prevents fallback. Claude CLI error results are thrown as FailoverError, unclassified text falls back as reason `unknown` rather than `billing`, and the model fallback loop continues to the next configured candidate while any candidates remain. Temporary quota-window 402 cases are also explicitly kept on the rate-limit path.

## Best Possible Solution

Keep the shipped behavior: Claude CLI quota-limit errors should be FailoverError-backed failures, never long billing disables unless the text is explicitly billing, and configured model fallbacks should continue. A future narrow test/classifier for the exact `You've hit your limit ... resets` string would be acceptable polish, but the reported fallback blockage is already handled on main.
