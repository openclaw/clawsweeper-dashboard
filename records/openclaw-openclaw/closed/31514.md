## Summary

Issue #31514's original symptom appears implemented on current `main` and already shipped in `v2026.4.22`: Slack explicit reply tags still route through threaded transport, and the Slack dispatch path now suppresses same-turn duplicate deliveries and avoids preview/native-stream fallback double-posts that match the reported "edited message + fresh duplicate" behavior.

## Evidence

- **Explicit Slack reply tags still map to threaded transport:** Slack outbound keeps explicit reply targets as `replyToId`, and `resolveReplyTransport` maps that to the actual Slack reply transport instead of a separate thread field.
  - file: [extensions/slack/src/channel.ts:617](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/extensions/slack/src/channel.ts#L617)
  - command: `nl -ba extensions/slack/src/channel.ts | sed -n '606,623p'`

  - file: [extensions/slack/src/monitor.tool-result.test.ts:539](https://github.com/openclaw/openclaw/blob/c6684af682bcb59bd1ee85cb49f72d4074007e2c/extensions/slack/src/monitor.tool-result.test.ts#L539)

## Close Comment

Closing as implemented on current `main` after a Codex review. The current Slack delivery path already preserves explicit threaded replies for Slack (`extensions/slack/src/channel.ts:617`), suppresses duplicate same-turn deliveries in the Slack dispatcher (`extensions/slack/src/monitor/message-handler/dispatch.ts:146` and `extensions/slack/src/monitor/message-handler/dispatch.ts:470`), and has regression coverage for preview/native-stream fallback cases that previously produced the visible "edited message + fresh duplicate" symptom (`extensions/slack/src/monitor/message-handler/dispatch.preview-fallback.test.ts:323`). This behavior is also present in the latest shipped tag, `v2026.4.22`. If someone is still seeing a current reproduction on `v2026.4.22` or newer, it should be reopened as a fresh issue with exact config (`channels.slack.replyToMode` / `channels.slack.streaming`), whether it is a DM vs thread, and screenshots/logs showing whether Slack duplicated one message render or OpenClaw actually sent two deliveries.
