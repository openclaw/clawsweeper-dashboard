## Summary

Keep open: current main still treats enabled-only channel config as unconfigured in source, and the issue has an open linked fix PR, so it should remain open until that PR or a replacement fix lands.

## What This Changes

The issue asks OpenClaw to accept `channels.openclaw-weixin.enabled=true` as enough configuration to load the external Weixin channel plugin, or to report a clearer missing-config error.

## Best Possible Solution

Land a generic channel-presence fix that treats `enabled:true` as meaningful configuration for external channel startup while preserving `enabled:false` as disabled, with focused regression coverage for both auto-enable and Gateway startup planning.

## Reproduction Assessment

Yes, at source level: current main returns false for enabled-only channel config and Gateway startup derives configured external channels from that same presence signal. I did not live-run the packaged Weixin plugin in this read-only review.

## Solution Assessment

Yes, the best fix direction is a generic channel-presence/configured-state fix rather than a Weixin-specific loader exception. The linked PR should be reviewed or extended so the Gateway startup path is fixed too, not only the shared static helper.
