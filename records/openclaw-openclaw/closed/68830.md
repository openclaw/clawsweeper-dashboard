## Summary

Current `main` still computes hybrid `vectorScore`/`textScore` internally but does not expose them in the merged result or public `MemorySearchResult` types, so this PR is not already implemented and should stay open for maintainer review.

## Evidence

- **Hybrid merge still drops component scores on current main:** `mergeHybridResults()` stores `vectorScore` and `textScore` in its internal map, but its declared return type and the final returned object only expose `path`, `startLine`, `endLine`, `score`, `snippet`, and `source`. Consumers on `main` still cannot read the per-signal scores from merged hybrid results.
