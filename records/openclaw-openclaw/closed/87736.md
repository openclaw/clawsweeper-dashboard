## Summary

Keep this open: the report describes a real shipped regression on OpenClaw 2026.5.27, and current main still only treats structured `failure.reason` values as recoverable before throwing the raw preflight compaction error. The linked implementation PR at https://github.com/openclaw/openclaw/pull/87738 is the right place to review and land the narrow fix before this issue closes.

## What This Changes

Recover raw Codex app-server missing-thread preflight compaction failures so Telegram dispatch continues through the existing fresh-thread recovery path instead of surfacing a generic failure.

## Best Possible Solution

Land or revise the linked narrow PR so raw missing-thread compaction reasons enter the existing stale/missing binding recovery path, with focused regression coverage and live Telegram proof if feasible.

## Reproduction Assessment

Yes for a source-level repro: mock `compactEmbeddedAgentSession` to return `{ ok: false, compacted: false, reason: "thread not found: ..." }` and current main reaches the preflight throw because only `failure.reason` is classified. ## Solution Assessment
