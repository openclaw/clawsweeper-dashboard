## Summary

Current main already implements the central PR goal with a generic custom memory-slot status path: non-default memory plugins are probed through their registered runtime without requiring built-in memorySearch config or the default sqlite/qmd store. Main also routes JSON-output command logging to stderr at CLI startup, so the PR's per-probe subsystem filter is not the right change to merge. The implementation is on main after the latest release, with no containing release tag found yet.

## Best Possible Solution

Close this PR as obsolete on current main. Keep the generic memory-slot runtime status implementation and JSON-output stderr routing from main; if maintainers want a stronger guarantee against arbitrary third-party raw console writes during routed JSON commands, track that as a separate generic JSON-output integrity issue rather than merging this branch.
