## Label Justifications

- `P1`: The report describes a provider/session resume regression that can strip operator context from real Claude CLI sessions.
- `impact:session-state`: The affected behavior is resumed session state carrying a different system prompt context than fresh sessions.
- `impact:auth-provider`: The issue is tied to Claude CLI provider routing after switching authentication/provider mode.

## Summary

Close as implemented on main: the merged Anthropic Claude CLI fix re-sends the system prompt on resumed turns, current main retains the behavior and regression tests, and the fix is present in shipped release v2026.5.26.

## What This Changes

The issue reports that resumed Claude CLI sessions lose OpenClaw harness initialization files such as SOUL.md and IDENTITY.md because resume launches omit the system prompt file.

## Best Possible Solution

Keep the released resumed-prompt implementation; any new identity drift on v2026.5.26 or newer should be reported as a fresh issue with exact provider, version, resume command, and logs.

## Reproduction Assessment

Yes. The original regression is source-reproducible from the pre-fix `systemPromptWhen: "first"` default and `!useResume` gates, and the merged fix PR supplies real before/after Claude CLI marker proof.

## Solution Assessment

Yes. The released implementation is the right narrow fix because it updates the Claude CLI backend policy and the normal/live resumed argv gates while preserving explicit `first` and `never` behavior for other backends.
