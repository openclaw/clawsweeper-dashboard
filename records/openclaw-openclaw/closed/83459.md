## Summary

Keep open. Current main and the latest shipped v2026.5.18 path still leave the offline CLI cleanup dependent on a PATH `trash` command that the default Docker image does not install, and the related fix PR at https://github.com/openclaw/openclaw/pull/83472 was closed unmerged.

## What This Changes

The issue reports that the official Docker image lacks a trash provider, so `openclaw agents delete` can remove config state while leaving workspace and agent files behind.

## Best Possible Solution

Make offline CLI deletion use the shared fs-safe trash helper or an equally safe no-PATH fallback, with focused tests and Docker proof that unique workspaces are removed while shared workspaces are retained.

## Reproduction Assessment

Yes by high-confidence source inspection: the offline CLI delete path still calls the PATH `trash` command, and the default Docker runtime package list does not include `trash-cli`. I did not execute the Docker container in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to reuse the existing fs-safe trash helper or an equivalent safe no-PATH fallback, not to make every default Docker image install another package.

## Repair Work Prompt

Fix the Docker/minimal-Linux offline CLI cleanup bug reported in https://github.com/openclaw/openclaw/issues/83459. Current main's `src/commands/onboard-helpers.ts` shells out to PATH `trash` and logs on failure, while `src/commands/agents.commands.delete.ts` has already pruned config/session state before calling it. Implement a narrow fix that avoids requiring `trash-cli` in the default Docker image, preferably by routing `moveToTrash` through the existing `movePathToTrash` fs-safe helper with an explicit allowed root for configured cleanup targets, or by an equally safe no-PATH fallback. Preserve the shared-workspace retention behavior from https://github.com/openclaw/openclaw/issues/39701 and do not add `trash-cli` to the default Dockerfile package list unless maintainers explicitly choose that product direction. Add focused regression coverage for missing PATH trash and the existing agents-delete unique/shared workspace behavior. Include changelog if the repair is user-visible. Validate with `node scripts/run-vitest.mjs src/commands/onboard-helpers.test.ts src/commands/agents.delete.test.ts src/plugin-sdk/browser-maintenance.test.ts` and Docker/Testbox proof for unique workspace deletion with no `trash` binary plus shared-workspace retention.
