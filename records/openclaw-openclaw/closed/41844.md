## Summary

Current `main` now blocks the Discord self-feedback path that the reporter later identified as the root cause. The bot's own Discord messages are dropped before debounce and again during preflight, with regression tests and docs covering loop prevention.

## Evidence

- **Reporter root cause follow-up:** The issue comments say the failure was caused by accidentally allowlisting the Discord bot's own user ID, which fed bot-authored output back into OpenClaw and contaminated the session transcript rather than exposing a direct cron execution bug.
- **Early self-message drop in handler:** Discord inbound handling now returns before enqueueing when the inbound author matches `botUserId`, preventing self-authored bot messages from entering the debounce queue.
