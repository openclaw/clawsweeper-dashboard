Full review comments:

- **[P2] Keep template file loading out of reply rendering:** `src/auto-reply/usage-bar/template.ts:33`
  - body: `loadUsageBarTemplate` is called from the `/usage full` reply branch, and an uncached configured path still reaches `readFileSync`. Even though later calls are cached, the first read of a slow, blocked, or network-backed path can stall the gateway event loop while a user-facing reply is being prepared; preload or reload templates through a lifecycle/config path and keep reply rendering cache-only.
  - confidence: 0.86
- **[P2] Document the usage template contract before exposing it:** `docs/concepts/usage-tracking.md:46-48`

Since this PR adds a persisted operator config and a versioned template language, users need a supported format reference or example before the field is public.
