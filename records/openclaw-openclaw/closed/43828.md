Closing as implemented on `main` after a Codex review. The current code already preserves live assistant text across the exact class of streaming/tool-boundary truncation described here: the gateway now merges segmented/non-prefix assistant deltas into a per-run buffer and flushes the merged text before `final` (`src/gateway/server-chat.ts:109`, `src/gateway/server-chat.ts:685`, `src/gateway/server-chat.ts:762`), and the TUI stream assembler preserves previously streamed text when later payloads temporarily drop earlier text blocks around tool boundaries (`src/tui/tui-stream-assembler.ts:80`). There is direct regression coverage for both layers in `src/gateway/server-chat.agent-events.test.ts:417` and `src/tui/tui-stream-assembler.test.ts:23`, and the shipped release notes call out the gateway-side live-stream retention fix in `CHANGELOG.md:2953`. This behavior is present in released `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
