## Summary

Keep open: current main still performs the synchronous installed-plugin index read while building the loader cache context before the registry cache lookup, and https://github.com/openclaw/openclaw/pull/77379 is an open linked fix PR with closing syntax.

## What This Changes

Request to remove redundant synchronous installed-plugin index reads from the `resolvePluginTools` hot path by memoizing or reusing install records across loader-cache hits.

## Best Possible Solution

Review, rebase, and land https://github.com/openclaw/openclaw/pull/77379 or an equivalent invalidation-aware memoization fix, then close this issue once the fix reaches main.

Yes. Source inspection on current main shows the sync installed-index read happens before the loader registry cache can be reused, and the discussion includes live OpenClaw 2026.5.7 timing data matching that path.

## Solution Assessment

Yes, conditionally. The linked PR direction is the narrow maintainable fix because it removes repeated hot-path sync reads while preserving freshness through existing cache invalidation paths.
