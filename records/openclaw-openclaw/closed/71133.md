## Summary

Keep open. Current `main` still matches the reported failure path: `memory-core` narrative cleanup calls `subagent.deleteSession()`, the gateway plugin fallback runtime dispatches `sessions.delete` with a synthetic operator client that defaults to `operator.write`, and gateway auth still requires `operator.admin` for `sessions.delete`. A current unit test explicitly asserts the fallback delete fails with `missing scope: operator.admin`, which is the issue's reported behavior.
