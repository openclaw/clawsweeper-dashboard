## Summary

Current `main` already implements the reported cron-model-precedence fix: isolated cron resolves per-job `payload.model` ahead of agent/default selection, retries on `LiveSessionModelSwitchError` with the switched model, and has targeted regression coverage for the reported Opus-vs-Sonnet scenario.

## Evidence

- **Payload model precedence is explicit in current code:** `resolveCronModelSelection()` applies `payload.model` first, then only falls back to stored session override when no payload override is present, and finally returns agent/default selection otherwise.
  - file: [src/cron/isolated-agent/model-selection.ts:104](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/cron/isolated-agent/model-selection.ts#L104)
  - command: `nl -ba src/cron/isolated-agent/model-selection.ts | sed -n '1,180p'`
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Live-session model switch is retried instead of letting agent primary win:** `run-executor` catches `LiveSessionModelSwitchError`, updates `liveSelection`, persists the cron session entry, and retries up to the bounded limit instead of treating the requested model as a failed candidate and silently settling on the agent primary model.
  - file: [src/cron/isolated-agent/run-executor.ts:300](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/cron/isolated-agent/run-executor.ts#L300)

  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Regression test covers the reported Opus->Sonnet switch path:** The test simulates first-attempt `LiveSessionModelSwitchError` with agent default Opus and asserts the second attempt reruns with Sonnet; a companion test asserts the switched provider/model is persisted before retry.
  - file: [src/cron/isolated-agent/run.live-session-model-switch.test.ts:106](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/cron/isolated-agent/run.live-session-model-switch.test.ts#L106)

- **Regression test blocks hidden fallback to agent primary:** A dedicated test asserts that when cron `payload.model` is set, OpenClaw passes an explicit empty `fallbacksOverride` and does not append the agent primary model as a hidden extra retry target.
  - file: [src/cron/isolated-agent/run.cron-model-override-forwarding.test.ts:150](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/cron/isolated-agent/run.cron-model-override-forwarding.test.ts#L150)
