## Summary

Keep open: the feature direction is useful, but the branch currently exposes raw transcript previews through a new built-in session tool, bypassing the safety-filtered `sessions_history` contract, and the supplied proof is still test-only.

## What This Changes

The PR adds a built-in `sessions_search` agent tool with catalog, policy, display, docs registration, and focused tests for exact transcript search over visible sessions.

## Best Possible Solution

Implement `sessions_search` on the existing session visibility and session-history safety boundary: search or preview sanitized content, add regression coverage for redaction/tool scaffolding, and then provide a real redacted tool-run proof.

## Reproduction Assessment

Yes for the review finding: source inspection of the PR branch shows raw transcript lines are read and returned as previews. I did not run a live PR-branch repro because this was a read-only review.

## Solution Assessment

No. Reusing `sessions_list` for visibility is the right direction, but returning raw transcript snippets from a built-in session search bypasses the established sanitization and redaction contract.
