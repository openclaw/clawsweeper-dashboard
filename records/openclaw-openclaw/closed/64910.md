## Summary

Current main addresses the reported WebChat indefinite-loading failure for stalled model streams. The issue was filed against 2026.4.9; main now wraps embedded LLM streams with an idle watchdog, aborts the run when chunks stop arriving, surfaces timeout/error output instead of orphaning the turn, and the Control UI clears its active run on terminal chat events. The relevant timeout hardening is documented in the 2026.4.10 changelog and is present on current main.

## Best Possible Solution

Close this issue as implemented. Keep the shipped LLM idle-timeout and terminal chat-event behavior; if GLM `zai/glm-5v-turbo` still hangs on current releases, the best follow-up is a new focused bug with gateway logs, WebSocket `chat` events, model/provider config, and whether the stall occurs before first token or after partial output.
