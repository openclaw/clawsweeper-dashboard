## Summary

Current main already handles this path: final `MEDIA:` directives are parsed into `mediaUrls`, workspace-local absolute media is staged through the shared outbound media normalizer, and Slack replies with `mediaUrl` go through Slack file upload with the thread timestamp preserved. The proof commit is on `main` after the latest release, and no tag currently contains it.

## What This Changes

The issue requests that Slack thread replies containing valid `MEDIA:/absolute/path/...` directives be converted into uploaded Slack files instead of visible text.

## Best Possible Solution

Keep the current shared directive parsing and media staging path, and let the next release carry this main-only fix; new reports should include logs only if the same behavior reproduces on a build containing the proof commit.

## Reproduction Assessment

Yes. The source-level reproduction is a final agent payload containing a standalone `MEDIA:` line; current tests cover conversion into `mediaUrls`, staging of workspace paths, and Slack delivery uses those media URLs for upload.

## Solution Assessment

Yes. The current approach is the narrow maintainable path: shared reply parsing/staging remains channel-agnostic, while Slack owns only the upload and thread delivery step.
