## Summary

Current main already wires WhatsApp group turns into message-tool-only source reply mode and documents the expected behavior, so the reported routing-layer gap from v2026.5.7 is fixed on main but not yet in a tagged release.

## What This Changes

WhatsApp group chat replies should be routed through the message tool when `visibleReplies: "message_tool"` is configured.

## Best Possible Solution

Keep the current main behavior: `message_tool` mode should route group turns through message-tool-only prompting, while users who want normal final answers auto-posted should set `messages.groupChat.visibleReplies: "automatic"`.

## Reproduction Assessment

No current-main failing reproduction was established. Source inspection and regression coverage show WhatsApp group turns now receive `sourceReplyDeliveryMode: "message_tool_only"`, which addresses the reported routing layer.

## Solution Assessment

Yes. The current implementation is the narrow maintainable fix because it wires the existing cross-channel source reply policy into WhatsApp dispatch instead of adding WhatsApp-only auto-send behavior.
