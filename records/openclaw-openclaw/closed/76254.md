## Summary

Current main and release v2026.5.27 already implement the macOS transitive ancestor walk, so the stale-gateway cleanup gap reported here is fixed. The reporter’s follow-up correctly separates the original live SIGTERM loop into a different open issue, but this specific macOS `getSelfAndAncestorPidsSync` behavior no longer remains.

## What This Changes

The issue asks for macOS stale-gateway cleanup to walk beyond the immediate parent PID so nested caller descendants do not kill their live gateway ancestor.

## Best Possible Solution

Keep the shipped macOS `ps` ancestor walk for stale-gateway cleanup, and handle any remaining gateway-wide SIGTERM symptoms under the separate kill-tree follow-up.

## Reproduction Assessment

Yes. The pre-fix source path returned after `process.ppid` on non-Linux platforms, and current regression coverage models the nested macOS caller that used to leave the gateway grandparent killable.

## Solution Assessment

Yes. The current implementation is the narrow maintainable fix: reuse the existing ancestor-exclusion path and add a Darwin parent reader, without adding config or changing unrelated cleanup policy.
