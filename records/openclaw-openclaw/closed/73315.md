## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Load Gateway config through canonical contracts:** `apps/desktop/src-tauri/src/main.rs:296-298`
  - body: This parses `openclaw.json` with strict `serde_json` and hand-extracts a subset of Gateway fields, so valid OpenClaw configs using JSON5, `$include`, or SecretRef-managed Gateway auth can be treated as missing or different from the running Gateway. The desktop app should consume an existing config snapshot, CLI contract, or Gateway dashboard contract instead of carrying a partial Rust copy.
  - confidence: 0.9
- **[P2] Include the configured Control UI base path:** `apps/desktop/src-tauri/src/main.rs:492-504`
  - body: `build_dashboard_url()` always embeds `/`, but current Gateway config supports `gateway.controlUi.basePath` and the canonical dashboard link helper appends that path. When the Control UI is mounted under a path such as `/openclaw`, the iframe opens the wrong route.
  - confidence: 0.9
- **[P2] Permit the dashboard URLs this app can generate:** `apps/desktop/src-tauri/tauri.conf.json:14`
  - body: The Rust URL builder can produce HTTPS dashboard URLs and custom-host URLs, but this CSP only allows HTTP/WS localhost and 127.0.0.1 frame/connect origins. Valid generated dashboard URLs can be blocked by the WebView policy.
  - confidence: 0.86
- **[P2] Avoid restarting through an ambient shell command:** `apps/desktop/src-tauri/src/main.rs:540-558`
  - body: The tray/UI restart action invokes `openclaw` through `cmd` or `sh -lc`, so a first-party desktop control executes whichever binary the ambient shell resolves. Resolve a trusted CLI path or use a controlled Gateway restart surface before exposing this as a GUI action.
  - confidence: 0.88
- **[P2] Use the repo typecheck lane instead of raw tsc:** `apps/desktop/package.json:12`
  - body: This adds a package script that runs `tsc --noEmit`, but repo policy keeps typechecking on the `tsgo` and `check:test-types` lanes. Adding a raw `tsc` script gives contributors and CI another unsupported typecheck path for this workspace.
  - confidence: 0.84
- **[P3] Add labeler coverage for the new app surface:** `pnpm-workspace.yaml:3`
  - body: Registering `apps/desktop` creates a new app surface, but `.github/labeler.yml` has no desktop-app rule. Future desktop changes would miss the app ownership label required for new app/plugin/doc surfaces.
  - confidence: 0.8
