## Summary

Current `main` already preserves an established external delivery route when dashboard/webchat touches a session, and it reuses that persisted route for event-driven replies. This PR’s `persistedOriginChannel` / `persistedOriginTo` approach is redundant against the broader `deliveryContext` / `lastChannel` / `lastTo` solution already on `main`.

## Evidence

- **Dashboard/webchat no longer overwrites external route:** `resolveLastChannelRaw` explicitly preserves an established external route when the incoming channel is internal webchat, with an inline note that this prevents subagent completions from being delivered to the dashboard instead of the original channel.
