## Summary

Current `main` already supports cron over `POST /tools/invoke` via the generic tool-invoke path, with an explicit security gate: cron is denied by default on HTTP, but `gateway.tools.allow: ["cron"]` removes that default deny. The request/response surface and cron action/schema docs are now present, and regression coverage proves a 200 response for cron once explicitly allowed.

## Risks / Open Questions

- Cron over HTTP remains opt-in, not default: a bridge that assumes stock `/tools/invoke` access to `cron` will still get 404 until the operator sets `gateway.tools.allow: ["cron"]`.
- The enabling work is on `main` at `0ad058a9cbc67842d64eff6f4c1a89bf1f44779c`, but it does not appear to be in the latest tagged release `v2026.4.22`.
