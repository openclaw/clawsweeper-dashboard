## Summary

Current main has moved the affected ACPX path past the OpenClaw 2026.3.13 failure mode: Feishu still dispatches through the normal agent route, `sessions_spawn(runtime="acp")` now initializes an ACP child session through the ACP control plane, the bundled ACPX plugin embeds `acpx/runtime` instead of shelling through the old external CLI hop, and early `acpx exited with code 1` turn failures are retried once with a fresh handle. The reporter discussion also has a later validation comment confirming the command-layer/config problem was resolved after using the ACPX wrapper layer and fixing Codex auth.
