## Summary

Current main addresses the traced TUI busy-loop path: context-window warmup is process-cached, warmup model discovery skips provider/plugin normalization, models.json planning uses a stable ready cache, and provider discovery can reuse plugin metadata instead of repeatedly reloading manifests.

## What This Changes

The issue requests that `openclaw-tui` stop pegging CPU at idle or during turns by removing the context/model/plugin discovery busy-loop.

## Best Possible Solution

Keep the current cached context/model/plugin metadata implementation, and track any remaining orphaned TUI lifecycle behavior separately in #75289.
