## Summary

Keep this open: current `main` still has the source-level `NO_REPLY` silent-turn misclassification, and the linked open fix PR is the right ownership path rather than closing the issue before merge.

## What This Changes

The issue reports that `NO_REPLY`-only assistant turns are counted as empty failures, triggering retry, fallback classification, and sticky model override behavior during heartbeats.

## Best Possible Solution

Land a narrow embedded-runner fix that treats exact terminal `NO_REPLY` assistant text as successful `silent-empty` only when silent replies are allowed, while preserving error, side-effect, and post-tool guards with regression coverage.

## Reproduction Assessment

Yes. Current source shows exact `NO_REPLY` payloads are filtered to zero and then rejected as intentional silence because the classifier checks for non-empty assistant text before recognizing the silent token.

## Solution Assessment

Yes, the best path is a narrow runner-classification fix plus regression tests; the linked open PR appears to target that path, so this issue should remain open for PR review and merge rather than a parallel repair.
