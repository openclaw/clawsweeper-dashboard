## Summary

Current main, and the shipped `v2026.4.23` release, now handle this as an explicit `metadata-upgrade` approval flow instead of a silent reconnect loop: remote/browser clients still require re-approval by design, but the gateway returns structured metadata-upgrade details, the companion UI surfaces an approval-needed state, and the app pauses reconnects so it does not keep spinning or generate repeated pending requests.

## Evidence
