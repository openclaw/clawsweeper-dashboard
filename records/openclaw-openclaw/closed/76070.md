## Summary

Close as superseded. The same Feishu topic mention-control work is already tracked by open PR #76020, which is the more complete implementation path, and this PR misses normal group thread replies because it only treats `chat_type: "topic_group"` as topic context.

## What This Changes

The PR adds Feishu `topicRequireMention` schema and reply-policy handling, passes an `isTopic` flag from the inbound handler, and adds policy unit tests.

## Best Possible Solution

Use #76020 as the canonical implementation, or land an equivalent Feishu-owned change that keys topic replies from `thread_id`, documents the new config, and preserves fallback to `requireMention` when unset.

## Reproduction Assessment

Yes. Configure `requireMention: true` and `topicRequireMention: false`, then send an unmentioned Feishu event with `chat_type: "group"` and `thread_id`; this PR leaves `isTopic` false and the message still follows the normal mention gate.

## Solution Assessment

No. The Feishu-owned additive config is the right direction, but this PR is not the best active solution because it misses the existing `thread_id` reply path and lacks the docs/changelog/schema coverage already present in #76020.
