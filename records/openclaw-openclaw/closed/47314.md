## Summary

Current `main` already implements the Bailian/GLM-5 streaming-usage path: DashScope Coding endpoints are treated as native streaming-usage-compatible OpenAI-completions endpoints, requests include `stream_options.include_usage`, and the completions stream parser now captures top-level `chunk.usage` even when the chunk has no choices.

## Evidence

- **DashScope Coding endpoints are classified as native ModelStudio endpoints with streaming-usage compat:** `https://coding.dashscope.aliyuncs.com/v1` is in `MODELSTUDIO_NATIVE_BASE_URLS`, and `supportsNativeStreamingUsageCompat` is true for `modelstudio-native` endpoints.
  - file: [src/agents/provider-attribution.ts:130](https://github.com/openclaw/openclaw/blob/3d8e6a3aa3a26870c4ae61fd27853914fc66cf96/src/agents/provider-attribution.ts#L130)
  - sha: [3d8e6a3aa3a2](https://github.com/openclaw/openclaw/commit/3d8e6a3aa3a26870c4ae61fd27853914fc66cf96)
- **Generic providers on Bailian/DashScope Coding endpoints now request streaming usage:** `buildOpenAICompletionsParams()` always sets `stream_options: { include_usage: true }`, and there is a regression test specifically covering a generic provider pointed at `https://coding.dashscope.aliyuncs.com/v1`.
  - file: [src/agents/openai-transport-stream.ts:1725](https://github.com/openclaw/openclaw/blob/3d8e6a3aa3a26870c4ae61fd27853914fc66cf96/src/agents/openai-transport-stream.ts#L1725)
  - sha: [3d8e6a3aa3a2](https://github.com/openclaw/openclaw/commit/3d8e6a3aa3a26870c4ae61fd27853914fc66cf96)
- **Regression test covers generic GLM-5 provider on Bailian Coding endpoint:** Test `enables streaming usage compat for generic providers on native DashScope endpoints` uses `provider: "generic"`, `id: "glm-5"`, and `baseUrl: "https://coding.dashscope.aliyuncs.com/v1"`, and expects `include_usage: true`.
  - file: [src/agents/openai-transport-stream.test.ts:1451](https://github.com/openclaw/openclaw/blob/3d8e6a3aa3a26870c4ae61fd27853914fc66cf96/src/agents/openai-transport-stream.test.ts#L1451)
  - sha: [3d8e6a3aa3a2](https://github.com/openclaw/openclaw/commit/3d8e6a3aa3a26870c4ae61fd27853914fc66cf96)
- **Stream parser captures usage before requiring a choice payload:** `processOpenAICompletionsStream()` assigns `output.usage = parseTransportChunkUsage(chunk.usage, model)` before checking `chunk.choices[0]`; chunks with `choices: []` therefore still update usage.
  - file: [src/agents/openai-transport-stream.ts:1295](https://github.com/openclaw/openclaw/blob/3d8e6a3aa3a26870c4ae61fd27853914fc66cf96/src/agents/openai-transport-stream.ts#L1295)
  - sha: [3d8e6a3aa3a2](https://github.com/openclaw/openclaw/commit/3d8e6a3aa3a26870c4ae61fd27853914fc66cf96)
- **OpenClaw already models the final OpenAI-compatible usage chunk as `choices: []`:** Gateway stream code emits a final usage chunk with `choices: []`, and its test asserts that shape explicitly.
  - file: [src/gateway/openai-http.ts:193](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/gateway/openai-http.ts#L193)
