## Summary

Keep open: current main refreshes skill snapshots, but the model-visible full and compact skills prompts still render only name, description, and/or location, with no per-skill version or changed-skill cue.

## What This Changes

Expose per-skill version metadata in the available skills prompt so a live session can tell which SKILL.md changed and re-read only that skill.

## Best Possible Solution

Add a deterministic per-skill change marker to both full and compact skills prompts, plus guidance and regression tests showing that body-only SKILL.md changes become model-visible without adding a new config option.

## Reproduction Assessment

Yes. Source inspection shows the snapshot version is refreshed at the snapshot level, while both available_skills renderers omit any per-skill marker, so a body-only SKILL.md edit has no specific model-visible change cue.

## Solution Assessment

Yes, if scoped to core skills prompt metadata. The watcher already owns refresh; the narrow maintainable fix is to surface deterministic per-skill change identity in the existing prompt path and keep the token-budget behavior tested.
