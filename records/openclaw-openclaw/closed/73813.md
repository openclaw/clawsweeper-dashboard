## Summary

Keep open. Current main still has the cron bookkeeping gap: embedded yielded runs expose paused/yielded state, but isolated cron still turns the first embedded return into terminal `ok` status and duration before descendant work and resumed parent processing finish.

## What This Changes

The issue requests accurate isolated cron status and duration when `sessions_yield` pauses a parent run while spawned subagents continue.

## Best Possible Solution

Define the yielded isolated-cron lifecycle contract and update cron bookkeeping so yielded parent/subagent work is not reported as terminal success until the resumed parent reaches a real terminal outcome, unless maintainers deliberately choose an explicit partial-yielded outcome field.

## Reproduction Assessment

Yes. Source inspection shows a yielded embedded run returns paused/yielded metadata, while isolated cron finalization still records `ok` and duration from the first returned result without waiting for descendant completion or resumed parent work.

## Solution Assessment

Unclear. Deferring `finished` best matches the reported monitoring semantics, but adding yielded metadata is a smaller compatible step; maintainers should choose the public cron run-log contract before implementation.
