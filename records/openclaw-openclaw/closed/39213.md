## Summary

Keep PR #39213 open. Current main still rotates Telegram answer previews across assistant-message boundaries in `streamMode: "partial"`, the PR's stream-revival/coalescing mechanism is absent, and current tests still assert forced new preview messages in partial mode. The PR also has an explicit maintainer hold requiring rebase/alignment with #38906 and boundary-regression fixes before merge.

## Best Possible Solution

Keep #39213 open as the active Telegram partial-preview coalescing work, or replace it with a rebased PR under `extensions/telegram`. The implementation should gate answer-preview boundary rotation to block mode, add a safe partial-mode stream revival/refinalization path that preserves the preview message ID, align with current #38906 DM message-transport behavior, preserve block-mode rotation, and add focused regression coverage for early partials, sequential tool-call rounds, media-only assistant boundaries, and reasoning/answer lane separation.
