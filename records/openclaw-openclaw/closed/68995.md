## Summary

## What This Changes

Allow memory-flush write restrictions to accept structured paths under `memory/`, such as `memory/daily/YYYY-MM-DD.md`, instead of only the single root-level daily file.

## Best Possible Solution

Track one explicit, validated memory-flush additional-write-target contract in https://github.com/openclaw/openclaw/issues/66612, carrying this `memory/daily/YYYY-MM-DD.md` variant into that design.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with a flush plan target of `memory/YYYY-MM-DD.md`, a write to `memory/daily/YYYY-MM-DD.md` resolves to a different absolute path and the current wrapper throws.

## Solution Assessment

No. The raw prefix-check sketch is not the best fix because the append target remains the planned file and the sandbox expansion needs an explicit validated target contract.
