## Summary

Current main already covers the reported cron behavior: manual `cron run` is now an enqueue-only ack instead of a misleading immediate success result, main-session cron jobs explicitly enqueue the system event and wake/run heartbeat delivery on `target: "last"`, main-session delivery state is recorded as `not-requested` instead of `unknown`, and isolated cron turns still go through the normal embedded tool stack that includes allowed plugin tools such as `browser`. These changes are already present in the shipped `v2026.4.22` release.
