## What This Changes

The issue reports that standalone helper module files in the user plugin extensions root are treated as native plugin candidates and can block Gateway startup when no plugin manifest exists.

## Best Possible Solution

Keep explicit configured file-path plugins working, but make automatic plugin-root scans ignore unrelated standalone helper files or downgrade them to a clear non-fatal diagnostic, with docs updated only if user guidance changes.
