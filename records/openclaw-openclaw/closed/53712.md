## Summary

Current `main` already prevents pre-compaction memory flush from rewriting arbitrary workspace files. Memory-triggered runs are limited to `read` plus an append-only `write` for one canonical `memory/YYYY-MM-DD.md` path, and the same implementation is present in release `v2026.4.22`, so the reported `NOW.md` rewrite race is obsolete on current builds.

## Evidence

- **Flush run passes a single canonical write target:** `runMemoryFlushIfNeeded` resolves `activeMemoryFlushPlan.relativePath` and passes it into the embedded run as `memoryFlushWritePath` for `trigger: "memory"`.
  - file: [src/auto-reply/reply/agent-runner-memory.ts:739](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/auto-reply/reply/agent-runner-memory.ts#L739)

  - file: [src/agents/pi-tools.ts:348](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/pi-tools.ts#L348)
