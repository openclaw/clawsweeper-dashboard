## What This Changes

## Best Possible Solution

Land the split-key fix after maintainer beta-gate acceptance, keeping context/history identity on the canonical run session and sandbox/tool policy on the sandbox session key.

## Reproduction Assessment

## Solution Assessment

Yes. Separating contextSessionKey from sandboxSessionKey is the narrowest maintainable fix because it restores context continuity while preserving sandbox/tool/runtime policy scoping.
