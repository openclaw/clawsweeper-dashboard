Full review comments:

- **[P1] Route remote_trigger through the SSRF guard:** `src/agents/tools/claw-compat-tools.ts:635-640`
  - body: This tool performs a direct fetch on a model-supplied URL, bypassing the existing web_fetch SSRF guard that blocks localhost and private-network targets before fetch is called. Route this through the guarded web fetch path or remove the tool.
  - confidence: 0.95
- **[P1] Stop returning raw config to the model:** `src/agents/tools/claw-compat-tools.ts:577-583`
  - body: When path is omitted, config_compat returns the full runtime config object into tool output, and path can select sensitive subtrees. This can expose auth/provider secrets or SecretRef-bearing config to the model context; use an explicit allowlist or remove the readback surface.
  - confidence: 0.92
- **[P1] Gate config writes behind owner authorization:** `src/agents/tools/claw-compat-tools.ts:551-555`
  - body: config_compat writes parsed settings with writeConfigFile for any caller that has the coding-profile tool. Persisted config mutation is a control-plane operation and needs owner-only gating and the repo's config mutation boundary, or it should be removed from this compatibility tool.
  - confidence: 0.94
- **[P1] Restrict plan mode tools before writing config:** `src/agents/tools/claw-compat-tools.ts:238-239`
  - body: enter_plan_mode and exit_plan_mode persist permissions.defaultMode through writeConfigFile but are not owner-only. Non-owner tool access should not be able to change persisted permission defaults.
  - confidence: 0.9
- **[P2] Do not claim user message delivery without dispatch:** `src/agents/tools/claw-compat-tools.ts:191-192`
  - body: send_user_message returns delivered=true but only echoes JSON and never calls the real message transport. In message-tool-only flows this can silently drop visible user output while telling the model delivery succeeded.
  - confidence: 0.91
- **[P1] Parse LSP frames with byte slices:** `src/agents/pi-bundle-lsp-runtime.ts:78-84`
  - body: Content-Length is a byte count, but this parser slices a JavaScript string with that byte count. Multibyte LSP payloads can desynchronize parsing and time out even when the server responded; use a Buffer-based parser like current main's agent-bundle LSP runtime.
  - confidence: 0.96
- **[P1] Dispose failed LSP startup processes:** `src/agents/pi-bundle-lsp-runtime.ts:477-480`
  - body: If initializeSession or later setup throws, the catch block only logs and continues; the child process was spawned but is not pushed into sessions, so dispose cannot clean it up. Kill or dispose the session in the per-server catch path.
  - confidence: 0.88
- **[P2] Resolve safe MCP server names before dispatch:** `src/agents/tools/mcp-tool.ts:151-153`
  - body: The catalog exposes safeServerName for provider-safe tool names, but mcp call dispatch forwards the parsed server prefix directly to runtime.callTool, whose sessions are keyed by the original serverName. Qualified names built from safeServerName will fail for sanitized server keys.
  - confidence: 0.82
- **[P2] Use flat enum schemas for tool actions:** `src/agents/tools/mcp-tool.ts:9-12`
  - body: The new tool schemas use Type.Union over literals, producing anyOf schemas for action fields. Provider tool schemas in this repo prefer flat string enum helpers because some providers reject anyOf/oneOf/allOf tool inputs.
  - confidence: 0.78
