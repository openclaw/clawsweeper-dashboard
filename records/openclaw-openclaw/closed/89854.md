## Summary

Current main already handles the central heartbeat problem: the runtime reads workspace HEARTBEAT.md itself and injects the file's non-task directives into the heartbeat prompt, so a visible model read-tool call is no longer required for HEARTBEAT.md context to reach the model.

## What This Changes

The issue asks for heartbeat turns to honor HEARTBEAT.md instructions instead of returning HEARTBEAT_OK without reading or acting on the workspace heartbeat flow.

## Best Possible Solution

Keep the runtime-owned HEARTBEAT.md preflight read and prompt directive injection; if a provider still ignores injected directives, track that as a new provider-specific prompt-following report with a prompt trace.

## Reproduction Assessment

No. I did not establish a current-main failing reproduction; source and regression tests show current heartbeat runs read HEARTBEAT.md and inject its directives before the model call.

## Solution Assessment

Yes. Reading HEARTBEAT.md in the runtime and injecting its directives is the safer maintained solution because it does not depend on every model/provider choosing to make a visible read-tool call.
