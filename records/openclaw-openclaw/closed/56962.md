## Summary

Current `main` has the missing Control UI local slash-command interception path for `/compact`: the chat send handler now routes local slash commands before normal message send, `compact` is marked as a local command, and the local executor calls `sessions.compact` with visible feedback. The older `v2026.4.23` release still lacked that interception block, matching the report.

## Evidence

- **Control UI now intercepts local slash commands before normal send:** `handleSendChat` parses slash commands and, when `executeLocal` is set, dispatches them locally instead of falling through to the normal chat send path. That directly addresses the reported symptom where `/compact` vanished without reaching the Gateway.
  - file: [ui/src/ui/app-chat.ts:354](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/ui/src/ui/app-chat.ts#L354)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **`/compact` is classified as a local slash command:** `compact` is in the local-command set, and `toSlashCommand` marks local commands with `executeLocal: true`, so the parser/dispatcher can recognize `/compact` as client-side RPC-backed behavior.
  - file: [ui/src/ui/chat/slash-commands.ts:82](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/ui/src/ui/chat/slash-commands.ts#L82)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Local `/compact` execution now calls Gateway `sessions.compact` and renders feedback:** `executeCompact` performs `client.request("sessions.compact", { key: sessionKey })` and returns success/skip/failure content, so Control UI should no longer be silent even when compaction is skipped or fails.
  - file: [ui/src/ui/chat/slash-command-executor.ts:151](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/ui/src/ui/chat/slash-command-executor.ts#L151)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Released tag still matched the bug; current main differs:** Inspecting `v2026.4.23:ui/src/ui/app-chat.ts` shows the old release path went from the `/btw` handling straight to normal send logic, without the current local slash-command interception block. That matches the original report and shows the fix is on `main` after the latest release.
