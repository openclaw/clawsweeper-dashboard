## Summary

Keep open. Current main still lacks the QQBot slash-command auth, pre-dispatch registration, clear-storage path, and slash-command file-delivery fixes, so the PR is not obsolete; however, it is an active security-sensitive implementation with unresolved review findings and scope/release metadata drift that needs maintainer review before merge.

## What This Changes

The PR changes QQBot slash-command registration and dispatch to compute admin auth from allowFrom, add c2cOnly gating and /bot-me, fix clear-storage download-root scanning and log-file delivery, add an engine import-boundary test, bump QQBot plugin version metadata, and increase the QQBot outbound response timeout.

## Best Possible Solution

Land a reviewed, narrowed QQBot fix that hardens admin slash-command authorization, keeps auth-gated commands reachable in the QQ pre-dispatch path, cleans the actual downloads root, and allows slash-command log attachments to send, while preserving intended group access for safe commands, handling dm/private surfaces deliberately, and adding focused regression tests plus approved release metadata.
