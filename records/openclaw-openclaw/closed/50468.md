## Summary

Current main and the v2026.4.26 release implement the requested Chokidar v5 fix for the skills watcher: skill roots are watched as concrete directories, non-SKILL.md churn is filtered, and watcher events bump the skills snapshot version so existing sessions refresh on the next turn. The closed PR #50473 proposed the same direction, and the shipped implementation now covers it. The separate restart/config cache-invalidation gap is tracked by #73765 and does not keep this Chokidar glob issue open.

## Best Possible Solution

Keep the shipped v2026.4.26 concrete-root watcher implementation for this report. Any remaining cache invalidation work after restarts or config edits should be handled in the separate #73765 thread rather than keeping this Chokidar glob issue open.
