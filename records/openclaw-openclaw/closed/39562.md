## Summary

Current `main` already has an outbound `message_sending` plugin hook that runs immediately before delivery, exposes the outgoing text plus routing context, and supports rewrite (`content`) or block (`cancel`) semantics. Discord outbound delivery uses the shared outbound pipeline, so the requested minimal pre-send allow/block/rewrite hook is already present on `main`.

## Evidence
