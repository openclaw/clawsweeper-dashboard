## Summary

Keep open. Current main still omits `xhigh` from the Amazon Bedrock provider thinking profile for Claude Opus 4.7, and the non-empty plugin profile returns before the legacy xhigh fallback can append it. Open PRs #74740 and #74747, plus earlier PR #70821, target the same Bedrock profile gap, so this issue should stay open pending maintainer review or dedupe of those implementation candidates.

## What This Changes

Request that the `amazon-bedrock` provider advertise the correct thinking levels for Claude Bedrock models, especially `xhigh` for Claude Opus 4.7 and adaptive parity for Claude/Sonnet 4.6.

## Best Possible Solution

Land or revise one canonical Bedrock provider fix that exposes `xhigh` for regional, dotted, and ARN-shaped Claude Opus 4.7 Bedrock ids using the existing Bedrock matcher or a Bedrock-normalized shared helper, with regression tests for matching Opus 4.7 ids and non-matching models, then close or supersede the overlapping PRs.
