## Summary

Current main treats `webchat` as an internal-only channel, so `heartbeat.target = "last"` in a direct webchat session intentionally resolves to no external delivery target. That does not block the heartbeat turn itself: the runner still calls the model and returns `status: "ran"`; `openclaw system heartbeat last` only reports the last heartbeat event, which can be `skipped/no-target` for delivery even when the heartbeat work ran internally.

## Evidence
