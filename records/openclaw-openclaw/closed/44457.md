## Summary

Current main already covers the reported Ollama/Qwen no-think use case through per-model thinking defaults plus native Ollama `think` forwarding, even though it does not use the exact proposed `models.providers.ollama.models[].options` shape.

## Evidence

- **Per-model thinking config exists:** `resolveThinkingDefault` reads `agents.defaults.models["<provider>/<model>"].params.thinking` before global defaults, so thinking can be configured per model.

  - file: [src/agents/model-selection.test.ts:1299](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/agents/model-selection.test.ts#L1299)
