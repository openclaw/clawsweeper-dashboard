## What This Changes

The PR deletes abort/task entries and marks a channel stopped on `stopChannel` timeout, with a focused test changed to expect restart after timeout.

## Best Possible Solution

## Solution Assessment

No. The direct cleanup is not the safest fix because it releases the slot before the old lifecycle is fenced, and it conflicts with current main's manual-versus-recovery timeout split.
