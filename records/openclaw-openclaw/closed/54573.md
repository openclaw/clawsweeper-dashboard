## Summary

This PR should close because it is a low-signal upstream deployment-default change whose safe action is per-deployer configuration, not an OpenClaw repo change. Current main intentionally keeps the Render Blueprint on `starter` with a persistent `/data` disk, while the Render docs tell users who want free tier to change their fork and accept state resets. The existing bot review comments correctly identified that this PR removes the only persistent storage backing the configured state and workspace paths.

## Best Possible Solution

Keep the upstream Render Blueprint durable by default: `starter` plan plus the `/data` persistent disk. Users who want the no-cost Render trade-off should follow the public Render install docs and change `plan: free` in their own fork/deployment, accepting that state resets. If OpenClaw ever wants a diskless upstream default, it needs a separate maintainer-owned PR that moves state/workspace to writable ephemeral locations and documents the data-loss behavior.
