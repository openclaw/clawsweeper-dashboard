## What This Changes

The issue reports that `openclaw memory search <query> --json` emits valid JSON but leaves the process alive when `memory.backend=qmd` is active.

## Best Possible Solution

Make the QMD-backed `memory search --json` CLI path emit JSON and terminate cleanly by fixing or bounding the CLI-owned QMD manager/process cleanup path, then add regression proof that a successful JSON search exits.

## Reproduction Assessment

Yes, there is a high-confidence reproduction recipe from the report: run `openclaw memory search "QMD rebuild method" --json` with `memory.backend=qmd` on v2026.6.5. I did not run a live QMD setup here, but current main still follows the same JSON-return and cleanup path.

## Solution Assessment

No, the proposed local `process.exit` hotfix is a useful signal but not clearly the best permanent fix because it can mask leaked handles. The better fix is to identify the QMD/manager handle that survives after JSON output and close or bound it in the CLI-owned path.
