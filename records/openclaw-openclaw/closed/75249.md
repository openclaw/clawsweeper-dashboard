## What This Changes

The PR changes the plugin `registerHook` wrapper to pass the original event context with temporary `pluginConfig` injection, adds `agent:bootstrap` regression tests, and adds a changelog entry for the plugin hook fix.

## Best Possible Solution

Land this PR or an equivalent narrow patch after mergeability and CI are green, preserving mutable hook-context behavior while restoring/removing per-handler `pluginConfig` after each plugin hook call.

## Reproduction Assessment

Yes. Register an `agent:bootstrap` hook through `api.registerHook` that assigns `event.context.bootstrapFiles`, then call `applyBootstrapHookOverrides`; current main sends that assignment to the cloned wrapper context while core reads the original event context after dispatch.

## Solution Assessment

Yes. Passing the original context and restoring `pluginConfig` in `finally` is the narrowest compatible repair for the existing mutable hook contract; an immutable return-value API would be a larger plugin SDK change.
