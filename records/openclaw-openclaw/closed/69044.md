## Summary

Keep open. Current main still calls `entry.loadChannelPlugin()` and then reads `plugin.id` without a nullish guard. A broad catch now prevents the TypeError from escaping, but it converts the malformed undefined return into a warning/log path, so the issue’s noisy-stderr concern is not fully fixed. The linked PR #69049 and referenced commit 81af91adedb8a9b1f46604bfb34a09b4733ca1c3 contain the intended guard/test, but that patch is not present on current main.

## Best Possible Solution

Keep #69044 open and land a narrow current-main fix in `src/channels/plugins/bundled.ts`: after `entry.loadChannelPlugin()`, return `undefined` for a nullish result before touching `plugin.id`, without warning or populating the negative cache for that legacy malformed case. Add colocated regression coverage that asserts both the returned value and the absence of noisy warning/config-read stderr for the undefined-return path.
