## Summary

Keep open. Current main still lacks a configurable `openclaw mcp serve` operator-scope selector, and https://github.com/openclaw/openclaw/pull/74428 is the active implementation candidate for this requested least-privilege MCP bridge behavior.

## What This Changes

The issue requests a supported way for `openclaw mcp serve` to request narrower operator scopes for least-privilege MCP bridge identities.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/74428 or an equivalent narrow implementation that validates MCP bridge scopes, preserves current defaults when omitted, documents the contract, and adds focused CLI/server coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows no `--scope` or mode option, no scope field in `OpenClawMcpServeOptions`, and a fixed MCP bridge scope list passed to `GatewayClient`.

## Solution Assessment

Yes, with maintainer review. A repeatable, allowlisted `--scope` flag threaded through MCP serve options to `GatewayClient` is the narrow maintainable direction; the broader operator-scope-to-agent-tool-policy work remains separate in https://github.com/openclaw/openclaw/issues/49183.
