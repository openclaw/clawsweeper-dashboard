## Summary

Current main routes exec/tool subprocesses through a Windows-aware process supervisor that now settles waits from child exit state after stdout/stderr drain even when `close` never arrives, with explicit regression coverage for that missing-`close` case. That matches the hang/forced-`SIGKILL` behavior described in #66481, so this is implemented on main.

## Evidence

- **Exec tool still uses supervisor path:** Non-sandbox exec builds a child-mode spawn spec for non-PTY runs, then calls `supervisor.spawn(...)`; the supervisor dispatches child-mode runs to `createChildAdapter(...)`.
