## Summary

This is better scoped as a plugin/ClawHub policy than an OpenClaw core issue. Current main already exposes prompt-time model override hooks and ACP spawn model overrides, so task-tier routing can live in an external hook plugin without adding core ACP-specific routing behavior.

## Evidence

## Risks / Open Questions

- No ClawHub plugin was linked in the issue, so this closure depends on maintainers agreeing that the existing hook/plugin seam is sufficient and that a bundled default policy is out of scope.
- If maintainers later want an official first-party default routing heuristic for UX reasons, this item would need to be reconsidered as a product decision rather than an extensibility gap.
