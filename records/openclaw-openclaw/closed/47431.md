## Summary

Current `main` already fixes the reported Control UI context display bug by storing context usage from the latest call snapshot instead of accumulated lifetime input totals, and by suppressing context-utilization UI when the snapshot is stale. The same implementation is present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).

## Evidence

- **Session accounting uses latest-call context, not accumulated lifetime usage:** `persistSessionUsageUpdate` documents and implements `lastCallUsage` for `totalTokens`, explicitly avoiding accumulated per-run `usage.input` because it overstates actual context; it also marks missing fresh snapshots as stale via `totalTokensFresh`.

  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Control UI context notice only uses fresh context totals:** The Control UI notice returns `null` when `totalTokensFresh === false`, and otherwise compares `session.totalTokens` against `contextTokens` for the displayed ratio.
  - file: [ui/src/ui/chat/context-notice.ts:53](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/ui/src/ui/chat/context-notice.ts#L53)
