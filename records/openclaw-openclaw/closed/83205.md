## Summary

Keep this PR open: it targets a real schema/validator mismatch, but the patch currently relaxes provider validation for every provider id instead of preserving full validation for custom providers, and the supplied proof exercises a mirrored zod snippet rather than OpenClaw's patched validator.

## What This Changes

The PR makes `baseUrl` and `models` optional in `ModelProviderSchema` and adds schema regression tests for timeout-only provider entries.

## Best Possible Solution

Permit timeout-only overlays for recognized built-in providers while keeping `baseUrl` and `models` required for brand-new custom provider ids, then prove the actual OpenClaw validator or gateway accepts the documented config.

## Reproduction Assessment

Yes by source inspection: current main's provider schema requires `baseUrl` and `models`, while the resolver reads `providerConfig.timeoutSeconds` once validation succeeds. I did not run a gateway restart because this review is read-only.

## Solution Assessment

No. The bug is real, but making `baseUrl` and `models` optional for every provider id is broader than the accepted fix shape; keep custom-provider requirements and allow known built-in overlays.
