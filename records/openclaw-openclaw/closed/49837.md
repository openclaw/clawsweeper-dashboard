## Summary

Close #49837 as superseded by the later Feishu native-post-element work, especially #49850. Current main does not implement the requested blue-mention rendering fix, but this PR's lark_md/card-formatter approach has unresolved correctness concerns and is no longer the best canonical path for #49833.

## Best Possible Solution

Close #49837 as superseded and use #49850 as the canonical remaining implementation thread. The eventual fix should build native Feishu post `at` elements from structured mention targets, avoid changing the shared card lark_md formatter, avoid parsing arbitrary model body text into live mentions, and keep focused send-payload regression tests.
