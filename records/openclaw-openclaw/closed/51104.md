## Summary

Keep open. Current main still lacks Mattermost WebSocket protocol ping/pong keepalive; the Mattermost connector handles open/message/close/error plus the bot update_at REST poll, but not transport-level liveness. The secondary stale-event threshold is configurable now, and the KeepMyClaw comment is useful recovery context, but neither replaces the requested Mattermost keepalive. Open PR #57621 remains the relevant implementation candidate.

## Best Possible Solution

Keep this issue open and review or land PR #57621, or an equivalent Mattermost-owned patch in extensions/mattermost/src/mattermost/monitor-websocket.ts. The fix should add ws ping(), listen for pong, terminate after missed pong cycles so runWithReconnect reconnects, clean up timers on close/error/abort, preserve the existing update_at poll, add focused monitor-websocket tests, and include a user-facing changelog entry if the PR lands. After merge, close this issue and related Mattermost half-dead WebSocket reports such as #41837 and #44160 as implemented or duplicates of the landed fix.
