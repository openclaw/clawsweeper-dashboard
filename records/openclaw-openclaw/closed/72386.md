## Summary

Current main still matches the reported regression. The runtime-context helper emits the exact human-readable preface, queues it as a hidden `openclaw.runtime-context` `nextTurn` custom message, and the pinned Pi dependency converts custom messages into user-role LLM messages. Telegram-style channel envelopes still create the effective-prompt/transcript-prompt delta that triggers this path, while current sanitizers only cover delimited and legacy internal runtime-context headers, not the new header reported here.

## Best Possible Solution

Keep this issue open for a core embedded-runtime fix. Stop placing the human-readable runtime-context preface into ordinary custom-message content, or teach the `openclaw.runtime-context` conversion path to provide structured non-user model context. Add regression coverage for runtime-context prompt assembly, Telegram/channel envelope prompt deltas, Pi custom-message conversion, and user-facing outbound sanitization of this exact header.
