## Summary

Current main and the 2026.5.4 changelog already cover the reported 2026.5.3 Telegram forum-topic message-tool failure: numeric Telegram topic targets are accepted by the agent message tool, explicit `threadId` is carried through outbound resolution, and Telegram sends receive `messageThreadId`.

## What This Changes

The issue reports that the Telegram `message` tool fails when sending to forum topic thread IDs while cron announce fallback delivery to the same topics succeeds.

## Best Possible Solution

Keep the shipped message-tool routing and direct users still on 2026.5.3 to upgrade to 2026.5.4 or newer rather than adding another core path.

## Reproduction Assessment

Yes for the reported version: the issue includes repeated 2026.5.3 cron runs where `message(action=send)` failed for Telegram topic IDs while announce fallback delivered. I did not reproduce a failing current-main path because the current source and changelog show the topic send route has already been fixed.

## Solution Assessment

Yes. The maintainable fix is the current generic message-action path: normalize the Telegram target, preserve explicit `threadId`, and let the Telegram plugin convert it to `messageThreadId` for Bot API sends.
