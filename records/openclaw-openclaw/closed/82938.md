## Summary

Current main and the shipped v2026.5.12 tree already suppress the reported `codex-app-server` marker while preserving `PLAINTEXT_FOUND` for real plaintext `models.json` provider keys.

## What This Changes

The issue asks `openclaw secrets audit` to stop warning on OC-generated non-secret `codex-app-server` markers in generated agent `models.json` files.

## Best Possible Solution

Keep the shipped marker-aware audit behavior and have affected users upgrade to v2026.5.12 or newer before filing a fresh report with `openclaw secrets audit --json` output.

## Reproduction Assessment

No current-main failure path was found from source inspection: the scanner visits all agent `models.json` files but suppresses `PLAINTEXT_FOUND` for declared non-secret markers, including Codex's `codex-app-server`. I did not run the CLI because this review was read-only, but the code and regression tests cover the reported path.

## Solution Assessment

Yes. The implemented solution keeps the marker owned by the Codex plugin manifest, lets the generic audit scanner consume that contract, and still warns on arbitrary plaintext provider keys.
