## Summary

Current main implements the OpenClaw core/bundled Feishu fix: zero-final dispatches now carry no-visible-reply eligibility, Feishu tracks visible delivery separately from final-count accounting, and fallback is suppressed for message-tool-owned delivery. This is main-only and not in the latest release yet; the companion external Lark plugin rollout remains in its own repository.

## What This Changes

The issue requests distinct Feishu/Lark accounting for true no-reply turns versus turns where a same-turn side-path/message-tool send already provided visible output.

## Best Possible Solution

Keep the current main OpenClaw Feishu fallback/disposition tracking, and let plugin-specific Lark rollout continue in larksuite/openclaw-lark.

## Reproduction Assessment

Yes. The reporter supplied real Feishu logs from v2026.4.29, and the old source path is clear from the zero-final dispatch accounting that current main now separates through fallback eligibility and source delivery mode.

## Solution Assessment

Yes. The merged fix is the maintainable core path because it keeps message-tool-owned delivery distinct, adds a Feishu fallback only for no-visible completions, and covers direct, streaming/card, media, silent, and broadcast cases in focused tests.
