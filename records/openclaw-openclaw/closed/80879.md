## Summary

Keep open: current main still has source-visible paths where blocked subagent terminal metadata can be carried through as an ok completion, and the issue is already paired with the open closing PR https://github.com/openclaw/openclaw/pull/80886. The later swallowed-error cleanup from https://github.com/openclaw/openclaw/pull/75462 helps an adjacent failure mode, but it does not replace the blocked-liveness normalization needed here.

## What This Changes

The issue asks OpenClaw to report subagent `livenessState: "blocked"` or `context_overflow` terminal states as errors instead of “completed successfully” with no usable output.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/80886 or an equivalent shared blocked-liveness normalization across lifecycle snapshots, gateway dedupe, agent.wait callers, registry completion, and announcement rendering, then close this issue after merge.
