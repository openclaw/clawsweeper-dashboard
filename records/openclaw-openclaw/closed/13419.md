## Risks / Open Questions

- The shipped option is broader than the issue's proposed key name: `channels.slack.thread.requireExplicitMention` disables implicit thread mentions for bot-participated threads generally, which includes the bot-parent case reported here.

## Close Comment

Closing as implemented on `main` after a Codex review. OpenClaw already ships `channels.slack.thread.requireExplicitMention`, documented in `src/config/types.slack.ts:100`, wired in `extensions/slack/src/monitor/provider.ts:177`, and enforced in `extensions/slack/src/monitor/message-handler/prepare.ts:444` by suppressing implicit thread mention kinds when the option is enabled. The exact bot-parent case from this issue is covered by `extensions/slack/src/monitor/message-handler/prepare.test.ts:909`, which asserts that a reply with `parent_user_id` matching the bot is skipped unless there is an explicit `@mention`. This behavior is also present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
