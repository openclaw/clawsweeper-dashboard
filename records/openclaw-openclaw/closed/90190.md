## Summary

Close as implemented: current main and the latest release suppress default text-only Codex tool-progress payloads in the shared reply dispatch path before they can be delivered as standalone WhatsApp messages, while preserving media and exec-approval payloads and leaving verbose progress as the explicit opt-in path.

## What This Changes

The issue requests that Codex native tool-progress status lines stop appearing as standalone WhatsApp messages, either by default suppression on non-draft channels or by adding a supported opt-out.

## Best Possible Solution

Keep the shared dispatch-layer suppression shipped in v2026.6.1 and use verbose progress as the opt-in path instead of adding a WhatsApp-specific config knob.

## Reproduction Assessment

No high-confidence current-main live reproduction was established. Source inspection shows the old failure path is now blocked by default because text-only tool-progress payloads return before delivery unless verbose progress is enabled.

## Solution Assessment

Yes. The current shared dispatch-layer fix is the narrowest maintainable solution because it covers WhatsApp and other non-draft channels without adding channel-specific config, while preserving media, approvals, and explicit verbose progress.
