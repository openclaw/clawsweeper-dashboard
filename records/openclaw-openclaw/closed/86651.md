## What This Changes

The issue reports that memory-core Dreaming stopped adding new session transcripts to `seenMessages` and `memory/.dreams/session-corpus` after May 15, 2026.

## Best Possible Solution

Keep the shipped SQLite-backed Dreaming state path and legacy JSON doctor migration; any remaining WSL2-specific evidence should be filed against the current release with fresh logs.

## Reproduction Assessment

No, not as a current-main failure. The old WSL2/private-file-store failure is documented in the related thread, and current source plus release evidence show that memory-core Dreaming ingestion state no longer uses that workspace private-file path.

## Solution Assessment

Yes. Moving memory-core Dreaming ingestion state to SQLite plugin state with a doctor migration is a better fix than weakening private directory permission enforcement for all private file stores.
