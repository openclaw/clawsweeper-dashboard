## Summary

The report is valid on current main. The migrate preview header now counts conflicts only from non-hidden items, while the apply guard still blocks on the authoritative plan summary, so a hidden config conflict can be previewed as zero conflicts and then fail apply.

## What This Changes

The issue asks for migrate preview aggregate counts to stay accurate when config item details are hidden.

## Best Possible Solution

Keep config item rows hidden, but make the preview header report authoritative conflict and sensitive counts from the plan summary or an equivalent hidden-conflict aggregate.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a plan whose only conflict is kind=config is filtered out of visibleConflicts, while the same plan still fails assertConflictFreePlan through summary.conflicts.

## Solution Assessment

Yes. The narrow maintainable fix is to keep HIDDEN_KINDS for item rendering while using plan.summary.conflicts and plan.summary.sensitive, or explicit hidden aggregates, for the header.
