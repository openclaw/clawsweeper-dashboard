## Summary

Current main already counts cached prompt tokens in the persisted session prompt snapshot and uses that snapshot/projected totals for preflight compaction, matching the reported Anthropic 100% cache-hit failure mode. The fix is also documented as shipped in v2026.4.15.

## Evidence

- **Cached prompt tokens are part of the session prompt snapshot:** `deriveSessionTotalTokens()` builds the persisted prompt/context snapshot from `input + cacheRead + cacheWrite` via `derivePromptTokens(...)`, and returns it unclamped. That means Anthropic cache-hit tokens still contribute to `SessionEntry.totalTokens` instead of disappearing into near-zero fresh input.
