## Summary

Current main does not match the reported Control UI reset/patch path. The UI model-change path sends only `sessions.patch`; the gateway patch handler only updates session store metadata; `sessions.reset` creates a header-only transcript; blank `chat.send` requests are rejected; and persisted blank user turns are repaired before replay.

## Best Possible Solution

Close as not reproducible on current main. Keep the existing source behavior: `sessions.patch` remains metadata-only, reset stays header-only, blank `chat.send` is rejected, and transcript hygiene repairs any legacy blank user turns before provider replay.
