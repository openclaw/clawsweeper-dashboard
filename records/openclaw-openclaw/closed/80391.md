## Summary

Keep this PR open for maintainer review: the hang fix is real and the contributor supplied useful terminal proof, but the 5-second timeout is applied through a shared helper that also affects broker-config inspection, so maintainers should accept or tune that automation tradeoff before merge.

## What This Changes

The PR adds a 5-second `SIGKILL` timeout to Crabbox wrapper subprocess probes and adds a regression test for hung sanity checks.

## Best Possible Solution

Land this after maintainers explicitly accept the shared 5-second metadata-probe timeout, or scope/tune the timeout so broker-config inspection has the intended failure budget.

## Reproduction Assessment

Yes for the core hang shape: current main calls `spawnSync` without a timeout for Crabbox metadata probes, so a hung selected binary can block the wrapper. I did not run the full wrapper test suite in this read-only cleanup review.

## Solution Assessment

Mostly yes: adding a bounded timeout is the right repair for the hang path. The open question is whether the hard 5-second budget should apply through the shared helper or only to the named sanity probes.
