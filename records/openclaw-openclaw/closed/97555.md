## Summary

The issue reports cross-session corruption of the Skill Workshop `proposals.json` index during lifecycle actions because of stale reads, blind writes, missing locking, and missing proposal name fields.

## Best Possible Solution

Keep the shipped lock-backed Skill Workshop store behavior; treat any future corrupted local state report as a concrete data-repair issue with attached proposal records and manifest contents.
