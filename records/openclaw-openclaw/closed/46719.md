## Summary

Current `main` already defers upstream announce while descendants are still pending, records a wake-on-descendant-settle state, and wakes the orchestrator with child findings instead of surfacing a premature completion. The same code and regression tests are present in release tag `v2026.4.22`, so this is already implemented/shipped.

## Evidence

- **Announce path bails out while descendants are pending:** `runSubagentAnnounceFlow` counts pending descendant runs and returns without announcing when any remain; when `wakeOnDescendantSettle` is set and child findings exist, it wakes the child session instead of delivering completion upstream.
