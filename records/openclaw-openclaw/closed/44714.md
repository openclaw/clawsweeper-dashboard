## Summary

Current `main` already contains the handshake-timeout fix this issue describes: the gateway pre-auth WebSocket timeout is back to 10 seconds, the timeout is still enforced in the WS connection path, and the changelog records this exact regression fix as shipped in `v2026.3.22` via PR `#49262`.

## Evidence

- **Current timeout default on main:** `DEFAULT_PREAUTH_HANDSHAKE_TIMEOUT_MS` is `10_000`, not `3_000`.
  - file: [src/gateway/handshake-timeouts.ts:1](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/gateway/handshake-timeouts.ts#L1)
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **WS path uses shared pre-auth timeout:** The WebSocket handshake timer for unauthenticated clients reads `getPreauthHandshakeTimeoutMsFromEnv()` before closing on timeout, so the restored 10s budget is what `openclaw logs --follow` now gets.
  - file: [src/gateway/server/ws-connection.ts:366](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/gateway/server/ws-connection.ts#L366)
