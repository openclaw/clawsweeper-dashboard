Full review comments:

- **[P1] Read plugin installs through the SQLite index API:** `src/infra/profile-portability.ts:794-796`
  - body: On current main, `resolveInstalledPluginIndexStorePath()` returns the shared SQLite DB path. This export path reads that path as UTF-8 JSON and sends it to `sanitizePluginInstallRecordsPayload`, so profile export will fail or lose plugin records for existing installs; use the installed-plugin-index record reader API and sanitize that object instead.
  - confidence: 0.9
- **[P2] Merge plugin records instead of skipping the SQLite DB:** `src/infra/profile-portability.ts:1187-1190`
  - body: Because `targetPath` is now `state/openclaw.sqlite`, most target homes will already have it, so import skips the entire `plugin-installs` asset and never adds missing profile records. Read the current install records and refresh a merged set of missing portable records instead of treating DB existence as a per-record conflict.
  - confidence: 0.86
- **[P3] Wire the profile page into CLI docs navigation:** `docs/cli/profile.md:1-6`
  - body: This adds a new public CLI reference page, but `docs/docs.json` and the CLI index are unchanged, so the page is not surfaced with the other CLI commands in Mintlify navigation. Add it alongside backup/migrate in the CLI docs navigation and command index.
