## Summary

Keep this PR open. Current main still has the Telegram reply streaming regression targeted by the PR: non-off `replyToMode` populates the native-quote candidate map for a plain inbound message, then treats any non-empty map as a native quote and disables answer draft streaming. The related #73505 bug is still open, and this PR remains the focused implementation candidate with no security-sensitive drift in the provided patch.

## Best Possible Solution

Keep this PR open as the focused implementation candidate for #73505. Maintainers should review the narrowed `hasNativeQuoteReply` guard, run the targeted Telegram dispatcher/delivery tests plus changed-lane validation, and land it if those checks pass.
