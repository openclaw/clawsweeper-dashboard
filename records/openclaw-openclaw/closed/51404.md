## Summary

Current main and the v2026.4.26 tag now separate selected/default model state from active runtime model state during reset/startup and status reporting. The startup prompt has both `model=` and `default_model=` available, `/new`/`/reset` no longer carry stale auto-created runtime pins as the selected default, and status/model displays distinguish selected/current from active runtime when they differ.

## Best Possible Solution

Close as implemented and keep the shipped behavior: runtime model, selected/default model, and fallback state remain distinct in reset prompts and status surfaces, with v2026.4.26 as the shipped proof.
