## Summary

Current main fixes the reported empty Channels table symptom for configured Telegram accounts without making `openclaw status` eagerly load the full plugin registry. `status` still has `loadPlugins: "never"`, but its table now uses read-only channel metadata with setup-runtime fallback, so configured bundled channels such as Telegram are surfaced even when the active runtime registry is not preloaded. The latest shipped tag `v2026.4.26` still had the old `includeSetupRuntimeFallback: false` status-table path, so this is implemented on current main but not provably shipped in a release yet.

## Best Possible Solution

Keep the current main implementation: `openclaw status` should stay on the fast read-only path and use setup/manifest channel metadata for configured channels instead of loading the full runtime plugin registry. Ship the current status-table setup fallback in the next release; no core pre-action full-registry load is needed for this issue.
