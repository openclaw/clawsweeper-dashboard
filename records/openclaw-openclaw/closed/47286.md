## Summary

Current main already suppresses stale intermediate mutating-tool failures from the final user-visible outcome. The runtime clears a stored tool failure when the same mutating action later succeeds, and the final warning bubble is generated only from any remaining unresolved `lastToolError` at turn end.

## Evidence

- **Retry success clears prior mutating failure:** `handleToolExecutionEnd` records a tool error, but if a later success matches the same mutating action fingerprint it clears `lastToolError` instead of carrying the failure forward.
