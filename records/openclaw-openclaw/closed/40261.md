## Summary

Current `main` already handles `sessionTarget: "main"` cron `systemEvent` jobs by enqueueing the event onto `job.sessionKey` and waking heartbeat with `heartbeat.target = "last"`, and regression coverage explicitly locks that behavior for both `wakeMode: "now"` and `wakeMode: "next-heartbeat"`.

## Evidence

- **Main-session cron path uses originating session key:** `executeMainSessionCronJob` enqueues the cron `systemEvent` to `targetMainSessionKey = job.sessionKey`, not a generic main-session fallback.
  - file: [src/cron/service/timer.ts:1205](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/src/cron/service/timer.ts#L1205)
  - command: `nl -ba src/cron/service/timer.ts | sed -n '1194,1268p'`
