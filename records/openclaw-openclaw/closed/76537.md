## What This Changes

The PR changes subagent steering parsing so a bare `/steer <message>` auto-selects the sole active subagent, with regression tests and a changelog entry.

## Best Possible Solution

Keep #76934's current-session `/steer <message>` implementation and leave child-run steering explicit through `/subagents steer <id|#> <message>`.

## Reproduction Assessment

No against current main. Current main has a dedicated `/steer` handler that queues steering for the active current session before the subagent handler runs, so the old ID-required top-level path no longer reproduces from source inspection.

## Solution Assessment

No for this branch. The merged #76934 solution is the narrower maintainable path because it aligns runtime routing, native/text command metadata, docs, changelog, and tests around current-session steering while preserving explicit child-run targeting.
