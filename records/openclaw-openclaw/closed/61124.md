## Summary

Current `main` already handles late Discord `reconnect-exhausted` errors during teardown/dispose, which matches the crash path described in #61124. The fix is present on `main` in commit `78a431ed224b6f5b9d58672346551a5b411ad200`, but it does not appear to be part of the latest tagged release `v2026.4.23`.

## Evidence

- **Disconnect now keeps lifecycle/error handling attached until after gateway teardown:** `waitForDiscordGatewayStop()` calls `gateway.disconnect()` before detaching lifecycle listeners, with an inline comment explaining that late `error` events during disconnect must still be handled so they do not become uncaught.
