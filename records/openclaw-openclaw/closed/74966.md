## Summary

Keep open for maintainer review. Current main still conditionally omits the `dependencies` field when the bundled runtime-deps install manifest is created with an empty spec list, and this PR is a narrow, plausible compatibility fix with no blocking review findings. The linked crash report in #74949 is still open, so the right path is to validate and merge this PR or an equivalent fix rather than close either item early.

## What This Changes

The PR makes bundled runtime-deps install manifests always emit a `dependencies` object and adds an empty-spec regression test.

## Best Possible Solution

Merge this narrow manifest-helper change after maintainer validation, and keep #74949 open until this PR or an equivalent fix lands. The targeted unit test is enough for the empty-manifest contract; a packaged Gateway startup or `openclaw plugins deps --repair` smoke on Node v25/npm v11 would give stronger end-to-end proof for the reported crash loop.

## Reproduction Assessment

Unclear for the full crash loop because this review did not run a packaged Gateway under Node v25/npm v11. Yes for the manifest-shape edge case: current main statically omits `dependencies` when `ensureNpmInstallExecutionManifest` receives an empty spec list.

## Solution Assessment

Yes. Always emitting `dependencies: {}` is the smallest maintainable change for the generated manifest contract and leaves sorted non-empty dependency output, package-manager selection, install flags, and plugin policy unchanged.
