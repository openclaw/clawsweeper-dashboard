## Summary

Keep this PR open. Current main still builds the security-audit known node-command set from platform/default allowlists only, so valid dangerous commands in `gateway.nodes.denyCommands` can still be reported as unknown. The PR is a narrow audit-classification fix and does not appear to introduce security or supply-chain risk, but it should land with a focused regression test.

## Best Possible Solution

Keep this PR open and land or recreate the small `listKnownNodeCommands()` change on current main, adding a regression case in `src/security/audit-node-command-findings.test.ts` for dangerous-but-valid deny-only commands while preserving warnings for typos and pattern-like deny entries.
