Full review comments:

- **[P1] Preserve 256-color light-background detection:** `src/tui/theme/detect.ts:29-31`
  - body: Current main handles 256-color COLORFGBG values such as `0;231` and `0;255` by computing luminance, but this detector only returns `light` for `7` or `15`. Existing users on 256-color light terminals would silently fall back to the dark palette after upgrade.
  - confidence: 0.9
- **[P2] Keep mounted theme objects live after switching:** `src/tui/theme/theme.ts:223-235`
  - body: `setTheme()` rebuilds and reassigns `markdownTheme`, select-list themes, and `searchableSelectListTheme`, but existing Markdown, SelectList, and local selector components store the object they were constructed with. Runtime `/theme` therefore leaves already-mounted messages/selectors in stale colors, so the UI can become partially themed.
  - confidence: 0.88
- **[P1] Remove the root package override from the theme PR:** `package.json:447`
  - body: Adding a root `openclaw` override changes workspace package resolution and the lockfile from local links to a published package for internal importers. That supply-chain and local-development behavior is unrelated to TUI theming and should not ship in this feature branch.
  - confidence: 0.91
