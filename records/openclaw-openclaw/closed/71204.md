## Summary

Keep open. The protected maintainer label blocks automated cleanup, and although current main already has baseline NVIDIA provider support, the branch still proposes unmerged provider-prefix display behavior with a concrete plugin-runtime loading problem in the model picker.

## What This Changes

The PR updates NVIDIA provider onboarding, docs, catalog tests, and API exports, and adds a `preserveLiteralProviderPrefix` provider flag used by model-picker and auth-choice display paths.

## Best Possible Solution

Keep this in maintainer review and make the desired end state a narrow NVIDIA/model-picker change that keeps canonical config storage while sourcing literal-prefix display metadata from a manifest or other lightweight control-plane surface, with focused coverage for NVIDIA onboarding, default-model labels, and allowlist picker labels.
