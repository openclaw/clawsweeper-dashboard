## Summary

Close as duplicate. Current main still does not add context-window usage to the per-reply `/usage` footer, but open issue #40215 already tracks the same remaining footer work in a broader form: cumulative context usage plus model/context tokens/window.

## Best Possible Solution

Keep #40215 as the canonical issue for this work. The implementation should add context used/limit/percent to the existing per-reply usage footer using fresh session `totalTokens`/`contextTokens` data, update footer tests, and align the public token-use docs after behavior ships.

## Evidence
