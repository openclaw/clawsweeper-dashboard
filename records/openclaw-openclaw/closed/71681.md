## Summary

Keep PR #71681 open. Current main still has the unconditional Unix process-group kill path, and the child supervisor still calls killProcessTree(pid) without passing detachment state. The PR addresses a real core process-management bug, but the provided review discussion also identifies an unresolved fallback-spawn gap, so this is not a conservative cleanup close.
