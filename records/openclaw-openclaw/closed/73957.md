## Summary

Close as ClawHub/plugin scope. Current main still does not expose MiniMax speech-to-text, but the existing plugin SDK and media-understanding audio contract are sufficient for a MiniMax ASR provider plugin once the vendor endpoint/model contract is confirmed, so this does not need to remain an OpenClaw core issue.

## What This Changes

Request to expose MiniMax ASR speech-to-text through the MiniMax plugin for audio transcription use cases such as voice messages and audio analysis.

## Best Possible Solution

Publish a native MiniMax ASR media-understanding plugin through ClawHub after confirming the MiniMax ASR contract, using the existing audio transcription provider seam; only reopen core work if that implementation exposes a missing generic plugin API.

## Reproduction Assessment

Not applicable as a feature request. Current-main verification is high-confidence: source, manifest metadata, and docs show MiniMax media understanding is image-only while the generic audio transcription path exists for other providers.

## Solution Assessment

Yes. The narrowest maintainable path is outside core as a MiniMax-specific media-understanding provider plugin, because the SDK already exposes registration, metadata, auth/config, and audio transcription hooks needed for this capability.
