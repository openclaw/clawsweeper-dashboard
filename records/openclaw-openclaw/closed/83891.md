## Summary

Keep this open: the fallthrough still exists on current main and latest release, and the issue is already paired with the open fixing PR at https://github.com/openclaw/openclaw/pull/84284.

## What This Changes

The issue asks for the plugin payload smoke check to emit only one failure when a malformed or empty extension entry is accompanied by a missing declared main file.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/84284; the desired end state is a `continue` after terminal empty/invalid extension-entry failures plus regression coverage for the combined malformed-plugin case.

## Reproduction Assessment

Yes, from source inspection: `openclaw.extensions` that resolves to `empty` or `invalid` plus a declared missing `main` takes both failure paths. A completely absent extension field is not the repro because it resolves to `missing`.

## Solution Assessment

Yes: the linked PR’s shape, adding `continue` after the terminal extension-entry failure and covering that combined case, is the narrow maintainable fix while leaving valid-extension main checks intact.
