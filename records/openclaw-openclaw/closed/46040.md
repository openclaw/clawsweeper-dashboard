## Summary

Close as `cannot_reproduce`: the reporter’s own follow-up attributes the crash to plugin-side `embedding.ts` parsing, and current `main` shows the `before_prompt_build` hook path passing `messages` explicitly, tolerating `void` returns, and covering empty-message cases in tests without any core-side `reduce` path.

## Evidence

- **Reporter follow-up says root cause was plugin code, not OpenClaw:** The issue comment states the `reduce` crash came from the plugin’s `embedding.ts` handling of nested llama.cpp embeddings, and that after fixing the plugin plus upgrading OpenClaw, `before_prompt_build` worked normally.
- **Hook contract requires `messages` and allows empty/void results:** `before_prompt_build` receives `{ prompt, messages }` and its handler return type is `PluginHookBeforePromptBuildResult | void`, so a plugin that returns nothing is part of the supported contract.
  - file: [src/plugins/hook-before-agent-start.types.ts:21](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/plugins/hook-before-agent-start.types.ts#L21)
  - command: `nl -ba src/plugins/hook-before-agent-start.types.ts | sed -n '1,220p'`
  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)
- **Current hook runner has no `reduce`-style aggregation here:** `mergeBeforePromptBuild` combines fields with `concatOptionalTextSegments`, and `runModifyingHook` ignores `undefined`/`null` handler results before merging, so the current core path is defensive against empty hook returns.
  - file: [src/plugins/hooks.ts:231](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/plugins/hooks.ts#L231)
  - command: `nl -ba src/plugins/hooks.ts | sed -n '220,420p'`
  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)
- **Agent runtime passes `messages` directly into the hook and catches hook errors:** The embedded runner calls `runBeforePromptBuild({ prompt, messages }, ctx)` and wraps failures with a warning path rather than assuming a non-empty aggregate structure.
  - file: [src/agents/pi-embedded-runner/run/attempt.prompt-helpers.ts:43](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/agents/pi-embedded-runner/run/attempt.prompt-helpers.ts#L43)
  - command: `nl -ba src/agents/pi-embedded-runner/run/attempt.prompt-helpers.ts | sed -n '1,180p'`
  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)
- **Regression tests cover empty-message invocation without internal crash:** `extensions/memory-lancedb/index.test.ts` calls `before_prompt_build` with `messages: []` and expects it to resolve `undefined`; `src/plugins/hooks.model-override-wiring.test.ts` also verifies the hook receives the message array and prepends context normally.
  - file: [extensions/memory-lancedb/index.test.ts:240](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/extensions/memory-lancedb/index.test.ts#L240)
