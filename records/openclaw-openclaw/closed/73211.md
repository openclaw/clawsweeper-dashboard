## Summary

Close as implemented. Current main and v2026.5.12 accept the canonical Mattermost `streaming` config and gate draft-preview behavior on `streaming.mode`, so the reported hardcoded preview opt-out gap is fixed; broader Mattermost block-streaming and agent-to-agent routing concerns remain in a separate open issue.

## What This Changes

The issue requests a configurable way to opt out of Mattermost draft-preview streaming instead of hardcoding the preview behavior.

## Best Possible Solution

Keep the shipped canonical `channels.mattermost.streaming` implementation and route unrelated block-streaming or agent-to-agent routing behavior to the broader Mattermost follow-up.

## Reproduction Assessment

No current-main reproduction remains. With `channels.mattermost.streaming: "off"`, the schema accepts the field, account resolution returns `off`, and the monitor uses a disabled draft stream instead of creating or updating a preview post.

## Solution Assessment

Yes. Using the existing shared `streaming.mode` contract is the narrow maintainable fix; adding a Mattermost-only `draftPreview` boolean would duplicate the public config model.
