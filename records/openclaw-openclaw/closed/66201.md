## Summary

Current `main` does not reproduce the reported destructive "Reset Agent" behavior. The disconnected Control UI renders a reconnect/login form, not a reset action, and the only comparable chat control is a `New session` button. On the backend, `sessions.reset` rotates the session entry/file and archives the prior transcript; it does not wipe agent workspace files, memory, or configuration.
