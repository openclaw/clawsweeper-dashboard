## Summary

Keep open. Current main still selects `heartbeat.model` for the heartbeat run and then persists that runtime model/context back into the shared session store, and open PRs #75557 and #75567 already reference this issue with closing syntax.

## What This Changes

Ensure `agents.defaults.heartbeat.model` acts as a single-turn heartbeat override and does not persist into later user turns.

## Best Possible Solution

Land #75567 or an equivalent narrow core fix that preserves the pre-heartbeat shared-session runtime model and context metadata after heartbeat turns, with regression coverage for session persistence and `session_status`.

## Reproduction Assessment

Yes. Configure a shared-session heartbeat with `heartbeat.model` different from the primary model, let the heartbeat run, then inspect `session_status`; source inspection shows the heartbeat runtime model is written back to the shared session entry.

## Solution Assessment

Yes directionally. Preserving or restoring runtime model/context for heartbeat turns is narrower than changing model selection semantics or requiring all users to switch to isolated heartbeat sessions.
