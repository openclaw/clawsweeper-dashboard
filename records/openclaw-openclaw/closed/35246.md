## Summary

#35246 requests a useful bulk session model-switching workflow, but it is optional operator automation rather than missing OpenClaw core behavior. Current main does not implement `openclaw sessions switch-model`; the needed extension points already exist for a ClawHub skill/plugin: plugins can register CLI commands, can use the public gateway client, `sessions.list` exposes session rows, and `sessions.patch` already validates and persists per-session model changes.
