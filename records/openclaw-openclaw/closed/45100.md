## Summary

Current `main` now contains explicit `/v1/responses` lifecycle cleanup and bounded in-memory session tracking that directly address the gradual stall/timeout pattern described in #45100.

## Evidence

- **Bounded response-session cache:** `/v1/responses` now stores `previous_response_id` continuity in a TTL- and size-bounded map (`30m`, `500` entries), preventing unbounded accumulation across repeated requests.
