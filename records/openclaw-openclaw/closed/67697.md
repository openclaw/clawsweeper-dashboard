## Summary

Current `main` already keeps cron-run and heartbeat noise out of memory-core dreaming ingestion through session-store classification plus transcript sanitization, which supersedes this PR's trigger-persistence approach for the reported behavior.

## Evidence

- **Cron-run transcripts are classified from the session store:** `loadSessionTranscriptClassificationForSessionsDir()` builds `cronRunTranscriptPaths` from session keys that match isolated cron runs, so ingestion can identify those transcripts without persisting a new `SessionEntry.trigger` field.
  - file: [src/memory-host-sdk/host/session-files.ts:168](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/src/memory-host-sdk/host/session-files.ts#L168)
  - command: `nl -ba src/memory-host-sdk/host/session-files.ts | sed -n '140,240p'`
  - sha: [fa976e5b93b3](https://github.com/openclaw/openclaw/commit/fa976e5b93b3819f9e137fe08635764edee38290)
- **Dreaming ingestion skips classified cron-run transcripts:** `collectSessionIngestionBatches()` preclassifies each file, passes `generatedByCronRun` into `buildSessionEntry()`, and then short-circuits when a transcript was generated by dreaming or an isolated cron run.
  - file: [extensions/memory-core/src/dreaming-phases.ts:720](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/extensions/memory-core/src/dreaming-phases.ts#L720)
