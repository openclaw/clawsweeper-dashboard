## Summary

Current main implements the confirmed fix path: the security audit no longer depends on the loaded channel registry for `plugins.allow` phantom checks, and config reload only restores last-known-good for invalid snapshots, not valid allowlist edits. The related discussion around #68154 was useful and matches the source-backed part that is now fixed.

## Best Possible Solution

Close this issue as implemented on main. Keep the plugin-index-backed security audit and current reload recovery behavior; if someone can still reproduce a valid `plugins.allow` edit being restored on a build after 2026.4.25, track that as a fresh gateway config-reload bug with the clobbered config and `config-audit.jsonl` evidence.
