## Summary

Current main and the v2026.5.18 release now preserve topic-qualified Telegram forum origins through inbound context, audio preflight, and transcript echo delivery, so the reported v2026.5.12 forum-topic voice-note failure is implemented.

## What This Changes

The issue reports that Telegram forum-topic voice notes arrive as raw audio instead of following the same transcript and echoTranscript path that works in direct chats.

## Best Possible Solution

Keep the shipped Telegram origin-target and preflight echo implementation, and ask for a fresh issue only if someone can reproduce the forum-topic voice-note failure on v2026.5.18 or newer.

## Reproduction Assessment

Yes, at source level: the canonical related report showed v2026.5.x flattening Telegram forum-topic OriginatingTo while retaining thread metadata, and current tests now cover a non-General forum voice message with topic-qualified OriginatingTo and MessageThreadId. I did not run a live Telegram forum-topic repro in this read-only pass.

## Solution Assessment

Yes. The shipped fix is the narrow maintainable solution: centralize Telegram inbound origin target construction, pass that prepared target into audio preflight, and let the existing preflight echo path deliver with the same context instead of adding a new workflow or config option.
