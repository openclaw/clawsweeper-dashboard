## Summary

Keep open: the PR fixes the ordinary WhatsApp direct inbound route in the right direction, but it still leaves configured WhatsApp broadcast DMs rebuilding per-agent routes from the global DM scope, so the privacy-sensitive context-sharing bug remains on that path. The protected maintainer label also means this PR should not be cleanup-closed by automation.

## What This Changes

The PR adds a route-level DM-scope override, applies account-aware WhatsApp direct session keys and isolated last-route writes, and updates tests, docs, and changelog.

## Best Possible Solution

Make every WhatsApp inbound direct path, including broadcast fan-out, resolve account-aware per-contact session keys while preserving existing group routing and generic non-WhatsApp defaults.

## Reproduction Assessment

Yes. Source inspection on current main and the PR diff shows default WhatsApp direct routes collapse through `cfg.session?.dmScope ?? "main"`, and the proposed patch leaves direct broadcast fan-out rebuilding route keys from that same global scope.

## Solution Assessment

No. The per-call route override is the right narrow mechanism for ordinary inbound DMs, but it is incomplete until direct broadcast route construction uses the same account-aware scope and has regression coverage.
