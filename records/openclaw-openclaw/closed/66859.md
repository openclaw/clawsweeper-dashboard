## Summary

Current `main` now has native restart persistence and recovery for this path: restart commands persist a durable restart sentinel with routing/session context, startup consumes that sentinel to wake and continue the prior session, interrupted main sessions are recovered from stale transcript-lock evidence, and removed session transcripts are archived as timestamped `.deleted.*` artifacts instead of being dropped. The changelog tracks the main-agent restart recovery work under the unreleased `2026.4.24` section, so this appears fixed on `main` but not in the latest tagged release yet.
