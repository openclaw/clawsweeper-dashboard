## Summary

Current `main` no longer exposes a top-level `buttons` requirement on the shared `message` tool. `action=send` uses optional `message` plus optional `presentation`, and the send validator only requires `message` when there is no media/presentation/interactive payload. The implementation proving this is on `main` but is not contained in the latest release tag `v2026.4.23`.

## Evidence

- **Shared message-tool schema has no top-level required `buttons` field:** `src/agents/tools/message-tool.ts:70` defines `buttons` only as an optional field inside `presentation.blocks[]`, and `src/agents/tools/message-tool.ts:90` builds the send schema with optional `message` plus other optional fields, without any top-level `buttons` property.
  - file: [src/agents/tools/message-tool.ts:90](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/agents/tools/message-tool.ts#L90)

  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
- **Regression test asserts `buttons` is absent from the tool schema:** `src/agents/tools/message-tool.test.ts:446` asserts `presentation` is present while `components`, `blocks`, and top-level `buttons` are `undefined`, directly covering the reported schema problem.
  - file: [src/agents/tools/message-tool.test.ts:446](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/agents/tools/message-tool.test.ts#L446)

## Risks / Open Questions

- `src/agents/system-prompt.ts` and some docs still mention legacy `buttons=[[...]]` examples, so documentation/prompt wording may lag the current schema even though the reported validation bug is fixed on `main`.
- The proving commit is not in the latest tagged release, so users on released builds may still hit the old behavior until a newer release ships.
