## Summary

Current `main` does not add `WORKING.md` to the built-in bootstrap set, but the requested behavior now fits OpenClaw's documented hook/plugin extension path rather than core. This is optional agent-memory workflow customization, and project scope says optional capability should usually ship as plugins.

## Evidence

- **Plugin-first scope:** Vision doc says core should stay lean and optional capability should usually ship as plugins; new core additions need a strong reason.

- **Bootstrap mutation is now a documented extension point:** Hooks docs explicitly document `agent:bootstrap` with mutable `context.bootstrapFiles`, plus plugin/workspace hook discovery and npm hook-pack installation.
  - file: [docs/automation/hooks.md:123](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/docs/automation/hooks.md#L123)
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)

  - file: [src/agents/bootstrap-files.ts:147](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/src/agents/bootstrap-files.ts#L147)

Adding it would expand core surface area rather than restore missing behavior.

## Risks / Open Questions

- TypeScript hook authors would still need a cast or JS handler today because `WorkspaceBootstrapFileName` excludes `WORKING.md`; that is extension ergonomics debt, but it does not make this a core-only feature.
