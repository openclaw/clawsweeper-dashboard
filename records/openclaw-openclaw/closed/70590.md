## Summary

Issue #70590 is a broad Control UI hard-coded-language request. Current main still has hardcoded English in multiple Control UI surfaces, but the same remaining work is already tracked by open PR #39692, which explicitly moves remaining Control UI hardcoded strings into translation-backed labels for agents, logs, debug, config form, instances, exec approvals, and related views. This should close as duplicate/superseded, not as implemented.

## Best Possible Solution

Close #70590 as duplicate/superseded by active PR #39692. Continue the implementation there by rebasing onto current main, moving remaining user-facing Control UI literals into the English source locale map, regenerating all non-English locale bundles through the existing Control UI i18n pipeline, and preserving the reviewed UX details. If #39692 closes unmerged, maintainers should open or retarget a narrower canonical hard-coded Control UI localization issue.
