## Summary

Current `main` already implements per-cron-job tool scoping end to end. Cron agent-turn payloads support `toolsAllow`, the CLI exposes `--tools`, isolated cron execution filters tool definitions accordingly, the gateway protocol schema validates the field, and the latest release tree also documents the feature. The only mismatch with the issue body is naming: the supported JSON field is `payload.toolsAllow`, not `payload.tools`.
