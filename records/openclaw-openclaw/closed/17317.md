## Summary

Current `main` now documents and enforces `sessions_spawn` target allowlists for agent-to-agent calls, and `agents_list` reports the allowed target ids. The reported behavior (`allowAny: false`, only `main` visible) matches the documented default of same-agent-only when no `subagents.allowAgents` wildcard or explicit target list is configured.

## Evidence

- **Allowlist is enforced for native subagent spawns:** `sessions_spawn` checks `agents.list[].subagents.allowAgents` or `agents.defaults.subagents.allowAgents`, treats `"*"` as wildcard, and rejects cross-agent targets not in the allowlist.
  - file: [src/agents/subagent-spawn.ts:642](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/src/agents/subagent-spawn.ts#L642)

## Close Comment

Closing based on a Codex review of current `main`. OpenClaw now implements and documents agent-to-agent `sessions_spawn` allowlists: configure `agents.defaults.subagents.allowAgents` or `agents.list[].subagents.allowAgents` (see `docs/gateway/configuration-reference.md:1825` and `docs/tools/subagents.md:141`). The runtime enforces that in both native and ACP spawns (`src/agents/subagent-spawn.ts:642`, `src/agents/acp-spawn.ts:749`), and `agents_list` now reports the allowed target ids plus `allowAny` (`src/agents/tools/agents-list-tool.ts:49`). In this implementation, `allowAny: false` means no `"*"` wildcard is configured, so only the requester agent and any explicitly allowlisted targets are available. This landed on `main` in `b22bf36bc4165dfe89b563960498e27e8e4e5034`. I could not confirm a tagged release containing it yet.
