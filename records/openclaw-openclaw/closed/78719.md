## Summary

Keep open. This PR targets a valid current-main gateway privilege-escalation bug, but it still needs real behavior proof and a contract/docs pass before merge.

## What This Changes

The PR removes the shared-secret `trusted-operator` default-scope shortcut and updates gateway plugin-route scope tests to expect no admin escalation while preserving trusted-proxy defaults.

## Best Possible Solution

Land a security-reviewed patch that aligns the resolver, tests, docs, changelog, and real gateway proof so shared-secret plugin routes cannot gain implicit admin scopes.

## Reproduction Assessment

Yes. Current-main source inspection shows token/password auth sets `trustDeclaredOperatorScopes: false`, and the `trusted-operator` resolver branch converts that path into default scopes containing `operator.admin`; existing tests also asserted that behavior.

## Solution Assessment

Unclear. Removing the implicit admin escalation is the right direction, but this PR should also settle the intended shared-secret fallback contract, update docs/changelog, and provide real route proof before merge.
