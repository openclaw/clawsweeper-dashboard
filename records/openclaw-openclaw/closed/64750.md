## Summary

Keep open. Current main has explicit WhatsApp outbound-media handling and unit coverage, but the report is about live false-success delivery on real WhatsApp sends. Repo inspection does not provide high-confidence evidence that the end-to-end bug is fixed or obsolete, and this exact `message.send` payload path was still being changed after the latest release.

## Evidence

- **Current `message.send` payload path is still being adjusted on main:** `sendPayload` now normalizes `mediaUrl`/`mediaUrls` before delivery and routes media through the shared text+media sender. That is recent churn on the exact surface the issue reports, which argues against closing as obsolete.
