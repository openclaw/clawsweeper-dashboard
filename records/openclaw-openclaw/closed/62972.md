## Summary

Current main already solves the reported endless unknown-tool loop through the embedded-runner unknown-tool guard, while intentionally preserving raw non-policy unknown-tool errors so unrelated cases like invalid `tool_choice` are not masked. That makes this PR obsolete and directionally incorrect for main.
