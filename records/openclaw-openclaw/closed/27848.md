## Summary

Current `main` already implements the core request through the shipped memory stack: indexed hybrid `memory_search`, optional async session-transcript indexing, and the bundled `active-memory` plugin that retrieves relevant memory before the main reply and injects it into prompt context. The implementation is more incremental than the issue’s proposed nightly-only cron, but it solves the same problem well enough on main and is present in the `v2026.4.22` release.

## Risks / Open Questions

- The current implementation is not a literal nightly/off-peak cron indexer for ordinary memory files; it uses more responsive watch/search/interval sync instead.
- Pre-reply retrieval is opt-in through the bundled `active-memory` plugin, so the behavior is available on main but not forced on every agent by default.
