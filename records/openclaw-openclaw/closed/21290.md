## Summary

Keep open. Current `main` contains partial overlapping diagnostics work, but it does not implement this PR's proposed structured OpenTelemetry GenAI lifecycle end-to-end. The shipped/docs-backed surface still revolves around `model.usage`, the exporter ignores lifecycle start events needed for run/inference parent spans, and the discussion shows maintainers/users still treating this as an unresolved gap rather than a completed item.
