Full review comments:

- **[P1] Preserve pinned Docker build inputs:** `Dockerfile:29-33`
  - body: The PR switches the build image to a floating `node:24-bookworm` tag and reinstates `curl | bash` for Bun. Current main intentionally pins the Node and Bun images by digest so release builds use reviewed bytes instead of moving upstream tags or installer scripts.
  - confidence: 0.95
- **[P1] Keep the runtime-assets validation stage:** `Dockerfile:77-78`
  - body: Collapsing prune into the build stage and copying runtime files from `build` skips current `runtime-assets` checks, bundled-plugin postinstall hotfixes, plugin dist pruning, package export validation, runtime patches, and QA assets. That can ship broken or unpatched runtime files.
  - confidence: 0.93
- **[P1] Install the slim-runtime CA, Python, and tini packages:** `Dockerfile:107`
  - body: The runtime apt list drops `ca-certificates`, `python3`, and `tini`. On `node:24-bookworm-slim`, missing CA certificates break Node HTTPS egress, missing Python breaks skills that depend on it, and missing `tini` removes the current init entrypoint behavior.
  - confidence: 0.95
- **[P1] Keep the Docker apt primary-key guard:** `Dockerfile:172`
  - body: The Docker CLI install path compares the first fingerprint but no longer counts primary keys before dearmoring the key file. Current main checks that the Docker apt key has exactly one `pub` key so apt cannot trust extra keys bundled with the pinned one.
  - confidence: 0.9
- **[P1] Restore the node-owned state directory setup:** `Dockerfile:197`
  - body: The final runtime stage no longer pre-creates `/home/node/.openclaw` and related directories with `node:node` ownership and mode `0700` before `USER node`. Named volumes can therefore start root-owned or too permissive, breaking first-run writes or weakening secret-file assumptions.
  - confidence: 0.9
