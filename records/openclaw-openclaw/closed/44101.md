## Summary

Current `main` already aligns the gateway docs with the CLI: the CLI only accepts bind modes, Docker docs explicitly warn against host aliases like `0.0.0.0`, and Kubernetes docs now explain that the default pod bind is loopback and that Service/Ingress exposure requires switching to an appropriate non-loopback bind mode.
