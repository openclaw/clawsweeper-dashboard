## Summary

This RFC should close as ClawHub/plugin-track rather than OpenClaw core. Current main already has the plugin, hook, browser-fill, CLI, and ClawHub distribution surfaces needed to build this externally, while the project vision keeps optional commercial integrations out of core unless maintainers make a separate product/security ownership call.

## What This Changes

Request to add a bundled payment plugin for Stripe Link virtual cards, browser-fill sentinel substitution, and HTTP 402 machine payments.

## Best Possible Solution

Pursue this as an external ClawHub code plugin first, using existing tool, CLI, browser-hook, approval, and message-write hook APIs. If implementation proves a generic SDK gap, open a narrow plugin-API design item; bundled or official promotion should be a separate maintainer/security decision.

## Reproduction Assessment

Not applicable; this is an RFC/feature request, not a bug report. I verified current main has no bundled payment plugin and that the requested integration points are present through the plugin APIs.

## Solution Assessment

Yes. Routing this to ClawHub is the narrowest maintainable outcome because the capability is optional, service-specific, money-moving, and buildable with existing plugin surfaces without adding core payment behavior.
