## Summary

Current main already fixes the Feishu dispatch regression by passing the full gateway channel runtime into Feishu startup and dispatch paths; the fix is main-only and not included in v2026.5.28.

## What This Changes

The issue reports Feishu WebSocket DMs failing to dispatch after upgrading to v2026.5.28 with `TypeError: Cannot read properties of undefined (reading 'run')`.

## Best Possible Solution

Close this release-report as implemented on current main, keep the d05e4a4bc6f runtime-binding fix, and ship it in the next OpenClaw release.

## Reproduction Assessment

Yes for the affected v2026.5.27/v2026.5.28 releases: reporter logs and related confirmations show Feishu DMs reaching the channel and then failing at the inbound runtime `run` path. I did not rerun a live Feishu tenant, and current main has the binding fix.

## Solution Assessment

Yes. The current-main fix restores Feishu to the gateway-provided full channel runtime and has focused gateway coverage proving bundled channel startup receives `inbound.run`.
