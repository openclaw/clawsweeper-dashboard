## Summary

Keep open for maintainer review and possible merge: the PR is a focused cwd hardening fix with no blocking code findings, but it changes managed-update execution cwd and still lacks live managed-service platform proof.

## What This Changes

The PR makes managed Gateway update handoff, `update.run`, `isMainModule`, npm freshness config, and update-runner cwd handling tolerate deleted current/package directories, with regression tests and a changelog entry.

## Best Possible Solution

Land the cwd hardening after maintainers accept the stable-cwd policy and either accept the targeted Linux smoke or require a live managed-service smoke; keep https://github.com/openclaw/openclaw/issues/85246 separate for the skipped-update lifecycle decision.

## Solution Assessment

Yes with a maintainer policy check: hardening the startup guard, handoff helper, update-runner, and npm env cwd paths is a narrow fix for the deleted-cwd failure. The remaining question is whether the stable-cwd fallback and targeted proof are enough before merge.
