## Summary

This is a focused and useful gateway PATH bug fix, but it should stay open because the PATH-shim discovery can persist an unrelated `openclaw` bin when the active argv path cannot be realpath-verified.

## What This Changes

The PR adds active OpenClaw CLI bin discovery to gateway service PATH planning, aligns doctor service-audit expectations, adds focused tests, and records a changelog entry.

## Best Possible Solution

Keep the minimal service PATH policy, add only a verified active OpenClaw CLI bin, and keep doctor audit comparing against the generated expected PATH.

## Reproduction Assessment

Yes, at source level. Current main only adds the selected Node bin to the generated service PATH, and the review finding follows from the PR head condition that accepts PATH candidates when realpath verification is missing.

## Solution Assessment

No, not quite. The overall owner-boundary approach is good, but the PATH shim branch should fail closed unless it can prove the candidate is the active OpenClaw entrypoint.
