## Summary

Current main already routes bundled channel auto-enable through `channels.<id>.enabled` instead of creating `plugins.entries.<id>`, and Feishu is now a bundled channel in that path. The released `v2026.4.22` tag contains the same behavior, so the reported doctor-induced `plugins.entries.feishu` duplicate-warning path is obsolete on supported builds.

## Evidence

- **Doctor auto-enable now treats bundled channels as channel config, not plugin entries:** `registerPluginEntry` checks `normalizeChatChannelId(pluginId)` and, when it matches, writes `channels.<id>.enabled = true` instead of populating `plugins.entries.<id>`.
  - file: [src/config/plugin-auto-enable.shared.ts:596](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/src/config/plugin-auto-enable.shared.ts#L596)
  - command: `nl -ba src/config/plugin-auto-enable.shared.ts | sed -n '560,780p'`
  - sha: [7d4718373698](https://github.com/openclaw/openclaw/commit/7d471837369885c12d540d6bafa319ff41129b89)
- **Built-in channel ids are derived from the bundled channel catalog:** `normalizeChatChannelId` resolves only ids present in the bundled channel catalog, which is how the auto-enable logic recognizes bundled channels.
  - file: [src/channels/ids.ts:12](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/src/channels/ids.ts#L12)
