## Summary

Current `main` already separates volatile inbound metadata from the stable system prompt, adds an explicit system-prompt cache boundary, documents the behavior, and shipped it in `v2026.4.15`.

## Close Comment

Closing as `implemented_on_main` after Codex review. Current `main` already keeps volatile inbound chat/message identifiers out of the stable system prompt and moves them into the user-role conversation block instead (`src/auto-reply/reply/inbound-meta.ts:142`, `src/auto-reply/reply/inbound-meta.ts:198`). OpenClaw also now splits the system prompt at an internal cache boundary so stable prompt content stays above volatile suffix content (`src/agents/system-prompt.ts:941`, `docs/reference/prompt-caching.md:164`), with regression coverage for cache-stable prompt composition (`src/agents/prompt-composition.test.ts:26`, `src/agents/system-prompt-cache-boundary.test.ts:9`). This behavior is called out in the `v2026.4.15` changelog: `CHANGELOG.md:568`.
