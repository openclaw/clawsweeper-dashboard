## Summary

Current `main` already fixes this: TUI `/new` now allocates a unique `tui-<uuid>` session key per client, outgoing chat uses that session key, and TUI event handling ignores events for other session keys. The changelog records this fix in release `2026.3.7`.

## Evidence

- **`/new` now isolates the TUI client with a unique session key:** The `/new` command generates `tui-${randomUUID()}` and switches the client to that session instead of resetting the shared agent session.
  - file: [src/tui/tui-command-handlers.ts:531](https://github.com/openclaw/openclaw/blob/ed6094b3014c56882da15a7942130b00ea410c6e/src/tui/tui-command-handlers.ts#L531)
