## Summary

Keep open: current main still has the 64 KiB OpenAI SSE sanitizer inter-event cap and the issue is paired with an open closing PR that should be reviewed and landed or rejected before cleanup.

## What This Changes

The issue reports a main-only regression where `gpt-5.5` on `openai-chatgpt-responses` fails when a legitimate SSE event exceeds the sanitizer's 64 KiB inter-event buffer cap.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/98198, preserving a tight non-OK error-body cap while allowing legitimate large SSE events on the OpenAI ChatGPT Responses path.

## Reproduction Assessment

Yes, source-reproducible: current main throws when a sanitized SSE response buffers more than 64 KiB before an event boundary, matching the reported failure path. I did not run a live provider call in this read-only review.

## Solution Assessment

Yes, likely: separating the non-OK body cap from the successful SSE inter-event sanitizer cap is the narrow owner-path fix. The open linked PR carries that candidate implementation and should be reviewed on its merge-risk tradeoff.
