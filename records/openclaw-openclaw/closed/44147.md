## Close Comment

Closing as implemented on main. A Codex review found that this blank-dashboard path was addressed in the shipped v2026.3.23 fixes: `CHANGELOG.md:2144` explicitly calls out Control UI auth changes so operator sessions stop failing or blanking on read-backed pages, the browser client now ignores cached operator device tokens that lack read access (`ui/src/ui/gateway.ts:591` with regression coverage in `ui/src/ui/gateway.node.test.ts:309`), and the UI now shows a clear missing-scope warning instead of leaving pages blank (`ui/src/ui/app-settings.ts:670`). If this still reproduces on v2026.3.23 or newer, please open a fresh issue with browser console errors plus `openclaw --version` and `openclaw gateway status` output so maintainers can distinguish any new regression from the already-shipped fix.
