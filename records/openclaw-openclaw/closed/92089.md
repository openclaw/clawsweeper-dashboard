## Summary

Keep this PR open for redesign: it addresses a source-visible goal-lifecycle limitation, but the patch lets models delete active goals, can silently lose archive history, stores persistent state outside the canonical database, and omits required core-tool classification surfaces.

## What This Changes

The PR adds a model-facing `clear_goal` tool with archive options, makes `create_goal` replace terminal goals, and writes cleared-goal history to a workspace JSONL file.

## Best Possible Solution

Define the approved model authority boundary, implement archive-and-clear atomically in the canonical SQLite-backed session owner, preserve active goals unless explicitly authorized, and register the resulting tool across catalog, profiles, construction, display, mutation, docs, and focused tests.

Yes, from source at high confidence: current main rejects a second goal whenever one exists, and the proposed `clear_goal` path has no terminal-status guard. This review did not independently run the contributor's live Windows scenario.

## Solution Assessment

No. A model-side escape hatch may be useful, but this implementation grants broader destructive authority than its stated safety boundary and introduces noncanonical, non-atomic persistence.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.98

Full review comments:

- **[P1] Prevent `clear_goal` from deleting active goals:** `src/agents/tools/goal-tools.ts:235`
  - body: The explicit tool calls `clearSessionGoal` for every existing goal, with no equivalent of the terminal-status guard used by automatic replacement. A model can therefore erase an active in-flight objective despite the PR's stated protection; reject active goals or restrict this tool to the approved terminal statuses.
  - confidence: 0.99
- **[P1] Do not clear the goal after a failed archive write:** `src/agents/tools/goal-tools.ts:118-120`
  - body: `archiveReplacedGoal` suppresses all filesystem failures, after which both callers still remove the canonical goal. A read-only workspace, full disk, or append failure silently loses the promised history; make archive-and-clear one atomic canonical-state operation or return an error without clearing.
  - confidence: 0.99
- **[P1] Store cleared-goal history in canonical SQLite state:** `src/agents/tools/goal-tools.ts:98-100`
  - body: This creates OpenClaw-owned persistent session history under `memory/goal-archive.jsonl`. That state is outside the session transaction, shared through the workspace, and violates the repository's SQLite-only runtime-state ownership rule; put the history in the session or per-agent database instead.
  - confidence: 0.98
- **[P1] Register `clear_goal` as a mutating core tool:** `src/agents/tools/goal-tools.ts:219-223`
  - body: Adding the tool only to `createOpenClawTools` leaves it absent from the core catalog/profile groups, construction planning, display configuration, and explicit mutation classifier. It can be filtered from normal profiles and treated as read-only by mutation-aware behavior; add the same sibling integration and coverage used by the other goal tools.
  - confidence: 0.97
