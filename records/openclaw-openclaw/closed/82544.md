## Summary

Keep open: current main and the latest release still preserve auto-created fallback model pins across later turns, so the reported primary-reconciliation bug is not fixed. Several open PRs target the same behavior with different policies, so maintainer review should choose one path rather than closing this cleanup item.

## What This Changes

The issue asks OpenClaw to stop indefinitely pinning a session to a fallback model after a transient primary-model failure and to reconcile automatic session overrides with the configured primary.

## Best Possible Solution

Land one bounded recovery policy for automatic fallback pins that retries or reverts to the configured primary after transient failover while preserving explicit user-selected model overrides.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: current code writes modelOverrideSource="auto" on fallback, later reads that stored override as the turn's selected model, and tests expect it to persist until reset; I did not run a live Slack/provider outage.

## Solution Assessment

Unclear. The requested recovery direction is sound, but maintainers need to choose the exact policy between immediate rollback, TTL expiry, or throttled primary probing, and it must preserve explicit user model selections.
