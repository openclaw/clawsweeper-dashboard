## Summary

Keep this issue open. Current main still uses the bare `/new` or `/reset` startup instruction as the transcript-visible prompt, and neither the Gateway `chat.history` projection nor Control UI history rendering has a startup marker/filter path. Related #58821 was broader and closed, but it does not cover this specific bare reset transcript path.

## Best Possible Solution

Keep this issue open and fix the owner path. Preserve the startup instruction for the agent runtime, but stop persisting/rendering it as ordinary user chat history. A conservative fix would pass bare reset startup text as runtime-only context or persist a marker such as `__openclaw: { kind: "startup" }` and teach `chat.history` plus Control UI rendering to omit it or show a neutral session divider. Add focused regression coverage for `get-reply-run`, embedded `transcriptPrompt` behavior, `chat.history`, and `ui/src/ui/chat/build-chat-items.ts`.
