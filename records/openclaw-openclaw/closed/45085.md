## Summary

Current `main` already avoids eager context-window warmup for `openclaw status`, and the Anthropic model normalization no longer depends on a top-level `ANTHROPIC_MODEL_ALIASES` const. The same implementation is present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).

## Evidence

- **`status` is explicitly excluded from eager warmup:** `src/agents/context.ts:133` defines `SKIP_EAGER_WARMUP_PRIMARY_COMMANDS` and includes `"status"`; `src/agents/context.ts:152` returns `false` for skipped primary commands, so the module-scope warmup at `src/agents/context.ts:265` does not run for `openclaw status`.
