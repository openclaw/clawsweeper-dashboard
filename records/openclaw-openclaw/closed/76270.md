## Summary

## What This Changes

The PR changes the interactive `agents add` wizard to pre-fill non-default agent workspaces as siblings of the default agent workspace.

## Best Possible Solution

Keep #71941 as the canonical #71889 path: a narrow wizard-prefill fix that preserves #59789 custom shared-base behavior and includes tests plus changelog coverage.

## Reproduction Assessment

Yes. For the linked bug, current source gives a deterministic reproduction: setup persists `agents.defaults.workspace`, the wizard reads `resolveAgentWorkspaceDir`, and that resolver nests non-default agents under the fallback while docs describe peer-level workspaces.

## Solution Assessment

No. A wizard-only fix is the right direction, but this PR applies the sibling path to custom shared roots and omits the tests/changelog coverage already present in the canonical approach.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve custom shared workspace defaults:** `src/commands/agents.commands.add.ts:274-275`
  - body: When `agents.defaults.workspace` is an intentional shared base, current main resolves new non-default agents below that base and has regression coverage for it. This PR instead derives a sibling of the default agent workspace for every non-default agent, so accepting the prompt silently bypasses the configured shared root. Gate this behavior to the documented default/setup workspace case or defer to `resolveAgentWorkspaceDir` for custom roots.
  - confidence: 0.86
