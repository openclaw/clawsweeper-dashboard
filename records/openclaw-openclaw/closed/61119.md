## Summary

Current main already implements the PR’s intended behavior: the connect command helper no longer hardcodes the copy tooltip, aria-label, or nested copy-button label. It now reads locale-backed strings via `overview.connection.copyCommand` and `overview.connection.copyCommandAria`, with matching keys across the supported locale set and shipped in `v2026.4.26`.

## Best Possible Solution

Close this PR as implemented on main and keep the shipped `overview.connection.copyCommand` / `overview.connection.copyCommandAria` implementation. Any remaining hardcoded Control UI strings should be handled by separate, current PRs for those specific surfaces.
