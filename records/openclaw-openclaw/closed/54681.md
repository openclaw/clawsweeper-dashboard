## Summary

Current main already routes native Discord slash commands to the real target session instead of mutating the transient `discord:slash:*` session, and the shipped `v2026.4.22` release snapshot contains the same fix plus regression coverage.

## Evidence

- **Discord native command context carries an explicit target session:** `buildDiscordNativeCommandContext()` now includes both the transient slash `SessionKey` and a separate `CommandTargetSessionKey`, so downstream session mutations can target the real chat session.
  - file: [extensions/discord/src/monitor/native-command-context.ts:67](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/extensions/discord/src/monitor/native-command-context.ts#L67)
  - command: `nl -ba extensions/discord/src/monitor/native-command-context.ts | sed -n '54,70p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Discord native command routing resolves slash and target sessions separately:** `resolveNativeCommandSessionTargets()` returns the slash-session key for interaction routing and `commandTargetSessionKey` for the actual session to mutate; Discord native commands call that helper before dispatch.
  - file: [extensions/discord/src/monitor/native-command.ts:1115](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/extensions/discord/src/monitor/native-command.ts#L1115)
  - command: `nl -ba extensions/discord/src/monitor/native-command.ts | sed -n '1110,1135p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Session initialization now prefers native command target sessions:** `initSessionState()` checks `ctx.CommandTargetSessionKey` for native commands and rewrites `SessionKey` to that target before any bound-session lookup, which avoids creating/resetting state on the slash-only key that caused the reported failure mode.
  - file: [src/auto-reply/reply/session.ts:241](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/auto-reply/reply/session.ts#L241)

  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Regression coverage exists for native command target-session precedence:** A test asserts that `initSessionState()` uses `CommandTargetSessionKey` instead of the bound slash session, covering the exact class of bug described in the issue.
  - file: [src/auto-reply/reply/session.test.ts:1060](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/auto-reply/reply/session.test.ts#L1060)
