## Summary

Current `main` already answers this docs request: the new background-tasks docs explain that tasks are persisted tracking records, not durable execution state, that missing runtime backing becomes `lost` after a 5-minute grace period, and the new Task Flow docs explicitly reserve restart-durable progress for flows rather than plain tasks.
