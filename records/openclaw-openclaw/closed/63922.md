## Summary

Current main already serializes same-file OpenClaw edit/write mutations through the pinned upstream Pi edit tool. OpenClaw's host edit tool delegates to `@mariozechner/pi-coding-agent` `createEditTool`, current main and `v2026.4.24` pin Pi `0.70.2`, and that package wraps the edit read/apply/write cycle in a per-file mutation queue. Native `sessions_spawn` subagents use the same OpenClaw agent/tool path and inherited workspace, so the reported main-agent + subagent workspace edit race is covered.

## Best Possible Solution

Keep the shipped upstream Pi per-file mutation queue for OpenClaw edit/write tools. If someone can still reproduce a cross-process or ACP-specific collision that bypasses these tools, track that as a narrower follow-up for OS-level file locking.
