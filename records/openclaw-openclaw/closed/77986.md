## Summary

Keep open: current main and the latest shipped release still have no hard cap for gateway auth-rate-limiter entries, and the linked closing PR remains open and conflicting rather than merged.

## What This Changes

The issue requests a bounded gateway auth rate-limiter entries map so unique-IP failed-auth floods cannot grow in-memory state until the next prune.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/77987 or an equivalent bounded-limiter fix that caps insertions, preserves active lockouts, and keeps any configurable cap aligned across types, schema, docs, and tests.

## Reproduction Assessment

Yes. Source inspection on current main shows each new non-loopback scope/IP failure creates a Map entry before the periodic prune, with no hard cap or eviction path.

## Solution Assessment

Yes. The narrow maintainable fix is an insertion-time entry cap that evicts only non-locked entries; the linked PR is the active candidate once its config-contract and merge-state issues are resolved.
