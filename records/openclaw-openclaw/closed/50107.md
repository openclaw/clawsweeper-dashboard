## Summary

Keep open. The issue has an open, unmerged implementation PR that explicitly closes it, so the issue should remain open until that PR is reviewed/landed or rejected. Current main also only flattens OpenAI-compatible message content when `compat.requiresStringContent` is set; the NVIDIA bundled provider and the reporter’s shown config do not set that flag, so current main does not clearly solve the reported NVIDIA default/custom-provider path.

## Best Possible Solution

Keep the issue open and review PR #50152 as the implementation candidate. Maintainers should either land a fix that makes the NVIDIA/OpenAI-compatible path normalize text-only content for the affected providers, or explicitly document and accept the `compat.requiresStringContent` opt-in contract and close the PR/issue with that product decision.
