## Summary

Current `main` already has explicit handling for the reported `sessions_spawn(..., mode="run")` timeout/completion race, plus matching docs, regression coverage, and a changelog entry showing the ACP `agent.wait` timeout fix shipped after the reported `2026.4.14` version.

## Evidence

- **Subagent completion recheck:** The announce flow now does a cached `waitForSubagentRunOutcome(..., 0)` recheck when the first outcome was `timeout` but real child output exists, so the final completion event can prefer the authoritative terminal state instead of preserving a stale timeout.
