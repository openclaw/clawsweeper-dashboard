## Summary

Close as superseded: #73008 is the older open canonical implementation for the same DeepSeek V4 xhigh/max gap, and it covers the needed docs/shared model matching while preserving the non-V4 custom-catalog fallback that this branch would shadow. Current main still lacks the behavior, so the cleanup path is to keep review pressure on #73008 rather than carry parallel PRs.

## What This Changes

This PR adds DeepSeek provider thinking-profile constants and tests so `deepseek-v4-flash` and `deepseek-v4-pro` expose `xhigh` and `max` while non-V4 DeepSeek models expose the base levels.

## Best Possible Solution

Use #73008 as the canonical branch: land a DeepSeek-owned V4 profile that advertises `off|minimal|low|medium|high|xhigh|max`, preserves generic fallback behavior for non-V4/custom catalog rows, keeps focused DeepSeek regression coverage, updates the public docs, and then close #73718 and duplicate implementation PRs.
