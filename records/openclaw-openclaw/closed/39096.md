## Summary

Current main already implements the reported fix: channel health now treats stale-socket recovery as a transport-liveness check, not an inbound-message-freshness check, and the shipped v2026.4.22 release notes call out quiet Discord/Slack-style channels no longer being restarted just because no user traffic arrived.
