## Summary

Current `main` already has targeted protections for the reported behavior: canonical main-session alias handling, automatic orphaned-session migration on startup, and TUI stale-session/model snapshot guards. The linked model-bleed symptom also matches a shipped v2026.4.2 fix for session model-switch drift.

## Evidence

- **Canonical main-session alias handling:** `canonicalizeMainSessionAlias()` now normalizes bare `main`, configured `mainKey`, and legacy `agent:main:<mainKey>` aliases onto the configured agent’s canonical main session key instead of letting session reads/writes diverge.
  - file: [src/config/sessions/main-session.ts:47](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/config/sessions/main-session.ts#L47)
- **Round-trip regression coverage for orphaned session mismatch:** The session round-trip regression test documents the exact failure mode where writes produced `agent:main:work` while reads expected `agent:ops:work`, and asserts the keys now match so sessions survive restarts without jumping/loss.
  - file: [src/infra/state-migrations.session-roundtrip.test.ts:1](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/infra/state-migrations.session-roundtrip.test.ts#L1)
- **Legacy aliases map to canonical session store key:** Gateway session-store tests now require legacy aliases like `main`, `agent:ops:MAIN`, `agent:main:main`, and `agent:main:work` to all resolve to the same canonical key.
  - file: [src/gateway/session-utils.test.ts:100](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/gateway/session-utils.test.ts#L100)
- **Automatic startup repair for old broken state:** Gateway startup now runs an orphaned-session-key migration automatically, explicitly to clean up prior write-path bugs on upgrade without requiring a manual doctor pass.
  - file: [src/gateway/server-startup-session-migration.ts:9](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/gateway/server-startup-session-migration.ts#L9)
- **TUI stale snapshot guard plus shipped release note:** TUI session actions now ignore older session snapshots instead of overwriting newer patched model state, and the changelog records the related session model-switch drift fix in `v2026.4.2` (`Models/selection: resolve bare model ids in session model switches...`).
  - file: [src/tui/tui-session-actions.ts:155](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/tui/tui-session-actions.ts#L155)
