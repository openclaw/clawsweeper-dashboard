## Summary

Keep this PR open. Current main still has the unbounded pending tool-result delivery drain the PR targets, and the related deadlock report remains open. The PR is focused and has no apparent supply-chain or CI security regression, but the fixed 30s wall-clock timeout is not clearly safe because current code deliberately serializes tool-result delivery to preserve ordering.

## Best Possible Solution

Keep this PR open, but revise it before merge into a progress-aware or idle-aware bounded drain that preserves toolResultChain ordering. Add targeted regression tests for a never-settling pending tool-result delivery task and for slow healthy serialized deliveries, then review it alongside the still-open #53889 deadlock report.
