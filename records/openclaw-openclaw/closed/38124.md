## Summary

Current `main` already addresses this report’s same-host loopback node-host path and clarifies that WS node-host pairing is handled through device pairing, not `openclaw nodes pending`. The reported `pairing required` + empty `nodes pending` behavior is obsolete against current `main` for the Oracle/VPS-style `openclaw node run --host 127.0.0.1 --port 18789` flow.
