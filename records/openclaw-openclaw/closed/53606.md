## Summary

Current main already supports Feishu document/drive tools through the bundled Feishu plugin without the App Owner-only OAuth path described in the issue. The reported `openclaw-lark` owner-policy code is not present here; current tool execution uses app credentials, channel access policy, and trusted requester auto-grants.

## Evidence

- **Feishu tools use app credentials, not owner OAuth:** `createFeishuClient` builds the SDK client from `appId` + `appSecret` with `AppType.SelfBuild`; there is no per-user owner-OAuth gate in this path.
  - file: [extensions/feishu/src/client.ts:180](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/feishu/src/client.ts#L180)

- **`feishu_doc` executes against that client for read/write/create:** `registerFeishuDocTools` resolves a tool client and directly serves `read`, `write`, `append`, `insert`, and `create` actions.
  - file: [extensions/feishu/src/docx.ts:1420](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/feishu/src/docx.ts#L1420)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)

  - file: [extensions/feishu/src/drive.ts:754](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/feishu/src/drive.ts#L754)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
