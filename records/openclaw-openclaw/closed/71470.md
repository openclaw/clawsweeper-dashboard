## Summary

Current main already guards the BlueBubbles macOS Contacts SQLite lookup by normalizing lookup keys to digit-only strings before building the SQL IN list, and the same implementation is present in v2026.4.24. The PR's sqlite `.parameter set` path is not a better replacement because digit-only values bind as integers in the sqlite CLI and can fail to match the computed text `digits` alias.

## Best Possible Solution

Close this PR as already implemented on main and keep the shipped digit-only normalized literal lookup. If maintainers want additional coverage, add a separate narrow regression test for punctuation or quote-bearing raw participant IDs instead of merging the sqlite version-probe and `.parameter set` branch.
