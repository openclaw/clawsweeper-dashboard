## Summary

#69251 should close as a duplicate of the already-open canonical regression #48457. Current main still has the reported node-exec prepare/binding behavior, but #48457 already tracks the same `system.run.prepare` / `approval cannot safely bind` bug under `security=full` + `ask=off`, and its discussion explicitly broadened from one-liners to real interpreter/script-file invocations.

## Best Possible Solution

Close #69251 as duplicate/superseded by #48457, and keep #48457 as the canonical protected bug. The eventual fix should address the node `system.run.prepare` planner/policy split for `security=full` + `ask=off` and add focused coverage for POSIX `/bin/sh -lc` file/script payloads, while cross-checking the #69077/#69308 shell fallback context.
