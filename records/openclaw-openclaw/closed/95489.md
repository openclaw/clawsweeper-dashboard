## Summary

Keep open: current `main` and the latest release still have the source-level path where `claude-cli` generic failure text can be wrapped as normal visible output, and the issue already has an open closing PR that should be reviewed or rejected before the issue closes.

## What This Changes

The issue reports that `claude-cli` out-of-credits output can be treated as normal visible text, causing the embedded-agent fallback classifier to complete the run instead of trying configured fallback models.

## Best Possible Solution

Land one narrow classifier or CLI result-shaping fix that treats exact generic external runner failure text as fallback-eligible while preserving visible-output, committed-delivery, hook-block, and side-effect guards.

## Reproduction Assessment

Yes at source level: current `main` wraps CLI `output.text` as a normal payload, and the fallback classifier returns `null` before error classification when that payload is visible. I did not reproduce a live exhausted Claude Code account.

## Solution Assessment

Yes, the best fix is a narrow classifier or CLI result-shaping change; changing model ordering, defaults, or configuration would be broader and would not address the broken fallback contract directly.
