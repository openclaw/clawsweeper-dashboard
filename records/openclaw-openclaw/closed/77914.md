## Summary

Keep open: the underlying config-patch failure is source-reproducible, but this PR fixes it by suppressing every root-level unknown-field error for bundled channels, which would hide ordinary typos in channel config.

## What This Changes

The PR ignores root-level AJV `additionalProperties` errors in raw bundled channel validation and adds Feishu unit tests for `replyMode` and `footer` plugin extension fields.

## Best Possible Solution

Keep unknown-field validation for bundled channel roots, but make the config patch/apply raw check respect plugin-provided channel schemas or only suppress extension fields declared by the installed plugin.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes from source: JSON config patch dry-runs call raw bundled validation, and the current generated Feishu schema rejects root unknown fields; I did not run a live `openclaw-lark` setup in this read-only review.

## Solution Assessment

Is this the best way to solve the issue? No. The fix should be plugin-aware or field-aware; skipping every root `additionalProperties` error removes the only raw dry-run typo check for bundled channel root fields.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Avoid suppressing all root unknown channel fields:** `src/config/validation.ts:231-233`
  - body: This guard runs for every bundled channel in raw dry-run validation. Since `channels` is passthrough in the base Zod schema and the generated bundled schemas are the check that catches root typos, this would silently accept mistakes like an unknown `channels.feishu` root key whenever the only AJV error is `additionalProperties`. Please make this suppression plugin-aware or limited to declared extension fields instead of skipping all root unknown-field errors.
  - confidence: 0.87
