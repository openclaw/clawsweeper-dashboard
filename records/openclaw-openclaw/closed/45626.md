## Summary

Keep this PR open. Current main still omits `fileId` from the shared `message` tool schema while Slack still advertises `download-file` and requires `fileId` at dispatch/runtime. The related #64429 work is complementary because it concerns propagating Slack media result IDs, not accepting the `fileId` tool argument. Security review did not find supply-chain or CI risk in the supplied diff; it is a one-file schema addition.

## Best Possible Solution

Review and land a scoped schema fix with a regression test. Either accept the PR’s minimal shared optional `fileId` field, or move the field into Slack’s `describeMessageTool()` schema contribution and cover the current-channel schema path. Treat #64429 as complementary, not a replacement for this schema fix.
