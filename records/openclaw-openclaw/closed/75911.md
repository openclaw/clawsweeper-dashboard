## Summary

Current main and the shipped v2026.5.2 snapshot already implement the WhatsApp/generic dispatch reasoning-suppression behavior with boundary guards and regression coverage, while this PR branch is obsolete and broader than the shipped path.

## What This Changes

The branch adds shared `isReasoning` suppression in auto-reply payload/execution paths, runner tests, and a WhatsApp changelog entry for the reasoning leak.

## Best Possible Solution

Keep the shipped dispatch and WhatsApp delivery guards, close this branch, and track any remaining instability or model-fallback symptoms from #74886 separately if they still reproduce on v2026.5.2 or newer.

## Reproduction Assessment

Yes. Source inspection shows the PR defect clearly: the branch changes shared reasoning-payload routing while current main and v2026.5.2 already have focused dispatch/WhatsApp guards and tests for the leak path.

## Solution Assessment

No. The branch’s shared final-payload filtering is not the narrowest maintainable fix; the shipped boundary guards in `dispatch-from-config` and WhatsApp delivery solve the user-visible leak while preserving channel-specific reasoning lanes.
