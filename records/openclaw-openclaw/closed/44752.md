## Summary

Current main already persists embedded-run auth-profile billing failure state; the report’s expectation that `disabledUntil` should appear in `auth-profiles.json` is outdated because runtime failure-routing state now lives in `auth-state.json`, and the embedded runner explicitly writes it there.

## Evidence

- **Embedded runner marks auth-profile failures during embedded runs:** `runEmbeddedPiAgent` calls `markAuthProfileFailure` from the embedded-run failover path, passing the failing profile, reason, agent dir, run id, and model id.
  - file: [src/agents/pi-embedded-runner/run.ts:600](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/agents/pi-embedded-runner/run.ts#L600)
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Failure-state helper persists updated usage stats with the locked auth-store writer:** `markAuthProfileFailure` updates `usageStats` through `updateAuthProfileStoreWithLock`, which is the persistence path used for billing/auth disable windows and failure counts.
  - file: [src/agents/auth-profiles/usage.ts:682](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/agents/auth-profiles/usage.ts#L682)
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Persisted routing state is written to `auth-state.json`:** `saveAuthProfileStore` writes secret-bearing profiles to `auth-profiles.json` and separately calls `savePersistedAuthProfileState(store, agentDir)` for runtime routing/failure state.
  - file: [src/agents/auth-profiles/store.ts:375](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/agents/auth-profiles/store.ts#L375)
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Regression test documents the split-store design:** A dedicated test asserts that runtime scheduling state is written to `auth-state.json` only, and not into `auth-profiles.json`.
  - file: [src/agents/auth-profiles.store.save.test.ts:134](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/agents/auth-profiles.store.save.test.ts#L134)
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Docs and embedded-run tests match the current persisted location:** The failover docs say runtime auth-routing state and billing disable state live in `auth-state.json`, and the embedded auth-profile rotation e2e helper reads usage state from that file.
  - file: [docs/concepts/model-failover.md:59](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/docs/concepts/model-failover.md#L59)
