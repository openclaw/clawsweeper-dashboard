## Summary

Current `main` already implements the LINE doctor/status fix. The LINE status path now reports config issues from sanitized channel snapshots via `configured`/`tokenSource` instead of re-reading stripped credential fields, and there is regression coverage for the exact false-positive case.

## Evidence

- **Current LINE status adapter uses snapshot-safe status fields:** `lineStatusAdapter` builds account snapshots with `configured: hasLineCredentials(account)` and routes warnings through `collectLineStatusIssues`, so the channel status surface relies on the computed snapshot state instead of raw secret fields.
