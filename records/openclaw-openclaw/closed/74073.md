## Summary

Keep open. Current main and the latest release still have a source-level path where subagent completion selection can return raw tool/toolResult text before a post-compaction assistant reply is available, and the focused fix is still in the open maintainer PR at https://github.com/openclaw/openclaw/pull/80110.

## What This Changes

The issue reports that subagent announce output can freeze or forward raw tool-result JSON when auto-compaction happens before the final assistant reply is visible.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/80110 with an equivalent selector/capture fix that prevents raw tool/toolResult text from becoming successful completion output, preserves timeout partial-progress behavior, and includes regression coverage for compaction and frozen-result paths.

## Reproduction Assessment

Yes, at source level. A `chat.history` snapshot with raw `toolResult` text followed by the synthetic compaction system marker and no later assistant text can still leave `latestRawText` selectable on current main.

## Solution Assessment

No, current main is not the best solved state yet. The issue’s narrow retry idea helps one branch, but the maintainable fix should cover the shared selector, completion capture, frozen-result path, and announce fallback so raw child tool output is never treated as successful completion text.
