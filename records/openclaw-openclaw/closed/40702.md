## Summary

Current `main` already contains the Telegram approval-resolution path that addresses this report: Telegram callback handlers resolve approvals through the shared gateway resolver, non-`plugin:` IDs go to `exec.approval.resolve` first, and only approval-not-found responses can fall back to `plugin.approval.resolve`. The behavior is covered by tests and documented, and the same resolver/test files are already present in release `v2026.4.22`.

## Evidence

- **Telegram callback uses shared approval resolver:** Telegram approval callbacks bypass generic chat-command auth and call the shared resolver with `allowPluginFallback` when the sender is an authorized Telegram approver.
  - file: [extensions/telegram/src/bot-handlers.runtime.ts:1453](https://github.com/openclaw/openclaw/blob/b0d2003fe65bd7a9812f7c7f5ad9f608a7fd1503/extensions/telegram/src/bot-handlers.runtime.ts#L1453)
