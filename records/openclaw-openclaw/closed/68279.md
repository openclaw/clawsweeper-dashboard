## Summary

Keep open. Current main still does not export `stringEnum` or `optionalStringEnum` from the bare `openclaw/plugin-sdk` entry, while the helpers are available from narrower SDK subpaths. The open PR #66214 remains an implementation candidate, and the maintainer comment in the provided context explicitly confirmed this is still valid and left this issue open as the current tracker.

## Best Possible Solution

Keep this issue open and review or replace #66214 with a complete compatibility fix: re-export `stringEnum` and `optionalStringEnum` from `src/plugin-sdk/index.ts` if maintainers choose backward compatibility, then update the SDK API baseline/export checks and add an explicit contract test proving the bare `openclaw/plugin-sdk` entry exposes both helpers. If maintainers instead reject the root barrel compatibility path, document the migration clearly and coordinate affected external plugins.
