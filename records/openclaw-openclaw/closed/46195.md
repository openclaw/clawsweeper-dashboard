## Summary

Current `main` no longer matches this PR's reproduction path. `openclaw plugins list` is metadata-only now, Feishu registration is split through bundled channel entry/setup-entry contracts, and the proposed module-global `areToolsRegistered` guard would skip required tool registration for fresh registries while Feishu tools still capture the registration-time `api`.

## Evidence

- **`plugins list` no longer does full plugin registration:** `src/cli/plugins-cli.ts:149` calls `buildPluginSnapshotReport()`, and `src/plugins/status.ts:225-226` hard-codes that report to `buildPluginReport(..., false)`. In `src/plugins/status.ts:183-202`, `loadModules=false` routes to `loadPluginMetadataRegistrySnapshot(...)` instead of a full `loadOpenClawPlugins(...)` run, so the report path described in the PR does not execute Feishu tool registration on current main.
  - file: [src/plugins/status.ts:225](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/plugins/status.ts#L225)
  - command: `nl -ba src/cli/plugins-cli.ts | sed -n '149,158p'; nl -ba src/plugins/status.ts | sed -n '183,226p'`
  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)
- **Metadata snapshot is validate-only:** `src/plugins/runtime/metadata-registry-loader.ts:19-30` calls `loadOpenClawPlugins(...)` with `mode: "validate"`, `activate: false`, `cache: false`, and `loadModules: false`. That keeps metadata/reporting paths off the full Feishu runtime/tool-registration surface.
  - file: [src/plugins/runtime/metadata-registry-loader.ts:19](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/plugins/runtime/metadata-registry-loader.ts#L19)
  - command: `nl -ba src/plugins/runtime/metadata-registry-loader.ts | sed -n '1,30p'`
  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)
- **Feishu now uses bundled channel entry contracts instead of manual `register()` flow:** Current Feishu entry is `defineBundledChannelEntry(...)` in `extensions/feishu/index.ts:56-81`. The shared contract in `src/plugin-sdk/channel-entry-contract.ts:476-492` always wires runtime/channel registration, but only calls `registerFull(api)` when `api.registrationMode === "full"`. This replaces the old single `register()` shape the PR patched.
  - file: [src/plugin-sdk/channel-entry-contract.ts:476](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/plugin-sdk/channel-entry-contract.ts#L476)
  - command: `nl -ba extensions/feishu/index.ts | sed -n '56,81p'; nl -ba src/plugin-sdk/channel-entry-contract.ts | sed -n '476,492p'`
  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)
- **Feishu has a dedicated setup entry for setup-only/setup-runtime loads:** `extensions/feishu/package.json:22-27` declares `"setupEntry": "./setup-entry.ts"`, and `extensions/feishu/setup-entry.ts:1-12` defines that setup entry. That means startup/setup flows can load Feishu without running full tool registration, which is the opposite of the behavior this PR was trying to guard.
  - file: [extensions/feishu/setup-entry.ts:1](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/extensions/feishu/setup-entry.ts#L1)
  - command: `nl -ba extensions/feishu/package.json | sed -n '22,27p'; nl -ba extensions/feishu/setup-entry.ts | sed -n '1,12p'`
  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)
- **Fresh registries still need fresh Feishu tool registration:** Each plugin load creates a fresh `OpenClawPluginApi` in `src/plugins/loader.ts:2711-2734`, and `src/plugins/runtime.ts:76-89` replaces the active registry rather than mutating one global registry forever. A module-global `areToolsRegistered` flag would therefore skip tool registration for later fresh registries instead of fixing duplication safely.
  - file: [src/plugins/loader.ts:2711](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/plugins/loader.ts#L2711)
  - command: `nl -ba src/plugins/loader.ts | sed -n '2711,2734p'; nl -ba src/plugins/runtime.ts | sed -n '76,89p'`
  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)
- **Feishu tool handlers still depend on the registration-time `api`:** Feishu tool registration closes over `api` and uses it later during execution. For example, `extensions/feishu/src/docx.ts:1406-1414` builds helpers from `api`, and `extensions/feishu/src/tool-account.ts:50-70` resolves tool accounts from `api.config` at execution time. That matches the review concern on the PR: suppressing later registrations with a module-global guard would leave future registries using stale registration-time state.
  - file: [extensions/feishu/src/tool-account.ts:50](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/extensions/feishu/src/tool-account.ts#L50)
  - command: `nl -ba extensions/feishu/src/docx.ts | sed -n '1406,1414p'; nl -ba extensions/feishu/src/tool-account.ts | sed -n '50,70p'`
  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)
