## What This Changes

Make `session_status` and `/status` show live transcript compaction activity during active sessions instead of a stale persisted `compactionCount`.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/78600 so the rendered count uses the freshest safe compaction source, preferably the max of persisted and transcript-observed counts, with regression coverage and concrete proof.

## Solution Assessment

Yes, the narrow maintainable fix is to merge active or transcript-observed compaction evidence into the status display while preserving persisted-store reconciliation. The linked open PR follows that direction, but this issue should stay open until a fix merges.
