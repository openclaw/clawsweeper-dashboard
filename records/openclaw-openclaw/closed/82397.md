## Summary

Keep open for normal maintainer handling. The PR is a focused fix for the linked workspaceOnly skill-read bug, the implementation appears narrowly scoped, proof is sufficient, and I found no blocking review findings.

## What This Changes

The PR lets host `read` access runtime-resolved skill roots under `tools.fs.workspaceOnly` while preserving workspace-only blocking for unrelated reads, writes, edits, and symlink escapes.

## Best Possible Solution

Land a narrowly scoped read-only exception for exactly the runtime-resolved, prompt-visible skill roots, without adding a broad user-configured host read bypass.

## Reproduction Assessment

Yes. Source inspection shows current main emits absolute skill locations but wraps host read with only the workspace root under `workspaceOnly`; the PR proof also shows the before/after behavior through `createOpenClawCodingTools()`.

## Solution Assessment

Yes. Using `SkillSnapshot.resolvedSkills` is the narrowest maintainable fix because it follows the already-resolved visible skill set, avoids a broad `allowRead` config surface, and keeps write/edit plus alias containment checks intact.
