## Summary

Keep open. Current main still treats ACP live output as block deliveries routed through normal channel message paths, while Telegram draft editing is only wired through the native Telegram streaming pipeline; the remaining work is a new channel/plugin live-preview contract decision, not conservative cleanup.

## What This Changes

Requests ACP bound-session live output to update an editable channel draft instead of sending separate block messages, using `acp.stream.deliveryMode: "live"` rather than `streamTo: "parent"`.

## Best Possible Solution

Add an additive typed channel live-preview/draft capability for ACP live delivery, implement Telegram first, preserve block-message fallback for unsupported channels, and cover both ACP dispatch and channel draft behavior with focused tests and docs.

## Reproduction Assessment

Yes for a source-level current-main path: live ACP chunks become `block` deliveries and then route through `routeReply` or `sendBlockReply`. I did not establish a live Telegram/ACP harness reproduction in this read-only review.

## Solution Assessment

Unclear as an exact implementation, but the direction is right: `streamTo: "parent"` is not the draft-editing mechanism, so the maintainable fix is a typed channel draft/live-preview capability used by ACP dispatch.
