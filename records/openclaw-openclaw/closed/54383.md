## Summary

Keep open. Current main still forwards the configured compaction reserve into Pi summarization, Pi 0.71.1 still derives `maxTokens` as 80% of that reserve, and the Anthropic transport still preserves a positive requested `maxTokens`; open PR #54392 remains the linked implementation candidate.

## What This Changes

Clamp built-in compaction summarization output budgets for Anthropic 1M-context models so `/compact` does not request `max_tokens` above the model output limit.

## Best Possible Solution

Land PR #54392 or an equivalent patch that clamps the built-in LLM compaction reserve/output budget against the resolved summarization model output cap, with regression coverage for the Anthropic 1M-context reserve scenario.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `reserveTokensFloor: 300000` reaches Pi as `reserveTokens`, Pi computes `floor(0.8 * reserveTokens)`, and the Anthropic transport sends that positive request without clamping to 128000.

## Solution Assessment

Yes. The narrow maintainable fix is to clamp before Pi `generateSummary` or before Anthropic request construction; the linked open PR is the implementation candidate to review or replace.
