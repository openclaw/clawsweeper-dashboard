## Summary

Keep open. Current main still uses `new URL(...).pathname` for Codex inbound image file URLs, while this PR applies the existing safe file-URL helper through the plugin SDK boundary and includes focused regression coverage plus a changelog entry.

## What This Changes

The branch updates the Codex conversation input builder to decode safe local `file://` image paths, fall back to `mediaUrl` when local conversion fails, and adds regression tests plus a changelog entry.

## Best Possible Solution

Land this PR after normal maintainer review and CI, keeping normalization in the Codex adapter and the shared file-access helper as the contract.

## Reproduction Assessment

Yes, from current-main source inspection and a local Node URL semantics check. The adapter forwards `new URL(...).pathname` values directly, which leaves encoded paths and remote-host file URLs in the wrong shape before Codex receives them.

## Solution Assessment

Yes. Reusing `safeFileURLToPath` through the plugin SDK subpath is the narrowest maintainable fix, and falling back to `mediaUrl` only when local file conversion fails preserves existing local-first behavior.
