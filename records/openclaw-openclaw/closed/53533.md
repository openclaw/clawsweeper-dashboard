## Summary

Keep open for maintainer follow-up. Current main still does not implement Feishu connection health reporting, but this PR branch is stale against the current Feishu runtime path: normal gateway startup never supplies the new status sink, the WebSocket changes are based on the older one-shot startup path, and the branch predates current raw-body webhook signature hardening. The overlapping open PR #39489 covers the same health-monitor work more completely, so maintainers should choose the canonical branch and request a careful rebase rather than close this as implemented.

## What This Changes

The PR branch adds Feishu monitor status-sink plumbing, a WebSocket lifecycle logger/logger passthrough, webhook connect/disconnect status updates, and targeted Feishu monitor/client tests.

## Best Possible Solution

Land one canonical Feishu-owned health-monitor implementation that passes the gateway account status sink from startAccount through monitorFeishuProvider, monitorSingleAccount, event handlers, and both transports; reports connection lifecycle plus inbound activity timestamps; preserves current WebSocket reconnect/heartbeat behavior and raw-body webhook signature verification; then close or rebase overlapping Feishu health PRs.
