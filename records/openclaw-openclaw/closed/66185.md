## Summary

Current `main` contains a post-release fix for the managed `memory-core` dreaming cron path. The code now creates an isolated `agentTurn` dreaming job, handles `trigger === "cron"` directly instead of only heartbeat-driven execution, and has regression tests that cover the exact fast-success/no-artifact scenario. This appears fixed on `main` but not yet in the latest tagged release `v2026.4.23`.

## Evidence

- **Managed dreaming cron now uses isolated agentTurn payload:** `buildManagedDreamingCronJob` now creates the managed job with `sessionTarget: "isolated"`, `payload.kind: "agentTurn"`, the dreaming token in `message`, and `delivery.mode: "none"`, which matches the new direct cron execution path rather than the older heartbeat-only system-event flow.
  - file: [extensions/memory-core/src/dreaming.ts:153](https://github.com/openclaw/openclaw/blob/250acdd2a28c190ed7934e3ba99022f24123d3c9/extensions/memory-core/src/dreaming.ts#L153)
  - sha: [250acdd2a28c](https://github.com/openclaw/openclaw/commit/250acdd2a28c190ed7934e3ba99022f24123d3c9)
- **Promotion handler now accepts cron triggers and writes artifacts:** `runShortTermDreamingPromotionIfTriggered` now accepts both `heartbeat` and `cron`, runs the dreaming sweep, ranks/applies promotions, and writes the deep report artifact via `writeDeepDreamingReport(...)`. That directly addresses the reported `ok`/no-op cron behavior.
  - file: [extensions/memory-core/src/dreaming.ts:490](https://github.com/openclaw/openclaw/blob/250acdd2a28c190ed7934e3ba99022f24123d3c9/extensions/memory-core/src/dreaming.ts#L490)
  - sha: [250acdd2a28c](https://github.com/openclaw/openclaw/commit/250acdd2a28c190ed7934e3ba99022f24123d3c9)
- **before_agent_reply hook now recognizes managed cron runs:** The runtime hook now treats `ctx.trigger === "cron"` as a managed dreaming trigger when the dreaming token is present, instead of requiring a pending heartbeat event. This is the core behavior change that matches the issue symptom.
  - file: [extensions/memory-core/src/dreaming.ts:767](https://github.com/openclaw/openclaw/blob/250acdd2a28c190ed7934e3ba99022f24123d3c9/extensions/memory-core/src/dreaming.ts#L767)
  - sha: [250acdd2a28c](https://github.com/openclaw/openclaw/commit/250acdd2a28c190ed7934e3ba99022f24123d3c9)
- **Regression tests cover the exact cron no-op failure mode:** New tests verify cron triggers without a queued heartbeat event, wrapped cron-label bodies, promotion into `MEMORY.md`, and diary output into `DREAMS.md` for managed cron dreaming.
  - file: [extensions/memory-core/src/dreaming.test.ts:1559](https://github.com/openclaw/openclaw/blob/250acdd2a28c190ed7934e3ba99022f24123d3c9/extensions/memory-core/src/dreaming.test.ts#L1559)
