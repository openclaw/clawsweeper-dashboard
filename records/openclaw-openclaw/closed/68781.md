## Summary

Current `main` already addresses the reported startup-timeout path: secrets preflight now has a non-plugin fast path, startup-gated RPCs fail fast with retryable `UNAVAILABLE`, and Gateway exposes `/healthz` and `/readyz` while sidecars are still settling.

## Evidence

- **Secrets startup fast path:** `prepareSecretsRuntimeSnapshot()` now defaults to `loadAuthProfileStoreWithoutExternalProfiles` for the initial auth-store scan and returns early on a fast path when config/auth stores contain no `SecretRef`s, which avoids the plugin-backed auth overlay during startup preflight.
  - file: [src/secrets/runtime.ts:267](https://github.com/openclaw/openclaw/blob/0ec3b79c07973e746328a1d6e1be5057096029af/src/secrets/runtime.ts#L267)

  - sha: [0ec3b79c0797](https://github.com/openclaw/openclaw/commit/0ec3b79c07973e746328a1d6e1be5057096029af)
- **Auth-store loader without external overlays:** `loadAuthProfileStoreWithoutExternalProfiles()` loads and merges auth stores without calling `overlayExternalAuthProfiles()`, giving the secrets runtime an explicit non-plugin path.
  - file: [src/agents/auth-profiles/store.ts:285](https://github.com/openclaw/openclaw/blob/0ec3b79c07973e746328a1d6e1be5057096029af/src/agents/auth-profiles/store.ts#L285)
