- **[P1] Clear manager state when the relay closes:** `apps/ios/Sources/Voice/RealtimeTalkRelaySession.swift:310`
  - body: When a gateway `talk.event` close arrives, the relay closes itself and sets status to Ready, but nothing tells `TalkModeManager` to clear `isListening` or `captureMode`. Since `start()` returns while `isListening` is true, provider errors or session expiry can leave Talk Mode unable to restart until the user toggles it off/on.
  - confidence: 0.91
- **[P1] Rebind the relay when the chat changes:** `apps/ios/Sources/Voice/TalkModeManager.swift:257`
  - body: `startRealtimeRelay()` captures `self.mainSessionKey` once, and relay tool-call handoffs later use that fixed `options.sessionKey`. `updateMainSessionKey()` only resubscribes chat events, so switching chats while realtime is active can keep tool calls routed to the previous conversation.
  - confidence: 0.88
- **[P2] Report realtime auth from actual credentials:** `apps/ios/Sources/Voice/TalkModeManager.swift:2197-2198`
  - body: This makes the Settings API Key row show `Configured` for every realtime relay config, even when no OpenAI API key or OAuth-backed profile is available. Users without realtime auth will see a configured state until `talk.session.create` fails.
  - confidence: 0.86
- **[P2] Keep auth-profile lookup failures nonfatal:** `src/plugin-sdk/provider-auth.ts:315`
  - body: `resolveProviderAuthProfileApiKey()` now calls the new helper outside the existing catch path, so unreadable or malformed auth-profile state can throw instead of returning `undefined` and letting callers surface normal missing-auth handling.
