- **[P1] Clear manager state when the relay closes:** `apps/ios/Sources/Voice/RealtimeTalkRelaySession.swift:308-310`
  - body: When a `talk.event` close arrives, the relay closes itself and sets status to Ready, but `TalkModeManager` still has `isListening = true` from `startRealtimeRelay()`. Since `start()` returns immediately while `isListening` is true, a provider/session close can leave Talk Mode unable to restart until the user manually toggles it.
  - confidence: 0.91
- **[P1] Rebind realtime relay when the chat session changes:** `apps/ios/Sources/Voice/RealtimeTalkRelaySession.swift:336`
  - body: Tool calls are sent with `self.options.sessionKey`, which is captured when the relay is created. If the user switches chats while realtime Talk Mode is active, `updateMainSessionKey()` only updates subscription bookkeeping, so later tool-call handoffs can still target the old conversation.
  - confidence: 0.88
- **[P2] Report realtime auth status from credentials:** `apps/ios/Sources/Voice/TalkModeManager.swift:2197-2198`
  - body: This marks `gatewayTalkApiKeyConfigured` true for every realtime relay mode, even when no API key or OAuth-backed OpenAI profile is available. Settings can show `Configured` before `talk.session.create` fails with missing auth, so the row needs to reflect actual credential availability or avoid claiming configured.
  - confidence: 0.9
- **[P2] Preserve missing-auth fallback on store load errors:** `src/plugin-sdk/provider-auth.ts:315`
  - body: `resolveProviderAuthProfileApiKey()` now calls `resolveUsableProviderAuthProfiles()` outside a catch, so corrupted or unreadable auth profile data can throw instead of returning `undefined` for normal missing-auth handling. Keep resolution failures on the previous missing-auth path while still using the fallback store when it loads successfully.
  - confidence: 0.82
- **[P1] Restore the current-main Windows installer fixes:** `scripts/install.ps1:658-684`
  - body: This branch deletes the current-main portable Git PATH persistence and repo-pinned pnpm logic, plus the matching docs/tests. That regresses Windows git-backed installs and updates that rely on user-local Git and the repository package-manager version, so the iOS PR should rebase/drop those installer changes.
