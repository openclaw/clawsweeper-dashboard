## Summary

Keep open. Current `main` documents the `OPENCLAW_DISABLE_BONJOUR=1` workaround and plugin-disable path, but I did not find a code change that clearly fixes the reported headless-Docker CPU runaway itself. The gateway still starts local discovery unless it is explicitly disabled, and the bonjour advertiser still uses the same watchdog/re-advertise/recreate loop that matches the issue’s log signature.

## Evidence

- **Local discovery still enabled by default outside explicit opt-out:** `startGatewayDiscovery` only skips local discovery when `discovery.mdns.mode` is `off`, `OPENCLAW_DISABLE_BONJOUR` is truthy, or the process is in test mode. I found no container/headless auto-disable path here.
