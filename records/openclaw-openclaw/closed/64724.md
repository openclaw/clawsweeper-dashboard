## Summary

Current `main` explicitly runs Telegram audio preflight for inbound voice/audio attachments, replaces the `<media:audio>` placeholder with the transcript, and has tests covering both DM and group voice-note paths. Groq is wired as a bundled audio media-understanding provider using `tools.media.audio`, so the reported Telegram + Groq/Whisper path is implemented on `main` even though I could not verify a released tag containing it yet.
