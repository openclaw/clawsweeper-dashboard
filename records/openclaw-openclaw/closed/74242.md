## Summary

Keep this PR open. Current main and v2026.4.26 still only have the consecutive Bonjour advertiser restart cap, and the watchdog still resets that counter whenever a service briefly reaches `announced`, so the announced/probing flapping fix in this PR is not implemented on main. The PR is narrow and addresses prior off-by-one review feedback, but the process-lifetime disable policy needs maintainer review because it can turn repeated LAN-triggered flapping into Bonjour being disabled until restart.

## What This Changes

This PR adds a 30-minute sliding-window cap to the Bonjour advertiser restart watchdog and a regression test for announced/probing flapping in `extensions/bonjour/src/advertiser.ts` and `extensions/bonjour/src/advertiser.test.ts`.

## Best Possible Solution

Keep this PR open for normal maintainer review. If maintainers accept the 5-restarts/30-minute threshold and process-lifetime disable behavior, land this narrow Bonjour plugin fix with its regression test, then keep #74209 open for the broader bundled-plugin startup and isolation work. If the availability tradeoff is not acceptable, adjust this PR toward cooldown/backoff recovery while preserving the announced/probing flap regression coverage.

- This PR only fixes one concrete Bonjour flap shape from #74209; broader default bundled plugin startup and optional-plugin isolation concerns remain in that open issue.
