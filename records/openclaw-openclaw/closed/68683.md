## Summary

Current main now implements the Perplexity web_search path: it registers the bundled Perplexity provider, resolves credentials from plugin config or PERPLEXITY_API_KEY/OPENROUTER_API_KEY, normalizes empty values before env fallback, selects direct Perplexity vs OpenRouter transport, and sends the resolved key as a Bearer Authorization header. The proof commit is on main after the latest release, so this is implemented on main but not proven shipped in v2026.4.26.

## What This Changes

The issue reports that native web_search configured for the Perplexity provider returns Perplexity 401 errors even when the same PERPLEXITY_API_KEY works with direct API calls.

## Best Possible Solution

Keep the main-branch Perplexity plugin implementation and let the next release carry the fix to packaged/Homebrew installs; any remaining provider-specific failures after that should be reopened with current-version gateway logs and sanitized config showing the active credential source.
