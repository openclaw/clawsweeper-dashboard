## Summary

Current main and the shipped release already implement the requested Nextcloud Talk behavior: setup/docs include `--feature response`, 401 sends mention that missing feature, and doctor/status preflight reports a missing response feature. The merged fix in https://github.com/openclaw/openclaw/pull/79657 matches the issue discussion and release evidence.

## What This Changes

The issue requests that Nextcloud Talk onboarding/docs include `--feature response` and that OpenClaw diagnose the misleading 401 send failure when that feature is missing.

## Best Possible Solution

Keep the merged Nextcloud Talk plugin-owned fix and direct affected installs to enable the Talk bot response feature with `talk:bot:state` or reinstall using the current setup command.

## Reproduction Assessment

Yes. The original failure is source-reproducible from the old setup command plus Nextcloud Talk stable32 filtering outbound bot messages by `FEATURE_RESPONSE`; current main replaces that setup path and has regression coverage for the diagnostic and preflight behavior.

## Solution Assessment

Yes. The merged fix is the narrow maintainable solution because it stays in the Nextcloud Talk plugin/docs surface, preserves the real secret-mismatch hint, and adds doctor/status detection without changing core channel contracts.
