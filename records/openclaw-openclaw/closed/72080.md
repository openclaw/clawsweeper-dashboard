## Summary

Current main now addresses the reported ACP Claude early-completion path: the bundled `claude-agent-acp` dependency is patched so an `idle` session state cannot complete a prompt until a Claude `result` has been seen, and regression coverage exercises the exact idle-before-result ordering described in the issue. The fix is also documented as shipped in `v2026.4.26`.

## What This Changes

The issue reports that long-running ACP Claude child sessions can emit a zero-usage `stop` completion before the underlying harness finishes, waking the parent and orphaning work.

## Best Possible Solution

Keep the shipped patched Claude ACP adapter and regression test in place; any future reports should focus on new raw ACP event-order evidence or a different harness/runtime that can still complete before an authoritative result.
