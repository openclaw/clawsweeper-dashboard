## Summary

Current main and the shipped v2026.5.7 tree already implement channel-session `sessions_send` announce delivery, including Discord target resolution and Gateway `send` dispatch.

## What This Changes

The issue requests that `sessions_send` to a channel-bound session deliver the resulting agent-visible reply back to that channel.

## Best Possible Solution

Keep the shipped A2A announce-delivery implementation and direct users on older builds to upgrade rather than opening another core change.

## Reproduction Assessment

No high-confidence failing reproduction on current main was established; source and regression tests show the current path resolves the channel target and calls Gateway `send`. The report is consistent with an older 2026.5.3-1 build, while the inspected release/current source has the delivery path.

## Solution Assessment

Yes. The existing A2A announce path is the narrow maintainable solution because it preserves tool-routed session semantics while posting the target agent's announce back through the normal channel outbound pipeline.
