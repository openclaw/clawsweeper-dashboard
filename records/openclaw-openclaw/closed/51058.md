## Summary

Current `main` already implements the requested fix: Cerebras/OpenAI-compatible completions routes now resolve `supportsStore` as false and therefore omit the `store` field from outbound payloads.

## Evidence

- **Cerebras is explicitly treated as non-standard for completions compat:** `resolveOpenAICompletionsCompatDefaults()` marks both `endpointClass === "cerebras-native"` and default-route `provider === "cerebras"` as non-standard, which makes `supportsStore` default to `false`.
  - file: [src/agents/openai-completions-compat.ts:75](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/agents/openai-completions-compat.ts#L75)
  - command: `nl -ba src/agents/openai-completions-compat.ts | sed -n '54,112p'`
  - sha: [6944d7025d9c](https://github.com/openclaw/openclaw/commit/6944d7025d9c0ba4891cbe646ab5dab039d504ca)
- **Completions payload only sends `store` when compat says it is supported:** `getCompat()` respects explicit/detected `supportsStore`, and `buildOpenAICompletionsParams()` adds `params.store = false` only inside `if (compat.supportsStore)`. With Cerebras resolving to `false`, the field is omitted.
  - file: [src/agents/openai-transport-stream.ts:1530](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/agents/openai-transport-stream.ts#L1530)
