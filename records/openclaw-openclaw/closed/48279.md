## Summary

Current main fixes the ACP `session/list` regression via merged ACP lifecycle work, and the fix is present in the v2026.5.9-beta.1 release; the older unmerged fix PRs are now superseded by main.

## What This Changes

The issue requests restoring ACP `session/list` handling after the ACP TypeScript SDK renamed `unstable_listSessions` to `listSessions`.

## Best Possible Solution

Keep the merged stable ACP lifecycle implementation and treat the remaining open listSessions-only PR as superseded by current main.

## Reproduction Assessment

Yes. The original regression is source-reproducible because pre-fix OpenClaw exposed `unstable_listSessions` while SDK v0.16+ dispatches `session/list` to `listSessions`; current main no longer reproduces because it exposes the stable handler.

## Solution Assessment

Yes. The merged fix uses the SDK-required method name, routes through the existing Gateway `sessions.list` path, adds bounded paging/cwd filtering, and covers the dispatcher-facing contract in tests and docs.
