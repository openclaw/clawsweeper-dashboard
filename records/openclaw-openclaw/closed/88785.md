Adds an optional `chat.history.inFlightRun` snapshot from gateway active-run state and teaches the terminal TUI to re-adopt and render that run when switching back to a session or agent.

## Best Possible Solution

## Solution Assessment

Yes. Gateway active-run state is the right layer because a switched-away client cannot retain events it never receives, and the additive history snapshot is narrower than changing delivery routing or persisting partial responses.
