## Summary

Current `main` already rewires `memory-core` dreaming to use the typed `gateway_start` startup config and live cron getter, adds regression tests for the startup/runtime reconciliation paths described in the report, and documents the supported `plugins.entries.memory-core.config.dreaming` shape. The original silent-initialization failure reported against `v2026.4.5` is implemented on `main`, but I could not tie it to a released tag yet.

## Evidence

- **Current implementation uses startup config plus live cron getter:** `registerShortTermPromotionDreaming` now reconciles dreaming from `ctx.config` on `gateway_start`, stores a live cron getter from the typed hook context, and logs debug/warn/error instead of failing silently when cron is unavailable or startup reconciliation throws.
