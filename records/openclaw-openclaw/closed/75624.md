## What This Changes

Split the memory vector degradation warning so missing embedding readiness is not reported as sqlite-vec unavailability.

## Solution Assessment

Yes. Splitting the diagnostic on the existing load-error state is the narrow maintainable fix; sqlite-vec loading, provider selection, and FTS-only indexing should remain unchanged.
