## Summary

Keep open: the SDK fetch hook is the right seam for custom OpenAI-compatible Chat Completions, but current main still skips SSE sanitation for `provider: "openai"` on `api.openai.com`, so this PR is not complete as claimed without https://github.com/openclaw/openclaw/pull/97139 or an equivalent helper/test change.

## What This Changes

The PR adds `buildGuardedModelFetch(model)` as the custom OpenAI SDK fetch implementation in the OpenAI-compatible Chat Completions client.

## Best Possible Solution

Land the Chat Completions fetch hook together with shared default-endpoint sanitation and focused coverage, or explicitly scope this PR to custom/OpenAI-compatible endpoints only.

## Reproduction Assessment

Yes for the source path, but I did not run a live repro in this read-only review. Current main creates the OpenAI-completions SDK client without a custom fetch, while the proposed guarded fetch still leaves official `api.openai.com` sanitation disabled.

## Solution Assessment

No, not as a standalone PR. The SDK `fetch` option is the right hook, but the branch needs the sibling shared-helper change, equivalent coverage, or a narrower documented scope before it is the best fix.
