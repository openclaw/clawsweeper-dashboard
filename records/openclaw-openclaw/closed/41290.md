## Summary

Current main already fixes the heartbeat timer overflow targeted by PR #41290. The shipped implementation clamps heartbeat scheduler delays through a shared Node-safe timeout helper, warns once when the cap is exceeded, reschedules without hot-looping when the clamped timer fires before the real due time, and has regression coverage. The remaining schema-rejection proposal is a policy difference, not needed to fix the crash/log-spam bug.

## Best Possible Solution

Close #41290 as implemented on main and keep the shipped #71478-style fix: shared safe timeout cap helper, heartbeat scheduler clamp/warn behavior, not-due rescheduling, and regression tests. If maintainers later want a hard config validation limit for heartbeat.every, track that as a separate policy/API issue rather than reviving this obsolete PR.
