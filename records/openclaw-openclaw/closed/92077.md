## Summary

Keep this open. Current main and the latest release still only have partial ClawHub verification plus narrow installed-skill origin/lock metadata; the requested source provenance, moderation transparency, pinned install hash, rich lockfile/audit flow, and sandbox-by-default trust policy remain unresolved, while https://github.com/openclaw/openclaw/pull/92860 covers only a first sourceUrl slice.

## What This Changes

Requests mandatory ClawHub skill source provenance, transparent moderation reports, pinned hash installs, a richer workspace skill lockfile/audit flow, and sandbox-by-default trust promotion.

## Best Possible Solution

Define a phased ClawHub skill trust contract: verified source provenance and moderation transparency first, artifact/hash lockfile plus frozen/audit flows next, then a maintainer-approved sandbox/trust capability policy with upgrade guidance.

## Reproduction Assessment

Yes. The issue is reproducible at source/API level: the sampled live ClawHub API lacks sourceUrl, and current main's lock/origin, CLI, and loader paths omit the requested artifact, moderation, audit, and sandbox policy state.

## Solution Assessment

No, this is not best solved as one autonomous cleanup PR. The maintainable path is a phased security/product design, with the open sourceUrl PR reviewed as a first slice and the broader lockfile, integrity, and sandbox semantics decided by maintainers.
