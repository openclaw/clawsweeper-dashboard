## Summary

Current `main` already supports dragging image files into the Control UI chat, and the same behavior is present in the latest released snapshot `v2026.4.22`.

## Evidence

- **Current chat view accepts dropped image files:** `handleDrop` reads `dataTransfer.files`, filters to supported image MIME types, converts each file with `FileReader`, and appends `ChatAttachment` entries.
  - file: [ui/src/ui/views/chat.ts:272](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/ui/src/ui/views/chat.ts#L272)

  - sha: [d3f6783b16ec](https://github.com/openclaw/openclaw/commit/d3f6783b16ecc10c67b101bf019ca4294453206a)
- **Drop handlers are wired into the rendered chat surface:** The rendered chat section binds `@drop` to `handleDrop` and prevents default on `@dragover`, so dragged files are accepted by the UI instead of doing nothing.
  - file: [ui/src/ui/views/chat.ts:1068](https://github.com/openclaw/openclaw/blob/d3f6783b16ecc10c67b101bf019ca4294453206a/ui/src/ui/views/chat.ts#L1068)

## Close Comment

Closing as implemented. A Codex review of current `main` found drag-and-drop image attachment support already wired into the Control UI chat: `ui/src/ui/views/chat.ts:272` handles dropped files, filters to image MIME types, and converts them into chat attachments, and `ui/src/ui/views/chat.ts:1068` binds the drop/dragover handlers on the chat surface. The same code is present in the latest released snapshot `v2026.4.22` at commit `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`. The exact hover styling proposed in the issue body does not appear to be present, but the requested drag-and-drop image upload behavior itself is already shipped. If there is still a reproducible gap on a current build, please open a new issue with the exact UI surface, app/browser version, and repro steps.
