## Summary

Keep open. Current main has provider usage snapshots, provider-owned usage hooks, a gateway usage endpoint, and an Overview Model Auth hint, but it still lacks the requested manual Control UI fuel gauge at the run/model decision point and the requested source, confidence, stale, and unavailable semantics.

## What This Changes

Add a manual refresh Control UI affordance that shows provider-reported remaining capacity and reset windows before starting expensive agent work.

## Best Possible Solution

Define a normalized provider-capacity contract, keep provider-specific fetching in provider-owned hooks, and add a manual refresh affordance near the selected model/send decision point after the UI and metadata semantics are chosen.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence absence check: usage data exists in gateway/status and Overview surfaces, but no manual decision-point capacity gauge exists in the chat/run toolbar.

## Solution Assessment

Unclear. The existing usage hooks and gateway methods are the right foundation, but the metadata contract, provider reliability semantics, and Control UI placement need maintainer direction before a narrow implementation is clearly the best fix.
