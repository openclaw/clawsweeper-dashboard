## Summary

Close as duplicate. #60550 reports the same remaining read-tool alias/diagnostic question as the older open #56694. Current main still shows the diagnostic only checking `path` and `file_path`, while runtime validation now requires canonical `path`, so this is not implemented-on-main; the best maintainer cleanup is to keep the product/compat decision centralized in #56694. Jah-yee's linked fork PR is useful implementation context for the canonical thread.

## Best Possible Solution

Close #60550 as a duplicate and keep #56694 as the canonical OpenClaw issue for the remaining maintainer decision: either restore/officially support `file` and `filePath` aliases and align diagnostics/runtime/tests, or make the read-tool contract consistently canonical-`path` only across diagnostics, docs, and model guidance.
