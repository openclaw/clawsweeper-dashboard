## Summary

Keep this PR open. Current main does not implement the PR's intended Android Always/background location behavior: Android location mode is still only Off/WhileUsing, stored "always" values are migrated down to WhileUsing, the Settings UI exposes no Always choice, and background `location.get` is still rejected. The PR is a focused implementation candidate with security-sensitive Android permission changes, so it needs maintainer product/security review rather than conservative cleanup closure.

## Best Possible Solution

Keep the PR open for Android maintainer review. If the product decision is to support background location in the third-party Android flavor, review and land a version that preserves Play flavor removal, gates foreground-service location type on real capability, persists/reconciles the Always grant flow, updates docs, and includes targeted Android tests plus manual device proof. If maintainers decide this is out of scope, close it explicitly as a product/security policy decision rather than automated cleanup.
