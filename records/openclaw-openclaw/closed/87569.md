## Summary

Keep open: this targets a real Control UI bug, but the patch should not merge as submitted because moving the picker actions off `click` breaks native keyboard activation, and the PR body only lists tests rather than real after-fix behavior proof.

## What This Changes

The PR changes the Control UI chat session picker search, clear, session option, and load-more buttons from click handlers to left-mousedown handlers, and updates chat view tests to dispatch mousedown.

## Best Possible Solution

Keep the picker fix, but use mousedown only to prevent the focus/blur race while preserving click as the action path, then add keyboard coverage and real browser proof for mouse and keyboard activation.

## Reproduction Assessment

Yes from source: current main binds picker actions to `click` while the focused search input can run its blur handler before the click action. I did not run browser proof in this read-only review, but the linked report gives a concrete Chrome reproduction path.

## Solution Assessment

No as submitted: preventing blur on mousedown is a reasonable direction, but replacing the action `click` handlers removes native keyboard activation. The narrower fix is to keep click for actions and add mousedown only as a blur guard, or otherwise provide explicit keyboard parity.
