## Summary

Close as implemented on main. Current main resolves top-level agent runs from stored session delivery context, forwards that route into the embedded `sessions_spawn` tool context, and seeds child subagent session deliveryContext from the `agent` request. A targeted regression test now covers the pre-patched `sessions.patch` spawn flow described in the issue, and the same implementation is present in the checked-out `v2026.4.23` release snapshot.

## Evidence

  - file: [src/gateway/server-methods/agent.ts:783](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/gateway/server-methods/agent.ts#L783)

  - file: [src/agents/command/run-context.ts:36](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/agents/command/run-context.ts#L36)
