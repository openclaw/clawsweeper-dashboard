## Summary

Current main handles later inbound messages during active runs through the auto-reply follow-up queue, and this PR only adds an unused memory-host SDK helper that is not wired into the live reply path. The linked bug #69384 is already closed, and merging this PR would not change production behavior.

## Best Possible Solution

Close this PR rather than merge an unused SDK helper. Keep the shipped queue semantics on main; if a newer inbound message is still lost or mishandled under current queue settings, the fix should be in the active auto-reply queue/embedded runtime path with a regression test, not in src/memory-host-sdk/engine.ts.
