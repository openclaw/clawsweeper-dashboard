## Summary

Keep open: the report is credible and current main still has the exact source-level behavior where Codex/message-tool-only final text is kept private when no message-tool delivery evidence exists. ## What This Changes

Request a fallback that delivers Codex final agent text or `task_complete.last_agent_message` to the originating channel when message-tool-only delivery produced no outbound and the text is not the silent sentinel.

## Best Possible Solution

Choose one central fallback or warning policy for message-tool-only missed delivery, then implement it behind a shared predicate that covers Codex direct turns and the related group/channel path with duplicate, send-policy, room-event, and sentinel tests.

Yes, source-level: current main explicitly suppresses unmarked final replies under `message_tool_only`, and the Codex adapter separately exposes final assistant text and message-tool delivery evidence. I did not run a live Telegram/Codex reproduction in this read-only review.

## Solution Assessment

Unclear: the fallback would address the silent-response symptom, but it changes a documented privacy policy where final Codex text stays private unless `message` is called. A maintainer-approved shared policy is safer than a one-off task_complete delivery bypass.
