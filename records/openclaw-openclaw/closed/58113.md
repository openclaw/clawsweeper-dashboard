## Summary

Current `main` no longer uses the startup path described in the report: the bundled `memory-lancedb` plugin is lazy-initialized, first-use initialization is serialized behind a shared promise, and the reported secondary Ollama symptom was already fixed earlier.

## Evidence

- **LanceDB init is serialized:** `MemoryDB.ensureInitialized()` now gates first-use setup behind `initPromise`, so concurrent callers share one initialization instead of racing `openTable()` / `createTable()`.
  - file: [extensions/memory-lancedb/index.ts:67](https://github.com/openclaw/openclaw/blob/0783090da0090982c19f7803a9daf9a2ec29538e/extensions/memory-lancedb/index.ts#L67)
  - sha: [0783090da009](https://github.com/openclaw/openclaw/commit/0783090da0090982c19f7803a9daf9a2ec29538e)
- **Bundled memory plugin is lazy, not startup-bootstrapped:** Plugin registration logs `lazy init` and only wires hooks/services; it does not eagerly open LanceDB during registration.
  - file: [extensions/memory-lancedb/index.ts:343](https://github.com/openclaw/openclaw/blob/0783090da0090982c19f7803a9daf9a2ec29538e/extensions/memory-lancedb/index.ts#L343)
  - sha: [0783090da009](https://github.com/openclaw/openclaw/commit/0783090da0090982c19f7803a9daf9a2ec29538e)
- **Recall moved off legacy startup hook:** The plugin now registers recall on `before_prompt_build`, and the regression test explicitly asserts it does not use `before_agent_start`.
  - file: [extensions/memory-lancedb/index.test.ts:172](https://github.com/openclaw/openclaw/blob/0783090da0090982c19f7803a9daf9a2ec29538e/extensions/memory-lancedb/index.test.ts#L172)
  - sha: [0783090da009](https://github.com/openclaw/openclaw/commit/0783090da0090982c19f7803a9daf9a2ec29538e)
- **Memory plugins are separate from context engines:** Current docs state memory plugins and context engines are separate responsibilities, which no longer matches the issue’s assumed single startup bootstrap chain.
  - file: [docs/concepts/context-engine.md:253](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/docs/concepts/context-engine.md#L253)
- **Secondary Ollama symptom already fixed:** The changelog for `2026.3.7` already records a fix for native Ollama compaction/summarization failures with `No API provider registered for api: ollama` outside the main run loop.
  - file: [CHANGELOG.md:3045](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/CHANGELOG.md#L3045)
