## Summary

Keep open. Current main and the latest shipped tag still keep the native relay registry in module-local state and still throw the reported `native hook relay not found` / `Native hook relay unavailable` paths; https://github.com/openclaw/openclaw/pull/73950 is the open closing implementation candidate rather than merged behavior.

## What This Changes

The issue asks OpenClaw to keep or repair the Codex native hook relay after restart/resume so Codex-native shell and MCP `PreToolUse` calls do not remain blocked while ordinary hooks show ready.

## Best Possible Solution

Land or replace the shared native relay state fix so all relay and approval state is shared across duplicate module instances, then treat relay diagnostics or user opt-out as narrower follow-up work.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: current main still has the module-local registry, the explicit relay-miss throw, and the fail-closed CLI response, with multiple reporter comments providing live logs from released installs.

## Solution Assessment

Is this the best way to solve the issue? Yes for keeping it open: the narrow maintainable fix is a shared native relay state repair in the existing Codex/native relay path, not closing before the linked implementation lands.
