## Summary

Current main already preserves configured bundled channel startup and status visibility even under restrictive plugin allowlists, which matches the empty Channels table symptom described in #55744. The same implementation is present in the shipped `v2026.4.22` release commit.

## Evidence

- **Gateway startup now bypasses restrictive allowlists for explicitly configured bundled channels:** `canStartConfiguredChannelPlugin(...)` treats explicit bundled channel config as an allowlist bypass, so configured bundled channels are still started even when `plugins.allow` does not include the channel plugin id.
