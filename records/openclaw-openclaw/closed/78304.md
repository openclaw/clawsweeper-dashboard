## What This Changes

MCP stdio server processes launched by the Gateway should not accumulate after repeated Windows MemPalace MCP use.

## Best Possible Solution

Use #75621 as the canonical thread and fix the shared leak by making bundled MCP runtime ownership compatible across sessions/workspace/config with deterministic teardown, preserving the Windows/MemPalace evidence there.
