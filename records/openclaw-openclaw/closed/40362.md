## Summary

Current main already covers the reported subagent first-run model-selection gap through a newer implementation: subagent spawn now persists the child runtime model before the first run, the gateway agent path reuses the pre-patched session entry, and session model resolution reads persisted runtime model fields alongside explicit overrides. This behavior is already present in the shipped `v2026.4.22` tree, so this PR is superseded.
