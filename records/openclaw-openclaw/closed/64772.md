## Summary

The request is a specialized cross-session memory/continuity policy that fits OpenClaw's documented plugin/ClawHub scope. Current main already exposes stable hook seams to capture reset transcripts and inject next-session context, and it already ships an optional reset-memory hook pattern rather than baking this behavior into core.

## Risks / Open Questions

- No existing ClawHub plugin was linked in this issue, so someone still needs to implement and publish it.
- The bundled session-memory path only covers manual /new and /reset; a plugin that wants full daily-rollover continuity should use session_end/session_start or before_prompt_build instead.
