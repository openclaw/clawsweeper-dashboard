  - body: This branch only reads `entry?.acp?.runtimeOptions?.model`, but current ACP metadata is stored in SQLite and `upsertAcpSessionMeta()` removes `acp` from the JSON session entry after persistence. `listSessionsFromStore()` receives the combined JSON store without merging those ACP rows, so production ACP sessions can still miss the runtime model unless this uses the current ACP metadata reader/context.
  - confidence: 0.9

  - body: `isAcpSessionKey(key)` alone is not enough to decide that the row is an ACP runtime session. Current main documents and tests ACP bridge sessions with ACP-shaped keys but no `SessionAcpMeta`; those run the configured model, while this branch would suppress the default and report no model.
  - confidence: 0.88
- **[P1] Preserve explicit providers for prefixed models:** `src/gateway/session-utils.ts:1058-1060`
