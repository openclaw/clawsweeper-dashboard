## Summary

Current `main` already has the requested pre-execution validation seam: `before_tool_call` runs at the execution boundary with the exact payload/context, can block or require approval, fails closed on hook errors, and is exercised across OpenClaw tool execution paths; the base tool-boundary implementation was already shipped in `v2026.4.23`.

## Evidence

- **Public hook contract documents execution-boundary preflight:** `before_tool_call` receives `toolName`, `params`, optional `runId`/`toolCallId`, plus agent/session context, and may rewrite params, block execution, or require approval before the tool runs.
  - file: [docs/plugins/hooks.md:105](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/docs/plugins/hooks.md#L105)

  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **Runtime executes the hook before tool execution:** The Pi tool path checks for `before_tool_call`, passes normalized params and execution context into the hook runner, and stops execution immediately on `block` or routes through approval before continuing.
  - file: [src/agents/pi-tools.before-tool-call.ts:233](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/agents/pi-tools.before-tool-call.ts#L233)

## Risks / Open Questions

- Current `main` supports blocking/approval for Codex-native pre-tool events, but native Codex argument rewriting is still intentionally unsupported; OpenClaw-owned dynamic tools support param rewriting, native Codex tools do not (`docs/plugins/codex-harness.md:617`).
