## What This Changes

Fix fs-safe writes and copies in Docker images without `python3` so they either work without Python or report a clear actionable dependency error instead of the misleading regular-file error.

## Best Possible Solution

Land one canonical fs-safe fix that handles pinned write/copy helper spawn failures with regression coverage, while preserving existing root-boundary and identity checks. If Docker images intentionally omit `python3`, align Docker docs and diagnostics so operators have an explicit, supported setup path.

## Solution Assessment
