## Summary

Keep this PR open. Current main has an opt-in `compat.requiresStringContent` path for string-only OpenAI-compatible Chat Completions backends, but it does not definitely implement this PR's intended NVIDIA/default-provider behavior: bundled NVIDIA catalog entries and NVIDIA docs still do not opt into that flag, and the linked NVIDIA bug remains plausibly unresolved for users following the shipped setup.

## Best Possible Solution

Keep this PR open for maintainer review and reconcile it with the shipped `compat.requiresStringContent` design. The likely best fix is either to mark affected bundled NVIDIA/strict provider catalog entries with the compat flag plus NVIDIA-focused docs and regression tests, or to add a generic provider/plugin-declared compat path that preserves current newline joining and metadata behavior without blanket-normalizing every OpenAI-compatible backend.
