## Summary

Current main already prevents mixed-provider/API Responses replay from sending foreign `thinkingSignature` metadata as OpenAI `reasoning` items. The shipped path preserves replayable reasoning only for the same provider/API/model, downgrades foreign thinking to plain text, and also drops stale OpenAI Responses reasoning after model-route changes. The behavior is documented and present in release `v2026.4.24`.

## Best Possible Solution

Close #64999 as implemented. Keep the current shared transport transform and session-history sanitizer as the canonical fix; any remaining same-provider Azure reasoning/tool-call replay issue should stay in its narrower follow-up rather than keeping this mixed-provider replay issue open.

## Evidence
