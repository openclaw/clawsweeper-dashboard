## Summary

Keep open. Current main still short-circuits the empty/reasoning-only silent-turn flag behind group-chat detection, so the reported direct-chat `silentReply.direct: "allow"` case remains unfixed in the checked-out code. An open linked PR, #74446, uses closing syntax and appears to target this exact behavior, so the issue should stay open until that PR is reviewed and landed or replaced.

## What This Changes

Allow direct chats configured with `silentReply.direct: "allow"` to treat clean empty or reasoning-only assistant turns as intentional silence instead of entering empty-response retry/fallback.

## Best Possible Solution

Land a narrow fix that removes the group-only short-circuit for the already-resolved silent-reply policy, preserves the default direct disallow/rewrite behavior, adds direct-allow regression coverage, and updates the public docs/changelog to explain the opt-in behavior.
