## Summary

Current `main` already addresses the reported Discord-triggered gateway crash path: the Discord gateway plugin now enables reconnect attempts instead of `0`, and the Discord gateway supervisor/lifecycle code classifies `reconnect-exhausted` events and suppresses late teardown errors so they do not surface as uncaught process-killing crashes.

## Evidence
