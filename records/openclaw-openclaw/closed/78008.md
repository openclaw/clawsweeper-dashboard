## Summary

Keep open: current main and latest release still intentionally reject world-writable plugin paths, while the requested trusted 0777 bypass is a security/product decision rather than a narrow regression fix.

## What This Changes

The issue asks OpenClaw to allow a trusted override for Feishu/Lark plugins installed under world-writable permissions after a Docker/Synology upgrade left the plugin blocked as unsafe.

## Best Possible Solution

Keep the fail-closed plugin path-safety gate and have maintainers choose between Docker/NAS ownership repair, clearer diagnostics, or a narrowly reviewed trusted-source override.

## Reproduction Assessment

No, not for the full Docker/Synology loop. Source and tests clearly show the 0777 plugin block, but the report lacks the affected path, owner/mode, and config write failure needed for a high-confidence end-to-end repro.

## Solution Assessment

No. A broad trusted 0777 bypass is not the narrowest safe fix; the maintainable direction is to preserve the trust boundary unless maintainers approve a narrow override or Docker ownership repair path.
