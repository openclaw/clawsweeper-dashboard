## Summary

Keep open: this is a plausible memory-core bug-fix PR, but it is not mergeable as-is because the recall-only gate would disable daily-only and grounded-only promotion behavior that current tests/docs still encode, and contributor-supplied real behavior proof is missing.

## What This Changes

The PR changes memory-core short-term promotion ranking and apply gates so `minRecallCount` checks `recallCount` instead of aggregate `signalCount`.

## Best Possible Solution

Settle one memory-core promotion signal contract, then update the gate logic, diagnostics, tests, docs, and release notes together so daily/session and grounded evidence are either preserved intentionally or retired explicitly.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence. Current main aggregates recall, daily, and grounded counts for the gate, and current fixtures show daily-only and grounded-only candidates clearing default promotion paths; I did not run tests in this read-only review.

## Solution Assessment

No, not as-is. A recall-only gate addresses the reported false eligibility, but it silently disables current daily and grounded promotion behavior unless maintainers intentionally retire or replace those contracts.
