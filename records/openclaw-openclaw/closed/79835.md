## Summary

Keep open: the PR addresses a real documented gap and has supplied live behavior proof, but it still has a merge-blocking fallback bug where accepted image-model overrides do not carry their prepared image fallbacks into the runner failover path.

## What This Changes

The PR adds Gateway image attachment content sniffing, conditional imageModel per-turn overrides and fallback preparation, auth-profile handling for temporary model switches, and focused regression tests.

## Best Possible Solution

Land the Gateway image detection and temporary override after the prepared image fallback list is threaded into the existing runner `fallbacksOverride` path.

## Reproduction Assessment

Yes, source-reproducible. Current docs define `imageModel` for primary-model image incompatibility, while current main `chat.send` only changes attachment routing and never dispatches an image-model override for the main agent run.

## Solution Assessment

No, not yet. Gateway sniffing plus a per-turn image override is the right boundary, but the accepted override path must also carry `imageModel.fallbacks` into runner `fallbacksOverride` so failover remains vision-capable.
