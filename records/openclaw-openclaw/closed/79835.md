Full review comments:

- **[P2] Thread image fallbacks into run failover:** `src/auto-reply/reply/get-reply.ts:335-337`
  - body: When the primary image override is accepted, this path sets `provider` and `model` but drops `opts.modelOverrideFallbacks`. The runner still builds fallback options from normal model config, so a vision turn can retry on text-model fallbacks instead of configured `imageModel.fallbacks`.

  - body: For cross-provider image overrides, setting `authProfileId` to `undefined` avoids forwarding the old provider profile but also skips provider-scoped auth order for the image provider. Resolve an ephemeral auth profile for the override provider without mutating the stored session profile.
  - confidence: 0.82
