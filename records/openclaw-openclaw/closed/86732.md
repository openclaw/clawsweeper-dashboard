## What This Changes

The PR adds an internal `messageChannel` subscription parameter, uses it to select channel-scoped embedded tool loggers with fallback to `agent/embedded`, and adds focused regression tests for subscriber log attribution and runner channel handoff.

## Best Possible Solution

Land the narrow channel-attribution fix after maintainer review and normal CI, keeping the `agent/embedded` fallback for channel-less, internal, or invalid runs.

## Reproduction Assessment

Yes, from source inspection: current main computes a normalized runtime channel but does not pass it into the embedded subscription, whose tool handlers receive a static `agent/embedded` logger. I did not run the reproduction because this review was read-only.

## Solution Assessment

Yes; passing the existing normalized runtime channel into subscription construction and resolving the logger there is the narrowest maintainable fix for this log attribution path. The fallback behavior remains intact for non-deliverable or missing channels.
