## Summary

Keep open. Current main still omits the `cacheTtlMs` range and default from the Active Memory docs, while the active-memory runtime and manifest define the 1000-120000 ms bounds and 15000 ms default that this PR documents. The linked user report #65708 is closed, but the docs gap remains on current main, and the PR diff is focused on docs/changelog cleanup.

## Best Possible Solution

Review and land this focused docs/changelog PR, or make an equivalent update on main that documents `config.cacheTtlMs` as 1000-120000 ms with a 15000 ms default in `docs/concepts/active-memory.md`, while keeping the credited changelog entry aligned with repository style.
