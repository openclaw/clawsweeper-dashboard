## Summary

Close as cannot_reproduce. Current main still sanitizes signed Anthropic `thinking` text during transport conversion, so the code shape in the report is real, but the related maintainer live verification in #70437 showed that mutated/sanitized thinking text did not trigger Anthropic signature validation failures while missing or invalid signatures did. Current main already strips missing/empty/blank thinking signatures before replay, with regression coverage for that actual live failure mode.

## Best Possible Solution

Close this report as not reproducible against current Anthropic behavior, keep the current invalid-signature cleanup on main, and avoid removing the sanitizer based on the lone-surrogate theory because the related live test showed raw lone surrogates produce invalid JSON while sanitized replay succeeds. If a current failure exists, the next actionable path is a new focused report or PR with a live failing payload, exact model/API version, and a regression test that matches that payload.
