## Summary

Current main already contains the generic gateway startup/restart hardening that matches this 2026.4.15 report: service-mode startup runs stale-gateway cleanup before binding, that cleanup now bounds slow/hung lsof probes, avoids duplicate lsof polling, and excludes the current process plus ancestors so startup helpers cannot kill or loop the active gateway. The changelog shows this shipped in 2026.4.18, and the issue’s only follow-up comment reports a negative repro on 2026.4.21.

## Best Possible Solution

Keep the shipped generic stale-gateway cleanup and bounded lsof behavior. Users still seeing this on 2026.4.15 should upgrade to 2026.4.18 or newer; any recurrence on current releases should be filed as a new issue with the current version, exact service environment, enabled plugins/channels, startup trace, and strace/log bundle.
