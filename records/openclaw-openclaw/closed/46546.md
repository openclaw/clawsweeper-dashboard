## Summary

Current `main` has newer Android voice-playback guardrails, but the issue is not clearly implemented away. The same gateway reply is still routed into both spoken playback and text surfaces, so simultaneous text-plus-voice delivery remains present by code inspection, and the item should stay open for maintainer follow-up.

## Evidence

- **Gateway replies still fan out to voice and chat handlers:** `NodeRuntime.handleGatewayEvent` forwards every gateway event to `micCapture`, `talkMode`, and `chat`, so voice-mode replies still reach both the voice path and the chat UI path.
