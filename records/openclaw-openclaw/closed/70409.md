## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Keep stage-only maintenance from becoming the upgrade default:** `src/memory-host-sdk/dreaming.ts:22`
  - body: Current main and v2026.6.9 automatically promote eligible deep Dreaming candidates into `MEMORY.md`. This PR sets maintenance on with `autoApply: false` and replaces that write path with staged output, so existing enabled Dreaming setups silently stop getting durable memory updates until someone runs `/dreaming apply`. Make stage-only behavior opt-in or add explicit maintainer-approved migration/upgrade handling before changing the default.
  - confidence: 0.86
- **[P1] Persist rollback metadata before applying file changes:** `extensions/memory-core/src/dreaming-maintenance.ts:1214-1217`
  - body: `applyDreamingMaintenance` writes each planned file change before `last-apply.json` is saved. If the process exits after a write but before line 1242, the workspace has modified memory files but no rollback record, and re-apply can conflict on the changed hash. Build and persist the rollback record first, then write the target files.
  - confidence: 0.92
- **[P1] Constrain staged maintenance paths to the workspace:** `extensions/memory-core/src/dreaming-maintenance.ts:1122`
  - body: `writeFileChange` trusts `change.path` from staged-plan and last-apply JSON and joins it directly to the workspace. A tampered staged file can use traversal to write or remove files outside the memory workspace during apply or rollback. Resolve the path and reject anything outside the workspace before any read, write, or unlink.
  - confidence: 0.93
- **[P1] Constrain configured daily signal paths before reading:** `extensions/memory-core/src/dreaming-phases.ts:1070`
  - body: `dailySignalFiles` values are user-configurable and are only string-normalized before `path.join`. A traversal value can read a file outside the workspace and then store its contents as Dreaming memory evidence. Normalize to a safe workspace-relative path and reject entries outside the intended memory-owned area before stat/read.
  - confidence: 0.9
- **[P2] Pass allowPaths when recording configured signal batches:** `extensions/memory-core/src/dreaming-phases.ts:1263-1272`
  - body: The PR makes `recordShortTermRecalls` drop non-date memory paths unless they are in `allowPaths`, but this collected/fallback batch loop never passes the configured signal paths. Configured files like `memory/daily-log.md` can be ignored here and then only first-seeded by the separate fallback, so later edits can go stale instead of feeding deep maintenance.
  - confidence: 0.88
