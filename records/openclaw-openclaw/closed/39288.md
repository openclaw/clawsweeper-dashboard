## Summary

Current `main` already contains Discord gateway lifecycle and gateway health-monitor changes that cover the reported 1006/1005 reconnect loop well enough: the Discord runtime now marks close/reconnect events as disconnected, actively restarts or force-stops stuck gateway lifecycles, and the gateway health monitor auto-restarts disconnected/stale Discord channels instead of requiring a manual restart.

## Evidence

- **Discord lifecycle now records broken gateway state instead of leaving the channel looking healthy:** `runDiscordGatewayLifecycle` pushes `connected: false` plus `lastDisconnect`/`lastError` when the Discord gateway closes or when Carbon schedules a reconnect. Regression tests assert the 1006-close and reconnect-scheduled cases.
