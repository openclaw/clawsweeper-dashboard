## Summary

Current main implements the requested sessions_spawn tolerance: ACP-only streamTo/resumeSessionId are documented as ACP-only, still pass through for runtime="acp", and are ignored for runtime="subagent" instead of triggering the old strict-provider rejection. The merged maintainer PR #72331 carried forward the prior related work and the changelog records this under Unreleased.

## Best Possible Solution

Close as implemented on main. Keep the merged #72331 behavior: tolerate ACP-only fields for native subagent spawns, preserve ACP passthrough, and rely on the existing regression tests and Unreleased changelog entry until the next release ships it.
