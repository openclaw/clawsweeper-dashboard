## Summary

Keep open. The PR is directionally aligned with a real source-level duplicate task-reconciliation path, but this external contribution is missing after-fix real behavior proof and still has small merge-readiness items around changelog coverage and helper scope.

## What This Changes

The branch adds a combined task registry/audit inspection helper, switches status summary to use it, disables unused channel-summary work for plain text `openclaw status`, and updates focused status/task tests.

## Best Possible Solution

A mergeable fix should reduce the status hot path to one task reconciliation pass, avoid unused channel-summary work, keep single-purpose helper APIs cheap for other callers, document the user-facing fix, and include real before/after status timing proof.

## Reproduction Assessment

Partially. Current main clearly reproduces the duplicate source path because status summary calls two helpers that each reconcile inspectable tasks, but I did not live-reproduce the linked 50s gateway stall on a Debian/RPI multi-Discord setup.

## Solution Assessment

Mostly, but not merge-ready. The status optimization is narrow and covered by focused tests, but the helper-wrapper change spreads extra work to summary-only callers and the PR still needs real behavior proof plus a changelog entry.
