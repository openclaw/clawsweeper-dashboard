## Summary

Current main now implements the Telegram-owned fixes that match the reported Windows/npm regression: proxy-aware HTTP/1 transport, webhook-cleanup recovery when Telegram already has no webhook, and bounded timeout/fallback retry for deleteMyCommands and setMyCommands. The final proving commit is on main but is not contained in any release tag yet, so this is fixed on current main rather than in v2026.4.26.

## What This Changes

The issue reports that Telegram initialization on Windows regressed after 2026.4.23, with deleteWebhook and command-sync network failures plus an intermittent installed-package runtime-loading symptom.

## Best Possible Solution

Keep the Telegram-owned fixes on current main and let the next release carry them to npm users; if Windows global-npm installs still show the intermittent runtime-module symptom after that release, reopen with a fresh package smoke log focused on installed dist/runtime files.
