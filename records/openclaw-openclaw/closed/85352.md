## Summary

Keep open: current main still has a source-level path where the macOS dashboard reloads the Control UI and renders the credentials gate before the gateway hello; the related open Control UI PR does not cover the macOS reload path.

## What This Changes

The issue asks that the macOS menu bar dashboard stop briefly rendering the credentials gate when an already authenticated user opens it.

## Best Possible Solution

Preserve authenticated dashboard state across macOS reopens or render an authenticated reconnecting shell during reload, while reserving the credentials gate for missing credentials, first connection, or explicit auth failure.

## Reproduction Assessment

Yes, at source level: the macOS controller reloads the dashboard on show, the Control UI starts disconnected, and renderApp shows the credentials gate for any disconnected state. I did not run a live macOS app proof in this read-only review.

## Solution Assessment

No patch is proposed here, and the related open PR is not sufficient for the macOS every-open reload path. The narrow fix should either avoid reloading an already authenticated dashboard or introduce an authenticated reconnecting state that still falls back to the gate on real auth failure.
