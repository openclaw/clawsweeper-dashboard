## Summary

Current `main` already treats zero-valued/default poll params on `message.send` as unset, which matches the reported misclassification path for media sends. The fix is documented in the `2026.3.22` changelog and covered by regression tests that specifically keep `send` + media requests from being rejected as polls when poll defaults are present.

## Evidence

- **Core poll-intent detection ignores zero/default numeric values:** `hasPollCreationParams` now treats only non-zero finite numeric poll params as poll intent, while still ignoring `false` booleans and empty strings/arrays. That directly addresses the reported "send" misclassification caused by default poll fields leaking into ordinary sends.
