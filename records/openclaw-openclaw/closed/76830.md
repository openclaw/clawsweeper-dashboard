## Summary

Current main already addresses the reported tool-loss path: the tool resolver now falls back to a standalone runtime registry and only warns for requested plugin IDs that still have no tools, so loaded `memory-lancedb` tools are not dropped because unrelated requested plugin IDs are absent. The proof commit is on main after `v2026.5.2`, so this is fixed on current main but not yet proven shipped in a release.

## What This Changes

The issue asks OpenClaw to preserve tools from loaded plugins when `onlyPluginIds` also includes unloaded or uninstalled plugin IDs.

## Best Possible Solution

Keep the current main fallback behavior and let the next release carry it; a future regression test for mixed present/missing tool plugin scopes would make the guarantee explicit.

## Reproduction Assessment

No current-main failing reproduction is available from source inspection. The reported `v2026.5.2` strict lookup path is clear, but current main falls back to a standalone registry and continues with the loaded scoped tool entries.

## Solution Assessment

Yes. The current main solution is narrower than changing `registryContainsPluginIds` globally because it preserves strict compatibility checks for other callers while allowing the tool resolver to use partially loaded scoped tools.
