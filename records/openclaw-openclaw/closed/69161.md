## Summary

Current main already fast-fails cron-triggered exec approval denials: scheduled cron runs carry `trigger: "cron"`, no-route approval requests resolve immediately with `decision: null`, cron exec handling resolves that inline, and denial throws the compact cron-specific error so the cron run records `status: "error"` instead of waiting on approval or returning an approval-pending tool result to the model.

## Best Possible Solution

Close this item as implemented and keep the shipped cron-specific inline denial path. If maintainers want extra assurance, the useful follow-up would be a narrow end-to-end regression that counts model-call diagnostics for a cron exec denial, but the current source and tests already implement the requested fast-fail behavior.
