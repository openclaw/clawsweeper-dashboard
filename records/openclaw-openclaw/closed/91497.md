## Label Justifications

- `P2`: The report affects memory search availability for configured agents but is scoped to the memory index path rather than a full runtime outage.
- `impact:session-state`: The issue concerns persistent memory index identity metadata and whether memory search state remains usable after reindex or reload.

## Summary

Current main covers the reported memory metadata failure: production sync initializes the embedding provider before reindexing, forced CLI reindex enters that sync path, and the gateway/background missing-metadata case was fixed and regression-tested by d46dc39b18 after v2026.6.1.

## What This Changes

The issue reports that forced memory reindexing or gateway reload can write missing or incorrect SQLite memory index metadata, causing later memory status/search identity checks to pause vector search.

## Best Possible Solution

Keep the current-main memory metadata recovery behavior and let it ship in the next release rather than opening another core fix for this already-covered path.

## Reproduction Assessment

Yes from source and regression coverage: current tests model deleting the meta row, observing the missing-metadata status, running sync, and recovering valid identity. I did not run a live CLI command because this review is read-only.

## Solution Assessment

Yes: current main resolves provider identity before production sync/reindex and added a narrow missing-metadata full-reindex condition with regression coverage, which is the maintainable fix shape for this bug.
