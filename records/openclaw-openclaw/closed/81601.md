## Summary

Keep open. The PR appears to be a focused, correct fix for a source-reproducible ClawHub archive body hang, with sufficient terminal proof; it should stay open for normal maintainer review and required checks rather than cleanup close.

## What This Changes

The PR routes ClawHub skill, package, and ClawPack archive downloads through a timed reader-owned body read, passes idle-timeout errors to stream cancellation, adds stalled-body tests, and updates the changelog.

## Best Possible Solution

Land this reader-owned ClawHub archive body timeout fix after maintainer review, then close https://github.com/openclaw/openclaw/issues/52073 and supersede https://github.com/openclaw/openclaw/pull/80006.

## Reproduction Assessment

Yes, at source level: current main clears the fetch timeout once headers arrive and then awaits unbounded arrayBuffer reads for all three archive paths. A headers-then-stalled ReadableStream follows the exact failing path described in the linked report and PR proof.

## Solution Assessment

Yes. Owning the body reader through the shared readResponseWithLimit helper is the narrow maintainable fix because it can time out and cancel the active read, unlike the older https://github.com/openclaw/openclaw/pull/80006 approach that tried to cancel the response body while arrayBuffer owned the stream.
