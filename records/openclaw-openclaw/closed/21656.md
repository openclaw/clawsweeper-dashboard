## Summary

Current main already closes the reported spoofing path: inbound user text that looks like internal `System:` content is rewritten to `System (untrusted):` before it reaches the agent, lower-trust runtime events are rendered as untrusted, and the post-compaction audit injection message was removed in the changelog-shipped fixes.

## Close Comment

Closing from a Codex review because current `main` already hardens this path. Inbound user text now neutralizes spoofed internal markers in `src/auto-reply/reply/inbound-text.ts:8` and applies that sanitization before agent-visible fields are built in `src/auto-reply/reply/inbound-context.ts:44`, so a Telegram/WhatsApp message starting with `System:` is rewritten to `System (untrusted):` before it reaches `BodyForAgent` or `BodyForCommands`. That exact downgrade is covered in `src/auto-reply/inbound.test.ts:85` and `src/auto-reply/inbound.test.ts:124`. Lower-trust queued runtime events are also rendered as `System (untrusted):` in `src/auto-reply/reply/session-system-events.ts:93`. Separately, the post-compaction audit injection message was removed in `CHANGELOG.md:3494`, and the broader prompt-spoofing hardening shipped in `v2026.3.2` per `CHANGELOG.md:3378`.
