## Summary

Current `main` already hardens the `openai-codex` GPT-5 path against the reported promise-only/toolless terminal states: `openai-codex` GPT-5 runs now auto-enter the strict-agentic lane, planning-only/incomplete terminal turns are converted into explicit error states, and the auto-reply fallback path classifies those empty/reasoning/planning-only results as failures instead of treating them as successful execution.
