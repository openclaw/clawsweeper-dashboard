## Summary

Keep open. Current main still continues Discord startup with `botUserId` unset when the bot identity fetch fails, and guild preflight still gates the no-mention drop on `botId`. This PR remains a direct implementation candidate for the linked open security bug; it is not implemented or superseded on main.

## Best Possible Solution

Keep this PR open and review or land it, or manually apply the same fail-closed Discord bot identity behavior plus regression tests. The fix belongs in `extensions/discord/src/monitor/provider.startup.ts` and `extensions/discord/src/monitor/message-handler.preflight.ts`, with focused tests for fetch failure, missing id, and absent-botId mention-pattern preflight.
