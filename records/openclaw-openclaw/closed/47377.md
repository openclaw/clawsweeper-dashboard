## Summary

Close as superseded: this PR addresses only the committed stream-segment path and leaves the cumulative live preview duplication intact, while #46985 is the open canonical Web UI streaming fix covering both paths with regression tests.

## What This Changes

The PR changes `ui/src/ui/app-tool-stream.ts` so text committed before a tool card stores only the suffix after earlier committed stream segments.

## Best Possible Solution

Use #46985, or an equivalent single Control UI fix, as the canonical path so cumulative assistant snapshots are deduped consistently for both committed pre-tool segments and the live preview, with focused regression coverage for repeated text/tool/text cycles.

## Reproduction Assessment

Yes. Current main's source path reproduces the remaining issue: the gateway broadcasts cumulative `mergedText`, the controller assigns it to `chatStream`, tool start commits a prior segment, and the renderer appends the next cumulative `chatStream` unchanged.

## Solution Assessment

No. Segment suffix slicing is necessary, but this PR is incomplete because the live preview needs the same committed-offset dedupe; #46985 is the better current path because it covers both render paths and tests them.
