## Summary

Keep open: the source path for the reported collapse is clear, but the PR broadens all explicit empty fallback overrides despite current docs and tests defining that path as strict, so this needs maintainer policy review and better runtime proof before merge.

## What This Changes

The PR changes model fallback candidate construction to append the configured primary when `fallbacksOverride` is undefined or empty, and updates agent fallback and command tests for pinned-session behavior.

## Best Possible Solution

Decide the intended session-pin policy, then encode it at the session-pin boundary or update the public contract, docs, changelog, and regression tests if empty overrides should no longer mean strict.

## Reproduction Assessment

Yes, by source inspection: user-pinned session models resolve to `fallbacksOverride: []`, and current candidate construction treats that as exact, leaving only the selected model. I did not run a live timeout session in this read-only review.

## Solution Assessment

No. The PR handles the visible pin-collapse by making every empty override append the configured primary, but the safer path is a session-pin-specific policy change or an explicit docs-backed contract update.
