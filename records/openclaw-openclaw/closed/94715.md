## What This Changes

The PR JSON-parses Nextcloud Talk webhook bodies before schema validation, returns HTTP 200/no dispatch for non-Note payloads, preserves HTTP 400 for malformed JSON or malformed Note-shaped payloads, and adds signed replay tests for Document/setup payloads.

## Best Possible Solution

Keep the JSON-before-schema split, classify supported chat messages by top-level Create plus Note shape, acknowledge other valid signed ActivityStreams hooks with 200/no dispatch, and add signed Like-hook proof through the real webhook path.

## Reproduction Assessment

Yes, source-level: current main rejects any signed schema miss with HTTP 400, and PR head still rejects official top-level Like reaction-added hooks because object.type is Note. I did not run a live Nextcloud server in this read-only review.

## Solution Assessment

No, not yet. JSON-before-schema is the right layer, but the nested object.type classifier is too broad for official Like reaction hooks; the durable fix should classify by top-level activity and cover signed reaction fixtures.
