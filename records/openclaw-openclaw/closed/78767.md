## Summary

Keep open: current main still has a source-backed duplicate WhatsApp media dispatch path, and the linked open fix PR should be reviewed or closed before this issue is resolved.

## What This Changes

The issue requests that WhatsApp inbound auto-replies send a single captioned attachment when an assistant reply contains caption text plus one `MEDIA:` directive.

## Best Possible Solution

Keep this issue as the regression tracker until https://github.com/openclaw/openclaw/pull/78770 or an equivalent WhatsApp dispatcher coalescing fix lands.

## Reproduction Assessment

Yes. The reporter supplied live WhatsApp logs for one assistant final reply with one `MEDIA:` directive, and current-main source shows the dispatcher can send a media-only payload before the later captioned payload for the same media URL.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the WhatsApp inbound dispatcher before `deliverReply`, because the duplicate originates from two dispatcher lifecycle deliveries rather than from low-level media sending.
