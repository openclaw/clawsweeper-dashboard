## Summary

Keep open. Current main still lacks an admin-owned, machine-wide hook policy tier, and the same author has a draft companion implementation PR that needs maintainer product/security review rather than cleanup closure.

## What This Changes

Request to add an administrator-owned machine-wide hook policy tier that loads before user hooks and cannot be disabled by user config or hook CLI controls.

## Best Possible Solution

Define the admin trust model first, then implement a hardened core policy-hook or system-plugin design with enforced ownership/ACL checks, reserved execution precedence, CLI/status visibility, docs, changelog, and focused regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The missing capability is high-confidence from current source and docs because hook discovery still has only bundled, plugin, managed, and workspace tiers, and normal hooks remain gated by user-controlled config.

## Solution Assessment

Unclear until maintainers settle the product and security model. Core involvement is likely required if OpenClaw accepts mandatory OS-owned policy hooks, but the current companion PR needs design approval before it is a safe fix path.
