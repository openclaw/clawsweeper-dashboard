Full review comments:

- **[P2] Clear the TUI limit when session info omits it:** `src/tui/tui-session-actions.ts:180-183`
  - body: `sessions.patch` can clear optional fields with `null`, and the resolved entry then omits `verboseLimit`. Because this branch only calls `chatLog.setVerboseLimit` inside the `!== undefined` guard, clearing the stored limit leaves the previous limit active in TUI tool cards.
  - confidence: 0.93
- **[P2] Stop carrying limits across loaded sessions:** `src/tui/tui-session-actions.ts:310-311`
  - body: When `loadHistory` receives a session without `verboseLimit`, this fallback keeps the prior session's value. Switching from a session with a custom limit to one without an override will render the new session's exec cards at the old limit instead of the default.
  - confidence: 0.94
- **[P2] Refresh the Web session after changing the limit:** `ui/src/ui/chat/slash-command-executor.ts:269-270`
  - body: `renderChat` reads exec display limits from the active session row, but this path returns no `action: "refresh"` after patching `verboseLimit`. Since `dispatchSlashCommand` only reloads on that action, Web tool cards keep using the old limit until a manual refresh or navigation.
  - confidence: 0.9
