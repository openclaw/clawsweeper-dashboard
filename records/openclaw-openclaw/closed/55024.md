## Summary

The reported `readStringParam` failure does not match current `main`: the SDK still exports `readStringParam`, duplicate plugin ids are explicitly reduced to a single winning plugin with a warning, and the reporter’s own follow-up identified a local global-vs-bundled Feishu plugin conflict that they resolved outside the repo.

## Evidence
