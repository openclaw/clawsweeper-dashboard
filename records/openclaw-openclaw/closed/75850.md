## Summary

Keep open. Current main still has the reported provider-string gate, so Anthropic-selected sessions with a stored Claude CLI binding and nonempty local history return before reading the Claude JSONL import path.

## What This Changes

The issue asks for Claude CLI history imports to be merged for Anthropic-routed sessions when a Claude CLI session binding exists, instead of requiring `provider === "claude-cli"`.

## Best Possible Solution

Adjust the helper to treat a valid Claude CLI session binding as the safety anchor for Anthropic-selected Claude CLI runtime sessions, preserve the existing merge/dedupe behavior, and add a focused regression test for `provider: "anthropic"` with nonempty local history and an imported Claude JSONL turn.

## Reproduction Assessment

Yes. A deterministic unit reproduction is to call `augmentChatHistoryWithCliSessionImports` with `entry.cliSessionBindings["claude-cli"]`, `provider: "anthropic"`, a valid Claude JSONL home, and at least one local message; current main returns the local array before reading the JSONL.

## Solution Assessment

Yes, with one scope guard. Generalizing the gate from provider string to Claude CLI binding presence for Anthropic/Claude CLI runtime sessions is the narrowest maintainable fix; dropping the gate for every unrelated provider would be riskier.
