## Summary

Current main does not implement this PR’s channel-side voice-format adaptation. The underlying gap still appears real for at least Feishu, even though parts of the PR description are now outdated for Telegram/Microsoft on current main.

## Evidence

- **No channel adaptation in current TTS flow:** `maybeApplyTtsToPayload()` still just forwards `result.audioPath` and sets `audioAsVoice` only when `result.voiceCompatible === true`; there is no shared `ensureVoiceFormat()` step or post-TTS transcode path on main.
