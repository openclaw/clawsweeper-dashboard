## Summary

Current `main` already covers the reported macOS gateway restart failure: channel config changes are hot-reloaded instead of forcing a full restart, restart-required config changes go through the SIGUSR1/in-process restart path with launchd-aware supervision, explicit LaunchAgent restarts use the detached handoff path, and the later stale-PID cascade described in comments is also fixed.
