## Summary

Close as superseded. Current main no longer matches the original post-log hang: supervised SIGUSR1 restarts log the same "supervisor restart" message and then exit. The remaining systemd `Restart=on-failure` recovery gap for existing/manual units is already tracked by open issue #70354, while watchdog support is separately tracked by #11973.

## Best Possible Solution

Close this older issue as superseded, keep #70354 as the canonical place to decide whether supervised SIGUSR1 restarts should exit non-zero or otherwise support existing/manual `Restart=on-failure` units, and keep #11973 for any future systemd watchdog/notify work.

## Evidence
