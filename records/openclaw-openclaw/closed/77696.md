## Summary

Current main and shipped release code already expose and honor `sessions_spawn.lightContext`; the unresolved config path is not part of the current subagent defaults contract.

## What This Changes

The issue asks for `lightContext` to be available on `sessions_spawn` and/or at `agents.defaults.subagents.lightContext`.

## Best Possible Solution

Keep `lightContext` as an explicit per-spawn tool option; a global subagent default should be a separate feature request if maintainers want that policy knob.

## Reproduction Assessment

No. Source inspection of current main shows `sessions_spawn.lightContext` is present in the tool schema and passed through to lightweight bootstrap mode; the reported missing config path is not part of the current subagent defaults schema.

## Solution Assessment

Yes. The narrow maintained solution is the per-call `sessions_spawn.lightContext` parameter, which avoids adding a new global default with broader policy implications.
