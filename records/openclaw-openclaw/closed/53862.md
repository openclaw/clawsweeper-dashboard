## Summary

Current main already handles the reported Telegram polling multi-account group path: it honors the configured default account, uses account-scoped group config in multi-account setups, processes grammY message updates for groups/supergroups, and lets `groupPolicy: "open"` pass. The same implementation is present in release `v2026.4.24`.

## Best Possible Solution

Close this issue as implemented. Keep the shipped Telegram account-scoped group handling and polling pipeline; if a forum-topic-specific variant still reproduces, track it separately on the existing #51119 thread.
