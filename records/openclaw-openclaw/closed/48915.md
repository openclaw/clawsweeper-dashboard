## What This Changes

The issue asks for session transcript repair to recognize snake_case tool call block types so malformed Anthropic `tool_use` history is sanitized before replay.

## Best Possible Solution

Land or rework the linked session transcript repair PR so the matcher handles `tool_use`, `tool_call`, and `function_call`, with focused regression tests for malformed snake_case blocks.

## Reproduction Assessment

Yes, from source inspection. A malformed assistant block with `type: "tool_use"` bypasses `isRawToolCallBlock`, so `sanitizeToolCallInputs` will not drop it even though the camelCase equivalent would be dropped.

## Solution Assessment

Yes, the proposed direction is the narrowest maintainable fix: extend the existing raw tool-call type guard and add regression coverage. Because PR #50802 is open with closing syntax, this issue should stay open while maintainers review or replace that patch.
