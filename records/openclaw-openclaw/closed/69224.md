## Summary

Keep open. Current main and the latest available release tag still have the same Docker packaging shape behind the report: root package metadata references a Baileys pnpm patch file, the source tree has that patch, but the final runtime image COPY list does not include `patches/`. The related in-repo fix PR #69248 was closed unmerged, so there is no canonical merged fix to close this issue against.

## Best Possible Solution

Keep the issue open and land a focused Docker packaging fix: either copy `/app/patches` from `runtime-assets` into the final runtime image, or strip `pnpm.patchedDependencies` from the runtime `package.json` if the patch is not needed after packaging. Add a Docker smoke assertion that the final image can resolve every `pnpm.patchedDependencies` patch path, preferably including a bounded `pnpm install` reproduction path against the built runtime image.
