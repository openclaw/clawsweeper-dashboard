## Label Justifications

- `P2`: This is a real session-routing bug in major messaging channels, but it is limited to native/slash commands under non-default DM scope settings.
- `impact:session-state`: Affected commands can read from or operate on a different direct-message session than ordinary messages in the same conversation.

## Summary

Keep open: current main still has the reported source-level mismatch for Discord native commands and Slack slash commands, while Telegram already uses a fresh runtime config path; the issue is also linked to an open implementation PR, so it should resolve through that PR or a refreshed replacement.

## What This Changes

Native Discord, Telegram, and Slack slash commands in DMs should honor configured `session.dmScope` when resolving the target agent session.

## Best Possible Solution

Refresh, review, or replace the linked PR so Discord and Slack native/slash command route resolution uses fresh route-sensitive config, Telegram's fresh path remains intact, and focused regression proof covers non-main DM scopes.

## Reproduction Assessment

Yes, at source level: `resolveAgentRoute` reads `cfg.session.dmScope`, while Discord and Slack command paths still pass captured startup config into route resolution. I did not run live transport proof in this read-only review.

## Solution Assessment

Yes. The best fix shape is a narrow channel-owned refresh of route-sensitive config in the native/slash command routing paths rather than changing `resolveAgentRoute` globally or adding a new config surface.
