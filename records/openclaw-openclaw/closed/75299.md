## Summary

Keep open. The starvation-guard idea is useful, but this patch is not merge-ready because the aged-priority comparator still lets aged foreground backlog outrank older aged background work, and the supplied proof does not exercise sustained contention.

## What This Changes

The PR adds a 30-second starvation promotion to command-queue dequeue selection and a regression test for aged background work versus fresh foreground work.

## Best Possible Solution

Define one canonical queue-aging policy where starved entries form a distinct tier above non-starved work and drain by enqueue age or sequence, then reconcile it with the open scheduler PR before merge.

## Reproduction Assessment

Yes, at source level against the PR diff. Aged background work gets effective priority 999 while aged foreground work gets 1001, so an older low-priority item can still sit behind an aged foreground backlog.

## Solution Assessment

No. The maintainable fix is to make starvation a separate aged tier ordered by enqueue age or sequence, or to fold that policy into the canonical queue-priority work.
