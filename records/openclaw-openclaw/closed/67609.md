## Summary

Current `main` no longer uses the reported `generativelanguage.googleapis.com` path for the Gemini CLI OAuth flow that the issue describes. The Gemini CLI login path now configures the canonical `google/*` model plus the `google-gemini-cli` runtime, legacy `google-gemini-cli/*` refs are compatibility aliases, and current tests explicitly assert that `google-gemini-cli/gemini-3.1-pro-preview` keeps the `cloudcode-pa.googleapis.com` transport metadata.

## Evidence
