## Summary

Keep open. Current main still starts model-pricing bootstrap during normal gateway startup, still performs OpenRouter and LiteLLM pricing catalog fetches with 60s timeouts when external pricing refs remain, and still has no `models.pricing.enabled` config gate. The open closing PR #68651 is the direct implementation candidate, so this issue should remain open until that PR is reviewed and merged, replaced, or closed.

## Best Possible Solution

Keep this issue open and review #68651 as the canonical fix candidate. A complete fix should add a validated `models.pricing.enabled` config key defaulting to current behavior, skip all pricing bootstrap and catalog network work when disabled, update schema/help/labels/docs/generated baselines and changelog as needed, and add focused tests proving disabled pricing does not fetch OpenRouter or LiteLLM while default/enabled behavior still works.
