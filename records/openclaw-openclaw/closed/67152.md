## Summary

Keep open. Current main still preserves the reported memory-core path: request-scoped subagent startup falls back to a local narrative, returns without a run id, and then the finally block still calls deleteSession on the same subagent runtime. The issue discussion correctly identifies this as a non-fatal lifecycle/scope mismatch, and PR #70554 is only an open implementation candidate, not a shipped fix.

## Best Possible Solution

Keep this issue open until PR #70554 or an equivalent memory-core fix lands. The fix should either use a gateway-valid background subagent runtime for dreaming narrative work or skip request-scoped cleanup after fallback, and it should add regression coverage proving fallback narrative generation does not emit the cleanup warning.
