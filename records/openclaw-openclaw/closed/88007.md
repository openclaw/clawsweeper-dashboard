## Summary

Keep open: current main and the latest release still mutate task-registry memory before sqlite persistence, and the linked open PR is the right implementation path rather than a reason to close this issue early.

## What This Changes

The issue asks for task-registry upsert and delete paths to keep in-memory and sqlite state consistent when sqlite persistence throws.

## Best Possible Solution

Review and land a focused persist-before-commit or rollback-on-persist-fail fix for task-registry updates and deletes, with regression coverage for failed upsert and failed delete paths.
