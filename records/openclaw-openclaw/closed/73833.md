## Summary

Current main already addresses the ACPX missing plugin-local runtime dependency problem: ACPX now declares the Codex ACP adapter as a runtime dependency, startup and doctor repair bundled runtime deps into the managed stage, and the ACPX wrapper resolves the staged package bin instead of trusting a global binary or the old release layout.

## Best Possible Solution

Close this as implemented on current main and keep the current managed runtime-deps staging design. The next shipped release should carry the ACPX runtime dependency and startup/doctor repair path so users no longer depend on stale globals or missing `dist/extensions/acpx/node_modules/.bin` entries in packaged installs.
