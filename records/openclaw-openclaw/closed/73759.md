## Summary

Keep this issue open. Current main has useful follow-up work for `infer model run`: the local path no longer calls `agentCommand`, and gateway raw model probes now skip the CLI backend/tool/context path. But the reported failure is specifically a fresh v2026.4.26 `secrets.resolve` timeout on both local and gateway Anthropic/Claude CLI OAuth runs, and current main still routes `--gateway` through the Gateway `agent` RPC and `resolveAgentRuntimeConfig`, which can still call `secrets.resolve` when agent-runtime SecretRefs are present. I found no regression test or live proof covering the reported Anthropic `anthropic:claude-cli` + gateway snapshot timeout, so this is not safe cleanup yet.

## Best Possible Solution

Keep the issue open and add a focused regression for `openclaw infer model run --local/--gateway --model anthropic/...` with an `anthropic:claude-cli` OAuth profile and any active agent-runtime SecretRef conditions needed to trigger `secrets.resolve`. If the gateway path still times out, fix it in the agent/runtime secret-resolution path, likely by avoiding a gateway self-call for already-active Gateway snapshots or by using a bounded local snapshot resolver for raw model probes. After that, close against the fixing commit and release provenance.
