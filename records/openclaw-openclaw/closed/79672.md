## Summary

Keep open: current main does not already contain this unified catalog feature, and the latest PR head still has source-reproducible blockers around timeout budgeting and live-cache eviction.

## What This Changes

This PR adds a unified model catalog registration path across SDK/plugin registry, manifests, provider docs, media list/help surfaces, live catalog caching, and OpenRouter video model capability overlays.

## Best Possible Solution

Keep the unified manifest-first catalog direction, but revise the timeout and cache behavior before SDK/API review and merge after CI is green.

## Reproduction Assessment

Not applicable for the feature request itself. The blocking PR defects are source-reproducible on the latest head: capability lookup uses a constant 5,000ms budget and the shared live catalog cache leaves expired keys resident unless that exact key is reused.

## Solution Assessment

No, not as currently patched. The direction fits OpenClaw's plugin-owned catalog boundary, but the implementation should consume the caller's remaining timeout budget and prune expired cache entries before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use the caller timeout for capability lookup:** `src/video-generation/runtime.ts:165`
  - body: Every video candidate now gets a fixed 5s capability lookup before provider execution. Requests with a smaller timeoutMs, or fallback chains with several candidates, can wait past the caller's intended deadline before the actual provider operation starts.
  - confidence: 0.94
- **[P2] Prune expired live-catalog cache entries:** `src/plugin-sdk/provider-catalog-shared.ts:57-64`
  - body: Expired cache entries are ignored only when the same hashed key is requested again. Because keys include provider, base-url, and auth facts, token rotation or many provider contexts can leave dead promises in this module-level map for the lifetime of the gateway.
  - confidence: 0.9
