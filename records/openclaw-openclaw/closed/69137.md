## Summary

Close #69137 as duplicate/superseded. The Paperclip `paperclip` root-field rejection is still a real unshipped bug on current main, but the same remaining work is already tracked by the earlier open issue #62102 and the dedicated open PR #69139, which explicitly closes #69137 and adds the schema/test change for this exact failure.

## Best Possible Solution

Track the remaining Paperclip/OpenClaw compatibility fix in the existing canonical surfaces: keep #62102 as the open issue tracker and review or replace #69139 with the maintainer-approved solution. The likely product/code path is either an additive Gateway protocol schema allowance for opaque Paperclip metadata with regression coverage, or a coordinated Paperclip adapter change to send that metadata through a documented supported path.
