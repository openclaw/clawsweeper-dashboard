## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Clear reserved markers before applying context:** `src/agents/bash-tools.exec-runtime.ts:265-268`
  - body: The resolver starts from `baseEnv` and only conditionally writes `OPENCLAW_SESSION_KEY`, and the ACP/TUI helpers repeat the same pattern. A stale parent `OPENCLAW_SESSION_KEY` or `OPENCLAW_AGENT_ID` can survive into a child process with no current context, so clear or overwrite the reserved marker keys before applying the current context.
  - confidence: 0.88
- **[P2] Forward agentId into approved gateway exec runs:** `src/agents/bash-tools.exec-host-gateway.ts:247-263`
  - body: `ProcessGatewayAllowlistParams` carries `agentId`, but the approved gateway execution path still calls `runExecProcess` with only `sessionKey`. In legacy or global session contexts the resolver can fall back to `main`, so approved commands from another agent can receive the wrong `OPENCLAW_AGENT_ID`.
  - confidence: 0.86
