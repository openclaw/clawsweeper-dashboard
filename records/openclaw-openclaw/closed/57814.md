## Summary

Current `main` no longer routes `openai-codex/gpt-5.4` embedded runs through the WebSocket path, so the reported `[ws-stream]` 500 loop is already addressed there.

## Evidence

- **OpenAI Codex is explicitly excluded from embedded WebSocket transport:** `shouldUseOpenAIWebSocketTransport` now returns `false` unless the provider is `openai` and the model API is `openai-responses`. The inline comment explicitly says `openai-codex` stays on the ChatGPT backend HTTP path rather than the public Responses WebSocket endpoint.
