## Summary

Keep open for maintainer review: the patch looks correct and the contributor proof is sufficient, but it intentionally changes oversized Mattermost success responses from unbounded buffering to hard failures at fixed caps, so maintainers should accept or tune that compatibility tradeoff before merge.

## What This Changes

The PR replaces Mattermost REST success JSON reads in client, upload, and probe paths with shared bounded JSON parsing, adds a throwing 64 KiB text fallback, and adds cancellation regression tests.

## Best Possible Solution

Land the bounded readers and regression tests if maintainers accept the caps; otherwise tune the JSON or text limits before merge while preserving bounded cancellation.

## Reproduction Assessment

Yes, source-level. Current main and v2026.6.10 still use unbounded res.json()/res.text() in Mattermost success paths, and the PR body provides terminal proof with a no-Content-Length streaming server.

## Solution Assessment

Yes, pending maintainer acceptance of the caps. The patch uses existing public SDK bounded readers in the client, upload, and probe paths; adding a config knob or a parallel custom reader would be broader than necessary.
