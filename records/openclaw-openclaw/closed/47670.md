## Summary

Current `main` already routes Discord REST traffic through the configured proxy for both startup REST lookups and the proxy-aware Discord request client, with focused tests covering both paths. The same implementation is present in release `v2026.4.22`, so the reported behavior from `2026.3.13` no longer matches current shipped/main behavior.
