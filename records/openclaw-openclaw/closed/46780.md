## Summary

Close as superseded by prior reviewed skill/tool hook decisions: #56885 covered the same skill/tool execution hook request and was closed after source review, while #78062 and #62039 route tool interception and skill usage tracking to plugin/ClawHub surfaces. Current main and v2026.6.6 provide tool-level hooks, skill allowlists, and skill usage diagnostics, but skills are not a separate executable lifecycle primitive.

## What This Changes

Request to add before/after skill execution hooks or plugin events for analytics, failure tracking, monitoring, and authorization before a skill is used.

## Best Possible Solution

Use code plugins with before_tool_call/after_tool_call, agent skill allowlists, and skill usage diagnostics; open a narrow plugin API issue only if a concrete use case cannot be expressed through those supported surfaces.

## Reproduction Assessment

Yes at source level for the absence of the exact hook family: current main has no skill execution hook names, while docs and runtime code show skills are prompt/context resources and runtime interception is tool-scoped.

## Solution Assessment

No as proposed; adding skill-script lifecycle hooks would duplicate prior closed proposals and create a second runtime policy surface where existing plugin tool hooks, allowlists, and diagnostics are the supported path.
