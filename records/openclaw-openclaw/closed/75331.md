## Summary

Close as duplicate/superseded. The crash is real on current main, but open PR #75172 already tracks the same #75168 gateway crash with the same logger guards plus the deferred plugin logger/runtime map backfill and changelog entry that this PR does not include.

## What This Changes

This PR changes gateway channel lifecycle logging to tolerate missing per-channel loggers and adds a regression test for an empty `channelLogs` map during a `startAccount` failure.

## Best Possible Solution

Use #75172, or an equivalent single gateway patch, as the canonical fix: keep the async logger guards, backfill both logger and runtime maps after deferred plugin reload, retain regression coverage for the real missing-map shape, and include the required changelog entry before closing #75168.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a deferred channel plugin that appears after `channelLogs` and `channelRuntimeEnvs` are built, then exits or errors during `startChannels()`, reaching the async `log.error?.(...)` path with `log` undefined.

## Solution Assessment

No. The `log?.` guards are necessary, but this PR is not the best standalone fix because it omits the paired `channelRuntimeEnvs`/`channelLogs` backfill and release-note entry that the canonical open PR already covers.
