## Summary

Keep open: current main still has a source-reproducible unbounded await path in `channels capabilities`, and the Discord targeted diagnostics path still does live permission work without carrying the CLI timeout through that call.

## What This Changes

The issue asks for `openclaw channels capabilities --channel discord` to complete quickly or fail with a bounded, actionable timeout instead of hanging and leaving a CPU-bound `openclaw-channels` child process.

## Best Possible Solution

Bound the `channels capabilities` diagnostic path end to end, including plugin hooks and Discord targeted permission diagnostics, while keeping any install-on-demand policy change separate unless maintainers approve it.

## Reproduction Assessment

Yes, from source: a configured channel plugin whose probe or diagnostics hook never settles will keep `channels capabilities` waiting because the command awaits those hooks directly. The exact Discord CPU-bound child process was not live-reproduced.

## Solution Assessment

Yes. A narrow repair that enforces the existing CLI timeout around capability hook work and carries the timeout through Discord target diagnostics is the most maintainable fix boundary.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/77040. In current main, `openclaw channels capabilities --channel discord` can wait indefinitely because `src/commands/channels/capabilities.ts` directly awaits `probeAccount` and `buildCapabilitiesDiagnostics`, and Discord targeted diagnostics do not carry the CLI timeout through permission fetching. Add a command-level timeout/fallback around capability hook work, propagate or enforce the timeout through Discord target permission diagnostics, and add focused regression tests with a never-settling probe/diagnostics hook. Keep the existing optional-channel install-on-demand product behavior unchanged unless maintainers explicitly approve changing it; if the install path is touched, only bound it or avoid unnecessary work for already-resolved configured channels. Add a single `CHANGELOG.md` Fixes bullet for this user-visible CLI hang. Do not broaden into the separate plugin-tool prep performance cluster.
