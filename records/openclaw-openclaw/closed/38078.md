## Summary

Current main already implements restart-scoped session wake and continuation: restart commands/tools persist a routed sentinel, startup consumes it to wake the saved session and dispatch a one-shot continuation back into the original chat, and the shipped release notes explicitly describe that behavior.

## Close Comment

Current main already implements restart-scoped session wake and continuation: restart commands/tools persist a routed sentinel, startup consumes it to wake the saved session and dispatch a one-shot continuation back into the original chat, and the shipped release notes explicitly describe that behavior.

What I checked:
