## Summary

Current `main` already contains the Feishu/Lark topic-thread routing changes that address this report: `topic_group` messages are handled as group chats, group-session resolution now sees `chatType`, and native topic groups use `thread_id` as the canonical thread key. The provided issue comment pointing to PR #67783 matches the behavior now present in code and tests.
