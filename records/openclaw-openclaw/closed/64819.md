## Summary

Keep open. Current main still does not implement the PR’s intended `openclaw agent --spawn` CLI surface or the spawn-specific gateway flow. The existing subagent machinery is present as `sessions_spawn`, and `--session-id` now appears to resolve explicit agent sessions without the old short-circuit, but the requested fresh CLI-created subagent session contract is not shipped on main.

## Best Possible Solution

Keep the PR open, but ask for a current-main refresh that contains only the `openclaw agent --spawn` feature, focused tests, and docs. The implementation should reuse the existing subagent/session primitives, fail closed if the spawn metadata patch cannot be established, reject contradictory `--local` and `--session-id` combinations, preserve least-privilege gateway scopes, handle `--run-id` retry idempotency, and document how this differs from using a caller-generated `--session-id`.
