## Summary

Current main does not ship the reported unauthenticated Google Places API server. Bundled skills are instruction-only `SKILL.md` entries, and the only Google Places bundled skill is the external-CLI-based `goplaces` skill; the older `local-places` skill/proxy is no longer present.

## Evidence

- **Bundled skills are loaded from `SKILL.md` files:** The bundled skills resolver only looks for a `skills/` directory containing markdown skill files or subdirectories with `SKILL.md`; it does not wire up a bundled HTTP service.
  - file: [src/agents/skills/bundled-dir.ts:6](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/src/agents/skills/bundled-dir.ts#L6)
  - sha: [3361593442e2](https://github.com/openclaw/openclaw/commit/3361593442e295e9bb1f55cbfb38861ea9413e45)
- **Skill loading reads `SKILL.md` metadata/instructions:** `loadSingleSkillDirectory` reads `SKILL.md`, parses frontmatter, and returns a `Skill` record with name/description/file paths. Current bundled skill loading is file-based, not a server runtime.
  - file: [src/agents/skills/local-loader.ts:44](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/src/agents/skills/local-loader.ts#L44)
  - sha: [3361593442e2](https://github.com/openclaw/openclaw/commit/3361593442e295e9bb1f55cbfb38861ea9413e45)
