## Summary

Current `main` already implements the requested `edit`-tool validation change, but in a more complete form than this PR: the runtime now requires `path` plus a validated `edits[]` array, rejects malformed edit entries via a dedicated validator, and includes regression tests for both the missing-`edits` error path and the accepted nested-`edits` path.
