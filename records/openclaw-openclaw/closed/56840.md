## Summary

Current `main` already covers this Discord abort/reconnect crash via the newer supervised gateway shutdown path, so PR #56840 is redundant.

## Evidence

- **Lifecycle now shuts the gateway down through the shared stop helper:** `runDiscordGatewayLifecycle` hands the running gateway to `waitForDiscordGatewayStop(...)` rather than using the older reconnect-sentinel pattern described in the PR.
  - file: [extensions/discord/src/monitor/provider.lifecycle.ts:441](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/extensions/discord/src/monitor/provider.lifecycle.ts#L441)

- **Abort keeps lifecycle listeners attached until disconnect completes:** On abort, `waitForDiscordGatewayStop` calls `gateway.disconnect()` and only then detaches the lifecycle handler, which is the key guard against disconnect-time gateway errors escaping uncaught.
  - file: [extensions/discord/src/monitor.gateway.ts:18](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/extensions/discord/src/monitor.gateway.ts#L18)
