## Summary

Current main already bounds unattended approval waits and converts them into timeout outcomes rather than leaving the subagent task running indefinitely.

## Evidence

- **Unresolved approvals are auto-expired:** `ExecApprovalManager.register()` starts a timer that calls `expire(record.id)`, so pending approvals are automatically resolved on timeout instead of remaining pending forever.

## Risks / Open Questions

- The default exec approval timeout is still 30 minutes, so a waiting task can remain `running` long enough to briefly trigger `stale_running` around that threshold before the timeout path settles.
- This was a read-only code/history review; I did not run a live subagent reproduction in this environment.
