## Summary

This PR should stay open, but it is not merge-ready: the current implementation still leaks ambient process environment into skill-authored setup scripts and only wires the hook into the metadata installer, leaving the linked install/update ClawHub flow uncovered.

## What This Changes

This PR adds `metadata.openclaw.setup` parsing and runs a skill-supplied setup script after the existing metadata install path succeeds, with focused tests for that path.

## Best Possible Solution

Define one constrained setup-hook contract that isolates the hook environment, composes with metadata and ClawHub install/update flows, preserves rollback semantics, documents the SKILL.md field, and has redacted real install/update proof.

## Reproduction Assessment

Yes for the PR defects: source inspection shows the setup runner inherits `process.env` through `runCommandWithTimeout`, and the hook is wired only into the metadata-installer path. As a feature request, current-main bug reproduction is otherwise not applicable.

## Solution Assessment

No. The direction may be viable, but this implementation is not the best merge candidate until it enforces the declared env contract, covers the actual install/update flows, and settles the security/product contract.

  - confidence: 0.9
