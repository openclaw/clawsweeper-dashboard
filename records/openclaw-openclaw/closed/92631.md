No blocking findings. Current main still classifies yielded-plus-aborted subagent lifecycle terminals through abort paths before the yield pause path, and this member-authored PR is the focused implementation candidate for the linked user-visible false-cancellation bug.

## What This Changes

This PR updates subagent lifecycle handling to let explicit yielded terminals pause before abort classification, clears pending lifecycle timers when yield wins, and adds regression coverage for yielded terminals that also carry abort metadata.

## Best Possible Solution

## Reproduction Assessment

Yes. Source inspection shows `sessions_yield` can record yield intent while aborting the run signal, and current main's lifecycle event handler classifies abort before the explicit yield branch.

## Solution Assessment

Yes. The PR changes the narrow owner path, aligns lifecycle event handling with the existing `agent.wait` yield precedence, and avoids broader task-delivery, task-taxonomy, or transport changes.
