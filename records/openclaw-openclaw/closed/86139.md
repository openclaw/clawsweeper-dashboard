## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Keep canonical broker routes under broker prefixes:** `extensions/channel-broker/src/channel.ts:93-96`
  - body: `buildCanonicalBrokerTarget` turns a broker-routed Discord DM into `discord:direct%3A...`. With native Discord installed, current prefix validation treats `discord:` as owned by the native plugin, so a channel-broker session can persist a `To` value that later fails validation or routes through the wrong owner. Keep canonical broker routes under `broker:` or another broker-owned prefix and cover the native-plugin-installed case.
  - confidence: 0.88
- **[P2] Infer broker Discord DMs as direct:** `extensions/channel-broker/src/channel.ts:55-65`
  - body: `inferBrokerTargetChatType` only checks the first component of the parsed conversation id, so `broker:discord:user:123` sees `discord` and falls through to `channel`. Any caller that uses the early inference path can treat a broker DM as group/channel even though the send and route resolver later preserve it as direct. Reuse the broker target parser or add Discord DM coverage here.
  - confidence: 0.82
