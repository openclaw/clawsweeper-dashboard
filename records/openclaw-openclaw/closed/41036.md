## Summary

Current `main` already implements the startup-order fix through a different mainline path: gateway recovery is activated only after `startGatewayPostAttachRuntime()` completes, with regression coverage proving recovery stays inert during initial setup and only runs after sidecars are ready. The open PR is therefore superseded, and its unrelated TUI history-refresh regression is not present on `main`.
