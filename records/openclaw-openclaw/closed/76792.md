## Summary

Keep open. Current main still has the reported early-return warning path when CLI-side memory backend resolution is null, and open PR #76797 is already the active implementation candidate, so this issue should remain open until that PR merges or is replaced.

## What This Changes

Suppress or correct `openclaw doctor`'s no-active-memory-plugin warning when memory search is actually healthy via the configured runtime or gateway status.

## Best Possible Solution

Land or revise #76797 so doctor considers a ready gateway memory probe before emitting the in-process no-runtime warning, then let the issue close through the merged PR.

Yes for a source-level repro: arrange `resolveActiveMemoryBackendConfig` to return null while passing a checked, ready `gatewayMemoryProbe` to `noteMemorySearchHealth`, and current main emits the warning before reading the probe. I did not run a live configured gateway in this read-only review.

## Solution Assessment

Yes, the narrow maintainable solution is to suppress or reword the no-runtime note when gateway memory status proves readiness, with regression coverage for that branch. Because #76797 already targets that fix, the issue should stay open for PR review rather than receive another repair lane.
