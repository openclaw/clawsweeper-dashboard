## Summary

Current main already handles the intended ghost-media boundary safely: ordinary Telegram inbound debounce is text-only, media-bearing default-lane updates bypass the debounce buffer, and forwarded bursts keep media aggregation so albums are not dropped. The PR branch is obsolete and should not be merged as-is because existing review feedback identified both a syntax error and a forwarded-burst media regression.

## Best Possible Solution

Close this PR as redundant. Keep the current main implementation: default-lane debounce remains text-only, forwarded bursts keep media aggregation, and any remaining Telegram redelivery symptom should be handled as a new focused bug with concrete runtime evidence rather than merging the broad last-entry-only patch.
