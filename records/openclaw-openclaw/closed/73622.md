## Summary

Current main implements the clean-install BOOTSTRAP.md fallback: embedded first-run bootstrap now keeps BOOTSTRAP.md out of privileged system context but passes its contents through the user prompt for full bootstrap turns, and constrained runs now receive limited bootstrap guidance instead of no bootstrap prompt. The fix is on main only and is not in the latest tagged release.

## Best Possible Solution

Close as implemented on current main, keep the user-prompt bootstrap fallback and limited-mode diagnostic behavior, and let the next release carry the fix to users who installed v2026.4.26.
