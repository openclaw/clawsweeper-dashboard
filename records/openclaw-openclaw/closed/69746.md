## Summary

Current main already fixes the reported WhatsApp personal-number/Meta AI leak. The inbound path now drops outbound `fromMe` direct messages unless the direct chat is the linked self phone, and regression coverage exercises exactly the non-self `fromMe` DM shape described by the report. The same behavior is present in the shipped v2026.4.23 tag, and the related #66153 ClawSweeper report already identified this as implemented on main.

## Best Possible Solution

Close #69746 as already implemented. Keep the current WhatsApp inbound guard and regression tests; if a future Meta AI case still bypasses this, it should be filed as a new issue with raw Baileys key fields (`remoteJid`, `fromMe`, `participant`), effective `channels.whatsapp` config, and logs from a current release.
