This should stay open: current `main` still has the reported Telegram mention-gate gap, and https://github.com/openclaw/openclaw/pull/84471 is an open fixing PR with closing syntax for this issue.

## What This Changes

The issue asks Telegram ingress to treat `/command@TargetBot` bot-command entities as explicit mentions for the targeted bot in mention-required groups.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/84471 after live Telegram proof, or replace it with the same shared-helper fix and regression coverage if that PR stalls.

## Reproduction Assessment

Yes, source-reproducible: current `main` ignores `bot_command` entities in `hasBotMention`, while both Telegram `requireMention` gate paths depend on that helper before skipping as `no-mention`. I did not execute a live Telegram run in this read-only pass.

## Solution Assessment

Yes: the narrow maintainable fix is to extend the shared Telegram bot-mention helper so both normal inbound handling and the pre-download media gate recognize current-bot targeted commands without weakening group mention policy.
