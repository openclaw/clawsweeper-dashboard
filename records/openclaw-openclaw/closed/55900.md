## Summary

Current `main` already implements the reported workaround as a supported opt-in config: `web_fetch` can allow RFC 2544 fake-IP proxy addresses via `tools.web.fetch.ssrfPolicy.allowRfc2544BenchmarkRange`, and the changelog shows it shipped in `2026.4.10`.

## Evidence

- **Runtime support in web_fetch:** `runWebFetch` reads `params.ssrfPolicy?.allowRfc2544BenchmarkRange` and passes `{ allowRfc2544BenchmarkRange }` into `fetchWithWebToolsNetworkGuard`, so fake-IP RFC 2544 allowance is now supported for `web_fetch`.
