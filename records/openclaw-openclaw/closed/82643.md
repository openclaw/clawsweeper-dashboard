## Summary

Close as already implemented: current main and the published 2026.5.12 package normalize Windows drive-letter paths before jiti/ESM import, so the raw generated call site named in the report is not handed directly to the loader.

## What This Changes

Fix Windows browser-control startup so a raw drive-letter path cannot reach the ESM import pipeline.

## Best Possible Solution

Keep the centralized toSafeImportPath normalization in the plugin loader and reopen only if a fresh install shows a different unnormalized import path with its matching bundle snippet and stack.

## Reproduction Assessment

No high-confidence failing reproduction remains for current main or the 2026.5.12 package: the underlying Node raw-path error is real, but OpenClaw normalizes the reported loaderFilename and target path before import.

## Solution Assessment

Yes. Centralizing Windows path normalization in the shared import helper and plugin loader cache is narrower and safer than patching one generated loader call site.
