## Summary

Current main still persists raw CLI argv into config-audit records, so this issue remains valid. The provided context also shows an open, unmerged closing PR (#75095), and because this is credential-handling/security-sensitive, the next step is maintainer review of that PR or an equivalent focused fix rather than closing this issue or queueing duplicate automation.

## What This Changes

The issue asks OpenClaw to stop writing plaintext secret-bearing CLI argv values into `config-audit.jsonl` audit records.

## Best Possible Solution

Land #75095 or an equivalent focused fix that sanitizes or omits argv at the shared config-audit process snapshot boundary before any write, observe, or observe-recovery record is serialized, with tests covering `--flag value`, `--flag=value`, common token shapes, caller-supplied process info, and forced audit redaction independent of debug logging settings.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to pass a secret-bearing CLI flag within the first eight argv entries and trigger a config write or observe anomaly; current main copies that argv array directly into the JSONL audit record.

## Solution Assessment

Yes. The best fix direction is centralized config-audit process-info sanitization or argv omission, not scattered call-site scrubbing; the open PR follows that direction but is unmerged.
