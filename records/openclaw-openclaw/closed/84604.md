## Summary

Close: current main now handles claude-cli as a provider-owned CLI runtime before the harness-missing branch, and the auth/profile path has current scoped Claude CLI profile creation and setup coverage; the fix is main-only and not in the latest tagged release yet.

## What This Changes

The issue reports that upgrading from OpenClaw 2026.4.15 to 2026.5.18 can leave claude-cli auth/profile runtime state unmigrated and route claude-cli aliases through a missing harness path.

## Best Possible Solution

Keep the current main implementation, ship it in the next release, and handle the related open selection PR separately if it has become redundant.

## Reproduction Assessment

Yes, source-level: the reported failing shape is the forced runtime selector reaching `MissingAgentHarnessError` for `claude-cli`, and current tests now cover that path. I did not run a macOS 2026.4.15 to current-main upgrade in this read-only review.

## Solution Assessment

Yes. The maintainable fix is to keep CLI runtime aliases out of plugin-harness lookup while preserving explicit missing-harness failures, plus provider-owned Claude CLI profile/setup handling instead of a startup-time core migration.
