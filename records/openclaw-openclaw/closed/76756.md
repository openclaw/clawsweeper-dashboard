## Summary

Keep open. Current main still does not expose a `PluginRuntime.gateway` or equivalent plugin-runtime accessor for `status` and `models.list`; the newer `gateway-method-runtime` bridge is explicitly limited to Gateway-authenticated plugin HTTP routes, so it does not cover the reported in-process plugin runtime use case.

## What This Changes

Expose a plugin SDK runtime surface that lets plugins call allowlisted host Gateway RPCs such as `status` and `models.list`.

## Best Possible Solution

Design and document a narrow `PluginRuntime.gateway` facade with allowlisted read-only accessors, redacted status behavior, exported response types, and deterministic replay snapshot semantics.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a request for a new public SDK capability. Source inspection on current main confirms the underlying Gateway RPC handlers exist while the plugin runtime still lacks the requested accessor.

## Solution Assessment

Unclear until maintainer API review. Curated read-only accessors look safer than exposing a generic Gateway client, but maintainers need to settle allowlisting, redaction, public types, and replay behavior first.
