## Summary

Keep open: the performance goal is plausible and current main already has a narrower PI discovery cache, but this branch still introduces stale hook-added bootstrap context reuse, authStorage-bound registry reuse, and lacks real behavior proof.

## What This Changes

The PR adds in-memory caches for bootstrap context and PI model discovery, expands auth profile cache keys, passes channel ids into bootstrap resolution, and adds a tool descriptor cache disable environment variable.

## Best Possible Solution

Refresh or replace the branch on top of current main with a narrower cache design that preserves hook-mutated bootstrap inputs and auth-bound model registry semantics, plus focused regression tests and real behavior proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: enable bootstrap-extra-files, resolve bootstrap context once, edit a matched extra file without changing root bootstrap files, then resolve again and the PR can return the old cached context before hooks rerun.

## Solution Assessment

No. Caching final hook-mutated context and caller-auth-bound registries is too broad; a safer approach is to cache only fully fingerprinted inputs, bypass final-context caching when hooks can add files, and scope model discovery to prepared auth/model store handles.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Fingerprint hook-added files before reusing context:** `src/agents/bootstrap-files.ts:374-376`
  - body: The cache hit returns before `resolveBootstrapFilesFromRaw` runs `applyBootstrapHookOverrides`. Because `bootstrap-extra-files` reads configured globs and file contents during `agent:bootstrap`, editing a matched extra file can keep serving the old context until another raw bootstrap input changes.
  - confidence: 0.92
- **[P2] Include auth storage in the model cache boundary:** `src/agents/pi-model-discovery.ts:317-323`
  - body: `discoverModels` accepts caller-provided `authStorage`, and `createOpenClawModelRegistry` builds a registry bound to that object. The new cache key only uses agent/config/file fingerprints, so later calls with another in-memory or runtime auth store can reuse a registry tied to stale credentials.
  - confidence: 0.88
