## Summary

Current main has not changed the default: `tools.fs.workspaceOnly` remains false/unset unless explicitly enabled, and the docs still present it as optional hardening. The same remaining product/security change is already tracked by open PR #42207, which proposes the default flip plus a sensitive-path denylist, so this issue is a duplicate/superseded tracker rather than an implemented-on-main close.

## Best Possible Solution

Close this issue as superseded by PR #42207 and use that PR as the single place to decide the default-policy change, sensitive-path denylist scope, tests, and docs updates. If maintainers reject or close that PR without a replacement, the product/security decision should be reopened in a fresh focused tracker.
