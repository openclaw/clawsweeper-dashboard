## What This Changes

The issue asks OpenClaw to avoid parse errors when Qwen3.6 27B is served through llama.cpp during dashboard agent initialization.

## Best Possible Solution

Keep OpenClaw on structured OpenAI-compatible tool calls and direct llama.cpp users to fix the chat template/parser or opt the model out of tools with existing config.

No high-confidence live reproduction was established because I did not run a Qwen3.6 27B llama.cpp server. The reporter's mitm trace is still strong evidence that the current failure is a llama.cpp server-side 500 after an OpenAI-compatible tools request.
