## Summary

Current main no longer has a bespoke `system event` WebSocket path: `openclaw system event` routes through the shared gateway RPC client, and that client explicitly handles `connect.challenge` and sends the authenticated `connect` request before issuing the RPC. The changelog also records a shipped Gateway handshake fix in release `2026.3.22` for local CLI connections timing out during pre-auth handshake.
