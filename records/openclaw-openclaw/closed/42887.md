## Summary

Keep open. Current `main` still treats inbound Feishu thread context as forcing thread replies regardless of `replyInThread: "disabled"`, and `main` now has an explicit test for that behavior. This PR is therefore not implemented on `main`, but it also is not safely auto-closeable as stale/incoherent/cannot-reproduce; it represents an unresolved behavior decision in `extensions/feishu/src/bot-content.ts` and `extensions/feishu/src/reply-dispatcher.ts`.
