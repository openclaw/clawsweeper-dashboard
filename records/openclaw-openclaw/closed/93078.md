## Summary

Keep this PR open: the security issue is real, but the home-directory guard is not merge-ready because it still allows same-home parent context injection for the default workspace and can drop trusted context entirely for workspaces outside the OS home directory.

## What This Changes

The branch adds a `homedir()` boundary check to `loadProjectContextFiles()` before reading ancestor `AGENTS.md`/`CLAUDE.md` files.

## Best Possible Solution

Define and use the trusted workspace or project boundary for context-file discovery, preserve project-local context loading inside that boundary, and add regression proof for same-home parent injection plus outside-home workspaces before merge.

## Reproduction Assessment

Yes, source-reproducible. Current main walks ancestor directories until `/`, and the PR guard still permits same-home parents for the default workspace while breaking before reading `cwd` for outside-home workspaces; I did not run a filesystem repro because this review is read-only.

## Solution Assessment

No. A home-directory guard is only a partial mitigation and has a compatibility regression; the maintainable fix is a workspace or project-boundary walk with tests showing trusted context is preserved and untrusted parents are excluded.
