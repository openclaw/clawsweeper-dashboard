## Summary

This PR should close as superseded: current main and the shipped provider runtime-auth path replaced the Paperclip-specific process-env approach, while this branch reintroduces an untyped Paperclip env path that still does not reach exec.

## What This Changes

The PR adds Paperclip runtime-auth env helpers, tries to pass `paperclipRuntimeEnv` into exec tool defaults, adds regression tests, and refactors several fetch mock casts into a shared helper.

## Best Possible Solution

Keep the shipped provider runtime-auth storage path; any future credential exposure to exec should be a generic, typed, security-reviewed contract with sanitizer coverage.

## Reproduction Assessment

Yes. Static inspection is enough to reproduce the PR defects: current main lacks the Paperclip fields, exec ignores the proposed default, and the attempt tool path does not forward it into tool construction.

## Solution Assessment

No. The branch is not the best solution because it revives a Paperclip-specific env path already superseded by provider runtime auth; the safer direction is the current generic authStorage contract or a new generic exec credential contract if maintainers want that capability.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Make exec consume the runtime env default:** `src/agents/pi-tools.ts:434`
  - body: This adds `paperclipRuntimeEnv` to the defaults passed into the lazy exec tool, but `ExecToolDefaults` and `createExecTool` do not define or merge that field into the child process env. The new tests assert behavior the implementation does not provide, so Paperclip values would still be absent from exec subprocesses.
  - confidence: 0.91
- **[P1] Forward Paperclip env into attempt tool creation:** `src/agents/pi-embedded-runner/run.ts:948-949`
  - body: Passing `paperclipRuntimeEnv` and `paperclipRuntimeAuth` into `runEmbeddedAttempt` here does not deliver them to `createOpenClawCodingTools`; the current attempt tool builder still constructs `exec` from `params.execOverrides` only. Real embedded runs would still build the exec tool without these values.
  - confidence: 0.9
- **[P2] Declare the Paperclip auth params before use:** `src/agents/pi-embedded-runner/run.paperclip-auth-test-utils.ts:36-38`
  - body: `buildPaperclipRuntimeEnv` narrows `RunEmbeddedPiAgentParams` to `paperclipRuntimeAuth`, and `run.ts` assigns Paperclip fields, but the current params type does not declare those fields. The branch is not type-safe before the env path can be exercised.
  - confidence: 0.86
