## Summary

Keep open. Current main still allows the installer to warn that the npm global bin directory is missing from the parent shell PATH, resolve the CLI internally, and print a success banner before `openclaw` is actually runnable from that shell.

## What This Changes

The issue reports that the curl-bash installer on Ubuntu 26.04 says OpenClaw installed successfully, but `openclaw` is not available afterward.

## Best Possible Solution

Make installer finalization self-verifying and actionable so a successful install either leaves a directly usable CLI in the user's shell path or clearly reports the exact command or PATH export needed before onboarding.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: run the npm installer where npm's global bin directory is absent from the parent shell PATH and do not pass `--verify`; current main can warn but still finish with the success banner.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `scripts/install.sh`: preserve npm prefix repair while making the final CLI path or required PATH export explicit, and fail only when the package or CLI truly cannot be resolved.
