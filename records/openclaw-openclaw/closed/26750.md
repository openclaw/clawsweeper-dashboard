## Summary

Current main already implements the prompt-cache-stability work this issue asks for: the system prompt has a stable/dynamic cache boundary, the time section is timezone-only, volatile project context is kept below the boundary, and tool ordering is normalized/deterministic. The same behavior is present in release v2026.4.22.
