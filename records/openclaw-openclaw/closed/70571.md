## Summary

Close this PR in favor of the already-open canonical issue #59600. The PR targets the same remaining exec safeBins compound-command request, but the current branch is not a safe or clean implementation: current main still treats safeBins as executable-name plus safe-bin profile validation, docs explicitly scope safeBins as stdin-only stream filters, and review comments correctly identify that the proposed compound-prefix branch would bypass per-bin argv policy. The underlying product/design question should remain on #59600 rather than merging or keeping this dirty PR open.

## Best Possible Solution

Close PR #70571 as superseded by the open canonical issue #59600. Keep #59600 open for the product/security decision, and require any future implementation to be a narrow, clean PR that either preserves safe-bin profile validation for compound entries or documents that explicit exec allowlist entries are the supported path for broader compound commands.
