## Summary

Current main no longer uses the legacy Browser Relay Chrome extension path. OpenClaw migrated host-local signed-in browser control to the built-in Chrome MCP `existing-session` / `user` profile, so the old per-tab extension attachment flow described in the issue is obsolete on main.

## Close Comment

Closing from a Codex maintainer review: current `main` no longer uses the legacy Browser Relay Chrome extension path this report was filed against. OpenClaw removed that relay/extension flow and migrated host-local signed-in browser control to Chrome MCP `existing-session` / `user` profiles (`CHANGELOG.md:2196`, `docs/gateway/doctor.md:207`). The supported browser path is now extension-free (`docs/tools/browser.md:82`), and current tab operations go directly through Chrome MCP instead of a manual tab-attachment model (`extensions/browser/src/browser/server-context.tab-ops.ts:75`). If you still see a persistence problem on current `existing-session` / `user` in a recent release, please open a new issue with exact repro steps, Chrome version, remote-debugging state, and whether the failure is attach, tab discovery, or navigation.
Codex evidence: Reviewed against main [50e36983bb2d](https://github.com/openclaw/openclaw/commit/50e36983bb2d827adc617282c090c0a0359c6eee). Fix evidence: release [2026.3.22](https://github.com/openclaw/openclaw/releases/tag/2026.3.22).
