## What This Changes

The PR reads a local iMessage `chat.db` startup ROWID watermark, passes it as `since_rowid` to `watch.subscribe`, drops watch notifications at or below that watermark, and adds monitor tests for stale, live, default-local, unreadable-DB, and missing-`node:sqlite` cases.

## Best Possible Solution

Land the narrow startup watermark fix only after maintainer review accepts the source-matching tradeoff and either current-head redacted real behavior proof is added or an explicit privacy-driven proof override is recorded.

## Solution Assessment
