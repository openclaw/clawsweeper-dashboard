## Summary

## What This Changes

Add a fresh-session startup config key that injects named AGENTS.md sections on the first turn, parallel to postCompactionSections.

## Best Possible Solution

Publish this as an optional hook-pack or code plugin that uses session_start to identify fresh sessions and before_prompt_build to inject selected AGENTS.md sections, extending the plugin API only if that path proves insufficient.

## Reproduction Assessment

Yes, source inspection reproduces the requested gap: current startupContext loads daily memory for /new/reset first-turn runs, while AGENTS.md section extraction is wired only to post-compaction paths. This is a feature gap rather than a broken documented contract.

## Solution Assessment

No for a new core config key. The maintainable path is an optional plugin or hook pack using existing session_start and before_prompt_build extension points; a core PR should only follow if that path exposes a concrete API gap.
