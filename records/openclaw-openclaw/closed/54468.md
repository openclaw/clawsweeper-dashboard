## Summary

Current `main` and the latest shipped release already cover this bundled-plugin naming case. Bundled plugin discovery now prefers `openclaw.plugin.json:id` for single-entry plugins, the packaged dist artifact list includes each bundled plugin `package.json`, and repo tests explicitly allow the historical `kimi-coding` directory name while mapping it to canonical plugin id `kimi`.

## Evidence

- **Discovery prefers manifest id:** `deriveIdHint()` returns the manifest id for single-entry plugin packages, so a bundled plugin with `openclaw.plugin.json` id `kimi` resolves to `kimi` instead of the directory name or package suffix.
  - file: [src/plugins/discovery.ts:430](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/plugins/discovery.ts#L430)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Bundled package scan uses package metadata path:** Bundled directory scanning reads `package.json`, resolves declared `openclaw.extensions`, loads the manifest id, and passes both into `deriveIdHint()` when building candidates.
  - file: [src/plugins/discovery.ts:883](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/plugins/discovery.ts#L883)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Official packaged dist includes plugin package.json:** Bundled pack artifact generation explicitly includes `dist/extensions/<id>/package.json` when a bundled plugin has one, which avoids the old missing-`package.json` dist case described in the issue.
  - file: [scripts/lib/bundled-plugin-build-entries.mjs:148](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/scripts/lib/bundled-plugin-build-entries.mjs#L148)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Kimi directory mismatch is explicitly allowlisted:** Bundled naming guardrails keep a documented legacy exception mapping `kimi-coding` directory name to manifest id `kimi`.
  - file: [src/plugins/bundled-plugin-naming.test.ts:30](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/plugins/bundled-plugin-naming.test.ts#L30)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)

## Risks / Open Questions

- If an unofficial/custom package build strips `dist/extensions/*/package.json`, discovery still has a directory-name fallback path, so a nonstandard build could reintroduce similar noise outside the shipped OpenClaw packaging flow.
