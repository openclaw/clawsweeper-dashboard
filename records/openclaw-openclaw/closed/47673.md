## Summary

Current `main` already implements first-class Telegram topic-bound ACP/ACPX session routing, topic-aware command/delivery behavior, and ACP status/session visibility. The same evidence is present in the shipped `v2026.4.22` snapshot, so this request is already covered well enough to close.

## Evidence

- **Docs explicitly describe Telegram topic-bound ACP sessions:** The ACP docs say thread-bound ACP sessions route follow-up messages to the bound session, deliver ACP output back to the same thread, and list Telegram topics as a built-in supported thread surface; they also document persistent Telegram forum-topic ACP bindings.
  - file: [docs/tools/acp-agents.md:134](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/docs/tools/acp-agents.md#L134)
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Telegram routing resolves configured and runtime topic bindings:** Telegram conversation routing canonicalizes topic conversations, applies configured ACP bindings, and then checks runtime conversation bindings for `<chatId>:topic:<topicId>` so a topic can act as the durable session identity.
  - file: [extensions/telegram/src/conversation-route.ts:94](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/extensions/telegram/src/conversation-route.ts#L94)
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Inbound Telegram topic messages initialize/use configured ACP bindings:** Inbound Telegram message handling resolves the topic route, ensures the configured ACP binding session is ready, and uses the resulting session key for the conversation; it also reports explicit binding-unavailable failures instead of silently misrouting.
  - file: [extensions/telegram/src/bot-message-context.ts:222](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/extensions/telegram/src/bot-message-context.ts#L222)
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Native Telegram commands preserve topic targeting and bound ACP sessions:** Tests cover `/status` and other native commands in Telegram topics using configured ACP topic bindings, runtime bound topic sessions, and preserving `MessageThreadId` plus `OriginatingTo` as `telegram:<chatId>:topic:<topicId>` for topic-scoped routing.
  - file: [extensions/telegram/src/bot-native-commands.session-meta.test.ts:560](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/extensions/telegram/src/bot-native-commands.session-meta.test.ts#L560)
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Canonical ACP binding support exists for Telegram forum topics:** Configured ACP binding resolution accepts Telegram forum-topic conversation IDs in canonical and split parent/topic forms and maps them to the same ACPX-backed session key.
  - file: [src/acp/persistent-bindings.test.ts:571](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/acp/persistent-bindings.test.ts#L571)
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **ACP status/session visibility already covers thread-bound sessions:** `/acp status` prints backend/runtime session identifiers and task state, and tests cover thread-bound sessions showing both `agent session id` and `acpx session id`; `/acp sessions` also annotates listed ACP sessions with their bound thread conversation id.
  - file: [src/auto-reply/reply/commands-acp/runtime-options.ts:106](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/auto-reply/reply/commands-acp/runtime-options.ts#L106)
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Shipped-release proof:** I checked the `v2026.4.22` release snapshot and confirmed the same ACP docs and Telegram topic-binding tests were already present there, so this was shipped by that release.
  - command: `git show 00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5:docs/tools/acp-agents.md | sed -n '130,175p' ; git show 00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5:extensions/telegram/src/bot-native-commands.session-meta.test.ts | sed -n '560,715p' ; git show 00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5:src/acp/persistent-bindings.test.ts | sed -n '560,610p'`
