## Summary

Current main fixes the reported Telegram native command owner-authorization path: Telegram native commands now treat an explicit Telegram owner as a command authorizer before dispatch, and the generic command gate still preserves owner-only enforcement for `/restart`. The fix is on main after the latest release, not yet in a tagged release.

## What This Changes

Allow a correctly owner-allowlisted Telegram sender to run the native `/restart` command without being rejected by the Telegram native command authorization handoff.

## Best Possible Solution

Keep the current main implementation and ship it in the next release; if a user can still reproduce on a build containing 81551ac24b69545067141d93635a9e0d4f6a7ed0, reopen with the exact Telegram DM/group config, sender/chat IDs, and logs showing which auth gate rejected the command.
