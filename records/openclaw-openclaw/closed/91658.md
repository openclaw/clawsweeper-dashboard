## Summary

Keep open: the stale same-session `clientContext` retention defect called out in the prior review is fixed at the PR head, but the PR still needs maintainer judgment because it introduces a new core diagnostic attribution contract visible to plugins and gateway clients.

## What This Changes

Adds an optional gateway `agent.clientContext` bag, normalizes and bounds it, stores it in diagnostic session state, emits it on `message.queued` and `session.state`, and regenerates Swift protocol models.

## Best Possible Solution

Land only if maintainers accept `clientContext` as a core diagnostic contract and explicitly decide the SDK, Swift initializer, and documentation scope; otherwise redesign a narrower attribution seam.
