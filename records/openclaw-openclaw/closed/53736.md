## Summary

Keep open. Current main still does not implement `tools.exec.denyPatterns`, and this PR is the active implementation attempt, but the proposed exec-policy surface needs maintainer product/security review before merge because its regex guard is applied to the whole command string and overlaps the broader open denylist-mode design in #71097.

## What This Changes

The PR adds `tools.exec.denyPatterns` to exec config types/schema/generated config artifacts, merges and forwards global/per-agent patterns, rejects matching exec commands, and adds deny-pattern tests.

## Best Possible Solution

Keep the PR open until a maintainer chooses the exec policy contract: either land a rebased soft guardrail with explicit matching limits and segment-aware coverage, or fold the work into #71097 so deny-pattern matching, deny actions, defaults, and approval behavior are designed together.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Match deny patterns against executable segments:** `src/agents/bash-tools.exec.ts:260`
  - body: The guard tests each regex only once against `params.command.trim()`. With documented patterns like `^claude\s`, a normal generated command such as `cd repo && claude --print hi` will not match even though the denied CLI still runs. Use the existing shell segment analysis or equivalent executable-segment matching before execution.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The diff adds a security-adjacent exec guardrail, but its whole-command regex matching can be bypassed by ordinary shell chaining and may be mistaken for stronger command blocking.

Concerns:

- **[medium] Deny patterns miss chained command segments:** `src/agents/bash-tools.exec.ts:260`
  - body: Because matching is applied once to the whole trimmed command string, start-anchored deny rules are bypassed by `cd ... && denied-binary` or environment-assignment wrappers while still executing the denied command.
  - confidence: 0.84
