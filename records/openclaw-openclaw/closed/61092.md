## Summary

Keep this PR open. Current main still has the top-level Skills screen labels hardcoded in English, and there is no `skillsPage` locale block in the current i18n files. The PR is still an implementation candidate, but it needs a refresh against the current Control UI i18n pipeline and expanded locale set before it can land cleanly.

## Best Possible Solution

Keep the PR open and ask for a rebase against current main. The best path is to add a `skillsPage` block to `ui/src/i18n/locales/en.ts`, wire the top-level Skills screen labels in `ui/src/ui/views/skills.ts` through `t(...)`, regenerate all supported non-English locale bundles and `.i18n` metadata with the Control UI i18n sync pipeline, and keep the change behavior-neutral. This belongs in the core Control UI, not ClawHub.
