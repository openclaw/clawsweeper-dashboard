## Summary

Close as implemented on main: current OpenClaw already supports the requested Claude Code/Claude CLI OAuth route through the bundled Anthropic provider, and the implementation is documented and shipped in v2026.4.27.

## What This Changes

Request to add Claude Code OAuth support so users with Claude Code plans can authenticate through Claude CLI instead of being limited to ChatGPT or API-key auth.

## Best Possible Solution

Keep the shipped Anthropic Claude CLI route as the supported answer for Claude Code plan users: log Claude Code in on the gateway host, then select Anthropic Claude CLI auth in OpenClaw onboarding or `openclaw models auth login --provider anthropic --method cli --set-default`. Handle the remaining external-store refresh write-back question in the narrower #74004 review path.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The observable verification path is source/docs/tests inspection showing that current main already registers Claude CLI auth, reads Claude Code OAuth credentials, and runs the `claude` backend.

## Solution Assessment

Yes. The current implementation keeps Claude-specific auth and runtime behavior inside the bundled Anthropic provider/CLI backend, which matches the repo ownership rules and avoids adding provider-specific logic to core.
