## Summary

Keep open: this PR targets a real exec self-update bug, but it is not merge-ready because the new parser misses npm's supported `--global=true` form and the external proof is still unit-test-only.

## What This Changes

The PR adds an exec preflight parser, regression tests, and a changelog entry to reject raw global OpenClaw package-manager installs and direct self-updates to `update.run` or `openclaw update`.

## Best Possible Solution

Land a complete exec preflight guard with regression coverage for equivalent global flag forms, then verify a real exec or Gateway rejection and close the linked bug as implemented.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: the linked issue has host logs for raw npm global update followed by stale chunk failures, and current main has no exec-side guard for that command class. I did not run a live Gateway repro in this read-only review.

## Solution Assessment

No, not as written. The exec preflight boundary is the right maintainable place for the fix, but the parser needs to cover npm true-valued global flags and the PR still needs real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Handle npm boolean global flags:** `src/agents/bash-tools.exec.ts:1151-1152`
  - body: The guard only treats `-g`, `--global`, and `--location` forms as global. npm also accepts `--global=true`; a raw self-update such as `npm install --global=true openclaw@latest` still performs a global OpenClaw install, but this parser returns false, so exec would allow the bypass. Please recognize true-valued global flags and add a regression case.
  - confidence: 0.86
