## Summary

Keep this PR open: the exec-boundary guard is the right narrow fix for the linked self-update bug, but the patch misses a valid npm global form and the external-PR behavior proof is only unit-test output.

## What This Changes

The PR adds an exec preflight guard and tests to reject raw global npm/pnpm/bun/yarn installs or updates of the `openclaw` package, plus a changelog entry.

## Best Possible Solution

Keep the exec-side guard, extend it to all valid global package-manager forms while preserving allowed commands, and require after-fix real behavior proof before merge.

## Reproduction Assessment

Yes. Current main source shows the exec preflight has no raw OpenClaw package-manager guard, while the linked maintainer issue includes concrete Ubuntu journal evidence for `sudo npm install -g openclaw@latest` followed by stale runtime chunk import failures.

## Solution Assessment

No. The exec preflight is the right place to enforce this, but the proposed parser must also handle npm's separate-value `--location global` form and the PR needs real behavior proof before it is merge-ready.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Handle npm `--location global` updates:** `src/agents/bash-tools.exec.ts:1157`
  - body: `npm` accepts `--location global` as equivalent to `--location=global`, but this guard only treats the equals form as global. A self-update command like `npm update --location global openclaw@latest` will still pass through exec, leaving the stale-gateway failure mode this PR is meant to block.
