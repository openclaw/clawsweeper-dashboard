## Summary

Current main now fixes the reported Claude CLI runtime-migration path: legacy default Claude CLI intent is backfilled into model-scoped runtime policy before gateway turns route, so the dispatcher no longer has to rely on the ignored whole-agent runtime key. The fix is post-v2026.5.12 and present only on current main so far.

## What This Changes

Cold-start Claude CLI configs using legacy default runtime intent should route gateway turns through Claude CLI instead of silently falling through to another provider.

## Best Possible Solution

Keep the provider/model-scoped runtime contract, ship the merged backfill/migration fix in the next release, and keep whole-agent runtime pins documented as legacy input repaired by defaults or doctor rather than active routing policy.

## Reproduction Assessment

Yes for the pre-fix source path: the merged fix PR reproduced that migrated configs carried only the ignored whole-agent runtime while the resolver returned no runtime. On current main, the same path is backfilled to model-scoped `claude-cli`, so the reported failure should not reproduce from source inspection.

## Solution Assessment

Yes. The fix changes the Anthropic plugin migration/defaulting layer to emit the runtime-policy shape the gateway already resolves, instead of reactivating legacy whole-agent runtime selection.
