## Summary

Keep open. Current main has related plugin-loader/hook-runner hardening, but I do not see a high-confidence implementation of the specific #42644 request. The direct fix PR #46360 is closed unmerged, and current main still allows activating plugin loads to replace the global hook runner, while Telegram delivery still snapshots `hasMessageSendingHooks` before sending replies.
