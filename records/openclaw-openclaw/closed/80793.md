## What This Changes

Request adding a native `script` cron payload/target that runs a command under the Gateway without an agent turn while preserving `openclaw cron` inventory and delivery.

## Best Possible Solution

Keep https://github.com/openclaw/openclaw/issues/18160 as the canonical design tracker and fold this issue's macOS TCC/token data into that discussion before implementing an additive script/exec cron contract.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection confirms current main lacks a script/exec cron payload and still limits cron to `systemEvent` and `agentTurn`.

## Solution Assessment

Yes for cleanup: closing this duplicate and keeping design on https://github.com/openclaw/openclaw/issues/18160 is the narrowest maintainer path. The actual product solution still needs owner/secops design before implementation because it would add persisted scheduled host-command execution.
