## Summary

Current `main` already supports SecretRef-managed credentials for the requested gateway, channel, and skill surfaces, and web-search credentials are supported on the current canonical paths. The only mismatch is that the issue body uses the old scoped `tools.web.search.gemini.apiKey` shape; current schema expects `plugins.entries.google.config.webSearch.apiKey` or compatibility `tools.web.search.apiKey` instead.
