## Summary

Current main addresses the reported npm Discord crash by adding package-local compiled runtime artifacts and publish-time metadata so installed plugin packages load `dist/*.js` instead of TypeScript source. The reporter's follow-up correctly retracted the `buildPluginLoaderJitiOptions` default as the root cause; the implemented fix follows the broader packaging path.

## What This Changes

The issue reports that `@openclaw/discord@2026.5.2` npm installs crash-loop when Discord source TypeScript imports a `.js` sibling that only exists after build.

## Best Possible Solution

Keep the package-local runtime artifact pipeline and publish the next Discord plugin release from current main so npm installs use compiled JS rather than source TypeScript.

## Reproduction Assessment

Yes for the reported release path by source evidence and reporter logs: `@openclaw/discord@2026.5.2` shipped source that can hit the `native-command.args.js` import shape. I did not run a live Discord gateway reproduction in this read-only pass, and current main changes the npm package path to compiled runtime artifacts.

## Solution Assessment

Yes. Shipping package-local `dist/*.js` artifacts plus `runtimeExtensions` is the narrower maintainable fix for npm installs than flipping the shared jiti default, because it avoids source TypeScript at runtime while preserving current dev/source loader policy.
