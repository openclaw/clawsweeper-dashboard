## Summary

Keep open: this is a draft MEMBER-authored PR with the protected `maintainer` label, and the latest patch still has source-backed Crabbox docs/config contract blockers.

## What This Changes

The PR updates Crabbox CI docs and skill guidance, changes `.crabbox.yaml` from AWS `beast` to `standard` with multiple regions and capacity hints, and adds a changelog entry.

## Best Possible Solution

Align the PR with Crabbox's actual contract: use Spot for multi-region scoring or remove the on-demand multi-region claim, and either add supported capacity-hints behavior in Crabbox or drop the unsupported key and docs claims before merge.

## Reproduction Assessment

Yes for the review findings: the PR patch can be compared directly with current Crabbox source for region selection and config parsing. I did not perform a live AWS launch.

## Solution Assessment

No. Reducing default `beast` usage is well supported by the Crabbox class mapping, but the current patch also documents on-demand multi-region behavior Crabbox does not perform and claims an unsupported `capacity.hints` config key.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Fix the on-demand region guidance:** `docs/ci.md:586-587`
  - body: The new fallback command sets `CRABBOX_CAPACITY_REGIONS` while forcing `--market on-demand`, but current Crabbox only calls region scoring when `Capacity.Market == "spot"`. Operators copying this during a regional on-demand quota/capacity problem will still launch in the configured AWS region, so either use Spot for regional scoring, add on-demand regional fallback in Crabbox first, or remove the multi-region claim for on-demand.
  - confidence: 0.93
- **[P2] Drop the unsupported capacity.hints setting:** `.crabbox.yaml:8`
  - body: Current Crabbox config parsing has no `capacity.hints` field and no `CRABBOX_CAPACITY_HINTS` environment override, so this added key is ignored while the docs and PR body say hints are enabled. Remove the key/claims or land Crabbox support before relying on it from OpenClaw's repo config.
  - confidence: 0.9
