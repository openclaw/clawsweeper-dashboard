## Summary

Current main already implements the iMessage attachment-root handling needed for this multi-agent case. iMessage publishes `/Users/*/Library/Messages/Attachments` as a default inbound attachment root, the channel contract exposes those roots, and media-understanding merges the channel roots into the attachment cache before the agent run, so the media pass is no longer limited to each agent workspace.

## Best Possible Solution

Close as implemented on main. Keep the current channel-owned inbound attachment root contract and media-understanding root merge; if a user still reproduces this on a build containing commit 92016b82, they should open a fresh issue with current version, `includeAttachments`/`attachmentRoots` config, logs, and affected agent/channel details.
