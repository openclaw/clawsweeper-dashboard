## Summary

Current main and release v2026.5.7 already cover the reported failure: agent-created cron reminders infer the announce delivery target from the active chat context before calling `cron.add`, so the saved job gets the live channel target instead of relying on a lowercased session key.

## What This Changes

Cron job creation should preserve or infer `delivery.to` for `openclaw-weixin` scheduled announcements created from an active chat.

## Best Possible Solution

Keep the shipped channel-agnostic active-chat delivery inference; users on v2026.4.26 should upgrade to v2026.5.7 or newer.

## Reproduction Assessment

No current-main failing reproduction was established. The reported old-version path is clear, but current source and regression tests show missing announce targets are filled from the active chat context before `cron.add`.

## Solution Assessment

Yes. The current solution is the maintainable owner-boundary fix because WeChat is an external plugin and core handles this generically through `currentDeliveryContext`, not Weixin-specific logic.
