## Summary

Keep open for maintainer review. Current main still has the Slack preview recovery mismatch: edit updates send a truncated blocks-only fallback, while readback verification expects the untruncated fallback, and this PR narrows the fix by sharing the exact edit text payload logic between both paths.

## What This Changes

This PR adds a shared Slack edit text payload helper, uses it for Slack chat.update and preview edit readback expectations, and adds regression coverage for long blocks-only fallback readback.

## Best Possible Solution

Land the shared Slack edit text payload helper with the added preview-finalize regression test, then run the Slack changed gate so future edit payload changes cannot drift from recovery readback expectations.

## Reproduction Assessment

Yes. On current main, a long blocks-only edit can produce an 8000-character truncated payload while `buildExpectedSlackEditText` expects the longer raw fallback, making mocked readback verification return false even when it contains the exact text Slack was sent.

## Solution Assessment

Yes. Sharing one helper for the Slack edit `text` payload is the narrowest maintainable fix because it aligns the write path and the recovery verifier instead of duplicating truncation rules in preview-finalize.
