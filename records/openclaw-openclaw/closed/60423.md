## Summary

Current main already implements the requested media-understanding error boundary. `getReplyFromConfig` runs media understanding through `applyMediaUnderstandingIfNeeded`, catches failures from loading or running the media-understanding runtime, clears the cached runtime import, logs, and continues through the normal reply pipeline with raw content. A regression test now mocks a media-understanding failure and verifies the reply still returns successfully. The behavior shipped in v2026.4.26 and is also reflected in the media-understanding docs as best-effort.

## Best Possible Solution

Keep the shipped v2026.4.26 boundary and regression test. If maintainers want different default log visibility for media-understanding failures, handle that under the separate media logging/reporting thread rather than keeping this crash-boundary issue open.
