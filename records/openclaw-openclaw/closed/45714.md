## Summary

Current `main` already contains the Telegram polling liveness watchdog, forced restart path, diagnostic logging, and regression coverage that address the reported half-alive polling stall; those fixes are also present in release `v2026.4.22`.

## Evidence

- **Polling watchdog restarts stalled runner:** `TelegramPollingSession` wraps `getUpdates`, tracks liveness, starts a 30s watchdog, logs `Polling stall detected`, stops the runner/bot, and restarts the polling cycle with backoff.
  - file: [extensions/telegram/src/polling-session.ts:246](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/extensions/telegram/src/polling-session.ts#L246)
