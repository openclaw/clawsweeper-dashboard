## Summary

The report’s premise does not match current `main`: port `18791` is the browser control service port, which the browser plugin starts during gateway startup, while an idle managed browser process is tracked separately and already reported as a non-error/info state (`"Browser is launchable but not running"` / `"browser not running"`).

## Evidence

- **Browser control service starts on plugin startup:** `startBrowserControlServiceFromConfig()` loads config, resolves the browser control port, creates the browser runtime state, and logs that the browser control service is ready when the bundled browser plugin is enabled.
  - file: [extensions/browser/src/control-service.ts:24](https://github.com/openclaw/openclaw/blob/c7f18a6b9d76a31f4daed5096c0a25b3794c76ac/extensions/browser/src/control-service.ts#L24)
  - command: `nl -ba extensions/browser/src/control-service.ts | sed -n '24,56p'`
  - sha: [c7f18a6b9d76](https://github.com/openclaw/openclaw/commit/c7f18a6b9d76a31f4daed5096c0a25b3794c76ac)
- **Gateway startup runs plugin services:** Gateway post-attach startup calls `startPluginServices(...)`, and the plugin service runner eagerly calls each service's `start(...)`, so the browser control service is not deferred until first browser use.
  - file: [src/gateway/server-startup-post-attach.ts:293](https://github.com/openclaw/openclaw/blob/c7f18a6b9d76a31f4daed5096c0a25b3794c76ac/src/gateway/server-startup-post-attach.ts#L293)
  - command: `nl -ba src/gateway/server-startup-post-attach.ts | sed -n '293,304p'`
  - sha: [c7f18a6b9d76](https://github.com/openclaw/openclaw/commit/c7f18a6b9d76a31f4daed5096c0a25b3794c76ac)
- **Default ports separate control service from browser CDP:** Current config code and docs define `18791` as the browser control service port and the managed browser CDP range starting at `18800`, so `18791` is not the on-demand browser-process port described in the issue body.
  - file: [src/config/port-defaults.ts:16](https://github.com/openclaw/openclaw/blob/c7f18a6b9d76a31f4daed5096c0a25b3794c76ac/src/config/port-defaults.ts#L16)
  - command: `nl -ba src/config/port-defaults.ts | sed -n '15,37p'`
  - sha: [c7f18a6b9d76](https://github.com/openclaw/openclaw/commit/c7f18a6b9d76a31f4daed5096c0a25b3794c76ac)
- **Idle browser availability is already treated as non-error:** Browser diagnostics mark missing CDP WebSocket as `info` with summary `Browser is launchable but not running` when the browser is not running, and the tabs route returns a plain `browser not running` response instead of treating that state as a control-service failure.
  - file: [extensions/browser/src/browser/doctor.ts:117](https://github.com/openclaw/openclaw/blob/c7f18a6b9d76a31f4daed5096c0a25b3794c76ac/extensions/browser/src/browser/doctor.ts#L117)
