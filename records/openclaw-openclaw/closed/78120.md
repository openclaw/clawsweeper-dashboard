## Summary

This matches the documented group/channel visible-reply policy rather than a WhatsApp dispatch bug: group turns default to message-tool-only visible output, while DMs still auto-route normal final replies.

## What This Changes

The issue asks OpenClaw to dispatch normal Claude CLI assistant text back to a WhatsApp group when the group is configured to process unmentioned messages.

## Best Possible Solution

Keep the current group safety default and have operators choose either a tool-calling model/path for visible group output or `messages.groupChat.visibleReplies: "automatic"` when they want normal final answers auto-posted to rooms.

## Reproduction Assessment

Yes. Source inspection shows a group chat defaults to `message_tool_only`, which suppresses ordinary final assistant text before WhatsApp delivery; DMs default to automatic delivery, matching the side-by-side report.

## Solution Assessment

Yes. The narrow solution for this setup is configuration or message-tool use, not changing WhatsApp dispatch; changing the default group-visible-reply policy would be a product decision.
