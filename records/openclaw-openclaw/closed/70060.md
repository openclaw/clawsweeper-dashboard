## Summary

The requested WeChat voice-bubble behavior is owned by the external `openclaw-weixin` plugin, not OpenClaw core. Current `main` already provides the generic `asVoice` / `audioAsVoice` plumbing for channel plugins and TTS media delivery, so this core issue is out of scope for `openclaw/openclaw`.

## Evidence

- **WeChat is documented as an external plugin, with WeChat-specific media/runtime owned outside core:** The WeChat docs say OpenClaw connects through Tencent's external `@tencent-weixin/openclaw-weixin` plugin, and explicitly state that WeChat login, media upload/download, context tokens, and monitoring are owned by the external plugin rather than core.
  - file: [docs/channels/wechat.md:10](https://github.com/openclaw/openclaw/blob/24e9924d6abd93560fb7bf721909257de533e57a/docs/channels/wechat.md#L10)

  - sha: [24e9924d6abd](https://github.com/openclaw/openclaw/commit/24e9924d6abd93560fb7bf721909257de533e57a)
- **Project vision prefers optional channel capability as plugins / ClawHub, not core:** VISION says core should stay lean, optional capability should usually ship as plugins, plugin promotion belongs in ClawHub, and the bar for adding optional plugins to core is intentionally high.
  - file: [VISION.md:52](https://github.com/openclaw/openclaw/blob/24e9924d6abd93560fb7bf721909257de533e57a/VISION.md#L52)
