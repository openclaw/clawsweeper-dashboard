## Summary

Keep open: the report is fresh, externally filed, and current main still has a source-visible Discord draft-preview path that can bypass the normal user-facing metadata sanitizer. The agent-prompt part needs owner/security triage because current code intentionally uses external-content wrappers as model-facing boundaries.

## What This Changes

Strip internal EXTERNAL_UNTRUSTED_CONTENT envelope markers from Discord-visible chat output and avoid duplicating them into agent-visible conversational input.

## Best Possible Solution

Keep the issue open and split the fix into a stage-level Discord repro plus a narrow user-visible sanitization patch, while preserving intentional model-facing untrusted-content boundaries unless owners approve a prompt-contract change.

## Reproduction Assessment

No, not as a live current-main reproduction. Source inspection gives a medium-confidence path: Discord draft partial previews can render text after only directive/reasoning stripping, while normal final delivery uses the broader user-facing sanitizer.

## Solution Assessment

Unclear. Sanitizing Discord draft-visible text is a narrow likely fix, but stripping wrappers from agent prompt assembly is not clearly best because those markers are part of the external-content security contract.
