## Summary

Current main already addresses this well enough: OpenClaw has a dedicated `apply_patch` tool for brittle multi-file/multi-hunk edits, enables it by default for OpenAI/OpenAI Codex models, and wraps `edit` with recovery that adds current-file-content hints on exact-match failures and treats post-write throws as success when the edit actually applied.

## Risks / Open Questions

- `edit` itself is still exact-match based; the closure relies on the combination of edit-recovery hardening plus the separate `apply_patch` tool rather than new fuzzy/anchor modes inside `edit`.
- `apply_patch` default enablement is limited to OpenAI/OpenAI Codex models, so non-OpenAI runtimes still depend primarily on the hardened exact-match `edit` path.

## Close Comment

Closing from a Codex review: current `main` already covers this well enough.
