## Summary

Current `main` contains the bundled-plugin Jiti loader fix and no longer uses the reported channel-metadata discovery path that re-entered `bundled-plugin-metadata` during chat metadata initialization.

## Evidence

- **Bundled plugin loads now scope Jiti to the target module path:** `createPluginJitiLoader` passes `jitiFilename: modulePath` when loading bundled plugins, instead of anchoring Jiti to the central loader module.
