## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Register `/activate` in command detection:** `src/auto-reply/group-activation.ts:31`
  - body: Changing only `parseActivationCommand()` leaves `/activate mention` outside the shared registry used by `hasControlCommand()`. WhatsApp group gating uses that detector for owner mention bypass, so the alias can still be skipped before the activation handler runs unless the registry and gating coverage are updated too.
  - confidence: 0.91
- **[P2] Update reload suppression for lowercase finals:** `ui/src/ui/controllers/chat.ts:18`
  - body: The controller now drops lowercase `no_reply`, but `ui/src/ui/chat-event-reload.ts` still treats the same final payload as renderable. A deferred session-message reload can therefore be cleared without replaying history, so the reload helper needs the same case-insensitive silent-token contract.
  - confidence: 0.86
- **[P2] Remove the unproven bare-N terminal suppression:** `ui/src/ui/controllers/chat.ts:77`
  - body: `isSilentReplyTerminalStream()` treats any terminal fallback text equal to `N` as silent even when the final or abort event has no NO_REPLY payload proving it was a sentinel fragment. That can make a legitimate one-character assistant reply disappear after streaming.
  - confidence: 0.78
