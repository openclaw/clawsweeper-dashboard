## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Constrain switch targets to permitted agents:** `extensions/discord/src/voice/manager.ts:543-545`
  - body: `switchAgent` only checks that the requested id exists in `cfg.agents.list`, so any user who can run `/vc` in the current channel can move the live session to any configured agent. Current Discord routing supports peer/guild/role-bound agents, and this bypasses those route boundaries plus any different agent tool policy; re-resolve or explicitly authorize the requested target before mutating the route.
  - confidence: 0.88
- **[P2] Preserve configured voice session targets:** `extensions/discord/src/voice/manager.ts:547-552`
  - body: This constructs the switched session from `entry.sessionChannelId`, but current `main` supports `channels.discord.voice.agentSession.mode="target"`, where the voice media session and agent supervisor session intentionally differ. A rebased switch must preserve that target route instead of silently moving agent turns back to the voice channel session.
  - confidence: 0.82
- **[P2] Snapshot routes in the current segment pipeline:** `extensions/discord/src/voice/manager.ts:641-647`
  - body: The route snapshot is added to the old inline segment path, while current `main` now runs STT/TTS through `processDiscordVoiceSegment` and `runDiscordVoiceAgentTurn`, which read `entry.route` during execution. After rebasing, the snapshot needs to be carried through those helper params, otherwise queued pre-switch utterances can still run under the post-switch route.
  - confidence: 0.79
