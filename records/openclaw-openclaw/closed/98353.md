## What This Changes

The PR changes iOS RootTabs so normal launches fall back to Chat, destination-only launches derive their owning tab, and the iPhone tab bar/test guards move to Chat-first order.

## Best Possible Solution

Have an iOS/product owner confirm Chat-first launch, or fold the default-tab decision into the adjacent Control redesign before merging.

## Reproduction Assessment

Yes from source inspection: current main and v2026.6.11 return .control on ordinary iOS launch without --openclaw-initial-tab, while the PR changes that fallback to Chat. I did not run an iOS simulator in this read-only review, but the PR body reports after-fix simulator/runtime proof.

## Solution Assessment

Yes technically, subject to product approval: RootTabs owns initial tab selection, and deriving destination-only launches through SidebarDestination.appTab avoids a second mapping. The unresolved question is whether Chat-first should replace Control-first as the iOS product default.
