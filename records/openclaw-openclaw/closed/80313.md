## Summary

Current main no longer routes the gateway startup models.json warmup through the legacy compatibility agent path, so the reported agents/main write target from v2026.5.7 is fixed on main but not yet shipped in a tagged release.

## What This Changes

Gateway startup should stop recreating agents/main when agents.list has a non-main configured default agent.

## Best Possible Solution

Keep the cfg-scoped default-agent write path on main and ship it in the next release; reopen with fresh current-main boot evidence only if another startup writer still recreates agents/main.

## Reproduction Assessment

No for current main: the release-era compat-path write is removed from the startup models.json path, and the non-main default-agent case is covered by source and test inspection. The v2026.5.7 reproduction remains credible because that release still used resolveOpenClawAgentDir().

## Solution Assessment

Yes: moving the startup write target to resolveDefaultAgentDir(cfg) is the narrow ownership-correct fix because it preserves single-agent main compatibility while honoring a configured non-main default agent.
