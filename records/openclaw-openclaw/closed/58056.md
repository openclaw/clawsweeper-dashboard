## Summary

Current `main` already sends the required GitHub Copilot IDE headers on both runtime inference requests and the GitHub token-exchange path, and the changelog shows this shipped in `v2026.4.5`.

## Evidence

- **Inference requests now inject Copilot IDE headers:** `src/agents/openai-transport-stream.ts` merges `buildCopilotDynamicHeaders(...)` for `github-copilot` OpenAI-family requests, and `src/agents/anthropic-transport-stream.ts` does the same for Copilot Claude requests. `src/agents/copilot-dynamic-headers.ts` includes `Editor-Version` and `User-Agent` in that shared header builder.
  - file: [src/agents/openai-transport-stream.ts:627](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/agents/openai-transport-stream.ts#L627)

- **GitHub token exchange also sends IDE headers:** `src/agents/github-copilot-token.ts` now calls `buildCopilotIdeHeaders({ includeApiVersion: true })` when exchanging the GitHub token, which covers the auth path mentioned in the report.
