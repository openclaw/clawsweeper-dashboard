## What This Changes

Requests that OpenAI model calls to api.openai.com stop being falsely blocked by SSRF protection behind SOCKS/fake-IP proxy setups.

## Best Possible Solution

Keep the current main scoped provider-host fake-IP SSRF policy and ship it in the next release; browser.ssrfPolicy does not need to control model provider calls.

## Reproduction Assessment

No high-confidence failing current-main reproduction is available. Current source and focused tests show api.openai.com provider fetches receive the host-scoped fake-IP exemptions that address the reported guard block.

## Solution Assessment

Yes. The current main approach is the narrowest maintainable fix because it scopes fake-IP exemptions to the configured model provider baseUrl host instead of broadening browser SSRF allowlists or private-network access.
