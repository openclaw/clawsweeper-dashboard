## Summary

Keep open: current main still leaves Control UI `/new` dashboard sessions without an automatic friendly label, and the local `openclaw sessions` projection still cannot show stored labels. The closed contributor PR proposed the right narrow area but was not merged, while the older labels/nicknames issue tracks broader manual naming work rather than fully superseding this WebUI auto-label gap.

## What This Changes

Request automatic friendly labels for dashboard sessions created by Control UI `/new`, plus session-list output that can show those labels.

## Best Possible Solution

Add a narrow default label for generated dashboard sessions, surface stored labels in local CLI JSON/text output while preserving raw keys, and leave first-message-derived titles or manual label commands to the broader labels issue.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: Control UI `/new` sends no label, Gateway stores only explicit labels for generated dashboard keys, and the local CLI projection/table omits label fields.

## Solution Assessment

Yes. Reusing the existing `sessions.create`/`sessions.patch` label contract is the narrow maintainable direction; generated titles from first messages should stay out of this initial fix.
