## Summary

Current main already prevents stale successful-delivery state from surviving a failed Discord cron send. Discord API non-OK responses throw, the isolated cron delivery path treats the send as a failure instead of marking it delivered, and cron state is overwritten from the current run result each time. The same logic is present in the shipped `v2026.4.22` release.
