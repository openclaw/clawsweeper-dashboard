The issue reports concurrent `openclaw agents add` commands racing on config writes and losing agent entries or surfacing config conflict errors.

## Best Possible Solution

Keep the centralized config mutation/retry implementation on main; any future interactive wizard conflict UX should be tracked separately with a concrete current-main repro.
