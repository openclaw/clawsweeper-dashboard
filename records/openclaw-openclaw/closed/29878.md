## Summary

Current `main` already treats Google Chat as a markdown-capable channel and routes outbound replies through the markdown path, so the requested feature is implemented on `main` even though it does not appear to have shipped in the latest release yet.

## Risks / Open Questions

- This appears implemented on `main` but not yet in the latest tagged release, so users on released builds may still not see it until a newer release ships.
- The implementation relies on Google Chat's native rendering of message `text`; if Google Chat itself does not render a specific markdown construct, that would be a product limitation rather than a missing OpenClaw markdown-mode toggle.
