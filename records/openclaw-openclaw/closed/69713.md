## Summary

Current `main` already contains a Teams-specific fix that suppresses native typing indicators while a DM streaming card is actively updating, with targeted regression tests covering the reported double-bubble artifact.

## Evidence

- **Typing gate in Teams reply dispatcher:** `extensions/msteams/src/reply-dispatcher.ts` now checks `streamActiveRef.current()` before sending a plain `typing` activity, specifically to avoid overlaying `...` on top of the live streaming card in supported Teams conversations.
