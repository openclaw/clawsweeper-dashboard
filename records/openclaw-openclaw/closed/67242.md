## Summary

Keep open. Current main has not implemented the neutral `openclaw/plugin-sdk/embeddings` subpath: package exports, SDK entrypoint lists, docs, and bundled imports still use `memory-core-host-engine-embeddings`. This is a plausible core Plugin SDK boundary change rather than a ClawHub-only plugin, but the PR should be rebased/reviewed against current main because concrete OpenAI/Gemini/Voyage embedding factories now live in provider plugins.

## Best Possible Solution

Keep this PR open for normal maintainer review. The best path is to rebase it onto current main and, if maintainers want the neutral stable import, add a narrow `openclaw/plugin-sdk/embeddings` facade backed by the current generic memory-host embedding contract, update package exports/entrypoint metadata/docs/API checks, and migrate bundled imports without moving provider-specific implementation out of provider plugins unless that is an explicit API decision.
