## Summary

This is a valid current-main Codex projector bug, but it should not be closed while the linked implementation PR is still open and references this issue with closing syntax.

## What This Changes

The issue requests that Codex app-server command results preserve streamed `item/commandExecution/outputDelta` text as a fallback when final `commandExecution.aggregatedOutput` is empty.

## Best Possible Solution

Land or revise https://github.com/openclaw/openclaw/pull/83200 so per-command streamed output is buffered and used only as the fallback for transcript, trajectory, and final tool-result surfaces.

## Reproduction Assessment

Yes from source inspection: emit `item/commandExecution/outputDelta` for a command, then complete the turn with the same `commandExecution` item and `aggregatedOutput: null`; current main has no retained delta text and final output helpers return no command output.

## Solution Assessment

Yes, buffering streamed text per command item and using it only when `aggregatedOutput` is missing is the narrow maintainable fix; the existing open PR is the right place to review that implementation and proof.
