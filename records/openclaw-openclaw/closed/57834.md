## Summary

Current `main` already contains session-reset safeguards that match this report: system-event traffic is excluded from session rebinding/reset logic, and CLI-backed session reuse now ignores volatile inbound metadata so a Kimi-backed session is not invalidated on each user turn. The exact reset text from the report is still the core bare-reset prompt, but the current code paths now prevent it from being triggered by those transient conditions.
