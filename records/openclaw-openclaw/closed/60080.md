## Summary

Current `main` plus the reporter’s own follow-up comments indicate this item is obsolete as filed: heartbeat routing to the agent main session is documented behavior, `cron` jobs with `sessionTarget: "main"` are intentionally main-session `systemEvent` jobs, and current heartbeat resolution now hard-falls back to the main session when a subagent session key is forced.

## Evidence

- **Reporter follow-up resolved the report as configuration, not a core bug:** In the issue’s final April 3, 2026 comment, the reporter states that stock OpenClaw already supports `heartbeat.session: "main"` plus explicit `target`/`to`, restored their local dist changes, and concluded `"无需修改 OpenClaw 核心代码"` / no core code changes were needed.
- **Heartbeat docs say main session is the default and isolated mode only changes transcript isolation:** The current heartbeat docs define `session` with `main` as the default, define `target: none` as the default delivery mode, and note that `isolatedSession` uses a fresh session history while delivery still uses the main-session context.
  - file: [docs/gateway/heartbeat.md:227](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/docs/gateway/heartbeat.md#L227)
  - command: `nl -ba docs/gateway/heartbeat.md | sed -n '220,240p'`
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Current heartbeat resolver explicitly falls back to the main session:** `resolveHeartbeatSession()` returns `mainSessionKey` when a forced session key is a subagent session, when `heartbeat.session` is empty, and when it is `main`/`global`. This matches the intended 'main session by default' behavior and blocks accidental routing into subagent sessions.
  - file: [src/infra/heartbeat-runner.ts:273](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/infra/heartbeat-runner.ts#L273)
  - command: `nl -ba src/infra/heartbeat-runner.ts | sed -n '245,372p'`
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Regression tests cover the main-session fallback:** Tests assert that a forced subagent session key and a subagent key supplied via `heartbeat.session` both fall back to the main session instead of using the subagent session.
  - file: [src/infra/heartbeat-runner.returns-default-unset.test.ts:975](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/infra/heartbeat-runner.returns-default-unset.test.ts#L975)
  - command: `nl -ba src/infra/heartbeat-runner.returns-default-unset.test.ts | sed -n '975,1065p'`
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Cron `sessionTarget: main` is an explicit product contract, not an accidental fallback:** The cron tool help and cron job validation both define `sessionTarget: "main"` as 'run in the main session' for `payload.kind="systemEvent"`, and reject non-main-session payload shapes for that mode.
  - file: [src/agents/tools/cron-tool.ts:445](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/agents/tools/cron-tool.ts#L445)
