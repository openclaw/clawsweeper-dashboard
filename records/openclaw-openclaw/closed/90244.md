## Summary

This should close because the reported npm/package warning comes from configuring Skill Workshop as a plugin, while current source, docs, schema, and the 2026.6.1 package all treat it as a core skills workflow configured under `skills.workshop`.

## What This Changes

The issue asks OpenClaw to either ship Skill Workshop as a bundled plugin in npm or stop exposing references that make `plugins.entries.skill-workshop` look valid.

## Best Possible Solution

Keep Skill Workshop documented and configured as a core skills workflow, and have affected users remove `plugins.entries.skill-workshop` in favor of `skills.workshop` settings and `openclaw skills workshop` commands.

## Reproduction Assessment

Yes for the warning path: source inspection shows `plugins.entries.skill-workshop` is a stale plugin reference because Skill Workshop is not a plugin id. I did not find a high-confidence package bug because the shipped docs/config path is `skills.workshop`.

## Solution Assessment

No, adding a `skill-workshop` bundled plugin is not the best fix for this report. The maintainable path is to remove the stale plugin config and use the documented core Skill Workshop surfaces.
