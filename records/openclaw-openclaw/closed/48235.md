  - confidence: 0.9
- **[P2] Carry reset hints into queued runs:** `src/auto-reply/reply/body.ts:48`
  - body: `applySessionHints` prepends and clears the reset hint before queue delivery is built, but followup/collect/steer paths send `queuedBody`, which is built from the unprefixed body. A queued first turn after reset can clear the hint without sending it to the model.
  - confidence: 0.88
- **[P2] Delimit prior assistant text before injecting it:** `src/auto-reply/reply/body.ts:47`
  - body: `resetHintMsg` comes from the prior transcript and can include newlines or instruction-shaped text reflected from earlier turns. Embedding it directly in the wrapper makes prior output hard to distinguish from reset instructions.
  - confidence: 0.84
- **[P3] Clear the in-memory hint without store context:** `src/auto-reply/reply/body.ts:49`
  - body: The hint fires whenever `sessionEntry` carries `lastAssistantMessageBeforeReset`, but clearing is gated on `sessionStore` and `sessionKey`. A caller that supplies only the entry can inject the same one-shot hint repeatedly.
  - confidence: 0.72
