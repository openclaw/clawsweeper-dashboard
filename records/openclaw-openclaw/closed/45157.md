## Summary

Current `main` no longer uses the TDZ-prone `ANTHROPIC_MODEL_ALIASES` path during config loading. Anthropic model alias normalization is inline and config pruning defaults now go through the lightweight bundled provider-policy artifact path, with regression coverage proving Anthropic defaults apply without loading the full provider plugin. The same runtime implementation is already present in the latest verified release tag, `v2026.4.22`.
