## Summary

Keep open: current main still suppresses verbose/tool progress for ordinary group sessions and has no schema-backed trusted-room config policy. The linked pull requests are open narrower implementation candidates, not merged fixes for the broader private-room configuration request.

## What This Changes

The issue requests an opt-in policy allowing private messaging group sessions to surface verbose/tool progress while preserving safe default suppression in ordinary groups.

## Best Possible Solution

Add a provider-agnostic, fail-closed group verbose-progress policy with typed global/per-channel/per-room config, docs privacy warnings, and regression coverage for ordinary groups, trusted private rooms, forum topics, and Slack non-direct suppression.

## Reproduction Assessment

Yes, at source level. Signal group messages enter dispatch as ChatType group, current dispatch gates verbose/tool progress off for ordinary groups, and existing tests assert text tool summaries stay suppressed there.

## Solution Assessment

Unclear pending maintainer product review. Explicit opt-in configuration is the right direction, but maintainers still need to choose naming, precedence, per-room semantics, and how it relates to https://github.com/openclaw/openclaw/pull/44027 and https://github.com/openclaw/openclaw/pull/64182.
