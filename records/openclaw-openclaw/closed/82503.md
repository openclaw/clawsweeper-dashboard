## Summary

Close as not actionable in the OpenClaw source repo: the report names the third-party `openclaw-channel-dmwork` plugin and describes DMWork plugin routing state, while current main does not bundle DMWork and already exposes generic channel routing, session, and turn-dispatch helpers for plugins.

## What This Changes

Request recovery and warning logs for DMWork group messages that arrive after a group session is `done` so they start a new session instead of being silently dropped.

## Best Possible Solution

Fix this in `openclaw-channel-dmwork` by making its inbound group routing recover or recreate plugin session state and log any drop reason; open an OpenClaw SDK issue only if that work proves a missing core API.

## Reproduction Assessment

No. The target repo does not contain the DMWork plugin or a DMWork runtime fixture, and the reported logs point to plugin-level MENTION/HISTORY/deliver-buffer stages rather than a current-main core path.
