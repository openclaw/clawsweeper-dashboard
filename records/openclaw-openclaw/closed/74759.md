## Summary

Keep open: current main and the latest release support configured SecretRefs through env/file/exec providers, including exec examples for 1Password and Vault, but they still do not provide the requested agent-callable `secret.get`/`secret.has`/`secret.list` broker or the needed redaction and trusted-session contracts.

## What This Changes

Request to add a first-class secret broker plugin with agent-callable `secret.get`, `secret.has`, and `secret.list` over Bitwarden, Keychain, 1Password, and Vault backends.

## Best Possible Solution

Define a trusted secret capability contract with metadata-only `has`/`list`, guarded `get`, redacted or non-persistent results, and host-enforced session gating before choosing a bundled plugin or plugin API extension.

## Reproduction Assessment

Not applicable: this is a feature and security-design request, not a broken existing behavior report. Current-main source inspection confirms the requested agent-callable `secret.*` broker surface is absent.

## Solution Assessment

Unclear. The proposed API shape is plausible, but implementing it as an ordinary external plugin is not currently safe because external plugins cannot register pre-model result redaction middleware and no `owner_dm`-style tool scope contract exists.

## Security Review

Status: needs_attention

Summary: This issue is security-sensitive because it would intentionally expose live credential material through agent-callable tools.

  - confidence: 0.94
- **[high] Gate broker tools by trusted session scope:** `VISION.md:43`
  - body: The requested group-chat and untrusted-participant guard needs a host-enforced tool availability contract before the tool is offered or executed; general redaction is not enough as the primary boundary.
  - confidence: 0.9
- **[medium] Do not treat exec SecretRefs as an arbitrary broker:** `docs/gateway/secrets.md:19`
  - body: Current SecretRefs resolve configured credential targets into a runtime snapshot and docs explicitly keep resolution off hot request paths, so a lazy agent-callable broker would be a distinct security and runtime contract.
  - confidence: 0.86
