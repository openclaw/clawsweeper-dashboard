## Summary

Current main already fixes the reported npm pruning path by maintaining a managed package.json in the shared plugin npm root before each npm install, and doctor missing-plugin repair uses that same installer.

## What This Changes

The issue reports that sequential `openclaw plugins install` and `openclaw doctor --fix` runs in v2026.5.2 prune other `@openclaw/*` plugin packages from the shared plugin npm root.

## Best Possible Solution

Keep the managed npm root manifest fix on main and let the next release carry it to users affected by v2026.5.2.

## Reproduction Assessment

Yes from source inspection: the reported v2026.5.2 behavior follows from npm installs in a shared root without a dependency manifest, and current main now creates and updates that manifest before install. I did not run a live install because this review is read-only.

## Solution Assessment

Yes. Maintaining the shared npm root package.json is the narrowest maintainable fix for npm pruning sibling plugin packages, and it also covers the doctor repair path because that path calls the same installer.
