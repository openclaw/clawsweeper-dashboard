## Summary

Keep open: this addresses a real Codex app-server MCP gap, but the latest diff still imports a private SDK subpath that is not emitted in package builds, and the supplied proof stops at helper/wiring tests rather than a live app-server run.

## What This Changes

The PR projects configured OpenClaw `mcp.servers` into Codex app-server `thread/start` and `thread/resume` config as `mcp_servers`, with a private SDK facade, alias allowlist, tests, generated baseline hash, and changelog entry.

## Best Possible Solution

Keep the projection approach, but route the helper through an emitted private SDK artifact or existing built boundary, add package-contract coverage for that path, and provide redacted live app-server proof that user MCP tools are visible.

## Reproduction Assessment

Yes, source-reproducible: current main has Codex CLI `mcp_servers` projection but the app-server path only sends native hook config plus curated plugin app config. I did not run a live app-server session in this read-only review.

## Solution Assessment

No, not yet. Projecting `mcp_servers` is the narrow maintainable fix, but the helper must be available from packaged dist and the PR needs live app-server proof before merge.
