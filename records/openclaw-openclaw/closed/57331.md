## Summary

Current `main` already routes both `/v1/responses` and `/v1/models` through the OpenAI-compatible HTTP scope resolver that restores default operator scopes when `x-openclaw-scopes` is absent, which addresses the reported 403 regression. The fix is documented in the `2026.4.5` changelog entry for Gateway/OpenAI HTTP scope restoration.

## Evidence

- **Compat scope fallback on current main:** `resolveTrustedHttpOperatorScopes()` now returns `CLI_DEFAULT_OPERATOR_SCOPES` when `x-openclaw-scopes` is absent, and `resolveOpenAiCompatibleHttpOperatorScopes()` restores the same defaults for shared-secret compat auth.
