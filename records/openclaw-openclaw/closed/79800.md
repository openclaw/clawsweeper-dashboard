## Summary

The report matches OpenClaw's documented group/channel visible-reply policy, not a WeCom core delivery bug: group final text is private by default unless the agent sends through the message tool or `messages.groupChat.visibleReplies` is set to `automatic`.

## What This Changes

The issue asks OpenClaw to deliver successful WeCom group-chat model final text back to WeChat the same way direct messages are delivered.

## Best Possible Solution

Keep the shared group/channel tool-only default, and use the existing `messages.groupChat.visibleReplies: "automatic"` opt-in or the WeCom message-tool path for visible group replies.

## Reproduction Assessment

No high-confidence bug reproduction is present. The supplied logs are source-reproducible as expected default behavior: group final text is private in message-tool-only mode unless sent through the message tool or `messages.groupChat.visibleReplies` is `automatic`.

## Solution Assessment

Yes. The existing config flag is the narrowest maintainable way to opt into automatic group replies without special-casing WeCom or bypassing the shared group/channel policy.
