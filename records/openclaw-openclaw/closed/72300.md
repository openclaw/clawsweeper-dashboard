## Summary

Current main already covers the Windows Node 25 drive-letter ESM loader failure reported for both browser-control and Telegram. The report and the follow-up confirmation match the same raw `C:` path root cause already closed in related #72221/#72198, while current main disables native Jiti loading on Windows, preserves non-native Jiti behavior when `tryNative` is false, and normalizes Windows absolute plugin import specifiers to `file://` URLs before module loading.

## Best Possible Solution

Close this as fixed on current main. Keep the shared Windows-safe plugin/channel loader behavior, and ask for a fresh issue only if the next release still reproduces with full Windows logs, OpenClaw version, Node version, and affected plugin/channel details.
