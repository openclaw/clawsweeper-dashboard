## Summary

Current `main` already contains the dashboard/TUI final-message reconciliation logic that matches this report: the dashboard now reloads persisted history on final chat events, and the TUI preserves streamed text when final payloads drop boundary text blocks. The fix is present on `main` in commit `9ab51bb66e0d75893de49863b8d1bb7fc9442dcd`, but that commit is newer than the latest tagged release `v2026.4.23`, so `fixedRelease` is unknown/null.
