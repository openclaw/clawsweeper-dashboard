## Summary

Current `main` already ships a more complete fix for the webhook-404 / stale-plugin-registry problem. The shipped solution pins startup HTTP-route and channel registries, resolves plugin auth/dispatch against the live HTTP-route registry, and includes regression tests for `/bluebubbles-webhook` surviving later registry churn and POST webhook passthrough through root-mounted Control UI. This PR is now redundant/outdated versus the implementation already on `main`.
