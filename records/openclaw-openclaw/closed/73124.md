## Summary

Current main and the v2026.4.27 release cover the concrete OpenClaw work requested here: a supported `models.pricing.enabled` opt-out skips remote pricing bootstrap, pricing refresh is no longer on the immediate startup path, MiniMax Coding Plan quota usage is provider-owned, and Chinese provider plan paths are now represented through bundled provider plugins and manifest pricing policy.

## What This Changes

Add a configuration switch to disable Gateway model-pricing fetches and improve Chinese provider subscription-plan pricing and quota handling.

## Best Possible Solution

Keep the shipped global pricing-fetch opt-out and provider-owned quota/pricing hooks. Future exact-provider gaps, such as a Baichuan-specific bundled plugin or quota API, should be tracked as narrow provider/plugin requests rather than reopening this umbrella.

## Reproduction Assessment

Yes. For the requested opt-out, the high-confidence check is a Gateway config with `models.pricing.enabled: false`; current tests assert neither bootstrap nor direct refresh calls the OpenRouter/LiteLLM fetch path in that state.

## Solution Assessment

Yes. A restart-gated global switch is the narrowest maintainable fix for restricted-network installs, while provider-specific quota and pricing behavior belongs in provider plugins and manifest metadata rather than hardcoded core branches.
