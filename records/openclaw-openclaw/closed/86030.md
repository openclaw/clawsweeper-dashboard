## What This Changes

The issue asks OpenClaw to expose provider API keys, or their secret-file locations, to agents so model-provider tasks can run without re-entering keys.

## Best Possible Solution

Keep the raw-secret boundary: model runtime should resolve credentials through auth profiles/env/config/SecretRefs, while sandboxed tasks that truly need keys should receive explicit operator-managed env or mounted secret delivery.

## Solution Assessment

No, exposing raw provider keys or secret-file paths to agents by default is not the best solution. The safer source direction is the current runtime credential resolver plus explicit operator-managed secret delivery for tasks that need credentials inside a sandbox.
