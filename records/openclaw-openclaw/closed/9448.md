## Summary

Current main already exposes chat type to messaging agents through the system prompt’s trusted inbound metadata, so the request’s functional goal is implemented even though the `Runtime:` line itself still does not render a `chatType=` field.

## Evidence

- **`Runtime:` line still does not render `chatType` literally:** `buildRuntimeLine` in `src/agents/system-prompt.ts` only includes agent/host/repo/os/node/model/default_model/shell/channel/capabilities/thinking, which shows the implementation moved elsewhere rather than into the runtime label itself.
