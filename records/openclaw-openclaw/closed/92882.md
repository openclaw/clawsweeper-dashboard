## Summary

Current main and the shipped v2026.4.27 release already restore the acpx ACP backend registration path that was missing from v2026.4.23, so the reported OpenClaw ACP backend failure is fixed.

## What This Changes

The issue reports that AutoClaw Desktop v2026.4.23 cannot configure ACP because bundled acpx/codex ACP backend registration fails at runtime.

## Best Possible Solution

Keep the shipped plugin-owned acpx startup registration path and direct affected v2026.4.23 users to upgrade to v2026.4.27 or newer.

## Reproduction Assessment

Yes. The v2026.4.23 source lacked the lazy acpx registration body and the registry throws the exact reported error when no backend is registered; current main and v2026.4.27 contain the registration path that addresses it.

## Solution Assessment

Yes. The shipped fix keeps ACP backend ownership in the acpx plugin while exposing only the generic ACP runtime registry to core, which is the narrow boundary the repository policy prefers.
