## Summary

Close as superseded: current main already has the merged `before_dispatch` hook for global pre-model interception, and this branch's unconditional `inbound_claim` broadcast conflicts with current fast-abort and plugin fallback semantics.

## What This Changes

The branch adds a broadcast `runInboundClaim` call in `dispatchReplyFromConfig` before `markProcessing` so global `inbound_claim` hooks can claim regular channel messages.

## Best Possible Solution

Keep `before_dispatch` as the global plugin interception point and leave `inbound_claim` targeted to plugin-bound conversations unless maintainers deliberately reopen that hook contract.

## Reproduction Assessment

Yes. Source inspection shows this patch would run a handled `inbound_claim` before the existing fast-abort branch and before the plugin-bound fallback reaches normal dispatch; current tests and docs make those paths concrete without needing to mutate the checkout.

## Solution Assessment

No. The narrower maintainable solution is the merged `before_dispatch` hook from https://github.com/openclaw/openclaw/pull/50444, while this PR changes `inbound_claim` semantics and introduces ordering regressions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve fast-abort before claim hooks:** `src/auto-reply/reply/dispatch-from-config.ts:418-419`
  - body: The new broadcast runs before `tryFastAbortFromMessage()`, so any plugin that returns `{ handled: true }` for overlapping messages will also swallow standalone abort messages like `/stop` or `abort`. That bypasses the documented fast-abort escape hatch and prevents the active run from being cancelled.
  - confidence: 0.9
- **[P1] Avoid generic claims after plugin fallback:** `src/auto-reply/reply/dispatch-from-config.ts:418-419`
  - body: When a plugin-owned binding falls through as missing or no-handler, the current path sends a fallback notice and proceeds to OpenClaw handling. This unconditional broadcast lets an unrelated global claimer intercept that same turn before the promised fallback reaches `replyResolver`, so plugin-bound fallback messages can disappear.
  - confidence: 0.86
