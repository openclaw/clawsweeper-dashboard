## Label Justifications

- `P2`: This is a normal-priority diagnostics bug for failed subagent completion delivery with a narrow agents/gateway-log surface.
- `impact:message-loss`: The report concerns failed or invisible subagent completion announcements and missing diagnostics for the underlying message delivery failure.

## Summary

Keep open: current main and the latest shipped release still omit the saved delivery failure from subagent announce retry-limit and suspended-delivery expiry warnings, while the only per-attempt direct failure diagnostic is on stderr, which the documented macOS managed gateway suppresses.

## What This Changes

The issue requests observability-only gateway logging so failed subagent completion announcements include the last concrete delivery error and phase context in retry-limit and expiry warnings.

## Best Possible Solution

Add a narrow logging fix that reuses the existing formatted delivery error in retry-limit and expiry warnings, and routes direct-attempt completion announce failures through the normal gateway log path without changing retry policy, delivery ordering, channel behavior, cleanup, or hook semantics.

## Reproduction Assessment

Yes at source level: a subagent run with `lastAnnounceDeliveryError` that reaches retry-limit or suspended-delivery expiry will hit current warnings that omit the saved error. I did not execute a live gateway reproduction because this review is read-only.

## Solution Assessment

Yes: the narrowest maintainable fix is to reuse the already formatted `lastAnnounceDeliveryError` in normal gateway warnings and add regression coverage for retry-limit, expiry, and direct-attempt failure logging.
