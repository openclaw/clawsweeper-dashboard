## Summary

Keep open. Current main still matches the reported launchd managed-env drop, and open PR #75507 is the direct closing fix rather than a reason to close before merge.

## What This Changes

The issue asks macOS service-env regeneration to export the values for keys listed in `OPENCLAW_SERVICE_MANAGED_ENV_KEYS` so env-backed gateway auth tokens survive launchd startup.

## Best Possible Solution

Review and land #75507 or an equivalent narrow daemon install fix that retains durable managed values for launchd's owner-only env-file writer while preserving systemd behavior and plist secret avoidance.

## Reproduction Assessment

Yes. Source inspection on current main shows a state-dir `.env` key can be collected, listed in `OPENCLAW_SERVICE_MANAGED_ENV_KEYS`, deleted from `environment`, and then omitted from the launchd env file writer.

## Solution Assessment

Yes. The best fix is in the daemon install to launchd env-file handoff, keeping durable managed values available only for the owner-only env file rather than changing gateway auth resolution or making the wrapper read `.env` directly.
