## Summary

Current `main` already routes Feishu session-store lookups through the actual agent id instead of hardcoding `main`, which matches the reported root cause for missing footer/session-derived metrics on non-main agents.

## Evidence

- **Reported failure mode:** The issue body and the April 8 confirmation comment both describe the same bug shape: footer/session-derived fields disappear for non-`main` Feishu agents because lookup hits `agents/main/sessions` instead of the active agent store.
