## Summary

Current main already covers the central request with the safer shipped SecretRef and provider-runtime-auth design: command-backed `exec` SecretRefs resolve dynamic provider credentials for `models.providers.*.apiKey` and related request/header surfaces, while provider plugins can use `prepareRuntimeAuth` to exchange credentials and return a runtime `baseUrl` when endpoint selection is provider-owned. The PR’s exact `preScript` field is not present, but the observable enterprise/vault/rotating-credential problem is implemented without adding a generic shell hook to provider config.

## Best Possible Solution

Do not merge this PR. Keep the shipped SecretRef resolver for dynamic command-backed provider credentials, and use provider plugin `prepareRuntimeAuth` for provider-specific token exchange or runtime endpoint selection. If maintainers want generic dynamic `baseUrl` SecretRef coverage later, track it as a separate focused config/API design rather than this broad `preScript` hook.
