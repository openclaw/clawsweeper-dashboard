## Summary

Keep this PR open for normal review. Current main does not have Feishu `streamingMode` or segment flushing, so the PR is not obsolete, but the proposed patch still has concrete ordering and repository-hygiene issues that should be fixed before merge.

## What This Changes

The PR adds Feishu `channels.feishu.streamingMode` with `card` and `segment`, implements segment-based flushing in the Feishu reply dispatcher, updates schema/metadata/docs/tests, and adds a root PR-description markdown file.

## Best Possible Solution

Land the Feishu segment streaming option only after the segment-mode delivery queue also orders media-only payloads, the root PR-description file is removed, generated config metadata is verified through the repo generator/check path, and the targeted Feishu dispatcher/config tests pass.

## Reproduction Assessment

Yes. The media ordering finding can be reproduced with a unit test that defers the first `sendMessageFeishu` segment flush, then delivers a media-only final payload; the PR branch can call `sendMediaFeishu` before that deferred flush resolves. The stray markdown file is directly visible in the PR diff.

## Solution Assessment

No. The feature direction is plausible and narrow to the Feishu plugin, but this patch is not yet the best merge shape because media-only segment payloads need to share the segment delivery queue and the PR-only markdown file should not be committed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Queue media-only segment payloads behind pending flushes:** `extensions/feishu/src/reply-dispatcher.ts:663-664`
  - body: In segment mode, media-only payloads take this branch with `shouldDeliverText` false, so they call `sendMediaReplies` without first waiting for `segmentDeliveryQueue`. If a tool-start or idle partial segment is still being sent, the attachment can appear before the text segment, breaking the ordering guarantee this mode is adding. Queue or await the media send behind the segment delivery queue even when there is no text.
