## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Pass the agent directory into model discovery:** `extensions/github-copilot/index.ts:54-57`
  - body: `fetchAndCacheModels` calls `resolveFirstGithubToken` with only `config` and `env`. Profile-only Copilot auth is stored under the agent directory, so the shared fetch can return an empty live catalog even though the active agent is authenticated.
  - confidence: 0.9
- **[P2] Scope the model cache to the active credentials:** `extensions/github-copilot/index.ts:44-45`
  - body: The module-level cache returns before resolving the current token, base URL, or agent. In a multi-agent gateway or after an auth change, one account's `/models` response can be reused for another account until restart.
  - confidence: 0.87
- **[P2] Preserve xhigh when discovery is unavailable:** `extensions/github-copilot/index.ts:463-464`
  - body: The fallback profile omits `xhigh`, replacing current main's known support for Copilot GPT/Codex ids such as `gpt-5.4`, `gpt-5.2`, and Codex variants. Any timeout, auth miss, or API failure would make those models reject `/think xhigh`.
  - confidence: 0.88
