## Summary

## What This Changes

WebChat is requested to always capture emitted reasoning tokens, keep them after completion, render them collapsed by default with an expandable control, and ideally add a default expansion setting.

## Best Possible Solution

Keep the older WebChat reasoning visibility issue as the canonical tracker, fold this collapsed-by-default preference into that design, and handle the narrower live repaint regression through its linked focused PR.

## Reproduction Assessment

## Solution Assessment

Yes for cleanup: consolidating this newer request into the older open canonical issue is the best current path. The eventual implementation should define one WebChat reasoning disclosure/default policy and then update rendering, persistence/reload behavior, docs, and tests together.
