## Summary

Current `main` already exposes native `openai-codex/gpt-5.4` metadata, separates native `contextWindow` from the effective runtime `contextTokens` cap, and documents the supported override path, so the issue is implemented on `main`.

## Evidence

- **GPT-5.4 native metadata is defined in the Codex provider:** The OpenAI Codex provider sets GPT-5.4 native `contextWindow` to `1_050_000`, default runtime `contextTokens` to `272_000`, and `maxTokens` to `128_000`.
  - file: [extensions/openai/openai-codex-provider.ts:57](https://github.com/openclaw/openclaw/blob/bfa6708c03f24c7caa5d0a96a298c292be0d85c1/extensions/openai/openai-codex-provider.ts#L57)
  - command: `nl -ba extensions/openai/openai-codex-provider.ts | sed -n '1,260p'`
- **Dynamic resolution applies the native window plus default cap:** `resolveDynamicModel` patches `openai-codex/gpt-5.4` to the native window and default cap instead of inheriting the old legacy Codex limits.
  - file: [extensions/openai/openai-codex-provider.ts:223](https://github.com/openclaw/openclaw/blob/bfa6708c03f24c7caa5d0a96a298c292be0d85c1/extensions/openai/openai-codex-provider.ts#L223)
  - command: `nl -ba extensions/openai/openai-codex-provider.ts | sed -n '1,260p'`

## Close Comment

Closing this as implemented after Codex review.

Current `main` already exposes native `openai-codex/gpt-5.4` metadata, separates native `contextWindow` from the effective runtime `contextTokens` cap, and documents the supported override path, so the issue is implemented on `main`.

What I checked:

- **GPT-5.4 native metadata is defined in the Codex provider:** The OpenAI Codex provider sets GPT-5.4 native `contextWindow` to `1_050_000`, default runtime `contextTokens` to `272_000`, and `maxTokens` to `128_000`. (`extensions/openai/openai-codex-provider.ts:57`)
- **Dynamic resolution applies the native window plus default cap:** `resolveDynamicModel` patches `openai-codex/gpt-5.4` to the native window and default cap instead of inheriting the old legacy Codex limits. (`extensions/openai/openai-codex-provider.ts:223`)
- **Catalog/model listing publishes the same values:** Catalog augmentation synthesizes `gpt-5.4` with `contextWindow: 1_050_000` and `contextTokens: 272_000`, so listing metadata matches runtime resolution. (`extensions/openai/openai-codex-provider.ts:549`)
- **Regression tests cover both runtime resolution and catalog exposure:** Tests assert that `openai-codex/gpt-5.4` resolves with native `contextWindow` plus the default `contextTokens` cap, and that augmented catalog entries expose the same metadata. ([`docs/gateway/config-tools.md:393`](https://docs.openclaw.ai/gateway/config-tools))

So I’m closing this as already implemented rather than keeping a duplicate issue open.
