## Summary

Keep open: current PR head still has two source-proven migration blockers around alternate Codex homes and pathless remote curated marketplaces, and the protected maintainer label plus intentional bridge removal need explicit approval before merge.

## What This Changes

The PR removes the OpenClaw Codex plugin bridge, adds native Codex app-server plugin activation for migration, updates docs/config baselines, normalizes newer app-server permission profiles, and adds wildcard tool-contract coverage.

## Best Possible Solution

Land the native Codex path after aligning migration activation with the same Codex home future sessions use, supporting pathless remote curated marketplaces where the app-server protocol allows it, and getting maintainer approval for bridge removal.

## Reproduction Assessment

Yes for the PR blockers by source inspection: --from flows into CODEX_HOME during apply while later app-server launches derive an agent-owned home, and plugin/install supports remoteMarketplaceName although apply rejects pathless marketplaces. I did not execute the PR branch in this read-only review.

## Solution Assessment

No: the native app-server direction is plausible, but this PR should first keep apply-time activation aligned with the future runtime home and handle remote-only curated marketplaces. Maintainers also need to approve the intentional bridge-removal compatibility break.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Use the runtime Codex home during migration activation:** `extensions/codex/src/migration/apply.ts:108`
  - body: `--from` is the source inventory path, but this exports it as `CODEX_HOME` for `plugin/install`. Normal app-server sessions later use an env-provided home or derive the per-agent home, so apply can enable a native plugin in a home future Gateway sessions will not read. Activate against the same resolved home future sessions use, or persist an explicit runtime home before reporting migration success.
  - confidence: 0.9
- **[P2] Pass remote marketplace names to plugin/install:** `extensions/codex/src/migration/apply.ts:215-220`
  - body: For an installed-but-disabled plugin in a remote-only `openai-curated` marketplace, `marketplace.path` is null, so this branch returns an error before activation. `PluginInstallParams` accepts `remoteMarketplaceName`, so pathless curated marketplaces should pass that name instead of requiring a local marketplace path.
  - confidence: 0.84
