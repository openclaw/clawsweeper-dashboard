## Summary

Close as duplicate/superseded: current main already fixes heartbeat-aware daily/idle freshness, and the remaining max-turn TTL/max-lifetime rotation work is tracked by open #45390.

## What This Changes

Request normal sessions to auto-reset after a configured turn count and prevent heartbeat traffic from extending reset freshness.

## Best Possible Solution

Keep the shipped heartbeat-aware freshness behavior and use #45390 as the canonical place to design normal-session max-age/max-turn rotation semantics.

## Reproduction Assessment

Yes for the remaining gap: source inspection on current main shows no reset maxTurns schema/config, no freshness turn input, and no durable turn counter. The heartbeat-staleness half no longer reproduces because runtime/tests use sessionStartedAt and lastInteractionAt instead of updatedAt for daily/idle freshness.

## Solution Assessment

Yes. Closing this as superseded keeps the implemented heartbeat fix in place while consolidating the public TTL API, counting semantics, and rotation behavior in #45390 instead of splitting one product decision across two open issues.
