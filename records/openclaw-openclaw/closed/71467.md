## Summary

Keep open. Current `main` still appears to mishandle whitespace-only streamed assistant text in `buildEmbeddedRunPayloads`, and this PR is a narrow fix with matching regression coverage rather than a stale or already-implemented change.

## Evidence

- **Current main still treats whitespace-only streamed text as a present reply:** `buildEmbeddedRunPayloads` uses `params.assistantTexts.length` and `params.assistantTexts.join(...)` directly when deciding whether to use streamed text or fall back to the stored final answer. A whitespace-only entry counts as present here, so fallback is skipped.

- **Later filtering can drop that whitespace-only reply entirely:** After `answerTexts` is chosen, the loop skips entries whose parsed text is empty and have no media/audio. That means a whitespace-only streamed entry can survive selection, then be discarded, leaving no user-facing payload.
