## Summary

Current `main` contains a post-release Claude CLI hardening change that strips Anthropic routing/auth and OTEL telemetry-related environment before any `claude` backend spawn, and the remaining Anthropic auth-detection path only reads local keychain/files. I did not find a current-main startup path in OpenClaw core that would still explain unsolicited `statsig.anthropic.com` traffic when not using Claude/Anthropic models.

## Evidence

- **Claude CLI backend now clears Anthropic and telemetry env by default:** `CLAUDE_CLI_CLEAR_ENV` explicitly removes Anthropic auth/routing vars plus OTEL exporter vars, with an inline comment calling out `telemetry bootstrap mode`.
  - file: [extensions/anthropic/cli-shared.ts:15](https://github.com/openclaw/openclaw/blob/4f4288e3b5be2e55abfce5213f0bcf182a252f5e/extensions/anthropic/cli-shared.ts#L15)
  - sha: [4f4288e3b5be](https://github.com/openclaw/openclaw/commit/4f4288e3b5be2e55abfce5213f0bcf182a252f5e)
- **Anthropic CLI backend applies that hardened clearEnv list:** The bundled `claude-cli` backend config sets `clearEnv: [...CLAUDE_CLI_CLEAR_ENV]`, so the hardening is active for Claude backend launches.
  - file: [extensions/anthropic/cli-backend.ts:15](https://github.com/openclaw/openclaw/blob/4f4288e3b5be2e55abfce5213f0bcf182a252f5e/extensions/anthropic/cli-backend.ts#L15)
  - sha: [4f4288e3b5be](https://github.com/openclaw/openclaw/commit/4f4288e3b5be2e55abfce5213f0bcf182a252f5e)
- **CLI runner deletes cleared vars before spawn:** The CLI runner removes every key from `backend.clearEnv` before launch and also force-deletes `CLAUDE_CODE_PROVIDER_MANAGED_BY_HOST`, preventing Claude host-managed routing/telemetry behavior from leaking in.
  - file: [src/agents/cli-runner/execute.ts:278](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/agents/cli-runner/execute.ts#L278)
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **Regression test locks the telemetry-env cleanup:** A dedicated test verifies `claude-cli` runs clear Anthropic routing/auth vars and OTEL vars from the child environment.
  - file: [src/agents/cli-runner.spawn.test.ts:1871](https://github.com/openclaw/openclaw/blob/4f4288e3b5be2e55abfce5213f0bcf182a252f5e/src/agents/cli-runner.spawn.test.ts#L1871)
  - sha: [4f4288e3b5be](https://github.com/openclaw/openclaw/commit/4f4288e3b5be2e55abfce5213f0bcf182a252f5e)
- **Current main Anthropic auth discovery is local-file/keychain based:** Claude synthetic auth reads cached local Claude CLI credentials; the credential loader reads macOS keychain or the local Claude credentials JSON, not a network endpoint.
  - file: [extensions/anthropic/register.runtime.ts:396](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/extensions/anthropic/register.runtime.ts#L396)
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **Credential loader itself is local only:** `readClaudeCliCredentialsCached()` delegates to `readClaudeCliCredentials()`, which reads macOS keychain or a local credential file path and does not call Anthropic services.
  - file: [src/agents/cli-credentials.ts:347](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/agents/cli-credentials.ts#L347)
