Full review comments:

- **[P1] Preserve self-contained providers during merge pruning:** `src/agents/models-config.merge.ts:223`
  - body: Starting the merge from only `nextProviders` drops any existing generated provider that is not in the next plan, including self-contained custom/baseUrl providers that current main preserves during regen.   - confidence: 0.9
- **[P1] Use stored reply context only as fallback:** `src/auto-reply/reply/agent-runner.ts:524-525`
  - body: `contextTokensOverride` returns before provider/model lookup, so including `activeSessionEntry?.contextTokens` here keeps a poisoned session window authoritative after the run already knows `providerUsed` and `modelUsed`. Pass only the agent config override and use the stored session value as `fallbackContextTokens`.
  - confidence: 0.88
- **[P1] Use stored follow-up context only as fallback:** `src/auto-reply/reply/followup-runner.ts:282-283`
  - body: This has the same stale-context problem as the reply runner: `sessionEntry?.contextTokens` in `contextTokensOverride` bypasses provider-qualified lookup and can keep follow-up accounting wrong indefinitely. The stored value should be a fallback, not the authoritative override.
  - confidence: 0.88
- **[P1] Keep transient network failures eligible for failover:** `src/agents/failover-error.ts:242-243`
  - body: Limiting code-based timeout classification to `ETIMEDOUT` and `ESOCKETTIMEDOUT` removes failover for current retryable network errors such as `ECONNRESET`, `ECONNREFUSED`, `EAI_AGAIN`, `ENETUNREACH`, and `EPIPE`. That can strand fallback chains on provider/network interruptions that current main deliberately treats as retryable.
  - confidence: 0.9
- **[P2] Preserve the mutating-tool side-effect warning:** `src/agents/pi-embedded-runner/run.ts:1097`
  - body: When a provider returns a friendly incomplete-turn error after mutating tools ran, `formattedAssistantErrorText || genericEmptyTurnError` hides the existing warning that tool actions may already have executed. Keep that side-effect caution visible so users know to verify before retrying.
  - confidence: 0.78
