## Summary

## What This Changes

The issue reports Telegram response truncation or non-delivery when literal angle-bracket reasoning/control tags such as `<think>` interact with HTML formatting and strict reasoning-tag stripping.

## Best Possible Solution

Keep this issue as the canonical tracker, then land or replace a narrow fix that prevents Telegram visible answer loss while preserving the strict reasoning privacy boundary and proving the channel path.

## Reproduction Assessment

Yes, at source level. Telegram answer splitting uses strict reasoning-tag stripping, and current tests still assert that a visible prefix before an unclosed `<think>` drops the tail; I did not run live Telegram proof in this read-only review.

## Solution Assessment

Unclear. The likely repair belongs at the shared strict-strip boundary plus Telegram regression coverage, but maintainers must choose the strict-mode recovery semantics so the fix does not trade message loss for reasoning exposure.
