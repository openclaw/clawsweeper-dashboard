## Summary

Keep this PR open. Current main still lacks the image-level `/home/node/.openclaw` ownership fix, while default config writes still create temporary `openclaw.json.*.tmp` files under that directory. The existing Docker setup chown path helps repo-managed bind mounts, but it does not make the proposed named-volume image fix obsolete for #61279. The PR diff is focused and does not add new dependencies, workflow execution, secrets access, or package resolution changes.

## Best Possible Solution

Review and land this PR, or an equivalent Dockerfile change, as the implementation candidate for #61279. Keep the directory creation before `USER node`, retain a release changelog entry if merged, and consider adding a Dockerfile unit assertion that `/home/node/.openclaw` is created with `node:node` ownership before the runtime image switches to the non-root user.
