## Summary

Current main already addresses the reported Telegram startup crash by preserving package-level staged runtime-deps supersets and by mirroring `json5` as a root runtime dependency. The fix is on main with targeted regression coverage, but it is not in the `v2026.4.29` tag.

## What This Changes

The issue asks OpenClaw to fix a packaged Telegram/channel startup crash where staged bundled-runtime-deps cannot resolve the root `json5` dependency imported by dist frontmatter code in OpenClaw 2026.4.29.

## Best Possible Solution

Keep the generic runtime-deps materialization fix and ship it in the next release so packaged channel startup preserves the pre-staged root dependency set instead of adding Telegram-specific dependency handling.

## Reproduction Assessment

Yes. The issue includes a concrete npm-global `v2026.4.29` failure path and missing-package stack, and current main has a focused regression test for the staged runtime-deps superset scenario that would otherwise reinstall/prune dependencies.

## Solution Assessment

Yes. The main fix is the narrow maintainable solution: preserve generated package-level runtime-deps supersets and continue declaring `json5` as a mirrored root runtime dependency, rather than changing shared frontmatter imports or adding Telegram-only recovery logic.
