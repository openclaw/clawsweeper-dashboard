## Summary

This PR duplicates the same Slack manifest-copy bug already tracked by #65751 and the earlier open PR #65773. Current main still has the bug, but this PR's console.log-based approach is not the best implementation because it bypasses the wizard transport used by gateway/UI setup flows.

## Best Possible Solution

Close this duplicate PR, keep #65751 as the canonical bug report, and implement the fix in #65773 or a replacement PR by emitting a copyable Slack manifest through the wizard/prompter contract so both local CLI and gateway/UI setup flows receive it.
