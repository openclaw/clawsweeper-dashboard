## What This Changes

The branch adds SIGINT/SIGTERM abort handling to `scripts/install.sh`, captures doctor exit 130, and only enables dashboard opening after a successful upgrade doctor.

## Best Possible Solution

Land a rebased installer fix that preserves source-install dashboard behavior, exits 130 for SIGINT and 143 for SIGTERM, and adds focused proof or coverage for interrupted and failed doctor paths.

## Reproduction Assessment

Yes, from source plus the PR's terminal proof: current main lacks an installer INT/TERM abort trap and the upgrade doctor path can treat interruption as ordinary failure. I did not run the live installer in this read-only review.

## Solution Assessment

No, not as written. The fix should keep the clean abort behavior but preserve SIGTERM's 143 status and the current successful git/source install dashboard path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve SIGTERM's exit status:** `scripts/install.sh:41`
  - body: `trap abort_install INT TERM` sends SIGTERM through a handler that always exits 130. That makes termination look like Ctrl+C instead of the conventional 143, which can break callers or supervisors that distinguish SIGINT from SIGTERM.
  - confidence: 0.91
- **[P2] Keep source installs opening the dashboard after doctor:** `scripts/install.sh:2937`
  - body: Removing the shared `should_open_dashboard=true` means `INSTALL_METHOD=git` runs `run_doctor` and then takes the source-install details branch, never reaching the new upgrade-only assignment. Successful source installs would no longer open the dashboard after doctor.
  - confidence: 0.9
