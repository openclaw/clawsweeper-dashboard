## Summary

Keep open: current main now has a generic `before_agent_run` block gate and improved external-content sanitization, but it still lacks the requested configurable tool-output prompt-injection scanner/blocker and the security contract for block/warn/redact behavior.

## What This Changes

The issue requests configurable content-based prompt-injection scanning or blocking for tool outputs before those outputs reach the model.

## Best Possible Solution

Define the security contract first, then implement either a tested opt-in bundled scanner or a versioned pre-model tool-result gate with documented policy, config, SDK boundaries, and focused regression coverage.

## Reproduction Assessment

Not applicable: this is a security hardening feature request, not a broken existing contract. Source inspection gives a high-confidence gap check that current main lacks the requested configurable scanner/blocker.

## Solution Assessment

Unclear: the requested direction is valid, but the exact solution needs maintainer security/product judgment. The newer `before_agent_run` gate helps plugin authors block whole runs, but it does not replace a tool-result scanner policy or a safe rewrite/redaction contract.
