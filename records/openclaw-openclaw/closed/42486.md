## Summary

Current main already contains a same-session followup-queue restart fix, matching the reported 'session B only resumes after unrelated activity' behavior. The queue is keyed per session, idle drains are explicitly restarted on that same key, and shipped regression tests cover the idle-window restart path.

## Evidence
