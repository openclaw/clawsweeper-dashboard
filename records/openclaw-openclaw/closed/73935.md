## Summary

Keep this PR open. Current main still does not implement the PR's intended `modelContextWindowOverrides` config path: Bedrock discovery uses the built-in known-model table or `defaultContextWindow`, the plugin schema has `additionalProperties: false` without this key, and the docs/tests do not cover the override behavior. This PR is a live implementation candidate for #73328, but maintainers should review it alongside open alternative PR #73382 and choose the canonical policy: configurable regex overrides versus a Claude family/table fallback. Security pass: the file list shows no CI, dependency, lockfile, package script, or supply-chain changes; the main security-sensitive point is runtime evaluation of admin-configured regexes, which needs maintainer acceptance or guardrails before merge.

## Best Possible Solution

Keep this PR open for maintainer review. The best outcome is to review it together with #73328 and #73382, decide whether OpenClaw wants a configurable per-model Bedrock override surface or a narrower built-in Claude fallback/table fix, then land one canonical implementation with discovery tests, plugin schema/type coverage, docs, and an appropriate changelog entry while closing the superseded alternatives.
