## Summary

Current main already fixes the reported runtime/provider selection path: configured web_search providers with manifest owners are treated as explicit selections, so a configured Brave provider no longer falls through to the loaded Claude CLI runtime registry and Gemini.

## What This Changes

The issue reports that `web_search` ignores `tools.web.search.provider: "brave"` under `claude-cli` and routes to Gemini instead.

## Best Possible Solution

Keep the current manifest-owner based provider selection on main and let the next release carry it; reopen only with evidence from a build at or after the fix SHA that still selects Gemini.

## Reproduction Assessment

No for current main. I did not run a live Claude CLI/Brave setup, but source and regression coverage show the reported fallback path is now guarded.

## Solution Assessment

Yes. The current implementation fixes the owner-boundary problem through generic manifest ownership and runtime-context selection instead of adding a Brave- or Claude-specific shim.
