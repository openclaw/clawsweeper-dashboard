## Summary

Current main now covers the central downgrade-safety gap: foreground and Docker-style gateway startup run the newer-config guard before automatic state migrations, and the fix is current-main-only rather than included in v2026.6.8.

## What This Changes

The issue requests a startup/layout version gate so downgrades cannot split OpenClaw state between legacy root and workspace directories.

## Best Possible Solution

Keep the current `meta.lastTouchedVersion`-based startup guard and ship it in the next release so foreground and Docker gateway rollbacks fail before state migration or server startup.

## Reproduction Assessment

No high-confidence current-main failure remains: current source and regression tests show gateway startup now blocks newer-written config before automatic state migrations. The earlier foreground gap is source-proven at f1b8827d20c8, but it is not present on current main.

## Solution Assessment

Yes. Reusing the existing `meta.lastTouchedVersion` guard at the gateway startup/migration boundary is narrower and more maintainable than adding a second `.openclaw_version` marker or symlink-routing scheme.
