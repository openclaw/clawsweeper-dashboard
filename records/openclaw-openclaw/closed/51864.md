## Summary

Current main already maps direct Google/Gemini `usageMetadata` into OpenClaw usage on the shipped Google transport path, with test and docs coverage, so this PR is obsolete.

## Risks / Open Questions

- The shipped fix is provider-specific in `extensions/google/transport-stream.ts`, not the shared `src/agents/usage.ts` normalization layer proposed here. If a future Gemini path bypasses that transport, similar normalization work could be needed there.
