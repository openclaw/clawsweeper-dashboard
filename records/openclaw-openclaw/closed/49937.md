## Summary

Current main already solves the timeout bug with a narrower implementation than this PR: inline memory embedding batches now honor `agents.*.memorySearch.sync.embeddingBatchTimeoutSeconds`, provider batch APIs still use `remote.batch.timeoutMinutes`, and docs/schema/tests cover the split. The extra retry-matcher and full-URL fetch-error changes from #49937 were not carried forward, which is appropriate given the review-thread risks.

## Best Possible Solution

Close #49937 and keep the current main implementation: use `sync.embeddingBatchTimeoutSeconds` for inline memory embedding batches, keep `remote.batch.timeoutMinutes` scoped to provider batch APIs, and avoid merging this PR's broader retry and full-URL fetch-error changes.
