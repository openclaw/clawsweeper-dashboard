## Summary

Current `main` no longer relies on `../../../src/...` imports for the affected bundled channels, and the needed helpers are exported through the public plugin SDK instead. The Docker runtime image still copies only built/runtime assets, so the original failure mode is addressed by removing the cross-boundary imports rather than by copying `src/` into the image.
