## Summary

Current main implements the requested delivery-boundary protection: internal runtime scaffolding tags are stripped from outbound payload text and nested payload fields before channel delivery, Discord now has explicit sanitizer wiring for the reported surface, and regression tests cover the helper, hook-rewritten payloads, normalized/rendered payloads, and Discord-native angle markup preservation. The reporter and follow-up commenter provided useful evidence that the tags came through Claude Code/claude-max and that Discord lacked sanitizer wiring; current main addresses both observable OpenClaw delivery gaps.

## Best Possible Solution

Close as implemented on current main. Keep the shared final outbound scaffolding sanitizer plus Discord-specific preservation behavior, and let the next release carry the fix since no current release tag contains the proof SHA.
