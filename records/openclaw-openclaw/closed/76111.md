## Summary

Keep this PR open, but it still needs one functional repair before merge. The earlier wrong-registry issue is fixed, but the current branch still misses the default Control UI `main` session alias, so the reported Stop-after-reload path remains broken for the common dashboard session.

## What This Changes

The PR adds authenticated Gateway WebSocket pings, exposes session active-run state on session list/change payloads, and updates Control UI abort handling so Stop can remain available after reconnect or reload.

## Best Possible Solution

Repair `sessions.list` to precompute active session keys plus their canonical store keys from `context.chatAbortControllers`, then keep the UI changes and changelog with focused alias and reconnect tests.

## Reproduction Assessment

Yes. Current Control UI starts with `sessionKey: "main"`, current `chat.send` registers the active abort entry under that raw key, and the PR compares only the canonical returned row key `agent:main:main`, so `hasActiveRun` stays false after reload for the default dashboard path.

## Solution Assessment

No. The product direction is right and the post-auth ping timer is reasonable, but the active-run lookup must canonicalize aliases and use a precomputed set so it works for the default Control UI session without regressing `sessions.list` responsiveness.
