## Summary

Current `main` already supports manual Ollama configuration and no longer requires the older workaround described in the report. The Ollama provider now accepts explicit `models.providers.ollama` config, recognizes the non-secret `ollama-local` marker, injects synthetic local auth for explicit Ollama configs without a real API key, and has regression coverage for both implicit discovery and manual remote config paths.
