## Summary

Current `main` contains a direct fix for this Control UI image-loss report: sent user messages keep optimistic image blocks, mid-run session-change/history reloads are deferred to avoid clobbering the pending user card, and history rendering now reconstructs offloaded user images from transcript `MediaPath`/`MediaPaths` metadata.

## Evidence

- **Optimistic send keeps image preview blocks in the user message:** `sendChatMessage` now appends `{ type: "image", source: { type: "base64", ... } }` blocks before the request is sent, so the just-sent message has a visible image immediately.
  - file: [ui/src/ui/controllers/chat.ts:315](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/ui/src/ui/controllers/chat.ts#L315)
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **Mid-run history reload race is explicitly blocked:** Control UI now defers `loadChatHistory` while a run is active because reloading history mid-run raced the optimistic user message update and delayed the user card until the first model delta.
  - file: [ui/src/ui/app-gateway.ts:496](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/ui/src/ui/app-gateway.ts#L496)
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **History renderer restores transcript media from `MediaPath`/`MediaPaths`:** `extractImages` now reads transcript `MediaPath` / `MediaPaths` plus `MediaType` / `MediaTypes` and converts allowed image paths back into renderable image blocks, which matches the issue's reported recovered history shape.
  - file: [ui/src/ui/chat/grouped-render.ts:194](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/ui/src/ui/chat/grouped-render.ts#L194)
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **Regression test covers user-history image rendering from transcript media metadata:** The grouped-render test asserts that user messages with `MediaPath` and `MediaPaths` render back through the authenticated Control UI media route, and that blocked/non-image paths are skipped.
  - file: [ui/src/ui/chat/grouped-render.test.ts:650](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/ui/src/ui/chat/grouped-render.test.ts#L650)
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **Regression tests cover the reload timing fix:** One test asserts Control UI does not reload chat history for each live event, and another asserts it reloads exactly once after the final event.
  - file: [ui/src/ui/app-gateway.node.test.ts:622](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/ui/src/ui/app-gateway.node.test.ts#L622)
