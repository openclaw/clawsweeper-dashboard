## Summary

Keep this PR open. Current main still discovers usage-counted session transcript files for memory-host-sdk session listing, so reset/deleted archive variants remain part of the QMD/session indexing source. The PR’s central behavior, primary-transcript-only discovery, is therefore not implemented on main. The PR context also shows useful follow-up after earlier review comments, but the branch still includes unrelated plugin loader, doctor, gateway, browser, and bundled-plugin trust/test-isolation changes that need maintainer review or splitting.

## Best Possible Solution

Keep the PR open for maintainer review. If maintainers accept primary-transcript-only memory indexing, land a scoped version that updates both `src/memory-host-sdk` and `packages/memory-host-sdk`, preserves exact compaction-checkpoint filtering, and keeps reset/deleted regression coverage. Split or explicitly justify the unrelated plugin loader, doctor, gateway, browser, and bundled-plugin trust changes before merge; if maintainers prefer the shipped recall behavior that includes reset/deleted usage-counted transcripts, close or revise the PR on that product basis.
