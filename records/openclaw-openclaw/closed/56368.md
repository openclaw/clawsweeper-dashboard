## Summary

Current main and the shipped v2026.5.28 release already add an unauthenticated ClawHub 429 hint, with regression coverage for the search path and shared ClawHub error formatting used by downloads as well.

## What This Changes

Request to append a helpful sign-in hint when ClawHub skill search or install receives a 429 rate-limit response without ClawHub authentication.

## Best Possible Solution

Keep the shipped shared ClawHub 429 error formatter so skills search/install and plugin/package downloads all give the same authenticated versus unauthenticated guidance.

## Reproduction Assessment

No current-main failing reproduction remains: source and tests show unauthenticated ClawHub 429 responses now include sign-in guidance, while authenticated 429 responses do not.

## Solution Assessment

Yes. The shipped shared ClawHub error path is the maintainable solution because it covers search, skill-card, package, and download failures instead of special-casing only one skills command.
