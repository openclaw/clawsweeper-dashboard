## Summary

Current main already addresses the reported Gateway/plugin reload loop: config watcher events are debounced, bookkeeping paths do not request plugin reloads, and in-process config write notifications now use canonical source config so runtime-shaped writes do not create phantom plugin diffs.

## What This Changes

Reduce redundant Gateway plugin hot-reloads triggered by multiple openclaw.json bookkeeping writes after an agent turn.

## Best Possible Solution

Keep the current main behavior for config reloads and leave #76268 to track the separate plugin registry cache-key optimization.

## Reproduction Assessment

Yes. The high-confidence path is the Gateway config reloader/write-notification path; existing tests cover watcher dedupe and runtime-shaped config writes, and source inspection shows bookkeeping writes no longer plan plugin reloads.

## Solution Assessment

Yes. Canonical source-config notifications plus reload-plan no-op classification and watcher debounce are the narrow maintainable fix for redundant hot-reloads without suppressing real plugin config changes.
