## Summary

Current main resolves the reported outbound local-media allowlist failure without adding the exact proposed config knob: outbound media access now derives allowed roots from concrete media sources when the configured fs policy permits host reads, passes those roots through message action dispatch, and Feishu forwards them into `loadWebMedia`. The 2026.3.24 changelog records this as shipped behavior for host-local files when `tools.fs.workspaceOnly` is off.

## Evidence
