## Summary

Current `main` already supports the requested setup: multiple local backends can be added as separate `models.providers.<id>` entries with different `baseUrl`s, the heartbeat can run on a different model than the main chat agent, and the reported tool-calling symptom matches a documented Ollama `/v1` misconfiguration that OpenClaw already warns against.

## Evidence

## Risks / Open Questions

- The issue reads like setup friction rather than a missing core capability; if the reporter is specifically asking for a more guided interactive workflow, that UX gap is real even though the underlying feature exists.
- If the reporter was trying to run two native Ollama hosts simultaneously under the single built-in `ollama` provider id, they may still need custom provider ids/config rather than the one-provider onboarding path.
