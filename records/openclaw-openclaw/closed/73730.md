## Summary

Keep this issue open. Current main still has the reported identity-keyed web-provider snapshot cache, and open PR #73847 is the active implementation candidate with closing syntax for this issue. The right path is to review and land that PR or an equivalent content-keyed cache fix, then close after merge.

## Best Possible Solution

Keep this issue open and review/land #73847 or an equivalent fix that replaces the outer config-identity WeakMap with a bounded content-keyed cache, including config fingerprints for provider-affecting plugin/web settings and regression tests for fresh-but-equal configs hitting while genuinely different configs miss.
