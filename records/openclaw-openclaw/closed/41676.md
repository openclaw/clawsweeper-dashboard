## Summary

## What This Changes

The issue requests bounded Gateway error logging and suppression or aggregation of repeated WebSocket heartbeat timeout lines so `gateway.err.log` cannot grow indefinitely.

## Best Possible Solution

Keep the shipped stderr-suppression behavior for the exact `gateway.err.log` bloat, and handle any remaining launchd stdout retention policy in a separate focused item or PR.

## Reproduction Assessment

No. Current main and `v2026.5.18` configure macOS LaunchAgent stderr as `/dev/null`, so the exact `gateway.err.log` spam path no longer exists; tests and docs pin the same behavior.

## Solution Assessment

Yes. Suppressing launchd stderr removes the unbounded `gateway.err.log` sink that was filling disk with duplicate heartbeat timeout lines; broader stdout retention should stay separate because it has different files and tradeoffs.
