## Summary

Keep open. Current `main` (`7d471837369885c12d540d6bafa319ff41129b89`) still does not enforce the `allowed-tools` list shape in `skills/skill-creator/scripts/quick_validate.py`, so the issue's reported gap remains present.

## Evidence

- **Validator allows the key but never validates its shape:** `validate_skill()` whitelists `allowed-tools`, then only validates `name` and `description` before returning success. There is no check that `allowed-tools` is a list of non-empty strings.
  - file: [skills/skill-creator/scripts/quick_validate.py:98](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/skills/skill-creator/scripts/quick_validate.py#L98)

  - sha: [7d4718373698](https://github.com/openclaw/openclaw/commit/7d471837369885c12d540d6bafa319ff41129b89)
- **Fallback parser would also accept malformed values:** The no-PyYAML fallback parser stores frontmatter values as plain strings and concatenated indented text. Without a later shape check, malformed `allowed-tools` content still passes validation.
  - file: [skills/skill-creator/scripts/quick_validate.py:29](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/skills/skill-creator/scripts/quick_validate.py#L29)
