## What This Changes

The PR changes `src/infra/heartbeat-runner.ts` to write heartbeat dedupe metadata through `updateSessionStore` and removes `saveSessionStore` from the sessions import.

## Best Possible Solution

Use #72766, or an equivalent current-main patch, as the canonical fix: preserve the still-needed `saveSessionStore` import, convert the heartbeat metadata writes that can clobber concurrent session-store updates to `updateSessionStore`, and add focused regression coverage if maintainers want explicit race protection.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a concurrent writer can update the same session store after heartbeat loads it and before heartbeat saves it, causing the stale heartbeat snapshot to overwrite the newer data.

## Solution Assessment

No, not as this PR currently stands. The helper choice is right for the dedupe write, but the branch is stale against current main and #72766 covers the same fix more completely.
