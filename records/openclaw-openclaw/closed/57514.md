## Summary

Current `main` no longer matches the reported reproduction: WebChat is handled as a core Gateway Control UI/WebSocket surface rather than a bundled plugin with `register()`/`registerHttpRoute("/webchat")`, and current plugin registry code rejects duplicate channel/HTTP-route registrations instead of accumulating them.

## Evidence

- **No current bundled WebChat plugin:** A read-only search for a bundled `webchat` extension returned no matches under `extensions/`, so there is no current `extensions/webchat` plugin whose `register()` could be hot-reloaded repeatedly.
  - command: `find extensions -maxdepth 2 \( -iname '*webchat*' -o -iname '*web-chat*' \) | sort`
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **WebChat docs describe core Gateway/WebSocket architecture:** The current WebChat docs say the macOS/iOS chat UI talks directly to the Gateway WebSocket and is 'no embedded browser and no local static server'.
  - file: [docs/web/webchat.md:8](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/docs/web/webchat.md#L8)
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **Remote docs say WebChat no longer uses a separate HTTP port:** Current remote docs explicitly state: 'WebChat no longer uses a separate HTTP port. The SwiftUI chat UI connects directly to the Gateway WebSocket.'
  - file: [docs/gateway/remote.md:124](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/docs/gateway/remote.md#L124)
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **`/webchat` is served by core Control UI handler:** Current HTTP handling routes Control UI/WebChat requests through `handleControlUiHttpRequest(...)` in core Gateway code, not through plugin route registration.
  - file: [src/gateway/control-ui.ts:684](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/gateway/control-ui.ts#L684)
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **WebChat websocket handling is centralized in Gateway:** Current WebChat connections are handled inside the shared Gateway websocket message handler (`isWebchatConnect(...)` logging), not by per-request plugin-created websocket servers.
  - file: [src/gateway/server/ws-connection/message-handler.ts:1233](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/gateway/server/ws-connection/message-handler.ts#L1233)
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **Current registry rejects duplicate registrations:** `registerHttpRoute` now errors on duplicate routes unless the same plugin explicitly replaces its own route, and `registerChannel` errors on duplicate channel ids instead of silently accumulating them.
  - file: [src/plugins/registry.ts:505](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/plugins/registry.ts#L505)
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)

## Close Comment

Closing as `cannot_reproduce` after a Codex review against current `main`.

- Current `main` does not have a bundled `webchat` plugin under `extensions/`, so the reported `webchat.register()` hot-reload loop does not match the present code layout.
- WebChat is now documented and implemented as a core Gateway surface that talks directly over the Gateway WebSocket: `docs/web/webchat.md:8`, `docs/gateway/remote.md:124`, `src/gateway/control-ui.ts:684`, `src/gateway/server/ws-connection/message-handler.ts:1233`.
- Current plugin registry code also rejects duplicate channel and HTTP-route registrations instead of silently accumulating them: `src/plugins/registry.ts:505` and `src/plugins/registry.ts:595`.
- Because the issue's reproduction depends on a WebChat plugin/register path that is not present on current `main`, this report appears obsolete relative to today's codebase.
