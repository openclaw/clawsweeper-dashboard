## Summary

Current `main` still has the proactive/reactive retry split described in #59107. Reactive WhatsApp auto-replies use a retry helper, but the proactive outbound path still calls the active listener once and propagates transient socket errors; the current test suite explicitly preserves that no-retry behavior.
