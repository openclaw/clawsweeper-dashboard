## Summary

Current main already remaps skills into the sandbox workspace for sandboxed sessions, so `<available_skills>` points at sandbox-readable `skills/.../SKILL.md` paths instead of host-only paths.

## Evidence

- **Sandbox setup syncs skills into sandbox workspace:** During sandbox workspace setup, read-only sandbox runs call `syncSkillsToWorkspace` with `sourceWorkspaceDir` set to the agent workspace and `targetWorkspaceDir` set to the sandbox workspace before the run starts.
