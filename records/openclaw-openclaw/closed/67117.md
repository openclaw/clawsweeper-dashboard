## Summary

Current main already implements Feishu/Lark per-account inbound lanes, non-blocking multi-account handler dispatch, and mention-gated early skips; the capability was shipped in v2026.4.23.

## Evidence

- **Per-account inbound lane:** The Feishu message receive handler derives the sequential queue key as `feishu:${accountId}:${chat_id}` and enqueues work on that key, so different bot accounts in the same group no longer share one inbound queue.
  - file: [extensions/feishu/src/monitor.message-handler.ts:172](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/extensions/feishu/src/monitor.message-handler.ts#L172)

  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Non-blocking account handler dispatch:** Feishu monitor handlers run in `fireAndForget` mode by default and call `void params.task()` when enabled, which avoids websocket event head-of-line blocking across long-running handlers.
  - file: [extensions/feishu/src/monitor.account.ts:313](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/extensions/feishu/src/monitor.account.ts#L313)
