## Summary

Current `main` now handles this in two places: the gateway advertises node-scoped canvas capability URLs so macOS WKWebView does not need to send `Authorization` headers, and the macOS app has a working `openclaw-canvas://` scheme handler that resolves session files like `openclaw-canvas://main/`. The fix is present on `main` but does not appear to be in the latest released tag yet.

## Evidence
