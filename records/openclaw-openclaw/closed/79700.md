## Summary

Keep open: this is a focused, currently useful PR for a real Telegram reverse-delivery bug on main. The current parser can misclassify duplicated DM bindings as topics, and the proposed guard addresses that path without broad ACP or routing changes.

## What This Changes

The branch adds a Telegram topic parser guard for duplicated DM conversation/parent ids, a colocated parser regression test, and a changelog entry.

## Best Possible Solution

Merge the focused parser guard, regression test, and changelog once normal maintainer review and exact-head gates complete.

## Reproduction Assessment

Yes. Source inspection on current main shows the duplicated DM shape follows the bare-topic parser branch and becomes a reverse-delivery target with a spurious threadId; the PR body also supplies before/after runtime output for that same path.

## Solution Assessment

Yes. Returning null for parent === conversation is the narrowest maintainable fix because the existing direct-target fallback already handles DM delivery while explicit topic strings and group parent/topic parsing remain unchanged.
