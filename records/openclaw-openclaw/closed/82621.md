## Summary

Keep open: current main still routes local CLI web search and web fetch through raw runtime config before provider execution, while the docs promise plugin-scoped `webSearch.apiKey` SecretRefs are resolved for selected providers. Open linked fix PRs already reference this issue with closing syntax, so this issue should stay open until one canonical fix lands.

## What This Changes

CLI `infer web search` should resolve selected plugin-scoped `webSearch.apiKey` SecretRefs before running the provider and leave non-selected provider refs inactive.

## Best Possible Solution

Review and land one canonical fix that resolves command SecretRefs before local web provider execution, preserves selected-provider scoping, adds regression coverage, and then closes this issue after merge.

## Reproduction Assessment

Yes. The issue gives a concrete `openclaw infer web search --provider tavily --query ping --limit 1 --json` repro, and current main's source shows that path still passes raw config to the web-search runtime before command SecretRef resolution.

## Solution Assessment

Yes. Resolving the CLI command config through the existing command SecretRef resolver before provider construction is the right direction, but the final fix should land through one canonical PR with selected-provider target scoping.
