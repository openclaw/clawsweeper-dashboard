## Summary

Current `main` already accepts Telegram's nested `streaming.mode` shape, but it still does not allow `allowedHostnames` under `tools.media.*.request` or media model request config. The PR's stated fix is therefore only partially implemented on `main`, so there is not enough basis to close it as implemented.
