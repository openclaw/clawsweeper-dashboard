## Summary

The reported `neodata`/`query.py` `/proxy/api` path is not present on current `main`; current OpenClaw HTTP auth requires an explicit bearer token and regression coverage expects unauthenticated plugin HTTP requests to 401, so this does not reproduce as an OpenClaw core issue.

## Evidence

- **No in-repo neodata/proxy implementation:** A repo-wide search on current `main` found no `neodata`, `query.py`, or `/proxy/api` under `skills/`, `extensions/`, `src/`, or `docs`, so the reported integration is not part of the code shipped in `openclaw/openclaw`.
  - command: `rg -n "neodata|query\.py|proxy/api" skills extensions src docs`
  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **Gateway HTTP auth requires explicit bearer input:** HTTP auth is derived from the incoming `Authorization` header and only passes `connectAuth` when a bearer token is present; otherwise the request falls through the normal auth failure path.
  - file: [src/gateway/http-utils.ts:43](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/gateway/http-utils.ts#L43)
  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **Protected plugin HTTP routes are expected to 401 without auth:** Regression coverage explicitly asserts that protected plugin HTTP routes return unauthorized until the caller supplies `Authorization: Bearer ...`, and then succeed once authenticated.
  - file: [src/gateway/server.plugin-http-auth.test.ts:230](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/gateway/server.plugin-http-auth.test.ts#L230)
  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **Plugin-owned HTTP auth stays with the plugin contract:** The plugin internals docs require plugin routes to declare auth explicitly and document that gateway-auth plugin routes are conservative, rather than implicitly inheriting privileged session behavior.
  - file: [docs/plugins/architecture-internals.md:551](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/docs/plugins/architecture-internals.md#L551)
  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **External plugins belong outside core:** The plugin guide says plugins should be published through ClawHub or npm instead of being added to the OpenClaw repository, which matches this issue targeting an external skill rather than core code.
  - file: [docs/plugins/building-plugins.md:16](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/docs/plugins/building-plugins.md#L16)
  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
