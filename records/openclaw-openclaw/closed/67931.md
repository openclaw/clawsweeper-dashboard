## Summary

Current main already supports llama.cpp through OpenClaw's generic local OpenAI-compatible provider path, with user-facing docs, compatibility handling, and regression tests. There is no dedicated bundled `llama.cpp` onboarding plugin, but the requested provider support is already implemented well enough to close this feature request.
