## Summary

Current main already handles the reported file descriptor cleanup in the shared boundary-file helper, and the PR’s proposed loader-side close calls target a failure variant that does not expose an fd. The shipped helper closes any fd opened before post-open validation fails and has a regression test for that case, so the PR should close rather than merge.

## What This Changes

The PR modifies `src/plugins/loader.ts` to add `fs.closeSync(...)` before three `openBoundaryFileSync()` failure-path `continue` branches, with small incidental formatting/import reorder changes.

## Best Possible Solution

Close this PR without merging and keep fd ownership in the shared `openVerifiedFileSync()` helper, where post-open validation cleanup is already centralized and tested.
