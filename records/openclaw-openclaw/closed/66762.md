## Summary

Current main already addresses the reported WebChat flicker path: chat sends still append the user message optimistically, and the Control UI gateway now defers `chat.history` reloads while a run is active instead of reloading immediately on `session.message`. New tests cover both the skip-during-run behavior and the deferred replay after terminal events.

## Evidence

- **Optimistic user message still appends locally on send:** `sendChatMessage()` appends a user message into `state.chatMessages` before the network request, so the current code path does not clear the message list when Enter is pressed.
  - file: [ui/src/ui/controllers/chat.ts:325](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/ui/src/ui/controllers/chat.ts#L325)
