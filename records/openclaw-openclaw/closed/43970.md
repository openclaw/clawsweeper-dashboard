## Summary

Current main already covers this request: Discord bound threads bypass mention gating, bot-created autoThreads are auto-exempt, `sessions_spawn({thread:true})` and `/focus` create or bind threads, and runtime commands already include `/activation mention|always` plus `/unfocus` for thread-level control.

## Close Comment

Closing as implemented on main after Codex review. Discord already bypasses mention gating for bound threads (`extensions/discord/src/monitor/message-handler.preflight.ts:696`), auto-exempts bot-created autoThreads (`extensions/discord/src/monitor/allow-list.ts:492`), and already supports the thread workflows this request asked for: `sessions_spawn({thread:true})` creates or binds a thread, `/focus` binds the current thread or creates one, and `/unfocus` removes that binding (`docs/tools/subagents.md:115`). OpenClaw also already exposes a runtime slash command for activation control, `/activation mention|always` (`src/auto-reply/commands-registry.shared.ts:629`, `src/auto-reply/reply/commands-session.ts:227`). I also verified those same pieces are present in the shipped `v2026.4.22` release (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
