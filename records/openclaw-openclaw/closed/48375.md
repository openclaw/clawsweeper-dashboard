## Review Findings

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve official control URLs in runtime validation:** `src/gateway/server-runtime-config.ts:155`
  - body: The config validator explicitly allows `https://controlplane.tailscale.com` with `serve`/`funnel`, but this runtime guard throws for any non-empty `controlUrl`. That makes a supported setup fail only at startup or via the CLI override path; gate this throw on `!isOfficialTailscaleControlServer(tailscaleConfig.controlUrl)` and cover the runtime path.
  - confidence: 0.92
- **[P1] Enforce the configured control URL in tailnet mode:** `src/gateway/server-runtime-config.ts:161-164`
  - body: The advertised Headscale path is `mode="off"` with `bind="tailnet"`, but the resolver never compares `gateway.tailscale.controlUrl` with the daemon's actual control server in that mode. A typo or stale value is silently ignored while the Gateway binds on the daemon's current tailnet, so either enforce the configured URL against a reliable source or remove the config/CLI surface and keep this as documentation.
  - confidence: 0.86
- **[P2] Redact daemon control URLs before throwing:** `src/gateway/server-runtime-config.ts:167`
  - body: This startup error includes `daemonControlUrl` verbatim. If a custom login URL contains userinfo or query tokens, the failed gateway start prints those details to normal terminal/log output; strip credentials, query, and fragment before including the URL in diagnostics.
  - confidence: 0.88
