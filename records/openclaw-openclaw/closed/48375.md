- **[P2] Allow official control URLs at runtime:** `src/gateway/server-runtime-config.ts:155-159`
  - body: Config validation now explicitly allows official Tailscale control URLs with `serve`/`funnel`, but runtime throws for any non-empty `controlUrl` before checking whether it is official.   - confidence: 0.91
- **[P2] Use a supported daemon source for the control server:** `src/gateway/server-runtime-config.ts:42-45`
  - body: The fast-fail path reads `Self.ControlURL`, `CurrentTailnet.ControlURL`, `Prefs.ControlURL`, and root `ControlURL` from `tailscale status --json`, but the published status shape does not establish those fields. A Headscale daemon with `controlUrl` unset can therefore fall through to the old opaque Serve/Funnel failure instead of the clear error this PR promises.
  - confidence: 0.82
- **[P2] Redact daemon control URLs before throwing:** `src/gateway/server-runtime-config.ts:167`
  - body: This error formats `daemonControlUrl` directly into startup output. If the local daemon or future source returns a URL with userinfo or query tokens, gateway startup logs would expose that value; sanitize the URL before including it in diagnostics.
  - confidence: 0.86
