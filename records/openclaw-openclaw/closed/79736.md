## Summary

Current main already implements third-party `api.on(...)` typed hook registration for the requested prompt-injection hooks; the reported v2026.5.7 behavior is superseded by main-only code and regression coverage.

## What This Changes

Allow non-bundled plugins to register typed prompt hooks such as `before_prompt_build`, `agent_turn_prepare`, and `heartbeat_prompt_contribution` through `api.on(...)`.

## Best Possible Solution

Keep the current generic Plugin SDK `api.on(...)` implementation and leave the separate legacy `registerHook(...)` compatibility question to https://github.com/openclaw/openclaw/issues/77285.

## Reproduction Assessment

No high-confidence failing reproduction remains on current main. Source inspection and regression coverage show non-bundled `api.on("before_prompt_build", ...)` registrations populate `registry.typedHooks` and are dispatched by the prompt path.

## Solution Assessment

Yes. The current-main fix uses the generic Plugin SDK registry path, not a bundled-plugin special case, and preserves the existing `allowPromptInjection` and `allowConversationAccess` policy boundaries.
