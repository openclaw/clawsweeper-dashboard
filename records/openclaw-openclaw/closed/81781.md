## Summary

Keep open. Current main still derives session titles from display name, subject, the first transcript user message, then session id/date fallback; the requested compaction-summary or later meaningful-message behavior is not implemented, and https://github.com/openclaw/openclaw/pull/81948 is an open unmerged implementation candidate that explicitly closes this issue.

## What This Changes

Request improved session `derivedTitle` generation so session pickers show conversation content using compaction summaries and/or later meaningful user messages instead of low-information first prompts.

## Best Possible Solution

Define a bounded derived-title contract that prefers existing metadata, then a stored compaction summary or clearly meaningful recent user prompt, with regression coverage for gateway listing, `sessions_list`, and the TUI picker.
