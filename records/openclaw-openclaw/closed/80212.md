## Summary

This should stay open: current main and the latest release still route proactive Telegram DM-topic sends and inbound DM-topic replies through different session-key shapes, so the cron continuity gap remains source-reproducible.

## What This Changes

The issue asks for cron-initiated Telegram DM-topic conversations to use a session key that matches the inbound reply routing key.

## Best Possible Solution

Align Telegram outbound direct-topic session routing for isolated DM scopes with the inbound `chatId:threadId` suffix, using the existing outbound routing seam rather than adding arbitrary session-key overrides or aliases.

## Reproduction Assessment

Yes for a source-level reproduction: current source shows inbound DM-topic replies use `:thread:<chatId>:<threadId>` while proactive outbound direct-topic routing can use only `:thread:<threadId>`. I did not run a live Telegram cron scenario because this review is read-only.

## Solution Assessment

Yes: the narrowest maintainable path is Option A, aligning the Telegram outbound route with the existing inbound key grammar. The broader cron session-key override and aliasing options are larger product surfaces and should not be needed for this report.
