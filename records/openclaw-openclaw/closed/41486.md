## Summary

Current `main` already implements the requested mitigation for isolated cron jobs: cron payloads support `toolsAllow`/CLI `--tools`, the executor forwards that allowlist into embedded runs, the embedded runner filters the tool list, and when a tool allowlist is present it also switches to a smaller minimal prompt and drops the skills catalog.

## Evidence

## Risks / Open Questions

- `lightContext` by itself still only trims injected bootstrap files; the prompt-size reduction path on current main is the explicit `toolsAllow`/`--tools` allowlist, not automatic context-budget detection.
