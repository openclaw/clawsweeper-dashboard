## Summary

Current `main` already exposes plugin-owned same-origin gateway HTTP routes and plugin gateway RPC methods, which cover the requested Mission Control proxy-path workaround without widening Control UI CSP.

## Evidence

- **Control UI CSP is intentionally same-origin only:** `buildControlUiCspHeader()` still emits `connect-src 'self' ws: wss:`; the supported way to keep browser fetches working is to serve them from the gateway origin rather than relaxing CSP.
  - file: [src/gateway/control-ui-csp.ts:35](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/src/gateway/control-ui-csp.ts#L35)

  - file: [src/plugins/types.ts:2037](https://github.com/openclaw/openclaw/blob/c1b9aa7d9630efe79f366b98564854aa935d00f4/src/plugins/types.ts#L2037)
