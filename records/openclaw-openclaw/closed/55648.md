## Summary

Current `main` already covers the reported stale non-main-agent auth failure: auth stores remain per-agent by design, but runtime loads merge main+agent auth, empty subagents inherit main auth on first writable load, and stale/expired OAuth in a subagent is replaced from main when a fresher main credential exists. The remaining on-disk file mismatch is expected design, not a current bug.

## Risks / Open Questions

- Hash mismatches across per-agent `auth-profiles.json` files are still possible by design, so file-diff checks alone can look alarming even when runtime auth resolution is correct.
- This review was repository/history based; direct `gh` hydration was unavailable in this environment because `GH_TOKEN` was not set.
