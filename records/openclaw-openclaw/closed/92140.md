## What This Changes

The issue asks OpenClaw to stop Codex `message_tool_only` Telegram DM turns from instructing the model to use `message` when the Codex app-server toolset does not expose that tool.

## Best Possible Solution

Keep the issue open and fix the Codex app-server delivery invariant so `message_tool_only` is only selected when `message` is truly model-visible on fresh and resumed threads, with any same-session `sessions_send` hardening handled as a separate explicit decision.
