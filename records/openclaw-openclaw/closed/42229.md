## Summary

Current main keeps `openai-codex/*` on the PI/OpenClaw-managed runtime by default, the bundled Codex app-server harness only claims the `codex` provider, and the OpenClaw tool construction path still creates and policy-filters the explicit `exec` tool. The reported `openai-codex/gpt-5.4` gateway case is therefore implemented on current main and is present in the latest release commit checked here.
