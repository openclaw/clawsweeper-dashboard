## Summary

Current main and the latest release already provide programmatic per-session model selection for WebSocket clients through the session-control RPC surface. The exact `chat.send.model` field was not added, but `sessions.patch`/`sessions.create` solve the observable dashboard workflow before the next `chat.send` run.

## Risks / Open Questions

- This closes on the basis that a two-step dashboard flow, `sessions.patch` then `chat.send`, satisfies the requested per-session model selection. Clients that specifically require a single `chat.send` payload with `model` would still need a new protocol field.
