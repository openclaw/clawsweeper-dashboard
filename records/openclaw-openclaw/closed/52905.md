## Summary

Current main implements the requested lightweight probe behavior for the affected status paths: `probeGateway` can skip full detail RPCs, `openclaw status` defaults to presence-only probing, and default `openclaw gateway status` uses connect-only probing. The full `status` RPC remains available only for explicit read/detail diagnostics.

## Best Possible Solution

Keep the shipped split: default status/connectivity checks stay lightweight, while explicit read/detail diagnostics can still exercise the full `status` RPC. If maintainers want additional scale work, open a narrower follow-up to paginate/cache/optimize the explicit Gateway `status` RPC and session summary for very large agent fleets.
