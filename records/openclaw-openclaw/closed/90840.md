## Summary

Keep open. Current main and v2026.6.5 still route the child `task_completion.result` through the direct-message text fallback when the parent announce run has no message-tool delivery evidence. The bug is source-reproducible, but the correct repair requires a maintainer decision about whether these completions should be silent under `message_tool_only` or gain an explicitly public parent-reply signal; the open related PRs address narrower `sessions_yield` behavior rather than this ordinary `sessions_spawn` reproduction.

## What This Changes

Fix direct-chat subagent completion routing so users receive the intended safe parent response instead of raw child output.

## Best Possible Solution

Define an explicit safe-to-deliver parent completion signal, then coordinate exactly-once direct-message delivery around that signal so raw child results remain internal whenever a parent completion run handles the event.

## Reproduction Assessment

Yes, at source level with high confidence. The supplied trace matches the current `:text-direct` fallback, which reads the child completion result and mirrors it externally when the parent announce run records no message-tool delivery.

## Solution Assessment

Unclear pending maintainer contract direction. Gating or deleting the fallback stops the raw-output leak but does not safely deliver the intended parent response; auto-delivering arbitrary parent final text has a separate privacy risk.
