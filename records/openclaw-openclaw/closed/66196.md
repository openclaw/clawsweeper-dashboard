## Summary

Keep this PR open. The WhatsApp prompt-boundary hardening goal is still relevant on current main, but the proposed patch is not merge-ready: it wraps legacy envelope `Body` text while the reply pipeline still prefers raw `BodyForAgent` for the model-facing prompt, and it also makes combined echo keys nondeterministic by putting a random wrapper into the dedupe key path.

## What This Changes

The PR imports `wrapExternalContent` into the WhatsApp monitor and wraps direct-message and group-history envelope body strings before passing them to `formatInboundEnvelope`.

## Best Possible Solution

Revise or replace the PR with a WhatsApp-specific prompt-boundary fix that wraps the actual model-facing user text, audio transcript, reply-context text, and visible group-history text while keeping trusted operator prefix/context outside the untrusted markers and keeping echo keys based on deterministic unwrapped content. Add focused WhatsApp regression tests for direct DMs, audio transcripts, reply context, group history, prefix handling, and duplicate suppression.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Wrap the field the agent actually reads:** `extensions/whatsapp/src/auto-reply/monitor/message-line.ts:43-47`
  - body: The patch wraps only the legacy envelope `Body`, but `processMessage` still passes raw `msgForAgent.body` as `BodyForAgent`, and session setup prefers `BodyForAgent` for `BodyStripped` before the prompt is built. Direct WhatsApp text and audio transcripts therefore remain trusted despite the advertised mitigation.
  - confidence: 0.93
- **[P1] Keep combined echo keys deterministic:** `extensions/whatsapp/src/auto-reply/monitor/message-line.ts:43-47`
  - body: `buildInboundLine` feeds `combinedBody`, which is used for WhatsApp echo dedupe and later remembered after sending. `wrapExternalContent` creates a fresh random marker id on each call, so duplicate deliveries of the same inbound message would no longer match the remembered combined key and could trigger duplicate replies.
  - confidence: 0.9
- **[P2] Keep trusted prefix text outside the untrusted block:** `extensions/whatsapp/src/auto-reply/monitor/message-line.ts:43-47`
  - body: `baseLine` includes the operator-configured `messagePrefix` and reply-context framing before the user body. Wrapping the whole string marks trusted prefix/context as untrusted, unlike the Discord reference that wraps only external message content.
  - confidence: 0.82
