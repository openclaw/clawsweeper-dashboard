## Summary

Current main already implements durable post-update restart reporting: `update.run` writes a persisted restart sentinel with session routing metadata before restart, gateway startup consumes it once after boot, and the restart-sentinel handler wakes and notifies the originating session/channel. Direct CLI `openclaw update` also waits through the restart and reports completion. The same implementation is present in the released `v2026.4.22` tag.
