## Summary

Keep open: current main and the latest release still leave code-span backticks on IDENTITY.md avatar values, and the avatar resolver can then surface the misleading unsupported image type state.

## What This Changes

The issue asks OpenClaw to strip markdown code-span backticks from IDENTITY.md field values so backticked avatar paths resolve normally.

## Best Possible Solution

Strip markdown code-span delimiters at the same parser boundary that already strips emphasis markers, then add regression coverage without weakening the avatar extension policy.

## Reproduction Assessment

Yes, source inspection plus a minimal current-parser expression reproduce the exact bad value and extension. I did not run the full gateway/UI path in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is to normalize IDENTITY.md markdown formatting before avatar resolution. Changing the avatar policy to accept malformed extensions would be the wrong layer.
