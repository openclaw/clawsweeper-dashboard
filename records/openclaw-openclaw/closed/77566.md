## Summary

Keep open: current main and the latest release still contain source-level paths that remove or omit Gemini tool-call thought signatures during cross-provider replay, and the overlapping open PR is not merged and appears to cover only part of the reported gap.

## What This Changes

The issue requests preserving or repairing Gemini `thoughtSignature` data across cross-provider subagent replay so Gemini tool-call flows do not hang or fail.

## Best Possible Solution

Preserve real Gemini signatures through replay boundaries, use Google's sentinel only for unsigned transferred function-call history, and add regression coverage for native Google plus OpenAI-compatible cross-provider replay.

## Reproduction Assessment

Yes, by source inspection rather than live execution: current main can serialize a Gemini-targeted assistant tool call without `thoughtSignature` after cross-provider or cross-API replay. That path matches the reporter's transcripts and Google's documented missing-signature failure mode.

## Solution Assessment

Yes on direction, but no complete fix is merged. The narrow maintainable solution is to repair the replay serialization/sanitization boundary and review or extend PR #75713 against the three identified drop points.
