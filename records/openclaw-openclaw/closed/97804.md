## Summary

This should close as plugin-owned work: the reported WeChat inbound image gap is in the external @tencent-weixin/openclaw-weixin runtime, while current OpenClaw core already exposes generic channel media/storage seams for plugins.

## What This Changes

The issue requests inbound WeChat image messages from @tencent-weixin/openclaw-weixin to be converted into standard Agent image attachments so multimodal handling can run.

## Best Possible Solution

Fix inbound image download/decrypt and media-fact injection in Tencent/openclaw-weixin, then publish through npm/ClawHub; open a narrow OpenClaw SDK issue only if that work exposes a missing generic channel media seam.
