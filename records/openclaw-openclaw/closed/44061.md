## Summary

Current `main` does not support the issue's top-level `compaction` config example, already performs prompt-budget prechecks before submission in safeguard mode, and documents that injected workspace/system-prompt context counts toward `/status` while compaction only summarizes conversation history. The reported reproduction path therefore does not demonstrate a current-main safeguard compaction bug.

## Close Comment

Closing based on a Codex review of current `main` (`50e36983`). The report’s reproduction does not match how compaction works today: safeguard mode is only read from `agents.defaults.compaction` (`src/agents/pi-embedded-runner/extensions.ts:72`, `docs/gateway/config-agents.md:532`), the runtime already does a prompt-budget precheck before submission and routes overflow into compaction (`src/agents/pi-embedded-runner/run/preemptive-compaction.ts:41`, `src/agents/pi-embedded-runner/run/attempt.ts:2260`), and `/status` context includes prompt-side usage such as injected workspace/system-prompt context (`src/status/status-message.ts:533`, `docs/concepts/context.md:10`). Compaction itself summarizes conversation history only and intentionally skips when there are no real conversation messages to compact (`src/agents/pi-embedded-runner/compact.ts:1003`, `docs/concepts/compaction.md:9`). Given the top-level config snippet in the report and the stated reproduction using large injected context, I could not verify a current-main safeguard bug from this issue as written. If you can still reproduce this on current `main` with `agents.defaults.compaction.mode: "safeguard"` and a session that has enough real conversation history to compact, please open a fresh issue with the exact nested config plus `/context detail` output and any `[context-overflow-precheck]` or `[compaction]` log lines.
