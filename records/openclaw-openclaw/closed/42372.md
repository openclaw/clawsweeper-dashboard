## Summary

Current main already preserves media-only TTS replies, flushes orphaned tool audio after `NO_REPLY`, and sends WhatsApp audio as PTT voice notes; the same implementation is present in the shipped `v2026.4.22` release snapshot.

Closing from a Codex review: current `main` already implements this path, and the same behavior is present in the shipped `v2026.4.22` snapshot (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). Based on current main and the released code, this specific WhatsApp-group TTS auto-delivery gap is already implemented on main.
