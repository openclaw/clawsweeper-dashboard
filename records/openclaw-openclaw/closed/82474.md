## Summary

Keep open: the PR is a focused improvement, but as written the new actionable message points operators at `agents.defaults.agentRuntime.id`, which current docs and runtime policy say is ignored. It also only provides mocked test proof, not after-fix behavior from a real gateway request.

## What This Changes

The PR adds an OpenAI-compatible unregistered-harness error mapper, wires it into chat-completions streaming and non-streaming handlers, and adds gateway regression tests.

## Best Possible Solution

Keep the 503 mapping, but point users at supported provider/model `agentRuntime` config and verify the response with a real `/v1/chat/completions` request before merge.

## Reproduction Assessment

Yes. Current source shows the exact harness-selection error path and the OpenAI-compatible chat-completions fallback to a generic 500; I did not run the live gateway image.

## Solution Assessment

No, not as written. Mapping this error to 503 is a reasonable narrow fix, but the proposed message directs users to an ignored runtime key instead of provider/model runtime policy.
