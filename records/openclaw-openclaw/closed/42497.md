## What This Changes

The branch adds a global in-process gateway dispatch registry, registers it from gateway startup, and makes `callGatewayTool` use it for embedded gateway tool calls with added unit coverage.

## Best Possible Solution

Replace this branch with a fresh narrow fix on current main that reuses `dispatchGatewayMethodInProcessRaw` or the scoped request context and preserves auth, scopes, timeout, expectFinal, and request-normalization semantics with real active-session cron proof.

## Reproduction Assessment

## Solution Assessment

Is this the best way to solve the issue? No. The maintainable path should integrate with current scoped in-process dispatch instead of adding a parallel global registry that skips existing gateway contracts.

Full review comments:

- **[P1] Preserve auth and scopes before local dispatch:** `src/agents/tools/gateway.ts:154-161`
  - body: The local-dispatch branch returns before `resolveGatewayOptions`, approval-runtime token setup, and least-privilege or caller-supplied scopes run. In an embedded gateway process, calls that should use an explicit token or constrained scopes can instead execute through the registered in-process dispatcher, changing auth-sensitive behavior.
  - confidence: 0.91
- **[P1] Keep caller timeouts on the in-process path:** `src/agents/tools/gateway.ts:156-160`
  - body: For calls without `expectFinal`, this branch never applies `opts.timeoutMs`; current callers such as approval waits and `node.invoke` pass timeouts specifically to bound stuck gateway work. If the in-process handler never responds, the active turn can hang instead of failing like the WebSocket path.
  - confidence: 0.88
- **[P2] Normalize request params like the WebSocket path:** `src/agents/tools/gateway.ts:157-160`
  - body: The new path forwards `params` directly into gateway method dispatch, so fields explicitly set to `undefined` are no longer removed by JSON serialization. Validators that accept omitted fields but reject `undefined` can fail for embedded calls that still work over WebSocket.
  - confidence: 0.8
