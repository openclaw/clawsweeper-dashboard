## Summary

Current `main` already removes plugin-owned `channels.<id>` config during plugin uninstall, including preview/reporting and regression coverage for external channel plugins. The fix is present on `main` but appears unreleased.

## Evidence

- **Uninstall now removes owned channel config:** `removePluginFromConfig` explicitly removes owned `channels` keys for installed plugins via `resolveUninstallChannelConfigKeys(...)`, which covers the reported stale `channels.<pluginId>` case.
