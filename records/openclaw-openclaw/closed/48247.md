## Summary

Current `main` already treats plugin manifests as authoritative during discovery/registry loading, and the bundled Brave/Google/Perplexity/xAI plugins all declare canonical manifest ids. On that codepath, the `*-plugin` package names in this PR do not produce a user-visible id-mismatch warning anymore, so the PR is obsolete against current `main`.

## Evidence

## Risks / Open Questions

- If a future install path omits `openclaw.plugin.json`, `deriveIdHint` still falls back to package-name normalization and only strips `-provider`, not `-plugin`. That is a residual edge case, but it is not how current `main` bundles these plugins.
