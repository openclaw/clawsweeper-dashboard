## Summary

Keep open. Current `main` has adjacent ACPX improvements and a newer bundled `acpx`, but I did not find code or test evidence that the specific post-spawn persistent-session failure `runtime: status=dead` with `summary: "queue owner unavailable"` is fixed. The issue is also recent and a maintainer comment from 2026-04-25 already leaves it open pending a fresh repro on current `main`/next release.
