## Summary

Keep open. Current `main` still uses the pre-PR restart-sentinel flow: it atomically consumes and deletes the sentinel before continuation delivery, has no durable `session-delivery-queue` implementation, and only recovers outbound deliveries at startup. The PR therefore is not implemented on `main`, and the latest review context also leaves a plausible unresolved ordering bug in the proposed recovery cutoff, so conservative maintainer cleanup should not auto-close it.

## Evidence
