## Summary

Current main already supports SCNet through the shipped Custom Provider flow for any OpenAI-compatible endpoint, so a dedicated built-in SCNet provider is not required to use `https://api.scnet.cn/api/llm/v1` with a bearer token.

## Close Comment

Closing this as implemented on current main after Codex review. OpenClaw already ships a generic Custom Provider path for any OpenAI-compatible endpoint, which matches the SCNet request (`https://api.scnet.cn/api/llm/v1` + bearer token + OpenAI-format API). The current docs say `Custom Provider` can connect "any OpenAI or Anthropic compatible endpoint, including hosted providers not listed" in `docs/cli/onboard.md:172` and expose that flow in onboarding in `docs/start/wizard.md:67`. The implementation in `src/commands/onboard-custom.ts:43` and `src/commands/onboard-custom-config.ts:364` prompts for base URL/API key, detects OpenAI compatibility, and saves the provider as `openai-completions` with the configured base URL and model. This shipped in release `v2026.4.22` at `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`. If the remaining ask is a dedicated SCNet-branded preset or one-click onboarding shortcut rather than API support itself, that would need a new, narrower feature request.
