## Summary

Keep open. Current main still has the key-existence based schema guard that can let typed object schemas with `properties: undefined`, `properties: null`, or another non-object value bypass parameter-free normalization, and PR #75401 is the open closing implementation path.

## What This Changes

Fix the regression where OpenAI GPT-5.4 rejects parameter-free MCP tool schemas after OpenClaw normalizes them without a valid top-level `properties` object.

## Best Possible Solution

Land PR #75401 or an equivalent central normalizer fix that treats top-level object schemas with absent, null, undefined, or non-object `properties` as parameter-free and adds regression coverage on the shared and OpenAI strict paths.

## Reproduction Assessment

Yes. A high-confidence source reproduction is passing `{ type: "object", properties: undefined }` or `{ type: "object", properties: null }` through `normalizeToolParameters` or `normalizeOpenAIStrictToolParameters`; current main does not rewrite those to a valid empty `properties` object.

## Solution Assessment

Yes in direction, but not yet on main. The narrow maintainable fix belongs in the shared schema normalizer rather than a Flux-specific workaround or provider transport special case.
