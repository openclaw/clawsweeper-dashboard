## Summary

Current `main` no longer recursively treats nested skill subdirectories as candidate skills. `openclaw skills` now loads workspace skill status through `loadWorkspaceSkillEntries`, which only considers the root skill folder or its immediate child directories and silently skips children without `SKILL.md`; warnings are reserved for real escaped/symlinked paths outside the configured root. That matches the bug report’s requested behavior closely enough to close as implemented on `main`.

## Evidence

- **Local loader is non-recursive:** `loadSkillsFromDirSafe()` reads either one root skill directory or the direct child directories returned by `readdirSync()`. It does not recurse into grandchildren, which prevents warning storms from flattened skill contents.
  - file: [src/agents/skills/local-loader.ts:96](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/agents/skills/local-loader.ts#L96)
