## Summary

Current `main` already filters iMessage `is_from_me=true` rows out of the inbound path before session dispatch, with explicit regression coverage for both normal DMs and self-chat echo cases; the changelog places the relevant iMessage echo/self-chat hardening in version `2026.4.10`, which is newer than the reported affected version `2026.4.1`.

## Evidence

- **Inbound gate drops non-self-chat from-me messages:** `resolveIMessageInboundDecision` checks `params.message.is_from_me` first and returns `{ kind: "drop", reason: "from me" }` for normal outbound rows, preventing them from being treated as inbound session traffic.
  - file: [extensions/imessage/src/monitor/inbound-processing.ts:230](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/extensions/imessage/src/monitor/inbound-processing.ts#L230)

  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **Dropped decisions are not dispatched:** `monitorIMessageProvider` calls `resolveIMessageInboundDecision(...)` and returns immediately on `decision.kind === "drop"`, so `from me` / echo cases never reach inbound dispatch.
