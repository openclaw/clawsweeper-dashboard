## Summary

Keep open. Current main and v2026.5.12 still leave the cache-trace and current Anthropic payload diagnostic JSONL sinks without diagnostic rotation/default caps, and https://github.com/openclaw/openclaw/pull/76407 is the open closing PR that should land or be replaced before this issue closes.

## What This Changes

Add default bounded rotation plus configurable max-size and archive retention for cache-trace and provider/Anthropic payload diagnostic JSONL files.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/76407 with bounded diagnostic JSONL rotation, schema/docs/tests, and focused runtime proof, while keeping gateway.err.log and journald rotation reports separate.

## Reproduction Assessment

Yes, at source level. Enabling diagnostics.cacheTrace or OPENCLAW_ANTHROPIC_PAYLOAD_LOG reaches writers that append through getQueuedFileWriter without diagnostic max-size or archive-retention options on current main.

## Solution Assessment

Yes, directionally. Extending the shared queued writer and wiring only the affected diagnostic sinks through config/env/schema/docs/tests is the narrow maintainable path, but the open closing PR must be rebased/reviewed or replaced before this can close.
