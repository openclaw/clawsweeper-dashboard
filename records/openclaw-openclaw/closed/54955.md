## Summary

Current main already supports cross-channel DM context sharing. `session.dmScope` can collapse DMs into one shared main session, and `session.identityLinks` can explicitly merge the same person across channels while keeping other users isolated. The docs, routing code, and tests all confirm this, and the same behavior is present in the shipped `v2026.4.22` release.

## Evidence

- **Session docs describe the feature:** The session docs state that direct messages are shared by default, list `per-peer` for cross-channel sender isolation, and recommend `session.identityLinks` when the same person contacts the agent from multiple channels.

## Risks / Open Questions

- This closes on the basis that the requested capability exists via configuration; it does not imply automatic cross-channel identity matching with zero setup.
- Group/channel conversations are still intentionally isolated unless separately configured; the evidence here is specific to DM/session sharing.
