## Summary

Keep open. Current main has stable/beta/dev release lanes, validation profiles, staged update hardening, and changelog-backed release notes, but it still does not expose an official per-release Production-ready, Early adopter, Known issues, or equivalent readiness marker; related release-stability discussions route the broader policy question here.

## What This Changes

The issue requests an official per-release production-readiness indicator in release notes, GitHub release metadata, or an update-facing metadata field.

## Best Possible Solution
