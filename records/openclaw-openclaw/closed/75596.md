## Summary

This should close as a ClawHub/plugin-boundary item: the requested streaming behavior is specific to the external WeChat channel plugin, while current OpenClaw core already provides the generic channel plugin and block-streaming seams needed for plugin-owned implementation.

## What This Changes

Add WeChat plugin support for sending partial AI responses while generation is still in progress instead of waiting for the completed reply.

## Best Possible Solution

Keep OpenClaw core channel-agnostic and have WeChat partial/block-reply behavior implemented and documented in the external `@tencent-weixin/openclaw-weixin` package or its ClawHub listing; only bring work back to core if the plugin maintainers identify a missing generic SDK seam.

## Reproduction Assessment

Not applicable as a feature request. The source review verified the ownership boundary and existing generic streaming seams; no live WeChat account test is needed to decide that plugin-specific send behavior is outside this repository.

## Solution Assessment

Yes. The narrow maintainable path is a WeChat plugin change using the existing channel runtime/outbound/block-streaming APIs, not a core OpenClaw change tied to `openclaw-weixin`.
