## Summary

Current `main` prevents `video_generate` from silently falling through from an explicit `model` override to configured/default video models. The old released code still appended fallback candidates after the override; current `main` short-circuits to the override alone, which matches the reported bug and removes the MiniMax fallback path for explicit Seedance requests.

## Evidence

- **Current main short-circuits valid media model overrides:** `resolveCapabilityModelCandidates()` now returns `[override]` immediately when `modelOverride` parses, so a valid explicit video model no longer falls through to configured/default or auto-provider fallback candidates.
  - file: [src/media-generation/runtime-shared.ts:181](https://github.com/openclaw/openclaw/blob/5376a4a5d6dd6332904481cf39be9537132b5c8a/src/media-generation/runtime-shared.ts#L181)

  - file: [src/agents/tools/video-generate-tool.ts:835](https://github.com/openclaw/openclaw/blob/a4fc6c2409f9c28e55e04e22c2e243c6900d66a3/src/agents/tools/video-generate-tool.ts#L835)
