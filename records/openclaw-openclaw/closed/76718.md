## Summary

Current main already addresses the QQ Bot disappearance path by treating QQ Bot as the external `@openclaw/qqbot` plugin, persisting install records for setup/login, and migrating previously bundled enabled QQ Bot installs during update; this appears fixed on main but not yet shipped in a release.

## What This Changes

The issue reports that QQ Bot stops running and disappears from the plugin list after upgrading to OpenClaw 2026.5.2 or restarting the Gateway.

## Best Possible Solution

Keep the current main migration and install-record persistence path, then let the next OpenClaw release ship the external QQ Bot package flow.

## Reproduction Assessment

No, not as a failing current-main runtime path. The report is source-reproducible as a 2026.5.2 release packaging/update issue, and current main has targeted code and tests for the missing external-plugin install/persistence path.

## Solution Assessment

Yes. Persisting installed external channel plugins and migrating previously bundled enabled plugins through the update path is the narrow maintainable fix for plugin-list visibility after restart.
