## Summary

Current `main` already addresses the actionable part of this report: TUI model changes are session-keyed, and the disappearing-reply path now flushes buffered assistant text into final chat events. The same-agent repro in the issue uses the shared default `main` session, so shared model state there is expected rather than cross-session bleed.

## Evidence

- **TUI docs define model changes as session overrides and default the session to `main`:** The TUI guide says sessions belong to the current agent, the model picker sets the session override, and `--session` defaults to `main` (or `global` in global scope).
  - file: [docs/web/tui.md:59](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/docs/web/tui.md#L59)
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **Blank TUI session input resolves to the agent's main session:** `resolveTuiSessionKey` returns `buildAgentMainSessionKey(...)` when no session is supplied, so two TUI windows on the same agent default to the same session key unless the user changes `--session` or `/session`.
  - file: [src/tui/tui.ts:132](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/tui/tui.ts#L132)
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **Model picker patches the active session key, not the whole agent:** The TUI model selector calls `client.patchSession({ key: state.currentSessionKey, model: value })`, which scopes model changes to the current session record.
  - file: [src/tui/tui-command-handlers.ts:113](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/tui/tui-command-handlers.ts#L113)
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **Gateway final chat events now flush buffered text before finalizing:** `emitChatFinal` flushes any throttled delta before broadcasting the final chat event and includes the final assistant message when buffered text exists.
  - file: [src/gateway/server-chat.ts:806](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/gateway/server-chat.ts#L806)
