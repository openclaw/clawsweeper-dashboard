## Summary

The reported `_skey` failure no longer matches current OpenClaw code or the latest shipped release. Current outbound and Telegram message-sent hook paths pass `params.sessionKeyForInternalHooks` directly, there is no `_skey` symbol in the repo, and existing tests cover the internal `message:sent` hook path with explicit session keys.
