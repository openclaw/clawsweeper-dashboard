## Summary

Current main no longer matches the PR premise. Isolated cron runs already pass the resolved delivery channel into embedded agent runs, so TTS tools are created with a bound `agentChannel`; the proposed fail-fast guard is therefore obsolete on current main and would also block valid channel-less `/tools/invoke` TTS calls.

## Evidence

- **Isolated cron now binds a message channel before the embedded run starts:** `executeCronRun` sets `messageChannel` from `params.resolvedDelivery.channel`, and `runEmbeddedPiAgent` is invoked with that `messageChannel`, plus the resolved target/thread context. That means isolated cron sessions with a resolvable delivery target are not channel-less on current main.

  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
