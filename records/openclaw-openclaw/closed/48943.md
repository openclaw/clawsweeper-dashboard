## Summary

Current main already implements the requested behavior: configured `typingMode: "never"` resolves into the reply run, disables all typing signal start paths, and Telegram only calls `sendChatAction` through those shared typing callbacks for normal inbound replies. The follow-up concern about 401 typing failures blocking delivery is also covered by the shared typing start guard swallowing start errors and the Telegram 401 backoff/suspension handler.
