## Summary

Keep open: current main still has a source-backed path where a short Gateway status timeout can mark Gateway health false and let noninteractive doctor repair auto-approve a restart of an already running service, contrary to the documented non-service repair boundary.

## What This Changes

The issue asks for `openclaw doctor --fix --non-interactive` to avoid SIGTERM/self-restart behavior when a healthy systemd-managed Gateway is already running.

## Best Possible Solution

Harden doctor Gateway health/daemon handling so an inconclusive status RPC cannot trigger noninteractive restart of an already running local service, while preserving expected-listener port suppression and adding focused regression plus Linux/systemd proof.

## Reproduction Assessment

No, not end-to-end for the SIGTERM symptom because this read-only review did not run a live Linux systemd Gateway. Source inspection gives a clear current-main path from 3000 ms status timeout to `healthOk=false` to an auto-approved running-service restart prompt.

## Solution Assessment

Yes. The narrow maintainable fix is in the existing doctor Gateway health/daemon path: avoid converting an inconclusive status timeout into a noninteractive running-service restart; the broader diagnostics drafts are adjacent, not the direct fix.
