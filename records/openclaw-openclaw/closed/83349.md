## Summary

## What This Changes

The issue reports that upgrade/doctor/plugin-state changes can leave the Codex harness unregistered while a live Telegram session silently falls back from GPT-5.5/Codex to Sonnet.

## Best Possible Solution

Keep the current fail-closed implementation on main, ship it in the next release, and track any remaining doctor auto-repair or session-rebind behavior as a separate narrower issue if it still reproduces.

## Reproduction Assessment

Yes. The original failure is source-reproducible from the old generic fallback path, and current regression tests now prove the missing Codex harness error stops before an Anthropic fallback can run.

## Solution Assessment

Yes. Failing closed inside the model-fallback boundary is the narrowest maintainable fix for the documented routing-integrity contract; plugin auto-install or session repair can remain separate recovery work.
