## Summary

Current main already preserves Telegram implicit reply threading for the reported user problem. The implementation records the triggering Telegram message id, auto-injects it as payload replyToId for replyToMode="all", and the Telegram send layer emits reply_to_message_id. The same implementation is present in v2026.4.24.
