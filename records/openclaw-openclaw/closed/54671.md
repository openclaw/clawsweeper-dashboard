## Summary

Keep open. Current main still has the Control UI pairing section that this PR is trying to improve: it only shows host-native `openclaw devices list` / `openclaw devices approve <requestId>` commands and still tells users to rerun `openclaw devices list`, while Docker-specific docs elsewhere use the `docker compose run --rm openclaw-cli ...` form.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to update or merge the docs change into `docs/web/control-ui.md`, aligning the Control UI pairing section with the Docker command pattern already documented in `docs/install/docker.md`, then run the docs check for the final branch.
