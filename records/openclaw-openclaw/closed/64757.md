## Summary

Current `main` does not have a matching OpenClaw core code path for the reported GBK write behavior: dreaming writes are explicitly UTF-8, the documented scheduler is a single dreaming sweep rather than separate `auto-log/auto-confirm` jobs, and since `v2026.4.15` the default dreaming output no longer writes Light/REM blocks into `memory/YYYY-MM-DD.md` at all.

## Evidence

- **Dreaming docs: current write surfaces:** Current docs describe dreaming as opt-in and disabled by default; machine state lives under `memory/.dreams/`, while human-readable output goes to `DREAMS.md` or `memory/dreaming/<phase>/YYYY-MM-DD.md`. That does not match the issue's description of Dreaming JSON being written into daily memory markdown as the normal current path.
  - file: [docs/concepts/dreaming.md:10](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/docs/concepts/dreaming.md#L10)
  - command: `nl -ba docs/concepts/dreaming.md | sed -n '1,28p'`
  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)
- **Dreaming docs: scheduler shape:** The current docs say `memory-core` auto-manages one cron job for a full dreaming sweep running `light -> REM -> deep`; the reported `auto-log + auto-dream + auto-confirm cron jobs` do not match the present OpenClaw dreaming surface.
  - file: [docs/concepts/dreaming.md:117](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/docs/concepts/dreaming.md#L117)
  - command: `nl -ba docs/concepts/dreaming.md | sed -n '116,132p'`
  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)
- **Inline and separate dream reports force UTF-8:** `writeDreamingMarkdown` reads existing inline content with `fs.readFile(..., "utf-8")` and writes both inline daily-note blocks and separate phase reports with explicit `"utf-8"` encoding.
  - file: [extensions/memory-core/src/dreaming-markdown.ts:74](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/extensions/memory-core/src/dreaming-markdown.ts#L74)
  - command: `nl -ba extensions/memory-core/src/dreaming-markdown.ts | sed -n '70,150p'`
  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)
- **Session corpus writes force UTF-8:** The session-corpus path mentioned in the issue is handled by `appendSessionCorpusLines`, which reads and appends `memory/.dreams/session-corpus/*.txt` using explicit `"utf-8"` encoding.
  - file: [extensions/memory-core/src/dreaming-phases.ts:650](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/extensions/memory-core/src/dreaming-phases.ts#L650)
  - command: `nl -ba extensions/memory-core/src/dreaming-phases.ts | sed -n '640,690p'`
  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)
- **Dreaming JSON state writes force UTF-8:** The JSON-backed short-term recall and phase-signal stores are serialized with `JSON.stringify(...)` and written via `fs.writeFile(..., "utf-8")`, so the current code does not rely on Windows default code pages for these artifacts either.
  - file: [extensions/memory-core/src/short-term-promotion.ts:848](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/extensions/memory-core/src/short-term-promotion.ts#L848)
  - command: `nl -ba extensions/memory-core/src/short-term-promotion.ts | sed -n '846,861p'`
  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)
- **Default storage no longer targets daily memory files:** Current dreaming code defaults `dreaming.storage.mode` to `"separate"`, and the changelog for `2026.4.15` says Light/REM phase blocks moved by default to `memory/dreaming/{phase}/YYYY-MM-DD.md` instead of `memory/YYYY-MM-DD.md`. That directly removes the main file surface named in the report for default configurations.
  - file: [extensions/memory-core/src/dreaming.ts:619](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/extensions/memory-core/src/dreaming.ts#L619)
  - command: `nl -ba extensions/memory-core/src/dreaming.ts | sed -n '610,626p'`
  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)
- **Release note for default output move:** `CHANGELOG.md` under `2026.4.15` records that Dreaming default storage changed from `inline` to `separate`, so daily memory files are no longer dominated by structured Light/REM phase output on current shipped builds.
  - file: [CHANGELOG.md:724](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/CHANGELOG.md#L724)
