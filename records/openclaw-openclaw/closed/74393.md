## Summary

Keep open. Current main still has the relevant bug surface because agent-end pending-media flushing does not compare pending generated media against media already committed by messaging tools, but this PR has the protected `maintainer` label and is currently dirty/unmergeable against `main`, so it needs maintainer review rather than conservative cleanup closure.

## What This Changes

The PR adds an agent-end helper that removes pending generated media URLs already recorded as sent by a messaging tool, wires it before pending-media flush, and adds focused unit/lifecycle regressions.

## Best Possible Solution

Rebase and adapt the PR onto current main so agent-end pending-media flushing subtracts URLs already delivered by successful messaging tools while preserving orphaned media delivery, no-block-callback retention, and trusted-media metadata semantics; then run the focused agent tests and the changed gate.
