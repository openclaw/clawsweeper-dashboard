## Summary

This is a good integration request, but it belongs with the external ClawHub WPS/kdocs skill and the upstream WPS Global API, not OpenClaw core. Current main has no bundled kdocs/WPS implementation, and the documented OpenClaw boundary already provides the skill/plugin surfaces needed for this kind of optional service integration.

## What This Changes

Request support for WPS Office Global wps.com authentication and endpoints in the external kdocs/WPS skill and kdocs-cli flow.

## Best Possible Solution

Keep OpenClaw core unchanged and route WPS Global support to the external ClawHub WPS/kdocs skill or a new external plugin once Kingsoft exposes a usable global API. A new OpenClaw issue would be appropriate only if that external implementation identifies a specific missing skill/plugin API in core.
