## What This Changes

The issue reports that session-memory writes `memory/YYYY-MM-DD-<slug>.md` files while Dreaming and rem-backfill only discover `memory/YYYY-MM-DD.md` daily notes.

## Best Possible Solution

Review and land the focused reader-side matcher fix in https://github.com/openclaw/openclaw/pull/81094, preserving the session-memory writer and avoiding a new config surface, then close this issue after merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a file such as `memory/2026-04-21-ai-assistant-config.md` and run `openclaw memory rem-backfill --path ./memory`; the current date-only matchers skip it before ingestion.

## Solution Assessment

Yes. Accepting bounded `YYYY-MM-DD-<slug>.md` files in the readers is the narrowest maintainable fix because changing only the writer or adding a naming config would leave existing slugged artifacts invisible.
