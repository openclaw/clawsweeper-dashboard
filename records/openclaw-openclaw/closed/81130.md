## Summary

Current main already handles `/models` and `/model` as control-plane command/session operations before any agent inference, so a rate-limited active provider should not block manual model browsing or selection.

## What This Changes

Report that `/models` and related model-switching commands become unusable when the active provider is rate-limited.

## Best Possible Solution

Keep the existing control-plane command and session override path; reopen only with a fresh current-release reproduction that identifies a remaining channel-specific bypass failure.

## Reproduction Assessment

No: I do not see a current-main failing path to reproduce. Source inspection shows `/models` stops in command handling and `/model` writes session state directly before any provider inference call.

## Solution Assessment

Yes: the current direct command/session-store implementation is the narrow maintainable solution because model selection is handled in the control layer and not by the active provider.
