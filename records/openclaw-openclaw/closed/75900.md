## Summary

Close as implemented on current main. The current package/update path removes the legacy bundled plugin `plugin-runtime-deps` state, runs post-install doctor repair before restart, and treats activated plugin load errors as restart failures instead of letting channel workers crash downstream.

## What This Changes

The issue reports that the 2026.4.29 macOS update/rollback path left a stale versioned bundled plugin runtime dependency tree missing `json5`, causing Gateway, Telegram, and browser plugin startup failures.

## Best Possible Solution

Keep the current mainline package/update cleanup behavior and ship it in the next release so affected 2026.4.29 rollback installs move off stale legacy runtime-deps state.

## Reproduction Assessment

Yes. The report gives a concrete macOS package update/rollback path plus the missing `json5` runtime-deps error; the current-main verification here is source, docs, and e2e assertion evidence rather than a live macOS LaunchAgent replay.

## Solution Assessment

Yes. Retiring and cleaning the mutable legacy bundled plugin runtime-deps tree before package restart is narrower and more maintainable than channel-specific fallback installs, and restart health now blocks activated plugin errors.
