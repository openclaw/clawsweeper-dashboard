## Summary

Keep open. Current main still has the reported iMessage echo-cache behavior: the text fallback TTL is 4 seconds, text normalization only converts CRLF and trims, and the focused test suite still asserts that text fallback misses after 6 seconds. The related fix PR #59980 was closed unmerged, while open PR #62191 appears to cover only the leading U+FFFD/control-character normalization part, not the full delayed-echo TTL issue.

## Best Possible Solution

Keep this issue open until the iMessage owner path gets a complete fix: extend or otherwise redesign the text fallback window with a false-positive assessment, strip leading replacement/control characters in the sent-message and self-chat normalization paths, and add focused regression tests for delayed 5-10 second corrupted-prefix DM reflections. #62191 can be used as part of that work, but the TTL behavior also needs to be addressed or explicitly rejected by maintainers.
