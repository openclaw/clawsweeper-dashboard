## Summary

Close as implemented. Current main and shipped v2026.4.27 now handle plugin-local config validation failures as degraded gateway startup with a skipped/error plugin entry plus doctor quarantine, so the reported single-plugin config crash-loop is no longer current worker behavior.

## What This Changes

The issue asks for invalid per-plugin runtime config to disable or skip only that plugin during worker startup, keep other plugins and channels online, and expose a doctor recovery path.

## Best Possible Solution

Keep the shipped v2026.4.27 behavior: gateway startup should continue in degraded mode for plugin-local validation failures, leave the bad plugin unregistered, and rely on doctor --fix to persistently quarantine the invalid entry.
