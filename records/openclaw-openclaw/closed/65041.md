## Summary

Current main and release v2026.4.24 already remove the reported stale `lastGood` credential-selection path: runtime auth scheduling state is split out to `auth-state.json`, `lastGood` is not prioritized for auth-profile ordering, billing-disabled profiles are surfaced/skipped through explicit usage state, and `models auth login` clears stale billing/auth lockouts before re-authentication.

## Best Possible Solution

Close the stale `lastGood`/billing fallback bug as implemented. Keep the shipped split between secret-bearing auth profiles and `auth-state.json`, the non-prioritized `lastGood` ordering, billing-disabled surfacing, and login-time lockout clearing. A dedicated `openclaw auth rotate` or BWS-specific recovery UX would be a separate enhancement if maintainers want that workflow.
