## Summary

Keep open for review purposes: live GitHub now shows this PR was merged into `main`, but the merged patch still only recognizes top-level/provider API stream identities and misses the `createAgentSession` anonymous default wrapper that OpenClaw passes to the resolver.

## What This Changes

This PR classifies `streamSimple` and `getApiProvider(api).stream`/`streamSimple` identities as default PI streams, adds a resolver test, and updates the changelog.

## Best Possible Solution

Add coverage for the real `createAgentSession` default stream and classify that SDK-created default as boundary-aware without changing explicit provider-owned or custom session stream precedence.

## Reproduction Assessment

Yes, source-level: `createAgentSession` installs an anonymous default wrapper, and OpenClaw passes the cached `session.agent.streamFn` into `resolveEmbeddedAgentStreamFn`. The PR's helper still returns false for that wrapper.

## Solution Assessment

No. The PR fixes a narrower API-provider identity case, but the production default-session path receives the SDK wrapper, so the safer fix is to mark or detect that default wrapper explicitly and test the session-created path.
