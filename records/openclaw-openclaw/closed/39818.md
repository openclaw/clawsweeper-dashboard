## Summary

Current `main` already surfaces model fallback state across runtime, UI, status, and logs. The reply runner emits fallback lifecycle events and verbose inline notices, `/status` prints the active fallback model and reason, Control UI shows fallback toasts, and the fallback path logs warn-level decision details. The same implementation is present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
