## Summary

Close as implemented on current main: the local web search CLI now resolves command-scoped web search SecretRefs before provider execution, and the resolver path supports exec SecretRefs. The reported release, v2026.5.12, predates this current-main fix.

## What This Changes

The issue asks for direct CLI web search to resolve exec SecretRefs configured at plugins.entries.xai.config.webSearch.apiKey before running the Grok provider.

## Best Possible Solution

Keep the current command-scoped SecretRef resolver path and ship it in the next release so users on v2026.5.12 receive the fix.

## Reproduction Assessment

No current-main failure reproduction is needed for closure: v2026.5.12 source shows the raw-config CLI path, while current main source and tests show command SecretRefs are resolved before provider execution. The issue's release reproduction is consistent with that diff.

## Solution Assessment

Yes. The current implementation reuses the shared command/gateway SecretRef resolution path and selected-provider scoping instead of adding an xAI-only special case.
