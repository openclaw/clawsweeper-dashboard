## Summary

Current `main` already uses the same skill-loading path for `~/.agents/skills` in both prompt-visible `available_skills` generation and skill slash/native command registration. The only remaining case where a skill can register a command but stay out of `available_skills` is the documented `disable-model-invocation: true` contract, which is intentional.
