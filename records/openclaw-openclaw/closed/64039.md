## Summary

Current main already implements the Telegram default final-reply delivery behavior this issue asks for: text finals finalize or retain the visible preview, fall back to normal sends when preview finalization is unsafe, and cleanup no longer clears completed previews. The behavior is documented and covered by targeted Telegram delivery tests, and the relevant finalization path is present in the v2026.4.24 release tree as well as current main.

## Best Possible Solution

Close this older Telegram disappearance report and keep the shipped lane-finalization behavior. If a new disappearance is observed on current OpenClaw, it should be filed as a fresh issue with Telegram debug logs, streaming mode, whether draft transport or sendMessage/editMessageText was used, and the affected provider/model.
