## What This Changes

The issue requests that provider catalog discovery avoid sending unauthenticated `GET /models` requests when a resolved provider `apiKey` is only a non-secret auth marker.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83462 or a replacement that skips only non-runnable marker-backed discovery, preserves env/SecretRef catalog behavior, covers the self-hosted helper path, and adds focused regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: resolve a provider key to a truthy non-secret marker, let `toDiscoveryApiKey()` reduce it to `undefined`, and catalog discovery can call `/models` without `Authorization`.

## Solution Assessment

Yes for the issue direction, but the linked implementation still needs careful review. The narrow maintainable fix is to skip marker-only discovery without suppressing valid env/SecretRef-backed catalog entries and to cover the self-hosted helper path.
