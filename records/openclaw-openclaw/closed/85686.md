## Summary

Keep open. Current main has structured cron denial handling and denied-exec follow-up suppression, but it does not yet have a fixed-wrapper cron payload or an existing invariant that prevents a model-mediated agentTurn from choosing diagnostic command variants; the paired open feature request at https://github.com/openclaw/openclaw/issues/85685 should be decided before locking this down with regression coverage.

## What This Changes

The issue requests regression coverage for isolated cron agentTurn wrapper jobs that drift into model-selected diagnostic exec variants after an approval denial.

## Best Possible Solution

Decide the fixed-wrapper cron payload contract in https://github.com/openclaw/openclaw/issues/85685, then add focused regression tests for command immutability, structured denial state, delivery suppression, and run-history metadata.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection does show the current isolated cron path remains model-mediated through agentTurn and lacks a fixed-wrapper command contract, so the reported drift is plausible but not yet a narrow current-main bug repro.

## Solution Assessment

No for a test-only change today. The maintainable path is to first decide whether OpenClaw should add a fixedWrapper/localExec cron payload, then write tests against that explicit contract rather than against prompt-only agent behavior.
