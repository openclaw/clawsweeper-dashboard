## Summary

Current main and v2026.6.6 now solve the fixed-wrapper cron drift problem with first-class command cron payloads, so a separate test-only tracker for model-rewritten wrapper approval loops is no longer necessary.

## What This Changes

The issue requests regression coverage for isolated scheduled wrapper jobs where approval denial lets the model rewrite the command, self-debug, and produce noisy or misleading cron output.

## Best Possible Solution

Keep the shipped command cron payload as the supported fixed-wrapper path, and leave remaining model-mediated agentTurn status-classification bugs to the narrower open cron outcome trackers.

## Reproduction Assessment

No current-main failing reproduction remains for the fixed-wrapper path I checked. Current source and shipped docs show command cron jobs execute configured argv directly without a model turn that can rewrite the wrapper.

## Solution Assessment

Yes. A dedicated command cron payload is the best solution for deterministic wrappers because it removes the model from the command-spelling path while preserving cron history, output capture, delivery modes, and failure accounting.
