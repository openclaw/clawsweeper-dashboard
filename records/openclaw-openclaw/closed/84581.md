## Summary

Needs changes before merge: the previous inherited-profile blocker is addressed, but the latest branch still has a credential migration regression that can turn an env-shaped plaintext key into an unresolved env SecretRef and erase the original stored value from models.json.

## What This Changes

The PR strips plaintext provider apiKey values from generated and legacy models.json, migrates eligible legacy catalog keys into auth-profiles.json, preserves non-secret markers, and adds regression coverage.

## Best Possible Solution

Keep the prompt-facing catalog stripped, but only migrate legacy catalog values to env-backed keyRef when there is a reliable env-marker or env-presence signal; ambiguous env-shaped values should remain preserved as local auth-profile keys.

## Reproduction Assessment

Yes, by source inspection. The PR uses isValidEnvSecretRefId to classify legacy catalog values, and current main defines that as a broad env identifier regex rather than proof that the value is an env marker.

## Solution Assessment

No, not yet. The overall stripping direction is right, but ambiguous env-shaped values need a safer migration rule so real keys are not converted to missing env references.
