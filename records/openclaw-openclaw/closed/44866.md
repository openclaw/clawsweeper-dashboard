## Summary

Current main already stamps Feishu inbound DMs with channel/origin `feishu`, the reply runner prefers that originating channel over relayed `webchat` metadata, and session-routing guards preserve external channel routing instead of letting webchat steal it. The same implementation is present in release `v2026.4.22`.
