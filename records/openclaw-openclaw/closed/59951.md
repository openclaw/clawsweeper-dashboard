## Summary

Current `main` already has Telegram inbound dedupe keyed by `update_id` and fallback `chat_id:message_id`, plus webhook timeout behavior that returns early enough to prevent the retry pattern described in the report. The shipped `v2026.4.23` release contains those protections and regression tests.

## Evidence
