## Summary

Close as `clawhub`: the reported `(unconfigured)` line is owned by the external Lossless Claw plugin’s startup banner, not by OpenClaw core on current `main`. Current core docs and tree treat `lossless-claw` as a community-installed plugin, and the current loader passes validated plugin config on normal registration paths; the only empty `pluginConfig` path is a channel setup-only/setup-runtime branch that does not apply to a normal context-engine plugin startup.

## Evidence

- **Lossless Claw is documented as a community plugin with its own repo/package:** The community plugins doc lists Lossless Claw separately from bundled extensions and points users to install `@martian-engineering/lossless-claw` from its own repository.
  - file: [docs/plugins/community.md:64](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/docs/plugins/community.md#L64)
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Context-engine docs describe lossless-claw as an externally installed plugin:** The context-engine guide tells users to install `@martian-engineering/lossless-claw` and then enable it via `plugins.entries["lossless-claw"]`, which matches third-party plugin ownership rather than bundled core code.
  - file: [docs/concepts/context-engine.md:30](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/docs/concepts/context-engine.md#L30)
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Project vision keeps optional capability in plugins, not core:** `VISION.md` states core should stay lean, optional capability should usually ship as plugins, plugin discovery/review belongs in ClawHub, and plugin authors should host and maintain plugins in their own repositories.
  - file: [VISION.md:52](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/VISION.md#L52)
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Current bundled extensions tree does not contain a lossless-claw implementation:** A tree scan of `extensions/` on current `main` shows many bundled plugins but no `extensions/lossless-claw`, so there is no in-repo plugin code here to fix or verify for this startup banner.
  - command: `find extensions -maxdepth 2 -type d -name 'lossless-claw' -o -type f -name 'openclaw.plugin.json'`
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Normal plugin registration passes validated plugin config:** On the normal plugin registration path, `createApi(...)` receives `pluginConfig: validatedConfig.value`, so current core does pass plugin-specific config into plugin `register(api)` during normal startup.
  - file: [src/plugins/loader.ts:2721](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/plugins/loader.ts#L2721)
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **CLI metadata path also passes validated plugin config:** The separate CLI metadata load path builds the plugin API with `registrationMode: "cli-metadata"` and `pluginConfig: validatedConfig.value`, which further reduces the case for a core-wide config-loss bug here.
  - file: [src/plugins/loader.ts:3144](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/plugins/loader.ts#L3144)
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **The only empty pluginConfig path is limited to setup-only/setup-runtime channel setup loading:** The `pluginConfig: {}` branch is guarded by `registrationMode === "setup-only" || "setup-runtime"` plus `manifestRecord.setupSource`; SDK docs describe those modes as lightweight setup/channel-entry paths rather than normal full startup.
  - file: [src/plugins/loader.ts:2447](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/plugins/loader.ts#L2447)
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
