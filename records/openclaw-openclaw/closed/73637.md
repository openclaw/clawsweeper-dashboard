## Summary

Keep this PR open. Current main still has the three prompt assembly paths where `systemPromptOverride` short-circuits the builder and drops `extraSystemPrompt`, while subagent spawn still uses `extraSystemPrompt` to carry the task-bearing `## Your Role` block. The PR remains a plausible implementation candidate for open bug #73624, but needs maintainer review on prompt-authority/security and docs semantics before landing.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to preserve subagent `extraSystemPrompt` across the embedded attempt, compaction retry, and CLI harness prompt assembly paths; add regression coverage for no override, override-only, and override-plus-extra behavior; then choose whether this combined-system-prompt fix is acceptable or whether a separate lower-priority message contract should be threaded through providers. If the PR lands, update `docs/gateway/config-agents.md` so `systemPromptOverride` semantics match shipped behavior.
