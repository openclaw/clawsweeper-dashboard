## Label Justifications

- `P1`: The latest Docker release regressed browser/CDP startup for users running Chromium as the default `node` user.
- `impact:crash-loop`: The permission regression makes Chromium abort before the CDP health endpoint can bind.

## Summary

Current main already fixes the reported Docker image ownership regression by creating `/home/node/.config` as `node:node` before the OpenClaw config leaf directory. The latest release image `v2026.5.22` is still affected, but the fix is present on main and recorded as unreleased release-note work.

## What This Changes

The issue reports that the v2026.5.22 container image leaves `/home/node/.config` root-owned, causing headless Chromium Crashpad startup failure as the `node` user.

## Best Possible Solution

Keep the current Dockerfile ownership fix and let the next image release carry the corrected `/home/node/.config` permissions.

## Reproduction Assessment

Yes. The issue includes a concrete Docker repro against `v2026.5.22`, and the release Dockerfile shows the exact parent-directory creation pattern that leaves `/home/node/.config` root-owned.

## Solution Assessment

Yes. Current main uses the narrowest maintainable fix: create `/home/node/.config` with `node:node` ownership first, then create the private OpenClaw leaf directory and assert both permissions.
