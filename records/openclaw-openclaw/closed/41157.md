## Summary

Keep open. Current `main` still appears to send `store: false` for Google Gemini `openai-completions` requests on `generativelanguage.googleapis.com`, which matches the reporter's root-cause comment rather than showing a fix on `main`. The Bearer-header path also exists only as an explicit `authHeader: true` opt-in, so the original title is not clearly resolved by default either.
