## Summary

Current `main` already supports binding auth-profile selection per agent well enough via per-agent auth-order overrides stored in each agent’s `auth-state.json`, and auth resolution prefers that per-agent order over global `auth.order`. Agent state is also isolated by default per agent directory, with validation blocking shared `agentDir` setups that would cause auth collisions.

## Risks / Open Questions

- The implementation uses per-agent `auth-state.json` / CLI overrides instead of the exact proposed `agents.list[].auth` config shape, so users expecting a single `openclaw.json` field may not discover it immediately.
- `models auth login` in current main still resolves the default agent store rather than taking an explicit `--agent`, so setting up credentials for a non-default agent may still require copying profiles or using other agent-scoped auth/secrets flows.
