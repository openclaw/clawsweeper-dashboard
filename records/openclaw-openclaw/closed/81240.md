## Summary

The reported Slack channel symptom matches the documented group/channel visible-reply policy, not a missing Slack outbound dispatch path: normal final assistant text is intentionally private unless the model uses `message(action=send)` or the operator opts back into automatic room replies.

## What This Changes

The issue reports that Slack channel and DM session replies are generated with delivery context but are not posted back through Slack `chat.postMessage`.

## Best Possible Solution

Keep the existing tool-only group/channel default and have operators set `messages.groupChat.visibleReplies: "automatic"` when they want legacy automatic Slack room posts.

## Reproduction Assessment

Not applicable as a bug reproduction: current source and docs already define this Slack channel result as expected when group/channel visible replies are in `message_tool` mode. The separate DM aside lacks enough focused data to reproduce as part of this report.

## Solution Assessment

Yes. The narrow existing solution is configuration or message-tool delivery, not changing Slack `chat.postMessage` dispatch for normal final replies.
