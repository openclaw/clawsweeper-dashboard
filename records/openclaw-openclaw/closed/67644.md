## Summary

Keep open. Current `main` and the latest shipped release still use a blanket `$VAR` regex for Python/Node script preflight, so the reported false positive for `$` tokens inside Python string/comment text is not implemented on `main` and not obsolete.

## Evidence

- **Current main still matches `$VAR` anywhere in script content:** `validateScriptFileForShellBleed` on current `main` runs `/\$[A-Z_][A-Z0-9_]{1,}/g` directly against the full file content and throws on the first match, with no parser/range skipping for Python strings or comments.
