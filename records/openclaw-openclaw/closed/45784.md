## What This Changes

This PR adds an explicit OPENCLAW_HOME branch in src/utils.ts so resolveConfigDir returns the resolved OPENCLAW_HOME directory directly instead of appending .openclaw.

## Best Possible Solution

Close this stale branch and keep https://github.com/openclaw/openclaw/issues/45765 as the canonical fix path; the correct repair should preserve parent-home semantics and update both config and state resolution with focused tests.

## Reproduction Assessment

Yes, at source level. Current main preserves explicit OPENCLAW_HOME and then both config/state helpers append .openclaw when direct overrides are absent, while this PR's regression is visible from the diff changing every explicit home to a direct config root.

## Solution Assessment

No. Returning every explicit OPENCLAW_HOME directly is broader than current docs/tests and leaves state-dir resolution inconsistent; a narrow known-state-directory basename guard in both helpers is the safer fix shape.
