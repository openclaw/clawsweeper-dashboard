Full review comments:

- **[P1] Keep app-url configs without add-on principals parseable:** `src/config/zod-schema.providers-googlechat.ts:105`
  - body: This check rejects every `audienceType: "app-url"` config without `appPrincipal`, but current docs show that shape and current auth tests accept Chat issuer app-url tokens without an add-on principal.   - confidence: 0.91
- **[P1] Inherit appPrincipal when validating account overrides:** `src/config/zod-schema.providers-googlechat.ts:121`
  - body: The account loop inherits `audienceType` from the parent but then checks only `account.appPrincipal`. Runtime account resolution merges top-level and `accounts.default` fields into the selected account, so a valid account override that inherits `appPrincipal` will now fail schema parsing.
  - confidence: 0.9
