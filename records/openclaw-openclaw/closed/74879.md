## Summary

Keep open: the reporter's latest comment is useful evidence that recent releases feel faster, and current main does include a channel-selection mitigation, but the shell-completion/onboarding path is still not fully resolved in source or in the latest shipped tag. Manual completion still loads plugin CLI commands by default, while onboarding and doctor can still wait on the 30-second completion-cache child process.

## What This Changes

The issue reports a regression where Docker onboarding and shell-completion cache generation became much slower in OpenClaw 2026.4.26 and newer.

## Best Possible Solution

Keep the channel-selection improvement, then either land or replace the bounded completion-cache work and explicitly decide whether automatic onboarding/doctor refreshes should skip full plugin-command completion while preserving a separate full manual rebuild path.

## Reproduction Assessment

Yes for the shell-completion portion: current source and the latest shipped tag still route onboarding/doctor through completion-cache generation, and manual write-state still loads plugin commands unless an internal env flag is set. The channel-selection stall is less certain because current source has a deferred-status mitigation and the latest reporter comment says recent versions are faster.

## Solution Assessment

Unclear because there is no patch on this issue and the related open PR only addresses update/doctor timeout length and progress. The narrow maintainable direction is to separate automatic onboarding/doctor cache refresh from full plugin-command completion loading, then verify Docker onboarding timing before closing.
