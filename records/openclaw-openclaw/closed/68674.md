## Summary

The reported behavior is already implemented in the shipped codebase: OpenClaw v2026.4.23 fans managed Dreaming across configured agent workspaces, includes unit coverage for a two-agent fan-out case, and documents that automatic runs apply across configured memory workspaces. The linked #67413 remains the broader per-agent configuration discussion, but this specific bug no longer matches current shipped behavior.
