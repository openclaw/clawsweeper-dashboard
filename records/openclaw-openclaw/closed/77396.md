## Summary

Close as implemented: current main and release v2026.5.3 already encode active-memory transcript agent directory names and avoid colon-bearing channel IDs for embedded recall, so QQ c2c IDs should no longer hit bundled plugin dirName validation.

## What This Changes

The issue requests that active-memory safely handle QQ c2c agent IDs containing colons when deriving bundled plugin transcript agent directory names.

## Best Possible Solution

Keep the shipped v2026.5.3/current-main behavior; if a different QQ path still fails on v2026.5.3 or newer, track it with fresh logs showing which value is still passed as a bundled plugin dirName.

## Reproduction Assessment

No high-confidence current-main failing reproduction exists: source inspection shows current main and v2026.5.3 encode the reported unsafe path segment and avoid colon-bearing channel IDs in the embedded recall call. The original report is consistent with older 2026.5.2 behavior, not the current release.

## Solution Assessment

Yes; encoding the active-memory transcript path segment while keeping bundled plugin dirName validation strict is the narrow maintainable fix. The adjacent embedded-recall channel guard also addresses the same validation failure mode without weakening plugin loader safety.
