## Summary

Current `main` already preserves structured tool-result media for embedded replies by extracting `details.media`, queueing it as pending tool media, and flushing it into outbound block replies at turn end. The PR’s reported user-visible drop before outbound delivery does not match current behavior on `a6aa151653ab478e9aa23e95c6446c41adc4fce1`.

## Evidence
