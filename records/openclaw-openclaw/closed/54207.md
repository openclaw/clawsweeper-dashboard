## Summary

Close as a duplicate of the open canonical report #31331. Current main still shows the same Docker-outside-of-Docker bind-source path flow, but #31331 is already labeled as the parent for this cluster and covers the identical `/home/node/.openclaw/sandboxes...` host/container namespace mismatch.

## What This Changes

The issue reports that Docker sandbox creation under Docker Compose/DooD uses gateway-container paths such as `/home/node/.openclaw/sandboxes` as Docker bind sources instead of host-visible paths, causing Docker Desktop on macOS to reject the mount.

## Best Possible Solution

Track and fix the Docker gateway sandbox bind-source contract under #31331, folding this issue's macOS Docker Desktop evidence into the canonical acceptance criteria and regression coverage.
