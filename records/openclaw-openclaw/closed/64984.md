## Summary

Current main implements the MCP lifecycle cleanup needed for #64984: session-scoped bundled MCP runtimes are retired on session lifecycle paths, gateway shutdown, MCP config hot reload, and idle expiry; stdio MCP transports now reap their spawned process trees on close. The related #70419 was only an OOM victim-selection mitigation and explicitly left #64984 as lifecycle work, while the later #70808 maintainer discussion closed the same lifecycle leak as fixed on main.

## Best Possible Solution

Close #64984 as implemented on main. Keep the current lifecycle model: session-scoped MCP runtimes are explicitly retired on session/reset/rollover, gateway shutdown, MCP config reload, one-shot run end, and idle expiry, while stdio transport close owns process-tree teardown. Any future dead-client self-healing after an externally killed MCP child should be tracked as a narrower follow-up rather than keeping this process-leak issue open.
