## Summary

Current main implements provider wildcard expansion for `agents.defaults.models` across configured model visibility; the v2026.5.7 behavior reported here was fixed after that release and is now covered by source and regression tests.

## What This Changes

Request for `agents.defaults.models` entries like `opencode-go/*` to expand to that provider's catalog in `/model`, `/models`, and `openclaw models list` instead of surfacing a literal wildcard row.

## Best Possible Solution

Keep the current main implementation and let it ship in the next release so provider wildcard allowlists work consistently across CLI, Gateway, and picker surfaces.

## Reproduction Assessment

Yes. The issue body provides concrete v2026.5.7 steps and output, and current-main source inspection traces the fixed `/models` and `models.list` path; I did not run the CLI locally.

## Solution Assessment

Yes. The current fix is the narrow maintainable path: parse only literal `provider/*` entries, load a real catalog when those entries exist, and filter visibility by provider instead of treating the wildcard as an exact model.
