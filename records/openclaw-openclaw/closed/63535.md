Full review comments:

- **[P1] Avoid age-only recovery of active cron runs:** `src/cron/session-reaper.ts:106-109`
  - body: This branch finalizes any `running` cron row older than two hours as `done`, but current timeout policy supports long and explicitly no-timeout cron work. An active run can therefore be marked terminal and later pruned while it is still executing.
  - confidence: 0.93
- **[P1] Update the visible cron session row:** `src/cron/session-reaper.ts:106`
  - body: The recovery branch writes only the matched `:run:` alias. Isolated cron state is persisted under the stable `agent:...:cron:<job>` key and session listing hides run aliases, so the user-visible row can remain stuck as `running`.
  - confidence: 0.92
- **[P2] Run recovery when retention pruning is disabled:** `src/cron/session-reaper.ts:83-85`
  - body: The `retentionMs === null` return exits before the new orphan scan, so users who set `cron.sessionRetention: false` still get stuck `running` rows after a crash even though they only disabled pruning.
  - confidence: 0.94
