## Summary

Keep open: current main still contains the Dockerfile parser directive that matches the reported failure path, and the linked fix PR is open but not merged. The report slightly misstates the mechanism: Docker BuildKit treats `# syntax=` as a parser directive that can pull the external Dockerfile frontend, not as a shell command.

## What This Changes

Make Docker setup on WSL avoid failing when the root Dockerfile's `# syntax=docker/dockerfile:1.7` directive requires a Docker Hub frontend pull before the build.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80868 if its proof and checks are acceptable, then close this issue from the merge.

## Reproduction Assessment

No, not with high-confidence live reproduction from this review. The source path is clear on current main, but I did not run a WSL Docker build or capture the Docker Hub timeout locally.

## Solution Assessment

Yes, the linked PR's narrow removal of only the root `# syntax=` directive is the right direction; removing all comments would be too broad and unnecessary.
