## Summary

Current main already surfaces provider-side auth and rate-limit failures through the embedded-agent error path instead of mapping them to a workspace-state code. The issue report matches behavior that is no longer present in the checked-out tree.

## Evidence

- **Lifecycle error path uses formatted provider errors:** Embedded agent end now derives the emitted/logged error from `formatAssistantErrorText(...)` or the assistant's raw provider error, then writes that text into the lifecycle error payload.
  - file: [src/agents/pi-embedded-subscribe.handlers.lifecycle.ts:65](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/agents/pi-embedded-subscribe.handlers.lifecycle.ts#L65)
  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **OAuth refresh failures are classified explicitly:** Provider runtime failure classification treats `OAuth token refresh failed ...` as `auth_refresh`, which is then rewritten to a re-authentication message instead of a workspace-state failure.
  - file: [src/agents/pi-embedded-helpers/errors.ts:857](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/agents/pi-embedded-helpers/errors.ts#L857)
  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **Refresh-token errors have direct regression coverage:** Tests cover the exact refresh-failure family from the report and expect the user-facing message `Authentication refresh failed. Re-authenticate this provider and try again.`
  - file: [src/agents/pi-embedded-helpers.formatassistanterrortext.test.ts:247](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/agents/pi-embedded-helpers.formatassistanterrortext.test.ts#L247)
  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **Usage-limit errors stay on the rate-limit lane:** Regression coverage keeps ChatGPT usage-limit messages classified as `rate_limit`, matching the report's expected behavior for provider limit failures.
  - file: [src/agents/pi-embedded-helpers.isbillingerrormessage.test.ts:1088](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/agents/pi-embedded-helpers.isbillingerrormessage.test.ts#L1088)
  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **No current deactivated-workspace remap in repo:** A repo-wide search for `deactivated_workspace` returned no matches in the checked-out main tree, which is consistent with the current implementation surfacing provider/auth errors directly instead of remapping them to that code.
