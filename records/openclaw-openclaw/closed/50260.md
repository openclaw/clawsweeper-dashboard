## Summary

Current `main` already contains a Feishu-specific fix path for this report: the Feishu reply dispatcher disables core block streaming, suppresses internal block payload fan-out, merges streamed text into the streaming card, and deduplicates repeated final text. The same implementation is present in the latest release tag `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).

## Evidence

- **Feishu dispatcher suppresses duplicate block/final delivery:** `extensions/feishu/src/reply-dispatcher.ts` drops `info.kind === "block"` payloads unless they can be folded into the active streaming card, closes the streaming card with merged text, and skips repeated final text via `deliveredFinalTexts`.
