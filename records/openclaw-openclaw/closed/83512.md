## Label Justifications

- `P2`: The report describes dropped CLI transcript persistence on affected older installs, but current main and the stable release already contain the bounded fix.
- `impact:data-loss`: The reported failure silently skips CLI transcript persistence for affected turns.
- `impact:session-state`: The affected behavior is CLI transcript and compaction state for agent sessions.

## Summary

Current main and the shipped stable release already initialize built-in context engines before CLI compaction resolves the default `legacy` engine, with regression coverage; the reported install is an older `2026.5.7` build from before the fix.

## What This Changes

The issue reports released CLI compaction resolving the default `legacy` context engine without first initializing built-in context engines, dropping transcript persistence on each CLI turn.

## Best Possible Solution

Keep the shipped init-order fix and direct affected users on `2026.5.7` to upgrade to `v2026.5.12` or newer; reopen only with logs from a fixed version.

## Reproduction Assessment

No current-main failure reproduction remains: current main calls initialization before context-engine resolution and has a regression test for that order. The historical `v2026.5.7` source lacks that initializer call, matching the reporter's installed-version failure.

## Solution Assessment

Yes: the merged fix is the narrow maintainable solution, keeping initialization at the CLI compaction boundary and preserving the existing dependency-injection test seam.
