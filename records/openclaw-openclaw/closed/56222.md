## What This Changes

Request to add the running gateway version to `openclaw gateway status --json` and MCP `gateway_status` structured output for update smoke checks.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73225, or an equivalent narrow patch, so daemon status exposes `gateway.version` from handshake metadata with `status.runtimeVersion` as the read-probe fallback.

Yes. Source inspection on current main shows `GatewayStatusSummary` still omits `version`, and the `--require-rpc` probe still discards the status RPC payload that already contains `runtimeVersion`.

## Solution Assessment

Yes. Reusing existing handshake `server.version` and status RPC `runtimeVersion` is the narrow maintainable path; no new RPC method, auth model, or config option is needed.
