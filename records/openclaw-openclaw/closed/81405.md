## Summary

Keep open: current main still lacks the requested provider-owned callback-settled or cancellable manual-prompt contract; the related merged work fixed copy only, while the current code and pinned helper still expose a browser/manual race without a signal to dismiss an already visible remote paste prompt.

## What This Changes

Request adding a callback-settled or cancellable prompt signal so remote OpenAI Codex OAuth can dismiss manual paste input when the browser callback completes automatically.

## Best Possible Solution

Keep this open as a provider-owned design item and add an explicit callback-settled or cancellable manual-input contract with focused coverage for both remote paste-only and SSH-forwarded callback paths.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a failing bug, but source inspection verifies the current behavior: remote OAuth starts the paste prompt immediately and the pinned helper exposes only a browser/manual race, not a callback-settled event.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the contract shape; the requested direction is reasonable, but the maintainable fix should keep OpenAI-specific OAuth behavior provider-owned and add any shared prompt-cancellation seam intentionally.
