## What This Changes

The issue reports that `event.messages.push()` from a `message:received` internal hook silently has no user-visible effect despite the hook docs saying pushed messages are sent to the user.

## Best Possible Solution

Use #39688 as the canonical tracker and review, fix, and land #41375 or an equivalent maintainer-approved patch that drains `message:received` hook messages only on reply-capable surfaces, with docs and regression tests aligned. Keep #61657 as useful prior art rather than reopening parallel duplicate work.

## Reproduction Assessment

Yes. Source inspection provides a high-confidence reproduction path: a `message:received` hook can mutate `event.messages`, but the inbound dispatch path currently fires the hook without retaining or draining that array; live WebChat/Telegram was not run because this review is read-only.
