## Summary

Current `main` no longer silently drops isolated cron deliveries when `delivery.channel` is `"last"`: unresolved targets now surface as explicit errors/not-delivered state, and cron list/show/docs expose the route status ahead of time.

## Close Comment

This looks fixed on current `main`, so I'm closing it.
