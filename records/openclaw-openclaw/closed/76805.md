## Summary

Keep open: current `main` still interpolates Kova step outputs directly inside the performance workflow shell block, and this PR is a narrow hardening change that addresses that risk without changing artifact paths, permissions, or dependency sources.

## What This Changes

The PR updates `.github/workflows/openclaw-performance.yml` to pass Kova report outputs through step `env` variables and copy them via quoted shell variables.

## Best Possible Solution

Land this narrow workflow hardening after exact-head CI and the normal maintainer security review pass.

## Solution Assessment

Yes. Moving the dynamic outputs into step `env:` and referencing quoted shell variables is the narrowest maintainable mitigation, and the diff preserves the workflow's artifact layout and copy behavior.
