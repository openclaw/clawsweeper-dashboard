## Summary

Current `main` already contains a targeted fix for this regression. The shipped `2026.4.5` changelog calls out both symptoms from #61050: writing a real `anthropic:claude-cli` auth profile when Claude CLI becomes default, and replacing migrated `agents.defaults.models` allowlists so stale `anthropic/*` entries do not linger. The current Anthropic CLI migration code writes that profile, rewrites the default model selection, and opts into replacing default models; the bundled Anthropic plugin also registers the `claude-cli` backend and synthetic auth path. Tests cover both storing the profile and doctor surfacing the exact missing-profile wiring. The follow-up comments match the pre-fix failure mode, but the fix is now on `main` and documented as shipped in `v2026.4.5`.
