## Summary

Current main no longer matches the reported Web UI update behavior. The Gateway `update.run` schema accepts omitted optional fields, the Control UI sends only the current session key, and global npm fallback failures from the internal `global update (omit optional)` step are normalized to the clearer `global-install-failed` reason with user-facing guidance. The same implementation is present in the latest available release tag, `v2026.4.26`.

## Best Possible Solution

Close this as implemented on current main and in `v2026.4.26`; keep the current optional `update.run` protocol contract and normalized `global-install-failed` UI message. Any remaining Windows-specific package replacement failure should continue under #44614 or a new report with updater logs.
