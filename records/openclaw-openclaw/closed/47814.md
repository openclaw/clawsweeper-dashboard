## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reject collapsed empty segments in named-DM parsing:** `src/sessions/session-key-utils.ts:184-185`
  - body: `parseNamedDmSessionKey` relies on `parseAgentSessionKey`, which drops empty `:` segments before named-DM validation runs. A malformed key like `agent:main:dm-named:123::work` can therefore be accepted as `{ peerId: "123", name: "work" }`; parse the raw scoped suffix or explicitly reject empty segments before returning a valid named-DM key.
  - confidence: 0.92
- **[P2] Fail closed for malformed stored active names:** `src/gateway/session-utils.ts:866-870`
  - body: `activeNamedSession` is persisted session-store data, but `getActiveNamedSessionKey` passes it straight into `buildNamedDmSessionKey`. If a store row contains `:`, callers get an exception during active-session resolution instead of the advertised `string | null` behavior; validate the stored name and return `null` or repair it before building the key.
  - confidence: 0.9
- **[P2] Prioritize dm-named before token chat-type checks:** `src/sessions/session-key-utils.ts:44-50`
  - body: Adding `dm-named` to the direct-token branch after the `group` and `channel` checks still misclassifies valid named DM keys whose user-chosen name is `group` or `channel`. Since send policy can derive chat type through this helper, those direct sessions can pick up group/channel policy rules unless `dm-named` is recognized before arbitrary name tokens.
  - confidence: 0.91
- **[P2] Keep the changelog entry in the active section only:** `CHANGELOG.md:109`
  - body: The branch repeats the same named-DM changelog bullet in a historical release section. That rewrites old release history and can confuse release-note tooling; keep a single entry in the active changelog section for this change.
  - confidence: 0.86
