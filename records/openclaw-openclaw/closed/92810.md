## What This Changes

The PR makes `MediaStreamHandler` reject `start` frames when no `shouldAcceptStream` validator is configured and adds a regression test for that path.

## Best Possible Solution

Land this fail-closed guard with the focused regression test while preserving the existing webhook-level validator wiring for real voice-call streams.

## Reproduction Assessment

Yes. By source inspection, current main accepts a direct `MediaStreamHandler` start frame when `shouldAcceptStream` is omitted and proceeds toward `createSession`; the PR adds the focused failing-closed regression path.

## Solution Assessment

Yes. The start-frame handler is the right layer to prevent unvalidated streams before STT/TTS state is created, and the normal webhook caller already supplies the validator that enforces active-call and token checks.
