## Summary

Keep open. Current `main` still does not implement the QQ Bot plain-reply `audioAsVoice`/TTS path described in the PR: non-structured replies go through `sendPlainReply`, which has no `audioAsVoice` handling, while QQ Bot voice/TTS delivery still exists only in the structured `mediaType === "audio"` path. The PR discussion is coherent and scoped to a real plugin bug, not a stale or clawhub item.
