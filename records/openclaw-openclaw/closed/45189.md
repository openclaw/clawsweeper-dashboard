## Summary

Current main already contains the Discord voice-message upload fix described in #45189: the `/channels/{id}/attachments` step now uses a direct JSON `fetch` instead of `@buape/carbon` `RequestClient.post()`, explicitly to avoid Carbon's multipart conversion on `files` payloads. The `asVoice=true` message action still routes into that code, and the same implementation is present in the latest shipped release commit provided for `v2026.4.22`.
