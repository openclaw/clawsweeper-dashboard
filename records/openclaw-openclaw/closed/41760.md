## Summary

Issue #41760 reads as a one-off workstation/Tailscale outage, not a reproducible OpenClaw defect on current `main`. The report and follow-up both point to the remote host being offline and explicitly say no gateway-side action was needed.

## Evidence

- **Issue describes host outage, not app behavior:** The report says `rusty` was `offline` in `tailscale status`, with 100% ping loss and SSH timeouts. The follow-up comment adds `No action needed on gateway side - awaiting physical access to machine.` That is an external workstation availability incident, not a code-path failure in OpenClaw.
- **Current Tailscale docs assume the remote machine itself must be reachable:** `docs/gateway/tailscale.md:117` says remote browser control works by running a node host on the browser machine and keeping both machines on the same tailnet; if that machine is offline, browser automation is unavailable by design.
  - file: [docs/gateway/tailscale.md:117](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/docs/gateway/tailscale.md#L117)
  - command: `nl -ba docs/gateway/tailscale.md | sed -n '115,121p'`
  - sha: [350d3221800a](https://github.com/openclaw/openclaw/commit/350d3221800a18283915cd2f3b612ae04ddd490c)
- **Current dashboard docs treat this class of failure as basic reachability/auth troubleshooting:** `docs/web/dashboard.md:65` and `docs/web/dashboard.md:76` document remote access via Tailscale/SSH tunnel and explicitly say to first ensure the gateway is reachable. That matches the issue's failed network checks rather than a product regression.
  - file: [docs/web/dashboard.md:65](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/docs/web/dashboard.md#L65)
  - command: `nl -ba docs/web/dashboard.md | sed -n '65,76p'`
  - sha: [350d3221800a](https://github.com/openclaw/openclaw/commit/350d3221800a18283915cd2f3b612ae04ddd490c)
- **Current main only reads Tailscale state for discovery:** `src/infra/tailscale.ts:182` reads `tailscale status --json`; `src/infra/tailnet.ts:24` enumerates tailnet addresses from host interfaces. These are inspection helpers, not code that would make a PopOS workstation sleep, drop off the network, or become unreachable over SSH.
  - file: [src/infra/tailscale.ts:182](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/src/infra/tailscale.ts#L182)
