## Summary

This is a useful Nextcloud Talk UX enhancement, but it is channel/plugin-scoped rather than OpenClaw core work. Current main already has the Nextcloud Talk channel packaged as `@openclaw/nextcloud-talk`, the project vision routes optional capabilities and plugin promotion through ClawHub, and the plugin SDK exposes the reply/typing lifecycle needed to implement this without a new core API. Current `extensions/nextcloud-talk` does not implement the `/typing` endpoint, so the right outcome is a focused plugin update rather than keeping a core issue open.

## Best Possible Solution

Keep OpenClaw core unchanged. Implement this as a focused `@openclaw/nextcloud-talk` plugin enhancement: add a typed `sendTypingNextcloudTalk` helper for the Nextcloud Talk `/typing` endpoint, wire it through the existing reply lifecycle with cleanup on success/error/timeout, add config/docs/tests, and publish through the plugin/ClawHub path.
