## Summary

Current main already suppresses HEARTBEAT_OK from live WebChat output and from chat history projection, with UI-side filtering as defense in depth. The reported 2026.4.26 behavior is fixed on main but was not proven shipped in stable v2026.5.2.

## What This Changes

Suppress HEARTBEAT_OK/no-op heartbeat sentinel replies from Control UI/WebChat transcripts instead of rendering them.

## Best Possible Solution

Keep the current main suppression path and ship it in the next stable release; users on 2026.4.26 need a build containing the main fix.

## Reproduction Assessment

Yes for the reported old build via the WebChat heartbeat steps. Against current main, source and regression coverage show the same HEARTBEAT_OK payload is filtered before live broadcast and history display, so the failure does not reproduce on main.

## Solution Assessment

Yes. Suppressing at the gateway live/history projection layer, with UI filtering as defense in depth, is the narrow maintainable fix and still preserves meaningful heartbeat alert text through ackMaxChars.
