## Summary

Keep open: the latest PR head still has blocking installer and supply-chain findings, and the protected `maintainer` label makes this explicit maintainer handling rather than cleanup-close material. Exact-head CI and real behavior proof are present, but source inspection shows regressions and token/security issues that should be repaired before merge.

## What This Changes

The PR makes OpenClaw own and verify website installer scripts, adds website sync automation, adds CLI/CMD installer entry points, updates installer smoke routing, and hardens Linux Node selection after package-manager installs.

## Best Possible Solution

Keep the source-owned installer sync direction, but move or harden token-bearing workflow steps, verify or remove the portable Git bootstrap, fix the installer regressions, and merge only after exact-head CI plus maintainer/security review.

## Reproduction Assessment

Yes. The blocking findings are reproducible by source inspection on the current PR head: the workflow runs a third-party action after a token checkout, the Windows installer downloads and extracts mutable portable Git, `-Tag main` maps to `openclaw@main`, `install-cli.sh` writes a malformed PATH export and runs cleanup outside the selected git install, and the PowerShell success path returns no `$true`.

## Solution Assessment

No. The overall source-owned sync direction is reasonable, but this is not the best mergeable form until the token exposure, portable binary trust path, and installer regressions are fixed narrowly.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Keep the website token away from third-party actions:** `.github/workflows/website-installer-sync.yml:163`
  - body: `sync-website` checks out `openclaw.ai` with `OPENCLAW_GH_TOKEN` and then runs `oven-sh/setup-bun@v2` after checkout has persisted push credentials. Move Bun setup before the token checkout, disable persisted credentials and push explicitly, or use a trusted pinned setup path so a mutable third-party action cannot access the website write token.
  - confidence: 0.86
- **[P1] Write a valid PATH export when setting npm prefix:** `scripts/install-cli.sh:477`
  - body: The generated rc line is `export PATH=\"...\"`, which persists literal quote characters in PATH when a later shell sources `.bashrc` or `.zshrc`. Users who hit the unwritable-prefix branch can end up with broken command lookup, so write the normal quoted export instead.
  - confidence: 0.9
- **[P2] Verify portable Git before extracting it:** `scripts/install.ps1:287-288`
  - body: This fallback resolves the latest Git-for-Windows release, downloads a MinGit ZIP, and extracts it without pinning the release or checking a digest/signature. Fresh Windows installs without Git would trust a mutable third-party binary path, so either keep the manual/package-manager path or pin and verify the archive before extraction.
  - confidence: 0.88
- **[P2] Return success after the PATH warning:** `scripts/install.ps1:763`
  - body: When `Ensure-OpenClawOnPath` fails, this branch prints that installation completed but then returns no explicit success value. The top-level handler treats only `$true` as success, so a completed install can exit non-zero and make automation or `install.cmd` report failure unnecessarily.
  - confidence: 0.9
- **[P2] Scope legacy cleanup to the selected git checkout:** `scripts/install-cli.sh:711`
  - body: `main` calls `cleanup_legacy_submodules` before knowing whether this is a git install and without passing the parsed `GIT_DIR`. The helper can fall back to the default checkout and delete `Peekaboo` outside the requested target, including during npm installs.
  - confidence: 0.9
- **[P2] Preserve main as the GitHub branch install spec:** `scripts/install.ps1:495`
  - body: `Resolve-NpmOpenClawInstallSpec` no longer special-cases `main`, so `install.ps1 -Tag main` resolves to `openclaw@main` instead of the documented `github:openclaw/openclaw#main` branch spec. That can fail or install the wrong artifact when no `main` npm dist-tag exists.
  - confidence: 0.9
