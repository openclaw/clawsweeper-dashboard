## Summary

Current main already preserves Telegram inbound image paths in `MediaPaths`, prepends them to the agent prompt as `[media attached: ...]`, and the embedded runner parses those prompt attachments into vision inputs. The placeholder-only root cause described in the issue/comments is obsolete on current main.

## Evidence
