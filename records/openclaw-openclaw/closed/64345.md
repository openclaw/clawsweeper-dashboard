## Summary

Current main already implements generic real-time session transcript push into the Control UI: gateway transcript updates emit `session.message`/`sessions.changed`, and the UI reloads the active session when those events arrive. The same implementation is present in the shipped `v2026.4.23` tag.

## Evidence
