The branch adds a private bundled `extensions/temporal-context` runtime plugin with README, manifest, tests, lockfile importer, and a plugin-boundary test update to inject live temporal context via `before_prompt_build`.

## Best Possible Solution

Publish and maintain temporal-context as an external ClawHub native plugin, while keeping core’s cache-stable date/time contract unchanged unless maintainers deliberately expand it.

## Reproduction Assessment

Not applicable; this is a new optional plugin feature rather than a bug report. The PR body provides copied live output from a similar workspace plugin, but there is no current-main failure path to reproduce.

## Solution Assessment

No; bundling this optional runtime plugin in core is not the best repo-level solution. Current plugin APIs can support it externally, and OpenClaw policy routes optional plugin promotion through ClawHub unless there is a clear core ownership reason.
