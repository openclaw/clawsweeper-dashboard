## Summary

Keep open: current main still appends memory-core promotions to MEMORY.md without a disk-side size budget, and the open closing PR has not merged. The later production comment strengthens the report with real deployment evidence of oversized bootstrap-loaded files and token growth.

## What This Changes

Bound memory-core dreaming promotions so MEMORY.md cannot grow past bootstrap limits and stall gateway sessions.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/74088 or an equivalent memory-core compaction/cap that preserves user-authored memory while bounding auto-promoted sections.

## Solution Assessment

Yes on boundary: the durable fix belongs in memory-core's promotion writer because bootstrap trimming only bounds injected context, not the file on disk. The open closing PR follows that direction, so the issue should stay open until that or an equivalent fix merges.
