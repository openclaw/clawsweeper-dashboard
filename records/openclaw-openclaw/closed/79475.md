## Summary

Close as implemented: current main keeps `runningAtMs` as the reservation but now bounds the detached agent startup path with setup/pre-execution watchdogs that record an error run and clear the running marker instead of silently sticking.

## What This Changes

Report one-shot `at` cron jobs targeting `session:cody-coder` silently remaining in `runningAtMs` with no spawned session, run record, log, or on-disk evidence.

## Best Possible Solution

Keep the shipped watchdog-and-diagnostics implementation for this failure class, while leaving broader cron observability under https://github.com/openclaw/openclaw/issues/13593 and adjacent watchdog tuning/progress PRs to their own reviews.

## Reproduction Assessment

No high-confidence live reproduction was established for the reporter's production load pattern. Source inspection and regression tests cover the former stall boundary: a detached `agentTurn` can now time out, record an error, and clear `runningAtMs` when startup or pre-execution progress never arrives.

## Solution Assessment

Yes. Preserving `runningAtMs` as the duplicate-run reservation while adding bounded setup/pre-execution watchdogs and durable diagnostics is narrower and safer than moving the marker until after spawn completion.
