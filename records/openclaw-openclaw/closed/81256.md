## What This Changes

The branch keeps `status --json` secret diagnostics in the JSON payload by omitting the logging runtime for that scan path, adds a regression test, and adds a changelog entry.

## Best Possible Solution

Land the narrow PR after maintainer review and green required checks, preserving JSON-only stdout while keeping diagnostics in `secretDiagnostics`.

## Reproduction Assessment

Yes. Current main has a source-reproducible path: `status --json` passes a runtime into secret resolution, and that resolver logs `[secrets]` diagnostics through stdout before the JSON payload is written.

## Solution Assessment

Yes. Omitting the logging runtime only for the `status --json` scan is the narrowest maintainable fix because status JSON already carries `secretDiagnostics`, while the closed alternative at https://github.com/openclaw/openclaw/pull/81390 changed the shared diagnostic emitter for more callers.
