## Summary

Current main already addresses the report well enough: Ollama is supported natively again, the Ollama docs now explicitly warn against `/v1` except for legacy compat mode and document `injectNumCtxForOpenAICompat`, and Telegram now sends a user-visible fallback reply instead of leaving the conversation silently stuck on typing. The same code/docs are present in the released `v2026.4.22` tree.
