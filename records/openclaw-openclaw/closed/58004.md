## Summary

Current OpenClaw already supports unified cross-surface DM sessions through existing session routing: DMs default to the agent main session, `session.identityLinks` collapses provider-prefixed identities across channels, and WebChat/TUI use the same gateway session model. The issue’s underlying use case is already covered on `main` and in the shipped `v2026.4.23` release.

## Evidence

- **Session behavior docs:** Docs state that all direct messages share one session by default, and that `session.identityLinks` links the same person across multiple channels so they share one session.
  - file: [docs/concepts/session.md:24](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/docs/concepts/session.md#L24)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Inbound routing defaults:** Inbound route resolution defaults DM routing to `session.dmScope ?? "main"` and passes `identityLinks` into session-key generation.
  - file: [src/routing/resolve-route.ts:623](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/routing/resolve-route.ts#L623)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Session-key implementation:** `buildAgentPeerSessionKey` sends direct messages to the agent main session for `dmScope: "main"`; for other DM scopes it canonicalizes linked peers via `resolveLinkedPeerId`, enabling cross-channel identity collapse.
