## Summary

Keep open: current main and the latest release still produce the reported generic `config-patch ok` restart-sentinel wording, and two open linked PRs already own candidate fixes. The issue should remain open until one linked PR lands or maintainers explicitly replace that fix path.

## What This Changes

The issue asks for `config.patch` restart sentinel notices to say a gateway restart is required, rather than implying the restart already completed.

## Best Possible Solution

Review and land or replace a linked fix that makes pending restart-required config writes say `Gateway restart required (...)` while preserving completed-restart and hot-reload wording.

## Reproduction Assessment

Yes, from source inspection: current main still formats a successful `config-patch` sentinel from kind, status, and mode, producing the misleading ok restart notice. I did not run a live Gateway repro because this review is read-only.

## Solution Assessment

Yes, the maintainable fix belongs at the restart sentinel formatter/payload boundary so it can distinguish pending restart-required, completed restart, and hot-reloaded config writes. A blanket config.patch wording change would be too narrow unless it preserves those states.
