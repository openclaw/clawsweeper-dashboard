## Summary

Keep open. Current main and the latest release still derive the Dreaming Advanced Daily Log summary and staged section from `groundedCount`, while the UI copy and controller data model keep `dailyCount` and `groundedCount` as separate concepts; the same reporter has an open paired fix PR that should be reviewed before creating another repair lane.

## What This Changes

The issue asks OpenClaw to make the Dreaming Advanced page's Daily Log summary, section, and origin labeling consistent with the underlying `dailyCount` versus `groundedCount` semantics.

## Best Possible Solution

Review and either land or supersede https://github.com/openclaw/openclaw/pull/72552 so the Daily Log summary and staged section use one consistent counter semantic with focused Dreaming view coverage.

## Reproduction Assessment

Yes from source inspection: render the advanced tab with an entry where `dailyCount > 0` and `groundedCount === 0`, and current main excludes it from the Daily Log summary and staged section because both use `groundedEntries`. I did not run a browser repro in this read-only review.

## Solution Assessment

Yes, the narrowest maintainable fix is to align the Daily Log summary and section with `dailyCount` while keeping grounded replay metadata visible. The existing paired PR is the right review target before opening another fix path.
