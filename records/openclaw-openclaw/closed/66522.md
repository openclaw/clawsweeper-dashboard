## Summary

Current `main` does not match the report’s failure model: Gateway session loading reads the persisted `sessions.json`, startup only runs an orphan-key migration, `sessions.list` does not drop ended sessions, and the issue thread already includes a non-repro report on v2026.4.14 for both graceful and hard restarts.

## Evidence

- **Session store load path reads persisted index:** `loadSessionStore()` reads and parses the existing `sessions.json` from disk, then applies migrations/normalization in memory; it is not a startup-only rebuild from active sessions.
  - file: [src/config/sessions/store-load.ts:76](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/config/sessions/store-load.ts#L76)
  - command: `nl -ba src/config/sessions/store-load.ts | sed -n '60,150p'`
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **Writes re-read current store before mutating:** `updateSessionStore()` explicitly re-loads the current store under a lock before saving, to avoid clobbering existing entries.
  - file: [src/config/sessions/store.ts:421](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/config/sessions/store.ts#L421)
  - command: `nl -ba src/config/sessions/store.ts | sed -n '420,440p'`
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **Gateway startup session work is migration, not index rebuild:** Gateway startup wires in `runStartupSessionMigration()`, and that helper is documented and implemented as an orphan-key canonicalization pass, best-effort on startup.
  - file: [src/gateway/server-startup-plugins.ts:38](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/gateway/server-startup-plugins.ts#L38)
  - command: `nl -ba src/gateway/server-startup-plugins.ts | sed -n '32,55p'`
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **Startup migration scope is narrow:** `runStartupSessionMigration()` only calls `migrateOrphanedSessionKeys()` and logs changes/warnings; it does not rebuild the session index from running sessions.
  - file: [src/gateway/server-startup-session-migration.ts:9](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/gateway/server-startup-session-migration.ts#L9)
  - command: `nl -ba src/gateway/server-startup-session-migration.ts | sed -n '1,50p'`
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **Session listing does not filter out ended sessions:** `listSessionsFromStore()` filters cron/global/unknown/agent/label/spawnedBy, but there is no `ended=false` filter in the listing path described by the report.
  - file: [src/gateway/session-utils.ts:1473](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/gateway/session-utils.ts#L1473)
  - command: `nl -ba src/gateway/session-utils.ts | sed -n '1473,1545p'`
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **Backup behavior exists for the store:** `rotateSessionFile()` creates `sessions.json.bak.<timestamp>` copies and explicitly keeps the live `sessions.json` authoritative to avoid a startup empty-store window.
  - file: [src/config/sessions/store-maintenance.ts:320](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/config/sessions/store-maintenance.ts#L320)
  - command: `nl -ba src/config/sessions/store-maintenance.ts | sed -n '320,372p'`
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **Issue discussion already contains a latest-version non-repro:** On 2026-04-15, @SimbaKingjoe reported that v2026.4.14 preserved the session list after both `openclaw gateway restart` and a `kill -9` followed by manual restart.

## Risks / Open Questions
