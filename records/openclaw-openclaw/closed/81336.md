## Summary

Close: this PR is superseded by the open canonical draft that preserves raw QMD lexical search while normalizing only semantic QMD searches. This PR sanitizes the whole query before QMD mode/tool selection, which would also change lexical recall for dashed identifiers and dates.

## What This Changes

The PR adds a QMD query sanitizer that replaces word-internal hyphens with spaces before all QMD search paths and adds two direct-CLI tests.

## Best Possible Solution

Follow the canonical semantic-only normalization path in https://github.com/openclaw/openclaw/pull/81423 or an equivalent patch that keeps lex/raw BM25 recall intact.

## Reproduction Assessment

Yes. The source path is high-confidence: QMD v2.0.1 rejects hyphens in vec/hyde validation while OpenClaw builds hybrid QMD queries with lex, vec, and hyde entries; I did not run a live QMD binary in this read-only review.

## Solution Assessment

No. This PR is broader than the narrow fix because it sanitizes the query before lexical and semantic paths split; preserving raw lex and normalizing only vec/hyde is the safer solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve raw lexical QMD queries:** `extensions/memory-core/src/memory/qmd-manager.ts:1112`
  - body: Sanitizing `trimmed` here feeds the modified text to every later QMD path, including lexical `search`/`lex` queries. The failure comes from vec/hyde semantic validation, so replacing `sqlite-vec` with `sqlite vec` globally fixes the rejection by weakening the exact lexical recall for dashed identifiers and dates that QMD can otherwise match. Keep the raw query for lex and normalize only semantic QMD payloads.
