## Summary

Current `main` already prevents cron management commands from loading plugins. The implementation differs from the issue's suggested explicit `cliCommandCatalog` row, but the effective behavior is present, covered by tests, and shipped in `v2026.4.23`.

## Evidence

- **Default CLI policy keeps unmatched commands off plugin preload:** `resolveCliCommandPathPolicy` starts from `loadPlugins: "never"` and only overrides that for catalog entries that opt in. With no `cron` override, cron commands inherit the non-loading default.
  - file: [src/cli/command-path-policy.ts:5](https://github.com/openclaw/openclaw/blob/7875092f4ddc6f2dbe218bea8506c9ac1da87c39/src/cli/command-path-policy.ts#L5)
  - command: `nl -ba src/cli/command-path-policy.ts | sed -n '1,80p'`
  - sha: [7875092f4ddc](https://github.com/openclaw/openclaw/commit/7875092f4ddc6f2dbe218bea8506c9ac1da87c39)
- **Startup policy converts cron paths into `loadPlugins: false`:** `shouldLoadPluginsForCommandPath` returns true only for `always` or `text-only` policies. A cron command therefore resolves to `false` on current `main`.
  - file: [src/cli/command-startup-policy.ts:19](https://github.com/openclaw/openclaw/blob/7875092f4ddc6f2dbe218bea8506c9ac1da87c39/src/cli/command-startup-policy.ts#L19)
  - command: `nl -ba src/cli/command-startup-policy.ts | sed -n '1,80p'`
  - sha: [7875092f4ddc](https://github.com/openclaw/openclaw/commit/7875092f4ddc6f2dbe218bea8506c9ac1da87c39)
- **Bootstrap only loads the plugin registry when explicitly enabled:** `ensureCliCommandBootstrap` returns early when `loadPlugins` is falsy, so a cron command on the resolved startup policy does not initialize plugins before running.
  - file: [src/cli/command-bootstrap.ts:31](https://github.com/openclaw/openclaw/blob/7875092f4ddc6f2dbe218bea8506c9ac1da87c39/src/cli/command-bootstrap.ts#L31)

  - sha: [7875092f4ddc](https://github.com/openclaw/openclaw/commit/7875092f4ddc6f2dbe218bea8506c9ac1da87c39)
- **Cron commands are explicitly treated as config-bypass paths:** `isGatewayConfigBypassCommandPath` returns true for `commandPath[0] === "cron"`, matching the expectation that cron management commands are handled specially and do not require normal startup config loading.
  - file: [src/gateway/explicit-connection-policy.ts:20](https://github.com/openclaw/openclaw/blob/7875092f4ddc6f2dbe218bea8506c9ac1da87c39/src/gateway/explicit-connection-policy.ts#L20)
