## Summary

Current `main` no longer uses the old bare reset greeting behavior from the report. Bare `/new` and `/reset` now explicitly instruct Session Startup file reads, add BOOTSTRAP-aware variants, and can prepend runtime-owned startup context, so the reported "generic greeting before reading files" problem is already addressed on `main` without adding a separate `sessionResetPrompt` config key.

## Close Comment

Closing from a Codex review because current `main` already addresses the reported reset behavior. The bare reset prompt now tells the agent to execute Session Startup and read required files before replying (`src/auto-reply/reply/session-reset-prompt.ts:11`), including BOOTSTRAP-specific paths that explicitly avoid a generic first greeting (`src/auto-reply/reply/session-reset-prompt.ts:14`). Bare `/new` and `/reset` turns also prepend runtime-owned startup context before that prompt (`src/auto-reply/reply/get-reply-run.ts:376`), and the reset startup-context controls are now part of the documented config surface (`src/config/types.agent-defaults.ts:67`, `docs/gateway/configuration-reference.md:1014`). The behavior is covered by tests in `src/auto-reply/reply/session-reset-prompt.test.ts:12` and `:70`. I do not see the old "generic greeting before reading files" behavior on current `main`, so this request looks implemented well enough by the current reset flow even though it landed as a behavior fix plus startup-context controls rather than a separate `sessionResetPrompt` config field.
