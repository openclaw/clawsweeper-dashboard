## Summary

Current `main` already includes the context-window fixes that match this report: Bedrock discovery now supplies known model limits instead of falling back to `32000` for known models, explicit `contextWindow`/`maxTokens` values are preserved when implicit catalogs merge with user config, and compaction now uses the resolved config/model context metadata. That fix surface is present in release `v2026.4.23` and unchanged on current `main`.

## Evidence

- **Issue discussion identified the failing path:** The issue comments, especially `mobilinkd`'s diagnosis, narrowed the failure to wrong resolved `contextWindow` values during compaction/precheck and linked follow-up work including PR `#65952` for Bedrock discovery context windows.
- **Known Bedrock model limits are now encoded:** `extensions/amazon-bedrock/discovery.ts` now carries a `KNOWN_CONTEXT_WINDOWS` table for discovered Bedrock models, including `mistral.mistral-large-3-675b-instruct: 128000`, instead of relying only on the old conservative fallback.
