## Summary

Keep open: current main still has a source-level path where cron Telegram announce delivery can pre-render Markdown links to Telegram HTML, route that HTML through the CLI lazy runtime sender, and lose the formatting/plain fallback metadata before the Telegram adapter sends it again.

## What This Changes

Fix Telegram cron announce delivery so Markdown links remain clickable and HTML parse fallback does not expose rendered anchors as literal text.

## Best Possible Solution

Carry formatting and real plain-text fallback metadata through the generic lazy outbound sender, then add regressions for cron Telegram announce links and HTML parse fallback plaintext.

## Reproduction Assessment

Yes, at source level: current main can hand pre-rendered Telegram HTML chunks to a lazy runtime sender that drops formatting/textMode/plainText before the Telegram adapter handles the text again. I did not run a live Telegram job in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is to preserve generic formatting and fallback state through the lazy sender and avoid raw rendered HTML on plain retries. A new `delivery.format` option would be a broader product/config change and is not required for this bug.
