## Summary

Current `main` already supports remote MCP servers over HTTP: URL-based configs resolve to HTTP transports, `streamable-http` is wired via `StreamableHTTPClientTransport`, the bundle MCP runtime connects through that transport, and the changelog shows the feature shipped in `2026.3.31`. The original "only stdio MCP servers are supported right now" behavior is no longer present on `main`.

## Evidence

## Risks / Open Questions

- The original repro used only `{"url":"http://localhost:3000/mcp"}` for a Streamable HTTP server. On current main, `streamable-http` support exists, but the config now needs `"transport": "streamable-http"`; otherwise OpenClaw defaults URL configs to SSE as documented in `docs/cli/mcp.md:476-481`.
