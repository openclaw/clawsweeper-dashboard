## Summary

Keep this PR open. Current main has baseline Discord/ACP thread-bound support, but it does not implement this PR's multi-phase delivery overhaul: the MessageClass/surface-policy layer, delivery receipts, Discord live E2E harness, respawn/end-binding lifecycle, strict visibility helpers, and new agent tools are absent or materially different. The PR also carries broad security/process-sensitive changes that need normal review and likely splitting, not conservative auto-closure.

## Best Possible Solution

Keep this PR open but do not merge it as-is. The best maintainer path is to split the useful work into focused PRs: first the generic MessageClass/surface-policy contract with tests, then Discord thread-binding lifecycle/respawn, then delivery receipts/tools, then the live E2E harness after a security review of auth/env handling. Remove fork/GitNexus artifacts and root planning docs from any upstream PR, and use the related open thread-bound ACP reports as follow-up context rather than as a reason to close this PR.
