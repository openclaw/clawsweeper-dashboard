Keep open: current main still accepts a terminal ACP `done` event as successful task completion, and the only linked fix PR was closed unmerged after its author agreed the core heuristic could reject valid text-only completions. ## What This Changes

## Best Possible Solution

Capture a redacted real Codex/acpx event stream for the premature completion and fix the adapter/runtime contract so premature close reports `ACP_TURN_FAILED` while legitimate text-only completions still succeed.

## Reproduction Assessment

## Solution Assessment

No for the closed PR's approach: failing based on progress-looking text or missing tool calls is not the narrowest maintainable fix because valid ACP turns can complete with text plus `done`. The safer direction is adapter/runtime-level failure signaling for premature termination.
