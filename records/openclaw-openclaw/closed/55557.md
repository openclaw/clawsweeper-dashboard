## Summary

Current main already ships a machine-readable setup wizard/session protocol for GUI-driven onboarding: Gateway RPC methods `wizard.start`/`wizard.next`/`wizard.status`/`wizard.cancel`, typed step payloads, docs that position it for GUI clients, an end-to-end test, and a macOS app consumer that renders the steps instead of reimplementing setup logic.

## Risks / Open Questions

- The shipped surface is Gateway RPC for onboarding, not a file-backed `openclaw configure session ...` CLI family. If maintainers specifically want a local file transport or broader post-install reconfigure coverage, that would be a follow-up enhancement rather than evidence this request is still unimplemented.
