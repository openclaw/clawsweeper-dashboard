## Summary

Keep open. Current main still has the hardcoded trusted web-tool policy gap this PR describes, so the contribution is relevant, but the patch is not merge-ready because it changes a security-sensitive default without updating tests or the required changelog.

## What This Changes

The PR changes `src/agents/tools/web-guarded-fetch.ts` so trusted web-tool endpoints allow RFC2544 and IPv6 ULA fake-IP ranges, and self-hosted endpoints also include the IPv6 ULA allowance.

## Best Possible Solution

Land a tested policy update only after maintainers accept the trusted-helper SSRF exception, with `web-guarded-fetch` regression coverage and a single-line Unreleased changelog entry.

## Reproduction Assessment

Yes. Current main passes `{}` for trusted web-tool endpoints, while the SSRF implementation only exempts RFC2544 or IPv6 ULA DNS answers when the active policy has the corresponding flag, so a fake-IP lookup on this path is source-reproducible.

## Solution Assessment

No. The direction is plausible, but this PR is not the best mergeable form because it leaves failing test expectations, lacks regression coverage and changelog, and needs explicit maintainer/security acceptance for the default trusted-policy widening.
