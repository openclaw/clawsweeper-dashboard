## Summary

Current `main` now blocks the synthetic `"No result provided"` replay path for OpenAI-compatible transports and has an explicit regression test for it. The latest tagged release in this checkout (`v2026.4.23`) predates that exact transport-layer guard, so this appears fixed on `main` but not yet tied to a shipped release tag here.

## Evidence

- **Current main transport gate:** `transformTransportMessages()` now enables synthetic missing-tool-result insertion only for a small allowlist of transport-owned replay APIs (`anthropic-messages`, Anthropic/Google transport aliases, and Bedrock). OpenAI-compatible transports are excluded, so they no longer inject the `"No result provided"` synthetic tool result.
