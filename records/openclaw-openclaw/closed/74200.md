## Summary

Keep this PR open: current main still has the destructive target skills directory removal, so the proposed fix is not obsolete, but the branch has blocking correctness and security-sensitive stale-skill issues plus missing contributor real behavior proof.

## What This Changes

The PR changes `syncSkillsToWorkspace` from whole-target removal to incremental per-skill copying with newest-mtime skips, debug counters, and non-destructive sync regression tests.

## Best Possible Solution

Implement an owned-manifest sync that preserves unknown user-created target content, prunes OpenClaw-owned stale skills, detects file-set changes, and preserves failed-copy cleanup.

## Reproduction Assessment

Yes, by source inspection rather than a live run: current main removes `<target>/skills` before copying entries, and the PR branch can leave stale directories or deleted source files in the target after eligibility or tree changes.

## Solution Assessment

No. Avoiding the whole-target wipe is the right direction, but preserving every non-manifest directory and using newest mtime as the freshness contract is not the narrowest maintainable fix.
