## Summary

This PR adds a useful optional context-injection format, but current OpenClaw direction and extension points make it better suited for a ClawHub/plugin implementation than a one-off core bootstrap format. Current main does not include Context Book support, and the PR’s own review comments identify unresolved behavior gaps in the proposed core implementation.

## Best Possible Solution

Close the core PR and keep the idea alive as a ClawHub code plugin or context-engine plugin. If implementation exposes a real missing public API, open a focused plugin-API proposal instead of adding a core-owned `context-books/` file format.
