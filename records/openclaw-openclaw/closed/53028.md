## Summary

Current main now has a bundled LM Studio/local-model path that targets localhost directly, plus local OpenAI-compatible transport handling and tests for local auth/no-auth behavior. The related #62684-style local timeout class is also covered by the later transport timeout fix in the changelog.

## Best Possible Solution

Close this issue and keep the shipped LM Studio/local-provider implementation. Users who still see an offline localhost failure on current releases should open a fresh bug with a current version, exact provider config, local server curl proof, logs, and whether hosted fallbacks or online tools were enabled.
