## Summary

Current main implements the PR’s central goal through the shared memory embedding provider path: `memory-lancedb` can delegate to registered providers, and the bundled Google plugin registers a Gemini embedding provider. The implementation uses provider id `gemini` rather than this PR’s proposed direct `google` branch, so the stale PR should close instead of merging its separate implementation.

## Best Possible Solution

Close this PR and keep current main’s shared memory embedding provider implementation. If maintainers want more ergonomics, the follow-up should be a small docs/config improvement for `memory-lancedb` Gemini examples and dimension defaults, not this stale direct Google-specific embedding branch.
