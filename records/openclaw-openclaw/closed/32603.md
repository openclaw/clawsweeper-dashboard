## Summary

Current `main` already provides a supported ACPX config surface to override the Claude harness command, which lets operators launch Claude Code with proxy env vars before `/acp spawn` starts it.

## Close Comment

Closing from a Codex review: current `main` already has a supported way to do this. The bundled ACPX runtime now exposes per-agent command overrides at `plugins.entries.acpx.config.agents.<id>.command` (`extensions/acpx/src/config-schema.ts:111`, `extensions/acpx/openclaw.plugin.json:144`), and the runtime wires those overrides into the ACP agent registry before `/acp spawn` launches the harness (`extensions/acpx/src/service.ts:49`). In practice that allows a Claude Code launch command like `env HTTP_PROXY=... HTTPS_PROXY=... claude --acp`, which matches the requested proxy-before-launch behavior. The implementation is covered by `extensions/acpx/src/config.test.ts:44`. I could confirm this on `main` in commit `c0a7b6a510758645c5267734b984f4885a605b67`; I could not confirm a tagged release containing it yet, so `fixedRelease` is left null.
