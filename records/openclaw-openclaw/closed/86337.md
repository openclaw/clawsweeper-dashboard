Full review comments:

- **[P2] Preserve replacement metadata during scoped close:** `extensions/memory-core/src/memory/manager.ts:149-150`
  - body: This deletes the shared lastAccessAt/inflightCount entries before `manager.close()` has finished. The old manager can still be awaiting provider or sync work, and the adjacent close/getOrCreate pattern allows a replacement to be installed while that close is in flight; if an old busy release fires after the replacement acquires its own ref, it can delete the replacement's count and make the active manager eligible for idle eviction. Keep the metadata until the old close is past the async window, or make the release accounting identity-scoped, and add a regression for this closeForAgent interleaving.
  - confidence: 0.8
