## Summary

Keep open. Current main appears to have addressed part of the original symptom by sanitizing inbound channel transcript entries and preserving sender labels for user messages, but it does not implement the requested Dashboard-vs-Feishu source labeling/distinct bubble treatment, and the only hardcoded UI `CHANNEL_LABELS` map in Control UI still does not include `feishu`.
