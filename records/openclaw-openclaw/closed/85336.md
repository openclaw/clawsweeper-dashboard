## Summary

Keep open: this PR suppresses one `gateway status` hint path, but the linked false-positive report is about the top-level `openclaw status --deep` service rows. Current main still derives those rows from user-systemd service state without using gateway reachability, so merging this branch could leave the reported behavior in place while closing https://github.com/openclaw/openclaw/issues/85094.

## What This Changes

This PR changes CLI gateway-status printing to skip Linux systemd-unavailable hints when the RPC probe succeeds, adds regression coverage for that printer path, and adds a changelog entry.

## Best Possible Solution

Teach the top-level status service summary/overview to treat a reachable Gateway under external or system supervision as externally managed, or otherwise suppress the user-systemd not-installed row, while preserving gateway-status failure hints when RPC is not reachable.

## Reproduction Assessment

Yes, source-reproducible: the linked report provides concrete Linux system-unit steps, and current main shows top-level status ignores `gatewayReachable` when formatting service rows. I did not run a live Linux systemd-system setup in this read-only review.

## Solution Assessment

No. Suppressing the daemon-status hint is useful but not sufficient; the maintainable fix should cover `src/commands/status.service-summary.ts` or the top-level status overview path, with regression coverage for `openclaw status --deep`.
