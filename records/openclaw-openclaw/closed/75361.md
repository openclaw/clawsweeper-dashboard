Keep open. This is an active maintainer-labeled PR, and the latest head still has a concrete supply-chain blocker: the new published-baseline input can reach `npm install` as a broad `openclaw@*` spec before strict OpenClaw package validation; the author follow-up addressed the fixture split but not this baseline validation path.

## Best Possible Solution

Keep the PR open, validate `published_upgrade_survivor_baseline` with the same strict OpenClaw package-spec rules before any npm install, add focused rejection coverage for alias/file-like specs, then rerun the targeted workflow/script tests plus the regular and published survivor Docker lanes.
