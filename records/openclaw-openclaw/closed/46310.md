## Summary

Current `main` already gates `config.patch` restarts through the reload planner, and the same behavior is present in the released `v2026.4.22` snapshot.

## Evidence

- **`config.patch` no longer restarts unconditionally:** `src/gateway/server-methods/config.ts:285` defines `shouldScheduleDirectConfigRestart()` to schedule a direct SIGUSR1 restart only when `gateway.reload.mode` is `off`, or when hot mode still yields `plan.restartGateway`. `src/gateway/server-methods/config.ts:592` uses that guard before calling `scheduleGatewaySigusr1Restart()` for `config.patch`.
