## Summary

Keep open: the PR fixes a real ClawDock dashboard port bug, but it is not merge-ready because the token URL-encoding helper still passes the raw dashboard token through child-process argv.

## What This Changes

The PR updates the ClawDock shell helper and changelog so dashboard URLs are rewritten to the Docker-published gateway port with token display redacted.

## Best Possible Solution

Land a helper-scoped remap that resolves the published Compose port, preserves scheme/path and redacted terminal output, encodes tokens without argv exposure, and preserves the `--no-deps` dashboard lookup behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set `OPENCLAW_GATEWAY_PORT` to a non-default port, run `clawdock-dashboard`, and current main opens the loopback URL reported from inside the CLI container unchanged.

## Solution Assessment

No, not as written. The helper is the right surface, but the patch should encode the token without passing it through child-process argv and should be coordinated with the open `--no-deps` dashboard helper change.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Avoid passing dashboard tokens through argv:** `scripts/clawdock/clawdock-helpers.sh:74-78`
  - body: `_clawdock_urlencode` receives the gateway token and then starts `python3` or `node` with that token as a positional argument. Local process listings can expose argv values while the encoder runs, so feed the token through stdin or use another non-argv encoding path before appending it to the browser URL.
  - confidence: 0.9
