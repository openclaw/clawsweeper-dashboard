## What This Changes

The PR replaces QQBot's hardcoded 300s outbound response watchdog with a resolver based on existing agent/provider timeoutSeconds config and adds resolver plus dispatch regression tests.

## Best Possible Solution

## Solution Assessment

Yes. Reusing existing timeoutSeconds config is the narrowest maintainable fix and avoids a QQBot-only knob; the only maintainer choice is whether max-across-providers is the desired bound.
