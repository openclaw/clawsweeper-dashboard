## Summary

Current `main` already implements cooldown-aware model fallback that skips repeatedly failing primaries/providers, persists cooldown state with model-aware scoping for rate limits, and only re-probes on a throttled window. The behavior is covered by regression tests and shipped in `v2026.4.23`.

## Evidence

- **Primary cooldown skip/probe policy:** `runWithModelFallback` keeps per-provider probe state, throttles probes to 30s, and only re-probes a cooldowned primary near expiry; otherwise it skips the primary instead of retrying it every request.
  - file: [src/agents/model-fallback.ts:537](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/agents/model-fallback.ts#L537)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Fallback loop skips unhealthy primary/provider:** When all auth profiles for a provider are unavailable, the fallback loop records `skip_candidate` and continues to the next fallback; repeated transient cooldown probes for the same provider are suppressed within a run.
  - file: [src/agents/model-fallback.ts:764](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/agents/model-fallback.ts#L764)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Persisted route/model health state:** Auth-profile failure tracking persists cooldown/disable windows, records `cooldownModel` for rate limits, and widens to profile-wide suppression for auth and other non-rate-limit failures.
  - file: [src/agents/auth-profiles/usage.ts:578](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/agents/auth-profiles/usage.ts#L578)

  - file: [src/agents/model-fallback.probe.test.ts:246](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/agents/model-fallback.probe.test.ts#L246)
