## Summary

Keep this PR open for normal maintainer review: current main still has the reported raw-entry budget regression in grouped skill discovery, and the patch is a narrow loader/test fix rather than obsolete work.

## What This Changes

The PR changes workspace skill discovery so candidate limits apply to filtered child directories, adds a separate raw-entry scan cap, truncates nested grouped candidates after sorting, and adds regression coverage for ignored raw files before valid nested skill directories.

## Best Possible Solution

The desired end state is a bounded one-level grouped skill scanner where configured candidate limits apply to visible child directories, with a separate raw-entry guard for pathological roots and focused regression coverage for ignored non-directory entries.

## Reproduction Assessment

Yes. On current main, a grouped skills folder with ignored raw files before valid skill directories and a small `maxCandidatesPerRoot` can exhaust `scannedEntryCount` before any valid directories are collected; the code path is direct from `loadWorkspaceSkillEntries()` through `listChildDirectories()`.

## Solution Assessment

Yes. The PR is the narrow maintainable fix: it separates raw-entry scanning from candidate-directory truncation, keeps deterministic sorting and existing containment/size checks, and avoids broad recursive scanning or public config churn.
