## Summary

Keep open: current main still has a source-reproducible plugin discovery path that skips global TypeScript package entries without built JavaScript, and https://github.com/openclaw/openclaw/pull/80557 is an open closing PR for the fix.

## What This Changes

The issue asks OpenClaw to stop silently dropping local TypeScript-source plugins when package metadata routes a global extension directory through compiled-package validation.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80557 or a maintainer-adjusted variant that preserves strict managed installs while allowing intentional local source-checkout plugins.

## Reproduction Assessment

Yes. Source inspection shows a global package entry pointing at TypeScript without a built JavaScript peer follows the package-entry path and is excluded with the compiled-runtime diagnostic.

## Solution Assessment

Yes, with maintainer policy acceptance. The linked PR's install-record based managed-directory classification is narrower than flipping manifest precedence, but it changes global source-loading semantics and should be reviewed before merge.
