The issue asks OpenClaw to strip redundant self-provider prefixes for Google, Google Gemini CLI, Google Vertex, XAI, OpenAI, and Vercel AI Gateway model ids after the Anthropic-only fix.

## Best Possible Solution

Add a shared self-prefix normalization step at the built-in provider model-id boundary, cover each affected provider with focused lookup and fallback tests, and preserve proxy/upstream provider prefixes.

Yes, at source level. The current code path can retain a self-prefixed `modelId` for the reported non-Anthropic providers and then format it as `provider/modelId`, matching the doubled-prefix failure shape; I did not execute a live gateway repro in this read-only review.

## Solution Assessment

Yes, a narrow normalization repair is the best fit. The maintainable shape is a shared post-alias self-prefix strip with tests for the affected providers, rather than another one-off Anthropic-style branch that could miss future built-ins.
