## Summary

Current `main` already exposes inbound WhatsApp message IDs to hooks and uses them for native quoted replies; the remaining mismatch in the report is that the supported internal hook event is `message:received`, not `message:inbound`.

## Close Comment

Closing after Codex review. Current `main` already implements the requested path for WhatsApp quoted replies: inbound WhatsApp message IDs are mapped into the shared inbound context at `extensions/whatsapp/src/auto-reply/monitor/inbound-dispatch.ts:120`, plugin hooks receive that as `metadata.messageId` in `src/hooks/message-hook-mappers.ts:304`, internal hooks receive it as `context.messageId` in `src/hooks/message-hook-mappers.ts:337`, and both hook paths are fired from `src/auto-reply/reply/dispatch-from-config.ts:579` and `src/auto-reply/reply/dispatch-from-config.ts:590`. Native quoted reply sending is also wired on current `main`: `extensions/whatsapp/src/outbound-base.ts:160` resolves `replyToId` into a WhatsApp `quotedMessageKey`, using inbound message metadata cached in `extensions/whatsapp/src/inbound/monitor.ts:617`. One correction from the original report: the supported internal hook event on current `main` is `message:received` (see `docs/automation/hooks.md:47`), not `message:inbound`. The best proving commit on `main` is `6f04eee2a12e665fb4b0e3b0021dbbf90efdf2a2`. This does not appear to be in the latest tagged release `v2026.4.22` yet, so users may need a newer build than the March report used.
