## Summary

Close as a configuration/support item: MiniMax M2.7 is intentionally text-only in OpenClaw, while MiniMax image understanding is routed through the plugin-owned MiniMax-VL-01 path that already sends base64 data URLs to the VLM endpoint.

## What This Changes

The issue asks OpenClaw to make the image tool work with `agents.defaults.imageModel.primary` set to `minimax/MiniMax-M2.7` by base64-encoding images for MiniMax's Anthropic-compatible API.

## Best Possible Solution

Keep MiniMax M2.7 as a text chat model and use `minimax/MiniMax-VL-01` for `agents.defaults.imageModel`, or remove the explicit image override so OpenClaw can auto-pair the MiniMax image provider.

## Reproduction Assessment

No. The reported `chat content is empty` path was not established on current main; source and tests show the supported MiniMax image route is `MiniMax-VL-01` with base64 VLM payloads, while `MiniMax-M2.7` is text-only.

## Solution Assessment

No. Encoding images in the generic MiniMax Anthropic-compatible chat path is not the right fix for this configuration; the maintainable path is to keep image understanding on the plugin-owned MiniMax-VL endpoint/default model.
