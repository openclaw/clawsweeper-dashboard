## Summary

Current main already implements the #58621/#58583 logging behavior: gateway file logs default to a dated rolling path, rotate at `logging.maxFileBytes` with a 100 MB default, keep five numbered archives, and continue writing to a fresh active file instead of suppressing diagnostics. The current implementation is simpler than the PR branch and avoids the branch-specific persistent-FD review issues.

## Best Possible Solution

Close #58621 as implemented on main. Keep the shipped simple rotation path; track any future persistent-FD optimization or alternate archive naming as a separate, narrower proposal if maintainers want it.
