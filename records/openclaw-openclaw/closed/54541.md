## Summary

Current `main` does not auto-run arbitrary skill-side scripts during discovery/load. Skill loading reads and parses `SKILL.md`, ClawHub install copies a skill folder into `workspace/skills`, and executable install actions only run through explicit `metadata.openclaw.install` handling. The issue’s reported `main()` auto-execution path does not match current OpenClaw behavior.
