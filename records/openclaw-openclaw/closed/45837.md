## Summary

Current `main` already provides the requested pre-delivery outbound interception as the plugin hook `message_sending`, which exposes outbound message fields and allows mutation or cancellation before channel delivery.

## Evidence

- **Pre-send hook contract exists:** `message_sending` is a first-class plugin hook, and its event/result types expose outbound `to`/`content` plus mutable `content` and `cancel` return fields.
  - file: [src/plugins/hook-message.types.ts:45](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/plugins/hook-message.types.ts#L45)
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Hook runner explicitly supports modify/cancel semantics:** The hook runner documents `message_sending` as allowing plugins to modify or cancel outgoing messages, with terminal `cancel=true` behavior.
  - file: [src/plugins/hooks.ts:733](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/plugins/hooks.ts#L733)
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Delivery path invokes pre-send hook before adapter send:** `applyMessageSendingHook()` runs before outbound delivery, updates the payload text when a hook returns new content, and skips delivery when the hook returns `cancel`.
  - file: [src/infra/outbound/deliver.ts:567](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/infra/outbound/deliver.ts#L567)
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Core delivery loop uses the hook before sending:** `deliverOutboundPayloadsCore()` calls the `message_sending` hook before `sendPayload`/channel send work and `continue`s without sending when the hook cancels.
  - file: [src/infra/outbound/deliver.ts:834](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/infra/outbound/deliver.ts#L834)
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Regression coverage proves cancellation blocks delivery:** The outbound delivery test verifies that when a `message_sending` hook returns `cancel: true`, the lower-priority hook is skipped and the channel send function is not called.
  - file: [src/infra/outbound/deliver.test.ts:1096](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/infra/outbound/deliver.test.ts#L1096)
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Docs describe the pre-send hook publicly:** The SDK docs document `message_sending` cancel semantics as the outbound pre-send hook behavior, while the internal hooks docs still show `message:sent` as post-delivery only.
  - file: [docs/plugins/sdk-overview.md:239](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/docs/plugins/sdk-overview.md#L239)
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)

## Risks / Open Questions

- The implemented surface is `message_sending` in the plugin hook system, not a workspace internal hook literally named `message:before_send`, so anyone depending specifically on internal workspace hooks would still need to migrate to a plugin-based hook.
