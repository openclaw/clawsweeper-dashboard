## Summary

Current main already carries the existing-session/Chrome MCP attach hardening for `openclaw browser` on the built-in `user` profile: the profile is synthesized without config, attach now waits for tabs to become usable and returns a structured browser-profile error instead of falling through, and regression tests cover both normal tab operations and DevToolsActivePort attach failures. The changelog also records follow-up fixes after `v2026.3.13` for this exact attach path.
