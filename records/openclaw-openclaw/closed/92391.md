## Summary

This issue should stay open: current main and the latest release still lack the doctor repair path for persisted `models.providers.ollama-cloud` endpoint config, and the matching implementation is the still-open https://github.com/openclaw/openclaw/pull/92594.

## What This Changes

The issue reports that first-class `ollama-cloud/...` models can fail against a persisted retired `https://ai.ollama.com` endpoint while the older `ollama/<model>:cloud` route works.

## Best Possible Solution

Merge a bounded Ollama doctor migration that repairs retired persisted endpoint config before runtime uses canonical config, while keeping runtime free of a silent fallback for stale config.

## Reproduction Assessment

Yes, at source level. A persisted `models.providers.ollama-cloud.baseUrl` of `https://ai.ollama.com` is passed into the current runtime stream path, and live DNS checks show `ollama.com` works while `ai.ollama.com` does not resolve.

## Solution Assessment

Yes. The linked doctor-migration approach is the narrowest maintainable fix under the repo policy because it repairs stale persisted config instead of hiding it with runtime fallback behavior.
