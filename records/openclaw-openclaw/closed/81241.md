## Summary

Keep open: current main still has a source-reproducible path where Telegram/current-turn metadata is prepended into the submitted user prompt, and the linked fix PR is open but unmerged. The issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Prevent Telegram inbound runtime-context metadata from being appended to the user message body while preserving out-of-band runtime-context delivery.

## Best Possible Solution

Route channel current-turn metadata through the hidden runtime-context custom-message path for embedded runs, while submitting only the user-authored Telegram body as the user prompt.

## Reproduction Assessment

Yes. Source inspection shows Telegram/auto-reply metadata becomes currentTurnContext, buildCurrentTurnPrompt prepends it to promptForModel, and promptActiveSession submits that combined string as the user turn.

## Solution Assessment

Yes. The narrow fix is the linked PR direction: merge currentTurnContext into the existing hidden runtime-context delivery path for embedded runs and leave only the typed body in the submitted prompt.
