## Summary

Current main already implements the reported webchat image-upload path end-to-end, and the same behavior is present in the shipped release `v2026.4.23`: Control UI sends image attachments through `chat.send`, the gateway normalizes and accepts image-only attachment payloads, `chat.send` forwards parsed images into `replyOptions.images`, and the reply runner no longer rejects webchat pure-image turns as empty.
