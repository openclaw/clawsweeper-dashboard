## Summary

Current main already implements the central Slack outbound target-account routing through a safer shared resolver, and that fix shipped in v2026.5.2 with regression coverage; this branch is superseded and still has unsafe account-selection behavior.

## What This Changes

The PR adds a peer-aware raw binding scan in `runMessageAction` so Slack outbound sends choose an account by channel and target peer before the existing agent-keyed fallback.

## Best Possible Solution

Keep the released resolver-based implementation and close this superseded branch instead of merging its raw binding scan.

## Reproduction Assessment

Yes for the historical bug, but not as a current-main failure. The PR body gives a concrete two-account Slack reproduction, and current source plus regression coverage show the old fallback was replaced by target-aware resolution.

## Solution Assessment

Yes for current main, no for this branch. The shipped resolver is narrower and safer because it is route-only, agent-scoped, peer-aware, and already covered by a focused regression test.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Preserve agent scoping in the peer lookup:** `src/infra/outbound/message-action-runner.ts:862-865`
  - body: This new fast path matches only channel plus `peer.id`, then assigns the first matching account before the existing agent-keyed fallback can run. If two agents intentionally bind different Slack accounts to the same peer, sends for the later agent will use whichever binding appears first, crossing bot identity and permission boundaries.
  - confidence: 0.9
- **[P1] Exclude ACP bindings from outbound route selection:** `src/infra/outbound/message-action-runner.ts:849-851`
  - body: Iterating raw `cfg.bindings` includes ACP bindings, while outbound account routing should only consider route bindings. An ACP binding with the same channel, peer, and account id can be selected for a message send even though it is not a route binding.
  - confidence: 0.88
