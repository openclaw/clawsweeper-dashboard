## Summary

Keep open: this is a focused test-only PR for a real Discord resolver coverage gap, but one added isolation case still does not prove the stated non-inheritance behavior because the named account defines its own `dmPolicy`.

## What This Changes

Adds `resolveDiscordAccountDmPolicy` coverage in `extensions/discord/src/accounts.test.ts` for defaults, account/root precedence, default-account selection, and invalid policy normalization.

## Best Possible Solution

Keep the colocated Discord account-unit tests, change the named-account isolation fixture so `work` omits `dmPolicy` and expects the resolver's default `pairing`, then merge once normal PR gates remain green.

## Reproduction Assessment

Not applicable as a user-facing bug reproduction. For the coverage gap, yes: current main has `resolveDiscordAccountDmPolicy` and production consumers, while `accounts.test.ts` lacks direct resolver precedence coverage.

## Solution Assessment

Mostly yes. Colocated resolver unit tests are the narrowest maintainable solution, but the isolation case should omit the named account policy so it locks the intended default-account non-inheritance rule.

Overall correctness: patch is correct
