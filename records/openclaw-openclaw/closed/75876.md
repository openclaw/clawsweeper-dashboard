## Summary

Current main has moved past the v2026.4.29 failure mode: legacy `plugin-runtime-deps` roots are cleanup-only, bundled plugins are no longer supposed to create runtime-dependency staging, and root dist bare imports such as `json5` are backed by package-root dependencies plus a postpublish guard. The proof is on main only; no release tag contains the proof commit yet.

## What This Changes

Ensure generated plugin runtime `json5` imports resolve through the package entrypoint so channel agents do not crash before replying.

## Best Possible Solution

Keep the current main direction: root-owned dist imports resolve from the package root, bundled plugin dependency staging stays retired, and doctor remains the cleanup path for stale v2026.4.29 cache roots.

## Reproduction Assessment

Yes for the released failure signature: the issue gives concrete v2026.4.29 cache paths and `json5/index.js` resolution errors. Current-main verification is source/docs/test based rather than a live gateway run because this checkout has no built dist or installed `node_modules`.

## Solution Assessment

Yes. Retiring generated bundled runtime-deps staging and guarding root dist imports is more maintainable than rewriting `json5` imports to `json5/lib/index.js`, because it preserves normal package entrypoint resolution for this and future root dependencies.
