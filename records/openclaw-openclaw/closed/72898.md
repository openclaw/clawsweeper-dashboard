## Summary

Keep open. Current main still treats iMessage `chat.db` permission denial as ordinary stderr or a non-fatal probe error, while this PR adds the right plugin-local fatal classification and test coverage but still needs the required changelog entry before merge.

This PR detects iMessage RPC Messages `chat.db` permission-denied output, fails pending RPC calls, terminates the child process, marks probes fatal, and adds a probe regression test.

## Best Possible Solution

Ship the plugin-local fatal permission-denial handling with regression coverage and a single `CHANGELOG.md` Fixes entry, while keeping #43440 open for the separate process-identity/TCC redesign.
