## Summary

Current main already removes the CLI/sandbox dependency on `src/channels/registry.ts` for channel ID/bootstrap data, and the registry itself is now lightweight enough that the original TDZ import path is no longer present.

## Evidence

- **CLI bootstrap uses `ids.ts`, not `registry.ts`:** `resolveCliChannelOptions()` now reads `CHAT_CHANNEL_ORDER` from `src/channels/ids.ts`, so CLI program-context initialization no longer pulls `src/channels/registry.ts` into startup.
  - file: [src/cli/channel-options.ts:4](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/cli/channel-options.ts#L4)

  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Sandbox constants also use `ids.ts`:** `DEFAULT_TOOL_DENY` now spreads `CHANNEL_IDS` imported from `src/channels/ids.ts`, covering the other registry-import cleanup that this PR was trying to land.
  - file: [src/agents/sandbox/constants.ts:2](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/agents/sandbox/constants.ts#L2)
  - command: `nl -ba src/agents/sandbox/constants.ts | sed -n '1,220p'`
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Registry no longer eagerly loads plugin runtime:** `src/channels/registry.ts` re-exports channel IDs from `ids.ts` and reads registered-channel state through `getActivePluginChannelRegistryFromState()`, rather than importing the heavier runtime registry path that originally caused init-order trouble.
  - file: [src/channels/registry.ts:1](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/channels/registry.ts#L1)

  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **State accessor is lazy/global-state based:** The registry state helper only reads a symbol on `globalThis`; it does not construct or require the active plugin registry at module evaluation time, which removes the old TDZ-style eager-init edge.
  - file: [src/plugins/runtime-channel-state.ts:20](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/plugins/runtime-channel-state.ts#L20)
