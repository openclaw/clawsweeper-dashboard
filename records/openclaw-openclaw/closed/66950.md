## Summary

Current `main` already contains the Telegram native-command fixes implicated by this report: Telegram `"auto"` native-command defaults are restored via the channel plugin registry, and the command-sync hash cache is now process-local instead of persisting across restarts. Those fixes were shipped in `v2026.4.15`.

## Evidence

- **Process-local Telegram command-sync cache on current main:** `syncTelegramMenuCommands` now keeps cached command hashes in an in-memory `Map` with the explicit comment `Keep the sync cache process-local so restarts always re-register commands.` There is no file-backed `~/.openclaw/telegram/command-hash-*.txt` path in the current implementation.
  - file: [extensions/telegram/src/bot-native-command-menu.ts:214](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/extensions/telegram/src/bot-native-command-menu.ts#L214)
