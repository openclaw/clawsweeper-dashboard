The PR addresses a real remaining lifecycle-control gap, but it is not ready as-is because current main moved Gateway method advertisement and scope ownership into core descriptors, while this branch still wires `channels.restart` through the older static method/scope lists and also lacks real linked-channel proof.

Refresh the branch onto current main, register `channels.restart` through the current core descriptor contract, add redacted real linked-channel proof, and land the additive admin-scoped lifecycle API if maintainers approve the public semantics.

## Reproduction Assessment

Yes for the actionable CLI/RPC gap: current main source and docs lack standalone `openclaw channels start`, `stop`, and `restart`, and `channels.restart` is absent. The transient live WhatsApp DNS recovery path itself was not reproduced in this read-only review.

## Solution Assessment
