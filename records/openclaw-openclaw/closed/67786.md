## Summary

Close as cannot reproduce: current main intentionally requires Google Chat `app-url` add-on tokens to match both `audience` and `appPrincipal`/JWT `sub`, and the issue discussion already found that adding the correct `appPrincipal` fixes the reported 401s.

## Evidence

- **Auth contract on main:** For `audienceType: "app-url"`, OpenClaw accepts Chat issuer tokens directly, but Google Workspace add-on tokens are only accepted when `expectedAddOnPrincipal` matches JWT `sub`; otherwise it returns `missing add-on principal binding` or `unexpected add-on principal` by design.
  - file: [extensions/googlechat/src/auth.ts:151](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/extensions/googlechat/src/auth.ts#L151)
  - sha: [fa976e5b93b3](https://github.com/openclaw/openclaw/commit/fa976e5b93b3819f9e137fe08635764edee38290)
- **Regression coverage for principal binding:** Tests cover the exact cases at issue: missing principal binding is rejected, matching binding is accepted, and mismatched binding is rejected for `app-url` tokens.
  - file: [extensions/googlechat/src/targets.test.ts:333](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/extensions/googlechat/src/targets.test.ts#L333)
  - sha: [fa976e5b93b3](https://github.com/openclaw/openclaw/commit/fa976e5b93b3819f9e137fe08635764edee38290)
- **Current main no longer fails silently:** Webhook handling now logs auth rejection reasons and warns when `appPrincipal` is missing or looks like an email, directly addressing the issue's 'silent 401' symptom.
  - file: [extensions/googlechat/src/monitor-webhook.ts:122](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/extensions/googlechat/src/monitor-webhook.ts#L122)
  - sha: [fa976e5b93b3](https://github.com/openclaw/openclaw/commit/fa976e5b93b3819f9e137fe08635764edee38290)
- **Webhook logging tests:** Tests assert 401 cases log `missing token` and `unexpected add-on principal`, and that misconfigured `appPrincipal` values emit actionable warnings.
  - file: [extensions/googlechat/src/monitor-webhook.test.ts:161](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/extensions/googlechat/src/monitor-webhook.test.ts#L161)
  - sha: [fa976e5b93b3](https://github.com/openclaw/openclaw/commit/fa976e5b93b3819f9e137fe08635764edee38290)
- **Issue discussion identified the config fix:** In the provided issue context, commenter @AlephLau reported that `appPrincipal` must be the JWT `sub` rather than the service-account email, and @sycshk confirmed that this immediately fixed their setup.
  - command: `Provided GitHub context for issue #67786 comments 4275177350 and 4280040282`

## Risks / Open Questions

- The public Google Chat docs page still does not obviously surface `appPrincipal`, so similar reports may recur even though current main now logs actionable warnings.
