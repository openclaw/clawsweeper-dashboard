Overall confidence: 0.86

Full review comments:

- **[P2] Mark resolver candidates as untrusted:** `src/auto-reply/reply/deictic-context.ts:119-120`
  - body: The new resolver block prepends reply/thread snippets into the prompt but describes them as primary candidate referents without saying they are untrusted data or that instructions inside them must not be followed. Existing inbound metadata uses explicit untrusted labels for this boundary; this new higher-priority block should preserve that framing too.
  - confidence: 0.9
- **[P2] Require nearby context for bare English pronouns:** `src/auto-reply/reply/deictic-context.ts:26`
  - body: This regex makes any short sentence starting with `this`, `that`, or `it` a deictic reference before the nearby-context check runs, so ordinary updates like `It still fails after reboot.` receive the resolver note even when no reply or thread context exists. Gate bare pronouns on nearby context or restrict them to fragment/question forms.
  - confidence: 0.88
- **[P2] Select message text before metadata lines:** `src/auto-reply/reply/deictic-context.ts:109`
  - body: `pushCandidate` keeps the last non-empty line, but Slack thread history appends a trailing `[slack message id: ...]` line after the actual message text. In those threads the resolver candidate becomes transport metadata instead of the referent, so pick the latest semantic content line or strip known metadata suffixes first.
  - confidence: 0.87
- **[P2] Include thread starters in the group shaping gate:** `src/auto-reply/reply/chat-shaping.ts:66-67`
  - body: The group-chat early return ignores `threadStarterBody`, even though the later nearby-context check includes it. First-turn thread replies can have only a starter body, so this branch suppresses the shaping note for the exact thread context the PR is trying to use.
  - confidence: 0.86
- **[P2] Treat channel sessions as shared chat for shaping:** `src/auto-reply/reply/get-reply-run.ts:547`
  - body: The PR passes the existing `isGroupChat` flag into chat shaping, but on this head that flag is true only for `ChatType === "group"`, not `channel`. Channel conversations therefore bypass the shared-chat guard and can get persona-shaping guidance without reply/thread context.
  - confidence: 0.84
- **[P2] Limit punctuation shaping to short follow-ups:** `src/auto-reply/reply/chat-shaping.ts:25`
  - body: The punctuation regex makes any message ending in `?` or `!` a conversational signal after only a 1200-character cap and deny-list checks. Detailed technical or product questions that end with punctuation can receive the persona-shaping note even when they are not short follow-ups; bound this trigger to the intended short conversational shape.
  - confidence: 0.82
- **[P3] Remove the loose json matcher:** `src/auto-reply/reply/chat-shaping.ts:13`
  - body: `/json/u` duplicates the earlier word-boundary JSON matcher but also matches substrings such as `jsonl` and `jsonify`, unnecessarily suppressing chat shaping for words that are not structured-output requests.
  - confidence: 0.8
