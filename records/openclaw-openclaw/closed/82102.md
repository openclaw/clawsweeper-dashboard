## Summary

Keep open: current main still compacts large Slack modal submissions into agent-visible system events and the existing plugin interactive dispatcher is only wired for block actions, so the requested plugin modal lifecycle seam remains real work rather than cleanup.

## What This Changes

The issue requests a Slack plugin API path for full modal `view_submission` or `view_closed` state before the agent-visible system event is compacted.

## Best Possible Solution

Add a backward-compatible Slack modal lifecycle plugin contract that lets plugin-owned modals receive full submitted state and return or persist a compact reference while keeping default agent system events small.

## Reproduction Assessment

Yes. Current source and tests show oversized Slack modal inputs are reduced to at most 3 agent-visible inputs, and the modal lifecycle path does not invoke the plugin interactive dispatcher.

## Solution Assessment

Unclear until maintainers choose the API shape. The requested direction is sound, but a new modal lifecycle seam plus summary/reference contract is safer than raising the compact input limit as a transport workaround.
