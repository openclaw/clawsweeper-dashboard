## Summary

Keep open. Current main still skips expired base cron session keys, and the PR is a plausible narrow fix, but it still needs user-facing docs/help alignment and external-contributor real behavior proof before merge.

## What This Changes

The PR changes cron session retention to sweep both base and run cron sessions, adds expired-base coverage, updates reaper wording/log output, and modifies the changelog.

## Best Possible Solution

Land a clean version that switches the cron-session predicate, keeps focused expired-base coverage, updates docs/help, and closes the linked issue only after merge.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: seed an expired `agent:main:cron:<jobId>` row and run `sweepCronRunSessions`; the current `isCronRunSessionKey` filter rejects that base key before pruning.

## Solution Assessment

Mostly yes. Switching the sweep predicate to `isCronSessionKey` with an expired-base regression test is the narrow maintainable fix, but the PR is incomplete until docs/help and real behavior proof are supplied.
