## What This Changes

The PR changes LLM idle-timeout precedence so explicit provider request timeouts still apply to no-timeout runs and explicit run timeouts are not re-bounded by agent defaults, with focused helper tests.

## Best Possible Solution

Land the focused helper change after redacted real run proof shows a no-timeout run honoring a configured provider timeout, or after maintainers explicitly accept helper-test-only coverage for this timeout policy change.

## Reproduction Assessment

Yes, the helper is the narrow owner boundary because the attempt caller already passes both resolved run timeout and provider request timeout there.
