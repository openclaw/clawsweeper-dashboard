## Summary

Current main contains a targeted fix for the dropdown flash path and regression coverage that matches the report; the reporter was right that v2026.5.27 still has the old behavior, but the central problem is fixed on main and not yet shipped in a tagged release.

## What This Changes

The issue reports that selecting another session from the top-right Control UI chat session dropdown flashes the picker but does not switch the active chat session, while sidebar session selection works.

## Best Possible Solution

Keep the current-main guarded blur behavior and ship it in the next release so v2026.5.27 users receive the fixed top-right session picker.

## Reproduction Assessment

Yes, by source comparison: v2026.5.27 always applied the picker search on blur, while current main adds a regression test showing an empty blur no longer clears/reloads the picker before clicking another session. I did not run a live browser repro in this read-only cleanup review.

## Solution Assessment

Yes, the current-main fix is the narrowest maintainable solution for the observed flash: preserve meaningful blur-to-search behavior for non-empty queries while leaving empty blur alone so option clicks remain clickable. No new UI mode, config, or broader session-routing change is needed.
