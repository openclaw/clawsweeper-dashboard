## Summary

Current `main` already implements the requested core capability: OpenClaw’s shared `message` tool can send Telegram media through the channel abstraction, including local file/media inputs, reply/thread metadata, and native Telegram `sendPhoto`/`sendDocument` dispatch with scoped media-access checks.

## Evidence

- **Shared core message abstraction already exists:** The plugin architecture docs say OpenClaw keeps one shared core `message` tool and channel plugins provide scoped discovery/execution behind it, including media-source normalization hooks rather than Telegram-only ad hoc behavior.
  - file: [docs/plugins/architecture.md:178](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/docs/plugins/architecture.md#L178)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Telegram action handler already accepts media send inputs:** Telegram `sendMessage` reads `mediaUrl`/`media`, threading and reply parameters, `silent`, and `forceDocument`, then forwards them into `sendMessageTelegram(...)`.
  - file: [extensions/telegram/src/action-runtime.ts:308](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/extensions/telegram/src/action-runtime.ts#L308)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Telegram send path maps media to native Bot API sends:** `sendMessageTelegram` loads media via the shared outbound media loader and routes images to `sendPhoto`, documents to `sendDocument`, plus other native Telegram media methods as appropriate.
  - file: [extensions/telegram/src/send.ts:814](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/extensions/telegram/src/send.ts#L814)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Agent/message tool path already applies scoped media-access policy:** Before dispatch, the outbound runner normalizes sandbox media params and resolves agent-scoped media access rooted in the agent workspace/local roots.
  - file: [src/infra/outbound/message-action-runner.ts:861](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/infra/outbound/message-action-runner.ts#L861)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Local file sends are constrained by allowed roots/workspace resolution:** Agent-scoped outbound media access resolves workspace/local roots and `loadWebMedia` resolves relative paths against the workspace, then rejects local paths outside allowed roots.
  - file: [src/media/read-capability.ts:83](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/media/read-capability.ts#L83)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **User-facing docs already document Telegram media sends:** The CLI docs already document `send` with `--media` and Telegram-specific `--force-document`, including a Telegram example sending `./diagram.png` through the supported path.
  - file: [docs/cli/message.md:68](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/docs/cli/message.md#L68)
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
