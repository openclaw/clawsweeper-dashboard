## Summary

Issue #37022 can be closed as implemented on main. OpenClaw already hard-stops runaway single-agent runs with an outer retry/iteration cap, returns a structured `retry_limit` error when that cap is hit, and also ships configurable tool-loop and agent-to-agent ping-pong protections. The exact `recursion_limit` parameter name is not exposed, but the requested infinite-loop safeguard is already present.
