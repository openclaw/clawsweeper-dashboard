## Summary

Current main already guards the reported sessionId/sessionFile mismatch on the restart persistence path. The write path for session initialization now goes through `resolveAndPersistSessionFile`, which only reuses a persisted `sessionFile` when `baseEntry.sessionId === sessionId`; otherwise it clears/rebuilds the transcript path and persists the corrected pair. A regression test covers the exact case of rotating to a new transcript path when the same session key gets a new sessionId, and the same code is present in the shipped `v2026.4.22` release.

## Evidence

- **Restart/session init write path uses guarded persistence helper:** `initSessionState` persists the session transcript path via `resolveAndPersistSessionFile`, so restart/session rehydration does not write `sessionFile` directly through the naive resolver.
  - file: [src/auto-reply/reply/session.ts:720](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/auto-reply/reply/session.ts#L720)
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Helper stops reusing stale sessionFile after sessionId rotation:** When the stored entry's `sessionId` differs from the requested `sessionId`, `resolveAndPersistSessionFile` sets `entryForResolve.sessionFile` to the fallback or `undefined` instead of reusing the old path, then persists the new `sessionId`/`sessionFile` pair.
  - file: [src/config/sessions/session-file.ts:21](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/config/sessions/session-file.ts#L21)
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Regression test matches the reported failure mode:** `resolveAndPersistSessionFile` has a test named `rotates to a new transcript path when sessionId changes on the same session key`, asserting that the old transcript path is not reused and the saved store entry gets the new path.
  - file: [src/config/sessions/sessions.test.ts:400](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/config/sessions/sessions.test.ts#L400)
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Fix is already in the latest shipped release:** The `v2026.4.22` release snapshot contains the same guarded helper logic and the same regression test, so this is not just on unreleased main.
