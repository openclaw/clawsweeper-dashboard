## Summary

Keep this PR open. Current main still leaves `ltm list` as a count-only command, so the PR's named behavior is not implemented, even though the newer merged `ltm query` command now provides a broader way to inspect LanceDB memory rows. The patch is focused and the earlier full-scan review concern was addressed in its second commit; the remaining decision is whether maintainers want `ltm list` to become a record-listing alias/shortcut now that `ltm query` exists.

## What This Changes

The PR changes the memory-lancedb `ltm list` command from count-only output to pretty-printed JSON memory rows with a private `MemoryDB.list()` helper, `--limit`, and `--order-by-created-at`.

## Best Possible Solution

Settle the memory-lancedb CLI contract explicitly: either land this PR or an equivalent current-main patch so `ltm list` lists records with tests and docs, or keep `ltm list` count-only and document `ltm query` as the supported inspection command so `ltm list` and `ltm stats` are not accidentally duplicate surfaces.
