## Summary

Keep open. Current main still constructs image, video, and music generation tools eagerly; each tool resolves providers through resolvePluginCapabilityProviders, which builds capability-specific compat config and calls resolveRuntimePluginRegistry without onlyPluginIds. The existing cache only stores manifest-derived plugin ids, so it does not address the redundant unscoped snapshot registry loads reported here.

## Best Possible Solution

Keep this issue open and fix the media capability-provider path directly. The likely maintainer path is either a stable provider-result cache keyed by capability plus config/env/workspace inputs, or passing the manifest-derived onlyPluginIds into resolveRuntimePluginRegistry so image, video, and music do not each materialize the full plugin registry. Add regression coverage that constructs or lists image, video, and music providers in sequence and proves redundant unscoped registry loads no longer occur. Coordinate with #73728 and #73730, but do not treat them as duplicates of this specific resolver bug.
