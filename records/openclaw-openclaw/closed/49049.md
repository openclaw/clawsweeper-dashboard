## Summary

Current `main` already carries the requested gateway timeout behavior: the preauth handshake default is 10s, the server uses that shared timeout, and the client `connect.challenge` timeout also resolves to the same 10s baseline. The shipped `v2026.4.22` release contains the same implementation.
