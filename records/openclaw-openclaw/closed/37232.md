## Summary

Keep open. Current `main` still treats `USER.md` and `MEMORY.md` as durable/bootstrap context, but there is no documented or scaffolded TTL-aware `STATUS.md`/`NOW.md` layer, and even the extra-bootstrap hook rejects nonstandard basenames like `STATUS.md`. The request is therefore not implemented on `main` and remains a plausible docs/product improvement.

## Evidence

- **Memory docs still position `MEMORY.md` as durable memory only:** The memory overview says `MEMORY.md` is for long-term durable facts and daily notes live under `memory/YYYY-MM-DD.md`; there is no TTL/review-date/status-file guidance here.
  - file: [docs/concepts/memory.md:17](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/docs/concepts/memory.md#L17)
- **Workspace file map has no status-layer file:** The workspace docs enumerate `USER.md`, `MEMORY.md`, daily notes, and other bootstrap files, but no `STATUS.md`/`NOW.md` or time-bounded status convention.
  - file: [docs/concepts/agent-workspace.md:62](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/docs/concepts/agent-workspace.md#L62)
- **Prompt injection still uses a fixed bootstrap file set:** Context docs say Project Context injects only `AGENTS.md`, `SOUL.md`, `TOOLS.md`, `IDENTITY.md`, `USER.md`, `HEARTBEAT.md`, and `BOOTSTRAP.md` by default.
  - file: [docs/concepts/context.md:101](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/docs/concepts/context.md#L101)
- **Runtime bootstrap validation excludes `STATUS.md`:** `VALID_BOOTSTRAP_NAMES` and `loadExtraBootstrapFilesWithDiagnostics()` only accept the built-in basenames; unsupported names are rejected as `invalid-bootstrap-filename`. That blocks using the bundled extra-bootstrap hook for a `STATUS.md` layer.
  - file: [src/agents/workspace.ts:171](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/src/agents/workspace.ts#L171)
- **Hook docs explicitly limit extra bootstrap files to recognized names:** The bundled `bootstrap-extra-files` docs say only recognized basenames are loaded, listing `AGENTS.md`, `SOUL.md`, `TOOLS.md`, `IDENTITY.md`, `USER.md`, `HEARTBEAT.md`, `BOOTSTRAP.md`, and `MEMORY.md`.
  - file: [docs/automation/hooks.md:175](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/docs/automation/hooks.md#L175)
- **Default workspace scaffolding does not create a status file:** The dev workspace bootstrap creates `AGENTS.md`, `SOUL.md`, `TOOLS.md`, `IDENTITY.md`, and `USER.md` only; there is no scaffold/template for `STATUS.md` or similar.
  - file: [src/cli/gateway-cli/dev.ts:78](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/src/cli/gateway-cli/dev.ts#L78)
