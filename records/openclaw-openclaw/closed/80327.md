## Summary

Current main already addresses the LM Studio/local OpenAI-compatible disconnect pattern by disabling the unconfigured 120s model idle watchdog for loopback, private-network, and .local provider base URLs; the fix is current-main-only and is not in v2026.5.7.

## What This Changes

The issue reports OpenClaw 2026.5.7 disconnecting long-running Windows LM Studio OpenAI-compatible completions after about two minutes.

## Best Possible Solution

Keep the current main local-provider idle-timeout behavior and ship it; any remaining failure with explicit provider timeouts or non-local hostnames should be tracked as a narrower follow-up.

## Reproduction Assessment

No live high-confidence reproduction was run against Windows and LM Studio. The reported timestamps line up with the 120s idle watchdog, and current source/tests show the local-provider path now avoids that default timeout.

## Solution Assessment

Yes. The current fix is the narrow maintainable path because it scopes the no-idle-watchdog behavior to local provider baseUrls instead of raising global agent timeouts for all providers.
