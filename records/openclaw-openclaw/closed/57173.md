## Summary

Current `main` already fixes the reported root cause: channel config validation now pulls plugin-provided channel schemas from the manifest registry and lets higher-precedence installed plugins override bundled channel schemas, so an installed Feishu plugin can supply its own `channels.feishu` schema during startup validation.

## Evidence

- **Startup validation now uses plugin channel schema metadata:** `validateConfigObjectWithPlugins` builds a channel-schema map from generated bundled metadata plus `collectChannelSchemaMetadata(info.registry)`, then validates `channels.<id>` against that schema during config validation.
  - file: [src/config/validation.ts:833](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/config/validation.ts#L833)
