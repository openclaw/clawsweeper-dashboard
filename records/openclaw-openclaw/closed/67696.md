## Summary

Current main and the v2026.4.24 release already normalize chat content arrays into text before agent prompting and final outbound delivery, preventing the `[object Object]` artifacts reported against 2026.4.14 for Telegram and Mattermost.

## Best Possible Solution

Close this issue as implemented. Keep the shipped content-array text extraction and string-only reply payload path; if a new provider/channel combination still emits `[object Object]`, it should be reported as a fresh bug with current-version logs and the exact channel/provider payload shape.
