## Summary

This PR is a docs-only recommendation for an author-linked third-party Mistral MCP server. Current main already has the generic MCP, bundle-plugin, and community-plugin paths for this kind of extension, while project scope says plugin discovery, provenance, and security review belong in ClawHub rather than first-party provider docs. The proposed `npx -y` snippet also creates a supply-chain concern by silently executing an unvetted community package from official docs.

## Best Possible Solution

Close this PR and keep first-party Mistral docs focused on the shipped provider surface. The useful path is for the author to publish or list `mistral-mcp` through ClawHub as a plugin/bundle or community package, and for maintainers to consider a separate generic MCP note only if provider docs should mention unsupported specialized Mistral APIs without promoting one package.
