## Summary

Current main no longer sends ACP/ACPX liveness checks as ordinary `session/prompt` user turns. The bundled ACPX runtime is pinned to `acpx@0.6.0`, exposes a `getStatus` control path, and OpenClaw's cached-handle probe uses that status path rather than constructing a prompt. The pinned upstream package's probe starts and closes the ACP client without calling `prompt`, and its status method reads the session record instead of sending a turn.
