## Summary

Current main already preserves the originating delivery context for spawned worker/subagent runs and explicitly falls back to the requester session's stored Telegram route when a completion only carries the channel. That behavior is covered by a regression test, and the same implementation is present in release v2026.4.22.

## Evidence
