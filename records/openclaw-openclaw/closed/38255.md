## Summary

Keep open. Current main still has only internal Telegram draft-stream throttle/min-initial parameters; it does not expose `channels.telegram.streamThrottleMs` or `channels.telegram.minInitialChars` through config/schema/types/docs or wire them from Telegram dispatch. The PR is useful but still needs maintainer review because it changes shared draft-stream loop behavior and leaves a timer-safe upper-bound review point unresolved.

## What This Changes

The PR adds optional Telegram `streamThrottleMs` and `minInitialChars` config fields, passes them into Telegram draft streaming, adjusts draft/message throttle and final-drain behavior, regenerates config metadata/docs, and adds focused config, Telegram, and shared draft-stream tests.

## Best Possible Solution

Expose the two Telegram preview knobs consistently across schema, exported config types, generated metadata/docs, and Telegram dispatch; keep native draft defaults smoother while preserving message-edit fallback throttling; enforce timer-safe bounds; and land only after targeted Telegram plus shared draft-loop validation confirms final flush and fallback behavior.
