Full review comments:

- **[P1] Clear recognized rate-limit sends from ambiguous delivery state:** `extensions/telegram/src/draft-stream.ts:178-180`
  - body: A wrapped 429 can be recognized by `isTelegramRateLimitError` through `retry_after` and error text without satisfying either condition here, leaving `messageSendAttempted` true.   - confidence: 0.98
- **[P1] Schedule a wake-up when retry-after expires:** `extensions/telegram/src/draft-stream.ts:220-221`
  - body: Returning false during suspension only preserves pending text; it does not arrange another loop attempt, and `flush()` clears the existing throttle timer. A quiet stream therefore never resumes unless another update or manual flush happens, so schedule a retry at the suspension deadline.
  - confidence: 0.97
- **[P1] Make finalization honor the active flood suspension:** `extensions/telegram/src/draft-stream.ts:220`
  - body: The `!streamState.final` exemption lets `stop()` immediately retry while Telegram's `retry_after` is still active, because `stop()` sets final before flushing and directly sends remaining text. Finalization must await or share the same scheduled suspension instead of bypassing it.
  - confidence: 0.99
