## Summary

Current source and the latest release already allow active-agent workspace files through the WebChat MEDIA/local-media root path, with tests covering the workspace root while keeping broad agent state directories excluded.

## What This Changes

The issue asks WebChat MEDIA attachment validation to accept files generated inside an agent workspace instead of showing “Outside allowed folders.”

## Best Possible Solution

Keep the shipped agent-scoped workspace media-root behavior; do not widen WebChat preview roots to broad `agentDir` state paths without a separate security/product case.

## Reproduction Assessment

No, not on current main. Source and release-tag inspection show active-agent workspace roots are already included before WebChat decides whether to show “Outside allowed folders.”

## Solution Assessment

Yes. The shipped workspace-root allowlist is the maintainable fix for generated workspace files, while keeping broad agent state directories out of the preview roots preserves the file-access boundary.
