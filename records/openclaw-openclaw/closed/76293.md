## Summary

Keep open: current main still lacks a bounded consecutive idle-timeout cost breaker, and open PR #76345 uses closing syntax for this issue, so cleanup should wait for that PR or an equivalent fix to merge.

## What This Changes

Request a bounded retry/backoff or circuit-breaker path for LLM idle timeouts so heartbeat/model calls cannot loop rapidly and create paid-provider cost spikes.

## Best Possible Solution

Ship a cross-attempt idle-timeout breaker or equivalent cost cap in the embedded runner, with focused regression coverage for repeated zero-output timeouts across retries, profile rotation, and model fallback.

## Reproduction Assessment

Yes, by source inspection: current main can still re-enter the model stream path after an idle timeout without a 3-5 consecutive idle-timeout cap. I did not run a live paid-provider reproduction.
