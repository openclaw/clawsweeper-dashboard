## Summary

Current main already fixes the reported ZAI/GLM `/new` failure by giving bare `/new` and `/reset` turns a non-empty transcript-facing user prompt (`[OpenClaw session new]` / `[OpenClaw session reset]`) while keeping startup instructions out of the visible transcript. The latest release cited in the report, v2026.4.26, still had the empty prompt behavior, so the report matches released behavior but is fixed on main.

## Best Possible Solution

Keep the current main behavior: bare `/new` and `/reset` should submit a non-empty transcript/user prompt marker while keeping startup instructions runtime-only, then ship it in the next release. No provider-specific ZAI padding is needed for this reported `/new` path.
