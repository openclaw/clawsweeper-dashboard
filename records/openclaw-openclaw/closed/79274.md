## Summary

Keep open. Current main and the latest release still have the reported serial deep code-safety scan shape, and the related bounded-concurrency PR was closed unmerged, so the central performance problem is not implemented on main.

## What This Changes

The issue asks to reduce long silent `openclaw security audit --deep` runs over many skill/plugin files with bounded parallel scanning, progress/timing output, caching, collector limits, and optional collector selection.

## Best Possible Solution

Scope a first fix around bounded deterministic scan concurrency plus visible progress/timing, then treat persisted cache, per-collector limits, and `--only` as approved follow-ups unless maintainers want the full bundle together.
