## Summary

Current `main` already covers the reported subagent/ACP completion-notification gaps: ACP `sessions_spawn` runs are registered for lifecycle/completion tracking, completion announces retry transient delivery failures, active requesters can receive queued/steered follow-up delivery, and spawned subagent sessions now preserve delivery context for correct announce routing.

## Evidence

- **ACP spawns now register completion tracking:** `sessions_spawn` reads `expectsCompletionMessage` and, for accepted ACP child runs, calls `registerSubagentRun(...)` with the child run/session, requester origin, cleanup mode, and completion-message flag. This is the missing tracking path the issue described for ACP spawns.
