## Summary

Current main implements the requested WebChat real-time refresh behavior. Instead of idle polling, the Control UI now subscribes to session events, reloads chat history on same-session `session.message` events while idle, defers that reload until active runs end, and reloads after final chat events. The same implementation is present in the latest release tag v2026.4.24.
