## Summary

Keep open. The PR adds useful coverage, but the new `~\\...` assertion does not match current `main` behavior on POSIX and has already been flagged in review. That makes this a concrete fixable PR issue, not a conservative auto-close candidate.

## Evidence

- **Current implementation expands `~` but does not normalize backslashes on POSIX:** `normalizeExecutablePath` only replaces a leading `~` and then passes the result directly to `path.resolve(...)`. For `~\\AppData\\...`, that preserves backslashes on Linux/macOS instead of converting them to `/`.
  - file: [extensions/browser/src/browser/config.ts:131](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/extensions/browser/src/browser/config.ts#L131)
  - command: `nl -ba extensions/browser/src/browser/config.ts | sed -n '105,140p'`
  - sha: [d068cb960de4](https://github.com/openclaw/openclaw/commit/d068cb960de47d5587c5942f6beb86e0f124b449)
- **Current main tests do not yet cover the new edge cases:** `main` currently tests `~/...`, non-tilde paths, and blank values, but not bare `~`, `~\\...`, or embedded `~` cases. The PR is additive rather than obsolete.
  - file: [extensions/browser/src/browser/config.test.ts:144](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/extensions/browser/src/browser/config.test.ts#L144)
  - command: `nl -ba extensions/browser/src/browser/config.test.ts | sed -n '135,185p'`
  - sha: [d068cb960de4](https://github.com/openclaw/openclaw/commit/d068cb960de47d5587c5942f6beb86e0f124b449)
- **POSIX reproduction shows the PR's Windows-style expectation is wrong on current main:** A direct Node reproduction on Linux produced `actual: /home/runner\\AppData\\Local\\Chromium\\chrome.exe` and `expected: /home/runner/AppData/Local/Chromium/chrome.exe`, with `equal: false`.
  - command: `node -e "const os=require('node:os'); const path=require('node:path'); const value='~\\\\AppData\\\\Local\\\\Chromium\\\\chrome.exe'; const actual=path.resolve(value.replace(/^~(?=$|[\\\\/])/, os.homedir())); const expected=path.resolve(os.homedir(), 'AppData/Local/Chromium/chrome.exe'); console.log(JSON.stringify({platform:process.platform, homedir:os.homedir(), actual, expected, equal:actual===expected}, null, 2));"`
