## Summary

Keep open: this is a useful default-off Telegram transport feature, but the current patch has a source-level delivery bug around late progress callbacks and the attached proof is still not clear enough to satisfy the live Telegram proof bar.

## What This Changes

The PR adds default-off, allowlist-gated Telegram DM native `sendMessageDraft` previews for transient tool progress, plus Telegram-scoped config/schema/docs/changelog updates and focused dispatch tests.

## Best Possible Solution

Track native draft progress as part of the final-answer boundary, suppress all late progress after final delivery, keep final answers on the persistent path, and land only with visible redacted Telegram proof plus maintainer acceptance of the default-off transport policy.

## Reproduction Assessment

Not applicable as a user bug report, but the review finding is source-reproducible: native draft success stops the native draft for final delivery without setting the same suppression state used by edited-preview progress. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

No: the feature boundary is reasonable, but the current patch does not fully preserve the final-answer boundary under late progress callbacks. The safer path is to fix the native-progress state transition and prove it in live Telegram before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Suppress native progress after final delivery:** `extensions/telegram/src/bot-message-dispatch.ts:701-703`
  - body: When `nativeToolProgressDraft.update()` succeeds, this branch only records `streamToolProgressLines` and returns. Unlike the edited-preview path, it never marks the answer lane as tool-progress-only or otherwise suppresses later progress after the final answer. If a late progress callback fires after final delivery, `nativeToolProgressDraft.stop()` makes the native update return false and the code falls back to the persistent edited-preview lane, so canary users can still get tool chatter after the final answer. Track a native-progress-active/finalized state or set suppression on final delivery, and cover native success -> final -> late progress.
  - confidence: 0.82
