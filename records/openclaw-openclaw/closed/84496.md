## Summary

Current main and the stable v2026.5.18 release already implement the requested Ollama guard, with regression coverage for both configured and runtime non-reasoning model paths.

## What This Changes

The issue reports native Ollama `/api/chat` requests forwarding truthy `think` values to models marked `reasoning: false`, causing Ollama HTTP 400 responses for non-thinking models.

## Best Possible Solution

Keep the shipped v2026.5.18 guard and point affected 2026.4.x users to upgrade rather than carrying another issue for the same fixed Ollama request-shaping bug.

## Reproduction Assessment

Yes. The reported 2026.4.23 failure path is clear from the old unconditional native Ollama think forwarding and the upstream Ollama contract; current main has focused request-body tests showing the fixed non-reasoning paths omit `think`.

## Solution Assessment

Yes. The merged fix is the narrow provider-owned solution: it suppresses only truthy native Ollama `think` for models positively marked non-reasoning while preserving supported thinking models and explicit `think: false`.
