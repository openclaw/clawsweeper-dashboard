## Label Justifications

- `P3`: This is a low-risk provider-configuration feature/ergonomics request that is already covered by current memory search configuration.
- `impact:auth-provider`: The request is about embedding provider routing, model selection, and avoiding OpenAI API-key requirements for memory search.

## Summary

Current main and the latest release already support the central request through the canonical `agents.defaults.memorySearch.provider` plus `model` and `remote` settings, including Ollama and custom Ollama provider IDs for local GPU hosts.

## What This Changes

Request to let memory semantic search use a configurable embedding provider/model such as local Ollama instead of always using OpenAI embeddings.

## Best Possible Solution

Keep the canonical `agents.defaults.memorySearch.provider`/`model`/`remote` configuration path and point users to that instead of adding a second `memory.embeddingProvider` alias.

## Reproduction Assessment

Not applicable as a feature request; the review did not need a failing current-main reproduction. Source and docs show the requested provider/model configuration is already present.

## Solution Assessment

Yes. The existing `agents.defaults.memorySearch.provider` with optional `model` and `remote` settings is the narrow canonical solution, and adding a parallel top-level `memory.embeddingProvider` key would conflict with the repo’s config-schema guidance.
