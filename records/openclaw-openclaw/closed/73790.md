## Summary

Keep open: the branch targets a real fallback persistence bug, but the current head still misclassifies user live-switch redirects handled inside the fallback runner, and contributor real behavior proof is still missing.

## What This Changes

The PR changes direct-agent and isolated-cron session persistence to avoid saving automatic fallback models as runtime models, preserve contextTokens, add regression tests, and update the changelog.

## Best Possible Solution

Have runWithModelFallback return explicit selection-source metadata and make both direct-agent and cron persistence use that contract so automatic fallbacks are not persisted while user-initiated live switches are.

## Reproduction Assessment

Yes, by source inspection rather than execution. Configure a primary plus later fallback candidate, have the first attempt throw LiveSessionModelSwitchError targeting that later candidate, and runWithModelFallback redirects internally and returns the target with attempts: [], which this PR classifies as fallback.

## Solution Assessment

No. The no-persist fallback direction is right, but comparing only final provider/model values is not the best contract; an explicit selection-source result is the narrower maintainable fix.
