## Summary

Current main and shipped releases now contain the bounded bootstrap-cache implementation requested here, with regression coverage and merged-PR provenance, so this issue is no longer necessary to keep open.

## What This Changes

The issue requested capping the per-session workspace bootstrap snapshot cache and evicting older session keys to prevent unbounded gateway memory retention.

## Best Possible Solution

Keep the bounded bootstrap-cache implementation and track broader gateway memory-retention work separately, such as https://github.com/openclaw/openclaw/issues/92844, because that report describes a different systemPromptReport/session-registry retainer path.

## Reproduction Assessment

Yes. The old release source showed an unbounded session-keyed `Map` with `cache.set(...)` and no eviction, and current source shows the bounded prune path on the same cache.

## Solution Assessment

Yes. The implemented 64-entry cap with oldest-key pruning and active-key refresh is the narrow maintainable fix for this cache layer without adding configuration or broader product policy.
