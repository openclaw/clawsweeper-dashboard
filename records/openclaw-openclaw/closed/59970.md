## Summary

This is a useful behavior-policy bundle, but it fits OpenClaw's plugin/ClawHub path better than a new default core behavior. Current main already exposes the needed seams: reply metadata is present in the prompt, prompt/context policy can be injected by plugins or context engines, retrieval policy already lives in memory/context plugins, and sub-agent/reporting behavior is exposed through documented sub-agent hooks/runtime helpers. No concrete missing core API is identified.

## Best Possible Solution

Close this umbrella as ClawHub/plugin scope. The best path is a behavior-pack plugin or skill bundle using `before_prompt_build`, context-engine/memory seams, message hooks, and sub-agent runtime hooks. If implementation exposes a specific missing core seam, open a narrower core issue for that API gap.
