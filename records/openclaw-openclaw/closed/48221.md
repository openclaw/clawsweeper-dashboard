## Summary

Close PR #48221 as ClawHub/provider-plugin work rather than a core fallback. The PR describes third-party proxy behavior, and current main has explicit provider-owned replay hooks that can express this without teaching core that every non-native `anthropic-messages` endpoint must drop thinking blocks. The current core fallback is intentionally narrow, and tests now preserve thinking blocks for newer Claude models on unowned Anthropic transports, so the PR’s broad core change is both stale and in tension with current replay policy.

## Best Possible Solution

Do not merge this broad core fallback. Keep current main’s provider-owned replay design, and move brconnector or other non-native Anthropic Messages proxy behavior into a ClawHub/provider plugin that uses `buildReplayPolicy` or `sanitizeReplayHistory`; open a focused plugin API issue only if those hooks prove insufficient.
