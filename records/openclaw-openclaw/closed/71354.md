## Summary

This PR is superseded by the maintainer fix for the same Tailscale status confusion. Current main does not adopt this PR's mode-off hostname probe; instead it shipped the product direction from #71790: label the config value as `Tailscale exposure`, show daemon state separately when available, and document that `off` means OpenClaw is not managing Serve/Funnel, not that the local Tailscale daemon is stopped.

## Best Possible Solution

Close this PR as superseded by #71790 and commit 1fe0e6fc4aaa1fcb47ab4c0f9d0f390dd11bb086. Keep the shipped wording/docs fix; if OpenClaw should actively probe and print MagicDNS while exposure mode is off, track that as a new focused issue with explicit privacy/PATH requirements and bootstrap regression tests.
