## Summary

Current `main` already carries macOS image attachments end to end: the macOS composer accepts pasted and picked images, the chat view model base64-encodes and sends them as `attachments`, the macOS gateway transport includes those attachments in `chat.send`, and gateway tests verify those attachments are normalized and dispatched as images rather than being reduced to plain text.

## Evidence

- **macOS composer accepts pasted and picked images:** The macOS chat composer handles pasteboard images and file-backed image references, delivers them via `onPasteImageAttachment`, and also exposes an image-only file picker. Dedicated tests cover PNG clipboard payloads and file-URL clipboard payloads.
