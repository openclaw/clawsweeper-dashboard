## Summary

Close #72271 as a duplicate of the earlier open tracker #42856. The new issue reports the same Telegram-visible `config.schema.lookup` path-not-found warning leak, and current `main` still matches the root-cause evidence already tracked there: the gateway tool treats the full dotted action as mutating, while gateway scopes classify the underlying method as read-only.

## Best Possible Solution

Close #72271 as a duplicate and keep the remaining product work on #42856. The best fix is still the root-cause classification change from the #42898/#42962 discussion: make read-only compound gateway actions such as `config.schema.lookup` non-mutating for warning policy, while preserving warnings for real gateway writes/restarts and adding regression coverage for both cases.
