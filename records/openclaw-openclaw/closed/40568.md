## Summary

Current `main` already contains the tool-schema normalization needed for this report: empty/typed-object tool schemas are normalized away from malformed `required: null`, the OpenAI strict compat path coerces non-array `required` to `[]`, and the shipped tests lock that behavior in. The same implementation is present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
