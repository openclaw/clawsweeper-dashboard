## Best Possible Solution

Keep the current main implementation and ship it in the next release; reopen only if a build containing the fix still reproduces the 401 after a Control UI config save.

## Reproduction Assessment

Yes for the reported release path: v2026.5.18 lacks the config-write refresh option that current main added, and the issue gives concrete local token-auth Control UI save steps. I did not run a live gateway because this review was read-only.

## Solution Assessment

Yes. The main-branch fix is the narrow owner-boundary repair: config writes request a lighter secrets refresh, the active auth-store snapshot is preserved, and existing bootstrap config auth tests keep the endpoint protected for shared and paired device tokens.
