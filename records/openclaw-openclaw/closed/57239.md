Full review comments:

- **[P1] Restore reset-triggered queue admission:** `src/auto-reply/reply/queue-policy.ts:10`
  - body: The PR replaces `resetTriggered` with `forceRunNowWhenActive`, but current main uses `resetTriggered` to let reset/interrupt turns run immediately while another run is active, and the PR-head test file still passes `resetTriggered`. Please preserve the existing reset parameter and add the fast-lane override alongside it.
  - confidence: 0.95
- **[P1] Isolate the fast-lane reply operation key:** `src/auto-reply/reply/get-reply-run.ts:1196`
  - body: The fast-lane path still passes the parent `sessionKey` into `runReplyAgent`, and `runReplyAgent` uses that key for `createReplyOperation`; with the parent run active, the fast-lane turn can hit `ReplyRunAlreadyActiveError` and return the shutdown/suppression path instead of responding immediately.
  - confidence: 0.9
- **[P1] Keep sandboxSessionKey separate from lane identity:** `src/agents/pi-embedded-runner/run/params.ts:41-42`
  - body: `sessionLaneKey` is a scheduling concern, but `sandboxSessionKey` is the current runtime-policy and process-tool scope identity used by embedded attempts. Replacing the sandbox field with the lane key breaks that boundary; preserve `sandboxSessionKey` and add any lane override as a separate field.
  - confidence: 0.92
- **[P1] Preserve session-key backfill before lane resolution:** `src/agents/pi-embedded-runner/run.ts:398-400`
  - body: Current main backfills `sessionKey` before resolving the embedded session lane so hooks, LCM, compaction, and downstream consumers receive a non-null session key. This branch removes that backfill while changing lane resolution, so callers that omit `sessionKey` regress to missing session identity downstream.
  - confidence: 0.86
- **[P2] Enforce fast-lane tool limits in runtime code:** `src/auto-reply/reply/get-reply-run.ts:1171-1173`
  - body: The fast-lane path only appends a prompt asking the model to avoid tools. Since `RunEmbeddedPiAgentParams` already supports hard tool controls, a parallel quick turn should set a runtime no-tool or allowlist policy instead of relying on prompt text while another session run is active.
  - confidence: 0.84
