## Summary

Keep open. Current main still forwards raw read-only session model fields into trim-based model resolution, and this PR is a narrow, appropriate fix with focused regression coverage and a changelog entry.

## What This Changes

The PR normalizes persisted status session model/provider fields before resolving the selected model, adds regression tests for non-string session values, and records the user-facing status crash fix in the changelog.

## Best Possible Solution

Land this narrow status normalization fix after normal maintainer review; broader model display hardening can stay separate unless a second reachable crash is shown.

## Reproduction Assessment

Yes. A non-string `model`, `modelProvider`, `providerOverride`, or `modelOverride` in a read-only session entry can flow from `readSessionStoreReadOnly` into `.trim()` calls when `openclaw status` builds session rows.

## Solution Assessment

Yes. Normalizing unknown persisted values at the status resolver boundary preserves the typed model-selection contract and fixes the read-only path that bypasses `loadSessionStore` normalization.
