## Summary

Current `main` already contains a targeted `sessions_send` guard for parent-owned background ACP child sessions that skips the A2A ping-pong/announce follow-up responsible for the reported repeated completion loop, and the fix is documented as shipped in `v2026.4.23`.

## Evidence

- **Loop-prevention guard in current main:** `sessions_send` now skips the A2A follow-up when the requester is the parent of its own background one-shot ACP child, and returns `delivery.status = "skipped"` instead of starting the announce ping-pong flow.
  - file: [src/agents/tools/sessions-send-tool.ts:294](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/agents/tools/sessions-send-tool.ts#L294)
  - command: `nl -ba src/agents/tools/sessions-send-tool.ts | sed -n '288,327p'`
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **Ownership classifier for affected session type:** The guard is driven by a dedicated classifier for parent-owned background ACP sessions, keyed off `spawnedBy` / `parentSessionKey`, so the skip is narrowly scoped to parent-child background runs.
  - file: [src/acp/session-interaction-mode.ts:8](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/acp/session-interaction-mode.ts#L8)
