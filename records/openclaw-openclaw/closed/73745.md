## Summary

Current main fixes the reported Discord native `/reset` empty-prompt path. Discord native commands still dispatch a generated `/reset` command through the shared reply pipeline, and the shared reset handler now short-circuits bare `/new` and `/reset` with a local acknowledgement instead of falling through to the model run. Regression coverage asserts bare `/RESET` returns `✅ Session reset.` and does not call the embedded model runner. The v2026.4.26 tag still shows the reported fall-through behavior, so this is fixed on current main but not proven shipped in a later release tag.

## Best Possible Solution

Close this issue as fixed on current main, keep the shared bare reset short-circuit and non-empty transcript marker, and ship the fix in the next tagged release so v2026.4.26 users have an upgrade path.
