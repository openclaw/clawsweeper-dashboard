## Label Justifications

- `P1`: The reported state can leave memory recall degraded across normal gateway syncs until repaired, which is a broken agent memory workflow for affected users.
- `impact:session-state`: The issue is about memory index identity state drifting from the indexed SQLite chunks and leaving agents in a dirty/degraded state.

## Summary

Current main and the latest release already include the narrow missing-index-identity self-heal trigger for chunk-backed memory indexes, with regression coverage for non-forced runtime sync and a safety guard that avoids downgrading semantic chunks when the provider is unavailable.

## What This Changes

The issue requests widening memory-core's missing `indexIdentity` reindex trigger so agents with indexed chunks but missing metadata can self-heal during normal sync cycles, not only CLI-triggered sync.

## Best Possible Solution

Keep the shipped safe self-heal behavior, and use the broader canonical memory lifecycle issue for remaining live-manager publication or provider-unavailable semantic-index cases.

## Reproduction Assessment

Yes, at source level. The pre-fix failure path is the missing identity plus existing chunks early-return in `runSync`, and current regression coverage exercises the repaired non-forced sync path.

## Solution Assessment

Yes for this narrow issue. The current implementation widens the runtime reindex trigger while preserving a semantic-index safety guard, which is safer than unconditionally rebuilding every missing-meta chunk state.
