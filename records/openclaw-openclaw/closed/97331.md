## Summary

Keep open: current main and the latest release still copy cached parent `contextTokens` into dashboard child sessions, and the issue has a protected `maintainer` label plus an open linked candidate fix.

## What This Changes

The issue reports that dashboard child sessions can inherit stale cached context-budget metadata from a parent session instead of deriving the active model budget.

## Best Possible Solution

Review and land the linked candidate fix if it preserves provider/model/auth/runtime inheritance while dropping cached context and usage metadata, then let the merged PR close this issue.

## Reproduction Assessment

Yes. Current source shows `sessions.create` copies `parentEntry.contextTokens` into the child entry and row projection prefers that stored value before model-derived context tokens; I did not run a live dashboard repro in this read-only review.

## Solution Assessment

Yes, the linked PR is the narrow owner-boundary fix shape: keep inherited provider/model/auth/runtime settings but stop copying cached context and usage metadata. A separate ClawSweeper repair branch would duplicate the open candidate fix.
