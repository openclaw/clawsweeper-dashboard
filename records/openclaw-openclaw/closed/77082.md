## What This Changes

The issue reports that OpenClaw 2026.5.2 cannot route first-class `web_search` to an installed Brave plugin and that explicitly pinned DuckDuckGo searches return bot-detection challenges.

## Best Possible Solution

Keep #77073/#77074 as the canonical Brave runtime-provider fix path, and treat pinned DuckDuckGo challenges as documented experimental-provider behavior unless a new report isolates a separate DuckDuckGo transport regression.
