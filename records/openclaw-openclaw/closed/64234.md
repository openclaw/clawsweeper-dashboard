- **[P1] Wire encryption before accepting the at-rest config:** `src/config/zod-schema.ts:1047-1049`
  - body: The schema accepts `privacy.atRest.enabled`/`passphrase`, and the PR advertises encrypted session files, but the canonical transcript path still creates and appends plaintext JSONL through `writeJsonlEntry`/`appendJsonlEntry`. Enabling this config would give users a false security boundary unless transcript reads and writes are encryption-aware, so either wire it through the session store or remove the config and claims for this PR.
  - confidence: 0.91
- **[P1] Pass privacy config through embedded prompts:** `src/agents/pi-embedded-runner/system-prompt.ts:60-89`
  - body: `buildEmbeddedSystemPrompt` never accepts or forwards `privacyConfig` to `buildAgentSystemPrompt`, and the embedded attempt call only passes `contextFiles`. That means channel/gateway users do not get system-prompt masking, context-file suppression, or context PII redaction even when configured, while the CLI path does pass `params.config?.privacy`.
  - confidence: 0.93
- **[P2] Honor the documented PII default:** `src/privacy/payload-redact.ts:117-119`
  - body: The new redactor returns no-op unless `config.pii.enabled` is explicitly true, but the new type comments and PR body describe PII as enabled by default under `privacy.enabled=true`. A user setting only the master privacy switch would still leak tool output, context, and TTS PII; make this guard default-on (`pii.enabled !== false`) or update the public contract and tests to require explicit opt-in.
