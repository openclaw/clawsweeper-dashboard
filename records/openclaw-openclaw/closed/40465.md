## Summary

Current `main` already contains the logging-threshold fix that addresses this report: child loggers inherit the parent `minLevel`, and the shipped changelog for `2026.4.7` explicitly calls out restoring `logging.level`/`logging.consoleLevel` threshold behavior plus child logger inheritance.

## Evidence

- **Child logger now inherits parent minLevel:** `getChildLogger()` uses `base.settings.minLevel` when no explicit level override is passed, which prevents child loggers from defaulting to allow-all.
  - file: [src/logging/logger.ts:253](https://github.com/openclaw/openclaw/blob/9f4ef6162dbde384797abc002cd4241080eb65e3/src/logging/logger.ts#L253)

- **Regression tests cover the exact inheritance bug:** `src/logging/level-filter.test.ts` asserts that child loggers inherit the parent `minLevel`, do not fall back to `0`, and that pino-style child loggers propagate the parent threshold too.
  - file: [src/logging/level-filter.test.ts:100](https://github.com/openclaw/openclaw/blob/9f4ef6162dbde384797abc002cd4241080eb65e3/src/logging/level-filter.test.ts#L100)
