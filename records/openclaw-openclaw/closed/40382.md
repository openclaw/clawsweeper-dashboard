## Summary

Current main no longer matches the reported unhandled-ffprobe crash path: Telegram audio preflight failures are caught and degraded locally, tool exceptions are wrapped into tool-error JSON results, and missing ffprobe is surfaced as a normal install-hint error instead of a raw ENOENT spawn failure.

## Evidence

- **Telegram preflight failure is caught at message-context boundary:** Inbound Telegram audio preflight is wrapped in try/catch and only logs the failure instead of letting it escape.

- **Audio preflight runner already swallows transcription/probe failures:** The transcription helper catches errors, logs them, and returns undefined so processing continues.
  - file: [src/media-understanding/audio-preflight.ts:48](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/src/media-understanding/audio-preflight.ts#L48)
