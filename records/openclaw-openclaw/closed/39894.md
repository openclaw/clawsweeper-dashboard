## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Avoid passing the token to PR-controlled code:** `.github/workflows/ci.yml:304-306`
  - body: The workflow exports GITHUB_TOKEN and then runs scripts/ci-list-pr-files.mjs from the PR checkout. A contributor can modify that script in the same branch, so this gives branch-controlled JavaScript direct token access; keep tokened PR-file listing in trusted workflow code or remove this fallback from the bug-fix PR.
  - confidence: 0.87
- **[P2] Move stripping to the current outbound plan:** `src/infra/outbound/payloads.ts:49`
  - body: The branch changes the older normalizeReplyPayloadsForDelivery loop, but current main now routes delivery and JSON normalization through createOutboundPayloadPlanEntry, where parseReplyDirectives still receives raw payload.text. Rebase the fix so stripping happens before that current parse point.
  - confidence: 0.9
- **[P2] Preserve timestamp-prefixed replies:** `src/infra/outbound/payloads.ts:49`
  - body: stripInboundMetadata removes leading OpenClaw timestamp prefixes even when no metadata block is present. Applying it to every outbound payload can silently change legitimate assistant replies such as logs or transcripts; use the leading-metadata-only helper instead.
  - confidence: 0.82
