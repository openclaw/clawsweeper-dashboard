## Summary

Current main already prevents the reported Feishu reconnect-loop leak by creating a fresh WSClient per monitor start, closing that client on abort/restart cleanup, and covering the cleanup path with regression tests. The same Feishu cleanup files are unchanged between the latest release tag `v2026.4.22` and current main, so this behavior is already shipped.

## Evidence
