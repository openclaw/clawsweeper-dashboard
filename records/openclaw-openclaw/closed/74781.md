## Summary

Keep open: current main still lacks the all-run wait-snapshot behavior this PR adds, and the proposed patch looks narrowly aligned with the open Gateway abort/wait work. The remaining question is maintainer handling of the stacked base PR, not conservative cleanup closure.

## What This Changes

The PR normalizes every run ID returned by session-wide `sessions.abort`, writes the RPC timeout wait snapshot for each aborted run while preserving the first `abortedRunId` response field, and adds a multi-run Gateway regression test.

## Best Possible Solution

Land or fold this with the Gateway cancel/wait semantics work so session-wide abort records terminal wait snapshots for every aborted chat-send or agent-kind run while preserving the existing first-`abortedRunId` response shape and run-kind-specific dedupe namespaces.
