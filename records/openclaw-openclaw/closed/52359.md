## Summary

Current main already deduplicates pre-compaction memory flushes by recording the compaction cycle where a flush ran and refusing another flush in that same cycle, including the transcript-size force-flush path. The docs describe this as “once per compaction cycle,” and regression tests cover the skip behavior. This satisfies the issue’s core cooldown/dedup request without adding the exact proposed cooldownMs field.
