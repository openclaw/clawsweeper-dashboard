## Summary

Current main and the shipped v2026.3.8 macOS release cover this launch regression. The closest proven root cause is the arm64-only macOS release artifact regression tracked by #28877/#28376 and fixed by #33891; #40469 reports the same affected surface, version window, and install artifact family, with the reporter confirming `OpenClaw-2026.3.7.dmg`.

## Best Possible Solution

Close #40469 as implemented by the universal macOS release packaging fix. Keep the current release packaging defaults and direct any remaining current-release launch failures to a fresh, diagnostic issue rather than tracking the obsolete 2026.3.7 DMG report.

## Evidence

- **Current release packaging defaults to universal binaries:** `scripts/package-mac-app.sh` now keeps explicit `BUILD_ARCHS` overrides, but for `BUILD_CONFIG=release` defaults to `all`, which expands to `arm64 x86_64`. This is the core implementation that prevents Apple Silicon build hosts from producing arm64-only release apps.
