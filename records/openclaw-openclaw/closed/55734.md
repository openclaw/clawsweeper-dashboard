## Summary

Keep open. Current main still lets Chrome internal CDP targets enter tab listing and default navigation selection, and the directly related fix PR is closed unmerged.

## What This Changes

OpenClaw should avoid selecting Chrome internal CDP targets such as `chrome://omnibox-popup.top-chrome/`, `chrome://tab-search.top-chrome/`, and `chrome-untrusted://` for browser navigation.

## Best Possible Solution

Add one browser-plugin predicate for selectable web-content CDP targets, apply it consistently before tab aliases/default selection/explicit resolution, fall back to opening `about:blank` when only internal targets remain, and add focused regressions for raw CDP and Playwright-backed listings.

## Reproduction Assessment

Yes. The issue provides a concrete `/json/list` fixture, and current main would map a `chrome://omnibox-popup.top-chrome/` entry as a selectable `page` before navigation; a focused unit test can reproduce this without launching Chrome.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the browser plugin's target-selection/listing code, not in core routing or a new plugin API; the closed related PR is useful source context but should not be copied without correcting its test sequencing issue.
