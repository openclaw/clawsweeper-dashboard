## Summary

Keep this PR open. The PR is a fresh implementation candidate for the same-author open tracker #72484, and current main still appears to use the serial per-collection QMD CLI search path rather than the proposed source-grouped invocation behavior. It needs normal maintainer review, not conservative cleanup closure.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to decide whether source-grouped QMD invocations or the overlapping parallelization approach in #69583 is the desired runtime behavior, preserve mixed memory/session diversification, and resolve the collection-field fallback question with a test or documented QMD JSON contract. After an implementation merges, the paired tracker #72484 can close normally.
