## Summary

Current main and the v2026.4.27 release address the reported WebChat duplicate-plus-metadata leak: history reads the active transcript branch, display projection strips inbound metadata before both history and live transcript broadcasts, and Control UI keeps optimistic/local messages reconciled with canonical history. The closely related #72975 was closed and shipped in v2026.4.27, so this specific report no longer needs a separate open tracker.

## What This Changes

The issue requests that Control UI/WebChat show one visible user turn per send and never expose `Sender (untrusted metadata)` or the `openclaw-control-ui` metadata label as user-visible chat content.

## Best Possible Solution

Keep the v2026.4.27 WebChat active-branch, display-projection, and optimistic-message reconciliation behavior. Leave broader transcript duplication analysis in the umbrella tracker, but close this concrete report unless someone provides a fresh reproduction on v2026.4.27 or newer.

## Reproduction Assessment

Yes. The issue has a high-confidence reproduction shape from the report and related #72975: an append-only transcript with a metadata-wrapped abandoned user entry and a clean replacement. Current regression coverage recreates that shape and verifies only the clean active-branch messages are returned.

## Solution Assessment

Yes. The current fix is the narrow maintainable path: prevent abandoned prompt copies from entering `chat.history`, sanitize all display projections, and keep Control UI optimistic messages reconciled instead of adding channel-specific special cases for `openclaw-control-ui`.
