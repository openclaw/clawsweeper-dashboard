## Summary

Keep open. Current main has adjacent fixes for Bedrock AWS SDK default-chain auth, durable config/.env service env, and preservation of safe existing service env vars, but it still does not implement this PR’s requested AWS/Azure/GCP host-env passthrough for fresh gateway service installs. The PR also has unresolved maintainer-relevant tradeoffs around persisting short-lived AWS credentials, so it should not be cleanup-closed.

## Best Possible Solution

Keep PR #62673 open for maintainer review. The best path is either a narrowed cloud env passthrough with regression tests and an explicit policy for short-lived/static AWS credentials, or a maintainer decision to prefer the existing durable `~/.openclaw/.env`/`config.env` service-env mechanism plus Bedrock default-chain behavior and close the PR manually with that rationale.
