## Summary

Keep open. Current main still has the host-exec SSRF gap described by the linked report, but this PR is not merge-ready because the new parser guard leaves common curl/wget localhost bypasses and fails open on unsupported shell syntax.

## What This Changes

This PR adds a gateway-host exec SSRF guard for curl/wget/httpx/aria2c-style commands, regression tests, and a changelog entry for #76260.

## Best Possible Solution

Land a security-owner-approved gateway exec egress policy that either uses a robust isolation boundary or a clearly documented, fail-closed argv mitigation with regression tests for common network-tool forms and known bypass classes.

## Reproduction Assessment

Yes, at source level. Current main has web_fetch SSRF enforcement but no equivalent gateway-host exec destination check, and the PR branch source shows direct parser bypasses without needing a live localhost harness.

## Solution Assessment

No, not as written. A parser-based mitigation may be acceptable only after maintainer/security approval and after it stops skipping common URL forms or failing open on unsupported network-tool commands.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Do not skip URLs after standalone curl flags:** `src/infra/exec-ssrf-filter.ts:623-625`
  - body: Because `CURL_OPTIONS_WITH_VALUE` is checked before the standalone set and includes flags such as `-L`, `-k`, `-I`, `-4`, and `--compressed`, `curl -L http://127.0.0.1:18760/status/500` advances past the URL as if it were an option value and remains allowed.
  - confidence: 0.94
- **[P1] Validate URL-bearing option assignments:** `src/infra/exec-ssrf-filter.ts:634-637`
  - body: The blanket `--option=value` skip means direct curl URL forms such as `curl --url=http://127.0.0.1:18760/status/500` are never checked. Parse URL-bearing assignments explicitly instead of discarding every long option assignment before hostname validation.
  - confidence: 0.91
- **[P1] Classify wget flags before consuming values:** `src/infra/exec-ssrf-filter.ts:663-665`
  - body: `WGET_OPTIONS_WITH_VALUE` also contains standalone flags that are checked first, including `-q`. As a result, `wget -q http://localhost/file` skips the blocked URL and is allowed.
  - confidence: 0.93
- **[P2] Fail closed when network command parsing is unsupported:** `src/infra/exec-ssrf-filter.ts:762-765`
  - body: When `analyzeShellCommand` rejects syntax such as command substitution, the filter returns allowed even though gateway exec still runs the original shell command. A command like `curl http://$(printf 127.0.0.1):18760/status/500` can hide a blocked destination from the new guard.
  - confidence: 0.88
