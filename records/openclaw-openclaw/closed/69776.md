## Summary

Current `main` already supports multi-file config loading via `$include`, including root-level include arrays with deep-merge and later-file-wins behavior that covers the requested base + local overlay pattern. There is no hardcoded `openclaw.local.json` convention, but the underlying capability is implemented, documented, and tested on `main`.

## Evidence

- **Include resolver supports overlay-style deep merge:** `deepMerge` recursively merges objects, later values win for primitives, and `processInclude` accepts both single-file and array `$include` values, merging sibling keys after includes.

## Risks / Open Questions

- There is no special built-in filename convention like `openclaw.local.json`; users must opt into the pattern with `$include`.
- Root-level include arrays are read-only for OpenClaw-owned write-through operations, so some CLI/plugin writes fail closed instead of rewriting a flattened root config.
- I could not tie this implementation to a released tag from the current checkout; it appears present on `main` after `v2026.4.23`.
