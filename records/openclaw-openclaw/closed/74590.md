## Summary

Current main already implements the requested Slack Socket Mode timeout surface: Slack config now accepts `channels.slack.socketMode`, the runtime passes `clientPingTimeout` and `serverPingTimeout` through Bolt's public `SocketModeReceiver` options with a 15s default pong timeout, and focused tests/docs cover the behavior. The fix is on current main after `v2026.4.26`, so release users need a post-fix build or the next release.

## What This Changes

Expose Slack Socket Mode `clientPingTimeout` and `serverPingTimeout` configuration under `channels.slack.socketMode` so high-latency WebSocket connections can avoid false pong-timeout disconnects.

## Best Possible Solution

Keep the current main implementation and let the next release carry it to stable users; no additional OpenClaw core work is needed for this request unless follow-up reports show the new defaults or config surface are insufficient.
