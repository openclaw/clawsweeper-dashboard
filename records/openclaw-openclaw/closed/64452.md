## Summary

Current `main` already handles Telegram voice-only inbound turns by awaiting audio preflight transcription during message-context construction and then dispatching that context immediately. Regression coverage now exists for both direct-message voice notes and mention-gated group voice notes, and the changelog records the fix under unreleased `2026.4.24`, so this issue is implemented on `main` but not yet tied to a shipped release tag.

## Evidence
