## Summary

This request is useful, but the Selenium/Puppeteer-specific part belongs as a ClawHub skill or plugin rather than new OpenClaw core code. Current main already ships a bundled browser plugin and browser-automation skill backed by Playwright/CDP for the major browser-automation use cases, and the project vision explicitly routes new optional skills/plugins through ClawHub unless a missing core API is identified.

## Best Possible Solution

Keep OpenClaw core on the existing bundled browser plugin and browser-automation skill. Publish Selenium/Puppeteer-specific workflows as a ClawHub skill or external tool plugin using the current plugin API; only open a focused core issue if that implementation hits a concrete missing extension API.
