## Summary

Keep open: current main documents that normal final answers should be durable, but the runner can build deliverable final payloads from streaming assistant text independently of the persisted assistant snapshot, and Telegram then delivers those payloads through its final lane.

## What This Changes

Ensure Telegram-visible final assistant replies after tool-call turns are also persisted as assistant entries in the active session JSONL.

## Best Possible Solution

Add a core runner/session invariant so visible final assistant delivery cannot complete from normal agent runs unless the same final assistant turn is durable exactly once in the active transcript.

## Reproduction Assessment

No, not with high-confidence live proof: I did not run a Telegram/Coolify turn. Source inspection does show a plausible current-main path where assistantTexts can drive final delivery separately from the persisted assistant message snapshot.

## Solution Assessment

Unclear until patched; the best direction is a runner/session persistence invariant rather than a Telegram-only transcript write, because JSONL is the canonical model transcript and channel payloads are only delivery projections.
