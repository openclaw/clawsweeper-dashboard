## Summary

Current main already scopes Gateway bundled runtime-deps pre-staging through the startup plugin plan, and that plan filters out explicit `channels.*.enabled=false` entries before dependency scanning. Main also contains the later retained-deps pruning needed to avoid carrying stale disabled-channel specs forward across repairs, so the specific disabled-channel startup install behavior is no longer actionable in current code.

## What This Changes

The issue asks for Gateway startup bundled runtime-dependency pre-staging to skip bundled channels whose `channels.<id>.enabled` config is explicitly `false`.

## Best Possible Solution

Keep the current manifest/startup-plan boundary: bundled runtime-deps pre-staging should continue consuming the filtered `startupPluginIds`, while retained dependency manifests should keep pruning inactive or disabled-channel specs after upgrades.
