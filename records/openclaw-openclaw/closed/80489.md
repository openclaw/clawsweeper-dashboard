## Summary

Keep open: the PR is pointed at the right Telegram transport boundary, but it still uses only `OPENCLAW_VERSION`, which leaves managed services stamping `OpenClawBot/unknown`, and the supplied terminal proof does not exercise the patched transport path.

## What This Changes

The PR wraps Telegram transport fetches to add an `OpenClawBot/<version>` User-Agent, adds header preservation tests, and includes a lint-only test change.

## Best Possible Solution

Land the Telegram transport UA wrapper after it uses the established runtime/service version fallback and has redacted proof through `resolveTelegramTransport` or an equivalent live Telegram transport probe.

## Reproduction Assessment

No for the ISP throttling end-to-end because it depends on a specific network path. Yes at source level: current main sends Telegram transport requests without a User-Agent, and undici appends its default when that header is absent.

## Solution Assessment

No as currently written. Stamping at the Telegram transport boundary is the narrow maintainable approach, but the version source should match managed service runtime sources and the proof should exercise the patched path.
