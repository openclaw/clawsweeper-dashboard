## Summary

Keep open. The checked-out main still uses the local agent-aware model preparation path for `infer model run --model`, and #74028 is still open/unmerged as the closing PR for honoring fully qualified local provider/model overrides.

## What This Changes

`openclaw infer model run --model <provider/model>` should route local probes to the requested provider/model instead of falling back to the agent default.

## Best Possible Solution

Review and land or replace #74028 with a narrow CLI fix that prepares fully qualified local overrides directly while preserving existing unqualified/default selection behavior, and reconcile it with #73717 if both touch the same seam.

## Reproduction Assessment

No high-confidence live reproduction was established here. The issue has exact commands and current source still shows the local dispatch seam targeted by the open fix PR, so the issue is source-reproducible with medium confidence.

## Solution Assessment

Yes. A CLI-layer fix for fully qualified local overrides is the narrowest maintainable direction; changing the generic runtime resolver would be broader than the reported local `infer model run` scope.
