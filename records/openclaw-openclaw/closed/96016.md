## Summary

Current main fixes the OpenShell mirror self-nesting problem: the upload path no longer sends the generated `openclaw-openshell-upload-*` staging directory as the remote workspace root, and merged maintainer PR https://github.com/openclaw/openclaw/pull/96303 provides the fix provenance. The latest release still has the old upload shape, so this is fixed on current main only until the next release.

## What This Changes

The issue reports that OpenShell mirror mode can recursively nest previous workspace snapshots after OpenShell began preserving uploaded directory basenames.

## Best Possible Solution

Keep the current main OpenShell plugin implementation and let the next release ship the fixed mirror upload behavior.

## Reproduction Assessment

Yes. The old release source uploaded a generated staging directory and upstream OpenShell now preserves named directory basenames; linked PR proof also exercised the behavior against OpenShell v0.0.68, while current main has the corrected upload shape.

## Solution Assessment

Yes. The current plugin-owned fix avoids uploading the generated staging root and preserves workspace cwd compatibility, which is narrower than the earlier `upload .` candidate PRs.
