## Summary

Keep open: this is a new Telegram transport/config feature, and the current PR head is not mergeable because native drafts can still finish without a durable message when the final text matches the last draft preview; CI also has a concrete lint failure. Prior maintainer context and the closed removal/restoration threads make the overall transport direction an owner/product decision rather than cleanup closure.

## What This Changes

The PR adds opt-in Telegram `channels.telegram.streaming.nativeTransport` support that routes private-chat stream previews through `sendMessageDraft`, updates config schema/types, and adds draft-stream tests.

## Best Possible Solution

If maintainers want to revisit native Telegram drafts, keep it as a Telegram-plugin owner decision with a finalization path that always persists the final answer exactly once, scoped config/docs, regression tests for unchanged-final text and ambiguous sends, and live DM proof through OpenClaw.

## Reproduction Assessment

Yes for the blocking review findings: source inspection shows the final same-text path can bypass durable draft materialization, and CI annotations identify the lint failure at the PR head. I did not run a live OpenClaw Telegram gateway repro.

## Solution Assessment

No. The current patch is not the best mergeable solution because it leaves a no-op finalization gap, fails lint, and reopens a transport direction that prior maintainer context says needs explicit owner review and live proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Persist unchanged final drafts:** `extensions/telegram/src/draft-stream.ts:191-192`
  - body: The durable `sendMessage` materialization only happens inside the new final branch here, but `sendOrEditStreamMessage` returns early when the rendered final text equals the last draft preview before this sender is selected. In normal streaming the final text often matches the last preview, so `stop()` can leave `streamMessageId` unset and lane delivery can retain an ephemeral draft instead of sending a permanent Telegram reply. Make final draft stops bypass the duplicate-text no-op when there is no message id.
  - confidence: 0.93
- **[P2] Remove the redundant previewMode cast:** `extensions/telegram/src/draft-stream.ts:456`
  - body: CI is failing oxlint on this new return value because the assertion does not change the inferred type. Removing the `as "draft" | "message"` cast should unblock the lint shard.
  - confidence: 0.98
