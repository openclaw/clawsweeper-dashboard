## Summary

The reported delivery-mirror/webchat duplication path is already fixed on current `main` and was shipped in `v2026.4.22`. Current code both avoids appending a redundant `delivery-mirror` transcript row when the latest assistant text already matches and sanitizes chat-history responses before webchat/HTTP history consumers render them.
