## Summary

Current `main` contains a targeted fix for slow embedded Ollama requests timing out at Undici's default 60s body/header timeout. The issue report and follow-up comment match that failure mode closely enough to close as implemented on `main`, although the fix appears to be newer than the latest tagged release in the review context.

## Evidence

- **Issue symptom matches timeout bug:** The report says `ollama run` succeeds on an Oracle Cloud Ubuntu VPS, but `openclaw agent --local --timeout 300` stalls for ~5 minutes and ends with `stopReason=error` and zero token usage. A follow-up comment reports the same behavior on the same Oracle Cloud VPS shape with a newer OpenClaw version.
- **`--local` uses embedded agent path:** The CLI docs state that `openclaw agent` uses the embedded path when `--local` is set, and that `--timeout` overrides the agent timeout.
  - file: [docs/cli/agent.md:10](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/docs/cli/agent.md#L10)
  - command: `nl -ba docs/cli/agent.md | sed -n '1,60p'`
  - sha: [ecac696643c6](https://github.com/openclaw/openclaw/commit/ecac696643c604c6d854341bfe6424bfa31423e9)
- **Embedded runs now wire run timeout into HTTP runtime:** Current `main` calls `ensureGlobalUndiciStreamTimeouts` from the embedded attempt HTTP runtime, using the run timeout (clamped against the shared default) before model work starts.
  - file: [src/agents/pi-embedded-runner/run/attempt-http-runtime.ts:7](https://github.com/openclaw/openclaw/blob/6fea42fc2d372d2cc76776859060dc59e4a2f261/src/agents/pi-embedded-runner/run/attempt-http-runtime.ts#L7)
  - command: `nl -ba src/agents/pi-embedded-runner/run/attempt-http-runtime.ts | sed -n '1,30p'`
  - sha: [6fea42fc2d37](https://github.com/openclaw/openclaw/commit/6fea42fc2d372d2cc76776859060dc59e4a2f261)
- **Guarded fetch now propagates per-request timeout into Undici dispatchers:** `fetchWithSsrFGuard` resolves a dispatcher timeout and passes it into the created dispatcher; the dispatcher factory sets `bodyTimeout` and `headersTimeout` to that value. That directly addresses the symptom of slow local Ollama calls dying at Undici's default 60s timeout before first token.
  - file: [src/infra/net/fetch-guard.ts:359](https://github.com/openclaw/openclaw/blob/6fea42fc2d372d2cc76776859060dc59e4a2f261/src/infra/net/fetch-guard.ts#L359)

  - sha: [6fea42fc2d37](https://github.com/openclaw/openclaw/commit/6fea42fc2d372d2cc76776859060dc59e4a2f261)
- **Dispatcher implementation sets body/header timeouts:** The HTTP/1-only Undici dispatcher helper now writes `bodyTimeout` and `headersTimeout` when a timeout is provided.
  - file: [src/infra/net/undici-runtime.ts:54](https://github.com/openclaw/openclaw/blob/6fea42fc2d372d2cc76776859060dc59e4a2f261/src/infra/net/undici-runtime.ts#L54)
