## Summary

Current main already implements the host-side fallback requested here, and the fix was merged and shipped before this report; the reported beta commit also contains the same runtime-name dispatch path, so remaining failures are likely an installed/package/runtime mismatch rather than missing OpenClaw core code.

## What This Changes

The issue requests that factory-style plugin tool registrations without explicit names either execute correctly from returned tool names, fall back safely to declared manifest tool names, or produce an actionable diagnostic.

## Best Possible Solution

Keep the shipped runtime-name dispatch in OpenClaw core; any remaining reports should investigate whether the installed gateway package is actually running the fixed source.

## Reproduction Assessment

No current-main failure path is visible: current main and the reported `dc8790d3b4091bef59b6f1327fc6fe5e9cccdb4f` source both include the unnamed-candidate runtime-name fallback. The pre-fix failure shape is high-confidence from the prior report and merged PR proof.

## Solution Assessment

Yes. Resolving by the returned runtime tool name is the narrow host-side fix because it preserves named-factory dispatch while allowing factory-style registrations that rely on manifest ownership.
