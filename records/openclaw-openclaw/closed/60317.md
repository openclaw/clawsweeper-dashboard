## Summary

#60317 reports a hosted clawhub.ai browser/OAuth login redirect problem: after authorizing, the website still shows the user as logged out and page interactions stop responding. The current openclaw/openclaw repo contains OpenClaw’s ClawHub registry client, docs, plugin/skill install flows, and release helpers, but not the clawhub.ai hosted web app login callback/session implementation. The additional comments confirm other users saw the same hosted-site symptom, but they do not identify an OpenClaw CLI, gateway, plugin API, or docs bug that can be fixed in this repository.

## Best Possible Solution

Close this OpenClaw source-repo issue as external to the repo, and route the underlying incident to the maintainers/operators of the hosted ClawHub web service or its service-specific tracker. No OpenClaw code/docs change is apparent unless a separate report identifies a failing OpenClaw CLI/gateway ClawHub API path or a missing public support/documentation link that belongs in this repo.
