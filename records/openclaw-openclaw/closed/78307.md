## Summary

Keep open: current main still does not surface live/dormant state in the Sessions table, and the existing `Active` filter is a recency window rather than a live-only filter.

## What This Changes

Request to add a visible running/live status indicator or live-only filter to the dashboard Sessions view and clarify the `Active N` recency label.

## Best Possible Solution

Add a compact visible status/live affordance using existing `status` and `hasActiveRun`, rename the `Active` recency filter to clearer updated-within wording, and cover the UI, i18n, CSS, tests, and changelog together.

## Reproduction Assessment

Yes. The issue is source-reproducible on current main: the table lacks a visible status/live column while `sessions.list` already carries `status`/`hasActiveRun`, and `activeMinutes` filters by `updatedAt`.

## Solution Assessment

Unclear until maintainers choose the UI shape. The narrow maintainable direction is a compact row-level badge or column plus clearer recency copy, without changing the gateway protocol unless a live-only filter is explicitly desired.
