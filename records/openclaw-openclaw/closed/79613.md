## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/79613. When `session_status(sessionKey: "current", model: "provider/model")` changes a session model, the Control UI chat model select should update without a page refresh and match `/status`. Start from `src/agents/tools/session-status-tool.ts` and the existing `sessions.patch`/`sessions.changed` contract in `src/gateway/server-methods/sessions.ts`, `src/gateway/server-session-events.ts`, and UI consumers in `ui/src/ui/controllers/sessions.ts`, `ui/src/ui/chat-model-select-state.ts`, and `ui/src/ui/chat/session-controls.ts`. Prefer reusing the existing session-change notification or shared session patch/update helper; do not add broad UI polling or duplicate model caches. Add regression coverage for the status-tool model mutation causing an updated session row/broadcast with `model` and `modelProvider`, plus UI/state coverage that the picker follows that row. Add a single-line `CHANGELOG.md` fix entry because this is user-facing. Stop and ask for maintainer review if the implementation requires a new public API, config option, or product-policy change.
