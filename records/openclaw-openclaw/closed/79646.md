## Summary

Keep open: current main still closes the blockquote style after the inner paragraph separator is appended, and Telegram renders that style span directly as `<blockquote>` HTML, so the reported extra blank line remains source-reproducible.

## What This Changes

The issue requests that Markdown blockquote spans exclude the trailing paragraph separator so Telegram HTML closes the `<blockquote>` before following body text.

## Best Possible Solution

Fix the shared Markdown IR blockquote span boundary so terminal paragraph separators fall outside the blockquote style, then add focused IR and Telegram HTML regressions while preserving current spacing behavior.

## Reproduction Assessment

Yes: this is source-reproducible on current main with `> `gpt`\n\nbody`; the parser path appends `\n\n` while the blockquote style is still open, and Telegram renders that span as native `<blockquote>` HTML.

## Solution Assessment

Yes: the maintainable fix belongs in the shared Markdown IR/block style boundary handling rather than a Telegram-only HTML post-process, with tests proving the blockquote span ends before the separator.
