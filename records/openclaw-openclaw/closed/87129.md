## Summary

Close: this is the same `memory-lancedb-pro` Active Memory default request already decided in the closed canonical discussion and companion PR; current shipped docs direct non-core memory plugins to use explicit `config.toolsAllow` and prompt guidance instead of adding a one-off core default.

## What This Changes

The issue asks Active Memory to treat `memory-lancedb-pro` as a LanceDB default slot so its hidden recall sub-agent uses `memory_recall` without explicit `toolsAllow`.

## Best Possible Solution

Keep the shipped explicit `config.toolsAllow` plus `promptAppend` path for non-core memory plugins; revisit generic capability-driven recall-tool discovery only through a separate plugin API/product discussion.

## Reproduction Assessment

Yes, source-level: current main and `v2026.5.22` exact-match only `memory-lancedb`, so `memory-lancedb-pro` falls through unless explicit `toolsAllow` is configured. I did not run a live third-party plugin deployment for this read-only review.

## Solution Assessment

No, not as a core one-off slot allowlist: a prior maintainer decision already chose the explicit custom `toolsAllow` plus prompt path for this third-party plugin case. A capability-driven default would be a separate plugin API design discussion.
