## Summary

Keep open. Current main still lets an aborted Feishu lifecycle signal produce an empty bot identity, write it as unknown, and skip the only background recovery path; the open Feishu PRs checked cover startup serialization, latency, or health reporting rather than this abort-gated recovery gap.

## What This Changes

The issue requests a Feishu WebSocket identity-recovery fix so config reload aborts cannot permanently leave bot open_id unknown without retry.

## Best Possible Solution

Preserve gateway lifecycle abort semantics while ensuring an aborted old Feishu lifecycle cannot overwrite active identity state or suppress current-lifecycle bot identity recovery.

## Reproduction Assessment

Yes, at source level. Current main has a concrete path where fetchBotIdentityForMonitor returns an empty identity for an aborted signal and monitorSingleAccount then skips startBotIdentityRecovery because that same signal is aborted.

## Solution Assessment

Yes. The narrow maintainable direction is a Feishu-local lifecycle/recovery fix that keeps shutdown aborts effective while making unknown bot identity recoverable in the active lifecycle.
