## Summary

Current main and v2026.4.24 route installer completion setup through bounded cache generation and non-interactive profile writes, so accepting zsh completion should no longer leave the installer hanging indefinitely.

## Evidence

- **Issue context and workaround comments:** The report is for OpenClaw 2026.3.8 on Ubuntu 24.04, hanging after answering yes to the zsh completion prompt. Comments already suggested skipping the prompt and enabling completion later with `openclaw completion --shell zsh --install --yes`; one also linked the community ClawAid repair tool.
- **Installer hands setup to onboard with a real TTY:** On current main, after install the script only starts onboarding when `/dev/tty` is readable/writable, then execs `openclaw onboard` with stdin redirected from `/dev/tty`; otherwise it tells the user to run onboarding later.

  - file: [src/wizard/setup.completion.ts:77](https://github.com/openclaw/openclaw/blob/e640c0a95fcbd06a1311f18fc635518d011d45da/src/wizard/setup.completion.ts#L77)
