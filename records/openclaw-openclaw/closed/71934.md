## Summary

Current main already provides the requested Active Memory controls: a dedicated recall model/fallback model, configurable embedded-run timeout up to 120 seconds, runtime timeout enforcement, docs, and regression coverage. The linked timeout-ceiling PR is superseded by current main; the separate memory-core dreaming cleanup issue remains #71956.

## Best Possible Solution

Keep the shipped Active Memory model and timeout controls. Users on high-latency Bedrock Opus should pin `plugins.entries.active-memory.config.model` or `modelFallback` to a faster recall model, or raise `config.timeoutMs` up to the documented 120000 ms ceiling. Treat memory-core Dreaming cleanup separately under #71956.
