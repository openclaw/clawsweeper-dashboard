## Summary

Current main implements the central WebChat image-turn fix via the newer gateway/media-understanding path: text-only chat.send image uploads are offloaded into MediaPath/MediaPaths before dispatch, media understanding prefers agents.defaults.imageModel, and regression tests cover the behavior. The stale PR's exact late get-reply model-promotion approach is no longer the best branch to merge, especially given the review concern about double-processing images after media understanding already ran.

## Best Possible Solution

Close this PR and keep the current main implementation: WebChat text-only image uploads are routed through staged media fields and described with agents.defaults.imageModel, while native vision-capable models continue receiving inline image payloads.
