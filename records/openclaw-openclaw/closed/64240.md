## Summary

Current main already handles the reported ACP failure mode in OpenClaw core: ACP `state:error` turns resolve as `end_turn`, and configured persistent ACP sessions in `error` state are reinitialized instead of staying poisoned. The proposed repo-local patch against bundled `acpx` dist output is therefore no longer needed on `main`.

## Evidence

- **ACP error-state turns now finish cleanly:** `AcpGatewayAgent` maps gateway `state:error` to `stopReason: "end_turn"` unless the error is an explicit refusal, which directly covers the PR's "null final reply" symptom at the OpenClaw boundary.
  - file: [src/acp/translator.ts:967](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/acp/translator.ts#L967)
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **Regression tests cover error-state stop-reason mapping:** Tests assert both `state:error` with an `errorMessage` and `state:error` with no `errorMessage` resolve to `{ stopReason: "end_turn" }`.
  - file: [src/acp/translator.stop-reason.test.ts:16](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/acp/translator.stop-reason.test.ts#L16)
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **Configured persistent ACP sessions no longer stay poisoned:** Configured binding reconciliation rejects sessions whose stored ACP metadata is already in `state:error`, then closes and reinitializes them.
  - file: [src/acp/persistent-bindings.lifecycle.ts:21](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/acp/persistent-bindings.lifecycle.ts#L21)
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **Regression test verifies error-state session reinitialization:** The persistent-bindings test suite explicitly checks that a matching session in `state:error` is reinitialized instead of reused.
  - file: [src/acp/persistent-bindings.test.ts:930](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/acp/persistent-bindings.test.ts#L930)
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **Fresh-session recovery exists in ACP session manager:** When discarding persistent state or recovering from backend-unavailable/init failures, the ACP manager prepares a fresh runtime session and clears cached runtime state instead of leaving a broken handle behind.
  - file: [src/acp/control-plane/manager.core.ts:1266](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/acp/control-plane/manager.core.ts#L1266)
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)

- **Scoped ACPX guidance prefers upstream fixes over OpenClaw-local runtime patches:** The ACPX subtree guide says shared ACP runtime behavior should be fixed in `openclaw/acpx` first and consumed here from a published release, not maintained as a local dist-file patch.
  - file: [extensions/acpx/AGENTS.md:7](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/extensions/acpx/AGENTS.md#L7)
