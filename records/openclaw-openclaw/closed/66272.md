## Summary

Keep open. Current main does not implement this PR's intended audit change, and the proposed patch is stale against the split gateway-audit collector. The remaining fix also needs a narrower security predicate based on effective gateway exposure and same-host reverse-proxy context, not a raw `gateway.bind !== "loopback"` check.

## What This Changes

This PR changes the gateway security audit to suppress `gateway.trusted_proxies_missing` for raw loopback binds, emit it for non-loopback binds, and updates the loopback logging regression test and warning text.

## Best Possible Solution

Revise the PR against `src/security/audit-gateway-config.ts` so `gateway.trusted_proxies_missing` is based on effective gateway exposure plus explicit reverse-proxy context, while preserving documented same-host loopback proxy behavior and adding coverage for loopback, LAN, custom loopback, custom exposed, auto/container behavior, and `allowLoopback` proxy cases.
