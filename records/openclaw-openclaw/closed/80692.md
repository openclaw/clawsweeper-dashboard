## What This Changes

The issue reports that Docker image 2026.5.3-1 shipped metadata-only external channel plugin directories for Discord, WhatsApp, LINE, QQBot, and Feishu, preventing those channels from registering.

## Best Possible Solution

Keep the v2026.5.4+ packaging contract: Docker images should not expose partial external plugin runtime directories, and official external channels should be installed through the plugin install flow with clear diagnostics.

## Reproduction Assessment

Yes for the historical 2026.5.3-1 image, based on the reporter’s deterministic container inventory. I did not reproduce a current-main failure because current source now prunes the package-excluded dist directories during Docker image assembly.

## Solution Assessment

Yes. Pruning misleading metadata-only runtime dirs while keeping externalized official channel plugins install-on-demand is the narrow maintainable fix, and the separate install-hint work addresses the operator guidance part.
