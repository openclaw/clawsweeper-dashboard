## Summary

Current main no longer performs unconditional Anthropic/Qwen external CLI auth-profile sync on auth-store load. The runtime external CLI overlay only registers `minimax-portal`, and the reported `synced anthropic oauth credentials from claude cli` log path is absent. The same implementation is present in release `v2026.4.22`.

## Evidence

- **External CLI runtime overlay only includes MiniMax:** `EXTERNAL_CLI_SYNC_PROVIDERS` contains a single entry for `MINIMAX_CLI_PROFILE_ID` / `minimax-portal`; there is no Anthropic or Qwen provider in the auth-profile external CLI sync path.
