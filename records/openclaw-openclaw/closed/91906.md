## Summary

Keep this PR open, but it is not merge-ready: source inspection shows subagent-lane runs can still have direct or inline live delivery consumers, so lane-wide suppression can remove live partial/block/reasoning updates, and the contributor proof remains tests/lint only.

## What This Changes

This PR threads `suppressLiveStreamOutput` through embedded agent runs and enables it for subagent-lane attempts to skip per-chunk live stream parsing while preserving final `message_end` delivery.

## Best Possible Solution

Gate the optimization on an actual no-live-consumer condition, preserve live updates for deliverable subagent paths, and add redacted real-run or benchmark proof before merge.

## Reproduction Assessment

Yes. Source inspection shows native and ACP session spawns can use `AGENT_LANE_SUBAGENT` while direct or inline delivery gives embedded subscriptions live callbacks; I did not run a live subagent session in this read-only review.

## Solution Assessment

No. The optimization is plausible for detached subagents, but the current lane-only predicate is too broad; the safer fix is to key suppression off the absence of live delivery consumers.
