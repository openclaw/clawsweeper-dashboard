## Summary

Current `main` already has a dedicated Voice Call realtime voice path backed by the bundled OpenAI realtime voice provider, with configurable instructions and tool definitions, and the same implementation is present in release `v2026.4.22`. This issue's requested capability is already in the product.

## Close Comment

Closing from a Codex review because the requested capability is already implemented in OpenClaw. Current `main` has a dedicated Voice Call realtime voice path (`extensions/voice-call/src/config.ts:215`, `extensions/voice-call/src/webhook/realtime-handler.ts:249`) and the bundled OpenAI plugin registers a realtime voice provider that opens a Realtime API audio session with instructions and optional tools (`extensions/openai/index.ts:53`, `extensions/openai/realtime-voice-provider.ts:398`). The docs also describe this Voice Call realtime voice support and bidirectional tool calling in `docs/providers/openai.md:415`, and the same implementation is already present in shipped release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). If there is still a separate gap around automatic agent-memory/prompt composition in this mode, that would need a narrower follow-up issue.
