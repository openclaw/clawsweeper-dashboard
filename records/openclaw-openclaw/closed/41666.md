## Summary

Current `main` already covers the reported Telegram cron-burst failure mode: cron delivery routes through the outbound chunking pipeline, Telegram outbound enforces a 4000-char chunk limit, the Telegram send helper chunks long HTML-mode sends, and the polling session has a watchdog that forces a restart when a non-`getUpdates` API call such as `sendMessage` stays stuck past the stall threshold. The same protections are present in release `v2026.4.22`.

## Evidence
