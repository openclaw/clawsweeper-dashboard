## Summary

Current main and the shipped changelog show this issue’s OpenClaw-facing proxy failures were addressed: provider HTTP helper requests now auto-upgrade to trusted env-proxy mode when appropriate, Slack Socket Mode/WebClient honor proxy env vars, and Slack dispatcher-backed media fetches avoid the incompatible global fetch path. The fixes were shipped by v2026.4.8.

## Risks / Open Questions

- This does not mean OpenClaw now globally calls `setGlobalDispatcher` for every `globalThis.fetch`; current main intentionally preserves strict DNS-pinned fetch behavior unless a trusted env-proxy path is selected.
- Other proxy-adjacent issues mentioned in the timeline remain separate scope and should stay tracked on their own issue numbers.
