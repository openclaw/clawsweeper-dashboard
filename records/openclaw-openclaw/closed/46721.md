## Summary

Keep open. Current main still builds session-memory filenames, fallback slugs, and Markdown headers from UTC `toISOString()` values, so the reported local-timezone filename bug remains present. The PR is a focused implementation with regression-test intent, and the discussion shows the initial review concerns were addressed in follow-up commits rather than superseded elsewhere.

## Best Possible Solution

Keep the PR open or land an equivalent focused fix in `src/hooks/bundled/session-memory/handler.ts`: resolve the configured/host user timezone, format the filename date and fallback slug in that timezone with `hourCycle: "h23"`, keep the Markdown header internally consistent, and add the regression coverage proposed in the PR. Use #46703 as the canonical bug until the implementation ships.
