## Summary

Keep open: current main still has the source-reproducible Slack outbound mirror path, and two open linked PRs now own the candidate fix work rather than making the issue closeable.

## What This Changes

Fix Slack outbound delivery-mirror routing so replies in an existing Slack thread append to that thread session instead of minting per-reply mirror-only sessions.

## Best Possible Solution

Land one narrow Slack-specific route fix that treats inherited same-thread replies as current-session-first while preserving explicit reply targets and proving the behavior with focused tests plus redacted real Slack evidence.

## Reproduction Assessment

Yes. Source inspection shows the current Slack outbound mirror route can choose a child replyToId before the recovered current thread session, and the issue includes live deployment evidence; I did not run a live Slack replay in this read-only review.

## Solution Assessment

Yes. The best fix is Slack-specific route precedence based on explicit versus inherited reply metadata; changing the generic helper default would be broader and risk other channels.
