Full review comments:

- **[P1] Handle existing Discord voice sessions before reporting start:** `extensions/discord/src/voice/meeting-notes-source.ts:43-55`
  - body: `start` treats any successful `manager.join` as a captured meeting-notes session, but `DiscordVoiceManager.join` still returns `ok: true` for an existing same-channel session without applying the new `meetingNotes` callback. In that common case the tool reports success while no transcript utterances are wired to the store.
  - confidence: 0.93
- **[P1] Finalize notes when provider stop is already disconnected:** `extensions/meeting-notes/src/tool.ts:175-184`
  - body: The stop path throws on any provider stop error before writing `stoppedAt`, clearing active state, or generating the summary. If Discord has already disconnected and `leave` returns a non-ok result, users cannot finalize a session that may already contain transcript data.
  - confidence: 0.9
- **[P2] Reject ambiguous Discord manager selection:** `extensions/discord/src/voice/meeting-notes-source.ts:20-25`
  - body: When `accountId` is omitted, start/stop/status select the first manager inserted in `managersByAccountId`. Multi-account gateways can therefore join, leave, or report status for the wrong bot based on insertion order instead of caller intent.
  - confidence: 0.88
- **[P2] Merge meeting-notes manifest providers when active:** `src/plugins/capability-provider-runtime.ts:69`
  - body: Adding `meetingNotesSourceProviders` to the capability keys is not enough because `resolvePluginCapabilityProviders` returns active providers early unless the key is in the manifest-merge allowlist. Once `manual-transcript` is active, provider listing can hide manifest-discovered sources such as `discord-voice`.
  - confidence: 0.86
- **[P2] Stop late-resolving auto-start sessions during shutdown:** `extensions/meeting-notes/src/tool.ts:358-363`
  - body: If service shutdown happens while `startMeetingNotes` is still in flight, `stop()` runs before a session id is recorded, then this callback records the id after shutdown. That leaves a capture session outside the service teardown path.
  - confidence: 0.84
