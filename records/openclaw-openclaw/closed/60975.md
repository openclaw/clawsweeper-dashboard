## Summary

The request is for optional cron/memory observability output that fits the repo’s plugin-first scope. Current main already exposes structured cron/task data and plugin hooks/runtime seams that are sufficient for a ClawHub plugin to emit a deterministic JSONL receipt ledger without adding new core behavior.

## Risks / Open Questions

- There is not yet an exact off-the-shelf ClawHub plugin for this precise `cron.machineReceipts.*` schema, so closing this assumes the requested value is in plugin scope rather than demanding a core-maintained format.
- If plugin authors later find they cannot derive a stable enough terminal receipt from existing hooks/task/run-log surfaces, the right follow-up would be a narrow SDK seam expansion rather than a new core cron feature.
