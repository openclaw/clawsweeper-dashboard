## Summary

Current main already fixes the reported WebChat/Pi reasoning leak. The gateway transcript builder now skips `ReplyPayload` entries marked `isReasoning: true`, WebChat media/audio helpers also ignore reasoning payloads, and regression tests/docs cover the behavior. The latest release tag `v2026.4.24` lacks the guard, so I could not identify a shipped fixed release yet.

## Best Possible Solution

Close #71910 as fixed on current main and keep the existing main implementation. The linked PR #72014 usefully identified the same root cause, but the issue itself no longer needs to remain open; maintainers can handle that PR separately as obsolete or as a refactor if still desired.
