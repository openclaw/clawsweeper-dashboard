## Review Findings

Overall confidence: 0.86

Full review comments:

- **[P1] Keep retry config on the model, not a global registry:** `src/agents/provider-retry.ts:15`
  - body: `providerRetryRunners` is process-global and keyed only by provider id, but model config is prepared per agent/config. If two agents use `vllm` with different retry settings, whichever normalization ran last clears or replaces the runner for every later OpenAI-compatible request, so one agent can unexpectedly inherit or lose another agent's retry policy. Carry this with the model/request metadata instead of module state.
  - confidence: 0.88
- **[P1] Do not expose retry for providers that ignore it:** `src/config/zod-schema.core.ts:342`
  - body: Adding `retry` to the generic provider schema makes the knob valid for Anthropic, Google, Bedrock, and other provider APIs, but this PR only reads the runner in the OpenAI-compatible transport wrappers. A user can configure retry for the linked Anthropic-style outage case and get no retry at all. Either wire the shared/provider-owned transport paths too, or restrict the public config/docs to the APIs that actually honor it.
  - confidence: 0.84
