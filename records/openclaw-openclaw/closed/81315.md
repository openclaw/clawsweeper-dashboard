## Summary

Keep open, but do not merge as-is: this PR adds an opt-in Telegram native-draft transport that conflicts with the current Telegram streaming decision, lacks patched OpenClaw runtime proof, and misses the strict config schema needed for the documented option.

## What This Changes

Adds an optional Telegram `streaming.nativeTransport` path that streams previews through `sendMessageDraft`, with tests, docs, and a shared config type change.

## Best Possible Solution

Keep the current Telegram send/edit preview behavior unless a Telegram owner approves an experimental native-draft mode with aligned config schema, docs, tests, and real OpenClaw runtime proof.

## Reproduction Assessment

Not applicable: this is a feature PR rather than a bug report. The changed behavior is source-traceable from the diff because `streaming.nativeTransport: true` would route Telegram preview lanes into `sendMessageDraft`.

## Solution Assessment

No. The proposed direction needs Telegram owner approval first, and even if approved the diff must update the strict config schema and show patched OpenClaw runtime behavior before it is the maintainable solution.
