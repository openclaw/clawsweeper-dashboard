## Summary

The report targets Lossless Claw, which current OpenClaw docs and code treat as an external community context-engine plugin rather than a bundled/core extension. Current `main` already sanitizes replay history to remove empty assistant text before provider replay, so the remaining DB write/replay behavior described in the issue is plugin-owned and should be handled in the Lossless Claw repo/ClawHub path, not as an OpenClaw core issue.
