## Summary

Current main already fixes the reported Windows WhatsApp plugin install failure through the shared managed npm-root command fix, and that fix is included in the v2026.5.9-beta.1 release.

## What This Changes

The issue reports that `openclaw channels login --channel whatsapp` fails on Windows 11 npm installs with `npm install failed: Cannot read properties of undefined (reading 'spec')`.

## Best Possible Solution

Keep the shared managed npm-root command fix and have affected v2026.5.7 users update to v2026.5.9-beta.1 or the next stable release containing it.

## Reproduction Assessment

Yes for the pre-fix command shape: the merged fix PR recorded Windows proof that `--prefix .` reproduced the same npm `spec` crash. No current-main failing reproduction remains because the managed npm install command no longer uses that command shape.

## Solution Assessment

Yes. Removing the redundant `--prefix .` from the shared managed npm-root commands is the narrow maintainable fix, rather than adding WhatsApp-specific install handling.
