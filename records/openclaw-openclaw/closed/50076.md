- **[P2] Move validation into the shared mutation path:** `src/cli/config-cli.ts:1035-1039`
  - body: This branch wires the new checks into the older `runConfigSet`/`runConfigUnset` shape, but current main now routes set, patch, batch, and dry-run unset through `runConfigOperations`.   - confidence: 0.86
- **[P2] Reuse the runtime and audit command catalog:** `src/config/validate-deny-commands.ts:33-35`
  - body: `collectAllKnownNodeCommands` rebuilds a separate catalog by calling `resolveNodeCommandAllowlist` over synthetic platform values and manually unioning dangerous defaults. Current audit/runtime logic already has a fuller vocabulary for device family, approved desktop commands, talk commands, and plugin policy, so this duplicate helper can false-mark valid commands as unknown.
  - confidence: 0.84
- **[P2] Allow exact pattern-looking command IDs:** `src/config/validate-deny-commands.ts:134-138`
  - body: The validator rejects pattern-looking strings before checking whether the exact value is known through `allowCommands`. The gateway protocol accepts arbitrary non-empty command strings and runtime policy exact-matches them, so allowlisted IDs like `foo*bar` or `group:sync` would become impossible to deny through `config set`.
  - confidence: 0.88
