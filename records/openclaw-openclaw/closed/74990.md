## Summary

Keep this PR open: the feature is not already on current main, but the branch still has a source-reproducible correctness blocker around custom Claude CLI command paths and lacks contributor-supplied real behavior proof.

## What This Changes

The PR relabels Anthropic CLI onboarding as a Claude subscription path, adds Claude CLI detect/install/sign-in handling, and updates Anthropic docs, changelog, and tests.

## Best Possible Solution

Keep the Anthropic-owned subscription onboarding direction, but resolve the effective `claude-cli` command from config, offer npm install only for the default PATH command case, then require real proof and maintainer UX review.

## Reproduction Assessment

Yes. Configure `agents.defaults.cliBackends.claude-cli.command` to an absolute Claude Code binary on a host where plain `claude` is not on PATH; source inspection of the PR head shows detection and login still spawn literal `claude`.

## Solution Assessment

No. The feature direction is plausible, but the current implementation bypasses the existing CLI backend command override contract instead of reusing it in Anthropic onboarding.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Respect the configured Claude CLI command:** `extensions/anthropic/cli-install.ts:40`
