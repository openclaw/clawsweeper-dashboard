## Summary

Current `main` and the shipped `v2026.4.23` release already include the speech-provider registry path that backfills bundled TTS providers from plugin manifests when the active runtime registry is empty, which addresses the gateway-side ElevenLabs registration gap described in #59979.

## Evidence

- **Gateway TTS methods enumerate providers through the speech registry with config-aware lookup:** `tts.status`, `tts.setProvider`, and `tts.providers` all call `listSpeechProviders(cfg)` / `getSpeechProvider(..., cfg)`, so the fix hinges on the speech registry returning bundled providers for the current config.
  - file: [src/gateway/server-methods/tts.ts:37](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/gateway/server-methods/tts.ts#L37)
  - command: `nl -ba src/gateway/server-methods/tts.ts | sed -n '20,190p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Speech registry now resolves providers via capability-provider runtime compatibility loading:** `src/tts/provider-registry.ts` delegates to `resolvePluginCapabilityProviders({ key: "speechProviders", cfg })`, which is the config-aware path used by gateway TTS inventory and provider validation.
  - file: [src/tts/provider-registry.ts:16](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/tts/provider-registry.ts#L16)
  - command: `nl -ba src/tts/provider-registry.ts | sed -n '1,120p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Capability runtime explicitly backfills bundled provider plugins from manifests when no active registry entries exist:** `resolvePluginCapabilityProviders()` loads bundled plugin ids from manifest contracts, applies bundled-plugin compat enablement, and then resolves the runtime registry with that config. That is the missing bridge between config-selected ElevenLabs and gateway-visible speech providers.
  - file: [src/plugins/capability-provider-runtime.ts:45](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/plugins/capability-provider-runtime.ts#L45)
  - command: `nl -ba src/plugins/capability-provider-runtime.ts | sed -n '1,170p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Regression test covers the exact failure mode:** `src/tts/provider-registry.test.ts` asserts that when config is provided and no active speech providers exist, OpenClaw loads bundled speech providers from plugin manifests and returns them through `listSpeechProviders(cfg)` / `getSpeechProvider(..., cfg)`.
  - file: [src/tts/provider-registry.test.ts:96](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/tts/provider-registry.test.ts#L96)
  - command: `nl -ba src/tts/provider-registry.test.ts | sed -n '1,180p'`

  - file: [extensions/elevenlabs/speech-provider.ts:75](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/extensions/elevenlabs/speech-provider.ts#L75)
