## Summary

Current `main` already implements automatic rollback to a last-known-good config on failed startup, with backup rotation, preserved clobbered files, operator-facing warnings, tests, and released docs.

## Evidence

## Risks / Open Questions

- The implementation does not match every proposed surface exactly: the issue suggested dedicated `openclaw config backup|restore|list-backups` commands and a configurable backup count, while shipped behavior solves the startup rollback problem through automatic backup rotation and recovery instead.

## Close Comment

Closing as implemented. A Codex review of current `main` confirmed that OpenClaw now restores invalid startup configs from the last-known-good backup in `src/gateway/server-startup-config.ts:60`, preserves the rejected payload as `.clobbered.*` in `src/config/io.observe-recovery.ts:620`, and maintains rotating config backups on write in `src/config/io.ts:1836` plus `src/config/backup-rotation.ts:16`. The behavior is covered by `src/gateway/server-startup-config.recovery.test.ts:57`, documented in `docs/gateway/configuration.md:87` and `docs/gateway/troubleshooting.md:265`, and it is already shipped in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). The exact CLI/config-surface proposed here was not adopted verbatim, but the requested automatic rollback on failed startup is present.
