## Summary

Keep open: current main has partial default-preservation plumbing and an open narrow setup fix PR, but the broader paid/preview default-model consent, warning, and changelog policy remains unresolved.

## What This Changes

The issue asks OpenClaw to stop silent upgrade or setup default-model promotions to paid preview variants without explicit user consent and pricing warnings.

## Best Possible Solution

Define default-model changes as explicit user intent: preserve existing primaries across upgrade/setup unless confirmed, add non-interactive consent semantics for costly or preview promotions, and document pricing/quota impact where defaults change.

## Reproduction Assessment

Partial yes. Current source shows setup provider auth can still promote a Google preview default without the preservation flag, but the exact historical 2026.3.2 to 2026.3.22 migration and billing artifact requires the reporter's private backups and logs.

## Solution Assessment

Unclear for automation. The narrow setup-preservation direction in https://github.com/openclaw/openclaw/pull/81304 looks plausible for the related heartbeat issue, but this issue's broader policy around price tiers, preview defaults, non-interactive consent, and release-note warnings needs maintainer judgment.
