## What This Changes

The issue requests Slack thread-broadcast support so agents can detect inbound “Also send to channel” replies and mirror that visibility on outbound thread replies.

## Best Possible Solution

Keep the shipped explicit outbound opt-in, then add a Slack-owned inbound broadcast-visibility field and an intentional mirroring policy after maintainer review.

## Reproduction Assessment

Yes, from source: current main accepts Slack `thread_broadcast` messages but does not expose a broadcast visibility field in inbound agent context. I did not run a live Slack workspace repro in this read-only review.

## Solution Assessment

Unclear until maintainers choose the policy. The best path is not another outbound-only parameter; it is a Slack-owned inbound metadata shape plus an explicit decision about whether and when agents should mirror channel broadcasts.
