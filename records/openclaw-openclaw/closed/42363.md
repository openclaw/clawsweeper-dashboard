## Summary

Current main and the latest release no longer route the reported valid numeric/boolean config values through containsEnvVarReference. Config env application is scoped to env, full-config substitution only processes strings, and number/boolean/null primitives pass through unchanged. The repro fields remain valid numeric schema fields.
