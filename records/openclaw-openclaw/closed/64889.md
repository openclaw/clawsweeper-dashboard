## Summary

Issue #64889 should close as a duplicate of #60965. Current main still shows the reported `sessions_spawn(runtime="subagent", streamTo="parent")` rejection and the `/subagents spawn` direct native path, but the remaining actionable defect is already tracked by the earlier open issue #60965: `sessions_spawn` exposes ACP-only `streamTo` while rejecting it for native subagent runtime.

## Best Possible Solution

Close #64889 as duplicate/superseded by the earlier open #60965. Keep #60965 as the single tracker for making `sessions_spawn` runtime-aware, either by using a discriminated schema or by normalizing/stripping ACP-only fields before native subagent dispatch. No separate public command-execution workaround is needed unless maintainers choose to open that as a distinct product/API proposal.
