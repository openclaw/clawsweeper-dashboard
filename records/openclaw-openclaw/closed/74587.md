Current main still emits the reported Active Memory warning text, while the implementation and docs show `modelFallback` is a last-resort model-resolution candidate rather than runtime failover. The issue asks for Active Memory's `modelFallbackPolicy` deprecation warning to clarify that `modelFallback` is a last-resort model-resolution fallback, not runtime failover.
