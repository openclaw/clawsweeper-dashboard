## Summary

Current `main` already ships automatic session lifecycle maintenance for stale/oversized session state, transcript archival/retention, disk-budget cleanup, and a manual `openclaw sessions cleanup` command. The implementation was already present in release `v2026.4.23` and the core code remains unchanged on `main` since that release.

## Evidence

- **Built-in maintenance config:** `session.maintenance` already supports stale-age pruning, entry caps, oversized `sessions.json` rotation, archive retention, and per-store disk budgets.
  - file: [src/config/sessions/store-maintenance.ts:12](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/config/sessions/store-maintenance.ts#L12)
- **Automatic archival and cleanup on writes:** Store updates prune stale sessions, cap overflow, archive removed transcripts as deleted artifacts, clean old archives by retention, rotate oversized stores, and enforce disk budgets.
  - file: [src/config/sessions/store.ts:287](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/config/sessions/store.ts#L287)
- **Disk-budget eviction order:** Budget enforcement removes old archived/orphan transcript artifacts first, then evicts oldest session entries/transcripts until usage reaches the configured high-water mark, while protecting an active session key.
  - file: [src/config/sessions/disk-budget.ts:192](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/config/sessions/disk-budget.ts#L192)
- **Manual maintenance CLI:** `openclaw sessions cleanup` already provides dry-run, enforce, JSON output, missing-transcript cleanup, per-agent/all-agent scope, and active-key protection.
  - file: [src/commands/sessions-cleanup.ts:37](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/commands/sessions-cleanup.ts#L37)
- **Docs and shipped release proof:** Docs describe automatic session maintenance and orphan transcript archival, and release tag `v2026.4.23` already contains the implementation/docs files (`src/commands/sessions-cleanup.ts`, `src/config/sessions/store.ts`, `src/config/sessions/store-maintenance.ts`, `docs/cli/sessions.md`, `docs/concepts/session.md`, `docs/cli/doctor.md`).
  - file: [docs/cli/sessions.md:56](https://github.com/openclaw/openclaw/blob/9b18ae877018a34933e27bb7b2986cb97e9602aa/docs/cli/sessions.md#L56)

## Risks / Open Questions

- The shipped UX uses `openclaw sessions cleanup` / `session.maintenance`, not the issue's proposed `openclaw sessions gc` naming.
- The implementation archives in place as timestamped session artifacts rather than the exact external archive-path + hook design proposed in the issue.
