## Summary

Issue #50891 reports a real current-main gap, but the remaining work is now directly tracked by open PR #61675, which explicitly closes #50891 and implements daily/idle lazy-reset hook dispatch for session-memory and before_reset. Current main still only wires session-memory to manual command reset/new events, so this should not close as implemented; it should close as superseded by the canonical PR thread.

## Best Possible Solution

Close #50891 as superseded by open PR #61675 and keep review/implementation discussion on that PR. Do not mark the behavior fixed or released until a PR lands on main; handle #57369 separately as the manual-only reset mode feature request, and consider deduping older PR #50933 against the chosen canonical fix.
