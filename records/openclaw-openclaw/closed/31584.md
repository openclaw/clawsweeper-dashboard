## Summary

Issue #31584 is a valid feature request, and current main still does not implement non-interactive GitHub Copilot onboarding. However, the same remaining work is now tracked by open PR #50003, which explicitly adds `openclaw onboard --non-interactive --github-copilot-token`, env-var fallback, auth-choice inference, and focused tests. This should close as superseded by that implementation track, not as shipped.

## Best Possible Solution

Close #31584 as superseded by PR #50003 and keep the implementation discussion there. The eventual fix should land through a current-main rebase or equivalent PR that adds Copilot manifest flag metadata, a provider-owned `runNonInteractive` auth path, token/profile persistence through the auth-profile secret model, current default-model behavior, and focused docs/tests. Adjacent PAT/paste-token work in PR #56650 should be reconciled if maintainers want one headless Copilot auth design.
