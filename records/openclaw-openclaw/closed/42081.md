## Summary

Issue #42081 reports URL-callback behavior for the external WeCom channel plugin, not a bug in OpenClaw core. Current `main` has no bundled WeCom implementation; it only documents and catalogs `@wecom/wecom-openclaw-plugin` as a community-maintained npm plugin, and `VISION.md` says optional capability should usually ship as plugins maintained in their own repos.
