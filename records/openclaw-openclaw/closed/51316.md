## Summary

Close this draft PR as superseded by #50694. This PR’s latest draft state is unresolved, its final diff asserts the oversized-reserve case should return false, and it includes unrelated lockfile churn.

## Best Possible Solution

Close this draft as superseded and route review/landing to #50694 or an equivalent focused fix. The canonical fix should centralize a context-window-aware effective reserve/threshold used by both preflight compaction and token-triggered memory flush, add regression coverage for reserveTokensFloor >= contextWindowTokens, and leave unrelated dependency-lockfile churn out of the patch.
