## Summary

Current `main` already implements the reported cron model-override precedence: isolated cron resolves `payload.model` ahead of stored session/default models, forwards that resolved provider/model into execution, and persists it before the run so logs/session state do not fall back to the agent default. The relevant implementation and regression coverage are already present in release `v2026.4.23`.

## Evidence
