## Summary

Keep open and do not merge as-is. Current main does not expose `agents.<id>.args` in the OpenClaw acpx plugin config, but this PR flattens argv with a raw space join, so arguments containing spaces, quotes, or empty strings will not survive the downstream command parser; it also misses the required docs and changelog coverage.

## What This Changes

The PR adds an optional `args` array to acpx agent command overrides and resolves each override by concatenating `command` with those args.

## Best Possible Solution

Land the config shape only after serializing each `args` element with quote-safe command-string semantics, adding regression coverage for whitespace, quotes, and empty args, and documenting the new acpx config field with a changelog entry.

## Reproduction Assessment

Yes for the PR-introduced defect: source inspection shows `cmdArgs.join(" ")` feeds a command-string parser that splits unquoted whitespace before spawn. A config with `args: ["/path/with space/adapter.mjs"]` would be parsed as multiple argv elements.

## Solution Assessment

No. Adding the field in the acpx plugin is the right boundary, but raw string concatenation is not the best implementation; quote-safe serialization or a structured runtime contract is needed before merge.
