## What This Changes

The PR adds a Telegram config-only accessor for `allowFrom` and `defaultTo` so status metadata reads do not resolve SecretRef-backed bot tokens.

## Best Possible Solution

Keep the released main implementation and close this branch as already shipped.

## Reproduction Assessment

Yes by source inspection. The SDK fallback would call Telegram runtime `resolveAccount` for metadata accessors when no accessor projection exists, and Telegram runtime token resolution is strict for non-env SecretRefs.

## Solution Assessment

Yes. The shipped fix keeps runtime token resolution strict while giving read-only allow/default accessors a Telegram config-only projection, matching the Slack and Discord accessor pattern.
