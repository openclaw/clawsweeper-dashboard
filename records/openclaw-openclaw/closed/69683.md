## Summary

Current `main` already handles native `api: "ollama"` through OpenClaw's provider-owned stream registration path for normal runs, compaction, and simple-completion/summarization. The docs explicitly direct users to `api: "ollama"` for the native Ollama transport, and the changelog records this fix as shipped in `2026.3.7`.
