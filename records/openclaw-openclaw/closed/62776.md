## Summary

Close #62776 as superseded by the already-related open PR #62134, which tracks the same remote MCP OAuth/authProvider/callback/token-persistence work. Current main still lacks native OAuth support for remote MCP servers like Heptabase, so this should not be closed as implemented; the remaining product and security discussion belongs on the canonical PR.

## Best Possible Solution

Close this issue as duplicate/superseded and keep the implementation discussion on #62134. The best path is a maintainer-reviewed core MCP runtime change that adds an opt-in remote-server OAuth config/schema/docs surface, preserves auth mode through resolution, passes an OAuthClientProvider to SSE and streamable-http transports, implements hardened callback/finishAuth/token persistence/retry behavior, and covers static-header, stale-token, SSE, and streamable-http cases in tests.
