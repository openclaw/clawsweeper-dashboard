## Summary

Current main already solves the Telegram Web readability problem by making Bot API rich messages explicit opt-in through `channels.telegram.richMessages: true` and using standard Telegram `sendMessage`/HTML delivery by default. The fix came through https://github.com/openclaw/openclaw/pull/93279 with a follow-up explicit default in https://github.com/openclaw/openclaw/pull/94048; it is present on current main but not in the latest `v2026.6.8` release.

## What This Changes

Request to make Telegram rich-message delivery disableable or fallback-compatible so Telegram Web users receive readable standard text instead of unsupported rich messages.

## Best Possible Solution

Keep the current main behavior: standard Telegram text delivery by default, rich messages only by explicit `richMessages` opt-in, and ship that fix in the next release.

## Reproduction Assessment

Yes for the shipped regression at source/report level: `v2026.6.8` sent default Telegram text through `sendRichMessage`, while multiple related reports describe Telegram Web unsupported placeholders. There is no failing current-main reproduction path because current main restores standard `sendMessage` delivery by default.

## Solution Assessment

Yes. The merged fix is the maintainable owner-boundary solution: keep the compatible text path as default and make native rich Telegram delivery explicit opt-in rather than adding a separate disable flag.
