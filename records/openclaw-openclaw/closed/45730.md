## Summary

Current `main` no longer matches the report's restart path. SIGUSR1 is handled through the gateway run loop, which drains work and calls graceful server shutdown; shutdown stops channel tasks, and the WhatsApp monitor/controller then closes the active listener/socket before restart.

## Evidence

- **SIGUSR1 restart uses graceful server close:** `runGatewayLoop()` handles restart by draining active work and awaiting `server.close({ reason: "gateway restarting" ... })` rather than exiting immediately on SIGUSR1.
  - file: [src/cli/gateway-cli/run-loop.ts:204](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/cli/gateway-cli/run-loop.ts#L204)

  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Gateway shutdown stops all channel plugins:** Server close iterates all channel plugins and awaits `stopChannel(plugin.id)` during shutdown.
  - file: [src/gateway/server-close.ts:137](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/gateway/server-close.ts#L137)
