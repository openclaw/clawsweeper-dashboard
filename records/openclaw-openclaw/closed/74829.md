## Summary

Keep this PR open. The branch addresses the conformance diagnostic assertion direction, but the latest diff still moves the full kitchen-sink runtime config before plugin install, which writes a plugin-owned channel id before the plugin exists and can make `plugins install` fail config validation.

## What This Changes

The PR updates the kitchen-sink plugin E2E harness to keep conformance out of invalid-probe diagnostics, creates the config directory before writing runtime config, moves runtime configuration before plugin install, and adjusts the prerelease-plan test guard.

## Best Possible Solution

Keep conformance on the no-error diagnostics path, but split the harness setup so only the personality choice is available before install. Write `channels.kitchen-sink-channel` after install and before enable/list/inspect, with a focused regression guard for that ordering.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path is the PR branch's success scenario: it runs `configure_kitchen_sink_runtime` before `plugins install`, and that helper writes an unknown plugin-owned channel id before install-time config validation runs.

## Solution Assessment

No. The assertion change is directionally correct, but moving the whole runtime config before install is not the narrowest maintainable fix; pre-install setup should be limited to personality selection or diagnostics should be filtered by phase.
