## What This Changes

The issue requests preventing large embedded-agent session JSONL files from crashing the Gateway by avoiding duplicate whole-file loads in the Pi `SessionManager.open()` context rebuild path.

## Best Possible Solution

Keep #63691 as the canonical focused repair for the embedded run pre-open large-transcript guard, keep #63998 for broader startup/restart-safe transcript quarantine, and carry this report’s duplicate-load evidence into those threads rather than tracking a third open issue.
