## Best Possible Solution

Keep the shipped direct configured-provider resolution and close the obsolete report; the still-open contributor PR can be handled separately as superseded by the shipped fix.

Yes. The current implementation is the narrow maintainable fix because configured realtime voice provider resolution asks the manifest-backed capability resolver for the requested provider before alias/list fallback, preserving lazy plugin loading.
