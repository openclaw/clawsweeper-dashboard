## Summary

Current `main` and shipped release `v2026.4.23` already suppress late exec `onUpdate` callbacks after backgrounding, exit, and abort, which matches the reported `Agent listener invoked outside active run` race.

## Evidence

- **Emit guard on current main:** `emitUpdate()` now returns early when the session is backgrounded, exited, or `updatesDisabled` is set, and the inline comment names the same `Agent listener invoked outside active run` crash.
  - file: [src/agents/bash-tools.exec-runtime.ts:600](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/src/agents/bash-tools.exec-runtime.ts#L600)
  - command: `nl -ba src/agents/bash-tools.exec-runtime.ts | sed -n '600,625p'`
  - sha: [2011de69d3dc](https://github.com/openclaw/openclaw/commit/2011de69d3dc8f1a11ebc774ba4af23f54892461)
- **Teardown disables late updates:** The exec runtime sets `updatesDisabled = true` before `markExited()` in the settled path and in the catch path, and exposes `disableUpdates()` on the handle so callers can suppress late stdout/stderr-driven updates before teardown completes.
  - file: [src/agents/bash-tools.exec-runtime.ts:834](https://github.com/openclaw/openclaw/blob/2011de69d3dc8f1a11ebc774ba4af23f54892461/src/agents/bash-tools.exec-runtime.ts#L834)
