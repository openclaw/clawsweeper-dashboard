## Summary

Current `main` already denies `nodes` on the HTTP `/tools/invoke` surface, applies that denylist in gateway tool resolution, and includes regression coverage that `nodes` stays blocked over HTTP. The behavior was already present in release `v2026.4.22`, so PR #55906 is redundant.
