## Summary

Keep open. Current main and the latest release still persist redacted transcript/tool-result content by default, and the available `logging.redactSensitive: "off"` workaround is too broad to close a security-sensitive local-provider cache regression.

## What This Changes

The issue requests a way to avoid mutating model-replayed session history when tool-result secret redaction matches local-provider prompt/KV-cache-sensitive text.

## Best Possible Solution

Define a maintainer-approved trusted-local or replay-copy policy that keeps secure defaults for cloud and persisted transcripts while preserving byte-stable model replay for local providers, with focused tests and docs for the chosen boundary.

## Reproduction Assessment

Partially: the transcript mutation is source-reproducible from the append/redaction path and covered by tests, but I did not establish a live local LLM KV-cache reproduction on current main.

## Solution Assessment

Unclear. A blanket local-provider default-off change is not clearly safe; the better direction is a maintainer-approved trusted-local policy or separate persisted-redacted versus model-replay copy with security review.
