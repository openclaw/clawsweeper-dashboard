## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Drop stale generated reset files when rotation misses:** `src/gateway/session-reset-service.ts:83`
  - body: If an operator is already in the reported bad state where `sessionId` changed but `sessionFile` points at another generated transcript, `rewriteSessionFileForNewSessionId` returns `undefined` and this fallback reuses `currentEntry.sessionFile` again. A post-upgrade `sessions.reset` can therefore keep showing the unrelated transcript; detect generated transcript names that do not match the current id and derive the new path instead, while preserving true custom overrides.
  - confidence: 0.87
- **[P2] Keep normalized session entries with a real sessionId:** `src/config/sessions/store-entry-shape.ts:50-52`
  - body: This branch keeps a record with a safe-but-non-transcript id by deleting `sessionId` and returning the remainder as `SessionEntry`. `SessionEntry.sessionId` is still required, and removal/archival paths use `entry.sessionId` when capping or deleting rows, so a metadata-only row can escape validation and later feed an undefined id into cleanup. Use an explicit metadata shape/path or keep pruning rows that cannot satisfy the entry contract.
  - confidence: 0.82
