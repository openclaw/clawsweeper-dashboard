## What This Changes

The branch changes `openclaw agent --model` gateway calls to use backend `gateway-client` admin scope while keeping default agent calls on CLI scope, with command tests for both paths.

## Best Possible Solution

Land this narrow backend/admin-scope switch after the contributor adds redacted real Gateway proof, while preserving CLI scope for ordinary agent calls.

## Reproduction Assessment

Yes, source-reproducible: current docs advertise `openclaw agent --model`, current main sends it as a CLI-scoped gateway `agent` request, and the handler rejects model overrides for callers without admin or internal override authorization. I did not live-run a Gateway reproduction in this read-only review.

## Solution Assessment

Yes, the proposed code direction is the narrowest maintainable fix because it requests admin scope only when a model override is present and matches the existing gateway model-run pattern. Merge should still wait for real behavior proof.
