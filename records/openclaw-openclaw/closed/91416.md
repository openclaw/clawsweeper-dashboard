## Summary

Current main already solves the central history-visibility problem: terminal `status: done` sessions are kept visible, older chat sessions are loaded without the active-window cutoff, and explicitly archived rows can be included with the existing Control UI toggle.

## What This Changes

Request to add a Control UI session visibility toggle so users can switch from running-only sessions to all historical sessions.

## Best Possible Solution

Keep the current main behavior that separates terminal status from explicit archival and lets users include archived sessions without adding a new protocol parameter.

## Reproduction Assessment

No. On current main, source and tests show `status: done` rows remain visible by default and explicitly archived rows can be included, so the reported running-only list behavior does not reproduce from source inspection.

## Solution Assessment

Yes. The current implementation solves the user problem with existing session-list filters and UI projection rather than adding a new backend parameter or treating terminal sessions as archived.
