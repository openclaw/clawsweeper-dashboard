## Summary

Keep open: the latest PR head does not deliver the claimed event-loop fix and introduces separate prompt-authority and plugin-contract regressions. The linked event-loop issue remains open, so this branch needs contributor revision and maintainer review rather than cleanup closure.

## What This Changes

The PR changes CLI credential keychain reading, injects the current prompt into `extraSystemPrompt`, and loosens bundled plugin contract assertions.

## Best Possible Solution

Land a focused auth/gateway performance patch that moves proven keychain hot paths off the event loop while preserving credential expiry, caller contracts, prompt authority boundaries, and exact plugin contract tests.

## Reproduction Assessment

No high-confidence live reproduction was run here. Source inspection confirms current main and the PR head still route macOS keychain reads through synchronous `execSync` calls, matching the reported event-loop warning path.

## Solution Assessment

No. The async direction is plausible, but this PR keeps synchronous keychain reads in the public path and mixes unrelated prompt/security and contract-test changes; the safer fix is a narrow auth/gateway revision.

Overall confidence: 0.97

Full review comments:

- **[P1] Move keychain lookup off the event loop:** `src/agents/cli-credentials.ts:251-253`
  - body: The latest head still calls `execSyncImpl` for `security find-generic-password` inside the sync credential path. That leaves the gateway-capable keychain read blocking the Node event loop, so the central fix is not present.
  - confidence: 0.96
- **[P1] Keep user prompts out of extraSystemPrompt:** `src/agents/pi-embedded-runner/run.ts:365-368`
  - body: This copies raw `params.prompt` into `extraSystemPrompt`, and prompt assembly appends that field to the model system prompt. Marking the text untrusted does not keep user content at user-role authority.
  - confidence: 0.94
- **[P2] Keep per-turn prompt text out of session hashes:** `src/agents/pi-embedded-runner/run.ts:371-373`
  - body: Appending the current user message to `extraSystemPrompt` changes that field every turn. CLI session reuse hashes this prompt material, so normal chat flows can repeatedly invalidate reusable CLI sessions.
  - confidence: 0.88
- **[P2] Preserve last_refresh anchored keychain expiry:** `src/agents/cli-credentials.ts:265`
  - body: The fallback `Date.now() + 60 * 60 * 1000` makes non-JWT keychain credentials look freshly valid on every read. The current `last_refresh` anchor avoids a sliding expiry window for stale credentials.
  - confidence: 0.87
- **[P2] Keep provider compat IDs an exact set check:** `src/plugins/contracts/loader.contract.test.ts:84`
  - body: Changing this equality to `arrayContaining` allows extra provider IDs in the compatibility list without failing the contract, weakening the guard against unintended bundled provider-scope expansion.
  - confidence: 0.88
- **[P2] Restore exact registry coverage assertions:** `src/plugins/contracts/registry.contract.test.ts:21-22`
  - body: The subset assertion allows runtime registry entries to disappear while manifest IDs remain, so the contract no longer verifies that registry coverage matches every bundled capability.
  - confidence: 0.89
