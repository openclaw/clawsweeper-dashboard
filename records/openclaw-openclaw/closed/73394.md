## Summary

Keep open. The PR targets a real source-reproducible Discord autoThread duplicate-reply bug, but the proposed prompt and skill wording makes final-message delivery unconditional and can hide default tool-only guild/channel replies; after-fix live Discord proof is also missing.

## What This Changes

The PR changes the Codex embedded developer instruction, the bundled Discord skill text, and the changelog to reserve the message tool for out-of-band actions and use final assistant output for current-conversation replies.

## Best Possible Solution

Land one canonical delivery-mode-aware Codex and Discord guidance change that uses final assistant output only when OpenClaw will deliver the current reply automatically, while preserving `message(action=send)` guidance for tool-only visible turns.

## Reproduction Assessment

Yes for source-level review. Current main shows autoThread final delivery can target the created thread, while the Codex prompt and Discord skill can still steer the agent to also call `message.send`; I did not run a live Discord session.

## Solution Assessment

No. Prompt and skill wording are the right narrow surface, but this patch is not delivery-mode aware and would undercut the existing `message_tool_only` visible-reply mode.
