## Summary

Current main now implements the requested WebChat/Control UI attachment fix. The add, paste, and drop paths still read the file for send serialization, but the large data URL is kept in a module-level side store rather than Lit reactive state; the state object now carries metadata plus a short object URL preview. Regression tests cover send-time serialization from the side store and confirm optimistic chat state no longer includes base64 payload bytes.

## Best Possible Solution

Close as implemented on main by b22926601fca9e22798515c7d6b1c3c696d20d0b. Keep the side-store/object-URL approach and ship it in the next release; no further core tracking issue is needed for the reported add/preview RangeError unless a new reproduction appears on a build containing this commit.
