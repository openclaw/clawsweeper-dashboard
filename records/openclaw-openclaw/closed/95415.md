## Summary

Current main already fixes the Telegram Web unreadable-message regression by using standard Telegram sendMessage/HTML delivery by default and keeping Bot API rich messages behind explicit richMessages opt-in; the fix is not in the latest v2026.6.8 release yet.

## What This Changes

The issue reports Telegram Web rendering OpenClaw Telegram replies as unsupported because outbound messages are routed through the rich message entity pipeline instead of a standard sendMessage path.

## Best Possible Solution

Keep current main’s standard Telegram delivery default plus explicit richMessages opt-in, and let the next release carry that behavior to affected users.

## Reproduction Assessment

Yes from source and sibling reports: the released regression routed ordinary replies through rich-message payloads that Telegram Web showed as unsupported. Current main no longer uses that default path; live Telegram Web was not run in this read-only review.

## Solution Assessment

Yes. The main fix keeps standard Telegram sendMessage/HTML as the default and leaves rich Bot API messages as explicit opt-in, which is narrower than adding another disable flag.
