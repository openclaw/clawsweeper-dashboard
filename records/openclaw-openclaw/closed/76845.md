## Summary

This PR has already landed on main, and the merged implementation fixes the elevated exec follow-up state loss by carrying trusted elevated defaults through an internal, session-scoped token into the embedded follow-up run. The source path, focused regression tests, merge provenance, and current-main ancestry are strong; the only residual gap is that the contributor proof did not include a complete redacted Telegram/host transcript.

## What This Changes

This PR adds an internal session-scoped one-use elevated follow-up token, carries restored `bashElevated` through gateway agent dispatch into embedded execution, and adds regression tests plus a changelog entry.

## Best Possible Solution

Keep the merged internal token propagation on main and ship it in the next release; add a live Telegram or host approval smoke only if the elevated follow-up flow regresses again.

## Reproduction Assessment

Yes, by source inspection. The pre-merge path dropped `bashElevated` before the follow-up embedded run, and the exec tool reports elevated unavailable when those defaults are missing.

## Solution Assessment

Yes. The merged fix preserves only already-trusted elevated defaults through a short-lived internal token and leaves the approval decision itself on the normal per-command approval path.
