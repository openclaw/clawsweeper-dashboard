## Summary

Keep open. Current main still treats Discord guild/channel turns as message-tool-only unless visible replies are explicitly restored, and the shipped docs/changelog describe that policy, but Discord doctor still has no visible-reply upgrade warning or migration path for existing configs.

## What This Changes

Requests an upgrade-safe Discord group/channel visible-reply behavior, migration, or doctor warning when existing configs lack `messages.groupChat.visibleReplies` and final replies become hidden.

## Best Possible Solution

Add a Discord-owned `openclaw doctor` preview warning, and optionally a conservative repair hint if maintainers want it, for configured Discord guild/channel usage where no explicit visible-reply choice preserves legacy behavior. Keep the current private-by-default runtime policy unless maintainers explicitly choose a broader product change.

## Reproduction Assessment

Yes. The source and tests provide a high-confidence reproduction path: a Discord guild message is treated as `ChatType: "channel"`, absent visible-reply config resolves to `message_tool_only`, and setting `messages.groupChat.visibleReplies` to `automatic` restores visible final replies.

## Solution Assessment

Yes for the narrow doctor diagnostic; no for globally reverting the default, because #73046 intentionally made group/channel replies private-by-default. A Discord-local warning is the smallest maintainable fix for the upgrade confusion reported here.
