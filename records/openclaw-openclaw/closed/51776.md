## Summary

Keep open. Current `main` does not implement Azure Speech TTS, but this PR targets TTS core paths that have since been replaced by plugin-owned speech-provider contracts. The maintainer guidance and current code both point to a rewrite as an extension speech provider, not an auto-close.

## Evidence

- **Current TTS entrypoint is plugin-runtime based:** `src/tts/tts.ts` on current `main` is now only a re-export surface from `../plugin-sdk/tts-runtime.js`, which shows the old in-core provider implementation path this PR edits is no longer the active architecture.
  - file: [src/tts/tts.ts:1](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/src/tts/tts.ts#L1)
  - sha: [42514156e03c](https://github.com/openclaw/openclaw/commit/42514156e03c53dec6be6c581069d8c5b2d4cd42)
- **Speech providers resolve from plugin capability registry:** `src/tts/provider-registry.ts` resolves speech providers through `resolvePluginCapabilityProviders({ key: "speechProviders" })`; it does not register builtin `src/tts/providers/*` implementations.
  - file: [src/tts/provider-registry.ts:19](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/src/tts/provider-registry.ts#L19)
  - sha: [42514156e03c](https://github.com/openclaw/openclaw/commit/42514156e03c53dec6be6c581069d8c5b2d4cd42)
- **Config contract is provider-owned under providers map:** Current TTS config still uses a generic `providers?: TtsProviderConfigMap` field rather than provider-specific core config blocks.
  - file: [src/config/types.tts.ts:26](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/src/config/types.tts.ts#L26)
  - sha: [42514156e03c](https://github.com/openclaw/openclaw/commit/42514156e03c53dec6be6c581069d8c5b2d4cd42)
- **Docs still require messages.tts.providers.<id>:** The user-facing TTS docs say committed config should use `messages.tts.providers.<id>` and describe provider-owned settings under `providers.<id>`. That conflicts with this PR's core-config reshaping.
  - file: [docs/tools/tts.md:426](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/docs/tools/tts.md#L426)
  - sha: [42514156e03c](https://github.com/openclaw/openclaw/commit/42514156e03c53dec6be6c581069d8c5b2d4cd42)
- **Bundled speech providers are extension-owned:** Current bundled providers declare `contracts.speechProviders` in their extension manifest, and the provider implementation lives in `extensions/<id>/speech-provider.ts` (for example Microsoft).
  - file: [extensions/microsoft/openclaw.plugin.json:1](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/extensions/microsoft/openclaw.plugin.json#L1)
  - sha: [42514156e03c](https://github.com/openclaw/openclaw/commit/42514156e03c53dec6be6c581069d8c5b2d4cd42)
- **No Azure speech provider exists on main:** Repository search over `extensions/**` found speech-provider plugins for `openai`, `microsoft`, `elevenlabs`, `google`, `minimax`, `xai`, `xiaomi`, `vydra`, `gradium`, `openrouter`, and `tts-local-cli`, but no `extensions/azure*` speech-provider implementation on current `main`.
  - command: `rg --files extensions | rg 'azure|speech-provider\\.ts$|openclaw\\.plugin\\.json$'`
  - sha: [42514156e03c](https://github.com/openclaw/openclaw/commit/42514156e03c53dec6be6c581069d8c5b2d4cd42)
- **Maintainer already documented the required rewrite:** The April 25, 2026 maintainer comment says this branch cannot be landed as-is because it targets the old core TTS layout, and that Azure must be rebuilt as an `extensions/azure-speech`-style speech-provider plugin with current tests/docs/contracts.

## Risks / Open Questions

- Closing now would discard a still-valid feature request even though Azure Speech TTS is not implemented on current `main`.
- Merging this branch as written would cut across the plugin-owned TTS boundary and conflict with the current speech-provider architecture.
