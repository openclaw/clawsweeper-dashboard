## Summary

Current main already carries the Telegram media proxy-download fix, with code and tests showing media downloads reuse the Telegram transport's proxy dispatcher path and trusted explicit-proxy DNS handling. The changelog records this as shipped in v2026.4.14.

## Evidence

- **Telegram media downloads now reuse the transport proxy path:** `downloadAndSaveTelegramFile` passes `transport.sourceFetch`, `transport.dispatcherAttempts`, and `trustExplicitProxyDns` into `fetchRemoteMedia`, so Telegram file downloads use the same proxy-aware transport metadata as the main Telegram transport.

  - file: [src/media/fetch.ts:127](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/media/fetch.ts#L127)
