## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve the explicit SDK tool name:** `packages/sdk/src/client.ts:185`
  - body: Because `gatewayParams` is spread after `{ name: toolName }`, any JavaScript caller or cast can pass `{ name: "other-tool" }` in the params object and silently invoke a different tool than the explicit first argument. On this direct tool execution surface, strip or reject `params.name`, or spread params first so the explicit `name` argument always wins.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The diff adds a sensitive direct tool invocation RPC, and this branch’s SDK helper allows caller params to override the explicit tool name.

Concerns:

- **[medium] SDK params can override tool name:** `packages/sdk/src/client.ts:185`
  - body: The SDK helper spreads caller params after the explicit tool-name argument. If an app forwards a user-controlled params object, a hidden `name` field can switch the invoked tool while the call site appears to request another tool.
  - confidence: 0.88
