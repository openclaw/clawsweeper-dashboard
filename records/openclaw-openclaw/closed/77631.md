## Summary

Current main fixes the released external-plugin contract discovery gap that made Discord per-account SecretRef tokens stay unresolved and surface as `configured_unavailable`; the fix is main-only and not in `v2026.5.3`.

## What This Changes

Restore SecretRef resolution for per-account Discord tokens when `@openclaw/discord` is installed as an external npm plugin.

## Best Possible Solution

Keep the current main dist-aware external contract loader and ship it in the next patch release so users can return Discord tokens from plaintext to SecretRefs.

## Reproduction Assessment

Yes, source-reproducible: the released loader missed npm plugin `dist/secret-contract-api` sidecars, leaving the Discord account token SecretRef unresolved and driving the documented `configured_unavailable` path. I did not run a live macOS gateway because this review was constrained to read-only inspection.

## Solution Assessment

Yes. Resolving the generic external plugin contract lookup is the narrowest maintainable fix; changing Discord token fallback behavior would bypass the plugin secret-contract model and risk reintroducing plaintext/env coupling.
