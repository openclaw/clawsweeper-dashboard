## Summary

The one-line guard in this PR is directionally correct, but #74989 is an open mergeable superset for the same undefined plugin-command result bug and includes central normalization, text and Telegram consumer hardening, and regression tests, so keeping this parallel minimal PR open would duplicate maintainer review.

## What This Changes

This PR changes `executePluginCommand` to coerce nullish plugin command handler results to an empty reply object before returning to callers.

## Best Possible Solution

Consolidate on a single canonical fix, preferably #74989 or an equivalent branch that normalizes plugin command results at the shared executor boundary and keeps focused regression coverage for the shared text and Telegram native command paths.

## Reproduction Assessment

Yes. A plugin command handler that returns `undefined` is enough: current main returns that value from `executePluginCommand`, and the text and Telegram command paths read fields on the result immediately.

## Solution Assessment

No for this branch as the merge vehicle. The executor-boundary guard is the right primitive, but #74989 is the stronger current solution because it includes regression tests and adjacent consumer normalization while tracking the same root bug.
