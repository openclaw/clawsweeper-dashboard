## What This Changes

The issue asks OpenClaw to address Weixin runtime initialization timeouts and Content-Length failures after upgrading with @tencent-weixin/openclaw-weixin@2.4.1.

## Best Possible Solution

Ship the runtime, Content-Length, and long-poll timeout fixes in the external @tencent-weixin/openclaw-weixin plugin or its ClawHub listing, and keep OpenClaw core's generic channelRuntime contract unchanged unless a missing cross-plugin API is identified.
