## What This Changes

The PR changes the old memory manager fallback to return keyword-backed merged entries when strict hybrid scoring returns nothing and adds memory regression tests for BM25 and temporal-decay cases.

## Best Possible Solution

Close this obsolete branch and keep future memory hybrid-search tuning against extensions/memory-core, with https://github.com/openclaw/openclaw/pull/78834 as the current-main implementation anchor.

## Reproduction Assessment

Yes. The linked report and current tests give a high-confidence source reproduction for BM25-only hybrid hits falling below minScore, and current main now covers that failure class.

## Solution Assessment

Yes for closing, not for merging this branch. The maintained memory-core implementation preserves the keyword-backed fallback set directly, while this branch targets a removed path and still keeps only the top post-merge score group.
