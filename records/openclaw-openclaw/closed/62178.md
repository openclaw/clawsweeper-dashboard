Current `main` and the shipped `v2026.4.23` release already implement the issue’s requested opt-in fix: `web_fetch` now accepts `tools.web.fetch.ssrfPolicy.allowRfc2544BenchmarkRange`, threads it into the guarded fetch path, and has regression coverage for RFC2544/FakeIP addresses.
