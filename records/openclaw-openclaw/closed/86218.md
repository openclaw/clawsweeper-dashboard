## Summary

Keep open: current main still has a source-proven stable-channel git updater bug. The stable git path filters out beta tags but not alpha tags, so an alpha tag can be selected as the latest “stable” tag when it sorts above the npm latest stable release.

## What This Changes

The issue reports that a git-based stable-channel update checked out `v2026.5.24-alpha.1` instead of the npm/latest stable target `v2026.5.22`.

## Best Possible Solution

Make stable git tag resolution exclude prereleases or validate against the stable npm/latest target, then add focused regression coverage for alpha, beta, and stable tag ordering.

## Reproduction Assessment

Yes from source inspection, though not from a live command. Current main treats alpha tags as stable because stable only means “not beta,” and the git updater checks out the first sorted matching tag.

## Solution Assessment

Yes. The narrow maintainable fix is to tighten updater channel/tag resolution and tests, not to add an operator setting or a second updater path.
