## Summary

Keep open: current main and the latest release still apply the shared history limiter after compaction-summary construction, and that limiter slices away any leading summary when the retained tail exceeds the configured user-turn limit.

## What This Changes

The issue reports that `historyLimit` and `dmHistoryLimit` silently drop the leading post-compaction summary from model context after auto-compaction.

## Best Possible Solution

Patch the central history limiter so a leading `compactionSummary` is preserved before the trimmed tail, without adding new config or channel-specific branches.

## Reproduction Assessment

Yes. Current source shows `buildSessionContext` prepends a `compactionSummary`, and `limitHistoryTurns` can then slice from a later user-turn index and drop that prefix.

## Solution Assessment

Yes. The best fix is in the shared limiter because both the normal attempt path and compaction path already funnel through it.
