## Summary

Issue #16332 asks OpenClaw to perform a bespoke Oracle PL/SQL-to-C# application migration for an external system. On current `main`, that kind of optional, task-specific automation is explicitly scoped to plugins/skills and new skills are expected to ship via ClawHub rather than OpenClaw core.

## Close Comment

Current project direction keeps optional capability in plugins/skills (`VISION.md:54-70`), and specifically says new skills that can live outside core should go to ClawHub first (`VISION.md:99-101`). If you want this in the OpenClaw ecosystem, the right path is a ClawHub skill or plugin for codebase migration rather than a core OpenClaw issue.
