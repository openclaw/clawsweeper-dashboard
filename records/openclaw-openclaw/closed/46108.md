## Close Comment

Closing as fixed after a Codex review of current `main`.

This report matches a plugin HTTP-route registry regression that is already implemented and shipped on `main`. The current gateway path pins webhook routes to the startup plugin registry and resolves request dispatch against that same live route registry (`src/plugins/runtime.ts:13`, `src/gateway/server-runtime-state.ts:108`, `src/gateway/server/plugins-http.ts:63`). There is also direct regression coverage proving that routes registered after later registry churn still dispatch instead of falling through to 404 (`src/gateway/server/plugins-http.test.ts:300`).

The fix is called out in `CHANGELOG.md:2391` under release `2026.3.22`, so this was shipped in `v2026.3.22`. If you still see `404 Not Found` on a newer build than that, please open a fresh issue with the exact version, gateway logs, and current `channels.synology-chat` / reverse-proxy setup.
