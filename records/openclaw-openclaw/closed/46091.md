## Summary

The reported failure mode is already fixed in current OpenClaw, and the fix is present in the shipped `v2026.4.22` tree. Control UI `/model` accepts bare model ids, sends the raw id through `sessions.patch`, and normalizes the local session override back to a provider-qualified value.

## Evidence

- **Released Control UI test accepts bare `/model` ids:** In the `v2026.4.22` tree, the Control UI test covers `/model gpt-5-mini`, expects `sessions.patch` to receive the bare id, and expects the local override to become `openai/gpt-5-mini`. That is the exact failure this issue reports, and the released test asserts it works.
