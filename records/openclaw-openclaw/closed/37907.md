## Summary

Current main already suppresses this warning in the recovered-turn case by clearing `lastToolError` when the same mutating action later succeeds, so the payload builder no longer emits the synthetic `⚠️ ... failed` message for that path.

## Evidence

- **Recovery happens before payload rendering:** `handleToolExecutionEnd` clears `state.lastToolError` after a successful retry of the same mutating action via `isSameToolMutationAction`, which prevents the later warning path from seeing a lingering tool error.
