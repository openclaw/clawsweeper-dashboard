## Summary

Current `main` already restores MiniMax image routing through the plugin-owned `MiniMax-VL-01` path, and the shipped `v2026.4.23` release includes code, tests, docs, and changelog evidence that cover this regression class.

## Evidence

- **Image tool now prefers MiniMax VL for MiniMax primaries:** When the active primary provider is `minimax` or `minimax-portal` and no explicit `imageModel` is set, the image tool injects `MiniMax-VL-01` as the primary image candidate instead of reusing the M2.7 chat model.

- **Regression scenario is covered by test:** The test suite explicitly asserts that `agents.defaults.model.primary = minimax/MiniMax-M2.7` resolves the image tool to `minimax/MiniMax-VL-01` without requiring an explicit `agents.defaults.imageModel` override.
  - file: [src/agents/tools/image-tool.test.ts:624](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/agents/tools/image-tool.test.ts#L624)
