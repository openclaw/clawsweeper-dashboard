## Summary

Current `main` already avoids stale-socket restarts for WhatsApp-style idle connections. The gateway health policy only classifies `stale-socket` when a channel reports `lastTransportActivityAt`, and the WhatsApp runtime publishes `lastEventAt` but not transport-liveness timestamps. Regression tests also cover the no-transport-liveness case, so the issue’s reported last-event-based restart loop is obsolete on current `main` and in the latest release snapshot inspected.

## Evidence
