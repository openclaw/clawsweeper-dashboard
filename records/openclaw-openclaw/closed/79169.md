## Summary

Keep open: current main still has directory-only safe-bin executable trust, while the requested path/hash trust would add a new security-sensitive exec trust and config surface that overlaps the existing exec approvals allowlist.

## What This Changes

Request per-binary safe-bin trust by absolute path, with optional SHA-256 pinning, instead of trusting every executable in a configured directory.

## Best Possible Solution

Decide the canonical exec trust contract first; if accepted, add typed path/hash trust with schema, docs, audit/doctor guidance, and regression coverage around the chosen surface.

## Reproduction Assessment

Not applicable as a feature request. Current-main source inspection gives high confidence that directory-only safe-bin trust remains, but there is no failing existing contract to reproduce.

## Solution Assessment

Unclear. Path/hash trust could solve the wrapper-maintenance problem, but maintainers should settle the product/security boundary with the existing host allowlist before implementation.
