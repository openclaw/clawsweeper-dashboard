## Summary

Current main already routes bundled Telegram sidecar loads through the shared Windows-safe plugin/channel loader, and the shipped changelog records the Windows native-loader fix for Telegram-class channel plugins after the reported v2026.4.24 regression.

## What This Changes

The issue asks OpenClaw to stop passing raw Windows drive-letter paths from the Telegram channel loader to Node's ESM loader.

## Best Possible Solution

Keep the shared Windows-safe plugin/channel loader path and direct users on v2026.4.24 to a release containing the 2026.4.25 loader fix or newer.

## Reproduction Assessment

Yes, for the failure class. The issue's Windows v2026.4.24 steps are plausible, and current regression coverage simulates a raw C:\ sidecar path and verifies current main converts it to a file URL before module loading.

## Solution Assessment

Yes. The current shared loader normalization is the narrow maintainable fix because it protects Telegram and other bundled plugin sidecar loads without adding Telegram-specific path handling.
