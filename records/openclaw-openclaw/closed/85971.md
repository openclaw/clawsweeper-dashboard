## Summary

Keep this issue open: source/tag evidence and multiple related reports show the Android v2026.4.5 app advertises protocol 3..3 while current and latest released Gateway code requires protocol v4 and rejects that range. Current source is v4-compatible, but that does not prove the public Play Store app can pair; the broader compatibility tracker https://github.com/openclaw/openclaw/issues/83736 and Android distribution tracker https://github.com/openclaw/openclaw/issues/9443 overlap but do not fully supersede this exact Play Store regression.

## What This Changes

The issue reports that the Google Play Android app v2026.4.5 cannot pair with Gateway releases that require Gateway protocol v4 because the app advertises only protocol v3.

## Best Possible Solution

Keep this as the canonical Play Store protocol regression until maintainers either publish and verify a v4-compatible Play/beta/official Android build or deliberately ship and test a Gateway compatibility policy for v2026.4.5 clients, while cross-linking https://github.com/openclaw/openclaw/issues/83736 and https://github.com/openclaw/openclaw/issues/9443 for the broader compatibility and distribution decisions.

## Reproduction Assessment

Yes from source inspection and reporter logs: Android v2026.4.5 advertises protocol 3..3, while current/latest Gateway code requires protocol v4 and rejects non-probe clients outside that range. I did not perform a live Play Store device repro.
