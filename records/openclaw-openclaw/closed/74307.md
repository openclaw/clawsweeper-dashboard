## Summary

Keep open. Current main deliberately creates the staged `openclaw/plugin-sdk` alias as a wrapper directory, so the symlink part of the report is not the right invariant. The remaining runtime-deps concern is still actionable: doctor’s packaged repair selection is channel/plugin-entry oriented, while `anthropic-vertex` is a provider-only bundled plugin with staged runtime dependencies, and the current tests only cover provider deps when the provider plugin is explicitly enabled.

## What This Changes

The issue asks `openclaw doctor --fix` on macOS global npm installs to converge bundled provider runtime-dependency repair for `anthropic-vertex` instead of reporting success while the staged dependency exposure remains incomplete.

## Best Possible Solution

Keep the wrapper-directory `plugin-sdk` alias, but make doctor’s bundled runtime-deps repair include provider-owned bundled plugins that are active or configured through model/provider/auth surfaces, including `anthropic-vertex`, while respecting deny/disable/allowlist policy and surfacing a hard repair failure when active provider imports remain unsatisfied.
