## Summary

Current `main` already implements the requested WebChat multi-agent/session switching. The Control UI chat header renders a grouped session selector, session labels are grouped/disambiguated by agent, switching is wired to change the active `agent:<id>:...` session and reload chat state, and the docs explicitly describe WebChat as attaching to the selected agent. I also verified the same implementation is present in release `v2026.4.22`.

## Evidence

- **Chat header has an agent-grouped session selector:** `renderChatSessionSelect` renders a `<select>` for the active chat session, with `<optgroup>` sections built from resolved session groups. `resolveSessionOptionGroups` groups `agent:<id>:...` sessions by agent label and disambiguates duplicate labels so multiple agents/sessions are selectable from WebChat.
  - file: [ui/src/ui/chat/session-controls.ts:23](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/ui/src/ui/chat/session-controls.ts#L23)
  - sha: [4630ce3d9e2f](https://github.com/openclaw/openclaw/commit/4630ce3d9e2f089062ccd386b0ae241325961d1a)
- **Chat wiring switches to the selected agent/session:** The chat view passes `onAgentChange` as `switchChatSession(state, buildAgentMainSessionKey({ agentId }))` and `onSessionSelect` as `switchChatSession(state, key)`, so picking another agent or session changes the active chat target directly from Control UI.
  - file: [ui/src/ui/app-render.ts:2287](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/ui/src/ui/app-render.ts#L2287)
  - sha: [4630ce3d9e2f](https://github.com/openclaw/openclaw/commit/4630ce3d9e2f089062ccd386b0ae241325961d1a)
- **Session switch reloads the correct chat context:** `switchChatSession` resets session-scoped chat state, refreshes assistant identity/avatar, refreshes slash commands for the parsed agent id, reloads chat history, and refreshes session options. That is the expected behavior for a real agent/session switcher rather than a cosmetic selector.
  - file: [ui/src/ui/app-render.helpers.ts:509](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/ui/src/ui/app-render.helpers.ts#L509)
  - sha: [4630ce3d9e2f](https://github.com/openclaw/openclaw/commit/4630ce3d9e2f089062ccd386b0ae241325961d1a)
- **Docs now describe WebChat as agent-selectable:** The routing docs explicitly state: WebChat attaches to the selected agent and defaults to that agent’s main session.
  - file: [docs/channels/channel-routing.md:130](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/docs/channels/channel-routing.md#L130)
