## Summary

Keep open. Current main still leaves configured speech-provider ownership out of Gateway startup planning, Microsoft is declared as a non-startup speech provider, and open PR #76540 explicitly closes this issue, so the issue should track that PR until it merges or is rejected.

## What This Changes

Selecting the Microsoft or legacy Edge TTS provider after the 2026.5.2 upgrade should activate the owning speech plugin during Gateway startup so Telegram voice replies can synthesize audio.

## Best Possible Solution

Review and land the linked startup-planner fix so configured speech-provider owners are included in Gateway startup while preserving plugin disablement and allowlist rules.

## Reproduction Assessment

Yes at the startup-planner level: current source shows `messages.tts.provider=microsoft` has no path into `startup.pluginIds` while Microsoft is `onStartup: false`. I did not run a live Telegram TTS smoke in this read-only review.

## Solution Assessment

Yes, the linked PR direction is the narrow maintainable fix: add configured speech-provider owner selection to Gateway startup planning, including the legacy `edge` alias, without changing synthesis, Telegram delivery, or plugin install behavior.
