## Summary

Current main and the shipped v2026.4.26 tag no longer rely on a stale per-session bootstrap snapshot for heartbeat workspace files. Heartbeat preflight reads HEARTBEAT.md from the agent workspace for each normal interval run, session bootstrap snapshots refresh on each turn, and transcript persistence stores a neutral heartbeat marker instead of replaying the full heartbeat prompt as the visible user prompt.

## Best Possible Solution

Keep the shipped v2026.4.26 behavior: heartbeat runs should continue refreshing workspace bootstrap files per turn, reading HEARTBEAT.md during heartbeat preflight, and persisting only the neutral heartbeat transcript marker instead of stale HEARTBEAT.md prompt text.
