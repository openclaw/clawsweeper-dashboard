## Summary

Current `main` implements browser realtime voice chat for the WebChat/Control UI chat surface: a Talk button starts microphone capture, requests an ephemeral realtime session from Gateway, streams audio over WebRTC, autoplays assistant audio, and can consult the full OpenClaw agent during the live session. This was not present in the latest tagged release `v2026.4.22`, so the feature appears implemented on `main` but not yet released.
