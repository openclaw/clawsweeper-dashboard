## Summary

Keep open: the reported gateway auth issue is valid, but this PR is not merge-ready because the latest patch creates a new bootstrap-token pairing-bypass path that needs maintainer/security review.

## What This Changes

The PR changes gateway backend self-pairing logic, helper tests, message-handler comments, and gateway security docs so shared token/password auth no longer triggers the backend pairing skip and bootstrap-token/device-token auth does.

## Best Possible Solution

Land a security-reviewed fix that removes shared token/password from backend self-pairing and keeps bootstrap-token handshakes on the existing pairing/handoff path unless maintainers explicitly approve a narrower backend exception.

## Reproduction Assessment

Yes. Source inspection on current main shows the shared-token/password backend skip, and PR-head inspection shows a bootstrap-token backend skip that would be evaluated before `requirePairing`.

## Solution Assessment

No. Removing shared token/password from the bypass is the right direction, but adding bootstrap-token to the skip is not the narrowest maintainable fix because bootstrap tokens already have guarded pairing and handoff logic.
