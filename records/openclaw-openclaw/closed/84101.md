## Summary

Close as implemented: current main and v2026.5.18 still fail closed for cron models outside `agents.defaults.models`, but the cron error now prints the configured allowlist and docs explain that cron `--model` must be allowed.

## What This Changes

The issue asks for cron `payload.model` allowlist rejection to be clear and consistent when the selected model is not present in `agents.defaults.models`.

## Best Possible Solution

Keep the shipped fail-closed allowlist behavior; operators should add the exact Anthropic model to `agents.defaults.models` with a merge, remove or widen the allowlist, and leave the separate explicitly-configured rejection case tracked at https://github.com/openclaw/openclaw/issues/83617.

## Reproduction Assessment

No for current main as reported against 2026.5.7: source and regression coverage show the confusing echo has been replaced by an error that includes the configured allowlist. The strict rejection remains source-proven when the payload model is outside `agents.defaults.models`.

## Solution Assessment

Yes. The narrow maintainable solution is to keep cron fail-closed against explicit allowlists while printing the effective configured keys and documenting how to add or clear the allowlist.
