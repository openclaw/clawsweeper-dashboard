## Summary

Current main already solves the Google Vertex ADC sentinel problem through the pinned @mariozechner/pi-ai 0.70.5 provider: built-in google-vertex models use the pi-ai google-vertex transport, and that transport now ignores both the current gcp-vertex-credentials marker and placeholder markers like <authenticated> before choosing the ADC client path. The PR is obsolete and should not be merged as-is because it targets the older workaround and includes unrelated bird skill content.

## Best Possible Solution

Close this PR as implemented on current main. Keep the shipped pi-ai 0.70.5 google-vertex marker handling and avoid merging the stale wrapper plus unrelated bird skill addition; open a new focused issue only if a separate custom-transport Vertex configuration still leaks markers.
