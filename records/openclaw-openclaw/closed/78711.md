## Summary

Current main already covers the reported Control UI reconciliation bug: local optimistic user messages are preserved across stale history reloads, and same-session history reloads are deferred while a chat run is active. I could prove the fix on main, but not in a shipped tag yet.

## What This Changes

Keep the submitted user message visible in the Control UI chat during long Codex agent runs instead of disappearing until the final response.

## Best Possible Solution

Keep the current main UI reconciliation behavior and ship it in the next release so older tagged builds inherit the fix.

## Reproduction Assessment

Yes for the original report: the issue gives concrete `/chat` steps and backend transcript evidence, and current source shows the exact history-reload/optimistic-message race that was guarded. I did not establish a current-main failing path because the guard and regression coverage are already present.

## Solution Assessment

Yes. Preserving the optimistic tail in `chat.history` reconciliation and deferring same-session reloads during an active run is the narrow UI-side fix; it leaves backend persistence and transcript storage unchanged.
