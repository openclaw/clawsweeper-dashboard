## Summary

Keep open: current main still hydrates every stored session before trimming to recent output, and https://github.com/openclaw/openclaw/pull/84813 is an open, unmerged closing PR for the focused fix path.

## What This Changes

The issue reports `openclaw status` and `openclaw status --json` taking 40-50 seconds because session summary/model-runtime resolution scales across many stored sessions.

## Best Possible Solution

Review and land or reject the linked focused PR; after it merges, close this issue as implemented, otherwise apply the same bounded summary-hydration optimization with regression coverage.

## Reproduction Assessment

Yes for the source-level path: current main resolves model/runtime/context details for every stored session before slicing to recent rows. I did not perform a live timing run because this review is read-only and the exact latency depends on a large local session store.

## Solution Assessment

Yes: the maintainable fix is to count all sessions with lightweight metadata and hydrate only the recent rows that status renders, rather than adding a workaround flag or changing the user-visible status contract.
