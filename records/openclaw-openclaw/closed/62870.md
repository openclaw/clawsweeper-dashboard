## Summary

Current `main` does not support the PR's stated root cause. The tool-policy pipeline already preserves plugin-only allowlists, and plugin tool discovery receives the merged explicit allowlist before policy filtering. The proposed change would also disable existing unknown-allowlist warnings for subagent agent/group steps and still does not add coverage for the claimed path.

## Evidence

- **Pipeline does not strip plugin-only allowlists:** `applyToolPolicyPipeline()` analyzes allowlists for warnings when `stripPluginOnlyAllowlist` is set, but it keeps the resolved policy and then expands plugin groups before filtering. There is no branch here that removes plugin-only allowlists from agent/group steps.
  - file: [src/agents/tool-policy-pipeline.ts:116](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/agents/tool-policy-pipeline.ts#L116)
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Plugin-only entries are explicitly recognized and preserved:** `analyzeAllowlistByToolType()` treats `group:plugins`, plugin ids, and plugin tool ids as plugin entries and returns the original policy object with `pluginOnlyAllowlist` metadata; it does not rewrite the allowlist away.
  - file: [src/agents/tool-policy.ts:167](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/agents/tool-policy.ts#L167)
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Current tests already cover preserved plugin-only allowlists:** The unit tests assert that plugin-only allowlists and `group:plugins` are preserved, which directly contradicts the PR's claim that current `main` strips them before subagent policy is applied.
  - file: [src/agents/tool-policy.plugin-only-allowlist.test.ts:15](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/agents/tool-policy.plugin-only-allowlist.test.ts#L15)
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Gateway tool loading already includes subagent/group allowlists before filtering:** `resolveGatewayScopedTools()` passes a merged explicit allowlist including `groupPolicy` and `subagentPolicy` into `createOpenClawTools()`, so optional plugin tools are loaded from those explicit entries before the pipeline filters the final tool list.
  - file: [src/gateway/tool-resolution.ts:83](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/gateway/tool-resolution.ts#L83)
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Optional plugin tools are admitted from explicit plugin/group allowlists:** `resolvePluginTools()` allows optional plugin tools when the explicit allowlist contains the tool name, the plugin id, or `group:plugins`. That matches the inheritance path described in the PR without requiring the proposed `preservePluginTools` toggle.
  - file: [src/plugins/tools.ts:42](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/plugins/tools.ts#L42)
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Prior review discussion identified regressions in the proposed patch:** The provided PR review comments already called out two unresolved problems in the patch itself: it suppresses unknown-allowlist warnings for subagent agent/group steps, and it adds no regression coverage for the new branch. Those comments strengthen the case that this PR is not the right fix.
