## Summary

Current main already supports the requested `/new` handoff flow through the bundled `session-memory` hook plus the startup-context prelude that injects recent memory into the fresh session. The same implementation is also present in the latest confirmed release snapshot.

## Evidence

- **Bundled hook docs match the request:** Docs describe a bundled `session-memory` hook for `command:new` and `command:reset` that saves session context to `<workspace>/memory/`, including extracting the last 15 user/assistant messages and writing `YYYY-MM-DD-slug.md`.
  - file: [docs/automation/hooks.md:156](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/docs/automation/hooks.md#L156)
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
- **Hook implementation saves the pre-reset session into memory:** The handler triggers on `new`/`reset`, prefers `previousSessionEntry`, reads recent session content, generates a slug, and writes a Markdown file with a `Conversation Summary` section under `memory/`.
  - file: [src/hooks/bundled/session-memory/handler.ts:53](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/src/hooks/bundled/session-memory/handler.ts#L53)
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
- **Startup context loads saved daily memory into the new session:** `buildSessionStartupContextPrelude` gathers recent `memory/*.md` files and returns a `[Startup context loaded by runtime]` prelude for the new session.
  - file: [src/auto-reply/reply/startup-context.ts:309](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/src/auto-reply/reply/startup-context.ts#L309)
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
- **Gateway injects the startup prelude on bare `/new` and `/reset`:** The agent gateway path prepends the startup context prelude before dispatching the fresh-session run.
  - file: [src/gateway/server-methods/agent.ts:906](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/src/gateway/server-methods/agent.ts#L906)
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
- **Regression test covers the handoff behavior:** A test verifies that bare `/new` prepends runtime-loaded startup memory, including recent `memory/YYYY-MM-DD.md` content, into the next run.
  - file: [src/gateway/server-methods/agent.test.ts:1123](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/src/gateway/server-methods/agent.test.ts#L1123)
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
