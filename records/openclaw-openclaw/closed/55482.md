## Summary

Current `main` already routes Telegram health/channel-status through the resolved default or bound account instead of a placeholder `default` account, and the gateway runtime snapshot publishes `running: true` for active accounts. That matches the bug report’s false `running:false` / `tokenSource:none` symptom and makes #55482 obsolete on `main`.

## Evidence

- **Telegram default-account resolution now prefers the real bound/configured account:** `resolveDefaultTelegramAccountSelection` first checks the default agent’s bound Telegram account, then `channels.telegram.defaultAccount`, before falling back. That prevents status surfaces from summarizing an unconfigured placeholder `default` account when a different Telegram account is actually in use.
  - file: [extensions/telegram/src/account-selection.ts:121](https://github.com/openclaw/openclaw/blob/fc5920fb51346a682b269fdbeeec0971ae4ae723/extensions/telegram/src/account-selection.ts#L121)
  - sha: [fc5920fb5134](https://github.com/openclaw/openclaw/commit/fc5920fb51346a682b269fdbeeec0971ae4ae723)
- **Telegram status metadata is built from inspected credentials for the resolved account:** The Telegram config adapter wires `defaultAccountId: resolveDefaultTelegramAccountId`, and `describeAccount` uses `inspectTelegramAccount(...)` to expose the selected account’s `configured` and `tokenSource` fields.
  - file: [extensions/telegram/src/shared.ts:102](https://github.com/openclaw/openclaw/blob/fc5920fb51346a682b269fdbeeec0971ae4ae723/extensions/telegram/src/shared.ts#L102)
  - sha: [fc5920fb5134](https://github.com/openclaw/openclaw/commit/fc5920fb51346a682b269fdbeeec0971ae4ae723)
- **`channels.status` summarizes the resolved default account, not a hard-coded placeholder:** The gateway `channels.status` handler resolves `defaultAccountId`, builds snapshots for each account, then picks `accounts.find((entry) => entry.accountId === defaultAccountId)` for the top-level channel summary.
  - file: [src/gateway/server-methods/channels.ts:253](https://github.com/openclaw/openclaw/blob/fc5920fb51346a682b269fdbeeec0971ae4ae723/src/gateway/server-methods/channels.ts#L253)
  - sha: [fc5920fb5134](https://github.com/openclaw/openclaw/commit/fc5920fb51346a682b269fdbeeec0971ae4ae723)
- **Active channel accounts publish `running: true` into the runtime snapshot:** When an account starts, the channel manager sets `running: true`, `lastStartAt`, and clears `lastError`, so a functioning Telegram bot should no longer be surfaced as stopped once the correct account is selected.
  - file: [src/gateway/server-channels.ts:408](https://github.com/openclaw/openclaw/blob/fc5920fb51346a682b269fdbeeec0971ae4ae723/src/gateway/server-channels.ts#L408)
  - sha: [fc5920fb5134](https://github.com/openclaw/openclaw/commit/fc5920fb51346a682b269fdbeeec0971ae4ae723)
- **Gateway runtime snapshots expose the selected default account at channel level:** `getRuntimeSnapshot()` resolves each plugin’s `defaultAccountId` and publishes that account as `channels[plugin.id]`, with all account snapshots preserved under `channelAccounts[plugin.id]`.
  - file: [src/gateway/server-channels.ts:645](https://github.com/openclaw/openclaw/blob/fc5920fb51346a682b269fdbeeec0971ae4ae723/src/gateway/server-channels.ts#L645)
  - sha: [fc5920fb5134](https://github.com/openclaw/openclaw/commit/fc5920fb51346a682b269fdbeeec0971ae4ae723)
