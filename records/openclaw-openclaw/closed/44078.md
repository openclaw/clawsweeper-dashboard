## Summary

Current `main` now widens outbound local-media access to include the parent directory of explicit local media sources, and the Telegram outbound path consumes that access when loading uploads. That addresses the reported `LocalMediaAccessError` for `openclaw message send --channel telegram --media <local-file>` when the file lives outside the default OpenClaw-managed roots.

## Evidence

- **Direct outbound delivery now resolves media access from concrete payload media sources:** `deliverOutboundPayloadsCore` collects media sources from the outbound payload plan and passes them into `resolveAgentScopedOutboundMediaAccess`, so direct CLI sends no longer rely only on the default OpenClaw roots.
  - file: [src/infra/outbound/deliver.ts:737](https://github.com/openclaw/openclaw/blob/5f702b464bb5500f3b0739741548c729c9011943/src/infra/outbound/deliver.ts#L737)

## Risks / Open Questions
