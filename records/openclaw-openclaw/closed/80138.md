The PR updates doctor Codex route repair so legacy `openai-codex/*` model refs become canonical `openai/*` refs while preserving Codex auth through model-scoped runtime policy, with docs, changelog, and tests.

## Best Possible Solution

Preserve Codex auth during doctor repair while keeping model visibility unchanged, either by separating runtime-policy storage from visibility or by making visibility semantics ignore runtime-only entries.

## Reproduction Assessment

Yes, source-reproducible: latest PR head and live main create `agents.defaults.models` for a bare default-agent repair, and current model selection treats any default model-map key as visibility state. I did not run tests because this review is read-only.

## Solution Assessment

No. Preserving Codex auth intent is the right direction, but storing that intent by creating `agents.defaults.models` changes `/model` and session override visibility for configs that did not already opt into an allowlist.
