## Summary

Keep open. Current main documents and tests two distinct supported paths: `existing-session` uses Chrome MCP and must not use `cdpUrl`, while raw CDP `attachOnly` profiles do support loopback `ws://` endpoints. The issue body/comments mix those modes and later report the local `127.0.0.1:9222` endpoint returning `404` again, so there is not high-confidence evidence for a safe auto-close.
