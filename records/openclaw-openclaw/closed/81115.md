## Summary

Current main and the shipped v2026.5.7 code already normalize MiniMax usage into internal usedPercent and render remaining quota as `% left`; the new report repeats the prior issue but does not include a redacted raw MiniMax payload showing a changed or uncovered response shape.

## What This Changes

The issue asks for `openclaw status` and status cards to stop labeling a MiniMax used percentage as `% left`.

## Best Possible Solution

Keep the normalized `usedPercent` contract and `% left` display; investigate only if a redacted MiniMax payload proves a changed or uncovered response shape.

## Reproduction Assessment

No. Source, docs, and regression tests on current main show the known MiniMax remaining-field shapes normalize to internal usedPercent and then render as remaining `% left`; the issue lacks the raw provider payload needed to reproduce a different live shape.

## Solution Assessment

Yes. Normalizing provider-specific remaining fields into the existing `usedPercent` contract and rendering `100 - usedPercent` is the narrow maintainable solution; changing the label globally or adding a MiniMax-specific display mode would conflict with the documented contract.
