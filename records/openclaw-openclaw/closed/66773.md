## Summary

Current main still returns the freshly rotated device token in the `device.token.rotate` gateway RPC response. The behavior is implemented in the gateway handler, backed by device-pairing token creation, asserted by tests, and still consumed by CLI/UI surfaces; the related redaction PR #66794 is closed and unmerged, so there is no shipped implementation to close against.

## Best Possible Solution

Keep this issue open as a gateway protocol/security design bug. The likely fix belongs in `src/gateway/server-methods/devices.ts` and the device-token contract: stop returning `entry.token` from normal rotate responses, add regression tests proving the fresh secret is absent or only exposed through an explicit safer provisioning/bootstrap path, and update CLI, Control UI, and docs to the chosen contract. Review or replace the closed #66794 approach rather than closing this issue before a merged fix lands.
