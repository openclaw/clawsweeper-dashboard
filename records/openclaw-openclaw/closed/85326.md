## Summary

This PR should stay open: the diff adds an explicit `--agent` path, but it still misses the documented `OPENCLAW_AGENT_DIR`/`PI_CODING_AGENT_DIR` workflow from the linked report and has only mock-based proof. The protected `maintainer` label also rules out conservative cleanup closure.

## What This Changes

The branch adds `--agent` to `infer/capability model auth logout`, routes logout through the selected agent id, adds focused mocked regression coverage, and records a changelog entry.

## Best Possible Solution

Resolve logout targets as explicit `--agent`, then `OPENCLAW_AGENT_DIR`/`PI_CODING_AGENT_DIR`, then configured default; validate configured agent ids and prove real before/after non-default auth-store writes.

## Reproduction Assessment

Yes, source-reproducible. Current main always resolves logout through the configured default agent id, while the linked report gives a concrete env-scoped command and the docs establish the env override contract.

## Solution Assessment

No. Adding explicit `--agent` is useful, but the maintainable fix should reuse the existing target resolution semantics and validation used by the models auth/status surfaces.
