## Summary

## What This Changes

The issue requests adding `video` as a supported `models.providers.*.models[].input` value for video-understanding models.

## Best Possible Solution

Keep the shipped `v2026.5.18` model input contract and use the dedicated media-understanding/provider capability path for actual video analysis behavior.

## Reproduction Assessment

Yes for the older reported release by source inspection: `v2026.3.24` limited model inputs to `text` and `image`. Current `main` does not reproduce that schema failure because `video` is now accepted.

## Solution Assessment

Yes; expanding the existing model input contract is the maintainable fix for the reported validation error, while provider-specific video behavior stays in media-understanding capability code.
