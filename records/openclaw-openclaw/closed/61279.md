## Summary

Keep this issue open. Current main still does not pre-create `/home/node/.openclaw` in the runtime image before switching to the non-root `node` user, while the Compose and config-write paths still match the reported Docker named-volume EACCES failure. The remaining work is already represented by open maintainer PR #72662, but it has not landed on current main.

## Best Possible Solution

Review and land #72662, or an equivalent narrow Dockerfile change that creates `/home/node/.openclaw` before `USER node` with `node:node` ownership and restrictive permissions. Preserve credit for #48072 and #63959, add Docker named-volume smoke proof or equivalent maintainer validation, and document recovery for already root-owned volumes before closing this issue as implemented.
