## Summary

Close as implemented on main: current main and v2026.5.2 already provide plugin-command continuation via the shipped `continueAgent` field, while this stale PR would now regress that contract by switching the handler to `shouldContinue`.

## What This Changes

The PR adds a `shouldContinue` field to plugin command results, maps it to agent continuation, suppresses replies when continuing, and updates tests/JSDoc.

## Best Possible Solution

Keep the shipped `continueAgent` contract and close this stale PR; any remaining wording polish should be a small docs/JSDoc follow-up against current main.

## Reproduction Assessment

Yes for the PR regression. Source inspection shows current main reads `result.continueAgent === true`, while this PR reads `shouldContinue`; a plugin using the shipped field would no longer continue if this patch merged.

## Solution Assessment

No. The early feature shape has been superseded by the shipped `continueAgent` API, so merging this branch would replace the current public field instead of preserving it.
