## Summary

Keep open: current `main` and latest `v2026.6.9` still route Mattermost public-channel turns through prompt prose that says `group chat` while trusted metadata can say `chat_type: "channel"`; several open PRs are candidate fixes, but none has merged yet.

## What This Changes

The issue requests that Mattermost channel prompt prose stop labeling `chat_type: "channel"` conversations as `group chat`.

## Best Possible Solution

Land one narrow PR that makes the shared group/channel prompt opener `ChatType`-aware or neutral, adds focused regression coverage, then leave the deeper private-channel target-inference work to https://github.com/openclaw/openclaw/issues/95646.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: Mattermost public channel type `O` maps to `channel`, `get-reply-run` routes `channel` through `buildGroupChatContext`, and that helper emits the `group chat` opener. I did not run a live Mattermost server or tests in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is to make the opener use the authoritative `ChatType` or neutral non-DM wording while preserving existing delivery guidance. The target-string inference problem should stay in https://github.com/openclaw/openclaw/issues/95646 unless maintainers intentionally combine the work.
