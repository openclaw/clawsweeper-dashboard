## Summary

Current main already contains a targeted fix path for this class of loopback gateway CLI failure: the browser command now registers through a dedicated lazy `cli-metadata` entry instead of loading the full browser runtime during CLI startup, and the browser status path is a lightweight `browser.request` call with explicit timeout coverage. The same code is present in the latest shipped release `v2026.4.22` at `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`.

## Evidence

- **Changelog calls out the startup regression class:** `CHANGELOG.md` explicitly says `openclaw devices list` and similar loopback gateway commands were fixed by isolating heavy import-time side effects from the normal CLI path.
  - file: [CHANGELOG.md:2569](https://github.com/openclaw/openclaw/blob/8fade9df27a9a25cd017db1154409b3ad9bd3741/CHANGELOG.md#L2569)
  - sha: [8fade9df27a9](https://github.com/openclaw/openclaw/commit/8fade9df27a9a25cd017db1154409b3ad9bd3741)
- **Browser CLI now uses a lazy cli-metadata entry:** The bundled browser plugin registers CLI commands from `cli-metadata.ts`, and only lazy-imports `runtime-api.js` when the `browser` command is actually used.
  - file: [extensions/browser/cli-metadata.ts:3](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/extensions/browser/cli-metadata.ts#L3)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Plugin loader executes cli-metadata separately from full runtime registration:** The plugin loader loads dedicated CLI metadata entries in `registrationMode: "cli-metadata"`, which is the mechanism that avoids full runtime side effects on CLI startup.
  - file: [src/plugins/loader.ts:3057](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/plugins/loader.ts#L3057)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **`openclaw browser status` now goes through a lightweight browser.request path:** Browser CLI requests are funneled through `callGatewayFromCli("browser.request", ...)` with normalized query/timeout handling instead of a heavier direct runtime path.
  - file: [extensions/browser/src/cli/browser-cli-shared.ts:30](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/extensions/browser/src/cli/browser-cli-shared.ts#L30)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Status command has explicit longer timeout and regression coverage:** `browser status` uses a built-in 45s timeout, and there is a dedicated regression test asserting that timeout for the status path.
  - file: [extensions/browser/src/cli/browser-cli-manage.ts:19](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/extensions/browser/src/cli/browser-cli-manage.ts#L19)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Related loopback fallback hardening shipped in the same CLI area:** Current `devices` CLI now falls back to local pairing state when loopback gateway calls fail with pairing-required errors, with regression tests covering the fallback behavior.
  - file: [src/cli/devices-cli.ts:122](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/cli/devices-cli.ts#L122)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
