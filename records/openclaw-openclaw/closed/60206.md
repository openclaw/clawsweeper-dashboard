## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Validate against Feishu emoji_type values before sending:** `extensions/feishu/src/typing.ts:125-131`
  - body: This gate treats any word string as Feishu-compatible, so global values like `eyes` or `party_parrot` pass through to `emoji_type` even though Feishu documents a fixed emoji_type set. Since non-backoff create failures are swallowed later, existing multi-channel configs can lose Feishu typing indicators instead of falling back to `Typing`.
  - confidence: 0.88
- **[P2] Preserve empty ackReaction as disabled:** `extensions/feishu/src/typing.ts:124-131`
  - body: `resolveAckReaction` uses an empty string as the shared disable signal, but this path converts `""` back into `Typing`. That makes `messages.ackReaction: ""` or `channels.feishu.accounts.<id>.ackReaction: ""` still send Feishu typing reactions instead of disabling them.
  - confidence: 0.9
- **[P3] Sync the Feishu plugin config metadata:** `extensions/feishu/src/config-schema.ts:189`
  - body: This adds a public Feishu config key in zod/docs, but `openclaw.plugin.json` still lists the top-level and account config schema without `ackReaction`. Setup, discovery, or config UI consumers that read plugin metadata will not see the new option until the manifest schema is updated too.
  - confidence: 0.82
