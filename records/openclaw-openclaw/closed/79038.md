## Summary

Keep open: current main still accepts and forwards a caller-supplied Webhooks `run_task.childSessionKey` without a route-session-tree guard, and the issue is already paired with an open closing PR for the narrow fix.

## What This Changes

The issue requests that Webhooks `run_task.childSessionKey` be accepted only for the route-bound session or recorded child sessions in that route session tree.

## Best Possible Solution

Review and land or supersede https://github.com/openclaw/openclaw/pull/79041, then close this issue after the Webhooks guard is merged on main.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a route-bound Webhooks `run_task` can pass an unrelated `childSessionKey`, TaskFlow stores it, and `cancel_flow` later reaches cancellation through that stored child session key.

## Solution Assessment

Yes. The narrow maintainable fix is a Webhooks-boundary guard that accepts the bound session or recorded spawned/parented child sessions before calling `runTask`; the open linked PR follows that direction.
