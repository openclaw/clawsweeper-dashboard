## Summary

Keep open: this is a focused bug-fix PR and current main still disables the root-alias native path on Windows. The code direction looks right, with one non-blocking guardrail issue: the PR deletes the explicit Windows contract case instead of changing it to expect the new native path.

## What This Changes

The PR removes the Windows-only `tryNative` override from `src/plugin-sdk/root-alias.cjs` and deletes the old Windows root-alias contract case that expected native loading to be disabled.

## Best Possible Solution

Land the narrow loader change after restoring the Windows-specific contract row with `expectedTryNative: true`; keep the broader Windows startup policy aligned with https://github.com/openclaw/openclaw/pull/74173.

## Reproduction Assessment

Yes, at source level: current main maps `getModuleLoader(true)` to `false` on `win32`, and the existing contract test asserts that old behavior for production/dist Windows. I did not execute a Windows runtime in this read-only review.

## Solution Assessment

Yes for the code direction: passing `tryNative` through is the narrow owner-boundary fix and matches the adjacent loader policy. The test change should preserve the explicit win32 case and flip it to the new expected behavior.
