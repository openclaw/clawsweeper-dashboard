## Summary

Close as superseded: current main still has the source-level Slack cron delivery regression, but the exact shared outbound-target bootstrap change requested here is already tracked by the open pull request at https://github.com/openclaw/openclaw/pull/78809 and its canonical resolver issue at https://github.com/openclaw/openclaw/issues/77712.

## What This Changes

The issue reports that OpenClaw 2026.5.12 isolated cron announce deliveries to Slack fail with `Unsupported channel: slack` because target resolution does not bootstrap the externalized non-startup Slack plugin.

## Best Possible Solution

Land the existing generic outbound target bootstrap fix, add Slack/5.12 isolated-cron proof if feasible, and keep this report linked from the canonical resolver thread so the Slack-specific regression is not lost.

## Reproduction Assessment

Yes, source-level: current main falls from isolated cron target resolution into `resolveOutboundTarget` without bootstrap, while Slack has `activation.onStartup: false`; a cold Slack plugin can therefore produce `Unsupported channel: slack`. I did not run a live Slack/Homebrew cron job in this read-only pass.

## Solution Assessment

Yes. The narrow maintainable fix is the generic `allowBootstrap: true` resolver change already open at https://github.com/openclaw/openclaw/pull/78809, not forcing Slack back to unconditional startup loading.
