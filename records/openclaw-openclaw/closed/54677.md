## Summary

Keep this PR open. Current main still lacks the proposed Hetzner warning, while the repository default compose file still publishes the gateway and bridge ports without a loopback host binding. Docker Compose merge behavior supports the reported footgun, and the proposed diff is docs-only with no security or supply-chain regression found.

## Best Possible Solution

Keep the PR open for maintainer review and merge or fold the same warning into the Hetzner guide. The best product/docs outcome is a visible callout near the Hetzner compose snippet explaining that operators adapting the repository default compose file must replace the `ports:` list, not layer a naive override that leaves public gateway or bridge bindings in place.
