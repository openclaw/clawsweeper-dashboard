## Summary

Current `main` no longer forwards raw `Codex error: {...}` JSON blobs to channels. The shared formatter parses provider error payloads into summarized text, and the reply normalization path sanitizes error payloads before external delivery. The fix is on `main` in commit `c6684af682bcb59bd1ee85cb49f72d4074007e2c`, but I could not confirm a tagged release that contains it.
