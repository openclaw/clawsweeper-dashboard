## Summary

Close as implemented on main: current main addresses the runtime-deps mirror ENOENT window that can make a transient plugin register failure look permanent, and the proof commit is present only on main after v2026.4.27.

## What This Changes

The issue asks OpenClaw to prevent bundled plugin runtime-deps ENOENT races during hot channel reloads and to avoid leaving Telegram outbound permanently unavailable after a transient failed register.

## Best Possible Solution

Keep the current atomic, locked runtime-deps mirror implementation and ship it in the next release; only reopen with fresh logs if a non-mirror failed-register state can still permanently block outbound after this fix.
