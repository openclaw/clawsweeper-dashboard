## Summary

Keep open. Current main still has the reported Discord outbound mention reliability gap: outbound sends run through the known-mention rewriter, but that rewriter only resolves account-scoped in-memory directory cache entries and preserves unknown handles as plaintext.

## What This Changes

Request deterministic Discord outbound mention rewriting so configured known `@Name` references become real `<@userId>` mentions instead of cache-dependent plaintext.

## Best Possible Solution

Implement a Discord-plugin-local configured alias-to-userId map that resolves before the transient directory cache, preserves reserved/unknown mentions, and is documented with focused tests across outbound send modes.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction exists: with an empty Discord directory cache, `rewriteDiscordKnownMentions("@Vladislava", { accountId })` returns plaintext, and every inspected outbound Discord send path depends on that helper.

## Solution Assessment

Yes. A plugin-local deterministic alias map is the narrowest maintainable direction because it fixes known teammate mentions without moving Discord-specific policy into core or changing unknown-handle behavior.
