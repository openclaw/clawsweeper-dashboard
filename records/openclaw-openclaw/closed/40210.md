## Summary

Keep this open. Current `main` still rejects bootstrap-file symlinks that resolve outside the workspace boundary and renders the failed reads as missing entries, while open implementation candidates are still pending maintainer review.

## What This Changes

The issue requests that recognized workspace bootstrap files load normally when they are symlinks to readable files outside the workspace.

## Best Possible Solution

Review and land one canonical narrow fix, likely in the workspace bootstrap reader layer, that allows recognized fixed bootstrap files to follow readable external symlink targets while preserving the existing guardrails and leaving extra/glob bootstrap paths boundary-bound. After that fix ships, close or consolidate #38622 and #64472 based on the final scope.
