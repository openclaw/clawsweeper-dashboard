## Summary

Current main and the latest release now cover the reported WebChat blanking path: assistant replies are preserved in durable session repair, normal agent-run transcript ownership is documented, and Control UI history reloads no longer replace visible optimistic/backscroll state in the stale-history cases that made older assistant bubbles disappear.

## What This Changes

The issue reports that switching the Control UI WebChat model from the chat header makes earlier assistant replies render as blank bubbles in v2026.5.2.

## Best Possible Solution

Keep the shipped transcript-ownership and Control UI history-reconciliation fixes, and treat any remaining post-upgrade blank transcript data as a new recovery report with raw history/JSONL evidence.

## Reproduction Assessment

Yes for the affected v2026.5.2 family: this issue plus related reports describe the disappearing-assistant path, and current source/tests show the stale-history and transcript-preservation fixes. I did not run a live browser reproduction against current main.

## Solution Assessment

Yes. The shipped fix is the maintainable boundary: preserve Pi-owned assistant transcript turns and prevent stale Control UI history reloads from replacing visible local chat state, rather than adding a model-switch-specific history reload workaround.
