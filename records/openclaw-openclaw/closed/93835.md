## Summary

Keep open: current main still resolves native `/think` menu choices without the runtime model catalog, so live-discovered Ollama reasoning metadata can be lost. The issue is already paired with open candidate PRs, and https://github.com/openclaw/openclaw/pull/94067 remains the strongest fix path because it supplies runtime catalog metadata to the native menu surfaces instead of broadening Ollama's unknown-reasoning policy.

## What This Changes

The issue reports that Telegram's bare `/think` menu shows only `default, off` for live-discovered Ollama thinking models even though `/think <level>` accepts supported levels.

## Best Possible Solution

Land or replace the runtime-catalog native `/think` menu fix, covering Telegram and the sibling Slack/Discord native menu surfaces while preserving explicit Ollama `reasoning:false` behavior.

## Reproduction Assessment

Yes. Source inspection shows the menu resolver uses a configured-only catalog unless a runtime catalog is supplied, while Ollama live discovery records `thinking` capability as `reasoning: true`.

## Solution Assessment

Yes. Passing the runtime catalog into native `/think` menu/autocomplete resolution is the narrowest maintainable fix; provider-wide undefined-reasoning fallbacks risk advertising thinking levels for unknown non-reasoning Ollama models.
