## Summary

Close. Current main already isolates channel/plugin startup failures from gateway availability, so a Discord startup problem should log a warning instead of closing the gateway, and the same implementation is present in the shipped `v2026.4.22` release commit `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`.
