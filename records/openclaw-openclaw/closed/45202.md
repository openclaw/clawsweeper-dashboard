## Summary

Current main does not match the reported behavior: gateway startup iterates every configured Telegram account, the Telegram plugin enumerates all configured account IDs, and current docs/doctor warnings explain that missing `defaultAccount` or missing account-scoped bindings can make multi-account setups look like only the default account is active even though per-account state still exists.

## Evidence

- **Gateway starts every listed account:** `startChannelInternal` loads `plugin.config.listAccountIds(cfg)` and runs `startAccount(...)` for each ID via `Promise.all`, so startup is not limited to `default`.
  - file: [src/gateway/server-channels.ts:299](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/src/gateway/server-channels.ts#L299)

  - sha: [4630ce3d9e2f](https://github.com/openclaw/openclaw/commit/4630ce3d9e2f089062ccd386b0ae241325961d1a)
- **Telegram account enumeration includes all configured accounts:** `listTelegramAccountIds` collects every key from `channels.telegram.accounts` and combines them with bound account IDs before startup/routing.
  - file: [extensions/telegram/src/account-selection.ts:30](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/extensions/telegram/src/account-selection.ts#L30)

- **Per-account startup log exists for each Telegram account:** Telegram `startAccount` logs `[{accountId}] starting provider...` before handing control to the monitor, so non-default accounts should appear separately when they are configured and started.
  - file: [extensions/telegram/src/channel.ts:866](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/extensions/telegram/src/channel.ts#L866)

  - sha: [4630ce3d9e2f](https://github.com/openclaw/openclaw/commit/4630ce3d9e2f089062ccd386b0ae241325961d1a)
- **Status summary only exposes the default account at top level:** Runtime snapshots store all Telegram accounts under `channelAccounts[plugin.id]`, but the top-level `channels[plugin.id]` entry is only the resolved default account. That can look like 'only one Telegram provider running' if the caller inspects only the summary view.
  - file: [src/gateway/server-channels.ts:645](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/src/gateway/server-channels.ts#L645)

  - file: [docs/channels/telegram.md:909](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/docs/channels/telegram.md#L909)
