## Summary

The reported `startsWith` crash path existed in `v2026.4.15`, but current `main` hardens `src/logging/subsystem.ts` against malformed subsystem/message values and adds regression tests. The merged fix landed in PR #70535 (`a63939d29592dcf803dde5721740a50955191ba9`) and is present in release `v2026.4.23`.

## Evidence

- **Old release had the unsafe `subsystem.startsWith` calls:** In `v2026.4.15`, `shouldSuppressProbeConsoleLine()` used `params.subsystem.startsWith(...)` directly, so a missing or malformed subsystem label could throw the exact `startsWith` TypeError reported here.
  - file: [src/logging/subsystem.ts:272](https://github.com/openclaw/openclaw/blob/041266a6699cac3baef8ef39db41fa26f29f9db3/src/logging/subsystem.ts#L272)
