## Summary

Keep this issue open: current main no longer reproduces the reported `xdg-open` branch because MS Teams delegated OAuth setup now uses a manual OAuth path, but #67660 remains an open same-author closing PR and maintainers still need to resolve whether the requested automatic Windows launcher is superseded or should be restored.

## What This Changes

The issue asks MS Teams delegated OAuth setup to stop selecting `xdg-open` on Windows and align the launcher with the repository's Windows browser-open behavior.

## Best Possible Solution

Resolve #67660 alongside this issue: either close it as superseded by the current manual delegated OAuth setup, or adapt it if maintainers still want automatic Windows browser launch for MS Teams setup.

## Reproduction Assessment

No. On current main, the reported `win32` reproduction no longer reaches `xdg-open`; setup forces manual OAuth and the exported opener rejects with a manual-open message instead of spawning a command.

## Solution Assessment

Unclear. The current manual flow is a narrow way to avoid wrong platform spawns, but whether it is preferable to #67660's automatic `explorer.exe` launcher is a maintainer behavior decision.
