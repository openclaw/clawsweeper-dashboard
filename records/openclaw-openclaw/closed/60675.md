## Summary

Current main covers the reported 2026.3.22 failure mode: browser/Gateway `chat.send` enters the normal reply runtime, the runtime resolves the configured agent workspace instead of bootstrapping an ad hoc home path, and the Docker image now pre-creates `/home/node/.openclaw` before switching to the non-root `node` user. The fix is present in the latest release tag `v2026.4.26`.

## Best Possible Solution

Keep the shipped Docker home pre-creation and configured-workspace runtime path. If a similar failure appears on a newer release, it should be reopened as a fresh report with the exact release, container image digest, `agents.defaults.workspace`, session key, and sanitized Gateway runtime logs around the `chat.send` run.
