## Summary

Keep open for normal review and merge. Current `main` rejects the documented xAI `pending` video poll status, while this PR fixes the parser boundary narrowly, adds a focused regression test, and includes sufficient after-fix live-output proof.

## What This Changes

The PR normalizes xAI video poll statuses `pending` and `in_progress` to `processing`, adds a pending-to-done regression test, and records the fix in `CHANGELOG.md`.

## Best Possible Solution

Merge the narrow xAI parser fix after required checks, preserving strict rejection for other unknown statuses and keeping the pending-to-done regression test.

## Reproduction Assessment

Yes. A current-main poll JSON with `status: "pending"` reaches `readXaiStatusResponse` and throws before `pollXaiVideo` can continue; xAI’s docs also describe `pending` as an in-flight video status. ([docs.x.ai](https://docs.x.ai/developers/rest-api-reference/inference/videos))

## Solution Assessment

Yes. Normalizing vendor in-flight statuses at the xAI response parser boundary is the narrowest maintainable fix because the downstream polling loop already handles `processing`.
