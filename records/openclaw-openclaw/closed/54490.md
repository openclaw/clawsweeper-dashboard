## Summary

Close as ClawHub/plugin work. Current main already exposes supported plugin/runtime surfaces for background subagent orchestration, task tracking, and plugin-owned CLI/runtime integration, so this PR’s bespoke `--mode=worker` core entry point is not the right core change. The PR-mentioned `openclaw-teams` style capability should live as an external plugin on ClawHub/npm unless a concrete missing plugin API is identified.

## Best Possible Solution

Do not merge this bespoke core CLI worker mode. Keep `api.runtime.subagent`, `sessions_spawn`, hooks, and the background task ledger as the supported extension points; publish `openclaw-teams`-style orchestration as a ClawHub/npm plugin, and open a focused core API proposal only if those shipped surfaces cannot express a specific requirement.
