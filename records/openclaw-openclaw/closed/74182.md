## Summary

Keep open. Current main and the latest release still have the run-scoped embedded-run abort wrapper and timeout abort path that match the suspected retention surface, and open draft PR #75008 is the active unmerged implementation candidate rather than a shipped fix.

## What This Changes

The issue asks OpenClaw Gateway to release embedded LLM run state after timeout or abort so repeated embedded-run failovers do not cause unbounded RSS growth.

## Best Possible Solution

Review and land PR #75008 or an equivalent replacement only after it extracts the abort wrapper and adds deterministic proof that hung provider aborts release run scope, `ACTIVE_EMBEDDED_RUNS` drains, and the remaining waitForIdle/provider stream/subagent announce surfaces do not retain multi-GB state.

## Reproduction Assessment

Unclear. The report gives a credible live reproduction over several hours with provider instability and RSS growth, but current main lacks a deterministic heap/RSS harness; source inspection still shows a plausible retained-promise path.

## Solution Assessment

Yes for keeping this open. The narrow maintainable path is to review the active draft PR and require targeted memory/drain proof before treating the issue as fixed, rather than closing from registry cleanup alone.
