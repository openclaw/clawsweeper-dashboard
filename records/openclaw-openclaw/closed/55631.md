## Summary

Current `main` already prevents heartbeat/cron/exec system events from resetting stale sessions, which covers the reported main-session loss after QQ Bot reconnects.

## Evidence

- **System events no longer participate in reset/binding retargeting:** `initSessionState()` explicitly treats `heartbeat`, `cron-event`, and `exec-event` as system events and skips conversation-binding retargeting for them.
  - file: [src/auto-reply/reply/session.ts:230](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/src/auto-reply/reply/session.ts#L230)

  - sha: [4630ce3d9e2f](https://github.com/openclaw/openclaw/commit/4630ce3d9e2f089062ccd386b0ae241325961d1a)
- **Heartbeat/system events are forced fresh instead of expiring into a reset:** When an existing session entry is present, `isSystemEvent` bypasses `evaluateSessionFreshness()` and marks the entry fresh, preventing heartbeat-driven idle/daily rollover.
  - file: [src/auto-reply/reply/session.ts:431](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/src/auto-reply/reply/session.ts#L431)
