## Label Justifications

- `P1`: The report describes a real Telegram channel workflow stall where inbound messages stop processing until restart.

## Summary

Current main and the latest release already cover the reported Telegram isolated-ingress stall: the main-thread bot is initialized before draining spooled updates, and drain handling no longer lets one failed lane stop unrelated queued updates.

## What This Changes

The issue reports Telegram isolated polling getting stuck on a `Bot not initialized` spooled update, blocking later inbound Telegram messages until gateway restart.

## Best Possible Solution

Keep the v2026.5.19+ Telegram isolated-ingress implementation; if the symptom appears on that release or newer, it should be filed as a fresh report with current logs and version details.

## Reproduction Assessment

No high-confidence current-main failure path was reproduced. Source and tests show the reported 2026.5.12 path is now covered by initializing the bot before drain and by per-lane spool processing.

## Solution Assessment

Yes. The current implementation fixes the initialization race directly and preserves per-lane Telegram ordering while allowing unrelated lanes to continue, which is narrower than deleting arbitrary failed updates.
