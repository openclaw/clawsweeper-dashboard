## Summary

Keep this PR open. Current main still has the cancellation gap the PR is intended to fix: the Ollama stream wrapper passes the caller abort signal only through `init.signal`, while `fetchWithSsrFGuard` composes timeout cancellation from its top-level `signal` and then overwrites the request init signal. The PR is a focused implementation candidate, not obsolete or superseded.

## Best Possible Solution

Review and land this focused PR, or an equivalent narrow patch, so Ollama passes `options.signal` as the top-level `fetchWithSsrFGuard` signal whenever timeout handling is active, with the added regression test kept in `extensions/ollama/src/stream-runtime.test.ts`. No docs or changelog change looks necessary for this internal cancellation regression.
