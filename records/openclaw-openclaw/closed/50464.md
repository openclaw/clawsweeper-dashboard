## Summary

The report describes hosted ClawHub account/client block state after account deletion and re-linking. Current OpenClaw main only consumes ClawHub as an external registry for skill/plugin search, install, update, and package download; account login/delete/unblock workflows are handled by the separate ClawHub service/CLI and are not fixable by changing this repository.

## Evidence

- **Issue scope is hosted account recovery:** #50464 reports deleting a ClawHub account, signing in again with the same account, and receiving a "client block" response. The follow-up comment also identifies this as retained ClawHub client ID/account state and asks for server-side association cleanup, cooldown, or manual unblock.
- **OpenClaw docs separate native repo flows from ClawHub account/auth workflows:** The ClawHub guide says native `openclaw` commands search/install/update skills and install plugins, while registry auth, publish, delete, undelete, and sync workflows use the separate `clawhub` CLI.
