## Summary

Close as duplicate/superseded by #50040. Current main already carries explicit Telegram forum-topic routing for `message(action=send, threadId=...)`; the unresolved symptom is the generic intermittent Telegram `sendMessage` network failure after idle/recovery windows, which #50040 tracks as the open canonical Telegram outbound delivery reliability issue. Open PR #48537 is also directly related to one likely root cause: polling restart aborting non-polling Telegram API sends.

## Best Possible Solution

Close this as duplicate/superseded by #50040. Keep the remaining work on the canonical Telegram outbound reliability tracker: preserve account/chat/topic/reply metadata, classify safe versus ambiguous send failures, add durable retry/drain behavior for eligible sends, and review/refresh #48537 as the narrower polling-restart abort fix if it still applies to current `bot-core.ts`.
