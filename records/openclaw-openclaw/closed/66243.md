## Summary

Close as implemented on main: v2026.5.2 contains the narrower shared FormData normalization and regression coverage that solve the proxy multipart failure, while the PR branch's Request-like flattening is unnecessary for real Request inputs and still carries proxy-normalization risks.

## What This Changes

The PR branch changes shared proxy fetch code and tests to convert global FormData to undici FormData, flatten Request or Request-like inputs, strip selected hop-by-hop headers, and preserve signal/header override semantics.

## Best Possible Solution

Keep the released v2026.5.2 shared FormData normalization and the merged Discord multipart coverage; avoid adding broad Request-like duck typing unless a concrete caller and security design justify it.

## Reproduction Assessment

Yes for the original FormData proxy failure: the PR discussion includes a production Discord repro, current tests encode the conversion behavior, and undici source shows multipart serialization depends on undici-recognized FormData. The Request-flattening claim is not reproduced, and undici v8.1.0 already accepts real Request inputs.

## Solution Assessment

No for the PR branch as written; yes for the current-main outcome. The shipped fix is narrower, preserves current env proxy option handling, and relies on undici's Request contract instead of adding unsafe Request-like duck typing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Require own request-like URL properties:** `src/infra/net/proxy-fetch.ts:60`
  - body: The request-like guard uses `"url" in input`, so an inherited `url` property can make an arbitrary object look like a proxy request and steer the outbound URL. Require an own property or remove the duck-typed Request-like path and let undici handle real Request objects.
  - confidence: 0.9
- **[P2] Preserve current env proxy option handling:** `src/infra/net/proxy-fetch.ts:280-284`
  - body: Current main builds `EnvHttpProxyAgent` with resolved env options, including ALL_PROXY fallback. This branch keeps the older HTTPS-only/default-constructor path, so rebasing it as-is would regress current env proxy behavior.
  - confidence: 0.84
- **[P2] Strip Proxy-Connection with hop-by-hop headers:** `src/infra/net/proxy-fetch.ts:69-80`
  - body: The new strip set removes several hop-by-hop/framing headers but still forwards `Proxy-Connection`. Add it to the sanitizer, and consider removing headers nominated by the `Connection` header before forwarding through the proxy path.
  - confidence: 0.8
