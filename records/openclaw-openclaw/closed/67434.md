## Summary

Keep open. ## What This Changes

The issue asks OpenClaw to stop long GPT-5/Codex strict-agentic runs from spending many consecutive turns on `update_plan` before returning to real tool use.

## Best Possible Solution

Land or revise the focused fix path in https://github.com/openclaw/openclaw/pull/68673 so no-progress `update_plan` churn is collapsed or capped under strict-agentic runs while preserving the documented `planTool: false` opt-out.
