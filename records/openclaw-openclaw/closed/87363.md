## Summary

Keep this issue open: current main and the latest release still allow the source-level Slack draft cleanup path described here, and the linked fixing PR is open but unmerged.

## What This Changes

The issue reports that Slack can delete an already visible final reply when a late same-turn final/error payload reuses the live draft cleanup path.

## Best Possible Solution

Review and land the linked Slack fix PR if it passes code review and real Slack or focused regression proof; then close this issue as implemented.

## Reproduction Assessment

Yes, at source level: normal Slack final delivery can leave draftPreviewCommitted false, and a later final/error payload can still receive a draft adapter whose fallback clears the Slack draft. I did not run a live Slack replay in this read-only review.

## Solution Assessment

Yes, the linked PR's direction is the narrow maintainable fix: track visible final delivery independently of preview finalization and stop passing the draft adapter to later same-turn finals after that point.
