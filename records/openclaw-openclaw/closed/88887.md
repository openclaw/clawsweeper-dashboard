## Summary

This PR is still necessary and appears to be the right narrow fix: current main still lets QMD's unified MCP `query` default reranking run for `search`/`vsearch`, while the branch disables it only for those reduced modes and adds focused regression coverage.

## What This Changes

Adds `rerank:false` to memory-core's QMD mcporter unified `query` payload for `search` and `vsearch` modes, plus a focused `vsearch` call-args regression test.

## Best Possible Solution

Land this focused patch, or an equivalent maintainer-owned patch, so QMD mcporter `search` and `vsearch` preserve lexical/vector-only behavior while full `query` mode keeps reranking.

## Reproduction Assessment

Yes. Current main builds QMD unified `query` call arguments without `rerank:false` for `search` and `vsearch`, while upstream QMD defaults `rerank` to true; the contributor also supplied live before/after terminal proof.

## Solution Assessment

Yes. Setting `rerank:false` at the unified mcporter call site for reduced search modes is the narrowest maintainable fix; direct CLI modes already use dedicated commands and full `query` mode keeps its existing rerank behavior.
