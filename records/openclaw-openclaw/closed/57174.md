## Summary

This issue is still a valid core feature request, but the remaining implementation is now tracked by the canonical open PR #69988. Current main still lacks per-agent compaction support: agent entries do not accept a compaction field, resolved agent config does not merge one, docs expose defaults-only configuration, and runtime compaction paths still read agents.defaults.compaction. Close this issue as superseded, not implemented.

## Best Possible Solution

Close this issue as superseded by #69988 and keep review focused there. The best implementation is still a core config/runtime change: add agents.list[].compaction to the type and strict schema, merge it over agents.defaults.compaction, thread the effective agent-specific config through Pi settings, safeguard runtime, compaction timeout/model/provider/post-compaction paths, update generated config docs/help, and add focused schema/resolver/runtime tests.
