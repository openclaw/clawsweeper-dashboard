## Summary

Close PR #62349 as implemented on current main. The PR targeted a real stale subsystem file-logger rollover bug, but main now solves the observable problem with a broader implementation: subsystem file writes no longer retain a forever-cached child logger, the root file transport resolves the active rolling log file at write time, and regression tests cover long-lived subsystem loggers crossing a date boundary.

## Best Possible Solution

Close PR #62349 as implemented on current main. Keep the broader main implementation that avoids stale subsystem child loggers and makes rolling-file writes current-date aware; no code from the generation-counter PR is needed unless maintainers later want a separate performance optimization.
