## Summary

Keep open. Current main still has the Docker-daemon-unavailable success path this PR targets, but the PR is not ready to merge because its browser regression test bypasses the earlier Docker network preflight that would fail before the new image-inspection error is reached.

## What This Changes

The PR changes Docker and sandbox browser image preflight to throw formatted Docker-daemon-unavailable errors and adds focused regression tests for daemon-unavailable image inspection and sandbox.mode off cron/sub-agent sessions.

## Best Possible Solution

Update the PR so Docker-daemon-unavailable handling is consistent across image inspection and browser network setup, with regression coverage proving disabled sandbox sessions never initialize Docker and enabled Docker/browser sandboxes fail with actionable daemon-unavailable messages.
