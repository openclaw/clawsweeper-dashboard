## Summary

Keep open. Current main and the latest shipped release still drop DNS hostnames before building Comfy's private-network SSRF policy, and the linked fix PR was closed unmerged.

## What This Changes

Allow the Comfy plugin in local/private-network mode to use configured DNS `baseUrl` hostnames that resolve to LAN/private IPs.

## Best Possible Solution

Patch the Comfy-owned network policy to trust only the configured local/private `baseUrl` origin or hostname for private DNS answers, with focused regression coverage and maintainer security review.

## Reproduction Assessment

Yes. Source inspection on current main shows local mode sets `allowPrivateNetwork`, but a DNS `baseUrl` fails the `isPrivateOrLoopbackHost` gate and then the shared SSRF resolver blocks private DNS answers.

## Solution Assessment

Yes, the fix should stay in the Comfy policy builder rather than changing core SSRF defaults. The safest current direction is an origin- or host-scoped policy for the configured `baseUrl`, preserving redirect and metadata protections.
