## Summary

Keep open. Current main still has the reported Comfy local-mode gap: the plugin opts local mode into private-network access, but then drops the SSRF policy for DNS hostnames because the host string is not already a private/loopback literal.

## What This Changes

The issue requests allowing Comfy local-mode `baseUrl` values that are DNS hostnames resolving to private/LAN IPs when `allowPrivateNetwork` is enabled.

## Best Possible Solution

Update the Comfy-owned network policy so local mode or explicit `allowPrivateNetwork` builds a hostname-scoped private-network policy for the configured `baseUrl`, with regression tests for DNS hostnames resolving to private IPs and no core-wide SSRF default change.

## Reproduction Assessment

Yes, source-reproducible on current main: use Comfy local mode with a DNS `baseUrl` resolving to RFC1918, and the policy builder returns `{}` before strict fetch blocks the resolved private IP. I did not run the live LAN setup in this read-only review.

## Solution Assessment

Yes, the best direction is a narrow plugin-side policy fix that keeps the configured hostname allowlist while allowing the opted-in private DNS answer. A shared SSRF guard change or new config knob would be broader than needed.
