## What This Changes

The issue reports that `sessions_spawn({ runtime: "acp", agentId: "claude", runTimeoutSeconds })` fails because OpenClaw sends `session/set_config_option` with `configId: "timeout"` to Claude Code.

## Best Possible Solution

Keep one canonical lane: land a fix that treats `runTimeoutSeconds` as an OpenClaw manager-owned timeout and stops forwarding unsupported `timeout` config to Claude ACP.

## Reproduction Assessment

Yes; source inspection on current main traces `runTimeoutSeconds` into a forwarded ACP `timeout` config option, and the locked Claude ACP dependency rejects unknown config IDs. I did not run a live Claude Code smoke in this read-only cleanup pass.
