## Summary

Current `main` already fixes the underlying gateway-registry churn that this PR targets. Instead of guarding `activatePluginRegistry`, OpenClaw now pins the gateway startup HTTP-route registry and resolves plugin route auth/dispatch from that pinned registry, with changelog and regression-test coverage shipped in `v2026.3.22`.

## Evidence

- **Changelog shipped fix:** `CHANGELOG.md` for release `2026.3.22` explicitly says: gateway/plugins pin runtime webhook routes to the gateway startup registry so channel webhooks keep working across plugin-registry churn, and plugin auth + dispatch resolve routes from the same live HTTP-route registry. Entry references PR `#47902` and fixes `#46924` and `#47041`.
