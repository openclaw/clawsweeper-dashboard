## Summary

Current main already prevents this failure mode: `config.apply` and `config.patch` validate and return before any disk write on invalid payloads, `writeConfigFile` re-validates and rejects destructive clobbers, and the actual persistence path is temp-file then rename. Adjacent regression coverage also checks that a failed `config.apply` leaves the config hash/raw unchanged.
