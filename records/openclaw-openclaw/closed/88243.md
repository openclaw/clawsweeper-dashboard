## Summary

Keep open: the latest shipped Bonjour handler already suppresses known ciao failures, but the reported current-release SIGTERM loop still exposes a real diagnostics gap because SIGTERM logging remains generic and diagnostics export does not surface effective mDNS state or recent Bonjour events.

## What This Changes

The issue reports a macOS LaunchAgent SIGTERM restart loop after a physical network migration where an apparent Bonjour/mDNS failure is indistinguishable from an external stop in logs and diagnostics.

## Best Possible Solution

Add generic Bonjour/mDNS diagnostics and attribution: record effective mDNS enablement plus recent Bonjour recovery/failure events in support exports, and correlate nearby internal Bonjour failures with shutdown logs without masking real external SIGTERM stops.

## Reproduction Assessment

No high-confidence physical-network reproduction was run. Source inspection does reproduce the diagnostic gap: current main logs SIGTERM generically and the support export lacks effective mDNS state or recent Bonjour event fields.

## Solution Assessment

Unclear as a one-step fix. The maintainable path is not to hardcode this one report, but to add generic Bonjour diagnostic events/state and only correlate SIGTERM with recent internal mDNS failures when that evidence exists.
