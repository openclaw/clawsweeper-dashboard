## Summary

Keep open: the shared guard direction is sound, but the latest branch is not merge-ready because it documents snapshot coverage while the `/snapshot` route still has ungated collection paths; the protected `maintainer` label also means this PR needs maintainer handling rather than cleanup closure.

## What This Changes

The branch adds an opt-in current-tab URL gate to browser route helpers, enables it for several browser debug/export/read routes, expands route tests, and adds a changelog entry.

## Best Possible Solution

Keep the shared opt-in guard, add the same pre-collection gate and regression coverage to `/snapshot`, and keep the changelog scoped to the routes that are actually guarded.

## Reproduction Assessment

Yes. Source inspection on current `main` shows read/export handlers can collect from an already-selected tab before response-only URL redaction, and PR-head inspection shows `/snapshot` still has ungated collection paths despite the new coverage claim.

## Solution Assessment

No, not yet. The shared helper is the right maintainable direction, but the branch should either guard `/snapshot` before collection and test it or remove snapshot from the documented fixed surface.
