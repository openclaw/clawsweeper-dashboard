## Summary

Needs follow-up rather than cleanup close: live GitHub state shows this PR has been merged into main, but the merged code still leaves `GET /snapshot` outside the new current-tab route gate while the changelog claims snapshot coverage.

## What This Changes

The PR adds an opt-in current-tab URL policy gate to browser route helpers, enables it for several browser debug/export/read routes, extends route tests and harness fixtures, and adds a changelog entry.

## Best Possible Solution

Apply the shared current-tab execution gate to `GET /snapshot` before any snapshot collection or CDP fallback, and add HTTP-level regression coverage for a blocked current-tab snapshot.
