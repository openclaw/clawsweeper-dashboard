## Summary

Keep this PR open. Current main still contains the Docker troubleshooting command that passes `--url` without explicit auth, while the runtime code and tests still require explicit `--token` or `--password` for CLI URL overrides. The PR remains a focused docs fix, not implemented or superseded.

## Best Possible Solution

Keep the PR open for normal maintainer review or apply an equivalent current-main docs fix in `docs/install/docker.md`: update the Docker troubleshooting `devices list --url` example to pass explicit gateway credentials and add a short note that URL overrides do not reuse implicit config or environment credentials. Treat related CLI hint work as separate from this Docker docs fix.
