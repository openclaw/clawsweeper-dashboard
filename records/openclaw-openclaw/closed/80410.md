## Summary

Close as duplicate/superseded: the linked warning bug is already covered by an open classifier-level fix, and this branch's payload-only suppression does not hit the current-main path where `config.schema.lookup` is still marked mutating. The branch also bundles unrelated gateway provider-attribution API changes without proof.

## What This Changes

This PR adds a payload-level suppression for gateway lookup path-not-found warnings and also includes unrelated embedded-runner usage and `chat.inject` provider-attribution changes.

## Best Possible Solution

Review and land the classifier-level fix in https://github.com/openclaw/openclaw/pull/42898 or a successor, keeping provider-attribution API work separate.

## Reproduction Assessment

Yes, at source level. Current main returns `config schema path not found` for a missing `config.schema.lookup` path, classifies the compound gateway action as mutating, and then renders mutating tool errors as channel-visible warnings.

## Solution Assessment

No. The suppression added here only runs when `lastToolError.mutatingAction` is false, but current main sets it true for the reported call; the maintainable fix belongs in the mutation classifier, as in https://github.com/openclaw/openclaw/pull/42898.

  - body: This branch only suppresses the warning when `lastToolError.mutatingAction` is false, but the reported `gateway` call is currently classified as mutating because `config.schema.lookup` is compared as a full compound action. As a result, the linked bug still warns on current main; fix the gateway action classifier or carry the read-only action facts into this policy.
  - confidence: 0.91

  - confidence: 0.82
