## Summary

Keep open. Current main still has the reported behaviors this PR is trying to address, but the patch should not merge as-is: the OpenRouter thinking profile drops an existing supported level, the cron fallback still lets error payload text win before the final assistant text, and the tool allowlist change conflicts with the existing disabled-tools contract test.

## What This Changes

This PR removes `--skip-trust` from the Google Gemini CLI backend defaults, adds an OpenRouter `xhigh` thinking profile, guards undefined plugin command results, changes cron summary fallback text selection, and skips empty-tool-allowlist errors when tools are disabled.

## Best Possible Solution

Revise the PR before merge: keep the straightforward Gemini CLI and plugin-command guard direction, preserve the existing OpenRouter thinking levels while adding `xhigh`, make cron prefer final assistant text over textual error fallbacks for the reported self-correction case, and either narrow or deliberately update the disabled-tools allowlist contract with tests.

## Reproduction Assessment

Yes for the main reported code paths by static inspection: current main still has the Gemini flag, the plugin command undefined-result crash path, the cron error-payload fallback behavior, and the disabled-tools allowlist guard. Live Gemini/OpenRouter reproduction was not run in this read-only sweep.

## Solution Assessment

No, this PR is not the best fix as written. The safer path is to keep each targeted fix narrowly aligned with the existing contracts and add or update focused tests for the OpenRouter thinking menu, cron all-error-payload case, plugin undefined-result boundary, and LLM-only allowlist behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve the minimal thinking level:** `extensions/openrouter/index.ts:163-167`
  - body: This custom OpenRouter profile replaces the base reasoning profile but omits `minimal`. OpenClaw currently treats `minimal` as part of the default `off` through `high` set, so after this change `/think minimal` and stored `minimal` settings would be rejected or remapped for OpenRouter reasoning models. Add `minimal` back or append `xhigh` to the existing base profile instead of replacing it.
  - confidence: 0.93
- **[P2] Prefer final assistant text before error payloads:** `src/cron/isolated-agent/helpers.ts:265-269`
  - body: `pickSummaryFromPayloads()` already falls back to error payload text when there are no non-error payloads, so this new fallback still will not use `finalAssistantVisibleText` for the reported case where only tool-error payloads are present. Split the non-error and error fallback steps, or check the normalized final assistant text before allowing error payload text to become the summary.
  - confidence: 0.9
- **[P2] Keep the disabled-tools allowlist contract passing:** `src/agents/tool-allowlist-guard.ts:24-26`
  - body: This early return makes `buildEmptyExplicitToolAllowlistError({ sources: [{ label: "runtime toolsAllow", ... }], disableTools: true })` return `null`, but the existing targeted test asserts that explicit runtime allowlists still fail closed when tools are disabled. Narrow the bypass to the inherited LLM-only allowlist case or update the contract and test together.
  - confidence: 0.88
