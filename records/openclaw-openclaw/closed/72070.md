## Summary

PR #72070 is safe to keep closed. Current main already fixes the reported Windows Telegram/native-Jiti crash at the shared loader policy boundary by disabling native Jiti on Windows, matching the working workaround from #71749. The PR’s file-URL normalization approach is not needed for the urgent bug and was not safe to merge as-is due to review-identified platform-sensitive path handling, incomplete Windows path coverage, and failing checks on the PR head.

## Best Possible Solution

Keep the shipped current-main policy that disables native Jiti on Windows for correctness. Leave #72040 as the user-facing tracker for a package containing that fix and the separate Telegram dependency/install concern. Any future Windows native-Jiti performance recovery should be a new measured PR with explicit drive-letter, backslash, UNC, and non-C-drive file-URL regression coverage.
