## Summary

Keep open. Current main still lacks a catalog-layer filter for registry-inferred aggregator-prefix shadow rows, and the provided GitHub context shows #62323 as an open same-repo fix PR with a closing reference.

## What This Changes

The issue asks OpenClaw to stop showing a phantom native provider in `/model status` when an OpenRouter model id starts with a provider-like prefix.

## Best Possible Solution

Land one narrow catalog-layer fix that removes only registry-inferred aggregator-prefix duplicates, preserves explicitly configured native providers, and adds regression coverage for an OpenRouter `google/...` model id.

## Reproduction Assessment

Yes. The issue gives a concrete fixture, and current source inspection shows a registry-emitted `google` row and a configured `openrouter/google/...` row can still coexist in `loadModelCatalog`.

## Solution Assessment

Yes. A catalog-layer filter is the narrowest maintainable fix because it protects status, picker, gateway catalog, and any other catalog consumers; a rendering-only skip would hide one symptom.
