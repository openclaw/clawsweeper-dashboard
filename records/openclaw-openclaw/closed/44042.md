## Summary

The reported self-update failure path is no longer present on current main or the latest shipped release. The repo no longer resolves the reported `@tloncorp/api` git dependency, and the updater now explicitly preserves the repo package manager for dev preflight/update instead of falling back into an npm-in-pnpm-workspace path.

## Close Comment

Closing as implemented on current main after a Codex review. The specific failure path reported here no longer exists: the repo no longer resolves `@tloncorp/api` as a dependency, and the Tlon plugin now depends on `@tloncorp/tlon-skill` instead (`extensions/tlon/package.json:6`). The current updater also resolves and uses the repo’s preferred package manager for dev preflight/update, including pnpm bootstrap, rather than falling back to an npm-in-pnpm-workspace path (`src/infra/update-package-manager.ts:150`, `src/infra/update-runner.ts:814`, `docs/cli/update.md:105`). The latest shipped release `v2026.4.22` already has that same dependency/update state. If you can still reproduce a self-update failure on `v2026.4.22` or newer, please open a fresh issue with the full `openclaw update --json` output and the failing step stderr.
