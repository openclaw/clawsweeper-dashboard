## Summary

Keep open: current main and v2026.6.1 still have a plausible CLI/fallback path where a successful empty Claude CLI result is converted into `empty_response`, allowing the configured fallback model to produce a second visible Telegram group reply.

## What This Changes

The issue asks OpenClaw to treat Claude CLI turns that already delivered via the Telegram message tool as successful even when final assistant text is empty.

## Best Possible Solution

Teach the CLI/fallback boundary to treat a clean empty Claude CLI completion as successful when current-source message-tool delivery or the existing silent-empty policy proves no further model attempt is needed.

## Reproduction Assessment

Source-reproducible, but not live-reproduced in this read-only pass. The reporter's logs plus current main show empty Claude CLI output becomes `empty_response`, and `empty_response` remains a fallback reason.

## Solution Assessment

Yes, this is best solved at the CLI/fallback evidence boundary. A Telegram-only workaround would leave other `message_tool_only` CLI surfaces exposed, while a broad fallback disable would hide real provider failures.

Fix the Claude CLI `message_tool_only` duplicate-reply fallback bug reported in https://github.com/openclaw/openclaw/issues/91302. Current main still throws `FailoverError(reason: "empty_response")` from `src/agents/cli-runner.ts` when Claude CLI returns clean empty text, while Telegram room events require visible replies to go through the `message` tool. Repair the CLI/fallback boundary so a clean empty Claude CLI completion does not trigger model fallback when current-source message-tool delivery evidence or the already-computed `allowEmptyAssistantReplyAsSilent` policy proves the turn is complete. Inspect both `runAgentTurnWithFallback` and queued follow-up paths so the fix is not one-sided. Add focused regression coverage for a Claude CLI Telegram group/message-tool-only turn with fallback configured, proving only the primary tool delivery is treated as terminal and the fallback candidate is not run. Do not add a config option and do not edit CHANGELOG.md.
