## Summary

Keep open. Current main only has the prior shared/final-text `antml:thinking` sanitizer; the embedded streaming/block-reply scanners and write-time thinking promotion paths that this PR targets still omit namespaced tags, so the PR is not obsolete and the linked Slack reasoning-leak bug remains plausibly actionable.

## Best Possible Solution

Keep the PR open and have a maintainer review it as a real focused bug fix. The likely product direction is to align embedded streaming, write-time promotion, and reasoning extraction with the shared `stripReasoningTagsFromText` tag contract, then add regression coverage for `<antml:thinking>` across reasoning stream, block reply, and promoted assistant-message paths before resolving #69235.
