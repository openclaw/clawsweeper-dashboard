## Summary

Current main already fixes the reported pre-provider image optimization failure by falling back to the original in-limit image when the optional optimizer is unavailable, and the shared loader path is still used by the built-in image tool.

## What This Changes

The issue reports that the built-in image-understanding tool fails before provider calls with `Failed to optimize image` for PNG/JPEG local files and URLs across configured vision providers.

## Best Possible Solution

Keep the shared media-loader fallback on main and ship it in the next release; no provider-specific image-model change is needed.

## Reproduction Assessment

No current-main failure reproduction is needed: source inspection shows the reported pre-provider path now falls through successfully for in-limit PNG/JPEG images when the optimizer is unavailable, and regression tests cover that path. The older 2026.4.26 behavior matches the fixed shared-loader failure mode.

## Solution Assessment

Yes. Fixing `loadWebMedia` is the narrow maintainable solution because the image tool, URL/local-file media sends, and channel media paths share that preprocessing code while preserving caps and HEIC safety checks.
