## Summary

Current main implements the reported cron announce behavior. The exact `⚠️ ✉️ Message failed` payload is now treated as a pending presentation warning, not a final job error, and it is only promoted to an error if delivery does not succeed. When delivery succeeds, the isolated cron runner returns `status: "ok"` with no error, so cron state persists `lastDelivered: true` / `lastDeliveryStatus: "delivered"` without `lastError`.

## Best Possible Solution

Close as implemented on main. Keep the current isolated-agent pending-presentation-warning behavior and regression tests; only open a narrower follow-up if maintainers want a generic defensive invariant in `applyJobResult()` for any future runner that returns both `error` and `delivered: true`.
