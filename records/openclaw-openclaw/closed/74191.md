## Summary

Keep this PR open. Current main still drops Feishu card-action sibling fields because the live parser reconstructs `action` from only `value` and `tag`, the decoder/fallback only reads `action.value`, and existing tests do not cover `form_value`, `option`, or `options`. The PR is a plausible focused fix, but it overlaps open PRs #42908 and #43953 and needs maintainer review of the canonical payload encoding before landing or superseding the other candidates.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to choose one canonical Feishu card-action payload contract, ensure the live parser preserves the raw sibling fields before dispatch, decide whether values should be emitted as structured JSON or safely quoted text, cover `input_value`/`name` intentionally if #43202 remains in scope, add parser-to-handler regression coverage, then land the selected PR and close or supersede the overlapping candidates.
