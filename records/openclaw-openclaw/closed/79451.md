## Summary

## What This Changes

The issue reports that per-agent `tools.deny` policy is not applied when `claude-cli` receives MCP servers through the generated CLI backend MCP config.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/79480 after correcting any remaining policy matching concerns; keep broader per-agent MCP process scoping separate at https://github.com/openclaw/openclaw/issues/72591.

Yes. Source inspection shows the documented deny-wins per-agent policy contract, while the current CLI MCP writer still receives config/workspace inputs without agent or effective-policy filtering; I did not run the external CRM MCP setup live.

## Solution Assessment

Yes. Applying effective policy before writing the generated CLI MCP config is the narrowest maintainable boundary because `claude-cli` reads that config directly; the existing linked PR should be reviewed or repaired rather than closing this issue early.
