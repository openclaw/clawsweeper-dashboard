## Summary

Current `main` already implements ACP one-shot parent completion delivery: default non-`streamTo:"parent"` ACP runs are registered for completion tracking/announce cleanup, and explicit `streamTo:"parent"` relays lifecycle completion events back to the parent session. Docs and changelog also describe this as shipped behavior in `v2026.3.31`.
