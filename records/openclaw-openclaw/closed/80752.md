## Summary

Keep open: current main still lacks a commitments-specific model override in the public config type, schema, help/docs, and resolver. The related fast-mode issue and open PR address a different extractor-routing bug, so they do not supersede this model-override request.

## What This Changes

The issue requests a public `commitments.model` override so hidden commitment extraction can use a different model than the main agent default.

## Best Possible Solution

Decide the commitments model-routing contract, then add the config type/schema/help/docs plus resolver and focused regression coverage for override precedence and default inheritance.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection gives high confidence that current main has no `commitments.model` key and that extraction inherits the agent/default model through `resolveCommitmentDefaultModelRef`.

## Solution Assessment

Unclear until maintainers choose the public config contract. A narrow resolver/schema/docs/test change looks maintainable, but fallback semantics and interaction with fast-mode behavior need explicit product approval first.
