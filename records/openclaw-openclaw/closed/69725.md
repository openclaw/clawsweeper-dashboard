## Summary

Keep PR #69725 open. Current main still has the reported version-skew failure: agent `exec host=node` always invokes `system.run.prepare`, gateway policy rejects commands not declared by the node, and the current macOS node command list still advertises `system.run`/`system.which` without `system.run.prepare`. Related #38781/#37591 track the same macOS prepare gap, but they do not make this fallback PR obsolete because #69725 addresses compatibility with nodes that only declare `system.run`.

## Best Possible Solution

Keep #69725 open for maintainer review. The best path is to either land a narrow, tested fallback for `exec host=node` when a node declares `system.run` but not `system.run.prepare`, or explicitly choose #38781's direct macOS prepare implementation and leave a clear compatibility decision for older macOS nodes. Current main is not implemented enough to close this PR.
