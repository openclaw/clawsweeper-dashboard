## Summary

Current main and the shipped v2026.5.4 tree already acknowledge Discord native slash commands immediately, so the 2026.5.3-1 interaction-timeout report is obsolete for the current supported code.

## What This Changes

The issue requests deferring Discord slash-command interactions before doing model, status, session, or routing work so Discord does not hit its 3-second acknowledgement deadline.

## Best Possible Solution

Keep the shipped v2026.5.4 Discord native-command defer behavior and direct affected 2026.5.3-1 users to update rather than track a separate core fix.

## Reproduction Assessment

No for current main: the reported 2026.5.3-1 failure is credible from the issue, but current main and v2026.5.4 defer Discord slash commands before slow command processing. The focused source path and tests show the current failure path no longer matches.

## Solution Assessment

Yes: immediately deferring in the Discord-owned native slash-command entrypoint is the narrow maintainable fix for Discord's acknowledgement deadline. The current follow-up delivery path then handles model pickers, status replies, plugin commands, and agent replies after the acknowledgement.
