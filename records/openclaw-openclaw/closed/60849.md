## Summary

Close this PR as superseded. The linked crash-loop report was closed by a maintainer after current main shipped an alternate fix: invalid startup configs are restored from last-known-good backups, and the public logging contract remains `off | tools` rather than adding a third `all` mode. Merging this branch now would reintroduce a stale public config value with unclear behavior.

## What This Changes

The PR adds `"all"` as a `logging.redactSensitive` mode in config schemas/types/help text, preserves it in redaction normalization, and updates security audit remediation text.

## Best Possible Solution

Keep the shipped two-mode redaction contract plus invalid-config recovery. If maintainers later want `"all"`, it should be a new product/API decision with clear semantics, docs, generated schema updates, and tests that distinguish it from `"tools"`.

## Reproduction Assessment

Yes. The linked report has concrete config steps, and current main source shows `"all"` remains invalid while startup recovery covers the crash-loop path for installs with a last-known-good backup.

## Solution Assessment

No. The proposed branch is no longer the best solution because current main resolved the user-visible failure through recovery and documentation, while the PR adds a third public mode whose behavior is not distinct from the existing non-`off` redaction path.
