## Summary

Current main no longer matches the reported v2026.4.5 failure mode where every CLI command reaches the agent message-tool schema at startup. The root version path returns before full CLI import, `status` is route-first with plugin loading disabled, and `message` is registered as a plugin-free core command whose CLI send path uses `commands/message.ts`, not `src/agents/tools/message-tool.ts`. The same relevant release-smoke coverage is present in v2026.4.24.

## Best Possible Solution

Close this v2026.4.5 startup-crash report as implemented on current main. Keep the current lazy/route-first CLI startup behavior and release packed-CLI smoke coverage; open a new focused issue only if the poll-schema TDZ is reproduced in a current release on an agent message-tool path.
