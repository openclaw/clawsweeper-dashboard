## Summary

Current `main` already covers the reported failure modes: QQ media downloads now allow the Tencent host family, custom provider ids are no longer caught by Qwen's `qwen3.6-plus` Coding Plan suppression, custom image-capable providers are auto-registered for media understanding/image-tool routing, and gateway attachment parsing now offloads images for text-only models instead of dropping them. The changelog entries are in the unreleased `2026.4.24` section, so I could not tie this to a shipped release yet.

## Evidence

- **Qwen suppression is scoped to bundled ids only:** `suppressBuiltInModel` now returns a suppression only when the provider id is `qwen` or `modelstudio`; unrelated custom ids such as `bailian` are excluded from the `qwen3.6-plus` Coding Plan filter.
  - file: [extensions/qwen/index.ts:177](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/extensions/qwen/index.ts#L177)
  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)
- **Custom image-capable providers auto-register for media understanding:** The media-understanding registry auto-adds config providers whose model definitions declare `input: ["image", ...]`, wiring them to the generic image runtime instead of requiring a bundled provider-specific registration.
  - file: [src/media-understanding/provider-registry.ts:41](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/src/media-understanding/provider-registry.ts#L41)
  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)
- **Regression test covers image tool pairing for custom providers:** There is a targeted test proving the image tool selects a custom provider when that provider declares an image-capable model, covering the reported `Unknown model` class of failure for custom providers.
  - file: [src/agents/tools/image-tool.test.ts:709](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/src/agents/tools/image-tool.test.ts#L709)
  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)
- **Gateway no longer drops images for text-only primary models:** `parseMessageWithAttachments` now offloads image attachments as `media://` refs when `supportsImages` is false, and the test suite asserts that text-only model runs keep the attachment via offload instead of dropping it.
  - file: [src/gateway/chat-attachments.ts:309](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/src/gateway/chat-attachments.ts#L309)
  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)
- **QQ/Tencent host allowlist covers the reported QQ media hostname:** QQ Bot media downloads now use an SSRF policy that includes `*.qq.com.cn`; that wildcard covers `multimedia.nt.qq.com.cn`, matching the prior comment that pointed to PR `#65788` for the QQ-specific part.
  - file: [extensions/qqbot/src/engine/utils/file-utils.ts:15](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/extensions/qqbot/src/engine/utils/file-utils.ts#L15)
  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)
- **Docs and changelog match the new behavior:** The public Qwen docs now say `qwen3.6-plus` should use the Standard pay-as-you-go endpoint, and the changelog records the attachment-offload and media-understanding fixes in the `2026.4.24 (Unreleased)` section rather than the latest shipped release.
  - file: [docs/providers/qwen.md:214](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/docs/providers/qwen.md#L214)
  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)
