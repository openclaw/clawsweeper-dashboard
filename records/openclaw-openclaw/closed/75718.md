## Summary

Close as implemented on current main. The bundled runtime dependency path now checks materialized package-level plans before staging, reuses already materialized roots, and has regression tests that fail if unchanged generated runtime deps reinstall; local tag checks show this is present on main but not in the latest v2026.4.29 release.

## What This Changes

The issue asks OpenClaw to skip bundled plugin runtime dependency staging on restart when the plugin plan is unchanged and the runtime dependency directory is already complete.

## Best Possible Solution

Keep the current main behavior: use the generated plugin dependency plan and installed package sentinels as the startup fast path, only repair when the plan or materialization is missing/stale, and ship the fix in the next release.

## Reproduction Assessment

Yes. The report gives concrete v2026.4.29 restart logs, and current main has a focused fixture for the unchanged generated-manifest state that would fail if package-manager staging ran again; I did not run tests because this review was explicitly read-only.

## Solution Assessment

Yes. The current implementation is the narrow maintainable fix: it preserves startup verification while skipping package-manager work for complete materialization and does not require a broader async staging redesign to solve the repeated restart staging loop.
