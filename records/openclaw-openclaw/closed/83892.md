- `impact:crash-loop`: An unhandled restart-script spawn failure can terminate the CLI during update restart handoff, which is a process availability failure.

## Summary

Keep open. Current main and the latest release still run the detached restart script without a synchronous `spawn()` guard or `ChildProcess` `error` listener, and the linked open PR is the active implementation candidate that uses closing syntax for this issue.

## What This Changes

Make `runRestartScript` treat restart-script spawn failures as best-effort so update restart handoff failures do not crash the CLI.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/85761, or close that PR with an equivalent narrow fix that keeps restart-script handoff best-effort and covers both sync throws and child `error` events.

## Reproduction Assessment

Yes, at source level. Current main lacks a child-process error listener in `runRestartScript`, and a local Node probe shows a failed spawn emits an unhandled `error` event unless the child handles it.

## Solution Assessment

Yes. The narrow maintainable solution is to catch synchronous `spawn` failures and attach a child `error` listener inside `runRestartScript`, with focused regression tests, which the linked PR attempts.
