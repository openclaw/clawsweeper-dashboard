## Summary

Keep open. Current main still accepts a truthy context-engine assemble result without checking that `messages` is an array, so malformed plugin output can still replace session history with `undefined` and later crash prompt diagnostics.

## What This Changes

The issue reports a prompt-assembly crash when lossless-claw returns malformed context-engine assembly output.

## Best Possible Solution

Add a shared assemble-result validator or immediate runner guard that warns on malformed output and continues with the original pipeline messages without mutating session state.

## Reproduction Assessment

Yes, by source inspection. A fake context engine returning a truthy assemble result with `messages: undefined` can flow into `activeSession.agent.state.messages` and later hit `activeSession.messages.length` on current main.

## Solution Assessment

Yes. The narrow maintainable fix is generic assemble-result validation at the shared harness or runner boundary, not lossless-claw-specific handling or the broader metadata API proposed in https://github.com/openclaw/openclaw/issues/82137.
