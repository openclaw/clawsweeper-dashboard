## Summary

Current `main` already implements session-scoped webchat delivery for async/internal agent runs and the Control UI now appends final chat payloads that arrive under a different `runId`, which matches this report's missing-reply behavior.

## Evidence

- **Session-scoped agent events keep `sessionKey` for Control UI-visible runs:** `emitAgentEvent` now carries the run context/session key into emitted agent events, which is the prerequisite for routing inter-session/subagent follow-up turns back to the same webchat session.
  - file: [src/infra/agent-events.ts:209](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/infra/agent-events.ts#L209)
  - command: `git blame -L 209,218 src/infra/agent-events.ts`
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Gateway broadcasts async finals to webchat chat subscribers:** `emitChatFinal` builds a `chat` final payload with `sessionKey` and broadcasts it to webchat plus session subscribers, so a resumed/internal run no longer depends on a fresh `agent.wait` poll to surface the reply.
  - file: [src/gateway/server-chat.ts:827](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/gateway/server-chat.ts#L827)
  - command: `git blame -L 827,843 src/gateway/server-chat.ts`
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Control UI accepts finals from a different run id:** The chat event handler explicitly handles a `final` event whose `runId` differs from the currently tracked run and appends the assistant message instead of dropping it.
  - file: [ui/src/ui/controllers/chat.ts:437](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/ui/src/ui/controllers/chat.ts#L437)
  - command: `git blame -L 437,447 ui/src/ui/controllers/chat.ts`
  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Regression coverage exists for the reported path:** Control UI tests now assert that a final payload from another run is appended without being discarded, covering the subagent/internal follow-up delivery path described in the issue.
  - file: [ui/src/ui/controllers/chat.test.ts:112](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/ui/src/ui/controllers/chat.test.ts#L112)
