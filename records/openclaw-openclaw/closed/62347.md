## Summary

Current `main` already covers the reported source-checkout failure path: bundled public-surface resolution now falls back from `dist/extensions` and `dist-runtime/extensions` into the source `extensions/<id>` tree, `memory-core` still exposes the needed `runtime-api` surface there, and the behavior is locked by regression coverage. The same implementation is present in release `v2026.4.23`.

## Evidence

- **Bundled public-surface loader now falls back from dist/dist-runtime to source extensions:** `resolveBundledPluginPublicSurfacePath()` explicitly treats `dist/extensions` and `dist-runtime/extensions` as package bundled dirs and falls back to `extensions/<id>` source artifacts when the built public-surface file is absent.
  - file: [src/plugins/public-surface-runtime.ts:61](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/plugins/public-surface-runtime.ts#L61)
