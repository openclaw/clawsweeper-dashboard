## Summary

Current `main` already has the guardrails the report asks for: the macOS app loads existing gateway config on startup without immediately rewriting it, only flips into remote mode on explicit remote-selection actions, prefers attaching to an already-running local gateway, keeps local auto-mode on loopback even when Tailscale is present, and classifies same-host tailnet discovery as local. The latest release `v2026.4.22` contains the same relevant code paths.

## Evidence

- **Startup reads config before enabling sync writes:** `AppState` initializes from `openclaw.json` and selected defaults, then only enables config syncing after initialization is complete. The sync path is guarded by `!isInitializing`, so init itself does not rewrite config.
  - file: [apps/macos/Sources/OpenClaw/AppState.swift:328](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/apps/macos/Sources/OpenClaw/AppState.swift#L328)
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Remote rewrite path is user-driven:** The onboarding flow only switches to remote mode from `selectRemoteGateway()`, and the remote URL/SSH target writes happen inside `GatewayDiscoverySelectionSupport.applyRemoteSelection()`. There is no startup auto-selection path here.
  - file: [apps/macos/Sources/OpenClaw/OnboardingView+Actions.swift:23](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/apps/macos/Sources/OpenClaw/OnboardingView%2BActions.swift#L23)
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Local startup attaches to an existing gateway:** `GatewayProcessManager.startIfNeeded()` first calls `attachExistingGatewayIfAvailable()`, which health-probes the running listener and marks status as `attachedExisting` instead of spawning or replacing it. Regression coverage exists for this path.
  - file: [apps/macos/Sources/OpenClaw/GatewayProcessManager.swift:117](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/apps/macos/Sources/OpenClaw/GatewayProcessManager.swift#L117)
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Tailscale does not force local auto-mode off loopback:** For local `bindMode: "auto"`, `GatewayEndpointStore.resolveLocalGatewayHost()` returns `127.0.0.1` even when a Tailscale IP exists. The matching test asserts exactly that behavior.
  - file: [apps/macos/Sources/OpenClaw/GatewayEndpointStore.swift:592](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/apps/macos/Sources/OpenClaw/GatewayEndpointStore.swift#L592)
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Same-host tailnet discovery is recognized as local:** `GatewayDiscoveryModel.isLocalGateway()` treats matching `tailnetDns` as local identity, and the regression test covers that case. That prevents the app from classifying its own tailnet-advertised gateway as a remote peer.
  - file: [apps/macos/Sources/OpenClawDiscovery/GatewayDiscoveryModel.swift:559](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/apps/macos/Sources/OpenClawDiscovery/GatewayDiscoveryModel.swift#L559)
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Latest release already contains these paths:** There are no diffs in the relevant macOS gateway files between release tag `v2026.4.22` and current `HEAD`, so the behavior on `main` is already shipped in the latest release.
  - command: `git diff --stat v2026.4.22..HEAD -- apps/macos/Sources/OpenClaw/AppState.swift apps/macos/Sources/OpenClaw/OnboardingView+Actions.swift apps/macos/Sources/OpenClaw/GatewayDiscoverySelectionSupport.swift apps/macos/Sources/OpenClaw/GatewayProcessManager.swift apps/macos/Sources/OpenClaw/GatewayEndpointStore.swift apps/macos/Sources/OpenClawDiscovery/GatewayDiscoveryModel.swift`
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)

## Risks / Open Questions

- If someone still sees this on a current build, the remaining likely causes are stale app preferences/UserDefaults or running an older macOS app binary, not the current `main` code path.
