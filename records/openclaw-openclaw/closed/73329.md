## Summary

This issue reports the same remaining core problem already tracked by open #53639: gateway model-pricing bootstrap performs remote pricing fetches during startup and has no user-facing opt-out. Current main still lacks the requested config gate, but open PR #68651 is already the direct implementation candidate for the canonical issue.

## Best Possible Solution

Close this as a duplicate of #53639 and keep the implementation discussion on #68651 or its replacement. The final fix should add a validated, documented pricing-bootstrap opt-out that skips both OpenRouter and LiteLLM catalog fetches for all configured model refs when disabled, while preserving current default behavior.
