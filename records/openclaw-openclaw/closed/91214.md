## Summary

Keep open. Current main and v2026.6.1 still route active-memory recall through `prependContext` into the model user prompt; current OpenAI-compatible transports forward `prompt_cache_key` from `promptCacheKey` or `sessionId` rather than hashing the request body, so the exact key-change claim needs live provider proof, but the cache-boundary issue remains actionable.

## What This Changes

The issue reports that active-memory `prependContext` changes OpenAI-compatible prompt-cache behavior on otherwise similar turns and collapses provider cache reuse.

## Best Possible Solution

Add an owner-approved cache-aware path for active-memory dynamic recall, likely a below-boundary system-prompt addition or an explicit documented limitation, without changing generic `prependContext` semantics for all plugins.

## Reproduction Assessment

Source-reproducible, but not live-reproduced: active-memory still returns `prependContext`, and current runner code prepends it into the model prompt. The reported provider cache-hit collapse and changing key value need a real OpenAI-compatible provider trace.

## Solution Assessment

Unclear. The requested direction is plausible, but the best fix depends on whether maintainers want active-memory to use a cache-aware system-prompt suffix, a new plugin hook boundary, or documentation of the current `prependContext` limitation.
