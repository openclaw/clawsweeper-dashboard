## Summary

Safe Browsing URL screening is better scoped as a plugin/ClawHub package, and current main already exposes plugin hooks that can intercept and enforce pre-tool checks for `web_fetch` and `web_search` without adding core-specific behavior.

## Evidence

- **Vision keeps optional capability out of core:** The project vision says core should stay lean, optional capability should usually ship as plugins, and the bar for adding optional plugins to core is intentionally high.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/VISION.md#L54)
  - command: `nl -ba VISION.md | sed -n '35,80p'`
- **Plugin hook contract exposes pre-tool interception:** `before_tool_call` receives `toolName` and `params`, and can return rewritten params, `block`, or an approval request. That is the enforcement surface needed to screen URLs before tool execution.
  - file: [src/plugins/hook-types.ts:318](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/plugins/hook-types.ts#L318)
  - command: `nl -ba src/plugins/hook-types.ts | sed -n '318,348p'`
- **SDK docs describe terminal before_tool_call behavior:** The public SDK docs explicitly document that `before_tool_call` can make a terminal block decision, which is appropriate for unsafe URL screening.
  - file: [docs/plugins/sdk-overview.md:232](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/docs/plugins/sdk-overview.md#L232)
  - command: `nl -ba docs/plugins/sdk-overview.md | sed -n '225,245p'`
- **All agent tools are wrapped with before_tool_call:** Tool construction on current main wraps normalized tools with `wrapToolWithBeforeToolCallHook`, so plugin enforcement applies across tool execution rather than only at prompt level.
  - file: [src/agents/pi-tools.ts:707](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/agents/pi-tools.ts#L707)
  - command: `nl -ba src/agents/pi-tools.ts | sed -n '700,724p'`
- **Hook runner blocks or rewrites before execution:** The runtime calls `runBeforeToolCall`, honors `block`, merges adjusted params, and throws before the underlying tool executes, which matches the requested scan-before-use workflow.
  - file: [src/agents/pi-tools.before-tool-call.ts:263](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/agents/pi-tools.before-tool-call.ts#L263)

## Close Comment

Closing this as better suited for [ClawHub](https://clawhub.ai/)/community plugin work after Codex review.

Safe Browsing URL screening is better scoped as a plugin/ClawHub package, and current main already exposes plugin hooks that can intercept and enforce pre-tool checks for `web_fetch` and `web_search` without adding core-specific behavior.

What I checked:

- **Vision keeps optional capability out of core:** The project vision says core should stay lean, optional capability should usually ship as plugins, and the bar for adding optional plugins to core is intentionally high. (`VISION.md:54`)
- **Plugin hook contract exposes pre-tool interception:** `before_tool_call` receives `toolName` and `params`, and can return rewritten params, `block`, or an approval request. That is the enforcement surface needed to screen URLs before tool execution. (`src/plugins/hook-types.ts:318`)

So I’m closing this as a scope-fit item for the plugin/community path rather than keeping it open as an OpenClaw core request.
