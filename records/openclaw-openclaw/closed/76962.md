## Summary

Keep open: current main still has a source-visible late-output path where subagent timeout completion records a terminal outcome without the explicit kill/queue-clear path, while detached exec approval follow-up can still send directly after session resume failure.

## What This Changes

The issue asks that subagent timeouts terminate the underlying Claude CLI work, cancel child-owned exec approvals, and prevent late child output from bypassing parent routing.

## Best Possible Solution

Bind subagent timeout to explicit abort and queue-clear semantics, expire child-owned exec approval continuations, and make late child follow-up fail closed once the owning run is terminal.

## Reproduction Assessment

Unclear for the full physical Claude CLI zombie claim because I did not run that live setup. Source inspection does show a concrete late-output path through timed-out subagent completion, detached exec approval continuation, and direct fallback delivery after session resume failure.

## Solution Assessment

Yes for the maintainer direction, but the exact fail-closed routing policy needs owner/security review. The narrow maintainable fix is lifecycle cleanup parity with explicit kill plus a late-follow-up guard for terminal child runs.
