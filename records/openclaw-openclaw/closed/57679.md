## Summary

The request is for optional context-management behavior that OpenClaw already scopes to the plugin ecosystem rather than core. Current main has a first-class `context-engine` plugin slot, documents third-party context engines, and already points users to an existing community plugin (`lossless-claw`) for DAG-style context compaction; exec-output compaction is also already handled via the bundled `tokenjuice` plugin.
