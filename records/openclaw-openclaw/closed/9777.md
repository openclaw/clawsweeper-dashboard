## Summary

Issue #9777's reported JSON-parsing break is already fixed on current `main`: explicit `--json` CLI commands now suppress doctor preflight stdout so machine-readable output stays clean, including `openclaw sessions --json`.

## Evidence

- **`sessions` is an explicit JSON-output command:** The `sessions` CLI command declares `--json` and forwards it to `sessionsCommand`, so it participates in the JSON-output startup path.
  - file: [src/cli/program/register.status-health-sessions.ts:124](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/src/cli/program/register.status-health-sessions.ts#L124)

  - file: [src/cli/command-startup-policy.ts:41](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/src/cli/command-startup-policy.ts#L41)
