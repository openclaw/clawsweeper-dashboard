## Summary

Current `main` already contains a targeted Discord gateway-readiness fix for the `awaiting gateway readiness` startup stall reported from `v2026.4.14`, plus adjacent `gateway status` startup trimming that matches the supporting Windows datapoint in the comments.

## Evidence

- **Changelog: Discord gateway startup stall fixed on main:** The unreleased changelog explicitly says: `Discord/gateway: prevent startup from getting stuck at `awaiting gateway readiness` when Carbon gateway registration races with a lifecycle reconnect.` That matches this issue's reported symptom and log line.
  - file: [CHANGELOG.md:175](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/CHANGELOG.md#L175)
  - command: `nl -ba CHANGELOG.md | sed -n '160,185p'`
  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **Current Discord lifecycle now times out and restarts a startup-stuck gateway:** `extensions/discord/src/monitor/provider.lifecycle.ts` now enforces a 15s READY timeout, logs `gateway was not ready after 15000ms; restarting gateway`, waits for stale socket close, reconnects, and only fails after a second timeout. That is exactly the sort of protection missing in the issue's 60s `awaiting gateway readiness` stall.
  - file: [extensions/discord/src/monitor/provider.lifecycle.ts:22](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/extensions/discord/src/monitor/provider.lifecycle.ts#L22)

- **Regression tests cover startup-never-ready and reconnect recovery paths:** `extensions/discord/src/monitor/provider.lifecycle.test.ts` has explicit tests that restart once when startup never reaches READY, wait for a stale startup socket to close before reconnecting, and fail only after a second timeout. Those tests directly exercise the fixed startup-race behavior.
  - file: [extensions/discord/src/monitor/provider.lifecycle.test.ts:307](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/extensions/discord/src/monitor/provider.lifecycle.test.ts#L307)
