## Summary

Close as implemented: current main already archives/deletes transcript artifacts when age/count maintenance removes session index entries, removes trajectory sidecars for those removed sessions, and disk-budget cleanup handles unreferenced transcript/trajectory artifacts. The broader first-class directory orphan sweep remains tracked by #75658 and PR #75672.

## What This Changes

Request session maintenance to garbage-collect transcript and trajectory files when stale or over-cap `sessions.json` entries are pruned, with an optional orphan sweep.

## Best Possible Solution

Keep the shipped row-driven transcript and trajectory cleanup, and continue any broader directory-wide orphan/archive cleanup UX in #75658 and PR #75672.

## Reproduction Assessment

Yes. The report can be reproduced with a stale or over-cap session store plus `.jsonl` and trajectory sidecars; current integration tests model that path and assert the stale/capped transcripts and trajectory sidecars are removed or archived.

## Solution Assessment

Yes. The current solution is the narrow maintainable fix because runtime writes and `sessions cleanup --enforce` share the session-store writer that owns pruning, capping, transcript archival, and trajectory cleanup; a standalone orphan sweep is separate follow-up work.
