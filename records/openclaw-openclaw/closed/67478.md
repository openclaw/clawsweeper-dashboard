## Summary

Keep open. Current main still lacks a CLI-wide root-execution guard or equivalent DigitalOcean 1-Click mitigation, and the unmerged open PR #67509 is already paired as the concrete implementation candidate.

## What This Changes

The issue asks OpenClaw to prevent or prominently warn against direct root execution on DigitalOcean 1-Click images before root-owned state and competing gateway services can be created.

## Best Possible Solution

Review and land PR #67509 or an equivalent maintainer-approved early guard/wrapper before state or service mutation, with DigitalOcean guidance updated for any legitimate root workflows that remain.

## Reproduction Assessment

Yes. The report gives deterministic droplet steps, and current source inspection still shows the necessary path: no entrypoint uid guard, root HOME resolves to root state, and doctor service repair can write and activate a user-scoped systemd unit; I did not live-run the destructive scenario.

## Solution Assessment

Yes at the issue level. Preventing direct root execution before state or service mutation is the right boundary, but the exact core versus managed-image scope should be decided with the existing root-based install docs in mind.
