## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] List every approval action in pending text:** `src/infra/exec-approval-reply.ts:280-282`
  - body: `buildExecApprovalPendingText` renders only `descriptors[0]`, so text-only clients and copied logs show one allow command but hide other allowed actions such as `deny`. This regresses the approval safety path; the pending text should show all allowed action commands, not only the first.
  - confidence: 0.96
- **[P1] Refresh session-store snapshots during maintenance yields:** `src/tasks/task-registry.maintenance.ts:381`
  - body: `runTaskRegistryMaintenance()` creates one pass-wide cache context and then yields between batches. Because session-store reads are snapshot-style, entries created after a yield can be invisible to later `shouldMarkLost` checks, incorrectly marking active tasks as lost.
  - confidence: 0.92
- **[P2] Make hidden exec stripping opt-in for rendered markdown:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatMarkdownPreprocessor.swift:49-55`
  - body: The new `stripHiddenSystemEvents` default is `true`, and `ChatMarkdownRenderer` still calls `preprocess(markdown:)` without an override. User-authored `System ... Exec ...` log quotes can therefore disappear at render time even after decode preserves them.
  - confidence: 0.88
- **[P2] Do not warn on healthy symlinked installs:** `src/infra/update-check.ts:397-398`
  - body: `detectInstallState` marks every symlinked active root as suspicious and emits a recovery hint even when the resolved root is healthy and consistent. Symlinked package roots are common for linked/version-manager installs, so this would produce false drift warnings.
  - confidence: 0.86
- **[P2] Recognize the openclaw gateway binary fallback:** `src/infra/ports-format.ts:32-33`
  - body: The fallback listener identity check only accepts basename `openclaw-gateway`. If listener discovery reports only `command: "openclaw"` for a real `openclaw gateway run` process, the port is misclassified as unknown and status can show a false busy-port warning.
  - confidence: 0.82
