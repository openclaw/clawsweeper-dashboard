## What This Changes

The PR adds an Ollama provider-policy `resolveThinkingProfile` export, a regression test, and a changelog entry so the lightweight artifact advertises `max` for reasoning-capable Ollama models.

## Best Possible Solution

Land the provider-owned lightweight policy hook after exact-head validation, keeping Ollama startup-lazy while aligning the runtime and public-artifact thinking profiles.

## Reproduction Assessment

Yes, source-reproducible: current main can fall back to base thinking levels without `max` because Ollama is startup-lazy and its lightweight provider-policy artifact lacks `resolveThinkingProfile`. I did not run a live Ollama/Fedora session.

## Solution Assessment

Yes, this is the narrow maintainable fix: expose the existing Ollama thinking profile through the startup-safe provider-policy artifact. Flipping `activation.onStartup` would mask the gap and weaken the plugin laziness boundary.
