## Risks / Open Questions

- Changing `allowAgents: ["*"]` from any valid agent ID to configured agents only may break deployments that intentionally rely on dynamically named subagent identities.
- ACP `runtime: "acp"` supports external harness IDs that are not normal OpenClaw config agents, so the fix must avoid collapsing ACP harness policy into native agent registry policy by accident.
- The open `SPAWN_ALLOWLIST=*` work would make the wildcard easier to configure from Docker env, increasing the importance of deciding the wildcard security contract before landing related allowlist changes.
