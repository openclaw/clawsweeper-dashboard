## What This Changes

The PR removes the duplicate cron human status formatter, routes `cron list` and `cron show` through `computeStatus`, and adds a regression test for `lastRunStatus`-only jobs.

## Best Possible Solution

Land the PR after normal maintainer review so cron JSON and human output share the single canonical status resolver.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: on current main, a cron job with `state.lastRunStatus: "ok"` and no `lastStatus` renders JSON status as `ok` but human `cron list`/`cron show` status as `idle`.

## Solution Assessment

Yes. Reusing `computeStatus` for human output is the narrowest maintainable fix; updating the old `formatStatus` helper would preserve duplicate status logic.
