## Summary

The reported Feishu/Lark `chat:` prefix path is already handled by current main: `member-info` resolves chat targets through the Feishu normalizer before calling the chat-member API, and the same behavior is present in the shipped release source. The live report may still indicate a stale or differently loaded Feishu plugin package, but there is no remaining source change to make in the repository for the reported prefix handling.

## What This Changes

The issue reports Feishu/Lark group replies failing because the message tool receives `chatId: "chat:oc_..."` for `member-info`, causing a Feishu API 400 and no visible reply.

## Best Possible Solution

Keep the existing Feishu target normalization in the plugin; if this still appears in a live install, the next actionable report should include the resolved `@openclaw/feishu` package version and `openclaw doctor --fix` output to check for a stale or differently loaded plugin package.

## Reproduction Assessment

No. The reported raw `chat:oc_...` parameter reaches a current-main path that strips the `chat:` prefix before the Feishu chat-member API call, and the same path is present in the reported `v2026.5.7` source.

## Solution Assessment

Yes. The existing channel-boundary normalization is the narrow maintainable solution because it preserves OpenClaw's internal `chat:<id>` target form while passing raw Feishu `oc_...` IDs to Feishu API calls.
