## What This Changes

The issue reports that Ollama models still produce user-visible `[assistant reasoning omitted]` placeholders despite `reasoningDefault: "off"` and model `reasoning: false`.

## Best Possible Solution

Keep this issue open and fix Ollama reasoning-disabled runs so reasoning-only output never becomes the user-visible placeholder, while preserving internal replay placeholders where provider adapters need non-empty turns.

## Reproduction Assessment

Yes from source for the core shape: current Ollama conversion preserves reasoning-only output as `thinking`, and current history sanitizers can replace stripped thinking-only turns with the placeholder. I did not run a live Ollama repro in this read-only review.

## Solution Assessment

No, the suggested broad sanitizer change keyed only on `reasoningDefault` is riskier than needed because the placeholder has an internal replay purpose. The better fix is provider/runtime or delivery-aware handling for Ollama reasoning-disabled, reasoning-only turns plus regression coverage.
