## What This Changes

## Best Possible Solution

Land the narrow wording if maintainers accept instruction-level safety as the right layer, and keep broader runtime enforcement, allowlists, audit logging, and bulk-send policy under https://github.com/openclaw/openclaw/issues/56898.

## Reproduction Assessment

Yes for the documentation gap: current `main` has broad confirmation wording and direct mutating examples, while the PR diff binds approval to resolved targets and previews. I did not run live integrations because the patch changes skill instructions, not executable command behavior.

## Solution Assessment

Yes for the stated scope: editing the bundled skill instructions is the narrowest maintainable way to improve model-followed guidance. If maintainers want non-bypassable guarantees, that should be separate runtime/tool enforcement rather than hidden in this prose-only PR.
