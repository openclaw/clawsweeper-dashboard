## Summary

Current main already fixes the central Feishu streaming-card delivery bug by sending cumulative full text to CardKit updates and final close, matching Feishu's full-content streaming API contract; the reporter used v2026.6.1, which still had the suffix-only update behavior.

## What This Changes

Reported Feishu streaming-card replies can render empty or partial on PC/mobile with `channels.feishu.streaming=true` even though gateway dispatch logs report a completed final reply.

## Best Possible Solution

Keep the cumulative CardKit streaming update behavior on main, ship it in the next release, and treat any remaining table-limit behavior as a separate Feishu platform-limit follow-up if it still reproduces.

## Reproduction Assessment

Yes from source and dependency contract, though not from a live Feishu client in this checkout. v2026.6.1 sent suffix-only CardKit text updates while Feishu documents the field as full text content, and current main changes that path to cumulative content.

## Solution Assessment

Yes. The current-main fix is the narrow owner-boundary repair because it keeps Feishu streaming enabled while aligning the existing CardKit update path with Feishu's API contract instead of adding a new config surface or disabling streaming globally.
