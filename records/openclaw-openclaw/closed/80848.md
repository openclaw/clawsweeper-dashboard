## What This Changes

The branch filters Telegram prompt-context selection after reset/session-start boundaries, carries the session-start timestamp through Telegram buffering, and adds focused regression tests plus a changelog entry.

## Best Possible Solution

Land the targeted Telegram session-boundary fix after normal CI and maintainer merge checks; leave broader live QA or custom reset-trigger refinements as follow-up only if maintainers request them.

## Solution Assessment

Yes. Using persisted sessionStartedAt as the primary boundary, with cached /new or non-soft /reset markers as a secondary guard, is a narrow Telegram-owned fix that avoids core policy changes.
