## Summary

Current `main` already canonicalizes Discord DMs to `user:<id>` instead of `channel:<dmChannelId>`, preserves DM reply routing on the direct-session path, and has regression tests covering both the binding restore path and `lastRoute` persistence. The same implementation is present in the `v2026.4.22` release.
