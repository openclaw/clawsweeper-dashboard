## Summary

Keep this open: it is maintainer-authored/protected, and current main still has a source-level path where a same-role/same-text history copy can replace a richer optimistic message without checking media equivalence.

## What This Changes

The issue reports that Control UI media previews for user attachments and generated media disappear after chat history refresh or final reconciliation.

## Best Possible Solution

Fix Control UI reconciliation so a richer local media message remains visible until history carries equivalent media-backed content, and cover both user attachments and generated/source-reply media with focused tests.

## Reproduction Assessment

No live/browser reproduction was run in this read-only review. Source inspection gives a clear current-main fixture: a previous user message with text plus media can share the same role/text signature as a poorer text-only history message and be discarded during refresh.

## Solution Assessment

Yes, the best path is a narrow Control UI reconciliation fix plus regression coverage. It should compare media richness or equivalence before replacing local optimistic messages, while separately verifying that generated/source-reply media is transcript-backed when available.
