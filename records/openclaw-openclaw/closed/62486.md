## Summary

Current main and the shipped v2026.4.26 tag cover the reported 2026.4.2 UX gaps: direct-message sessions are gateway-backed, WebChat reads gateway transcripts, docs now give the official source-of-truth recommendation instead of promising client-to-client mirroring, and active runs can be queued or stopped through WebChat and inbound /stop fast-abort paths. Broader hardening for stale side effects after interruption remains tracked separately in #70319.

## Best Possible Solution

Keep the shipped v2026.4.26 implementation and docs as the resolution for this broad 2026.4.2 report. Use #70319 for the remaining stronger run-invalidation work, and file a separate focused feature/plugin request if full QQ native-history mirroring is desired beyond the gateway-backed transcript model.
