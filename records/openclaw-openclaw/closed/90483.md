## Summary

Close as implemented: current main and the latest release already redact thinking signatures from cloned display/history objects, and replay hygiene strips stale compaction-bound Anthropic signatures before provider submission.

## What This Changes

The issue asks OpenClaw to stop corrupting Anthropic thinking block signatures when display, history, truncation, or replay paths handle Claude 4.x thinking blocks.

## Best Possible Solution

Keep the shipped shape: preserve native Claude thinking blocks where supported, redact display/history views from cloned objects, and strip only stale or invalid replay signatures before Anthropic submission.

## Reproduction Assessment

No; current main does not expose the reported shared-reference delete path because both display/history sanitizers copy before deleting, and replay has a stale-signature strip before provider conversion.

## Solution Assessment

Yes; the current implementation is the maintainable fix because it keeps Claude 4.x thinking preservation for native replay while preventing display/history redaction from mutating source objects and handling stale compaction signatures separately.
