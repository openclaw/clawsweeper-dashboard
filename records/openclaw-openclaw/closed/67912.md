## Summary

Current `main` already changes dreaming’s default storage away from `memory/YYYY-MM-DD.md`, documents the new output locations, and ships a regression test specifically asserting daily memory files should not be polluted by default.

## Risks / Open Questions

- I did not run a live reproduction in this review because the checkout is read-only and `pnpm` is unavailable here; the decision is based on current code, tests, docs, and the shipped `v2026.4.23` tag.
- Users who explicitly set `plugins.entries.memory-core.config.dreaming.storage.mode` to `inline` or `both` can still write managed phase blocks into daily files by design; the implemented fix is that this is no longer the default behavior.
