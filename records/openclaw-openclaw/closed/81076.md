## Summary

The PR remains useful and should stay open for normal maintainer review: current main still drops top-level browser act fields when `request` is already an object, while the proposed patch narrowly backfills only allowlisted fields without overriding nested values. I found no actionable correctness finding and no diff-introduced security or supply-chain concern.

## What This Changes

The PR changes the browser plugin act parser to copy missing allowlisted top-level act fields into nested `request` objects and adds parser regression tests for backfill and nested precedence.

## Best Possible Solution

Merge the focused parser backfill with its regression tests after normal CI and dependency-audit gate handling, keeping the browser service `/act` contract unchanged.

## Reproduction Assessment

Yes. Source inspection on current main shows a high-confidence path: `action: "act"` with top-level `ref` and `request: { kind: "click" }` returns only the nested request, then the `/act` normalizer throws because that body lacks `ref` or `selector`.

## Solution Assessment

Yes. Backfilling only existing allowlisted top-level act fields when missing from `request` is the narrowest maintainable fix and preserves nested-field precedence.
