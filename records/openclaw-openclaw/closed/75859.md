## Summary

Current main already removes the bundled plugin runtime-dependency install path that produced the v2026.4.29 `Unable to resolve a safe npm executable` failure. Plugin dependency work is now limited to explicit plugin install/update flows, while doctor repair only cleans legacy staging debris; this fix is on main but not yet in a shipped stable release.

## What This Changes

Fix the Ubuntu npm/global-install upgrade path where v2026.4.29 fails gateway startup because bundled plugin runtime dependency repair cannot resolve a safe npm executable.

## Best Possible Solution

Keep the current main behavior and ship it in the next stable patch release; if the same safe-npm error appears on a newer build, it should be reopened as a new regression because the bundled runtime-deps installer should no longer exist.

## Reproduction Assessment

Yes. The issue includes concrete v2026.4.29 Ubuntu commands and logs for the failure path; current-main verification is source-backed rather than a live package repro because this review was read-only.

## Solution Assessment

Yes. Moving bundled dependency staging to legacy cleanup and keeping npm installs only in explicit plugin install/update flows is the narrow maintainable fix for this class of startup/doctor failure.
