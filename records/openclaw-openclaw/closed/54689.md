## Summary

Current main already exposes Telegram file/media sending through the agent message tool, routes Telegram sends with media through `sendMessageTelegram`, and covers PDF/document delivery in tests. The same implementation is present in the shipped `v2026.4.22` release.

## Evidence

- **Shared message tool exposes file/media parameters:** The agent-facing `message` tool schema includes `media`, `buffer`, `path`, `filePath`, and Telegram-specific `forceDocument`/`asDocument`, so file sends are part of the current tool contract.
  - file: [src/agents/tools/message-tool.ts:101](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/agents/tools/message-tool.ts#L101)
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Telegram plugin still advertises send support:** Telegram message-tool discovery always includes the `send` action when the channel is configured, so the send capability is not hidden from the agent on current main.
  - file: [extensions/telegram/src/channel-actions.ts:124](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/extensions/telegram/src/channel-actions.ts#L124)
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Telegram action runtime accepts media inputs:** `handleTelegramAction` reads `mediaUrl` or `media` for `sendMessage` and forwards that media payload into `sendMessageTelegram`, along with `forceDocument`/`asDocument`.
  - file: [extensions/telegram/src/action-runtime.ts:313](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/extensions/telegram/src/action-runtime.ts#L313)
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Telegram send path delivers documents:** `sendMessageTelegram` loads outbound media and falls through to `api.sendDocument(...)` for document-kind files, which is the exact capability requested in the issue.
  - file: [extensions/telegram/src/send.ts:814](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/extensions/telegram/src/send.ts#L814)
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **PDF/document delivery is covered by tests:** The Telegram send tests explicitly verify that a PDF (`report.pdf`) is sent via `sendDocument` with the expected caption parameters.
  - file: [extensions/telegram/src/send.test.ts:1302](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/extensions/telegram/src/send.test.ts#L1302)
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
