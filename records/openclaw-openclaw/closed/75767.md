## Summary

Keep open: current main and the latest release still reach `lsof` in the macOS/Unix gateway port paths implicated by the SMB stat-timeout hang, so the reported bug is not fully fixed.

## What This Changes

The issue asks OpenClaw to avoid `lsof`-based gateway port checks on macOS because SMB-mounted volumes can make `gateway restart` and `gateway --force` hang.

## Best Possible Solution

Make Unix/macOS gateway availability checks socket-probe-first, and reserve process enumeration for verified OpenClaw-owned listener termination with bounded fallbacks.

## Reproduction Assessment

Yes for source-level reproduction: current main still invokes `lsof` from the busy-port `gateway --force`, restart diagnostics, and Unix stale-cleanup paths. A live timing reproduction still needs macOS with a slow or unresponsive SMB/Time Machine mount.

## Solution Assessment

Yes: the socket-probe-first direction is the narrow maintainable solution for busy/free checks. Process enumeration should remain only where OpenClaw must terminate a verified gateway listener.
