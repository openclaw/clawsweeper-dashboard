## Summary

Current main already exposes the requested session lifecycle automation through plugin hooks `session_end` and `session_start`, including reset/idle/daily/delete reasons and transcript context, so the request is implemented well enough.

## Evidence

- **Public docs list session lifecycle hooks:** The agent-loop docs explicitly document `session_start` / `session_end` as plugin hook lifecycle boundaries.
  - file: [docs/concepts/agent-loop.md:103](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/docs/concepts/agent-loop.md#L103)
- **Hook contract includes end reasons and transcript metadata:** `PluginHookSessionEndEvent` carries `reason`, `sessionFile`, `transcriptArchived`, and next-session linkage; reasons include `new`, `reset`, `idle`, `daily`, `compaction`, and `deleted`.
  - file: [src/plugins/hook-types.ts:402](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/plugins/hook-types.ts#L402)
- **Main session init emits lifecycle hooks on rollover:** Session initialization maps stale sessions to `idle`/`daily` and emits `session_end` followed by `session_start` when replacing a session.
  - file: [src/auto-reply/reply/session.ts:121](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/auto-reply/reply/session.ts#L121)
- **Gateway reset/delete paths emit the same hooks:** Gateway session reset service fires `session_end`/`session_start` and resolves the stable archived transcript path for the end event payload.
  - file: [src/gateway/session-reset-service.ts:112](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/gateway/session-reset-service.ts#L112)
- **Tests cover reset, delete, idle, and daily cases:** Lifecycle tests assert `session_end` payloads for reset/delete plus idle/daily rollover reasons and archived transcript/session-file behavior.
  - file: [src/gateway/server.sessions.gateway-server-sessions-a.test.ts:2528](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/gateway/server.sessions.gateway-server-sessions-a.test.ts#L2528)
- **Shipped in latest release tree:** The `v2026.4.23` tag already contains the same `session_start` / `session_end` docs and implementation files.
  - command: `git grep -n "session_start\|session_end" v2026.4.23 -- docs/concepts/agent-loop.md src/plugins/hook-types.ts src/plugins/hooks.ts src/auto-reply/reply/session.ts src/gateway/session-reset-service.ts`
  - sha: [9b18ae877018](https://github.com/openclaw/openclaw/commit/9b18ae877018a34933e27bb7b2986cb97e9602aa)

## Risks / Open Questions

- The implemented surface is the plugin hook API (`session_end` / `session_start`), not an internal HOOK.md event spelled `session:end`. If maintainers consider that distinction material, this close decision would be too aggressive.
