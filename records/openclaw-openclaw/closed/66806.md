## Summary

Current `main` no longer treats an unspecified memory-embedding provider as OpenAI-only. The memory search path now resolves unspecified `memorySearch.provider` to `auto`, tolerates missing embedding credentials in auto mode, and keeps builtin memory search running in FTS-only mode instead of failing agent startup with an OpenAI API-key error.

## Evidence

- **Memory provider default is `auto`, not `openai`:** Regression coverage on current main asserts that when `agents.defaults.memorySearch.provider` is omitted, `resolveMemorySearchConfig()` resolves it to `auto` with fallback `none`. That directly contradicts the issue's reported implicit OpenAI lookup path.
