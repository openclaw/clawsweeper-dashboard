## Summary

The reported Docker auto-create path is already covered by current main and the latest release: writable sandbox skill materialization syncs into a state-owned `skillsWorkspaceDir`, creates `<skillsWorkspaceDir>/skills` recursively before Docker mount construction, and the mount builder only emits existing skill source directories.

## What This Changes

The issue asks OpenClaw to pre-create the materialized sandbox skills directory before Docker receives the bind mount so Docker does not create it as `root:root`.

## Best Possible Solution

Keep the shipped sync-before-mount implementation; any separate Gateway-in-Docker host path translation should stay with the DooD issue rather than adding duplicate directory creation for this report.

## Reproduction Assessment

No. Current main and `v2026.6.8` both create `<skillsWorkspaceDir>/skills` during skill sync before Docker mount construction, and missing materialized skill sources are filtered out instead of being passed to Docker.

## Solution Assessment

Yes. The existing implementation prepares the exact directory Docker would bind and keeps mount construction gated on existing sources, so adding another mkdir in the mount formatter would duplicate the current owner boundary.
