## Summary

Current main already fixes the missing `chokidar` runtime-deps failure: `chokidar` is declared as a mirrored root runtime dependency, the runtime-deps planner folds mirrored package deps into the generated install manifest, and tests now guard both root bare-import drift and the memory-core `chokidar` install path. The fix appears to be current-main only; I found no shipped tag containing the proof commit, and `v2026.4.27` still lacks the mirrored-root declaration.

## What This Changes

The issue asks OpenClaw to include `chokidar` in the generated plugin-runtime-deps dependency package so CLI commands that import the plugin manager work after a fresh global install.

## Best Possible Solution

Keep the current-main runtime-deps mirror behavior and carry it into the next stable or correction release so global installs generate a runtime-deps package that includes `chokidar`.
