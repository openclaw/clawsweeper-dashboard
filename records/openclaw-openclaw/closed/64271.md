## Summary

Current main already has the browser-owned per-profile circuit breaker/backoff that the report was asking for, and the release changelog says this shipped in v2026.4.26. The implementation bounds repeated managed Chrome launch/readiness failures instead of relying only on the model to stop retrying.

## What This Changes

The issue asks OpenClaw to stop repeated `browser.request` calls from spawning managed Chrome indefinitely when CDP startup keeps failing, with bounded retries/backoff and clearer disable-browser guidance.

## Best Possible Solution

Keep the shipped browser-plugin per-profile managed-launch circuit breaker/backoff as the fix; any future request for a permanent disabled-until-restart policy should be tracked separately as policy tuning.
