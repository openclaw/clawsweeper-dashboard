## Summary

Keep #70853 open. Current main still has the reported silent auto-reply reset path: it rebinds the stable session key to a new sessionId/sessionFile, and I did not find the PR’s proposed tail-replay helper or equivalent logical hydration on main. The linked PR #70898 is directly relevant and should remain the active fix path, but it is still unmerged and its head differs from current main.

## Best Possible Solution

Keep the issue open and use PR #70898 as the active implementation path: add a narrowly scoped replay or logical-hydration helper for `resetReplyRunSession`, preserve only safe recent user/assistant transcript records, and add regression coverage proving a silent compaction/role-ordering reset retains DM context while explicit `/new` and `/reset` remain fresh starts.
