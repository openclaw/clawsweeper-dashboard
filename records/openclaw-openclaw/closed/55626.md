## Summary

Current `main` already clears transient mutating-tool failures when the same action later succeeds, including the edit retry/path-alias case described in #55626, so the stale warning should no longer be emitted from preserved `lastToolError` state.

## Evidence

- **Recovered mutating failures are cleared on successful retry:** `handleToolExecutionEnd` records tool errors into `lastToolError`, but if a later completion is successful and matches the same mutating action, it clears `lastToolError` instead of leaving the stale failure in place.
