## Summary

## What This Changes

Request to trim shared `message` tool schema properties to the actions and enabled channel capabilities actually exposed in a session.

## Best Possible Solution

Implement action-aware shared schema trimming in `src/agents/tools/message-tool.ts`, reusing the resolved action set while preserving cross-channel action support and plugin-owned schema contribution behavior.

## Solution Assessment

Yes. Passing the resolved action set into the shared property builder, or gating shared property groups at that same boundary, is the narrow maintainable fix if gateway and cross-channel behavior stay covered.
