## Summary

Keep open: current `main` at `af56926e2fc410809fcb0399435787c7fa8daef8` still lacks an agent-level TTS config surface and still resolves reply TTS from global `messages.tts`/prefs rather than the active session agent. The request is recent, coherent, and corroborated by the issue discussion; there is no strong evidence that it has already been implemented.

## Evidence

- **Agent config has no per-agent TTS field:** `AgentConfig` lists per-agent overrides such as model, identity, tools, runtime, etc., but no `tts` property. A repo search only found `tts?: TtsConfig` on global messages config and Discord voice config, not on `agents.list[]`.
  - file: [src/config/types.agents.ts:70](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/src/config/types.agents.ts#L70)
  - command: `nl -ba src/config/types.agents.ts | sed -n '70,125p'`
  - sha: [af56926e2fc4](https://github.com/openclaw/openclaw/commit/af56926e2fc410809fcb0399435787c7fa8daef8)
- **Speech-core TTS resolution is still global:** `resolveTtsConfig` reads `const raw: TtsConfig = cfg.messages?.tts ?? {};` and builds the active TTS config from that global block.
  - file: [extensions/speech-core/src/tts.ts:313](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/extensions/speech-core/src/tts.ts#L313)
  - command: `nl -ba extensions/speech-core/src/tts.ts | sed -n '300,390p'`
  - sha: [af56926e2fc4](https://github.com/openclaw/openclaw/commit/af56926e2fc410809fcb0399435787c7fa8daef8)
- **Reply dispatch resolves the session agent, but does not thread it into TTS calls:** `dispatch-from-config` resolves `sessionAgentId`, but the later `maybeApplyTtsToReplyPayload` calls pass only payload/config/channel/kind/inboundAudio/ttsAuto. There is no agent id in the TTS call contract, so agent-specific voice selection cannot happen on this path.
  - file: [src/auto-reply/reply/dispatch-from-config.ts:305](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/src/auto-reply/reply/dispatch-from-config.ts#L305)

- **TTS runtime API has no agent parameter and resolves from config/prefs only:** `maybeApplyTtsToPayload` accepts `payload`, `cfg`, `channel`, `kind`, `inboundAudio`, and `ttsAuto`; it immediately resolves TTS state from `cfg` and prefs, with no agent-specific input.
