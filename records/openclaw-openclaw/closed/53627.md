## Summary

Current `main` already supports Discord `user:<id>` DM targets through the Discord plugin delivery path, and this PR's proposed core `src/infra/outbound/*` changes no longer match how target resolution works.

## Evidence

- **Current architecture resolves channel and target separately:** `resolveChannel()` only sends the `channel` argument through `resolveMessageChannelSelection()`, while `resolveActionTarget()` resolves the recipient later from `to`/`target`. That means `user:<id>` is handled as a Discord target, not as a channel-selection value.
