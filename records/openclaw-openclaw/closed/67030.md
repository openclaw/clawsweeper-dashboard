## Summary

Current `main` already prevents wake-triggered exec completion events from a base session from being treated as user-facing exec prompts in heartbeat isolated sessions, and `isolatedSession` by itself does not strip `SOUL.md`/persona context.

## Close Comment

Closing this as implemented after Codex review.

Current `main` already prevents wake-triggered exec completion events from a base session from being treated as user-facing exec prompts in heartbeat isolated sessions, and `isolatedSession` by itself does not strip `SOUL.md`/persona context.

What I checked:

- **Wake-event queue guard on current main:** `resolveHeartbeatPreflight` now only inspects wake-triggered pending events when the queue being peeked is the same queue the heartbeat run will actually drain. With `isolatedSession: true`, base-session wake events are excluded from the isolated-session prompt path. (`src/infra/heartbeat-runner.ts:569`)
- **Direct regression test for this failure mode:** There is an explicit test asserting that a base-session `hook:wake` exec completion is not classified as an `exec-event` when heartbeat isolated sessions are enabled; the run stays `Provider === "heartbeat"`, uses the `:heartbeat` session key, and sends nothing outward. (`src/infra/heartbeat-runner.ghost-reminder.test.ts:379`)
- **Isolated session does not imply lightweight bootstrap:** Heartbeat runs only switch to lightweight bootstrap when `heartbeat.lightContext === true`; otherwise the heartbeat run uses normal bootstrap context. (`src/infra/heartbeat-runner.ts:1068`)
- **Only lightweight heartbeat mode strips bootstrap files down to HEARTBEAT.md:** Bootstrap filtering keeps all workspace bootstrap files unless `contextMode` is `lightweight`; only then do heartbeat runs reduce to `HEARTBEAT.md` alone. (`src/agents/bootstrap-files.ts:165`)
- **Docs match current behavior:** The public heartbeat docs distinguish `lightContext` from `isolatedSession`: `lightContext` keeps only `HEARTBEAT.md`, while `isolatedSession` only removes prior conversation history and still uses main-session delivery routing. ([`docs/gateway/heartbeat.md:223`](https://docs.openclaw.ai/gateway/heartbeat))

So I’m closing this as already implemented rather than keeping a duplicate issue open.
