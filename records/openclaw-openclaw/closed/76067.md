## Summary

Keep open: this is a member-authored, maintainer-labeled PR, GitHub reports the latest head as dirty, and the new descriptor cache has an unbounded process-wide map that regresses the bounded cache behavior on current main.

## What This Changes

The PR replaces plugin tool factory-result caching with cached plugin tool descriptors for repeated prompt-time planning, plus docs, tests, and a changelog entry.

## Best Possible Solution

Keep the lazy descriptor-planning direction, but add bounded eviction for descriptor cache entries and rebase onto current main before normal maintainer review and CI.

## Reproduction Assessment

Yes. A source-level reproduction is deterministic: resolve descriptor-cached plugin tools across many distinct sessionId/context values and observe each cache write inserts a new process-wide Map entry with no eviction path.

## Solution Assessment

No as-is. The latest context-scoped cache appears to preserve sandbox visibility, but it should retain the old bounded-cache property or use a narrower bounded key before merge.
