## Summary

Current `main` does not support the reported orphaned-`grep` outcome strongly enough to keep this open: the exec path now runs through the process supervisor, disables detached child mode for service-managed gateway runs, has explicit process-tree cleanup coverage, and a direct repeated shell-spawn repro did not leave lingering matching `grep` processes.

## Evidence

- **Exec host path uses supervisor child spawn:** Non-PTY host exec builds a child spawn spec with `stdinMode: "pipe-closed"` and runs it through `supervisor.spawn(...)`, so current exec is mediated by the process supervisor rather than ad-hoc shell spawning.
  - file: [src/agents/bash-tools.exec-runtime.ts:720](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/agents/bash-tools.exec-runtime.ts#L720)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Service-managed gateway disables detached children:** The child-process adapter sets `detached` only when not in a service-managed runtime; when `OPENCLAW_SERVICE_MARKER` is present it keeps children attached instead of letting them escape under launchd/systemd.
  - file: [src/process/supervisor/adapters/child.ts:42](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/process/supervisor/adapters/child.ts#L42)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Gateway services set the service marker:** Gateway service environment generation exports `OPENCLAW_SERVICE_MARKER`, so launchd/systemd-managed gateway execs take the non-detached path above.
  - file: [src/daemon/service-env.ts:303](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/daemon/service-env.ts#L303)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Hard kill is process-tree scoped:** Current kill logic targets the full Unix process group (`process.kill(-pid, ...)`) or Windows task tree, and the child adapter uses that process-tree kill path for default `SIGKILL` cleanup.
  - file: [src/process/kill-tree.ts:46](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/process/kill-tree.ts#L46)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Regression coverage for no-detach + process-tree kill:** Tests assert both `killProcessTree(...)` on default kill and `detached: false` in service-managed runtime, matching the launchd/systemd cleanup path relevant to this report.
  - file: [src/process/supervisor/adapters/child.test.ts:106](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/process/supervisor/adapters/child.test.ts#L106)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Direct repeated repro did not leave lingering grep processes:** Ran the reported style of piped command 50 times through Node child-process spawning (`ps aux | grep __openclaw_exec_probe__ | grep -v grep`) and then checked `ps -axo pid=,ppid=,command= | rg '__openclaw_exec_probe__'`; only the probe shell and final `rg` were present, with no orphaned matching `grep` processes left behind.
  - command: `node -e 'const {spawn}=require("node:child_process"); const cmd="ps aux | grep __openclaw_exec_probe__ | grep -v grep"; const run=()=>new Promise((resolve,reject)=>{ const child=spawn(process.env.SHELL||"/bin/sh",["-c",cmd],{detached:true,stdio:["pipe","pipe","pipe"]}); child.stdin.end(); child.on("error",reject); child.on("close",()=>resolve()); }); (async()=>{ for(let i=0;i<50;i++) await run(); })().catch(err=>{ console.error(err); process.exit(1); });' && ps -axo pid=,ppid=,command= | rg '__openclaw_exec_probe__'`
- **Latest release carries the same relevant cleanup files as main:** `sha256sum` of current `src/process/supervisor/adapters/child.ts` and `src/process/kill-tree.ts` matched the same files from release commit `a9797214338ba31c52c796adbb75afb16e0684a9` (`v2026.4.23`).
  - command: `sha256sum src/process/supervisor/adapters/child.ts && git show a9797214338ba31c52c796adbb75afb16e0684a9:src/process/supervisor/adapters/child.ts | sha256sum; sha256sum src/process/kill-tree.ts && git show a9797214338ba31c52c796adbb75afb16e0684a9:src/process/kill-tree.ts | sha256sum`
