## Summary

Current main no longer matches the reported bundled-channel failure path: bundled plugins resolve from the package runtime tree instead of a per-version `plugin-runtime-deps` mirror, `json5` is a root runtime dependency, and legacy runtime-deps roots are now explicit doctor cleanup targets. The proof is on `main` after `v2026.4.29`, with no containing release tag yet.

## What This Changes

The issue reports that bundled Telegram, Discord, and WhatsApp plugins crash after upgrade because built chunks import `json5` from a runtime-deps tree where it is not installed.

## Best Possible Solution

Keep the current package-tree bundled plugin loading model and ship it in the next release, with legacy runtime-deps cleanup handled by doctor.

## Reproduction Assessment

Yes. The report gives a precise ESM missing-package path and upgrade flow; current-main source verification shows that path is now legacy and bundled channels resolve from the package runtime tree instead.

## Solution Assessment

Yes. Loading bundled plugins from the package runtime tree and treating old runtime-deps mirrors as cleanup targets is narrower and more maintainable than adding another package to a legacy startup staging path.
