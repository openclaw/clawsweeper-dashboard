## Summary

Current `main` already rejects invalid `config.patch` payloads before any config write or restart, and the same behavior is present in release `v2026.4.22`.

## Evidence

- **`config.patch` validates before write:** `src/gateway/server-methods/config.ts` merges the patch, restores redacted values, validates with `validateConfigObjectWithPlugins`, and returns an `INVALID_REQUEST` on validation failure before `writeConfigFile(...)` is reached later in the handler.
  - file: [src/gateway/server-methods/config.ts:516](https://github.com/openclaw/openclaw/blob/9f4ef6162dbde384797abc002cd4241080eb65e3/src/gateway/server-methods/config.ts#L516)
  - command: `nl -ba src/gateway/server-methods/config.ts | sed -n '471,579p'`
  - sha: [9f4ef6162dbd](https://github.com/openclaw/openclaw/commit/9f4ef6162dbde384797abc002cd4241080eb65e3)
- **Unknown keys are schema errors:** The root config schema is strict, and the `gateway` object is also strict, so unrecognized keys like `gateway.hooks` are rejected during validation rather than silently persisted.
  - file: [src/config/zod-schema.ts:237](https://github.com/openclaw/openclaw/blob/9f4ef6162dbde384797abc002cd4241080eb65e3/src/config/zod-schema.ts#L237)
