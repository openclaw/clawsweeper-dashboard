## Summary

Issue #45042 proposes useful memory and compaction capabilities, but current main already treats this area as plugin territory: memory is an exclusive plugin slot, context assembly/compaction are pluggable via context-engine and compaction-provider APIs, and service-backed memory retrieval is already documented through external plugins rather than core changes.

## Risks / Open Questions

- There is no in-tree Supabase/pgvector plugin on current main, so closing as ClawHub assumes the community or reporter will carry that integration externally.
- Privacy isolation and decay/ranking semantics would still need careful implementation and tests in the plugin; core only provides the seams.
