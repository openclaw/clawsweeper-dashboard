## Summary

Close: current main and the latest release already cover the durable service-environment outcome through preserved safe service env values, state-dir `.env`, and the managed proxy path for Telegram egress; persisting raw `HTTP_PROXY`/`HTTPS_PROXY` in generated service metadata is intentionally not the supported route.

## What This Changes

Request durable user-managed gateway environment variables so custom proxy settings survive OpenClaw upgrades.

## Best Possible Solution

Keep the shipped durable service-env and managed proxy paths: put safe durable values in `$OPENCLAW_STATE_DIR/.env`, use `proxy.proxyUrl` or `OPENCLAW_PROXY_URL` for Telegram egress, and leave `service-env/*.env` as OpenClaw-managed generated files.

## Reproduction Assessment

Not applicable as a feature request. Source confirms generated LaunchAgent env files are rewritten by design, while current main already provides durable env/proxy sources for the requested outcome.

## Solution Assessment

Yes for the central outcome: the existing durable `.env` and managed proxy path is the safer maintainable solution. Adding a second arbitrary override file sourced after generated service env would be a new config/security surface and is not needed for the Telegram proxy case.
