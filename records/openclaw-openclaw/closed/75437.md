## Summary

Current main already addresses the reported v2026.4.29 runtime-deps restaging loop: bundled runtime dependency installs now persist through the generated install-root package manifest, materialized package-level supersets are treated as converged, lazy plugin activation fast-exits instead of rescanning/reinstalling, and Jiti alias normalization is cached. The fix is on main but not yet in a tagged release.

## What This Changes

The issue asks OpenClaw to stop Gateway event-loop stalls caused by repeated bundled plugin runtime-dependency staging on startup and per message, plus repeated Jiti alias/cache work.

## Best Possible Solution

Keep the current main implementation and make sure the next release includes the runtime-deps materialization fix; no separate source change is needed for this report.

## Reproduction Assessment

Yes. The report includes a clean v2026.4.29 `openclaw --dev gateway run` reproduction and per-message timing/profile evidence; this read-only review verified the current-main code and regression tests rather than running a state-writing gateway repro.

## Solution Assessment

Yes. The current fix targets the durable convergence check and hot alias path directly, avoiding repeated package-manager staging while preserving per-plugin activation correctness.
