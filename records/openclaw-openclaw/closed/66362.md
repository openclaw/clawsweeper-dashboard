## Summary

Current `main` already contains a QMD slugified-path recovery fix plus targeted regression tests for `memory_search` -> `memory_get` roundtrips. The fix appears to have landed after the latest tagged release, so it is implemented on `main` but not yet tied to a shipped release tag.

## Evidence

- **QMD file URI parsing and slug recovery:** `QmdMemoryManager` now parses `qmd://<collection>/...` file refs, normalizes slugified lookup segments, and matches them back to indexed real paths instead of treating the slugified path as a literal filesystem path.
  - file: [extensions/memory-core/src/memory/qmd-manager.ts:2329](https://github.com/openclaw/openclaw/blob/30aa7e0d4d225650b58c2fba79e71bffa5627b08/extensions/memory-core/src/memory/qmd-manager.ts#L2329)

- **Roundtrip-safe search path and read resolution:** Search results are built as real workspace-relative paths when possible, otherwise as `qmd/<collection>/...`, and `resolveReadPath()` resolves those collection-aware paths back to the correct root for `memory_get`/read operations.
  - file: [extensions/memory-core/src/memory/qmd-manager.ts:2599](https://github.com/openclaw/openclaw/blob/30aa7e0d4d225650b58c2fba79e71bffa5627b08/extensions/memory-core/src/memory/qmd-manager.ts#L2599)
