## What This Changes

This PR adds a `homedir()` boundary check to `loadProjectContextFiles()` so ancestor `AGENTS.md` / `CLAUDE.md` discovery stops above the user's home directory.

## Best Possible Solution

Bound project-context discovery to the configured workspace or project root, preserve ancestor ordering only inside that boundary, and add regression plus real-behavior proof for excluded parent files.

## Reproduction Assessment

Yes, source-level. Current main walks ancestors to the filesystem root, and the PR head still walks through same-home parents outside the workspace while disabling context loading for valid workspaces elsewhere.

## Solution Assessment

No. A home-directory boundary is only a partial mitigation; the safer fix is to use the configured workspace or project root as the trust boundary.
