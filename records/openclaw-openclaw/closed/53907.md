## Summary

Close as implemented on main: the WhatsApp channel now resolves through the installable-channel flow instead of failing immediately as unsupported, and that behavior was already shipped in `v2026.4.22`.

## Evidence

- **Login now resolves installable channels before rejecting them:** `runChannelLogin` calls `resolveInstallableChannelPlugin(... allowInstall: true ...)` before any `Unsupported channel` throw, so `openclaw channels login --channel whatsapp` can recover through plugin resolution/install.
  - file: [src/cli/channel-auth.ts:105](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/cli/channel-auth.ts#L105)
  - sha: [8f64cd3e4d83](https://github.com/openclaw/openclaw/commit/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf)
- **Missing channel plugins are installable from the channel catalog:** `resolveInstallableChannelPlugin` looks up the channel catalog, loads scoped setup plugins, and if needed calls `ensureChannelSetupPluginInstalled(...)` for the selected channel.
  - file: [src/commands/channel-setup/channel-plugin-resolution.ts:175](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/commands/channel-setup/channel-plugin-resolution.ts#L175)
