## Summary

Keep open. The reporter corrected this from a core compatibility regression to a setup mistake, but current main still turns this specific misconfiguration into a Gateway startup failure without a targeted doctor/status recovery hint.

## What This Changes

Request a friendlier diagnostic and recovery path when `hooks.transformsDir` points outside the canonical hook transforms directory, especially at workspace skill paths.

## Best Possible Solution

Keep the fail-closed transform containment rule and add a targeted doctor/status/config preflight hint that tells users to move transforms under `~/.openclaw/hooks/transforms` or use the relevant plugin webhook flow.

## Reproduction Assessment

Yes. A config with hooks enabled, a transform mapping, and an absolute `hooks.transformsDir` outside `<configDir>/hooks/transforms` reaches `resolveHooksConfig` during Gateway startup and throws the same containment error covered by existing tests.

## Solution Assessment

Yes for a diagnostic-only repair. Preserving the containment check is the narrow maintainable path; accepting workspace skill directories for dynamic imports would be the less safe alternative.
