## Summary

Keep this PR open: the additive Gateway delta field is useful, but the current SDK fallback would duplicate cumulative text when connected to an existing protocol-v4 Gateway that does not send `deltaText`.

## What This Changes

The branch adds `deltaText`/`replace` chat delta payload fields, updates Gateway state cleanup, SDK projection, protocol schemas/generated native artifacts, docs, tests, and a changelog entry.

## Best Possible Solution

Keep the additive `deltaText` contract, but preserve cumulative-message fallback for existing v4 Gateways unless maintainers intentionally move the incompatible behavior behind a new protocol version.

## Reproduction Assessment

Yes. Source inspection on current main shows protocol-v4 chat deltas only carry cumulative `message` snapshots, and the PR body supplies live output for the new additive `deltaText` path.

## Solution Assessment

No. The additive field is the right direction, but the SDK fallback should still compute a suffix from cumulative `message` when `deltaText` is absent, or the protocol version must be bumped for an intentional break.

## Review Findings
