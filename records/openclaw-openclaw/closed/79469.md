## Summary

Keep open: current main still does not provide completed WebChat reasoning review. Reasoning-marked delivery payloads remain excluded from WebChat replay/display paths, and the Control UI only renders thinking that is already present on a stored message as a static block, so the requested retention/expansion behavior still needs a maintainer UX and privacy decision.

## What This Changes

Add a Control UI/WebChat control or disclosure behavior so reasoning remains reviewable after a reply completes.

## Best Possible Solution

Define one WebChat reasoning disclosure and retention contract, then update gateway replay semantics, Control UI active/completed disclosure UI, docs, and focused tests together.

## Reproduction Assessment

Yes, at source level: current docs, gateway code, and tests deliberately suppress reasoning-flagged WebChat payloads from completed replay, while the UI only displays thinking already present on stored messages. I did not run a live browser reproduction.

## Solution Assessment

Unclear until maintainers define the UX and privacy contract. A UI-only always-show toggle would not be enough because current gateway replay intentionally excludes reasoning payloads from durable WebChat content.
