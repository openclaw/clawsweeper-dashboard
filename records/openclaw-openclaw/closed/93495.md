## What This Changes

The PR adds four `openclaw cron edit --clear-*` delivery-routing flags, sends explicit `null` values in the `cron.update` delivery patch, updates cron edit tests, and documents the flags.

## Best Possible Solution

Land this narrow CLI/docs/test patch after normal maintainer review and required checks, keeping the gateway nullable patch contract as the single runtime clear path.

## Reproduction Assessment

Yes. Source inspection shows current main drops empty delivery values from the CLI patch, while the RPC merge path only clears when a nullable key is present; the PR body also includes after-fix `cron get` proof for the real CLI flow.

## Solution Assessment

Yes. Adding explicit clear flags is the narrowest maintainable fix because the gateway and agent cron tool already use nullable delivery clears; treating empty strings as clears would change existing no-op CLI behavior.
