## Summary

Close #51055 as duplicate/superseded by the active fix PR #50968, not as implemented on main. Current main still hardcodes the LiteLLM and copilot-proxy defaults, while #50968 remains open and specifically tracks the same remaining work: probing the proxy model-info endpoint for per-model context and output-token limits. The other related PR, #51238, was closed unmerged, so #50968 is the canonical thread.

## Best Possible Solution

Close #51055 as superseded by #50968 and keep the remaining work tracked there. The product/code path should add LiteLLM-compatible `/v1/model/info` discovery for LiteLLM and copilot-proxy onboarding, map `max_input_tokens` to `contextWindow` and `max_output_tokens` to `maxTokens`, keep the 128k/8192 fallback for unavailable endpoints, and add regression coverage for larger-context models.
