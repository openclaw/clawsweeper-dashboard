## Label Justifications

- `P1`: The report describes a current released agent-to-agent messaging failure for Anthropic/Pi-backed agents, which breaks a real agent workflow.
- `impact:message-loss`: The failure rejects outbound `sessions_send` calls before delivery, dropping the intended inter-session message.

## Summary

Keep open: current `main` and the latest release still require the canonical `message` field for `sessions_send`, while the related alias fix only covers the outbound `message` action. The failure is source-reproducible at the schema/validation boundary, so this is a narrow core bug rather than stale cleanup.

## What This Changes

The issue asks for `send_to_task`/`sessions_send` to normalize model-generated body aliases such as `SendMessage`, `content`, and `text` to `message` before validation rejects the call.

## Best Possible Solution

Add a narrow `sessions_send` argument-normalization path before validation, preserve canonical `message` as the primary contract, and cover aliases with focused sessions-send regression tests.

## Reproduction Assessment

Yes for source-level reproduction: current `sessions_send` requires `message`, has no alias normalization, and agent-core validates before execute. I did not run the full live Anthropic/Pi A2A scenario in this read-only review.

## Solution Assessment

Mostly yes: the requested alias normalization is the right narrow fix, but it must happen before schema validation, not only inside `execute`. A small shared helper or `prepareArguments` path with direct execute safety would be the maintainable version.
