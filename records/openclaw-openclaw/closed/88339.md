## Summary

Keep open: current main and the reported release intentionally route canonical `openai/*` agent turns through the Codex runtime/Codex Responses path, and I did not find an implemented ChatGPT conversation-backend route for this requested quota split.

## What This Changes

The issue asks for `openai/gpt-5.5` with ChatGPT Pro OAuth to use the ChatGPT conversation quota while `openai-codex/gpt-5.4` continues using the Codex quota.

## Best Possible Solution

Maintainers should decide whether ChatGPT conversation-backend routing is a supported OpenAI OAuth path, then document or implement that route without disturbing the existing Codex app-server default for `openai/*`.

## Reproduction Assessment
