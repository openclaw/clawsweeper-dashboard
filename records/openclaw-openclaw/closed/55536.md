## Summary

Current main already has native model/provider failover with auth-profile cooldown state, model-scoped rate-limit cooldowns, cooldown skipping/probing, success reset, and documented configuration knobs. The exact proposed `models.circuitBreaker` config shape was not adopted, but the observable problem is handled through `agents.defaults.model.fallbacks` plus `auth.cooldowns`.

## Best Possible Solution

Close this issue as implemented on current main and keep the shipped `auth.cooldowns` plus model-fallback design. If maintainers want the exact proposed config spelling or a separate model-only circuit-breaker API, that should be a new, narrower product/API issue.
