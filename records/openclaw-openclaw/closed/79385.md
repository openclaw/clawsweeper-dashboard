## Summary

Keep open: the unified run-control surface is plausible, but PR head still lets one-shot `thinking: "off"` fall through the model default-reasoning path because the explicitness guard ignores `opts.thinkingLevelOverride`.

## What This Changes

This PR adds one-shot thinking/fast-mode controls across chat directives, `infer model run`, Talk/voice realtime consult config, gateway protocol models, docs, tests, and changelog.

## Best Possible Solution

Keep the unified run-control PR open, patch the explicitness guard for one-shot `off`, and land it with focused regression coverage plus the existing docs/changelog surface.

## Reproduction Assessment

Yes for the review finding by source inspection: PR head resolves the one-shot thinking override, but the explicit-thinking guard excludes that override. The existing target-session test contract shows explicit `off` must not call the model reasoning default.

## Solution Assessment

No, not yet. The combined approach is reasonable, but the narrow maintainable fix is to include the normalized one-shot override in `thinkingExplicitlySet` and add regression coverage in `get-reply-directives.target-session.test.ts`.
