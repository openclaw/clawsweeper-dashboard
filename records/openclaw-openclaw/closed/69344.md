## Summary

Keep this PR open for revision. The bug it targets is real, but the current diff weakens an existing narrow-terminal guard and uses an over-broad punctuation exemption that can disable wrapping for non-code tokens; open PR #39565 overlaps with the safer dotted-token direction but does not fully supersede this PR.

## What This Changes

The PR raises TUI long-token chunking from 32 to 128 characters, adds a punctuation-based copy-sensitive exemption for code-like fragments, and updates formatter tests.

## Best Possible Solution

Keep the generic 32-character wrapping guard for ordinary long tokens, add a constrained copy-sensitive classifier for real code/file/entity fragments, and cover both preserved code-like examples and generic overflow protection in `src/tui/tui-formatters.test.ts`. Coordinate with #39565 if its narrower dotted-token classifier becomes the canonical implementation.

## Reproduction Assessment

Yes. The behavior follows directly from the regex thresholds: current main chunks `b`.repeat(90) to <=32, while this PR's 129+ regex would leave that ordinary 90-character token unchanged; a >128-character token with incidental punctuation would also hit the new copy-sensitive branch and skip chunking.

## Solution Assessment

No. The PR addresses copy/paste corruption by weakening a broad layout guard; the safer solution is a narrower code/file-like classifier that preserves truly copy-sensitive fragments while retaining wrapping for generic long text.
