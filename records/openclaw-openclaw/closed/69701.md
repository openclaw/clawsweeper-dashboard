## What This Changes

The branch adds a Windows-only early return in `resolveGatewayListenHosts`, win32/non-win32 listen-host tests, and an Unreleased changelog entry.

## Best Possible Solution

Land this narrow listen-host guard after normal CI/Testbox gates, or use an equivalent Windows-specific fix that avoids binding both loopback families on the same gateway port while preserving non-Windows behavior.

## Reproduction Assessment

Yes at the source level: default loopback can still resolve to both `127.0.0.1` and `::1`, and gateway startup listens each returned host. This review did not run a fresh live Windows repro, so it is source-reproducible rather than live-reproduced.

## Solution Assessment

Yes. The PR is a narrow platform guard with focused tests; if Windows `::1` support is required later, that should be a separate IPv6-only socket design rather than expanding this fix.
