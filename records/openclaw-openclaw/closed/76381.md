## Summary

Current main already has a TTL-aware, file-backed GitHub Copilot API token cache on the shared provider-auth path used by GitHub Copilot model catalog setup and runtime auth, and regression coverage proves warm cache hits skip the token exchange.

## What This Changes

The issue requests caching the GitHub token to Copilot API token exchange so warm github-copilot agent runs avoid repeated multi-second model-resolution latency.

## Best Possible Solution

Keep the shared provider-auth cache; if warm Windows npm runs still spend seconds in model-resolution, track that as a fresh report with cache-file and stage-specific timing evidence.

## Reproduction Assessment

No; no GitHub Copilot credential was present in the environment or profile for a live Windows/npm timing run. Source and regression tests instead show the current-main token-exchange path should hit cache on warm runs.

## Solution Assessment

Yes; the shared TTL-aware resolver is the narrow maintainable solution because both model catalog setup and runtime auth already call it, avoiding provider-specific duplicate caching logic.
