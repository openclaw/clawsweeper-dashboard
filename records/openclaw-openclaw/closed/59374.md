## Summary

Close as ClawHub/plugin scope. The request is a Tavily-specific key-rotation/failover feature, OpenClaw's vision keeps optional capability out of core when plugin paths work, and an existing community Tavily plugin already advertises automatic API key rotation on HTTP 429 with comma-separated keys.

## Evidence

- **Scope anchor:** VISION.md says core should stay lean, optional capability should usually ship as plugins, and the bar for adding optional plugins to core is intentionally high.
  - file: [VISION.md:47](https://github.com/openclaw/openclaw/blob/d6ef1fcf24eea3a5985d653a3baf400e32aa1b75/VISION.md#L47)
  - sha: [d6ef1fcf24ee](https://github.com/openclaw/openclaw/commit/d6ef1fcf24eea3a5985d653a3baf400e32aa1b75)
- **Current bundled Tavily config:** The Tavily docs describe a single apiKey under plugins.entries.tavily.config.webSearch.apiKey or the TAVILY_API_KEY environment variable; they do not document multi-key failover for the bundled provider.
  - file: [docs/tools/tavily.md:23](https://github.com/openclaw/openclaw/blob/d6ef1fcf24eea3a5985d653a3baf400e32aa1b75/docs/tools/tavily.md#L23)

## Risks / Open Questions

- The existing community plugin is third-party and not reviewed here for security or quality; users should evaluate it before enabling it.
- If maintainers want bundled Tavily to become a production failover provider, this could be reopened as an explicit maintainer-owned product decision.
