## Summary

Current main has the post-2026.3.8 CJK keyword-search fix: configurable trigram FTS, short-CJK substring fallback for two-character terms such as `法海`, regression coverage for short Chinese queries, and changelog evidence that the Memory/FTS fix shipped after the reported version.

## Best Possible Solution

Keep the shipped trigram FTS plus short-CJK fallback implementation. For affected users on older releases or existing unicode61 indexes, point them to `memorySearch.store.fts.tokenizer: "trigram"` in the memory config reference and `openclaw memory index --force` to rebuild.
