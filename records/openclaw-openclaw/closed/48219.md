## Summary

Current main already includes Telegram polling/network hardening that matches this report’s macOS/Node abort pattern: sticky IPv4 fallback across polling restarts, recoverable polling retry classification, explicit transport teardown/rebuild, bounded undici pools, and forced HTTP/1.1 for Telegram polling.
