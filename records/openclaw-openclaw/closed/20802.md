## Summary

Close as superseded: the useful mutable `llm_input`/`llm_output` subset is now tracked by the narrower hardened PR at https://github.com/openclaw/openclaw/pull/61485, while the pre-model tool-result mutation/security question is tracked by https://github.com/openclaw/openclaw/issues/79168. Current main still keeps these hooks observation-only, so this broad branch should not be treated as implemented on main.

## What This Changes

The PR changes plugin hook contracts, hook runners, embedded agent call sites, adapter-side tool execution, and tests so `llm_input`, `llm_output`, and adapter-side `after_tool_call` can return modifications.

## Best Possible Solution

Review or land a focused hardened mutable `llm_input`/`llm_output` implementation, and handle pre-model tool-result mutation separately with explicit API, security policy, validation, docs, and live proof.

No. The feature direction is useful, but this broad branch mixes LLM prompt/output mutation with tool-result mutation; the narrower maintainable path is https://github.com/openclaw/openclaw/pull/61485 plus the separate security/API discussion at https://github.com/openclaw/openclaw/issues/79168.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Block prompt overrides from bypassing hook policy:** `src/plugins/types.ts:1563-1565`
  - body: Adding `prompt` and `systemPrompt` result fields makes `llm_input` a prompt-injection surface, but the patch does not extend the existing `allowPromptInjection=false` constraints to this hook. A plugin blocked from prompt injection through the documented policy could still rewrite the final prompt here.
  - confidence: 0.9
- **[P1] Re-resolve images after prompt rewrites:** `src/agents/pi-embedded-runner/run/attempt.ts:2839-2840`
  - body: `effectivePrompt` is replaced after prompt-local image detection already ran, so a redaction plugin can remove or change image references while the original image attachments still go to the model. Re-detect images or roll back the override when the prompt changes.
  - confidence: 0.91
- **[P2] Clear stale assistant data on output replacement:** `src/agents/pi-embedded-runner/run/attempt.ts:3177-3178`
  - body: When `llm_output` replaces `assistantTexts`, the original `lastAssistant` is still returned downstream. That leaves original assistant content, reasoning, or error metadata reachable after a plugin requested output redaction.
  - confidence: 0.84
- **[P2] Keep after_tool_call dispatch single-owned:** `src/agents/pi-tool-definition-adapter.ts:177-184`
  - body: The adapter now calls `after_tool_call` while the subscription handler still fires the same hook for embedded tool completion, only through a void runner. That changes the current single-owner contract and can invoke the same plugin hook twice for one embedded tool call.
  - confidence: 0.82
- **[P2] Validate replacement tool results before returning them:** `src/agents/pi-tool-definition-adapter.ts:185-187`
  - body: The adapter trusts `hookResult.result` with a direct cast to `AgentToolResult`. A malformed plugin result can bypass the normal tool-result normalization boundary before being sent back to the model.
  - confidence: 0.79

  - confidence: 0.84
