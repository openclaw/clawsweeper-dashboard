## Summary

The request is for additive spend-observability/reporting that current OpenClaw scope documents direct to plugins/ClawHub, not new bundled core behavior. Current main already exposes the core primitives a plugin would need: plugin hooks, diagnostic event subscription, `model.usage` events with `channel`/session/model/cost fields, and built-in session-log cost summaries.
