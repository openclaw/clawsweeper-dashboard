## Summary

Current `main` already implements the requested suppression control for user-facing tool error warnings. OpenClaw now exposes `messages.suppressToolErrors`, applies it in embedded-run payload shaping, covers the behavior with tests, and the changelog shows it shipped in `v2026.2.14`.

## Evidence

- **Config surface:** `MessagesConfig` defines `messages.suppressToolErrors` as a user-facing config knob to suppress tool-error warnings.
  - file: [src/config/types.messages.ts:121](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/src/config/types.messages.ts#L121)
  - command: `sed -n '110,145p' src/config/types.messages.ts`

  - command: `sed -n '220,380p' src/agents/pi-embedded-runner/run/payloads.ts`

## Risks / Open Questions

- Current implementation does not add a custom replacement message or per-agent override; it implements suppression of non-mutating warnings, while mutating tool failures remain visible by design.
