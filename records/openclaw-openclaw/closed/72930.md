## Summary

Keep open. Current main still has a Feishu setup/status probe path that inspects only top-level credentials, and open PR #74060 remains the linked closing fix candidate with regression coverage.

## What This Changes

Fix Feishu status/health reporting so multi-account configs with credentials under `channels.feishu.accounts.<id>` are recognized and probed as configured.

## Best Possible Solution

Land PR #74060 or an equivalent Feishu plugin change that resolves the selected account before setup/status probing and keeps account-scoped regression coverage.

## Reproduction Assessment

Yes. A config with `channels.feishu.defaultAccount` and credentials only under `channels.feishu.accounts.<id>` exercises the source-level mismatch: configured detection can pass, while the setup/status probe still inspects only top-level Feishu credentials.

## Solution Assessment

Yes. Reusing the existing Feishu account resolver for the setup/status probe is the narrow plugin-owned fix; core health already has generic account-aware seams.
