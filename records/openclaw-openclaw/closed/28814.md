## Summary

Current `main` already implements default cron staggering for top-of-hour recurring jobs, persists/migrates the stagger setting, and exposes `--exact` / `--stagger` overrides. The changelog entry is under `## Unreleased`, so this appears fixed on `main` but not yet in the latest tagged release `v2026.4.22`.
