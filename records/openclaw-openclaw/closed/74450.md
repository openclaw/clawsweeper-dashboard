## Summary

Current main and the latest shipped release already cover the failing path: the Control UI bootstrap fetch sends configured shared-secret credentials and retries alternate credentials on auth failures, while the gateway also accepts valid paired operator device tokens for the config endpoint. Regression coverage exists for gateway-token bootstrap auth, retry after 401, and paired operator-token access returning 200.

## What This Changes

The issue asks for GET /__openclaw/control-ui-config.json to authenticate with the configured gateway shared-secret token instead of failing with 401 after sending a cached operator/device token.

## Best Possible Solution

Keep the shipped Control UI/gateway auth behavior: bootstrap config fetches should continue using available shared-secret credentials with bounded retry, and the gateway should continue accepting scoped paired operator tokens for read-only Control UI HTTP routes so paired dashboards do not regress.
