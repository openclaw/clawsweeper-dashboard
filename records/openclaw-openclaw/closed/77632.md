## What This Changes

The PR hoists Matrix plugin runtime dependencies into the root manifest and changes dependency-contract tests to require hoisting for bundled plugins.

## Best Possible Solution

Keep Matrix on the current external ClawHub/npm plugin path with plugin-local dependencies and the existing guardrail that prevents root dependency creep.

No current-main failure path remains: source inspection shows the root package excludes Matrix dist chunks and current docs/catalog route Matrix through the external plugin install path. The original 2026.5.x bundled-root failure is supported by the PR’s logs, but it is obsolete against current main.

## Solution Assessment

No for this PR now. Hoisting Matrix dependencies was aimed at the old bundled package shape, but current main’s externalized plugin path is the narrower maintainable solution and this branch would reverse that dependency boundary.
