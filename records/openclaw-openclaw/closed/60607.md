## What This Changes

The issue asks OpenClaw to avoid silent successful completions by sending a fallback or error message when an agent run produces no replyable payload.

## Best Possible Solution

Keep the current main fallback behavior and let any remaining channel-specific no-visible delivery gaps be tracked in narrower channel/plugin issues.

## Reproduction Assessment

No current-main failing path was established for the reported Feishu symptom. Source inspection shows current main now turns the zero-visible dispatch result into a Feishu fallback notice unless the silence is intentional or policy-denied.

## Solution Assessment

Yes for the reported path: the current implementation preserves intentional silence and message-tool-only suppression while adding a fallback for unintentional no-visible completion.
