## What This Changes

Fix bundled plugin runtime dependency materialization so a package-level dependency superset is treated as already staged instead of re-running installs on each plugin activation.

## Best Possible Solution

Resolve the linked PR path: either merge the subset-based fallback fix or close that PR as superseded by the equivalent main-branch implementation, ensure a focused package.json-fallback regression is present, then close this issue once the fix is merged and release provenance is clear.

## Reproduction Assessment

Yes. A focused fixture can create an install root whose normal package.json contains a dependency superset, install only the requested package sentinels in node_modules, and call `isRuntimeDepsPlanMaterialized()` or `ensureBundledPluginRuntimeDeps()` to verify that activation does not reinstall.

## Solution Assessment

Yes. Subset matching for package-level specs while retaining `hasSatisfiedInstallSpecPackages()` is the narrow maintainable fix: it avoids strict array equality without trusting package.json alone.
