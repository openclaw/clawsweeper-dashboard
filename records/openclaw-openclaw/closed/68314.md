## Summary

Close as implemented on current main. The reported false missing-lockfile path was tied to the Discord plugin's former `@buape/carbon` dependency chain; current main removed that dependency and the two optional lockfile entries named in the repro are no longer present in `pnpm-lock.yaml`. The proof fix is on main after the latest release, so no shipped release tag is available yet.

## What This Changes

The issue requests that the source checkout install path `pnpm install --no-optional` stop reporting missing lockfile entries for optional dependencies that are already present in `pnpm-lock.yaml`.

## Best Possible Solution

Keep the current Discord dependency split and trimmed lockfile on main; if a different optional-pruned install failure appears later, track it as a fresh issue with the new dependency names and pnpm version.
