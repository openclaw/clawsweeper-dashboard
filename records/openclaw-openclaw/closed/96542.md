## Summary

Keep open: current main and the latest release still use the colliding cwd-to-session-dir encoding and cwd-blind recent-session selection. An open linked PR is a candidate fix, so this issue should close only after a compatibility-safe fix merges.

## What This Changes

The issue requests fixing project-scoped sessions so two distinct cwds such as `/home/alice/dev/client/app` and `/home/alice/dev/client-app` cannot share a session directory or silently resume each other's transcripts.

## Best Possible Solution

Land one compatibility-safe session-manager fix that prevents project-scoped resume/list from adopting a transcript whose header cwd differs from the requested cwd, while preserving or deliberately migrating shipped session directories.

## Reproduction Assessment

Yes at source level: current main still maps the two example cwds to the same directory and resumes the newest valid JSONL file without checking header cwd. I did not run an artifact-writing repro in this read-only review, but the issue body provides a concrete real-file repro against the same code path.

## Solution Assessment

Yes. The right boundary is the shared session directory/recent-session selection path; a header-cwd filter is the narrowest shipped-directory-safe approach, while an encoding change needs explicit migration or compatibility proof.
