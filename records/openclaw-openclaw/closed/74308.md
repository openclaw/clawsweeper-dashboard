## Summary

Keep open: current main still documents the worker-owned `openclaw message send` completion model, while this PR proposes a different parent-owned stdout/logs contract. The diff is focused and does not add supply-chain or code-execution surface, but it changes bundled skill operating policy and has a small result-template consistency review comment that should be handled by a maintainer or the contributor before merge.

## What This Changes

This PR rewrites `skills/coding-agent/SKILL.md` so background coding-agent workers print a structured `CODING_AGENT_RESULT` stdout block while the parent assistant captures the `sessionId`, inspects process logs/status, and reports completion or failure itself.

## Best Possible Solution

Align the bundled coding-agent skill with the intended completion policy. If parent-owned completion is the desired contract, land a focused docs update after normalizing the result-block examples; otherwise keep the direct-send model and close the PR with that explicit policy decision.
