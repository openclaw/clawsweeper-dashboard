## Summary

The report does not describe a current reproducible gateway freeze. The reporter later confirmed the breakage was caused by Slack channel-name allowlist entries after Slack routing became ID-first by default; switching `channels.slack.channels` entries from names to stable channel IDs restored normal behavior immediately. Current main also documents that unresolved channel-name entries are ignored for routing unless `channels.slack.dangerouslyAllowNameMatching` is enabled, and the Slack doctor/audit surfaces now warn on mutable allowlist entries.
