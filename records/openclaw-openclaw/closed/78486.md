## What This Changes

The PR adds an `agents.defaults.revertAfterImageModel` config flag plus reply-runner/schema/help changes to clear auto session model overrides after claimed image or PDF analysis.

## Best Possible Solution

If maintainers accept the policy, implement an opt-in media/PDF-specific auto-revert from an explicit completion signal, clear and persist the full session override/auth/runtime state, and cover it with docs, tests, and real runtime proof.

## Reproduction Assessment

Not applicable as a current-main bug reproduction: current source and docs define sticky session and auto-fallback behavior. The PR defects are source-reproducible from the diff and session-store merge contract.

## Solution Assessment

No. The PR uses a broad post-run condition and a partial persisted patch; the safer solution is a media/PDF-specific completion signal plus full persisted cleanup of every session field the helper clears.
