## Summary

Keep #44908 open. Current main has WhatsApp audio transcription for messages that survive group gating, but the group `requireMention` path still evaluates mentions before the audio preflight can supply a transcript, so an audio-only `<media:audio>` group message can still be stored and dropped before STT runs.

## Best Possible Solution

Keep the issue open and fix the WhatsApp plugin path: perform a minimal, authorization-checked audio preflight before `applyGroupGating` for eligible group voice notes, feed that transcript into mention detection, preserve raw `<media:audio>` for command authorization, and add a regression test proving a transcript mention passes `requireMention` while unauthorized senders do not trigger STT. Treat the separate media carryover idea as a follow-up if it remains after the pre-gating fix.
