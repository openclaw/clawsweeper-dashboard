## Summary

Current `main` already implements non-blocking `cron.run`: the gateway calls `enqueueRun`, and the cron service immediately returns `{ ok: true, enqueued: true, runId }` while the actual job runs in the background. The report matches older `2026.3.2` behavior, and the changelog says the enqueue/ack change shipped in `2026.3.7`.
