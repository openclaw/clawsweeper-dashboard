## Summary

Close as implemented on current main: the managed npm root now preserves sibling plugin dependencies and recovers missing/partial install records from the shared npm root, with regression coverage for both paths. The fix is main-only after v2026.5.2.

## What This Changes

The issue asks for external plugin install/repair to preserve all configured official npm plugins in the shared `~/.openclaw/npm` root so restoring Brave does not remove Feishu and vice versa.

## Best Possible Solution

Keep the generic managed npm root and install-ledger recovery on main and ship it in the next release; no Brave- or Feishu-specific core workaround remains necessary.

## Reproduction Assessment

Yes for the released regression from source/history rather than a current-main failure: the old install-record reader trusted `plugins/installs.json` alone, while current main reconstructs records from the managed npm root and has regression tests. Current main itself is fixed.

## Solution Assessment

Yes. Recovering install metadata from the shared npm root and installing from the full root manifest is the narrow generic fix, and it avoids Brave/Feishu-specific core logic.
