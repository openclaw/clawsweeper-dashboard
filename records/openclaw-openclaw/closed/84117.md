## Summary

Current main and the v2026.5.18 release no longer match the reported 2026.5.12 behavior: group/channel replies default to automatic visible final delivery, Feishu preserves the group chat target, and the exact `replies=0` symptom is now documented as an explicit message-tool-only configuration mode.

## What This Changes

The issue reports that Feishu group messages reach the agent and produce session output, but no group reply is sent while DMs still work.

## Best Possible Solution

Keep the v2026.5.18/current-main behavior, ask users on 2026.5.12 to upgrade, and reopen only with a latest-release repro plus full `messages.*visibleReplies` config and verbose suppressed-payload logs.

## Reproduction Assessment

Yes for the historical failure class: the source and docs show `queuedFinal=false, replies=0` occurs when group/channel visible replies are message-tool-only and the agent does not call `message(action=send)`. Current main and v2026.5.18 default normal group/channel turns back to automatic visible final delivery.

## Solution Assessment

Yes: the shipped fix keeps group/channel automatic final delivery as the compatibility default, retains `message_tool` as an explicit opt-in, and documents the exact silence symptom for configured tool-only rooms.
