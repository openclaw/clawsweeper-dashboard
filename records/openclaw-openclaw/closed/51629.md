## Summary

Current main already covers the reported failure mode: the Google provider normalizes bare Gemini API origins to /v1beta before request dispatch, the transport path builds embedded-runtime requests from that normalized base URL, and tests assert both the base URL rewrite and correct google/gemini-2.5-flash model parsing. Those code paths are already present in the shipped v2026.4.22 release and are unchanged on current main except for docs.

## Evidence

- **Google provider normalizes bare Gemini API origins to /v1beta:** The Google provider sets DEFAULT_GOOGLE_API_BASE_URL to https://generativelanguage.googleapis.com/v1beta and normalizeGoogleApiBaseUrl() rewrites https://generativelanguage.googleapis.com to /v1beta; resolveGoogleGenerativeAiTransport() applies that normalization for google-generative-ai transports.
