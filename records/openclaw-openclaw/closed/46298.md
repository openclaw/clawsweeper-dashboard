## Summary

Current `main` already handles isolated cron + `sessions_yield` follow-up work: cron delivery waits for descendant subagents, avoids prematurely marking the run complete while children are still active, and reuses descendant output for the final delivery. The same implementation is present in the released `v2026.4.22` tree.

## Evidence

- **Session-tool contract still promises follow-up turn behavior:** The docs for `sessions_yield` still describe it as ending the current turn so the next message can be the follow-up event, which matches the implemented orchestration path rather than the old immediate-termination behavior.
