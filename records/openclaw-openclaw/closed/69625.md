## Summary

The reported behavior is scoped to the third-party DingTalk channel plugin, not OpenClaw core on current main. DingTalk is documented as a community plugin, channel inbound handling is plugin-owned, and the cited `lane wait exceeded` core log path is warning-only rather than a drop/timeout path.

## Evidence

- **DingTalk is a community plugin, not a bundled core channel:** The docs list DingTalk under community-maintained plugins, with package `@largezhou/ddingtalk` and repo `github.com/largezhou/openclaw-dingtalk`.
  - file: [docs/plugins/community.md:52](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/docs/plugins/community.md#L52)
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **OpenClaw expects plugins to ship outside the core repo:** Plugin docs say plugins should be published to ClawHub or npm and installed with `openclaw plugins install <package-name>`, rather than added to the OpenClaw repository.
  - file: [docs/plugins/building-plugins.md:16](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/docs/plugins/building-plugins.md#L16)
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **Inbound message handling is channel-plugin-owned:** The channel plugin guide says the plugin receives platform messages, forwards them to OpenClaw, and that each channel plugin owns its own inbound pipeline.
  - file: [docs/plugins/sdk-channel-plugins.md:560](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/docs/plugins/sdk-channel-plugins.md#L560)
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **Current main does not drop queued work at `lane wait exceeded`:** `src/process/command-queue.ts` logs `lane wait exceeded` after a warning threshold, then immediately dequeues and runs the task. This path does not implement a queue timeout/drop on its own.
  - file: [src/process/command-queue.ts:171](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/process/command-queue.ts#L171)
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **No bundled DingTalk implementation exists on current main:** A repo search found DingTalk only in docs and routing tests, not as a bundled channel implementation under `src/` or `extensions/`.
  - command: `rg -n "dingtalk|DingTalk" -S extensions src docs test | sed -n '1,40p'`
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)

## Risks / Open Questions
