## What This Changes

The issue requests removing the provider gate so `apply_patch` can be offered to non-OpenAI providers when tool policy and config allow it.

## Best Possible Solution

Decide the provider contract first, then either make `apply_patch` provider-independent with focused runtime/policy/docs coverage or explicitly keep and document the OpenAI-only boundary.
