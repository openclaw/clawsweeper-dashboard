## Summary

Keep open. The durable delivery direction is useful, but the current PR still has source-level blockers: command delivery persists raw reply text before outbound filtering, and heartbeat replay returns stored text without preserving the original delivery route.

## What This Changes

The PR adds durable pending-final-delivery state for main and subagent sessions, restart and heartbeat replay paths, dispatch clearing, tests, and a changelog entry.

## Best Possible Solution

Keep durable retry state, but persist only route-bound sendable payloads and replay or clear them through the same delivery contract used by the original final reply.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: command delivery can store raw payload text before filtering, and heartbeat replay can return stored text without the original channel/recipient context.

## Solution Assessment

No. Durable retry is the right direction, but this patch is not the narrow maintainable fix until it preserves original routing and reuses sendable-payload filtering before persistence or replay.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Replay pending replies through the original route:** `src/auto-reply/reply/get-reply.ts:343`
  - body: The heartbeat replay path returns stored pendingFinalDeliveryText as a normal reply, while the new main-session markers do not persist or use the original delivery context. A main session reply originally targeting one channel can later be replayed through whatever route the heartbeat resolves.
  - confidence: 0.87
- **[P2] Filter command retry text through sendable payloads:** `src/agents/agent-command.ts:1258-1261`
  - body: This stores every raw payload's text before deliverAgentCommandResult builds the outbound plan. That plan suppresses reasoning payloads, so a restart or delivery failure after this write can make hidden reasoning text replay later as a plain reply.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The patch introduces concrete delivery-boundary risks around wrong-route replay and hidden reasoning text persistence.

Concerns:

- **[medium] Pending replies can replay to the wrong route:** `src/auto-reply/reply/get-reply.ts:343`
  - body: Heartbeat replay returns stored text without binding it to the original channel, target, thread, or account that failed, so retry can surface a completed reply in a different delivery context.
  - confidence: 0.84
- **[medium] Reasoning text can be persisted for replay:** `src/agents/agent-command.ts:1258`
  - body: Command retry state is captured from raw payload text rather than the outbound sendable payload plan, allowing reasoning payloads that normal delivery suppresses to become replayable.
  - confidence: 0.87
