## Summary

Current `main` already repairs invalid `openclaw.memorySearchManagerCache` singleton state before the module-level cache destructure runs, and it includes a regression test that poisons the shared global slot then verifies `getMemorySearchManager()` succeeds. That makes PR #67630 redundant for the reported `memory_search` TypeError. The fixing commit is on `main` but is not contained in any local release tag, so `fixedRelease` is unknown.
