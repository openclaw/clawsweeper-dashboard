## Summary

Keep open. Current `main` still has the shared-state gap the issue describes: channel preflight transcription can replace the user-facing body, but `applyMediaUnderstanding` rebuilds fresh attachments from plain `MsgContext` media fields, so an `alreadyTranscribed` flag from preflight does not survive into downstream audio selection.

## Evidence

- **WhatsApp code documents the unresolved shared gap:** Current `main` preserves `mediaPath`/`mediaType` after preflight transcript substitution and explicitly says avoiding a second STT pass requires shared SDK-level attachment state tracked separately.
  - file: [extensions/whatsapp/src/auto-reply/monitor/process-message.ts:251](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/extensions/whatsapp/src/auto-reply/monitor/process-message.ts#L251)
  - command: `nl -ba extensions/whatsapp/src/auto-reply/monitor/process-message.ts | sed -n '210,280p'`
  - sha: [af56926e2fc4](https://github.com/openclaw/openclaw/commit/af56926e2fc410809fcb0399435787c7fa8daef8)
- **Downstream media understanding always normalizes fresh attachments from context:** `applyMediaUnderstanding` calls `normalizeMediaAttachments(ctx)` at the start of the downstream path, so it does not receive any attachment object mutated during earlier preflight work.
  - file: [src/media-understanding/apply.ts:494](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/src/media-understanding/apply.ts#L494)

- **Attachment normalization has no place to carry preflight transcript state:** `normalizeAttachments` rebuilds attachments only from `MediaPath`/`MediaUrl`/`MediaType` and array variants on `MsgContext`; it does not ingest persisted attachment objects or any transcript-state marker.
  - file: [src/media-understanding/attachments.normalize.ts:27](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/src/media-understanding/attachments.normalize.ts#L27)

  - file: [src/media-understanding/audio-preflight.ts:62](https://github.com/openclaw/openclaw/blob/af56926e2fc410809fcb0399435787c7fa8daef8/src/media-understanding/audio-preflight.ts#L62)
