Keep open. Current main still has no agent-id-scoped restart lifecycle surface, and this is a valid core Gateway/CLI feature request that needs maintainer design rather than cleanup closure.

## What This Changes

Request for a per-agent restart command such as `openclaw agents restart <agent-id>` that can reset one agent without restarting the whole Gateway.

## Best Possible Solution

Design an additive admin-scoped Gateway lifecycle API plus CLI command for per-agent restart, with explicit soft/hard semantics, authorization, health checks, docs, and focused tests.

## Reproduction Assessment

Yes for the capability gap: current source and docs show no agent-id-scoped restart command, Gateway method, protocol schema, or handler. The runaway-provider recovery scenario is a feature use case rather than a deterministic current-main bug reproduction.
