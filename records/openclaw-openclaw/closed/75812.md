## Summary

Close as superseded by the open distributed agent runtime RFC. Current main still has only whole-Gateway restart and session/lane recovery primitives, while the remaining per-agent lifecycle design is already tracked in the broader runtime-control-plane work.

## What This Changes

The issue requests a per-agent CLI/admin restart capability so one agent can be reset without restarting the whole Gateway.

## Best Possible Solution

Keep the per-agent lifecycle work centralized in the distributed runtime/control-plane RFC and avoid adding a standalone restart command until the runtime ownership, health, lock, and authorization semantics are designed there.

## Reproduction Assessment

Yes. Source and docs show the current agents CLI has no restart subcommand, Gateway restart is global, and the runtime docs still describe one embedded agent process per Gateway.

## Solution Assessment

No, not as a standalone cleanup item. The durable fix needs the broader per-agent runtime lifecycle/control-plane design already tracked by https://github.com/openclaw/openclaw/issues/42026.
