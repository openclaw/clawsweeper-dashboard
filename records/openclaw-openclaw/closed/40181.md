## Summary

Current `main` already carries the symlink-safe media-root handling this issue requests, and the same implementation is present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).

## Evidence

- **Agent workspace is now an explicit allowed media root:** `getAgentScopedMediaLocalRoots()` adds the active agent workspace path to the allowed roots set instead of relying only on the generic state `workspace` directory, which addresses symlinked per-agent workspaces.
  - file: [src/media/local-roots.ts:54](https://github.com/openclaw/openclaw/blob/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b/src/media/local-roots.ts#L54)
  - command: `nl -ba src/media/local-roots.ts | sed -n '30,80p'`
  - sha: [b98a6a46fb54](https://github.com/openclaw/openclaw/commit/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b)
- **Containment check resolves each root through realpath:** `assertLocalMediaAllowed()` realpaths both the candidate media path and every allowed root before the `startsWith` containment check, which is the symlink-safe behavior described in the report.
  - file: [src/media/local-media-access.ts:30](https://github.com/openclaw/openclaw/blob/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b/src/media/local-media-access.ts#L30)
