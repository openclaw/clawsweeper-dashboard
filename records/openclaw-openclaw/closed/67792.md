## Summary

Keep open. Current main still builds the Telegram model button fallback label from the provider-local model id, and open draft PR #72752 is explicitly paired with this issue, so the issue should stay open until that PR is reviewed and landed or closed.

## What This Changes

The issue asks the Telegram plugin model selector to display the full provider/model ref for nested OpenRouter model ids instead of the provider-local id.

## Best Possible Solution

Review and land a narrow Telegram plugin fix that changes only the fallback display label for nested provider-local ids, while preserving display-name overrides, callback data, selection resolution, pagination, truncation, and current-model marking, with focused OpenRouter regression tests.

## Reproduction Assessment

Yes, source-level. With `provider="openrouter"`, `model="openai/gpt-5.4-mini"`, and no `modelNames` entry, current `buildModelsKeyboard` falls back to the bare model string at `extensions/telegram/src/model-buttons.ts:220`.

## Solution Assessment

Yes. The narrowest maintainable fix is in the Telegram `buildModelsKeyboard` fallback label plus focused unit coverage; the parser, callback data, and core model resolution do not need to change.
