## Summary

Current main now backfills per-channel logger and runtime entries before lifecycle handlers capture them, so deferred channel plugins no longer crash through an undefined `log` lookup. The fix is present on `main` in `4cca1b2399f12b81ae03e36d37aa763ab1cfd6a6`, but I found no containing release tag yet.

## What This Changes

The issue asks OpenClaw to stop gateway crashes caused by undefined channel loggers in async channel lifecycle error and exit handlers.

## Best Possible Solution

Keep the on-demand channel logger/runtime backfill on main and let the next release carry it; no separate OpenClaw source change remains for this issue.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a channel manager constructed without a `channelLogs` entry for a later-registered channel, followed by channel start or lifecycle failure; current main now covers that missing-map path by creating the sink before the handler runs.

## Solution Assessment

Yes. The current fix is stronger than only changing the async continuations to `log?.level?.(...)` because it preserves the formatted subsystem logger and runtime invariants while eliminating the undefined lookup.
