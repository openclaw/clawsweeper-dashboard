## Summary

Current `main` already fails Discord startup on invalid/revoked bot tokens before the gateway registers, and the Discord lifecycle now stops on fatal or reconnect-exhausted gateway errors instead of spinning indefinitely. The same behavior is present in release `v2026.4.23`.

## Evidence

- **Invalid token fails before gateway registration:** `createDiscordGatewayPlugin()` fetches Discord `/gateway/bot` with the bot token during `registerClient()`. Non-OK responses are thrown as fatal metadata errors, and only transient fetch failures use the fallback path, so a `401` does not proceed into websocket reconnects.
  - file: [extensions/discord/src/monitor/gateway-plugin.ts:274](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/extensions/discord/src/monitor/gateway-plugin.ts#L274)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Regression test covers 401 unauthorized startup:** The Discord monitor tests explicitly assert that a `401 Unauthorized` metadata response rejects startup and that the base gateway `registerClient` is never called, which blocks the reconnect loop described in the issue.
  - file: [extensions/discord/src/monitor/provider.proxy.test.ts:306](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/extensions/discord/src/monitor/provider.proxy.test.ts#L306)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Fatal gateway states stop the lifecycle:** Gateway errors classified as fatal or `reconnect-exhausted` are marked `shouldStopLifecycle: true`, and the lifecycle handler stops rather than continuing to run indefinitely.
  - file: [extensions/discord/src/monitor/gateway-supervisor.ts:89](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/extensions/discord/src/monitor/gateway-supervisor.ts#L89)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Behavior is already shipped in the latest release:** I checked the tagged release tree and found the same invalid-token startup guard and 401 regression test present in `v2026.4.23`.
  - command: `git show v2026.4.23:extensions/discord/src/monitor/gateway-plugin.ts | sed -n '113,170p;215,230p;360,410p' && git show v2026.4.23:extensions/discord/src/monitor/provider.proxy.test.ts | sed -n '178,212p;297,312p'`
