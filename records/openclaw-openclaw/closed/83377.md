## Summary

Keep open. The protected `maintainer` label blocks cleanup closing, and the current diff still has a delivery-state regression: explicit no-deliver cron runs that successfully use the message tool can be reported as undelivered.

## What This Changes

The PR adds a shared `SourceDeliveryPlan`/`SourceDeliveryOutcome` helper and rewires isolated cron delivery planning, execution, dispatch, and focused tests to use it.

## Best Possible Solution

Keep the shared plan abstraction, but model explicit no-deliver cron targets as message-tool-owned without runner fallback, add delivered/deliveryAttempted plus pending-warning regression coverage, then collect redacted live cron proof or a maintainer override.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main treats a matched message-tool send as delivered, while the PR's mode-none plan cannot mark the same send as satisfying delivery; I did not run a live cron job.

## Solution Assessment

No. The shared contract is a maintainable direction, but this implementation must preserve explicit mode-none message-tool sends as completed visible delivery while still disabling runner fallback.
