## Summary

Keep #40256 open. Current main partially added an internal cache boundary and moved some suffix content, but it still leaves channel/session-dependent Messaging and Reactions text before the large Project Context block. For local OpenAI-compatible transports, that boundary is stripped before provider delivery, so the observable prompt prefix can still change before the stable workspace context that the issue is trying to protect.

## Best Possible Solution

Keep #40256 open. The maintainer path should land a current-main version of #40296 or a fresh fix that moves volatile Messaging, Reactions, channel/session context, and other per-turn metadata below the cache-stable workspace prefix or into a separate late message. Add regression coverage for section order and provider payload stability, and separately decide how to stabilize or relocate dynamic message tool descriptions for local OpenAI-compatible/llama.cpp-style prefix caches.
