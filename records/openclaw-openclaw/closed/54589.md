## Summary

Current main already fixes the stale dated subsystem log file problem that PR #54589 targets. The PR and linked #54381 correctly identified the old cached subsystem file logger as the failure mode, but main now removes that cache, resolves the active rolling file at write time, and includes regression tests for long-lived loggers crossing dates.

## Best Possible Solution

Close this PR as obsolete on current main and keep the broader shipped logger implementation, which fixes both cached subsystem loggers and active rolling-file writes. If #54381 is still open, it can be reviewed separately against the same current-main evidence.

## Evidence

- **Subsystem logger no longer caches a file child logger:** `createSubsystemLogger()` now calls `getChildLogger({ subsystem: resolvedSubsystem })` on file writes instead of retaining a closure-scoped `fileLogger`, so long-lived subsystem logger instances no longer pin the child logger created before midnight.
