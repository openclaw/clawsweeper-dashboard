## Summary

Keep open. The reported CLI MIME bug and overall fix direction are valid and the PR includes real provider proof, but the current head is not merge-ready because it fails test type checking and still reads full local image files before the configured media size guard.

## What This Changes

The PR adds local-file MIME detection to `openclaw infer image describe` and `describe-many`, passes the detected MIME through configured and explicit model routes, and adds CLI regression tests.

## Best Possible Solution

Keep MIME propagation for local image describe inputs, but implement detection with bounded header sniffing or the existing media cache contract while preserving the current URL handling and maxBytes behavior.

## Reproduction Assessment

Yes. Source inspection shows current main omits `mime` for local image describe dispatch while the explicit image runtime defaults missing MIME to `image/jpeg`; the PR branch blockers are also confirmed by source and CI output.

## Solution Assessment

No. Passing MIME through the existing runtime parameters is the right direction, but this implementation needs a type fix and bounded MIME sniffing before merge.
