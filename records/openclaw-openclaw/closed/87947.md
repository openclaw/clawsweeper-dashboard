## Summary

Close as implemented: current main and shipped releases no longer have the reported gateway-runtime package-manager install path under root-owned bundled plugin directories, and the central dependency-handling refactor is already in stable releases.

## What This Changes

The issue requests fixing a non-root gateway startup regression where bundled plugin runtime dependency installs wrote under root-owned `dist/extensions/*/node_modules` directories and silently dropped affected plugins.

## Best Possible Solution

Keep the shipped package-manager-owned plugin dependency model and ask for a fresh current-release report only if gateway startup still launches package-manager work or writes under `dist/extensions`.

## Reproduction Assessment

Yes for the historical issue: the report includes a root-owned global install, non-root systemd-user gateway startup, and EACCES logs. No for current main: I did not run a live Debian non-root smoke, and the current source/release evidence shows the failing runtime install path is gone.

## Solution Assessment

Yes: the best current solution is the shipped package-manager-owned dependency model, with explicit plugin install/update/doctor repair points and no gateway startup writes into packaged extension directories.
