## Summary

Keep open. Current main still enforces hard-coded Claude live-session turn caps and the CLI backend config/schema only expose watchdog reliability tuning, while the latest v2026.4.29 release contains the same guard path.

## What This Changes

The issue requests configurable or higher Claude CLI live-session turn output limits to avoid `FailoverError` aborts on tool-heavy Opus 4.7 runs.

## Best Possible Solution

Add bounded, documented live-session output-limit tuning to the CLI backend reliability config, apply it to the Claude stdio parser, and cover default and operator-raised limits with focused tests plus a changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: feed a Claude live-session turn enough parsed JSONL to push `turn.rawChars` over 2 MiB or `rawLines.length` to 5,000, and current main aborts with the reported `FailoverError` format path.

## Solution Assessment

Yes. Bounded configurability is the narrowest maintainable fix; removing the guard would risk memory growth, and raising only the default would not handle different host capacities or Claude tool-output profiles.
