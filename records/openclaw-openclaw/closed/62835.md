## Summary

Current `main` already suppresses `NO_REPLY`/lead-fragment leakage in the gateway chat stream used by webchat, and the underlying fix shipped in `v2026.4.9` via commit `153e3add688521b2b0ead6e257def80d0c2fb58d`. The linked PR #62845 identified the same symptom, but the bug is already fixed on the shipped path.

## Evidence

- **Gateway chat stream suppresses silent control replies and lead fragments:** `emitChatDelta` stores the raw streamed text, drops exact suppressed control replies, drops lead fragments like `NO`/`NO_`, and strips glued `NO_REPLY...` prefixes before broadcasting to webchat clients.
