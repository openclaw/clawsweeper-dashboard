## Summary

Current `main` already resolves the reported Nextcloud Talk callback-contract mismatch by making `processMessage` explicitly support replay-status returns, and both runtime code and tests now depend on that broader contract. This PR's `Promise<void>` narrowing is obsolete against `main`.
