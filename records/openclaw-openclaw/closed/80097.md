## Summary

Current main and the latest release already use unambiguous Apple Reminders date contracts: the bundled skill documents ISO-style `--due` inputs, and the node Reminders API accepts `dueISO` and rejects non-ISO input before EventKit can locale-parse it.

## What This Changes

Request to prevent Apple Reminders reminder creation from misparsing slash-formatted due dates such as `05/06/2026 08:00 PM`.

## Best Possible Solution

Keep OpenClaw on ISO/unambiguous Reminders inputs and route any unsupported slash-date parsing behavior in `remindctl` or macOS outside the OpenClaw repo.

## Reproduction Assessment

No, I did not establish a failing current-main reproduction. Source inspection shows the current OpenClaw paths either document unambiguous formats or require `dueISO` and reject non-ISO input.

## Solution Assessment

Yes, the current solution is the best repo-side direction: ISO-only node input plus documented unambiguous skill formats avoids locale-dependent slash-date parsing rather than trying to infer regional date order.
