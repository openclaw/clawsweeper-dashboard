## Label Justifications

- `P1`: The reported bug skips CLI transcript compaction on live sessions and can lead to accumulated context and session resets for real users.
- `impact:session-state`: The failure affects transcript compaction and session continuity rather than message delivery itself.

## Summary

Current main already contains the exact initializer-before-resolve fix for the CLI compaction context-engine path, and GitHub compare shows that fix is included in the v2026.6.6 release.

## What This Changes

The issue requests fixing CLI transcript compaction so `runCliTurnCompactionLifecycle()` initializes context engines before resolving the default `legacy` engine.

## Best Possible Solution

Keep the shipped initializer-order fix and regression coverage; no new core work is needed for this report.

## Reproduction Assessment

Yes. The source path and prior PR review show the old lazy CLI compaction runtime could resolve the default context engine before the registry initialized, while current main has the init-before-resolve ordering and test coverage.

## Solution Assessment

Yes. Calling the existing idempotent initializer immediately before resolving the CLI compaction engine is the narrowest maintainable fix and matches the context-engine contract.
