## Summary

Keep open. Current `main` still does not persist WebChat queue or composer state across refresh, and PR #56542 is not merge-ready as written because its patch has an unresolved broken import and an incomplete restored-queue flow.

## Evidence

- **Current main keeps queued messages only in memory:** `enqueueChatMessage`, `flushChatQueue`, and `removeQueuedMessage` only mutate `host.chatQueue`; there is no storage-backed persistence or restore path in current `main`.
  - file: [ui/src/ui/app-chat.ts:112](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/ui/src/ui/app-chat.ts#L112)
  - command: `nl -ba ui/src/ui/app-chat.ts | sed -n '90,360p'`
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Current main keeps draft and attachments only in memory:** The render layer still uses `onDraftChange: (next) => (state.chatMessage = next)` and `onAttachmentsChange: (next) => (state.chatAttachments = next)`, with no persistence calls.
  - file: [ui/src/ui/app-render.ts:2257](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/ui/src/ui/app-render.ts#L2257)

That means the PR's 'restore history' claim overlaps existing code, but queue/draft persistence is still missing.
