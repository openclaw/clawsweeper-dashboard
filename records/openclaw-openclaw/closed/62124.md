## Summary

Current `main` already has the fail-closed gateway stage behavior this PR proposes: `runGatewayHttpRequestStages` now rethrows by default and only skips stages that explicitly opt into `continueOnError`, while the auth stages remain fail-closed and the behavior is covered by tests.
