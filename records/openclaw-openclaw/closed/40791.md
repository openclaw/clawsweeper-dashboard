Full review comments:

- **[P2] Keep the rescue runner explicitly opt-in:** `scripts/systemd/openclaw-rescue-watchdog.service:9`
  - body: The PR body and docs frame the runner as optional, but this service template exports `OPENCLAW_RESCUE_RUNNER` by default. An operator who enables the timer for monitoring will also launch Claude Code with `acceptEdits` on every incident, which expands command and file-editing surface without a separate opt-in.
  - confidence: 0.92
- **[P2] Clear duplicate suppression after a healthy probe:** `scripts/rescue-watchdog.sh:175-177`
  - body: When the probe is healthy, the script exits without clearing `state.env`. In a fail, recover, fail sequence inside the cooldown window, the second outage keeps the old fingerprint and is skipped as a duplicate even though recovery happened in between.
  - confidence: 0.9
- **[P2] Generate collision-free incident directories:** `scripts/rescue-watchdog.sh:196-198`
  - body: Incident IDs have only second-level precision and `mkdir -p` reuses an existing path. Two overlapping watchdog runs in the same second can write the same incident bundle, leaving mixed artifacts or a partial capture instead of independent incidents.
  - confidence: 0.88
- **[P2] Persist suppression state before running the rescue command:** `scripts/rescue-watchdog.sh:240-244`
  - body: `state.env` is written only after `RUNNER_CMD` returns, so a timer overlap or manual rerun during a long rescue run sees the old state and starts another runner against the same outage. Save state before launch or guard the run with a lock.
  - confidence: 0.88
