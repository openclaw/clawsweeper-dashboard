## Summary

Keep open. Current main still has a source-backed Claude CLI runtime-policy mismatch, and the linked open draft PR at https://github.com/openclaw/openclaw/pull/82546 uses closing syntax for this issue, so the issue should remain open until that PR lands or is closed.

## What This Changes

Upgrading from 2026.5.7 to 2026.5.12 should preserve Claude CLI-backed gateway, Dashboard, and Telegram turns instead of falling through to Anthropic API billing.

## Best Possible Solution

Land or replace the linked fix so Claude CLI migration/defaulting preserves provider/model-scoped claude-cli runtime policy for canonical Anthropic refs while keeping legacy whole-agent runtime keys ignored.

## Reproduction Assessment

Yes, at source level: current migration can output canonical Anthropic model entries with no model-scoped runtime while the resolver ignores agents.defaults.agentRuntime, so the CLI runtime is not selected. I did not run a live Telegram or Dashboard repro in this read-only sweep.

## Solution Assessment

Yes, the linked PR direction is the narrow maintainable fix: write and backfill model-scoped Claude CLI runtime policy through the Anthropic migration/defaulting path without re-enabling legacy whole-agent runtime selection.
