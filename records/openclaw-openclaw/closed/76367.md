## Summary

Keep open: current main still has the source-visible cold-registry doctor/status warning path, and the open PR that targets this issue has not merged.

## What This Changes

The issue asks doctor/status to stop reporting no active memory plugin when bundled memory-core is selected, loaded, and memory search works.

## Best Possible Solution

Review and land #76393 or an equivalent narrow fix that loads the configured memory-slot plugin for cold standalone doctor/status runtime resolution without broad plugin activation.

## Reproduction Assessment

Yes. Source inspection shows the exact warning is emitted when the local backend lookup returns null, and the current helper only checks already-loaded runtime registry state, which can miss selected `memory-core` in standalone CLI doctor/status paths.

## Solution Assessment

Yes. The narrow maintainable direction is to make the memory runtime bootstrap for doctor/status load only the configured memory-slot plugin when the registry is cold, rather than adding config or changing memory search behavior.
