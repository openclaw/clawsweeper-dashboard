## Summary

Close as implemented on main: current code and the shipped v2026.4.29 tag route persisted session model/provider overrides into the next agent run instead of silently using the configured primary default.

## What This Changes

Make TUI/session-level `modelOverride` and `providerOverride` take effect for subsequent agent runtime routing instead of falling back to `agents.defaults.model.primary`.

## Best Possible Solution

Keep the shipped session-override routing; users still on v2026.4.21 should upgrade to v2026.4.29 or newer and file fresh logs if a specific provider still misroutes.

## Reproduction Assessment

Yes. The issue gives a concrete TUI `/model` to `sessions.json` to `openclaw agent --session-id` path, and current source/tests cover the equivalent persisted user override routing path; no live Ollama run was executed because this review is read-only.

## Solution Assessment

Yes. The current implementation is the narrow maintainable fix: persist the session-scoped selection at the Gateway boundary, normalize it, and consume it directly before model fallback selection.
