## Summary

Current main implements the provider-scoped timeout request through `models.providers.<id>.timeoutSeconds`, which is threaded into resolved model request metadata, guarded fetch abort handling, Undici connect/header/body timeouts, OpenAI SDK request timeouts, and the LLM idle watchdog. The older PR is superseded by the safer main implementation in 18b76e399579a9fc3794beb0c64d166a7256a975, and its global-dispatcher approach had documented concurrency/reliability review concerns.

## Best Possible Solution
