## Repair Work Prompt

Fix the cron retry classifier false positive reported in https://github.com/openclaw/openclaw/issues/90947. On current main, `resolveCronExecutionRetryHint` in `src/cron/retry-hint.ts` treats any bare 500-599 number as `server_error`, causing messages like `context limit 512 exceeded`, `process exited with 503 lines of output`, `ENOENT: no such file '/var/run/app-540.sock'`, `killed worker pid 511 after deadline`, and `assertion failed: expected 500 got 0` to retry by default. Tighten only the `server_error` classification so incidental numbers are not retryable, while preserving genuine HTTP/status 5xx phrases and structured provider `server_error` classification. Add focused regression tests in `src/cron/retry-hint.test.ts`; include timer regression coverage only if the classifier-level test is insufficient. Do not add config options, change retry defaults, or edit CHANGELOG.md.
