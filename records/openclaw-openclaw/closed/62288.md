## Summary

Keep open. Current main has the managed browser as the documented/default lane and has partial DevToolsActivePort fallback text, but it still does not meet the maintainer-defined canonical acceptance bar for existing-session reason propagation, doctor/profile diagnostics, and internal-tab avoidance.

## What This Changes

The issue asks to make the managed `openclaw` browser the reliable default while hardening `profile="user"` existing-session attach fallback, diagnostics, and internal-tab handling.

## Best Possible Solution

Finish this as a narrow browser-plugin fix: preserve explicit existing-session failure reasons through status/profiles/doctor, keep managed `openclaw` as the default recommendation, and filter Chrome/DevTools/internal tabs from default selection unless explicitly targeted.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is available by mocking Chrome MCP `list_pages` with `chrome://inspect` before a real tab for default selection, then mocking DevToolsActivePort/transport failures and verifying `/profiles`, `/doctor`, and CLI output preserve actionable reasons.

## Solution Assessment

No. Current main is only a partial solution; the maintainable fix belongs in the browser plugin with focused regression tests, not in core startup migration or a broader browser redesign.
