## Summary

Current main already loads bootstrap files from the configured workspace and injects those workspace-local files at runtime. This PR only reorders template seeding lookup under `resolveWorkspaceTemplateDir`, which is not the runtime path for loading existing workspace bootstrap files, so the linked behavior is already implemented and this patch is no longer the needed fix.
