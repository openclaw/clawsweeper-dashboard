## Summary

Close as `clawhub`. Current main treats new channel support as plugin work rather than OpenClaw core, and DingTalk already has a documented community plugin path.

## Evidence

- **Vision scope keeps optional capabilities out of core:** `VISION.md:54-61` says optional capability should usually ship as plugins, plugin packages should be hosted and maintained in their own repositories, and the bar for adding optional plugins to core is intentionally high.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/VISION.md#L54)
  - sha: [7d4718373698](https://github.com/openclaw/openclaw/commit/7d471837369885c12d540d6bafa319ff41129b89)
- **Current docs already list a DingTalk plugin:** `docs/plugins/community.md:52-62` documents a DingTalk community plugin (`@largezhou/ddingtalk`) and shows the install command `openclaw plugins install @largezhou/ddingtalk`.
  - file: [docs/plugins/community.md:52](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/docs/plugins/community.md#L52)
  - sha: [7d4718373698](https://github.com/openclaw/openclaw/commit/7d471837369885c12d540d6bafa319ff41129b89)
- **Plugin SDK explicitly supports external channel plugins:** `docs/plugins/sdk-setup.md:248-284` shows channel plugins are first-class plugin manifests (`kind: "channel"`, `channels: [...]`) and should be published through ClawHub/package distribution rather than added to core.
  - file: [docs/plugins/sdk-setup.md:248](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/docs/plugins/sdk-setup.md#L248)
  - sha: [7d4718373698](https://github.com/openclaw/openclaw/commit/7d471837369885c12d540d6bafa319ff41129b89)
- **Core runtime has a dedicated channel-plugin registration seam:** `src/plugins/types.ts:1939-1940` exposes `registerChannel(...)` for native messaging channel plugins, which matches the plugin-based architecture rather than a new core `src/providers/dingtalk` implementation.
  - file: [src/plugins/types.ts:1939](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/src/plugins/types.ts#L1939)
  - sha: [7d4718373698](https://github.com/openclaw/openclaw/commit/7d471837369885c12d540d6bafa319ff41129b89)
- **Project changelog records the DingTalk plugin listing on main:** `CHANGELOG.md:2269` notes: `Docs/plugins: add the community DingTalk plugin listing to the docs catalog. (#29913)`.
