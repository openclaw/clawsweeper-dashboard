## Summary

The reported GPT-5 approval storm is already fixed for the observable path: normal exec security is now derived from configured policy and host approvals, so model-supplied `security="allowlist"` cannot combine with `ask="on-miss"` to force approvals under a configured full/off policy.

## What This Changes

The issue asks OpenClaw to prevent GPT-5 tool-call `ask` and `security` arguments from overriding configured exec approval policy on 2026.5.7.

## Best Possible Solution

Keep the shipped configured-only handling for normal exec `security`; if maintainers want `ask` removed as a per-call prompt control too, track that as a separate policy decision.

## Reproduction Assessment

No: the reported full/off plus model `security=allowlist` path no longer reproduces from current source because normal exec does not read `params.security`. Current coverage also asserts configured full/off plus request `ask=on-miss` completes without an approval request.

## Solution Assessment

Yes for the reported observable bug: keeping normal exec security under configured policy is the narrow maintainable fix. Ignoring `ask` as well would be a separate policy/API change because docs and tests still expose it as per-call prompt behavior.
