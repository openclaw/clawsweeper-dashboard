## Summary

Keep open. Current main and the latest release still only synchronously bootstrap builtin memory search when the index is empty; a populated but dirty or reindex-needed index can query immediately while search-triggered sync runs in the background, matching the reported empty-result failure mode.

## What This Changes

The issue reports builtin `memory_search` returning empty results against a populated SQLite memory index until a forced full reindex is triggered.

## Best Possible Solution

Add a builtin memory-core search preflight that detects dirty, session-dirty, or full-reindex-needed state before querying and either waits for bounded catch-up/reindex or returns an explicit transient unavailable result without changing QMD behavior.

## Reproduction Assessment

Yes by source inspection: a populated builtin index skips synchronous bootstrap, dirty search sync is started in the background, and the query proceeds against the current index. I did not run a live BigModel endpoint repro in this read-only pass.

## Solution Assessment

Yes: the narrow maintainable fix is a builtin memory-core search preflight/guard with regression coverage. The manual reindex workaround and existing background search sync do not protect the current query.
