## Summary

Current main already addresses the reported QMD duplicate/false-positive collection path: OpenClaw no longer creates the legacy `memory-alt-*` default collection, and startup now repairs stale/conflicting managed QMD collections instead of leaving the duplicate-warning state described in #58935.

## Evidence

- **Default QMD collections no longer include legacy lowercase root:** `resolveDefaultCollections` now creates only `memory-root-<agent>` for `MEMORY.md` and `memory-dir-<agent>` for `memory/**`; the companion test explicitly asserts `memory-alt-main` is absent.
