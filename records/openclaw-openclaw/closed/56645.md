## Summary

Current `main` already implements a server-driven WebSocket keepalive for gateway/control-ui connections by broadcasting periodic `tick` events every 30 seconds to all clients, including webchat. The implementation is covered by tests and post-dates the reported 2026-03-28 environment, but it does not appear in the latest tagged release `v2026.4.23` yet.
