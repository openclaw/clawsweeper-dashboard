## What This Changes

The branch relaxes read-only channel manifest gating and warning text, adds session.maintenance.preserveKeys plumbing, and adds two PR templates.

## Best Possible Solution

Implement a focused manifest-channel fallback where the read-only plugin object is built, keep diagnostics accurate, remove unrelated session/template changes, and add regression coverage for manifest-only configured channels.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a configured external channel declared only in manifest channels[] reaches addManifestChannelPlugins, but buildManifestChannelPlugin returns undefined without channelCatalogMeta or channelConfigs and the listing helper skips that undefined value.

## Solution Assessment

No. The PR's guard-only channel change is not the best fix because the downstream builder still rejects the reported manifest shape; the safer solution is a focused fallback in the builder/listing path with tests and no unrelated session or template changes.
