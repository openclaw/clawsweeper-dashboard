## Summary

Current main already implements live session model-switch handling for explicit `/model` and session-patch overrides: it persists the new selection as a pending live switch, clears stale runtime model/provider state, retargets queued followups, and restarts active runs onto the persisted provider/model instead of sticking to stale fallback/runtime identity.
