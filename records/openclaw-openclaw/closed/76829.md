## Summary

Keep open: current `main` still lets `format` failover reasons reach shared auth-profile failure recording, and open PRs #76866 and #76963 are the active implementation candidates.

## What This Changes

A corrupted embedded-agent transcript can produce Anthropic `format` errors that are recorded as shared auth-profile cooldowns, blocking unrelated requests.

## Best Possible Solution

Land one narrow fix that suppresses session-local `format` failures from shared auth-profile health while preserving real auth, billing, rate-limit, overload, and fallback behavior.

## Reproduction Assessment

Yes at source level: a `format` prompt failure can flow through `resolveAuthProfileFailureReason`, into `maybeMarkAuthProfileFailure`, and then into profile-wide cooldown state. I did not run a live Telegram/Anthropic scenario in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to filter `format` at the shared auth-profile failure boundary rather than resetting sessions globally or changing provider fallback classification broadly.
