## Summary

Close as `clawhub`: current main already exposes the resolved runtime model through generic Runtime/status surfaces, while the remaining value of this PR is provider/model-family prompt tuning that current provider plugin hooks can express outside core. The submitted patch also has an unresolved prompt-injection concern because it interpolates the runtime model string into a high-salience instruction without sanitizing it.

## Best Possible Solution

Close this core PR and leave the current generic Runtime/status behavior in place. Provider/model-family identity salience should be implemented through a provider plugin or ClawHub package using `resolveSystemPromptContribution`; a core API/design discussion is only needed if that hook cannot express the desired behavior safely.

## Risks / Open Questions

- Landing the PR as written could let a custom or provider-supplied model id with control characters or markdown delimiters alter the high-salience identity instruction.
- If maintainers later decide every provider needs a universal dedicated identity sentence in core, that should be a new sanitized core change with prompt regression coverage rather than this PR as-is.
