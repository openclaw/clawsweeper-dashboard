## Summary

Current main already prevents `openclaw doctor --fix` from auto-installing a user-systemd gateway service when a system-level OpenClaw gateway service exists. The doctor flow now scans system unit directories first, emits an explicit warning, and returns before `service.install` or restart. The behavior is covered by a regression test and documented for Linux system-service installs. The guard is present in the shipped `v2026.4.26` tag and remains present on current main.

## Best Possible Solution

Close this issue as implemented by the shipped system-service guard. Keep the separate Signal multi-account doctor bug tracked independently in #62763 if it still reproduces.
