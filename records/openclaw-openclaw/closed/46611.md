## Summary

Current `main` already fixes the reported Docker/plugin-load problem through a newer, stricter SDK layout: bundled channels now load outbound send deps from `openclaw/plugin-sdk/outbound-runtime`, that subpath is exported in `package.json`, and guardrail tests now forbid both legacy `src/infra/outbound/send-deps.*` imports and `openclaw/plugin-sdk/compat` inside bundled extension sources. This PR’s compat-based approach is therefore obsolete on current `main`.

## Evidence

- **Stable SDK subpath exists for outbound send deps:** `src/plugin-sdk/outbound-runtime.ts` re-exports `resolveOutboundSendDep` and `OutboundSendDeps` from the core outbound helper, giving bundled plugins a shipped SDK path instead of a relative `src/` import.
  - file: [src/plugin-sdk/outbound-runtime.ts:1](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/src/plugin-sdk/outbound-runtime.ts#L1)
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
- **Published exports include the runtime subpath:** `package.json` exports `./plugin-sdk/outbound-runtime` and `./plugin-sdk/infra-runtime`, so the Docker/runtime package surface includes the supported path used by bundled plugins.
  - file: [package.json:240](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/package.json#L240)
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
- **Bundled channels already use the supported runtime subpath:** Current channel implementations import `resolveOutboundSendDep` from `openclaw/plugin-sdk/outbound-runtime` rather than from `src/infra/outbound/send-deps.js`, e.g. Signal, Discord, Slack, Telegram, iMessage, WhatsApp, and MSTeams.
  - file: [extensions/signal/src/channel.ts:12](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/extensions/signal/src/channel.ts#L12)
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
- **Repo guardrails now forbid the legacy import and compat fallback:** `src/channels/plugins/contracts/channel-import-guardrails.test.ts` rejects bundled extension imports from both `openclaw/plugin-sdk/compat` and `src/infra/outbound/send-deps.*`, which covers the regression this PR targeted and also blocks the PR’s compat-based implementation path.
  - file: [src/channels/plugins/contracts/channel-import-guardrails.test.ts:553](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/src/channels/plugins/contracts/channel-import-guardrails.test.ts#L553)
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
- **Compat path is now explicitly deprecated:** Current `main` treats `openclaw/plugin-sdk/compat` as legacy-only and warns new plugins away from it; the migration docs also state that `compat` is deprecated and bundled plugins should use focused SDK subpaths instead.
  - file: [src/plugin-sdk/compat.ts:1](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/src/plugin-sdk/compat.ts#L1)
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
