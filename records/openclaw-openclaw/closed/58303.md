## Summary

Current `main` already prevents fallback-driven live-session model changes from being treated as pending live switches, normalizes any remaining `LiveSessionModelSwitchError` into failover, and has regression coverage plus a changelog entry describing the shipped fix.

## Evidence

- **Only explicit user-driven switches mark pending live switch state:** Session override persistence sets `liveModelSwitchPending` only when `markLiveSwitchPending` is requested, which is the explicit model-switch path rather than automatic fallback.
  - file: [src/sessions/model-overrides.ts:117](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/sessions/model-overrides.ts#L117)

  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Session contract explicitly excludes system fallbacks:** The session entry type documents that `liveModelSwitchPending` is for explicit user-driven changes and that system-initiated fallbacks never set it, so fallback transitions are no longer mistaken for live switches.
  - file: [src/config/sessions/types.ts:189](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/config/sessions/types.ts#L189)

- **Embedded runner only throws on pending live switch flag:** `shouldSwitchToLiveModel()` is consulted during embedded runs, and `LiveSessionModelSwitchError` is thrown only when `liveModelSwitchPending` is set and the persisted selection differs from the current run.
  - file: [src/agents/pi-embedded-runner/run.ts:974](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/agents/pi-embedded-runner/run.ts#L974)

  - file: [src/agents/model-fallback.ts:921](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/agents/model-fallback.ts#L921)

  - file: [src/agents/live-model-switch.test.ts:400](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/agents/live-model-switch.test.ts#L400)
