## Summary

Keep open: current main and v2026.6.9 still contain the agent-side `web_search` provider-preference gate that can bypass live runtime provider discovery, and the direct closing fix remains open at https://github.com/openclaw/openclaw/pull/88684.

## What This Changes

The issue reports that agent-embedded `web_search` can lose the Brave runtime provider while CLI/plugin inspection still sees it loaded, requiring a gateway restart.

## Best Possible Solution

Review, prove, and land or supersede https://github.com/openclaw/openclaw/pull/88684, then close this issue from the merged fix.

## Reproduction Assessment

Yes, at source level: current main can set `preferRuntimeProviders` to false for a manifest-owned configured search provider, and `createWebSearchTool` forwards that into `runWebSearch`. I did not live-reproduce the multi-hour gateway uptime failure in this read-only review.

## Solution Assessment

Yes for the linked narrow direction, but it is not merged yet. Keeping agent `web_search` runtime-first while leaving `web_fetch`'s bundled-owner gate intact is the maintainable fix shape shown by the current source and open PR evidence.
