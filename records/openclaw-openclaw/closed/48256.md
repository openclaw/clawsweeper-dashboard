## Summary

Current `main` already preserves provider-qualified model IDs in the Control UI and sends the selected model value unchanged in `sessions.patch`; changelog entries show the fix shipped in `2026.4.2`.

## Evidence

- **Picker sends selected model unchanged:** `switchChatModel` patches `model: nextModel || null` directly, so the UI no longer prepends the current session provider when the selected option is already provider-qualified.
  - file: [ui/src/ui/chat/session-controls.ts:251](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/ui/src/ui/chat/session-controls.ts#L251)
