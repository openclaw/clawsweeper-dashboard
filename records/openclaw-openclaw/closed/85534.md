## Summary

This PR should stay open for maintainer review: it appears to be a coherent follow-up that broadens the already-merged Codex node-exec fix from the normal native Code Mode shell surface to other Codex app-server native surfaces, with no discrete blocking code finding found in this pass.

## What This Changes

The branch adds a private Codex native-execution policy that blocks Codex app-server native methods, compaction, `/btw`, native `/codex` paths, and app-server conversation bindings when effective OpenClaw exec resolves to `host=node`, while preserving OpenClaw `exec` and `process` tools for node-routed harness turns.

## Best Possible Solution

Land the private Codex policy only after maintainers explicitly approve the broader fail-closed node-exec behavior and accept OpenClaw harness tools as the supported node execution path for Codex sessions.

## Reproduction Assessment

Yes, at source/proof level. Current main only gates the normal native tool surface while direct app-server requests, native commands, `/btw`, compaction, and app-server bindings remain sandbox-only gated; the PR body also includes AWS Crabbox before/after proof for the broader policy.

## Solution Assessment

Yes, if maintainers approve the compatibility tradeoff. Keeping the policy private to Codex and preserving OpenClaw `exec`/`process` under `host=node` is a maintainable owner-boundary fix; the safer alternative is a documented compatibility mode if native app-server controls must keep working.
