## Summary

Current `main` no longer matches the reported failure mode: missing session transcript files are treated as no-history/new-session state rather than a fatal condition, and maintainer tooling now detects and prunes stale `sessions.json` entries that point at missing transcripts.

## Evidence

- **Missing transcript means no persisted history:** `sessionFileHasContent()` returns `false` when the `.jsonl` file is missing, and agent runs use that to avoid assuming prior transcript history.
  - file: [src/agents/command/attempt-execution.helpers.ts:67](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/agents/command/attempt-execution.helpers.ts#L67)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Agent run path gates history on transcript existence:** `runWithModelFallback` passes `sessionHasHistory: !isNewSession || sessionFileHasContent(sessionFile)`, so a missing transcript does not block the run path or force reuse of broken history.
  - file: [src/agents/agent-command.ts:914](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/agents/agent-command.ts#L914)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **PI runner explicitly handles nonexistent session files:** The run attempt checks whether the transcript exists, still opens the session manager, and then calls `prepareSessionManagerForRun()`. That helper documents that when the file does not exist yet, the session manager creates and flushes a new session normally.
  - file: [src/agents/pi-embedded-runner/run/attempt.ts:1043](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/agents/pi-embedded-runner/run/attempt.ts#L1043)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Missing-transcript cleanup is now built in:** Current main ships maintainer tooling that warns about recent sessions with missing transcripts and can remove stale store entries with `openclaw sessions cleanup --enforce --fix-missing`.
  - file: [src/commands/doctor-state-integrity.ts:852](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/commands/doctor-state-integrity.ts#L852)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Checkout stayed clean:** Verified `git status --porcelain` before and after inspection; no changes in the worktree.
  - command: `git status --porcelain`
