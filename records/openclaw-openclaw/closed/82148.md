## Summary

Keep this PR open, but do not merge it as-is: the native subagent path is plausible, while the ACP path still records `fastMode` only in local session state and does not apply it through the ACP runtime-control path used before the child turn.

## What This Changes

Adds a `sessions_spawn.fastMode` parameter for native subagent and ACP spawns, with docs, tests, prompt snapshots, and changelog updates.

## Best Possible Solution

Keep the native subagent session override, but either carry ACP fast-mode overrides through ACP runtimeOptions/backendExtras before the first ACP `runTurn`, or remove the ACP-facing schema/docs until that runtime contract is implemented.

## Reproduction Assessment

Yes for the PR finding by source inspection. The PR adds `fastMode` only to ACP session patch state, while current ACP initialization and turn controls apply model/thinking/timeout/backendExtras from runtime options before `runTurn`; I did not execute the branch in this read-only review.

## Solution Assessment

No. Persisting `SessionEntry.fastMode` is a narrow fit for native subagents, but ACP needs the override applied through the ACP runtime-control path or withheld from ACP docs/schema.
