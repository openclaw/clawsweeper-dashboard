## Summary

Current main and the shipped v2026.4.26 release already implement the requested behavior. Plugin runtime subagent cleanup now carries the creating plugin id, uses a synthetic admin-scoped client only for plugin-owned `sessions.delete`, and keeps generic synthetic fallback calls at `operator.write`, which addresses the memory-core dreaming cleanup failure without granting broad admin deletion to all plugin calls.

## What This Changes

The issue requests that plugin subagent cleanup have enough scoped authority to delete memory-core dreaming narrative sessions without `missing scope: operator.admin` warnings or orphaned sessions.

## Best Possible Solution

Close as implemented and direct affected operators to upgrade to v2026.4.26 or newer. Keep the owner-scoped synthetic admin cleanup model rather than widening the default synthetic plugin client to broad admin scope.
