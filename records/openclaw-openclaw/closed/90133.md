## Summary

## What This Changes

Request to add a `/sticky` command or `/btw --system` flag that stores client-supplied session instructions in the system prompt instead of transcript/message history.

## Best Possible Solution

Publish a portal-specific plugin through ClawHub that stores authorized per-session rules in plugin state and injects them through `before_prompt_build` or a context engine, while keeping `/btw` ephemeral.

## Reproduction Assessment

Not applicable; this is a feature request, not a bug. Source and docs checks confirm current `/btw` is ephemeral and no first-class `/sticky` or `/btw --system` surface exists on current main.

## Solution Assessment

Yes for cleanup: routing this out of core is the best current solution because existing plugin/session/prompt hooks can cover the portal-specific behavior without widening OpenClaw's core command and security surface. If a plugin implementation hits a concrete missing API, that should be opened as a narrower plugin API issue.
