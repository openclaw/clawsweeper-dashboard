## Summary

Close as support/config rather than an OpenClaw source bug: Telegram privacy mode controls whether Telegram delivers updates, but OpenClaw intentionally applies its own group and sender authorization before mention handling. Current source, docs, tests, and the follow-up comments point to configuration/routing evidence rather than a reproducible core failure.

## What This Changes

The issue asks Telegram bots to respond to group messages and @mentions after BotFather privacy mode is disabled, or to expose/document the group-message gate.

## Best Possible Solution

Keep the fail-closed Telegram group authorization model and route this to configuration; open a new redacted routing report only if documented group/account settings are still ignored.

## Reproduction Assessment

Yes for the observed support path: source inspection shows unconfigured or unauthorized Telegram group messages are skipped before mention handling. No high-confidence core-bug reproduction is present because the issue does not prove that an authorized group and sender are ignored on current main.

## Solution Assessment

Yes. The maintainable solution is to keep group authorization before mention dispatch and configure channels.telegram.groups plus groupPolicy/groupAllowFrom/allowFrom, not to let @mentions bypass the access gate.
