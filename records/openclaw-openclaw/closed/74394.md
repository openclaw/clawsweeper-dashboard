## Summary

Current main implements the enabled opencode-go DeepSeek V4 replay fix: the OpenCode Go plugin routes DeepSeek V4 Pro/Flash through the shared DeepSeek V4 wrapper, and that wrapper now backfills or preserves `reasoning_content` on every replayed assistant message when thinking is enabled. The fix is present on main but not proven shipped in a release; the disabled-thinking extra_body edge remains tracked separately in #74374.

## What This Changes

DeepSeek V4 Pro/Flash conversations through opencode-go should replay prior `reasoning_content` in multi-turn history so follow-up turns do not fail with DeepSeek 400 errors.

## Best Possible Solution

Keep the current main implementation for enabled DeepSeek V4 replay and use #74374 as the canonical follow-up for disabled-thinking payload ordering, rather than reopening this broader release-regression report.
