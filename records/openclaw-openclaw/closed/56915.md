## Summary

Current main still skips the reported default layout because the managed skills loader only accepts the configured root itself or immediate child directories containing `SKILL.md`; it does not descend from `~/.openclaw/skills/coze/` into `koze-retrieval/`. The issue should remain open because two open PRs with closing syntax already target this exact bug, so maintainers should review or deduplicate those candidates rather than closing the report early.

## What This Changes

The issue requests default skills discovery for nested/grouped managed skills such as `~/.openclaw/skills/coze/koze-retrieval/SKILL.md` so `openclaw skills list` and runtime loading can see them.

## Best Possible Solution

Land or adapt one bounded grouped managed-skill discovery implementation in `src/agents/skills/workspace.ts`, add regression coverage for `~/.openclaw/skills/<group>/<skill>/SKILL.md`, then let the merged PR close this issue and close or deduplicate the overlapping PR.
