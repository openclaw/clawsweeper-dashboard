## Summary

Current main does not expose a supported code path where `sessions_spawn(runtime="subagent")` executes an `agent.sh` script from `agentDir`. Native subagents are OpenClaw-managed background agent runs; external harnesses belong on the ACP runtime instead. The reported setup therefore does not reproduce as a current-main core bug.

## Evidence

- **Subagents are documented as native OpenClaw runs:** `docs/tools/subagents.md` defines sub-agents as background agent runs in their own `agent:<agentId>:subagent:<uuid>` session and explicitly says external harness sessions like Codex/Claude Code/Gemini CLI should use `sessions_spawn` with `runtime: "acp"`.
  - file: [docs/tools/subagents.md:10](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/docs/tools/subagents.md#L10)
  - command: `nl -ba docs/tools/subagents.md | sed -n '1,120p'`
  - sha: [6944d7025d9c](https://github.com/openclaw/openclaw/commit/6944d7025d9c0ba4891cbe646ab5dab039d504ca)
- **External harnesses are ACP, not native subagents:** `docs/tools/acp-agents.md` states ACP is the path for external coding harnesses and contrasts ACP sessions with native sub-agent runs.
  - file: [docs/tools/acp-agents.md:13](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/docs/tools/acp-agents.md#L13)
  - command: `nl -ba docs/tools/acp-agents.md | sed -n '1,110p'`
  - sha: [6944d7025d9c](https://github.com/openclaw/openclaw/commit/6944d7025d9c0ba4891cbe646ab5dab039d504ca)
- **Runtime config only supports embedded or ACP:** `AgentRuntimeConfig` only allows `{ type: "embedded" }` or `{ type: "acp" }`; there is no shell-script or `subagent` runtime type that would point at `agent.sh`.
  - file: [src/config/types.agents.ts:28](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/config/types.agents.ts#L28)
  - command: `nl -ba src/config/types.agents.ts | sed -n '1,60p'`
  - sha: [6944d7025d9c](https://github.com/openclaw/openclaw/commit/6944d7025d9c0ba4891cbe646ab5dab039d504ca)
- **Native subagent spawn path calls the gateway agent runtime:** `spawnSubagentDirect()` creates a child subagent session key and starts work by calling gateway method `agent` with a task message, system prompt, timeout, and metadata. There is no script execution path in this flow.
  - file: [src/agents/subagent-spawn.ts:686](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/agents/subagent-spawn.ts#L686)

  - sha: [6944d7025d9c](https://github.com/openclaw/openclaw/commit/6944d7025d9c0ba4891cbe646ab5dab039d504ca)
- **No `agent.sh` implementation exists in repo surfaces searched:** A repo search for `\bagent\.sh\b` returned no matches in `src`, `docs`, `packages`, `extensions`, or `test`, which supports that current main has no native shell-script subagent runner to reproduce against.
