## Summary

Current `main` already supports Mattermost auto-thread style replies, shipped as `channels.mattermost.replyToMode` rather than an `autoThread` boolean. The Mattermost plugin now routes channel/group replies into the triggering post's thread and forks session keys by thread root.

## Risks / Open Questions

- The shipped config name is `replyToMode`, not the issue's proposed `autoThread` boolean.
- The current implementation is plugin/account-level; the issue's optional per-group config example is not present in the inspected surface.

## Close Comment

Closing as implemented on `main` after Codex review. Mattermost now supports the requested threaded-reply behavior via `channels.mattermost.replyToMode` rather than an `autoThread` boolean: the docs describe `first`/`all` as starting a thread under the triggering post (`docs/channels/mattermost.md:140`), the config/type surface exposes the option (`extensions/mattermost/src/config-schema-core.ts:102`, `extensions/mattermost/src/types.ts:56`), and the runtime/test coverage shows replies and session keys being scoped to the Mattermost thread root (`extensions/mattermost/src/mattermost/monitor.ts:342`, `extensions/mattermost/src/mattermost/monitor.test.ts:458`). This is already present in the released `v2026.4.22` tag (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). If you still need a separate per-group toggle or an `autoThread` alias specifically, please open a new focused feature request for that config surface.
