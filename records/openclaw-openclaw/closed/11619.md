## Summary

The request is a valid idea, but current main’s documented scope and extension seams place periodic memory persistence in plugin/community-plugin territory rather than OpenClaw core.

## Evidence

- **Core scope prefers plugins for optional capability:** VISION says core should stay lean, optional capability should usually ship as plugins, and memory is an exclusive plugin slot.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/VISION.md#L54)
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **Bundled session-memory is intentionally narrow:** The bundled hook is documented as saving session context only when `/new` or `/reset` is issued.
  - file: [src/hooks/bundled/session-memory/HOOK.md:19](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/hooks/bundled/session-memory/HOOK.md#L19)
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **Current implementation only fires on reset/new:** The handler exits unless the event is a `command` with action `new` or `reset`, with no periodic/token autosave path in core.
  - file: [src/hooks/bundled/session-memory/handler.ts:53](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/hooks/bundled/session-memory/handler.ts#L53)
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)
- **Plugin SDK already exposes the needed extension seams:** OpenClaw plugins can register event hooks and background services, and memory plugins can register memory capability and pre-compaction flush behavior.
  - file: [src/plugins/types.ts:1928](https://github.com/openclaw/openclaw/blob/86f69ba5a06ae256906c3a26308801e0b84e0091/src/plugins/types.ts#L1928)
  - sha: [86f69ba5a06a](https://github.com/openclaw/openclaw/commit/86f69ba5a06ae256906c3a26308801e0b84e0091)

## Risks / Open Questions

- Some users may still want a bundled default autosave policy; closing this as scope does not solve that product demand.
- The issue body proposes hook config in core terms, so closing should clearly point future work toward a plugin/community-plugin implementation path.
