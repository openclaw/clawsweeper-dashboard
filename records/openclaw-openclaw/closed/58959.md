## Summary

Issue #58959 is a ClawHub hosted-service account/OAuth problem, not an OpenClaw `main` code issue. Current `main` only contains ClawHub registry client/install flows, and repo scope docs place ClawHub behavior outside OpenClaw core.

## Evidence

- **Issue asks for service-side account reset:** The report says deleting the ClawHub account after a GitHub username rename caused `Sign in with GitHub` on `clawhub.ai` to loop, and explicitly asks maintainers to clear/reset a stale ClawHub auth/account record tied to the reporter's numeric GitHub identity. That is a hosted-service/account-state problem, not a local OpenClaw runtime behavior.
- **VISION scopes ClawHub outside core:** `main` explicitly says plugin discovery, publisher status, provenance, and security review live in ClawHub, and that OpenClaw should document extension points rather than absorb ClawHub responsibilities.
  - file: [VISION.md:72](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/VISION.md#L72)
