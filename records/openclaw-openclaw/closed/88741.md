## Label Justifications

- `P1`: The issue reports a channel-visible reasoning leak in normal agent replies, which is an urgent user-facing messaging and privacy failure.
- `impact:security`: The reported behavior can expose raw model deliberation and potentially sensitive tool-use reasoning to chat recipients.
- `impact:message-loss`: The parser is routing content to the wrong presentation stream, causing thinking text to be duplicated into channel-visible reply text.
- `impact:session-state`: The issue also concerns persisted assistant content parts containing duplicated text and thinking blocks in session history.

## Summary

Keep open. Current main mitigates the default bundled MiniMax Anthropic route, but the OpenAI-compatible stream parser still routes raw string `delta.content` to visible text before separately emitting reasoning fields as thinking, so the reported parser boundary remains actionable for mirrored or inline tagged reasoning.

## What This Changes

The issue asks OpenClaw to stop duplicating inline `<think>` reasoning into channel-visible text parts while still preserving it as thinking content.

## Best Possible Solution

Add stateful OpenAI-compatible stream partitioning that removes or de-dupes inline `<think>/<reasoning>` spans from text, emits reasoning only as thinking, and covers both active and legacy parser paths with regression tests.

## Reproduction Assessment

Yes at source level, but not as a live MiniMax run. A streamed OpenAI-compatible chunk with `delta.content` containing `<think>` text and/or matching `reasoning_content` would be appended to text and thinking by the current parser paths.

## Solution Assessment

Yes, the best fix belongs in the stream parser/normalizer rather than only in channel sanitizers. Channel sanitization is useful defense in depth, but it does not prevent duplicated session content or reasoning from entering text parts.
