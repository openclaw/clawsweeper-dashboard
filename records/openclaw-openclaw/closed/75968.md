## Summary

Current main already addresses the specific coupled failure path: bundled plugin runtime-dep staging has been removed from the plugin loader path, and `sessions.list` now uses an async, bounded, yielding session-row builder with targeted coverage. The proof is on `main` only; I could not prove it is included in a shipped tag newer than `v2026.4.29`.

## What This Changes

The issue reports that `v2026.4.29` blocks the Gateway event loop because inbound messages synchronously restage plugin runtime dependencies while Control UI `sessions.list` repeatedly hydrates large session transcripts.

## Best Possible Solution

Keep the main-branch runtime/session mitigations, and use #75703 for any remaining ARM64 event-loop starvation that persists without this specific coupling.

## Reproduction Assessment

Yes. The report gives a concrete stable-release reproduction path and current main has source/test evidence for the two implicated hot paths, though this read-only review did not run a live macOS/M4 gateway.

## Solution Assessment

Yes. Removing per-turn bundled runtime-dep staging and making `sessions.list` bounded and yielding is the narrow maintainable fix for this specific event-loop exhaustion path; remaining platform-wide starvation belongs in the broader tracker.
