## Summary

#69589 reports a real current-main bug, but the same remaining work is already tracked by open PR #69664. Current `openclaw.mjs` still lacks Bun `err.specifier` handling for direct optional import misses, and PR #69664 is the active unmerged fix for exactly that bootstrap path.

## Best Possible Solution

Consolidate tracking on PR #69664 and land that PR or an equivalent narrow bootstrap fix with a regression test for Bun-style `ERR_MODULE_NOT_FOUND` objects using `err.specifier`.

## Evidence

- **Current main lacks Bun error-shape handling:** `isDirectModuleNotFoundError` recognizes `ERR_MODULE_NOT_FOUND` through Node-style `err.url` or resolved absolute-path message checks, but it does not inspect Bun's `err.specifier`.
  - file: [openclaw.mjs:53](https://github.com/openclaw/openclaw/blob/ee2ab9a6449219e1c425308a4832ddbb7eeb0ca2/openclaw.mjs#L53)

  - file: [openclaw.mjs:71](https://github.com/openclaw/openclaw/blob/ee2ab9a6449219e1c425308a4832ddbb7eeb0ca2/openclaw.mjs#L71)
