## Summary

Current `main` already implements ACP one-shot child-result routing back to the requester via the background-task completion path. The fix landed after the reported `2026.3.13` environment in commit `564f820efa2bb81ae49b8f55a93e41492ad31917`, which added ACP background task creation, progress/terminal updates, requester-channel delivery, and docs describing `sessions_spawn({ runtime: "acp", mode: "run" })` as a parent-owned completion flow rather than a direct inline return.
