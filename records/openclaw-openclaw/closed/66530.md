## Summary

Current `main` already treats runtime fallback model selection as transient: after a fallback turn succeeds, the session stores that override as `auto`, and the next turn clears the auto override and retries the configured primary model. The same logic and test coverage are present in release `v2026.4.23`, so this request is already implemented and shipped.

## Evidence

- **Primary model is retried on the next turn:** `createModelSelectionState()` explicitly treats auto-failover overrides as transient, clears the stored fallback override, writes the cleared session state back, and resets in-memory provider/model to the configured default primary.
