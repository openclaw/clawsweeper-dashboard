## Summary

Current `main` already treats transient network failures like `ENETUNREACH` as non-fatal, tags Telegram polling transport errors, and restarts the Telegram polling loop instead of letting the gateway process exit. The same implementation is present in release `v2026.4.23`.

## Evidence
