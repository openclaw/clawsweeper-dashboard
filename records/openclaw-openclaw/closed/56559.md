## Summary

Current main already wraps the default `read` tool with adaptive paging when `limit` is omitted, and regression coverage explicitly exercises `read(path)` without a `limit`. The same implementation is already present in release `v2026.4.23`.

## Evidence

- **Missing-limit read path is handled in current main:** `createOpenClawReadTool()` routes every `read` call through `executeReadWithAdaptivePaging()`. That helper checks whether `limit` is explicitly provided; if not, it starts from `offset=1` and pages file content instead of forwarding an undefined-limit call to the upstream tool.
  - file: [src/agents/pi-tools.read.ts:206](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/agents/pi-tools.read.ts#L206)
- **Default coding tools use the wrapped read tool:** `createOpenClawCodingTools()` replaces the raw upstream `createReadTool()` with `createOpenClawReadTool(...)` for host workspaces, so normal `default_api.read(...)` calls go through the safe wrapper.
  - file: [src/agents/pi-tools.ts:473](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/agents/pi-tools.ts#L473)
- **Regression test covers `read(path)` without `limit`:** The test `auto-pages read output across chunks when context window budget allows` calls `readTool.execute(..., { path: "big.txt" })` with no `limit` and asserts it returns file content through the last line, with no fallback-state payload.
  - file: [src/agents/pi-tools.create-openclaw-coding-tools.adds-claude-style-aliases-schemas-without-dropping-g.test.ts:48](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/agents/pi-tools.create-openclaw-coding-tools.adds-claude-style-aliases-schemas-without-dropping-g.test.ts#L48)
- **Fix is already in the latest release:** `git diff --name-only v2026.4.23..HEAD -- src/agents/pi-tools.read.ts src/agents/pi-tools.create-openclaw-coding-tools.adds-claude-style-aliases-schemas-without-dropping-g.test.ts` returned no output, so the shipped `v2026.4.23` release already contains the same implementation and regression test.
  - command: `git diff --name-only v2026.4.23..HEAD -- src/agents/pi-tools.read.ts src/agents/pi-tools.create-openclaw-coding-tools.adds-claude-style-aliases-schemas-without-dropping-g.test.ts`
