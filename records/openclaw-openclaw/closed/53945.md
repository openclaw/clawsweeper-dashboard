## Summary

Current main already solves the observable multi-gateway targeting problem without adding an agent-specific `--url` flag: `openclaw agent` uses the shared Gateway RPC path, which resolves the active config, honors `gateway.remote.url` in remote mode, and uses `gateway.port`/`OPENCLAW_GATEWAY_PORT` for local multi-gateway profiles. The useful Discord mention workaround from the issue remains valid but is no longer the required core path for config-driven gateway targeting.

## Best Possible Solution

Keep the shipped config-driven targeting model: use `gateway.mode="remote"` plus `gateway.remote.url` for remote Gateway clients, or isolated profiles/config/state with unique `gateway.port` values for local multi-gateway setups. Add an agent-specific `--url` only if maintainers later want explicit CLI parity with lower-level Gateway commands.
