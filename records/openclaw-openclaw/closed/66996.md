## Summary

Keep open. Current `main` does contain a macOS-local MLX Talk path, but the user-facing docs and config surface still do not clearly document how to select it (`talk.provider: "mlx"` / provider-owned `modelId`) or what model repo it expects. From this Linux checkout I could verify code paths and tests, but not reproduce the macOS runtime behavior itself, so this is not a safe conservative auto-close.
