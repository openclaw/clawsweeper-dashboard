## Summary

Keep open: current main still has a source-reproducible agent-switch bug, and https://github.com/openclaw/openclaw/pull/81858 is an open linked fix candidate with closing syntax. The issue should not be closed until that PR, or an equivalent maintainer replacement, lands.

## What This Changes

The issue asks for the Control UI session picker to stop presenting sub-agent sessions as normal choices and to avoid retaining or selecting a sub-agent session when switching agents.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81858, or an equivalent maintainer replacement, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes. Source inspection shows a concrete current-main reproduction: seed `sessionsResult.sessions` with a target-agent subagent row newer than the target agent's main/dashboard row, and `resolvePreferredSessionForAgent` can return the subagent key.

## Solution Assessment

Yes for the open PR direction, not for a literal always-reset-to-main change. The maintainable fix is to exclude subagent/internal runtime rows from automatic selection and normal picker options while preserving saved dashboard sessions as documented.
