## Summary

Current main has removed the expensive plugin metadata freshness path that matched the reported node_modules-style doctor stall, and the added regression coverage protects the fast memoized path; the fix is only on main so far, not in v2026.5.22.

## What This Changes

The issue reports that `openclaw doctor` in v2026.5.22 stalls before showing the initial doctor UI while scanning the distribution or node_modules on a low-CPU ARM device.

## Best Possible Solution

Keep the current main plugin metadata memoization behavior and ship it in the next release so doctor no longer pays the v2026.5.22 per-plugin file scan cost.

## Reproduction Assessment

Yes. The report gives a concrete v2026.5.22 doctor startup path with `strace` evidence, and the release-to-main diff shows the removed plugin metadata freshness scan that matches the observed distribution/node_modules traversal.

## Solution Assessment

Yes. The main-branch solution moves plugin metadata freshness to the persisted registry envelope and reload boundary, which fits the repository rule that plugin metadata is process-stable rather than re-polled on hot paths.
