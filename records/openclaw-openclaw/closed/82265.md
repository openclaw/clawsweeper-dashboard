## Summary

Keep open: current main still only switches Active Memory defaults for the exact `memory-lancedb` slot, and the issue is paired with an open closing PR that should be reviewed before the issue is closed.

## What This Changes

Configure Active Memory so `memory-lancedb-pro` uses the `memory_recall` default instead of falling back to `memory_search` and `memory_get`.

## Best Possible Solution

Review the linked PR and either land the narrow slot-default fix with its regression test or replace it with a capability-driven default that derives the recall tool from the selected memory plugin.

## Reproduction Assessment

Yes, source-reproducible: with `plugins.slots.memory = "memory-lancedb-pro"` and no `config.toolsAllow`, current main takes the `memory_search`/`memory_get` default branch. If the selected plugin only registers `memory_recall`, the embedded allowlist has no matching callable tools.

## Solution Assessment

Unclear for final merge: the linked PR is a narrow fix for the known `memory-lancedb-pro` slot, while registered-tool discovery would be more future-proof. Because an open closing PR already exists, the right next step is PR review rather than closing or starting a separate repair job.
