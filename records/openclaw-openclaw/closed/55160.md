## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Return the current targeted wake result:** `src/infra/heartbeat-runner.ts:1250`
  - body: After deferred replay, `ran` can already be true before the newly requested targeted wake runs. If that current call fails or skips, this return still reports `ran`, so callers lose the actual result for the requested wake.
  - confidence: 0.88
- **[P1] Recheck deferred cron replay before future schedule slots:** `src/cron/service/timer.ts:625-638`
  - body: Deferred recurring jobs keep a future `nextRunAtMs`, but replay checks are only armed when no next wake exists. That can delay replay until the next normal cron slot instead of the phase transition that makes the job allowed.
  - confidence: 0.86
- **[P2] Preserve global FIFO for deferred cron runs:** `src/cron/service/timer.ts:600-610`
  - body: The replay builder sorts by each job's first deferred timestamp and then appends all runs for that job. Multiple deferred runs from one job can jump ahead of older deferred runs from another job, breaking the promised FIFO ordering.
  - confidence: 0.84
- **[P2] Bound or coalesce blocked heartbeat deferrals:** `src/infra/heartbeat-runner.ts:1263-1264`
  - body: Every blocked non-interval wake can append to `deferredRuns`. Repeated targeted, hook, or action wakes during a long blocked phase can grow memory and replay a large burst later; add a cap, dedupe, or coalescing rule.
  - confidence: 0.83
- **[P2] Reject invalid maintenance agent IDs:** `src/infra/maintenance-phase.ts:143-146`
  - body: The allowlist normalizes configured entries through `normalizeAgentId`, which can map malformed values to `main`. A typo can silently make the default agent maintenance-only, so invalid IDs should be rejected or ignored before normalization.
  - confidence: 0.87
- **[P2] Restart schedulers when user timezone changes:** `src/gateway/config-reload-plan.ts:78-82`
  - body: Maintenance windows using timezone `user` depend on `agents.defaults.userTimezone`, but this reload rule only watches `cron.maintenance`. Changing the user timezone can leave cron and heartbeat evaluating with stale inputs.
  - confidence: 0.82
- **[P3] Use the tick snapshot for replay gating:** `src/infra/heartbeat-runner.ts:1060`
  - body: `replayDeferredRuns` accepts a `now` snapshot but calls `Date.now()` for the maintenance check. Long replay loops can evaluate a different phase from the rest of the tick; use the passed snapshot consistently.
  - confidence: 0.73
