## Close Comment

Closing from a Codex review because current `main` already has this in place. `docs/automation/hooks.md:38-50` documents `message:received`, `src/auto-reply/reply/dispatch-from-config.ts:575-597` emits both the plugin `message_received` hook and the internal `message:received` hook on inbound turns, and `src/auto-reply/reply/dispatch-from-config.test.ts:2439-2471` covers that wiring. The Slack thread-context use case is also now handled directly: `docs/channels/slack.md:517-519` documents `channels.slack.thread.initialHistoryLimit`, `extensions/slack/src/monitor/message-handler/prepare-thread-context.ts:211-233` builds `threadHistoryBody`, and `extensions/slack/src/monitor/message-handler/prepare.test.ts:448-486` verifies new thread sessions get prior replies injected. I also checked the `v2026.4.22` tag and it already contains the same hook/docs wiring. If you still need a stricter blocking or mutating pre-processing contract beyond the current message hook model, please open a new issue scoped to that narrower gap.
