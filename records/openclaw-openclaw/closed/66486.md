## Summary

Keep open. Current `main` still has a real gap: the image-tool/media-understanding path resolves `agents.defaults.imageModel` directly from the model registry and throws `Unknown model` on miss, while the main session model path first runs provider `prepareDynamicModel`. The bundled LM Studio provider relies on that dynamic-model hook, so the reported behavior remains plausible on `main`, and the related fix PR #66622 is still only referenced as open rather than landed.

## Evidence

- **Image runtime skips dynamic-model preparation:** `resolveImageRuntime` ensures `models.json`, creates the discovery registry, and immediately does `modelRegistry.find(...)`; on miss it throws `Unknown model` without any provider dynamic-model warm-up step.
  - file: [src/media-understanding/image.ts:107](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/media-understanding/image.ts#L107)
  - command: `nl -ba src/media-understanding/image.ts | sed -n '96,126p'`
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **Primary-model resolution does prepare dynamic models first:** The embedded runner resolves provider config, calls `prepareProviderDynamicModel(...)`, and only then re-resolves the model from the registry. That is a materially different path from the image tool and matches the reporter's claim that the same model works when used as the primary model.
  - file: [src/agents/pi-embedded-runner/model.ts:810](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/agents/pi-embedded-runner/model.ts#L810)

- **LM Studio depends on dynamic-model hooks:** The bundled LM Studio provider populates cached runtime models in `prepareDynamicModel` and serves lookups from `resolveDynamicModel`, so callers that skip preparation can miss otherwise valid configured LM Studio models.
  - file: [extensions/lmstudio/index.ts:105](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/extensions/lmstudio/index.ts#L105)
