Keep open: current main still throws raw ClawHub 429 response bodies without an auth-state-aware login hint, while this active PR directly implements the missing behavior and appears functionally correct after the guarded-test repair. ## What This Changes

The PR threads ClawHub authentication state through request handling, appends a login hint only to unauthenticated 429 ClawHub errors, adds guarded regression tests, and records the fix in the changelog.

## Best Possible Solution

Land this PR or an equivalent scoped change in the shared ClawHub request/error layer so ClawHub search, detail, package archive, and skill archive failures all get consistent unauthenticated 429 guidance while authenticated and non-429 errors stay unchanged.
