## Summary

Current main already fixes the reported local `openai-codex/*` probe failure, and the fix is present in the `v2026.5.9-beta.1` pre-release. The report is accurate for stable `v2026.5.7`, which lacks the merged `src/cli/capability-cli.ts` changes, but no source fix remains on main.

## What This Changes

The issue reports that `openclaw infer model run --model openai-codex/gpt-5.5 --local` returns a generic no-text error on stable `v2026.5.7` while the same model succeeds through `--gateway`.

## Best Possible Solution

Keep the merged Codex-only local probe shim and direct stable users on `v2026.5.7` to `v2026.5.9-beta.1`, current main, or the next stable release.

## Reproduction Assessment

Yes. The `v2026.5.7` source path sends only `messages` and throws the exact generic no-text error, while current main contains the provider-gated instructions fix; I did not need an OAuth live rerun to establish the release/current-main difference.

## Solution Assessment

Yes. The current-main fix is the narrow maintainable solution because it only adds a minimal instruction for `openai-codex` / `openai-codex-responses` local probes and leaves non-Codex probes on the lean raw path.
