  - body: The protected list omits active OpenClaw bootstrap files like `AGENTS.md`, `USER.md`, `HEARTBEAT.md`, and `BOOTSTRAP.md`, even though those files are injected or loaded as workspace instructions. As written, an agent can still overwrite core instruction inputs while this guard reports the instruction-file poisoning class as covered; derive this list from the workspace/bootstrap contract and add coverage for the real names.
  - confidence: 0.94
- **[P1] Resolve tool paths before guarding aliases:** `src/agents/pi-tools.instruction-file-guard.ts:50-53`
  - body: `safeRealpath(filePath)` runs on the raw model argument before the workspace/container guard maps the path to the tool root. A relative workspace alias or sandbox container path can be checked against the gateway process cwd, pass the guard, and then be resolved by the underlying write/edit tool to a protected file; move the guard to the canonical resolved target or pass the same root/mapping into it.
  - confidence: 0.88
- **[P1] Guard resolved apply_patch targets:** `src/agents/apply-patch.ts:155-163`
  - body: The apply_patch guard checks `hunk.path` and `movePath` before `resolvePatchPath` applies cwd, sandbox, and alias resolution. A patch whose displayed path is harmless but whose resolved target is protected can bypass this security check in configurations that allow those aliases, so the protected-file check needs to run on the resolved target used for the write/remove.
  - confidence: 0.82
