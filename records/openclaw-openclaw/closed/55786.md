## Summary

Current `main` already treats Telegram reaction failures as soft results instead of tool errors, so warning-only `REACTION_INVALID` responses no longer synthesize the user-facing `Message failed` alert described in #55786.

## Evidence

- **Telegram react path returns soft results:** The Telegram action runtime documents that all `react` failures return soft `jsonResult({ ok: false, ... })` payloads instead of throwing. Warning-only invalid reactions return `{ ok: false, warning: ... }`.
  - file: [extensions/telegram/src/action-runtime.ts:227](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/extensions/telegram/src/action-runtime.ts#L227)

  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)

  - file: [src/agents/tools/common.ts:285](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/agents/tools/common.ts#L285)
