## Summary

Current main already blocks the reported non-sandbox host-file exfiltration path for `sendAttachment`/`setGroupIcon` by routing unset-`sandboxRoot` cases through host media policy and enforcing allowed local roots before any file read; regression tests cover rejection of absolute local paths without a sandbox.

## Evidence

- **Host-mode attachment policy when sandboxRoot is unset:** `resolveAttachmentMediaPolicy` returns `mode: "host"` when `sandboxRoot` is absent and carries resolved media access/local-root constraints instead of a sandbox bypass.
  - file: [src/infra/outbound/message-action-params.ts:172](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/src/infra/outbound/message-action-params.ts#L172)
  - command: `nl -ba src/infra/outbound/message-action-params.ts | sed -n '160,233p'`
- **Message actions now use host media policy for non-sandbox sends:** `runMessageAction` first normalizes params, then builds host `mediaAccess` via `resolveAgentScopedOutboundMediaAccess(...)` and passes that policy into attachment hydration when `sandboxRoot` is unset.
  - file: [src/infra/outbound/message-action-runner.ts:861](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/src/infra/outbound/message-action-runner.ts#L861)
  - command: `nl -ba src/infra/outbound/message-action-runner.ts | sed -n '860,909p'`
- **Local file reads are root-checked before hydration reads bytes:** `loadWebMedia` rejects local paths unless they are sandbox-validated or under allowed local roots, then applies extra host-read restrictions for host-local sends.
  - file: [src/media/web-media.ts:527](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/src/media/web-media.ts#L527)
  - command: `nl -ba src/media/web-media.ts | sed -n '527,580p'`
- **Regression test rejects absolute local paths without sandbox:** `expectRejectsLocalAbsolutePathWithoutSandbox` asserts `sendAttachment`/`setGroupIcon` fail on absolute host paths when no sandbox is provided, and the enclosing test applies that to `media`, `mediaUrl`, and `fileUrl`.
  - file: [src/infra/outbound/message-action-runner.media.test.ts:306](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/src/infra/outbound/message-action-runner.media.test.ts#L306)
  - command: `nl -ba src/infra/outbound/message-action-runner.media.test.ts | sed -n '306,336p'`
- **Changelog records the shipped fix:** The changelog entry under `## 2026.2.24` says: `Security/Message actions: enforce local media root checks for sendAttachment and setGroupIcon when sandboxRoot is unset...`
  - file: [CHANGELOG.md:3930](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/CHANGELOG.md#L3930)
  - command: `nl -ba CHANGELOG.md | sed -n '3930,4007p'`

## Risks / Open Questions

- Host-local attachment sends are still intentionally possible when operators disable `tools.fs.workspaceOnly`, but current main constrains them to allowed roots and buffer-verified media/safe document types rather than arbitrary host files.

## Close Comment

Closing as implemented on `main`. Codex review checked `src/infra/outbound/message-action-runner.ts:861`, `src/infra/outbound/message-action-params.ts:172`, `src/media/web-media.ts:527`, and `src/infra/outbound/message-action-runner.media.test.ts:306`. Current code no longer treats an unset `sandboxRoot` as a free local-file read path: non-sandbox attachment hydration now goes through host media policy, enforces allowed local roots before any read, and has regression coverage that absolute local paths without a sandbox are rejected for `sendAttachment`/`setGroupIcon`. The changelog also records this fix under `v2026.2.24` in `CHANGELOG.md:4006`. If you still see a bypass on a newer build, please open a new issue with the exact channel/plugin, config (especially `tools.fs.workspaceOnly`), and the attachment path form that still gets through.
