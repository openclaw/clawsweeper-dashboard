## Summary

Current main already addresses the reported Bonjour/ciao crash-loop signature from v2026.4.24. The Bonjour plugin classifies `CIAO PROBING CANCELLED`/`CIAO ANNOUNCEMENT CANCELLED` as handled ciao cancellations, registers scoped process-level handlers through the plugin SDK runtime, keeps those handlers alive across advertiser restarts, and the global unhandled-rejection handler returns before process-fatal exit when a registered handler accepts the reason. The watchdog also recreates stuck probing/announcing advertisers and disables Bonjour for the current process after repeated failed restarts instead of taking down the gateway. No release tag containing this fix is present in the checkout; the latest available tag remains v2026.4.24.

## Best Possible Solution

Keep the current main implementation: scoped ciao cancellation/error handlers stay registered across Bonjour advertiser restarts, stuck mDNS probing/announcing disables Bonjour for the current gateway process, and operators retain documented `OPENCLAW_DISABLE_BONJOUR=1`, `discovery.mdns.mode: "off"`, or plugin-disable controls. A new issue should be opened only if the same crash reproduces on a build containing ddc2036956031f812485d9b6cde0c70a9d699f58 or later.
