## Repair Work Prompt

Fix the current provider-runtime prompt-cache regression described in https://github.com/openclaw/openclaw/issues/89386. In `src/llm/providers/anthropic.ts`, preserve the existing Anthropic cache-boundary contract by splitting `context.systemPrompt` at `SYSTEM_PROMPT_CACHE_BOUNDARY` so only the stable prefix receives cache control and the dynamic suffix is sent without cache control; reuse existing helpers if practical rather than inventing a parallel parser. In `src/llm/providers/openai-completions.ts`, strip the internal cache-boundary marker from system prompts before sending and gate `prompt_cache_key` on `compat.supportsPromptCacheKey && cacheRetention !== "none"` using the existing prompt-cache-key clamp; keep `prompt_cache_retention` tied to explicit long retention and compatible support as intended by current docs. Add focused tests in the provider-runtime test files for Anthropic split behavior, OpenAI-compatible prompt_cache_key on non-OpenAI base URLs, disabled caching suppression, and marker stripping. Do not add new config flags or change provider defaults beyond restoring the documented cache behavior.
