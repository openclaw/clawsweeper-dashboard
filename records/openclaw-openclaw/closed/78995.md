## Summary

Current main and the v2026.5.6 release already provide a supported managed-service wrapper path that is written into service ProgramArguments, so operators can inject Node startup flags such as heap sizing without hand-editing generated launchd or systemd files.

## What This Changes

The issue requests a supported gateway service mechanism, preferably a config key, to pass Node heap flags to the managed Gateway process.

## Best Possible Solution

Keep the shipped wrapper-based service customization path; add a heap-specific wrapper example only if operators keep missing it, rather than adding arbitrary Node flags to core config without product and security review.

## Reproduction Assessment

Not applicable: this is a feature/configuration request rather than a broken documented behavior. Source inspection verifies the requested supported service customization path exists in current main and v2026.5.6.

## Solution Assessment

Yes: the wrapper path is the narrowest maintainable solution because it gives operators control over Node startup flags while preserving the intentional block on raw NODE_OPTIONS service environment injection.
