## Summary

Current main already fixes the reported WhatsApp voice-note ordering bug. Group audio placeholders with a saved media path now defer only the missing-mention decision, run the shared audio transcription preflight once, rerun group gating against the transcript, and pass the transcript through to the agent context. Regression tests and user-facing WhatsApp docs cover the behavior.

## Best Possible Solution

Close this issue as implemented on current main and keep the current WhatsApp auto-reply implementation: policy/allowlist gating stays before expensive STT, missing mention decisions for audio are deferred until transcription, and transcript-based gating/history behavior remains covered by regression tests. No separate core issue is needed unless a new report shows a failing live WhatsApp voice-note path on this implementation.
