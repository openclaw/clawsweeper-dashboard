## Summary

Current main already handles successful cron announce delivery as a successful cron run, and the shipped `v2026.4.22` tag contains the same implementation plus regression coverage for delivered-status persistence.

## Evidence

- **Fatal-error gating now ignores recovered payloads:** `resolveCronPayloadOutcome` only treats an error payload as fatal when no successful text payload follows it, which prevents stale delivery errors from forcing the overall cron run into `error` after a successful announce.
