## Summary

Current main already handles persisted cron jobs that omit schedule kind: the cron store loader normalizes every loaded job, the normalizer infers missing schedule.kind from legacy schedule shapes like {at: ...}, gateway tests cover add/update payloads without schedule.kind, and the scheduler now isolates malformed schedules instead of crash-looping. The same normalization files are present in the shipped v2026.4.22 release, so this appears fixed for versions newer than the reporter's 2026.3.13 build.

## Evidence
