## Summary

Keep this PR open. Current main still has the avatar override race the PR targets: delayed server avatar refresh state can remain authoritative over a browser-local assistant avatar override, and the render resolver still prefers fetched blob candidates before the local identity image. The PR remains a recent implementation candidate, with the Greptile P2 notes addressed in its later commits and no supply-chain-sensitive changes observed in the patch.

## Best Possible Solution

Review and land this PR, or an equivalent focused UI fix, so browser-local assistant avatar overrides own both assistant identity state and chat avatar render state across delayed refreshes and clear-override reloads. Keep the existing data-image/same-origin safety gates, retain the focused regression tests in `ui/src/ui/app-chat.test.ts`, `ui/src/ui/controllers/assistant-identity.test.ts`, and `ui/src/ui/views/agents-utils.test.ts`, and decide whether the two extension test-lint cleanups should stay in this branch or be split.
