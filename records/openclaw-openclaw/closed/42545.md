## Summary

Current `main` already implements the requested `/v1/chat/completions` multimodal image support, including `image_url` parsing, guarded URL/data-URI handling, forwarding to the agent as multimodal images, config knobs, tests, and a shipped changelog entry in `v2026.4.22`.

## Risks / Open Questions

- The issue title says "Images/Files", but the concrete request/body is about OpenAI chat-completions `image_url` parts. Current evidence is strong for `image_url` multimodal image support on `/v1/chat/completions`, not for generic `input_file` semantics on that older endpoint.

## Close Comment

Closing as implemented. Codex review against current `main` found that `/v1/chat/completions` now parses active-turn `image_url` parts, accepts base64 data URIs plus guarded URL sources, enforces image budgets, and forwards the result into the agent run as multimodal `images` in `src/gateway/openai-http.ts:257` and `src/gateway/openai-http.ts:552`. Regression coverage is in `src/gateway/openai-http.test.ts:360`, and the shipped changelog entry is `CHANGELOG.md:3100`. This is also present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). If you meant broader `input_file` parity beyond chat-completions `image_url`, that would be a separate follow-up.
