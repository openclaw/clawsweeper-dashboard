## Summary

Keep this open: current main still has the Telegram account-selection bug, and an active linked PR already uses closing syntax to fix it, so the issue should remain open until that PR is merged or closed.

## What This Changes

The issue reports that adding a named Telegram account can remove the implicit top-level Telegram default account from runtime polling/status enumeration.

## Best Possible Solution

Review and land the focused linked PR, then close this issue as implemented once current main preserves the implicit Telegram default account and has regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: top-level Telegram credentials plus any named or bound account make listTelegramAccountIds return only explicit IDs, and Gateway startup only starts those returned IDs.

## Solution Assessment

Yes. The narrow maintainable fix is to include the implicit default account when top-level Telegram default credentials exist; the active linked PR follows that approach and should be reviewed instead of closing the issue early.
