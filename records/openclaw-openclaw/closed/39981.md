## Summary

Current main no longer matches the reported failure mode: enabling Discord is handled as a channel-scoped hot reload, an unconfigured Discord account is marked not configured instead of taking down the gateway, and bundled Discord runtime dependencies are repaired before import with Docker-focused coverage in tests/scripts.

## Evidence

- **Discord config changes are channel-scoped hot reloads:** The Discord channel plugin declares `reload.configPrefixes: ["channels.discord"]`, and gateway reload planning maps channel config prefixes to `restart-channel:<id>` actions instead of a full gateway/container crash path.
  - file: [extensions/discord/src/shared.ts:111](https://github.com/openclaw/openclaw/blob/bfa6708c03f24c7caa5d0a96a298c292be0d85c1/extensions/discord/src/shared.ts#L111)

  - file: [src/gateway/server-channels.ts:367](https://github.com/openclaw/openclaw/blob/bfa6708c03f24c7caa5d0a96a298c292be0d85c1/src/gateway/server-channels.ts#L367)
