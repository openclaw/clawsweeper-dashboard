## Summary

Current `main` now implements post-compaction refresh injection: OpenClaw exposes `agents.defaults.compaction.postCompactionSections`, builds a post-compaction refresh prompt from AGENTS.md, and appends that prompt before the next follow-up run after compaction. That covers the underlying request, even though the final config surface is not the exact `compaction.postCompactionPrompt` name proposed in the issue.
