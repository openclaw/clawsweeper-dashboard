## Summary

Current main and the shipped v2026.4.26 release already handle the reported Discord cron DM case: explicit `user:<id>` targets are preserved through cron announce resolution and the Discord outbound send path opens a DM channel for user targets. The attached repro was from OpenClaw 2026.4.12, which is older than the verified shipped code.

## What This Changes

The issue reports that Discord cron announce delivery rewrites an explicit `user:<id>` DM target to `channel:<id>`, causing Discord delivery to fail with `Unknown Channel`.

## Best Possible Solution

Keep the shipped target-normalization path and direct affected users on 2026.4.12 to update to v2026.4.26 or later; if this reproduces on a current build, reopen with the current queue artifact and cron run log.
