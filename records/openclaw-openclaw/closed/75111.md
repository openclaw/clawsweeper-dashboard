## Summary

Keep open for revision: current main still matches the reported OAuth transport bypass, and the PR fixes the main path, but the patch introduces a metadata regression for custom/proxy Codex-compatible endpoints when payload hooks return a replacement payload.

## What This Changes

The PR preserves already-wrapped OpenAI Codex streams in the embedded runner, scopes unsupported Responses payload-field stripping for native ChatGPT/Codex backends, and stores account/plan metadata from Codex OAuth tokens with regression tests and changelog credit.

## Best Possible Solution

Revise the PR so native ChatGPT/Codex requests keep the OAuth wrapper and strip rejected fields, while custom/proxy Codex-compatible endpoints retain caller controls and transport metadata after payload hooks; then land after targeted tests, build, and optional live OAuth validation.

## Reproduction Assessment

Yes. The main bug has a high-confidence static reproduction path: current main resolves a boundary-aware OAuth-injecting Codex stream, then the attribution wrapper replaces it with a fresh Responses transport. The PR finding can be reproduced with a custom openai-codex-responses baseUrl and an onPayload hook that returns a replacement payload without metadata.

## Solution Assessment

No, not as-is. Preserving the wrapped stream is the right narrow direction, but the metadata merge guard should be scoped to the native ChatGPT Codex backend or custom endpoints should re-merge metadata before the sanitizer no-ops.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Re-merge metadata for custom Codex endpoints:** `src/agents/openai-transport-stream.ts:769-771`
  - body: This guard skips `mergeTransportMetadata` for every `openai-codex-responses` model. For a custom/proxy Codex baseUrl the new sanitizer is a no-op, so if `onPayload` returns a replacement payload, the turn metadata that current main re-adds after hooks is now dropped. Gate this on the native ChatGPT Codex backend, or re-merge before the no-op sanitizer, so custom endpoints keep metadata.
  - confidence: 0.86
