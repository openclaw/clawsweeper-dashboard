## Summary

Current main already provides the requested supported CLI/RPC path for external services to inject messages into existing sessions via `openclaw gateway call sessions.send ...`. The exact `openclaw sessions send --session-key` wrapper is not present, but the observable workflow is implemented through the Gateway call surface and documented in the protocol docs.
