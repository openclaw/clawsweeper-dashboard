## Summary

The reported Discord guild-channel behavior matches OpenClaw's documented tool-only visible-reply default, not an unhandled Discord transport failure; the repository already provides the config escape hatch and diagnostics guidance.

## What This Changes

The issue asks OpenClaw to post generated Discord channel replies visibly again after upgrading to 2026.5.7.

## Best Possible Solution

Keep the current tool-only guild/channel default and direct affected users to set messages.groupChat.visibleReplies: "automatic" or use a model that reliably calls message(action=send).

## Reproduction Assessment

Yes for the expected no-auto-post path: source inspection shows Discord guild/channel turns default to message_tool_only and suppress automatic final reply delivery unless config is set to automatic or the model calls the message tool. No high-confidence reproduction of an unintended no-typing or transport failure is available from the report because it has no logs or didSendViaMessagingTool metadata.

## Solution Assessment

Yes, closing as support/config is the narrowest maintainable outcome because current docs, source, tests, and the shipped 2026.5.7 behavior already define this policy. Reverting the guild/channel default would be a product-policy change rather than a focused bug fix.
