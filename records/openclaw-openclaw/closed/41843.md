## Summary

Current `main` does not have an `openclaw crontab` command or a tracked code path that manages the system crontab. OpenClaw scheduling is implemented as `openclaw cron` gateway jobs persisted under `~/.openclaw/cron/jobs.json`, so the reported overwrite path does not reproduce against current `main`.
