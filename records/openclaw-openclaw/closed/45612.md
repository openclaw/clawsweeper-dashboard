## Summary

Current main still uses UTC `Z` backup archive names, so the PR is not obsolete; keep it open, but it needs maintainer judgment because it changes the documented default filename contract and still lacks real `openclaw backup create` proof.

## What This Changes

The PR changes `openclaw backup` archive filenames from UTC `Z` timestamps to local-time timestamps with UTC offsets, then updates backup docs, CLI help, and formatter tests for that format.

## Best Possible Solution

Keep UTC archive basenames as the default; if local-offset names are desired, add an explicit opt-in mode with docs, focused tests for both filename shapes, and real backup-run proof.

## Reproduction Assessment

Yes for the behavior delta: current main builds default backup names through the UTC session timestamp helper, while the PR changes the default helper to local-offset formatting. I did not run the CLI because this read-only review cannot create backup artifacts in the checkout.

## Solution Assessment

No, not as an unconditional default change: the formatter itself is narrow, but replacing a documented filename shape needs maintainer acceptance or a compatibility-preserving opt-in path before merge.
