## Summary

Current `main` already contains the Control UI reconnect/session-restoration work that addresses this report: the browser gateway client now auto-reconnects with backoff after gateway disconnects, the app reconnect path clears the restart error state on `hello`, reapplies session defaults from the gateway snapshot, and reloads the session/bootstrap state needed to restore chat UI state after a restart.
