The PR changes shared and native Anthropic payload shaping so conversation cache markers prefer the latest real user text/image block through trailing tool-result loops, with fallback tool-result marking and marker-budget tests.

## Best Possible Solution

The end state should be one shared/native Anthropic payload policy that keeps conversation cache markers stable through tool-result loops while staying within Anthropic's four-breakpoint contract.

## Reproduction Assessment

## Solution Assessment

Yes, this is an acceptable best current fix: it changes the existing Anthropic payload-shaping owners, preserves tool-result fallback behavior, and bounds message markers by the Anthropic breakpoint limit. The remaining question is maintainers accepting the compatibility impact, not a concrete patch defect.
