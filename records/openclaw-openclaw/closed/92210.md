The same issue also has other open implementation PRs, so maintainers should keep the issue and candidate PRs open until one branch proves the complete runtime handoff.

## What This Changes

The PR adds a fallback in `resolveRuntimeModel` that synthesizes a minimal model for configured `agentRuntime` aliases missing from the discovered model registry.

## Best Possible Solution

Use one canonical implementation that routes `/btw` through the configured runtime’s actual execution owner, preserves sanitized no-tools side-question context, rejects genuinely unknown unconfigured models, and proves the no-direct-provider-credential setup end to end.
