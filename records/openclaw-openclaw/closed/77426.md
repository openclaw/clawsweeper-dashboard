## Summary

Keep open: current main still constructs the MCP loopback tool surface without the run-bound `sessionId` and `onYield` context that `sessions_yield` requires, and https://github.com/openclaw/openclaw/pull/77437 remains the open closing fix path with unresolved proof/review gaps.

## What This Changes

The issue requests that `sessions_yield` work on the MCP/claude-cli agent runtime path instead of always returning `No session context`.

## Best Possible Solution

Bind MCP loopback tools to the active run's yield context, pause that run through the subagent registry, add focused regression coverage, and land after redacted real claude-cli MCP proof.

## Reproduction Assessment

Yes. Source inspection on current main shows MCP loopback tools are resolved without `sessionId` or `onYield`, and `sessions_yield` deterministically returns `No session context` when `sessionId` is missing.

## Solution Assessment

Yes at the issue level. The maintainable fix is to pass run-bound yield context through the MCP loopback path with MCP-specific pause semantics; the open PR is the current vehicle but still needs revision and real proof.
