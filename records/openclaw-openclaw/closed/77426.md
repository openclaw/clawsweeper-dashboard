## Summary

Keep open: current `main` and v2026.6.6 still lack the MCP/CLI loopback session/yield context required by `sessions_yield`, and the issue is already paired with an open closing PR that has not merged.

## What This Changes

The issue asks OpenClaw to make `sessions_yield` work on the MCP/claude-cli runtime path by passing the active run session context and yield callback through gateway loopback tool resolution.

## Best Possible Solution

Refresh, review, and land https://github.com/openclaw/openclaw/pull/90861 or an equivalent narrow fix that threads run session/yield context through the MCP loopback path and preserves yielded lifecycle state.

## Reproduction Assessment

Yes, source-level. Current `main` constructs gateway/MCP tools without the `sessionId` and `onYield` fields that `sessions_yield` requires, so a loopback call reaches the documented `No session context` error path.

## Solution Assessment

Yes, the remaining fix direction is clear: update the gateway/MCP loopback context plumbing and lifecycle handling, preferably by landing the open linked PR after conflicts and maintainer review. Creating a separate automated fix PR now would duplicate that open implementation path.
