## Summary

Current main and the latest release contain a Feishu WebSocket event transport that registers `im.message.receive_v1` and routes inbound messages into Feishu sessions. The user-facing Feishu docs also now direct operators to use persistent connection/WebSocket mode for receiving messages, so the reported v2026.3.11 long-connection receive gap is covered by the shipped implementation.
