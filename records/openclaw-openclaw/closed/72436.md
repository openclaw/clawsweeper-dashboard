## Summary

Keep this PR open. Current main still lacks the central proposed behavior: an append-only `recovery-events.jsonl` sidecar next to the session store, with sanitized breadcrumbs for inbound metadata, route resolution, manual `/compact`, and outbound delivery. The PR is an active draft implementation candidate, and adjacent current-main session persistence, transcripts, and outbound delivery queue recovery do not make it obsolete. #72209 is related recovery work, but it tracks passive task/checkpoint recovery rather than this session-store-side event ledger.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to evaluate or narrow the proposed core implementation as a best-effort append-only recovery event log in `src/config/sessions`, with call sites for inbound metadata, route resolution, manual compaction, and outbound delivery. Review it alongside #72209 so OpenClaw does not grow competing recovery ledgers, and require focused tests for redaction, file permissions, non-blocking append failures, and no message-body logging before merge.
