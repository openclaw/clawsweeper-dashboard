## Summary

Keep this issue open because current main and the latest release still build prompt-preface runtime context with an unwrapped body that the user-facing sanitizer does not fully remove; the linked open fix PR should be reviewed and landed or replaced before closing.

## What This Changes

The issue reports that Feishu streaming card replies can expose internal runtime context such as relevant memories, sender metadata, and conversation metadata to users.

## Best Possible Solution

Review and land or replace the linked focused PR so the shared runtime-context builder emits a fully removable internal block for every channel, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes, source-reproducible: current main builds the prompt-preface runtime context with an unwrapped body, and the shared display sanitizer removes only the preface when no delimiters are present. I did not run a live Feishu streaming-card repro in this read-only pass.

## Solution Assessment

Yes, the best current path is the shared prompt-boundary fix represented by the linked PR, because Feishu-specific filtering would leave other user-visible surfaces dependent on heuristic stripping.
