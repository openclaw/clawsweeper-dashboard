## Summary

This PR is superseded by current main's service proxy design. The branch preserves legacy LaunchAgent `HTTP_PROXY`/`HTTPS_PROXY` style values from an existing plist, but current main intentionally routes installed-service proxy configuration through `proxy.proxyUrl` or durable `OPENCLAW_PROXY_URL`, tests that ambient proxy variables are not persisted, and audits inline proxy values as service configuration drift.

## What This Changes

The PR branch adds macOS LaunchAgent proxy-environment preservation on reinstall, tests preserve/override cases, and documents that LaunchAgents do not inherit macOS Network Settings proxy values.

## Best Possible Solution

Keep the current config-first proxy path: installed services should use `proxy.proxyUrl` or durable `OPENCLAW_PROXY_URL`, while doctor/audit continues to identify embedded legacy proxy variables for cleanup rather than preserving them across reinstalls.
