## Summary

Close as not actionable in this repository: current main and the public docs require plugin-defined keys under `plugins.entries.<id>.config`, runtime passes that object as `api.pluginConfig`, and `memory-tencentdb` is an external plugin that is not present in the OpenClaw source tree or official catalogs.

## What This Changes

The issue requests accepting memory-tencentdb config keys directly under `plugins.entries.memory-tencentdb`.

## Best Possible Solution

Keep the current OpenClaw config contract and have the external memory-tencentdb plugin document and parse `plugins.entries.memory-tencentdb.config` via `api.pluginConfig`.

## Reproduction Assessment

Yes. Source inspection shows direct sibling keys under a plugin entry are rejected by the strict core plugin-entry schema, which matches the reported error but is expected by the documented config contract.

## Solution Assessment

No. Changing OpenClaw to accept or merge arbitrary top-level plugin-specific keys would add a new compatibility mode; the maintainable path is for the external plugin to read the supported `.config` object.
