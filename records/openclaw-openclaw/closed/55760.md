## Summary

Current `main` already preserves embedded-runner auth when custom/provider-owned stream functions are selected, and the same implementation is present in release `v2026.4.22`.

## Evidence

- **Embedded runner no longer overwrites session transport with raw `streamSimple`:** `run/attempt.ts` now assigns `activeSession.agent.streamFn` via `resolveEmbeddedAgentStreamFn(...)`, passing the original session stream plus `resolvedApiKey`/`authStorage` instead of directly replacing the stream with an unauthenticated function.
