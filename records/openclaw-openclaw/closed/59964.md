- **[P1] Keep whole-file recovery out of Gateway startup/reload:** `src/gateway/server.impl.ts:469-472`
  - body: Current main and shipped docs reserve whole-file config repair for `openclaw doctor --fix`, but this branch calls `recoverConfigFile()` before startup validation and wires it into hot reload.   - confidence: 0.94

  - body: `notifyReloadFailure` uses the cached `lastWriteEvent` for every recovery alert. A later filesystem-only invalid edit can therefore send config failure details to the previous session or thread instead of the context that caused the bad write.
  - confidence: 0.91
- **[P1] Harden backup candidates before reading them:** `src/config/io.ts:992-993`
  - body: `validateRecoveryCandidate` reads `.bak` files directly, and the sync path does the same, before checking that the candidate is a regular bounded trusted file. A hostile or malformed backup can hang or exhaust startup/reload recovery before fail-closed handling runs.
  - confidence: 0.84
- **[P2] Return no selected source for invalid backups:** `src/config/io.ts:2881-2890`
  - body: When `--from` selects an invalid candidate, this branch still returns `selectedSource` and `restoredPath`. `runConfigRecover` treats `selectedSource` as the dry-run success signal, so it can report that it would recover from an unusable backup and exit successfully.
  - confidence: 0.88
- **[P2] Remove the draft proposal from docs:** `docs/PR-PROPOSAL.md:3-8`
