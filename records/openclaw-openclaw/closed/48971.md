## Best Possible Solution

Keep the shipped v2026.5.7 sanitizer from https://github.com/openclaw/openclaw/pull/72914 and close this superseded PR rather than merging the older, looser branch.

## Reproduction Assessment

Yes. The pre-fix source regex matched a valid-looking MIME prefix without validating the full string, and current main has targeted tests for malicious suffixes and malformed parameter tails; I did not run tests in this read-only review.

## Solution Assessment

No for this PR branch. Current main's shipped implementation is the better fix because it validates RFC-style parameter tokens and quoted strings instead of accepting arbitrary semicolon tails.

## Review Findings

Overall confidence: 0.9

- **[P2] Reject malformed parameter tails:** `src/media-understanding/apply.ts:81`
