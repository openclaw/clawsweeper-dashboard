## Summary

Keep open. Current main still resolves embedded compaction to one provider/model and retries summarization on that same model; the linked open PR #74470 is the active implementation candidate and uses closing syntax, so this issue should remain open until that PR is reviewed and merged or closed.

## What This Changes

Add compaction model fallback handling so Azure-filtered summarization can retry an alternate model without persisting a session model change.

## Best Possible Solution

Review and land #74470, or an equivalent narrow fix, that retries implicit embedded compaction over documented fallback candidates, keeps model selection per-attempt only, preserves explicit compaction.model behavior, and coordinates harder overflow escalation with #64962.

## Reproduction Assessment

Yes. A focused regression can configure model fallbacks, stub summarization to throw an Azure-style filtered HTTP 400 for the primary compaction model, and verify current main retries the same params.model instead of trying a fallback.

## Solution Assessment

Unclear until #74470 is reviewed. Reusing the active session fallback chain for implicit compaction looks narrower than adding a new config surface, but maintainers still need to confirm explicit compaction.model semantics and non-persistence guarantees.
