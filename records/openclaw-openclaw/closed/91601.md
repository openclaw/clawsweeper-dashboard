## Summary

Current main and released Feishu/Lark code already route agent Markdown through Feishu rich-text post payloads and interactive-card paths, so the reported raw-Markdown behavior is not an open OpenClaw source gap.

## What This Changes

The issue asks for Feishu/Lark outbound agent Markdown to be rendered as native rich text or cards instead of being sent as literal Markdown text.

## Best Possible Solution

Keep the shipped Feishu rich-text/card implementation; if DeepSeek GUI still shows raw Markdown, the next report should identify the exact bridge version and payload path that bypasses OpenClaw's Feishu outbound sender.

## Reproduction Assessment

No high-confidence current-main repro is available. Source inspection shows the current Feishu outbound path sends Markdown through rich post/card payloads rather than raw plain text.

## Solution Assessment

Yes, the current implementation is the right repository-side solution: Feishu channel ownership handles transport-specific rich post/card encoding inside the plugin boundary.
