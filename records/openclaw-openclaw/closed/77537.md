## Summary

Keep open: the PR addresses a real TUI commentary-progress gap, but current PR head still has a source-backed P2 bug where late commentary for a finalized run can reappear as visible assistant output.

## What This Changes

The PR moves TUI pending run tracking earlier, renders commentary-phase assistant agent events as live progress, adds focused TUI tests, and adds a changelog entry.

## Best Possible Solution

Land the PR after commentary rendering is limited to active, pending, or session-tracked runs, while finalized-run support remains available for late tool events only.

## Reproduction Assessment

Yes, via source inspection. Current main can drop early commentary before pendingChatRunId is set, and PR head can render assistant commentary for a finalized run because finalizedRuns passes the known-run filter before commentary rendering.

## Solution Assessment

No as proposed. The approach is narrow and maintainable, but it needs a split guard so finalized runs are accepted for late tool events without accepting finalized assistant commentary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Gate commentary to active, pending, or session-tracked runs while preserving finalized-run support for tool events.
