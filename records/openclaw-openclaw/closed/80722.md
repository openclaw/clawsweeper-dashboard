## Summary

Keep open: current `main` still emits the restart hint unconditionally for config mutations, and the open linked fix PR should be reviewed or landed before this issue closes.

## What This Changes

The issue asks `openclaw config set` to avoid telling users to restart the gateway for active-agent model config changes that apply on the next invocation.

## Best Possible Solution

Land a narrow CLI hint-selection change that keeps restart wording for restart-required paths and uses next-invocation wording only for positively classified hot-loadable agent config mutations.

## Reproduction Assessment

Yes. Source inspection on current `main` shows the CLI always appends the restart hint on config mutation success, while `agents.list` is already a hot reload path and the issue/linked PR include terminal proof of active-agent next-invocation behavior.

## Solution Assessment

Yes. The linked PR is the right implementation path because it changes only success-message hint selection after mutation classification and keeps the conservative restart hint for paths it cannot prove are hot-loadable.
