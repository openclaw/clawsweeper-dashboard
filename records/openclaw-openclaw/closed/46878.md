## Summary

Current main already avoids the reported plugin-slot false positive. The disabled note is only emitted when memory search itself is turned off, while doctor now resolves the active memory runtime/backend, cross-checks gateway memory probe results, and points users to the active-plugin `openclaw memory status --deep` flow for runtime verification.

## Evidence

- **Disabled note is now tied only to memorySearch.enabled:** `resolveMemorySearchConfig()` returns `null` only when the merged `agents.*.memorySearch.enabled` value is false; it does not inspect `plugins.slots.memory`, so an active memory plugin slot no longer forces the "explicitly disabled" path.

If no runtime is registered it reports that directly instead of saying memory search is disabled.
