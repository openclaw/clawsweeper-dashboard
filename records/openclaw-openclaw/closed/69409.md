## Summary

Current main already provides the operational gate needed to prevent unapproved restarts from direct config-file changes: `gateway.reload.mode: "hot"` applies hot-safe edits but skips restart-required edits with a warning, while `off` disables live reload entirely until an explicit manual restart. `plugins.*` changes, including `plugins.allow`, are classified as restart-required. The same behavior is present in v2026.4.24, and the later Slack socket-mode crash comment is also covered by the current bounded reconnect path.

## Best Possible Solution

Close this item and direct operators who require human-controlled restarts to set `gateway.reload.mode: "hot"` or `"off"`. Keep the current core reload-policy implementation; only open a new narrower issue if a maintainer explicitly wants a queued restart-notification workflow on top of the existing policy.
