## Summary

Close as implemented on main: OpenClaw already provides the requested per-turn pre/post model middleware surface through typed plugin hooks rather than new internal `agent:turn:*` events.

## What This Changes

The issue asks for additive per-agent-turn hook events before and after each LLM call for middleware such as compression, observability, prompt inspection, and enrichment.

## Best Possible Solution

Keep the existing typed plugin hooks as the supported per-turn middleware surface and point Compresh-style integrations to `https://docs.openclaw.ai/plugins/hooks` instead of adding duplicate internal hook event names.

## Reproduction Assessment

Not applicable; this is a feature/API request rather than a bug. Source and docs show the per-turn middleware path already exists via `before_agent_run`, `llm_input`, `llm_output`, and `agent_end`.

## Solution Assessment

Yes for the central workflow: the existing typed plugin hooks are the maintainable OpenClaw extension boundary for pre/post model middleware. Adding duplicate internal hook events with new naming or mutation semantics would be a separate product/API decision.
