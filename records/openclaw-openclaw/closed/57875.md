## Summary

Current `main` already adds Discord-specific block-stream coalescing that keeps short multi-paragraph replies buffered and merged into one outbound message instead of sending chunk-boundary fragments.

## Evidence

- **Discord now has provider-default block coalescing:** The Discord channel plugin advertises `blockStreamingCoalesceDefaults: { minChars: 1500, idleMs: 1000 }`, which means a 500-1000 character reply stays buffered and is not emitted as multiple block-stream messages.
