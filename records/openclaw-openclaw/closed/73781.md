## What This Changes

The issue requests preventing OpenClaw from replaying a previous tool call after a failure in Feishu/Qwen and similar channel runs.

## Best Possible Solution

Land or consolidate a canonical agents-runtime fix that disables Pi SDK auto-retry for OpenClaw-managed runs, preserves failed-tool context, and adds focused replay-loop regression coverage.

## Solution Assessment

Is this the best way to solve the issue? Yes: the narrow maintainable fix is to keep retry ownership in OpenClaw by disabling embedded Pi SDK auto-retry, while broader self-healing or default-on circuit-breaker policy belongs in separate trackers.
