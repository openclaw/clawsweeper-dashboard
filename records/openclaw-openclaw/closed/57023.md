## Summary

Close as implemented_on_main. Current `main` and the shipped `v2026.4.23` release both contain the cache-first facade sentinel guard and explicit circular re-entry regression tests that address the bundled plugin stack-overflow path described in #57023.

## Evidence

- **Shipped changelog entry:** The `v2026.4.23` changelog explicitly says bundled plugin facade loads are guarded with a cache-first sentinel so circular re-entry stops crashing `xai`, `sglang`, and `vllm` during gateway plugin startup.
  - file: [CHANGELOG.md:1479](https://github.com/openclaw/openclaw/blob/9b18ae877018a34933e27bb7b2986cb97e9602aa/CHANGELOG.md#L1479)

  - sha: [9b18ae877018](https://github.com/openclaw/openclaw/commit/9b18ae877018a34933e27bb7b2986cb97e9602aa)
- **Facade loader guard on current main:** The loader now inserts a sentinel into the module cache before Jiti/module evaluation, back-fills it after load, and removes it on failure. That is the concrete recursion guard against re-entrant facade loads.
  - file: [src/plugin-sdk/facade-loader.ts:201](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/plugin-sdk/facade-loader.ts#L201)

- **Public surface loader uses same pattern:** The bundled public-surface loader also seeds a sentinel before loading and clears it on failure, matching the same anti-recursion pattern in the broader plugin-loading path.
  - file: [src/plugins/public-surface-loader.ts:174](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/plugins/public-surface-loader.ts#L174)
