## What This Changes

This PR adds an immediate-timeout path for embedded-runner pending-tool-result cleanup, passes it during aborted attempts, and adds a unit test for zero and negative timeout values.

## Best Possible Solution

Merge the abort cleanup fast path after adding the required Unreleased changelog entry and trimming the test whitespace, keeping broader session-lock latency work separate.

## Reproduction Assessment

Yes. On current main, a guarded session manager plus an agent whose `waitForIdle` never resolves exercises the existing wait path before pending tool results are cleared and before the cleanup helper can release the session lock.

## Solution Assessment

Yes, with small process fixes. Reusing the existing timeout cleanup semantics for `timeoutMs <= 0` is the narrowest maintainable way to skip idle waiting only for abort cleanup while preserving the normal idle wait for successful tool completion.
