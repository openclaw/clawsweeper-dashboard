## Summary

Keep open. Current main and the latest shipped tag still stage the backup manifest under ambient `os.tmpdir()`, so a macOS service `TMPDIR` inside the state directory can put the staged manifest under a recursively archived source; the focused repair PR #75031 was closed without merging.

## What This Changes

Fix `openclaw backup create` so staging `manifest.json` under a state-local `TMPDIR` does not add duplicate root manifest entries to the backup archive.

## Best Possible Solution

Backup creation should stage its manifest in a temp root proven outside every selected backup source, keep the verifier’s exactly-one-root-manifest invariant, and add focused regression coverage for state-local `TMPDIR`.

## Reproduction Assessment

Yes. The high-confidence path is source-level: run a normal state/full backup with `TMPDIR=<stateDir>/tmp`, where the staged manifest is under the recursive state source and is also passed explicitly to tar.

## Solution Assessment

Yes. Reusing `resolvePreferredOpenClawTmpDir()` with a containment regression test is the narrow maintainable fix; backup verification should remain strict rather than accepting duplicate root manifests.
