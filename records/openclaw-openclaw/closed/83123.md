## Summary

Close as implemented on current main: the MiMo `reasoning_content` replay path for multi-turn tool calls is now handled for Xiaomi MiMo and custom OpenAI-compatible MiMo proxy routes, with regression coverage. The fix is main-only after `v2026.5.12`, so users on that release still need a newer build or the next release.

## What This Changes

The issue requests a declared strategy for replaying prior `reasoning_content` in assistant tool-call messages for reasoning-capable OpenAI-compatible models such as MiMo v2.5 Pro.

## Best Possible Solution

Keep the current provider/replay-boundary implementation and ship it in the next release; a separate public `reasoningEchoPolicy` would duplicate the main fix unless maintainers later want a broader compatibility knob.

## Reproduction Assessment

Yes for the reported release path: `v2026.5.12` lacks the replay-preservation code and the provider contract requires `reasoning_content` passback after tool calls. On current main, the failing path is covered by source and regression tests rather than reproducing as a failure.

## Solution Assessment

Yes: current main fixes the behavior at the provider/replay boundary, preserving MiMo and DeepSeek-style replay while still stripping unsupported reasoning fields for stock OpenAI. Adding the proposed new public policy field is no longer the narrowest required fix for the reported failure.
