## Summary

Close as a duplicate of the broader open macOS sqlite-vec/node:sqlite tracking issue #66977. Current main still uses the same bundled sqlite-vec loader path, and the published sqlite-vec loader is already a thin `db.loadExtension(getLoadablePath())` wrapper, so this report’s observed macOS vector degradation belongs with the existing macOS extension-loading issue rather than a separate better-sqlite3 API mismatch.

## What This Changes

Report requests fixing macOS ARM64 v2026.4.26 memory indexing so sqlite-vec loads under node:sqlite and vector recall stays available instead of degrading to FTS/BM25-only.

## Best Possible Solution

Keep #66977 as the canonical macOS/node:sqlite sqlite-vec load tracker and fold this v2026.4.26 Apple Silicon reproduction into that fix, including macOS packaging/runtime validation for extension loading.
