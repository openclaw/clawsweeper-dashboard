## Summary

Current `main` already has the control-plane routing and sandbox-policy behavior this issue was missing on `2026.3.8`: the `cron` tool now calls Gateway methods directly, explicit sandbox allow entries can remove default-denied tools like `cron`/`gateway`, and the shipped docs/runtime guidance now matches the expected chat behavior.

## Evidence

- **Cron tool is gateway-routed on current main:** The agent-side `cron` tool executes `callGateway("cron.status")`, `callGateway("cron.list")`, and `callGateway("cron.add")` rather than shelling out or persisting sandbox-local state.
