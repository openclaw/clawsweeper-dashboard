## Summary

Keep open. Current `main` does not contain the PR's process-wide shared-runtime fix, and the current gateway/plugin lifecycle still matches the reproduced multi-context `voice-call` `EADDRINUSE` path described in the PR discussion.

## Evidence

- **Current main still uses module-local voice-call runtime state:** `extensions/voice-call/index.ts` on `main` still declares `let runtimePromise` and `let runtime` inside `register(api)`, and `ensureRuntime()` only reuses that closure-local state. There is no process-wide `globalThis`/`Symbol.for(...)` sharing on current `main`.
  - file: [extensions/voice-call/index.ts:171](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/extensions/voice-call/index.ts#L171)
  - command: `nl -ba extensions/voice-call/index.ts | sed -n '150,360p'`
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Service lifecycle still binds stop/start to the same closure-local runtime:** The registered `voicecall` service starts with `ensureRuntime()` and stops by awaiting the same closure-local `runtimePromise`, then clears only those local variables. That means a second registration context would not see the already-started runtime from the first context.
  - file: [extensions/voice-call/index.ts:559](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/extensions/voice-call/index.ts#L559)
  - command: `nl -ba extensions/voice-call/index.ts | sed -n '520,620p'`
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Gateway still starts plugin services once, then later reloads deferred gateway plugins separately:** `startPluginServices(...)` is called during post-attach startup, but `server.impl.ts` later calls `reloadDeferredGatewayPlugins(...)` when deferred configured channel plugins exist. That second load path rebuilds plugin registrations after services have already been started.
  - file: [src/gateway/server-startup-post-attach.ts:285](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/gateway/server-startup-post-attach.ts#L285)

  - file: [src/gateway/server-plugin-bootstrap.ts:107](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/gateway/server-plugin-bootstrap.ts#L107)
