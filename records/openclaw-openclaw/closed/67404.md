## Summary

Keep this PR open. Current main still does not refresh QMD session exports from transcript-update events: QMD creates and indexes the managed sessions collection, but it only exports sessions during normal QMD updates, does not subscribe to transcript updates, and its filesystem watcher skips session collections. The closed predecessor #65017 is explicitly superseded by this PR, so this remains the active implementation candidate rather than obsolete work.

## Best Possible Solution

Keep this PR open, but ask for a narrower patch or split. The best implementation is to make QMD subscribe to canonical transcript-update events only when QMD session export is enabled, filter updates to the current agent's session transcript directory, debounce refreshes, use the forced queued QMD update path so transcript writes during an in-flight update are not dropped, clean up listener and timer state on close, and cover that with focused qmd-manager tests. Review the plugin-loader trust and other unrelated test-isolation changes separately.
