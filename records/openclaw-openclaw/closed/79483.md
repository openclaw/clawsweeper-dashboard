Keep open: this is a plausible focused fix for a source-reproducible Feishu group-session bug, with sufficient terminal proof and no blocking review findings; it still needs normal maintainer review and landing coordination.

## What This Changes

The PR updates shared group session resolution to prefer the originating chat target for group/channel contexts whose speaker-shaped `From` lacks an explicit group kind, adds a Feishu regression test, and adds a changelog entry.

## Best Possible Solution

Land a narrow fix that derives Feishu group session/tool context from the channel-owned chat id while preserving `From`/`SenderId` for speaker attribution.

## Reproduction Assessment

Yes, at source level. Current main Feishu group turns set `From` to `feishu:<sender>` and `OriginatingTo`/`To` to `chat:<group>`, while `resolveGroupSessionKey` derives the group id from `From`.

## Solution Assessment

Yes. Correcting the resolver to use trusted `OriginatingTo`/`To` for speaker-shaped group turns addresses the mismatch without weakening the tool-policy fail-closed guard; the overlapping `GroupId`-field alternative would need a supported context contract first.
