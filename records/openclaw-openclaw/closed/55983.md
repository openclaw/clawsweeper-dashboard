## Summary

Keep PR #55983 open. Current main still has the reported guard-ordering behavior: cross-agent targets are rejected before `tree` visibility checks spawned-session membership, while docs/config define `tree` as current session plus spawned sessions. The PR is a focused linked bug fix for open issue #54878, not obsolete cleanup. It still needs maintainer review because adjacent `session_status` and `sessions_send` behavior includes additional A2A semantics that should be decided explicitly.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to fix the shared session visibility checker so `tree` mode authorizes proven spawned-lineage children before generic cross-agent visibility rejection, add the cross-agent spawned regression test from the PR, retain a negative non-spawned cross-agent assertion, and explicitly decide/document whether spawned lineage also bypasses A2A for `sessions_send` and `session_status`. If status is intended to work for parent-owned ACP children, update `session-status-tool.ts` in addition to the shared guard.
