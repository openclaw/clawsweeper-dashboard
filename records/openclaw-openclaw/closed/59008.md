## Summary

Current `main` no longer matches the reported 2026.4.1 TUI silent-failure behavior: TUI chat finals now render provider/gateway `errorMessage` text instead of collapsing to `(no output)`, `chat` error events are surfaced in-session, and a streaming watchdog reports silently dropped runs instead of leaving the UI stuck at `connected | idle`. The changelog records the TUI error-recovery work in `v2026.4.2`.

## Evidence

- **TUI renders envelope error text on empty finals:** `resolveFinalAssistantText` returns formatted `errorMessage` text before falling back to `(no output)`, and `handleChatEvent` passes `evt.errorMessage` into final rendering.
  - file: [src/tui/tui-formatters.ts:157](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/tui/tui-formatters.ts#L157)
  - command: `nl -ba src/tui/tui-formatters.ts | sed -n '145,180p'`
- **TUI surfaces silent stalls and run errors in-session:** The streaming watchdog resets a stuck run and adds a visible system message when no deltas arrive; explicit `error` events also add `run error: ...` text instead of failing silently.
  - file: [src/tui/tui-event-handlers.ts:93](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/tui/tui-event-handlers.ts#L93)
