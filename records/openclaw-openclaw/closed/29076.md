## Summary

Close as ClawHub/plugin-scope. Current main explicitly routes optional providers to the plugin ecosystem rather than bundling every provider in core, and the repo already documents a first-class provider-plugin path plus ClawHub distribution for community providers.

## Evidence

- **Vision keeps optional capability out of core:** `VISION.md` says core should stay lean, optional capability should usually ship as plugins, the preferred path is an external plugin repository, and the bar for adding optional plugins to core is intentionally high.

## Close Comment

Closing as `clawhub` after a Codex review. Current main points new provider integrations toward the plugin ecosystem rather than bundling them in core: `VISION.md:54` says optional capability should usually ship as plugins and that the bar for adding optional plugins to core is intentionally high, `docs/plugins/building-plugins.md:11` says model providers are a plugin capability and do not need to be added to this repository, `docs/plugins/sdk-provider-plugins.md:11` documents the provider-plugin path, and `docs/plugins/community.md:9` says community plugins can add providers and should be published on ClawHub/npm. Based on that current scope, a Replicate integration fits better as a community provider plugin than as an OpenClaw core issue.
