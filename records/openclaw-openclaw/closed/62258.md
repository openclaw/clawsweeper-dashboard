## Summary

Current `main` already resets the WhatsApp watchdog baseline on reconnect by creating a fresh live connection with a new `startedAt` and empty `lastInboundAt`, and it now includes a regression test that explicitly covers a reconnected listener getting a fresh watchdog window. This fix is present on `main` in commit `ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46`, but I could not tie it to a released tag yet.

## Evidence

## Risks / Open Questions

- Status snapshots still keep the previous `lastInboundAt`/`lastMessageAt` until a new inbound message arrives, so status surfaces may continue to show an old last-message timestamp even though the reconnect watchdog itself now resets correctly.
- This was a read-only code/history review; I relied on the checked-in regression test and did not run the test suite.
- `gh` inspection was unavailable here because `GH_TOKEN` is not configured, so GitHub-side cross-reference checking was limited to the provided issue context and local git history.
