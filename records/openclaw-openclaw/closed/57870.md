## Summary

PR #57870 should close because its implementation path is obsolete on current main. The bundled ACPX plugin is now a thin wrapper around the published `acpx/runtime` package, while this PR patches the older in-repo CLI argument construction path. Maintainer feedback also blocked merge on upstream ACP passthrough proof and a clear platform contract, so the right next step is upstream ACPX support plus live verification before a small OpenClaw dependency/config/docs follow-up.

## Best Possible Solution

Close this PR as not actionable in `openclaw/openclaw` as-is. Implement and verify Claude Code computer-use support in the `openclaw/acpx` runtime first, including real ACP-managed end-to-end proof and a clear platform contract; then follow up in OpenClaw with a small dependency bump plus config/schema/docs wiring if the released runtime exposes a stable option.
