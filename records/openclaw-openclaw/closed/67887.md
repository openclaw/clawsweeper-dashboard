## Summary

Current `main` already covers the reported failure mode: bundled public-surface resolution falls back from requested `runtime-api.js` to source artifacts, qa-lab staging always carries `speech-core`, and packaging/tests now require the `speech-core` runtime sidecar. The shipped `v2026.4.23` release already contains these safeguards.

## Evidence

- **Public-surface resolver accepts source fallbacks for `.js` runtime artifacts:** `resolveBundledPluginSourcePublicSurfacePath()` strips the requested `.js` suffix and searches `.ts/.mts/.js/.mjs/.cts/.cjs`, so a request for `speech-core/runtime-api.js` can resolve to source-side `runtime-api.ts` in source checkouts or staged trees.
  - file: [src/plugins/public-surface-runtime.ts:56](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/src/plugins/public-surface-runtime.ts#L56)

- **QA staging now always includes `speech-core`:** `QA_ALWAYS_STAGE_RUNTIME_PLUGIN_IDS` explicitly includes `speech-core`, and `collectQaBundledPluginIds()` adds it whenever a source/built plugin directory exists, preventing unrelated parity runs from omitting the speech runtime facade.
