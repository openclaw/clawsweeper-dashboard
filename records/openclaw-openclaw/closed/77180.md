## Summary

Current main already implements the core behavior this report asks for: repeated prompt-time plugin tool planning uses cached descriptors and skips plugin runtime factory calls for matching warm contexts, with regression coverage for reuse and context invalidation.

## What This Changes

The issue reports that 2026.5.3-beta.4 still pays per-turn plugin tool factory cost during embedded agent turns and asks for architectural caching instead of incremental factory optimization.

## Best Possible Solution

Keep the descriptor-first plugin tool planning path on main; if a current-main or final-release build still shows second-turn factory timing warnings, track that as a fresh narrowed regression with current trace logs.

## Reproduction Assessment

No: I did not establish a failing current-main reproduction path. The inspected current-main path and regression tests show the reported warm prompt-time factory re-run is avoided for matching contexts.

## Solution Assessment

Yes: descriptor-first planning is the safest maintainable fix because it skips prompt-time runtime factory work while still late-loading the live plugin implementation on actual tool execution and invalidating on trusted context changes.
