## Summary

Keep open. Current main still has the heartbeat/commitment coupling that can degrade unrelated HEARTBEAT.md task runs, and this PR is the right narrow repair, but it is missing the required user-facing changelog entry that the automerge gate already flagged.

## What This Changes

The PR changes heartbeat preflight and prompt routing so due HEARTBEAT.md tasks run with normal tools when inferred commitments are also due, deferring commitment delivery for that turn, and adds focused regression coverage.

## Best Possible Solution

Land this PR after adding the required changelog entry, preserving the separation between tool-capable HEARTBEAT.md task turns and no-tools commitment-only delivery turns.

## Solution Assessment

Yes. The PR is the narrowest maintainable fix because it defers untrusted commitment delivery instead of weakening the no-tools safety rule, while keeping due heartbeat tasks on their normal tool-capable path.
