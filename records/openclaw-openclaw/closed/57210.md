## Summary

This issue is already covered by the open canonical session-management issue #54397. Current main still does not implement the independent Control UI New Chat workflow: the toolbar New Session action sends `/new` to the current session reset path, while the backend already exposes `sessions.create` and related session management APIs. The remaining work belongs under #54397, which tracks topic/session UI creation, switching, rename/delete, metadata, and export/import.

## Best Possible Solution

Close this issue as a duplicate of open #54397 and keep the implementation work consolidated there. The best product path is a focused Control UI change under #54397 that wires the visible New Session/New Chat affordance to `sessions.create`, switches to the returned session key, refreshes the Sessions list, and leaves slash reset semantics intact.
