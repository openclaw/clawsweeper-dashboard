Close as implemented: current main now pushes terminal session lifecycle updates over `sessions.changed`, and WebChat applies those patches to clear the local active run. The proof commit is on current main only in the local tag set; no shipped tag contains it yet.

## What This Changes

The issue reports WebChat staying in an in-progress state after an agent turn completes because the terminal session state is not pushed to WebSocket clients.

## Best Possible Solution

Keep the current main lifecycle `sessions.changed` implementation and ask for a fresh current-main report only if WebChat still stays busy after a build containing `91080fde684f6a7acc8fe6995e305033a85cd288`.

## Solution Assessment

Yes, current main uses the narrow maintainable fix: the Gateway emits terminal session lifecycle patches and WebChat reconciles those patches to clear active-run UI state without adding config or product policy.
