## Summary

Current `main` already prevents fallback runs from being mistaken for live user model switches. The embedded runner now only throws `LiveSessionModelSwitchError` when `liveModelSwitchPending` is explicitly set, and the session schema documents that system fallback rotations never set that flag. Targeted regression tests cover the no-pending-switch case, so the reported primary-model pullback loop is already fixed.

## Evidence

- **Live switch now gated by explicit pending flag:** `shouldSwitchToLiveModel` returns a switch request only when `entry.liveModelSwitchPending` is true; otherwise it returns `undefined`, so persisted/default primary selection alone no longer forces a switch during fallback.
