## Summary

Keep open. Current main still shows the three reported source-level gaps, and the related open PRs do not close or supersede this exact lifecycle recovery bug.

## What This Changes

The issue asks OpenClaw to treat clean gateway close 1000, pending `agent.wait` results, and still-running backing sessions as recoverable subagent lifecycle states instead of terminal lost failures.

## Best Possible Solution

Implement a focused agents-runtime fix in the existing wait classifier, subagent wait supervision, and stale-sweep reattach path while preserving fail-closed behavior for missing, terminal, stale, drifted, or truly orphaned sessions.

## Reproduction Assessment

Yes, at source level. Current main still omits clean-close 1000 from the recoverable wait classifier, returns on pending wait without rearming supervision, and can finalize a still-running backing session as lost context.

## Solution Assessment

Yes. The narrow maintainable repair is to extend the existing recoverable wait classifier, reuse the guarded delayed wait retry pattern for pending waits, and add a running-session reattach or re-wait guard before lost-context finalization.
