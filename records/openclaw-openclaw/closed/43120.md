## Summary

Current main already makes workspace/ClawHub skills available to isolated cron runs by building a fresh skill snapshot for the cron session, passing it into both embedded and CLI-backed agent execution, and preserving the Windows Scheduled Task PATH behavior needed for command lookup.

## Best Possible Solution

Close this as implemented on current main. Keep the shipped isolated-cron skill snapshot and Windows Scheduled Task PATH behavior; if a user can still reproduce on a current release, the best follow-up is a new focused bug with `openclaw cron runs --id <jobId>`, the affected skill name, `openclaw skills check --json`, Gateway service status/config context, and the command PATH details from the Windows task environment.
