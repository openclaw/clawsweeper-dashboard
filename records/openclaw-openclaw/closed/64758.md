## Summary

This PR should close as superseded by the newer maintainer-owned implementation path. Current main still has the active-but-not-streaming steer gap, but open #73365 covers the same remaining behavior with a lifecycle-aware `isStopped` approach, direct-injection bounds, focused tests, and changelog coverage, while this PR is dirty against main and uses a broader abort-on-failed-enqueue fallback.

## What This Changes

The PR changes auto-reply steer handling to attempt injection for active runs rather than streaming-only runs, abort non-heartbeat active embedded runs when steer injection fails, removes the runner `isStreaming` parameter, and adds focused regression tests.

## Best Possible Solution

Close this PR in favor of reviewing and landing #73365 or a maintainer-approved successor, while keeping #10960 open as the product/runtime tracker until the lifecycle-aware steer behavior ships.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is an active session using `messages.queue.mode: "steer"` where `isActive` is true but `isStreaming` is false; current main skips immediate injection and enqueues a followup. The PR-specific abort risk is also source-reproducible because `queueEmbeddedPiMessage` returns false for compacting handles and this patch treats any false result as an abort trigger.

## Solution Assessment

No. The intended direction is valid, but this PR's abort-on-failed-enqueue fallback is less precise than the newer lifecycle-aware path in #73365, which distinguishes stopped/non-injectable states and adds direct-injection bounds instead of killing the active run after an untyped queue failure.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Avoid aborting on every failed steer enqueue:** `src/auto-reply/reply/agent-runner.ts:223-225`
  - body: `queueEmbeddedPiMessage` returns `false` for more than the active-but-not-streaming case, including compacting handles. This fallback will abort the current run for any non-heartbeat steer enqueue failure, so a compacting or otherwise non-injectable active run can be killed even though the turn then proceeds through followup/drop policy. Use a typed failure reason or the lifecycle-aware injection guard from the canonical replacement path before aborting.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The patch does not touch supply-chain files, but it broadens an inbound runtime control path in a way that can repeatedly abort active work without the lifecycle and rate/size guards present in the superseding PR.

Concerns:

- **[medium] Unbounded steer fallback can abort active runs:** `src/auto-reply/reply/agent-runner.ts:223`
  - body: A non-heartbeat steer message now aborts the active embedded run whenever enqueue returns false. In shared or noisy sessions, repeated steer messages that hit non-injectable states could become an availability/control-surface regression; the replacement PR adds more explicit lifecycle checks and injection bounds.
  - confidence: 0.78
