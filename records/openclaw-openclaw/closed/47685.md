## Summary

The reported bug is in a community memory plugin package, not in OpenClaw core. Current main has no `@memtensor/memos-local-openclaw-plugin` implementation to fix; the repo only treats that id generically as an external plugin.

## Evidence

- **Core scope policy:** OpenClaw's vision says optional capability should usually ship as plugins and that plugin authors should host and maintain plugins in their own repository.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/VISION.md#L54)
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
- **Community plugin ownership:** The community plugin docs say community plugins are third-party packages built and maintained by the community, published on ClawHub or npm.
  - file: [docs/plugins/community.md:9](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/docs/plugins/community.md#L9)
  - sha: [d38ed0831d73](https://github.com/openclaw/openclaw/commit/d38ed0831d7372c8505dc944aea69b5b83da1435)
- **External-vs-core distinction:** The plugin docs explicitly distinguish core plugins shipped with OpenClaw from external plugins published on npm by the community.
  - file: [docs/tools/plugin.md:11](https://github.com/openclaw/openclaw/blob/d38ed0831d7372c8505dc944aea69b5b83da1435/docs/tools/plugin.md#L11)
