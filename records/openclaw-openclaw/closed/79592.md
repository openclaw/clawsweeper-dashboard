## Summary

Close as implemented on current main: the v2026.5.7 report is coherent, but current source, docs, and tests now resolve per-agent model config before global defaults and persist runtime model state without creating an automatic session override.

## What This Changes

Agent-level `model` settings should be honored when initializing routed sessions instead of pinning sessions to `agents.defaults.model.primary` with an automatic override.

## Best Possible Solution

Keep the current main behavior: resolve `agents.list[].model` before global defaults and persist runtime model fields only after an actual run or fallback.

## Reproduction Assessment

No current-main failing reproduction was established. The report provides a clear v2026.5.7 path, while current-main source and regression tests show the routed agent model is now selected before the global default.

## Solution Assessment

Yes. The current implementation fixes this at the shared model-resolution and session-persistence path, which is narrower and more maintainable than adding a new config flag or channel-specific workaround.
