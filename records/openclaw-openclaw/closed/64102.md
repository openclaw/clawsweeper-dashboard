## Summary

Keep this PR open. The provided discussion correctly identifies that current main only makes `security_scan_blocked` and `security_scan_failed` terminal before hook-pack fallback when `--dangerously-force-unsafe-install` is set; normal no-flag blocked installs can still fall through to hook-pack fallback and produce the misleading follow-up error this PR targets.

## Best Possible Solution

Keep this PR open for maintainer review or adaptation. The best fix is to make `security_scan_blocked` and `security_scan_failed` terminal before local-path, linked-path, and npm hook-pack fallback regardless of `--dangerously-force-unsafe-install`, while preserving non-security hook-pack fallback and bundled-plugin npm recovery. Keep or adapt the focused no-flag npm regression test from this PR.
