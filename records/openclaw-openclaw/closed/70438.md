## Summary

Current main implements the reported non-image WebChat attachment path via merged PR #72299 / commit 6b6dcafc: Control UI accepts non-video files, the client serializes them as file attachments, Gateway preserves them through attachment normalization, stores non-image uploads as managed inbound media, injects a `media://inbound/<id>` reference into the user message, and persists transcript metadata. The exact proposed text-inline config knob was not added, but the silent-drop/local-patch problem described by #70438 is solved on main.

## Best Possible Solution

Close #70438 and keep the shipped #72299 behavior: non-video WebChat uploads should flow through the official Control UI and Gateway media-ref path. If maintainers want a separate policy surface for text-file inlining, per-tenant attachment policy, or attachment size knobs, that should be tracked as a narrower follow-up rather than keeping this silent-drop issue open.
