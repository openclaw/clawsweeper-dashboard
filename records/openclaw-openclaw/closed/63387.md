## Summary

Close as implemented_on_main. Current `main` already guards the stale Discord gateway heartbeat race described in #63387 by clearing any leftover `heartbeatInterval` and `firstHeartbeatTimeout` before delegating to the parent `connect()` path, and it includes regression tests for both stale-timer cases. The fixing code is present at `86f8c826e20a01d1bba67c0ba6e73182fbf0455f` and was introduced by commit `14934f0b7c874e6af4ed93a1c307a17d222ef739`. That commit is newer than the latest tagged release `v2026.4.23`, so there is no shipped release tag to record yet.
