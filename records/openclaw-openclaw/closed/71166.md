## Summary

Close #71166 as superseded by the active implementation PR #71382. Current checked-out main does not yet provide a WhatsApp transport-level read-only gate, and PR #71382 explicitly tracks this exact WhatsApp read-only mode, including channel/account `readOnly` config and outbound blocking paths. The PR discussion also contains the remaining review follow-ups, so the work should stay centralized there.

## Best Possible Solution

Close #71166 as superseded by open PR #71382 and keep all remaining WhatsApp read-only implementation and review follow-up on that PR. The shipped solution should be a transport-level WhatsApp gate with channel/account `readOnly` config, not just session `sendPolicy`.
