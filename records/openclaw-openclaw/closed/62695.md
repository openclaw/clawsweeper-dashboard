## Summary

Close as duplicate/superseded. Current main still does not expose a per-run tool allowlist on `sessions_spawn`, but the same remaining spawned-subagent tool restriction work is already tracked by the earlier open #23906, which covers `none`/subset/default-inherit semantics for local-model subagents.

## What This Changes

Request support for a per-run `sessions_spawn` tool allowlist so spawned subagent runs can temporarily receive a narrower tool set without creating separate agent definitions.

## Best Possible Solution

Keep one canonical tracker for the remaining design and implementation work in #23906, then add an explicit per-spawn tool restriction parameter to `sessions_spawn`, validate it at the tool/gateway boundary, and forward it into the existing embedded-runner `toolsAllow` filter without widening existing agent, sandbox, or subagent policy.

## Reproduction Assessment

Not applicable as a feature request, but the absence is source-verifiable: current `sessions_spawn` and gateway `agent` schemas have no per-run tool allowlist, while cron has `toolsAllow` plumbing into the embedded runner.

## Solution Assessment

Yes for cleanup: closing this as a duplicate of the earlier open #23906 is the narrowest maintainer path because current main still lacks the feature and #23906 already tracks the same spawned-subagent behavior with more detailed local-model examples.
