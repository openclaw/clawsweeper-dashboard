## Summary

Keep this PR open for normal merge review: current main still ignores prerelease precedence in plugin minHostVersion checks, and the updated branch now fixes that at the shared helper while preserving OpenClaw stable correction-release ordering. The remaining risk is an intentional compatibility change: prerelease hosts that were incorrectly allowed to load/install/update incompatible plugins will now fail closed.

## What This Changes

The PR switches plugin minHostVersion checks from the runtime triple parser to a channel-aware OpenClaw release comparator with semver fallback and adds prerelease/correction-release regression tests.

## Best Possible Solution

Land the centralized helper fix with the current prerelease and correction-release regression coverage after normal required checks, treating stricter prerelease gating as the intended compatibility contract.

## Reproduction Assessment

Yes. Current main's parseSemver drops prerelease identifiers, and checkMinHostVersion uses it for both current and minimum versions; the PR's direct helper proof covers the before/after cases.

## Solution Assessment

Yes. The updated fix is in the central helper used by load/install/update and reuses existing OpenClaw release ordering before falling back to generic semver, which is narrower than changing each caller.
