## Summary

Keep open. Current main still contains the exact unconditional `system.run.prepare` path and approval-plan denial that the report describes, and the same logic is present in the latest release snapshot `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).

## Evidence

- **Node exec still always calls prepare first:** The remote node exec path builds a shell command and immediately invokes `system.run.prepare` before any `system.run` call. There is no `security`/`ask` gate before the prepare call.
  - file: [src/agents/bash-tools.exec-host-node.ts:100](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/agents/bash-tools.exec-host-node.ts#L100)
  - command: `nl -ba src/agents/bash-tools.exec-host-node.ts | sed -n '70,150p'`
  - sha: [4013c658537e](https://github.com/openclaw/openclaw/commit/4013c658537ee2051c44186f197b0710e84c9049)
- **Node host prepare handler still hard-fails on planner errors:** `system.run.prepare` decodes params, calls `buildSystemRunApprovalPlan(params)`, and returns `INVALID_REQUEST` with the planner message when `prepared.ok` is false. No effective approval-policy check happens in this handler before the planner runs.
  - file: [src/node-host/invoke.ts:498](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/node-host/invoke.ts#L498)

- **Planner still emits the reported denial message:** When no bindable mutable file operand is found and stable interpreter binding is required, the planner returns `SYSTEM_RUN_DENIED: approval cannot safely bind this interpreter/runtime command`.
  - file: [src/node-host/invoke-system-run-plan.ts:1051](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/node-host/invoke-system-run-plan.ts#L1051)

  - file: [src/node-host/invoke-system-run-plan.ts:1289](https://github.com/openclaw/openclaw/blob/4013c658537ee2051c44186f197b0710e84c9049/src/node-host/invoke-system-run-plan.ts#L1289)
