## Summary

Current main already routes gateway `agent` requests through the selected agent/session and resolves workspace bootstrap files from that agent’s workspace, not the default agent workspace. The same implementation is present in released `v2026.4.23`, so the reported default-workspace fallback is already fixed on current code.

## Evidence

- **Gateway resolves explicit per-agent session keys before dispatch:** The gateway `agent` handler derives `requestedSessionKey` from `resolveExplicitAgentSessionKey({ cfg, agentId })` when `--agent` is supplied and no explicit session id/key is provided, so non-default agents are routed to their own `agent:<id>:main` session instead of the default agent session.
  - file: [src/gateway/server-methods/agent.ts:503](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/gateway/server-methods/agent.ts#L503)
  - command: `nl -ba src/gateway/server-methods/agent.ts | sed -n '500,515p'`
  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **Agent runtime chooses workspace from the resolved session agent:** `agentCommand` computes `sessionAgentId` from the explicit agent override or resolved session key, then sets `workspaceDirRaw = normalizedSpawned.workspaceDir ?? resolveAgentWorkspaceDir(cfg, sessionAgentId)`. That means workspace bootstrap files come from the selected agent’s workspace.
  - file: [src/agents/agent-command.ts:339](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/agents/agent-command.ts#L339)

- **Per-agent workspace resolver honors configured agent workspace:** `resolveAgentWorkspaceDir` first checks the specific agent entry’s `workspace` field before falling back to defaults, matching the expected per-agent workspace behavior described in the report.
  - file: [src/agents/agent-scope-config.ts:149](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/agents/agent-scope-config.ts#L149)

- **Runner reports systemPromptReport.workspaceDir from effective workspace:** The embedded runner builds `systemPromptReport` with `workspaceDir: effectiveWorkspace`, so `--json` output now reflects the runtime workspace actually used for injected bootstrap files.
