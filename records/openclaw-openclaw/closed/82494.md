## Summary

The correctness blocker still applies: the PR relaxes persisted cron `agentTurn.message` validation for every session target, but current runtime only has blank-payload skip handling on main-session jobs. Detached targets would now be admitted and forwarded to isolated execution with a whitespace-only prompt, so this should stay open for contributor changes and real behavior proof.

## What This Changes

The PR adds session-store metadata-only coverage, relaxes persisted cron `agentTurn.message` validation, updates cron store expectations, and routes GitHub Copilot device-flow POSTs through a fixed `github.com` SSRF policy.

## Best Possible Solution

Preserve repairable blank payloads only for the main-session path that already records skipped state, or add matching detached-target skip handling before broadening persisted validation.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: persist a non-main `agentTurn` cron job with a whitespace-only `message`; the PR admits it at load time and current detached execution forwards that message to isolated runtime.

## Solution Assessment

No. The session metadata coverage and Copilot SSRF guard are reasonable, but the cron validation change is wider than the runtime contract supports; target-scoped preservation or detached skip handling is the safer fix.
