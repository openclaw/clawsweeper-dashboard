## Summary

Close as implemented: current main passes the full gateway channel runtime into Feishu account startup and handler registration, so the reported undefined `channel.inbound.run` path is fixed and the fix is included in `v2026.6.6`.

## What This Changes

Feishu WebSocket inbound messages on v2026.5.27 were received by the gateway but failed before agent dispatch with `Cannot read properties of undefined (reading 'run')`.

## Best Possible Solution

Keep the shipped full gateway channel-runtime propagation and ask for a new report only if `v2026.6.6` or newer still hits the Feishu `channel.inbound.run` failure on a real tenant.

## Reproduction Assessment

Yes. The affected-release logs and source path show Feishu messages reached the gateway and then failed at `channel.inbound.run`; current main no longer follows the broken runtime handoff path.

## Solution Assessment

Yes. Passing the existing full gateway channel runtime through Feishu startup and monitor registration is the narrow owner-boundary fix, and it avoids adding a Feishu-specific fallback or compatibility shim.
