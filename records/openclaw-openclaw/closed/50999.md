Current main already contains multiple Telegram polling fixes that directly cover the reported 2026.3.13 stall/restart loop: a higher configurable stall threshold, bounded getUpdates timeouts, HTTP/1.1 long-poll transport hardening, transport cleanup/rebuild on stalls, and explicit 409 conflict recovery, all backed by targeted tests and shipped changelog entries.
