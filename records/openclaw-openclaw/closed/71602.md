## Summary

Current main has post-v2026.4.24 Docker/Bonjour hardening that addresses the reported Docker gateway restart loop: bundled Compose disables Bonjour by default on bridge networking, Docker setup persists that default, the Bonjour advertiser contains known @homebridge/ciao process errors, and tests/docs cover those behaviors. The issue author is not a maintainer and only has the non-protected `bug` label.

## Best Possible Solution

Keep the current main Docker/Bonjour hardening and ship it in the next release. Users still on v2026.4.24 can use `OPENCLAW_DISABLE_BONJOUR=1` in Docker, and any remaining loop on a fixed build should be reported as a new, narrower issue with full Docker logs, restart count/exit code, stability bundle, and whether Bonjour was disabled.
