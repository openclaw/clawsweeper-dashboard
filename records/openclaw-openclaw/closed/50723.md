## Summary

Keep open. Current `main` still gives all `systemEvent` cron jobs the 10-minute default timeout even though `sessionTarget="main"` + `wakeMode="now"` jobs synchronously await `runHeartbeatOnce()`, so the PR appears to fix a real bug that is not yet implemented on `main`.

## Evidence

- **Timeout policy on main still treats all systemEvent jobs as 10-minute jobs:** `resolveCronJobTimeoutMs()` returns `AGENT_TURN_SAFETY_TIMEOUT_MS` only for `agentTurn`; every other payload kind still falls back to `DEFAULT_JOB_TIMEOUT_MS`.
  - file: [src/cron/service/timeout-policy.ts:16](https://github.com/openclaw/openclaw/blob/ebd7f19a3b20001960474ec9ba9cc424e45f7b23/src/cron/service/timeout-policy.ts#L16)
