## Summary

Current `main` already supports Telegram original-quality image delivery via `forceDocument`/`asDocument`, which sends images and GIFs as Telegram documents instead of compressed photos and is exposed through the outbound tooling and CLI.

## Evidence

## Close Comment

Closing from a Codex review: current `main` already implements this. Telegram outbound now exposes `forceDocument` / `asDocument` (`src/agents/tools/message-tool.ts:125`, `src/channels/plugins/outbound.types.ts:24`), and the Telegram send path disables image optimization and sends image/GIF uploads via `sendDocument` instead of `sendPhoto` when that flag is set (`extensions/telegram/src/send.ts:821`, `extensions/telegram/src/send.ts:891`, `extensions/telegram/src/send.ts:955`). The CLI also documents `openclaw message send --force-document` specifically for avoiding Telegram compression (`docs/cli/message.md:74`, `src/cli/program/message/register.send.ts:28`). Regression coverage is in `extensions/telegram/src/send.test.ts:1191`. This behavior is already present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
