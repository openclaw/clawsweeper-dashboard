## Summary

Current main already provides the requested wedged-gateway escape hatch through plain `openclaw gateway restart`: `--safe` is the RPC-based drain path, while the normal restart path uses the platform service manager and keeps JSON service responses.

## What This Changes

Requested an out-of-band gateway restart primitive for fully wedged gateway processes.

## Best Possible Solution

Keep the current `gateway restart` service-manager behavior as the documented escape hatch and use `gateway restart --safe` only when the running Gateway can still answer RPC.

## Reproduction Assessment

Not applicable; this is a feature/API request rather than a bug with failing steps. Source and docs inspection on current main show the requested out-of-band path is already available through plain `openclaw gateway restart`.

## Solution Assessment

Yes; using the existing `gateway restart` command for the supervisor path avoids adding a second command name while preserving `--json`, `--force`, `--wait`, and the separate RPC-aware `--safe` mode.
