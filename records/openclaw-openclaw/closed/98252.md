## Summary

Current main already strips direct Anthropic thinking blocks with missing, empty, or blank replay signatures before provider replay, and the same behavior is present in the shipped `v2026.6.11` release; the broader non-empty invalid-signature umbrella remains separately tracked.

## What This Changes

The issue reports that direct Anthropic API-key sessions can persist unsigned thinking blocks and then loop with `non_deliverable_terminal_turn` after restart when replaying history.

## Best Possible Solution

Keep the shipped replay-history sanitizer and recovery path; direct Anthropic API-key users on `2026.5.19` should upgrade to `v2026.6.11` or newer instead of disabling thinking globally.

## Reproduction Assessment

Yes for the historical bug: the report includes concrete 2026.5.19 Anthropic `Invalid signature in thinking block` logs with `sig_len=0`, and current source/tests show that exact replay shape is now stripped before provider conversion. I did not run live Anthropic traffic in this read-only cleanup pass.

## Solution Assessment

Yes. The shipped central replay sanitizer plus recovery path is the narrow maintainable fix because it repairs provider-facing history without adding config surface or asking users to disable thinking globally.
