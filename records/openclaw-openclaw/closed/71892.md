## Summary

Issue #71892 asks for the same remaining core behavior already tracked by open PR #38881: auto-switching the main agent run to `agents.defaults.imageModel` when an inbound image is attached but the selected model is text-only. Current main preserves/offloads text-only image attachments and uses `imageModel` for media-understanding/tool paths, but it still does not switch the main reply model. #38881 is open and is the better canonical thread for the implementation and design review.

## Best Possible Solution

Close #71892 as a duplicate/superseded issue and keep upstream PR #38881 as the canonical thread. The eventual fix should implement the main reply-model switch in the normal reply/model-selection path, respect explicit model overrides, use the real model alias/catalog machinery, support configured `imageModel` fallbacks, and add focused regression tests proving the final agent run receives the configured image model when inbound images meet the auto-switch criteria.
