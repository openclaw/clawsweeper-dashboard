## Best Possible Solution

Keep one canonical emitter-side fix: land https://github.com/openclaw/openclaw/pull/81593 or an equivalent `completed: true` Codex app-server event change, then close https://github.com/openclaw/openclaw/issues/82470 after proof passes.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path: Codex app-server emits `contextCompaction` end without `completed`, and the reply runner treats that shape as incomplete when user notifications are enabled.

## Solution Assessment

No. The narrower maintainable fix is to mark the successful Codex app-server compaction event with `completed: true`, which preserves the existing completion contract and fixes both strict consumers.
