## Summary

Keep open: current main and the latest release still scope Skill Workshop writes, proposal storage, and reviewer context to one workspace. The requested shared-skill targeting and cross-workspace promotion are new, security-sensitive product behavior rather than stale cleanup.

## What This Changes

The issue requests opt-in Skill Workshop support for shared-skill targeting, cross-workspace pattern promotion, and a manual promote-to-shared action.

## Best Possible Solution

Stage this as an opt-in Skill Workshop design with explicit target metadata, separated shared/workspace proposal stores, strict shared-root validation, and manual-only shared applies; defer cross-workspace promotion until privacy and security policy are approved.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, and v2026.5.12 release inspection give high confidence that Skill Workshop writes, stores, and reviews only workspace skills today.

Unclear. Shared targeting and explicit manual promotion are plausible, but the full proposal crosses write-authority and cross-workspace data-flow boundaries, so the maintainable path needs split design plus product/security approval first.
