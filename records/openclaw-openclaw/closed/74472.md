This PR changes native operator-approval Gateway clients to pass `deviceIdentity: null` only for loopback token/password bootstrap auth, adds unit and stale-baseline regression coverage, and adds an Unreleased changelog entry.

## Best Possible Solution

Land the narrow Gateway change after maintainer review and green latest-head checks, keeping the identity omission limited to local shared-auth native approvals while preserving normal paired-device identity behavior for remote and unauthenticated clients.

## Reproduction Assessment

Yes. On current main, a loopback token-auth native approvals client can attach the local paired Gateway identity and hit the server's approved-scope baseline path; the PR's added stale-baseline regression test captures that scenario.

## Solution Assessment

Yes. The PR mirrors the existing loopback shared-auth exception in `callGateway`, scopes it to native approvals, and covers the remote shared-auth and loopback no-auth cases so the default identity path is preserved elsewhere.
