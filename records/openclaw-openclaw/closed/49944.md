## Summary

Keep open: current main and the latest release still drop the effective verbose level before Control UI tool-card expansion, and the open linked PR with closing syntax is the active fix candidate.

## What This Changes

Control UI should expand tool cards by default when the effective session verbose level from `agents.defaults.verboseDefault` or a session override is `full`.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/74398 or an equivalent narrow fix that carries `chat.history.verboseLevel` into Control UI state and feeds the existing expansion helper while preserving manual tool-card toggles.

## Reproduction Assessment

Yes, source-level: `chat.history` returns the effective `verboseLevel`, but current Control UI does not store it and still passes `autoExpandToolCalls: false`. I did not run a live browser repro in this read-only review.

## Solution Assessment

Yes. The linked PR is the best current fix shape because it consumes the already-returned gateway value and existing expansion helper instead of adding a separate persisted preference or duplicating default resolution in the UI.
