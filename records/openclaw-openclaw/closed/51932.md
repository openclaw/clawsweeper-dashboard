## Summary

Current `main` already handles the reported plain-HTTP Control UI approval path by preserving operator scopes for localhost `allowInsecureAuth` sessions, and the repo includes regression coverage proving a device-less localhost Control UI client can still perform an `operator.admin` RPC. That means the reported `scopes=[] -> no-approval-route -> unknown or expired approval id` flow no longer matches current `main`.
