## Summary

Close as duplicate/superseded. The requested native usage-footer default for new Discord thread sessions is the same remaining work tracked by open #12578, and open #69618 explicitly closes #12578 while adding inherited `agents.defaults.responseUsage` semantics that would cover this issue's accepted global-default option. Current `main` still lacks the behavior, so this should not close as implemented; the active canonical tracker and PR are the right place to finish the work.

## What This Changes

Request a native default or inheritance path so new Discord thread sessions can start with `responseUsage=tokens` or another configured usage-footer mode instead of requiring `/usage` in every thread.

## Best Possible Solution

Close this narrower Discord-thread report as superseded and finish the shared usage-footer default work in #12578/#69618, or in a maintainer-chosen narrower replacement if #69618 is too broad. The accepted end state should provide one validated inherited default for new sessions, preserve explicit per-session `/usage` overrides including explicit `off`, update schema/help/docs, and keep Discord thread sessions covered by the same default path.
