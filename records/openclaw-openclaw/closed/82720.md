## Summary

The stated run-identity stale CLI task behavior is already implemented on current main and shipped; the PR's unique runtime change targets legacy no-run-identity CLI tasks instead, which is a separate documented contract change and should not merge through this PR.

## What This Changes

The PR changes task maintenance so stale persistent CLI owner-session tasks are marked lost when their live run context is gone, adds regression coverage, and updates docs and changelog.

## Best Possible Solution

Keep the shipped run-identity fail-closed implementation; if legacy no-run-identity main-session CLI tasks should also fail closed, track that separately with real reproduction and a docs contract update.

## Reproduction Assessment

No: current main does not reproduce the stated run-identity stale task problem because CLI tasks with sourceId/runId already fail closed when the live run context is gone. The PR's added regression setup sets sourceId and runId, so it exercises existing behavior.

## Solution Assessment

No: the central fix already exists in the run-identity branch, while the new check is reached only after no run identity is present. Changing that legacy fallback should be handled as a separate contract decision.
