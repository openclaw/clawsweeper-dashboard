## Summary

Current main and the shipped v2026.5.4 source already keep unconfigured Ollama discovery silent and keep the reported JSON CLI commands off provider-runtime startup paths.

## What This Changes

Stop CLI JSON commands from printing the Ollama unreachable warning unless Ollama is actually configured or selected.

## Best Possible Solution

Keep the current opt-in discovery behavior: unconfigured Ollama stays silent, explicit Ollama config remains loud, and JSON status/list commands stay on config-only or read-only paths.

## Reproduction Assessment

No. Source and regression coverage show current main returns before probing unconfigured Ollama, and the reported JSON commands avoid provider runtime or full catalog startup unless the user explicitly asks for broader model discovery.

## Solution Assessment

Yes. The current implementation is the narrow maintainable fix: it suppresses ambient/unconfigured probes without hiding warnings for explicitly configured Ollama endpoints.
