## Summary

Current main already fixes the Feishu parse-card failure path by changing the Schema 2.0 card builders from legacy `wide_screen_mode` to `width_mode: "fill"`, with regression coverage and a merged PR (#53395) that explicitly referenced this issue. The reporter's proposed top-level `elements` workaround is a Schema 1.x-compatible shape, but current Schema 2.0 `schema` plus `body.elements` is intentionally retained and covered while the incompatible config key was removed.

## Best Possible Solution

Close #53771 as implemented. Keep the current Schema 2.0 card structure and `width_mode: "fill"` fix; any future Feishu card rejection should be filed as a new issue with the exact generated payload, Feishu error response, OpenClaw version, and affected render path.
