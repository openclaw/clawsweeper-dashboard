## Summary

Current `main` already carries Control UI image attachments from paste/drop/file input through `chat.send`, and the gateway normalizes and accepts the Control UI attachment shape. The same implementation is present in the latest shipped release tag, with regression coverage for dashboard/control-ui image payloads.

## Evidence

- **Control UI collects pasted and dropped images as chat attachments:** The browser chat view reads pasted images and drag-dropped files with `FileReader`, stores them as `ChatAttachment { dataUrl, mimeType }`, and shows attachment previews before send.
  - file: [ui/src/ui/views/chat.ts:206](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/ui/src/ui/views/chat.ts#L206)
