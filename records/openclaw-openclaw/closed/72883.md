The issue asks for `gateway config.patch` to allow safe `diagnostics.cacheTrace.enabled` changes when message, prompt, and system content capture are disabled.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73098, or an equivalent narrow resulting-config guard that permits cache-trace decision observability only when content capture resolves disabled.

Yes. Source inspection on current main shows the reported patch changes `diagnostics.cacheTrace` leaves that are not allowlisted, so the guard rejects them before forwarding `config.patch`.

## Solution Assessment
