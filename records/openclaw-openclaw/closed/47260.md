## Summary

Current `main` no longer relies on `/app/src/infra/outbound/send-deps.js` in packaged Docker installs. Bundled plugin discovery now prefers compiled `dist/extensions` artifacts in installed/package roots, bundled channel sources import `resolveOutboundSendDep` through the public plugin SDK runtime seam, and regression tests guard against falling back to legacy `src/.../send-deps` imports.

## Evidence

- **Bundled plugin discovery prefers built artifacts in installed packages:** `resolveBundledPluginsDir()` returns `dist-runtime/extensions` or `dist/extensions` before source `extensions/` outside source-checkout flows, so packaged/Docker installs do not need `/app/src/**` available at runtime.

- **Channel catalog follows the same bundled-plugin resolver:** Bundled channel metadata resolution explicitly documents `source extensions/ in dev/test, dist/extensions in published installs, dist-runtime/extensions when paired with dist`.
  - file: [src/channels/bundled-channel-catalog-read.ts:30](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/channels/bundled-channel-catalog-read.ts#L30)
