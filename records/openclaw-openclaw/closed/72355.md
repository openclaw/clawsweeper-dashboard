## Summary

Keep this issue open. Current main mitigates the fatal Bonjour/CIAO rejection path, but it still defaults the advertised hostname and `lanHost` to `openclaw`/`openclaw.local` when `OPENCLAW_MDNS_HOSTNAME` is unset. The provided GitHub context also shows an open, unmerged PR explicitly intended to close this issue, so the issue should not be closed before that PR is reviewed and merged or rejected.

## Best Possible Solution

Keep this issue open and use PR #72694 as the implementation candidate. Maintainers should review whether Bonjour should default to a sanitized system hostname, add a supported hostname config surface, or both; then update `extensions/bonjour/src/advertiser.ts`, Bonjour tests, plugin/config schema if needed, and the public mDNS docs while preserving the current CIAO cancellation handling and bounded advertiser recovery.
