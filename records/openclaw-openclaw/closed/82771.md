## Summary

Keep open: the patch is focused, source-backed, and has sufficient Fedora rootless Podman proof, but it intentionally crosses an auth/security boundary by exposing host Claude CLI credentials to the setup container, so it needs maintainer approval before merge.

## What This Changes

The PR adds setup-only guarded read-only mounting of the host Claude CLI credential directory to the Podman launcher, allows `OPENCLAW_HOST_CLAUDE_DIR` from the launcher env file, and documents the override.

## Best Possible Solution

Merge only after provider/security owners accept the setup-only credential boundary; if they want stricter least privilege, narrow the mount to the minimum credential material or make the credential mount explicitly opt-in before refreshing proof.

## Reproduction Assessment

Yes from source inspection and the supplied live proof, though I did not run a fresh current-main Fedora repro. Current main sets container `HOME=/home/node` and mounts only `.openclaw` state, while the Claude credential reader looks for `.claude/.credentials.json` under that home.

## Solution Assessment

Unclear as a final merge choice: the implementation is a small, well-proven fix for the source-visible failure, but the broad directory mount and persisted override are auth/security boundary decisions. A narrower credential-file mount or explicit opt-in would be stricter if maintainers prefer that tradeoff.
