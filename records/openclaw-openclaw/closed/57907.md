## Summary

Current `main` intentionally treats `claude-cli/*` as the valid model-ref family for Anthropic Claude CLI setup, so the report's premise that the prefix is stale/invalid no longer matches the codebase.

## Evidence

- **Configure flow uses provider auth setup for model section:** `openclaw configure --section model` calls `applyAuthChoice(... setDefaultModel: true)` in `src/commands/configure.gateway-auth.ts:149`, so the current wizard behavior is whatever the provider auth flow returns.
