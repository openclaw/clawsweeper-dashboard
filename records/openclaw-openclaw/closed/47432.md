## Summary

Current `main` intentionally uses the base path without a trailing slash for the Control UI WebSocket URL, and the gateway WebSocket upgrade path does not require `/openclaw/` specifically. The reported failure is better explained by the nginx rule only matching `/openclaw/`, not by an OpenClaw frontend bug.

## Evidence

- **Frontend default URL omits trailing slash by design:** `deriveDefaultGatewayUrl()` normalizes `controlUi.basePath` and builds `pageUrl` as `${proto}://${location.host}${basePath}`, so `/openclaw` becomes `wss://host/openclaw`, not `.../openclaw/`.
  - file: [ui/src/ui/storage.ts:83](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/ui/src/ui/storage.ts#L83)
  - command: `nl -ba ui/src/ui/storage.ts | sed -n '83,96p'`
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Current tests assert no-slash base-path gateway URL:** `loadSettings()` is expected to return `expectedGatewayUrl("/openclaw")`, and that helper produces `wss://gateway.example:8443/openclaw` with no trailing slash.
  - file: [ui/src/ui/storage.node.test.ts:62](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/ui/src/ui/storage.node.test.ts#L62)
  - command: `nl -ba ui/src/ui/storage.node.test.ts | sed -n '41,71p'`
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Browser routing tests also treat `/openclaw` as the canonical gateway URL:** Navigation tests expect `app.settings.gatewayUrl` to equal `wss://other-gateway.example/openclaw` after confirmation.
  - file: [ui/src/ui/navigation.browser.test.ts:33](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/ui/src/ui/navigation.browser.test.ts#L33)
  - command: `nl -ba ui/src/ui/navigation.browser.test.ts | sed -n '30,38p'`
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Gateway link generation distinguishes HTTP slash from WS no-slash:** `resolveControlUiLinks()` builds `httpUrl` with `${basePath}/` but `wsUrl` with `${basePath}`. This is explicit current behavior, not an accidental omission.
  - file: [src/gateway/control-ui-links.ts:30](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/gateway/control-ui-links.ts#L30)
  - command: `nl -ba src/gateway/control-ui-links.ts | sed -n '8,36p'`
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Gateway does not require a trailing slash for non-canvas WebSocket upgrades:** The HTTP base-path router redirects `/openclaw` to `/openclaw/` for page requests, but the WebSocket upgrade handler does not enforce a control-ui base-path match before calling `wss.handleUpgrade()`. That means `/openclaw` vs `/openclaw/` is not an OpenClaw-side WS requirement on current `main`.
  - file: [src/gateway/control-ui-routing.ts:41](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/gateway/control-ui-routing.ts#L41)
