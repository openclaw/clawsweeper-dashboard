## Summary

Current `main` already normalizes stringified Ollama tool-call arguments back into JSON objects on both the native `/api/chat` replay path and the OpenAI-compatible Ollama path, with regression tests covering the exact multi-turn round-trip described in the issue. The changelog also records this fix, but it appears unreleased as of `v2026.4.23`.

## Evidence

- **Native Ollama replay now deserializes tool-call arguments before sending history back to Ollama:** `ensureArgsObject()` parses stored string arguments back into objects, and `extractToolCalls()` uses it when rebuilding assistant tool calls for native Ollama chat requests.
