## Summary

Current main routes the Web UI Update button through `update.run`, verifies the installed global package after package-manager updates, and explicitly avoids restarting the gateway after failed updates. The issue timeline cross-references PR #63065, and the changelog says that change shipped in `v2026.4.9` to fix npm packaging/runtime-dependency omissions so installs fail fast instead of crashing at runtime.
