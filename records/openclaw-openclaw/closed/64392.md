## Summary

PR #64392 is a substantial multi-agent orchestration subsystem, but current OpenClaw direction explicitly routes optional/heavy orchestration into plugins/ClawHub rather than core. Current main already exposes plugin, subagent, node, Task Flow, and detached-task surfaces that cover the right extension path, so this should be externalized instead of merged into core.

## Evidence
