## What This Changes

The PR adds npm freshness-bypass env/argv helpers for managed package and plugin installs, propagates npm config scope into install paths, updates tests, and adds a changelog entry.

## Best Possible Solution

Derive the npm freshness bypass mode once from the same config scope used by each install command, including staged prefixes, then prove exact-head managed install/update behavior for before and min-release-age configs.

## Reproduction Assessment

Yes, at source level: npm documents before and min-release-age as exclusive, and the PR head can still compute env freshness without the staged prefix while argv uses the staged prefix. I did not run a full package update flow.

## Solution Assessment

No, not as currently patched. The right fix is to share the same scoped freshness decision across env and argv, then add coverage/proof for the staged-prefix path.
