## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Wire composingDebounceMs into strict schemas:** `src/config/types.whatsapp.ts:82-85`
  - body: The new config field is added only to the WhatsApp TypeScript type. Because the root and per-account WhatsApp schemas are strict and still expose only `debounceMs`, user config that enables this feature is rejected as an unknown key.
  - confidence: 0.97
- **[P2] Update the public debounce SDK contract:** `src/auto-reply/inbound-debounce.ts:164`
  - body: `createInboundDebouncer` is exported through `plugin-sdk/channel-inbound-debounce` and injected runtime helpers, so adding `touch` and `touchByPrefix` changes a public plugin-facing return shape. Either keep this helper internal to WhatsApp or update SDK docs, mocks, contract tests, and compatibility coverage with the new methods.
  - confidence: 0.88
- **[P2] Use debounce-compatible keys for group touches:** `extensions/whatsapp/src/inbound/monitor.ts:489-490`
  - body: For group messages, active debounce buffers are keyed with sender identity/JID before E.164 on current main, but the presence handler touches only an E.164 prefix. Normal group composing updates therefore do not extend the matching sender's debounce buffer.
  - confidence: 0.91
- **[P2] Deduplicate presence subscriptions:** `extensions/whatsapp/src/inbound/monitor.ts:397-402`
  - body: This calls `presenceSubscribe(remoteJid)` for every accepted inbound message whenever composing debounce is configured, including repeated messages from the same chat and cases where no debounce listener is registered. Guard it by active debounce state and per-chat subscription tracking with retry after failure.
  - confidence: 0.9
- **[P2] Bound the LID-to-E164 cache:** `extensions/whatsapp/src/inbound/monitor.ts:78`
  - body: `jidToE164Cache` grows one entry per distinct mapped JID and never expires entries. Since the mapping only needs to bridge nearby composing events, use a short TTL or size cap to avoid stale contact growth in long-running listeners.
  - confidence: 0.84
