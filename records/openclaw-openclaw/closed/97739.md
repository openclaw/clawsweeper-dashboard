## Label Justifications

- `P1`: A subagent completion can be marked delivered without requester consumption, breaking active multi-agent workflows for real users.
- `impact:message-loss`: The reported failure loses or suppresses the subagent completion wake event after queue acceptance.
- `impact:session-state`: The parent session and subagent delivery bookkeeping can disagree with actual completion consumption, leaving the parent waiting.

## Summary

Close as duplicate: current main still has the queued-active-requester false-delivery path, and the same active-run transcript-commit versus consumption race is already tracked with source-level evidence and maintainer discussion in https://github.com/openclaw/openclaw/issues/92433.

## What This Changes

The issue requests at-least-once subagent completion delivery so a parent waiting in `sessions_yield` is not marked woken merely because the completion event was queued.
