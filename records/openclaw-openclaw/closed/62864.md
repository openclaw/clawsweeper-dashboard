## Summary

Current main, and the shipped `v2026.4.23` release, already implement the slow-local-model timeout handling this report asked for: explicit idle-timeout config is honored, one silent same-model idle timeout is retried automatically, and remaining timeouts are surfaced as visible errors instead of blank turns.

## Evidence

- **Idle timeout now honors configured run / agent timeouts:** The embedded runner resolves `agents.defaults.llm.idleTimeoutSeconds` first, then falls back to the explicit run timeout or `agents.defaults.timeoutSeconds`, instead of hardwiring a shorter idle window. That directly matches the report that increasing timeout settings previously did not help.
