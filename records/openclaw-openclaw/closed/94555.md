## Summary

Current main still has the Workboard CLI/API inconsistency: the terminal list path filters only by board/status while the tool and slash-command list paths hide archived cards by default. This issue should stay open because open candidate PRs, especially https://github.com/openclaw/openclaw/pull/94562, already use closing syntax and need maintainer selection before the issue can close.

## What This Changes

The issue requests that `openclaw workboard list` hide soft-archived Workboard cards by default, matching Workboard tool behavior, while providing an explicit way to include archived cards.

## Best Possible Solution

Land one canonical Workboard CLI fix, preferably the JSON-preserving candidate at https://github.com/openclaw/openclaw/pull/94562, then close this issue after the merged fix is on main.

## Reproduction Assessment

Yes. Current source shows terminal `openclaw workboard list` filters only by status while the Workboard tool and slash-command list paths hide archived cards by default, matching the reporter's concrete archived-card reproduction.

## Solution Assessment

Yes, the narrow best fix is to put archived-card visibility in the CLI list presentation layer with an explicit include-archived escape hatch, while leaving `WorkboardStore.list` as the full-card boundary.
