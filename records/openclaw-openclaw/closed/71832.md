## Summary

Issue #71832 asks for a new core `subclaw` command and parent/child orchestration model for per-directory agents. Current main does not ship a `subclaw` command, but the requested capability fits OpenClaw's plugin/ClawHub boundary: core already exposes isolated agents, sub-agent runs, plugin CLI registration, task-flow helpers, and runtime subagent APIs. The issue discussion also already points to an external ClawHub skill for multi-agent coordination patterns.

## Best Possible Solution

Close the OpenClaw core feature request and direct the workflow to ClawHub/plugin space. If an external `subclaw` implementation finds a concrete missing SDK seam, open a narrower plugin-API issue for that specific gap.
