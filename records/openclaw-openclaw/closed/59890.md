## Summary

The issue’s only concrete crash evidence is a missing `grammy` runtime import from Telegram’s `sticker-cache` chunk. OpenClaw already fixed bundled channel runtime-dependency packaging on main and shipped it in `v2026.4.9`, and current WebChat still treats pasted text as a plain `chat.send` message rendered through the generic markdown path.
