## Summary

Keep this PR open. Current main still has the old no-argument ContextEngineFactory contract, resolveContextEngine still accepts only config, and the run, queued compaction, and subagent cleanup paths do not pass agentDir/workspaceDir to factories. The PR remains an active, unmerged implementation candidate with maintainer follow-up rather than obsolete work.

## Best Possible Solution

Keep this PR open for maintainer review and completion. If the API direction is accepted, land the additive ContextEngineFactoryContext/ResolveContextEngineOptions contract, pass normalized agentDir/workspaceDir at run, queued compaction, and subagent cleanup resolution sites, keep the SDK export/docs/API baseline aligned, and run the targeted changed gates before merge. If maintainers reject the public contract, close it explicitly after that product/API decision.
