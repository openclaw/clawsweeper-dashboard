## Summary

Current main already implements deterministic active heartbeat checks through the documented `HEARTBEAT.md` `tasks:` block. The runner reads `HEARTBEAT.md`, parses named tasks, builds the heartbeat prompt from only due tasks, skips entirely when no task is due, and stores per-task timestamps. This solves the reported static-checklist clutter without adding brittle arbitrary heading inference. The related #56941 session-caching report is separate from this active-check selection behavior.

## Best Possible Solution

Keep the shipped structured `HEARTBEAT.md` `tasks:` mode as the deterministic active-check mechanism. Users who want checks to disappear entirely should model them as named tasks in `HEARTBEAT.md`; exact-time or standalone checks should continue to use cron jobs.
