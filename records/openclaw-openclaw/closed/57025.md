## Summary

Current `main` no longer has a 5-minute default timeout for isolated agent-turn cron jobs. The cron runtime now gives agent-turn jobs a 60-minute safety timeout by default, supports explicit per-job `timeoutSeconds` overrides (including `0` for no timeout), and the issue's reflection-specific job names/API calls do not exist in the OpenClaw codebase.

## Evidence

- **Agent-turn cron default timeout expanded:** `resolveCronJobTimeoutMs` returns `AGENT_TURN_SAFETY_TIMEOUT_MS = 60 * 60_000` for `agentTurn` jobs without an explicit timeout, instead of a 300000 ms default.

  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
- **Timeout behavior covered by tests:** Tests assert the expanded default timeout for `agentTurn`, explicit positive `timeoutSeconds`, and `timeoutSeconds = 0` disabling the outer timeout.
  - file: [src/cron/service/timeout-policy.test.ts:31](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/cron/service/timeout-policy.test.ts#L31)
