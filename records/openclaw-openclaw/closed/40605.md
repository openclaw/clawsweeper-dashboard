## Summary

Current `main` already sends sub-agent completion through the requester session instead of short-circuiting after a direct external send, and it has explicit coverage for both active requester sessions and Telegram topic delivery.

## Evidence

- **Direct completion path still injects requester-session trigger:** `sendSubagentAnnounceDirectly()` resolves requester activity, wakes the requester session when active, and otherwise calls `callGateway({ method: "agent", params: { sessionKey: canonicalRequesterSessionKey, message: params.triggerMessage, deliver: ... } })`. That means the completion trigger is sent to the requester session even when external delivery is enabled.
