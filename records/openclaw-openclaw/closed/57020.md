## Summary

Current `main` already rotates `sessionFile` when a reset creates a new `sessionId`, and the same implementation is present in shipped release `v2026.4.23`. The reported stale-transcript behavior is covered by an explicit regression test.

## Evidence

- **Resolver no longer reuses stale transcript after sessionId change:** `resolveAndPersistSessionFile()` now gates persisted `sessionFile` reuse on `baseEntry.sessionId === sessionId`; if the session id changed, it clears or replaces the old path before resolving the transcript file.
