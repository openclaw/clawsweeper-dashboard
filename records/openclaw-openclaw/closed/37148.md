## Summary

Keep open. Current `main` has prompt-level 12h/24h support, but the UI display surfaces named in the issue still rely on locale-default `toLocaleTimeString()` behavior and do not provide a dedicated 24-hour display setting, so the request is not implemented well enough to close conservatively.
