## Summary

Current `main` does not implement this PR’s exact behavior. `resolveOutboundAttachmentFromUrl` still always stages outbound media through `saveMediaBuffer`, and the saved path is intentionally rewritten to `"<sanitized-base>---<uuid>.<ext>"` rather than preserving the original local filename/path. Signal then passes that staged path through unchanged as the attachment path. That means the PR is not redundant on current `main`/`v2026.4.22`, even though main already improved the temp filename from UUID-only to `original-name---uuid`.
