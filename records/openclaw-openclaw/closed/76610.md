## Label Justifications

- `impact:data-loss`: The issue is about possible corruption or loss of persisted Matrix crypto snapshot data after an unclean shutdown.

## Summary

Current main has removed the reported direct JSON snapshot write path: Matrix IDB snapshots now persist through Matrix plugin-state SQLite, with legacy JSON snapshots migrated and archived. The fix is merged in https://github.com/openclaw/openclaw/pull/91100 and is included in the stable v2026.6.5 release and later releases.

## What This Changes

The issue reports that Matrix E2EE IndexedDB snapshots were written directly to `crypto-idb-snapshot.json`, risking corrupted crypto state after an unclean shutdown.

## Best Possible Solution

Keep the merged SQLite-backed Matrix crypto sidecar implementation; the broader timer/shutdown persistence lifecycle remains separately tracked in https://github.com/openclaw/openclaw/issues/76611.

## Reproduction Assessment

No for current main: the exact direct `crypto-idb-snapshot.json` write path reported here is gone. The original report was source-reproducible on older main, but current main persists through Matrix SQLite plugin state instead.

## Solution Assessment

Yes. Moving Matrix crypto sidecars into plugin-state SQLite removes the specific direct-file truncation window more completely than only swapping in an atomic JSON helper, while preserving legacy JSON import/archive paths for upgrades.
