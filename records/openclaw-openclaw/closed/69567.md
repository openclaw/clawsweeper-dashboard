## What This Changes

The branch adds `::add-mask::${auth_header}` after derived GitHub checkout auth header construction in 13 custom checkout blocks across `.github/workflows/ci.yml` and the two Testbox CI workflows.

## Best Possible Solution

Merge the focused mask registrations after workflow-security review and after required checks are green or explicitly handled; track the unrelated zizmor failures separately.

## Reproduction Assessment

Yes, at source level. Current main has 13 custom checkout blocks that derive `auth_header` from `github.token` and pass it to Git before registering any matching mask; I did not run a live leaking-token workflow against current main.

## Solution Assessment

Yes. Registering the derived header with `::add-mask::` immediately after construction and before the first Git invocation is the narrowest maintainable fix under the GitHub Actions workflow-command contract.
