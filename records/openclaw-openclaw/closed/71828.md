## Summary

Keep open: current main still lets the run-side typing TTL expire during active keepalive ticks, and this PR is a narrow functional fix for that Discord-visible liveness gap. The only concrete blocker I found is the missing required changelog entry for a user-facing fix.

## What This Changes

This PR refreshes the typing controller TTL after each successful keepalive tick and adds a fake-timer regression test for active runs that continue past the original TTL window.

## Best Possible Solution

Add the missing changelog entry, land this focused TTL renewal fix, and keep the broader active-run presence lease discussion under #65919 if maintainers still want lifecycle-owned presence beyond this narrow repair.

## Reproduction Assessment

Yes. On current main, create a typing controller with a 6s interval and 10s TTL, start the loop, advance fake timers to the first keepalive tick, then advance past 10s total; cleanup can fire because the keepalive tick does not refresh the original TTL.

## Solution Assessment

Yes for the narrow bug. Refreshing the TTL after a successful keepalive tick preserves the existing run-complete guard and dispatch-idle cleanup while avoiding a broader presence-policy change.
