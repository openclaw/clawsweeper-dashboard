## Summary

Current `main` and shipped release `v2026.4.23` already accept and use `channels.msteams.blockStreaming`, so the reported validation failure matches older behavior, not current OpenClaw.

## Evidence

- **Validator uses bundled channel runtime schema:** `ChannelsSchema` normalizes plugin-backed channel configs by resolving a direct channel runtime schema with `getDirectChannelRuntimeSchema()` and then `safeParse()`-ing `channels.<id>` through it. That matches the root-cause direction from the issue discussion.
