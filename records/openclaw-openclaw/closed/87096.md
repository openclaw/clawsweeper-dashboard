## Summary

Keep open: this is a focused Codex runtime fix, but the patch currently broadens watchdog progress to unscoped notifications and can mask genuine idle stalls. The maintainer label also means this PR should stay in human review rather than being cleaned up automatically.

## What This Changes

The PR updates Codex app-server terminal/completion idle guards and adds a regression test for queued terminal completion behind blocked notification projection.

## Best Possible Solution

Restrict receive-time activity updates to notifications that positively match the active turn, keep the short terminal queued guards, and add a regression proving repeated unscoped account/rate-limit notifications cannot keep a wedged turn alive.

## Reproduction Assessment

Yes, source-level: current main can queue turn/completed before projection while the short completion/terminal guards still ignore terminalTurnNotificationQueued. I did not run the regression because this review is read-only, but the PR's harness models the race clearly.

## Solution Assessment

No, not as written. The short terminal queued guard is a reasonable direction, but enqueue-time timestamp updates need to be limited to active-turn notifications so global account/rate-limit events remain non-progress.
