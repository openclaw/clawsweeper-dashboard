## Summary

Close as a duplicate of the already-open generic skill-workshop/session-directory bug: current main still has the Teams failure path, but the remaining root cause is the same raw conversation-id-as-channel path already represented by the Matrix report.

## What This Changes

The issue reports skill-workshop and embedded session-bundle failures when Microsoft Teams conversation IDs containing characters like `:` and `@` are used as bundled plugin directory names.

## Best Possible Solution

Track the generic skill-workshop embedded-review channel-context fix under https://github.com/openclaw/openclaw/issues/77636, with regression cases for both Matrix room IDs and Teams `19:...` conversation IDs.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: Teams session keys can expose a raw conversation id as hook `channelId`, and skill-workshop currently forwards that value as `messageChannel`. I did not run a live Teams gateway in this read-only review.

## Solution Assessment

No separate Teams-only fix is needed. The maintainable fix is the generic skill-workshop embedded-review channel resolver, mirroring the active-memory provider-vs-conversation-id guard and covering Teams plus Matrix regression cases.
