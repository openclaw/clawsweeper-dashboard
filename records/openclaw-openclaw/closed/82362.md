## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Do not preserve stale refresh expiry:** `extensions/xai/xai-oauth.ts:336`
  - body: `parseXaiOAuthTokenResponse` allows `expires_in` to be absent, but refresh spreads the old credential and only overwrites `expires` when the new value is present. If the stored credential is already expired, the refreshed access token is saved with that old expired timestamp, so `hasUsableOAuthCredential` rejects it and every use can refresh again; clear/replace the expiry or fail the refresh when no usable lifetime is returned.
  - confidence: 0.86
- **[P2] Guarantee OAuth listener cleanup on early failures:** `extensions/xai/xai-oauth.ts:255-269`
  - body: The callback server is started before `noteXaiOAuthUrl` and `ctx.openUrl`, and the promise is detached. If either step throws before `await callbackPromise`, `waitForLocalOAuthCallback` keeps port 56121 bound until callback, server error, or timeout, so retries can fail with the port still in use; add an abort/cleanup path or avoid starting the listener before fallible pre-callback work.
  - confidence: 0.82
