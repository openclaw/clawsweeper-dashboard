## Summary

Current main already addresses the two concrete behaviors in this report: built-in channel disablement now blocks BlueBubbles auto-enable, and gateway startup now refuses loopback/custom fallback to 0.0.0.0 with an explicit error instead of silently widening the bind. Because auto-enable now no-ops in the reported disabled-BlueBubbles case, startup also skips the config rewrite path there.

## Evidence

- **BlueBubbles disable respected:** `isPluginExplicitlyDisabled` now treats `channels.<builtInChannel>.enabled === false` as an explicit disable before plugin auto-enable runs, so `channels.bluebubbles.enabled=false` blocks BlueBubbles startup auto-enable.
