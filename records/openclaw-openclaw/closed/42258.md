- **[P2] Treat terminal timeout wait snapshots as finished:** `ui/src/ui/app-chat.ts:84`
  - body: This recovery path only clears the stale run for `agent.wait` statuses `ok` and `error`. Gateway terminal snapshots can also return `status: "timeout"` with terminal metadata such as `endedAt`, `error`, `stopReason`, or timeout phase, while plain wait-deadline timeouts mean the run is still active; retrying all `timeout` results leaves timed-out completed runs stuck behind the watchdog, preserving the bug this PR is trying to fix.
  - confidence: 0.89
- **[P3] Keep watchdog probe failures out of lastError:** `ui/src/ui/app-chat.ts:98`
  - body: A transient background `agent.wait` failure writes `host.lastError`, which can show a visible chat error even though the primary WebSocket and run may still be healthy. Since the watchdog retries automatically, route this to non-visible diagnostics or suppress it unless the run is proven failed.
  - confidence: 0.78
