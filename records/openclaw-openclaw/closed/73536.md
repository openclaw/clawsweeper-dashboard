## Summary

Keep open. Current main still omits bundled-MCP per-call request options, and this PR is a plausible fix path, but it is not merge-ready because it overloads a connection-only config field, lacks real behavior proof, and overlaps with the separate request-timeout direction in https://github.com/openclaw/openclaw/pull/78160.

## What This Changes

The branch adds a resolved bundled-MCP request timeout, propagates it through transport/session state, passes it to `Client.callTool()`, and adds resolver tests.

## Best Possible Solution

Land one reconciled patch that exposes and documents a per-server bundled-MCP request-timeout contract, passes it to `Client.callTool`, adds focused regression coverage, and includes redacted real behavior proof from a long-running MCP tool.

## Reproduction Assessment

Yes for source-level reproduction: configure a bundled MCP tool that runs longer than 60 seconds and current main reaches `Client.callTool` without `RequestOptions.timeout`, so SDK 1.29.0 applies its 60-second default. I did not run a live slow MCP server in this read-only review.

## Solution Assessment

No, not as submitted. Passing `RequestOptions.timeout` into `Client.callTool` is the right code path, but reusing `connectionTimeoutMs` changes a connection-only config contract; a separate documented request-timeout field like the direction in https://github.com/openclaw/openclaw/pull/78160 is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep request timeouts separate from connection timeouts:** `src/agents/mcp-transport-config.ts:74`
  - body: This makes any explicit `connectionTimeoutMs` also cap `callTool` requests. Current docs/types describe that field as connection-only, and examples use short values such as `10000`; after this patch those servers would time out tool calls after that short connect budget instead of the SDK's 60s default. Please add/use a separate request-timeout config or otherwise preserve the connection-only contract.
  - confidence: 0.88
