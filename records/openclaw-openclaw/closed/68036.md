## Summary

Keep this PR open. Current main partially moved the dreaming/cron transcript classification path away from `loadSessionStore`, but it still does not implement the PR's central readonly cache-hit behavior: `readSessionStoreCache` still deep-clones cached stores, `LoadSessionStoreOptions` has no `readonly` contract, and `readSessionUpdatedAt` still uses the default cloning load path. The PR remains useful, but it needs rebasing/adaptation to current main plus tests/type-safety around shared cache references.

## Best Possible Solution

Keep the PR open and ask for a current-main rebase/adaptation: preserve the independent direct transcript-classification path, add a typed readonly/no-clone cache API only where it is still needed, convert `readSessionUpdatedAt` or other proven read-only hot callers, and add tests for cache hit, cache miss, default clone isolation, and misuse-resistant readonly return types.
