## Summary

Current `main` already treats the reported `TypeError: fetch failed` unhandled rejection as a transient network error, so the gateway no longer exits on that condition. The changelog shows this shipped in `v2026.2.24`, which is newer than the reporter's `2026.1.24-3` environment.

## Evidence

- **Transient fetch-failed classification on main:** `isWrappedFetchFailedMessage()` matches bare `fetch failed`, `isTransientNetworkError()` treats it as transient, and `installUnhandledRejectionHandler()` logs a non-fatal warning and returns instead of calling `process.exit(1)` for transient unhandled rejections.
  - file: [src/infra/unhandled-rejections.ts:123](https://github.com/openclaw/openclaw/blob/a52b50ef3c8bb05d4a18302aad6862c37278cc57/src/infra/unhandled-rejections.ts#L123)
