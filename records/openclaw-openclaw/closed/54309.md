## Summary

This PR is proposing a new core-internal worker-spawn seam for a community plugin orchestration pattern. Current `main` points the other way: optional orchestration belongs in plugins, and plugins already have supported runtime APIs for agent/subagent execution without adding a new `src/infra` contract.

## Evidence

- **Repo vision keeps optional capability in plugins:** `VISION.md` says core stays lean, optional capability should usually ship as plugins, and plugin authors should host and maintain plugins in their own repos.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/VISION.md#L54)
- **Repo vision rejects core agent-hierarchy orchestration:** `VISION.md` lists agent-hierarchy frameworks and heavy orchestration layers that duplicate existing agent/tool infrastructure under 'What We Will Not Merge (For Now)'.
  - file: [VISION.md:99](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/VISION.md#L99)
- **Official plugin runtime already supports plugin-started agent work:** `docs/plugins/sdk-runtime.md` documents `api.runtime.agent.runEmbeddedAgent(...)` as the neutral supported way to start a normal OpenClaw agent turn from plugin code, and documents `api.runtime.subagent.*` for background subagent runs.
  - file: [docs/plugins/sdk-runtime.md:53](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/docs/plugins/sdk-runtime.md#L53)
- **Injected plugin runtime exposes these supported surfaces:** `src/plugins/runtime/runtime-agent.ts` wires `runEmbeddedAgent`/`runEmbeddedPiAgent` into plugin runtime, and `src/plugins/registry.ts` scopes `subagent.run/waitForRun/getSessionMessages/deleteSession` per plugin.
  - file: [src/plugins/runtime/runtime-agent.ts:17](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/plugins/runtime/runtime-agent.ts#L17)
- **The proposed worker CLI path is not part of the current root CLI surface:** `src/cli/program/help.ts` defines the root options and `src/cli/run-main.ts` routes normal CLI execution; there is no supported `--mode=worker` entrypoint on current `main`.
  - file: [src/cli/program/help.ts:47](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/cli/program/help.ts#L47)

## Risks / Open Questions

- If maintainers later decide that OS-process isolation itself should become a first-class plugin contract, that would need a documented plugin-SDK/runtime design rather than a new internal `src/infra` hook.
