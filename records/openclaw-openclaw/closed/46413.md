## Summary

Current `main` already implements the requested background completion path: subagents announce results back to the requester chat, direct completion delivery survives dormant/disconnected requester sessions by reusing stored routing, and Feishu has channel-specific subagent delivery hooks to preserve the original DM/topic target.

## Evidence

- **Subagent completions are documented as push-based final updates to the requester chat:** The subagents guide says `/subagents spawn` sends one final completion update back to the requester chat, uses resilient delivery, and fills missing target/account fields from the requester session's stored route so direct delivery still works.

## Risks / Open Questions

- This evidence covers the implemented background/subagent completion path, not automatic recovery of an already-running top-level live session after a transport drop.
- Feishu thread-bound subagent session UX is still narrower than Discord; the implemented fix is resilient completion delivery back to the original Feishu target.
