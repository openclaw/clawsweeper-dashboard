## Summary

Current `main` already supersedes this refactor. Signal outbound no longer uses the old standalone adapter shape from the PR; it now composes `outbound.base` plus shared `attachedResults` handling inside `createChatChannelPlugin`, deduping sender/max-bytes/result-wrapping in shared seams and avoiding the `sendPayload` divergence raised in review. The same implementation is present in release `v2026.4.23`.
