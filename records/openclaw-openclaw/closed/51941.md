## Summary

Current `main` does not have the UTC-only manual cron path described in #51941. Manual runs go through `src/cron/service/ops.ts` and shared schedule helpers, and run-history entries do not contain the reported `scheduledTime` field.

## Evidence

- **Manual run path does not parse cron expressions separately:** `inspectManualRunPreflight` checks due state from the persisted job, and `finishPreparedManualRun` applies the result back to job state/run logs. There is no separate `cron-parser`/`parseExpression(... without tz)` path for manual runs.
  - file: [src/cron/service/ops.ts:502](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/src/cron/service/ops.ts#L502)
  - command: `nl -ba src/cron/service/ops.ts | sed -n '500,740p'`
  - sha: [aa3a5f9ff790](https://github.com/openclaw/openclaw/commit/aa3a5f9ff7907c2a41d6259d24b529bda41f293a)
- **Timezone handling is centralized in shared cron scheduling:** `computeJobNextRunAtMs` delegates cron schedules to `computeNextRunAtMs`, and `computeNextRunAtMs` resolves `schedule.tz` (or host timezone) and builds `new Cron(expr, { timezone, catch: false })`. That same helper is used for cron scheduling generally, not a scheduler/manual split.
  - file: [src/cron/service/jobs.ts:247](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/src/cron/service/jobs.ts#L247)

  - sha: [aa3a5f9ff790](https://github.com/openclaw/openclaw/commit/aa3a5f9ff7907c2a41d6259d24b529bda41f293a)
- **Reported files/API do not exist on current main:** Repository search found no `cron-parser`, no `parseExpression`, no `src/cron/commands/run.ts`, and no `scheduledTime` or `triggeredBy` cron-run field matching the report's root-cause section.
  - command: `rg -n "cron-parser|parseExpression|src/cron/commands/run\.ts|scheduledTime|triggeredBy" .`
  - sha: [aa3a5f9ff790](https://github.com/openclaw/openclaw/commit/aa3a5f9ff7907c2a41d6259d24b529bda41f293a)
- **Run-log schema has `runAtMs` and `nextRunAtMs`, not `scheduledTime`:** Current cron run-log entries store `runAtMs`, `durationMs`, and `nextRunAtMs`. The UI renders those same fields in run history.
  - file: [src/cron/run-log.ts:18](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/src/cron/run-log.ts#L18)
  - command: `nl -ba src/cron/run-log.ts | sed -n '18,34p'; nl -ba ui/src/ui/views/cron.ts | sed -n '1728,1736p'`
  - sha: [aa3a5f9ff790](https://github.com/openclaw/openclaw/commit/aa3a5f9ff7907c2a41d6259d24b529bda41f293a)
- **Current docs describe manual runs as queue/force operations, with timezone applied to cron schedules:** Docs say `openclaw cron run` queues a manual run and `--due` preserves due-only behavior; timezone guidance is attached to cron schedules themselves (`--tz` or host timezone), not to a separate manual-trigger parser.
