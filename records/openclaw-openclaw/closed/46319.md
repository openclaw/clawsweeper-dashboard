## Summary

Current `main` already applies a second tool-use/tool-result repair pass immediately after `limitHistoryTurns()`, which directly addresses the reported truncation bug. The shared repair helper also drops free-floating orphan `toolResult` entries, and the same post-truncation safeguard is present in both normal embedded runs and compaction flows.
