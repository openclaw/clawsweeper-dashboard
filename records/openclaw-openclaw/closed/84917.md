Full review comments:

- **[P2] Preserve symlinked-root edit paths:** `src/agents/pi-tools.read.ts:962-963`
  - body: This compares the edit tool's incoming absolute path against the canonical workspace root. When the configured workspace root itself is a symlink, upstream edit/read paths are still under the symlink path, so this can throw a workspace escape before edit can read or access the file. Current main avoids that by keeping lexical root checks for read/access and using canonicalization only where needed for writes.
  - confidence: 0.78
- **[P2] Keep existing-ancestor resolution for missing parents:** `src/agents/pi-tools.read.ts:871-873`
  - body: The new helper only calls realpath on the immediate parent and falls back to the original symlinked path when that parent does not exist. A nested write such as memory/new/note.md under an in-workspace symlink would still send fs-safe a symlink component; current main's helper resolves the nearest existing ancestor and appends the missing suffix.
  - confidence: 0.76
