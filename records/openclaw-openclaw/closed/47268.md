## Summary

Current `main` already implements the reported MSTeams FileConsentCard behavior: the consent-card activity id is captured when the card is sent, and acceptance now replaces that original card in place with `updateActivity`, with regression coverage. The shipped `v2026.4.22` release also contains that fix.
