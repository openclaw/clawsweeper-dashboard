## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Track enough data to skip long delivered commentary:** `src/agents/pi-embedded-subscribe.ts:924-928`
  - body: `rememberDeliveredCommentarySegment` stores only an 8,000-character prefix while recording the full delivered length. For a longer commentary segment that was already sent live, final payload filtering can only verify the prefix, so the tail after 8,000 characters is treated as unsent and can be resent in the final reply; store a tail/hash or otherwise verify the full delivered prefix before deciding what suffix remains.
  - confidence: 0.86
- **[P2] Preserve the current WhatsApp delivery path:** `extensions/whatsapp/src/auto-reply/monitor/inbound-dispatch.ts:452`
  - body: The PR funnels final/block replies through `deliverPayload`, but current main now routes text through durable message-send context, media-only coalescing, and `sourceReplyDeliveryMode` forwarding before direct fallback. Rebase the commentary callback onto that pipeline so message-tool-only rooms, accepted-reply tracking, and media-only duplicate handling do not regress.
  - confidence: 0.82
