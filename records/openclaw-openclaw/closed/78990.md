## Summary

Keep open: current main and the latest tagged release still lack a Matrix table-mode default, so Matrix outbound markdown tables remain source-reproducible as fenced code blocks. The exact follow-up PR with reporter confirmation was closed unmerged, so it does not resolve this issue.

## What This Changes

The issue asks Matrix outbound formatting to stop rendering markdown tables as fenced code blocks, likely by giving Matrix a bullets table default and treating native HTML tables as a separate opt-in follow-up.

## Best Possible Solution

Approve and land a Matrix-owned `defaultMarkdownTableMode: "bullets"` with focused regression coverage and docs; keep native Matrix HTML table opt-in as a separate follow-up after validating formatter and client behavior.

## Reproduction Assessment

Yes, at source level: Matrix send/edit paths resolve `channel: "matrix"`, the resolver falls back to `"code"` without a Matrix plugin default, and shared tests show code mode marks tables as code blocks. I did not run a live homeserver/Element round trip.

## Solution Assessment

Unclear until Matrix owner approval: setting the existing Matrix plugin default to `"bullets"` is the narrowest likely fix, while native HTML tables should remain a separate opt-in discussion.
