## Summary

Keep #57477 open. Current main still has the silent skipped-update path: `update.run` reports `ok: true` for `status: "skipped"`, and the Control UI only surfaces feedback when `ok === false`. Main also lacks the proposed `force` parameter and `update.progress` event wiring, so this PR is not implemented or superseded on main.

## Best Possible Solution

Keep the PR open and either rebase/finish it or split it: first land the low-risk skipped-response/UI feedback fix with regression tests, then separately review force-update recovery and live progress events with explicit authorization scoping and run correlation.
