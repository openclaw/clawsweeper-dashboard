## Summary

Keep open: current main and v2026.6.9 still omit the compaction abort signal at the context-engine delegate bridge, and the linked open fix PR is not merged yet.

## What This Changes

The issue asks OpenClaw to cancel or discard in-flight auto-compaction when the user stops a run so the next model turn does not see an unintended compacted context.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/89886, or replace it with an equivalent narrow bridge fix that forwards the existing abort signal and keeps focused regression coverage.

## Reproduction Assessment

Yes for a high-confidence source-level reproduction: current main threads a run abort signal into context-engine compaction, but delegateCompactionToRuntime does not forward it to runtime compaction. I did not run the full live stop-button and large-context scenario in this read-only review.

## Solution Assessment

Yes. Forwarding the already-defined abortSignal through the delegate bridge is the narrow maintainable fix because the public contract and downstream runtime already support it; broader stop-button lock and manual /compact cancellation work remain separate.
