## What This Changes

This PR changes the OpenAI Responses and Chat Completions parameter builders to strip a leading `openrouter/` segment from OpenRouter model IDs before sending the request body.

## Best Possible Solution

Use https://github.com/openclaw/openclaw/pull/79313 as the canonical path for the OpenRouter payload fix and keep this narrower-but-overbroad duplicate closed.

## Reproduction Assessment

Yes, at source level. Current main copies `model.id` into OpenAI-compatible payloads, and the linked issue provides live OpenRouter 400/200 evidence for prefixed versus bare StepFun ids; I did not rerun a live OpenRouter request in this read-only review.

## Solution Assessment

No. This PR strips too broadly in the shared transport builders; the safer fix is the route-scoped payload normalization in https://github.com/openclaw/openclaw/pull/79313, which preserves `openrouter/auto` and custom proxy routes.
