## Summary

Current main already implements the relevant Discord/gateway cleanup: stale-socket restarts are no longer driven by stale app-event timestamps for Discord-like channels, Discord reconnects clear stale heartbeat timers, and long Discord runs already use a typing keepalive loop.

## Evidence

- **Stale-socket health checks now require explicit transport activity:** The gateway health policy only returns `stale-socket` when `lastTransportActivityAt` is present; stale `lastEventAt` alone is explicitly treated as non-liveness for quiet Slack/Discord-style channels.
  - file: [src/gateway/channel-health-policy.ts:115](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/gateway/channel-health-policy.ts#L115)

- **Regression test covers the quiet-Discord case:** There is a dedicated test asserting that a connected Discord snapshot with only an old `lastEventAt` stays healthy instead of being classified as `stale-socket`.
  - file: [src/gateway/channel-health-policy.test.ts:139](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/gateway/channel-health-policy.test.ts#L139)
