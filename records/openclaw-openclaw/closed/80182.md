## Summary

Keep open. This PR is active, protected by the `maintainer` label, and still removes public SDK named bindings from package-exported subpaths even though the scoped SDK policy treats breaking removals as major-version work.

## What This Changes

The PR reduces and deprecates Plugin SDK public subpaths, moves bundled plugin imports to narrower seams, and updates SDK docs, metadata, scripts, tests, and changelog entries.

## Best Possible Solution

Keep bundled plugins off broad or provider-owned SDK seams, but preserve existing public named bindings as deprecated shims until an explicit major-version SDK removal decision is made.

## Reproduction Assessment

Yes for the review findings: current-main source and docs show the public SDK names, while the provided PR diff removes those named bindings from still-exported SDK subpaths. An external plugin with existing named imports would fail module binding after upgrade.

## Solution Assessment

No. Deprecating the surfaces and moving bundled imports is reasonable, but deleting public named exports is not the narrowest compatible solution; retaining deprecated shims or using an explicit major-version process is safer.
