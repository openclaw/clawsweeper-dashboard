## Summary

Keep open: current main and v2026.6.1 still contain the health-only post-refresh shortcut, and the related systemd/provenance items are open rather than merged fixes.

## What This Changes

The issue reports that `openclaw update` can remove a systemd gateway unit and then skip reinstall when a surviving gateway process reports the updated version.

## Best Possible Solution

Repair update-time service refresh so success requires the expected gateway version plus proof that the intended supervisor unit is present, enabled, and active, with a conservative failure path when that cannot be proven.

## Reproduction Assessment

No live high-confidence reproduction was run in this review. The issue is source-reproducible: current main and v2026.6.1 have the health-only shortcut that can accept a surviving updated process without proving systemd supervision.

## Solution Assessment

Unclear. The narrow technical direction is supervisor-unit proof in the update restart path, but the exact behavior for system-scoped units and generated-unit provenance needs maintainer direction before an automatic fix.
