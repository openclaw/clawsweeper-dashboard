## Summary

Current `main` already has the load-before-resolve behavior and regression coverage for this report. Gateway/plugin startup now loads plugin runtime before context-engine resolution paths, and the repo contains an explicit regression test describing the 2026.3.7 `lossless-claw` failure.

## Evidence

- **Gateway startup loads startup plugins during bootstrap:** `prepareGatewayPluginBootstrap()` computes startup plugin ids, then calls `loadGatewayStartupPlugins(...)` before returning the startup bootstrap state, so gateway startup no longer depends on a later plugin-registration phase for those plugins.
  - file: [src/gateway/server-startup-plugins.ts:55](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/src/gateway/server-startup-plugins.ts#L55)
  - command: `nl -ba src/gateway/server-startup-plugins.ts | sed -n '1,140p'`
  - sha: [d4e93e791bc5](https://github.com/openclaw/openclaw/commit/d4e93e791bc5093ea14666bf4d8202b45a5132b4)
- **Gateway plugin bootstrap installs runtime environment before loading plugins:** `prepareGatewayPluginLoad()` installs the gateway plugin runtime environment, then immediately calls `loadGatewayPlugins(...)`, which is the path that executes plugin registration.
  - file: [src/gateway/server-plugin-bootstrap.ts:67](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/src/gateway/server-plugin-bootstrap.ts#L67)
  - command: `nl -ba src/gateway/server-plugin-bootstrap.ts | sed -n '55,120p'`
  - sha: [d4e93e791bc5](https://github.com/openclaw/openclaw/commit/d4e93e791bc5093ea14666bf4d8202b45a5132b4)
- **Plugin registration wires context engines into the shared registry:** The plugin API handler for `registerContextEngine` stores plugin-provided engines through `registerContextEngineForOwner(...)` and records the engine id on the plugin record.
  - file: [src/plugins/registry.ts:1385](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/src/plugins/registry.ts#L1385)
  - command: `nl -ba src/plugins/registry.ts | sed -n '1380,1415p'`
  - sha: [d4e93e791bc5](https://github.com/openclaw/openclaw/commit/d4e93e791bc5093ea14666bf4d8202b45a5132b4)
- **Runtime run path loads plugins before resolving the configured context engine:** `runEmbeddedPiAgent` calls `ensureRuntimePluginsLoaded(...)` first, then resolves the configured context engine later in the same flow. That addresses the reported order-of-operations failure for active runs.
  - file: [src/agents/pi-embedded-runner/run.ts:288](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/src/agents/pi-embedded-runner/run.ts#L288)
  - command: `nl -ba src/agents/pi-embedded-runner/run.ts | sed -n '280,320p'`
  - sha: [d4e93e791bc5](https://github.com/openclaw/openclaw/commit/d4e93e791bc5093ea14666bf4d8202b45a5132b4)
- **Subagent restore/cleanup path also loads runtime plugins before resolving the engine:** `notifyContextEngineSubagentEnded()` first calls `ensureSubagentRegistryPluginRuntimeLoaded(...)`, then resolves the context engine. This covers the startup-resume path that can run during gateway boot.
  - file: [src/agents/subagent-registry.ts:317](https://github.com/openclaw/openclaw/blob/d4e93e791bc5093ea14666bf4d8202b45a5132b4/src/agents/subagent-registry.ts#L317)
