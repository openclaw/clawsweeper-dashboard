## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Run version probing without a shell:** `src/commands/doctor-install.ts:188-190`
  - body: `displayPath` is interpolated into a shell command here. Because candidate paths can be influenced by npm config or environment-derived install roots, a path containing shell metacharacters can execute code during `openclaw doctor`; use `execFileSync(displayPath, ["--version"], ...)` instead.
  - confidence: 0.94
- **[P2] Compute npm global bin candidates from the prefix:** `src/commands/doctor-install.ts:132`
  - body: `npm root -g` returns the global `node_modules` directory, so joining `..`, `bin`, `openclaw` maps a normal `<prefix>/lib/node_modules` root to `<prefix>/lib/bin/openclaw`. Custom-prefix and version-manager installs that rely on this probe will be missed; derive the bin path from the npm prefix/bin contract or inspect package roots directly.
  - confidence: 0.9
- **[P3] Show single service context after duplicate binaries:** `src/commands/doctor-install.ts:95`
  - body: Once duplicate binaries are found, a single OpenClaw-related systemd service can still be the stale supervisor causing confusion. The `> 1` threshold hides that clue, so report any found service in the duplicate-install warning with singular/plural wording.
  - confidence: 0.78
- **[P2] Restore process.env after the suite:** `src/commands/doctor-install.test.ts:54`
  - body: This test suite replaces the `process.env` object in `beforeEach` but never restores it. Later tests sharing the same Vitest worker can inherit the mocked environment reference and HOME value, so add `afterEach` or `afterAll` cleanup.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The PR introduces a concrete shell-injection risk in doctor version probing and should not merge until the shell command is replaced with a safe process API.

Concerns:

- **[high] Shell injection in version fallback:** `src/commands/doctor-install.ts:188`
  - body: `execSync(`"${displayPath}" --version ...`)` runs through a shell with an interpolated path. Since candidate paths can originate from npm/global install configuration and environment-derived roots, a crafted path can execute arbitrary shell code when doctor resolves versions.
