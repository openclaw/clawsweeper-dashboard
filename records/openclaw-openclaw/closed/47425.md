## Summary

Current `main` already routes Feishu tools by the active routed account context (`agentAccountId`) before falling back to `channels.feishu.defaultAccount`, and docs/tests on `main` already cover the multi-account routing flow this draft PR is trying to add.

## Evidence

- **Tool context already carries routed account:** Plugin tool factories receive `agentAccountId` in their trusted execution context, and gateway tool resolution passes the active route account into tool creation.
  - file: [src/plugins/tool-types.ts:25](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/plugins/tool-types.ts#L25)
  - command: `nl -ba src/plugins/tool-types.ts | sed -n '7,31p'; nl -ba src/gateway/tool-resolution.ts | sed -n '80,90p'`
- **Feishu account resolution prefers contextual account over configured default:** `resolveImplicitToolAccountId` checks explicit `executeParams.accountId` first, then `defaultAccountId` from context if it is a real enabled Feishu account, and only then falls back to `channels.feishu.defaultAccount`.
  - file: [extensions/feishu/src/tool-account.ts:15](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/extensions/feishu/src/tool-account.ts#L15)
  - command: `nl -ba extensions/feishu/src/tool-account.ts | sed -n '1,120p'`
- **Feishu tools pass routed account context into client creation:** Bitable, Drive, and Doc tools all forward `ctx.agentAccountId` into Feishu client/account resolution, so tool calls inherit the active routed account instead of always using the configured default.
  - file: [extensions/feishu/src/bitable.ts:571](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/extensions/feishu/src/bitable.ts#L571)
  - command: `nl -ba extensions/feishu/src/bitable.ts | sed -n '556,590p'; nl -ba extensions/feishu/src/drive.ts | sed -n '752,775p'; nl -ba extensions/feishu/src/docx.ts | sed -n '1405,1430p;1598,1610p'`
- **Tests cover contextual account routing:** Current tests verify that Feishu tools route to `agentAccountId`, that contextual account beats `defaultAccount`, and that docs tools use account context unless explicitly overridden.
  - file: [extensions/feishu/src/tool-account-routing.test.ts:69](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/extensions/feishu/src/tool-account-routing.test.ts#L69)

  - file: [docs/gateway/config-channels.md:739](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/docs/gateway/config-channels.md#L739)

## Risks / Open Questions

- Current `main` solves this through routed account context, not by re-deriving an account from `agentId` alone. If a future use case needs account selection outside routed/channel context, that would be separate work.
