## Summary

Current main already solves the linked OpenRouter per-model pass-through problem through the shipped `agents.defaults.models["provider/model"].params.extra_body` path. The PR's exact `models.providers.*.models[].extraParams` field is not present, but the user-visible need from #69075 is covered by a documented, released per-model params contract.

## Best Possible Solution

Keep the shipped and documented per-model `agents.defaults.models["provider/model"].params` contract, and use `params.extra_body` / `extraBody` for OpenAI-compatible proxy request-body fields such as OpenRouter `reasoning.max_tokens`. Do not add a parallel `models.providers.*.models[].extraParams` field unless maintainers intentionally expand the model-definition contract and document precedence across both surfaces.
