## Summary

Current `main` already implements dashboard run cancellation well enough to cover this report: active chat runs are registered for abort, the Control UI keeps a Stop control for abortable runs, and disconnect-time aborts are queued and replayed on reconnect.

## Evidence

- **Current-main changelog explicitly covers stuck-run abort routing:** The unreleased `2026.4.24` changelog says `Gateway/chat` now registers `chat.send` runs in the chat run registry so lifecycle errors reach the client and `/abort` no longer reports no active run.
