## Summary

Current main already removes the cross-plugin `normalizeConfig` fallback that made the LM Studio hook run on foreign providers. The PR's bug report was valid, but its plugin-local guard is now obsolete because main fixed the affected behavior more broadly.

## Best Possible Solution

Close this PR as implemented on current main. Keep the broader provider-runtime no-scan implementation from `8ba9c9098a6baf6864f32376fd724c9e5d896987`; if maintainers still want defense-in-depth, track that as a new narrow hardening/docs cleanup rather than merging this stale PR.
