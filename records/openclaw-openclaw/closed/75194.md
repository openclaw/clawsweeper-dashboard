## Summary

Close as implemented. The issue discussion correctly points at Telegram `BodyForAgent` staying as standalone user text, but the downstream prompt path also receives structured `ReplyToBody`/quote fields and renders them into the assistant-facing prompt before the current message; current main and the reported v2026.4.27 release both contain that path with Telegram reply/quote coverage.

## What This Changes

The issue reports that Telegram direct-message replies or selected quotes no longer provide the assistant with the referenced bot-message context after upgrading to 2026.4.27.

## Best Possible Solution

Keep the structured design: Telegram should continue to store clean current-message text in `BodyForAgent` while carrying quoted/replied message data through typed reply fields that core renders as untrusted prompt context. If a live Telegram update still fails, it should be handled as a new payload-specific bug with a redacted update sample or prompt trace.

## Reproduction Assessment

Yes. A source-level reproduction using the described Telegram `reply_to_message` plus optional `quote` payload does not reproduce the loss: current code sets `ReplyToBody`/quote metadata and core prepends it to the model prompt.

## Solution Assessment

Yes. The current implementation is the maintainable solution because it avoids overloading `BodyForAgent` while still passing reply context to the assistant through structured, sanitized prompt metadata.
