## Summary

Keep open. Current main still leaves terminal cleanup dependent on the delayed `clear()` path, while the terminal transition itself only applies the done/error emoji and does not remove tracked stale lifecycle reactions; the related repair PR was closed unmerged, so the reported Discord status UI bug still has actionable remaining work.

## What This Changes

The issue requests that Discord lifecycle status reactions never leave a non-terminal emoji such as thinking as the final visible state after a successful turn, especially when `messages.removeAckAfterReply` is enabled.

## Best Possible Solution

Land a narrow shared-controller fix that removes tracked non-terminal lifecycle reactions when entering done/error, keeps the terminal emoji visible for the configured hold, and still clears all tracked lifecycle reactions afterward when `removeAckAfterReply` is enabled. Add regression coverage for the shared controller and the Discord success path, including removal failure diagnostics or retry semantics.

## Reproduction Assessment

Yes. The issue provides concrete live Discord steps on `v2026.4.29`, and current tests/source give a deterministic unit path: walk queued/thinking/tool to done with a remove-capable adapter and assert stale non-terminal reactions are removed at terminal completion rather than waiting solely for delayed clear.

## Solution Assessment

No, current main is not the best complete solution. The narrow maintainable direction is the closed #75570-style terminal cleanup adapted to the current active-emoji controller, plus focused Discord coverage and clear handling for failed removals.
