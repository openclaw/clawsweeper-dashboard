## Summary

Current main has the agent-run timeout lifecycle covered: gateway agent runs are registered for abort before acceptance, expired runs are reaped, CLI live sessions close on timeout/abort, and timeout fallback no longer reuses the original session.

## What This Changes

The issue requests that `openclaw agent --timeout N` terminate both the CLI wait and the corresponding gateway warm session so orphaned processes and file descriptors do not accumulate.

## Best Possible Solution

Keep the current main lifecycle behavior and ensure the next release includes the gateway abort/reaper and CLI fallback isolation changes.

## Reproduction Assessment

No high-confidence failing reproduction was run against current main; the issue has concrete v2026.4.25 steps, but current source and regression tests now show the timeout/abort path is bounded.

## Solution Assessment

Yes; the fix is in the maintainable ownership boundary: gateway registers and reaps active runs, the runner aborts warm CLI sessions, and CLI fallback avoids touching the original gateway session.
