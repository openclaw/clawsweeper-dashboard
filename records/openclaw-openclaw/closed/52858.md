## Summary

Current `main` already fixes the reported legacy `systemEvent.message` crash path for main-session cron jobs. The exact runtime fallback and a regression test are present on `main`, so this PR is superseded even though it also proposed extra store/API normalization cleanup.

## Evidence

- **Legacy payload fallback exists on main:** `normalizePayloadToSystemText` no longer assumes `payload.text` exists for `systemEvent`; it checks `text`, then legacy `message`, and returns an empty string instead of throwing.
  - file: [src/cron/service/normalize.ts:64](https://github.com/openclaw/openclaw/blob/443b837bd521b0485c87bb7359f370a4ccec10fa/src/cron/service/normalize.ts#L64)
  - sha: [443b837bd521](https://github.com/openclaw/openclaw/commit/443b837bd521b0485c87bb7359f370a4ccec10fa)
- **Main-session execution uses the safe resolver:** Main-session cron execution calls `resolveJobPayloadTextForMain(job)` before enqueueing. If no usable text exists, it skips with an error instead of dereferencing `undefined.trim()`.
  - file: [src/cron/service/timer.ts:1194](https://github.com/openclaw/openclaw/blob/c070509b7f8fdbe479e5d829643782c272e30e83/src/cron/service/timer.ts#L1194)
  - sha: [c070509b7f8f](https://github.com/openclaw/openclaw/commit/c070509b7f8fdbe479e5d829643782c272e30e83)
- **Regression coverage matches the reported shape:** Current tests include a `systemEvent` payload persisted with `message` instead of `text` and assert it resolves to `legacy text` without throwing.
  - file: [src/cron/service.jobs.test.ts:692](https://github.com/openclaw/openclaw/blob/443b837bd521b0485c87bb7359f370a4ccec10fa/src/cron/service.jobs.test.ts#L692)
  - sha: [443b837bd521](https://github.com/openclaw/openclaw/commit/443b837bd521b0485c87bb7359f370a4ccec10fa)
- **Fix landed after the latest release:** Diffing `v2026.4.23` against `HEAD` shows the legacy-message fallback and matching regression test were added on main after the latest shipped tag.
  - command: `git diff --unified=20 a9797214..HEAD -- src/cron/service/normalize.ts src/cron/service.jobs.test.ts`
  - sha: [443b837bd521](https://github.com/openclaw/openclaw/commit/443b837bd521b0485c87bb7359f370a4ccec10fa)

## Risks / Open Questions

- Current main fixes the crash path, but it does not include the PR's extra store/API canonicalization of legacy `payload.message` into `payload.text`. That is cleanup, not evidence of an unfixed bug.
- The fix is on `main` but not in the latest release tag `v2026.4.23`, so downstream users on that release would still need a newer build to get it.
