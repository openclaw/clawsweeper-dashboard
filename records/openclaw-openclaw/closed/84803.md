## Summary

Keep open: current main still records skipped WhatsApp group history as text-only while the inbound path already has media metadata and the shared history-media contract supports bounded local image history. The report is source-reproducible, distinct from the closed quoted-media WhatsApp issues, and needs a security-conscious channel fix rather than cleanup closure.

## What This Changes

Preserve media from unmentioned WhatsApp group messages stored by requireMention gating so a later mentioned turn can inspect the historical image.

## Best Possible Solution

Wire the WhatsApp mention-gating history path to the existing bounded HistoryEntry.media/recordWithMedia contract, carry only allowed local image media through filtered group history into InboundHistory, and add regression coverage for the image-before-later-mention flow.

## Reproduction Assessment

Yes at source level: a WhatsApp group image can reach the skipped mention-gating path with mediaPath/mediaType, but the stored GroupHistoryEntry and later InboundHistory mapping keep only text. I did not run live WhatsApp or tests because this review is read-only.

## Solution Assessment

Yes: the right fix is WhatsApp-owned wiring to the existing HistoryEntry.media and createChannelHistoryWindow().recordWithMedia contract after visibility filtering, not a new core history format or parallel retention path.
