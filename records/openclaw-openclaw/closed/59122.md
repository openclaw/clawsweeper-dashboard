## Summary

Current `main` already normalizes provider-qualified persisted model refs into separate provider/model values before selection, and the OpenAI-compatible transport sends the resolved bare `model.id`. The exact `ollama-beelink2/qwen2.5-coder:7b` regression is covered by tests, and the same fix/test bodies are present in the shipped `v2026.4.23` release.

## Evidence

- **Persisted override refs are reparsed into provider + bare model:** `resolvePersistedOverrideModelRef` rebuilds an encoded ref and runs it through `parseModelRef`, so a stored `provider/model` string is split back into separate provider/model fields instead of being treated as a raw model id.
  - file: [src/agents/model-selection.ts:76](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/agents/model-selection.ts#L76)
- **Duplicated provider prefixes are stripped from stored overrides:** `normalizeStoredOverrideModel` removes a leading `<provider>/` from `modelOverride` when `providerOverride` is already stored separately.
  - file: [src/agents/model-selection.ts:165](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/agents/model-selection.ts#L165)
- **Live session selection applies the normalization before runtime use:** `resolveLiveSessionModelSelection` loads the session entry, normalizes stored overrides, reparses them via `resolvePersistedSelectedModelRef`, and returns separate `provider` and `model` fields.
  - file: [src/agents/live-model-switch.ts:19](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/agents/live-model-switch.ts#L19)
- **Regression test covers the exact Ollama-style ref from the report:** The test stores `modelOverride: "ollama-beelink2/qwen2.5-coder:7b"` and expects `{ provider: "ollama-beelink2", model: "qwen2.5-coder:7b" }`.
  - file: [src/agents/live-model-switch.test.ts:209](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/agents/live-model-switch.test.ts#L209)
- **Transport sends the resolved model id, not a provider-qualified ref:** `buildOpenAICompletionsParams` serializes `model: model.id` into the outbound request body, so once resolution/normalization has run, the OpenAI-compatible/Ollama request uses the bare model id.
  - file: [src/agents/openai-transport-stream.ts:1719](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/agents/openai-transport-stream.ts#L1719)
- **Shipped in the latest release:** The same normalization code and regression tests are already present in release `v2026.4.23` (`a9797214338ba31c52c796adbb75afb16e0684a9`), verified with `git show a9797214338ba31c52c796adbb75afb16e0684a9:src/agents/model-selection.ts`, `git show a9797214338ba31c52c796adbb75afb16e0684a9:src/agents/live-model-switch.ts`, and `git show a9797214338ba31c52c796adbb75afb16e0684a9:src/agents/live-model-switch.test.ts`.
  - command: `git show a9797214338ba31c52c796adbb75afb16e0684a9:src/agents/model-selection.ts && git show a9797214338ba31c52c796adbb75afb16e0684a9:src/agents/live-model-switch.ts && git show a9797214338ba31c52c796adbb75afb16e0684a9:src/agents/live-model-switch.test.ts`
