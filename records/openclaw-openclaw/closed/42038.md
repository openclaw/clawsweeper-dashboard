## Close Comment

Closing from a Codex review. This is already implemented in current `main`, and the released `v2026.4.22` code already contains the browser exec-approval flow: the WebChat/Control UI client subscribes to `exec.approval.requested` / `exec.approval.resolved` and queues them (`ui/src/ui/app-gateway.ts:583`), renders an approval dialog with `Allow once` / `Always allow` / `Deny` (`ui/src/ui/views/exec-approval.ts:56`), and sends the selected decision back with `exec.approval.resolve` (`ui/src/ui/app.ts:815`). The embedded chat path also emits structured exec-approval events/tool payloads for web sessions (`src/agents/pi-embedded-subscribe.handlers.tools.ts:930`), with regression coverage in `src/agents/pi-embedded-subscribe.handlers.tools.test.ts:449`. Web UI/WebChat is treated as an enabled initiating approval surface (`src/infra/exec-approval-surface.ts:48`). Since the report was against `2026.3.2` and the released `v2026.4.22` tree already has this support, closing as implemented on main.
