## Summary

Close as a duplicate of open #41195; this issue's concatenated prompt is a downstream symptom of replayed Feishu WebSocket messages entering the existing same-sender inbound debounce batch with new message IDs, and the remaining fix should be tracked in the canonical reconnect/dedup issue.

## What This Changes

The issue reports Feishu WebSocket messages intermittently being delivered to the agent with previous user messages concatenated into the current request.

## Best Possible Solution

Keep #41195 as the canonical tracker and fix the Feishu replay/staleness filter before inbound debounce so replayed historical messages cannot be batched into a new agent turn.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: replayed same-chat/same-sender Feishu text events with new message IDs bypass ID dedupe and are joined by the inbound debounce flush before dispatch.

## Solution Assessment

Yes. The best maintainer path is the canonical #41195 fix, likely timestamp/grace-window replay filtering in the Feishu plugin before debounce/dispatch rather than changing core prompt history assembly.
