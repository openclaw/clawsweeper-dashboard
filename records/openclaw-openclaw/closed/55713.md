## Summary

Keep this PR open. Current main still omits Mattermost from the workspace reaction-capable platform list and from the central reactions semantics page, so the PR is not obsolete. The underlying Mattermost reaction implementation exists and is covered by tests, making this a valid docs-only contribution, but it needs the existing review fixes before merge: alphabetical ordering and explicit documentation that Mattermost requires a non-empty emoji.

## Best Possible Solution

Keep the PR open as a focused docs candidate. The best path is to revise it so `docs/reference/templates/AGENTS.md` says `Discord, Mattermost, Slack`, add the Mattermost accordion to `docs/tools/reactions.md` in alphabetical position, and explicitly document that Mattermost uses emoji names and requires a non-empty emoji for both add and remove. No runtime change appears necessary.
