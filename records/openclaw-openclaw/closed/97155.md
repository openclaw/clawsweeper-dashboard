## Summary

Keep open: this is a useful narrow replacement for the earlier socket-factory seam, but it is not merge-ready because the new WebSocket URL override is resolved after the WebSocket proxy agent is selected, so override traffic can still use the default WhatsApp host's proxy and NO_PROXY decision.

## What This Changes

Adds `OPENCLAW_WHATSAPP_WEB_SOCKET_URL` and an explicit `createWaSocket` `waWebSocketUrl` option that forward Baileys' WebSocket endpoint override, plus session and dotenv tests.

## Best Possible Solution

Keep the narrow Baileys `waWebSocketUrl` seam, but resolve proxy behavior against the effective URL, add the missing proxy/NO_PROXY regression coverage, and have maintainers explicitly accept the trusted env surface before landing.

## Solution Assessment
