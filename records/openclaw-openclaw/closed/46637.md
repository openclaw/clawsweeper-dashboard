## Summary

Keep open. Current main still captures OpenAI-compatible `reasoning_content` as thinking metadata and only enables historical reasoning stripping for Gemma 4, so the Qwen/oMLX follow-up replay path remains uncovered; the related fix PRs are closed and unmerged.

## What This Changes

The issue asks OpenClaw to prevent Qwen/oMLX `reasoning_content` from poisoning later OpenAI-compatible conversation-history requests.

## Best Possible Solution

OpenAI-compatible local/custom replay should strip or downgrade historical Qwen/oMLX-style provider reasoning blocks before follow-up requests while preserving current tool-call continuations and explicit preserved-reasoning contracts.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a second `openai-completions` turn for a Qwen/oMLX custom provider after the prior assistant turn contains a `thinkingSignature: "reasoning_content"` block; current main captures that block and does not enable `dropReasoningFromHistory` outside Gemma 4.

## Solution Assessment

Yes. The narrow maintainable fix is replay-history policy hygiene for affected strict OpenAI-compatible Qwen/oMLX-style backends, not a generic JSON-escaping patch, because current source points to replaying provider reasoning metadata into later requests.
