## Summary

Close as not actionable in this source repository. Current OpenClaw source and docs define `openclaw skills install` and gateway `skills.install` as ClawHub download/local workspace flows, while public install counts are documented as coming from logged-in `clawhub sync` snapshots; inspecting or correcting the hosted count for one ClawHub page belongs to ClawHub service/backend operations rather than `openclaw/openclaw` code.

## What This Changes

The issue asks for the ClawHub skill page install count to reflect confirmed installs, or for the product/docs to clarify which commands and events count.

## Best Possible Solution

Leave the OpenClaw source behavior as-is unless maintainers decide to change the ClawHub privacy/product contract. If the hosted count for `agenttrust/agenttrust` is wrong under the documented `clawhub sync` snapshot model, a ClawHub operator should inspect the backend metric attribution using the slug, rough install/sync dates, and commands used.

## Reproduction Assessment

No high-confidence source-level reproduction exists in this repository. The report depends on the hosted ClawHub page/backend count, while the local source review can only verify that OpenClaw install paths are download/local metadata flows and that docs point counting at `clawhub sync`.

## Solution Assessment

Not applicable as an OpenClaw source fix. The narrow maintainable path is a ClawHub service/backend metric check or a product decision to redefine what the public count means; adding install telemetry to OpenClaw would be a new privacy/product behavior, not a bug fix for this repo.
