The PR adds a Quick Settings Text-to-Speech card, per-provider voice persistence, `tts.setVoice`, `tts.preview`, UI controller wiring, tests, and a changelog entry.

## Best Possible Solution

Land or replace this PR with a rebased implementation whose UI, gateway descriptors/scopes, speech-core runtime exports, dynamic voice listing path, docs, and focused tests all describe the same TTS Settings contract for the linked issue.

## Reproduction Assessment

Yes for source-level review. Current main lacks the requested TTS Settings UI, and the PR-head defects are visible by comparing the new facade/method/UI code against the speech-core runtime API, method-scope contract, and dynamic voice-list provider contract.

## Solution Assessment

No as submitted. The direction is reasonable, but the patch needs runtime exports, gateway write-scope descriptors, and a provider-backed voice-list path before it is the narrow maintainable solution.

Full review comments:

- **[P2] Export the new TTS voice helpers from speech-core:** `src/plugin-sdk/tts-runtime.ts:115-120`
  - body: `src/tts/tts.ts` reaches speech-core through this lazy facade, but the PR adds `setTtsVoice` and `getTtsVoiceByProvider` lookups here without exporting those names from `extensions/speech-core/runtime-api.ts`. In a real gateway, `tts.status` or `tts.setVoice` will load a runtime module with no matching export.
  - confidence: 0.94
- **[P2] Classify the new TTS RPCs as write methods:** `src/gateway/server-methods-list.ts:29-30`
  - body: The PR advertises `tts.setVoice` and `tts.preview`, but does not add them to the method-scope table. The method-scope coverage requires every listed gateway method to be classified, and both active TTS operations should require `operator.write`; after rebasing onto current main, add them to the core method descriptors.
  - confidence: 0.93
- **[P2] Populate voices from provider voice listing:** `ui/src/ui/views/config-quick.ts:1091-1092`
  - body: The new card only renders a voice selector when `tts.providers` returns static `voices`, but providers such as Microsoft expose voices through `listVoices` and no static array. That means the linked Microsoft default-voice use case hides the picker; add a selected-provider voice-list RPC/path or populate the provider payload from `listSpeechVoices`.
  - confidence: 0.86
