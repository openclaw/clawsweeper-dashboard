## Summary

Close as implemented on main. A newer merged Feishu fix already covers the same empty-message failure path more generally by skipping blank-content/no-media events after media resolution, with regression coverage and changelog provenance.

## What This Changes

The PR adds an early Feishu text-message guard for whitespace-empty text payloads and a unit test asserting that routing and Feishu replies are not invoked.

## Best Possible Solution

Keep the merged main implementation from #74661 as the canonical fix because it handles the empty-content/no-media condition at the point where the handler knows whether media exists, preserving richer Feishu messages while preventing blank session turns.

## Reproduction Assessment

Yes. The reproduction path is a Feishu text event whose content parses to an empty string, such as {"text":""}; current main has a focused regression test for that event and a source guard that returns before reply dispatch/session writes.

## Solution Assessment

No for merging this PR now. The proposed guard was a valid narrow fix on its original base, but current main’s merged guard after media resolution is the better current solution because it covers blank text with no media without treating all non-text or media-bearing events as empty.
