## Summary

Current main already covers PR #41694's central request through the shipped `messaging.inferTargetChatType` channel-plugin seam. The exact `defaultTargetKind` field from the PR is not present, but plugins can now steer bare targets to direct/group/channel resolution before directory lookup, outbound resolution consumes that inferred kind, direct/user session routing is covered by tests, and the contract is documented for plugin authors.

## Best Possible Solution

Close PR #41694 as implemented on main. Keep `messaging.inferTargetChatType` plus `looksLikeId`/`resolveTarget` as the supported plugin target-resolution API, and direct any remaining provider-specific prefix or routing edge cases to focused follow-up issues against that current seam.
