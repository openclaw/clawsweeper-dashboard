## Summary

Keep this issue open. Current main still performs a redundant `chat.history` reload for normal final chat events even though the final assistant payload is already incorporated locally, and the related open maintainer PR #72325 remains the right implementation candidate for the remaining reconciliation race.

## Best Possible Solution

Keep this issue open and resolve it through #72325 or an equivalent narrow Control UI/WebChat patch. The fix should suppress redundant history reloads for same-run final assistant payloads already applied locally, preserve legitimate reloads for tool/no-message/out-of-band finals and deferred `session.message` hydration, and update the gateway, chat-event-reload, and controller tests before closing.
