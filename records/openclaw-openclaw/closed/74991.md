## Summary

Close as not reproducible on current main. The reported `conversation-logger` hook and `memory/raw/YYYY-MM-DD-conversations.jsonl` writer are not present in the repository or latest release, and the closest in-repo hook surfaces either store true UTC ISO timestamps from a `Date` or intentionally format local display text without a `Z` suffix.

## What This Changes

The issue reports that an internal hook named `conversation-logger` writes local Asia/Shanghai wall time into JSONL while marking the timestamp as UTC.

## Best Possible Solution

Keep the current core behavior: internal hook timestamps should remain `Date` objects and built-in loggers should serialize absolute instants with `toISOString()`. If a custom `conversation-logger` depends on OpenClaw APIs and still reproduces, the next report should include the hook source or plugin link plus the exact OpenClaw version and generated JSONL line.

## Reproduction Assessment

No. The current source and latest release do not contain the named hook or reported JSONL writer, and the closest bundled hook paths cannot generate the sample `16:22Z` value from a `Date` without custom formatting outside core.

## Solution Assessment

Not applicable for a code fix in this repository. The narrow maintainable path is to fix the custom/external writer if one exists, or reopen with a concrete current-main in-repo surface if OpenClaw core is actually producing the bad JSONL.
