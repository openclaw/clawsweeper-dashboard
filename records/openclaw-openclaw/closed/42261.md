## Summary

Keep this issue open. Current main still matches the reported producer-side bug: live reasoning is enabled only when an optional `onReasoningStream` callback exists, and WebChat/Control UI `chat.send` does not provide that callback, so callback-less WebSocket clients still cannot receive live `stream: "thinking"` events.

## Best Possible Solution

Keep this issue open. Fix `src/agents/pi-embedded-subscribe.ts` so `streamReasoning` depends on `reasoningMode === "stream"` and visible reasoning rather than callback presence, emit `stream: "thinking"` before optionally invoking `params.onReasoningStream`, update the Telegram-only `/reasoning stream` acknowledgement/docs if WebSocket support is intended, and add callback-absent regression coverage proving WebChat-style clients receive thinking deltas.
