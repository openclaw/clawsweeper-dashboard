## Summary

Keep open. Current main still allows the direct Google/Gemini transport to build an empty or effectively empty `contents` payload, while this PR is a narrow fix in the provider-owned conversion path and does not introduce a clear functional or security regression. The main gap is missing adjacent regression coverage before merge.

## What This Changes

This PR changes the Google Generative AI transport conversion so empty user text, filtered-to-empty user parts, and fully empty converted contents fall back to a whitespace user part.

## Best Possible Solution

Keep the fix in the Google plugin transport and land it with focused regression coverage in `extensions/google/transport-stream.test.ts` for the three new fallback paths, then run the targeted Google transport test before merge.
