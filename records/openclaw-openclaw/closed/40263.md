## Summary

Current `main` already handles the reported isolated-cron 429 path: classified rate-limit exhaustion escalates into model fallback, and exhausted retries now surface an explicit error payload that isolated cron delivery treats as a fatal visible outcome. The behavior is covered by a regression test and is present in release `v2026.4.22`.

## Evidence

- **Rate-limit exhaustion escalates to fallback:** The embedded runner counts rate-limit profile rotations and throws a `FailoverError` once the configured cap is exceeded, explicitly escalating to model fallback instead of silently ending the run.
