## What This Changes

The issue reports a 2026.4.29 regression where simple Discord replies take roughly 70-85 seconds despite a reduced OpenAI-only configuration, with delays concentrated in model resolution, auth, plugin tool scaffolding, prompt setup, stream setup, trajectory cleanup, and gateway RPC stalls.

## Best Possible Solution

Keep the current main implementation that moves optional provider/catalog work off the reply registration path, then ship it in the next release; if a post-fix build still shows 70-85 second Discord replies, open a fresh performance report with current-main or release build SHA and the same stage timing trace.
