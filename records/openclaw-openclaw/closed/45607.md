## Summary

Current `main` now loads same-day slugged `session-memory` artifacts into startup context for `/new` and `/reset`, so the reported write/read mismatch is implemented on `main` even though the hook still writes `memory/YYYY-MM-DD-slug.md` files.

## Evidence

- **Startup context now enumerates slugged memory files:** `listStartupMemoryPathsByDate()` scans `memory/`, keeps the canonical `YYYY-MM-DD.md`, and also includes up to four `YYYY-MM-DD-*.md` files for the requested date stamps before building the startup prelude.
