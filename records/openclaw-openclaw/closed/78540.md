## Summary

Keep open: current main still has a source-reproducible false `openclaw doctor` warning for the documented `memory-lancedb` memory slot. The open runtime-registration fix at https://github.com/openclaw/openclaw/pull/80211 is the right path, but it is not merged and is currently conflicting with an unresolved review blocker.

## What This Changes

`openclaw doctor` reports a missing active memory plugin when `memory-lancedb` is installed, enabled, and selected as `plugins.slots.memory`.

## Best Possible Solution

Land a memory-lancedb runtime/capability registration fix with doctor/status regression coverage, preferably by repairing and merging https://github.com/openclaw/openclaw/pull/80211.

## Reproduction Assessment

Yes, at source level. Current main documents `memory-lancedb` as the active memory slot plugin, but the plugin does not register the memory runtime that `resolveActiveMemoryBackendConfig` needs, so doctor warns on the null backend path.

## Solution Assessment

Yes for the runtime-registration direction, not for a doctor-only suppression. The maintainable fix is to make `memory-lancedb` visible through the generic memory capability/runtime contract and then validate doctor/status against that contract.
