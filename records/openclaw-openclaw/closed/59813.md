## Summary

Current `main` already bootstraps the global Undici env-proxy dispatcher in the CLI startup path before any `openclaw message ...` command reaches `runMessageAction()`, so the PR's added bootstrap inside `runMessageAction()` is redundant for the reported CLI proxy timeout repro.

## Evidence

- **CLI startup bootstraps env proxy before command routing:** `runCli()` normalizes env and calls `ensureGlobalUndiciEnvProxyDispatcher()` before command dispatch, so CLI invocations already install the global proxy dispatcher up front.
  - file: [src/cli/run-main.ts:216](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/src/cli/run-main.ts#L216)
  - command: `nl -ba src/cli/run-main.ts | sed -n '180,235p'`
  - sha: [42514156e03c](https://github.com/openclaw/openclaw/commit/42514156e03c53dec6be6c581069d8c5b2d4cd42)
- **CLI entrypoint reaches run-main:** The main entry imports `./cli/run-main.js` and invokes `runCli(argv)`, so normal `openclaw ...` execution flows through the bootstrap above.
  - file: [src/entry.ts:179](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/src/entry.ts#L179)
  - command: `nl -ba src/entry.ts | sed -n '170,195p'`
  - sha: [42514156e03c](https://github.com/openclaw/openclaw/commit/42514156e03c53dec6be6c581069d8c5b2d4cd42)

That means `openclaw message read` runs after CLI startup has already applied the proxy bootstrap.

  - sha: [42514156e03c](https://github.com/openclaw/openclaw/commit/42514156e03c53dec6be6c581069d8c5b2d4cd42)
- **Current runMessageAction has no local proxy bootstrap:** `runMessageAction()` on current `main` does not call `ensureGlobalUndiciEnvProxyDispatcher()`, yet the CLI path is still covered because bootstrap happens earlier in `runCli()`. That makes the PR change unnecessary for the reported CLI-only repro.
