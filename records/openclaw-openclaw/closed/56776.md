## Summary

Keep open. Current `main` does not show a strong fix for ordinary Control UI `chat.send` turns: the webchat handler still builds `ctx` without `MediaPath`/`MediaPaths`/`MediaType`/`MediaTypes` for normal sessions, and current tests still lock in that omission. The code does pass inline images separately in `replyOptions.images`, so some multimodal paths may work, but the specific path-based/tool-based failure described in #56776 is not clearly implemented away on `main`.

## Evidence

- **`chat.send` still omits media fields from normal webchat `ctx`:** In `chat.send`, `persistedImagesPromise` is created, but the `ctx: MsgContext` only spreads `...pluginBoundMediaFields`. Those media fields are only populated when `explicitOriginTargetsPlugin && parsedImages.length > 0`, so ordinary Control UI sessions still do not attach `MediaPaths`/`MediaTypes` to `ctx`.
