## What This Changes

Add incremental CLI backend preview streaming so channels can show progressive updates during long CLI-backed agent runs instead of only the final message.

## Best Possible Solution

Keep the merged bridge through the existing agent-event and `onPartialReply` preview pipeline; track any future backend capability or diagnostic knobs separately only if maintainers want that product surface.

## Reproduction Assessment

Yes for the pre-fix behavior: a JSONL CLI backend could emit assistant deltas while channel previews only consumed `onPartialReply`, and the linked narrower issue includes live Telegram/Claude CLI proof. Current main now has the bridge, so the old failure path should not reproduce there.

## Solution Assessment

Yes. Reusing the existing agent-event and channel `onPartialReply` path is narrower than adding a new backend-level streaming config contract, and it solves the reported final-only preview UX for eligible CLI deltas.
