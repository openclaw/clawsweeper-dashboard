## Summary

Keep this PR open, but it should not merge as-is: the auth merge fix targets a real linked regression, yet the latest diff still broadens explicit selected-agent auth order and includes an unrelated cron validation loosening.

## What This Changes

The PR changes auth-profile store merging so selected-agent profiles sort before inherited main-agent profiles, adds focused auth coverage, exposes shared test-env helpers for plugin tests, and adjusts one cron persisted-shape validator/test.

## Best Possible Solution

Land a focused auth merge fix that keeps selected-agent credentials local-first without broadening explicit provider order, retains inherited main profiles as fallback, and leaves unrelated cron/test-SDK stabilization out unless separately justified.

## Reproduction Assessment

Yes. Source inspection shows current main merges main and requested auth stores before runtime order resolution, and the linked issue describes the same-provider MiniMax local profile being ordered behind inherited main auth when profile IDs differ.

## Solution Assessment

No. The shared merge boundary is the right place to fix this, but the current PR must preserve explicit selected-agent provider order and should not loosen unrelated persisted cron payload validation.
