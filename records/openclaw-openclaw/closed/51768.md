## Summary

Keep open. Current `main` no longer contains the ACPX runtime implementation this PR edits; `extensions/acpx` is now a thin wrapper over the external `acpx/runtime` package. The shipped dependency has a newer detached queue-owner architecture with different cancel/close semantics, but I did not find repo-backed proof that it already implements this PR’s specific per-turn process-group cleanup for orphaned grandchildren, so conservative cleanup should not auto-close it.
