## Summary

Current `main` and shipped release `v2026.4.23` already restore the default operator scope set for bearer-authenticated OpenAI-compatible Gateway HTTP requests, which includes `operator.write`, so the reported 403 path is fixed.

## Evidence

- **Compat HTTP scope resolver restores default scopes:** `resolveOpenAiCompatibleHttpOperatorScopes()` returns `CLI_DEFAULT_OPERATOR_SCOPES` for shared-secret gateway auth (`token`/`password`), which includes `operator.write`.

- **`/v1/chat/completions` uses that resolver:** The OpenAI-compatible chat completions handler wires `resolveOpenAiCompatibleHttpOperatorScopes` into the authorization path for `/v1/chat/completions`.
  - file: [src/gateway/openai-http.ts:505](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/gateway/openai-http.ts#L505)
