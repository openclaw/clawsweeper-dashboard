## Summary

Issue #62255 is a valid global model-call throttling request, but the same remaining work is already covered by open issue #13615. Current main still lacks a proactive global/provider model-call limiter, queue, or budget policy; existing behavior is reactive auth-profile cooldown/fallback after rate-limit failures. The best cleanup is to close #62255 as duplicate/superseded and keep implementation/design discussion on #13615.

## Best Possible Solution

Close #62255 as duplicate/superseded by open #13615. Keep the remaining design and implementation under #13615 as an opt-in outbound provider/model policy layer with config/schema/docs, shared queue/backpressure across agents/sessions/cron, token/request budgets, 429 retry handling, and observability.
