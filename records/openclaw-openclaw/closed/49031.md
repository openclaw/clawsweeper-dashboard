## Summary

Close as duplicate/superseded. The obsolete macOS NodeServiceManager command path described here has been corrected on current main, but the remaining macOS companion `system.run.prepare` gap is still real and is better tracked by open issue #37591 and open PR #38781.

## Evidence

- **Canonical remaining tracker:** Provided GitHub context shows open issue #37591 tracks the same remaining macOS node failure: the node advertises `system.run` but not `system.run.prepare`, causing node exec to fail.
