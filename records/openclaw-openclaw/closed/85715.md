## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Defer DSML recovery until native calls are ruled out:** `src/agents/openai-transport-stream.ts:2642`
  - body: This calls the DSML recovery flush while processing each text delta. If a complete DSML block is seen in one chunk and a native tool_calls chunk arrives later in the same response, the synthetic call has already been appended and the later native call will still be appended, so one model turn can execute duplicate or conflicting tools. Buffer DSML recovery until the stream is complete or remove/reconcile synthetic calls when native tool_calls appear, and add a regression where DSML arrives before a later native call.
  - confidence: 0.88
- **[P1] Reject unparseable DSML before emitting a tool call:** `src/agents/openai-transport-stream.ts:2652-2654`
  - body: parseStreamingJson returns {} when strict and partial parsing fail, so this object check treats malformed non-empty JSON as valid tool arguments. A bad DSML body such as an incomplete object can become an executable zero-argument tool call instead of being dropped or left as text; use a strict parse result for DSML recovery and only emit when the argument body parses cleanly.
  - confidence: 0.9
- **[P2] Recover every executable wrapper kind the filter strips:** `src/agents/openai-transport-stream.ts:86`
  - body: The new parser only recognizes tool_calls, but the existing DeepSeek filter strips tool_call and function_calls wrappers too. Those executable DSML forms would still disappear from the assistant output without becoming tool calls, so the recovery path should share the wrapper contract with the filter or explicitly preserve unsupported executable wrappers as non-executed text.
  - confidence: 0.78
