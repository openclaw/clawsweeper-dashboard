## Summary

Keep this issue open. Current main still exposes SecretRefs as built-in env/file/exec sources only, documents 1Password through the generic exec provider, and has no native keyring/Keychain provider or plugin-owned SecretRef provider-source seam. Open PR #72548 by the same author is the active implementation candidate for the remaining seam and bundled keyring work, so closing this issue now would race that review.

## Best Possible Solution

Review #72548 as the active implementation candidate. If it lands, close this issue as implemented for the pluggable SecretRef provider-source seam and bundled native keyring/Keychain path, then open or link a focused native 1Password follow-up only if maintainers want that bundled rather than published as a plugin. If maintainers decline the PR, narrow this issue to the accepted keyring/Keychain, native 1Password, or plugin SecretRef provider API work, with broad cloud-provider tracking left to #13610 or focused provider issues.
