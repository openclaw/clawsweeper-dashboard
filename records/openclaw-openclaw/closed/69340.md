## Summary

This PR is a real attempt at the TUI token-corruption bug, but its one-line classifier widening is now superseded by the open generalized fix in #77335 and still has a narrow wrapping regression risk.

## What This Changes

The PR changes `src/tui/tui-formatters.ts` so hyphenated file-like tokens are treated as copy-sensitive and skip long-token chunking.

## Best Possible Solution

Close this narrow PR in favor of the canonical generalized formatter fix in #77335 while keeping #48432 tracked until that fix lands.

## Reproduction Assessment

Yes. From current-main source, `ubuntu-budgie-desktop-environment` is exactly 33 characters and lacks the existing `_`, slash, URL, Windows-drive, and mixed-alnum credential guards, so it reaches the long-token chunker and gets a visible inserted space.

## Solution Assessment

No as written. The narrow one-line hyphen guard fixes the example package-name case but also reclassifies generic hyphenated runs, while #77335 is the better current path because it covers code spans, code blocks, dotted IDs, tests, and changelog credit in one canonical PR.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep generic hyphenated tokens chunkable:** `src/tui/tui-formatters.ts:82`
  - body: This guard treats every long `[A-Za-z0-9._-]` token containing `-` as copy-sensitive. That includes `-`.repeat(60) and long hyphenated prose-like runs, which currently fall through to `chunkToken` to keep narrow-terminal segments bounded; after this change they bypass chunking and can overflow. Please narrow the hyphen heuristic or use the guarded canonical formatter fix.
  - confidence: 0.88
