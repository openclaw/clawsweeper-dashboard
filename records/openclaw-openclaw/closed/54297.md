## Summary

Current main already routes WhatsApp outbound media through Baileys, pins Baileys 7.0.0-rc.9, and that dependency cleanup path unlinks both encrypted and original temp files in the upload/metadata `finally` path as well as on encryption errors. The latest release also ships the same Baileys pin and OpenClaw media hotfix patch.

## Best Possible Solution

Close this OpenClaw issue and keep the current Baileys-backed implementation. Future Baileys upgrades should preserve the same temp-file cleanup contract or add an OpenClaw-side regression test before changing the pinned media upload path.
