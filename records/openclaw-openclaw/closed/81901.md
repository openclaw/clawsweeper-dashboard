## Summary

Keep open: current main and the latest shipped release still expose only the short 256-character plugin approval description, while the linked implementation PR is open, dirty, and previously reviewed with blockers.

## What This Changes

The issue requests additive plugin approval fields for long-form context and optional custom prompt delivery across richer chat channels.

## Best Possible Solution

Land a focused additive longDescription contract through schema, payload storage, SDK/view-model rendering, channel presentation, docs, and tests, then split deliverPrompt unless maintainers explicitly approve that lifecycle change together.

## Reproduction Assessment

Yes, by source inspection: current main rejects descriptions over 256 characters, rejects extra fields, and slices node-policy approval descriptions to 256. I did not run a live chat flow because this is an additive API feature request rather than a broken existing contract.

## Solution Assessment

Unclear as a complete proposal. The longDescription field is a plausible narrow path, but deliverPrompt:false and exact channel ownership should be decided by maintainers or split into a follow-up.
