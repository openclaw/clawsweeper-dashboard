## Summary

Keep open. Current main and the latest release still gate compaction notices on the active turn `onBlockReply` callback, while docs and config types promise `notifyUser` notices when compaction starts and completes.

## What This Changes

The issue requests that `agents.defaults.compaction.notifyUser` notices be delivered for asynchronous or non-inbound compactions instead of depending only on the active turn streaming callback.

## Best Possible Solution

Deliver `notifyUser` compaction notices through the canonical session/outbound delivery path when the active turn callback is unavailable, with regression coverage for callback and async compaction paths.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: enable `agents.defaults.compaction.notifyUser`, emit a compaction event or run preflight compaction without `opts.onBlockReply`, and the current notice helper has no delivery path.

## Solution Assessment

Yes. Decoupling the notice from `onBlockReply` is the right direction; the maintainable fix is to reuse existing session/outbound delivery rather than only logging the missing callback.
