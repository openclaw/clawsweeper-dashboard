## Summary

Close #67184 as superseded by #70947. The requested non-image WebChat attachment support is still not implemented on current main, but #67184 only changes the UI accept/filter helper and leaves the downstream serializer, preview, gateway parser, and tests image-only. #70947 is the newer open canonical PR for the same remaining WebChat generic-file attachment work.

## Best Possible Solution

Close #67184 as superseded and keep the remaining implementation review on #70947 or a maintainer-owned successor. The final fix should cover the full Control UI plus gateway path: accept/filter, filename/MIME preservation, non-image preview/rendering, chat.send serialization, inbound media persistence or agent-visible refs, and regression tests.
