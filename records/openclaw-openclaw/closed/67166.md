## Summary

Current `main` already implements inline rendering for user-uploaded images in Control UI/WebChat, including the persisted history/reload path described in issue #67166. The changelog records this exact fix as shipped in `v2026.4.18`.

## Evidence

- **Optimistic user-image rendering on send:** `sendChatMessage` appends uploaded images to the user message as `content` blocks of type `image`, so Control UI can render them inline immediately after send.
  - file: [ui/src/ui/controllers/chat.ts:310](https://github.com/openclaw/openclaw/blob/6f72b74cecf8d1fec5c147ae932161bf0f37a68b/ui/src/ui/controllers/chat.ts#L310)

- **History-backed transcript image extraction:** `extractImages` reads persisted transcript `MediaPath`/`MediaPaths` entries, recognizes image media types/extensions, and appends them for rendering. This is the reload/history path the issue reported missing.
  - file: [ui/src/ui/chat/grouped-render.ts:114](https://github.com/openclaw/openclaw/blob/6f72b74cecf8d1fec5c147ae932161bf0f37a68b/ui/src/ui/chat/grouped-render.ts#L114)
