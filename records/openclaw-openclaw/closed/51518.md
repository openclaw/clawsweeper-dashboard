## Summary

Current main already supports gateway-loaded plugins spawning subagents after the request-scoped context is gone. `agent_end` hooks are still fired asynchronously, but gateway plugin runtimes are late-bound to the gateway subagent runtime and `dispatchGatewayMethod(...)` falls back to the server's gateway context when no request scope is present, which covers the reported `agent_end` case. The same code is present in released tag `v2026.4.22`.

## Evidence
