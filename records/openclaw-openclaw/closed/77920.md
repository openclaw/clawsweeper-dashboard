## Summary

Current main resolves Telegram forum topic messages against the bare group chat ID for group config, then applies topic/group `requireMention` before dispatch, so the reported 2026.4.27 bypass is already fixed and shipped in v2026.5.4.

## What This Changes

Issue reports that Telegram forum topic messages with `requireMention: true` are processed by all configured agents because topic-qualified chat IDs bypass mention gating.

## Best Possible Solution

Close this as fixed and direct affected 2026.4.27 users to upgrade to v2026.5.4 or newer.

## Reproduction Assessment

No for current main: I did not run a live Telegram forum, and source inspection shows the reported topic-suffix lookup failure is no longer present. The old 2026.4.27 behavior is plausible from the report, but current main routes the same policy through the bare chat ID and topic config separately.

## Solution Assessment

Yes: the current implementation keeps topic IDs for session isolation while resolving access and mention policy against the parent group plus topic overrides, which is the narrow maintainable fix for this class of bypass.
