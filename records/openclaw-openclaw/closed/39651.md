## Summary

Current main already covers the reported local-Ollama 'only talks, won’t act' regression: local onboarding now defaults to the `coding` tool profile instead of `messaging`, weak no-action turns are surfaced as explicit errors instead of silently stalling, and leaked Qwen/Ollama tool-call text is stripped from visible replies/history.

## Evidence

- **Local onboarding now enables coding tools by default:** `applyLocalSetupWorkspaceConfig` sets `ONBOARDING_DEFAULT_TOOLS_PROFILE` to `"coding"` and applies it when `tools.profile` is unset, directly addressing the regression cluster where legacy/local setups were left without filesystem/runtime tools.
