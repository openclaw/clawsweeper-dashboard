## Summary

PR #48225 is superseded by the shipped #61830 solution: current main and v2026.4.24 already expose a scoped `tools.web.fetch.ssrfPolicy.allowRfc2544BenchmarkRange` opt-in for Clash/Surge/mihomo fake-IP environments, wire it into `web_fetch`, document it in generated config metadata, and test both strict default blocking and opt-in success. The exact automatic env-proxy retry design from this PR did not land, but the user-visible fake-IP `web_fetch` failure is now covered by the maintained config path.

## Best Possible Solution

Close this PR as superseded by the shipped #61830 config-based fix. Keep the explicit `tools.web.fetch.ssrfPolicy.allowRfc2544BenchmarkRange` path as the maintained solution for fake-IP proxy setups, and require a new focused proposal if maintainers want automatic env-proxy retry behavior later.
