## Summary

Keep this PR open. Current main still does not contain the PR's central cron handoff from the run-scoped key to the canonical agent key for sandbox policy, and the linked bug #73586 remains an open implementation target. The PR is a focused candidate, but the sandbox context fallback change still needs maintainer review before merge.

## Best Possible Solution

Keep this PR open as the active implementation candidate for #73586. The best maintainer path is to review the narrow sandboxSessionKey handoff, require regression coverage proving Docker backend initialization is not reached when agents.defaults.sandbox.mode is "off", and verify off, non-main, and all behavior across isolated cron plus the heartbeat/sub-agent surfaces named in the linked bug. The context.ts non-null assertion should either be justified by caller proof or split out.
