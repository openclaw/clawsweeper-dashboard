The issue requests an official lightweight Docker image variant with fewer preinstalled plugins and reduced node_modules size.

## Best Possible Solution

Keep the shipped slim/pruned Docker release path and the explicit plugin install flow instead of adding another parallel minimal image tag.

Yes: the maintainable solution is the existing slim runtime plus `OPENCLAW_EXTENSIONS` keep-list and explicit ClawHub/plugin installs, which avoids shipping all optional plugin dependencies by default.
