## Summary

Current `main` already handles OpenResponses `input_file` base64 payloads by decoding `source.data` directly and using `filename` only as metadata, so the reported same-name workspace-file precedence bug is not present on `main` and is also present in the latest shipped release inspected (`v2026.4.22`).

## Evidence

There is no workspace/path lookup in this branch.
