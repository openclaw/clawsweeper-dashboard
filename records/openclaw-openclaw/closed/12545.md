## Summary

Current main no longer stores any Nostr private key material in `extensions/nostr/src/nostr-state-store.ts`; that file now persists only non-secret bus/profile metadata, so the reported plaintext-key-in-state-file condition is obsolete.

## Evidence

- **Current state-store schema contains no private key field:** `NostrBusState` and `NostrProfileState` only define timestamps, recent event IDs, last published event ID, and relay publish results. No secret/private-key field is part of either persisted shape.

## Risks / Open Questions

- The shallow local history did not expose the exact landing commit for this behavior, so I could not pin a precise fix SHA or first shipping release.
- Nostr private keys can still be supplied directly in config if an operator chooses `--private-key` instead of `--use-env`; that is a separate at-rest config choice, not the reported `nostr-state-store.ts` plaintext-state-file bug.
