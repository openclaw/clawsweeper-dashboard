## Summary

Keep this issue open. Current main only partially addresses the report: `agents.list[].verboseDefault` is accepted now, but `agents.list[].elevatedDefault` is still absent from the strict per-agent schema, public agent config type, and resolved agent config path; live GitHub also shows an open PR with closing syntax for this issue, so it should stay open until that PR or an equivalent replacement lands.

## What This Changes

Allow per-agent `agents.list[].verboseDefault` and `agents.list[].elevatedDefault` defaults instead of forcing operators to use only global `agents.defaults.*` settings.

## Best Possible Solution

Review, rebase, and land https://github.com/openclaw/openclaw/pull/74643 or an equivalent narrow fix that aligns schema, public types, generated config metadata, docs, resolver precedence, and regression coverage.

## Reproduction Assessment

Yes. Current main validates `agents.list[]` through strict `AgentEntrySchema`, and that schema still omits `elevatedDefault`; because `AgentsSchema` uses that entry schema, the reported `openclaw config set agents.list.0.elevatedDefault ask` rejection path remains source-reproducible without running the CLI.

## Solution Assessment

Yes. The narrow maintainable solution is to accept and document the missing per-agent fields and carry them through the existing per-agent default precedence; the open closing PR is the canonical implementation path to review or replace.
