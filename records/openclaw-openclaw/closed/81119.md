## Summary

## What This Changes

Request to lift or clarify the ACP `mcpCapabilities` HTTP/SSE guard so OpenClaw agents can consume remote MCP transports without local patches or bridge processes.

## Best Possible Solution

Keep the shipped `mcp.servers` gateway/agent configuration path for native MCP transports; consider ACP per-session MCP only as a separate design decision if maintainers want that protocol surface.
