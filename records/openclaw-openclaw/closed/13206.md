## Summary

Current main already supports per-agent identity (name/avatar) via `agents.list[].identity`, surfaces it in the Control UI, and threads it into outbound identity handling.

## Close Comment

Closing as implemented on current `main` after Codex review. OpenClaw already supports per-agent identity through `agents.list[].identity` (including `name` and `avatar`) in `docs/gateway/configuration-reference.md:1763`, and the CLI explicitly writes those fields via `openclaw agents set-identity` in `docs/cli/agents.md:163`. The Control UI resolves the displayed assistant identity from the selected agent in `src/gateway/assistant-identity.ts:81`, and gateway/session payloads expose that identity data in `src/gateway/session-utils.ts:688`. Current `main` at `3361593442e295e9bb1f55cbfb38861ea9413e45` therefore already covers the requested per-agent name/avatar behavior, even though the shipped schema is `agents.list[].identity` rather than the proposed `ui.agents` shape.
