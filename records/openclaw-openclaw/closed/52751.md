## Summary

Keep open. Current main still exposes compaction and context pruning as defaults-only config, and this PR branch makes per-agent keys parse but does not thread the effective values into the standard Pi runtime consumers. The related open PR #69988 overlaps the compaction half with deeper runtime plumbing, so this needs maintainer consolidation rather than cleanup closure.

## What This Changes

The PR adds per-agent `compaction` and `contextPruning` type/schema fields, extracts shared Zod schemas, and returns those fields from agent config resolution.

## Best Possible Solution

Land one canonical implementation that accepts both per-agent keys, merges each agent override over `agents.defaults`, passes the effective settings through Pi settings, extension factories, direct compaction/reload paths, generated config surfaces, and docs, then resolve #52732 and any overlapping PR only after that implementation merges.
