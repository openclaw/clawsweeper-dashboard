## Summary

Keep open. Current main still wires the plugin API resolver to the generic user-path resolver even though the SDK docs describe `api.resolvePath` as plugin-root-relative, and the issue is now paired with open PR #74718 using closing syntax. That PR is the right place to finish the fix, so this issue should not be closed before the PR merges.

## What This Changes

The issue requests that `api.resolvePath` for externally installed extension plugins return a plugin-root-relative path during service startup instead of `undefined` or a process-CWD path.

## Best Possible Solution

Land the narrow root-aware resolver in the paired PR after any required review fixes, preserve existing absolute and home-relative behavior, add regression coverage for service-start use of a captured plugin API, and then close this issue through the merged PR.
