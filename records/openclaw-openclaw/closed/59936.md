## Summary

Current `main` fixes this in the executor/runtime path rather than the `openclaw setup` template: unsupported or reasoning-mandatory thinking settings are retried automatically, and OpenRouter requests no longer require legacy `reasoning_effort` injection in the default/off case.

## Evidence

- **Setup template is still minimal:** `setupCommand()` still only persists `agents.defaults.workspace` and `gateway.mode`; there is no new required `reasoning_effort` or `format` key in the generated config, so the reported beta failure is no longer solved via template changes.
