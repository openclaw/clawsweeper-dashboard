## Summary

Current `main` already addresses both reported failure modes for `existing-session` Chrome MCP attach: it uses the Chrome DevTools MCP tool names (`list_pages`) instead of `browser_tab_list`, and it now waits for tabs to become usable after `--autoConnect` before declaring the attach ready.
