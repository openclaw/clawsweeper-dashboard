## Summary

Current main already addresses the reported QMD `openclaw memory search` CLI hang. The CLI search path now uses a transient `purpose: "cli"` manager, closes it in a `finally` path, creates non-cached QMD CLI managers, skips watcher/background update startup in QMD CLI mode, and bounds QMD/mcporter subprocess searches by configured timeouts. The shipped v2026.4.26 changelog includes this one-shot memory CLI fix.

## Best Possible Solution

Keep the shipped transient one-shot memory CLI manager path. If users still see search failures on current releases, route intermittent remote embedding socket failures to the separate open #71784 thread, or ask for a fresh QMD-specific report with current OpenClaw/QMD versions, `searchMode`, logs, and whether the process hangs before output or only exits slowly after output.
