## Summary

Close #63925 as superseded by the already-linked open PR #63939, which is the focused implementation thread for exposing `web.messageTimeoutMs`. Current `main` still only accepts `web.enabled`, `web.heartbeatSeconds`, and `web.reconnect.*`, and the WhatsApp watchdog timeout remains passed from internal tuning/defaults, so this is not implemented-on-main yet.

## Best Possible Solution

Use PR #63939 as the canonical review and implementation thread for this feature. The best code path is the narrow config addition proposed there: add `web.messageTimeoutMs` to the config type/schema/help/docs, resolve it in the WhatsApp web monitor while preserving the 30-minute default, and keep test-only tuning overrides higher precedence.
