## Summary

Keep open. Current main has runtime-pair tiers and the 20 runtime tool fixtures, but the default-tool check is not yet a hard required-tool gate: the standard runtime-tool rows still carry tracking/knownHarnessGap metadata, and release checks run the advisory runtime parity report rather than the `qa coverage --tools` gate.

## What This Changes

Request to turn runtime tool fixtures into a strict required default-tool Pi/Codex gate while keeping optional or known-gap tools report-only.

## Best Possible Solution

Wire the tool-coverage verdict into the standard runtime parity release or scheduled lane, remove tracking from fixed required rows, and leave only genuinely open searchable/deferred gaps linked to https://github.com/openclaw/openclaw/issues/80319 as report-only.

## Solution Assessment
