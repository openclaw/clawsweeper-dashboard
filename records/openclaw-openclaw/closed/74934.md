## What This Changes

The PR modifies `src/agents/pi-embedded-runner/run/attempt.ts` so the default tool-result context guard prefers `params.contextTokenBudget` before falling back to model `contextWindow`, `maxTokens`, or `DEFAULT_CONTEXT_TOKENS`.

## Best Possible Solution

Land the targeted fallback change, preferably with a small wiring regression for the attempt call site, then let the linked bug close through the PR merge.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by configuring a model where resolved `contextTokenBudget` differs from `model.contextWindow`; current main resolves the larger/different budget but installs the default guard with the model window, and the guard thresholds derive directly from that supplied value.

## Solution Assessment

Yes. The proposed fix is the narrowest maintainable path because the attempt already receives the resolved budget and nearby SessionManager/context-engine paths already use it; no broader normalization or policy change is needed for this bug.
