## Summary

Current main already protects the Nostr profile mutation surface behind gateway auth, trusted-operator runtime scope, explicit `operator.admin` checks, and loopback-only mutation guards. The changelog records this exact security issue as fixed in release `v2026.2.12`.

## Evidence

- **Nostr route is not unauthenticated:** The Nostr profile HTTP route is registered with `auth: "gateway"` and `gatewayRuntimeScopeSurface: "trusted-operator"`, so it is not exposed as a plugin-managed public endpoint.
  - file: [extensions/nostr/index.ts:86](https://github.com/openclaw/openclaw/blob/3361593442e295e9bb1f55cbfb38861ea9413e45/extensions/nostr/index.ts#L86)
