## Summary

Current main already has a shipped, documented repair path for the user-visible problem: `openclaw sessions cleanup --fix-missing` prunes session-store entries whose resolved transcript files are missing, and doctor points users to that dry-run/enforce flow. The PR's automatic gateway save-time scan is a different workflow, but the missing-transcript cleanup itself is implemented and released.

## Best Possible Solution

Keep the shipped `openclaw sessions cleanup --fix-missing` plus doctor guidance as the supported cleanup path for missing transcript files, and close this PR rather than adding automatic gateway save-time deletion. Revisit automatic pruning only with a focused maintainer-approved design.
