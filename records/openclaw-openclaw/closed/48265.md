  - body: When output contains only [[SPLIT]] markers, the parser falls back to returning the original marker text. That makes delivery send the control token to users instead of suppressing an empty payload.
  - confidence: 0.94
- **[P2] Suppress split-only media captions:** `src/infra/outbound/deliver.ts:1256`
  - body: For media payloads whose caption is only split markers, the media path can use the original marker text as the caption. That leaks [[SPLIT]] with the attachment instead of sending an empty caption and no follow-up.
  - confidence: 0.92
- **[P2] Store the full opening fence marker:** `src/utils/split-tag.ts:87-88`
  - body: The parser stores only the first fence character, so a three-backtick line can close a four-backtick fence. That lets [[SPLIT]] inside still-open code blocks split outbound content.
  - confidence: 0.96
- **[P2] Buffer split markers across streamed chunks:** `src/agents/pi-embedded-subscribe.ts:778`
  - body: Streaming splits each emitted chunk independently. If the model emits a marker across chunk boundaries, such as [[SP then LIT]], the raw directive leaks and no split occurs.
  - confidence: 0.88
- **[P2] Do not dedupe adjacent split segments:** `src/agents/pi-embedded-subscribe.ts:778-780`
  - body: The streaming path feeds split segments into the existing lastBlockReplyText dedupe. Output like OK [[SPLIT]] OK will drop the second intended bubble because both segments are identical.
  - confidence: 0.86
- **[P2] Reject relative media local roots:** `src/media/local-roots.ts:73`
  - body: media.localRoots expands host file access, but relative entries are resolved against the gateway working directory and added to the allowlist. Config validation should require absolute, normalized paths.
  - confidence: 0.9
