## Label Justifications

- `P1`: The reported release bug disables semantic memory recall after safe reindex, breaking a real agent memory workflow until current main is released.
- `impact:session-state`: The issue is about memory index identity state being lost, which suppresses semantic recall and affects long-term context behavior.

## Summary

Current main already resets the metadata write cache before safe reindex writes into the temporary database and includes regression coverage, so the reported release bug can close as implemented on main. The fix is current-main-only and is not in v2026.6.1.

## What This Changes

Fix builtin memory safe reindex so unchanged-index rebuilds still write `memory_index_meta_v1` into the swapped-in temporary database.

## Best Possible Solution

Keep the current-main fix and let the next release carry it; the related provider-empty-model report remains a separate open path at https://github.com/openclaw/openclaw/issues/90042.

## Reproduction Assessment

Yes, we have a high-confidence source-level reproduction path: v2026.6.1 switches to the temp DB without clearing `lastMetaSerialized`, so `writeMeta` can skip the INSERT into an empty temp `meta` table. I did not run a live repro because this review was read-only.

## Solution Assessment

Yes, the current-main fix is the narrowest maintainable repair: clear the serialized-meta cache when the manager switches DB handles, then keep `writeMeta` idempotent within a single DB handle.
