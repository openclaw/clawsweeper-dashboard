## Summary

## What This Changes

The issue reports a v2026.5.12 Telegram regression where multi-account isolated-ingress startup stalls the event loop during `channels.telegram.start-account`.

## Best Possible Solution

Keep the current-main startup probe limiter and ask for a fresh report only if the same stall reproduces from a build that includes the limiter fix.

## Reproduction Assessment

Yes for the reported release path: the issue includes concrete v2026.5.12 liveness and fetch-timeout logs, and the release SHA lacks the startup limiter. I did not establish a live failing current-main path because current main contains the fix.

## Solution Assessment

Yes. The current-main fix targets the maintainable boundary by limiting Telegram-owned startup `getMe` fanout and leaving the channel lifecycle tracker intact, instead of making the long-running monitor untracked fire-and-forget work.
