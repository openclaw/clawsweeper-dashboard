## Summary

Current main already routes normal subagent completion updates through the detached task delivery path instead of repeatedly waking heartbeat from the parent session, and the heartbeat runner now consumes inspected pending events exactly once so stale wake/event state is not replayed on later turns.

## Evidence

- **Subagent runs now carry requester-origin delivery metadata into the detached task system:** Subagent run registration stores `requesterOrigin` on the run record and immediately creates a `runtime: "subagent"` task run with that same requester origin, so completions have a direct delivery target instead of only a parent-session fallback.
  - file: [src/agents/subagent-registry-run-manager.ts:312](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/agents/subagent-registry-run-manager.ts#L312)

  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Normal task terminal delivery is direct; heartbeat wake is now fallback-only:** `maybeDeliverTaskTerminalUpdate()` only enqueues a system event and calls `requestHeartbeatNow()` when there is no deliverable requester origin. When a requester origin exists, it sends the completion directly via `sendMessage()` and marks delivery as `delivered`.
  - file: [src/tasks/task-registry.ts:984](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/tasks/task-registry.ts#L984)

  - file: [src/infra/heartbeat-runner.ts:564](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/infra/heartbeat-runner.ts#L564)

  - file: [src/infra/heartbeat-runner.ts:966](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/infra/heartbeat-runner.ts#L966)
