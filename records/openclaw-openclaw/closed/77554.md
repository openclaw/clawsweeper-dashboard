## Summary

Keep-open/repair verdict for the review snapshot: the patch still routes the new BTW model-refresh workspace lookup from `sessionKey` alone, so non-default `/btw` calls without a session key can refresh the worker agent with the default workspace. The `maintainer` label also means cleanup automation should not close this PR as stale or obsolete.

## What This Changes

The PR threads a resolved `workspaceDir` into BTW, embedded-run, compaction, and PDF model-refresh calls, adds focused expectations, and updates the changelog.

## Best Possible Solution

Keep the embedded-run, compaction, and PDF workspace threading, but change BTW to carry the trusted command/session agent or already resolved workspace into `runBtwSideQuestion` and cover the non-default no-sessionKey case.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: invoke `/btw` for `params.agentId = worker-1` without `sessionKey`; command routing uses the worker `agentDir`, while the PR's new runner lookup falls back to the default workspace.

## Solution Assessment

No. Passing `workspaceDir` into model refreshes is the right direction, but the BTW path should use the command/session agent or already resolved workspace instead of recomputing from `sessionKey` alone.
