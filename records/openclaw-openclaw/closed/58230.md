## Summary

Current main already contains the relevant Telegram/startup and restart-loop hardening: disabled Telegram accounts are skipped before startup, Telegram polling restarts stay in-process, and stale-gateway cleanup now excludes the current process and its ancestors. The changelog shows these fixes shipped after 2026.3.28.

## Evidence

- **Disabled Telegram accounts are not started:** `resolveTelegramAccount` derives `enabled` from `channels.telegram.enabled !== false`, and gateway startup refuses to call `startAccount` when the resolved account is disabled.
  - file: [extensions/telegram/src/accounts.ts:148](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/extensions/telegram/src/accounts.ts#L148)
- **Gateway skips disabled channel accounts:** `startChannelInternal` resolves the account, checks `enabled`, and returns early with a disabled runtime status instead of starting the provider.
  - file: [src/gateway/server-channels.ts:351](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/gateway/server-channels.ts#L351)
- **Telegram restart path is polling-local, not gateway self-restart:** The current Telegram watchdog stops/restarts only the polling runner with backoff; there is no gateway restart helper call in the Telegram runtime path reviewed here.
  - file: [extensions/telegram/src/polling-session.ts:313](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/extensions/telegram/src/polling-session.ts#L313)
  - command: `rg -n "triggerOpenClawRestart\(|scheduleGatewaySigusr1Restart|process\.kill|process\.exit" extensions/telegram/src -g '!**/*.test.ts'`
- **Stale PID cleanup now excludes self and ancestors:** The stale-gateway cleanup explicitly collects the current process and ancestor PIDs and filters them out before termination, preventing supervisor restart loops caused by killing the active gateway lineage.
  - file: [src/infra/restart-stale-pids.ts:107](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/infra/restart-stale-pids.ts#L107)
- **Shipped fixes are recorded after 2026.3.28:** The changelog records follow-up fixes for Telegram polling hardening and Linux/systemd restart-loop hardening after the reported version, including Telegram polling transport/liveness fixes and gateway restart-loop fixes.
  - file: [CHANGELOG.md:508](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/CHANGELOG.md#L508)
