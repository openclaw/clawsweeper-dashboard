## Summary

Current main already contains ACPX recovery paths for long-running session loss: early `acpx` process exits are retried with a fresh runtime handle, and stale persistent-session resume failures are retried after clearing persisted ACPX resume state. The same behavior is present in the shipped `v2026.4.23` tree, which is newer than the reported `2026.3.28` environment.

## Evidence
