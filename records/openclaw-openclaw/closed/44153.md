## Summary

Current main already preserves parent external routing for inter-session `sessions_send` traffic, including Telegram, so the reported delivery-context flip is implemented/fixed in code that is also present in release `v2026.4.22`.

## Evidence

- **Inter-session provenance is wired into session routing:** `initSessionState` reads `ctx.InputProvenance`, detects `kind === "inter_session"`, and passes that flag into both `resolveLastChannelRaw` and `resolveLastToRaw`, which is the exact path used for `sessions_send` deliveries.
