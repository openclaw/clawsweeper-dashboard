## Summary

Current main implements the central request in memory-core: built-in Dreaming now ingests redacted session transcripts into a per-day short-term corpus, records those as short-term recall signals, supports optional inline daily-note phase output, and keeps durable promotion isolated to Deep promotion into MEMORY.md. The implementation is documented and shipped in v2026.4.24.

## Best Possible Solution

Keep the shipped memory-core Dreaming design: platform-owned transcript ingestion feeds the short-term dreaming corpus, optional inline storage can write managed daily-note phase blocks, and Deep promotion remains the only durable MEMORY.md writer. Future refinements should be narrower follow-ups against this pipeline rather than leaving this broad feature request open.
