## Summary

Current `main` does not yet contain the PR's substantive fixes. The heartbeat runner still inspects the forced/base session before remapping isolated heartbeats onto the `:heartbeat` sibling lane, and the public-surface loader still has no retry path when `require()` of source artifacts fails on emitted-style `.js` imports. The PR discussion is useful and shows follow-up fixes, but the item is not safely closable as already implemented or obsolete.

## Evidence

- **Heartbeat preflight still inspects the pre-isolation session key on main:** `resolveHeartbeatPreflight()` resolves a single `session` from the forced session key and immediately peeks pending events from `session.sessionKey`; later, `runHeartbeatOnce()` checks the busy lane for that same `sessionKey` before only afterwards remapping isolated runs onto `runSessionKey = isolatedSessionKey`. That means forced-session isolated heartbeats can still inspect/drain or busy-check the wrong lane on current `main`.
