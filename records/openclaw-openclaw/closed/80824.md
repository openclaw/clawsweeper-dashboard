## Summary

Keep open: the branch addresses a real OAuth retry problem and has usable proof, but latest head still lets forced refresh return an adoptable main-store or external credential without calling the refresh adapter in some agentDir paths.

## What This Changes

The PR adds terminal OAuth refresh-failure caching, threads a force-refresh flag through auth-profile OAuth resolution, removes the Codex app-server temporary expires:0 write, and adds focused auth-profile/Codex tests.

## Best Possible Solution

Keep the refresh-failure cache design, but make forceRefresh bypass every usable-credential adoption fast path and cover local-agent plus inherited-main Codex refresh cases before merging.

## Reproduction Assessment

Yes, at source level. Current main lacks terminal failure caching, and the PR's force-refresh bug can be reproduced with an agentDir-backed OAuth profile plus an adoptable main-store credential or usable external bootstrap credential.

## Solution Assessment

No, not yet. The cache and hashed fingerprint approach is a good fit, but forceRefresh must bypass the main-store and external-bootstrap adoption returns before this is the narrow correct fix.
