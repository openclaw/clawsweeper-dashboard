## Summary

Close as superseded by the broader shared-Gateway RBAC tracker that was closed as not planned. Current main still has fixed Gateway roles plus scoped operator tokens, but not configurable named roles; adding them would reopen the same Gateway authorization and multi-user trust-boundary decision.

## What This Changes

The issue requests configurable custom Gateway roles, role-to-scope mappings in config, pairing-time role assignment, and an `openclaw roles create` CLI.

## Best Possible Solution

Keep the current fixed-role plus scoped-token model in core; use separate Gateways/hosts, trusted-proxy scope headers, or external ClawHub/plugin sidecars for multi-user role separation unless maintainers reopen a core RBAC RFC.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs give a high-confidence current-main check that custom named Gateway roles and `gateway.roles` do not exist while scoped operator tokens do.

## Solution Assessment

No for the requested direct core change. The overlapping RBAC direction was closed as not planned, and current docs point users toward separate trust boundaries or external proxy/plugin approaches rather than configurable core roles.
