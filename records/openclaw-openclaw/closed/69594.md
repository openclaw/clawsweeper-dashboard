## Summary

Current main already fixes the reported phantom cron announce status: isolated cron fallback delivery now goes through the real outbound adapter, only records delivered after a non-empty successful delivery result, and keys direct-delivery dedupe per cron execution so recurring scheduled runs cannot reuse a prior delivered result.

## Best Possible Solution

Close this core issue as implemented. Keep the channel-agnostic cron direct-delivery and per-execution idempotency behavior on main; if openclaw-weixin or WeCom still drops messages on a current release, track that as a fresh plugin-specific report with current OpenClaw version, plugin version, run-log delivery trace, and gateway/plugin send logs.
