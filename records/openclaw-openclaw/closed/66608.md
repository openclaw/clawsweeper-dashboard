## Summary

Keep open. Current main still advertises bundled Z.AI image understanding through `zai/glm-4.6v`, but the reviewed code does not show a source-backed fix for the reported immediate `Request was aborted` failure. The image tool still uses the generic media-understanding completion path with a single timeout AbortController, and the OpenAI-compatible completions transport still builds streaming requests. Recent main changes around media-understanding model resolution and reasoning-only retries do not specifically address Z.AI `glm-4.6v` immediate aborts or add a Z.AI image regression/live proof.

## Best Possible Solution

Keep #66608 open and treat it as a bundled Z.AI media-understanding bug. The best next maintainer path is to add a focused regression or live-gated test around the `image` tool with `zai/glm-4.6v`, verify whether Z.AI vision needs non-streaming completions, different prompt placement, disabled reasoning/tool_stream, or provider stream-wrapper application in the media-understanding path, then patch `src/media-understanding/image.ts`, the Z.AI plugin, or the OpenAI-compatible transport accordingly.
