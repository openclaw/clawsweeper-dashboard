## Summary

Keep open. The PR addresses the reported API-key billing copy bug for profile-backed OAuth/token paths, but the patch still misses supported no-profile OAuth/token auth such as Claude CLI synthetic auth, so one narrow correction is needed before merge.

## What This Changes

The PR makes Anthropic billing-shaped error copy accept an auth mode and threads authProfileMode through assistant formatting, failover, lifecycle subscription, payload synthesis, and focused tests.

## Best Possible Solution

Carry the canonical resolved credential mode into all billing-error formatting paths for both profile-backed and no-profile OAuth/token auth, while preserving existing API-key billing copy and classifier behavior.

## Reproduction Assessment

Yes, source-reproducible. Current main routes billing-shaped errors to API-key/top-up copy, and supported OAuth/token auth can exist without an auth profile via Claude CLI synthetic auth.

## Solution Assessment

No, not as written. The fix should use the resolved auth mode, not only the auth-profile-store type, so no-profile OAuth/token routes also receive neutral recovery copy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

- **[P2] Use resolved auth mode without an active profile:** `src/agents/pi-embedded-runner/run.ts:1489-1492`

Fall back to the resolved auth mode when no active profile is present.
