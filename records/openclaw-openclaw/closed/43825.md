## Summary

Current main already treats pre-reply recall and prompt injection as plugin/context-engine behavior, and bundled plugins (`active-memory`, `memory-lancedb`) implement that path today. Adding a new core `agents.*.memorySearch.recallBeforeResponse` surface does not fit the repo's current scope direction.

## Evidence
