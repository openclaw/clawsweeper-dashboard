## Summary

Keep open. Current main has useful Bonjour/ciao mitigations for cancellation rejections and stuck probing/announcing loops, but it still does not cover the issue's exact `IP address version must match` / netmask assertion or contain ciao synchronous `uncaughtException` crashes. The issue is recent and has fresh 2026.4.24 comments showing the same ciao crash family is still user-visible.

## Best Possible Solution

Keep the issue open and harden the bundled `bonjour` plugin or its dependency boundary so malformed or inconsistent mDNS packets cannot terminate the gateway. The likely maintainer path is to add exact coverage for the `IP address version must match` / netmask assertion, cover both IPv4 interface assertion directions, make ciao packet-handler failures non-fatal before they reach the global `uncaughtException` exit path, and retain the documented disable-Bonjour workaround as a deployment mitigation rather than the final fix.
