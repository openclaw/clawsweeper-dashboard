## Summary

Keep open: the patch targets a real provider-stream timeout, but it currently suppresses the new keepalive when reasoning output is disabled, which is one of the linked report's explicit failing configurations. It also has only synthetic/local verification, so it still needs a code update plus redacted real OpenAI-compatible stream proof before merge.

## What This Changes

The PR changes the OpenAI-compatible chat-completions stream parser to emit zero-length thinking activity for usage-only chunks with positive reasoning tokens, with regression tests for usage-only and trailing-usage cases.

## Best Possible Solution

Land a fix that treats positive reasoning-token usage chunks as non-visible stream progress even when reasoning output is disabled, with an `emitReasoning: false` regression test and redacted live sidecar proof.

## Reproduction Assessment

Yes, by source inspection: current main consumes usage-only chunks without yielding assistant events, and the PR still does that when `emitReasoning` is false. I did not run the live Vertex sidecar setup.

## Solution Assessment

No. The parser is a plausible fix location, but the keepalive is still coupled to visible reasoning emission; the safer fix preserves hidden-reasoning suppression while still resetting the idle watchdog.
