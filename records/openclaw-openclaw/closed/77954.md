## Summary

Close as ClawHub/plugin-scope: OpenClaw already exposes runtime plugin hooks for pre-tool blocking, approval, parameter rewriting, and post-tool observation, while new optional skills or policy bundles belong outside core unless a missing plugin API is identified.

## What This Changes

The issue requests a Skill Hook mechanism that automatically runs configured skills before and after tool execution, especially before `exec` commands.

## Best Possible Solution

Publish the security and optimization logic as a ClawHub code plugin using before_tool_call/after_tool_call, optionally bundling skills for agent guidance, without adding a new core skill-hook YAML contract.

## Reproduction Assessment

Not applicable; this is a feature/surface request, not a broken existing behavior. Source and docs inspection show the supported runtime interception path is plugin hooks, while skills remain instruction and command-dispatch assets.

## Solution Assessment

Yes for closing from core cleanup: the existing plugin hook surface is the narrowest maintained way to run code before and after tool calls. Adding automatic skill execution would create a new skill runtime contract and needs product/security design rather than a repair job.
