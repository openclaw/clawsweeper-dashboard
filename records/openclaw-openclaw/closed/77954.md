## Summary

Close as ClawHub/plugin work: current main already exposes plugin hooks around tool calls, compaction, and tool-result persistence, and plugins can ship skills, so these optional safety/optimization behaviors do not need a new core skill-hook schema.

## What This Changes

Request automatic skill-defined pre/post tool hooks so skills such as bash-security can inspect or block `exec` and clean/compress results without manual invocation.

## Best Possible Solution

Implement these safety and optimization checks as a ClawHub code or bundle plugin that registers the existing hook APIs and ships any supporting skills alongside the plugin.

## Reproduction Assessment

Not applicable: this is a feature request, not a report of broken existing behavior. Source and docs inspection confirms the current extension path is plugin hooks plus plugin-shipped skills, not skill-authored hook YAML.

## Solution Assessment

Yes: the existing plugin hook surface is the narrowest maintainable path for automatic tool review. Adding declarative skill-level hooks would duplicate a security-sensitive runtime extension surface and needs product/API design if maintainers still want it.
