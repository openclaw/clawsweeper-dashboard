## Summary

Current main already applies Anthropic prompt caching for direct Anthropic-family requests and documents the default `cacheRetention: "short"` behavior for API-key auth, so there is no missing Haiku-specific core implementation to track here.

## Evidence

- **Released docs say Anthropic API-key auth defaults to short prompt caching:** The latest released docs explicitly state that Anthropic API-key auth seeds `cacheRetention: "short"` when unset and maps `short` to Anthropic's default 5-minute ephemeral cache.
  - file: [docs/reference/prompt-caching.md:100](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/docs/reference/prompt-caching.md#L100)

- **Released Anthropic config defaults seed short retention automatically:** The released Anthropic provider config-defaults code backfills `params.cacheRetention = "short"` for Anthropic API-key model entries and the resolved primary model when unset.
  - file: [extensions/anthropic/config-defaults.ts:232](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/extensions/anthropic/config-defaults.ts#L232)
