## Summary

Keep open: current main still lacks the Ollama lightweight provider-policy thinking hook, so startup-lazy Ollama can still fall back to base thinking levels that omit `max`; #77617 is the active open implementation candidate.

## What This Changes

Allow `/think max` for reasoning-capable `ollama/deepseek-v4-pro:cloud` before the full Ollama plugin runtime activates.

## Best Possible Solution

Keep Ollama startup-lazy and expose the existing reasoning-capable `off/low/medium/high/max` profile through the lightweight provider-policy artifact with focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible: current main has Ollama startup-lazy, no lightweight Ollama `resolveThinkingProfile`, and a fallback base profile that excludes `max`. I did not run a live Ollama/Fedora session.

## Solution Assessment

Yes, the narrow fix is to add the existing Ollama thinking profile to the startup-safe provider-policy artifact. Forcing `activation.onStartup` would mask the regression and weaken the lazy plugin boundary.
