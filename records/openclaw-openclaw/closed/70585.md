## Summary

Current main already implements the central PR #70585 fix: queued subagent announces resolve an external target first and send `deliver: false` with no route fields when no deliverable `channel` + `to` target exists. Regression tests cover no-route, channel-only, internal-origin, and valid external-route queued cases. The fix is on main via `76a0abc768c71ccd48d57288f46c1ad389ccf13e`; it is not in the latest `v2026.4.24` release tag.

## Best Possible Solution

Close PR #70585 as implemented on current main. Keep the resolver-based queued delivery behavior and its regression tests; do not merge the duplicate PR branch unless maintainers specifically want a separate follow-up to add an explicit `bestEffortDeliver` marker for queued no-target announces.

If maintainers want that exact flag contract, it should be tracked as a separate refinement.
- The fix is not in `v2026.4.24`; users on the latest released tag still have the older behavior until the next release.
