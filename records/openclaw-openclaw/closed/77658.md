## Summary

Keep open; this protected maintainer/member PR still has source-backed P1/P2 correctness regressions in the Telegram preview reuse path, so it needs a narrow branch repair before merge.

## What This Changes

The PR changes Telegram text-final delivery to split oversized finals through the active preview, adds Telegram live QA/mock coverage and docs/changelog, and adds diagnostics-otel no-op event cases.

## Best Possible Solution

Keep the preview-reuse direction, but gate it behind existing fresh-final conditions and make remainder delivery derive from the actually finalized visible prefix with regression coverage.

## Reproduction Assessment

Yes. Source inspection shows current main falls back to standard send for oversized text finals after a preview, and the PR-introduced regressions are source-reproducible through the regressive-skipped path and the missing fresh-final guard.

## Solution Assessment

No. Reusing the preview is the right narrow fix, but this implementation needs to preserve the existing stale/overwritten-preview fallback and avoid sending long-final remainders after a skipped first-chunk edit.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Handle regressive skips before sending long-final remainders:** `extensions/telegram/src/lane-delivery-text-deliverer.ts:389-402`
  - body: When the streamed draft has already advanced past `firstChunk`, `tryUpdatePreviewForLane` can return `regressive-skipped`; this branch then sends `remainingChunks.join("")` as if the preview was edited to exactly `firstChunk`. Because the draft stream stops once rendered text exceeds Telegram's max, the visible preview can still be shorter than `lane.lastPartialText`, so a normal long streamed final can lose the middle of the answer while marker-only QA passes. Handle `regressive-skipped` by editing, falling back, or computing the remainder from the actual visible chunk.
  - confidence: 0.9
- **[P2] Keep fresh-final fallback for stale or overwritten previews:** `extensions/telegram/src/lane-delivery-text-deliverer.ts:675-681`
  - body: This oversized-final branch runs before the normal fallback and does not check `shouldUseFreshFinalForLane(lane)`. That regresses the existing Telegram behavior for previews older than about a minute or previews followed by visible non-preview output: those should be replaced by a fresh final and cleanup, not edited into the first long-final chunk. Bail out to the standard send/cleanup path when the fresh-final guard is true.
  - confidence: 0.86
