## Summary

Current `main` no longer uses the generic Feishu ack-reaction removal path described in the report. Feishu now has a dedicated `typingIndicator` lifecycle that starts on reply start and is cleaned up only when the reply dispatcher goes idle after delivery settles, so the reported early reaction removal flow does not match current code.

## Evidence
