## Label Justifications

- `P1`: The original report describes a gateway crash loop when Bonjour starts on affected network interfaces.
- `impact:crash-loop`: The affected behavior was an unhandled rejection that exited the gateway and caused systemd restart loops.

## Summary

Close as implemented on main. Current OpenClaw pins @homebridge/ciao 1.3.9, and that upstream version changed NetworkManager from asserting on no-address interfaces to skipping them, so the reported Bonjour gateway crash path no longer exists even though OpenClaw did not add a local classifier.

## What This Changes

The issue requests preventing Bonjour/CIAO no-valid-address interface assertions from escaping as unhandled rejections and crash-looping the gateway.

## Best Possible Solution

Keep the ciao 1.3.9 or newer dependency behavior; add an OpenClaw-side classifier only if a future ciao release reintroduces this assertion.

## Reproduction Assessment

No current-main failure path remains: OpenClaw now pins ciao 1.3.9, whose NetworkManager skips no-address interfaces instead of throwing the reported assertion. The old 1.3.8 tarball confirms the original assertion path.

## Solution Assessment

Yes. Removing the throw at the dependency source is narrower than carrying an OpenClaw classifier for an obsolete ciao assertion, and the shipped pin proves the runtime uses that behavior.
