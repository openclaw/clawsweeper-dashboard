## Summary

Current `main` already mitigates Telegram command-menu overflow for large skill sets by capping menu entries, trimming descriptions to fit Telegram's payload budget, and retrying with progressively fewer commands on `BOT_COMMANDS_TOO_MUCH`. The behavior is covered by regression tests and documented in current Telegram troubleshooting docs. The changelog shows the sub-100 payload-budget fix shipped in `v2026.4.5`.

## Evidence
