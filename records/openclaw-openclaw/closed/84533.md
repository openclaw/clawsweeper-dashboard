## Summary

Keep open: the PR fixes a real source-level installer bug, but the current diff introduces a macOS compatibility regression for missing-Git installs and still lacks real behavior proof beyond stubbed installer tests.

## What This Changes

The PR moves macOS Homebrew installation behind the Node version check in `scripts/install.sh` and adds installer regression tests for supported and missing/outdated Node flows.

## Best Possible Solution

Move Homebrew bootstrapping to the macOS dependency paths that actually need it, at least `install_node()` and `install_git()`, then update installer docs and require redacted real macOS proof for the supported-Node path.

## Reproduction Assessment

Yes at source level: current `main()` invokes Homebrew setup before checking Node, and `install_homebrew()` can exit before `check_node()` accepts a supported Node. I did not run a live non-admin macOS installer session in this read-only review.

## Solution Assessment

No as implemented: moving Homebrew behind the failed Node check is the right direction for Node, but Homebrew also backs macOS Git installation. The maintainable fix is to make Homebrew conditional inside each Homebrew-backed dependency path and cover both Node and Git cases.
