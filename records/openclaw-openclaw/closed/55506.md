## Summary

Current main already addresses the reported stale Telegram DM workspace/bootstrap context path. Bootstrap context is reloaded from workspace truth per run, deleted workspace files become missing entries rather than stale injected content, run systemPromptReports are built from that freshly resolved context, and regression tests cover refreshed session snapshots after workspace edits. The same refresh code is present in the verified v2026.4.26 tag.

## Best Possible Solution

Keep the current main implementation. If someone still sees this on v2026.4.26 or newer, the next useful report should include a fresh /context output or trajectory bundle from the affected Telegram session so maintainers can inspect the actual current injectedWorkspaceFiles and transcript state.
