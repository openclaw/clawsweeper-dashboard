## Summary

Current `main` already contains the model-switch cleanup needed for this report: `session_status(model=...)` persists the override, clears stale runtime model/provider fields so status surfaces stop preferring the old model, and the TUI refresh path re-reads session info after the run completes. This appears implemented on `main` after the reported 2026.3.8 release.
