## Summary

Current main already fixes the reported Web UI context badge behavior, and the same implementation is present in the shipped v2026.4.29 release: message metadata now bases ctx% on the largest single assistant call prompt/cache size instead of accumulating cache reads across the whole session.

## What This Changes

The issue reports that Web UI message metadata can show a false red 100% ctx badge in long sessions by accumulating per-call cacheRead across assistant messages.

## Best Possible Solution

Keep the shipped grouped-message implementation and ask affected users on 2026.4.24 to update to v2026.4.29 or newer.

## Reproduction Assessment

Yes. The reported old behavior can be checked with a grouped assistant message containing multiple usage records; current main does not reproduce it because the covered path uses the largest single prompt/cache usage for ctx%.

## Solution Assessment

Yes. The current fix is narrow and maintainable: cacheRead remains visible as usage metadata, while the ctx% badge no longer uses an accumulated cache-read denominator.
