The branch adds a `mcporterEnv` that removes `XDG_CONFIG_HOME` and `XDG_CACHE_HOME` from QMD manager mcporter spawns and updates the adjacent unit test to assert those variables are absent.

## Best Possible Solution

Separate mcporter config discovery from QMD server execution: keep mcporter out of OpenClaw's QMD-scoped XDG tree while preserving user XDG discovery and ensuring local/generated `qmd mcp` execution receives agent-scoped QMD config/cache env, with redacted real memory-search proof before merge.
