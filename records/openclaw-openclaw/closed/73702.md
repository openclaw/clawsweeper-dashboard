## Summary

Current main implements the requested same-turn inter-session prompt hardening. `sessions_send` and A2A reply prompts are now wrapped with a model-visible `[Inter-session message ... isUser=false]` envelope before dispatch, and provenance is preserved in metadata and docs. The fix is on main only, not in the latest `v2026.4.26` release.

## Best Possible Solution

Close this issue as implemented on current main. Keep the current same-turn inter-session envelope and transcript provenance behavior, ship it in the next release, and track any stronger cryptographic sender-envelope or capability-intersection design as separate hardening work.
