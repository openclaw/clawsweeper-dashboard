- **[P2] Keep @default from inheriting stale cron sessions:** `src/cron/isolated-agent/model-selection.ts:119`
  - body: `normalizeStoredModelOverride("@default")` returns `undefined`, so this branch later enters `if (!modelOverride && !hooksGmailModelApplied)` and can apply `sessionEntry.modelOverride`. A cron explicitly edited to `--model @default` can therefore keep firing on the old stored session pin instead of the live default; keep a distinct `payloadRequestedDefault` state and skip stored session fallback for that case.
  - confidence: 0.91
- **[P2] Exclude @default from user override metadata:** `src/agents/subagent-spawn-plan.ts:74-75`
  - body: This predicate is still true for the literal sentinel, so a subagent spawn with `modelOverride: "@default"` resolves through the default path but persists `modelOverrideSource: "user"`. That makes later session handling treat the default-derived model as an exact user pin; exclude `isModelDefaultSentinel(params.modelOverride)` here.
  - confidence: 0.94
- **[P2] Align object-form coverage with the public type:** `src/agents/openclaw-tools.subagents.sessions-spawn.model.test.ts:239`
  - body: This new test passes an object to `resolveSubagentModelAndThinkingPlan`, but that function still declares `modelOverride?: string`. `check:test-types` should reject the call; either widen this boundary intentionally or move object-form coverage to `resolveSubagentSpawnModelSelection` / `normalizeStoredModelOverride`, where `unknown` is accepted.
  - confidence: 0.9
- **[P3] Document the new @default model value:** `src/agents/model-default-sentinel.ts:9`
  - body: The PR makes `@default` a public value operators are expected to store via cron/subagent model override fields, but the current docs still describe cron `--model` as an exact per-job primary. Add the user-facing contract, precedence, and migration example so operators know when it follows `agents.defaults.model.primary` and when explicit pins remain exact.
