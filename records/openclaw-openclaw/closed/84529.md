Full review comments:

- **[P2] Parse config backups as JSON5:** `src/secrets/audit.ts:271-274`
  - body: `openclaw.json` is documented as JSON5, and these backup files are snapshots of that authored config. Reading them with `readJsonObjectIfExists` means backups with comments, unquoted keys, or trailing commas are silently skipped, so `secrets audit` can still miss plaintext values in valid OpenClaw config snapshots. Please use the normal config JSON5 parser and cover a JSON5 backup in tests if this branch remains active.
  - confidence: 0.9
