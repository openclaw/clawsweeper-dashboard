## Summary

Current main already supports persistent shell workflows by keeping a backgrounded `exec` session alive and continuing it through the `process` tool, which preserves shell state within that session.

## Evidence

## Risks / Open Questions

- The current UX is explicit (`exec` + `process` on one live shell session), not implicit sticky state across unrelated `exec` calls. That is a discoverability gap, but the requested capability is already present.
