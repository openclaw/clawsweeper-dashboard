## Summary

The issue is not an actionable OpenClaw change request. It asks for cloud security-group / host-firewall changes for a separate Node.js app on TCP 3001 (`CardioNews Station` on “MaxClaw”), while current OpenClaw main documents only its own single gateway port and leaves public exposure/firewall policy to the deployment operator.
