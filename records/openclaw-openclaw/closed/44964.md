## Summary

Current main already fixes the reported 0-byte sandbox write behavior, and the fix is present in the latest tagged release v2026.4.22.

## Evidence

- **Pinned mutation helper keeps stdin for payload bytes:** The sandbox write plan now executes the Python helper via an inline script literal (`-c "$python_script"`) instead of consuming stdin for the helper source, so stdin remains available for file payload bytes during `write` operations.
  - file: [src/agents/sandbox/fs-bridge-mutation-helper.ts:293](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/src/agents/sandbox/fs-bridge-mutation-helper.ts#L293)

  - sha: [4630ce3d9e2f](https://github.com/openclaw/openclaw/commit/4630ce3d9e2f089062ccd386b0ae241325961d1a)
