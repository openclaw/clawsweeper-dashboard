## Summary

This is a valid core feature request, but the remaining work is now tracked by the canonical open implementation PR #69988. Current main still does not implement per-agent compaction overrides: the per-agent type and strict schema lack `agents.list[].compaction`, resolver/runtime paths still read defaults-only compaction settings, and docs expose only `agents.defaults.compaction`. Earlier implementation attempts #14598 and #19329, including the live-gateway testing noted by curtismercier, are closed unmerged; #69988 is the active focused path.

## Best Possible Solution

Close this issue as superseded by #69988 and keep review focused on that PR. The best implementation remains a core config/runtime change: add optional `agents.list[].compaction` to the public type and strict schema, reuse the defaults compaction schema, deep-merge per-agent values over `agents.defaults.compaction` including nested `memoryFlush`, thread the effective config through Pi settings, safeguard runtime, compaction model/provider/timeout paths, memory-flush/preflight behavior, generated config help/docs, and focused schema/resolver/runtime tests.
