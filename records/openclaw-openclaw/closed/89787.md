## Summary

Keep open. Current `main` still has the reported bypass: a counted reasoning payload can make `payloadCount` nonzero, then `resolveIncompleteTurnPayloadText` returns before the non-visible turn detectors run; the related open PR targets the truncation path, not this clean `stop` plus counted-thinking case.

## What This Changes

The issue reports that embedded agent turns can silently stall when a local model returns `stopReason="stop"` after emitting only a thinking/reasoning block and no structured tool call.

## Best Possible Solution

Teach the non-visible turn recovery path to treat reasoning-only payloads as non-final output for this guard, or otherwise evaluate the existing reasoning/empty/incomplete detectors before the payload-count short-circuit, with a focused regression test for clean `stop` plus counted thinking and no tool call.

## Reproduction Assessment

No live high-confidence reproduction was established in this read-only review. Source inspection gives a medium-confidence reproduction path: clean `stop`, unsigned thinking extracted as a reasoning payload, no visible assistant text, no tool call, and `payloadCount > 0` bypassing the later recovery checks.

## Solution Assessment

No current implementation is sufficient for the reported edge case. The reporter’s suggested guard restructuring is plausible, but the repair should preserve the existing side-effect, messaging-delivery, and accepted-spawn gates.
