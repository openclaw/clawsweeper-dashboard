## Summary

Current `main` does not show a code path that strips literal `https` from Discord message content, the empty-message path already hydrates from REST, and the linked bug report `#67151` was closed by the reporter as self-resolved. This PR adds a plausible `embed.url` fallback, but the issue discussion does not establish that payload shape as the active bug on current OpenClaw.
