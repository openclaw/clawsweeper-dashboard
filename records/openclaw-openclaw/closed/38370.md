## Summary

Current `main` already repairs tool/result pairing after `limitHistoryTurns` in both embedded-runner paths, and the repair helper drops orphaned `toolResult` messages. The reported failure mode is therefore already implemented/fixed on `main`, and the same code is present in release `v2026.4.22`.
