## Summary

Close as implemented on current main and shipped in v2026.4.27. QQ Bot C2C messages preserve the raw `/new` text into the shared command path, and the shared reset command handler now returns a local acknowledgement for bare `/new` before the reply pipeline can invoke the embedded agent or provider.

## What This Changes

A bare `/new` sent in a QQ Bot C2C private chat should reset the session and acknowledge locally instead of making a Claude-compatible request with an empty messages payload.

## Best Possible Solution

Keep the shipped shared reset-command behavior: bare `/new` and `/reset` should acknowledge locally without model execution, while reset commands with trailing user text continue through the agent path. Leave the broader embedded-runner empty-prompt hardening to the existing #73926/#73929 track.
