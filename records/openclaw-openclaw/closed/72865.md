## Summary

Current main and the shipped v2026.4.27 tag already handle the reported Bedrock/Anthropic prefill shape: outgoing Anthropic-class replay strips trailing non-tool assistant turns before provider calls, and transcript-only OpenClaw assistant records are filtered out of replay. That directly addresses the Feishu `/new` first-message path described here, while the linked Lark PR usefully points to the same reset/greeting area rather than a remaining OpenClaw core gap.

## What This Changes

The issue requests a fix for Feishu `/new` sessions on Bedrock Claude Opus 4.6 failing on the first user message because the outgoing conversation can end with an assistant message prefill.

## Best Possible Solution

Keep the v2026.4.27 replay hygiene behavior: transcript-only channel messages should remain visible to users but excluded from model replay, and Anthropic-class provider calls should continue to end with a user turn unless a real assistant tool-use turn must be preserved.

## Reproduction Assessment

Yes. The issue gives a concrete versioned path, and current focused tests reproduce the provider-facing shape by passing replay messages ending in an assistant turn through the stream wrapper and asserting the provider sees only the user-ended transcript.

## Solution Assessment

Yes. The current fix is at the shared replay/provider boundary, so Feishu and other channel surfaces do not need owner-specific hacks for a Bedrock/Anthropic wire-contract problem.
