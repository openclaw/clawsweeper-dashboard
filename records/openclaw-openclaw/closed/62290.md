## Summary

Current `main` already carries the Control UI image-upload path end to end: the browser reads selected files and sends them in `chat.send`, the gateway preserves them for text-only models by offloading to `media://inbound/...` instead of dropping them, and the embedded runner resolves those refs back into prompt images. That matches the issue discussion pointing at model image-support handling rather than a broken Control UI picker.
