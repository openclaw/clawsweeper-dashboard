## Summary

Current main no longer rejects Gemini multimodal memory just because the embedding provider registry is not preloaded, and memory runtime loading no longer depends on the old hashed root `memory-tool.runtime-*` chunk path. The packaged path is now the bundled `dist/extensions/memory-core/runtime-api.js` sidecar, with release guards that require bundled plugin runtime artifacts in npm packs.

## Evidence

- **Multimodal validation no longer fails on an unloaded Gemini adapter:** `resolveMemorySearchConfig(...)` now only throws the multimodal-support error when a provider adapter is already registered and explicitly lacks multimodal support. If Gemini is not yet registered on a status/startup path, config resolution does not fail just for that reason.
