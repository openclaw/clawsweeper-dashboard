## Summary

Current `main` now has the plugin-compatibility strategy this issue asked for: documented deprecation policy, a central compatibility registry covering legacy root SDK imports, runtime deprecation warnings for broad SDK compatibility imports, and install/load-time host-version gating so incompatible plugins fail with explicit messages instead of cryptic breakage.

## Risks / Open Questions

- This closes on the basis that the requested compatibility strategy now exists on `main`; it does not mean every previously removed root-barrel helper was retroactively restored.
- Compatibility signaling is split between deprecation docs/registry, `openclaw.compat` metadata, and enforced `openclaw.install.minHostVersion`, so some plugin authors may still need docs follow-up for clarity.
