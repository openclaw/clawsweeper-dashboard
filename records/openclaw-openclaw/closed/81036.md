## Summary

Current main already implements the reported reset semantics for Feishu/group chat and Gateway session resets: reset commands rotate the persisted session id, update the session store, clear runtime reset state, and archive the previous transcript so old context is not reloaded.

## What This Changes

The issue asks for `/new` and `/reset` in Feishu/group chats to clear persistent session state so old context is not restored from `sessions.json` or transcript files.

## Best Possible Solution

Keep the current-main behavior: preserve the stable chat `sessionKey`, rotate its `sessionId`, archive the old transcript, and ship that reset path in the next release.

## Reproduction Assessment

No high-confidence current-main failing reproduction is established. This read-only sweep did not run a live Feishu group, but source and regression tests show current main rotates persisted session state instead of reusing the old transcript.

## Solution Assessment

Yes. Rotating the `sessionId` under the same `sessionKey` matches the documented session contract and is narrower than deleting the entire metadata row, while still preventing old transcript context from being reloaded.
