## Summary

Current main already handles long-running execs with background sessions plus real completion signaling, and it includes a blocking `process poll timeout` path for agents that need to wait before replying.

## Evidence

- **Exec no longer double-resolves after backgrounding:** The exec tool resolves `status: "running"` when `yieldMs/background` kicks in, then the later `run.promise` completion path returns early if `yielded || run.session.backgrounded`, which prevents the same command from immediately producing a second completion result in the same turn.
  - file: [src/agents/bash-tools.exec.ts:1737](https://github.com/openclaw/openclaw/blob/bfa6708c03f24c7caa5d0a96a298c292be0d85c1/src/agents/bash-tools.exec.ts#L1737)
  - command: `nl -ba src/agents/bash-tools.exec.ts | sed -n '1737,1809p'`
  - sha: [bfa6708c03f2](https://github.com/openclaw/openclaw/commit/bfa6708c03f24c7caa5d0a96a298c292be0d85c1)
- **Completion wake happens only on actual process exit:** Backgrounded exec sessions only enqueue the `Exec completed/failed` system event and request a heartbeat inside `maybeNotifyOnExit`, after the process has actually exited.
  - file: [src/agents/bash-tools.exec-runtime.ts:279](https://github.com/openclaw/openclaw/blob/bfa6708c03f24c7caa5d0a96a298c292be0d85c1/src/agents/bash-tools.exec-runtime.ts#L279)
