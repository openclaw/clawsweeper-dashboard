## Summary

Current main already handles NVIDIA refs with slash-containing model IDs correctly: `nvidia/moonshotai/kimi-k2.5` is parsed into provider `nvidia` plus bare model ID `moonshotai/kimi-k2.5`, then the OpenAI-compatible transport sends the bare `model.id` in the request payload. The released `v2026.4.22` tag contains the same behavior, and the reporter later confirmed it was fixed.

## Evidence
