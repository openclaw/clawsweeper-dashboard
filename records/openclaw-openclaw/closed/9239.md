## Summary

Current `main` already keeps Telegram subagent runs announce-only: subagent child runs are started with `deliver=false` unless they are thread-bound persistent sessions with a bound delivery origin, and the docs say only Discord currently supports thread-bound subagent sessions. That means Telegram subagents no longer stream per-tool activity into the requester chat and instead announce on completion.

## Evidence

- **Subagent spawn disables direct child delivery by default:** `src/agents/subagent-spawn.ts:958-980` computes `deliverInitialChildRunDirectly` only for `thread: true` + `mode: "session"` + a bound thread delivery origin, then passes `deliver: deliverInitialChildRunDirectly` into the child `agent` call.
