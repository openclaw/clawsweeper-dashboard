## Summary

Close as implemented. Current main and the v2026.6.6 release preserve the embedded session tool allowlist across session tool-registry refreshes, so the reported Telegram/direct-Anthropic path should no longer leave `context.tools` empty after the runner computed tools.

## What This Changes

The issue reports Telegram/direct-Anthropic embedded runs sending an empty tools array even though the runner computed coding tools, preventing real `tool_use` blocks and exec approvals.

## Best Possible Solution

Keep the shipped session-runtime allowlist refresh behavior in v2026.6.6 and newer; reopen only with fresh Telegram/direct-Anthropic evidence against that release or current main.

## Reproduction Assessment

Yes for the original bug: the issue has concrete Telegram/direct-Anthropic logs and a later CLI-vs-Telegram comparison, and the old source path matched the reported active-tool wipe. I did not run a live Telegram setup in this read-only review because current main and v2026.6.6 now show the shipped fix path in source.

## Solution Assessment

Yes. Preserving the allowed registered tools inside `AgentSession.refreshToolRegistry` is the maintainable fix point; an Anthropic transport fallback would hide legitimate no-tool runs and would not repair the session state that produces real tool calls.
