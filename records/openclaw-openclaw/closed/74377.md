## Summary

Current main and the v2026.6.6 release already preserve the embedded-session tool allowlist across session tool-registry refreshes, which addresses the reported Telegram/direct-Anthropic path where tools were computed but disappeared before provider dispatch.

## What This Changes

The issue reports that Telegram-triggered Anthropic embedded runs lose the computed tool list before provider dispatch, causing tool calls and approval IDs to be hallucinated as plain text.

## Best Possible Solution

Keep the shipped session-runtime allowlist refresh behavior in v2026.6.6 and newer; reopen only with fresh Telegram/direct-Anthropic evidence against that release or current main.

## Reproduction Assessment

Yes for the original bug: the issue includes concrete Telegram/direct-Anthropic logs plus a later CLI-vs-Telegram comparison, and the old source path matched an active-tool refresh wipe. I did not run a live Telegram setup in this read-only review because current main and v2026.6.6 now show the fixed source path.

## Solution Assessment

Yes. Preserving the allowed registered tools inside the session refresh path is the maintainable fix point; an Anthropic transport fallback would hide legitimate no-tool runs and would not repair the session state that produces real tool calls.
