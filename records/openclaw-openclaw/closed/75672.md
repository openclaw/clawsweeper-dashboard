- **[P1] Use the preview maintenance parameter:** `src/config/sessions/cleanup-service.ts:535-536`
  - body: `previewStoreCleanup` receives maintenance as `params.maintenance`, but the new planner call reads bare `maintenance.pruneAfterMs` and `maintenance.resetArchiveRetentionMs`. This branch will fail type/build validation before dry-run cleanup can run.
  - confidence: 0.98

  - body: When cleanup runs without `--enforce` and maintenance mode is `warn`, current cleanup avoids destructive file sweeps. This new call always passes `dryRun: false`, so archive files can be deleted despite warn mode.
  - confidence: 0.94
- **[P2] Preserve the checkpoint age cutoff:** `src/config/sessions/cleanup-service.ts:347-350`
  - body: Current unreferenced-artifact cleanup only removes files older than the retention cutoff. This branch removes orphan checkpoints as soon as their session id is absent from the index, which can delete fresh or in-flight checkpoint files.
  - confidence: 0.91
- **[P2] Count archives before maintenance removes them:** `src/config/sessions/cleanup-service.ts:690-696`
  - body: Apply mode runs `updateSessionStore` before this planner, and current store maintenance already prunes `.deleted` and `.reset` archives. The later `archivedFiles` plan can therefore report zero even when this command removed archives.
  - confidence: 0.88
- **[P3] Document public retention keys and defaults:** `docs/cli/sessions.md:125-126`
  - body: `pruneAfterMs` is an internal resolved field, so the docs should name `session.maintenance.pruneAfter`. Also, unset `resetArchiveRetention` defaults to the prune window; only `false` disables reset archive cleanup.
  - confidence: 0.9
