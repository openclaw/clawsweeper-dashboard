## Summary

Current `main` already contains transcript-repair logic that removes an orphaned trailing user message before retrying, merges it into the next prompt when needed, and explicitly avoids duplicating queued user text. The coverage includes background triggers such as `heartbeat` and `overflow`, which matches the reported failure mode.

## Evidence
