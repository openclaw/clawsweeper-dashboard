## Summary

Current `main` already exposes a plugin-consumable real-time telemetry surface for model dispatches via `openclaw/plugin-sdk` diagnostics events. Plugins can subscribe with `onDiagnosticEvent`, receive `model.usage` events with usage/session/provider/model/duration data, and also observe `model.call.completed` / `model.call.error` lifecycle events. The bundled `diagnostics-otel` plugin consumes that exact surface.
