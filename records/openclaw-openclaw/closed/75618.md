## Summary

Current main and the latest shipped tag preserve Feishu top-level allowlist fields when resolving a named runtime account. The account resolver performs a generic top-level-plus-account merge, the shared Feishu schema exposes the reported fields at both levels, and the message handler reads the merged `account.config` for group gating.

## What This Changes

The issue asks for Feishu runtime account resolution to preserve top-level `allowFrom`, `groupAllowFrom`, and `groupSenderAllowFrom` when `channels.feishu.accounts.main` is configured.

## Best Possible Solution

Keep the existing generic account-resolution path: Feishu should inherit shared top-level access-control config through `resolveMergedAccountConfig`, with account-level values overriding only when explicitly set.

## Reproduction Assessment

Yes. The high-confidence verification path is source-level: resolve `channels.feishu.defaultAccount = "main"` with top-level allowlist fields and credentials only under `accounts.main`; current main’s merge path preserves the reported fields before group gating reads them.

## Solution Assessment

Yes. The current generic base-plus-account merge is the narrowest maintainable solution because it preserves all shared Feishu fields without hard-coding a Feishu-specific allowlist of property names.
