## What This Changes

Request to add persistent vector-searchable Dynamic Parts that agents can create, update, retrieve, merge, split, evaluate over time, and archive.

## Best Possible Solution

Build this as an external ClawHub code plugin, likely combining a memory plugin or memory corpus supplement with a context-engine plugin, plugin-owned JSON/vector storage, tools or commands for JSON mutations, and scheduler/next-turn hooks for due evaluations. If implementation discovers a concrete missing SDK seam, open a narrow API issue for that seam rather than adding the whole Dynamic Parts framework to core.
