## Summary

Keep open: this PR addresses a real CLI diagnostic gap, but the current diff is not merge-ready because the disabled-by-default hint is not origin-aware and the branch also adds a deadcode allowlist entry for a file removed on current main.

## What This Changes

This PR carries plugin `enabledByDefault` into command-alias lookup, adds a missing-command diagnostic for disabled bundled plugin aliases, updates regression tests and changelog, and changes the deadcode unused-file allowlist.

## Best Possible Solution

Keep the CLI diagnostic, but make alias ownership carry enough metadata to distinguish bundled default-disabled plugins from external enabled-by-default plugins, then remove the stale allowlist change during rebase.

## Reproduction Assessment

Yes. Source inspection on current main shows `voice-call` owns the `voicecall` alias without `enabledByDefault`, docs define that as bundled default-disabled, and the current missing-command policy has no default-disabled alias hint; I did not run CLI/tests because this review is read-only.

## Solution Assessment

No. The intended fix is narrow and appropriate, but the implementation should gate the new hint on bundled origin or effective enable state and should drop the stale deadcode allowlist entry after rebasing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Gate the disabled alias hint on bundled origin:** `src/cli/run-main-policy.ts:145-147`
  - body: This condition treats every alias owner without `enabledByDefault: true` as a bundled plugin disabled by default. External, workspace, and global plugins also normally omit `enabledByDefault` and are enabled by default unless config disables them, so a missing external alias can get the wrong `openclaw plugins enable <id>` repair. Carry plugin origin or effective enable state through alias lookup before emitting the bundled-plugin message.
  - confidence: 0.88

  - body: Current main removed `src/infra/command-analysis/index.ts`, and `check-deadcode-unused-files` fails when required allowlist entries are stale. After rebasing, this added allowlist line would fail the deadcode check unless the barrel is intentionally restored.
  - confidence: 0.9
