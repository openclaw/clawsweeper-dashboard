## Summary

Close as superseded by the broader open Discord access-list design in #69748. Current main still lacks per-sender Discord mention policy, but #69748 directly tracks the remaining product work with bypass/mention/deny lists, role support, per-channel overrides, precedence, and migration; #23896 is an older narrower version of the same gap.

## Best Possible Solution

Close this issue as superseded by #69748, keep #69748 as the canonical Discord access/mention-policy design discussion, and fold the narrower compatibility details from #23896 into that implementation plan. The code fix should live in the bundled Discord plugin/config contract, add sender-aware mention resolution, validate the new schema, document precedence, and test mixed senders in the same channel.
