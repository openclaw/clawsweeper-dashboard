## Summary

Keep open: current main still injects active-memory summaries through the cache-hostile prepend path, and the related openviking report is useful context but does not track this active-memory-specific bug.

## What This Changes

The issue asks active-memory to avoid dynamic prepended prompt context, or make the injection position configurable, so llama.cpp prompt cache reuse remains high on long sessions.

## Best Possible Solution

Move active-memory's dynamic recall summary to the existing appendContext path, update active-memory tests/docs/changelog, and add real llama.cpp cache proof if feasible without adding a new core hook surface.

## Reproduction Assessment

No live high-confidence reproduction was run in this read-only review. Source inspection does reproduce the relevant current-main path: active-memory returns prependContext, and both main prompt runners place prependContext before the prompt.

## Solution Assessment

Likely yes: using the existing appendContext hook for active-memory is the narrowest maintainable fix because core already supports it. A new injection-position config should be avoided unless maintainer review finds a compatibility need for preserving prepend behavior.
