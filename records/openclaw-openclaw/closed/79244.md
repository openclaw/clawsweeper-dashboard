## Summary

Keep open: current main still contains the source-reproducible managed-image authorization weakness, and https://github.com/openclaw/openclaw/pull/79204 is the active unmerged fix path.

## What This Changes

Fix managed outgoing image byte serving so ownership cannot be proven by client-declared requester-session headers.

## Best Possible Solution

Review and land or replace the linked gateway fix so managed image byte serving derives owner/admin authority from authenticated request context while preserving transcript membership and scoped operator checks.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: a request with `chat.history` scope can send the target `x-openclaw-requester-session-key`, and the current helper treats equality with the target session as ownership. I did not run live gateway traffic because this was a read-only review.

## Solution Assessment

Yes, the best direction is narrow: keep the existing transcript and operator-scope checks, but replace caller-declared requester ownership with authenticated owner/admin request state.
