## What This Changes

The PR adds a fallback that auto-selects the sole active subagent for targetless `/steer` input and adds two unit tests for that path.

## Best Possible Solution

Keep the shipped current-session `/steer <message>` implementation and require `/subagents steer <id|#> <message>` for child runs.

## Reproduction Assessment

No for this PR branch: the merged current-session command is the narrower maintainable solution because it aligns runtime routing, native/text command metadata, docs, changelog, and tests while preserving explicit child-run steering.
