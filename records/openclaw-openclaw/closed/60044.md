## Summary

Current `main` already supersedes PR #60044. `writeConfigFile` now refreshes the committed runtime snapshot before notifying listeners, then emits the refreshed runtime config plus the persisted source config separately. That fixes the reported `__OPENCLAW_REDACTED__` hot-reload rewrite path and also avoids the PR review concern about sending source-shaped `nextCfg` into the gateway reloader.

## Evidence
