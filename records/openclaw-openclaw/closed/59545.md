## Summary

Keep open. Current main still has the macOS General settings path that can turn Tailscale settings hydration into a save-on-change config rewrite, and the existing tests only smoke-render the views rather than proving that initial hydration is read-only.

## What This Changes

The issue asks the macOS General settings page to avoid rewriting `~/.openclaw/openclaw.json` unless the user changes a control.

## Best Possible Solution

Keep the fix in the macOS General/Tailscale settings owner path: distinguish initial config hydration from user edits and prove `gateway`, `auth`, `meta`, and `wizard` fields stay unchanged on view entry.

## Reproduction Assessment

Yes. The issue provides concrete Settings -> General mtime/diff steps, and current source inspection shows the same hydration-to-save route that can produce the write.

## Solution Assessment

Yes. Dirty-tracking or suppressing Tailscale save handlers during initial `loadConfig()` hydration is the narrowest maintainable fix; changing gateway auth, bind defaults, or global config stamping would be broader than the reported UI side effect.
