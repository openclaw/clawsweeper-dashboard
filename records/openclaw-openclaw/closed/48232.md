## Summary

This is a useful idea, but it fits OpenClaw's plugin/ClawHub extension layer rather than OpenClaw core. Current main already exposes the needed seams: per-turn prompt injection hooks, Discord thread/session binding, and separate identity/skill configuration.

## Evidence

- **Vision keeps optional behavior out of core:** The project vision says optional capability should usually ship as plugins, and new skills should go to ClawHub first rather than core.
