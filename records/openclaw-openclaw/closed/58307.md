## Summary

Current `main` already preserves the cron-selected model through isolated runs and live-model-switch retries. The cron path now persists the selected runtime model before execution, and live model selection resolves from persisted runtime model fields when no explicit session override exists, so the issue's described fallback to the agent default no longer matches current code.
