The PR changes ACPX Codex wrapper generation to symlink canonical `~/.codex/auth.json` into the isolated `codex-home` at wrapper startup and adds wrapper-content plus symlink behavior tests.

## Risks / Open Questions

- The PR intentionally exposes host Codex `auth.json` to the managed Codex ACP adapter process through a symlink; that may be the right fix, but the trust boundary needs maintainer/security sign-off.
- The branch hard-codes `os.homedir()/.codex/auth.json` rather than respecting an explicit `CODEX_HOME`, so it may miss users whose working Codex auth lives outside the default home.
- Symlink failures are silently ignored, preserving existing fallback behavior but leaving the downstream auth failure without a clearer OpenClaw diagnostic.
- This branch lacks docs/changelog updates that #73943 already includes, so merging this exact diff would leave user-facing ACP documentation behind the behavior.
- There are now two open PRs for the same auth half of #73910; maintainers should pick one canonical implementation or consolidate them.
