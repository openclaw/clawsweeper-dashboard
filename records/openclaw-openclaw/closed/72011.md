## Summary

Keep PR #72011 open. Current main still lacks the streaming-preview guard this PR is trying to add: the partial reply path builds `cleanedText` directly from parsed stream text and can pass it to `onPartialReply`, while existing sanitizers only cover XML/bracketed downgraded tool scaffolding or block-reply chunks. The PR is a focused follow-up to open bug #70797 and the closed earlier attempt #71954, not obsolete or already implemented.

## Best Possible Solution

Keep the PR open and treat it as an active bug-fix candidate for #70797. The best path is maintainer review plus revisions that harden the streaming sanitizer edge cases, followed by targeted regression coverage for `onPartialReply` and the shared assistant-visible-text sanitizer.
