## Summary

Current main still lacks the bundled TUI live transcript subscription/handler, so this is not implemented. It can close as superseded because the same remaining TUI work is now tracked by canonical issue #38829 and open maintainer PR #73402, whose body explicitly says it covers this report's duplicate Linux/Telegram reproduction.

## Best Possible Solution

Close this issue as superseded by #38829 and route the remaining work through review of #73402. The desired fix is still to add TUI backend/client subscribe and unsubscribe methods, subscribe to the active session transcript after connect and session switches, handle `session.message` without disrupting active local streams, refresh session metadata from `sessions.changed`, and keep focused regression tests for cross-surface delivery and session filtering.
