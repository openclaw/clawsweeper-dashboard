## Summary

Current `main` already implements a Windows scheduled-task restart handoff for gateway self-restarts, and the reported version (`2026.3.23-1`) predates the shipped Windows restart fixes called out in `CHANGELOG.md` under `2026.4.5`.

## Evidence

- **Gateway run loop now uses full-process/supervised restart path:** After gateway shutdown for restart, the run loop calls `restartGatewayProcessWithFreshPid()` and exits cleanly when that returns `supervised`, instead of depending on an in-process restart.
  - file: [src/cli/gateway-cli/run-loop.ts:82](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/cli/gateway-cli/run-loop.ts#L82)
  - command: `nl -ba src/cli/gateway-cli/run-loop.ts | sed -n '82,103p'`
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Windows supervised gateway processes are detected and restarted via schtasks:** `restartGatewayProcessWithFreshPid()` detects Windows `schtasks` supervision and calls `triggerOpenClawRestart()` before returning `supervised`; it avoids unsafe detached respawn on Windows without task markers.
  - file: [src/infra/process-respawn.ts:30](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/infra/process-respawn.ts#L30)
  - command: `nl -ba src/infra/process-respawn.ts | sed -n '20,67p'`
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Windows restart path uses detached helper instead of relying on the original process tree:** `triggerOpenClawRestart()` routes Windows to `relaunchGatewayScheduledTask()`, which writes a temporary `.cmd` helper and launches detached `cmd.exe` to retry `schtasks /Run` and fall back to the installed `gateway.cmd` if needed.
  - file: [src/infra/restart.ts:353](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/infra/restart.ts#L353)
  - command: `nl -ba src/infra/restart.ts | sed -n '353,393p'`
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Detached schtasks relaunch helper implementation:** The helper script logs a `windows-task-handoff`, queries the task, retries `/Run`, optionally falls back to the installed task script, and self-deletes. This is the mechanism that survives the original gateway process exiting during an agent-triggered restart.
  - file: [src/infra/windows-task-restart.ts:24](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/infra/windows-task-restart.ts#L24)
