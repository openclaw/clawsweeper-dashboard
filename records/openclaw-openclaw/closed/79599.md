## What This Changes

The issue requests surfacing model-pricing bootstrap failures in gateway status and health output while keeping overall health true.

## Best Possible Solution

Expose a model-pricing degraded warning from the last bootstrap/refresh failure in health and gateway status without changing liveness semantics.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection gives a clear path: fail the remote pricing catalog fetch during scheduled pricing bootstrap, then query gateway status/health and observe that no model-pricing degradation field can be emitted.

## Solution Assessment

Yes, an additive health/status warning sourced from structured model-pricing state is the narrow maintainable fix. It should not make `/healthz` or gateway health fatal, and it should not add a config option.
