## What This Changes

The issue reports that the bundled Active Memory `before_prompt_build` recall pass adds 7-15s of per-turn overhead on Docker Linux ARM64 and notes that `sqlite-vec` was missing from the released runtime-dependency install.

## Best Possible Solution

Use #69182 as the canonical tracker for the blocking Active Memory latency work, carry over this issue's Docker ARM64/event-loop diagnostics there, and verify the sqlite-vec runtime-deps mirror in the next release acceptance path.
