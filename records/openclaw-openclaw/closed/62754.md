## Summary

Current main already contains the targeted fix for the reported pi-agent-core late-listener crash: exec update callbacks are suppressed after exit, backgrounding, or abort, regression tests cover the late-update paths, and the changelog records the exact crash signature under the 2026.4.10 release notes. The related duplicate #63220 points to the same failure mode and is already closed.

## Best Possible Solution

Close as implemented. Keep the shipped exec late-update suppression and idle-aware cleanup; if a different non-exec pi-agent-core listener path still reproduces on current main or a release at or after 2026.4.10, it should be filed as a fresh, narrower report with logs and the affected tool/runtime path.

## Evidence

- **Runtime guard suppresses late exec updates:** `runExecProcess` tracks `updatesDisabled` and makes `emitUpdate()` return when the session is backgrounded, exited, or disabled. The inline comment names the exact `Agent listener invoked outside active run` crash and explains why try/catch cannot catch pi-agent-core's rejected update promises.

  - file: [src/agents/bash-tools.exec-runtime.ts:838](https://github.com/openclaw/openclaw/blob/0f3a9d812b40cbc8e545abcb80a1f12a93946cf6/src/agents/bash-tools.exec-runtime.ts#L838)
