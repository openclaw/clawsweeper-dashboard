## Summary

Close as superseded. Current main has moved the cron model contract in the opposite direction from this branch: a disallowed explicit cron `payload.model` now fails closed with a validation error, and the source, tests, docs, and changelog all pin that behavior. The branch should not be merged as cleanup because it would reintroduce allowlist bypass behavior, still has provider/model validation feedback, and carries an unrelated security-facing SSRF message change.

## What This Changes

This PR changes isolated cron model selection to relax the allowed-model gate for explicit `payload.model` values found in the catalog and also updates an SSRF resolved-IP block error message.

## Best Possible Solution

Keep the current fail-closed allowlist contract on main. Any future allowlist-bypass proposal should be a fresh maintainer-approved change with exact provider/model catalog validation, focused tests for collision cases, matching docs/changelog updates, and no unrelated SSRF edits.
