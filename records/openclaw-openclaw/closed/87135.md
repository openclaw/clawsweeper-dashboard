## What This Changes

Request to add `agents.defaults.baseline` or `agents.defaults.meta` so deployment-specific tier tags can be stored under agent defaults and read by a CatClaw channel plugin.

## Best Possible Solution

Keep `agents.defaults` strict and have CatClaw define/read deployment tier metadata under its own `plugins.entries.<id>.config`, publishing the plugin through ClawHub if the integration should be shared.

## Solution Assessment
