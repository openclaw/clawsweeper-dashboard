## Summary

## What This Changes

The branch adds a `{ handled: true }` provider usage-auth result, uses it to stop Anthropic standard API keys from falling through to usage-status bearer calls, and updates SDK exports, docs, API baseline, boundary cache inputs, and regression tests.

## Best Possible Solution

Have maintainers decide the public `{ handled: true }` SDK contract; if accepted, land this focused fix after normal checks with the supplied CLI proof, SDK checks, docs checks, and regression tests.

## Reproduction Assessment

Yes by source inspection, not by running current main: current main lets Anthropic's usage hook return no OAuth token and then falls through to generic API-key usage auth. The linked report provides concrete `openclaw status --usage` and `/status` steps.

## Solution Assessment

Mostly yes: the patch creates a narrow explicit handled result and keeps `resolveOAuthToken()` token-only. The unresolved part is whether maintainers want `{ handled: true }` as the public SDK contract instead of an internal Anthropic-only suppression path.
