## Summary

Current main already normalizes OpenAI-style `prompt_tokens`/`completion_tokens` usage in the assistant-message/runtime path, preserves streaming usage for DashScope/Qwen-compatible `openai-completions` endpoints, and the latest release tag `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`) contains that implementation plus regression coverage.
