  - body: The branch removes `shared` from `MemoryDreamingWorkspace`, but the PR's own memory-core isolation code reads `entry.shared` to decide whether to fail closed and filter shared workspaces. That makes the new shared-workspace security checks see `undefined` instead of true and defeats the isolation path.
  - confidence: 0.93
- **[P2] Keep primary workspace resolution options:** `src/memory-host-sdk/dreaming.ts:606`
  - body: Current main callers pass `primaryWorkspaceDir` and `primaryAgentId` so the runtime's active workspace is included even when it is not listed in config. Removing that options parameter breaks the existing memory dreaming contract and can omit the primary workspace when configured subagent workspaces exist.
  - confidence: 0.9
- **[P1] Validate the transport gate instead of hard-coding pass:** `src/memory-host-sdk/clawsweeper.ts:364-373`
  - body: `validateTransportLayer()` returns `passed: true` without checking that `sanitizeForGateway` or any fail-closed transport gate is actually wired. A missing or regressed transport sanitizer would still produce a green sweep, so this cannot enforce the security boundary it claims to validate.
  - confidence: 0.92
- **[P1] Use a non-reversible hash for blocked content:** `src/security/external-content.ts:179-187`
  - body: `createHashFromContent()` is documented as a SHA-256 audit hash, but it returns the byte length plus the first 64 bytes encoded as hex. If a sanitization failure is logged or surfaced, this leaks the beginning of the blocked external content instead of recording a non-reversible digest.
  - confidence: 0.91
