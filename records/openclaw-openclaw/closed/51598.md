## Summary

Keep #51598 open. Current main has `agents.defaults.systemPromptOverride` and `agents.list[].systemPromptOverride`, but those are full replacement/debug controls, not the requested additive `agents.defaults.systemPrompt` plus per-agent append semantics. The related #54504 subagent-instruction-file request is adjacent, not a canonical replacement for this main/cron/subagent-wide agent config gap.

## Best Possible Solution

Keep the issue open and implement an additive agent-level prompt config in the core agent config path, likely with docs and tests covering defaults plus per-agent composition, main/CLI/embedded runs, cron session creation, and subagent spawning. The implementation should preserve `systemPromptOverride` as an explicit full-replacement debug/control knob and define how additive prompt text composes with plugin hook system context and cache-boundary behavior.
