## Summary

Current main addresses the reported Dashboard/CLI state mismatch. The session usage path now persists context utilization from the last/fresh API-call snapshot instead of accumulated run totals, the Control UI context warning renders only fresh `totalTokens / contextTokens`, and stale token snapshots are hidden. Device pairing is also no longer a separate raw-file-only path for the local CLI: `openclaw devices list` calls the Gateway pairing API and falls back to the local pairing store on loopback pairing-required failures. This shipped in v2026.4.26; PR #48364 was closed unmerged and is not the implementation path.

## Best Possible Solution

Close as implemented. Keep the shipped fresh-context usage path and local pairing fallback; if a user still sees a mismatch on current releases, ask for a new report with `openclaw status --json`, `openclaw devices list --json`, the session key, and the fresh Dashboard screenshot so maintainers can investigate a current reproduction rather than the 2026.3.13 behavior.
