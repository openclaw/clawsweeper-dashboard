## Summary

Keep open. The PR targets a real heartbeat runtime-model bleed path, but the proposed preserve logic can write a mismatched provider/model for legacy model-only session entries, and the user-facing fix is missing the required changelog entry.

## What This Changes

The PR adds a `preserveRuntimeModel` option for heartbeat runs so session-store updates keep the pre-existing runtime model/provider/context window instead of persisting the heartbeat run's model.

## Best Possible Solution

Keep the heartbeat model override as a per-run selection, preserve the prior runtime model/context identity after heartbeat turns, handle partial legacy runtime entries without borrowing the heartbeat provider, and add the required Unreleased Fixes changelog entry.

## Reproduction Assessment

Yes. Current main can be reproduced at unit level by seeding a session with a primary runtime model, running `updateSessionStoreAfterAgentRun` with a heartbeat result that reports a different model/context window, and observing the stored runtime fields change.

## Solution Assessment

No, not as currently written. The chosen seam is appropriate, but the preserve logic should treat the previous runtime provider/model as an atomic identity or leave partial legacy fields unresolved instead of mixing a stored model with the heartbeat provider.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve model-only entries without the heartbeat provider:** `src/agents/command/session-store.ts:126-127`
  - body: When a stored session has `model` but no `modelProvider`, this preserve path writes `provider: entry.modelProvider ?? providerUsed` together with `model: entry.model ?? modelUsed`. For heartbeat runs, `providerUsed` is the heartbeat override, so legacy model-only entries can be saved as an invalid heartbeat-provider/primary-model pair. Preserve both stored runtime fields only when both exist, or keep the provider unresolved/inferred from the stored model instead of borrowing the heartbeat provider.
  - confidence: 0.86
- **[P2] Add the required changelog entry:** `src/agents/agent-command.ts:1153`
  - body: This is a user-facing fix for heartbeat model persistence, but the PR only changes agent code and tests. OpenClaw policy requires user-facing fixes to add an Unreleased `CHANGELOG.md` entry with contributor credit before merge.
  - confidence: 0.94
