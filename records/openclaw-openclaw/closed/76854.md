## Summary

Keep open: the PR is close, but the current head can still preserve unrelated live PIDs whose argv ends in generic entrypoints like `dist/index.js`, so it does not fully fix the false-live lock cleanup case.

## What This Changes

The PR adds owner-argv inspection to session lock cleanup, tests, docs, and changelog coverage so `doctor --fix` can remove fresh live-PID locks proven to belong to a non-OpenClaw process.

## Best Possible Solution

Use a single stricter owner classifier that preserves only verified OpenClaw argv, then keep the new doctor/session-lock tests with an added regression for generic entrypoint false positives.

## Reproduction Assessment

Yes, at source level. Current main does not classify fresh live non-OpenClaw owner PIDs as stale, and PR head still preserves generic entrypoint argv such as `node /srv/app/dist/index.js`.

## Solution Assessment

No. Owner argv inspection is the right narrow direction, but preserving locks based only on generic entrypoint filenames leaves the reported false-live class partly unfixed.
