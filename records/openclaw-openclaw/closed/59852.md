## Summary

Current main already implements the PR's intended LSP request-timeout cleanup. `sendRequest()` stores the timeout handle, successful responses clear it, and `disposeSession()` clears pending timers before rejecting outstanding requests. The same behavior landed on main in `dc96886378d52c36ef8582fac97fcf4e19c602c8` as part of bundled LSP cleanup, so this PR is redundant.

## Best Possible Solution

Close this PR as implemented on main and keep the existing `dc96886378d52c36ef8582fac97fcf4e19c602c8` implementation. A future maintainer could add a focused timer-regression test, but the requested runtime behavior is already present.
