## Summary

Keep this PR open. Current main still does not implement the proposed per-skill `persist` override, `metadata.openclaw.persist`, or `skills.trustedSources` scanner-bypass config. The PR is open, unmerged, and is an implementation candidate for still-open #73549; the `trustedSources` portion is security-sensitive because it would skip installed-skill code-safety scanning for configured source classes.

## Best Possible Solution

Keep the PR open and review it alongside #73549. Maintainers should decide whether to accept a schema-backed operator override for skill eligibility, how it relates to existing `metadata.openclaw.always`, and whether any trusted-source scan bypass is acceptable with clear threat-model wording, docs, and tests.
