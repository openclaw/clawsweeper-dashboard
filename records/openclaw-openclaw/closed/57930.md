## Summary

Current `main` no longer sends Codex Responses requests through the old top-level `instructions` payload shape. `openai-codex-responses` is routed through the boundary-aware OpenAI Responses transport, which serializes the system prompt into `input` messages instead. The same behavior is already present in release `v2026.4.23`.

## Evidence
