## Summary

Current main already refreshes Slack assistant status through the shared channel typing keepalive: Slack supplies status start/clear callbacks, and the shared lifecycle repeats the start callback until idle/cleanup.

## What This Changes

Refresh Slack assistant thread status during long-running turns so Slack does not clear the typing/status indicator after its two-minute timeout.

## Best Possible Solution

Keep the shared channel typing keepalive as the source of truth for Slack status refreshes, with future Slack-specific timing changes handled as narrow regressions if needed.

## Reproduction Assessment

No. The reported single-call path is not present on current main: Slack status updates are routed through shared typing callbacks that keep refreshing until idle or cleanup.

## Solution Assessment

Yes. The existing shared typing lifecycle is the narrower maintainable solution because Slack only provides start/clear status callbacks while core owns keepalive, cleanup, and failure-guard behavior.
