## What This Changes

The PR preserves raw fetched Feishu parent-message content, resolves quoted parent media, merges it into inbound agent media context, and adds focused Feishu tests.

## Best Possible Solution

Land a current-main Feishu fix that preserves raw fetched parent content, resolves quoted media before quote-only replies can be skipped, keeps visibility checks, adds quote-only regression coverage, and includes redacted live Feishu proof.

## Reproduction Assessment

Yes, at source level: current main fetches quoted parent messages as parsed placeholder text and builds MediaPaths before the parent fetch, so quoted file/image media cannot reach agent context. I did not run a live Feishu account in this read-only review.

## Solution Assessment

No, not as patched yet. Preserving rawContent and reusing the existing media resolver is the right direction, but quoted-media lookup must happen before the empty-message skip or the skip must account for quoted media, and merge should wait for real Feishu proof.
