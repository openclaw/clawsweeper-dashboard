## Summary

Current `main` already contains OpenAI Responses replay hygiene that drops orphaned `reasoning` signatures, preserves valid reasoning+function-call pairs, and applies that path to `azure-openai-responses`. The same implementation is present in release `v2026.4.23`, which is newer than the reported `2026.3.31` environment.

## Evidence

- **OpenAI replay sanitizer drops orphaned reasoning and rewrites broken fc pairings:** `downgradeOpenAIReasoningBlocks` removes standalone `rs_*` reasoning signatures without a following non-thinking block, and `downgradeOpenAIFunctionCallReasoningPairs` rewrites `call_id|fc_id` pairs when replayable reasoning metadata is missing.
