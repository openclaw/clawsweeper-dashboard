## Summary

Current main still has the reported builtin memory-core failure shape: `corpus="sessions"` is a supported `memory_search` route, but keyword/FTS failures are converted to empty result arrays, trigram MATCH execution has no local retry/fallback, and `memory status --deep` still reports FTS availability rather than representative query health. The issue is concrete and current, and no open PR found by public search references this issue as its fix.

## Best Possible Solution

Keep this issue open and fix the builtin `memory-core` keyword path for session searches: capture/log FTS MATCH failures safely, add a bounded fallback that preserves model/source filters and limits, and add regression coverage for `corpus="sessions"` with English and CJK terms plus a thrown-MATCH fallback path. Add a cheap `memory status --deep` FTS query probe only if it avoids sensitive output and expensive scans.
