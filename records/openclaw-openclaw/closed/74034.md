## Summary

Current main and the shipped v2026.4.26 release address the reported heartbeat feedback loop by removing both self-reinforcing transcript inputs from provider replay: blank user text turns are dropped, and `openclaw/delivery-mirror` / `gateway-injected` assistant entries are omitted from the in-memory replay copy. The original report was against 2026.4.25, and the v2026.4.26 changelog/tag contain the replay repair.

## Best Possible Solution

Close this issue as fixed by the shipped replay sanitization in v2026.4.26. Keep the existing implementation, because it directly breaks the reported cascade while preserving intentional heartbeat and message-tool behavior; any broader heartbeat tool-policy change should be opened as a separate product hardening request.
