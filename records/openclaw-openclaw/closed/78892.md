## Summary

Keep open: current main still limits official-plugin syncing to `openclaw update` and explicit plugin update/doctor flows, while raw package-manager postinstall only runs packaged cleanup and registry migration. The QQBot catalog entry is eligible when the supervised sync path runs, but the requested raw `npm install -g openclaw@latest` behavior remains an unresolved upgrade-contract decision.

## What This Changes

The issue requests that `@openclaw/qqbot` auto-sync to the new OpenClaw release when the host is upgraded with raw `npm install -g openclaw@latest`.

## Best Possible Solution

Define the upgrade contract explicitly: either add a safe supervised post-package official-plugin sync for raw package-manager upgrades, or document that manual npm/pnpm/bun upgrades require `openclaw update`, `openclaw doctor --fix`, or `openclaw plugins update --all` afterward.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: raw npm postinstall runs cleanup and registry migration only, while official plugin sync is wired to `openclaw update`; I did not run a live global npm upgrade in this read-only review.

## Solution Assessment

Unclear. The requested auto-sync may not be the best maintainable solution because npm postinstall network mutation is a product and security policy choice; clarifying docs plus the existing supervised updater/plugin-update commands may be safer.
