## Summary

Current main no longer matches the reported silent zombie-session behavior. Pre-reply embedded-agent failures now return an explicit final error payload, and failed reply operations clear the active-run registry so later messages are not left queued behind a dead session.

## Evidence

- **Final error payload on run failure:** `runReplyAgent()` treats `runAgentTurnWithFallback()` final errors as terminal output, marks the reply operation failed, and returns the payload instead of leaving the run silent.

- **Embedded init/failure path is user-visible:** Generic embedded-agent failures fall back to `Something went wrong...`, OAuth refresh failures fall back to explicit re-auth guidance, and the failure path calls `replyOperation.fail(...)` before returning the final payload.
  - file: [src/auto-reply/reply/agent-runner-execution.ts:371](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/auto-reply/reply/agent-runner-execution.ts#L371)
