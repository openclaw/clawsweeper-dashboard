## Summary

Issue #43146 is still reproducible in current main, but the same remaining work is already tracked by open PR #43276, which explicitly closes #43146 and patches the exact prompt in buildGroupChatContext to allow message tool sends for attachments/files while preserving normal text auto-replies.

## Best Possible Solution

Close #43146 as superseded by the open canonical fix PR #43276, and continue review/landing work there. The product fix should update buildGroupChatContext and its tests so text replies stay auto-routed while attachments/files/images may use the shared message tool in the same Telegram group/topic.
