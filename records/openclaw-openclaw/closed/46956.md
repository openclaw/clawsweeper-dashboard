- **[P1] Register trust RPCs in core method descriptors:** `src/gateway/server-methods/exec-approvals.ts:263`
  - body: Current main builds core Gateway handlers through createCoreGatewayMethodDescriptors, which throws for any handler missing from CORE_GATEWAY_METHOD_SPECS. This PR adds three core trust handlers but only updates the old method-list surface, so a rebase that keeps this shape can fail registry construction instead of exposing the RPCs.
  - confidence: 0.91
- **[P1] Use gateway-owned locality for shared-auth trust callers:** `src/gateway/server-methods/exec-approvals.ts:136`
  - body: The shared-auth fallback requires isLoopbackAddress(client?.clientIp), but current main deliberately stores clientIp as undefined for local direct clients. A localhost token/password CLI without device identity therefore gets rejected even though the PR documents that path as allowed; carry a Gateway-owned locality/auth fact instead of testing this field.
  - confidence: 0.9
- **[P1] Enforce agent-session blocking on the server:** `src/cli/exec-approvals-cli.ts:485-490`
  - body: The self-escalation guard only checks OPENCLAW_SESSION_KEY and OPENCLAW_AGENT_ID in the CLI, and current main does not populate those variables in production paths. An agent-spawned CLI process can therefore bypass this guard if it reaches the trust RPC; the RPC needs a server-side operator/session authority check.
  - confidence: 0.84
- **[P1] Preserve or reject hard-deny policies:** `src/infra/exec-approvals.ts:624-629`
  - body: An active trust window replaces a resolved agent policy with full/off, including when the stored policy is security: "deny". Since current docs treat deny as the way to block remote execution, the grant path should reject hard-deny agents unless maintainers intentionally document and test trust as a stronger override.
  - confidence: 0.88
- **[P2] Audit commands that started under trust:** `src/agents/bash-tools.exec.ts:570-574`
  - body: The audit append re-checks that the window is still active after command completion. Commands that started while trust bypassed approvals but finish after expiry or untrust are omitted from the audit summary, weakening the accountability mechanism for this security-sensitive feature.
  - confidence: 0.87
