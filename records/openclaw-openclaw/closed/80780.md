## Summary

Current main fixes the reported state-corruption path: ack-only heartbeat output is no longer persisted as pending final delivery, stale ack-only pending delivery is cleared before heartbeat replay, and the busy-window check ignores ack-only pending state. The proof commit is on origin/main and is not included in a tagged release yet.

## What This Changes

Prevent Telegram/group heartbeat HEARTBEAT_OK replies from being persisted as pending final delivery and wedging later message delivery.

## Best Possible Solution

Keep the current-main heartbeat pending-delivery normalization and include it in the next release; broader durable-delivery queue work can remain in the separate canonical tracker.

## Reproduction Assessment

Yes for the affected releases: the issue includes concrete session-store state and a second reporter reproduced the Telegram group failure on v2026.5.4. Current main source and regression tests now cover the ack-only pending-delivery state instead of reproducing the failure.

## Solution Assessment

Yes. The current fix targets the narrow corruption path by not storing ack-only heartbeat text as pending final delivery and by clearing stale ack-only state, while preserving real heartbeat content delivery semantics.
