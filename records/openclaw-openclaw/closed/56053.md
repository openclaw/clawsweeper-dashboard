## Summary

Current `main` already classifies HTTP 402 provider failures as failover-worthy, converts embedded assistant-side 402 failures into `FailoverError` instead of letting the run appear successful, and has regression coverage for both direct model fallback and embedded cross-provider fallback paths. The same implementation is present in release tag `v2026.4.22`.
