## Summary

Keep open. Current main still drops short explicit tagged TTS at the unconditional under-10-character guard, while this PR applies the right narrow runtime/test fix and addresses the markdown-only edge case; it still needs docs and changelog cleanup before merge.

## What This Changes

The PR changes speech-core TTS handling so explicit `[[tts:text]]` content can bypass untagged short-text skips, adds a post-markdown empty guard, and adds regression tests.

## Best Possible Solution

Land this PR after a docs/changelog-only follow-up, preserving the explicit-tag bypass and leaving #74955 as separate outbound send-path parity work.

## Reproduction Assessment

Yes. On current main, `[[tts:text]]hello[[/tts:text]]` is parsed into `ttsText`, then the unconditional under-10-character guard returns before synthesis; the PR adds direct regression coverage for that path.

## Solution Assessment

Yes, with docs and release-note cleanup. Separating explicit tagged speech from untagged auto-TTS is the narrowest maintainable fix, and the added empty post-markdown guard covers the reviewed edge case.
