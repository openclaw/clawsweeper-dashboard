## Summary

Keep open. Current main still computes Peak Error Hours by proportionally spreading session-wide message/error counts across session hour slices, and the backend `SessionCostSummary` still has no UTC quarter-hour message-count contract for the UI to consume. The PR remains a focused, actively reviewed implementation candidate; its provided patch is limited to the usage summary/UI/test/changelog surface and does not introduce obvious security-sensitive or supply-chain changes.

Keep this PR open for normal maintainer review and merge readiness.
