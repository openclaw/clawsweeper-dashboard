## Summary

Keep open. The report matches documented Telegram topic and Discord ACP binding behavior, and current shared dispatch source can select the bound ACP/non-main session but suppress origin-channel delivery before routed replies are sent.

## What This Changes

Non-main or specialist agent sessions bound to Telegram topics or Discord channels should deliver normal assistant replies back to the originating conversation.

## Best Possible Solution

Restore origin-channel delivery for conversation-bound non-main and ACP turns while keeping true parent-owned background ACP child sessions silent unless they are the user-facing bound route.

## Reproduction Assessment

No live end-to-end reproduction was run. Source inspection gives a concrete path: bound ACP routing can select the target session, then parent-owned ACP suppression disables origin-channel delivery before the coordinator can route the reply.

## Solution Assessment

Yes, the best fix is a narrow shared dispatch routing-policy change with regression coverage, not Telegram- or Discord-specific adapter work, because both providers feed the same suppression path.
