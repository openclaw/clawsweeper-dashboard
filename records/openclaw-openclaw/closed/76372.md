## Summary

Close as implemented: current source, tests, docs, and the published v2026.5.2 npm tarball already carry the narrower owner-scoped admin cleanup path. The reported `operator.write` default line is real, but it is intentionally kept least-privilege and is no longer the path that blocks same-plugin `subagent.deleteSession` cleanup.

## What This Changes

The issue asks for the synthetic plugin subagent cleanup scope fix to be shipped in v5.x, or for a public release disposition if it is not shipped.

## Best Possible Solution

Keep the shipped least-privilege default plus owner-scoped admin cleanup; if a local install still lacks the lower `deleteSession` block, handle that as an install/package-integrity report with the full bootstrap snippet and package integrity output.

## Reproduction Assessment

No. The grep for the default `operator.write` line reproduces, but the claimed missing `deleteSession` admin dispatch does not: the current source and the published v2026.5.2 tarball include the owner-scoped synthetic admin cleanup path.

## Solution Assessment

Yes. The shipped fix is narrower than widening every synthetic plugin gateway client to admin, while still allowing same-plugin `deleteSession` cleanup and preserving the documented ownership boundary.
