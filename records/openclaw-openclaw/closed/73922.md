## Summary

Current main fixes the reported Bonjour/CIAO gateway crashloop path. The Bonjour plugin now handles CIAO cancellation/assertion process errors through both plugin-runtime handlers and a scoped direct unhandled-rejection listener, recreates the advertiser on CIAO cancellation, logs name/hostname conflict resolution, and disables Bonjour for the current process after repeated stuck probing/announcing restarts instead of taking down the Gateway. The fix is present on main after the latest tagged release; no tag in this checkout contains the final proof commit.

## Best Possible Solution

Close this issue as implemented on current main for the Bonjour/CIAO crashloop. Keep the current plugin-owned recovery behavior and ship it in the next release; open a focused diagnostics issue only if maintainers want full fatal stack capture in stability bundles beyond this fixed crash path.
