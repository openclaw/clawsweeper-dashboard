## Summary

Keep open: the docs-only direction is useful, but the PR still contains an incorrect archive-root assumption that can make the restore examples fail for custom-named or renamed backup archives.

## What This Changes

The PR adds manual restore guidance to `docs/cli/backup.md`, including archive inspection, same-machine restore, staged new-machine restore, and a migration-doc link.

## Best Possible Solution

Update the restore docs to derive the archive root from `openclaw backup verify`, `manifest.json`, or an archive listing before using it, then land the docs fix while leaving native restore-command design to the existing feature tracks.

## Reproduction Assessment

Yes. Current main reproduces the original docs gap, and the PR-specific defect is source-reproducible because `archiveRoot` is timestamp-derived while `--output` can choose an unrelated archive filename.

## Solution Assessment

No, not yet. Manual restore documentation is the narrow maintainable solution for the linked docs gap, but the examples need to stop inferring the archive root from the archive basename.
