## Summary

Current main already handles the reported `openclaw update` failure path: missing private QA scenario files no longer throw during completion-cache refresh, and the shipped `v2026.4.23` release contains that behavior.

## Evidence

- **Current main treats missing QA scenario pack as optional:** `readQaScenarioPack()` now returns an empty QA pack when `qa/scenarios/index.md` is absent, with an inline note that npm distributions may omit that directory so completion-cache consumers do not crash.
  - file: [extensions/qa-lab/src/scenario-catalog.ts:336](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/extensions/qa-lab/src/scenario-catalog.ts#L336)

  - sha: [fd65caf4b0ba](https://github.com/openclaw/openclaw/commit/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc)
- **Current main no longer aborts completion-cache generation on subcli registration errors:** `registerSubcommandsForCompletion()` wraps each subcommand registration in `try/catch` and emits a warning like `skipping subcommand ... while building completion cache` instead of failing the whole completion-cache write.
  - file: [src/cli/completion-cli.ts:55](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/src/cli/completion-cli.ts#L55)

  - file: [src/cli/completion-cli.write-state.test.ts:82](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/src/cli/completion-cli.write-state.test.ts#L82)
