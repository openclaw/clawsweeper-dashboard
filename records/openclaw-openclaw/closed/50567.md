## Summary

Keep open. Current `main` still lacks both core changes from PR #50567: log transport sharing across host/jiti module instances, and opt-in OTEL content capture (`diagnostics.otel.includeContent` with `gen_ai.*`/Langfuse attributes). Recent OTEL work landed on `main`, but it is adjacent work, not this implementation.

## Evidence

- **Log transport state is still module-local:** `src/logging/logger.ts` still defines `externalTransports` as a module-scoped `Set` and `registerLogTransport()` only late-attaches to `loggingState.cachedLogger`. There is no `globalThis` singleton or all-loggers registry, so the PR’s main logger-isolation fix is not present on `main`.
