This report was filed against `2026.2.22-2`, and OpenClaw later shipped a Windows file-tool path fix in `v2026.4.14`: `CHANGELOG.md:712` explicitly notes that drive-letter paths like `C:\Users\...` are now treated as absolute instead of being prefixed with the workspace root. If this still reproduces on a current release, please open a new issue with the current OpenClaw version, exact provider/model route, and the full tool-result/error transcript.
