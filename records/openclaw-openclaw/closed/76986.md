## Summary

Current main already persists and reuses Claude CLI session bindings instead of leaving resume metadata empty, and it has documented/tested handling for stored Claude session ids, transcript validation, and fresh-session fallback.

## What This Changes

The issue asks OpenClaw to persist Claude CLI resume-chain metadata for agent sessions and pass the stored Claude session id on later spawns.

## Best Possible Solution

Keep the current provider-owned cliSessionBindings/cliSessionIds mechanism and documented transcript validation path rather than adding the older null field names back as a parallel contract.

## Reproduction Assessment

No live reproduction was run in this read-only review. The failing behavior was reported against 2026.4.21, while current main has source and regression-test coverage showing Claude CLI session ids are persisted and reused when a readable transcript exists.

## Solution Assessment

Yes. The current implementation is the narrower maintainable fix: store the provider-owned CLI binding, verify Claude's transcript before resume, pass the id through resumeArgs, and clear stale bindings instead of persisting a separate transcriptPath chain.
