## Summary

Current `main` already rate-limits automatic WebSocket fallback attempts for embedded OpenAI sessions, and the same behavior is present in released `v2026.4.23`, so the reported repeated HTTP 500 warn spam is obsolete on supported builds.

## Evidence

- **Auto-mode cool-down on connect failure:** `createOpenAIWebSocketStreamFn` checks `isWsSessionDegraded(session)` before reconnecting, and after a failed connect it marks the session degraded and falls back to HTTP instead of retrying every turn.
  - file: [src/agents/openai-ws-stream.ts:754](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/agents/openai-ws-stream.ts#L754)

  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Bounded retry policy:** The transport limits automatic WebSocket runtime retries to 1 and uses a default 60 second degrade cool-down.
  - file: [src/agents/openai-ws-stream.ts:304](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/agents/openai-ws-stream.ts#L304)
