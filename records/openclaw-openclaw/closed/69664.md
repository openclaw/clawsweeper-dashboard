## What This Changes

The branch modifies `openclaw.mjs` so `isDirectModuleNotFoundError` treats Bun `ERR_MODULE_NOT_FOUND` errors with matching `specifier` or relative-specifier message text as direct launcher misses.

## Best Possible Solution

Land a narrow launcher fix that recognizes Bun direct optional-import misses only when both the missing specifier and Bun referrer match the launcher, with regression coverage and redacted live Bun CLI proof.

## Reproduction Assessment

Yes, source-backed: the linked report gives `bun openclaw.mjs --version`, current `origin/main` only checks Node-style `url` or absolute-path messages, and upstream Bun exposes `specifier` and `referrer`. I did not run Bun in this read-only checkout.

## Solution Assessment

No, not as written. The fix direction is right, but the Bun-specific branches should require the launcher `referrer` instead of accepting any matching relative specifier or message text.
