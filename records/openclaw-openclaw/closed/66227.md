## Summary

Current `main` already contains the gateway-client teardown fix for the post-output CLI hang, with direct regression coverage, and `agents list` no longer uses the gateway path that was reported. The released `v2026.4.23` snapshot also contains the fixed teardown logic.

## Evidence

- **Gateway RPC teardown now waits for client shutdown:** `callGateway` clears the wrapper timeout, then waits for `stopGatewayClient(client)` before resolving/rejecting. `stopGatewayClient` uses `client.stopAndWait({ timeoutMs: 1000 })` and only falls back to `client.stop()` if teardown throws.
