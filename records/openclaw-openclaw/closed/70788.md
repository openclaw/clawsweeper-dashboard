## Summary

Keep open. Current main and latest release still generate the Windows Startup-folder gateway login item as a minimized `cmd.exe` launcher, and the related open Windows launcher PRs do not implement this specific gateway fallback fix.

## What This Changes

The issue requests a silent Windows Startup-folder gateway launcher so login startup does not flash a visible command window.

## Best Possible Solution

Preserve the existing `OpenClaw Gateway.cmd` detection path, delegate Startup-folder launch through a hidden launcher, clean up any sidecar on uninstall, and prove the login flow on Windows 11.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: force the Scheduled Task denied/hung fallback and inspect the generated Startup-folder script, which current tests assert contains the minimized `cmd.exe` launcher. I did not run a live Windows login reproduction in this read-only review.

## Solution Assessment

Yes on the fix boundary: a narrow hidden Startup-folder launcher is the maintainable direction. The exact VBScript/wscript implementation should be validated on Windows and should preserve `.cmd` discovery plus uninstall cleanup.
