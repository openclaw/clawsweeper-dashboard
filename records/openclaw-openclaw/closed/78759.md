## Summary

Keep open. The PR is a plausible fix for the linked gateway auth bug, but it is not merge-ready because it changes a security-sensitive plugin-route scope contract while only updating the pure resolver test; route-dispatch expectations and real behavior proof are still missing.

## What This Changes

The PR changes trusted-operator plugin HTTP route scope resolution so shared-secret token/password callers receive only operator.write, and updates the focused resolver unit test.

## Best Possible Solution

Land a security-reviewed patch that aligns resolver behavior, route-level tests, user-facing docs, release notes, and real gateway proof so shared-secret trusted-operator routes cannot receive implicit admin scopes.

## Reproduction Assessment

Yes, from source and existing tests. Token/password auth sets trustDeclaredOperatorScopes false, the trusted-operator resolver returns default scopes containing operator.admin, and route-dispatch tests assert admin helpers pass on that path.

## Solution Assessment

No, not yet. Returning operator.write in the central resolver is a plausible narrow direction, but this PR must also update route-level expectations and provide real runtime proof before it is the best mergeable solution.
