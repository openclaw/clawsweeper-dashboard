## Summary

Keep open: I found no blocking correctness or security findings, and the PR has sufficient terminal proof, but it changes the plugin discovery contract for untracked global plugin directories and still needs maintainer acceptance before merge.

## What This Changes

The PR changes plugin discovery to require compiled runtime output only for install-record-managed plugin directories, allows untracked global source checkouts to load TypeScript package entries, and updates focused tests, docs, and changelog coverage.

## Best Possible Solution

Land the install-record-based discovery policy if maintainers accept the global source-checkout contract, keeping the strict managed-install behavior, docs, changelog, and regression tests.

## Reproduction Assessment

Yes. Current main is source-reproducible: global package discovery resolves openclaw.extensions before manifest/default fallback, and global TypeScript package entries without built JavaScript hit the compiled-runtime warning path.

## Solution Assessment

Yes, with maintainer policy acceptance. Classifying managed paths from install records is narrower than flipping manifest precedence and preserves strict handling for managed installs and explicit runtimeExtensions.
