## Summary

Close as duplicate/superseded by #45706. The reporter confirmed the original Gemini/Feishu symptom was fixed in their upstream OpenAI-compatible backend/proxy. The remaining OpenClaw-side compatibility gap is the generic OpenAI-compatible request-shaping issue: `buildOpenAICompletionsParams` still always sends `stream_options.include_usage` even when compat says streamed usage is unsupported, and #45706 is already open as the broader tracker for that same request-shaping fix.

## Best Possible Solution

Close this issue as superseded by #45706. Track the remaining OpenAI-compatible request-shaping work there: honor `compat.supportsUsageInStreaming: false` when deciding whether to emit `stream_options.include_usage`, preserve usage reporting for native and manifest-declared compatible providers, and avoid changing Feishu card streaming policy.
