## Summary

Current main already fixes the reported completion-loss path: completion-mode dispatch no longer depends on a queue-first path, and an active requester wake failure now falls through to direct completion fallback with regression coverage. This is present on main but not in the latest tagged release v2026.5.3.

## What This Changes

The issue asks for successful subagent completions to reach the active requester session even when `queueEmbeddedPiMessage(...)` cannot wake or queue the parent run.

## Best Possible Solution

Keep the current main implementation and let the next release carry the fix to users still on v2026.5.3-1.

## Reproduction Assessment

No current-main failing reproduction remains. Source inspection shows the queue-false path now reaches direct completion fallback, and the regression test covers that exact active-requester failure shape.

## Solution Assessment

Yes. The current fix is the narrow maintainable path because it preserves active-session wake when possible, then directly delivers the captured child result instead of silently losing it.
