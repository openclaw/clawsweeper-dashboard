## Summary

Current main fixes the reported Discord owner-auth gap: the Discord config adapter now resolves owner allowlists through the effective account allowFrom helper, so top-level `channels.discord.allowFrom` is included before command authorization derives `ownerList` and `senderIsOwner`. The fix is present on main but not in the latest tagged release `v2026.4.26`.

## What This Changes

The issue requests that Discord DM owner authorization honor top-level `channels.discord.allowFrom` when `commands.ownerAllowFrom` is unset, so owner-only tools such as `cron` and `gateway` remain available to the allowlisted sender.

## Best Possible Solution

Keep the current-main Discord adapter fix, let it ship in the next release, and leave the broader security-audit warning tracked separately in #39341.
