## What This Changes

The branch moves `messages.suppressToolErrors` ahead of mutating-tool warning generation in `buildEmbeddedRunPayloads` and updates the focused regression expectation.

## Best Possible Solution

Merge the focused guard reorder and regression after normal CI/proof gate reconciliation; keep broader warning-delivery modes tracked in https://github.com/openclaw/openclaw/issues/39406.

## Solution Assessment

Yes. Moving the existing suppression guard before mutating-tool warning generation is the narrow maintainable fix, and it preserves the stronger per-run `suppressToolErrorWarnings` and `sessions_send` behavior.
