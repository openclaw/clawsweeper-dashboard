## Summary

Keep this PR open. Current main still has the exact extra-params lookup bug the PR targets: `resolveExtraParams` builds `provider/modelId` with raw string concatenation, so `provider: "openrouter"` plus `modelId: "openrouter/auto"` still misses an `agents.defaults.models["openrouter/auto"].params` entry. Adjacent model-selection paths already canonicalize this class of model key, but that canonicalization has not been applied to the extra-params path that feeds stream options like temperature, transport, and cache retention.

## Best Possible Solution

Review and land this PR or an equivalent maintainer patch that changes `resolveExtraParams` to use `modelKey` with `legacyModelKey` fallback, matching `resolveThinkingDefault`, and adds regression coverage for `openrouter/auto` plus nested OpenRouter model ids.
