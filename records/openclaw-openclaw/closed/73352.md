## Summary

This issue should close as duplicate/superseded. Current main still lacks MCP OAuth Authorization Code + PKCE support, but the exact remaining work is already covered by the earlier MCP OAuth issue #61611 and the open implementation PR #62134.

## Best Possible Solution

Close this issue as a duplicate/superseded tracker. The best path is for maintainers to review, land, or close #62134 against the prior #61611 product decision; if the OAuth direction changes, update that canonical thread rather than splitting the same MCP client auth work across another issue.

## Evidence

  - file: [src/agents/mcp-transport.ts:102](https://github.com/openclaw/openclaw/blob/cd1343c2447adc2551c16b8e81f9104cf9231af3/src/agents/mcp-transport.ts#L102)
