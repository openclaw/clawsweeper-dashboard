## Summary

Keep open: the PR addresses a real CLI hint bug and includes strong terminal proof, but the proposed classifier can incorrectly soften the restart hint for whole-agent replacements that remove an existing `agentRuntime.id`. That edge case would contradict the PR's conservative contract for runtime-structure changes.

## What This Changes

The PR adds config CLI hint selection and tests so hot-loadable `agents.list[N]` mutations can say they apply on next agent invocation while restart-required paths keep the gateway restart hint.

## Best Possible Solution

Land the CLI hint change after making the classifier conservative for whole-agent replacements or before/after `agentRuntime` structural changes, with a regression test for replacing a dormant agent entry.

## Reproduction Assessment

Yes. Current main source still emits the restart hint unconditionally while `agents.list` is a hot-reload prefix, and the linked issue plus PR body provide terminal proof for the active-agent case; the PR edge case is also source-reproducible from the helper logic.

## Solution Assessment

Not quite. The narrow hint-selection approach is the right shape, but the current post-mutation-only classifier should stay restart-required for whole-agent replacements that may remove or change `agentRuntime`.
