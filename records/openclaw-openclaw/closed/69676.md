## Summary

Close PR #69676 as implemented_on_main. Current main already resolves `github-copilot` as a memory embedding provider when the memory provider registry is cold, using the bundled plugin capability contract instead of preloading all plugins for `openclaw memory` commands. The linked issue #69180 is closed, and the implementation is present in the v2026.4.24 release tag.

## Best Possible Solution

Do not merge PR #69676's broad `memory` command plugin-preload policy. Keep the shipped config-aware capability-provider implementation, which resolves plugin-owned memory embedding providers on demand; any future regression test should target that cold-registry provider resolution path rather than forcing all memory CLI commands to preload every plugin.
