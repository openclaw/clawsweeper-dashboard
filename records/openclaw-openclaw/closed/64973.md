## Summary

Keep this open. Current main has a partial macOS config hardening fix that preserves `gateway.auth` and unrelated keys, but the stale `config.set` path can still fall back to a direct macOS file write and `AppState` still has a direct config writer outside the gateway base-hash contract.

## What This Changes

The issue asks for macOS Talk Mode config writes to avoid stale `config.set` fallback writes, gateway auth token drift, Talk provider loss, and local gateway restarts.

## Best Possible Solution

Land a focused macOS config-writer fix that treats stale gateway config writes as conflicts, preserves auth and unrelated keys, and adds Swift regression coverage before closing this issue.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the reported failure shape: the issue includes concrete Talk Mode steps/logs, and current main still contains the stale `config.set` plus direct local-write path. The exact live Talk Mode race was not replayed during this read-only review.

## Solution Assessment

Is this the best way to solve the issue? No current-main fix is complete: the #75631-era preservation work reduces auth clobbering, but stale gateway rejections should not be silently converted into direct full-file writes. The safer path is the focused macOS ConfigStore/AppState/OpenClawConfigFile repair already represented by #75815 or an equivalent narrow PR.
