## Summary

Current main already covers the reported zhipu-0ki/GLM-5 ByteString failure path: custom-provider API keys are normalized before use, and the embedded runner now honors `models.providers.*.authHeader` by injecting `Authorization: Bearer <apiKey>` for OpenAI-compatible providers.

## Evidence

- **Custom provider API keys are normalized before use:** `getCustomProviderApiKey()` reads `models.providers.<id>.apiKey` through `normalizeOptionalSecretInput(...)`, so direct config values are sanitized before becoming runtime auth.
  - file: [src/agents/model-auth.ts:73](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/agents/model-auth.ts#L73)
