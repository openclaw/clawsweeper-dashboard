## Summary

Close: current main already removes the hardlink-based Telegram spool claim by routing Telegram ingress claims through the shared SQLite channel ingress queue; the fix is current-main-only after v2026.5.28.

## What This Changes

The issue reports Telegram ingress updates on Android 12 / Termux staying in the spool because v2026.5.28 claimed files with fs.link(), which SELinux rejects with EACCES.

## Best Possible Solution

Keep the SQLite-backed channel ingress queue implementation and ship it in the next release; no per-platform hardlink fallback or env knob is needed.

## Reproduction Assessment

Yes for the reported release by source inspection: v2026.5.28 hardlinks during the claim step, matching the EACCES log. Current main no longer has that failing path because claims are SQLite-backed.

## Solution Assessment

Yes. Moving the spool claim to the shared SQLite ingress queue is a narrower and more maintainable fix than adding an Android-specific fs.link fallback or a new claim-mode environment option.
