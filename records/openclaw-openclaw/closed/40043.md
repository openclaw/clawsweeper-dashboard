## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Enforce the candidate cap during recursive discovery:** `src/agents/skills/workspace.ts:204-247`
  - body: The recursive walker only stops after it finds `maxSkillsLoadedPerSource` skill roots, so a large tree with few or no `SKILL.md` files can still be traversed up to depth 10. This regresses the old bounded scan behavior and can make startup/refresh unexpectedly expensive for repo-style skill sources.
  - confidence: 0.9
- **[P1] Keep auth diagnostics off the throwing startup path:** `src/gateway/server-startup-log.ts:15-16`
  - body: `hasAuthForProvider` calls `ensureAuthProfileStore` directly from startup logging without a non-throwing wrapper. A best-effort warning should not be able to propagate auth-store I/O or parsing failures and block gateway startup.
  - confidence: 0.86
- **[P2] Use the runtime auth checker for provider availability:** `src/gateway/server-startup-log.ts:10-16`
  - body: The helper checks only environment API keys and stored profiles, while runtime auth also accepts config-backed custom keys, AWS SDK/Bedrock auth, synthetic local providers, and plugin-provided synthetic auth. Valid configurations can therefore get a misleading warning that requests will fail.
  - confidence: 0.9
- **[P1] Do not recurse through uncontained symlink directories:** `src/agents/skills/workspace.ts:217-223`
  - body: The new scanner follows symbolic-link directories before checking for skill roots, which expands traversal beyond the configured skill root. Recursive discovery over user-controlled skill folders needs realpath containment before descending, matching the bounded implementation now on main.
  - confidence: 0.8

## Security Review

Status: needs_attention

Summary: The PR adds no dependencies, workflows, lockfiles, or publishing changes, but it broadens user-controlled filesystem traversal in a way that needs attention.

Concerns:

- **[medium] Recursive skill scan can escape configured roots:** `src/agents/skills/workspace.ts:217`
  - body: The scanner follows symlink directories and recurses before any realpath containment check, so a configured skill root can lead discovery into directories outside that root. That is security-sensitive because skills are untrusted local content that affects agent prompt/tool behavior.
  - confidence: 0.8
- **[low] Recursive scan lacks a scanned-directory budget:** `src/agents/skills/workspace.ts:204`
  - body: The traversal is bounded by found skills and depth, but not by scanned candidates. Large user-controlled trees can create startup or refresh denial-of-service behavior compared with the bounded current-main implementation.
  - confidence: 0.9
