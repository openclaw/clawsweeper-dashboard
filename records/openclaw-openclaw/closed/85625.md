## Summary

Keep this PR open. The intended fix targets a real P1 session-state problem, but the patch skips final `result` usage without first reading the nested assistant `message.usage` shape that the linked report and current Claude transcript importer rely on, so real sessions could end with missing token snapshots instead of corrected ones.

## What This Changes

The branch changes `createCliJsonlStreamingParser` to skip usage updates from `result` events and adds a regression test expecting assistant-event usage to remain the final parser usage.

## Best Possible Solution

Land the accounting fix after the parser reads the real assistant-message usage shape, ignores cumulative result usage for session snapshots, and has redacted live CLI JSONL or terminal proof for a tool-heavy turn.

## Reproduction Assessment

Yes for the review finding: source inspection shows the proposed guard drops `result` usage while the reader still misses nested assistant `message.usage`. I did not run a live Claude CLI turn in this read-only review.

## Solution Assessment

No. Ignoring cumulative `result` usage is the right direction, but the patch should first parse assistant `message.usage` and test that real shape rather than a top-level assistant fixture.
