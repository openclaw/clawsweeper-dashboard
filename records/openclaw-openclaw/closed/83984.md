## Label Justifications

- `P1`: The report describes an immediate crash that blocks Codex OAuth GPT agent turns in the affected beta.

- `impact:auth-provider`: The affected workflow is the OpenAI Codex OAuth provider and GPT model route.

## Summary

The reported beta-package missing export is fixed in current `main` and in published packages after the affected beta; the first npm version I found with the export is `v2026.5.16-beta.2`, and the latest stable `v2026.5.18` also contains it.

## What This Changes

The issue reports that Codex OAuth GPT models crash immediately because the built harness runtime calls `hasBeforeToolCallPolicy` but the affected beta dist did not export it.

## Best Possible Solution

Keep the shipped SDK runtime export and ask affected beta users to upgrade to `v2026.5.16-beta.2` or newer, preferably the latest stable release.

## Reproduction Assessment

Yes, at package-artifact level: the affected `2026.5.14-beta.2` dist entrypoint lacks `hasBeforeToolCallPolicy`, while later package artifacts export it. I did not run a live Codex OAuth turn because this read-only review avoided runtime setup and credentials.

## Solution Assessment

Yes. The narrow maintainable fix is for the supported plugin SDK harness runtime entrypoint to export the helper that the Codex app-server path imports, and that is now present in source and published packages.
