## Summary

Keep this PR open: the Slack assistant direction is useful and matches the linked feature request, but the patch still has a concrete assistant-context fallback bug and the PR body only provides tests/CI output, not real Slack assistant-thread proof. The protected `maintainer` label also means this is not a cleanup-close candidate.

## What This Changes

The PR adds Slack assistant thread lifecycle events, assistant context caching, thread-scoped assistant DM routing, forced assistant-thread replies/status/streaming hints, setup/docs updates, and regression tests.

## Best Possible Solution

Land the Slack assistant lifecycle feature after fixing assistant context normalization and adding redacted live Slack assistant-thread proof for the changed behavior.

## Reproduction Assessment

Yes: a source-level repro is clear by invoking `assistant_thread_context_changed` with `assistant_thread.context` present but partial and top-level context populated. The new normalizer selects the partial nested object as a whole, so the populated fallback fields are stored as undefined.

## Solution Assessment

No: the feature direction is reasonable, but this is not ready as-is. The safer solution is to resolve assistant context fields independently, avoid unintended context loss, and prove the lifecycle in a real Slack assistant thread before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Fall back per assistant context field:** `extensions/slack/src/monitor/events/assistant.ts:67-69`
  - body: When `assistant_thread.context` exists but omits one of the optional context fields, `(thread.context ?? event.context)` prevents fallback to the top-level context for every field. A context-changed event with a partial nested object can therefore save `undefined` for channel/team/enterprise and overwrite the cached assistant context, so later turns lose the Slack context metadata this PR is trying to carry. Resolve each field independently, and add a regression case for a partial nested context with top-level fallback values.
  - confidence: 0.87
