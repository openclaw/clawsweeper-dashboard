## Summary

Close as implemented on current main. WhatsApp now has a documented `channels.whatsapp.exposeErrorText` setting, with channel-level and account-level config support, that suppresses user-visible WhatsApp error payloads marked `isError` while leaving ordinary agent text delivery intact. The fix is present on current main but is not in the latest inspected release tag, so no shipped release can be named yet.

## What This Changes

Request to add a WhatsApp configuration knob that suppresses gateway-synthesized error replies, matching the user-visible behavior requested from Telegram error-reply controls.

## Best Possible Solution

Keep the current `channels.whatsapp.exposeErrorText` implementation and documentation as the maintained WhatsApp error-visibility control; add a `silentErrorReplies` alias only if maintainers explicitly want name-level compatibility later.
