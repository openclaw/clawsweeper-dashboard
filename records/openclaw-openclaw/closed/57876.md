## Summary

Current `main` already guards the `read` tool against missing or blank `path` values before execution, and the embedded runner now only logs the bad call instead of proceeding into filesystem access. The reported empty-path `ENOENT` crash path is fixed on `main`, but it does not appear to be in a tagged release yet.

## Evidence

- **Read tool now enforces required path before execution:** `createOpenClawReadTool()` calls `assertRequiredParams(..., REQUIRED_PARAM_GROUPS.read, ...)` before any read execution, so null/empty/missing `path` is rejected before filesystem access.
  - file: [src/agents/pi-tools.read.ts:680](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/src/agents/pi-tools.read.ts#L680)
  - command: `nl -ba src/agents/pi-tools.read.ts | sed -n '674,700p'`
  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
- **Required-param validator treats blank strings as missing:** `REQUIRED_PARAM_GROUPS.read` requires `path`, and `assertRequiredParams()` rejects non-object, missing, non-string, or whitespace-only values.
  - file: [src/agents/pi-tools.params.ts:75](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/src/agents/pi-tools.params.ts#L75)
  - command: `nl -ba src/agents/pi-tools.params.ts | sed -n '60,130p'`
  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
- **Bad read calls are now warn-only at tool-start logging:** Embedded tool-start handling checks `path`/`file_path`, trims it, and emits `read tool called without path` when empty instead of attempting execution there.
  - file: [src/agents/pi-embedded-subscribe.handlers.tools.ts:586](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/src/agents/pi-embedded-subscribe.handlers.tools.ts#L586)
  - command: `nl -ba src/agents/pi-embedded-subscribe.handlers.tools.ts | sed -n '586,606p'`
  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
- **Regression coverage added for missing/blank params and warn path:** Tests now cover runtime validation failures for blank/missing params and the specific warn-only case when a read call has neither `path` nor `file_path`.
  - file: [src/agents/pi-tools.params.test.ts:44](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/src/agents/pi-tools.params.test.ts#L44)

  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
