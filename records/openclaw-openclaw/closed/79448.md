## Summary

Close as implemented: current main removes the Telegram private-topic threadless fallback, locks the behavior with regression tests, and the fix shipped in v2026.5.9-beta.1 via the merged Telegram PR.

## What This Changes

The issue asks for Telegram cron deliveries that fall back from a DM topic to the bare DM to be surfaced instead of persisted as ordinary delivered.

## Best Possible Solution

Keep the fail-closed Telegram private-topic behavior so rejected DM-topic sends surface as delivery errors instead of becoming plain-DM successes.

## Reproduction Assessment

No for current main: source inspection and regression tests show DM-topic thread-not-found errors are not retried threadless, so the v2026.5.7 fallback-to-delivered path is no longer present. The old behavior is well evidenced by the issue report and the merged PR's before/after live proof.

## Solution Assessment

Yes. Failing private-topic sends loudly is narrower and safer than adding a delivered-fallback status while still allowing wrong-surface delivery.
