## Summary

Current `main` already addresses the issue’s concrete Ollama root cause: embedded runs now default to `thinking=off`, and native Ollama `/api/chat` requests only send `think:true` when the user explicitly enables thinking. The behavior is covered by regression tests and documented, but it does not appear in the latest tagged release `v2026.4.23` yet.

## Evidence

- **Embedded runs now default thinking off:** The embedded runner initializes `initialThinkLevel` as `params.thinkLevel ?? "off"`, so a plain `openclaw infer model run ...` no longer implicitly opts local models into thinking mode.
  - file: [src/agents/pi-embedded-runner/run.ts:501](https://github.com/openclaw/openclaw/blob/306c0f73bff1338482dd6533f61a86905bf541b1/src/agents/pi-embedded-runner/run.ts#L501)

  - sha: [306c0f73bff1](https://github.com/openclaw/openclaw/commit/306c0f73bff1338482dd6533f61a86905bf541b1)
- **Native Ollama transport maps off to think=false:** The Ollama stream wrapper applies top-level `think: false` for native Ollama transport when `thinkingLevel === "off"`, and only applies `think: true` for explicit non-off levels.
  - file: [extensions/ollama/src/stream.ts:199](https://github.com/openclaw/openclaw/blob/306c0f73bff1338482dd6533f61a86905bf541b1/extensions/ollama/src/stream.ts#L199)

  - file: [extensions/ollama/src/stream-runtime.test.ts:100](https://github.com/openclaw/openclaw/blob/306c0f73bff1338482dd6533f61a86905bf541b1/extensions/ollama/src/stream-runtime.test.ts#L100)
