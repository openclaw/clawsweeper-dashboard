## Repair Work Prompt

Fix the false-clean plain `openclaw memory status` path for builtin session transcript indexing. Current main constructs status-purpose memory managers as transient, so `ensureSessionStartupCatchup()` is skipped, while `status().dirty` only reports manager dirty flags and index identity. Add a bounded status-path session drift signal, preferably by reusing the existing `listSessionFilesForAgent` + `loadMemorySourceFileState({ source: "sessions" })` + `resolveMemorySessionStartupDirtyFiles` comparison without running a full reindex from plain status. Plain status should report dirty=true, or an equivalent status field that makes the CLI print Dirty: yes, when usage-counted session transcripts exist on disk but are missing/stale in `memory_index_sources` for source `sessions`. Keep `--index` behavior as the repair path, do not add config/env options, do not broaden to QMD or memory-file watcher policy, and do not edit CHANGELOG.md. Coordinate with https://github.com/openclaw/openclaw/issues/62625 and https://github.com/openclaw/openclaw/pull/82341 as prior partial fixes, and avoid bundling broader uninvoked-agent memory-file policy from https://github.com/openclaw/openclaw/issues/94888.
