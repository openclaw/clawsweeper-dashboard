## Summary

Keep this PR open: it targets a real `agents add` model-inheritance bug, but the current patch is not merge-ready because it suppresses the provider-selected per-agent override even when no global default exists, lacks real CLI proof, and is currently conflicting with main.

## What This Changes

The PR changes provider auth-choice default-model handling and related tests so `setDefaultModel: false` restores existing defaults and stops returning `agentModelOverride` during agent setup.

## Best Possible Solution

Land a narrow conditional fix that omits per-agent overrides only when an existing global primary should be inherited, preserves provider-selected overrides when no global default exists, and includes real CLI proof for both setup cases.

## Reproduction Assessment

Yes, source-reproducible. Current main’s `agents add` path calls `applyAuthChoice` with `setDefaultModel: false` and persists only `authResult.agentModelOverride`, while provider auth returns the selected default as that override.

## Solution Assessment

No. The PR targets the right seam, but the best fix is conditional: inherit an existing global primary while preserving the selected provider model when no global primary exists.
