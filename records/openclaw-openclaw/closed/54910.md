## Summary

Current `main` already supports overriding the Bonjour/mDNS hostname via `OPENCLAW_MDNS_HOSTNAME`, and that value is used for the advertised hostname plus the default instance name/TXT `lanHost`, which addresses the collision problem described in #54910.

## Evidence

- **Advertiser reads configurable mDNS hostname:** The Bonjour advertiser reads `process.env.OPENCLAW_MDNS_HOSTNAME`, normalizes it, and uses it for the advertised `hostname`, default `instanceName`, and `lanHost` TXT field.
  - file: [extensions/bonjour/src/advertiser.ts:194](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/extensions/bonjour/src/advertiser.ts#L194)

  - file: [docs/gateway/configuration-reference.md:551](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/docs/gateway/configuration-reference.md#L551)
