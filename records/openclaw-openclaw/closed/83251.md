Full review comments:

- **[P1] Register typed hooks through api.on:** `packages/agent-shield/src/index.ts:24`
  - body: `registerHook` is the internal hook registration API and currently requires a hook name in options; calling it this way for `before_prompt_build` will throw `hook registration missing name` during plugin load. Use the typed plugin hook API (`api.on("before_prompt_build", ...)` / `api.on("message_sending", ...)`) so these lifecycle hooks actually register.
  - confidence: 0.94
- **[P1] Ship the runtime entry named in package metadata:** `packages/agent-shield/package.json:10`
  - body: The package tells OpenClaw to load `./src/index.ts`, but the published `files` list only includes `dist`, `openclaw.plugin.json`, and `README.md`. A ClawHub/npm install would not contain the configured entry, so point `openclaw.extensions` at the built `./dist/index.js` entry or ship the source intentionally.
  - confidence: 0.9
- **[P2] Use the supported agent tool registrar:** `packages/agent-shield/src/index.ts:175`
  - body: `OpenClawPluginApi` exposes `registerTool`, not `registerAgentTool`; with optional chaining this silently skips both advertised `agent_shield_*` tools even after the plugin loads. Register these through the documented tool API so runtime inspection and agent tool dispatch can see them.
  - confidence: 0.88
- **[P2] Use the plugin CLI registration API:** `packages/agent-shield/src/index.ts:274`
  - body: `registerCliCommand` is not part of the current plugin API, so the advertised `openclaw agent-shield ...` commands will not be registered. Use `registerCli` with descriptors or an existing command registration surface instead.
  - confidence: 0.86
- **[P2] Update the lockfile for the new workspace package:** `packages/agent-shield/package.json:35-39`
  - body: `pnpm-workspace.yaml` includes `packages/*`, so adding this package creates a new workspace importer with new dependency metadata. Without the corresponding `pnpm-lock.yaml` importer update, frozen installs and dependency review will not have a reproducible package graph.
  - confidence: 0.82
