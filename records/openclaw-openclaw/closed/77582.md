## Summary

Current main already fixes the reported Telegram selected-quote prompt loss by carrying reply context separately from transcript text and appending it to the model-visible prompt before `finalPromptText` is recorded; the fix is on main only and is not contained in the latest `v2026.5.3` release tag.

## What This Changes

Telegram selected quote text should remain model-visible in `finalPromptText` when normalized reply context includes it.

## Best Possible Solution

Keep the generic current-turn reply context path on main and let the next release carry it; no Telegram-only prompt assembly fork is needed.

## Reproduction Assessment

No for current main. The 2026.5.2 reproduction is credible, but current main source and regression coverage show `ReplyToBody`/`ReplyToIsQuote` are appended to `promptForModel` and recorded in `finalPromptText`.

## Solution Assessment

Yes. The current implementation is the narrow maintainable fix: preserve clean transcript text while adding explicit reply context only to the current model input.
