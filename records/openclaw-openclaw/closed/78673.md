## Summary

Current main and the latest release contain the two relevant fixes: Gemini 3.1 Pro Preview thought-signature-only stream chunks now count as activity before visible text, and Discord no longer applies a channel-owned queued-run timeout to long replies. The report is against OpenClaw 2026.5.4, so the conservative cleanup outcome is to close as already implemented and ask the reporter to update.

## What This Changes

Discord background worker turns using google/gemini-3.1-pro-preview should keep sustained generation alive instead of hitting the LLM idle timeout while the same prompt works in the TUI.

## Best Possible Solution

Keep the current Google stream-activity handling and shared Discord run queue behavior; users on 2026.5.4 should update to v2026.5.6 or newer.

## Reproduction Assessment

No live high-confidence current-main reproduction was run. The 2026.5.4 report matches the known source-level Gemini thinking-only idle path and Discord queued-run timeout path, both of which current main now covers.

## Solution Assessment

Yes. The current solution is the narrow maintainable fix: provider streaming emits activity for Gemini thoughtSignature-only chunks, while Discord relies on shared runtime/provider timeout policy instead of a channel-specific long-run cutoff.
