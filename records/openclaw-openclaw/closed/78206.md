## Summary

Keep open: current main has partial Matrix room-case protections for stored delivery metadata, cron, ACP, and poisoned queue recovery, but source inspection still shows outbound/tool producers can pass a lowercased session/group-derived target and the queue will preserve that bad `to` exactly.

## What This Changes

Matrix outbound delivery should preserve provider-exact mixed-case room IDs instead of deriving send/read targets from lowercased session or group identifiers.

## Best Possible Solution

Use preserved route metadata as authoritative for Matrix outbound targets whenever a candidate target came from a session/group key, while keeping lowercased identifiers only for indexing and lookup.

## Reproduction Assessment

Yes, from source inspection and the reporter's detailed logs, but not from a live Matrix run here. Current code intentionally lowercases session/group identities, and outbound producers plus queue recovery still trust whatever `to` they are handed while Matrix helpers cannot recover original room ID casing.

## Solution Assessment

Yes. Fixing the outbound target-selection seam to prefer `deliveryContext.to`, `lastTo`, `origin.to`, or `origin.nativeChannelId` is narrower and more maintainable than changing generic session-key normalization or Matrix room ID semantics.
