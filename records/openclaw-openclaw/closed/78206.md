## Summary

Current main already keeps Matrix room IDs case-exact through inbound context, reply routing, message actions, cron inference, session target resolution, and queue replay, so the old-beta lowercased-target failure is no longer actionable as an open core bug.

## What This Changes

The issue requests that Matrix outbound/read/retry routing preserve provider-exact mixed-case room IDs instead of leaking lowercased session or group keys into delivery targets.

## Best Possible Solution

Keep the current main implementation and ship it in the next release; operators on the older beta should upgrade or correct any legacy lowercased Matrix delivery targets to the exact room ID.

## Reproduction Assessment

No, not against current main. Source inspection and regression coverage show the lowercased session key is no longer used as the Matrix delivery target when exact `deliveryContext.to`, `lastTo`, or current chat context exists.

## Solution Assessment

Yes. The current approach is the narrow maintainable fix: session/group keys can stay lowercase for lookup, while Matrix sends, reads, cron delivery, and queue replay use the provider-exact delivery target.
