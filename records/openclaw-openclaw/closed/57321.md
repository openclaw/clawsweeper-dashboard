## Summary

Current `main` addresses both parts of the report: Matrix restart/recovery now has a canonical-route fallback for lowercased session keys, and Matrix `User ... not in room` recovery failures are treated as permanent so poisoned deliveries are moved to `failed/` instead of causing an endless restart loop.

## Evidence

- **Canonical route recovery from session metadata:** `extractDeliveryInfo` reads the stored session entry and returns routable delivery context from persisted metadata instead of trusting the session key shape. If a thread key lacks route metadata, it falls back to the base session entry, which is the relevant fix for lowercased Matrix session keys on restart flows.
