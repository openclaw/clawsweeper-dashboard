## Summary

Keep open: this PR is not implemented on current main, and it addresses a real shipped/current cron retry-classification bug with focused tests and supplied classifier proof. The remaining merge question is compatibility: narrowing default `server_error` matching may stop retries for real transient 5xx text that lacks the accepted HTTP/status context or canonical phrases.

## What This Changes

The PR replaces cron's broad `server_error` 5xx regex with a context-aware pattern and adds retry-hint tests for incidental 5xx-looking numbers versus genuine HTTP/status 5xx messages.

## Best Possible Solution

Land a context-aware `server_error` classifier after maintainers accept the default retry recall tradeoff, or add any known real uncontextualized 5xx message before merge.
