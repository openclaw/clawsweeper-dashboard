## Summary

Current `main` already contains the targeted fix for this startup-race: CLI gateway RPCs yield one event-loop turn before opening the WebSocket, and `openclaw logs` still uses that shared CLI gateway path. The behavior is also covered by a regression test, and the same fix is present in release `v2026.4.22`.

## Evidence

- **Shared CLI gateway path now yields before WS connect:** `src/gateway/call.ts:479`-`483` explicitly adds `await new Promise<void>((r) => setImmediate(r));` with a comment that heavy synchronous module loading can starve the event loop and cause `connect.challenge` handshake timeouts.
