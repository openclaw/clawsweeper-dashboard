## Summary

Current `main` does not implement this PR's `existing-session` `cdpPort` targeting. The browser plugin still drops `cdpPort` for `existing-session` profiles, launches Chrome MCP with only `--autoConnect`/optional `--userDataDir`, and its create-profile/API/CLI paths do not accept or persist an `existing-session` port. This PR is not safely closable as already implemented or obsolete.
