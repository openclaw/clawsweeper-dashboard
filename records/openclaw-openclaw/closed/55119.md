## Summary

Already implemented and shipped: the Ollama plugin now sends native top-level `think=false` when thinking is off, and it preserves thinking-only/reasoning-only Ollama output so the empty-response retry path described in the issue no longer matches current code.

## Evidence

- **Native Ollama requests now set `think` from resolved thinking level:** `createConfiguredOllamaCompatStreamWrapper` wraps native Ollama transport and sets top-level `think=false` when `ctx.thinkingLevel === "off"`, or `think=true` for non-off thinking levels.
  - file: [extensions/ollama/src/stream.ts:154](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/extensions/ollama/src/stream.ts#L154)
  - command: `nl -ba extensions/ollama/src/stream.ts | sed -n '140,230p'`
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **The wrapper is on the real provider path:** The bundled Ollama provider registers `wrapStreamFn: createConfiguredOllamaCompatStreamWrapper`, so the `think` patch applies to normal Ollama streaming requests.
  - file: [extensions/ollama/index.ts:172](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/extensions/ollama/index.ts#L172)
