## Summary

The reported Windows `.backupignore` permission-check bug does not reproduce against current `main` because the shipped backup implementation on `aa21d4ea7e9aab034acbf0b13f8c3575057c5c54` does not contain the `.backupignore` / `--exclude-file` feature or the referenced `src/infra/backup-exclude.ts` code path.

## Evidence

- **Current backup CLI surface has no exclude-pattern options:** `backup create` currently exposes only `--output`, `--json`, `--dry-run`, `--verify`, `--only-config`, and `--no-include-workspace`. There is no `--exclude-file` or related option on current `main`.
  - file: [src/cli/program/register.backup.ts:20](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/cli/program/register.backup.ts#L20)
  - sha: [aa21d4ea7e9a](https://github.com/openclaw/openclaw/commit/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54)
- **Current backup implementation options have no exclude support:** `BackupCreateOptions` only includes `output`, `dryRun`, `includeWorkspace`, `onlyConfig`, `verify`, `json`, and `nowMs`; there is no exclude-pattern or permission-check path in the current backup create implementation.
  - file: [src/infra/backup-create.ts:26](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/src/infra/backup-create.ts#L26)
  - sha: [aa21d4ea7e9a](https://github.com/openclaw/openclaw/commit/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54)
- **Published backup docs match the reduced surface:** The backup docs list only create/verify, output, dry-run, verify, no-include-workspace, and only-config flows. They do not mention `.backupignore` or `--exclude-file`.
  - file: [docs/cli/backup.md:13](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/docs/cli/backup.md#L13)
  - sha: [aa21d4ea7e9a](https://github.com/openclaw/openclaw/commit/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54)
- **Repo-wide search finds no reported code path on current main:** A repo-wide search for `backupignore|exclude-file|ExcludeFileError|readPatternFile|permissionCheck` found no backup implementation matches; the only hit was unrelated gateway docs text, and `src/infra/backup-exclude.ts` is not present in the checkout.
