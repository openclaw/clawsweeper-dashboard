## Summary

Keep open: the PR targets a real Slack name-allowlisted read gap, but the patch is not merge-ready because it can authorize an explicit target channel using the inbound channel's name, and the external PR has only test output rather than real Slack behavior proof.

## What This Changes

The PR changes Slack read-like action authorization to use channel names for name-based allowlists, adds Slack channel names to threading context, returns names from `conversations.info`, and adds focused Slack tests.

## Best Possible Solution

Keep the target-name fallback, but bind any inbound `currentChannelName` to a matching `currentChannelId`; otherwise resolve the target channel's own name before applying name-based allowlist matching and require redacted real Slack proof before merge.

## Reproduction Assessment

Yes. Source inspection shows an explicit Slack read target can be different from the inbound context channel, while the PR passes the inbound `currentChannelName` into the target authorization check.

## Solution Assessment

No. Lazy target-name resolution is the right general shape for ID-only targets, but the current implementation must only reuse the inbound context name when it is proven to describe the same target channel.
