## Summary

Current `main` already supersedes this PR. `resolveAnnounceTargetFromKey()` no longer uses the narrow numeric-only `:topic:`/`:thread:` regex path; it now delegates to shared session-conversation parsing, and the Feishu parser on `main` accepts alphanumeric topic IDs via `:topic:([^:]+)` with coverage for Feishu topic conversation handling.

## Evidence
