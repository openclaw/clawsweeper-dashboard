## Summary

Keep open: current main still has a source-reproducible yielded media-completion path where active wake rejects a yielded, non-streaming run and the fallback announce run can write the same session without publishing that write to the original fence. Related open issues cover broader `EmbeddedAttemptSessionTakeoverError` symptoms, but none safely supersedes this more specific image-generation reproducer.

## What This Changes

The issue asks OpenClaw to deliver image-generation completion into a yielded requester session without triggering `EmbeddedAttemptSessionTakeoverError`.

## Best Possible Solution

Forward owned transcript-write publication through the prompt-lock wrapper used during prompt submission and add a yielded media-completion regression that preserves external-edit takeover detection.

## Reproduction Assessment

Yes for a source-level reproduction path: the current code can take the reported image-generation completion route through `not_streaming` active wake failure into a new announce run on the same session. I did not run a live provider image-generation repro in this read-only pass.

## Solution Assessment

Yes, a narrow repair appears available: preserve the existing owned-write publication contract instead of adding a new config option or changing channel behavior. A broader yielded-run resume mechanism can be considered later, but it is not required to fix this fence mismatch.
