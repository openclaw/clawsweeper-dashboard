## Summary

PR #71612 is now obsolete on current main. The same cron false-positive is fixed by merged commit a3c51f91 (#72292), which runs default isolated cron executions under per-run session keys so loop-detection diagnostic state no longer accumulates across repeated runs of the same cron job.

## Best Possible Solution

Close #71612 as implemented by the merged #72292 run-scoped cron session-key fix. Keep the current main approach because it scopes all isolated cron context and diagnostic state to the real run boundary instead of adding an idle-gap heuristic only inside loop detection; open a narrower follow-up only if a non-cron isolated caller can still share a persistent diagnostic session key across independent runs.
