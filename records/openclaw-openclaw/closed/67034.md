## Summary

Current `main` already contains shipped mitigations for the reported Telegram multi-account restart avalanche: native approval handlers now become ready after gateway auth and replay pending approvals in the background, replay failures no longer block startup, Telegram polling uses jittered restart backoff, and stale transports are rebuilt/closed on recovery paths. The issue timeline also already points at merged PR #70649 for the loopback-storm side of this report.
