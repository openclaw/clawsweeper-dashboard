## Summary

Keep this draft PR open. Current main does not implement locale path preservation for the Mintlify language switcher: the docs tree only has the existing nav-tabs helper, no `locale-path-preserve.js`, and no `localization-select` click handler. The PR is a plausible docs-site fix, but it still needs maintainer review because it adds sitewide client JavaScript and hard-codes the generated locale prefixes that are owned by the docs publish/i18n pipeline.

## Best Possible Solution

Keep the PR open. The best path is to review and revise the draft so the locale list is derived from, generated with, or clearly synchronized with the docs publish/i18n locale source, then verify the Mintlify preview behavior before merge. A small focused docs-site script is reasonable if maintainers accept the sitewide JS surface.
