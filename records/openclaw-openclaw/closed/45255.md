## Summary

Current main already resolves doctor orphan-transcript checks by sessionId-derived transcript path for every session-store entry, and isolated cron runs persist each run's own transcript path. That matches the requested fix, and the changelog says it shipped in 2026.4.12.

## Evidence

- **Doctor now checks referenced transcripts by resolved session path for every store entry:** The orphan-transcript scan builds `referencedTranscriptPaths` from every `sessions.json` entry with a `sessionId`, using `resolveSessionFilePath(entry.sessionId, entry, sessionPathOpts)`, then only treats files outside that set as orphaned.
  - file: [src/commands/doctor-state-integrity.ts:883](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/src/commands/doctor-state-integrity.ts#L883)
