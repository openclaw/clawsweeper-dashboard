## Summary

Keep open: this PR addresses the right heartbeat model-selection seam, but the current patch can still keep a stale auto fallback override after fallback notice metadata has been cleared.

## What This Changes

This PR adds heartbeat-aware stale auto fallback override clearing, regression coverage, model-failover docs, and a changelog entry.

## Best Possible Solution

Persist durable auto-fallback origin metadata when writing fallback-owned selection, then compare that origin to the current default for heartbeat stale clearing while preserving explicit heartbeat, user, legacy, parent, and unchanged-default overrides.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a heartbeat session with `modelOverrideSource: "auto"` but no `fallbackNoticeSelectedModel`, followed by a default-model change, makes the PR helper return false and keep the stale override.

## Solution Assessment

No. The PR changes the right area, but using fallback notice fields as the origin record is not durable enough because those fields are display lifecycle state and can be cleared while the auto override remains.
