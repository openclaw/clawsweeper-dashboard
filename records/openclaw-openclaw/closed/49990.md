## Summary

Current `main` already applies `channels.discord.proxy` to both Discord startup REST traffic and guild/channel allowlist resolution, matching the requested fix.

## Evidence

- **Guild/channel resolution uses proxied fetch:** `monitorDiscordProvider` builds `discordRestFetch` from `channels.discord.proxy` and passes it into `resolveDiscordAllowlistConfig`; that resolver uses the provided `fetcher` for `resolveDiscordChannelAllowlist`, which is the guild/channel resolve path described in the report.

  - file: [extensions/discord/src/monitor/provider.startup.ts:180](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/extensions/discord/src/monitor/provider.startup.ts#L180)
