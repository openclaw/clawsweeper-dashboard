## What This Changes

The issue requests that the TUI footer context indicator use current context-window usage instead of cumulative totalTokens cost/accounting totals.

## Best Possible Solution

Keep the shipped fresh-session-token accounting path; reopen only if v2026.5.26 or newer still diverges between the TUI footer and /context after a new turn refreshes the session row.

## Reproduction Assessment

Yes for the reported 2026.5.22 behavior: the issue gives concrete footer and /context outputs, and current regression tests cover the accumulated-usage versus last-call-usage shape. There is no failing current-main reproduction path left in source because the session row now persists a fresh context snapshot.

## Solution Assessment

Yes. The current implementation fixes the shared session-token source that feeds both TUI and /context, which is narrower and safer than adding a TUI-only duplicate currentContextTokens field.
