## Summary

Current `main` already supports overriding the base URL for built-in provider catalog models via `models.providers.<id>.baseUrl`, without redeclaring the full built-in model list. The model resolver applies provider config overrides to registry-found models, tests cover the Anthropic override case directly, and docs now describe explicit `models.providers.<id>` entries as the way to override default base URLs for bundled providers.

## Evidence

## Risks / Open Questions

- The issue body also mentioned env-var-based overrides like `ANTHROPIC_BASE_URL`; the evidence here only proves the config-based override path on current `main`.
- This appears to be on unreleased `main` after `v2026.4.23`, so users on released builds may still see the older behavior until a later release ships it.
