## Summary

Current main still treats configured but undiscoverable channel ids as fatal config validation issues, and gateway startup still throws on invalid config snapshots. The same-author PR #53315 is open and is the active implementation candidate, so this issue should stay open until that PR or a replacement lands.

## Best Possible Solution

Keep this issue open while maintainers review PR #53315 or a replacement patch. The likely fix belongs in `src/config/validation.ts` and gateway startup behavior: configured but undiscoverable plugin-owned channels should warn and be skipped while remaining channels still start, with targeted regression coverage and docs updated if the validation contract changes.
