## Summary

Keep open: current main and the latest release still open the dashboard URL reported inside the Docker Compose CLI container unchanged, and an open paired PR already targets this exact bug.

## What This Changes

ClawDock should open the dashboard on the Docker host-published gateway port when `OPENCLAW_GATEWAY_PORT` differs from the container-internal `18789` port.

## Best Possible Solution

Implement the host-published URL translation in `scripts/clawdock/clawdock-helpers.sh`, avoid exposing dashboard tokens through child-process argv, and preserve the related `--no-deps` dashboard helper change.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set `OPENCLAW_GATEWAY_PORT=3200`, start the Docker Compose gateway, source `scripts/clawdock/clawdock-helpers.sh`, then run `clawdock-dashboard`; current main opens the CLI-container URL instead of translating it to the host-published port.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `scripts/clawdock/clawdock-helpers.sh`, translating only the host-browser URL while leaving Docker's internal gateway port, dashboard command, and gateway runtime unchanged.
