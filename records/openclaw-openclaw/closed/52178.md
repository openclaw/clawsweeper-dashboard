## Summary

Current main already handles the reported localhost plain-HTTP Control UI approval path by preserving Control UI operator scopes during insecure localhost auth, and that behavior is present in the released v2026.4.23 code/docs/tests. Remote plain HTTP is now intentionally blocked unless using the documented exceptions, so this PR's client-id fallback is redundant on main.

## Evidence
