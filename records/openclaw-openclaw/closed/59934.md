## Summary

Current `main` already supports Telegram negative group IDs for `config.patch`: Telegram groups are validated as `Record<string, ...>`, the docs explicitly require negative chat IDs under `channels.telegram.groups`, and the current gateway config-mutation guard includes regression coverage using a negative Telegram group key in a `config.patch` payload.

## Evidence

- **Telegram group keys are arbitrary strings:** `TelegramAccountSchema` validates `groups` with `z.record(z.string(), TelegramGroupSchema.optional())`, so negative Telegram chat IDs are accepted as object keys rather than parsed as array indexes.
