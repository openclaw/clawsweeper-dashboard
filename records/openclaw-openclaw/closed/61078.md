Full review comments:

- **[P1] Keep sandbox tool arguments in container form:** `src/agents/pi-tools.read.ts:635`
  - body: `aliased` now contains host-remapped paths, but sandbox-backed read/write/edit tools pass those values to `SandboxFsBridge`, which treats absolute `/workspace/...` paths as container paths first. In layouts such as `root=/workspace/openclaw` with `containerWorkdir=/workspace`, this makes existing container paths resolve under `openclaw/openclaw/...` instead of the intended file, so the guard should validate the remapped path without forwarding that host path to sandbox tools.
  - confidence: 0.89
- **[P2] Avoid remapping host-resolved temp aliases as container paths:** `src/agents/pi-tools.read.ts:374`
  - body: After `mapWorkspaceTempAliasToRoot` resolves `@tmp/...` to a host path under `root/run/tmp`, this call sends that host path back through the container-workdir mapper. When the host root sits under the container prefix, the new alias itself points at `root/openclaw/run/tmp/...` instead of `root/run/tmp/...`; add coverage with a `/workspace/openclaw` root and skip container remapping once the temp alias matched.
  - confidence: 0.87
