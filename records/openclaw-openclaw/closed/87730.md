## Summary

Keep open: current main and the latest release still reject persisted bundled plugin roots outside the single active bundled tree, and two open implementation PRs already reference this issue. The right cleanup is to review and land one focused fix rather than close the issue early.

## What This Changes

The issue asks for `openclaw plugins registry --refresh` to stop leaving a source-checkout persisted plugin registry marked as `persisted-registry-stale-source` when bundled plugin roots are mixed between `dist/extensions` and `extensions`.

## Best Possible Solution

Land one focused plugin-registry fix that accepts same-checkout mixed bundled roots while retaining stale-source protection for different checkouts and missing files.

## Reproduction Assessment

Yes. Source inspection shows current main can persist same-checkout bundled plugins from both `dist/extensions` and `extensions`, then reject the persisted registry because validation accepts only the active bundled root; I did not run the reporter's macOS CLI sequence live.

## Solution Assessment

Yes. The best path is a narrow plugin-registry validator/discovery-alignment fix, not a new flag or disabling the stale-source guard; the linked PRs are plausible implementation candidates to review.
