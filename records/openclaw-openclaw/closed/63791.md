## Summary

Current `main` already contains a targeted fix for dispatcher-backed runtime fetch multipart uploads: it rebuilds `FormData` into the runtime `undici` implementation and removes stale `content-type`/`content-length` headers before calling runtime fetch. Groq STT uses that shared OpenAI-compatible audio path, and there is a regression test covering this exact multipart case.

## Evidence

- **Groq STT uses shared OpenAI-compatible audio upload path:** The Groq media-understanding provider delegates `transcribeAudio` to `transcribeOpenAiCompatibleAudio(...)`, so Groq audio transcription follows the shared multipart upload code path.
