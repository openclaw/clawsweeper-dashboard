## Summary

Current main and the shipped v2026.5.7 release already cover the reported failure class: disabled plugins short-circuit before runtime import, the CLI metadata loader honors disabled state before loading plugin code, and Feishu is now handled as an externalized plugin with its own dependency/setup-entry boundary.

## What This Changes

The issue asks OpenClaw to stop unconfigured bundled Feishu from crashing on a missing `@larksuiteoapi/node-sdk` dependency and to make `openclaw plugins disable feishu` work without loading that dependency.

## Best Possible Solution

Keep the shipped loader/setup-entry/externalized-Feishu behavior; add only a narrow regression test if maintainers want extra coverage for disabled heavy-runtime imports while preserving setup-only channel loading.

## Reproduction Assessment

No high-confidence current-main failure path remains: source inspection shows both the runtime loader and CLI metadata loader stop before importing disabled plugin code. The older-release reproduction in the report is credible, but it no longer matches current main or v2026.5.7.

## Solution Assessment

Yes. The current solution addresses the user-visible failure at the loader and packaging boundaries without moving Feishu runtime dependencies into core or breaking setup-only channel registration.
