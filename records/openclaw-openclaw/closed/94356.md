Full review comments:

- **[P1] Escape the generated extension regex:** `src/agents/agent-tools.params.ts:33`
  - body: `ext.replace(".", "\\")` builds a regex like `/\docodex$/`, where `\d` means a digit. As a result `report.docodex` is not matched at all and the proposed correction never runs for the reported pattern; escape the dot as `\.` or avoid dynamic regex construction here.
  - confidence: 0.98
- **[P2] Cover the read-tool normalization path:** `src/agents/agent-tools.params.ts:254-258`
  - body: This only wires correction into `wrapToolParamValidation`, but read tools use `createOpenClawReadTool`, which currently strips XML suffixes directly and never calls the new correction helper. The linked bug explicitly includes read-style operations, so read calls with `.docodex` would still reach the base tool unchanged after the regex is fixed.
  - confidence: 0.92
- **[P1] Keep Matrix token errors fatal:** `extensions/matrix/src/matrix/monitor/sync-lifecycle.ts:34-35`
  - body: The recoverable-error list includes `m_unknown_token`, but current Matrix code treats 401 `M_UNKNOWN_TOKEN` as an invalidated access token. Marking that recoverable would hide auth failures as `SYNCING` and keep the fatal-stop wait from settling, so this unrelated Matrix change should be removed or kept narrowly fatal for token errors.
  - confidence: 0.9
