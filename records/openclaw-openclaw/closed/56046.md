## Summary

Current main already wires Signal proactive sends through the channel outbound adapter, so the reported `message action=send` failure path is implemented/fixed in code that is also present in the shipped `v2026.4.22` release.

## Evidence

- **Signal plugin defines proactive outbound send handlers:** `signalPlugin` includes an `outbound` block with `attachedResults.sendText` and `attachedResults.sendMedia`, which provide the send handlers the message tool path expects.
  - file: [extensions/signal/src/channel.ts:361](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/extensions/signal/src/channel.ts#L361)
  - command: `nl -ba extensions/signal/src/channel.ts | sed -n '320,430p'`
  - sha: [7130e56f872a](https://github.com/openclaw/openclaw/commit/7130e56f872aea96577ac9fcbda0c7555a9ecc69)
- **Chat channel builder materializes attached outbound handlers into `plugin.outbound`:** `createChatChannelPlugin` calls `resolveChatChannelOutbound`, which merges `outbound.base` with inline adapters created from `attachedResults`, producing `sendText`/`sendMedia` on `plugin.outbound`.
  - file: [src/plugin-sdk/core.ts:711](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/plugin-sdk/core.ts#L711)
  - command: `nl -ba src/plugin-sdk/core.ts | sed -n '560,750p'`

- **Proactive send path loads `plugin.outbound` and only errors if `sendText` is absent:** `loadChannelOutboundAdapter` returns `entry.plugin.outbound`, and `createChannelHandler` throws `Outbound not configured for channel` only when the loaded adapter lacks `sendText`. With the current Signal plugin shape, that condition is satisfied.
