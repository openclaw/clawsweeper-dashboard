## Summary

Current `main` already mitigates the reported failure mode: session stores are now bounded by age/count/disk-budget maintenance, startup recovery is limited to orphaned subagent sessions instead of all `sessions.json` bindings, and those recovery resumes use the `subagent` lane with a 10s gateway call timeout rather than contending on `lane=main` indefinitely.

## Evidence

- **Session store maintenance exists for `sessions.json`:** `session.maintenance` now resolves retention/count/rotation/disk-budget controls, including `pruneAfter` and `maxEntries`, so stale bindings no longer accumulate indefinitely when enforcement is enabled.
  - file: [src/config/sessions/store-maintenance.ts:135](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/config/sessions/store-maintenance.ts#L135)
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Oversized stores are pruned on load:** Loading a session store applies maintenance when the store exceeds the configured max entries, pruning stale entries and capping the store before use.
  - file: [src/config/sessions/store-load.ts:131](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/config/sessions/store-load.ts#L131)
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Maintenance also runs on save:** Session-store writes prune stale entries, cap entry count, archive removed transcripts, and enforce disk-budget cleanup before persisting.
  - file: [src/config/sessions/store.ts:287](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/config/sessions/store.ts#L287)
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Startup recovery is scoped, not blanket resume:** Gateway startup schedules only subagent orphan recovery, and recovery resumes only sessions that are still active in the subagent registry and marked `abortedLastRun`. The recovery call uses `lane: "subagent"` and `timeoutMs: 10_000`.
  - file: [src/agents/subagent-orphan-recovery.ts:104](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/agents/subagent-orphan-recovery.ts#L104)
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Session-scoped embedded lanes replaced shared main-lane queuing:** Embedded reply runs resolve to `session:<key>` lanes, and reply execution uses that per-session lane key rather than sending all work through a shared `main` lane queue.
  - file: [src/agents/pi-embedded-runner/lanes.ts:3](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/agents/pi-embedded-runner/lanes.ts#L3)
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
