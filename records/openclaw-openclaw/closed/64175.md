## Summary

Keep open. Current main still drops OpenRouter's `supported_parameters` tool-support signal for dynamic models, so a normal agent run can still treat `openrouter/perplexity/sonar-deep-research` as tool-capable; the one-shot model-run path is now no-tools, but that does not solve the reported agent/runtime path.

## What This Changes

Request to handle OpenRouter models that do not advertise tool support without sending OpenClaw tool payloads or falling back unclearly.

## Best Possible Solution

Preserve OpenRouter `supported_parameters.includes("tools")` in runtime capabilities and map known toolless dynamic models to `compat.supportsTools: false`, with focused regression coverage and a changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level path is clear: live OpenRouter metadata for `perplexity/sonar-deep-research` lacks `tools`, current dynamic OpenRouter resolution drops that signal, and normal agent runs enable tools unless `compat.supportsTools` is explicitly false.

## Solution Assessment

Yes. The narrowest maintainable fix is to feed OpenRouter metadata into the existing `compat.supportsTools` and `supportsModelTools()` gate; adding a new non-tool agent mode would be broader than needed for this bug.

## Repair Work Prompt

Fix OpenRouter dynamic model tool capability propagation for this issue. Current main sees `perplexity/sonar-deep-research` in the live OpenRouter catalog without `tools` in `supported_parameters`, but `src/agents/pi-embedded-runner/openrouter-model-capabilities.ts` drops that signal and `extensions/openrouter/index.ts` builds a dynamic model without `compat.supportsTools`, so normal agent turns treat it as tool-capable. Update the OpenRouter capability cache/type/parser to preserve whether tool support is known, set `compat.supportsTools: false` on dynamic OpenRouter models when metadata explicitly lacks `tools`, and preserve existing unknown/absent fallback behavior when metadata is unavailable. Add focused regression tests for capability parsing, dynamic model resolution, and existing tool gating; do not add a new non-tool agent mode or broad fallback-policy change. Add a single-line `CHANGELOG.md` fix entry because this is user-facing.
