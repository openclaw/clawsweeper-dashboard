## Summary

Keep open: current main still has a source-visible chat-continuation path where prompt additions split away from `transcriptPrompt` are appended into system-prompt space, and the linked closing PR #77521 is still open rather than merged.

## What This Changes

Restore prompt-cache stability by keeping `before_prompt_build` `prependContext` and volatile chat metadata out of the cache-sensitive system prompt on chat continuations.

## Best Possible Solution

Preserve the documented hook contract by delivering `prependContext` and volatile per-turn metadata as current-turn user/runtime context while keeping only stable prompt text or explicit system-context hook fields in system-prompt space.

## Reproduction Assessment

Yes. Source inspection on current main shows the chat-continuation path: auto-reply provides a distinct `transcriptPrompt`, prompt additions become `runtimeContext`, and the embedded runner appends that runtime context to the system prompt before submission.

## Solution Assessment

Yes, the requested direction is the narrow maintainable fix. The linked PR #77521 appears to target the right boundary, but this issue should remain open until that PR or an equivalent fix is merged and validated.
