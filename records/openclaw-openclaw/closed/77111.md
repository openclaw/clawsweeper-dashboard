## Summary

The PR fixes the core credential-in-URL path, but it should not merge as-is because the new blob-link document flow can remove the gateway response headers that currently protect rendered assistant-media documents.

## What This Changes

The PR removes assistant-media `token` query authentication, switches Control UI local attachment requests to bearer-authenticated fetches, adds blob URL rendering for fetched attachments, and updates tests, docs, and the changelog.

## Best Possible Solution

Keep the bearer-authenticated fetch direction, but handle non-preview document/open-in-new-tab flows through a path that preserves security headers or forces a safe download before merging.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path: local assistant attachments are rendered as assistant-media URLs with `&token=...`, and the gateway accepts that query token on the media route.

## Solution Assessment

No. The header-auth direction is right, but replacing document links with same-origin blob URLs is not the safest complete fix until active document content is sandboxed, forced to download, or served through a route that keeps the gateway CSP headers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P1] Keep document blobs from opening without gateway headers:** `ui/src/ui/chat/grouped-render.ts:1394-1395`
  - body: The PR turns local document attachments into `blob:` links opened in a new tab. The fetched assistant-media response is currently protected by the gateway's CSP/nosniff headers, but the generated blob URL does not carry those response headers; for document attachments such as `.html`, clicking the link can render active content under the Control UI origin. Keep document downloads on a response that preserves the gateway headers, force a non-rendering download, or sandbox this path before replacing document hrefs with blob URLs.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The patch removes the credential-in-query leak, but it introduces a security-sensitive blob document opening path that needs repair before merge.

Concerns:

- **[high] Blob document links bypass route response headers:** `ui/src/ui/chat/grouped-render.ts:1394`
  - body: Local document attachments now use generated blob URLs as `target=_blank` links. That avoids exposing bearer credentials, but it also means rendered document content no longer comes from the assistant-media route response that sets CSP and nosniff headers, which is risky for active document types such as HTML.
  - confidence: 0.78
