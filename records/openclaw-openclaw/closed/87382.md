## Summary

Current main fixes the repeated already-paired macOS node approval prompt by reconciling paired-node reconnects before creating new pairing requests and clearing stale same-node pending reapproval after a successful hello; the fix is merged and shipped in v2026.6.8-beta.1.

## What This Changes

The issue asks OpenClaw to stop showing repeated macOS node-pair approval prompts when an already-paired node relaunches with the same approval surface.

## Best Possible Solution

Keep the current gateway-side reconnect cleanup and reapproval diagnostics behavior; reopen only with a repro against v2026.6.8-beta.1 or later showing same-surface prompts still appear.

## Reproduction Assessment

No current-main failure path remains. The historical 2026.5.26 and 2026.5.28-beta reports reproduced the prompt loop, and current source plus regression coverage now show same-surface paired-node reconnects clear stale pending prompts instead of creating another approval.

## Solution Assessment

Yes. Fixing this at the gateway reconnect and cleanup boundary is the best shape because it suppresses stale same-surface prompts without auto-approving expanded commands, capabilities, or permissions.
