The PR adds `BrowserStatus.pageReady` for Chrome MCP existing-session profiles, computes it with an ephemeral page-level reachability probe, threads the option through browser availability actions, and updates status tests.

## Best Possible Solution

Keep the additive `pageReady` contract and ephemeral probe, but make the route and public status/doctor timeout budgets share a tested contract that reliably returns a status response.
