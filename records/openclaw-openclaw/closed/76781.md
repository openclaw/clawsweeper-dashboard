## Summary

Keep open; current main still has the partial registry reuse bug, and this PR is a focused fix with no blocking review findings from the latest patch.

## What This Changes

The PR changes plugin tool registry reuse to require full selected plugin-id tool coverage, adds an ephemeral forced cold-load path for partial registries, and covers it with regression tests plus a changelog entry.

## Best Possible Solution

Land this PR after normal CI because it fixes the core registry reuse semantics without adding plugin-specific policy.

## Reproduction Assessment

Yes, by source inspection. Current main accepts a registry when any selected plugin has tools, so a registry with plugin A's tools can prevent plugin B from being cold-loaded for the same selected tool scope.

## Solution Assessment

Yes. The patch narrows registry reuse to full selected tool-owner coverage and uses an ephemeral cold load only when an existing registry is incomplete, which is the smallest maintainable fix for the reported behavior.
