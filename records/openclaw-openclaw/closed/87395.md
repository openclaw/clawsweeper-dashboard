## Summary

Current main already addresses the reported v2026.5.26 native hook relay generation-staleness failure: Codex app-server resumes now reuse the stored relay generation, legacy pre-generation bindings get a bounded bootstrap grace window, and regression tests cover both acceptance and stale-rejection paths. The fix is not in v2026.5.26, so this is a current-main-only close until the next release ships it.

## What This Changes

The issue asks maintainers to confirm and diagnose intermittent `Native hook relay unavailable` failures on macOS after v2026.5.26 that block shell, filesystem, memory, and `gh` tool access.

## Best Possible Solution

Keep the current-main generation persistence and bounded bootstrap grace fix, then ship it in the next release so v2026.5.26 users no longer need a hard restart workaround.

## Reproduction Assessment

Yes. The v2026.5.26 source path unconditionally rejects a stale required generation after relay replacement, and current-main tests encode the resume/restart paths; I did not run a live macOS reproduction.

## Solution Assessment

Yes. The current-main fix keeps the relay generation stable for true resumed Codex threads, adds a bounded compatibility window for pre-generation bindings, and still rejects stale generations when a fresh thread is started.
