## Summary

PR #69050 is redundant on current main. The exact `build-all` one-liner was not merged, but the user-visible failure it targeted is already covered: gateway startup now rebuilds missing Control UI assets before serving, update/onboarding paths explicitly run `ui:build`, and the release/local-dist contract still documents `pnpm build && pnpm ui:build` rather than folding UI bundling into every build-all profile.

## Best Possible Solution

Close this PR rather than merge the build-all change. Keep the current main behavior: build/pack/release paths require an explicit Control UI bundle, while gateway startup, setup, and update flows repair missing `dist/control-ui` assets so the original 503 symptom is covered without expanding every build-all profile.
