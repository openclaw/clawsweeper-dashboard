## Summary

Current main already routes Telegram turns through the same skills snapshot and CLI prompt/env injection path used by agent runs, so the reported Dashboard-vs-Telegram skill availability mismatch is implemented on main. The bundled Notion skill is present, declares `NOTION_API_KEY`, and release `v2026.4.24` contains the relevant snapshot/injection code.

## Best Possible Solution

Close this as implemented on main and keep the shared channel skill-snapshot path. If a user still sees Notion unavailable on a current release, the best follow-up is a new focused issue with the current OpenClaw version, `openclaw skills list --verbose`, relevant `agents.*.skills` config, Telegram account binding, and redacted gateway logs.
