## Summary

Current main already implements configurable provider-scoped LLM request timeouts as `models.providers.<id>.timeoutSeconds`. The key name is seconds-based rather than the proposed `timeoutMs`, but it solves the reported slow-provider/GLM timeout problem by threading the provider config into model request metadata, guarded fetch, OpenAI SDK transport options, and the model idle watchdog. The implementation is present in the shipped `v2026.4.26` tag.

## Best Possible Solution

Keep the shipped `models.providers.<id>.timeoutSeconds` implementation and documentation. It matches the maintainer-friendly framing from the discussion: a provider-scoped request/idle timeout knob for slow local, self-hosted, or OpenAI-compatible providers without increasing the global agent runtime timeout.
