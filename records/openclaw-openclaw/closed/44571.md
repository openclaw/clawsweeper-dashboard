## Summary

Current main solves the reported multi-agent missing-auth case through runtime read-through from the default/main auth store, and the fix shipped in v2026.5.7. The remaining stale-local-profile/key-rotation UX is already tracked separately by https://github.com/openclaw/openclaw/issues/47684.

## What This Changes

The issue asks onboarding to make credentials available to every configured agent, either by writing each agent auth store or by letting non-default agents inherit the main auth store.

## Best Possible Solution

Keep the shipped read-through inheritance model, and handle stale-profile rotation or bulk-sync UX in the narrower follow-up rather than reopening this onboarding distribution report.

## Reproduction Assessment

No failing current-main reproduction remains from source inspection: non-main agents without a local profile now merge the main auth store through `ensureAuthProfileStore*` and runtime profile loading. I did not run a live onboarding flow because this review had to keep the checkout read-only.

## Solution Assessment

Yes. The implemented read-through path is the safer maintainable fix for missing non-default agent auth because it avoids cloning secret material into every agent while still making main credentials available when no local override exists.
