## Summary

Keep this issue open. Current main still has plausible Windows console-popup paths in bundled plugin runtime dependency staging: Telegram stages grammY dependencies, Anthropic is default-enabled and staged, and the npm child-process calls in the relevant runtime/postinstall/staging paths still do not pass `windowsHide`.

## Best Possible Solution

Keep this open and patch the bundled plugin runtime dependency npm staging entrypoints. Add `windowsHide: true` to the npm `spawn`/`spawnSync` options in `src/plugins/bundled-runtime-deps.ts`, `scripts/postinstall-bundled-plugins.mjs`, and `scripts/stage-bundled-plugin-runtime-deps.mjs`; add focused Windows-option regression assertions; then verify on Windows 11 with Telegram configured and a provider route matching the report.
