## Summary

The reported colon-in-description failure does not match current main. OpenClaw's frontmatter parser preserves inline values containing colons during skill load, that behavior is covered by a unit test, and the skill-creator packaging path now fails validation on invalid YAML instead of packaging silently.

## Evidence

- **Runtime parser preserves colon-containing descriptions:** `parseFrontmatterBlock` falls back to line parsing when YAML parsing fails and also prefers the inline value when YAML would coerce a colon-containing scalar into a structured value, so `description: Use ...: ...` is preserved as a plain string.
  - file: [src/markdown/frontmatter.ts:201](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/markdown/frontmatter.ts#L201)

  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Unit coverage for colon case:** There is an explicit test asserting that an inline `description` containing a colon is preserved unchanged.
  - file: [src/markdown/frontmatter.test.ts:71](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/markdown/frontmatter.test.ts#L71)
  - command: `nl -ba src/markdown/frontmatter.test.ts | sed -n '68,78p'`
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Skill loading uses that parser:** Workspace skill loading reads `SKILL.md`, calls `parseFrontmatter`, and only drops the skill if `name` or `description` are missing after parsing.
  - file: [src/agents/skills/local-loader.ts:60](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/agents/skills/local-loader.ts#L60)
  - command: `nl -ba src/agents/skills/local-loader.ts | sed -n '60,71p'`
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Packaging no longer silently succeeds on invalid YAML:** `package_skill.py` runs validation before packaging, and `quick_validate.py` returns an explicit `Invalid YAML in frontmatter` error on YAML parser failures.
  - file: [skills/skill-creator/scripts/package_skill.py:56](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/skills/skill-creator/scripts/package_skill.py#L56)

  - file: [skills/skill-creator/scripts/quick_validate.py:83](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/skills/skill-creator/scripts/quick_validate.py#L83)
