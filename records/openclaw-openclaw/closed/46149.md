## Summary

Current `main` already has a broader shared assistant-text sanitizer that strips raw tool XML from user-visible replies across delivery and history paths, with regression coverage. This PR's one-file patch is obsolete against the current implementation.

## Evidence

- **Shared sanitizer strips raw tool XML:** `stripToolCallXmlTags` removes leaked `<tool_call>`, `<tool_result>`, `<function_call(s)>`, `<tool_calls>`, truncated tool-call streams, and preserves code regions; the visible-text pipeline applies it for user-facing assistant text.
  - file: [src/shared/text/assistant-visible-text.ts:199](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/shared/text/assistant-visible-text.ts#L199)
- **Delivery path uses sanitized visible assistant text:** Assistant text extraction for outbound replies runs through `sanitizeAssistantVisibleText`, then `sanitizeUserFacingText`, so successful reply payloads are built from already-sanitized visible text.
  - file: [src/agents/pi-embedded-utils.ts:33](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/agents/pi-embedded-utils.ts#L33)
- **Reply payload fallback uses visible-text extraction:** Embedded runner reply assembly falls back to `extractAssistantVisibleText(lastAssistant)` when building final answer text for delivery.
  - file: [src/agents/pi-embedded-runner/run/payloads.ts:233](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/agents/pi-embedded-runner/run/payloads.ts#L233)
