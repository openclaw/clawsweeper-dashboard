## Summary

Keep open because current main still initializes the Control UI Sessions tab with an empty active-window filter and limit 120, so the first `sessions.list` request omits `activeMinutes`; the linked fix PR #76051 is open and unmerged with closing syntax for this issue.

## What This Changes

The issue asks for the Control UI Sessions tab to default to a bounded recent-session query with a smaller limit.

## Best Possible Solution

Review and land #76051 or an equivalent small UI-only default change, keeping the existing Gateway API and editable filters intact.

## Reproduction Assessment

Yes for the source-level query shape: current main initializes an empty active-window filter and constructs the initial `sessions.list` call without `activeMinutes`. A full latency reproduction still requires a long-running gateway with substantial session history.

## Solution Assessment

Yes, the proposed direction is the narrowest maintainable fix because the Gateway already supports `activeMinutes` and `limit`; changing Control UI defaults avoids broader API or storage changes.
