## Summary

Keep open: current main and the latest release still select the last outbound heartbeat payload without excluding reasoning, and the open linked PR is the active fix path.

## What This Changes

The issue asks OpenClaw to stop heartbeat delivery from selecting reasoning payloads as the main visible reply when `includeReasoning` is false.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/92356 as the canonical fix; after it merges, this issue can close through the linked PR.

## Reproduction Assessment

Yes, at source level. A reply result shaped as final `HEARTBEAT_OK` followed by `{ text, isReasoning: true }` makes current `resolveHeartbeatReplyPayload` return the reasoning payload because it scans backward and checks only outbound content.

## Solution Assessment

Yes. The narrow maintainable fix is to change heartbeat main-reply selection so SDK-classified reasoning is not selected as the visible reply while preserving the separate `includeReasoning` Thinking path.
