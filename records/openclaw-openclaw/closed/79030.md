## Summary

Keep open: current main still has a source-provable provider transport path where a private model endpoint can be upgraded to trusted env-proxy mode because the OpenClaw NO_PROXY matcher does not understand CIDR or IPv4 octet-wildcard entries.

## What This Changes

Private model-provider requests with `allowPrivateNetwork` should not be routed through env proxy when `NO_PROXY` covers the target only by CIDR or IP-wildcard entries.

## Best Possible Solution

Extend OpenClaw's target-aware env-proxy bypass decision for provider fetches to handle CIDR and narrowly defined IP wildcards, with regression coverage for CGNAT/Tailscale endpoints and no managed-proxy policy change.

## Reproduction Assessment

Yes for source-level reproduction: current main selects trusted env-proxy mode for provider requests when `shouldUseEnvHttpProxyForUrl` returns true, and the matcher cannot match `100.64.*` or `100.64.0.0/10` to `100.64.0.3`. I did not run a live proxy capture in this read-only review.

## Solution Assessment

Yes. The narrowest maintainable fix is to extend OpenClaw's NO_PROXY decision before selecting trusted env-proxy mode; defaulting all `allowPrivateNetwork` providers to direct transport would be a broader product/security change.

## Repair Work Prompt

Fix the private model-provider env-proxy bypass regression reported at https://github.com/openclaw/openclaw/issues/79030. Current main lets `buildGuardedModelFetch` select trusted env-proxy mode when `shouldUseEnvHttpProxyForUrl(url)` is true, but `matchesNoProxy` only mirrors Undici exact/suffix/port matching and does not match `NO_PROXY=100.64.0.0/10` or `100.64.*` for `http://100.64.0.3:8990`. Implement a narrow OpenClaw-owned bypass decision for provider/env-proxy selection that supports CIDR entries and carefully scoped IP-literal wildcards for the target host. Prefer reusing `src/shared/net/ip.ts` helpers such as `isIpInCidr`; do not broaden hostname wildcard behavior and do not change managed proxy behavior that intentionally clears NO_PROXY. Add regression coverage for `matchesNoProxy`/`shouldUseEnvHttpProxyForUrl` with `100.64.0.0/10`, RFC1918 CIDRs, exact host, host:port, and nonmatching CIDRs, plus provider transport coverage showing no env-proxy mode for a CIDR-covered private provider target and preserving explicit dispatcher policies. Add a user-facing changelog fix entry crediting the external reporter if policy allows; do not add a new config option or default all `allowPrivateNetwork` providers to direct transport without maintainer approval.
