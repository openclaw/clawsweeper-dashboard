## Summary

Current main now implements the missing `fileId` passthrough in the shared message tool schema and has a focused regression test for Slack `download-file`. The fix is only proven on main, not in the latest shipped `v2026.4.27` release.

## What This Changes

Expose the Slack `download-file` `fileId` argument through the message tool schema so agents can pass Slack file IDs to the handler.

## Best Possible Solution

Keep the current main implementation that exposes `fileId` as a backward-compatible optional message-tool parameter, reconcile the now-superseded focused PRs separately, and let any additional model-facing Slack parameter guidance proceed as a follow-up rather than blocking this cleanup.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: `v2026.4.27` lacks `fileId` in the message tool schema while Slack dispatch/runtime require it, and current main adds `fileId` plus a Slack schema test.

## Solution Assessment

Yes. Adding an optional `fileId` schema field is the narrowest maintainable fix for the pass-through failure; Slack-owned descriptions can improve model behavior later but are not required to solve this report.
