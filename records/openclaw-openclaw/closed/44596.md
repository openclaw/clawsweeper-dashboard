## Summary

Current main already fixes the reported stale `lastDeliveryStatus: delivered` behavior for isolated cron announce delivery. Best-effort delivery failures now mark the run as not delivered instead of silently succeeding, cron state persists `not-delivered`, and isolated cron delivery resolution now falls back to a bound account when session context lacks `lastAccountId`.

## Evidence
