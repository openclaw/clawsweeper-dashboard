## Summary

Keep open: the fix is aimed at the right source-level crash path, but the current patch can bypass the existing zero-token empty-stop replay repair after normalizing malformed null content.

## What This Changes

This PR normalizes malformed assistant replay and transport content object/null shapes into arrays and adds focused regression tests.

## Best Possible Solution

Merge the malformed-content normalization only after the replay normalizer applies the existing zero-token empty-stop sentinel repair to the normalized message and adds the missing regression case.

## Reproduction Assessment

Yes, source-level: current main can pass object/null assistant content through replay sanitization to transport conversion, where assistant content is iterated. The linked Windows Feishu/GptGod report provides matching `v.content is not iterable` logs, but I did not run a live provider reproduction.

## Solution Assessment

No, not as currently patched. Normalizing malformed content before replay/transport is the right direction, but the patch should use the normalized assistant message for the existing zero-token empty-stop repair and cover that case in tests.
