## Summary

Keep open. Current `main` still does not implement a configurable Brave Search `baseUrl`: the Brave provider runtime hardcodes Brave endpoints, the Brave plugin schema exposes only `apiKey` and `mode`, and docs show provider-specific Brave settings now live under the Brave plugin config rather than the old `tools.web.search.brave.*` path this PR edits. The PR discussion was useful: the follow-up commit `e66095242a563c3501c80f9611104518b7956164` addresses the cache-key concern raised in review, but that work is not present on `main` and the branch now targets stale integration points.
