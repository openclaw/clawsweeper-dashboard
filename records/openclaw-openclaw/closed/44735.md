## What This Changes

The PR adds a plugin auto-enable guard and regression test so an enabled openclaw-lark entry prevents doctor from auto-enabling the built-in feishu plugin for channels.feishu.

## Best Possible Solution

Keep the shipped effective channel-owner repair behavior; any future auto-enable change should use that same effective ownership contract in the split auto-enable modules, not this raw entries.enabled guard.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path for the PR defect: a restrictive plugins.allow config can make openclaw-lark configured but not effectively active, while the PR guard would still skip Feishu auto-enable.

## Solution Assessment

No. The PR direction is useful, but the submitted implementation is not the best fix; current main's effective channel-owner/preferOver approach is narrower and already shipped for the doctor repair paths.
