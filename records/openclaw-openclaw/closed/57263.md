## Summary

Current `main` already contains explicit guardrails and parity coverage for this failure family: short approval messages are converted into act-now retry instructions, repeated plan-only turns are retried and then surfaced as a blocked error instead of looping, and QA scenarios now require real tool followthrough for approvals, subagent delegation, and repo-contract write tasks.

## Evidence
