## Summary

Close as ClawHub/plugin-scoped: current core does not define a `tier-*` model-alias contract, and optional tier-routing policy can be handled by existing model aliases or a plugin hook without adding a new core config surface.

## What This Changes

The issue requests that `tier-*` model aliases in agent configuration resolve to concrete LiteLLM model names instead of being sent as literal model ids.

## Best Possible Solution

Keep core model selection on explicit `provider/model` refs plus configured aliases, and route optional task/tier routing to a ClawHub plugin unless a concrete missing plugin API is identified.

## Reproduction Assessment

Yes, at source level: current main does not load a `tier-*` map, and unconfigured bare model strings are treated as aliases only when `agents.defaults.models` defines that alias. I did not run a packaged v2026.6.8 live LiteLLM setup because this review is read-only.

## Solution Assessment

Yes for cleanup: the maintainable end state is not a new core `tier-*` contract, because current aliases cover simple shortcuts and `before_model_resolve` lets plugins own richer tier-routing policy.
