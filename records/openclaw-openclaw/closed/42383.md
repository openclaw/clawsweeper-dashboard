## Summary

Issue #42383 targets the old Chrome extension relay, but current `main` no longer has that relay path. OpenClaw migrated host-local signed-in browser control to Chrome MCP `existing-session`, and docs now direct network/response-body use cases to managed browser or raw CDP profiles instead. The report is therefore obsolete against current `main`, not an open cleanup item for the current browser stack.

## Close Comment

Closing after Codex review because this request targets the legacy Chrome extension relay, and that path no longer exists on current `main`. The relay hook is now an intentional no-op in `extensions/browser/src/browser/server-lifecycle.ts:9`, and the migration docs explicitly say the old `driver: "extension"` path was removed and replaced with Chrome MCP `existing-session` (`docs/gateway/doctor.md:207`). The removal is also recorded in `CHANGELOG.md:2191` under release `2026.3.22`. For current browser automation, advanced network-style features such as `responsebody` are documented as requiring a managed browser or raw CDP profile rather than `existing-session` (`docs/help/faq.md:432`, `docs/gateway/configuration-reference.md:204`). Since the reported target no longer matches the current product surface, this issue is obsolete. If there is still a concrete gap on the current `existing-session` or raw-CDP flows, please open a new issue with exact current commands, profile type, expected behavior, and actual behavior.
