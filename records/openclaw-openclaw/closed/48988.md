## Summary

Current `main` already addresses both parts of #48988: Z.ai 1113/1311 are no longer collapsed into rate-limit errors, and fallback cooldown state is resolved from provider-filtered auth profiles with model-scoped rate-limit metadata rather than a single global bucket.
