## Summary

Current main already resolves the reported missing QR runtime dependency: `qrcode` is now a root runtime dependency, the lockfile includes `qrcode@1.5.4`, and the install-smoke workflow has a QR import smoke. The fix is present on `main` only; no release tag contains the proof commit yet, so users on `v2026.4.29` remain affected until the next package release.

## What This Changes

Add or ship the `qrcode` runtime dependency so WhatsApp terminal and Control UI QR rendering can resolve the dynamic QR renderer on fresh installs.

## Best Possible Solution

Keep the current-main dependency and QR import smoke, then let the normal release train ship it in the next OpenClaw package.

## Reproduction Assessment

Yes. The report has a high-confidence source-level reproduction path: WhatsApp QR rendering calls the shared helper that dynamically imports `qrcode`, and older installs without that package fail before rendering; current main resolves that path through `package.json` and `pnpm-lock.yaml`.

## Solution Assessment

Yes. Adding `qrcode` as a root runtime dependency is the narrowest maintainable fix because both terminal and PNG QR renderers already use that package, and the Docker QR import smoke now covers package-resolution regressions.
