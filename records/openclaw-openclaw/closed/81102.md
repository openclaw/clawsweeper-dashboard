## Summary

Close as implemented on current main: the supported `openclaw update` path now derives bundled-to-external relocation bridges from the old persisted plugin registry, syncs plugins before restart, and has regression coverage for default-enabled bundled plugins without explicit config.

## What This Changes

Request an operator-facing OpenClaw upgrade flow that identifies and installs newly externalized bundled plugins without requiring a destructive manual global npm upgrade first.

## Best Possible Solution

Use current `openclaw update` as the supported migration-aware upgrade path; keep raw manual package-manager upgrades documented as an alternative that cannot coordinate plugin migration.
