## What This Changes

The issue asks OpenClaw to add bounded backoff, enough retry budget, and clearer terminal errors when Codex app-server startup closes while a replacement server is warming up.

## Best Possible Solution

Keep this open until a focused Codex app-server startup recovery fix lands, either by repairing https://github.com/openclaw/openclaw/pull/83639 or by landing a narrower patch that adds bounded backoff, adequate retry budget, exhaustion-specific error text, and focused regression coverage.

## Solution Assessment
