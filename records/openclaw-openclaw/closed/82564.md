## Summary

Keep open: this is a plausible CI bug fix with supplied real-behavior proof, but the latest diff still executes an unverified downloaded OpenGrep release asset in a security-labeled workflow.

## What This Changes

The PR adds a repo-owned OpenGrep installer, switches the PR-diff and full OpenGrep workflows to it, and updates local missing-tool guidance.

## Best Possible Solution

Keep the repo-owned installer direction, but verify the pinned v1.19.0 asset before execution and land it through security-owner review.

## Reproduction Assessment

Yes. Current main runs the upstream installer with v1.19.0, and the upstream script validates that version through an echo-to-grep pipeline under pipefail; the related failing PR context shows the install step failing before any scan ran.

## Solution Assessment

No as submitted. Replacing curl-pipe-bash with a repo-owned installer is a reasonable fix shape, but the installer should pin SHA-256 digests or preserve cosign verification before executing the release asset.
