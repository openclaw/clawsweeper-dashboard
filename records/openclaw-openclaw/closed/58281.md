## Summary

Close as superseded: current main still needs the path-resolution fix, but this PR duplicates open #53718 while adding an unsafe missing-variable fallback that #53718 already avoids.

## What This Changes

The PR adds environment-variable placeholder expansion to shared home-relative path helpers and extends `src/infra/home-dir.test.ts` coverage for `$VAR` and `${VAR}` paths.

## Best Possible Solution

Close this PR as superseded and keep #53718, or a maintainer-picked replacement, as the canonical fix path with unresolved placeholders preserved and a changelog before merge.

## Reproduction Assessment

Yes. On current main, `resolveHomeRelativePath("${XDG_CONFIG_HOME}/workspace", { env: { XDG_CONFIG_HOME: "/home/node/.openclaw" } })` follows the non-tilde branch and resolves a literal placeholder path; the PR regression is reproduced with `${MISSING}/workspace`.

## Solution Assessment

No. The PR targets the correct shared helper, but replacing unresolved variables with an empty string can redirect configured paths to root; #53718 already tracks the same issue with safer unknown-placeholder preservation.

## Review Findings

Overall correctness: patch is incorrect
