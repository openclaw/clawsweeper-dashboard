## Summary

Keep open. Current `main` does not contain the proposed isolated `scripts/ops` upgrade-proof tooling, and the PR discussion surfaced a substantive timeout-contract problem against the current `openclaw update --timeout` API. There is adjacent upgrade validation on `main`, but only in the broader cross-OS release-checks lane, not as this repo-owned targeted proof script.
