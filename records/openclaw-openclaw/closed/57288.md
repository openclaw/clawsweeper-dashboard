## Summary

Close this issue as duplicate/superseded by the focused open PR #46881. Current main still does not implement WhatsApp inbound reaction forwarding, but #46881 already tracks the same remaining work by adding a `messages.reaction` handler and reaction fields on the inbound message contract; #48570 is a broader overlapping PR.

## Best Possible Solution

Use #46881 as the focused implementation candidate, or explicitly supersede it with #48570 if maintainers want the broader WhatsApp event work. The shipped fix should live in the WhatsApp plugin owner module, add a `messages.reaction` inbound path, extend the inbound contract deliberately, and include targeted tests for group/DM access control, empty reaction removals, dedupe, status/broadcast filtering, and cleanup.
