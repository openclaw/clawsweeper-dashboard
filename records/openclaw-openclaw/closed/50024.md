## Summary

Current main already fixes the user-visible routing bug targeted by PR #50024 via the merged #58013 route-preservation implementation. The shipped fix keeps inter-session `sessions_send` traffic from overwriting established external `lastChannel`/`lastTo` routes, with source, tests, and changelog evidence in current main.

## Best Possible Solution

Close PR #50024 as implemented on main and keep the shipped #58013 resolver-based route-preservation fix. If maintainers still want source-channel threading in `sessions_send`, track it as a separate cleanup/API semantics discussion with tests around nested agent RPC channel behavior and A2A follow-up steps.
