## Summary

Current main already addresses the reported v2026.5.2 failure: npm plugin installs now preserve existing managed npm-root dependencies and merge persisted install records instead of replacing the other installed plugin's state. The fix is present only on main in this checkout; no release tag contains the proof commit yet.

## What This Changes

The issue reports that installing one externalized npm channel plugin, such as Feishu or QQ Bot, removes the other plugin's npm package and config entry on v2026.5.2.

## Best Possible Solution

Keep the current managed npm-root and install-record merge behavior, and ship it in the next release so v2026.5.2 users can update out of the conflict.

## Reproduction Assessment

No high-confidence current-main failing reproduction was run in this read-only review. The reported v2026.5.2 behavior is source-explained by the old npm plugin install/repair path, and current main now has code and regression coverage for preserving sibling npm plugin dependencies and install records.

## Solution Assessment

Yes. The current fix is the narrow maintainable path: preserve the npm root dependency manifest and merge persisted install records, without adding a new user-facing option or channel-specific special case.
