## Summary

Close as `clawhub`: current main has already shipped the core `message_sending` hook coverage this PR needed for Discord and inbound auto-replies, while the remaining `outbound-whitelist` capability is an optional code plugin that fits ClawHub/npm distribution rather than a new bundled core plugin.

## Best Possible Solution

Keep the shipped core `message_sending` hook coverage and route the optional outbound-recipient allowlist as a ClawHub/npm code plugin. Reopen or file a focused core issue only if building that plugin exposes a concrete missing hook field, recipient identity contract, or delivery-policy API.
