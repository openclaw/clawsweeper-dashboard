## Summary

Keep open. Current main shipped the newer `agents.list[].tts` per-agent voice override path, but it has not implemented Feishu account-level TTS defaults or a generic channel/account default contract. The maintainer re-review explicitly says this PR should not land as-is, while leaving it open because the account-default gap remains valid and #44058 is still the broader tracker.

## Best Possible Solution

Keep the PR open but do not merge it as-is. The best path is a rebase/rewrite around a generic channel/account TTS default contract that composes explicitly with `messages.tts`, `agents.list[].tts`, local `/tts` preferences, and inline directives, with Feishu opting in through plugin-owned schema/runtime helpers and docs/tests tied to #44058.
