## Summary

Keep open: current main still has the reported agent-path preload asymmetry, and the issue is now paired with an open closing PR rather than a merged fix.

## What This Changes

Preload the standalone plugin tool registry in the agent-path plugin tool resolver before resolving plugin tools.

## Best Possible Solution

Review and land PR #77657, or an equivalent change that preloads with the same plugin-tool inputs in the agent resolver and adds focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible: current main still has the agent resolver skip preload while `resolvePluginTools` can return cached descriptor wrappers before loading the runtime registry. No live plugin run was executed in this read-only review.

## Solution Assessment

Yes, mirroring the catalog/MCP preload in the shared agent resolver is the narrowest maintainable fix because it uses the existing helper without changing selection or policy semantics.
