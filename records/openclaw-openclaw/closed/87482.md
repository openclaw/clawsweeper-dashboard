## Summary

Keep open: the report matches a current-main source bug where the materialized bundle MCP runtime is shadowed and never handed to cleanup, so MCP leases can stay active and keep spawned stdio servers from being retired.

## What This Changes

The issue asks OpenClaw to prevent duplicate `@modelcontextprotocol/server-filesystem` sidecars from accumulating for a single configured MCP server.

## Best Possible Solution

Fix the current cleanup bug by wiring the materialized bundle MCP runtime into the existing cleanup path and add regression coverage that proves the lease is released; keep broader shared-runtime policy in https://github.com/openclaw/openclaw/pull/79882 or a separate product review.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes from source: run an embedded attempt that materializes bundle MCP tools, then observe that the shadowed runtime handle prevents cleanup from releasing the lease; a live filesystem process-tree repro was not run in this read-only pass.

## Solution Assessment

Is this the best way to solve the issue? Yes: the narrow maintainable fix is to stop shadowing `bundleMcpRuntime` and prove cleanup releases the materialized runtime, without adding a new shared-runtime config surface for this leak.

## Repair Work Prompt

Fix this issue by preventing leaked bundle MCP runtime materializations in embedded agent attempts. In `src/agents/embedded-agent-runner/run/attempt.ts`, ensure the runtime returned by `materializeBundleMcpToolsForRun` is assigned to the outer cleanup-scoped `bundleMcpRuntime` instead of a shadowing local constant, so `cleanupEmbeddedAttemptResources` disposes it and releases the active MCP lease. Add focused regression coverage proving the materialized bundle MCP runtime is passed to cleanup/disposed after a successful attempt and on early cleanup; do not add `mcp.runtimeScope`, shared runtime behavior, config options, docs, or changelog entries. Mention https://github.com/openclaw/openclaw/pull/79882 only as related broader work, not as the fix for this cleanup leak.
