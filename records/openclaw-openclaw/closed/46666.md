## Summary

Current `main` already ships Slack-native interactive-reply prompt guidance, runtime handling, tests, and docs. The open PR targets the generic Telegram-style `inlineButtons` branch in `src/agents/system-prompt.ts`, but Slack is implemented through channel-owned `messageToolHints` plus `[[slack_buttons]]` / `[[slack_select]]` directives instead.

## Evidence

- **Slack-owned prompt guidance exists on main:** The Slack plugin emits interactive-reply instructions when `interactiveReplies` is enabled, including `[[slack_buttons: ...]]` and `[[slack_select: ...]]`, and emits the disabled-state config hint otherwise.
  - file: [extensions/slack/src/shared.ts:105](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/extensions/slack/src/shared.ts#L105)
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
- **System prompt already injects channel-specific hints:** Prompt assembly resolves `messageToolHints` from the current channel plugin and passes them into the embedded system prompt build for runtime channels.
  - file: [src/agents/channel-tools.ts:123](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/src/agents/channel-tools.ts#L123)
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
- **Slack runtime implements interactiveReplies directly:** Slack parses `[[slack_buttons]]` / `[[slack_select]]` directives and treats `interactiveReplies` as the enabling capability, including object-format config `{ interactiveReplies: true }`.
  - file: [extensions/slack/src/interactive-replies.ts:8](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/extensions/slack/src/interactive-replies.ts#L8)
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
- **Existing tests cover the shipped Slack guidance:** Slack channel tests assert the enabled-state prompt hints for `interactiveReplies`, including both button and select directives.
  - file: [extensions/slack/src/channel.test.ts:658](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/extensions/slack/src/channel.test.ts#L658)
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)

Routing Slack `interactiveReplies` through this branch would steer Slack toward the wrong payload shape.
