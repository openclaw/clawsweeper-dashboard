## Summary

Related open items such as https://github.com/openclaw/openclaw/issues/54397 and https://github.com/openclaw/openclaw/issues/39127 cover adjacent slices, but neither fully supersedes this request.

## What This Changes

Add a first-class Web Control UI and TUI sessions workspace with overview, non-attaching peek, inline reply, explicit waiting/working/done status, and background-session actions.

## Best Possible Solution

Define a maintainer-owned shared sessions-workspace contract that reuses sessions.list, sessions.preview, and sessions.send where possible, coordinates status semantics with https://github.com/openclaw/openclaw/issues/39127, and then stage Web Control UI and TUI UX slices deliberately.
