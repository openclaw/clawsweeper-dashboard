## Summary

Current main now fixes the reported macOS A2UI availability path: the macOS node runtime uses the node session’s scoped canvas surface URL, refreshes that capability after an A2UI readiness miss, and has focused coverage for the injected refresher path.

## What This Changes

The issue requests that Gateway A2UI host access work for macOS nodes so `canvas.a2ui.push` no longer fails with `A2UI_HOST_UNAVAILABLE`.

## Best Possible Solution

Keep the current node-scoped capability refresh implementation and let the next release carry it to users; Gateway canvas auth should remain unchanged.

## Reproduction Assessment

Yes. The source-level path is high-confidence: a macOS node A2UI invoke with a missing or stale scoped canvas capability used to reach `A2UI_HOST_UNAVAILABLE`; current main now resolves via the active node session and retries once after refreshing that capability.

## Solution Assessment

Yes. The implemented fix is the narrow maintainable solution because it reuses `GatewayNodeSession` surface URL and refresh contracts instead of weakening Gateway auth or adding a second A2UI routing path.
