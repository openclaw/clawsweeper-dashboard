## Summary

Current `main` already implements Ollama vision auto-detection during model discovery and shipped it in `v2026.4.7`, so this PR is superseded by work already on `main`.

## Risks / Open Questions

- Current `main` solves this via the `/api/show` `capabilities` field rather than the PR's extra metadata-name heuristics, so very old or nonstandard Ollama responses that omit `capabilities` may still require manual model config.
