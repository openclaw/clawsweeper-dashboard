## Summary

Current main still has the reported git-ref handling bug, and an open same-author fix PR with closing syntax already targets it, so this issue should remain open until that PR lands or is closed.

## What This Changes

The issue reports that explicit `git:` plugin refs beginning with option syntax can be interpreted by Git as checkout flags instead of refs.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79901 or an equivalent fix that uses a Git ref boundary and adds regression coverage for option-like refs.

## Reproduction Assessment

Yes. Current main parses `@--ignore-skip-worktree-bits` as `parsed.ref` and invokes `git checkout --detach <ref>`, and a scratch Git 2.53.0 check confirms that checkout form exits successfully while the separated `git switch` form rejects the invalid ref.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable fix: pass the requested ref after a Git option separator with `git switch --detach -- <ref>` and lock the behavior with a focused regression test.
