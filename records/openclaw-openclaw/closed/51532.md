## Summary

Current `main` already implements the requested behavior. The gateway agent handler now skips automatic agent main-session key derivation when a caller supplies `sessionId`, so `agent({ agentId, sessionId })` keeps the caller session instead of forcing `agent:<agentId>:main`. It also still forwards `agentId` when no conflicting resolved session key exists. Regression coverage was added on `main` for both the explicit `sessionId` case and the whitespace fallback case.

## Evidence
