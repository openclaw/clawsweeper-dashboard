## Summary

Keep open. Current main still lacks equivalent token usage buckets and still drives Usage Mosaic and hour filtering from session spans, while the proposed patch is a focused implementation with no blocking diff finding found in this review.

## What This Changes

This PR adds optional UTC quarter-hour token usage buckets to session cost summaries, aggregates them from transcript usage entries, updates Control UI Usage Mosaic/hour filtering to prefer those buckets with legacy fallback, and adds backend/UI tests plus a changelog entry.

## Best Possible Solution

Land a reviewed version that keeps the token-bucket field optional, derives sparse UTC quarter-hour token buckets from existing transcript usage, validates bucket coordinates in the UI, preserves legacy span fallback for older summaries, and keeps the focused backend/UI coverage for aggregation, timezone mapping, invalid data, and fallback behavior.

## Reproduction Assessment

Yes. On current main, a session with token usage in one quarter-hour but an activity span crossing another hour will still have Usage Mosaic tokens proportionally spread across the span and selected-hour filtering matched by first/last activity; the current files show that path directly.

## Solution Assessment

Yes, with maintainer review. The proposed direction is the narrowest maintainable extension of the already-merged #49396 pattern: add an optional sparse bucket contract, consume it in the token mosaic/hour filter, and leave dailyBreakdown and broader UsageTimeBucket work out of scope.
