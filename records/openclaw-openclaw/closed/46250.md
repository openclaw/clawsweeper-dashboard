## Summary

Current main already persists a fresh transcript file on session reset and archives the old transcript separately, with regression coverage for both the gateway reset path and session rollover path. The shipped v2026.4.22 release also contains this implementation.

## Evidence

- **Gateway reset creates the replacement transcript immediately:** `performGatewaySessionReset` mints a new `sessionId`, derives a new `sessionFile`, archives the old transcript, then creates the new `.jsonl` header file if it does not exist.
  - file: [src/gateway/session-reset-service.ts:544](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/gateway/session-reset-service.ts#L544)

  - file: [src/auto-reply/reply/session.ts:720](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/auto-reply/reply/session.ts#L720)
