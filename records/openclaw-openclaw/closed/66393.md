## Summary

Current `main` preserves WebChat image attachments for text-only models instead of dropping them. That matches the reported MiniMax M2.7 setup and no longer matches the shipped/current code path on `main`.

## Evidence

- **WebChat still accepts pasted images into attachments:** The WebChat composer intercepts pasted clipboard image items, converts them to data URLs, and stores them as chat attachments.
  - file: [ui/src/ui/views/chat.ts:208](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/ui/src/ui/views/chat.ts#L208)
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **WebChat sends attachments through `chat.send`:** The UI serializes attachment data URLs into base64 image attachments and includes them on the `chat.send` RPC payload.
  - file: [ui/src/ui/controllers/chat.ts:213](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/ui/src/ui/controllers/chat.ts#L213)
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **Current main offloads text-only image attachments instead of dropping them:** `parseMessageWithAttachments` now force-offloads images when `supportsImages === false`, preserving them as `media://inbound/*` refs rather than returning no images/offloaded refs.
  - file: [src/gateway/chat-attachments.ts:309](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/gateway/chat-attachments.ts#L309)
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **Gateway uses the session model capability to decide preservation path:** `chat.send` resolves whether the session model supports images and passes that into attachment parsing, so text-only sessions now preserve images through the offload path.
  - file: [src/gateway/server-methods/chat.ts:2204](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/gateway/server-methods/chat.ts#L2204)
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **Regression tests now require preservation for text-only sessions:** Current tests assert that text-only sessions produce `imageOrder === ["offloaded"]` and keep a `[media attached: media://inbound/...]` marker instead of losing the image entirely.
  - file: [src/gateway/server-methods/chat.directive-tags.test.ts:2080](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/gateway/server-methods/chat.directive-tags.test.ts#L2080)
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **Reported model is documented as text-only:** The bundled MiniMax docs define `MiniMax-M2.7` with `input: ["text"]`, which matches the reporter's setup and explains why the old text-only drop path would have affected this issue.
  - file: [docs/providers/minimax.md:144](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/docs/providers/minimax.md#L144)
