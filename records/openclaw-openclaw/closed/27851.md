## Summary

Current `main` implements the requested Discord EventQueue timeout fix: OpenClaw now defaults Discord listener timeout to 120s, exposes `channels.discord(.accounts.<id>).eventQueue.listenerTimeout`, and separates long-running agent work into the inbound worker so agent turn length is no longer tied to the gateway listener budget.
