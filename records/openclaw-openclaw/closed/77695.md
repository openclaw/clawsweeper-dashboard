## Summary

Close: current OpenClaw exposes Task Flow management under `openclaw tasks flow`, while the missing direct `acpx flow` subcommands are part of the separate published `acpx` CLI surface that this repo only wraps and pins.

## What This Changes

Request for `acpx flow list/status/cancel/inspect/logs` to exist or be clarified because `acpx` 0.6.1 only exposes `flow run`.

## Best Possible Solution

Keep OpenClaw's flow-management surface on `openclaw tasks flow`; track any desired direct `acpx flow` management commands or docs in `openclaw/acpx`, then update the OpenClaw plugin pin after an upstream release.

## Reproduction Assessment

Yes, from source inspection: published `acpx@0.6.1` registers only `acpx flow run <file>`, while current OpenClaw main registers and documents flow management as `openclaw tasks flow list|show|cancel`.

## Solution Assessment

No for this repository; adding direct `acpx flow` management commands inside OpenClaw would cross the ACPX package boundary. The maintainable path is an upstream `openclaw/acpx` feature or docs decision, followed by an OpenClaw dependency update if needed.
