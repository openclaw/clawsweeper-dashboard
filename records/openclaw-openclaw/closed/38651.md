## Summary

Current main already contains the scheduler startup and legacy-`every` fallback logic needed for this report, plus an exact regression test asserting that a previously broken stored job now yields a finite `nextWakeAtMs`. The same code is present in shipped release `v2026.4.22`.

## Evidence

- **Startup recomputes and persists wake times before reporting status:** `start()` reloads the store, runs `recomputeNextRuns(state)`, persists changes, arms the timer, and then logs `nextWakeAtMs`, so stored jobs are repaired on startup rather than left with a null wake.
  - file: [src/cron/service/ops.ts:87](https://github.com/openclaw/openclaw/blob/51186d2725439646f7fea0e59bb466f957404c46/src/cron/service/ops.ts#L87)
  - sha: [51186d272543](https://github.com/openclaw/openclaw/commit/51186d2725439646f7fea0e59bb466f957404c46)
- **Legacy `every` jobs without `createdAtMs` still get a valid anchor:** `computeJobNextRunAtMs` falls back to `nowMs` when `createdAtMs` is missing, then computes a valid next run for `every` schedules instead of leaving them unscheduled.
  - file: [src/cron/service/jobs.ts:251](https://github.com/openclaw/openclaw/blob/51186d2725439646f7fea0e59bb466f957404c46/src/cron/service/jobs.ts#L251)
  - sha: [51186d272543](https://github.com/openclaw/openclaw/commit/51186d2725439646f7fea0e59bb466f957404c46)

  - file: [src/cron/service.issue-regressions.test.ts:63](https://github.com/openclaw/openclaw/blob/51186d2725439646f7fea0e59bb466f957404c46/src/cron/service.issue-regressions.test.ts#L63)
