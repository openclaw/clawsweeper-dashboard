## Summary

Keep PR #72105 open. Current main still routes selected harnesses through the V1 adapter only, has no native AgentHarnessV2 factory registry/resolution seam, and the built-in PI harness is still V1-only. The PR is a fresh draft tied to the open RFC #72072 and implements a real core-agent lifecycle refactor rather than a ClawHub/plugin-side capability or an obsolete duplicate.

## Best Possible Solution

Keep the PR open for maintainer review. The appropriate path is to evaluate or revise the internal native AgentHarnessV2 factory registry on the PR branch, preserving the public V1 plugin contract while adding the core selection seam and PI parity tests described by RFC #72072.
