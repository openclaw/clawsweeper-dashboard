## Summary

Keep open: the proposed TTL sweep can delete start timestamps for active long-running runs before their terminal lifecycle event arrives, which would make `agent.wait` snapshots lose `startedAt`; the external PR also still lacks inspectable real behavior proof.

## What This Changes

The PR adds a TTL-based sweep of `agentRunStarts` inside `pruneAgentRunCache()` to delete orphaned start timestamps after the existing agent-run cache TTL.

## Best Possible Solution

Use lifecycle-aware cleanup that drains true orphaned starts without deleting active-run timestamps, with focused coverage for both orphan cleanup and long active runs crossing the cache TTL.

## Reproduction Assessment

Yes. By source inspection against the PR diff, a lifecycle `start` with `startedAt`, a prune after the 10-minute TTL, and a later terminal `end` without `startedAt` loses the original timestamp.

## Solution Assessment

No. Reusing the completed snapshot cache TTL for active-run starts is not the narrowest maintainable fix because active runs can last up to 48 hours and terminal events still rely on the fallback map.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P2] Preserve start timestamps for active runs:** `src/gateway/server-methods/agent-job.ts:39-42`

Since active runs can last far longer than the 10-minute cache TTL, a later `end` or `error` can make `agent.wait` return a snapshot without the original start time.
