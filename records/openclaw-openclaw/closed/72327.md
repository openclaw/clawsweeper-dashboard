## Summary

#72327 should close as a duplicate of the already-open canonical tracker #55334. Current main still persists the same static `skillsSnapshot` and `systemPromptReport` payloads per session entry, and #55334 is already kept open specifically for the remaining sessions.json bloat/OOM work.

## Best Possible Solution

Close #72327 as duplicate/superseded by #55334, and keep the remaining product/code work consolidated there: deduplicate or reference static `skillsSnapshot` and `systemPromptReport` data instead of persisting it per session, then improve hot paths such as `sessions.list`/`chat.history` so they do not need to materialize oversized whole-store JSON before serving bounded metadata.
