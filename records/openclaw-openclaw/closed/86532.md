## What This Changes

The issue reports that WebChat’s stop/thinking control can remain visually active after an assistant response has completed, requiring a page refresh to recover.

## Best Possible Solution

Keep the shipped terminal-status-based WebChat run-state handling, and request a fresh report only if a current release still reproduces with matching build SHA and browser/runtime evidence.

## Reproduction Assessment

No current-main failing reproduction remains from source inspection. Current main and the v2026.5.22 source have terminal-status gates plus regression tests for stale active-run flags hiding the stop button after completion.

## Solution Assessment

Yes. Treating terminal session status as authoritative over stale active-run flags is the narrow maintainable fix for this UI state problem.
