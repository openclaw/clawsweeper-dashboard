## Summary

Keep open: current main and the latest release have Bedrock inference-profile discovery, but they do not implement the reported `params.modelId` model-id override and do not document the shorthand Haiku 4.5 profile alias the reporter used, so the Bedrock-to-Anthropic fallback remains a plausible provider-routing bug rather than cleanup.

## What This Changes

The issue requests Amazon Bedrock cron jobs to route Haiku 4.5 through a supported inference profile ID/ARN instead of falling through to direct Anthropic when `params.modelId` is configured.

## Best Possible Solution

Teach the Bedrock plugin/runtime to keep an `amazon-bedrock` profile route when a supported inference profile ID or ARN is configured, with focused cron/fallback tests and docs; avoid silently adopting `params.modelId` unless maintainers explicitly make it a documented provider contract.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection supports the reported path because per-model `params.modelId` is merged but not used to change the selected Bedrock `modelId`, and cron model docs say the selected job model should be attempted before fallbacks.

## Solution Assessment

Unclear: the issue offers several fixes, and the safest permanent path is a provider-owned contract for canonical inference profile IDs/ARNs or documented aliases rather than a generic undocumented `params.modelId` override.
