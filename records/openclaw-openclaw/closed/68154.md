## Summary

Current main no longer uses the narrower channel-plugin inventory for `plugins.allow_phantom_entries`. The audit now compares allowlist entries against the plugin registry snapshot, whose discovery path includes bundled plugin directories before global extensions, so bundled IDs such as `telegram`, `openai`, `browser`, `acpx`, and `memory-core` are treated as known plugin IDs rather than phantom entries. The latest release tag still has the old `listChannelPlugins()` implementation, so no fixed release is identified yet.

## Best Possible Solution

Close this issue as implemented on current main and keep the registry-based phantom check. A future release should carry the fix; no separate OpenClaw core issue is needed unless users can reproduce the warning on a post-7c0fdae main build with a refreshed plugin registry.
