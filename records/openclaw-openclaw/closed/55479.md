## Summary

Current `main` already routes specialist-agent skill injection through the same merged workspace-skill loader used by `openclaw skills list`, so external skills from `~/.openclaw/skills` and agent/workspace skill roots are now included in the runtime snapshot and prompt path rather than diverging to bundled-only skills.

## Evidence

- **Shared loader includes managed and workspace skill roots:** `loadWorkspaceSkillEntries` merges `CONFIG_DIR/skills` (`~/.openclaw/skills`), personal/project `.agents/skills`, and `<workspace>/skills`, with managed skills taking precedence over bundled skills and workspace skills overriding all of them.

- **`skills list` uses that same loader:** `buildWorkspaceSkillStatus`, which backs the skills-status path used by `openclaw skills list`, calls `loadWorkspaceSkillEntries(...)` directly instead of a separate registry path.
