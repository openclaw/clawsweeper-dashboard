## Summary

Close as duplicate: this report adds useful OpenAI/OpenRouter evidence, but the same top-level `ref` plus nested `request` browser act bug is already tracked in https://github.com/openclaw/openclaw/issues/38762 and current main still shows the same parser path.

## What This Changes

Accept browser `act` requests where `ref` or related act fields are supplied at the browser tool top level while `request` contains the act kind and action-specific fields.

## Best Possible Solution

Keep https://github.com/openclaw/openclaw/issues/38762 as the canonical issue and fix the browser plugin parser to backfill missing supported top-level act fields into nested `request` without overriding nested values.

## Reproduction Assessment

Yes, by source inspection. With `action: "act"`, top-level `ref`, and `request: { kind: "click" }`, the parser returns the nested request unchanged, so `/act` receives no `ref` or `selector`.

## Solution Assessment

Yes. The best handling is to close this duplicate, link its OpenAI/OpenRouter model details to https://github.com/openclaw/openclaw/issues/38762, and land one narrow parser/test fix through the canonical issue.
