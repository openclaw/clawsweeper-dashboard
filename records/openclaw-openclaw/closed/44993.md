## What This Changes

The issue reports that heartbeat/cron prompts preserve an old `Current time:` line instead of refreshing it between recurring runs.

## Best Possible Solution

The shared helper should refresh or collapse helper-generated `Current time` blocks across heartbeat/reset callers, with focused regression tests for stale refresh, duplicate collapse, and user-authored passthrough.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: pass text containing an existing generated `Current time:` block into `appendCronStyleCurrentTimeLine` with a later `nowMs`, and current `main` returns the stale base unchanged.

## Solution Assessment

Yes. Fixing the shared helper is the narrowest maintainable solution because heartbeat and session-reset callers already pass fresh timestamps; caller-specific patches would leave sibling prompt paths inconsistent.
