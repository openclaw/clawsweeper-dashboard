- **[P1] Make legacy state probes best-effort:** `src/agents/workspace.ts:269`
  - body: `readWorkspaceSetupStateForDir` falls back to `.openclaw/workspace-state.json` whenever the new root state has no markers, but `readWorkspaceSetupState()` rethrows non-ENOENT errors. On the dot-restricted filesystems this PR targets, that legacy probe can abort setup before the new root state is written, so catch and ignore legacy-probe I/O failures.
  - confidence: 0.91
- **[P1] Reject foreign root state files:** `src/agents/workspace.ts:207`
  - body: Resolving OpenClaw state to `<workspace>/workspace-state.json` gives the setup writer a generic project-root target. A preexisting JSON file with no OpenClaw markers is parsed as empty and later overwritten, so choose a collision-resistant non-dot name or detect and reject foreign content before writing.
  - confidence: 0.88
- **[P2] Promote fallback state to the new path:** `src/agents/workspace.ts:401`
  - body: When `ensureAgentWorkspace` loads state from the legacy path here, `stateDirty` remains false, so the later write never creates the new root state file. Persist fallback-loaded state to the new path so existing workspaces complete the migration and do not keep depending on the dot-prefixed directory.
  - confidence: 0.86
