## Summary

Keep open. Current main recognizes stop/abort requests and has improved abort cleanup, but the runtime still relies on cooperative abort signals and Pi dependency behavior rather than an explicit hard-stop contract that fences off future tool dispatch after an authorized stop.

## What This Changes

The issue requests a hard emergency-stop mechanism for in-flight agent turns that can terminate the turn or prevent later tool execution after an authorized stop request.

## Best Possible Solution

Define and implement an owner-approved emergency-stop contract that cancels active work where possible, records a stopped outcome, clears queued work, and blocks any later tool dispatch after an authorized stop signal.

## Reproduction Assessment

No high-confidence live reproduction was established. Source and dependency inspection do show the current path is cooperative cancellation through AbortController and AgentSession.abort rather than an explicit hard-stop state machine.

## Solution Assessment

No, not as a small direct patch in runEmbeddedAttempt. The maintainable solution needs a designed core cancellation/fencing contract across provider streams, tool execution, subagents, queues, and transcript state.
