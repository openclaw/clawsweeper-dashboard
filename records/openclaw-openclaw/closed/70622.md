## Summary

Current main covers the reported failure mode: failed assistant turns are no longer replayed into OpenAI/Codex transport history, OpenAI Responses/Codex replay repair drops errored tool-call turns and synthesizes Codex-style `aborted` outputs only for genuinely missing tool results, bare `terminated` transport errors are classified for fallback, and stream-error turns now carry a non-empty fallback content block with the real error retained in `errorMessage`.

## Best Possible Solution

Close this issue as implemented on current main. Keep the current OpenAI/Codex replay repair, failed-turn filtering, stream-error fallback message, and `terminated` transport failover behavior; open a new focused issue only if the same repeated `terminated` state can be reproduced on current main with fresh session history/logs.
