## Summary

Keep this PR open. Current main has a built-in Bedrock known-context-window table plus default fallbacks, but it still does not implement the PR's intended `modelContextWindowOverrides` config surface or regex override behavior. The linked #73328 remains open, and this PR is still a plausible implementation candidate rather than obsolete work.

## Best Possible Solution

Keep the PR open for normal maintainer review. The best path is to finish and validate the plugin-owned Bedrock override implementation: keep the behavior in `extensions/amazon-bedrock`, expose the field through the plugin manifest and typed config surface, keep docs and regression tests for unknown foundation models and inference profiles, and decide whether legacy `models.bedrockDiscovery` compatibility should accept the new field. After targeted Bedrock tests and the appropriate changed gate pass, maintainers can merge it or explicitly close it in favor of a built-in-table-only solution for #73328.
