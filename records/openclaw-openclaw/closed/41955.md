## Summary

This PR is superseded by the overlapping open PR #48671. Current main no longer has the `resolveSessionOptions()` helper this branch modifies; the active Control UI dropdown path uses `resolveSessionOptionGroups()` and still needs a current-code fix there. The underlying bug should stay tracked by #41933 until #48671 or an equivalent current-main patch lands.

## Best Possible Solution

Close this outdated PR as superseded by #48671. The current fix should be reviewed or consolidated in `ui/src/ui/chat/session-controls.ts`, where `resolveSessionOptionGroups()` can merge configured sessionless agents after stored rows and the active key, dedupe normalized keys, use `agentsList.mainKey`, preserve configured names, and avoid advertising per-agent switching under global session scope.
