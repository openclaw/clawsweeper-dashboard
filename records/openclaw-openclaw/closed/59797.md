## Summary

The reported failure reaches the external `@tencent-weixin/openclaw-weixin` plugin’s own QR-login code and exact error string. Current OpenClaw core provides the generic channel-plugin login contract and does not implement WeChat QR login or Tencent iLink fetch behavior, so this belongs with the external plugin/ClawHub path rather than this source repo.

## Best Possible Solution

Leave OpenClaw core unchanged for this item. Track the QR-start AbortError in the external `@tencent-weixin/openclaw-weixin` plugin/ClawHub package, and keep host-side preflight hangs in a separate OpenClaw core issue when they reproduce before plugin `auth.login` starts.
