## Summary

Keep open: current main and v2026.5.2 still carry and accept reusable Control UI media credentials in `?token=`, and open remediation work is already linked through #77111 and the maintainer-labeled #70830 tracker.

## What This Changes

The issue reports that Control UI assistant media URLs expose reusable gateway or device credentials through the `token` query parameter.

## Best Possible Solution

Serve Control UI assistant media without putting reusable gateway or device credentials in URLs, then close this issue when the reviewed fix from #77111 or the #70830 design path lands.

## Reproduction Assessment

Yes. Source inspection on current main shows the UI builds assistant-media URLs with `token=<auth>` and the gateway accepts that query value; existing tests also assert the same behavior.

## Solution Assessment

Yes on the remediation direction: replace reusable query credentials with a browser-compatible media auth flow, then remove query-token parsing. The exact ticket, cookie, or fetch/blob design needs maintainer security review.
