## Summary

Keep open for maintainer review: the PR addresses a real Swift/OpenClawKit compatibility gap with older v2-only gateways, but it intentionally changes the signed auth payload from v3 to v2 and needs an explicit auth-compatibility decision before merge.

## What This Changes

The PR adds a Swift v2 device-auth payload builder, switches Swift gateway connect signing to that v2 payload, adds reset helpers, and covers the v2 payload vector with a Swift test.

## Best Possible Solution

Land a reviewed version of this compatibility fix if maintainers accept the temporary v2 signing tradeoff, then restore or negotiate v3 signing once deployed gateways are no longer v2-only.

## Reproduction Assessment

Yes. Source inspection shows current main Swift signs only v3 while the gateway still verifies v2, and the PR body includes redacted live logs showing an older v2-only gateway rejecting before the patch and connecting after it.

## Solution Assessment

Yes, with maintainer sign-off. The patch mirrors the existing TypeScript v2 canonical contract and uses a server-supported compatibility path; the long-term safer endpoint is returning to v3 once deployed gateways are upgraded.
