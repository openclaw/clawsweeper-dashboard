## Summary

Current `main` already supports configurable browser timeouts via per-call `timeoutMs`, default `browser.actionTimeoutMs` handling for browser `act` requests, and gateway/browser-request timeout propagation, so issue #55611 can be closed as implemented on `main`.

## Evidence

- **Browser tool schema exposes timeout controls:** `BrowserToolSchema` includes a top-level `timeoutMs` field for browser tool calls, and the nested act request schema also includes `timeoutMs`.
  - file: [extensions/browser/src/browser-tool.schema.ts:120](https://github.com/openclaw/openclaw/blob/fc5920fb51346a682b269fdbeeec0971ae4ae723/extensions/browser/src/browser-tool.schema.ts#L120)

  - sha: [fc5920fb5134](https://github.com/openclaw/openclaw/commit/fc5920fb51346a682b269fdbeeec0971ae4ae723)
- **Browser tool execution propagates caller timeout:** `createBrowserTool()` normalizes `params.timeoutMs` into `toolTimeoutMs` and passes it through browser status/start/stop/profiles/tabs/open/focus/close/screenshot/upload/dialog flows.
  - file: [extensions/browser/src/browser-tool.ts:413](https://github.com/openclaw/openclaw/blob/fc5920fb51346a682b269fdbeeec0971ae4ae723/extensions/browser/src/browser-tool.ts#L413)

  - file: [extensions/browser/src/browser-tool.actions.ts:79](https://github.com/openclaw/openclaw/blob/fc5920fb51346a682b269fdbeeec0971ae4ae723/extensions/browser/src/browser-tool.actions.ts#L79)

  - sha: [fc5920fb5134](https://github.com/openclaw/openclaw/commit/fc5920fb51346a682b269fdbeeec0971ae4ae723)
- **Browser config documents the default timeout knob:** The browser docs document `browser.actionTimeoutMs` as the default budget for browser `act` requests when the caller does not pass `timeoutMs`.
  - file: [docs/tools/browser.md:184](https://github.com/openclaw/openclaw/blob/fc5920fb51346a682b269fdbeeec0971ae4ae723/docs/tools/browser.md#L184)
