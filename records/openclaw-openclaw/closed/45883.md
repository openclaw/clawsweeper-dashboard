## Summary

Current `main` already changed gateway update/restart behavior to use a fresh process/supervisor restart on macOS/Linux instead of keeping the old process alive with stale hashed module imports, which matches the issue's requested fix.

## Evidence

- **Gateway restart now prefers full process restart:** `handleRestartAfterServerClose` releases the gateway lock, calls `restartGatewayProcessWithFreshPid()`, and exits when restart mode is `spawned` or `supervised`, logging `full process restart`. That directly addresses stale in-memory module references after update.
  - file: [src/cli/gateway-cli/run-loop.ts:82](https://github.com/openclaw/openclaw/blob/37c37eecfbe4bef413c33b8606db62cf51c2d1d4/src/cli/gateway-cli/run-loop.ts#L82)
  - command: `git blame -L 82,118 src/cli/gateway-cli/run-loop.ts`
  - sha: [37c37eecfbe4](https://github.com/openclaw/openclaw/commit/37c37eecfbe4bef413c33b8606db62cf51c2d1d4)
- **Fresh-PID restart helper added:** `restartGatewayProcessWithFreshPid()` explicitly implements fresh-process restart behavior: supervised installs exit and let launchd/systemd/schtasks relaunch; otherwise OpenClaw spawns a detached child with the current argv/execArgv. The doc comment states the intent is a fresh PID.
  - file: [src/infra/process-respawn.ts:20](https://github.com/openclaw/openclaw/blob/37c37eecfbe4bef413c33b8606db62cf51c2d1d4/src/infra/process-respawn.ts#L20)

- **Update flow respawns into updated install before post-update work:** For package-manager updates, `continuePostCoreUpdateInFreshProcess()` launches the updated `dist/entry.js` with `OPENCLAW_UPDATE_POST_CORE=1`, so follow-up tasks run in a fresh process from the new install root rather than the pre-update module graph.
  - file: [src/cli/update-cli/update-command.ts:895](https://github.com/openclaw/openclaw/blob/37c37eecfbe4bef413c33b8606db62cf51c2d1d4/src/cli/update-cli/update-command.ts#L895)

- **Regression test covers updated-root respawn:** There is an explicit test asserting that update respawns into the updated package root before post-update tasks, and that plugin updates/restart are not run in the old process.
  - file: [src/cli/update-cli.test.ts:475](https://github.com/openclaw/openclaw/blob/37c37eecfbe4bef413c33b8606db62cf51c2d1d4/src/cli/update-cli.test.ts#L475)
