## Summary

Current OpenClaw already handles the actionable OpenClaw-side parts of this report: official plugin installs skip lifecycle scripts, core docs already define third-party plugins as trusted in-process code, the community plugin bar already requires public source repos, and released builds already ship those safeguards/policies.

## Risks / Open Questions

- Direct `npm install -g <package>` outside OpenClaw bypasses OpenClaw's guarded install path and remains outside core's control.
- OpenClaw does not attest arbitrary npm package provenance; third-party plugins remain an operator trust decision, not a core security boundary.
