## Summary

Close as superseded. The bug is still present on current main, and this one-line guard is directionally valid, but open PR #74989 is the stronger canonical fix for the same root issue because it includes executor normalization, defensive text/Telegram consumer handling, and focused regression tests; the prior identical PR #74838 was already closed for the same overlap.

## What This Changes

This PR changes `executePluginCommand` to return an empty object when a plugin command handler result is nullish.

## Best Possible Solution

Consolidate review on #74989, or an equivalent single canonical branch, with shared executor normalization, defensive consumers for text and Telegram native command paths, and regression coverage for undefined plugin command returns.

## Reproduction Assessment

Yes. A registered plugin command whose handler completes without returning a payload will flow through the current executor raw return and hit unguarded result reads in the generic command path and Telegram native path.

## Solution Assessment

No for this PR as the merge vehicle. The executor nullish fallback is a valid primitive, but #74989 is the safer maintainable solution because it covers non-object results, caller-side hardening, and tests across the affected surfaces.
