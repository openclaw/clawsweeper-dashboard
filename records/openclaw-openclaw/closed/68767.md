## Summary

Current `main` already implements isolated subagent spawns as the default, requires explicit `context:"fork"` to inherit requester transcript, keeps `lightContext` scoped to lightweight bootstrap injection, and runs subagents in minimal prompt mode that omits memory/docs-heavy sections. That addresses the issue's reported root cause on `main` even though the latest stable release predates the change.

## Evidence

- **`sessions_spawn` now models transcript inheritance separately from `lightContext`:** The tool schema defines `context` as `isolated|fork` with docs saying `isolated` is the clean default and `fork` is only for transcript carry-over; `lightContext` is described separately as lightweight bootstrap context only.
