## Summary

Current `main` already implements the requested maintainer/operator escape hatch for plugin-update scan false positives: `openclaw plugins update` now supports `--dangerously-force-unsafe-install`, the option is wired through the update path into the installer, and the scan layer bypasses only built-in critical blocking while still honoring hook blocks and scan-failure blocking. This does not narrow the scanner heuristics themselves, but it does implement the issue’s suggested unblock path well enough to close the item as addressed on `main`.
