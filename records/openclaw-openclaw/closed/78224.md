## Summary

Keep open: current main still recursively watches configured memory extraPaths with chokidar `awaitWriteFinish` while `sync.watch` defaults to true. The small stop-polling PR was closed unmerged, and the related open FD/default-policy reports do not supersede this specific large-extraPaths leak report.

## What This Changes

The issue requests fixing memorySearch watcher file descriptor growth when `sync.watch` watches large `extraPaths` trees.

## Best Possible Solution

Stabilize memorySearch watching by eliminating or bounding per-file write polling for large extraPaths while preserving one-shot sync and documenting any watch policy change.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. Current source confirms the reported recursive extraPaths watcher with `awaitWriteFinish`, and the issue includes concrete macOS lsof/ulimit evidence, but the multi-hour FD growth was not rerun.

## Solution Assessment

Unclear. Removing `awaitWriteFinish` is the narrowest code-only mitigation and has a closed unmerged PR, but maintainers should decide whether to pair it with per-path watch policy, caps, or gateway-default changes and require macOS large-tree FD proof.
