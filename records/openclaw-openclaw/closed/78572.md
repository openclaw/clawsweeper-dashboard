## Summary

Keep open: current main still has a source-backed Discord message-tool routing bug where a bare allowlisted Discord user ID can be resolved to `channel:<id>` before delivery, bypassing the allowFrom-aware Discord DM resolver.

## What This Changes

Fix Discord `message(action="send")` so bare numeric targets listed in effective `allowFrom` deliver as DMs instead of failing as unknown channels.

## Best Possible Solution

Wire Discord's allowFrom-aware target resolver into the generic message-tool target path, preserve explicit `channel:<id>` behavior, and add regression coverage for a bare numeric target in effective `allowFrom`.

## Reproduction Assessment

Yes by source inspection, though not by live Discord execution: the current message-tool path normalizes a bare Discord numeric target to `channel:<id>` before delivery even though the Discord docs promise an allowFrom DM exception.

## Solution Assessment

Yes: the narrow maintainable fix is to delegate Discord generic target resolution to the existing allowFrom-aware resolver and cover the message-tool path with focused tests, not to add a new option or broaden core policy.
