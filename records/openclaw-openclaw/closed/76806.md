## Summary

Keep open. The PR targets a real source-reproducible context-overflow loop and addressed the earlier reset-kind concern, but the current head still misclassifies recoverable history-only overflows as irreducible and leaves user guidance incomplete.

## What This Changes

The branch adds an `irreducible_overflow` precheck route, non-resettable overflow metadata/reset skips, runner tests, docs, and a changelog entry for context-overflow compaction loops.

## Best Possible Solution

Keep the fail-fast route only for zero-history baseline overflow, preserve `compact_only` for history-only overflow, carry non-resettable metadata through auto-reply, and align copy/tests with system prompt plus current message causes.

## Reproduction Assessment

Yes. Source inspection against current main shows an over-budget zero-history baseline can route through `compact_only`, exhaust overflow compaction, then be reset as generic `context_overflow`; I did not run a live gateway scenario in this read-only review.

## Solution Assessment

No, not yet. The baseline detector and non-resettable kind are the right direction, but the PR must not turn recoverable history-only overflow into terminal `irreducible_overflow`, and the user-facing guidance should include oversized current input.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Preserve compaction for history-only overflow:** `src/agents/pi-embedded-runner/run/preemptive-compaction.ts:119-120`
  - body: When the zero-history baseline fits but session history pushes the prompt over budget, `toolResultReducibleChars` is often `0` because there are no tool-result messages. This branch now returns `irreducible_overflow`, so ordinary long-history sessions skip compaction and surface a terminal system-prompt error even though history compaction is the recoverable path. Keep this branch as `compact_only` unless the baseline itself exceeds budget.
  - confidence: 0.95
- **[P2] Mention current input in irreducible-overflow guidance:** `src/agents/pi-embedded-runner/run.ts:1447-1449`
  - body: The route is triggered by `systemPrompt + prompt` with no history, but this payload blames only the system prompt and skills/plugins. A very large current message can also trigger this path, so users need to be told to shorten the current message as well as reduce active skills/plugins or choose a larger-context model.
  - confidence: 0.87
