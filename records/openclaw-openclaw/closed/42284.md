## Summary

Current main already loads skills from the active workspace `skills/` directory, installs ClawHub skills into that active workspace, and documents that skill visibility is session-snapshotted. The reported failure is more consistent with installing into a different workspace or checking in an existing session than with a current-main discovery bug.
