## Summary

Keep open: current main still bundles the reported `@openai/codex` 0.130.0 harness and the Codex adapter still relies on native thread/resume, turn/start, and app-server tool-call handoff paths where this cache regression would occur. The open related prompt-cache reports are useful context, but they do not fully supersede this more specific tool-continuation regression report.

## What This Changes

The issue reports that Codex app-server tool-call continuation turns rebuild the OpenAI prompt instead of append-only continuing with the tool result, causing prompt-cache misses after exec or wait calls.

## Best Possible Solution

Keep this as a focused Codex harness regression until a live current-main trace compares 0.130.0 tool continuations against a known-good Codex version and proves whether OpenClaw should upgrade, pin, or change the harness request contract.

## Reproduction Assessment

## Solution Assessment

Unclear. The best fix should be chosen after live trace proof identifies whether the regression belongs to OpenClaw request assembly or upstream Codex app-server behavior; a speculative prompt rewrite in OpenClaw would be premature.
