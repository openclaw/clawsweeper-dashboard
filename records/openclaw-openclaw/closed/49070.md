## Summary

Current `main` already clears stale auth-profile lockouts during `openclaw models auth login`, which removes `usageStats[profileId].disabledUntil` and `disabledReason` before the provider auth flow runs. That matches the reported re-login failure mode, and there is direct test coverage for an OAuth profile with an active disabled window.

## Evidence

- **Login flow now clears lockouts before provider auth runs:** `runProviderAuthMethod()` calls `clearStaleProfileLockouts()` before invoking the provider auth method, so a re-login clears stale provider lockouts first.
