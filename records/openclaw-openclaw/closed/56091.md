## Summary

Keep open. Current `main` still appears to match key parts of the report: skipped Telnyx verification still participates in replay dedupe, outbound conversation startup does not start listening after the initial greeting, and configured TTS voice selection is only forwarded for Twilio so Telnyx custom voice IDs are ignored.

## Evidence

- **Skipped Telnyx verification still marks replays:** `verifyTelnyxWebhook(..., { skipVerification: true })` still builds a `telnyx:skip:` replay key and returns `isReplay` from the replay cache instead of bypassing replay handling.
