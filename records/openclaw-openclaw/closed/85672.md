## Label Justifications

- `P2`: This is a normal-priority feature/product gap for multi-agent browser workflows, with real workflow impact but no core outage or regression proof.
- `impact:session-state`: The issue is about subagents drifting from the intended logged-in browser/profile state and lacking explicit ownership metadata.
- `impact:security`: The requested inheritance and attachment behavior would govern access to logged-in cookies, SSO sessions, and private browser state.

## Summary

Keep open: current main has manual browser profile selection and existing-session attachment, but it does not provide first-class subagent browser ownership, explicit parent-browser inheritance, auth-state mismatch reporting, or dashboard ownership visibility. The nearby browser-isolation and browser-handoff issues overlap but do not supersede this specific logged-in subagent attachment problem.

## What This Changes

Subagents should gain an explicit browser-session ownership or inheritance contract so they do not ambiguously use a different logged-in browser than the parent or operator expects.

## Best Possible Solution

Design a browser-session ownership contract covering per-agent profile binding, explicit opt-in parent-browser inheritance/attachment, auth-state mismatch reporting, and visible owner/profile state in the browser/dashboard surfaces, coordinated with https://github.com/openclaw/openclaw/issues/37487 and https://github.com/openclaw/openclaw/issues/63926.

## Reproduction Assessment

Not applicable as a strict bug reproduction because the expected behavior is a new first-class browser ownership and attachment contract. Source inspection gives high confidence that current main lacks a sessions_spawn browser session/profile parameter and only offers manual browser-tool profile selection.

## Solution Assessment

Unclear as a direct implementation: a narrow hidden default change would be risky. The maintainable path needs explicit product and security design for dedicated versus inherited behavior, auth-state checks, and dashboard visibility.
