## Summary

Current main already fixes the reported QQBot crash by removing the bundled-channel startup path that passed only a minimal `runtimeContexts` object; QQBot now receives the full channel runtime with `inbound.run`, and the fix is current-main-only after v2026.5.28.

## What This Changes

The issue reports that QQ Bot inbound messages reach QQ successfully but crash during OpenClaw agent message processing with `Cannot read properties of undefined (reading 'run')` after upgrading to v2026.5.28.

## Best Possible Solution

Keep the current full-runtime startup path and let the next release carry the already-merged fix to QQBot users.

## Reproduction Assessment

Yes. The v2026.5.28 source path shows bundled channels receiving a minimal startup runtime while QQBot calls `runtime.channel.inbound.run`, matching the reported undefined `run` crash.

## Solution Assessment

Yes. The current-main fix removes the partial bundled startup runtime path and adds coverage that bundled channel startup receives a full runtime with `inbound.run`.
