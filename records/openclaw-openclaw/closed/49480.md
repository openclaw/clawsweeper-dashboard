## Summary

#49480 is a valid idea, but current OpenClaw scope puts new model-provider integrations on the plugin/ClawHub path rather than in core. Main already supports custom/self-hosted providers and has a documented provider-plugin SDK for non-bundled backends.

## Evidence

- **Vision keeps optional capability out of core:** OpenClaw's vision says core stays lean, optional capability should usually ship as plugins, plugins should be hosted and maintained in their own repository, and the bar for adding optional plugins to core is intentionally high.

## Risks / Open Questions

- If maintainers later decide a specific P2P backend is strategic enough to bundle, this scope decision could change.
- Some P2P systems may need new SDK seams or an agent-harness plugin rather than only models.providers config, but that still points to plugin work first, not a core issue.
