## Summary

Current `main` already carries a Feishu-side fix for this report: outbound upload now preserves UTF-8 filenames instead of percent-encoding them, and regression tests explicitly cover Chinese and other Unicode filenames. The issue’s claimed core `loadWebMedia` root cause does not match current `main`, which resolves local file URLs to paths and uses the path basename as the filename.

## Evidence
