## Summary

Keep open: current main still has a source-visible path where terminal ACP background tasks with a deliverable group/channel origin send generic lifecycle text directly to the channel, and linked PR #77365 is already the implementation candidate.

## What This Changes

The issue requests parent-owned ACP/subagent completions in group channels to route through the parent-agent reply path instead of stopping at generic `Background task ...` status messages.

## Best Possible Solution

Review and land PR #77365, or an equivalent narrow fix, so parent-owned ACP completions wake the parent announce path with useful child-result context while generic lifecycle text remains fallback telemetry.

## Reproduction Assessment

Yes, at source level: create a pending ACP task with a deliverable `requesterOrigin`, let it reach a terminal state, and current main sends formatted `Background task ...` text directly while the parent queue remains empty. I did not run a live group-channel scenario in this read-only pass.

## Solution Assessment

Yes, the narrow maintainable direction is to route parent-owned ACP completions through the existing parent announce machinery and keep direct lifecycle delivery only as fallback/status telemetry.
