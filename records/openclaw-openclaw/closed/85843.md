- **[P1] Preserve fresh cold plugin metadata reads:** `src/plugins/plugin-metadata-snapshot.ts:276-278`
  - body: Current main revalidates watched manifest/source/package/install-record files before reusing a snapshot, and the scoped plugin policy says metadata stays fresh unless a caller owns an explicit snapshot. This branch drops the watched-file hash and returns a memo based only on the registry envelope, so cold readers can miss plugin metadata edits until restart. Keep the watched-file freshness path, or make the reload-required contract explicit before merging.
  - confidence: 0.91
- **[P2] Do not cache cold metadata policies by config only:** `src/plugins/manifest-model-id-normalization.ts:52`
  - body: These policy caches are keyed by `configFingerprint`, which does not include plugin file content. Current main marks cold-loaded snapshots `cacheable: false`; this PR sets `cacheable: true`, so a first cold model-normalization or setup-backend lookup can pin stale plugin metadata even if the underlying snapshot reload later becomes fresh. Only cache when the resolver actually returned the Gateway-owned current snapshot, or include a metadata-content fingerprint in the cache key.
  - confidence: 0.86
