## Summary

Keep this PR open: current main still has only the docx read path, and this PR contains meaningful Feishu legacy-document support that is not obsolete, but PR head still has two correctness issues plus missing real behavior proof before merge.

## What This Changes

The PR adds Feishu legacy `/docs/:token` read support by detecting doc v2 metadata, reading legacy docs through doc v2 REST endpoints, routing upgraded docs to docx, and adding old-doc types/tests.

## Best Possible Solution

Land a Feishu-plugin-owned legacy read path after old-doc read failures propagate clearly, raw text can still be returned when only rich block content fails, and `/docs/:token` extraction plus old-doc permissions are documented.

## Reproduction Assessment

Yes, for the review findings via source inspection: PR head can detect `is_upgraded: false`, then a `/content` or `/raw_content` failure from `readOldDoc` is caught by the metadata fallback and retried as docx with the legacy token. I did not run tests because this review is read-only.

## Solution Assessment

No. The Feishu plugin boundary is appropriate, but this patch is not the best final solution until the fallback catch only covers metadata detection, rich content is best-effort for plain-text reads, and caller-facing legacy token/scope guidance is added.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Move old-doc reads out of the meta catch:** `extensions/feishu/src/docx.ts:876`
  - body: `readOldDoc` is still awaited inside the `try` whose catch falls through to the docx path. If metadata identifies a legacy doc but either old-doc content request fails, the real old-doc failure is swallowed and retried as docx with the legacy token, masking the cause and likely failing incorrectly. Only `fetchOldDocMeta` should be inside the catch.
  - confidence: 0.91
- **[P2] Keep raw old-doc reads working when rich content fails:** `extensions/feishu/src/old-doc.ts:155-158`
  - body: `readOldDoc` returns `content` from `/raw_content`, but `Promise.all` makes `/content` a hard dependency. A transient rich-content failure discards raw text that was already readable; make the rich block payload best-effort and still return raw text when `/raw_content` succeeds.
  - confidence: 0.82
- **[P3] Update legacy document token guidance:** `extensions/feishu/src/docx.ts:869-876`
  - body: This exposes `read` for `/docs/:token` documents, but the schema and bundled skill still tell callers to use `/docx/` tokens and list only docx scopes. Add legacy token extraction and required old-doc permissions so agents and operators can use the new path.
  - confidence: 0.79
