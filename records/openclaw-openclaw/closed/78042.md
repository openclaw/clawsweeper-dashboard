## Summary

Current main has removed the path-keyed bundled runtime-deps startup cache path described here; packaged installs now keep plugin dependency work out of Gateway startup and treat old plugin-runtime-deps roots as legacy cleanup.

## What This Changes

Request a content-keyed or operator-overridden bundled-runtime-deps cache key so identical OpenClaw installs at different realpaths do not pay a fresh startup dependency install.

## Best Possible Solution

Keep the current package-local dependency model and legacy-cache cleanup path; no new content-keyed cache or env-var bypass is needed for the old startup cache surface.

## Reproduction Assessment

Not applicable as a current-main bug reproduction. The reported 2026.4.27 behavior is addressed by source inspection on current main because the path-keyed startup staging functions and state cache are gone.

## Solution Assessment

Yes. Removing startup package-manager work and treating `plugin-runtime-deps` as legacy debris is a narrower operational fix than adding another cache-key mode for a cache OpenClaw no longer relies on.
