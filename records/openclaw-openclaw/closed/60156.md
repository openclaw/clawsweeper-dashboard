## Summary

The item is a broad proposal for agent-behavior layers, but current OpenClaw intentionally routes this kind of optional workflow/context/hygiene logic into plugins, skills, and ClawHub-distributed bundles rather than core. Main already exposes plugin seams for context assembly/compaction, native harness control, tool-result shaping, and reusable workflow skills, and there are already community plugins covering parts of the proposal.

## Evidence

- **Vision keeps optional behavior out of core:** Project direction says core should stay lean, optional capability should usually ship as plugins, bundle-style plugins are preferred when they can express the capability, new core skills should live on ClawHub when possible, and heavy orchestration layers are explicitly out of scope for default core architecture.

## Risks / Open Questions

- A narrower plugin-API gap may still be worth a focused core issue if a specific bundle cannot be implemented with the current seams.
- Closing this umbrella should not be read as rejecting the ideas themselves; it only scopes them to plugins/skills/ClawHub instead of OpenClaw core.
