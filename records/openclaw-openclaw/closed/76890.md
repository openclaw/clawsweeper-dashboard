## Summary

Keep open: current main still has a source-backed path from memory-core session-start warming into session transcript indexing without a session-loop yield, and linked PR #76978 is still open with closing syntax for this issue.

## What This Changes

The issue reports that memory-core session-start indexing can block the Gateway event loop for 30+ seconds when large session corpora are indexed.

## Best Possible Solution

Land or revise #76978 so memory-core session sync yields, defers, targets, or offloads heavy transcript indexing while preserving existing session source semantics.
