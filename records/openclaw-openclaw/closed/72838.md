## Summary

Close as implemented on main. The PR’s intended allowlist behavior is already present in current main through `resolveSubagentTargetPolicy` and `spawnSubagentDirect`, with regression coverage and docs, and the same behavior is present in the shipped `v2026.4.27` release.

## What This Changes

The branch changes `sessions_spawn` so any explicitly supplied `agentId`, including a self-target, is checked against `subagents.allowAgents`, and adds regression coverage for the denied explicit self-spawn path.

## Best Possible Solution

Close this PR and keep the shipped main implementation. Future changes to subagent topology policy should extend `resolveSubagentTargetPolicy` with focused tests and matching sub-agent docs.

## Reproduction Assessment

Yes. The reproduction path is an agent calling `sessions_spawn` with `agentId` equal to its own id while its `subagents.allowAgents` omits itself; current main has tests asserting this returns `forbidden` and does not call the gateway.

## Solution Assessment

Yes. Main’s shared target-policy helper is the narrow maintainable solution: it checks every explicit `agentId` target against the allowlist while preserving implicit same-agent spawns when no `agentId` is supplied.
