Full review comments:

- **[P1] Do not authorize cross-channel recall with a model-only reason:** `extensions/memory-core/src/tools.shared.ts:36-40`
  - body: Adding `scopeOverride` to `memory_search` lets the agent request arbitrary `slack_channel:<id>` scopes, and the resolver only checks that `reason` is non-empty. When this feature is enabled, the model can expand recall across Slack channel collections by self-supplying that reason, so this needs a maintainer-approved allowlist or non-model authorization gate before exposing the parameter.
  - confidence: 0.86
- **[P2] Make scoped QMD names match managed collections:** `extensions/memory-core/src/memory/qmd-manager.ts:1132-1135`
  - body: The resolver emits `memory-global-main`, `memory-private-main`, and `memory-slack-<id>`, but QMD currently registers defaults as `memory-root-<agent>`/`memory-dir-<agent>` and scopes workspace custom names through backend config. Since this filter intersects requested names with managed names, a normal enabled setup drops every collection and returns no memory results; wire scoped names into collection registration or derive them from the resolved collections.
  - confidence: 0.9
