## Summary

Keep open. Current main and v2026.5.12 still enable Gateway pricing unless `models.pricing.enabled` is explicitly false, so `models.mode: "replace"` can still reach the OpenRouter/LiteLLM pricing fetch path; the closing PR is open and unmerged.

## What This Changes

The issue requests that Gateway startup skip remote model pricing fetches when `models.mode` is `replace`.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/74038 with the shared helper default change, regression coverage, and docs/schema alignment.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure `models.mode: "replace"` with OpenRouter/LiteLLM-backed configured models and omit `models.pricing.enabled`; current main still treats pricing as enabled and can reach the remote fetch path.

## Solution Assessment

Yes. The narrow maintainable fix is to derive pricing enablement in the shared helper used by startup and refresh paths, preserving explicit true/false overrides while changing the omitted-pricing default for replace mode.
