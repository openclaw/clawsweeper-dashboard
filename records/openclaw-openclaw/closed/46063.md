## Summary

Current `main` still loads workspace skills into the shared skills status report used by both the CLI and the web Skills tab. The only concrete follow-up in the issue thread points to malformed `SKILL.md` YAML/frontmatter rather than an OpenClaw regression.

## Evidence

- **Workspace skills are still loaded from the active workspace:** `loadWorkspaceSkillEntries` resolves `<workspace>/skills`, loads those entries, and merges them with other skill sources at the highest precedence.
  - file: [src/agents/skills/workspace.ts:533](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/agents/skills/workspace.ts#L533)
  - command: `nl -ba src/agents/skills/workspace.ts | sed -n '533,629p'`
  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)
- **Web UI gets its data from the workspace skill status report:** The `skills.status` gateway method resolves the agent workspace and returns `buildWorkspaceSkillStatus(workspaceDir, ...)` to the UI.
  - file: [src/gateway/server-methods/skills.ts:68](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/gateway/server-methods/skills.ts#L68)
  - command: `nl -ba src/gateway/server-methods/skills.ts | sed -n '68,110p'`
  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)
- **CLI and web UI consume the same skill inventory:** `openclaw skills list` builds the same `SkillStatusReport`, and the Skills view renders `props.report?.skills` directly without filtering out workspace sources.
  - file: [src/cli/skills-cli.ts:27](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/cli/skills-cli.ts#L27)
  - command: `nl -ba src/cli/skills-cli.ts | sed -n '1,70p'`
  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)
- **Invalid or incomplete SKILL.md files are intentionally skipped:** The loader returns `null` when frontmatter parsing fails or when `description` is missing, so directories alone are not enough to appear in CLI/UI.
  - file: [src/agents/skills/local-loader.ts:60](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/agents/skills/local-loader.ts#L60)
  - command: `nl -ba src/agents/skills/local-loader.ts | sed -n '40,90p'`
  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)
- **Docs require valid YAML frontmatter with required metadata:** The docs explicitly require `SKILL.md` YAML frontmatter and call out `name` and `description` as required fields.
  - file: [docs/tools/skills.md:144](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/docs/tools/skills.md#L144)
  - command: `nl -ba docs/tools/skills.md | sed -n '136,190p'`
