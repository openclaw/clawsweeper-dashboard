## Summary

Current `main` already prevents `prompter.text()` from surfacing `undefined` into the onboarding wizard. The PR's local `?? ""` guards are redundant, and the `setup.plugin-config.ts` change would also blur cancel-vs-clear semantics in a way the existing PR discussion already called out as risky.

## Evidence

- **Wizard prompter contract is string-only on current main:** `WizardPrompter.text` is typed as `Promise<string>`, so wizard callers are not expected to receive `undefined`.
