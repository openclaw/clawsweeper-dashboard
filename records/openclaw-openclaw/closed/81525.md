## Summary

Keep open. Current main and the latest release still trust user-declared image capability for configured media-understanding model selection, and failed summaries still drop the attempted provider/model plus the diagnostic tail.

## What This Changes

The issue asks OpenClaw to validate or warn on misdeclared media-understanding image capability and emit actionable failed image-model diagnostics.

## Best Possible Solution

Adopt a maintainer-approved capability authority policy: at minimum warn on catalog-narrower configured image rows and preserve provider/model/input detail in failed summaries, with fail-closed behavior only if maintainers explicitly want strict catalog authority.

## Reproduction Assessment

Yes for the OpenClaw-side path: source inspection shows configured image rows are trusted for model selection and failed summaries strip useful detail. I did not re-call NVIDIA, so that rejection portion relies on the reporter's logs.

## Solution Assessment

Unclear. The diagnostic improvement is a narrow fix, but strict catalog cross-checking needs maintainer policy because configured provider rows currently act as operator-supplied capability metadata.
