Fix the Feishu WebSocket proxy mismatch reported in https://github.com/openclaw/openclaw/issues/85385. Keep the change inside the Feishu plugin and public SDK proxy-helper boundaries: the Lark SDK WSClient first calls pullConnectConfig through its httpInstance before opening the WebSocket, so ensure that HTTP preflight and the WebSocket dial use the same explicit OpenClaw/Proxyline proxy decision and do not let axios/follow-redirects select a stale SOCKS env proxy.
