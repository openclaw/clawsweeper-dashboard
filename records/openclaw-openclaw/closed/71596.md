## Summary

Keep PR #71596 open. Current main still has the stale-fallback behavior the PR targets: configure applies the allowlist before fallback reconciliation, fallback reconciliation cannot clear the last/deselected visible fallback, and the allowlist picker does not seed configured fallback refs into the visible selection. The PR is recent, focused, assigned for review, and the provided review context reports no major issues, only minor P2 hardening comments.

## Best Possible Solution

Keep the PR open for normal maintainer review. The best path is to continue with a focused fix that reconciles `agents.defaults.model.fallbacks` before allowlist pruning, seeds existing visible fallbacks into the picker, preserves hidden and out-of-scope fallbacks, and keeps the added regression coverage for singleton, scoped, alias-backed, and no-catalog selections.
