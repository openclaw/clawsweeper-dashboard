## Summary

Current main fixes the reported stdout pollution by routing command SecretRef diagnostics through stderr while preserving structured `secretDiagnostics` in the JSON payload; the fix is current-main-only and has not shipped in `v2026.5.12`.

## What This Changes

`openclaw status --json` should keep stdout parseable JSON when SecretRef resolution emits `[secrets]` diagnostics.

## Best Possible Solution

Keep the current-main stderr routing and structured JSON diagnostics, then ship it in the next OpenClaw release.

## Reproduction Assessment

Yes for the original defect: the tagged release source wrote SecretRef diagnostics through `runtime.log` before status JSON was emitted. Current main no longer has that failing stream path because the shared helper writes those diagnostics to stderr.

## Solution Assessment

Yes. Changing the shared command-secret diagnostic emitter from stdout to stderr is the narrow maintainable fix and preserves `secretDiagnostics` in the JSON payload without adding new config or product surface.
