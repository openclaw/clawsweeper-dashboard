Full review comments:

- **[P1] Preserve Slack session-key fallback routing:** `extensions/slack/src/approval-native-gates.ts:234-242`
  - body: Current main treats a parseable Slack sessionKey as a native plugin approval origin even when there is no `turnSourceTo` or persisted session-store entry. By moving forwarding eligibility entirely through `createNativeApprovalChannelRouteGates`, this path now only sees turn-source targets or persisted session targets, so session-key-only Slack plugin approvals that used to route natively can stop being delivered. Please keep the `resolveApprovalRequestSessionConversation` fallback in the Slack forwarding gate or extend the shared gate to accept that fallback before switching over.
  - confidence: 0.86
- **[P1] Keep wildcard approvals scoped to the native chat:** `extensions/slack/src/approval-auth.ts:35`
  - body: This authorizes any Slack sender whenever `allowFrom` contains `*` and no concrete approvers resolve, but the helper has no approval request, channel, thread, or message context and is also used by generic approval resolution paths. That is broader than the stated same-chat native-button behavior; keep wildcard approval authorization tied to the native interaction/request context instead of making it a global sender-level approval grant.
  - confidence: 0.84
