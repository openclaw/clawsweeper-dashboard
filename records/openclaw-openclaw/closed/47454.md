## Summary

Current main already contains the newline-chunking behavior needed for Telegram to preserve single-message line breaks. Telegram reply delivery routes newline-mode text through the shared chunker, and that chunker now keeps single-newline content in one chunk, only splitting on blank-line paragraph boundaries or real length limits. The same implementation and regression tests are present in release v2026.4.22.

## Evidence

- **Telegram reply delivery uses shared newline-aware chunker:** Telegram text replies call `chunkMarkdownTextWithMode(markdown, textLimit, chunkMode)` when `chunkMode === "newline"`, so the reported per-line split behavior is governed by the shared chunking path.
