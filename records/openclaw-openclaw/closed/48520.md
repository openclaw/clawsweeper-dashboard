## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve the caller working directory:** `scripts/build-sea.mjs:228`
  - body: The generated SEA launcher changes cwd to the package root before loading OpenClaw. Current CLI behavior reads cwd-local `.env` files and several commands infer workspace context from `process.cwd()`, so running the SEA binary from a mounted workspace would silently operate from `dist-sea` instead. Resolve sidecar paths without changing cwd, or restore the caller cwd before importing `openclaw.mjs`.
  - confidence: 0.92
- **[P1] Keep startup-env respawn behavior under SEA:** `src/entry.respawn.ts:72`
  - body: Because the SEA bootstrap sets `OPENCLAW_SEA=1`, this early return bypasses the current startup-env respawn path before it can apply `NODE_EXTRA_CA_CERTS` and related readiness guards. Current main intentionally separates startup TLS repair from warning-suppression respawn, including for interactive commands, so SEA needs an equivalent path rather than disabling the whole plan.
  - confidence: 0.87
- **[P2] Align the SEA Node runtime with sidecar dependencies:** `scripts/fetch-node-for-sea.mjs:10`
  - body: The build pins the SEA executable to Node 25.9.0 while staging `node_modules` from whatever supported Node install built the package. Since OpenClaw supports Node 22.19+ and ships native optional dependencies, the package needs same-runtime/container dependency proof or a build step that materializes sidecar dependencies for the exact Node/runtime being shipped.
  - confidence: 0.78
