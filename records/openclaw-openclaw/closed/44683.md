## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve existing Kubernetes resource names:** `charts/openclaw/templates/_helpers.tpl:8-9`
  - body: The documented release name `openclaw` renders `openclaw-openclaw` and `openclaw-openclaw-home-pvc`, which does not match the shipped manifests' `openclaw` objects. Users adopting the chart from the current Kustomize path would get a parallel deployment and fresh PVC instead of reusing their existing state.
  - confidence: 0.9
- **[P1] Trigger rollouts when config or secrets change:** `charts/openclaw/templates/deployment.tpl:15-17`
  - body: The pod template has only static labels and no checksum annotations for the rendered ConfigMap or Secret. Because the init container copies config at startup and credentials are env vars, `helm upgrade` can leave running pods on stale config or tokens until an operator manually restarts the Deployment.
  - confidence: 0.88
- **[P2] Fail when an external secret name is missing:** `charts/openclaw/templates/_helpers.tpl:26-30`
  - body: When `secret.create=false` and `secret.existingSecret` is empty, the helper still returns the generated name even though no Secret is created. The Deployment's gateway token reference is required, so the rendered release installs a pod that fails at container config time instead of failing during Helm rendering.
  - confidence: 0.9
- **[P2] Keep service.port synchronized with gateway config:** `charts/openclaw/values.yaml:84`
  - body: `service.port` controls the container port and probes, but the default `config.openclawJson` still hardcodes gateway port `18789`. Overriding `service.port` alone makes Kubernetes probe and route one port while the gateway listens on another.
  - confidence: 0.87
- **[P2] Avoid documenting credentials through Helm values:** `charts/openclaw/README.md:10-11`
  - body: The primary install command passes the gateway token and provider key with `--set`, which can leak through shell history, process listings, CI logs, or Helm release values. A first-party chart should lead with an existing Kubernetes Secret or another documented secret-management path.
  - confidence: 0.88
- **[P2] Guard replicas when using a ReadWriteOnce PVC:** `charts/openclaw/values.yaml:4`
  - body: The chart exposes `replicaCount` while the default PVC access mode is `ReadWriteOnce`. Setting replicas above one can leave extra pods pending on multi-node clusters, and `strategy: Recreate` does not prevent steady-state multi-replica scheduling.
  - confidence: 0.82
