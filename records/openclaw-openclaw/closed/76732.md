## Summary

Keep open: the source-level bug is valid, but this PR still misses prompt-local Telegram media, uses process-global pruning state, and lacks required after-fix real behavior proof.

## What This Changes

The PR adds an aggressive history-image pruning option for fresh-image turns, wires it into prompt-build and provider replay, adds a focused pruning test, and adds an unreleased changelog note.

## Best Possible Solution

Implement aggressive history-image pruning with attempt-local fresh-media state that covers prompt-local media notes and inline images while preserving raw transcript history.

## Reproduction Assessment

Yes, at source level. Current main preserves recent prior image blocks and media markers in replay while separately loading current prompt image refs, so an image-heavy Telegram/media-path turn can still present prior and current images together.

## Solution Assessment

No. Aggressive pruning is the right direction, but the branch should derive fresh-image state from attempt-local current prompt/media data and pass that state through provider replay without module-global state.
