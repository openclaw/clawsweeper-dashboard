## Summary

Close as not actionable in this repository: the reported duplicate-ingest branch is in the external lossless-claw plugin, while current OpenClaw and the v2026.4.29 tag do not pass `autoCompactionSummary` into context-engine `afterTurn` callers.

## What This Changes

Issue reports duplicate user messages when safeguard compaction interacts with the external lossless-claw context engine.

## Best Possible Solution

Keep OpenClaw's context-engine lifecycle unchanged; fix lossless-claw to ignore `autoCompactionSummary` when its transcript batch already contains the same source messages, or file a new OpenClaw issue with a minimal core reproducer.

## Reproduction Assessment

No. Source inspection of current main and v2026.4.29 shows OpenClaw does not pass `autoCompactionSummary` into the relevant `afterTurn` callers, so the claimed core-to-plugin duplication path is not reproducible from this repository as described.

## Solution Assessment

Yes for repo triage. Expanding OpenClaw's 60-second duplicate-message compaction window would be broad and does not address the observed external plugin branch; the narrow fix belongs in lossless-claw unless a core lifecycle repro appears.
