## Summary

Current `main` no longer preflights MiniMax image-edit requests into a resolution-override failure. The tool skips inferred resolution when a provider advertises `supportsResolution: false`, the image-generation runtime drops unsupported explicit resolution overrides instead of throwing, and MiniMax edit mode still maps the reference image into `subject_reference` for the API call.

## Evidence

- **Tool no longer infers edit resolution for providers that do not support it:** `image_generate` now leaves `resolution` undefined whenever edit mode is active and `supportsResolution === false`, which avoids the old preflight block described in the issue.

- **Regression test covers successful edit flow with `supportsResolution: false`:** The test `does not treat inferred edit resolution as an OpenAI override` exercises edit mode with a provider whose edit capabilities have `supportsResolution: false` and asserts the call succeeds with `resolution: undefined`.
  - file: [src/agents/tools/image-generate-tool.test.ts:718](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/agents/tools/image-generate-tool.test.ts#L718)
