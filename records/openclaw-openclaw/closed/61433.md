## Summary

Keep open. Current main still has partial wrapper/display mitigations, but the broad hardening request is not implemented: completion announce fallback can still deliver raw task result text directly, and isolated cron announce delivery still forwards selected assistant-visible text without a fail-closed rewrite/sanitization boundary for operational summaries, file paths, or routing labels. The related open issues cover narrower surfaces rather than fully superseding this broad cron/subagent/announce boundary issue.

## Best Possible Solution

Keep this issue open as the broad hardening tracker. Implement a shared fail-closed completion/announce delivery boundary: avoid raw event.result external fallback, require a clean synthesized user-facing payload or narrowly sanitized fallback, apply the same rule to isolated cron and subagent completion paths, and add regression coverage for internal wrappers, session ids, stats, file paths, routing labels such as Message for Telegram, and operational summaries across user-visible channel delivery and visible chat history.
