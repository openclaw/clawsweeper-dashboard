## Summary

Current `main` already applies per-agent skill filtering in the command/export system-prompt path and refreshes cached skill snapshots when the effective filter changes, including the persisted gateway-session and CLI-session paths. The same implementation is present in the shipped `v2026.4.22` release commit.

## Close Comment

Closing from a Codex review: current `main` already implements this. The command/export prompt path now builds its skills snapshot with the resolved agent context (`src/auto-reply/reply/commands-system-prompt.ts:60`), and the snapshot builder applies the effective per-agent allowlist from `config + agentId` (`src/agents/skills/workspace.ts:724`). Both persisted session paths also refresh stale snapshots when the effective skill filter changes (`src/auto-reply/reply/session-updates.ts:148`, `src/agents/agent-command.ts:578`). I also checked the shipped `v2026.4.22` release commit `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`, which contains this logic. If this is still reproducible on `v2026.4.22` or newer, please open a new issue with the exact config, session type, and current exported prompt/output.
