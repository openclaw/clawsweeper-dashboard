## Summary

Current main already contains generic recovery for aborted/failed reply runs: failed dispatches release inbound dedupe and mark the session idle, reply operations are always completed in a finally block, and queued followups recheck liveness instead of silently leaving the session stuck. The current tests also cover retry-after-failed-dispatch and the visible "still shutting down" recovery path.

## Evidence

- **Dispatch errors no longer leave inbound messages stuck in-flight:** On any dispatch exception, the dedupe claim is released and the session is marked idle, so the next inbound message can be processed instead of being silently dropped.
