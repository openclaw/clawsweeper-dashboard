## Summary

The request is primarily to bundle a new default skill, but current main scope explicitly routes new optional skills through ClawHub instead of core. OpenClaw already has core exec guardrails; the cited `exec-security` capability is better handled as the existing ClawHub skill rather than a bundled core skill.

## Evidence

- **Project vision keeps optional skills/plugins out of core:** `VISION.md` says optional capability should usually ship as plugins, bundle-style plugins/skills are preferred when they can express the capability, new skills should be published through ClawHub first, and 'New core skills when they can live on ClawHub' are explicitly listed under 'What We Will Not Merge (For Now)'.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/VISION.md#L54)

- **ClawHub is the documented install/distribution path for skills:** `docs/tools/clawhub.md` documents ClawHub as the public registry for OpenClaw skills and plugins, with native `openclaw skills install <skill-slug>` flows. It also distinguishes skill installs from plugin installs.
  - file: [docs/tools/clawhub.md:10](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/docs/tools/clawhub.md#L10)
