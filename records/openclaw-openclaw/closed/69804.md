## Summary

Current `main` already threads the raw/source config through gateway startup and plugin loading, and the loader now merges trust/allowlist data from that activation-source snapshot before `warnWhenAllowlistIsOpen()` runs. Regression tests on `main` cover the runtime-snapshot and gateway-startup variants of this bug. The fix is present on `main` in commit `07cf1dd65c937c75cf48a1645ea580404f5056b4`, but it does not appear to be in the latest tagged release `v2026.4.23`, so `fixedRelease` is `null`.
