Keep open: the latest PR head fixed the earlier `override` build blocker, but it still introduces a Discord voice access-control bug, has a false mode-setting confirmation, lacks real Discord proof, and appears to solve the linked `/voice` routing bug through a new `/vc` UX that needs maintainer judgment.

## What This Changes

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Authorize list before exposing voice sessions:** `extensions/discord/src/voice/command.ts:286-288`
  - body: `/vc list` reads active sessions and returns channel mentions without the `ensureVoiceCommandAccess` gate used by `status` and `leave`. In allowlisted guilds, an unauthorized user could learn active voice session channel IDs.
  - confidence: 0.92

  - confidence: 0.9
