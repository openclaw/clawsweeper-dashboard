## Summary

Current main still does not implement ProcessType=Interactive, but the same remaining macOS launchd/App Nap work is already tracked by open PR #62308 and the older open launchd/App Nap reports #58061 and #62294.

## Best Possible Solution

Close this duplicate and keep the work centralized in #62308 for the narrow ProcessType=Interactive code/test change, with #58061 and #62294 retaining the older launchd/App Nap evidence. Maintainers should review or rebase that PR, add the plist key in src/daemon/launchd-plist.ts, cover fresh install and restart rewrite behavior in src/daemon/launchd.test.ts, and decide whether service-audit/doctor should detect or repair existing plists missing the key.
