## Summary

Current main now resolves Gateway auth SecretRefs before emitting the doctor warning, with regression coverage for the resolving env SecretRef and exec opt-in paths; the same fixed source is present in the v2026.6.6 release.

## What This Changes

Doctor should not report a configured Gateway auth SecretRef as unavailable when that SecretRef resolves successfully.

## Best Possible Solution

Keep the shipped SecretRef-aware doctor behavior for resolvable refs, and handle the broader runtime-healthy but CLI-unavailable diagnostic question in https://github.com/openclaw/openclaw/issues/88022.

## Reproduction Assessment

No for current main. Reporters gave high-confidence reproduction on 2026.5.x, but current main has a source-level early return and regression test for a resolving `gateway.auth.token` SecretRef.

## Solution Assessment

Yes. The landed fix uses the existing SecretRef-aware Gateway token resolver in both doctor paths while preserving explicit `--allow-exec` behavior for exec providers.
