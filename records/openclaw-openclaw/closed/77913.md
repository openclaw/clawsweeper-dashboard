## Summary

Keep open: the PR addresses a valid existing-session split-DNS browser false positive and has sufficient real behavior proof, but the patch still leaves downstream existing-session result checks using direct-mode DNS pinning.

## What This Changes

The PR adds an `external-browser-proxy` browser navigation mode for non-OpenClaw driver profiles, skips Node DNS pinning for that mode, exports the SSRF hostname helper, and adds proxy-mode/navigation-guard tests.

## Best Possible Solution

Keep the new external browser proxy mode, but thread the profile-derived `browserProxyMode` through post-action result checks, new-tab checks, and tab-list redaction with focused route tests.

## Reproduction Assessment

Yes. Source inspection shows current main classifies existing-session as direct and DNS-pins hostnames, and the PR body gives a deterministic hosts-file or split-DNS reproduction path.

## Solution Assessment

No, not yet. The new mode is the right narrow fix, but the current patch is incomplete until all existing-session result-check paths receive the same derived mode.
