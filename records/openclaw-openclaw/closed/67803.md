## Summary

Current main already fixes the Codex app-server active-run/lane-lock behavior. The Codex harness now resolves completion from scoped `turn/completed` notifications, handles terminal `turn/start` responses without waiting for a later notification, clears the active embedded run in `finally`, and has regression coverage for the stale-lane failure. The fix is documented as shipped in v2026.4.24.

## Best Possible Solution

Keep the shipped v2026.4.24 Codex app-server completion and lane-cleanup behavior. If the separate Control UI or Gemini symptoms still reproduce on current OpenClaw, track them as focused issues with current logs, versions, and affected runtime details.
