## Summary

Close as duplicate/superseded by #51251. Current main still does not implement the requested reset behavior for explicit `/model` selections: `/new`, `/reset`, and `sessions.reset` clear auto-sourced fallback overrides but preserve user or legacy session overrides. #51251 is the broader canonical tracker for the same remaining session modelOverride/default-precedence problem.

## Best Possible Solution

Close this issue as a duplicate and keep the remaining behavior/product decision centralized in #51251. If maintainers decide bare `/new` and `/reset` should clear explicit user model pins, implement it centrally in `resolveResetPreservedSelection()`, preserve `/new <model>` as the opt-in fresh override path, and update auto-reply, gateway reset, Telegram/menu, docs, and changelog coverage together.
