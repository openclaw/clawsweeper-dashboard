## Summary

Keep open: the PR targets a real managed-media authorization bug, but the current patch still allows trusted identity requests without `x-openclaw-scopes` to inherit default admin owner semantics, and the PR body does not provide real gateway media-fetch proof.

## What This Changes

The PR removes requester-session ownership checks for managed outgoing image bytes, gates serving on OpenAI-compatible sender-owner auth, updates managed-image tests, and adds a changelog bullet.

## Best Possible Solution

Use a media-specific owner/admin predicate that rejects requester-session self-declaration and implicit default-admin fallback, cover the real helper semantics in tests, and land after redacted gateway media-fetch proof.

## Reproduction Assessment

Yes. Source inspection on current main shows a managed image GET authenticated as trusted-proxy or device-token reaches byte serving without requester-session ownership because `privilegedAccess` skips that branch; no live gateway fetch was run in this read-only review.

## Solution Assessment

No. The patch removes the client-declared requester-session check but reuses a generic OpenAI-compatible owner helper whose documented default-scope fallback is too permissive for this media endpoint.
