## Summary

The reviewed PR head should stay open: current main lacks the repair, the PR has a protected maintainer label, and the patch still has blocking doctor/plugin-contract issues.

## What This Changes

The PR adds plugin-owned doctor session-state repair metadata for stale Codex cleanup, wires it into doctor state integrity, updates docs/tests/changelog, changes the kitchen-sink canary to @latest, and bumps the axios override.

## Best Possible Solution

Keep the plugin-owned doctor repair direction, but make cleanup state-kind aware and race-safe, load built contract sidecars, identify auto auth state by credential ownership, and regenerate the SDK API baseline before rerunning the targeted checks.

## Reproduction Assessment

Yes, source-level. A PI route configured as openai-codex/gpt-5.4 with stale agentHarnessId: "codex" or codex-cli bindings reaches routeAllowsOwnerState and skips the cleanup branch.

## Solution Assessment

No. The plugin-owned seam is a maintainable direction, but the reviewed diff misses important current-main behaviors around provider/runtime separation, published plugin sidecars, store races, arbitrary auth profile IDs, and SDK baseline drift.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Separate provider routes from runtime cleanup:** `src/commands/doctor-session-state-providers.ts:187-191`
  - body: Because this returns true for any configured owner provider, a PI route like openai-codex/gpt-5.4 suppresses cleanup of stale native Codex runtime and codex-cli state. The docs define openai-codex/* as the PI OAuth provider route, not native Codex execution, so runtime/CLI cleanup needs to be state-kind aware.
  - confidence: 0.92
- **[P2] Load built doctor contract sidecars:** `src/plugins/doctor-contract-registry.ts:242-244`
  - body: The new sessionRouteStateOwners contract is loaded through resolveContractApiPath, but that resolver only probes the package root for doctor-contract-api.* or contract-api.*. A published TypeScript plugin such as @openclaw/codex will expose the compiled sidecar under dist/doctor-contract-api.js, so doctor will not find this contract on the npm-installed plugin path.
  - confidence: 0.88
- **[P2] Revalidate entries inside the store writer:** `src/commands/doctor-session-state-providers.ts:472-477`
  - body: The scan and confirmation happen before updateSessionStore reloads the current sessions store, but the mutator applies the old repair reasons without checking the current entry still has the same stale auto state. If the gateway or user changes the session between scan and write, doctor can delete a newly selected model/runtime/auth binding.
  - confidence: 0.86
- **[P2] Clear auto auth profiles by credential owner:** `src/commands/doctor-session-state-providers.ts:293-296`
  - body: This only records auto auth-profile cleanup when the profile ID starts with a configured prefix. Auth profile IDs are arbitrary keys, so a stale auto-selected Codex profile named work with provider openai-codex will be left behind even while the rest of the stale Codex route state is cleared.
  - confidence: 0.82
- **[P2] Regenerate the SDK API baseline:** `src/plugin-sdk/runtime-doctor.ts:20`
  - body: This new public export changes the openclaw/plugin-sdk/runtime-doctor surface, but the PR leaves docs/.generated/plugin-sdk-api-baseline.sha256 unchanged from main. The plugin SDK API check will continue to report baseline drift until the generated baseline hash is committed.
  - confidence: 0.84
