## Summary

Current code and docs show the reported `Krilling...` and `🛠️ Exec` text is Telegram tool-progress preview output, enabled by default when preview streaming is active, with existing config opt-outs; changing that default would be a product decision rather than a source bug fix.

## What This Changes

The issue asks OpenClaw to suppress Telegram-visible internal-looking progress/reasoning markers during multi-step tool runs with thinking enabled.

## Best Possible Solution

Keep the documented default and direct users who want quiet Telegram previews to set `channels.telegram.streaming.preview.toolProgress=false`, `commandText="status"`, or `streaming.mode="off"`; track any default change separately as a product decision.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the reported visible text via source and tests: Telegram preview streaming emits progress labels and tool lines by default. No evidence here shows an unintended stale final message after completion.

## Solution Assessment

Is this the best way to solve the issue? No as a bug fix, because hiding this by default would reverse documented released behavior. The narrow existing solution is the Telegram streaming config opt-out unless maintainers want a product-level default change.
