## Summary

Keep open. This is a draft, maintainer-authored, maintainer-labeled docs PR, so it should not be closed by cleanup automation; the patch also needs a small wording correction because it overstates that non-interactive repair is limited to non-service repairs.

## What This Changes

The PR updates `docs/cli/doctor.md` to clarify how `doctor --repair`/`--fix --non-interactive` handle missing or stale gateway service definitions and to point users at explicit `openclaw gateway install` commands.

## Best Possible Solution

Revise the docs to distinguish gateway service definition installs/rewrites from other runtime service repairs: missing installs and stale definition replacements should point to interactive confirmation, update repair mode, or explicit `gateway install` commands, while the option text should not claim all service repairs are excluded.

## Reproduction Assessment

Not applicable as a runtime reproduction; this is a docs accuracy review. The high-confidence verification path is static inspection of current-main doctor prompter, daemon repair, service config repair code, and the related tests.

## Solution Assessment

No. The documentation direction is right, but the current wording is too broad; a narrower wording around service definition installs and rewrites would match the implementation without understating other `doctor --fix --non-interactive` service actions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Narrow the non-service repair wording:** `docs/cli/doctor.md:32-35`
  - body: The new option text says `--repair`/`--non-interactive` perform only non-service repairs, but current `confirmRuntimeRepair` still auto-accepts runtime repairs in non-interactive repair mode unless the call sets `requiresInteractiveConfirmation`. Missing service installs and service definition rewrites are the interactive-only cases here, so this should be phrased around service definition installs/rewrites rather than all service repairs.
