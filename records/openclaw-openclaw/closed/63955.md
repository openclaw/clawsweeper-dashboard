## Summary

Current main now addresses the observable analysis-to-execution stall class: the embedded runner detects plan-only/intent-only turns, retries with an act-now steer, and surfaces an explicit blocked error instead of silently ending; the shared prompt also tells agents not to finish with a plan/promise when tools can advance. Heartbeat and inbound-message interruptions are handled by active-run queue policy, per-session lanes, and restart recovery. Durable task and Task Flow docs/code cover the background-progress tracking side. The only issue comment was unrelated promotional/joke content and did not add a distinct remaining requirement.

## Best Possible Solution

Close #63955 as implemented on current main. Keep the shipped plan-only retry/blocked-state path, per-session queue/heartbeat handling, restart recovery, and Task Flow/background-task progress model; open narrower follow-ups only for a concrete provider/channel where current main still reproduces an intent-only stall.
