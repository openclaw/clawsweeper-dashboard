## Summary

Current main already preserves and replays Gemini thought signatures for both the native Google Gemini transport and the Google/OpenAI-compatible tool-call replay path, with regression coverage. The shipped v2026.4.26 tree contains the same implementation, and related prior reports for this exact error were closed against the same behavior.

## Best Possible Solution

Close this as implemented and keep the shipped native-Google Part-level thoughtSignature replay plus the OpenAI-compatible extra_content.google.thought_signature replay path. If a fresh session after restart/update still fails, the useful follow-up is a narrower report with a redacted trajectory or outgoing contents payload around the assistant functionCall and following functionResponse so maintainers can identify a new edge case rather than the already-covered missing-signature path.
