## Summary

Keep open: current main and latest v2026.6.1 still have the status-only blank-model identity path, so this is a narrow memory-core bug rather than a stale or duplicate cleanup item.

## What This Changes

Fix `openclaw memory status` and `memory_search` so a blank memory embedding model resolves to the provider default before the index is declared stale.

## Best Possible Solution

Resolve the effective status-time provider model through the same provider-default logic used by initialization, then add a regression test for blank model plus prebuilt default-model metadata while preserving true mismatch warnings.

## Reproduction Assessment

Yes, from source inspection: with `memorySearch.model` as an empty string and existing index metadata built for the provider default, plain status never initializes the provider and compares the metadata against the blank configured model. I did not run a live Homebrew repro because this review was read-only.

## Solution Assessment

Yes, the best fix is a narrow identity-resolution change, not a new config option or a provider initialization probe in plain status. Plain status should stay fast while still resolving adapter default model identity consistently.
