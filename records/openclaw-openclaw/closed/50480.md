## Summary

Current main implements the WebChat WebSocket push path well enough to close this old 2026.3.13 report: Gateway emits chat delta/final/error events over the WebSocket, WebChat consumes those events into live chat state, and regression coverage exercises assistant deltas/finals including buffered final flushes. The latest release changelog also includes the chat-run registry fix for stuck chat state.
