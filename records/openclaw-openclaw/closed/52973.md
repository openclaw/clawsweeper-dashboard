## Summary

Current `main` already contains a targeted fix for bundled plugin SDK subpath resolution, including the exact `openclaw/plugin-sdk/provider-entry` failure mode described in #52973.

## Evidence

- **Root alias now emits explicit SDK subpath aliases and orders the bare root last:** `buildPluginSdkAliasMap()` adds `openclaw/plugin-sdk/<subpath>` aliases for every exported SDK subpath, then appends the bare `openclaw/plugin-sdk` alias last with a comment explaining this avoids prefix-based resolvers treating the root alias as the base for subpaths.
  - file: [src/plugin-sdk/root-alias.cjs:159](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/plugin-sdk/root-alias.cjs#L159)

  - file: [src/plugins/sdk-alias.ts:379](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/plugins/sdk-alias.ts#L379)

  - file: [src/plugins/sdk-alias.test.ts:702](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/plugins/sdk-alias.test.ts#L702)
