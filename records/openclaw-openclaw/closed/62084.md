## Summary

Current main isolates builtin `devices` commands from bundled plugin CLI loading and includes tested loopback fallback paths, so the reported Google-plugin crash no longer blocks `openclaw devices list` or `openclaw devices approve`. The shipped `v2026.4.23` release already contains this implementation.

## Evidence

- **Builtin primary commands skip plugin registration:** `runCli()` registers the primary builtin command first, then skips plugin CLI registration when that primary already resolves to a builtin command. This prevents `openclaw devices ...` from traversing bundled plugin startup just to parse the command.
  - file: [src/cli/run-main.ts:243](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/cli/run-main.ts#L243)
  - command: `git show a9797214338ba31c52c796adbb75afb16e0684a9:src/cli/run-main.ts | nl -ba | sed -n '239,275p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Skip policy is explicit:** `shouldSkipPluginCommandRegistration()` returns `true` whenever the primary command is builtin, which is the guard that keeps plugin registration out of the `devices` path.
  - file: [src/cli/command-registration-policy.ts:14](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/cli/command-registration-policy.ts#L14)
  - command: `git show a9797214338ba31c52c796adbb75afb16e0684a9:src/cli/command-registration-policy.ts | nl -ba | sed -n '1,40p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **`devices` is a lazy core subcommand:** The core subcommand registry maps `devices` directly to `../devices-cli.js`, so the command is handled by the builtin CLI registrar rather than by bundled plugin command discovery.
  - file: [src/cli/program/register.subclis-core.ts:99](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/cli/program/register.subclis-core.ts#L99)
  - command: `nl -ba src/cli/program/register.subclis-core.ts | sed -n '94,101p'`
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Devices path has direct pairing fallback:** `devices-cli` calls gateway pairing RPCs directly and falls back to local pairing storage for loopback `pairing required` failures in both list and approve flows, so basic device commands continue without depending on a successful bundled plugin load.
  - file: [src/cli/devices-cli.ts:151](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/cli/devices-cli.ts#L151)
