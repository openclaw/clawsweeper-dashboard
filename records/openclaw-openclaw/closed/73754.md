## Summary

Close as implemented. Current main unwraps the staged CommonJS JWT runtime modules before validation, the regression-shaped test covers the reported jsonwebtoken ESM/CJS shape, and the published `openclaw@2026.4.27` tarball contains the fixed compiled Microsoft Teams artifact while `2026.4.26` shows the broken destructuring.

## What This Changes

The issue reports that `openclaw@2026.4.26` shipped a Microsoft Teams bundle that calls `jwt.verify` on the CommonJS namespace instead of the default export, causing inbound Teams messages to 401.

## Best Possible Solution

Keep the shipped `v2026.4.27` implementation as the release fix for the broken bundled artifact; track warn-level JWT diagnostics or stronger live smoke coverage separately if maintainers still want those follow-ups.

## Reproduction Assessment

Yes. The release regression is reproducible at the package-artifact level: the `2026.4.26` tarball has the broken `jwt` namespace destructure, while the `2026.4.27` tarball has the default-unwrapping runtime loader before `jwt.verify`.

## Solution Assessment

Yes. For the release-facing failure, shipping a new OpenClaw package whose compiled Microsoft Teams bundle unwraps the runtime modules is the narrow maintainable fix; logging improvements are useful but not required to resolve the broken release.
