## Summary

Current main still matches the reported failure shape. Gateway `infer model run` sends a raw model-run request with tools intentionally suppressed, but the embedded runner still evaluates explicit tool allowlist sources against an empty callable-tool set and treats the resulting guard error as a precheck prompt failure. The normal fail-closed guard is intentional, so this should stay open for a narrow maintainer-approved fix rather than be swept as stale or implemented.

## Best Possible Solution

Keep this issue open. The best fix is a narrow maintainer-reviewed change that lets raw model probes bypass the empty explicit allowlist precheck when tools are intentionally disabled, while preserving documented fail-closed behavior for ordinary agent runs. The `tools.alsoAllow` orphan case should either be handled in the same PR with explicit tests or split into a follow-up after confirming whether orphan additive entries should be filtered, repaired by doctor/fix, or continue to fail closed.
