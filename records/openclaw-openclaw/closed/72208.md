## Summary

Current main and the shipped v2026.5.2 release now address the gateway pricing hang: pricing refresh starts after readiness, in-flight pricing fetches are abortable on Gateway stop, and post-stop timer/cache work is guarded.

## What This Changes

The issue reports that global npm openclaw@2026.4.24 on Windows hangs during gateway startup when LiteLLM/OpenRouter pricing fetches time out, leaving the dashboard unreachable and Ctrl+C ineffective.

## Best Possible Solution

Keep the shipped lifecycle fix and stable release notes; no further repository change is needed for this report.

## Reproduction Assessment

Yes for the historical failure: the report gives concrete 2026.4.24 Windows npm-global steps, and the regression test models the remaining lifecycle failure with hanging pricing fetches plus Gateway stop. Current main is not expected to reproduce because the fetches now receive the stop signal and post-stop work is guarded.

## Solution Assessment

Yes. The current fix is narrower than disabling pricing globally: it defers optional pricing work until readiness and makes the existing refresh lifecycle stop-aware without changing provider behavior.
