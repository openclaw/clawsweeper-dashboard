## Summary

Close as duplicate/superseded by #38458. Current main still has the reported mutating-tool warning behavior, but the older open issue already tracks the same tool-error visibility problem, includes this report in its cluster, and #51091 is the active unmerged implementation candidate needing maintainer alignment.

## What This Changes

The issue requests that `agents.defaults.messages.suppressToolErrors` suppress user-visible warnings for mutating tools such as `message` and `exec`, or that a new per-agent mutating-warning directive be added.

## Best Possible Solution

Use #38458 as the canonical design and implementation tracker, then resolve the product boundary there: either broaden `messages.suppressToolErrors` deliberately, add a clearer visibility policy, or land an aligned version of #51091 with tests that preserve agent-visible errors and avoid silent false-success behavior.

## Reproduction Assessment

Yes: current main has a source-level reproduction path. The policy order and unit tests show `messages.suppressToolErrors` suppressing non-mutating errors but not mutating `write`/`message` warnings, while `suppressToolErrorWarnings` suppresses mutating warnings only as a run-level option.

## Solution Assessment

No proposed patch is attached to this issue. The best maintainer path is to keep the remaining implementation/design work on the older open canonical tracker #38458 and review #51091 there rather than split the same work across duplicate issues.
