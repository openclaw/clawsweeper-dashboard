## Summary

Keep open. Current `main` contains the low-level 515 restart handling, but the QR web-login flow is still split across `web.login.start` and `web.login.wait`, and the UI still requires a separate explicit wait action. I did not find evidence that the reported QR-scan hang is fully implemented away on `main`.

## Evidence

- **515 restart is handled only inside the waiter helper:** `waitForWhatsAppLoginResult(...)` retries once on status 515 by creating a new socket, then returns success/failure to its caller; this is the primitive the report describes, not a full UI flow fix.
