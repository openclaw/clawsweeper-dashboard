## Summary

Current `main` already preserves external routing for inter-session/webchat turns and resolves A2A announce delivery from stored session delivery metadata before sending the announce, which addresses the reported `webchat` misrouting.

## Evidence

- **Announce target resolution uses session delivery metadata:** `resolveAnnounceTarget()` falls back to `sessions.list` and prefers `deliveryContext.channel`, `deliveryContext.to`, `deliveryContext.accountId`, and `deliveryContext.threadId` (with legacy fallbacks) before returning the announce target.
  - file: [src/agents/tools/sessions-announce-target.ts:14](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/agents/tools/sessions-announce-target.ts#L14)
- **A2A announce send uses resolved target, not requester channel:** `runSessionsSendA2AFlow()` calls `resolveAnnounceTarget()` and then uses the returned `channel`, `to`, `accountId`, and `threadId` for the final gateway `send` of the announce reply.
  - file: [src/agents/tools/sessions-send-tool.a2a.ts:64](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/agents/tools/sessions-send-tool.a2a.ts#L64)
- **Inter-session turns no longer overwrite external route with webchat:** `resolveLastChannelRaw()` and `resolveLastToRaw()` explicitly preserve an established external route for `sessions_send` / inter-session inputs instead of replacing it with internal `webchat` routing.
  - file: [src/auto-reply/reply/session-delivery.ts:102](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/auto-reply/reply/session-delivery.ts#L102)
- **Session state persists normalized deliveryContext for announce routing:** `initSessionState()` derives `lastChannel`/`lastTo` from the guarded routing helpers and writes them back into `deliveryContext`, so channel-scoped sessions keep their real delivery target.
  - file: [src/auto-reply/reply/session.ts:529](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/auto-reply/reply/session.ts#L529)
- **Regression coverage matches the reported failure mode:** Tests assert that `OriginatingChannel: "sessions_send"` preserves a persisted external route and that an internal/webchat first turn on a Telegram session key keeps `deliveryContext.channel === "telegram"`; announce-target tests also verify delivery-context-based announce routing.
  - file: [src/auto-reply/reply/session.test.ts:3283](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/auto-reply/reply/session.test.ts#L3283)
