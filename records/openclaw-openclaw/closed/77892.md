## Summary

Current main and shipped releases already support the requested OpenRouter provider routing through per-model params, generic OpenAI-compatible extra_body pass-through, and provider/model header configuration for preset-style headers.

## What This Changes

Request to let model configuration forward OpenRouter provider routing metadata, extra request body fields, and preset headers per model.

## Best Possible Solution

Keep using the shipped `params.provider` and `params.extra_body` paths plus provider/model headers rather than adding another OpenRouter-specific allowlist-map shape.

## Reproduction Assessment

Not applicable; this is a configuration feature request, not a broken-behavior report. Source, docs, tests, and release provenance show the requested routing and body-extension paths already exist.

## Solution Assessment

Yes; the shipped design keeps generic extra params in shared runner code and OpenRouter-specific routing in the OpenRouter plugin, which matches the repo's ownership boundaries.
