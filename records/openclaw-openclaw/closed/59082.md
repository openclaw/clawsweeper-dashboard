## Summary

This PR packages useful workspace automation, but current main’s documented direction is to keep optional orchestration/plugins out of core and publish them as external plugins/bundles. The official workspace template surface already exists on main; the remaining value here is better suited to ClawHub or a maintained external repo.

## Evidence

- **Vision: optional capability should live in plugins, not core:** Current project direction says core should stay lean, optional capability should usually ship as plugins, plugin discovery/promotion belongs in ClawHub, and OpenClaw does not want default agent-hierarchy frameworks or heavy orchestration layers that duplicate existing agent/tool infrastructure.
