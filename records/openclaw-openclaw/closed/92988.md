Full review comments:

- **[P1] Guard cleanup against active reindexes:** `extensions/memory-core/src/memory/manager-atomic-reindex.ts:301-302`
  - body: `runSafeReindex()` creates the temp DB before rebuild work starts, so a large in-flight rebuild can last more than 30 seconds. A second startup against the same store would classify that live temp DB as stale and remove it; add an ownership or live-process guard before deletion.
  - confidence: 0.9
- **[P2] Use the actual subsystem logger contract:** `extensions/memory-core/src/memory/manager.ts:393`
  - body: `cleanupOrphanedTempFiles` requires `info`, `warn`, and `error` to accept arbitrary `unknown[]`, but `createSubsystemLogger` exposes `(message: string, meta?: Record<string, unknown>)`. Passing `log` here is consistent with the branch's type-check failures, so narrow the helper logger type to the real contract.
  - confidence: 0.96
- **[P3] Remove sidecar-only temp groups:** `extensions/memory-core/src/memory/manager-atomic-reindex.ts:295-298`
  - body: If startup sees only `*.sqlite.tmp-*-wal` or `-shm` after a crash between publishing the main DB and post-swap sidecar cleanup, `fs.stat` on the base path fails and this branch marks the group fresh forever. Cleanup should remove aged sidecar-only groups too.
  - confidence: 0.82
