## Summary

Close as implemented on current main. The reported request-scoped fallback path now returns without creating a subagent run, and the cleanup loop only calls `deleteSession` for attempts with a non-null run id; current regression tests cover the direct narrative call and full dreaming phase sweep. The fix is also documented in the v2026.4.26 changelog and present in the tagged release source.

## What This Changes

The issue asks memory-core dreaming to avoid noisy narrative cleanup warnings when request-scoped subagent fallback writes a local diary entry without creating a subagent run.

## Best Possible Solution

Keep the shipped behavior: request-scoped fallback should write the local Dream Diary entry and skip subagent-session cleanup unless a subagent run was actually accepted, while preserving cleanup for real accepted runs.
