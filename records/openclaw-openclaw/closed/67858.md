## Summary

Current `main` already addresses the reported sticky-fallback behavior. The runtime now refuses to persist a fallback over a user-selected or legacy session model override, and the next turn clears any auto-failover override so the configured primary model is retried instead of staying pinned to the fallback.

## Evidence

- **Fallback persistence now skips user-selected and legacy overrides:** The fallback persistence path explicitly returns early when the active session override is user-selected (`modelOverrideSource === "user"`) or when a legacy override exists without `modelOverrideSource`, preventing fallback models from overwriting the user's chosen model in session state.

  - command: `nl -ba src/auto-reply/reply/agent-runner-execution.ts | sed -n '736,779p'`
  - sha: [306c0f73bff1](https://github.com/openclaw/openclaw/commit/306c0f73bff1338482dd6533f61a86905bf541b1)
- **Auto-failover overrides are treated as transient on the next turn:** Model selection now clears direct `modelOverrideSource: "auto"` session overrides before allowlist validation and resets in-memory provider/model back to the configured primary, so a previous fallback does not remain sticky across later turns.
