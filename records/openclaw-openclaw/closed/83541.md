## What This Changes

The PR gates transcript-derived `/status` Context promotion on a new `promptTokensFresh` flag and adds a cache-heavy legacy-session status test.

## Best Possible Solution

Separate the Context numerator from aggregate transcript cache totals by using a latest/current prompt snapshot for Context, keep aggregate cache values on the Tokens/Cache lines, and require a fixture that fails on current main or a maintainer-approved proof override.

No high-confidence live reproduction of the original >100% status output is available. The PR defect is source-reproducible: current main sets `totalTokensFresh` true for derived cache-inclusive totals, and the contributor's synthetic proof shows no branch delta.

## Solution Assessment

No. The intended boundary is right, but this implementation reuses a freshness signal that current parser code already applies to derived usage, so it should be replaced by a current/latest prompt snapshot separation or paused for maintainer override.
