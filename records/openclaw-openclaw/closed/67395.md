## Summary

Keep open. Current main does not implement either core behavior from PR #67395: Windows `pnpm.exe` is still not executed directly from `npm_execpath`, and the Lobster plugin still statically imports `@clawdbot/lobster/core` with no installed-package fallback. The related open issue #69456 tracks Windows regression coverage and explicitly references this PR’s fixes; it does not supersede the code changes.

## Best Possible Solution

Keep #67395 open. The best maintainer path is to rebase or port the focused fixes onto current main, preferably preserving the targeted `pnpm-runner` regression tests and the Lobster fallback/serialization/abort-signal tests. If maintainers want smaller review units, split the pnpm runner fix from the Lobster embedded-runtime fallback, and track the broader Windows CI lane hardening under #69456.
