## Summary

Current main and the latest v2026.4.24 release already cover the concrete Cloudflare Markdown for Agents use case: web_fetch sends an Accept header preferring text/markdown by default and logs Cloudflare's x-markdown-tokens hint when present. The exact general-purpose tools.web.fetch.defaultHeaders config field is not present, but the observable problem described in #43149 is solved without user config. Related PR #43477 proposed the exact knob and is useful context, but the core behavior now exists on main/release.

## Risks / Open Questions

- This does not implement arbitrary per-install default headers; it only resolves the reported Markdown Accept negotiation use case. A future request for broader custom headers should be evaluated separately because global headers could leak sensitive values across unrelated URLs.
