## Summary

Current `main` already avoids the reported onboarding failure mode: the recommended hatch path now launches the TUI in local embedded mode, so onboarding no longer depends on approving a gateway-paired device from another terminal.

## Evidence

- **Onboarding hatch now launches local TUI:** The onboarding finalize flow handles the recommended terminal hatch by calling `launchTuiCli({ local: true, deliver: false, message: ... })`. That means the onboarding path no longer enters the gateway-backed TUI flow that could get stuck behind pairing approval.
