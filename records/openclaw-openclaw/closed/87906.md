## Summary

Current main and v2026.6.6 already fix this issue's central delivered-then-failed contradiction: successful media generation no longer routes an unconfirmed completion delivery through the generation-failure path, and generated-media fallback delivery is treated as delivery evidence when it succeeds.

## What This Changes

The issue requests that successful long-running media generation remain a generation success when completion delivery falls back to text/direct media delivery without message-tool evidence.

## Best Possible Solution

Keep the shipped blocked-success/direct-fallback media completion behavior, and leave the broader missing-target no-delivery contract to the canonical issue and linked PR.

## Reproduction Assessment

No, not on current main: the old source path was clear in the prior review, but current source and focused tests now exercise the opposite behavior by completing the generated-media task instead of failing it.

## Solution Assessment

Yes, current main uses the safer fix shape: it preserves the delivery contract by marking unconfirmed delivery as blocked or using direct generated-media fallback, rather than blindly treating every failed completion notice as success.
