## Label Justifications

- `P1`: The issue reported a user-visible Discord channel workflow regression where accepted guild messages appeared idle for several seconds before typing feedback started.
- `impact:other`: The concrete impact is Discord channel responsiveness and operator observability, which is maintainer-visible but not message loss, session drift, auth, security, or availability.

## Summary

Current main and shipped releases now honor explicit Discord accepted-reply typing for `typingMode: "instant"`, carry the typing feedback through queue wait and dispatch, and the merged related fix directly addresses the reported guild-channel silent delay.

## What This Changes

Discord guild/channel messages with `typingMode: "instant"` were reported to wait several seconds before showing native typing feedback after acceptance.

## Best Possible Solution

Keep the merged Discord plugin typing lifecycle implementation and close this issue as fixed in shipped releases.

## Reproduction Assessment

Yes. Source inspection reproduces the old `v2026.5.6` DM-only accepted-typing gate and confirms current main now prestarts typing for explicit `typingMode: "instant"`; I did not run a live Discord account in this read-only review.

## Solution Assessment

Yes. The merged fix is the narrow owner-boundary solution: Discord owns native typing feedback, starts it after accepted preflight, dedupes queue-key keepalives, and restarts it at dispatch without adding new config.
