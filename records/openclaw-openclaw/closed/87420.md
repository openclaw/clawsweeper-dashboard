## Summary

Close: current main and the v2026.5.26 release already run media understanding before the followup queue stores the turn, and Telegram DM voice preflight is present, so queued voice followups should receive transcript-enriched prompt text instead of raw `<media:audio>`.

## What This Changes

The issue reports that Telegram DM voice notes can enter a busy-run followup turn as raw `<media:audio>` instead of a transcript.

## Best Possible Solution

Keep the shipped v2026.5.26 media-understanding-before-queue path; reopen or file a new report only with current-release logs showing a queued Telegram voice followup still reaches the model as raw `<media:audio>`.

## Reproduction Assessment

## Solution Assessment

Yes; the existing path is the narrow maintainable solution because shared media understanding runs once before prompt-envelope construction and enqueue, while Telegram DM preflight remains in the Telegram plugin.
