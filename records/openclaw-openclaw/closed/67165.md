## Summary

Current main already covers the three requested surfaces: SearXNG `web_search` trusts configured private/loopback base URLs, `web_fetch` has an RFC2544 opt-in for fake-IP proxy environments, and browser CDP reachability avoids self-blocking configured loopback/private control endpoints.

## Evidence

- **`web_fetch` exposes an RFC2544 SSRF opt-in:** The shipped v2026.4.23 tree includes `tools.web.fetch.ssrfPolicy.allowRfc2544BenchmarkRange`, threads it into `runWebFetch`, and uses it to relax the fetch guard only for that benchmark range.
  - file: [src/agents/tools/web-fetch.ts:274](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/agents/tools/web-fetch.ts#L274)
  - command: `git show a9797214338ba31c52c796adbb75afb16e0684a9:src/agents/tools/web-fetch.ts | rg -n "allowRfc2544BenchmarkRange|ssrfPolicy"`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **`web_fetch` behavior is covered by regression test:** `web-fetch.ssrf.test.ts` proves RFC2544 URLs stay blocked by default, succeed only when `ssrfPolicy.allowRfc2544BenchmarkRange` is enabled, and remain blocked again for stricter configs.
  - file: [src/agents/tools/web-fetch.ssrf.test.ts:159](https://github.com/openclaw/openclaw/blob/6f72b74cecf8d1fec5c147ae932161bf0f37a68b/src/agents/tools/web-fetch.ssrf.test.ts#L159)
  - command: `nl -ba src/agents/tools/web-fetch.ssrf.test.ts | sed -n '159,180p'`
  - sha: [6f72b74cecf8](https://github.com/openclaw/openclaw/commit/6f72b74cecf8d1fec5c147ae932161bf0f37a68b)
- **SearXNG accepts private/loopback HTTP and uses the trusted web-search path:** The SearXNG provider validates `http://` base URLs only for trusted private/loopback hosts, then performs searches through `withTrustedWebSearchEndpoint(...)` rather than the strict user-URL fetch path.
  - file: [extensions/searxng/src/searxng-client.ts:94](https://github.com/openclaw/openclaw/blob/6f72b74cecf8d1fec5c147ae932161bf0f37a68b/extensions/searxng/src/searxng-client.ts#L94)
  - command: `nl -ba extensions/searxng/src/searxng-client.ts | sed -n '82,101p'; nl -ba extensions/searxng/src/searxng-client.ts | sed -n '179,205p'`

- **Public docs still document localhost SearXNG:** The user-facing docs explicitly show `SEARXNG_BASE_URL="http://localhost:8888"` and state that `http://` is accepted for trusted private-network or loopback hosts.
  - file: [docs/tools/searxng-search.md:41](https://github.com/openclaw/openclaw/blob/6f72b74cecf8d1fec5c147ae932161bf0f37a68b/docs/tools/searxng-search.md#L41)
