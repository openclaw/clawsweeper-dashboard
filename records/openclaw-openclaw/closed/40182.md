## Summary

Current main already routes the agent session through the message tool and outbound media access path, so the reported missing-`agentId` fallback is implemented on main and present in the latest release snapshot.

## Evidence

- **Message tool passes resolved agentId into outbound runner:** `createMessageTool()` derives `resolvedAgentId` from `agentSessionKey` and passes both `sessionKey` and `agentId` to `runMessageAction()`. That directly addresses the report that the message tool invocation could omit the current agent id.
