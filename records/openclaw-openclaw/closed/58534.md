## Summary

Keep this issue open. The GitHub context shows an open, unmerged implementation PR that explicitly marks this issue as the one it would close, and current main only has partial mitigations for the reported session-scale problem.

## Best Possible Solution

Keep this issue as the canonical tracker until PR #58550 is reviewed and either merged or closed. If the SQLite store is too broad, the best narrower core path is to land a smaller fix that indexes child-session lookups, makes `sessions.list` truly pageable or incremental, avoids unbounded UI refreshes, and documents the resulting retention/performance behavior.
