## Summary

Current main already addresses the reported failure modes well enough: OpenAI Responses replay sanitizes orphaned `rs_*` reasoning references before send, and the WebSocket path now retries one retryable pre-output disconnect or falls back/fails fast instead of waiting for a long timeout.
