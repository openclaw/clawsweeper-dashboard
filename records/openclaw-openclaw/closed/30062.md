## Summary

Issue #30062 asks for a first-class GitHub channel in OpenClaw core, but current project direction is to keep optional integrations out of core and ship them as plugins published on ClawHub or npm. Current docs explicitly route new channels/integrations to the plugin SDK, and GitHub is not a supported built-in channel on `main`.

## Risks / Open Questions

- The issue is a plausible product request; closing it as `clawhub` assumes maintainers want GitHub automation to live as a separately maintained plugin rather than revisiting core scope later.
- GitHub is not a chat app in the same sense as existing channels, so a future implementation might fit better as a tool/hook plugin than a channel plugin, but that still supports closing this core-feature request as out of scope for main repo.

## Close Comment

Closing this as a `clawhub` item after Codex review of current `main`.

The current repo direction is to keep optional integrations out of OpenClaw core and ship them as plugins: `VISION.md:54` says optional capability should usually ship as plugins, hosted and maintained in their own repo, and `VISION.md:103` lists commercial service integrations outside the model-provider category as out of scope for core. The plugin docs also explicitly route new channels/integrations to the external plugin path: `docs/plugins/building-plugins.md:8`, `docs/plugins/sdk-channel-plugins.md:11`, and `docs/plugins/community.md:9`.

I also checked the current supported-channel docs on `main`; GitHub is not a built-in channel in `docs/channels/index.md:12`, and the only in-repo GitHub integration I found is the `extensions/github-copilot/` provider plugin, not a GitHub repo-activity channel.

Given that, this request is better pursued as a ClawHub/community plugin rather than an OpenClaw core issue. If someone wants to build it, the current Plugin SDK and channel/tool plugin docs are the intended path.
