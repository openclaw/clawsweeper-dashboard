## Summary

PR #60605 adds an optional standalone wrapper around the external `clawhub` CLI and Cisco AI Skill Scanner. That is useful third-party tooling, but current project scope points optional tooling/plugins to ClawHub/npm rather than adding repo-level scripts, and a native OpenClaw skill-install security change should be a separate focused core design.

## Best Possible Solution

Close this PR from the core repository and direct the wrapper to ClawHub/npm as external tooling. If the remaining product need is native ClawHub skill-install scanning, open or use a focused core security issue/PR that wires scanning into `src/agents/skills-clawhub.ts` and the existing install-scan/hook contracts.
