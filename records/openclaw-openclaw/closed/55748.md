## Summary

Current `main` already handles `null` and `undefined` at runtime. `src/utils.ts:125-133` returns `""` for any falsy `input`, and `src/utils.test.ts:139-142` already asserts `resolveUserPath(undefined as unknown as string)` and `resolveUserPath(null as unknown as string)` both return `""`. The submitted PR only widens the TypeScript signature and would remove test casts; it does not fix a reproducing runtime bug on current `main` (`d3d9b5738f83d92787f57db9186ddcebda60af9a`).
