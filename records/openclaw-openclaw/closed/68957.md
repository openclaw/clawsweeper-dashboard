## Summary

Current `main` contains targeted GitHub Copilot request-compatibility fixes that match this report: Copilot IDE headers are now added on the OpenAI Responses path and built-in compaction summarization, with regression tests and docs updated. The latest shipped tag in this checkout (`v2026.4.23`) still lacks the Responses-header change, so this appears fixed on `main` but not yet in a tagged release.

## Evidence

- **Copilot Responses requests now get compatibility headers:** `buildCopilotRequestHeaders()` is applied to both Anthropic and OpenAI Responses Copilot transports, which directly addresses provider-side schema/tool-payload rejection for ordinary Copilot prompts.
  - file: [extensions/github-copilot/stream.ts:25](https://github.com/openclaw/openclaw/blob/0ec3b79c07973e746328a1d6e1be5057096029af/extensions/github-copilot/stream.ts#L25)

  - file: [extensions/github-copilot/stream.test.ts:104](https://github.com/openclaw/openclaw/blob/0ec3b79c07973e746328a1d6e1be5057096029af/extensions/github-copilot/stream.test.ts#L104)
