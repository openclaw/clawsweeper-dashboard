## Summary

Current main already solves the PR’s central Slack channel leakage problem through the broader group/channel visible-replies model. Group/channel turns now default to message-tool-only source delivery, suppressing normal final, block, draft/preview, tool/progress output while leaving explicit message-tool sends available. The Slack-specific `suppressAssistantText` key is no longer the best path to land.

## Best Possible Solution

Close this PR and keep the broader current-main implementation: `messages.groupChat.visibleReplies: "message_tool"` is the product-level control for suppressing normal group/channel room replies while preserving explicit message-tool sends. If maintainers later want direct-DM or Slack-account-scoped suppression, that should be opened as a narrower follow-up against the shared source-delivery model rather than landing this stale Slack-only key.
