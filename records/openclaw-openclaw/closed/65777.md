## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Keep accepted decisions aligned with custom actions:** `src/infra/approval-view-model.ts:112`
  - body: This branch now prefers `pluginRequest.request.actions` for the pending view, but `plugin.approval.resolve` still validates against `snapshot.request.allowedDecisions`. For a custom action list that only exposes `deny`, a manual `/approve <id> allow-always` can still be accepted unless `allowedDecisions` was also set, so the visible/native approval choices and the resolver policy can diverge. Please make the resolver derive from the same decision actions, or require and normalize matching `allowedDecisions` when custom decision actions are used.
  - confidence: 0.86
- **[P1] Preserve the public approval action contract:** `src/infra/approval-actions.ts:22`
  - body: `ApprovalActionView` is exported through `openclaw/plugin-sdk/approval-handler-runtime`, and this new union means action entries can lack `decision`. Existing channel plugins that iterate `view.actions` as decision actions will either fail typechecks or mishandle command-only entries after upgrading. Please preserve a decision-only public view, or make this a documented/versioned SDK migration with contract tests for old and new consumers.
  - confidence: 0.8
