## Summary

Keep open. Current main still performs broad implicit provider discovery, so the requested cleanup is not implemented. The PR also should not be landed as-is because the current patch would skip profile-backed providers such as openai-codex when any models.providers entry exists.

## Best Possible Solution

Keep the PR open but do not land the current patch unchanged. The best fix is an owner-aware discovery scope that uses explicit providers, preserves profile-backed providers and auth-profile discovery, falls back safely when ownership is unknown, and adds regression tests for configured-provider scoping plus openai-codex/profile-backed discovery. Maintainers may later close this as superseded if #63552 or another canonical PR lands that behavior.
