## Summary

Close as duplicate/superseded: the same missing main-session dropdown behavior is already tracked by #43737 with open PR #43984 as the implementation candidate. Current main still has the underlying active-list/dropdown gap, so this issue is useful context but should be centralized on the canonical thread.

## What This Changes

The issue requests a stable pinned canonical `agent:main:main` entry and main-alias normalization in the Control UI chat session dropdown.

## Best Possible Solution

Keep #43737 as the canonical issue and review, land, or replace #43984 so the gateway consistently returns the real main session to every consumer.

## Reproduction Assessment

Yes. A high-confidence static reproduction is a stale `agent:main:main` row plus a fresher heartbeat/current session, followed by `sessions.list` with a positive `activeMinutes`; current main filters main out and the dropdown builder does not re-add it.

## Solution Assessment

Yes. Centralizing on #43737/#43984 is the best maintenance path because the root API behavior should be fixed once in `sessions.list`; a UI-only injected row should be considered only if the gateway can still omit a real main session after that fix.
