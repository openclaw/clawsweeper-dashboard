## Summary

Close: the observed typing-then-no-visible-reply behavior matches the documented group/topic visible-reply policy, not a current Telegram forum-topic routing bug. The follow-up workaround of setting `messages.groupChat.visibleReplies: "automatic"` is the documented way to restore legacy public topic replies.

## What This Changes

The issue asks OpenClaw to visibly respond to normal Telegram supergroup topic messages after an upgrade where topic messages show typing but no chat reply.

## Best Possible Solution

Keep the documented tool-only group reply default and have users who want legacy public topic replies set `messages.groupChat.visibleReplies: "automatic"`; reopen only with logs if that explicit config still fails.

## Reproduction Assessment

Yes for the observed symptom, not for a source bug. Current source maps group/topic chats without `messages.groupChat.visibleReplies: "automatic"` to message-tool-only delivery, which can show typing while suppressing normal final chat output.

## Solution Assessment

Yes. The narrow maintainable answer is the existing documented config policy, not a Telegram topic-routing code change.
