## Summary

Current main contains a two-part fix path for this report: the runner now emits an explicit idle-timeout error payload instead of ending the turn silently, and Telegram delivery now falls back/cleans up when no visible final reply was produced so stale preview state is not reused. Current tests explicitly cover stale-preview and no-final-response cases that match the reported duplicate-resend symptom.
