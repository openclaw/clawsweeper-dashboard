## Summary

Current main already makes `openclaw doctor --fix` the first-class repair path for missing bundled plugin runtime dependencies, prints the direct next step when deps are missing, documents the behavior, and includes regression coverage. The implementation is also present in the latest release, `v2026.4.24`.

## Best Possible Solution

Keep the shipped doctor-first repair path as the canonical bundled runtime-dependency maintenance surface. Add separate `plugins runtime-deps` subcommands only if maintainers later want extra operational UX beyond the already documented `doctor --fix` repair flow.
