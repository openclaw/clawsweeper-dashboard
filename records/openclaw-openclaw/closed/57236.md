## Summary

Keep this PR open. Current main still has the dirty watched source short-circuit this PR is trying to change, the current tests and gateway-watch regression helper still encode that behavior, and the PR is paired with the same-author open issue #57204 via closing syntax. This is an implementation candidate that needs maintainer product judgment, not conservative cleanup.

## Best Possible Solution

Keep this PR open as the implementation candidate for #57204. A maintainer should decide the intended run-node contract for dirty watched source trees; if the PR direction is accepted, update resolveBuildRequirement(), the dirty_watched_tree test expectations, and the gateway-watch regression helper together, then run targeted run-node coverage plus the appropriate changed gate.
