## Summary

Current main already fixes the observable #48518 zombie-after-compaction-timeout path with run-level compaction grace, abort-controller-wrapped compaction retry waiting, and handle-guarded active-run cleanup. The PR's force-clear approach is unnecessary and would bypass the handle guard in a way review already flagged as risky.

## Best Possible Solution

Close this PR as implemented by current main. Keep the shipped run-level compaction grace, abortable compaction wait, and handle-guarded active-run cleanup rather than merging the PR's unconditional force-clear. If someone can reproduce a zombie active run on current main or a current release, open a narrower issue with logs and version details.
