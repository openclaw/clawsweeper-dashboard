## Summary

Current main intentionally routes local/self-hosted OpenAI-compatible providers through the plugin surface, not new core provider ids. This PR adds a generic `local-api` provider by patching core `src/agents`/`src/commands`, but main already exposes dedicated provider-plugin helpers for discovery/onboarding and already supports arbitrary OpenAI-compatible endpoints through custom provider setup. That makes this a better fit for a ClawHub/provider plugin than OpenClaw core.
