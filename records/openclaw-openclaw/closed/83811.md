## Label Justifications

- `P2`: The report is a normal provider-specific web_search bug with limited blast radius and an implementation already shipped in the latest release.

## Summary

Current main and the latest stable release already implement Kimi web_search citation handling: citation URLs are extracted when Moonshot exposes them, and ungrounded chat-only responses now fail explicitly instead of being wrapped as a citation-unavailable search result.

## What This Changes

Kimi web_search should return source URLs/citations instead of a `citationsUnavailable: true` result.

## Best Possible Solution

Keep the shipped Kimi provider behavior and ask for a new report only if v2026.5.18 or newer drops citation URLs that are present in a redacted Moonshot response payload.

## Reproduction Assessment

No, I do not have a live current-main Kimi credential/repro run. The issue was reported against OpenClaw 2026.3.13, while current source, tests, docs, and release contents show the cited behavior has been replaced by citation extraction or an explicit ungrounded error.

## Solution Assessment

Yes. Handling citations inside the Moonshot-owned Kimi web_search provider is the narrow maintainable fix because the provider owns Moonshot `$web_search` parsing, base URL/model handling, and grounding semantics.
