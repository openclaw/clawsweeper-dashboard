## Summary

Current main already prevents the reported Claude CLI spawn crash for the described reproduction path by stripping null bytes from inbound metadata before that data is serialized into the prompt context, and the same code is present in the shipped v2026.4.23 release.

## Evidence

- **Inbound metadata now strips NULs at source:** `stripNullBytes()` removes `\u0000`, and both metadata strings and prompt bodies pass through it before prompt serialization.
