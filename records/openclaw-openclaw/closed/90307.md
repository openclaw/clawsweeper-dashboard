## Label Justifications

- `P1`: The reported failure is a regression-class embedded run abort that can break real channel replies for users in message-tool-only rooms.
- `impact:session-state`: The report is about session transcript ownership, session file fences, and active embedded session writer state.
- `impact:message-loss`: The takeover abort can stop the final visible channel reply after a successful message-tool send.

## Summary

Close: current main and the latest release already route `delivery-mirror` transcript appends through the owned embedded session write context, so OpenClaw-internal mirror writes no longer look like external session takeover writes.

## What This Changes

The issue asks OpenClaw to prevent `message_tool_only` delivery-mirror transcript writes from tripping `EmbeddedAttemptSessionTakeoverError` during an active embedded run.

## Best Possible Solution

Keep the shipped owned transcript write context and leave broader session-takeover or delivery-mirror follow-ups to their narrower open trackers if new evidence appears.

## Reproduction Assessment

Yes for the old bug shape: the report identifies a clear source path where message-tool delivery appends a `delivery-mirror` transcript entry while the embedded prompt fence is active. Current main now has source and regression-test proof that owned mirror appends refresh/publish the fence instead of reproducing the abort.

## Solution Assessment

Yes. Routing mirror writes through the active owned session write context is the narrow maintainable fix because it preserves transcript persistence and the external takeover guard instead of swallowing errors or changing delivery policy.
