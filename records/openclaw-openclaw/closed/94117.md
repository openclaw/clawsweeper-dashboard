## Summary

Keep open: current main still emits rich Telegram anchors for explicit local or relative Markdown hrefs, and this issue is already paired with open candidate PRs that use closing references. The source path plus two live reports support a real Telegram final-reply message-loss bug.

## What This Changes

Telegram rich replies should degrade local or relative Markdown link hrefs to visible label text instead of emitting rich anchors that Telegram rejects.

## Best Possible Solution

Land one narrow Telegram formatter fix with regression coverage and real Telegram or Mantis proof, then close this issue through the merged PR.

## Reproduction Assessment

Yes. Two live reports show RICH_MESSAGE_URL_INVALID on v2026.6.8, and current source still converts explicit local or relative Markdown hrefs into rich anchors before sendRichMessage; I did not run a live Telegram repro here.

## Solution Assessment

Yes. The best fix is local to the Telegram rich formatter: suppress unsupported href anchors while preserving label text and supported links, rather than adding a parse-mode/config workaround.
