## Summary

Current `main` implements main-session cron `systemEvent` delivery instead of silently no-oping: cron enqueues the event, wakes heartbeat with `heartbeat.target: "last"`, and heartbeat turns tagged cron events into a cron-event prompt. Unsupported non-default-agent `sessionTarget: "main"` configs are now rejected up front with an explicit error.

## Evidence

- **Main-session cron execution now enqueues + wakes with target=last:** `executeMainSessionCronJob` enqueues the `systemEvent` text and, for both `wakeMode: "now"` and `"next-heartbeat"`, calls heartbeat with `heartbeat: { target: "last" }` instead of silently returning.
  - file: [src/cron/service/timer.ts:1205](https://github.com/openclaw/openclaw/blob/171322644dece55402569c52b26370fd81769ca6/src/cron/service/timer.ts#L1205)
  - command: `nl -ba src/cron/service/timer.ts | sed -n '1190,1288p'`
- **Gateway cron adapter canonicalizes the scoped main-session key:** Gateway cron wiring resolves the target agent/session before calling `enqueueSystemEvent`, `requestHeartbeatNow`, and `runHeartbeatOnce`, so main-session cron jobs target the correct scoped session instead of a swallowed alias.
  - file: [src/gateway/server-cron.ts:281](https://github.com/openclaw/openclaw/blob/171322644dece55402569c52b26370fd81769ca6/src/gateway/server-cron.ts#L281)
  - command: `nl -ba src/gateway/server-cron.ts | sed -n '281,330p'`

  - file: [src/cron/service.main-job-passes-heartbeat-target-last.test.ts:56](https://github.com/openclaw/openclaw/blob/171322644dece55402569c52b26370fd81769ca6/src/cron/service.main-job-passes-heartbeat-target-last.test.ts#L56)
