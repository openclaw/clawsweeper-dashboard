## Summary

Close this PR as superseded by the already-open canonical PR #74002. Current main still has the underlying exec-approvals state-dir bug, but #74002 covers the same JSON/socket/default-host-source behavior with broader docs, changelog, and store/effective-policy regression coverage; this PR also repeats the generic `resolveStateDir()` fallback problem that prior duplicate review found in #75210.

## What This Changes

This PR changes exec approval JSON/socket path resolution and effective-policy host-source reporting to derive defaults from `resolveStateDir()`, and updates policy/allowlist tests for the new path behavior.

## Best Possible Solution

Consolidate review on #74002 or a maintainer-selected replacement that honors `OPENCLAW_STATE_DIR` for approval JSON, socket, and effective-policy display paths while preserving the unset `~/.openclaw` default, adding store/effective-policy regression tests, and updating public approvals docs/changelog.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: set `OPENCLAW_STATE_DIR` to a non-default path and current main still resolves `exec-approvals.json` and `.sock` through the hard-coded `~/.openclaw` constants; this PR's regression is reproducible by following `resolveStateDir()` into its legacy `.clawdbot` fallback when the override is unset.

## Solution Assessment

No. The intended direction is valid, but this branch is not the best landing path because #74002 is the broader canonical fix and this patch directly reuses `resolveStateDir()`, which can relocate the default approval store to legacy `.clawdbot` for legacy-only homes.
