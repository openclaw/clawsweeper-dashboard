## Summary

Keep open for maintainer judgment. Current main still intentionally steers rapid same-session messages into the active run by default, while the Feishu-specific wrong-current-message claim has not been live-reproduced or narrowed beyond the broader session-context tracker.

## What This Changes

The issue reports that rapid Feishu messages queued behind an active session are answered as merged or misaligned prompts instead of independent turns.

## Best Possible Solution

Maintainers should decide whether Feishu or direct-message traffic should keep default `steer` semantics with documented `followup` guidance, or change that default with focused queue and prompt-context regression coverage.

## Reproduction Assessment

Yes for the documented merge path: current source and docs show default `steer` routes rapid follow-up text into the active run. No for a distinct Feishu prompt-builder bleed, which still needs a live transcript or deterministic replay.

## Solution Assessment

Unclear. Existing `followup` mode is the narrowest way to get independent later turns today, while changing Feishu or global defaults is a maintainer product decision.
