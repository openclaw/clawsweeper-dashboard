## Summary

Close as ClawHub/external-plugin scope. PR #72091 adds a new optional Fleet Orchestrator bundled plugin plus contract/integration tests, not just a narrow test improvement. Current main does not contain an orchestrator extension, but it already exposes the plugin seams this PR uses: HTTP routes, CLI registration, background services, plugin config, runtime subagent helpers, and TaskFlow APIs. The project vision says optional capability should usually ship as plugins through ClawHub/npm, and specifically calls out agent-hierarchy frameworks and heavy orchestration layers as not-for-core for now. The existing Webhooks plugin already covers authenticated HTTP ingress into TaskFlows for trusted external automation. The provided related title-search reports are unrelated cleanup/flaky-test items and do not supersede this PR.

## Best Possible Solution

Do not merge this as a bundled `extensions/orchestrator` core plugin. Publish and iterate the orchestrator as an external ClawHub/native plugin using the existing plugin SDK, TaskFlow runtime, and Webhooks-style HTTP patterns. If that external implementation hits a concrete missing SDK seam, bring back a narrow API-gap PR or design issue rather than the whole orchestration layer.
