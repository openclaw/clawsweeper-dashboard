## Summary

Current `main` already supersedes this PR with a broader cron table hardening change: `printCronList` now stringifies unknown cell values before padding and truncating, which covers the `undefined`/`null` cases this PR targeted and also closes the `truncate(job.name)` crash path raised in review. The current tests also include a malformed-row case with `name: undefined` and `sessionTarget: undefined` and assert `printCronList` does not throw.

## Evidence

- **Current formatter hardens both padding and name truncation:** `stringifyCell()` converts non-string cell values to a printable fallback, `pad()` uses that helper, and `printCronList()` now calls `truncate(stringifyCell(job.name), ...)`, so the formatter no longer relies on `job.name` being defined before either `padEnd()` or `length` access.
