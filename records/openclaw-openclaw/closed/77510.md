## Summary

Close because the reported WhatsApp behavior does not match current main or the latest release: configured `allowFrom` entries are included in the effective DM allowlist before `dmPolicy="pairing"` creates a pairing challenge, and the WhatsApp docs already describe pairing approvals as merged with configured `allowFrom`.

## What This Changes

Request clearer docs or a default-policy change for how WhatsApp `dmPolicy="pairing"` interacts with `allowFrom` for DMs.

## Best Possible Solution

Keep the current contract: configured `allowFrom` pre-authorizes WhatsApp DMs, pairing approvals add more DM senders, and `allowlist` mode remains an explicit stricter choice.

## Reproduction Assessment

No. Current-source inspection and checked-in WhatsApp inbox coverage show that a sender in `channels.whatsapp.allowFrom` is accepted under the default `pairing` policy, so the reported drop path is not reproducible from current main.

## Solution Assessment

Yes. Closing as cannot reproduce is the best cleanup because documenting that allowlisted WhatsApp numbers still need pairing would contradict the current resolver, tests, and published docs.
