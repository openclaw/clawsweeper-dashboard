## Summary

`openclaw-weixin` is not a bundled/core OpenClaw plugin on current `main`, and the current bundled `MEDIA:` outbound path already preserves per-file filenames through `loadWebMedia(...) -> saveMediaBuffer(..., originalFilename)`. This makes the report plugin-scope rather than an `openclaw/openclaw` core issue.

## Evidence

- **Vision keeps optional channels in plugins, not core:** `VISION.md:54-73` says optional capability should usually ship as plugins, channel/runtime extension points belong in plugins, plugin ownership should live in the plugin's own repository, and promotion/discovery belongs in ClawHub.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/VISION.md#L54)

- **Plugin docs distinguish bundled vs external plugins:** `docs/tools/plugin.md:11-15` states some plugins are core while others are external community packages. `docs/tools/plugin.md:71-72` says external plugins must be installed through `openclaw plugins install`. The official installable/core plugin lists at `docs/tools/plugin.md:88-119` do not include any `weixin` plugin.
  - file: [docs/tools/plugin.md:11](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/docs/tools/plugin.md#L11)

  - command: `find extensions -mindepth 1 -maxdepth 1 -type d | sed 's#^extensions/##' | sort`
