## Summary

#53339 is a plugin-specific instance of the same root `stringEnum` plugin-SDK removal already tracked by open #52842. Current `main` has not restored `stringEnum` on the bare root SDK surface, but the helper exists on focused compatibility subpaths, so this issue should close as duplicate rather than as fixed.

## Best Possible Solution
