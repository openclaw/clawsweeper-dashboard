## Summary

Keep open. Current `main` and the latest release still do not implement the draft-preview/block-streaming mutual exclusion described in the PR, so this is not `implemented_on_main`. At the same time, the PR's teardown guard is not strong enough to justify closing it as obsolete: under current `SlackStreamSession` types it appears dead, and it would bypass the existing `stopped` state transition if that type ever changed.

## Evidence

- **Main still allows preview draft updates without forcing block streaming off:** `dispatch.ts` enables draft preview updates via `onPartialReply` when `previewStreamingEnabled` is true and native streaming is off, but `disableBlockStreaming` only checks `useStreaming` and the account block-streaming flag. There is no Slack-specific mutual exclusion for preview streaming here, so both paths can still be active together.
