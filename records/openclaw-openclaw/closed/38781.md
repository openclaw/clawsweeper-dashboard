## Summary

Current main has shipped the user-visible fix for this PR: `exec host=node` no longer fails when a macOS node advertises `system.run` without `system.run.prepare`. The Swift macOS node still does not declare or dispatch `system.run.prepare`, but the agent exec path now routes on declared node commands, builds a local `systemRunPlan` when prepare is absent, and forwards `system.run` with that plan. That fallback shipped in `v2026.4.26` and supersedes this PR’s direct Swift implementation approach.

## Best Possible Solution

Close this PR as implemented/superseded by the shipped `host=node` fallback on current main. Keep the v2026.4.26 agent-side behavior, and only revisit a direct Swift `system.run.prepare` implementation in a new maintainer-reviewed macOS exec parity PR if node-side canonical preparation is still desired.
