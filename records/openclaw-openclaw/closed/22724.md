## Risks / Open Questions

- The proposal bundles two pieces: ClawHub-side aggregation/UI and an OpenClaw-side runtime hook. If maintainers want the latter later, it should be tracked as a separate narrowly scoped core issue.

## Close Comment

Closing this as `clawhub` after a Codex review. Current `main` treats ClawHub as an external registry/discovery surface rather than the place for marketplace-quality telemetry inside OpenClaw core: `src/gateway/server-methods/skills.ts:132` only exposes `skills.search`, `skills.detail`, and `skills.install`, `src/infra/clawhub.ts:123` only models skill summary/version/platform/owner metadata, and `.github/workflows/plugin-clawhub-release.yml:27` points ClawHub publishing at the separate `openclaw/clawhub` repository. `VISION.md:55` also keeps optional capability out of core where possible. The telemetry/ratings idea is useful, but the aggregation, ranking, and review UI belong on ClawHub rather than as an OpenClaw core cleanup item. If you want to pursue it, please open it in the ClawHub project; if that later needs a small OpenClaw runtime hook, that can be tracked separately.
