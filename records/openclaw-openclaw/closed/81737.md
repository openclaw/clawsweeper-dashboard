## Summary

Keep open: current main and the latest release still preserve final-only `--json` stdout, while the requested stdout NDJSON event stream would add a new public CLI/API contract.

## What This Changes

Request an optional `--stream` or `--event-stream` mode for `openclaw agent --json` that emits real-time NDJSON events on stdout while preserving existing final-only JSON behavior by default.

## Best Possible Solution

Design and document an additive `--stream` or `--event-stream` CLI contract that emits redacted NDJSON for both Gateway and local runs while keeping plain `--json` final-only by default.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection confirms the current behavior: there is no stream flag, and `--json` emits a terminal JSON response only.

## Solution Assessment

Unclear until maintainers choose the public event schema. The requested direction is plausible, but the safest path is a designed CLI contract rather than exposing raw internal event shapes directly.
