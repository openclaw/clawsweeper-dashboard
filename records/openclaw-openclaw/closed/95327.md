## Summary

Keep this PR open, but the patch is not merge-ready because the new context-engine override can undercount prompt-local runtime context that is still sent to the model, and the external PR only shows unit-test proof rather than after-fix real behavior proof.

## What This Changes

The PR wires positive `AssembleResult.estimatedTokens` from a context engine into the embedded-runner pre-prompt overflow check and adds a CJK-heavy unit test for the override path.

## Best Possible Solution

Keep the context-engine estimate path, but compute the final precheck pressure from the exact model-boundary payload by adding any prompt-local runtime-context/custom message cost and covering it with a runner-level regression plus real after-fix proof.

## Reproduction Assessment

Yes for the review finding by source inspection: current main estimates `hookMessagesForCurrentPrompt`, which includes prompt-local runtime context, while the PR override only adds system and prompt overhead to the context-engine message estimate. I did not run a live CJK/lossless-claw repro.

## Solution Assessment

No. The direction is plausible, but the best fix must preserve current main's full provider-boundary accounting and add runner-level coverage plus real after-fix proof.
