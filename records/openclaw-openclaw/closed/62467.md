## Summary

Current main and the v2026.4.24 release already remove the reported script-preflight rejection from the default gateway YOLO exec path (`host=gateway`, `security=full`, `ask=off`). The old complex-interpreter preflight still exists for tightened approval-gated modes, but the reported default/skill automation problem and the workaround discussed in the comments are implemented on main.

## Best Possible Solution

Keep the shipped YOLO/default exec behavior and its docs/tests. Do not add a broad `cd && python` bypass to approval-gated preflight without a separate security design; any remaining approval-mode ergonomics should be tracked as a narrower follow-up around workdir-aware approval parsing.
