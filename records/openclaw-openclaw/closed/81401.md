## What This Changes

The branch adds an `agents.defaults.agentRuntime` fallback to `resolveModelRuntimePolicy` and adds unit tests for that fallback precedence.

## Best Possible Solution

Keep the shipped provider/model-scoped runtime policy unless maintainers explicitly decide to revive defaults inheritance; use https://github.com/openclaw/openclaw/pull/81421 as the canonical branch for that decision.

## Reproduction Assessment

Yes for the observed behavior: current main's resolver ignores `agents.defaults.agentRuntime`, and a current test locks that legacy key as ignored. It is not a high-confidence bug reproduction because current docs define that behavior as intentional legacy handling.

## Solution Assessment

No. This branch reintroduces a legacy whole-agent key and mislabels the source as `provider`; if maintainers choose to restore inheritance, the broader source-propagation path in https://github.com/openclaw/openclaw/pull/81421 is the safer starting point.
