## Summary

Close as ClawHub/plugin scope. The linked implementation PR was closed unmerged with maintainer guidance that this should not become a new core `memorySearch.recallBeforeResponse` setting, and current main already has plugin hooks plus bundled Active Memory and LanceDB memory plugins for pre-reply recall and context injection.

## What This Changes

Request to add a runtime-enforced rule or memory recall check before agent responses so recalled rules can be injected and failed recall can block generation.

## Best Possible Solution

Leave the core request closed and route tuned rule-checking or recall-before-reply behavior through ClawHub plugins; if a plugin cannot express a specific fail-closed workflow, open a focused plugin API design issue instead of adding a memory-specific core setting.
