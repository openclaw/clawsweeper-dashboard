## Summary

Close as superseded by prior Discord channel-override discussions: #466, #618, and #2614 were closed after a MEMBER explicitly kept the implicit channel allowlist behavior and pointed users to the `"*"` channel entry for allow-all-plus-overrides. Current main still implements and tests that model, so this issue is not a separate remaining core bug.

## What This Changes

The issue requests changing Discord guild `channels` entries from an implicit allowlist into additive per-channel overrides where omitted channels inherit guild defaults.

## Best Possible Solution

Keep the current documented implicit channel allowlist behavior and use `channels.discord.guilds.<id>.channels["*"]` for allow-all defaults with specific channel overrides; revisit semantics only through a maintainer-approved Discord access-control redesign.

## Reproduction Assessment

Yes. A source-level reproduction is a guild config with `groupPolicy: "open"`, guild-level `requireMention: false`, and a non-empty `channels` map that omits the incoming channel and has no `"*"` entry; current main resolves it to `{ allowed: false }` and preflight drops it.

## Solution Assessment

Yes for cleanup. Closing to the prior maintainer decision is the narrowest outcome because current main supports the intended allow-all-plus-overrides shape through a wildcard channel entry, while a global semantic flip would broaden existing access-control behavior.
