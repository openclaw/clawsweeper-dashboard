## Summary

Current `main` already resolves gateway image support from the model catalog and configured provider metadata, so custom providers with `models.providers.<id>.models[].input` including `"image"` are no longer gated by a fixed provider allowlist. I did not find a release tag containing the fixing commit yet, so this appears fixed on `main` but not yet in a tagged release.

## Evidence

- **Gateway image support now comes from catalog metadata:** `resolveGatewayModelSupportsImages()` loads the gateway model catalog, finds the matching provider/model entry, and returns true when that catalog entry declares `input` containing `image`. The remaining special cases are stale-metadata shims for Foundry and Claude CLI, not a general provider whitelist.
