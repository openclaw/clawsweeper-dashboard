## Summary

Current main already supports OpenRouter provider pinning/routing through the existing model-config param path: provider-specific `params` are merged into runtime extra params, the OpenRouter plugin reads `params.provider` and forwards it as routing metadata, the behavior is documented, and it was already shipped in `v2026.4.23`.

## Evidence

- **Model entries already allow provider-specific params:** `AgentModelEntryConfig` documents `params` as provider-specific API parameters and types it as an open `Record<string, unknown>`, so nested routing objects are accepted in model config.
  - file: [src/config/types.agent-defaults.ts:31](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/config/types.agent-defaults.ts#L31)
  - command: `nl -ba src/config/types.agent-defaults.ts | sed -n '1,80p'`
  - sha: [245451b6a9c5](https://github.com/openclaw/openclaw/commit/245451b6a9c529464e177afa65013adca0d63eb5)
- **Configured params are merged into runtime extra params:** `resolveExtraParams()` merges `agents.defaults.params`, per-model `agents.defaults.models["provider/model"].params`, and per-agent overrides, then returns them as runtime extra params.
  - file: [src/agents/pi-embedded-runner/extra-params.ts:66](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/agents/pi-embedded-runner/extra-params.ts#L66)

- **OpenRouter plugin consumes `params.provider`:** `wrapOpenRouterProviderStream()` reads `ctx.extraParams.provider` when it is an object and injects it as `compat.openRouterRouting`, which is the OpenRouter routing metadata path.
  - file: [extensions/openrouter/stream.ts:30](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/extensions/openrouter/stream.ts#L30)

- **Regression test covers provider routing:** The OpenRouter plugin test passes `extraParams.provider = { order: ["moonshot"] }` and asserts that the wrapped model receives `compat.openRouterRouting.order`, proving the config path is exercised.
  - file: [extensions/openrouter/index.test.ts:88](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/extensions/openrouter/index.test.ts#L88)

## Risks / Open Questions

- The reporter asked for a friendlier alias like `providerPreference` or `preferDirect`; current shipped behavior uses the raw upstream OpenRouter field name `params.provider`, so discoverability is lower even though the capability exists.
