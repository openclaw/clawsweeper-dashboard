## Summary

Keep open: current main still does not resolve a target ACP agent's configured primary model when sessions_spawn omits an explicit model, and the acpx dependency contract also requires OpenClaw to map that model into sessionOptions before the delegate can apply it.

## What This Changes

The issue asks OpenClaw to honor per-agent model.primary when spawning ACP runtime sessions and to forward the resolved model into acpx session creation.

## Best Possible Solution

Resolve the target ACP agent's configured primary model when no explicit spawn model is supplied, then map the resulting model into the acpx sessionOptions contract before session creation.

## Reproduction Assessment

Yes, from source inspection: an ACP sessions_spawn without a model passes undefined through the ACP spawn path, and acpx only applies requested models from sessionOptions.model. I did not run a live ACP process in this read-only review.

## Solution Assessment

Yes, the maintainable fix is to reuse existing configured-model resolution instead of inventing another model policy path, then bridge OpenClaw's model field to acpx sessionOptions.
