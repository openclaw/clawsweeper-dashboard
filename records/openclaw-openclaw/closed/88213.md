## What This Changes

The issue reports that Chrome MCP existing-session browser operations can fail with an implicit 300ms attach timeout even when browser doctor reports the profile as reachable.

## Best Possible Solution

Implement a narrow timeout-propagation fix for Chrome MCP existing-session tabs, tab selection, and snapshot setup paths, while leaving broader reconnect and product timeout policy to https://github.com/openclaw/openclaw/issues/68779.

## Reproduction Assessment

Yes at source level, but not with a live browser in this review. Current main shows /tabs calling isReachable(300), and Chrome MCP turns that budget into the reported attach timeout when first attach/list-pages exceeds 300ms.

## Solution Assessment

Yes for the narrow bug: forwarding the caller or route timeout into Chrome MCP list-tabs and tab-selection setup is the maintainable fix. Raising constants or adding config should be avoided unless maintainers separately choose a broader timeout policy.
