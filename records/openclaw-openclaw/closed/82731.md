  - body: When a session has `contextTokens` but no total/input/output counters, returning `0` makes the footer say no tokens were used even though usage is unavailable or stale. The prior related PR at https://github.com/openclaw/openclaw/pull/43044 was closed around this contract, so this path should keep unknown usage unknown and fix the session data source instead.
  - confidence: 0.92
- **[P2] Restore the RTL fixtures before adding token-total tests:** `src/tui/tui-formatters.test.ts:364-367`
  - body: The PR replaces Arabic RTL strings with ASCII question marks while keeping expectations for RTL isolation. `sanitizeRenderableText` only wraps text matching `RTL_SCRIPT_RE`, so these cases would either fail or stop covering the RTL behavior they name.
  - confidence: 0.94
- **[P3] Keep the box-drawing fixture in the table-rule test:** `src/tui/tui-formatters.test.ts:476`
  - body: This test protects horizontal table rules made from box-drawing characters. Replacing `─` with `?` makes it cover generic punctuation instead of the table rendering case.
  - confidence: 0.86
