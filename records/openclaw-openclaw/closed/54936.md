## Summary

Current `main` already honors `agents.defaults.subagents.runTimeoutSeconds` when `sessions_spawn` omits `runTimeoutSeconds`, and the same behavior is present in release `v2026.4.22`.

## Evidence

- **Current spawn path resolves config default before registration:** `spawnSubagentFromTool` loads config and resolves omitted `runTimeoutSeconds` via `resolveConfiguredSubagentRunTimeoutSeconds(...)` before registering the subagent run.
  - file: [src/agents/subagent-spawn.ts:594](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/agents/subagent-spawn.ts#L594)

- **Resolver uses configured default when override is absent:** `resolveConfiguredSubagentRunTimeoutSeconds` returns the configured `agents.defaults.subagents.runTimeoutSeconds` when no explicit timeout is supplied, otherwise falls back to `0` only if config also omits it.
