## Summary

Current `main` now auto-enables configured bundled chat channels under restrictive `plugins.allow`, feeds that auto-enable pass through both doctor and runtime startup, and documents/tests the behavior. The reported outage path no longer matches `main`, although the fix appears unreleased as of `v2026.4.23`.

## Evidence

- **Auto-enable detects configured channels:** `resolveConfiguredPluginAutoEnableCandidates` now turns configured channels into `channel-configured` auto-enable candidates, which is the missing path behind the original report.
  - file: [src/config/plugin-auto-enable.shared.ts:463](https://github.com/openclaw/openclaw/blob/0783090da0090982c19f7803a9daf9a2ec29538e/src/config/plugin-auto-enable.shared.ts#L463)

  - file: [src/config/plugin-auto-enable.shared.ts:596](https://github.com/openclaw/openclaw/blob/0783090da0090982c19f7803a9daf9a2ec29538e/src/config/plugin-auto-enable.shared.ts#L596)
