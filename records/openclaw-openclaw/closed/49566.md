## Summary

Keep open. Current `main` does not contain the same-provider embedded model normalization or downstream runtime-model-id propagation described in PR #49566, and the PR remains a plausible bug fix for #48876 rather than an item that is already implemented or safely auto-closable.

## Evidence

- **No generic same-provider normalization on main:** `normalizeStaticProviderModelId` only special-cases Anthropic, Hugging Face, Google, OpenRouter, xAI, and Vercel AI Gateway. There is no generic branch that strips a repeated provider prefix such as `polza/gpt-4o-mini` for a custom provider, so the PR's core behavior is not present on current `main`.

- **Configured override lookup still misses same-provider qualified refs:** Embedded model resolution still looks up configured overrides by exact `modelId`, then optionally by discovered `discoveredModel.id`. For a request like `polza/gpt-4o-mini` against a configured model `gpt-4o-mini`, both `applyConfiguredProviderOverrides` and `resolveConfiguredFallbackModel` can miss the intended config row and keep the prefixed runtime id/header set. That matches the bug described in the PR body.
