## Summary

Current main already implements the Windows Telegram loader fix reported in #72318. The failing 2026.4.24 policy allowed native Jiti on Windows, while current main disables native Jiti on win32 through the shared plugin-loader helper and has focused regression coverage for bundled Telegram dist sidecars.

## Best Possible Solution

Close #72318 as implemented on current main. Keep the shared Windows plugin-loader policy that disables native Jiti on win32; affected 2026.4.24 Telegram users should upgrade to the next release containing this main-branch fix rather than tracking another duplicate issue.
