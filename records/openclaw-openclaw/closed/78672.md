## What This Changes

The issue reports that @tencent-weixin/openclaw-weixin@2.4.1 crash-loops on OpenClaw 2026.5.6 because the monitor waits for a module-global runtime instead of using the channel worker context.

## Best Possible Solution

Keep the remaining work consolidated in https://github.com/openclaw/openclaw/issues/78376, then coordinate either a core loader compatibility fix or an external Weixin plugin release that passes ctx.channelRuntime into the monitor and removes the manual Content-Length header.
