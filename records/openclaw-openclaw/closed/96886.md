## Summary

Keep open: current main and v2026.6.10 still have unbounded fal music/video success JSON reads, and this focused patch uses the existing bounded provider JSON reader with sufficient loopback proof. The remaining blocker is maintainer acceptance of the compatibility tradeoff from fail-closed 16 MiB successful-response metadata caps.

## What This Changes

This PR routes fal music and video successful JSON parsing through `readProviderJsonResponse` and updates the provider tests to use real `Response` bodies.

## Best Possible Solution

## Solution Assessment

Yes. Reusing `readProviderJsonResponse` is the narrow maintainable fix because it applies the existing SDK byte cap at the affected fal success reads without adding fal-local parsing, configuration, or a second cap mechanism.
