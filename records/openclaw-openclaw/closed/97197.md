## Best Possible Solution

Land the narrow prompt hint after normal CI and maintainer review; keep delimiter-to-tag conversion as a separate product decision.

## Reproduction Assessment

No live current-main model run was performed, but the source and API path is clear: current main says only `formulas`, while Telegram rich messages and OpenClaw's formatter use `<tg-math>` and `<tg-math-block>`.

## Solution Assessment

Yes. This is the narrowest maintainable fix for the prompt gap because the rich formatter already supports the target tags, while delimiter parsing would be broader and easier to misapply to code or currency text.
