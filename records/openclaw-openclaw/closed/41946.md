## Summary

Keep open. Current main still sends Feishu non-card text through a markdown post element with only table conversion, so the underscore URL behavior is not implemented. This PR is also not mergeable as submitted: the diff imports a helper that is not exported, uses a URL regex that can break links with parentheses, and only targets markdown-link URLs while the linked bug reproduces with a bare URL. The related #41880 remains an open implementation candidate, not shipped main behavior.

## Best Possible Solution

Keep this PR open only as a rework candidate, or consolidate it with the more direct #41880 path. The correct Feishu-owned fix should cover the bare URL reproduction from #41860 and any intended markdown-link case without corrupting parentheses or escaped characters, add focused send/edit regression tests, and then close the linked bug after the corrected implementation lands on main.
