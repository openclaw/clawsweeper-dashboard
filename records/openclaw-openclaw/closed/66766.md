## Summary

Keep open: current `main` still has the 200k synchronous session/status resolver path for Anthropic Opus/Sonnet 4.6 when `context1m` is absent, and the closing PR #66862 remains open and unmerged.

## What This Changes

The issue asks OpenClaw to show the 977k/1M Anthropic Opus/Sonnet 4 session context limit without requiring users to set `params.context1m: true`.

## Best Possible Solution

Land #66862 or an equivalent narrow patch that aligns resolver, session/status denominator, Control UI display, tests, and Anthropic docs while preserving explicit opt-out and auth-specific fallback behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the current resolver fixture for `anthropic/claude-opus-4-6` with empty params and `allowAsyncLoad: false` expects 200k, and `/status` plus session persistence call that same synchronous resolver path.

## Solution Assessment

Yes for the direction, but this issue should stay open until the linked fix lands. The narrow maintainable solution is resolver/status alignment that does not accidentally promise 1M request behavior for auth modes where the Anthropic beta header is unavailable.
