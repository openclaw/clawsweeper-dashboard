## Summary

Keep open. Current main still lacks DeepSeek V4 `reasoning_content` replay for OpenRouter-routed DeepSeek models: the native DeepSeek path and shared wrapper exist, but OpenRouter only applies proxy reasoning and Anthropic prefill wrappers. Related #75105/#75619 are useful context, but no merged fix currently covers the bundled OpenRouter route.

## What This Changes

OpenRouter-routed `deepseek/deepseek-v4-flash` should get the same DeepSeek V4 follow-up/tool-turn `reasoning_content` replay handling as the native DeepSeek provider.

## Best Possible Solution

Add a provider-owned or model-compat-driven DeepSeek V4 replay hook for OpenRouter-routed `deepseek/deepseek-v4-*` models, with focused payload tests for tool-call and plain follow-up turns plus docs/changelog.

## Reproduction Assessment

Yes. The issue has a high-confidence source-level reproduction: an OpenRouter DeepSeek V4 model resolves as provider `openrouter`, so the native DeepSeek V4 provider guard and replay wrapper do not run on current main.

## Solution Assessment

Yes, the requested direction is valid, but the safest implementation should reuse the existing shared DeepSeek V4 wrapper and align with the open #75105 model-compat path rather than duplicating provider-specific replay code.
