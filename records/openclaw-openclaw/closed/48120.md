## Summary

Current `main` already matches plugin HTTP routes case-insensitively at gateway dispatch, so the proposed LINE-specific `.toLowerCase()` route registration change is unnecessary and the reported path-case 404 does not reproduce from the current code.

## Evidence

- **LINE monitor still registers the configured path directly:** `extensions/line/src/monitor.ts` registers `normalizePluginHttpPath(webhookPath, "/line/webhook") ?? "/line/webhook"` without lowercasing, so if case mismatches were still a live bug they would still be visible on `main`.

  - file: [src/gateway/security-path.ts:34](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/src/gateway/security-path.ts#L34)
