## Summary

Keep open. Current `main` still has the schema/type mismatch this PR targets: the exec approvals runtime can persist `source: "allow-always"`, but the gateway protocol schema for `exec.approvals.set` / `exec.approvals.node.set` does not accept `source` and is strict about additional properties. The Control UI type also still lacks both `argPattern` and `source`. The related PRtags duplicate context is real, but the linked successor #60073 is also open and #69575 is closed unmerged, so there is no shipped or canonical merged replacement to close this against.

## Best Possible Solution

Keep the item open as a valid focused fix candidate. The best maintainer path is to land or port the minimal protocol/UI alignment: add `source?: "allow-always"` to `ExecApprovalsAllowlistEntrySchema`, keep the Control UI allowlist type aligned with runtime metadata including `argPattern`, and add validator coverage for both gateway and node set params plus an invalid-source rejection test. After one canonical fix lands, close the remaining duplicate PRs and the linked bug #60000 as implemented.
