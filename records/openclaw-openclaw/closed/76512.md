## Summary

Current main and the stable v2026.5.3 release implement the requested `/steer <message>` behavior: top-level `/steer` now targets the current session active run instead of being parsed as a sub-agent id.

## What This Changes

A Telegram text-command regression report asks for `/steer <message>` to steer the active agent session without requiring a sub-agent id.

## Best Possible Solution

Keep the shipped current-session `/steer` command behavior; if a live Telegram case still errors on a supported release, it should be filed as a new focused regression with the exact command, channel/session context, and logs.

## Reproduction Assessment

No high-confidence failing current-main reproduction remains. I did not run live Telegram, but current source and focused tests show top-level `/steer` is claimed by the current-session handler and no longer parsed as a sub-agent target.

## Solution Assessment

Yes. The merged shared command handler is the best fix because it corrects the common text-command surface for Telegram and other channels without adding a Telegram-only workaround.
