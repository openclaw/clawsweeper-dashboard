## Summary

Close as implemented: current main supports argument-level exec allowlist matching via the documented `argPattern` regex field, with cross-platform matcher coverage and a merged fix. The exact proposed `argsPattern` glob spelling was not used, but the central capability is present.

## What This Changes

Request to let exec allowlist entries match a binary plus specific argv so restricted agents can allow one trusted script invocation without allowing the whole interpreter.

## Best Possible Solution

Keep the current `argPattern` regex contract; any future glob alias, CLI/UI editor enhancement, or `argsPattern` compatibility request should be tracked separately.

## Reproduction Assessment

Not applicable as a feature request. The original missing capability is now verifiable from source and tests by matching a path/bare executable plus `argPattern` against parsed argv on Linux, macOS, and Windows.

## Solution Assessment

Yes. The implemented central matcher change is the narrow maintainable fix because it preserves path-only entries while allowing operators to omit broad path-only interpreter entries when they want argv-scoped trust.
