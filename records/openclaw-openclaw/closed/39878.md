## Summary

Current `main` already removes plugin-owned channel config during uninstall and has regression coverage for the reported case, so issue #39878 is implemented on `main` but not yet in the latest tagged release.

## Evidence

- **Uninstall removes channel config before config write:** `removePluginFromConfig` now deletes owned `channels.<id>` entries for installed plugins before returning the next config, preventing stale unknown-channel references during uninstall.
