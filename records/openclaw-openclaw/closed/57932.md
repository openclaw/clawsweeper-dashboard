## Summary

Current main, and the shipped v2026.4.24 tag, solve the reported sandbox-off `workspaceOnly` native-vision failure through centralized managed-inbound-media handling rather than by merging this PR's workspace-copy fallback. Inbound `media://inbound/<id>` refs and first-level absolute paths under the managed inbound media store are now allowed through the media loader before local-root checks, and regression coverage proves native prompt images load under `tools.fs.workspaceOnly: true`.

## Best Possible Solution

Close this older PR and keep the shipped managed-media implementation. It fixes the native vision and media-tool failure without copying every sandbox-off inbound file into the workspace; any remaining request for generic read-tool access to inbound media should be tracked separately with explicit maintainer security review.
