Fix https://github.com/openclaw/openclaw/issues/77645 in one focused memory-core PR. Current behavior: with `memory.backend = "qmd"` and `memory.qmd.searchMode = "search"`, `QmdMemoryManager.probeVectorAvailability()` intentionally returns false without probing vectors, but `probeEmbeddingAvailability()` turns that into `QMD semantic vectors are unavailable`, and `openclaw memory status --deep` prints `Embeddings: unavailable` plus `Embeddings error`. Expected behavior: lexical/BM25-only QMD mode should be shown as skipped/not applicable or should suppress the error, while true vector/embedding failures for `query` and `vsearch` still report unavailable with useful errors. Prefer a narrow owner-boundary fix in memory-core; if adding optional skipped-state fields to `MemoryEmbeddingProbeResult`, update both memory host SDK and plugin SDK facade types plus focused renderer tests. Do not change QMD search behavior, add config/env flags, alter fallback behavior, or edit CHANGELOG.md. Preserve the documented contract that `searchMode: "search"` skips semantic vector readiness probes and embedding maintenance.
