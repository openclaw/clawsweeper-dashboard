## Risks / Open Questions

- This is implemented as Task Flow plus webhook ingress/egress, not as a native cron job `dependsOn`/`onComplete` field, so the exact requested API shape still does not exist.

## Close Comment

Closing as implemented on `main` after Codex review. The underlying completion-driven pipeline use case is now covered by OpenClaw's Task Flow and webhook surfaces rather than by adding a new cron `dependsOn`/`onComplete` field. Current docs explicitly direct multi-step pipelines to Task Flow (`docs/automation/taskflow.md:19`, `docs/automation/index.md:27`), managed flows can spawn child tasks (`src/tasks/task-executor.ts:485`), the bundled webhooks plugin lets external systems create flows and enqueue child tasks (`docs/plugins/webhooks.md:11`, `docs/plugins/webhooks.md:112`, `extensions/webhooks/index.ts:20`, `extensions/webhooks/src/http.ts:122`), and cron can already emit finished-run webhooks for handoff triggers (`src/gateway/server-cron.ts:419`). The changelog shows this shipped in `v2026.4.7` on top of the Task Flow substrate added in `v2026.4.2` (`CHANGELOG.md:1039`, `CHANGELOG.md:1500`).
