## Summary

Keep this PR open. Current main still has the Mattermost inbound-post path enqueueing every ordinary user post as a system event immediately before also sending the same content through `formatInboundEnvelope`, so the PR's intended fix is not implemented on main and remains a plausible focused implementation for #71795.

## Best Possible Solution

Review and land this PR or an equivalent Mattermost-owned change that removes the per-message system-event enqueue while preserving `formatInboundEnvelope` metadata. Best follow-up is focused regression coverage proving ordinary inbound posts do not call `enqueueSystemEvent`, while reactions and slash/interactions still can.
