## Summary

Keep open. Current main contains several adjacent post-report mitigations for slow model catalog calls, Telegram polling timeout churn, and packaged runtime-deps repair loops, but those changes are narrower than the reported Linux Gateway CPU saturation with Telegram-triggered stalls, WS/RPC timeouts, stuck heartbeat processing, and cron timeout. The discussion also includes a fresh confirmation that downgrading to 2026.4.20 avoids the problem while 2026.4.29 beta still reproduced it, so there is not enough source, test, release, or live-profile evidence to close this as fixed.

## What This Changes

The issue reports a Linux Gateway regression where CPU saturates, WebSocket/RPC calls time out or close, and agent/cron work stalls after upgrading from 2026.4.20 to 2026.4.23 through 2026.4.29 beta.

## Best Possible Solution
