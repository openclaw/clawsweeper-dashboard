## Summary

Current `main` already contains a targeted fix for the persisted live-model-switch retry loop behind this report: when a restored session requests a different persisted model during fallback, `LiveSessionModelSwitchError` is now normalized into failover handling instead of being re-thrown into the outer retry path. Current tests also cover the all-providers-fail case and assert a terminal `All models failed ...` error instead of endless retries.

## Evidence
