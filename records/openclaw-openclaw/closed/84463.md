## Summary

Close as implemented: current main and v2026.5.18 already prevent inherited main OAuth credentials from being persisted into secondary agent stores and route shared refreshes through the main owner, while the reported 2026.4.5 full-store write path is gone.

## What This Changes

The issue reports that per-agent OAuth profile persistence duplicates shared upstream credentials, causing single-use refresh tokens to race across agents.

## Best Possible Solution

Keep the shipped main-store OAuth owner and refresh behavior; users still seeing this should upgrade to v2026.5.18 or newer and file a fresh report with v2026.5.18+ logs if it reproduces.

## Reproduction Assessment

No high-confidence failing reproduction exists against current main: source inspection shows the reported v2026.4.5 full-store save path has been replaced by inherited-main filtering, main-owner refresh writes, and targeted regression tests. I did not run tests because this was a read-only checkout review.

## Solution Assessment

Yes. The current implementation is the maintainable fix shape because it avoids cloning shared OAuth refresh tokens, resolves the canonical owner before refresh, and preserves per-agent overrides when they are genuinely local.
