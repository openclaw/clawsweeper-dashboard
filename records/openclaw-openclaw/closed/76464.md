## Summary

Keep this issue open because current main still omits `systemPrompt` from the local `model.run` simple-completion context and still collapses empty error results into a generic no-text error; two open closing PRs already target the same bug, so the issue should remain open until one lands.

The issue asks for local `openclaw infer model run` to send instructions for `openai-codex/gpt-5.5` and surface provider errors when the local completion returns no text.

## Best Possible Solution
