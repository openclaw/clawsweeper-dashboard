## Summary

Current main addresses the issue’s central gateway cold-start problem: Bedrock’s AWS SDK is lazy-loaded instead of imported at plugin registration/module load, gateway startup now scopes plugin loading through the persisted cold registry instead of broadly importing default-enabled provider plugins, and allow/deny policy is evaluated before runtime imports for disabled plugins. The related #71739/#71746 CLI item was a narrower sibling fix, not the remaining canonical work for this gateway report.

## Best Possible Solution

Close as implemented on main. Keep the shipped cold-registry and lazy Bedrock import approach; if operators still see gateway cold-start misses, open a new narrow issue with fresh current-main timings and the specific remaining hot path.
