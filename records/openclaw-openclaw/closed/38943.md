## Summary

Current main already handles Feishu streaming cards by merging cumulative snapshots instead of blindly appending them, and the same implementation is present in release v2026.4.22.

## Evidence

- **Reply dispatcher treats Feishu partials as snapshots:** `onPartialReply` feeds streaming text through `queueStreamingUpdate(..., { dedupeWithLastPartial: true, mode: "snapshot" })`, so repeated cumulative chunks are merged rather than appended.
  - file: [extensions/feishu/src/reply-dispatcher.ts:495](https://github.com/openclaw/openclaw/blob/c971c58fc7592344539ac3cf87c7c4229be9978e/extensions/feishu/src/reply-dispatcher.ts#L495)
  - sha: [c971c58fc759](https://github.com/openclaw/openclaw/commit/c971c58fc7592344539ac3cf87c7c4229be9978e)
- **Streaming card session merges overlaps and skips unchanged updates:** `mergeStreamingText` prefers the newer full snapshot when it already contains prior text, and `update`/`close` avoid sending another card update when the merged text matches the current card state.
  - file: [extensions/feishu/src/streaming-card.ts:115](https://github.com/openclaw/openclaw/blob/c971c58fc7592344539ac3cf87c7c4229be9978e/extensions/feishu/src/streaming-card.ts#L115)
  - sha: [c971c58fc759](https://github.com/openclaw/openclaw/commit/c971c58fc7592344539ac3cf87c7c4229be9978e)

  - file: [extensions/feishu/src/streaming-card.test.ts:4](https://github.com/openclaw/openclaw/blob/c971c58fc7592344539ac3cf87c7c4229be9978e/extensions/feishu/src/streaming-card.test.ts#L4)
