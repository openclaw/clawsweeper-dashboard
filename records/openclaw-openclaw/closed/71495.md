## Summary

Current main and the shipped v2026.4.27 release now route native `/subagents list` commands through the command target session instead of the isolated slash-command session, while the list path reads the merged persisted/in-memory subagent registry. That addresses the reported empty-list regression from v2026.4.23/v2026.4.24 for native slash-command surfaces.

## What This Changes

The issue requests that `/subagents list` show active and recently completed subagents after a spawn instead of returning empty sections on v2026.4.23/v2026.4.24.

## Best Possible Solution

Keep the shipped v2026.4.27 native-command target-session routing and merged registry snapshot behavior. Future work should add a package/QA scenario only if a user can still reproduce empty `/subagents list` output on v2026.4.27 or newer.

## Reproduction Assessment

Yes. The old failure path is well specified: spawn subagents in the routed chat session, then invoke native `/subagents list` from an isolated slash-command session; current main has focused resolver coverage for that command-target selection.

## Solution Assessment

Yes. The current fix is the narrow maintainable solution for the observed empty list: keep command ownership filtering intact, but resolve native slash commands to their target chat session and read the shared persisted registry snapshot.
