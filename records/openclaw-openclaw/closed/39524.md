## Summary

PR #39524 is obsolete for its stated `commands.allowFrom` startup-failure fix. Current main still keeps the documented provider-keyed record contract, but the user-visible failure from the linked bug is now handled by last-known-good config recovery, and that recovery is shipped in the latest release.

## Best Possible Solution

Close this PR and keep the shipped direction: record-shaped `commands.allowFrom`, docs that show the `"*"` global fallback, and last-known-good recovery for invalid edits. If maintainers want flat-array shorthand as a public config contract later, it should be a fresh focused change covering schema, types, runtime lookup, docs, and tests; the unrelated Feishu media config work should be handled separately.
