## Summary

Current `main` already routes QQ Bot CLI and proactive/cron sends through the media-tag-aware outbound path, and the same implementation is present in the latest release `v2026.4.23`. The reported markdown-only bypass does not match the shipped code on `main`.

## Evidence

- **CLI send keeps message text as a string and forwards it through outbound send flow:** `handleSendAction` reads `--message` as text, parses reply/media directives, and passes the final string into `executeSendAction` instead of a QQ-specific markdown-only API shortcut.
  - file: [src/infra/outbound/message-action-runner.ts:491](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/infra/outbound/message-action-runner.ts#L491)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **QQ Bot plugin outbound adapter uses `sendText` for CLI/core sends:** The QQ Bot channel plugin wires outbound `sendText` and `sendMedia` directly to `extensions/qqbot/src/engine/messaging/outbound.ts`, so `openclaw message send --channel qqbot ...` goes through the QQ Bot outbound implementation.
  - file: [extensions/qqbot/src/channel.ts:104](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/extensions/qqbot/src/channel.ts#L104)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **QQ Bot proactive outbound path parses inline media tags:** `sendText(...)` normalizes media tags, matches `<qqimg>`, `<qqvoice>`, `<qqvideo>`, `<qqfile>`, and `<qqmedia>`, builds an ordered send queue, and dispatches text/media parts in sequence.
  - file: [extensions/qqbot/src/engine/messaging/outbound.ts:793](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/extensions/qqbot/src/engine/messaging/outbound.ts#L793)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Cron/proactive queue path reuses the same media-tag-aware sender:** `sendCronMessage(...)` falls through to `sendText(...)` for both decoded cron payloads and plain text, so proactive queued delivery uses the same tag parser instead of a separate markdown-only path.
  - file: [extensions/qqbot/src/engine/messaging/outbound.ts:1195](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/extensions/qqbot/src/engine/messaging/outbound.ts#L1195)
