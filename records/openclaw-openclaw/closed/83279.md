## What This Changes

The PR adds an optional heartbeat `hint`, threads it through skipped heartbeat events, and renders that hint in `openclaw status --deep` while preserving `reason: "target-none"`.

## Best Possible Solution

Land the additive hint/status rendering change after maintainer review and normal CI, while keeping `target-none` as the machine-readable reason.

## Reproduction Assessment

Yes. Current main source shows the `target=none` resolver emits `reason: "target-none"` without a hint, and the status formatter falls back to `unknown` when the skipped event has no channel.

## Solution Assessment

Yes. Keeping the stable `target-none` reason and adding an optional human-readable hint is narrower and safer than renaming the reason, and suppressing `unknown` only for skipped no-channel rows preserves sent-heartbeat diagnostics.
