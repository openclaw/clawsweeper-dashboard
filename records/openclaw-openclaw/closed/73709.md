## Summary

Keep this open. Current main still requests `GuildVoiceStates` unconditionally for Discord gateway sessions, while `channels.discord.voice.enabled=false` only suppresses the voice runtime/VoicePlugin path. The related #28587 CPU report is closed but describes a different eager-loading problem, not this intent-selection bug.

## Best Possible Solution

Fix this in the bundled Discord plugin by threading resolved voice enablement into gateway intent resolution, so `channels.discord.voice.enabled=false` removes `GuildVoiceStates`. Add regression coverage around `resolveDiscordGatewayIntents` or provider startup, update the config schema/docs if an explicit `channels.discord.intents.voiceStates` override is added, and keep default behavior aligned with the documented Discord voice default.
