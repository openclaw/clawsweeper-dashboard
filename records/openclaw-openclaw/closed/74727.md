## Summary

Close as not actionable in the OpenClaw source repository. The reported false-negative behavior is in the external Lossless Claw plugin's `lcm_grep`/`/lossless` surfaces; current OpenClaw main only documents and hosts the generic context-engine/plugin APIs for that plugin and does not contain the LCM recall implementation to change.

## What This Changes

Request that Lossless Claw current-conversation recall avoid silent false negatives after session-key drift, likely via fallback all-conversation search or diagnostics for multiple active conversations.

## Best Possible Solution

Track the session-key drift recall policy in the Lossless Claw plugin repository, using OpenClaw's existing context-engine lifecycle and plugin tool/command surfaces; reopen in OpenClaw only if the plugin maintainers identify a concrete missing core API or incorrect session metadata passed by current main.
