## Summary

Keep open. Current `main` still writes file logs to the `settings.file` captured when the logger was built, and OpenClaw keeps child loggers alive across long-running work. I did not find equivalent rotation-on-date-change logic on `main` or in the latest release tag, so this PR still addresses a live gap rather than an already-shipped fix.

## Evidence

- **File transport still uses the captured path:** `buildLogger()` creates the log directory once, computes initial byte count once, and the attached transport continues to call `appendLogLine(settings.file, ...)` and size-cap warnings with `settings.file`. There is no per-write recompute of the rolling filename.
