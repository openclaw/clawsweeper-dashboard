## Summary

Current `main` already fixes this. The memory-core plugin now registers `memory_search` and `memory_get` independently, so one tool failing to initialize no longer suppresses the other in new sessions, and the shipped changelog records that exact fix for `#50198`.

## Evidence

- **Independent tool registration on main:** `extensions/memory-core/index.ts:42` registers `memory_search` and `extensions/memory-core/index.ts:51` registers `memory_get` in separate `api.registerTool(...)` calls, matching the reported failure mode where coupled registration could drop both tools.
