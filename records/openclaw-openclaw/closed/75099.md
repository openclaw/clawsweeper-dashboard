## Summary

Keep open for maintainer review. The PR targets a real current-main regression and includes the right warning/config/docs/test shape, but it has a protected `maintainer` label and the new fallback does not yet match all policy layers that can remove the `message` tool.

## What This Changes

The branch adds a one-shot warning for implicit private group/channel replies, allows visible-reply config keys through `config.patch`, falls back to automatic delivery when the message tool is judged unavailable, and updates docs, changelog, tests, and config baseline metadata.

## Best Possible Solution

Land a version that keeps the intentional private group/channel default when `message` is truly available, warns and documents the repair path, allows the bounded visible-reply config edits, and bases fallback on the same effective tool inventory/policy layers used by the agent run.

## Reproduction Assessment

Yes. Current main returns `message_tool_only` for group/channel source turns without explicit visible-reply config and dispatch suppresses automatic delivery, while #74868 and #74876 provide concrete post-upgrade silent-room scenarios.

## Solution Assessment

No. The PR is directionally right, but the current availability check is not yet the best complete fix because it can disagree with the actual filtered tool list used by the agent runtime.
