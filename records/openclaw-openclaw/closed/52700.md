## Summary

Current `main` does not support the reported cross-run compaction runtime pollution path. The issue’s key code claim about `compactEmbeddedPiSessionDirect` trusting raw caller `provider`/`model` is obsolete on current `main`, and the only current `setCompactionSafeguardRuntime(...)` call is scoped to a freshly opened `SessionManager` for that run.

## Evidence

- **Compaction target is resolved before compaction runs:** `compactEmbeddedPiSessionDirect()` now resolves the effective compaction provider/model through `resolveEmbeddedCompactionTarget(...)` before model loading, instead of directly trusting raw caller values.
  - file: [src/agents/pi-embedded-runner/compact.ts:326](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/pi-embedded-runner/compact.ts#L326)
  - command: `nl -ba src/agents/pi-embedded-runner/compact.ts | sed -n '320,345p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Compaction override logic is centralized:** `resolveEmbeddedCompactionTarget(...)` applies `agents.defaults.compaction.model` overrides and otherwise falls back to the run’s provider/model, so the old direct `params.provider ?? ...` / `params.model ?? ...` path described in the issue is not present on current `main`.
  - file: [src/agents/pi-embedded-runner/compaction-runtime-context.ts:34](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/pi-embedded-runner/compaction-runtime-context.ts#L34)
  - command: `nl -ba src/agents/pi-embedded-runner/compaction-runtime-context.ts | sed -n '1,120p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Each normal run opens a fresh SessionManager:** The main run path calls `SessionManager.open(params.sessionFile)` immediately before `buildEmbeddedExtensionFactories(...)`, so the safeguard runtime is attached to that run-local manager instance.
  - file: [src/agents/pi-embedded-runner/run/attempt.ts:1058](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/pi-embedded-runner/run/attempt.ts#L1058)
  - command: `nl -ba src/agents/pi-embedded-runner/run/attempt.ts | sed -n '1050,1125p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **Each compaction run also opens a fresh SessionManager:** The compaction path separately opens `SessionManager.open(params.sessionFile)` and passes that instance into `buildEmbeddedExtensionFactories(...)` and `sanitizeSessionHistory(...)`, so the current code does not reuse one shared SessionManager object across runs by default.
  - file: [src/agents/pi-embedded-runner/compact.ts:804](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/pi-embedded-runner/compact.ts#L804)
  - command: `nl -ba src/agents/pi-embedded-runner/compact.ts | sed -n '800,840p;915,935p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **The safeguard runtime is only written once per run-local SessionManager:** `setCompactionSafeguardRuntime(...)` is called from `buildEmbeddedExtensionFactories(...)` with the current run’s `sessionManager`; no other production call site writes this registry.
  - file: [src/agents/pi-embedded-runner/extensions.ts:100](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/pi-embedded-runner/extensions.ts#L100)
  - command: `nl -ba src/agents/pi-embedded-runner/extensions.ts | sed -n '80,125p'`
  - sha: [8866544ffefb](https://github.com/openclaw/openclaw/commit/8866544ffefb42ca234c66c337134fb60743d8d0)
- **The cache does not retain SessionManager objects:** `session-manager-cache.ts` caches only `true` by session-file path for page-cache warming/tracking; it does not store or reuse `SessionManager` instances, so it cannot explain WeakMap key reuse across runs.
  - file: [src/agents/pi-embedded-runner/session-manager-cache.ts:44](https://github.com/openclaw/openclaw/blob/8866544ffefb42ca234c66c337134fb60743d8d0/src/agents/pi-embedded-runner/session-manager-cache.ts#L44)
  - command: `nl -ba src/agents/pi-embedded-runner/session-manager-cache.ts | sed -n '1,120p'`

## Risks / Open Questions

- The runtime registry is still keyed by SessionManager object identity in `src/agents/pi-hooks/session-manager-runtime-registry.ts:1`, so a future caller that intentionally reuses one SessionManager instance across distinct model contexts could recreate this class of bug.
- I did not run a live end-to-end repro because this review was constrained to read-only inspection and the checkout lacks `pnpm` in PATH, so the conclusion is based on current main code paths rather than runtime execution.
