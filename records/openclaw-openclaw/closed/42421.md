## Summary

Current `main` already fixes the reported OpenAI-compatible streaming-usage regression: the compat normalization no longer overwrites an explicit `supportsUsageInStreaming` opt-in, streaming `openai-completions` requests always send `stream_options.include_usage`, and the changelog records that fix as shipped in `v2026.3.22`.

## Close Comment

Closing from a Codex review because the regression described here is already fixed on current `main` and was shipped in `v2026.3.22`. The changelog entry at `CHANGELOG.md:2428` calls out this exact fix, `src/plugins/provider-model-compat.ts:108` now preserves an explicit `compat.supportsUsageInStreaming` override instead of forcing it off, and `src/agents/openai-transport-stream.ts:1692` now sends `stream_options.include_usage` on streaming `openai-completions` requests. There is focused regression coverage in `src/agents/model-compat.test.ts:191` and `src/agents/openai-transport-stream.test.ts:1404`. If someone can still reproduce zero usage on a current build, that would need a new issue with the exact provider config and captured response shape, because the code path reported here is no longer the one on `main`.
