## Summary

Keep this PR open. Current main still treats a pre-hello code-1000 empty-reason gateway close as fatal in one-shot gateway calls, and GatewayClient still surfaces the close/error before reconnect can complete. The PR remains a focused implementation candidate, but it overlaps with same-author PR #54475, so maintainers should choose one canonical branch before closing either one.

## Best Possible Solution

Keep this PR open for maintainer review and consolidate it with #54475. The best path is a narrow gateway fix that tracks `hello-ok` in `src/gateway/call.ts`, suppresses only pre-hello code-1000 empty-reason closes in `src/gateway/client.ts`, keeps suppression bounded and keyed to the relevant connect attempt, and retains regression coverage in `src/gateway/call.test.ts` and `src/gateway/client.test.ts`. After one branch lands, close the other overlapping PR as superseded.
