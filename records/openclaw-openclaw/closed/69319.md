  - body: Current main derives dispatch metadata from `CORE_GATEWAY_METHOD_SPECS`, and `createCoreGatewayMethodDescriptors` throws for any core handler without a descriptor.   - body: This deletes `skills/<slug>` whenever the directory exists, even if there is no `.clawhub/origin.json` and no lockfile entry.   - confidence: 0.94
- **[P2] Classify invalid slugs as request errors:** `src/gateway/server-methods/skills.ts:278`
  - body: The schema accepts any non-empty string, so malformed slugs such as path traversal attempts reach backend normalization. Mapping every backend failure to `UNAVAILABLE` turns client input errors into retryable server outages; slug normalization and path errors should return `INVALID_REQUEST`.
  - confidence: 0.87
- **[P2] Use the request runtime config:** `src/gateway/server-methods/skills.ts:254-255`
  - body: Current Gateway skills handlers use `context.getRuntimeConfig()` so request-scoped config changes and tests are honored. This new handler reloads config from disk with `loadConfig()`, which can target the wrong workspace after runtime config updates; take `context` and resolve the workspace from it.
  - confidence: 0.8
