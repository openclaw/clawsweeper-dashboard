## Summary

Keep open. The report describes broken existing subagent orchestration, current docs promise push-based `sessions_yield` completion delivery, and current source still has a plausible no-retry `agent.wait` pending path; the exact 5-8s parallel-completion race was not reproduced in this read-only pass and overlaps with existing subagent delivery work.

## What This Changes

The issue reports that parallel `sessions_spawn` subagents completing within seconds of each other can fail to deliver expected `sessions_yield` completion events to the parent session.

## Best Possible Solution

Verify whether this timing report is covered by the pending-wait supervision gap or an existing completion-delivery PR, then land one focused core fix that makes every terminal child run either wake the parent or remain durably inspectable without requiring polling.

## Reproduction Assessment

No, not as a high-confidence current-main runtime reproduction. The issue gives concrete live timing evidence, and source inspection shows a plausible no-retry pending wait path, but this review did not run the Windows/openclaw-weixin parallel-spawn scenario.

## Solution Assessment

Unclear. A narrow fix to keep supervising `agent.wait` pending results may solve the source-backed gap, but maintainers should first check it against the adjacent durable-delivery and sessions_yield wake PRs so OpenClaw does not grow competing completion paths.
