## Summary

Keep open. Current main and the latest release still classify implicit cron `text` payloads as `systemEvent` before considering agent-turn hints, so nested and legacy top-level `text` plus `model` inputs can still drop the model override. This PR remains a focused implementation candidate, and the later branch discussion says the earlier top-level gap was addressed.

## Best Possible Solution

Keep this PR open and review or land this branch, or an equivalent mainline fix, that updates both `coercePayload()` and `inferTopLevelPayload()` so `text` plus agent-turn-only fields promotes to `agentTurn` while preserving explicit `systemEvent` pruning. Keep the nested and top-level create/patch regression tests and refresh the changelog against current main before merge.
