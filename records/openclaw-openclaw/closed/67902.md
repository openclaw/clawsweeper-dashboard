## Summary

Current main and v2026.4.26 implement conservative cleanup/liveness handling for stale unended subagent runs, persist terminal lifecycle status for normal endings, and bound retained session-store rows through session maintenance.

## Best Possible Solution

Close as implemented and keep the current conservative behavior: supported subagent/session APIs no longer treat stale unended subagent rows as active forever, normal lifecycle completion persists terminal state, and retained raw `sessions.json` history is bounded by session maintenance rather than being an authoritative live-process registry.
