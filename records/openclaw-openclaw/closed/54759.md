## Summary

Close: the linked bug was already resolved on current main through provider-agnostic subagent session keys with provider metadata stored separately, while this PR is an obsolete alternative that changes the OpenAI-compatible HTTP generated-key namespace and still has a provider-override correctness bug.

## What This Changes

The PR derives OpenAI-compatible `/v1/chat/completions` generated session-key prefixes from the selected agent's configured default provider, passes a pre-resolved agent id into gateway request context, and adds session-prefix tests.

## Best Possible Solution

Keep the current provider-agnostic subagent session-key implementation with provider metadata stored separately; any separate OpenAI-compatible HTTP generated-key namespace change should be proposed as a fresh, documented product/backcompat decision.

## Reproduction Assessment

Yes. Source inspection confirms current main no longer reproduces the linked subagent `:openai:` key bug, and PR-head inspection reproduces the remaining patch defect where `x-openclaw-model` overrides are resolved after the proposed prefix is computed.

## Solution Assessment

No. The source bug is already solved on main through the subagent session-key path, and this PR’s HTTP-prefix rewrite is both a separate backward-incompatible policy choice and currently ordered before final model override resolution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Derive the prefix after model override resolution:** `src/gateway/openai-http.ts:452-458`
  - body: The OpenAI-compatible endpoint supports `x-openclaw-model` as the backend provider/model override, but this computes `providerPrefix` from the agent default before `resolveOpenAiCompatModelOverride` runs. A default Anthropic agent handling `x-openclaw-model: openai/gpt-5.4` would dispatch an OpenAI run under an `anthropic-*` generated session key, so the generated key still would not reflect the actual provider for override requests.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `src/gateway/openai-http.ts:458`
  - body: This PR changes user-visible generated session keys and marks the change as not backward compatible, but `CHANGELOG.md` is not touched. OpenClaw requires a changelog entry for user-facing fixes, so this namespace change should be recorded before merge if maintainers decide to proceed.
  - confidence: 0.78
