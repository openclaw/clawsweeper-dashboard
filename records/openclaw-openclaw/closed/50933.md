## What This Changes

The PR adds automatic daily/idle reset hook dispatch in session initialization, updates the bundled session-memory hook to handle those events, and adds related docs and tests.

## Best Possible Solution

Keep the remaining implementation discussion on https://github.com/openclaw/openclaw/pull/61675 and land one bounded, off-path automatic reset memory preservation path with tests and docs.

## Reproduction Assessment

Yes, at source level. Current main rotates stale daily/idle sessions and archives the old transcript, while the bundled session-memory hook is registered for and handles only manual command reset events.

## Solution Assessment

No, not this branch as-is. The useful behavior is covered by the canonical open PR, and this branch awaits hook dispatch in session initialization instead of keeping reset memory capture bounded or off the reply path.
