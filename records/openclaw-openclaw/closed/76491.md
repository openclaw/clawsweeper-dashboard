## Summary

Close as implemented on current main. The Gateway web_search path now late-binds to the active secrets runtime config, while startup secret resolution materializes env-backed web-search credentials into that active config before Tavily/Exa provider execution.

## What This Changes

Make bundled Tavily and Exa web_search honor Gateway-process `TAVILY_API_KEY` and `EXA_API_KEY` credentials at runtime.

## Best Possible Solution

Keep the #76449 active-runtime-config fix on main; users on 2026.4.29 or 2026.5.2 should update to a build that includes b1f817, and local `openclaw infer web search` should use local env/config rather than relying only on a systemd Gateway env.

## Reproduction Assessment

Yes for the pre-fix path: the related #76416 report and #76449 fix show late-bound web_search was losing the active runtime config. Current main source no longer has that same failure path because it reads the active secrets runtime snapshot before provider execution.

## Solution Assessment

Yes. The merged fix is the narrow maintainable solution: keep provider code owner-local and pass the active runtime config into the generic web_search runtime rather than adding Tavily/Exa-specific core behavior.
