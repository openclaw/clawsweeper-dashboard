## Summary

This is useful Feishu/Lark functionality, but it is an optional provider-native card tool rather than a core OpenClaw change. Current main already has the generic Feishu message/presentation path and internal card helpers, while the plugin SDK and ClawHub path can support a separately maintained `feishu_card` native tool plugin without a new core API. No already-posted ClawHub plugin was linked in the provided PR context.

## Best Possible Solution

Close this bundled PR as ClawHub/plugin scope. The best path is a separately maintained native plugin that registers a `feishu_card` tool via `api.registerTool(...)`, with its own config and security posture; core should keep the shipped semantic `message`/`presentation` Feishu path unless a concrete missing plugin API is identified.
