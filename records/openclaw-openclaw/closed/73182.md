## Summary

Keep open. Current main has implemented global reasoning visibility defaults and the Telegram/Feishu preview fix, but the API-thinking cost path remains source-reproducible for Claude 4.6 and generic reasoning-capable models, and the Microsoft Teams renderer still has no reasoning-payload suppression if such payloads reach that plugin-local delivery path.

## What This Changes

Request to make Claude/API thinking opt-in by default, prevent hidden reasoning blocks from rendering in chat surfaces, and add safer global/default-warning controls.

## Best Possible Solution

Adopt a maintainer-approved policy that API thinking is off unless explicitly configured or provider-required, keep cron and normal turns on the same policy, and suppress reasoning-marked payloads in every channel renderer unless visibility is explicitly enabled.

## Reproduction Assessment

Yes, source-level. Current main still resolves some Claude and reasoning-capable models to non-off API thinking when no explicit thinking override is set, and the Teams renderer lacks a local `isReasoning` suppression check.

## Solution Assessment

No, current main is only a partial solution. The global reasoning visibility config and Telegram/Feishu preview fixes should stay, but the remaining API-thinking default policy and Teams suppression path need maintainer direction before a repair PR changes runtime behavior.
