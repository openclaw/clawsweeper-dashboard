## Summary

Keep open: current main has better Codex app-server side-effect timeout diagnostics and a related open PR for the terminal-idle budget, but cron still has no declared-artifact reconciliation contract that can verify an already-created external artifact and convert a timed-out run into success or partial success.

## What This Changes

The issue requests scheduled cron runs distinguish or reconcile Codex app-server timeouts that happen after external side effects so final artifact loss does not leave misleading failed run state.

## Best Possible Solution

Define an opt-in cron completion-artifact contract that can verify declared external artifacts, record verified partial or successful outcomes without hiding ordinary failures, and coordinate any remaining terminal-idle diagnostics with the existing Codex app-server timeout work.

## Reproduction Assessment

No, not as a high-confidence live reproduction against current main. Source inspection and the reporter's production observation show the current timeout/finalization path can only record error/timed_out state and lacks any declared-artifact verifier.

## Solution Assessment

Unclear until maintainers choose the artifact declaration and trust model. The requested reconciliation hook is plausible, but the maintainable path is an explicit opt-in cron artifact contract plus coordinated timeout diagnostics, not an implicit GitHub-specific check.
