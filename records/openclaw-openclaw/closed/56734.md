## Summary

Current `main` already covers both reported symptoms. Config writes go through a `0o600` temp-file path with an explicit `chmod(0o600)` fallback on copy replacement, and doctor has a regression test ensuring consecutive `--fix` runs do not keep re-reporting the same `talk.provider/providers` normalization.

## Evidence

- **Config write path hardens mode to 0600:** `writeConfigFile` writes the temp config with `mode: 0o600`; on copy fallback it explicitly reapplies `chmod(configPath, 0o600)`. On normal POSIX rename, the temp file's mode is preserved.
