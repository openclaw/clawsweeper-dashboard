## Summary

Close as `clawhub`: current main keeps model fallbacks as failure-time failover only, while this opt-in prompt-optimization layer fits the existing plugin/ClawHub extension surfaces rather than OpenClaw core.

## Close Comment

Closing as `clawhub` after a Codex review of current `main`. OpenClaw currently treats `agents.defaults.model.fallbacks` as failure-time failover only, not as a preprocessing stage (`docs/concepts/models.md:16`, `docs/concepts/model-failover.md:10`). The project scope also says optional capability should usually ship as plugins, and new skills should go to ClawHub first (`VISION.md:54`). This request is also a better fit for the existing plugin seams: `before_prompt_build` is the documented hook for prompt mutation, `before_agent_reply` can intercept a turn before the normal LLM path, and bundled plugin code already runs auxiliary embedded agent calls (`docs/plugins/architecture.md:100`, `docs/concepts/agent-loop.md:93`, `src/plugins/hooks.ts:530`, `extensions/skill-workshop/src/reviewer.ts:227`). Because this is an opt-in prompt-optimization layer rather than core reliability behavior, we should keep it out of OpenClaw core and pursue it as a ClawHub/community plugin instead. If someone hits a concrete SDK gap while building that plugin, please open a focused issue for the missing hook/runtime seam.
