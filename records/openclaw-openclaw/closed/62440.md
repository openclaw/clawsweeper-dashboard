## Summary

Current `main` already implements the requested stdout/stderr contract: the local agent delivery path now writes the JSON envelope through the shared runtime JSON writer instead of `runtime.log`, and the latest release tag available in this checkout contains the same path. The issue comments and the closed related PRs identified the right call site; current source, docs, and focused tests now cover the behavior.

## What This Changes

Ensure `openclaw agent --local --json` writes the machine-readable result envelope to stdout while diagnostics remain on stderr.

## Best Possible Solution

Keep the shared `writeRuntimeJson` path as the contract for all agent JSON output and preserve the focused delivery test so future local, gateway, and embedded-fallback changes cannot route machine-readable payloads through diagnostic logging again.
