## Summary

Issue #18047 is a useful idea, but it fits OpenClaw's documented ClawHub/community plugin-skill path rather than OpenClaw core. Current project scope explicitly says new skills should go to ClawHub first and that new core skills that can live on ClawHub are out of scope.

## Close Comment

Closing after Codex review because this request fits ClawHub/community extension scope rather than OpenClaw core. The current project vision says new skills should be published to ClawHub first, not added to core by default (`VISION.md:66`), and explicitly lists 'New core skills when they can live on ClawHub' as out of scope for now (`VISION.md:99`). The docs also route new optional capabilities through third-party plugins/skills published on ClawHub or npm (`docs/plugins/community.md:9`, `docs/tools/clawhub.md:10`). For an MT4/MT5 trading executor, the maintainable path here is a separately maintained plugin/skill repo and ClawHub publication rather than a core OpenClaw item.
