The PR adds a WeakMap memoization layer around `resolveOwnerAllowFromList` plus a colocated test for repeated owner allowlist resolution.

## Best Possible Solution

Revise or replace the branch so command authorization caches a compiled owner lookup shape, including wildcard flags and Set-backed membership, and include redacted large-list Gateway proof.

## Reproduction Assessment

Yes at source level: current main and v2026.6.10 still walk and linearly check large owner allowlists in command authorization. I did not run a live 9,000-entry Gateway benchmark in this read-only review.

## Solution Assessment

No: this is a plausible partial mitigation, but the best fix is to cache or compile the owner authorization lookup state itself so repeated authorization checks use O(1) membership instead of rebuilding and scanning arrays.
