## Summary

Keep open. Current `main` still drops the spoken transcript for audio-only auto-TTS payloads, so the underlying gap is real, and the proposed `spokenText`/`hookContent` PR still has an unresolved path where `message_sending` hooks can turn that hook-only transcript back into channel-visible delivery text.
