## Summary

Keep open. Current main still auto-stamps a missing `job.sessionKey` from the caller session during `cron.add`, even for `payload.kind="agentTurn"` jobs that default to or explicitly use isolated execution. That contradicts the cron tool docs, which say current-session binding requires explicit `sessionTarget="current"`, and it can feed the inherited key into isolated-run session selection.
