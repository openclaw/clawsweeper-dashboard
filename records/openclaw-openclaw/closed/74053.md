## Summary

Keep open: the runtime change is narrow and likely right, but this external PR still lacks after-fix real behavior proof, and the new regression test is too weak to prove the seeded `lastTo` case would fail on current main.

## What This Changes

The PR changes `src/infra/heartbeat-runner.ts` to set heartbeat `ctx.To` from `delivery.to`, adds a target-none regression test, and adds a changelog fix entry.

## Best Possible Solution

Land the narrow `ctx.To = delivery.to` fix after redacted real behavior proof is added and the regression test pins target-none `ctx.To` to `undefined`.

## Reproduction Assessment

Yes, for the source-level context behavior: current main sets `ctx.To` to `sender`, target-none delivery leaves `delivery.to` unset, and the sender resolver can fall back to `heartbeat`. The full persisted-corruption path is less direct on current main because system-event turns now preserve existing `lastTo`, and I did not run live proof in this read-only review.

## Solution Assessment

Yes. Deriving `ctx.To` from the resolved delivery target is the narrow maintainable runtime fix, but the regression test should assert target-none `ctx.To` is exactly `undefined` and the PR still needs real behavior proof.
