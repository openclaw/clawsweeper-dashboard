## Summary

Current main does not use a repo-root `config/` directory for runtime state; mutable config/state lives under `~/.openclaw` or an explicit `OPENCLAW_STATE_DIR`. The proposed unanchored `config/` ignore rule would also create a real regression by hiding new files under the tracked `src/config/` tree.

## Evidence

- **Runtime state location in code:** `resolveStateDir()` documents and implements `~/.openclaw` as the default mutable state directory, and `resolveCanonicalConfigPath()` defaults the config file to `~/.openclaw/openclaw.json`, not a repo-root `config/` directory.
  - file: [src/config/paths.ts:55](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/src/config/paths.ts#L55)
  - sha: [d068cb960de4](https://github.com/openclaw/openclaw/commit/d068cb960de47d5587c5942f6beb86e0f124b449)
- **Runtime state location covered by tests:** Current tests assert that the default state dir resolves to `<home>/.openclaw` and that `OPENCLAW_STATE_DIR` is the override path.
  - file: [src/config/paths.test.ts:111](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/src/config/paths.test.ts#L111)
  - sha: [d068cb960de4](https://github.com/openclaw/openclaw/commit/d068cb960de47d5587c5942f6beb86e0f124b449)
- **Public docs match current behavior:** The user-facing docs say all persisted state lives in `~/.openclaw/`, including `openclaw.json`, auth profiles, channel/provider state, and session data.
  - file: [docs/platforms/oracle.md:284](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/docs/platforms/oracle.md#L284)
  - sha: [d068cb960de4](https://github.com/openclaw/openclaw/commit/d068cb960de47d5587c5942f6beb86e0f124b449)
- **Existing root ignore conventions are anchored:** The current `.gitignore` anchors root-only sensitive/local directories as `/memory/` and `/local/`. The PR's proposed `config/` pattern is broader than the repo's existing convention for root-only ignores.
  - file: [.gitignore:100](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/.gitignore#L100)
  - sha: [d068cb960de4](https://github.com/openclaw/openclaw/commit/d068cb960de47d5587c5942f6beb86e0f124b449)
- **Tracked `src/config` tree would be affected:** Current main tracks 315 files under `src/config/`, so an unanchored `config/` rule would create a contributor footgun by hiding newly added files in that subtree.
  - command: `git ls-files 'src/config/**' | wc -l`
  - sha: [d068cb960de4](https://github.com/openclaw/openclaw/commit/d068cb960de47d5587c5942f6beb86e0f124b449)
- **Prior review already identified the regression:** Both review comments on this PR correctly call out that unanchored `config/` would match nested directories like `src/config/` and should not merge as written.
  - file: [.gitignore:5](https://github.com/openclaw/openclaw/blob/2e53838dde3c1ad44e2de5eef5cf1ffc3caedd16/.gitignore#L5)
  - sha: [2e53838dde3c](https://github.com/openclaw/openclaw/commit/2e53838dde3c1ad44e2de5eef5cf1ffc3caedd16)

## Risks / Open Questions

- Undocumented out-of-tree workflows could still create a repo-root `config/` directory, but that is not current OpenClaw main behavior and is not supported by the repo code/docs reviewed here.
- `gh` inspection was not available in this environment because `GH_TOKEN` was unset, so the decision relies on the provided PR context plus local code/docs/history review.
