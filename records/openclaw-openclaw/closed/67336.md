## Summary

Keep open. Current main still contains the two macOS Remote over SSH write paths described in #67336: the SSH config sync path rewrites `gateway.remote.url` from the SSH target host, and the discovery-selection path persists the discovered endpoint URL even when the selected transport remains SSH. The linked PR #67470 is the right shape of fix, but it is closed/unmerged in the provided context and its regression coverage is not present on current main.

## Best Possible Solution

Keep #67336 open and revive or reimplement the #67470 approach on current main: in macOS SSH transport, persist `gateway.remote.url` as the local tunnel URL (`ws://127.0.0.1:<port>`, preserving an existing loopback custom port where appropriate), store discovered hosts only in `gateway.remote.sshTarget`, leave direct transport behavior unchanged, and add regression tests for both `AppState.updatedRemoteGatewayConfig` and `GatewayDiscoverySelectionSupport.applyRemoteSelection`.
