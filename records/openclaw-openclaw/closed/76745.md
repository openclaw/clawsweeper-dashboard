## Best Possible Solution

Keep the current main slot-based memory runtime bootstrap and ship it in the next release; no user config workaround or new product behavior is needed.

## Reproduction Assessment

## Solution Assessment

Yes. The current main approach is the narrow maintainable fix because doctor consults the generic selected memory slot and loads only that runtime instead of hard-coding a bundled plugin or requiring `plugins.slots.memory`.
