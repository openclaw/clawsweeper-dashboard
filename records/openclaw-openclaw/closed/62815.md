## Summary

Close as implemented_on_main. Current `main` already fixes the late exec-update crash by proactively suppressing `onUpdate` callbacks before exit/error and on tool-call abort, and the fix shipped in `v2026.4.23` via the changelog entry for `#62821`, so this PR is superseded.

## Evidence

- **Current main suppresses late exec updates before process settlement:** `runExecProcess` now tracks `updatesDisabled`, makes `emitUpdate()` a no-op once disabled, and flips that flag before `markExited()` on success and in the runtime-error path.
