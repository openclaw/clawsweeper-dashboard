## Summary

Current main and the latest release already handle the actionable `chat.send` image-attachment repro without the reported stack overflow. Webchat now sends normalized attachment content, the Gateway normalizes both direct `content` and nested base64 `source` attachment shapes before parsing, request-handler failures are converted to RPC errors instead of crashing the WebSocket, and regression coverage exercises WebSocket `chat.send` with image attachments.

## Evidence
