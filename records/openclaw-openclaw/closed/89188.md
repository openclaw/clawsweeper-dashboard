## Summary

Keep this PR open for maintainer review: the latest head appears to address the prior subtree-race finding and the Linux descriptor fan-out fix is still useful, but it remains an availability and memory-freshness-sensitive watcher replacement.

## What This Changes

The PR changes memory-core so Linux memory directories use a custom non-recursive directory-only fs.watch tree while keeping file paths on chokidar and expanding watcher configuration tests.

## Best Possible Solution

Land the directory-only Linux watcher after the responsible maintainer accepts the targeted Linux proof or requests one broader Linux gateway soak.

## Reproduction Assessment

Yes. Current main routes Linux memory directories to chokidar, while inspected Node and chokidar source both support the report that this allocates per-entry watchers; the PR body adds a concrete Linux descriptor probe and targeted manager smoke.

## Solution Assessment

Yes, with maintainer acceptance of the proof scope. A directory-only Linux fs.watch tree is the narrow layer that reduces descriptor fan-out while preserving sync.watch, and the main alternative of disabling watch by default is tracked separately at https://github.com/openclaw/openclaw/issues/71335.
