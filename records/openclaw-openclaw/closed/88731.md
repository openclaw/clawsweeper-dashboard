## Best Possible Solution

Keep the doctor-owned migration direction, but merge only after the full-store path preserves or intentionally retries retained legacy records, adds focused regression coverage, and includes redacted doctor migration proof.

## Reproduction Assessment

Yes. By source inspection, seed the SQLite event store with 1000 call records, add valid newer `calls.jsonl` records, and run the migration; `selectEntriesForImport` selects none and the archive path can still rename the source.

## Solution Assessment

No. Doctor-owned migration is the right layer, but the current selection/archive rule is not the safest implementation because it does not preserve retained legacy records when the target store is full.
