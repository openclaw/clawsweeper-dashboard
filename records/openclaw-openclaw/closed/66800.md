## Summary

Current main already bounds the Control UI reconnect flow for `AUTH_TOKEN_MISMATCH`: it caches the paired device token from a successful session, does one trusted retry with that token after a shared-token mismatch, and then suppresses further reconnect attempts if that retry also mismatches. The same behavior is present in release `v2026.4.23`.

## Evidence
