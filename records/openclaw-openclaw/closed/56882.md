## Summary

Current `main` no longer matches the reported Feishu core-schema mismatch. OpenClaw now allows plugin-owned `channels.<id>` config through the core `ChannelsSchema` and validates it with the plugin runtime schema, while the current Feishu docs and Feishu runtime config schema no longer advertise or define a `footer` block at all.

## Evidence

- **Core channel config no longer hard-rejects plugin-owned keys:** `ChannelsSchema` is now `.passthrough()` for extension channel configs and then runs `normalizeBundledChannelConfigs(...)`, which resolves the bundled plugin runtime schema and validates `channels.<id>` there instead of rejecting unknown channel properties in core.
  - file: [src/config/zod-schema.providers.ts:143](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/config/zod-schema.providers.ts#L143)
  - command: `nl -ba src/config/zod-schema.providers.ts | sed -n '30,160p'`
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Feishu channel registers its own runtime config schema:** The bundled Feishu channel plugin wires `configSchema: buildChannelConfigSchema(FeishuConfigSchema)`, so current validation comes from the Feishu plugin schema rather than a stale manifest-only core schema.
  - file: [extensions/feishu/src/channel.ts:604](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/extensions/feishu/src/channel.ts#L604)
  - command: `nl -ba extensions/feishu/src/channel.ts | sed -n '596,612p'`
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Current Feishu runtime schema has no `footer` config:** The current shared Feishu config shape includes keys such as `streaming`, `blockStreamingCoalesce`, `replyInThread`, `reactionNotifications`, `typingIndicator`, and `resolveSenderNames`, but no `footer`, `footer.elapsed`, or `footer.status` fields.
  - file: [extensions/feishu/src/config-schema.ts:155](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/extensions/feishu/src/config-schema.ts#L155)
  - command: `nl -ba extensions/feishu/src/config-schema.ts | sed -n '130,220p'`
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Current Feishu docs no longer document footer settings:** The current Feishu configuration reference lists the supported settings through `channels.feishu.resolveSenderNames`; it does not list `channels.feishu.footer.*`. The same absence is present in the latest shipped tag `v2026.4.23` from `git show v2026.4.23:docs/channels/feishu.md`.
  - file: [docs/channels/feishu.md:375](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/docs/channels/feishu.md#L375)
  - command: `nl -ba docs/channels/feishu.md | sed -n '372,405p'`
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Repo-wide search finds no current footer-config surface for Feishu:** A repo search for `channels.feishu.footer`, `footer.elapsed`, `footer.status`, `resolveFooterConfig`, and `footer config` returned no matches, which supports that the reported footer surface is obsolete in current `main`.
