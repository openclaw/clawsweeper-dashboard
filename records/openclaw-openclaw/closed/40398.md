## Summary

Close PR #40398 because current `main` already implements the requested zero-default poll detection fix, adds direct regression coverage for the `send` path, and intentionally preserves negative poll durations on the poll-validation path. The PR's proposed `> 0` check is now stale relative to `main` and would weaken current validation behavior for malformed negative durations.
