Full review comments:

- **[P1] Gate skill model overrides behind user policy:** `src/agents/skills/model-router.ts:103-126`
  - body: `buildSkillModelMap` trusts any skill's SKILL.md metadata to pick a configured provider/model. Managed skills can be third-party content, and provider choice controls where transcript context and credentials go; this needs an explicit user/admin policy or allowlist before runtime honors the hint.
  - confidence: 0.88
- **[P1] Resolve provider routing before swapping models:** `src/agents/skills/model-router.ts:206-210`
  - body: The wrapper only replaces the `model` argument, but the attempt already resolved provider stream, auth profile, API key, transport, and extra params for the session model. A skill override to another provider can therefore run through the wrong provider/auth wrapper instead of the selected model's runtime path.
  - confidence: 0.86
- **[P2] Recognize current read-tool path aliases:** `src/agents/skills/model-router.ts:164-171`
  - body: The read interceptor checks `path`, `file`, and `filename`, but current OpenClaw tooling also uses `file_path` and `filePath`. Reads using those aliases will leave `rawFilePath` empty and the model override will silently never activate.
  - confidence: 0.82
