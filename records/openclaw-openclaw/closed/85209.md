## Summary

Keep open for maintainer review. The patch matches the requested Telegram formatting path and I did not find a concrete correctness or security defect, but it intentionally widens the public text-chunking SDK marker contract, so that compatibility choice should be accepted before merge.

## What This Changes

The branch adds optional fence-language metadata to the Markdown IR, allows style marker openers to use span metadata, and updates Telegram formatting/tests to emit language-specific fenced code blocks.

## Best Possible Solution

Land the formatter change after maintainers accept callable style openers as the public text-chunking SDK shape; otherwise use a narrower approved seam that still carries sanitized fence language through chunking.

## Reproduction Assessment

Yes. Source inspection on current main shows Telegram code blocks use a static opener and the Markdown IR drops fence token info, so a language fence cannot produce `class="language-bash"` today.

## Solution Assessment

Yes, mostly. Carrying optional fence metadata through the existing IR/render-aware chunking path is the maintainable fix, but the public SDK type widening needs maintainer acceptance.
