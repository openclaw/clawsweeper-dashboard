## Summary

Keep #52327 open. Current main has improved subagent run persistence, restart recovery, retryable `sessions.delete` sweeps, and a safer sessions.json rotation path, but it does not implement the requested persistent retention policy for completed subagent/ACP session-store entries. `agents.defaults.subagents` still exposes `archiveAfterMinutes` only, session-mode subagents intentionally get no `archiveAtMs`, and the session-mode sweeper TTL removes the registry run record rather than deleting the `sessions.json` entry. ACP sessions also use `agent:<id>:acp:<uuid>` keys without a cron-style retention reaper. The Ryce/KeepMyClaw comment is a useful external backup workaround, but the issue itself is a core session-store retention gap, not a ClawHub-only request.

## Best Possible Solution

Keep the issue open and implement a cron-style durable reaper for completed subagent and ACP session-store rows. The likely path is to add a validated config surface such as `agents.defaults.subagents.sessionRetention` and/or an ACP equivalent, document it beside `archiveAfterMinutes`, and run a lock-safe retention sweep on gateway startup and session-store writes for `agent:*:subagent:*` and `agent:*:acp:*` keys whose `endedAt`, terminal status, or cleanup metadata proves the run is complete. Preserve active/main sessions and add focused tests for restart catch-up, session-mode subagents, ACP keys, and transcript archival.
