## Summary

Current main already handles flattened `cron add` arguments by reconstructing a `job` object from top-level `schedule`/`payload` fields, and regression tests cover the exact missing-`job` shape reported here. The latest shipped release tag `v2026.4.22` points at a commit that already contains this recovery path.

## Evidence

- **Cron tool recovers flattened add params:** `src/agents/tools/cron-tool.ts` includes `schedule`, `payload`, and related fields in `CRON_RECOVERABLE_OBJECT_KEYS`, then rebuilds a synthetic `job` from top-level params when `action:"add"` arrives without a nested `job`. That matches the issue's failing call shape.
