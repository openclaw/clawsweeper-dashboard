## Summary

Keep this PR open: the Feishu queue/session direction is valid, but the current head is merge-conflicting, lacks required real Feishu behavior proof, and can still break same-topic FIFO for native topic starters whose thread id is hydrated only after queue selection.

## What This Changes

The PR threads Feishu account config into sequential-key resolution so grouped Feishu messages can queue by the same group, sender, topic, or topic-sender peer identity used for session routing.

## Best Possible Solution

Refresh or replace this with a narrow Feishu-only fix that derives queue keys from the same hydrated session identity used by handling, preserves chat-wide /stop and /btw lanes, removes the changelog edit, and includes redacted live Feishu proof.

## Reproduction Assessment

## Solution Assessment

No. Reusing Feishu group-session identity is the right direction, but this branch computes the queue key from the raw event before native topic hydration and lacks real behavior proof, so it is not the safest fix yet.
