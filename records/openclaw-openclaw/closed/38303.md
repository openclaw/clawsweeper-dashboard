## Summary

Keep open. Current `main` does not implement the PR’s stuck-session auto-recovery work, and the timeout behavior has diverged rather than being superseded: core agent timeout now defaults to 48 hours, diagnostics still only warn on stuck sessions, there is no `diagnostics.stuckSessionAutoRecoverMs` config surface or `session.auto_recover` event, and the gateway CLI still falls back to 600 seconds when no timeout is provided.
