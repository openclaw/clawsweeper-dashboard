## Summary

Current `main` and released `v2026.4.23` already guard deferred context-engine turn maintenance with per-session coalescing and regression tests, so the reported immediate restart loop is implemented away on the current code path.

## Evidence

- **Per-session deferred-run coalescing:** `scheduleDeferredTurnMaintenance` keeps a single active deferred run per session. If another trigger arrives while one is active, it only sets `rerunRequested` and updates `latestParams`, then returns. When the active run finishes, it may schedule at most one follow-up run.
