## Summary

Keep open: current main still replays historical user-role messages without stripping OpenClaw inbound metadata, and the existing strip helper also does not recognize the 2026.5.12 delivery hint or chronological chat-window block called out by the reporter.

## What This Changes

The issue requests stripping OpenClaw inbound metadata from historical user-role messages during prompt replay while preserving the current turn's full context envelope.

## Best Possible Solution

Strip inbound metadata from historical user-role replay content at the replay-history boundary, extend `stripInboundMetadata` for the new delivery and chat-window envelope shapes, and keep current-turn metadata generation unchanged.

## Reproduction Assessment

Yes from source inspection: the current turn's metadata is built separately, while historical user replay content is passed through without `stripInboundMetadata`. I did not run a live Telegram replay in this read-only pass.

## Solution Assessment

Yes: the narrowest maintainable fix is to sanitize only historical user-role replay text and teach the existing helper the two newer envelope shapes, instead of changing Telegram context generation or adding a new config mode.
