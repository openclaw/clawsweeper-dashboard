## Summary

Current `main` already contains the route-registry fix for plugin-owned webhook paths. The gateway pins the startup plugin HTTP-route registry, `registerPluginHttpRoute(...)` resolves against that pinned registry, and request dispatch re-resolves against the active route registry rather than a stale captured object. Current tests explicitly cover the reported failure mode where the active registry changes after startup and dynamic webhook routes would otherwise 404.
