## Summary

Keep open: this is a plausible fix for an open abort-output bug, but the proposed cleanup runs before the tracked prompt/abort settlement and can miss the aborted assistant message it is trying to remove; the external PR also still lacks real behavior proof.

## What This Changes

This PR adds a helper in the embedded Pi runner and calls it on runner aborts to remove trailing assistant messages with `stopReason: "aborted"` from memory and session file entries.

## Best Possible Solution

Rework the patch to wait for tracked abort settlement before removing untrusted aborted assistant output, preserve or explicitly revise the legitimate abort-partial contract, add focused regression coverage, and provide redacted real runtime proof.

## Reproduction Assessment

No, not as a high-confidence live Telegram/provider reproduction in this review. Source inspection shows abort buffers plus abort-settlement timing paths that can preserve or broadcast nonblank aborted assistant text, matching the reported failure class.

## Solution Assessment

No. The cleanup direction is plausible, but this patch strips too early in the abort path and needs to preserve or deliberately revise the existing legitimate abort-partial contract.
