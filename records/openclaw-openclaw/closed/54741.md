## Summary

Current `main` already routes `echoTranscript` as an immediate pre-agent outbound send to the originating Telegram chat, so the reported buffering-behind-restart behavior is obsolete against the checked-in implementation.

## Evidence

- **Pre-agent echo path is explicit:** `applyMediaUnderstanding` now awaits `sendTranscriptEcho(...)` immediately after transcription and before agent processing continues.
  - file: [src/media-understanding/apply.ts:544](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/media-understanding/apply.ts#L544)
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Echo uses direct outbound delivery:** `sendTranscriptEcho` resolves `channel` from `ctx.Provider/Surface`, `to` from `ctx.OriginatingTo/From`, and calls `deliverOutboundPayloads(...)` directly with `bestEffort: true` instead of going through a session-only queue.
  - file: [src/media-understanding/echo-transcript.ts:31](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/media-understanding/echo-transcript.ts#L31)
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Telegram DM context carries the correct return target:** Inbound Telegram context for direct chats sets `From`, `Provider`, `Surface`, and `OriginatingTo` to the DM chat id (`telegram:${chatId}`), which is exactly what the echo path consumes.
  - file: [extensions/telegram/src/bot-message-context.session.ts:328](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/extensions/telegram/src/bot-message-context.session.ts#L328)
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Telegram dispatch is eager on main:** Current Telegram message processor statically imports `dispatchTelegramMessage` and calls it eagerly; the lazy import path was removed in commit `f03252aaf9cc946dd6a4f6189f30d5c5dcd64022` (`fix: keep telegram dispatch eager`).
  - file: [extensions/telegram/src/bot-message.ts:12](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/extensions/telegram/src/bot-message.ts#L12)

  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Echo delivery contract is covered by tests:** Unit tests assert that enabling `echoTranscript` causes an immediate outbound delivery call with the resolved destination and formatted transcript.
  - file: [src/media-understanding/apply.echo-transcript.test.ts:257](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/media-understanding/apply.echo-transcript.test.ts#L257)
