## Label Justifications

- `P1`: The report described an urgent broken channel workflow where existing WeChat accounts silently stopped processing messages after a new login.
- `impact:message-loss`: The affected older account stopped receiving and sending channel messages until a manual gateway restart.

## Summary

Current main and the shipped v2026.5.18 release already contain the generic gateway hot-reload lifecycle fix: plugin reload stops are manager-owned/non-manual, and non-manual channel exits remain eligible for account-specific recovery instead of leaving older accounts manually stopped.

## What This Changes

Ensure WeChat hot reload after adding a new QR-login account preserves or restarts previously running account monitors.

## Best Possible Solution

Keep the shipped generic gateway lifecycle fix in v2026.5.18 and handle the linked open PR separately as already covered or superseded by current main.

## Reproduction Assessment

No current-main failing reproduction remains because the fix is present in v2026.5.18 and current main. The original failure was source-reproducible from the reporter logs plus the WeChat plugin's account-index reload contract and the old manual-stop hot-reload path.

## Solution Assessment

Yes. The implemented solution stays channel-agnostic by fixing gateway lifecycle semantics rather than adding WeChat-specific policy, and regression coverage now exercises the non-manual reload/recovery behavior.
