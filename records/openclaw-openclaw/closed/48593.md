## Summary

Keep open: this is a valid user-visible Gemini tag leak fix attempt, but the PR is not merge-ready because the widened regex can treat custom or malformed tags as real final/thinking delimiters and there is still no after-fix real Telegram or equivalent proof.

## What This Changes

The PR broadens embedded final/thinking tag regexes, updates sanitizer/fallback/live-test helper patterns, adds self-closing tag handling, and adds focused regression tests for Gemini-style wrapper tags.

## Best Possible Solution

Land one reconciled constrained or shared reasoning/final tag parser across streaming, sanitizer, persistence, and UI display paths, with focused regression tests and redacted real Telegram proof.

## Reproduction Assessment

Yes, at source level: current main's narrow streaming and sanitizer regexes leave attributed and self-closing final tags visible, and a May 14 Telegram report confirms the leak on v2026.5.7. I did not run a live Telegram or Gemini session in this read-only review.

## Solution Assessment

No, not as-is. The touched surfaces are right, but the widened regex should use a constrained tag-name and attribute grammar or a shared parser so strict final-tag enforcement fails closed before merge.
