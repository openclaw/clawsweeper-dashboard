## Summary

Current main already addresses the reported invariant: runtime-required packages such as `chokidar` and `sqlite-vec` are declared in the package manifest, startup dependency installation is documented as unsupported, and release/postpublish guardrails now fail missing runtime declarations or legacy staging debris.

## What This Changes

The issue asks OpenClaw to stop relying on gateway boot-time bundled runtime-deps installs by declaring runtime-required dependencies such as `chokidar` and `sqlite-vec` in package or plugin manifests.

## Best Possible Solution

Keep the current manifest-backed dependency model and release guardrails; older 2026.4.x users should update to a build containing the current main changes once released.

## Reproduction Assessment

Yes. The historical reproduction is a packaged 2026.4.29 gateway boot showing bundled runtime-deps staging, while current main verification checks the now-declared dependencies and the absence of startup install as the supported contract.

## Solution Assessment

Yes. Declaring runtime requirements in package manifests and limiting package-manager work to explicit install/update/doctor paths is the narrow maintainable fix for both cold-start reliability and supply-chain surface.
