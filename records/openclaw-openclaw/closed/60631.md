## Summary

Current main implements the Feishu Schema 2.0 card-action callback fix described by #60631. The parser no longer requires the missing `context.open_id`, `context.user_id`, or legacy `context.chat_id` fields; it accepts `operator.open_id`, top-level SDK-style IDs, `context.open_message_id`, and `context.open_chat_id`, and lifecycle coverage verifies the callback reaches the reply dispatcher instead of being dropped as malformed.

## Best Possible Solution

Close #60631 as implemented on current main. Keep the shipped parser/test behavior; any remaining Feishu card-action payload pass-through or reply-target edge cases should stay with the narrower follow-up issues already tracking those separate surfaces.
