## Summary

Keep open: current main has label/displayName display primitives, but the WebChat/sidebar session row path still does not clearly derive a missing display name from configured agent identity, and the overlapping title/session-naming PRs do not fully own this exact fallback behavior.

## What This Changes

The issue requests that WebChat sidebar session labels use each agent's configured display name or ACP session title before falling back to a generic subagent label.

## Best Possible Solution

Unify this through the existing session title/displayName path: session rows and ACP/WebChat sidebar rendering should share one priority order, with configured agent identity used only as a well-defined fallback and no new WebChat-specific config map unless maintainers choose that product direction.

## Reproduction Assessment

Yes, source-level: current main has configured agent names in the agent list, but the session-row displayName and sidebar resolver fall back to generic labels unless label/displayName is already set on the session. I did not establish a live browser reproduction for the exact compiled WebChat bundle in the report.

## Solution Assessment

Partly. Respecting the existing session title/displayName contract is the maintainable path, but adding a separate `webchat.agentDisplayNames` config map would create a competing source of truth unless maintainers explicitly choose it.
