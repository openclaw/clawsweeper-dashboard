Keep open. Current main still lacks explicit external `wake` / `system event` session targeting, and two open unmerged closing PRs are already the canonical implementation candidates for the remaining work.

## What This Changes

The issue reports lost or misrouted async/background completion reports when external `system event` / `wake` calls do not reliably preserve the originating session context.

## Best Possible Solution

Review and land or replace the paired fixes in https://github.com/openclaw/openclaw/pull/78687 and https://github.com/openclaw/openclaw/pull/80214, then close this issue once current main and a release contain the complete targeted completion path.

Yes from source inspection, though not from a live channel run. Current main has no CLI `--session-key`, no typed wake `sessionKey`, and the gateway wake handler forwards only `mode` and `text`.
