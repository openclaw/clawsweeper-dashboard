## Summary

Keep this PR open but do not merge as-is: the claimed Zod conflict is not reproduced, and the patch forces a repo-wide major-version override into Jimp packages that publish a Zod 3-only dependency contract.

## What This Changes

The PR adds a root `pnpm.overrides.zod` entry and regenerates `pnpm-lock.yaml` so Jimp and other transitive consumers resolve to Zod 4.4.1.

## Best Possible Solution

Use an upstream-compatible Jimp/Baileys dependency update or a maintainer-approved targeted resolution, backed by dependency-change evidence and real WhatsApp/Jimp media proof.

## Reproduction Assessment

Unclear. Current main visibly contains both `zod@3.25.76` and `zod@4.4.3`, but the PR does not include a concrete failing command, TypeScript diagnostic, runtime stack, or WhatsApp/Jimp behavior proof.

## Solution Assessment

No. A repo-wide override is broader than the reported Jimp transitive issue, uses a non-exact range, and crosses Jimp's published Zod 3 dependency contract; a targeted upstream-compatible dependency change would be safer.
