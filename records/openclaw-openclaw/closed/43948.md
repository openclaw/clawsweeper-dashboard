## Summary

Current `main` already supports Feishu image/file sends through the shared `message` tool: the tool schema exposes a `media` field, Feishu `send` reads that field and dispatches to `feishuOutbound.sendMedia`, regression tests cover the path, and `CHANGELOG.md` says this shipped in `2026.3.23`.

## Close Comment

Closing as implemented on current `main`. Codex review found that the shared `message` tool already exposes a `media` input (`src/agents/tools/message-tool.ts:89`), Feishu’s `send` action reads that media payload and routes it to `feishuOutbound.sendMedia(...)` (`extensions/feishu/src/channel.ts:182`, `extensions/feishu/src/channel.ts:697`), and regression coverage exists for the send-with-media path (`extensions/feishu/src/channel.test.ts:432`, `extensions/feishu/src/outbound.test.ts:478`). This was also shipped in release `2026.3.23` (`CHANGELOG.md:2166`). The `capabilities=none` runtime line is not the gate for shared `message` media sends here. If you still see a failure on `v2026.4.22` or newer, please open a fresh issue with the exact prompt/tool call, actual result, and expected result.
