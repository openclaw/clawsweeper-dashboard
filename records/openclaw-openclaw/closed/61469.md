## What This Changes

This PR makes `emitReasoningStream` broadcast WebSocket `thinking` events without requiring the channel reasoning callback, and widens message handlers to call it outside the prior stream gate.

## Best Possible Solution

Keep https://github.com/openclaw/openclaw/pull/79687 as the canonical implementation and close duplicate reasoning-stream branches that add no unique safe behavior.

## Reproduction Assessment

Yes. Source tracing is high-confidence: current main requires `onReasoningStream`, while Gateway `chat.send` does not provide that callback, so callback-less WebSocket runs cannot emit live `thinking` events through the current path.

## Solution Assessment

No for this branch. The safer solution is the canonical PR because it removes only the callback dependency, preserves `canShowReasoning` and silent-run gates, and adds focused no-callback regression coverage plus real behavior proof.
