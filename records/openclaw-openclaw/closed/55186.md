## Summary

Close #55186 as superseded. Current main still needs Mattermost routing work, but this one-line PR is not the right landing path because `effectiveReplyToId` also drives session-key threading. Open PR #57565 tracks the decoupled `messageThreadId` approach for existing thread delivery with `replyToMode: "off"`, while #60115 separately tracks the docs-consistent DM non-threading behavior.

## Best Possible Solution

Close #55186 as superseded and continue with a decoupled Mattermost fix: preserve `effectiveReplyToId` as the session-routing value, introduce or use separate delivery-thread metadata for existing channel/group threads, update all Mattermost delivery/typing/context call sites, and keep DM behavior aligned with the current docs or an explicit maintainer product decision.

## Evidence

Making the return unconditional would affect session routing, not only delivery.
