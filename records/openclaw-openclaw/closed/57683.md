## Summary

Current `main` already strips `thinkingBudget=0` for Gemini 2.5 Pro in both embedded-runner and native Google payload paths, matching the reported 400 failure. The fix is present on `main` but appears newer than the latest tagged release, so it has not clearly shipped in `v2026.4.23` yet.

## Evidence

- **Shared sanitizer removes zero budget for Gemini 2.5 Pro:** `stripInvalidGoogleThinkingBudget` deletes `thinkingBudget` when the model id is a Gemini 2.5 Pro variant, and `sanitizeGoogleThinkingPayload` applies that sanitizer to both `config` and `generationConfig` payload containers.
  - file: [src/plugin-sdk/provider-stream-shared.ts:237](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/src/plugin-sdk/provider-stream-shared.ts#L237)
  - command: `nl -ba src/plugin-sdk/provider-stream-shared.ts | sed -n '150,390p'`
  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
- **Native Google transport no longer sends disabled zero budget:** The native Google transport builds disabled-thinking config through `getDisabledThinkingConfig()` and immediately normalizes it, which strips the invalid zero budget before the request is sent.
  - file: [extensions/google/transport-stream.ts:214](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/extensions/google/transport-stream.ts#L214)
