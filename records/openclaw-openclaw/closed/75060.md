## Summary

Keep open. Current main still restores invalid configs from last-known-good with generic logs/notices and no rejected validation details, and the issue is paired with open PR #75081 using closing syntax, so it should remain open until that PR lands or is superseded.

## What This Changes

The issue asks OpenClaw to stop silently restoring last-known-good config after invalid `agents.defaults.execution` or malformed `gateway.auth.password` edits by surfacing clear validation diagnostics while preserving safe recovery.

## Best Possible Solution

Land or supersede PR #75081 with a narrow diagnostics-only fix that keeps last-known-good recovery and strict schemas intact while showing concise rejected validation paths/messages in service logs and the main-agent recovery notice.

## Reproduction Assessment

Yes. Static current-main inspection is enough: the reported config shapes are invalid under strict schemas, and the startup/reload recovery paths restore last-known-good with generic notices/logs that do not include rejected validation details.

## Solution Assessment

Yes. The linked diagnostics-only direction is the narrow maintainable solution; accepting the invalid config shapes or relaxing LAN/auth behavior would be a broader product and security decision.
