## What This Changes

The issue asks for `agents.defaults.thinkingDefault: "low"` to be honored for `google/gemini-3-flash-preview` in cron and interactive sessions.

## Best Possible Solution

Keep the shipped `v2026.5.19` Gemini provider profile and advise affected 2026.5.18 users to upgrade.

## Reproduction Assessment

No: the reported 2026.5.18 downgrade path does not match current main or `v2026.5.19`, where `low` is supported for Gemini 3 Flash and maps to `LOW`. The issue's logs are credible for the older release, but current source shows the failing path is no longer reachable for that model/level.

## Solution Assessment

Yes: the current provider-owned profile and transport mapping are the right boundary because they align runtime validation, cron handling, interactive `/think`, docs, and Google's Gemini thinking contract.
