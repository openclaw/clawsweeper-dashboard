## Summary

Keep open. Current main still routes OpenAI Responses, Chat Completions, Azure/OpenAI-compatible fallback, and forced SSE/WebSocket HTTP fallback through the OpenAI SDK parser before OpenClaw can filter parsed chunks. The pinned openai dependency is 6.34.0, the latest checked 6.35.0 still has the same empty-data `JSON.parse(sse.data)` behavior, and I found no OpenClaw regression guard for event-only SSE frames on the OpenAI transport path.

## Best Possible Solution

Keep this issue open and fix it in the OpenAI transport boundary. The best narrow fix is to normalize OpenAI SDK event-stream `Response` bodies before the SDK parser sees them, dropping or coalescing event-only/whitespace-only SSE frames while preserving valid `data:` frames and CRLF handling. Add focused regression coverage for event-only SSE, event-only followed by valid data, and CRLF variants across the OpenAI Responses/Completions transport path; do not conflate this with the separate Control UI UTF-8 streaming issue unless that path is independently reproduced.
