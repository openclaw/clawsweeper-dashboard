## What This Changes

Constrain the diffs viewer toolbar helper so button icons cannot be supplied as arbitrary SVG/HTML strings.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83955 or an equivalent fix that replaces raw icon markup with closed icon IDs or DOM-created SVG elements and rebuilds the viewer runtime.

## Reproduction Assessment

Source-reproducible, but not exploit-reproduced: current main has `iconMarkup: string` flowing into `button.innerHTML`, while current callers are hardcoded icon factories and the payload schema exposes no icon field.

## Solution Assessment

Yes, the closed icon-name union or DOM-created SVG direction is the best maintainable fix. The linked PR follows that direction, while the discussion correctly notes that DOMParser alone would not sanitize event-handler attributes.
