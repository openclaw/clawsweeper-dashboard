## Summary

Keep this PR open. Current main still lacks Telegram `allowBots` config, schema/help metadata, and runtime filtering, while the PR is a focused implementation candidate for the still-open Telegram parity request (#8295). It needs maintainer review rather than cleanup closure, especially because the default behavior would affect existing `channel_post` bot-to-bot flows.

## Best Possible Solution

Keep this PR open and review it as the implementation candidate for Telegram `allowBots` parity. The best path is to land a scoped Telegram plugin/config change with schema/help/generated metadata, regression coverage for both default-drop and opt-in delivery, and explicit docs/changelog or logging for the `channel_post` behavior change.
