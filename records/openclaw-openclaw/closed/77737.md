## Summary

Current main already addresses the reported failure mode: managed `web_search` now resolves runtime web-search providers, keeps external provider discovery enabled, and falls back to a scoped plugin load when the active Gateway registry has no web providers. The proof commit is on `main` only and is not included in a tagged release yet.

## What This Changes

The issue reports that a configured Brave `web_search` provider with a valid API key is unavailable after upgrading to 2026.5.3.

## Best Possible Solution

Keep the generic runtime-provider fallback on main and ship it in the next release; no Brave-specific core workaround is needed.

## Reproduction Assessment

No live Docker/Brave API reproduction was run, but the reported error path and current-main provider-discovery fix are clear from source and regression tests. The current-main path should no longer reach the reported empty-provider error when the Brave plugin manifest and provider are available.

## Solution Assessment

Yes. The current solution fixes the generic plugin web-provider resolution path, including external providers, instead of hard-coding Brave into core.
