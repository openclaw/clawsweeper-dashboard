## Summary

Current `main` already waits for child-process `close`/stream drain before finalizing backgrounded exec sessions, so the reported lost-output race is implemented and shipped in `v2026.4.23`.

## Evidence

- **Child wait settles on close:** The child-process supervisor resolves `wait()` from the process `close` event, not `exit`, which is the key guard against finalizing before pipe buffers flush.
  - file: [src/process/supervisor/adapters/child.ts:251](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/process/supervisor/adapters/child.ts#L251)
  - command: `git show a9797214338ba31c52c796adbb75afb16e0684a9:src/process/supervisor/adapters/child.ts | nl -ba | sed -n '120,255p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Windows path also waits for drain:** On Windows, the adapter tracks stdout/stderr drain and only settles from `exit` after both streams end/close, covering the same early-exit race when `close` is missing.
  - file: [src/process/supervisor/adapters/child.ts:204](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/process/supervisor/adapters/child.ts#L204)
