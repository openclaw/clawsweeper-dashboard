## Summary

Current main no longer uses a launchctl kickstart/bootstrap handoff for config-triggered gateway restarts on macOS launchd. `config.patch`/`config.apply` now schedule a SIGUSR1 restart, the gateway run loop treats launchd as supervised, and the respawn path exits cleanly so launchd `KeepAlive` performs the restart instead of calling `kickstart -k` against a process that just died.

## Evidence

## Risks / Open Questions

- The fix is present on current `main` but I could not tie it to a tagged release newer than `v2026.4.22`, so affected released builds may still need an upgrade once a newer release ships.
- `src/infra/restart.ts` still contains launchctl `kickstart` logic for explicit restart paths, but the config-triggered macOS launchd flow described in this issue no longer goes through that code path.
