## Summary

Current `main` already records successful mutating tool side effects and, if the provider then ends the turn with no visible assistant reply, returns a user-facing warning instead of leaving the session silent. That behavior is also present in the shipped `v2026.4.23` release.

## Evidence

- **Successful mutating tools are preserved as side effects:** `handleToolExecutionEnd` records every completed mutating tool call and sets `replayState.hadPotentialSideEffects=true` on successful mutation, so an `edit`/`write` success is remembered even if the provider errors afterward.

  - file: [src/agents/pi-embedded-runner/run/incomplete-turn.ts:180](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/agents/pi-embedded-runner/run/incomplete-turn.ts#L180)
