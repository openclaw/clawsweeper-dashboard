## What This Changes

The PR adds a subsystem logger and module-level one-shot warning in `resolveSourceReplyDeliveryMode` when group/channel replies fall back to `message_tool_only` because neither group nor global visible-reply config is set.

## Best Possible Solution

Land the resolver-level one-shot warning if maintainers accept the wording and CI proof, while keeping the broader linked issue open or split for changelog wording, docs, config-editing, and migration-policy work.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to call `resolveSourceReplyDeliveryMode` with empty config and `ChatType` set to `group` or `channel`; current tests show it resolves to `message_tool_only`, and dispatch coverage shows the final reply is not sent automatically.

## Solution Assessment

Yes for this PR's warning slice. Placing the warning in the shared resolver is channel-agnostic and fires only when the affected fallback is actually used, but it is not a complete solution for the broader migration issue.
