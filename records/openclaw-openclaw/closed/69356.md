## Summary

Current main already implements the task timestamp ordering fix through a broader normalization path. Task records are normalized on create, update, and restore so audit-visible records no longer surface the startedAt-before-createdAt false positive that #69356 targeted. The PR is now obsolete, though its analysis and follow-up test discussion were useful.

## Best Possible Solution

Close #69356 as implemented on main and keep the broader #71871 normalization path. It solves the same audit-noise bug while also covering update and restore paths; any separate Task Flow timestamp issue should stay in a narrower follow-up rather than merging this now-obsolete PR.
