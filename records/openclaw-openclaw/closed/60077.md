## Summary

The reported `openclaw` ACP harness regression is already implemented on current `main` and was shipped in `v2026.4.23`: ACPX now routes `openclaw acp` bridge sessions through a separate bridge-safe delegate without MCP servers, with regression tests that match the reporter’s `openclaw`-fails/`codex`-works split.

## Evidence

- **ACPX routes OpenClaw bridge sessions through a bridge-safe delegate:** Current runtime detects `openclaw acp` bridge commands, creates a distinct delegate with `mcpServers: []`, and sends `ensureSession` through that delegate instead of the default MCP-enabled path.
  - file: [extensions/acpx/src/runtime.ts:163](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/extensions/acpx/src/runtime.ts#L163)
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Regression test matches the reported harness-specific failure mode:** The test explicitly asserts that `openclaw` `ensureSession` uses the bridge-safe delegate when MCP servers are configured.
  - file: [extensions/acpx/src/runtime.test.ts:147](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/extensions/acpx/src/runtime.test.ts#L147)
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Control test preserves normal behavior for other ACP harnesses:** A neighboring test confirms non-`openclaw` ACP sessions still use the default delegate, matching the reporter’s finding that `codex` worked while `openclaw` failed.
  - file: [extensions/acpx/src/runtime.test.ts:178](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/extensions/acpx/src/runtime.test.ts#L178)
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)

  - file: [docs/cli/acp.md:181](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/docs/cli/acp.md#L181)
