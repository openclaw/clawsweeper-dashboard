## Summary

Keep open: the PR is aimed at the right bug, but the patch currently breaks the gateway server-methods build and still misses the fetch failures it is meant to surface.

## What This Changes

The PR adds in-memory model-pricing bootstrap failure tracking and attempts to expose it through health/status JSON plus human-readable health and status output.

## Best Possible Solution

Keep the additive health/status direction, but preserve the status RPC import, record per-source catalog fetch failures where they are currently swallowed, and render gateway-sourced pricing degradation in health, openclaw status, and gateway status with focused regression coverage.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: make the pricing catalog fetch throw, observe refreshGatewayModelPricingCache catches the source errors and resolves, then the new outer ledger never records the degraded state. A typecheck would also reproduce the missing getStatusSummary import.

## Solution Assessment

No, this is not the best way to solve the issue yet. The fix should record failures at the per-source catch points or in shared pricing cache state, and status rendering should consume gateway-owned state rather than a CLI-local summary.
