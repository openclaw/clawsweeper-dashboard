## What This Changes

Request to encode cross-field config dependencies so enabling hooks requires a non-empty hooks.token before config writes or startup accept the configuration.

## Best Possible Solution

Use PR #54831 as the canonical implementation target: land a tested shared validation rule for hooks.enabled=true requiring a non-empty hooks.token across config.patch, config.set/apply, writeConfigFile, and plugin-aware validation, while keeping resolveHooksConfig as the startup backstop. Broader dependency metadata in config.schema.lookup should be split only if maintainers want clients to preflight dependencies generically.
