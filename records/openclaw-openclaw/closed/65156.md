## Summary

Keep open. Current main and v2026.5.27 still trust a successful sqlite-vec load call without probing that vec0/vec_version actually registered, so the reported load-success/no-functions failure remains a valid memory vector readiness bug. The related Windows suffix PR is narrower and does not cover the cross-platform ABI/readiness check.

## What This Changes

The issue requests fixing builtin memory vector indexing when sqlite-vec loads but registers no functions, causing chunks_vec writes to be skipped and semantic recall to degrade to FTS.

## Best Possible Solution

Add a post-load sqlite-vec readiness probe on the actual memory DB, fail soft with an explicit loadError when vec_version or vec0 is absent, and keep the existing FTS fallback without adding a config or dependency-pinning workaround.

## Reproduction Assessment

Yes for a source-level repro, not live-confirmed here: use a DB/mock where sqlite-vec load returns success but vec_version or vec0 is absent, then index any chunk with embeddings. The current code will trust load success before proving registered functions.

## Solution Assessment

Yes: the narrowest maintainable fix is a readiness probe and fail-soft unavailable state at the sqlite-vec loader/vector boundary. Rebuilding or pinning sqlite dependencies may be useful later, but OpenClaw should first detect the bad loaded state reliably.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/65156 in the builtin memory sqlite-vec readiness path. Current main treats a successful loadExtension/sqliteVec.load as availability, but a bad ABI or broken extension can load without registering vec_version/vec0. Add a post-load health probe on the active DB, preferably in packages/memory-host-sdk/src/host/sqlite-vec.ts or the adjacent memory-core sync boundary, that verifies sqlite-vec functions/modules are usable before returning ok:true or setting vector.available=true. If the probe fails, return ok:false with an actionable error, make ensureVectorReady fail soft, and preserve FTS fallback. Add regression tests that simulate load success with missing vec_version/vec0 and keep existing explicit extensionPath/platform-variant behavior. Do not add config knobs, pin or patch dependencies, change QMD behavior, or broaden memory defaults.
