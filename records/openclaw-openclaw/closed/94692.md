## Label Justifications

- `P1`: The reported latest-release regression makes Telegram channel replies unreadable and uncopyable for real users, even though current main already fixes it.
- `impact:message-loss`: Readable Telegram reply content is replaced by unsupported-message placeholders, effectively losing the delivered message content for affected clients.

## Summary

Current main already fixes the reported Telegram v2026.6.8 unsupported-message regression by making standard sendMessage/HTML delivery the default and rich-message delivery explicit opt-in; the latest release tag remains affected, so this is a main-only implementation close.

## What This Changes

The issue asks for Telegram bot replies in v2026.6.8 to render as normal copyable text, or for a supported config option to disable rich-message delivery.

## Best Possible Solution

Keep the current-main default-off rich-message implementation and ship it in the next patch release rather than opening another Telegram delivery switch issue.

## Reproduction Assessment

Yes. The v2026.6.8 tag shows ordinary Telegram text routed through `sendRichMessage` with no `richMessages` schema key, and the reporter supplied matching screenshots; I did not run a live Telegram repro in this read-only review.

## Solution Assessment

Yes. Current main solves the issue at the right boundary by making compatible sendMessage/HTML delivery the default and keeping Bot API rich messages as explicit opt-in.
