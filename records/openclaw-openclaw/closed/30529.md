## Summary

Current main solves the central user problem through automatic Discord typing prestart plus a keepalive heartbeat for active runs; the exact manual message-tool action remains unimplemented, but it is no longer necessary for the requested “agent is working” feedback and would require a separate portable API decision.

## What This Changes

Provide visible Discord typing feedback while an agent processes a message, either automatically or through a message-tool action.

## Best Possible Solution

Keep automatic, channel-owned Discord typing as the supported processing signal; track richer placeholders and tool-progress feedback in https://github.com/openclaw/openclaw/issues/90866, and require a separate cross-channel design if explicit agent-controlled presence is still wanted.
