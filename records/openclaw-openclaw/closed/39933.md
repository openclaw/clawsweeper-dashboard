## Summary

Close as `cannot_reproduce`. Current `main` is a thin wrapper around `@mariozechner/pi-tui`, now pinned to `0.70.0`, while the mirrored upstream Ghostty/macOS report against `pi-tui` `0.55.3` was closed as non-repro and a related stray-`c` Kitty CSI-u bug was fixed upstream long ago. I did not find any OpenClaw-local input path that would still independently inject printable `c` characters on top of pi-tui.

## Evidence

- **OpenClaw TUI delegates input handling to pi-tui:** `src/tui/tui.ts:513-521` constructs `new TUI(new ProcessTerminal())` and only adds a backspace deduper. `src/tui/components/custom-editor.ts:16-64` forwards all unmatched input to `super.handleInput(data)`, so OpenClaw does not add its own printable-`c` insertion path here.
  - file: [src/tui/tui.ts:513](https://github.com/openclaw/openclaw/blob/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b/src/tui/tui.ts#L513)
  - command: `nl -ba src/tui/tui.ts | sed -n '500,540p'; nl -ba src/tui/components/custom-editor.ts | sed -n '1,120p'`
  - sha: [b98a6a46fb54](https://github.com/openclaw/openclaw/commit/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b)
- **Current main and latest release both ship much newer pi-tui than the report referenced:** `package.json:1588` on current `main` pins `@mariozechner/pi-tui` `0.70.0`. The latest OpenClaw release commit `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5` already pinned `0.69.0` in `package.json:1551`. The mirrored upstream report referenced `pi-tui` `0.55.3`.
  - file: [package.json:1588](https://github.com/openclaw/openclaw/blob/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b/package.json#L1588)
