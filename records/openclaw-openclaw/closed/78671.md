## Summary

Keep open: current main still has a source-reproducible cached plugin tool dispatch bug where manifest-level tool contracts can be used as per-factory identity for unnamed registrations.

## What This Changes

The issue reports that cached plugin descriptor tools can dispatch to the first unnamed tool factory because plugin-level declared tool names are treated as per-factory runtime identity.

## Best Possible Solution

Update cached descriptor execution to treat `declaredNames` as plugin-level ownership only, dispatch by the actual returned runtime tool name, and lock the multi-unnamed-factory case with a focused regression test.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: cache descriptors for a plugin with two unnamed factories under `contracts.tools = ["a", "b"]`, then execute cached tool `b` and the wrapper can select the first factory by shared `declaredNames`.

## Solution Assessment

Yes. A wrapper-side dispatch fix is the narrowest maintainable path because docs already define `contracts.tools` as ownership/discovery metadata and execution must call the live registered implementation.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/78671. In cached descriptor execution, do not treat plugin-level `declaredNames` as per-factory identity for unnamed `api.registerTool(ctx => tool)` registrations. In `src/plugins/tools.ts`, load only the owner plugin and resolve cached execution by actual returned `runtimeTool.name`; use explicit `entry.names` as a direct identity hint and use `declaredNames` only as a coarse ownership/availability filter. Add a regression in `src/plugins/tools.optional.test.ts` with one plugin whose manifest `contracts.tools` contains two names and whose registry has two unnamed factories in order; after the first resolve populates descriptors, cached execution of the second tool must call/return the second factory instead of throwing `plugin tool runtime missing`. Keep the test on the small fake registry helpers and do not depend on lossless-claw. Add a single-line `CHANGELOG.md` fix entry with appropriate human credit, such as Thanks @100yenadmin, and do not add a new config option or SDK requirement.
