## Summary

Current main already handles this failure mode conservatively: stale persisted ACP resume ids are dropped after one failed resume attempt, stale bound conversations are unbound on `Resource not found`/missing-metadata ACP init failures, and regression tests cover both the dead-session recovery and duplicate-thread-binding cleanup paths.
