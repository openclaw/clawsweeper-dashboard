## Repair Work Prompt

Fix the Feishu Docker/package startup regression reported at https://github.com/openclaw/openclaw/issues/76291. Current main bundles `@larksuiteoapi/node-sdk@1.62.1` into the ESM Feishu output, and the published SDK uses bare `__dirname`, causing Feishu to exit in Docker. Keep the fix in the packaging/plugin-runtime boundary: likely update `tsdown.config.ts` so the Lark SDK is not bundled into the ESM plugin output, add or update focused regression coverage in the tsdown/package-manifest/runtime staging tests, and verify the final Docker/runtime image can still resolve the externalized SDK when `OPENCLAW_EXTENSIONS=feishu` is used. Add a user-facing `CHANGELOG.md` fix entry. Do not move Feishu-owned behavior into core, do not broaden plugin APIs unless strictly necessary, and stop if the repair would require a new config option or product decision.
