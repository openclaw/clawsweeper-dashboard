## Summary

Current `main` does not unconditionally start a gateway on nodes where no service is installed; the PR's `shouldRestart = false` change would also suppress the intentionally supported unmanaged-gateway restart path.

## Evidence

- **Update flow still delegates restart handling to daemon lifecycle:** `updateCommand` only prepares a restart script when `resolveGatewayService().isLoaded()` is true, but otherwise still passes `shouldRestart` into `maybeRestartService`, which calls `runDaemonRestart()`. The behavior therefore depends on the current daemon restart lifecycle, not on unconditional process launch in `updateCommand` itself.
  - file: [src/cli/update-cli/update-command.ts:1376](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/src/cli/update-cli/update-command.ts#L1376)
  - command: `nl -ba src/cli/update-cli/update-command.ts | sed -n '1368,1408p'`
  - sha: [d068cb960de4](https://github.com/openclaw/openclaw/commit/d068cb960de47d5587c5942f6beb86e0f124b449)
- **Current restart lifecycle does not start anything when service is not loaded and no recovery path exists:** `runServiceRestart()` handles the not-loaded case by calling `onNotLoaded`; if that returns nothing, it emits `Gateway service not loaded.` plus start hints and returns `false` instead of launching a new gateway.
  - file: [src/cli/daemon-cli/lifecycle-core.ts:398](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/src/cli/daemon-cli/lifecycle-core.ts#L398)
  - command: `nl -ba src/cli/daemon-cli/lifecycle-core.ts | sed -n '345,470p'`
  - sha: [d068cb960de4](https://github.com/openclaw/openclaw/commit/d068cb960de47d5587c5942f6beb86e0f124b449)
- **Unmanaged restart is an intentional current-main behavior:** `runDaemonRestart()` explicitly prefers sending `SIGUSR1` to a verified unmanaged gateway listener when the service is not loaded. Tests cover that path and assert it is preferred over launchd recovery when a listener exists.
  - file: [src/cli/daemon-cli/lifecycle.ts:203](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/src/cli/daemon-cli/lifecycle.ts#L203)
  - command: `nl -ba src/cli/daemon-cli/lifecycle.ts | sed -n '120,220p'; nl -ba src/cli/daemon-cli/lifecycle.test.ts | sed -n '358,415p'`
  - sha: [d068cb960de4](https://github.com/openclaw/openclaw/commit/d068cb960de47d5587c5942f6beb86e0f124b449)
- **Mac recovery is limited to already-installed LaunchAgents:** The only auto-recovery path after `onNotLoaded` is `recoverInstalledLaunchAgent()`, which first checks for an existing LaunchAgent plist; if the service was never installed, it returns `null` and nothing is started. Tests cover the installed-LaunchAgent recovery path separately.
  - file: [src/cli/daemon-cli/launchd-recovery.ts:14](https://github.com/openclaw/openclaw/blob/d068cb960de47d5587c5942f6beb86e0f124b449/src/cli/daemon-cli/launchd-recovery.ts#L14)
  - command: `nl -ba src/cli/daemon-cli/launchd-recovery.ts | sed -n '1,40p'; nl -ba src/cli/daemon-cli/lifecycle.test.ts | sed -n '386,415p'`
  - sha: [d068cb960de4](https://github.com/openclaw/openclaw/commit/d068cb960de47d5587c5942f6beb86e0f124b449)
- **Existing PR review already identified the regression in this patch:** The Codex review comment on this PR points out that setting `shouldRestart = false` when `isLoaded()` is false would skip the intentional unmanaged-process restart path and leave manually run gateways on old code/config until a manual restart.

## Risks / Open Questions

- If someone still sees `openclaw update` start a gateway on a truly non-installed node, the remaining likely differentiators are platform-specific state such as an existing unmanaged listener or a leftover macOS LaunchAgent plist; that would need a fresh reproduction against current `main`.
