## Summary

Close #69645 as duplicate/superseded by the earlier open PR #69421. Current main still does not implement the requested package-update no-op skip, and this PR duplicates the same #69412 work while carrying unresolved contract concerns around JSON output and forced package-spec reinstall flows.

## Best Possible Solution

Close #69645 as the duplicate PR and keep the remaining product/code review on #69421 or a maintainer-authored follow-up. The final implementation should preserve the existing UpdateRunResult JSON contract, avoid skipping explicit forced reinstall/package-spec flows, and align src/cli/update-cli/update-command.ts, src/cli/update-cli.test.ts, and docs/cli/update.md.
