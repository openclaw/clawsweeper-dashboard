## Summary

Keep open: current main and the latest release still contain the reported stdout write followed by synchronous exit, and the linked fixing PR is open but unmerged.

## What This Changes

The issue reports that the codex-cli metadata script can truncate piped JSON output because it calls process.exit(0) immediately after process.stdout.write.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84239 or an equivalent drain-safe script fix, ideally with a focused regression check, then close this issue after merge.

## Reproduction Assessment

Yes, from source inspection: current main and v2026.5.20 still write JSON to stdout and immediately call process.exit(0), while Node documents this as a stdout truncation risk. I did not run a live failing pipe reproduction in this read-only review, so this is source-reproducible rather than directly reproduced. ([nodejs.org](https://nodejs.org/api/process.html))

## Solution Assessment

Yes: the maintainable fix is to let the process exit naturally after emitting exactly one JSON payload, such as by setting process.exitCode and skipping the rest of the script. The open linked PR appears to take that direction, but the issue should remain open until the fix is merged.
