## Summary

Current `main` already contains and ships the session-routing fix that preserves established external channel delivery routes when Control UI/webchat touches an externally originated session. That matches this report’s core symptom: replies still appear in UI history, but later automatic deliveries stop reaching Telegram/Slack after the route gets overwritten away from the external channel.

## Evidence

- **Route preservation logic on main:** `src/auto-reply/reply/session-delivery.ts:103` prevents webchat/internal turns from overwriting an established external route; `:113`-`:159` also preserves `lastTo` for inter-session/webchat cases so later deliveries do not get lost.
