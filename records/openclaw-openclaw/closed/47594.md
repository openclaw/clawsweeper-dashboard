## Summary

Current main already threads `--profile` into `process.env` before gateway lifecycle handlers run, and `resolveGatewayLifecyclePort()` already reads that environment. The PR’s stated root cause does not match current code. Its added fallback logic would also overwrite explicit `OPENCLAW_LAUNCHD_LABEL` overrides, which current launchd code and tests treat as higher priority than `OPENCLAW_PROFILE`.

## Evidence
