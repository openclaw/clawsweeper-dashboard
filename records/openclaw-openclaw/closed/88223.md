Keep open. Current main and v2026.6.9 still wire WhatsApp composing to the shared reply lifecycle and still send composing immediately before outbound message send, so the requested accepted-inbound responsiveness behavior is not implemented and needs maintainer direction plus live WhatsApp proof.

## What This Changes

The issue requests faster WhatsApp typing feedback and message-display responsiveness, including accepted-inbound composing, removal of redundant send-time composing, a shorter typing refresh cadence, and a possible WhatsApp-specific config option.

## Best Possible Solution

Have maintainers choose the WhatsApp typing lifecycle/API boundary, then implement a focused WhatsApp or shared accepted-inbound composing path with live transport proof and without adding a redundant config surface unless explicitly approved.

## Reproduction Assessment
