## Summary

Current main already implements the WAL-mode memory index behavior through the memory-core database opener and shared WAL maintenance helper. The implementation is now in the current `extensions/memory-core` path rather than the older `src/memory` path touched by this PR, retains `busy_timeout` to avoid writer-vs-writer regressions, and has focused WAL helper coverage. The PR diff is obsolete and does not introduce security-sensitive infrastructure, dependency, or CI changes.

## Best Possible Solution

Close this PR as implemented on current main. Keep the shipped shared WAL maintenance implementation, which solves the central memory index WAL request in the current memory-core location and intentionally preserves `busy_timeout`; any future desire to force `synchronous = NORMAL` specifically should be a separate maintainer-reviewed tuning follow-up.
