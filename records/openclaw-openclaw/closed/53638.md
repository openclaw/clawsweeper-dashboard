## Summary

Keep open. Current main and the latest release already provide the shared `channels.modelByChannel` surface for channel/group overrides and wildcard direct-chat defaults, but ordinary per-DM target keys still do not reach the resolver; the open linked PR is the current fix candidate and should be reviewed or replaced before this issue closes.

## What This Changes

The issue requests persistent config-level model selection by channel, group, and DM instead of ephemeral per-session `/model` overrides.

## Best Possible Solution

Keep the existing shared `channels.modelByChannel` surface, add a direct-chat-only DM candidate path with documented key forms, preserve channel/group precedence, and close this issue after the linked fix or an equivalent replacement merges.

## Reproduction Assessment

Yes, at source level. Current main has no direct peer parameter in `ChannelModelOverrideParams`, and the direct match candidate list uses only group and parent-session keys, so ordinary DM-specific entries cannot match except through wildcard fallback.

## Solution Assessment

Yes for the generic direction. Finishing `channels.modelByChannel` for direct conversations is narrower and more maintainable than adding provider-specific nested `model` fields, but the linked PR still needs review before it can close the issue.
