## Summary

Keep open. Current main still blocks IPv6 ULA DNS answers under web_fetch even when the RFC2544 fake-IP opt-in is enabled, and the remaining question is a security/product policy decision rather than a cleanup closure.

## What This Changes

The issue requests that tools.web.fetch.ssrfPolicy.allowRfc2544BenchmarkRange also allow fake-IP proxy IPv6 ULA answers in fc00::/7, or that OpenClaw add a separate opt-in for that range.

## Best Possible Solution

Define the SSRF contract explicitly, then either add a separate default-off IPv6 ULA fake-IP opt-in with schema, help text, docs, cache-key, audit, and regression coverage, or document that ULA remains intentionally blocked for web_fetch.
