## Summary

The report targets OpenClaw 2026.3.31, but current main already contains Matrix gateway crash-hardening and crypto-runtime recovery paths that cover the reported startup-crash class, with changelog and test evidence showing the gateway now degrades to channel-scoped failure/warnings instead of taking down the whole process.

## Evidence

- **Matrix changelog records shipped gateway crash-hardening:** The Matrix changelog says release 2026.4.9 waits for Matrix sync readiness, contains background handler failures, and routes fatal Matrix sync stops through channel-level restart handling instead of crashing the whole gateway.
  - file: [extensions/matrix/CHANGELOG.md:72](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/extensions/matrix/CHANGELOG.md#L72)
- **Matrix plugin startup no longer treats crypto bootstrap failure as fatal:** During plugin registration, Matrix now attempts crypto bootstrap and downgrades failures to warnings (`crypto runtime bootstrap failed` / `failed loading crypto bootstrap runtime`) instead of aborting startup.
  - file: [extensions/matrix/index.ts:18](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/extensions/matrix/index.ts#L18)
- **Missing native crypto runtime is auto-repaired via download-lib.js:** `ensureMatrixCryptoRuntime` detects missing `@matrix-org/matrix-sdk-crypto-nodejs`, runs the dependency's `download-lib.js`, and retries the load.
  - file: [extensions/matrix/src/matrix/deps.ts:149](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/extensions/matrix/src/matrix/deps.ts#L149)
- **Encrypted-room runtime degrades to warning instead of crash when crypto is unavailable:** Matrix room-event handling emits a one-time warning with a native dependency hint if encryption is enabled but crypto bindings are unavailable.
  - file: [extensions/matrix/src/matrix/monitor/events.ts:342](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/extensions/matrix/src/matrix/monitor/events.ts#L342)
- **Regression coverage for crypto-unavailable and fatal-sync behavior:** Tests cover both the one-time warning path for unavailable crypto bindings and the fatal-sync path as a channel-task failure with error-state reporting, rather than an uncontrolled gateway crash.
  - file: [extensions/matrix/src/matrix/monitor/events.test.ts:1403](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/extensions/matrix/src/matrix/monitor/events.test.ts#L1403)
- **Channel-task failure behavior is explicitly tested:** The monitor test expects a fatal Matrix sync error to reject the Matrix channel task and mark status as error after releasing the shared client.
  - file: [extensions/matrix/src/matrix/monitor/index.test.ts:605](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/extensions/matrix/src/matrix/monitor/index.test.ts#L605)
