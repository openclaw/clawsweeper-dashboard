The issue reports that OpenAI-compatible Chat Completions replay drops `reasoning_content` from historical assistant tool-call messages for MiMo/Kimi thinking models, causing follow-up turns to fail.

## Best Possible Solution

Add narrow provider/model-specific replay preservation for Kimi K2.6 preserved-thinking tool-call history and MiMo V2.6 Pro, while keeping the stock OpenAI/Qwen reasoning-strip default intact.

## Solution Assessment

Yes. The best fix is narrow provider/model compatibility logic with regression coverage, not a global revert of the OpenAI-compatible reasoning-strip behavior needed for other providers.
