## Summary

Current `main` points this kind of optional command-guard behavior to plugins/ClawHub, and the repo already exposes the needed seams: `before_tool_call` can require user approval before execution, while the bundled `command-logger` hook covers audit logging. The issue itself and a follow-up comment already point to working third-party implementations, so this does not need new OpenClaw core work.
