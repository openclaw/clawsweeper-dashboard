## Summary

Current main already fixes the Control UI blob-avatar problem this PR targets: the gateway CSP allows `blob:` images, the chat avatar fetch path converts authenticated local avatar routes into managed object URLs, and the chat rendering helper accepts those locally-created blob URLs without broadening the config-sourced avatar regex. The PR’s discussion and Greptile review correctly identify the affected CSP/UI surfaces, but main now covers the observable bug with narrower source-backed behavior and tests.

## Best Possible Solution

Close this PR as already implemented on main. Keep the narrower main implementation: allow `blob:` in the Control UI CSP and in the locally-managed chat render path, while continuing to reject arbitrary config-sourced `blob:` strings through the generic avatar URL regex unless a maintainer explicitly decides to broaden that trust boundary later.
