## Summary

Current `main` and shipped `v2026.4.23` already route `cron` thin-RPC commands through lazy CLI/gateway boundaries and a loaded-only channel registry path, so the client-side Codex harness startup path described in this report is no longer present in code.

## Evidence

- **Cron subcli is lazy-loaded:** The root CLI registers `cron` via `loadModule: () => import("../cron-cli.js")`, so the cron command group is not materialized until that subcommand is selected.
  - file: [src/cli/program/register.subclis-core.ts:119](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/cli/program/register.subclis-core.ts#L119)

- **Gateway RPC path stays behind a cached dynamic import:** CLI gateway calls go through `loadGatewayRpcRuntime()` and `import("./gateway-rpc.runtime.js")`, keeping the heavier RPC runtime off the registration path.
  - file: [src/cli/gateway-rpc.ts:9](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/cli/gateway-rpc.ts#L9)
