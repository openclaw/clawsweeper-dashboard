## Summary

Current `main` already resolves isolated cron runs from `payload.model`, persists that selection before the run, feeds the selected provider/model into the embedded runner's Runtime prompt, and bounds `LiveSessionModelSwitchError` retries. The changelog says the cron live-switch loop fix shipped in `v2026.3.31`.

## Evidence

- **Cron model selection prefers payload.model:** `resolveCronModelSelection()` reads `payload.model`, validates it, and sets the provider/model from that override before falling back to stored session or agent defaults.
  - file: [src/cron/isolated-agent/model-selection.ts:104](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/cron/isolated-agent/model-selection.ts#L104)
- **Isolated cron persists selected model before execution:** `prepareCronRunContext()` resolves the model selection, then calls `markCronSessionPreRun()` and persists the session entry before the run starts.
  - file: [src/cron/isolated-agent/run.ts:518](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/cron/isolated-agent/run.ts#L518)
- **Runtime prompt uses the actual run provider/model:** The embedded runner builds Runtime info from `params.provider` and `params.modelId`, so the system prompt Runtime line reflects the selected cron run model rather than a separate primary-model default.
  - file: [src/agents/pi-embedded-runner/run/attempt.ts:887](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/agents/pi-embedded-runner/run/attempt.ts#L887)
- **Model-switch retries are bounded and persisted:** `executeCronRun()` catches `LiveSessionModelSwitchError`, updates the live selection, persists it, and aborts after 2 retries instead of looping forever.
  - file: [src/cron/isolated-agent/run-executor.ts:300](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/cron/isolated-agent/run-executor.ts#L300)
- **Regression tests cover forwarding and retry behavior:** Tests assert that cron payload model overrides reach `runWithModelFallback` and the embedded runner, and that `LiveSessionModelSwitchError` retries switch to the requested model and do not loop indefinitely.
  - file: [src/cron/isolated-agent/run.cron-model-override-forwarding.test.ts:108](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/cron/isolated-agent/run.cron-model-override-forwarding.test.ts#L108)
- **Shipped release note:** The changelog for `2026.3.31` says isolated cron jobs now carry the full live-session provider/model/auth-profile selection across retry restarts so model overrides no longer fail or loop on mid-run model-switch requests.
  - file: [CHANGELOG.md:1741](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/CHANGELOG.md#L1741)
