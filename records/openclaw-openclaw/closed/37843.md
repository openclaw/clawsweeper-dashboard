## Summary

Close: current main still intentionally treats cron `payload.model` as a strict explicit model ref, while merged follow-up work superseded the request with doctor cleanup for bad persisted sentinels and a `--clear-model` inheritance path.

## What This Changes

Support cron agent-turn `payload.model: "default"` as a special value that resolves to the runtime default model instead of a literal model id.

## Best Possible Solution

Keep cron runtime model refs strict, use doctor and `--clear-model` to remove bad or stale stored overrides, and open a new maintainer-sponsored design only if a portable sentinel is still desired.

## Reproduction Assessment

Yes. Current source gives a high-confidence reproduction path: `payload.model: "default"` is a non-empty string, so cron validates it as an explicit model ref rather than inheriting the default.

## Solution Assessment

No. The one-line sentinel guard is no longer the best fit because merged maintainer work chose strict runtime refs plus doctor cleanup and explicit clear semantics instead.
