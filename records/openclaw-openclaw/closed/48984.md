## Summary

Current `main` already prevents a subagent from resolving `/subagents kill` against sibling sessions or falling back to killing itself. Control is now scoped to the caller's own child tree, and leaf subagents are denied subagent-control tools entirely.

## Evidence

- **Leaf subagents no longer get control tools:** The subagent tool policy always denies `sessions_send`, and additionally denies `subagents`, `sessions_list`, `sessions_history`, and `sessions_spawn` for leaf subagents. That matches the default two-sibling reproduction path from the report.
  - file: [src/agents/pi-tools.policy.ts:34](https://github.com/openclaw/openclaw/blob/925d11d89054c5dd1ba31de5537d2ee3cb5cdd3b/src/agents/pi-tools.policy.ts#L34)
  - command: `nl -ba src/agents/pi-tools.policy.ts | sed -n '1,170p'`
  - sha: [925d11d89054](https://github.com/openclaw/openclaw/commit/925d11d89054c5dd1ba31de5537d2ee3cb5cdd3b)
- **Controller resolution is session-scoped:** `resolveSubagentController()` uses the caller session as the controller for subagent calls, `listControlledSubagentRuns()` only returns runs whose latest controller/requester session key matches that caller, and `ensureControllerOwnsRun()` rejects other sessions' runs. This removes the sibling-target resolution failure described in the issue.
  - file: [src/agents/subagent-control.ts:115](https://github.com/openclaw/openclaw/blob/925d11d89054c5dd1ba31de5537d2ee3cb5cdd3b/src/agents/subagent-control.ts#L115)
  - command: `nl -ba src/agents/subagent-control.ts | sed -n '1,210p'`
  - sha: [925d11d89054](https://github.com/openclaw/openclaw/commit/925d11d89054c5dd1ba31de5537d2ee3cb5cdd3b)
- **Leaf subagents are explicitly forbidden from killing other sessions:** `killControlledSubagentRun()` returns `forbidden` with `Leaf subagents cannot control other sessions.` before any kill happens when the caller control scope is `none`.
  - file: [src/agents/subagent-control.ts:350](https://github.com/openclaw/openclaw/blob/925d11d89054c5dd1ba31de5537d2ee3cb5cdd3b/src/agents/subagent-control.ts#L350)
