## Summary

Keep open. The reporter narrowed the original incident to a manual source-level global install, but current main still lacks the requested guard/diagnostic for an npm-global package root that resolves through a symlink into a source checkout; the closest implementation candidate is still the open related PR #74920.

## What This Changes

Detect and report package-installed OpenClaw layouts where the global package root or service entrypoint resolves into a mutable source checkout, especially after update/dev-channel flows.

## Best Possible Solution

Add a shared install-layout diagnostic that preserves both the lexical package root and resolved realpath, detects package roots or service entrypoints pointing into source checkouts, and surfaces that state in status, doctor, and update pre/post verification while keeping documented dev-channel source installs explicit.

## Reproduction Assessment

Yes. The issue includes direct systemd log evidence for the source install command, and current main has focused test/code evidence that a symlinked OpenClaw root into a git repo is classified as a git install rather than flagged as a mixed package/source layout.

## Solution Assessment

Yes, the diagnostic/guard direction is the narrowest maintainable fix. Removing dev-channel source installs would conflict with documented behavior, while preserving lexical root plus realpath lets OpenClaw distinguish intentional source installs from hazardous mixed layouts.
