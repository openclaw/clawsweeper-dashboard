## Summary

Current main already downloads inbound Feishu chat attachments and exposes them to the agent context automatically, which covers the requested attachment-download/analyze workflow without needing a separate `feishu_download_attachment` tool.

## Close Comment

Closing from a Codex review because current `main` already covers the requested behavior well enough. Feishu inbound attachments are downloaded through `extensions/feishu/src/media.ts:276`, processed for inbound message types in `extensions/feishu/src/bot-content.ts:329`, and injected into the agent turn context in `extensions/feishu/src/bot.ts:712` and `extensions/feishu/src/bot.ts:1015`. Those attachments are then surfaced to the agent through `MediaPath`/`MediaPaths` (`src/plugin-sdk/agent-media-payload.ts:13`) and prompt plumbing is covered by `src/auto-reply/reply.media-note.test.ts:5`. I also confirmed the same implementation is present in shipped release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).

The only missing piece is the exact standalone tool shape proposed in the issue, but the stated user workflow of sending a Feishu file/video/PDF to the agent so it can download and analyze it is already implemented. If there is still a need for an explicit callable tool beyond the existing automatic attachment-ingestion path, please open a narrower follow-up issue for that API surface.
