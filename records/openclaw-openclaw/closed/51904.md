## Summary

Current main already routes `memory_search` through the resolved `agents.*.memorySearch` provider config instead of hardcoding OpenAI, and the same wiring is present in the shipped `v2026.4.22` tag.

## Evidence

- **Config resolution:** `resolveMemorySearchConfig` resolves the provider from agent override/default config with `overrides?.provider ?? defaults?.provider ?? "auto"`, alongside the model, remote config, and fallback.
  - file: [src/agents/memory-search.ts:153](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/src/agents/memory-search.ts#L153)
- **Provider request propagation:** The memory manager forwards the resolved provider, model, remote settings, fallback, and local settings into embedding setup via `resolveMemoryPrimaryProviderRequest`.
  - file: [extensions/memory-core/src/memory/manager-provider-state.ts:20](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/extensions/memory-core/src/memory/manager-provider-state.ts#L20)
- **Manager initialization:** `MemoryIndexManager` calls `createEmbeddingProvider(...)` with the resolved memory-search provider request and stores `settings.provider` as `requestedProvider` for status/reporting.
  - file: [extensions/memory-core/src/memory/manager.ts:143](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/extensions/memory-core/src/memory/manager.ts#L143)
- **Tool wiring:** `memory_search` obtains the shared memory manager with `{ cfg, agentId }` and returns `status.provider` / `status.model`; there is no OpenAI-specific instantiation path in the tool.
  - file: [extensions/memory-core/src/tools.ts:204](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/extensions/memory-core/src/tools.ts#L204)
- **Shipped release proof:** The same provider-resolution and `memory_search` wiring are present in the `v2026.4.22` tag, so this behavior is not only on current `main` but also in the latest shipped release provided in the review context.
  - command: `git show v2026.4.22:src/agents/memory-search.ts | nl -ba | sed -n '145,210p'; git show v2026.4.22:extensions/memory-core/src/tools.ts | nl -ba | sed -n '200,320p'; git show v2026.4.22:extensions/memory-core/src/memory/manager.ts | nl -ba | sed -n '143,170p;692,728p'`
