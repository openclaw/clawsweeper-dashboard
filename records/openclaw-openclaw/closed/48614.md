## Summary

Keep open. Current `main` does wire `MediaPath`/`MediaType` from the Signal inbound entry, but the Signal fetch path still ignores `attachment.filename` and saves attachments using only `attachment.contentType`. With `signal-cli` voice notes that omit `contentType`, `saveMediaBuffer` gets no filename context for MIME inference, so the attachment can still degrade to `application/octet-stream` instead of audio. An unmerged PR (#61143) exists specifically to add the missing filename-based MIME resolution, so this is not implemented on `main` yet.
