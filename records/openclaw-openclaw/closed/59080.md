## Summary

Current main has a narrow transport-sentinel fix from #59118, but the raw unclassified `JSON.parse` message shape from this issue is still deliberately preserved in formatter coverage, and #60593 is the active broader tracker for recurring Anthropic/tool-call JSON parse errors leaking to user channels and missing failover.

## What This Changes

The issue reports that malformed tool-call JSON after session compaction can surface a raw JavaScript `JSON.parse` error to user-facing channels instead of a graceful error or retry.

## Best Possible Solution

Keep the remaining work consolidated under #60593: classify malformed streaming/tool-call JSON parser failures at the provider/transport boundary, make them retryable or failover-worthy where appropriate, and ensure every channel receives friendly sanitized text without hiding provider request-validation diagnostics.
