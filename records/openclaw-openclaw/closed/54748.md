## Summary

SambaNova is not implemented on current main, but the PR is a service-specific model-provider integration that current OpenClaw plugin surfaces can support outside the core repo. The project vision and plugin docs direct optional capabilities, including providers, toward external plugin packages and ClawHub unless there is a maintainer-owned reason to bundle them.

## Best Possible Solution

Close this core PR and move the SambaNova work into a standalone native provider plugin published through ClawHub or npm. If maintainers later want SambaNova bundled by default, open a separate maintainer-owned proposal covering official ownership, security review, support expectations, and catalog maintenance.
