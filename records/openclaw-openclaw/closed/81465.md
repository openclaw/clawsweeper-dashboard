## Summary

Keep open. Current main and the latest release still match the reported Dream Diary failure class: request-scoped narrative fallback can write raw snippets, timeout/no-text paths return before a diary append, transcript denoising does not directly reject the literal routing text named in the issue, and `memory status` still has no narrative fallback count.

The issue asks memory-core Dreaming to stop silently falling back to raw snippets, filter internal routing metadata from session-corpus ingestion, and make Dream Diary fallback visible in status output.

## Best Possible Solution

Keep this as the canonical v2026.5.7 Dream Diary fallback/denoising tracker, land or replace the focused fallback and diary-hardening PRs, and leave the public execution fallback contract to the linked config issue.
