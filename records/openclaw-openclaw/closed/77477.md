## Summary

Keep open: the source-level bug is real and the active-config pass-through is the right direction, but the current PR head still sends already-handled bundled channels into plugin fallback when a non-channel plugin entry is configured.

## What This Changes

The PR passes active config into plugin doctor contract discovery and doctor warning/fix collection, adds load-path doctor contract regression tests, and appends a changelog entry.

## Best Possible Solution

Keep the active-config pass-through, but compute fallback plugin ids from unresolved channels plus configured non-channel plugin entries only, with regression coverage for a handled bundled channel plus a non-channel plugin.

## Reproduction Assessment

Yes, at source level. Current main omits active config when resolving plugin doctor contracts while manifest discovery uses config.plugins.load.paths, so a load-path-only plugin can be runtime-visible but doctor-invisible.

## Solution Assessment

No for the current PR head. Passing active config through doctor discovery is the narrow maintainable direction, but fallback id selection should exclude bundled channels already handled by channel doctor normalizers.
