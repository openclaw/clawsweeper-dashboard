## Summary

Keep open: the doctor load-path bug is valid and the PR takes the right direction, but the latest head still routes already-handled channel ids back through plugin doctor fallback during `doctor --fix`. The protected `maintainer` label also means merge remains maintainer-gated.

## What This Changes

The branch passes active config through plugin doctor contract, warning, session-owner, and `doctor --fix` discovery paths; adds load-path plugin coverage and a changelog entry; and includes small cleanup fixes.

## Best Possible Solution

Keep the active-config pass-through, but restrict `doctor --fix` plugin fallback to unresolved channel ids plus configured non-channel plugin entries and add a handled-channel-plus-plugin regression.

## Reproduction Assessment

Yes, at source level. Current main omits active config when resolving plugin doctor contracts while manifest discovery depends on `config.plugins.load.paths`, so a load-path-only plugin can be runtime-visible but doctor-invisible.

## Solution Assessment

No for the current PR head. Passing active config through doctor discovery is the narrow maintainable fix, but fallback id selection should exclude configured channels that were already handled by bundled channel doctor normalizers.
