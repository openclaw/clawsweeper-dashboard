## Summary

Current `main` already contains multiple bounded shutdown/restart protections that address this report's indefinite CLI hang during `openclaw gateway restart` / `openclaw gateway stop` on Linux systemd.

## Evidence

- **Restart path is now explicitly bounded:** `runDaemonRestart` computes a fixed restart-health window and fails with a timeout instead of waiting forever.
  - file: [src/cli/daemon-cli/lifecycle.ts:193](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/cli/daemon-cli/lifecycle.ts#L193)

  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)
- **Gateway shutdown no longer waits indefinitely on websocket/http close:** Shutdown now uses websocket and HTTP grace windows plus force-continue / force-close handling, which directly matches the reported symptom of restart/stop hanging after the service action already happened.
  - file: [src/gateway/server-close.ts:13](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/gateway/server-close.ts#L13)
