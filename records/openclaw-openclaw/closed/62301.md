## Summary

Current `main` already contains a targeted fix for the exact late-exec-update race behind `Error: Agent listener invoked outside active run`, plus cleanup changes for pending tool results. The fix is documented in the unreleased changelog, so this looks resolved on `main` but not yet shipped in the latest tagged release `v2026.4.23`.

## Evidence

- **Exec runtime now suppresses late updates after run end:** The exec runtime tracks `updatesDisabled` specifically to prevent the `"Agent listener invoked outside active run"` crash, and disables updates before marking the session exited or on runtime failure.
  - file: [src/agents/bash-tools.exec-runtime.ts:561](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/agents/bash-tools.exec-runtime.ts#L561)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Abort path disables updates before teardown:** The exec tool abort handler now calls `run.disableUpdates()` before kill/background guards, covering the reported race where stdout/stderr arrives after the agent run has already ended.
  - file: [src/agents/bash-tools.exec.ts:1716](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/agents/bash-tools.exec.ts#L1716)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Cleanup now waits for idle before flushing pending tool results:** Embedded-run cleanup waits for agent idle and can clear pending tool results on timeout, which matches the report's secondary symptoms around aborted tools and orphaned messages.
  - file: [src/agents/pi-embedded-runner/run/attempt.subscription-cleanup.ts:17](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/src/agents/pi-embedded-runner/run/attempt.subscription-cleanup.ts#L17)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Changelog documents the exact crash fix on main:** `Unreleased` changelog entry: `Agents/exec: prevent gateway crash ("Agent listener invoked outside active run") when a subagent exec tool produces stdout/stderr after the agent run has ended or been aborted. (#62821)`
  - file: [CHANGELOG.md:1011](https://github.com/openclaw/openclaw/blob/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46/CHANGELOG.md#L1011)
  - sha: [ec3dbd22a4de](https://github.com/openclaw/openclaw/commit/ec3dbd22a4dea8da4bcc0d0832da48a83ae27d46)
- **Related discussion already pointed at the same race:** The issue timeline cross-references closed issue `#62520` (`Gateway crash: background exec output after subagent run completes triggers unhandled rejection in pi-agent-core (exec-after-run race)`), which is the same failure mode the current code comments and changelog reference.
