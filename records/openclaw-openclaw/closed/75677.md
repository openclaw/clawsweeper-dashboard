## Summary

Keep open, but the current patch should not merge: it turns semantic TUI activity state keys into display text, removes the English default without a locale contract, and lacks after-fix real behavior proof.

## What This Changes

The PR replaces TUI status, connection, Ctrl+C/tool messages, activity status values, and default waiting phrases with hardcoded Simplified Chinese across four `src/tui` files.

## Best Possible Solution

Add a TUI localization boundary that keeps typed activity states stable, maps display strings by locale, defaults and falls back to English, and documents/tests locale selection.

## Reproduction Assessment

Yes for the PR-introduced regressions: source inspection shows `setActivityStatus` writes the state later compared against exact `waiting` and `streaming` keys. I did not run tests because this review is read-only, but current tests already assert those keys.

## Solution Assessment

No. Hardcoding zh-CN over existing literals is not the narrow maintainable solution; the safer path is a locale-aware display layer with stable semantic state and English fallback.
