## Summary

Current main does not reproduce the reported docs/code mismatch: `appPrincipal` is still optional in schema and types, and runtime only requires it for the narrower Google Workspace add-on `app-url` token path. The related fix PR was closed unmerged after the same conclusion, because making the field globally required would reject valid `project-number` and normal Chat-issuer configs.

## What This Changes

Report asks for Google Chat config schema/docs to stop treating `channels.googlechat.accounts.*.appPrincipal` and `accounts` as optional because webhook auth appears to require `appPrincipal`.

## Best Possible Solution

Keep `appPrincipal` optional globally and handle the add-on `app-url` token case with the existing targeted rejection/warning; any remaining concern should be a narrower docs clarification for that add-on path, not a global required-field change.

## Reproduction Assessment

No. Source and tests on current main show valid app-url Chat-issuer and project-number flows without `appPrincipal`; only add-on issuer tokens reject a missing binding by design.

## Solution Assessment

Yes. The current optional schema/type contract plus add-on-specific runtime check is the narrow maintainable behavior; making `appPrincipal` universally required would break valid configs and the current setup flow.
