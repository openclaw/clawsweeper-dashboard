## Summary

Current `main` already implements direct HTTP hook ingress for agent runs. OpenClaw exposes `POST /hooks/agent`, accepts agent/task fields including `message`, `agentId`, `sessionKey`, delivery options, and model overrides, then dispatches the run and returns a `runId`. The behavior is documented and covered by gateway hook tests, so the feature request is already satisfied on `main` and in the latest release evidence checked.
