## Summary

Close as implemented on current main: the v2026.4.29 ACPX package still declared bundled runtime-deps staging, while current main removes that staging contract, excludes ACPX's dist tree from the core npm package, and treats `plugin-runtime-deps` as legacy cleanup only. The proof commit is on main and not contained in a release tag yet.

## What This Changes

The report asks OpenClaw to stop recreating and populating `~/.openclaw/plugin-runtime-deps` on gateway startup when plugins and ACPX are disabled.

## Best Possible Solution

Keep the current externalized ACPX/plugin-dependency model, ship it in the next release, and leave stale `plugin-runtime-deps` removal to `openclaw doctor --fix`.

## Reproduction Assessment

Yes. The issue gives concrete macOS/npm v2026.4.29 steps, and the v2026.4.29 tag still contains the ACPX `stageRuntimeDependencies` metadata that matches the reported log; this read-only review verified the source path rather than running the gateway.

## Solution Assessment

Yes. Removing bundled startup dependency staging and externalizing ACPX is the narrow maintainable fix because it makes startup/reload dependency-free instead of adding more skip-specific guards.
