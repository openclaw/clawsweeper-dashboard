## Summary

Close as `clawhub`: the PR adds optional `sessions_send` observability/audit logging to core, but current OpenClaw already exposes plugin hooks that can observe tool calls, results, errors, duration, and session context. The related structured signal-layer proposal #50072 was already closed toward the ClawHub/plugin path, and the current PR does not identify a missing core API that blocks an external plugin implementation.

## Best Possible Solution

Close this core PR and move the idea to a ClawHub/plugin implementation that registers an `after_tool_call` hook, filters `sessions_send`, and writes operator-controlled audit JSONL with opt-in config, retention, and privacy controls. If that plugin path exposes a concrete missing SDK field later, open a narrow plugin API issue instead of merging one-off core logging.
