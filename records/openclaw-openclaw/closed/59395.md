## Summary

Current main implements the reported fake-IP/TUN workaround as an explicit `web_fetch` SSRF policy opt-in. The original linked PR #59443 was closed unmerged, but the central request's proposed allowlist path shipped later as `tools.web.fetch.ssrfPolicy.allowRfc2544BenchmarkRange`, with schema support, runtime wiring, SSRF policy handling, tests, and a v2026.4.10 changelog entry.

## Best Possible Solution

Keep the shipped opt-in implementation: it solves the Clash/Surge fake-IP `198.18.0.0/15` failure without globally weakening SSRF protection. If maintainers want proxy-mode DNS delegation beyond this, track that separately from this now-implemented fake-IP allowlist request.
