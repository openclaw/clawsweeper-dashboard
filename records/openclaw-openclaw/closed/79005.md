## Summary

Current main and the latest release already route Telegram forum messages through topic-qualified session keys, with source, docs, and regression coverage matching the requested behavior.

## What This Changes

Route Telegram forum-group topics to separate session keys that include `:topic:<topic_id>`.

## Best Possible Solution

Keep the shipped Telegram plugin behavior and use the existing topic-qualified routing/session contract for any future topic-specific fixes.

## Reproduction Assessment

No, current main does not reproduce the reported bleed: forum supergroup messages with `message_thread_id` route to `agent:main:telegram:group:<group>:topic:<thread>`, and regression tests assert that behavior.

## Solution Assessment

Yes, the existing plugin-owned Telegram routing is the narrow maintainable solution; no core change is needed because the forum thread id is already part of the route/session key.
