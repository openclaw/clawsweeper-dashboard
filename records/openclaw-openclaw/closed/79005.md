## Summary

Current main already isolates Telegram forum topics in session keys, and the behavior shipped in v2026.1.29; the source, tests, and user docs all now include the `:topic:<threadId>` routing path.

## What This Changes

The issue requests topic-aware Telegram group session keys so forum topics in the same group do not share conversation context.

## Best Possible Solution

Keep the shipped Telegram plugin behavior and route any fresh reports to a new issue with current-version logs if a specific forum update still misses `is_forum` or `message_thread_id`.

## Reproduction Assessment

No; current-main source and regression coverage show a forum supergroup with `message_thread_id: 99` routes to `agent:main:telegram:group:-1001234567890:topic:99`, so the reported shared group session does not match the checked-out code.

## Solution Assessment

Yes; the current solution stays inside the Telegram plugin, derives topic identity from Telegram’s forum thread metadata, and feeds the topic-qualified peer id into the generic session-key builder without adding core channel-specific policy.
