## Summary

Keep open. Current main and v2026.6.9 still use a one-shot post-stop port check that can report a false macOS LaunchAgent stop/update failure, and this maintainer-labeled PR is a focused fix with no blocking review finding from this read-only pass.

## What This Changes

The PR adds a short bounded poll to macOS LaunchAgent stop verification so a briefly busy gateway port can finish releasing after bootout before OpenClaw reports failure.

## Best Possible Solution

Land or revise this as the scoped stop/update postcondition fix after normal maintainer and CI gates, while leaving the distinct restart-before-kickstart work to https://github.com/openclaw/openclaw/issues/88309 and https://github.com/openclaw/openclaw/pull/89096.

## Reproduction Assessment

Yes from source inspection: current main has a single busy-port postcondition check after LaunchAgent stop, so a busy-then-free port sequence throws before a second observation. I did not run the failing test because this review was required to keep the checkout read-only.

## Solution Assessment

Yes for the scoped stop/update race: a bounded post-bootout poll keeps the existing fail-closed behavior while tolerating launchd's asynchronous port release. Broader restart semantics are separate and already tracked by the related restart issue/PR.
