  - body: This second-pass check only exact-matches normalized tool names, so supported entries such as `group:plugins`, plugin ids like `bundle-mcp`, and globs like `outlook__*` can pass the original allowlist path and then get stripped here.   - confidence: 0.91

  - body: For `message_tool_only` turns, current main force-adds `message` so the model can deliver the visible source reply; the PR now returns an empty or plugin-only allowlist unchanged, leaving no `message` tool. That can turn a channel reply into an undelivered private final or no model-facing delivery path.
  - confidence: 0.9

If Gateway requested tools are part of the threat model, thread the actual turn policy into those callers or remove this unproven surface.
