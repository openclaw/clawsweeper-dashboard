## Summary

The issue reports that model-fallback retries can duplicate queued user messages or assistant failure stubs in persisted session transcripts, corrupting future provider requests.

## Best Possible Solution

Land a reviewed fix through https://github.com/openclaw/openclaw/pull/83417 or a narrow replacement that makes fallback-cascade transcript persistence idempotent for queued users and assistant error stubs, then close this issue with merge and proof links.
