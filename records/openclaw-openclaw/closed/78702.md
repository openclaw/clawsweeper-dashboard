## Summary

Keep open. Current main has useful mitigations for missing externalized channel plugins, but it still does not fully solve the reported Feishu upgrade/status outage: Feishu is excluded from the core package, missing-plugin validation is warning-only, and the merged recovery path is scoped to onboarding/doctor repair rather than the package-upgrade/status path.

## What This Changes

The issue asks for upgraded Feishu installs to either auto-install `@openclaw/feishu` after built-in-to-external migration or surface a hard, actionable offline error.

## Best Possible Solution

Keep Feishu externalized, but make configured missing official external channels impossible to miss in status/startup with the exact install or `openclaw doctor --fix` repair command unless maintainers explicitly approve update-time installation.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction shape: keep `channels.feishu` or `plugins.allow: ["feishu"]` configured while `@openclaw/feishu` is absent from the core install. I did not run the historical macOS npm upgrade end to end in this read-only review.

## Solution Assessment

Unclear. The current onboard fix in https://github.com/openclaw/openclaw/pull/78328 is useful, but a complete fix should address upgrade/status/startup visibility without violating the documented no-package-manager-on-startup policy unless maintainers explicitly change that policy.
