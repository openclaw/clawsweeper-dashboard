## Summary

Current `main` already suppresses TUI/non-webchat agent events from the Control UI chat path. Runs are marked `isControlUiVisible` only for the internal `webchat` channel, hidden runs drop `sessionKey` during event emission, and the gateway chat broadcaster skips chat/session delivery when `isControlUiVisible` is false. Regression tests cover both the hidden-session-key case and the no-broadcast case.

## Evidence
