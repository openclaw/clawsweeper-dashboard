## Summary

Current main already implements the PR’s intended Discord allowlist behavior in the active bundled Discord plugin: when a guild has a non-empty channels map, unmatched channels resolve to allowed=false and are blocked by groupPolicy=allowlist. The paired bug report #54277 is closed, and any future change to make channels additive overrides is now tracked separately by #73387.

## Best Possible Solution

Close this stale PR as implemented on current main. Keep the current fail-closed Discord allowlist behavior for the original #54277 bug, and use #73387 for any future product decision to change Discord channel override semantics.
