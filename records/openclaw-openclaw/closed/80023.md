## Summary

Keep this PR open for normal review: current main still drops ACP cause-chain detail, and the PR has real log/CLI proof plus green GitHub checks. I found one low-priority formatter edge case, but no blocking correctness or security issue.

## What This Changes

The PR adds redacted ACP error-chain formatting and propagates it through ACP lifecycle, background-task, gateway, and reply failure surfaces, with regression tests, a changelog entry, and small CLI command-hint formatting changes.

## Best Possible Solution

Land the narrow ACP diagnostic propagation after maintainers decide whether to clean up the top-level non-Error formatter edge case; keep the ACP timeout-control root cause in the separate linked ACP PRs.

## Reproduction Assessment

Yes from source inspection: current main stores or emits only `acpError.message`/`String(err)` on the ACP lifecycle, task, and gateway failure paths, which drops nested cause detail. The PR body and https://github.com/openclaw/openclaw/issues/78546#issuecomment-4414010521 provide live after-fix log evidence, but I did not run a live current-main ACP repro.

## Solution Assessment

Yes, mostly: formatting the ACP error chain at the ACP/gateway lifecycle boundary and redacting before log/reply exposure is the narrow maintainable shape. A small direct non-Error object case should use the same JSON helper if maintainers want the formatter’s no-`[object Object]` guarantee to be complete.
