## Summary

Needs changes before merge: the occupied-port preflight is the right direction, but the current branch drops the OAuth authorize URL and can leave users at a manual prompt with no URL to open.

## What This Changes

Adds a localhost:1455 availability preflight to the OpenAI Codex OAuth core wrapper and bundled OpenAI runtime copy, with a regression test and changelog entry.

## Best Possible Solution

Keep the port-conflict preflight, suppress automatic browser opening, and show or log the authorize URL with manual-open instructions in both OAuth wrappers, with regression coverage for URL availability.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the upstream helper still calls onAuth with the authorize URL, and this PR's busy-port branch returns before that URL is exposed.

## Solution Assessment

No, not as written. The preflight is maintainable, but the fix must preserve a manual authorization URL while still avoiding openUrl on the occupied callback port.
