Keep this PR open for maintainer judgment. Current main still omits the proposed Slack `assistant_view` and `assistant_thread_started` manifest entries, and the PR is not superseded by a shipped implementation. A maintainer already commented that OpenClaw is intentionally holding off on this Slack Assistant UI feature for now, so this is a product/default-experience decision rather than conservative auto-close cleanup.
