## Summary

Keep open. Current main still has overflow compaction, memory-flush, memory-promotion, and session cleanup primitives, but it does not surface the requested deduped promote-then-rotate lifecycle suggestion or operator confirmation flow.

## What This Changes

Add a lifecycle suggestion that detects repeated session context overflow, dry-runs memory promotion candidates, and offers operator-confirmed promote/rotate actions.

## Best Possible Solution

Define a lifecycle policy that tracks repeated per-session overflow pressure, emits one deduped operator suggestion with a stable id and candidate count, and keeps promotion or rotation behind explicit confirmation.

## Reproduction Assessment

Yes for the product gap: current-main source and the overflow-loop regression show repeated overflow still flows through auto-compaction and terminal overflow handling without a promote-then-rotate suggestion. I did not run tests because this was a read-only review.

## Solution Assessment

Unclear pending maintainer design approval. A suggestion-and-confirmation flow is safer than automatic promotion or rotation, but it should be reconciled with existing memoryFlush, truncateAfterCompaction, and memory review-quality work.
