## Summary

Close as implemented on main and shipped in v2026.5.2. The reported 2026.4.29 symptoms line up with plugin/pricing hot paths that are now addressed by scoped plugin runtime loading, request-time registry reuse, plugin tool descriptor caching, and deferred pricing refresh.

## What This Changes

The issue asks OpenClaw to stop the 2026.4.29 startup and per-turn slowdown where plugin prep appears to scan/load all registered plugins and block on pricing fetches.

## Best Possible Solution

Keep the v2026.5.2 plugin hot-path implementation, and ask for a fresh current-version startup trace only if the same Windows configuration still shows multi-second `core-plugin-tools` or pricing delays.

## Reproduction Assessment

No high-confidence current-main reproduction was established. The historical 2026.4.29 report is concrete, but current v2026.5.2 source and release notes show the reported eager plugin runtime and pricing-startup paths have been changed.

## Solution Assessment

Yes. Closing against the shipped v2026.5.2 hot-path fixes is the narrowest maintenance outcome; a new issue should carry fresh `OPENCLAW_GATEWAY_STARTUP_TRACE=1` and per-turn prep timings if current releases still regress.
