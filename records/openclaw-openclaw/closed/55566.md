## Summary

Current main still uses a shared fallback global lane for embedded runs, but that lane is configured to allow parallel cross-session execution by default (`agents.defaults.maxConcurrent`, default 4). The report’s claim that different Discord-channel sessions still serialize solely because they share the embedded fallback lane does not match current main behavior.

## Evidence

- **Embedded runner still uses session lane plus shared global lane:** `runEmbeddedPiAgent` enqueues by per-session lane, then enqueues inside `resolveGlobalLane(params.lane)`. With no explicit lane, `resolveGlobalLane()` returns `main`.
  - file: [src/agents/pi-embedded-runner/run.ts:249](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/agents/pi-embedded-runner/run.ts#L249)
  - command: `nl -ba src/agents/pi-embedded-runner/run.ts | sed -n '240,320p'`
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Fallback lane is not single-file serialization on current main:** Gateway startup applies concurrency for `CommandLane.Main` from `resolveAgentMaxConcurrent(cfg)`, so the shared `main` lane can process multiple runs in parallel.
  - file: [src/gateway/server-lanes.ts:6](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/gateway/server-lanes.ts#L6)
  - command: `nl -ba src/gateway/server-lanes.ts | sed -n '1,20p'`
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Default main-lane concurrency is 4:** `resolveAgentMaxConcurrent({})` falls back to `DEFAULT_AGENT_MAX_CONCURRENT`, which is `4`.
  - file: [src/config/agent-limits.ts:3](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/config/agent-limits.ts#L3)
  - command: `sed -n '1,160p' src/config/agent-limits.ts`
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Runtime config materialization injects the default when unset:** Config materialization calls `applyAgentDefaults`, which writes `agents.defaults.maxConcurrent = 4` when the setting is absent.
  - file: [src/config/defaults.ts:296](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/config/defaults.ts#L296)
  - command: `nl -ba src/config/defaults.ts | sed -n '286,308p'`
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
