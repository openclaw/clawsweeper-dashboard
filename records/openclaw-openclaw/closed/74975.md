## Summary

Current main and the latest release already address the reported restart path: OpenClaw-owned config writes now publish canonical source snapshots to the reload watcher, compare source-to-source, and skip the duplicate filesystem watcher pass by persisted hash. The issue was reproduced on 2026.4.26, while v2026.4.27 shipped the related phantom plugin diff/unnecessary Gateway restart fix.

## What This Changes

Patching `agents.defaults.model.primary` through `config.patch` should hot-apply without a Gateway restart or unrelated runtime/materialized config diffs.

## Best Possible Solution

Keep the v2026.4.27/current-main source-snapshot reload behavior and have affected 2026.4.26 users upgrade to the shipped release or newer.

## Reproduction Assessment

Yes. The issue provides concrete CLI steps and logs for 2026.4.26, and current-main regression coverage exercises the same phantom runtime-materialized diff class without scheduling a restart.

## Solution Assessment

Yes. Canonicalizing post-write source snapshots and deduping the watcher event is the narrow maintainable fix because it preserves normal external edit validation while preventing OpenClaw-owned writes from being re-diffed against runtime-only fields.
