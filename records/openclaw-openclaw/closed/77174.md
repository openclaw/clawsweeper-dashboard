## Summary

Keep open: the bug is valid at source level, but this draft, maintainer-labeled PR still has a correctness blocker because the reaction cleanup can turn a successful tapback into a failed action.

## What This Changes

The PR exports BlueBubbles typing to the action runtime, clears typing after reaction actions, and tracks pending typing restart promises before final reply cleanup.

## Best Possible Solution

Keep the reaction typing cleanup, but make the stop-typing call best-effort and logged while preserving the monitor pending-restart ordering.

## Reproduction Assessment

Yes, at source level: current main's BlueBubbles react handler sends the reaction and returns without clearing typing, and the PR's action test target can cover the missing cleanup. I did not run a live BlueBubbles/iMessage E2E in this read-only review.

## Solution Assessment

No. Clearing typing after reactions and ordering final reply cleanup are the right boundaries, but the reaction cleanup should be best-effort/logged like the existing reply stop path so stop failures do not mask a successful tapback.
