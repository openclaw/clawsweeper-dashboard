## Summary

Current `main` already contains the WhatsApp split-runtime listener-state fix, and the changelog shows it shipped in `v2026.3.23`, which postdates the reported broken `2026.3.13` build. The outbound send path now resolves the active listener from a process-global registry shared across split runtime chunks, matching the issue’s suspected root cause.
