## Summary

Current main already covers the linked gateway restart failure with a safer launchd handoff plus restart-intent path. The PR’s hand-rolled SIGUSR1 self-signal is obsolete and the existing review comments correctly identify it as risky because it bypasses the authorized restart helper and can self-signal helper processes.

## Best Possible Solution

Close PR #44518 and keep current main’s launchd handoff plus restart-intent implementation. Do not merge the PR’s unauthenticated manual SIGUSR1 self-signal; any remaining launchd restart bug should be filed against the current handoff/restart-intent path with fresh logs and runtime state.
