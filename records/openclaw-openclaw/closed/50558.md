## Summary

Issue #50558 is the same remaining request as open issue #18160: a direct shell/exec cron payload that avoids agentTurn/LLM sessions. Current main still only exposes systemEvent and agentTurn cron payloads in types, protocol schema, CLI, and runtime validation, and the related implementation PR #51276 was closed unmerged, so #18160 is the canonical tracker rather than this issue being implemented.

## Evidence
