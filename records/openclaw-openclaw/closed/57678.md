## Summary

Current `main` already preserves unquoted frontmatter values containing `: `, and both `skills.status` and prompt generation now use the same workspace skill loader, so the reported silent-drop/inconsistent-visibility bug is no longer present on `main`.

## Evidence

- **Frontmatter parser falls back safely:** `parseFrontmatterBlock` combines a YAML parse with a line-based parser. If YAML parsing fails it returns the line-based result, and if YAML interprets a colon-bearing inline value as structured data it prefers the original inline string instead.
  - file: [src/markdown/frontmatter.ts:55](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/src/markdown/frontmatter.ts#L55)
  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
- **Regression coverage for colon-containing descriptions:** There is an explicit unit test asserting that an unquoted description containing `: ` is preserved verbatim.
  - file: [src/markdown/frontmatter.test.ts:71](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/src/markdown/frontmatter.test.ts#L71)
  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
- **Status and prompt share the same skill-entry loader:** `buildWorkspaceSkillStatus` calls `loadWorkspaceSkillEntries`, and prompt generation also starts from the same `loadSkillEntries` path, so `skills.status` and the available-skills prompt no longer rely on divergent loaders for this case.
  - file: [src/agents/skills-status.ts:228](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/src/agents/skills-status.ts#L228)
  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
- **Local skill loading no longer drops this case:** `loadSingleSkillDirectory` only drops a skill if `name` or `description` are absent after parsing. With the current frontmatter fallback/parser merge, the reported colon-in-description case still produces a description and remains loadable.
  - file: [src/agents/skills/local-loader.ts:44](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/src/agents/skills/local-loader.ts#L44)
  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
