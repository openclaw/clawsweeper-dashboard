## What This Changes

The issue requests that `openclaw gateway status` wait for a bounded GatewayClient WebSocket close drain before the probe promise resolves.

## Best Possible Solution

Update `probeGateway()` to use a bounded `stopAndWait()` cleanup before resolving, with focused coverage that preserves existing probe timeout/error semantics and proves teardown ordering.
