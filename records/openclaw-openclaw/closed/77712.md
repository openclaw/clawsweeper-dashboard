## What This Changes

The issue reports cron/status delivery previews showing `Unsupported channel: feishu` for configured Feishu announce jobs even though delivery succeeds.

## Best Possible Solution

Land a generic outbound target resolver bootstrap fix after exact-head validation so cron previews use configured channel plugin availability without adding Feishu-specific cron code.

## Reproduction Assessment

Yes, at source level: cron preview dry-runs `resolveDeliveryTarget`, which can call `resolveOutboundTarget` without channel bootstrap and return the reported unsupported-channel error. I did not run a live Feishu cron setup in this read-only review.

## Solution Assessment

Yes, the open PR’s generic use of the existing `allowBootstrap` seam is the narrowest maintainable direction. It keeps channel target ownership in plugins and avoids hard-coding Feishu in cron preview code.
