## Summary

Current main and the shipped v2026.4.26 release already fix the Windows drive-letter ESM loader class behind this Slack startup crash. Slack is a bundled channel that loads through the shared bundled-channel module loader, and that path now converts raw Windows absolute paths to file:// URLs before Jiti sees them and disables native Jiti on Windows.

## Best Possible Solution

Close as fixed and point affected Windows users to upgrade from 2026.4.24 to v2026.4.26 or newer. If Slack still reports the same raw c: ESM loader error on v2026.4.26/current main, the follow-up should be a fresh issue with logs from that version because the shared bundled channel loader now covers the reported import path.
