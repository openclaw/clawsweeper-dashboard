## Summary

Current main already fixes the reported onboard/setup crash through a different implementation: bundled channel metadata is read directly from the resolved bundled plugin tree in installed builds, and docs-link formatting now tolerates missing `docsPath`. This PR’s catalog-only approach is obsolete on main and would still synthesize broken npm install targets for private bundled channels under the current external catalog loader.

## Evidence

- **Bundled channel metadata now loads from the resolved bundled plugin tree first:** `listBundledChannelCatalogEntries()` reads `package.json` files from `resolveBundledPluginsDir()` and only falls back to `dist/channel-catalog.json` if that yields nothing, so installed builds can surface bundled channel metadata from `dist/extensions` directly.
