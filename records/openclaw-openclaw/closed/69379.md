## Summary

Keep this PR open. Current main still computes the preflight-compaction and memory-flush token thresholds from `reserveTokensFloor` only, while the config help and Pi settings path treat the floor as a minimum under `reserveTokens`. The paired bug #66830 remains open, the earlier related PR #67297 is closed unmerged, and the supplied PR diff is a focused source/test/changelog change rather than a security-sensitive infrastructure change.

## Best Possible Solution

Keep this PR open as the implementation candidate for #66830. The best path is to review and land this helper or equivalent `max(reserveTokens, reserveTokensFloor)` logic through both preflight-compaction and memory-flush gates, keep the regression coverage, and let the paired bug close only after the fix merges.
