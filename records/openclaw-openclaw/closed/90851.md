## Summary

The exact `/model --set-default` spelling is not present, but adding another global config-write path is a follow-up UX decision rather than a necessary core item.

The issue requests an in-chat command path to persist the default agent model instead of only setting a session-scoped model override.

## Best Possible Solution

Keep the shipped owner-approved Crestodian rescue path as the supported chat-to-config write for persistent model defaults; consider a `/model --set-default` alias only as a separate UX/product decision.

## Reproduction Assessment

Not applicable; this is a feature request, and source/docs/test evidence show current main already has a chat-originated persistent default-model write path through Crestodian rescue.

## Solution Assessment

Yes for cleanup; the existing Crestodian path is the safer solution because it wraps persistent config writes in owner-DM policy and explicit approval instead of making ordinary `/model` silently change global defaults.
