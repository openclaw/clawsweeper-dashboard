## What This Changes

The PR makes Google browser Talk prefer Gateway relay, adds a realtime transport config escape hatch, adjusts relay readiness/rebinding, changes Google Live silence handling, and includes a small diagnostic-session cleanup.

## Best Possible Solution

Rework the useful transport-default idea on the current unified Talk APIs, use the existing talk.realtime.transport contract, preserve connection-bound relay ownership, and prove the result in a real browser/Gateway Google relay run.

## Reproduction Assessment

Not applicable as a strict bug reproduction. Current source and docs show the observable behavior: browser Talk tries client-owned provider sessions first and uses Gateway relay only after fallback or explicit transport selection.

## Solution Assessment

No. The direction may be useful, but this branch is stale against the current Talk architecture and needs a safer relay ownership design plus maintainer agreement on the default transport policy.
