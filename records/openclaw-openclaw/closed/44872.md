## Summary

Close as ClawHub/plugin-scope. Current main does not ship `/default_model`, but it already has core paths for temporary model switching (`/model`), model browsing (`/models`), and persistent default changes via owner-gated config writes or CLI `openclaw models set`. The PR is an optional convenience command plus Telegram picker, and current plugin surfaces can implement it outside core with command registration, config write helpers, model-provider data helpers, and Telegram interactive callbacks.

## Best Possible Solution

Do not merge this stale core/Telegram patch into OpenClaw core. Publish the command as a ClawHub plugin that registers `/default_model`, uses model-provider runtime helpers for the picker/list, applies the same command/config-write authorization rules, writes `agents.defaults.model.primary` through config-runtime, and handles Telegram callbacks through the plugin interactive handler surface. If maintainers want this built in, open a focused design issue for first-party command semantics and tests.
