## Summary

Keep open. This is a maintainer-labeled MEMBER PR, so cleanup closure is not allowed, and current main still does not implement the artifact RPC surface. The PR also still has a P2 correctness blocker: run/task queries resolve a session and then discard transcript artifacts unless each message already carries matching run/task provenance, but current transcript reads only attach `id` and `seq` metadata and no current-main writer stores those provenance fields.

## What This Changes

The PR adds SDK-facing Gateway `artifacts.list/get/download` RPCs, read-scope registration, protocol and Swift models, SDK `oc.artifacts` helpers/types/docs, changelog coverage, and task-scope lookup/download-source hardening.

## Best Possible Solution

Land one canonical maintainer-approved artifact RPC implementation that keeps the read-scope gating, SDK/docs/protocol synchronization, and unsafe download guardrails, while either persisting and reading real run/task provenance before filtering or narrowing the public contract until that provenance exists.

## Reproduction Assessment

Yes. On PR head, read ordinary session messages through current `readSessionMessages()` semantics, then call `collectArtifactsFromMessages()` with a `runId` or `taskId`; messages without explicit run/task metadata are skipped even though the same artifacts appear for `sessionKey`.

## Solution Assessment

No, not as-is. The RPC direction is maintainable, but the advertised run/task query surface is not backed by persisted transcript provenance, so the safer fix is to add that provenance or defer/narrow those scopes.
