## Summary

Keep open. Current main still creates isolated cron run session keys and the general agent ingress path can materialize a user-triggered run under an explicit requested key without blocking `agent:*:cron:*:run:*`, so the reported duplicate side-effect path remains plausible and actionable.

## What This Changes

Prevent inbound user or DM turns from reusing a closed isolated cron run session key and re-executing the cron task context.

## Best Possible Solution

Add a generic core guard so cron run session keys are internal-only: cron-triggered execution may use them, while user/channel ingress rejects or remaps them to the normal routed session.

## Reproduction Assessment

Unclear for a full live Telegram timing reproduction in this read-only sweep. Source inspection does show a narrow reproducible boundary: a non-cron `agent.run` can target an `agent:<id>:cron:<job>:run:<run>` key and be dispatched as a user run.

## Solution Assessment

Yes. The narrowest maintainable fix is a gateway/session ingress guard around cron run keys with regression coverage, rather than a Telegram-specific workaround or new configuration option.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/78074. Current main lets non-cron user/channel `agent.run` requests target `agent:<id>:cron:<jobId>:run:<runId>` session keys, which can materialize a fresh user-triggered session under a closed isolated cron run's prompt. Add a generic core guard in gateway/session routing so cron run keys are internal-only: allow them for cron-triggered execution (`bootstrapContextRunKind === "cron"` or the cron executor path), but reject or remap user/channel/webchat/DM ingress to the normal routed session rather than persisting or running under the cron run key. Do not add Telegram-specific behavior, do not add new config, and do not weaken isolated cron execution. Add regression coverage for `agent.run` with a cron run key and keep isolated cron execution coverage green. Add a CHANGELOG.md fix entry because this is user-facing behavior.
