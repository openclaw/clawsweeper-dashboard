## Summary

Current main already covers the reported behavior: normal user agent turns run through the embedded agent path, configured `mcp.servers` entries are merged into the bundle-MCP runtime, materialized as provider-safe agent tools, filtered through the normal tool-policy pipeline, and included in the effective tool list. The behavior is also documented and present in the `v2026.4.26` release tree.

## What This Changes

Expose MCP tools configured under `mcp.servers` to the main OpenClaw agent, or clearly document if that registry is limited to embedded runtimes.

## Best Possible Solution

Keep the shipped runtime shape: `mcp.servers` remains the central registry, and configured MCP servers are exposed during agent turns as `bundle-mcp` tools with provider-safe names such as `server__tool`, governed by the existing profile and allow/deny policy.
