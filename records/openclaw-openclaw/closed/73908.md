## Summary

This issue should stay open. Current `main` still has the unscoped external CLI auth overlay path: `models.authStatus` loads the auth store before provider eligibility is computed, the auth overlay calls `resolveExternalCliAuthProfiles` without config/provider scope, and that function iterates Codex, Claude CLI, and MiniMax credential readers. There is also an open unmerged closing PR, #73952, that explicitly targets this issue, so the right cleanup path is to review and land or replace that PR rather than close the issue now.

## Best Possible Solution

Keep this issue open and treat #73952 as the active implementation candidate. The best fix is to thread provider eligibility and `allowKeychainPrompt: false` through external CLI auth overlays for gateway auth-status/runtime paths, while preserving unrestricted behavior for callers that intentionally want all external CLI auth sources. Add regression coverage for an opencode/api-key-only setup that excludes Claude/Anthropic and verifies no Claude CLI Keychain reader is invoked.
