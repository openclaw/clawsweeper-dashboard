## Summary

Keep this PR open: it is a focused Telegram bug fix with credible real behavior proof, but the new native `/status` thinking-default resolver still uses the pre-session provider/model and can miss target-session model overrides.

## What This Changes

The PR hydrates model catalog metadata and threads default or explicit thinking levels through Telegram native `/status` and `/think` menu display paths with focused regressions.

## Best Possible Solution

Land the display fix after the native `/status` default resolver derives provider/model from the target session, matching the already-fixed Telegram `/think` menu context behavior.

## Reproduction Assessment

Yes. Current main's native `/status` source path passes no thinking default resolver, and the supplied Telegram screenshots show the visible `Think: off` to `Think: medium` behavior after the patch.

## Solution Assessment

No. The direction is right, but the native `/status` resolver should use the target session's stored model context before merge so model-derived thinking defaults match the session being reported.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

- **[P2] Resolve status defaults against the target session model:** `src/auto-reply/reply/get-reply-native-slash-fast-path.ts:133-134`
