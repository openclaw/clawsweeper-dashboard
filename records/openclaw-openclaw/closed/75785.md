## Summary

Keep open. Current main still has a source-level path where the parent announce flow accepts a raw `agent.wait` timeout without reconciling persisted child session state, and the open unmerged closing PR remains the right implementation candidate.

## What This Changes

The issue requests that subagent completion handoff reconcile a timed-out `agent.wait` result against persisted child session state before announcing a child run as timed out.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/75786, or an equivalent narrow fix, with regression coverage for timeout wait results reconciled against terminal child session state.

## Reproduction Assessment

Yes, at source level: mock `agent.wait` returning `timeout` while the child session store entry is already `done`, then assert the parent announce resolves to success and includes the child result. I did not run the Vitest harness because this review is read-only.

## Solution Assessment

Yes. Reconciling only timeout wait outcomes against persisted terminal child session state is the narrow maintainable fix; normal ok/error wait handling should remain unchanged.
