## Summary

Keep open. Current `main` only overlaps part of this PR: Gemini 2.5 Flash is present, but the browser `lockTab` surface and the Windows CLI shell-fallback change are not on `main`. The PR also targets browser-extension paths that no longer exist in the current tree, so this is stale work that would need a rework/rebase, not a conservative close.

## Evidence

- **`lockTab` is not in the current browser config contract:** `BrowserProfileConfig` and `BrowserConfig` expose `attachOnly`, but there is no `lockTab` field in the current typed config surface.
  - file: [src/config/types.browser.ts:1](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/config/types.browser.ts#L1)

- **`lockTab` is not accepted by the current schema:** The browser profile schema includes `cdpPort`, `cdpUrl`, `userDataDir`, `driver`, `attachOnly`, and `color`; no `lockTab` key is accepted on `main`. A repo-wide `rg -n "lockTab" extensions/browser/src src docs packages` returned no matches.
  - file: [src/config/zod-schema.ts:383](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/config/zod-schema.ts#L383)

  - file: [docs/gateway/configuration-reference.md:168](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/docs/gateway/configuration-reference.md#L168)
