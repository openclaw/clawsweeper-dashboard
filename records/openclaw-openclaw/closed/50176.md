## Summary

Close as superseded by #52614. Current main still lacks the historical replay guard, but this older PR uses an object-identity `preexistingMessages` filter that review discussion flagged as incomplete for cloned/deserialized replay events. The newer open PR #52614 tracks the same remaining bug with the broader replay-state approach, so cleanup should consolidate review there rather than keep both PRs open.

## Best Possible Solution

Close #50176 as superseded, keep the duplicate historical-replay work consolidated in open PR #52614, and make the eventual fix use a replay guard robust enough for cloned/deserialized historical assistant events rather than relying only on object identity.

## Evidence
