## Summary

Current `main` already avoids MCP tool-name collisions for OpenClaw-managed `mcp.servers` by exposing provider-safe names in the form `serverName__toolName`, with collision disambiguation and docs/tests covering the behavior.

## Evidence

- **`mcp.servers` feed the embedded Pi MCP runtime:** `loadEmbeddedPiMcpConfig()` merges top-level `cfg.mcp.servers` into the MCP server set consumed by embedded Pi runtimes, so the issue's config path is the one used by the current implementation.
