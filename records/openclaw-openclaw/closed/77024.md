## Summary

Keep open: current main and latest release still return count-only primary MCP `content` for the reported tools while placing the usable payload under `structuredContent`, and open PR #77041 already tracks a fix with closing syntax.

## What This Changes

The issue requests that `conversations_list` and `messages_read` return full conversation/message payloads in primary MCP tool content instead of only in `structuredContent`.

## Best Possible Solution

Review and land #77041 or a focused equivalent that serializes the returned payload into primary MCP text content while preserving `structuredContent` for capable clients.

## Reproduction Assessment

Yes, at source level: any successful `conversations_list` or `messages_read` call with data returns primary text from `summarizeResult`, which is only a count. I did not run the live Claude.ai path, so this is source-reproducible rather than fully live-reproduced.

## Solution Assessment

Yes, the narrow maintainable fix is in MCP channel tool response formatting with focused MCP call regression tests. Gateway session routing and transcript retrieval do not need to change.
