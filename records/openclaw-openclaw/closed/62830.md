## Summary

Current `main` already contains a cron-specific timeout fix that matches the reported long-running isolated-session exec regression: unconfigured LLM idle timeouts are disabled for cron runs, the embedded runner passes the cron trigger into that resolver, and regression tests cover the cron behavior. The changelog shows this shipped in 2026.4.9, after the reports on 2026.4.5 and 2026.4.8.

## Evidence

- **Issue symptom and confirmation:** The issue reports isolated cron `agentTurn` runs timing out after roughly 60-90s after upgrading to 2026.4.5, despite the underlying exec succeeding; a follow-up comment confirms the same behavior on 2026.4.8 and says downgrading mitigates.
  - command: `Provided GitHub context`
- **Release note matches the regression:** The 2026.4.9 changelog explicitly says OpenClaw now disables the unconfigured LLM idle watchdog for cron runs and points idle-timeout errors at `agents.defaults.llm.idleTimeoutSeconds`.
