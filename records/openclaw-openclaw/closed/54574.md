## Summary

Keep open. Current main still hard-blocks Docker `network: "host"` and has no `dangerouslyAllowHostNetwork` config surface, so this PR is not obsolete; because it deliberately changes sandbox network isolation, it needs maintainer/secops review and validation before merge.

## What This Changes

The PR adds a Docker sandbox `dangerouslyAllowHostNetwork` boolean and threads it through network validation, config/schema/help/types, Docker create-args behavior, security-audit remediation, and targeted tests.

## Best Possible Solution

Land a maintainer-approved version that keeps host networking blocked by default while adding a clearly named Docker-only break-glass flag across runtime validation, config resolution, public schema/help/docs, Docker create args, audit messaging, dangerous-flag reporting, and focused sandbox/config/security tests.
