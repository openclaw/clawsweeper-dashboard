- **[P1] Keep the documented footer fields as the default:** `src/tui/tui.ts:845-852`
  - body: The new `footerParts` list renders only model, tokens, progress, percent, agent, and elapsed time.   - confidence: 0.9
- **[P2] Refresh the footer when elapsed time should advance:** `src/tui/tui.ts:843`
  - body: `elapsedLabel` is computed only when `updateFooter()` runs, but the busy/waiting timers update only the status message. During a run this footer segment will stay at its last render, often `0s`, unless the footer is refreshed from the same timer or the elapsed segment is removed.
  - confidence: 0.88

  - body: When token usage is unavailable, `renderCompactProgressBar(null)` reports percent `0`, so `progressColor(formatCompactPercent(...))` paints `?%` with the success color. Unknown data should use a neutral/dim color instead of signaling low pressure.
  - confidence: 0.82
