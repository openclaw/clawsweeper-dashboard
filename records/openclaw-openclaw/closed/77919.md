Request to add native Deepgram `utterances` query support alongside the existing `smartFormat` option for audio transcription.

## Best Possible Solution

Keep the canonical `providerOptions.deepgram.utterances` path and avoid adding a new deprecated Deepgram-specific runtime alias; a separate issue would be appropriate only for exposing structured Deepgram utterance arrays downstream.
