## Summary

Keep open: current main can already pass `utterances=true` through generic Deepgram provider options, but native support remains incomplete because OpenClaw does not document the option or expose returned utterance metadata. Choosing docs-only pass-through versus a new transcription metadata contract needs maintainer API/product judgment.

## What This Changes

Request adds native Deepgram `utterances` support so users can enable segmented transcription and consume timestamp/confidence metadata.

## Best Possible Solution

Decide whether support means documented pass-through only or preserving structured `results.utterances`; then update Deepgram docs and, if needed, add a typed metadata field through the plugin SDK/media output with tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that the query flag can already be sent, while the missing structured metadata surface is visible in the current parser and output types.

## Solution Assessment

No. The proposed two-line compat change is not the best complete solution because generic `providerOptions` already preserves `utterances`; native support needs docs and likely an additive media-output metadata design.
