## Summary

Current `main` already routes the managed Dreaming cron through an isolated `agentTurn` job with the newer 60-minute default cron safety timeout, and regression tests cover the old false 10-minute timeout behavior.

## Evidence

- **Cron timeout policy no longer caps isolated agent turns at 10 minutes:** `resolveCronJobTimeoutMs()` keeps the generic 10-minute timeout for non-agent cron jobs but uses `AGENT_TURN_SAFETY_TIMEOUT_MS = 60 * 60_000` for `agentTurn` jobs when no explicit `timeoutSeconds` is set.
  - file: [src/cron/service/timeout-policy.ts:14](https://github.com/openclaw/openclaw/blob/0ec3b79c07973e746328a1d6e1be5057096029af/src/cron/service/timeout-policy.ts#L14)
