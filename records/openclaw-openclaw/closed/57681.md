## Summary

Current `main` already contains the model-switch self-heal and retry-bounding logic that addresses the reported permanent "noodling" state after fallback/provider switches, and the same code is present in release `v2026.4.23`.

## Evidence

- **Transient fallback overrides now self-heal:** `createModelSelectionState` clears auto failover session overrides and resets the in-memory provider/model back to the configured primary on the next turn, instead of leaving the session pinned to the fallback.
