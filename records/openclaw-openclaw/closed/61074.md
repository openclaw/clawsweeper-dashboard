## Summary

Issue #61074 is about the external `@tencent-weixin/openclaw-weixin` channel plugin's inbound metadata, not an OpenClaw core bug. Current main documents WeChat as an external plugin, assigns inbound handling to channel plugins, and already supports the plugin's `weixin` alias generically.

## Evidence

- **WeChat is external to core:** The WeChat docs say OpenClaw connects through Tencent's external `@tencent-weixin/openclaw-weixin` plugin, and that the WeChat code does not live in the OpenClaw core repo; the external plugin owns login, monitoring, and inbound normalization.
  - file: [docs/channels/wechat.md:10](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/docs/channels/wechat.md#L10)

  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **Inbound pipeline is plugin-owned:** The channel plugin SDK docs state that inbound message handling is channel-specific and each channel plugin owns its own inbound pipeline.
  - file: [docs/plugins/sdk-channel-plugins.md:556](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/docs/plugins/sdk-channel-plugins.md#L556)
