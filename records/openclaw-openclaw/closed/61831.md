## Summary

Close as duplicate/superseded. Current main still has the shared default host/openclaw browser path when agents omit `profile`, and the remaining per-agent isolated browser-instance/default-profile/concurrency design is already tracked in the open canonical issue at https://github.com/openclaw/openclaw/issues/37487.

## What This Changes

The issue asks for browser tool calls from different agents to use isolated Chrome instances/profiles instead of sharing the default host browser profile.

## Best Possible Solution

Keep the remaining browser isolation/default-profile/concurrency work centralized in https://github.com/openclaw/openclaw/issues/37487 and use this report as linked default-host reproduction context there.

## Reproduction Assessment

Yes. With sandboxing off, two agents that call `browser` without `target` or `profile` both take the host path and pass `profile` as undefined, which resolves to the configured default `openclaw` profile.

## Solution Assessment

Yes. Closing this as duplicate/superseded is the best cleanup path because the remaining fix needs the broader canonical product/security design for agent-bound profiles, concurrent isolated instances, and default/shared-profile compatibility.
