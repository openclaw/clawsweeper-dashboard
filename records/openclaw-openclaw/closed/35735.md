Full review comments:

- **[P1] Route deletes through the session lifecycle:** `src/commands/sessions-delete.ts:144`
  - body: `updateSessionStore` saves through maintenance by default, so `sessions rm` can prune unrelated stale/capped entries even when the requested key is missing, and filtered `clear` can remove more than the reported predicate matches. This also skips the existing `sessions.delete` cleanup, plugin-runtime checks, lifecycle unbinds, and transcript archival; route configured deletes through the Gateway lifecycle or make direct mutation an explicit offline repair path with matching reporting.
  - confidence: 0.91
- **[P2] Report transcript removal only after success:** `src/commands/sessions-delete.ts:97-98`
  - body: The command swallows any `fs.promises.rm` failure and still pushes the path into `removedTranscripts`, so permission or filesystem errors are reported as successful transcript deletion. Only count the transcript after a successful remove, or surface the failed removal in the summary/error path.
  - confidence: 0.86
- **[P3] Keep JSON fields stable across actions:** `src/commands/sessions-delete.ts:212`
  - body: For `clear`, `key` is `undefined` and disappears from the JSON output, while the analogous unused `olderThanMinutes` field is rendered as `null` for `rm`. Emit `key: null` when no key applies so JSON consumers get a consistent schema.
  - confidence: 0.78
