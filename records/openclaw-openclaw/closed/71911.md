## Summary

Current main fixes the Discord-DM duplicate-turn mechanism described in #71911. The v2026.4.24 path submitted the runtime-enriched prompt and then rewrote the transcript by branching; current main instead submits the transcript-facing prompt directly and moves runtime context into a hidden next-turn custom message, with doctor repair for already-affected 2026.4.24 transcripts.

## Best Possible Solution

Close #71911 as implemented on main. Keep the current runtime-context submission model, keep the doctor repair path for affected 2026.4.24 transcripts, and direct any remaining reports to a fresh issue with logs from a build containing this mainline fix if duplicate Discord DM delivery still reproduces.
