## Summary

Current main has moved Ollama setup into the bundled Ollama plugin, but the equivalent `/api/pull` setup path still has no per-call timeout. The submitted one-line PR also has unresolved review feedback: a 30s `AbortSignal.timeout()` on the fetch would cap the whole streaming model pull, not just connection startup, so it should not be closed as implemented or landed as-is.

## Best Possible Solution

Keep the item open, but request a redesigned fix in `extensions/ollama/src/setup.ts`: add a streaming-safe startup/connect/headers timeout or no-progress watchdog that does not cap healthy long-running `/api/pull` streams, and add focused tests for stalled pull startup plus a long streaming pull that continues past 30 seconds.
