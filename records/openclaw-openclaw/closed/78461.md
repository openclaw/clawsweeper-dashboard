## Summary

Keep open: current main still has the source-level reader-side model-normalization snapshot miss. Newer process-local metadata memoization can reduce fallback cost, but it does not make this path reuse the Gateway-owned workspace-scoped snapshot, and related implementation PRs remain open.

## What This Changes

Avoid repeated plugin metadata/index discovery by reusing Gateway-owned plugin metadata snapshots during manifest model-id normalization.

## Best Possible Solution

Fold the narrow unscoped manifest-normalization snapshot reuse fix into the active plugin metadata performance work, then keep broader discovery, realpath, and status-specific regressions tracked by their narrower items.

## Reproduction Assessment

Yes, source-level: store a workspace-scoped current metadata snapshot, leave the active plugin runtime workspace unset, and call manifest model-id normalization without `workspaceDir`. Current main rejects the compatible current snapshot and falls back to `loadPluginMetadataSnapshot`.

## Solution Assessment

Yes for the original miss: applying the existing unscoped `allowWorkspaceScopedSnapshot` pattern to manifest model-id normalization is the narrowest maintainable fix. Broader process-level realpath and status-path fixes should be reviewed in the related performance PRs/issues instead of closing this early.
