## Summary

Keep open. Current `main` still matches the reported failure mode: Active Memory defaults to the current session model, Anthropic/Claude CLI extra-usage strings are explicitly classified as `billing`, and billing-classified auth failures still disable the shared auth profile in `auth-state.json`. I did not find a post-`v2026.4.23` fix in the relevant code paths.

## Evidence
