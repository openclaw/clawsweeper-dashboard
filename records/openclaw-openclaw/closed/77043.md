## Summary

Current `main` and the `v2026.5.2` tagged source already normalize the reported parameter-free MCP schema shapes on the OpenAI paths, so the report no longer matches the source shipped for that release.

## What This Changes

The issue reports that `flux-operator-mcp` parameter-free tools still reach OpenAI without a valid empty object schema in `v2026.5.2`.

## Best Possible Solution

Keep the central OpenAI/MCP schema normalization shipped in `v2026.5.2`; investigate only if a new report includes an image digest and payload showing a path that bypasses it.

## Reproduction Assessment

No. Current `main` and the `v2026.5.2` tagged source normalize the reported `{}`, missing, null, and invalid `properties` schema shapes before OpenAI payload construction; the issue lacks logs or payload evidence for a distinct path.

## Solution Assessment

Yes. The central schema normalizer plus OpenAI transport call sites are the narrow maintainable fix; if a deployed image still fails, the next useful step is payload/image-digest evidence rather than another core change.
