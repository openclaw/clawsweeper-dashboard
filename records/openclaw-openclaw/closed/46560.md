## Summary

Current main already ships the supported plugin-side gateway client and protocol contract for this flow. The repo documents that all WS clients must include device identity and sign the `connect.challenge` nonce, and the public plugin SDK exports `GatewayClient`, which auto-loads device identity and sends the signed challenge-bound payload. Server-side tests also show that device-less shared-secret operator connects intentionally lose self-declared scopes, so the remaining breakage is in the external A2A plugin's custom WS client rather than an unfixed core gap on main.
