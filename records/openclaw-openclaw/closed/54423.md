## Summary

Current `main` already fixes the non-loopback Control UI startup failure that this PR targets, using explicit `allowedOrigins` seeding instead of the PR's dangerous Host-header fallback. The remaining script-side Windows symlink handling is also already implemented on `main`, and the current Docker image preserves the non-root runtime plus `HEALTHCHECK` that this PR would regress.

## Evidence

- **Startup now seeds Control UI origins before gateway runtime validation:** Gateway startup runs a pre-runtime migration that persists seeded `gateway.controlUi.allowedOrigins` for existing non-loopback/container configs, specifically to avoid startup crash loops.
  - file: [src/gateway/server.impl.ts:321](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/gateway/server.impl.ts#L321)
