## Summary

Current main no longer matches the reported failure mode. Media understanding is applied on inbound messages before session initialization and before queued followup prompts are constructed, so followup queue entries inherit the transcribed/enriched body instead of raw `<media:audio>` placeholders.

## Evidence

- **Inbound media understanding runs before queueing:** `getReplyFromConfig` finalizes inbound context, then calls `applyMediaUnderstandingIfNeeded(...)` before `initSessionState(...)` and before later handing off to prepared-reply execution.
  - file: [src/auto-reply/reply/get-reply.ts:246](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/auto-reply/reply/get-reply.ts#L246)
  - command: `nl -ba src/auto-reply/reply/get-reply.ts | sed -n '120,270p'`
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Media understanding rewrites the inbound body and transcript:** `applyMediaUnderstanding` updates `ctx.Body`, sets `ctx.Transcript`, records `ctx.MediaUnderstanding`, and re-finalizes the inbound context so downstream prompt fields use the enriched content.
  - file: [src/media-understanding/apply.ts:531](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/media-understanding/apply.ts#L531)
  - command: `nl -ba src/media-understanding/apply.ts | sed -n '520,580p'`
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Session state copies enriched body into prompt source:** `initSessionState` builds `sessionCtx.BodyStripped` from `BodyForAgent`/`Body`, so any media-understanding rewrite is preserved in the session context used for prompts.
  - file: [src/auto-reply/reply/session.ts:811](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/auto-reply/reply/session.ts#L811)
  - command: `nl -ba src/auto-reply/reply/session.ts | sed -n '780,840p'`
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Queued followup prompt is built from the enriched session body:** `runPreparedReply` derives `baseBody` from `sessionCtx.BodyStripped ?? sessionCtx.Body`, then stores `queuedBody` as `followupRun.prompt`. That means queued followups are built from the already-processed transcript/body, not from raw attachment placeholders.
  - file: [src/auto-reply/reply/get-reply-run.ts:326](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/auto-reply/reply/get-reply-run.ts#L326)
