## Summary

Current `main` already routes inbound attachment reads through channel-specific allowed roots, and the iMessage plugin now publishes `/Users/*/Library/Messages/Attachments` as its default inbound attachment root with configurable overrides. The same implementation is present in release `v2026.4.23`, so this issue is already covered by shipped code.

## Evidence

- **Media understanding now merges channel attachment roots into the local-media allowlist:** `resolveMediaAttachmentLocalRoots()` merges the default media roots with `resolveChannelInboundAttachmentRoots(params)`, so inbound attachment reads are no longer limited to the generic `~/.openclaw/...` roots.
  - file: [src/media-understanding/runner.ts:188](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/media-understanding/runner.ts#L188)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Core loads per-channel inbound attachment roots from the bundled plugin public surface:** `resolveChannelInboundAttachmentRoots()` looks up a channel's `media-contract-api.js` and returns its `resolveInboundAttachmentRoots()` result for the current context/account.
  - file: [src/media/channel-inbound-roots.ts:68](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/media/channel-inbound-roots.ts#L68)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **iMessage declares the Messages attachments directory as an allowed inbound root:** The iMessage media contract exports `DEFAULT_IMESSAGE_ATTACHMENT_ROOTS = ["/Users/*/Library/Messages/Attachments"]` and merges account/global overrides for both local and remote attachment roots.
  - file: [extensions/imessage/src/media-contract.ts:5](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/extensions/imessage/src/media-contract.ts#L5)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Config/docs/tests all expose and validate the iMessage attachment-root contract:** The public config type includes `channels.imessage.attachmentRoots` / `remoteAttachmentRoots`; docs document the same default root; and the contract test covers default + override resolution.
  - file: [src/config/types.imessage.ts:61](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/config/types.imessage.ts#L61)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **The fix is already in the latest release:** `git show a9797214338ba31c52c796adbb75afb16e0684a9:src/media-understanding/runner.ts` and `git show a9797214338ba31c52c796adbb75afb16e0684a9:extensions/imessage/src/media-contract.ts` show the same channel-root merge and iMessage default attachment-root implementation in `v2026.4.23`.
