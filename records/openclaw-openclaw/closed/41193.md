## Summary

Keep open. The issue reports several crash-loop paths, and the related WeakMap/null-message crash path is not implemented on current main: `estimateMessageCharsCached` still calls `WeakMap.get(msg)` before any null/object guard, so null or undefined messages would still throw. The linked PR #41178 that commenters said might resolve this was closed unmerged in the provided GitHub context, and current tests only cover malformed content blocks, not null/undefined message entries.
