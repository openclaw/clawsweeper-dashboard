## Summary

Keep open. Current main intentionally keeps workspace-local explicit QMD collections agent-scoped in source, tests, and docs; this PR weakens that isolation without a collision policy or scoped-to-unscoped migration, and it is currently unmergeable against the package-owned implementation path.

## What This Changes

This PR changes custom QMD collection resolution so every explicit collection name is used verbatim, removes the workspace-containment check, and updates backend-config and memory-core tests from agent-suffixed names to unscoped explicit names.

## Best Possible Solution

Settle the product contract first. If authoritative workspace-local explicit names are desired, implement that in the package resolver with a collision-safe multi-agent policy, a scoped-to-unscoped compatibility migration in memory-core, matching docs, and regression coverage for older QMD list output.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep local custom names collision-safe across agents:** `src/memory-host-sdk/host/backend-config.ts:261-263`
  - body: This makes a workspace-relative custom path such as `path: "notes", name: "workspace"` resolve to `workspace` for every agent even though each agent has a different workspace root. QMD collections are managed and searched by collection name, so searches or rebinds can cross the isolation boundary that current docs and tests promise.
  - confidence: 0.9
- **[P1] Add scoped-to-unscoped collection migration:** `src/memory-host-sdk/host/backend-config.ts:261-263`
  - body: Existing workspace-local explicit collections are already stored as `<name>-<agent>`, but after this change searches target `<name>`. The current migration only handles the opposite unscoped-to-scoped direction, and older name-only QMD list output deliberately avoids destructive rebinds, so upgraded installs can keep searching a missing collection.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: No dependency or CI surface changed, but the naming change can weaken per-agent memory isolation for workspace-local QMD collections.

Concerns:

- **[high] Potential cross-agent memory collection collision:** `src/memory-host-sdk/host/backend-config.ts:261`
  - body: Preserving the same explicit name for workspace-relative paths can collapse distinct agent workspaces onto one QMD collection key, allowing memory search or collection repair behavior to cross an intended agent isolation boundary.
  - confidence: 0.84
