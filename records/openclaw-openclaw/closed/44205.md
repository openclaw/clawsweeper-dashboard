## Summary

Keep open; the PR has useful freshness intent, but the current patch would regress isolated cron fallback delivery by dropping distinct child outputs and by applying an overly broad interim-message heuristic to arbitrary child result text.

## What This Changes

The PR changes cron isolated-agent descendant fallback to return the newest usable descendant reply, updates related tests, and adds a generic type assertion in `src/shared/global-singleton.ts`.

## Best Possible Solution

Keep latest-run-per-child selection, aggregate one usable latest reply per distinct child, narrow acknowledgement filtering to true child acknowledgements, and add the required changelog entry.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path for the PR regression: current main has a focused multi-child aggregation test, while the PR replaces that expectation and returns after the first usable descendant reply.

## Solution Assessment

No. Selecting the newest run per child is useful, but the maintainable fix should preserve distinct-child fan-out aggregation and avoid applying the broad cron interim heuristic to arbitrary child output.
