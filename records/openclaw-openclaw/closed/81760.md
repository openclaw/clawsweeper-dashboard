## Summary

Keep this issue open: current main still lacks the per-agent thinking metadata needed by the Agents panel, and the open closing PR is still the focused fix path rather than a merged implementation.

## What This Changes

The issue reports that Control UI agent thinking defaults are derived from the global default model instead of the selected agent’s configured model and thinking setting.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81795 or an equivalent fix that exposes resolved per-agent thinking metadata through agents.list and prevents global-default fallbacks when the session model differs.

## Reproduction Assessment

Yes, source-level. Current main omits thinking metadata from agents.list rows and still has global-default fallback paths that can mislabel sessions whose model differs from the defaults.

## Solution Assessment

Yes. The narrow maintainable solution is to expose gateway-resolved per-agent thinking metadata and guard defaults by model match; the linked open PR implements that direction but has not merged.
