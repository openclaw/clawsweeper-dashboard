## Summary

Close #53586 as `clawhub`. The LiteLLM `/search/<tool>` use case is valid, but current main does not expose it through the bundled Perplexity provider, and the repo’s current architecture routes optional provider/search integrations to plugins published through ClawHub/npm. Existing web-search provider SDK seams are sufficient for a dedicated LiteLLM Search plugin, including internal/private trusted endpoints.

## Best Possible Solution

Keep the shipped Perplexity provider focused on native Perplexity Search plus Sonar/OpenRouter compatibility. If this workflow is pursued, implement and publish a dedicated LiteLLM web-search provider plugin through ClawHub/npm, reusing the existing `api.registerWebSearchProvider(...)` contract and trusted web-tools fetch helpers to call LiteLLM `/search/<search_tool_name>` endpoints.
