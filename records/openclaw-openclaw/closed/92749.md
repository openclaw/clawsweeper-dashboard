## Summary

Keep open: the PR targets a valid stuck-session recovery bug, but it is not merge-ready because the added test fails TypeScript checking and the production guard disables warning backoff for all recovery-eligible stuck sessions instead of separating recovery scheduling from log throttling.

## What This Changes

The PR changes diagnostic stuck/long-running warning backoff guards and adds a regression test intended to let recovery-eligible sessions bypass throttling.

## Best Possible Solution

Keep the narrow recovery fix, but split warning/event throttling from recovery scheduling so recovery-eligible classifications still reach `requestStuckSessionRecovery` while repeated stuck warnings remain backed off, with compiling regression coverage and real after-fix proof.

The fix should let recovery scheduling proceed without disabling the warning throttle for every recovery-eligible stuck classification.
