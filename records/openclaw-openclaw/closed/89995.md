## Summary

Current main already fixes the reported duplicate-send path: outbound transcript mirror failures after a successful channel send are now warning-only, and the regression test uses the exact lock-change error from the report.

## What This Changes

The issue reports duplicate subagent completion announce messages when a post-send embedded session lock conflict makes delivery look failed and cleanup retries the announce.

## Best Possible Solution

Keep the current main implementation that treats post-send transcript mirror failures as non-fatal; users on older beta builds need a build containing 79896a24d9bb941f7c9f3801a69f3ea01fb31043.

## Reproduction Assessment

Yes, by source and regression-test evidence: pre-fix outbound delivery threw when the post-send transcript mirror failed with the exact lock-change message, and current main’s test now proves the channel send remains successful instead of retrying.

## Solution Assessment

Yes; making transcript mirroring warning-only after platform send is the narrowest durable fix because it preserves the successful channel delivery and avoids duplicate retries without changing subagent retry policy or adding a new config surface.
