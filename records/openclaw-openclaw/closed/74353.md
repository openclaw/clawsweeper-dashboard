## Summary

Current main wires `models.providers.<id>.timeoutSeconds` through the configured model, OpenAI-compatible request transport, guarded fetch timeout, OpenAI SDK timeout options, and the embedded runner's no-token idle watchdog. The proof SHA is on current main but is not contained in any local release tag, so this should close as implemented on main rather than shipped in v2026.4.26.

## What This Changes

The issue reports that `models.providers.<id>.timeoutSeconds` may not cover the full LM Studio/OpenAI-compatible cold-load request path after `agents.defaults.llm.idleTimeoutSeconds` was retired.

## Best Possible Solution

Keep the current-main implementation that routes provider `timeoutSeconds` through model request metadata, OpenAI-compatible transport timeouts, guarded fetch, and the idle watchdog; if a user can still reproduce on a build from current main, the next issue should include logs showing which timeout fired and the effective provider/model config.
