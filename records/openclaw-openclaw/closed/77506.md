## Summary

Keep open: current main still hardcodes `memory_recall` into active-memory's embedded recall allowlist while the default `memory-core` plugin only declares and registers `memory_search` and `memory_get`; two open PRs already target this issue with closing syntax, so the issue should remain open until one is reviewed and landed or superseded.

## What This Changes

Fix active-memory's embedded memory sub-agent allowlist so default memory-core setups do not request unavailable recall tools or disrupt sessions.

## Best Possible Solution

Land one canonical fix that derives or filters the recall sub-agent's allowed memory tools from the selected/registered backend, preserving `memory_recall` for memory-lancedb while keeping memory-core-only runs on `memory_search` and `memory_get`.
