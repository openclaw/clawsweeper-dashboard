## Summary

`openclaw-weixin` voice sending is owned by the external WeChat plugin, not OpenClaw core. Current `main` already carries the generic outbound `audioAsVoice` seam to plugins, so this core issue can close as plugin-scope work.

## Evidence

- **WeChat is documented as an external plugin:** The WeChat docs state that OpenClaw connects through Tencent's external `@tencent-weixin/openclaw-weixin` plugin and mark its status as `external plugin`.
  - file: [docs/channels/wechat.md:10](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/docs/channels/wechat.md#L10)
  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **WeChat-specific media/runtime ownership stays with the plugin:** The docs explicitly say the WeChat code does not live in the core repo and that Tencent iLink API calls, media upload/download, context tokens, and monitoring are owned by the external plugin.
  - file: [docs/channels/wechat.md:27](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/docs/channels/wechat.md#L27)
  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **Project scope prefers plugin-owned optional capability:** VISION says core stays lean, optional capability should usually ship as plugins, the preferred path is npm-distributed plugins, and plugin promotion/discovery belongs in ClawHub.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/VISION.md#L54)
  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **Core already exposes the voice-send hint to channel plugins:** The generic channel outbound context already includes `audioAsVoice?: boolean`, which is the core seam a channel plugin would use for native voice-note delivery.
  - file: [src/channels/plugins/outbound.types.ts:14](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/channels/plugins/outbound.types.ts#L14)
  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **Current main tests that plugin media delivery receives `audioAsVoice`:** `deliver.test.ts` verifies generic plugin media delivery forwards `audioAsVoice: true`, which indicates the missing work is plugin-side implementation rather than a missing core transport field.
  - file: [src/infra/outbound/deliver.test.ts:603](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/infra/outbound/deliver.test.ts#L603)
  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **No bundled WeChat extension exists in this checkout:** A repo scan for `weixin|wechat` under `extensions/` returned no matching bundled plugin paths, consistent with the docs that this channel is maintained outside the core repo.
  - command: `rg --files extensions | rg 'weixin|wechat'`
  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
