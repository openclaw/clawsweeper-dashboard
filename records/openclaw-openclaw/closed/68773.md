## Summary

Keep this PR open. Current main still has the `asRecord(details?.debug) ?? {}` fallback in `readActiveMemorySearchDebug`, so a newer payload-less `memory_search` transcript result can still block an older useful debug payload. The PR remains a focused bug fix with a targeted regression test and no obvious security-sensitive unrelated changes in the provided diff.

## Best Possible Solution

Keep this PR open and land it, or an equivalent patch, after normal maintainer validation. The best code path is to remove the `?? {}` fallback in `readActiveMemorySearchDebug` and keep the regression test proving a newer payload-less `memory_search` transcript entry does not hide older real debug telemetry.
