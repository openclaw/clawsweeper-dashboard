## Summary

Close: the shipped Control UI code in `v2026.4.23` already defers `session.message` history reloads while a chat run is active, matching the optimistic-send/history race described in issue #66176 and its `#67037` cross-reference.

## Evidence

- **Released UI code now guards the exact race described in the issue:** `ui/src/ui/app-gateway.ts:490` in release `v2026.4.23` explicitly skips `chat.history` reloads during an active run and documents that reloading mid-run races with the optimistic user message and delays the visible card until the first LLM delta.
  - file: [ui/src/ui/app-gateway.ts:490](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/ui/src/ui/app-gateway.ts#L490)
  - command: `git show a9797214338ba31c52c796adbb75afb16e0684a9:ui/src/ui/app-gateway.ts | nl -ba | sed -n '458,500p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
