## Summary

The reported WeChat false-success path is owned by the external `@tencent-weixin/openclaw-weixin` plugin rather than OpenClaw core. Current main does not convert thrown channel-adapter failures into delivered cron results, and the current WeChat plugin source shows the session guard throws before outbound send while Tencent send-response validation is implemented in that external package.

## What This Changes

Cron announce delivery to `openclaw-weixin` is reported as `delivered: true` even though the WeChat message does not arrive after the WeChat session token has expired.

## Best Possible Solution

Keep OpenClaw core channel-agnostic and fix/publish the external WeChat plugin so its Tencent send API wrapper throws when the API response body reports `ret` or `errcode` failure, including session expiry. Track dynamic account selection separately in #73230.

## Reproduction Assessment

No high-confidence live reproduction is available here because it requires an expiring WeChat session and Tencent service behavior. Source inspection gives a high-confidence ownership check: the reported session guard throws in the plugin, and current core does not mark thrown non-best-effort sends as delivered.

## Solution Assessment

Yes for this repository: the maintainable fix is plugin-side send-response validation in `@tencent-weixin/openclaw-weixin`, not a WeChat-specific core branch. A separate generic core test for raw `ok:false` plugin results would be valid as its own focused issue if maintainers want that contract tightened.
