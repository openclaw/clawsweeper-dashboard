## Summary

Current main implements the reported behavior: `alsoAllow` now becomes an explicit wildcard allowlist for discovery, and plugin tool resolution cold-loads incomplete registries so optional plugin tools are available to effective/invoke/native surfaces.

## What This Changes

The issue reports that optional plugin tools listed by `tools.catalog` were unavailable to `tools.effective`, `tools.invoke`, and native agent tool assembly when allowed via `alsoAllow`.

## Best Possible Solution

Keep the current main implementation and let the unreleased plugin-tool discovery fixes ship in the next release.

## Solution Assessment

Yes. The current fix uses the existing tool-policy and plugin-registry seams rather than adding a new config mode, and it keeps the documented `alsoAllow` path working for optional plugin tools.
