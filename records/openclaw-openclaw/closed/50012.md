## Summary

Current main already documents and uses `tools.fs.workspaceOnly` as the canonical runtime-backed config path, and invalid schema keys are rejected during config writes.

## Evidence

- **Canonical schema key exists:** `tools.fs.workspaceOnly` is defined as the filesystem-tools workspace restriction in the config types and schema help.
  - file: [src/config/types.tools.ts:296](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/src/config/types.tools.ts#L296)

  - command: `nl -ba src/agents/tool-fs-policy.ts | sed -n '1,80p'`

  - file: [src/agents/tool-fs-policy.test.ts:8](https://github.com/openclaw/openclaw/blob/45146913007da723b4f06aba6a66cf4a695974c0/src/agents/tool-fs-policy.test.ts#L8)
