## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Await script banner emission before command execution:** `src/cli/banner.ts:207`
  - body: This changes `emitCliBanner` to async, but `command-execution-startup.ts` and the gateway fast path still call it without `await`. In script mode the first dynamic import yields before the banner is written, so command execution can race ahead of the banner or a short command can exit before the tagline appears; make the presentation path await this promise or keep emission synchronous after a separate priming step.
  - confidence: 0.9
- **[P2] Run tagline scripts in an abortable boundary:** `src/cli/tagline.ts:287-293`
  - body: The timeout races work on the same event loop, so it cannot stop top-level module work or a synchronous default export such as the documented `execSync` example. Since this runs during CLI startup/help, a bad configured script can still hang OpenClaw indefinitely; use a worker/child process with a hard timeout or avoid script execution here.
  - confidence: 0.88
- **[P2] Wire script mode through real help rendering paths:** `src/cli/run-main.ts:574`
  - body: This priming call only covers the `runCli` root-help branch. The installed entrypoint handles `openclaw --help` before `run-main.ts`, and the help renderer still calls `formatCliBannerLine` with `mode: "default"`, so configured script taglines remain missing on common help paths while the PR adds extra startup work. Centralize this in the help/startup policy and cover root flag help, the help subcommand, and piped help.
  - confidence: 0.83
