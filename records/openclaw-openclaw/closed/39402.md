## Summary

Current main already supports Telegram bot-authored group traffic in the normal inbound path, and the repo ships a live Telegram QA lane that exercises a private-group bot-to-bot reply flow with mention gating. The issue’s premise that bots in the same Telegram group cannot see/respond to each other no longer matches main.

## Evidence

- **Telegram handler accepts other bots in group message flow:** The Telegram `message` handler only suppresses self-echoes (`from.id === ctx.me?.id`) and otherwise forwards the message into `handleInboundMessageLike`, so bot-authored group messages from another bot are not filtered out by OpenClaw.
