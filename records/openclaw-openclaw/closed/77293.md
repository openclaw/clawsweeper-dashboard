## Summary

Keep open: current main still routes `2026.5.3-1` through the generic plugin API semver range check, and the linked closing PR #77450 is open and unmerged. The runtime-chunk side now has source guardrails and legacy alias coverage, but the remaining plugin API compatibility bug is still present on main.

## What This Changes

The issue reports that npm global OpenClaw 2026.5.3-1 cannot update plugins because a hashed install runtime chunk is missing and `2026.5.3-1` fails plugin API range `>=2026.5.3`.

## Best Possible Solution

Review and land PR #77450, or an equivalent narrow fix, so stable CalVer correction host versions satisfy base stable plugin API ranges while preserving current package artifact guardrails.

## Reproduction Assessment

Yes for the plugin API mismatch by source inspection: `2026.5.3-1` reaches the generic semver comparator and sorts below `2026.5.3`. The missing runtime chunk half is not fully reproducible from this source checkout because it depends on the published npm artifact.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable fix for the remaining compatibility bug: normalize only stable correction host versions in the plugin API range path, without changing visible runtime versions or prerelease behavior.
