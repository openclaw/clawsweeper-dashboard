## Summary

Close as implemented on current main. The report targets v2026.4.29, and current main removes the request-time package-manager and broad plugin-registry hot paths it names: bundled dependency work is no longer runtime loading, ACPX is deferred, and plugin tool discovery is scoped/reuses gateway registries. The proof is main-only; no release tag contains the proof SHA yet.

## What This Changes

Investigate and remove severe Gateway agent-run startup event-loop blocking before streaming when many workspace skills and bundled plugins are enabled.

## Best Possible Solution

Keep the current main implementation, ship it in the next release, and reopen only if a current-main diagnostics export still shows multi-second event-loop starvation in the same startup stages.

## Reproduction Assessment

Yes for the v2026.4.29 failure signature: the report gives concrete setup, request path, stage names, and liveness metrics. Current-main verification here was source/provenance based rather than a live Docker replay.

## Solution Assessment

Yes. Removing dependency repair and broad plugin activation from runtime/tool discovery, deferring ACPX, and reusing scoped registries is narrower and more maintainable than adding event-loop yields around still-heavy synchronous work.
