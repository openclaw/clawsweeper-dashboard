## Summary

Current main intentionally limits OpenRouter `cache_control` marker injection to Anthropic model refs; DeepSeek is already handled as OpenRouter-managed automatic prompt caching, so the proposed guard swap would not match the upstream contract.

## What This Changes

The issue requests applying OpenRouter `cache_control` breakpoints to cache-eligible non-Anthropic models such as `openrouter/deepseek/deepseek-v4-pro`.

## Best Possible Solution

Keep Anthropic marker injection scoped to Anthropic OpenRouter refs, and treat DeepSeek cache failures as actionable only with repeated stable-prefix usage evidence showing `cached_tokens` or cache discount stays absent.

## Reproduction Assessment

No. The payload inspection reproduces the expected absence of DeepSeek `cache_control` markers, but it does not reproduce the claimed cost/cache failure; a high-confidence bug report would need OpenRouter usage metrics from repeated same-prefix requests.

## Solution Assessment

No. Replacing `isAnthropicModelRef` with cache-TTL eligibility would conflate pruning eligibility with Anthropic-specific payload shaping; current behavior is the narrower maintainable solution.
