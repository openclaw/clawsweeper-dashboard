## Summary

The request is for an optional real-time market-data integration for a WhatsApp trading assistant, not a missing OpenClaw core API. Current main already has the generic plugin, web fetch, web search, automation, and browser surfaces needed to build this as a ClawHub/npm plugin, while the project vision explicitly keeps optional and commercial service integrations out of core by default.

## Best Possible Solution

Close this as ClawHub scope. The best path is an external stock-data plugin or skill that registers a quote tool, optional web fetch/search provider behavior, and configurable market-data providers through existing plugin config; cron or heartbeat can handle periodic summaries and alerts. If building that plugin reveals a concrete missing SDK seam, open a narrow plugin-API issue for that seam rather than adding a one-off finance integration to core.

## Risks / Open Questions

- Closing this in core does not provide an official stock quote plugin; users still need browser automation, Firecrawl/Tavily, or an external market-data plugin.
- A real trading-data plugin would need to handle provider licensing, latency, market coverage, credentials, and financial-data reliability outside the OpenClaw core repository.
