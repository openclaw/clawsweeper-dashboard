## Summary

Keep open: the updated head fixes the earlier system-prompt promotion and moves toward scoped, data-only recall, but it is not merge-ready because scoped auto-captured memories become invisible to existing management tools and the supplied proof is still mock-only.

## What This Changes

The PR changes memory-lancedb automatic recall/capture to require agent/session scope, store automatic memories in hashed `auto-recall/` LanceDB paths, keep recalled text in `prependContext`, and add hook-level regression tests.

## Best Possible Solution

Keep the scoped, data-only automatic recall direction, but make scoped automatic persistence a managed memory store with recall/list/forget or explicit maintainer-approved migration and cleanup semantics plus real setup proof.
