## Summary

Current `main` has already removed the reported orphaned `openclaw-channels` login path and replaced it with in-process WhatsApp login code that explicitly cleans up sockets on completion and failure.

## Evidence

- **CLI login now calls channel auth directly:** `runChannelLogin` resolves the channel plugin and invokes `plugin.auth.login(...)` directly; there is no subprocess launch in the current login path.
  - file: [src/cli/channel-auth.ts:197](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/cli/channel-auth.ts#L197)
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **WhatsApp login is in-process:** The WhatsApp channel plugin routes `channels login` to `loginWeb(...)` in the runtime module, not to a spawned `openclaw-channels` helper.
  - file: [extensions/whatsapp/src/channel.ts:161](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/extensions/whatsapp/src/channel.ts#L161)
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Interactive login now closes the socket in `finally`:** `loginWeb` creates the WA socket, waits for login completion, and always calls `closeWaSocketSoon(sock)` in `finally`, including failure paths.
  - file: [extensions/whatsapp/src/login.ts:20](https://github.com/openclaw/openclaw/blob/37c37eecfbe4bef413c33b8606db62cf51c2d1d4/extensions/whatsapp/src/login.ts#L20)

  - file: [extensions/whatsapp/src/login-qr.ts:61](https://github.com/openclaw/openclaw/blob/37c37eecfbe4bef413c33b8606db62cf51c2d1d4/extensions/whatsapp/src/login-qr.ts#L61)
  - sha: [37c37eecfbe4](https://github.com/openclaw/openclaw/commit/37c37eecfbe4bef413c33b8606db62cf51c2d1d4)
