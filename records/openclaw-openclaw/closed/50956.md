## Summary

Current main no longer matches the reported failure mode: busy-session messages live in an in-memory followup queue and are drained as a separate turn after the active run finishes, rather than being persisted into the current transcript chain and replayed inside the same agent loop.

## Evidence

- **Busy messages are queued in process memory, not appended to the active transcript:** `FOLLOWUP_QUEUES` is a global in-memory map of queued followups; queue state is `items[]` plus drain metadata, with no transcript/session-file persistence on this path.
  - file: [src/auto-reply/reply/queue/state.ts:23](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/auto-reply/reply/queue/state.ts#L23)

  - sha: [6944d7025d9c](https://github.com/openclaw/openclaw/commit/6944d7025d9c0ba4891cbe646ab5dab039d504ca)
- **Enqueue path only pushes followups into the queue:** `enqueueFollowupRun()` updates queue metadata and `queue.items.push(run)`; it does not write queued inbound messages into the parentId/session chain.
  - file: [src/auto-reply/reply/queue/enqueue.ts:60](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/auto-reply/reply/queue/enqueue.ts#L60)

  - sha: [6944d7025d9c](https://github.com/openclaw/openclaw/commit/6944d7025d9c0ba4891cbe646ab5dab039d504ca)
- **Queued prompt is created only when the queue drains:** `scheduleFollowupDrain()` builds the `[Queued messages while agent was busy]` prompt during drain and hands it to `effectiveRunFollowup(...)`, which makes it a fresh followup turn rather than a continuation of the prior loop.
  - file: [src/auto-reply/reply/queue/drain.ts:159](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/auto-reply/reply/queue/drain.ts#L159)

- **Main run finalization schedules followup drain after the run ends:** `finalizeWithFollowup()` just calls `scheduleFollowupDrain(...)`, and `agent-runner.ts` returns through that helper on final/empty/success paths, so queued work is kicked to the next turn after completion.
  - file: [src/auto-reply/reply/agent-runner-helpers.ts:59](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/auto-reply/reply/agent-runner-helpers.ts#L59)
