## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Patch Gemma baseUrl when forcing native API:** `extensions/google/provider-models.ts:177`
  - body: Setting `patch.api` to `google-generative-ai` for every Gemma clone also affects `google-gemini-cli` resolutions, where template selection can preserve a CLI/cloudcode baseUrl. The native transport then appends `/models/...:streamGenerateContent` to that incompatible base URL, so CLI Gemma users can be routed to the wrong host/path. Please either choose a native Google template for this override or patch `baseUrl` together with `api`.
  - confidence: 0.91
- **[P1] Gate baseUrl normalization on an API switch:** `extensions/google/provider-policy.ts:117-119`
  - body: The new explicit-Gemma check normalizes the provider baseUrl away from `/openai` even when a Gemma entry omits `model.api` and still inherits provider-level `openai-completions`. That sends OpenAI-compatible requests to the native `/v1beta` path and can also break sibling OpenAI-compatible models in the same provider config. Only normalize the baseUrl when the affected model API is actually switched to `google-generative-ai`.
  - confidence: 0.9
