## Summary

Keep open: this is a valid session-memory bug fix area, but the current PR still has blocking sanitizer correctness issues and lacks redacted after-fix proof from a real OpenClaw hook run.

## What This Changes

Adds a session-memory sanitizer, wires it into transcript extraction and memory writes, and adds tests for chat-template tokens, raw tool_call XML, orphan role lines, and NO_REPLY elision.

## Best Possible Solution

Revise the PR to reuse or match the shared model-token and assistant-visible tool-call sanitizers at the session-memory read/write boundaries, preserve concise cleaned replies, add focused regressions, and include redacted real hook proof.

## Reproduction Assessment

Yes, at source level. On current main, a JSONL assistant message containing `<tool_call>...</tool_call><|im_end|>` is extracted unchanged, appended as role-prefixed text, and written under `## Conversation Summary`.

## Solution Assessment

No. Sanitizing before persistence is the right boundary, but this implementation duplicates incomplete sanitizer logic and can still persist known markers while eliding valid short replies and prose examples.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Reuse the shared special-token sanitizer:** `src/hooks/bundled/session-memory/sanitize.ts:20-21`
  - body: The hard-coded token regex is narrower than the existing model-token contract, so markers such as `[INST]`, `</s>`, `<start_of_turn>`, harmony tokens, and `<|reserved_special_token_N|>` can still be persisted in session memory. Reuse or match the shared sanitizer instead of maintaining a partial list here.
  - confidence: 0.93
- **[P2] Require tool-call shape before stripping XML:** `src/hooks/bundled/session-memory/sanitize.ts:32`
  - body: The open-ended `<tool_call>` regex removes normal prose from any literal `<tool_call>` mention through the end of the turn. That corrupts legitimate explanations like `Use <tool_call> to invoke tools.`, so this should use the shared tag-aware stripping behavior or require a tool-call-shaped payload.
  - confidence: 0.92
- **[P2] Preserve concise replies after token cleanup:** `src/hooks/bundled/session-memory/sanitize.ts:91-95`
  - body: The `strippedRatio > 0.5 && cleaned.length < 80` heuristic elides useful short replies whenever one leaked token dominates the character count, such as `2+2=4<|im_end|>`. Short factual replies are valid memory context, so elide only empty, NO_REPLY, or structurally malformed remnants.
  - confidence: 0.9
