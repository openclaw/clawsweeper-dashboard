## Summary

Close: this PR adds a first-party community skills catalog, but current OpenClaw direction routes skill discovery, promotion, provenance, and scan status through ClawHub; the patch also documents an invalid owner/name install command.

## What This Changes

The PR adds a new `docs/tools/community-skills.md` page listing third-party skills and inserts it into the Skills docs navigation.

## Best Possible Solution

Keep community skill discovery and promotion in ClawHub, with official docs linking to ClawHub and the skill creation guide instead of maintaining a curated first-party list.

## Reproduction Assessment

Not applicable as a runtime bug report. For the PR defects, yes: source inspection shows the new page documents `openclaw skills install nerua1/skill-name`, while current CLI validation rejects slash-containing slugs.

## Solution Assessment

No. Current VISION and docs make ClawHub the maintainable discovery and scan surface; adding a first-party curated docs catalog with GitHub clone guidance is not the narrowest or safest path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Remove the invalid owner/name install example:** `docs/tools/community-skills.md:85`
  - body: `openclaw skills install` accepts a single ClawHub slug, but this example passes `nerua1/skill-name`. Current slug validation rejects `/`, so the new docs would tell users to run a command that fails before any install lookup.
  - confidence: 0.95
- **[P2] Route third-party installs through ClawHub:** `docs/tools/community-skills.md:80-85`
  - body: The page tells users to clone arbitrary GitHub repos into a loaded skill root even though current docs make ClawHub the registry with visible scan state before install. Official docs should not bypass that trust boundary for community skills.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The docs-only patch adds official install guidance for unverified third-party skill clones outside the ClawHub scan/discovery boundary.

Concerns:

- **[medium] Clarify third-party skill install trust boundary:** `docs/tools/community-skills.md:80`
  - body: The new page directs users to clone third-party repositories into a loaded skill root, bypassing the ClawHub scan visibility documented elsewhere. Official docs should either require ClawHub-published slugs with scan state or frame GitHub cloning as a manual review-before-use path.
  - confidence: 0.86
