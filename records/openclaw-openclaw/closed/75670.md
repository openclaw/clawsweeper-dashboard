## Summary

Keep open. Current main still has a source-reproducible Matrix opaque-ID case-collapse path: Matrix thread routing preserves mixed-case event IDs, but shared session construction, inbound recording, and store lookup still lowercase Matrix room/thread session keys; the adjacent Matrix thread-history PR does not close this case-normalization bug.

## What This Changes

The issue asks OpenClaw to preserve Matrix room and thread event ID casing through session keys, store lookup, and thread reply delivery.

## Best Possible Solution

Add a case-preserving opaque-identifier path for Matrix room/thread session components across construction, recording, store lookup, delivery metadata, transcript lookup, and outbound recovery, while keeping generic lowercase canonicalization for channels that require it.

## Reproduction Assessment

Yes, source-level. Current main preserves Matrix thread roots in the Matrix helper, then lowercases Matrix room/thread session identifiers in shared route construction, inbound recording, and store lookup before persistence and recovery.

## Solution Assessment

Yes. The narrow maintainable fix is a Matrix-aware opaque identifier seam; stuck-session recovery in https://github.com/openclaw/openclaw/issues/71127 or disabling thread replies would only mask the wedge, not preserve valid Matrix relation targets.
