## Summary

Current main already fixes the runtime compatibility problem this PR targets: the plugin loader maps openclaw/plugin-sdk to the compat-backed root alias, and that compat surface now exposes normalizeAccountId and resolvePreferredOpenClawTmpDir. The exact static root-index re-export proposed here is intentionally not used because current policy keeps the root SDK surface tiny and routes legacy broad imports through the compat wrapper.

## Best Possible Solution

Close this PR and keep the current-main compat-backed root alias implementation. It addresses the external plugin runtime failure without expanding the intentionally small static root SDK barrel; new plugin code should use focused subpaths such as openclaw/plugin-sdk/account-id and openclaw/plugin-sdk/temp-path.
