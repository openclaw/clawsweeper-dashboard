## Summary

Current `main` already threads plugin-registered tools into the agent runtime. The code path that the report called out as missing now exists end-to-end, and the remaining exclusion path is the intentional tool-policy filter, which explicitly preserves plugin-only allowlists and `group:plugins` on current `main`. Based on the code, tests, docs, and later issue discussion, this no longer reproduces as a runtime handoff bug.
