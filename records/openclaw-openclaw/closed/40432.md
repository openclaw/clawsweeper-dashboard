## Summary

Current `main` already carries the locale-sync fix for the Overview tab regression, and the same implementation is present in release `v2026.4.22`.

## Evidence

- **Overview uses persisted locale for the selector:** The Overview language `<select>` derives `currentLocale` from `props.settings.locale` before falling back to `i18n.getLocale()`, so revisiting the tab keeps the saved language instead of snapping back to English.

  - file: [ui/src/ui/app.ts:142](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/ui/src/ui/app.ts#L142)
