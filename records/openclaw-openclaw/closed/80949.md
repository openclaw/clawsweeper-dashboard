## Summary

Keep open: the PR addresses a real message-tool-only final-visibility gap, but it should not merge as-is because it turns normal final text into visible channel output without an explicit policy/config decision and without live Discord or Telegram proof.

## What This Changes

The branch adds message-tool-only prompt/tool guidance, suppressed-final diagnostics, metadata-based fallback delivery, and focused tests/snapshots across agent and auto-reply paths.

## Best Possible Solution

Keep the prompt guidance and privacy-safe audit, but make same-turn auto-delivery diagnostic-only unless maintainers explicitly approve a visible-delivery policy or config-gated fallback with live channel proof.

## Reproduction Assessment

Yes by source inspection: current main resolves group/channel turns to message_tool_only and dispatch suppresses ordinary final payloads, so a final answer without message(action=send) stays private. I did not run a live channel repro in this read-only review.

## Solution Assessment

No. The guidance and audit warning are narrow, but default auto-delivery of normal finals is not the safest maintainable fix without an explicit policy/config gate and live Discord or Telegram proof.
