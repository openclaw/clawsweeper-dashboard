## Summary

Current `main` TUI code switches the active session when `/agent <id>` runs, so subsequent messages are sent with the new agent-scoped session key instead of staying on `agent:main:*`. The fix is present on `main` after the latest tagged release, so the issue is resolved in code but not yet in `v2026.4.23`.
