## Summary

This PR is a new optional channel plugin, and current main explicitly steers that work toward external/community plugins published on ClawHub or npm rather than new in-repo bundled plugins. The proposed implementation also targets an outdated in-repo layout (`workspace/plugins/...`) that is not present on current main, where bundled plugins live under `extensions/` and use the current channel entry contract.

## Evidence

- **Vision sets the default scope boundary for optional plugins:** Current project vision says optional capability should usually ship as plugins, the preferred path is npm distribution plus local extension loading for development, plugin authors should host and maintain plugins in their own repository, and the bar for adding optional plugins to core is intentionally high.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/VISION.md#L54)

- **Plugin docs say new plugins do not need to be added to this repository:** The getting-started plugin docs explicitly say: do not add the plugin to the OpenClaw repo; publish to ClawHub or npm and install with `openclaw plugins install ...`. The same page shows external publish/install as the normal path.
  - file: [docs/plugins/building-plugins.md:16](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/docs/plugins/building-plugins.md#L16)
  - sha: [76a4c167f7ef](https://github.com/openclaw/openclaw/commit/76a4c167f7ef65040fd5b00d206c59b8bb28fd49)
- **Community plugin policy covers third-party channel plugins:** Community plugins are documented as third-party packages published on ClawHub or npm, and the submission guide says to publish there and host the source in a public repo instead of adding repo-local discoverability changes.
  - file: [docs/plugins/community.md:9](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/docs/plugins/community.md#L9)

Discord is a representative example: it declares install/release metadata in `extensions/discord/package.json` and registers through `defineBundledChannelEntry` in `extensions/discord/index.ts`.
