## Best Possible Solution

## Reproduction Assessment

Yes. The original failure path was a strict config/schema rejection plus missing runtime fallback; current source and regression tests provide a high-confidence check that the fields are accepted and `agentCfg.reasoningDefault` resolves when no directive or session override is present.

## Solution Assessment

Yes. The implemented path is additive, schema-backed, and centralized in the existing directive/status resolution code, preserving directive and session precedence while avoiding channel-specific policy in the config layer.
