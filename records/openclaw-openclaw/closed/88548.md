## Summary

Current main and the latest stable release now make refreshable provider catalogs runtime-owned, so the Copilot live `/models` catalog is no longer shadowed by bundled static rows for list and picker surfaces.

## What This Changes

The issue requests that GitHub Copilot model list and picker visibility prefer live `/models` entitlement discovery over bundled static catalog defaults, with static rows only as fallback.

## Best Possible Solution

Keep the shipped live-first refreshable catalog behavior and leave the remaining xhigh reasoning-only PR to be reviewed on its own merits.

## Reproduction Assessment

Yes. The old source path is source-reproducible: refreshable manifest rows could be treated as complete static catalog coverage before runtime discovery, and current tests now cover both list and picker static-mask cases.

## Solution Assessment

Yes. The current fix is the narrow maintainable path: refreshable catalogs are runtime-owned, model list and picker prefer runtime rows, and static rows remain only as explicit or failure fallback.
