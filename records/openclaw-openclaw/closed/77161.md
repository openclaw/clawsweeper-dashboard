## What This Changes

The PR updates `extensions/bluebubbles/src/monitor-processing.ts` so delayed BlueBubbles typing restart requests are tracked before final cleanup awaits pending typing work and the final stop-typing request.

## Best Possible Solution

Land the narrow BlueBubbles cleanup after maintainer review, with an Unreleased changelog entry and preferably a focused timer/order regression test in the BlueBubbles monitor coverage.

## Reproduction Assessment

Yes, from source inspection: current `main` can leave a delayed `typing=true` restart POST in flight while final cleanup sends an unawaited `typing=false`, so request ordering can leave the remote indicator active. I did not run the live BlueBubbles/iMessage repro in this read-only review.

## Solution Assessment

Yes, the proposed code is the narrowest maintainable fix for the race because it only tracks restart typing promises and awaits the final stop request in the owning BlueBubbles cleanup path. It should still get the required changelog entry, and a deterministic timer/order test would make the fix safer to land.
