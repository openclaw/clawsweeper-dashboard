## Summary

Current main already fixes the reported startup stall path. `shouldSuppressBuiltInModel` no longer imports or calls provider runtime suppression hooks, so it cannot recurse into `resolveProviderPluginsForCatalogHooks`/`loadOpenClawPlugins`; startup also schedules primary-model prewarm in the background, scopes provider discovery to the configured provider, bounds it to 5s, and splits `sidecars.model-prewarm` from `sidecars.channel-start`.

## Best Possible Solution

Keep the current main behavior: model suppression should stay manifest-backed and runtime-hook-free, and gateway startup should keep model prewarm metadata-only, provider-scoped, bounded, and off the `sidecars.channels` critical path. Any remaining startup stall should be filed as a fresh current-main report with split `sidecars.model-prewarm` and `sidecars.channel-start` trace data.
