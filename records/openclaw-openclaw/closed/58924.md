Current `main` already sanitizes assembled session history before model calls, repairs/drops orphaned tool-use/tool-result pairs for Anthropic replay, and includes a regression test for switching from OpenAI-style history to Anthropic.

## Risks / Open Questions

- The repair is request-time transcript hygiene, not a physical cleanup of persisted `lcm.db` rows. If the goal was database self-healing rather than avoiding Anthropic request failure, that narrower behavior is not shown here.
- `lossless-claw` itself is not bundled in this checkout, so this review proves the OpenClaw core replay path, not plugin-internal storage mechanics.
