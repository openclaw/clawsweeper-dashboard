## Summary

Close as not actionable in core: the report checks the root `openclaw` package, but Matrix is published and installed as `@openclaw/matrix`, whose shipped 2026.5.27 and 2026.5.28-beta.1 metadata already declares the runtime SDK dependencies.

## What This Changes

The issue asks OpenClaw to include the Matrix SDK runtime dependency in the package so fresh 2026.5.x installs do not crash-loop Matrix.

## Best Possible Solution

Keep Matrix runtime dependencies owned by `@openclaw/matrix` and use the documented plugin install/update path; reopen only with a crash trace from a clean plugin install.

## Reproduction Assessment

No. The root-package dependency absence can be observed, but that is expected because Matrix is a separate plugin; the supported `@openclaw/matrix` packages for 2026.5.27 and 2026.5.28-beta.1 declare the runtime SDK deps.

## Solution Assessment

No. Adding Matrix SDK deps to the root `openclaw` package would move plugin-owned runtime deps into core; the maintainable solution is the existing external plugin package and install/update path.
