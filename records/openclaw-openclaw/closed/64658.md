## Summary

Close as duplicate/superseded. The requested per-spawn `none | parent | channel` completion-delivery policy is already covered by older open canonical feature requests for parent routing, deterministic suppression, and channel-level suppression; current main still lacks the feature, so the remaining work should stay consolidated there.

## What This Changes

Add a public `sessions_spawn` completion-announcement control so sub-agent results can be suppressed, routed only to the parent, or delivered to the inherited channel.

## Best Possible Solution

Keep the implementation discussion in https://github.com/openclaw/openclaw/issues/27445 for parent routing, https://github.com/openclaw/openclaw/issues/8299 for deterministic suppression, and coordinate channel precedence with https://github.com/openclaw/openclaw/issues/13911.
