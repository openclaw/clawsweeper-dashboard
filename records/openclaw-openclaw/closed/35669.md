## Summary

Current `main` already fixes the BlueBubbles iMessage-vs-SMS routing bug through shipped resolver logic, tests, and docs, so this PR is superseded by the existing implementation.

## Evidence

## Risks / Open Questions

- `main` fixes the reported downgrade bug through handle-ranking plus `sms:+handle`, not by preserving DM `chat_guid` service prefixes. If maintainers still want that exact normalization contract as an extra feature, it would need a fresh proposal.
