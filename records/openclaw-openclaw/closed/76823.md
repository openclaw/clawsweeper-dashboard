## Summary

Keep open: current main still preserves fresh live-PID session locks during doctor repair, and open PR #76854 already tracks the narrow false-live owner-check repair for this report. Review target: https://github.com/openclaw/openclaw/issues/76823

## What This Changes

`openclaw doctor --fix` should clean blocking session lock files whose recorded PID is alive but no longer belongs to the Gateway or an active OpenClaw writer.

## Best Possible Solution

Review and land or replace #76854 with a conservative owner-verification fix that removes fresh live-PID locks only when the readable owner is clearly not OpenClaw.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a recent `.jsonl.lock` with a live PID and run repair cleanup; current logic gives it no stale reason and the existing test asserts that a fresh live lock remains.

## Solution Assessment

No for the literal suggested Gateway-down delete-all approach, because it is too broad for active or unknown owners. The narrower owner-argv verification proposed in #76854 is the safer maintainable direction.
