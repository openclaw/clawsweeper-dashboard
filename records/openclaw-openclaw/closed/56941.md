## Summary

Close as implemented: current `main` routes `heartbeat.isolatedSession` through a forced fresh isolated session, clears stale transcript/bootstrap state on rollover, and includes regression coverage for the stale `:heartbeat`/session-file reuse paths described in the report. The same implementation is present in the shipped `v2026.4.23` tag.

## Evidence

- **Docs contract still promises fresh heartbeat sessions:** The heartbeat docs and config type both state that `isolatedSession` means a fresh session with no prior conversation history.
  - file: [docs/gateway/heartbeat.md:227](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/docs/gateway/heartbeat.md#L227)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Heartbeat runner forces isolated runs onto a fresh cron-style session:** `runHeartbeatOnce()` detects `heartbeat.isolatedSession === true`, resolves the synthetic `:heartbeat` session key, and calls `resolveCronSession(..., forceNew: true)` before saving the session entry.
  - file: [src/infra/heartbeat-runner.ts:878](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/infra/heartbeat-runner.ts#L878)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Forced isolated sessions create a new sessionId and drop stale transcript pointers:** `resolveCronSession()` generates a new UUID when `forceNew` is set, clears bootstrap cache on rollover, and removes `sessionFile` from fresh session entries so later runs do not append to an old transcript.
  - file: [src/cron/isolated-agent/session.ts:55](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/cron/isolated-agent/session.ts#L55)

## Close Comment

Closing this as implemented on current `main`.

- Codex review found `heartbeat.isolatedSession` now routes through a dedicated `:heartbeat` session and calls `resolveCronSession(..., forceNew: true)`, so each heartbeat tick gets a fresh session instead of reusing persisted transcript state (`src/infra/heartbeat-runner.ts:878`).
- The shared isolated-session helper now creates a new `sessionId` on forced rollovers, clears stale `sessionFile` metadata, and clears bootstrap cache on rollover, which addresses the reported old-JSONL / stale-`HEARTBEAT.md` behavior (`src/cron/isolated-agent/session.ts:55`, `src/agents/bootstrap-cache.ts:23`).
- Regression coverage exists for both halves of this bug: fresh `sessionId` plus cleared `sessionFile` on forced isolated runs (`src/cron/isolated-agent/session.test.ts:151`) and canonical heartbeat isolated-session key handling without stale `:heartbeat:heartbeat` reuse (`src/infra/heartbeat-runner.isolated-key-stability.test.ts:102`).
- The docs still describe `isolatedSession` as a fresh session each run with no conversation history, and the current implementation matches that contract (`docs/gateway/heartbeat.md:227`).
- This was a Codex review, and the shipped `v2026.4.23` tag also contains the same implementation path.
