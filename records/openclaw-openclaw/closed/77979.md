## Summary

Keep open: current main still lets unauthenticated device handshakes reach device-id derivation and v3/v2 signature verification without a device-signature-specific limiter, and the linked closing PR remains open and conflicting.

## What This Changes

The issue requests rate limiting and cheap prechecks for pre-auth gateway device-signature verification to prevent CPU-amplification DoS.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/77492 or a narrow maintainer replacement that adds a pre-crypto per-IP device-signature guard, cheap input bounds/prechecks, and regression coverage, then close this issue through the merged fix.

## Reproduction Assessment

Yes. Current main source shows caller-controlled device public keys and signatures reach deriveDeviceIdFromPublicKey and v3/v2 verifyDeviceSignature before any device-signature limiter; the linked PR also supplies a concrete WS flood PoC.

## Solution Assessment

Yes for the issue direction. The maintainable fix belongs in the gateway handshake path as a pre-crypto limiter plus cheap input bounds/prechecks, with security review on the linked PR or replacement before merge.
