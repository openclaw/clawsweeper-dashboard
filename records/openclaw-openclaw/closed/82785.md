## Summary

Keep open: current main still drops the checked auth source from missing-key diagnostics, and the open closing PR is the right path to review instead of closing this issue or spawning a competing fix.

## What This Changes

The issue requests that missing API key errors include the checked auth source label so users know which environment variable, profile, or config key was evaluated.

## Best Possible Solution

Land the shared missing-auth formatter from the linked PR after normal review and CI, covering both requireApiKey and the direct agent runtime paths without changing credential resolution semantics.

## Reproduction Assessment

Yes. Source inspection shows a ResolvedProviderAuth with source: "env: OPENAI_API_KEY" reaches requireApiKey, and current main throws a message that omits that source; direct agent runtime paths have the same source-less formatting.

## Solution Assessment

Yes. The open linked PR centralizes the missing-auth formatter and routes the duplicated call sites through it, which is narrower than changing auth resolution or adding config.
