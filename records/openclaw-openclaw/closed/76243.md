## Summary

Current main fixes the reported 2026.4.29 gap: the agent message tool now resolves scoped channel SecretRefs even when constructed with an injected config snapshot, and Telegram registers `channels.telegram.botToken` as a supported command-secret target.

## What This Changes

The issue reports that Telegram `channels.telegram.botToken` SecretRefs can resolve for the channel runtime but remain unresolved in an agent/message-lane send path.

## Best Possible Solution

Keep the current generic scoped message SecretRef resolution and let the next release carry the main-branch fix rather than adding Telegram-specific credential handling.

## Reproduction Assessment

Yes. The `v2026.4.29` source reproduces the failure shape because `createMessageTool` skipped command-secret resolution whenever the agent supplied `options.config`; current main has a targeted regression test for that exact config-snapshot path.

## Solution Assessment

Yes. The merged fix uses the existing command-secret target registry and scoped resolver, which is narrower and more maintainable than adding a Telegram-only token workaround.
