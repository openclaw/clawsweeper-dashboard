## Summary

Current `main` already contains a Discord preview-streaming fix that prevents later assistant messages from overwriting the first visible Discord reply. The new logic rotates to a fresh Discord preview message on assistant-message boundaries, and there is a focused regression test for that exact behavior.

## Evidence
