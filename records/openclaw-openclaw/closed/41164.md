## Summary

Close #41164 as a duplicate of the already-open canonical tracker #9155. Current main still does not implement config-level primary cron delivery defaults, but #9155 tracks the same remaining feature with the same `cron.defaults.delivery`/default channel-target request, so this newer report can be deduped without losing the work.

## Best Possible Solution

Close #41164 as a duplicate of #9155, keep #9155 open as the canonical tracker, and implement the feature there by adding a real config-level primary cron delivery default through `CronConfig`, strict zod/generated config schema/help/docs, cron creation/update paths, CLI/UI surfaces as needed, and regression tests proving job-level delivery overrides the global default before any fallback to `last`.
