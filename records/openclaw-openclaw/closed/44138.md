## Close Comment

Closing from a Codex review: this specific issue is already implemented on current `main`. The compaction safeguard now checks both `messagesToSummarize` and `turnPrefixMessages`, so split-turn/thread-prefix content no longer falls into the old `no real conversation messages` cancel path (`src/agents/pi-hooks/compaction-safeguard.ts:727`). There is also a regression test for the exact case where only `turnPrefixMessages` has real content (`src/agents/pi-hooks/compaction-safeguard.test.ts:1875`). The fix was shipped in release `2026.3.22`, whose changelog explicitly calls out keeping split-turn prefixes on the normal compaction path (`CHANGELOG.md:2191`, `CHANGELOG.md:2440`). If you still hit a different manual compaction failure on a newer build, please open a fresh issue with current logs and session/channel details.
