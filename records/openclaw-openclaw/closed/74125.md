## Summary

Keep open. Current main still shows the reported cold wake-up path risk: system-event turns build CLI reuse prompt identity from the live heartbeat/cron context, CLI reuse still invalidates on prompt-hash drift, and the `cli exec` log still lacks the requested resume diagnostics. There is also an open unmerged PR (#74171) with closing syntax for this issue, so the right maintainer path is to review and land or close that PR before resolving this issue.

## Best Possible Solution

Keep this issue open and use #74171 as the canonical implementation candidate. The best fix is a narrow runtime change that restores persisted Discord prompt-only metadata for wake/heartbeat/cron system-event turns before CLI prompt identity hashing, adds redacted resume diagnostics, and keeps auth/MCP invalidation strict. Review should also keep #69118 separate unless that broader group-intro prompt drift is intentionally handled by the same patch.
