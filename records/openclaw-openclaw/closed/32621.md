## Summary

Keep open. Current `main` still only runs `message_sending` inside the outbound delivery pipeline, while explicit `message(action=send)` calls still route through the message-tool action runner and channel plugin `handleAction` paths without any pre-send `message_sending` emission. That matches the issue report, recent April 2026 confirmations from users, and the fact that the earlier fix PR #32834 was closed unmerged while related follow-up PRs remain open.

## Evidence

- **`message_sending` still lives in outbound delivery only:** `deliverOutboundPayloads()` checks `hasHooks("message_sending")` and calls `applyMessageSendingHook(...)` immediately before payload delivery. This is the concrete pre-send hook site on current `main`.
