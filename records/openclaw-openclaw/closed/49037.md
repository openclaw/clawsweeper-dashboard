## Summary

Current main already addresses the reported launchd lock-contention crash loop: supervised gateway startups retry instead of exiting, stale/dead lock owners are reclaimed, and the generated LaunchAgent plist includes `ThrottleInterval` with `KeepAlive=true`. The same behavior is present in the shipped `v2026.4.22` release commit.

## Evidence

- **Supervised startup no longer exits on healthy lock contention:** `openclaw gateway run` detects a respawn supervisor and, on `GatewayLockError("gateway already running...")`, logs a warning and sleeps 5000ms before retrying startup instead of exiting back to launchd/systemd.
  - file: [src/cli/gateway-cli/run.ts:669](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/cli/gateway-cli/run.ts#L669)
  - sha: [01bc49c88c0c](https://github.com/openclaw/openclaw/commit/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd)
- **Dead/stale gateway lock owners are cleaned automatically:** Gateway lock acquisition removes the lock immediately when the recorded owner is dead, and also reclaims non-alive stale locks after staleness checks instead of failing fast forever on leftover lockfiles.
  - file: [src/infra/gateway-lock.ts:299](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/infra/gateway-lock.ts#L299)
  - sha: [01bc49c88c0c](https://github.com/openclaw/openclaw/commit/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd)
- **LaunchAgent plist includes launchd throttling:** The launchd plist builder emits `RunAtLoad`, `KeepAlive`, and `ThrottleInterval`, so rapid relaunches are throttled by launchd rather than spinning with immediate restarts.
  - file: [src/daemon/launchd-plist.ts:116](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/daemon/launchd-plist.ts#L116)
  - sha: [01bc49c88c0c](https://github.com/openclaw/openclaw/commit/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd)
- **Regression coverage for stale-lock recovery on macOS:** Tests cover reclaiming stale locks when the Darwin lock owner is not actually a gateway process, and reclaiming locks when the configured gateway port is free.
  - file: [src/infra/gateway-lock.test.ts:249](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/infra/gateway-lock.test.ts#L249)
