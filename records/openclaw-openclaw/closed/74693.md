## Summary

Close as implemented on current main: the read-only channel setup fallback now detects bundled `dist/` importers, resolves stable `dist/plugins/*` loader candidates inside the installed OpenClaw package, and has regression coverage for the packaged path. The fix is not in the latest shipped `v2026.4.27` tag, so this is main-only until the next release.

## What This Changes

Requests that `openclaw status` and related read-only channel setup fallback paths resolve the packaged plugin runtime loader without escaping the installed OpenClaw package when an external channel plugin needs setup metadata.

## Best Possible Solution

Keep the current main implementation and ship it in the next release; the external WeChat plugin should still add `channelConfigs` metadata as its own follow-up, but OpenClaw core no longer needs an additional loader-path repair for this crash.

## Reproduction Assessment

Yes. The issue gives a concrete packaged-install reproduction with `openclaw status`/`status --deep`, and multiple commenters confirmed the same path escape on Linux and macOS; current-main source and tests directly cover the old URL calculation and the fixed packaged candidate list.

## Solution Assessment

Yes. Resolving fallback candidates from the detected `dist/` root and using stable `dist/plugins/*` build entries is the narrow maintainable fix because it preserves source-mode loading while making packaged chunks package-relative.
