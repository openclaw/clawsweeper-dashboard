## Label Justifications

- `P1`: The original bug could prevent SDK-using channel plugins from registering in pnpm global installs, breaking real channel workflows.
- `impact:message-loss`: Plugin registration failure can stop affected channel plugins from delivering or receiving messages.

## Summary

Current main and the shipped release line already fix the reported pnpm global-install zod resolution failure: the build now always bundles zod/zod subpaths for the unified dist graph, and package verification rejects a non-self-contained plugin-sdk zod artifact.

## What This Changes

The issue asks OpenClaw to make `openclaw/plugin-sdk/zod` self-contained so pnpm global installs do not fail plugin registration with a missing bare `zod` import.

## Best Possible Solution

Keep the merged artifact-bundling and verifier guardrail; affected users should upgrade to `v2026.5.18` or newer.

## Reproduction Assessment

Yes for the historical bug: the reported 2026.5.4 artifact shape and merged PR proof show a bare zod import failing under pnpm global installs. Current main no longer reproduces that failure because zod is bundled and the installed artifact verifier rejects regressions.

## Solution Assessment

Yes. Bundling `zod` and `zod/*` through the existing tsdown dependency policy plus a postpublish artifact verifier is narrower and more maintainable than a package-manager-specific postinstall or doctor workaround.
