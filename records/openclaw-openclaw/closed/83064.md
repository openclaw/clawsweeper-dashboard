## Summary

Current main already honors per-agent bootstrap budget overrides end-to-end, with source, schema, docs, and regression coverage all aligned; the fix is present only on main and is not in the latest tagged release yet.

## What This Changes

The issue reports that `agents.list[].bootstrapMaxChars` and `bootstrapTotalMaxChars` were accepted by config types/docs but ignored by runtime bootstrap truncation.

## Best Possible Solution

Keep the current main implementation that honors the documented per-agent budget fields and ship it in the next release; users on older builds can keep using `agents.defaults.*` until they update.

## Reproduction Assessment

Current main: no, the reported behavior no longer reproduces from source because the agent id now reaches the bootstrap budget resolvers. The original released behavior is source-proven in `v2026.5.12`, where the same resolver read only `agents.defaults.*`.

## Solution Assessment

Yes. Honoring the already-typed and documented per-agent fields is the narrowest maintainable fix, and current main adds resolver coverage instead of rejecting a valid config key.
