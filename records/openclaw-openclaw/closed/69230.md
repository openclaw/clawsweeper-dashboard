## Summary

Current main already implements the PR's central fix with a broader registry-based path: the phantom plugins.allow audit now trusts the plugin registry snapshot for the audited state directory instead of channel-only plugin IDs, so bundled non-channel plugin IDs such as openai, google, and acpx are no longer treated as phantom entries. The implementation also addresses the PR review concern about scoping the lookup to the audited stateDir.

## Best Possible Solution

Close this PR as implemented on current main and keep the registry-snapshot implementation, since it solves the same false-positive audit problem while also scoping the lookup to the audited state directory.
