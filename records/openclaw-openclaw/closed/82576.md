## Summary

Keep open: current `main` still has the reported stale-snapshot write path, and the issue is paired with an open fixing PR that references it with closing syntax.

## What This Changes

The issue reports that `persistSessionUsageUpdate` overwrites post-compaction `totalTokens` state when a model call lacks a fresh context snapshot.

## Best Possible Solution

Review and land a narrow fix that only updates `totalTokens` and `totalTokensFresh` when a fresh context snapshot exists, with regression coverage for preserving post-compaction token state.

## Reproduction Assessment

Yes, at source level. A usage update with no `lastCallUsage`, no positive `promptTokens`, and no `usageIsContextSnapshot` computes `totalTokens` as `undefined` and writes `totalTokensFresh=false`, which can overwrite the fresh post-compaction state.

## Solution Assessment

Yes, the proposed direction is the narrow maintainable fix: preserve existing total-token state when no fresh context snapshot is available. The linked PR should be reviewed and validated before this issue closes.
