## What This Changes

The PR changes QQBot debug logging to join sanitized arguments into one CRLF-neutralized string before calling `console.log`, `console.warn`, or `console.error`, and updates the debug-log test expectation.

## Best Possible Solution

Keep the PR open and land this narrow QQBot plugin fix if the targeted test, formatter check, changed gate, and CodeQL/security review confirm the single-line console sink resolves the alert without regressing debug logging.

## Reproduction Assessment

Yes. On current `main`, `debugLog("prefix", "line one\nline two")` sanitizes values but still calls `console.log` with spread varargs, and the existing colocated test exercises that exact path.

## Solution Assessment

Yes, with maintainer validation. Joining already-sanitized values into one string at the QQBot debug helper is the narrowest maintainable fix for a console varargs log-injection finding without changing call sites across the plugin.
