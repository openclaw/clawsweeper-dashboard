## Summary

Current main already implements the shared auto-reply queue/drain safeguards needed for this stale fast-follow reply race. WhatsApp and Discord route through that shared path, and current tests cover stale callback and stale active-run snapshot cases.

## Best Possible Solution

Close this as implemented on current main and keep the shared queue/drain safeguards. If a new stale-reply reproduction appears on a build containing bdfb408ce6eeb724a638c106104dbbcae50b612c or later, it should be filed as a fresh issue with channel, exact version/SHA, session key, message ids, queue mode, and gateway logs.
