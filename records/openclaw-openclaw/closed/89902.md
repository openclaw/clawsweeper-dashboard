## What This Changes

The branch adds a gateway discovery registration reader, uses snapshotted plugin/service metadata for advertiser logging while preserving the service receiver, and adds regression tests for unreadable metadata plus explicit unknown service ids.

## Best Possible Solution

Rebase this branch onto current main, keep the focused metadata guard plus regression tests, and merge it if normal gateway CI stays green.

## Reproduction Assessment

Yes. Source inspection shows current main can re-read throwing service metadata while logging a discovery advertisement failure, which can escape and prevent later healthy discovery services from starting; I did not run tests because this was a read-only review.

## Solution Assessment

Yes. Snapshotting the service and identifiers once before advertising is the narrowest maintainable fix; registry-only validation would not protect against malformed or mutated registration objects at startup.
