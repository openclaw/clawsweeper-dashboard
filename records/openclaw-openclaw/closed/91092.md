## What This Changes

The issue reports WeChat channel messages exposing OpenClaw’s inbound metadata context prefix to the end user.

## Best Possible Solution

Keep the shared core sanitizer and ask for a fresh report only if the leak still reproduces after updating OpenClaw and the external WeChat plugin.

## Reproduction Assessment

Yes for the core failure shape: current tests construct inbound metadata blocks and prove they are stripped from message-tool dispatch and restart-recovery delivery. I did not run a live WeChat session in this read-only review.

## Solution Assessment

Yes. The implemented shared sanitizer is the best layer because the metadata prefix is generic OpenClaw prompt context and WeChat runtime code lives in the external plugin.
