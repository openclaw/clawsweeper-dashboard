## Summary

Keep this PR open but do not merge yet: the state/report split is a good direction, but the current patch has build-breaking contract gaps and still loses once-mode warning history on the CLI path.

## What This Changes

The PR moves bootstrap truncation warning dedupe history from `systemPromptReport.bootstrapTruncation` into a new `SessionEntry.bootstrapPromptWarningState` field and updates related persistence/reset tests.

## Best Possible Solution

Keep the report/state separation, but update runner and session contracts, carry CLI warning signatures through prepare/result metadata, preserve existing state on unrelated usage updates, and add focused regression coverage.

## Reproduction Assessment

Yes. Source inspection plus CI logs reproduce the PR regressions: the PR reads a new runner meta field that is neither typed nor populated by CLI results, and the new `SessionEntry` key fails the reserved-slot guard.

## Solution Assessment

No. The direction is maintainable, but the implementation must first wire the new state through the CLI/embedded result contracts, session slot list, and usage update preservation paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.96

Full review comments:

- **[P1] Declare and populate warning meta before reading it:** `src/auto-reply/reply/agent-runner-execution.ts:1593-1598`
  - body: This new read of `result.meta.bootstrapPromptWarningSignaturesSeen` does not match `EmbeddedPiRunMeta`, so CI reports TS2339 at each consumer. It also leaves CLI runs without any replacement state because `buildCliRunResult()` still only emits `systemPromptReport`; add the meta field to the runner contract and populate it from CLI prepare/result state before consuming it here.
  - confidence: 0.97
- **[P1] Reserve the new session state key:** `src/config/sessions/types.ts:359`
  - body: Adding `bootstrapPromptWarningState` to `SessionEntry` also needs the corresponding reserved slot entry. Without updating `SESSION_ENTRY_RESERVED_SLOT_KEY_LIST`, the compile-time guard in `src/plugins/session-entry-slot-keys.ts` fails with TS2344 for the new key.
  - confidence: 0.96
- **[P2] Preserve existing warning state when no new state is passed:** `src/auto-reply/reply/session-usage.ts:147-152`
  - body: `persistSessionUsageUpdate()` now writes `bootstrapPromptWarningState: undefined` whenever the optional new parameter is absent, which can clear existing dedupe history during usage-only updates. Only overwrite this field when the caller provides new warning state, otherwise keep `entry.bootstrapPromptWarningState`.
  - confidence: 0.88
- **[P2] Avoid direct typed reads of the removed report field:** `src/agents/command/session-store.test.ts:506-508`
  - body: The test still directly reads `systemPromptReport.bootstrapTruncation.warningSignaturesSeen` after the PR removes that property from `SessionSystemPromptReport`, so test typechecking fails. Use an object-shape assertion or a legacy-metadata cast for this absence check.
  - confidence: 0.93
