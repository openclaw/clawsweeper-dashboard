## Summary

Current main and the v2026.4.24 release contain the maintained fix for the reported heartbeat timer overflow: oversized heartbeat scheduler delays are clamped to Node's 32-bit setTimeout cap before scheduling, with regression coverage for the former tight-loop behavior.

## Evidence

- **Scheduler delay is clamped before setTimeout:** The shipped v2026.4.24 heartbeat scheduler computes rawDelay, warns once when it exceeds MAX_SAFE_TIMEOUT_DELAY_MS, then passes it through resolveSafeTimeoutDelayMs before calling setTimeout. A 30-day delay therefore no longer reaches Node as 2,592,000,000ms.
