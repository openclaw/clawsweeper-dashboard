## Summary

Current main already fixes the Windows Feishu raw drive-letter ESM loader failure reported against OpenClaw 2026.4.24. The bundled Feishu channel now loads through the shared bundled-channel entry contract, Windows absolute paths are normalized to file:// URLs before Jiti/ESM loading, regression tests cover Feishu-style C:\ paths, and the fix shipped in v2026.4.26.

## Best Possible Solution

Close as implemented and keep the shipped shared Windows import normalization plus bundled-channel regression coverage. Users seeing this on 2026.4.24 should update to v2026.4.26 or newer.
