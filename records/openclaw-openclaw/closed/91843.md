Keep this PR open: current main and the latest release still use cumulative cron run usage for the persisted context snapshot, while the embedded runner exposes `lastCallUsage` specifically for final-call context accounting. ## What This Changes

The PR changes cron isolated-run finalization to prefer `agentMeta.lastCallUsage` when deriving the persisted context token snapshot while leaving cumulative usage for billing/token accounting.

## Best Possible Solution

Land the narrow cron writer fix after contributor-provided after-fix proof, keeping cumulative usage for accounting and using final-call usage only for the context snapshot.

## Reproduction Assessment

Yes, by source inspection: current `finalizeCronRun` persists the context snapshot from accumulated `usage`, while embedded agent metadata separately exposes `lastCallUsage` for the final API call. I did not run the repro because this review is read-only.

## Solution Assessment

Yes, this appears to be the best fix shape: update the cron writer at the persistence boundary to use the final-call context snapshot while preserving cumulative usage for billing and telemetry. A regression test would make the fix safer to land.
