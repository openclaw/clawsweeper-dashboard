## Summary

Keep open: current main still has a source-reproducible stale runtime model cache path, and https://github.com/openclaw/openclaw/pull/77339 is an open closing PR for the same bug.

## What This Changes

The issue reports that channel `/new` and `/reset` sessions preserve cached `model`/`modelProvider` after `agents.defaults.model.primary` changes when no explicit model override exists.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/77339, or an equivalent narrow patch that clears runtime `model`/`modelProvider` cache fields during reset-created channel sessions while preserving explicit user overrides.

## Reproduction Assessment

Yes. Seed a persisted channel session row with `modelProvider` and `model` but no explicit override, trigger `/new`, and current main's merge path preserves those fields before `resolveSessionModelRef` prefers them over the configured default.

## Solution Assessment

Yes. Clearing only the runtime cache fields in the existing reset cleanup block is the narrow maintainable fix because explicit user selections already live in `modelOverride` and `providerOverride`.
