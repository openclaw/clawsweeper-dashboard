## Summary

Close: current main and v2026.5.19 already drop unsafe GitHub Copilot Responses reasoning replay items at the provider payload boundary, with regression coverage for overlong reasoning IDs; the report was from 2026.5.18.

## What This Changes

The issue reports that GitHub Copilot Responses replay can resend overlong historical reasoning item IDs from thinkingSignature blocks and fail before generation.

## Best Possible Solution

Keep the provider-bound Copilot payload sanitizer and ask users on 2026.5.18 to upgrade to v2026.5.19 or newer before filing any new redacted final-payload evidence.

## Reproduction Assessment

Yes for the old failure class: the issue provides a concrete 2026.5.18 Copilot/Feishu transcript shape, and current tests use an overlong reasoning replay ID. I did not replay the private session, but current main source shows that item class is removed before dispatch.

## Solution Assessment

Yes; the provider-bound final payload sanitizer is the maintainable seam because it covers reasoning items after Responses conversion and immediately before Copilot dispatch without changing generic OpenAI Responses replay semantics.
