## What This Changes

The issue reports that dashboard child-session rollover can reject an already-allowed `openai/gpt-5.5` selection and revert the session to stale `openai/gpt-5.4` state.

## Best Possible Solution

Review and land the narrow stored-override normalization plus stale reset-diagnostic fix in https://github.com/openclaw/openclaw/pull/94752, then close this issue after it reaches main; keep the broader WebChat inheritance/default-picker contract in https://github.com/openclaw/openclaw/issues/86174.

## Reproduction Assessment

Yes, at source level. Current main still creates parent-linked dashboard child sessions, resolves parent stored overrides, and formats stale resets as allowlist rejections; I did not run a live dashboard rollover.

## Solution Assessment

Yes for the narrow issue. The maintainable fix is to normalize persisted overrides before reply reset and separate stale-session copy from true disallowed-copy, as targeted by https://github.com/openclaw/openclaw/pull/94752; the broader WebChat no-inherit/default display contract belongs to https://github.com/openclaw/openclaw/issues/86174.
