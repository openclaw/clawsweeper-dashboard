## Summary

Current `main` no longer matches the reported split state. The bundled browser plugin registers `browser.request` and the browser control service together, gateway startup loads bundled sidecar plugins like `browser` when enabled, and there is an integration test asserting that startup with `browser` present exposes both the method and the service. The issue's `plugins list` evidence is also not reliable for a live gateway, per current docs.
