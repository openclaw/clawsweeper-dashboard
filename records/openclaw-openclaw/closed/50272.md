## Summary

Current `main` already preserves external delivery routing for inter-session `sessions_send` turns and explicitly routes Feishu-origin replies back to Feishu, which matches the reported failure mode.

## Evidence

- **Inter-session sends no longer overwrite external routes:** `resolveLastChannelRaw` and `resolveLastToRaw` now preserve an established external delivery route when `isInterSession` is true, preventing `sessions_send` turns from replacing Feishu/other channel routing with `webchat` session data.
