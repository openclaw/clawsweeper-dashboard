## Summary

Close as implemented on current main: bare plugin installs now route to npm by default, and ClawHub install is reserved for explicit `clawhub:` specs or trusted catalog metadata, with regression coverage and docs. The proof commit is on `main` but is not contained in the latest `v2026.5.3` tag.

## What This Changes

The issue asks OpenClaw to prevent bare plugin installs from resolving to ClawHub names that could be squatted over trusted plugin names.

## Best Possible Solution

Keep the npm-default bare-spec resolver, reserve ClawHub for explicit `clawhub:` and trusted catalog flows, and handle signatures or allowlists as separate defense-in-depth hardening.

## Reproduction Assessment

No against current main. Source inspection and regression coverage show `plugins install demo` goes through npm without calling the ClawHub installer, so the reported current-main substitution path no longer reproduces.

## Solution Assessment

Yes. The current resolver policy removes the reported supply-chain path at the narrow install decision point without adding a new trust UI, config knob, or broader product policy change.
