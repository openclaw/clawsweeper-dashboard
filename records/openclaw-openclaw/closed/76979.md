## Summary

Current main already handles the reported CLI-backend token-counter failure by refusing to treat CLI cumulative usage as a fresh context-window snapshot, so the misleading >100% context display is no longer produced from that field.

## What This Changes

The issue asks OpenClaw to stop showing confidently wrong context-window percentages for CLI-routed sessions when the CLI reports cumulative/internal usage.

## Best Possible Solution

Keep the current main behavior that treats CLI backend usage as non-authoritative for context snapshots and include it in the next release.

## Reproduction Assessment

No, not on current main: the source path now clears the context snapshot for CLI cumulative usage, and the regression test covers the reported multi-million-token CLI usage case. The original production repro is credible for older builds but should no longer produce a fresh >100% context total from `sessions.json.totalTokens`.

## Solution Assessment

Yes. The current implementation chooses the safest suggested behavior, effectively a true-unknown mode for CLI context totals, while preserving raw input/output usage separately for observability.
