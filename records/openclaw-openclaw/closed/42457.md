## Summary

Current main already covers the reported gap: normal sessions auto-load workspace rule files, memory backends index `memory/**/*.md`, and the shipped Active Memory flow injects relevant memory before the main reply. Exact keyword/file routing is also already possible through the existing `before_prompt_build` hook surface.

## Evidence

- **Workspace rules are auto-injected:** Normal sessions automatically load `AGENTS.md` and `MEMORY.md`, so always-on task rules already enter session context without the agent having to remember to fetch them.
  - file: [docs/start/openclaw.md:75](https://github.com/openclaw/openclaw/blob/7d9172d5e093435d37ed3e00e476a4eaf84a252b/docs/start/openclaw.md#L75)
- **Memory files are indexed by default:** The default memory config auto-indexes `MEMORY.md` plus `memory/**/*.md`, which covers dedicated rule files stored under the workspace memory tree.
  - file: [docs/reference/memory-config.md:399](https://github.com/openclaw/openclaw/blob/7d9172d5e093435d37ed3e00e476a4eaf84a252b/docs/reference/memory-config.md#L399)
- **Non-dated memory files stay evergreen:** Non-dated files in `memory/` are explicitly treated as evergreen, which fits durable rule/reference files rather than ephemeral daily notes.
  - file: [docs/reference/memory-config.md:249](https://github.com/openclaw/openclaw/blob/7d9172d5e093435d37ed3e00e476a4eaf84a252b/docs/reference/memory-config.md#L249)
- **Pre-reply context injection is a first-class hook:** `before_prompt_build` is documented as running after session load to inject `prependContext` or system-context text before prompt submission.
  - file: [docs/concepts/agent-loop.md:94](https://github.com/openclaw/openclaw/blob/7d9172d5e093435d37ed3e00e476a4eaf84a252b/docs/concepts/agent-loop.md#L94)
- **Active Memory exists for this exact pain point:** The Active Memory docs say it was added because normal memory is too reactive and that it gives the system a bounded chance to surface relevant memory before the main reply is generated.
  - file: [docs/concepts/active-memory.md:13](https://github.com/openclaw/openclaw/blob/7d9172d5e093435d37ed3e00e476a4eaf84a252b/docs/concepts/active-memory.md#L13)
- **Shipped implementation injects recall before reply:** The bundled `active-memory` plugin hooks `before_prompt_build`, builds a query from the current prompt plus recent turns, resolves recall, and returns `prependContext` for the reply prompt.
  - file: [extensions/active-memory/index.ts:1964](https://github.com/openclaw/openclaw/blob/7d9172d5e093435d37ed3e00e476a4eaf84a252b/extensions/active-memory/index.ts#L1964)
- **Exact custom matching is already extensible:** Plugin hook types already allow `before_prompt_build` handlers to inspect `prompt` and `messages` and return injected context, so stricter keyword/file matching can be implemented on top of the current hook contract.
  - file: [src/plugins/hook-before-agent-start.types.ts:21](https://github.com/openclaw/openclaw/blob/7d9172d5e093435d37ed3e00e476a4eaf84a252b/src/plugins/hook-before-agent-start.types.ts#L21)
- **Feature shipped in latest release:** The latest release tag includes both the Active Memory docs and the bundled `active-memory` pre-reply recall implementation.
  - command: `git show v2026.4.22:docs/concepts/active-memory.md | sed -n '1,24p'`
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)

## Risks / Open Questions

- This is not a first-class keyword-to-file rules UI; the shipped path is workspace bootstrap plus semantic/prompt-time recall, with exact custom routing left to the existing hook/plugin surface.

## Close Comment

Closing after Codex review against current `main`: this is already covered well enough by the shipped workspace + memory pipeline. Normal sessions auto-inject `AGENTS.md` and `MEMORY.md` (`docs/start/openclaw.md:75`), memory backends auto-index `memory/**/*.md` for durable rule/reference files (`docs/reference/memory-config.md:399`), and Active Memory exists specifically to surface relevant memory before the main reply when the agent would otherwise forget to search (`docs/concepts/active-memory.md:13`, `extensions/active-memory/index.ts:1964`). If you want stricter keyword-to-file routing than the shipped recall path, current main already exposes `before_prompt_build` for that kind of plugin/hook implementation (`src/plugins/hook-before-agent-start.types.ts:21`). This is present in the released `v2026.4.22` build (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
