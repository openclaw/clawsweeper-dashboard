## Summary

Keep open. Current `main` still starts ACP startup identity reconciliation immediately, without any backend-readiness wait, and the same unavailable-backend error path still exists. The PR is stale against the refactor from `src/gateway/server-startup.ts` to `src/gateway/server-startup-post-attach.ts`, but I did not find an equivalent fix already landed on `main`.
