## Repair Work Prompt

Fix the FAL image-generation provider/model dispatch reported in https://github.com/openclaw/openclaw/issues/77444. Current main parses `model: "fal-ai/flux/dev"` as provider `fal-ai`, so `image_generate` fails with `No image-generation provider registered for fal-ai` before reaching the bundled `fal` provider or FAL auth. Keep the change narrow: make the FAL model shown to users resolve through provider `fal` either by normalizing media model selection, by presenting fully-qualified FAL refs in `image_generate action=list`, or by another small compatible fix consistent with existing provider/model contracts. Add regression coverage for the failing path and keep FAL auth provider-owned; do not add a broad new config option or change unrelated provider auth semantics. Add a user-facing CHANGELOG.md fix entry. If the reported auth-profile `invalid_type` needs a separate fix, only address it if the exact invalid shape is reproduced from current code; otherwise leave a targeted doc/test note rather than guessing.
