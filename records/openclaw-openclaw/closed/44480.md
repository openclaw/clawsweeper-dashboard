## Summary

Current main already gives Claude Code repo guidance through the canonical `CLAUDE.md -> AGENTS.md` symlink. The PR’s standalone replacement is stale and unsafe as written: reviewers correctly flagged wrong product naming, non-canonical commands, a raw `vitest` invocation, and a direct conflict with the repo’s symlink policy.

## Best Possible Solution

Keep the shipped canonical `CLAUDE.md -> AGENTS.md` model. If a standalone Claude Code guide is desired later, first update the root policy, then generate a short audited file that preserves OpenClaw naming and the current repo command/test guidance.
