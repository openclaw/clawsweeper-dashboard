## Summary

Keep open: current main and v2026.5.12 still use strict whole-string equality for `isAnnounceSkip`, the final-line token case is source-reproducible, and https://github.com/openclaw/openclaw/pull/74136 is still the open closing fix candidate.

## What This Changes

Accept `ANNOUNCE_SKIP` when it appears as a standalone final line in multi-line sub-agent announce output so the summary is not posted to the parent session.

## Best Possible Solution

Review, validate, and land or supersede https://github.com/openclaw/openclaw/pull/74136 with the final-line `ANNOUNCE_SKIP` contract, focused regression coverage, and docs alignment, then close this issue after merge.

## Reproduction Assessment

Yes. The high-confidence source path is `isAnnounceSkip("DM summary block\nANNOUNCE_SKIP")`: current main trims the whole string and compares it to `ANNOUNCE_SKIP`, which evaluates false while the exact padded token evaluates true.

## Solution Assessment

Yes. The narrow maintainable fix is to broaden only `isAnnounceSkip` for a standalone final token line, keep `REPLY_SKIP` strict, and cover the helper plus announce flow with tests and docs.
