## Summary

Current main already fixes the reported auth-profile corruption path: locked auth-store writes reload from disk instead of replaying stale runtime state, runtime routing state now lives in `auth-state.json` rather than `auth-profiles.json`, auth-order management has first-class CLI commands, and auth resolution prefers configured profiles/order before falling back to raw stored profiles. The same implementation is already present in release `v2026.4.22`.

## Evidence
