## Summary

Keep open. Current `main` still rewrites GitHub Copilot Responses item ids without exempting `reasoning` items that carry `encrypted_content`, and the wrapper still applies that normalization on the replay path this PR targets. The linked issue discussion provides concrete field evidence for the exact provider-side failure, while current `main` only partially addresses model-switch replay sanitization in core and does not make this plugin-side guard obsolete.

## Evidence

- **Current main still rewrites encrypted reasoning ids:** `rewriteCopilotConnectionBoundResponseIds` rewrites any connection-bound `id` once `looksLikeConnectionBoundId(id)` passes. There is no guard for `item.type === "reasoning" && encrypted_content`, so encrypted reasoning items are still normalized on `main`.
