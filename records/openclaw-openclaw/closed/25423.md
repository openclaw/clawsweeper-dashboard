## Summary

Current main already mitigates this class of spoofed inline system-message injection. Telegram ingress now derives inbound text from the actual caption/location/audio-transcript path, and all user-controlled inbound text is normalized through `sanitizeInboundSystemTags()` before `BodyForAgent` reaches the agent, with regression coverage for `[System Message]` and `System:` spoofing. The shipped `v2026.4.22` release contains that behavior.

## Evidence
