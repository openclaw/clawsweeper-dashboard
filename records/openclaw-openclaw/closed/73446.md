## Summary

Keep open. Current live docs still drop the page path when switching from English to Simplified Chinese, current main has no durable path-preserving locale switcher or proven docs asset hook, and the closed workaround PRs were rejected because standalone docs-root JavaScript is not served or synchronized with the generated locale source of truth.

## What This Changes

The issue asks for the Mintlify docs language switcher to preserve the current page path when changing locales.

## Best Possible Solution

Keep this as the canonical tracker until maintainers either get Mintlify to preserve locale-switch paths upstream or wire a supported publish/config hook derived from GENERATED_LOCALES and verify it in a Mintlify preview or live deployment.

## Reproduction Assessment

Yes. A live headless Chrome replay on May 15, 2026 reproduced the path loss from /install/docker to /zh-CN while /zh-CN/install/docker returned 200.

## Solution Assessment

Unclear. The requested behavior is correct, but the prior standalone docs-root JavaScript approach is not the best proven fix unless OpenClaw wires and verifies a supported Mintlify asset hook; an upstream Mintlify fix or publish-pipeline integration is safer.
