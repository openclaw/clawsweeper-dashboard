## Summary

This issue should stay open: current main still has the reported read-only registration/quarantine path, and the linked fix PR is open, clean, and unmerged rather than implemented on main.

## What This Changes

The issue reports a v2026.6.10 regression where read-only plugin registration can cause the configured lossless-claw context engine to be quarantined and replaced by `legacy`.

## Best Possible Solution

Keep this issue as the canonical bug and land https://github.com/openclaw/openclaw/pull/96357 or an equivalent lifecycle split that prevents read-only discovery factories from being constructed or quarantined while preserving runtime quarantine for real failures.

## Reproduction Assessment

Yes at source level: current main lets read-only discovery modes register context-engine factories, and resolution invokes/quarantines the selected factory before falling back. No live macOS gateway reproduction was run in this read-only review.

## Solution Assessment

Yes: separating read-only discovery metadata from runtime factory construction is the narrowest maintainable direction. This issue should remain open until the linked PR or equivalent fix is merged.
