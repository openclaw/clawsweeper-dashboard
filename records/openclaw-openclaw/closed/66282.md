## Summary

Current main already contains the cron/session fixes implicated by this report, the reporter confirmed the problem disappeared after restarting into the newer gateway code, and the public update docs already instruct operators to restart the background gateway after upgrades.

## Evidence

- **Reporter confirmed resolution after restart:** The issue author's April 15 follow-up says `openclaw gateway restart` immediately fixed the previously failing isolated cron jobs, including a heavier MiniMax run that then completed in 128s. The reported root cause was an older gateway process still running in memory after upgrade.
