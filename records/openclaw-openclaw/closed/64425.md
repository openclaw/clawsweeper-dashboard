## Summary

Current `main` no longer assumes the OpenAI Codex OAuth localhost callback can complete on a VPS. It now has a remote/VPS path that prints the auth URL for the local browser, prompts for the pasted redirect URL, adds a localhost-timeout manual fallback, and also exposes a device-code pairing option for OpenAI Codex. This appears fixed on `main` by commit `88c91675e277505fb64c13a00988757e092a38c9`, but I could not tie that fix to a tagged release yet.
