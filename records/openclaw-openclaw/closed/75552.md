## Summary

Keep this PR open, but not merge-ready. The PR is protected by maintainer/MEMBER handling, and the inspected head still has two source-level P2 blockers: a raw transcript append path that bypasses the parentId chain, and a new tree-transcript visitor that still materializes current-format transcripts through SessionManager.open(...).getBranch().

## What This Changes

The PR adds bounded Gateway transcript read/count helpers and routes session detail, HTTP/SSE history, artifact listing, send/subscribe sequencing, compaction-related reads, and injected assistant transcript writes through those helpers, with tests and a changelog entry.

## Best Possible Solution

Keep the bounded-read direction, but land it only after the transcript traversal and injected-write design preserve SessionManager parent chains while avoiding full-file materialization for current tree-format transcripts. Any legacy append-only fallback should be narrowly justified, isolated, and tested without weakening the parentId guardrail for normal transcripts.

## Reproduction Assessment

Yes. The PR-level regression is visible in the added oversized raw-append test that expects no parentId, and the remaining OOM-prone path is reproducible by following visitSessionMessages() on a parent-linked large transcript into SessionManager.open(filePath).getBranch().

## Solution Assessment

No. The requested performance fix is appropriate, but this implementation is not the best mergeable solution because it preserves a full SessionManager load for tree transcripts and adds a raw append path instead of a bounded, parent-chain-preserving SessionManager-compatible seam.
