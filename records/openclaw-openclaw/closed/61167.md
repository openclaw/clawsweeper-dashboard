- **[P1] Validate UID and GID before root chown:** `scripts/docker/setup.sh:505-506`
  - body: `OPENCLAW_UID` and `OPENCLAW_GID` come from the environment and are interpolated into the root container's `sh -c` chown command.   - confidence: 0.91
- **[P1] Make host-root deletion opt in:** `docker-teardown-web-whatsapp.sh:66`
  - body: The teardown helper always runs `sudo rm -rf "$HOST_ROOT"` whenever the default host root exists. That root contains the persisted config and workspace for this helper, so a normal teardown can delete user state without a separate destructive flag or prompt.
  - confidence: 0.92
