The issue asks OpenClaw to prevent context-engine deferred compaction debt from becoming stranded after bootstrap/reconcile or hot-cache maintenance paths.

## Best Possible Solution

Keep the issue open and land a narrow OpenClaw host fix that lets bootstrap/reconcile-created deferred debt schedule or run an allowed background maintenance follow-up; handle Lossless-Claw cache/dedup policy as separate plugin-owned work if it remains reproducible.

## Repair Work Prompt

Fix the OpenClaw host scheduler gap from https://github.com/openclaw/openclaw/issues/67716. Current `runContextEngineMaintenance()` only defers background context-engine maintenance when `reason === "turn"`, while bootstrap/reconcile maintenance is invoked with `reason: "bootstrap"` and then runs foreground, so engines that require `runtimeContext.allowDeferredCompactionExecution` cannot consume deferred compaction debt created during bootstrap/reconcile. Implement the narrow host-side fix so background-capable context engines get a safe deferred/background follow-up for bootstrap-created debt, or otherwise run that maintenance with `executionMode: "background"` only when the session-lane/rewrite semantics remain safe. Preserve existing turn deferral/coalescing behavior, do not add config flags, do not change Lossless-Claw internals or hot-cache policy in this PR, and add focused regression tests for bootstrap/reconcile maintenance plus existing turn behavior. Changelog is release-owned; put any user-visible release-note context in the PR body only.
