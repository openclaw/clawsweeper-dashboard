## Summary

Keep this PR open: the Codex watchdog fix is aimed at a real source-reproducible bug, but the patch still moves the completion-idle timestamp on raw assistant progress, so it does not prevent the same 60-second-after-status timeout; the PR also lacks after-fix real behavior proof.

## What This Changes

The PR changes Codex app-server watchdog bookkeeping for `rawResponseItem/completed`, and also includes unrelated Ollama ambient-discovery warning and Windows plugin-skill symlink fixes.

## Best Possible Solution

Separate diagnostic progress from completion-idle timing so raw assistant progress can update logs without moving the post-tool-response deadline, add focused regression coverage, and require after-fix runtime proof before merge.

## Reproduction Assessment

No live reproduction was run in this read-only sweep, but the current-main source path is high-confidence: a turn-scoped response arms the watchdog, raw progress updates the shared activity timestamp, and silence then reaches the completion timeout path.

## Solution Assessment

No. The proposed skip only avoids immediate rescheduling; because the same helper still updates `turnCompletionLastActivityAt`, the already-armed timer can re-arm from the raw progress timestamp and preserve the false-positive timeout shape.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Keep raw progress from moving the idle timestamp:** `extensions/codex/src/app-server/run-attempt.ts:876-878`
  - body: This still calls `touchTurnCompletionActivity` for `rawResponseItem/completed`, and that helper updates `turnCompletionLastActivityAt` before the new `skipCompletionWatch` option is checked. If a tool response arms the timer at T0 and raw progress arrives at T30, the existing timer fires at T60, sees only 30s idle, and re-arms until T90, which is still the 60s-after-status timeout the PR is meant to avoid. Keep a separate diagnostic timestamp/reason or avoid changing the completion-idle timestamp for skipped notifications.
  - confidence: 0.92
