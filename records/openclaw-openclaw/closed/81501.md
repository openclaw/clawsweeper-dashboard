## Summary

Current main already fixes the reported beta install-time scanner regression by limiting plugin code scanning to the plugin-owned runtime graph and leaving installed dependency trees to manifest denylist checks, so LanceDB/OpenAI-style dependency code no longer blocks normal installs.

## What This Changes

The issue asks OpenClaw to stop blocking legitimate third-party memory/vector plugin installs when runtime dependencies such as LanceDB, Apache Arrow, and OpenAI contain scanner-triggering implementation code.

## Best Possible Solution

Keep the current main scanner boundary: scan plugin-owned runtime code, preserve manifest denylist checks for installed dependencies, and ship the main-only fix in the next beta/stable release.

## Reproduction Assessment

No current-main failure remains to reproduce. The beta regression is source-backed by the removed installed-tree scan loop and by tests that were changed from blocking dependency runtime code to allowing it.

## Solution Assessment

Yes. Scoping install-time code scanning to the plugin-owned runtime graph is the narrow maintainable fix for dependency false positives while keeping direct plugin code and manifest denylist checks active.
