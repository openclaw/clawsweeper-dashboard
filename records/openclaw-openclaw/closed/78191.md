## Summary

Current main already removes the reported failure path: Claude migration writes MCP servers to `mcp.servers`, and unresolved `plugins.entries` are warning-only with install hints instead of gateway-startup blockers.

## What This Changes

The issue reports that `openclaw migrate apply` can import unresolved plugin entries and leave the Gateway crash-looping on invalid plugin config.

## Best Possible Solution

Keep the current main behavior: migrate Claude MCP data into `mcp.servers` and leave unresolved plugin-entry config as non-fatal warnings with install guidance.

## Reproduction Assessment

No: the reported crash path is not present on current main by source inspection and regression coverage. Claude migration no longer writes the reported MCP-style entries to `plugins.entries`, and missing plugin entries validate as warnings.

## Solution Assessment

Yes: the current implementation is the narrow maintainable fix because it preserves imported MCP configuration without pretending uninstalled plugins are active, while config validation still gives install guidance for real plugin entries.
