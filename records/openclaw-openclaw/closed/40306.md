## Summary

Current `main` already contains a targeted macOS LaunchAgent restart fix for the reported automation context: in-process restarts are handed off to a detached helper instead of self-kickstarting the current launchd-managed process, and restart recovery now re-bootstraps unloaded LaunchAgents. The same implementation is present in the shipped `v2026.4.22` release.

## Evidence

- **In-process LaunchAgent restart now uses detached handoff:** `restartLaunchAgent()` detects when the caller is the current launchd-managed gateway process and returns a scheduled restart after spawning a detached helper, rather than running `launchctl kickstart -k` in-process.
  - file: [src/daemon/launchd.ts:638](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/daemon/launchd.ts#L638)
  - command: `nl -ba src/daemon/launchd.ts | sed -n '625,685p'`
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Detached helper performs the restart after caller exit:** The helper script waits for the caller PID to exit, re-enables the service, then tries `launchctl kickstart -k` and falls back to `launchctl bootstrap` plus retry. That directly matches the reported 'manual Terminal works, automation kills itself' distinction.
  - file: [src/daemon/launchd-restart-handoff.ts:93](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/daemon/launchd-restart-handoff.ts#L93)
