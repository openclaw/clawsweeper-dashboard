## Summary

Keep open: current main still turns restart continuations into internal-provenance agent turns while leaving the raw continuation text in normal message fields, and the WebChat/Control UI projection has no branch that suppresses or visually separates `internal_system` restart-sentinel input.

## What This Changes

The issue asks Control UI/WebChat to treat `continuationMessage` and runtime continuation prompts as agent-only or clearly system/runtime content instead of ordinary visible user chat.

## Best Possible Solution

Define one shared transcript/display treatment for runtime-internal input so the agent still receives restart continuations while WebChat either suppresses them or renders them in a clearly labeled runtime/system block.

## Reproduction Assessment

Yes, at source level. A restart sentinel `continuationMessage` becomes an `agentTurn`, is copied into normal body fields with `internal_system` provenance, and the display/UI path has no restart-continuation or `internal_system` hiding/separation branch.

## Solution Assessment

Unclear until maintainers choose the UX/API boundary. A narrow fix could use existing `internal_system` provenance at projection time, but the durable path should align with the broader system-block and event-audience work.
