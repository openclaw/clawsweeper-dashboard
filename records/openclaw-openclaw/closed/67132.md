## Summary

Current `main` suppresses `commentary`-phase assistant text before user-visible channel delivery, prefers `final_answer` text for visible replies, and has regression coverage for both streaming updates and final block replies. The changelog shows this behavior shipped in `v2026.4.5`, so the reported iMessage narration leak is already implemented/fixed.

## Evidence
