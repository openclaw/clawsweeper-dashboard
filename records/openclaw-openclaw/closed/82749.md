## What This Changes

Resumed GitHub Copilot Responses sessions can replay historical overlong tool IDs and fail provider schema validation before generation.

## Best Possible Solution

Land a reviewed fix that normalizes GitHub Copilot Responses replay item IDs and paired tool-result IDs before dispatch, with focused regression coverage at the request-builder seam.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to build a `github-copilot` `openai-responses` payload from a replayed same-model assistant `toolCall` whose ID is `call_*|<overlong fc_...>` plus a paired `toolResult`; current main preserves the raw function-call item ID into outbound `input[].id`.

## Solution Assessment

Yes. The maintainable fix belongs in the Responses replay normalization/request-builder path so Copilot receives provider-valid IDs while preserving tool-result pairing; transcript reset or provider pinning should remain temporary workarounds.
