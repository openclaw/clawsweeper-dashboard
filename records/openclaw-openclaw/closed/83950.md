## Label Justifications

- `P1`: Default Telegram polling can silently lose its configured stall recovery path for real channel workflows.
- `impact:message-loss`: A silent isolated polling worker can leave inbound Telegram updates undelivered without the configured watchdog firing.

## Summary

Keep open: current main still ignores the configured Telegram polling stall threshold in the default isolated polling path, and https://github.com/openclaw/openclaw/pull/84013 is an open linked fix candidate that should be reviewed or closed before this issue is resolved.

## What This Changes

The issue asks OpenClaw to honor the existing Telegram `pollingStallThresholdMs` watchdog setting in the default isolated polling ingress path.

## Best Possible Solution

Honor the existing documented `pollingStallThresholdMs` in isolated ingress by feeding worker liveness into the same restart semantics, preferably by reviewing and landing the linked fix PR once its proof and risk questions are satisfied.

## Reproduction Assessment

Yes, at source level: current main defaults Telegram polling into isolated ingress and `#stallThresholdMs` is only passed to `detectStall` in `#runPollingCycle`, not `#runIsolatedIngressCycle`. I did not run a live Telegram wedge.

## Solution Assessment

Yes. Wiring the existing documented setting into the default isolated path is the narrowest maintainable fix; exposing a public isolated-ingress opt-out would be secondary and is not required to fix the broken config contract.
