## What This Changes

The issue requests outbound proxy support for Telegram and provider HTTP traffic when standard proxy environment variables or proxy configuration are set, especially for Docker deployments behind DPI.

## Best Possible Solution

Keep the current main proxy-aware Telegram transport and provider fetch behavior, and make sure the next release notes make the correct HTTP-forward-proxy versus SOCKS5 Telegram configuration path clear for Docker users behind DPI.
