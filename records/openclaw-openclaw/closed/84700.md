## Summary

## What This Changes

The issue reports Safari Control UI rendering the bundled alien-style assistant icon instead of the configured star emoji in message headers.

## Best Possible Solution

Keep the shipped v2026.5.19/current-main identity rendering path and ask for a fresh report only if the star emoji still fails on v2026.5.19 or newer with current Control UI bootstrap data.

## Reproduction Assessment

No; I did not recreate the Safari 2026.5.6 failure live. Source inspection of current main and v2026.5.19 shows configured short emoji values now flow through as text avatars rather than falling back to the bundled icon.

## Solution Assessment

Yes; the current implementation is the narrow maintainable solution because it carries identity emoji through the existing gateway bootstrap and Control UI avatar renderer without adding a new config surface.
