## Summary

Close: current main and the v2026.5.18 release already preserve Signal quote context through queued followups via `currentInboundContext`; the raw queued `prompt` remaining as the body is expected because runtime context is prepended later.

## What This Changes

Preserve Signal reply-quote metadata when queued inbound messages are later drained through followup, steer, or collect paths.

## Best Possible Solution

Keep the structured `currentInboundContext` path rather than duplicating untrusted quote blocks into the raw queued prompt string.

## Reproduction Assessment

No; I did not find a failing current-main path. Source and tests show Signal `ReplyTo*` metadata is carried on queued runs as `currentInboundContext` and prepended to the model input.

## Solution Assessment

Yes; the existing structured runtime-context path is the better fix because it keeps reply metadata model-visible without polluting the persisted transcript or duplicating untrusted blocks into raw queue prompts.
