## What This Changes

When `messages.queue.mode` is `steer`, inbound messages during an active Discord/Pi run should reach the active runtime at the documented steering boundary instead of silently falling back to followup.

## Best Possible Solution

Keep #48003 as the canonical open tracker, review or replace #73365/#75226 for the code fix, and carry this issue’s Discord diagnostics into that thread.

## Reproduction Assessment

Yes. Source inspection on current main shows same-session Discord work is serialized, steering is attempted only while `isStreaming` is true, non-streaming embedded handles reject steering, and active `steer` falls through to followup.
