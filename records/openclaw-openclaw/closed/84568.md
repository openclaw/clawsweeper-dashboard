- `impact:message-loss`: The affected behavior is outbound Discord reply delivery failing with `Outbound not configured for channel: discord`.

## Summary

Keep open: current main and v2026.5.19 still contain the active-registry fallback on the outbound adapter path, and the linked open PR is the right fix path rather than closing the issue before merge.

## What This Changes

The issue asks to stop resolving Discord outbound adapters through an unstable active plugin registry fallback after the channel registry has been pinned.

## Best Possible Solution

Review and land the linked PR, or an equivalent lifecycle fix, so outbound lookup uses the stable pinned channel surface when a channel is already pinned while handling truly dynamic channels explicitly.

## Reproduction Assessment

Yes, at source level: current main and v2026.5.19 have the active-registry fallback on the outbound adapter path that produces the reported error when no handler is resolved. I did not run the live Discord setup in this read-only review.

## Solution Assessment

Yes, the direction is right, but the safest implementation is the linked refined fix: do not consult the active registry when the pinned registry already contains the channel, while preserving explicit support for truly dynamic channels.
