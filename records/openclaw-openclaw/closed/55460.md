## Summary

Current main no longer exposes Slack-native `blocks` on the shared `message` tool, so the reported root cause for `message(action=send)` does not apply anymore. The released `v2026.4.22` tree already uses shared `presentation` instead of provider-native Slack `blocks` on that tool surface.

## Evidence

- **Shared message tool schema has `presentation`, not top-level `blocks`:** `src/agents/tools/message-tool.ts` defines optional `presentation` for rich sends and does not add a Slack-native `blocks` field to the shared send schema.
  - file: [src/agents/tools/message-tool.ts:78](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/agents/tools/message-tool.ts#L78)

- **Slack plugin explicitly does not contribute a native message-tool schema:** `extensions/slack/src/channel.test.ts` asserts `describeMessageTool(...).schema` is `undefined`, so Slack is not exposing a plugin-owned `blocks` parameter on the shared tool anymore.
  - file: [extensions/slack/src/channel.test.ts:227](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/extensions/slack/src/channel.test.ts#L227)
