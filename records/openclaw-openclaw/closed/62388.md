## Summary

Keep open. Current `main` still has the Comfy config-path/schema mismatch the report describes: runtime reads only `models.providers.comfy`, the core `models.providers.*` validator only accepts generic model-provider fields, the bundled Comfy manifest and public docs still advertise Comfy-specific keys, and the referenced fix is still only an open PR rather than code on `main`.

## Evidence

- Users following `docs/providers/comfy.md` can still hit config-validation failures before runtime, while users configured through plugin-owned paths can still miss the runtime read path.
