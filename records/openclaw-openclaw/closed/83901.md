## What This Changes

The issue asks for `openclaw acp --no-prefix-cwd` to actually suppress the working-directory prompt prefix by using Commander’s `prefixCwd` option key.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/84039; once a fix merges, close this issue as implemented by that PR.

## Reproduction Assessment

Yes. Source inspection on current main plus Commander 14.0.3’s option contract shows `--no-prefix-cwd` populates `prefixCwd`, while the CLI reads `noPrefixCwd` and therefore still forwards `prefixCwd: true`.

## Solution Assessment

Yes. The best fix is to remove the explicit false default, read Commander’s positive `prefixCwd` attribute, and add focused CLI forwarding coverage; the open linked PR follows that shape but has not merged.
