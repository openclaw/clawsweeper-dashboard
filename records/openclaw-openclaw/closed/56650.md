Full review comments:

- **[P1] Remove the argv token path:** `src/cli/models-cli.ts:350-352`
  - body: Adding `--token <value>` makes provider credentials part of the command line, where they can land in shell history, command echoing, CI logs, and process listings.   - confidence: 0.91
- **[P1] Attach Copilot headers through the runtime request path:** `extensions/github-copilot/index.ts:163-167`
  - body: The PAT branch says it goes to the enterprise API with editor-identification headers, but it returns only `apiKey`, `baseUrl`, and `expiresAt`, while `COPILOT_EDITOR_HEADERS` is imported but unused. Rebase onto current main and preserve/use the provider request-header path so PAT-backed runtime calls actually send the required Copilot headers.
  - confidence: 0.84
- **[P2] Do not treat classic PATs as supported without proof:** `src/agents/github-copilot-token.ts:17-19`
  - body: `isGitHubPAT` routes both `github_pat_` and `ghp_` tokens down the raw enterprise path, but current GitHub Copilot CLI docs mark classic `ghp_` PATs as unsupported. Limit this to documented token types or include live proof that OpenClaw's direct endpoint path supports classic PATs.
  - confidence: 0.82
