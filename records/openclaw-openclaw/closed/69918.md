## Summary

Current `main` already contains a targeted fix for this report: dreaming narrative runs now always clean up their temporary session and then scrub stale `dreaming-narrative-*` entries from `agents/<id>/sessions/sessions.json`, with regression coverage that matches the reported UI leakage. The fix landed after `v2026.4.23`, so it appears implemented on `main` but not yet in a tagged release.

## Evidence

- **Narrative runs now delete their temp session and scrub stale dreaming artifacts:** `generateAndAppendDreamNarrative` always calls `subagent.deleteSession({ sessionKey })` in `finally`, then calls `scrubDreamingNarrativeArtifacts`, which prunes stale dreaming session-store keys and archives orphan dreaming transcripts.
  - file: [extensions/memory-core/src/dreaming-narrative.ts:716](https://github.com/openclaw/openclaw/blob/712f7b218c092c9d0fae1f583f7c06e617920115/extensions/memory-core/src/dreaming-narrative.ts#L716)

  - sha: [712f7b218c09](https://github.com/openclaw/openclaw/commit/712f7b218c092c9d0fae1f583f7c06e617920115)
- **Regression test matches the reported leak:** The test seeds `agent:main:dreaming-narrative-light-1` into `sessions.json`, runs narrative generation/cleanup, and asserts the dreaming entry is removed while unrelated sessions remain.
  - file: [extensions/memory-core/src/dreaming-narrative.test.ts:792](https://github.com/openclaw/openclaw/blob/712f7b218c092c9d0fae1f583f7c06e617920115/extensions/memory-core/src/dreaming-narrative.test.ts#L792)

- **Session listing does not specially filter dreaming keys:** `listSessionsFromStore` filters cron-run sessions, but not dreaming narrative sessions, so cleaning stale dreaming entries out of `sessions.json` is the relevant fix for the Control UI switcher clutter described in the issue.
