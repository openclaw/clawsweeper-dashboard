## Summary

Current main already covers the reported Windows node-exec behavior: remote node exec no longer inherits the gateway cwd when no workdir was requested, `approvals get --node` is explicitly modeled as the merge of node-host approvals plus gateway `tools.exec` policy, wildcard-agent approvals are merged in code, and Windows `cmd.exe /c` wrapper runs are intentionally approval-gated in allowlist mode with explicit guidance.

## Evidence

- **Current CLI/docs explain why `approvals get --node` can still show a stricter effective policy:** The docs now state that `openclaw approvals get --node` shows requested policy, host approvals-file policy, and the effective result, and that `--node` combines the node host approvals file with gateway `tools.exec` policy because both apply at runtime.
