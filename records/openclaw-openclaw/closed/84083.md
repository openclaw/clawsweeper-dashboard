## Label Justifications

- `P1`: The PR targets an agent-turn hang where plugin-owned compaction can leave interactive work unresponsive until a broad run timeout.
- `merge-risk: 🚨 session-state`: As written, a timed-out plugin compaction can continue in the background and mutate session artifacts after the host moves on.
- `merge-risk: 🚨 availability`: The patch changes timeout and abort behavior in core agent compaction recovery paths that gate whether stalled turns recover cleanly.

## Summary

Keep open: this is a plausible fix for a real agent availability bug, but the patch currently documents timeout-driven cancellation without actually aborting the signal passed to plugin compaction, and the external PR still lacks real behavior proof.

## What This Changes

The PR adds a safety-timeout wrapper for plugin-owned `ContextEngine.compact()` calls, wires it into queued and run-loop compaction paths, extends compact params with optional `abortSignal`, and adds focused tests.

## Best Possible Solution

Keep the host timeout wrapper, but pass a composite compact abort signal that fires on both run abort and compaction timeout, then require redacted real proof with an installed ownsCompaction plugin before merge.

## Reproduction Assessment

Yes for source-level reproduction: current main awaits an ownsCompaction engine’s never-resolving `compact()` directly in queued, timeout-recovery, and overflow-recovery paths. I did not run a live lossless-claw stall in this read-only review.

## Solution Assessment

No for this PR as-is. Reusing the existing timeout wrapper is the right direction, but the helper must make the plugin-visible abort signal fire on timeout before the documented cancellation contract is true.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Abort plugin compact work when the safety timeout fires:** `src/agents/pi-embedded-runner/compaction-safety-timeout.ts:124-127`
  - body: The new helper only threads the caller’s `abortSignal` into `contextEngine.compact()`. When the safety timeout wins, `compactWithSafetyTimeout` rejects the host await, but the plugin-visible signal is not aborted, so a cooperative engine will keep running and can still write session state after OpenClaw has moved into failure handling; this also contradicts the new `ContextEngine` docs. Please pass a signal that aborts on both caller abort and timeout, and add a test that the compact params signal is aborted by timeout.
