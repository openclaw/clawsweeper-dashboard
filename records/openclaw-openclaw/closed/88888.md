The issue asks OpenClaw to make Feishu final-answer delivery reliable by distinguishing interactive API fallback artifacts from UI failure, tracking final delivery, and retrying or surfacing failed final sends.

## Best Possible Solution

Keep this open until the Feishu plugin has a live-proofed final-delivery contract that retries transient final-send failures, preserves or recovers the actual final answer when safe, and treats CardKit readback fallback text as diagnostic-only.
