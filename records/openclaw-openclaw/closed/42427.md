## Summary

The request targets the external WeCom channel plugin, not OpenClaw core. Current main already treats WeCom as an external/community plugin and the SDK/docs already provide multi-account channel seams, so this should be tracked in the WeCom plugin repository instead of openclaw/openclaw.

## Risks / Open Questions

- The external WeCom plugin repository could choose not to adopt the existing multi-account SDK seams, so the user-facing feature is still not available today.
- If maintainers intentionally want to use the core tracker for external official plugin requests, closing here could reduce visibility even though the implementation belongs upstream.

## Close Comment
