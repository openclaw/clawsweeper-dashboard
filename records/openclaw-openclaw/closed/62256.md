## Summary

Current `main` already contains a targeted fix for this exact exec/update race, regression coverage for the late-stdout/late-stderr cases, and a changelog entry showing it shipped in `v2026.4.10`.

## Evidence

- **Changelog entry names this exact crash and shipped fix:** The `2026.4.10` changelog explicitly says OpenClaw now prevents the gateway crash `"Agent listener invoked outside active run"` when a subagent exec tool emits stdout/stderr after the agent run has ended or been aborted.
  - file: [CHANGELOG.md:1011](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/CHANGELOG.md#L1011)
  - command: `nl -ba CHANGELOG.md | sed -n '900,1055p'`
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Runtime now suppresses late exec updates after run end:** `runExecProcess` tracks `updatesDisabled` and `emitUpdate()` returns early when the session is backgrounded, exited, or updates were disabled, with an inline comment calling out the `Agent listener invoked outside active run` crash and issue `#62520`.
  - file: [src/agents/bash-tools.exec-runtime.ts:561](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/agents/bash-tools.exec-runtime.ts#L561)
  - command: `nl -ba src/agents/bash-tools.exec-runtime.ts | sed -n '520,660p'`
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Updates are disabled on exit, error, and explicit abort:** The exec handle exposes `disableUpdates()`, sets `updatesDisabled = true` before exit/error finalization, and the tool abort path calls `run.disableUpdates()` before killing or returning, so late stdout/stderr can no longer enqueue stale `onUpdate` work after the agent run ends.
  - file: [src/agents/bash-tools.exec-runtime.ts:164](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/agents/bash-tools.exec-runtime.ts#L164)

  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Abort path explicitly documents this regression:** The foreground exec wrapper documents that `run.disableUpdates()` is intentionally called on abort so late process output cannot push rejected promises into pi-agent-core update handling after the agent run has ended (`#62520`).
  - file: [src/agents/bash-tools.exec.ts:1715](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/agents/bash-tools.exec.ts#L1715)

  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Regression tests cover the reported failure shape:** Tests assert that `onUpdate` is not invoked after backgrounding, after foreground process exit, or after abort, which directly covers the late-output race reported in this issue and echoed by commenters/cross-references.
  - file: [src/agents/bash-tools.test.ts:896](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/agents/bash-tools.test.ts#L896)
