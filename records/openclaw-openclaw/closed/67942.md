## Summary

Current `main` already filters the reported REM role-tag and boilerplate themes out of both concept-tag extraction and REM reflection ranking, with regression tests covering `assistant`/`system`/`user` tags and heartbeat/cron transcript noise. The issue was valid on the April 2026 releases, but the current `main` code now implements the requested cleanup well enough to close.

## Evidence

- **REM reflections now skip chat scaffolding tags:** `buildRemReflections()` blacklists `assistant`, `user`, `system`, `subagent`, and `the` before counting recurring themes, so those tags no longer become REM reflections.
  - file: [extensions/memory-core/src/dreaming-phases.ts:194](https://github.com/openclaw/openclaw/blob/306c0f73bff1338482dd6533f61a86905bf541b1/extensions/memory-core/src/dreaming-phases.ts#L194)
  - sha: [306c0f73bff1](https://github.com/openclaw/openclaw/commit/306c0f73bff1338482dd6533f61a86905bf541b1)
- **Concept-tag derivation now strips the same role/system words earlier:** Shared stopwords now include `assistant`, `subagent`, `system`, and `user`, and English stopwords now include `the`, `you`, and `your`, reducing chat-structure tokens before REM ranking even starts.
  - file: [extensions/memory-core/src/concept-vocabulary.ts:15](https://github.com/openclaw/openclaw/blob/306c0f73bff1338482dd6533f61a86905bf541b1/extensions/memory-core/src/concept-vocabulary.ts#L15)
  - sha: [306c0f73bff1](https://github.com/openclaw/openclaw/commit/306c0f73bff1338482dd6533f61a86905bf541b1)
- **Regression tests cover the exact reported failure mode:** Tests now assert that `previewRemDreaming()` keeps `ollama`/`provider` but does not surface `assistant` or `the`, and that concept-tag derivation drops `assistant`/`system`/`the`/`you`/`your` from a chat-style snippet.
  - file: [extensions/memory-core/src/dreaming-phases.test.ts:1342](https://github.com/openclaw/openclaw/blob/306c0f73bff1338482dd6533f61a86905bf541b1/extensions/memory-core/src/dreaming-phases.test.ts#L1342)
  - sha: [306c0f73bff1](https://github.com/openclaw/openclaw/commit/306c0f73bff1338482dd6533f61a86905bf541b1)
- **Session ingestion now drops heartbeat and cron boilerplate from the corpus:** Session transcript ingestion classifies cron-run transcripts and skips them, and regression coverage verifies that `Read HEARTBEAT.md`, `HEARTBEAT_OK`, and cron sync chatter do not enter the dreaming session corpus while real content still does.
  - file: [extensions/memory-core/src/dreaming-phases.ts:711](https://github.com/openclaw/openclaw/blob/306c0f73bff1338482dd6533f61a86905bf541b1/extensions/memory-core/src/dreaming-phases.ts#L711)
  - sha: [306c0f73bff1](https://github.com/openclaw/openclaw/commit/306c0f73bff1338482dd6533f61a86905bf541b1)
- **The fix landed after the latest release:** `v2026.4.23` still counted every concept tag in `buildRemReflections()` with no blacklist, while `git diff v2026.4.23..HEAD -- extensions/memory-core/src/concept-vocabulary.ts extensions/memory-core/src/dreaming-phases.ts extensions/memory-core/src/dreaming-phases.test.ts` shows the new stopwords, blacklist, and regression tests added on top of that release.
  - command: `git diff --unified=20 v2026.4.23..fa976e5b93b3819f9e137fe08635764edee38290 -- extensions/memory-core/src/concept-vocabulary.ts extensions/memory-core/src/dreaming-phases.ts extensions/memory-core/src/dreaming-phases.test.ts`

## Risks / Open Questions

- This fixes the concrete role-tag and boilerplate cases reported here, but broader conversational-theme quality may still need follow-up if new non-structural high-frequency noise patterns appear in real deployments.
- The change is on current `main` but does not appear in the latest tagged release `v2026.4.23`, so users on released builds may still see the old behavior until a newer release ships.
