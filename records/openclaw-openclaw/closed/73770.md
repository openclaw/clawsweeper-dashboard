## Summary

Keep open. Current main still has single-agent `memory_search`, while this PR’s implementation has blocking correctness and privacy issues around cross-agent search plus a stale stacked diff that needs maintainer/security review before it can be merged.

## What This Changes

This PR changes memory-core so `memory_search` derives searchable agents from runtime identity, adds agent/source provenance to returned hits, updates memory-core tests/mocks, and carries the earlier memory CLI embedding-provider registration change.

## Best Possible Solution

Keep the PR active, but require an explicit opt-in cross-agent memory policy, partial-failure handling for scoped manager errors, a rebase after #73769, updates to the package-owned SDK/output contract, focused tests/docs for the accepted visibility model, and a user-facing changelog entry.

## Reproduction Assessment

Yes. A high-confidence logic reproduction is a coordinator-scoped search across multiple configured agents where one `getMemoryManagerContext` call returns an error; the PR returns an unavailable payload before querying healthy managers. The implicit policy behavior is also demonstrated by the added test that allows `chief` to search configured agents by default.

## Solution Assessment

No. The feature direction is plausible, but this implementation is not the safest maintainable solution because it grants implicit cross-agent memory access, aborts all scoped results on one manager error, and updates a stale SDK type surface instead of the current package-owned contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Skip failed scoped managers instead of aborting all results:** `extensions/memory-core/src/tools.ts:237-245`
  - body: When any scoped agent's memory manager returns an error, this guard returns an unavailable payload before the search loop runs. A cross-agent search will drop valid results from healthy agents even though the loop below already has a per-agent `continue` path.
  - confidence: 0.91
- **[P1] Require explicit opt-in for cross-agent memory search:** `extensions/memory-core/src/memory-permissions.ts:56-58`
  - body: Defaulting missing `chiefAgentIds` to `["chief"]` gives any agent named `chief` cross-agent `memory_search` access without configuration. Current docs describe per-agent memory storage, so broad memory visibility should fail closed until an operator explicitly enables it.
  - confidence: 0.88
- **[P2] Update the package-owned memory result contract:** `extensions/memory-core/src/tools.ts:39`
  - body: Current main exposes `MemorySearchResult` through `packages/memory-host-sdk`, but this branch adds the provenance aliases to the older `src/memory-host-sdk/host/types.ts` surface. After rebasing, the changed `agent_id` usage in `tools.ts` needs the package-owned type or a separate serialized output type updated instead.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff changes the memory isolation boundary and grants implicit cross-agent read access to an agent named `chief` unless configuration opts out.
