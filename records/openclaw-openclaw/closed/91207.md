## Summary

Current main already covers the reported existing-session provider-wildcard allowlist path: `/model` and persisted session overrides now honor `provider/*` visibility after hot reload. The fix is current-main-only and is not in latest release `v2026.6.1`.

## What This Changes

The issue reports that adding `provider-id/*` to `agents.defaults.models` under hybrid gateway reload does not let an existing session use that provider after `/model`.

## Best Possible Solution

Keep the current main implementation and let the next release carry the hot-reloaded provider-wildcard session behavior.

## Reproduction Assessment

No current-main failing reproduction remains from source inspection; current tests cover the success case for provider-wildcard session overrides when catalog rows are unavailable. The old-release report is plausible, but I did not run a live 2026.5.28 repro.

## Solution Assessment

Yes. The current solution keeps the existing visibility-policy boundary, preserves session overrides when `provider/*` allows them, and treats `agents.defaults.models` as hot-reloadable model config without adding a new config surface.
