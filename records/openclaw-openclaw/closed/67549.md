## Summary

Current `main` already hardens the `memory_search` singleton cache path that could leave `qmdManagerCache` undefined and includes a regression test for the exact corrupt-singleton startup scenario from this report. The fix appears to have landed via a later mainline commit/PR than the issue-linked PR, and it is not present in the latest tagged release yet.

## Evidence

- **Current main repairs invalid memory search singleton state:** `getMemorySearchManagerCacheStore()` now validates that both `qmdManagerCache` and `pendingQmdManagerCreates` are real `Map` instances and replaces the global singleton with a repaired store when the shape is stale or invalid, preventing the `.get()` crash path described in the issue.
