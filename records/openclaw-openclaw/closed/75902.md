## Summary

Current main addresses the reported bundled-runtime resolver failure: the legacy `plugin-runtime-deps` cache path is no longer part of normal bundled plugin loading, canonical package `dist` imports resolve through root runtime dependencies such as `json5` and `tslog`, and the relevant post-release fix is on main but not yet in a tagged release.

## What This Changes

The issue asks for bundled plugin runtime loading to resolve bare package imports such as `json5` and `tslog` so Telegram does not crash-loop during startup.

## Best Possible Solution

Keep the current main implementation and include it in the next release so packaged Telegram startup no longer depends on the legacy generated runtime dependency cache.

## Reproduction Assessment

Yes for the affected release: the report includes concrete 2026.4.29 startup logs showing bare `json5` and `tslog` imports failing from the legacy `plugin-runtime-deps` root. I did not run a live Telegram startup in this read-only review, but the current-main code path no longer matches that failing layout.

## Solution Assessment

Yes. Resolving bundled runtime imports through the package-root dependency graph and removing the legacy generated cache path is narrower and safer than hardcoding package subpaths for `json5`, `tslog`, or each future dependency.
