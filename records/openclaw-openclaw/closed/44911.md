## Summary

The request is for monorepo promotion of an already-external channel plugin. Current main explicitly positions optional channel integrations like this as external plugins distributed through ClawHub/npm, with a documented channel-plugin SDK and optional setup surfaces, so this belongs in the community plugin path rather than OpenClaw core.

## Evidence

- **Vision scope prefers external plugins over core additions:** `VISION.md` says optional capability should usually ship as plugins, the preferred path is npm/local extension loading, and the bar for adding optional plugins to core is intentionally high.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/40be5ad58187fd20b8defe5fc30e649190c131a8/VISION.md#L54)

- **Community plugin docs make ClawHub/npm the canonical path:** `docs/plugins/community.md` describes community plugins as third-party packages for channels/tools/providers, says ClawHub is the canonical discovery surface, and sets the quality bar around publishing on ClawHub or npm rather than merging into core.
  - file: [docs/plugins/community.md:9](https://github.com/openclaw/openclaw/blob/40be5ad58187fd20b8defe5fc30e649190c131a8/docs/plugins/community.md#L9)
  - sha: [40be5ad58187](https://github.com/openclaw/openclaw/commit/40be5ad58187fd20b8defe5fc30e649190c131a8)

  - file: [docs/tools/plugin.md:11](https://github.com/openclaw/openclaw/blob/40be5ad58187fd20b8defe5fc30e649190c131a8/docs/tools/plugin.md#L11)
