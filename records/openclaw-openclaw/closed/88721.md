Keep this PR open for review: the typed action direction matches the repository policy, but the current head still has source-proven channel fallback bugs, a relaxed agent schema that can silently drop select options, and only test/mock validation for visible transport behavior.

## Best Possible Solution

Keep the typed action design, but require one-of action/value validation for select options, preserve fallback labels whenever a channel drops typed controls, and add redacted live transport proof before merge.
