## Summary

Current main solves the reported Control UI readability problem by keeping exec/system-event runtime context out of visible WebChat prompt/history paths and by stripping residual internal runtime context from realtime Control UI chat events. This overlaps prior related cleanup already recorded for #56489 and #61990; the current implementation removes the reported `System (untrusted): ... Exec completed ...` content from the normal conversation flow rather than adding a separate log panel.

## Best Possible Solution

Close this item and keep the shipped/current runtime-transcript separation plus Control UI sanitization. If maintainers want a broader product feature for a separate system-event log panel, track that as a new explicit UX design issue rather than keeping this already-fixed leak/readability report open.
