## Summary

Current main treats Google as a bundled provider with Gemini 3.1 support, and the provider-catalog contract now merges plugin catalogs into models.json instead of leaving Google absent and falling through to the 200k default.

## Evidence

- **Provider catalog contract writes plugin catalogs into models.json:** Current public docs state that provider plugins can inject model catalogs via registerProvider({ catalog }) and OpenClaw merges that output into models.providers before writing models.json.
  - file: [docs/concepts/model-providers.md:129](https://github.com/openclaw/openclaw/blob/d6ef1fcf24eea3a5985d653a3baf400e32aa1b75/docs/concepts/model-providers.md#L129)
  - sha: [d6ef1fcf24ee](https://github.com/openclaw/openclaw/commit/d6ef1fcf24eea3a5985d653a3baf400e32aa1b75)
- **Google provider is documented as built-in with Gemini 3.1:** Current docs list Google Gemini API-key provider `google`, `GEMINI_API_KEY` auth, and example model `google/gemini-3.1-pro-preview`.
  - file: [docs/concepts/model-providers.md:309](https://github.com/openclaw/openclaw/blob/d6ef1fcf24eea3a5985d653a3baf400e32aa1b75/docs/concepts/model-providers.md#L309)
  - sha: [d6ef1fcf24ee](https://github.com/openclaw/openclaw/commit/d6ef1fcf24eea3a5985d653a3baf400e32aa1b75)
- **Google plugin owns Gemini 3.1 compatibility:** Current docs describe `google` and `google-gemini-cli` as owning Gemini 3.1 forward-compat fallback and related Gemini runtime behavior.
  - file: [docs/concepts/model-providers.md:189](https://github.com/openclaw/openclaw/blob/d6ef1fcf24eea3a5985d653a3baf400e32aa1b75/docs/concepts/model-providers.md#L189)
  - sha: [d6ef1fcf24ee](https://github.com/openclaw/openclaw/commit/d6ef1fcf24eea3a5985d653a3baf400e32aa1b75)

Current main documentation describes the opposite contract for bundled provider catalogs and Google Gemini 3.1 availability.
