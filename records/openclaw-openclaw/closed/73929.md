## Summary

Current main already covers the embedded-runner blank replay-history dispatch problem through the merged #74168 guard. The active code now skips blank visible prompts before provider submission when replay history exists, keeps runtime-only and prompt-image submissions allowed, and has focused helper plus runner regression coverage; #74731 remains a separate MiniMax transport fallback and does not keep this PR open.

## What This Changes

This PR changes embedded-runner prompt submission detection to ignore blank replay-history content, preserve structured non-text replay such as images, add regression tests, and add a changelog entry for #73926.

## Best Possible Solution

Keep the merged main implementation from #74168 as the embedded-runner boundary fix, and leave the distinct MiniMax/Anthropic empty-message transport work to #74731.

## Reproduction Assessment

Yes. The pre-fix release guard treated any replay array as content, while current main has source and test coverage showing a blank visible prompt with replay history is skipped before provider submission.

## Solution Assessment

Yes. The current main fix is the narrowest maintainable endpoint behavior for this class: it guards the embedded-runner dispatch boundary, preserves runtime-only and prompt-image submissions, and avoids dispatching solely because stale replay history exists.
