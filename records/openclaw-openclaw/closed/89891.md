## Summary

Current main and the latest local release tag still build Google Vertex multi-region `eu`/`us` requests with a regional host prefix, while Google’s current docs and `@google/genai` source route those multi-regions to `.rep.googleapis.com` hosts. The report remains actionable as a narrow Google provider runtime bug.

## What This Changes

The issue requests that Google Vertex `eu`/`us` multi-region locations use the correct Vertex endpoint instead of the hardcoded `${location}-aiplatform.googleapis.com` host.

## Best Possible Solution

Patch the Google provider’s Vertex URL builder, endpoint metadata, and focused tests so `global`, `us`, and `eu` match Google’s current endpoint contract while regional locations and explicit `baseUrl` overrides keep working.

## Reproduction Assessment

Yes, source-backed: with `GOOGLE_CLOUD_LOCATION=eu` and the bundled Vertex placeholder base URL, current code builds `https://eu-aiplatform.googleapis.com/.../locations/eu/...`. I did not run a live request without Vertex credentials.

## Solution Assessment

No: current main is not the right solution because it only handles `global`. The best fix is provider-owned endpoint mapping for `global`, `us`, and `eu` using Google’s current docs/SDK contract, not a new config surface as the primary repair.
