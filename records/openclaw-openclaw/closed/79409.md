## What This Changes

The issue asks Feishu text slash commands such as `/compact`, `/new`, and `/stop` to be intercepted by Gateway command handling instead of dispatched as plain agent text.

## Best Possible Solution

Review and land a narrow Feishu or shared command-authorization fix that treats admitted senders as command-authorized when no explicit command allowlist exists, while preserving explicit allowlists and owner-command enforcement.

## Reproduction Assessment

Yes. Current main has a clear source path where a Feishu slash command can reach `CommandAuthorized=false` with `commands.useAccessGroups=true` and no configured command allowlist; the reply path then returns no response for whole-message control commands.

## Solution Assessment

Yes for the narrow fix direction, but not for closing now. The best path is to review and land or replace the linked implementation at https://github.com/openclaw/openclaw/pull/79556 rather than close this issue before a fix merges.
