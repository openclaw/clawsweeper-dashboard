## Summary

Current main has the package-install safeguards needed for this failure mode: packaged installs prune stale dist files while preserving imported Vite/Rollup chunks, and the npm update path stages and verifies a clean package tree before swapping it into the global prefix. The proof is on main after the latest shipped v2026.5.2 tag, so this is fixed on current main but not proven shipped yet.

## What This Changes

The issue asks for npm global upgrades from 2026.4.23 to 2026.5.2 to avoid leaving a mixed dist tree that makes Gateway HTTP imports fail with missing hashed chunks.

Keep the current main packaging safeguards and ship them in the next package release so direct npm installs and openclaw update both avoid mixed hashed dist trees.
