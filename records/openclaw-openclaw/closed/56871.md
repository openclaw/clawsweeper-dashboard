## Summary

Current `main` already contains the session-lane restart recovery that prevents stale interrupted runs from keeping channel lanes permanently busy, and `openai/gpt-5.4` is resolved through the OpenAI Responses transport rather than the Chat Completions path described in the report. The same implementation is present in release `v2026.4.23`.

## Evidence

- **`openai/gpt-5.4` resolves to Responses transport:** The OpenAI provider normalizes GPT-5.4 family models onto `api: "openai-responses"`, and the provider test asserts `openai/gpt-5.4` resolves with `api: "openai-responses"` and the native OpenAI base URL.

  - file: [extensions/openai/openai-provider.test.ts:199](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/extensions/openai/openai-provider.test.ts#L199)
