Full review comments:

- **[P1] Enforce absolute preHook executable paths:** `src/cron/normalize.ts:507`
  - body: The docs and tool schema describe `preHook.file` as an absolute executable path, but normalization accepts any non-empty string and passes it to `execFile`. Bare names can resolve through PATH, so this preserves the search-path execution risk raised in review discussion.
  - confidence: 0.91
- **[P2] Use process-tree cleanup for hook timeouts:** `src/cron/pre-hook.ts:66-70`
  - body: `runPreHook` calls `execFileAsync` directly, so timeout or abort kills only the immediate process. Current cron command jobs go through `runCommandWithTimeout(..., killProcessTree: true)` to avoid leaving child processes behind; preHook needs the same cleanup boundary before it can safely run scripts on the scheduler path.
  - confidence: 0.84
