## Summary

Keep open: the PR fixes the xAI reasoning transport direction, but its explicit `/think off` path still omits the field; xAI documents omission as defaulting to low reasoning, so Off would not actually disable reasoning.

## What This Changes

The PR updates xAI provider thinking profiles, runtime model compatibility, stream payload stripping, tests, and changelog so reasoning-capable Grok models forward `/think` levels on outbound requests.

## Best Possible Solution

Keep the provider-owned policy and transport approach, but map explicit Off to xAI's disabled `none` effort for reasoning-capable models and keep stripping unsupported fields for non-reasoning routes.

## Reproduction Assessment

Yes, source-reproducible: current main cannot send Grok 4.3 reasoning controls, and the PR's Off case still maps to omission. pi-ai source plus current xAI docs show that omission does not preserve OpenClaw's documented Off semantics.

## Solution Assessment

No. The provider-owned profile-plus-transport direction is the right boundary, but the implementation needs a narrow disabled-effort mapping and test/proof update before merge.
