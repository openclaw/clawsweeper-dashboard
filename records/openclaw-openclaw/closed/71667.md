## Summary

Keep open. Current main does not provide high-confidence evidence that the reported duplicate `--openai-api-key` registration path is fixed, and the issue is fresh with concrete failure text. The updater’s completion refresh is soft-failure in code, but onboard option registration still adds manifest-derived flags directly and only dedupes provider flags by `optionKey::cliFlag`, which would not protect against same long flag with a different option key or a separate static/dynamic source.
