## Summary

Keep open. The overflow-hint direction is useful, but the patch still has a stale-session fallback bug, includes unrelated GoogleChat config/SDK boundary churn, and lacks required real behavior proof.

## What This Changes

This PR changes the context-overflow recovery hint to compute `reserveTokensFloor` from model/session context-window tiers, adds tests, and also splits GoogleChat config schemas into a new provider module.

## Best Possible Solution

Keep the overflow fix scoped to `agent-runner-execution` and its focused tests, guard session fallback by matching model/provider, and split or drop the GoogleChat schema refactor unless maintainers explicitly want that separate boundary change.

## Reproduction Assessment

Yes. Current main has a source-level reproduction because the generic overflow recovery path still falls back to the static `20000 or higher` reset hint when heartbeat-bleed diagnosis is absent.

## Solution Assessment

No, not yet. The tiered hint is the right direction, but the session fallback needs a model/provider match guard and the unrelated GoogleChat schema extraction should not ride along in this bug-fix PR.
