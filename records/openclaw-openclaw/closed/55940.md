## Summary

Keep open. The PR targets a real Docker/Linux skill-install failure, but the proposed apt fallback still has blocking security and reliability issues, no after-fix real Linux proof, and overlaps with other open approaches for the same package-manager strategy decision.

## What This Changes

This PR adds a Linux apt-get fallback for missing-Homebrew brew skill installers, refactors Go apt installation through a shared helper, and expands mocked installer fallback tests.

## Best Possible Solution

Choose one documented Docker skill-dependency strategy; if apt fallback is selected, gate it to trusted or explicit opt-in installs, keep sudo non-interactive, update coverage/docs, and require real Linux proof.

## Reproduction Assessment

Yes, from source inspection: current main still returns the brew-missing failure for brew installers when no brew executable resolves, and openai-whisper remains brew-only. I did not run a live Docker or apt install in this read-only review.

## Solution Assessment

No. The apt fallback addresses the visible symptom, but it is not the best mergeable solution until maintainers choose the package-manager trust posture and the patch gates privileged installs and keeps sudo non-interactive.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate apt fallback to trusted skill sources:** `src/agents/skills-install.ts:553-555`
  - body: This missing-brew branch runs apt-get or sudo apt-get for any brew installer metadata even though current main only warns that workspace and personal skill metadata can be attacker-controlled. Gate the fallback to trusted or explicitly opted-in installers before invoking apt, otherwise a user-level skill install becomes privileged package installation on root/passwordless-sudo Linux hosts.
  - confidence: 0.92
- **[P2] Keep sudo apt commands non-interactive:** `src/agents/skills-install.ts:327-328`
  - body: The helper probes sudo with `sudo -n true` but then runs `sudo apt-get update/install` without `-n`. Because the command runner inherits stdin when no input is set, sudo can still prompt or block until timeout when apt-get needs a password or is denied; pass `-n` to the follow-up sudo commands or close stdin.
  - confidence: 0.78
