## Summary

Current `main` does not require manual `AGENTS.md` edits to make installed skills discoverable. Skills install into the workspace `skills/` directory, the loader treats `SKILL.md` frontmatter `name` + `description` as the discovery surface, and `user-invocable` skills are auto-exposed as slash commands, so the issue’s reported AGENTS-based trigger workflow no longer matches current behavior.

## Evidence
