## Summary

The report is credible, but the sandbox eligibility check lives in the separate Windows companion repository, while this repository only documents and promotes the signed Windows Hub installers into OpenClaw releases.

## What This Changes

The issue asks for Windows 11 25H2 build 26200.8655 to stop failing the Windows Hub MXC sandbox eligibility check that currently reports a build 26300 requirement.

## Best Possible Solution

Track and land the Windows Hub MXC sandbox compatibility fix in `openclaw/openclaw-windows-node`, then promote a new signed Windows Hub installer through the existing OpenClaw release workflow.

## Reproduction Assessment

Yes, at source level for the Windows companion implementation: current `MxcAvailability` rejects every Windows build except 26300, so build 26200 is rejected before any real host probe. I did not run a live Windows 11 25H2 machine in this read-only review.

## Solution Assessment

No, not in this repository: the code-level fix belongs in `openclaw/openclaw-windows-node`, and an external PR is already replacing the brittle build table with `wxc-exec --probe`.
