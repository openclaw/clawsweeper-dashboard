## Summary

Issue #38169 describes host-level Tailscale MagicDNS failure on one named node, not a current OpenClaw defect. Current main still passes the configured gateway URL straight to the WebSocket client and only reports the underlying DNS/network error; the supported remote-node paths are documented as either a working tailnet/Tailscale Serve endpoint or an SSH tunnel when the gateway stays loopback-bound.

## Evidence

- **Node host only logs the underlying connect error:** `src/node-host/runner.ts:211-213` shows `onConnectError` writing `node host gateway connect failed: ${err.message}`. There is no OpenClaw-side MagicDNS resolver or retry/fallback logic here; the reported `getaddrinfo ENOTFOUND ...ts.net` is the host OS/network stack failing name resolution.
  - file: [src/node-host/runner.ts:211](https://github.com/openclaw/openclaw/blob/53aac30f5104052db56da2b26bcee0f4e5140325/src/node-host/runner.ts#L211)
  - command: `nl -ba src/node-host/runner.ts | sed -n '200,220p'`
  - sha: [53aac30f5104](https://github.com/openclaw/openclaw/commit/53aac30f5104052db56da2b26bcee0f4e5140325)
- **Gateway client uses the configured URL directly and treats remote access as transport configuration:** `src/gateway/client.ts:224-282` constructs `const url = this.opts.url` and opens `new WebSocket(url, ...)`. The only built-in guidance is to use `wss://`, SSH tunneling, or Tailscale Serve/Funnel for remote hosts; there is no app-level workaround for a machine that cannot resolve its configured hostname.
  - file: [src/gateway/client.ts:224](https://github.com/openclaw/openclaw/blob/53aac30f5104052db56da2b26bcee0f4e5140325/src/gateway/client.ts#L224)

- **Current docs define the supported remote node-host fallback:** `docs/nodes/index.md:84-99` says remote node hosts cannot connect directly to a loopback-bound gateway and should use an SSH tunnel to `127.0.0.1:18789`. That is the documented OpenClaw path when direct hostname reachability is unavailable.
  - file: [docs/nodes/index.md:84](https://github.com/openclaw/openclaw/blob/53aac30f5104052db56da2b26bcee0f4e5140325/docs/nodes/index.md#L84)
