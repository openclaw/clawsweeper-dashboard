## Summary

Current `main` already routes Google Chat auth through a plugin-owned scoped `gaxios` transport with a custom `fetchImplementation`, and the same implementation is present in release `v2026.4.22`. That avoids the old `google-auth-library -> gaxios -> node-fetch -> fetch-blob -> node-domexception` startup-crash path even though the root `node-domexception` alias is still present.
