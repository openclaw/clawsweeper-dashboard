## Summary

Current main already fixes the WhatsApp `channels add` problem behind PR #61927: `--auth-dir` is accepted, passed through setup input, persisted by the WhatsApp setup adapter, documented, tested, and called out in the 2026.4.24 changelog as fixing #61923. The PR's exact generic auth-only error wording is not present, but WhatsApp no longer needs that fallback path on main.

## Best Possible Solution

Close PR #61927 as implemented by the later mainline WhatsApp setup fix. Keep the shipped `channels add --auth-dir` plus `channels login` flow and use a new focused issue only if a broader generic login-only plugin error message is still desired.
