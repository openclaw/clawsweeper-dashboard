## Summary

The request is coherent, but on current `main` it is better scoped as memory-plugin or hook work than a new core `sessions_spawn` flag. OpenClaw already treats memory as plugin-owned, exposes `session_end`/`subagent_ended` lifecycle hooks, and ships a bundled `session-memory` hook that demonstrates session-to-memory persistence outside core.
