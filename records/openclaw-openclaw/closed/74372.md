## Summary

Current main fixes the onboarding failure mode reported against v2026.4.26: bundled channel plugins such as WhatsApp are now detected as local bundled sources, the misleading npm download option is suppressed for that path, and the bundled plugin is enabled without creating a separate npm install record. The fix is present on main but not in the latest tagged release available in this checkout.

## What This Changes

Report that `openclaw onboard` offers an npm install for the WhatsApp plugin even though `@openclaw/whatsapp` is not available on npm.

## Best Possible Solution

Keep the current main behavior: onboarding should use the bundled WhatsApp plugin when it ships with OpenClaw, avoid sending users to npm for that bundled path, and let the next release deliver the corrected setup flow to v2026.4.26 users.
