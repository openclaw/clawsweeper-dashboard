## Summary

Keep open: the PR is a real bug-fix attempt, but the patch still has blocking setup-code credential exposure, missing real iOS behavior proof, and docs/changelog gaps.

## What This Changes

This PR changes setup-code generation from bootstrap tokens to shared token/password credentials, updates TypeScript and Swift auth precedence, permits iOS local-network `ws://` inputs, and adds targeted tests plus pairing docs.

## Best Possible Solution

Preserve bounded setup credentials, land the iOS local-network `ws://` and password-precedence fixes with aligned docs/changelog, and require real iOS onboarding proof before merge.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main rejects non-loopback plaintext iOS setup/deep-link hosts and setup-code generation emits only `bootstrapToken`. I did not establish a live iOS device reproduction.

## Solution Assessment

No. The local-network `ws://` and password-over-bootstrap parts are narrow, but replacing bounded bootstrap setup codes with shared gateway credentials is not the safest maintainable solution without explicit security design resolution.
