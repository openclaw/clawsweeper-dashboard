## Summary

Current `main` already fixes the macOS LaunchAgent lifecycle gap behind this report: `openclaw gateway stop` keeps the LaunchAgent installed, `openclaw gateway start` now repairs an installed-but-unloaded LaunchAgent, and integration tests explicitly cover stop-then-start without reinstall.

## Evidence

- **`gateway start` repairs installed-but-unloaded LaunchAgents on macOS:** `runDaemonStart()` wires `onNotLoaded` to `recoverInstalledLaunchAgent({ result: "started" })`, so a stopped-but-still-installed LaunchAgent is re-bootstrapped instead of treated as missing.
  - file: [src/cli/daemon-cli/lifecycle.ts:152](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/cli/daemon-cli/lifecycle.ts#L152)
  - command: `nl -ba src/cli/daemon-cli/lifecycle.ts | sed -n '152,163p'`
- **LaunchAgent recovery re-bootstraps and kickstarts the existing plist:** `recoverInstalledLaunchAgent()` first checks that the plist still exists, then calls `repairLaunchAgentBootstrap()`, which runs `launchctl enable`, `bootstrap`, and `kickstart` against the existing LaunchAgent.
  - file: [src/cli/daemon-cli/launchd-recovery.ts:14](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/cli/daemon-cli/launchd-recovery.ts#L14)
  - command: `nl -ba src/cli/daemon-cli/launchd-recovery.ts | sed -n '1,38p'`

  - file: [src/daemon/launchd.ts:466](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/daemon/launchd.ts#L466)
