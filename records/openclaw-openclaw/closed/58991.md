## Summary

Current `main` already prevents reload-time internal hook duplication by explicitly unregistering prior workspace and plugin hook registrations before re-registering them, and it now has direct tests for the exact duplicate-on-reload behavior this PR targets. The proposed `registerInternalHook()` dedupe guard is therefore redundant on current `main`.

## Evidence
