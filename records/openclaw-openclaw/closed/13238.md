## Summary

Current `main` already has a documented beta-first stable release policy plus enforced release preflight and release-check workflows, which covers the requested stable quality gate/canary process well enough.

## Evidence

- **Public release policy documents beta-first stable promotion and release checklist:** `docs/reference/RELEASING.md` defines beta-first stable releases, says stable follows only after the latest beta is validated, and lists the required release preflight/check steps and promotion flow from `beta` to `latest`.
  - file: [docs/reference/RELEASING.md:33](https://github.com/openclaw/openclaw/blob/68e2d6f0889e5365ecdc8f01ab5e3bf6065cbeeb/docs/reference/RELEASING.md#L33)

  - file: [.github/workflows/openclaw-npm-release.yml:107](https://github.com/openclaw/openclaw/blob/68e2d6f0889e5365ecdc8f01ab5e3bf6065cbeeb/.github/workflows/openclaw-npm-release.yml#L107)

## Risks / Open Questions

- The implementation is workflow- and docs-driven rather than a dedicated release PR template, so the proposal is satisfied by process/pipeline changes rather than the exact mechanism suggested in the issue.
- Public docs describe beta-first promotion and release gates, but any finer rollback/go-no-go criteria may live in maintainer-only release docs.

## Close Comment

Closing from a Codex review because this is already implemented on current `main`, and the public release tag `v2026.4.22` already contains it. `docs/reference/RELEASING.md` now documents a beta-first stable policy, required release preflight steps, and promotion from `beta` to `latest`; `.github/workflows/openclaw-npm-release.yml` enforces the gated preflight/publish flow with `preflight_run_id`; and `.github/workflows/openclaw-release-checks.yml` runs install smoke, cross-OS validation, QA parity, and live Matrix/Telegram release lanes before approval. The checklist/policy ended up landing in release docs and workflows rather than a dedicated release PR template, but the requested stable quality gate and canary process are in place well enough now.
