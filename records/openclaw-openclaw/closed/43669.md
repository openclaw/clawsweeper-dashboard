## Summary

Current main already solves the PR's central race through the pinned Pi dependency contract: `AgentSession.prompt()` now waits for auto-retry and then waits for the underlying agent to become idle before returning, so OpenClaw's existing post-`activeSession.prompt()` snapshot path is protected without the PR's local duplicate wait. This shipped in OpenClaw after the Pi package bump that first brought in `@mariozechner/pi-coding-agent` 0.65.0, and current main/release carry newer fixed versions.

## Best Possible Solution

Close this PR as implemented by the shipped Pi dependency behavior. Keep the current dependency-backed idle wait; if maintainers want extra defense in depth, add a focused OpenClaw regression test around `activeSession.prompt()` resolving only after auto-retry idle rather than landing this stale duplicate wait.
