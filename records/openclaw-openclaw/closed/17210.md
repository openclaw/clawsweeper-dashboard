## Summary

Current main treats domain-shaped context behavior as plugin-owned context-engine/memory work, not as OpenClaw core scope. This request is better suited to a ClawHub-published plugin or skill than a built-in core feature.

## Close Comment

Closing after a Codex review. On current `main`, domain-shaped context behavior is explicitly plugin-owned rather than a core feature: `docs/concepts/context-engine.md:15` says to install a context-engine plugin when you want different assembly/recall behavior, `docs/plugins/architecture.md:1367` assigns ingest/assembly/compaction to context-engine plugins, and `src/plugins/types.ts:2005` already exposes the registration seam for that. The project vision also keeps optional capability out of core and sends new skills toward ClawHub first (`VISION.md:55`, `VISION.md:69`, `VISION.md:101`). For that reason, native health/finance/projects context packets are better scoped as a ClawHub plugin/skill or a context-engine or memory plugin, not an OpenClaw core issue.
