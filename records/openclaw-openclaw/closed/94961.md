## Summary

Keep this PR open, but the current patch is not merge-ready: it turns `config.summaryModel` into runtime LLM override authorization, while current code and docs require explicit `plugins.entries.<id>.llm` operator trust.

## What This Changes

The PR changes plugin config normalization so a lossless-claw entry with `config.summaryModel` synthesizes an `llm` override policy on normal config load.

## Best Possible Solution

Preserve explicit `plugins.entries.<id>.llm` authorization as the only model-override trust source, then fix the context-engine/runtime config lifecycle with focused tests and real gateway proof.

No live current-main LCM run was executed in this read-only review. The patch defect is source-reproducible because the PR writes trusted `llm` policy from `summaryModel` while current runtime and docs require explicit `llm.allowModelOverride`.

## Solution Assessment

No. This is not the best fix because it turns plugin config into authorization; the safer path is to preserve explicit `llm` trust and repair the context-engine/runtime config lifecycle.
