## Summary

Current main already fixes the Telegram bare `/new` empty Codex Responses request path. The reset command handler now stops bare `/new` and `/reset` after reset hooks with a local acknowledgement, and regression tests assert the model runner is not invoked.

## Best Possible Solution

Keep the main implementation: bare `/new` and `/reset` should reset and acknowledge locally after hooks, while `/new <message>` and `/reset <message>` continue into the model with real user input. Ship this main-branch fix in the next release rather than tracking another core issue.
