## What This Changes

The branch changes the Dreaming advanced UI so the waiting/promoted origin badge returns `originDailyLog` from `dailyCount > 0` instead of `groundedCount > 0`.

## Best Possible Solution

Converge the Dreaming Advanced page on one provenance definition: either land #72552 or an equivalent patch that updates the badge, summary, staged section, and focused badge-matrix tests together, or deliberately rename the section if grounded replay is the intended concept.

## Reproduction Assessment

Yes. A focused view fixture with daily-only, grounded-only, and mixed entries reproduces the classification matrix; the linked issue also reports the live-store case of many `dailyCount > 0` entries and zero `groundedCount > 0` entries leaving the Daily Log Review area empty.

## Solution Assessment

No. The PR is a useful partial change, but as a standalone fix it is not the best solution because it leaves the same page using `dailyCount` for one badge path and `groundedCount` for the summary/staged section; the safer path is the broader #72552-style fix with regression tests.
