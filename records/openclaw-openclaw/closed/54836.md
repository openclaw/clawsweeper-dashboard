## Summary

Current `main` already has inbound reply deduplication in the shared auto-reply dispatch path, with regression coverage for duplicate `MessageSid` deliveries and the same inbound message arriving under both main and direct session keys. The changelog shows this shipped in `v2026.4.12` as the channels replay-dedupe fix.
