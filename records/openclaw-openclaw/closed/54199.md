## Summary

Current main implements the reported Control UI/WebChat non-image upload path: the file picker is no longer image-only, the UI serializes non-images as file attachments, Gateway persists non-image/non-video attachments as managed inbound media, and focused regression tests plus docs/changelog cover the behavior. The latest stable release provided in the review context predates the fix, so no shipped release is assigned.

## Best Possible Solution

Close this issue as implemented on current main and keep the merged Control UI/Gateway managed-media behavior. Any remaining request for video uploads or direct workspace-copy semantics should be tracked as a separate, narrower enhancement.
