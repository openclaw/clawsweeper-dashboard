## Summary

Current main already prevents the reported permanent Anthropic session corruption. The unmerged heartbeat-pruning PR #46987 was not the shipped path, but the shared strict-provider replay hygiene now repairs missing/displaced tool results and strips any remaining dangling Anthropic tool-use blocks before provider submission. The separate user-visible heartbeat `[TOOL_CALL]` text leak remains tracked by #54138 rather than this transcript-corruption issue.

## Best Possible Solution

Close #41329 as implemented and keep the shipped shared replay-hygiene path. Treat #54138 as the canonical remaining heartbeat user-visible tool-markup issue, and only reopen this if a fresh current-main repro shows Anthropic still receives orphaned heartbeat tool-use blocks after replay sanitization.
