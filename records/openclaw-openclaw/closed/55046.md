## Summary

Current main still preserves the reported ACP media-understanding gap. ACP dispatch invokes media understanding for inbound media, but still passes only ctx and cfg; provider-backed image understanding requires agentDir, and activeModel is still needed for correct native-vision skip and automatic media provider routing. This issue should remain open.

## Best Possible Solution

Keep this issue open. Fix ACP dispatch by resolving the effective ACP session agent directory from the canonical ACP session/agent and passing that agentDir into applyMediaUnderstanding. Also pass the active ACP model when the ACP runtime/session configuration can provide it, so native-vision skip and automatic media-provider routing match the normal reply path. Add a focused dispatch-acp.test.ts regression for an inbound image ACP turn with configured tools.media.image, asserting media understanding receives agentDir/activeModel and enriches the prompt before acpManager.runTurn.
