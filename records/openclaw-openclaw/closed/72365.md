## Summary

Keep this issue open. The current gateway path still runs `tools.effective` inventory resolution synchronously on the request path, the Control UI still calls that method for the visible tools panel, and the linked implementation candidate remains an open, unmerged PR with closing syntax. The related comment linking #65279 is useful corroboration of web UI messages disappearing/reappearing after delay, but it is not a merged fix or a canonical duplicate for this broader gateway/control-plane responsiveness regression.

## Best Possible Solution

Keep this issue open. Review #72558 as the active implementation candidate, but require a real event-loop-safe fix before closure: cache/debounce effective tool inventory, add incremental async yield points inside the expensive resolver, or move CPU-heavy inventory work off the main gateway event loop. Add focused regression coverage showing another gateway/control UI request can complete promptly while `tools.effective` resolution is in progress. If #72558 merges with sufficient proof, this issue can then close as implemented by that merged PR.
