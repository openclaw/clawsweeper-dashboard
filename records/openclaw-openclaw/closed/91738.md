## Summary

Current main already fixes the reported Feishu CardKit streaming behavior: the release sent only suffix deltas to the text-update endpoint, while main now sends the full merged card text that the Feishu API expects.

## What This Changes

The issue requests Feishu streaming card updates to show accumulated generated text instead of replacing the card with only the latest delta chunk.

## Best Possible Solution

Keep the current main behavior that sends full CardKit streaming content, and track separate Feishu streaming-card edge cases in their existing narrower reports.

## Reproduction Assessment

Yes, from source and release tests: v2026.6.1 passed only suffix deltas into the CardKit text update body, while Feishu expects full text content. I did not run a live Feishu bot in this read-only review.

## Solution Assessment

Yes: sending the full merged text from the Feishu streaming-card owner is the narrow maintainable fix for a CardKit endpoint that replaces text content and computes the visual increment itself.
