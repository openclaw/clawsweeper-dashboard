## Summary

Keep open. Current `main` still rolls fresh isolated cron sessions by spreading the prior session entry without clearing runtime `model`/`modelProvider`, so this cleanup is not implemented on `main`. The PR discussion also shows the branch is stale but the core fix remains useful after rebasing, even though the original `payload.model`-shadowing rationale is partly obsolete on current code.
