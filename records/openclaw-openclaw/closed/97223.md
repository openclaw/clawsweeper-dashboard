## Best Possible Solution

Keep repair mutations marked for final writeback, or split the state into distinct preview and writeback flags with config-flow coverage proving repair-mode writes still persist.

## Reproduction Assessment

## Solution Assessment

No. The patch treats `pendingChanges` as only a preview marker, but current main uses it as the final repair writeback signal; the safer fix is to keep it true or split the concepts with full caller coverage.
