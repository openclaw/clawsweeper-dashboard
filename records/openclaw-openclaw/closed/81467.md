## Best Possible Solution

Bounded `usage.cost.daily` should either be accepted as a dense calendar series for all gateway consumers, or densification should move to the UI/export layer while the public RPC stays sparse.

## Solution Assessment

Yes, with an API-contract caveat. Backend zero-fill is the clean consistent fix across cached and uncached summaries, but usage/API owner acceptance is required because the shared response row semantics change.
