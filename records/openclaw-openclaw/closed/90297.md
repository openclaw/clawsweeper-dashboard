## Summary

Current main already addresses the Feishu setup-runtime gap behind this dispatch failure by wiring the Feishu setup entry to initialize `setFeishuRuntime`; the fix is main-only and not in `v2026.6.1`.

## What This Changes

The issue reports Feishu inbound messages being received over WebSocket but failing during dispatch to the agent with `TypeError: Cannot read properties of undefined (reading 'run')`.

## Best Possible Solution

Keep the current main Feishu setup-runtime wiring and ship it in the next release; reopen or file a new report only if a build containing that wiring still fails.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: `v2026.6.1` lacks the Feishu setup-runtime setter, while current main adds it and tests that setup-only registration stores a runtime with `channel.inbound.run`. I did not run a live Feishu account.

## Solution Assessment

Yes. Wiring the Feishu setup entry to `setFeishuRuntime` is the narrow owner-boundary fix for setup-only plugin loading and avoids adding fallback behavior or user configuration.
