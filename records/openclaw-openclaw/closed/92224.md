## Summary

Keep open: current main and the latest release still derive the CLI row `available` value from registry availability or provider-auth fallback, so a local Ollama row can still render `local: true` with `available: false`; the two linked fix PRs were closed unmerged and do not resolve the issue.

## What This Changes

The issue reports that `openclaw models list --json` marks a reachable local Ollama model as unavailable even though sessions can use it.

## Best Possible Solution

Preserve provider-level Auth semantics while fixing the local Ollama row case with focused regression coverage and live or equivalent local-provider CLI proof.

## Reproduction Assessment

Yes at source level, but not as a live Ollama run in this review: a local row with `availableKeys` defined but missing the model key still renders `available: false`.

## Solution Assessment

Unclear whether an unconditional `local ? true` change is the best final shape; the safer fix should align with documented local-marker semantics and prove the real local-provider CLI case.
