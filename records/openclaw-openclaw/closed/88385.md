## What This Changes

The issue requests that restrictive `exec-approvals.json` defaults act as fallback values and not override an explicit permissive `agents.main` policy.

## Best Possible Solution

Keep the current main behavior and ask for a new report only if the same config still prompts on a build containing the current-main proof SHA.

Yes: the current implementation resolves `agents.main` before defaults and threads the main session agent id into the exec approval resolver. The remaining distinction is release availability, since the proof is current-main-only.
