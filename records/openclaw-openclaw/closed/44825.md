## Summary

Current main already covers the requested zero-token `sessions_spawn` notification path: OpenClaw exposes `after_tool_call` and `subagent_spawned` plugin hooks, and it also persists spawned-run metadata including `task` to `subagents/runs.json`. That gives both proposed solutions from the issue without adding a new core hook surface.

## Evidence

- **`after_tool_call` is a supported plugin hook:** The public hook list includes `after_tool_call`, `subagent_spawning`, and `subagent_spawned`. The hook payload type for `after_tool_call` includes `params`, `runId`, `toolCallId`, and `result`, which is enough for a `sessions_spawn` notification plugin to read the task text and returned child session metadata without spending model tokens.
