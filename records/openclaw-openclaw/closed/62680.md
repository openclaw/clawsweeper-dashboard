## Summary

Current `main` already handles Telegram voice notes through audio preflight before any later body wrapping, and the latest release `v2026.4.23` contains the same implementation. The current Telegram inbound path computes `hasUserText` from actual Telegram text/caption plus location only, not from `<media:audio>` placeholders or later envelope text, and it explicitly preflights DM audio-only messages.

## Evidence

- **Telegram inbound gating ignores placeholders/wrappers when deciding `hasUserText`:** `rawText` comes from Telegram `text`/`caption`, `hasUserText` is `Boolean(rawText || locationText)`, and `<media:audio>` is only assigned afterward as a placeholder. Preflight then runs when `hasAudio && !hasUserText`, including DMs via `!isGroup || (...)`.
