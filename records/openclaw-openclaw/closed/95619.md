The issue reports that the default `HEARTBEAT.md` template is treated as non-empty because HTML comments are not ignored, causing unintended scheduled heartbeat model calls.

## Best Possible Solution

Keep the current HTML-comment-aware empty detection, runtime template split, and doctor repair path; reopen only with evidence that a fresh `v2026.6.5-beta.1` or newer workspace still sends model heartbeats for an untouched template.

The old affected behavior is source-explained for older releases, but it no longer reproduces in the current code path.
