## Summary

Current `main` already contains the matching fix: the embedded PI runner now backfills missing/blank `sessionKey` values before downstream session-scoped logic runs, with regression tests covering the omitted-session-key path. That matches the issue timeline’s cross-reference to PR #60555 and is sufficient evidence that the heartbeat/main session-label mixup has been addressed on `main`, though it has not shipped in the latest tagged release yet.
