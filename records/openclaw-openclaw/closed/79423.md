## What This Changes

Restore Slack native reply/thread context to the model after runtime-context prompt-cache changes.

## Best Possible Solution

Keep the current prompt-local inbound context path; any remaining Slack-specific missing quoted-body case should be filed as a narrower Slack supplemental-context issue with sanitized event payloads.

## Reproduction Assessment

No failing current-main reproduction was established; the high-confidence path is source-level. Slack turns now carry ReplyToId into inbound context, and that context is prefixed into the model prompt for the current turn.

## Solution Assessment

Yes. The current implementation is the narrow maintainable fix because it keeps volatile reply metadata out of the cache-sensitive system prompt while still making it model-visible as current-turn user-role context.
