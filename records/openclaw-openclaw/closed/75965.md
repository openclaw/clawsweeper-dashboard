## Summary

Keep open. Current main still has the reported user-facing mismatch: Telegram model picker selections are stored as session overrides, while the success copy does not say the agent default remains unchanged; reset/new-session acknowledgements are also generic.

## What This Changes

Clarify Telegram and model-switch UX so session-scoped model overrides are visibly distinct from durable agent defaults after reset, recreation, or restart.

## Best Possible Solution

Keep the current session-override storage contract, add explicit channel-facing scope copy for model switches and resets, and leave durable default reconciliation to the broader model-state threads.

## Reproduction Assessment

Yes for the remaining UX mismatch: current source and tests show a Telegram picker selection writes session override fields while the confirmation omits session-only and agent-default wording. The exact production fallback-after-restart path still needs a live or targeted regression reproduction before changing precedence.

## Solution Assessment

Unclear for a broad behavior change. The safest first fix is a narrow Telegram/core command copy and diagnostic update; durable-default buttons or single-source reconciliation should stay with the broader design work.
