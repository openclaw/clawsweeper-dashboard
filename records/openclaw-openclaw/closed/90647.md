## Summary

Keep open for maintainer review: the patch appears correct and the new macOS/Windows real-behavior proof is sufficient, but it intentionally changes the watcher backend for session skill refresh and carries a maintainer-owned platform/timing tradeoff.

## What This Changes

The branch replaces chokidar per-file skills watching on macOS/Windows with native recursive `fs.watch` plus parent reattach/chokidar fallback, and adds watcher lifecycle regression tests.

## Best Possible Solution

Land this shape after maintainer acceptance of the native watcher tradeoff; keep the fallback and regression tests, and avoid the directory-only alternative because it drops documented `SKILL.md` edit refresh.

## Reproduction Assessment

Yes. Current main source still routes `SKILL.md` files through chokidar, and the linked issue/PR provide a concrete macOS lsof path showing one persistent FD per installed workspace skill.

## Solution Assessment

Yes, this appears to be the best fix shape: native recursive watching preserves `SKILL.md` edit refresh while removing per-file fan-out, whereas directory-only watching loses edits and polling adds broader platform cost.
