## Summary

Current OpenClaw main explicitly classifies Anthropic-style `overloaded_error`/503 responses as `overloaded`, routes known failover errors through the model fallback chain, and has direct regression coverage proving cross-provider fallback after an overloaded primary failure. The same regression test is present in the latest shipped tag `v2026.4.23`, so the issue is already fixed in a released build.

## Evidence

- **503 overloaded classification:** `classifyFailoverReasonFromHttpStatus` preserves `overloaded` for HTTP 503, and `classifyFailoverReasonFromCode` maps `OVERLOADED_ERROR` to `overloaded`, so Anthropic overloaded payloads are routed into failover rather than generic retry handling.
