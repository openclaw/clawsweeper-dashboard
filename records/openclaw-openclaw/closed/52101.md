## Summary

PR #52101 is superseded by the current SDK subpath direction and the still-open canonical compatibility tracker #52885. Current main intentionally keeps the bare `openclaw/plugin-sdk` runtime surface small, while the helpers named in this PR already live on focused SDK subpaths; the remaining Weixin/root-barrel compatibility question should be handled in the issue or by plugin-side migration, not by merging this PR as-is.

## Best Possible Solution

Close PR #52101 as superseded. Keep the remaining `@tencent-weixin/openclaw-weixin` compatibility work in #52885 and solve it through documented narrow SDK subpath imports or a targeted compatibility adapter if maintainers explicitly choose one, rather than expanding the bare root SDK barrel with heavier server/channel helpers.
