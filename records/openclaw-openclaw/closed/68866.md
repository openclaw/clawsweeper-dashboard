## Summary

Current main, shipped in v2026.4.24, already fixes the stale Codex OAuth credential failure through safer disk reloads in locked auth writes and OAuth refresh recovery. The unmerged PR's runtime-snapshot mtime approach is now stale and had review comments flagging credential/snapshot regressions, so it should not be merged as the fix.

## Best Possible Solution

Close this PR and keep the shipped v2026.4.24 implementation. If maintainers still want broader runtime auth snapshot mtime invalidation later, it should be designed as a new change with tests for subagent fallback, save-path refresh, auth-state changes, and runtime-only secret overlays.
