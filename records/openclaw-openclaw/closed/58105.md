## Summary

Issue #58105 requests cron multi-channel fanout, and the same remaining work is already tracked by open PR #48194 (`feat(cron): support additionalTargets for multi-channel delivery`). Current `main` still has a single-target cron delivery contract and runtime path, so this should close as a duplicate/superseded tracker, not as implemented.

## Best Possible Solution

Close #58105 as the duplicate issue and continue the product/code discussion in #48194 or a maintainer-approved replacement. The complete solution should add a reviewed cron fanout contract across types, protocol schema, normalization and patch merge, isolated-run dispatch, CLI/UI, docs, and regression tests.
