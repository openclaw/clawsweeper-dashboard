## Summary

Keep open: current main and v2026.6.9 still leave generated gateway Scheduled Tasks on the direct `.cmd` launcher unless the hidden-launcher flag is supplied, and https://github.com/openclaw/openclaw/pull/95480 is an open closing candidate rather than a merged fix.

## What This Changes

Windows gateway Scheduled Tasks should launch through a windowless wrapper instead of directly targeting `gateway.cmd`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/95480 after required checks and maintainer acceptance of the VBS/WScript default, then close this issue as implemented.

## Reproduction Assessment

Yes. Source inspection shows generated gateway env lacks the hidden-launcher flag while scheduler action resolution only switches to `.vbs` when that flag is truthy; I did not run a live Windows desktop repro in this read-only review.

## Solution Assessment

Yes, with maintainer compatibility review. Reusing the existing hidden VBS launcher at the generated gateway service-env boundary is narrower than adding a new launcher mechanism, but the WScript default needs maintainer acceptance before merge.
