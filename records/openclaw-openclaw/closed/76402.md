## Summary

Keep open. The performance goal is plausible and useful, but this PR is not merge-ready because the new lightweight `sessions.list` path drops public row metadata that current docs and tests expect, and the user-facing fix still needs a changelog entry.

## What This Changes

The PR adds oversized transcript line guards/tests and a `buildGatewaySessionRow()` lightweight mode that `sessions.list` uses to skip transcript fallback and row enrichment for bulk list rows.

## Best Possible Solution

Keep the sessions.list hot path lightweight only for transcript fallback and genuinely expensive recomputation, while preserving public row metadata, adding async-list regression coverage, and documenting the user-facing performance fix in the changelog.

## Reproduction Assessment

Yes. The PR diff source-reproduces a contract regression because `sessions.list` uses `listSessionsFromStoreAsync()` and the new lightweight branch omits documented list-row fields; I did not live-reproduce the reported production latency.

## Solution Assessment

No. Skipping transcript fallback and expensive recomputation is the right direction, but dropping cheap public metadata is not the narrowest maintainable fix.
