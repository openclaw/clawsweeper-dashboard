## Summary

Current main does not fully solve #65207. The non-`--all` `openclaw models list --provider ollama` path still only prints configured default/fallback/model-reference entries, while explicit `models.providers.ollama.models` rows are appended only through the `--all` row-source path. The related closed Codex/Kilo discovery reports do not supersede this Ollama explicit-config CLI behavior.

## Best Possible Solution

Keep the issue open. The best fix is a focused CLI behavior change plus regression test: make `openclaw models list --provider <id>` include explicit `models.providers.<id>.models` rows as configured provider models, or intentionally document that users must pass `--all` and adjust the command/help to avoid the current misleading empty result.
