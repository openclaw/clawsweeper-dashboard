## Summary

The fix is not present in v2026.6.5 and has not yet appeared in a release tag, so this is a current-main-only fix.

## What This Changes

Fix ordinary WhatsApp `message(action="send")` calls being rejected as poll creation after upgrading to OpenClaw 2026.6.5.

## Best Possible Solution

Keep the canonical shared-validator fix on main and include it in the next OpenClaw release rather than adding a separate WhatsApp-specific workaround.

## Reproduction Assessment

Yes, from source with high confidence: v2026.6.5 retains the false-positive validator path, while current main includes a focused test proving the same schema-padded plain-send shape is accepted.

## Solution Assessment

Yes. Narrowing poll intent in the canonical shared validator fixes the cross-channel root cause without adding a WhatsApp-only bypass, fallback, or configuration surface.
