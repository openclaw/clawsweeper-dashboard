## Summary

Keep PR #40200 open. Current main still does not implement the PR’s POSIX shell startup-suppression behavior for exec commands: `getShellConfig()` continues to return plain `['-c']` for bash/zsh/fish-derived command shells, and host exec/PTY execution both consume those args. The PR discussion shows useful review follow-through: the fish `--no-config` gap was addressed, and the later PATH-fallback concern was addressed by restoring the sh/bash fallback. The remaining state is maintainer review/rebase/validation, not an obsolete or closeable PR.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to rebase it onto current main, preserve the existing non-interactive-shell and PATH fallback behavior, keep regression tests for zsh `-f`, bash `--noprofile --norc`, and fish `--no-config`, then run the relevant changed exec/agents checks once the baseline is suitable.
