## Summary

Current OpenClaw already supports deterministic user-invocable skill -> tool handoff through SKILL.md frontmatter (`command-dispatch: tool`, `command-tool`, raw arg forwarding), and that support is documented, implemented, tested, and present in the shipped `v2026.4.23` release.

## Evidence

- **Docs: skills frontmatter supports direct tool dispatch:** The skills reference documents `command-dispatch: tool`, `command-tool`, `command-arg-mode: raw`, and the forwarded payload `{ command, commandName, skillName }`, which matches the requested deterministic skill-to-tool bridge.
