## Summary

Keep open. Current main and the latest release still use the usage-counted transcript listing for Dreaming and still classify cron transcripts only by direct cron session keys; the linked full fix remains open and unmerged.

## What This Changes

The issue asks Dreaming session-corpus ingestion to stop re-ingesting reset/deleted archive transcripts and to exclude cron-descended subagent transcripts by following session parentage.

## Best Possible Solution

Land a focused fix that adds a Dreaming-only primary-transcript eligibility boundary and SDK-owned cron-descendant classification through stored lineage, while preserving reset/deleted archive indexing for memory_search and QMD.

## Reproduction Assessment

Yes. Current main is source-reproducible: Dreaming enumerates usage-counted transcript files and cron classification only checks direct cron session keys, so reset/deleted archives and cron-descended subagents can still enter the corpus path.

## Solution Assessment

Yes for the reported defects. A Dreaming-only archive filter plus lineage-aware SDK classifier is the narrowest maintainable direction; the archive-only sibling does not cover cron parentage, and changing the shared usage-counted helper would risk memory_search/QMD behavior.
