## Summary

Current main routes Telegram reply typing through the shared typing lifecycle, which stops the 3-second keepalive loop after two consecutive start failures. That means the reported minute-long `sendChatAction` retry spam no longer matches current `main` or the latest shipped release.

## Evidence

- **Telegram reply typing uses the shared typing pipeline:** The Telegram reply dispatcher passes `sendTyping` into the shared channel reply pipeline, so repeat `sendChatAction` behavior is governed by the generic typing lifecycle rather than a Telegram-only loop.
  - file: [extensions/telegram/src/bot-message-dispatch.ts:745](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/telegram/src/bot-message-dispatch.ts#L745)
  - command: `nl -ba extensions/telegram/src/bot-message-dispatch.ts | sed -n '720,790p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Shared typing lifecycle keeps a 3s interval but adds a 2-failure breaker:** `createTypingCallbacks` still uses a 3000ms keepalive, but now defaults `maxConsecutiveFailures` to 2 and stops the keepalive loop when the guard trips.
  - file: [src/channels/typing.ts:25](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/channels/typing.ts#L25)

  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Guard blocks further retries after consecutive failures:** `createTypingStartGuard` increments consecutive failures, trips once the max is reached, and then blocks future starts.
  - file: [src/channels/typing-start-guard.ts:46](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/channels/typing-start-guard.ts#L46)
  - command: `nl -ba src/channels/typing-start-guard.ts | sed -n '1,220p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Regression test asserts the loop stops after the second failed start:** `typing.test.ts` advances time past multiple keepalive windows and still expects only two start attempts after repeated failures.
  - file: [src/channels/typing.test.ts:110](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/channels/typing.test.ts#L110)
  - command: `nl -ba src/channels/typing.test.ts | sed -n '90,170p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
