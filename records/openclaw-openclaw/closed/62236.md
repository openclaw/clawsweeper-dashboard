## Summary

Keep open. Current `main` contains explicit mitigations and regression tests for `refresh_token_reused`, but there is no strong evidence that the reported cron/isolated-session failure is resolved in shipped builds, and the issue has a fresh corroborating report from `v2026.4.20`. That is not enough to conservatively close as implemented or cannot-reproduce.

## Evidence

- **Refresh path persists refreshed OAuth credentials:** `doRefreshOAuthTokenWithLock()` writes refreshed OAuth credentials back to `auth-profiles.json` via `saveAuthProfileStore(store, params.agentDir)` and mirrors sub-agent refreshes into the main agent store. This shows the intended fix path exists on current `main`, but code presence alone does not prove the reported failure is gone.

  - file: [src/agents/auth-profiles/oauth-manager.ts:528](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/agents/auth-profiles/oauth-manager.ts#L528)
