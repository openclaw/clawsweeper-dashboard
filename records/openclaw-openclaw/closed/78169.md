## Summary

Current main and the latest release already cover the requested cleanup path: `tasks maintenance --json` is preview-only, while `tasks maintenance --apply` prunes expired terminal records and the Gateway runs the same sweeper automatically.

## What This Changes

The issue asks for expired `lost` task records to be auto-pruned after `cleanupAfter` or for a deterministic `openclaw tasks reap-orphans` CLI.

## Best Possible Solution

Keep `openclaw tasks maintenance --apply` and the Gateway maintenance sweeper as the canonical cleanup path instead of adding a narrower duplicate orphan-only command.

## Reproduction Assessment

No. Source and docs show the reported `openclaw tasks maintenance --json` command is a dry-run preview; current main prunes via `openclaw tasks maintenance --apply` and the Gateway sweeper.

## Solution Assessment

Yes. The shipped maintenance engine is the narrowest maintainable path because it handles recovery, cleanup stamping, pruning, and Task Flow state through one operator command and one scheduled sweeper.
