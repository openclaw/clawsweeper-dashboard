## Summary

## What This Changes

The branch updates Telegram dispatch to drop text-only post-final tool payloads and tagged non-terminal warning finals after a delivered final answer, with regression tests for suppressed and preserved warning cases.

## Best Possible Solution

Land this only if maintainers accept the scoped Telegram message-delivery tradeoff, keeping the existing core metadata contract and the regression coverage for terminal warning/media/approval preservation.

## Reproduction Assessment

Yes. Source inspection shows current Telegram dispatch only suppresses `tool` payloads after final delivery starts, while core can emit a tagged `nonTerminalToolErrorWarning` final after assistant output; the PR tests model that path directly.

## Solution Assessment

Yes. The best fix is channel-local suppression keyed to the existing core metadata and gated on `finalAnswerDelivered` for warning finals, rather than broad `isError` suppression or a core-wide change that could affect other channels.
