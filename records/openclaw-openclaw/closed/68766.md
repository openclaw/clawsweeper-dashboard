## Summary

Keep open. Current main still has a single-shot Feishu WebSocket monitor with no OpenClaw-owned reconnect/backoff loop, and two open PRs explicitly target this issue with closing syntax. The best maintainer path is to review and land or close the active Feishu repair PR before closing this issue.

## Best Possible Solution

Keep this issue open. Review #72411 as the canonical maintainer repair, or explicitly close/supersede it if maintainers choose a different implementation. The fix should add an OpenClaw-owned Feishu WebSocket supervisor with bounded exponential backoff, observe `WSClient.start()`/connect failure state, clean per-cycle state safely, include the minimal heartbeat `wsConfig` repair if still needed, and add regression tests for startup token timeout and runtime disconnect recovery. Close this issue only after that PR or an equivalent fix lands on main.
