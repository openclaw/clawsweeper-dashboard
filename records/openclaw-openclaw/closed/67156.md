## Summary

Keep open. Current main still ships `skills/canvas/SKILL.md` without YAML frontmatter, while the active skill loader requires a non-empty frontmatter-derived `description` before a skill is loaded. The PR is a narrow metadata-only implementation candidate, and the final provided patch avoids the hard `darwin` OS gate called out in review.

## Best Possible Solution

Review and land this narrow PR, or apply the equivalent change on main: add `name`, `description`, and safe optional `metadata.openclaw` fields to `skills/canvas/SKILL.md` without an OS hard gate, and consider a small bundled-skill regression check so future skills cannot ship without required frontmatter.
