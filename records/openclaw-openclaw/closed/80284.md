## What This Changes

Prevent diagnostics-otel OTLPExporterError rejections from crashing OpenClaw when the OTLP collector is unavailable.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Yes. A diagnostics-otel-owned registerUnhandledRejectionHandler path is the narrowest maintainable fix because the OTLP dependency and benign exporter failure policy belong to the plugin, not the core transient allowlist.
