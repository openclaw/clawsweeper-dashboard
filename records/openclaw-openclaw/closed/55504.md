## Summary

The request is for the separate `clawhub install` workflow, not OpenClaw core. Current `main` in this repo implements native `openclaw skills install` plus a declarative, safety-checked `metadata.openclaw.install` path; it does not contain the separate ClawHub CLI, and its install model intentionally avoids arbitrary post-install command execution from `SKILL.md`. This belongs on the ClawHub side rather than `openclaw/openclaw`.

## Evidence

- **Repo scope: ClawHub vs native OpenClaw flows:** `docs/tools/clawhub.md:12-15` says to use native `openclaw` commands for skill install/update, while the separate `clawhub` CLI is for registry-authenticated workflows like publish/delete/sync. `docs/tools/clawhub.md:107-120` also describes the separate `clawhub` CLI as its own installer flow.
  - file: [docs/tools/clawhub.md:12](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/docs/tools/clawhub.md#L12)
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **OpenClaw-owned install command in this repo:** `src/cli/skills-cli.ts:94-116` shows the repo-owned CLI surface is `openclaw skills install <slug>`, which only downloads and installs the skill into the workspace and reports `Installed <slug>@<version> -> <targetDir>`.
  - file: [src/cli/skills-cli.ts:94](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/cli/skills-cli.ts#L94)

That indicates the standalone `clawhub install` command mentioned in the issue is not implemented in this repo.
