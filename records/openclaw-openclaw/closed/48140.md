## Summary

The proposal is a plausible community capability, but current `main` scope routes this kind of optional payment integration to external plugins/skills via ClawHub or MCP rather than OpenClaw core. The repository vision explicitly prefers plugins hosted in their own repos, prefers the existing MCP bridge instead of first-class core MCP runtime, and lists non-model-provider commercial integrations / new core skills as out of scope for core.

## Risks / Open Questions

- Closing as `clawhub` does not evaluate the safety or correctness of the proposed payment system itself; any future community plugin would still need its own security and operator-trust review.
- The issue body suggests ACPX per-session `mcpServers`, but current main only supports gateway- or agent-scoped MCP configuration, so maintainers may still want separate docs or guidance if this becomes a common integration request.
