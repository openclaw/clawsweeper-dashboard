## Summary

Keep open: centralizing the comparisons is a reasonable direction, but this diff sends caller-controlled token/signature strings into a helper that pads to the provided length before comparing. The supplied real-behavior proof covers the runtime-token path only, while the assistant media ticket path also changes.

## What This Changes

This PR replaces gateway runtime approval token and assistant media ticket `timingSafeEqual` comparisons with `safeEqualSecret` and adds a focused runtime-token unit test.

## Best Possible Solution

Use the shared helper only after fixed-format length/canonical guards, or introduce a bounded comparison helper, and cover oversized invalid values for both changed paths.

## Reproduction Assessment

Yes, source-reproducible: oversized `mediaTicket` signatures and `approvalRuntimeToken` strings reach `safeEqualSecret`, whose implementation pads to the larger input length before comparison. I did not run a live oversized-request probe in this read-only review.

## Solution Assessment

No. The centralization goal is reasonable, but the implementation should preserve fixed-length/canonical rejection before `safeEqualSecret` so fixed-format secrets do not inherit attacker-controlled padding work.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Bound media ticket signatures before comparing:** `src/gateway/control-ui.ts:425`
  - body: Here `sig` comes directly from the `mediaTicket` query parameter. `safeEqualSecret` pads both operands to the larger input length before `timingSafeEqual`, so replacing the old decoded-length check lets an oversized malformed ticket force extra allocation and scanning before rejection. Keep a fixed-length/base64url guard before the helper; the HMAC signature length is generated by OpenClaw and is not secret.
  - confidence: 0.87
- **[P2] Bound runtime approval tokens before comparing:** `src/gateway/operator-approval-runtime-token.ts:17`
  - body: `approvalRuntimeToken` is still an unbounded connect-frame string on current main. The old code rejected length mismatches before `timingSafeEqual`; this line now sends oversized pre-auth values through `safeEqualSecret`, which pads to the attacker-controlled length. Retain a fixed-length guard or protocol cap before calling the helper.
  - confidence: 0.84
