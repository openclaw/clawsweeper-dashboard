## Summary

Current main already fixes the reported v2026.4.24 Bonjour/CIAO crash-loop. The bundled Bonjour plugin now passes scoped CIAO rejection/exception handlers from the plugin runtime into the advertiser, the global unhandled-rejection path consults registered handlers before exiting, the classifier handles both `CIAO PROBING CANCELLED` and `CIAO ANNOUNCEMENT CANCELLED`, and the watchdog disables the advertiser after repeated stuck probing/announcing restarts instead of crash-looping the gateway. The affected v2026.4.24 code path only passed the logger into the advertiser, matching the report. No fixed release tag was found in this checkout, though the changelog has current-main entries for the fix.

## Best Possible Solution

Close #72420 as implemented on current main. Keep the Bonjour-owned scoped CIAO handling, watchdog self-disable behavior, and public Bonjour troubleshooting docs; no additional core change is needed for this reported v2026.4.24 crash-loop.
