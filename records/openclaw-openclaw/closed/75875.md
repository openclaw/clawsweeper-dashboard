## What This Changes

The PR moves hot Gateway/session transcript reads to async bounded readers and adds serialized async transcript append helpers across Gateway, agent, TUI, session config, and Codex app-server paths.

## Best Possible Solution

Leave this merged PR closed, then repair the branch-tail and usage-fallback regressions on `main` while keeping the async transcript I/O direction.

## Reproduction Assessment

Yes. The original performance problem is evidenced by #75656 and the pre-merge `readSessionMessages()` full synchronous read; the two PR regressions are reproducible with focused transcript fixtures for incomplete parent-linked tails and usage entries older than the last 256 KiB.

## Solution Assessment

No, not as merged. The async conversion is the right solution class, but the implementation should preserve active-branch reconstruction and uncapped usage fallback semantics before considering the linked issue fully fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Fall back when the tail chain is incomplete:** `src/gateway/session-utils.fs.ts:395-396`
  - body: Tree transcript leaves can be non-message entries such as `session_info`, model/thinking changes, or labels because SessionManager advances the leaf for all parent-linked entries. If that leaf is inside the bounded tail but its parent is outside the byte window, this `break` returns only the tail fragment, so recent visible message history can disappear. Fall back to the async full index or expand the tail when the parent chain is incomplete.
  - confidence: 0.82
- **[P2] Keep the uncapped usage fallback:** `src/gateway/session-utils.ts:486-492`
  - body: When no `maxTranscriptBytes` is requested, current main scans the full transcript for latest usage. Defaulting this fallback to the last 256 KiB drops `totalTokens` and cost whenever the newest usage-bearing assistant entry is older than a large recent tail, so session rows and status can lose valid usage metadata.
  - confidence: 0.86
