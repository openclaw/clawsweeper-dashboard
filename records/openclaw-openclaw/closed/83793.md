## Summary

Current main and the v2026.5.18 release already address the reported old-release Telegram long-poll hang: getUpdates is hard-bounded, polling stalls force a transport rebuild/restart, and status only becomes connected after a successful getUpdates response.

## What This Changes

The issue asks Telegram long polling to stop hanging indefinitely on macOS, honor timeout/retry behavior, and avoid reporting connected while getUpdates is wedged.

## Best Possible Solution

Keep the shipped v2026.5.18+ Telegram polling implementation and direct affected 2026.5.7 users to upgrade before opening a new report with fresh logs.

## Reproduction Assessment

No current-main failing reproduction was established. The issue body gives an old-version 2026.5.7 repro, while current source, tests, docs, and the v2026.5.18 tag show the reported hang/status behavior is already addressed.

## Solution Assessment

Yes. The fix is in the Telegram plugin-owned polling path, with bounded getUpdates requests, getUpdates-based liveness, transport rebuild on stalls, and status based on successful polling rather than a broad core workaround.
