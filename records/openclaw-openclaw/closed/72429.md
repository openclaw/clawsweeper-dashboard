## Summary

Current main already has a MiniMax-specific runtime fix for the reported visible looping/duplicated dialogue class: MiniMax Anthropic-compatible streams are wrapped so OpenClaw injects `thinking: { type: "disabled" }` unless explicitly overridden, preventing MiniMax `reasoning_content` deltas from leaking into visible assistant output. The behavior is covered by focused tests and documented for the MiniMax provider; no release tag currently contains the fixing commit.

## Best Possible Solution

Close this issue as implemented on main and keep the shipped MiniMax stream-wrapper behavior. If users can still reproduce after a release containing ff9fefb7, open a fresh focused bug with current version, exact provider id (`minimax` or `minimax-portal`), model ref, thinking/fast settings, and logs/screenshot from that build.
