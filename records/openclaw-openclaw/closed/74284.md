## Summary

Keep open. Current main still has the heartbeat stored-override path that can reuse a stale auto-fallback model, and open PR #74932 is already linked with closing syntax, so this issue should remain open until that PR lands or is superseded.

## What This Changes

The issue asks heartbeat runs without an explicit `heartbeat.model` to stop reusing a stale auto-fallback session model after `openclaw.json` default model changes and gateway restarts.

## Best Possible Solution

Review and land or supersede #74932 with a narrow stored model-selection fix that clears stale direct heartbeat auto-fallback overrides after default-model changes while preserving explicit heartbeat, user, parent, legacy, and unchanged-default overrides.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a heartbeat run without `heartbeat.model` and with an existing direct `modelOverrideSource: "auto"` session override from an older fallback; current main applies that stored override because the explicit heartbeat override flag stays false.

## Solution Assessment

Yes in direction. The narrow maintainable fix belongs in the stored model-selection seam rather than a startup migration or blanket session reset, and PR #74932 is already the active linked implementation path.
