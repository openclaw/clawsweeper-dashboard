## Summary

Keep open: current main still warns unconditionally on compile-cache prune permission errors, and open PR #76362 is the active unmerged fix candidate.

## What This Changes

Suppress expected EACCES/EPERM warnings when postinstall prunes shared Node compile-cache directories while preserving warnings for unexpected prune failures.

## Best Possible Solution

Land PR #76362 or an equivalent narrow patch with regression coverage for EACCES/EPERM suppression while preserving unexpected prune warnings.

## Reproduction Assessment

Yes, source-backed. pruneOpenClawCompileCache currently routes readdirSync and rmSync permission failures to unconditional warning calls, though I did not run the root/non-root global install sequence.

## Solution Assessment

Yes. A narrow errno guard in the compile-cache prune catch paths is the maintainable fix; changing cache ownership or default cache locations would be broader than this cosmetic postinstall bug requires.
