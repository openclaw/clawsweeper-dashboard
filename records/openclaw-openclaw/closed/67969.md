## Summary

Current `main` already fixes the reported web UI `billing`/`surface_error` behavior through a different path: `surface_error` cases now throw a `FailoverError` upstream, and the existing `chat.send` rejection handler already broadcasts the terminal chat error to the web UI. This makes the PR’s lifecycle-timer patch unnecessary on current `main`.
