## Summary

Current `main` already uses the session context snapshot (`totalTokens`) for the chat warning, not accumulated `inputTokens`, and it includes a regression test for the inflated-count case described in the issue. That behavior is also present in the shipped `v2026.4.22` release.

## Evidence
