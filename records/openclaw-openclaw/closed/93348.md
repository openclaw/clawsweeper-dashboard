Full review comments:

- **[P2] Cover provider-qualified session overrides:** `src/status/status-text.ts:384-385`
  - body: This switches selected-usage routing to `selectedStatusProvider`, but that value is still computed from the caller `provider` rather than parsing `sessionEntry.modelOverride` when `providerOverride` is absent. Existing session logic supports provider-qualified overrides such as `modelOverride: "openai/gpt-5.5"`, so those sessions can still route usage to the configured/default provider instead of the selected model provider. Please parse the selected override provider before choosing the usage provider, or consolidate this fix with the sibling resolver change.
