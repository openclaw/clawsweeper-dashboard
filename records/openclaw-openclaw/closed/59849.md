## Summary

Current `main` already covers the requested fix path: OpenClaw documents config-driven remote CDP browser profiles, including a Browserbase example with residential proxies/CAPTCHA-solving, and local managed Chromium accepts proxy flags through `browser.extraArgs`, which are passed through to Chrome and covered by tests.

## Risks / Open Questions

- OpenClaw still cannot prevent third-party sites from blocking datacenter IP reputation on their side; the implemented answer is supported proxy/remote-browser configuration, not a core bypass of Cloudflare/CAPTCHA decisions.
- There is still no dedicated first-class `browser.proxyUrl` key; the supported solutions are remote CDP/browser services or Chrome launch flags via `browser.extraArgs`.
