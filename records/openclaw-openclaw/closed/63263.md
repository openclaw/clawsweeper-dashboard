## Summary

Keep this PR open. Current main has already superseded the local model-run session collision part with the lean local completion path and already passes agentDir for image description, but it still does not implement the remaining PR goals: `image describe --prompt`, structured web-search fallback for missing-key payloads, or `openai-codex` image autoPriority. The PR also needs rebase/adaptation because its broad structured-error fallback can mask provider validation errors.

## Best Possible Solution

Keep this PR open as an implementation candidate. Rebase it onto current main, drop or adapt the obsolete local session-key change, add and thread `image describe --prompt` and `image describe-many --prompt` with request-scoped precedence, narrow structured web-search fallback to provider availability/auth failures, add `openai-codex` image autoPriority in current manifest/runtime provider metadata if desired, then rerun the targeted CLI, web-search, and media-understanding tests before maintainer review.
