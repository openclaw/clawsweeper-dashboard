## Summary

Current main already satisfies the central Mac desktop-control request through the bundled Peekaboo skill plus OpenClaw.app PeekabooBridge hosting. It is not the exact proposed Codex ACP computer-use flag, but it covers screen inspection, UI element targeting, input, app/window/menu control, and permission-aware local brokerage; the related #41024 cleanup reached the same implemented-on-main conclusion.

## Best Possible Solution

Keep the shipped Peekaboo/PeekabooBridge macOS desktop automation path as OpenClaw's current answer for screen-level Mac tasks. If maintainers later want provider-native Codex Computer Use specifically, track it as a narrower Codex/acpx integration request rather than keeping this broad desktop-control issue open.
