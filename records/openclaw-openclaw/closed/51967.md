## Summary

Current `main` already fixes the underlying zero-duration false positive on `message.send`, and that behavior was shipped in `v2026.4.23` via a different implementation than this PR. The remaining difference is intentional: current code still treats negative non-zero durations as explicit poll intent so invalid poll durations stay on the poll validation path.
