## Summary

Current main already exposes a public pre-LLM plugin hook for synthetic replies, and repo vision says optional capability like rule-based auto-responses should ship as plugins/ClawHub rather than core.

## Risks / Open Questions

- This closes the core feature request, not the underlying idea; someone still has to ship and maintain the plugin.
- There is no built-in declarative `agents.defaults.quickReplies` config on current main; the supported path is plugin implementation via hooks.

## Close Comment

Closing from a Codex review as `clawhub`, not because the idea is bad. Current `main` already has the right extension seam for this: `before_agent_reply` runs after inline actions and before any LLM call, and a plugin can claim the turn with a synthetic `ReplyPayload` (`src/auto-reply/reply/get-reply.ts:567`, `src/plugins/hook-types.ts:167`, `docs/concepts/agent-loop.md:96`). That makes exact-match or regex-style ping/health-check replies a good community plugin fit with zero model tokens on matched turns. Repo direction also says optional capability should usually ship as plugins and new skills should go to ClawHub first (`VISION.md:54`, `VISION.md:68`).
