## Summary

Keep open. Current `main` still does not propagate `browser.ssrfPolicy` through the sandbox browser bridge path, so this PR is not implemented on `main` and the reported gap remains real. The only caution is that bridge reuse still keys on container/CDP port, so a policy change would likely need an extra invalidation check beyond this PR's basic plumbing.

## Risks / Open Questions

- If this PR lands unchanged, bridge reuse still appears to ignore SSRF policy changes because `shouldReuse` only compares container identity and mapped CDP port in `src/agents/sandbox/browser.ts:349`.
- There is no current sandbox-specific test asserting that `ensureSandboxBrowser` passes `ssrfPolicy` into the bridge-resolved config, so this regression can recur silently.
