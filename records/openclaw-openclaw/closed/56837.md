## Summary

Close as implemented: current main and `v2026.4.27` expose model-provider request proxy configuration, attach it to configured/inline model providers, route supported LLM transports through guarded fetch with explicit or environment proxy dispatchers, and test the supported API matrix. The dependency contract also supports SOCKS5 because OpenClaw uses undici `ProxyAgent`, whose v8.1.0 source routes `socks5:`/`socks:` proxy URLs through `Socks5ProxyAgent`.

## What This Changes

Add SOCKS5/HTTP proxy support for model-provider requests.

## Best Possible Solution

Keep the shipped model-provider request proxy surface as the supported path, and use narrower follow-up issues only for specific provider-owned transports that are not covered by the current transport-aware API matrix.
