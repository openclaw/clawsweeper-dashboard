## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Preserve stopAndWait before unrefing the socket:** `src/gateway/client.ts:408`
  - body: `beginStop()` also backs `stopAndWait()`, which current code and docs treat as waiting for the close event or terminate fallback. Since Node may exit when an unref'd socket is the only active handle, and both shutdown timers are also unref'd, this unconditional call can make a standalone `await client.stopAndWait()` exit before the close/terminate path runs. Keep the socket referenced for waitful shutdown, or make unref specific to the fire-and-forget `stop()` path.
  - confidence: 0.86
