## Summary

Close #44449 as a duplicate of the earlier open tracker #41355. The reported Discord thread-starter/context re-injection is still a real remaining behavior on current main, but #41355 already tracks the same Discord ThreadStarterBody re-injection problem with the same Slack comparison and suggested guard.

## Evidence

- **Canonical open tracker:** #41355 is an earlier open issue titled "Discord: ThreadStarterBody re-injected on every turn (echo contamination)". Its body describes the same Discord ThreadStarterBody being set unconditionally on every inbound thread message, causing repeated `[Thread starter - for context]`, token growth, and echo contamination, and it cites the same Slack fix pattern from #32133.
