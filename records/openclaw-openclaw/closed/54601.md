## Summary

The request is a useful workflow improvement, but current main positions it as plugin/hook-pack territory rather than OpenClaw core. OpenClaw’s memory model is intentionally file-backed with no hidden session state, and the repo already exposes hook/plugin seams to capture session decisions and inject bootstrap context for isolated runs.
