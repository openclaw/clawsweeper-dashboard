## Summary

## What This Changes

The PR changes model catalog `stripPrefixes` stripping to slice by `normalizedPrefix.length` instead of the raw `prefix.length`.

## Best Possible Solution

Land one canonical model-catalog fix that strips by the raw trimmed prefix span or another explicit matched-span mapping, adds focused regression tests, and coordinates with the overlapping guard PR before merge.

## Solution Assessment

No. The helper is the right layer, but the best fix should strip by the original matched span, not by the fully lowercased normalized string length.
