## Summary

Keep this PR open. The Slack-owned direction is right and fixes the common stale direct-field case, but the latest patch still makes the prefetch gate diverge from the canonical session reset path for legacy session rows, and it is missing the required changelog entry.

## What This Changes

The PR changes Slack thread-context preparation to load thread history for stale daily/idle thread sessions, passes config into the resolver, reexports session-store runtime helpers, and adds targeted regression tests.

## Best Possible Solution

Land the Slack stale-session fix after the prefetch gate uses the same lifecycle timestamp resolution as `initSessionState`, adds a legacy `sessionFile` header regression test, and includes a changelog bullet.

## Reproduction Assessment

Yes. The original bug is reproducible with a Slack thread session crossing a daily or idle reset while retaining a stored timestamp; the remaining PR gap is reproducible with a legacy session entry whose `updatedAt` is fresh, direct lifecycle fields are absent, and the JSONL header predates the reset boundary.

## Solution Assessment

No, not yet. The owner-module fix is the right shape, but the implementation should reuse or expose lifecycle timestamp resolution so Slack history prefetch and session initialization agree for both current and legacy rows.
