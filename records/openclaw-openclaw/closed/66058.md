## Summary

Current `main` already contains the reported memory-flush compaction guard and persists `memoryFlushCompactionCount`/`memoryFlushAt` after an in-flush compaction. Targeted unit coverage on `main` simulates the compaction-inside-flush path and asserts the session store fields the report said were staying null. The fix appears to be in commit `628f0e80553fe6d0d1d755eb2fef4679b0e2732b`, which is newer than the latest tagged release, so `fixedRelease` is unknown/null.
