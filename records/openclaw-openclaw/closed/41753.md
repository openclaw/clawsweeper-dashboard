## Summary

Current main covers the observable failure mode: Telegram DMs using `dmPolicy: "pairing"` and `allowFrom` are authorized by numeric sender id after `tg:` prefix normalization, and the polling path persists accepted update ids before long-running handler completion with tests for retry/replay behavior. The related #25793 reference was an allowlist-specific drop report, while this issue uses the pairing path that is now directly covered.
