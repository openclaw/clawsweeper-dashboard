## Summary

## What This Changes

Request to fix Slack native slash-command routing when multiple Slack Apps in one workspace register the same built-in command names for different agents.

## Best Possible Solution

Use #38302 as the canonical lane: either implement a deliberate account-specific Slack native naming/prefix mapping across config, spec generation, lookup, argument-menu callbacks, docs, and tests, or document that identical Slack native names across multiple Slack Apps are unsupported.

## Reproduction Assessment

Yes. The reporter provides concrete multi-app Slack steps, and source inspection gives a high-confidence static reproduction path: current native Slack command names are generated and looked up by provider while Slack config has no per-account native-name field.
