## Summary

The specific Google Chat group-session-key claim does not match current main or the v2026.5.7 release source: group spaces are routed as `group` peers and shared routing builds a Google Chat group session key, not the agent main key.

## What This Changes

The report asks for Google Chat group webhook messages to create `agent:<agentId>:googlechat:group:<spaceId>` sessions so replies return to Google Chat instead of another channel.

## Best Possible Solution

Keep the existing Google Chat group routing path; if a deployment still sees a main session key, handle it as a fresh runtime/package mismatch report with raw routing and session logs rather than changing the current source path speculatively.

## Reproduction Assessment

No. Current main and the v2026.5.7 release source both route non-DM Google Chat spaces through a `group` peer into the shared session-key builder, which does not produce the agent main key for groups.

## Solution Assessment
