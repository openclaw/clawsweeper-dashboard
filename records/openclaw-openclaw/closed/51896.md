## Summary

Close as cannot_reproduce. The report was resolved by freeing localhost:1455, and current main shows the OpenAI Codex OAuth flow preserves the authorize URL and intentionally falls back to manual redirect/code entry when the callback port is unavailable.

## Evidence

- **Reporter self-resolved:** In the issue follow-up, the reporter said they missed that port 1455 was already in use, killed that process, and the flow was fixed.
- **Docs describe bind-failure fallback:** The OpenAI Codex OAuth docs explicitly say the flow tries http://127.0.0.1:1455/auth/callback and, if the callback cannot bind, the user should paste the redirect URL or code.
  - file: [docs/concepts/oauth.md:114](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/docs/concepts/oauth.md#L114)

  - file: [src/plugins/provider-oauth-flow.ts:24](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/src/plugins/provider-oauth-flow.ts#L24)
