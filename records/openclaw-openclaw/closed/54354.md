## Summary

Keep open. Current main has a shipped `openclaw.install.minHostVersion` compatibility gate, but it does not implement this PR's intended `engines.openclaw` registry resolution or older-compatible npm version selection, and the proposed patch still has blocking selector, exact-pin, malformed-range, and external-metadata issues before it is safe to merge.

## What This Changes

The PR adds `engines.openclaw`-aware npm version resolution for plugin install/update, an `--ignore-engine` CLI flag, load-time compatibility warnings, custom semver/registry helpers, and related tests.

## Best Possible Solution

Decide the public plugin compatibility contract first. If `engines.openclaw` is accepted, rework the branch so npm tags, exact versions, prereleases, invalid ranges, non-release host versions, and external registry metadata preserve user intent and fail safely with focused install/update/load tests; otherwise keep the shipped `openclaw.install.minHostVersion` and ClawHub compatibility model as the supported path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Honor explicit dist-tags during engine resolution:** `src/plugins/install.ts:743-747`
  - body: For specs like `pkg@beta`, `parseRegistryNpmSpec` marks the selector as a tag and `selectorIsPrerelease` stays false. This resolver then scans all versions, excludes prereleases, and rewrites the request to an exact stable version, so a user asking for the beta channel can silently get a different channel.
  - confidence: 0.9
- **[P1] Block incompatible exact-version installs:** `src/plugins/install.ts:743`
  - body: Exact-version specs skip the new engine resolver entirely, so `pkg@1.2.3` can still install even when that version declares an incompatible `engines.openclaw` range. The new `--ignore-engine` flag implies incompatible exact pins should be blocked unless explicitly bypassed.
  - confidence: 0.86
- **[P1] Reject malformed engine ranges:** `src/infra/semver-range.ts:237-238`
  - body: When a non-empty branch has no tokenized comparators, `allComparators.every(...)` returns true. Unsupported ranges such as `>=2026.3` or `foo` therefore match every host version, bypassing the install/update gate and suppressing load warnings for bad metadata.
  - confidence: 0.92
- **[P2] Validate registry engine values before resolving:** `src/infra/npm-registry-versions.ts:82-87`
  - body: The registry `engines` object is cast to `Record<string, string>` without checking `engines.openclaw`. A package that publishes a non-string value can pass a number into `satisfiesRange`, which calls `range.trim()` and throws during install/update instead of treating the metadata defensively.
  - confidence: 0.88
- **[P2] Preserve prerelease lower bounds for caret and tilde ranges:** `src/infra/semver-range.ts:158-164`
  - body: Caret and tilde parsing replaces the lower-bound prerelease with `null`, so `^2026.3.14-beta.1` becomes `>=2026.3.14` instead of including the beta lower bound. That diverges from npm semver behavior and can reject or select the wrong versions for prerelease engine constraints.
  - confidence: 0.82
- **[P3] Suppress load warnings for unparsable host versions:** `src/plugins/loader.ts:987-995`
  - body: The loader only checks that `VERSION` is truthy before calling `satisfiesRange`. Local or development builds with a non-semver version string will fail every constrained range and warn for all constrained plugins even when compatibility is unknown rather than incompatible.
  - confidence: 0.8

## Security Review

Status: needs_attention

Summary: Security review needs attention because the diff introduces npm registry metadata reads and client-side version selection with concrete selector-mismatch and fail-open compatibility risks.

Concerns:

- **[medium] Explicit npm dist-tags can be overridden:** `src/plugins/install.ts:743`
  - body: The resolver treats tag selectors as broad package scans and can rewrite a user-requested tag such as `@beta` to a different exact stable version, changing the requested release channel during plugin installation or update.
  - confidence: 0.9
- **[medium] Exact incompatible versions bypass the engine gate:** `src/plugins/install.ts:743`
  - body: Exact-version specs skip engine resolution, so a package version with incompatible `engines.openclaw` metadata can still install without using the new bypass flag.
  - confidence: 0.86
- **[medium] Malformed compatibility ranges fail open:** `src/infra/semver-range.ts:237`
  - body: Unsupported `engines.openclaw` ranges can evaluate as compatible because an empty comparator list passes `every()`, bypassing the new install/update gate and hiding load warnings.
  - confidence: 0.92
- **[low] External registry metadata is not parsed defensively:** `src/infra/npm-registry-versions.ts:82`
  - body: The npm registry `engines.openclaw` value is trusted as a string, so malformed metadata can throw during install/update instead of producing a controlled compatibility result.
  - confidence: 0.88
