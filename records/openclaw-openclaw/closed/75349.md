## Summary

Current main now addresses the harmful Copilot failure mode: Copilot setup defaults to Opus 4.7, live account-specific `/models` discovery is wired in, and `format` rejections no longer poison shared auth-profile cooldown state. The remaining static Opus 4.6 fallback row is intentional broad catalog behavior because Copilot availability varies by account.

## What This Changes

The issue asks OpenClaw to stop advertising an unsupported GitHub Copilot `claude-opus-4.6` default and to prevent a model-specific Copilot rejection from cooling down the whole provider profile.

## Best Possible Solution

Keep the current provider-owned live catalog and auth-profile cooldown behavior; do not globally delete a Copilot model row that can still be account-specific while live discovery and docs handle availability.

## Reproduction Assessment

No. The original provider-wide cooldown path no longer reproduces on current main because `format` rejections are filtered before shared auth-profile health is updated, and Copilot discovery now uses account-specific live catalog data when available.

## Solution Assessment

Yes. The current solution is more maintainable than hard-deleting the row: it moves defaults to Opus 4.7, uses Copilot's live catalog for entitlement-specific availability, and prevents unsupported-model/request-shape errors from cooling down the whole provider.
