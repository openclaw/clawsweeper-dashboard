## Summary

Keep open: the opt-in direction matches maintainer guidance and the linked bug is real, but the current patch can still raw-reseed a missing-transcript reset before auth-profile or credential-epoch mismatches are checked.

## What This Changes

This PR adds a `reseedFromRawTranscriptWhenUncompacted` CLI backend config flag, threads it into CLI session reseed preparation, and updates tests, docs, changelog, and generated config baseline metadata.

## Best Possible Solution

Land the opt-in raw-tail reseed only after prepare-level auth-boundary coverage proves raw history is never sent across auth profile or credential-epoch changes.

## Reproduction Assessment

Yes from source. Current main makes uncompacted reseed empty, and the PR diff shows the new raw-reseed path can classify a combined missing-transcript plus auth-change case as `missing-transcript` before auth checks run.

## Solution Assessment

No as currently patched. The explicit opt-in is the narrow maintainable direction, but auth-profile and auth-epoch invalidations need to win before any missing-transcript raw reseed is prepared.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve auth-boundary checks before raw reseed:** `src/agents/cli-runner/prepare.ts:411-413`
  - body: When a saved Claude session has both a missing transcript and changed auth identity, `prepareCliRunContext` classifies it as `missing-transcript` before `resolveCliSessionReuse` can return `auth-profile` or `auth-epoch`. With the new allow-list, that sends raw prior transcript into a fresh CLI run under changed credentials, despite the intended no-raw auth boundary; resolve auth reuse first or suppress raw reseed for this combined case.
  - confidence: 0.88
