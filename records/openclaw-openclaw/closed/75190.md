## Summary

Close as implemented on current main. The reported v2026.4.27 Telegram DM disappearance matches the native draft-preview path that used Telegram `sendMessageDraft` and later cleared/materialized the draft; main now removes that native draft transport and uses normal `sendMessage`/`editMessageText` previews with regression coverage for DM preview finalization. The fix is on main only; no release tag contains it yet.

## What This Changes

Telegram bot DM replies intermittently vanished after upgrading from 2026.4.15 to 2026.4.27, likely around streaming preview edit/delete cleanup.

## Best Possible Solution

Keep the current main implementation that avoids Telegram native draft previews for DM streaming, ship it in the next release, and leave the separately linked quote-context regression to its own review if it still reproduces.

## Reproduction Assessment

Yes. There is a high-confidence code-level reproduction path in v2026.4.27: DM preview streaming could use `sendMessageDraft`, then materialize a normal message and clear the draft, which matches a message-like preview vanishing after it was shown.

## Solution Assessment

Yes. Removing the native draft preview transport is the narrowest maintainable fix because it keeps preview streaming on normal Bot API messages with stable message ids instead of relying on draft cleanup semantics.
