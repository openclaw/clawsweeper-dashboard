The issue asks for correctly configured `memory-lancedb` auto-recall and auto-capture typed hooks to fire in embedded PI, cron, restart, and hot-reload paths.

## Best Possible Solution

Keep the generic gateway startup hook-scope implementation and ship it in the next release; no memory-lancedb-specific workaround is needed.

## Reproduction Assessment

No against current main. The original v2026.5.2 report is credible, but current source and tests show the configured memory and hook-policy plugins are now loaded into the gateway-bindable runtime before embedded PI checks `hasHooks`.

## Solution Assessment

Yes. The current generic startup-scope fix is the narrow maintainable solution because it keeps selected and hook-capable plugins visible to embedded PI without hardcoding `memory-lancedb` or a channel.
