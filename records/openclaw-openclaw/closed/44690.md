## Summary

Current main already blocks the webchat/control-ui route inheritance path that would cause a webchat turn in an iMessage-scoped session to reuse the stored iMessage delivery route, and it has regression coverage for that exact scenario.

## Evidence

- **Webchat route inheritance guard:** `resolveChatSendOriginatingRoute` now refuses to inherit stored external delivery routes for webchat clients on channel-scoped sessions, returning internal `webchat` origin instead of the session’s external channel route.
  - file: [src/gateway/server-methods/chat.ts:554](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/gateway/server-methods/chat.ts#L554)
- **Regression test for iMessage-scoped session:** Test covers `sessionKey: "agent:main:imessage:direct:+8619800001234"` and asserts a webchat client gets `OriginatingChannel: "webchat"`, `OriginatingTo: undefined`, and `ExplicitDeliverRoute: false`, which blocks the duplicate iMessage delivery path described in the issue.
  - file: [src/gateway/server-methods/chat.directive-tags.test.ts:1314](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/gateway/server-methods/chat.directive-tags.test.ts#L1314)
- **Shipped changelog entry:** Changelog documents this behavior as `Gateway/webchat route safety: block webchat/control-ui clients from inheriting stored external delivery routes on channel-scoped sessions`, matching the reported cross-channel leakage pattern.
  - file: [CHANGELOG.md:3218](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/CHANGELOG.md#L3218)
- **Present in latest release tag:** `git grep` against `v2026.4.22` shows both the route-safety code and the regression test are included in the latest release tag, so this is not just unreleased main-only behavior.
  - command: `git grep -n "Webchat clients never inherit external delivery routes" v2026.4.22 -- src/gateway/server-methods/chat.ts; git grep -n "chat.send does not inherit external routes for webchat clients on channel-scoped sessions" v2026.4.22 -- src/gateway/server-methods/chat.directive-tags.test.ts`
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
