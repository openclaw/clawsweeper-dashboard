## Summary

Close as ClawHub/plugin scope: current main already exposes the lifecycle and prompt-injection hooks needed for this, and OpenClaw's scope guidance keeps optional memory policy out of core unless a missing plugin API is identified.

## What This Changes

Request adding a session-start memory recall hook/config that scans saved memory files and injects them into new session context.

## Best Possible Solution

Publish the automatic memory recall policy as a ClawHub code plugin using session_start and before_prompt_build, while keeping core focused on stable plugin hooks and the existing startupContext prelude.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection is enough to show the existing plugin surfaces can support an external implementation.

## Solution Assessment

Yes. A ClawHub code plugin is the narrowest maintainable path because it can combine session_start with before_prompt_build, while core already provides startupContext for the built-in bare /new and /reset daily-memory prelude.
