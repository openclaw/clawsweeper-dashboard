## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Gate compaction transcript messages:** `src/agents/pi-embedded-runner/compaction-hooks.ts:212`
  - body: This sends full transcript `messages` through `before_compaction`, but that hook is not part of `CONVERSATION_HOOK_NAMES`, so non-bundled plugins can register it without `hooks.allowConversationAccess=true`. Either put `before_compaction` behind the same conversation-access gate with docs/tests, or keep this hook metadata-only for plugins that lack that authorization.
  - confidence: 0.88
- **[P2] Keep compaction counts and messages aligned:** `src/agents/pi-embedded-runner/compaction-hooks.ts:211-212`
  - body: `messageCount` still comes from `params.metrics.messageCountBefore`, which is the post-limit in-memory compaction input, while `messages` now comes from the full on-disk transcript. When history limiting has trimmed the compaction input, plugins can receive `messages.length` far larger than `messageCount`, so analytics and budgeting consumers will read an incoherent event.
  - confidence: 0.82
- **[P2] Use a scoped active transcript for agent_end:** `src/agents/pi-embedded-runner/run/attempt.ts:1757-1763`
  - body: `agent_end` now prefers an async read of `params.sessionFile` after the run finishes instead of the run-scoped `messagesSnapshot`. If the active transcript is rotated or appended while the fire-and-forget chain is pending, the hook can observe messages outside the completed run's finalized snapshot; hydrate provenance into the scoped snapshot or capture the active transcript before releasing the attempt.
  - confidence: 0.72
