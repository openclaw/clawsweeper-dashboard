## Summary

Current main already treats workspace Markdown bootstrap files, including TOOLS.md, as UTF-8 text. The relevant loader builds a fixed allowlist of bootstrap Markdown filenames, reads file bytes into a string, and passes that string into prompt-context trimming without JSON parsing. The same behavior is present in v2026.4.24.

## Best Possible Solution

Close this broad workspace-bootstrap report as already handled. Keep the current plain-text Markdown bootstrap implementation; if the reporter can provide a stack trace showing a different JSON parser is consuming Markdown, track that as a narrower bug against the specific config, tool-call, or provider path.
