## Best Possible Solution

Treat stop during microphone setup as cancellation: clean up the newly resolved stream/peer, leave the UI idle, and prove the real Control UI stop/cancel path in a browser.

## Reproduction Assessment

## Solution Assessment

No. The guard is in the right file, but throwing after a user stop is not the best fix because the caller turns that cancellation into an error state; the closed path should return after cleanup.
