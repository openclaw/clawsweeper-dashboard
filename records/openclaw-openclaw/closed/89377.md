## Summary

Close as implemented on current main: the TUI `/new` path now creates an empty TUI session through `setEmptySession`, and that helper resets `sessionInfo.displayName` before the footer renders. I could not prove the fix shipped in `v2026.5.28`, so this is current-main-only cleanup.

## What This Changes

The issue requests that TUI `/new` clear stale `sessionInfo.displayName` so the footer does not keep the previous session title.

## Best Possible Solution

Keep the current `setEmptySession` behavior for TUI `/new`; if generic manual session switching shows the same stale-title symptom, track that as a separate focused bug.

## Reproduction Assessment

No, not against current main: `/new` now routes through `setEmptySession`, which clears `displayName` before footer rendering. The issue was source-reproducible against the older quoted path, but that path is no longer current.

## Solution Assessment

Yes for the reported `/new` bug: the empty-session helper is the right boundary because it resets fresh TUI session state in one place. A separate generic `setSession` cleanup would only be needed if manual session switching is independently reported.
