## Summary

Close this PR as implemented on main. The same user-visible fix for internal non-delivery agent channel hints was merged through #73282, is present in current main, has broader shared-constant and test coverage than this branch, and shipped in v2026.4.27.

## What This Changes

This PR accepts cron, heartbeat, and webhook as internal non-delivery channel hints in gateway agent validation, keeps them out of delivery routing, and adds a gateway-server regression test.

## Best Possible Solution

Keep the shipped #73282 implementation: the internal source-channel concept belongs in the shared message-channel constants, gateway validation should accept only the named non-delivery hints plus existing gateway channels, and routing should continue to treat those hints as non-deliverable provenance rather than outbound destinations.

## Reproduction Assessment

Yes. The linked bug reproduces by sending an agent or sessions_spawn request with `channel` or `replyChannel` set to `heartbeat`, `cron`, or `webhook`; current main has regression tests for those fields and the rejection path for `not-a-real-channel`.

## Solution Assessment

No, this PR is no longer the best merge target. The already-merged #73282 solution covers the same behavior with shared constants, tests for both hint fields, unknown-channel fail-closed coverage, and release provenance.
