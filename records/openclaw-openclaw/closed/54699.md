## Summary

Current `main` does not match the reported cached/no-op behavior. The macOS Settings view reruns a fresh gateway check on appear and on `Recheck`, the local version probe resolves executables from explicit search paths instead of relying on a GUI-shell `command -v`, and the reporter’s own follow-up narrowed the original repro to local environment issues rather than WKWebView cache persistence.

## Evidence

- **`Recheck` is wired to a fresh status refresh:** `GeneralSettings` refreshes gateway status on appear and the `Recheck` button calls `refreshGatewayStatus()`, which reruns `GatewayEnvironment.check()`.
  - file: [apps/macos/Sources/OpenClaw/GeneralSettings.swift:420](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/apps/macos/Sources/OpenClaw/GeneralSettings.swift#L420)
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Gateway check is recomputed, not read from WebView state:** `GatewayEnvironment.check()` resolves runtime/CLI each call, and `readGatewayVersion` runs the binary with `--version` through `Process` and parses live output.
  - file: [apps/macos/Sources/OpenClaw/GatewayEnvironment.swift:103](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/apps/macos/Sources/OpenClaw/GatewayEnvironment.swift#L103)
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Local macOS CLI resolution uses explicit path probing:** `CommandResolver.preferredPaths()` includes `/opt/homebrew/bin`, `/usr/local/bin`, `/usr/bin`, `/bin`, OpenClaw-managed bins, and node-manager bins, and `openclawExecutable()` uses direct filesystem lookup instead of a shell `command -v` for local resolution.
  - file: [apps/macos/Sources/OpenClaw/CommandResolver.swift:86](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/apps/macos/Sources/OpenClaw/CommandResolver.swift#L86)
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Current release matches current main for the relevant macOS files:** No diff between `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`) and current `main` for `apps/macos/Sources/OpenClaw/GeneralSettings.swift`, `apps/macos/Sources/OpenClaw/GatewayEnvironment.swift`, and `apps/macos/Sources/OpenClaw/CommandResolver.swift`.

- **Reporter follow-up retracted the cache diagnosis:** The issue comments state the original repro was traced to a stray `~/package.json` version conflict plus GUI PATH resolution problems, explicitly saying it was not a WKWebView cache issue.
