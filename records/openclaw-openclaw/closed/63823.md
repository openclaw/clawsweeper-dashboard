## Summary

Keep open. Current main still lacks the MiniMax manifest auth-alias mapping while auth lookup is manifest-backed; open PR #72798 references this issue with closing syntax and remains the active implementation path.

## What This Changes

Add MiniMax `providerAuthAliases` for `minimax-cn` and `minimax-portal-cn` so CN endpoint variants reuse the base MiniMax auth providers.

## Best Possible Solution

Land #72798 or apply an equivalent maintainer patch that adds the MiniMax manifest aliases with focused regression coverage while keeping the generic auth resolver manifest-backed.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: current main has `minimax-cn` runtime aliases but no MiniMax manifest `providerAuthAliases`, so `resolveProviderIdForAuth("minimax-cn")` falls back to the alias id before credential lookup.

## Solution Assessment

Yes. The manifest-level alias map is the narrowest maintainable fix because core docs and resolver code already define `providerAuthAliases` as the contract for provider variants sharing auth.
