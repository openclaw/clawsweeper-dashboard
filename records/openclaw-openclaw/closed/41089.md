## Summary

Current `main` does forward configured OpenAI TTS `instructions` to `/audio/speech` and gates them to `gpt-4o-mini-tts`, but the issue's requested per-message `[[tts:instructions=...]]` override path is still not implemented. Conservative outcome: keep open.

## Evidence

- **Configured OpenAI TTS instructions are implemented on main:** `resolveOpenAITtsInstructions()` trims `instructions` and only includes them for `gpt-4o-mini-tts` models; `openaiTTS()` sends the field in the request body when supported.
  - file: [extensions/openai/tts.ts:64](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/extensions/openai/tts.ts#L64)
  - command: `nl -ba extensions/openai/tts.ts | sed -n '60,160p'`
  - sha: [e94c0bf51542](https://github.com/openclaw/openclaw/commit/e94c0bf51542fe518e49ad4e2ee29882891b5603)
- **Provider config forwards instructions, but override handling does not:** OpenAI speech config reads `instructions` from provider config, and synthesize/synthesizeTelephony pass `config.instructions` through. However `readOpenAIOverrides()` only accepts `model`, `voice`, and `speed`, so there is no per-message override path for `instructions`.
  - file: [extensions/openai/speech-provider.ts:32](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/extensions/openai/speech-provider.ts#L32)
  - command: `nl -ba extensions/openai/speech-provider.ts | sed -n '20,160p' && nl -ba extensions/openai/speech-provider.ts | sed -n '238,285p'`
  - sha: [e94c0bf51542](https://github.com/openclaw/openclaw/commit/e94c0bf51542fe518e49ad4e2ee29882891b5603)
- **Directive parser still cannot express `instructions`:** `parseTtsDirectives()` delegates tokens to provider parsers, and the OpenAI provider parser only handles `voice` and `model`. There is no `instructions` token case, so `[[tts:instructions=...]]` from the issue body is still unsupported.
  - file: [src/tts/directives.ts:88](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/tts/directives.ts#L88)

- **Schema acceptance is no longer the blocker:** `TtsConfigSchema` accepts arbitrary provider keys via `TtsProviderConfigSchema.catchall(...)`, and there is a dedicated test asserting `providers.openai.instructions` parses successfully.
  - file: [src/config/zod-schema.tts.test.ts:4](https://github.com/openclaw/openclaw/blob/e94c0bf51542fe518e49ad4e2ee29882891b5603/src/config/zod-schema.tts.test.ts#L4)
