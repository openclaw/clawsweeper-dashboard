## Summary

Current main already fixes the codex native hook relay transport problem by sending the stable `features.hooks` flag together with `hooks.PreToolUse` into Codex thread config; the fix is on main only and has not shipped in a release yet.

## What This Changes

The issue reports that codex-harness agents do not receive `hooks.PreToolUse` config, bypassing plugin `before_tool_call` enforcement for native tool execution.

## Best Possible Solution

Keep the current main fix, ship it in the next release, and leave the separate Anthropic harness report tracked independently.

## Reproduction Assessment

Yes, by source and history: the reported stable build used the legacy Codex hook feature flag, while current main sends the stable `features.hooks` key through the `thread/start` config path. I did not run a live Codex app-server turn in this read-only review.

## Solution Assessment

Yes. Switching the generated per-thread feature flag to Codex's stable `hooks` key is the narrowest fix for the reported missing hook config, and the existing start/resume config tests cover the transport path.
