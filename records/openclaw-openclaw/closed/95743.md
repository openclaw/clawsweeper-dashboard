## Summary

Current main and v2026.6.9 still contain the raw-length slice in the `stripPrefixes` loop, so the core helper can still over-strip when handed an untrimmed policy entry. The linked fix PR is open and uses closing syntax for this issue, so the issue should stay open until that PR lands or is rejected.

## What This Changes

The issue asks OpenClaw to strip manifest model-id prefixes by the normalized matched prefix length instead of the raw manifest string length.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/95744 if it remains the narrow matched-length fix with focused regression coverage, then close this issue as implemented.

## Reproduction Assessment

Yes for source-level reproduction: current main still matches with the trimmed/lowercased prefix but slices by the raw prefix length. I did not run the Vitest repro because this review was required to keep the checkout read-only.

## Solution Assessment

Yes, the linked PR’s one-line change to slice by `normalizedPrefix.length` is the narrowest maintainable fix for the helper invariant, with regression tests covering padded/cased prefixes.
