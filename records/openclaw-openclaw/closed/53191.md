## Summary

Close as `clawhub`: the request is coherent and useful, but current main already routes reusable workflow/playbook patterns through skills, plugin bundles, OpenProse, Lobster, and Task Flow rather than adding a new native `playbooks` root in `openclaw.json`. The existing plugin and skill APIs cover discovery, prompt surfacing, tools, approval gates, and durable flow tracking without a concrete missing core API in this issue.

## Best Possible Solution

Do not add a native `playbooks` root to OpenClaw core. Package playbook patterns as ClawHub skills or a plugin bundle, likely using workspace `SKILL.md` files for discovery/institutional memory, OpenProse for multi-agent orchestration, and Lobster/Task Flow for approval-gated durable pipelines. If a concrete generic plugin API gap appears, track that as a narrow plugin-SDK issue.
