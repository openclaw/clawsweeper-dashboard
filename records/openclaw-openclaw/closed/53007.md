## Summary

Current main implements proxy-aware Memory Search remote embedding requests. Gemini memory embeddings call the shared memory-host remote HTTP helper, which switches guarded fetch to trusted env-proxy mode only when `HTTP_PROXY`/`HTTPS_PROXY` would actually proxy that target and keeps strict SSRF/DNS pinning for `NO_PROXY` or unsupported proxy cases. Checked-in tests cover the memory helper, proxy-env matching, and guarded fetch behavior. The changelog records the memory-host SDK proxy fix under unreleased 2026.4.25, so no shipped release tag is known yet.

## Best Possible Solution

Close this issue as implemented on main. Keep the current targeted memory-host proxy integration because it fixes the reported Memory Search failure without broadening SSRF bypasses for `NO_PROXY` or `ALL_PROXY`-only environments; any remaining proxy gaps in unrelated providers should be tracked as narrower provider-specific issues.
