## Summary

Keep open: this is the canonical active fix for a source-reproducible gateway status false positive on current main, with focused code/test changes, sufficient real behavior proof, and no blocking review finding.

## What This Changes

The PR expands gateway service-audit command parsing so shell-wrapped LaunchAgent payloads can satisfy the existing gateway subcommand and port checks.

## Best Possible Solution

Land this focused service-audit parsing fix or an equivalent narrow implementation, then close https://github.com/openclaw/openclaw/issues/81751 after merge.

## Reproduction Assessment

Yes. Current main reads LaunchAgent ProgramArguments into the status audit unchanged while `hasGatewaySubcommand` only checks exact argv elements, so the reported `/bin/zsh -lc ... gateway --port ...` wrapper is source-reproducible as a false `gateway-command-missing` warning.

## Solution Assessment

Yes. Expanding command tokens inside the existing service-audit parser is a narrow maintainable fix, and the added near-match regression keeps the change from becoming broad substring matching.
