## Summary

Current OpenClaw already enforces and documents the trust gate that explains the reported Hindsight retain no-op: non-bundled plugins cannot register conversation-content hooks such as `agent_end` unless `plugins.entries.<id>.hooks.allowConversationAccess=true` is configured. The supplied config omits that flag, so the remaining action belongs in the third-party plugin setup/release guidance rather than OpenClaw source.

## What This Changes

The issue asks OpenClaw to restore or diagnose retain hook dispatch for the third-party `hindsight-openclaw` plugin after SDK and manifest migrations.

## Best Possible Solution

Keep OpenClaw's explicit conversation-access gate, and have the Hindsight plugin setup or release notes require `plugins.entries.hindsight-openclaw.hooks.allowConversationAccess=true` before retain hooks can read completed turns.

## Reproduction Assessment

No high-confidence live reproduction was run because the Hindsight plugin implementation is external to this repo. Source inspection does show the matching OpenClaw path: a non-bundled `agent_end` retain hook is blocked when the config lacks `allowConversationAccess`.

## Solution Assessment

Yes. The narrow maintainable outcome is to keep the privacy/trust gate in core and route this setup to plugin configuration and third-party Hindsight docs, verified with `openclaw plugins inspect hindsight-openclaw --runtime --json`.
