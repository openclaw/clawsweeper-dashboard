## Summary

Current `main` already contains the async `exec-event` reply-routing fix. The root path now falls back to persisted session delivery context or stored last-route metadata for system-event replies, and regression tests cover the missing-originating-route case including a Discord fallback.

## Evidence

- **Merged fix PR:** PR #70258 (`fix(auto-reply): restore route replies for exec-event completions`) merged on 2026-04-23 with merge commit `2b45bc1fd353230ef6b51d50855063ebda514576`. The PR summary states the root cause was async completion turns losing direct channel metadata, causing channel route-reply delivery to be skipped.
