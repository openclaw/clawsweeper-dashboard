## Summary

Current main already supports Slack custom sender identity for routed agent replies: built-in Slack delivery resolves the routed agent identity, passes it into reply delivery, and `chat.postMessage` includes `username` plus `icon_url`/`icon_emoji` when `chat:write.customize` is available. The docs describe this as shipped behavior for the active agent identity, which covers the reported dedicated-channel-per-agent use case.

## Evidence

- **Built-in Slack routing applies routed agent identity:** `extensions/slack/src/monitor/message-handler/dispatch.ts:222` resolves `resolveAgentOutboundIdentity(cfg, route.agentId)` for Slack `chat:write.customize` overrides, and `extensions/slack/src/monitor/message-handler/dispatch.ts:486` passes that identity into `deliverReplies` for normal built-in Slack message delivery.
  - file: [extensions/slack/src/monitor/message-handler/dispatch.ts:222](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/extensions/slack/src/monitor/message-handler/dispatch.ts#L222)

  - file: [extensions/slack/src/send.ts:100](https://github.com/openclaw/openclaw/blob/aa21d4ea7e9aab034acbf0b13f8c3575057c5c54/extensions/slack/src/send.ts#L100)

## Risks / Open Questions

- The current solution is agent-scoped, not an additional Slack-channel-only override. If one agent must present different Slack names/icons in different channels, that narrower use case is still distinct.
