## Summary

Close as superseded: current main intentionally keeps `node-llama-cpp` out of the root package graph, and the prior merged packaging decision plus release checks/docs supersede the requested root optional dependency change.

## What This Changes

Issue requests shipping `node-llama-cpp` as a root optional dependency so configured local memory embeddings keep working after updates.

## Best Possible Solution

Keep the current operator-installed local runtime contract; any change to bundle or auto-install `node-llama-cpp` should be a new maintainer-owned packaging policy discussion, not a small bug fix.

## Reproduction Assessment

Yes from source inspection: `provider: "local"` reaches a dynamic import of `node-llama-cpp`, so a missing package will fail before local embeddings can load. That failure is expected under current docs when the optional runtime has not been installed next to OpenClaw.

## Solution Assessment

No: adding a root optional dependency is not the best current solution because current main has release guards and docs that intentionally keep `node-llama-cpp` operator-installed. The maintainable path is the documented local-runtime install flow unless maintainers reopen the packaging policy.
