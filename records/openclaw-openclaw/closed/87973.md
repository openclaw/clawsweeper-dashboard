## Summary

Current main already honors the documented empty/comment-only HEARTBEAT.md behavior before any heartbeat reply/model dispatch, and the latest release tree also contains that gate. The report was against an older 2026.5.19 build, while current source and tests cover the central skip behavior.

## What This Changes

The issue asks OpenClaw to skip recurring heartbeat API calls when an idle persistent ACP agent has only the empty/comment-only HEARTBEAT.md template.

## Best Possible Solution

Keep the current scheduler-side HEARTBEAT.md preflight gate and point affected users to upgrade to v2026.5.27 or newer rather than adding another config surface.

## Reproduction Assessment

No for current main: source and tests show comment-only HEARTBEAT.md interval runs skip before `getReplyFromConfig`, so the reported heartbeat API call path should not reproduce on current main or the inspected latest release tree.

## Solution Assessment

Yes: the existing preflight gate is the narrowest maintainable solution because it avoids the heartbeat turn before runtime dispatch, including ACP-backed agents, while preserving wake, cron, system-event, and actionable-task behavior.
