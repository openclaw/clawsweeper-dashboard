## Summary

Current main already suppresses `session.message`-triggered history reloads during an active chat run and replays a single deferred reload after terminal chat events; the same behavior is present in shipped release `v2026.4.23` and covered by checked-in UI tests.

## Evidence

- **Current main defers mid-run session.message reloads:** `handleSessionMessageGatewayEvent` now skips `loadChatHistory(...)` when `host.chatRunId` is active and stores a deferred session key instead, which directly addresses the reported flicker path from repeated `session.message` events during a live turn.
  - file: [ui/src/ui/app-gateway.ts:496](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/ui/src/ui/app-gateway.ts#L496)
