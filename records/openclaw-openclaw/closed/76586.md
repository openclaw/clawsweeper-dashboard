## Summary

Keep open. Current main has partial repairs and docs for the 5.2 externalized-plugin rollout, but source inspection still shows the central recovery loop: update-time doctor defers configured plugin installs, post-core plugin sync skips invalid configs, and manual install recovery is not available for the Brave web-search invalid-config case.

## What This Changes

The issue asks OpenClaw to make the 2026.4.29 to 2026.5.2 upgrade and recovery path work for configs that reference externalized official plugins such as Brave, Discord, voice-call, and WhatsApp.

## Best Possible Solution

Make configured official plugin installs complete before config-valid post-core sync is required, and keep manual invalid-config recovery scoped to the requested official plugin rather than opening a broad config bypass.

## Reproduction Assessment

Yes, from source inspection. A config selecting Brave web search becomes invalid when Brave is missing; update-mode doctor defers the install, and the later plugin sync returns `invalid-config` before repairing it.

## Solution Assessment

No, current main is not yet the best solution. The narrow maintainable fix is to run the official configured-plugin repair to completion in the update recovery path, with targeted tests for invalid-config recovery.
