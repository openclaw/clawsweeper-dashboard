## Summary

Current main already addresses the Slack multi-agent thread wakeup problem with the shipped `channels.slack.thread.requireExplicitMention` setting. When enabled, Slack suppresses implicit `reply_to_bot` and `bot_thread_participant` mention kinds, docs describe the behavior, regression tests cover the drop/allow/default cases, and PR #58276 shipped it in v2026.4.7.

## Best Possible Solution

Close as implemented and keep the shipped `channels.slack.thread.requireExplicitMention` control. If maintainers want `requireMention: true` itself to imply strict thread mention gating by default, that should be tracked as a separate compatibility/product decision against the current Slack config surface.

Changing that default would be a separate product/compatibility decision, not missing implementation of the current mitigation.
