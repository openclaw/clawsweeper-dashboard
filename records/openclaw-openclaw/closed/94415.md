## What This Changes

The PR routes `ensurePortAvailable` through the existing multi-family port probe, exports that probe internally, and adds regression tests for free ports, IPv4-only occupants, and unverifiable port availability.

## Best Possible Solution

Land this narrow multi-family CDP preflight fix, or one equivalent canonical PR, after maintainer acceptance of the SDK-helper behavior; keep the separate `isPortBusy` work under https://github.com/openclaw/openclaw/issues/94426.

## Solution Assessment

Yes: reusing the existing multi-family `checkPortInUse` helper is a better fix than adding a one-host browser workaround, and the latest head preserves fail-fast behavior for unverifiable ports. The `gateway --force` sibling should remain separate because it is a different caller surface with its own issue.
