## Summary

Current `main` already separates the ACP stall/recovery states that the report called out: ACP parent relays emit explicit `Started`, `No output`, `Resumed output`, `completed`, and `run failed` state changes; ACP control-plane handling retries stale/early `acpx` exits and stale missing-session resumes without leaving the session in an error state; and provider quota/rate-limit/capacity errors are surfaced as provider-facing lifecycle errors instead of collapsing into generic empty-response copy.

## Evidence

  - file: [src/agents/acp-spawn-parent-stream.test.ts:141](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/src/agents/acp-spawn-parent-stream.test.ts#L141)
