## Summary

Keep open. The issue is fresh, has an open implementation PR with closing syntax, and current main still lacks the explicit sandbox policy-key handoff or regression proof that would close the reported Docker probe path. Current source still sends isolated cron embedded runs with only the run-scoped session key, the embedded runner still falls back from sandboxSessionKey to sessionKey, and Docker image inspection still throws if the sandbox backend path is reached unexpectedly.

## Best Possible Solution

Keep this issue open until maintainers review and either land PR #73627, choose PR #73671, or apply an equivalent focused fix. The preferred outcome should ensure automated isolated runs pass the correct sandbox policy key, guarantee sandbox.mode: "off" cannot initialize Docker/backend code, and add regression coverage for Docker-unavailable cron, heartbeat, and sub-agent startup paths.
