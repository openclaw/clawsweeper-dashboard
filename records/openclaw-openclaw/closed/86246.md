  - body: `openClawToolsMcpBridge` is already documented as the selected built-in-tools bridge whose initial server exposes `cron`; adding `slack` here means any ACPX deployment that enabled that bridge for cron will also get Slack send/edit/delete/upload/download actions after upgrade.   - body: The reported Claude Code path is the loopback MCP server named `openclaw`, generated by `createMcpLoopbackServerConfig`, so tools appear as `mcp__openclaw__*`.   - confidence: 0.86
- **[P2] Preserve message-tool session context for Slack actions:** `src/agents/tools/slack-tool.ts:329-335`
  - body: This bridge calls `message.action` without the session key, agent id, requester, inbound kind, or `toolContext` that the existing `message` tool forwards. Slack sends/uploads then lose current-thread inference, same-channel thread guards, and transcript mirroring, so a Slack-threaded run can post or record messages differently than the supported channel action path.
  - confidence: 0.82
