  - body: The PR head returns raw cleanup paths and removes each existing target with `fs.rm`, but current main validates names, realpaths, cleanup roots, and symlink roots before removal.   - body: Pooling only by method/token/password can reuse a GatewayClient created for a different URL, TLS fingerprint, scopes, approval token, device identity, protocol range, client mode, or instance id. That can send later authenticated calls over the wrong gateway connection and scope set.
  - confidence: 0.9
- **[P1] Do not keep one-shot gateway clients alive indefinitely:** `src/gateway/call.ts:74-79`
  - body: `releasePooledClient` is a no-op, so calls that previously stopped their GatewayClient now leave WebSocket reconnect timers alive after the promise settles. That can keep CLI commands running and leak background connection work.
  - confidence: 0.88
- **[P1] Preserve gateway request abort and accepted callbacks:** `src/gateway/call.ts:711-714`
  - body: The PR head calls `client.request` without forwarding the caller `signal` or `onAccepted`, and the branch lacks current-main `onSignalAbort` handling. Rebase must keep those request contracts so caller cancellation and accepted acknowledgements still work.
