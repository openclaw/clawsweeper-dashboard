Close as superseded by the active canonical fix PR #71988. Current main still does not implement the requested broadcast call: dispatch only invokes the plugin-targeted inbound claim path for plugin-owned bindings, then continues to message_received and agent dispatch. The remaining work is already tracked directly by #71988, after the earlier #58862/#71644 attempts.

## Best Possible Solution

Close this issue as superseded and keep #71988 as the canonical implementation review. The best code path is to land #71988 or an equivalent narrow change that broadcasts inbound_claim from dispatch-from-config.ts after plugin-bound fallback handling, with tests covering the plugin fallback guard and no unwanted broadcast for ordinary core conversations.
