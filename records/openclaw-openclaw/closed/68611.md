## Summary

Current main already fixes the reported WebChat CLI-runner failure in both places that matter: CLI turns are persisted to the session transcript as user plus assistant messages, and the Control UI preserves optimistic user/assistant messages when a final-event history refresh briefly returns a stale snapshot. The behavior is documented and present in the v2026.4.24 release tag.

## Best Possible Solution

Close this as shipped. Keep the current CLI transcript persistence and Control UI stale-history preservation; any broader transcript-architecture cleanup should be tracked separately if maintainers want more symmetry across runners.
