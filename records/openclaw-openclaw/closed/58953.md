## Summary

Current `main` already resets and re-drains command lanes after SIGUSR1 gateway restarts, matching the reporter's lane-state diagnosis. The tagged `v2026.4.23` release contains that implementation and tests for queued work resuming immediately after restart.

## Evidence

- **Restart loop resets lane state after SIGUSR1:** The gateway restart loop installs a restart-iteration hook that calls `resetAllLanes()` on each in-process restart specifically to clear stale active counts left behind by interrupted tasks.
