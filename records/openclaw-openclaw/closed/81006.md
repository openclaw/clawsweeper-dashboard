## Summary

Close as superseded: the linked ACPX timeout bug is already fixed on current main by the narrower capability-aware implementation from https://github.com/openclaw/openclaw/pull/80812, while this PR still proposes a broader universal timeout drop that would regress advertised timeout support.

## What This Changes

The PR changes ACPX runtime handling so `timeout` and `timeout_seconds` return before backend-specific forwarding for every ACPX backend.

## Best Possible Solution

Keep the merged capability-aware implementation from https://github.com/openclaw/openclaw/pull/80812 and close this PR rather than replacing it with universal timeout suppression.

## Reproduction Assessment

Yes for the original released bug: the PR body provides before/after real logs, and dependency source shows Claude ACP rejects unknown `timeout` config ids. It is not a failing current-main path because current main already suppresses the unsupported Claude path and gates timeout by advertised capabilities.

## Solution Assessment

No. This PR fixes the observed Claude failure, but the current-main solution is narrower because it omits timeout only when unsupported and preserves timeout forwarding for backends that explicitly advertise `timeout` or `timeout_seconds`.
