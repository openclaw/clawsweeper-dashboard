## Summary

Current `main` already preserves existing session origin/routing for heartbeat and other system-event updates, with targeted regression coverage proving the reported behavior is fixed.

## Evidence

- **System-event origin is now skipped during session metadata derivation:** `deriveSessionOrigin` returns `undefined` for `heartbeat`, `cron-event`, and `exec-event` when `skipSystemEventOrigin` is set, and `deriveSessionMetaPatch` threads that option through.
  - file: [src/config/sessions/metadata.ts:54](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/config/sessions/metadata.ts#L54)

  - sha: [564f820efa2b](https://github.com/openclaw/openclaw/commit/564f820efa2bb81ae49b8f55a93e41492ad31917)
- **Session initialization applies the system-event safeguard:** `initSessionState` calls `deriveSessionMetaPatch(..., skipSystemEventOrigin: isSystemEvent)` and then strips stale thread metadata for non-thread system events, preventing heartbeat/system turns from replacing persisted origin data.
  - file: [src/auto-reply/reply/session.ts:648](https://github.com/openclaw/openclaw/blob/564f820efa2bb81ae49b8f55a93e41492ad31917/src/auto-reply/reply/session.ts#L648)
