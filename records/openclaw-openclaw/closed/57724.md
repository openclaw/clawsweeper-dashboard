## Summary

Current `main` and the latest shipped release already contain Discord native-slash handling that immediately defers interactions, routes replies through follow-up delivery, returns a direct `/status` reply, and falls back to an explicit acknowledgment instead of exiting silently.

## Evidence

- **Immediate defer + follow-up dispatch:** Native Discord commands now call `interaction.defer()` before any heavier work and then dispatch through follow-up semantics (`preferFollowUp: true`), which removes the silent no-response path described in the issue.
  - file: [extensions/discord/src/monitor/native-command.ts:701](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/extensions/discord/src/monitor/native-command.ts#L701)
  - sha: [7e49cc87f9e9](https://github.com/openclaw/openclaw/commit/7e49cc87f9e9cdad47d0572331d7a7a47281a83e)
- **No empty-run silent exit remains:** `/status` has a dedicated direct-reply branch, and if a native slash run produces no deliverable output the code sends an explicit `✅ Done.` fallback instead of returning without a Discord response.
  - file: [extensions/discord/src/monitor/native-command.ts:1123](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/extensions/discord/src/monitor/native-command.ts#L1123)
