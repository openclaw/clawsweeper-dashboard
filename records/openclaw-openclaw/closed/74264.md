## Summary

Close as superseded by the existing shared-memory-store work in #46542. Current main still scopes builtin memory indexes and embedding caches per agent/store path, but #46542 is the narrower canonical implementation candidate for shared cross-agent document memory using `sharedPaths`, `_shared.sqlite`, merged shared search results, and `openclaw memory ... --shared`. The issue body's Memory v2 reference (#67836) is relevant adjacent work, but that PR is a default-off sidecar/ingest/rerank foundation and is not the exact shared embedding-cache tracker.

## Best Possible Solution

Close this as superseded by #46542 and keep maintainer review focused on a single shared-memory-store path. The best product outcome is to rebase or replace #46542 in the current `extensions/memory-core` and `packages/memory-host-sdk` layout, resolve its security/data-scope review concerns, and document whether shared workspace embeddings are a first-class `memory-core` feature or an explicit non-goal.
