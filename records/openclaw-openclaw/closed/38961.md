## Summary

Current `main` already guards against the exact WebChat duplicate described here by reusing the latest matching assistant message instead of appending a second `openclaw/delivery-mirror` entry, and it also preserves established external routing when WebChat opens channel-scoped sessions.

## Evidence

- **Transcript dedupe on delivery mirror:** `appendExactAssistantMessageToSessionTranscript` now checks whether a `delivery-mirror` assistant message matches the latest assistant transcript entry and returns the existing message id instead of appending a duplicate.
  - file: [src/config/sessions/transcript.ts:204](https://github.com/openclaw/openclaw/blob/8d1f98ef08cda2809ce3c5b3491a67c10a8afa88/src/config/sessions/transcript.ts#L204)
  - sha: [8d1f98ef08cd](https://github.com/openclaw/openclaw/commit/8d1f98ef08cda2809ce3c5b3491a67c10a8afa88)
- **Exact matching logic:** The helper scans backward to the latest assistant message and only suppresses the mirror when that latest assistant text matches, which directly covers the issue's `original + delivery-mirror` double-record symptom.
  - file: [src/config/sessions/transcript.ts:263](https://github.com/openclaw/openclaw/blob/8d1f98ef08cda2809ce3c5b3491a67c10a8afa88/src/config/sessions/transcript.ts#L263)
  - sha: [8d1f98ef08cd](https://github.com/openclaw/openclaw/commit/8d1f98ef08cda2809ce3c5b3491a67c10a8afa88)
- **Regression test for duplicate mirror suppression:** Test proves that when an original assistant reply already exists, adding the same mirrored text does not append a second transcript line and reuses the original message id.
  - file: [src/config/sessions/transcript.test.ts:149](https://github.com/openclaw/openclaw/blob/8d1f98ef08cda2809ce3c5b3491a67c10a8afa88/src/config/sessions/transcript.test.ts#L149)
  - sha: [8d1f98ef08cd](https://github.com/openclaw/openclaw/commit/8d1f98ef08cda2809ce3c5b3491a67c10a8afa88)
- **WebChat routing preservation:** Current session routing logic keeps WebChat from overwriting an established external delivery route for channel-scoped/main sessions, preventing WebChat follow-ups from hijacking delivery paths.
  - file: [src/auto-reply/reply/session-delivery.ts:96](https://github.com/openclaw/openclaw/blob/8d1f98ef08cda2809ce3c5b3491a67c10a8afa88/src/auto-reply/reply/session-delivery.ts#L96)
  - sha: [8d1f98ef08cd](https://github.com/openclaw/openclaw/commit/8d1f98ef08cda2809ce3c5b3491a67c10a8afa88)
