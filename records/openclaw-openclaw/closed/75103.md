## What This Changes

The branch adds optional process start-time metadata to Plugin SDK lock payloads and tests stale same-PID lock recovery in `src/plugin-sdk/file-lock`.

## Best Possible Solution

Port same-PID start-time disambiguation into the current fs-safe-backed Plugin SDK lock payload and shared stale-lock policy, preserving fail-closed live-owner behavior and adding focused regression plus redacted runtime proof.

Yes. A high-confidence source-level path is a `.lock` payload whose recorded `pid` is live in the new process but belongs to a previous incarnation; current main has no Plugin SDK `starttime` field/comparison, so the active fs-safe wrapper treats that live PID as non-reclaimable.

## Solution Assessment
