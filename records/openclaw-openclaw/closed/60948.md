## Summary

Current `main` already handles the reported ended-session follow-up path: `sessions_send` goes through Gateway `agent`, completed session/subagent state is reactivated on a new send/start, lifecycle start clears terminal state back to `running`, and targeted regression tests cover the timeout-adjacent flow.
