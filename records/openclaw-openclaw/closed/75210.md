## Summary

Close this PR as superseded by #74002. Current main still has the underlying OPENCLAW_STATE_DIR exec-approvals bug, but #74002 is already the broader open canonical fix for the same path/socket/default-host-source problem and includes docs, changelog, and regression coverage; this narrower PR would split review and also introduces a legacy-state fallback regression.

## What This Changes

This PR changes `src/infra/exec-approvals.ts` so the default exec approvals JSON and socket paths are built from `resolveStateDir()` instead of hard-coded `~/.openclaw` paths.

## Best Possible Solution

Use the canonical #74002 direction: honor `OPENCLAW_STATE_DIR` for the exec approvals file, socket default, and effective-policy host-source display while preserving the documented unset default, adding regression tests, and updating approval docs/changelog in one reviewed change.

## Reproduction Assessment

Yes. The underlying bug is reproducible from source by setting `OPENCLAW_STATE_DIR=~/openclaw` and observing current main still resolves exec approvals through hard-coded `~/.openclaw` defaults; the PR regression is reproducible by following `resolveStateDir()` into its legacy `.clawdbot` fallback when the override is unset.

## Solution Assessment

No. The requested direction is right, but this one-file implementation is not the best fix because it reuses a generic state resolver with legacy fallback semantics and omits the docs/tests/display-path coverage already present in #74002.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve the default approvals root:** `src/infra/exec-approvals.ts:211-215`
  - body: `resolveStateDir()` can return an existing legacy `.clawdbot` directory when `OPENCLAW_STATE_DIR` is unset and `.openclaw` does not exist. Using it here would silently move the exec approval policy file and socket token away from the documented `~/.openclaw` default for those installs, so the fix needs an exec-approvals-specific resolver that only follows the override and otherwise preserves the current default.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The diff touches host exec approval policy and socket-token storage and can move that security state to a legacy directory when no override is set.

Concerns:

- **[medium] Approval state can move to legacy state dir:** `src/infra/exec-approvals.ts:211`

  - confidence: 0.86
