## Summary

Current main already handles plugin-local config validation failures as a degraded startup condition instead of aborting the gateway. The implementation is covered by startup recovery tests, documented, and shipped in v2026.4.26; the related secret-target-registry failure class noted in the comment also now has per-plugin error isolation.

## Best Possible Solution

Close this issue as implemented and keep the shipped plugin-local startup degradation path. Any remaining noisy repeated-log behavior should stay with the narrower open follow-up #63234 or a new plugin-specific report if a concrete startup-abort reproduction appears.
