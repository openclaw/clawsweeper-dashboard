## Summary

Keep open: current main intentionally disables the macOS LaunchAgent during `gateway stop`, and source, tests, and docs all confirm the behavior the reporter is challenging. This is a valid lifecycle semantics and diagnostics proposal, but changing it needs maintainer product judgment because it alters the documented stop contract.

## What This Changes

Request changing macOS `openclaw gateway stop` so persistent `launchctl disable` is opt-in and adding diagnostics for disabled LaunchAgent recovery.

## Best Possible Solution

Have maintainers choose the macOS lifecycle contract, then update stop/start/restart behavior, CLI docs, tests, and doctor/status recovery hints as one coordinated change.

## Reproduction Assessment

Yes for the narrow current behavior: source and tests show `openclaw gateway stop` unconditionally reaches `launchctl disable` on macOS. I did not run a live macOS LaunchAgent reproduction in this read-only review.

## Solution Assessment

Unclear until maintainer decision: the proposed `--disable` or `--permanent` mode is plausible, but it changes a documented CLI lifecycle contract and should be reviewed as product behavior, not a mechanical bug fix.
