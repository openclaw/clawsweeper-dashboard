## Summary

This issue is a duplicate/superseded feature request. Current main still lacks per-agent `agents.list[].userTimezone`, but the same remaining work is already tracked by open PR #39610, which implements the per-agent timezone override and was filed for the earlier matching issue #39579.

## Best Possible Solution

Close this issue as duplicate/superseded and keep the remaining implementation review on open PR #39610 or a successor PR. The eventual fix should add `agents.list[].userTimezone`, centralize precedence as per-agent -> defaults -> host timezone, update schema/generated config help/docs, and thread the active agent id through all user-timezone runtime paths.
