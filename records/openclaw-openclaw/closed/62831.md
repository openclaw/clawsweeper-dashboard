## Summary

Keep open. Current `main` still carries `audioAsVoice` through core outbound delivery, but the QQ Bot reply path drops that flag before plain replies are sent. QQ Bot only synthesizes voice in its structured `mediaType: "audio"` path, so the reported `mediaType=undefined, ttsText=undefined` behavior remains consistent with the code on `main`.
