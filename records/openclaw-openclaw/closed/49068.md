## Summary

Current `main` no longer surfaces `NO_REPLY` / `ANNOUNCE_SKIP` as the final gateway `agent` summary, so the silent-summary case this PR targets is obsolete against today's code.

## Evidence

- **Gateway agent success now hardcodes a non-silent summary:** The current gateway `agent` handler returns `summary: "completed"` on successful runs, regardless of whether the result payloads are empty.
  - file: [src/gateway/server-methods/agent.ts:267](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/gateway/server-methods/agent.ts#L267)
  - command: `nl -ba src/gateway/server-methods/agent.ts | sed -n '258,282p'`
  - sha: [01bc49c88c0c](https://github.com/openclaw/openclaw/commit/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd)
- **CLI gateway path logs the final response summary, not a silent token:** When the gateway returns zero payloads, the CLI logs `response.summary` if present; with the current gateway handler, that is `completed`, so the old `No reply from agent.` symptom is not reproduced in the success path.
  - file: [src/commands/agent-via-gateway.ts:166](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/commands/agent-via-gateway.ts#L166)
  - command: `nl -ba src/commands/agent-via-gateway.ts | sed -n '145,190p'`
  - sha: [01bc49c88c0c](https://github.com/openclaw/openclaw/commit/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd)
- **`expectFinal` waits for the final gateway response frame:** The gateway client keeps waiting past `accepted` frames when `expectFinal` is true, so `openclaw agent` consumes the final success payload from the gateway handler above.
  - file: [src/gateway/client.ts:774](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/gateway/client.ts#L774)
  - command: `nl -ba src/gateway/client.ts | sed -n '760,790p'`
  - sha: [01bc49c88c0c](https://github.com/openclaw/openclaw/commit/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd)
- **Current tests for the CLI gateway path no longer cover silent-summary handling:** `src/commands/agent-via-gateway.test.ts` on `main` has no `NO_REPLY` / `ANNOUNCE_SKIP` regression case, matching the fact that the current gateway success contract no longer emits those summaries here.
  - file: [src/commands/agent-via-gateway.test.ts:83](https://github.com/openclaw/openclaw/blob/01bc49c88c0cef9807b15df4d7486a0c5fe4afbd/src/commands/agent-via-gateway.test.ts#L83)
  - command: `nl -ba src/commands/agent-via-gateway.test.ts | sed -n '1,180p'`

## Risks / Open Questions

- If someone is still running an older gateway build that returns `NO_REPLY` or `ANNOUNCE_SKIP` as the final `agent` summary, this PR could still help that older code path, but that is not the behavior on current `main`.
- Current `main` still logs `completed` for zero-payload gateway runs, which may or may not be ideal UX, but that is a different issue from the silent-summary bug described here.
