## Summary

Keep open: current main still has the reported Telegram finalization bug at source level, and the only PR using closing syntax for this issue is closed unmerged. The documented `MEDIA:` contract and shared outbound parser support local media directives, but Telegram can still preview-finalize the raw directive text before that normalization happens.

## What This Changes

The issue asks for Telegram direct-chat replies containing local-path `MEDIA:` directives to send those files as attachments and strip the directive lines from visible text.

## Best Possible Solution

Normalize the final Telegram lane segment through the shared outbound payload plan, or an equivalent directive parser, before deciding that the reply is text-only preview-finalizable.

## Reproduction Assessment

Yes, at source level. A final Telegram answer segment containing standalone `MEDIA:/path` text can reach `deliverLaneText`, whose `hasMedia` check reads only `payload.mediaUrl(s)` and can preview-finalize before the shared directive parser strips and attaches the media.

## Solution Assessment

Yes. The narrow maintainable fix is to make Telegram's preview/finalization media check use the same normalized payload shape as final delivery, without changing Telegram upload behavior or local file access policy.
