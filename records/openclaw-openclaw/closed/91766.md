Change `openclaw cron status` so `storePath` identifies the canonical shared SQLite database rather than the legacy cron JSON path.

## Best Possible Solution

Land one reporting-only fix that resolves `storePath` through the canonical SQLite path helper, updates focused service and gateway assertions, and leaves the internal legacy-derived partition key unchanged.

## Solution Assessment

Yes, the narrow reporting correction is the best direction. Adding a configurable status path or a second legacy field would expand the contract without being necessary to correct the existing output.
