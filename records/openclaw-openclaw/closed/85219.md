## Summary

Keep this PR open: the core Telegram attachment-path fix is valid and well scoped, but the current collector can fan out singular media aliases into multiple Telegram sends, which is a merge-blocking delivery regression to fix before landing.

## What This Changes

The PR updates Telegram `sendMessage` actions to collect top-level and structured attachment media aliases, send them through Telegram media delivery, add regression tests, and note the user-visible fix in the changelog.

## Best Possible Solution

Land the Telegram attachment-path support after top-level singular media aliases are normalized to one primary media source, while preserving explicit multi-media delivery through `mediaUrls[]` and structured attachments.

## Reproduction Assessment

Yes by source inspection. Current main exposes `path`/`filePath`/structured attachments in the message tool while Telegram action runtime only forwards `mediaUrl`/`media`; the PR finding is also source-reproducible from the new collector appending singular aliases as separate sends.

## Solution Assessment

No in the current form. The right shape is to reuse the existing Telegram media sequencing idea, but first normalize top-level singular aliases to a single primary media source and reserve multi-send behavior for `mediaUrls[]` and structured attachments.
