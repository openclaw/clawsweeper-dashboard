## Summary

Current main already contains the health-monitor changes that remove the reported Discord stale-socket restart loop behavior. The stale-socket path now uses a separate 30-minute threshold and only fires for channels that report transport-level liveness, which avoids treating idle Discord accounts as stale based on ordinary message inactivity.
