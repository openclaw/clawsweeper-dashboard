## Summary

Current `main` adds documented and tested ways to disable Bonjour (`discovery.mdns.mode: "off"`, `OPENCLAW_DISABLE_BONJOUR=1`, or disabling the bundled `bonjour` plugin), but I did not find high-confidence evidence that the default WSL2 watchdog crash itself is fixed. There is still no WSL/WSL2-specific bypass in the Bonjour extension, and the production Bonjour entry does not wire the optional ciao unhandled-rejection suppressor that the new advertiser code supports.
