## What This Changes

The issue asks OpenClaw to keep the Discord gateway online after the SSH session that launched `openclaw chat` disconnects.

## Best Possible Solution

Use the supported Linux host setup path: run `openclaw doctor --fix` or `sudo loginctl enable-linger "$USER"`; if the failure persists with linger enabled, open a new bug with linger status, service status, and journal logs.

## Reproduction Assessment

Yes for the likely support path, not for a source bug. Current docs and code show that a Linux systemd user service without linger stops after logout, and the report did not include evidence that linger is enabled.

## Solution Assessment

Yes. The narrowest maintainable solution is the documented `loginctl enable-linger`/doctor repair path, with a new source-level bug only if the service still stops when linger is confirmed enabled.
