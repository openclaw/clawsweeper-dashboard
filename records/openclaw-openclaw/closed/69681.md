## Summary

Current `main` already adds node-targeted exec-approval controls, effective-policy inspection, and documented no-prompt/session-scoped escape hatches that address the reported approval friction for trusted maintenance workflows.

## Evidence

- **Node approvals are now a first-class node-host surface:** The node CLI docs explicitly route node-host exec approval management through `openclaw approvals --node <id|name|ip>` from the gateway.
  - file: [docs/cli/node.md:158](https://github.com/openclaw/openclaw/blob/07cf1dd65c937c75cf48a1645ea580404f5056b4/docs/cli/node.md#L158)

  - file: [docs/cli/approvals.md:68](https://github.com/openclaw/openclaw/blob/07cf1dd65c937c75cf48a1645ea580404f5056b4/docs/cli/approvals.md#L68)

  - file: [docs/cli/approvals.md:94](https://github.com/openclaw/openclaw/blob/07cf1dd65c937c75cf48a1645ea580404f5056b4/docs/cli/approvals.md#L94)

  - file: [docs/tools/exec-approvals.md:199](https://github.com/openclaw/openclaw/blob/07cf1dd65c937c75cf48a1645ea580404f5056b4/docs/tools/exec-approvals.md#L199)
