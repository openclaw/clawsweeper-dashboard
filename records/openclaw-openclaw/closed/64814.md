## Summary

Current `main` already threads `plugins.entries.<id>.config` into `register(api)` for non-bundled plugins, exposes it as `api.pluginConfig`, and includes a regression test that fails if that config is missing.

## Evidence

- **Loader validates plugin entry config before registration:** The main plugin loader reads `entry?.config` through `validatePluginConfig(...)` before plugin registration.
  - file: [src/plugins/loader.ts:2525](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/src/plugins/loader.ts#L2525)
