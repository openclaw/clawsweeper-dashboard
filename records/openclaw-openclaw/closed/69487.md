## Summary

Current `main` already avoids the reported cold-start path: gateway startup loads a gateway-bindable plugin registry, later runtime/subagent loads reuse that active registry instead of recompiling/re-registering non-bundled plugins, and there is a regression test covering that exact behavior.

## Evidence

- **Gateway startup installs gateway-bindable plugin runtime:** `prepareGatewayPluginLoad` installs the gateway subagent runtime up front, and `loadGatewayPlugins` loads startup plugins with `allowGatewaySubagentBinding: true`, so the startup registry is the gateway-bindable runtime later subagent calls can reuse.
  - file: [src/gateway/server-plugin-bootstrap.ts:74](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/gateway/server-plugin-bootstrap.ts#L74)
  - command: `nl -ba src/gateway/server-plugin-bootstrap.ts | sed -n '39,96p' && nl -ba src/gateway/server-plugins.ts | sed -n '493,507p'`
  - sha: [5fe333ada85a](https://github.com/openclaw/openclaw/commit/5fe333ada85afd0def9e85985ce57e36aa00108a)
- **Runtime loads reuse the active compatible registry:** `ensureRuntimePluginsLoaded` inherits gateway-bindable mode from the active registry, and `getCompatibleActivePluginRegistry` / `resolveRuntimePluginRegistry` return the already-active registry when the load context matches instead of triggering a fresh plugin load.
  - file: [src/plugins/loader.ts:1121](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/plugins/loader.ts#L1121)

  - file: [src/plugins/loader.test.ts:3548](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/plugins/loader.test.ts#L3548)
