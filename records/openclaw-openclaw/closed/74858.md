## What This Changes

The issue reports that Telegram group natural-language replies are generated in OpenClaw 2026.4.27 but never delivered back to the group, while DMs and native slash-command group replies still work.

## Best Possible Solution

Keep #73583 as the canonical implementation path, then land or replace it with a maintainer-approved fix that preserves the documented group visible-reply privacy contract while making Telegram group/topic fallback behavior explicit and tested.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: with a Telegram group turn and no messages.groupChat.visibleReplies: "automatic", current main resolves delivery to message_tool_only, suppresses normal finals, and lacks Telegram fallback wiring; native commands bypass that path.

## Solution Assessment

Yes, #73583 is the narrowest current solution track because it adds the missing Telegram/core suppressed-source fallback and tests the reported group/topic path. Maintainers still need to approve the privacy/product behavior before merge.
