## Summary

Current main already makes Telegram group allowlist setup explicit in the public docs and doctor warning while preserving the intended fail-closed runtime behavior for unconfigured groups; the same evidence is present in `v2026.4.29`.

## What This Changes

Request clearer Telegram group allowlist docs/setup/doctor guidance so work subagent group chats can be enabled without guessing the required config keys.

## Best Possible Solution

Keep the shipped fail-closed Telegram group behavior and the existing explicit docs/doctor remediation; future setup-wizard polish can be handled separately if maintainers want it.

Yes for the underlying configuration state: `groupPolicy="allowlist"` without configured groups or sender allowlists maps directly to the doctor warning and runtime fail-closed paths inspected above. No live Telegram run is needed to verify the reported confusion path is now documented and diagnosed.

## Solution Assessment

Yes. The safest maintainable solution is explicit documentation and doctor remediation around `channels.telegram.groups` and `groupAllowFrom`, while retaining allowlist defaults for group chats.
