## Summary

Keep open. Current main still has the reported control-plane/runtime mismatch: disabled bundled plugins with tool contracts can enter the runtime discovery scope, and registry reuse then rejects an otherwise usable registry before enabled plugin tools are appended.

## What This Changes

The issue reports that disabled bundled tool plugins can cause enabled plugin-backed tools such as browser, memory_search, and memory_get to disappear from the effective agent tool surface.

## Best Possible Solution

Filter runtime tool-discovery required plugin IDs to actually enabled/loadable plugin owners while preserving bundled manifest visibility for catalog, setup, and doctor flows.

## Reproduction Assessment

Yes, at source level. Current main can include disabled bundled tool-contract plugins in required runtime IDs, and the active registry lookup rejects registries unless all required IDs are loaded; I did not run a live effective-inventory trace.

## Solution Assessment

Yes. Excluding disabled plugins from the runtime tool-discovery required-ID set is the narrow maintainable fix; changing global control-plane availability would risk catalog, setup, and doctor flows that intentionally inspect bundled metadata without loading runtime code.
