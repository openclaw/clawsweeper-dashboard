## Summary

Current main already implements this PR's intended optimization in the moved Memory Core plugin path: fallback vector search streams chunk rows with SQLite iterate() and keeps a bounded top-K result set instead of materializing and sorting every candidate. The implementation landed via #73100 in commit 864c4f7ff492f0f514c12557d44f0d6b509231fc, with regression coverage. The changelog entry is still under Unreleased, so no fixed release tag is identifiable yet.

## Best Possible Solution

Close this PR as implemented on current main and keep the merged Memory Core implementation from #73100. Future tuning, if needed, should start from the current `extensions/memory-core/src/memory/manager-search.ts` bounded streaming fallback rather than reviving the old `src/memory` patch.
