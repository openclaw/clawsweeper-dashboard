## Summary

The request is for a LanceDB memory plugin variant, and current main explicitly scopes optional capability like this to plugins distributed outside core. OpenClaw already has the memory-plugin slot, external plugin install flow, community-plugin policy, and a bundled `memory-lancedb` plugin, so this should live as a ClawHub/npm plugin rather than an OpenClaw core issue.

## Evidence

- **Vision scope:** Project vision says optional capability should usually ship as plugins, plugin authors should host and maintain plugins in their own repositories, and the bar for adding optional plugins to core is intentionally high. It also calls out memory as a plugin slot.
