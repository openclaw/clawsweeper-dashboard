## Summary

Keep open. Current `main` still hardcodes the browser tool description around the isolated `openclaw` browser instead of reflecting the configured `browser.defaultProfile`, so the PR's requested behavior is not implemented on `main` even though the browser runtime itself still honors `defaultProfile` when `profile` is omitted.

## Evidence

- **Tool description still hardcodes managed-browser default:** On current `main` (`799a42bd13de586a1a7163cf0bd51673c31de253`), the browser tool description says: omit `profile` by default for the isolated OpenClaw-managed browser (`openclaw`), rather than describing the configured default profile.
  - file: [extensions/browser/src/browser-tool.ts:383](https://github.com/openclaw/openclaw/blob/799a42bd13de586a1a7163cf0bd51673c31de253/extensions/browser/src/browser-tool.ts#L383)
  - command: `nl -ba extensions/browser/src/browser-tool.ts | sed -n '370,391p'`
  - sha: [799a42bd13de](https://github.com/openclaw/openclaw/commit/799a42bd13de586a1a7163cf0bd51673c31de253)
- **Runtime still resolves configured defaultProfile:** The browser config resolver still takes `browser.defaultProfile` from config when set.
  - file: [extensions/browser/src/browser/config.ts:264](https://github.com/openclaw/openclaw/blob/799a42bd13de586a1a7163cf0bd51673c31de253/extensions/browser/src/browser/config.ts#L264)
  - command: `nl -ba extensions/browser/src/browser/config.ts | sed -n '248,295p'`
  - sha: [799a42bd13de](https://github.com/openclaw/openclaw/commit/799a42bd13de586a1a7163cf0bd51673c31de253)
- **Omitted profile uses resolved default at runtime:** When no profile is passed, the browser server context selects `current.resolved.defaultProfile`, so the runtime behavior and tool-description guidance are mismatched.
  - file: [extensions/browser/src/browser/server-context.ts:136](https://github.com/openclaw/openclaw/blob/799a42bd13de586a1a7163cf0bd51673c31de253/extensions/browser/src/browser/server-context.ts#L136)

- **Docs still describe configurable default profile:** The docs explicitly say `defaultProfile: "user"` opts into the signed-in user browser, which reinforces that the tool description should not hardcode `openclaw` as the default for every install.
  - file: [docs/tools/browser.md:161](https://github.com/openclaw/openclaw/blob/799a42bd13de586a1a7163cf0bd51673c31de253/docs/tools/browser.md#L161)
