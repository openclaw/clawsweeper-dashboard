## Summary

Current `main` already addresses the reported restart blocker: gateway lock acquisition now treats a free gateway port as proof that the prior lock owner is stale and removes the lock, the gateway run loop passes the port into lock acquisition on startup and restart, shutdown close paths are bounded, and the changelog records this stale-lock restart fix as shipped in release `2026.4.5`.

## Evidence
