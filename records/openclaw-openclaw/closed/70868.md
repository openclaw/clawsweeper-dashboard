## Summary

Current main now handles the dangerous failure mode reported here: zero-token empty Claude `stop` turns are detected, retried once, repaired for replay, and eligible for model fallback instead of silently succeeding or poisoning future replay. The plugin `before_prompt_build`/`prependSystemContext` path remains supported and documented; the later canonical empty-Claude-stop issue #71880 was closed by maintainers as fixed on main in 90cd9fce85949dc7118cb6ba7921698b07e091ef.

## Best Possible Solution

Keep the current main implementation from the empty-Claude-stop fix: detect zero-token empty `stop` turns, retry once, repair replay history, and allow fallback. Treat #71880 as the canonical shipped fix for this class of empty Claude terminal outputs while leaving the existing `before_prompt_build` system-context API in place.
