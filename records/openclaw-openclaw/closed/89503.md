## Summary

Keep open for maintainer review: the patch addresses a source-reproducible public SDK stream failure, but it intentionally changes public async-iterator semantics from clean completion to rejection, so the remaining decision is contract acceptance rather than a code defect.

## What This Changes

This PR changes the SDK EventHub and normalized event pump so event-pump failures reject `oc.events()` / `Run.events()` streams and adds focused SDK regression tests.

## Best Possible Solution

Land this PR or an equivalent SDK fix after maintainers accept rejected async iteration as the public pump-failure contract; if they prefer non-throwing streams, switch to a typed SDK error event with focused tests before merge.

## Reproduction Assessment

Yes. Source inspection on current main shows pump failures close the normalized EventHub without preserving the cause, and the linked issue plus PR body provide a focused transport-boundary reproduction.

## Solution Assessment

Yes, with a public-contract caveat. Carrying an error through `EventHub.close(error)` keeps the fix at the SDK stream owner boundary and covers active and future normalized consumers; a typed SDK error event is the main alternative if maintainers reject thrown iterator failures.
