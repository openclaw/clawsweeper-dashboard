## Summary

Keep open: the bug boundary is plausible, but this draft PR still has blocking stream-filter ordering defects, unrelated changelog entries, and insufficient DSML-exercising real behavior proof.

## What This Changes

The PR adds a DeepSeek-gated DSML text filter to OpenAI-compatible streaming, wires it into `processOpenAICompletionsStream`, adds parser/filter tests, and updates the changelog.

## Best Possible Solution

Keep the compat-gated suppress-only design, but preserve filter state across chunk boundaries, stream safe text promptly, remove unrelated changelog claims, and require real DSML-exercising proof before merge.

## Reproduction Assessment

Yes, at source level: current main appends string `choice.delta.content` directly to visible assistant text, and the PR's own parser tests show DSML-shaped chunks reaching this seam. I did not establish a fresh live DeepSeek response that naturally emits DSML.

## Solution Assessment

No. The suppress-only, `thinkingFormat: "deepseek"`-gated direction is maintainable, but this implementation finalizes the filter too early, delays safe text streaming, includes unrelated changelog entries, and lacks DSML-exercising real proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep DSML filter state across tool-call chunks:** `src/agents/openai-transport-stream.ts:1543-1544`
  - body: `flushDeepSeekTextFilter()` calls the filter's final `flush()` before same-chunk `delta.tool_calls` or reasoning are processed. If a DSML opener starts in that content chunk and closes in a later chunk, this clears the in-progress state or emits the partial opener, so the later DSML body/close text can leak as visible assistant text. Drain only provably safe text before changing block type, and keep DSML state for later content chunks.
  - confidence: 0.9
- **[P2] Emit safe DeepSeek text without waiting for flush:** `src/agents/deepseek-text-filter.ts:60-62`
  - body: When no DSML opener is present, the filter still retains `MAX_OPEN_TOKEN_LEN - 1` characters unconditionally. Normal short chunks such as `hello` can produce no `text_delta` until later chunks or final flush, regressing existing streaming behavior; retain only a suffix that is actually a prefix of a DSML open token.
  - confidence: 0.88
- **[P2] Remove unrelated changelog entries:** `CHANGELOG.md:253-256`
  - body: This branch only changes DeepSeek stream filtering, but the changelog hunk also claims Control UI/session, Windows exec-approval, and Slack fixes. If merged, release notes would advertise fixes not delivered by this PR, so keep only the DeepSeek entry or rebase onto the commits that actually contain those fixes.
  - confidence: 0.86
