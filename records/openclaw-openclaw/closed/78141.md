## Summary

Keep open: the fal plugin and docs define fal image generation as supported, and current main still has a source-level path where an already-loaded image provider can hide allowlisted fal from `image_generate`, matching the reported symptom.

## What This Changes

Fix the bundled fal image-generation provider so an enabled/allowlisted fal plugin is registered and visible to `image_generate` after upgrade.

## Best Possible Solution

Keep the issue open and patch the generic capability-provider resolver so enabled/allowlisted generation providers are discoverable even when another generation provider is already active.

## Reproduction Assessment

Yes, at source level: an active xai image provider makes `resolvePluginCapabilityProviders` return active providers only, while image-generation providers have no requested-provider collector to cold-load allowlisted fal. I did not run a live gateway with FAL credentials in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is in the generic capability-provider resolution path, not the fal provider implementation, while preserving manifest allow/deny and lazy-loading boundaries.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/78141. On current main, `image_generate` can miss bundled fal when another image-generation provider such as xai is already active: `resolvePluginCapabilityProviders({ key: "imageGenerationProviders" })` returns active providers only because image/video/music generation have no requested-provider collector. Patch the generic capability-provider runtime so enabled/allowlisted manifest-owned generation providers can be resolved/merged without broad-loading unrelated plugins, while preserving `plugins.enabled=false`, deny lists, explicit plugin disablement, and restrictive allowlists. Do not change the fal provider API implementation unless the resolver investigation proves it is necessary. Add regression coverage for an active registry containing one image provider plus a manifest-owned/allowlisted fal image provider, asserting `resolvePluginCapabilityProviders`, `listImageGenerationProviders`, or `getImageGenerationProvider("fal", cfg)` can see fal. Add a single-line CHANGELOG.md Fixes entry for the user-facing regression.
