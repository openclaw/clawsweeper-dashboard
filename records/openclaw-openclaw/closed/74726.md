Close as superseded by #74844. The later open issue tracks the same actionable MCP exposure failure with broader scope and stronger wire-level evidence, while this issue's image-specific symptom is one consequence of configured MCP tools not reaching the normal main-agent tool list.

## What This Changes

Expose configured `mcp.servers` tools to the main OpenClaw agent, including access to an MCP `understand_image` capability when the built-in image-provider path cannot handle the request.

## Best Possible Solution

Use #74844 as the canonical tracker for the remaining product bug: configured `mcp.servers` should be present as prefixed callable tools in normal main-agent outbound requests, with docs or implementation clarifying whether media-understanding MCP tools are called directly by name or through a built-in image fallback adapter.
