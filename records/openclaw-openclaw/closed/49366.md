## Summary

Current main already implements the PR’s intended gateway install hardening: config/env durable values are not embedded as inline service environment literals, auth-profile env refs are preserved for ref-mode daemon installs, gateway auth SecretRefs are not persisted, and newer main also has systemd/launchd service-env handling that makes the PR branch stale.

## Best Possible Solution
