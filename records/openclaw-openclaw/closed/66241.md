## What This Changes

The PR adds best-effort persistence for Discord component and modal registry entries so active buttons, selects, and forms survive gateway restarts.

## Best Possible Solution

Keep the shipped SDK-backed Discord persistence from https://github.com/openclaw/openclaw/pull/75584 and close this branch rather than merging its local JSON-file implementation.

## Reproduction Assessment

No current-main failing reproduction remains. The old source-level path was clear because process-global maps were lost on restart, but current main mirrors entries into `runtime.state.openKeyedStore` and falls back to it after memory is cleared.

## Solution Assessment

Yes for the repository state: the current SDK-backed keyed-store implementation is the maintainable solution. No for this PR branch: it persists bespoke JSON under `~/.openclaw/cache` and is superseded by the shipped state-store implementation.
