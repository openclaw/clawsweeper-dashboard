## Summary

The PR's reported failure path no longer matches current main. Supported thread-bound ACP spawns create a normal `agent:<agent>:acp:<uuid>` session and bind the conversation/thread to that key; inbound routing then uses the bound target session key instead of a derived `:thread:` ACP alias. The exact store-level ACP-copy fallback proposed here is not on main, so this is better closed as obsolete/cannot-reproduce rather than implemented-on-main.

## Best Possible Solution

Close this PR as obsolete against current main. Keep the current conversation-binding routing model, and only revisit store-level ACP inheritance if a current supported flow still produces stale `:thread:` ACP aliases with clear reproduction evidence.
