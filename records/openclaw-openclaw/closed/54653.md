## Summary

Current main already solves the observable stale bundled plugin path problem through a different implementation: doctor detects/removes redundant bundled `plugins.load.paths` aliases, discovery ignores packaged/legacy bundled load-path aliases, and legacy `plugins.installs` records are migrated out of authored config into the plugin index. PR #54653 is therefore obsolete even though its review follow-ups were useful.

## Best Possible Solution

Close this PR and keep the current main implementation: bundled plugin path drift is handled by doctor load-path alias cleanup plus plugin-index migration for legacy install metadata. Any future stale-path bug in the state-managed plugin index should be opened as a new, narrower issue against that state index path rather than reviving this config-level repair PR.
