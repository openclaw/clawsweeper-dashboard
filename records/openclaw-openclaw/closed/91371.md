## Summary

Current main and v2026.6.1 already fix the linked duplicate transcript behavior by overwriting the unflushed full buffer; the remaining append-on-exists optimization is redundant and would be unsafe for an existing session reset path.

## What This Changes

The branch changes `SessionManager.persist()` to append non-assistant entries immediately when the session file exists and adds a resume regression test.

## Best Possible Solution

Keep the shipped overwrite-on-unflushed implementation; any future immediate-append optimization should first prove the header is actually present on disk and cover the run-preparation empty-file reset path.

## Reproduction Assessment

No current-main failure was found from source inspection: the unflushed first-assistant path now overwrites the full buffer. The PR-introduced missing-header risk is source-reproducible through `prepareSessionManagerForRun()` emptying an existing no-assistant file before a user append.

## Solution Assessment

No; this PR is not the best fix because the central bug is already fixed by the shipped overwrite path, and the new `existsSync` optimization is unsafe for a sibling reset path.
