## Summary

Keep this PR open: current main still has the Active Memory channel-resolution bug, and the updated PR now contains the focused code fix, regression coverage, and changelog entry with no blocking review findings from this pass.

## What This Changes

The PR filters colon-containing Active Memory hook `channelId` values out of embedded recall `messageChannel` resolution, falls back to `messageProvider`, and adds a Telegram forum-topic regression test plus changelog entry.

## Best Possible Solution

Land the focused Active Memory resolver fix after normal exact-head CI or targeted Active Memory test proof.

## Reproduction Assessment

Yes. Source inspection shows Telegram forum-topic session keys can yield a colon-containing hook `channelId`, current Active Memory passes that value as the embedded recall `messageChannel`, and bundled plugin dir-name validation rejects `:`.

## Solution Assessment

Yes. Filtering scoped conversation ids inside the Active Memory resolver is the narrow maintainable fix because hook context can still expose raw conversation ids while the embedded runner receives the runnable plugin channel name.
