## Summary

Current `main` contains a targeted fix for the reported Codex-through-proxy failure: Codex OAuth refresh now bootstraps the global undici env-proxy dispatcher before calling the PI OAuth helpers, and the auth refresh path routes through that plugin hook. The fix is covered by a unit test on `main`, but it does not appear in the latest tagged release yet.

## Evidence

- **Codex runtime now enables env-proxy before OAuth calls:** `extensions/openai/openai-codex-provider.runtime.ts` wraps both `getOAuthApiKey` and `refreshOpenAICodexToken` with `ensureGlobalUndiciEnvProxyDispatcher()`, directly addressing the proxy-env gap described in the issue.
