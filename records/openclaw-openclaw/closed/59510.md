## Summary

Current main already resolves the central #59510 problem: gateway/node host exec now defaults to no-prompt YOLO policy (`security=full`, `ask=off`, `askFallback=full`), the effective-policy resolver and runtime honor those defaults, and users have CLI shortcuts/docs for synchronizing config plus the approvals file and adding wildcard allowlist entries. The exact proposed `exec.allowAlways` field and comma-batch syntax were not adopted, but the reported approval-spam and difficult no-prompt setup from v2026.4.1 are addressed well enough by the shipped implementation.

## Best Possible Solution

Close #59510 as implemented on main. Keep the shipped no-prompt defaults and `exec-policy`/approval CLI as the canonical UX; any future request for an exact batch-allowlist syntax or timed session-only mode should be filed as a narrower product enhancement if maintainers still want that specific workflow.
