## Summary

Current main already preserves the post-tool completion watchdog for the reported `rawResponseItem/completed` tool-output handoff, with regression coverage and docs; the fix is current-main-only and not in `v2026.5.12`.

## What This Changes

The issue asks for the Codex bridge's 60s turn-completion idle watch to stay armed when post-tool `rawResponseItem/completed` notifications arrive before `turn/completed`.

## Best Possible Solution

Keep the current type-aware implementation and ship it in the next release; only broaden the exclusion if new logs show a non-`custom_tool_call_output` raw item should preserve the short post-tool watchdog too.

## Reproduction Assessment

No current-main failure was reproduced. The reported path is now covered by a focused source-level regression test that exercises `item/tool/call` followed by raw `custom_tool_call_output` completion and confirms the short watchdog fires.

## Solution Assessment

Yes. Current main solves the reported tool-output handoff with a narrower type-aware guard instead of excluding every `rawResponseItem/completed`, preserving the separately tested raw-assistant behavior.
