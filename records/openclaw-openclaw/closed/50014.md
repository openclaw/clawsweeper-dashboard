## Summary

Current `main` already handles the stale auto-profile case conservatively: expired profiles are filtered out of automatic auth selection, auto session overrides are cleared when they are no longer eligible, explicit user locks remain strict, and `google-vertex` still falls back to ADC/env auth when no valid profile remains. This supersedes PR #50014's intended behavior for #49885 on current `main`.

## Evidence
