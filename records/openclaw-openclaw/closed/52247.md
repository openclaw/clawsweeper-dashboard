## Summary

Current main already solves the session_status model mismatch that #52247 targets. The shipped implementation uses the persisted runtime session model (modelProvider/model) for session_status when no explicit session override exists, which covers channel-routed modelByChannel sessions and keeps status-card auth/usage metadata aligned with the same provider/model path. The fix was merged via #55425 and shipped in v2026.3.28, so this open PR is now obsolete.

## Best Possible Solution

Close #52247 as already implemented on main. Keep the shipped #55425 runtime-model implementation because it fixes the user-visible session_status mismatch while preserving aligned auth, usage, and status-card metadata; any new channel override execution bugs should be tracked separately from this now-obsolete PR.
