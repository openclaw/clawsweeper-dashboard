## Summary

Keep open. Current main still runs the Linux installer apt paths directly with `apt-get install -y -qq` and no `DEBIAN_FRONTEND=noninteractive` or dpkg config-file options, including the exact build-tools and NodeSource path described in #41146. The linked fix PR #58399 was closed unmerged, so it does not supersede the issue, and the issue has multiple independent user confirmations.

## Best Possible Solution

Keep #41146 open and implement a focused installer fix in `scripts/install.sh`: set/preserve `DEBIAN_FRONTEND=noninteractive` before Linux apt/NodeSource operations, add dpkg `--force-confdef`/`--force-confold` options to apt installs for build tools, Node.js, and Git, and add static or Docker-based installer coverage for the Ubuntu 24.04 `curl | bash` non-TTY path. #58399 is useful prior art but should not close the issue unless the fix is landed or replaced by an equivalent implementation.
