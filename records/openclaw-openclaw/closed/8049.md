## Summary

Current main already covers the reported need with supported run-scoped env injection for skills and CLI backends, so the proposed exec-hook metadata is no longer necessary to solve the `.bashrc`/preprocessing use case.

## Evidence

## Risks / Open Questions

- The exact proposed `metadata.openclaw.execHooks` schema is not present; this closure treats the issue as resolved by the supported run-scoped env/config surfaces instead of the requested regex/prepend hook format.
- If someone is using the bundled `claude-cli` backend rather than plain `exec`, some ambient vars are intentionally cleared for hardening, so Claude-specific routing vars should be set through backend config instead of inherited shell state.
