## Summary

Keep open. The current Logs tab still depends on `logs.tail` returning lines from the configured gateway file log, and current `main` does not add a targeted fix for the reported empty Web UI Logs page after 2026.4.24. The report is fresh, labeled regression, and lacks enough runtime data to prove it is the already-handled cross-date CLI log issue or another resolved duplicate.

## What This Changes

The issue reports that after upgrading to 2026.4.24, the Web UI Logs page on Ubuntu 20.04 shows no log entries when logs are expected.

## Best Possible Solution

Keep the issue open and reproduce the 2026.4.24-to-current upgrade path on Linux. Maintainers should collect the `logs.tail` payload, browser console/network output, configured `logging.file`, file existence/permissions/size, and a comparison with `openclaw logs --follow`; then fix the confirmed generic logging or Control UI diagnostic path and add a regression test for that failure mode.
