## Summary

Current main already addresses the reported 3000ms WebSocket handshake/connect-challenge timeout: the gateway now uses a shared 10_000ms default, exposes env overrides for both preauth and connect-challenge timeouts, documents that contract, and has regression tests covering the new behavior. The latest release `v2026.4.22` contains the same implementation.
