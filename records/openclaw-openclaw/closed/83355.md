## Summary

Current main and shipped releases no longer use the failing Docker-era Discord bundle resolution path, and the loader/package changes address bare OpenClaw SDK imports before Discord subagent_ended hooks run.

## What This Changes

The issue asks for Discord subagent_ended hooks to load and process completion events without ERR_MODULE_NOT_FOUND failures in Docker-era packaged builds.

## Best Possible Solution

Keep the shipped loader fallback and external Discord package boundary; use the related direct-announce/orphan-prune issue for any remaining user-visible subagent completion loss.

No live current-main Docker repro was run. The old Docker-era failure is consistent with the reported bundled extension import path, while current source and release artifacts no longer use that path and include module-resolution fallback.

## Solution Assessment

Yes. The current solution fixes SDK import resolution at the plugin loader/package boundary and keeps Discord-specific hook logic in the Discord plugin; the newer direct-announce/orphan-prune failure is a separate remaining path.
