## Summary

Current `main` already returns matched chunk snippets and line ranges for memory search results instead of truncating from file start. The fix is present in both the builtin search path and the QMD path, with targeted tests covering the line/snippet behavior.

## Evidence

- **Builtin search returns chunk text, not file head:** Both vector and keyword search results are built from the matched chunk `text` plus its `start_line`/`end_line`, so CLI output uses the relevant chunk snippet rather than reading from byte 0 of the file.
  - file: [extensions/memory-core/src/memory/manager-search.ts:197](https://github.com/openclaw/openclaw/blob/ed6b487e2033d87a85b5713a964cc677c70353f8/extensions/memory-core/src/memory/manager-search.ts#L197)
  - command: `git blame -L 197,205 -- extensions/memory-core/src/memory/manager-search.ts`
  - sha: [ed6b487e2033](https://github.com/openclaw/openclaw/commit/ed6b487e2033d87a85b5713a964cc677c70353f8)
- **QMD parser preserves explicit line metadata:** The QMD JSON parser now reads `start_line` / `end_line` into `startLine` / `endLine`, which gives the renderer exact match coordinates instead of inferring from file start.
  - file: [packages/memory-host-sdk/src/host/qmd-query-parser.ts:101](https://github.com/openclaw/openclaw/blob/ed6b487e2033d87a85b5713a964cc677c70353f8/packages/memory-host-sdk/src/host/qmd-query-parser.ts#L101)

  - sha: [ed6b487e2033](https://github.com/openclaw/openclaw/commit/ed6b487e2033d87a85b5713a964cc677c70353f8)
- **QMD manager uses explicit lines or snippet headers for result ranges:** Search results keep the returned snippet text and resolve display ranges from explicit line metadata first, then fall back to snippet header widths if needed.
  - file: [extensions/memory-core/src/memory/qmd-manager.ts:1210](https://github.com/openclaw/openclaw/blob/ed6b487e2033d87a85b5713a964cc677c70353f8/extensions/memory-core/src/memory/qmd-manager.ts#L1210)
  - command: `nl -ba extensions/memory-core/src/memory/qmd-manager.ts | sed -n '1180,1260p'`

- **Regression tests cover relevant snippet ranges:** Tests assert that mcporter/QMD search returns `notes/welcome.md` with `startLine: 8`, `endLine: 10`, and the matched snippet text, including the fallback case where only a start line is returned.
  - file: [extensions/memory-core/src/memory/qmd-manager.test.ts:2482](https://github.com/openclaw/openclaw/blob/ed6b487e2033d87a85b5713a964cc677c70353f8/extensions/memory-core/src/memory/qmd-manager.test.ts#L2482)
