## Summary

Keep open: current main still has a source-reproducible empty-payload logging path that this PR is trying to address, but the branch is stale/unmergeable and should be ported against the current runner metadata before merge.

## What This Changes

The PR adds a silent-completion marker for exact `NO_REPLY` embedded runs, gates local and gateway CLI empty-result logging on that marker, and adds focused regression tests for the runner helper and both logging surfaces.

## Best Possible Solution

Port the fix to current main using one canonical delivery-consumable silent-completion signal from existing runner metadata, then cover exact `NO_REPLY` empty payloads in local delivery and gateway CLI tests.

## Reproduction Assessment

Yes, from source inspection. Exact `NO_REPLY` payloads can be filtered to zero payloads, and current local delivery still logs `No reply from agent.` for that empty result without checking the silent completion state.

## Solution Assessment

Mostly yes. The PR targets the right boundary, but the best current-main version should reuse or normalize the existing `finalAssistantRawText` / `terminalReplyKind` metadata instead of adding an uncoordinated parallel shape.
