## Summary

The central bug remains on current main: recovery retries the current Anthropic request without thinking blocks but leaves the rejected signed-thinking state persisted for later turns. This PR extends that established recovery path through the existing transcript rewrite and session refresh mechanisms, has convincing live Anthropic proof and focused regression coverage, and has no concrete correctness or security defect; because it intentionally removes all thinking/redacted-thinking blocks from the active persisted branch when the provider cannot identify the rejected block, the maintainer-protected PR still needs explicit session-state owner acceptance before merge.

## What This Changes

The branch adds a successful-retry notification to Anthropic thinking recovery, durably strips thinking blocks from the active persisted session branch, refreshes the runner’s in-memory session state, and adds helper and integration regression tests.

## Best Possible Solution

Land the existing-path durable repair if session-state owners accept provider-confirmed active-branch reasoning removal as the canonical recovery policy, while retaining fresh reasoning generated by the successful retry and avoiding signature-size heuristics, runtime fallback readers, or a second persistence path.
