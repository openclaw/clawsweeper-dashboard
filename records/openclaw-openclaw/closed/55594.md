## Summary

Current `main` does not support the reported root cause. OpenClaw still requests `GatewayIntents.MessageContent`, `channels status` reports `intents:content=limited` from Discord application flags rather than the gateway identify payload, and the reporter's wildcard guild config matches an already-fixed empty-channel-map path rather than a missing-intent negotiation bug.

## Evidence
