## Summary

This is a duplicate of the open Android background-location feature request at https://github.com/openclaw/openclaw/issues/68581; the Samsung/Tailscale details are useful validation context but point to the same foreground-only `location.get` limitation already tracked there.

## What This Changes

Request support for Android node `location.get` while the app is backgrounded or locked, using explicit background-location permission, a location foreground service, settings UI, and battery guidance.

## Best Possible Solution

Keep https://github.com/openclaw/openclaw/issues/68581 as the canonical design and implementation thread for Android Always/background location support, preserving this issue’s Samsung Galaxy A54 and Tailscale setup as validation context there.

## Reproduction Assessment

Yes for the current limitation: source and tests show Android `location.get` returns `LOCATION_BACKGROUND_UNAVAILABLE` whenever the app is not foregrounded. I did not run a live Android device repro, but the code path is direct and already covered by a unit test.

## Solution Assessment
