Full review comments:

- **[P2] Use the billed aggregate for usage.total:** `src/cron/isolated-agent/run.ts:1051-1065`
  - body: `totalTokens` comes from `deriveSessionTotalTokens`, which is documented as a prompt/context snapshot and intentionally excludes output tokens. When that value is present, this event can report a lower `usage.total` than the billed cron run, so downstream token and cost dashboards remain wrong.
  - confidence: 0.97
- **[P2] Emit the documented context shape:** `src/cron/isolated-agent/run.ts:1068-1070`
  - body: `DiagnosticUsageEvent.context` exposes `limit` and `used`, and the OTel exporter records only those fields. Emitting `contextTokens` and `promptTokens` here both violates the typed event contract and drops cron context metrics for subscribers.
  - confidence: 0.96
