## Summary

Keep open for maintainer review: the patch looks correct and proof is sufficient, but it intentionally changes existing terminal-link output by stripping all C0/C1 controls, so compatibility acceptance remains a maintainer decision.

## What This Changes

The branch replaces ESC-only terminal-link sanitization with C0/C1 control stripping for labels, URLs, and fallback text, and adds focused terminal-core regression tests.

## Best Possible Solution

Merge the boundary-level terminal-link sanitizer with the empty-fallback regression preserved if maintainers accept that terminal links should never emit C0/C1 controls; otherwise narrow the sanitizer to the specific OSC delimiter/control threat.

## Reproduction Assessment

Yes. Current main is source-reproducible because it strips ESC but not BEL before emitting OSC-8 links, while BEL is also the delimiter used in the generated OSC-8 sequence.

## Solution Assessment

Yes, if maintainers accept the compatibility tradeoff. Sanitizing at formatTerminalLink is the shared boundary for direct terminal links and docs-link/plugin-SDK callers; caller-side sanitizing would be incomplete, and sanitizeTerminalText would be broader than this fix needs.
