## Summary

Current main already fixes the reported OpenAI Codex encrypted-reasoning replay failure: it strips unproven or mismatched encrypted replay before request assembly and retries once without encrypted_content on invalid_encrypted_content. The fix is merged on main but not in the latest v2026.5.18 release the reporter used.

## What This Changes

The issue reports that openai-codex ChatGPT-plan OAuth sessions replay stale encrypted reasoning after token refresh, causing persistent 400 invalid_encrypted_content failures and empty user replies.

## Best Possible Solution

Keep the merged main implementation from https://github.com/openclaw/openclaw/pull/84367 and let the fix ship in the next release rather than opening another repair lane.

## Reproduction Assessment

I did not rerun a live OAuth token-refresh repro in this read-only review.

## Solution Assessment

Yes. The current main fix is the maintainable boundary: provenance-bound replay prevents stale encrypted_content from being sent, and the targeted retry recovers provider-side invalid_encrypted_content without routing to model fallback.
