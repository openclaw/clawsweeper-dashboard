## Summary

Current `main` already contains the relevant Codex OAuth hardening: it no longer bootstraps `openai-codex` from external `.codex` state, it returns an existing usable stored OAuth credential instead of refreshing/re-authorizing it, and it has explicit recovery for `refresh_token_reused` by reloading newer on-disk credentials or retrying once with the rotated refresh token.
