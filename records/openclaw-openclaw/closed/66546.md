## Summary

Current main supersedes this PR's central active-transcript growth fix with the compaction-based `agents.defaults.compaction.maxActiveTranscriptBytes` guard plus `truncateAfterCompaction`. Main does not carry the proposed `session.maintenance.transcriptRotateBytes` / `transcriptMaxLines` API, but it now uses semantic compaction and successor JSONL rotation instead of merging this XL raw transcript-rotation patch.

## Best Possible Solution

Close this PR as superseded by current main's compaction-based active transcript size guard. Keep the shipped `maxActiveTranscriptBytes` plus successor-transcript compaction direction, and open a narrower maintainer-owned follow-up only if OpenClaw still needs raw archival, line caps, or bulk inactive-transcript cleanup under `session.maintenance`.
