## Summary

Current main already implements the PR's central Windows update restart fix: the update restart handoff uses a hidden .cmd wrapper that runs embedded PowerShell via -Command, avoids -File/.ps1 execution-policy exposure, polls with Get-NetTCPConnection first, falls back to parsing netstat.exe output directly in PowerShell, and contains no findstr pipeline in the update restart helper. The PR's incidental diagnostics-prometheus lockfile sync is also already present on main.

## Best Possible Solution

Close this PR as implemented on main and keep the current main restart-helper implementation. If maintainers want additional changelog attribution for #59394 or #57682, that can be handled as a separate tiny docs/changelog follow-up rather than keeping this obsolete PR branch open.
