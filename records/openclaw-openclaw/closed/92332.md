## Summary

The proposed integration is useful, but current main already exposes the credential-free web-search provider seam it needs, while OpenClaw’s vision and repository policy route optional third-party providers to independently maintained plugins and ClawHub rather than bundling them into core.

## What This Changes

Request to bundle SCOUTS-AI as a no-key JSON fallback in the web search provider auto-detection chain.

## Best Possible Solution

Implement SCOUTS-AI as an independently maintained code plugin, publish it through ClawHub, and document its endpoint, query disclosure, rate limits, failure behavior, and explicit opt-in configuration.

## Reproduction Assessment

Not applicable; this requests a new bundled provider rather than reporting broken documented behavior. The external endpoint itself was live-verified and matches the proposed response shape.

## Solution Assessment

No; bundling is not the narrowest maintainable solution because the current credential-free web-search plugin contract can host the integration externally without making a public third-party endpoint part of OpenClaw’s default trust boundary.
