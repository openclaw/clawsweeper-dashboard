## Summary

This report is a same-day duplicate of the earlier open WebChat scrollback issue: both describe older current-session messages becoming unreachable because the UI loads only a fixed latest-history window and offers no upward pagination.

## What This Changes

Restore access to older messages in long WebChat conversations when users scroll upward.

## Best Possible Solution

Keep https://github.com/openclaw/openclaw/issues/92317 as the canonical report and implement bounded cursor- or page-based older-history loading across the gateway contract and WebChat while preserving current payload caps.

## Reproduction Assessment

Yes at source level: v2026.6.5 and current main request only the latest 100 entries and expose no older-page load path, so longer sessions cannot reveal earlier entries by scrolling.

## Solution Assessment

Yes, consolidating into the earlier report and adding pagination or cursor-based upward loading is the narrowest durable solution; increasing or configuring the limit would retain the same failure mode.
