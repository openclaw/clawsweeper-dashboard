## Summary

Keep open. Current main still uses the bundled `require("../config/config.js")` logging config path reported in the issue, still catches failures by returning `undefined`, and still falls back to the default rolling `/tmp/openclaw/openclaw-YYYY-MM-DD.log` path when no logging config is loaded. The issue discussion includes multiple confirmations on packaged releases, including 2026.4.14, and I did not find a current-main source or packaging change that closes the gap.
