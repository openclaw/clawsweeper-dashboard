## Summary

Current `main` already implements the bundled-channel allowlist exemption, keeps the bundled-origin trust boundary, and has regression coverage for the Telegram/channel-id collision cases this PR targets.

## Evidence

- **Current main routes plugin activation through the bundled-channel allowlist bypass:** `src/plugins/config-state.ts:182-193` calls `resolvePluginActivationDecisionShared(...)` with `allowBundledChannelExplicitBypassesAllowlist: true`, which is the active path imported by loader and other plugin activation callers from `config-state.js`.
  - file: [src/plugins/config-state.ts:182](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/plugins/config-state.ts#L182)
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Bypass is restricted to bundled origins and still enforces allowlists otherwise:** `src/plugins/config-activation-shared.ts:116-120` only marks channel-config enablement as explicit when `origin === "bundled"`, and `src/plugins/config-activation-shared.ts:221-240` allows that explicit bundled case past the allowlist while returning `not-in-allowlist` for other unmatched plugins.
  - file: [src/plugins/config-activation-shared.ts:116](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/plugins/config-activation-shared.ts#L116)
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Regression tests cover the reported behavior and the spoofing boundary:** `src/plugins/config-state.test.ts:192-217` asserts bundled Telegram stays enabled with a non-empty allowlist and that a non-bundled plugin reusing `telegram` still resolves to `{ enabled: false, reason: "not in allowlist" }`.
  - file: [src/plugins/config-state.test.ts:192](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/plugins/config-state.test.ts#L192)
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
