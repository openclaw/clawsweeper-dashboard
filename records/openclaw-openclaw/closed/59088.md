## Summary

Current main already includes an Agents > Files Control UI surface backed by agents.files.list/get/set. It exposes the core workspace files called out by the request, including USER.md, MEMORY.md, SOUL.md, TOOLS.md, HEARTBEAT.md, and related bootstrap files, and lets users select, edit, preview, reset, and save them from the dashboard. The changelog records the Agents dashboard for managing agent files in v2026.2.2, with later Files-panel polish in v2026.4.5.

## Best Possible Solution

Keep the existing allowlisted Agents > Files editor as the core workspace-file editing path. If maintainers want a full arbitrary workspace/media explorer later, track that separately with an explicit security and permission design rather than reopening this core-file editor request.
