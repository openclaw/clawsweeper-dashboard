## Summary

Current main no longer matches the reported buffered-dispatcher behavior: the provider wrapper delegates to dispatcher creation, and the dispatcher delivers each already-formed reply payload once without a DEFAULT_CHUNK_LIMIT or markdown chunking pass. The issue’s external atticus-channel workaround remains useful context, but the specific OpenClaw core splitting path described here is obsolete on main.

## Evidence

  - command: `GitHub issue #59097 page inspection`
