## Summary

Keep open: current main and the reported release still route configured-agent `sessions_send` traffic to the target agent’s main session, while task-level isolation would require an explicit API/product decision rather than cleanup closure.

## What This Changes

The issue requests task-level session isolation for agent-to-agent delegation via `sessions_send` so unrelated delegated work does not accumulate in the target agent’s fixed main session.

## Best Possible Solution

Add a deliberate, opt-in isolated delegation path for `sessions_send`, or formally steer this workflow to `sessions_spawn`, without silently breaking existing main-session continuity.

## Reproduction Assessment

Yes from source inspection: current `sessions_send` maps configured-agent `agentId` delegation to the target agent’s main session and lacks a task-scoped send option. I did not run the live Docker workflow that demonstrates eventual context exhaustion.

## Solution Assessment

Unclear as proposed: an unconditional default change would be compatibility-sensitive, while an opt-in send isolation option or clearer official `sessions_spawn` workflow looks safer.
