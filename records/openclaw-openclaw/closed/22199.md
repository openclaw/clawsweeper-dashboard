## Summary

Current `main` already gives the Control UI a per-agent visual identity, but the shipped surface is `agents.list[].identity.{name,emoji,avatar}` with top-level `ui.assistant` as an override/fallback layer, not a nested per-agent `ui.assistant` block. The active session/agent identity is resolved server-side, returned by `agent.identity.get`, refreshed by the web UI on session switches, and rendered into the chat welcome/header/message-avatar surfaces.

The same implementation is present in the released `v2026.4.22` snapshot (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). If you were specifically looking for the exact config key shape from the proposal, that part did not land, but the functional per-agent UI identity behavior already has.
