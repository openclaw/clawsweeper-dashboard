## Summary

Keep open: the PR is a focused, mostly correct fix for the linked Windows gateway Scheduled Task bug, but it still needs maintainer acceptance of the VBS/WScript default and required checks are not green. I found one low-priority cleanup defect around generated `gateway.vbs` uninstall behavior.

## What This Changes

The PR changes generated Windows gateway Scheduled Task installs to use the existing windowless `gateway.vbs` launcher by default, preserves caller-selected task identity, adds focused daemon tests, and updates Windows docs.

## Best Possible Solution

Merge the focused scheduler fix after required checks are green, maintainers accept the VBS/WScript default, and the generated `gateway.vbs` uninstall cleanup is either fixed or explicitly accepted as non-blocking.

## Reproduction Assessment

Yes. Current main and `v2026.6.9` build gateway service env without `OPENCLAW_WINDOWS_TASK_HIDDEN_LAUNCHER`, while scheduler launcher selection only chooses `.vbs` when that flag is truthy; the linked issue supplies Windows user steps and impact.

## Solution Assessment

Yes for the code shape, with maintainer compatibility review still required. Reusing the existing hidden-launcher path at the generated service-env boundary is narrower than adding a new launcher mechanism, but making VBS the default is upgrade-sensitive.
