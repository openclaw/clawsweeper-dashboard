## Summary

Current main already implements the PR's intended packaged CLI compile-cache isolation well enough, and the behavior is present in the v2026.4.27 release tag. The shipped implementation scopes packaged cache directories under OpenClaw/version/install metadata in both bootstrap paths, preserves source-checkout cache disablement, passes a string cacheDir to Node's API, and has focused tests.

## What This Changes

The PR derives a packaged CLI compile-cache directory from an install-root/version scope, updates both bootstrap paths to pass a string cacheDir, and adds tests plus changelog/docs notes for NODE_COMPILE_CACHE behavior.
