## Summary

Current main already routes Feishu message-tool media sends to inline image messages when the source is an image path/URL. The current Feishu send path no longer depends on a `filename`/`fileName` param mapping for the reported reproduction; it derives the loaded basename/MIME and chooses `msg_type: "image"` accordingly. The same implementation is present in release `v2026.4.22`.

## Evidence
