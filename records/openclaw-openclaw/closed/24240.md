## Summary

Current main already supports append-style preview replies instead of replacement via channel preview streaming `block` mode (with legacy Slack `append` mapping), and the changelog shows that surface shipped in `v2026.2.21`.

## Close Comment

Codex review: this appears to already be implemented on current `main`. OpenClaw now has explicit preview-streaming modes where `partial` replaces the current preview and `block` appends chunked updates instead (`docs/concepts/streaming.md:115-116`, `docs/channels/slack.md:556-557`). The legacy Slack `streamMode: "append"` path is still mapped to the canonical `streaming: "block"` setting in `extensions/slack/src/streaming-compat.ts:43-50`, and Discord’s block-preview implementation appends streamed deltas into accumulated draft text rather than overwriting prior content in `extensions/discord/src/monitor/message-handler.process.ts:721-744`. The changelog also shows this preview-streaming surface shipped in `v2026.2.21` (`CHANGELOG.md:4348-4360`). Closing as already implemented on main. If this is still happening on a recent build, please open a new issue with the specific channel/plugin and the relevant `channels.<channel>.streaming` setting so maintainers can check that surface directly.
