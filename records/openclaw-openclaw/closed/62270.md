## Summary

Keep open. Current `main` does not implement a `gmicloud` provider, auth choice, or docs entry, and the PR discussion already surfaced that the proposed onboarding commands and provider claims are unsupported. That makes the PR incorrect as written, but it does not meet any of the allowed conservative auto-close reasons.

## Evidence

- **No GMI Cloud implementation on current main:** Repository search on current `main` found no `gmicloud`, `GMI Cloud`, `gmi-serving`, `gmicloud-api-key`, or `GMI_CLOUD_API_KEY` matches under `docs/providers`, `src/cli`, `src/commands`, `src/plugins`, or `extensions`. This matches the review concern that the PR is documenting a provider not present in the codebase.
  - command: `rg -n "GMI Cloud|gmicloud|gmi-serving|gmicloud-api-key|GMI_CLOUD_API_KEY" -S docs/providers src/cli src/commands src/plugins extensions`

- **Onboard provider flags are generated from manifest-backed auth choices:** `onboard` builds provider auth flags from `resolveManifestProviderOnboardAuthFlags()` and registers those as CLI options, so a documented `--auth-choice gmicloud-api-key` or `--gmicloud-api-key` flow requires a real manifest entry first.
  - file: [src/cli/program/register.onboard.ts:50](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/cli/program/register.onboard.ts#L50)

Current bundled providers follow that contract; for example, OpenRouter declares `openrouter-api-key` and `--openrouter-api-key` in its manifest. There is no analogous bundled `gmicloud` manifest in `extensions/` on current main.
  - file: [src/plugins/provider-auth-choices.ts:185](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/plugins/provider-auth-choices.ts#L185)

- **Example of a valid bundled provider auth choice:** OpenRouter's bundled manifest declares provider id, env var, auth choice id, and CLI flag in metadata. That is the wiring the PR's docs assume exists for GMI Cloud, but no such bundled metadata exists on `main`.
  - file: [extensions/openrouter/openclaw.plugin.json:4](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/extensions/openrouter/openclaw.plugin.json#L4)
