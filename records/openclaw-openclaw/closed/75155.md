## What This Changes

Render macOS Talk Mode user voice transcripts in the WebChat/Control UI chat history like typed user messages.

## Best Possible Solution

Keep the current transcript-event bridge and let the next release ship it; track any remaining assistant-duplicate echo as a separate focused UI bug if it persists.

## Solution Assessment

Yes. Subscribing the active WebChat session to canonical transcript events and rendering user-role session.message payloads is narrower and more maintainable than adding a Talk-only echo setting.
