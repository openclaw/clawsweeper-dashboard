## Summary

Keep open. Current main has timeout and circuit-breaker mitigations, but Active Memory is still a foreground `before_prompt_build` hook that awaits embedded recall before the main prompt is built; the open, unmerged #75200 is the active implementation candidate for this issue.

## What This Changes

Active Memory direct-chat recall should complete quickly or fail open without making normal replies wait for a full recall timeout across model configurations.

## Best Possible Solution

Review and land #75200 or an equivalent focused fix that makes the memory-only recall path cheap and deterministic enough for foreground chat, then close this after merged validation.

## Reproduction Assessment

Yes for source-level reproduction: an eligible interactive chat turn reaches Active Memory's `before_prompt_build` hook, which awaits embedded recall before the main prompt is built. I did not run a live provider/gateway reproduction in this read-only review.

## Solution Assessment

No, current main is not the complete solution because it bounds the wait and can circuit-break later calls, but it still puts the first slow recall in the reply-critical path. The narrowest maintainable path is to review and land the focused memory-only recall latency fix in #75200 or a replacement with equivalent coverage.
