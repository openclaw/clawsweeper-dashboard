## Summary

Current main already contains the complete delete-confirm listener cleanup through the merged replacement https://github.com/openclaw/openclaw/pull/76318, including the same-button toggle path and regression tests. This older branch should not merge because it only covers Cancel/Delete and leaves a stale-listener path that the shipped replacement fixed.

## What This Changes

The branch updates `ui/src/ui/chat/grouped-render.ts` so Cancel and Delete in the Control UI delete confirmation popover call a shared cleanup helper for the document click listener.

## Best Possible Solution

Keep the shipped replacement implementation from https://github.com/openclaw/openclaw/pull/76318 and close this overlapping older branch instead of merging the partial fix.

## Reproduction Assessment

Yes. By source inspection, the original/base path and this PR branch can leave the document capture listener behind after a same-button toggle; current main no longer reproduces because every dismiss path goes through cleanup.

## Solution Assessment

Yes for current main, no for this PR branch. The merged replacement is the narrow maintainable solution because it centralizes cleanup, guards delayed listener attachment, and adds focused coverage for all dismiss paths.
