## Summary

Current main already creates an OpenClaw-owned package root before bundled runtime-dependency npm installs and has regression coverage for the parent-root/npm-prefix escape mode described in #72374.

## Best Possible Solution

Close #72374 as implemented on current main and leave any remaining upgrade-path/runtime SDK package staging symptoms to the narrower related report #72415 or a new focused issue if they still reproduce against a build containing 9be8d43c3182c2b773bbb25a79a08895320addab.

## Evidence

- **Implementation writes a local install manifest before npm:** `installBundledRuntimeDeps` creates the install root and execution root, then calls `ensureNpmInstallExecutionManifest` before spawning npm with `cwd` set to the execution root. The manifest writes a local `package.json` with `name: openclaw-runtime-deps-install` and `private: true`, preventing npm from relying on an ancestor package root.
