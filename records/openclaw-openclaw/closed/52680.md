## Summary

Current main already prevents the reported run abort for out-of-range `read` offsets by routing agent tools through the custom tool adapter, which catches non-abort tool exceptions and returns a structured diagnostic tool result. The lower-level pi read implementation still throws for EOF offsets, but that thrown error is now contained at the OpenClaw agent tool boundary and surfaced as `{status:"error", tool:"read", error:"Offset ..."}` instead of aborting the run.

## Best Possible Solution

Close this issue as implemented on main because the run-abort failure mode is now contained at the OpenClaw agent tool adapter boundary. If maintainers still want read-specific clamping with returned file content, that should be a narrower enhancement or follow-up PR with focused tests for `src/agents/pi-tools.read.ts`.
