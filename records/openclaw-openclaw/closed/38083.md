## Summary

Current main already replaces Slack’s tight auth retry loop with OpenClaw-owned reconnect handling: Bolt auto-reconnect is disabled, retries use bounded exponential backoff with a 12-attempt cap, and token-expired-class auth failures are treated as non-recoverable so the channel is skipped instead of looping indefinitely. The same implementation is present in the shipped release `v2026.4.22`.

## Evidence

- **SDK auto-reconnect disabled:** Slack socket mode is created with `autoReconnectEnabled: false`, so OpenClaw, not Bolt, owns reconnect behavior. This same code is present in `v2026.4.22`.
  - file: [extensions/slack/src/monitor/provider.ts:205](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/extensions/slack/src/monitor/provider.ts#L205)
