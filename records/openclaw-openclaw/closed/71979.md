## Summary

Current main already fixes the reported Control UI avatar CSP regression: the CSP `img-src` directive now allows locally generated `blob:` URLs, while keeping remote image URLs blocked. The report matches v2026.4.24, whose tag still had `img-src 'self' data:` without `blob:`.

## Best Possible Solution

Keep the current main implementation: allow `blob:` in the Control UI `img-src` CSP for locally managed avatar object URLs, keep remote `http(s)` images blocked, and let the next release carry this v2026.4.24 regression fix. The related 401 avatar route issue (#71551) remains handled by the authenticated fetch/blob path.
