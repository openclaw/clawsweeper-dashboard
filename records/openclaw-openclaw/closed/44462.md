## Summary

Current `main` already supports `hooks.mappings[].agentId` end-to-end for webhook routing, with docs, tests, and a changelog entry showing the feature shipped in release `2026.2.9`.

## Evidence

## Risks / Open Questions

- The issue body's optional idea of templated `agentId` values (for example `"{{workspace}}"`) is not covered by the current mapping implementation; the implemented and documented behavior is direct string `agentId` routing.
