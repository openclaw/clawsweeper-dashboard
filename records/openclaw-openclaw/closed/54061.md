## Summary

Current main already suppresses non-duplicate inline/final assistant payloads when a successful `message(action="send")` targets the same originating channel conversation. The fix path is target-based, not text-dedup-only, and it is present in the latest release tag v2026.4.26.

## Best Possible Solution

Close this as implemented on main and keep the shipped target-aware suppression. If maintainers still want a global opt-in mode that suppresses automatic delivery even without a same-target message-tool send, continue that broader design in #57914 rather than keeping this narrower bug open.
