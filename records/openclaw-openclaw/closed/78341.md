## What This Changes

The issue reports that QQBot native approvals fail after upgrading to OpenClaw 2026.5.4 because the handler rejects a self-signed Gateway certificate on CentOS 8.

## Best Possible Solution

Keep https://github.com/openclaw/openclaw/issues/41740 as the canonical shared fix: carry the guarded local Gateway TLS fingerprint through operator approval bootstrap/client code and add regression coverage that includes QQBot-style native approvals.

## Reproduction Assessment

Yes, at source level. With local Gateway TLS enabled and no URL override, current main selects `wss://127.0.0.1:<port>`, the native approval path uses the shared operator approval client, and that client reaches `GatewayClient` without the `tlsFingerprint` required for pinned self-signed TLS handling.

## Solution Assessment
