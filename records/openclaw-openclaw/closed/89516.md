- Existing users who relied on managed CLI installation of standalone plugin files would now receive an unsupported-file error and need to move those files to `plugins.load.paths`; the PR argues that path was shipped-but-broken/internal, but it is still a compatibility decision.
- Hosts that enable `security.installPolicy` can fail closed for normal installs and updates when the policy command is missing, invalid, timed out, silent, malformed, too chatty, rejected by path hardening, or returns a block decision.
