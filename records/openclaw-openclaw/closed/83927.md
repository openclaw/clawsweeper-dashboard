## Summary

Keep open: current main and the latest stable tag still expose the raw `qa` descriptor through `SUB_CLI_DESCRIPTORS`, and the linked closing PR is open rather than merged.

## What This Changes

The issue reports that `SUB_CLI_DESCRIPTORS` exposes the private `qa` subcommand without the same `isPrivateQaCliEnabled()` filtering used by the public sub-CLI accessors.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/83946 so argv and descriptor consumers use one gated descriptor surface with regression coverage for private QA disabled and enabled.

## Reproduction Assessment

Yes. Source inspection shows that with `OPENCLAW_ENABLE_PRIVATE_QA_CLI` unset, `getSubCliEntries()` filters `qa` while `SUB_CLI_DESCRIPTORS` remains raw and is consumed by `src/cli/argv.ts`.

## Solution Assessment

Yes. The narrow maintainable fix is to remove or gate the raw export and update argv to use a lazy gated descriptor accessor; the linked PR follows that shape but is not merged yet.
