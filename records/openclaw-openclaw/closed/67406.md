## Summary

Keep open. Current main still has a plausible gap in the image-understanding execution path for configured/provider-prefixed model refs, and the repo’s docs/tests say Ollama vision models configured with `input: ["text", "image"]` should work. That is not strong enough evidence to conservatively close this issue.

## Evidence

- **Image runtime still hard-fails on exact registry miss:** `resolveImageRuntime` normalizes the provider/model ref, does a single `modelRegistry.find(...)`, and immediately throws `Unknown model` on a miss. There is no fallback lookup in this path.
  - file: [src/media-understanding/image.ts:143](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/src/media-understanding/image.ts#L143)
  - command: `nl -ba src/media-understanding/image.ts | sed -n '138,152p'`
  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **Fallback lookup exists elsewhere, but not in the media-understanding runtime:** `resolveModelFromRegistry` in shared tool code retries with a provider-prefixed model id when the first lookup misses. That helper is not used by `src/media-understanding/image.ts`, which is the execution path behind `describeImageWithModel`.
  - file: [src/agents/tools/media-tool-shared.ts:425](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/src/agents/tools/media-tool-shared.ts#L425)
  - command: `nl -ba src/agents/tools/media-tool-shared.ts | sed -n '425,440p'`
  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **Current tests only prove config selection, not successful Ollama execution for this case:** The image-tool tests cover selecting a config-declared image-capable provider model and avoiding double-prefixing during selection, but they do not cover actually executing an Ollama image request through the media-understanding runtime with these model refs.
  - file: [src/agents/tools/image-tool.test.ts:748](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/src/agents/tools/image-tool.test.ts#L748)

Provider-registry tests also assert config providers with image-capable models auto-register for media understanding.
