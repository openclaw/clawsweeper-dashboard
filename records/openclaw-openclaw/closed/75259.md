The issue reports that Telegram on the 2026.4.29 beta npm-global line repeatedly stages bundled plugin runtime dependencies but still cannot resolve staged packages such as `json5`, causing channel exits, repeated `pnpm install`, high CPU, and gateway RPC timeouts.

## Best Possible Solution

Validate a new beta candidate on the same npm-global Ubuntu/VPS path, then patch the bundled runtime-deps mirror/stage flow so root dist chunks resolve staged ESM packages from the runtime root, Telegram stays running, repeated installs stop, and gateway status/RPC probes return promptly.
