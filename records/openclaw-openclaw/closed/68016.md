## Summary

Current `main` already allows QQ Bot payload files from the shared OpenClaw media root, which covers `media/outbound` and `media/inbound`, and the same fix is present in release `v2026.4.23`.

## Evidence

- **QQ Bot payload allowlist now includes shared OpenClaw media root:** `resolveQQBotPayloadLocalFilePath()` trusts both `getOpenClawMediaDir()` and `getQQBotMediaPath()`, so files under `~/.openclaw/media/outbound` and sibling media dirs are accepted instead of being rejected as outside QQ Bot storage.
