## Summary

Current `main` already contains the queueing, system-event hardening, auth-context splitting, and chat-history sanitization needed to prevent the phantom-user/fallback behavior described in #56632.

## Evidence

- **Active runs no longer start a second collect/followup turn immediately:** `resolveActiveRunQueueAction` returns `enqueue-followup` when a run is active and the queue mode is collect/followup/steer-backlog, so queued input is deferred instead of dispatching another live run during active tool work.

  - command: `nl -ba src/auto-reply/reply/queue-policy.ts | sed -n '1,80p'`
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
