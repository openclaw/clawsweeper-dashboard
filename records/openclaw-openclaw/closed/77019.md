## Summary

Close: current main already surfaces the message for real `Error` throws; the `[object Object]` symptom only matches non-Error object throws, which this PR does not change.

## What This Changes

This PR changes `installCompletion` to print `err.message` for caught `Error` values and `String(err)` otherwise when completion installation fails.

## Best Possible Solution

Leave this closed unless a new source-backed reproduction shows an object-shaped completion install failure; any future fix should format message-bearing non-`Error` throws and include a changelog entry if user-facing.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No. Source inspection plus a Node runtime check shows real `Error` interpolation produces `Error: <message>` on current main; `[object Object]` only appears for plain objects, and no such install throw path was shown.

## Solution Assessment

Is this the best way to solve the issue? No. The diff changes the already-message-bearing `Error` path and leaves the object fallback that would produce `[object Object]`, so it is not the narrow fix for the stated symptom.
