## Summary

Current `main` and the shipped `v2026.5.18` tag already expose the Codex turn-completion idle timeout through plugin config and record `timeoutMs` in timeout trajectories, so the 60s value is now a configurable fallback rather than an unoverrideable constant.

## What This Changes

The issue requests a configurable Codex app-server turn-completion idle timeout, preferably per agent profile, instead of relying on the 60s default for reasoning-heavy Codex runs.

## Best Possible Solution

Keep the plugin-owned `plugins.entries.codex.config.appServer.turnCompletionIdleTimeoutMs` contract as the supported override path; only add agent-profile aliases or per-model defaults if maintainers choose a broader per-agent Codex policy later.

## Reproduction Assessment

Not applicable as a feature/config request. Source, docs, tests, and the shipped tag show the timeout is already configurable and the effective `timeoutMs` is recorded.

## Solution Assessment

Yes. The maintainable solution is the existing plugin-owned `plugins.entries.codex.config.appServer.turnCompletionIdleTimeoutMs` surface; adding a second `agents.<name>.codex` alias would create a duplicate source of truth unless separately designed.
