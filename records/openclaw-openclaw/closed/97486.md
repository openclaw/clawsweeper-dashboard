## Summary

Keep open. Current main still parses the DeepInfra video success response with raw `response.json()`, while this PR routes that path through the existing bounded provider JSON helper and I found no patch-introduced correctness defect.

## What This Changes

The PR changes DeepInfra video generation to parse successful JSON responses with `readProviderJsonResponse` instead of raw `response.json()`.

## Best Possible Solution

Land this narrow helper swap for the current DeepInfra video path, or carry the same bounded JSON read into any canonical DeepInfra video rewrite before merging that rewrite.

## Reproduction Assessment

Yes. Source inspection shows current main uses raw `response.json()` on the DeepInfra video success path, and the PR body includes terminal proof comparing bounded stream reads against the old unbounded path.

## Solution Assessment

Yes. Reusing `readProviderJsonResponse` is the narrowest maintainable fix because it is already the provider SDK seam for capped successful JSON responses and preserves the existing malformed-JSON label.
