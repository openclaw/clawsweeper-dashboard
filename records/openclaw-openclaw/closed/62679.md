## Summary

Current `main` already makes bundled speech providers available from TTS config without requiring manual `plugins.allow` or `plugins.entries.microsoft.enabled` wiring. The Microsoft speech plugin is bundled, marked `enabledByDefault`, and the speech-provider resolution path applies bundled allowlist/enablement compatibility before loading providers, which covers the reporter's failure mode.

## Evidence

- **Bundled capability-provider resolution auto-adds bundled plugin ids before loading providers:** `resolveCapabilityProviderConfig` derives bundled capability plugin ids from manifest contracts, then passes them through `withBundledPluginAllowlistCompat` and `withBundledPluginEnablementCompat` before loading the runtime registry. This is the path used for speech providers, so bundled speech plugins no longer need separate manual allowlist/entry config to resolve.
  - file: [src/plugins/capability-provider-runtime.ts:45](https://github.com/openclaw/openclaw/blob/3ffd944e6ba6dfe4c7c4c275848f1c0a1f246e28/src/plugins/capability-provider-runtime.ts#L45)
  - command: `nl -ba src/plugins/capability-provider-runtime.ts | sed -n '40,90p'`
  - sha: [3ffd944e6ba6](https://github.com/openclaw/openclaw/commit/3ffd944e6ba6dfe4c7c4c275848f1c0a1f246e28)
- **Bundled compat explicitly adds missing allowlist entries and enabled plugin entries:** `withBundledPluginAllowlistCompat` appends missing bundled plugin ids to `plugins.allow`, and `withBundledPluginEnablementCompat` injects `plugins.entries.<id>.enabled = true` when absent.
  - file: [src/plugins/bundled-compat.ts:5](https://github.com/openclaw/openclaw/blob/3ffd944e6ba6dfe4c7c4c275848f1c0a1f246e28/src/plugins/bundled-compat.ts#L5)

  - file: [extensions/microsoft/openclaw.plugin.json:1](https://github.com/openclaw/openclaw/blob/3ffd944e6ba6dfe4c7c4c275848f1c0a1f246e28/extensions/microsoft/openclaw.plugin.json#L1)
