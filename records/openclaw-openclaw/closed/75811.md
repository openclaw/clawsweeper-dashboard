## Summary

Keep this open: current main fixed the specific model-supplied `security` merge path, but the broader exec policy boundary remains exposed and partly consumed, and an open fix PR already references this issue with closing syntax.

## What This Changes

The issue asks OpenClaw to stop exposing and accepting exec policy controls such as `security`, `elevated`, `host`, `ask`, and `node` as model-call arguments.

## Best Possible Solution

Land or replace the linked fix PR so the model-facing exec schema contains only command/runtime inputs while host, node, ask, security, and elevated routing come from trusted config or session defaults.

## Reproduction Assessment

Yes, from source for the remaining boundary: current main still exposes and consumes several operator-control fields from exec tool-call arguments. The exact `security` self-denial path is partially fixed by ignoring model-supplied `security`.

## Solution Assessment

Yes, in direction. Hiding policy controls from the model schema and reading them only from configured defaults is the narrow maintainable fix, with the linked PR as the current implementation candidate.
