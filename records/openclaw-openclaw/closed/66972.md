## Summary

Current `main` already covers this stale ACP-thread recovery path, and the same implementation was already shipped in `v2026.4.23`: `/new` is retargeted onto the bound conversation session during session init, and stale ACP bindings are automatically unbound on `ACP_SESSION_INIT_FAILED` so the thread can recover.

## Evidence

- **Bound conversation retargeting in session init:** `initSessionState` resolves the current conversation binding before session identity is finalized, then swaps `ctx.SessionKey` to the bound target session key. That means native slash commands like `/new` act on the bound conversation session instead of staying trapped on the stale slash-session key.
