## What This Changes

The PR adds a Linux sudo guard for `openclaw gateway install`, post-install linger/runtime diagnostics, guard tests, and a `tsdown.config.ts` sourcemap setting.

## Best Possible Solution

Land a narrow current-main daemon install fix that blocks or safely handles sudo before systemd writes across gateway install/configure/onboarding, keeps linger guidance separate, updates regression tests, and leaves build config unchanged.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with `HOME=/root`, `SUDO_USER=alice`, and uid 0, current main resolves the unit path from `HOME` while systemctl targets the sudo caller's user scope; the PR also has source-visible test failures.

## Solution Assessment

No. Rejecting sudo at the install boundary is a maintainable direction, but this patch only covers one install entrypoint, has failing test/mocking changes, emits wrong custom-unit hints, and includes an unrelated build-artifact change.
