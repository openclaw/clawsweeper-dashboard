## Summary

Specialized SQLite/local-filesystem search, especially language-specific search behavior like Chinese tokenization and pinyin handling, fits OpenClaw's plugin and memory-plugin architecture rather than a new core feature request.

## Risks / Open Questions

- If maintainers want an officially bundled Chinese-language search backend instead of a third-party plugin or memory backend, closing this as plugin-scope would be premature.
- The linked `simple-search` repo is an external implementation candidate, not a verified published ClawHub package, so adoption still depends on external packaging or a separate plugin release.
