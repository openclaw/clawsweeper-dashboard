## Summary

Current main already covers this launchd recovery path: restart-required config reloads still trigger a gateway restart, but launchd-managed gateways now use a supervised exit/restart path instead of an in-process launchctl restart, and the CLI start/restart flows repair installed-but-not-loaded LaunchAgents. The same behavior is present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).

## Evidence

- **Restart-required config changes still route through the managed gateway restart flow:** Unknown changed paths become `restartGateway` reasons, and the reload handler emits a gateway restart when no work is pending. `commands.allowFrom` is not a hot-reload prefix, so it follows this path.
  - file: [src/gateway/config-reload-plan.ts:303](https://github.com/openclaw/openclaw/blob/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b/src/gateway/config-reload-plan.ts#L303)
- **Launchd restart is now handled as a supervised restart, not an in-process launchctl handoff:** `restartGatewayProcessWithFreshPid()` returns `supervised` when launchd markers are present, and the gateway run loop releases the lock, waits briefly for launchd, and exits cleanly with code 0.
  - file: [src/infra/process-respawn.ts:30](https://github.com/openclaw/openclaw/blob/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b/src/infra/process-respawn.ts#L30)
- **Run loop uses the supervised restart path for restart-required reloads:** After server close, the gateway treats `supervised` as a full process restart and pauses briefly on launchd to avoid relaunch throttling.
  - file: [src/cli/gateway-cli/run-loop.ts:82](https://github.com/openclaw/openclaw/blob/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b/src/cli/gateway-cli/run-loop.ts#L82)
- **CLI start/restart repairs an installed-but-not-loaded LaunchAgent:** `runDaemonStart()` and `runDaemonRestart()` call `recoverInstalledLaunchAgent()`, which re-bootstraps launchd when the plist still exists but the service is not loaded.
  - file: [src/cli/daemon-cli/lifecycle.ts:152](https://github.com/openclaw/openclaw/blob/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b/src/cli/daemon-cli/lifecycle.ts#L152)
- **LaunchAgent restart/start path now bootstraps and retries when launchd reports the service is not loaded:** `restartLaunchAgent()` enables the service, retries via plist bootstrap when needed, and `recoverInstalledLaunchAgent()` uses `repairLaunchAgentBootstrap()` for the installed-but-unloaded case.
  - file: [src/daemon/launchd.ts:656](https://github.com/openclaw/openclaw/blob/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b/src/daemon/launchd.ts#L656)
- **Focused tests cover both sides of the fix:** There are tests asserting launchd uses the supervised/no-kickstart restart path and that installed-but-not-loaded LaunchAgents are repaired successfully.
  - file: [src/infra/process-respawn.test.ts:88](https://github.com/openclaw/openclaw/blob/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b/src/infra/process-respawn.test.ts#L88)
- **Release proof:** The `v2026.4.22` tag already contains the same `src/infra/process-respawn.ts` and `src/cli/daemon-cli/launchd-recovery.ts` implementations present on current main.
  - command: `git show v2026.4.22:src/infra/process-respawn.ts ; git show v2026.4.22:src/cli/daemon-cli/launchd-recovery.ts`
