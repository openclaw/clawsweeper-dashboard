## Summary

Keep open: current main and the latest release still have a TTL-only `agentRunCache`, and the existing discussion already links an open same-author implementation PR that should be reviewed or replaced before closing this issue.

## What This Changes

The issue asks for a hard bound on the gateway `agentRunCache` so sustained `agent.run` lifecycle fan-out cannot retain every terminal snapshot from the last 10 minutes.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/77973, or replace it with an equivalent narrow gateway patch that caps `agentRunCache` after inserts while preserving TTL pruning and `agent.wait` semantics.

## Reproduction Assessment

Yes. Source inspection shows many unique lifecycle `end` events can reach `recordAgentRunSnapshot`, where each fresh run ID is inserted into a TTL-only `Map` with no size cap.

## Solution Assessment

Yes. A hard max-entry cap enforced after insertion is the narrow maintainable fix because it composes with existing TTL pruning without changing lifecycle wait semantics.
