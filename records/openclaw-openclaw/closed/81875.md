## Summary

Keep open: the installer fix direction and supplied proof are useful, but the current head still checks remote branches before immutable tag checkout with pattern-matching `ls-remote`, so pinned release refs can resolve as moving branches and use the non-frozen install path.

## What This Changes

Updates the Bash installers, focused installer tests, and changelog to handle noninteractive git installs from moving refs without tag-fetch conflicts.

## Best Possible Solution

Land the installer fix after preserving immutable tag/semver/commit precedence and using exact remote-branch detection in both published installers.

## Reproduction Assessment

Yes. The PR-head source plus live remote-ref checks show the branch-first path can catch a release ref before tag fallback; I did not need a full installer run to prove that regression path.

## Solution Assessment

No, not in the current form. The main/noninteractive direction is right, but the maintainable fix must keep immutable refs tag-first or make branch detection exact before applying no-tags and non-frozen behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Preserve immutable refs before branch handling:** `scripts/install.sh:1972`

  - confidence: 0.94
