## Summary

Keep open: current main still does not reclaim the reported same-process lock payload when it has a valid `starttime`, and PR #75822 is already an open closing implementation candidate.

## What This Changes

The issue reports a gateway session `.jsonl.lock` left after a hung task/restart, causing high CPU lock polling until the lock file is manually deleted.

## Best Possible Solution

Review and land PR #75822 or an equivalent narrow session-lock fix that reclaims untracked self-owned valid-starttime locks while preserving active `HELD_LOCKS` ownership.
