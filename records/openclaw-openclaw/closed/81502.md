This PR should close as superseded by the earlier focused fix in https://github.com/openclaw/openclaw/pull/81446, which targets the same linked bug, preserves the lstat-before-readlink direction, handles generated Windows directory entries, and includes real behavior proof. The branch changes plugin skill publishing to inspect existing entries with `lstatSync` before `readlinkSync`, skip non-symlinks, and add a regular-directory collision test.

## Best Possible Solution

Close this duplicate branch and review or land https://github.com/openclaw/openclaw/pull/81446 as the canonical fix for the linked bug.

## Solution Assessment
