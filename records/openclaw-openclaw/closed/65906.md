## Summary

Keep open. Current main still lacks these response-prefix variables, and this PR is useful but not merge-ready because it can report inflated context usage, does not propagate the usage callback through all reply paths, and misses the public docs/changelog updates required for a user-facing feature.

## What This Changes

This PR adds `{context}` and `{contextPercent}` response-prefix template variables, an `onContextUsage` callback with partial runtime wiring, and focused interpolation/runtime tests.

## Best Possible Solution

Revise the PR so the displayed context is derived from prompt-side context counters, the usage callback is propagated anywhere `onModelSelected` is threaded, and docs/schema help/changelog describe the exact semantics before merge.

## Reproduction Assessment

Yes. Current main can be reproduced by configuring `messages.responsePrefix` with `{context}` or `{contextPercent}` and observing that the resolver leaves them literal; the PR bug is reproducible with a mocked `lastCallUsage.total` that includes output tokens.

## Solution Assessment

No. The feature direction and callback pattern are appropriate, but the implementation should reuse the existing prompt-context helpers and complete the reply-option propagation boundary instead of relying on output-inclusive totals and partial wiring.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Compute context from prompt tokens only:** `src/auto-reply/reply/agent-runner-execution.ts:1250-1254`
  - body: This branch prioritizes `lastCallUsage.total`, which can include completion/output tokens, so long answers can overstate context pressure and clamp `{contextPercent}` to 100% while prompt usage is still below the context window.   - confidence: 0.88

  - body: This adds user-facing template variables, but the configuration docs/schema help and required changelog entry still show only the old variable list. Users would not discover the new placeholders or their previous-turn context semantics.
  - confidence: 0.82
