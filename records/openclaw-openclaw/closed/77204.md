## Summary

Close as implemented: current main and v2026.5.3 defer automatic PDF model/auth resolution out of the tool-prep path, with regression coverage showing `createOpenClawTools` no longer calls PDF model resolution during registration.

## What This Changes

The issue reports that pdf-tool initialization blocks the Node.js event loop for about 10 seconds on every agent run.

## Best Possible Solution

Keep the shipped deferred PDF registration behavior and have affected 2026.5.2 users update to v2026.5.3 or newer; `tools.deny: ["pdf"]` remains only a workaround for older installs.

## Reproduction Assessment

No for current main: I did not run the Windows global-install live scenario, and current source/tests show tool prep no longer resolves PDF models or auth before use. The original 2026.5.2 failure mode is covered by the now-fixed eager-resolution path referenced in the regression tests and changelog.

## Solution Assessment

Yes. Deferring automatic PDF model/auth resolution during tool registration is the narrowest maintainable fix because explicit PDF model config still registers eagerly, while automatic resolution happens only when `pdf` is actually executed.
