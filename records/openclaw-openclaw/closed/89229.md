## What This Changes

The PR hardens Anthropic provider tool payload construction and OAuth Claude Code tool-name remapping against unreadable or provider-incompatible tool descriptors, with focused provider tests.

## Best Possible Solution

Land the focused provider guard after the draft is ready and either add a small redacted live Anthropic smoke or explicitly accept mocked API coverage for this narrowly defensive payload path.

## Reproduction Assessment

Yes from source inspection, but I did not run a current-main failing test. Current main directly dereferences Anthropic tool descriptor fields in convertTools and fromClaudeCodeName, so a descriptor getter that throws would fail on that path.

## Solution Assessment

Yes, this is an acceptable scoped fix because the raw LLM provider must defend direct legacy Context callers that bypass agent-runtime quarantine. A lower-level shared projection helper would reduce duplicate logic, but importing the existing agents-layer helper into the provider would be the wrong boundary.
