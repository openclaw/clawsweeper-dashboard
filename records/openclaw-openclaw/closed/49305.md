## Summary

Current main fixes the reported WhatsApp logout dead-end: terminal 401/440 closes now clear cached Web auth and active listener state, status reflects logged-out/not-linked rather than stale connected, and CLI login now asks the running gateway to start the channel runtime after relink. The fix is present on main but not in the latest release tag v2026.4.26.

## Best Possible Solution

Close as implemented on current main. Keep the current WhatsApp lifecycle behavior: clear stale auth/listener state on terminal 401/440, surface accurate status, and restart the live gateway runtime through channels.start after CLI relink. Users on v2026.4.26 need a later main build or the next release for this fix.
