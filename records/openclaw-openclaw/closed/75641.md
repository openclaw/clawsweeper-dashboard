## Summary

Keep open. Current `main` still projects Codex dynamic tool lifecycle as `item`/`codex_app_server.*` events or trajectory records, while Control UI/WebChat live tool cards only consume the standard `stream: "tool"` event shape with a `toolCallId`.

## What This Changes

Control UI / WebChat should show live tool cards and progress for Codex harness dynamic tool calls.

## Best Possible Solution

Project Codex-owned dynamic tool start/update/result/failure into the existing runtime-neutral `stream: "tool"` agent-event contract while preserving the current item/debug events.

## Reproduction Assessment

Yes, at source level. Route a Codex `item/tool/call` or `dynamicToolCall` through the current bridge/projector and the emitted UI-visible lifecycle is `item`/`codex_app_server.*`, while WebChat live cards accept only `stream: "tool"` with `data.toolCallId`.

## Solution Assessment

Yes. The narrow maintainable fix is Codex-side projection into the existing Gateway/Control UI tool-event contract, not a Codex-only renderer or a new config flag.
