## Summary

The PR targets a real WhatsApp reconnect problem, but current main already moved transport liveness into `WhatsAppConnectionController` and listens to raw Baileys frame events; this stale branch still tracks only higher-level events, lacks required real-behavior proof, and should be retargeted before merge.
