## Summary

The observed `replies=0` state matches documented group visible-reply policy: Feishu hands the turn to the shared dispatcher, but group/channel finals default to message-tool-only delivery unless configured for automatic visible replies.

## What This Changes

Request that Feishu group chats send normal generated replies back to the group like DMs do.

## Best Possible Solution

Leave the source behavior unchanged and direct users who want visible group finals to set `messages.groupChat.visibleReplies: "automatic"` or have the model send via `message(action=send)`.

## Reproduction Assessment

Yes, source inspection and existing tests reproduce the observed state: a group/channel turn with no automatic visible-reply override resolves to `message_tool_only`, suppresses final delivery, and returns `queuedFinal=false`. The reported logs align with that path rather than a Feishu-specific send failure.

## Solution Assessment

Yes, current behavior is the intended cross-channel group policy; changing Feishu routing would bypass that safety model. The narrow solution is the documented config/model/tool path, not a source patch.
