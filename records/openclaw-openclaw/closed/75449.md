## Summary

Close as implemented on current main. The main branch now keeps group/channel visible replies deliverable in tool-only mode, prompts the model to use the message tool for visible output, and preserves Telegram topic thread IDs when sending back to the current chat; the fix is not in the latest v2026.4.29 release yet.

## What This Changes

The issue asks for Telegram forum topic turns to either auto-deliver generated replies to the source topic or force visible topic replies through the message tool.

## Best Possible Solution

Keep the current main behavior: group and topic replies stay tool-mediated by default, while the message tool remains available and topic-aware; users who want automatic group posting can opt into messages.groupChat.visibleReplies="automatic".

## Reproduction Assessment

Yes. The report maps to a Telegram forum-topic inbound turn with ChatType=group and MessageThreadId; current source and tests show how automatic delivery is suppressed in tool-only mode and how visible delivery now remains available through the message tool.

## Solution Assessment

Yes. The current fix is narrower than Telegram-only auto-posting because it preserves the repository's group-room privacy policy while making visible output deliverable and thread-preserving for Telegram topics.
