## Summary

Current `main` already fixes the `/new` and `/reset` wrong-greeting/date problem through the bare reset prompt path. The shipped implementation appends a live `Current time:` line for bare reset/new turns, and the docs intentionally keep the main system prompt timezone-only for prompt-cache stability, so this PR's system-prompt approach is superseded.
