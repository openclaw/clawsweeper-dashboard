## Summary

Current main, shipped in v2026.4.24, already fixes the observable failure behind this PR: empty post-compaction preparations now write a compaction boundary instead of cancelling with “no real conversation messages,” repeated empty preparations are covered by tests, and the runner reports the operation as a completed compaction with `tokensAfter`. The PR's exact token-threshold override is not present, but the shipped implementation solves the central user problem without entering the model/auth-dependent path that the PR review flagged as a possible cancel-loop regression.
