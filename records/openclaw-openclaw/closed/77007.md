## Summary

Keep open. Current main still has the generic non-verbose external-chat fallback for these CLI timeout errors, and an open closing PR already targets the narrow fix, so this issue should remain open until that PR merges or is replaced.

## What This Changes

Telegram should surface Claude CLI no-output and overall timeout failures as actionable CLI timeout copy instead of the generic external-run failure message.

## Best Possible Solution

Review and land PR #77015 or an equivalent narrow fix, then track broader diagnostics/default-timeout policy separately if maintainers still want that behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence unit path: reject an external-chat run with a CLI timeout message while verbose detail is off and current main falls through to the generic failure copy.

## Solution Assessment

Yes for the narrow fix. Matching OpenClaw-generated CLI timeout templates before the generic fallback is the smallest maintainable repair; changing default timeouts or broad status diagnostics should be handled separately.
