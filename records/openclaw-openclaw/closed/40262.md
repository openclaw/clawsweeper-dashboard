## Summary

Current `main` already handles Discord forum/media thread creation correctly for `message action=thread-create`: it forwards the optional `message` field, detects forum-style parent channels, sends the required nested `message: { content }` payload, and includes `applied_tags` for forum threads. The behavior is covered by targeted tests and documented in the Discord channel docs.

## Evidence

- **Forum/media thread payload fixed in implementation:** `createThreadDiscord` now looks up the parent channel type for standalone thread creation, treats `GuildForum` and `GuildMedia` as forum-like, sends `body.message = { content: ... }`, and forwards `applied_tags` when present.
  - file: [extensions/discord/src/send.messages.ts:98](https://github.com/openclaw/openclaw/blob/51186d2725439646f7fea0e59bb466f957404c46/extensions/discord/src/send.messages.ts#L98)
