## Summary

Current main already addresses the reported npm-plugin disappearance path: npm installs now maintain a managed root package.json dependency set before running npm install, and regression coverage verifies that a second npm plugin install keeps the first sibling package present.

## What This Changes

The issue reports that sequential optional npm plugin installs on 2026.5.2 can leave only the latest package under ~/.openclaw/npm/node_modules while stale install records remain.

## Best Possible Solution

Keep the managed npm-root package.json preservation and ledger recovery now on main, then ship it in the next release so affected 2026.5.2 users can upgrade and reinstall any already-missing packages once.

## Reproduction Assessment

No current-main failing reproduction remains. The historical 2026.5.2 path is represented by regression coverage that performs two npm installs into the same managed root and verifies both sibling plugin packages remain present.

## Solution Assessment

Yes. The current main fix is the narrow maintainable solution because install/update preserves the managed npm root dependency manifest and avoids adding package-manager repair work to Gateway startup.
