## Summary

Current `main` has a shared CLI/Gateway teardown and timeout fix that covers `openclaw cron rm`, so the reported indefinite hang is already fixed on `main`.

## Evidence

- **`cron rm` uses the shared gateway RPC path:** The cron remove CLI subcommand is a thin wrapper over `callGatewayFromCli("cron.remove", opts, { id })`, so it inherits the generic one-shot CLI/Gateway behavior.
  - file: [src/cli/cron-cli/register.cron-simple.ts:83](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/cli/cron-cli/register.cron-simple.ts#L83)

  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **CLI gateway calls now run with an explicit timeout:** `callGatewayFromCliRuntime` sends CLI RPCs through `callGateway(...)` with `timeoutMs: Number(opts.timeout ?? 10_000)`, so stalled requests no longer wait forever without surfacing an error.
  - file: [src/cli/gateway-rpc.runtime.ts:20](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/cli/gateway-rpc.runtime.ts#L20)

  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **Shared gateway request wrapper always tears down the client:** `stopGatewayClient()` awaits `client.stopAndWait(...)`, and `executeGatewayRequestWithScopes()` clears the wrapper timer and always runs that teardown before resolving or rejecting the CLI RPC.
  - file: [src/gateway/call.ts:95](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/gateway/call.ts#L95)

  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **Gateway client shutdown no longer leaves lingering sockets alive:** `GatewayClient.stopAndWait()` waits for the socket to close and force-terminates lingering sockets after a short grace window, which directly addresses the class of "command finished work but process never exited" hangs.
  - file: [src/gateway/client.ts:352](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/src/gateway/client.ts#L352)
