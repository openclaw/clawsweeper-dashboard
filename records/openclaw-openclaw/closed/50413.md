## Summary

Current main already implements restart recovery for recurring cron jobs, persists runtime state separately from job definitions, records finished runs, and reconciles orphaned cron tasks so the false-positive/silent-drop behavior described for recurring jobs is no longer the current design.

## Evidence

- **Startup clears stale running markers and replays recurring jobs:** `start()` clears persisted `runningAtMs` markers on startup, skips only interrupted one-shot jobs, and immediately runs missed recurring jobs after restart.
  - file: [src/cron/service/ops.ts:93](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/cron/service/ops.ts#L93)

  - file: [src/cron/service.restart-catchup.test.ts:118](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/cron/service.restart-catchup.test.ts#L118)
