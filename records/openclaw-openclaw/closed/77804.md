## What This Changes

`openclaw doctor` should return a non-zero exit code when its final config validation prints `Invalid config:` diagnostics.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/81829 so final invalid-config diagnostics set a non-zero status and the doctor wrapper honors it, while keeping broader `doctor --fix` repair atomicity work separate.

Yes. Source inspection on current `main` shows invalid config is allowlisted for `doctor`, final validation prints `Invalid config:` without setting failure state, and the wrapper exits 0 after a resolved doctor run.
