## Summary

Close as duplicate/superseded. Current main still has the reported native `sessions_spawn` cold-registry failure path, but the same remaining work is already tracked by open canonical issue #73095 and open focused PR #73904, which explicitly targets initialization before subagent spawn preparation resolves the legacy context engine.

## What This Changes

The issue reports that native `sessions_spawn` fails on cold non-PI/cliBackend-style runtime paths because the built-in legacy context engine is not registered before spawn preparation resolves the context engine.

## Best Possible Solution

Keep #73095 as the canonical open bug and review/land #73904, or an equivalent narrow patch, so native subagent spawn preparation initializes built-in context engines before resolving the configured engine.

## Reproduction Assessment

Yes. At source level, `sessions_spawn` reaches `spawnSubagentDirect()`, then `prepareContextEngineSubagentSpawn()`, which resolves the default `legacy` engine before the built-in initializer is called; the registry has the exact missing-engine throw path reported here.

## Solution Assessment

Yes for cleanup. Closing this duplicate while #73095 and #73904 remain open keeps one canonical tracker and one focused implementation path; the code solution should be the spawn-prep initializer, not a broader registry side effect.
