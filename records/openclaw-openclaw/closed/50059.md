## Summary

Current `main` already handles this case: both the live session guard and transcript repair logic skip synthesizing `toolResult` entries for assistant turns with `stopReason: "error"` or `"aborted"`, which is the corruption path described in #50059. The released tag `v2026.4.22` also contains the same logic and explicit regression tests for errored/aborted tool-call turns.

## Evidence

- **Session guard skips errored/aborted assistant tool calls:** `installSessionToolResultGuard()` gates `extractToolCallsFromAssistant()` behind `stopReason !== "aborted" && stopReason !== "error"`, with an inline note matching the reported Anthropic `unexpected tool_use_id found in tool_result blocks` failure.
