## Summary

Keep this PR open: the reported Claude ACP crash is a real source-reproducible bug and the PR is aimed at the right ACPX boundary, but the latest head still has two blocking correctness issues in effort normalization.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve xhigh instead of mapping it to max:** `extensions/acpx/src/runtime.ts:436`
  - body: Claude's SDK and ACP adapter treat `xhigh` and `max` as distinct effort values, and the adapter validates against the selected model's advertised values. Mapping xhigh aliases to `max` can reject sessions on models that advertise `xhigh` but not `max`, or silently request stronger effort than the user selected.
  - confidence: 0.9
- **[P2] Normalize advertised effort values for Claude ACP:** `extensions/acpx/src/runtime.ts:1008`
  - body: The control plane can resolve stored `thinking` to the adapter-advertised `effort` key, and explicit `/config effort ...` reaches this branch as `key === "effort"`. Because this condition skips `effort`, values like `x-high` still bypass normalization and can be rejected by the adapter.
  - confidence: 0.94
