## Summary

Keep open: the reported TUI visibility symptom is source-reproducible, but this patch changes a shared final-answer-safe helper used by Gateway previews, transcript helpers, and Control UI, and it still lacks after-fix real TUI proof.

## What This Changes

The PR adds an unconditional commentary-phase fallback to `extractAssistantVisibleText` when no `final_answer` text is present.

## Best Possible Solution

Preserve shared final-answer-only extraction and implement any no-final fallback only in TUI rendering or behind a caller-scoped option, with regression coverage and real TUI proof.

## Reproduction Assessment

Yes from source inspection: a commentary-only phased assistant message returns no visible shared text, and the TUI stream assembler reaches that helper through `extractContentFromMessage`. I did not live-run a TUI session in this read-only review.

## Solution Assessment

No. The patch targets the right symptom, but changing the shared helper is too broad; the narrower maintainable fix is a TUI-scoped fallback or caller-scoped option that preserves non-TUI commentary suppression.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
