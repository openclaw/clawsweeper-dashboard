## Summary

Keep open: this PR targets a real memory-lancedb compatibility bug, but it still lacks real provider proof and unconditionally replaces provider-side dimensioning with local truncation, which is an upgrade-sensitive behavior change for existing dimension-aware configs and vector stores.

## What This Changes

The branch changes memory-lancedb's direct OpenAI-compatible embedding path to omit `dimensions`, locally truncate returned vectors, validate positive integer dimensions, and update related docs, manifest metadata, and tests.

## Best Possible Solution

Preserve provider-side dimensions when accepted and fall back to local truncation only for rejecting compatible endpoints, with live rejecting-provider proof plus upgrade coverage for existing dimension-aware configs.

## Reproduction Assessment

No high-confidence live reproduction is present yet. Source inspection proves current main sends `dimensions` upstream and tests assert that behavior, but the rejecting-provider failure still needs redacted real provider proof.

## Solution Assessment

No, not as written. Omitting `dimensions` everywhere addresses rejecting endpoints but is less upgrade-safe than preserving provider-side dimensions where accepted and falling back only for incompatible endpoints.
