## Summary

Current main already includes cron-first scheduling guidance in the system prompt and tool descriptions, plus a reminder guard when a reply promises a reminder without a successful `cron.add`. The same implementation is present in the shipped `v2026.4.22` release.

## Evidence

- **System prompt steers reminder requests to cron:** The core tool summary injected into the system prompt describes `cron` as the tool to use for reminders and reminder wording.
  - file: [src/agents/system-prompt.ts:494](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/agents/system-prompt.ts#L494)
  - command: `nl -ba src/agents/system-prompt.ts | sed -n '490,500p'`
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Cron tool description reinforces deferred follow-up guidance:** The cron tool explicitly says to use it for reminders, delayed follow-ups, and recurring tasks, and says not to emulate scheduling with exec sleep or process polling.
  - file: [src/agents/tools/cron-tool.ts:421](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/agents/tools/cron-tool.ts#L421)
  - command: `nl -ba src/agents/tools/cron-tool.ts | sed -n '418,426p'`
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Exec/process descriptions redirect scheduled work to cron:** When cron is available, exec/process descriptions tell the model not to use exec sleep loops or process polling for reminders and to use cron instead.
  - file: [src/agents/bash-tools.descriptions.ts:16](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/agents/bash-tools.descriptions.ts#L16)

- **Tests lock the cron guidance into the tool descriptions:** Unit tests assert the cron tool description contains the reminder/deferred follow-up guidance and that exec/process descriptions tell the model to use cron for scheduled follow-ups.
  - file: [src/agents/tools/cron-tool.test.ts:124](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/tools/cron-tool.test.ts#L124)

## Risks / Open Questions

- The guidance does not mention `crontab` by name; it is prompt steering plus a user-visible fallback note, not a hard exec-layer block on `crontab`.
