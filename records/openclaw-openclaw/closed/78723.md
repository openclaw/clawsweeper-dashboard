## Summary

Close as implemented on current main: managed outgoing image downloads now require authenticated owner/admin context before serving bytes, and regression coverage rejects the reported device-token self-declared ownership path. The fix is present on current main but was not found in the latest release tag.

## What This Changes

The issue reports that device-token-authenticated callers can download managed outgoing media from sessions they do not own.

## Best Possible Solution

Keep the merged owner/admin media gate from https://github.com/openclaw/openclaw/pull/79204 and track any future media authorization regressions as new, narrower reports.

## Reproduction Assessment

No, not against current main. The historical source-level path was valid, but current main now returns 403 for device-token self-declared session ownership and has regression coverage for that behavior.

## Solution Assessment

Yes. The landed fix is narrower and safer than the old requester-session-header approach because it stops trusting client-declared ownership and uses authenticated owner/admin request context while preserving the existing scope and transcript checks.
