## Summary

Issue #50074 is already addressed on current `main`: Windows stale-listener detection no longer returns `[]`, restart health only treats unknown listeners as stale on Windows when the runtime is not running, and the active runtime PID is excluded from the stale-PID fallback. The changelog records this Windows `EADDRINUSE` restart-loop fix in release `2026.4.5`.
