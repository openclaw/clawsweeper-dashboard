## Summary

Keep open: current main still lets an undefined plugin command handler result flow into unguarded Telegram and generic command reply paths, and the only direct closing PR, #74989, is still open/unmerged in the provided GitHub context.

## What This Changes

The issue asks OpenClaw to handle undefined or empty plugin command handler results without crashing Telegram command dispatch.

## Best Possible Solution

Land or supersede #74989 with a narrow fix that normalizes nullish/non-object command returns at the shared executor and keeps defensive caller coverage.

## Reproduction Assessment

Yes, source-reproducible: a plugin command handler that returns undefined flows through `executePluginCommand` unchanged and is then dereferenced by the generic command path; the Telegram native path has the same unguarded result use. I did not run a live Telegram command in this read-only review.

## Solution Assessment

Yes. Normalizing at the shared executor boundary with defensive guards in the Telegram and generic command callers is the narrow maintainable fix, and #74989 already targets that path.
