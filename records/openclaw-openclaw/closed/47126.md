## Summary

Current main already covers the two restart-related failure modes described in #47126: Telegram polling/webhook startup always requests `message_reaction` updates again, and `reactionNotifications: "own"` now uses a persisted sent-message cache that survives restart. The changelog shows the own-message reaction filtering fix shipped in v2026.4.22.

## Evidence
