## What This Changes

The issue requests changing cron session retention cleanup so expired base cron session keys are swept in addition to per-run cron session keys.

## Best Possible Solution

Land a clean version of https://github.com/openclaw/openclaw/pull/76545 or a replacement that prunes expired base and run cron session rows, updates retention docs/help, and tracks boot-session orphan cleanup separately if confirmed.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: seed an expired `agent:main:cron:<jobId>` entry and call `sweepCronRunSessions`; current main rejects that base key through `isCronRunSessionKey` before pruning.

## Solution Assessment

Yes for the central cron leak. Switching the sweep predicate to the broader cron-session classifier with regression coverage is the narrow maintainable fix, with docs/help alignment required for the changed retention contract.
