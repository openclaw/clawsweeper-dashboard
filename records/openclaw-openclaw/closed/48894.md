## Summary

This Discord MESSAGE_UPDATE request is real and still missing on current main, but the remaining work is already tracked by the older open #20173 issue. The useful added-mention/requireMention context from this issue should be carried into that canonical implementation discussion.

## Best Possible Solution

Close this as a duplicate of #20173 and keep #20173 as the canonical implementation tracker. The eventual Discord plugin fix should register a Carbon MessageUpdateListener, route eligible edit payloads through the existing preflight/inbound worker path, make replay dedupe edit-aware, preserve bot/self/access/mention filters, and add regression tests for the requireMention + newly added @mention case described here.
