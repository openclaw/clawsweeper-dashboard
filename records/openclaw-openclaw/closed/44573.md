## Summary

Current `main` still shows the exact mismatch this PR fixes: isolated cron runs compute and return a per-run `...:run:<sessionId>` key, but the actual CLI/embedded execution path and descendant accounting still use the stable job-scoped session key. The PR remains materially relevant and should stay open for maintainer review.
