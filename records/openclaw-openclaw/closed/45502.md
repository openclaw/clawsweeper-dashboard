## Summary

Current `main` does not support the issue as reported. The `evaluate` path is an intentional browser-control feature for trusted operators, the config opener uses `execFile` with literal argv plus regression tests for shell metacharacters, SSRF protection already blocks the cited hostname/IP classes, the trusted-proxy code fails closed, and there are no `skills/*/web/*.html` files matching the XSS claim.

## Evidence

- **`evaluate` is an intentional operator feature, not an accidental code-exec sink:** The browser action contract explicitly accepts `{ kind: "evaluate", fn: string }`, the route can reject it via `browser.evaluateEnabled`, and the docs warn that it executes arbitrary JavaScript in page context.

  - file: [docs/tools/browser-control.md:317](https://github.com/openclaw/openclaw/blob/028b6c9b13c960b6f59a7643aff4f6de04e07f71/docs/tools/browser-control.md#L317)
