## Summary

Keep open. Current main still skips unchanged daily memory files using only mtime/size, so later dreaming days cannot add another daily signal; open closing PR #76359 is the active implementation candidate and should be reviewed/landed before this issue closes.

## What This Changes

The issue reports that memory-core dreaming deep promotion remains stuck at Promoted 0 because recall/daily signals do not accumulate across daily sweeps.

## Best Possible Solution

Review and land #76359, or an equivalent memory-core fix that makes daily-ingestion checkpoints calendar-day aware while preserving same-day dedupe and adding persisted-checkpoint regression coverage.

## Reproduction Assessment

Yes, from source inspection. Ingest a daily note once, keep memory/.dreams/daily-ingestion.json, then run dreaming on a later calendar day without changing the note; current main skips the file and does not add another daily signal.

## Solution Assessment

Yes for the linked PR direction. Making the daily-ingestion checkpoint day-aware is narrower than lowering thresholds or changing promotion policy, and it targets the current defect rather than the older normalization-reset hypothesis.
