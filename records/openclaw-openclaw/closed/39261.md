## Summary

Current `main` already works around Mattermost’s server-side `/...` interception with opt-in native Mattermost slash commands (`/oc_*`), callback routing, docs, and regression coverage. The exact Matrix-style raw `/model` text passthrough is still not how Mattermost works, but the requested capability is implemented well enough on `main` via native command registration and documented fallback behavior.

## Evidence

- **Native Mattermost slash commands exist:** `extensions/mattermost/src/mattermost/slash-commands.ts:131` defines built-in Mattermost-native commands including `oc_status`, `oc_model`, `oc_models`, `oc_new`, `oc_help`, `oc_think`, `oc_reasoning`, and `oc_verbose`, with `originalName` mapping back to the normal OpenClaw command names.
  - file: [extensions/mattermost/src/mattermost/slash-commands.ts:131](https://github.com/openclaw/openclaw/blob/4d7c4b3298920c0e6e2c5cc8bd7a1546e8122abe/extensions/mattermost/src/mattermost/slash-commands.ts#L131)
- **Monitor registers commands after connect:** `extensions/mattermost/src/mattermost/monitor.ts:487` calls `registerMattermostMonitorSlashCommands(...)` on startup, and `extensions/mattermost/src/mattermost/monitor-slash.ts:137`-`209` resolves config, computes callback URLs, registers commands across teams, and activates token-backed slash handling.
  - file: [extensions/mattermost/src/mattermost/monitor.ts:487](https://github.com/openclaw/openclaw/blob/4d7c4b3298920c0e6e2c5cc8bd7a1546e8122abe/extensions/mattermost/src/mattermost/monitor.ts#L487)
- **Gateway callback route and handler are implemented:** `extensions/mattermost/index.ts:32` wires the route during plugin registration; `extensions/mattermost/src/mattermost/slash-state.ts:153`-`310` registers HTTP callback paths and dispatches by command token; `extensions/mattermost/src/mattermost/slash-http.ts:226`-`345` validates the callback token and reconstructs commands like `/status` from triggers such as `/oc_status`.
  - file: [extensions/mattermost/src/mattermost/slash-state.ts:153](https://github.com/openclaw/openclaw/blob/4d7c4b3298920c0e6e2c5cc8bd7a1546e8122abe/extensions/mattermost/src/mattermost/slash-state.ts#L153)
- **Docs now describe the Mattermost-native flow:** `docs/channels/mattermost.md:59`-`95` documents opt-in native slash commands, `commands.native: true`, callback path/URL requirements, and `oc_*` registration; `docs/channels/mattermost.md:114`-`131` also documents `chatmode: "onchar"` with `oncharPrefixes` as fallback behavior.
  - file: [docs/channels/mattermost.md:59](https://github.com/openclaw/openclaw/blob/4d7c4b3298920c0e6e2c5cc8bd7a1546e8122abe/docs/channels/mattermost.md#L59)
- **Shipped in changelog and covered by tests:** `CHANGELOG.md:4504`-`4512` lists Mattermost native slash command support in release `2026.2.19`; tests cover command mapping and registration in `extensions/mattermost/src/mattermost/slash-commands.test.ts:80`-`109`, startup registration in `extensions/mattermost/src/mattermost/monitor-slash.test.ts:82`-`145`, and callback validation in `extensions/mattermost/src/mattermost/slash-http.test.ts:81`-`150`.
  - file: [CHANGELOG.md:4504](https://github.com/openclaw/openclaw/blob/4d7c4b3298920c0e6e2c5cc8bd7a1546e8122abe/CHANGELOG.md#L4504)

## Risks / Open Questions

- This does not make raw text messages like `/model` travel through the Mattermost WebSocket path the way Matrix does; the implemented OpenClaw solution is Mattermost-native `/oc_*` registration plus fallback chat modes.
- Operators still need to enable `channels.mattermost.commands.native: true` and configure callback reachability correctly, so users expecting zero-config Matrix parity may still be surprised.
