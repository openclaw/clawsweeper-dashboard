## Summary

Current main, and the shipped v2026.4.23 release, no longer match the reported non-TTY `openclaw sessions --json` hang. `sessions` now runs through the routed read-only fast path, and the checked-in startup benchmark exercises `sessions --json` with non-TTY stdout and records 5/5 successful exits.

## Evidence

- **`sessions` is a routed fast-path command on current main:** The CLI command catalog marks `sessions` as an exact routed command with `ensureCliPath: false`, and the routed-command definition dispatches directly to `sessionsCommand`.
  - file: [src/cli/command-catalog.ts:62](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/cli/command-catalog.ts#L62)
  - command: `nl -ba src/cli/command-catalog.ts | sed -n '58,68p'`
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Routed commands return before the full console-capture startup path:** `runCli` returns immediately when `tryRouteCli(...)` handles the invocation, before `enableConsoleCapture()` is reached. That means routed `sessions` no longer goes through the heavier startup stack implicated by the original hang reports.
  - file: [src/cli/run-main.ts:206](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/cli/run-main.ts#L206)
  - command: `nl -ba src/cli/run-main.ts | sed -n '204,212p'`
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Benchmark harness uses non-TTY stdout:** The checked-in CLI startup benchmark runs child commands with `stdio: ["ignore", "ignore", "pipe"]`, so the command under test does not have a TTY stdout.
  - file: [scripts/bench-cli-startup.ts:307](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/scripts/bench-cli-startup.ts#L307)
  - command: `nl -ba scripts/bench-cli-startup.ts | sed -n '300,315p'`
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Shipped fixture records successful non-TTY `sessions --json` exits:** The committed startup-benchmark fixture includes a `sessions --json` case with 5 samples, each exiting with code 0, and an `exitSummary` of `code:0x5`. This same fixture is present in the v2026.4.23 release commit.
  - file: [test/fixtures/cli-startup-bench.json:397](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/test/fixtures/cli-startup-bench.json#L397)

That discussion supports that the report was valid, but the current main/release behavior no longer matches it.
