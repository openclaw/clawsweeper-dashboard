## Label Justifications

- `P1`: The report describes repeated provider-replay crashes that can keep an active Anthropic session unusable until transcript state is repaired.
- `impact:crash-loop`: The same stale thinking block can be replayed repeatedly after compaction, producing repeated request failures.
- `impact:session-state`: The failing state is persisted transcript/session history containing stale provider-owned thinking signatures.

## Summary

Current main already fixes the central crash loop: compacted Anthropic/Bedrock replay strips pre-compaction thinking signatures before provider replay and successor transcript rotation, while the latest release only had the generic replay-invalid classifier. The remaining age-trigger config idea is no longer the best fix for this failure because the runtime now removes the stale provider-owned signatures directly.

## What This Changes

The issue asks OpenClaw to stop replaying stale Anthropic thinking signatures after compaction and optionally add an inactivity/age-based compaction trigger.

## Best Possible Solution

Keep the current-main replay/rotation sanitization as the canonical fix and let the next release carry it, rather than adding a separate age-based compaction knob for this failure mode.

## Reproduction Assessment

Yes. Source inspection shows v2026.6.1 lacked compaction-stale signature stripping, while current main strips those signatures before provider replay and successor transcript rotation.

## Solution Assessment

Yes. The current-main fix is the narrower maintainable solution because it removes stale provider-owned signatures at the replay/rotation boundary while keeping fresh signed thinking intact.
