## Summary

Current main no longer runs `pnpm check` from the pre-commit hook. The hook is formatting-only, tests assert that `pnpm` must not run there, and shipped docs document the same behavior, so this PR's `corepack pnpm` fallback is obsolete on current OpenClaw.

## Evidence

- **Pre-commit hook no longer invokes pnpm:** `git-hooks/pre-commit` now only gathers staged files, filters format targets, runs `oxfmt`, and re-stages files. There is no `pnpm` or `corepack` execution path left in the hook.

- **Integration test proves pnpm must not run:** The shipped integration test creates a fake `pnpm` binary that exits 99 and still expects the hook to succeed for non-doc staged changes, which only works because current pre-commit no longer runs `pnpm`.
