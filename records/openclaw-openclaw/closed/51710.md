## Summary

Current `main` already strips sensitive headers on cross-origin media download redirects via the shared redirect-header helper, and it has regression tests for both cross-origin stripping and same-origin preservation. The PR review concern about preserving `Range` and conditional headers is also addressed on `main` by reusing the broader shared safe-list.
