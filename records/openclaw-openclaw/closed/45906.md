## Summary

Current main already treats successful visible Telegram delivery as a final response, so status reactions no longer fall through to the error emoji when `queuedFinal` stays false. The same logic and regression test are present in the shipped `v2026.4.22` release.

## Evidence

- **Successful Telegram deliveries are tracked explicitly:** `sendPayload()` marks the turn as delivered whenever `deliverReplies()` returns `delivered: true`, and lane text delivery can also mark the turn delivered.
  - file: [extensions/telegram/src/bot-message-dispatch.ts:631](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/extensions/telegram/src/bot-message-dispatch.ts#L631)
  - command: `nl -ba extensions/telegram/src/bot-message-dispatch.ts | sed -n '630,705p'`
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Status reaction finalization now treats visible delivery as success:** `hasFinalResponse` includes `deliverySummary.delivered`, and the final reaction outcome is `done` unless there was a real dispatch error or fallback send.
  - file: [extensions/telegram/src/bot-message-dispatch.ts:1193](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/extensions/telegram/src/bot-message-dispatch.ts#L1193)
