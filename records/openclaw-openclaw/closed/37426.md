## Summary

Keep open. Current `main` still does not implement the Windows local-path vs `file://` normalization described in #37426, so there is not strong evidence that the issue is fixed or obsolete.

## Evidence

- **Core media dedupe still compares raw Windows paths literally:** `filterMessagingToolMediaDuplicates` returns non-`file://` inputs unchanged and, for `file://` inputs, only decodes `pathname` or strips the prefix. It does not normalize Windows drive letters or slash variants, so `C:\foo\bar.jpg`, `C:/foo/bar.jpg`, and `file:///C:/foo/bar.jpg` would not normalize to the same key.
