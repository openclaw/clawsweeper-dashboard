## Summary

Keep open. Current main does not implement the requested LLM context headers, and the proposed patch is useful but not merge-ready because it sends stable identifiers to provider requests without a trusted/opt-in boundary and still has fallback/turn metadata correctness issues.

## What This Changes

The branch adds OpenClaw session, attempt, fallback, turn-index, and turn-id headers to embedded LLM requests and exposes fallback state on plugin LLM output events.

## Best Possible Solution

Adopt a maintainer-approved observability contract that is opt-in or trusted-target scoped, reconciles #69463 and #22021, uses accurate fallback state, keeps per-call IDs unique across retries, and includes tests, docs, and changelog coverage.

## Reproduction Assessment

Yes. The blockers are source-reproducible from the PR diff and current main: headers are merged into outbound model headers, fallback status can diverge from selected fallback candidates, and per-attempt turn counters reset across retry loops.

## Solution Assessment

No. The direction is useful, but unconditional core headers are not the safest path; an opt-in or trusted-target observability contract is the narrower maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Gate context headers to trusted gateway targets:** `src/agents/pi-embedded-runner/run/attempt.ts:2179-2193`
  - body: The new `gatewayContextHeaders` are merged into `params.model.headers`, so ordinary provider transports can receive stable OpenClaw session and run identifiers. Make these headers opt-in or restrict them to trusted gateway/proxy targets before exposing correlation IDs to third parties.
  - confidence: 0.86
- **[P2] Use fallback options for CLI fallback status:** `src/agents/agent-command.ts:520`
  - body: This path forwards `params.isFallbackRetry`, but `runWithModelFallback` can skip primary candidates before invoking the callback. In that case the first executed fallback request is still labeled primary in the new header and hook metadata; pass through the selected candidate's fallback option instead.
  - confidence: 0.88
- **[P2] Keep turn IDs unique across embedded retries:** `src/agents/pi-embedded-runner/run/attempt.ts:2458-2475`
  - body: `llmTurnCount` is local to one embedded attempt and is seeded from persisted assistant messages. Retry paths can restart before an assistant message is written, so a later HTTP request in the same run can reuse the same `X-OpenClaw-Turn-Id`.
  - confidence: 0.82
