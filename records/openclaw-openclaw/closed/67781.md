## Summary

Current main and the v2026.4.29 release already cover the PR's intended behavior: Google Vertex ADC and 1M-context model handling are implemented in the Google-owned transport/model path, and clean empty assistant turns now retry and surface an error before auto-reply delivery. The PR branch should not merge because its wrapper replacement and blanket empty-payload failures would regress existing Google thinking and successful zero-final-payload delivery paths.

## What This Changes

The PR adds Google provider tests and a provider `wrapStreamFn` that strips the Vertex ADC marker, plus `runReplyAgent` empty-payload guards and an e2e test for whitespace-only final payloads.

## Best Possible Solution

Keep the shipped current-main implementation and close this superseded PR. Leave the separate Gemini native-thinking/text-tag behavior in #69220 to its own issue rather than landing this broader branch.

## Reproduction Assessment

Yes. Current-main source and tests provide high-confidence reproduction paths for the relevant behavior: Vertex ADC marker handling, 1M Vertex model matching, and empty assistant retry/error handling are covered; the PR regressions are visible from the diff against the current hook and payload contracts.

## Solution Assessment

No. The PR branch is not the best way to solve the issue because it overrides the Google thinking hook and treats all empty post-filter payloads as failures; current main's provider-owned transport plus embedded incomplete-turn retry is the narrower maintainable solution.
