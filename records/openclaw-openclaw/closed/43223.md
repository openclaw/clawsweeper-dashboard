## Summary

Current main and the v2026.4.24 package already solve the bundled-hook discoverability problem this PR targeted. The package ships `HOOK.md` and `handler.js` together under `dist/bundled/session-memory`, and the shipped resolver is bundled into a `dist/` chunk that resolves `moduleDir/bundled`, so the runtime finds that directory. The PR's proposed `dist/hooks/bundled` destination is obsolete and incomplete against current main because handlers are still emitted under `dist/bundled`.

## Best Possible Solution

Close this PR as obsolete because the shipped implementation already makes bundled hooks discoverable from packaged installs. Keep the current `dist/bundled` packaging path, and if maintainers want a follow-up, make it a focused cleanup that aligns comments and adds a package-layout regression check for `HOOK.md` plus `handler.js` co-location.
