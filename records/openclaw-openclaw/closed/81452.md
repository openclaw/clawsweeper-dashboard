## Summary

Current main does not match the reported strip/drop behavior: webchat/agent image attachments are forwarded as native image blocks, `read` preserves image content blocks, and the exact placeholder strings are documented as old-history replay pruning rather than current-turn image removal.

## What This Changes

The issue asks for webchat uploads and `read` image results to reach vision-capable models as native image blocks instead of text pruning placeholders.

## Best Possible Solution

Keep the current native-image pipeline, and route any remaining Kimi-specific failure to correct model capability metadata or the narrower model/tool-calling thread at https://github.com/openclaw/openclaw/issues/62514.

## Solution Assessment

Yes. The current design is the maintainable path: model catalog capabilities gate inline images, current-turn images are passed as native blocks, and only older replay history is pruned to placeholders.
