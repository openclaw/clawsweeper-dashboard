## Summary

Current main preserves WebChat/chat.send image attachments for text-only primary models by offloading them to managed inbound media refs instead of dropping them, and v2026.4.24 shipped the related WebChat/media-understanding fixes. The related #43259 discussion correctly identified the gap; main now covers the observable failure through the gateway offload path and configured image-tool/image-model path rather than the exact proposed MediaPaths-only implementation.

## Evidence
