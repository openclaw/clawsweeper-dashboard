## Summary

Keep this PR open. Current main still has the Control UI behavior this PR targets: editing the Overview WebSocket URL clears the in-memory Gateway Token, and current browser coverage still asserts that clearing behavior. The PR is not obsolete, but its exact one-line fix still needs maintainer review because it can carry a bearer token from one gateway URL to another, which conflicts with the documented and tested per-gateway token scoping model.

## Best Possible Solution

Keep this PR open for maintainer review or rework. The best fix is to stop losing the token during harmless URL editing while preserving the per-gateway credential boundary, for example by using a draft URL field, preserving tokens only when the normalized gateway scope is unchanged, and requiring an explicit token or confirmation when switching to a different gateway scope.
