This PR adds an opt-in transient provider retry policy, exposes provider HTTP retry helpers, wires safe-stage retries into media, embedding, and video providers, and adds tests plus a changelog entry.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Check timeout-shaped errors before status codes:** `src/provider-runtime/operation-retry.ts:145-149`
  - body: `readErrorStatus` treats any numeric `code` as a status and returns before `hasTimeoutSignal` runs. Node's `AbortSignal.timeout()` produces a `TimeoutError` DOMException with `code === 23`, so opted-in retries still skip real timeout failures from SDK/provider wrappers.
  - confidence: 0.94
- **[P2] Recompute deadline timeouts for each retry attempt:** `src/media-understanding/shared.ts:183-187`
  - body: The helper closes over one `timeoutMs` value, but poll callers pass the remaining deadline computed before the first try. If that first try times out near the deadline, the retry gets the same full timeout and can run past the operation deadline.
  - confidence: 0.88
