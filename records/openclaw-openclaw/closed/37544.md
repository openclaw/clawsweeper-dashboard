## Summary

Current `main` already contains the Slack-side protections that address this report: outbound Slack writes use a no-retry client, `download-file` now requires `fileId` before invoking Slack, and non-retryable Slack dispatch failures are not replayed. Those same protections are present in the released `v2026.4.22` tag.

## Evidence
