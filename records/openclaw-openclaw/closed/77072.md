## Summary

Keep open: current main still has the Telegram preview path that can render exec command progressText, and open PR #77110 is the active closing implementation but has not merged.

## What This Changes

Telegram partial streaming should stop exposing command text/output as repeated transient tool progress bubbles in DMs.

## Best Possible Solution

Land a Telegram-local sanitization fix that drops raw exec output from preview command item progress while preserving final answers and the existing preview.toolProgress opt-out.

## Reproduction Assessment

Yes by source inspection: Telegram partial preview item events consume progressText, and exec command item updates can populate progressText from command output. I did not live-test Telegram in this read-only review.

## Solution Assessment

Yes: the narrow maintainable path is to sanitize command item preview progress in the Telegram plugin and review or land #77110, rather than closing the issue before its closing PR merges.
