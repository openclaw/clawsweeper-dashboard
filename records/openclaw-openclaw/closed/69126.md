## Summary

Current main already implements the requested global compaction default through the shipped `agents.defaults.compaction` surface. The exact proposed key `sessionCompactionLimit` was not added, but the observable request is covered by `agents.defaults.compaction.maxActiveTranscriptBytes` plus `truncateAfterCompaction`, which applies across sessions, triggers normal local compaction before a run when the active transcript exceeds the configured threshold, is documented, tested, and shipped in `v2026.4.26`.

## Best Possible Solution

Keep the shipped `agents.defaults.compaction.maxActiveTranscriptBytes` and `truncateAfterCompaction` design as the supported global compaction threshold instead of adding a duplicate `sessionCompactionLimit` key. Users asking for this behavior should be pointed to the public config docs for `agents.defaults.compaction`.
