## Summary

Current main already addresses the reported Telegram gateway stall pattern. The runtime-deps/filesystem side is covered by a new process-local bundled plugin alias lookup cache, avoiding repeated bundled plugin metadata scans from `normalizePluginId`, and Telegram outbound Bot API requests are now bounded with method-specific abort timeouts. The changelog records this as PR #74210 fixing Telegram/gateway stalls from slow Telegram sends or WSL2 filesystem scans. The fix is present on main but is not contained in any local release tag yet.

## Best Possible Solution

Close this issue as fixed on current main by PR #74210’s code path, keep the alias-cache and Telegram timeout changes, and ensure the next release includes them. If a build containing these commits still stalls on Telegram, the best follow-up is a new issue with the exact build SHA, liveness logs, and fresh strace/perf data.
