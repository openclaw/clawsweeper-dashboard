## Summary

Current main contains a targeted OpenAI provider fix for this regression: direct OpenAI Responses models now default to SSE instead of WebSocket auto-selection, matching the reported hang mode and the Unreleased changelog entry.

## What This Changes

The issue reports that after upgrading to OpenClaw 2026.5.2, OpenAI-backed agent turns hang indefinitely with models.list delays, event-loop delay warnings, idle watchdog timeouts, and /v1/responses fetch timeouts.

## Best Possible Solution

Keep the direct OpenAI Responses SSE default, preserve explicit transport overrides, and include the fix in the next shipped release; reopen only if a post-fix build still hangs with fresh Gateway diagnostics.

## Reproduction Assessment

No high-confidence current-main failure reproduction is available. The reported 2026.5.2 logs map to the old WebSocket-first path, but current main now defaults direct OpenAI Responses to SSE and has regression coverage for that default.

## Solution Assessment

Yes. Defaulting direct OpenAI Responses to SSE is the narrowest maintainable fix because it avoids the stalled WebSocket path without removing explicit `websocket` or `auto` transport overrides.
