## Summary

This is a duplicate of the existing Discord/group-channel visible-replies upgrade report in #74842. Current main confirms the same root cause: Discord guild/channel turns default to `message_tool_only`, so normal final replies are private unless the agent uses `message(action=send)` or `messages.groupChat.visibleReplies` is set to `automatic`; #74842 remains the canonical open thread for any remaining migration/default behavior decision.

## What This Changes

The issue reports that Discord guild-channel turns on OpenClaw 2026.4.27 complete locally but do not post the final assistant reply back into Discord.

## Best Possible Solution

Keep #74842 as the single canonical thread for whether Discord upgrade behavior should migrate, warn, doctor-check, or preserve automatic posting differently; until that changes, users should opt into legacy visible replies with `messages.groupChat.visibleReplies: "automatic"` or have the agent send visible output with `message(action=send)`.
