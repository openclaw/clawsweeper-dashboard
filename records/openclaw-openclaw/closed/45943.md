## Summary

Current main, and the shipped v2026.4.26 tag, implement the supported safe path for this request: `openclaw update` handles npm/pnpm/bun installs, refreshes the managed Gateway service from the updated install, restarts it, and verifies the restarted Gateway reports the expected version.

## Best Possible Solution

Keep the supported update behavior in `openclaw update` rather than restarting Gateway services from npm lifecycle scripts. Direct npm installs should remain documented as a manual/advanced path, with `openclaw update` as the safe official flow that refreshes, restarts, and verifies the managed Gateway.
