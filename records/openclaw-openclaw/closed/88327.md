This PR should stay open for normal maintainer review: current main still forces Control UI `chat.history` calls through a 4,000-character RPC override, while the PR removes that override and adds focused UI plus gateway regression coverage for the documented gateway config path.

## What This Changes

The branch removes Control UI's hardcoded `chat.history.maxChars` override and updates UI/gateway tests so `gateway.webchat.chatHistoryMaxChars` can control WebChat history truncation.

## Best Possible Solution

Land the narrow client override removal with the added regression tests after maintainer review and final CI, leaving the broader full-message reader work to the existing follow-up issue.

## Reproduction Assessment

Yes. Current main source shows Control UI always sends `maxChars: 4000`, and gateway source/docs show that RPC value overrides `gateway.webchat.chatHistoryMaxChars`; the PR body also supplies a live gateway/WebSocket comparison.

## Solution Assessment

Yes. Removing the Control UI RPC override is the narrowest maintainable fix because the gateway already owns config/default resolution and per-request overrides remain available to explicit callers.
