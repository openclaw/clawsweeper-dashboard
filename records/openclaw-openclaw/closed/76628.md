## Summary

Keep open. Current main and v2026.5.2 still grant `internalDiagnostics` only to bundled diagnostics services while official docs/catalog metadata expose install paths that can leave `diagnostics-prometheus` running as a global plugin with no event subscription.

## What This Changes

The issue asks OpenClaw to stop letting official `diagnostics-prometheus` npm/global installs load without the privileged diagnostics capability, which yields empty Prometheus scrapes and error logs.

## Best Possible Solution

Define one supported trusted distribution contract for diagnostics exporters and align the capability gate, install path, docs, and catalog so users cannot enable a metrics route that never receives events.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: managed installs are discovered as `global`, `internalDiagnostics` is granted only to `bundled`, and the Prometheus service returns before subscribing when the capability is absent.

## Solution Assessment

Unclear. The report identifies the broken contract, but choosing between trusted distribution, install rejection, docs limitation, or a broader trust mechanism is a maintainer/security policy decision.
