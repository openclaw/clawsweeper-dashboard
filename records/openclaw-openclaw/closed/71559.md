## Summary

Current `main` already fixes the cold-start CLI lookup path for the built-in `local` memory embedding provider. The reporter's `manager-runtime.js` observation matched older packaged behavior, but current `main` no longer depends on import-time provider registration for this case: `memory-core` now declares the `local` adapter in manifest contracts, and provider lookup falls back to bundled capability resolution when the registry is cold.
