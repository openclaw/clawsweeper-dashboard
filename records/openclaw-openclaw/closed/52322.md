## Summary

Issue #52322 is superseded by open PR #56650, which is the active implementation track for the same remaining GitHub Copilot PAT auth work and explicitly fixes #52322. Current main still has the reported gap: configured GitHub tokens are accepted from env/auth profiles, but runtime auth still unconditionally exchanges them through the Copilot token endpoint with a Bearer header rather than handling PATs directly.
