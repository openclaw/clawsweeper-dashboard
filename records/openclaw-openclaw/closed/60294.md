## Summary

Issue #60294 is a language-specific Android localization request and is superseded by the maintainer canonical i18n/localization tracker #3460. Current main still does not implement native Android zh-CN localization: Android resources are minimal and the key Compose surfaces remain hardcoded in English. A related mobile localization PR, #42089, attempted Android zh-rCN resources but was closed unmerged and pointed back to #3460.

## Best Possible Solution

Close this issue as superseded by #3460. Future Android zh-CN work should wait for a maintainer-requested i18n architecture, then return as a focused Android PR that extracts Compose UI copy into resources, adds `values-zh-rCN`, preserves Android system-locale behavior, and covers Connect, Chat, Voice, and Settings with appropriate validation.
