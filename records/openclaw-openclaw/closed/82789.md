## Summary

Keep open: current main does not include fal/OpenRouter music-generation support, and this protected-label feature PR still has a concrete OpenRouter streaming correctness blocker on the latest head.

## What This Changes

The PR adds fal and OpenRouter music-generation providers, SDK music helper exports, provider manifests/onboarding/docs, live coverage, and related provider/runtime tests.

## Best Possible Solution

Keep the provider feature, but repair OpenRouter stream assembly to join base64 audio fragments before decoding and add a non-aligned chunk regression test before maintainer review.

## Reproduction Assessment

Yes, at source level: latest PR head decodes each OpenRouter audio SSE fragment separately, and a non-4-character base64 split reproduces corrupted bytes without live API access.

## Solution Assessment

No, the PR should follow OpenRouter's stream contract by collecting `delta.audio.data` strings and decoding once after completion; that is narrower than changing provider selection or the shared music runtime.

## Review Findings

Overall correctness: patch is incorrect
