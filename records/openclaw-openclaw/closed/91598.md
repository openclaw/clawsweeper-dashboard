## Summary

Keep open: this maintainer-labeled Control UI performance PR is not redundant on current main, and source review did not find a blocking patch defect. Current main still schedules `commands.list` after startup metadata that only contains models, while the PR defers command hydration to composer slash intent or palette open.

## What This Changes

The PR removes the clean-startup `commands.list` follow-up from Control UI chat startup and adds lazy slash-command hydration from the chat composer and command palette, with focused unit and mocked E2E coverage.

## Best Possible Solution

Land the lazy hydration path after normal maintainer review and required checks, keeping command discovery out of the clean-startup path while preserving explicit slash and palette hydration.

## Reproduction Assessment

Yes. Source inspection of current main shows the clean-startup path can schedule `commands.list` after model-only startup metadata; the PR tests encode that path and the new lazy hydration triggers.

## Solution Assessment

Yes. Deferring to the existing `refreshSlashCommands` cache from composer slash intent and palette open is the narrowest maintainable fix I found; moving command discovery into `chat.startup` would keep command enumeration on the first-load path.
