## Label Justifications

- `impact:crash-loop`: Unbounded descriptor retention can drive the long-lived gateway toward EMFILE and process-level availability failure.

## Summary

Keep open. Current main and v2026.6.1 still route workspace skill roots through chokidar in a way that admits `SKILL.md` file entries, and chokidar 5 then creates file-level watchers for included files, matching the reported persistent FD pattern.

## What This Changes

Fix the gateway skills watcher so workspace `SKILL.md` changes still refresh snapshots without retaining one persistent read-only file descriptor per skill file.

## Best Possible Solution

Replace or constrain the skills refresh watcher so it does not register persistent per-`SKILL.md` file watchers, while preserving documented snapshot refresh behavior and proving the result with focused tests plus macOS FD proof.

## Reproduction Assessment

Yes, source-reproducible: current main still feeds `SKILL.md` file entries into chokidar, and chokidar 5 creates per-file watchers for included files. A live macOS `lsof` run remains the before/after proof for the eventual fix.

## Solution Assessment

No fix is present yet. The best solution is a narrow watcher refactor that keeps `skills.load.watch` semantics but avoids chokidar's per-file watcher path for `SKILL.md`.
