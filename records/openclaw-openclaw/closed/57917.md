## Summary

Current `main` already hardens Control UI model switching so bare model names/aliases are normalized against the catalog, stale provider prefixes are corrected, and `/model` results are mirrored back as provider-qualified overrides instead of drifting to the current provider.

## Evidence

- **UI normalizes bare model values against the catalog:** `resolvePreferredServerChatModelValue()` first resolves raw values through the catalog and only falls back to provider prefixing when no unique catalog match exists, which prevents a bare `gpt-5.4` from being rewritten to the current provider when the configured model entry is unique.
  - file: [ui/src/ui/chat-model-ref.ts:31](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/ui/src/ui/chat-model-ref.ts#L31)
  - command: `nl -ba ui/src/ui/chat-model-ref.ts | sed -n '20,155p'`
  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
- **Control UI picker uses resolved values:** The chat model select reads the current override through `resolveChatModelSelectState()` and submits the selected value directly to `sessions.patch`; the picker options are built from resolved/provider-qualified values rather than raw display text.
  - file: [ui/src/ui/chat-model-select-state.ts:34](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/ui/src/ui/chat-model-select-state.ts#L34)
  - command: `nl -ba ui/src/ui/chat-model-select-state.ts | sed -n '24,90p'`
  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
- **Slash-command path mirrors qualified resolved models:** `executeModel()` patches with the requested input, then rewrites the local session override from the server-resolved provider/model plus catalog normalization so the Control UI keeps the canonical qualified model instead of a stale-provider/raw mix.
  - file: [ui/src/ui/chat/slash-command-executor.ts:210](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/ui/src/ui/chat/slash-command-executor.ts#L210)
  - command: `nl -ba ui/src/ui/chat/slash-command-executor.ts | sed -n '208,245p'`
  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
- **Backend still resolves aliases before default-provider fallback:** `resolveModelRefFromString()` checks the alias index before parsing provider/model refs, and `resolveAllowedModelRefFromAliasIndex()` infers a unique configured provider for bare model ids before falling back to the session default provider.
  - file: [src/agents/model-selection-shared.ts:348](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/src/agents/model-selection-shared.ts#L348)
  - command: `nl -ba src/agents/model-selection-shared.ts | sed -n '348,370p;598,623p'`
  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
- **Regression tests cover this family of failures:** UI tests assert that `/model` changes are mirrored back as `openai/...`, that missing-provider patch responses are still qualified from the local catalog, and that stale patched providers are corrected; backend tests also cover preventing provider-prefix drift for bare model ids under a different current provider.
  - file: [ui/src/ui/chat/slash-command-executor.node.test.ts:275](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/ui/src/ui/chat/slash-command-executor.node.test.ts#L275)
