## Summary

Keep open: current main has resolved the restart-side LaunchAgent recovery, but the stop-side path still has the short confirmation window and degraded bootout fallback that can leave the service unloaded. The remaining question is a maintainer decision about intended `gateway stop` semantics, not a cleanup-close candidate.

## What This Changes

The issue asks OpenClaw to avoid leaving the macOS Gateway LaunchAgent unloaded when `gateway stop` or `gateway restart` exceeds the current launchd stop confirmation budget.

## Best Possible Solution

Keep the merged restart hardening, then make an explicit maintainer call on stop behavior: either extend/rework the stop fallback so a slow but healthy shutdown does not leave launchd unloaded, or document that degraded bootout is intentional and let #60885 carry the broader LaunchAgent downtime work.
