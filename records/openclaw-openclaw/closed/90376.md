## What This Changes

The PR changes IRC account tests to replace unconditional Windows skips for symlinked password-file cases with a runtime file-symlink capability probe.

## Best Possible Solution

Keep the dynamic capability-check approach, but make the probe use a temporary directory with best-effort cleanup and refresh proof with output that demonstrates the symlink-capable path or clearly logs the probe result.

## Reproduction Assessment

Yes from source for the test gap: current main unconditionally skips these symlink password-file tests on win32, so symlink-capable Windows environments do not exercise the existing guard.

## Solution Assessment

Mostly yes; a dynamic file-symlink capability check is the right layer for this test-only change, but the implementation should clean up probe artifacts and prove the path it intends to unlock.
