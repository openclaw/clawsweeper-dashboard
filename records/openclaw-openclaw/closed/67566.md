## Summary

Current main already covers the reported memory_search failure mode: empty memory meta rows are handled as missing index metadata, memory_search converts embedding/search failures into structured unavailable results instead of throwing, and the proxy-specific remote-embedding path raised in the issue comment is fixed in the memory-host SDK.

## Best Possible Solution

Close as implemented on current main. Keep the current memory-host SDK env-proxy gating, missing-meta reindex behavior, and structured memory_search unavailable payloads; #52162/#71506 remain the canonical proxy-specific history for the related failure mode.
