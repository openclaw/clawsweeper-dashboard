## Summary

Current main already has the Control UI logic this report asks for: chat history is rendered in forward array order, cross-run/final messages are appended rather than prepended, terminal chat events clear the active stream/run state, and deferred `session.message` reloads are replayed after the active run ends.

## Evidence

- **History renders in forward order:** `buildChatItems()` walks `props.messages` from `historyStart` upward and pushes each item in that order, so the chat thread renders oldest-to-newest rather than prepending newer items.
  - file: [ui/src/ui/chat/build-chat-items.ts:213](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/ui/src/ui/chat/build-chat-items.ts#L213)
