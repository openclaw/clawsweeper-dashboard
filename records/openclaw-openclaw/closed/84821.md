## What This Changes

The issue requests deduping live tool websocket delivery so overlapping run-scoped `agent` recipients do not also receive the same `session.tool` mirror while session-only late joiners still do.

## Best Possible Solution

Review and land or reject the linked Gateway dedupe PR, preserving `session.tool` for session-only late joiners while avoiding duplicate mirrors for run-scoped tool recipients.

## Reproduction Assessment

Yes. Source inspection shows a Control UI connection can advertise `tool-events`, subscribe to sessions, and then receive both run-scoped `agent` and session-scoped `session.tool` frames for the same tool event on current main.

## Solution Assessment

Yes, conditionally. The linked PR's recipient filtering is the narrow maintainable fix shape because it changes only delivery targeting and preserves session-only late joiners, but the issue should stay open until that PR merges or maintainers choose a different contract.
