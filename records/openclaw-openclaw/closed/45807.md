## Summary

Current `main` already preserves Control UI user image attachments across the post-response history reload path: gateway `chat.send` rewrites uploaded user images into persisted `MediaPath`/`MediaPaths`, and the Control UI renderer rebuilds user-message images from those history fields with regression coverage.

## Evidence

- **Gateway persists uploaded user images into transcript media fields:** `resolveChatSendTranscriptMediaFields(...)` builds `MediaPath`/`MediaPaths` + `MediaType`/`MediaTypes`, and `rewriteChatSendUserTurnMediaPaths(...)` rewrites the matching user transcript entry so history-backed reloads retain the uploaded image reference.
  - file: [src/gateway/server-methods/chat.ts:755](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/gateway/server-methods/chat.ts#L755)
  - command: `nl -ba src/gateway/server-methods/chat.ts | sed -n '755,829p'`
  - sha: [f03252aaf9cc](https://github.com/openclaw/openclaw/commit/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022)
- **Rewrite is invoked on the chat.send path after images are persisted:** `rewriteChatSendUserTurnMediaPaths(...)` is called from the live `chat.send` flow with `savedImages: await persistedImagesPromise`, which is the exact path needed for uploaded Control UI images to survive later `chat.history` reloads.
  - file: [src/gateway/server-methods/chat.ts:2363](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/gateway/server-methods/chat.ts#L2363)
