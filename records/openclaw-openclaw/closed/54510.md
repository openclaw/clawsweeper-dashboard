## Summary

Current `main` now routes Telegram forum/topic conversations through topic-scoped session keys and topic-scoped transcript files, and the generic embedded runner calls `contextEngine.assemble()` and `contextEngine.afterTurn()` for every run attempt without a DM/group/topic split. The missing group/topic context-engine lifecycle described in #54510 is implemented on `main`, but I could not tie the fix to a released tag yet.

## Evidence
