## Summary

Current main already supports Apple Reminders on a remote macOS node through the generic remote macOS skill path and node-host exec approvals, so a hard-coded `macky` or `remindctl` gateway allowlist should not be added to core.

## What This Changes

The issue requests allowing agents to invoke `remindctl` on a private macOS node for Apple Reminders access.

## Best Possible Solution

Keep the generic remote macOS skill and exec-approval path, and handle remaining failures as node setup: install `remindctl`, allow `system.run` on the node, and refresh pairing after command-surface changes.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a request to enable a private-node CLI. Source and docs show current main already supports the generic path when the macOS node advertises `system.run` and the binary is present.

## Solution Assessment

Yes. The maintainable solution is the existing generic `exec host=node` plus node-local Exec approvals, not a core allowlist entry for a private node name or a specific binary.
