## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Return real sessions from `_getAllSessions`:** `src/acp/session.ts:95`
  - body: This function body uses the TypeScript `Record` utility as a runtime value, and `saveToDisk()` depends on it to serialize sessions. That either fails typechecking or prevents file-backed ACP persistence from saving real session data.
  - confidence: 0.97
- **[P2] Keep enhanced compaction config session-local:** `src/agents/pi-extensions/compaction-safeguard.ts:290-292`
  - body: Calling `configureEnhancedCompaction()` inside an individual compaction rewrites module-level config shared by the process. Concurrent sessions with different settings can race and run under another session's config; pass the config into the compaction call instead.
  - confidence: 0.88
- **[P2] Replace prefix-based shared-context access:** `src/agents/shared-context.ts:371-372`
  - body: `parent-child` access should not be inferred from string prefixes. An agent ID like `main` would grant access to unrelated IDs such as `main-attacker`, so this needs explicit relationship tracking before shared context is enabled.
  - confidence: 0.86
- **[P2] Validate persisted session JSON before replaying it:** `src/acp/session.ts:120-125`
  - body: Parsed JSON entries are replayed into `createSession()` without checking that `sessionKey` and `cwd` are strings. A corrupt or unexpected state file can create malformed ACP sessions instead of being skipped cleanly.
  - confidence: 0.8
