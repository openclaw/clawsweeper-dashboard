## Summary

Close as `clawhub`: the PR is coherent and useful, but it changes a bundled core skill to depend on a contributor-maintained external fork. Current `main` scope/docs route that kind of optional skill distribution through ClawHub/plugins rather than core promotion.

## Evidence

- **Bundled apple-notes skill on main is still the baseline upstream memo integration:** Current `main` ships `skills/apple-notes/SKILL.md` pointing at `antoniorodr/memo`, with Homebrew install metadata for `antoniorodr/memo/memo` and interactive `memo notes` usage. The proposed PR would replace that bundled dependency with the contributor fork and add agent-only API docs.
