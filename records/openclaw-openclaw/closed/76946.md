## What This Changes

The issue asks OpenClaw to stop putting detailed bootstrap truncation diagnostics, including file sizes and config knobs, into routine WebChat/runtime conversation context.

## Best Possible Solution

Review and either land or revise PR #76994 as the canonical fix, keeping detailed counts/config hints on diagnostics while preserving only approved concise guidance in model context.

## Reproduction Assessment

Yes, from source inspection: oversized bootstrap files produce warning lines, the embedded runner appends them to effectivePrompt, and WebChat-style transcript/effective prompt differences can move that addition into hidden runtime context. I did not run a live WebChat turn because this review is read-only.

Unclear until maintainer review of PR #76994 finishes; its described approach matches the narrow surface, but the final default for agent-visible truncation guidance needs explicit approval.
