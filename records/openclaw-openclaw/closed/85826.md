## Summary

Keep open: current source and the reported release already expose global diagnostics knobs, but current main still lets long silent active model calls become stalled and active-abort eligible, and no merged fix fully owns the reported takeover/message-loss path.

## What This Changes

The issue asks OpenClaw to stop diagnostics/recovery from aborting legitimate long local-vLLM model calls, with configurable thresholds, visible still-working feedback, and recovery instead of failed sessions.

## Best Possible Solution

Treat this as a session-recovery policy item: keep the existing global diagnostics and provider timeout knobs, then decide provider-aware progress, abort, and channel-visible still-working semantics for long silent local model calls.

## Reproduction Assessment

Yes at source level, but not as a live vLLM run. Current code and tests show active model calls can be classified as stalled and become active-abort candidates; the exact production takeover timing was not reproduced here.

## Solution Assessment

Unclear. A duplicate timeout knob is not the best first fix because global diagnostics and provider timeout controls already exist; the maintainable path is a deliberate recovery policy for long silent local model calls.
