## Summary

Close PR #48583 as superseded. The same underlying bug/work is already tracked by #48573, while current main has a different subagent registry lifecycle with persistent cleanup, orphan reconciliation, delete-mode sweeping, and regression coverage. This PR is not a good patch to revive because it would delete run-mode subagents even when cleanup is explicitly keep, and it bundles unrelated Slack/json/gateway changes that review already flagged.

## Best Possible Solution

Close this stale PR and continue any remaining work in #48573. A future fix, if needed, should be narrowly scoped to current `src/agents/subagent-registry*` and `src/agents/subagent-control.ts`, preserve explicit `cleanup: "keep"` semantics, and include regression coverage for gateway-delete failure and parent/child termination behavior.
