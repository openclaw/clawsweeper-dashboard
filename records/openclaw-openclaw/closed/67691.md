## Summary

Current `main` already implements the requested model-agnostic local context reduction: in-memory session pruning for old tool results, configurable pruning heuristics, live tool-result truncation, and automatic conversation compaction/summarization across providers. The shipped `v2026.4.23` release already contains these paths.

## Evidence

- **Session pruning docs:** The user-facing docs say session pruning trims old tool results before each LLM call, is in-memory only, and can be enabled explicitly for non-Anthropic providers with `agents.defaults.contextPruning.mode: "cache-ttl"`.
