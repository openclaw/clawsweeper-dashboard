## Summary

## What This Changes

The issue reports isolated heartbeat wake re-entry creating nested `session:heartbeat:heartbeat...` sessions and orphan transcript files.

## Best Possible Solution

Keep the current main implementation: canonicalize isolated heartbeat sessions to one `:heartbeat` sibling and clean up stale accumulated keys on the next heartbeat tick.

## Solution Assessment

Yes. Canonicalizing the isolated heartbeat key and archiving stale accumulated entries directly addresses the nested-session loop without adding a new depth-limit policy or config option.
