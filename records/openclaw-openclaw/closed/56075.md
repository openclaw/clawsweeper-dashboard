## Summary

Current `main` already implements automatic shared-skill propagation for existing chats by versioning skill snapshots, watching shared/global skill roots for `SKILL.md` changes, and rebuilding stale session snapshots on the next turn. The explicit in-chat refresh UX requested in the body is still separate feature work, but the propagation bug described by the title is implemented.
