## Summary

Keep open. Current main still has the reported full-catalog paths: the Gateway configured model view can fall back to the complete catalog, Web chat renders every model returned by that view, and `/models` builds provider data from `allowed.allowedCatalog`, which is the full catalog when no allowlist is configured. The closed #74473 PR only touched the picker helper and was not merged; #42797 and #59693 overlap with parts of the problem but are open and do not provide a complete merged fix for both Web chat and `/models`.

## What This Changes

The issue asks for `/models` and the Web chat model picker to default to configured or authenticated provider models instead of showing the entire built-in catalog.

## Best Possible Solution

Introduce one shared default-visible-model policy used by Gateway/Web chat, `/models`, and the interactive picker: prefer explicit allowlists and configured provider models, include auth-backed provider catalog entries when usable credentials exist, and reserve the full catalog for an explicit `all` or `view: "all"` browse path with updated docs and regression tests.
