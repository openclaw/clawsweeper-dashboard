## Summary

Current `main` already provides both requested capabilities for `memory_search`: provider-less local keyword/BM25 fallback and a built-in local embedding provider that does not require an external API.

## Evidence

- **Docs describe local embeddings without API:** `memory_search` docs explicitly say to use `provider: "local"` for local embeddings with no API key.
  - file: [docs/concepts/memory-search.md:32](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/docs/concepts/memory-search.md#L32)
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **Docs describe keyword-only fallback:** Builtin memory docs state that without an embedding provider, keyword search remains available, and the engine provides FTS5/BM25 keyword search.
  - file: [docs/concepts/memory-builtin.md:14](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/docs/concepts/memory-builtin.md#L14)
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **Local embedding provider exists in code:** `createLocalEmbeddingProvider()` loads a local GGUF embedding model through `node-llama-cpp` and exposes `embedQuery`/`embedBatch` without any external API dependency.
  - file: [src/memory-host-sdk/host/embeddings.ts:23](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/src/memory-host-sdk/host/embeddings.ts#L23)
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **Provider-less FTS/BM25 search path exists:** The memory manager has an explicit `FTS-only mode: no embedding provider available` branch that runs keyword search, expands conversational queries into keywords, merges results, and returns scored results even with no provider.
  - file: [extensions/memory-core/src/memory/manager.ts:341](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/extensions/memory-core/src/memory/manager.ts#L341)
  - sha: [754acc4478fe](https://github.com/openclaw/openclaw/commit/754acc4478fe361db1e5303fd60f589d52beea7a)
- **Tests cover FTS-only indexing/search mode:** The FTS-only test suite mocks `provider: null` and verifies forced reindexing preserves and refreshes indexed chunks in no-provider mode.
  - file: [extensions/memory-core/src/memory/manager.fts-only-reindex.test.ts:10](https://github.com/openclaw/openclaw/blob/754acc4478fe361db1e5303fd60f589d52beea7a/extensions/memory-core/src/memory/manager.fts-only-reindex.test.ts#L10)

## Risks / Open Questions

- The issue body mentions ONNX/all-MiniLM-L6-v2 as one possible implementation; that exact backend is not what ships today. The close rationale is that the requested no-external-API capability is already covered by the existing local provider plus FTS fallback.
