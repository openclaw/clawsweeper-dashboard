## Summary

Current main already fixes the packaged memory runtime-deps gap: `sqlite-vec` is now declared as a mirrored root runtime dependency, the bundled runtime-deps planner merges those package-level mirrors into the memory plugin install plan, and regression coverage asserts `sqlite-vec@0.1.9` is staged for `memory-core`. The fix appears to be main-only in this checkout because no release tag contains the proof commit.

## What This Changes

Restore `sqlite-vec` in packaged default memory runtime dependencies after the v2026.4.27 upgrade so `memory_search` keeps vector ranking without a manual install.

## Best Possible Solution

Keep the current package-level mirrored dependency approach and ship it in the next release; changing the import subpath alone would not repair installs where `sqlite-vec` is absent from the managed runtime-deps tree.

## Reproduction Assessment

Yes. The v2026.4.27 package metadata has `sqlite-vec` as a dependency but no `openclaw.bundle.mirroredRootRuntimeDependencies` block, while current main adds that mirror and has a focused regression test for the memory-core staging plan.

## Solution Assessment

Yes. Mirroring `sqlite-vec` into the packaged bundled runtime-deps install plan is the narrowest maintainable fix because the failure is package absence in `plugin-runtime-deps`, not the TypeScript import surface.
