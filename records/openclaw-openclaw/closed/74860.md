## Summary

Keep this open. The report has concrete production profiling plus later 2026.4.29 confirmation, and current main still contains repeated tool-construction, model-resolution, and provider-auth paths that can rebuild manifest-backed lookup data during agent turns; related partial mitigations do not fully cover the per-turn auth/model/tool hot path described here.

## What This Changes

The issue asks OpenClaw to stop repeated gateway event-loop stalls caused by synchronous plugin registry and provider-auth lookup work during tool creation and agent-turn startup.

## Best Possible Solution

Make the agent-turn hot path reuse bounded, freshness-aware lookup products for provider env candidates/auth evidence, manifest capability provider IDs, and compatible active plugin registries across one runtime/config snapshot, while preserving plugin install/config changes on the next appropriate snapshot refresh.

## Reproduction Assessment

Yes. The issue gives a concrete production reproduction path for 2026.4.27 and a later 2026.4.29 stage-timing confirmation; current source inspection shows the same tool, model-resolution, and provider-auth lookup shapes still exist, though this review did not run a live gateway replay.

## Solution Assessment

Unclear because there is no patch yet. The reporter's memoization/instrumentation direction is sound, but the safest implementation should be scoped to request/config-snapshot lookup reuse rather than adding an unbounded persistent plugin metadata cache.
