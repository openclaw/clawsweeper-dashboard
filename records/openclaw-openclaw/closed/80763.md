## Summary

Current main already enforces the documented strict inline-eval approval boundary for gateway and node host exec, so the reported allowlist bypass is fixed on main but not yet in a tagged release.

## What This Changes

The issue requests that `tools.exec.strictInlineEval` force explicit approval for inline interpreter evals such as `python3 -c` and `node -e` even when the interpreter is allowlisted by path in allowlist mode.

## Best Possible Solution

Keep the current-main strict inline-eval approval enforcement and let the next release carry the fix to users.

## Reproduction Assessment

No. I did not run a live gateway, but current main source shows `python3 -c` and `node -e` are detected and forced through approval even when allowlist evaluation succeeds.

## Solution Assessment

Yes. The current implementation is the narrow maintainable fix because it adds the strict inline-eval approval boundary on top of allowlist satisfaction instead of changing path allowlist matching semantics.
