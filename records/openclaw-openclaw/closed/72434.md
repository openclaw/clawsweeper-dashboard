## Summary

Current main already fixes the reported 2026.4.24 regression. Legacy `agents.defaults.embeddedHarness.runtime: "claude-cli"` is migrated/read as `agents.defaults.agentRuntime.id`, `claude-cli` is treated as a CLI backend alias rather than an embedded AgentHarness id, canonical Anthropic model refs route through the Claude CLI runner, fallback paths no longer pass `claude-cli` as `agentHarnessId`, and startup planning can auto-enable the Anthropic plugin owner for the `claude-cli` backend. The later OAuth-token snapshot comments are useful but describe a separate auth-refresh concern, not the harness-registration outage reported here.

## Best Possible Solution

Close this as implemented on main. Keep the shipped `agentRuntime`/CLI-backend split, legacy config migration, fallback guard, and Anthropic plugin-owner loading. If the OAuth snapshot versus Keychain refresh behavior mentioned later still reproduces, track it as a separate auth-refresh issue with fresh logs/config rather than keeping this harness-registration regression open.
