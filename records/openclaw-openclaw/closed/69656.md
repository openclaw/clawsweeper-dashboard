## Summary

QQBot HTTP/SOCKS proxy support is a useful channel-specific transport variant, but current main already has the plugin surfaces needed to ship and select that behavior outside OpenClaw core. The issue itself points to a third-party QQBot proxy fork, and OpenClaw's docs/code support external channel plugins replacing the bundled qqbot owner via preferOver, so the best cleanup is to close this core issue as ClawHub/plugin scope.

## Best Possible Solution

Close this OpenClaw core issue as ClawHub/plugin scope. The practical path is a maintained QQBot proxy plugin or fork, published through ClawHub/npm and declaring preferOver for qqbot; only reopen/core-track if that plugin hits a concrete missing Plugin SDK or channel replacement API.
