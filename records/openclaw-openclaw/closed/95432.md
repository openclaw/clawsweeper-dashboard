This PR changes `createBlockReplyPipeline` to track streamed text fragments per assistant message and adds regression tests for duplicate final suppression after multi-message block streaming.

Yes. The fix stays in the existing block-streaming suppression predicate, mirrors the sibling per-message grouping rule, and avoids channel, protocol, or config changes.
