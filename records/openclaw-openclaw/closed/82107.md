## Summary

Close: current main and the shipped v2026.6.6 release now route main-session cron systemEvent ticks into per-run child sessions, so the reported recurring full-transcript amplification no longer matches the implementation.

## What This Changes

Prevent recurring cron systemEvent jobs targeted at long-lived sessions from resending the host transcript on every tick.

## Best Possible Solution

Keep the shipped per-run cron lane behavior, with broader model-call watchdog and no-model reminder cost work handled by their separate canonical issues.

## Reproduction Assessment

No current-main reproduction remains for the central issue. I did not run a multi-day live transcript, but source, tests, docs, and the shipped release now show main cron ticks using fresh per-run child session keys instead of the long-lived host transcript.

## Solution Assessment

Yes. The shipped per-run child-session lane is the narrow maintainable fix because it preserves main-session delivery behavior while avoiding routine cron transcript accumulation.
