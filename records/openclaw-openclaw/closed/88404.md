## Summary

Close as implemented on current main: OpenClaw now derives effective session context budgets from active provider/model metadata, treats numeric `agents.*.contextTokens` as a cap rather than an expansion, passes the effective budget into embedded-run compaction, and clears stale session budget metadata on model changes. The exact `"auto"` string and safety-factor config were not added, but the central user problem is solved by default behavior on main.

## What This Changes

The issue requests model-aware session context budgeting so OpenClaw derives the effective compaction limit from the active model's catalog window instead of relying on a static global `contextTokens` cap.

## Best Possible Solution

Keep the current implicit model-aware budget resolution on main and leave explicit per-model override policy to the narrower related config item if maintainers still want that extra surface.

## Reproduction Assessment

No current-main failing reproduction is needed for closure: this is a feature/config request, and source plus regression tests show the central model-aware behavior is now present. The original and follow-up live reports remain credible for older releases before the current-main implementation.

## Solution Assessment

Yes. The current implementation is the better end state because it makes unset/default behavior model-aware without adding a second `"auto"` string mode, while still respecting numeric caps and provider/model `contextTokens` overrides.
