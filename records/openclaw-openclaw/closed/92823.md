## Summary

Keep open for maintainer review rather than cleanup: current main and the latest release still allow QQBot lower-layer send errors or missing platform IDs to become empty successful receipts, while this PR fixes that bounded adapter failure path with tests and redacted live logs. I found no blocking patch defect, but the shared text/media fail-fast behavior is a compatibility choice before merge.

## What This Changes

The branch makes QQBot message adapter text and media sends reject on outbound errors or missing QQ platform message IDs, with adapter regression tests.

## Best Possible Solution

Land the bounded adapter error-propagation fix after maintainer acceptance of shared QQBot text/media fail-fast semantics, while keeping the broader QQ C2C media delivery work on the canonical issue.

## Reproduction Assessment

Yes at source level: current main and v2026.6.6 can convert QQBot lower-layer errors or missing ids into an empty successful receipt, and the PR tests cover the reject path. I did not run a credentialed QQBot live call, but the PR body includes redacted after-fix runtime logs.

## Solution Assessment

Yes for the bounded error-propagation problem: the adapter normalizer is the narrowest maintainable layer to reject lower-level send errors and empty receipts. It is not the full solution for QQ C2C media delivery, which remains tracked separately at https://github.com/openclaw/openclaw/issues/92246.
