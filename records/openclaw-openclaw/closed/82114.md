## Summary

Keep this PR open: the Feishu muxer change addresses the reported `.ogg.part` ffmpeg failure, but the Discord fake-IP SSRF exemption should be narrowed before merge because the new policy is broader than the existing host-scoped fake-IP patterns.

## What This Changes

The PR allows Discord voice upload guarded fetches through RFC2544/IPv6 ULA fake-IP DNS ranges, adds `-f ogg` to Feishu voice transcoding, updates focused tests, and adds a changelog entry.

## Best Possible Solution

Keep the Feishu `-f ogg` fix and Discord fake-IP compatibility, but make the Discord SSRF policy host-scoped per upload/API URL before landing.

## Reproduction Assessment

Yes for the Feishu failure: current main sends ffmpeg a staged `.ogg.part` output without `-f ogg`, matching the linked logs. The Discord fake-IP upload blockage is source-provable from the missing policy but was not live-tested here.

## Solution Assessment

No for the PR as written: the Feishu muxer fix is the narrow path, but the Discord SSRF policy should use a host-scoped fake-IP exemption rather than an unscoped special-range allow policy.
