## Summary

Current main already covers the reported stale macOS LaunchAgent entrypoint after package/non-git updates: package updates run the updated install's doctor in update-repair mode, doctor detects entrypoint drift, and update repair stages a rewritten service plist even when the gateway is not currently loaded. Loaded services still get the force install refresh before restart.

## Evidence
