## Summary

Close as duplicate/superseded: the report matches the active-requester wake-failure dead-end already tracked by open PR https://github.com/openclaw/openclaw/pull/79059, while the broader durable subagent delivery problem remains tracked at https://github.com/openclaw/openclaw/issues/44925.

## What This Changes

Subagent completions spawned via `sessions_spawn` and followed by `sessions_yield` should reach the requester instead of failing after active-run wake retries with `active requester session could not be woken`.

## Best Possible Solution

Consolidate the fix through the existing active-requester fallthrough or queue-first completion PRs, preserving requester-agent mediation and linking this v2026.5.12 evidence there.

## Reproduction Assessment

Yes. A source-level repro is high-confidence: set `expectsCompletionMessage`, resolve an active requester session, make `queueEmbeddedPiMessageWithOutcome` return `not_streaming` or `no_active_run`, and current main returns before `runAnnounceAgentCall`; the focused test asserts that `callGateway` is not called.

## Solution Assessment

Yes for the requested fix direction, but not as a separate tracker. Falling through to the requester-agent handoff on `no_active_run` or `not_streaming` aligns with the docs and the open narrow PR, while maintainers should consolidate it with the protected queue-first completion path.
