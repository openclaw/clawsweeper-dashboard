## What This Changes

The issue reports that interrupted writes to `auth.json` can corrupt the credential store and lock out all stored providers.

## Best Possible Solution

Land a focused fix that routes both auth-storage persistence sites through atomic same-directory replacement with `0o600` permissions and regression coverage for interrupted writes.

## Solution Assessment

Yes. Reusing the existing atomic replacement helper for both credential write sites is the narrowest maintainable fix direction, with review focused on permissions, temp-file cleanup, and regression proof.
