## Best Possible Solution

Define one maintainer-approved sandbox contract for Codex with OpenClaw Docker sandboxing: run Codex native execution inside the active per-agent sandbox, or fail/warn/diagnose clearly when that boundary cannot be honored.

No, the requested config opt-out alone is not clearly the best fix because the managed app-server still runs on the Gateway host. The safer path is a maintainer-approved boundary decision, likely app-server/native execution inside the OpenClaw sandbox or an explicit fail/warn diagnostic path.
