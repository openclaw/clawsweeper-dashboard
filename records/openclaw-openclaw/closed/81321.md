## Summary

Close: this is a useful Honcho plugin capability request, but current OpenClaw core already exposes and documents the generic `publicArtifacts.listArtifacts` seam that `memory-wiki` consumes. The missing implementation belongs in the external Honcho plugin and its ClawHub/plugin distribution path, not in OpenClaw core.

## What This Changes

Request to make the external `openclaw-honcho` memory plugin export `publicArtifacts.listArtifacts` so `memory-wiki` bridge mode can import Honcho memory artifacts.

## Best Possible Solution

Implement and publish `publicArtifacts.listArtifacts` in the external Honcho plugin, then surface it through ClawHub while leaving OpenClaw core’s generic SDK seam unchanged.

## Reproduction Assessment

Not applicable as an OpenClaw core bug: this repository does not contain the Honcho plugin runtime to reproduce or fix. The reported zero-artifact behavior is consistent with an active memory plugin that has not registered `publicArtifacts.listArtifacts`.

## Solution Assessment

No, changing OpenClaw core is not the best way to solve this. The maintainable path is for the external Honcho memory plugin to implement the existing memory capability API and publish that update through its plugin distribution path.
