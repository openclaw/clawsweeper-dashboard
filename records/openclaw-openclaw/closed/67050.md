## Summary

Keep open. Current main still blocks env-plus-network scanner findings as critical and still scans test-named paths, so the linked false positive is not fixed. This PR remains a plausible fix path for #66840, but the proposed blanket test-path skip is security-sensitive and would let dangerous runtime-capable code evade all install-time scanner rules by filename or directory name.

## Best Possible Solution

Keep this PR open, but require a safer design before merge: keep scanning runtime-capable files, relax only the blocking decision or severity for credible test-file false positives, and add regression tests covering both the reported ClawHub archive false positive and a malicious test-named file that must not bypass the scanner.
