## Summary

Current `main` now turns the reported CLI no-output stall into a timeout-class `FailoverError`, and the fallback machinery explicitly treats related transport failures as failover-worthy and advances to the next configured model. The repo also has regression coverage for `terminated` / `Request failed` transport failures tied to the related consolidated report #69368.
