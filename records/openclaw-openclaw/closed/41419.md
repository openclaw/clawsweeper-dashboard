## Summary

Current main still has the Discord MIME/WebP bug this PR is trying to fix: normal sends, component sends, and native interaction replies all still create upload Blobs without preserving the loaded media content type, and shared media loading still optimizes non-GIF images including WebP. The PR is not safely closeable as implemented or duplicate because #41256 and overlapping PRs #41265 and #41714 remain open with no merged canonical fix, and this PR still needs maintainer review for the native Discord path plus shared WebP validation/privacy behavior.

## Best Possible Solution

Keep this PR open for maintainer review or supersede it with a single canonical fix for #41256. The best implementation should preserve verified `media.contentType` in every Discord upload Blob path, including standard sends, component sends, and native interaction replies; decide whether WebP and animated-image preservation belongs in shared media loading or Discord-specific send policy; add regression tests for MIME preservation and frame/format preservation; resolve WebP metadata stripping and fail-closed unknown-dimension validation before bypassing optimization; and update WhatsApp tests/docs if shared media behavior intentionally changes.

## Risks / Open Questions

- The PR does not cover `extensions/discord/src/monitor/native-command.ts`, leaving one Discord reply upload path with typeless Blobs.
- The shared WebP optimization bypass changes WhatsApp-observed behavior while current tests and docs still expect image recompression/optimization.
- WebP passthrough may retain EXIF/XMP/ICC metadata that re-encoding would usually strip.
- The proposed passthrough relies on image dimension validation, but current non-sips behavior allows unknown dimensions when metadata parsing fails.
- There are overlapping open implementations (#41265 and #41714), so maintainers should pick or combine a canonical fix rather than merge or close this PR in isolation.
- The PR contains unrelated formatting-only churn in `src/plugins/jiti-loader-cache.test.ts`, which should be dropped from a focused Discord/media fix.
