## Summary

This request is coherent, but it fits OpenClaw's plugin/channel-extension layer rather than core. Current main already exposes plugin hooks and channel reply callbacks for tool/progress/approval/command-output events, while channel plugins own outbound delivery and threading UX. Core also already ships only coarse message-based tool-progress previews on some channels, which reinforces that richer per-channel execution visibility should be implemented as a plugin or channel extension and published through ClawHub.
