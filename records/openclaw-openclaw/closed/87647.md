## What This Changes

The branch moves npm-sourced plugin installs from one shared managed npm root into per-package projects under `npm/projects/<hash>`, and updates recovery, doctor repair, uninstall, peer relinking, Codex lookup, docs, tests, and Crabbox/Testbox config for that layout.

## Best Possible Solution

Merge only after maintainers accept the per-plugin npm project layout and either accept the fixture plus Crabbox proof or request one live public registry install/update/uninstall smoke.

## Reproduction Assessment

Yes. Source inspection shows current main installs all npm plugins through one managed npm root and runs npm from that cwd; a two-plugin fixture with conflicting dependency graphs is the high-confidence reproduction path, though I did not run a live registry repro.

## Solution Assessment

Yes, with maintainer acceptance. Isolating npm package-manager cwd per plugin is a cleaner fix than adding more shared-root repair heuristics, and the PR preserves legacy flat-root recovery paths for shipped installs.
