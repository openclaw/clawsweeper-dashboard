## Summary

Keep open. Current main still rejects Xiaomi per-call TTS `voice`/`model` overrides through the CLI/Gateway path, and the related VoiceDesign issue and Token Plan PR cover different Xiaomi TTS work.

## What This Changes

Request standard Xiaomi TTS CLI/Gateway per-call `voice` and `model` overrides to map into the provider's existing request-scoped Xiaomi model and `audio.voice` handling.

## Best Possible Solution

Add a Xiaomi-plugin-local `resolveTalkOverrides` implementation that maps `voiceId` to Xiaomi `voice` and `modelId` to Xiaomi `model`, with regression coverage for CLI/Gateway override resolution and Xiaomi request payloads.

## Reproduction Assessment

Yes, at source level. CLI and Gateway `voiceId`/`modelId` inputs enter `resolveExplicitTtsOverrides`, which throws before synthesis because the selected Xiaomi provider lacks `resolveTalkOverrides`.

## Solution Assessment

Yes. The narrow maintainable fix is provider-local resolver plumbing in `extensions/xiaomi`; VoiceDesign support and Token Plan endpoint/auth changes should stay with https://github.com/openclaw/openclaw/issues/72927 and https://github.com/openclaw/openclaw/pull/79185.
