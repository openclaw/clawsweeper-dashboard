## Summary

The request is coherent and useful, but current main already positions context assembly, retrieval filtering, and transcript/result transformation as plugin-owned extension points rather than new OpenClaw core API surface. This fits a ClawHub/community plugin or context-engine plugin, not core cleanup.

## Close Comment

Closing as `clawhub` after Codex review. Current `main` treats this as plugin territory, not new core API surface: `VISION.md:54-59` keeps optional capability out of core, `docs/concepts/context-engine.md:10-16` and `docs/concepts/context-engine.md:67-96` give context-engine plugins control over which messages are included in model context, and `docs/concepts/agent-loop.md:93-102` plus `src/plugins/hooks.ts:530-542` / `src/plugins/hooks.ts:828-848` already expose prompt-build and transcript-persist hooks for exclusion/redaction behavior. `src/plugin-sdk/index.ts:116-120` and `src/context-engine/types.ts:150-157` also expose the SDK/runtime seam needed for a plugin-owned implementation. Given that scope, this belongs as a ClawHub/community plugin or context-engine plugin rather than an OpenClaw core feature request.
