## Summary

Keep open. Current main does not implement the stale menubar cleanup, and this branch is a focused macOS bug-fix attempt, but it is not merge-ready because the watchdog can hide quiet active runs and real macOS behavior proof is missing.

## What This Changes

The PR adds a macOS `WorkActivityStore` watchdog and tests intended to clear stale menubar activity when provider overloads or timeouts leave no terminal run event.

## Best Possible Solution

Use lifecycle, disconnect, error, or explicit run-liveness ownership to clear orphaned UI state while keeping active quiet work visible, then back it with focused tests and redacted macOS proof.

## Reproduction Assessment

Yes for the PR regression at source level: after a start event, any job or tool with no timestamp refresh for more than 180 seconds is removed even if the underlying work has not ended. I did not run a live macOS provider-overload scenario in this read-only review.

## Solution Assessment

No. A watchdog is a plausible mitigation, but the best fix should distinguish orphaned UI state from quiet active work instead of using unconditional age-based eviction.
