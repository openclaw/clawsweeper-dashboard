## Summary

Keep open: current main already has channel-plugin contracts, lifecycle commands, and multiple official external channel packages, but Telegram and Signal are still bundled and the remaining official-channel distribution policy is unresolved.

## What This Changes

The issue proposes separately versioned, independently installable and updatable channel plugins while keeping OpenClaw core focused on shared runtime infrastructure.

## Best Possible Solution

Define the official-channel distribution policy, then split any remaining work into per-channel externalization tasks with SDK, release, rollback, docs, onboarding, and repair proof.

## Reproduction Assessment

Not applicable: this is an architecture and packaging proposal, not a broken existing behavior report. Source and docs inspection shows partial support plus a remaining distribution-policy gap rather than a failing current-main path.

## Solution Assessment

Unclear as a direct implementation path: the direction matches the plugin architecture, but the maintainable solution is a maintainer-owned distribution decision followed by narrower per-channel work.
