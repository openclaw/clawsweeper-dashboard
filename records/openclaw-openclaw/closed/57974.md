## Summary

Current `main` no longer does timer-based Control UI `sessions.list` polling. The UI subscribes once to `sessions.changed` and refreshes on push events, so the reported per-tab 5-7s polling saturation from `v2026.3.28` is obsolete on current code.

## Evidence

## Risks / Open Questions

- `sessions.list` on current main still builds and sorts the full session row set before applying `limit` (`src/gateway/session-utils.ts:1483` and `:1556`), so very large manual refreshes may still be expensive. That is a different scalability problem than the reported timer-based per-tab polling saturation.
