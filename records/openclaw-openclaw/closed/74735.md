## Summary

Keep open: this PR adds a useful session-scoped bootstrap-extra-files contract that current main does not have, but the current PR head still introduces a literal bracket-path regression and misleading short docs that should be repaired before merge.

## What This Changes

The PR adds exact `bootstrap-extra-files.sessions[sessionKey]` path lists, scoped arbitrary-basename loading through workspace guards, deterministic ordering/dedupe, docs/changelog updates, and focused tests.

## Best Possible Solution

Repair literal bracket path compatibility, correct the short hook docs, then let maintainers review whether to land exact-session hook config while broader arbitrary/global/session-type policy remains in the related threads.

## Reproduction Assessment

Not applicable to the requested feature itself. The introduced PR regression is high-confidence reproducible with a scratch workspace containing `docs/[draft]/TOOLS.md`, where `fs.promises.glob("docs/[draft]/TOOLS.md")` returns no matches and the PR code does not fall back to the literal path.

## Solution Assessment

No, not until the PR preserves backward-compatible literal path handling and fixes the misleading summaries. After that narrow repair, the exact-session approach is a maintainable solution if maintainers accept this hook-owned config surface.

  - body: The PR now treats any `[` as a glob marker.   - confidence: 0.88

Readers can configure a global `PROJECT.md` that is skipped, so use a recognized bootstrap filename or mention the session-specific requirement here.
  - confidence: 0.78
