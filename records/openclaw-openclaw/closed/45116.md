## Summary

Current main already routes Telegram inbound media downloads through the same proxy-aware transport used for Bot API calls, and the changelog shows this shipped in 2026.3.13 with later explicit-proxy hardening in 2026.4.14.

## Evidence

- **Release note matches reported regression:** The 2026.3.13 changelog explicitly says Telegram media downloads were fixed by threading the same direct/proxy transport policy into SSRF-guarded file fetches, which matches this report's 2026.3.12 proxy-backed media-download failure.
  - file: [CHANGELOG.md:2542](https://github.com/openclaw/openclaw/blob/40be5ad58187fd20b8defe5fc30e649190c131a8/CHANGELOG.md#L2542)
