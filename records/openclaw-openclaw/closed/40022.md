## Summary

Current `main` already covers this Linux fresh-install regression: the systemd detail parser concatenates `stderr` and `stdout`, `isSystemdServiceEnabled` short-circuits to `false` when the managed unit file is missing, regression tests cover both missing-unit and `code 4 / not-found` cases, and `CHANGELOG.md` shows the fixes shipped in `v2026.3.7`.

## Evidence

- **Current parser keeps both streams:** `readSystemctlDetail` now returns ```${result.stderr} ${result.stdout}``` with an inline comment explaining that stdout must remain visible for `not-found` matching even when `execFileUtf8` fills stderr from the Node error fallback.
  - file: [src/daemon/systemd.ts:272](https://github.com/openclaw/openclaw/blob/bfa6708c03f24c7caa5d0a96a298c292be0d85c1/src/daemon/systemd.ts#L272)
