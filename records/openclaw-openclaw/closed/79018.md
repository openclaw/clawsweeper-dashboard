## Summary

Keep open. Current main and the shipped v2026.5.6 Compose file still put `openclaw-cli` in the gateway network namespace with dropped network capabilities, and the docs continue to direct Docker users to run post-start CLI commands there; the reported failure is not implemented, duplicated, or stale enough to close.

## What This Changes

The issue asks OpenClaw to make documented Docker CLI-container plugin installs work despite the current shared-network and capability-drop hardening, or to document a supported workaround.

## Best Possible Solution

Keep Docker plugin installs working from the documented CLI path without broadening container privileges if possible; if the hardening must stay, document the gateway-container install workaround in the Docker and upgrade/plugin docs.
