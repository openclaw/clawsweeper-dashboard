## Summary

Keep open. The PR is a plausible attempt at the linked status reporting bug, but it does not fix the current `status --json` path and it adds debug stdout output that would corrupt command output if the summary path runs.

## What This Changes

The PR applies plugin auto-enable during CLI plugin registry loading and adds a channel-summary plugin list debug print.

## Best Possible Solution

Decide the `status --json` channel-summary contract, then fix the actual fast JSON and read-only summary resolver path with focused regression coverage and no debug stdout output.

## Reproduction Assessment

Yes. Source inspection shows the current `status --json` fast path disables channel data and passes `includeChannelSummary: false`, while the summary resolver still excludes setup-fallback plugins when reached.

## Solution Assessment

No. Applying auto-enable in the old registry loader path is not the narrow current-main fix for the reported command, and the debug stdout write must be removed.
