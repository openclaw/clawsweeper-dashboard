## Summary

Current main already fixes the central #43850 failure mode by giving isolated cron executions a run-scoped session key and using that key for subagent follow-up, so stale descendants from previous isolated runs are outside the current wait tree. This PR is obsolete and also carries an unrelated gateway restart script that should not be merged as part of the cron fix.

## Best Possible Solution

Close this PR as obsolete. Keep current main's run-scoped isolated cron session implementation, and if the stale-wait behavior regresses, add focused tests around `runSessionKey` descendant registry behavior rather than reviving this branch. The unrelated restart script should be left out of this cron fix path.
