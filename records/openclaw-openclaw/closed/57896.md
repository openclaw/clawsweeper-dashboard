## Summary

Current `main` no longer matches the reported stale-exec-config-after-restart path: SIGUSR1 restart clears the active runtime snapshot, startup reactivates secrets/config, and chat runs rebuild tools from the current config rather than reusing a session-baked exec tool.

## Evidence

- **Restart shutdown clears runtime snapshot:** `startGatewayServer` passes `clearSecretsRuntimeSnapshot` into the gateway close prelude, and `runGatewayClosePrelude` invokes it during shutdown/restart.
  - file: [src/gateway/server.impl.ts:564](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/gateway/server.impl.ts#L564)

- **SIGUSR1 restart re-enters fresh startup:** The gateway run loop handles authorized `SIGUSR1` by closing the server, then starting a new gateway iteration instead of continuing on the old runtime state.
  - file: [src/cli/gateway-cli/run-loop.ts:140](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/cli/gateway-cli/run-loop.ts#L140)

  - file: [src/gateway/server-startup-config.ts:262](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/gateway/server-startup-config.ts#L262)
