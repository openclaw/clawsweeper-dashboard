## Summary

Current main at 68561a8c948b49e85ae012646de550a49d6c63a still appears to have the reported restart/resume gap. Exec approval state is process-local, stale approval IDs still fail as unknown/expired, and the restart/session replay paths I inspected do not invalidate or re-register persisted approval-pending exec tool results.

## Best Possible Solution

Keep this issue open. The fix should live in restart/session replay recovery or in an explicit approval-manager persistence contract: when resumed history contains an exec approval-pending result whose approval ID is absent from the live manager, convert it to a clear expired/error result so the agent reissues exec, or securely persist and re-register pending approvals with the original bound execution metadata. Add a regression test for restart with an approval-pending transcript tail.
