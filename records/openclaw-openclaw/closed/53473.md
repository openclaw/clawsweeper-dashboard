## Summary

Keep open. Current main still lacks the useful doctor/FAQ clarification, but this PR should not merge as-is because its auto-mode local-provider guidance says an `hf:` model path works with `provider = "auto"`, while runtime auto-selection and the reference docs require an existing local file.

## What This Changes

The PR updates `openclaw doctor` memory-search guidance and FAQ text for local GGUF/node-llama-cpp embeddings, adds regression assertions, and includes a formatting-only subagent workspace test change.

## Best Possible Solution

Land a narrow version that aligns doctor guidance, helper behavior, tests, and FAQ text with the memory embedding contract: explicit `provider = "local"` may use the default downloadable GGUF or `hf:` refs, `provider = "auto"` should only present local as ready for an existing local file, Ollama remains explicit, and pnpm build-approval guidance stays scoped to node-llama-cpp.
