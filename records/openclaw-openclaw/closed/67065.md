## Summary

Current main already supports the requested next-turn ownership path: normal user replies run `before_agent_reply` after inline actions but before media staging, active-memory `before_prompt_build`, and the main LLM path. A workflow can persist pending state keyed by `ctx.sessionKey`, claim the clarifier follow-up with `{ handled: true }`, and return either a synthetic reply or silence. The current behavior is covered by focused hook-runner and get-reply wiring tests, and the same wiring is present in the v2026.4.24 release tag.

## Best Possible Solution

Close as implemented. Keep the existing `before_agent_reply` first-claim-wins contract as the core primitive: media workflows should store their pending state by `ctx.sessionKey` and claim the immediate follow-up with `{ handled: true }` before active-memory or the main model run. A separate docs/example follow-up could improve ergonomics, but the core routing/suppression capability is already present.
