## Solution Assessment

No for this PR head: the boundary is right, but the implementation is incomplete because disk-only/local-update save inputs can still overwrite the active snapshot without runtime-only external OAuth profiles. The safer fix is to preserve those profiles from the previous runtime snapshot and prove that path.
