## Summary

Close as duplicate_or_superseded. PR #49954 is an older implementation of the Anthropic native web-search request in #49949; it bridges the feature by making a separate Anthropic Messages API call, while the newer open PR #50692 tracks the same request using the native payload-injection direction that #49954 itself called out as future work. Current main still does not ship Anthropic as a web_search provider, so the remaining work should continue in #49949/#50692 rather than keeping two overlapping PRs open.

## Best Possible Solution

Close this older PR and keep the remaining feature work in the canonical #49949/#50692 thread. The best implementation path is to rebase onto current main's manifest contract shape, declare Anthropic webSearchProviders in the Anthropic plugin, register the web-search provider, and compose native Anthropic server-tool injection through the existing provider wrapStreamFn path with credential/config/docs/tests updated together.
