## Summary

Keep open: the backfill idea is useful, but this patch is not merge-ready because unrelated label events can produce a successful skipped Dependency Guard check and cancel a real in-progress guard run.

## What This Changes

The PR adds a `pull_request_target` `labeled` trigger to Dependency Guard and tests that only `dependency-guard-backfill` labels are intended to run the guard.

## Best Possible Solution

Use a backfill path that cannot satisfy or replace the required Dependency Guard check on unrelated labels, such as a separate non-required backfill workflow/status or a design where labeled runs execute the real guard without skipped-success bypass.

## Reproduction Assessment

Yes. Source review plus GitHub Actions documentation establishes the bypass path: a non-backfill label starts the workflow, the job-level condition skips it successfully, and same-group concurrency can cancel a real guard run.

## Solution Assessment

No. The proposed job-level label gate is not the safest way to solve the backfill problem because skipped required jobs count as success; the backfill mechanism needs a design that cannot replace the real guard check.
