## Summary

Keep open: current main still has the source-reproducible false positive, and the focused closing PR at https://github.com/openclaw/openclaw/pull/81778 is still open, so closing before merge would be premature.

## What This Changes

The issue reports that `gateway status` warns a macOS LaunchAgent command lacks the `gateway` subcommand even when a shell-wrapped command contains `gateway --port`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81778 or an equivalent narrow service-audit tokenization fix, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes. Current main reads LaunchAgent `ProgramArguments` into the service audit and only accepts an argv element exactly equal to `gateway`, so the reported `/bin/zsh -lc ... gateway --port ...` wrapper is source-reproducible as a false `gateway-command-missing` warning.

## Solution Assessment

Yes. A narrow service-audit command-token normalization fix with regression tests for the shell wrapper and near matches is the maintainable path; the active linked PR is already taking that approach.
