## Summary

Close as ClawHub/plugin work. Current main still documents the default memory flush as pre-compaction, but the repo already exposes the needed post-turn/session lifecycle seams and a bundled memory plugin demonstrates automatic post-turn capture. There is no concrete missing core API in this report.

## Best Possible Solution

Keep OpenClaw core's compaction semantics as-is and move short-session memory capture to a memory plugin or ClawHub plugin that uses agent_end, session_end, heartbeat_prompt_contribution, or context-engine afterTurn. Reopen core only if plugin authors find a concrete missing API or maintainers explicitly want a new default memory-core policy.
