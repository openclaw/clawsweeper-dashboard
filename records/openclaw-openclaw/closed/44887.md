## Summary

PR #44887 is superseded by merged PR #71506 and current main. The old `src/memory/remote-http.ts` target no longer exists; current memory-host remote embedding/batch HTTP uses `TRUSTED_ENV_PROXY` only when `HTTP_PROXY`/`HTTPS_PROXY` will actually proxy the target, with tests and an unreleased changelog entry.

## Best Possible Solution

Close #44887 as superseded by merged #71506 and keep the current main implementation: memory-host remote embedding and batch HTTP should use the shared URL-aware env-proxy gate rather than reviving the stale `src/memory/remote-http.ts` patch.
