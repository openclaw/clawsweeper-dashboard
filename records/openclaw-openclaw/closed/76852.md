## Summary

Keep open: current main still sends Telegram health getMe probes through a cached fetch closure that cannot call the Telegram transport fallback hook, and the focused closing PR remains open and unmerged.

## What This Changes

The issue reports a regression where periodic Telegram getMe health probes time out every roughly 60 seconds on networks with IPv4 egress but broken IPv6 egress.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/76856, or an equivalent Telegram-owned probe transport fix, then close this issue after merge and release verification.

## Reproduction Assessment

Yes. The reporter provides concrete broken-IPv6 logs, and current main still routes the periodic Telegram getMe health probe through a cached fetch closure that cannot promote the Telegram transport fallback.

## Solution Assessment

Yes. The narrow maintainable fix is to keep fallback state in TelegramTransport and let probe retries promote the same cached transport; this issue should remain open until the linked implementation lands or is superseded.
