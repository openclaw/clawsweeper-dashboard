## Summary

Keep open. Current target main `cf834e2a` still lacks the PR's long-quiet typing lease or an equivalent active-run presence mechanism: typing TTL still expires from a watchdog, tool events only refresh on start/update, compaction handling does not lease typing, preflight/memory flush paths do not receive the typing controller, and explicit timeout/overflow compaction recovery does not emit compaction lifecycle events. The linked #65919 remains the active user problem, and the existing Greptile comment on PR head `7f526d20` is positive supporting evidence rather than a reason to close.

## Best Possible Solution

Keep PR #66365 open and update/review it against current main. The best path is to land an internal active-run typing/presence lease, or an equivalent implementation, covering preflight compaction, memory flush, in-run compaction, and long tool execution, with regression tests for TTL cleanup, nested leases, error release, and explicit timeout/overflow compaction lifecycle events; then close or update #65919 from that shipped fix.
