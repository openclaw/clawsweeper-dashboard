## What This Changes

The issue reports that Control UI root-level public assets such as favicons, apple-touch icons, and the web manifest 404 when requested under the `/__openclaw__/` SPA prefix.

## Best Possible Solution

Keep the canonical tracker https://github.com/openclaw/openclaw/issues/55600 open and fix the shared Control UI asset URL/routing behavior there with regression coverage for prefixed and deep SPA routes.

## Reproduction Assessment

Yes. Source inspection on current main shows `/__openclaw__/favicon.svg` resolves to a missing nested file and then hits the known static-asset 404 branch, while nested `/assets/` paths are specially remapped.
