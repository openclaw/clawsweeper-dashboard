## Risks / Open Questions

- Merging the current PR head as-is would still leave broken Dockerfile examples because the new `latest/download/gogcli_linux_amd64.tar.gz` and `latest/download/goplaces_linux_amd64.tar.gz` URLs return 404 against the current releases.
- The PR now changes the Linux `wacli` URL even though the current `wacli` release only publishes a macOS universal tarball; maintainers should decide whether to wait for the linked upstream Linux artifact or omit `wacli` from the Linux example.
- The latest patch uses `curl -L` and separates `tar` from `chmod` with `;`. In a Docker `RUN`, a failed download or failed `tar` can be masked by a succeeding `chmod`; the docs should prefer `curl -fSL` plus `&&` or an equivalent fail-fast pattern.
- Security review pass: the diff is documentation-only and does not touch workflows, lockfiles, package manifests, publishing metadata, secrets, generated/vendor files, or repository code execution paths, but the documented sample itself is a supply-chain-sensitive binary download path and should be accurate and fail closed.
