## Summary

Keep open. Current main still has the source-level foreground/background lifecycle gap for `openclaw agent --json` with spawned descendants, and the remaining repair is a CLI/subagent completion-contract decision rather than a stale, duplicate, or already-implemented item.

## What This Changes

The issue asks for `openclaw agent --json` to avoid returning misleading partial output when a parent turn spawns sub-agents that continue asynchronously.

## Best Possible Solution

Define a single foreground CLI contract for spawned descendants, then update the CLI/Gateway lifecycle code, JSON metadata, docs, and focused regression tests to match that contract.

## Reproduction Assessment

Yes, at source level: `openclaw agent --json` waits for the parent Gateway `agent` RPC result while `sessions_spawn` child completion is registered and waited on asynchronously. I did not run the reporter's Docker/SRE-MAS setup live in this read-only review.

## Solution Assessment

Unclear. The retracted last-turn and `replayInvalid` fixes are not the narrow fix; maintainers should choose whether the CLI drains descendants, exposes explicit partial-output/wait semantics, or relies on a separate blocking delegate primitive.
