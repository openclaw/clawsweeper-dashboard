## Summary

Current main and the latest shipped release already include eligible `skills.load.extraDirs` skills in the model-facing skills prompt, with per-skill `disable-model-invocation` as the documented opt-out.

## What This Changes

The issue requests an opt-in way for `skills.load.extraDirs` skills to appear in the agent `<available_skills>` prompt block so shared machine-level skills can be auto-selected.

## Best Possible Solution

Keep the shipped behavior: eligible extra-dir skills are included in the prompt by default, and individual skills can opt out with `disable-model-invocation`.

## Reproduction Assessment

No: the old 2026.4.23 behavior does not reproduce from current-main source inspection. `skills.load.extraDirs` entries flow into the skills snapshot prompt, and the shipped regression test asserts those names appear in `snapshot.prompt`.

## Solution Assessment

Yes: the current implementation solves the observable shared-skill discovery problem without adding another config knob. The existing per-skill `disable-model-invocation` field remains the narrower opt-out for prompt injection.
