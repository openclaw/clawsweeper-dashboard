## Summary

Close as superseded by the existing canonical optional-provider-models work. PR #71636 identifies a real Ollama remote autodiscovery/config-schema bug linked to #70982, but current main still has the bug and this PR only changes the Zod source. The broader open PR #39766 already tracks the same remaining `models.providers.<id>.models` optionality gap with the needed contract/test direction, while Greptile correctly flagged this PR as incomplete because generated schema and exported types still diverge.

## Best Possible Solution

Close #71636 as a duplicate/superseded one-line PR, keep #39766 and linked bug #70982 as the canonical path, and land a complete config-contract fix that updates the Zod schema, generated `schema.base.generated.ts`, exported config type/default behavior, and Ollama remote-baseUrl regression coverage for omitted `models`.
