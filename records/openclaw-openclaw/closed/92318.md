## What This Changes

The PR adds an explicit-target-evidence requirement for cron announce delivery so targetless message-tool sends no longer suppress direct fallback delivery.

## Best Possible Solution

Land this narrow fix after maintainer acceptance of the message-delivery risk, keeping legacy implicit target synthesis for non-cron message-tool-owned flows.

## Reproduction Assessment

I did not run a live cron job in this read-only review, but the PR includes live Telegram proof for the same shape.

## Solution Assessment

Yes. This is the best narrow fix I found: put the stricter evidence requirement on cron announce source-delivery accounting, instead of disabling tools globally or changing legacy message-tool-owned flows that still rely on implicit plan targets.
