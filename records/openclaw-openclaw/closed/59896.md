## Summary

Close as `clawhub`: current `main` already provides lifecycle hooks, transcript metadata, and startup-memory loading that let this behavior live as a plugin or hook pack instead of adding more OpenClaw core session-state machinery.

## Evidence

- **Vision keeps optional memory/continuity features out of core:** Project direction says optional capability should usually ship as plugins, and that plugin promotion belongs in ClawHub rather than OpenClaw core.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/VISION.md#L54)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Plugin hooks already cover the needed lifecycle surfaces:** Plugin hooks are documented for session lifecycle, compaction, message flow, and agent-end observation, which matches the requested trigger points.
  - file: [docs/plugins/hooks.md:10](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/docs/plugins/hooks.md#L10)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Session-end hook payload already carries machine-usable state:** `session_end` includes `reason`, `sessionFile`, transcript-archived state, and next-session metadata. Reasons include `idle`, `reset`, `new`, `daily`, `compaction`, and `deleted`.
  - file: [src/plugins/hook-types.ts:401](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/plugins/hook-types.ts#L401)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Current main emits lifecycle hooks on rollover and compaction:** Reply-session rollover resolves idle/daily reasons and fires `session_end`/`session_start`; compaction also emits `session_end` with reason `compaction` and the stable transcript path.
  - file: [src/auto-reply/reply/session.ts:121](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/auto-reply/reply/session.ts#L121)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **There is already a bundled continuity workaround:** The bundled `session-memory` hook saves prior-session context to `memory/YYYY-MM-DD-slug.md` on `/new` and `/reset`, which is a direct existing workaround in this problem space.
  - file: [src/hooks/bundled/session-memory/HOOK.md:17](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/hooks/bundled/session-memory/HOOK.md#L17)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **New sessions already have a load path for saved memory:** Runtime startup context can prepend recent daily memory on bare `/new` and `/reset`, so loading saved continuity data is already on an extension-friendly path.
  - file: [src/auto-reply/reply/startup-context.ts:309](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/auto-reply/reply/startup-context.ts#L309)
