## Summary

Current main already addresses the reported large Project Context startup crash: MEMORY.md is still part of workspace bootstrap context, but injected bootstrap content is capped per file and in total, truncation is surfaced to the agent and /context, and prompt pressure now has a pre-submission recovery path. Session-store rotation also copies the live store before atomic rewrite and keeps recent backups.

## Best Possible Solution

Close this issue and keep the current main implementation: bounded bootstrap injection, visible truncation diagnostics, pre-submission overflow recovery, and conservative session-store backup rotation. Any remaining desire for different defaults or additional circuit-breaker UI should be tracked as a narrower follow-up rather than keeping this crash report open.
