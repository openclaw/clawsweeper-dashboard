## Summary

This issue should close as a duplicate of the open Gateway CPU/liveness tracker #72338. The reported 2026.4.25/2026.4.26 one-core Gateway spin, loopback/TUI WebSocket timeouts, and delayed Telegram/WhatsApp replies match #72338's current-release Linux Gateway CPU spin/status-timeout thread, including a later Node 22 Linux reproduction with WebSocket/status hangs and a concrete subagent-registry hot path. Current main still has that disk-backed subagent snapshot path, so this is not implemented; the best cleanup is to centralize the new Ubuntu/x86_64 details on #72338.

## Best Possible Solution

Close this as a duplicate and keep #72338 as the canonical current Gateway CPU/liveness tracker. The likely maintainer path is to fix the disk-backed subagent registry/status hot path or any newly proven equivalent loop, add focused performance/regression coverage, and have affected users add sanitized logs, profiles, or strace output to #72338.
