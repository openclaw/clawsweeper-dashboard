## Summary

Current `main` has partial scroll guards, but it does not clearly implement the requested behavior well enough to close. The Control UI still uses a 450px "near bottom" heuristic and no separate manual-scroll lock during streaming, so the report remains plausible on `main` and on the latest release snapshot.

## Evidence
