## Summary

Current main already supports same-provider multi-account routing through auth profiles: multiple profiles can coexist for one provider, sessions can pin a specific profile with `/model ...@<profileId>`, and runtime failover rotates auth profiles within a provider before moving to model fallbacks.

## Evidence

- **OAuth docs: multiple profiles per provider:** The OAuth docs explicitly document multiple profiles for the same provider in one agent, with global selection via `auth.order` and per-session selection via `/model ...@<profileId>`.
