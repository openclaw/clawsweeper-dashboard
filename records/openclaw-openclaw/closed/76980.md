## What This Changes

The issue requests that Nextcloud Talk direct-delivery replies never disappear silently when an agent emits visible text without using the `message` tool.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/77020 for the diagnostic gap, while leaving behavior-changing automatic fallback policy to the cross-channel tracker.

## Reproduction Assessment

Yes, at source level. Nextcloud Talk inbound enters shared dispatch, group/channel turns default to `message_tool_only`, automatic source delivery is suppressed, and dispatch can still record `completed` after generated visible output is not posted.

## Solution Assessment

Yes for the narrow bug: a shared suppressed-delivery diagnostic is the maintainable first fix because it preserves the current tool-only delivery contract. Automatic fallback or auto-promotion should remain a separate product-policy decision.
