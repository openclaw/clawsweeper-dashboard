## Summary

Keep open. Current main still matches the reported Active Memory / Memory Core composition failure, and open PR #74592 already references this issue as the implementation candidate, so this issue should stay open until that PR is merged or closed.

## What This Changes

Active Memory and Memory Core should compose so lightweight embedded recall runs can access memory_search and memory_get without leaving persistent placeholder assistant turns when tool resolution fails.

## Best Possible Solution

Land #74592 or an equivalent generic fix that lets runtime toolsAllow participate in plugin-tool materialization for embedded runs, preserves the final allowlist filter, keeps Active Memory extension-owned, and prevents unavailable memory tools from leaving persistent user-visible placeholder transcript turns.

## Reproduction Assessment

Yes. Enable active-memory and memory-core, trigger Active Memory's pre-summarizer, and current main's source path shows the memory-only runtime allowlist is applied after plugin tools have not been materialized from that runtime allowlist.

## Solution Assessment

Yes. The narrowest maintainable direction is a generic plugin-tool materialization fix for runtime toolsAllow, rather than hard-coding Memory Core tool IDs or broadening all lightweight embedded bootstraps.
