## Summary

Keep open. The PR targets a real current-main bug, but latest head still drops deferred zero-interval exec-event wakes and lacks after-fix real behavior proof, so it should wait for contributor revision and proof before merge.

## What This Changes

The PR changes `src/infra/heartbeat-runner.ts` and focused heartbeat tests so targeted `exec-event` wakes can run when periodic heartbeat intervals are disabled with `heartbeat.every: "0m"`.

## Best Possible Solution

Repair or replace the narrow disabled-interval exec-event fix so only real targeted exec completions bypass disabled periodic heartbeats, deferred fallback wakes are retried or scheduled correctly, and the contributor provides redacted real behavior proof before merge.

## Reproduction Assessment

Yes for the narrowed bug by source inspection: configure `agents.defaults.heartbeat.every` to `0m`, let a notify-on-exit background exec finish, and current main requests an `exec-event` wake that both direct and scheduler paths treat as disabled. I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

No. The PR is aimed at the right current code path, but its fallback state is outside normal scheduler state, so cooldown-deferred exec-event wakes can still be lost; a safer fix must requeue or arm the fallback wake for its due time.
