## Summary

PR #58300 is superseded. Current main now has the shared visible-history projection that hides default heartbeat prompts and HEARTBEAT_OK acknowledgments from chat.history, while the remaining safer internal-prompt/source-tag work is tracked by the newer open PR #68262. The text-prefix patch in #58300 also has substantive review concerns around user-message suppression and missing tests, so it should not remain the canonical implementation path.

## Best Possible Solution

Close #58300 as superseded. Keep the current shared display-projection behavior for the already-covered default heartbeat artifact, and continue any remaining custom/internal prompt work on #68262's provenance/source-tagging path rather than merging this older text-prefix chat.ts patch.
