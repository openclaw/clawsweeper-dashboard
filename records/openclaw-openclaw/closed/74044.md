## Summary

Current main now addresses the reported Discord startup wedge path for normal text-only Discord configs: Discord voice is opt-in, the voice runtime is not loaded by default, and the `GuildVoiceStates` intent is omitted unless voice or the explicit intent override is configured. This is present on main but not yet in the latest tagged release.

## What This Changes

The issue reports that enabling the Discord channel on Raspberry Pi arm64 with OpenClaw 2026.4.26 wedges the gateway around 100% CPU while disabling Discord restores gateway health.

## Best Possible Solution

Keep the current main behavior: text-only Discord configs should not start voice support or subscribe to `GuildVoiceStates`, while existing voice users retain an explicit opt-in path and documented overrides.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the reported release path: the issue gives a concrete OpenClaw version, platform, Discord config, startup sequence, gateway probes, CPU symptom, and disabled-Discord recovery check. Current-main verification is source/test-based rather than a live Raspberry Pi run.

## Solution Assessment

Is this the best way to solve the issue? Yes for the observed default-config wedge: making Discord voice opt-in removes the unnecessary voice intent/runtime from text-only startup without changing the normal Discord text channel path. A broader gateway CPU watchdog would be less targeted and would not address the unnecessary subscription directly.
