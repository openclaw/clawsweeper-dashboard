## Summary

Keep this PR open: the patch is a narrow bug fix with no blocking code-review findings found, but it is a maintainer-authored compatibility-sensitive fallback removal that should be landed only after maintainers accept the fail-closed behavior and proof level.

## What This Changes

This PR removes the run-scoped `sessions_send` durable-parent fallback after active queue rejection and updates agent tests to expect a caller-visible error with no fallback agent run.

## Best Possible Solution

Land the narrow fail-closed path once maintainers accept the compatibility tradeoff and proof level, while preserving the transcript-commit retry and normal non-run-scoped send behavior.

## Reproduction Assessment

Yes. Current main and `v2026.6.1` source show a rejected run-scoped queue falling through to a durable-parent `agent` call, and the existing test asserted that fallback path; I did not run the live cron gateway repro.

## Solution Assessment

Yes, this appears to be the best narrow fix. Removing the fallback at `startAgentRun` stops wrong-session A2A/transcript mutation without adding a second routing policy, while docs only describe the peer A2A follow-up path.
