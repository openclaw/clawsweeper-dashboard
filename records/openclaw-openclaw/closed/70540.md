  - body: Current main uses `openclaw.plugin.json` contracts for manifest-driven tool discovery, and the manifest/docs require registered tools to match `contracts.tools`.   - confidence: 0.88

  - body: Current main keeps memory tool implementations behind lazy entrypoint wrappers so tool discovery and gateway startup do not cold-load the memory runtime.   - confidence: 0.9
- **[P2] Insert a boundary before appended raw memory:** `extensions/memory-core/src/memory/add-memory-block.ts:52`
  - body: If the daily file already has manual/separator-free content, appending only blank lines plus the new block makes the parser treat the old raw text and the new memory as one block until the new trailing separator. Insert a separator before the appended block when needed and cover that upgrade/manual-edit case.
  - confidence: 0.87

  - body: The new tool is registered, but `buildPromptSection` still only detects `memory_search` and `memory_get`, so the model receives recall guidance but no availability-gated instruction to use `memory_add` when the user asks it to remember something. Add prompt/docs/test coverage for the write path.
