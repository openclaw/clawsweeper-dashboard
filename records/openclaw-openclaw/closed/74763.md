## Summary

Current main already persists the selected native subagent model into child-session override fields before launch, regression coverage asserts those override fields are present, and the runtime reads those fields when choosing the model. The helper cited in the report still updates runtime last-used fields, but it now runs after the initial child-session override patch and merges without dropping those override fields; the same implementation is present in shipped release v2026.4.27.

## What This Changes

The issue asks for `sessions_spawn` and native subagent spawn model overrides to be stored as explicit override fields so child runs use the requested model instead of the agent default.

## Best Possible Solution

Keep the shipped native subagent model-selection behavior and regression coverage; no repository change is needed for this report unless a new reproduction shows a different spawn path that bypasses the child-session patch.
