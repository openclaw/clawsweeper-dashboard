## Summary

Keep this open: current main still has the reported `claude-cli` poisoned-resume path, and https://github.com/openclaw/openclaw/pull/81247 is an open draft PR with closing syntax for this issue. The issue should close only after that PR, or a replacement such as the overlapping https://github.com/openclaw/openclaw/pull/79386, lands.

## What This Changes

Invalidate the stored `claude-cli` resume session id after `AbortError` or non-`session_expired` `FailoverError` so the next turn does not resume a poisoned CLI session.

## Best Possible Solution

Review and land, or replace, https://github.com/openclaw/openclaw/pull/81247 with a focused fix that clears only failed reused `claude-cli` sessions while preserving the existing `session_expired` retry path.

## Reproduction Assessment

Yes. Source inspection on current main plus the reporter's live traces give a high-confidence path: reuse a stored `claude-cli` binding, hit `AbortError` or timeout `FailoverError`, and observe that current main does not clear the binding.

## Solution Assessment

Yes. Clearing the stored reused `claude-cli` binding through `clearCliSessionInStore` before rethrowing abort or non-expired failover is the narrow maintainable fix; broader watchdog or fallback changes would not address the poisoned resume id as directly.
