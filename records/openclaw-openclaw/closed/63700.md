## Summary

Keep this issue open. Current main still has no repeatable `--file` path on `openclaw infer model run`: the capability metadata and Commander registration are prompt/model/transport/json only, local execution sends only a text message to the lean completion path, and gateway execution calls `agent` without forwarding attachments. Separate `image describe` and `audio transcribe` commands exist, and the gateway agent method can accept image attachments, but that does not satisfy the requested custom-prompt multimodal `model run --file` workflow.

## Best Possible Solution

Keep this issue open and implement the core CLI capability. Add repeatable `--file` to `model.run` metadata and Commander registration, read files with MIME/size validation, forward image inputs through the local lean completion path or a supported image-content adapter, pass gateway image files as `agent.attachments`, add focused tests for local/gateway forwarding and unsupported files, and update `https://docs.openclaw.ai/cli/infer`. Treat `wav`/`mp3` support as a follow-up unless the model-run audio content contract is defined in the same change.
