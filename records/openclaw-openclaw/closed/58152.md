## Summary

Current `main` now wires the Control UI chat-header model dropdown through `sessions.patch`, covers it with focused UI regression tests, and the gateway/session resolution path honors the stored override. The fix appears to be on `main` at commit `0783090da0090982c19f7803a9daf9a2ec29538e`, but it does not appear in a tagged release yet.

## Evidence

- **Chat header dropdown now sends sessions.patch:** `switchChatModel()` is called from the chat-header model `<select>` change handler and sends `state.client.request("sessions.patch", { key: targetSessionKey, model: nextModel || null })`.
  - file: [ui/src/ui/chat/session-controls.ts:100](https://github.com/openclaw/openclaw/blob/0783090da0090982c19f7803a9daf9a2ec29538e/ui/src/ui/chat/session-controls.ts#L100)
  - sha: [0783090da009](https://github.com/openclaw/openclaw/commit/0783090da0090982c19f7803a9daf9a2ec29538e)
- **Regression test for the reported behavior:** The UI test explicitly changes the chat-header model picker and asserts a `sessions.patch` RPC with the selected model, matching the issue report’s missing transmission path.
  - file: [ui/src/ui/chat/session-controls.test.ts:139](https://github.com/openclaw/openclaw/blob/0783090da0090982c19f7803a9daf9a2ec29538e/ui/src/ui/chat/session-controls.test.ts#L139)
  - sha: [0783090da009](https://github.com/openclaw/openclaw/commit/0783090da0090982c19f7803a9daf9a2ec29538e)

  - file: [src/gateway/server-methods/sessions.ts:1284](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/gateway/server-methods/sessions.ts#L1284)
- **Session model resolution prefers stored overrides:** `resolveSessionModelRef()` prefers explicit session overrides ahead of runtime/default model fields, and the test suite covers that precedence.
