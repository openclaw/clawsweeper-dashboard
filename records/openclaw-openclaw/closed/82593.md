## Best Possible Solution

## Reproduction Assessment

Yes, at source level: current main has the detail-less `response.failed` fallback, and the PR head's new redaction helper returns original deep subtrees at `depth > 6`. ## Solution Assessment

No, not as submitted: logging a redacted observation and preserving the response ID is the right narrow direction, but returning deep original subtrees is not a safe redaction strategy. The safer fix is to drop, summarize, or placeholder deep fields before stringifying them for logs.
