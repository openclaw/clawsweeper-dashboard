## Summary

Keep this open. Current main still has the reported Mattermost docs/runtime mismatch: docs advertise canonical `channels.mattermost.streaming`, but the Mattermost strict schema, public config type, generated metadata, and inbound draft-preview runtime still do not honor preview mode selection; open PR #73252 remains the active implementation candidate, while final notification policy needs a separate maintainer choice.

## What This Changes

The issue requests Mattermost support for the documented canonical streaming config modes plus a fix for draft-preview notification and visual-noise UX.

## Best Possible Solution

Land or replace PR #73252 with Mattermost-local canonical `streaming.mode` schema/type/metadata/runtime support while preserving legacy `blockStreaming`, then make a separate product decision for final notifications or silent draft previews.

## Reproduction Assessment

Yes for the documented config mismatch: current main advertises `channels.mattermost.streaming`, while the strict Mattermost schema/type/metadata omit it and the runtime always registers draft-preview callbacks. The notification symptom has reporter evidence but was not live-reproduced here.

## Solution Assessment

Yes, the narrowest maintainable direction is Mattermost-local support for the existing channel streaming helpers with regression coverage for `off` and final delivery. A new `finalNotification` option should wait until the documented mode contract is honored.
