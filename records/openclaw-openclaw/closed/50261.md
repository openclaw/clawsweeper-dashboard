## Summary

Current `main` already contains code and tests covering both reported problems: the web-search setup flow now merges provider changes into the existing config instead of replacing it, and daemon/runtime proxy handling now forwards proxy env vars into service installs and boots Undici with env-proxy support when those vars are present.

## Evidence

- **Web-search setup now merges onto existing config:** `applySearchKey` and `applySearchProviderSelection` start from the current config object and only update `tools.web.search` plus provider-owned plugin config, instead of building a tiny replacement object.
