## Summary

The reported failure shape is credible, but it is not an OpenClaw core bug to fix in this repository. Current main already provides a generic plugin-owned target-resolution contract, and the missing behavior is in the external Weixin plugin package, which does not resolve `filehelper` or opaque WeChat targets before OpenClaw's generic resolver fails closed.

## What This Changes

The issue asks OpenClaw core to let `openclaw-weixin` outbound CLI targets such as `filehelper` or opaque WeChat IDs reach the plugin instead of failing host target resolution.

## Best Possible Solution

Keep OpenClaw core channel-agnostic and have the external Weixin plugin publish target-resolution support, including `resolveTarget` and appropriate `looksLikeId` or directory behavior for `filehelper`, `wxid_*`, `@im.wechat`, and other valid WeChat `to_user_id` forms.

## Reproduction Assessment

Yes. Current main gives a high-confidence static reproduction: `openclaw message send --dry-run` resolves `--target` before dispatch, and an `openclaw-weixin` plugin with no directory match and no `resolveTarget` returns `Unknown target` before its outbound send path can run.

## Solution Assessment

No, the requested generic core pass-through is not the best fix. The narrower maintainable solution is plugin-owned Weixin target resolution, because accepting arbitrary opaque strings in core would weaken validation across all plugin channels.
