## What This Changes

Prevent WebChat from returning to an in-progress indicator after a model reply has already completed.

## Best Possible Solution

Add a source-level reconciliation that preserves local terminal session state across a racing sessions.list refresh for the selected session while leaving genuine recovered active runs abortable, with focused controller and chat rendering regression tests.

## Reproduction Assessment

Yes for source-level reproduction: final chat events clear local run state, then a stale sessions.list row can replace the locally corrected selected-session row and make hasAbortableSessionRun true again. I did not run a live browser repro.

## Solution Assessment

No: the pasted minified bundle patch would clear any selected active row whenever there is no local chatRunId/chatStream, which conflicts with expected recovered active-run behavior. The maintainable fix should live in source and be gated on a recent local terminal reconcile or preserved terminal row.

## Repair Work Prompt

Fix the WebChat Control UI stale in-progress regression reported at https://github.com/openclaw/openclaw/issues/87875. Current main clears chatRunId/chatStream on a final chat event, but a later sessions.list refresh can overwrite sessionsResult with a stale selected-session row whose status is running or hasActiveRun is true; hasAbortableSessionRun then makes renderChat show phase in-progress again. Implement the fix in source, not the built bundle, and do not broadly clear every selected active row when there is no local chatRunId/chatStream because WebChat intentionally recovers active runs from sessions.list. Prefer preserving or reapplying the local terminal state only when the selected row was locally reconciled as terminal or a recent chatRunStatus/previous row proves the active-looking list row is stale. Add focused regression coverage for the sessions.list overwrite path plus the existing abortable recovered-run behavior. Do not edit CHANGELOG.md; include release-note context in the PR body or commit message.
