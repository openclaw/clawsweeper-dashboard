## Summary

Keep this PR open. Current main at e962381dbf1ecdb1cd4784fdeb6c2e07ddd3435b still does not expose the requested trust identity fields on PluginHookAgentContext or the main agent/compaction hook contexts. The message-hook sessionKey portion is partly already present, but the core identity-context feature remains missing; this PR is also paired with the same-author open umbrella #36671 and has plugin-author comments showing real demand.

## Best Possible Solution

Keep this PR open for maintainer review or a narrowed rebase. The best path is an additive plugin SDK/context update that exposes sourceProvider, senderId, senderName, senderIsOwner, groupId, and spawnedBy on PluginHookAgentContext; derives a normalized origin provider consistently; preserves null semantics; threads the fields through early run, attempt, queued/memory, compaction, reset, and message-related hook contexts as appropriate; updates SDK/API/docs/tests; and drops any sessionKey message-hook hunks already implemented on main.
