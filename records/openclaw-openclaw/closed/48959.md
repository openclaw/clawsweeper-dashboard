## Summary

The reported CLI gateway handshake timeout is already addressed on current `main` and was documented as shipped in release `2026.3.22`: the shared pre-auth WebSocket handshake timeout is now 10 seconds, with an `OPENCLAW_HANDSHAKE_TIMEOUT_MS` override, and the gateway connection path uses that shared timeout.

## Evidence

  - file: [CHANGELOG.md:2446](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/CHANGELOG.md#L2446)
