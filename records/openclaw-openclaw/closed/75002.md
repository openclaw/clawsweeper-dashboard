## What This Changes

The PR changes `.github/workflows/stale.yml` so assigned PR backfill candidates must also be older than the 7-day update close window, and `max_closures=0` selects zero candidates instead of disabling the cap.

## Best Possible Solution

Merge this narrow workflow fix after rerunning or clearing CI so the manual stale backfill matches the scheduled stale close grace window and treats `max_closures` as an actual cap, including zero.
