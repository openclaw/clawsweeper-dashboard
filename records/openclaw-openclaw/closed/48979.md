## What This Changes

The issue asks for Telegram to deliver images read by the `read` tool as actual outbound media attachments.

## Best Possible Solution

Use https://github.com/openclaw/openclaw/issues/41744 and https://github.com/openclaw/openclaw/pull/50359 as the canonical path, adapting the start-args fallback to current main while preserving trusted local media gates and Telegram delivery tests.

## Reproduction Assessment

## Solution Assessment

Yes. The canonical issue and open PR target the narrow shared fix: derive media from trusted read start args before the existing pending-media and Telegram delivery paths run, instead of adding Telegram-specific Bot API plumbing.
