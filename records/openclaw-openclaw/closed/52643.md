## Summary

Keep this PR open. Current main does not implement the PR's main hook-metadata feature: plugin hook events are still forwarded unchanged, there is no bot-name registry SDK subpath, and Feishu bot names remain internal monitor state. The Feishu probe has since moved to the newer pingBotInfo shape, so that part of the PR may need rebasing, but the central plugin-hook request is not obsolete.

## Best Possible Solution

Keep the PR open for maintainer review or contributor revision. The best path is an account-aware, documented hook-metadata contract that injects OpenClaw version and bot display name consistently, updates SDK entrypoints/package exports/docs/API baselines/tests, and addresses the existing review comments around multi-account Feishu, recovery, and stale registry cleanup before merge.
