## Summary

Current main already fixes the Feishu runtime-binding failure by giving bundled channel startup the full channel runtime, including `inbound.run`, and the same symptom was previously confirmed fixed on main in the canonical Feishu dispatch thread.

## What This Changes

Report that Feishu WebSocket inbound messages on v2026.5.28 are received but fail during dispatch with `Cannot read properties of undefined (reading 'run')`.

## Best Possible Solution

Keep the current full-channel-runtime startup fix and ship it in the next release rather than adding Feishu-only null checks around a missing runtime surface.

## Reproduction Assessment

Yes. Multiple stable-release reports and the v2026.5.28 source show bundled Feishu startup could receive a runtime surface without `inbound.run`; I did not run a live Feishu tenant repro in this read-only pass.

## Solution Assessment

Yes. The current fix is at the right layer because gateway startup now supplies the full channel runtime to bundled channel accounts, which repairs Feishu and related synthetic dispatch paths without a Feishu-only defensive shim.
