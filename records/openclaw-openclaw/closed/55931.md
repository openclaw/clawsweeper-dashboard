## Summary

Current main no longer routes subagent completion announces from `inputProvenance.sourceChannel`; it uses the stored requester delivery context, so the reported `webchat` reroute path is obsolete.

## Evidence

- **Spawn stores requester origin:** `sessions_spawn` registers each child run with `requesterOrigin` from the caller's actual channel/account/target/thread context.
  - file: [src/agents/tools/sessions-spawn-tool.ts:307](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/agents/tools/sessions-spawn-tool.ts#L307)
- **Announce flow resolves direct origin from requester context:** Subagent completion announce flow loads the requester session entry, resolves `directOrigin`/`completionDirectOrigin`, and passes those through to delivery; `sourceChannel` is only metadata on `inputProvenance`.
  - file: [src/agents/subagent-announce.ts:521](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/agents/subagent-announce.ts#L521)
- **Delivery uses requester origin for routing:** Direct announce delivery computes `effectiveDirectOrigin` and then sets outbound `channel`/`accountId`/`to`/`threadId` from that origin, not from `inputProvenance.sourceChannel`.
  - file: [src/agents/subagent-announce-delivery.ts:468](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/agents/subagent-announce-delivery.ts#L468)
- **Regression test prefers requesterOrigin over stale channel:** E2E coverage asserts queued announce delivery uses `requesterOrigin.channel` (`telegram`) instead of stale session `lastChannel`.
  - file: [src/agents/subagent-announce.format.e2e.test.ts:2899](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/agents/subagent-announce.format.e2e.test.ts#L2899)
- **Regression test keeps external completion delivery:** Delivery test asserts dormant completion requesters still send directly to the external channel (`slack`) with the original route context.
  - file: [src/agents/subagent-announce-delivery.test.ts:141](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/src/agents/subagent-announce-delivery.test.ts#L141)
- **Latest release contains same delivery logic:** Release `v2026.4.22` already contains the same `effectiveDirectOrigin`/external delivery code path now present on `main`.
  - command: `git show 00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5:src/agents/subagent-announce-delivery.ts | sed -n '460,555p'`
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)

## Risks / Open Questions

- `inputProvenance.sourceChannel` is still internal metadata, so if a future report is specifically about provenance fidelity rather than delivery routing, that would be a narrower follow-up issue.
