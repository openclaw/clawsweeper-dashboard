## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Store Sentinel keys in the existing secret boundary:** `scripts/docker/full-local.mjs:230-234`
  - body: Here `seedNvidiaVaultFromRuntime` writes resolved NVIDIA keys into a new JSON vault under the OpenClaw state/workspace tree. Current NVIDIA auth already goes through provider API key resolution, and `models.providers.*.apiKey` is a supported SecretRef path, so persisting raw keys in a new sidecar vault bypasses the credential audit/storage boundary and leaves duplicated secrets behind after rotation.
  - confidence: 0.9
- **[P1] Keep Docker publishing upgrade-safe:** `docker-compose.yml:64-66`
  - body: Current `main` publishes these ports using Docker's default host binding, but this changes existing `docker compose up` installs to loopback-only unless users discover the new publish-host env vars. That can break LAN clients, remote Docker, or Teams callback flows after upgrade; keep the old default and make loopback opt-in, or add maintainer-approved migration docs and tests.
  - confidence: 0.88
- **[P1] Allow documented ticket states in Blackboard:** `scripts/docker/sidecars/blackboard-cli.cjs:151`
  - body: The new Agent OS contract accepts `WAITING_APPROVAL` and `BLOCKED`, but this SQLite schema still rejects both. Any adapter that follows the documented contract can fail ticket inserts or updates in Blackboard storage; use the shared status contract or include every documented persisted state in all ticket schemas.
  - confidence: 0.93
- **[P1] Preserve existing capability-agent skills:** `scripts/agents/capability-agent-profile.mjs:190`
  - body: When `apply` refreshes a matching agent id, this replaces the user's `skills` list with the bundled profile list. Users who already created `research_agent` or another matching id with custom skills lose persisted choices even though model/workspace/params are preserved; merge or preserve existing skills unless maintainers intentionally choose a migration.
  - confidence: 0.86
