## Summary

Keep this PR open. Current main still has the old session-path behavior and Docker setup does not include the proposed JSON path normalization. The PR addresses a plausible bug, but its Docker-side rewrite is security-sensitive and the provided review discussion still leaves a concrete Node fallback failure to resolve.

## Best Possible Solution

Keep this PR open for maintainer review or author revision. The best path is to land a focused session resolver change that rebases canonical stale session transcript paths to the current state root with regression tests, then either remove the Docker JSON rewrite or narrow it to session metadata only with explicit Docker-out-of-Docker path-parity tests and a fixed Node fallback.
