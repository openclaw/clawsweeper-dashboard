## Summary

Keep this PR open. Current main still rejects `document` media in both the explicit image tool and native prompt image auto-injection paths, so the PR’s intended behavior is not implemented on main. The proposed change remains relevant, but it needs maintainer review/rework because `document` is a broad media kind, the provided diff includes a patch artifact, and the bot review already noted a MIME fallback inconsistency.

## Best Possible Solution

Keep the PR open for maintainer review or a rebased replacement. The best fix is to remove the patch artifact, address the fallback MIME mismatch, and narrow document-kind acceptance to media that is actually image-compatible, with regression tests for both `src/agents/tools/image-tool.ts` and `src/agents/pi-embedded-runner/run/images.ts`.
