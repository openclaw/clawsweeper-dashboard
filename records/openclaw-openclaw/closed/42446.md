## Close Comment

Closing as implemented on main after Codex review. OpenClaw now strips the exact leaked tool scaffolding from assistant-visible output, including malformed MiniMax XML such as `<invoke ...>` / `</minimax:tool_call>` and plain-text `<tool_result>...</tool_result>` blocks (`src/shared/text/assistant-visible-text.ts:310`, `src/shared/text/assistant-visible-text.ts:620`). The outbound reply path uses `extractAssistantVisibleText(...)` before delivery (`src/agents/pi-embedded-subscribe.handlers.messages.ts:630`), and targeted tests cover the same leak shape reported here (`src/shared/text/assistant-visible-text.test.ts:482`, `src/agents/tools/sessions.test.ts:190`). This behavior is already present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). If you still hit raw tool markup on a newer build, please open a fresh issue with the current version, provider/model, and a minimal repro transcript.
