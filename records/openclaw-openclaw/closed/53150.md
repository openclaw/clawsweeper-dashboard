## Summary

Current main already supports dynamic backend model switching on the Gateway OpenAI-compatible APIs, but via the documented agent-first contract: the request `model` selects an OpenClaw agent target, and `x-openclaw-model` selects the provider/model override for that agent.

## Evidence

- **Docs define the current contract:** The Gateway docs state that the OpenAI `model` field is an agent target (`openclaw`, `openclaw/default`, `openclaw/<agentId>`), and that `x-openclaw-model` is the supported backend model override.

- **Docs include a dynamic-switch example:** The examples show a chat-completions request using `model: "openclaw/research"` together with `x-openclaw-model: openai/gpt-5.4` to switch the backend model dynamically.
  - file: [docs/gateway/openai-http-api.md:237](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/docs/gateway/openai-http-api.md#L237)

## Risks / Open Questions

- Raw provider/model ids in the JSON body are still unsupported by design; callers must use the documented `x-openclaw-model` header for backend model switching.
