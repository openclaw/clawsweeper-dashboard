## Summary

Current `main` already contains Telegram-side reasoning stripping/splitting plus a MiniMax-specific anti-leak guard, and the same behavior is present in the shipped `v2026.4.23` release.

## Evidence

- **Reporter narrowed repro to MiniMax on Telegram:** The issue discussion includes a follow-up config and screenshot showing `minimax/MiniMax-M2.7` on the Telegram channel with reasoning enabled, which matches the code paths now guarded in current main.
- **MiniMax-specific leak prevention exists on main:** `extra-params.ts` installs `createMinimaxThinkingDisabledWrapper` with an inline comment explaining that MiniMax's Anthropic-compatible stream can leak `reasoning_content` into visible replies; the wrapper injects `thinking: { type: "disabled" }` unless a prior wrapper already set thinking explicitly.
