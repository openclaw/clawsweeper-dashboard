## Summary

Current `main` already contains the fallback-order and live-session-switch fixes described in #57063, and the same implementation is present in released tag `v2026.4.23`.

## Evidence

- **Fallback ordering keeps the requested session model first:** `resolveFallbackCandidates` adds the normalized requested model first, uses full configured fallbacks only for same-provider chains, and only appends the configured primary as a final candidate.
  - file: [src/agents/model-fallback.ts:460](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/agents/model-fallback.ts#L460)
- **Regression tests cover session override fallback order:** Tests assert that a session model differing from the configured primary still runs first, then falls through to configured fallbacks; cross-provider requests skip configured fallbacks and only try the configured primary last.
  - file: [src/agents/model-fallback.test.ts:1472](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/agents/model-fallback.test.ts#L1472)
- **Live-session switch state is scoped to explicit user changes:** Session metadata now distinguishes `modelOverrideSource` and documents that `liveModelSwitchPending` is only for explicit user-driven switches, not system fallbacks.
  - file: [src/config/sessions/types.ts:179](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/config/sessions/types.ts#L179)
- **Stale live switch flags are cleared instead of retriggering retries:** `shouldSwitchToLiveModel` returns early when the current model already matches the persisted selection and clears `liveModelSwitchPending` from the session store.
  - file: [src/agents/live-model-switch.ts:144](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/agents/live-model-switch.ts#L144)
- **Release tag contains the same fix:** `git show v2026.4.23:src/agents/model-fallback.ts`, `git show v2026.4.23:src/agents/model-fallback.test.ts`, and `git show v2026.4.23:src/agents/live-model-switch.test.ts` show the same fallback-order logic and regression tests in the released tree.
  - command: `git show v2026.4.23:src/agents/model-fallback.ts; git show v2026.4.23:src/agents/model-fallback.test.ts; git show v2026.4.23:src/agents/live-model-switch.test.ts`
  - sha: [9b18ae877018](https://github.com/openclaw/openclaw/commit/9b18ae877018a34933e27bb7b2986cb97e9602aa)
