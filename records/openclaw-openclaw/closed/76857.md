## Summary

Keep open. Current main still shows the affected guarded-fetch path creating per-request undici dispatchers without inheriting the global Happy Eyeballs setting, and open PR #76870 explicitly closes this issue, so this should stay paired with that PR until it lands or is rejected.

## What This Changes

The issue asks OpenClaw to restore openai-codex agent reliability on networks where DNS returns IPv6 records but outbound IPv6 is unavailable.

## Best Possible Solution

Ship a generic transport-layer regression fix that makes guarded per-request undici dispatchers inherit the system Happy Eyeballs setting, with coverage for pinned direct and proxy dispatchers.

## Reproduction Assessment

Yes, source-reproducible with high confidence: openai-codex Responses traffic uses guarded fetch, guarded fetch attaches per-request dispatchers, and those dispatchers currently omit `autoSelectFamily` while tests assert it is undefined. I did not live-recreate the network in this read-only pass.

## Solution Assessment

Yes. The proposed direction in #76870 is the right layer because the failure is in the shared guarded undici dispatcher path, not in openai-codex-specific model routing or user config.
