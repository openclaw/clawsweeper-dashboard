## Summary

This is a Lossless Claw context-engine/plugin behavior rather than an OpenClaw core bug. Current core already exposes the context-engine assemble/compact/maintenance contract needed for a plugin to preserve tool-call/tool-result pairs or omit orphaned tool calls, and the external Lossless Claw plugin is already documented as a community plugin.

## Best Possible Solution

Close this OpenClaw core issue and route the concrete fix to the Lossless Claw plugin/ClawHub path: during LCM compaction or assemble, keep assistant tool calls and matching tool results together, or omit orphaned tool calls from the assembled model context. Open a new core/API issue only if the plugin maintainers can show that the existing context-engine assemble/compact/maintenance and rewriteTranscriptEntries surfaces are insufficient.
