## Summary

Keep open. Current main and the latest release still contain the reported early-return path; the linked fix PR was closed unmerged, so this remains a narrow gateway bug suitable for a repair PR.

## What This Changes

Ensure gateway chat lifecycle finalization still emits a terminal chat event when registry peek succeeds but the subsequent shift misses.

## Best Possible Solution

Patch the gateway terminal finalizer to fall back to the already resolved session/run identifiers on a `shift()` miss, preserve terminal cleanup, and cover that exact branch with a focused regression test.

## Reproduction Assessment

Yes for a source-level reproduction: make `registry.peek(evt.runId)` return a chat link while the subsequent `registry.shift(evt.runId)` returns undefined, then send lifecycle `end` or `error`. Current main returns before the final chat payload and skips the remaining terminal cleanup.

## Solution Assessment

Yes. The narrow maintainable fix is in `finalizeLifecycleEvent`: on a `shift()` miss, emit the same fallback final chat payload used when no chat link exists instead of returning early.
