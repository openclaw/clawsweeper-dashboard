## Summary

Keep this PR open. Current main still has the unguarded `WebSocketTaskBox.sendPing()` continuation resume path, and the keepalive loop still calls it, so the crash fix has not been implemented on main. The functional change remains useful, but the PR should not land as-is because it also adds an unrelated Swift workflow that duplicates existing CI coverage and touches security-sensitive infrastructure.

## Best Possible Solution

Keep this PR open, ask the contributor or a maintainer to remove or split the new workflow, and land a focused `GatewayChannel.swift` fix that guarantees the ping continuation is resumed at most once. If maintainers prefer #51311 or another implementation, make that PR canonical first, then close the duplicate after the chosen fix merges.
