## Summary

Keep open. Current main still has the first-fingerprint/full-dearmor Docker apt key gap, and this PR is a narrow, green hardening patch, but it is still missing the required `CHANGELOG.md` entry for a user-facing Docker fix.

## What This Changes

The PR adds a Dockerfile guard requiring exactly one GnuPG `pub` primary key in Docker's apt signing-key file before fingerprint comparison and `gpg --dearmor`, plus a focused Dockerfile ordering test.

## Best Possible Solution

Add an Unreleased Fixes changelog entry crediting @zozo123, keep the Dockerfile guard and focused ordering test, then merge this PR or an equivalent narrow hardening change so #74234 closes through the merged fix.

## Reproduction Assessment

Yes. A key file whose first `fpr` matches `OPENCLAW_DOCKER_GPG_FINGERPRINT` but also contains another primary `pub` key would pass current main's first-fingerprint extraction and then be fully dearmored into the apt keyring.

## Solution Assessment

Yes, aside from the missing changelog. Counting primary `pub` records before trusting the first fingerprint is a narrow maintainable fix, and live Docker key inspection confirms the official key remains a single-primary-key file with subkeys reported separately.
