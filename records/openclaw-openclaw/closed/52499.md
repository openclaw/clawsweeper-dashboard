## Summary

Current main now covers the requested coordinator/reporter workflow through documented sub-agents, session tools, and channel delivery routing. A primary agent can spawn allowed reporter agents into isolated child sessions, those child runs do not take over inbound routing by default, and their completion is announced back through the requester session and originating chat/thread. The supported path shipped in v2026.4.26.

## Best Possible Solution

Keep the shipped sub-agents plus `sessions_send` coordinator/reporter pattern as the supported implementation. Open a narrower follow-up only if maintainers want a literal owner/contributor binding syntax in channel configuration.
