## Summary

Current main's control-panel model picker sends qualified `provider/model` refs into `sessions.patch`, so the PR's reported control-panel failure mode no longer matches the shipped UI path.

## Evidence

- **Picker options are provider-qualified:** `buildChatModelOptionFromLookup` sets each option value with `buildQualifiedChatModelValue(entry.id, provider)`, so control-panel selections are emitted as qualified refs rather than bare model ids.
  - file: [ui/src/ui/chat-model-ref.ts:266](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/ui/src/ui/chat-model-ref.ts#L266)

  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)
- **Control-panel state avoids bare duplicates:** The current chat model select state tests normalize bare overrides to qualified values and explicitly exclude bare duplicates like `gpt-5-mini` from picker options.
  - file: [ui/src/ui/chat-model-select-state.test.ts:65](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/ui/src/ui/chat-model-select-state.test.ts#L65)
