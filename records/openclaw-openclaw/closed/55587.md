## Summary

Current main implements the requested split between a truly missing LaunchAgent and a plist that still exists while launchd has no loaded job. The linked PR #55591 proposed the same shape but was closed unmerged; the equivalent implementation landed later in #72616 and is contained in v2026.4.26.

## Best Possible Solution

Close as implemented on main by #72616. Keep the shipped `missingSupervision` status path and restart guidance; any broader macOS LaunchAgent lifecycle or auto-upgrade recovery work should stay in its narrower follow-up threads rather than this false not-installed status report.
