## Summary

Issue appears to have been a transient Kimi Coding upstream streaming outage on March 30, 2026, not a persistent OpenClaw mainline regression. Current main also contains guardrails that turn this parser failure into a retryable user-facing error and avoid cascading transcript repair damage from aborted/errored tool-call turns.

## Evidence

- **Issue discussion points to an upstream outage window, not a version-specific core regression:** Multiple reporters said the exact same failure started suddenly on March 30, 2026 around 11:50 with no local OpenClaw/config changes, across v2026.3.13 and v2026.3.28. Later comments from fishel-feng and the original reporter said the problem recovered without configuration changes by March 31, 2026.
  - command: `provided GitHub issue context for #57523 comments/timeline`
