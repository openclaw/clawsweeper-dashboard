## Summary

Keep open: the PR still contains potentially useful artifact pagination/type-filter work, but it is not merge-ready because it regresses the current artifact provenance contract, changes shipped SDK helper signatures, has unstable cursor semantics, mixes in an under-wired Hyprland path, and lacks external real behavior proof.

## What This Changes

The branch adds cursor/limit/type filtering to artifact RPCs, changes SDK artifact helper shapes and protocol/generated models/docs, and adds an opt-in Hyprland screenshot capture path.

## Best Possible Solution

Rebase on current artifact RPC hardening, keep run/task provenance fail-closed, add cursor/type filtering with artifact-level stable cursor semantics, preserve current SDK helper signatures, and move or fully wire the Hyprland fix through https://github.com/openclaw/openclaw/pull/71940 with redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection gives high-confidence reproduction paths for the PR blockers: untagged artifacts pass scoped queries, messageSeq-only cursors skip same-message artifacts, the SDK call shape diverges from current main, and hyprlandCapture is checked but not resolved from config.

## Solution Assessment

No. The current PR is not the best merge vehicle until it preserves current SDK/provenance contracts, uses stable cursor semantics, and separates or fully owns the Hyprland config/runtime path with proof.
