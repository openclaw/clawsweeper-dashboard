## Summary

Current main already provides the requested capability in practice: durable JSONL session transcripts on disk, non-truncating compaction by default, pre-compaction checkpoint snapshots with restore/branch flows, opt-in transcript indexing for `memory_search`, and a shipped backup CLI that includes sessions.

## Evidence

- **Session persistence design:** The session-management deep dive says OpenClaw persists sessions in two layers and that `<sessionId>.jsonl` is an append-only transcript storing the actual conversation, tool calls, and compaction summaries.
  - file: [docs/reference/session-management-compaction.md:42](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/docs/reference/session-management-compaction.md#L42)
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Transcript format is JSONL:** The same doc defines transcript structure as JSONL with a session header followed by message/custom/compaction entries.
  - file: [docs/reference/session-management-compaction.md:165](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/docs/reference/session-management-compaction.md#L165)
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Transcript writer persists session files:** The transcript writer creates the session JSONL header on disk, including the session id and timestamp, as the durable transcript file.
  - file: [src/config/sessions/transcript.ts:18](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/src/config/sessions/transcript.ts#L18)
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Transcript persistence is verified in tests:** The transcript test reads the written JSONL file and asserts the next line is a `message` entry containing the assistant text.
  - file: [src/config/sessions/transcript.test.ts:75](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/src/config/sessions/transcript.test.ts#L75)
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Compaction does not delete history by default:** `agents.defaults.compaction.truncateAfterCompaction` only rewrites the session JSONL when explicitly enabled, and the documented default is `false`.
  - file: [src/config/schema.help.ts:1224](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/src/config/schema.help.ts#L1224)
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Pre-compaction snapshots exist:** Compaction checkpoint capture copies the live transcript to a sibling `.checkpoint.*.jsonl` snapshot before compaction.
  - file: [src/gateway/session-compaction-checkpoints.ts:72](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/src/gateway/session-compaction-checkpoints.ts#L72)
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Checkpoint restore flow exists:** Gateway session methods can restore a session from the saved pre-compaction snapshot transcript.
  - file: [src/gateway/server-methods/sessions.ts:1123](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/src/gateway/server-methods/sessions.ts#L1123)
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Session transcripts can be searched:** The memory config docs expose opt-in session transcript indexing for `memory_search` via `experimental.sessionMemory` plus `sources` including `"sessions"`.
  - file: [docs/reference/memory-config.md:342](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/docs/reference/memory-config.md#L342)
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)
- **Backup CLI already includes sessions:** `openclaw backup create` is documented as backing up OpenClaw state including sessions, and its backup sources include the state directory, usually `~/.openclaw`.
  - file: [docs/cli/backup.md:11](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/docs/cli/backup.md#L11)
  - sha: [cec34821754a](https://github.com/openclaw/openclaw/commit/cec34821754ad935c780805f3116f99449d46ac2)

## Risks / Open Questions

- The implementation does not use the exact path/config shape proposed in the issue; it uses per-agent transcript storage under `agents/<agentId>/sessions/` instead of `~/.openclaw/sessions/<session-id>/history.jsonl`.
- Transcript-backed `memory_search` is opt-in/experimental rather than enabled by default.
