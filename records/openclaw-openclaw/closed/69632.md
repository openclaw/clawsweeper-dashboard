## Summary

Current main already contains automatic cleanup for completed subagent sessions plus enforced session-store retention/capping, so the indefinite accumulation path described in #69632 is implemented away on shipped OpenClaw builds.

## Evidence

- **Session-mode completed runs are swept after cleanup:** Current main applies an absolute 5-minute TTL to session-mode subagent runs after cleanup completes, then removes them from the in-memory registry and emits a swept lifecycle end.

## Risks / Open Questions

- This was a read-only code/docs/history review; I did not run a live long-duration RSS repro against a large real session store.
- The cleanup and retention behavior is configurable. Users who set `archiveAfterMinutes: 0`, disable enforcement, or greatly raise `session.maintenance.maxEntries` can still choose larger memory footprints.
