## Summary

Close as ClawHub/community-discovery scope. The linked watchdog appears useful, but this issue asks OpenClaw to promote or vendor an external optional self-healer; current main routes plugin/tool discovery and provenance through ClawHub or other community submission channels, while the Gateway already exposes the health/readiness and restart surfaces the tool needs.

## What This Changes

Request to link or vendor the external `openclaw-watchdog` community CLI that monitors Gateway health, restarts with backoff, logs incidents, and sends notifications.

## Best Possible Solution

Keep OpenClaw core docs focused on the Gateway health/restart contracts and plugin extension points. The watchdog should remain externally maintained and be submitted through ClawHub if packaged as a plugin/skill, or through the existing community showcase channels; reopen core work only if the tool exposes a missing health, readiness, or restart API.
