## Summary

Current `main` already adds a post-compaction context refresh. After auto-compaction, OpenClaw enqueues a system event that reloads selected `AGENTS.md` sections and explicitly tells the agent to rerun its startup sequence and read required files before replying. The behavior is documented, configurable, and covered by tests. The implementing commit is on `main` after the latest listed release, so this looks fixed on `main` but not yet released in `v2026.4.22`.
