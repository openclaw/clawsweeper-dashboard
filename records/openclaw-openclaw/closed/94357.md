## Summary

Keep open: the observe-only hook direction is useful, but the current patch is not merge-ready because observe-only WhatsApp messages can still be marked read and can still emit ack/status reactions, while the supplied proof is only local test/typecheck output.

## What This Changes

The PR expands WhatsApp `pluginHooks.messageReceived` so disabled DM/group policy messages are admitted as observe-only, adds an early return after hook emission, and adds focused unit tests.

## Best Possible Solution

Make observe-only a true hook-only WhatsApp path that honors `shouldMarkRead`, suppresses ack/status/preflight reactions, adds focused regression coverage, and then require redacted real WhatsApp proof.
