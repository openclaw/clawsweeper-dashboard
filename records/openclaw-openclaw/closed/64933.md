## Summary

Current main already handles the reported case: plugin loading merges trust settings from the activation/source config before warning evaluation, the open-allow warning exits when `plugins.allow` is populated, and the provenance warning is skipped for loaded non-bundled plugins whose id is in the allowlist. Regression coverage includes an auto-discovered `extensions/` plugin allowlisted by id and a source-runtime-snapshot allowlist case with no false warnings. The related status-path report #62049 and the #63848 diagnosis were useful context, but no separate core work remains for this item.

## Best Possible Solution

Close this issue as implemented on main. Keep the current source-snapshot trust merge and allowlist/provenance warning gates; treat #62049 and #63848 as useful historical status-path context rather than separate remaining work for this lossless-claw warning report.
