## Close Comment

This looks fixed on current `main`, so I would close it as implemented.

- Codex review of `src/plugins/gateway-startup-plugin-ids.ts:251` shows bundled plugins now stay in the gateway startup set when their activation source is `default` or `explicit`, which covers `enabledByDefault: true`.
- The current regression test in `src/plugins/channel-plugin-ids.test.ts:382` explicitly expects `browser` to load at idle startup with empty config, and `src/plugins/channel-plugin-ids.test.ts:401` keeps it when `plugins.allow` includes `browser`.
- The bundled browser plugin still declares `enabledByDefault: true` in `extensions/browser/openclaw.plugin.json:2`, and it still registers the `browser.request` RPC in `extensions/browser/plugin-registration.ts:36`.
- The fix is called out in the `2026.4.5` changelog: `CHANGELOG.md:1518` (`#57931`, “keep bundled enabledByDefault plugins in the gateway startup set”).
