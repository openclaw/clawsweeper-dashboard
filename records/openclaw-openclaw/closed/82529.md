## Summary

Keep open: current main still rejects the default Active Memory agent when `config.agents` is empty, and this PR is a narrow fix, but it still needs actual after-fix command proof and a small docs alignment before merge.

## What This Changes

The PR changes Active Memory agent gating to treat missing or empty `config.agents` as the default `main` agent and adds regression coverage for `/active-memory on` and `/active-memory status`.

## Best Possible Solution

Land the shared gate fallback with regression coverage after actual command-path proof, and update Active Memory docs so missing or empty `config.agents` means the default `main` agent while explicit non-empty allowlists still fail closed.

## Reproduction Assessment

Yes. Source inspection on current main shows omitted `config.agents` normalizes to `[]`, then `/active-memory` rejects `main` through `isEnabledForAgent` before the toggle/status action runs.

## Solution Assessment

Yes, mostly. Reusing `isEnabledForAgent` for the default-agent fallback is the narrow maintainable code path, but the docs and real behavior proof need to match the new contract before merge.
