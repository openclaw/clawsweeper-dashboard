## Summary

Close as cannot_reproduce. On current main, media-understanding CLI prompts are interpolated into an argv array and executed through `execFile` with shell execution disabled, so the PR’s built-in Gemini `$(...)`/backtick CWE-78 claim does not reproduce as described.

## Evidence

- **CLI runner passes templated values as argv entries:** `runCliEntry()` builds `templCtx`, substitutes into each arg with `applyTemplate()`, and then calls `runExec(argv[0], argv.slice(1), ...)`. That is argv execution, not shell interpolation.
  - file: [src/media-understanding/runner.entries.ts:733](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/src/media-understanding/runner.entries.ts#L733)
  - command: `nl -ba src/media-understanding/runner.entries.ts | sed -n '720,770p'`
  - sha: [4630ce3d9e2f](https://github.com/openclaw/openclaw/commit/4630ce3d9e2f089062ccd386b0ae241325961d1a)
- **Process launcher hard-disables shell mode:** `runExec()` uses `execFile`, and `shouldSpawnWithShell()` explicitly returns `false` with a security comment saying argv-based execution must never enable `shell`.
  - file: [src/process/exec.ts:117](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/src/process/exec.ts#L117)
  - command: `nl -ba src/process/exec.ts | sed -n '1,170p'`
  - sha: [4630ce3d9e2f](https://github.com/openclaw/openclaw/commit/4630ce3d9e2f089062ccd386b0ae241325961d1a)
- **Built-in Gemini CLI entry is a direct binary invocation:** The built-in media-understanding Gemini entry is `command: "gemini"` with `"{{Prompt}}"` as its own argv element. It is not configured as `sh -c`, `bash -lc`, or another shell wrapper.
  - file: [src/media-understanding/runner.ts:418](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/src/media-understanding/runner.ts#L418)
  - command: `nl -ba src/media-understanding/runner.ts | sed -n '412,432p'`
  - sha: [4630ce3d9e2f](https://github.com/openclaw/openclaw/commit/4630ce3d9e2f089062ccd386b0ae241325961d1a)
- **Windows shim already rejects real cmd.exe metacharacters:** When Windows must route through `cmd.exe` for `.cmd`/`.bat`, `escapeForCmdExe()` rejects `& | < > ^ %` and newlines before building the command line.
  - file: [src/process/exec.ts:15](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/src/process/exec.ts#L15)
  - command: `nl -ba src/process/exec.ts | sed -n '1,60p'`
  - sha: [4630ce3d9e2f](https://github.com/openclaw/openclaw/commit/4630ce3d9e2f089062ccd386b0ae241325961d1a)
- **Tests codify the no-shell contract:** The process tests assert that Windows command launch still does not enable shell execution.
  - file: [src/process/exec.test.ts:94](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/src/process/exec.test.ts#L94)
