## Summary

Current main already removes the per-turn plugin-file freshness checks and adds package realpath caching in the plugin metadata path, so the reported stock-plugin latency regression is handled on main but has not shipped in the latest release yet.

## What This Changes

The issue reports a regression where stock bundled plugins cause 8-12 second delays before LLM activity on each chat turn in packaged releases.

## Best Possible Solution

Keep the current main plugin-metadata snapshot and realpath caching behavior, ship it in the next release, and reopen only with post-main or post-release timings that still show per-turn plugin scans.

## Reproduction Assessment

Yes, from source for the affected release path: v2026.5.22 still hashed watched plugin files during hot memo validation, matching the repeated-stat latency report. I did not rerun a live latency benchmark, and current main removes that hot-path check.

## Solution Assessment

Yes. Treating plugin metadata as process-stable during a running gateway and caching package realpaths matches the repository hot-path policy better than per-turn freshness polling.
