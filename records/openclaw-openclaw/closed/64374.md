## Summary

Closing as `cannot_reproduce`: the issue report was resolved in-thread as a Slack token-selection/configuration problem, not a current OpenClaw core bug. The reporter confirmed a configured `channels.slack.userToken` was being preferred for reads; removing that token made `message(action="read")` work. Current `main` intentionally preserves that read-path precedence, with tests and docs matching the behavior.

## Evidence

- **Reporter confirmed root cause and workaround:** In the issue discussion, the reporter confirmed the failure came from a configured Slack `userToken` being selected for reads; removing `userToken` made `message(action="read")` work again.
  - command: `GitHub issue #64374 comment 4225528123 (provided context)`
- **Current read path intentionally prefers user token:** Slack action runtime chooses `userToken ?? botToken` for read operations, so a configured user token is expected to override the bot token on reads.
  - file: [extensions/slack/src/action-runtime.ts:162](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/extensions/slack/src/action-runtime.ts#L162)
  - command: `nl -ba extensions/slack/src/action-runtime.ts | sed -n '150,190p'`
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **Behavior is covered by tests on main:** Regression test explicitly asserts that reads use the user token when available.
  - file: [extensions/slack/src/action-runtime.test.ts:653](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/extensions/slack/src/action-runtime.test.ts#L653)
  - command: `nl -ba extensions/slack/src/action-runtime.test.ts | sed -n '648,706p'`
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **Docs match the implemented token model:** Slack docs state that `userToken` defaults to read-only behavior and that for actions/directory reads, the user token can be preferred when configured.
  - file: [docs/channels/slack.md:408](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/docs/channels/slack.md#L408)
  - command: `nl -ba docs/channels/slack.md | sed -n '404,430p'`
