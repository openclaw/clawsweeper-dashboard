## Summary

Current main, and the shipped v2026.4.24 tag, already render the Control UI cron one-shot Run at field as a time-capable datetime-local input, preserve date/time values when editing, and serialize one-shot schedules from that value. That addresses the reported scheduled-task inability to select time; the only issue comment also pointed to native browser picker support rather than a missing OpenClaw schedule model.
