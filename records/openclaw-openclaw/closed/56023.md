  - body: These commands require a `<chunk-id>`, but the normal `memory search` output shows only score, path, line range, and snippet, and the public search result type does not expose chunk ids.   - confidence: 0.88

  - body: The feedback path writes `WeightedRecallStore` directly instead of going through the active memory manager. When `memorySearch` resolves to QMD, CLI/search uses `QmdMemoryManager`, which never reads `memory-weights.json`, so the command can print success while ranking is unchanged; either implement QMD support or fail clearly when the active backend cannot honor feedback.
  - confidence: 0.9

  - body: `flush()` writes the entire cached weight map back to disk.
