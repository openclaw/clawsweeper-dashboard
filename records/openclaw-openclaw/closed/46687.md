## Summary

Current main already handles the reported mismatch well enough: the bundled `session-memory` hook still writes `memory/YYYY-MM-DD-slug.md`, but startup context now enumerates and loads both canonical daily files and slugged same-day files, and the AGENTS template tells agents to use runtime-provided startup context first instead of manually rereading a guessed `memory/YYYY-MM-DD.md` path.

## Evidence

- **Hook still writes slugged daily files:** The bundled hook continues to save session memory as `YYYY-MM-DD-slug.md` via `const filename = `${dateStr}-${slug}.md``.
