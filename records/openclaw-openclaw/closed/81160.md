## Summary

Close as implemented: current main and the stable v2026.5.12 release provide an agent-accessible `canvas.a2ui.reset` recovery path that forces A2UI host navigation with capability refresh, while `canvas present` preserving explicit targets is tested behavior.

## What This Changes

The issue asks for macOS Canvas to recover A2UI after WebView navigation or capability-token expiry, either by making `canvas present` force a fresh A2UI URL or by adding an explicit reset path.

## Best Possible Solution

Keep `canvas present` as a non-destructive show command and use the shipped `canvas.a2ui.reset` path when an agent needs to force Canvas back to a fresh A2UI host.

## Reproduction Assessment

Yes for the narrow `canvas present` behavior: current source and smoke coverage show user-selected targets are preserved. The broader no-recovery failure is addressed on current main by the explicit reset path.

## Solution Assessment

Yes. An explicit A2UI reset is the narrower maintainable solution because it restores a fresh capability URL without breaking tested `canvas present` target-preservation semantics.
