## Summary

Keep open: this is a draft, maintainer-labeled PR, and the diff has merge-blocking issues in the live Telegram setup path and TDLib artifact verification.

## What This Changes

Adds a Telegram real-user QA credential shape, chunked Convex credential payload leasing, a Crabbox proof runner, docs, package script, and focused tests.

## Best Possible Solution

Land a maintainer-reviewed proof lane that keeps the shared credential approach but clears Telegram webhook state before polling and requires trusted TDLib artifact verification.

## Reproduction Assessment

No high-confidence live reproduction was run in this read-only review. Source inspection gives a clear failure path when the SUT bot still has a webhook configured, because the runner calls `getUpdates` before webhook cleanup.

## Solution Assessment

No. The direction is useful, but the current implementation should be hardened by deleting the webhook before draining updates and by requiring a trusted TDLib digest instead of accepting a sibling checksum from the same URL.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Delete the webhook before draining updates:** `scripts/e2e/telegram-user-crabbox-proof.ts:397`
  - body: The runner calls `getUpdates` in `drainSutUpdates` before the local gateway starts. Current Telegram monitor coverage expects `deleteWebhook({ drop_pending_updates: false })` before polling; if the SUT bot was left in webhook mode, this proof exits before it can start the gateway. Clear the webhook first or let the gateway startup own draining.
  - confidence: 0.86
- **[P2] Require a trusted digest for TDLib downloads:** `scripts/e2e/telegram-user-crabbox-proof.ts:708-713`
  - body: When `--tdlib-sha256` is omitted, the script fetches `<tdlib-url>.sha256` from the same supplied URL and then installs the extracted `libtdjson.so`. The documented proof command uses an HTTP artifact URL, so a network or mirror compromise can replace both archive and digest before code is loaded. Require HTTPS plus an explicit pinned digest, or fetch the digest from a trusted source.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The PR introduces a concrete supply-chain risk in the Crabbox proof runner's TDLib download path.

Concerns:

- **[medium] TDLib archive checksum is fetched from the same URL:** `scripts/e2e/telegram-user-crabbox-proof.ts:708`
  - body: The runner accepts `--tdlib-url`, downloads the archive, and if no digest is provided downloads `<tdlib-url>.sha256` from the same location before installing `libtdjson.so`. With the documented HTTP artifact URL, transport or artifact compromise can replace both files.
  - confidence: 0.9
