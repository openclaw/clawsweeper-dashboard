## Summary

Current `main` already implements model-scoped auth-profile cooldowns and threads the candidate model through fallback/auth-profile selection, so an Opus cooldown no longer blocks Sonnet fallback on the same profile.

## Evidence

- **Model-scoped cooldown bypass in auth state:** `isProfileInCooldown` bypasses an active cooldown when the recorded `cooldownModel` differs from the requested model, while still honoring profile-wide disables.
