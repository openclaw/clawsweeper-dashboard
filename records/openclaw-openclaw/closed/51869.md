## Summary

Close as implemented: current main and the shipped v2026.4.27 release already have custom-provider image-input inference, an interactive prompt for unknown models, non-interactive override flags, docs, and regression coverage, so the reported silent text-only generation no longer matches current behavior.

## What This Changes

The issue requests that non-Azure custom-provider onboarding stop writing all custom models as text-only and instead infer or ask for image input support for vision-capable models.

## Best Possible Solution

Keep the shipped onboarding behavior: infer known vision-capable custom models, ask for unknown interactive models, and use explicit non-interactive flags for scripted setups. Users who generated text-only custom model entries on older versions should upgrade and re-run onboarding or update those existing config entries once.

## Reproduction Assessment

Yes. The old reproduction path was to run custom onboarding with a vision-capable non-Azure model and inspect the generated model `input`; current main has source and regression-test proof that known vision IDs now produce `['text', 'image']`, and unknown models are no longer silently forced text-only in the interactive flow.

## Solution Assessment

Yes. The current implementation is the narrow maintainable fix: it avoids defaulting every custom model to image-capable, covers common vision families automatically, prompts when uncertain, and exposes explicit non-interactive flags for automation.
