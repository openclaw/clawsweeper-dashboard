## Summary

Keep open: current main still resolves Windows batch wrappers through `process.env.ComSpec`, while the latest PR head replaces that path with the shared Windows install-root resolver, adds focused regression coverage and changelog, and fixes the earlier WINDIR CI blocker; the protected `maintainer` label means this should proceed through normal maintainer handling rather than cleanup close.

## What This Changes

The PR changes Windows `.cmd`/`.bat` wrapper launches to derive `cmd.exe` from `getWindowsInstallRoots()`, adds regression tests for malicious `ComSpec` and unsafe roots, and adds a changelog entry.

## Best Possible Solution

Land this through normal maintainer/security review so the shared Windows install-root resolver becomes the command-wrapper trust boundary.
