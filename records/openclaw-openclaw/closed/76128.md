## Summary

Keep open. Current main still lets `openclaw models set` update and report only `agents.defaults.model` while an explicit default-agent model remains authoritative, and the open related PRs cover adjacent config-set/UI surfaces rather than this command path.

## What This Changes

The issue reports that changing the global default model with `openclaw models set` can leave the active default `main` agent pinned to an older explicit `agents.list[].model.primary`.

## Best Possible Solution

Keep per-agent model precedence intact, but make `openclaw models set` report explicit default-agent model overrides that will continue to shadow the newly selected global default.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a new `agents.defaults.model.primary` while `agents.list[].id == "main"` has an older explicit model, then `models set` updates only defaults while effective resolution still chooses the agent model.

## Solution Assessment

Yes. A focused warning is the narrowest maintainable fix because the docs define per-agent model overrides as intentional, so automatic reconciliation would risk changing valid configurations.
