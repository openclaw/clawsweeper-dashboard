## Summary

Current main already implements PR #71991's requested `device.token.revoke` caller-scope containment. The gateway now passes the caller session scopes into `revokeDeviceToken`, and the infra mutation checks `resolveMissingRequestedScope` under the same lock before setting `revokedAtMs`, which also closes the TOCTOU shape discussed in the PR review. Regression tests and gateway protocol docs are already aligned.

## Best Possible Solution

Close PR #71991 as already implemented on main. Keep the current infra-layer implementation, because it enforces caller-scope containment inside the locked token mutation and covers both ordinary revoke and missing-device race cases. No additional OpenClaw core change is needed for this PR; future work should only reopen with a new reproduction if the current main behavior regresses.

## Evidence
