## Summary

Keep open. Current main still lets the missing-configured-plugin install repair collect the downloadable `feishu` channel package from `channels.feishu` without checking whether an enabled alternative plugin already owns that channel; the related open PR covers adjacent auto-enable behavior, not this doctor install path.

## What This Changes

`openclaw doctor --repair` should avoid installing or enabling the downloadable `@openclaw/feishu` plugin when `openclaw-lark` is already enabled and handling `channels.feishu`.

## Best Possible Solution

Apply the existing channel ownership and `preferOver` arbitration before doctor installs channel-selected candidates, then add a regression for `channels.feishu` with enabled `openclaw-lark`.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with `channels.feishu` configured, `plugins.entries.openclaw-lark.enabled=true`, no usable `feishu` install record, and `feishu` available from the official channel catalog, the installer can still select and install `@openclaw/feishu`.

## Solution Assessment

Yes. The narrow maintainable fix is generic channel-owner/preferOver filtering in the missing configured plugin install path; a hard-coded `feishu` guard would leave the shared replacement-plugin contract inconsistent.
