## Summary

Close as duplicate of #76367: this report describes the same memory-core cold-registry doctor false positive already tracked there, with #76393 open as the implementation path.

## What This Changes

The issue reports that `openclaw doctor --non-interactive` warns that no active memory plugin is registered even though `memory-core` is enabled, selected, and reported healthy by Gateway/status.

## Best Possible Solution

Keep #76367 as the canonical tracker and land #76393 or an equivalent narrow fix that loads only the configured memory-slot plugin for cold standalone doctor/status runtime resolution.

## Reproduction Assessment

Yes, source-reproducible. Current main emits the exact doctor warning when backend resolution returns null, and the helper only checks already-loaded runtime registry state before returning the current memory runtime.

## Solution Assessment

Yes. The best solution is the canonical scoped runtime-load fix tracked by #76367/#76393, rather than keeping a second issue open for the same observable failure.
