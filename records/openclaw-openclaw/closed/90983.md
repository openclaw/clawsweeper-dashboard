## Summary

Close as implemented: current main and the latest release already normalize the reported root-level `oldText`/`newText` edit shape before schema validation, so the specific validation-retry timeout path is covered even though the tool description itself did not gain the proposed example.

## What This Changes

The issue requests adding an explicit example and warning to the edit tool description so models keep `oldText` and `newText` inside the `edits` array.

## Best Possible Solution

Keep the shipped `v2026.6.1` argument-preparation behavior for this edit-shape case, and leave the broader repeated-tool-failure stop policy to https://github.com/openclaw/openclaw/issues/86454.

## Reproduction Assessment

No for current main: the reported root-level edit shape is normalized before validation, so the specific rejection loop should not reproduce on `v2026.6.1` or current main. I did not run a live MoE model session.

## Solution Assessment

Yes: normalizing this malformed edit call before validation is a stronger fix than only adding a prompt example, because it prevents the retry loop even if a model emits the old root shape again.
