## Summary

Current main already supports multiple auth profiles for the same provider, per-session profile selection, per-agent profile routing, and channel/account-to-agent bindings, which covers the requested multi-account OAuth use case without adding `pi-multi-pass`.

## Evidence

- **Docs: multi-account OAuth is already documented:** The OAuth docs explicitly describe two supported patterns for multiple accounts: separate agents for isolated credentials/workspaces, and multiple profile IDs for one provider inside a single agent, selected via `auth.order` or `/model ...@<profileId>`.
