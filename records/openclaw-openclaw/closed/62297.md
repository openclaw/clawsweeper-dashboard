## Summary

Current `main` now rewrites local/file assistant media paths to authenticated Control UI HTTP routes, serves those routes in the gateway, and has basePath-aware bootstrap config handling with regression coverage for Windows `file:///...` media paths and `/__openclaw/control-ui-config.json` under a configured base path.

## Evidence

- **Control UI rewrites local/file assistant media to HTTP:** `buildAssistantAttachmentUrl()` converts local assistant attachment sources into `__openclaw__/assistant-media?...` URLs instead of leaving them as raw `file://` or local filesystem paths in rendered chat output.
  - file: [ui/src/ui/chat/grouped-render.ts:900](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/ui/src/ui/chat/grouped-render.ts#L900)

  - file: [src/gateway/control-ui.ts:185](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/gateway/control-ui.ts#L185)
