## Summary

Current main already fixes the reported Bonjour/CIAO startup crash: the bundled Bonjour plugin now registers scoped CIAO process-error handlers, the global unhandled-rejection path honors registered handlers before exiting, the CIAO classifier covers ANNOUNCEMENT and PROBING cancellations, and the advertiser self-disables after repeated stuck restarts instead of crash-looping. The affected v2026.4.24 tag did not wire the handler into the plugin, but current main does.

## Best Possible Solution

Close this issue as fixed on current main. Keep the shipped/main implementation: scoped Bonjour/CIAO process-error handling, bounded advertiser restarts, self-disable behavior for multicast-blocked environments, and the documented WSL/Docker workaround. Any patch-release/backport decision should be handled through the normal release process rather than leaving this core bug open.
