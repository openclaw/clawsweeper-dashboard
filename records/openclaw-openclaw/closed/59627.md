## Summary

Current main addresses the reported TUI silent-stall failure mode: `chat.send` now uses the bounded GatewayClient request path, unresolved sends fail visibly instead of remaining silent, the tick watchdog closes stale sockets after pending RPCs settle, and the linked merged PR #59800 preserves/reconciles pending local TUI sends across reload/reconnect paths.

## Best Possible Solution

Keep the shipped GatewayClient timeout/tick-watch behavior and TUI pending-send reconciliation from #59800. If a current version still silently stalls, track it as a new focused bug with modern logs and a reproduction against the current GatewayClient/TUI paths.
