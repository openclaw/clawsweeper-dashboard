## Summary

Close PR #61510 as superseded by the canonical Control UI math-rendering work. Current main still does not implement KaTeX/MathJax rendering, so this is not an implemented-on-main close. The remaining feature is already tracked by open issue #42840, and newer open PR #67103 targets the current markdown-it renderer while this PR is an older unmerged marked-based attempt with unresolved review blockers.

## Best Possible Solution

Close this older PR and consolidate the remaining feature under #42840 and the newer current-renderer implementation path in #67103 or its successor. The eventual fix should live in ui/src/ui/markdown.ts, preserve existing markdown/link/security behavior, avoid currency false positives and code-span parsing, narrowly handle KaTeX styling/sanitization, import required CSS/dependencies, and add focused rendering tests for inline, block, invalid, and mixed Markdown/math messages.
