## Summary

Keep this PR open because it targets a real Telegram edit-message limit bug, but the submitted patch is not mergeable as-is: it counts rendered HTML rather than Telegram's post-entity text, leaves a CI-breaking unused helper, needs to preserve the current HTML-to-plain fallback, and lacks real Telegram proof.

## What This Changes

The PR adds a Telegram edit payload truncation helper, wires it into `editMessageTelegram` for HTML/plain fallback edits, and adds focused `send.test.ts` coverage.

## Best Possible Solution

Keep the fix in the Telegram plugin, but count/truncate against Telegram's post-entity parsed text, preserve the current HTML-to-plain fallback, add entity-expanded regression coverage, remove or wire the unused helper, and verify a real Telegram edit flow.

## Reproduction Assessment

Yes, source-reproducible rather than live-reproduced. Current main sends edit text directly to `editMessageText`, and Telegram documents the edit text limit as 1-4096 characters after entities parsing.

## Solution Assessment

No, not as submitted. The Telegram edit path is the right owner boundary, but the patch should measure what Telegram counts, preserve the current plain fallback, and satisfy the live-proof gate.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Count edit length after entity parsing:** `extensions/telegram/src/send.ts:215-218`
  - body: The guard fires on `htmlText.length`, but Telegram's `editMessageText` limit is after entities parsing. Escaped characters or long link hrefs can make rendered HTML exceed 4096 while the visible Telegram text is still valid, and `omittedChars` can remain zero because it is raw-text based. Base the limit and omitted count on Telegram-counted text and cover entity-expanded input.
  - confidence: 0.93
- **[P2] Remove the unused message-too-long helper:** `extensions/telegram/src/send.ts:192-194`
  - body: `isMessageTooLongTelegramError` is added but never called, and the PR's CI logs already show the repo's lint gate failing with `Function 'isMessageTooLongTelegramError' is declared but never used.` Remove it or wire it into the error path before merge.
  - confidence: 0.96
- **[P2] Preserve the HTML-to-plain edit fallback:** `extensions/telegram/src/send.ts:1428`
  - body: The truncation call passes `text` as the plain fallback input, but current main now derives readable plain text with `telegramHtmlToPlainTextFallback` when `textMode: "html"`. Rebase the branch and truncate that derived fallback instead, otherwise an HTML parse retry can show raw tags/attributes to the user.
  - confidence: 0.88
