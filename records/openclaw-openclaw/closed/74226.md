## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Base the truncation footer on the rendered payload:** `extensions/telegram/src/send.ts:218`
  - body: `truncateTelegramEditPayload` truncates whenever `htmlText` is over 4096 chars, but the footer count only uses `rawText.length - 4000`. If rendering expands a shorter raw source past the hard limit, this returns `0`, `buildTelegramEditTruncationFooter` returns an empty string, and the user-visible edit is cut with no truncation marker. Base the omitted count and footer reservation on the actual rendered/truncated payload and cover an HTML-expanded case.
  - confidence: 0.9
- **[P2] Preserve the HTML plain-text retry source:** `extensions/telegram/src/send.ts:1428`
  - body: Current main derives the plain retry with `telegramHtmlToPlainTextFallback(htmlText)` when `textMode` is `html`, so parse-mode retries send readable labels/links instead of raw markup. This PR passes raw `text` into the truncator as the plain variant, so a fallback after an HTML parse error can send raw tags/anchors; compute the existing plain fallback first and truncate that value.
  - confidence: 0.86
- **[P2] Remove or use the message-too-long helper:** `extensions/telegram/src/send.ts:192-194`
  - body: `tsconfig.extensions.json` enables `noUnusedLocals`, but `isMessageTooLongTelegramError` is added as a local helper and never called. That leaves the branch typecheck-sensitive even if the runtime logic is fixed; remove it or wire it into a real path.
  - confidence: 0.84
