## Summary

Keep open. The issue is fresh, the reporter is not a maintainer, and current main does not provide enough evidence that either reported behavior is fixed. Main has recently added OpenCode Go DeepSeek V4 model entries, but the tool-payload compatibility flag/wrapper needed for an Anthropic transport that must emit OpenAI-style `function.name` tools is not applied to those entries. The fallback/Telegram leak side also remains plausible because the visible reply path sanitizes envelopes/directive tags, then forwards text content; I found no current-main regression proof for stripping plain planning text from fallback replies.
