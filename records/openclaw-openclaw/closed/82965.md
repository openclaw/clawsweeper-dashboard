The PR adds a duplicate Gemini CLI OAuth refresh helper/hook and replaces native Google tool declarations for Gemini CLI with a ReAct-style prompt/parser path.

## Best Possible Solution

Keep the current-main OAuth refresh implementation and native Google tool-calling path, then close this redundant branch rather than merging the extra ReAct parser changes.

## Solution Assessment

No, this PR is not the best way to solve the issue because current main already has the narrow OAuth fix and the PR adds an unrelated ReAct tool parser. The best solution is to keep the current provider-owned refresh hook and native Google tool declarations.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Respect toolChoice before prompting ReAct tools:** `extensions/google/transport-stream.ts:603-604`
  - body: This branch appends ReAct tool instructions whenever Gemini CLI has tools, while the later `params.tools`/`toolConfig` block is skipped for CLI. A request with `toolChoice: "none"` would still expose tools in the prompt and the parser can turn `Action:` text into tool calls, bypassing the existing tool-disabled boundary.
  - confidence: 0.88
- **[P3] Remove the shadowed refreshOAuth hook:** `extensions/google/gemini-cli-provider.ts:122`
  - body: The new `refreshOAuth` is added before the existing current-main `refreshOAuth` property, so the later property wins and `refreshGeminiTokens` is never called. Please update the existing hook or drop the duplicate helper instead of adding dead token-refresh code.
  - confidence: 0.86
