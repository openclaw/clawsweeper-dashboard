## Summary

Keep open. Current main still routes the Slack manifest JSON through the framed setup intro note, so the underlying bug remains, but this PR should not merge as-is because the `console.log` prepare hook bypasses wizard-session delivery and can print after the env-var shortcut path.

## What This Changes

The PR exports the Slack manifest builder, removes the JSON manifest from the Slack intro note text, and prints the raw manifest from a Slack setup `prepare` hook.

## Best Possible Solution

Add a generic wizard/prompter copyable or plain-output step that emits raw manifest text in the CLI and also becomes a `WizardStep` for session clients, preserve env-shortcut suppression, update focused Slack and wizard tests, and include the required user-facing changelog entry.

## Reproduction Assessment

Yes. Current main can be verified from source: Slack setup lines include the manifest, `setup-wizard` sends those lines through `prompter.note`, and the CLI prompter delegates notes to the clack note path that frames output.

## Solution Assessment

No. Printing from Slack `prepare` is narrower than a core API change, but it bypasses the existing wizard delivery contract and cannot see `usedEnvShortcut`; a prompter-backed plain/copyable step is the safer maintainable fix.
