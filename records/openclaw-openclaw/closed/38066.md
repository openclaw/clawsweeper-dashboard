## Summary

Keep this open: current main still has no Telegram config surface for draft preview cadence, and the earlier target-repo implementation PRs were closed unmerged. The exact wording is partly stale because current Telegram streaming now uses durable message edits rather than native draft state, but the missing configurable throttle remains visible in source.

## What This Changes

The issue requests a Telegram configuration option for live preview stream cadence, plus optional consideration of a lower default cadence for smoother DM streaming.

## Best Possible Solution

Add a validated, documented Telegram preview cadence setting that follows the current nested `streaming.preview` config shape, wire it into `createTelegramDraftStream`, and preserve the existing default unless maintainers explicitly choose a new one.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current dispatch never passes `throttleMs`, so the helper uses its 1000 ms default, and the strict config schema rejects the requested cadence keys.

## Solution Assessment

Yes for the configurable-key direction, with a boundary. The maintainable fix is to add a narrow validated config surface for the current Telegram preview transport; changing the default cadence should be a separate maintainer decision.
