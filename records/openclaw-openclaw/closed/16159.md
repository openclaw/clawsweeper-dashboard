## Summary

Current main implements coordinate-based browser clicks via the browser act surface as `kind: "clickCoords"`, with x/y validation, optional button/double-click/delay handling, managed-CDP and existing-session execution paths, CLI/docs coverage, and contract tests. The implementation uses Playwright's coordinate mouse API, which maps to Chromium `Input.dispatchMouseEvent` mousePressed/mouseReleased under the hood, satisfying the issue's observable canvas/Guacamole use case even though the public action name is `clickCoords` rather than overloading `click`.

## Risks / Open Questions

- `clickCoords` does not currently accept modifier keys. The issue listed modifiers as optional enhancement, not the core blocked use case.
