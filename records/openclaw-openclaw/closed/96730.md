## Summary

Keep open for maintainer review: the patch is a plausible and well-proven Codex Computer Use fix, but it intentionally changes the default macOS managed app-server executable when Codex Desktop is installed, so the remaining question is a compatibility decision rather than an automated cleanup close.

## What This Changes

The PR makes the Codex plugin prefer `/Applications/Codex.app/Contents/Resources/codex` for macOS managed stdio app-server startup, carries plugin-local managed fallback candidates, retries stale unsupported desktop versions, and updates Computer Use docs/tests.

## Best Possible Solution

Land after a maintainer accepts desktop-first managed startup as the intended macOS Computer Use default; otherwise keep plugin-local managed startup as default and require an explicit command override for Codex.app.

## Solution Assessment
