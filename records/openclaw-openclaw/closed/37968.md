## Summary

Current `main` already carries the selected Node runtime directory into supervised service `PATH`, so an `n`-managed `node` like `~/.n/bin/node` already yields `~/.n/bin` on the service path without this PR's extra special-casing.

## Close Comment

Closing after Codex review. Current `main` already handles this path case through the generic install flow: `src/daemon/program-args.ts:189` uses the selected `nodePath` in `ProgramArguments`, `src/commands/daemon-install-plan.shared.ts:47` turns that into `path.dirname(nodePath)`, and both install paths pass it into service env construction (`src/commands/daemon-install-helpers.ts:288` and `src/commands/node-daemon-install-helpers.ts:58`). `src/daemon/service-env.ts:220` then prepends those extra runtime dirs onto the supervised service `PATH`. In practice, if the selected runtime is `~/.n/bin/node`, the service PATH already includes `~/.n/bin`, which covers the behavior requested here without adding `n`-specific special cases. This was also already shipped in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`), which contains the same install-path wiring and service PATH tests.
