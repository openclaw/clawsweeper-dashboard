## Summary

Keep open. Current main and the latest release still expose `sessionKey`, `label`, and `agentId` as independent `sessions_send` fields and still reject mixed `sessionKey` plus `label` calls before resolution, while the linked runtime/schema fix PRs remain unmerged.

## What This Changes

Request to make `sessions_send` safer by clarifying or normalizing mutually exclusive `sessionKey` and `label` addressing so redundant model calls do not fail.

## Best Possible Solution

Land one canonical fix that makes `sessionKey` authoritative over redundant `label`/`agentId` hints, updates model-facing guidance, and adds focused regression coverage.

## Reproduction Assessment

Yes. The source path in `createSessionsSendTool` returns the reported mutual-exclusion error whenever both `sessionKey` and `label` are present, before any gateway send can happen.

## Solution Assessment

Yes. The narrow maintainable fix is to prefer the more specific `sessionKey` while also clarifying schema/description guidance; the open linked PRs already target those layers.
