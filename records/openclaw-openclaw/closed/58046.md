## Summary

Current main already contains the generic plugin-registry and `/tools/invoke` reload fixes that match this report, and the changelog shows they shipped after the reporter's `2026.3.24` version in release `2026.3.31`.

## Evidence

- **Release notes match the reported symptom:** `CHANGELOG.md` has a `2026.3.31` section with two directly relevant entries: one says plugin registries were made to reuse only compatible active runtimes so plugins stop 're-registering from mismatched runtime loads', and another says HTTP `/tools/invoke` now reuses the session workspace so workspace plugins do not re-register on repeated HTTP tool calls.
  - file: [CHANGELOG.md:1678](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/CHANGELOG.md#L1678)
  - command: `nl -ba CHANGELOG.md | sed -n '1678,2056p'`
- **Loader now reuses compatible active plugin registries:** `getCompatibleActivePluginRegistry(...)` and `resolveRuntimePluginRegistry(...)` reuse the active registry when the cache key matches and refuse to recurse into an in-flight plugin load, preventing the repeated reload/reentry pattern described in the issue.
  - file: [src/plugins/loader.ts:1015](https://github.com/openclaw/openclaw/blob/0783090da0090982c19f7803a9daf9a2ec29538e/src/plugins/loader.ts#L1015)
  - command: `nl -ba src/plugins/loader.ts | sed -n '1015,1068p'`
  - sha: [0783090da009](https://github.com/openclaw/openclaw/commit/0783090da0090982c19f7803a9daf9a2ec29538e)
- **Gateway tool resolution now prefers the active gateway-bindable registry:** `resolvePluginToolRegistry(...)` returns the already-active gateway-bindable plugin registry instead of rebuilding plugin tools for each gateway tool-resolution path, which is the direct code path used by `/tools/invoke`.
  - file: [src/plugins/tools.ts:61](https://github.com/openclaw/openclaw/blob/0783090da0090982c19f7803a9daf9a2ec29538e/src/plugins/tools.ts#L61)
  - command: `nl -ba src/plugins/tools.ts | sed -n '61,105p'`
  - sha: [0783090da009](https://github.com/openclaw/openclaw/commit/0783090da0090982c19f7803a9daf9a2ec29538e)
- **End-to-end regression coverage for repeated `/tools/invoke` calls:** The gateway E2E test creates a workspace plugin that increments a registration counter, calls `POST /tools/invoke`, and asserts the counter does not increase, proving repeated HTTP tool calls no longer re-register the plugin.
  - file: [src/gateway/gateway.test.ts:221](https://github.com/openclaw/openclaw/blob/0783090da0090982c19f7803a9daf9a2ec29538e/src/gateway/gateway.test.ts#L221)
  - command: `nl -ba src/gateway/gateway.test.ts | sed -n '221,296p'`
  - sha: [0783090da009](https://github.com/openclaw/openclaw/commit/0783090da0090982c19f7803a9daf9a2ec29538e)
- **Regression coverage for plugin-load reentry:** Loader tests now assert that a plugin attempting to reenter the same snapshot load fails with `PluginLoadReentryError` instead of spinning a nested reload loop.
  - file: [src/plugins/loader.test.ts:2390](https://github.com/openclaw/openclaw/blob/0783090da0090982c19f7803a9daf9a2ec29538e/src/plugins/loader.test.ts#L2390)
