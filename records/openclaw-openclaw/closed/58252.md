## Summary

The issue targets an obsolete QClaw/Electron desktop runtime. Current `main` ships a native Swift macOS app, so the reported `QClaw Helper (GPU)` Electron helper process is not part of the current OpenClaw architecture and the report does not reproduce against current `main` as written.

## Evidence

- **Current macOS app is native:** The current macOS app docs describe OpenClaw as a native menu-bar companion that manages or attaches to the Gateway locally/remotely.
  - file: [docs/platforms/macos.md:9](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/docs/platforms/macos.md#L9)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Build target is a Swift executable, not Electron:** The macOS app is defined as a Swift Package executable target named `OpenClaw` with Swift package dependencies; there is no Electron app target in the current macOS app build definition.
  - file: [apps/macos/Package.swift:11](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/apps/macos/Package.swift#L11)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Current chat UI is SwiftUI:** The macOS WebChat docs say the app embeds WebChat as a native SwiftUI view connected to the Gateway.
  - file: [docs/platforms/mac/webchat.md:8](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/docs/platforms/mac/webchat.md#L8)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Current embedded web surface uses WebKit:** The current canvas window controller imports WebKit and constructs a `WKWebView`; the reported Electron `QClaw Helper (GPU)` process is not the current implementation.
  - file: [apps/macos/Sources/OpenClaw/CanvasWindowController.swift:41](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/apps/macos/Sources/OpenClaw/CanvasWindowController.swift#L41)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Repo history shows the macOS app transition away from QClaw branding:** The changelog records completion of the OpenClaw macOS app rename for sources, bundle identifiers, and shared kit paths.
  - file: [CHANGELOG.md:5627](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/CHANGELOG.md#L5627)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Electron is absent from current app sources:** A repo search for `Electron|electron` only surfaced a transitive `pnpm-lock.yaml` entry and no current Electron-based macOS app/runtime sources.

  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)

## Risks / Open Questions

- Current OpenClaw still uses SwiftUI and some WebKit-backed surfaces, so a fresh high-CPU bug on a current macOS build could still exist under a different process name and would need a new reproduction on the current app/version.
