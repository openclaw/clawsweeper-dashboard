## Summary

Current `main` already routes spawned sessions through the normal tool-policy pipeline. The child session key resolves back to its target agent, then both agent-level `tools.allow`/`tools.deny` and subagent-specific policy are applied before the tool surface is exposed, so the reported `tools.deny` bypass no longer matches current code.

## Evidence

- **Child session resolves target agent policy:** `resolveEffectiveToolPolicy()` derives `agentId` from the spawned `sessionKey` and then loads that agent's `tools` config, so a child key like `agent:configurator:subagent:*` is evaluated against `configurator` policy instead of bypassing it.
