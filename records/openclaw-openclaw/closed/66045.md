## Summary

Close this PR as superseded by merged #65984. Current main deliberately canonicalizes gateway service entrypoints in index.js -> entry.js order and keeps doctor repair aligned with that canonical path. The PR's alias-equivalence behavior would now reverse shipped behavior rather than fill a remaining gap.

## Best Possible Solution

Close this PR as superseded and keep the shipped #65984 design: gateway service/update/doctor paths should use the canonical index-first entrypoint resolution. A new issue should be opened only if current releases still produce an unsafe or unnecessary service rewrite after that canonicalization.
