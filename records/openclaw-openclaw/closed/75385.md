## Summary

Current main already covers the reported WebChat duplication class: chat history is display-normalized, optimistic UI messages are reconciled by signature when history catches up, duplicate in-flight sends are coalesced, and redundant delivery-mirror assistant transcript appends are skipped. The relevant WebChat duplicate-render/process fix is recorded in the v2026.4.29 changelog, which postdates the reported 2026.4.24 build.

## What This Changes

The issue asks for WebChat sends to create and render one user/assistant transcript entry instead of duplicate visible and file-backed messages.
