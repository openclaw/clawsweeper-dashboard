## What This Changes

The PR adds a gateway `resolveSessionModelRef()` post-processing helper and tests so bare selected model names can infer a unique provider from configured model families.

## Best Possible Solution

Move configured-provider inference into the shared selected-model resolver contract while preserving explicit overrides, wrapper-provider refs, stored runtime semantics, ambiguity handling, and current Codex routing policy.

## Reproduction Assessment

Yes for the PR defects: source inspection of the PR diff and current resolver path shows stored runtime providers and non-OpenAI defaults can bypass the intended configured-provider inference. No live configured OpenClaw run was performed in this read-only review.

## Solution Assessment

No. The requested behavior is useful, but a gateway-only post-processor is not the best merge shape until the shared selected-model resolver defines override, runtime, wrapper-provider, and Codex routing semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Do not let stored providers suppress bare-model inference:** `src/gateway/session-utils.ts:1073-1078`
  - body: The new normalizer returns `entry.modelProvider` before the inferred configured provider, and current sessions commonly have stored provider/model state after one run. A bare selected model like `gpt-5.4` can therefore keep routing through the old provider instead of the unique configured provider, so add coverage for stored runtime state and make the selected-model contract decide when the explicit selected model outranks stale runtime metadata.
  - confidence: 0.86
- **[P2] Infer configured providers for any default provider:** `src/gateway/session-utils.ts:1079-1084`
  - body: The inference branch only runs when the resolved provider is empty or `openai`; if an agent defaults to another provider, the same bare override still stays pinned to that default even when configured models uniquely identify a different provider. This makes the feature depend on the current default provider rather than the configured model family.
  - confidence: 0.84
