## Summary

The fix is on main after `v2026.4.26`, not in a shipped release I could prove yet.

## Best Possible Solution

Close this issue as implemented on main. Keep the current Slack `socketMode` config/runtime/docs/tests, and track broader Slack event-loop isolation or channel restart lifecycle problems in their narrower related issues rather than reopening this config request.
