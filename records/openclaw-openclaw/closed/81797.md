## Summary

Close as implemented on main: current main now emits and requires the isolated Telegram polling worker at the root `dist/telegram-ingress-worker.runtime.js` path that the loader resolves, with provenance from the same-day build-entry fix and no release tag containing it yet.

## What This Changes

The issue reports that Telegram polling crash-loops in `2026.5.12-beta.7` because the worker loader looks for `dist/telegram-ingress-worker.runtime.js` while the package only shipped the worker under `dist/extensions/telegram/`.

## Best Possible Solution

Keep the current main root dist entry and release-check contract, then let the next beta or stable package carry the fix instead of relying on a local symlink workaround.

## Reproduction Assessment

Yes for the affected beta package: the report identifies a missing `dist/telegram-ingress-worker.runtime.js` artifact while the loader resolves that sibling path. I did not rerun a global Telegram install in this read-only review; current main source and packaging checks now cover the missing artifact.

## Solution Assessment

Yes. Shipping the worker as a stable root dist entry preserves the existing loader contract and is narrower than changing Telegram runtime behavior or adding a startup workaround.
