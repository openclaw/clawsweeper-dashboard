## Summary

Current `main` no longer matches the reported WhatsApp hook gap: the WhatsApp inbound path now emits canonical `message_received` hooks behind an explicit privacy opt-in, that contract is documented/schema-backed, and dedicated regression tests cover the behavior. The change is on `main` after `v2026.4.23`, so it appears fixed but not yet in the latest tagged release.

## Risks / Open Questions

- The fix is on `main` but not in the latest tagged release `v2026.4.23`, so users on the release build may still see the old behavior until the next release.
- Current behavior is intentionally privacy-gated: WhatsApp `message_received` hooks require `channels.whatsapp.pluginHooks.messageReceived: true` (optionally per account).
