## Summary

Issue #41487 should close as superseded by open issue #63139. The remaining problem is the same `before_model_resolve` and `runWithModelFallback` boundary: fallback candidates invoke `runEmbeddedPiAgent`, which then re-runs the model-resolve hook and may make the outer fallback candidate metadata diverge from the actual hook-selected execution model. #63139 already tracks that boundary with a concrete reproduction, downstream workaround links, and proposed fixes.

## Best Possible Solution

Close #41487 as superseded by #63139 and keep the remaining fix work in the canonical issue: define the hook/fallback boundary so `before_model_resolve` is either hoisted to the logical run level or receives retry context, and ensure observable final metadata matches the actual executed provider/model or reports fallback clearly.
