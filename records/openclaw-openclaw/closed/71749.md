## Summary

Keep open. Current main still intentionally enables native Jiti for built JavaScript plugin/channel modules on Windows, including the shared helper path the reporter identified as the decisive local hotfix. I found no post-release change that fixes this loader path after v2026.4.24.

## Evidence

There is no `process.platform === "win32"` guard in the helper the report says fixed Telegram locally.
