## Summary

Keep open. The reported DeepSeek replay problem is plausible, but this PR only changes the unowned transport fallback. On current `main`, DeepSeek is a provider-owned OpenAI-compatible plugin, and provider-owned replay policy takes precedence over that fallback. The bundled DeepSeek path still enables `validateAnthropicTurns`, so the proposed change does not appear sufficient to fix the stated bug.
