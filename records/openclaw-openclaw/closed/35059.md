## Summary

Issue #35059 is a useful security idea, but it fits the ClawHub/community skill-plugin surface rather than OpenClaw core. Current main’s project scope explicitly pushes new skills to ClawHub first, and core already has built-in security-audit/code-safety hooks for installed skills and gateway-backed installs.

## Risks / Open Questions

- Current main does not pre-scan the `openclaw skills install <slug>` ClawHub download path before extraction; the existing built-in coverage is post-install audit plus gateway-backed install scanning, so the security need itself is real even if the right home is ClawHub.
- If maintainers later want registry-side moderation or a mandatory pre-install scan in core/ClawHub service, this issue’s scope decision may need to be revisited.
