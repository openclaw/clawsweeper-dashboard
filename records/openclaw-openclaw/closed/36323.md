## Summary

Current main already has the Telegram delivery-layer formatting gate this issue asks for: Telegram send, edit, and preview paths resolve markdown tables for the Telegram channel and render code-mode tables as Telegram `<pre><code>...</code></pre>` blocks. The same implementation is present in the shipped `v2026.4.22` release.

## Evidence

- **Shared table-mode resolver defaults unsupported channels to code mode:** `resolveMarkdownTableMode` falls back to `"code"` when a channel has no special override, which covers Telegram in the current registry-backed implementation.
  - file: [src/config/markdown-tables.ts:92](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/src/config/markdown-tables.ts#L92)
  - sha: [7d4718373698](https://github.com/openclaw/openclaw/commit/7d471837369885c12d540d6bafa319ff41129b89)
- **Code mode renders markdown tables as code blocks:** The markdown IR renderer routes `tableMode === "code"` to `renderTableAsCode`, producing code-block output instead of raw pipe-table markdown.
  - file: [src/markdown/ir.ts:718](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/src/markdown/ir.ts#L718)
  - sha: [7d4718373698](https://github.com/openclaw/openclaw/commit/7d471837369885c12d540d6bafa319ff41129b89)
- **Telegram HTML renderer maps code blocks to `<pre><code>`:** Telegram formatting renders `code_block` spans with `<pre><code>...</code></pre>`, which is the Telegram-safe rendering the issue requested.
  - file: [extensions/telegram/src/format.ts:66](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/extensions/telegram/src/format.ts#L66)
  - sha: [7d4718373698](https://github.com/openclaw/openclaw/commit/7d471837369885c12d540d6bafa319ff41129b89)

  - sha: [7d4718373698](https://github.com/openclaw/openclaw/commit/7d471837369885c12d540d6bafa319ff41129b89)
