## Summary

Current main already routes isolated cron agentTurn provider failures through the shared model fallback loop. Cron calls runWithModelFallback with payload fallbacks, embedded provider failures now throw FailoverError instead of returning ordinary meta.error payloads, and meta.error is limited to local terminal states that cron should surface rather than use for model fallback.

## Best Possible Solution

Close as implemented. Keep the shared embedded-runner FailoverError path and cron runWithModelFallback integration; only revisit cron-local meta.error normalization if a future provider-failover state is intentionally added to EmbeddedPiRunMeta.error.
