## Summary

Current `main` already treats sandboxing as explicit/optional rather than silently forcing all skills into sandboxes: the resolved default mode is `off`, the Docker setup path only turns on `non-main` sandboxing when explicitly requested and rolls back to `off` on failure, and doctor/docs now surface sandbox status plus skill-requirement summaries.

## Risks / Open Questions

- `doctor`'s skill-status check is host/eligibility based; it does not by itself prove every custom sandbox image contains the same binaries as the host. That said, the issue's reported default-all sandbox regression is no longer present on current main.
