## Summary

Current main already contains a targeted fix for transient auto-failover model overrides and ships it in `v2026.4.22`, so this issue can be closed as implemented.

## Evidence

- **Primary model is explicitly restored before each turn:** `createModelSelectionState` now clears auto-failover session overrides and resets in-memory selection back to the configured primary model before running the turn.
  - file: [src/auto-reply/reply/model-selection.ts:173](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/src/auto-reply/reply/model-selection.ts#L173)
  - command: `nl -ba src/auto-reply/reply/model-selection.ts | sed -n '170,205p'`
- **Regression test covers this exact behavior:** A unit test asserts that an auto-failover override is cleared and the configured primary model is retried instead of staying pinned to the fallback model.
  - file: [src/auto-reply/reply/model-selection.test.ts:635](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/src/auto-reply/reply/model-selection.test.ts#L635)
  - command: `nl -ba src/auto-reply/reply/model-selection.test.ts | sed -n '630,690p'`
- **Shipped release note:** The changelog for `2026.4.22` documents: "Agents/model selection: clear transient auto-failover session overrides before each turn so recovered primary models are retried immediately..."
  - file: [CHANGELOG.md:399](https://github.com/openclaw/openclaw/blob/f3c37a946cc49d26e5fe65a1043f7286a86aa4a4/CHANGELOG.md#L399)
