## Summary

Keep open: the PR has a useful Feishu topic-anchor direction and sufficient live proof, but the current head reintroduces the normal-group `root_id` reply regression that current main explicitly guards against.

## What This Changes

The PR changes Feishu/Lark routing so topic replies, message-tool sends, reactions, media/cards, direct-message topics, and subagent completion notices preserve separate inbound, root/thread, and merged-forward anchors.

## Best Possible Solution

Keep the topic-anchor split, but restrict root/thread send targets to topic, direct-topic, or explicit `replyInThread` paths while preserving normal-group `root_id` replies on the triggering message.

## Reproduction Assessment

Yes, source-reproducible: current main has a focused normal-group `root_id` regression test, and the PR branch changes that expectation from the triggering message to the root message. I did not run tests because this was a read-only review.

## Solution Assessment

No, not as-is. Splitting typing/current-message anchors from topic send anchors is the right boundary, but `threadedReply` should not be true for every plain group event that happens to carry `root_id`.
