## Summary

The reported Feishu symptom is already addressed on current `main`: routed final replies are counted as successful sends before the Feishu channel logs `dispatch complete`, so the old `queuedFinal=false, replies=0` outcome for a successful routed reply is obsolete.

## Evidence

- **Routed final replies now count as sent:** When `routeReplyToOriginating(...)` succeeds, `dispatchReplyFromConfig` returns `queuedFinal: result.ok` and increments `routedFinalCount` by 1.
  - file: [src/auto-reply/reply/dispatch-from-config.ts:665](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/src/auto-reply/reply/dispatch-from-config.ts#L665)
  - sha: [7d4718373698](https://github.com/openclaw/openclaw/commit/7d471837369885c12d540d6bafa319ff41129b89)
- **Returned reply counts include routed sends:** Before returning, the dispatcher adds routed final sends into the public result via `counts.final += routedFinalCount`.
  - file: [src/auto-reply/reply/dispatch-from-config.ts:1144](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/src/auto-reply/reply/dispatch-from-config.ts#L1144)
  - sha: [7d4718373698](https://github.com/openclaw/openclaw/commit/7d471837369885c12d540d6bafa319ff41129b89)
- **Feishu log reads the fixed count:** The Feishu handler logs `replies=${counts.final}` from the `dispatchReplyFromConfig` result, so with the current counting logic a successful routed final reply no longer logs as `replies=0`.
  - file: [extensions/feishu/src/bot.ts:1238](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/extensions/feishu/src/bot.ts#L1238)
  - sha: [7d4718373698](https://github.com/openclaw/openclaw/commit/7d471837369885c12d540d6bafa319ff41129b89)
- **Shipped release contains the same fix path:** The same routed-final counting logic is present in release tag `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`), so this was already shipped by that release.
