## Summary

Keep open. Current main still relies on the raw launcher shebang and respawn path rather than resolving or avoiding a Volta shim in non-interactive Linux contexts. The related #52184 is open but scoped to macOS LaunchAgent path selection and does not supersede this raw CLI/systemd report.

## Best Possible Solution

Keep #68672 open. The likely maintainer path is to either make the raw CLI launcher/respawn path robust under Volta-managed non-interactive shells or document wrapper-based execution such as `/opt/openclaw/bin/openclaw-env openclaw ...` as a supported deployment pattern, while preserving the reporter's warning that doctor repair must not blindly rewrite custom SOPS-loading systemd units.
