## Summary

Current `main` routes OpenClaw-owned tool failures through a new Pi tool-definition adapter that converts thrown tool exceptions into structured error tool results, and the embedded runner now uses that adapter for all custom tools. The image tool still throws provider/model failures, so on current `main` those failures are propagated back as tool errors instead of escaping and hanging the turn. The specific `AbortError: Request was aborted` shape from the report is also classified as a timeout/failover case rather than a user abort.
