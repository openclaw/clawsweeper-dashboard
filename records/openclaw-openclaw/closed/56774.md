## Summary

Keep open. Current `main` still has a live code path where `read` image handling depends on a dynamic `import("sharp")`, and the inspected history/tests do not provide strong evidence that the reported desktop packaging/runtime-missing-`sharp` failure was fixed on `main`. This is not safe to close as implemented or non-reproducible from read-only inspection alone.

## Evidence

- **Read tool always runs image sanitization after image reads:** `createOpenClawReadTool()` normalizes image reads and then always calls `sanitizeToolResultImages(...)` on the read result.
  - file: [src/agents/pi-tools.read.ts:674](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/agents/pi-tools.read.ts#L674)
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Sanitization can invoke JPEG resize for read images:** `sanitizeContentBlocksImages()` calls `resizeImageBase64IfNeeded()`, which in turn calls `resizeToJpeg(...)` for oversized or dimension-unknown images.
  - file: [src/agents/tool-images.ts:148](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/agents/tool-images.ts#L148)
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Image ops still dynamically import sharp at runtime:** `loadSharp()` is a raw `import("sharp")` dynamic import, so a missing runtime package would still surface on this path.
  - file: [src/media/image-ops.ts:37](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/media/image-ops.ts#L37)
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)

  - file: [src/media/image-ops.ts:504](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/media/image-ops.ts#L504)

  - file: [src/agents/tool-images.test.ts:1](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/agents/tool-images.test.ts#L1)
