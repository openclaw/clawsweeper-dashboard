## Summary

Keep open. Current main still does not implement the PR's central behavior: subagent completion output selection returns the latest assistant text before raw tool-result text, so exec stdout/stderr can still be lost when the final assistant summary does not echo it. The PR has review tradeoffs around run scoping and chained tool calls, but it is a focused bug-fix PR tied to #57965 and is not obsolete or superseded by current main.

## Best Possible Solution

Keep the PR open or ask for a revised patch. The best fix belongs in `src/agents/subagent-announce-output.ts`: include current completion-run toolResult/tool text with the final assistant summary, handle nested `AgentToolResult.content` arrays, and add regression tests for exec toolCall -> toolResult -> assistant summary, multiple exec results, nested content, and stale-output scoping across reused child sessions.
