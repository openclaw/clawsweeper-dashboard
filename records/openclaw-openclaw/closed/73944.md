## Summary

The request is useful, but current OpenClaw policy and extension surfaces make it a ClawHub/code-plugin fit rather than a core default. Main has no built-in CJK/Latin path-space auto-correction, and the existing `before_tool_call` hook can rewrite tool parameters before execution for an opt-in implementation.

## Best Possible Solution

Close this core issue and encourage a ClawHub code plugin that registers `before_tool_call`, detects configured path parameters or exec command strings, and optionally rewrites only when a safer validation rule passes, such as the original path missing and the corrected path existing.
