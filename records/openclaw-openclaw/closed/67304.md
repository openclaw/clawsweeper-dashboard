## Summary

Keep open. Current main still lacks `OPENCLAW_JITI_CACHE_DIR` support, but this PR should not merge as-is because it forces `fsCache: true` when the OpenClaw override is unset, bypasses existing Jiti env semantics, lacks cache-key/docs coverage, and is stale against current Matrix/runtime loader refactors.

## What This Changes

The PR adds an env-driven Jiti `fsCache` override in the shared plugin loader, SDK root alias, Matrix runtime wrapper, and disables `fsCache` in one git-path regression fixture.

## Best Possible Solution

Land a narrow current-main implementation that only supplies a trimmed OpenClaw cache directory when non-empty, otherwise leaving Jiti defaults untouched, with cache isolation, focused tests, and operator-facing docs/release notes.

## Reproduction Assessment

Yes. Apply the PR, leave `OPENCLAW_JITI_CACHE_DIR` unset, set `JITI_FS_CACHE=false`, and the added fallback still passes `fsCache: true` to `createJiti()`, overriding Jiti 2.6.1's env-derived default.

## Solution Assessment

No. The requested override is reasonable, but this implementation is not the best fix because preserving existing Jiti defaults requires omitting `fsCache` unless the OpenClaw override is explicitly set.
