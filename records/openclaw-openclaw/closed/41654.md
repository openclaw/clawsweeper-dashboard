## Summary

Current main implements the concrete failure path reported in #41654: Anthropic thinking-block modification rejections are recognized and recovered before user-visible delivery, replay sanitization preserves/strips thinking blocks according to provider policy, and subagent announce delivery error handling no longer turns caught delivery errors into channel content. The cross-referenced canonical report #40579 is closed, and the replay safety fix is reflected in the v2026.4.24 changelog.

## Best Possible Solution

Keep the shipped Anthropic replay/recovery behavior and current subagent announce delivery error handling. If a different provider error still appears as a visible channel message, it should be tracked as a narrower new bug with provider, model, channel, logs, and reproduction details rather than keeping this already-fixed thinking-block report open.
