## Summary

The reported `apiKey: false` rejection is not an OpenClaw core contract bug. Current `main` validates `plugins.entries.<id>.config` against each plugin's own manifest-declared `configSchema`, and OpenClaw's Honcho docs already treat `@honcho-ai/openclaw-honcho` as an external plugin whose self-hosted setup should omit `apiKey` entirely. If Honcho wants `false`/`null` as explicit sentinels or different warning suppression, that belongs in the Honcho plugin package/ClawHub surface, not OpenClaw core.

## Evidence
