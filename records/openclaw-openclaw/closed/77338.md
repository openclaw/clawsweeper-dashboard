## Summary

Keep this open: current main and v2026.6.10 still use the strict `.jsonl` session-status denominator, while the current session corpus/indexing path uses the usage-counted transcript predicate. The open linked PR is the active fix candidate, so this issue should close only after that or an equivalent fix lands.

## What This Changes

The issue requests fixing memory status session file counts so reset/deleted session archives and live session indexing use one consistent session-file predicate.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/95452 or an equivalent predicate-alignment fix, then close this issue once the denominator fix reaches main and split any remaining dirty-state concern into a narrower follow-up.

## Reproduction Assessment

Yes at source level: current main renders memory status using a strict `.jsonl` denominator while the current session corpus/indexing predicate includes reset/deleted archives. I did not run a live CLI repro in this read-only review.

## Solution Assessment

Yes for the central counter overflow: reusing the existing usage-counted session transcript predicate in `scanSessionFiles()` is the narrowest maintainable fix. Skipping reset/deleted archives in the indexer would conflict with the current archive recall/usage contract and belongs in a separate product discussion if desired.
