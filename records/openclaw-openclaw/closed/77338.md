## Summary

Keep open: current main still has the sessions status counter mismatch, while reset/deleted archive recall is now intentional in source and tests. The counter can be fixed narrowly, but the requested archive-skipping/privacy direction needs maintainer policy judgment rather than cleanup closure.

## What This Changes

The issue asks OpenClaw memory status and indexing to stop counting reset/deleted session archive artifacts as live sessions and also reports a possible dirty-state race after forced indexing.

## Best Possible Solution

Align memory status with the same usage-counted session transcript contract used by indexing, and leave archive recall, deletion, and retention semantics to the dedicated policy thread.

## Reproduction Assessment

Yes for the counter mismatch by source inspection: current main and v2026.5.12 scan only filenames ending in .jsonl for status totals while the indexer lists usage-counted reset/deleted archives. The dirty-state race is unclear because this read-only review did not run a live index cycle.

## Solution Assessment

No for the requested archive-skipping direction: current code and tests intentionally keep reset/deleted archives searchable. The maintainable fix is to align the status denominator with the indexer contract, while deciding privacy and retention policy separately.
