## Summary

Current `main` already removes dreaming from outbound channel delivery: the managed dreaming cron runs as an isolated maintenance `agentTurn` with `delivery.mode: "none"`, and the Dream Diary subagent run uses `deliver: false`. Tests and docs on `main` match that behavior, so the Feishu `to: "default"` push path described in the report is no longer present on `main`.

## Evidence
