## Summary

Close as implemented on current main. The media model-config resolver now accepts lazy provider sources, the image/video/music helpers pass provider thunks, and regression coverage verifies explicit model configs no longer load runtime providers; the fix landed after v2026.5.2, so it is main-only for now.

## What This Changes

The issue requests deferring image/video/music provider listing during explicit model-config resolution to avoid repeated plugin registry reloads.

## Best Possible Solution

Keep the merged lazy-provider implementation on main and let the now-redundant paired PR be handled as superseded in normal PR triage.

## Reproduction Assessment

No against current main. Source inspection and regression tests show complete explicit image/video/music model configs return without invoking the runtime provider lists that previously triggered plugin reloads.

## Solution Assessment

Yes. Deferring providers behind a thunk in the shared resolver is the narrow maintainable fix because it avoids unused provider discovery only on the explicit-config path while preserving fallback provider discovery when modelConfig is incomplete.
