## Summary

This should close as a duplicate of the older open Feishu audio-duration report. Current main still has the underlying missing-duration behavior, but #53798 already tracks the same Feishu voice-message duration problem with clearer reproduction and linked PR context, so keeping both issues open would split the same remaining work.

## What This Changes

The issue reports that Feishu TTS/audio replies call `uploadFileFeishu` without `duration`, causing Feishu voice messages to display 0 seconds.

## Best Possible Solution

Keep the canonical #53798 thread open and fix the bundled Feishu plugin there with a narrow patch that preserves existing media routing/download metadata, computes a finite millisecond duration for native Opus/Ogg and successful voice-transcode uploads, and passes it through `uploadFileFeishu`.

## Reproduction Assessment

Yes. A focused mock reproduction is to call `sendMediaFeishu` with Ogg/Opus or TTS voice media and inspect that `client.im.file.create` receives no `duration`; #53798 provides equivalent user-facing message-tool steps against the same code path.

## Solution Assessment

Yes. Closing this duplicate is the clean maintainer cleanup; the code solution should remain in the canonical Feishu audio-duration thread or a replacement for #50399, not split across duplicate issues.
