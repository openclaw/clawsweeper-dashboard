## Summary

Keep open. Current main still suppresses the Telegram answer draft stream for selected/native quote replies, which makes tool-progress previews no-op without a docs or doctor warning; open closing PRs #73495 and #73508 should be reviewed rather than closing this issue early.

## What This Changes

Request to surface or fix Telegram tool-progress preview suppression when `channels.telegram.replyToMode` uses native quote replies.

## Best Possible Solution

Land one canonical Telegram-scoped clarification, preferably docs plus a matching doctor warning; leave a persistent separate-message fallback for separate behavior work.

## Reproduction Assessment

Yes. The source-backed path is to enable Telegram preview streaming/tool-progress, set non-off `replyToMode`, and use a selected/native quote so `ReplyToQuoteText` is present; current code then skips the answer draft stream that renders tool-progress previews.

## Solution Assessment

Yes for the near-term direction. A Telegram docs caveat plus doctor warning is the narrowest maintainable fix for the silent mismatch; a runtime fallback to separate persistent messages is broader behavior work.
