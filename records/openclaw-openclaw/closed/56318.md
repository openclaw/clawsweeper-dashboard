## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve host-read local media compatibility:** `src/media/web-media.ts:277-285`
  - body: Removing the `localRoots: "any"` compatibility path makes existing host-read and plugin SDK media callers fall back into the root guard instead of the explicit read-file path. Current `main` still documents and tests that escape hatch, so this needs a compatibility-preserving migration rather than a hard removal.
  - confidence: 0.89
- **[P1] Do not treat backpressure as output closure:** `src/terminal/stream-writer.ts:49-51`
  - body: `stream.write()` returning `false` only means backpressure, but callers such as `openclaw logs` stop tailing when the writer returns false. This change can truncate healthy high-volume output instead of waiting for drain or preserving the old success semantics.
  - confidence: 0.93
- **[P1] Keep exec resolver environment explicit:** `src/secrets/resolve.ts:701-710`
  - body: Docs and current source make exec provider environments minimal unless a key is listed in `passEnv` or provider `env`. Adding HOME, USER, TERM, and temp variables by default changes that security boundary and can leak environment details to every configured exec resolver.
  - confidence: 0.87
- **[P1] Preserve approved local shell command semantics:** `src/tui/tui-local-shell.ts:120-121`
  - body: The existing TUI local shell path intentionally runs through a shell after an in-session operator approval prompt. Parsing to argv with `shell: false` breaks approved commands that use pipes, redirects, globs, or environment expansion, so this needs either a product-approved behavior change with proof or a different injection mitigation.
  - confidence: 0.85
- **[P2] Bypass stale directory caches for session file resolution:** `src/infra/session-cost-usage.ts:297-308`
  - body: Caching `readdirSync` results here can return a stale primary session file after transcript reset or rotation, causing usage/session lookup to choose a path that no longer exists until the TTL expires. This resolver should read fresh directory state or prove the cache is invalidated on file churn.
  - confidence: 0.77
- **[P1] Drop the already-shipped cost cache rewrite:** `src/gateway/server-methods/usage.ts:63-64`
  - body: Current `main` already bounds `costUsageCache` with a tested 256-entry helper that prefers evicting settled entries before in-flight entries. Replacing it with a new `LruMap(100)` duplicates the shipped fix and loses that in-flight-aware eviction behavior.
  - confidence: 0.84
