## Summary

Closing because the report targets a user-installed `openclaw-lark` manifest/config shape, while current OpenClaw already documents and enforces the generic third-party channel manifest contract.

## What This Changes

The issue asks how to restore Feishu/Lark loading for `openclaw-lark` by adding `channelConfigs` metadata for `channels.feishu` after upgrading to OpenClaw 2026.5.2.

## Best Possible Solution

Keep the core contract generic; update and release the external `openclaw-lark` manifest with inline `channelConfigs.feishu.schema` JSON Schema, or switch the install to the official `@openclaw/feishu` plugin.

## Reproduction Assessment

Partially: source inspection shows current main emits the exact `without channelConfigs metadata` warning for active non-bundled channel manifests missing `channelConfigs`. The reported load failure depends on the external plugin package and local `plugins.allow` config, which are not in this repository.

## Solution Assessment

Yes for repo cleanup: the narrow maintainable fix is in the external plugin manifest and local plugin allowlist, not a core migration for one third-party package. The manifest should use inline JSON Schema under `channelConfigs`, while `plugins.allow` should name the installed plugin id.
