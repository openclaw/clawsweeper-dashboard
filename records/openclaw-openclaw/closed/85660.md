- `impact:security`: The reported symptom involved a Copilot token being sent to the wrong provider endpoint.
- `impact:auth-provider`: The affected behavior is provider/model routing and auth-token selection for `agents.defaults.imageModel`.

## Summary

Current main and the latest release already keep unknown `github-copilot/*` image model refs on the GitHub Copilot provider path, including synthetic vision-capable Gemini model definitions, so the reported OpenAI-provider fallback should not remain open as a core bug.

## What This Changes

Setting an unknown `github-copilot/*` `agents.defaults.imageModel` should route through GitHub Copilot via forward-compatible model resolution or fail with a provider-specific error.

## Best Possible Solution

Keep the shipped plugin-owned dynamic model resolver and Copilot transport metadata, and ask for a fresh issue only if the latest release still sends a Copilot token to a non-Copilot endpoint.

## Reproduction Assessment

No current-main failing path was reproduced. Source inspection shows the configured `github-copilot/*` image model stays provider-prefixed and reaches the Copilot dynamic resolver before auth is prepared.

## Solution Assessment

Yes. The current owner-local fix keeps Copilot model discovery and forward-compatible routing inside the GitHub Copilot plugin instead of adding core special cases or a new config knob.
