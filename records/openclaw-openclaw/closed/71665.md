## Summary

Current main already implements Slack native slash command handling for explicit native mode. The issue targets v2026.4.23 behavior, but main registers Slack Bolt command handlers, acknowledges slash payloads, parses command text, dispatches through the gateway command path, and documents the Slack configuration. Slack `auto` mode intentionally remains off, so explicit `channels.slack.commands.native: true` is the supported path.
