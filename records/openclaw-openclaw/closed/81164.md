## Summary

## What This Changes

The PR adds an optional `ContextEngine.interceptCompaction()` contract, Pi extension wiring, compaction settings changes, plugin SDK exports, docs, changelog, and tests for context-engine plugins that want to replace Pi auto-compaction summaries.

## Best Possible Solution

Land the additive intercept API only after the handler is bounded by the existing compaction timeout/abort pattern, maintainer API review approves the compatibility semantics, conflicts are resolved, and redacted live output shows an opted-in engine handling a real `session_before_compact` event.

## Reproduction Assessment

Not applicable as a current-main bug; this PR proposes a new plugin API and runtime path. The merge-blocking availability issue is source-reproducible from the direct `await engine.interceptCompaction(request)` call inside a Pi handler that the upstream runner awaits sequentially.

## Solution Assessment

No as currently proposed: the intercept seam is plausible, but the maintainable version should bound the plugin callback with the same finite timeout and abort policy already used for plugin-owned compaction, then prove the live handled-event path.
