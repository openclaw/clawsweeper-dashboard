The PR updates self-hosted OpenAI-compatible provider discovery to read llama.cpp runtime context from `/props`, map it to `contextTokens`, preserve native `contextWindow` metadata, and add tests plus a changelog entry.

## Best Possible Solution

Keep the merged current-main implementation and ship it in the next release so self-hosted llama.cpp sessions budget against the runtime context cap while preserving native model metadata.

## Reproduction Assessment

Yes. The original failure path is clear from the linked report, current pre-merge helper behavior, and llama.cpp docs: `/v1/models` exposes `n_ctx_train` while `/props` exposes the runtime `n_ctx`; current main now has regression tests for that path.

## Solution Assessment

Yes. The merged solution is narrow: it keeps `n_ctx_train` as native `contextWindow`, sets `contextTokens` from `/props`, preserves explicit configured `contextWindow`, and avoids router autoload side effects with `autoload=false`.
