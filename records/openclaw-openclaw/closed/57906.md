## Summary

Current `main` already limits overloaded same-provider retries and advances through the fallback chain, with documented operator knobs and regression coverage. The latest release tag `v2026.4.23` also contains this behavior.

## Evidence

- **Overload retry defaults now favor fast fallback:** `src/agents/pi-embedded-runner/run/helpers.ts:32` sets `DEFAULT_OVERLOAD_FAILOVER_BACKOFF_MS = 0` and `DEFAULT_MAX_OVERLOAD_PROFILE_ROTATIONS = 1`, so overloaded requests do not sit behind a built-in 30-60s retry/backoff loop before fallback.
  - file: [src/agents/pi-embedded-runner/run/helpers.ts:32](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/agents/pi-embedded-runner/run/helpers.ts#L32)
- **Overloaded assistant failures escalate to model fallback after the cap:** `src/agents/pi-embedded-runner/run/assistant-failover.ts:116` increments overload rotations and, once the configured cap is exceeded, throws a `FailoverError` with reason `overloaded` to the outer model-fallback path instead of continuing to hammer the same primary.
  - file: [src/agents/pi-embedded-runner/run/assistant-failover.ts:116](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/agents/pi-embedded-runner/run/assistant-failover.ts#L116)
- **Model fallback avoids repeated same-provider cooldown probes:** `src/agents/model-fallback.ts:809` explicitly limits transient cooldown probing to once per provider per fallback run, with the inline comment at `:810-812` noting that re-probing every same-provider candidate can stall cross-provider fallback.
  - file: [src/agents/model-fallback.ts:809](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/agents/model-fallback.ts#L809)
- **Regression tests cover overloaded-primary to fallback progression:** `src/agents/model-fallback.probe.test.ts:388` asserts that non-primary fallbacks are attempted after an overloaded primary probe failure, and `src/agents/model-fallback.test.ts:1653` asserts same-provider fallbacks are attempted during overloaded cooldown.
  - file: [src/agents/model-fallback.probe.test.ts:388](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/src/agents/model-fallback.probe.test.ts#L388)
- **Config surface matches the requested tunability:** `docs/gateway/configuration-reference.md:751` documents `auth.cooldowns.overloadedProfileRotations` and `auth.cooldowns.overloadedBackoffMs`, which provide the operator control requested in the issue comment/suggestion.
  - file: [docs/gateway/configuration-reference.md:751](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/docs/gateway/configuration-reference.md#L751)
