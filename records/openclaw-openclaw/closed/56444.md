## Summary

Current main already routes WhatsApp media sends through the normal message-send path, the WhatsApp channel still declares media support, and current docs/tests cover WhatsApp media delivery. This matches the report closely enough to treat it as already implemented/fixed on main.

## Close Comment

Closing after a Codex review because current `main` already implements this path. The agent message tool routes through `runMessageAction` (`src/agents/tools/message-tool.ts:807`), outbound send handling routes media sends through the channel/plugin send path (`src/infra/outbound/outbound-send-service.ts:143`), and the runtime adapter calls `sendMedia(...)` whenever media is present (`src/cli/send-runtime/channel-outbound-send.ts:60`). The changelog also records the shipped fix in `v2026.4.10` (`CHANGELOG.md:896`).
