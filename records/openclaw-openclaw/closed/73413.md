## Summary

Current main already implements the subagent/yield/compaction behavior needed for the reported 2026.4.14 failure mode. Compaction retries now clear buffered interim output and wait for retry completion, yielded subagent completion delivery has a direct fallback when the dormant parent announce turn produces no visible reply, and regression/QA coverage verifies yielded subagent fallback plus avoiding placeholder/intermediate child results.

## Best Possible Solution

Keep the shipped v2026.4.26 subagent/yield delivery and compaction-retry implementation. If a current release still loses the final child output after compaction, track it as a new, narrower regression with logs and run ids against the current code path.
