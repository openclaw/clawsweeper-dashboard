## Summary

Keep open: current main and v2026.6.10 still let the gateway rotate provider-owned CLI sessions across the implicit daily boundary, while docs and the inbound path define those sessions as exempt; the PR is a focused fix with sufficient runtime proof and no blocking findings.

## What This Changes

The PR exports the existing provider-owned CLI session predicate, applies it to both gateway agent freshness decisions, and adds a gateway regression test for the implicit daily reset boundary.

## Best Possible Solution

Land the focused gateway alignment after normal required CI finishes, keeping explicit reset policies and /reset behavior unchanged.

## Reproduction Assessment

Yes. Source inspection shows current main and v2026.6.10 evaluate gateway session freshness directly at the daily boundary, then rotate and clear CLI bindings, while the docs and inbound path define provider-owned CLI sessions as exempt from implicit daily reset.

## Solution Assessment

Yes. Reusing the existing provider-owned predicate at both gateway freshness decisions is the narrowest maintainable fix; routing through the full entry-freshness loader would fight the gateway's already-loaded and store-locked entry flow.
