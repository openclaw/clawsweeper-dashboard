## Summary

Keep this PR open. Current main still omits `sessionKey` when `memory-wiki` calls shared memory search, while the shared manager contract and QMD implementation both support and require session context for scoped QMD searches.

## What This Changes

This PR forwards `agentSessionKey` as `sessionKey` in `memory-wiki` shared memory search and adds a regression test for that call shape.

## Best Possible Solution

Land a narrow fix that forwards the existing `agentSessionKey` into the shared memory search call, keep QMD scope policy unchanged, and retain a focused regression test proving `wiki_search` passes the session context.
