## Summary

Close this issue as superseded by the active Google Chat thread-routing work in open PR #74235. Current main still matches the reported invalid-thread failure mode, but #74235 now directly covers the remaining fix with inbound thread preservation, outbound thread normalization, and a shared Google Chat thread-resource validator; #67055/#42510 and #69422 cover narrower adjacent symptoms.

## What This Changes

The issue asks the Google Chat plugin to stop forwarding raw message, session, or action ids as `thread.name`, omit or normalize invalid thread values, and honor `replyToMode: "off"` for flat DM replies.

## Best Possible Solution

Use #74235, or an equivalent maintainer-approved replacement, as the canonical path to land one Google Chat-owned thread-routing helper that validates `spaces/{space}/threads/{thread}` resources, preserves inbound thread resources separately from message ids, suppresses threading when configured off, and covers natural replies, streamed/block replies, media, attached results, and message actions.

Yes.
