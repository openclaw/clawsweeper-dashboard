## Summary

PR #65275 adds a new service-specific Preferred Networks/PLaMo model provider, but current project scope routes optional provider integrations to external native plugins/ClawHub unless there is a clear missing core API or maintainer product decision. Current main does not already bundle PLaMo, and the PR discussion is useful, but the provider SDK already exposes the needed provider hooks and transport helpers for an external plugin path.

## Best Possible Solution

Move the PLaMo provider implementation to an external native provider plugin and publish it through ClawHub/npm. Keep only any narrowly demonstrated, provider-agnostic SDK or transport fixes in OpenClaw core as separate reviewed PRs with focused tests.
