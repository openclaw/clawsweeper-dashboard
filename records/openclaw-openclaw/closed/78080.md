## Summary

Keep open: current main still has a source-backed silent-drop path for ordinary Discord guild text control commands, and the reporter's follow-up identifies a second text-command unauthorized path that suppresses a visible reply.

## What This Changes

Request that Discord guild text slash/control commands like `/steer ...` route through text command handling or return a visible rejection instead of disappearing.

## Best Possible Solution

Keep the native Discord interaction echo drop, but let ordinary guild text control commands reach command handling and produce visible Discord replies for unauthorized, unavailable, or idle-session outcomes.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: call preflightDiscordMessage for a normal non-DM guild message with `/steer test` and it returns null at the early hasControlCommand guard; if that guard is bypassed, unauthorized text commands still return shouldContinue:false without a reply.

## Solution Assessment

Yes. The narrow fix is to distinguish actual Discord native command echo messages from ordinary text and make text-command denial/help paths visible without adding new config.
