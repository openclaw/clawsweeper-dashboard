## Label Justifications

- `P2`: The report describes a normal-priority config preservation bug with limited blast radius.
- `impact:data-loss`: The issue is about silently losing persisted per-agent workspace fields from config.
- `impact:session-state`: The reported workspace loss can make agents load the wrong workspace-scoped instruction files.

## Summary

Current main already uses source-config snapshots for config reload planning and persisted writes, so a model-default hot reload should not rewrite away untouched per-agent workspace entries.

## What This Changes

The issue asks OpenClaw to preserve `agents.list[].workspace` when a config hot reload follows changes such as `agents.defaults.model.primary`.

## Best Possible Solution

Keep the existing source-config-based reload/write path; if a narrower writer still drops workspaces, track it with an exact command or RPC payload and before/after config.

## Reproduction Assessment

No: source inspection did not find a current-main failing path for a model-default hot reload. The current code compares and notifies with source config, and targeted config patches merge by agent id rather than replacing the whole agent list.

## Solution Assessment

Yes: preserving authored source config through reload planning and persisted writes is the narrow maintainable solution for this class of field-drop bug.
