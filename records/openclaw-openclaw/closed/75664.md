## What This Changes

The issue reports `openclaw wiki status --json` and `openclaw wiki doctor --json` hanging through the top-level CLI even though the memory-wiki helpers complete directly.

## Best Possible Solution

Keep the shipped generic CLI lifecycle cleanup; any new hang on current releases should be reported with exact version, TTY/stdin context, and redacted debug output.

## Reproduction Assessment

Yes for the original bug shape: the issue gives concrete top-level commands, and current source shows those commands return through the shared CLI finalizer. I did not live-run the old hang in this read-only review, but the retained-stdin exit path is now covered and shipped.

## Solution Assessment

Yes. The maintainable fix is the shared CLI exit cleanup rather than adding a memory-wiki-specific forced exit, because the reported helpers already completed and the hang was in the wrapper lifecycle.
