## Summary

Current main already covers the startup-warmup performance problem with a refactored gateway path: startup model prewarm is opt-out via an equivalent env flag, bounded to 5 seconds, scoped to the configured provider, and scheduled without blocking channel startup. The PR’s review feedback was addressed on the branch, but the branch is now obsolete against the current gateway startup implementation.

## What This Changes

The PR adds an opt-in `OPENCLAW_SKIP_MODEL_WARMUP` env check to skip gateway startup primary-model warmup and registers that env var in gateway test helpers.

## Best Possible Solution

Close this PR and keep the current main implementation: use the existing `OPENCLAW_SKIP_STARTUP_MODEL_PREWARM` opt-out plus bounded, background, provider-scoped warmup. If maintainers want the shorter env alias for compatibility with the PR branch, track that as a small follow-up rather than merging this stale branch.
