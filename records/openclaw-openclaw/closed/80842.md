## What This Changes

Request better DingTalk app/process handling and clearer user reminders when OpenClaw opens a browser login instead of reading the DingTalk desktop app.

## Best Possible Solution

Keep DingTalk-specific app/process/login diagnostics in the external DingTalk plugin or ClawHub package, and only add a generic SDK/setup seam to core if that plugin hits a documented missing extension point.
