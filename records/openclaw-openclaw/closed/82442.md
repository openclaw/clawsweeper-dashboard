## What This Changes

The issue reports that the debug proxy CONNECT tunnel lacks a client socket error handler, so abrupt client disconnects can crash the proxy or leave the upstream socket uncleaned.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82444, or an equivalent narrow fix, so the CONNECT tunnel handles errors from both sockets and preserves diagnostic capture without crashing.

## Reproduction Assessment

Yes at source level: current main and v2026.5.18 still lack a clientSocket error listener after CONNECT tunneling, and Node throws unhandled error events. I did not run the full live kill-client repro.

## Solution Assessment

Yes. The narrow maintainable solution is symmetric socket error handling in src/proxy-capture/proxy-server.ts, plus focused coverage or real proxy proof for abrupt client disconnect behavior.
