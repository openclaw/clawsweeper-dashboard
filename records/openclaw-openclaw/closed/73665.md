## Summary

Current main already addresses the reported v2026.4.26 packaged-install optimizer failure. The image optimizer now loads the media-understanding-core public artifact through the bundled public-surface runtime-dependency preparation path, so the plugin-owned sharp dependency is staged/mirrored before resize code runs. Current MiniMax media metadata also keeps image understanding on MiniMax-VL-01 rather than treating MiniMax-M2.7 chat metadata as the vision provider by default.

## Best Possible Solution

Close as implemented on main. Keep the current bundled public-surface runtime-dependency staging path and MiniMax-VL-01 media-provider routing; the remaining practical step is shipping a release that includes the fix.
