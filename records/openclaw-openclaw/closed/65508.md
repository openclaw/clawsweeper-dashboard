## Summary

Current main already strips raw tool-call XML from user-visible assistant text through the shared assistant-visible sanitizer, with broader tag coverage than the original PR and regression tests for closed, dangling, self-closing, mismatched, Qwen-style, and embedded-closing-token cases. The latest PR refresh is therefore obsolete for the stated sanitizer leak and now carries unrelated release/QA/test rebase fallout.

## Best Possible Solution

Close this PR as obsolete against current main. Keep the shared assistant-visible sanitizer as the source of truth for outbound/history text cleanup, and open a narrow follow-up only if maintainers want different literal XML example preservation semantics.
