## Summary

Current `main` already normalizes Telegram `allowFrom` entries through string coercion before DM sender matching, so quoted numeric IDs and JSON numbers both match. The Telegram setup flow also explicitly accepts numeric sender IDs only, and the same implementation is already present in the shipped `v2026.4.23` release.

## Evidence

- **Telegram runtime coerces allowFrom entries to strings before matching:** `normalizeAllowFrom()` maps each configured entry through `String(value)` and keeps numeric sender IDs as normalized strings before `resolveSenderAllowMatch()` compares them to the incoming sender id.
