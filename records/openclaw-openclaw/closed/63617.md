## Summary

Current main already hardens the exact failure surfaces described here: isolated cron runs are forced onto fresh per-run sessions/transcripts, malformed Kimi/OpenAI-compatible tool-call arguments are repaired before execution, and malformed persisted tool calls are dropped before replay. The matching fixes were shipped across v2026.4.10 through v2026.4.22, so this report is implemented on main rather than still actionable as a separate cleanup item.

## Evidence
