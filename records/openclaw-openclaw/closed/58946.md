## Summary

Current main already fixes the central user-visible bug from #58903: disconnected or unavailable node paths no longer produce false API rate-limit guidance. The open branch adds a more specific node-unavailable message, but main shipped safer underlying fixes for node connectivity, cooldown reason inference, and mixed-cause fallback rendering; the remaining branch also still carries a text-matching mid-turn classifier that reviewers flagged as brittle.

## Best Possible Solution

Close this PR as obsolete because current main and v2026.4.26 already fix the misleading rate-limit behavior through lower-risk node connectivity and fallback/cooldown classification changes. If maintainers still want a dedicated node-unavailable user message, it should be proposed as a fresh narrow PR on top of current main using structured attempt/error data for both pre-turn and mid-turn paths.
