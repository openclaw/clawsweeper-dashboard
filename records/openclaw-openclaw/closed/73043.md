## Summary

Keep open. Current main still routes the reported claude-cli simple-completion path through pi-ai's Anthropic HTTP transport without a Claude CLI dispatch path or a clear unsupported-runtime error, and I found no canonical merged or open fix.

## What This Changes

The issue reports that Lossless/pi-ai simple completion fails against OpenClaw's synthetic claude-cli Sonnet model with a generic connection error.

## Best Possible Solution

Define the runtime.llm/simple-completion contract for CLI runtime models, then either add a Claude CLI one-shot completion path or reject CLI runtime models with a specific unsupported-runtime error and regression coverage.

## Reproduction Assessment

Yes, by source inspection rather than a live macOS run. The reported model still reaches pi-ai completeSimple without a Claude CLI dispatch path or unsupported-runtime guard, while pi-ai's Anthropic implementation treats model.baseUrl as an HTTP SDK baseURL.

## Solution Assessment

Unclear. The reported direction identifies the right boundary, but maintainers should choose whether this API should support Claude CLI one-shot completions or fail fast with a clear unsupported-runtime error.
