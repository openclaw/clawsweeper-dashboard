## Summary

The reported Windows packaged-install failure path is already addressed on current `main` and was shipped in `v2026.4.23`: bundled-plugin startup now restores `openclaw/plugin-sdk/*` resolution for packaged installs and external staged runtime-dependency roots, with regression tests and smoke coverage aimed at the Telegram/Discord-after-dep-repair case.

## Evidence

- **Changelog calls out the exact packaged-install startup fix:** The changelog explicitly says bundled plugin `openclaw/plugin-sdk/*` resolution was restored for packaged installs and external runtime-deps stage roots, so Telegram/Discord no longer crash-loop after missing dependency repair.
