## What This Changes

Report asks OpenClaw to prevent macOS native node WebSocket ping `CheckedContinuation` double-resume crashes during long-running `system.run` commands.

## Best Possible Solution

Keep the implementation discussion on https://github.com/openclaw/openclaw/pull/51371, land or replace it with a focused one-shot ping continuation guard, and add duplicate-callback regression coverage plus real macOS/iOS behavior proof.

## Reproduction Assessment

Yes. Current main still routes keepalive pings through an unguarded checked continuation, and a `WebSocketTasking` double that calls `pongReceiveHandler` twice would hit the same double-resume path; I did not run a live macOS app repro.

## Solution Assessment

Yes. The narrow maintainable fix is the canonical PR's one-shot guard in `GatewayChannel.swift`, ideally paired with a duplicate-callback regression test and after-fix real behavior proof.
