## Close Comment

Closing from a Codex review because this is already fixed on current `main`. The sandbox write path in `src/agents/sandbox/fs-bridge.ts:96` still sends file data on stdin, but the helper launcher in `src/agents/sandbox/fs-bridge-mutation-helper.ts:300` now uses `python -c` instead of the old stdin-consuming heredoc form, so stdin remains available for payload bytes. There is direct regression coverage in `src/agents/sandbox/fs-bridge-mutation-helper.test.ts:120` (verifies a non-empty write stays non-empty) and `src/agents/sandbox/fs-bridge.shell.test.ts:128` (asserts the old `python3 - "$@" <<'PY'` path is gone). `CHANGELOG.md:2623` also records this as shipped in `v2026.3.12`. On that basis, this issue appears implemented on `main` already.
