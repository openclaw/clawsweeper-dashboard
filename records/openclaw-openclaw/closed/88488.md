## What This Changes

The issue reports Android node rows shown by `openclaw nodes status` failing removal with `openclaw nodes remove --node <id>` because the gateway returns `unknown nodeId`.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/90373 or an equivalent gateway-layer fix so displayed node rows and removal share the same node/device pairing boundary while preserving unrelated device roles and tokens.

## Reproduction Assessment

Yes, source-reproducible. The reporter's output plus current source show `node.list` can display a device-backed node row while `node.pair.remove` only deletes legacy node-pairing records and returns `unknown nodeId`.

## Solution Assessment

Yes, pending maintainer acceptance of the credential semantics. The maintainable fix belongs in the gateway node removal and device-pairing role boundary, not in a CLI-only workaround.
