## Summary

Keep open: this PR is the right canonical path for the linked multi-agent cron-list feature, but the current branch is not merge-ready because the new filter compares unnormalized agent IDs and the user-facing surface lacks required docs, changelog, and regression coverage.

## What This Changes

The PR adds an optional `agentId` filter to `cron.list` across the protocol, service, gateway handler, agent cron tool auto-fill, CLI `--agent`, and generated Swift protocol models.

## Best Possible Solution

Land this as a focused cron feature after normalizing the list filter value consistently and adding regression tests, docs, and a changelog entry.

## Reproduction Assessment

Yes. Source inspection is enough: current main has no `agentId` list filter, and the PR defect follows directly because cron add/edit normalize stored IDs while the new list filter compares the raw requested string.

## Solution Assessment

No, not yet. The API shape is reasonable, but the maintainable implementation should normalize the requested agent ID before filtering and include tests plus user-facing documentation and changelog coverage.
