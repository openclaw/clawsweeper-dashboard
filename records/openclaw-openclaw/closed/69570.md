## Summary

Keep open. Current main still models exec approval completion as one approval-id-scoped detached follow-up agent turn per approval result, and the linked production repro plus docs/tests show this remains an active scheduler-policy problem rather than cleanup.

## What This Changes

The issue asks OpenClaw to prevent approved exec completion cascades from blocking later user commands by batching, cancelling, deprioritizing, or stopping stale async approval follow-ups.

## Best Possible Solution

Define a core exec approval scheduler contract that can coalesce or cancel stale approval follow-ups, prioritize stop or newer user input, and preserve a trustworthy audit trail for every approved command outcome.

## Reproduction Assessment

Yes, at source level. Configure exec approvals, approve multiple same-session foreground commands, and current main still returns approval-pending while creating one detached approval-id follow-up agent turn after each command finishes.

## Solution Assessment

Unclear. The requested direction is valid, but the best fix requires maintainer policy on whether to pause, batch, drop, direct-deliver, or audit stale exec completion messages.
