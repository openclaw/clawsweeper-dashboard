## Summary

Current `main` does not contain the PR's write-verification changes. `src/agents/sandbox/fs-bridge-mutation-helper.ts` still performs unchecked `os.write()` calls in both `write_atomic` and the EXDEV `move_entry` copy path, and the tree does not contain the new fakeowner/network-mount error strings described by PR #44736. This is not implemented on `main`, and there is not enough contrary evidence to close the PR.
