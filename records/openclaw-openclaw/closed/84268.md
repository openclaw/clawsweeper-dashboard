## What This Changes

The PR adds gateway projection and session-history logic to mirror successful current-session `message.send` replies into visible chat history when the run ends with a silent `NO_REPLY`.

## Best Possible Solution

Keep the legacy-history backfill, but make it dedupe against durable source-reply transcript mirrors and derive displayed text from the sanitized delivery payload contract.

## Reproduction Assessment

Yes. A transcript containing a `message.send` tool call/result followed by an assistant `NO_REPLY` has a clear current-main projection path where the `NO_REPLY` is dropped and hidden tool calls do not become a normal assistant reply; the PR body also supplies real-session after-fix projection proof.

## Solution Assessment

No. The fallback direction is useful, but the current implementation should be repaired to reuse delivered/sanitized source-reply data and avoid duplicating the durable transcript mirror path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Skip mirrors when a durable reply already exists:** `src/gateway/chat-display-projection.ts:793`
  - body: Current main already appends successful internal source replies back into the transcript through `sourceReplyTranscriptMirror`. This fallback blindly pushes a synthetic assistant mirror when it reaches `NO_REPLY`, so transcripts where the durable mirror is appended after the final silent assistant message can render the same reply twice after reload. Please dedupe against the durable mirror path or restrict this fallback to legacy transcripts without one.
  - confidence: 0.87
- **[P2] Mirror delivered text instead of raw tool arguments:** `src/gateway/chat-display-projection.ts:565`
  - body: This path builds visible history from raw tool-call args, but the actual send path normalizes escaped newlines, caption fallback, reply directives, media hints, citation controls, cross-context markers, and final reply sanitization before delivery. History should mirror the sanitized delivered source-reply payload; otherwise reload can show text that was stripped or changed before the user saw it.
  - confidence: 0.9
