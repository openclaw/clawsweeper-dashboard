## Summary

#55704 should close as superseded by the exact open PR #55761. Current main still has the reported coarse shared-hook fallback behavior, but the remaining work is already tracked by #55761, which explicitly fixes #55704 and covers the broader announce policy slice.

## Best Possible Solution

Close #55704 as superseded by #55761, and keep the actual fix work on the PR. The PR or an equivalent maintainer patch should update `src/gateway/server/hooks.ts` with an explicit announce policy that suppresses successful silent/no-deliver shared-hook output while still surfacing real hook errors, with regression tests for direct and mapped hooks.
