## Summary

Current main and the latest shipped release already address the reported inflated session cost class: session usage cost is now snapshotted instead of accumulated across repeated persists, with regression coverage for that behavior.

## What This Changes

The issue reports session-level `estimatedCostUsd` being much higher than manual and transcript-derived cost for a multi-turn Xiaomi MiMo run on OpenClaw 2026.5.18.

## Best Possible Solution

Keep the shipped `v2026.5.27` snapshot behavior; if cost still diverges on a current release, open a new report with redacted transcript usage and `sessions.json` from that version.

## Reproduction Assessment

No current-main failure was reproduced; the issue targets 2026.5.18, and current main plus `v2026.5.27` include regression coverage for the same repeated-persist cost inflation class.

## Solution Assessment

Yes. The shipped fix is the narrow maintainable path: calculate cost from normalized usage and configured rates, then snapshot the run estimate instead of accumulating it in the session store.
