## What This Changes

The issue requests preserving `reasoning_content` across multi-turn tool-call replay for non-DeepSeek OpenAI-compatible reasoning providers such as Xiaomi MiMo.

## Best Possible Solution

Keep the current main implementation and ship it in the next release; users on 2026.5.7 or the current stable should upgrade once that release is available or use the documented `reasoning: false` workaround until then.

## Reproduction Assessment

No current-main failure is reproducible from source: the reported 2026.5.7 path is covered by current Xiaomi/native and custom-proxy replay code plus focused tests. I did not run a live Xiaomi call in this read-only review.

## Solution Assessment

Yes. The current solution keeps the behavior in the provider/OpenAI-compatible replay layer, preserves the provider-required field only for known compatible routes, and avoids changing stock OpenAI/Qwen replay behavior.
