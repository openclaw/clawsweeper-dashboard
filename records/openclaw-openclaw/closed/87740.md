## Summary

Keep open. Current `main` and the latest release still contain the fallback path that can persist an unsupported thinking fallback over a stored session override, and the GitHub Copilot Claude thinking profile can be suppressed by catalog `reasoning: false`, so the reported session-state bug remains actionable.

## What This Changes

The issue requests that explicit session `thinkingLevel` overrides persist across agent turns instead of being overwritten to `off` by runtime fallback resolution.

## Best Possible Solution

Preserve explicit session thinking overrides across turns, apply unsupported fallback only to the active execution path unless the user deliberately changes models or thinking settings, and add a GitHub Copilot Claude regression test for the profile/catalog mismatch.

## Reproduction Assessment

Yes from source inspection, though not from a live gateway run: a stored session thinking override can still be replaced by the unsupported-level fallback in the reply path. The GitHub Copilot Claude profile/catalog path also matches the reporter's explanation because catalog `reasoning: false` can suppress the provider-advertised `high` level.

## Solution Assessment

Yes, a narrow runtime fix is the best path: stop persisting turn-local fallback values over explicit session overrides and add focused regression coverage. The adjacent `/reset` behavior in https://github.com/openclaw/openclaw/issues/80377 should stay separate unless maintainers choose to combine that broader policy work.
