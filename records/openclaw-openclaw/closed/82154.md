## Summary

Keep open: current main does not show a completed path for gateway-visible provider credentials to reach native subagent model/auth execution, and the report is recent with no duplicate or merged fix found.

## What This Changes

The issue reports that subagents spawned from a gateway process cannot see provider API-key environment variables that the gateway itself can use.

## Best Possible Solution

Define a narrow credential propagation contract for subagent model/auth resolution, or document explicit sandbox env configuration if isolation is intended; avoid broad host env forwarding.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No live failing run was performed, but source inspection gives a medium-confidence path: spawn a sandboxed subagent using a gateway-only provider env var and check whether the child model call or exec can see it.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the credential boundary; the safest direction is provider auth/profile or explicit allowlisted env propagation, not wholesale process.env inheritance.
