- `P1`: The reported release regression breaks a supported channel's inbound agent dispatch for affected users.

## Summary

Current main already fixes the Feishu inbound dispatch crash by passing the full plugin channel runtime into channel startup and Feishu message handling; the fix is current-main-only and has not shipped in v2026.5.28.

## What This Changes

The issue reports Feishu inbound messages being received but failing to dispatch to the agent with `Cannot read properties of undefined (reading 'run')`.

## Best Possible Solution

Keep the current-main runtime handoff fix and ship it in the next release; reopen or file a new report only if Feishu dispatch still fails in a build containing d05e4a4bc6f22aaaa17ca566568556d46a67dee9.

## Reproduction Assessment

Yes, at source level for the affected release path: the old startup path could hand Feishu a channel runtime without `inbound.run`, matching the reported crash. Current main changes that handoff to a full plugin channel runtime.

## Solution Assessment

Yes. Passing the full gateway channel runtime into Feishu startup fixes the missing `inbound.run` surface at the owner boundary instead of adding a guard that would still drop messages.
