## Summary

Close PR #44878 as superseded by the canonical feature request #43760. Current main has not implemented separate fallback remote/model config: fallback is still typed and validated as a string, and fallback activation reuses the primary remote config. The PR is not the right active vehicle because it targets older memory files and bundles unrelated SSRF, compaction, and binary-artifact changes with unresolved review comments. Keep #43760 open for a narrow implementation on current main.

## Best Possible Solution

Close this PR, keep #43760 as the active tracker, and implement a fresh narrow patch on current main. The new implementation should add an object fallback shape without losing generic plugin-defined provider IDs, thread fallback-specific model/remote/secret handling through the current memory-core fallback path, update schema/help/docs/secret surfaces, and add focused tests. Do not carry forward the unrelated SSRF, compaction, or binary-artifact changes from this branch.
