## Summary

The broader open session-activity issue already owns the explicit awaiting-user and awaiting-subagent state contract, including the producer-side signaling problem described here. Consolidating the production workaround and requirements there avoids two competing gateway protocol designs.

## What This Changes

Requests an explicit agent-to-gateway end-of-turn signal for awaiting user or subagent input, integrated with session activity events and removed before channel delivery.

## Best Possible Solution

Keep one canonical design in https://github.com/openclaw/openclaw/issues/39127 and incorporate a typed agent-runtime attention transition that is emitted before channel rendering, explicitly cleared, and never inferred from message text.

## Solution Assessment

No, not as a separate token-oriented mechanism; the best solution is to define the producer-side typed signal within the canonical session activity and attention-state design.
