## Summary

The PR's claimed SPA-route bug does not match current `main`: missing Control UI routes already resolve to non-IO boundary failures that fall through to the existing `index.html` SPA fallback. The proposed `io: () => null` change would instead hide unexpected filesystem faults such as permission or disk errors.

## Evidence

- **Current fallback only throws real IO failures:** `resolveSafeControlUiFile` handles boundary open failures by throwing only the `io` branch; all other failure reasons already fall through `fallback: () => null`, and the caller then tries the SPA `index.html` fallback for unknown non-asset routes.
  - file: [src/gateway/control-ui.ts:657](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/gateway/control-ui.ts#L657)
  - command: `nl -ba src/gateway/control-ui.ts | sed -n '643,670p'`
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **Missing paths are classified as `path`, not `io`:** `openVerifiedFileSync` maps `ENOENT`, `ENOTDIR`, and `ELOOP` to `reason: "path"`; only other failures become `reason: "io"`. A missing `/chat` file therefore does not take the throwing `io` branch.
  - file: [src/infra/safe-open-sync.ts:17](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/infra/safe-open-sync.ts#L17)

- **Boundary resolution already tolerates missing lexical segments:** Boundary path resolution treats not-found path errors during lexical traversal as a missing in-boundary suffix instead of throwing, which supports the later null/fallback behavior for client-side routes without physical files.
  - file: [src/infra/boundary-path.ts:226](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/infra/boundary-path.ts#L226)
  - command: `nl -ba src/infra/boundary-path.ts | sed -n '226,267p'`
  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **Main handler already has explicit SPA fallback:** After a requested file misses, non-static-asset routes fall through to `index.html`; only recognized asset extensions return a direct 404. This is the code path the PR claims is blocked.
  - file: [src/gateway/control-ui.ts:857](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/gateway/control-ui.ts#L857)
