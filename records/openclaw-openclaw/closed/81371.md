## What This Changes

The PR adds `attempt_dispatch` to the cron execution-start milestone classifier and adds a focused timer regression test for isolated cron runs that pass the old 60-second pre-execution watchdog.

## Best Possible Solution

Land this narrow classifier and regression-test change, or an equivalent runner milestone fix, then close https://github.com/openclaw/openclaw/issues/81368.

Yes. ## Solution Assessment

Yes. The PR changes the private classifier at the point where the watchdog decides whether execution started and adds a focused regression test; a later, more exact `model_call_started` emission could improve semantics but is not required to fix this false abort.
