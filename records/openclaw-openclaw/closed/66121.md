## Summary

Current `main` preserves the original/source plugin config through the doctor/status plugin-loading path and has regression tests showing that a populated `plugins.allow` no longer triggers the `plugins.allow is empty` warning. The behavior reported against `2026.4.12` appears fixed on `main` by commit `628f0e80553fe6d0d1d755eb2fef4679b0e2732b`, but I could not tie that commit to a shipped release tag.

## Evidence

- **Warning condition only fires when effective allowlist is empty:** `warnWhenAllowlistIsOpen` returns early when `params.allow.length > 0`; the reported false positive therefore depends on the wrong activation-source config reaching the loader.
  - file: [src/plugins/loader.ts:1871](https://github.com/openclaw/openclaw/blob/fe930b987e33f4e1acd93b5ef1ea76c8b7260047/src/plugins/loader.ts#L1871)
  - command: `nl -ba src/plugins/loader.ts | sed -n '1871,1908p'`
  - sha: [fe930b987e33](https://github.com/openclaw/openclaw/commit/fe930b987e33f4e1acd93b5ef1ea76c8b7260047)
- **Doctor/status now pass raw config as activation source:** `buildPluginReport` uses `rawConfig` and explicitly passes `activationSourceConfig: rawConfig` into both `loadOpenClawPlugins(...)` and `loadPluginMetadataRegistrySnapshot(...)`, which is the path doctor/status use.
  - file: [src/plugins/status.ts:160](https://github.com/openclaw/openclaw/blob/fe930b987e33f4e1acd93b5ef1ea76c8b7260047/src/plugins/status.ts#L160)
  - command: `nl -ba src/plugins/status.ts | sed -n '156,201p'`

- **Activation-source helper preserves source snapshot:** `resolvePluginActivationSourceConfig` prefers an explicit `activationSourceConfig`, otherwise falls back to the runtime source snapshot when the normalized runtime config object is in use.
  - file: [src/plugins/activation-source-config.ts:7](https://github.com/openclaw/openclaw/blob/fe930b987e33f4e1acd93b5ef1ea76c8b7260047/src/plugins/activation-source-config.ts#L7)

- **Regression test covers source snapshot with populated allowlist:** The load-context test sets a runtime config with empty plugins, a separate source config with `plugins.allow = ["trusted-plugin"]`, and asserts that the context keeps `activationSourceConfig === sourceConfig`.
  - file: [src/plugins/runtime/load-context.test.ts:97](https://github.com/openclaw/openclaw/blob/628f0e80553fe6d0d1d755eb2fef4679b0e2732b/src/plugins/runtime/load-context.test.ts#L97)
