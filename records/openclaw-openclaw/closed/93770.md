## Label Justifications

- `P1`: The reported release regression made Telegram channel replies unreadable for affected users, breaking a real channel workflow.
- `impact:message-loss`: Readable Telegram reply content was replaced by unsupported-message placeholders, effectively losing delivered message content for affected clients.

## Summary

Current main already implements the requested Telegram rich-message opt-out behavior: standard Telegram text delivery is the default, and Bot API rich messages only run when `channels.telegram.richMessages` is explicitly true.

## What This Changes

The issue requests a Telegram configuration path to disable `sendRichMessage` and force readable standard text delivery.

## Best Possible Solution

Keep the current-main Telegram contract: standard `sendMessage`/HTML delivery by default, `richMessages: true` as explicit opt-in, and release follow-through tracked separately for users still on `v2026.6.8`.

## Reproduction Assessment

Yes for the shipped regression: `v2026.6.8` source routes default Telegram text through `sendRichMessage`, matching the reported unsupported-message behavior. Current main no longer reproduces the default-path problem because rich delivery is gated on `richMessages === true`.

## Solution Assessment

Yes. Current main solves the central user problem more cleanly than a separate disable flag by making standard text delivery the default and keeping rich messages as an explicit opt-in.
