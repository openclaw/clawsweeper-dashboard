## Summary

This request is valid, but on current `main` it fits the documented plugin/ClawHub extension surface rather than OpenClaw core. The repo already exposes the needed primitives: inbound message hooks, prompt/context-engine injection, bounded cross-session session recall, and an existing cross-session memory plugin model.

## Risks / Open Questions

- There is not an exact off-the-shelf `<#channelId>` mention plugin linked on the issue, so closing assumes a ClawHub/plugin implementation rather than a built-in replacement.
- If maintainers want a first-party security policy for automatic channel-to-session resolution across all channels, that could still justify future plugin-API refinement without reopening this as core work.
