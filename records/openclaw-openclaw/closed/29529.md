## Close Comment

Closing as `clawhub` after a Codex review of current `main` (`c6684af682bcb59bd1ee85cb49f72d4074007e2c`). This is a useful idea, but current project scope keeps optional capability like distributed knowledge sharing out of OpenClaw core: `VISION.md:54` says optional capability should usually ship as plugins, `VISION.md:68` says new skills should go to ClawHub first, and `VISION.md:101` lists new core skills that can live on ClawHub as out of scope. The existing extension seams already fit this proposal: `docs/tools/plugin.md:11` documents external/community plugins and ClawHub install flow, `docs/plugins/sdk-overview.md:184` exposes memory-plugin public-artifact APIs for companion plugins, and `docs/plugins/memory-wiki.md:40` plus `docs/plugins/memory-wiki.md:80` describe shared artifact bridge patterns. If this is pursued, it should be built as a ClawHub plugin or companion memory plugin rather than merged into OpenClaw core.
