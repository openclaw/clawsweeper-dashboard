## Summary

Current main implements configurable non-Batch memory indexing concurrency through `agents.defaults.memorySearch.remote.nonBatchConcurrency`. The setting is accepted by the strict config schema, resolved into `ResolvedMemorySearchConfig`, used by `getIndexConcurrency()` for inline/non-Batch indexing, covered by focused tests, and documented in the public memory config reference. The fix is on current main after `v2026.4.26`; no shipped tag contains the proof commit yet.

## Best Possible Solution

Close this issue as implemented on current main, keep `agents.defaults.memorySearch.remote.nonBatchConcurrency` as the documented user-facing knob, and leave broader retry/backoff configurability to #68812/#68822 if maintainers still want that additional scope.
