## Summary

Current `main` already backfills session `contextTokens` from transcript usage or the resolved model when the stored session entry does not have an explicit context window, and the Control UI already uses that field to compute the transcript context percentage. The same behavior is present in the shipped `v2026.4.23` tag, including an Anthropic regression test.

## Evidence

- **Gateway session rows now derive missing contextTokens from the resolved model:** `buildGatewaySessionRow` falls back from stored `entry.contextTokens` to transcript usage and then `resolveContextTokensForModel(...)`, which is the missing input the UI needs for provider-default context windows such as Anthropic/Claude.
