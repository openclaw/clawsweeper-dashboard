## Summary

Current `main` already hardens the Telegram delivery path against this leak family: Telegram tool/progress chatter is suppressed by default, unmatched `<think>` partials are kept out of the answer lane, and core embedded-agent handling suppresses `commentary`-phase user-visible output. The relevant Telegram hardening landed in commit `55d3e7bc473e34883a2d2cce51ce904b87c32eed` and is on `main`, but it is not in a tagged release yet.

## Evidence
