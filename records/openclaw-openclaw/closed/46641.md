## What This Changes

Request to add a configurable, user-visible active-channel prompt before auto-compaction so the user can tell OpenClaw what to preserve before memory flush and compaction.

## Best Possible Solution

Fold this report’s active-channel prompt and human response-window requirement into the open compaction lifecycle and confirmation trackers, then design one core compaction UX/config contract instead of another ad hoc notification knob.
