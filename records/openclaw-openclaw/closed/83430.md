## Summary

Current main and v2026.5.12 already route generated-media completions through the requester agent with message-tool-only visible delivery, which addresses the reported Telegram direct-chat outbound adapter failure from 2026.5.6.

## What This Changes

Ensure Telegram direct-chat background media generation completions wake the requester and deliver generated artifacts instead of failing with `Outbound not configured for channel: telegram`.

## Best Possible Solution

Keep the v2026.5.12/current-main message-tool-only generated-media completion path; request a fresh report only if v2026.5.12 or newer still fails with new gateway logs and the task run id.

## Reproduction Assessment

No live current-main Telegram reproduction was run in this read-only sweep. The reporter provided concrete v2026.5.6 logs, and current source/tests show the failing direct final-delivery path is bypassed for generated media completions.

## Solution Assessment

Yes. Routing generated-media completions through requester-agent handoff with `sourceReplyDeliveryMode: "message_tool_only"` is the narrow maintainable fix because the completion agent must send the saved media through the normal message tool path.
