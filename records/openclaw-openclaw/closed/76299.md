## Summary

Close as implemented on current main. OpenClaw uses the term `thinking` for the effort/level control and `reasoning` for visibility; the requested global and per-model default thinking-level mechanism is present, and both model-switch and cron paths consume it.

## What This Changes

Add an `openclaw.json` mechanism for global and per-model default reasoning/thinking levels used when switching models.

## Best Possible Solution

Keep the existing `thinkingDefault` plus per-model `params.thinking` implementation; a future docs polish could make the naming clearer, but no core feature gap remains.

## Reproduction Assessment

Not applicable as a bug report. For this feature request, the high-confidence verification path is static: configure `agents.defaults.thinkingDefault` or `agents.defaults.models["provider/model"].params.thinking` and `resolveThinkingDefault` returns that value before built-in model defaults.

## Solution Assessment

Yes. Reusing `thinkingDefault` and per-model `params.thinking` is the narrow maintainable solution because `reasoningDefault` is already reserved for visibility (`off|on|stream`) while thinking controls the effort level requested here.
