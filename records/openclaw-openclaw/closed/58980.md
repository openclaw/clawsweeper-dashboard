## Summary

Keep open. Current main still has the exact gap this PR targets: the OpenRouter/Kilocode proxy wrapper removes `reasoning_effort` but returns on `thinkingLevel` `off` or `undefined` without removing a nested `reasoning` object. The pinned upstream `@mariozechner/pi-ai` 0.70.5 OpenAI-completions transport can emit `reasoning: { effort: "none" }` for OpenRouter-format reasoning models when no reasoning effort is set, so the linked 400 path is still plausible on main. The PR is narrow and the security pass did not find supply-chain or secret-handling changes.

## Best Possible Solution

Review and land this PR, or an equivalent scoped patch, after rebasing onto current main. The code should delete `payloadObj.reasoning` in the off/undefined branch of `normalizeProxyReasoningPayload`, and the regression tests should cover OpenRouter and Kilocode for off, undefined, and active thinking so the mandatory-reasoning endpoint behavior stays fixed.
