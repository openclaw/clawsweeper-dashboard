## Summary

## What This Changes

The issue reports that the bundled Telegram plugin is auto-enabled on Windows AutoClaw 1.1.1.91 / OpenClaw 2026.3.11 but its channel runtime never starts polling.

## Best Possible Solution

Keep the current runtime-only auto-enable startup implementation and direct affected users to a build containing 6e5042cd62e7b54d57ad4dddb65a32f5d52d54b0; reopen or file a new issue only if a post-fix Windows build still skips Telegram startup.
