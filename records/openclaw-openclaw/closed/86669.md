## Summary

Keep open. The report is actionable: current source already has the vLLM/Nemotron payload wrapper, but the `/v1/chat/completions` ingress path appears to miss the documented per-agent `thinkingDefault` before that wrapper can fire, which matches the reporter's HTTP-only failure pattern.

## What This Changes

The issue asks OpenClaw's OpenAI-compatible `/v1/chat/completions` path to send vLLM/Nemotron `chat_template_kwargs` for thinking-off sessions, matching the CLI/embedded behavior.

## Best Possible Solution

Propagate the selected agent's `thinkingDefault` into agent-command thinking resolution before the global fallback, then add a focused regression proving OpenAI-compatible HTTP with a `vllm/nemotron-3-*` agent and per-agent thinking off sends the expected chat-template kwargs.

## Reproduction Assessment

Yes, source-reproducible with high confidence: the reporter provides a concrete MITM setup, and current main shows the HTTP path reaches `agentCommandFromIngress` while fresh agent runs ignore `agents.list[].thinkingDefault`. I did not run the live vLLM/Nemotron server locally.

## Solution Assessment

Yes, the best fix is a narrow default-resolution repair, not a new config surface or a duplicate HTTP-only vLLM wrapper. The existing vLLM wrapper should remain the single owner of Nemotron payload mutation once the correct thinking level reaches it.
