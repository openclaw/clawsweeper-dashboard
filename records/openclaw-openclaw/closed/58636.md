## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep upload context until send succeeds:** `src/tui/tui-command-handlers.ts:572`
  - body: `pendingFileContext` is cleared before `client.sendChat` runs, so any send rejection after the user attaches files drops that context and a retry sends only the typed message. Clear it only after `sendChat` resolves, or restore it in the catch path.
  - confidence: 0.9
- **[P1] Clear stale context when replacement upload attaches nothing:** `src/tui/tui-command-handlers.ts:278`
  - body: If a previous upload is queued and the user confirms a new upload where every selected file is skipped or unreadable, this branch leaves the old `pendingFileContext` intact. Clear pending context before processing the replacement upload or explicitly clear it when `blocks.length === 0`.
  - confidence: 0.91
- **[P1] Exclude non-regular entries from selectable uploads:** `src/tui/components/file-browser-select-list.ts:80-81`
  - body: Every non-directory entry is pushed as a selectable file, including FIFOs, sockets, and device nodes; the upload handler later calls `readFileSync` on the selected path. Only regular files should be selectable, and the handler should revalidate before reading.
  - confidence: 0.88
- **[P2] Bound file size before reading the whole file:** `src/tui/tui-command-handlers.ts:257`
  - body: The handler reads the entire selected path before sampling for binary content, so a very large regular file can freeze or exhaust the TUI before the binary heuristic runs. Stat the file first and enforce a clear size limit before `readFileSync`.
  - confidence: 0.84
- **[P2] Use the current pi-tui package import:** `src/tui/components/file-browser-select-list.ts:10`
  - body: Current main has moved TUI imports and the root dependency to `@earendil-works/pi-tui`, but this added component imports `@mariozechner/pi-tui`. Rebasing this branch as-is would leave the new file on the old package surface and fail the current dependency contract.
  - confidence: 0.82
