## Summary

Close this PR as superseded. Current main contains the accepted safer warning policy from merged #73750: suppress mutating-tool warnings when a user-facing error reply or explicit failed-action acknowledgement already covers the turn, keep warnings visible for success-looking or ambiguous replies, and leave broader configurable delivery policy to #39406. The PR's proposed `hasUserFacingReply` rule conflicts with current tests and would hide unresolved mutating failures behind replies like `Done.`.

## Best Possible Solution

Close this PR as superseded by merged #73750 for the accepted safety fix, keep the current main behavior that warns on unresolved success-looking mutating failures, and continue the broader configurable transient-warning policy in #39406.
