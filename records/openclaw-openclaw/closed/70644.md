## Summary

Current main routes Signal heartbeat/session/cron output through the shared outbound delivery path instead of transcript-only bookkeeping, and cron delivery status now derives from the actual outbound result rather than internal completion alone.

## Best Possible Solution

Close as implemented on current main and keep the shared Signal outbound, heartbeat outbound, and cron delivery-status paths as the product boundary. If this recurs, the next issue should include a fresh live Signal reproduction with signal-cli/RPC send logs, cron run diagnostics, and the affected session key/config.
