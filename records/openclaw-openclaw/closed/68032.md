## Summary

Current `main` no longer matches the reported failure mode: skills are exposed to the model as metadata-only catalog entries, duplicate skill names are merged before prompt generation, the skills prompt is budget-limited, overflow compaction retries are capped at three, and the compaction-failure restart path does not delete the prior transcript.

## Evidence

- **Skills prompt is metadata-only:** `formatSkillsForPrompt()` renders each available skill with only `<name>`, `<description>`, and `<location>`. The system prompt then tells the model to read at most one selected `SKILL.md` afterward, so current main does not bulk-inline skill file contents into `<available_skills>`.
