## Summary

Current main already derives the GitHub Copilot API base URL from the exchanged token's `proxy-ep` hint and applies that resolved host across Copilot discovery, runtime auth, and embeddings, which matches the reported 421/Misdirected Request failure mode for Business tokens.

## Evidence

- **Token flow derives provider-specific baseUrl:** `deriveCopilotApiBaseUrlFromToken` parses `proxy-ep` from the Copilot token, rewrites `proxy.*` to `api.*`, validates it, and `resolveCopilotApiToken` returns that `baseUrl` for both cached and freshly fetched tokens.
  - file: [src/agents/github-copilot-token.ts:79](https://github.com/openclaw/openclaw/blob/a94763ee224e858e29dd177cd6f7f4daa438c2f1/src/agents/github-copilot-token.ts#L79)
  - sha: [a94763ee224e](https://github.com/openclaw/openclaw/commit/a94763ee224e858e29dd177cd6f7f4daa438c2f1)
- **Provider catalog now publishes resolved Copilot baseUrl:** The bundled `github-copilot` provider uses `resolveCopilotApiToken(...)` during catalog discovery and writes `provider.baseUrl = token.baseUrl` instead of always keeping the individual-host default.
  - file: [extensions/github-copilot/index.ts:121](https://github.com/openclaw/openclaw/blob/a94763ee224e858e29dd177cd6f7f4daa438c2f1/extensions/github-copilot/index.ts#L121)
  - sha: [a94763ee224e](https://github.com/openclaw/openclaw/commit/a94763ee224e858e29dd177cd6f7f4daa438c2f1)
- **Runtime auth uses resolved Copilot baseUrl:** At runtime the provider returns `{ apiKey: token.token, baseUrl: token.baseUrl }`, so actual chat/model requests use the token-derived endpoint.
  - file: [extensions/github-copilot/index.ts:168](https://github.com/openclaw/openclaw/blob/a94763ee224e858e29dd177cd6f7f4daa438c2f1/extensions/github-copilot/index.ts#L168)
  - sha: [a94763ee224e](https://github.com/openclaw/openclaw/commit/a94763ee224e858e29dd177cd6f7f4daa438c2f1)
- **Embeddings path also uses resolved host:** Copilot embeddings resolve the session with `token.baseUrl` and send `/embeddings` requests to that host, covering the same misrouting class outside normal chat requests.
  - file: [extensions/github-copilot/embeddings.ts:206](https://github.com/openclaw/openclaw/blob/a94763ee224e858e29dd177cd6f7f4daa438c2f1/extensions/github-copilot/embeddings.ts#L206)
  - sha: [a94763ee224e](https://github.com/openclaw/openclaw/commit/a94763ee224e858e29dd177cd6f7f4daa438c2f1)
