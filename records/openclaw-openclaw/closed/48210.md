## Summary

Closing as `cannot_reproduce`. Current `main` still uses plain `server.close()` for Feishu monitor shutdown, but on the current supported Node runtime the reported stuck-port / `EADDRINUSE` behavior does not reproduce: local read-only repros showed immediate same-port rebinding after `server.close()` for both idle keep-alive and in-flight requests.

## Risks / Open Questions

- The repros were runtime-level and did not exercise a live Feishu account or a full gateway restart sequence.
- `closeAllConnections()` could still change shutdown behavior for truly hung or very long-running webhook requests, but that is a different claim than the PR's reported current-main `EADDRINUSE` / stuck-port bug.
