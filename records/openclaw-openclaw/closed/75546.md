## Summary

Close as duplicate/superseded because the remaining CLI compaction init-order bug is already tracked by the active fix PR #75129 and the broader canonical context-engine cold-registry issue #73095. Current main still contains the source-level gap, but the separate issue no longer needs to remain open as its own tracker, and the detailed duplicate #75548 preserves the reporter's reproduction context.

## What This Changes

The issue reports that CLI transcript persistence fails because the CLI compaction lifecycle resolves the default legacy context engine before registering built-in context engines.

## Best Possible Solution

Use PR #75129, or an equivalent narrow patch, to initialize built-in context engines at the CLI compaction boundary before resolving the active engine, with regression coverage for the cold registry path. Keep #73095/#75129 as the remaining canonical tracking path instead of maintaining this separate duplicate issue.

## Reproduction Assessment

Yes. A cold CLI-backed turn with a persisted `sessionFile` and positive `contextTokens` reaches `runCliTurnCompactionLifecycle()`, which currently resolves the default `legacy` engine before any local initializer registers it; #75548 supplies the same reporter's shipped-version reproduction steps.

## Solution Assessment

Yes for cleanup: closing this as superseded by the already-open targeted PR is the narrowest tracker state. The code solution itself should be the PR #75129 shape after its regression fixture is corrected, not a broader registry side-effect change.
