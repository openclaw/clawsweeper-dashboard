## Summary

Current `main` already supports SAP AI Core well enough through OpenClaw's generic custom-provider path: arbitrary `models.providers.<id>` entries can supply a custom base URL, model list, static headers, and request-level auth/header overrides, which matches SAP AI Core's documented OpenAI-compatible chat-completions surface and required bearer/resource-group headers. ## Close Comment

Closing from a Codex review because current `main` already supports this integration through the generic custom-provider path. OpenClaw documents arbitrary `models.providers.<id>` entries with custom `baseUrl`, `headers`, and `request.auth` / `request.headers` overrides in `docs/gateway/configuration-reference.md:2543-2605`, and the runtime/config types implement that in `src/config/types.models.ts:114-123`, `src/config/types.provider-request.ts:3-46`, and `src/agents/provider-request-config.ts:435-542`. SAP AI Core's own docs describe its generative model surface as OpenAI-compatible and using bearer auth plus an `AI-Resource-Group` header, which fits that existing path.
