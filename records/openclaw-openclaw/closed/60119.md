## Summary

Keep open. Current `main` does not contain the proposed voice-call session pinning, but this PR's stated root cause does not match the live-switch code that actually throws `LiveSessionModelSwitchError`, and the review discussion surfaced two concrete correctness problems in the patch: stale persisted overrides and concurrent session-store clobbering. The underlying voice-call bug may still need a different fix, so there is not strong enough evidence to close conservatively.

## Evidence

- **Requested behavior is not implemented on current main:** `generateVoiceResponse()` creates/saves a new session only when missing, then resolves `provider`/`model` and calls `runEmbeddedPiAgent()` directly. There is no current-main write of `providerOverride`/`modelOverride` for voice-call sessions.
