## Summary

Keep open: current main and the latest shipped iOS UI still appear to accept private-LAN QR setup codes but then fail to persist the bootstrap handoff tokens needed for the post-pairing operator/chat session.

## What This Changes

The issue asks for iOS QR pairing over LAN to persist the gateway/operator auth handoff so mobile chat and Control UI remain connected without re-entering the Gateway Token.

## Best Possible Solution

Align iOS bootstrap handoff persistence with the existing private-LAN/.local setup-code allowlist, keep public/tailnet plaintext fail-closed, and add regression coverage proving node plus operator tokens persist and the operator chat loop starts after QR pairing.

## Reproduction Assessment

Yes for the source path, but not from a live iPhone run: private-LAN QR setup is accepted, the iOS bootstrap flow clears old tokens, and current Swift persists the replacement handoff only for `wss` or loopback.

## Solution Assessment

No, current main is not the best state: the setup-code trust predicate and the handoff persistence predicate disagree for private-LAN QR pairing, so the narrow fix is to reuse the established private-LAN/.local boundary and cover the iOS reconnect flow.

## Repair Work Prompt

Fix the source-reproducible iOS QR/private-LAN bootstrap persistence bug from https://github.com/openclaw/openclaw/issues/98033. Current setup-code parsing accepts private LAN and `.local` `ws://` payloads, but `GatewayChannel.shouldPersistBootstrapHandoffTokens()` only persists bootstrap handoff tokens for `wss` or loopback, so the iOS QR flow clears old tokens, clears the spent bootstrap token after node connect, and never starts the operator/chat loop because no operator token was stored. Align the Swift persistence trust predicate with the already-supported private-LAN/.local setup-code allowlist without allowing public, tailnet, CGNAT, or attacker-like plaintext hosts; do not add config flags or broad compatibility shims. Add focused Swift tests proving private-LAN and `.local` bootstrap hello responses persist both node and bounded operator tokens, while public/tailnet/CGNAT plaintext still do not. Add or update iOS model tests if needed to prove the operator loop starts after the handoff is persisted. Update docs only if the trusted local-pairing wording needs clarification; do not edit CHANGELOG.md.
