## Summary

Current main already fixes the reported Control UI CSP failure by allowing the exact tweakcn registry origin in `connect-src`, with regression coverage for the served CSP. The fix is present on main but not yet included in a tagged release.

## What This Changes

Allow the Control UI tweakcn theme importer to work behind Tailscale Serve by permitting or otherwise handling the tweakcn registry request instead of showing a generic network error.

## Best Possible Solution

Keep the narrow `https://tweakcn.com` CSP allowlist and ship it in the next release; a gateway proxy is not needed for the current single registry origin.

## Reproduction Assessment

No current-main failure path remains: the importer fetches `https://tweakcn.com/r/themes/<id>` and the served Control UI CSP now allows `https://tweakcn.com`. I did not run a live Tailscale Serve browser repro in this read-only review, but source and regression tests cover the reported root cause.

## Solution Assessment

Yes. The current fix is the narrowest maintainable solution because it allows only the registry origin the importer actually fetches and tests that the CSP does not broaden to wildcard tweakcn hosts or all HTTPS.
