## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve advertised default-agent aliases:** `src/gateway/http-utils.ts:229-230`
  - body: This new guard makes `openclaw/<agentId>` valid only when the id appears in `cfg.agents.list`. The current models endpoint always advertises `openclaw/<defaultAgentId>` from `resolveDefaultAgentId(cfg)`, so an install with the implicit default agent but no list would see `/v1/models` advertise `openclaw/main` and then have compatible requests using that model rejected.
  - confidence: 0.94
- **[P1] Pass the context-token override and fallback:** `src/gateway/session-utils.ts:948-953`
  - body: `getSessionDefaults` used to honor `cfg.agents.defaults.contextTokens` and fall back to `DEFAULT_CONTEXT_TOKENS`; this call passes neither `contextTokensOverride` nor `fallbackContextTokens`. That can make default model status display lose the user-configured cap or return `null` instead of the normal fallback when discovery has no value.
  - confidence: 0.95
- **[P1] Keep backend model overrides on the documented header:** `src/gateway/http-utils.ts:250-261`
  - body: This turns `model: "openclaw/<provider>/<model>"` into a backend model override, but the current docs say the body `model` field selects agent targets and backend provider/model overrides belong in `x-openclaw-model`. Sibling embeddings handling still follows that contract, so this would create endpoint-specific model routing behavior and a second source of truth.
  - confidence: 0.9
