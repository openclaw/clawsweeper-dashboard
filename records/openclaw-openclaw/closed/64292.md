## Summary

Current main treats cross-agent `/subagents spawn <agentId> ...` as allowlist-gated behavior, not something unlocked by `tools.allow: ["*"]`; with only a default `main` agent configured, the reported `/subagents spawn coder test` repro does not establish a current bug.

## Evidence

- **Cross-agent spawns are intentionally blocked unless the target agent is allowlisted:** `spawnSubagentDirect` compares the requested target agent to the requester agent and returns `agentId is not allowed for sessions_spawn (allowed: ...)` when `agents.list[].subagents.allowAgents` or `agents.defaults.subagents.allowAgents` do not include the target.
  - file: [src/agents/subagent-spawn.ts:667](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/agents/subagent-spawn.ts#L667)
  - command: `nl -ba src/agents/subagent-spawn.ts | sed -n '640,685p'`
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **The slash command always treats the first argument as an explicit target agent id:** `/subagents spawn` requires `<agentId> <task>` and passes that `agentId` directly into `spawnSubagentDirect`, so `/subagents spawn coder test` is a cross-agent spawn request, not a same-agent spawn.
  - file: [src/auto-reply/reply/commands-subagents/action-spawn.ts:11](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/auto-reply/reply/commands-subagents/action-spawn.ts#L11)

  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
- **Current user docs say same-agent is the default and document the correct config key:** The Sub-agents docs state that `agents.list[].subagents.allowAgents` and `agents.defaults.subagents.allowAgents` control cross-agent targeting, and that the default is only the requester agent.
  - file: [docs/tools/subagents.md:152](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/docs/tools/subagents.md#L152)

- **Current tests lock in same-agent-only default behavior:** Tests assert that cross-agent spawning is forbidden by default, forbidden when not allowlisted, and accepted once `allowAgents` includes the target or the default allowlist does.
  - file: [src/agents/openclaw-tools.subagents.sessions-spawn.allowlist.test.ts:91](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/agents/openclaw-tools.subagents.sessions-spawn.allowlist.test.ts#L91)
