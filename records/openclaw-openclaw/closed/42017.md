## Summary

Current `main` already exposes an OpenClaw-native ACPX per-agent command override path and wires it into the embedded ACP runtime, so the core request is implemented. There is also current docs coverage for ACPX command overrides in the ACP docs set.

## Evidence

## Risks / Open Questions

- The docs do not appear to spell out the new `plugins.entries.acpx.config.agents.<id>.command` path as directly as the implementation does, so there may still be room for a follow-up docs polish issue even though the requested capability is present on `main`.

## Close Comment

Closing from a Codex review: current `main` already implements the requested OpenClaw-native ACPX command override path. The bundled ACPX config schema accepts per-agent overrides at `plugins.entries.acpx.config.agents.<id>.command` (`extensions/acpx/src/config-schema.ts:26`, `extensions/acpx/src/config-schema.ts:111`), the runtime wires those overrides into the ACP agent registry (`extensions/acpx/src/service.ts:43`), and there is test coverage proving the override is accepted (`extensions/acpx/src/config.test.ts:44`). The bundled plugin metadata also exposes this as `Agent Commands` (`extensions/acpx/openclaw.plugin.json:144`). Docs now also mention ACPX command overrides in the ACP docs set, including `~/.acpx/config.json` for the `openclaw` ACP bridge target (`docs/cli/acp.md:196`) and acpx-config-defined agent aliases on the ACP agents setup page (`docs/tools/acp-agents-setup.md:33`). This appears fixed on `main`; I do not see a tagged release containing it yet, so `fixedRelease` is left unset.
