## What This Changes

The issue asks OpenClaw to preserve causal transcript ordering so a delayed inbound chat user turn cannot be appended after, or parented to, the assistant delivery mirror it triggered.

## Best Possible Solution

Add a core transcript-ordering guard that gives the triggering inbound user turn an authoritative causal anchor before same-rollout delivery mirrors append, with a regression for delayed inbound persistence plus immediate visible delivery.
