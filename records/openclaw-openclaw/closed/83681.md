## Summary

Current main now routes prompt-time delivery-mirror transcript appends through the active session write context, which prevents the visible assistant mirror from racing ahead of the triggering inbound user turn on the reported path.

## What This Changes

The issue asks for chat-sourced transcript mirroring to preserve causal order so delayed inbound user turns cannot be appended after same-turn assistant delivery mirrors or parented to them.

## Best Possible Solution

Keep the current owned transcript write context as the prevention layer, and let the adjacent replay-repair work for already-poisoned sessions continue separately in https://github.com/openclaw/openclaw/pull/84708.

## Reproduction Assessment

Yes for the pre-fix source path: the issue and follow-up comment give a concrete bad transcript shape, and pi-agent-core/OpenClaw source shows why prompt-time mirror writes needed the owned session lock. I did not establish a failing current-main repro because current main already carries the prevention fix.

## Solution Assessment

Yes. The current fix is at the core session/prompt transcript boundary instead of channel-specific policy, and it makes same-prompt delivery mirrors share the active session write lock after the user turn is emitted.
