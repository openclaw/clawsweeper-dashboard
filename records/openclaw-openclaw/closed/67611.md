## Summary

Current `main` already cleans up completed dreaming narrative sessions, scrubs stale dreaming session-store entries/orphan transcripts, and surfaces dream output in the Dream Diary UI instead of leaving it as ordinary sessions.

## Evidence

- **Narrative session cleanup on completion:** `generateAndAppendDreamNarrative()` always attempts `subagent.deleteSession({ sessionKey })` in `finally`, then runs `scrubDreamingNarrativeArtifacts()` to remove leftover dreaming artifacts after the run.
  - file: [extensions/memory-core/src/dreaming-narrative.ts:843](https://github.com/openclaw/openclaw/blob/355c92d69b3803644a6a56247e5e9273bc1d17b9/extensions/memory-core/src/dreaming-narrative.ts#L843)
  - sha: [355c92d69b38](https://github.com/openclaw/openclaw/commit/355c92d69b3803644a6a56247e5e9273bc1d17b9)
- **Stale dreaming entries and orphan transcripts are scrubbed:** The scrubber removes stale `dreaming-narrative-*` session-store entries whose transcript no longer exists and archives orphan dreaming transcripts, while leaving non-dreaming sessions alone.
  - file: [extensions/memory-core/src/dreaming-narrative.ts:716](https://github.com/openclaw/openclaw/blob/355c92d69b3803644a6a56247e5e9273bc1d17b9/extensions/memory-core/src/dreaming-narrative.ts#L716)
