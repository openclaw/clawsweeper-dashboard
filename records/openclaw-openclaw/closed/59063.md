## Summary

Current `main` already has sandbox-path translation and media fallback handling that address the reported host-path vs `/workspace` mismatch for sandboxed agents, including explicit coverage for `/workspace/...`, `file:///workspace/...`, `/tmp/openclaw/...`, and inbound `media/inbound/*` paths.

## Evidence

- **Sandbox path mapper now accepts container `/workspace` paths and OpenClaw temp paths:** `resolveSandboxedMediaSource` maps `/workspace/...` and `file:///workspace/...` into the active sandbox root, then separately allows absolute paths under the preferred OpenClaw temp root instead of rejecting them as sandbox escapes.
  - file: [src/agents/sandbox-paths.ts:103](https://github.com/openclaw/openclaw/blob/4f4288e3b5be2e55abfce5213f0bcf182a252f5e/src/agents/sandbox-paths.ts#L103)
  - command: `nl -ba src/agents/sandbox-paths.ts | sed -n '103,228p'`
  - sha: [4f4288e3b5be](https://github.com/openclaw/openclaw/commit/4f4288e3b5be2e55abfce5213f0bcf182a252f5e)
- **Regression tests cover the reported path classes:** Tests assert that sandbox media resolution accepts `/tmp/openclaw/...`, maps `/workspace/media/pic.png`, maps `file:///workspace/media/pic.png`, and still rejects true out-of-sandbox paths.
  - file: [src/agents/sandbox-paths.test.ts:75](https://github.com/openclaw/openclaw/blob/4f4288e3b5be2e55abfce5213f0bcf182a252f5e/src/agents/sandbox-paths.test.ts#L75)
  - command: `nl -ba src/agents/sandbox-paths.test.ts | sed -n '72,204p'`
  - sha: [4f4288e3b5be](https://github.com/openclaw/openclaw/commit/4f4288e3b5be2e55abfce5213f0bcf182a252f5e)
- **Bridge media resolver rewrites inbound absolute paths into sandbox-visible `media/inbound/*`:** `resolveSandboxedBridgeMediaPath` first resolves through the sandbox bridge, and when that fails it falls back to `media/inbound/<basename>` inside the sandbox workspace. It also returns the container path when the bridge has no host path, avoiding host-only path leakage.
  - file: [src/agents/sandbox-media-paths.ts:21](https://github.com/openclaw/openclaw/blob/4f4288e3b5be2e55abfce5213f0bcf182a252f5e/src/agents/sandbox-media-paths.ts#L21)

- **Tool-level test proves inbound host paths are rewritten for sandbox use:** The image tool test passes a host inbound-media path like `@/Users/.../.openclaw/media/inbound/photo.png` and verifies successful execution via the sandbox with `rewrittenFrom` recorded, which matches the issue's host-path-to-sandbox-path complaint.
  - file: [src/agents/tools/image-tool.test.ts:1207](https://github.com/openclaw/openclaw/blob/4f4288e3b5be2e55abfce5213f0bcf182a252f5e/src/agents/tools/image-tool.test.ts#L1207)
