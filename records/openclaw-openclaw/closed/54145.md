## Summary

Keep PR #54145 open. Current main still has the reported behavior: native Anthropic and Google PDF analysis calls go through direct fetch requests without a timeout signal, and the PR is a narrow fix for that specific gap rather than obsolete or superseded.

## Best Possible Solution

Keep the PR open or land an equivalent focused fix in `src/agents/tools/pdf-native-providers.ts`: add timeout signals to both native PDF provider fetch calls and add a small regression test that verifies the fetch options include a signal for Anthropic and Google. If maintainers prefer a different timeout duration or a named constant, update the PR rather than closing it as obsolete.
