## Summary

Keep this PR open: it targets a real P1 session-state failure, but the patch is not merge-ready because the cleanup only runs behind opt-in transcript rotation and deletes provider-owned reasoning signature metadata outside the existing replay policy.

## What This Changes

The PR adds a successor-transcript transform that strips `signature`, `thinkingSignature`, and `thought_signature` from assistant thinking blocks after compaction, plus regression coverage for rotated transcripts.

## Best Possible Solution

Fix the default compaction/replay path that affected Anthropic sessions use, and make signature stripping obey provider replay policy instead of deleting all compacted thinking signatures generically.

## Reproduction Assessment

Yes for source-level reproduction: current main preserves non-blank thinking signatures through default compaction unless provider replay policy strips them, and the linked report shows the Anthropic error after compaction. I did not run a live Anthropic compaction continuation in this read-only review.

## Solution Assessment

No; the proposed patch is too narrow for the default path and too broad across provider signature fields. The better fix should run where default compaction/replay builds the next request and should respect provider replay policy.
