## Summary

Close as ClawHub/plugin-scope cleanup. The PR adds only a planning document for an optional Mission Control dashboard launcher, while current project guidance says optional plugins should live in their own repositories and publish through ClawHub/npm. Current main already exposes plugin CLI registration, so the proposed v1 launcher does not need a new core API; the requested `dashboard` namespace also conflicts with the core `openclaw dashboard` Control UI command now on main.

## Evidence
