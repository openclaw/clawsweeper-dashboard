## Summary

Keep open. Current main still has flat, budget-limited skill catalog injection and on-demand `SKILL.md` reads, but it does not implement a skill dependency graph, depth-limited dependency expansion, `skills.maxDepth`, or `openclaw skills graph --dot`; related open work covers broader prompt overhead, lazy skill catalogs, or hook narrowing without owning this graph-specific design.

## What This Changes

Request adding a skill dependency graph with on-demand dependency-chain loading, default depth/config controls, and dot-output CLI visualization.

## Best Possible Solution
