## Summary

Close as ClawHub/plugin scope. The request is for a named optional Ombre security/audit integration, while current main already exposes plugin hooks, text transforms, tool/install blocking, and plugin security-audit collector surfaces that are sufficient for an externally maintained Ombre-style plugin. OpenClaw scope docs put optional integrations, official publisher status, provenance, and promotion in ClawHub; broader vendor-neutral security hardening remains tracked separately in #7604.

## What This Changes

Request to add Ombre as an official OpenClaw security and audit integration for prompt-injection blocking, PII redaction, audit logging, hallucination detection, compliance exports, and semantic caching.

## Best Possible Solution

Close the OpenClaw core issue as ClawHub/plugin scope. The Ombre author can publish an externally maintained native OpenClaw plugin or ClawHub package using the existing hook, text-transform, tool-policy, install-policy, and security-audit collector surfaces; any missing generic plugin API should be proposed separately, and non-vendor core security hardening should remain with #7604.
