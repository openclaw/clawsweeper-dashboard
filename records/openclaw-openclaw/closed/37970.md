## Summary

Current `main` already handles bare `/new` and `/reset` with a broader runtime-owned startup flow that covers the PR’s goal: startup files stay in Project Context, recent daily memory is injected explicitly for the first turn, and the default AGENTS template now instructs agents to use that runtime-provided startup context first. The PR’s older approach of re-reading and inlining `## Session Startup` from `AGENTS.md` is obsolete on current `main`.

## Close Comment

Closing as implemented on `main` after Codex review. Current OpenClaw already handles bare `/new` and `/reset` with a runtime-owned startup path: `runPreparedReply()` resolves bare reset state and prepends `buildSessionStartupContextPrelude()` (`src/auto-reply/reply/get-reply-run.ts:358`), and that prelude explicitly injects recent daily memory while noting that bootstrap files are already provided separately (`src/auto-reply/reply/startup-context.ts:309`). `AGENTS.md` also remains part of normal Project Context, and the current template now tells agents to use runtime-provided startup context first (`docs/reference/templates/AGENTS.md:16`). This behavior is covered by the bare `/new` e2e (`src/auto-reply/reply.triggers.trigger-handling.targets-active-session-native-stop.e2e.test.ts:378`) and was shipped in `v2026.4.22`, so this older PR approach is no longer needed.
