## Summary

Current OpenClaw already implements account-scoped DM isolation for multi-account channels. The report's repro uses `session.dmScope: "per-channel-peer"`, which intentionally omits the account dimension; current main and the latest release already provide `per-account-channel-peer`, preserve `deliveryContext.accountId`, and expose that accountId in session listings.

## Evidence

## Risks / Open Questions

- No live Telegram reproduction was run in this read-only review; confidence comes from current main/release code, tests, and docs.
- Users who stay on `session.dmScope: "per-channel-peer"` will continue sharing DM session identity across accounts by design; they need `per-account-channel-peer` for multi-account isolation.
