## Summary

Current main already covers the reported zombie session write-lock cascade: abort cleanup releases after the settle timeout, retained embedded-attempt locks are disposed from outer teardown, and untracked same-process lock files are reclaimable. This fix is current-main-only; it is not in v2026.5.22.

## What This Changes

The issue reports embedded-run abort-settle timeouts leaving session transcript write locks that block later messages from the same chat.

## Best Possible Solution

Keep the current main teardown and lock-reclaim behavior and let it ship in the next release instead of adding another stale-time workaround.
