## Summary

Keep open: the xAI bug fix is valid and the supplied provider proof is strong, but the current patch still has a blocking Grok 4.3 regression in the supported-effort metadata/default payload path.

## What This Changes

The PR adds xAI compat metadata and transport/wrapper tests so older Grok reasoning models with `supportsReasoningEffort=false` omit unsupported reasoning-effort payload controls while Grok 4.3 remains configurable.

## Best Possible Solution

Keep the provider-owned xAI compat gate for unsupported older Grok models, but separate explicit disabled reasoning from the supported-effort list so Grok 4.3 preserves documented default-low behavior unless the user intentionally turns reasoning off.

## Reproduction Assessment

Yes. Current main's source path can send reasoning controls for older xAI reasoning models, and the PR body adds live xAI output showing the unsupported payload is rejected while the fixed payload is accepted.

## Solution Assessment

No. The provider-owned fix direction is right, but the patch should not make default Grok 4.3 Responses calls resolve to `reasoning: { effort: "none" }`.
