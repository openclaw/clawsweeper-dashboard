## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Limit default denies to untrusted channel providers:** `src/agents/pi-tools.message-provider-policy.ts:41-43`
  - body: `messageProvider` is also used for internal and synthetic surfaces, not only anonymous chat channels. With this fallback, any non-empty provider outside the allow map loses exec/read/write before normal policy runs, so owner webchat, heartbeat, ACP, and future trusted provider flows can regress; scope this deny list to known untrusted external auto-reply channels or explicitly exempt trusted/internal providers.
  - confidence: 0.88
- **[P1] Preserve memory flush read and write:** `src/agents/pi-tools.ts:593-597`
  - body: Memory-triggered runs are first reduced to `read` and append-only `write`, then passed through this message-provider filter. Because the default deny list removes both tools for channel providers, channel memory flushes can end up with no tools and silently stop persisting memory unless a group allow happens to re-enable them; bypass or explicitly preserve the memory-flush tool set.
  - confidence: 0.9
- **[P2] Wire exec owner-only through the runtime policy:** `src/agents/tools/owner-only-tools.ts:1`
  - body: Adding `exec` to `OPENCLAW_OWNER_ONLY_CORE_TOOL_NAMES` does not affect the branch's runtime owner-only filter: `applyOwnerOnlyToolPolicy` uses `OWNER_ONLY_TOOL_APPROVAL_CLASS_FALLBACKS`, and the exec tool itself does not set `ownerOnly`. If this defense-in-depth claim stays in scope, add `exec` to the actual runtime policy source and cover that path.
  - confidence: 0.86
