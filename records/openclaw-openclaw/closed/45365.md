## Summary

Current main already guards the agent chat avatar path so non-URL avatar strings do not render as literal text/SVG path data, and the same behavior is present in the shipped release v2026.4.22.

## Evidence

- **Control UI only accepts renderable local/data image avatar URLs:** `isRenderableControlUiAvatarUrl` only matches `data:image/...` and local `/...` URLs, and `resolveAgentAvatarUrl` returns `null` for anything else instead of passing raw avatar text through to the renderer.
  - file: [ui/src/ui/views/agents-utils.ts:201](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/ui/src/ui/views/agents-utils.ts#L201)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Assistant chat avatar falls back to the local logo for non-URL avatar values:** `renderAvatar` renders the assistant avatar as an `<img>` only when the avatar passes the URL check; otherwise it renders the local logo image, which prevents raw SVG path strings from appearing in the chat UI.
  - file: [ui/src/ui/chat/grouped-render.ts:700](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/ui/src/ui/chat/grouped-render.ts#L700)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Regression tests cover the fallback behavior:** Tests assert that non-URL avatar values are rejected and that the assistant chat avatar falls back to the local logo instead of rendering arbitrary avatar text/URLs.
  - file: [ui/src/ui/views/agents-utils.test.ts:117](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/ui/src/ui/views/agents-utils.test.ts#L117)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Chat rendering test verifies logo fallback in the assistant message UI:** The test expects a remote/non-approved assistant avatar to render as `/openclaw-logo.svg`, matching the intended safe fallback path in the session chat view.
  - file: [ui/src/ui/chat/grouped-render.test.ts:245](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/ui/src/ui/chat/grouped-render.test.ts#L245)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Release tag contains the fix path:** I verified the same avatar-sanitizing code and tests are present at the latest shipped tag, so this is not only on unreleased main.
