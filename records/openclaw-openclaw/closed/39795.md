## Summary

Telegram partial-stream preview/final mapping is already implemented on current `main`, and the shipped `v2026.4.22` release tag also contains the fix path. The current dispatcher no longer blindly rotates on every assistant boundary for same-reply cases, archives superseded previews, and finalizes archived previews before falling back to a new send.

## Evidence

- **Same-reply boundary no longer forces a new preview:** `onAssistantMessageStart` now skips rotation when `skipNextAnswerMessageStartRotation` or `pendingCompactionReplayBoundary` is set, instead of always calling `forceNewMessage()`.
