## Summary

Keep open. Current `main` still has the image-generation gap this PR targets: chat/Codex transport normalizes legacy ChatGPT backend URLs to `/backend-api/codex`, but the OpenAI image path still forwards `models.providers.openai-codex.baseUrl` unchanged and then appends `/responses`, with no regression test for the legacy `https://chatgpt.com/backend-api` form.

## Evidence

- **Image path still uses configured Codex base URL verbatim:** `generateOpenAICodexImage` reads `req.cfg.models.providers["openai-codex"].baseUrl` and passes it directly to `resolveProviderHttpRequestConfig`; the request URL is later built as `${baseUrl}/responses`, so a legacy `https://chatgpt.com/backend-api` setting would target `/backend-api/responses` on current `main`.
