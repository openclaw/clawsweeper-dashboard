## Summary

Current `main` already routes native Discord slash/model commands through `CommandTargetSessionKey` to the conversation session, so the slash-session vs channel-session mismatch described in #54485 is obsolete. The same behavior is present in shipped release `v2026.4.22`.

## Evidence

- **Discord native command context carries both slash session and routed target session:** Discord native command contexts still use `SessionKey` for the isolated slash interaction and separately pass `CommandTargetSessionKey` for the routed chat session, which is the mechanism needed to avoid slash/channel key mismatches.
  - file: [extensions/discord/src/monitor/native-command-context.ts:66](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/discord/src/monitor/native-command-context.ts#L66)
