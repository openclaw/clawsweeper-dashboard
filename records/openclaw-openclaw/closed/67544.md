## Summary

Current `main` already hardens the Windows gateway restart path against lingering listeners and mixed Scheduled Task/runtime state, which covers the `EADDRINUSE` restart failure described in #67544. The fix is present on `main` but not in the latest tagged release.

## Evidence

- **Windows scheduled-task restart now clears lingering listeners before relaunch:** `restartScheduledTask` ends the task, terminates verified gateway listeners and Startup-folder fallback runtimes, waits for port release, force-kills any remaining busy-port listeners, and only then reruns the task.
  - file: [src/daemon/schtasks.ts:968](https://github.com/openclaw/openclaw/blob/355c92d69b3803644a6a56247e5e9273bc1d17b9/src/daemon/schtasks.ts#L968)
  - command: `nl -ba src/daemon/schtasks.ts | sed -n '968,1007p'`
  - sha: [355c92d69b38](https://github.com/openclaw/openclaw/commit/355c92d69b3803644a6a56247e5e9273bc1d17b9)
- **Mixed schtasks/runtime state is now detected from the live listener:** `readScheduledTaskRuntime` reports the gateway as running when a verified listener is present even if `schtasks` does not report a running task, matching the issue's mixed-state symptom.
  - file: [src/daemon/schtasks.ts:431](https://github.com/openclaw/openclaw/blob/355c92d69b3803644a6a56247e5e9273bc1d17b9/src/daemon/schtasks.ts#L431)
  - command: `nl -ba src/daemon/schtasks.ts | sed -n '431,458p'`
  - sha: [355c92d69b38](https://github.com/openclaw/openclaw/commit/355c92d69b3803644a6a56247e5e9273bc1d17b9)
- **Daemon restart path retries after stale Windows listeners:** `runDaemonRestart` on Windows enables stale-listener detection, kills stale gateway PIDs, and retries restart before surfacing a failure.
  - file: [src/cli/daemon-cli/lifecycle.ts:241](https://github.com/openclaw/openclaw/blob/355c92d69b3803644a6a56247e5e9273bc1d17b9/src/cli/daemon-cli/lifecycle.ts#L241)
  - command: `nl -ba src/cli/daemon-cli/lifecycle.ts | sed -n '241,268p'`
  - sha: [355c92d69b38](https://github.com/openclaw/openclaw/commit/355c92d69b3803644a6a56247e5e9273bc1d17b9)
- **Windows restart health check treats unattributed busy listeners as stale:** `inspectGatewayRestart` includes unknown busy listeners as stale on Windows when the service is not running, which directly targets leftover `EADDRINUSE` listeners after crashes/overlaps.
  - file: [src/cli/daemon-cli/restart-health.ts:159](https://github.com/openclaw/openclaw/blob/355c92d69b3803644a6a56247e5e9273bc1d17b9/src/cli/daemon-cli/restart-health.ts#L159)
