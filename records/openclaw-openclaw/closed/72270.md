## Summary

Current main implements the bundled plugin runtime-dependency staging fix for the reported v2026.4.24 ENOTEMPTY failures under ~/node_modules. The dependency install path is now OpenClaw-managed, npm is anchored to a project-local install root, inherited global npm config is stripped, and tests/e2e coverage guard against writing bundled runtime deps into HOME node_modules or the packaged plugin tree.

## Best Possible Solution

Close this issue as implemented on current main. Keep the managed bundled-runtime-deps staging path and regression coverage; if ENOTEMPTY still reproduces after an upgrade containing this fix, ask for a fresh report with startup logs, install method, OPENCLAW_PLUGIN_STAGE_DIR or systemd STATE_DIRECTORY details, and the affected channel/plugin list.
