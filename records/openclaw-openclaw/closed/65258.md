## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Show auth mismatch errors instead of reconnecting forever:** `ui/src/ui/app-gateway.ts:375-386`
  - body: This path clears `lastError` and returns for every recoverable close after a prior connection. That includes an exhausted `AUTH_TOKEN_MISMATCH` retry path, leaving operators in a reconnecting state without the credential error they need to fix chat access.
  - confidence: 0.88
- **[P2] Mark first-seen result events as complete:** `ui/src/ui/app-tool-stream.ts:573-574`
  - body: When a tool call is first observed on a successful `phase === "result"` event, the new entry initializes as `running` and never reaches the later branch that marks it complete, so recovered completed tools can stay running indefinitely.
  - confidence: 0.9
- **[P2] Replay queued aborts after reconnect:** `ui/src/ui/app-gateway.ts:338-345`
  - body: `handleAbortChat` still queues `pendingAbort` while disconnected, but this reconnect hello path clears local run state without replaying it. Pressing Stop during a disconnect can make the UI look stopped while the gateway run keeps executing.
  - confidence: 0.86
- **[P2] Preserve attachments when retrying prompts:** `ui/src/ui/chat/grouped-render.ts:1197-1202`
  - body: The retry action forwards only extracted markdown text. For user turns that included files or images, retry silently drops the attachments and sends a materially different prompt.
  - confidence: 0.82
- **[P2] Allow stored space overrides to win when edited:** `ui/src/ui/chat-spaces.ts:47-50`
  - body: `setChatSpace` writes a local override, but `resolveChatSpace` always returns backend `sessionSpace` first. Sessions that already have a space appear editable but snap back on the next render.
  - confidence: 0.8
- **[P2] Use attachment kind in generated labels:** `ui/src/ui/views/chat.ts:700-703`
  - body: The context rail labels every attachment as `Image N` even after computing the MIME-derived kind. Audio, video, or document attachments therefore get incorrect visible and accessibility labels.
  - confidence: 0.78
