## What This Changes

Remove CLI proxy SIGTERM/SIGINT listeners after normal command completion so late signals cannot interrupt remaining teardown.

## Best Possible Solution

Keep the shipped runCli cleanup as-is; reopen or file a new issue only if a concrete signal-race reproduction bypasses the existing final handler removal.

## Solution Assessment

Yes. The existing implementation is the narrow maintainable solution: remove the signal handlers in both the signal shutdown path and normal completion path before proxy and memory-manager teardown.
