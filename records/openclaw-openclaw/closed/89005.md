## What This Changes

The PR adds title-preservation logic to session-store upserts and regression tests for empty/null versus non-empty replacement title metadata.

## Best Possible Solution

Land the narrow guard only if maintainers agree that empty/null replacement upserts should preserve human titles, with regression coverage and `sessions.patch` retaining explicit clear/rename semantics.

## Reproduction Assessment

Yes, from source: current main persists `params.entry` directly in `upsertSessionEntry`, so empty/null replacement title metadata can overwrite stored `label` and `displayName`. I did not run tests because this review is read-only.

## Solution Assessment

Yes, with maintainer-owned compatibility acceptance: the session-store upsert boundary is the narrowest place to prevent replacement metadata from wiping stable titles, while `sessions.patch` remains the explicit rename/clear path.
