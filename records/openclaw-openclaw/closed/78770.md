## Summary

Keep open: this PR addresses a real WhatsApp duplicate-media regression and includes sufficient live proof, but the current coalescer can still drop legitimate non-overlapping media from deferred payloads.

## What This Changes

The PR buffers WhatsApp media-only interim auto-reply payloads in the inbound dispatcher and drops same-URL duplicates before sending a later captioned media payload, with dispatcher test updates.

## Best Possible Solution

Keep dispatcher-level coalescing, but make dedupe URL-granular so only duplicate media is suppressed while unique media still sends, with regression coverage and a changelog entry.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path through the dispatcher and its existing test, and the linked issue plus PR proof provide redacted live WhatsApp logs and screenshots for the duplicate-send behavior.

## Solution Assessment

No, not yet. The dispatcher is the right boundary, but this implementation suppresses whole deferred media payloads on partial overlap instead of filtering only duplicated URLs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve non-overlapping deferred media:** `extensions/whatsapp/src/auto-reply/monitor/inbound-dispatch.ts:171-173`
