## What This Changes

## Best Possible Solution

Add a narrow OpenAI/Codex raw server-error classifier with regression coverage and a changelog entry.

## Solution Assessment

Yes: a narrow classifier and regression test for the exact provider-error wording is the best path. A broad policy change that treats generic server prose as failover-worthy would be riskier.
