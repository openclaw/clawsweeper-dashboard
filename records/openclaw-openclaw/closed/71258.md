## Summary

Current `main` and the released `v2026.4.23` docs already cover the issue’s core guidance: same-host loopback proxies must use token/password instead of `trusted-proxy`, non-loopback browser access requires explicit `gateway.controlUi.allowedOrigins`, and Linux startup paths already auto-apply `NODE_EXTRA_CA_CERTS` for the CLI/service path.

## Evidence
