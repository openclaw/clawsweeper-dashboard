## What This Changes

The issue requests durable session-transcript persistence for assistant text from WebChat agent runs so history reloads do not erase prior replies.

## Best Possible Solution

Keep the current Pi-owned persistence boundary and route any remaining transcript branch/rewrite loss to #77012 instead of adding duplicate WebChat JSONL mirrors.

## Reproduction Assessment

## Solution Assessment

Yes. The maintainable fix is to keep normal agent-run transcript writes in Pi SessionManager and prevent stale history reloads from wiping live UI state, rather than mirroring every WebChat final payload into JSONL and risking duplicate assistant turns.
