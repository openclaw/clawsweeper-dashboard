## Summary

Keep open: the PR fixes the standard Telegram `deliverReplies` send path, but visible final replies delivered by preview finalization still bypass that path, and the new mirror append does not preserve the configured session write-lock timeout.

## What This Changes

The PR adds a Telegram `transcriptMirror` option to `deliverReplies`, passes session and agent context from message dispatch, re-exports the transcript mirror helper through the plugin SDK agent-harness runtime, and updates the SDK API hash.

## Best Possible Solution

Mirror every successful Telegram final assistant delivery, including fresh sends and preview-finalized edits, through the existing transcript helper with focused regression coverage and a changelog entry.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main has no transcript append in `deliverReplies`, and the existing preview-finalized test shows a visible final reply can avoid `deliverReplies` entirely. I did not run a live Telegram session in this read-only review.

## Solution Assessment

No. The proposed direction is right for normal `deliverReplies` sends, but it is incomplete because preview-finalized final replies remain outside the mirror path and the new append should carry the configured write-lock settings.
