## Summary

Keep open: current main and v2026.6.10 still use the unbounded home-prefix check in the session tool path renderer, while this PR is a narrow, mergeable fix with focused tests and now-sufficient helper-level real behavior proof.

## What This Changes

The PR updates the session tool `shortenPath` helper to shorten only exact home paths or separator-bounded home subpaths, and adds focused regression tests.

## Best Possible Solution

Land the narrow helper and regression-test fix after ordinary maintainer review, preserving copy-pasteable session tool paths without changing the broader path-display policy.

## Reproduction Assessment

Yes. Source inspection shows current main and v2026.6.10 rewrite any string starting with `os.homedir()`, so a sibling path such as `${home}extra/app.ts` would be displayed as `~extra/app.ts`; I did not run a live TUI scenario in this read-only review.

## Solution Assessment

Yes. The PR fixes the local session renderer at the implicated helper and matches the separator-bounded behavior already used by `shortenHomePath`; importing the shared helper would also change this renderer's current OS-home-only display semantics.
