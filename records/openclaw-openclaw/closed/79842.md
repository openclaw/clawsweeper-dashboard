## Summary

Current main already preserves the Claude CLI execution route while canonicalizing legacy `claude-cli/*` model refs to `anthropic/*`, so the reported cold-start billing failure is fixed on main but not yet proven shipped in v2026.5.7.

## What This Changes

The issue asks doctor/upgrade to stop converting a Claude CLI billed default model into a direct Anthropic billing route during config migration.

## Best Possible Solution

Keep the current canonical `anthropic/*` model ref plus model-scoped `claude-cli` runtime policy, and let the next release carry the fix to affected v2026.5.7 users.

## Reproduction Assessment

No. I did not establish a failing path on current main; the current source and regression test show doctor now adds the `claude-cli` runtime policy when it rewrites legacy Claude CLI refs.

## Solution Assessment

Yes. The current implementation uses the documented provider/model split and preserves billing behavior through model-scoped runtime policy, which is narrower than keeping legacy whole-agent runtime pins alive.
