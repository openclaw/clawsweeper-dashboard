## Summary

Keep open: current main has merged the compaction read-side guard and the Codex dynamic-tool classifier fix, but the generic embedded runner still preserves an incoming `isError: true` flag for an accepted `sessions_spawn` result. The remaining source-side fix is already represented by the open linked PR, so this issue should stay open until that path is either landed or explicitly closed as superseded.

## What This Changes

Fix accepted `sessions_spawn` results so transcript persistence and compaction summaries treat accepted child launches as successes while keeping forbidden or errored launches as failures.

## Best Possible Solution

Review and land a narrow source-side accepted-spawn guard for the generic embedded-runner path, preserving the already-merged compaction and Codex behavior without duplicating those changes.

## Reproduction Assessment

Yes. Source inspection shows a current-main path where an accepted `sessions_spawn` tool result that arrives with `event.isError === true` is still returned with `isError: true` by the generic embedded-runner middleware.

## Solution Assessment

Yes. The narrowest maintainable fix is to reuse the existing accepted-spawn predicate at the embedded-runner classification boundary, while keeping forbidden/error statuses and non-spawn lookalikes classified as failures.
