# Commit 1fc92dd

## Summary

Found one low-severity table-rendering regression in the new ANSI continuation logic. It affects multiline cells whose wrapped styled section has multiple active SGR sequences, such as bold plus color.

## Findings

### Low: Wrapped multiline cells drop combined ANSI styles after the second continuation line

- Kind: regression
- File: `src/terminal/table.ts`
- Line: 349
- Evidence: `activeSgrAfter` returns active styles as one concatenated string at `src/terminal/table.ts:302`, then `flushAt` stores that string as a single synthetic ANSI token at `src/terminal/table.ts:336` and `src/terminal/table.ts:349`. On the next wrap, `parseSgrParams` at `src/terminal/table.ts:174` can only parse one `ESC[...m` sequence per token, so a rehydrated value like `\x1b[1m\x1b[31m` is ignored. A focused runtime probe with `plain\n\x1b[1m\x1b[31m${"a".repeat(80)}\x1b[0m` showed the first two wrapped styled lines kept bold/red, but the remaining three wrapped content lines lacked both style prefixes.
- Impact: The commit’s intended “preserve multiline table colors” behavior still fails for composed styles after an unstyled prefix or explicit newline. Any table cell that uses multiple active SGR properties in the wrapped multiline portion will lose styling partway through the cell.
- Suggested fix: Keep rehydrated active SGR as separate ANSI tokens, or make the synthetic-token parser split and process concatenated SGR sequences before calculating active state. Add a regression test using two active SGR sequences after an explicit newline and enough content to wrap at least three styled lines.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Run a focused probe or add a test with a cell value like `plain\n\x1b[1m\x1b[31m${"a".repeat(80)}\x1b[0m` at narrow table width, then assert every wrapped `a...` line contains both `\x1b[1m` and `\x1b[31m`.
- Is this the best way to solve the issue? yes. The bug is in the representation of rehydrated active SGR state, so preserving token boundaries or parsing concatenated SGR sequences directly addresses the failure without changing table layout behavior.
