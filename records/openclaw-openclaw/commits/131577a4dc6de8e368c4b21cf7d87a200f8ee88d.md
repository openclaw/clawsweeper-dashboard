# Commit 131577a

## Summary

Found one reliability regression in the new queue-preservation gate: stale `queued` workflow runs now block all later Mantis Telegram runs indefinitely.

## Findings

### Medium: stale queued Mantis runs can block every later Telegram proof run

- Kind: reliability
- File: `.github/workflows/mantis-telegram-desktop-proof.yml:314`
- Line: 314
- Evidence: The new loop treats every older run with status `queued` as a blocker. The same logic is duplicated in `.github/workflows/mantis-telegram-live.yml:278`. Live Actions state currently has seven older Mantis Telegram runs still in `queued` from `2026-05-15T07:46:13Z` through `2026-05-15T08:06:34Z`, and `gh run view` shows queued runs with no jobs. Running the committed blocker filter against a later `current_created` returns all seven queued runs, so a fresh run would keep sleeping behind them until the account job times out.
- Impact: A stale GitHub Actions queued run that is not holding the Telegram account can make new desktop-proof/live QA runs unusable. This is especially risky because the affected workflows are manual/maintainer proof workflows, and the failure mode is a long wait rather than an immediate actionable error.
- Suggested fix: Bound queued blockers with a stale-age policy or explicitly fail with a cleanup instruction when an older queued run has not created jobs/updated within a reasonable TTL. Keep blocking active `in_progress`/`waiting` account consumers, but do not let indefinitely stale pre-job queue entries hold the account lock forever.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. The read-only reproduction is the committed `gh run list --status ... | jq` blocker expression using a synthetic later `current_created`; it returns the stale queued run ids.
- Is this the best way to solve the issue? yes. The account lock should be tied to live or recently queued work, not unbounded historical queued runs that have no jobs and cannot currently be using the shared Telegram account.
