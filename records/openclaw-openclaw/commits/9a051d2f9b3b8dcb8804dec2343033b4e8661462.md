# Commit 9a051d2

## Summary

Found one low-severity regression in the Parallels smoke scripts: the new dev-channel preseed assumes `openclaw.json` already exists, but the documented `--skip-latest-ref-check` path can reach the update step without onboarding and without a config file.

## Findings

### Low: `--skip-latest-ref-check` can break dev update lanes before update runs

- Kind: regression
- File: `scripts/e2e/parallels/macos-smoke.ts`, `scripts/e2e/parallels/windows-smoke.ts`
- Line: `scripts/e2e/parallels/macos-smoke.ts:861`, `scripts/e2e/parallels/windows-smoke.ts:802`
- Evidence: `runUpgradeLane()` sets `upgradePrecheck = "skipped"` when `--skip-latest-ref-check` is passed, so it skips `captureLatestRefFailure()` and its `runRefOnboard()` config creation. The macOS lane also removes `$HOME/.openclaw` before installing with `OPENCLAW_NO_ONBOARD=1`; Windows installs with `-NoOnboard`. The commit then added unconditional reads of `.openclaw/openclaw.json` before `openclaw update --channel dev`. A focused local reproduction of the inserted read path fails with `ENOENT` for Node and `ItemNotFoundException` for PowerShell when the config is absent.
- Impact: macOS and Windows upgrade smoke runs using the documented `--skip-latest-ref-check` option, or any path where the precheck fails before writing config, fail in the preseed step before exercising the dev update they were meant to test.
- Suggested fix: make the preseed create the state directory and start from `{}` when `openclaw.json` is missing or unreadable, then set `update.channel = "dev"`.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Run the macOS or Windows upgrade smoke with `--skip-latest-ref-check` from a clean/pre-openclaw snapshot; the preseed reads `.openclaw/openclaw.json` before any onboarding step creates it.
- Is this the best way to solve the issue? yes. The intent is to preseed one config field for update status, so creating a minimal config when absent is narrower than forcing the skipped precheck to run.
