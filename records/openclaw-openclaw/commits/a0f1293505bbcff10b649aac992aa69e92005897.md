# Commit a0f1293

## Summary

Found one actionable test-gate regression in the bundled plugin runtime smoke: channel plugin visibility is now logged as a skip instead of failing the lane.

## Findings

### Low: Channel plugins can disappear from `channels.status` without failing the bundled runtime smoke

- Kind: test_gap
- File: `scripts/e2e/lib/bundled-plugin-install-uninstall/runtime-smoke.mjs`
- Line: 376
- Evidence: This commit replaces the previous `assertChannelVisible(status, channel)` failure with a `console.log(...)` skip when `channels.status` does not include a manifest-declared channel. The gateway handler builds `channelMeta`, `channels`, and `channelAccounts` from `listChannelPlugins()` for every loaded channel plugin, even with `probe: false` (`src/gateway/server-methods/channels.ts:289`). The smoke also now explicitly enables the plugin and its channel before startup, so absence from dry `channels.status` is a concrete “plugin did not load/register as a channel” signal, not just an expected unconfigured state.
- Impact: A bundled channel plugin can install successfully but fail to register with the gateway channel registry, and this Docker lane will still pass as long as the gateway stays up. That weakens a release-path smoke that is supposed to catch broken bundled channel/plugin runtime loading.
- Suggested fix: Keep the assertion for `channelMeta`/`channels`/`channelAccounts` visibility after `activateSmokePlugin(...)`, or only allow a skip behind an explicit, documented per-plugin exception list with a reason.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Any regression that prevents a bundled channel plugin from registering in `listChannelPlugins()` will make `channels.status` omit the channel; after this commit, `runtime-smoke.mjs` logs the skip and continues instead of failing.
- Is this the best way to solve the issue? yes. The existing `channels.status` contract already exposes loaded channel plugins without probing, so restoring the assertion preserves the lane’s intended signal.
