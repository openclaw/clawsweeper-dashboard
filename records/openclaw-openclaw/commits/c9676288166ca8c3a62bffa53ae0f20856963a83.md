# Commit c967628

## Summary

Found one actionable compatibility issue: the restored Telegram outbound poll cap is stale at 10 options, while current Telegram Bot API and pinned grammY types allow 12.

## Findings

### Low: Telegram outbound rejects valid 11- and 12-option polls

- Kind: compatibility
- File: `extensions/telegram/src/outbound-adapter.ts`
- Line: 27
- Evidence: This commit adds `TELEGRAM_POLL_OPTION_LIMIT = 10` and wires it into `pollMaxOptions` at `extensions/telegram/src/outbound-adapter.ts:199`. Core and gateway poll paths use that adapter field to call `normalizePollInput(..., { maxOptions })`, so 11-option Telegram polls fail with `Poll supports at most 10 options`. Current official Telegram Bot API documents `sendPoll.options` as a JSON-serialized list of 2-12 answer options, and the installed `grammy@1.42.0` type docs at `node_modules/grammy/out/core/api.d.ts:417` also say 2-12 strings. ([core.telegram.org](https://core.telegram.org/bots/api))
- Impact: Telegram users cannot send otherwise valid native polls with 11 or 12 options through OpenClaw’s shared outbound path. This also conflicts with OpenClaw’s gateway schema, which already accepts up to 12 options at `src/gateway/protocol/schema/agent.ts:115`.
- Suggested fix: Raise the Telegram poll limit to 12 and keep all Telegram poll guards aligned, including `extensions/telegram/src/send.ts:1560` and the outbound adapter test.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. `pnpm exec tsx -e "import { normalizePollInput } from './src/polls.ts'; const opts=Array.from({length:11},(_,i)=>String(i+1)); try { normalizePollInput({question:'Q', options:opts}, {maxOptions:10}); console.log('accepted'); } catch (err) { console.log(err instanceof Error ? err.message : String(err)); }"` prints `Poll supports at most 10 options`.
- Is this the best way to solve the issue? yes, update the Telegram-owned cap to match Telegram’s current contract and the pinned grammY type surface; do not widen the generic core default beyond each channel’s adapter contract.
