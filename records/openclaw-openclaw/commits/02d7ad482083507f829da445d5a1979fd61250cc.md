# Commit 02d7ad4

## Summary

Found one regression in the new plugin-only allowlist fast path: some explicit core tool allowlists are misclassified as plugin-only, so those runs skip core tool construction and fail with no callable tools.

## Findings

### Medium: Explicit `process` and heartbeat allowlists can drop the requested core tool

- Kind: regression
- File: `src/agents/pi-embedded-runner/run/attempt.ts`
- Line: 500
- Evidence: `shouldBuildCoreCodingToolsForAllowlist()` decides whether to pass `includeCoreTools: false`, but its hard-coded core set omits the shipped `process` tool and contains `heartbeat_response` while the actual heartbeat tool name is `heartbeat_respond`. `runEmbeddedAttempt()` passes that result into `createOpenClawCodingTools()` at `src/agents/pi-embedded-runner/run/attempt.ts:888`; when false, `src/agents/pi-tools.ts:668` only appends `processTool` if `includeCoreTools` is true, and heartbeat is only created through the core `createOpenClawTools()` path at `src/agents/pi-tools.ts:672`.
- Impact: A restricted run such as a cron/agent turn with `toolsAllow: ["process"]`, or a heartbeat run restricted to `["heartbeat_respond"]`, is now treated as plugin-only. Core tools are never built, final allowlist filtering cannot recover them, and the run hits the existing preflight “No callable tools remain” path instead of submitting to the model.
- Suggested fix: Stop maintaining a separate string list here, or derive it from the canonical tool catalog/actual tool constants. At minimum add `process` and replace `heartbeat_response` with `HEARTBEAT_RESPONSE_TOOL_NAME`, with focused tests for both runtime allowlists.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Run an embedded attempt with `toolsAllow: ["process"]` or a heartbeat attempt with `toolsAllow: ["heartbeat_respond"]`; the new predicate returns false, `includeCoreTools` is false, and the requested core tool is never constructed.
- Is this the best way to solve the issue? yes for the immediate fix: the fast path should only skip core construction when it can prove no requested non-plugin tool is needed. A catalog/constant-backed predicate is safer than another hand-maintained duplicate list.
