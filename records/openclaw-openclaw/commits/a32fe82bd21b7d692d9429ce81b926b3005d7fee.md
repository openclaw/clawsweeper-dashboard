# Commit a32fe82

## Summary

Found one low-severity regression in the oversized-line placeholder path for tree-shaped transcripts.

## Findings

### Low: Oversized placeholders are dropped from tree transcript history

- Kind: bug
- File: `src/gateway/session-utils.fs.ts`
- Line: 276
- Evidence: `buildOversizedTranscriptRecord()` returns a placeholder record without the original tree `id` / `parentId`, and `parseTailTranscriptRecord()` returns it before parsing the JSON line. For tree transcripts, `selectBoundedActiveTailRecords()` only keeps records with tree links. An oversized active parent is therefore absent from `byId`, so traversal from the leaf stops at the child and drops both the placeholder and earlier ancestors.
- Impact: For current version 3 tree transcripts, `chat.history` / recent history can still omit the oversized turn instead of showing the new explicit `[chat.history omitted: message too large]` placeholder. A focused smoke with `root -> huge(300 KiB) -> tail` returned only `tail` for both sync and async recent readers, with no placeholder.
- Suggested fix: Preserve enough tree metadata for oversized placeholders. A cheap option is to parse only the small structural envelope (`id`, `parentId`, `type`) before replacing the message payload, or otherwise keep unparsed oversized records in active-branch traversal so the placeholder remains between parent and child.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Create a version 3 transcript with `root` parent, an oversized `huge` message whose `parentId` is `root`, and a `tail` message whose `parentId` is `huge`; call `readRecentSessionMessages()` or `readRecentSessionMessagesAsync()` with a byte window large enough to include all lines. Current output contains only `tail`.
- Is this the best way to solve the issue? unclear. The fix should preserve tree linkage without parsing the full oversized payload; exact implementation depends on whether transcript JSONL has a stable structural envelope that can be scanned safely.
