# Commit 7641783

## Summary

Found one concrete regression in the bundled capability runtime helper: declared bundled plugin tool contracts are not copied onto the runtime record before the new enforcement runs, so declared tools are falsely rejected on that path.

## Findings

### Low: Bundled capability runtime rejects declared bundled tools

- Kind: bug
- File: `src/plugins/bundled-capability-runtime.ts`
- Line: 263
- Evidence: `loadBundledCapabilityRuntimeRegistry` reads the manifest at lines 255-263, but `createCapabilityPluginRecord(...)` does not accept or set `contracts` on the returned `PluginRecord` at lines 131-176. The new enforcement then calls `normalizePluginToolContractNames(record.contracts)` at line 484, so the declared set is always empty on this path. A focused runtime probe against current `main` confirmed `loadBundledCapabilityRuntimeRegistry({ pluginIds: ["tavily"] })` returns `tools: []` and diagnostics `plugin must declare contracts.tools for: tavily_search` and `tavily_extract`, even though `extensions/tavily/openclaw.plugin.json:22` declares both tool names.
- Impact: Any consumer of `loadBundledCapabilityRuntimeRegistry(...).tools` now loses declared bundled tools and receives false error diagnostics. This does not appear to break the normal `resolvePluginTools` path, but it makes the bundled capability registry internally inconsistent and leaves the new contract enforcement untested on that runtime path.
- Suggested fix: Pass `manifest.contracts` into `createCapabilityPluginRecord` and store it on the returned record, matching `createPluginRecord` behavior in `src/plugins/loader-records.ts`.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes: after installing missing deps, `pnpm exec tsx -e 'import { loadBundledCapabilityRuntimeRegistry } from "./src/plugins/bundled-capability-runtime.ts"; ... pluginIds: ["tavily"] ...'` showed the false undeclared-tool diagnostics and empty tool registry.
- Is this the best way to solve the issue? yes: the manifest record already has the correct contracts; the capability record just drops them before enforcement.
