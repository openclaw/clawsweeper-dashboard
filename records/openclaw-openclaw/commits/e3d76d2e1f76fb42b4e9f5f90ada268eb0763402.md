# Commit e3d76d2

## Summary

Found one packaging/runtime regression affecting bundled Matrix setup in published installs.

## Findings

### Medium: Bundled Matrix cannot bootstrap its runtime dependencies from the packaged manifest

- Kind: regression
- File: `extensions/matrix/package.json`
- Line: 19
- Evidence: This commit makes `dist/extensions/matrix/**` part of the core package by removing the root package exclusion, while the Matrix package manifest that is copied into dist still contains workspace dev dependencies: `@openclaw/plugin-sdk: "workspace:*"` and `openclaw: "workspace:*"` at `extensions/matrix/package.json:19`. Matrix’s dependency bootstrap runs `npm install --omit=dev --silent` when the packaged plugin root has no `pnpm-lock.yaml` (`extensions/matrix/src/matrix/deps.ts:321`). Live check with the exact Matrix `package.json` failed immediately: `npm error Unsupported URL Type "workspace:": workspace:*`. Root `package.json` also does not include Matrix runtime deps like `matrix-js-sdk`, `@matrix-org/matrix-sdk-crypto-nodejs`, `@matrix-org/matrix-sdk-crypto-wasm`, `fake-indexeddb`, or `music-metadata`, so a fresh published core install has to rely on that failing bootstrap path.
- Impact: New packaged OpenClaw installs can show Matrix as bundled, but Matrix setup/runtime dependency installation fails before installing required SDK packages. Users cannot complete Matrix setup unless they manually repair the packaged plugin manifest or install dependencies some other way.
- Suggested fix: Strip workspace-only `devDependencies` from bundled plugin `package.json` files when copying metadata into `dist`, or write a sanitized Matrix runtime-install manifest before packaging. Add a focused test that runs `npm install --omit=dev` against the packaged Matrix manifest shape. A heavier alternative is to internalize all Matrix runtime deps in root `dependencies`.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Copy `extensions/matrix/package.json` into an empty temp package and run `npm install --omit=dev --package-lock=false`; npm 11.11.0 rejects the `workspace:*` dev dependency before installing runtime deps.
- Is this the best way to solve the issue? yes. Sanitizing the packaged plugin manifest preserves the intended lazy Matrix dependency install without adding large Matrix deps to the root package.
