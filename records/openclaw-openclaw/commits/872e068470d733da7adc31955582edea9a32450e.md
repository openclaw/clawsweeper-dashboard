# Commit 872e068

## Summary

Found one low-severity compatibility risk in the test helper change. The focused contract test passes in this full checkout, but the new `git ls-files` enumeration can return paths that are absent from a sparse working tree and then crash when the scanner reads them.

## Findings

### Low: Git-index source scan can break sparse-checkout test runs

- Kind: compatibility
- File: `src/plugins/contracts/plugin-sdk-subpaths.test.ts`
- Line: 334
- Evidence: `listTrackedRepoTsFiles()` now uses `git ls-files -- <root>` and returns tracked `.ts` paths without checking whether those files exist in the working tree. `findRepoFilesContaining()` later reads each returned path via `readCachedSource()`. A disposable sparse-checkout fixture showed `git ls-files -- src` still listed omitted tracked files, while only the sparse path existed on disk.
- Impact: In a sparse checkout that includes this contract test but not all of `src`, the test can fail with `ENOENT` instead of scanning the present tree or reporting a clear full-checkout requirement. The repo validation guidance explicitly calls out linked/sparse checkouts for local proof commands, so this is a plausible local validation regression.
- Suggested fix: Filter `git ls-files` results through an existence check before returning them, or detect sparse checkout/missing indexed files and fall back to the filesystem walk for present files. If the contract intentionally requires a full checkout, fail with an explicit skip/error message before `readFileSync`.
- Confidence: medium

## Details

- Do we have a high-confidence way to reproduce the issue? Yes for the underlying Git behavior: in a temporary sparse Git fixture, `git ls-files -- src` listed `src/b/b.ts` even though only `src/a/a.ts` existed in the sparse worktree. I did not mutate the target repo into sparse mode.
- Is this the best way to solve the issue? Unclear. Filtering to existing files preserves sparse local proof behavior; an explicit full-checkout requirement is safer if maintainers want this contract to scan every tracked source file.
