# Commit 99c88629c3

## Summary

One low-severity compatibility issue found: the OpenClaw-hosted Peekaboo bridge now inherits newly allowlisted Browser MCP operations from Peekaboo 3.2.1, but OpenClaw’s custom service provider does not implement those browser methods.

## Findings

### Low: OpenClaw bridge advertises unsupported Browser MCP operations

- Kind: compatibility
- File: `apps/macos/Sources/OpenClaw/PeekabooBridgeHostCoordinator.swift`
- Line: 68
- Evidence: `PeekabooBridgeServer` is constructed without an explicit `allowedOperations`, so Peekaboo 3.2.1 uses its default remote allowlist. That default now includes `browserStatus`, `browserConnect`, `browserDisconnect`, and `browserExecute`; the same dependency’s `PeekabooBridgeServiceProviding` supplies default browser methods that throw `operationNotSupported`. OpenClaw’s `OpenClawPeekabooBridgeServices` adds `desktopObservation` but does not override any browser methods.
- Impact: signed Peekaboo clients can see Browser MCP operations as supported/enabled in handshake metadata, then fail when they try to use the OpenClaw-hosted bridge for browser status/connect/execute. This is a false capability contract introduced by the dependency bump.
- Suggested fix: pass an explicit `allowedOperations` set for OpenClaw’s bridge that excludes the four browser operations, or implement the browser bridge methods against OpenClaw’s intended browser-control service.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes: enable the macOS Peekaboo bridge, perform a Peekaboo bridge handshake, observe browser operations in `supportedOperations`/`enabledOperations`, then call `browserStatus` or `browserExecute`; the dependency default implementation throws `Browser MCP is not supported by this bridge host`.
- Is this the best way to solve the issue? yes: the bridge should advertise only operations backed by this host’s service provider. Excluding unsupported browser operations is the bounded fix unless OpenClaw intentionally wants this bridge to own Browser MCP.
