## Findings

### Low: About screen reports “Up to date” when gateway update status is unknown

- Kind: bug
- File: `apps/android/app/src/main/java/ai/openclaw/app/ui/SettingsScreens.kt`
- Line: 808
- Evidence: Current `main` still carries the reviewed commit’s About logic after the later v2 settings rename. `latestVersion` is derived from `gatewayUpdateAvailable`, which is `null` before any operator gateway connection. The row then renders `latestVersion?.let { "v$it available" } ?: "Up to date"` and marks it healthy with `latestVersion == null`, so a disconnected or not-yet-loaded gateway shows `Update: Up to date` with an OK pill even though no update check result is available.
- Impact: Users can get false assurance that the gateway is current when the app is offline or has not received gateway update state yet.
- Suggested fix: Distinguish “unknown/not connected” from “connected and no update available”. For example, only show `Up to date` when `isConnected` and gateway version/update state has loaded; otherwise show `Waiting`/`Not connected` and avoid the success status.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes: launch the Android app with no operator gateway connection and open Settings > About. With `isConnected=false`, `gatewayVersion=null`, and `gatewayUpdateAvailable=null`, the code renders the Update row as `Up to date`.
- Is this the best way to solve the issue? yes: the fix should stay in the UI state derivation, because the gateway protocol correctly uses `null` to mean no update available once connected, while the UI also needs a separate “no data yet” state.
