# Commit 7cfa12f

## Summary

Found one regression in the embedded raw model-run path: the new model identity line is reintroduced after raw runs explicitly clear the system prompt.

## Findings

### Medium: raw gateway model probes no longer send only the supplied prompt

- Kind: regression
- File: `src/agents/pi-embedded-runner/run/attempt.ts`
- Line: 2721
- Evidence: Raw runs are detected with `params.modelRun === true || params.promptMode === "none"` and twice reset the session/system prompt to `""` (`src/agents/pi-embedded-runner/run/attempt.ts:786`, `src/agents/pi-embedded-runner/run/attempt.ts:1688`, `src/agents/pi-embedded-runner/run/attempt.ts:2199`). The new unconditional append at `src/agents/pi-embedded-runner/run/attempt.ts:2721` then calls `appendModelIdentitySystemPrompt` after hooks are skipped for raw runs, and that helper turns an empty system prompt into `Current model identity: ...`. `openclaw infer model run --gateway` sets `modelRun: true` and `promptMode: "none"` at `src/cli/capability-cli.ts:786`, while docs say gateway model run remains a raw probe sending the supplied prompt and attachments without agent context at `docs/cli/infer.md:137`.
- Impact: Gateway raw model probes are no longer byte/semantics-equivalent raw provider checks. This can skew provider smoke tests, exact-output probes, and debugging comparisons that rely on `model run --gateway` not adding hidden instructions.
- Suggested fix: Skip the post-hook `appendModelIdentitySystemPrompt` block when `isRawModelRun` is true, and add coverage with a non-empty `runtimeInfo.model` asserting raw runs keep `session.agent.state.systemPrompt === ""`.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. The source path is deterministic: `openclaw infer model run --gateway ...` sets `modelRun: true`/`promptMode: "none"`, raw setup clears `systemPromptText`, then the new unconditional append restores a model-identity system prompt before `activeSession.prompt(...)`.
- Is this the best way to solve the issue? yes. The feature is correct for normal agent prompts and hook/custom overrides, but raw probes are an explicit exception; gating the append with `!isRawModelRun` preserves both contracts.
