# Commit 4babd925

## Summary

Found one compatibility regression in the public plugin SDK compatibility shim. The internal runtime behavior of the touched infra helpers otherwise appears unchanged.

## Findings

### Medium: `infra-runtime` no longer exports approval account-binding helpers

- Kind: compatibility
- File: `src/infra/exec-approval-session-target.ts`
- Line: 11
- Evidence: The commit removes the re-export of `doesApprovalRequestMatchChannelAccount`, `resolveApprovalRequestAccountId`, and `resolveApprovalRequestChannelAccountId` from `exec-approval-session-target.ts`. `src/plugin-sdk/infra-runtime.ts:25` publicly re-exports `../infra/exec-approval-session-target.ts`, and docs describe `openclaw/plugin-sdk/infra-runtime` as deprecated but still present “for external compatibility.” A focused probe now shows those three names are absent from `infra-runtime`, while still present in `approval-runtime`. `pnpm plugin-sdk:api:check` also fails with Plugin SDK API baseline drift.
- Impact: External plugins that still use the documented compatibility shim, for example `import { resolveApprovalRequestChannelAccountId } from "openclaw/plugin-sdk/infra-runtime"`, will fail at import/typecheck after this change even though the focused replacement subpaths still export the helpers.
- Suggested fix: Preserve the compatibility shim by explicitly re-exporting these three helpers from `src/plugin-sdk/infra-runtime.ts`, or restore the removed re-export if keeping it on the infra module is acceptable. If this is an intentional breaking SDK removal, update the API baseline and migration docs, but `src/plugin-sdk/AGENTS.md` says breaking removals are major-version work.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. `pnpm exec tsx -e 'import("./src/plugin-sdk/infra-runtime.ts").then(...)'` reported all three removed helper names as `false`, while the same probe against `src/plugin-sdk/approval-runtime.ts` reported them as `true`. `pnpm plugin-sdk:api:check` failed with `Plugin SDK API baseline drift detected`.
- Is this the best way to solve the issue? yes, keeping `infra-runtime` as a compatibility shim is the narrowest fix. It preserves the public deprecated surface without re-expanding unrelated internal exports.
