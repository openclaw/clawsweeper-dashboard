# Commit 2601453

## Summary

The commit moves chat session search from the picker popover into inline controls beside the session selector. The final merged code still has a mobile layout regression: the new inline search submit and clear buttons are generic `.btn--icon` buttons, so the existing mobile dropdown rule shrinks them to effectively unusable widths.

## Findings

### Medium: Inline session search buttons collapse in the mobile controls dropdown

- Kind: regression
- File: `ui/src/ui/chat/session-controls.ts`
- Line: 421
- Evidence: The new submit and clear buttons use only `class="btn btn--ghost btn--icon"` at `ui/src/ui/chat/session-controls.ts:421` and `ui/src/ui/chat/session-controls.ts:432`. The old popover buttons used `chat-session-picker__icon-button`, which is still the only mobile fixed-size exception at `ui/src/styles/layout.mobile.css:549`. The broad mobile dropdown rule at `ui/src/styles/layout.mobile.css:608` applies `width: 100%; min-width: 0; height: 44px` to all `.btn--icon` controls. A focused Playwright layout probe of the committed CSS and matching DOM measured the new submit and clear buttons at `2px` wide at 390px, 360px, and 320px viewports.
- Impact: On mobile, users opening inline session search from the controls dropdown can get nearly untappable submit and clear buttons. Search still has some keyboard paths, but the visible action controls are effectively broken.
- Suggested fix: Give the inline search action buttons a dedicated class and add a mobile override equivalent to the old picker icon rule, e.g. fixed `flex: 0 0 44px`, `width: 44px`, `min-width: 44px`, `height: 44px`, `padding: 0`. Update the mobile style test to assert the new inline-search selector, not only the now-stale popover icon selector.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. The committed CSS/DOM path reproduces with a browser layout probe at mobile widths; the measured submit and clear buttons collapse to `2px` wide.
- Is this the best way to solve the issue? yes. The prior fixed-size class protected the old popover buttons; the inline buttons need the same mobile sizing contract.
