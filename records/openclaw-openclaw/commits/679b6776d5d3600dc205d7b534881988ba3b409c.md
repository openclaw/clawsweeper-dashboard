# Commit 679b677

## Summary

Found one regression in the TLS inheritance change: explicitly passing the same saved endpoint is treated as a retarget and drops the saved TLS pin.

## Findings

### Medium: Explicitly reusing the saved endpoint clears its TLS settings

- Kind: regression
- File: `src/cli/node-cli/register.ts`
- Line: 67
- Evidence: `retargetedGateway` is true whenever `opts.host !== undefined || opts.port !== undefined`, before comparing the resolved `host`/`port` to `existing.gateway`. With saved config `{host:"10.0.0.2", port:19001, tls:true, tlsFingerprint:"old-fingerprint"}`, `openclaw node run --host 10.0.0.2 --port 19001` now passes `gatewayTls: undefined` and `gatewayTlsFingerprint: undefined` to `runNodeHost`. `runNodeHost` then persists `tls: opts.gatewayTls ?? getRuntimeConfig().gateway?.tls?.enabled ?? false` and `tlsFingerprint: opts.gatewayTlsFingerprint`, so the saved pin can be erased even though the endpoint did not change.
- Impact: Scripts and documented-style invocations that always pass `--host`/`--port` can break or weaken a previously working pinned TLS node connection when they point at the exact saved gateway. For self-signed/private-CA gateways this can fail to reconnect; for trusted CA endpoints it silently drops the explicit pin.
- Suggested fix: Decide retargeting from the resolved endpoint, not option presence. Inherit saved TLS settings when `host` and `port` match the saved gateway endpoint; clear them only when the resolved endpoint actually differs, unless `--tls` or `--tls-fingerprint` is explicitly supplied.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes: configure a saved node gateway with TLS and a fingerprint, then run `openclaw node run --host <same-host> --port <same-port>` without TLS flags. The new branch at `src/cli/node-cli/register.ts:67` drops inherited TLS solely because options were present.
- Is this the best way to solve the issue? yes: comparing the resolved endpoint matches the commit’s stated contract, “only when using the saved gateway endpoint,” and avoids reusing stale pins for truly different targets.
