# Commit 201f690d54

## Summary

One test-only regression found: the new assertion helper no longer rejects unexpected Matrix mention metadata for scenarios whose contract depends on sending an unmentioned message.

## Findings

### Low: Helper allows unintended mention metadata in unmentioned Matrix scenarios

- Kind: test_gap
- File: `extensions/qa-matrix/src/runners/contract/scenarios.test.ts`
- Line: 191
- Evidence: `expectSentTextMessage` only checks `mentionUserIds` when the caller supplies an expected value. Several converted tests intentionally omit `mentionUserIds`, for example `matrix-allowbots-mentions-unmentioned-open-room-block` at `extensions/qa-matrix/src/runners/contract/scenarios.test.ts:1261` and DM/shared-DM sends at `extensions/qa-matrix/src/runners/contract/scenarios.test.ts:3774`. In the runtime, `withMention === false` is the branch that must omit `mentionUserIds`, while any supplied list becomes Matrix `m.mentions` metadata.
- Impact: A future regression that sends `mentionUserIds: ["@sut:matrix-qa.test"]` while keeping an unmentioned body would still pass these tests, even though `allowBots=mentions` and DM/no-mention scenarios rely on not tagging the SUT. The previous exact `toHaveBeenCalledWith({ body: ..., roomId: ... })` assertions rejected that extra field.
- Suggested fix: Make `expectSentTextMessage` assert absence when `expected.mentionUserIds` is omitted, or add an explicit `mentionUserIds: undefined` / `expectNoMentionUserIds` option and use it on the unmentioned scenarios. Consider restoring exact-object checks for `createPrivateRoom` where omitted fields such as `encrypted` are also part of the contract.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes: call `expectSentTextMessage` with a mock first call containing the expected `body` and `roomId` plus an unexpected `mentionUserIds`; the helper skips the assertion because `expected.mentionUserIds` is absent.
- Is this the best way to solve the issue? yes: the helper can keep the clearer body checks while preserving the old exactness for mention metadata.
