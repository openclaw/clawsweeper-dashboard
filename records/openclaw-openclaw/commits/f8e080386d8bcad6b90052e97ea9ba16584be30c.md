# Commit f8e0803

## Summary

Found one low-severity edge case in the benchmark trace suppression path. Normal default benchmark runs are covered and the focused test passes, but an empty inherited `OPENCLAW_TRACE_SYNC_IO` defeats the new suppression in tmux mode.

## Findings

### Low: Empty trace env still allows benchmark tmux runs to re-enable sync-I/O spam

- Kind: bug
- File: `scripts/gateway-watch-tmux.mjs`
- Line: 100
- Evidence: Benchmark mode defaults `nextEnv.OPENCLAW_TRACE_SYNC_IO ??= "0"`, which leaves an existing empty string unchanged. Later, `buildGatewayWatchTmuxCommand` drops empty child env values at `scripts/gateway-watch-tmux.mjs:158`, so the tmux pane starts `scripts/watch-node.mjs` without `OPENCLAW_TRACE_SYNC_IO`. `watch-node` treats an absent value as the gateway-watch default and sets `OPENCLAW_TRACE_SYNC_IO = "1"` at `scripts/watch-node.mjs:280`, which causes `run-node` to add `--trace-sync-io` at `scripts/run-node.mjs:492`.
- Impact: `OPENCLAW_TRACE_SYNC_IO= pnpm gateway:watch --benchmark` still produces the sync-I/O stack trace flood that this commit is intended to suppress, as long as the tmux wrapper path is used. This is narrow, but it violates the documented тАЬunless explicitly requestedтАЭ behavior because an empty value is not an explicit request for tracing.
- Suggested fix: Treat an empty inherited value as unset when applying the benchmark default, for example set `"0"` when `nextEnv.OPENCLAW_TRACE_SYNC_IO == null || nextEnv.OPENCLAW_TRACE_SYNC_IO === ""`.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. A focused command-construction probe with `env: { OPENCLAW_TRACE_SYNC_IO: "", SHELL: "/bin/zsh" }` and `args: ["gateway", "--force", "--benchmark"]` produced a tmux command containing `OPENCLAW_RUN_NODE_CPU_PROF_DIR` but no `OPENCLAW_TRACE_SYNC_IO=0`, which leaves `watch-node` to restore the default trace value.
- Is this the best way to solve the issue? yes. Normalizing only the benchmark default preserves explicit `"1"` and `"0"` values while making the documented default robust to empty inherited env.
