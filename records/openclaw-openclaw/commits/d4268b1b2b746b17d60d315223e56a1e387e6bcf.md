# Commit d4268b1

## Summary

Found one actionable regression: the new official external npm catalog can repair/install `diagnostics-otel` from the obsolete npm `latest` package even though that package no longer loads against current OpenClaw.

## Findings

### Medium: `diagnostics-otel` npm-first catalog installs a package that cannot load

- Kind: regression
- File: `scripts/lib/official-external-plugin-catalog.json`
- Line: 31
- Evidence: The catalog lists `diagnostics-otel` with `clawhubSpec`, unversioned `npmSpec: "@openclaw/diagnostics-otel"`, and `defaultChoice: "npm"` at `scripts/lib/official-external-plugin-catalog.json:31`. The plugin is not bundled in packaged OpenClaw (`package.json:39`). Doctor repair preserves the npm-first choice and skips ClawHub when `defaultChoice === "npm"` (`src/commands/doctor/shared/missing-configured-plugin-install.ts:230`), and update relocation bridges also prefer npm for this entry (`src/cli/plugins-location-bridges.ts:42`).
- Evidence: `npm view @openclaw/diagnostics-otel dist-tags` returns `latest: 2026.3.13` and `beta: 2026.5.2-beta.2`; `npm view @openclaw/diagnostics-otel version deprecated openclaw` reports `2026.3.13` as deprecated. A live `installPluginFromNpmSpec({ spec: "@openclaw/diagnostics-otel", expectedPluginId: "diagnostics-otel" })` returned `ok: true` and resolved `@openclaw/diagnostics-otel@2026.3.13`.
- Evidence: The installed/extracted `2026.3.13` package imports `openclaw/plugin-sdk/diagnostics-otel`, but current OpenClaw does not export `./plugin-sdk/diagnostics-otel`. Importing that package with current OpenClaw available fails with `ERR_PACKAGE_PATH_NOT_EXPORTED`.
- Impact: A user with `diagnostics-otel` configured, or a user updating from a previously bundled enabled `diagnostics-otel`, can be “repaired” to an installed package that fails at runtime. Because install succeeds and records the npm package, later repair/update flows may think the missing plugin was handled.
- Suggested fix: Do not point npm-first catalog entries at unversioned `@openclaw/*` packages while `latest` is obsolete. For this entry, prefer a current ClawHub artifact if available, pin/use an installable compatible npm tag or exact version, or keep the plugin bundled until a compatible stable npm artifact is published. Also consider rejecting deprecated/compat-missing OpenClaw-owned npm artifacts during official catalog repair.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. `npm view @openclaw/diagnostics-otel dist-tags --json` shows `latest` is `2026.3.13`; installing `@openclaw/diagnostics-otel` through `installPluginFromNpmSpec` resolves that version and returns success; importing its `index.ts` against current OpenClaw fails with `ERR_PACKAGE_PATH_NOT_EXPORTED`.
- Is this the best way to solve the issue? yes, directionally. The catalog should only advertise artifacts that are compatible with the current host, or it should prefer the compatible source instead of silently taking npm `latest`.
