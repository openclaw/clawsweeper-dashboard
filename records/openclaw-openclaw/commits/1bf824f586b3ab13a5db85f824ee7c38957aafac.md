The low-level command explainer now detects `exec bash -lc ...` and `exec eval ...`, but the gateway exec approval summary path still reports no risk warnings for those same commands.

## Findings

### Low: Exec shell-wrapper and eval risks are still omitted from approval summaries

- Kind: bug
- File: `src/infra/command-analysis/explain.ts`
- Line: 53
- Evidence: `summarizeCommandSegmentsForDisplay` only adds `inline-eval` and `command-carrier` warnings at `src/infra/command-analysis/explain.ts:58` and `src/infra/command-analysis/explain.ts:67`. Gateway approval requests use this summary through `resolveCommandAnalysisSummaryForDisplay` at `src/gateway/server-methods/exec-approval.ts:211`. A focused smoke check returned empty `riskKinds`/`warningLines` for both `exec bash -lc 'id && whoami'` and `exec eval 'echo hi'`, while `exec python -c 'print(1)'` did report `inline-eval`.
- Impact: Approval prompts can still miss the exact `exec`-carried shell-wrapper and eval/source risks this commit adds to the full shell explainer, so users reviewing an approval may not see the hidden carrier risk.
- Suggested fix: Extend `summarizeCommandSegmentsForDisplay` to include the new carrier shell-wrapper and carried eval/source detectors, or factor a shared risk-summary helper used by both the shell explainer and approval-summary path. Add `explain.test.ts` coverage for `resolveCommandAnalysisSummaryForDisplay({ host: "gateway", commandText: "exec bash -lc ..." })` and `exec eval`.
- Confidence: medium
