# Commit 0f1f9525f3

## Summary

Found one low-severity reliability risk in the new cleanup jobs: reaction deletion is not best-effort, so a stale or concurrently removed reaction can fail an otherwise successful Mantis workflow.

## Findings

### Low: reaction cleanup can fail if the shared bot reaction is already gone

- Kind: reliability
- File: `.github/workflows/mantis-discord-status-reactions.yml`
- Line: 606
- Evidence: The new cleanup jobs list `github-actions[bot]` `eyes` reactions and call `github.rest.reactions.deleteForIssueComment(...)` without handling delete failures. The same pattern is duplicated in `.github/workflows/mantis-discord-thread-attachment.yml:628`, `.github/workflows/mantis-telegram-desktop-proof.yml:660`, and `.github/workflows/mantis-telegram-live.yml:573`. The add path is explicitly best-effort via `.catch(...)`, but the delete path is not.
- Impact: A single PR comment can plausibly trigger more than one Mantis workflow, for example Discord text containing both `status reaction` and `thread attachment`. Since GitHub reactions are keyed to the comment and bot user, both cleanup jobs operate on the same `github-actions[bot]` `eyes` reaction. If one cleanup deletes it first, another cleanup that already listed the reaction can fail on delete and mark the workflow failed after the actual scenario succeeded. Manual removal during a long run creates the same failure mode.
- Suggested fix: Treat cleanup deletion as best-effort: wrap each delete in `try/catch`, ignore not-found/already-removed responses, and warn rather than failing for cleanup-only errors.
- Confidence: medium

## Details

- Do we have a high-confidence way to reproduce the issue? No. Reproducing it cleanly requires a live issue-comment workflow run or mutating a comment reaction. The code path and race are concrete, but I did not perform live reaction mutation.
- Is this the best way to solve the issue? Yes. Reaction cleanup is auxiliary status UI, so deletion failures should not override the result of the underlying Mantis proof run.
