Found one concrete heartbeat prompt/tool contract regression in the due-commitment path.

## Findings

### Medium: Due commitment heartbeats can ask for a disabled `heartbeat_respond` tool

- Kind: regression
- File: `src/infra/heartbeat-runner.ts`
- Line: 786
- Evidence: In response-tool mode, `buildCommitmentHeartbeatPrompt()` tells the model: “If none should be sent, use heartbeat_respond with notify=false” at `src/infra/heartbeat-runner.ts:785-787`. The same run sets `{ disableTools: true, skillFilter: [] }` whenever `hasDueCommitments` is true at `src/infra/heartbeat-runner.ts:1576-1580`. Existing commitment tests assert this tool-disable behavior, but they do not cover the new response-tool prompt branch.
- Impact: A message-tool or Codex heartbeat with a due inferred commitment and no due heartbeat tasks gets an instruction to call a tool that the runner deliberately withholds. That can make the model emit an invalid/tool-shaped text response, fail to produce the intended quiet ack, or create noisy user-visible commitment check-ins instead of a silent dismissal.
- Suggested fix: Keep the no-tool commitment branch on the text sentinel contract, or explicitly allow only `heartbeat_respond` while preserving the no-other-tools safety boundary. Add a focused test with `messages.visibleReplies: "message_tool"` plus a due commitment and no due tasks, asserting the prompt and tool policy agree.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes: configure `messages.visibleReplies: "message_tool"` or run a Codex response-tool heartbeat, seed a due commitment with heartbeat target `last`, ensure no due `HEARTBEAT.md` tasks, then call `runHeartbeatOnce`. The constructed prompt contains `heartbeat_respond`, while `getReplyFromConfig` receives `disableTools: true`.
- Is this the best way to solve the issue? yes for the direction: the prompt contract must match tool availability. The safer minimal fix is to keep commitment-only runs on `HEARTBEAT_OK` unless the runner can allow only the heartbeat response tool without exposing other tools to untrusted commitment metadata.
