# Commit 12debcb

## Summary

Found two actionable regressions: the guided `config --section ...` bypass does not work for the common space-separated option form, and the newly added run-node progress test is broken before it reaches the behavior under test.

## Findings

### Medium: `openclaw config --section models` still runs the startup config guard

- Kind: regression
- File: `src/cli/program/preaction.ts`
- Line: 79
- Evidence: `isGuidedConfigAction()` checks `actionCommand.parent?.parent === undefined`, but Commander roots have `parent === null`; a direct Commander probe returned `config true null null false true` for `c.parent?.parent === undefined` vs `== null`. The focused test added by this commit also fails: `config --section models` calls `ensureConfigReady` with `commandPath: ["config", "models"]`.
- Impact: the changelog-promised guided config flow only bypasses startup validation for bare `openclaw config`. `openclaw config --section models` can still be blocked by invalid config before the guided config action can repair or own validation, and it keeps the redundant startup config read.
- Suggested fix: make the top-level command check nullish, for example `actionCommand.parent?.parent == null`, or classify guided config from the Commander action/root relationship instead of argv-derived command paths that treat option values as positionals.
- Confidence: high

### Low: New run-node progress test fails because the fixture touches a missing source file

- Kind: reliability
- File: `src/infra/run-node.test.ts`
- Line: 1403
- Evidence: the new test passes `oldPaths: [ROOT_SRC, ROOT_TSCONFIG, ROOT_PACKAGE]`, but `setupTrackedProject()` only writes `BASE_PROJECT_FILES` plus `options.files`; `BASE_PROJECT_FILES` does not include `src/index.ts`. The focused test fails with `ENOENT: no such file or directory, utime '/tmp/.../src/index.ts'`.
- Impact: the focused `src/infra/run-node.test.ts` suite fails on current main, and the progress behavior is not actually verified.
- Suggested fix: add `[ROOT_SRC]` to the test fixture’s `files`, or remove `ROOT_SRC` from `oldPaths` since the test already forces a build with `OPENCLAW_FORCE_BUILD=1`.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Run `node scripts/run-vitest.mjs src/cli/program/preaction.test.ts -t "lets guided config sections" --reporter=verbose` and `node scripts/run-vitest.mjs src/infra/run-node.test.ts -t "shows tty progress" --reporter=verbose`.
- Is this the best way to solve the issue? yes for the first issue: Commander uses `null` for the root parent, so the current strict `undefined` check is the direct cause. For the second, the fixture should create or stop touching `ROOT_SRC`.
