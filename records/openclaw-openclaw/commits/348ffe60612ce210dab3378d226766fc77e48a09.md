# Commit 348ffe6061

## Summary

Found one edge-case reliability regression in the same auth retry state machine this commit changes. The normal no-token and successful one-shot retry paths are covered and pass.

## Findings

### Low: Queued device-token retry flag can survive after the cached token disappears

- Kind: reliability
- File: `ui/src/ui/gateway.ts:502`; `src/gateway/client.ts:728`
- Evidence: Both clients queue `pendingDeviceTokenRetry = true` after a retryable `AUTH_TOKEN_MISMATCH`, but clear it only when the next connect plan actually includes `authDeviceToken` (`ui/src/ui/gateway.ts:739`, `src/gateway/client.ts:515`). If local storage or the node device-auth file loses the cached token between queuing and the retry connect, the retry sends the stale shared token again while `pendingDeviceTokenRetry` remains true. The next `AUTH_TOKEN_MISMATCH` then schedules another reconnect in the browser client, and the node client returns “do not pause” from `shouldPauseReconnectAfterAuthFailure`, so the stale retry loop can continue even though no device-token fallback is available.
- Impact: This can recreate the retry loop this commit is meant to stop when the cached fallback is removed, revoked locally, corrupted, or otherwise unavailable during the backoff window.
- Suggested fix: Treat the retry opportunity as consumed whenever a pending retry connect is built. Clear `pendingDeviceTokenRetry` even if no `authDeviceToken` is found, so the next mismatch pauses instead of looping.
- Confidence: medium

## Details

- Do we have a high-confidence way to reproduce the issue? yes, source-level: store a cached device token, trigger an `AUTH_TOKEN_MISMATCH` that queues fallback retry, clear the cached token before the 800ms retry connect, then return `AUTH_TOKEN_MISMATCH` again. Current code will still consider `pendingDeviceTokenRetry` true.
- Is this the best way to solve the issue? yes. Clearing the pending flag when the retry attempt is built preserves the successful one-shot fallback path and makes “fallback no longer available” terminal.
