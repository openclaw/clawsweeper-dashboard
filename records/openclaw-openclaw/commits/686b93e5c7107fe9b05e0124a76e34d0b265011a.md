# Commit 686b93e

## Summary

Found one concrete regression risk in the new command-style cron classifier: natural-language cron jobs that start with lowercase words like `make` or `go` are now treated as shell commands and lose bootstrap context by default.

## Findings

### Medium: Natural-language cron payloads can be misclassified as exact commands

- Kind: regression
- File: `src/cron/isolated-agent/run-executor.ts`
- Line: 64
- Evidence: `COMMAND_STYLE_CRON_PREFIX` includes bare `go` and `make` as command prefixes. `resolveCronBootstrapContextMode()` then returns `"lightweight"` for any matching single-line payload unless `lightContext: false` is explicitly set. Lightweight cron/default context intentionally drops all bootstrap files in `src/agents/bootstrap-files.ts:187`.
- Impact: Existing scheduled jobs such as `make a nightly status summary` or `go over today’s queue` are plausible natural-language agent tasks, but they now run without workspace identity/bootstrap/memory context. That can change behavior for summary/report cron jobs that depend on agent instructions or memory, even though the user did not opt into lightweight context.
- Suggested fix: tighten the heuristic for ambiguous command names. For `make`/`go`, require a command-shaped continuation such as a target/subcommand/operator/path, or exclude English-overlapping verbs from automatic inference and rely on explicit `lightContext` for those cases. Add classifier coverage for natural-language examples like `make a nightly status summary` and `go over today’s queue`.
- Confidence: medium

## Details

- Do we have a high-confidence way to reproduce the issue? yes. A focused regex check with the committed pattern returns `true` for `"make a nightly status summary"` and `"go over today’s queue"`, and the code path then maps that to `"lightweight"` bootstrap context.
- Is this the best way to solve the issue? yes, broadly. The least risky fix is to narrow only the ambiguous prefixes while keeping obvious exact commands such as `cd /srv/openclaw && ./scripts/nightly-report.sh` lightweight.
