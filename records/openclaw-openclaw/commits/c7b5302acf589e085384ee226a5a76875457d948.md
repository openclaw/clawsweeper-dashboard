## Summary

Found one actionable regression: the new Prometheus diagnostics ClawHub-first install metadata and docs point users at an unversioned ClawHub package that currently has no installable latest version.

## Findings

### Medium: Prometheus diagnostics quick-start now points to a ClawHub spec that cannot install

- Kind: regression
- File: `extensions/diagnostics-prometheus/package.json`
- Line: 18
- Evidence: The commit adds `clawhubSpec: "clawhub:@openclaw/diagnostics-prometheus"` and `defaultChoice: "clawhub"`. The docs now tell users to run `openclaw plugins install clawhub:@openclaw/diagnostics-prometheus` in `docs/gateway/prometheus.md:30` and `docs/install/docker.md:174`. A live ClawHub package lookup returned package `@openclaw/diagnostics-prometheus` with `latestVersion: null`, and `installPluginFromClawHub({ spec: "clawhub:@openclaw/diagnostics-prometheus", dryRun: true })` returned `{ ok: false, code: "no_installable_version" }`.
- Impact: Users following the updated Prometheus docs cannot install the plugin from the documented ClawHub command. Any install-on-demand flow that honors this new ClawHub-first metadata will prefer a source that fails before npm is tried unless the caller explicitly chooses npm or the registry gets a stable/latest version.
- Suggested fix: Publish/mark an installable ClawHub latest version for `@openclaw/diagnostics-prometheus` before advertising the unversioned `clawhub:` spec, or keep the docs/default choice on npm until that registry state exists. If beta is intentionally required, document a versioned/tagged ClawHub spec instead of the unversioned one.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes: run the focused installer probe against `clawhub:@openclaw/diagnostics-prometheus`; it returns `no_installable_version`.
- Is this the best way to solve the issue? yes: the safest fix is aligning repo metadata/docs with actual registry availability, because the failure is caused before archive install or plugin runtime code runs.
