# Commit 3918d69587

## Summary

The commit hardens transcript JSONL reading by filtering malformed session entries before `TranscriptFileState` builds branches and compaction context. I found one remaining correctness bug in the new rejected-id bookkeeping: malformed duplicate ids can poison already accepted valid entries and detach later valid children from their real parent.

## Findings

### Medium: Malformed duplicate ids detach later valid children from accepted parents

- Kind: bug
- File: `src/agents/pi-embedded-runner/transcript-file-state.ts`
- Line: 379
- Evidence: `readableSessionEntries()` records every rejected row id in `rejectedIds` / `rejectedParentById` before checking whether that id already belongs to an accepted entry. A focused inline repro with a valid `user-1`, then a malformed duplicate `user-1`, then a valid `assistant-1` whose `parentId` is `user-1` produced entries where `assistant-1.parentId` was rewritten to `null`; `getBranch()` returned only `assistant-1`, dropping the valid user prompt.
- Impact: A corrupted or imported transcript containing a malformed duplicate id can still lose valid branch context after this fix. Compaction, rewrite, truncation, and provider context can then run from an incomplete branch even though the valid parent row was already accepted.
- Suggested fix: Make rejected-id bookkeeping occurrence-safe. Do not mark an id rejected when it is already accepted, and clear or ignore prior rejected bookkeeping when a valid row becomes the canonical accepted entry for that id. Add a regression case covering valid row + malformed duplicate id + valid child.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. The inline `node --import tsx --input-type=module` repro wrote a temporary JSONL transcript and confirmed `assistant-1` was detached from valid `user-1`.
- Is this the best way to solve the issue? yes. The bug is local to the new rejected-id maps; fixing that bookkeeping preserves the commit’s malformed-row filtering without changing the broader transcript state contract.
