# Commit ba2b033

## Summary

Found one concrete parser gap: the new Harmony parser handles the simplified LM Studio `commentary to=read code {...}` shape, but still rejects the documented full Harmony function-call header with `functions.<name>` and `<|constrain|>json`.

## Findings

### Medium: Full Harmony tool-call headers still parse as text

- Kind: bug
- File: `src/plugin-sdk/tool-payload.ts`
- Line: 149
- Evidence: `parseHarmonyOpening` reads the recipient with `isToolNameChar`, which excludes `.`, then requires the next token to start with literal `code`. The documented Harmony tool-call form uses recipients like `to=functions.get_current_weather` and may include `<|constrain|>json` before `<|message|>`. ([cookbook.openai.com](https://cookbook.openai.com/articles/openai-harmony/?utm_source=openai)) A focused parser probe on current `main` returned structured blocks for the commit’s two examples, but returned `null` for `<|channel|>commentary to=functions.read <|constrain|>json<|message|>{"path":"/tmp/file.txt"}<|call|>`. The LM Studio promotion path returns without promotion when this parser returns `null` at `extensions/lmstudio/src/stream.ts:194` and `extensions/lmstudio/src/stream.ts:226`.
- Impact: If LM Studio leaks the full documented Harmony call instead of the simplified `code` variant, the same failure class remains: no structured `toolUse`, no tool invocation, and the tool-call text can remain an assistant text reply.
- Suggested fix: Accept an optional `functions.` namespace, map it back to the registered OpenClaw tool name before the allowed-name check, and accept optional `<|constrain|>json` before `<|message|>`, with parser and LM Studio stream tests for that exact shape.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes: the parser probe above deterministically returns `null` for the documented full Harmony call shape.
- Is this the best way to solve the issue? yes: keep the shared parser approach, but make the Harmony header parser cover the documented delimiter and namespace form instead of only the observed simplified LM Studio form.
