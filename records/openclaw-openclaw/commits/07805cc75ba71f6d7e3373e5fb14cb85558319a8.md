## Findings

### Low: Filename helper accepts extra date-prefix segments

- Kind: test_gap
- File: `src/hooks/bundled/session-memory/handler.test.ts`
- Line: 153
- Evidence: `expectDatedMemoryFile()` checks `filename.endsWith("-${slug}.md")`, then destructures only the first three parts from `datePrefix.split("-")`. Extra hyphenated segments are ignored, so a filename like `2026-05-11-2330-simple-math.md` would pass for slug `simple-math`, even though the replaced regex required exactly `YYYY-MM-DD-simple-math.md`.
- Impact: The two updated LLM-slug tests are meant to pin the session-memory filename contract. This weaker helper would miss a regression that accidentally inserts a time or other extra segment before the model slug.
- Suggested fix: Assert there are exactly three date parts before the slug, or use an anchored pattern equivalent to `^[0-9]{4}-[0-9]{2}-[0-9]{2}-${escapedSlug}\\.md$`.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes; by inspection of the helper, `split("-")` can return more than three parts and the extras are ignored.
- Is this the best way to solve the issue? yes; the helper should preserve the exact filename shape previously enforced by the anchored regex while keeping the ASCII digit checks.
