# Commit 5245289

## Summary

Found one high-confidence regression: session repair now permanently removes ordinary completed assistant turns before the next prompt is appended.

## Findings

### High: Session repair deletes normal completed assistant replies

- Kind: data_loss
- File: `src/agents/session-file-repair.ts`
- Line: 274
- Evidence: `repairSessionFileIfNeeded` unconditionally pops every trailing non-tool-call assistant entry. The helper is called before `SessionManager.open()` and before `activeSession.prompt(...)` appends the next user prompt in `src/agents/pi-embedded-runner/run/attempt.ts:1312`, and also before compaction loads the session in `src/agents/pi-embedded-runner/compact.ts:805`. A normal completed transcript at rest is commonly `session -> user -> assistant`; the new repair treats that ordinary tail as corrupt and rewrites the file to `session -> user`.
- Impact: On the next run or compaction, the last successful assistant answer is removed from the durable JSONL transcript and from future model history. This is broad data loss, not limited to Anthropic thinking-prefill failures, because the file repair has no provider/model/trigger guard.
- Suggested fix: Do not persistently trim normal assistant tails in the generic session-file repair pass. Keep trailing-assistant stripping in the outbound provider-specific replay path, or only trim a persisted assistant when there is concrete corruption/interruption evidence such as an error/aborted stop reason or an incomplete active-turn marker.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. I created a temp session JSONL with `session -> user -> assistant(stop)` and called `repairSessionFileIfNeeded`; it returned `trimmedTrailingAssistantMessages: 1` and rewrote the file from 3 lines to 2, leaving only `session,user`.
- Is this the best way to solve the issue? yes. The safe boundary is provider-request sanitization or an interruption-specific on-disk repair predicate; generic durable repair should not delete valid completed assistant replies.
