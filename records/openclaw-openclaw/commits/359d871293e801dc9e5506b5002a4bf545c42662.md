# Commit 359d871

## Summary

Found one concrete security regression in the Parallels npm update smoke helper: macOS/Linux guest scripts are now left world-readable even though their generated contents include live provider API keys.

## Findings

### Medium: Guest update scripts expose provider API keys to all local users

- Kind: security
- File: `scripts/e2e/parallels/npm-update-smoke.ts`
- Line: 739
- Evidence: `writeGuestScript()` writes the macOS/Linux update payload to `/tmp/openclaw-parallels-npm-update-*.sh` and this commit changes the final mode from `700` to `755`. The generated macOS and Linux payloads embed `input.auth.apiKeyValue` directly in the script text at `scripts/e2e/parallels/npm-update-scripts.ts:49`, `scripts/e2e/parallels/npm-update-scripts.ts:74`, `scripts/e2e/parallels/npm-update-scripts.ts:157`, and `scripts/e2e/parallels/npm-update-scripts.ts:183`. A focused static check with a fake key confirmed both generated POSIX scripts contain the secret.
- Impact: during a live Parallels update smoke, any local user in the macOS or Linux guest can read the temporary `/tmp` script and recover the host-provided model provider API key. If cleanup fails, the readable secret can persist beyond the run.
- Suggested fix: keep Linux at `700`. For macOS, make the script readable only by the selected execution user: write it as `--current-user` when that path is available, or `chown` it to the resolved fallback desktop user and use `700` before running `sudo -H -u <user> /bin/bash <scriptPath>`. Since `/bin/bash scriptPath` only needs read permission, avoid granting world read/execute.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. The static path is enough: generate a macOS/Linux update script with a fake API key, observe the secret in the script body, then observe the committed `chmod 755` on the `/tmp` guest script path.
- Is this the best way to solve the issue? no. The intended macOS fallback-user fix can be preserved without world-readable secrets by transferring ownership/readability to only the user that executes the script.
