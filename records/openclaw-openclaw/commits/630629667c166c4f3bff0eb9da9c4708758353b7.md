# Commit 6306296

## Summary

Found one low-severity formatting bug in the new loose-list paragraph detection. The fix works for root-level and nested lists, but still collapses loose list paragraphs when the list is inside a blockquote because the token-level check does not account for enclosing container depth.

## Findings

### Low: Loose list paragraphs still collapse inside blockquotes

- Kind: bug
- File: `src/markdown/ir.ts`
- Line: 272
- Evidence: `appendParagraphSeparator` treats a direct list paragraph as `state.env.listStack.length * 2`. For a loose bullet list inside a blockquote, markdown-it emits direct list `paragraph_close` tokens at level `3`, while `listStack.length` is `1`, so the check expects level `2` and suppresses the paragraph separator. Local smoke:
  `markdownToIR("> - first paragraph\n>\n>   second paragraph\n> - next").text`
  returned `"• first paragraphsecond paragraph\n• next"`.
- Impact: quoted loose lists still produce concatenated text in shared outbound formatting, leaving the same unreadable failure mode this commit fixes for top-level lists.
- Suggested fix: store the opening list token level in `ListState` and compare direct paragraph closes against `currentList.openLevel + 2`, or otherwise track the current list item’s paragraph level instead of deriving it only from list depth.
- Confidence: high
