## Summary

Found one low-severity test reliability issue. The commit changed a presence assertion into an exact ordered array assertion, but the helper under test does not expose a sorted-order contract.

## Findings

### Low: Exact persisted-auth channel test depends on unsorted discovery order

- Kind: reliability
- File: `src/channels/plugins/persisted-auth-state.test.ts`
- Line: 9
- Evidence: The test now asserts `listBundledChannelIdsWithPersistedAuthState()` equals `["matrix", "whatsapp"]`. That helper delegates through `src/channels/plugins/package-state-probes.ts:96`, which filters `listChannelCatalogEntries({ origin: "bundled" })` without sorting. The catalog path reaches `src/plugins/discovery.ts:725`, where directory entries are read with `fs.readdirSync(..., { withFileTypes: true })` and iterated in returned order. Node's filesystem docs describe directory iteration order as "no particular order" from the underlying OS mechanisms, and `fs.readdirSync()` with `withFileTypes` returns `fs.Dirent` entries from that same directory-entry surface. ([nodejs.org](https://nodejs.org/download/release/v22.10.0/docs/api/fs.html?utm_source=openai))
- Impact: A checkout or packaged install whose bundled plugin directory returns `whatsapp` before `matrix` can fail this test even though both shipped persisted-auth channels are present and runtime behavior is correct.
- Suggested fix: Keep the enumeration check but remove order sensitivity, for example `expect(listBundledChannelIdsWithPersistedAuthState().toSorted()).toEqual(["matrix", "whatsapp"])`, or sort in `listBundledChannelIdsForPackageState()` if callers should receive a deterministic API.
- Confidence: medium

## Details

- Do we have a high-confidence way to reproduce the issue? No. The focused test passes in this checkout because the current directory order is `matrix,whatsapp`; the issue is the missing order contract in the call path.
- Is this the best way to solve the issue? Yes. Sorting in the assertion is the smallest fix if the test only wants to enumerate membership; sorting the helper is better if production callers should get deterministic channel-id ordering.
