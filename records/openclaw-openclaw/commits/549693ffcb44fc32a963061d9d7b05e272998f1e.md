# Commit 549693f

## Summary

Found two actionable regressions in the installer change: the PATH promotion rewrite can replace symlinked shell rc files, and the commit makes the installer ShellCheck workflow fail on `main`.

## Findings

### Medium: PATH promotion replaces symlinked shell profiles with regular files

- Kind: regression
- File: `scripts/install.sh`
- Line: 1386
- Evidence: `persist_shell_path_prepend` writes `${rc}.openclaw-tmp` and then `mv "$tmp_rc" "$rc"` whenever the first line is not the promoted PATH export. For a dotfiles-managed `~/.bashrc -> dotfiles/bashrc` containing the same PATH export after an interactive-shell guard, current `main` changes `~/.bashrc` from a symlink to a regular file. A focused repro printed `is_link=no`; running the parent helper on the same fixture printed `is_link=yes`.
- Impact: Users who manage `.bashrc`/`.zshrc` as symlinks lose the link to their dotfiles source during install/upgrade. The copied profile may work immediately, but future dotfile updates no longer apply, and the managed source remains unpromoted.
- Suggested fix: When `rc` is a symlink, rewrite the resolved target or update through the link without replacing the symlink. Add a regression test with symlinked `.bashrc` and an existing PATH line after a noninteractive guard.
- Confidence: high

### Medium: Installer ShellCheck workflow now fails on SC2016

- Kind: reliability
- File: `scripts/install.sh`
- Line: 1765
- Evidence: The commit removed the local `# shellcheck disable=SC2016` block and now passes `'$HOME/.npm-global/bin'` directly to `persist_shell_path_prepend`. The repository workflow `.github/workflows/website-installer-sync.yml` runs `shellcheck -e SC1091 scripts/install.sh scripts/install-cli.sh`; that command now fails with SC2016 at this line. The exact `Website Installer Sync` run for this commit also completed with `static` failed and overall conclusion `failure`.
- Impact: Any push/PR touching `scripts/install.sh` trips the installer sync static gate, and the website sync job is skipped.
- Suggested fix: Restore a targeted ShellCheck suppression for the intentional literal `$HOME` argument, or assign the literal to a clearly named variable with a local `SC2016` suppression.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Symlink repro: create a symlinked `.bashrc` with the target PATH export after a noninteractive guard, source `scripts/install.sh` with `OPENCLAW_INSTALL_SH_NO_RUN=1`, call `persist_shell_path_prepend`, then check `test -L "$HOME/.bashrc"`. ShellCheck repro: `shellcheck -e SC1091 scripts/install.sh scripts/install-cli.sh`.
- Is this the best way to solve the issue? yes for direction. The intended PATH promotion can stay, but the rc rewrite should preserve symlinks, and the intentional literal `$HOME` should be documented for ShellCheck.
