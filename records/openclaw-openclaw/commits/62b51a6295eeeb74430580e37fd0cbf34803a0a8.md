# Commit 62b51a6

## Summary

Found one high-confidence user-visible message loss regression. The merge includes the intended centralized reply admission work, but visible dispatch still inherits the reply-run registry’s 15 second default wait and then commits the inbound message as skipped.

## Findings

### High: Visible reply admission still times out and commits the user message as skipped

- Kind: regression
- File: `src/auto-reply/reply/reply-turn-admission.ts`
- Line: 57
- Evidence: `admitReplyTurn` calls `replyRunRegistry.waitForIdle(params.sessionKey, params.waitTimeoutMs, ...)`; when `waitTimeoutMs` is omitted, `reply-run-registry.ts:442-475` defaults to `15_000`. `dispatch-from-config.ts:978-985` calls this without a timeout for visible dispatch, and if admission returns `skipped`, `dispatch-from-config.ts:1419-1427` records `reply-operation-active`, marks idle, and commits the inbound dedupe claim.
- Impact: A visible Telegram topic turn that arrives behind a heartbeat/system reply operation lasting over 15 seconds can still be treated as processed with no final response. That is the same message-loss class this commit is meant to fix.
- Suggested fix: For visible turns, wait abortably until the active operation ends, or preserve/retry the inbound instead of committing dedupe on an active-run timeout. Keep bounded/deferred behavior for heartbeat and queued follow-up lanes if needed.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Hold an active `ReplyOperation` for the same Telegram topic session key longer than 15 seconds, then dispatch a visible turn for that session. The second dispatch waits through `admitReplyTurn`, receives `active-run`, and `finishReplyOperationBusyDispatch` commits the dedupe claim with `queuedFinal: false`.
- Is this the best way to solve the issue? yes. The core admission design is reasonable, but visible user turns need either an unbounded abortable wait or a retry-preserving path; committing dedupe after a registry timeout is the unsafe part.
