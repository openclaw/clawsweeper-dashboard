# Commit cb9d788

## Summary

Found one source-compatibility regression in the generated Swift protocol models. The wire change is additive, but the public Swift initializer now requires a new `sessionid` argument, so existing Swift callers that construct `ChatSendParams` no longer compile.

## Findings

### Medium: Optional `sessionId` breaks existing Swift `ChatSendParams` callers

- Kind: compatibility
- File: `apps/shared/OpenClawKit/Sources/OpenClawProtocol/GatewayModels.swift`
- Line: 4973
- Evidence: `ChatSendParams` gained `public let sessionid: String?` and the public initializer now requires `sessionid: String?` between `sessionkey` and `message`. The mirrored macOS model has the same required initializer at `apps/macos/Sources/OpenClawProtocol/GatewayModels.swift:4973`. A focused Swift type-check of an existing-style call without `sessionid` fails with `missing argument for parameter 'sessionid' in call`.
- Impact: Swift clients pinned to this generated protocol model must update every `ChatSendParams(...)` call even though the Gateway field is optional on the wire. That turns an additive protocol change into a source-breaking client API change.
- Suggested fix: Give the new initializer parameter a default, e.g. `sessionid: String? = nil`, in both generated Swift model copies, or adjust the generator so newly added optional schema fields preserve source compatibility.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Type-check a Swift caller that uses the previous `ChatSendParams(sessionkey:message:thinking:deliver:originatingchannel:originatingto:originatingaccountid:originatingthreadid:attachments:timeoutms:systeminputprovenance:systemprovenancereceipt:idempotencykey:)` initializer shape; Swift reports the missing `sessionid` argument.
- Is this the best way to solve the issue? yes. The Gateway schema already makes `sessionId` optional, so defaulting the Swift initializer parameter to `nil` preserves the additive contract without changing runtime behavior.
