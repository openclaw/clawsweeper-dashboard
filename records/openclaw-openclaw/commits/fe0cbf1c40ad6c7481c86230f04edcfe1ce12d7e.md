# Commit fe0cbf1c

## Summary

Found one medium-confidence regression in the Docker runtime workspace file that this commit restores into the final image.

## Findings

### Medium: Runtime image now exposes every bundled plugin as a pnpm workspace

- Kind: regression
- File: `Dockerfile`
- Line: 128
- Evidence: The runtime-assets stage saves the full source `pnpm-workspace.yaml`, temporarily replaces it for `pnpm prune --prod`, then restores the full source file before the final image copies `/app/pnpm-workspace.yaml`. That source workspace contains `extensions/*`, and the final image also copies `/app/extensions`, so any downstream `pnpm install` from `/app` sees every bundled plugin package, not just `.`/`ui` plus `OPENCLAW_EXTENSIONS`. A local pnpm 11.0.8 temp-workspace probe confirmed that a root install with `packages: extensions/*` includes dependencies from all matching extension package manifests.
- Impact: The same downstream `pnpm install` layers this change is meant to repair can now install or resolve dependencies for unselected bundled plugins. That undermines the documented `OPENCLAW_EXTENSIONS` opt-in model, can bloat derived images, and can trigger native/package-manager failures from plugins the image builder did not opt into.
- Suggested fix: Preserve the source workspace metadata keys needed for pnpm policy, patches, overrides, and build policy, but keep the runtime `packages` list restricted to the same generated subset used for prune: `.`, `ui`, and the selected `OPENCLAW_EXTENSIONS`.
- Confidence: medium

## Details

- Do we have a high-confidence way to reproduce the issue? yes: build the Dockerfile with no or one selected `OPENCLAW_EXTENSIONS`, then build a child image with `RUN pnpm install --prod` from `/app` and inspect the resulting workspace resolution or installed deps for unselected `extensions/*` packages. I did not run the full Docker build locally because it is a broad Docker proof.
- Is this the best way to solve the issue? yes: the Dockerfile already generates the correct restricted package set for prune, so the safer fix is to merge non-package metadata from the source workspace into that restricted runtime workspace instead of restoring the full source workspace wholesale.
