# Commit ad264a9

## Summary

Found one release-packaging regression that remains actionable on current `main`: Discord is marked as core-included for inventory generation, but current package packing excludes the Discord dist tree, so generated inventories can reference tarball entries that npm omits.

## Findings

### Medium: Discord inventory entries can reference files omitted from the npm package

- Kind: compatibility
- File: `src/infra/package-dist-inventory.ts`
- Line: 111
- Evidence: The reviewed change makes `includeInCore: true` return `false` from `isPublishableExternalizedBundledManifest`, so `dist/extensions/discord/**` is retained in `dist/postinstall-inventory.json`. Current `extensions/discord/package.json:71` sets `"includeInCore": true`, while current `package.json:43` excludes `"!dist/extensions/discord/**"` from the packed package. A temp-package reproduction using the current generator produced `["dist/extensions/discord/index.js"]`, then `scripts/check-openclaw-package-tarball.mjs` failed with `inventory references missing tar entry dist/extensions/discord/index.js`.
- Impact: A release/package artifact can fail package integrity checks, or ship an inventory whose postinstall expectations do not match packed contents.
- Suggested fix: Align the inventory source of truth with the npm `files` exclusions. If Discord should be externalized, remove `includeInCore: true` or make the inventory collector omit root-package-excluded extension dirs. If Discord should ship in core, remove the package exclusion and ensure Discord runtime dependencies are root-owned.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes: generate inventory from a package root containing `extensions/discord/package.json` and a built `dist/extensions/discord/index.js`, then pack/check a tarball that omits `dist/extensions/discord/**`; the tarball checker fails on the missing inventory entry.
- Is this the best way to solve the issue? unclear: the code has conflicting signals about whether Discord should be core-included or externalized, so the fix should first choose that product contract, then make package `files`, inventory generation, and dependency ownership agree.
