# Commit e4b09e1

## Summary

Found one concrete MCP serialization edge case in the new raw-result handling.

## Findings

### Medium: Raw array tool results become invalid MCP content blocks

- Kind: bug
- File: `src/mcp/plugin-tools-handlers.ts`
- Line: 60
- Evidence: The commit now treats a tool result without a `content` envelope as `rawContent = result`, but then passes any array directly through as `content`. For a raw plugin result like `[{ title: "one" }]`, the handler returns `{"content":[{"title":"one"}]}`. The installed MCP SDK validates `tools/call` handler results against `CallToolResultSchema`, whose `content` must be MCP `ContentBlock[]`; the schema probe rejected that response with `Invalid input`.
- Impact: Raw object results are now serialized correctly, but raw array results, a common shape for list/search tools, can make the MCP `tools/call` request fail as an invalid protocol result instead of returning usable JSON text.
- Suggested fix: Track whether the original result was an actual tool-result envelope. Only pass arrays through when they came from `result.content`; otherwise serialize the whole raw value as text. Add a test for a raw array result.
- Confidence: high
