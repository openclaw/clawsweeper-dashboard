# Commit b418c08a22

## Summary

Found one regression in model-list availability heuristics for providers whose env auth is resolved by special resolver logic rather than `providerAuthEnvVars`.

## Findings

### Medium: Google Vertex env credentials are no longer reflected in `models list` availability

- Kind: regression
- File: `src/commands/models/list.auth-index.ts`
- Line: 57
- Evidence: This commit routes list rows through the precomputed auth index at `src/commands/models/list.list-command.ts:79` and `src/commands/models/list.rows.ts:84`. The index only preloads env-backed providers by iterating `Object.keys(envCandidateMap)` at `src/commands/models/list.auth-index.ts:57`, but `google-vertex` is intentionally resolved by `resolveEnvApiKey()` special handling when no candidate-map entry exists (`src/agents/model-auth-env.ts:76`). The Google plugin manifest only declares `providerAuthEnvVars` for `google`, not `google-vertex` (`extensions/google/openclaw.plugin.json:75`), and the existing auth tests assert `GOOGLE_CLOUD_API_KEY` and ADC credentials are valid for `google-vertex` (`src/agents/model-auth.profiles.test.ts:809`).
- Impact: `models list --all --provider google-vertex` now shows Vertex rows as unavailable even when `GOOGLE_CLOUD_API_KEY` or ADC env credentials are present. I reproduced current behavior with an isolated config and dummy `GOOGLE_CLOUD_API_KEY`: the command returned 13 `google-vertex/*` rows, all with `"available": false`. Before this refactor, the per-row fallback called `resolveEnvApiKey(model.provider)` directly, so those rows would have been marked available.
- Suggested fix: Make `ModelListAuthIndex.hasProviderAuth()` do a cached on-demand `resolveEnvApiKey(provider, env, { aliasMap, candidateMap })` check for providers missing from the precomputed set, or seed the index from all provider ids that can be handled by resolver-only auth paths. Add a regression test for `createModelListAuthIndex(... env: { GOOGLE_CLOUD_API_KEY }) .hasProviderAuth("google-vertex")`.
- Confidence: high
