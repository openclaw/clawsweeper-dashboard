## Findings

### Low: Hook docs invert which `event.messages` surfaces actually deliver replies

- Kind: bug
- File: `docs/automation/hooks.md`
- Line: 125
- Evidence: The new text says `event.messages` is delivered on `message:received`, and that `session:*` events ignore pushed messages. Current code does the opposite for relevant cases: `message:received` is fired fire-and-forget in `src/auto-reply/reply/dispatch-from-config.ts:1418` and never reads `hookEvent.messages`, while `session:compact:before` and `session:compact:after` explicitly forward `hookEvent.messages` through `onHookMessages` in `src/agents/pi-embedded-runner/compaction-hooks.ts:199` and `src/agents/pi-embedded-runner/compaction-hooks.ts:297`, then route them to visible compaction notices via `src/agents/pi-embedded-runner/run.ts:1213` and `src/auto-reply/reply/agent-runner-execution.ts:2120`. The same docs page still lists `compaction-notifier` as sending visible chat notices from `session:compact:*`.
- Impact: Hook authors following the updated docs may build `message:received` hooks that silently drop replies, and may avoid or mis-debug working `session:compact:*` message hooks such as `compaction-notifier`.
- Suggested fix: Change the docs to say that `command:*` and `session:compact:before/after` currently route `event.messages`, while `message:received`, `message:sent`, `agent:bootstrap`, and gateway lifecycle events are observation/mutation-only unless a caller explicitly wires delivery. If `message:received` should become replyable, implement and test synchronous message routing instead.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. A `message:received` internal hook that pushes `event.messages` is triggered by `dispatch-from-config`, but the event is not retained or routed; a `session:compact:*` hook that pushes `event.messages` is forwarded through the compaction event path.
- Is this the best way to solve the issue? yes. This commit is docs-focused, and the source already has explicit surface-specific delivery behavior, so the safest fix is to document the actual contract precisely.
