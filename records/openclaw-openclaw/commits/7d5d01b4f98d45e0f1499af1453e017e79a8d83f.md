- Kind: reliability

- Impact: The media test lane is red on current `main`, and the retry/cleanup behavior those tests were meant to prove is no longer being exercised.

- Confidence: high

### Low: fs-safe race test no longer accepts the dependency’s boundary error code

- Kind: compatibility
- File: `src/infra/fs-safe.test.ts`
- Line: 40
- Evidence: The symlink-retarget write race test accepts only `outside-workspace`, `path-mismatch`, `path-alias`, or `invalid-path`, but current `@openclaw/fs-safe` returns `FsSafeError` with `code: "not-file"`. `pnpm test src/infra/fs-safe.test.ts src/infra/boundary-file-read.test.ts src/infra/fs-safe-defaults.test.ts src/infra/fs-safe-import-boundary.test.ts` fails this one assertion.
- Impact: The fs-safe wrapper gate is red. The protected behavior still appears to fail closed, but OpenClaw’s expected error-code contract is now out of sync with the dependency.
- Suggested fix: Decide whether `not-file` is an acceptable fail-closed boundary result for this race. If yes, update the OpenClaw assertion and any caller mappings that depend on the narrower code set; if no, normalize the dependency/wrapper back to a boundary-style code.
- Confidence: high
