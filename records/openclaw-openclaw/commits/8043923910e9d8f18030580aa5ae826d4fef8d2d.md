## Summary

Found one compatibility regression: the commit removes a named export from the published `openclaw/plugin-sdk/plugin-test-contracts` SDK subpath. The current tree still has the removal, and the SDK API baseline check fails.

## Findings

### Medium: Published plugin test contract export was removed without a compatible replacement

- Kind: compatibility
- File: `src/plugin-sdk/plugin-test-contracts.ts`
- Line: 11
- Evidence: The parent exported `loadRuntimeApiExportTypesViaJiti` from `./test-helpers/jiti-runtime-api.js`; this commit deletes both that export and `src/plugin-sdk/test-helpers/jiti-runtime-api.ts`. `v2026.4.29` also contains the export, while current `main` does not. A direct import now fails with `SyntaxError: The requested module './src/plugin-sdk/plugin-test-contracts.ts' does not provide an export named 'loadRuntimeApiExportTypesViaJiti'`. `pnpm plugin-sdk:api:check` also fails with `Plugin SDK API baseline drift detected`.
- Impact: External plugin tests or downstream code that adopted the shipped `openclaw/plugin-sdk/plugin-test-contracts` helper from `v2026.4.29` will break on upgrade. It also blocks the repo’s SDK API check until the export is restored or the API baseline is intentionally regenerated.
- Suggested fix: Restore a backwards-compatible `loadRuntimeApiExportTypesViaJiti` export, ideally as a wrapper or deprecated alias around the replacement direct-smoke path if Jiti-specific behavior is no longer wanted. If the removal is intentional, update the SDK API baseline and docs/changelog explicitly, but this is a breaking public test-SDK removal.
- Confidence: high
