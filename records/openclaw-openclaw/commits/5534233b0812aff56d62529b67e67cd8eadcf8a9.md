## Findings

### Medium: Null guard removal breaks strict test typecheck

- Kind: regression
- File: `extensions/qa-channel/src/channel.test.ts`
- Line: 392
- Evidence: The commit changes `mediaCtx` to `{ ... } | null` and removes the explicit throw-based guard, but the next assertions still dereference `mediaCtx.MediaType`, `mediaCtx.MediaPaths`, and `mediaCtx.MediaTypes`. On current `main`, `pnpm tsgo:extensions:test` fails with `TS18047: 'mediaCtx' is possibly 'null'` on lines 392-394, plus `TS2339` on line 391.
- Impact: The qa-channel extension test typecheck lane is broken, so changed-gate/CI validation for extension tests can fail even though the runtime Vitest test still passes.
- Suggested fix: Restore a real narrowing guard before dereferencing, e.g. `expect(dispatchedCtx).not.toBeNull(); if (!dispatchedCtx) throw ...; const mediaCtx = dispatchedCtx as ...;`, or assert through a helper that narrows the value.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes: after installing missing dependencies, `pnpm tsgo:extensions:test` fails on the changed lines.
- Is this the best way to solve the issue? yes: the previous explicit guard narrowed the type and preserved the intended stronger assertion; restoring equivalent narrowing fixes the lane without changing runtime behavior.
