# Commit 8654144

## Summary

Found one concrete regression in the new Slack `allowBots: "mentions"` mode. The implementation accepts bot-authored Slack control commands without a real mention because it treats command mention-bypass as if the bot were mentioned.

## Findings

### Medium: Slack `allowBots: "mentions"` accepts unmentioned bot control commands

- Kind: regression
- File: `extensions/slack/src/monitor/message-handler/prepare.ts`
- Line: 492
- Evidence: The new mentions gate uses `isDirectMessage || effectiveWasMentioned || shouldBypassMention`. `shouldBypassMention` is true for authorized control commands without any mention, so a channel-allowlisted bot message like `/new incident room` passes `allowBots: "mentions"` even though it does not mention the receiving bot. A focused repro on current `main` returned `prepared: true`, `CommandAuthorized: true`, and `WasMentioned: true` for an unmentioned bot-authored `/new incident room`.
- Impact: This breaks the advertised Discord-parity contract for `allowBots: "mentions"`: bot messages without `@mention` should be dropped. It can also reintroduce bot-to-bot activation or loop behavior for bot relays that are allowlisted only so mentioned traffic can pass.
- Suggested fix: Match Discord’s gate and only count direct messages, real mentions, and implicit mention kinds for bot-authored `"mentions"` mode. Do not count `shouldBypassMention`; add a Slack regression test equivalent to `extensions/discord/src/monitor/message-handler.preflight.test.ts:946`.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Configure Slack with `allowBots: "mentions"`, `commands.text: true`, default room mention requirement, and channel `users: ["B0AGV8EQYA3"]`; then prepare a bot-authored room message from `B0AGV8EQYA3` with text `/new incident room` and no `<@B1>` mention. Current `main` prepares the message.
- Is this the best way to solve the issue? yes. The issue is localized to the new bot mentions gate; using `wasMentioned` plus implicit mention, rather than `effectiveWasMentioned`/`shouldBypassMention`, preserves command bypass for human messages while keeping bot `"mentions"` strict.
