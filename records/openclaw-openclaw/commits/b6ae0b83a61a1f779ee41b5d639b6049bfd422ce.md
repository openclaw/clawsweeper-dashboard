# Commit b6ae0b8

## Summary

The main Telegram inbound DM/group message paths now honor `accessGroup:*` allowlists, and the added focused tests pass. I found one remaining callback/reaction authorization path where access-group expansion is still skipped for DMs.

## Findings

### Medium: Access-group-authorized DM users are still blocked from Telegram callbacks and reactions

- Kind: bug
- File: `extensions/telegram/src/bot-handlers.runtime.ts`
- Line: 793
- Evidence: `resolveTelegramEventAuthorizationContext` now receives `senderId`, but `authorizeTelegramEventSender` still builds DM access from raw `groupAllowOverride ?? allowFrom` and passes it straight into `normalizeDmAllowFromWithStore`. That normalizer treats `accessGroup:owners` as a nonnumeric invalid entry and produces no numeric entries. This path gates `message_reaction` and every `callback_query` before plugin/model/native callback handling.
- Impact: A sender allowed by `channels.telegram.allowFrom: ["accessGroup:owners"]` can send a DM, but follow-up inline buttons or reaction callbacks from the same DM sender are silently denied. That leaves the “callback authorization” part of this fix incomplete.
- Suggested fix: Expand DM `allowFrom` with `expandTelegramAllowFromWithAccessGroups` inside the event authorization context, then reuse the expanded/effective DM allowlist in `authorizeTelegramEventSender` and `isTelegramModelCallbackAuthorized`. Also keep valid `accessGroup:*` aliases out of the numeric invalid-entry warning path when they simply do not match the current sender.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? Yes. Configure Telegram DM access with `allowFrom: ["accessGroup:owners"]`, put the callback sender in `accessGroups.owners.members.telegram`, generate any DM inline callback, then press it. Source tracing shows the callback path normalizes the raw access-group entry without expansion.
- Is this the best way to solve the issue? Yes. The message paths already use the narrow Telegram-owned expansion helper; the callback/reaction path should share that expanded DM allowlist instead of loosening numeric Telegram ID validation.
