# Commit 71c855f

## Summary

Found one concrete regression in the refactored `findOcPaths` recursive wildcard dispatcher. Terminal `**` searches now emit duplicate concrete paths, and JSONL terminal `**` also includes the file root where the previous JSONL walker explicitly avoided a root match.

## Findings

### Medium: Recursive `**` find emits duplicate matches

- Kind: regression
- File: `extensions/oc-path/src/oc-path/find.ts`
- Line: 197
- Evidence: The new shared dispatcher handles `**` by emitting the current zero-length match, then for every child recursing with both `i + 1` and `i` (`extensions/oc-path/src/oc-path/find.ts:197`). For a terminal pattern, the `i + 1` child recursion emits the child once, and the retained-`i` recursion emits the same child again. A focused smoke on current `main` produced:
  - `findOcPaths(parseJsonc('{"a":{"b":1},"c":2}'), parseOcPath('oc://x/**'))`
  - Output paths: `oc://x`, `oc://x/a`, `oc://x/a`, `oc://x/a.b`, `oc://x/a.b`, `oc://x/c`, `oc://x/c`
  - For Markdown: `oc://x`, `oc://x/tools`, `oc://x/tools`
  - For JSONL terminal `**`: `oc://s` root plus duplicate `L1` and `L2` line paths.
- Impact: `openclaw path find 'oc://file/**'` over public CLI/API output overcounts matches and can cause downstream scripts to process the same node more than once. That is especially risky for automation that treats find output as a worklist.
- Suggested fix: Restore the old recursive wildcard shape: take the zero-segment branch once at the current node (`ops.walk(node, subs, i + 1, walked, onMatch)`), then descend children only with `i` retained. Alternatively dedupe `concretePaths`, but fixing the walker avoids the extra work and preserves previous JSONL root behavior.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. The inline `pnpm exec tsx -e ...` smoke above reproduces duplicate paths on current `main`, and the same duplicate branch exists in the reviewed commit.
- Is this the best way to solve the issue? yes. The regression comes directly from the new shared dispatcher’s extra consumed child branch; restoring the prior traversal semantics is narrower than deduping after expansion.
