# Commit f6f8d74419

## Summary

Found one privacy regression in the new restart-drain blocker diagnostics: active task labels and raw task titles can be written into Gateway logs, and those log messages can flow into diagnostics support exports.

## Findings

### Medium: Restart blocker logs can expose raw task text

- Kind: privacy
- File: `src/gateway/server-reload-handlers.ts`
- Line: 190
- Evidence: `getInspectableActiveTaskRestartBlockers()` copies `task.label` and `task.task` into blocker objects (`src/tasks/task-registry.maintenance.ts:856` and `src/tasks/task-registry.maintenance.ts:858`). Both restart-drain formatters then interpolate those fields directly into warning messages (`src/gateway/server-reload-handlers.ts:190` and `src/cli/gateway-cli/run-loop.ts:348`). `TaskRecord.task` is the raw task text (`src/tasks/task-registry.types.ts:67`), and the diagnostic support log sanitizer keeps ordinary `msg` strings unless they match a narrow unsafe-message regex (`src/logging/diagnostic-support-log-redaction.ts:141`). The Gateway docs describe diagnostics exports as shareable sanitized log summaries that omit chat text and similar user payloads (`docs/cli/gateway.md:242`).
- Impact: A background task like “summarize this private contract …” can be logged as `title=summarize this private contract ...` when restart deferral is blocked. That local log line may also survive into `logs/openclaw-sanitized.jsonl` because the generic message does not necessarily contain words such as `prompt` or `message contents`. This weakens the support-export privacy contract.
- Suggested fix: Keep blocker diagnostics to operational identifiers and bounded metadata: `taskId`, `runId`, `status`, and `runtime`. Omit `label` and `title` from restart-blocker logs, or route them through an explicit privacy-safe task summary helper that is also covered by the support-log sanitizer tests.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Create or mock a running/queued `TaskRecord` whose `task` or `label` contains private text, then trigger config restart deferral or a restart with active work. The current formatter emits that text in `restart blocked by active task run(s): ...`.
- Is this the best way to solve the issue? yes. Restart blocking only needs IDs and status to let an operator inspect the task separately; raw labels/titles are not needed in logs.
