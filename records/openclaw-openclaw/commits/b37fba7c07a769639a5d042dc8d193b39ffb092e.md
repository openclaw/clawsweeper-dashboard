### High: ClawHub publish workflow calls a non-existent pnpm script

- Kind: regression
- File: `.github/workflows/plugin-clawhub-release.yml`
- Line: 244
- Evidence: The new `Verify package-local runtime build` step runs `pnpm release:plugins:npm:runtime:check --package "${{ matrix.plugin.packageDir }}"`, but `package.json` only defines `release:plugins:clawhub:check`, `release:plugins:clawhub:plan`, `release:plugins:npm:check`, and `release:plugins:npm:plan`. - Impact: Any ClawHub release run with candidates reaches `preview_plugin_pack` and fails before the dry-run or publish step, blocking ClawHub plugin publication. A live plan for `@openclaw/diagnostics-otel` currently produced a candidate, so this path is executable.
- Suggested fix: Either add the missing package script, likely pointing to `node scripts/check-plugin-npm-runtime-builds.mjs`, or call the existing script directly from the workflow: `node scripts/check-plugin-npm-runtime-builds.mjs --package "${{ matrix.plugin.packageDir }}"`.
- Confidence: high
