# Commit 00ad13b

## Summary

Found one concrete regression: the new bare `@openclaw/codex` default currently resolves to a prerelease on npm, and OpenClaw rejects prereleases for bare npm specs.

## Findings

### Medium: Bare Codex npm default breaks the live plugin install path

- Kind: regression
- File: `scripts/e2e/codex-npm-plugin-live-docker.sh`
- Line: 50
- Evidence: This commit changes the live Docker default from `npm:@openclaw/codex@beta` to `npm:@openclaw/codex`. Current npm metadata resolves `@openclaw/codex` to `2026.5.1-beta.1` with tags `latest=2026.5.1-beta.1` and `beta=2026.5.2-beta.2`. The installer explicitly rejects prerelease resolutions for bare specs in `src/infra/npm-registry-spec.ts:125`, and `installPluginFromNpmSpec` enforces that after `npm view` in `src/plugins/install.ts:1127`. A focused CLI repro with an isolated state dir fails before install: `openclaw plugins install npm:@openclaw/codex` reports `Resolved @openclaw/codex to prerelease version 2026.5.1-beta.1, but prereleases are only installed when explicitly requested.`
- Impact: The Codex npm plugin live Docker E2E now fails under current registry state unless callers override `OPENCLAW_CODEX_NPM_PLUGIN_SPEC`. The changed docs also include `openclaw plugins install npm:@openclaw/codex` in `docs/plugins/manage-plugins.md:53`, which points users at the same failing path while `latest` is a prerelease.
- Suggested fix: Either keep the live check/default docs on an explicit prerelease selector such as `@beta` until npm `latest` points at a non-prerelease, or retag the official packages so bare/default specs resolve to stable versions before relying on bare installs. Add a focused test that covers the current registry contract: bare official npm spec resolving to a prerelease must not be used as a passing default.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. With an isolated state dir, `OPENCLAW_STATE_DIR=<tmp> OPENCLAW_CONFIG_PATH=<tmp>/openclaw.json pnpm openclaw plugins install npm:@openclaw/codex` fails with the prerelease rejection above.
- Is this the best way to solve the issue? unclear. The safest immediate fix is restoring the explicit `@beta` live default; the longer-term fix is aligning npm dist-tags with the installer’s prerelease policy.
