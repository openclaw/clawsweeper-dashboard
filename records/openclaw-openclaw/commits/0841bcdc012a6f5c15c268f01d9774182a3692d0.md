## Summary

Found one validation coverage gap in the new suppressed-model check.

## Findings

### Low: Suppressed subagent default models are not validated

- Kind: bug
- File: `src/config/validation.ts`
- Line: 1111
- Evidence: `collectConfiguredModelRefs()` scans `model`, media/PDF model fields, and `agents.*.models` keys, but it does not scan `subagents.model`. That field is documented and used at runtime: `resolveSubagentConfiguredModelSelection()` reads `agents.list[].subagents.model` before agent/default model fallback, then reads `agents.defaults.subagents.model`.
- Impact: `openclaw config validate` can still accept `agents.defaults.subagents.model` or `agents.list[].subagents.model` set to an unconditionally suppressed provider/model ref, such as the commit’s rejected `openai-codex/gpt-5.3-codex-spark`. The failure is deferred until `sessions_spawn` uses the configured default, so the new “reject unavailable provider refs” validation is not consistently applied to documented subagent model defaults.
- Suggested fix: Have `collectFromAgent()` call `collectModelConfig(`${path}.subagents.model`, agent.subagents?.model)` for both defaults and list entries, and add tests for `agents.defaults.subagents.model` plus `agents.list[].subagents.model`.
- Confidence: medium
