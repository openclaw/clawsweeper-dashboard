# Commit c5bc4b6892

## Summary

Found one regression risk in the Telegram audio preflight echo path: transcript echoes now happen before the Telegram group mention gate has decided whether the message should be ignored.

## Findings

### Medium: Mention-gated Telegram groups can echo transcripts for messages the bot later skips

- Kind: privacy
- File: `src/media-understanding/audio-preflight.ts`
- Line: 63
- Evidence: `transcribeFirstAudio` now calls `sendTranscriptEcho` whenever `tools.media.audio.echoTranscript` is enabled. Telegram calls this preflight path in `extensions/telegram/src/bot-message-context.body.ts` before computing `matchesMentionWithExplicit` and before the `no-mention` skip branch. For group audio with `requireMention: true`, no text, and a transcript that does not mention the bot, the new echo is sent first, then `resolveTelegramInboundBody` returns `null` as a skipped no-mention message.
- Impact: A mention-gated Telegram group can get a system transcript echo for voice notes that were not addressed to the bot. That bypasses the visible “no mention means no bot response” contract and can expose a machine transcript to the whole group even though the agent turn is dropped.
- Suggested fix: Do not emit preflight echoes inside the generic preflight helper until the caller has accepted the inbound message. For Telegram, either suppress echo during group mention-detection preflight and send it only after `mentionDecision.shouldSkip` is false, or restrict this immediate echo to DM preflight and let accepted group messages use the existing post-accept media echo path.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Configure Telegram group `requireMention: true`, set `messages.groupChat.mentionPatterns` so mention detection is available, enable `tools.media.audio.echoTranscript`, then send an audio-only group voice note whose transcript does not mention the bot. The code path sends the echo during preflight and then hits the no-mention skip branch.
- Is this the best way to solve the issue? yes. The fix should keep the DM repair from this commit, but move the echo decision to the Telegram caller after routing/access/mention acceptance, rather than making every preflight transcription immediately deliver a user-visible echo.
