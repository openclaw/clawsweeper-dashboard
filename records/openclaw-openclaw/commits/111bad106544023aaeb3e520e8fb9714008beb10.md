# Commit 111bad1065

## Summary

The commit updates the doctor warning from `openclaw migrate codex --dry-run` to `openclaw migrate plan codex`, but the linked failure mode is still present when the Codex migration provider is unavailable.

## Findings

### Low: Doctor still recommends a Codex migration command that can hit the same unknown-provider failure

- Kind: bug
- File: `src/commands/doctor/shared/codex-native-assets.ts`
- Line: 204
- Evidence: The warning is emitted whenever Codex runtime/plugin config and personal Codex assets are detected, and now tells users to run `openclaw migrate plan codex`. That subcommand routes through `migratePlanCommand`, which calls `createMigrationPlanWithProgress`, then `createMigrationPlan`, then `resolveMigrationProvider`. `resolveMigrationProvider` still throws `Unknown migration provider "codex"` when the Codex migration provider is not registered. The old default path `openclaw migrate codex --dry-run` also delegates to the same plan path, so this string-only change does not address the provider-missing state reported in #84948.
- Impact: Users in the reported state can still follow doctor’s guidance and get the same unknown-provider error, just from the new canonical preview syntax.
- Suggested fix: Make the doctor warning conditional on Codex migration-provider availability, or change the warning to first direct users through the plugin/doctor repair path when `codex` is not resolvable. Add a focused regression test for “Codex assets found, Codex runtime configured, migration provider unavailable.”
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Source trace is enough: `src/commands/doctor/shared/codex-native-assets.ts:136` allows the warning based on configured Codex runtime/plugin state, while `src/commands/migrate.ts:295` and `src/commands/migrate/providers.ts:18` show `migrate plan codex` still fails if provider resolution returns none.
- Is this the best way to solve the issue? unclear. Guarding the warning is the smallest fix; making the Codex migration provider always resolvable may be better if that is the intended product contract.
