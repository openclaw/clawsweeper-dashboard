# Commit 8a406528b4

## Summary

Found one privacy regression: the new user MCP “fingerprint” persists the full projected MCP config, including sensitive URL/header values, into the Codex app-server binding file.

## Findings

### Medium: User MCP fingerprint stores raw MCP secrets in the session binding

- Kind: privacy
- File: `extensions/codex/src/app-server/thread-lifecycle.ts`
- Line: 554
- Evidence: `fingerprintUserMcpServersConfigPatch()` returns `JSON.stringify(stabilizeJsonValue(configPatch))`, and `writeCodexAppServerBinding()` writes `userMcpServersFingerprint` into `<session>.codex-app-server.json` at `extensions/codex/src/app-server/session-binding.ts:151`. The projected patch can contain static `http_headers` from `src/agents/cli-runner/bundle-mcp-codex.ts:49` and URLs/env copied by `src/agents/cli-runner/bundle-mcp-adapter-shared.ts:48` and `:55`. Config tests already mark `mcp.servers.*.headers.*` and `mcp.servers.*.url` as sensitive in `src/config/schema.test.ts:115`.
- Impact: A config such as `mcp.servers.remote.headers.Authorization = "Bearer secret-token"` or a URL containing credentials/tokens is duplicated into the app-server binding file beside the session. A focused smoke script confirmed the binding contained `secret-token`, `secret123`, and a stdio env value after `startOrResumeThread()`.
- Suggested fix: Store a digest of the stable projection, for example SHA-256 over the stabilized JSON, instead of the raw JSON string. Existing raw persisted values should be treated as stale or migrated by recomputing on next start without preserving the secret-bearing string.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Start a Codex app-server thread with `mcp.servers.remote.headers.Authorization: "Bearer secret-token"` and inspect `<session>.codex-app-server.json`; the ad hoc local check showed `containsHeaderSecret`, `containsUrlSecret`, and `containsEnvSecret` all `true`.
- Is this the best way to solve the issue? yes. The binding only needs equality comparison for thread rotation, so a stable hash preserves behavior without persisting config contents.
