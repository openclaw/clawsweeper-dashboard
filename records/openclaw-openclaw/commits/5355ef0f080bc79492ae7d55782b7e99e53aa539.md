# Commit 5355ef0

## Summary

Found one low-severity diagnostics regression: the new registry-wide dedupe can collapse distinct plugin discovery/package-entry errors when their text matches but their source paths differ.

## Findings

### Low: Registry diagnostic dedupe hides distinct broken plugin sources

- Kind: regression
- File: `src/plugins/manifest-registry.ts`
- Line: 497
- Evidence: `dedupePluginDiagnostics` keys only on `level`, `pluginId`, and `message`, then `loadPluginManifestRegistry` applies it to every registry diagnostic at `src/plugins/manifest-registry.ts:803`. Discovery/package-entry diagnostics often distinguish separate failures by `source`, not message; for example `src/plugins/package-entry-resolution.ts:570` reports the same `package.json openclaw.runtimeExtensions length ...` error with a different source label per plugin directory.
- Impact: if two configured plugin load paths fail with the same structural error, only the first diagnostic is returned. Config validation/status can then report one broken configured plugin while silently omitting the other, making `plugins.load.paths` repair incomplete.
- Suggested fix: include `diagnostic.source ?? ""` in the general dedupe key, or narrow this dedupe to the manifest compatibility diagnostic path that actually produces the config-vs-global duplicate warning.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. A focused `tsx` smoke created two distinct temp plugin directories, each with `openclaw.extensions` length 1 and `openclaw.runtimeExtensions` length 2, then loaded them through `plugins.load.paths`. Current `main` returned only one diagnostic even though both paths were independently invalid.
- Is this the best way to solve the issue? yes. Preserving `source` in the general key keeps the intended duplicate-message suppression from hiding separate source-specific failures; if the original repeated manifest warning should still collapse across different sources, that case should be handled narrowly where those compatibility diagnostics are pushed.
