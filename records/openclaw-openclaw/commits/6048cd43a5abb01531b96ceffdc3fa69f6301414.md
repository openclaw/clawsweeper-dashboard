Found one concrete gap in the recovered-warning fix: media-only successful cron output can still be converted into a fatal warning payload because the new metadata marker is assigned before tool media is merged into terminal payloads.

## Findings

### Medium: Media-only cron output is still fatal after a recovered middleware warning

- Kind: bug
- File: `src/agents/pi-embedded-runner/run/payloads.ts`
- Line: 517
- Evidence: `buildEmbeddedRunPayloads` only sets `nonTerminalToolErrorWarning` when `hasUserFacingAssistantReply` is true. But structured tool media is merged later in `src/agents/pi-embedded-runner/run.ts:2662` via `mergeAttemptToolMediaPayloads`, so a media-only successful cron run has no assistant text at marker time. A focused runtime probe using the production helpers produced one unmarked payload: `{ text: "⚠️ 🛠️ Exec failed", isError: true, mediaUrl: "file:///tmp/cron-report.png" }`, and `resolveCronPayloadOutcome` returned `hasFatalErrorPayload: true` with delivery reduced to the warning text.
- Impact: The source PR explicitly claims media/structured cron output after a recovered middleware warning remains deliverable and nonfatal, but the integrated production path can still record the scheduled run as failed and deliver the warning instead of the generated media.
- Suggested fix: Thread successful tool-media evidence into `buildEmbeddedRunPayloads`, or have `mergeAttemptToolMediaPayloads` preserve/create a separate successful media payload before the marked diagnostic warning. Add an integrated regression test that composes `buildEmbeddedRunPayloads` + `mergeAttemptToolMediaPayloads` + `resolveCronPayloadOutcome`.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes: run the production helper composition for `assistantTexts: []`, `lastToolError.middlewareError: true`, then merge `toolMediaUrls` and resolve cron outcome.
- Is this the best way to solve the issue? yes: the fix should mark or preserve the actual post-run success signal at the same layer where media-only success is introduced, not rely on helper-only manually marked payloads.
