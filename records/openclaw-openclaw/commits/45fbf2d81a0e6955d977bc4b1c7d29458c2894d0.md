# Commit 45fbf2d

## Summary

Found one privacy/regression risk in the verbose gate change: inherited `verboseDefault: "on"` now exposes text-only tool summaries in normal groups, where the base commit suppressed them.

## Findings

### Medium: inherited verbose defaults now expose normal group tool summaries

- Kind: privacy
- File: `src/auto-reply/reply/dispatch-from-config.ts`
- Line: 1496
- Evidence: Current `fallbackLevel` includes `sessionAgentCfg?.verboseDefault` / `cfg.agents?.defaults?.verboseDefault`, then `shouldSendToolSummaries()` mirrors that verbose state and `resolveToolDeliveryPayload()` returns text-only tool payloads when it is true. The new regression test at `src/auto-reply/reply/dispatch-from-config.test.ts:1635` confirms a Matrix `ChatType: "group"` with `agents.defaults.verboseDefault: "on"` sends `🔧 exec: pwd`. In the base SHA, normal non-forum groups were hard-gated by `ctx.ChatType !== "group" || ctx.IsForum === true`, so the same text-only summary was suppressed.
- Impact: Operators who enabled global or per-agent verbose defaults for direct-chat debugging can start exposing command names, arguments, progress, or tool output snippets in shared group rooms after this upgrade, without an explicit per-group `/verbose on`.
- Suggested fix: keep separate predicates for inherited verbose defaults and explicit session verbose state, and require the explicit session state for normal group/channel text-only tool summaries. Direct chats and forum topics can continue using the effective verbose state if that is the intended contract.
- Confidence: medium

## Details

- Do we have a high-confidence way to reproduce the issue? yes: the committed test case at `src/auto-reply/reply/dispatch-from-config.test.ts:1635` demonstrates the behavior on current `main`; comparing against the base code shows it was previously suppressed for normal groups.
- Is this the best way to solve the issue? unclear: if maintainers intentionally want `verboseDefault` to apply to all group rooms, then the code is consistent with the test. If the intended safety contract is explicit group `/verbose` only, the suggested split is the narrowest fix.
