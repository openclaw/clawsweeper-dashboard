# Commit 46a0409

## Summary

The production Windows temp-dir fix looks narrow and source-backed. I found one low-severity test hygiene regression: the WhatsApp media test still calls the filesystem-creating temp resolver while `process.platform` is mocked to `win32`, which creates a repo-root `\tmp\openclaw-<uid>` directory on POSIX runners.

## Findings

### Low: Windows-path media test leaves a repo-local temp directory on POSIX runners

- Kind: reliability
- File: `extensions/whatsapp/src/media.test.ts`
- Line: 386
- Evidence: The test mocks `process.platform` to `win32` at `extensions/whatsapp/src/media.test.ts:380`, then computes `localRoots: [resolvePreferredOpenClawTmpDir()]` while that mock is active. After this commit, the resolver takes the Windows early return at `src/infra/tmp-openclaw-dir.ts:175` and uses `path.win32.join` for fallback construction at `src/infra/tmp-openclaw-dir.ts:81`. On this Linux runner, `path.win32.join("/tmp", "openclaw-1001")` produced `\tmp\openclaw-1001`, which POSIX treats as a relative repo-root path. Running the focused test passed but left `"\\\\tmp\\\\openclaw-1001/"` visible via `git ls-files --others --exclude-standard --directory`; I removed that generated artifact after confirming it.
- Impact: Focused local/CI test runs can leave unexpected untracked directories in the repository root, confusing workspace cleanliness checks and accumulating artifacts. The test already fixed the same pattern in the previous Windows-stat case by resolving the real temp root before mocking platform, but missed this later network-path case.
- Suggested fix: Resolve `realTmpRoot` before installing the `win32` platform spy in the network-path test too, or pass a fixed harmless explicit root that does not call `resolvePreferredOpenClawTmpDir()` while the platform mock is active.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. On a POSIX runner with dependencies installed, run `pnpm test extensions/whatsapp/src/media.test.ts` or the combined focused command below, then inspect `git ls-files --others --exclude-standard --directory`.
- Is this the best way to solve the issue? yes. The production helper should keep the Windows branch; the safer fix is in the test setup, matching the nearby `realTmpRoot` pattern already added at `extensions/whatsapp/src/media.test.ts:358`.
