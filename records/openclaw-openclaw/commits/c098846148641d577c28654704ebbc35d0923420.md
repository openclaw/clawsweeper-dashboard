# Commit c098846

## Summary

Found one regression in the overflow-recovery compaction path: it does not pass the newly added effective fallback override into `buildEmbeddedCompactionRuntimeContext`, unlike the timeout-recovery path.

## Findings

### Medium: Overflow compaction can ignore explicit session model override fallback rules

- Kind: regression
- File: `src/agents/pi-embedded-runner/run.ts`
- Line: 1430
- Evidence: `agent-command.ts:940` computes `effectiveFallbacksOverride`, and `attempt-execution.ts:577` passes it into embedded runs. `run.ts:1290` forwards it into timeout-recovery compaction, but the overflow-recovery context at `run.ts:1430` omits `modelFallbacksOverride: params.modelFallbacksOverride`. The runtime delegate then spreads that `runtimeContext` into `compactEmbeddedPiSessionDirect` (`src/context-engine/delegate.ts:54`). Without the override, `compact.ts:330` falls back to `resolveRunModelFallbacksOverride`/default configured fallbacks, bypassing `resolveEffectiveModelFallbacks` behavior where explicit user session model overrides return `[]` (`src/agents/agent-scope.ts:207`).
- Impact: A user-selected exact session model can still use the global fallback chain during overflow-recovery compaction. That can send the transcript-summary prompt to an unintended fallback provider, and it diverges from normal attempt fallback semantics.
- Suggested fix: Add `modelFallbacksOverride: params.modelFallbacksOverride` to the overflow-recovery `buildEmbeddedCompactionRuntimeContext` call, mirroring the timeout-recovery call, and add a focused overflow-compaction test asserting the runtime context preserves an empty override.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? Yes. Run an embedded session with `params.modelFallbacksOverride: []`, trigger context overflow, and inspect the `contextEngine.compact` runtime context; current code omits the field in the overflow path.
- Is this the best way to solve the issue? Yes. The timeout path already uses the intended one-line propagation pattern.
