# Commit b7704b917e

## Summary

Found three actionable issues in the new bundled `meme-maker` skill: one medium privacy/security issue around password handling, plus two low-severity user-visible command problems.

## Findings

### Medium: Imgflip passwords can be placed in argv and agent transcripts

- Kind: privacy
- File: `skills/meme-maker/scripts/meme.mjs`
- Line: 300
- Evidence: `renderImgflip` accepts `flags.password` before `IMGFLIP_PASS`, and the error/docs advertise `--username/--password` as a supported credential path (`skills/meme-maker/SKILL.md:22`, `skills/meme-maker/scripts/meme.mjs:300-304`). In agent-run shell commands, that puts a third-party password into command text, process argv, and potentially saved tool transcripts.
- Impact: Users or agents may leak Imgflip credentials while following the documented hosted-render path.
- Suggested fix: Remove `--password` support and require `IMGFLIP_PASS`, secret config, or stdin-style secret input that does not appear in argv. Keep `--username` only if needed.
- Confidence: high

### Low: Advertised PNG quick start fails with the declared skill requirements

- Kind: compatibility
- File: `skills/meme-maker/SKILL.md`
- Line: 16
- Evidence: The skill declares only `node` as required (`skills/meme-maker/SKILL.md:4`) and advertises a “Local PNG” quick-start command, but PNG rendering dynamically imports `sharp` (`skills/meme-maker/scripts/meme.mjs:282-290`). In this checkout, `node -e 'import("sharp")...'` reports `ERR_MODULE_NOT_FOUND`, and the documented PNG render exits with `error: PNG output needs the optional sharp package`.
- Impact: A bundled skill visible to users by default presents PNG rendering as a quick-start feature even though a normal install may only support SVG.
- Suggested fix: Either remove PNG from the quick start/changelog and document it as explicitly optional, or ship/declare a supported PNG conversion dependency path.
- Confidence: high

### Low: The documented `suggest` example returns no suggestions

- Kind: bug
- File: `skills/meme-maker/SKILL.md`
- Line: 14
- Evidence: The quick start says to run `suggest "slow python image scripts"`, but `scoreTemplate` only scores token overlap and `suggest` filters out score `0` entries (`skills/meme-maker/scripts/meme.mjs:99-111`, `skills/meme-maker/scripts/meme.mjs:358-367`). Running that exact example with `--json` returns `[]`.
- Impact: The first suggested workflow for users can silently produce no usable result.
- Suggested fix: Change the example to match curated tags, or make `suggest` fall back to a small ranked/default set when no token overlap exists.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. The password issue is visible from the script and docs. The PNG failure reproduces with the documented `.png` render command in this checkout. The suggest issue reproduces with the documented query returning `[]`.
- Is this the best way to solve the issue? yes for credential handling: removing password argv is the safer contract. For PNG and suggest behavior, either tightening docs or changing implementation would be acceptable; the current docs and runtime do not align.
