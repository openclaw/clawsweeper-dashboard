# Commit d94012a

## Summary

Found one actionable workflow bug: the commit serializes the dry-run preview matrix, but the real ClawHub publish matrix still runs in parallel.

## Findings

### Low: Real ClawHub publish jobs are still parallel

- Kind: reliability
- File: `.github/workflows/plugin-clawhub-release.yml`
- Line: 208
- Evidence: The commit adds `strategy.max-parallel: 1` only under `preview_plugin_pack` at `.github/workflows/plugin-clawhub-release.yml:146`, whose command is `bash scripts/plugin-clawhub-publish.sh --dry-run`. The actual `publish_plugins_clawhub` job at `.github/workflows/plugin-clawhub-release.yml:200` still has a matrix strategy without `max-parallel`, and its command is `bash scripts/plugin-clawhub-publish.sh --publish`. GitHub Actions runs jobs in parallel by default and uses `jobs.<job_id>.strategy.max-parallel` to limit matrix parallelism. ([docs.github.com](https://docs.github.com/en/actions/reference/workflows-and-actions/workflow-syntax?utm_cta=events-page-featured-vhol))
- Impact: A manual `publish_scope=all-publishable` run can still start all real publish matrix children concurrently. Current `main` has 22 ClawHub-publishable plugin packages, so the release workflow still has the concurrent publish behavior this commit appears intended to remove.
- Suggested fix: Add `max-parallel: 1` to `publish_plugins_clawhub.strategy`; optionally keep or remove the preview serialization depending on whether serial dry-runs are useful.
- Confidence: high
