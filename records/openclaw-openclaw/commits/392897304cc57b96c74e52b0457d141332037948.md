# Commit 3928973

## Summary

Found one low-severity Matrix progress-draft regression. The delayed progress gate is not wired when Matrix progress lines are disabled, so the documented label-only progress draft no longer appears.

## Findings

### Low: Matrix progress mode loses label-only drafts when `toolProgress` is disabled

- Kind: regression
- File: `extensions/matrix/src/matrix/monitor/handler.ts`
- Line: 1589
- Evidence: Matrix still suppresses standalone progress messages in progress mode (`shouldSuppressDefaultToolProgressMessages` is true at `extensions/matrix/src/matrix/monitor/handler.ts:1480`), but `buildPreviewToolProgressReplyOptions()` returns immediately when `shouldStreamPreviewToolProgress` is false at `extensions/matrix/src/matrix/monitor/handler.ts:1589`. That means no tool/item/plan callbacks call `progressDraftGate.noteWork()`. This commit also removed the old `onReplyStart` draft update, so `streaming.mode: "progress"` plus `progress.toolProgress: false` now has no path to create the label-only Matrix draft. The docs still promise this mode keeps the draft while hiding tool lines: `docs/concepts/progress-drafts.md:209`.
- Impact: Matrix users who intentionally hide progress lines lose all visible in-progress feedback for long tool chains, while standalone tool-progress messages are also suppressed. Other touched channels still note work even when progress lines are disabled.
- Suggested fix: For Matrix progress mode, install work-event callbacks whenever `shouldSuppressDefaultToolProgressMessages` is true, and only skip adding lines when `shouldStreamPreviewToolProgress` is false. This matches the Discord/Slack/Telegram/Teams shape.
- Confidence: high

- Do we have a high-confidence way to reproduce the issue? yes: configure Matrix with `streaming.mode: "progress"` and `progress.toolProgress: false`, then trigger a long tool run. The current handler sets `suppressDefaultToolProgressMessages: true` but exposes no `onToolStart` callback, matching the existing focused test at `extensions/matrix/src/matrix/monitor/handler.test.ts:2784`.
