# Commit fa68929

## Summary

Found one concrete regression in the new channel streaming `commandText: "status"` policy. It hides command details from `progressText`, but not from `meta` or `summary`, and current Codex command item events put command details in `meta`.

## Findings

### Medium: `commandText: "status"` still leaks command text from item metadata

- Kind: regression, privacy
- File: `src/plugin-sdk/channel-streaming.ts`
- Line: 343
- Evidence: In the `item` formatter, `input.meta` and `input.summary` are always preferred before the status-mode suppression check, so only `progressText` is hidden. Current Codex app-server command item events populate `meta` from the command at `extensions/codex/src/app-server/event-projector.ts:618` and `extensions/codex/src/app-server/event-projector.ts:1061`. Focused repro after dependency install:

  ```text
  formatChannelProgressDraftLineForEntry(
    { streaming: { mode: "partial", preview: { commandText: "status" } } },
    { event: "item", itemKind: "command", name: "bash", meta: "run tests, `pnpm test -- --watch=false`" }
  )
  => 🛠️ Bash: run tests, `pnpm test -- --watch=false`
  ```

  A second repro with `summary: "exec ls ~/Desktop"` also returned `🛠️ Exec: exec ls ~/Desktop`.

- Impact: The new config is documented as hiding raw command/exec text in preview/progress lines, but command item events can still surface command details in Telegram, Slack, Discord, Matrix, and other compact-progress channels that use `formatChannelProgressDraftLineForEntry`. This undercuts the fix for `#77072`, especially in group chats or shared channels where command text may include sensitive local paths or operational details.
- Suggested fix: When `options.commandText === "status"` and `isCommandProgressItem(input)` is true, suppress all command detail fields (`meta`, `summary`, and `progressText`) or route them through a redacted/status-only formatter. Also consider scoping `streaming.progress.commandText` to `mode: "progress"` in `resolveChannelStreamingPreviewCommandText`; it currently overrides `preview.commandText` even in partial/block mode.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. The `pnpm exec tsx` formatter repro above uses current `main` source and shows the raw command detail survives despite `preview.commandText: "status"`.
- Is this the best way to solve the issue? yes for the main leak: the suppression belongs in the shared SDK formatter because all affected channel plugins call the shared `*ForEntry` helpers.
