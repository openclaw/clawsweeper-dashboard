# Commit 27b35c5

## Summary

The new Groq static manifest catalog includes multiple model IDs whose Groq shutdown dates already passed in 2025. Because the provider is marked `static`, OpenClaw now surfaces those rows directly in `models list --all --provider groq`, so users can pick dead Groq model IDs and hit provider-side failures.

## Findings

### Medium: Static Groq catalog exposes shutdown/deprecated model IDs as normal rows

- Kind: bug
- File: `extensions/groq/openclaw.plugin.json`
- Line: 37
- Evidence: The manifest adds `deepseek-r1-distill-llama-70b`, `gemma2-9b-it`, `llama3-70b-8192`, `llama3-8b-8192`, `mistral-saba-24b`, and `qwen-qwq-32b` as catalog models, with no `status`, so normalization defaults them to `available` in `src/model-catalog/normalize.ts:481`. Groq’s deprecation page lists shutdown dates for those IDs: Qwen QwQ on 2025-07-14, Mistral Saba on 2025-07-30, both Llama 3 IDs on 2025-08-30, DeepSeek R1 Distill Llama 70B on 2025-10-02, and Gemma 2 9B on 2025-10-08.  The same manifest sets `discovery.groq` to `static` at `extensions/groq/openclaw.plugin.json:291`, and provider-filtered `--all` listing uses static manifest rows without falling back to runtime discovery in `src/commands/models/list.source-plan.ts:71` and `src/commands/models/list.source-plan.ts:91`. A focused smoke confirmed `pnpm openclaw models list --all --provider groq --json` returns all 18 new rows, including the shutdown IDs.
- Impact: Groq users browsing or selecting Groq models can choose IDs that Groq has already shut down, causing model requests to fail even though OpenClaw’s static catalog presents them as valid selectable models. The deprecated `moonshotai/kimi-k2-instruct` row is less severe because Groq says it redirects to `moonshotai/kimi-k2-instruct-0905`, but it is also listed as an ordinary row instead of deprecated/replaced. 
- Suggested fix: Remove the shutdown IDs from the static catalog, or mark them disabled only if callers actually suppress disabled rows before selection. For redirecting models such as Kimi K2, add `status: "deprecated"` and `replacedBy`, and make list/picker surfaces respect that status if they do not already.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes: run `pnpm openclaw models list --all --provider groq --json` on this commit and observe the shutdown IDs in the returned Groq rows.
- Is this the best way to solve the issue? yes: a static catalog should only be authoritative for active/static-safe rows; dead vendor IDs should not be advertised as selectable models.
