# Commit 2f8cb86381

## Summary

The commit fixes the immediate `CLAUDE.md` symlink failure, but the new `--changed` logic skips every changed symlink under first-party paths. That creates a concrete OpenGrep PR-diff bypass for source-path symlinks whose targets live outside the changed-scan prefixes.

## Findings

### Medium: Changed source symlinks can hide target content from the OpenGrep PR-diff scan

- Kind: security
- File: `scripts/run-opengrep.sh`
- Line: 119
- Evidence: `scripts/run-opengrep.sh:119` now does `if [[ -L "$path" ]]; then continue`, after the changed-path filter has already limited candidates to `src|extensions|apps|packages|scripts`. A temp-repo repro adding `src/opengrep-hidden.ts -> ../docs/opengrep-hidden.ts` plus the target file produced `→ No changed first-party paths for opengrep.`, so the scanner was never invoked for the new target content.
- Impact: a PR can introduce executable first-party surface through a symlink under `src/` while storing the actual file under an unscanned prefix such as `docs/`. The PR-diff OpenGrep gate then passes as if no scanable first-party path changed.
- Suggested fix: narrow the skip to the intended duplicate guide aliases, for example `CLAUDE.md -> AGENTS.md`, or resolve symlinks and add the in-repo target to `SCAN_PATHS`; fail closed for source symlinks that resolve outside the repo or outside approved scan roots.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. In a temporary git repo using this script, commit a `docs/opengrep-hidden.ts` file and a `src/opengrep-hidden.ts` symlink pointing to it, then run `OPENCLAW_OPENGREP_BASE_REF='HEAD~1...HEAD' scripts/run-opengrep.sh --changed --error` with a fake `opengrep` binary. The script prints `No changed first-party paths for opengrep.`
- Is this the best way to solve the issue? yes. The intended fix target is duplicate guide symlinks; applying it to all changed symlinks widens the exception beyond the failure mode. Resolving or narrowly recognizing guide aliases preserves the CI fix without creating a scanner blind spot.
