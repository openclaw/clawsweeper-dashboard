# Commit 336303e

## Summary

Found one low-severity uninstall cleanup regression in the npm-managed missing-directory path.

## Findings

### Low: Missing npm package directories skip package-manager cleanup

- Kind: reliability
- File: `src/plugins/uninstall.ts`
- Line: 568
- Evidence: The new early return exits `applyPluginUninstallDirectoryRemoval` when `fs.access(removal.target)` says the package directory is absent, before the existing `npm uninstall --prefix <npmRoot> <packageName>` cleanup runs. In the parent commit, npm cleanup ran even when `existed` was false. OpenClaw npm plugin installs use `npm install --prefix <npmRoot> <spec>`, which records the plugin in the managed npm root’s `package.json`/lockfile (`src/plugins/install.ts:1153`). The docs also describe npm uninstall as the mechanism that prunes hoisted npm-managed dependencies (`docs/plugins/dependency-resolution.md:49`).
- Impact: If a user has an install record for an npm-managed plugin but its package directory is already missing, `openclaw plugins uninstall <id>` now removes the OpenClaw install record but leaves the npm root dependency entry and lockfile state behind. A later npm plugin install can rehydrate the supposedly uninstalled package into `~/.openclaw/npm/node_modules`, and stale hoisted dependencies are not pruned.
- Suggested fix: For `removal.cleanup?.kind === "npm"`, run `npm uninstall` before returning for a missing target. Then skip only the `fs.rm` call when `existed` is false.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. A focused local npm-prefix smoke showed `npm uninstall --prefix <root> demo-plugin` removes the dependency from `package.json` even after `node_modules/demo-plugin` has been deleted; if skipped, a later `npm install --prefix <root> other-plugin` restores `demo-plugin`.
- Is this the best way to solve the issue? yes. The old behavior already separated `directoryRemoved: false` from npm cleanup, so keeping npm cleanup before the missing-target return preserves idempotent file deletion without leaving package-manager state stale.
