# Commit 84ec0c27

## Summary

Found three actionable regressions that still exist on current `main`: one config redaction risk from the schema response TTL cache, one sidebar drawer rendering bug, and one generated-locale regression.

## Findings

### Medium: Config schema cache can redact new snapshots with stale sensitivity hints

- Kind: security
- File: `src/gateway/server-methods/config.ts`
- Line: 281
- Evidence: `loadSchemaWithPlugins()` returns a cached `ConfigSchemaResponse` for five seconds based only on time. `config.get` then reads the current config snapshot and redacts it with that cached `schema.uiHints` at lines 298-300. The cache is cleared only by `config.set`, `config.patch`, and `config.apply`, so gateway reloads, direct file edits, or plugin metadata changes can leave `config.get` applying old hints to a newer snapshot.
- Impact: Plugin schema hints are part of the sensitive-field contract. If a newly active plugin/config field is marked `sensitive: true` but is not caught by the fallback key-name regex, an authenticated Control UI `config.get` can expose it until the TTL expires.
- Suggested fix: Key the schema cache to runtime config and plugin metadata fingerprints, clear it from the same reload/write listener path that updates runtime config/plugin metadata, or remove this cache.
- Confidence: medium

### Low: Sidebar recent-session shortcuts stay collapsed inside an expanded drawer

- Kind: bug
- File: `ui/src/ui/app-render.ts`
- Line: 267
- Evidence: `renderApp()` computes drawer-aware `navCollapsed = state.settings.navCollapsed && !navDrawerOpen` at lines 915-916, but `renderSidebarSessions()` reads `state.settings.navCollapsed` directly. The helper is called without the computed value at line 1762.
- Impact: With a saved collapsed sidebar, opening the responsive drawer expands the main nav but still hides the New session label and recent sessions. That breaks the new shortcut UI in the exact mobile/drawer path it should support.
- Suggested fix: Pass the computed `navCollapsed` into `renderSidebarSessions()` and use that rendered state for labels/recent-session visibility.
- Confidence: high

### Low: New non-English Cron strings are marked translated but shipped in English

- Kind: regression
- File: `ui/src/i18n/locales/de.ts`
- Line: 1221
- Evidence: The generated German bundle adds `emptyTitle`, `emptyHint`, `emptyFilteredHint`, and `advancedJob` as English strings while `ui/src/i18n/.i18n/de.meta.json` has `fallbackKeys: []` and `translatedKeys: 1117`. The same English additions appear across the generated non-English locale bundles.
- Impact: Non-English Control UI users see English copy in the Cron empty state and advanced creation label, and the metadata incorrectly reports full translation coverage.
- Suggested fix: Regenerate or repair the locale bundles so these keys are translated, or mark them as explicit fallback keys instead of counted translated output.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. The sidebar bug is reproducible by setting `settings.navCollapsed = true`, opening the drawer, and observing that `renderSidebarSessions()` still treats it as collapsed. The locale issue is directly visible in the generated bundles. The schema-cache issue is source-evident; a focused repro would warm `config.schema`, change runtime/plugin schema inputs outside these handlers, then call `config.get` within five seconds.
- Is this the best way to solve the issue? yes for the sidebar and locale fixes. For the schema cache, the safest fix is either real invalidation/fingerprinting or removing the cache because redaction depends on the freshness of schema hints.
