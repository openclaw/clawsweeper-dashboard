## Summary

Found one low-severity live-test coverage regression: the default prioritized live-model path now bypasses plugin catalog discovery, so `minimax-portal/minimax-m2.7` remains in the priority list but is never loaded by the new direct pi-ai registry path.

## Findings

### Low: Default priority live sweep silently drops MiniMax Portal

- Kind: regression
- File: `src/agents/models.profiles.live.test.ts`
- Line: 122
- Evidence: `useDefaultPriorityOnly` now calls `loadPrioritizedHighSignalModels()` instead of `discoverAuthStorage()` + `discoverModels()`. That helper only loads providers returned by `@mariozechner/pi-ai` `getProviders()`, but `src/agents/live-model-filter.ts:31` still includes `minimax-portal/minimax-m2.7` and `extensions/minimax/provider-registration.ts:272` registers `minimax-portal` through the OpenClaw plugin catalog. A focused dependency check showed 19 priority refs, with `minimax-portal/minimax-m2.7` as the only ref whose provider is absent from pi-ai.
- Impact: `OPENCLAW_LIVE_MODELS=modern` with no provider filter no longer exercises the MiniMax Portal route, even when MiniMax Portal credentials exist. That can let regressions in the OAuth/portal transport path pass the default live sweep.
- Suggested fix: keep the fast direct pi-ai path for pi-ai-owned providers, but merge in prioritized refs whose provider is plugin-owned by resolving just those provider catalogs, or special-case unknown priority providers through the existing `discoverModels()` path.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. The direct registry check showed `minimax-portal/minimax-m2.7` is the only priority ref whose provider is missing from `@mariozechner/pi-ai`; the live test branch at `src/agents/models.profiles.live.test.ts:808` uses only that direct loader for the default modern sweep.
- Is this the best way to solve the issue? unclear. The intended performance win is valid, but the safer direction is a hybrid loader that preserves plugin-owned priority refs without returning to full model discovery for every default run.
