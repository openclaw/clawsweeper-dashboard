# Commit 098b729

## Summary

Found one actionable Control UI regression in the new agent-scoped chat session picker. The issue is still present on current `main`.

## Findings

### Medium: Agent filter can switch into hidden cron sessions instead of the agent main chat

- Kind: regression
- File: `ui/src/ui/chat/session-controls.ts`
- Line: 565
- Evidence: `resolvePreferredSessionForAgent` picks the most recently updated row tied to the selected agent before applying the same visibility rules used by `resolveSessionOptionGroups`. Cron sessions are hidden later when `sessionsHideCron` is true unless they are already the active `sessionKey` (`ui/src/ui/chat/session-controls.ts:663`). A focused DOM check with `sessionsHideCron: true`, current session `agent:alpha:main`, and only one beta row `agent:beta:cron:nightly-check` produced `switch-call agent:beta:cron:nightly-check`.
- Impact: selecting an agent in the chat header can land the user in a hidden cron transcript rather than `agent:<id>:main`, despite the documented fallback behavior. Follow-up chat input can then target the cron session, and the hidden session becomes visible only because it was selected.
- Suggested fix: make `resolvePreferredSessionForAgent` filter out hidden/non-chat rows before sorting, at least cron rows when `sessionsHideCron` is true and `global`/`unknown` rows, then fall back to `buildAgentMainSessionKey({ agentId })` when no visible chat/dashboard/main row exists.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Render `renderChatSessionSelect` with `agent:alpha:main` active, `sessionsHideCron: true`, agents `alpha` and `beta`, and a sessions list containing `agent:alpha:main` plus `agent:beta:cron:nightly-check`; changing the agent select to `beta` calls the switch handler with the cron key.
- Is this the best way to solve the issue? yes. The selection helper should use the same eligibility rules as the rendered session options, so the agent filter cannot choose a row the UI otherwise hides.
