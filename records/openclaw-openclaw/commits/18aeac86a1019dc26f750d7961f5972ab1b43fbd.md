# Commit 18aeac8

## Summary

Found one regression in include-backed configs: an unrelated `openclaw config set` can now fail if `agents` is supplied through `$include` and contains a legacy normalizable Gemini model ref.

## Findings

### Medium: Unrelated config writes fail for included agent configs with legacy Gemini refs

- Kind: regression
- File: `src/cli/config-cli.ts`
- Line: 1472
- Evidence: `runConfigOperations` now calls `normalizeConfigMutationModelRefs(next as OpenClawConfig)` after every mutation, not only when the mutation touches agent model config. That helper rewrites `agents.defaults.model` and `agents.defaults.models` at lines 117-141. The config writer intentionally rejects writes whose generated patch touches an include-owned subtree, via `src/config/io.write-prepare.ts:267`.
- Impact: A user with root config like `agents: { $include: "./config/agents.json" }` can no longer run an unrelated command such as `openclaw config set gateway.port 18789 --strict-json` if the included agents file contains a legacy but normalizable ref like `google/gemini-3.1-flash-preview`. The command now attempts to normalize `agents` while writing `gateway`, so the writer rejects it with `Config write would flatten $include-owned config at agents`.
- Suggested fix: Do not run whole-config agent model normalization for every config mutation. Normalize only when an operation touches `agents.defaults.model` or `agents.defaults.models`, or make the normalizer include-aware so untouched include-owned subtrees are preserved before patch calculation.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. With `agents` included from `./config/agents.json`, put `google/gemini-3.1-flash-preview` in `agents.defaults.model.fallbacks` and `agents.defaults.models`, then run `OPENCLAW_HOME=<temp> pnpm openclaw config set gateway.port 18789 --strict-json`. Current `main` fails with `Config write would flatten $include-owned config at agents`.
- Is this the best way to solve the issue? yes. The failure comes from broad normalization creating an unintended `agents` diff; narrowing normalization to touched paths preserves the intended include protection while keeping the Gemini fix for actual model mutations.
