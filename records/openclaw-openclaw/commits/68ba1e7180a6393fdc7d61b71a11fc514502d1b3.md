# Commit 68ba1e7

## Summary

Found one regression in the new tmux wrapper: current invocation environment is only partially forwarded into the tmux pane, so several documented and runtime-supported Gateway environment controls stop working reliably once a tmux server/session already exists.

## Findings

### Medium: tmux watch mode drops current Gateway env overrides

- Kind: regression
- File: `scripts/gateway-watch-tmux.mjs`
- Line: 11
- Evidence: `TMUX_CHILD_ENV_KEYS` forwards only `NODE_OPTIONS`, config/state/home/profile/port, and `OPENCLAW_SKIP_CHANNELS`, then `buildGatewayWatchTmuxCommand` injects only those keys into the pane command. Env-backed Gateway auth and runtime controls are read elsewhere, for example `OPENCLAW_GATEWAY_TOKEN` in `src/gateway/startup-auth.ts:108`, `OPENCLAW_SKIP_CANVAS_HOST` in `src/gateway/server-runtime-config.ts:126`, `OPENCLAW_SKIP_CRON` in `src/gateway/server-cron.ts:92`, and `OPENCLAW_SKIP_PROVIDERS` in `src/gateway/server-startup-post-attach.ts:395`. A focused command-construction check showed `OPENCLAW_GATEWAY_TOKEN`, `OPENCLAW_SKIP_PROVIDERS`, `OPENCLAW_SKIP_CRON`, and `OPENCLAW_SKIP_CANVAS_HOST` are absent from the tmux command, while a local tmux 3.4 server check showed those OpenClaw vars are not automatically refreshed from a later client invocation.
- Impact: Existing workflows such as `OPENCLAW_GATEWAY_TOKEN=... pnpm gateway:watch`, `OPENCLAW_RAW_STREAM=1 pnpm gateway:watch`, or `OPENCLAW_SKIP_PROVIDERS=1 pnpm gateway:watch` can start/restart the tmux watcher with stale or missing env. That can make the Gateway fail auth startup, miss raw-stream logging requested by docs, or start providers/channels/cron/canvas that the caller explicitly tried to disable.
- Suggested fix: Preserve the old foreground watcher env contract for tmux mode. Prefer passing a reviewed set of Gateway env controls through a tmux-safe mechanism that does not expose secrets in the shell command text; at minimum include the non-secret skip/debug controls and either support env auth safely or document that env auth requires `gateway:watch:raw`/CLI flags.
- Confidence: high
