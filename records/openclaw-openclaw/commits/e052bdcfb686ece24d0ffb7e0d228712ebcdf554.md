## Summary

Found one regression in the new GPT-5.5 release-gate timeout setup. The touched files are unchanged on current `main`, so this still applies.

## Findings

### Medium: OpenAI cross-OS lanes fail when setting the new provider timeout

- Kind: regression
- File: `scripts/openclaw-cross-os-release-checks.ts`
- Line: 2643
- Evidence: The commit adds `timeoutSeconds: 600` for the OpenAI provider, then `runModelsSet` and `runInstalledModelsSet` call `openclaw config set models.providers.openai.timeoutSeconds 600 --strict-json`. The release onboard path for `--auth-choice openai-api-key` does not create `models.providers.openai`; it writes the default model, plugin enablement, and auth profile only. In an isolated temp `OPENCLAW_HOME`, `onboard --non-interactive --auth-choice openai-api-key --secret-input-mode ref ... --skip-health` succeeded, then the new timeout command exited 1 with `Config validation failed: models.providers.openai.baseUrl: Invalid input: expected string, received undefined`.
- Impact: Every OpenAI cross-OS release lane reaches `models-set` before gateway startup or the agent turn, then aborts on config validation instead of stabilizing GPT-5.5.
- Suggested fix: Do not write a partial `models.providers.<id>` entry. Either add/use a supported config API for provider timeout overrides that does not require a full provider catalog entry, or seed a complete provider-owned config shape before setting the timeout. Add a focused test for `onboard openai-api-key -> set release timeout`.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes: with a temp config, run non-interactive OpenAI onboard using an env ref, then run `pnpm openclaw config set models.providers.openai.timeoutSeconds 600 --strict-json`; it fails with the validation error above.
- Is this the best way to solve the issue? unclear: the safest direction is to avoid partial `models.providers` writes. The exact fix depends on whether provider request timeout should be modeled as a provider override, model override, or release-script-only knob.
