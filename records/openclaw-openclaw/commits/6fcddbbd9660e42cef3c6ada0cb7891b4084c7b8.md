# Commit 6fcddbbd96

## Summary

Found two auth-profile regressions around inherited agent credentials and stale OAuth drift handling.

## Findings

### Medium: Save-time filtering drops valid per-agent state for inherited profiles

- Kind: regression
- File: `src/agents/auth-profiles/store.ts`
- Line: 233
- Evidence: `buildLocalAuthProfileStoreForSave` now filters `order`, `lastGood`, and `usageStats` to only IDs left in `localStore.profiles`. `modelsAuthOrderSetCommand` validates requested IDs against the merged store, so an inherited main profile is accepted, then `setAuthProfileOrder` saves only local state. In a temp-dir repro, an inherited `openai-codex:default` profile was visible, `setAuthProfileOrder` returned `{"openai-codex":["openai-codex:default"]}`, but no `auth-state.json` was written and the next load returned `order: undefined`.
- Impact: `openclaw models auth order set --agent <agent> ...` can report success but lose the override for profiles inherited from the main agent. The same filter can discard per-agent `lastGood` and cooldown/usage state for inherited OAuth profiles, so agent-specific routing and backoff state does not persist.
- Suggested fix: Do not filter auth-state entries against only locally persisted secrets. Either keep state references that resolve in the effective merged store, or split secret filtering from state persistence so inherited profile IDs remain valid in `auth-state.json`.
- Confidence: high
