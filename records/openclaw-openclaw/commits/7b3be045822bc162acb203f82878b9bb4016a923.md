# Commit 7b3be04582

## Summary

Found one low-severity regression in Codex trajectory recording. The runtime turn itself and the prompt accounting paths use the new combined developer-instruction rendering, but the trajectory `context.compiled` event still records only the inherited thread-level developer instructions.

## Findings

### Low: Codex trajectory context omits turn-scoped workspace instructions

- Kind: bug
- File: `extensions/codex/src/app-server/run-attempt.ts`
- Line: 1795
- Evidence: `recordCodexTrajectoryContext(...)` still receives `developerInstructions: promptBuild.developerInstructions`, which is now only the inherited thread developer-instruction layer. In the same commit, the actual `turn/start` request adds `workspaceBootstrapContext.turnScopedDeveloperInstructions` through `collaborationMode.settings.developer_instructions`, and other accounting/reporting paths were updated to use `buildRenderedCodexDeveloperInstructions()`.
- Impact: trajectory exports no longer reconstruct the full model-bound developer context for Codex turns with `SOUL.md`, `IDENTITY.md`, or `USER.md`. A trajectory user investigating a run would see `context.compiled.data.systemPrompt` missing the turn-scoped persona/user instructions that Codex actually received.
- Suggested fix: pass `buildRenderedCodexDeveloperInstructions()` into `recordCodexTrajectoryContext(...)`, matching `systemPromptReport`, `createCodexTrajectoryRecorder(...)`, and the LLM input hook.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? Yes. Enable `OPENCLAW_TRAJECTORY=1`, run a Codex app-server attempt from a workspace containing `SOUL.md`/`IDENTITY.md`/`USER.md`, and compare `turn/start.params.collaborationMode.settings.developer_instructions` with the trajectory `context.compiled.data.systemPrompt`; the latter follows line 1795 and omits those turn-scoped files.
- Is this the best way to solve the issue? Yes. The commit already introduced `buildRenderedCodexDeveloperInstructions()` as the canonical combined rendering; using it for the trajectory context keeps the recorder aligned without changing runtime prompt delivery.
