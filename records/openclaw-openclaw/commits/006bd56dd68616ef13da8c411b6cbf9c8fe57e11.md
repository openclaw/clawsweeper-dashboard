# Commit 006bd56d

## Summary

Found one user-facing install-path bug: the new `@openclaw/acpx` trusted-package coverage is not reachable for the documented bare npm install because current npm `latest` resolves to a prerelease and the prerelease guard returns before the trust/scan path runs.

## Findings

### Medium: `@openclaw/acpx` documented install still fails before the new trust path

- Kind: bug
- File: `src/plugins/install.npm-spec.test.ts`
- Line: 347
- Evidence: This commit adds `@openclaw/acpx` to the trusted official npm package map in `src/plugins/install.ts:119` and adds a passing test case for bare `spec: "@openclaw/acpx"` at `src/plugins/install.npm-spec.test.ts:347`, but the test mocks the registry version as stable `2026.5.2` at `src/plugins/install.npm-spec.test.ts:374`. Live npm metadata currently reports `@openclaw/acpx` `latest` and `beta` as `2026.5.2-beta.2`. `installPluginFromNpmSpec({ spec: "@openclaw/acpx", dryRun: true })` returns: `Resolved @openclaw/acpx to prerelease version 2026.5.2-beta.2, but prereleases are only installed when explicitly requested.`
- Impact: The documented packaged ACP setup command `openclaw plugins install @openclaw/acpx` in `docs/tools/acp-agents-setup.md:133` and `docs/tools/acp-agents.md:45` fails before install, scan, or the new trusted-package logic. The scoped package path also does not get bundled fallback because `src/cli/plugin-install-plan.ts:73` only falls back for `NPM_PACKAGE_NOT_FOUND`, not prerelease rejection.
- Suggested fix: Either publish/promote a stable `@openclaw/acpx` version to `latest`, change the docs/install hints to use `@openclaw/acpx@beta` while only a beta tag exists, or teach the official launch-package path to request the beta tag when the package has no stable latest and that is intended for launch.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Run `pnpm exec tsx -e '...'` against `installPluginFromNpmSpec({ spec: "@openclaw/acpx", dryRun: true })`, or follow the documented `openclaw plugins install @openclaw/acpx` path in a packaged setup while npm `latest` is `2026.5.2-beta.2`.
- Is this the best way to solve the issue? unclear. If `@openclaw/acpx` should be stable for this launch, promoting/publishing stable npm metadata is the least invasive fix. If the beta channel is intentional, docs and install hints should opt into `@beta` explicitly so the existing prerelease guard remains intact.
