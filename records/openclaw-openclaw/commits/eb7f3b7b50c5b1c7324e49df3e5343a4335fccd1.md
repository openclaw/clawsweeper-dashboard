# Commit eb7f3b7

## Summary

Found one actionable regression/security-boundary issue: explicit `agent:<id>:` session keys can select an undeclared agent id without passing `--agent`, bypassing the known-agent validation that the CLI applies to `--agent`.

## Findings

### Medium: `--session-key agent:<id>:` bypasses configured-agent validation

- Kind: security
- File: `src/commands/agent-via-gateway.ts`
- Line: 139
- Evidence: `validateExplicitSessionKeyForDispatch` returns early when no `--agent` is supplied, even if `--session-key` is agent-prefixed. The embedded runner then derives the runtime agent from the session key in `src/agents/agent-command.ts:425-430`, and `resolveSessionAgentId` accepts that parsed id in `src/agents/agent-scope.ts:291-295`. A focused source check with config `{agents:{list:[{id:"main"}]}}` resolved `agent:ghost:incident-42` to `sessionAgent: "ghost"` while known agents were only `["main"]`.
- Impact: `openclaw agent --session-key agent:ghost:incident-42 ...` can create/use `ghost` agent state and workspace paths even though `openclaw agent --agent ghost ...` is rejected. That bypasses the configured agent allowlist and can also bypass per-agent policy by running as an implicit undeclared agent that only gets defaults.
- Suggested fix: when an explicit session key has shape `agent`, validate its parsed agent id against `listAgentIds(cfg)` even when `--agent` is absent. Apply the same validation in the shared embedded/gateway ingress path, or explicitly require an existing stored session if deleted-agent session resurrection is intended.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes: inspect `src/commands/agent-via-gateway.ts:124-149` and `src/agents/agent-command.ts:346-383`; then evaluate `resolveSessionAgentId({ config: { agents: { list: [{ id: "main" }] } }, sessionKey: "agent:ghost:incident-42" })`, which returns `ghost`.
- Is this the best way to solve the issue? yes: the existing `--agent` guard already establishes the contract that CLI-selected agents must be configured; explicit session keys should not be a second path around that guard.
