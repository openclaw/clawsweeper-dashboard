One low-severity test-gap finding. The commit is test-only and the touched test passes locally, but the new deterministic gate can release before the competing async call has had an I/O turn to reach the write lock, so it may not reliably prove the concurrency regression it is meant to guard.

## Findings

### Low: write serialization test can release before the second call reaches the lock

- Kind: test_gap
- File: `src/agents/models-config.write-serialization.test.ts`
- Line: 293
- Evidence: The test waits for the first `models.json` write to start, yields only `await Promise.resolve()`, then calls `releaseModelsWrites()`. The production path computes the fingerprint with async `fs.stat` work before `withModelsJsonWriteLock` is entered in `src/agents/models-config.ts:195` and `src/agents/models-config.ts:219`, so the second `ensureOpenClawModelsJson(...)` call can still be in pre-lock async work when the first write is released.
- Impact: If the per-path write lock regresses or is removed, this test can still pass in schedules where the second call has not reached `writeText` before the first write is released. That weakens the release gate for the concrete bug this test is meant to catch: overlapping writes to the same `models.json`.
- Suggested fix: Hold the first write through at least a real event-loop turn, not just a microtask, before releasing, and assert no second write starts while the first is gated. A more robust option is to add a test-only signal around lock acquisition/queueing so the test waits until the second call is actually contending for the same target path before releasing the first write.
- Confidence: medium

## Details

- Do we have a high-confidence way to reproduce the issue? No. The touched test passes as-is; this is a schedule-dependent gap derived from the async call path and the new `Promise.resolve()` gate.
- Is this the best way to solve the issue? Unclear. A timer or `setImmediate` would improve the current test, but an explicit test hook/signal around the lock queue would be more deterministic.
