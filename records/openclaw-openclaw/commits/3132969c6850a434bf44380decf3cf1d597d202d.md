# Commit 3132969c68

## Summary

The commit adds npm fallback when official ClawHub artifact installation fails. One fallback path is too broad and remains present on current `main`: ClawHub download blocks, including security/moderation 403s, are converted into npm installs.

## Findings

### High: ClawHub artifact blocks can be bypassed through npm fallback

- Kind: security
- File: `src/plugins/clawhub.ts`
- Line: 1157
- Evidence: `downloadClawHubPackageArchive` throws `ClawHubRequestError` for any non-OK `/artifact/download` response, including 401, 403, 429, and 5xx (`src/infra/clawhub.ts:920`). This commit maps every such request error that is not the special 404 case to `ARTIFACT_UNAVAILABLE` (`src/plugins/clawhub.ts:1152`), then treats that code as eligible for npm fallback (`src/plugins/update.ts:439`, `src/commands/onboarding-plugin-install.ts:72`, `src/commands/doctor/shared/missing-configured-plugin-install.ts:90`). The added update test explicitly models a 403 body saying the release was “flagged as malicious” and expects fallback to `@openclaw/discord@beta` (`src/plugins/update.test.ts:2296`).
- Impact: Official plugin update, onboarding, doctor repair, and externalized bundled-plugin sync paths can install from npm after ClawHub rejects an artifact download. That bypasses ClawHub’s auth, rate-limit, outage, moderation, or security decision instead of failing closed.
- Suggested fix: Only emit a fallback-eligible code for explicit benign artifact rollout gaps, such as missing artifact metadata or a well-defined artifact-missing response. Preserve fatal behavior for 401/403/429/5xx and add regression coverage proving malicious/security-blocked 403s do not fall back to npm.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. The committed test case already reproduces the unsafe policy by asserting that a 403 “flagged as malicious” artifact block falls back to npm; the focused test passes.
- Is this the best way to solve the issue? yes. The narrow fix is to split benign artifact-unavailable states from request failures and fail closed on server/security/auth/throttle responses while preserving the intended launch fallback.
