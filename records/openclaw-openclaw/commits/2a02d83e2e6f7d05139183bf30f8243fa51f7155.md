## Findings

### Low: `PreflightFacts.media` was widened to a lazy function union on the public adapter path

- Kind: compatibility
- File: `src/channels/turn/types.ts`
- Line: 240
- Evidence: `PreflightFacts.media` changed from `InboundMediaFacts[]` to `readonly InboundMediaFacts[] | (() => MaybePromise<...>)`, while `ChannelTurnAdapter.resolveTurn` receives `preflight: PreflightFacts` and existing context assembly still accepts only `media?: InboundMediaFacts[]` at `src/channels/turn/context.ts:39`.
- Impact: third-party channel plugins that read `preflight.media` in `resolveTurn` and pass it to `buildChannelTurnContext({ media: preflight.media })` now see a function union and can fail TypeScript compilation after upgrading, even if their own `preflight` implementation only ever returns an array. This is a public SDK surface through `runInboundReplyTurn`.
- Suggested fix: keep `PreflightFacts.media` array-only and put the lazy dropped-history media resolver on the new history-specific options, for example `history.media` or `history.resolveMedia`, so dispatched-turn media consumers keep the old contract.
- Confidence: medium

## Details

- Do we have a high-confidence way to reproduce the issue? Yes: compile a channel adapter that consumes the `resolveTurn(..., preflight)` parameter and passes `preflight.media` into `buildChannelTurnContext`’s `media` field.
- Is this the best way to solve the issue? Yes: separating dropped-history lazy media from normal preflight media preserves the old public type while keeping the new behavior.
