# Commit e310824

## Summary

Found one low-severity `/codex account` regression that still exists on current `main`: when a ChatGPT subscription profile is the active credential and its Codex usage is exhausted, the new auth overview replaces the old “Codex is paused” summary with an “active now” ladder entry.

## Findings

### Low: Active subscription limit is shown as active instead of paused

- Kind: regression
- File: `extensions/codex/src/command-account.ts`
- Line: 305
- Evidence: `formatAccount()` now returns the auth overview whenever `readCodexAccountAuthOverview()` finds auth order data, so the previous `summarizeCodexAccountRateLimits()` “Codex is paused...” path is bypassed. In the overview path, `buildProfileRow()` sets any selected profile to `active now` before considering `params.usage?.blocked`, and `formatSubscriptionUsageLine()` only appends usage/reset text. A focused smoke with one active `openai-codex` OAuth profile plus a `rateLimitReachedType: "rate_limit_reached"` payload printed:
  ```text
  Subscription  personal-email@gmail.com
    Weekly 100% · Short-term 0% · Resets in 24 hours

  Auth order
    1. personal-email@gmail.com   ChatGPT subscription   — active now
  ```
- Impact: users with no API-key fallback, or with the subscription still selected, no longer get the clear paused/rate-limited account message that existed before this commit. The output says the selected ChatGPT subscription is “active now” even though Codex is blocked until reset.
- Suggested fix: when `activeIsSubscription && activeUsage?.blocked`, surface the blocked state in the overview, either by marking the active row as `rate-limited`/`active but rate-limited` and adding the reset, or by carrying the existing `summarizeCodexAccountRateLimits()` paused text into the auth overview.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Seed an auth store with a single active `openai-codex` OAuth profile, have `/codex account` return a ChatGPT account, and have `rateLimits` return the Codex snapshot with weekly `usedPercent: 100` and `rateLimitReachedType: "rate_limit_reached"`.
- Is this the best way to solve the issue? yes. The overview is useful, but it needs to preserve the blocked-rate-limit contract when it replaces the older account formatter.
