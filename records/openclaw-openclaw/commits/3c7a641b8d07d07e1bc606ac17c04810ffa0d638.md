# Commit 3c7a641b

## Summary

Found one low-severity runtime regression in the production timezone fallback added to support the thread-pool test path. The memory extension lane itself passes under the new `threads` pool.

## Findings

### Low: Raw `TZ` fallback can make dream diary writes fail

- Kind: reliability
- File: `extensions/memory-core/src/dreaming-narrative.ts`
- Line: 325
- Evidence: `formatNarrativeDate()` now passes `timezone ?? process.env.TZ` as the explicit `Intl.DateTimeFormat` `timeZone`. When the user has no configured dreaming timezone, narrative callers pass `undefined` through `runLightDreaming`, `runRemDreaming`, and `generateAndAppendDreamNarrative`, so the raw host `TZ` is used. A focused Node probe showed the previous implicit-default formatter works for `TZ=""` and `TZ=":/etc/localtime"`, while the new explicit option throws `RangeError: Invalid time zone specified`. That exception reaches `appendNarrativeEntry()` and is caught as a best-effort narrative failure, so the generated dream diary entry is skipped.
- Impact: Users running OpenClaw with an unset-but-present empty `TZ`, or a colon/path-style `TZ`, can lose Dream Diary entries whenever no explicit `plugins.entries.memory-core.config.dreaming.timezone` is configured.
- Suggested fix: Only pass an explicit `timeZone` when the configured timezone or `process.env.TZ` is a non-empty valid `Intl` timezone. Otherwise omit the option and let `Intl` use the host default. Keep the thread-pool test by setting a valid IANA `TZ`.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Run the formatter shape with `TZ=:/etc/localtime` or `TZ=""` and no configured `timezone`; the new explicit `timeZone` throws, while the old implicit default formats successfully.
- Is this the best way to solve the issue? no. The desired behavior from issue `#65027` is host-local formatting when no user timezone is configured, but passing raw `process.env.TZ` is narrower and can reject host values that the implicit local default path tolerates.
