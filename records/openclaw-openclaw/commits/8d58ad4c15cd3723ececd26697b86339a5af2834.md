## Findings

### Low: Startup retry handshakes can flood warning logs

- Kind: reliability
- File: `src/gateway/server/ws-connection.ts`
- Line: 328
- Evidence: The new startup-pending path returns retryable `UNAVAILABLE` and closes before registering a client at `src/gateway/server/ws-connection/message-handler.ts:456`. The retry interval is 500ms at `src/gateway/protocol/startup-unavailable.ts:2`. Every one of those expected pre-connect closes reaches the generic `!client` close branch, which logs `closed before connect` with `logWsControl.warn` at `src/gateway/server/ws-connection.ts:328`. Issue #73652 describes a roughly 15-22s startup window, so each early Control UI/TUI/CLI client can produce dozens of warnings while doing exactly what the new protocol asks it to do.
- Impact: Normal startup retries may make gateway logs look unhealthy and bury real pre-auth failures during the startup window.
- Suggested fix: Treat `closeCause === "startup-sidecars-pending"` or close code `1013` as debug/info, or rate-limit this specific expected close reason.
- Confidence: high
