# Commit 233765b

## Summary

Found one low-severity regression in the packaged launcher fast path: `OPENCLAW_CONTAINER=<name> openclaw nodes --help` and `OPENCLAW_CONTAINER=<name> openclaw secrets --help` can be answered from host precomputed metadata before the runtime container routing code runs.

## Findings

### Low: Launcher command-help fast path bypasses `OPENCLAW_CONTAINER`

- Kind: regression
- File: `openclaw.mjs`
- Line: 456
- Evidence: `tryOutputPrecomputedCommandHelp()` now handles `secrets` and `nodes` directly from `dist/cli-startup-metadata.json`, but the launcher’s defer check only considers plugin-related env/config at `openclaw.mjs:398`; it does not consider `OPENCLAW_CONTAINER`. The runtime entry and run-main paths do handle this: `src/entry.ts:242` skips precomputed command help when a container target exists, and `src/cli/run-main.ts:445` routes through `maybeRunCliInContainer()`, with `OPENCLAW_CONTAINER` recognized by `src/cli/container-target.ts:55`.
- Impact: In a packaged install with `OPENCLAW_CONTAINER=demo`, `openclaw nodes --help` can silently print host/bundled precomputed help instead of routing into the target container. That is most visible for `nodes --help`, because plugin config can add or remove `nodes` subcommands; it also means a missing/stopped container is not reported for these new help invocations.
- Suggested fix: Make the launcher’s command-help fast path defer when `process.env.OPENCLAW_CONTAINER` is set, or share a tiny container-target check with the launcher before reading precomputed command metadata. Add launcher e2e coverage for `OPENCLAW_CONTAINER=demo openclaw nodes --help`.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. A temporary packaged-launcher fixture with `dist/cli-startup-metadata.json` containing `nodesHelpText`/`secretsHelpText` and `dist/entry.js` printing `RUNTIME ENTRY` emitted the precomputed help for `NODE_DISABLE_COMPILE_CACHE=1 OPENCLAW_CONTAINER=demo node openclaw.mjs nodes --help` and `... secrets --help`, proving the runtime entry was bypassed.
- Is this the best way to solve the issue? yes. The lower runtime layers already treat `OPENCLAW_CONTAINER` as a routing target; the packaged launcher should preserve that contract before taking any host-only precomputed help shortcut.
