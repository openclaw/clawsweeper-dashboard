# Commit a31b75f

## Summary

Found one concrete regression in the macOS deep-link delivery path: removing the `bluebubbles` enum case makes stale explicit BlueBubbles deep links silently fall back to `last`, which can route a keyed unattended delivery to an unintended previous channel instead of failing closed.

## Findings

### Medium: Retired BlueBubbles deep links now deliver to `last` instead of being rejected

- Kind: privacy
- File: `apps/macos/Sources/OpenClaw/GatewayConnection.swift`
- Line: 21
- Evidence: This commit removes `case bluebubbles`, but `GatewayAgentChannel(raw:)` still maps any unknown non-empty channel to `.last`. `DeepLinkAgentPolicy.effectiveDelivery` uses that initializer for keyed unattended deep links, then calls `shouldDeliver`; `.last` is deliverable. The gateway then receives `channel: "last"` from `sendAgent`, so a stale `channel=bluebubbles&deliver=true` deep link no longer reaches gateway-side unknown-channel rejection and can resolve against the session’s previous delivery route.
- Impact: An old macOS automation/deep link intended for the retired BlueBubbles route can send the prompt to the last chat/channel instead of failing. Because unattended delivery requires the deep-link key, this is not an unauthenticated path, but it is still a plausible wrong-recipient delivery regression for stale configured automations.
- Suggested fix: Fail closed for explicit retired/unknown deep-link channels. A narrow fix is to map `bluebubbles` to a non-deliverable value in the macOS client or make `effectiveDelivery` set `deliver = false` when `link.channel` is non-empty and not a known deliverable channel.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes: with a valid macOS deep-link key, open an `agent` deep link containing `channel=bluebubbles`, `deliver=true`, and any `to`; source inspection shows the app serializes it as `channel: "last"` after this commit.
- Is this the best way to solve the issue? yes: explicit retired channel names should fail closed at the client boundary rather than being normalized into implicit last-route delivery.
