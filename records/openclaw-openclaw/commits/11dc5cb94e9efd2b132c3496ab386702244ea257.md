# Commit 11dc5cb

## Summary

Found one low-severity Discord scrubber regression: compact command lifecycle lines now render without `Exec:`, but the final-send scrubber does not recognize status-only compact lines.

## Findings

### Low: Compact command-output status lines can leak into final Discord replies

- Kind: regression
- File: `extensions/discord/src/monitor/reply-safety.ts`
- Line: 7
- Evidence: This commit makes command-output progress render as compact status-only lines, e.g. `formatChannelProgressDraftLine({ event: "command-output", name: "exec", phase: "end", exitCode: 0 })` now returns `🛠️ completed`, and exit failures return `🛠️ exit 1` from `src/plugin-sdk/channel-streaming.ts:404`. The new Discord compact trace regex only matches backticked commands or command verbs such as `run`, `git`, `pnpm`, etc.; it does not match `completed` or `exit 1`. A focused check confirmed `sanitizeDiscordFrontChannelText("🛠️ completed\nVisible reply.")` preserves the internal status line, while `🛠️ run git status` is stripped.
- Impact: Discord final replies can still expose internal tool-progress status rows after this compact-display change. The leaked content is usually generic, so severity is low, but it regresses the intended final-send trace scrubbing for command-output progress.
- Suggested fix: Extend the compact Discord trace predicate to cover compact command lifecycle/status lines produced by `command-output` (`completed`, `exit <code>`, and configured statuses), or keep a label on lifecycle-only command rows so the existing labeled trace regex continues to catch them. Add Discord scrubber tests for `🛠️ completed` and `🛠️ exit 1`.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes: run `pnpm exec tsx -e` importing `sanitizeDiscordFrontChannelText` and `formatChannelProgressDraftLine`; current output preserves `🛠️ completed` / `🛠️ exit 1` but strips `🛠️ run git status`.
- Is this the best way to solve the issue? yes, at the scrub boundary or by preserving a labeled lifecycle form; both keep Discord final-send sanitation aligned with the new compact progress format.
