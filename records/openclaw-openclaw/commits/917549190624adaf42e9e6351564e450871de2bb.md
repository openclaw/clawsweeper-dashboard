# Commit 9175491

## Summary

Found one reliability regression in the new cron parser path: plugin target parser exceptions can now escape before cron delivery target resolution reaches its structured error handling.

## Findings

### Medium: Plugin target parser exceptions can reject cron target resolution

- Kind: reliability
- File: `src/cron/isolated-agent/delivery-target.runtime.ts`
- Line: 18
- Evidence: `parseExplicitTargetForDelivery` calls `messaging.parseExplicitTarget` directly, and `resolveDeliveryTarget` passes that parser into `resolveSessionDeliveryTarget` before `resolveOutboundTargetWithRuntime` can catch target failures. Existing plugin parsers are not no-throw: `extensions/clickclack/src/target.ts` throws for unsupported prefixed targets, and `extensions/qa-channel/src/bus-client.ts` throws for malformed `thread:` targets. A focused probe confirmed `parseExplicitTargetForDelivery({ channel: "clickclack", rawTarget: "workspace:general" })` throws `Unsupported ClickClack target: workspace:general`.
- Impact: A malformed cron `delivery.to` for a bootstrapped channel plugin can reject target resolution instead of returning a controlled `{ ok: false, error: Invalid delivery target... }` result. This is a user-visible availability regression for cron announce delivery and any caller expecting `resolveDeliveryTarget` to resolve rather than throw.
- Suggested fix: Wrap the injected parser path used by cron, preferably around the `resolveSessionDeliveryTarget` calls in `resolveDeliveryTarget`, and convert parser failures into the same structured invalid-target result used by outbound target resolution. Add a regression test with a throwing parser.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes; call the new parser seam with ClickClack and an unsupported prefixed target, or inject a throwing parser into `resolveSessionDeliveryTarget`.
- Is this the best way to solve the issue? yes; keeping plugin-owned parsing is correct, but cron should preserve structured delivery errors instead of allowing parser exceptions to escape.
