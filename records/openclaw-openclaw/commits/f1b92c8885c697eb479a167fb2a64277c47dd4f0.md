### High: Public R2 publishing includes raw Mantis evidence

- Kind: privacy

- Evidence: `publishArtifactFiles` iterates over `manifest.artifacts` and uploads each file to the public R2 prefix. The same function then writes `index.json` listing all artifact URLs. Current manifest producers include non-inline/raw artifacts: Telegram live adds `observed-messages.json`, transcript HTML, reports, error text, full videos, and metadata in `scripts/mantis/build-telegram-evidence.mjs:289`; Slack adds `slack-desktop-command.log`, report, error text, full video, and metadata in `.github/workflows/mantis-slack-desktop-smoke.yml:328`; Discord workflows add full videos, reports, comparison JSON, and preview metadata in `.github/workflows/mantis-discord-status-reactions.yml:493` and `.github/workflows/mantis-discord-thread-attachment.yml:514`. The docs still state that raw logs, observed messages, and other bulky evidence stay in the Actions artifact in `docs/concepts/mantis.md:637`.
- Impact: Mantis runs on public PRs can publish channel/user/message evidence and raw run artifacts to `https://artifacts.openclaw.ai/...` instead of keeping them behind the short-lived GitHub Actions artifact boundary. That widens exposure beyond the intended public evidence surface.
- Suggested fix: Make R2 publishing explicit and allowlisted. Upload only approved redacted public media, such as `inline: true` artifacts or a new `public: true` manifest field, and keep metadata, reports, observed messages, logs, transcript HTML, and full/raw videos only in the Actions artifact. The generated `index.json` should list only public artifacts.
- Confidence: high
