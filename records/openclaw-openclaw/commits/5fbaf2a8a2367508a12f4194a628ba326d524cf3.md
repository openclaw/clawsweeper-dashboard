# Commit 5fbaf2a

## Summary

Found one low-severity user-facing bug: WhatsApp approval prompts can advertise thumb reactions in configurations where the reaction resolver intentionally refuses to honor them.

## Findings

### Low: Reaction hint is shown even when reactions are not authorized

- Kind: bug
- File: `extensions/whatsapp/src/approval-handler.runtime.ts`
- Line: 101
- Evidence: Native prompt construction appends `React with:` whenever the approval’s decision set includes `allow-once` or `deny`, without checking whether the WhatsApp account has explicit approvers. The shared WhatsApp renderer does the same through `extensions/whatsapp/src/approval-native.ts:429`. But reaction handling rejects every reaction when `getWhatsAppApprovalApprovers(...)` is empty in `extensions/whatsapp/src/approval-reactions.ts:462`, and the added test at `extensions/whatsapp/src/approval-reactions.test.ts:349` confirms direct reactions with no explicit approvers are ignored.
- Impact: A valid session-mode WhatsApp approval can be delivered to a direct chat with no `allowFrom` configured, show `👍 Allow Once` / `👎 Deny`, and then silently ignore the user’s reaction. Manual `/approve` still works, but the new reaction affordance is broken in that common/default-ish configuration.
- Suggested fix: Only append/register WhatsApp reaction hints when explicit WhatsApp approvers are configured for that account, or make direct same-chat reactions follow the same implicit authorization model as manual `/approve` if that is the desired contract.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Current tests already demonstrate the mismatch: prompt/render paths add reaction hints unconditionally, while `approval-reactions.test.ts` verifies direct reactions without explicit approvers are handled but not resolved.
- Is this the best way to solve the issue? yes for the contract stated in the PR/docs: if reactions require explicit `allowFrom` or `"*"`, the prompt should not advertise reactions until that condition is true.
