# Commit af765100

## Summary

The latest-assistant preservation path is covered and the focused tests pass, but the new recovery behavior does not work for the real `StreamFn` contract used by the built-in Anthropic/Bedrock providers.

## Findings

### Medium: Thinking recovery does not retry real provider error streams

- Kind: reliability
- File: `src/agents/pi-embedded-runner/thinking.ts`
- Line: 427
- Evidence: `wrapAnthropicStreamWithRecovery` only retries rejected promises or thrown async-iterator errors. `@earendil-works/pi-agent-core` documents `StreamFn` failures as non-throwing error streams, and `pi-ai` Anthropic/Bedrock catch provider failures, push `{ type: "error", error: output }`, then end. A focused probe using `createAssistantMessageEventStream()` with `errorMessage: "ValidationException: invalid signature on thinking block in message history"` produced `{"calls":1,"stopReason":"error",...}`, so the retry never happened.
- Impact: The commit wires this wrapper into `preserveSignatures` runs to recover from opaque thinking-signature rejections, but built-in provider failures still end the turn as an error assistant. Sessions with invalid opaque/stale thinking signatures can remain wedged instead of retrying once without thinking replay.
- Suggested fix: In `pumpStreamWithRecovery`, detect a provider `error` event or final error assistant matching `shouldRecoverAnthropicThinkingError` before any visible assistant content is forwarded, suppress that error event, and retry with `stripAllThinkingBlocks`. Add a regression test that uses `createAssistantMessageEventStream()` rather than only `Promise.reject`.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. The inline `node --import tsx -e ...` probe against `wrapAnthropicStreamWithRecovery` simulated the real `AssistantMessageEventStream` error shape and showed only one provider call.
- Is this the best way to solve the issue? yes. The wrapper already owns retry policy; it just needs to honor the stream error contract it is wrapping.
