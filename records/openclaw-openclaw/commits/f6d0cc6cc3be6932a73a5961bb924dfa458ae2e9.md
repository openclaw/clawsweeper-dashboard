# Commit f6d0cc6c

## Summary

Found one low-severity CLI help regression in the metadata-missing path.

## Findings

### Low: Missing startup metadata leaves message and agent channel help blank or misleading

- Kind: regression
- File: `src/cli/channel-options.ts`
- Line: 39
- Evidence: The commit changes `resolveCliChannelOptions()` to return `[]` when `cli-startup-metadata.json` is absent. `createProgramContext()` then formats that empty list directly: `messageChannelOptions` becomes `""` and `agentChannelOptions` becomes `"last"` (`src/cli/program/context.ts:25`, `src/cli/program/context.ts:28`). Those strings are rendered into user-visible help as `Channel: ` for message commands and `Delivery channel: last ...` for agent commands (`src/cli/program/message/helpers.ts:114`, `src/cli/program/register.agent.ts:39`). This affects source/dev checkouts where `scripts/run-node.mjs` builds `dist` but does not invoke `scripts/write-cli-startup-metadata.ts`.
- Impact: In source/dev CLI help, `openclaw message ... --help` no longer shows a usable generic channel placeholder, and `openclaw agent --help` implies only `last` is valid even though direct channel ids are still accepted. The command behavior still works, so this is a low-severity help regression rather than a runtime break.
- Suggested fix: Keep `resolveCliChannelOptions()` lightweight, but make the help formatters fall back to generic text consistently. For example, have `createProgramContext()` expose `messageChannelOptions: "channel"` and `agentChannelOptions: "last|channel"` when the resolved list is empty; similarly consider whether `formatCliChannelOptions(["all"])` should render `all|channel`.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes: in a source checkout without `dist/cli-startup-metadata.json`, run the source launcher path and inspect `openclaw message send --help` or `openclaw agent --help`.
- Is this the best way to solve the issue? yes: the fix should stay in help-string formatting so the performance goal of avoiding channel runtime imports remains intact.
