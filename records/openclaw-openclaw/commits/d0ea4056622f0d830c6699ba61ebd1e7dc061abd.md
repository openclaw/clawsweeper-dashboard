### Low: Exact header assertion ignores the supported `OPENCLAW_VERSION` override

- Kind: reliability
- File: `src/plugin-sdk/provider-stream.test.ts`
- Line: 243
- Evidence: The new assertion expects `version: VERSION` and `User-Agent: openclaw/${VERSION}`. Production header resolution goes through the OpenAI attribution wrapper and runtime version resolution, where `OPENCLAW_VERSION` takes precedence over the package `VERSION`.
- Impact: `OPENCLAW_VERSION=2026.3.22 pnpm test src/plugin-sdk/provider-stream.test.ts` now fails, even though the runtime behavior is intentional and covered by `src/version.ts`. This can create false local or release-environment failures when that documented env override is present.
- Suggested fix: Build the expected value with the same runtime version resolver used by production, or explicitly clear/stub `OPENCLAW_VERSION` in this test before asserting package `VERSION`.
- Confidence: high
