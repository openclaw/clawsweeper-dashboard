## Findings

### Low: WhatsApp shared runtime mock ignores global `messages.visibleReplies: "automatic"` for groups

- Kind: test_gap
- File: `extensions/whatsapp/src/test-helpers.ts`
- Line: 273
- Evidence: the new helper returns `"automatic"` for group/channel only when `messages.groupChat.visibleReplies === "automatic"`. Production resolves `messages.groupChat.visibleReplies ?? messages.visibleReplies`, and the local `inbound-dispatch.test.ts` mock uses the same fallback. So `{ messages: { visibleReplies: "automatic" } }` with `ChatType: "group"` is `"automatic"` in production but `"message_tool_only"` in this shared helper.
- Impact: WhatsApp web auto-reply tests that import `extensions/whatsapp/src/test-helpers.ts` will simulate silent/tool-only group replies when the real runtime would visibly reply under the global automatic setting. That can mask regressions around the group visible-reply fallback fixed by the adjacent production contract.
- Suggested fix: mirror the production helper exactly for group/channel mode:
  `const configuredMode = params.cfg.messages?.groupChat?.visibleReplies ?? params.cfg.messages?.visibleReplies; return configuredMode === "automatic" ? "automatic" : "message_tool_only";`
- Confidence: high
