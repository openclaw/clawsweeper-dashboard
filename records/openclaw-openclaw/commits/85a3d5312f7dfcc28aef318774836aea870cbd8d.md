# Commit 85a3d531

## Summary

Found one actionable compatibility issue in the new npm freshness tests. The runtime code paths read consistently, but the added npm JSON assertion is version-sensitive and fails under a supported Node/npm combination used by this repo.

## Findings

### Medium: npm freshness test fails on Node 24.15.0 / npm 11.12.1

- Kind: compatibility
- File: `src/infra/npm-install-env.test.ts`
- Line: 375
- Evidence: `node scripts/run-vitest.mjs src/infra/npm-install-env.test.ts src/infra/update-global.test.ts src/infra/update-runner.test.ts src/infra/install-source-utils.test.ts src/infra/safe-package-install.test.ts` fails 1/108 tests. The failing test expects `npmConfig["min-release-age"]` to be `0` or `"0"`, but with Node `v24.15.0` / npm `11.12.1`, `npm config list --json` reports `before: <current-date>` and `"min-release-age": null` after npm flattens `min-release-age=0` into `before`. A probe with npm `11.14.1` reports `"min-release-age": 0`, which explains why the PR proof can pass on newer npm while failing on the pinned/local supported toolchain.
- Impact: Supported developer/CI environments using Node 24.15.0 can fail the focused infra test lane even though the behavior being tested is effectively bypassed. Release workflows in this repo also pin Node 24.15.0 in several places, so this is likely to become a check blocker.
- Suggested fix: Make the assertion tolerate npm’s older JSON encoding by accepting `null` when `before` has been flattened to the effective current bypass date, or assert the helper’s emitted env plus npm’s effective `before` behavior without requiring `"min-release-age"` to remain visible in `npm config list --json`.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes: run the focused wrapper test command above under Node `v24.15.0` / npm `11.12.1`.
- Is this the best way to solve the issue? yes: the product helper already sets `npm_config_min_release_age="0"`; the brittle part is the test’s assumption about npm’s JSON reporting shape across npm 11 patch versions.
