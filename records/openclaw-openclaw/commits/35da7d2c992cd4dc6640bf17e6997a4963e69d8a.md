Found one auth-profile regression that still matters on current `main`.

## Findings

### Medium: Implicit main auth store no longer honors `OPENCLAW_AGENT_DIR`

- Kind: regression
- File: `src/agents/auth-profiles/path-resolve.ts`
- Line: 13
- Evidence: `resolveAuthStorePath()` now falls back to `resolveDefaultAgentDir({})`, which ignores `OPENCLAW_AGENT_DIR` and `PI_CODING_AGENT_DIR`. Auth store merge paths still treat `resolveAuthStorePath()` with no `agentDir` as the canonical main store in `src/agents/auth-profiles/store.ts:436` and `src/agents/auth-profiles/store.ts:499`. Existing coverage proves the behavior is broken: `pnpm test src/agents/auth-profiles.ensureauthprofilestore.test.ts src/agents/auth-profiles.store.save.test.ts src/agents/auth-profiles.store-cache.test.ts` fails 6 tests in `src/agents/auth-profiles.ensureauthprofilestore.test.ts`, including the case where a secondary agent should inherit `anthropic:default` from the env-selected main auth dir.
- Impact: users and test harnesses that isolate the main agent with `OPENCLAW_AGENT_DIR` lose read-through auth inheritance for non-main agents. That can make provider auth appear missing, prevent OAuth fallback/repair from seeing the healthy main profile, and leave stale per-agent OAuth state in use.
- Suggested fix: restore env-aware behavior for the implicit main auth path, or thread the selected main agent dir/config explicitly through the auth-store helpers. The minimal compatibility direction is for the no-argument auth path fallback to keep the old `OPENCLAW_AGENT_DIR`/`PI_CODING_AGENT_DIR` override semantics.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes: run `pnpm test src/agents/auth-profiles.ensureauthprofilestore.test.ts src/agents/auth-profiles.store.save.test.ts src/agents/auth-profiles.store-cache.test.ts`.
- Is this the best way to solve the issue? yes for compatibility: the store merge code is built around “no `agentDir` means main store,” so that implicit main-store resolver needs to keep resolving the operator-selected main store.
