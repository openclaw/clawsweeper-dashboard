### Medium: Anthropic SDK bump is not applied to the workspace graph

- Kind: supply_chain

- Evidence: This commit changes root `package.json:1717` and `extensions/amazon-bedrock-mantle/package.json:8` to `@anthropic-ai/sdk` `0.95.2`, but current `pnpm-workspace.yaml:38` still overrides `@anthropic-ai/sdk` to `0.95.1`. The committed lockfile mirrors that override at `pnpm-lock.yaml:7` and records both the root importer and Bedrock Mantle importer as `specifier: 0.95.1` / `version: 0.95.1` at `pnpm-lock.yaml:47` and `pnpm-lock.yaml:357`. `corepack pnpm list @anthropic-ai/sdk --recursive --depth 0 --lockfile-only` confirms both root OpenClaw and `@openclaw/amazon-bedrock-mantle-provider` resolve to `@anthropic-ai/sdk@0.95.1`.
- Impact: source installs, CI, and local testing continue to run against `0.95.1` while package manifests and changelog say the workspace refreshed to `0.95.2`. That creates an untested release/source split and can leave any intended Anthropic SDK fix absent from the workspace graph.

- Confidence: high
