## Findings

### High: Focused release-check tests fail after the package-exclusion change

- Kind: regression
- File: `test/scripts/bundled-plugin-build-entries.test.ts`
- Line: 101
- Evidence: `listBundledPluginPackArtifacts()` now excludes any plugin directory matched by root `package.json` `files` exclusions in `scripts/lib/bundled-plugin-build-entries.mjs:169`, but the tests added in this commit still expect `dist/extensions/acpx/index.js`, `dist/extensions/googlechat/index.js`, and `dist/extensions/line/index.js` to be present. Root `package.json:34`, `package.json:47`, and `package.json:48` exclude those directories. Focused check `OPENCLAW_VITEST_MAX_WORKERS=1 pnpm test test/scripts/bundled-plugin-build-entries.test.ts test/release-check.test.ts` fails 3 tests, including this assertion and the matching `test/release-check.test.ts:543` expectations.
- Impact: The tooling/release-check test lane is red on current `main`, so future release/package validation work will be blocked or noisy.
- Suggested fix: Align the tests with the new default behavior by expecting root-excluded plugins to be absent, or call `listBundledPluginPackArtifacts({ includeRootPackageExcludedDirs: true })` only in tests that intentionally validate build outputs before npm package filtering.
- Confidence: high

### Medium: Plugin docs now claim external plugins are included in the core package

- Kind: regression
- File: `docs/plugins/plugin-inventory.md`
- Line: 33
- Evidence: The commit moves `acpx`, `googlechat`, and `line` into “Core npm package” / “included in OpenClaw” docs at `docs/plugins/plugin-inventory.md:33`, `docs/plugins/plugin-inventory.md:62`, and `docs/plugins/plugin-inventory.md:71`, and the individual reference pages say the same. But the root package excludes those dist directories, and each package still has `openclaw.install.npmSpec` plus publish metadata. The inventory generator classifies excluded installable packages as external, and `pnpm plugins:inventory:check` fails with `docs/plugins/plugin-inventory.md is stale. Run pnpm plugins:inventory:gen`.
- Impact: Published docs can tell users and maintainers that these plugins ship in the core npm package when they actually need the external install path, causing setup/debugging errors.
- Suggested fix: Regenerate with `pnpm plugins:inventory:gen` if these remain external, or remove the root package exclusions and update release/package checks if the intent is to include them in the core npm package.
- Confidence: high
