## Findings

### Medium: media defaults test now asserts an unshipped OpenAI Codex fallback

- Kind: test_gap
- File: `src/media-understanding/defaults.test.ts`
- Line: 87
- Evidence: The commit adds a mock for `loadManifestMetadataSnapshot()` that returns the local fixture. That fixture gives `openai-codex` `autoPriority: { image: 20 }` at `src/media-understanding/defaults.test.ts:63` and expects it in the image auto-provider order at `src/media-understanding/defaults.test.ts:143`. Current shipped metadata in `extensions/openai/openclaw.plugin.json:791` declares `openai-codex` image capability and default model, but no image `autoPriority`. Production `resolveAutoMediaKeyProviders()` only includes providers with a finite capability priority (`src/media-understanding/defaults.ts:151`), so manifest-backed resolution excludes `openai-codex` from the automatic image fallback order.
- Impact: The test is now green while protecting behavior that the manifests do not ship. If OpenAI Codex OAuth is meant to be an automatic image-understanding fallback, users with Codex auth can lose that fallback while this test still says it is present. If it is not meant to be automatic, the test expectation is misleading and can drive future fixes in the wrong direction.
- Suggested fix: Either add the intended image `autoPriority` for `openai-codex` in `extensions/openai/openclaw.plugin.json`, or remove `openai-codex` from the fixture’s expected auto order. Prefer adding a small manifest-backed regression test that exercises the real metadata path, not only a hand-written fixture.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. `pnpm test src/media-understanding/defaults.test.ts` passes, while structured inspection of the shipped manifests computes image auto providers as `openai,anthropic,google,minimax,minimax-portal,zai` and shows `openai-codex` image priority is `null`.
- Is this the best way to solve the issue? unclear. The correct fix depends on intended product behavior: either manifest priority should be added, or the test should stop asserting `openai-codex` as an automatic fallback.
