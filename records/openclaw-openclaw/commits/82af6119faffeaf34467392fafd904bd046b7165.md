# Commit 82af611

## Summary

Found one concrete regression in `install-cli.sh`: documented tilde-form `OPENCLAW_HOME` values are expanded incorrectly for the default git checkout path.

## Findings

### Medium: `install-cli.sh` expands `OPENCLAW_HOME=‾/...` to `$HOME/‾/...`

- Kind: bug
- File: `scripts/install-cli.sh`
- Line: 44
- Evidence: The new `¥‾/*` branch runs `echo "${HOME}/${openclaw_home#‾/}"`. In Bash, that parameter-removal pattern does not strip the literal `‾/` here, so sourcing the script with `HOME=/tmp/home` and `OPENCLAW_HOME='‾/svc'` produced `git=/tmp/home/‾/svc/openclaw`. The main `install.sh` helper uses `${openclaw_home:1}` and correctly produced `/tmp/home/svc/openclaw`.
- Impact: Users following the docs that say `OPENCLAW_HOME` may be a tilde path get a checkout under a literal `‾` directory inside `$HOME` instead of the intended custom home. That can create unexpected directories, miss an existing checkout, and continue violating the `OPENCLAW_HOME` default-path contract this commit is meant to fix.
- Suggested fix: Change the branch to the same shape as `install.sh`, for example `echo "${HOME}${openclaw_home:1}"`, or escape the removal pattern as `${openclaw_home#¥‾/}` and add an `install-cli.sh` regression test for `OPENCLAW_HOME='‾/svc'`.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Source `scripts/install-cli.sh` with `OPENCLAW_INSTALL_CLI_SH_NO_RUN=1`, `HOME` set to a temp home, and `OPENCLAW_HOME='‾/svc'`; `$GIT_DIR` becomes `$HOME/‾/svc/openclaw`.
- Is this the best way to solve the issue? yes. The bug is isolated to one shell parameter expansion; mirroring `scripts/install.sh` keeps both installers aligned.
