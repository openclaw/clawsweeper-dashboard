# Commit 14a9bb98

## Summary

Found one remaining data-loss race in the new scrub rename guard. The added post-temp-write `stat` catches appends during the temp write, but an append between that final `stat` result and the `rename` can still be overwritten.

## Findings

### Medium: Final stat-to-rename window can still drop appended audit entries

- Kind: data_loss
- File: `src/config/io.audit.ts`
- Line: 597
- Evidence: `scrubConfigAuditLog` checks `finalPreRenameSize` against `originalByteLength`, then immediately renames `tmpPath` over `auditPath`. There is no lock, fd-bound compare, or append-preserving merge between the check and `rename`. I reproduced the remaining interleaving with an injected `stat` that appends one audit line after observing the original size but before `rename`; the scrub returned `{"aborted":false}`, and the final audit log no longer contained the appended `theme` record.
- Impact: A live gateway or config command can append a new `config-audit.jsonl` record in that final window, and `openclaw doctor --fix` can silently replace the log with the scrubbed snapshot, losing the new audit event. This preserves redaction of old secrets, but drops forensic/config-write history.
- Suggested fix: Avoid replacing the log based only on a size precheck. Use an exclusive/locked rewrite strategy, or after writing the temp file, append/merge any records added since the original read before rename and repeat validation, or make the scrub operate under a lock shared with `appendConfigAuditRecord`.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. A targeted injected-filesystem smoke simulated an append between the final size check and rename; `scrubConfigAuditLog` returned `aborted:false`, `afterHasAppended:false`, `afterLines:1`.
- Is this the best way to solve the issue? unclear. A shared lock is the cleanest data-loss prevention if config audit appends can participate; otherwise a merge-and-retry rewrite can preserve appended JSONL records without requiring all writers to lock.
