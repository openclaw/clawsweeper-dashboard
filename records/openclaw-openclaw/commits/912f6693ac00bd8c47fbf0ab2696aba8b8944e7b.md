## Findings

### Low: Direct published-upgrade-survivor runs no longer write the documented summary path

- Kind: regression
- File: `scripts/e2e/upgrade-survivor-docker.sh`
- Line: 18
- Evidence: The new default artifact dir always appends a lane suffix, using `default` when `OPENCLAW_DOCKER_ALL_LANE_NAME` is unset: `.../.artifacts/upgrade-survivor/$LANE_ARTIFACT_SUFFIX`. The direct package script at `package.json:1515` invokes `scripts/e2e/upgrade-survivor-docker.sh` without setting `OPENCLAW_DOCKER_ALL_LANE_NAME`, so a direct `pnpm test:docker:published-upgrade-survivor` now writes `summary.json` under `.artifacts/upgrade-survivor/default/`. The documented path remains `.artifacts/upgrade-survivor/summary.json` in `docs/reference/test.md:47`, `docs/help/testing.md:623`, and `docs/help/testing-updates-plugins.md:255`.
- Impact: Users or automation following the documented focused command can miss the upgrade-survivor summary after a successful run. The aggregate scheduler path is still fine because expanded lanes pass explicit per-lane artifact dirs, but the standalone command’s documented artifact contract changed.
- Suggested fix: Preserve the old root artifact dir when `OPENCLAW_DOCKER_ALL_LANE_NAME` is unset, and append the sanitized lane suffix only when the scheduler provides a real lane name. Updating docs alone is possible, but preserving the direct command’s old path is the lower-risk compatibility fix.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Run `pnpm test:docker:published-upgrade-survivor` without `OPENCLAW_DOCKER_ALL_LANE_NAME`; by the shell expansion in `scripts/e2e/upgrade-survivor-docker.sh:16-18`, the mounted artifact dir is `.artifacts/upgrade-survivor/default`, not the documented `.artifacts/upgrade-survivor`.
- Is this the best way to solve the issue? yes. The scheduler-specific isolation should be conditional on scheduler-provided lane identity, while explicit `OPENCLAW_UPGRADE_SURVIVOR_ARTIFACT_DIR` should continue to override both paths.
