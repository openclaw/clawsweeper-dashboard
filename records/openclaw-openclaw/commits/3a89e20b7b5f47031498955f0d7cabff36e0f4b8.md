# Commit 3a89e20

## Summary

Found one concrete hardlink-policy bypass in the new wrapper. The pre-scan closes the same-filesystem gap, but the wrapper stops forwarding `sourceHardlinks: "reject"` to the upstream EXDEV fallback, so a hardlink introduced after the pre-scan can still be copied and published.

## Findings

### Medium: EXDEV fallback drops the hardlink rejection option after the pre-scan

- Kind: security
- File: `src/infra/replace-file.ts`
- Line: 34
- Evidence: `movePathWithCopyFallback()` checks `options.sourceHardlinks === "reject"` before delegating, but then calls `movePathWithCopyFallbackBase({ from: options.from, to: options.to })` without forwarding `sourceHardlinks`. The pinned `@openclaw/fs-safe` helper still enforces `sourceHardlinks: "reject"` inside its copy fallback. A focused live check forced the first rename to throw `EXDEV`, created a hardlink after OpenClawŌĆÖs pre-scan, and OpenClawŌĆÖs wrapper completed successfully; the same check against `@openclaw/fs-safe/atomic` rejected with `Refusing to move hardlinked file with copy fallback`.
- Impact: callers such as package install/update paths pass `sourceHardlinks: "reject"` expecting hardlink-safe moves. On cross-device moves, a concurrent source change between the pre-scan and fallback copy can bypass that policy and publish copied content from a now-hardlinked source.
- Suggested fix: keep the pre-scan for same-device moves, but delegate with `sourceHardlinks: options.sourceHardlinks` as well. Add a regression test that forces `EXDEV` and creates the hardlink after the pre-scan but before fallback copy.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. A `pnpm exec tsx` smoke patched `fs.rename` to create a hardlink and throw `EXDEV` after the wrapperŌĆÖs pre-scan; OpenClawŌĆÖs wrapper returned success, while the upstream pinned helper rejected the same scenario when passed `sourceHardlinks: "reject"`.
- Is this the best way to solve the issue? yes. Forwarding the existing option preserves upstream fallback enforcement while retaining the new OpenClaw same-device pre-scan behavior.
