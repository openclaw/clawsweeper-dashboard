# Commit de186a8b66

## Summary

Found one audit false-negative regression: the commit downgrades small-model findings to `info` when `web_search`, `web_fetch`, and `browser` are denied, but it still does not account for `x_search`, which is part of `group:web` and can be available through the xAI plugin.

## Findings

### Medium: Small-model audit can clear exposure while `x_search` remains available

- Kind: security
- File: `src/security/audit-extra.summary.ts`
- Line: 237
- Evidence: `collectSmallModelRiskFindings()` only adds `web_search`, `web_fetch`, and `browser` to `exposed`, then treats `exposed.length === 0` as safe. But `group:web` includes `x_search` in `docs/gateway/config-tools.md:38`, `x_search` is in the coding profile in `src/agents/tool-catalog.ts:121`, and the xAI plugin registers it as a tool in `extensions/xai/index.ts:224`. A focused inline audit check with `x_search` enabled and only `web_search`/`web_fetch`/`browser` denied returned `severity: "info"` and `web=[off]`.
- Impact: Operators can follow the new regression test pattern of denying only `web_search`, `web_fetch`, and `browser` for a small model while leaving `x_search` available. The audit then reports no web/browser exposure, even though a small model can still query untrusted external X content.
- Suggested fix: Include `x_search` in the small-model exposure calculation, or compute exposure from the actual `group:web` expansion used by tool policy. The regression test should deny `group:web` plus `browser`, or assert that explicit denies must include `x_search`.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. The inline `collectSmallModelRiskFindings()` check returned `info`/`web=[off]` with `tools.web.x_search.enabled=true`, xAI plugin config present, and byProvider deny limited to `web_search`, `web_fetch`, and `browser`.
- Is this the best way to solve the issue? no. Reusing provider/model deny resolution is right, but the safe predicate should cover every tool in the web exposure family before downgrading the finding.
