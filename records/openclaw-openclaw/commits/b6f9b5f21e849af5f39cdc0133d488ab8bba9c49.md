# Commit b6f9b5f

## Summary

Found one grouped-completion fallback edge case: the new label-prefixed combined fallback text can prevent the existing incomplete-prefix guard from recognizing a truncated raw child result.

## Findings

### Medium: Grouped fallback text can miss truncated first-result prefixes

- Kind: reliability
- File: `src/agents/subagent-announce-delivery.ts`
- Line: 565
- Evidence: For grouped completions, the new formatter prefixes each result with `taskLabel:\n` before joining the blocks. Later, `hasIncompleteCompletionPrefix` only falls back when the normalized full fallback text starts with the visible requester-agent output (`expected.startsWith(visible)`). The adjacent existing test proves the intended guard for a mid-word raw child-result prefix in the single-completion case; with grouped completions, a visible payload like `34/34 tests pass... Now dock` is no longer a prefix of `first task:\n34/34 tests pass...`, so `shouldSendCompletionFallback` returns false and the truncated requester output is treated as delivered.
- Impact: If a grouped completion announce produces a visible but truncated raw prefix of the first child result, OpenClaw can skip the direct fallback and leave the requester chat with an incomplete completion instead of delivering the full grouped results.
- Suggested fix: Keep a raw per-completion integrity candidate list in addition to the formatted grouped delivery text, or make the prefix check test both the formatted block and each extracted completion text before deciding no fallback is needed. Add a grouped variant of the existing mid-word prefix test.
- Confidence: medium

## Details

- Do we have a high-confidence way to reproduce the issue? Unclear; the code path is concrete and mirrors the existing single-completion regression test, but my one-off `tsx` probe was blocked by package export resolution outside the Vitest harness.
- Is this the best way to solve the issue? Unclear; preserving formatted grouped delivery is reasonable, but the integrity check should not depend only on the formatted aggregate when visible output may be copied from an unformatted child result.
