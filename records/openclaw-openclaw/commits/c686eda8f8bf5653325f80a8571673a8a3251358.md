Current `main` already has a follow-up commit, `87f3501b91`, that removes the reviewed commit’s scoped external CLI auth sync for filtered gateway live runs, so I did not report that transient risk as actionable.

## Findings

### Low: Setup timeout cannot stop synchronous setup stalls

- Kind: reliability
- File: `src/gateway/gateway-models.profiles.live.test.ts`
- Line: 250
- Evidence: `withGatewayLiveSetupTimeout` takes an already-created `Promise` and relies on `Promise.race` with `setTimeout`. The setup call sites at `src/gateway/gateway-models.profiles.live.test.ts:2414` and `src/gateway/gateway-models.profiles.live.test.ts:2426` wrap synchronous setup helpers in `Promise.resolve().then(...)`. If one of those helpers blocks the event loop, the timer callback cannot run. Focused repro on current `main`: `OPENCLAW_LIVE_GATEWAY_SETUP_TIMEOUT_MS=5000 OPENCLAW_LIVE_GATEWAY_PROVIDERS=codex-cli OPENCLAW_LIVE_GATEWAY_MODELS=codex-cli/__clawsweeper_missing_model__ pnpm test:live:gateway-profiles` was still stuck after 25s at `[all-models] loading auth profiles`, instead of failing after 5s.
- Impact: The live gateway profile Docker/setup lane can still hang until a much larger suite or process timeout, defeating the commit’s intended setup bound and making provider-filtered live checks harder to diagnose.
- Suggested fix: Do not rely on `Promise.race` to bound synchronous setup work. Make the slow setup path async/cooperative, move blocking auth/model discovery into a killable worker/subprocess, or add a process-level phase timeout around this live setup path.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. The command above sets the setup timeout to 5s and uses a nonexistent filtered model to avoid real model calls; it still exceeded a 25s shell timeout while in setup.
- Is this the best way to solve the issue? unclear. The safest direction depends on which setup helper is allowed to block in CI/Testbox, but the current wrapper cannot provide a hard wall-clock bound for synchronous work.
