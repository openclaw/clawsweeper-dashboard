Found one current-main regression from the commit’s shared direct-room inspection change. The later follow-up commit fixes repair persistence, but outbound direct-user resolution still pays the new joined-room discovery cost even when `m.direct` already has a valid strict room.

## Findings

### Medium: Direct sends now call joined-room discovery even when `m.direct` is already valid

- Kind: regression
- File: `extensions/matrix/src/matrix/direct-management.ts`
- Line: 282
- Evidence: This commit removed the `!mappedStrict` guard, so `inspectMatrixDirectRooms()` always awaits `client.getJoinedRooms()`. `resolveDirectRoomId()` calls this shared inspector for `user:@...` sends at `extensions/matrix/src/matrix/send/targets.ts:66`, even when account data already maps the user to a strict DM. The existing regression test at `extensions/matrix/src/matrix/send/targets.test.ts:56` still expects no joined-room scan for that fast path and now fails: `expected "vi.fn()" to not be called at all, but actually been called 1 times`.
- Impact: Valid mapped direct-user sends now depend on an unnecessary `/joined_rooms` lookup. Failures are swallowed, but a slow or stuck homeserver request can delay or hang outbound sends that already had enough `m.direct` data to resolve the room.
- Suggested fix: Keep joined-room discovery enabled for repair/inspection flows that need to surface additional rooms, but let send resolution skip it once a strict mapped room is found. A small option on `inspectMatrixDirectRooms()` or a send-specific fast path would preserve the issue fix without widening the outbound send dependency.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Run `pnpm test extensions/matrix/src/matrix/send/targets.test.ts -- --reporter=verbose`; the `uses m.direct when available` test fails because `getJoinedRooms()` is called.
- Is this the best way to solve the issue? yes, directionally. The repair/CLI discovery behavior should stay broad, while outbound direct-user sends should not enumerate joined rooms after a valid mapped strict DM has already been selected.
