## Findings

### Low: Simplified Chinese love phrases are captured but categorized as `other`

- Kind: bug
- File: `extensions/memory-lancedb/index.ts`
- Line: 613
- Evidence: The new trigger regex captures simplified Chinese `爱` in phrases such as `我爱猫` at `extensions/memory-lancedb/index.ts:516`, but `detectCategory` only includes traditional `愛`, not simplified `爱`, in its preference regex at `extensions/memory-lancedb/index.ts:613`. A focused probe returned `{"captureLoveSimplified":true,"categoryLoveSimplified":"other","captureLoveTraditional":true,"categoryLoveTraditional":"preference"}`.
- Impact: Auto-captured simplified Chinese preference memories using `爱` are stored with category `other` instead of `preference`, so recall context, list output, and category-based inspection/query workflows get incorrect metadata.
- Suggested fix: Add simplified `爱` to the `detectCategory` preference regex and add a regression test for `detectCategory("我爱猫") === "preference"`.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes: run `pnpm exec tsx -e "import { shouldCapture, detectCategory } from './extensions/memory-lancedb/index.ts'; console.log(JSON.stringify({ captureLoveSimplified: shouldCapture('我爱猫'), categoryLoveSimplified: detectCategory('我爱猫'), captureLoveTraditional: shouldCapture('我愛猫'), categoryLoveTraditional: detectCategory('我愛猫') }))"`.
- Is this the best way to solve the issue? yes: the trigger already intentionally supports simplified `爱`; the category detector should mirror that same preference vocabulary.
