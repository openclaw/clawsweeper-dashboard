## Summary

Found one concrete incomplete-fix bug: the commit fixes active model display for `session_status(current)`, but the same active-run status path still ignores transient thinking overrides, which was explicitly reported on the linked issue before it was closed.

## Findings

### Low: `session_status(current)` still reports stored/default thinking instead of the active run thinking level

- Kind: bug
- File: `src/agents/tools/session-status-tool.ts`
- Line: 715
- Evidence: The commit threads only `activeModelProvider` and `activeModelId` through `createOpenClawTools` into `createSessionStatusTool`. When building the status card, the tool still passes `resolvedThinkLevel: statusSessionEntry.thinkingLevel`, so a heartbeat or inline `/think:medium` run with no persisted `thinkingLevel` falls back to the agent default in `buildStatusText`. The linked issue comment from May 7 specifically reports this alongside the model bug: heartbeat prompt uses `/think:medium`, but status still says `Think: high`.
- Impact: Heartbeat/self-introspection status can still lie about the active execution settings. Users relying on heartbeat status reports will see the correct active model after this commit but still see the wrong thinking level for the same run.
- Suggested fix: Thread the resolved active run thinking level into the session-status tool context alongside the active model, and apply it only for the same semantic-current/implicit-current cases. Keep it transient; do not persist it into `SessionEntry`.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes: configure an agent default `thinkingDefault: high`, run a heartbeat or current turn with `/think:medium`, and call `session_status({"sessionKey":"current"})`; source shows the tool has no active-run thinking input and passes only persisted `statusSessionEntry.thinkingLevel`.
- Is this the best way to solve the issue? yes: the model fix already uses transient tool context to avoid leaking heartbeat state into persistence, and the same pattern should be used for active thinking.
