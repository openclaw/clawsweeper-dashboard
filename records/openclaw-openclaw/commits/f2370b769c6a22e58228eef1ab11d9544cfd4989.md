# Commit f2370b7

## Summary

Found one concrete remaining bug: the commit allows `MINIMAX_OAUTH_TOKEN` from env, but MiniMax OAuth onboarding stores OAuth credentials in auth profiles, not in that env var. OAuth-only onboarding can still hit `missing_minimax_api_key`.

## Findings

### Medium: OAuth-profile MiniMax setups still cannot supply web_search credentials

- Kind: bug
- File: `extensions/minimax/src/minimax-web-search-provider.runtime.ts`
- Line: 55
- Evidence: `resolveMiniMaxApiKey()` only reads `plugins.entries.minimax.config.webSearch.apiKey` / merged search config plus env vars. MiniMax OAuth onboarding returns `buildOauthProviderAuthResult()` with an auth profile and a `models.providers.minimax-portal` config patch, and the provider-auth flow persists that profile separately; the web-search provider context has config/search/runtime metadata but no auth-profile token resolver.
- Impact: A user who completes `minimax-portal` OAuth through onboarding, but does not also export `MINIMAX_OAUTH_TOKEN` or configure `plugins.entries.minimax.config.webSearch.apiKey`, can still get `missing_minimax_api_key`. That contradicts the changelog/docs claim that OAuth-authorized MiniMax Token Plan setups do not need a separate web-search key.
- Suggested fix: Add a web-search credential seam that can resolve the `minimax-portal` OAuth profile token for MiniMax search, or have MiniMax OAuth setup populate a web-search credential/SecretRef that `resolveMiniMaxApiKey()` can consume. Add a regression for auth-profile-only OAuth setup, not just `MINIMAX_OAUTH_TOKEN` env.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Start from a MiniMax OAuth profile/config patch with no `MINIMAX_CODE_PLAN_KEY`, `MINIMAX_CODING_API_KEY`, `MINIMAX_OAUTH_TOKEN`, `MINIMAX_API_KEY`, or webSearch apiKey; the resolver returns no key and `executeMiniMaxWebSearchProviderTool()` returns `missing_minimax_api_key` before any network call.
- Is this the best way to solve the issue? unclear. The right fix depends on whether web-search providers should be allowed to resolve provider OAuth profiles directly, or whether onboarding should write an explicit web-search credential reference.
