# Commit afbc395

## Summary

Found one high-confidence regression: `tools.catalog` can evict the pinned channel plugin registry when it loads non-startup plugin tools for the catalog.

## Findings

### High: Catalog tool discovery can replace the live channel registry

- Kind: regression
- File: `src/gateway/server-methods/tools-catalog.ts`
- Line: 98
- Evidence: The new catalog path calls `ensureStandalonePluginToolRegistryLoaded(...)` with `toolAllowlist: ["group:plugins"]`. That helper loads with `surface: "channel"` in `src/plugins/tools.ts:690`, and the standalone loader installs the loaded snapshot with `setActivePluginRegistry(...)` and `pinActivePluginChannelRegistry(...)` in `src/plugins/runtime/standalone-runtime-registry-loader.ts:38` and `src/plugins/runtime/standalone-runtime-registry-loader.ts:43`. A focused smoke reproduced the state change: a pinned startup registry containing `telegram` became a tool-discovery registry with no channels after loading eligible non-startup `tavily` tools.
- Impact: A UI/RPC `tools.catalog` request can silently replace the gateway’s channel registry with a scoped tool registry. After that, plugin-backed channel lookups can return no loaded channel plugin, breaking channel operations until another reload/restoration path runs.
- Suggested fix: Do not load catalog-only tool discovery into the `channel` surface. Either load into a non-channel/tool-specific surface, return the scoped registry directly to `resolvePluginTools`, or make `ensureStandalonePluginToolRegistryLoaded` preserve an existing pinned channel registry when it loads extra tool-only plugin IDs.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. In-process smoke: pin a startup registry with `telegram`, call `ensureStandalonePluginToolRegistryLoaded` for config allowing `tavily` with `TAVILY_API_KEY` set, then compare `getActivePluginChannelRegistry()` before/after. Result: `same: false`, before channels `["telegram"]`, after channels `[]`, after tools include `tavily`.
- Is this the best way to solve the issue? no. Loading plugin tool factories for catalog is reasonable, but installing the scoped discovery snapshot as the channel registry crosses runtime surfaces and can evict live channel state.
