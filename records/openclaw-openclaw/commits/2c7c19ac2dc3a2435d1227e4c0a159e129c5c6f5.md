### Low: Codex CLI backend lane requires OpenAI auth but plan JSON does not advertise it

- Kind: reliability
- File: `scripts/lib/docker-e2e-scenarios.mjs`
- Line: 367
- Evidence: The commit changed `live-cli-backend-codex` to run with `OPENCLAW_LIVE_CLI_BACKEND_AUTH=api-key`. The wrapper exits if that mode is used for `codex-cli` without `OPENAI_API_KEY` (`scripts/test-live-cli-backend-docker.sh:66`). But `laneCredentialRequirements()` only marks `openwebui`, `openai-web-search-minimal`, and `live-codex-npm-plugin` as requiring `openai`, so `OPENCLAW_DOCKER_ALL_LANES=live-cli-backend-codex node scripts/test-docker-all.mjs --plan-json` returns `"credentials": []`.
- Impact: Focused Docker planner runs for `live-cli-backend-codex` skip the existing “Validate Docker E2E credentials” OpenAI check and fail later inside the lane with a less useful setup error when `OPENAI_API_KEY` is absent. This also makes the plan JSON’s credential contract inaccurate for CI/manual lane selection.
- Suggested fix: Add `live-cli-backend-codex` to the OpenAI branch in `scripts/lib/docker-e2e-plan.mjs:331`, or derive credential requirements from lane auth metadata, and add a planner test asserting selected `live-cli-backend-codex` emits `credentials: ["openai"]`.
- Confidence: high
