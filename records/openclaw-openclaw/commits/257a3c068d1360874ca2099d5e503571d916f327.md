# Commit 257a3c0

## Summary

Found one regression in the archive install path: ClawHub code-plugin installs now copy package archives without installing declared runtime dependencies, but ClawHub publishes source package files and excludes `node_modules`.

## Findings

### Medium: ClawHub code-plugin installs lose runtime dependencies

- Kind: regression
- File: `src/plugins/install.ts`
- Line: 896
- Evidence: ClawHub installs call `installPluginFromArchive` in `src/plugins/clawhub.ts:867`, and this commit forces native package archive/directory installs through `hasDeps: false`. The lower-level installer only runs `npm install` when `hasDeps` is true (`src/infra/install-package-dir.ts:251`). ClawHub `package publish` dry-run for `extensions/diffs` uploads source files plus `package.json`, not `node_modules`, while `extensions/diffs/package.json` declares runtime deps including `typebox`. A focused local install/load check reproduced the failure: the installed plugin had no `node_modules`, and loading `diffs` failed with `Cannot find module 'typebox'` from `extensions/diffs/src/tool.ts`.
- Impact: ClawHub is the preferred install path for bare plugin specs. Any ClawHub code plugin with runtime dependencies can install successfully but fail at runtime. This affects multiple publishable official plugins, including `diffs`, `discord`, `memory-lancedb`, `whatsapp`, and others with `dependencies`.
- Suggested fix: restore safe dependency convergence for ClawHub/native package archives that declare runtime deps, or make ClawHub package publishing/install validation enforce truly self-contained archives and reject dependency-declaring packages with an actionable error. Given current ClawHub publishing excludes `node_modules`, restoring install-time deps for ClawHub code-plugin archives is the compatible fix.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Publish/dry-run `extensions/diffs` as a ClawHub code plugin, install/copy it through the archive/package-dir path with this commit, then load only `diffs`; it fails on missing `typebox`.
- Is this the best way to solve the issue? unclear. The product direction may want local archives to be self-contained, but current ClawHub publishing is not self-contained, so the ClawHub install path needs either dependency install restored or a matching publisher/install contract change.
