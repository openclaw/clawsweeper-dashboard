## Findings

### Low: Missing-summary guard can still succeed when the Mantis command exits 0

- Kind: bug
- File: `.github/workflows/mantis-slack-desktop-smoke.yml`
- Line: 254
- Evidence: The workflow now captures `mantis_exit`, then handles a missing `mantis-slack-desktop-smoke-summary.json` with `exit "$mantis_exit"`. If the CLI exits `0` but fails to produce the required summary, this branch exits `0` and ends the step before status/evidence generation. I confirmed the shell behavior with a focused snippet: missing file plus `mantis_exit=0` returns `exit=0`.
- Impact: A broken harness path that returns success without writing the summary can make a manual run without `pr_number` appear green while producing no status/evidence. With `pr_number`, the later evidence publish step likely fails on a missing manifest, but the root cause is obscured.
- Suggested fix: Make the missing-summary branch fail unconditionally, or preserve non-zero status while mapping zero to one, for example `exit "$([[ "$mantis_exit" -eq 0 ]] && echo 1 || echo "$mantis_exit")"` or a clearer `if` block.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes: run the workflow shell branch with no summary file and `mantis_exit=0`; it exits successfully.
- Is this the best way to solve the issue? yes: the summary is required by the following `jq`, evidence manifest, and report-summary steps, so missing summary should be a hard failure independent of the command exit code.
