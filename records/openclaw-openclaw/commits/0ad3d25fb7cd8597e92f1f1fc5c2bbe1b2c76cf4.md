# Commit 0ad3d25

## Summary

Found one medium-severity regression: the new shared cron subagent model helper preserves and extends the known wrong precedence where a parent agent's own model shadows `agents.defaults.subagents.model`.

## Findings

### Medium: Cron subagent selection ignores the global subagent default when the parent agent has a model

- Kind: regression
- File: `src/agents/agent-scope.ts`
- Line: 194
- Evidence: `resolveSubagentModelConfigSelectionResult` checks `agentConfig.model` before `cfg.agents.defaults.subagents.model`, and `src/cron/isolated-agent/model-selection.ts:86` now uses that helper for isolated cron model selection. This contradicts the current native spawn helper in `src/agents/model-selection.ts:323`, which checks per-agent `subagents.model`, then `agents.defaults.subagents.model`, then the parent agent model. It also contradicts `docs/tools/subagents.md`, which says subagents inherit the caller unless `agents.defaults.subagents.model` or per-agent `subagents.model` is set.
- Impact: With a common config where `main.model` is set and `agents.defaults.subagents.model` is set, isolated cron subagent-style runs pick the parent model instead of the configured subagent default. That can route scheduled subagent work to the wrong provider/model and skip the intended subagent fallback chain. This is the still-open behavior tracked by `https://github.com/openclaw/openclaw/issues/58822`.
- Suggested fix: Order the shared helper as `agentConfig.subagents.model`, then `cfg.agents.defaults.subagents.model`, then `agentConfig.model`; keep the explicit fallback-only per-agent subagent override handling before selection so `{ fallbacks: [] }` still disables fallback inheritance.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. A direct probe with `main.model = anthropic/claude-opus-4-6` and `agents.defaults.subagents.model.primary = openai/gpt-5.4` returned `cronShared.primary = anthropic/claude-opus-4-6`, while the native spawn helper returned `openai/gpt-5.4`.
- Is this the best way to solve the issue? yes. It aligns cron with the existing native helper and the documented subagent model contract without changing payload model override precedence.
