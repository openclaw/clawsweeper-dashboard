## Summary

Found one correctness gap: the commit handles only the `37` / `Operation already in progress` launchctl text, but the linked bug’s user-visible bootstrap failure was `Bootstrap failed: 5: Input/output error`. That shape still bypasses the new state probe and throws.

## Findings

### Medium: Observed bootstrap I/O race still bypasses the recovery probe

- Kind: bug
- File: `src/daemon/launchd.ts`
- Line: 681
- Evidence: `bootstrapLaunchAgentOrThrow` only calls `probeLaunchAgentState` when `isLaunchctlOperationAlreadyInProgress(detail)` matches. The new matcher accepts `operation already in progress` or `bootstrap failed: 37`, but issue #84721 reports the CLI failure as `launchctl bootstrap failed: Bootstrap failed: 5: Input/output error` while unified logs showed the concurrent bootstrap loaded/spawned the service. That returned detail will still fall through to `throw new Error(...)` without probing.
- Impact: `openclaw gateway restart` can still report a false macOS Gateway restart failure for the exact race this commit claims to fix, leaving issue #84721 only partially resolved.
- Suggested fix: include the observed code-5/I/O bootstrap race shape in the recoverable branch, but keep the current safety condition: return success only after `launchctl print <serviceTarget>` confirms the LaunchAgent is loaded (`running` or `stopped`). Add a regression test where bootstrap returns `Bootstrap failed: 5: Input/output error` and the follow-up print confirms the service is loaded, plus a genuine I/O failure case that remains fatal.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? Yes. In the added restart reload test setup, use `state.bootstrapError = "Bootstrap failed: 5: Input/output error"` with `state.bootstrapLoadsServiceOnFailure = true`; current code will not call the new print probe because the matcher rejects that detail.
- Is this the best way to solve the issue? Yes. The probe-after-specific-bootstrap-failure shape is sound; the gap is that the detector does not include the reported launchctl error surface.
