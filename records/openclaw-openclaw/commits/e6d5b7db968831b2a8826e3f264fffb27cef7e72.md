# Commit e6d5b7d

## Summary

The runtime adapter added in `src/plugins/registry.ts` fixes the callback gap for JavaScript plugins and the focused loader regression passes. One contract issue remains: the public TypeScript SDK still types gateway handlers as callback-only, so the newly supported async return-value form is rejected for typed plugin authors.

## Findings

### Medium: Return-value gateway methods are still rejected by the public handler type

- Kind: compatibility
- File: `src/plugins/registry.ts`
- Line: 356
- Evidence: `adaptPluginGatewayMethodHandler` now treats a non-`undefined` handler result as a successful gateway response, but the exported `GatewayRequestHandler` contract remains `(opts) => Promise<void> | void` in `src/gateway/server-methods/shared-types.ts:151`, and `OpenClawPluginApi.registerGatewayMethod` still consumes that type in `src/plugins/types.ts:2624`. A focused TypeScript compiler probe for `const handler: GatewayRequestHandler = async () => ({ ok: true })` reports `TS2322: Type 'Promise<{ ok: boolean; }>' is not assignable to type 'void | Promise<void>'`.
- Impact: TypeScript plugins cannot use the newly supported async return-value completion path without casts or `@ts-ignore`, so the merged fix is runtime-only and leaves the public plugin API inconsistent with shipped behavior.
- Suggested fix: update the plugin-facing gateway handler result type to allow returned payloads, keep explicit `respond(...)` precedence documented, and add a public contract/type test plus SDK docs for return-style completion.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. The minimal TypeScript assignability probe above fails against the current exported handler type, while `src/plugins/loader.test.ts` proves the runtime accepts returned payloads.
- Is this the best way to solve the issue? yes. The registry adapter is the right runtime seam; the remaining fix is to align the SDK type/docs/contract tests with that behavior rather than requiring external plugins to cast around the public API.
