## Findings

### Low: `model.run` capability metadata omits the new `--thinking` flag

- Kind: bug
- File: `src/cli/capability-cli.ts`
- Line: 125
- Evidence: The actual command registers `.option("--thinking <level>", "Thinking level override")` at `src/cli/capability-cli.ts:1674`, but the static `CAPABILITY_METADATA` entry for `model.run` still lists only `--prompt`, `--file`, `--model`, `--local`, `--gateway`, and `--json` at `src/cli/capability-cli.ts:125`. A focused smoke confirmed `pnpm openclaw infer inspect --name model.run --json` returns that stale `flags` array without `--thinking`.
- Impact: Scripts or users that rely on `infer inspect` as the machine-readable capability contract will not discover the new thinking override even though the command accepts it and docs mention it.
- Suggested fix: Add `--thinking` to the `model.run` metadata flags and add/extend a CLI metadata test so future option additions keep metadata aligned.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Run `pnpm openclaw infer inspect --name model.run --json`; the returned `flags` array omits `--thinking` while `infer model run --help` and the command registration include it.
- Is this the best way to solve the issue? yes. The metadata is a static list, so updating that list and testing the inspect output is the narrow fix.
