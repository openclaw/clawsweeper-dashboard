# Commit 112dedd

## Summary

Found one concrete packaging/runtime regression in `memory-lancedb`: the commit removes the fallback runtime install path but leaves `@lancedb/lancedb` outside the root OpenClaw package dependencies and nested plugin `node_modules` are explicitly omitted from the npm package.

## Findings

### Medium: Bundled `memory-lancedb` can no longer load LanceDB in packaged installs

- Kind: regression
- File: `extensions/memory-lancedb/lancedb-runtime.ts`
- Line: 46
- Evidence: `createLanceDbRuntimeLoader()` now only does `import("@lancedb/lancedb")` and, on failure, throws `bundled @lancedb/lancedb dependency is unavailable`. The removed code previously read `extensions/memory-lancedb/package.json`, installed `@lancedb/lancedb` into a state runtime directory, and imported that resolved copy. Current package metadata still declares `@lancedb/lancedb` only in `extensions/memory-lancedb/package.json:6`, while root `package.json` does not declare it and also excludes `dist/extensions/node_modules/**` plus `dist/extensions/*/node_modules/**` from the npm package. `scripts/postinstall-bundled-plugins.mjs:2` also says plugin package dependencies are never installed by postinstall.
- Impact: Users enabling the bundled `memory-lancedb` plugin from the OpenClaw package can hit the new hard failure even though the plugin is documented as bundled. Node resolution from `dist/extensions/memory-lancedb/lancedb-runtime.js` will search plugin/root `node_modules`, but the root package does not install `@lancedb/lancedb` and the plugin-local `node_modules` tree is omitted.
- Suggested fix: either restore an explicit repair/install path before the dynamic import, or make packaged OpenClaw install this native dependency through a supported package dependency path. If the intended contract is “separate plugin install only,” stop presenting bundled `memory-lancedb` as usable from the root package and gate activation/config accordingly.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? Yes. In an installed/package-shaped tree without plugin-local `node_modules`, enable `plugins.slots.memory = "memory-lancedb"` and configure `plugins.entries.memory-lancedb.config.embedding`; the first DB initialization calls `loadLanceDbModule()`, which now only attempts `import("@lancedb/lancedb")`.
- Is this the best way to solve the issue? Unclear. The safer direction depends on the intended product contract: either bundled memory should bring its dependency with OpenClaw, or plugin-package dependencies must be installed/verified before activation.
