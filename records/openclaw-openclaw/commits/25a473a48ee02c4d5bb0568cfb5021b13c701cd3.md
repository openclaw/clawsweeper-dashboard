# Commit 25a473a

## Summary

The added Vitest rules are currently ineffective because the Oxlint config does not enable the Vitest plugin.

## Findings

### Low: Vitest safety rules are configured but not active

- Kind: bug
- File: `.oxlintrc.json`
- Line: 3
- Evidence: `.oxlintrc.json:3` sets `"plugins": ["unicorn", "typescript", "oxc"]`, omitting `vitest`, while the commit adds `vitest/no-focused-tests`, `vitest/no-disabled-tests`, `vitest/no-conditional-tests`, `vitest/no-conditional-expect`, and `vitest/no-standalone-expect` at `.oxlintrc.json:111`. `oxlint --print-config src/infra/ports.test.ts` showed no `vitest/*` rules in the resolved config. A synthetic repo-root `*.test.ts` containing `describe.only`, a conditional `expect`, and a standalone `expect` passed with `node_modules/oxlint/bin/oxlint --config .oxlintrc.json <file>` (`Found 0 warnings and 0 errors`, exit 0). The same file failed when the Vitest plugin and these denies were enabled explicitly.
- Impact: CI/local lint will still allow focused tests, disabled tests, conditional tests/expectations, and standalone expectations despite the changelog saying these safety rules are enabled.
- Suggested fix: add `vitest` to the Oxlint `plugins` list or enable it in the lint wrapper, then rerun `pnpm lint:all` and handle any newly surfaced violations or intentional overrides.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. - Is this the best way to solve the issue? yes. Rule entries alone are not enough here; the resolved config must include the Vitest plugin so Oxlint registers and applies `vitest/*` rules.
