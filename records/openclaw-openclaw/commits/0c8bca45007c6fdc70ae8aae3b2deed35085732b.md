# Commit 0c8bca4

## Summary

Found one low-severity behavior gap: the live no-config render is pluginless, but generated startup metadata still passes a config object, which trips the new `|| renderOptions?.config` condition and keeps packaged root help opted into plugin descriptors.

## Findings

### Low: Packaged root help still includes plugin descriptors through startup metadata

- Kind: bug
- File: `src/cli/program/root-help.ts`
- Line: 30
- Evidence: The new guard loads plugin descriptors when either `includePluginDescriptors === true` or any `config` is present. `scripts/write-cli-startup-metadata.ts:196` and `scripts/write-cli-startup-metadata.ts:231` still render root help with `{ config, env }`, and `scripts/write-cli-startup-metadata.ts:380` writes that output as `rootHelpText`. Runtime root help then prefers that precomputed text in `src/cli/run-main.ts:483` and `src/entry.ts:192`.
- Impact: The commit makes `renderRootHelpText()` pluginless in the fallback path, but the default installed/package path can still show plugin-provided commands in root help. A focused source smoke reproduced the mismatch: no-config root help had no extra plugin command, while rendering with the same `{ config, env }` shape used by startup metadata added `memory`.
- Suggested fix: Make plugin descriptor loading depend only on explicit `includePluginDescriptors === true`, or have the startup metadata generator omit `config`/`env` or pass an explicit false mode for the root-help render.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Run a source render comparison: `renderRootHelpText()` is pluginless, while `renderRootHelpText({ config, env, pluginSdkResolution: "src" })` includes the plugin descriptor `memory`; the metadata generator uses that second call shape.
- Is this the best way to solve the issue? yes, if the intended contract is truly “pluginless by default.” The current implicit config opt-in conflicts with that contract; explicit `includePluginDescriptors` is the safer control.
