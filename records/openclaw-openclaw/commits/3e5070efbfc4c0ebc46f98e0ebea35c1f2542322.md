# Commit 3e5070e

## Summary

Found one regression in the Z.AI preserved-thinking contract. The commit makes `reasoning_content` replay unconditional for Z.AI completions, but the Z.AI plugin and docs only enable preserved thinking when `params.preserveThinking` sets `thinking.clear_thinking: false`.

## Findings

### Medium: Z.AI reasoning history is replayed even when preserved thinking is not enabled

- Kind: regression
- File: `src/agents/openai-transport-stream.ts`
- Line: 2517
- Evidence: `shouldPreserveReasoningContentReplay()` now returns true whenever `compat.thinkingFormat === "zai"`, and `buildOpenAICompletionsParams()` passes that into `sanitizeCompletionsReasoningReplayFields()`, so assistant `reasoning_content` survives for all Z.AI requests. The Z.AI provider wrapper only sends `thinking: { type: "enabled", clear_thinking: false }` when `params.preserveThinking` or `params.preserve_thinking` is true; otherwise it returns without adding a `thinking` payload at `extensions/zai/index.ts:91`. OpenClaw’s Z.AI docs describe replaying prior `reasoning_content` only when preserved thinking is enabled at `docs/providers/zai.md:165`.
- Impact: users who did not opt into preserved thinking now still send prior hidden reasoning blocks back to Z.AI on follow-up requests. That breaks OpenClaw’s documented cost/privacy control and is inconsistent with Z.AI’s API contract, where `clear_thinking` defaults to true and prior `reasoning_content` is ignored/removed unless preserved thinking is explicitly enabled. Z.AI’s official docs say preserved thinking is enabled by setting `clear_thinking: false` and returning complete historical `reasoning_content`; the chat-completion API documents `clear_thinking` defaulting to true and recommends that default for general chat to reduce context length and cost. ([docs.z.ai](https://docs.z.ai/guides/capabilities/thinking-mode?utm_source=openai))
- Suggested fix: gate Z.AI `reasoning_content` preservation on the same provider-owned `preserveThinking` decision that sets `clear_thinking: false`, or move the replay sanitizer decision into a provider hook that can see resolved extra params. Keep native DeepSeek preservation separate.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. The added `openai-transport-stream.test.ts` case proves `buildOpenAICompletionsParams()` preserves Z.AI `reasoning_content` without any `preserveThinking` input, while `extensions/zai/index.test.ts` proves the wrapper omits `thinking.clear_thinking: false` unless `preserveThinking` is set.
- Is this the best way to solve the issue? yes. The provider already owns the opt-in flag and payload patch, so the replay decision should share that same source of truth instead of treating all Z.AI requests as preserved-thinking requests.
