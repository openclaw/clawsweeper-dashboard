# Commit d22da87

## Summary

Found one likely layout regression in the standalone PWA containment fix: safe-area body padding shrinks the app root, but the existing shell still sizes itself to the full dynamic viewport.

## Findings

### Medium: Standalone safe-area padding can clip the bottom of the app shell

- Kind: regression
- File: `ui/src/styles/base.css`
- Line: 557
- Evidence: In standalone mode, the commit fixes `body` to the viewport and adds `padding-top`/`padding-bottom` from `--safe-area-*`, then sets `openclaw-app` to `height: 100%` and `overflow: hidden` at `ui/src/styles/base.css:557` and `ui/src/styles/base.css:566`. Because `* { box-sizing: border-box; }` is active, the body content box becomes `100dvh - safe-area-top - safe-area-bottom`. But the actual app shell inside it still uses `height: 100dvh`, and `.shell--chat` also keeps `height: 100dvh` plus `min-height: 100vh` in `ui/src/styles/layout.css:13` and `ui/src/styles/layout.css:32`. The root clips overflow, so the bottom safe-area amount of the shell can be hidden rather than merely kept above the home indicator.
- Impact: On iOS standalone devices with a non-zero bottom safe area, bottom-anchored UI such as the chat composer can be partially clipped or shifted below the visible app root, undermining the Add-to-Home-Screen fix.
- Suggested fix: Under `@media (display-mode: standalone)`, make the shell consume the padded root instead of the viewport, for example override `.shell, .shell--chat { height: 100%; min-height: 0; }`, or move safe-area padding to a border-box `.shell` that itself owns the `100dvh` height.
- Confidence: medium

## Details

- Do we have a high-confidence way to reproduce the issue? Yes: launch the Control UI as an iOS standalone PWA on a device with non-zero safe-area insets and inspect the app root versus `.shell` heights; the static CSS arithmetic shows the shell remains taller than the padded root by the safe-area total.
- Is this the best way to solve the issue? No. The document lock is reasonable, but the safe-area contract needs the top-level app shell and chat shell to size against the padded containing block, not `100dvh`.
