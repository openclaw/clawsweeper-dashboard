# Commit e6cd90e

## Summary

Found two concrete regressions: one runtime auth persistence bug that can discard a newer local OAuth credential, plus one SDK API baseline gate failure.

## Findings

### Medium: Newer local OAuth credentials can be dropped as “inherited” during save

- Kind: data_loss
- File: `src/agents/auth-profiles/store.ts`
- Line: 84
- Evidence: `isInheritedMainOAuthCredential` treats any same-identity main OAuth credential as inherited when the agent has no already-persisted local profile. It does not check whether the credential being saved is newer than main. The existing test `src/agents/auth-profiles.ensureauthprofilestore.test.ts > keeps a newer agent replacement credential while repairing stale default references` now fails: the merged store returns `older-main-access` instead of the saved `newer-agent-access`.
- Impact: A secondary agent trying to persist a fresh OAuth profile for the same profile id/account can silently lose that new refresh token and continue using an older main-agent credential. This undermines separate-agent auth repair/login and can leave the agent on stale credentials.
- Suggested fix: Only filter a same-identity OAuth profile as inherited when the main credential is identical or at least as fresh as the candidate being saved. Preserve newer local credentials.
- Confidence: high

### Low: Plugin SDK API baseline was not updated for new public exports

- Kind: compatibility
- File: `src/plugin-sdk/agent-runtime.ts`
- Line: 48
- Evidence: This commit adds `findPersistedAuthProfileCredential` and `resolvePersistedAuthProfileOwnerAgentDir` to the public `agent-runtime` SDK export, but `pnpm plugin-sdk:api:check` fails with `Plugin SDK API baseline drift detected` and a hash mismatch for `docs/.generated/plugin-sdk-api-baseline.sha256`.
- Impact: The SDK API gate fails for an intentional public surface change, blocking CI/release validation until the generated baseline hash is updated.
- Suggested fix: Run `pnpm plugin-sdk:api:gen` and commit the generated baseline/hash changes, or remove the public exports if they were not intended as SDK surface.
- Confidence: high
