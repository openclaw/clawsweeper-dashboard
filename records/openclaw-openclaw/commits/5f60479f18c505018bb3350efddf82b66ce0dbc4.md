# Commit 5f60479

## Summary

The commit breaks the core TypeScript gate. The new transport-normalization calls put `config` and `workspaceDir` inside `ProviderNormalizeTransportContext`, but that public hook context only accepts `provider`, `api`, and `baseUrl`.

## Findings

### High: Core TypeScript no longer compiles

- Kind: regression
- File: `src/agents/pi-embedded-runner/model.ts`
- Line: 167
- Evidence: `pnpm tsgo:core` fails with `TS2353` at `src/agents/pi-embedded-runner/model.ts(167,7)` and `(317,7)`: `'config' does not exist in type 'ProviderNormalizeTransportContext'`. The declared type at `src/plugins/types.ts:541` only includes `provider`, `api`, and `baseUrl`.
- Impact: Core typecheck/CI fails on current `main`, blocking normal validation and release gates.
- Suggested fix: Either remove `config`/`workspaceDir` from the nested `context` objects and rely on the existing top-level hook params, or intentionally widen `ProviderNormalizeTransportContext` and update the plugin SDK contract/tests accordingly.
- Confidence: high

### Medium: Computed workspace fallback is dropped in direct registry resolution branches

- Kind: bug
- File: `src/agents/pi-embedded-runner/model.ts`
- Line: 968
- Evidence: `resolveModelWithRegistry` computes `workspaceDir` from `params.workspaceDir ?? cfg.agents.defaults.workspace` at `src/agents/pi-embedded-runner/model.ts:966`, but then calls `resolveExplicitModelWithRegistry(normalizedParams)`, `resolvePluginDynamicModelWithRegistry(normalizedParams)`, and `resolveConfiguredFallbackModel(normalizedParams)` without the computed value at lines 968, 994, and 999. Direct callers such as `src/commands/models/list.rows.ts:363`, `src/media-understanding/image.ts:157`, and `src/agents/btw.ts:234` pass `cfg` but no `workspaceDir`.
- Impact: Once the compile error is fixed, direct `resolveModelWithRegistry` callers can still run provider hooks without the config-derived workspace scope, which can resolve plugin/runtime state from the wrong workspace or miss workspace-scoped plugins.
- Suggested fix: Build a scoped params object, e.g. `{ ...normalizedParams, workspaceDir }`, and use it for every branch that calls resolver helpers.
- Confidence: medium
