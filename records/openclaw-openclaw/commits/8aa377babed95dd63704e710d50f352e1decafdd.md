# Commit 8aa377babe

## Summary

Found one responsive Control UI regression in the Sessions table. The commit’s mobile column-hiding rules are off by one because the real table includes the Runtime column, while the new browser fixture omits it.

## Findings

### Medium: Compaction disclosure is hidden on tablet and mobile widths

- Kind: regression
- File: `ui/src/styles/layout.mobile.css`
- Line: 831
- Evidence: The real Sessions table column order is checkbox, Key, Label, Kind, Runtime, Updated, Tokens, Compaction, Thinking, Fast, Verbose, Reasoning (`ui/src/ui/views/sessions.ts:618`). The new CSS hides `nth-child(8)` at `max-width: 900px` (`ui/src/styles/layout.mobile.css:829`), which is the Compaction column, not Thinking. A focused Playwright computed-style check with the real 12-column order showed `compactionDisplay: "none"` at 899px, 768px, 430px, and 375px; at 375px the visible headers were only Key, Tokens, and Reasoning.
- Impact: The commit’s main UI affordance, the compact `N Checkpoint(s)` disclosure, disappears on common tablet/mobile widths, so users cannot expand compaction/session details from the responsive Sessions table.
- Suggested fix: Recalculate the responsive `nth-child` rules against the 12-column table. Keep column 8 visible for Compaction, hide lower-priority columns instead, and update `ui/src/ui/views/sessions.browser.test.ts` so its fixture includes the Runtime column.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Render the Sessions table with the real 12-column order and current CSS at width `899px` or below; `.session-compaction-col` computes to `display: none`.
- Is this the best way to solve the issue? yes. The bug is in CSS column indexing; the test missed it because its headers omit Runtime at `ui/src/ui/views/sessions.browser.test.ts:37`.
