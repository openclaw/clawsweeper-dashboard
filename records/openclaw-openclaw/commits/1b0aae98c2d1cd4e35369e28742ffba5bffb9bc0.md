# Commit 1b0aae98c2d

## Summary

Found one low-severity test portability regression. The production code is unchanged, but the tightened assertion now hard-codes Linux’s default temp directory even though the implementation intentionally uses Node’s platform/env-dependent `os.tmpdir()`.

## Findings

### Low: Temp-dir assertion fails outside default `/tmp`

- Kind: compatibility
- File: `src/plugins/plugin-control-plane-context.test.ts`
- Line: 107
- Evidence: The test expects `stock: "/tmp/openclaw-empty-bundled-plugins"`, while the implementation builds the disabled bundled-plugin root with `path.join(os.tmpdir(), "openclaw-empty-bundled-plugins")` in `src/plugins/bundled-dir.ts:10`, reached through `resolvePluginSourceRoots` in `src/plugins/roots.ts:22`. Node documents `os.tmpdir()` as overrideable by `TMPDIR`, `TMP`, and `TEMP` on non-Windows, and by `TEMP`/`TMP` on Windows. ([nodejs.org](https://nodejs.org/api/os.html))
- Impact: The affected test fails when the temp dir is not `/tmp`, including ordinary Linux environments with `TMPDIR` set and likely Windows/macOS lanes or local developer runs. I reproduced this with `TMPDIR=/var/tmp pnpm test src/plugins/plugin-control-plane-context.test.ts`; the received value was `/var/tmp/openclaw-empty-bundled-plugins`.
- Suggested fix: Derive the expected value with `path.join(os.tmpdir(), "openclaw-empty-bundled-plugins")`, or assert the invariant without baking in the temp root.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes: `TMPDIR=/var/tmp pnpm test src/plugins/plugin-control-plane-context.test.ts` fails at `src/plugins/plugin-control-plane-context.test.ts:103`.
