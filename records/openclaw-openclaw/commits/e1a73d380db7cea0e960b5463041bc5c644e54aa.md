## Summary

Found one concrete test-gate weakness in the new reviewed critical finding allowlist.

## Findings

### Medium: Reviewed critical allowlist ignores the actual finding evidence

- Kind: test_gap
- File: `src/plugins/npm-install-security-scan.release.test.ts`
- Line: 145
- Evidence: The reviewed key is only `${plugin.packageName}:${finding.ruleId}:${packedPath}`. It ignores `finding.line` and `finding.evidence`, even though unreviewed failures include both at line 150. The scanner records line/evidence for each finding in `src/security/skill-scanner.ts:280`, then stops after the first line-rule match per file at `src/security/skill-scanner.ts:289`.
- Impact: A published plugin file that is already allowlisted can change the concrete critical command execution finding without making this release test fail. For example, replacing or inserting a different first `dangerous-exec` call in `@openclaw/codex:dangerous-exec:src/app-server/transport-stdio.ts` still produces the same allowlist key and is treated as reviewed. I verified with `scanSource` that `spawn("codex", [])` and `execFileSync("sh", ["-c", "echo pwn"])` in the same file both produce the same package/rule/path identity while the evidence changes.
- Suggested fix: Make `REVIEWED_PUBLISHABLE_CRITICAL_FINDINGS` snapshot the exact reported finding, at least `packageName`, `ruleId`, packed path, line, and evidence/message. Alternatively compare full normalized finding objects and assert duplicates/counts so a changed or newly earlier critical finding requires explicit review.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. The focused `pnpm tsx --eval ... scanSource(...)` check showed the same file/rule identity with different critical evidence; the test would only compare the shared identity.
- Is this the best way to solve the issue? yes. The commit’s intent is to require reviewed critical findings, and exact finding snapshots are the narrowest way to preserve that intent without blocking unrelated non-critical scanner output.
