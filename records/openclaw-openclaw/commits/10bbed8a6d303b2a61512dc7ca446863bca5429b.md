# Commit 10bbed8

## Summary

Found one cleanup regression in the new retained overflow preview path.

## Findings

### Medium: Retained overflow draft pages survive unfinalized cleanup

- Kind: bug
- File: `extensions/telegram/src/draft-stream.ts`
- Line: 252
- Evidence: When an over-limit draft page is completed, `sendOrEditStreamMessage` emits `retain: true` for the old `streamMessageId` and resets the stream to a new active message. `clear()` later deletes only the current `streamMessageId` at `extensions/telegram/src/draft-stream.ts:333`, while dispatch intentionally ignores retained previews at `extensions/telegram/src/bot-message-dispatch.ts:444` and calls `stream.clear()` for unfinalized lanes at `extensions/telegram/src/bot-message-dispatch.ts:1367`. A focused `node --import tsx` probe against the actual modules sent message 1 and message 2 for an overflow preview, then `clear()` deleted only message 2; retained message 1 was left behind.
- Impact: If an overflow preview is abandoned before it becomes the final reply, for example after a dispatch error or final delivery falling back to a non-stream payload, Telegram can keep stale partial draft content in the chat. The commit’s intended retained pages are correct only for finalized continuation messages; unfinalized cleanup needs to own and remove them.
- Suggested fix: Track retained overflow message IDs in the draft stream or in the dispatch lane, and delete them when `clear()`/unfinalized cleanup runs. Keep them only after successful finalization. Add tests for `clear()` after an overflow chain and for a dispatch fallback path after a retained page exists.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Use `createTelegramDraftStream` with a small `maxChars` and a renderer that expands text, call `update()`/`flush()` to create two preview messages, then call `clear()`; only the active tail message is deleted.
- Is this the best way to solve the issue? yes on direction. The retained page needs lifecycle ownership so cleanup can distinguish finalized continuation pages from abandoned draft pages.
