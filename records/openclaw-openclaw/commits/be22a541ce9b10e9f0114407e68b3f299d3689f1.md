# Commit be22a54

## Summary

Found one remaining regression in the blank tool-name guard: a final assistant message with both an allowed tool call and a blank tool call is classified as allowed, so the blank call is still returned for dispatch.

## Findings

### Medium: Mixed allowed-plus-blank tool calls bypass blank-name rejection

- Kind: reliability
- File: `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts`
- Line: 707
- Evidence: `classifyToolCallMessage` records `sawBlankStringToolCall` for blank names, but returns `{ kind: "allowed" }` whenever any block resolves to an allowed tool before it checks the blank-name malformed case. `wrapStreamTrimToolCallNames` only rewrites final unrecoverable blanks when the classifier returns `malformed`, so a result containing `read` plus `name: ""` keeps both tool calls. A focused `node --import tsx` probe against current `main` returned `{"content":[{"name":"read"},{"name":""}]}` for exactly that mixed final message.
- Impact: providers that emit parallel/multiple tool calls can still dispatch or persist a blank tool call whenever the same turn also contains a valid call, preserving the `Tool  not found` retry-loop class this commit intended to close.
- Suggested fix: make blank final tool calls dominate the `allowed` classification, or classify per-block and rewrite/drop only the malformed blank blocks before dispatch. Add a regression test for a final message containing one allowlisted call and one unrecoverable blank call.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes; use `wrapStreamFnTrimToolCallNames` with `allowedToolNames = new Set(["read"])` and a final assistant message containing `{ name: "read" }` plus `{ name: "" }`. The wrapper returns both tool calls instead of rewriting to text.
- Is this the best way to solve the issue? yes; the bug is in whole-message classification order, so the fix should keep id-based recovery while ensuring unrecoverable blank calls cannot be masked by sibling allowed calls.
