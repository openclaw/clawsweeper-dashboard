# Commit bc735f4

## Summary

Found two concrete regressions in the new `openclaw path set` surface.

## Findings

### Medium: Markdown and YAML `path set` can persist the redaction sentinel

- Kind: data_loss
- File: `src/commands/path.ts`
- Line: 141
- Evidence: `pathSetCommand` expects `emitForKind(result.ast, ...)` to reject sentinel-bearing values, but `emitForKind` calls `emitYaml(ast, opts)` and `emitMd(ast, opts)` without render/strict mode. Those default round-trip emitters trust `ast.raw`, and both markdown/YAML edit paths pre-render caller-provided values into `raw`.
- Impact: `openclaw path set` can write `__OPENCLAW_REDACTED__` into markdown workspace files and YAML files, despite the documented emit guard. Live smoke: setting `oc://AGENTS.md/Section/token/token` and `oc://config.yaml/token` to that value exited 0 and persisted the sentinel; the same JSONC smoke correctly exited 1 with `OC_EMIT_SENTINEL`.
- Suggested fix: Make mutated markdown/YAML writes go through render/strict sentinel checking before returning bytes, ideally in `setMdOcPath`/`setYamlOcPath` as well as the CLI boundary. Add CLI and universal `setOcPath` tests for md/yaml sentinel writes.
- Confidence: high

### Medium: Markdown `set` rejects positional and ordinal item paths that `resolve` accepts

- Kind: regression
- File: `src/oc-path/edit.ts`
- Line: 65
- Evidence: `resolveMdOcPath` handles `#N`, `$first`, `$last`, and `-N` for markdown items, but `setMdOcPath` only lowercases `path.item` and compares it to item slugs. Live smoke: `openclaw path resolve 'oc://AGENTS.md/tools/#1/foo'` and `$last` both resolved, while `openclaw path set` on the same paths returned `{"reason":"unresolved"}`.
- Impact: the advertised positional/ordinal addressing cannot edit markdown items, including duplicate-slug cases that `#N` was added to disambiguate.
- Suggested fix: Share the markdown item-selection logic between resolve and edit, or add equivalent ordinal/positional handling before choosing `itemIdx`. Add tests for markdown `set` with `#N`, `$first`, `$last`, and `-N`.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Both findings were reproduced with `pnpm openclaw path ...` in temporary directories.
- Is this the best way to solve the issue? yes. The fix should be at the write/edit boundary so both CLI and direct substrate callers get the same protection and addressing semantics.
