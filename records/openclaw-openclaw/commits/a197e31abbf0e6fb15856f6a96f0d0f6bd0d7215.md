# Commit a197e31a

## Summary

Found two actionable user-visible regressions that still exist on current `main`: embedded Codex onboarding can hide the itemized migration preview and warnings before apply, and the public migrate docs still describe the removed `Skip for now` selector behavior.

## Findings

### Medium: Embedded onboarding suppresses the migration plan and warnings before apply

- Kind: regression
- File: `src/wizard/setup.post-install-migration.ts`
- Line: 153
- Evidence: the post-install migration helper now calls `migrateDefaultCommand(..., { provider, suppressPlanLog: true })`. That reaches `migratePlanCommand`, where `formatMigrationPlan(plan)` is skipped when `suppressPlanLog` is true. `formatMigrationPlan` is the only pre-apply output that includes warnings, counts for conflicts/sensitive items, and the first 25 itemized plan entries.
- Impact: in the interactive onboarding path, a user can accept `Migrate Codex at <source> into this agent now?`, choose recommended skills/plugins, and reach `Apply this migration now?` without seeing provider warnings or the itemized plan that the migrate safety model normally shows before mutation. This weakens review for archive/config/manual/sensitive items not represented in the checkbox rows.
- Suggested fix: do not suppress the formatted plan unless an equivalent embedded preview/warnings view is shown before the final apply confirmation. If the intent is only to avoid duplicate noise for Codex selectors, at minimum log warnings and non-selector plan summary/items before `Apply this migration now?`.
- Confidence: high

### Low: Public migrate docs still document the removed Skip row

- Kind: regression
- File: `docs/cli/migrate.md`
- Line: 129
- Evidence: current docs still say interactive Codex migration previews the full plan, that `Skip for now` skips skill copies while continuing to plugin selection, and that plugin `Skip for now` stops before applying. The commit removed `MIGRATION_SELECTION_SKIP`, removed both picker rows, and added `Accept recommended`.
- Impact: users following the CLI docs will look for a selector action that no longer exists and may misunderstand the new default Enter path.
- Suggested fix: update `docs/cli/migrate.md` to describe `Accept recommended`, the removed skip row, and the current “Toggle all off” exit behavior.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Source inspection traces the onboarding call into `migrateDefaultCommand`, and `docs/cli/migrate.md` still contains the stale selector text.
- Is this the best way to solve the issue? yes for the docs fix; for the plan suppression, either restore the normal plan log or replace it with a concise embedded warnings/plan preview before apply.
