# Commit e44d907

## Summary

Found one low-severity execution bug in the new additional-mount file URL path: the guard accepts `file:///agent/...`, but the wrapped read tool still receives the original file URL and fails.

## Findings

### Low: `file:///agent/...` is validated but not normalized before read execution

- Kind: bug
- File: `src/agents/pi-tools.read.ts`
- Line: 640
- Evidence: `mapContainerPathToRoot` now maps `file:///agent/...` for `additionalContainerMounts`, and the added test asserts that guard mapping. But `wrapToolWorkspaceRootGuardWithOptions` only rewrites the actual tool args when `normalizeGuardedPathParams` is true; the sandboxed read wiring in `src/agents/pi-tools.ts:618` does not set it. A focused smoke using the new wrapper with `file:///agent/url.txt` failed with `Sandbox FS error (ENOENT): .../sandbox/file:/agent/url.txt`. The same smoke passed when `normalizeGuardedPathParams: true` was set.
- Impact: callers using the file URL form that the new guard test explicitly covers still cannot read the read-only `/agent` mount. Plain `/agent/...` works, so this is a narrow compatibility hole rather than a broader sandbox regression.
- Suggested fix: normalize matched guarded path params for sandboxed read, or teach sandboxed read path resolution to consume the same mapped candidate before calling upstream `createReadTool`. Add an execute-level regression for `file:///agent/...`, not just an `assertSandboxPath` call assertion.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. The direct wrapper smoke against `createSandboxedReadTool` and `wrapToolWorkspaceRootGuardWithOptions` reproduces the `file:///agent/...` failure; enabling `normalizeGuardedPathParams` makes the same smoke pass.
- Is this the best way to solve the issue? unclear. Normalizing guarded params is the smallest proven direction, but maintainers should confirm whether all sandboxed read display/details behavior can tolerate receiving host paths instead of original container/file URL strings.
