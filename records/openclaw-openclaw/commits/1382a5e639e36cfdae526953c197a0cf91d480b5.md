# Commit 1382a5e

## Summary

Found one concrete regression: the fix avoids IPv6 on IPv4-working hosts by hiding AAAA records from automatic pinned lookups, but that also removes Node/undici Happy Eyeballs fallback for dual-stack hosts when IPv4 is unavailable and IPv6 works.

## Findings

### Medium: Automatic pinned lookups no longer fall back to IPv6 for dual-stack hosts

- Kind: compatibility
- File: `src/infra/net/ssrf.ts`
- Line: 348
- Evidence: `automaticRecords` is set to IPv4-only whenever any IPv4 record exists, and the non-explicit family path uses `automaticRecords` for both single-address and `{ all: true }` lookups. Node and undici call custom lookup functions with `{ family: undefined, hints: 32, all: true }` when `autoSelectFamily` is enabled, so this now returns only A records even when validated AAAA records are present.
- Impact: Guarded fetch/model/provider paths can fail on IPv6-only or IPv4-broken networks for ordinary dual-stack endpoints. Before this commit, the automatic `{ all: true }` response exposed both pinned A and AAAA records, letting Node’s auto-family connection logic reach IPv6 if IPv4 failed.
- Suggested fix: Preserve both families for `{ all: true }` automatic lookups while preventing round-robin single-address calls from rotating to IPv6 by default. Alternatively, only force IPv4 under a scoped WSL2/IPv4-preferred policy and keep Happy Eyeballs inputs intact elsewhere.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. A local smoke with an HTTP server bound only to `::1`, a pinned lookup containing `127.0.0.1` and `::1`, and Node `http.get(..., autoSelectFamily: true)` fails on current main with `ECONNREFUSED`. The same smoke using the pre-patch lookup behavior succeeds with `status:200`.
- Is this the best way to solve the issue? no. The EADDRNOTAVAIL classification is fine, and default single-address lookup should avoid rotating onto IPv6 for the reported WSL2 class, but filtering AAAA out of automatic `{ all: true }` removes the fallback that auto-family selection is designed to provide.
