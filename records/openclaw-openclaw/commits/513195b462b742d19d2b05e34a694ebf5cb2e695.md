# Commit 513195b

## Summary

Found one actionable regression in the Prometheus diagnostics exporter: the new async queue drop metric path is unreachable for the real queue-drop event emitted by core.

## Findings

### Medium: Prometheus never reports real async diagnostic queue drops

- Kind: regression
- File: `extensions/diagnostics-prometheus/src/service.ts`
- Line: 719
- Evidence: `recordDiagnosticEvent` returns immediately for any `!metadata.trusted` event at `extensions/diagnostics-prometheus/src/service.ts:451`, but the core drop summary is dispatched as `{ trusted: false }` at `src/infra/diagnostic-events.ts:949`. The new `case "diagnostic.async_queue.dropped"` therefore only works in the new unit test because the test calls `recordDiagnosticEvent(..., trusted)` directly, not through the actual queue overflow path.
- Impact: Prometheus users still get no `openclaw_diagnostic_async_queue_dropped_total` or queue-length metric when async diagnostics overflow, which defeats the commit’s stated behavior for that exporter.
- Suggested fix: allow this specific internal summary event through the Prometheus trust gate, or change the core summary dispatch to carry trusted internal metadata while preserving the public stream behavior through a separate public/internal distinction. Add a test that overflows the real async queue, starts `createDiagnosticsPrometheusExporter()`, drains diagnostics, and asserts the rendered metrics include `openclaw_diagnostic_async_queue_dropped_total`.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. I started `createDiagnosticsPrometheusExporter()`, emitted 10,001 async `model.call.started` events with `emitDiagnosticEvent`, awaited `waitForDiagnosticEventsDrained()`, and `exporter.render()` produced `<no async queue metrics>`.
- Is this the best way to solve the issue? yes, directionally. The exporter must distinguish the core-generated queue summary from ordinary untrusted plugin-emitted diagnostics; otherwise the metric added by this commit remains unreachable in production.
