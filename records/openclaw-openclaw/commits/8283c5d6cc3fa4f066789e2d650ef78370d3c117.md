# Commit 8283c5d

## Summary

Found one concrete regression: web-search and web-fetch plugin provider discovery no longer cold-loads provider registries when no compatible active runtime registry exists.

## Findings

### Medium: Web provider discovery returns no providers outside a preloaded runtime registry

- Kind: regression
- File: `src/plugins/web-provider-runtime-shared.ts`
- Line: 195
- Evidence: The non-setup path now checks `getLoadedRuntimePluginRegistry(...)`, handles in-flight/empty scope, then unconditionally returns `[]`. In the parent commit, the same path fell back to `loadOpenClawPlugins(loadOptions)` and mapped providers from that snapshot. Focused tests reproduce the break: `pnpm test src/plugins/web-search-providers.runtime.test.ts src/plugins/web-fetch-providers.runtime.test.ts` fails 11 tests, including `resolvePluginWebSearchProviders({})` returning `[]` instead of bundled providers and `resolvePluginWebFetchProviders({})` returning `[]` instead of `firecrawl:firecrawl`.
- Impact: Standalone or cold provider-discovery paths that do not already have a compatible active registry lose web_search/web_fetch providers. That can break provider auto-detection, configured-provider listing, onboarding/setup fallback discovery, and runtime definition resolution in cold CLI/test paths.
- Suggested fix: Restore a non-in-flight cold fallback for non-empty scopes, likely by loading `loadOpenClawPlugins(loadOptions)` in `resolvePluginWebProviders` when no compatible active registry exists, while keeping the in-flight guard and explicit-empty-scope behavior.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes: run `pnpm test src/plugins/web-search-providers.runtime.test.ts src/plugins/web-fetch-providers.runtime.test.ts`; the failures show provider arrays are empty and `loadOpenClawPlugins` is never called.
- Is this the best way to solve the issue? yes, directionally: the parent behavior already had the correct cold fallback, and the current tests still encode that contract.
