# Commit 984174f

## Summary

Found one actionable security/privacy risk in the new `publish --full-artifacts` path. The default GIF-only mode behaved as intended in a local smoke, but the full-artifacts mode can publish the unsanitized session directory.

## Findings

### High: `--full-artifacts` can publish pre-finish Telegram credential material

- Kind: security
- File: `scripts/e2e/telegram-user-crabbox-proof.ts`
- Line: 1590
- Evidence: `publishSessionArtifacts` sets `publishDir` to `session.outputDir` whenever `opts.publishFullArtifacts` is true, without requiring `finish` to have run or building a sanitized bundle. A held session creates `localRoot = path.join(outputDir, ".session")`, and `leaseCredential` writes `payload.json` there via `--payload-output`. That payload can contain `sutToken`, `telegramApiHash`, `tdlibDatabaseEncryptionKey`, TDLib archive data, and Telegram Desktop tdata archive data from `scripts/e2e/telegram-user-credential.ts`.
- Impact: If an operator runs `publish --session <file> --pr <number> --full-artifacts` before `finish`, the publisher receives the whole session directory, including `.session/**` unless the external publisher independently filters it. That can expose bot tokens and Telegram account/session material through a PR artifact/comment.
- Suggested fix: Make `publish` require a completed session summary even in `--full-artifacts` mode, and publish from a freshly created allowlisted bundle. Exclude `session.json`, `.session/**`, lease files, payload files, TDLib databases, Telegram Desktop profiles, and raw remote-state archives.
- Confidence: medium

## Details

- Do we have a high-confidence way to reproduce the issue? Yes for the unsafe directory selection: a local fake-publisher smoke showed `--full-artifacts` passes the session root as `--dir`. The actual GitHub upload was not run because this is report-only and `crabbox` is unavailable here.
- Is this the best way to solve the issue? Yes: filtering at the OpenClaw wrapper is safer than relying on operator ordering or undocumented `crabbox artifacts publish` exclusions.
