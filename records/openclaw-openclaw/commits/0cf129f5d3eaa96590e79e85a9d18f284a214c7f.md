# Commit 0cf129f

## Summary

Found one low-severity catalog regression: the new Venice manifest catalog exposes stale model ids through manifest-backed model listing surfaces.

## Findings

### Low: Venice manifest catalog lists models absent from the live Venice catalog

- Kind: bug
- File: `extensions/venice/openclaw.plugin.json`
- Line: 155
- Evidence: The commit declares `qwen3-4b`, `mistral-31-24b`, `minimax-m21`, `gemini-3-pro-preview`, and `grok-code-fast-1` in `modelCatalog.providers.venice.models` at lines 155, 190, 311, 443, and 487. A live `curl https://api.venice.ai/api/v1/models?type=all` comparison showed all five ids absent from Venice’s current text model list, while `pnpm openclaw models list --all --provider venice --json` still emitted all five as Venice model choices. Venice documents `/api/v1/models` as the available-model listing endpoint. ([docs.venice.ai](https://docs.venice.ai/api-reference/endpoint/models/list))
- Impact: Users can see and select manifest-backed Venice models that the provider no longer advertises, leading to failed inference/configuration paths. This is newly exposed beyond the older runtime fallback because the commit puts those stale rows into cold manifest metadata used by list/picker surfaces.
- Suggested fix: Refresh the manifest rows from the current Venice model list before declaring them, or omit unavailable ids from `modelCatalog`; if a removed id must remain for migration, mark it as deprecated/disabled only where consumers will not offer it as an available choice.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Compare manifest ids with `curl -fsS 'https://api.venice.ai/api/v1/models?type=all'`, then run `pnpm openclaw models list --all --provider venice --json`; the five stale ids appear in OpenClaw output but not in the live Venice model response.
- Is this the best way to solve the issue? yes. The bug is bad static metadata; the narrow fix is to remove or correctly status those rows rather than changing generic manifest catalog behavior.
