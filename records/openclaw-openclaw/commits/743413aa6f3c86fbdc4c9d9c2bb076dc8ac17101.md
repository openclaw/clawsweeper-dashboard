# Commit 743413a

## Summary

Found one retry-path regression. The commit fixes one-off model overrides by passing a sanitized `sessionEntryForAttempt`, but that snapshot is created once before the live-switch retry loop. Later live model switches update `sessionEntry`, not `sessionEntryForAttempt`, so retries can lose the selected auth profile.

## Findings

### Medium: Live model-switch retries can drop the switched auth profile

- Kind: regression
- File: `src/agents/agent-command.ts`
- Line: 1045
- Evidence: `sessionEntryForAttempt` is initialized before the attempt loop and is what `runAgentAttempt` receives. On `LiveSessionModelSwitchError`, the catch block updates `sessionEntry.authProfileOverride` and `sessionEntry.authProfileOverrideSource`, but never updates `sessionEntryForAttempt`. `runAgentAttempt` then reads `params.sessionEntry?.authProfileOverride` to build the runtime auth plan in `src/agents/command/attempt-execution.ts`.
- Impact: a live switch that includes `authProfileId` can retry on the new provider/model without the requested auth profile. Depending on auth order, this can run under the wrong account/profile or fail auth entirely. The existing test named for propagating the auth profile only checks `authProfileProvider`, not the retried `sessionEntry.authProfileOverride`, so it does not catch this.
- Suggested fix: recompute or update the per-attempt session entry whenever `sessionEntry` is reassigned for retry, especially after the live-switch block sets `authProfileOverride`. Keep the one-off override sanitization, but do not freeze the snapshot across live-switch retries.
- Confidence: medium

## Details

- Do we have a high-confidence way to reproduce the issue? Yes: trigger a live model switch that throws `LiveSessionModelSwitchError` with `authProfileId`, then assert the second `runAgentAttempt` receives that profile in `sessionEntry`.
- Is this the best way to solve the issue? Yes: the fix should preserve the intended transient sanitization while keeping retry state current; updating `sessionEntryForAttempt` in the live-switch path is the narrowest direction.
