# Commit b27bae3

## Summary

Found one concrete regression risk: the Gemini API-key onboarding default now points fresh users at a paid-only Gemini Pro Preview model, replacing the prior free-tier-friendly Flash default.

## Findings

### Medium: API-key onboarding now defaults fresh users to a paid-only Gemini model

- Kind: regression
- File: `extensions/google/onboard.ts`
- Line: 6
- Evidence: The commit changes `GOOGLE_GEMINI_DEFAULT_MODEL` from `google/gemini-2.5-flash` to `google/gemini-3.1-pro-preview`. `buildGoogleProvider()` wires that constant into the Google API-key auth method as both `defaultModel` and `applyConfig`, and the provider auth path applies it to `agents.defaults.model.primary` during onboarding. Google’s current pricing page lists Gemini 3.1 Pro Preview with no free tier, while Gemini 2.5 Flash has a free tier. citeturn0view1
- Impact: A fresh user running `openclaw onboard --auth-choice gemini-api-key` with a normal AI Studio API key but no paid billing can end up with a default model that fails on first use for billing/quota reasons. The previous default avoided that by choosing `google/gemini-2.5-flash`.
- Suggested fix: Keep API-key onboarding on a generally available/free-tier model such as `google/gemini-2.5-flash` or another verified free-tier Gemini default, and move Gemini 3.1 Pro Preview coverage to explicit live/test configuration or an opt-in onboarding choice. If Pro must be the default, add a fallback/free-tier default for unbilled API-key users.
- Confidence: medium

## Details

- Do we have a high-confidence way to reproduce the issue? No live reproduction here; after sourcing `~/.profile`, `GEMINI_API_KEY` and `GOOGLE_API_KEY` were missing. The source path and Google pricing docs give a concrete failure mode for unbilled API-key users.
- Is this the best way to solve the issue? Yes: onboarding defaults should work for the broadest fresh-user credential path. Test coverage for Gemini 3.1 Pro should be separate from the default user model unless onboarding can detect paid-tier eligibility.
