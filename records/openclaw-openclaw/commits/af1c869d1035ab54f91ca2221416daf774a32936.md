# Commit af1c869

## Summary

Found one remaining provider-policy reachability bug: the new Anthropic artifact hook fixes `anthropic/*`, but the added `claude-cli` branch is not reachable through the bundled policy resolver, so non-runtime callers still see only base thinking levels for `claude-cli/claude-opus-4-7`.

## Findings

### Medium: Claude CLI Opus 4.7 still loses bundled `max`/`xhigh`/`adaptive` thinking levels

- Kind: bug
- File: `extensions/anthropic/provider-policy-api.ts`
- Line: 19
- Evidence: The commit adds `case "claude-cli"` to `resolveThinkingProfile`, but non-runtime callers do not import this artifact directly. They call `resolveBundledProviderPolicySurface(provider)`, whose manifest fallback only checks `plugin.providers`. The Anthropic manifest has `providers: ["anthropic"]` and `cliBackends: ["claude-cli"]`, so `resolveBundledProviderPolicySurface("claude-cli")` returns no Anthropic policy artifact. A focused source probe on current `main` confirms the split: `listThinkingLevels("anthropic", "claude-opus-4-7")` returns `off,minimal,low,medium,adaptive,high,xhigh,max`, while `listThinkingLevels("claude-cli", "claude-opus-4-7")` returns only `off,minimal,low,medium,high`.
- Impact: Cron/session/default validation paths that do not have the full runtime registry active can still reject or remap `/think max`, `/think xhigh`, and `/think adaptive` for the default Claude CLI Opus 4.7 model, despite the new hook appearing to handle `claude-cli`.
- Suggested fix: Either make bundled provider-policy ownership resolve CLI backends as well as `providers[]` (matching `resolveOwningPluginIdsForProvider` behavior), or add an explicit manifest-level policy ownership mapping for `claude-cli`. Add a resolver-level regression test such as `listThinkingLevels("claude-cli", "claude-opus-4-7")` or `resolveBundledProviderPolicySurface("claude-cli")`.
- Confidence: medium

## Details

- Do we have a high-confidence way to reproduce the issue? yes: `pnpm exec tsx -e "import { listThinkingLevels } from './src/auto-reply/thinking.ts'; console.log(listThinkingLevels('claude-cli', 'claude-opus-4-7').join(','));"` prints `off,minimal,low,medium,high` on current `main`.
- Is this the best way to solve the issue? unclear: the safer architectural fix is likely in the bundled policy owner resolver, but maintainers may prefer keeping `cliBackends` out of policy ownership and declaring an explicit policy alias instead.
