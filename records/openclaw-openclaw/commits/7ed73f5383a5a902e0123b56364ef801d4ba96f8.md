# Commit 7ed73f5

## Summary

Found one low-severity test coverage bug in the added plugin-update E2E assertions. The new npm and ClawHub update checks can pass while validating pre-update plugin state, so they do not fully prove the update no-op contract they were added to cover.

## Findings

### Low: Update assertions reuse pre-update plugin snapshots

- Kind: test_gap
- File: `scripts/e2e/lib/plugins/assertions.mjs`
- Line: 389
- Evidence: `scripts/e2e/lib/plugins/sweep.sh:80`-`88` captures npm list/inspect/CLI output before `plugins update demo-plugin-npm`, then `assertNpmPluginUpdateUnchanged()` checks the update log and calls `assertNpmPlugin()`, which reads those pre-update files. Similarly, `scripts/e2e/lib/plugins/clawhub.sh:52`-`56` writes `/tmp/plugins-clawhub-updated.json` and `/tmp/plugins-clawhub-updated-inspect.json`, but `assertClawHubUpdated()` only checks the update log and calls `assertClawHubInstalled()`, which reads `/tmp/plugins-clawhub-installed.json` and `/tmp/plugins-clawhub-inspect.json`.
- Impact: Regressions where `plugins update` exits successfully and prints the expected no-op message but corrupts enabled state, status, inspect output, or runtime command visibility after the update would not be caught by these new checks. The ClawHub updated snapshots are currently dead writes.
- Suggested fix: After npm update, capture fresh list/inspect/CLI output and validate those files. For ClawHub, make `assertClawHubUpdated()` validate `/tmp/plugins-clawhub-updated.json` and `/tmp/plugins-clawhub-updated-inspect.json` directly, or parameterize the installed-state assertion with file paths.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. This is directly reproducible by inspection: the post-update files are either never generated for npm or generated but never read for ClawHub.
- Is this the best way to solve the issue? yes. The safest fix is to point the assertions at post-update artifacts and rerun the plugin Docker lane or targeted helper tests.
