# Commit c136bb0

## Summary

The commit excludes Discord’s built plugin tree from the root npm package, but Discord is still declared as an in-core bundled plugin. Current release/package validation still requires 29 `dist/extensions/discord/**` artifacts, so this one-line exclusion conflicts with the existing package contract and should break release preflight once `dist` is built.

## Findings

### Medium: Discord package exclusion conflicts with in-core bundled plugin metadata

- Kind: regression
- File: `package.json`
- Line: 43
- Evidence: The commit adds `!dist/extensions/discord/**` to the root package `files` list. But `extensions/discord/package.json:71` still has `"bundle": { "includeInCore": true }`, and `scripts/lib/optional-bundled-clusters.mjs:42` only skips plugin build artifacts when `includeInCore === false`; Discord also has an `openclaw.install.npmSpec`, which forces `shouldBuildBundledCluster()` to return true. `scripts/release-check.ts:48` includes `...listBundledPluginPackArtifacts()` in required pack paths, and `scripts/release-check.ts:793` fails the release check when any required path is missing. A focused probe of `listBundledPluginPackArtifacts()` on current `main` reports 29 required `dist/extensions/discord/**` artifacts, and the new `files` rule excludes all 29.
- Impact: Release preflight/package validation should fail after a build because required Discord bundled artifacts are intentionally omitted by npm pack. If the pack validation is bypassed, the root package can still advertise/catalog Discord as bundled while the runtime files are absent, causing installed-package Discord setup/runtime paths to fail.
- Suggested fix: Choose one packaging model. Either remove `!dist/extensions/discord/**` so Discord remains in-core, or make Discord fully externalized by setting `openclaw.bundle.includeInCore` to `false` and updating release/package/catalog expectations so root package validation no longer requires Discord bundled artifacts.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. In a built checkout, `pnpm release:check` should fail with missing Discord pack artifacts. I did not run that broad/mutating check; instead I ran a focused non-mutating Node probe showing `listBundledPluginPackArtifacts()` requires 29 Discord artifacts while the new `package.json` exclude removes the whole `dist/extensions/discord/**` tree.
- Is this the best way to solve the issue? no. The one-line root package exclusion is incomplete because the Discord plugin metadata and release validator still model Discord as in-core.
