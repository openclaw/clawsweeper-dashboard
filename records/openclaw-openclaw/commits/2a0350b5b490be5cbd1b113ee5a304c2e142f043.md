## Findings

### Medium: Plugin SDK API baseline hash does not match the generated API surface

- Kind: regression
- File: `docs/.generated/plugin-sdk-api-baseline.sha256`
- Line: 1
- Evidence: This commit adds public SDK exports for `AgentPromptGuidance`, `AgentPromptGuidanceEntry`, and `AgentPromptSurfaceKind` in `src/plugin-sdk/core.ts`, `src/plugin-sdk/plugin-entry.ts`, and `src/plugins/types.ts`, and updates the tracked baseline hash. Re-running the repo’s source-of-truth check fails:
  `node --max-old-space-size=8192 --import tsx scripts/generate-plugin-sdk-api-baseline.ts --check`
  reports `Plugin SDK API baseline drift detected` and `Hash mismatch: docs/.generated/plugin-sdk-api-baseline.sha256`.
  The generator currently computes `e3b36491... plugin-sdk-api-baseline.json` and `7f4ca083... plugin-sdk-api-baseline.jsonl`, while the committed file contains `efac8656...` and `bf676f45...`.
- Impact: Any CI or maintainer gate that runs `plugin-sdk:api:check` fails on current `main`, and the committed public SDK baseline no longer represents the actual exported contract.
- Suggested fix: Run `pnpm plugin-sdk:api:gen` from the current dependency set and commit the regenerated `docs/.generated/plugin-sdk-api-baseline.sha256`.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes: run `node --max-old-space-size=8192 --import tsx scripts/generate-plugin-sdk-api-baseline.ts --check`.
- Is this the best way to solve the issue? yes: the change intentionally adds public SDK types, so the tracked generated hash should be refreshed rather than removing the exports.
