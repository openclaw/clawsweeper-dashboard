# Commit dd32254607

## Summary

Found one low-severity regression in Discord progress mode: the new text-only block suppression also drops opt-in compaction notices.

## Findings

### Low: Discord progress mode drops configured compaction notices

- Kind: regression
- File: `extensions/discord/src/monitor/message-handler.process.ts`
- Line: 433
- Evidence: The new guard returns for every progress-mode block payload that has no media and is not `isError`. Compaction notices are emitted as text-only block replies with `isCompactionNotice: true` when `agents.defaults.compaction.notifyUser` is enabled (`src/auto-reply/reply/agent-runner-execution.ts:944`). The config help promises those notices are sent to the user (`src/config/schema.help.ts:1331`). Matrix explicitly exempts `payload.isCompactionNotice` from draft-stream suppression (`extensions/matrix/src/matrix/monitor/handler.ts:1654`), which confirms this payload class is meant to bypass preview handling.
- Impact: Users who enable `agents.defaults.compaction.notifyUser` and `channels.discord.streaming.mode: "progress"` stop seeing “Compacting context...” / “Compaction complete” notices during long Discord runs. The dispatcher still counts the block as queued, so this fails silently.
- Suggested fix: Let compaction notice blocks through before suppressing text-only progress blocks, e.g. `if (!payload.isCompactionNotice && !reply.hasMedia && !payload.isError) return;`. Consider whether other non-progress block metadata such as explicit reply targets should also bypass the suppression.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? Yes. Configure Discord progress drafts plus `agents.defaults.compaction.notifyUser: true`, then trigger compaction. The agent runtime emits `isCompactionNotice` text-only block payloads; the new Discord guard returns before `deliverDiscordReply`.
- Is this the best way to solve the issue? No. The intent is to suppress interim progress text, but the current predicate is broader than that. Matching Matrix’s `!payload.isCompactionNotice` exemption is the narrowest safe fix.
