# Commit a0358bb

## Summary

Found one low-severity regression in the release E2E log assertion helper. The helper can now pass as soon as it sees the initial reload-detection log, before the asynchronous restart-required log has a chance to be written.

## Findings

### Low: Log assertion can pass before a restart-required line is emitted

- Kind: test_gap
- File: `scripts/e2e/lib/config-reload/assert-log.mjs`
- Line: 29
- Evidence: The polling loop breaks when either `restartLines` or `reloadLines` is non-empty. In current gateway code, `src/gateway/config-reload.ts:244` writes `config change detected; evaluating reload` before the restart path is scheduled, and `src/gateway/server-reload-handlers.ts:706` awaits `activateRuntimeSecrets(...)` before `requestGatewayRestart(...)` can log `config change requires gateway restart` at `src/gateway/server-reload-handlers.ts:541` or `src/gateway/server-reload-handlers.ts:593`. A focused temp-log simulation appended the reload line first and the restart-required line shortly after; `node scripts/e2e/lib/config-reload/assert-log.mjs` exited with status 0 even though the final log contained both lines.
- Impact: `pnpm test:docker:config-reload` can report success if the watched config change is delayed and then takes the restart path, because the helper may stop at the first “detected” proof and miss the later restart proof that it is supposed to reject.
- Suggested fix: Treat `config change requires gateway restart` as an immediate failure while polling, but do not treat the initial `config change detected; evaluating reload` line as terminal success. For this scenario, wait for a terminal hot-reload proof such as `config hot reload applied (gateway.channelHealthCheckMinutes)`, then scan the full log for restart-required lines and forbidden runtime-only diff paths.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. A focused temp-log check appending `config change detected; evaluating reload (gateway.port)` and then `config change requires gateway restart (gateway.port)` showed the helper exiting successfully before the second line was considered.
- Is this the best way to solve the issue? yes. The assertion’s contract is to prove a hot reload occurred without restart, so success should be tied to the hot-reload completion log, not the earlier reload-evaluation log.
