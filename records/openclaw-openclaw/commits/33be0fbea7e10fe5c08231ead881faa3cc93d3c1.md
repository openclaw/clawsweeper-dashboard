# Commit 33be0fb

## Summary

Found one low-severity compatibility edge case in the new `deactivate` hook alias: alias-specific per-hook timeout config is ignored.

## Findings

### Low: `hooks.timeouts.deactivate` cannot apply to aliased shutdown hooks

- Kind: bug
- File: `src/plugins/registry.ts`
- Line: 2354
- Evidence: `registerTypedHook` normalizes `hookName === "deactivate"` to `effectiveHookName === "gateway_stop"` at `src/plugins/registry.ts:2301`, then resolves timeout config using only the effective name at `src/plugins/registry.ts:2354`. `resolveTypedHookTimeoutMs` only reads `policy.timeouts[params.hookName]` at `src/plugins/registry.ts:251`. The new test covers global `hooks.timeoutMs`, but not `hooks.timeouts.deactivate` (`src/plugins/loader.test.ts:5925`).
- Impact: An operator following the documented `hooks.timeouts.<hookName>` contract for a legacy plugin that registers `api.on("deactivate", ...)` can set `plugins.entries.<id>.hooks.timeouts.deactivate`, but the registered `gateway_stop` handler will not receive that budget. Slow cleanup may then use the global timeout, plugin-authored timeout, or no per-hook timeout instead of the intended alias-specific shutdown budget.
- Suggested fix: Preserve both the registered hook name and canonical hook name for timeout resolution. For the alias path, accept `timeouts.deactivate` as well as `timeouts.gateway_stop`, with a clear precedence, and add a loader regression test using `hooks.timeouts: { deactivate: 250 }`.
- Confidence: medium

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Use the existing loader fixture shape from `src/plugins/loader.test.ts`, register `api.on("deactivate", ...)`, configure only `hooks.timeouts.deactivate`, and inspect `registry.typedHooks[0].timeoutMs`; the current resolver will look up `gateway_stop`, not `deactivate`.
- Is this the best way to solve the issue? yes. The alias should be normalized for execution, but policy lookup should still honor the alias spelling that the plugin registered and docs expose.
