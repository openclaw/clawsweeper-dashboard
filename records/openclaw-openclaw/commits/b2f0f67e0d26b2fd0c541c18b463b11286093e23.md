### Low: Ubuntu WSL import path is unusable on the configured Windows runner

- Kind: compatibility
- File: `.github/workflows/windows-testbox-probe.yml`
- Line: 111
- Evidence: The workflow imports with `wsl.exe --import UbuntuProbe ... --version 2` on `blacksmith-16vcpu-windows-2025`. An exact-SHA run for `b2f0f67e0d26b2fd0c541c18b463b11286093e23` with `IMPORT_UBUNTU_WSL2=true` reached that import and failed with `HCS_E_HYPERV_NOT_INSTALLED`, then `Enforce WSL2 requirement` failed with `OPENCLAW_WSL2_PROBE_OK=false`: https://github.com/openclaw/openclaw/actions/runs/25301054871
- Impact: The new `import_ubuntu_wsl2` option does not make the manual Windows probe pass with `require_wsl2` on the runner this workflow selects for `openclaw/openclaw`, so it cannot currently validate WSL2 availability by importing Ubuntu.
- Suggested fix: Run this workflow on a Windows runner image with Virtual Machine Platform/nested virtualization enabled, or add a pre-import capability check that reports the runner as unsupported before downloading/importing and avoids presenting import as a path to WSL2 success on this runner.
- Confidence: high
