# Commit 1c45592e62

## Summary

Found one regression in the empty plugin-load fast path. The new path activates an empty registry but does not clear all process-global plugin registrations that the cached activation path restores.

## Findings

### Medium: Empty activating plugin loads can leave stale plugin-owned providers registered

- Kind: regression
- File: `src/plugins/loader.ts`
- Line: 2238
- Evidence: The new `onlyPluginIds: []` early return clears harnesses, commands, interactive handlers, detached task runtime, and memory plugin state, then activates an empty registry. It skips `clearCompactionProviders()` and `clearMemoryEmbeddingProviders()`, even though the normal cached activation path restores those process-global registries from the cached snapshot at `src/plugins/loader.ts:2263` and `src/plugins/loader.ts:2266`. A read-only smoke confirmed current behavior: after loading a plugin that registers memory embedding provider `embedder` and compaction provider `compactor`, `loadOpenClawPlugins({ onlyPluginIds: [], cache: false })` returned `plugins.length === 0` but `getMemoryEmbeddingProvider("embedder")` and `listCompactionProviderIds()` still reported the stale providers.
- Impact: An activation-planner miss can make OpenClaw’s active plugin registry empty while stale plugin-owned embedding or compaction providers remain available through process-global lookup paths such as `src/gateway/embeddings-http.ts:138`, `src/agents/memory-search.ts:155`, and `src/agents/pi-hooks/compaction-safeguard.ts:858`. That can bypass plugin disablement/reload expectations and continue using a provider after the empty activation says no plugin is loaded.
- Suggested fix: Make the empty-scope activation use the same side-effect reset semantics as cached registry activation. At minimum clear or restore empty snapshots for compaction providers and memory embedding providers in the fast path; preferably factor the activation side-effect reset/restore code so empty, uncached, and cached paths cannot drift.
- Confidence: high
