# Commit 8a2207f

## Summary

Found one audit/loader mismatch introduced by the new installed-plugin-dir filter. The false-positive fix for `.openclaw-install-backups` is valid, but the audit now skips every dot-prefixed plugin root while runtime discovery can still discover dot-prefixed roots.

## Findings

### Medium: Security audit skips dot-prefixed plugin roots that runtime discovery still loads

- Kind: security
- File: `src/security/installed-plugin-dirs.ts`
- Line: 13
- Evidence: `shouldIgnoreInstalledPluginDirName()` returns true for any `normalized.startsWith(".")`, so both `collectPluginsTrustFindings()` and deep `collectPluginsCodeSafetyFindings()` ignore `stateDir/extensions/.shadow`. Runtime discovery does not have the same rule: `src/plugins/discovery.ts` only ignores specific names such as `.git`, `.hg`, `.svn`, `.turbo`, `.yarn`, `.yarn-cache`, `dist`, and `node_modules`, then processes other directories through manifest/index discovery. A temp-dir smoke confirmed `discoverOpenClawPlugins()` returns a candidate for `extensions/.shadow/index.js`, while `collectPluginsTrustFindings({ cfg: {}, stateDir })` returns no plugin audit findings. A second smoke with dangerous code under `extensions/.shadow/index.js` returned no code-safety findings.
- Impact: `openclaw security audit` and `openclaw security audit --deep` can report no plugin allowlist/tool-policy/code-safety issue for a hidden extension root that the plugin loader can still discover. That creates a blind spot for exactly the security audit surface this commit changes.
- Suggested fix: Make the audit filter match plugin discovery. Either narrow this helper to explicit debris names such as `.openclaw-install-backups` and known stage/cache names, or update plugin discovery/loader to reject all dot-prefixed extension roots too, with a regression test proving the runtime and audit enumerate the same install roots.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Create a temp `OPENCLAW_STATE_DIR` with `extensions/.shadow/openclaw.plugin.json` and `extensions/.shadow/index.js`; `discoverOpenClawPlugins()` sees `.shadow`, but the changed audit collectors skip it.
- Is this the best way to solve the issue? unclear. If hidden plugin dirs are meant to be un-loadable, the safer fix is to enforce that in discovery/loader. If they remain loadable, the audit should not blanket-ignore them.
