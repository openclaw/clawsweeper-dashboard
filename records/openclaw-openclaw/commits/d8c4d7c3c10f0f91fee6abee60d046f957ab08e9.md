# Commit d8c4d7c

## Summary

Found one regression in provider wizard metadata handling.

## Findings

### Medium: Method-level provider model allowlists are ignored

- Kind: regression
- File: `src/commands/configure.gateway-auth.ts`
- Line: 60
- Evidence: `resolveProviderPluginChoice` returns method-level wizard metadata for auth choices such as explicit method `choiceId`s (`src/plugins/provider-wizard.ts:268`-`273`), and the parent commit read `resolved?.wizard?.modelAllowlist`. This commit instead reads only `resolved?.provider.wizard?.setup` at `src/commands/configure.gateway-auth.ts:60`, so auth-method `wizard.modelAllowlist` is dropped. A bundled example exists at `extensions/anthropic/register.runtime.ts:494`-`505` for `anthropic-cli`.
- Impact: Provider plugins that put setup metadata on an auth method no longer get their configured allowlist, initial selections, or prompt message applied after `openclaw configure`. This breaks the plugin wizard contract and can leave `/model` picker defaults/fallbacks unscoped or unseeded for third-party method-level setup choices.
- Suggested fix: Prefer the resolved choice wizard before falling back to provider-level setup, e.g. `const wizard = resolved?.wizard ?? resolved?.provider.wizard?.setup`, and restore/add a method-level modelAllowlist test.
- Confidence: high
