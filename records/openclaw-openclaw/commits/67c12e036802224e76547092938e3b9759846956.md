# Commit 67c12e0368

## Summary

Found one regression in the benchmark budget wrapper: after relaxing missing baseline cases for narrowed presets, an invalid preset can now produce zero benchmark cases and still exit successfully.

## Findings

### Medium: Invalid benchmark presets can make the budget check pass with zero cases

- Kind: regression
- File: `scripts/test-cli-startup-bench-budget.mjs`
- Line: 127
- Evidence: The commit changed missing baseline cases to fail only when `opts.preset === "all"` (`scripts/test-cli-startup-bench-budget.mjs:127-139`). The underlying runner accepts arbitrary preset strings and filters cases by membership (`scripts/bench-cli-startup.ts:407-418`, `scripts/bench-cli-startup.ts:432-434`), so an unknown preset resolves to an empty case list. I reproduced this on current `main`: `node scripts/test-cli-startup-bench-budget.mjs --preset typo --runs 1 --warmup 0 --timeout-ms 1000` exited `0` and wrote a report with `caseCount: 0`. In the parent commit, the same empty current report would have failed because every missing baseline case was unconditional (`989e53c20d395d3c8bf47efc21fdb9d56e7227b0:scripts/test-cli-startup-bench-budget.mjs:130-136`).
- Impact: A typo, preset rename, or bad caller can turn the startup budget check into a vacuous pass, hiding performance or response-contract regressions while reporting success.
- Suggested fix: Validate `--preset` against the runner’s supported preset names, or at minimum fail when the generated current report has zero cases. For baseline comparison, skip only baseline cases that are outside the requested valid preset set, rather than treating every non-`all` preset as overlap-only.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Run `node scripts/test-cli-startup-bench-budget.mjs --preset typo --runs 1 --warmup 0 --timeout-ms 1000`; it exits `0` with an empty `primary.cases` array.
- Is this the best way to solve the issue? yes. The wrapper should reject invalid or empty benchmark selections before comparing budgets; that preserves the intended narrowed-preset behavior without allowing an empty suite to pass.
