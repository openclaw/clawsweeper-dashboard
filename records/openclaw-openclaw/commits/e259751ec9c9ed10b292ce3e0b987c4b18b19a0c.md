# Commit e259751

## Summary

Found one actionable issue: the new iMessage action toggles are enforced only during message-tool discovery, not during action execution.

## Findings

### Medium: Disabled iMessage private actions can still execute through direct dispatch

- Kind: security
- File: `extensions/imessage/src/actions.ts`
- Line: 378
- Evidence: `describeMessageTool` filters actions through `createActionGate(account.config.actions)` at `extensions/imessage/src/actions.ts:341`, and the test at `extensions/imessage/src/actions.test.ts:109` only verifies disabled actions disappear from discovery. The execution path starts at `extensions/imessage/src/actions.ts:378` and then dispatches each action branch directly (`react` at `extensions/imessage/src/actions.ts:434`, `leaveGroup` at `extensions/imessage/src/actions.ts:597`, etc.) without rechecking the configured gate. The central dispatcher only calls `supportsAction` before `handleAction`, and iMessage returns true for every supported private action at `extensions/imessage/src/actions.ts:376`.
- Impact: A direct `message.action`/CLI/tool replay that names a hidden action can still perform reactions, edit/unsend, attachment send, or group management even when the operator set `channels.imessage.actions.<action>: false`. That turns the new config from an enforcement control into a discoverability hint.
- Suggested fix: At the start of `handleAction`, resolve `IMESSAGE_ACTIONS[action]` and reject when `createActionGate(account.config.actions)(spec.gate)` is false. Also enforce `groupOnly` and selector/capability gates in execution, not only discovery.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Configure `channels.imessage.actions.reactions: false`, then dispatch `imessage:react` directly with valid bridge/message parameters; the code path reaches the `react` branch because execution never consults the action gate.
- Is this the best way to solve the issue? yes. Discovery controls are advisory; action handlers need their own authorization/capability checks because dispatch can be reached outside the generated tool schema.
