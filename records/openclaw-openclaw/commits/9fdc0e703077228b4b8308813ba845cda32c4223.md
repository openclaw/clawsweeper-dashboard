# Commit 9fdc0e7

## Summary

Found one low-severity edge case in the new Codex bootstrap forwarding path: `AGENTS.md` is excluded only after it has already consumed shared bootstrap budget, so configured low budgets or very large `AGENTS.md` files can prevent `SOUL.md` from being forwarded.

## Findings

### Low: Codex-native `AGENTS.md` can consume the budget for files Codex actually needs OpenClaw to forward

- Kind: bug
- File: `extensions/codex/src/app-server/run-attempt.ts`
- Line: 1660
- Evidence: `buildCodexWorkspaceBootstrapInstructions()` calls `resolveBootstrapContextForRun()` first, which applies `agents.defaults.bootstrapTotalMaxChars` while iterating all bootstrap files. Only later does `renderCodexWorkspaceBootstrapInstructions()` filter out Codex-native `AGENTS.md`. The shared budget is decremented in `src/agents/pi-embedded-helpers/bootstrap.ts:280` and `src/agents/pi-embedded-helpers/bootstrap.ts:323`, before the Codex-specific filter at `extensions/codex/src/app-server/run-attempt.ts:1660`.
- Impact: With `agents.defaults.bootstrapTotalMaxChars` set below the injected `AGENTS.md` size, `resolveBootstrapContextForRun()` can return only `AGENTS.md`; Codex then drops that file and sends no `SOUL.md` instructions, preserving the original issue for those configurations. I verified this with a temp workspace containing a 2000-character `AGENTS.md`, `SOUL.md`, and both bootstrap limits set to 1000: the resolver returned only `AGENTS.md`, which the new renderer would discard.
- Suggested fix: Exclude Codex-native project docs before applying the bootstrap character budget for the Codex path, for example by adding an exclude option to the bootstrap resolver or resolving bootstrap files, filtering `AGENTS.md`, then building Codex context files from the filtered list.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes: create a workspace with large `AGENTS.md`, a present `SOUL.md`, and low `agents.defaults.bootstrapTotalMaxChars`; `resolveBootstrapContextForRun()` spends the budget on `AGENTS.md`, then the Codex renderer filters it out.
- Is this the best way to solve the issue? yes: the intended contract is to leave `AGENTS.md` to Codex and forward the remaining OpenClaw bootstrap files, so the Codex path should not let `AGENTS.md` participate in that forwarding budget.
