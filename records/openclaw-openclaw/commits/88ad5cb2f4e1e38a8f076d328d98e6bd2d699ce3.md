# Commit 88ad5cb

## Summary

Found one low-severity regression in the rewritten autoreview helper: output paths no longer create missing parent directories.

## Findings

### Low: `--output` and `--json-output` fail when the parent directory is missing

- Kind: regression
- File: `.agents/skills/autoreview/scripts/autoreview`
- Line: 525
- Evidence: The new Python helper writes `Path(args.json_output).write_text(...)` and later opens `Path(args.output).open("w")` without creating parent directories. The previous Bash helper created the destination parent with `mkdir -p "$(dirname "$review_output")"` before writing. A focused fake-engine smoke check with a valid structured review result and an output path under a missing child directory failed with `FileNotFoundError` at both line 525 for `--json-output` and line 529 for `--output`.
- Impact: Existing closeout workflows that save autoreview output into a fresh artifact/report directory now fail after the expensive review finishes, even though the review itself succeeded. This can drop the human report or structured JSON artifact and turn a clean review into a helper failure.
- Suggested fix: Before writing either output path, create `Path(...).parent` when it is not `.`: `parent.mkdir(parents=True, exist_ok=True)`.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Use a fake Codex binary that writes valid structured JSON to `--output-last-message`, then run the helper with `--mode commit --commit 88ad5cb2f4e1e38a8f076d328d98e6bd2d699ce3 --output "$tmp/missing/out.txt"` or `--json-output "$tmp/missing/result.json"`.
- Is this the best way to solve the issue? yes. Restoring parent directory creation preserves the old `--output` contract and applies the same behavior to the new `--json-output` artifact path.
