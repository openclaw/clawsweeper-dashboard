# Commit 79220d7832

## Summary

Found one regression in message-tool action discovery. The Slack runtime opt-out itself is covered for current-channel sends, but the new optional `upload-file` schema fragment makes Slack uploads disappear from cross-channel tool schemas.

## Findings

### Medium: Slack `upload-file` is hidden for cross-channel message tool use

- Kind: regression
- File: `extensions/slack/src/message-tool-api.ts`
- Line: 88
- Evidence: The commit adds a schema contribution for `actions: ["upload-file"]` without setting `visibility`, so it defaults to `current-channel`. Cross-channel discovery treats current-channel schema actions as blocked in `src/channels/plugins/message-action-discovery.ts:254` and filters them out before `src/agents/tools/message-tool.ts:596` adds other configured channels’ actions. A local probe with active `signal` plus loaded/configured Slack produced action enum `["send","react","read","list-pins","member-info","emoji-list"]`, omitting Slack `upload-file`.
- Impact: Agents running from another channel, cron, or an isolated non-Slack context no longer see or validate `message({ channel: "slack", action: "upload-file", ... })`, even though normal Slack upload-file parameters still work and the issue-closing comment says `upload-file` is part of the fix.
- Suggested fix: Do not make optional `topLevel` block cross-channel `upload-file`. Either remove the separate upload-file schema contribution, mark it `visibility: "all-configured"` if exposing `topLevel` globally is acceptable, or merge/update the existing `topLevel` description without adding `upload-file` to current-channel blocked schema actions.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. The focused `tsx` probe loaded Slack and a second channel in the active plugin registry, created the message tool with `currentChannelProvider: "signal"`, and showed `upload-file` missing from the action enum.
- Is this the best way to solve the issue? yes, directionally. The problem is schema visibility/action filtering, not Slack delivery. The fix should keep `upload-file` available cross-channel while preserving the new top-level opt-out for current Slack contexts.
