## Summary

Found one actionable regression: the commit’s OpenAI/Codex runtime contract is internally inconsistent, and the new implicit-OpenAI Codex startup path is not implemented. Focused tests on touched files fail on current `main`.

## Findings

### Medium: Implicit `openai/*` Codex startup contract is not wired

- Kind: regression
- File: `src/agents/openai-codex-routing.ts`
- Line: 27
- Evidence: `openAIRouteRequiresCodexRuntime()` returns true only for `openai-codex`, so `modelSelectionRequiresCodexRuntime("openai/gpt-5.5")` is false. `src/agents/harness-runtimes.ts:70` only adds `codex` when that helper returns true, so `resolveGatewayStartupPluginIds()` omits the Codex plugin for implicit `openai/gpt-5.5`. This contradicts the new docs at `docs/providers/openai.md:62` and `docs/plugins/codex-harness.md:143`, plus the new startup test at `src/plugins/channel-plugin-ids.test.ts:1404`.
- Impact: users who select `openai/gpt-5.5` expecting the new default native Codex route can start the Gateway without the Codex harness loaded. That makes the documented subscription/OAuth route unavailable and can fall back to PI/API-key behavior or fail auth/runtime selection.
- Suggested fix: choose one contract and align all paths. If `openai/*` is intended to imply Codex for official OpenAI agent turns, make the routing/runtime helper treat official `openai/*` as Codex-required with the custom-base-url exemption, then update the older PI-default tests. If the intended behavior is install-only without runtime forcing, revert the new implicit-Codex docs/startup expectations and remove the failing test.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. `pnpm test src/plugins/channel-plugin-ids.test.ts -- -t 'includes Codex when an OpenAI agent model uses the implicit runtime default|does not include Codex for custom OpenAI-compatible agent models'` fails because the returned startup plugin list lacks `codex`.
- Is this the best way to solve the issue? unclear. The merged code, tests, docs, and PR body disagree on whether implicit `openai/*` should force Codex or only ensure plugin availability, so the maintainer needs to pick the intended contract before patching.
