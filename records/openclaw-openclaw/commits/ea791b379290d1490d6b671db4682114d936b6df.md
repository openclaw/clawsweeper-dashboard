# Commit ea791b3

## Summary

Found one actionable cross-feature regression: the new orphan transcript sweep can delete primary `.jsonl` transcripts that QMD session indexing currently treats as memory source material.

## Findings

### Medium: QMD-indexed orphan transcripts can now be deleted by cleanup

- Kind: data_loss
- File: `src/config/sessions/cleanup-service.ts`
- Line: 348
- Evidence: non-warn `sessions cleanup` now always calls `pruneUnreferencedSessionArtifacts(...)` after store maintenance. That helper deletes any old unreferenced primary `.jsonl` file because `isUnreferencedSessionArtifactFile` includes `isPrimarySessionTranscriptFileName(...)` (`src/config/sessions/disk-budget.ts:224`, `src/config/sessions/disk-budget.ts:296`). But QMD session indexing enumerates every usage-counted transcript file in the agent sessions directory (`packages/memory-host-sdk/src/host/session-files.ts:296`), and doctor explicitly suppresses orphan transcript repair when `memory.backend=qmd` and `memory.qmd.sessions.enabled` are true (`src/commands/doctor-state-integrity.ts:594`, `src/commands/doctor-state-integrity.ts:980`).
- Impact: a user with QMD session indexing enabled can have old primary transcript files that are not in `sessions.json` but are still part of QMD-backed recall. Running `openclaw sessions cleanup --enforce` now removes those originals; on the next QMD export pass, the exported markdown for that source is dropped too, so searchable conversation memory can disappear.
- Suggested fix: make the orphan sweep aware of the agent config. For QMD session indexing, either treat primary orphan transcripts as retained until `memory.qmd.sessions.retentionDays` permits removal, or archive/export them through the existing QMD/doctor-safe path before deleting. The trajectory/checkpoint cleanup can stay independent.
- Confidence: medium

## Details

- Do we have a high-confidence way to reproduce the issue? Yes for the cleanup behavior: a focused `tsx` smoke created an old unreferenced `qmd-memory-orphan.jsonl`, called `pruneUnreferencedSessionArtifacts`, and confirmed it was removed. The QMD conflict is from current source contracts and existing doctor tests.
- Is this the best way to solve the issue? Unclear. The commit is correct for normal orphan bloat, but it should not ignore the QMD retention/indexing contract for primary transcripts.
