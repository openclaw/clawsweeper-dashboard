# Commit ecb9028

## Summary

Found one remaining redaction gap in the Chrome MCP attach failure path. The new code redacts URL userinfo and query secrets, but still exposes direct WebSocket `/devtools/.../<id>` path tokens in user-visible attach errors.

## Findings

### Medium: Direct Chrome MCP WebSocket path secrets remain visible in attach errors

- Kind: privacy
- File: `extensions/browser/src/browser/chrome-mcp.ts`
- Line: 383
- Evidence: `redactChromeMcpDiagnosticText()` replaces URL-like text via `redactCdpUrl()`, and `redactCdpUrl()` preserves the full URL path while only stripping username/password and redacting recognized query secrets. A focused check against current `main` showed `ensureChromeMcpAvailable()` returning: `configured Chrome endpoint (wss://browserless.example/devtools/browser/secret-session-id-1234567890)` and `Details: ... attach failed for wss://browserless.example/devtools/browser/secret-session-id-1234567890`.
- Impact: OpenClaw docs support direct CDP WebSocket URLs shaped as `wss://.../devtools/browser/<id>`. For hosted or shared browser endpoints, that `<id>` can be enough to identify or attach to a live browser session, so this can leak browser attach material into tool/error surfaces even after this redaction fix.
- Suggested fix: use a stricter CDP diagnostic redactor for Chrome MCP attach failures, e.g. preserve scheme/host but mask userinfo, query, and direct `/devtools/<kind>/<id>` identifiers for non-loopback endpoints. Add a regression test with a direct `wss://host/devtools/browser/<secret>` URL, not only query/userinfo secrets.
- Confidence: medium

## Details

- Do we have a high-confidence way to reproduce the issue? yes: create a fake MCP process that rejects initialize with `attach failed for <cdpUrl>`, then call `ensureChromeMcpAvailable()` with `cdpUrl: "wss://browserless.example/devtools/browser/secret-session-id-1234567890"` and observe the full path token in the thrown `BrowserProfileUnavailableError`.
- Is this the best way to solve the issue? yes: the issue is localized to diagnostic CDP URL redaction, not MCP transport behavior.
