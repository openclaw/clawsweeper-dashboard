# Commit 5d09b4b

## Summary

Found one low-severity progress-display regression in Pi raw detail mode: raw mode can add both compact and raw lines for the same tool start in progress drafts.

## Findings

### Low: Pi raw progress drafts can show duplicate tool-start lines

- Kind: bug
- File: `src/agents/pi-embedded-subscribe.handlers.tools.ts`
- Line: 633
- Evidence: Pi tool start now computes raw-aware `meta` at `src/agents/pi-embedded-subscribe.handlers.tools.ts:620`, but still emits a separate `tool` event with only sanitized args at `src/agents/pi-embedded-subscribe.handlers.tools.ts:633` and then an `item` event with `meta` at `src/agents/pi-embedded-subscribe.handlers.tools.ts:643`. `agent-runner-execution` forwards both to channel callbacks at `src/auto-reply/reply/agent-runner-execution.ts:1528` and `src/auto-reply/reply/agent-runner-execution.ts:1543`. Progress channels format the `tool` callback without passing `detailMode`; for example Telegram calls `formatChannelProgressDraftLine(...)` without options at `extensions/telegram/src/bot-message-dispatch.ts:1176`, so `formatChannelProgressDraftLine` falls back to `explain` at `src/plugin-sdk/channel-streaming.ts:192`. The following `item` callback uses the raw meta, so the two adjacent lines no longer dedupe.
- Impact: With `agents.defaults.toolProgressDetail: "raw"` and progress drafts enabled, a single Pi exec start can consume two progress lines: first compact, then compact plus raw command. This is noisy and can push real later progress out of the bounded `maxLines` draft.
- Suggested fix: Carry the resolved tool progress detail mode through the progress callback layer, or suppress the generic `tool` progress line when an immediate `item` line with mode-aware meta will follow. Add a Pi/channel progress regression test that raw mode produces only the raw line for one tool start.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. A focused formatter smoke using the Pi raw meta path produced two distinct strings for the same exec start: compact `Exec: run tests` from the `tool` event path, then raw `Exec: run tests, <raw command>` from the `item.meta` path.
- Is this the best way to solve the issue? yes, the safest direction is to keep one source of formatted progress detail per event or pass the same detail mode into both paths, preserving current dedupe behavior.
