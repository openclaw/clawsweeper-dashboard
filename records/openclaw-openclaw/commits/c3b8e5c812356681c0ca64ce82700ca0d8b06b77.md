# Commit c3b8e5c

## Summary

Found one concrete issue: the Bedrock runtime npm override added here does not affect `npm install -g openclaw`, so the Windows npm install path can still resolve the failing `@aws-sdk/client-bedrock-runtime@3.1041.0`.

## Findings

### Medium: Published npm installs ignore the new Bedrock runtime override

- Kind: bug
- File: `package.json`
- Line: 1677
- Evidence: The commit adds root `overrides["@aws-sdk/client-bedrock-runtime"] = "3.1024.0"` and matching `pnpm.overrides`, but npm only applies `overrides` from the install root. `npm help package-json` for npm 11.11.0 states that overrides in installed dependencies are not considered and published packages must pin dependencies or use `npm-shrinkwrap.json`. `@mariozechner/pi-ai@0.71.1` declares `@aws-sdk/client-bedrock-runtime: ^3.1030.0`, and an isolated temp-prefix `npm install -g` of a package with the same dependency/override shape installed Bedrock runtime `3.1041.0`, not `3.1024.0`.
- Impact: The advertised Windows npm install stabilization is ineffective for the default installer path in `scripts/install.ps1`, which runs `npm install -g openclaw...`. Users can still get the Node 24 Windows ARM npm resolver failure before onboarding.
- Suggested fix: Move the pin to a place npm honors for published installs: publish/use a `@mariozechner/pi-ai` version that pins or ranges below the bad Bedrock runtime, add a published `npm-shrinkwrap.json` that locks this transitive dependency, or otherwise make the default installer install from an npm root that carries the override.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. The isolated temp-prefix global install reproduced npm’s global-install resolution behavior and resolved `@aws-sdk/client-bedrock-runtime` to `3.1041.0`; local npm docs explain why the package-level override is ignored.
- Is this the best way to solve the issue? no. The current root override helps pnpm workspace installs, but it is not a reliable fix for published npm/global installs.
