# Commit 267f5e0

## Summary

Found one packaged-install regression in the Bonjour dependency move.

## Findings

### Medium: Packaged Bonjour can no longer resolve `@homebridge/ciao`

- Kind: regression
- File: `package.json`
- Line: 1615
- Evidence: This commit removes `@homebridge/ciao` from root runtime dependencies while `package.json:34` and `package.json:35` explicitly exclude `dist/extensions/**/node_modules/**` from the npm package. Bonjour still dynamically imports `@homebridge/ciao` at `extensions/bonjour/src/advertiser.ts:105` and `extensions/bonjour/src/advertiser.ts:112`, and the plugin is startup-enabled by default in `extensions/bonjour/openclaw.plugin.json:3`. The updated test at `extensions/bonjour/manifest.test.ts:19` now asserts the root dependency is absent, but a nested bundled-plugin `package.json` dependency is not installed by npm for the root `openclaw` package. I also built a temporary Bonjour bundle and ran it with package-like `node_modules` that omitted `@homebridge/ciao`; calling the registered advertiser failed with `Cannot find package '@homebridge/ciao' imported from /tmp/openclaw-bonjour-runtime-smoke/index.mjs`.
- Impact: npm/package installs can lose Bonjour/mDNS startup discovery even though the bundled plugin remains enabled by default.
- Suggested fix: Keep `@homebridge/ciao` in the root package runtime dependencies for packaged OpenClaw, or change packaging/build so the Bonjour runtime bundles or otherwise ships the dependency and add a package-layout smoke that calls the Bonjour advertiser.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes: build/import the Bonjour entry in an installed-package-like layout without root `@homebridge/ciao`, set `OPENCLAW_DISABLE_BONJOUR=0`, call the registered discovery service, and observe the missing package error.
- Is this the best way to solve the issue? yes: root dependency restoration is the narrowest fix unless the packaging strategy is changed to bundle the dynamic import or install bundled plugin package dependencies.
