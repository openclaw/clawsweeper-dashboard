# Commit 25ce2e8

## Summary

Found one regression in the metadata refactor: manifest model suppressions now ignore bundled plugin disablement, so a disabled/denied/global-off provider plugin can still block models.

## Findings

### Medium: Disabled bundled provider plugins still suppress models

- Kind: regression
- File: `src/plugins/manifest-model-suppression.ts`
- Line: 25
- Evidence: `listManifestModelCatalogSuppressions` now filters `snapshot.plugins` through `isManifestPluginAvailableForControlPlane`; that helper returns `true` for every bundled plugin in `src/plugins/manifest-contract-eligibility.ts:17`. Before this commit, the same path used `loadPluginManifestRegistryForPluginRegistry` without `includeDisabled`, which filters disabled plugin records out. Focused smoke confirmed current behavior: `openai/gpt-5.3-codex-spark` is still suppressed with `plugins.enabled: false`, `plugins.deny: ["openai"]`, and `plugins.entries.openai.enabled: false`; the old registry helper excludes `openai` for those same configs.
- Impact: Operators who disable a bundled provider plugin can still get that plugin’s manifest `modelCatalog.suppressions`, causing OpenClaw to reject model refs that the disabled plugin should no longer control.
- Suggested fix: make model suppression use enabled plugin semantics, not broad control-plane availability. For example, filter with `isInstalledPluginEnabled(snapshot.index, plugin.id, config)` for all origins, or keep using the default manifest-registry path that excludes disabled records. Add a regression test covering disabled, denied, and global plugin-off config.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. `pnpm exec tsx -e` against `shouldSuppressBuiltInModel` returned `true` for disabled, denied, global plugin-off, and baseline OpenAI configs.
- Is this the best way to solve the issue? yes. The old behavior was tied to enabled registry records; model suppressions should follow the same activation boundary instead of treating bundled metadata as always available.
