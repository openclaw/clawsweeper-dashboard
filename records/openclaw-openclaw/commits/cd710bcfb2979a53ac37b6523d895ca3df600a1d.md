# Commit cd710bc

## Summary

Found one actionable issue: the new ClawHub artifact-download hint applies to every ClawHub HTTP failure, not only the intended missing-rollout artifact case.

## Findings

### Medium: ClawHub download denials now suggest bypassing to npm

- Kind: security
- File: `src/plugins/clawhub.ts`
- Line: 342
- Evidence: `formatClawHubClawPackDownloadError` returns the “not available yet” message plus `Use "npm:..."` for any `ClawHubRequestError`. The catch at `src/plugins/clawhub.ts:1054` calls it for every failed ClawPack download when `expectedClawPackSha256` exists. `downloadClawHubPackageArchive` throws `ClawHubRequestError` for any non-2xx artifact download response, not just 404s, in `src/infra/clawhub.ts:866`. The docs state scan-held or blocked releases may be unavailable on public catalog and install surfaces in `docs/tools/clawhub.md:166`.
- Impact: a 401/403/429/5xx, including a ClawHub moderation/quarantine refusal, is now framed as temporary rollout unavailability and tells the operator to install the same version from npm. That can steer users away from the ClawHub refusal signal and its registry moderation/provenance path.
- Suggested fix: restrict the launch-rollout npm hint to the specific expected artifact-routing miss, likely `ClawHubRequestError.status === 404`. Preserve the original `formatErrorMessage(error)` for auth, policy, rate-limit, and server failures.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. In the added test, change the mocked `ClawHubRequestError` from `status: 404, body: "Not Found"` to `status: 403, body: "Quarantined"`; the current helper still emits the “not available yet” npm fallback message because it only checks `instanceof ClawHubRequestError`.
- Is this the best way to solve the issue? yes. The commit’s intended behavior is explicitly about unavailable rollout artifacts, and the existing test only proves 404. Status-gating keeps that path while preserving accurate ClawHub failure messages for denial or policy cases.
