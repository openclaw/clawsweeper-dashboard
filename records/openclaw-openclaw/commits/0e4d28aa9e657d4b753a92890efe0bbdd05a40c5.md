- Kind: reliability

On the current checkout after `pnpm install`, this focused test failed twice with `expected app-server method thread/start; saw ; mock saw`. The command was `pnpm test extensions/codex/src/app-server/run-attempt.test.ts -- -t "forces the message dynamic tool"`. A nearby pre-existing app-server test, `passes the selected auth profile`, passed in the same environment, and the shared tool-factory unit proving `forceMessageTool` keeps `message` under the coding profile also passed.
- Impact: The regression test added by this commit can fail on a supported local Node 24 path before exercising the assertion, making the Codex app-server test suite unreliable for local/changed-gate validation on slower or fresh dependency installs.
- Suggested fix: Avoid the full app-server attempt startup in this specific regression test. Add a lower-level testing seam around Codex dynamic-tool construction or mock the `createOpenClawCodingTools` boundary and assert that `forceMessageTool: true` is passed when `sourceReplyDeliveryMode` is `message_tool_only`. If keeping the integration shape, make the harness wait budget configurable for this expensive tools-enabled path.
- Confidence: medium
