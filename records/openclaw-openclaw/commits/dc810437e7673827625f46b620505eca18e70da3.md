### Low: Connect RPC timeout no longer uses the retry window

- Kind: reliability
- File: `scripts/e2e/mcp-channels-harness.ts`
- Line: 223
- Evidence: `sendGatewayRequest("connect", ...)` now times out with `gateway request timeout: connect`, but `isRetryableGatewayConnectError()` still only matches the previous `gateway connect timeout` text at lines 290-298. A focused reproduction of the classifier showed `gateway request timeout: connect => false`.
- Impact: if the WebSocket opens but the Gateway does not answer the connect RPC within 60s, `connectGateway()` now exits after one attempt instead of retrying inside the 420s startup window. That can reintroduce flakes in the Docker MCP channel/cron smoke lanes this commit is trying to harden.
- Suggested fix: either preserve the old connect-specific timeout message for the handshake request or add `gateway request timeout: connect` to the retryable error classifier.
