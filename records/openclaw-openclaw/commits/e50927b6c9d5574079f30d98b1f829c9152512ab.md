# Commit e50927b

## Summary

Found one actionable regression in the merged Telegram TTS fix. The visible chat behavior is improved, but successful late voice delivery still mirrors hidden fallback text into hooks/transcripts as if a fallback text message had been sent.

## Findings

### Medium: Successful voice sends still mirror hidden `spokenText`

- Kind: bug
- File: `extensions/telegram/src/bot/delivery.replies.ts`
- Line: 844
- Evidence: `contentForSentHook` is initialized from `reply.text || resolveVoiceFallbackText(reply)`, and `resolveVoiceFallbackText` falls back to `reply.spokenText`. For the new late-media voice payload, `lane-delivery-text-deliverer.ts` strips visible `text` and stores the already-streamed answer in `spokenText`. If `sendVoice` succeeds, `deliverMediaReply` does not set `visibleFallbackText`, so lines 913-923 still mirror and emit hooks with the hidden fallback text.
- Impact: In the intended TTS path, the streamed text has already been finalized and mirrored by `emitPreviewFinalizedHook`. The later successful audio follow-up can then duplicate the answer in the session transcript and report incorrect `message_sent` content to hooks, even though Telegram only received a media message with no caption.
- Suggested fix: Start `contentForSentHook` from visible `reply.text` only. Assign `mediaDelivery.visibleFallbackText` only when voice fallback text is actually sent. For successful media-only voice sends, mirror `{ mediaUrls }` without the hidden `spokenText`.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. I ran an inline `deliverReplies` smoke with `{ mediaUrl, audioAsVoice: true, spokenText: "Hidden voice fallback" }`, a successful mocked `sendVoice`, and `transcriptMirror`; output showed `sendVoiceCaption: null` but `transcriptPayloads[0].text: "Hidden voice fallback"`.
- Is this the best way to solve the issue? yes. The fix should keep `spokenText` available only for actual voice rejection/caption-too-long fallback, not for successful media sends.
