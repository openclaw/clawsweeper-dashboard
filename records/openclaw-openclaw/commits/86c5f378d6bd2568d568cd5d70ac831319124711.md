# Commit 86c5f378d6

## Summary

Found one low-severity triage enforcement bypass in the new Clownfish exemption.

## Findings

### Low: Any contributor can bypass the active-PR-limit label with a `clownfish/` branch name

- Kind: bug
- File: `.github/workflows/labeler.yml`
- Line: 379
- Evidence: The new exemption treats a PR as Clownfish-owned solely when `pullRequest.head?.ref` starts with `clownfish/`, then skips and removes `r: too-many-prs` at `.github/workflows/labeler.yml:381`. That head ref is the submitter-controlled PR branch name. Existing Barnacle code uses the same prefix at `scripts/github/barnacle-auto-response.mjs:241` and removes the same label before applying auto-close rules at `scripts/github/barnacle-auto-response.mjs:1034`, so the intended bot exemption is clear, but the workflow does not verify the PR author/app identity.
- Impact: An unprivileged contributor with more than 10 open PRs can name their branch `clownfish/<anything>` and avoid the `r: too-many-prs` label in this workflow. That weakens the repo’s active PR limit automation and hides the condition from maintainers.
- Suggested fix: Require both the branch prefix and a trusted Clownfish identity, for example the known bot/app login such as `app/openclaw-clownfish`, or another stable trusted signal like the expected head repository/owner.
- Confidence: high
