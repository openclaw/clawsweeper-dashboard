# Commit dd4613a

## Summary

Found one low-severity regression in local changed-test routing. It affects untracked source/helper files, not committed CI diffs.

## Findings

### Low: Untracked changed files can bypass import-graph test routing

- Kind: regression
- File: `scripts/test-projects.test-support.mjs`
- Line: 752
- Evidence: `listChangedPathsFromGit` includes untracked files via `git ls-files --others --exclude-standard` in `scripts/changed-lanes.mjs:247`, but the new import graph file list uses plain `git ls-files` in `scripts/test-projects.test-support.mjs:752`, so untracked files are absent from `fileSet`. When the changed file is missing, `resolveAffectedTestsFromTargetedImportScan` returns `[]` at `scripts/test-projects.test-support.mjs:855` instead of falling back to the recursive graph. For `test/helpers/*`, the default broad fallback is skipped at `scripts/test-projects.test-support.mjs:1152`, which can leave `scripts/test-projects.mjs:219` with zero Vitest specs.
- Impact: `pnpm test:changed` can report “no changed test targets” for an untracked helper/source file even when tracked tests import it. A local pre-stage/pre-commit changed check can therefore pass without running the affected test.
- Suggested fix: include untracked files in the fast-path graph and grep candidates, or return `null` from the targeted scan when the changed path is not in the tracked-only file set so the filesystem recursive fallback preserves prior behavior.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. In a temp git repo with tracked `src/consumer.test.ts` importing untracked `test/helpers/fixture.ts`, `resolveChangedTargetArgs(["--changed", "HEAD"], repo)` returned `[]` and `buildVitestRunPlans(["--changed", "HEAD"], repo)` returned `[]`. The same file layout outside git, which exercises the recursive fallback, returned `src/consumer.test.ts`.
- Is this the best way to solve the issue? yes, the fix should keep the fast path but make its file universe match the changed-file contract, especially because the changed-file lister intentionally includes untracked worktree files.
