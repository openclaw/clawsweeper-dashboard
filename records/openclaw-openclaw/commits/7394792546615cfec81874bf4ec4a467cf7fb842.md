## Findings

### Low: `/model status` can hide explicitly allowlisted direct provider rows

- Kind: regression
- File: `src/auto-reply/reply/directive-handling.model.ts`
- Line: 272
- Evidence: `filterMissingAuthNestedProviderDuplicates` protects only keys from `buildConfiguredModelCatalog`, which reads `models.providers.*.models`. But `/model status` also builds rows from `agents.defaults.models`, fallback/default model config, and the loaded `allowedModelCatalog`; `buildAllowedModelSetWithFallbacks` turns exact `agents.defaults.models` refs into allowed catalog rows. If a user allowlists both `google/gemini-3-flash-preview` and `openrouter/google/gemini-3-flash-preview`, and direct Google auth is missing, the new filter removes the direct Google row because `configuredKeys` does not include the allowlist key.
- Impact: `/model status` can omit the missing-auth diagnostic for a real user-configured direct model, including a current/default direct model, whenever a nested OpenRouter row covers the same provider/model suffix. The model can still be allowed/selectable, but status stops showing the auth problem the command is meant to diagnose.
- Suggested fix: build the â€śdo not hideâ€ť set from all explicit status inputs, not only `models.providers`: exact `agents.defaults.models` refs, resolved default/primary/fallback refs, and possibly current selected/active refs. Alternatively have `buildModelPickerCatalog` tag entries with whether they came from explicit config and only filter unconfigured catalog-derived shadow rows.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Add an existing-unit-test variant with `agents.defaults.models` containing both `"google/gemini-3-flash-preview": {}` and `"openrouter/google/gemini-3-flash-preview": {}`, with matching allowed catalog rows and no Google auth; the current filter drops `[google]`.
- Is this the best way to solve the issue? yes. The fix should preserve the intended phantom-row suppression while exempting every explicitly configured model source used by `/model status`.
