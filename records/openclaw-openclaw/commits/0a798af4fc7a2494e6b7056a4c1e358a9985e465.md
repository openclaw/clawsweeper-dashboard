# Commit 0a798af

## Summary

Found one low-severity regression: the documented `FORCE_COLOR=0` opt-out for managed `gateway:watch` tmux logs does not actually disable subsystem ANSI colors unless `NO_COLOR` is also present.

## Findings

### Low: `FORCE_COLOR=0` opt-out still emits ANSI colors without `NO_COLOR`

- Kind: regression
- File: `scripts/gateway-watch-tmux.mjs`
- Line: 77
- Evidence: `resolveColorEnv` preserves explicit `FORCE_COLOR=0` by adding only `FORCE_COLOR=0` and no `NO_COLOR`. The gateway subsystem logger treats `FORCE_COLOR=0` only as “not force color”; if `NO_COLOR` is absent and the environment looks rich (`TERM`, TTY, etc.), it still returns `new Chalk({ level: 1 })` in `src/logging/subsystem.ts:96`. A focused live check reproduced ANSI output with `env -u NO_COLOR FORCE_COLOR=0 TERM=xterm-256color ... createSubsystemLogger('gateway/auth').info('hello')`, producing `"\u001b[33m[auth]\u001b[39m \u001b[36mhello\u001b[39m"`. With `NO_COLOR=1 FORCE_COLOR=0`, the same check produced plain `"[auth] hello"`.
- Impact: The new docs say `FORCE_COLOR=0 pnpm gateway:watch` disables ANSI output, but users still get colored managed tmux logs in common terminal environments. Since the default path now unsets `NO_COLOR` and sets `FORCE_COLOR=1`, the practical opt-out becomes the undocumented `NO_COLOR=1 FORCE_COLOR=0`.
- Suggested fix: In the explicit `FORCE_COLOR=0` path, also set or preserve a no-color signal that the subsystem logger honors, such as assigning `NO_COLOR=1`, or update `getColorForConsole()` to treat `FORCE_COLOR=0` as an explicit disable before rich-terminal fallback.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes: run the focused `tsx` subsystem logger check with `FORCE_COLOR=0`, `TERM=xterm-256color`, and no `NO_COLOR`; ANSI escapes are emitted.
- Is this the best way to solve the issue? yes: the fix should align the tmux wrapper’s advertised opt-out with the logger’s actual color decision path, preferably by making `FORCE_COLOR=0` an explicit disable.
