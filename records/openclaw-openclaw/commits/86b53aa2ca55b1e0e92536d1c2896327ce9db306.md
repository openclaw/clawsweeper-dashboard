# Commit 86b53aa

## Summary

Found one concrete regression in the quiet DM no-response path. The commit stops sending the synthesized silent-reply text in DMs, but the dispatcher still treats that successful quiet turn as “no final response,” so Telegram reaction cleanup/finalization can take the error/no-response branch.

## Findings

### Medium: Quiet DM no-response turns are not counted as handled

- Kind: regression
- File: `extensions/telegram/src/bot-message-dispatch.ts`
- Line: 1652
- Evidence: The new `isGroup` guard at line 1622 skips fallback delivery for DMs. For the intended DM path (`queuedFinal: false`, no dispatch error, no delivery), `hasFinalResponse` at lines 1652-1653 remains false. That immediately triggers error status finalization at lines 1655-1660 and returns at lines 1663-1665 before the normal ack-reaction cleanup path at lines 1708-1725.
- Impact: With Telegram status reactions enabled, an intentionally quiet DM turn can end with the error reaction instead of a done/restored state. With simple ack reactions and `messages.removeAckAfterReply: true`, the ack reaction can be left behind because the cleanup path is skipped. This replaces the visible chatter bug with a smaller but still user-visible reaction-state bug.
- Suggested fix: Add an explicit handled-success flag for no-response DMs, for example `quietNoResponseDm`, and include it in the final-response/handled condition so normal done finalization and ack cleanup run without sending fallback text. Keep real error and group fallback behavior unchanged.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes: dispatch a direct Telegram context where the buffered dispatcher returns `queuedFinal: false`, no payloads are delivered, and either a `statusReactionController` or `reactionApi` plus `removeAckAfterReply: true` is present. The current code takes the `!hasFinalResponse` branch.
- Is this the best way to solve the issue? unclear as submitted. The `isGroup` guard is the right delivery boundary, but the turn-completion state also needs to model “quiet DM success.”
