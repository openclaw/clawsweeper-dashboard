## Summary

Found one supply-chain regression in the Mantis Telegram desktop proof workflow: the commit replaces exact ffmpeg npm package versions with an unauthenticated moving GitHub `latest` binary download that is installed and executed before the workflow enters later secret-bearing steps.

## Findings

### Medium: Secret-bearing workflow now executes a moving third-party ffmpeg binary without integrity verification

- Kind: supply_chain
- File: `.github/workflows/mantis-telegram-desktop-proof.yml`
- Line: 343
- Evidence: the workflow downloads `BtbN/FFmpeg-Builds/releases/download/latest/ffmpeg-master-latest-linux64-gpl.tar.xz`, extracts it, installs `ffmpeg`/`ffprobe` into `/usr/local/bin`, and executes both binaries without pinning an immutable release or checking a digest. The same job later exposes OpenAI, Crabbox, Convex, Mantis, and `GH_TOKEN` secrets to subsequent steps, and has `contents`, `issues`, and `pull-requests` write permissions.
- Impact: a changed or compromised upstream `latest` asset can execute arbitrary code on the runner and persist into later secret-bearing steps, creating a path to secret exfiltration or repository mutation. This also makes the proof workflow non-reproducible.
- Suggested fix: pin an immutable release/tag or known artifact digest and verify it with `sha256sum -c` before extraction or execution. Prefer a repo-owned/cacheable artifact or a package-manager path with lockfile-backed integrity if that is the intended trust boundary.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? Yes. Inspect the workflow step at `.github/workflows/mantis-telegram-desktop-proof.yml:341`; it always resolves the moving `latest` release URL and performs no checksum verification before executing the installed binaries.
- Is this the best way to solve the issue? Yes. Verification must happen before `tar`, `sudo install`, `ffmpeg -version`, or any later secret-bearing step; pinning plus checksum verification preserves the direct-download approach while restoring deterministic integrity.
