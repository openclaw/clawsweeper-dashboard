# Commit 1831e124b2

## Summary

Found one low-severity regression in the Canvas legacy config migration. A lint rewrite changed object spread to `Object.assign`, which changes `__proto__` handling for user-authored config objects.

## Findings

### Low: Canvas migration now allows `__proto__` to affect migrated host config through the prototype

- Kind: security
- File: `extensions/canvas/src/config-migration.ts`
- Line: 14
- Evidence: `mergeHostConfig` now uses `Object.assign({}, params.legacyHost, params.existingHost)`. For JSON-authored `{"canvasHost":{"__proto__":{"enabled":false,"root":"/proto-root"}}}`, Node sets the returned host object's prototype instead of creating an own `__proto__` data property. A focused `pnpm exec tsx` reproduction showed `hostOwnKeys: []`, `inheritedEnabled: false`, `inheritedRoot: "/proto-root"`, `parseCanvasPluginConfig({host}).host` returning those inherited values, and `JSON.stringify(host)` producing `{}`. `validateConfigObjectRawWithPlugins` also returned `ok: true` for the migrated config because the host has no own keys.
- Impact: The migration can accept and act on inherited Canvas host values that are not actually persisted and are not own config fields. That creates a validation/persistence mismatch in the doctor migration path and weakens the previous object-spread behavior, where `__proto__` stayed an own property and could be rejected or serialized predictably.
- Suggested fix: Revert this merge to object spread, e.g. `{ ...params.legacyHost, ...(params.existingHost ?? {}) }`, or use a safe own-property copy helper that defines data properties instead of invoking the `__proto__` setter.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes; the focused `pnpm exec tsx` migration check reproduced inherited `enabled`/`root` values and showed the serialized migrated host as `{}`.
- Is this the best way to solve the issue? yes; the regression comes from replacing spread with `Object.assign`, so restoring spread or equivalent data-property copy preserves the prior safer semantics.
