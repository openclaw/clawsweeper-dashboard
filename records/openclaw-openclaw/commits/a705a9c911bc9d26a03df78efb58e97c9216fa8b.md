## Findings

### Low: App-server wait helper now consumes the whole Vitest test budget

- Kind: reliability
- File: `extensions/codex/src/app-server/run-attempt.test.ts`
- Line: 99
- Evidence: This commit changes the shared app-server harness wait from 30s to `120_000`, while `test/vitest/vitest.shared.config.ts` also sets `testTimeout: 120_000`. The focused file run failed with `starts active OpenClaw sandbox turns with Codex native execution disabled` timing out at Vitest’s 120s limit, instead of `waitForMethod("turn/start")` surfacing its own detailed “expected app-server method…” error.
- Impact: Any missing app-server request now burns the entire per-test timeout and loses the harness diagnostic, making CI failures slower and harder to debug. In the current checkout this already turns the explicit touched-file proof into a 120s test timeout.
- Suggested fix: Keep the harness default below the Vitest per-test timeout, or raise only the specific long-running tests’ timeout so `waitForMethod` has time to fail with its own assertion and `afterEach` cleanup still has budget.
- Confidence: high
