# Commit 277ad4a

## Summary

Found regressions in the new guarded link-understanding path. The SSRF guard is added, but the new fallback and argument filtering can break existing CLI summarizers and can inject raw fetched page content when the configured summarizer fails or intentionally emits nothing.

## Findings

### Medium: failed link summarizers now inject raw fetched page content

- Kind: regression
- File: `src/link-understanding/runner.ts`
- Line: 231
- Evidence: `runCliEntry` throws on nonzero exit and returns `null` for empty stdout at `src/link-understanding/runner.ts:145` and `src/link-understanding/runner.ts:149`. `runLinkEntries` catches those failures and returns `null` at `src/link-understanding/runner.ts:174` and `src/link-understanding/runner.ts:184`. The new caller then converts any `null` result into `fetched.content` at `src/link-understanding/runner.ts:231`, so a missing binary, failed CLI, unsupported entry, or intentionally empty output now appends the raw guarded page body to `LinkUnderstanding`.
- Impact: before this commit, a failed or empty CLI entry produced no link-understanding output. After this commit, the agent can receive up to the fetched content limit of unsummarized HTML/JSON/text, which can bloat prompts and bypass operator intent when the configured CLI was supposed to filter, summarize, or decline content.
- Suggested fix: remove the global `?? fetched.content` fallback. If curl/wget compatibility needs pass-through behavior, have `runCliEntry` return the guarded content explicitly for that detected case, or return a tagged result that distinguishes intentional pass-through from failure/empty output.
- Confidence: high

### Medium: URL-template filtering leaves dangling CLI options

- Kind: regression
- File: `src/link-understanding/runner.ts`
- Line: 66
- Evidence: `buildLinkCliArgs` filters out every arg containing `LinkUrl` or `LinkFinalUrl`, without considering the surrounding option shape. The new test itself configures `args: ["--source", "{{LinkUrl}}"]` and asserts the command is invoked as `["summarize", "--source"]`, leaving `--source` without its value. The base implementation templated all args, so this is a behavior change for existing configs using patterns like `--url {{LinkUrl}}`, `--source {{LinkUrl}}`, or `--metadata={{LinkFinalUrl}}`.
- Impact: existing non-fetcher summarizers can fail due to malformed argv, and with the fallback above those failures become raw page-content injection instead of a clean skip.
- Suggested fix: do not silently drop only the value side of URL-template options. Either reject/skip URL-template CLI configs with a clear verbose warning, preserve metadata through the new environment variables only and remove known paired flags safely, or add an explicit config contract for stdin-only link processors.
- Confidence: high

### Low: CLI output is no longer bounded by `CLI_OUTPUT_MAX_BUFFER`

- Kind: reliability
- File: `src/link-understanding/runner.ts`
- Line: 137
- Evidence: the base code used `runExec(..., { maxBuffer: CLI_OUTPUT_MAX_BUFFER })`, capping CLI output at 5 MiB. The new code uses `runCommandWithTimeout` at `src/link-understanding/runner.ts:137`, whose implementation appends stdout/stderr chunks into arrays at `src/process/exec.ts:386` and `src/process/exec.ts:390` and concatenates them at `src/process/exec.ts:450` without a byte cap.
- Impact: a noisy or looping configured summarizer can accumulate unbounded stdout/stderr until timeout, increasing memory pressure compared with the previous bounded `execFile` path.
- Suggested fix: add a max-output option to `runCommandWithTimeout` or wrap this call with capped stream collection equivalent to the previous `CLI_OUTPUT_MAX_BUFFER` behavior.
- Confidence: medium

## Details

- Do we have a high-confidence way to reproduce the issue? yes. A mocked guarded fetch plus a CLI mock that exits nonzero or returns empty stdout will produce `outputs: [fetched.content]`; the existing test also demonstrates the dangling `--source` argv shape.
- Is this the best way to solve the issue? yes for the main fallback regression: pass through guarded content only for the explicit curl/wget replacement path, and keep failure/empty-output semantics as skip.
