# Commit f1340be

## Summary

The commit expands OpenRouter’s attribution category header from 1 category to 7, but OpenRouter documents `X-OpenRouter-Categories` as a comma-separated list with up to 2 categories per request. That makes the emitted header out of contract for every verified OpenRouter LLM route.

## Findings

### Medium: OpenRouter attribution category header exceeds the documented per-request limit

- Kind: compatibility
- File: `src/agents/provider-attribution.ts`
- Line: 126
- Evidence: `OPENROUTER_ATTRIBUTION_CATEGORIES` is now `cli-agent,cloud-agent,programming-app,creative-writing,writing-assistant,general-chat,personal-agent`, and `buildOpenRouterAttributionPolicy()` sends it as `X-OpenRouter-Categories` on verified OpenRouter routes. The updated stream-wrapper and request-config tests assert the same 7-category header. OpenRouter’s app-attribution docs describe that header as accepting a comma-separated list of up to 2 categories per request. citeturn0search0
- Impact: OpenClaw no longer follows OpenRouter’s documented request contract. Depending on OpenRouter’s validation behavior, the extra categories may be ignored, attribution may not be applied as intended, or OpenRouter LLM requests could be rejected if the limit is enforced.
- Suggested fix: Cap the emitted header to at most 2 documented categories per request, for example a stable pair such as `cli-agent,personal-agent`, or select two categories by capability/request context. Keep tests asserting the vendor limit instead of the expanded 7-value string.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? Yes for the local contract violation: `resolveProviderRequestAttributionHeaders({ provider: "openrouter" })` and the OpenRouter wrapper emit the 7-category header asserted by the changed tests. Live server-side rejection could not be checked without an OpenRouter API key.
- Is this the best way to solve the issue? Yes: the safest fix direction is to obey the documented request-header limit and send no more than two categories.
