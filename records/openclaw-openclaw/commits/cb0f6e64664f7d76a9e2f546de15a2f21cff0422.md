# Commit cb0f6e6466

## Summary

Found one regression: removing Matrix sidecars from the checked baseline also removes them from global-install critical sidecar verification, while existing `update-global` tests still require those legacy/corrupt install cases to fail closed.

## Findings

### Medium: Matrix sidecar removal weakens global install verification and breaks focused infra tests

- Kind: regression
- File: `scripts/lib/bundled-runtime-sidecar-paths.json`
- Line: 1
- Evidence: This commit removes `dist/extensions/matrix/helper-api.js`, `runtime-api.js`, `runtime-setter-api.js`, and `thread-bindings-runtime.js` from `BUNDLED_RUNTIME_SIDECAR_PATHS`. `src/infra/update-global.ts:221` derives critical installed package sidecar checks only from that list. With Matrix removed, `collectInstalledGlobalPackageErrors` now returns `[]` for an installed `dist/extensions/matrix/package.json` missing those sidecars.
- Impact: global update/package verification can pass a legacy or corrupt install with Matrix plugin metadata present but required Matrix runtime sidecars missing. The current focused regression tests confirm the behavioral change: `src/infra/update-global.test.ts:801` and `src/infra/update-global.test.ts:815` both fail because the expected missing-sidecar error is no longer emitted.
- Suggested fix: keep root-package-excluded Matrix sidecars out of the root package baseline if that packaging change is intentional, but preserve a separate critical/legacy verification inventory for installed Matrix plugin dirs, or intentionally update the legacy verification contract and tests together.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes: `pnpm test src/infra/update-global.test.ts -- -t "legacy sidecars|critical sidecars"` fails both Matrix sidecar assertions on current `main`.
- Is this the best way to solve the issue? unclear: the packaging intent looks valid, but the update verifier still has an explicit contract for installed bundled plugin sidecars. The safer fix is to decouple packaging inventory from legacy critical verification instead of dropping Matrix from both.
