# Commit a3cacf3

## Summary

Found one remaining regression on current `main`: cron isolated sessions can still reuse stripped skill snapshots without hydrating `resolvedSkills`, so Claude CLI cron runs silently lose the session-scoped skills plugin after a cold resume.

## Findings

### Medium: Cron CLI resumes lose Claude skills after snapshot stripping

- Kind: regression
- File: `src/cron/isolated-agent/skills-snapshot.ts`
- Line: 34
- Evidence: `resolveCronSkillsSnapshot` returns `existingSnapshot` unchanged when the version/filter match (`src/cron/isolated-agent/skills-snapshot.ts:30-35`). After this commit, persisted snapshots no longer contain `resolvedSkills`. Cron then passes that snapshot into CLI execution at `src/cron/isolated-agent/run-executor.ts:144-162`, and the Claude skills plugin reads only `snapshot?.resolvedSkills ?? []` at `src/agents/cli-runner/claude-skills-plugin.ts:33-36`, returning no `--plugin-dir` when the field is absent.
- Impact: A scheduled/cron isolated agent using the Claude CLI backend can resume an existing session whose snapshot was loaded from `sessions.json`; skills remain listed in the lightweight snapshot, but the Claude plugin receives no materialized skill directories, so Claude Code skill integration is disabled for that run. The later hydration fixes cover auto-reply and `agent-command`, but not this cron path.
- Suggested fix: hydrate in `resolveCronSkillsSnapshot` when returning a non-refresh existing snapshot, using the centralized `hydrateResolvedSkillsAsync` helper and `runtime.buildWorkspaceSkillSnapshot`, preserving prompt/skills/version like the other resume paths.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. A focused smoke of `prepareClaudeCliSkillsPlugin` showed a stripped snapshot returns `args: []`, while the same snapshot with `resolvedSkills` returns `["--plugin-dir", ...]`.
- Is this the best way to solve the issue? yes. Hydrating at the cron snapshot resolver keeps downstream cron embedded and CLI paths consistent with `session-updates` and `agent-command`, without repersisting the large `resolvedSkills` field.
