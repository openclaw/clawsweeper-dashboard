- Kind: regression

- Impact: A PR that already has `proof: sufficient` can receive an unrelated status label and end up with both `proof: sufficient` and `triage: needs-real-behavior-proof`. That is contradictory state for the proof gate and can confuse review/automerge label consumers.
- Suggested fix: When preserving existing sufficient proof for override, exact-head proof, or unrelated label churn, also filter proof-candidate labels such as `triage: needs-real-behavior-proof` and `triage: mock-only-proof` from the labels added in that pass.
