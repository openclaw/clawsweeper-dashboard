# Commit 59c523c

## Summary

Found one actionable gap in the new source-only package rejection: explicit `openclaw.runtimeExtensions` can still point at a TypeScript file, so a package can pass install/discovery without compiled JavaScript output.

## Findings

### Low: Explicit runtimeExtensions can bypass the compiled-runtime requirement

- Kind: bug
- File: `src/plugins/package-entry-resolution.ts`
- Line: 142
- Evidence: `validatePackageExtensionEntriesForInstall` validates an explicit `runtimeExtensions[index]` path only for existence and then `continue`s. Discovery has the same shape: `resolvePackageRuntimeEntrySource` returns `runtimeEntryPath` before the global-origin TypeScript compiled-output check runs. A focused smoke check with `extensions: ["./src/index.ts"]` and `runtimeExtensions: ["./src/index.ts"]` returned install `{ ok: true }`, resolved `src/index.ts`, and emitted no diagnostics.
- Impact: A source-only npm/package install can still be accepted if its manifest explicitly points `runtimeExtensions` at TypeScript source. That contradicts the new docs and leaves the same class of provider-unavailable/runtime-load failures this commit is meant to fail early.
- Suggested fix: Reject TypeScript `runtimeExtensions` and `runtimeSetupEntry` for package installs, and make global-origin discovery reject explicit TypeScript runtime paths unless they resolve to a JavaScript built peer or come from a source/dev origin.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. After installing deps, I ran a focused `pnpm exec tsx -e ...` probe that called `validatePackageExtensionEntriesForInstall` and `resolvePackageRuntimeExtensionSources` against a temp package containing only `src/index.ts` with `runtimeExtensions: ["./src/index.ts"]`; it passed and resolved the TS source.
- Is this the best way to solve the issue? yes. The guard already distinguishes install/global package paths from source/dev paths, so extending that validation to explicit runtime entries is the narrowest fix.
