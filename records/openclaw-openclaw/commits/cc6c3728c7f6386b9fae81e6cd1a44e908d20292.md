## Summary

Found one CI plumbing regression: one Testbox hydration workflow still does not pass `FACTORY_API_KEY`, even though the commit made missing Factory auth fatal for Droid ACP Docker proof.

## Findings

### Low: Build-artifacts Testbox cannot hydrate Factory auth for Droid proof

- Kind: regression
- File: `.github/workflows/ci-build-artifacts-testbox.yml`
- Line: 199
- Evidence: `ci-build-artifacts-testbox.yml` runs `bash scripts/ci-hydrate-testbox-env.sh` at line 221, but its env block at lines 199-220 omits `FACTORY_API_KEY`. That helper delegates to `scripts/ci-hydrate-live-auth.sh`, which now includes `FACTORY_API_KEY` in the profile only if it is present in the step environment. The new Droid guard in `scripts/test-live-acp-bind-docker.sh:303` exits with error when `FACTORY_API_KEY` is missing. The same commit added `FACTORY_API_KEY` to analogous hydration steps in `.github/workflows/ci-check-testbox.yml:106` and `.github/workflows/crabbox-hydrate.yml:137`, which makes this workflow the remaining uncovered Testbox path.
- Impact: a maintainer-dispatched build-artifacts Testbox session will not get Factory credentials through `openclaw-testbox-env`, so `OPENCLAW_LIVE_ACP_BIND_AGENT=droid` / `pnpm test:docker:live-acp-bind:droid` fails even when the repository secret exists.
- Suggested fix: add `FACTORY_API_KEY: ${{ secrets.FACTORY_API_KEY }}` to the `Hydrate Testbox provider env helper` step in `.github/workflows/ci-build-artifacts-testbox.yml`, and broaden the workflow test to check every workflow that calls `ci-hydrate-testbox-env.sh`.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes: run the `ci-build-artifacts-testbox.yml` workflow with a configured `FACTORY_API_KEY`, then in the Testbox use `openclaw-testbox-env env | grep FACTORY_API_KEY` or run `openclaw-testbox-env pnpm test:docker:live-acp-bind:droid`; the env will not be hydrated from this workflow’s step.
- Is this the best way to solve the issue? yes: all nearby workflows already pass provider secrets directly into the hydration step, so adding the missing env entry keeps the existing contract.
