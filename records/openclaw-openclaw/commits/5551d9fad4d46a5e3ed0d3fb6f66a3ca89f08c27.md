# Commit 5551d9f

## Summary

Found one contract-inventory regression around the new packaged-core boundary. The reviewed commit removes the old `includeInCore` gates from discovery, but adds a `package.json#files` exclusion gate to the static bundled capability inventory, so source/external bundled plugin contracts can diverge from manifest discovery.

## Findings

### Medium: Static bundled contract inventory excludes source/external plugin contracts

- Kind: regression
- File: `src/plugins/contracts/inventory/bundled-capability-metadata.ts`
- Line: 94 in the reviewed commit
- Evidence: The reviewed commit adds `EXCLUDED_PACKAGED_EXTENSION_DIRS` from root `package.json#files` and returns `undefined` for excluded plugin dirs before building `BUNDLED_PLUGIN_CONTRACT_SNAPSHOTS`. In the same commit, `src/plugins/contracts/registry.contract.test.ts` expects the static registry to cover every bundled manifest provider/contract without the old package gate. Source/external plugins such as `codex`, `diffs`, `qqbot`, `brave`, and others have manifest contracts but are excluded from the package tarball.
- Impact: Vitest/static contract paths can omit source-checkout plugin contracts even though discovery now exposes those plugins. That breaks contract coverage and can make CI fail or hide missing provider/tool contract registration for source-only checkout plugins.
- Suggested fix: Keep package tarball excludes limited to package/dist inventory. Static contract inventory and its contract tests should use the same manifest/source checkout boundary as plugin discovery, then separately assert package excludes where package contents are being tested.
- Confidence: medium

## Details

- Do we have a high-confidence way to reproduce the issue? Yes for the remaining current-main mismatch: after `pnpm install`, `pnpm test src/plugins/contracts/registry.contract.test.ts -- --reporter=dot` fails because the static and manifest-derived bundled contract sets disagree. The exact reviewed commit was not checked out, but its code shows the inverse mismatch: static snapshots filter package-excluded plugins while the reviewed contract test expects unfiltered manifest discovery.
- Is this the best way to solve the issue? Yes: use package excludes only for packaged artifact boundaries, not source-checkout contract inventory. Current `main` already removed the runtime inventory exclusion in follow-up `4e312d9b0e`, but later test filters still need to be aligned.
