# Commit 0c50957

## Summary

Found one low-severity diagnostic regression. The new agent-tool lookup can say a bundled plugin tool is тАЬavailableтАЭ even when the owning bundled plugin is disabled by default or denylisted.

## Findings

### Low: Disabled bundled plugin tools are reported as available

- Kind: bug
- File: `src/plugins/manifest-command-aliases.runtime.ts`
- Line: 93
- Evidence: `resolveManifestToolOwner` relies on `isManifestPluginAvailableForControlPlane` before returning `availability: "loaded"`, but that helper returns `true` for every bundled plugin in `src/plugins/manifest-contract-eligibility.ts:20`. The later CLI policy guard in `src/cli/run-main-policy.ts:197` checks `plugins.entries.<id>.enabled=false` and restrictive `plugins.allow`, but not `plugins.deny` and not bundled `enabledByDefault=false`.
- Impact: `openclaw google_meet` can report `"google_meet" is an agent tool available from the "google-meet" plugin` even though `google-meet` is disabled by default. Similarly, a denylisted bundled plugin such as `browser` can still be attributed as available. This does not enable the tool or create a security bypass, but it reintroduces the class of misleading CLI guidance this commit is trying to fix.
- Suggested fix: use the same effective activation policy used for bundled plugin loading before returning `availability: "loaded"` for tool ownership, or downgrade blocked/default-disabled bundled owners to no owner or `manifest-only`. Include `plugins.deny`, global `plugins.enabled`, `enabledByDefault`, and platform default enablement in the check.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. A direct source probe returned `availability: "loaded"` for `google_meet` with empty config and for `browser` with `plugins.deny:["browser"]`; a separate activation-state probe showed those owners are actually `activated:false` (`bundled (disabled by default)` / `blocked by denylist`).
- Is this the best way to solve the issue? yes, the fix should be in ownership availability, not in string wording, because the current return value is what makes the CLI choose the stronger тАЬavailable fromтАЭ diagnostic.
