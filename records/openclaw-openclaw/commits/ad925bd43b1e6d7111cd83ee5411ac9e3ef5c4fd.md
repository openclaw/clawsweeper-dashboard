# Commit ad925bd

## Summary

The commit fixes the original middle-policy truncation case, and the focused tests pass. I found one remaining regression in the merged implementation: the new AGENTS-specific truncation path can leave thousands of characters of the configured per-file budget unused when the digest is sparse, so some oversized `AGENTS.md` files now inject much less raw policy text than before.

## Findings

### Medium: AGENTS.md truncation drops raw context when the digest is sparse

- Kind: regression
- File: `src/agents/pi-embedded-helpers/bootstrap.ts`
- Line: 229
- Evidence: `trimAgentsBootstrapContent` fixes the retained slices at 45% head, 15% tail, and 35% digest budget, then only shrinks those allocations when the rendered content overflows. It never reallocates unused digest budget back to head or tail. A read-only probe of current `main` showed a 24,000-character `AGENTS.md` with no digest candidate lines injects only 7,316 chars (`kept 5400+policy 0+1800`), while the same content through the generic path used by `TOOLS.md` injects 11,999 chars (`kept 8919+2973`). Before this commit, `AGENTS.md` used that generic path too.
- Impact: Oversized `AGENTS.md` files written as prose, or with sparse/non-matching digest candidates, can lose roughly 4.7k chars of raw bootstrap policy that previous `main` would have delivered. A rule around char 8,000, for example, was previously in the retained head slice but is now omitted unless the digest heuristic selects it.
- Suggested fix: After building the digest, compute the actual rendered length and reallocate unused digest capacity into head/tail slices, or fall back to the generic splitter when the digest is empty or too small to justify the reduced raw context.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Call `buildBootstrapContextFiles` with a 24,000-character `AGENTS.md`, `maxChars: 12000`, and content that does not match digest candidates; compare injected length with the same payload named `TOOLS.md`.
- Is this the best way to solve the issue? yes. Keeping the digest is a good fix direction, but it should preserve the configured budget by reusing empty digest space instead of lowering the effective AGENTS.md context cap.
