# Commit a0d5372

## Summary

Found one packaging/runtime risk from externalizing `qrcode-terminal`.

## Findings

### Medium: externalized QR terminal dependency is emitted without a declared runtime dependency

- Kind: compatibility
- File: `tsdown.config.ts`
- Line: 129
- Evidence: `tsdown.config.ts:129` adds `qrcode-terminal` to `deps.neverBundle`. The real tsdown graph still bundles QQBot setup code from `extensions/qqbot/src/bridge/setup/finalize.ts:22`, and the emitted connector chunk contains a top-level bare `import "qrcode-terminal"`. The root `package.json:1648` runtime dependencies include `qrcode`, but not `qrcode-terminal` or `@tencent-connect/qqbot-connector`. An isolated import of that emitted chunk from a package root with no dependencies fails with `ERR_MODULE_NOT_FOUND: Cannot find package 'qrcode-terminal'`.
- Impact: packaged builds can ship a root dist chunk that cannot load unless `qrcode-terminal` happens to be installed by the surrounding environment. This can break QQBot QR setup if that chunk is reachable, and it also creates package-validation drift for emitted root dist imports.
- Suggested fix: either keep `qrcode-terminal` bundled in the QQBot connector chunk, or make the packaging boundary explicit: exclude/prune the QQBot-owned root chunks when QQBot is externalized from the core package. Adding `qrcode-terminal` to root dependencies would also satisfy runtime resolution, but it conflicts with the current plugin-local dependency ownership direction.
- Confidence: medium

## Details

- Do we have a high-confidence way to reproduce the issue? yes for the emitted chunk: build the real tsdown graph with this commit, observe the generated bare `qrcode-terminal` import, then import that chunk from a temp package root with no installed deps.
- Is this the best way to solve the issue? unclear. The safer direction depends on whether QQBot should be present in the core package at all. If QQBot remains externalized, the build/package inventory should stop shipping its root chunks; if it remains bundled, `qrcode-terminal` should not be externalized without a declared root dependency.
