# Commit f06b77c

## Summary

Found two low-severity CLI parser-output regressions in the new formatter. Both are user-facing guidance issues, not runtime/security failures.

## Findings

### Low: Parse errors can suggest an invalid help command

- Kind: bug
- File: `src/cli/program/error-output.ts`
- Line: 25
- Evidence: `resolveHelpCommand()` uses `getCommandPathWithRootOptions(argv, 2)` and formats the first two non-option tokens as a command path. Live check on current `main`: `pnpm openclaw plugins isntall` prints `Too many arguments for this command.` followed by `Try: openclaw plugins isntall --help`, but `isntall` is the mistyped/extra argument, not a valid subcommand.
- Impact: Users are sent to a help command that cannot work, which makes parser errors harder to recover from.
- Suggested fix: derive the help target from the last recognized Commander command or known command tree, and stop before unknown/extra positionals. For the example above, point to `openclaw plugins --help`.
- Confidence: high

### Low: Commander “Did you mean” suggestions are dropped

- Kind: regression
- File: `src/cli/program/error-output.ts`
- Line: 49
- Evidence: Commander 14.0.3 has suggestion output enabled by default for unknown commands/options. The new regex branches match only the leading `unknown command` / `unknown option` text and discard the remaining suggestion suffix. Live check: `pnpm openclaw plugins install --helpp` prints no `Did you mean --help?` suggestion.
- Impact: Typos in flags or commands lose Commander’s closest-match recovery hint.
- Suggested fix: preserve and append Commander’s suggestion lines after the rewritten error, or intentionally replace them with equivalent suggestions from the same candidate set.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes: `pnpm openclaw plugins isntall` and `pnpm openclaw plugins install --helpp`.
- Is this the best way to solve the issue? unclear: the formatter is reasonable, but it needs either Commander command context or a known-command-path resolver so it does not invent invalid help targets or discard upstream suggestions.
