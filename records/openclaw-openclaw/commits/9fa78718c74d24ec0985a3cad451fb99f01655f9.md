# Commit 9fa78718c7

## Summary

Found one regression in the fix: the new OpenRouter Anthropic/xAI branch strips valid `reasoning_details` arrays, not only the bogus string provenance tags.

## Findings

### Medium: Anthropic/xAI OpenRouter replay now drops valid `reasoning_details` arrays

- Kind: regression
- File: `src/agents/openai-transport-stream.ts`
- Line: 2943
- Evidence: `shouldPreserveOpenRouterReasoningReplay()` returns `false` for OpenRouter `anthropic/*` and `x-ai/*`, so `sanitizeCompletionsReasoningReplayFields()` takes the `stripCompletionsReasoningReplayFields()` path and deletes `reasoning_details` entirely. A local payload probe showed `deepseek/deepseek-v4-flash` preserved a valid `reasoning_details` array, while `anthropic/claude-sonnet-4.6` and `x-ai/grok-4.3` dropped it and kept only `content`, `role`, and `tool_calls`. OpenRouter’s docs say `reasoning_details` arrays should be passed back unmodified for supported reasoning models, including Anthropic and xAI. ([openrouter.ai](https://openrouter.ai/docs/guides/best-practices/reasoning-tokens?utm_source=openai))
- Impact: Multi-turn OpenRouter conversations on Anthropic/xAI lose valid structured reasoning continuity, especially around tool calls. The original bug was a string sentinel problem, but the patch removes the valid array case too.
- Suggested fix: Keep using OpenRouter-specific sanitation for Anthropic/xAI, but make it value-shape based: strip or ignore string sentinel fields like `"reasoning_details"`, `"reasoning"`, `"reasoning_content"`, and `"reasoning_text"`, while preserving `Array.isArray(record.reasoning_details)`.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Run `buildOpenAICompletionsParams()` with provider `openrouter`, model `anthropic/claude-sonnet-4.6`, and an assistant tool call carrying a JSON `thoughtSignature`; the resulting assistant message drops `reasoning_details`.
- Is this the best way to solve the issue? yes. The failure is caused by model-family gating where field-shape gating is needed.
