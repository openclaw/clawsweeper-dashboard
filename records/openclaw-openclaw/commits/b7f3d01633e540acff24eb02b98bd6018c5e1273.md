# Commit b7f3d016

## Summary

The commit fixes the narrow Notion-style direct `$ref`/`oneOf` case, but the new shared inliner weakens other valid tool schemas and breaks an existing focused schema-normalization test on current `main`.

## Findings

### Medium: `$ref` sibling validation constraints are dropped

- Kind: bug
- File: `src/agents/pi-tools-parameter-schema.ts`
- Line: 446
- Evidence: When a local `$ref` resolves, the code clones only the resolved schema and then `copySchemaMeta` restores just `title`, `description`, and `default` from the referring node. Validation siblings such as `minLength`, `pattern`, `minimum`, or object constraints are discarded. A focused probe showed raw TypeBox/pi-ai validation rejects `{ token: "short" }` for `{ $ref: "#/$defs/Token", minLength: 8 }`, while `normalizeToolParameterSchema` rewrites it to `{ type: "string" }` and accepts the same invalid argument. TypeBox’s validator checks string constraints and `$ref` together in `node_modules/typebox/build/schema/engine/schema.mjs:273`.
- Impact: Bundled MCP and other normalized tool schemas can become more permissive, allowing invalid model-generated arguments through OpenClaw validation and onward to the MCP server/provider.
- Suggested fix: Preserve non-meta `$ref` siblings, for example by emitting `allOf: [resolvedSchema]` plus the sibling assertions, or by using a resolver that keeps the full JSON Schema assertion semantics. Add a regression for `$ref` plus `minLength`/`pattern`.
- Confidence: high

### Medium: Recursive local refs are replaced with permissive `{}`

- Kind: bug
- File: `src/agents/pi-tools-parameter-schema.ts`
- Line: 430
- Evidence: Cycle detection returns `{}` when the same local `$ref` appears in `refStack`. A focused probe with a recursive `Node` definition showed the raw schema rejects `{ node: { id: "root", child: {} } }`, but the normalized schema becomes `child: {}` and accepts it.
- Impact: Recursive MCP/tool input schemas lose validation below the first repeated ref. Invalid nested arguments can pass local validation.
- Suggested fix: Do not replace cycles with `{}`. Keep the unresolved `$ref` and preserve the needed `$defs`/`definitions`, or implement recursion-safe inlining that preserves the recursive constraint. Add a recursive `$defs` regression.
- Confidence: high

### Low: Object rebuilding drops own `__proto__` schema properties

- Kind: regression
- File: `src/agents/pi-tools-parameter-schema.ts`
- Line: 456
- Evidence: `inlineLocalSchemaRefsWithDefs` rebuilds every schema object using a plain `{}` and `result[key] = ...`. For an own key named `__proto__`, that assignment does not create a normal own property. The existing focused test `src/agents/pi-tools.schema.test.ts:645` now fails, and a probe showed normalized properties lose the own `__proto__` entry.
- Impact: Current focused tests fail on `main`, and any tool schema with a literal `__proto__` parameter loses that parameter’s schema during normalization.
- Suggested fix: Rebuild maps with `Object.fromEntries`, `Object.create(null)`, or `Object.defineProperty` for arbitrary property names. Avoid rebuilding unchanged subtrees when no refs were inlined.
- Confidence: high
