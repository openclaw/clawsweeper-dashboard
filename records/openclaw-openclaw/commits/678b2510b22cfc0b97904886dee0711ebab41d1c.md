# Commit 678b2510b2

## Summary

Found one reliability bug: the new generic critical loop block fires once, but an immediate retry of the same tool call is allowed because the blocked result is recorded into loop history and breaks the identical-outcome streak.

## Findings

### Medium: Generic loop blocks are not sticky after the first critical block

- Kind: reliability
- File: `src/agents/tool-loop-detection.ts`
- Line: 618
- Evidence: The new generic critical branch blocks when `noProgressStreak >= criticalThreshold`, but `wrapToolWithBeforeToolCallHook` records the returned blocked tool result back into the same history at `src/agents/pi-tools.before-tool-call.ts:706`. That blocked result has a different result hash than the repeated successful outcome, so the next identical call sees only a warning and executes.
- Impact: A model stuck in the same generic tool call can run another 20 identical calls after each critical block. That undermines the commit’s stated behavior of aborting generic no-progress loops and still permits runaway tool/model cycles for stubborn agents.
- Suggested fix: Do not record `deniedReason: "tool-loop"` blocked results as normal outcomes for that same tool/args, or have no-progress streak calculation ignore loop-blocked outcomes and preserve the last non-blocked result. Add an e2e regression: after the first generic critical block, an immediate identical retry must also be blocked and must not increment the underlying tool execute count.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. A focused probe through `wrapToolWithBeforeToolCallHook` ran 20 identical successful `read` calls, saw the 21st call blocked, then saw the 22nd identical call execute successfully. The probe output included `blockedStatus: "blocked"`, `retryStatus: "executed"`, and `executed: 21`.
- Is this the best way to solve the issue? yes. Keeping loop-denied synthetic results out of the no-progress history preserves the evidence that caused the critical block while avoiding repeated unblock windows.
