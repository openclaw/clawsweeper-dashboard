# Commit 2d231ce

## Summary

Found one low-severity migration UI regression: the preview now exposes raw internal block codes for skipped manual-review Codex plugin items, replacing the actionable message text.

## Findings

### Low: Manual-review plugin preview now hides actionable messages behind raw block codes

- Kind: regression
- File: `src/commands/migrate/output.ts`
- Line: 160
- Evidence: In preview mode, every `conflict`, `skipped`, or `error` item now returns `humanizeReason(item.reason) ?? item.message`. Codex migration creates manual-review plugin items with `kind: "manual"`, `status: "skipped"`, `reason: plugin.migrationBlock.code`, and an actionable `message` in `extensions/codex/src/migration/plan.ts:203`. Those block codes include values such as `plugin_disabled`, `plugin_read_unavailable`, `app_inaccessible`, and `app_disabled` in `extensions/codex/src/migration/source.ts:50`, but most are not in `REASON_CODE_MESSAGES`. A focused smoke produced: `🔍 gmail (app_disabled)` instead of the existing explanatory message.
- Impact: `openclaw migrate plan --provider codex --verify-plugin-apps` can show raw internal status codes for manual-review plugin rows and omit the concrete remediation message, such as enabling/authenticating the app in Codex. This weakens the migration preview right where users need the reason for manual follow-up.
- Suggested fix: Scope the new reason-first preview behavior to the intended skill/plugin install conflict rows, or preserve `item.message` first for `manual` and other non-install/copy items. Add an output test for a skipped manual item with both `reason` and `message`.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes: format a preview plan containing a skipped manual item with `reason: "app_disabled"` and a `message`; current output renders `(app_disabled)`.
- Is this the best way to solve the issue? yes: keeping manual item messages first restores the previous contract while preserving the new conflict wording for skill/plugin rows.
