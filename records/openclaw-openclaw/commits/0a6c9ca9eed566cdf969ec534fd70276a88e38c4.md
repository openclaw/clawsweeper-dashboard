# Commit 0a6c9ca

## Summary

Found one actionable ClawHub publishing regression risk: several channel plugins are opted into ClawHub publishing, but the ClawHub publish path does not include the generated `channelConfigs` metadata that npm publishing overlays.

## Findings

### Medium: ClawHub-published channel plugins can lose setup/config metadata

- Kind: regression
- File: `extensions/twitch/package.json`
- Line: 45
- Evidence: The commit enables `openclaw.release.publishToClawHub` for Twitch, and similarly for Feishu, Synology Chat, and Tlon at `extensions/feishu/package.json:56`, `extensions/synology-chat/package.json:42`, and `extensions/tlon/package.json:81`. Their `openclaw.plugin.json` files declare `channels`, but do not contain `channelConfigs`; Synology Chat, Tlon, and Twitch also have empty root `configSchema.properties`. `scripts/plugin-clawhub-publish.sh:46` publishes the raw package directory with `clawhub package publish`, while npm publishing uses `scripts/lib/plugin-npm-package-manifest.mjs` to overlay generated channel metadata.
- Impact: ClawHub installs of these newly publishable channel plugins can register as external channel plugins without `channelConfigs`, so channel config schema/setup surfaces are missing before runtime loads. The manifest registry explicitly warns for non-bundled channel plugins in this state at `src/plugins/manifest-registry.ts:451`.
- Suggested fix: Apply the same generated manifest overlay to `scripts/plugin-clawhub-publish.sh` before `clawhub package publish`, or commit the required `openclaw.plugin.json#channelConfigs` metadata before enabling `publishToClawHub` for these channel plugins.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. `node scripts/lib/plugin-npm-package-manifest.mjs --run extensions/twitch -- node -` shows npm overlay adds `channelConfigs: ["twitch"]`, while the checked-in ClawHub source manifest has `channels: ["twitch"]` and no `channelConfigs`.
- Is this the best way to solve the issue? yes. Reusing the existing npm manifest overlay keeps ClawHub and npm artifacts aligned without duplicating generated channel schema by hand.
