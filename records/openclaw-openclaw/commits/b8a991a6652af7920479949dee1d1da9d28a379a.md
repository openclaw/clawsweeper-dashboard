# Commit b8a991a

## Summary

Found one incomplete-fix issue: the heartbeat sanitizer added by this commit still misses the colon-style `[TOOL_CALL]{tool: ..., args: ...}[/TOOL_CALL]` format described in the linked bug report, so that class of raw tool-call marker can still reach user-facing heartbeat replies.

## Findings

### Medium: Colon-style legacy TOOL_CALL blocks are still delivered

- Kind: security
- File: `src/shared/text/assistant-visible-text.ts`
- Line: 357
- Evidence: The new heartbeat path calls `stripLegacyBracketToolCallBlocks` from `src/auto-reply/reply/agent-runner-payloads.ts:95`, but `isLegacyBracketToolCallPayload` only matches `tool => "..."` plus `args =>`. The issue body for #54138 explicitly reports `[TOOL_CALL]{tool: "exec", args: {...}}[/TOOL_CALL]`. A focused probe confirmed the current sanitizer strips arrow-style calls and `TOOL_RESULT` blocks but leaves the reported colon-style call unchanged:
  - arrow style -> `"Before\n\nAfter"`
  - colon style -> `"Before\n[TOOL_CALL]{tool: \"exec\", args: {\"command\":\"ls\"}}[/TOOL_CALL]\nAfter"`
- Impact: Heartbeat replies can still leak raw tool invocation text, including command arguments, paths, or token-bearing tool payloads, to Telegram/Discord/other channel recipients for the reported colon-style pseudo-call syntax.
- Suggested fix: Extend `isLegacyBracketToolCallPayload` to accept the same JSON-ish/colon style already accepted for `TOOL_RESULT`, while still preserving fenced-code examples. Add a heartbeat `buildReplyPayloads` regression test using `[TOOL_CALL]{tool: "exec", args: {...}}[/TOOL_CALL]`.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes: call `stripLegacyBracketToolCallBlocks` with `Before\n[TOOL_CALL]{tool: "exec", args: {"command":"ls"}}[/TOOL_CALL]\nAfter`; the marker remains unchanged.
- Is this the best way to solve the issue? yes: the commit already introduced a narrow predicate-based sanitizer, so broadening the predicate and covering the exact reported syntax is the smallest fix.
