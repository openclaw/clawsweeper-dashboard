### Medium: New video assertions fail extension test typecheck

- Kind: regression
- File: `extensions/openrouter/video-generation-provider.test.ts`
- Line: 254
- Evidence: `requireGeneratedVideo(result, 0)` only proves `result.videos[index]` exists. The returned `GeneratedVideoAsset` still has `buffer?: Buffer`, so `video.buffer.toString()` fails strict TypeScript. `pnpm tsgo:extensions:test` reports `TS18048: 'video.buffer' is possibly 'undefined'` at `extensions/openrouter/video-generation-provider.test.ts:254` and `extensions/openrouter/video-generation-provider.test.ts:338`.
- Impact: The extension test typecheck lane is broken on current `main`, blocking changed-gate/CI validation for extension test changes even though the Vitest runtime test passes.

- Confidence: high
