## Findings

### Low: New preflight examples are not valid channel-turn API usage

- Kind: bug
- File: `docs/plugins/sdk-channel-turn.md`
- Line: 345
- Evidence: The new examples return `history: { key, limit, ... }` from `preflight`, but `ChannelTurnDroppedHistoryOptions` requires `historyMap` at `src/channels/turn/types.ts:319`. The same section also returns `access` from `preflight` at `docs/plugins/sdk-channel-turn.md:409`, but `PreflightFacts` only accepts `admission`, `command`, `message`, `media`, `supplemental`, and `history` at `src/channels/turn/types.ts:236`.
- Impact: Plugin authors copying these TypeScript examples will either fail type-checking or, in untyped code, can crash dropped-history recording because `recordDroppedChannelTurnHistory` passes `history.historyMap` into history helpers at `src/channels/turn/kernel.ts:211`.
- Suggested fix: Add `historyMap` to each `history` example, and move mention `access` facts into the resolved turn or show `resolveTurn` consuming a preflight-local mention decision.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. Copy the documented `preflight` snippets into a typed `ChannelTurnAdapter`: `historyMap` is missing from the required history contract, and `access` is not part of `PreflightFacts`.
- Is this the best way to solve the issue? yes. The runtime contract already requires `historyMap`, and access facts belong in the resolved turn shape, so the docs should match the existing API rather than changing runtime behavior.
