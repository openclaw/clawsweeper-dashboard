# Commit d033c369

## Summary

Found one remaining Telegram access-group regression on current `main`: authorized access-group users can pass the generic callback allowlist gate, but model-selection callbacks still re-check raw `allowFrom` entries and deny them.

## Findings

### Medium: Telegram model callback auth skips access-group expansion

- Kind: regression
- File: `extensions/telegram/src/bot-handlers.runtime.ts`
- Line: 858
- Evidence: The generic direct callback gate expands access groups before authorizing DMs at `extensions/telegram/src/bot-handlers.runtime.ts:794`, but `isTelegramModelCallbackAuthorized` later recomputes DM authorization with raw `dmAllowFrom` at `extensions/telegram/src/bot-handlers.runtime.ts:858`. `normalizeAllowFrom` treats `accessGroup:operators` as an invalid non-numeric Telegram ID, so `isSenderAllowed` returns false. A focused helper smoke returned `expanded:["123456789"]`, `genericCallbackAllowed:true`, and `modelCallbackRawAllowed:false`.
- Impact: A Telegram user authorized through `channels.telegram.allowFrom: ["accessGroup:operators"]` can send messages and reach model UI callbacks, but pressing `mdl_prov`, `mdl_list_*`, or `mdl_sel_*` is silently blocked as “not authorized for /models”.
- Suggested fix: Make `isTelegramModelCallbackAuthorized` async and expand `dmAllowFrom` with `expandTelegramAllowFromWithAccessGroups` before `normalizeDmAllowFromWithStore`, matching `authorizeTelegramEventSender`. Add a callback test with `dmPolicy: "allowlist"` and `allowFrom: ["accessGroup:operators"]`.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes: configure a Telegram DM allowlist via `accessGroups`, open the `/models` UI as that sender, then press any model inline button. The direct helper smoke reproduces the exact raw-vs-expanded authorization mismatch.
- Is this the best way to solve the issue? yes: the model callback helper should reuse the same expanded allowlist semantics as the generic callback authorization path introduced by this commit.
