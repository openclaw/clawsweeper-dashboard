# Commit 5b528f4

## Summary

Found one low-severity docs regression: the generated plugin inventory now says bare plugin installs try ClawHub first, but current CLI code and tests still route bare installs through npm unless the user explicitly prefixes `clawhub:`.

## Findings

### Low: Plugin inventory documents the wrong default install source

- Kind: bug
- File: `scripts/generate-plugin-inventory-doc.mjs`
- Line: 561
- Evidence: The new generated text says “Bare package specs try ClawHub first, then npm fallback” in `scripts/generate-plugin-inventory-doc.mjs:561` and `docs/plugins/plugin-inventory.md:43`. Current CLI flow does not do that: explicit `clawhub:` is handled only after bundled/official-external npm plans, and ordinary bare specs fall through to npm at `src/cli/plugins-install-command.ts:830`. The focused contract test at `src/cli/plugins-cli.install.test.ts:829` asserts bare `demo` installs through npm and `installPluginFromClawHub` is not called.
- Impact: Users reading the plugin inventory may believe an unprefixed install is using ClawHub scanning/metadata and fallback semantics when it is actually npm-backed. That is especially relevant for plugin supply/source trust decisions.
- Suggested fix: Change the generated inventory prose to match current behavior, for example “Bare package specs install from npm by default; use `clawhub:` to force ClawHub.” If ClawHub-first is the intended new policy, update the installer and its tests together instead of changing only docs.
- Confidence: high

## Details

- Do we have a high-confidence way to reproduce the issue? yes. The focused test `pnpm test src/cli/plugins-cli.install.test.ts -- -t "installs ordinary bare plugin specs through npm without ClawHub lookup"` passed after installing dependencies, confirming the current npm-first/no-ClawHub behavior.
- Is this the best way to solve the issue? yes for this docs-only commit: align the generated prose with existing CLI behavior. A behavior change to ClawHub-first would be broader and needs code, tests, and docs updated consistently.
