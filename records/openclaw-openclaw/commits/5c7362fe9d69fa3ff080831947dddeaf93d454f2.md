# Commit 5c7362f

## Summary

The commit removes the runtime export for `LEGACY_DAEMON_CLI_EXPORTS`, but the build helper still imports it. This breaks the build’s `write-cli-compat` step.

## Findings

### High: Build helper imports a value that is no longer exported

- Kind: regression
- File: `src/cli/daemon-cli-compat.ts`
- Line: 1
- Evidence: `src/cli/daemon-cli-compat.ts:1` now declares `const LEGACY_DAEMON_CLI_EXPORTS` without `export`, while `scripts/write-cli-compat.ts:5` imports `LEGACY_DAEMON_CLI_EXPORTS` and uses it at `scripts/write-cli-compat.ts:75`, `scripts/write-cli-compat.ts:166`, and `scripts/write-cli-compat.ts:185`. `scripts/build-all.mjs:93` runs `node --import tsx scripts/write-cli-compat.ts` as part of `pnpm build`, and `package.json:1261` also runs it in `build:docker`.
- Impact: `pnpm build` / Docker artifact builds fail before the legacy daemon CLI shim can be written, blocking package/build workflows.
- Suggested fix: Restore `export const LEGACY_DAEMON_CLI_EXPORTS` in `src/cli/daemon-cli-compat.ts`, or move the runtime list into `scripts/write-cli-compat.ts` so the script no longer imports a private value. The minimal fix is to keep the value exported while leaving the type-only shape exports trimmed.
- Confidence: high
