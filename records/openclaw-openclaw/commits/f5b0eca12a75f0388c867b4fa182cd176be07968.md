## Findings

### Low: Canonical tool-profile docs are incomplete after moving policy details

- Kind: regression
- File: `docs/tools/index.md`
- Line: 15
- Evidence: The new overview says exhaustive tool policy, defaults, and group membership live in `docs/gateway/config-tools.md`. That table currently lists the `coding` profile without `music_generate` at `docs/gateway/config-tools.md:26`, while `src/agents/tool-catalog.ts:288` defines `music_generate` with `profiles: ["coding"]`. The removed `/tools` profile table in the parent commit did include `music_generate`, and it also documented that `coding` and `messaging` allow configured `bundle-mcp`; current source still adds `"bundle-mcp"` to both profiles at `src/agents/tool-catalog.ts:328`.
- Impact: Users following the new routing page for exact profile membership can wrongly conclude that `music_generate` or bundle MCP tools are not present in the default `coding` profile, leading to unnecessary allowlist changes or incorrect troubleshooting.
- Suggested fix: Update `docs/gateway/config-tools.md` so the profile table matches `src/agents/tool-catalog.ts`, at minimum restoring `music_generate` and the `bundle-mcp` profile behavior that this refactor moved out of `/tools`.
- Confidence: high
