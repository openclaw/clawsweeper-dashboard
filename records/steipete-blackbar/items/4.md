## Summary

## What This Changes

The branch changes `BlacksmithStatus.isOperational` to require no maintenances, adds a maintenance-only model test, and adds an unreleased changelog note.

## Best Possible Solution

Keep maintenance notices visible, but make operational state status-aware so scheduled `NOTSTARTEDYET` maintenance remains operational while active maintenance states are non-operational, with regression coverage for both cases.

## Reproduction Assessment

Yes. Source inspection shows current main returns `All systems operational` for an `UP` status with no incidents and a maintenance entry, and the PR regression is source-reproducible because scheduled `NOTSTARTEDYET` entries are mapped into `maintenances` without filtering. ([status.blacksmith.sh](https://status.blacksmith.sh/public-api?utm_source=openai))

## Solution Assessment

No, not as written. The narrow maintainable fix is to distinguish scheduled maintenance notices from maintenance that is actually underway, rather than treating every maintenance entry as non-operational.
