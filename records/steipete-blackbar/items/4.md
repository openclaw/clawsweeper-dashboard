## What This Changes

The branch adds `maintenances.isEmpty` to `BlacksmithStatus.isOperational`, adds a maintenance-only model test, and adds an unreleased changelog note.

## Best Possible Solution

Keep maintenance notices visible, but make operational state maintenance-status-aware so planned `NOTSTARTEDYET` entries remain operational while active maintenance states are non-operational, with regression coverage and real behavior proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main returns `All systems operational` for `pageStatus: "UP"` with no incidents and a maintenance entry. The PR regression is also source-reproducible because the public API can include planned `NOTSTARTEDYET` entries in `activeMaintenances`.

## Solution Assessment

No, not as written: the fix should distinguish scheduled maintenance notices from maintenance that is actually underway. A status-aware predicate is narrower and avoids changing the operational signal before a planned window starts.
