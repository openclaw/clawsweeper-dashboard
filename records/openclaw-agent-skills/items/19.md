## Summary

Keep this PR open, but it should not merge as-is: the useful Claude fallback support is not on current main, yet the patch applies global fallback defaults to non-Claude reviewers and can break the default Codex path.

## What This Changes

The PR adds autoreview environment defaults for model, thinking, and fallback settings, forwards Claude `--fallback-model`, and expands autoreview model documentation.

## Best Possible Solution

Scope fallback defaults so global fallback values are consumed only by Claude reviewers, while explicitly keyed non-Claude fallback values such as `codex=x` still fail clearly.

## Reproduction Assessment

Yes from source inspection: with the PR diff, a global `--fallback-model` or `AUTOREVIEW_FALLBACK_MODEL` becomes `fallback_model` for every reviewer and is then rejected when the reviewer is Codex. I did not mutate the checkout to run the PR branch, but the patched control flow is direct.

## Solution Assessment

No; the useful Claude fallback support should not globally fail non-Claude reviewers. The narrower maintainable path is to apply global fallback defaults only to Claude and reject only explicit non-Claude keyed fallback settings.
