## Summary

This PR is useful and has strong live proof, but it should not merge unchanged because its advertised fail-closed Cursor isolation misses the supported root `.cursorrules` project instruction file; since the runner then launches Cursor with `--trust`, an untrusted checkout can still influence the review.

## What This Changes

Adds Cursor as an optional autoreview engine with Ask-mode execution, workspace-instruction and MCP trust gates, structured output parsing, documentation, test-harness support, unit tests, and CI validation.

## Best Possible Solution

Extend the preflight scan and focused tests to cover every repository-controlled instruction source Cursor currently supports, at minimum root `.cursorrules`, before retaining the documented fail-closed guarantee and merging the engine.

## Reproduction Assessment

Yes at source level: a repository containing root `.cursorrules` but none of the currently scanned files passes the preflight guard even though Cursor supports that file as project instructions. A focused unit test can establish this without invoking Cursor.

## Solution Assessment

No, not yet. The engine abstraction, output parsing, and explicit trust controls are reasonable, but the narrowest safe solution must align the refusal scan with Cursor's full supported project-rule discovery contract.
