## Summary

Keep this PR open: Droid reasoning-effort support is useful and current main does not have it, but the PR's Droid `stream-json` switch is not ready because the extractor does not handle the event fields shown in the contributor's own live proof.

## What This Changes

The PR updates the autoreview skill and helper to allow Droid reasoning-effort values, pass them through to `droid exec`, and use `stream-json` for Droid streaming output.

## Best Possible Solution

Keep the Droid reasoning-effort support, but land it with either compatible Droid streaming output or explicit extraction and validation for the Droid `stream-json` event fields.

## Reproduction Assessment

Yes. Source inspection plus the contributor's live stream proof gives a high-confidence path: Droid stream-json emits the final report in top-level `text` or `finalText`, but the extractor does not read those fields.

## Solution Assessment

No, not as-is. Forwarding Droid reasoning effort is the right narrow direction, but the streaming output-format change needs matching parser support or should be split out.
