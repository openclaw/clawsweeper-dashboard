## Summary

Keep open: current main still carries the v2026.5.12/pnpm 11 gateway dependency FOD that the report targets, and no newer issue or PR appears to supersede the reported current-main hash instability. The prior live-fetch issue was closed after offline install work, but this report is narrower: the fixed-output dependency derivation itself is producing changing hashes.

## What This Changes

The issue reports that `openclaw-gateway-pnpm-deps` for the current v2026.5.12/pnpm 11 pin downloads npm packages inside the fixed-output derivation and produces different output hashes across repeated builds.

## Best Possible Solution

Make the gateway pnpm dependency fixed-output derivation deterministic for the current pnpm 11/OpenClaw v2026.5.12 pin, then update the stored hash only after repeated rebuilds produce the same output.
