## What This Changes

The issue reports that `openclaw-gateway-pnpm-deps` still downloads npm tarballs during the dependency fixed-output derivation and produces different output hashes across repeated realizes.

## Best Possible Solution

Make the gateway dependency fixed-output derivation deterministic for the current pnpm 11/OpenClaw v2026.5.12 pin, update the pinned `pnpmDepsHash`, and add a focused maintainer validation path for repeated realizes of the dependency output.
