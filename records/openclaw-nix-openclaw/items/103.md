## Best Possible Solution

Land the path routing if maintainers accept the pre-1.0 migration away from the internal profile symlink; otherwise add a deliberate compatibility or cleanup path before merge.

## Reproduction Assessment

## Solution Assessment

Yes, with a compatibility caveat: using OpenClaw tools.exec.pathPrepend and Codex shell_environment_policy.set.PATH is the narrower supported contract. The unresolved part is whether maintainers want to accept removing the old internal profile path without cleanup.
