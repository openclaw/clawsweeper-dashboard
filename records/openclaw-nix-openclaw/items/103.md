Full review comments:

- **[P1] Preserve or migrate inherited HOME state:** `nix/scripts/openclaw-codex-app-server-wrapper.sh:13`
  - body: This unconditionally starts the managed Codex app-server with `HOME=$CODEX_HOME/home` instead of the gateway's inherited HOME. Existing users can have Codex credentials, provider choices, CLI config, `.agents` files, or caches under the inherited HOME, so the upgrade can make Codex start with fresh state even though `runtimePackages` resolve. Please either preserve/migrate the needed state, make the behavior opt-in, or provide explicit maintainer-approved upgrade proof.
  - confidence: 0.88
- **[P1] Do not abort Codex startup on profile conflicts:** `nix/scripts/openclaw-codex-app-server-wrapper.sh:28-38`
  - body: When `$CODEX_HOME/home/.nix-profile/bin` already exists as a non-store symlink or as a regular path, this wrapper exits before `exec`ing Codex. That turns pre-existing native-home profile state into an app-server availability failure; handle it non-fatally, migrate it, or document a tested repair path before merge.
  - confidence: 0.86
- **[P3] Split or explicitly accept the oversized proof files:** `nix/checks/openclaw-runtime-path.nix:1`
  - body: `maintainers/packaging.md` says to keep files under 400 lines unless a maintainer explicitly accepts the larger file. This PR adds a 436-line Nix check and a 511-line JavaScript smoke, so either split the proof into smaller audit units or get visible maintainer acceptance for the size.
