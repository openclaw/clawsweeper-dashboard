## What This Changes

This PR splits generic runtime tool PATH generation from the Codex app-server adapter, adds a Nix Codex launcher that sets `HOME=$CODEX_HOME/home` and links runtime bins, removes activation-time Codex profile linking, and adds runtime-path proof coverage and docs.

## Best Possible Solution

Land a maintainer-approved version that preserves or migrates existing Codex home/auth state, explicitly handles the fail-closed profile case, keeps user path precedence, and either splits or accepts the large proof files.
