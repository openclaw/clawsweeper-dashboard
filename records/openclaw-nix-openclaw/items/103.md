Full review comments:

- **[P1] Preserve or migrate inherited HOME state:** `nix/scripts/openclaw-codex-app-server-wrapper.sh:13`
  - body: This starts managed Codex with `HOME=$CODEX_HOME/home`, while upstream v2026.6.8 explicitly documents that normal local app-server launches keep inherited `HOME` so subprocesses can find user-home config and tokens. Existing users can silently lose access to Codex/CLI credentials, provider choices, caches, or other HOME-dependent state on upgrade unless the PR adds a tested migration/compatibility path or maintainer-visible acceptance of that break.
  - confidence: 0.89
- **[P1] Do not abort Codex startup on profile conflicts:** `nix/scripts/openclaw-codex-app-server-wrapper.sh:29-30`
  - body: When `$CODEX_HOME/home/.nix-profile/bin` is a non-store symlink or non-symlink, the new launcher exits before starting Codex. That turns local profile state into an app-server availability failure at runtime; add a tested repair path or preserve startup with a warning unless maintainers intentionally want this fail-closed upgrade behavior.
  - confidence: 0.84
- **[P3] Split or explicitly accept the oversized proof files:** `nix/scripts/openclaw-runtime-path-smoke.mjs:1`
  - body: The new runtime proof adds a 436-line Nix check and a 511-line JavaScript smoke, while the maintainer packaging guide says to keep files under 400 lines unless a maintainer explicitly accepts the larger file. Split these or record explicit maintainer acceptance before merge.
  - confidence: 0.93
