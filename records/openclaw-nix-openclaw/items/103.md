## What This Changes

The branch routes Home Manager `runtimePackages` into the gateway wrapper and generated `tools.exec.pathPrepend`, replaces the Codex-home profile linker with legacy symlink cleanup, and adds runtime-path checks.

## Best Possible Solution

Land the `tools.exec.pathPrepend`-based runtime package path after maintainers explicitly accept the legacy-profile cleanup and the supported Linux/Darwin proof gates pass.

## Reproduction Assessment

Yes. Current main source shows `runtimePackages` wired into the gateway wrapper PATH and legacy Codex-home profile activation, while the generated OpenClaw config lacks the new `tools.exec.pathPrepend` injection that this PR adds.

## Solution Assessment

Yes, with a maintainer compatibility decision. Using upstream `tools.exec.pathPrepend` plus the gateway wrapper is the narrow package-owned path, but removing the old symlink should be accepted explicitly before merge.
