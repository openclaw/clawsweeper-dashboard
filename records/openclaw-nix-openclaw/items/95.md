## Summary

This should stay open: current main still generates `apiKey` and other SecretRef-shaped options as overlapping `t.oneOf` submodules, while the earlier fix only made `secrets.providers` evaluable.

## What This Changes

The issue requests support for resolving SecretRefs such as `models.providers.<name>.apiKey` through non-`env` secret providers like `exec` and `file`.

## Best Possible Solution

Fix the config-option generator so same-key `source`-discriminated SecretRef unions become an evaluable Nix type, then regenerate options and add coverage for a non-env `apiKey` reference.

## Reproduction Assessment

Yes. A Home Manager eval using `models.providers.groq.apiKey = { source = "exec"; provider = "aws_test"; id = "value"; }` should exercise the generated `apiKey` type; source inspection shows it still has the overlapping `oneOf` shape that the related issue documented as failing in Nix.

## Solution Assessment

Yes. The narrowest maintainable fix is in `nix/scripts/generate-config-options.ts`, with regenerated golden options and regression coverage, rather than hand-editing generated Nix output.
