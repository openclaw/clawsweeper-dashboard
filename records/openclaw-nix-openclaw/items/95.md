## Summary

Keep open. Current main still emits the reported SecretRef apiKey shape as nested oneOf submodules, while the related provider-only fix covers secrets.providers but not same-key SecretRef unions.

## What This Changes

The issue reports that SecretRef options such as model provider apiKey cannot reference non-env secret providers because generated Nix types collapse env/file/exec SecretRef variants through nested oneOf submodules.

## Best Possible Solution

Update the generator so SecretRef-style unions with identical fields still collapse the source discriminator into one accepted enum, then regenerate options and add coverage beside the existing secrets.providers check.

## Reproduction Assessment

Yes from source inspection, but not from an executed Nix evaluation in this read-only review. Current main still generates provider apiKey SecretRefs as nested t.oneOf submodules for env, file, and exec, and the generator falls back for same-key object unions.

## Solution Assessment

Yes. A narrow generator fix plus regenerated options and targeted default-instance regression coverage is the maintainable solution; hand-editing individual generated apiKey blocks would drift from the schema pipeline.

## Repair Work Prompt

Fix the SecretRef oneOf-of-submodules regression in nix-openclaw. Current main still emits apiKey and other SecretRef options as t.oneOf [ t.str (t.oneOf [ env/file/exec submodules ]) ], because nix/scripts/generate-config-options.ts refuses to collapse source-discriminated object unions when every variant has the same id/provider/source keys. Update the generator, not hand-written generated Nix, so same-key SecretRef variants accept source values env, file, and exec without Nix's submodule oneOf ambiguity. Regenerate nix/generated/openclaw-config-options.nix and add focused coverage beside nix/checks/openclaw-default-instance.nix proving a provider apiKey SecretRef with source = "exec" or "file" evaluates, while preserving the existing secrets.providers file variant coverage from the related fix. Do not add new config options or change runtime secret semantics.
