## Summary

Keep open for maintainer judgment under this repo's conservative cleanup policy: current main does not install ByteRover 4.0.0 as published, and the remaining question is whether to close this as third-party packageability/support or change the fail-closed Nix plugin contract.

## What This Changes

The issue asks how to install the third-party ByteRover context-engine plugin alongside packaged runtimePlugins in nix-openclaw.

## Best Possible Solution

Keep the Nix runtime-plugin contract fail-closed, ask ByteRover to publish bundled dependencies or npm-shrinkwrap.json, then use runtimePluginSources plus plugins.slots.contextEngine config instead of raw load paths.

## Reproduction Assessment

Yes, at source level: ByteRover 4.0.0 declares a runtime dependency, the tarball lacks node_modules and npm-shrinkwrap.json, and current installer code fails that artifact shape. I did not run a Nix build because this was a read-only review.

## Solution Assessment

No automatic repo fix is clearly best: the current fail-closed behavior matches README and module policy. Installing ByteRover as published needs an upstream artifact change or a maintainer decision to expand nix-openclaw's dependency materialization model.
