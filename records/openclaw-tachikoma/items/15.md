## Summary

Keep open: current main still sends Google/Gemini tool schemas with unfiltered orphan `required` entries, and the companion Peekaboo root-cause fix does not replace this Tachikoma provider guard.

## What This Changes

The PR changes `GoogleProvider.convertTool` to filter `required` names to declared properties, warn on dropped names, and omit empty `required` before serializing Gemini tool schemas.

## Best Possible Solution

Land the Google provider sanitization with focused request-serialization regression coverage so malformed external tool schemas cannot make Gemini requests fail.

## Reproduction Assessment

Yes, at source level: create an `AgentTool` whose `required` array names a property missing from `properties`; current main serializes that orphan unchanged in the Google request body. I did not run a live Gemini call in this read-only review.

## Solution Assessment

Yes. Filtering at the Google provider boundary is a narrow defensive fix for strict Gemini validation while still leaving upstream schema conversion fixes useful.
