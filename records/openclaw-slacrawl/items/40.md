## Summary

Keep open: the direct-download Slack path candidate is useful, but the patch changes no-path auto-detection into a hard `desktop path missing` error and makes Redux ingestion silently omit collided entities; it also lacks real behavior proof.

## What This Changes

The branch adds a direct-download macOS Slack Desktop cache candidate to desktop path detection and changes Redux desktop ingestion to continue after workspace-collision errors for users, channels, and messages.

## Best Possible Solution

Keep the added direct-download Slack candidate while preserving non-fatal auto-detect misses, then only tolerate collisions with explicit tests plus reporting, schema support, or maintainer-approved partial-ingest semantics.

## Reproduction Assessment

Yes for the PR-introduced regression from source inspection: the patch can assign an empty detected path, and `slackdesktop.Discover` rejects empty roots with `desktop path missing`. I do not have a real Slack Desktop fixture proving the original direct-download or collision cases end to end.

## Solution Assessment

No. The direct-download path candidate is the right narrow direction, but returning an empty path on misses and silently continuing after store collisions is not the safest maintainable behavior without tests, reporting, or a storage-policy decision.

Full review comments:

- **[P1] Keep auto-detect misses non-fatal:** `internal/config/config.go:418`
  - body: When no desktop candidate exists, this returns an empty path. `Normalize` assigns that value for desktop auto-detection, and `slackdesktop.Discover` rejects empty roots with `desktop path missing`, so an auto-detect miss can become a hard desktop error instead of the previous unavailable-source result. Keep the direct-download candidate, but preserve a non-empty fallback or make the downstream miss explicitly non-fatal.
  - confidence: 0.92
- **[P1] Do not silently drop collided Redux rows:** `internal/slackdesktop/redux.go:181-183`
  - body: Continuing on `already belongs to workspace` makes the import report success while the colliding user, channel, or message is omitted from the store. That is risky for common cross-workspace IDs under the current global-key schema; add explicit reporting/tests or store duplicates under a schema-compatible key before continuing.
  - confidence: 0.86
