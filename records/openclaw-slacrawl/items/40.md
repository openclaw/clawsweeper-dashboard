## Summary

Keep this PR open, but it is not merge-ready: the direct path fallback is useful, while the empty-path fallback and silent Redux collision skips introduce blocking correctness risks and the PR has no real Slack Desktop proof.

## What This Changes

The branch adds a direct-download Slack Desktop path candidate and catches cross-workspace collision errors during Redux user, channel, and message ingestion.

## Best Possible Solution

Keep the direct-download path fallback, preserve the existing no-error unavailable behavior when no desktop path exists, and handle Redux workspace collisions through a tested workspace-aware storage or ingestion policy rather than string-matched silent skips.

## Reproduction Assessment

Yes for the introduced regression by source inspection: default configs enable desktop auto-detect, the PR can leave the path empty, and slackdesktop.Discover rejects empty paths. The reported real Slack collision case itself was not reproduced from a live desktop cache.

## Solution Assessment

No. The direct-download fallback is useful, but returning an empty path and swallowing store collision errors is not the narrowest maintainable fix; preserve unavailable-path behavior and make the collision policy explicit and tested.
