The issue asks for desktop wiretap imports to honor the configured Slack workspace ID and excluded-channel filters.

Extend the desktop import path to accept and enforce the existing workspace and excluded-channel filters before writing desktop-derived workspaces, channels, messages, users, and sync-state rows.

## Repair Work Prompt

Fix desktop/wiretap sync so it respects the existing configured workspace filter and excluded-channel filters. In internal/syncer/syncer.go, pass syncer.Options.WorkspaceID and ExcludeChannels into the desktop path instead of dropping them. In internal/slackdesktop/desktop.go and internal/slackdesktop/redux.go, add a focused options path so desktop ingestion skips non-selected workspaces and excluded channels before upserting desktop-derived workspaces, users, channel hints, drafts/messages, Redux channels/messages, and relevant sync-state rows. Reuse the existing normalization semantics for excluded channel names/IDs where possible; do not add a new config key or change API/MCP sync behavior. Add regression coverage in internal/slackdesktop/desktop_test.go and/or internal/syncer/syncer_test.go for workspace filtering and case-insensitive excluded-channel filtering on desktop data.
