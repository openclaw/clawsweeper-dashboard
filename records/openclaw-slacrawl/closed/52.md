## Summary

Keep open: current main and the latest release still only auto-detect macOS Slack Desktop paths, so Linux `${XDG_CONFIG_HOME}/Slack` or `~/.config/Slack` discovery remains missing while the explicit `slack.desktop.path` workaround is available.

## What This Changes

Add Linux Slack Desktop auto-discovery for the existing blank `[slack.desktop].path` flow.

## Best Possible Solution

Extend `DetectDesktopPath` with OS-aware Linux candidates, preferring `$XDG_CONFIG_HOME/Slack` and falling back to `~/.config/Slack`, with focused config tests and user-facing docs updates.

## Reproduction Assessment

Yes. Source inspection shows a blank desktop path calls `DetectDesktopPath`, and that function only tests macOS `~/Library/...` candidates, so a Linux-only `~/.config/Slack` install would leave the path empty.

## Solution Assessment

Yes. Adding Linux candidates to the existing detection helper is the narrowest maintainable path because explicit `slack.desktop.path` already handles custom locations.
