## Summary

Keep this PR open: it fixes a real current-main concurrency gap, but the PR still has a concrete `/stream` cache correctness blocker before merge.

## What This Changes

The PR routes daily cache misses through a per-host Durable Object, coalesces concurrent generation, preserves `/stream`, stores fallback text, and updates docs/tests.

## Best Possible Solution

Keep the DO coalescing approach, but gate `/stream` edge caching on a complete generated or fallback record and cover the emitted-chunk failure path.

## Reproduction Assessment

Yes from source inspection: current main can generate outside the Durable Object on cold misses, and the PR blocker is reproducible by a `/stream` request whose upstream SSE emits a chunk and then fails. I did not run Cloudflare or local Wrangler in this read-only review.

## Solution Assessment

No as-is: moving generation into the Durable Object and preserving `/stream` is the right maintainable direction, but the streaming response must not let partial failed output become the cached daily page.
