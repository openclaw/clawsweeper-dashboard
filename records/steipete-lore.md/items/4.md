## Summary

Keep open: the PR is a useful fix direction with sufficient happy-path proof, but it can still cache a partial streamed response after a midstream AI failure while the Durable Object stores fallback text for the same host/day.

## What This Changes

The PR routes cold daily generation through the per-host Durable Object, adds in-flight coalescing, preserves `/stream`, stores fallback text, and updates docs/tests.

## Best Possible Solution

Keep the DO coalescing approach, but gate `/stream` edge caching on a complete generated or fallback record and cover the emitted-chunk failure path.

## Reproduction Assessment

Yes from source inspection: current main can generate outside the Durable Object on cold misses, and the PR blocker is reproducible by a `/stream` request whose upstream SSE emits a chunk and then throws. I did not run Cloudflare or local Wrangler in this read-only review.

## Solution Assessment

No as-is: moving generation into the Durable Object and preserving `/stream` is the right maintainable direction, but the streaming response must not let partial failed output become the cached daily page.
