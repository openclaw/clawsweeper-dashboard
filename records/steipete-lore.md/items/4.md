## Summary

Keep open: current main still has the cold-miss Durable Object bypass this PR targets, but this PR head is not merge-ready because the `/stream` response can still be cloned into the daily edge cache before generation is known complete.

## What This Changes

The PR routes normal and streaming cold misses through per-host Durable Object endpoints, adds in-memory generation coalescing and fallback persistence, and updates README/spec/tests.

## Best Possible Solution

Keep the Durable Object coalescing design, but only populate the daily edge cache for `/stream` after the stream has produced a complete record that matches Durable Object storage, with regression coverage for midstream failure after at least one chunk.

## Reproduction Assessment

Yes. Current main source shows cache misses can generate outside the Durable Object after a DO lookup, and the PR blocker is source-reproducible with an SSE stream that emits one chunk and then errors before completion.

## Solution Assessment

No as-is. Moving generation into the Durable Object is the right maintainable direction, but the streaming response must not let partial failed output become the cached daily page.
