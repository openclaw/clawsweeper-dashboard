## Summary

Keep this draft PR open: the updated branch now preserves a DO-backed streaming path and includes usable local Wrangler proof, but it still has a concrete correctness blocker where a failed partial stream can be cached at the edge while the Durable Object stores fallback text.

## What This Changes

The PR routes daily cache misses through `DomainDO`, adds in-memory coalescing for per-version/day generation, preserves `/stream` via a DO streaming path, stores fallback records on generation failure, and updates README/spec/tests.

## Best Possible Solution

Keep the DO-owned coalescing and streaming direction, but make the first `/stream` response cache only a complete generated or fallback daily record and add regression coverage for partial upstream stream failure.

## Reproduction Assessment

Yes from source inspection: current main can generate outside the Durable Object on cold misses, and the PR blocker is also source-reproducible by following a `/stream` request whose upstream SSE emits a chunk and then throws. I did not run Cloudflare or local Wrangler in this read-only review.

## Solution Assessment

No as-is: moving generation into the Durable Object and preserving `/stream` is the right maintainable direction, but the streaming response must not let partial failed output become the cached daily page. A narrow cache/fallback consistency fix should be made before merge.
