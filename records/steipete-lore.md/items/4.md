## Summary

Keep open: the patch addresses a source-visible coalescing bug, but it removes the documented live `/stream` behavior and has no after-fix real behavior proof, so it needs maintainer decision and proof before merge.

## What This Changes

The branch moves daily cache-miss generation into `DomainDO` with an in-memory in-flight map, stores fallback records, routes `/stream` through the same daily path, and updates docs/tests.

## Best Possible Solution

Keep the DO-owned coalescing and stored fallback direction, but preserve live `/stream` behavior or explicitly deprecate it with maintainer approval and upgrade proof.

## Reproduction Assessment

Yes from source inspection: current `main` can generate outside the Durable Object on cold misses and uses the live streaming path, so concurrent edge misses can bypass a single DO-owned generation path. I did not run a live Cloudflare reproduction in this read-only review.

## Solution Assessment

No as-is: moving generation into the DO is the right repair direction, but removing the documented live `/stream` behavior is a compatibility decision rather than a narrow bug fix. The safer path is to preserve streaming while coalescing, or get explicit maintainer approval for deprecation.
