## Summary

Keep open: current main still lacks DO-owned get-or-create coalescing, and this PR is useful, but the head still has a partial `/stream` failure path that can make edge cache content diverge from Durable Object storage.

## What This Changes

This PR routes Worker cache misses through per-host Durable Object `/daily` and `/stream` paths, adds in-memory coalescing and fallback handling, and updates README/spec/tests.

## Best Possible Solution

Keep the Durable Object coalescing design, but cache only a complete generated or fallback daily record that matches Durable Object storage and add regression coverage for midstream SSE failure.

## Reproduction Assessment

Yes. Current main source shows cache misses can generate outside the Durable Object after a DO lookup, and the PR blocker is source-reproducible with an SSE stream that emits one chunk and then errors before completion.

## Solution Assessment

No as-is. Moving generation into the Durable Object is the right maintainable direction, but the streaming response must not let partial failed output become the cached daily page.
