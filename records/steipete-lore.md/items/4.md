## Summary

Keep open: the branch addresses a real main-branch cache/coalescing bug, but it is not merge-ready because the `/stream` response can be cloned into the daily edge cache before upstream streaming has completed, allowing partial content to diverge from the Durable Object fallback record.

## What This Changes

The branch routes normal and streaming cold misses through per-host Durable Object paths, coalesces in-flight daily generation, persists fallback text, and updates README, spec, and Worker tests.

## Best Possible Solution

Keep the Durable Object coalescing design, but only populate the daily edge cache for `/stream` after a complete generated or fallback record matches Durable Object storage, with a chunk-then-error regression test.

## Reproduction Assessment

Yes. Current main source shows cold misses can generate outside the Durable Object after a lookup miss, and the PR blocker is source-reproducible from the cloned `/stream` response being cached before stream completion.

## Solution Assessment

No as-is. Moving generation into the Durable Object is the right maintainable direction, but `/stream` must not let partial failed output become the cached daily page.
