## Summary

Keep open: the Durable Object coalescing direction matches the documented one-generation-per-host/day contract and the contributor proof is sufficient, but the PR still caches the `/stream` response before proving generation completed, so a midstream upstream failure can leave edge cache serving partial HTML while the Durable Object stores fallback text.

## What This Changes

The PR routes daily and streaming cache misses through the per-host Durable Object, adds in-flight coalescing, persists fallback text, and updates README/spec docs plus Worker tests.

## Best Possible Solution

Keep the Durable Object coalescing approach, but make `/stream` edge caching depend on a complete generated or fallback record that matches Durable Object storage, with regression coverage for emitted-then-error SSE.

## Reproduction Assessment

Yes from source inspection: current main can generate outside the Durable Object on cold misses, and the PR blocker follows from a `/stream` request whose upstream SSE emits a chunk and then throws. I did not run Cloudflare or local Wrangler in this read-only review.

## Solution Assessment

No as-is: moving generation into the Durable Object is the narrow maintainable direction, but `/stream` must not let a partial failed stream become the cached daily page.
