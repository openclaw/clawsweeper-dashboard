## Summary

Keep open: the PR is a useful fix direction with live proof for the successful coalescing path, but it still queues edge caching for `/stream` before the stream is known complete, so a midstream failure can cache partial daily content.

## What This Changes

Routes daily and streaming cache misses through per-host Durable Object generation/coalescing, persists fallback text, and updates README, spec, and worker tests.

## Best Possible Solution

Keep the Durable Object coalescing design, but only populate the daily edge cache for `/stream` after a complete generated or fallback record matches Durable Object storage, with a chunk-then-error regression test.

## Reproduction Assessment

Yes. Current main source shows cold misses can generate outside the Durable Object after a lookup miss, and the PR blocker is source-reproducible from the cloned `/stream` response being cached before stream completion.

## Solution Assessment

No as-is. Moving generation into the Durable Object is the right maintainable direction, but the streaming response must not let partial failed output become the cached daily page.
