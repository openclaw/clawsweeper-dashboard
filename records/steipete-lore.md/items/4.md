## Summary

Keep open: the PR addresses a real documented one-generation-per-host/day gap, but current head can still cache a partial `/stream` response when upstream streaming fails after emitting chunks while the Durable Object stores fallback text.

## What This Changes

The PR moves cold daily generation into the per-host Durable Object, adds in-memory coalescing, preserves `/stream`, stores fallback output, and updates docs/tests.

## Best Possible Solution

Keep the DO coalescing design, but cache `/stream` only after it has a complete generated or fallback record matching Durable Object storage, with regression coverage for midstream failure.

## Reproduction Assessment

Yes from source inspection: current main can generate outside the Durable Object on cold misses, and the PR blocker is reproducible with an SSE stream that emits a chunk and then throws. I did not run Wrangler in this read-only review.

## Solution Assessment

No as-is: moving generation into the Durable Object and preserving `/stream` is the right maintainable direction, but the streaming response must not let partial failed output become the cached daily page.
