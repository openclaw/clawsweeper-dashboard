## Summary

This PR is still necessary if maintainers want single-chat imports: current main lacks an `import --chat` mode, while the branch adds a scoped CLI/importer/store path with real terminal proof and focused store coverage. I did not find a blocking correctness or security finding in the diff, but the new partial-import semantics still deserve maintainer product/API review before merge.

## What This Changes

The branch adds `telecrawl import --chat ID`, passes the selected chat to the Python Telegram Desktop importer, skips global folder enumeration in that mode, upserts only the selected chat's archive rows, and adds store coverage for preserving unrelated chats.

## Best Possible Solution

Land the scoped single-chat import mode if maintainers accept partial archive updates as an official import behavior, keeping full-import defaults unchanged and retaining the focused store regression coverage.
