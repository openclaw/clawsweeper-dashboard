## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Isolate recall from same-session transcript history:** `scripts/behavior-eval.mjs:1211`
  - body: The eval sends the seed and recall turns through the same `sessionKey`, so a normal gateway prompt can include the first user message in chat history even if LCM contributes nothing. Since the mock model answers whenever the request text contains both the seed and recall strings, the forward gate can pass on transcript carryover rather than proving context-engine memory; drive the recall from a fresh or compacted session, or otherwise assert the fact reaches the model only through LCM-assembled context.
  - confidence: 0.9
- **[P1] Avoid shelling out with profile-controlled specs:** `scripts/behavior-eval.mjs:894-898`
  - body: The runner executes generated command strings with `shell: true`, while OpenClaw versions and plugin specs come from profiles or CLI overrides. A profile or override containing shell metacharacters can execute more than the intended npm/openclaw command when behavior execution is enabled; build argv arrays and spawn with `shell: false`, following the existing workspace executor pattern.
  - confidence: 0.88
- **[P2] Redact reportable provider errors:** `scripts/behavior-eval.mjs:1065`
  - body: The mock provider writes `formatErrorForReport(error)` directly into JSON responses, and the readiness path can wrap raw gateway logs into error messages. CodeQL has already flagged this flow as stack-trace-dependent information exposure, so report a stable failure class/message while keeping raw stderr or stack traces out of user-facing JSON/HTTP output.
  - confidence: 0.78
