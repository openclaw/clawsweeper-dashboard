## Summary

Keep open: current main still routes the plugin-inspector smoke wrapper to hard-failing `ci` mode under `--check`, while this PR is a narrow, reviewable fix with direct terminal proof and no blocking patch or security findings.

## What This Changes

The PR changes the plugin-inspector smoke wrapper to always invoke `plugin-inspector report` and adds a source-level test to prevent reintroducing ci or author-filtered smoke behavior.

## Best Possible Solution

Land the report-only smoke wrapper after normal CI/review, keeping compatibility findings visible in generated reports while leaving hard-failure policy to the dedicated policy gates.

## Reproduction Assessment

Yes. Source inspection shows `npm run plugin-inspector:smoke` and the static suite pass `--check`, and current main maps that to `plugin-inspector ci`; I did not run the artifact-writing smoke command in the read-only checkout.

## Solution Assessment

Yes. Calling `plugin-inspector report` directly in the wrapper is the narrowest maintainable fix because existing callers can keep their `--check` invocation while the smoke job remains advisory.
