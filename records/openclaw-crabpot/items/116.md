## Summary

Keep open: this member-authored draft addresses a real current-main lifecycle-capture hang, but the patch should wait for a stdout-flush repair before merge.

## What This Changes

The PR wraps the OpenClaw lifecycle capture CLI with explicit success/failure exits and adds a regression test for a loader that leaves an active handle open.

## Best Possible Solution

Keep the explicit exit behavior, but make stdout writes await completion or drain before calling `process.exit()` and preserve active-handle regression coverage.

## Reproduction Assessment

Yes. Source inspection plus the added regression fixture show a loader can leave an active `setInterval()`, while current main only awaits `main()` and has no forced exit path.

## Solution Assessment

No, not quite. Explicitly exiting is the right shape for the active-handle hang, but the stdout path must flush before `process.exit()` to avoid truncating valid CLI output.
