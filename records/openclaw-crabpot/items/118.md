## Summary

Keep open: current main still has unbounded fixture-only profile fallbacks, and this member-authored draft PR needs normal maintainer review, CI, and an explicit acceptance of the best-effort dashboard behavior when profile refreshes fail.

## What This Changes

This PR routes track-dashboard OpenClaw and fixture-only profile commands through bounded timeouts, continues with warnings when fallback profile refreshes fail, skips runtime profile diff when no runtime profile artifact exists, and updates workflow assertions.

## Best Possible Solution

Land a bounded fallback workflow only after CI confirms the assertions and maintainers accept the best-effort profile artifact policy for dashboard refreshes.

## Reproduction Assessment

Yes, at source level: current main times out only the OpenClaw profile attempts, then runs fixture-only fallback profile commands without a timeout. A hanging fallback command would keep the dashboard step stuck; I did not run the live scheduled workflow.

## Solution Assessment

Yes, with a policy caveat: the patch narrowly bounds the existing profile fallback calls and avoids a guaranteed compare failure when no runtime profile exists. Maintainers still need to accept the best-effort dashboard behavior before merge.
