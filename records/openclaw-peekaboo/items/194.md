## Summary

Keep open: current main still has source-visible legacy `elem_*` and `menuitem_*` output while CLI and MCP help advertise B1/T2-style IDs, and the related docs PR only partially mitigates guidance while leaving the runtime contract open.

## What This Changes

The issue asks Peekaboo to make live `see`/`inspect_ui` element IDs and CLI/MCP/docs examples consistently use one public ID contract.

## Best Possible Solution

Adopt one explicit element-ID contract across runtime snapshots, CLI help, MCP descriptions, docs, tests, and the Peekaboo skill, preserving legacy aliases if runtime output changes.

## Reproduction Assessment

Yes. Source inspection shows current AX/menu collectors generate `elem_*`, `menu_*`, and `menuitem_*` IDs while current CLI/MCP help advertises B1/T2-style examples; the reporter also provided live Calculator output from the current main SHA.

## Solution Assessment

Unclear until maintainers choose the supported public contract. Updating runtime output to B1/T1 aligns many help and test examples, but preserving or aliasing existing `elem_*` IDs is important for upgrade safety.
