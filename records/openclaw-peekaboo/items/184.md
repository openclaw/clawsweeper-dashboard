## Summary

Keep this open: current main still permits competing app, on-demand daemon, and MCP bridge hosts to replace the shared socket, and no relevant repair has landed since v3.4.1; the initial sustained-load wedge still needs live macOS diagnosis.

## What This Changes

Fix sustained-capture bridge wedging, ungranted socket takeover, daemon CPU spin, and MCP processes hosting the shared bridge socket.

## Best Possible Solution

Make the permission-bearing app host authoritative for its socket, add observable bounded recovery for host liveness loss, and allow daemon or MCP takeover only through a verified stale-owner protocol with sustained-load regression coverage.

## Solution Assessment

No, the watchdog is a useful diagnostic and recovery workaround but should not become the product architecture. The narrow maintainable direction is authoritative socket ownership, bounded host self-recovery, explicit fallback arbitration, and live macOS regression coverage.
