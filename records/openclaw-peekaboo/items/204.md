## Summary

Keep this PR open: current main and v3.5.2 still have the speech observer polling loop without a stored cancellable task or post-sleep cancellation check, and the merged capture cancellation PR is adjacent rather than superseding. The patch is narrow and plausible, but the provided proof is a standalone poll-loop script rather than production-path proof of the patched SpeechRecognizer/Whisper lifecycle.

## What This Changes

The PR stores the Whisper recorder observer task, cancels it from `stopListening()`, checks `Task.isCancelled` after the polling sleep, and adds a standalone Swift cancellation proof script.

## Best Possible Solution

Land the narrow observer-task lifecycle fix only after production-path proof or maintainer override, ideally with teardown coverage for any speech UI path that can dismiss while recording.

## Reproduction Assessment

Yes at source level: current main has a fire-and-forget speech observer polling loop whose `try? await Task.sleep` swallows cancellation. A live Whisper dismissal/background reproduction was not shown.

## Solution Assessment

Partially. Storing the observer task and checking `Task.isCancelled` is the right narrow direction, but the merge-ready path should prove the patched production lifecycle and either cover or explicitly scope out dismissal teardown paths.
