## Summary

Keep open: current main still routes the built-in MiniMax path through one `MINIMAX_API_KEY` and Tachikoma's International endpoint, while custom Anthropic-compatible providers are only a workaround and do not satisfy the requested built-in MiniMax China support.

## What This Changes

Request support for MiniMax China alongside the current built-in MiniMax International credential and endpoint handling.

## Best Possible Solution

Add an explicit MiniMax China path that coexists with the current International default, such as a provider alias, separate credential, or documented base-URL override with regression tests.

## Reproduction Assessment

Yes, source reproduction is high-confidence: current code maps built-in MiniMax to one key and the International endpoint. I did not run a live China-key request, but the reported failure follows directly from the provider mapping.

## Solution Assessment

No, replacing the single hardcoded URL with the China URL is not the best fix because it would break International users. The maintainable direction is a coexistence surface selected by maintainers.
