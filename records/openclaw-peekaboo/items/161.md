## Summary

Keep open: current main still routes built-in `minimax` through a single `MINIMAX_API_KEY` into Tachikoma's International endpoint, while the existing custom-provider surface is only a workaround and does not satisfy the reported `peekaboo agent --model minimax` path.

## What This Changes

The issue requests MiniMax China support while preserving MiniMax International support.

## Best Possible Solution

Choose an additive contract, such as a separate MiniMax China provider/credential or explicit base-URL override, then update Tachikoma, bump the submodule, and add Peekaboo CLI/Mac/docs/tests while keeping the International default intact.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: built-in `minimax` only resolves `MINIMAX_API_KEY` and Tachikoma `.minimax` points at `api.minimax.io`, so a China-region key is sent to the wrong endpoint.

## Solution Assessment

Unclear until maintainers choose the provider contract. The reporter's endpoint swap proves the gap, but replacing the endpoint is not the best fix because it would break MiniMax International users.
