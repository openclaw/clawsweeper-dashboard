## Summary

Keep this PR open: current main still has the daemon poll loops that swallow cancellation, and this branch is a narrow, useful fix with sufficient terminal proof and no concrete correctness or security finding.

## What This Changes

The PR adds `Task.isCancelled` guards after three `DaemonLaunchPolicy` poll-loop sleeps and adds a standalone Swift proof script for the cancellation behavior.

## Best Possible Solution

Land the narrow daemon cancellation guards, preferably with durable CLI regression coverage if maintainers want this contract protected beyond the standalone proof script.

## Reproduction Assessment

Yes. Current main has the three daemon `try? await Task.sleep` poll loops without cancellation checks, and the PR body includes terminal proof that the swallowed-sleep pattern ignores cancellation until the deadline while the guarded pattern exits promptly.

## Solution Assessment

Yes. Checking `Task.isCancelled` immediately after the swallowed sleeps is the narrow maintainable fix for these nonthrowing poll loops; the remaining judgment is whether maintainers require integrated regression coverage before merge.
