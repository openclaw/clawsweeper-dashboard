## Best Possible Solution

Expose Cloudflare actions only when the installed crawler advertises or passes a version/capability check, while preserving the existing local database fallback for unknown kinds.

- **[P1] Gate cloud actions on actual CLI support:** `Sources/CrawlBarCore/BuiltInCrawlApps.swift:50-52`

Add explicit Cloudflare kind aliases, but leave unknown kinds on the existing local fallback or add a narrow compatibility path.
