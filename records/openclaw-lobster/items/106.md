## Summary

Keep open: the linked bug is valid and this PR is aimed at the right area, but the patch is still draft/dirty, lacks real behavior proof, and broadens `AbortError` retry behavior beyond the workflow timeout path.

## What This Changes

This PR changes `withRetry` so `AbortError`s only short-circuit when `options.signal` is aborted, and updates retry unit tests for external cancellation and synthetic timeout aborts.

## Best Possible Solution

Land a refreshed fix that preserves bare `AbortError` propagation by default, opts workflow step timeouts into retry through `shouldRetry` or a dedicated timeout error, adds workflow-level regression coverage, and then closes https://github.com/openclaw/lobster/issues/105 when merged.

## Reproduction Assessment

Yes, source-reproducible with high confidence: current main's `withRetry` throws abort-shaped errors before workflow `shouldRetry`, while step timeout execution feeds abort signals into retried attempts. I did not run the workflow under the read-only review contract.

## Solution Assessment

No, not as written: the fix targets the right bug but changes default `AbortError` handling for the exported retry helper. A narrower solution should let workflow timeouts opt into retry while preserving the generic cancellation boundary.
