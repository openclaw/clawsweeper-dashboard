## Summary

Keep open: current main still records zero cost for unknown non-null model IDs, the workflow cost limit checks that zero estimate, and no canonical open or merged repair was found.

## What This Changes

The issue asks Lobster to stop silently charging $0 for current or unknown LLM model IDs so workflow `cost_limit` remains meaningful.

## Best Possible Solution

Add non-breaking diagnostics for unknown non-null model IDs and invalid `LOBSTER_LLM_PRICING_JSON`, with focused regression tests; leave dynamic pricing or fail-closed policy for a separate maintainer decision.

## Reproduction Assessment

Yes, from source inspection: any non-null model absent from `this.pricing` records token counts but computes zero cost, and `checkLimit` compares against that zero estimate. The existing test suite also asserts unknown models cost `$0`.

## Solution Assessment

Yes for the first repair: warnings for unknown model IDs and invalid pricing JSON are the narrowest maintainable fix for the silent failure mode. A dynamic pricing URL or fail-closed behavior should be treated as separate policy work because it changes configuration and upgrade behavior.
