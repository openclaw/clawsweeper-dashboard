## Summary

Keep this open: current main supports provider/model LLM invocation and a generic OpenClaw tool bridge, but it still does not define a first-class configured-agent invocation path, and the remaining work needs maintainer API direction.

## What This Changes

The issue asks for a native Lobster workflow surface to call or spawn a specific OpenClaw agent, including agent/model selection rather than only invoking `llm-task` or a generic model adapter.

## Best Possible Solution

Define a stable OpenClaw gateway operation for invoking a configured agent, including agent identity, session reuse or creation, model override rules, and structured output, then expose it through a thin Lobster command with docs and contract tests.

## Reproduction Assessment

Not applicable: this is a feature/API request rather than a bug report. Source and docs inspection show provider/model selection exists, while configured-agent invocation is not a defined first-class Lobster capability.
