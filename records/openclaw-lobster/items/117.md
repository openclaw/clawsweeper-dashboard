## Summary

Keep this open: current main and the latest release expose LLM provider/model selection plus a generic OpenClaw tool bridge, but they still do not define a native way for Lobster to invoke a configured OpenClaw agent.

## What This Changes

Request for Lobster workflows to invoke a configured OpenClaw agent, not only a model or generic OpenClaw tool.

## Best Possible Solution

Define a stable OpenClaw gateway operation for invoking a configured agent, including agent identity, session creation or reuse, model override rules, and structured output, then expose it through a thin Lobster command with docs and contract tests.

## Reproduction Assessment

Not applicable: this is a feature/API request rather than a bug report. Source inspection shows model/provider selection exists, while configured-agent invocation is not a defined first-class Lobster capability.

## Solution Assessment

Unclear: a thin gateway-backed command is likely the maintainable shape, but maintainers need to choose the permanent agent, session, and model-override contract before implementation.
