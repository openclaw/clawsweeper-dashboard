## Summary

Keep this issue open: current main and v2026.6.11 support LLM provider/model selection and a generic OpenClaw tool bridge, but they still do not define a first-class way to invoke a configured OpenClaw agent.

## What This Changes

The issue asks for a native Lobster mechanism to call or select an OpenClaw agent rather than only invoking an LLM provider/model or generic OpenClaw tool.

## Best Possible Solution

Define a stable OpenClaw gateway operation for invoking a configured agent, including agent identity, session reuse/creation, model override rules, and structured output, then expose it through a thin Lobster command with docs and contract tests.

## Reproduction Assessment

Not applicable: this is a feature/API request rather than a bug report. Source and docs inspection show provider/model selection exists, while configured-agent invocation is not a defined first-class Lobster capability.

## Solution Assessment

No, not as a direct code change yet. A thin gateway-backed Lobster command is likely the maintainable shape, but maintainers need to choose the permanent agent, session, and model-override contract before implementation.
