## Summary

Keep this issue open: current main supports model-backed LLM calls and generic OpenClaw tool forwarding, but it still does not define a first-class way to invoke a configured OpenClaw agent. The remaining work needs maintainer API/product direction for agent identity, session behavior, and model override semantics.

## What This Changes

The issue asks for a native Lobster workflow surface to invoke a configured OpenClaw agent rather than only sending prompts or arbitrary tool calls.

## Best Possible Solution

Define a stable OpenClaw gateway operation for invoking configured agents, then expose it through a thin Lobster command or workflow syntax with docs and contract tests.
