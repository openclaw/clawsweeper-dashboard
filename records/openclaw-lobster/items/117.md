## Summary

Keep this issue open: current main supports selecting a model for LLM prompting and forwarding arbitrary OpenClaw tool calls, but it does not define a first-class contract for invoking a configured OpenClaw agent. The request is clear and vision-aligned, but agent identity, session behavior, and model-override semantics require a maintainer product/API decision.

## What This Changes

Requests a native Lobster workflow mechanism to invoke a specific configured OpenClaw agent, with explicit agent and optional model targeting.

## Best Possible Solution

Define a stable OpenClaw gateway operation for invoking a configured agent, including agent identity, session creation/reuse, optional model override, and structured output, then expose it through a thin Lobster command with documentation and contract tests.

## Reproduction Assessment

Not applicable as a bug reproduction: source inspection establishes that model selection exists, while configured-agent invocation is not a defined first-class Lobster capability.

## Solution Assessment

Unclear. A thin gateway-backed command is likely the maintainable direction, but maintainers must first decide the permanent agent, model-override, and session contract rather than inferring it from the generic tool bridge.
