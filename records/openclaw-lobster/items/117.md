Keep this open: current main supports LLM provider/model selection and a generic OpenClaw tool bridge, but it still does not define a first-class way to invoke a configured OpenClaw agent.

## What This Changes

The issue requests a native Lobster workflow surface for invoking a configured OpenClaw agent and selecting the agent/model target.

## Best Possible Solution

Define a stable OpenClaw gateway operation for configured-agent invocation, then expose it through a thin Lobster command with docs and contract tests.
