## Summary

Keep this issue open: current main supports choosing a model for prompt-only LLM calls and forwarding arbitrary OpenClaw tool calls, but it has no first-class configured-agent target. The remaining agent identity, session, model-override, and approval semantics require a maintainer product/API decision.

## What This Changes

Add a supported Lobster path for invoking a configured OpenClaw agent with explicit agent, session, and model targeting.

## Best Possible Solution

Define one OpenClaw gateway-backed agent invocation contract covering target identity, session lifecycle, model overrides, approval propagation, and result envelopes, then expose only a thin deterministic forwarding surface in Lobster.

## Reproduction Assessment

Not applicable: source inspection confirms that configured-agent targeting is absent, but the issue requests a new capability rather than reporting broken documented behavior.

## Solution Assessment

No, adding an ad hoc `--agent` option to one existing command is not yet the best solution; maintainers should first establish the shared OpenClaw agent and session contract to avoid parallel or conflicting APIs.
