## Summary

Keep this issue open: current main and the latest release support LLM provider/model selection plus a generic OpenClaw tool bridge, but they still do not define a first-class way to invoke a configured OpenClaw agent.

## What This Changes

The issue requests a native Lobster way to invoke a configured OpenClaw agent, including how agent/model identity should be specified from workflows.

## Best Possible Solution

Define a stable OpenClaw gateway operation for invoking a configured agent, including agent identity, session creation/reuse, model override rules, and structured output, then expose it through a thin Lobster command with docs and contract tests.

## Reproduction Assessment

Not applicable: this is a feature/API request rather than a bug report. Source inspection shows model/provider selection exists, while configured-agent invocation is not a defined first-class Lobster capability.

## Solution Assessment

Unclear: a thin gateway-backed command is likely the maintainable shape, but maintainers need to choose the permanent agent, session, and model-override contract before implementation.
