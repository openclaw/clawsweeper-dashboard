## Summary

Keep open: current main still reads LLM cache JSON without handling malformed content and writes cache entries with plain `fsp.writeFile`, so the corrupt-cache recovery bug remains despite the adjacent state-layer fix in https://github.com/openclaw/lobster/pull/110.

## What This Changes

Make `llm.invoke` and `llm_task.invoke` cache persistence atomic and resilient so a partial cache file becomes a cache miss instead of a permanent `SyntaxError`.

## Best Possible Solution

Use the existing atomic file replacement helper for LLM cache writes and make cache reads discard malformed JSON as a miss, with focused regression coverage for both corrupt existing files and normal cache reuse.

## Reproduction Assessment

Yes from source inspection: a malformed file at the computed LLM cache path is read before any provider call, and `JSON.parse` errors are rethrown. I did not run an end-to-end failing workflow in this read-only review, so the structured status is source-reproducible rather than executed reproduction.

## Solution Assessment

Yes, the proposed Option C is the narrowest durable fix: atomic writes prevent new partial files, and malformed-cache recovery lets existing corrupt files fall through to a fresh invocation. It should stay scoped to cache persistence and tests, without changing provider routing or cache key semantics.

## Repair Work Prompt

Fix the LLM cache corruption recovery bug for `llm.invoke` and `llm_task.invoke`. In `src/commands/stdlib/llm_invoke.ts`, route `writeCacheEntry` through the existing `writeFileAtomic` helper after creating the cache namespace directory, and update `readCacheEntry` so malformed JSON/SyntaxError in an existing cache file is treated as a cache miss, with best-effort corrupt-file removal. Add focused regression coverage in `test/llm_invoke.test.ts` or `test/llm_task_invoke.test.ts` for a pre-existing invalid cache file falling through to a fresh provider call and being replaced by a valid cache entry; keep existing successful cache reuse behavior. Do not change cache key computation, provider selection, default cache enablement, state semantics, or `CHANGELOG.md`.
