- **[P1] Split out the autoreview skill before merging:** `.agents/skills/autoreview/scripts/autoreview:1`
  - body: This Cloudflare remote archive PR also adds a new executable `.agents/skills/autoreview` toolchain with subprocess-backed reviewer engines and a shell harness.   - confidence: 0.91
- **[P2] Reuse keyring auth for cloud publish:** `internal/cli/cloud_commands.go:63-69`
  - body: `remote login` stores the bearer token in `cfg.Remote.Auth`, and `remoteClient` knows how to read that keyring entry, but `cloud publish` builds a fresh publisher config with only `TokenEnv`. A user who logged in through the new OAuth/token bootstrap can still get `ErrMissingToken` when publishing unless they also expose the token in the environment.
  - confidence: 0.9
- **[P2] Resolve keyring tokens in cloud doctor:** `internal/cli/app.go:3115-3127`
  - body: After the new login flow saves `token_source = keyring`, cloud-mode `doctor` still calls the env-only `ResolveRemoteToken`, so it reports `missing_token` and skips the remote status check even though `remote status` would authenticate through the keyring. Use the same keyring-aware resolver here.
  - confidence: 0.93
- **[P2] Clamp remote integers before converting to int:** `internal/cli/remote_commands.go:499-500`
  - body: CodeQL flagged this helper because `strconv.ParseInt` can return values outside the target `int` range before the unchecked conversion. Since these values come from a remote response, clamp or reject out-of-range values before returning an `int`.
