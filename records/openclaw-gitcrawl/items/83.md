## Summary

Keep open: current main and the latest release support targeted `sync --numbers --with pr-details`, but they do not provide a first-class bounded command that selects missing PR-detail rows, reports quota/progress, and resumes without external SQL orchestration.

## What This Changes

The issue requests a bounded PR-detail fill command that hydrates only missing pull request detail rows with limits, quota-aware progress, and resumable behavior.

## Best Possible Solution

Add a focused `fill-pr-details` flow that queries local missing PR-detail rows, hydrates them through the existing syncer, honors caller limits and rate-limit reserves, emits progress, and resumes by skipping rows already filled.

## Reproduction Assessment

Not applicable; this is a feature request rather than a reproducible bug. Source inspection gives high confidence that the requested first-class command is absent and users must currently select missing PR numbers outside Gitcrawl.

## Solution Assessment

Yes, the requested direction is a maintainable fit if it wraps the existing targeted PR-detail hydration path instead of introducing a separate hydrator or hydrating every PR blindly.
