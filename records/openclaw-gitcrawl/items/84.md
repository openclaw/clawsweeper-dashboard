## Summary

Keep open: current main still has only aggregate sync run history, while failed sync or PR-detail hydration attempts are not persisted as queryable per-thread attempts with retry and resolved state.

## What This Changes

Persist failed sync and PR-detail hydration attempts so coverage and JSON output can distinguish failed, retried, resolved, skipped, and unattempted rows.

## Best Possible Solution

Define a durable failure-attempt ledger, wire sync and PR-detail hydration to record failures and resolve them on later success, then expose the rows through coverage or status JSON.

## Reproduction Assessment

Yes for the missing current capability: source inspection and the existing PR-detail failure test show failed hydration currently leaves no durable queryable attempt row.

## Solution Assessment

Unclear: persisting attempts is the right direction, but maintainers should confirm the durable schema, JSON output shape, and resolved-state semantics before implementation.
