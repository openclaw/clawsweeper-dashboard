- `impact:other`: The issue affects maintainers' ability to distinguish known failed fetches from unattempted or intentionally skipped archive rows.

## Summary

Keep open: current main has aggregate run history, but failed sync or PR-detail hydration still returns without writing a queryable per-thread failure attempt, retry count, or resolved state.

## What This Changes

The issue requests a durable, queryable ledger for failed sync and hydration attempts, including retry and resolved-state metadata.

## Best Possible Solution

Add a small durable failure-attempt model for sync and hydration operations, then expose it through JSON coverage/status-oriented output so agents can distinguish failed, unattempted, skipped, and resolved rows.

## Reproduction Assessment

Not applicable; this is a request for a new durable failure-tracking capability rather than a broken existing command contract. Source inspection confirms current behavior lacks the requested ledger.

## Solution Assessment

Yes on direction, but the exact schema and resolution semantics need maintainer design. Existing aggregate run records are not sufficient for the requested per-thread retry and resolved-state decisions.
