## Summary

Keep open: current main has aggregate sync run history, but failed sync or PR-detail hydration attempts are not persisted as queryable per-thread attempts with retry/resolved state.

## What This Changes

Persist queryable failed sync or hydration attempts with retry and resolved state so coverage can distinguish failed, missing, skipped, and later-resolved archive rows.

## Best Possible Solution

Define a durable failure-attempt ledger, then wire sync and PR-detail hydration to record failures, increment retries, mark later success as resolved, and expose those rows through coverage or JSON status output.

## Reproduction Assessment

Yes. Source inspection and the existing PR-detail failure test show failed hydration currently leaves no durable queryable attempt row.

## Solution Assessment

Unclear. Persisting attempts is the right direction, but the durable schema, JSON output shape, and resolved-state semantics need maintainer confirmation before implementation.
