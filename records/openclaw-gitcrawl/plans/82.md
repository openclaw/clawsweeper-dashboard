## Summary

Keep open: current main and v0.6.4 still lack a first-class coverage command, and the existing status surface only reports aggregate archive counts rather than per-repository completeness or missing PR-detail rows.

## Plan

Implement a read-only `gitcrawl coverage` command for https://github.com/openclaw/gitcrawl/issues/82. Add CLI parsing in the existing command dispatcher, JSON output for agent use, a human-readable table mode, default all-repository reporting, `--repos owner/a,owner/b` filtering, and a filter for repositories with at least N missing PR-detail rows. Compute per-repository issue count, PR count, comment/review count, PR details present/missing, files/checks/runs/review-thread presence where available, and last sync time from existing store tables. Keep the command read-only, preserve existing `status` behavior, add focused store/CLI tests and docs command-reference coverage, and do not edit CHANGELOG.md; put release-note context in the PR body instead.
