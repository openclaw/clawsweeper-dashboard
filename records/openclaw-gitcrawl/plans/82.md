## Summary

Keep open: current main and latest release v0.6.4 still lack a first-class archive coverage command or equivalent per-repository missing-detail report, and the related issues are complementary rather than replacements.

## Plan

Implement a read-only `gitcrawl coverage` command for openclaw/gitcrawl. Add JSON and human-readable table output; default to all repositories in the local DB; support filtering with `--repos owner/a,owner/b`; support `--min-missing-pr-details`; report repository name, issue count, PR count, comment/review counts, PR details present/missing, files/checks/runs/review-thread presence or missing counts, and last sync state/time where available. Reuse existing SQLite/store tables; do not perform network syncs or hydration from this command. If durable failed/skipped hydration tracking from https://github.com/openclaw/gitcrawl/issues/84 is not implemented yet, expose that field as unavailable or omit it rather than inventing a ledger. Keep bounded fill orchestration from https://github.com/openclaw/gitcrawl/issues/83 out of scope. Add focused store and CLI tests, update command docs, and do not edit CHANGELOG.md.
