## Summary

Keep open: current main and v0.6.4 still lack a first-class archive coverage command or equivalent per-repository missing-detail report, while the related issues are adjacent follow-ups rather than replacements.

## Plan

Implement a read-only `gitcrawl coverage` command for local SQLite archives. Add CLI dispatch, help/usage, JSON output, and a human-readable table mode. Aggregate per repository: repository name, issue count, PR count, comment/review count where available, PR details present and missing, PR files/checks/runs/review-thread coverage as supported by existing tables, and last successful sync time. Support all repos by default, `--repos owner/a,owner/b`, and `--min-missing-pr-details`; approximate DB size may be omitted or clearly marked approximate. Keep mutation, hydration/fill orchestration, and durable failure-ledger schema out of scope except for leaving room to integrate https://github.com/openclaw/gitcrawl/issues/84 later. Add focused store and CLI tests plus docs; do not edit CHANGELOG.md.
