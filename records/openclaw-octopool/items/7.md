  - body: This routes every `/git/` request through `handleGitRequest` without passing `ctx` or `requestId`, and the new handler never records `insertAudit`.   - confidence: 0.91

  - body: After caller policy is loaded, the handler selects a GitHub App identity and forwards to GitHub without `ensurePublicGitHubRepo` or an explicit private-repo grant.   - confidence: 0.88
- **[P1] Gate pushes on write-scoped identities:** `src/git-proxy.ts:426`
  - body: The push path reuses any active GitHub App identity that matches owner/repo and filters only by `kind`. A caller Git policy should not be enough to send receive-pack through an identity configured for the read relay; require an explicit Git/write scope or separate write identity selection and test both denied and allowed push cases.
  - confidence: 0.86

  - body: `parseRepoName` lowercases owner but preserves repo casing, while lookup/delete compare `lower(repo)` and the table primary key is case-sensitive. Updating `OpenClaw/Octopool` and then `openclaw/octopool` can create two policy rows and leave stale grants ambiguous.
  - confidence: 0.8
