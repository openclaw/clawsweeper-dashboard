Full review comments:

- **[P1] Keep gh pr checks live by default:** `cmd/octopool/gh_top.go:520-524`
  - body: This helper now reads check-runs and statuses without the cache-bypass headers while `gh pr checks` still exits from those values. Since completed check/status responses are cached for up to 300 seconds for the same SHA, automation can see stale success or failure after a rerun; preserve the current live default or make cached checks an explicit opt-in path with coverage for both modes.
  - confidence: 0.88
- **[P2] Translate REST file statuses to gh changeType values:** `cmd/octopool/gh_top.go:706`
  - body: This maps REST `status` values directly into `files[].changeType`, yielding values like `added` or `removed` instead of the normal `gh pr view --json files` values such as `ADDED` or `DELETED`. Scripts comparing Octopool output with regular `gh` output will miss matches, especially deletions and renames, so translate these before returning the hydrated field.
  - confidence: 0.86

  - body: OpenClaw release notes are release-owned, and this contributor PR already carries the release-note context in the PR body. Leaving this edit in the branch creates release-note churn that should be handled by the release process instead.
