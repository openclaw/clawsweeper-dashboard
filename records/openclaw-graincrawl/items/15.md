## Repair Work Prompt

Implement only the short-term diagnostic for encrypted-only Granola desktop state. When plaintext `supabase.json` is missing but encrypted state such as `supabase.json.enc`, `cache-v6.json.enc`, or `storage.dek` is present, make `doctor`/source reporting and the relevant sync failure path clearly explain that Granola desktop state is encrypted-only, `encrypted-json` unlock/import is not implemented in this version, and re-signing into Granola will not help unless plaintext state is emitted. Add focused regression tests using temp profile/config paths. Do not implement Electron safeStorage, AES-GCM decryption, Keychain prompting, helper binaries, token persistence, or CHANGELOG edits; preserve no-secret logging behavior.
