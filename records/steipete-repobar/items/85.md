- **[P1] Remove the global ATS bypass:** `Scripts/package_app.sh:123-126`
  - body: This writes `NSAllowsArbitraryLoads=true` into the packaged app while the new GitLab PAT flow accepts `http://` hosts and sends the token in `PRIVATE-TOKEN`. That allows cleartext token traffic for the whole app; keep HTTPS as the default or add a narrow maintainer-approved opt-in instead, and mirror the fix in the resource plist.
  - confidence: 0.92
- **[P1] Keep GitLab PATs out of GitHub clients:** `Sources/RepoBar/App/AppState+Accounts.swift:231-232`
  - body: For a GitLab active account this branch falls back to `legacyGitHub`, but the GitHub token provider still returns the active account token and startup/diagnostics still call GitHub rate-limit paths. That can send the GitLab PAT to `api.github.com` as a bearer token; make GitHub token providers and diagnostics nil/skip unless the active account is GitHub.
  - confidence: 0.86
- **[P2] Use the active provider host for checkout:** `Sources/RepoBar/StatusBar/StatusBarMenuManager.swift:431-434`
  - body: `cloneURL` still builds remotes from `session.settings.githubHost`, and GitLab accounts intentionally do not mirror their host into that setting. With a GitLab repo selected, Checkout Repo will try to clone from GitHub instead of the GitLab instance.
  - confidence: 0.86
- **[P2] Use GitLab project paths for issue and MR menus:** `Sources/RepoBar/StatusBar/StatusBarMenuManager+Actions.swift:39-44`
  - body: The provider-aware work only special-cases CI; issue and pull-request actions still append GitHub paths like `/issues` and `/pulls`. For GitLab projects these should route to `/-/issues` and `/-/merge_requests`, otherwise visible menu actions open the wrong pages.
  - confidence: 0.78
