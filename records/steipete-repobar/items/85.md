- **[P1] Remove the global ATS bypass:** `Scripts/package_app.sh:123-126`
  - body: This writes `NSAllowsArbitraryLoads=true` into the packaged app while the new GitLab PAT flow accepts `http://` hosts and sends the PAT in `PRIVATE-TOKEN`.   - confidence: 0.93
- **[P1] Keep GitLab tokens out of GitHub legacy paths:** `Sources/RepoBar/App/AppState+Accounts.swift:164-166`
  - body: This guard only covers the fresh-login path, but switching to or falling back from a GitLab account still mirrors active-account credentials to legacy keys, and the GitHub token provider reads the active account token. A GitLab PAT can therefore be used by GitHub-only clients; gate all legacy mirroring and GitHub token-provider paths on `.github`.
  - confidence: 0.87
- **[P2] Use GitLab paths beyond CI:** `Sources/RepoBar/Support/RepoWebURLBuilder.swift:43-49`
  - body: The provider-aware URL work only special-cases CI; issue and merge-request actions still append GitHub paths like `/issues` and `/pulls`, and checkout still builds remotes from `settings.githubHost`. For GitLab projects these visible actions should use GitLab paths and the active provider host.
