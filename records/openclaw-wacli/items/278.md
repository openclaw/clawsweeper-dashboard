## Best Possible Solution

Land a narrow stale-stream fix that force-closes or otherwise truly replaces the connected stale stream before reconnecting, keeps heartbeat writes private, and includes focused regression tests plus redacted real setup proof.

## Reproduction Assessment

Yes, by source inspection: with `--stale-threshold` enabled while the WA client still reports connected, the PR's stale branch calls reconnect, and the real client returns nil without reconnecting when already connected. I did not run a live WhatsApp reproduction in this read-only review.

## Solution Assessment

No: opt-in stale detection is a plausible direction, but this implementation needs to force-close or otherwise force-replace the stale connected stream before reconnecting. It also needs real behavior proof before merge.
