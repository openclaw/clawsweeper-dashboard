- **[P2] Gate DetachSocket off for auth bootstrap:** `internal/app/sync.go:238`
  - body: `connectForSync` is also used by `wacli auth`, which passes `AllowQR: true` and can use phone pairing. Setting `DetachSocket: true` unconditionally means auth bootstrap dials no longer honor signal or root-timeout cancellation until the fixed 30-second timer fires; keep detachment to already-authenticated sync/reconnect paths.
  - confidence: 0.9
- **[P2] Respect caller cancellation during detached dials:** `internal/wa/client.go:158-159`
  - body: With `DetachSocket`, `ConnectContext` receives a background context canceled only by the 30-second timer. A shorter `--max-reconnect`, root timeout, or signal cancellation while inside one connect attempt is ignored until that timer fires; bridge caller cancellation only until the socket is established, then detach the established socket lifetime.
  - confidence: 0.88
- **[P2] Bound available presence sends before cleanup:** `internal/app/sync_events.go:103-107`
  - body: The added available-presence handlers use `context.WithoutCancel(ctx)`, so a slow `SendPresence` has no deadline. Since whatsmeow holds the event-handler read lock while invoking handlers and `RemoveEventHandler` needs the write lock, shutdown can block before the final bounded unavailable send is reached; use a bounded context for these sends too.
  - confidence: 0.82
