## What This Changes

The issue asks wacli to handle whatsmeow `ReceiptTypeReadSelf` receipts as a fallback for clearing local chat unread state when `regular_high` app-state read events fail.

## Best Possible Solution

Add a narrow `ReceiptTypeReadSelf` live-event fallback that reuses existing JID canonicalization and unread store helpers, with regression coverage for read-self and non-read receipts.

## Reproduction Assessment

No live reproduction was run. The issue is source-reproducible with high confidence because the current event switch has no receipt case while the only inbound read-state handler is app-state `MarkChatAsRead`.

Add regression coverage in `internal/app/sync_test.go` or a nearby app test using `newTestApp`/`fakeWA`: an unread chat is cleared by a read-self receipt, non-read-self receipts do not clear it, and empty/unresolvable receipt data is handled gracefully.
