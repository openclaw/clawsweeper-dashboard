## Repair Work Prompt

Fix the inconsistent JSON field casing for message command payloads. Current `messages list --json` and related message JSON outputs serialize `store.Message` directly, causing PascalCase keys such as `ChatJID`, `MsgID`, `Timestamp`, and `DisplayText`, while chats and other command outputs use snake_case JSON tags. Add an explicit snake_case message JSON contract, either by adding appropriate tags to `store.Message` or by introducing a command-layer DTO if that better preserves internal model semantics. Cover at least `messages list --json` with a regression test that asserts snake_case keys like `chat_jid`, `msg_id`, `timestamp`, `from_me`, and `display_text`, and asserts representative PascalCase keys are absent. Consider whether `messages search`, `messages starred`, `messages show`, `messages context`, and `messages export` should share the same payload helper. Do not change the SQLite schema, WhatsApp behavior, store locking, build tooling, or add dependencies. Keep stdout JSON wrapped by `internal/out.WriteJSON` and human output unchanged.
