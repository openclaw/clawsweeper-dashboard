## Summary

Current main already uses WhatsApp reply context to build human-readable display text, but it still does not persist or expose quoted-message references as machine-readable message fields, so the requested receive-side metadata remains open.

## What This Changes

Expose quoted-message metadata from incoming WhatsApp context info on stored and JSON-rendered messages so clients can detect replies to prior agent messages.

## Best Possible Solution

Add narrow persisted fields for the quoted stanza ID and quoted sender JID, wire them through parser, sync, store, and JSON output, and only expose full context_info after an explicit API decision.

## Reproduction Assessment

Not applicable as a bug reproduction, but source inspection gives a high-confidence current-main check: reply ContextInfo is parsed only into transient display fields, then dropped before storage and JSON output. I did not run a live WhatsApp fixture in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public shape; flattened quoted_msg_id plus quoted_sender_jid is the narrowest maintainable solution for the stated use case, while full context_info is a broader API choice.
