Full review comments:

- **[P1] Remove the production demo runtime seed:** `apps/web/src/ChatApp.svelte:217-219`
  - body: `seedDemoChannelRuntime()` runs for every mounted app session, so a channel with no real bridge-stamped runtime row still shows fabricated model, context, and cache values. That makes the picker/meter look authoritative when the backend has not reported those facts; gate this behind an explicit demo path or render the unknown/hidden state until real runtime data exists.
  - confidence: 0.9
- **[P2] Clear stale runtime data for empty channels:** `apps/web/src/lib/chat/channel-runtime-store.svelte.ts:120-124`
  - body: When `/runtime` returns an empty record, `loadChannelRuntime` returns without resetting the global store. After visiting a stamped channel, switching to an unstamped channel can keep showing the previous channel's model/context values and can send stale override data to the new channel; reset to an empty/unknown state when no real record is present.
  - confidence: 0.86
- **[P2] Route runtime PATCHes through the app API helper:** `apps/web/src/lib/chat/channel-runtime-store.svelte.ts:144-148`
  - body: `persistRuntimeOverride` sends the model/thinking override with raw `fetch`, so a normal browser session with a session cookie will hit the server's unsafe-request CSRF middleware without `X-ClickClack-CSRF: 1` and be rejected. Use the existing API helper or add the same CSRF header behavior here so the composer picker can actually persist overrides.
  - confidence: 0.88
