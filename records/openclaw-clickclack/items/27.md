## Solution Assessment

- **[P1] Filter activity rows before browser notifications:** `apps/web/src/ChatApp.svelte:2138-2139`

  - confidence: 0.91

The new API and schema persist only `override_model` and `override_thinking`, so those controls reset on reload/channel refresh and are not applied to the next turn. Either hide them until supported or extend the runtime override contract and tests to persist/apply them.
