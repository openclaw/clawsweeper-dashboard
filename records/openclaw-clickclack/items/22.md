## Summary

Keep open: current main still allows the same members to get separate DM conversations, and no merged fix or canonical open issue/PR owns the remaining work.

## What This Changes

The issue requests stable reuse or lookup of a normal 1:1 DM so repeated human/bot replies do not split across fresh direct conversations.

## Best Possible Solution

Define a canonical 1:1 DM reuse contract, then implement it across the store/API and adapter-facing path with regression coverage and a plan for existing duplicate rows.

## Reproduction Assessment

Yes, from source inspection and reporter logs: repeat POST /api/dms for the same workspace/member_ids reaches CreateDirectConversation, which allocates and inserts a fresh direct_conversations row without an exact-member lookup. I did not run a live test because this review is read-only.

## Solution Assessment

Unclear until maintainers choose the contract. The narrowest durable fix is likely a ClickClack lookup/create-by-members or canonical 1:1 reuse invariant, but that should be decided against existing group-DM semantics and the OpenClaw adapter route behavior.
