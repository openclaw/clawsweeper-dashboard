## Summary

Keep open: current main still routes landing-page App links on every non-localhost, non-app host to the hosted app, which conflicts with the documented single-host `/app` path for self-hosted deployments.

## What This Changes

The issue requests that self-hosted landing-page App/Open app links use the instance-local `/app` route instead of `https://app.clickclack.chat` on non-localhost hostnames.

## Best Possible Solution

Update the landing-page app URL logic to use the hosted app only for known product-site hosts and `/app` everywhere else, with focused e2e coverage for hosted and self-host hostnames.

## Reproduction Assessment

Yes. The current hostname rule can be reproduced directly from source: non-localhost self-host names evaluate to `https://app.clickclack.chat` instead of `/app`.

## Solution Assessment

Yes. The narrow maintainable fix is to make hosted product hosts explicit and keep self-hosted or reverse-proxied instances on the local `/app` route, with regression coverage for both cases.
