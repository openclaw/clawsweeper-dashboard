## Summary

Source inspection confirms this remains necessary: the in-drawer arrow toggles the desktop sidebar-collapse state without clearing the mobile drawer state, so the backdrop and open-drawer styling can persist together.

## What This Changes

Fix the iOS Safari mobile drawer becoming stuck in a partially shifted overlay state after the in-drawer collapse/back control is used.

## Best Possible Solution

Route the arrow through a viewport-aware handler that closes the mobile drawer without changing the desktop collapse preference on narrow screens, retains normal collapse behavior on desktop, and adds WebKit regression coverage for close and reopen geometry.

## Reproduction Assessment

Yes at source level with high confidence: open the mobile drawer and press its `Collapse sidebar` arrow; the handler changes `sidebarCollapsed` but leaves `mobileNavOpen` and its backdrop active. The reporter’s screenshots match that conflicting state, though this read-only review did not run a real iPhone Safari session.

## Solution Assessment

Yes: separating mobile close behavior from the desktop collapse state is the narrowest maintainable fix, with regression coverage for the exact control sequence.
