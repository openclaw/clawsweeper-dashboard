## Summary

Keep open. Current main still lets the mobile drawer state and desktop sidebar-collapse state coexist, and the concrete repair is the open, cleanly mergeable closing PR at https://github.com/openclaw/clickclack/pull/21, so this issue should stay open until that PR lands or is replaced.

## What This Changes

The issue reports that iOS Safari can leave the mobile navigation drawer in a partial overlay state after using the collapse-arrow control inside the open drawer and then returning via the hamburger button.

## Best Possible Solution

The desired end state is a narrow state-model fix that keeps desktop sidebar collapse as a desktop preference while making the mobile drawer close or reset coherently, with regression coverage for both mobile and desktop behavior.

## Reproduction Assessment

Yes, at source level: current main lets `mobileNavOpen` and `sidebarCollapsed` coexist through separate handlers/classes, and the reporter supplied a concrete iPhone Safari sequence with screenshots. I did not rerun the sequence on a physical iPhone.

## Solution Assessment

Yes. The linked repair path is narrow: close the mobile drawer at the mobile breakpoint instead of toggling desktop collapse, preserve desktop collapse behavior, and cover the regression in Playwright.
