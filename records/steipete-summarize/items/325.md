## Summary

Keep this PR open: it fixes the linked Anthropic PDF sub-path gateway bug, but the submitted URL join can break existing endpoint-style `ANTHROPIC_BASE_URL` overrides that already end in `/v1`, so it needs a compatibility-safe adjustment and inspectable real behavior proof before merge.

## What This Changes

The PR changes Anthropic PDF document requests to append `/v1/messages` to the configured base URL and adds URL regression tests for prefixed, trailing-slash, and default bases.

## Best Possible Solution

Use a URL builder that preserves arbitrary gateway path prefixes while treating an existing trailing `/v1` as the Anthropic API root, then cover both shapes in tests and merge after proof is attached.
