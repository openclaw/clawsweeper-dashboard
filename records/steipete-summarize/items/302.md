## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Cancel oversized transcript streams:** `packages/core/src/content/transcript/providers/podcast/rss-transcript.ts:159`
  - body: When the byte cap is exceeded, this throws out of the read loop and the `finally` only calls `reader.releaseLock()`. Releasing the lock does not cancel the response body, so an attacker-controlled chunked transcript can keep the underlying fetch/connection alive after the function has already returned failure; the existing capped media downloads cancel readers in `finally`. Please cancel the reader/body on the over-limit path and add a cancellation assertion.
  - confidence: 0.87
