## Summary

Keep open: current main and v0.20.0 still have no way for a synthetic custom Anthropic gateway model to opt into pi-ai's adaptive thinking path, so the reported Bedrock Opus empty-summary failure is still plausible and actionable.

## What This Changes

The issue reports that custom Anthropic-on-Bedrock gateways for Opus 4.7/4.8 fail with thinking enabled because summarize sends budget-based thinking instead of adaptive thinking plus output_config.effort.

## Best Possible Solution

Add a narrow adaptive-thinking compatibility path for custom Anthropic gateway models while preserving existing budget-based models, then cover synthetic Opus 4.7/4.8 and any captured signature-only streaming fixture with focused tests.

## Reproduction Assessment

No live high-confidence reproduction was possible without the reporter's Bedrock-backed gateway. Source inspection does confirm the custom synthetic-model path that would send budget-based thinking once a reasoning effort is present.

## Solution Assessment

Unclear pending maintainer direction. The narrowest maintainable fix is likely either explicit adaptive compat for custom Anthropic models or a targeted retry on the Bedrock validation error, without changing existing budget-based Claude behavior.
