## Summary

Keep this PR open for maintainer review: the patch is focused, proven through real CLI output, and matches the current Node >=24 runtime, but it intentionally turns malformed `file:` URLs that current main coerces into parser errors.

## What This Changes

The branch replaces manual `file:` URL pathname decoding in CLI input resolution with Node `fileURLToPath` and adds regression tests for POSIX host/encoded-separator rejection plus Windows drive/UNC conversion.

## Best Possible Solution

Land the Node-based conversion with the added regression coverage if maintainers accept stricter malformed `file:` URL handling as the supported CLI contract.

## Reproduction Assessment

Yes. Source inspection and a focused Node check show current main manually decodes and resolves `file:` URL pathnames, coercing non-local hosts and encoded separators that `fileURLToPath` rejects.

## Solution Assessment

Yes, subject to a maintainer compatibility call. Delegating file URL conversion to Node is the narrow maintainable fix, but the stricter malformed-URL behavior should be accepted explicitly before merge.
