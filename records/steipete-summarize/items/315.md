## Summary

Keep this open: current main still emits the UTF-8 XML declaration in the Windows Scheduled Task XML, the scheduled-task test asserts that declaration, and canonical searches found no open fix PR for the localized `schtasks` failure.

## What This Changes

The issue reports that Windows `summarize daemon install` fails on non-UTF-8 ANSI code pages because the generated Scheduled Task XML declares UTF-8 before `schtasks /Create /XML` imports it.

## Best Possible Solution

Patch the Scheduled Task XML builder to omit only the unnecessary encoding attribute, update the focused scheduled-task test, and keep the existing task settings and install lifecycle unchanged.

## Reproduction Assessment

Yes at source level: current main writes `daemon-task.xml` with `encoding="UTF-8"` and imports it via `schtasks /Create /XML`. A live Chinese Windows runtime reproduction was not run in this Linux checkout.

## Solution Assessment

Yes: removing only the unnecessary encoding attribute is the narrowest maintainable fix because the XML content and existing task settings can remain unchanged.
