## Summary

Keep open: current main and v0.20.0 still block Direct provider hover summaries when the daemon token is empty, and this PR appears to fix that path without weakening the background token or URL boundaries.

## What This Changes

The PR removes the Chrome hover content-script daemon-token preflight and adds unit plus installed-extension coverage for Direct provider hover summaries with an empty daemon token.

## Best Possible Solution

Merge the focused fix after maintainer review, keeping content-script hover enablement separate from background runtime, token, and URL enforcement.

## Reproduction Assessment

Yes from source inspection: current main exits before sending `hover:summarize` when hover summaries are enabled but `settings.token` is empty. The PR body also provides installed Chrome extension Playwright proof for the fixed Direct provider path.

## Solution Assessment

Yes: moving token gating to the mode-aware background controller is the narrow maintainable fix. The PR preserves daemon-token rejection and public-URL enforcement there.
