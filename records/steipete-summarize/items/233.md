## Summary

Keep open: the PR targets a real Anthropic `--thinking` forwarding gap, but it should not merge as-is because it leaves the streaming Anthropic path unpatched and can force thinking onto known models whose SDK metadata marks reasoning unsupported.

## What This Changes

The PR forwards `requestOptions.reasoningEffort` into Anthropic text completions and maps it to pi-ai `reasoning`, including a `reasoning: true` override for models whose metadata is false.

## Best Possible Solution

Share a safe Anthropic thinking mapping across non-streaming and streaming text calls, preserve known model compatibility metadata, and cover supported, synthetic, and unsupported model cases with focused tests.

## Reproduction Assessment

Yes from source inspection: current main passes request options into OpenAI paths but not into Anthropic text completions, and the streaming Anthropic branch also omits reasoning despite receiving requestOptions.

## Solution Assessment

No; forwarding the effort is the right direction, but the current patch misses streaming calls and bypasses the SDK's unsupported-model metadata.
