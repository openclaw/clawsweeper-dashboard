## Summary

Keep this PR open: the amended branch fixes the earlier streaming gap, but it still forwards `reasoning` for registered Anthropic models whose locked pi-ai metadata says `reasoning: false`; pi-ai 0.75.5 enables Anthropic thinking from `options.reasoning`, so existing Claude 3/3.5 setups with a global thinking option could start failing after merge.

## What This Changes

The PR threads `--thinking`/`reasoningEffort` into Anthropic text generation and streaming, adds a shared Anthropic reasoning helper, and adds focused helper tests.

## Best Possible Solution

Land the Anthropic thinking support after the helper only forwards pi-ai `reasoning` for registered supported models and synthetic override cases, with regression coverage proving registered unsupported models omit the option.

## Reproduction Assessment

Yes. Current main source clearly drops `requestOptions.reasoningEffort` on both Anthropic text paths, and the PR body/comment provide before/after outbound request logs for the intended fix.

## Solution Assessment

No, not yet. Threading the option through both paths is the right shape, but the helper must omit `reasoning` for registered `reasoning:false` Anthropic models instead of relying on pi-ai to inspect that metadata.
