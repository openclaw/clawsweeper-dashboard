- `P2`: The PR addresses a bounded wrong-video transcript race, but its URL identity guard has a lower-blast-radius correctness gap.

## Summary

This is a focused fix for a real YouTube SPA navigation race, but the proposed identity comparison can still accept an unrelated page carrying the same `v` parameter, so the stale-result guard is incomplete.

## What This Changes

The PR passes the discovered YouTube page URL into injected caption and transcript-panel work, rejects results after navigation changes the page identity, and adds navigation-race regression tests.

## Best Possible Solution

Derive identity only from recognized YouTube hosts and include the normalized host with the video ID, falling back to the complete URL for non-video pages, then cover navigation to an off-origin URL with the same `v` value.

## Reproduction Assessment

Yes at source level: navigate from `https://www.youtube.com/watch?v=test` to an unrelated URL such as `https://example.com/?v=test`; the new helper computes `youtube:test` for both and permits stale caption or panel work.

## Solution Assessment

No, not yet: passing the expected page identity is the right narrow approach, but identity must validate the YouTube host rather than treating every URL with the same `v` parameter as the same video page.
