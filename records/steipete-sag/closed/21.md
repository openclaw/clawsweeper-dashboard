## Summary

This should stay open: current main still lets output-extension inference overwrite an explicit `--format`, and the linked fix pull request is open rather than merged.

## What This Changes

The issue reports that `sag speak --output out.mp3 --format mp3_44100_192` requests the extension-inferred 128 kbps MP3 format instead of honoring the explicit format flag.

## Best Possible Solution

Keep this issue as the canonical bug until https://github.com/steipete/sag/pull/22 lands or maintainers choose an equivalent fix with the same precedence guarantee and regression coverage.

## Reproduction Assessment

Yes, by source inspection. Current main overwrites `opts.outputFmt` from a recognized output extension before request construction, matching the reported `--output out.mp3 --format mp3_44100_192` scenario; I did not run the live ElevenLabs command in this read-only review.

## Solution Assessment

Yes. Guarding extension inference with `cmd.Flags().Changed("format")`, as the linked pull request proposes, is the narrow maintainable fix that matches the docs; the issue should remain open until that fix merges.
