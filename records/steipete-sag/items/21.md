## Summary

This should stay open: current main still lets output-extension inference override an explicit ElevenLabs `--format`, while the linked fix is an open pull request rather than merged code.

## What This Changes

The issue asks for `sag speak --format` to take precedence over format inference from `--output` file extensions.

## Best Possible Solution

Review and merge https://github.com/steipete/sag/pull/22 or an equivalent narrow fix that preserves explicit `--format` and keeps extension inference only as the default.

## Reproduction Assessment

Yes. Source inspection shows a high-confidence path: pass `--output out.mp3 --format mp3_44100_192`, and current main overwrites `opts.outputFmt` with the `.mp3` default before building the ElevenLabs request.

## Solution Assessment

Yes. Guarding extension inference with `!cmd.Flags().Changed("format")` is the narrowest maintainable fix and matches the existing docs contract.
