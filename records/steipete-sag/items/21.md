## Summary

This should stay open: current main still lets output-extension inference overwrite an explicit `--format`, while the linked pull request is open and should close this issue only if it lands.

## What This Changes

The issue reports that `sag speak --format ... --output out.mp3` sends the extension-inferred default format instead of the explicit CLI format.

## Best Possible Solution

Keep this issue as the canonical bug until https://github.com/steipete/sag/pull/22 lands or maintainers choose a replacement fix with the same precedence guarantee and regression coverage.

## Reproduction Assessment

Yes by source inspection: current main overwrites `opts.outputFmt` from a recognized output extension before request construction, matching the issue's `--output out.mp3 --format mp3_44100_192` scenario. I did not run the live ElevenLabs command in this read-only review.

## Solution Assessment

Yes. Guarding extension inference with `cmd.Flags().Changed("format")`, as the linked PR proposes, is the narrow maintainable fix that matches the docs; the issue should remain open until that fix merges.
