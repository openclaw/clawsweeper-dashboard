## What This Changes

The PR changes session lifecycle handling, status output, docs, changelog, and focused tests so cached session PIDs are cleared or hidden unless a live queue-owner lease proves the helper is running.

## Best Possible Solution

Land the queue-owner-liveness contract after maintainers explicitly accept the status-output compatibility change and required checks are green.

Yes, if maintainers accept the compatibility change: using queue-owner health as the status PID authority and clearing stopped lifecycle PIDs is the narrow maintainable fix. The branch also synchronized the CLI/session docs and focused regression coverage.
