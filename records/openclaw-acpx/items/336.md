## Summary

This PR is useful but not merge-ready because it changes the persisted built-in Codex command without an upgrade path for existing sessions; the package/version claims and real adapter smoke proof are otherwise well supported.

## What This Changes

The branch switches the built-in Codex adapter from `@zed-industries/codex-acp` to `@agentclientprotocol/codex-acp`, bumps the Claude adapter range, narrows the Codex `thought_level` alias, and updates matching tests and docs.

## Best Possible Solution

Land the adapter switch only with an explicit upgrade story: preserve or intentionally migrate/retire old Zed-backed Codex session records, document any breaking behavior, and test fresh-install plus upgrade paths.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes from source: persisted sessions are filtered by exact `agentCommand`, and this PR changes the default `codex` command string without adding a legacy lookup or migration path.

## Solution Assessment

Is this the best way to solve the issue? Not yet; the adapter switch is plausible, but the maintainable path needs explicit upgrade handling for old Codex sessions and documented compatibility behavior before merge.
