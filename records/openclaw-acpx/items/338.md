## What This Changes

The branch clears stopped helper PIDs from session records, makes status PID output depend on queue-owner liveness, and updates focused tests and session docs.

## Best Possible Solution

Land the queue-owner-liveness contract with the CLI reference synchronized, after maintainers explicitly accept that cached session PIDs are no longer a status liveness signal.

## Reproduction Assessment

Yes from source inspection: current main's statusPid returns health.pid ?? record.pid, so a saved session with no queue owner can still surface a cached record.pid. I did not run a runtime reproduction because this review is read-only and avoids generated artifacts.

## Solution Assessment

Yes, using queue-owner health as the live PID authority and clearing stopped helper PIDs is the narrow maintainable fix. The remaining issue is merge policy for the status-output compatibility change, plus syncing the full CLI reference.
