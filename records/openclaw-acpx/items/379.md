## What This Changes

The PR changes public file session store loading to treat corrupt or ill-shaped session JSON as an undefined record and adds a regression test for that recovery path.

## Best Possible Solution

Land this focused fix after normal code-change validation passes, keeping the public file store aligned with existing internal session-reader recovery without changing the public type contract.

## Reproduction Assessment

Yes from source inspection: current main reads and JSON.parses inside FileSessionStore.load(), then rethrows any non-ENOENT error, so a truncated session file deterministically surfaces SyntaxError. I did not run the test suite because this review is constrained to read-only repository inspection.

## Solution Assessment

Yes. Splitting read errors from parse/shape recovery is the narrow maintainable fix here because it preserves real filesystem error propagation while matching the existing internal reader behavior for unusable session content.
