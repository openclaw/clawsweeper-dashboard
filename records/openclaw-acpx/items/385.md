## Summary

Keep open: current main still keeps non-Claude model IDs strict and forwards Cursor `set model` values unchanged, so Cursor shorthand model support or clearer native-ID documentation remains an unresolved product decision.

## What This Changes

The issue asks ACPX to decide or clarify how Cursor should handle shared `--model` and `set model` shorthand such as `composer-2.5` when Cursor advertises bracketed native model IDs.

## Best Possible Solution

Maintainers should choose the Cursor model contract first, then either normalize unambiguous shorthand to advertised Cursor IDs in both `--model` and `set model` paths or document native advertised IDs with clearer errors.

## Reproduction Assessment

No high-confidence live reproduction was run in this review. Source inspection gives a medium-confidence path because non-Claude `--model` uses exact advertised IDs and Cursor `set model` values are forwarded unchanged, matching the reporter's live output.

## Solution Assessment

Unclear; shorthand alias normalization would improve portability, but native-ID-only docs and errors would preserve the current strict adapter contract. The best path depends on a maintainer decision about shared model-label semantics.
