## Summary

Keep open: current main still requires exact advertised model IDs for non-Claude agents and does not document a Cursor shorthand-alias contract, so the reported Cursor model behavior remains an unresolved product decision.

## What This Changes

The issue requests that Cursor model selection either accept provider-agnostic/shared model labels like other built-in agents or clearly document that Cursor requires native advertised model IDs.

## Best Possible Solution

Maintainers should choose one contract: either add a narrow, tested Cursor model-alias normalization path, or document and diagnose that Cursor requires exact native advertised model IDs such as the values returned by status.availableModels.

## Reproduction Assessment

Not a high-confidence live reproduction in this review. The issue provides concrete Cursor commands, and source inspection confirms ACPX exact-match validation plus raw set-model forwarding, but I did not run the reported Cursor CLI version.

## Solution Assessment

Unclear until maintainers pick the product contract. Alias normalization would improve portability, while explicit native-ID documentation would preserve the current ACP-advertised-values model with less new surface.
