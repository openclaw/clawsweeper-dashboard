## Summary

Keep open: current main still preserves the strict non-Claude model-ID contract and forwards Cursor `set model` values unchanged, so the shared-label Cursor workflow remains unresolved and needs maintainer product direction.

## What This Changes

The issue asks ACPX to handle or document shared/global model labels for Cursor so values such as `composer-2.5` do not fail when other agents accept provider-agnostic model labels.

## Best Possible Solution

Maintainers should choose the Cursor model contract first, then either add a narrow advertised-ID alias resolver with docs/tests or explicitly document native Cursor model IDs as required.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No live high-confidence repro was run here; source inspection gives a medium-confidence path because non-Claude model selection requires exact advertised IDs and Cursor `set model` values are forwarded unchanged.

## Solution Assessment

Is this the best way to solve the issue? Unclear; alias normalization would improve portability, but native-ID-only docs/errors would preserve the current strict adapter contract, so maintainer direction is needed before implementation.
