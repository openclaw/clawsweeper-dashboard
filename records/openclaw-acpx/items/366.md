## Summary

Keep open. Current main still lacks a caller-provided response schema path for `exec` or `prompt`: the CLI exposes transport output modes and prompt input options, while the ACP prompt call forwards only the session id and normalized prompt. The request fits acpx’s backend-friendly structured-output direction, but it adds durable CLI/API conventions and adapter fallback semantics that need maintainer product design first.

## What This Changes

Request an ACPX-level way to pass a JSON Schema that constrains the agent’s final response separately from `--format` transport output.

## Best Possible Solution

Define one protocol-aligned response-contract surface that keeps `--format` for transport output, negotiates native schema enforcement where supported, and returns typed unsupported-capability or validation errors under an explicit fallback policy.

## Reproduction Assessment

Not applicable; this is a new response-contract capability request rather than a bug in documented behavior. Source inspection gives high-confidence confirmation that current main lacks the requested CLI/API path.

## Solution Assessment

Unclear until maintainers settle the public contract. Separating response constraints from `--format` is the right direction, but flag naming, schema input form, adapter capability handling, and fallback behavior need design first.
