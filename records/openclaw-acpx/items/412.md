## What This Changes

The PR adds a top-level `npmjs` Dependabot registry for `https://registry.npmjs.org` with `replaces-base: true` and makes the npm update block use it.

## Best Possible Solution

Keep Dependabot on the default public npm registry unless maintainers intentionally add a supported authenticated registry configuration and verify Dependabot accepts it.

## Reproduction Assessment

## Solution Assessment

No: the current patch uses a private-registry-style Dependabot entry without the required auth shape. The maintainable path is either no override for the public npm registry or a supported authenticated Dependabot registry setup.
