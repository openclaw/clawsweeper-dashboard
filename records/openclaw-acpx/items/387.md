## Summary

Keep open: current main and v0.11.0 still only support `mcpServers` through fixed global/project config and do not provide the requested per-invocation MCP source or config-location override.

## What This Changes

Request a session-scoped way to provide MCP server definitions without writing `<cwd>/.acpxrc.json` or changing global config.

## Best Possible Solution

Add a maintainer-chosen, redaction-aware per-invocation MCP/config source with explicit precedence, session-scope semantics, docs, and regression tests for fresh and reused sessions.

## Reproduction Assessment

Not applicable as a feature/config-surface request. Source inspection confirms the reported gap: current main reads fixed global/project config files and has no per-invocation MCP or config-path flag.

## Solution Assessment

Unclear until maintainers choose the permanent API. A file-path based MCP/config override looks safer than raw inline JSON because `mcpServers` can contain credentials and should not be echoed by default.
