## Summary

Keep open: current main and the latest release still only load `mcpServers` from the fixed global/project config files, and the requested per-invocation MCP or config-location entry point remains a product-surface decision.

## What This Changes

The issue asks for a per-invocation way to supply MCP servers or override config location without writing `.acpxrc.json` into the current working directory.

## Best Possible Solution

Choose one stable per-invocation config or MCP-server input surface, add precedence and forwarding coverage, and document the supported `mcpServers` shape without exposing secret-bearing values through routine `config show` output.

## Reproduction Assessment

Yes. Source inspection on current main shows MCP servers can be forwarded from fixed config files, but there is no CLI/env/config-path entry point for session-scoped MCP input.

## Solution Assessment

Unclear. A config-path override or MCP file flag could be narrow, but maintainers need to choose the API and secret-display behavior because this repo treats config and output conventions as durable product surface.
