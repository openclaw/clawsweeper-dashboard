## Summary

Keep open: current main still starts a fresh flow run every time and has no flow-level resume or per-node memoization surface. The request fits acpx's robustness goals, but it adds new CLI/API and persistence semantics that need maintainer design before implementation.

## What This Changes

The issue requests opt-in per-node memoized resume for ACP flows so reruns can skip completed deterministic nodes whose resolved inputs have not changed.

## Best Possible Solution

Design a default-off flow resume mode with explicit per-node memoization metadata, stable cache-key semantics, and tests proving both fresh-run compatibility and resumed deterministic-node skipping.

## Reproduction Assessment

Not applicable; this is a feature request, not a bug report. Source inspection does confirm the requested flow resume and per-node memoization surfaces are absent on current main.

## Solution Assessment
