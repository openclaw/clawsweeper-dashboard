## Summary

Keep open: current main still creates a fresh flow run every time and has no flow-level resume flag or per-node memoization contract. The request is useful and fits flow robustness, but it adds public CLI/API and persistence semantics that need maintainer design before implementation.

## What This Changes

Request opt-in per-node flow memoization so re-running a flow can skip completed deterministic nodes keyed by node id and resolved input.

## Best Possible Solution

Design a default-off flow resume mode with explicit per-node memoization metadata, stable cache-key semantics, clear run-bundle behavior, and tests covering both fresh runs and resumed deterministic-node skipping.

Not applicable; this is a feature request, not a bug report. Source inspection does confirm the requested flow resume and per-node memoization surfaces are absent on current main.

Unclear; the proposal is directionally plausible, but the exact CLI flag, node field, cache key, invalidation, and persistence semantics need maintainer product/API review first.
