The PR tags regular-file identity mismatches as `FsSafeError("path-mismatch")`, retries async JSON readers on that transient code, and adds regression coverage plus a changelog entry.

## Best Possible Solution

Land the narrow typed-sentinel and bounded async JSON retry in `fs-safe`, then release it so downstream OpenClaw can bump the package version.

## Solution Assessment

Yes. The typed `path-mismatch` sentinel plus bounded retry in only async JSON readers is the narrowest maintainable fix while preserving parse errors and non-race read failures.
