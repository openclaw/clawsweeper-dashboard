## What This Changes

This PR changes regular-file identity mismatches to throw `FsSafeError("path-mismatch")`, adds bounded retry for async standalone JSON readers, updates tests, and adds a changelog entry.

## Best Possible Solution

Land the dependency-level typed sentinel and bounded async JSON retry in `fs-safe`, then release it so downstream OpenClaw can consume the fixed package.

## Reproduction Assessment

Yes. Current main source shows a direct, non-retried async JSON read path and a raw `File changed during read` throw; the PR body adds real-disk race output showing upstream failures and patched success.

## Solution Assessment

Yes. The typed `path-mismatch` sentinel plus bounded retry in only async JSON readers is the narrowest maintainable fix while preserving parse errors and non-race read failures.
