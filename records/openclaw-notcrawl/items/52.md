## Best Possible Solution

## Solution Assessment

Is this the best way to solve the issue? Likely yes: reusing CrawlKit's PushSnapshot and SyncForWrite keeps the Git retry and tag-retargeting logic in the shared mirror layer instead of duplicating it in Notcrawl, but live upgrade proof is still needed before merge.
