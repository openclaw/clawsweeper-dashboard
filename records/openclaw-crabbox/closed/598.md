## Summary

Keep open: current main still routes E2B and Upstash Box credential-bearing requests through HTTP clients that do not enforce the documented same-origin redirect boundary, and no open PR appears to own this exact fix yet.

## What This Changes

The issue reports that E2B and Upstash Box provider clients can replay custom provider credential headers across cross-origin redirects.

## Best Possible Solution

Land a focused security fix that wraps E2B API/envd and Upstash Box HTTP clients with same-origin redirect guards, preserves caller redirect hooks and default redirect-count behavior, and adds regression tests for credential header non-delivery to cross-origin redirect targets.

## Reproduction Assessment

Yes at source level: current main sets custom credential headers and sends the requests through clients without a same-origin redirect guard. A high-confidence regression path is an httptest server that redirects to a different origin and asserts the credential headers never reach the target.

## Solution Assessment

Yes: the narrowest maintainable fix is the same provider-local CheckRedirect wrapper pattern already used by RunPod, Cloudflare, OpenComputer, and other provider clients. The fix should add tests rather than changing user-facing provider behavior or adding a new config knob.
