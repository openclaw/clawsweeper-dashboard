## What This Changes

The branch requires manage/admin access for changed run-event lease attribution, defers explicit `crabbox run --id ...` run-history creation until lease resolution, and adds Worker plus CLI regression coverage.

## Best Possible Solution

Land this focused security fix if maintainers accept fail-closed post-create attribution; otherwise add a dedicated server-validated attachment path that blocks forged audit re-tagging.

## Reproduction Assessment

Yes. Source inspection of current main shows an owned run can post a different visible use-shared lease ID, which event summary and attribution code can persist into owner-visible run audit surfaces; I did not run the Worker harness in this read-only review.

## Solution Assessment

Mostly yes. The patch takes a narrow security path by requiring manage/admin access for changed event lease IDs and preserving explicit-lease runs by creating history after lease resolution, but maintainers still need to accept the compatibility contract.
