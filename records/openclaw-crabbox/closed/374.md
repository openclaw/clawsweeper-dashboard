## Summary

Keep open. Current main and the latest release still make Proxmox template image verification optional, while maintainer discussion has accepted the report as compatibility-preserving supply-chain hardening rather than closing it as already implemented.

## What This Changes

The issue requests mandatory Proxmox template cloud-image identity verification for the documented default build path and custom image URLs.

## Best Possible Solution

Pin the documented default Proxmox cloud image to a dated release with a verified SHA256, require a matching digest for any custom image URL, and cover missing, wrong, and correct checksum paths in the helper tests and docs.

## Reproduction Assessment

Yes. Source inspection shows the default path leaves the checksum empty, skips sha256sum when it is empty, and continues into image conversion and template creation; I did not run the helper because this review is read-only.

## Solution Assessment

Yes. The accepted narrow path is to verify a pinned default image and require an operator-supplied digest for overrides, which preserves custom templates while closing the unverified default path.
