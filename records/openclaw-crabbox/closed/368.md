## Summary

Keep open: current main has Cloudflare Containers and Cloudflare Dynamic Workers providers, but not the requested distinct `cloudflare-sandbox` provider backed by the official Sandbox SDK bridge, and no canonical issue or PR already owns that exact work.

## What This Changes

Add a separate `cloudflare-sandbox` delegated-run provider that runs Linux commands through the official Cloudflare Sandbox SDK bridge instead of the existing Crabbox Cloudflare Containers runner.

## Best Possible Solution

If maintainers want this first-party integration, land it as a distinct provider adapter with the official Sandbox bridge, env/user-config-only secrets, conservative v1 features, focused docs, unit coverage, and guarded live proof before advertising future Sandbox-native capabilities.
