Keep open: the PR is useful, but the current patch still misses wrapped `RunInstances` failures at the region-fallback boundary and it does not include after-fix hosted broker proof.

## What This Changes

The PR makes XML-declaration-only EC2 HTTP 400 provisioning errors retryable as AWS capacity errors and adds AWS classifier regression assertions.

## Best Possible Solution

Keep the AWS-adapter classifier approach, but make it cover both raw and wrapped XML-only EC2 errors, add a region-fallback regression test, and require after-fix hosted broker proof before merge.

## Reproduction Assessment

Yes, from source inspection: a wrapped message such as `c7a.8xlarge: aws RunInstances: http 400: <?xml ...?>` can reach the region fallback, and the new anchored regex does not match it. I did not run tests because this was a read-only review.

## Solution Assessment

No; classifying only bare raw AWS Query messages is too narrow. The maintainable fix is to classify the XML-only suffix after existing wrappers or add an explicit region-level predicate/test for wrapped `RunInstances` failures.
