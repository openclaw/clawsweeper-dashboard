## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Reject data-run output path collisions:** `internal/cli/data.go:192-197`
  - body: `manifestLocal` and `summaryLocal` are each checked for being repo-relative, but nothing rejects them being the same path or matching a configured `run.Downloads` local target. In that case a successful data run can validate the downloaded manifest and then overwrite it with the summary, or validate a file clobbered by another configured download. Please preflight the cleaned manifest, summary, and download local targets as one set before starting the remote run.
  - confidence: 0.93
- **[P1] Reject URL-shaped proof paths:** `internal/cli/data.go:604-605`
  - body: The new data-run manifest validator says artifact paths must be safe relative paths, but `safeArtifactGlob` still allows values like `https://token@example.com/proof` or `s3://bucket/key` because `:`, `/`, and `@` are permitted. Since this feature is specifically meant to keep signed URLs and sensitive proof out of manifests and run records, reject scheme/URL-shaped values here and mirror that restriction in the Worker `dataSummary` sanitizer.
  - confidence: 0.86
