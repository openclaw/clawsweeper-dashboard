## Summary

Keep open: current main still has the reported Cloudflare ownership gap, with `stop` resolving a missing local claim to the raw identifier and then sending a runner delete request.

## What This Changes

The issue reports that `crabbox stop --provider cloudflare <raw-id>` can destroy a Cloudflare container when no local Crabbox claim matches the identifier.

## Best Possible Solution

Require an existing local Cloudflare claim before `status` or `stop` calls the runner, preserve claimed stale-lease pruning, and review `run --id` because it shares the same resolver.

## Reproduction Assessment

Yes, at source level: with an empty local claim store, `Stop` on a raw ID follows the fallback in `resolveSandboxID` and sends a runner DELETE. I did not run a dynamic exploit or test in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to fail closed in the Cloudflare claim-resolution path before destructive runner calls, with regression coverage for no-delete-on-unclaimed-ID. The shared `run --id` path should be checked as part of the same repair.
