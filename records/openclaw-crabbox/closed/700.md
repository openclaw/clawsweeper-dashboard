## Summary

Current main already removes the reported python3 fail-open path: the Proxmox live-smoke helper preflights perl before proof capture, uses a Perl redactor, and the public docs now name jq/perl plus distinguish private raw logs from redacted PR-ready evidence.

## What This Changes

The issue reports that the Proxmox live-smoke helper can leave unredacted temporary doctor evidence if python3 is unavailable.

## Best Possible Solution

Keep the shipped jq/perl redaction dependency model and current docs that separate private raw evidence from redacted shareable logs.

## Reproduction Assessment

No for current main: the reported missing-python3 path is gone because the helper does not call python3 and preflights perl before proof capture. The issue body's historical reproduction is credible, but it no longer matches the checked-out script.

## Solution Assessment

Yes: using a checked redaction dependency before evidence capture is the narrow maintainable fix for the reported fail-open path. Broader policy questions about retaining private .raw.log files are already documented separately from the redacted public evidence contract.
