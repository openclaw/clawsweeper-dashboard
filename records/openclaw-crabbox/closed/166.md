Replace the page/limit ListSandboxesPaginated loop in internal/providers/daytona/client.go with Daytona's supported cursor-based ListSandboxes API from v0.180.0 or newer, preserving label filtering for crabbox=true and X-Daytona-Organization-ID handling. Preserve existing lease inventory, slug allocation, status/list, resolve, stop, doctor, and delegated-run creation behavior; map SandboxListItem fields to the existing server/lease resolution needs if the new API no longer returns full Sandbox values. Add regression coverage for multiple cursor pages, empty/nil responses, and label/slug resolution. Update go.mod/go.sum only as required for the Daytona SDK/API client version bump and do not change unrelated providers or generated docs.
