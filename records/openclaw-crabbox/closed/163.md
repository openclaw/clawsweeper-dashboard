## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve explicit provider overrides for doctor runs:** `internal/cli/doctor.go:118-120`
  - body: When `--from-run` is used with an explicit `--provider`, this assignment overwrites the parsed flag with the recorded run provider before `cfg.Provider` is applied. That makes `crabbox doctor --from-run <id> --provider <x>` silently run the recorded provider path instead of the provider the user requested; only use the run provider as a default when the flag was not set.
  - confidence: 0.9
- **[P2] Quote shell retry payloads in digest suggestions:** `internal/cli/run_failure.go:232`
  - body: For shell-mode runs, `commandDisplay` is a shell fragment such as `npm test && npm run lint`, but the digest appends it raw after `--`. A copied retry hint can therefore let the local shell interpret `&&`, pipes, redirects, or semicolons instead of passing one remote payload; quote/build this as argv-safe text or suppress shell-mode retry hints.
  - confidence: 0.87
- **[P3] Avoid printing failure tails twice:** `internal/cli/run.go:1253-1268`
  - body: The new digest prints stderr/stdout tails, but the same failure path still calls the legacy tail printers later in `run.go`. Failed runs will show duplicate tail sections, so let either the digest or the legacy path own this output.
  - confidence: 0.84
