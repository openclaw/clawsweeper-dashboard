## Summary

Keep open: current main still returns WebVNC bridge commands and egress profile/allow details from visibility-gated paths, even though the documented share model distinguishes visible `use` access from manageable `manage` access.

## What This Changes

Restrict WebVNC bridge-command and egress diagnostic disclosure so `use` share viewers can use shared leases without seeing manage-only operator details.

## Best Possible Solution

Redact the WebVNC bridge command and egress profile/allow values for `use` viewers while preserving shared viewer access, with focused Worker tests for owner, manage-share, admin, and use-share behavior.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: a `use` share is visible but not manageable, and the WebVNC/egress status paths return manage-only details after only visibility resolution. I did not run a live Worker test in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is to gate or redact only the manage-only diagnostic fields while leaving visible shared WebVNC use intact.
