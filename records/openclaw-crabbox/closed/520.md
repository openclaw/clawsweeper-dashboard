## Summary

Keep this PR open: it is a useful, proof-backed Freestyle ownership hardening, but the changed predicate still accepts any already-canonical `crabbox-...` provider name rather than only names Crabbox actually emits. The compatibility-sensitive fail-closed direction is acknowledged in the discussion, but the remaining security gap needs a narrow fix before merge.

## What This Changes

The PR tightens Freestyle raw-ID ownership recovery, adds malformed-name and stop regression tests, updates Freestyle docs, and adds a changelog entry.

## Best Possible Solution

Land a Freestyle-adapter fix that accepts raw-ID recovery only for provider names matching the shape emitted by `newFreestyleSandboxName`, with regression tests for both malformed names and already-canonical manual names.

## Reproduction Assessment

Yes. Source inspection shows current main can route a malformed or overly broad Freestyle name through `resolveLeaseID` into `Stop`, and the PR tests/proof cover the malformed case; the remaining gap is source-reproducible with an already-canonical manual name like `crabbox-personal`.

## Solution Assessment

No. The adapter-local direction is right, but the PR should match the emitted Freestyle name shape rather than accepting every canonical `crabbox-...` prefix before the linked security issue is considered fixed.
