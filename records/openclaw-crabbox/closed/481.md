## Summary

Keep open: current main still has explicit egress cleanup only under `crabbox egress stop`; ordinary stop/release paths proceed from Actions/Tailscale cleanup to lease release, while coordinator release/expiry only closes sockets and does not remove local daemon pid state or SSH-kill the lease-side helper.

## What This Changes

`crabbox stop` should best-effort clean up mediated egress host daemons and lease-side egress clients before releasing a lease.

## Best Possible Solution

Add a provider-neutral best-effort egress cleanup helper to the CLI stop/release lifecycle, reuse the existing local daemon and remote kill behavior, document it, and leave coordinator socket closure as a fallback.

## Reproduction Assessment

Yes, at source level: start egress with `--daemon`, then inspect current main and ordinary `stop`/release paths do not call the existing local daemon cleanup or remote egress-client kill path. I did not run a live lease reproduction in this read-only review.

## Solution Assessment

Yes for the CLI stop path: reusing the existing egress cleanup pieces before `ReleaseLease` is the narrowest maintainable fix. Remote expiry cleanup may need complementary stale-state handling, but it should not replace explicit stop cleanup.
