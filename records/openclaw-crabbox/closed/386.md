- `impact:security`: The issue concerns mutable base images for runtimes that can execute operator jobs or run the coordinator with deployment secrets.

## Summary

This security report should stay open because current main still has the seven tag-only base-image references in the documented Cloudflare, Azure dynamic-sessions, and portable Node coordinator runtime Dockerfiles.

## What This Changes

The issue asks Crabbox to digest-pin shipped runtime image base layers, add a repository check for unpinned Dockerfile FROM lines, and document the digest update procedure.

## Best Possible Solution

Pin the affected runtime Dockerfile base images to reviewed digests while retaining readable tags, then add a focused check and docs for refreshing those digests.

## Reproduction Assessment

Yes. Source inspection on current main shows the named Dockerfiles still contain seven FROM lines without @sha256: digest pins, and the docs/config confirm those files are runtime image inputs.

## Solution Assessment

Yes in direction, but maintainer policy is still needed. Digest-pinning plus a focused check and update docs is the narrow maintainable fix for making base-image changes explicit and reviewable.
