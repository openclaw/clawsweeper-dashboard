## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Gate the macOS bridge on the resolved target:** `internal/cli/webvnc.go:1249-1256`
  - body: This helper treats any provider that advertises a macOS target as a macOS lease. Providers such as `parallels` advertise Linux, macOS, and Windows targets, so `crabbox webvnc --provider parallels --target linux` would be routed into the tart/macOS bridge instead of the existing direct SSH WebVNC path. Gate this on the resolved target being macOS, or narrowly on the tart provider when the target is still unresolved, and add a regression test for a multi-target provider.
  - confidence: 0.94
- **[P1] Keep the tart password out of URLs and argv:** `internal/cli/macos_webvnc.go:78-84`
  - body: `macOSWebVNCViewerURL` embeds the tart account password in the query string, then the bridge prints that URL and `--open` passes it to `open`/`xdg-open` as an argument. A custom `--tart-password` or `CRABBOX_TART_PASSWORD` is therefore exposed in logs, browser history, and local process argv, which violates the repo guidance not to pass secrets as command-line arguments. Use a prompt, local session token, or another in-memory handoff that keeps the password out of URLs and argv.
  - confidence: 0.91
