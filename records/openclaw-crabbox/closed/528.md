## Summary

Keep this PR open: it appears to be a focused fix for the linked coordinator credential redirect disclosure, but it still needs maintainer/security review for the redirect compatibility tradeoff and one release-note cleanup before merge.

## What This Changes

The branch blocks cross-origin redirects in the Go coordinator client, disables redirect following and ambient curlrc processing in the curl fallback, documents the redirect boundary, and adds redirect regression coverage.

## Best Possible Solution

Land a maintainer-approved fix that prevents coordinator bearer, Access, and local identity headers from being replayed across origins, with focused regression coverage for both Go and curl fallback paths.

## Reproduction Assessment

Yes, source inspection of current main shows the curl fallback still emits `location` together with bearer, Access, and identity headers. I did not run tests in this read-only review, but the PR adds a real-curl harness for the attack path.

## Solution Assessment

Yes, the implementation path is narrow: reject cross-origin redirects in the Go path and make the curl fallback non-redirecting. The safer alternative, if redirect compatibility is required, is an explicit same-origin curl redirect design with matching tests.
