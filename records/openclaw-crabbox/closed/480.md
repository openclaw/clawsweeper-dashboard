## Summary

## What This Changes

The issue asks Crabbox to preserve valid JUnit results when one report is malformed, avoid silently dropping oversized auto-discovered reports, and add an opt-in way for JUnit failures to affect run status.

## Best Possible Solution

Ship a focused robustness fix for partial result preservation and skipped-file warnings, with a separate maintainer-approved opt-in policy for making JUnit failures affect run exit status.

## Reproduction Assessment

Yes for the parser and auto-drop portions from source inspection: malformed XML returns an error, `parseJUnitResults` returns nil, and auto filtering only keeps files whose capped body fully parses. I did not execute an end-to-end run because this was a read-only review.

## Solution Assessment

Partly: preserving valid parsed files and warning on skipped result files is the narrow maintainable repair. The fail-on-JUnit-failures request needs an opt-in flag/config design and maintainer policy decision before implementation.
