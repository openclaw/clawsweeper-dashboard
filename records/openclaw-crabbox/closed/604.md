## Label Justifications

- `P1`: The issue reports a high-impact coordinator authorization failure where revoked shared users can retain live interactive bridge access.
- `impact:security`: The affected behavior is an authorization boundary failure in coordinator ownership and sharing controls.
- `impact:session-state`: The stale authorization lives in active WebSocket/session state after the persisted lease share state changes.

## Summary

Keep open: current main still persists share changes without closing or re-authorizing existing WebVNC and Code viewer sockets, so the reported coordinator authorization gap remains source-reproducible and is not owned by a merged fix or open PR.

## What This Changes

Ensure revoking a lease share immediately terminates or invalidates existing WebVNC and Code viewer sockets for principals that lost access.

## Best Possible Solution

Add Worker coordinator share-shrink revocation that removes live WebVNC and Code viewer access for principals that no longer satisfy leaseAccessRole, with regression tests for API and portal share removal paths.

## Reproduction Assessment

Yes. Source tracing on current main shows share mutation routes persist the new share while live WebVNC and Code viewer sockets continue forwarding from stored bridge attachments without a fresh leaseAccessRole check.

## Solution Assessment

Yes in principle. The narrow maintainable fix is to revoke affected live bridge state when sharing shrinks, while regression tests prove new requests and already-open viewer sockets both lose access after revocation.
