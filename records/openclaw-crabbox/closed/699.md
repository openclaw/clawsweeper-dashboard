## Summary

Current main and release v0.33.0 already block the reported Proxmox label-to-local-key path escape by validating lease IDs before constructing the stored testbox key path, and the Proxmox resolve path now reaches that shared validation before setting target.Key.

## What This Changes

The issue reports that Proxmox-provided lease labels can drive Crabbox's local per-lease SSH key path outside the expected key directory.

## Best Possible Solution

Keep the shipped shared lease-ID validation and traversal regression test; open a new issue only if current main can still select a key outside the Crabbox testboxes directory.

## Reproduction Assessment

No for current main: the pre-fix source path was plausible, but current main and v0.33.0 reject traversal-style lease IDs before constructing the local key path.

## Solution Assessment

Yes: validating at the shared stored-key path boundary is the narrow maintainable fix because it protects Proxmox and other callers without duplicating provider-specific path checks.
