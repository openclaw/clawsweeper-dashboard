## Summary

Keep open: current main still rejects `--emit-proof` for delegated providers and Blacksmith does not advertise proof support, while this branch adds the missing opt-in feature and appears directionally sound. ## What This Changes

The branch adds an opt-in `FeatureRunProof` capability, enables Blacksmith delegated runs to return proof metadata and bounded local artifacts, writes delegated `--emit-proof` output after successful runs, and documents/tests the behavior.

## Best Possible Solution

Land the opt-in delegated proof implementation after CI and inspectable Blacksmith proof evidence confirm the generated proof and local artifacts in a real run.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature PR. Source inspection does show the current-main baseline rejects delegated `--emit-proof`, which is the behavior the branch changes.

## Solution Assessment

Yes, the implementation direction fits the existing design: delegated providers opt into proof rendering with a feature flag and reuse core proof rendering. The safer merge path is to keep it opt-in and require inspectable real-run proof before landing.
