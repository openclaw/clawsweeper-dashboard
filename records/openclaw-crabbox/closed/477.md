## Summary

Keep open: current main still shows the reported unvalidated AWS region dataflow into SigV4-signed AWS endpoints, and no merged PR or duplicate issue appears to own this exact fix yet.

## What This Changes

The issue reports that unvalidated AWS region inputs can be interpolated into AWS SigV4 service endpoints, allowing an authenticated non-admin caller to redirect signed coordinator requests to a caller-controlled host.

## Best Possible Solution

Validate AWS regions at the lease config and readiness ingress points, add adapter-level validation in `EC2SpotClient`, and cover malicious path-bearing regions in Worker regression tests.

## Reproduction Assessment

Yes. Source inspection on current main shows `region`/`awsRegion` reaching AWS endpoint construction without the existing sanitizer, though no live AWS request was executed.

## Solution Assessment

Yes. The narrowest maintainable fix is to reuse or centralize the existing AWS region allowlist at the affected ingress paths and keep a defense-in-depth check inside the AWS adapter.
