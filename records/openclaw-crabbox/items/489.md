## Repair Work Prompt

On https://github.com/openclaw/crabbox/pull/489, fix the xclip branch generated by `desktopPasteRemoteCommand` so an early status-0 exit from `xclip -loops 1` before Ctrl+V is treated as a successful clipboard-manager handoff rather than `clipboard helper exited before paste`. Preserve failure propagation for nonzero early exits and for nonzero helper failures after Ctrl+V. Add a focused regression in `internal/cli/desktop_test.go` with a fake xclip that exits 0 before paste and assert the script still sends Ctrl+V. Do not change unrelated desktop launch/proof semantics, provider behavior, or release files. Related canonical issue: https://github.com/openclaw/crabbox/issues/482.
