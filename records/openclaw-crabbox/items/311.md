## Summary

This should stay open: current main still uses the Tailscale package install script in both managed Linux bootstrap renderers, has no Tailscale install-mode/version/checksum config, and does not record `tailscale version` metadata outside the Islo-specific pinned path.

## What This Changes

Request an opt-in deterministic Tailscale bootstrap mode with pinned checksum verification and recorded client-version metadata while preserving the current default install behavior.

## Best Possible Solution

Add a provider-neutral Tailscale install policy that keeps the current installer default, offers a pinned static archive mode with checksum verification, shares or validates version data against the Islo path, records `tailscale version`, and updates CLI/Worker docs and tests together.

## Reproduction Assessment

Not applicable as a feature/hardening request; source inspection against current main confirms the starting point: CLI and Worker bootstrap still use `install.sh` and no `tailscale-version` metadata/config exists.

## Solution Assessment

Unclear until maintainers choose the config and version-ownership model; the requested direction is maintainable if it stays provider-neutral and default-compatible.
