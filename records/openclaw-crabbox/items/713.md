## Summary

Keep this PR open: current main and v0.33.0 still rewrite the credential-bearing user config with a direct write, while this PR provides a focused atomic-replace fix and regression coverage. The remaining maintainer decision is the compatibility tradeoff for custom config paths that were writable as a file but not replaceable through same-directory temp-file creation and rename.

## What This Changes

The PR changes the Crabbox CLI user-config writer to stage rewrites through a same-directory temporary file, fsync, and atomic replacement, with regression tests for success, failed rename cleanup, and symlinked config paths.

## Best Possible Solution

Land the focused atomic config rewrite after maintainers accept or document the custom config-path directory-entry permission requirement.
