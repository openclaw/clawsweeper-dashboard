Full review comments:

- **[P1] Avoid selecting an unsupported sandbox provisioner:** `mobile/App/Settings/AppSettings.swift:120-124`
  - body: `makeProvisioner()` returns `CoordinatorProvisioner` whenever a crabbox.sh token is saved, but that provisioner's `launch`, `list`, and `stop` methods all throw unsupported. Users with a portal token, including users who also configured islo, will see Sandboxes/targeting controls enabled and then hit failures instead of the working direct-islo path. Make the selection capability-aware or hide unsupported coordinator lifecycle actions.
  - confidence: 0.9
- **[P1] Fail launches that never become ready:** `mobile/App/Sandboxes/SandboxesView.swift:404-410`
  - body: The islo bootstrap now requires an image that already has `ollama`, while the default spec is still plain `ubuntu:26.04`; after 90 failed readiness polls this code still registers the engine and marks the launch ready. That can offer a non-working sandbox engine for chat, so the launch should fail unless readiness succeeds or the default image should be Ollama-ready.
  - confidence: 0.88
