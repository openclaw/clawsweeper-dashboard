- **[P1] Use the existing coordinator lease API:** `mobile/Sources/CrabboxKit/CoordinatorClient.swift:42`
  - body: The primary `crabbox.sh` provisioner posts to `/v1/sandboxes`, but the current coordinator contract exposes lease creation/list/release/share under `/v1/leases`. As written, the app's default sandbox path will fail against the deployed coordinator instead of launching a sandbox.
  - confidence: 0.92
- **[P1] Validate provider URLs before sending credentials:** `mobile/App/Settings/AppSettings.swift:111-119`
  - body: `makeProvisioner()` passes stored coordinator and islo base URLs directly into clients that attach Bearer/session/API credentials. A mistyped or hostile `http://`/foreign URL can receive those secrets; normalize and require trusted HTTPS endpoints before constructing either provisioner.
  - confidence: 0.95
- **[P2] Pin or replace the root bootstrap installer:** `mobile/Sources/CrabboxKit/IsloClient.swift:276`
  - body: Every direct islo LLM launch runs `curl https://ollama.com/install.sh | sh` as root in the leased sandbox and ignores installer failure.   - confidence: 0.77
