## Label Justifications

- `impact:security`: The central impact is leaking trusted credentials across a redirected-origin security boundary.
- `impact:auth-provider`: The affected headers include coordinator bearer tokens and Cloudflare Access auth headers.

## Summary

Current main and the latest release still contain the reported coordinator curl fallback behavior, so this security issue should stay open rather than being cleaned up as already implemented.

## What This Changes

The issue asks Crabbox to stop the coordinator curl fallback from replaying bearer, Cloudflare Access, and local identity headers to redirected origins.

## Best Possible Solution

Patch the coordinator curl fallback so credentialed requests do not follow redirects with custom auth or identity headers, and add focused regression coverage.

## Reproduction Assessment

Yes from source inspection: current main still emits `location` and credential-bearing custom headers in the curl fallback config for eligible coordinator GET/HEAD failures. I did not run a live redirect harness because the read-only source path is explicit and sufficient for triage.

## Solution Assessment

Yes: disabling redirect following for credentialed curl fallback requests is the narrowest maintainable fix; if redirect support is required, same-origin policy should be explicit and covered by tests.
