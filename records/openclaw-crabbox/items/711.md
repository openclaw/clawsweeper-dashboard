## Label Justifications

- `P1`: The issue describes root execution of unverified third-party installer bytes in documented managed lease bootstrap workflows, with high impact if an upstream installer path is compromised.
- `impact:security`: The affected behavior concerns supply-chain integrity, root bootstrap execution, trusted lease setup, and possible exposure or tampering of lease contents.

## Summary

Keep open: current main still executes unpinned third-party installer scripts as root for managed Linux Code and default Tailscale bootstrap, and repository policy treats downloaded artifact integrity failures in documented workflows as security-sensitive.

## What This Changes

The issue asks Crabbox to replace managed Linux Code and default Tailscale root `curl | sh` installer paths with pinned, verified bootstrap installs.

## Best Possible Solution

Use one canonical security-tracked issue for the combined installer-integrity problem, then add pinned, versioned, digest-verified Code and default Tailscale bootstrap paths in both Worker and CLI generators with regression coverage.

## Reproduction Assessment

Yes. Source inspection of current main shows the exact Code and default Tailscale root `curl | sh` bootstrap paths, and existing tests assert those installer URLs; no live upstream compromise was attempted.

## Solution Assessment

Yes on the security direction, but maintainers need to choose the upgrade policy. Pinned, digest-verified artifacts match the repository's existing bootstrap integrity pattern, while default-mode changes and fail-closed behavior need explicit maintainer approval.
