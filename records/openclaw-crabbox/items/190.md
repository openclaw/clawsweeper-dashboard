## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Emit --json before the harness path:** `index.js:321-322`
  - body: `crabbox_harness_validate` currently builds `harness validate HARNESS.md --json`. The real `harnessValidate` path uses Go `flag.Parse`, which stops at `HARNESS.md`; `--json` is then left as a second positional arg and the command returns usage instead of JSON. Emit `--json` before the path or make the parser accept interspersed flags, and update the test to exercise the real parser behavior.
  - confidence: 0.93
- **[P2] Match required artifact names:** `internal/cli/harness.go:493-496`
  - body: When `required_artifacts` is set, this only checks `len(artifacts) == 0`, so any unrelated artifact makes every required artifact pass. A harness that requires `screenshots` can pass with only a different artifact kind present, undermining the compliance failure promised by the docs. Check each required artifact against the collected artifact kind/name/path and report the specific missing entries.
  - confidence: 0.88
- **[P3] Move harness flags before the job name:** `README.md:97`
  - body: This new example puts `--harness` after the job name, but `jobRun` also parses flags before positionals. Copying it leaves the flag as an extra positional and fails usage; use `crabbox job run --harness HARNESS.md full-ci` here and in the matching feature docs example.
  - confidence: 0.9
