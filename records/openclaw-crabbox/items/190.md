Full review comments:

- **[P1] Forward documented job harness defaults to the nested run:** `internal/cli/job.go:314-318`
  - body: `JobConfig` now loads the full `harness:` block and the docs advertise `harness.compliance`, `template`, `plan_file`, and `scope`, but this nested `run` invocation only receives `--harness` and `--index`. A job that sets `require_junit` or `required_artifacts` in `.crabbox.yaml` therefore runs without those compliance requirements unless they are duplicated in `HARNESS.md`; preserve those defaults in the nested run or stop documenting them as job config.
  - confidence: 0.91
- **[P2] Create the no-sync workdir before uploading harness grounding:** `internal/cli/run.go:1056-1058`
  - body: For `--no-sync` runs, this remote evidence upload runs before the later no-sync workdir creation block. The upload helper starts by `cd`ing into `workdir`, so `crabbox run --no-sync --harness ...` can fail before the command starts when the workdir has not already been created; move the upload after the no-sync mkdir or make the upload create the workdir first.
  - confidence: 0.88
