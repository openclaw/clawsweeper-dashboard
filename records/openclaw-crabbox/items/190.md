- **[P1] Forward all job harness defaults to the nested run:** `internal/cli/job.go:314-319`
  - body: `JobConfig.Harness` now accepts `template`, `plan_file`, `scope`, and `compliance`, and the docs advertise those job-level defaults, but `jobRunArgs` only forwards `--harness` and `--index`. A configured job can therefore silently ignore required compliance evidence unless the same fields are duplicated in the harness frontmatter; either pass these defaults into the run path or reject/remove the unsupported job config fields.
  - confidence: 0.91
- **[P1] Create the workdir before uploading no-sync grounding:** `internal/cli/run.go:1056-1058`
  - body: With `--no-sync --harness` and the default `--index light`, this block uploads harness grounding before the later no-sync `remoteMkdir(workdir)` block runs. The upload helper starts with `cd <workdir>`, so a kept lease without that repo directory fails before the command runs; create the workdir before `writeHarnessRemoteEvidence` or skip remote upload for this path.
  - confidence: 0.88
