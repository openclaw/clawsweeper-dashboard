Full review comments:

- **[P2] Forward all documented job harness defaults:** `internal/cli/job.go:314-319`
  - body: `job.Harness` can contain `template`, `plan_file`, `scope`, and `compliance` from `.crabbox.yaml`, and the docs advertise those keys, but the nested `run` argv only receives `--harness` and `--index`. That means job-level compliance defaults are silently dropped unless duplicated in the harness file, so either pass/apply the full harness config or stop documenting those job defaults.
  - confidence: 0.92
- **[P2] Create the no-sync workdir before uploading harness evidence:** `internal/cli/run.go:1051-1053`
  - body: With `--no-sync`, the existing code creates the remote workdir later in the command path, but the new harness upload runs before that block and uses an upload helper that first `cd`s into the workdir. A no-sync harness run against a lease without that directory will fail before running the command; move the upload after the no-sync mkdir or make the upload path create the workdir first.
  - confidence: 0.87
- **[P3] Remove project-specific names from the generic roadmap:** `docs/plan/crabbox-harness-station-reasoning-roadmap.html:379-380`
  - body: The new roadmap page is not a legacy or release-history file, but it frames the generic Crabbox roadmap around OpenClaw, Codex, and Peter-specific wording. AGENTS.md asks new Crabbox docs to avoid project/person-specific workflows outside explicit exceptions, so this page should be neutralized or moved to an explicitly scoped place.
  - confidence: 0.84
