Full review comments:

- **[P1] Honor Docker context precedence when replaying daemon scope:** `internal/cli/checkpoint.go:209-210`
  - body: `DOCKER_CONTEXT` overrides `DOCKER_HOST` for the Docker CLI, but this records `DOCKER_HOST` whenever it is present and the replay helpers treat that host as the pinned daemon. If both variables are set, `docker commit` uses the context daemon while verify/delete/fork can target the host daemon or inherit a later context, so a checkpoint can appear missing or be deleted/forked on the wrong daemon. Record only the effective scope and scrub `DOCKER_CONTEXT` when replaying a true `DOCKER_HOST`, with coverage for both env vars set.
  - confidence: 0.88
