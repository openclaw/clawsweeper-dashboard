Full review comments:

- **[P1] Honor Docker context precedence for checkpoint scope:** `internal/cli/checkpoint.go:1672-1675`
  - body: Docker's CLI treats DOCKER_CONTEXT/--context as overriding DOCKER_HOST. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/?utm_source=openai)) This function records the host whenever it is set, and the create path uses the same assumption while clearing DOCKER_CONTEXT before docker commit. A local-container lease resolved under a named context while DOCKER_HOST is also present can therefore commit against a different daemon or fail to find the source container, and verify/delete/fork then replay the wrong scope. Record/replay the context when DOCKER_CONTEXT is set and only pin DOCKER_HOST when it is the effective Docker scope.
  - confidence: 0.9
