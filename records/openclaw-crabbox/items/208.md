## Summary

This PR remains useful and has strong live Docker proof, but it is not merge-ready: it rejects valid checkpoints created by the recently merged foundation, and mutable Docker daemon scope can leak from a checkpoint fork into later unscoped lease operations on the reused backend.

## What This Changes

Adds native local-container checkpoint forks from Docker-commit checkpoints with recorded daemon-scope replay, immutable-image validation, lease metadata persistence, and workspace relocation.

## Best Possible Solution

Make checkpoint daemon scope lease-local or reset it before every operation, then derive safe user and work-root values for foundation-era records while preserving all immutable-image and recorded-daemon validation.

## Reproduction Assessment

Yes. A foundation-era record reaches the explicit missing-relocation-metadata rejection, and a scoped lease followed by an unscoped lease on the same backend leaves the prior checkpoint metadata and runtime active.

## Solution Assessment

No in its current form. Native forks are the appropriate provider capability, but scope must be reset per lease and already-valid checkpoint records need a narrow compatibility path before the implementation is safe.
