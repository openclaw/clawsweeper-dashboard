Full review comments:

- **[P1] Honor docker-socket lease labels before enabling docker commit:** `internal/providers/localcontainer/provider.go:54-55`
  - body: `NativeCheckpointCapability` only rejects docker-socket mode from `req.Config`, but `checkpoint create` resolves the existing lease and then calls `checkpointCreateMode` without applying the resolved server labels back onto `cfg`. For a socket-enabled local-container lease whose setting only comes from the recorded `docker_socket` label, `--mode native` can still take the docker-commit path and produce a checkpoint image missing the host-mounted work root. Please make the native capability check see the resolved lease label, and add a regression for that existing-lease path.
  - confidence: 0.88
- **[P3] Advertise fork support in the provider feature set:** `internal/providers/localcontainer/provider.go:27`
  - body: This provider now implements `ApplyNativeCheckpointForkConfig`, but `Spec().Features` adds only `FeatureCheckpoint`. Provider docs reserve `FeatureFork` for providers that can create a new workspace from a checkpoint, so `crabbox providers` and feature consumers would not see the fork capability this PR adds.
  - confidence: 0.84
