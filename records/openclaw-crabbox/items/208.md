## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Make Docker checkpoint image references immutable:** `internal/cli/checkpoint_native.go:167`
  - body: This PR carries the base docker-commit create path, where `--name` becomes the reusable Docker tag. Creating another checkpoint with the same friendly name retags `crabbox-checkpoint-<name>`, while verify/delete later prefer that name, so an older record can verify or delete the newer image. Use a per-record unique tag or verify/delete by an immutable recorded image reference.
  - confidence: 0.91
- **[P2] Guard docker-socket forks from hiding the saved workdir:** `internal/providers/localcontainer/provider.go:73`
  - body: `checkpoint fork` applies normal lease flags before this hook, so a current config or flag can leave `LocalContainer.DockerSocket` true. The local-container backend then mounts a host work root over the container work root, hiding the committed image's saved workspace before relocation runs. Reject or clear docker-socket mode for docker-commit forks, or add proof and coverage for preserving the workdir.
  - confidence: 0.84
