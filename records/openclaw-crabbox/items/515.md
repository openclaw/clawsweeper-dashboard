## What This Changes

The issue asks Crabbox to reject malformed Islo sandbox identifiers that only look Crabbox-owned after lossy slug normalization.

## Best Possible Solution

Land a narrow Islo-provider fix that requires raw and `isb_` sandbox identifiers to already be canonical Crabbox-created names, with regression tests for malformed resolver and bridge inputs.

## Reproduction Assessment

Yes. Source inspection shows the resolver accepts names through `isCrabboxIsloSandboxName`, that predicate slugifies before checking `crabbox-`, and the original raw name is then returned to provider operations.

## Solution Assessment

Yes. Requiring the actual Islo identifier to already equal its normalized/canonical form is the narrowest maintainable fix and matches the documented generated-name contract.
