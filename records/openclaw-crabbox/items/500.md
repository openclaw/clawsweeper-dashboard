## Summary

Keep open. Current main still renders the Proxmox API URL raw in both config-show outputs, the docs still promise redacted secret output, and no merged/current fix owns the narrower Proxmox-specific problem.

## What This Changes

The issue reports that `crabbox config show` and `--json` expose Proxmox API URL userinfo instead of using the existing URL redaction helper.

## Best Possible Solution

Make a narrow maintainer decision for Proxmox: either redact Proxmox API URL userinfo in config-show with regression tests, or update the documented redaction contract to explicitly exclude unsupported URL userinfo.

## Reproduction Assessment

Yes. Source inspection on current `main` shows `configShowView` assigns `proxmox.apiUrl` from `cfg.Proxmox.APIURL` and `writeConfigShowText` prints `cfg.Proxmox.APIURL` directly, while adjacent XCP-ng output routes the same shape through `redactedConfigURL`.

## Solution Assessment

Unclear. The code fix is small and matches existing XCP-ng behavior, but https://github.com/openclaw/crabbox/issues/421 was closed with maintainer pushback against broad URL-userinfo redaction, so the final Proxmox policy needs maintainer confirmation.
