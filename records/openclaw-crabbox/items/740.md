## Summary

Keep open: this PR is the live candidate for https://github.com/openclaw/crabbox/issues/716, and current main still contains the unsafe AWS raw-ID release path. The patch looks directionally correct, but the external PR still needs real AWS behavior proof and maintainer acceptance of the raw-ID compatibility change before merge.

## What This Changes

Adds an AWS provider ownership-tag predicate for raw `i-*` resolution and regression tests for rejecting external or wrong-provider instances while allowing managed legacy instances.

## Best Possible Solution

Land a provider-local AWS ownership predicate with focused regression coverage after real behavior proof is added and maintainers explicitly accept fail-closed raw-ID behavior for non-Crabbox EC2 instances.

## Reproduction Assessment

Yes from source inspection: current main resolves a raw AWS `i-*` ID into a release target, and the stop path can pass it to AWS termination without Crabbox ownership tags. I did not run live AWS because that would require real credentials and a disposable EC2 instance.

## Solution Assessment

Yes, mostly: the provider-local guard mirrors Azure/GCP ownership checks and fixes the release-target boundary. The remaining merge question is proof and explicit acceptance of fail-closed raw-ID compatibility impact.
