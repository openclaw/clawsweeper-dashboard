## Label Justifications

- `P2`: This is a real configuration-safety bug in an opt-in path, but it is not an emergency or active core runtime outage.
- `impact:data-loss`: The current auto-bootstrap path can silently drop operator comments and formatting from a shared configuration file.
- `impact:security`: The affected file is the tailnet policy that controls Tailscale access rules for the operator's tailnet.

## Summary

Keep open: current main still strips HuJSON comments/trailing commas and rewrites the whole pond ACL policy as pretty JSON, and no canonical related issue or PR appears to supersede this safety fix.

## What This Changes

Make pond ACL auto-bootstrap preserve an existing Tailscale HuJSON policy while adding only the missing pond tag owner and self-peer rule.

## Best Possible Solution

Use a HuJSON-preserving minimal edit, or refuse unsupported HuJSON with a manual snippet, while keeping ETag compare-and-set behavior and adding golden/idempotence tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a commented HuJSON policy missing the pond row flows through hujsonStandardize, then json.Unmarshal/json.MarshalIndent, which drops comments before PUT.

## Solution Assessment

Unclear on the exact implementation choice. The requested direction is sound, but maintainers should choose between a HuJSON-preserving editor and a fail-closed manual fallback because the path edits an ACL policy.
