Full review comments:

- **[P1] Keep delegated downloads fail-closed per adapter:** `internal/cli/provider_backend.go:835-837`
  - body: This gate now allows `--download` for any delegated provider with `FeatureRunArtifacts`. Blacksmith already advertises that feature for artifact globs/required artifacts, but its `Run` method only calls artifact collection when `ArtifactGlobs` or `RequiredArtifactGlobs` are set and never processes `Downloads`, so `crabbox run --provider blacksmith-testbox --download ...` can succeed without writing the requested file instead of failing closed. Please gate downloads on an adapter path that actually handles them, or extend the common delegated artifact contract and update the providers/tests together.
  - confidence: 0.91
- **[P2] Remove the unsupported W&B data-run example:** `docs/commands/data.md:54-67`
  - body: The docs tell users with delegated providers that reject `--shell` to use `commandArgs` and show `provider: wandb`, but `crabbox data run` always adds `--require-artifact` and `--download`, and W&B still advertises no artifact retrieval feature. Copying this example will fail before the data command can run; use Islo for the delegated example or explicitly state that W&B remains unsupported for data runs until it implements bounded retrieval.
