- **[P1] Gate delegated downloads on real download support:** `internal/cli/provider_backend.go:835-836`

  - confidence: 0.94

  - body: The example config uses `provider: wandb`, but `crabbox data run` always adds `--require-artifact` and `--download` for the manifest, while W&B has no run-artifact feature and its docs still say those flags are rejected. Users copying this example will hit the delegated-provider fail-closed path, so use Islo or a supported SSH provider here, or explicitly mark W&B as unsupported for data runs until artifact retrieval exists.
  - confidence: 0.9
