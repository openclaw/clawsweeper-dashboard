Full review comments:

- **[P1] Keep unsupported delegated downloads fail closed:** `internal/cli/provider_backend.go:835-837`
  - body: This guard now lets any provider with `FeatureRunArtifacts` accept `--download`, but current main's Blacksmith implementation only collects `ArtifactGlobs` and `RequiredArtifactGlobs` and never materializes `req.Downloads`. That means an existing Blacksmith delegated run can appear to accept a requested download while no local file is written; keep `--download` rejected unless the selected provider has an actual download backend, or route downloads through a shared provider contract that Blacksmith implements.
  - confidence: 0.91
- **[P2] Avoid duplicating configured data-run downloads:** `internal/cli/data.go:469`
  - body: `dataRunJobConfig` copies `run.Downloads` into `JobConfig`, and `jobLeaseCreateArgsFor` already emits those as `--download`; `dataRunArgs` then appends the same `run.Downloads` again after the manifest download. Any `dataRuns` entry with extra downloads will generate duplicate local output targets and fail run validation before execution, so only one of those paths should emit user-configured downloads.
  - confidence: 0.88
