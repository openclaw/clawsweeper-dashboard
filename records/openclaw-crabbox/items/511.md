## Summary

This issue should stay open because current main still embeds the GitHub Actions runner registration token in SSH command text for both Linux/WSL2 and native Windows runner registration, which conflicts with Crabbox's documented secret-handling boundary.

## What This Changes

The issue asks Crabbox to stop placing the GitHub Actions runner registration token in `actions register` SSH command arguments and carry it through an off-argv channel instead.

## Best Possible Solution

Keep the existing `actions register` workflow, but make the SSH command generic and have the remote wrapper read runner metadata plus the registration token from stdin or another off-argv channel, with regression tests for Linux/WSL2 and native Windows.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: call `githubActionsRunnerInstallRemoteCommand` with a sentinel token and the returned Linux/WSL2 or Windows command contains that token.

## Solution Assessment

Yes. The requested direction is the narrowest maintainable fix because it preserves the runner feature while moving credential transport off argv-like command text.
