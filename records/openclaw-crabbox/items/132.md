## Summary

Keep this PR open: current main does not have a W&B provider, but the proposed branch still has a blocking auth-path defect and lacks successful real W&B sandbox proof.

## What This Changes

The PR adds a W&B Sandboxes delegated-run provider backed by an embedded Python cwsandbox shim, plus provider config, docs, registration, and tests.

## Best Possible Solution

Fix the W&B auth wiring to use cfg.Wandb.APIKey consistently, add regression coverage for the CRABBOX_WANDB_API_KEY-only and override cases, then attach redacted successful live provider proof before merge.

## Reproduction Assessment

Yes. From source, setting only CRABBOX_WANDB_API_KEY populates cfg.Wandb.APIKey, but Run/Doctor check os.Getenv("WANDB_API_KEY") and the shim receives os.Environ() unchanged, so the documented CRABBOX-only auth path fails.

## Solution Assessment

No. The provider shape is plausible, but the current implementation should route the configured key into the shim environment and preserve transport/API errors separately from remote command exit codes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Map the configured W&B key into the shim environment:** `internal/providers/wandb/backend.go:42-43`
  - body: The docs/config accept `CRABBOX_WANDB_API_KEY` as the preferred secret, but `Run` rejects unless `WANDB_API_KEY` exists, and the shim is launched with `os.Environ()` unchanged while reading only `WANDB_API_KEY`. With only `CRABBOX_WANDB_API_KEY` set, auth fails despite config parsing succeeding; use `cfg.Wandb.APIKey` to set `WANDB_API_KEY` for the shim and test the override case.
  - confidence: 0.96
- **[P2] Preserve shim failures instead of treating them as command exits:** `internal/providers/wandb/client.go:255-258`
  - body: `Exec` converts every `*wandbAPIError` into a sandbox command exit code. That is only valid after the remote process starts; shim failures such as bad auth, missing sandbox, import errors, or SDK failures also arrive as `*wandbAPIError` and will be reported as a generic sandbox exit, making provider failures look like user-command failures.
  - confidence: 0.8
