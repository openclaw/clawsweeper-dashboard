## Summary

Keep open: the new live output is enough real-behavior proof for the basic W&B sandbox lifecycle, but the current PR head still has blocking auth-provider bugs in the documented CRABBOX-prefixed credential path and can hide shim failures as sandbox command exits.

## What This Changes

Adds a W&B Sandboxes delegated-run provider with config and env plumbing, an embedded Python cwsandbox shim, provider registration, docs, changelog, and tests.

## Best Possible Solution

Land the provider only after auth uses cfg.Wandb.APIKey end-to-end, shim errors remain distinguishable from remote command exits, and regression tests cover WANDB-only, CRABBOX-only, and override paths.

## Reproduction Assessment

Yes. Source inspection shows applyEnv sets cfg.Wandb.APIKey from CRABBOX_WANDB_API_KEY, but Run and Doctor gate on os.Getenv("WANDB_API_KEY") and runShim passes os.Environ(), so a CRABBOX-only setup fails before or inside the shim.

## Solution Assessment

No. The provider shape is plausible and the live lifecycle proof now helps, but the implementation should route the configured key into every shim call and preserve provider/shim errors separately from remote command exits.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Use the configured W&B API key in preflight:** `internal/providers/wandb/backend.go:42-43`
  - body: The PR documents `CRABBOX_WANDB_API_KEY` as accepted and `applyEnv` writes it into `cfg.Wandb.APIKey`, but `Run` checks only `os.Getenv("WANDB_API_KEY")` before building the client. With only the namespaced variable set, `crabbox run --provider wandb --no-sync` exits before the shim despite valid config; `Doctor` repeats the same direct env gate.
  - confidence: 0.96
- **[P1] Propagate the configured key to the shim process:** `internal/providers/wandb/client.go:160`
  - body: `runShim` launches Python with `Env: os.Environ()`, while the shim requires `WANDB_API_KEY`. Credentials provided via `CRABBOX_WANDB_API_KEY` or any config path that fills `cfg.Wandb.APIKey` do not reach `list`, `status`, `stop`, or `exec`, so append `WANDB_API_KEY` from the configured key when it is present.
  - confidence: 0.97
- **[P2] Preserve shim errors on exec failures:** `internal/providers/wandb/client.go:255-258`
  - body: `Exec` converts every `*wandbAPIError` into a sandbox process exit code. For an existing `--id` run, auth failures, missing SDK/import errors, or sandbox lookup failures become `wandb sandbox exit=N` instead of actionable provider errors.
  - confidence: 0.86
