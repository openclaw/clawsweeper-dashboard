## Summary

Keep this PR open, but it needs a small auth-precedence fix before merge: the branch documents config-file W&B keys as higher precedence than the generic `WANDB_API_KEY`, while the config env pass currently lets `WANDB_API_KEY` overwrite that config value before the client resolves credentials.

## What This Changes

Adds a built-in W&B/CoreWeave Sandboxes delegated-run provider with native Go gRPC client, generated v1beta2 stubs, config/env/flags, docs, provider registration, unit tests, and live-smoke support.

## Best Possible Solution

Fix the documented credential precedence and add a regression test, then merge only if maintainers intentionally accept W&B/CoreWeave v1beta2 as an experimental built-in provider.

Not applicable as a feature PR rather than a bug report. The main run path is source-verifiable from the PR head and has live output proof, while the auth-precedence blocker is a source-level configuration-path issue.

## Solution Assessment
