## Summary

Keep this issue open: current main and v0.29.0 do not include an OVHcloud provider, and no related issue or pull request owns the request. The shipped external-provider protocol is a credible experimentation path, but it does not deliver the requested named first-party configuration and support contract, so maintainers still need to decide whether OVH should become a built-in adapter.

## What This Changes

Add a first-party direct OVHcloud Public Cloud SSH-lease provider with signed authentication, typed configuration, lifecycle and cleanup support, tests, documentation, and optional live validation.

## Best Possible Solution

Adopt OVH as a built-in adapter only after maintainers approve its support and endpoint policy; meanwhile, validate demand and lifecycle safety with a separately versioned `provider: external` implementation, then require redacted live proof of create, SSH readiness, inventory, stop, and conservative cleanup before core adoption.
