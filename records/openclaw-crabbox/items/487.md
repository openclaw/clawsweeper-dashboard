## Summary

Keep open: the follow-up commit addresses the shell-injection issue and removes the timeout-default change, but the PR still lacks promised real Windows + Hyper-V + WSL2 proof and introduces a compatibility blocker when WSL probe execution fails for reasons other than target reachability.

## What This Changes

The PR adds a WSL-side TCP reachability probe before Windows WSL rsync selection, falls back to native rsync when the probe reports unreachable or unsafe input, and adds Go tests for the probe and input validation.

## Best Possible Solution

Preserve the WSL rsync path unless an actual target reachability failure is proven, or prove that native rsync is available before falling back, then add redacted real Windows + Hyper-V + WSL2 proof before merge.

## Solution Assessment

No. The reachability-check direction is narrow and plausible, but this implementation should distinguish probe-helper failure from confirmed TCP unreachable before changing existing WSL-only setups to native rsync.
