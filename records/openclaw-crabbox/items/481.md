## Summary

Keep open: current main still has explicit egress cleanup only under `crabbox egress stop`, while ordinary lease stop/release paths only run Actions/Tailscale cleanup before releasing the lease. Coordinator socket closure is a fallback, not local daemon pid-file or remote helper cleanup, and no canonical issue or PR currently supersedes this bug.

## What This Changes

The issue requests that `crabbox stop` best-effort clean up mediated egress host daemons and lease-side egress clients before releasing a lease.

## Best Possible Solution

Add a provider-neutral best-effort egress cleanup helper in the CLI stop/release lifecycle, reuse the existing local daemon and remote `pkill` behavior, document it, and keep coordinator socket closure as a fallback rather than the primary cleanup mechanism.

## Reproduction Assessment

Yes at source level: start egress with `--daemon`, then inspect the ordinary stop/release paths and they do not call the existing explicit local daemon or remote egress-client cleanup helpers. I did not run a live lease reproduction in this read-only review.

## Solution Assessment

Yes for `crabbox stop`: reusing the existing egress cleanup pieces before release is the narrowest maintainable fix. Remote expiry may also need daemon-side stale-state handling, but that should complement rather than replace explicit stop cleanup.
