## Summary

Keep open: current main still has the raw public `/v1/internal/*` prefix check before the shared router collapses empty path segments, so the reported route-boundary bypass remains source-reproducible.

## What This Changes

Authenticated non-admin callers can use empty path segments to bypass the public internal-route block and reach scheduled maintenance with a caller-supplied internal header.

## Best Possible Solution

Fix the Worker public entrypoint so normalized or empty-segment internal routes cannot reach Fleet, strip external internal headers, and add regression tests for double-slash scheduled-maintenance requests.

## Reproduction Assessment

Yes, source-reproducible: current main still has the raw prefix check and normalized Fleet dispatch, and a URL/path split check confirms double-slash paths evade the prefix while normalizing to the internal route. I did not run the full Worker request harness during this read-only pass.

## Solution Assessment

Yes, the narrow maintainable fix is to align the public security check with normalized routing and prevent external callers from supplying internal-only headers. Replacing the header with a non-forgeable internal capability would be a stronger follow-up if maintainers want to broaden the fix.
