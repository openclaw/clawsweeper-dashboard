Full review comments:

- **[P2] Distinguish OpaqueRefs from UUIDs during rollback:** `internal/providers/xcpng/client.go:892-895`
  - body: `CloneVM` rollback calls `c.deleteServer(..., ref, true)` with the ref returned by `VM.clone` or `VM.copy`, but `vmRefForID` treats any long hyphenated string as a UUID before accepting it as a ref. Real XAPI refs can be UUID-shaped `OpaqueRef:` values, so rollback can call `VM.get_by_uuid` with the ref, fail, and leave the failed clone or disk behind because those rollback errors are ignored. Please distinguish OpaqueRefs from UUIDs and add a rollback test with a UUID-shaped OpaqueRef.
  - confidence: 0.82
