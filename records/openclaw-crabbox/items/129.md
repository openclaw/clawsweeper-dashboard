## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Wire crew through coordinator leases:** `worker/src/config.ts:203`
  - body: `leaseConfig` can only label brokered provider resources when `input.crew` is present, but the CLI `CreateLease` request never sends a `crew` field and the Worker lease record does not persist one. A coordinator-backed `crabbox warmup --crew alpha` therefore creates leases without the advertised crew metadata, so `list --crew` and fallback lease views cannot work on the main managed-provider path.
  - confidence: 0.92
- **[P1] Preserve coordinator fallback rows when filtering:** `internal/cli/pool.go:75`
  - body: This filter drops every coordinator fallback lease because `coordinatorLeasesToServers` builds `Server` labels from `CoordinatorLease` fields that do not include `crew`. Users without an admin token can get an empty `crabbox list --crew <name>` even when matching leases exist, so the fallback path needs crew-aware lease records or should avoid applying a lossy label-only filter.
  - confidence: 0.9
- **[P2] Filter typed JSON list payloads:** `internal/cli/pool.go:93`
  - body: The JSON filter only handles `[]any`, but coordinator JSON list returns typed slices such as `[]CoordinatorMachine` or fallback `[]CoordinatorLease`. For those valid payloads, `--json --crew <name>` becomes a no-op and can return unfiltered leases.
  - confidence: 0.87
- **[P3] Use the .cbx hostname suffix in the changelog:** `CHANGELOG.md:18`
  - body: The changelog says crew peers resolve as `<slug>.box`, while the implementation, docs, and PR body use `<slug>.cbx` to avoid the real `.box` TLD. This release note should match the shipped hostname suffix.
  - confidence: 0.98
