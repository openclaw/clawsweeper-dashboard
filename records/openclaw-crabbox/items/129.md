## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Defer ACL bootstrap until validation finishes:** `internal/cli/lease_flags.go:115`
  - body: `maybeBootstrapPondACL` can mutate the operator's tailnet policy before later local validation runs. For example, with `TS_API_KEY` and a Tailscale auth key present, an invalid `--expose` value or invalid duration can still install the pond ACL row before the command fails, leaving a policy side effect from a rejected lease request. Move bootstrap after all local validation or prevalidate everything that can reject the command before calling it.
  - confidence: 0.9
- **[P2] Keep healthy providers in pond peer fan-out:** `internal/cli/pond_bridge.go:326`
  - body: When resolving all providers for a pond, this returns immediately on the first provider error. That means one misconfigured or unsupported provider can make `crabbox pond peers --pond ...` fail and hide healthy peers from other providers, which breaks the cross-provider discovery behavior this command is adding. Accumulate successful provider results and only fail the whole request when no provider can be resolved, or surface per-provider warnings without dropping the rest.
  - confidence: 0.88
