- **[P1] Persist pond in common claim writes:** `internal/cli/claim.go:59-69`
  - body: The common claim wrappers still call the pond-aware helper with an empty pond, while `pond peers`, `pond connect`, and `pond release` first filter local claim sidecars by `claim.Pond`.   - confidence: 0.92
- **[P1] Return failed SSH forwards from pond connect:** `internal/cli/pond_mesh.go:621-624`
  - body: `runPondMeshForwards` cancels the context when any tunnel exits, then only returns `firstErr` if `ctx.Err() == nil`; after `<-ctx.Done()` that condition is never true. A failed `ssh -L` process can therefore make `crabbox pond connect` exit successfully even though the advertised loopback forward is dead.
  - confidence: 0.95
- **[P2] Keep --export stdout shell-safe:** `internal/cli/pond_mesh.go:303-317`
  - body: The `--export` path writes export lines, then falls through to status text, duplicate exports, and file-path messages on stdout. That breaks the shell-export contract for callers that consume stdout, so status should move to stderr or be suppressed while stdout contains only valid export statements.
  - confidence: 0.86
