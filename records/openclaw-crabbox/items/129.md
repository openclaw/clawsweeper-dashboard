Full review comments:

- **[P1] Clean up export daemons on startup failure:** `internal/cli/pond_mesh.go:334-335`
  - body: The `--export` path detaches each `ssh -L` as it starts, but returns immediately on a later `Start()` failure or immediate `Wait()` error without stopping handles already launched. A multi-forward pond can leave stale loopback tunnels running even though the command reports failure.
  - confidence: 0.92
- **[P2] Use the HuJSON standardizer for doctor ACL checks:** `internal/cli/doctor_pond.go:124`
  - body: `normalizeHuJSON` only removes trailing commas when the comma is directly followed by `]` or `}`, so normal multiline HuJSON policy rows still fail `json.Unmarshal` and `doctor --pond` can report a missing ACL row when it is present. Reuse `hujsonStandardize` or the same whitespace-aware scanner here.
  - confidence: 0.86
