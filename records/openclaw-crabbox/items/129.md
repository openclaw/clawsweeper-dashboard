The live proof is useful, but this needs repair plus maintainer approval of the new network/security boundary.

## What This Changes

The branch adds preview pond grouping with `--pond`/`--expose`, `crabbox pond peers/connect/release`, Tailscale ACL checks, URL bridge adapters, SSH-mesh forwarding, Worker lease metadata, docs, and tests.

## Best Possible Solution

Land a narrower repaired pond preview where endpoint and exposed-port metadata round-trip through every documented production path, then have maintainers explicitly approve the reserved-label and Tailscale ACL boundary.

Yes for the blocking findings by source inspection: the PR reads claim endpoint fields that the production claim writer never assigns, and it sends exposed ports to the coordinator without adding them to persisted/echoed lease records. I did not run live leases in this read-only review.
