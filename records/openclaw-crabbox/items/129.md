Full review comments:

- **[P1] Use the robust HuJSON normalizer for ACL doctor checks:** `internal/cli/doctor_pond.go:99`
  - body: `normalizeHuJSON` only replaces literal `,]` and `,}`, so a valid HuJSON ACL section formatted with `,\n]` or `,\n}` makes `json.Unmarshal` fail and `doctor --pond` can report a missing rule even when the self-grant exists. Reuse the string-aware standardizer from the ACL merge path and cover multiline trailing commas.
  - confidence: 0.94
- **[P2] Render pending peers in the pond reachability matrix:** `internal/cli/pond_bridge_doctor.go:245-249`
  - body: Pending peers are counted under `pending`, but `observedTransports` and the renderer only iterate `tailnet`, `url`, `ssh`, and `none`. Ponds with provisioning or endpoint-missing members can show fewer breakdown entries and rows than actual members, making doctor diagnostics incomplete.
  - confidence: 0.9
- **[P3] Canonicalize provider aliases before filtering claims:** `internal/cli/pond_bridge.go:515-521`
  - body: `--provider` accepts registered aliases elsewhere, but this filter compares the raw flag to canonical claim provider names. Alias inputs like `exedev` or `docker` can hide valid pond members, so resolve the flag through `ProviderFor` before filtering.
  - confidence: 0.82
