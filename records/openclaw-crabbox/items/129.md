## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use the matched claims when collecting SSH members:** `internal/cli/pond_mesh.go:408`
  - body: `collectPondMembersAcrossProviders` narrows to pond claims, but then throws that membership set away and filters each backend's `List` output by `server.Labels["pond"]`. Several advertised SSH-mesh providers still claim through helpers that do not persist `Pond`, and their list views synthesize labels from the current config rather than provider state, so `pond connect` can miss leases created with `--pond` or include the wrong provider leases if the config default is set. Carry pond into those provider claims and join/resolve against the matched claim lease IDs instead of trusting synthetic list labels.
  - confidence: 0.93
- **[P1] Verify export tunnels before reporting them ready:** `internal/cli/pond_mesh.go:337`
  - body: In `--export` mode, `handle.Start()` only proves the ssh process forked. OpenSSH can still fail authentication or fail the local `-L` bind after this line, and `sshBaseArgs` does not set `ExitOnForwardFailure`, so the command can print exports and return 0 while the advertised loopback endpoint is dead. Add a readiness/failure handshake, such as `ExitOnForwardFailure` plus a short wait or local probe, before printing daemon success.
  - confidence: 0.87
