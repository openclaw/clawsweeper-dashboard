Full review comments:

- **[P1] Strip paths for pond connect and release:** `internal/cli/cli_kong.go:516-520`
  - body: `pondConnectKongCmd` and `pondReleaseKongCmd` are passthrough commands but still pass `c.Args` directly, unlike `pond peers` which strips the Kong path. If passthrough args include `pond connect` or `pond release`, the handlers parse the command path as the pond name, so `crabbox pond connect demo` or `crabbox pond release demo` can operate on the wrong pond.
  - confidence: 0.9
- **[P1] Start export tunnels before exiting:** `internal/cli/pond_mesh.go:313-314`
  - body: The `--export` path starts `runPondMeshForwards` only as an in-process goroutine and then returns. When the Go process exits, that goroutine is not a daemon and may never reliably start or supervise the SSH commands, leaving `eval $(crabbox pond connect demo --export)` with exports for dead loopback ports.
  - confidence: 0.92
- **[P2] Standardize HuJSON before checking ACL rules:** `internal/cli/doctor_pond.go:119-120`
  - body: The live doctor client requests HuJSON, but these rule checks unmarshal the raw `acls`/`grants` sections with `encoding/json`. A real policy with comments or trailing commas around an existing self-peer rule will be treated as missing, so `doctor --pond` can fail even after the ACL row is present.
  - confidence: 0.82
