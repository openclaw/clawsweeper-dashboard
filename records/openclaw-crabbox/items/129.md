## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Strip the Kong path for pond subcommands:** `internal/cli/cli_kong.go:517-520`
  - body: `pond peers` strips the passthrough command path before dispatch, but `pond connect` and `pond release` pass `c.Args` directly. With Kong passthrough args that still include `pond connect` or `pond release`, `pondConnect` parses the pond name as `pond` and `pondRelease` normalizes the whole command path, so the advertised commands operate on the wrong pond or no pond.
  - confidence: 0.86
- **[P1] Start export tunnels before returning:** `internal/cli/pond_mesh.go:313-314`
  - body: In `--export` mode the command prints exports, starts `runPondMeshForwards` in a goroutine, and returns immediately. The Go process can exit before the goroutine starts the ssh commands or reports `Start` failures, so `eval $(crabbox pond connect demo --export)` can publish loopback ports that are not actually forwarded.
  - confidence: 0.88
- **[P1] Persist pond labels before filtering claims:** `internal/cli/pond_bridge.go:518-519`
  - body: Discovery starts by dropping every claim whose `claim.Pond` is empty, but several delegated providers newly advertised as bridge participants still write claims through legacy helpers that never set that field. Those leases never reach the provider bridge adapter, so Modal/Cloudflare/Tensorlake pond members disappear instead of showing the documented unsupported URL-bridge state.
  - confidence: 0.8
