Full review comments:

- **[P1] Implement pond release before documenting lifecycle cleanup:** `docs/features/pond.md:16`
  - body: The docs and PR smoke show `crabbox release --pond NAME`, but the stop/release implementation still only accepts one positional lease/server id and registers no `--pond` flag. Users following the documented lifecycle command will fail cleanup and can leave every pond lease running.
  - confidence: 0.96
- **[P1] Stop claiming Modal peers are reachable as .cbx hosts:** `docs/features/pond.md:44-46`
  - body: This example says a Hetzner API box can use `ML_HOST=ml.cbx` for a Modal peer, but the PR's Modal adapter returns `ErrBridgeNotImplemented` and the Tailscale hosts refresh only discovers tailnet peers. Either implement Modal reachability or change the example/proof to supported providers.
  - confidence: 0.93
- **[P1] Keep tunnels alive for exported SSH-mesh ports:** `internal/cli/pond_mesh.go:294-298`
  - body: `--export` prints variables and returns before `runPondMeshForwards`, so `eval $(crabbox pond connect ... --export)` creates environment variables for local ports where no `ssh -L` processes are listening. The export flow needs a live tunnel strategy or the docs should require a separate foreground connect process.
  - confidence: 0.91
- **[P2] Return SSH forward failures from pond connect:** `internal/cli/pond_mesh.go:543-545`
  - body: When any spawned SSH process exits with an error, the goroutine stores `firstErr` and calls `cancel()`, but the function later checks `ctx.Err() == nil` after waiting on `ctx.Done()`. That condition is never true after this cancellation path, so failed SSH tunnels can make `pond connect` exit successfully.
  - confidence: 0.88
- **[P2] Standardize HuJSON before checking ACL rows:** `internal/cli/doctor_pond.go:119-120`
  - body: The live doctor client requests `application/hujson`, but `pondACLSelfPeerRule` and the grants counterpart unmarshal the raw section with `encoding/json`. Real policies with comments or trailing commas inside `acls`/`grants` will be reported as missing even when the row exists, and bootstrap can append duplicates after the same false negative.
  - confidence: 0.84
