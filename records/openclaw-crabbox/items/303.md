Full review comments:

- **[P1] Reuse configured nsc context for lifecycle commands:** `internal/providers/namespaceinstance/backend.go:265-273`
  - body: `createInstance` appends the configured region/endpoint/keychain, but `describe`, `list`, `destroy`, and `extend` do not. A lease created with `namespaceInstance.endpoint`, `keychain`, or `region` can then be invisible to status/stop/cleanup/touch because those commands hit the default `nsc` context. Please thread the common args through every lifecycle command and cover those paths with fake-runner tests.
  - confidence: 0.92
- **[P2] Validate namespace-instance work roots:** `internal/providers/namespaceinstance/flags.go:65-68`
  - body: The new docs say broad roots such as `/`, `/tmp`, and `/workspaces` are rejected, and the existing Namespace provider enforces that safety check. Here `--namespace-instance-work-root` is accepted and `ApplyProviderFlags` returns nil without validation, while `Configure` also accepts file/env values, so the provider can sync into unsafe broad remote roots. Add the matching validation and regression coverage.
  - confidence: 0.86
