## What This Changes

The issue requests isolating lease-controlled Code portal content from the authenticated coordinator origin.

## Best Possible Solution

Add a maintainer-approved untrusted-origin boundary for Code bridge content, preserving existing portal URLs and Code/WebSocket behavior through a compatible redirect or routing design with focused security and live bridge proof.

## Reproduction Assessment

Yes for source-level reproduction: current main still authenticates `/portal` with `crabbox_session`, proxies Code bridge responses under `/portal/leases/{id}/code/...`, and permits same-origin script execution in the Code CSP. I did not run a live exploit against a coordinator and bridge.

## Solution Assessment

Yes on direction, but the exact implementation still needs maintainer design. Origin isolation is the durable solution; a blanket CSP or cookie-only change would not satisfy the accepted compatibility requirements.
