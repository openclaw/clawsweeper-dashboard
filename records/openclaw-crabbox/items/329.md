## Best Possible Solution

Land a narrow version that rejects Windows PATH shim tools before choosing WSL rsync, preserves the native MSYS2/Cygwin fallback, and keeps focused neutral fixture tests.

## Reproduction Assessment

No high-confidence local reproduction was run from this Linux checkout. Source inspection shows current main chooses WSL whenever wsl is present, and the PR body describes the Windows Docker Desktop/local-container failure and after-fix smoke path.

## Solution Assessment

No. The proposed direction is right, but the WSL probe must distinguish native WSL binaries from Windows PATH shims before this is the narrow maintainable fix.
