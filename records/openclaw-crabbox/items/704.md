- `impact:security`: The issue concerns coordinator authorization and cloud provider resource boundaries for brokered AWS leases.

## Summary

Keep open: current main and latest release still preserve non-admin AWS selector fields and pass them to AWS API calls under broker authority, and no merged or viable open PR appears to own the AWS-specific fix.

## What This Changes

Issue reports that non-admin broker lease requests can set AWS resource selector fields consumed by the coordinator's AWS principal.

## Best Possible Solution

Land a security-reviewed coordinator policy that rejects non-admin AWS selector overrides unless they match broker-approved defaults, promoted images, or an explicit operator allowlist, with regression tests and broker docs updated.

## Reproduction Assessment

Yes. Source inspection shows a crafted non-admin POST /v1/leases can carry AWS selector fields through leaseConfig into AWS provider calls; no live AWS exploit was run.

## Solution Assessment

Yes on the fix boundary, unclear on the exact compatibility policy. The narrow maintainable direction is a coordinator-side authorization gate before provider calls, while maintainers choose admin-only, default-match, or allowlist semantics.
