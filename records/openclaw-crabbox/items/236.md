## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Honor the oc config API URL before the built-in default:** `internal/providers/opencomputer/apiclient.go:79-83`
  - body: `baseConfig` always seeds `OpenComputer.APIURL`, so this expression never falls through to `fileCfg.APIURL` during normal CLI loading. A user whose `~/.oc/config.json` points at a non-default API would have that config-file API key sent to `https://app.opencomputer.dev` instead of the configured host; preserve Crabbox/env/flag overrides, but use the oc config URL before the built-in default when the user has not set a Crabbox URL.
  - confidence: 0.9
- **[P2] Return API errors from status lookups:** `internal/providers/opencomputer/backend.go:238-256`
  - body: `api.getSandbox` can fail with 404, auth, or network errors, but `Status` folds that into an empty view and returns nil on ordinary non-wait status requests. Handle `getErr` before returning the view, and avoid making `--wait` spin until timeout on persistent API failures.
  - confidence: 0.88
