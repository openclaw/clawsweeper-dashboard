## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Keep forwarded secrets out of argv:** `internal/providers/opencomputer/cli.go:185`
  - body: `uploadEnvProfile` sends allowed env values through `writeRemoteFile`, which builds an `oc exec ... bash -lc "echo '<base64>' | base64 -d > ..."` command. Base64 is reversible and still appears in process argv wherever the local or remote command is visible, so this breaks the promised secret boundary. Please disable OpenComputer env forwarding or use a transport that never places secret material in argv.
  - confidence: 0.95
- **[P2] Advertise archive sync before relying on force-sync:** `internal/providers/opencomputer/provider.go:24`
  - body: Core rejects `--force-sync-large` for delegated providers unless `FeatureArchiveSync` is advertised. This spec only includes `FeatureURLBridge`, while the backend/docs expect large archive sync to be overrideable, so large OpenComputer runs fail before the backend can honor the flag.
  - confidence: 0.9
