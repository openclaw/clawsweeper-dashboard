## Summary

Keep open. Current main and v0.31.0 still make Proxmox cloud-image SHA256 verification optional, and no open or merged canonical PR appears to own this exact security hardening.

## What This Changes

Require the Proxmox template build helper and docs to make cloud-image SHA256 verification mandatory by default.

## Best Possible Solution

Fail closed when no expected SHA256 is provided, document a pinned image and digest update process, and add regression coverage for missing, wrong, and correct hash paths.

## Reproduction Assessment

Yes, by source inspection: current main defaults the image hash to empty and only runs `sha256sum -c -` when the variable is non-empty before converting and importing the image. I did not execute the helper because this review is read-only and the script is designed to mutate a Proxmox host.

## Solution Assessment

Unclear pending maintainer policy: requiring a digest by default is the narrow security direction, but it intentionally changes the documented setup flow. The safer implementation should pair the fail-closed script behavior with docs and tests for the upgrade path.
