## Summary

This issue should stay open: current main still models `status` as a string and only reads top-level `networks`, so the reported Apple `container` 1.0.0 JSON shape remains unsupported. I found no duplicate issue or open fix PR that already owns the work.

## What This Changes

The issue asks Crabbox's existing `apple-container` provider to parse Apple `container` 1.0.0 inspect/list JSON where lifecycle state is under `status.state` and IPs may be under `status.networks`.

## Best Possible Solution

Add a provider-local compatibility decoder/helper for both JSON shapes, update status/IP helpers and docs as needed, and cover inspect/list parsing with focused regression tests without changing the CLI-backed provider architecture.

## Reproduction Assessment

Yes, there is a high-confidence source reproduction path: current main unmarshals `status` into a Go string, so the supplied object-status JSON fails before IP discovery. I did not execute the Apple silicon smoke in this read-only review.

## Solution Assessment

Yes, the proposed provider-local compatibility fix is the narrowest maintainable path. It stays within `internal/providers/applecontainer` and avoids a new SDK integration or cross-provider behavior change.
