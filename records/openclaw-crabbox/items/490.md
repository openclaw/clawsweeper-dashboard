## Summary

Current main and the latest release still match the reported Cloudflare delegated-run redirect exposure, so this should stay open for maintainer security handling rather than cleanup closure.

## What This Changes

The issue reports that Cloudflare delegated-run HTTP requests can replay command and upload bodies to a different origin after 307/308 redirects because the provider client lacks a same-origin redirect guard.

## Best Possible Solution

Add a Cloudflare provider same-origin redirect guard around the selected HTTP client, matching adjacent credentialed provider patterns, with regression tests for exec-stream, upload, and same-origin redirect behavior.

## Reproduction Assessment

Yes, from source inspection: Cloudflare exec and upload requests use replayable bodies through a client with no same-origin redirect guard. I did not rerun the temporary PoC because this review is read-only and avoids artifact-producing tests in the checkout.

## Solution Assessment

Yes, the reported remediation is the narrow maintainable fix: clone the chosen client, reject redirects to a different scheme/host/effective port, and preserve same-origin redirect behavior plus any caller policy.
