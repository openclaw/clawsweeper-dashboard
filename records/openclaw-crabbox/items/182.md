## What This Changes

Track adoption of `RunSessionHandle` and `FeatureRunSession` across reusable Crabbox providers so orchestrators can acquire provider-neutral kept-run session handles.

## Best Possible Solution

Keep the existing provider-neutral contract, review and land the Islo PR after real proof, then add run-session support provider-by-provider only where reuse and cleanup semantics are clear.

## Reproduction Assessment

## Solution Assessment
