## Summary

Keep open: current main has implemented the run-session contract for many provider slices, but the tracker is not fully resolved because `cloudflare-sandbox` is now a delegated-run provider with lease/reuse/keep/cleanup behavior and still does not advertise `FeatureRunSession` or return `RunResult.Session`.

## What This Changes

Track provider-by-provider adoption of Crabbox `RunSessionHandle` / `FeatureRunSession` support for reusable delegated-run leases.

## Best Possible Solution

Keep the tracker open or split a Cloudflare Sandbox-specific follow-up, and advertise `FeatureRunSession` only when each provider has a validated session handle, cleanup command, conformance coverage, docs, and real provider proof.

## Reproduction Assessment

Not applicable: this is a provider-capability adoption tracker rather than a broken existing behavior report. The source-level check is high confidence that at least one current delegated provider still lacks the advertised run-session contract.

## Solution Assessment

No, closing as implemented is not the best outcome yet because current main still leaves a meaningful provider-scope gap. The maintainable path is provider-by-provider adoption or an explicit maintainer decision that a provider should remain unsupported.
