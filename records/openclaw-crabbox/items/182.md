## Summary

Keep open: current main has the generic run-session contract and several provider implementations, including the merged Islo work, but provider-by-provider adoption is still incomplete and the latest discussion explicitly narrows this tracker to remaining providers with unproven reusable lifecycle semantics.

## What This Changes

Track provider-by-provider adoption of Crabbox run-session handles so external orchestrators can acquire reusable leases through a provider-neutral contract.

## Best Possible Solution

Keep this as the canonical umbrella until each eligible provider is audited in small provider-owned PRs with real behavior proof, tests, and docs for the run-session lifecycle.

## Reproduction Assessment

Not applicable: this is a feature/provider capability tracker rather than a bug. Source inspection gives a high-confidence current-main check that several delegated providers still do not advertise FeatureRunSession and e2b is still tested as unsupported for --lease-output.

## Solution Assessment

Yes, the current direction is the narrow maintainable path: keep the provider-neutral contract in core and add FeatureRunSession only inside provider adapters after lifecycle and cleanup semantics are proven. Closing this now would lose the remaining per-provider product decision work.
