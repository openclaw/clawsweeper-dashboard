## Summary

Keep open: current main has the generic run-session contract and some provider implementations, but the tracker’s central work is still incomplete and the latest discussion plus open Cloudflare PR show provider-by-provider adoption is active remaining work.

## What This Changes

Track adoption of `RunSessionHandle` / `FeatureRunSession` across reusable Crabbox providers so provider-neutral orchestrators can acquire kept leases and cleanup handles.

## Best Possible Solution

Keep this as the canonical umbrella while maintainers review small provider-local PRs with tests and redacted real behavior proof, excluding providers like railway unless their ownership model changes.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/provider capability tracker. Source inspection gives a high-confidence current-main check that several delegated providers still do not advertise `FeatureRunSession` and e2b is still tested as unsupported for `--lease-output`.

## Solution Assessment

Yes, the current direction is the narrow maintainable path: keep core provider-neutral and add `FeatureRunSession` only inside provider adapters after lifecycle and cleanup semantics are proven. The open Cloudflare PR should be reviewed as one slice, not used to close the umbrella.
