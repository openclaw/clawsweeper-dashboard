## Summary

Keep this issue open: current main and the latest release still return Cloudflare runner JSON/text error bodies without redacting the configured runner bearer token. The merged redirect fix at https://github.com/openclaw/crabbox/pull/505 is adjacent transport hardening, but it does not address reflected error-body secrets.

## What This Changes

The issue requests redaction of Cloudflare runner bearer tokens from CLI-visible Cloudflare provider API errors.

## Best Possible Solution

Add Cloudflare-container error redaction for the configured runner token and bearer-shaped Authorization text, with focused tests for JSON error and plain-text response bodies while preserving useful status context.

## Reproduction Assessment

Yes. Source inspection shows Cloudflare requests set the runner bearer token and both responseError branches return remote error text without redaction; I did not run the temporary probe because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable fix is to redact the configured token and generic bearer-token shapes before formatting Cloudflare provider API errors, matching adjacent provider patterns.
