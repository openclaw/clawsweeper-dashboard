## What This Changes

The issue requests a first-party `vast` direct Linux SSH-lease provider so Crabbox can run GPU workloads on Vast.ai instances through the normal SSH data plane.

## Best Possible Solution

Keep the issue open for maintainer product/API acceptance; if accepted, add a focused `internal/providers/vast` SSH-lease adapter with no API-key flag, provider-local REST client, docs, generated provider metadata, and tests for auth, offer selection, SSH resolution, list/status, release, cleanup, and recovery.
