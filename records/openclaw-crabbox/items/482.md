## What This Changes

The issue asks Crabbox desktop paste/type and POSIX launch/proof helpers to stop reporting success after unverified clipboard delivery or detached GUI-command failure.

## Best Possible Solution

Make desktop helper success contingent on verified clipboard delivery or terminal/app visibility, with focused generated-command tests and a live Linux desktop smoke when available.

## Reproduction Assessment

Yes by source inspection: the generated paste script ignores clipboard-helper failure, and proof/terminal paths proceed after detached launch success without a liveness or visibility gate. I did not execute a live desktop lease during this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to propagate clipboard-helper failure and add a targeted terminal/proof launch liveness or visibility gate before success output and proof artifacts. This should reuse the existing browser-check precedent without adding new provider or config surface.
