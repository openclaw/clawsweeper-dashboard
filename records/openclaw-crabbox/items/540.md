## Summary

Keep open: the security direction and live proof are strong, but the PR still computes non-admin host visibility from the paginated portal lease list, so a visible attached Mac host can become a false 404 once it falls outside the first page.

## What This Changes

The PR requires admin auth for explicit brokered host pinning, filters portal Dedicated Host visibility for non-admin users, updates regression tests, and revises host-pinning documentation.

## Best Possible Solution

Keep the admin-only host-pinning gate, but compute portal host visibility from an unpaginated caller-visible active macOS lease set and add a regression test for an attached host outside the first page before merge.

## Reproduction Assessment

Yes. Current main’s source path accepts non-admin hostId lease requests and passes full portal Mac host inventory to portal rendering, and the PR body includes live Wrangler output showing the new 403 gate.

## Solution Assessment

No, not quite yet. The admin gate is the right narrow security fix, but the portal host visibility path should use an unpaginated caller-visible lease set instead of the paginated portal list helper.
