## Summary

This PR should stay open: the main admin-only host-pinning gate is well scoped and proven, but the current branch still has a source-confirmed portal authorization regression where non-admin host visibility is derived from the paginated portal lease list.

## What This Changes

The PR requires admin authentication for explicit brokered AWS Mac host pinning, filters non-admin portal Dedicated Host visibility to hosts attached to visible active macOS leases, and updates related tests and docs.

## Best Possible Solution

Keep the admin-only host-pinning gate, fix host visibility to use an unpaginated caller-visible active macOS lease set, and have maintainers explicitly accept the fail-closed compatibility change before merge.

## Reproduction Assessment

Yes from source: on the PR head, non-admin host visibility is computed from portalLeases(request), and that path is paginated before the new host filter sees it. A visible active macOS lease outside the first 100 portal leases can therefore lose host detail/VNC authorization.

## Solution Assessment

Not yet. The admin gate is the right narrow security fix, but the portal host visibility path should use an unpaginated caller-visible lease set instead of the paginated portal list helper.
