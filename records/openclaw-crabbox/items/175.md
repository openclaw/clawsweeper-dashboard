## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Scope provider secrets before installing clients:** `.github/workflows/live-provider-e2e.yml:121-123`
  - body: The job exports the full provider secret surface to every matrix leg before installing the Modal or Tensorlake Python clients. Those installs run unpinned third-party code, so a Modal/Tensorlake smoke can read unrelated AWS, GCP, broker, SSH, and other provider credentials. Keep provider credentials out of the job env and inject only the selected provider's secrets after any client installation, or use pinned/preinstalled clients.
  - confidence: 0.89
- **[P1] Build the matrix from registered providers:** `.github/workflows/live-provider-e2e.yml:52-76`
  - body: The workflow hard-codes the same provider inventory that current main already exposes through the registered provider matrix and `crabbox providers --json`. The next built-in provider will make `providers=all` silently incomplete unless two lists are kept in sync. Generate the list from the CLI/registry and keep this workflow responsible only for filtering aliases/user input.
  - confidence: 0.82
