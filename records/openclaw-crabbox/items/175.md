Full review comments:

- **[P2] Require the Azure Dynamic Sessions auth path actually used:** `scripts/live-provider-e2e.sh:266-269`
  - body: For `azure-dynamic-sessions`, this preflight treats service-principal env vars as sufficient, and the workflow only exports those plus endpoint/pool. Current main's provider reads `CRABBOX_AZURE_DYNAMIC_SESSIONS_TOKEN` or shells out to an already-authenticated `az account get-access-token`; it does not log in from `AZURE_CLIENT_SECRET`, so a configured matrix leg can pass preflight and then fail during client creation. Please pass a supported token secret or add a trusted Azure login/token step and make the preflight match that path.
  - confidence: 0.9
- **[P2] Guard tool installation for non-Ubuntu runner labels:** `.github/workflows/live-provider-e2e.yml:123-128`
  - body: `runner_label` is documented as the escape hatch for macOS/self-hosted providers, but every provider job runs `sudo apt-get update/install` before the provider smoke script. A macOS runner for `apple-container` or a non-Debian self-hosted runner for local-tool providers will fail here before preflight can skip or run the provider. Please make this install conditional on a Linux/apt runner, or split/document the workflow so non-Ubuntu runner labels do not hit this step.
  - confidence: 0.86
