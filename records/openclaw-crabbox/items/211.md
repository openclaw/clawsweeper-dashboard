## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Validate WASI guest roots before host writes:** `internal/providers/wasi/backend.go:426`
  - body: `copyManifestToGuest` creates the workdir root with `os.MkdirAll(target)` before any no-follow validation of that root. If a kept guest workdir root is already a symlink, the later child checks treat that symlink as trusted and can copy files through it, so validate or recreate the guest root/workdir root safely before any mkdir/copy/mount path.
  - confidence: 0.92
- **[P1] Gate the WASI live smoke behind provider selection:** `scripts/live-smoke.sh:460`
  - body: Every other provider smoke in this script is guarded by `has_provider`, but `wasi_smoke` runs unconditionally. That makes existing `CRABBOX_LIVE_PROVIDERS=aws` or `hetzner` jobs run extra WASI sync/warmup/status/stop work and fail on unrelated local-provider behavior, so wrap it in `if has_provider wasi; then ... fi`.
  - confidence: 0.98
- **[P3] Use the same slug in the warmup example:** `docs/providers/wasi.md:40-41`
  - body: The docs warm up a WASI session without naming a slug, then immediately run with `--id my-wasi`; following that example will not find the generated lease. Add `--slug my-wasi` to the warmup command or use the printed slug in the run command.
  - confidence: 0.86
