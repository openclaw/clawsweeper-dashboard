## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Reject a symlinked WASI workdir before copying:** `internal/providers/wasi/backend.go:426-427`
  - body: A kept guest workdir can itself be replaced with a symlink, and `os.MkdirAll(target)` will accept a symlinked directory before the later no-follow checks begin below `target`. That lets resync writes and `/work` mounting follow the workdir symlink outside the intended guest root; lstat and reject the workdir root before copying or mounting.
  - confidence: 0.78
- **[P1] Gate the WASI live smoke behind provider selection:** `scripts/live-smoke.sh:460`
  - body: `scripts/live-smoke.sh` uses `CRABBOX_LIVE_PROVIDERS` to narrow which provider smokes run, but this new call runs WASI every time. That makes an `aws`, `hetzner`, or other selected smoke run an extra provider unexpectedly; wrap it in `if has_provider wasi; then wasi_smoke; fi` while preserving the current KubeVirt/external blocks during conflict resolution.
  - confidence: 0.95
- **[P2] Return proof output before advertising run-proof support:** `internal/providers/wasi/backend.go:205-214`
  - body: The provider declares `FeatureRunProof`, but the `RunResult` built here never sets `LogExcerpt`; current `writeDelegatedRunProof` therefore writes `(no console output captured)` even when the WASI command printed the live result. Capture a bounded stdout/stderr excerpt or remove `FeatureRunProof` and the docs claim until proof output is real.
  - confidence: 0.92
