## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Validate WASI guest roots with no-follow checks:** `internal/providers/wasi/backend.go:358`
  - body: This provider treats kept guest directories as part of a sandbox boundary, but `prepareGuest` creates `guestRoot` with `os.MkdirAll` before checking whether an existing path is a symlink. If a kept guest root or workdir root is replaced with a symlink before the next sync/no-sync/run, later copy and mount paths can follow it outside the intended `crabbox-wasi-*` tree. Add explicit `Lstat`/no-follow validation for the guest root and workdir root before creating, syncing, or mounting them.
  - confidence: 0.86
- **[P2] Honor sync.include when building the WASI manifest:** `internal/providers/wasi/backend.go:372`
  - body: Current main supports `sync.include` as a whitelist for provider sync, but the WASI path calls `syncManifest(repoRoot, excludes)` and its wrapper calls `BuildSyncManifest` without `cfg.Sync.Includes`. A user who configures an include whitelist would still copy the full non-excluded checkout into `/work` for this provider. Thread `cfg.Sync.Includes` into the manifest builder and add a regression test.
  - confidence: 0.9
