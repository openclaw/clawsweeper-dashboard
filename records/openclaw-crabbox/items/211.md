## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Reject symlinked WASI guest roots before writes:** `internal/providers/wasi/backend.go:358`
  - body: `os.MkdirAll(guestRoot, ...)` follows an existing symlink before the provider has validated the guest root. With a kept/configured guest base, that lets later sync and cleanup operate under a host path outside the intended WASI guest tree, so the root itself needs an Lstat/no-follow check before any create, remove, or copy step.
  - confidence: 0.9
- **[P1] Reject symlinked WASI workdir roots before copying:** `internal/providers/wasi/backend.go:426`
  - body: `copyManifestToGuest` calls `os.MkdirAll(target, ...)` on the configured workdir root before `prepareGuestDestination` runs, and the later no-follow helper starts below that root. If a kept workdir root is a symlink, resync can write outside `/work`; validate the workdir root itself before this call and cover it with a regression test.
  - confidence: 0.9
- **[P2] Gate the WASI live smoke behind provider selection:** `scripts/live-smoke.sh:460`
  - body: The script calls `wasi_smoke` unconditionally, so `CRABBOX_LIVE_PROVIDERS=aws` or any other narrowed smoke now also runs the local WASI flow. Wrap this call in the same `if has_provider wasi; then ... fi` pattern used by the other provider smokes to avoid surprising existing automation.
  - confidence: 0.95
