## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Reject symlinked WASI workdir roots before copying:** `internal/providers/wasi/backend.go:426`
  - body: `target` is the guest workdir root, but `os.MkdirAll(target)` follows an existing symlink before any no-follow validation. In a kept guest workdir, that lets a symlinked workdir root redirect later sync writes and the `/work` mount outside the intended guest root. Validate the guest root/workdir root with `Lstat` before sync, no-sync, and runtime mount, and add a root-symlink regression test.
  - confidence: 0.93
- **[P2] Gate the WASI live smoke behind provider selection:** `scripts/live-smoke.sh:460`
  - body: `wasi_smoke` is called unconditionally, so `CRABBOX_LIVE_PROVIDERS=wandb` or the default `aws,hetzner` selection will still run the new WASI checks. That breaks the live-smoke provider filter and can fail unrelated live-provider jobs; wrap it in a `has_provider wasi`/alias guard like the other providers.
  - confidence: 0.98
- **[P2] Return proof output before advertising run-proof support:** `internal/providers/wasi/provider.go:27`
  - body: The provider advertises `FeatureRunProof`, so delegated `--emit-proof` is accepted, but the WASI backend never fills `RunResult.LogExcerpt` or proof artifacts. Successful module output will generate proof with `(no console output captured)`, which is misleading for a provider claiming run-proof support. Either tee bounded stdout/stderr into `LogExcerpt` with a test, or remove `FeatureRunProof` for now.
  - confidence: 0.9
