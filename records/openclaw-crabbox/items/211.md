Overall confidence: 0.92

Full review comments:

- **[P1] Map the synced guest workdir to `/work` for wasmtime:** `internal/providers/wasi/backend.go:729`
  - body: This builds the wasmtime invocation as `--dir /work::<guestWork>`, but wasmtime parses that form as `HOST_DIR::GUEST_DIR`. As a result `--wasi-runtime wasmtime` will not mount the synced checkout at guest `/work`; it will usually fail to open host `/work`, or, if host `/work` exists, preopen the wrong host tree under an unintended guest path. Please pass the host guest-work path first and `/work` second, then update the assertion and real/e2e proof for the wasmtime path.
  - confidence: 0.96
