## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Reject symlinked guest roots before syncing:** `internal/providers/wasi/backend.go:358`
  - body: `prepareGuest` accepts an existing `guestRoot` with `os.MkdirAll` before checking whether it is a symlink. If `crabbox-wasi-<leaseID>` already points to another directory, the subsequent workdir creation and manifest copy operate through that symlink, so a kept or locally controlled lease can escape the intended guest tree. Lstat/no-follow the guest root before any write path and cover it with a regression test.
  - confidence: 0.93
- **[P1] Reject symlinked workdir roots before copying:** `internal/providers/wasi/backend.go:426`
  - body: `copyManifestToGuest` calls `os.MkdirAll(target)` before validating `target` itself. When the kept `/work` directory is already a symlink to another directory, the later child-destination checks use that symlinked root and file copies can write outside the guest tree. Validate or create the workdir root one component at a time before copying, and add a direct regression for a symlinked workdir root.
  - confidence: 0.92
- **[P1] Gate the WASI live smoke behind provider selection:** `scripts/live-smoke.sh:460`
  - body: `wasi_smoke` is called unconditionally, so `CRABBOX_LIVE_PROVIDERS=aws` or any other narrowed live-smoke run still performs a local WASI warmup, sync-only run, status, and stop against `CRABBOX_LIVE_REPO`. That changes existing automation semantics and can fail on repos that did not opt into this provider; wrap it in `if has_provider wasi; then ... fi` like the other provider smokes.
  - confidence: 0.96
