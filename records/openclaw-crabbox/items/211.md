## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve excludes when copying symlinks:** `internal/providers/wasi/backend.go:437-438`
  - body: This follows any in-repo symlink and copies the resolved target bytes into `/work`. Because the manifest filters only the symlink path, a repo can include `fixtures/token -> .env` while `.env` is excluded by `.crabboxignore` or `sync.exclude`, and the WASI guest still receives the ignored file contents. Existing archive-based delegated sync preserves symlink entries instead of dereferencing them, so please preserve that behavior or reject symlinks unless the resolved target also passes the same exclude checks.
  - confidence: 0.88
