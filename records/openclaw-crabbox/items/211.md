## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Replace guest paths without following symlinks:** `internal/providers/wasi/backend.go:489-490`
  - body: `copyManifestToGuest` syncs into a kept, writable `/work` tree, but `copyFile` opens `dst` directly. If a previous run or earlier sync leaves a symlink at that path, `os.OpenFile` follows it and truncates the target instead of replacing the manifest entry. Lstat/remove the destination or write to a safe temp file and rename without following existing links before copying.
  - confidence: 0.9
- **[P2] Restrict the wasmtime child environment:** `internal/providers/wasi/backend.go:697-704`
  - body: `buildWasmtimeRunArgsAndEnv` starts from `os.Environ()` and passes that to the wasmtime process. The guest only receives the listed `--env` keys, but the external runtime still inherits every Crabbox secret/config env var, which is wider than the explicit-env boundary this provider advertises. Build the child env from a minimal baseline plus the allowed keys.
  - confidence: 0.83
