## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Reject symlinked WASI guest roots before syncing:** `internal/providers/wasi/backend.go:358`
  - body: `prepareGuest` creates `guestRoot` with `os.MkdirAll` before any no-follow validation, and `copyManifestToGuest` then creates the workdir under that path. If the guest root or workdir root already exists as a symlink, the later destination-component checks start too late and sync writes can be redirected outside the intended guest tree. Please create/validate the guest root and workdir with no-follow checks, and add a regression test for a preexisting symlinked root.
  - confidence: 0.86
- **[P2] Gate the WASI live smoke on provider selection:** `scripts/live-smoke.sh:460`
  - body: All other live provider smokes are guarded by `has_provider`, but this calls `wasi_smoke` unconditionally. That means `CRABBOX_LIVE_PROVIDERS=aws` or the default provider set will still run the new WASI flow, adding unexpected local state and possible failures to narrowed live-smoke runs. Wrap it in `if has_provider wasi; then ...; fi` when refreshing the script against current main.
  - confidence: 0.93
