## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Reject symlinked WASI workdir roots:** `internal/providers/wasi/backend.go:426`
  - body: The resync hardening starts with `os.MkdirAll(target)` and then treats `target` as the trusted root for no-follow checks. If a kept guest has the workdir itself replaced by a symlink to another host directory, `MkdirAll` accepts it and later copy/read checks operate under that resolved directory, so `/work` can escape before descendant symlink checks run. Lstat and reject the workdir root, including the no-sync path, before using it.
  - confidence: 0.91
- **[P2] Capture WASI output before enabling run-proof:** `internal/providers/wasi/backend.go:205-213`
  - body: The provider advertises `FeatureRunProof`, so `--emit-proof` is accepted, but `Run` builds the result without `LogExcerpt` while writing command output directly to stdout/stderr. Generated delegated proofs will report no console output even for successful WASI runs. Tee stdout/stderr into a bounded excerpt or remove `FeatureRunProof` until that contract is implemented.
  - confidence: 0.89
- **[P3] Remove PR-specific review-history comments:** `internal/providers/wasi/backend.go:448-449`
  - body: These comments bake temporary review history and a bare PR reference into provider code; the repo policy asks for full GitHub URLs for issue/PR references, and this detail does not explain the lasting invariant. Keep the invariant comments, but remove the PR-specific wording here and in the wasmtime env helper.
  - confidence: 0.86
