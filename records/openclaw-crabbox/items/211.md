Overall confidence: 0.9

Full review comments:

- **[P1] Validate the WASI workdir root before copying:** `internal/providers/wasi/backend.go:426`
  - body: `os.MkdirAll(target)` follows an existing symlink at the guest workdir root before any no-follow check runs. For a kept WASI lease, replacing `<guestRoot>/<workdir>` with a symlink lets the next sync copy manifest files through that symlink, so the root itself needs an `Lstat`/no-follow validation path before mkdir/copy writes and a regression test for a symlinked workdir root.
  - confidence: 0.92
- **[P2] Flip wasmtime `--dir` to host::guest:** `internal/providers/wasi/backend.go:735`
  - body: Wasmtime parses `--dir` as `HOST_DIR[::GUEST_DIR]`, but this builds `/work::<guestWork>`. That tries to open host `/work` and exposes it as the temp path in the guest, so `--wasi-runtime wasmtime` will not provide the synced tree at `/work`; use `<guestWork>::/work` and update the test expectation.
  - confidence: 0.95
- **[P1] Refresh live-smoke onto current main:** `scripts/live-smoke.sh:460`
  - body: The PR head adds `wasi_smoke` at the script tail where current main now also runs KubeVirt and external provider smokes, and GitHub reports the branch as dirty. Refresh this script so kubevirt, external, and wasi smoke blocks all survive in the merge result.
