## Summary

Keep open: this is a focused experimental WASI provider with useful proof and tests, but the current sync implementation still has a security-boundary bug because resync can follow preexisting guest symlinks when delete-sync is disabled. After that narrow fix, maintainers still need to explicitly accept the new built-in sandbox and runtime dependency posture.

## What This Changes

The PR adds an experimental built-in `wasi` delegated-run provider with `wasm`/`wazero` aliases, wazero execution by default, optional wasmtime execution, config/docs wiring, live-smoke coverage, and Go tests.

## Best Possible Solution

Harden WASI sync so destination writes never follow preexisting guest symlinks or symlinked parent directories, add regression coverage for `sync.delete: false`, then have maintainers decide whether the built-in experimental sandbox should ship.

## Reproduction Assessment

We have a source-level reproduction path, not a runtime-run reproduction: use a kept WASI guest with `sync.delete: false`, place a symlink or symlinked parent at a future manifest destination, then resync a regular file. The source shows that path reaches `os.OpenFile` without no-follow protection.

## Solution Assessment

No, not yet. The provider-adapter shape is reasonable, but the sync destination path needs no-follow or clean-staging hardening before this is the maintainable way to ship a trusted experimental sandbox.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.83

Full review comments:

- **[P1] Do not follow guest symlinks while resyncing:** `internal/providers/wasi/backend.go:491`
  - body: `prepareGuest` only removes the workdir when `sync.delete` is true. With `sync.delete: false`, a kept WASI workdir can already contain a symlink or symlinked parent at a manifest destination; this `os.OpenFile` follows that guest-controlled path during the next sync, so the host-side copier can write outside the intended `/work` tree. Copy through a no-follow/temp-and-rename path, or reject symlinked destination components before writing.
  - confidence: 0.84
