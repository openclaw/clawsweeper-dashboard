## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Validate WASI guest roots before use:** `internal/providers/wasi/backend.go:358`
  - body: `prepareGuest` creates the guest root with `os.MkdirAll` before any no-follow validation, and the workdir root is also created with ordinary path operations. For kept leases, replacing either root with a symlink can make later sync or runtime mount paths operate outside the intended `crabbox-wasi-*` tree even though nested destination components are checked. Add explicit `Lstat`/no-follow validation for the guest root and workdir root before creating, syncing, or mounting them.
  - confidence: 0.86
- **[P2] Honor sync.include in WASI manifests:** `internal/providers/wasi/backend.go:372`
  - body: Current main treats `sync.include` as a whitelist for provider sync, but this path calls `syncManifest(repoRoot, excludes)` and the wrapper delegates to `BuildSyncManifest` without includes. A user who configures a whitelist would still copy the full non-excluded checkout into `/work` for this provider. Thread `cfg.Sync.Includes` into `BuildSyncManifestFiltered` and add a regression test.
  - confidence: 0.9
