## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Do not pass forwarded env values through wasmtime argv:** `internal/providers/wasi/backend.go:638`
  - body: When users forward secrets with `--allow-env` and choose `--wasi-runtime wasmtime`, this serializes every value into the wasmtime process arguments as `--env KEY=value`. That violates the repo policy against passing secrets as command-line arguments and exposes tokens to local process inspection; use a safer inheritance mechanism or reject env forwarding for wasmtime.
  - confidence: 0.96
- **[P1] Keep copied symlinks inside the guest tree:** `internal/providers/wasi/backend.go:453`
  - body: After `resolveRepoSymlink` only checks that the resolved target is somewhere under `repoRoot`, this copies the original link target unchanged. An absolute repo symlink, or a link to a manifest-excluded path, leaves the guest tree pointing back at the host checkout rather than the synced `/work` tree; reject these links or rewrite them only when the target is also copied into the guest.
  - confidence: 0.88
