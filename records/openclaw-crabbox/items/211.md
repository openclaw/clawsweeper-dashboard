## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Validate the WASI workdir root before copying:** `internal/providers/wasi/backend.go:426`
  - body: `copyManifestToGuest` calls `os.MkdirAll(target)` before proving the guest workdir root itself is a real directory. If a kept guest workdir has been replaced with a symlink to a host directory, that call succeeds through the symlink and the later no-follow checks only validate child components, so sync can still write outside the intended guest tree. Lstat/validate `guestRoot` and `target` before any copy or mkdir under them.
  - confidence: 0.9
- **[P2] Gate the WASI live smoke on provider selection:** `scripts/live-smoke.sh:460`
  - body: The script normally runs provider smokes only when `CRABBOX_LIVE_PROVIDERS` includes that provider, but this block calls `wasi_smoke` unconditionally. A default `aws,hetzner` run or any single-provider smoke would now also create and run a WASI lease the caller did not request; wrap it with `if has_provider wasi` when resolving the current-main smoke conflict.
  - confidence: 0.86
