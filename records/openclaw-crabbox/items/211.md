## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Validate WASI guest roots before syncing:** `internal/providers/wasi/backend.go:426`
  - body: `copyManifestToGuest` creates the workdir root with `os.MkdirAll` before any no-follow check, and `prepareGuestDestination` treats that root as trusted. A kept guest can replace the workdir root with a symlink, so the next sync can write root-level files through the symlink or later mount the outside directory as `/work`; validate the guest root and workdir root with Lstat/no-follow checks before any mkdir/copy/no-sync/mount path.
  - confidence: 0.9
- **[P2] Gate the WASI live smoke behind provider selection:** `scripts/live-smoke.sh:460`
  - body: This unconditional call runs the new WASI smoke for every `CRABBOX_LIVE=1` invocation, even when `CRABBOX_LIVE_PROVIDERS` is left at the default `aws,hetzner` or set to a different provider. Wrap it in `if has_provider wasi; then ... fi` so the smoke matrix remains opt-in like the other provider-specific blocks.
  - confidence: 0.92
- **[P3] Reuse the same slug in the warmup docs:** `docs/providers/wasi.md:40-41`
  - body: The example warms up without `--slug my-wasi`, then tells users to run, status, and stop `my-wasi`; that named session will not exist. Add `--slug my-wasi` to the warmup command or change the follow-up commands to use the generated slug.
  - confidence: 0.87
