## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Reject symlinked guest and workdir roots before writes:** `internal/providers/wasi/backend.go:358`
  - body: `prepareGuest` and `copyManifestToGuest` still call `os.MkdirAll` on the guest root/workdir root before any no-follow check. If a kept guest root or configured workdir root already exists as a symlink, later sync/no-sync writes can be redirected outside the WASI guest tree even though descendant parents are validated. Add Lstat/no-follow validation for both roots before every host write path and cover both cases with regression tests.
  - confidence: 0.94
- **[P2] Gate the WASI live smoke behind provider selection:** `scripts/live-smoke.sh:460`
  - body: The live-smoke script is provider-filtered through `CRABBOX_LIVE_PROVIDERS`, but this call runs `wasi_smoke` unconditionally. A user asking for only `aws` or `hetzner` now also runs local WASI warmup/sync/status/stop, which can fail or mutate state for unrelated live-smoke runs. Wrap it in the same `if has_provider wasi; then ... fi` pattern as the other providers.
  - confidence: 0.9
