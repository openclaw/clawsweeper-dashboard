- **[P1] Keep forwarded env values out of wasmtime argv:** `internal/providers/wasi/backend.go:638`
  - body: `req.Env` comes from explicit env forwarding and can contain secrets, but this builds `--env KEY=VALUE` arguments for the wasmtime child.   - confidence: 0.94

  - body: The provider rejects arbitrary host paths and only supports `.wasm` modules plus the small builtin set, so this `/bin/false` example exercises the unsupported-command error rather than the intended nonzero builtin. Use `false` here to match the implementation and the live smoke script.
  - confidence: 0.88
