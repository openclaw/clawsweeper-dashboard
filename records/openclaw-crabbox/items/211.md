## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Reject symlinked WASI roots before using them:** `internal/providers/wasi/backend.go:426`
  - body: `copyManifestToGuest` creates the workdir root with `os.MkdirAll(target)` before any no-follow check. If a kept guest workdir root, or the guest root via the configured base dir, has been replaced with a symlink, later copies and runtime mounts can follow it outside the intended guest tree. Validate the guest root and workdir root with `Lstat`/no-follow before `MkdirAll`, no-sync setup, and runtime mounting, and cover both root-symlink cases with regressions.
  - confidence: 0.88
- **[P2] Gate WASI live smoke on provider selection:** `scripts/live-smoke.sh:460`
  - body: `scripts/live-smoke.sh` uses `has_provider` for the other provider smokes, but this new call runs `wasi_smoke` unconditionally. A selected run such as `CRABBOX_LIVE_PROVIDERS=wandb` will now also create and stop a local WASI lease, which changes unrelated live-smoke automation and can fail on an unrequested provider path. Wrap it in a `has_provider wasi` check.
  - confidence: 0.95
- **[P2] Populate proof excerpts before advertising run-proof:** `internal/providers/wasi/provider.go:27`
  - body: The provider declares `FeatureRunProof`, so `--emit-proof` is allowed, but the WASI backend streams stdout/stderr directly and never fills `RunResult.LogExcerpt`. Generated proof will therefore report no captured console output even for a successful visible module run. Capture a bounded combined excerpt while streaming, like the existing delegated proof provider does, or drop `FeatureRunProof` until useful proof data is returned.
  - confidence: 0.9
