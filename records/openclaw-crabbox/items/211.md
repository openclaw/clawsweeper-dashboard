## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Reject symlinked WASI guest roots before syncing:** `internal/providers/wasi/backend.go:358`
  - body: `os.MkdirAll(guestRoot)` follows an existing symlink at the computed guest root before the provider has established a trusted directory. For kept/reused local WASI state, that lets a preexisting `crabbox-wasi-<lease>` symlink redirect later sync writes outside the intended guest tree; create/validate the guest root with no-follow semantics and add a regression before merging.
  - confidence: 0.91
- **[P1] Reject symlinked WASI workdir roots before copying:** `internal/providers/wasi/backend.go:426`
  - body: `copyManifestToGuest` trusts `target` after `os.MkdirAll(target)`, but if the workdir root already exists as a symlink, the following `joinUnder`/copy path treats the symlink as the root and writes files through it. Please Lstat and reject a symlinked workdir root before copy/no-sync reuse, with a regression for this exact root case.
  - confidence: 0.93
- **[P1] Gate WASI live smoke behind provider selection:** `scripts/live-smoke.sh:460`
  - body: `scripts/live-smoke.sh` uses `CRABBOX_LIVE_PROVIDERS` to decide which provider smokes run, and the surrounding provider calls are guarded with `has_provider`. This unconditional call means default `aws,hetzner` runs, or a narrowed selection like `CRABBOX_LIVE_PROVIDERS=e2b`, will also run WASI sync/warmup/status/stop and can fail or create local WASI state without opting into that provider.
  - confidence: 0.97
