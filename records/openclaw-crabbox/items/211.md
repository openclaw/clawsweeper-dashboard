Full review comments:

- **[P1] Restrict the wasmtime child environment:** `internal/providers/wasi/backend.go:697-700`
  - body: `buildWasmtimeRunArgsAndEnv` still seeds the subprocess env from `os.Environ()`, and `runWithWasmtimeCLI` passes that full slice to wasmtime. That means `--wasi-runtime wasmtime` exposes ambient host tokens/config to the subprocess even though the provider boundary is described as explicit env forwarding; start from a minimal env plus the allowed map and add a regression that unrelated host env vars are absent.
  - confidence: 0.92
- **[P3] Reject unsupported flags for WASI aliases:** `internal/providers/wasi/backend.go:52-58`
  - body: This guard only matches the canonical provider name, but the provider also registers `wasm` and `wazero` aliases.   - confidence: 0.82
