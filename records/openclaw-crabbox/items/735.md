Full review comments:

- **[P2] Enforce native VNC connection caps:** `worker/src/fleet.ts:3272-3275`
  - body: The existing workspace terminal route checks per-workspace, owner, and global limits before accepting WebSockets, but this new ticket handoff goes straight to `createWebSocketUpgrade` and starts a coordinator SSH relay. An authenticated client can mint many short-lived tickets and open many relays, bypassing the existing terminal caps while each connection holds buffers and an SSH client. Please add native VNC connection accounting and limits before upgrading.
  - confidence: 0.88
- **[P2] Use the native VNC public URL check:** `worker/src/fleet.ts:12511`
  - body: `workspacePublicURL` accepts loopback HTTP for native grants and the CLI also accepts loopback HTTP grant URLs, but `workspaceNativeVNCError` delegates to `workspaceTerminalError`, which rejects every non-HTTPS `CRABBOX_PUBLIC_URL`. With a local loopback coordinator, grant creation returns `native_vnc_unavailable` before minting a ticket. Please use the native URL check here or make the loopback exception consistent.
  - confidence: 0.82
