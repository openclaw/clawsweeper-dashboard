Full review comments:

- **[P1] Treat State=running as running when Running is omitted:** `internal/providers/tart/backend.go:185-186`
  - body: Resolve rejects any non-status call when `inst.Running` is false, but the PR discussion reports Tart can omit that JSON field. In that case a live VM with `State: "running"` decodes with `Running=false`, so run/ssh/code/cache/egress can reject a valid claimed VM; gate this on `instanceRunning(inst.State)` or a combined helper and add a regression with `Running` omitted.
  - confidence: 0.94
- **[P1] Check cancellation before launching kept VMs:** `internal/providers/tart/backend.go:409-414`
  - body: The keep branch ignores `ctx` and starts `tart run` with `exec.Command` even if acquisition has already been canceled. Check `ctx.Err()` before `Start` and include cancellation in the startup observation path so a canceled request cannot create local VM state before the later cleanup path runs.
  - confidence: 0.86
- **[P2] Make tart disk help match validation:** `internal/providers/tart/flags.go:24`
  - body: The flag help says `--tart-disk 0` uses the clone default, but the next block rejects zero as invalid. Either allow explicit zero to mean clone default or change the help/docs to tell users to omit the flag for clone-default behavior.
  - confidence: 0.98
