Full review comments:

- **[P2] Preserve shared SSH and workRoot overrides:** `internal/cli/config.go:1095-1096`
  - body: Because `baseConfig` now pre-populates `cfg.Tart.User` and `cfg.Tart.WorkRoot`, the later Tart defaulting path treats those provider fields as set and overwrites `cfg.SSHUser`/`cfg.WorkRoot` with `admin` and `/Users/admin/crabbox`. A user following the documented top-level `ssh.user`/`CRABBOX_SSH_USER` or `workRoot`/`CRABBOX_WORK_ROOT` contract will inject the key and sync into the wrong account/path unless they duplicate the value under `tart.*`. Keep these as fallback defaults only when Tart-specific config was not explicitly set, and add a regression for top-level overrides.
  - confidence: 0.9
- **[P3] Align the disk flag help with validation:** `internal/providers/tart/flags.go:24`
  - body: The flag help advertises `--tart-disk` value `0` as “use clone default”, but `applyFlags` rejects `<= 0` immediately. Either accept `0` as no resize or change the help text, otherwise users following `--help` get a validation error.
  - confidence: 0.86
