## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Reject non-positive Tart resources from config/env:** `internal/providers/tart/flags.go:61-68`
  - body: The non-flag validation only rejects values below zero, while YAML loading ignores zero/negative values and env parsing falls back on non-numeric input. That means `CRABBOX_TART_CPUS=0`, `CRABBOX_TART_CPUS=abc`, or `tart.cpus: -1` can silently default instead of failing before the VM is configured, so users do not get the size they requested or a useful config error.
  - confidence: 0.91
- **[P2] Report Tart image as the effective server type:** `internal/cli/config.go:774-789`
  - body: `config show` still reaches generic `serverTypeForConfig` after the Tart provider defaults run, and that function has no Tart case, so provider=tart reports the default Hetzner type instead of the Tart VM image. This makes the new provider diagnostics misleading even when `cfg.Tart.Image` is set correctly.
  - confidence: 0.9
- **[P2] Keep malformed Tart claims visible to cleanup:** `internal/providers/tart/backend.go:644-647`
  - body: `providerClaims()` drops Tart claims that lack `labels.instance` or an `instance:` providerScope before `Cleanup()` does its orphan-claim pass. As a result, malformed Tart claim files are invisible to `crabbox cleanup --provider tart` and cannot be removed by the repair path that should handle stale local state.
  - confidence: 0.93
