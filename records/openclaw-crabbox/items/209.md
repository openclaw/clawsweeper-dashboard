Full review comments:

- **[P1] Preserve Tart lease state when release fails:** `internal/providers/tart/backend.go:272-275`
  - body: `ReleaseLease` discards both `stopVM` and `deleteVM` errors and then removes the lease claim and stored key. If Tart fails to stop or delete a real VM, Crabbox loses the state needed to target or clean it up later; only known-missing stale claims should be pruned this way.
  - confidence: 0.93
- **[P2] Reject running Tart VMs that have no IP:** `internal/providers/tart/backend.go:596-599`
  - body: `Resolve` treats `State=running` as running even when the `Running` bool is omitted, but this branch checks only `!inst.Running` when the IP is empty. That can return a lease with no SSH host for non-status commands instead of failing fast; use the normalized state check here too.
  - confidence: 0.86
