## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve YAML Tart resource presence:** `internal/cli/config.go:1537-1539`
  - body: `fileTartConfig` stores `cpus`, `memory`, and `disk` as plain `int`s, and `applyFileConfig` only copies values greater than zero. That means `.crabbox.yaml` entries like `tart: { cpus: 0 }` or `cpus: -1` are discarded before the Tart validation in `applyFlags` runs, so the provider falls back to defaults and can launch a VM with resources the user explicitly tried to set. Use pointer/presence-tracked fields and add file-config regressions for zero, below-minimum, and negative values.
  - confidence: 0.93
