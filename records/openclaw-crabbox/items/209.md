## Summary

Keep open. The Tart provider is useful and proof-backed, but current head still lets a stale Tart claim reach `ReleaseLease`, where `tart delete` runs before the claim/key are removed, so `crabbox stop` cannot clear stale state when the VM was already deleted outside Crabbox.

## What This Changes

Adds a local Cirrus Labs Tart SSH-lease provider for Apple Silicon macOS VMs, with config/env/CLI wiring, status plumbing, docs, registration, and provider tests.

## Best Possible Solution

Land the macOS Tart provider after `ReleaseLease` treats an already-missing VM as a stale-claim prune path, with Linux/Ubuntu Tart support tracked separately if maintainers want that scope.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a Tart lease claim whose VM is absent from `tart list` resolves as `State: "missing"`, then `ReleaseLease` calls `tart delete` and returns before pruning the claim/key.

## Solution Assessment

No. The PR should keep propagating real delete failures for existing VMs, but it needs a separate stale-missing branch that prunes local Crabbox state without requiring Tart to delete an already-absent VM.
