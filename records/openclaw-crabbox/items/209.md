## Summary

## What This Changes

The branch adds a new `tart` local macOS SSH-lease provider with CLI/YAML/env config, provider registration/status plumbing, docs, and provider/CLI tests.

## Best Possible Solution

Fix the missing-VM release path before merge, keep this provider scoped to macOS Tart inside the provider adapter, and leave Linux Tart support to a separate follow-up after the macOS provider lands cleanly.

## Reproduction Assessment

Yes. From source inspection, `crabbox stop` resolves with `ReleaseOnly`, `Resolve` returns a named `missing` Tart instance, and `ReleaseLease` then skips the only missing-prune branch because the name is already present.

## Solution Assessment

No. The latest repair is the right idea, but it is attached to the fallback-only path; the maintainable fix is to handle missing resolved leases before calling `tart stop`/`tart delete` and cover the stop-shaped path.
