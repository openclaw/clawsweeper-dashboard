## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Reject invalid Tart resources from config/env:** `internal/cli/config.go:2649-2656`
  - body: These YAML branches only copy resource values when they are `> 0`, while the env path uses `getenvInt` and the Tart flag hook only rejects `< 0` for non-CLI sources before defaults rewrite `<= 0`. That means explicit typos such as `tart.cpus: -1`, `tart.memory: 0`, `CRABBOX_TART_CPUS=0`, or a non-numeric env value can be silently ignored/defaulted instead of failing the public Tart sizing contract. Track whether config/env values were supplied and reject non-positive or unparsable values before `applyDefaults` can mask them.
  - confidence: 0.96
- **[P2] Set Tart's effective server type before config show:** `internal/cli/config.go:774-789`
  - body: This Tart provider-default branch returns without setting `cfg.ServerType`, and `serverTypeForConfig` has no Tart case. For `provider=tart`, `config show --json` can therefore render the class fallback such as `ccx63` instead of the effective Tart image that the backend later uses. Populate `ServerType` from `cfg.Tart.Image` here or expose it through the provider server-type hook before config rendering.
  - confidence: 0.93
- **[P2] Filter doctor lease counts to Crabbox Tart VMs:** `internal/providers/tart/backend.go:227`
  - body: `Doctor` reports `leases` as `len(instances)` from raw `tart list`, but `List` filters to Crabbox claims or `crabbox-` names. On a Tart host with unrelated VMs or OCI cache entries, doctor will report those as Crabbox leases, making setup and cleanup diagnostics misleading. Count the same filtered inventory as `List`, or rename the raw count and add a separate filtered lease count.
  - confidence: 0.95
