## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve generic SSH and workRoot settings for Tart:** `internal/providers/tart/backend.go:65-68`
  - body: `applyDefaults` always copies Tart defaults onto `cfg.SSHUser` and `cfg.WorkRoot`. Because Tart defaults are already populated in base config, a user who sets generic `ssh.user` or top-level `workRoot` still gets `admin` and `/Users/admin/crabbox`.
  - confidence: 0.88
- **[P1] Check cancellation before launching kept VMs:** `internal/providers/tart/backend.go:409-418`
  - body: For `--keep`, `startVM` uses `exec.Command` and starts `tart run` without checking whether the request context is already canceled. A canceled acquisition can still launch a local VM before cleanup can reason about it.
  - confidence: 0.87
- **[P2] Treat running State as running when Running is omitted:** `internal/providers/tart/backend.go:185-186`
  - body: `Resolve` rejects non-status callers when `inst.Running` is false, but Tart's `State` is also present and older or variant JSON can omit the bool. A claimed VM with `State: running` and no `Running` field is rejected for normal SSH-target commands.
  - confidence: 0.84
- **[P2] Make tart disk-zero behavior match the flag help:** `internal/providers/tart/flags.go:24`
  - body: The flag help advertises `0 = use clone default`, but `applyFlags` rejects `--tart-disk 0`. Either allow zero as the documented no-resize value or remove that promise from the flag help and related tests/docs.
  - confidence: 0.82
- **[P2] Let stop prune stale claims whose Tart VM is gone:** `internal/providers/tart/backend.go:252-254`
  - body: When release has only a lease ID, `ReleaseLease` returns immediately if `resolveInstance` says the claimed VM no longer exists. That leaves `crabbox stop --provider tart <lease>` unable to prune the stale claim reported in the thread.
  - confidence: 0.77
- **[P1] Refresh the config merge against current main:** `internal/cli/config.go:17-20`
  - body: GitHub reports the PR as dirty, and the conflicting area is `internal/cli/config.go`, where current main has post-base config work. Refresh the branch so the final tested result preserves both current config fields and Tart's new fields.
  - confidence: 0.9
