## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve generic SSH and work root overrides:** `internal/cli/config.go:777-785`
  - body: Because `baseConfig()` pre-populates `Tart.User` and `Tart.WorkRoot`, this block always copies the Tart defaults back onto `cfg.SSHUser` and `cfg.WorkRoot`. A config that sets `provider: tart` with top-level `ssh.user` or `workRoot` therefore runs as `admin` in `/Users/admin/crabbox` unless the user also knows to set the Tart-specific keys, so existing generic config conventions are silently lost.
  - confidence: 0.95
- **[P2] Allow zero disk to keep the clone default:** `internal/providers/tart/flags.go:50-51`
  - body: The flag help and provider docs describe disk `0` as clone default, and `configureVM` already skips `--disk-size` unless the value is greater than zero. Rejecting `--tart-disk 0` here prevents users from clearing a configured disk override back to clone default; the later explicit-disk validation should allow zero and reject only negative values.
  - confidence: 0.92
