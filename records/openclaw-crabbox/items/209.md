Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Propagate real delete failures before dropping lease state:** `internal/providers/tart/backend.go:273-275`
  - body: `ReleaseLease` ignores `tart delete` errors and then removes the Crabbox claim and stored key. That is only safe for a stale claim whose VM is already gone; for a real delete failure, Crabbox forgets a VM that still exists, so please propagate real delete failures or narrowly detect the missing-VM case before removing state.
  - confidence: 0.91
- **[P2] Use State when deciding whether a no-IP VM is running:** `internal/providers/tart/backend.go:595-600`
  - body: This branch only checks `inst.Running`, but Tart JSON can omit that bool while `State` is still `running`. In that case Resolve can return a LeaseTarget with no SSH target instead of the intended no-IP error, so this guard should also use `instanceRunning(inst.State)`.
  - confidence: 0.88
