## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Preserve generic Tart SSH user and work root:** `internal/cli/config.go:777-785`
  - body: Because `baseConfig()` pre-populates `cfg.Tart.User` and `cfg.Tart.WorkRoot`, this branch always copies the Tart defaults back over top-level `ssh.user` and `workRoot`. A user who configures the generic SSH account/path for an SSH lease still gets `admin` and `/Users/admin/crabbox` unless they know to duplicate the values under `tart:`, so the provider should distinguish explicit Tart values from defaults.
  - confidence: 0.93
- **[P1] Preserve kept Tart VMs after startup failures:** `internal/providers/tart/backend.go:132-135`
  - body: After `startVM` succeeds, every later acquisition failure still stops and deletes the VM even when the caller passed `--keep`. That defeats the recovery/debugging purpose of `--keep` exactly when IP, SSH-key, readiness, claim, or endpoint setup fails; guard the post-start cleanup with `if !req.Keep`.
  - confidence: 0.9
- **[P2] Honor cancellation before starting kept Tart VMs:** `internal/providers/tart/backend.go:409-433`
  - body: The keep path uses `exec.Command` and the startup select only waits for process exit or a timer, so an already-canceled context can still launch a detached Tart VM and return success after the observation window. Check `ctx.Err()` before start and handle `ctx.Done()` during the observation window, killing the just-started process if cancellation wins.
  - confidence: 0.88
- **[P2] Treat Tart State=running as running when Running is omitted:** `internal/providers/tart/backend.go:185-186`
  - body: `Running` is an optional decoded bool, so a `tart list` record with `State:"running"` but no `Running` field is rejected as stopped. Use the normalized state helper here so valid claimed running leases do not break reuse paths.
  - confidence: 0.9
- **[P2] Allow the documented Tart disk zero reset:** `internal/providers/tart/flags.go:49-51`
  - body: The flag help documents `--tart-disk 0` as clone-default, and the backend skips disk resizing when the value is zero, but flag/config validation rejects explicit zero. That makes it impossible to override a previously configured disk size back to clone-default from the CLI/env/YAML path.
  - confidence: 0.91
- **[P2] Let stop prune Tart claims whose VM is already gone:** `internal/providers/tart/backend.go:528-538`
  - body: When a local Tart claim exists but the referenced VM is missing from `tart list`, release-only resolution errors before `ReleaseLease` can remove the stale claim/key. For stop-by-lease/slug, return a release target that lets the local claim be pruned without attempting `tart stop` or `tart delete` for a missing VM.
  - confidence: 0.86
