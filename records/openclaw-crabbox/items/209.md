## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Preserve generic Tart SSH and work-root overrides:** `internal/providers/tart/backend.go:49-50`
  - body: When `tart.user`/`tart.workRoot` are not explicitly configured, the provider defaults still overwrite top-level `ssh.user` and `workRoot`, so a normal Crabbox config can inject the key into `admin` and sync to `/Users/admin/crabbox` instead of the user's configured account/path. Track Tart-specific explicitness and let the generic settings flow through unless the Tart-specific fields are set.
  - confidence: 0.93
- **[P1] Use Tart state when Running is omitted:** `internal/providers/tart/backend.go:185-186`
  - body: `Running` is an optional JSON field from `tart list`; if Tart reports `State:"running"` without that boolean, this branch treats the claimed VM as stopped and rejects non-status reuse. Use the normalized state helper here so running VMs remain usable even when the boolean is absent.
  - confidence: 0.94
- **[P1] Do not delete kept Tart VMs after post-start failures:** `internal/providers/tart/backend.go:132-135`
  - body: After `startVM` succeeds, every later acquisition failure still stops and deletes the VM, even for `--keep`. That removes the partially provisioned local VM exactly when the operator needs to inspect DHCP, SSH injection, SSH readiness, or claim-write failures; guard the post-start cleanup with `!req.Keep`.
  - confidence: 0.92
- **[P1] Honor cancellation before starting kept VMs:** `internal/providers/tart/backend.go:410-418`
  - body: The keep path uses `exec.Command` instead of `exec.CommandContext` and does not check `ctx.Err()` before `cmd.Start`, so a canceled acquire can still launch a detached VM and return success after the observation timer. Check cancellation before launch and handle `ctx.Done()` during startup observation.
  - confidence: 0.91
- **[P2] Let release prune claim-backed missing VMs:** `internal/providers/tart/backend.go:538`
  - body: When a local claim points at a Tart VM that no longer appears in `tart list`, `Resolve(ReleaseOnly)` errors here before `ReleaseLease` can remove the stale claim/key. For release-only resolution, return a missing-instance target that prunes Crabbox-owned local state without calling `tart stop/delete`.
  - confidence: 0.9
- **[P3] Align explicit disk zero with the clone-default contract:** `internal/providers/tart/flags.go:50-51`
  - body: The flag help says `--tart-disk 0` means clone default, but this validation rejects zero before the provider can skip `--disk-size`. Either accept zero as the documented reset-to-default sentinel or change the help/docs so CLI behavior and docs agree.
  - confidence: 0.91
