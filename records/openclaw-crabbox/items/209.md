## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Honor cancellation before starting kept Tart VMs:** `internal/providers/tart/backend.go:410-418`
  - body: When `keep` is true this path uses `exec.Command` and never checks `ctx` before `cmd.Start` or during the startup observation window. A canceled acquire can still launch a detached Tart VM and return nil after the timer, leaving work running after the caller abandoned it.
  - confidence: 0.93
- **[P1] Preserve kept VMs after post-start failures:** `internal/providers/tart/backend.go:133-135`
  - body: After `startVM` succeeds, these failure paths always stop and delete the VM even when the request used `--keep`. That defeats the purpose of keeping a failed VM around for diagnosis; the adjacent Multipass acquire path only removes the instance after these phases when `Keep` is false.
  - confidence: 0.9
- **[P1] Treat State=running as a running Tart instance:** `internal/providers/tart/backend.go:185-186`
  - body: This check depends on the optional `Running` JSON bool, so a `tart list` record with `State: "running"` but no `Running` field defaults to false and gets rejected as stopped. Use the normalized state helper here so lease reuse works across Tart output shapes.
  - confidence: 0.9
- **[P1] Preserve generic SSH user and work root defaults:** `internal/cli/config.go:777-785`
  - body: Because base config pre-populates `Tart.User` and `Tart.WorkRoot`, this branch overwrites top-level `ssh.user` and `workRoot` even when the user did not set Tart-specific values. That makes generic Crabbox SSH settings silently ineffective for the new provider.
  - confidence: 0.87
- **[P2] Align --tart-disk zero handling with help text:** `internal/providers/tart/flags.go:50-51`
  - body: The flag help says `0 = use clone default`, but `applyFlags` rejects zero before the provider can run. Either accept zero as the documented sentinel or change the help/docs so users do not get contradictory guidance.
  - confidence: 0.88
