## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve lease state when Tart delete fails:** `internal/providers/tart/backend.go:273-276`
  - body: `ReleaseLease` now discards `tart delete` errors and then removes the claim and per-lease key. If delete fails for a real still-present VM rather than a known-missing stale claim, Crabbox loses the ownership state needed to stop or clean it up later; only remove state after a successful delete or a confirmed already-gone VM.
  - confidence: 0.91
- **[P1] Honor keep on every post-start acquire failure:** `internal/providers/tart/backend.go:162-164`
  - body: After the VM has started, failures from `prepareLease`, claim creation, or endpoint update still stop/delete the VM unconditionally. That contradicts the PR's kept-lease behavior for post-start failures and can destroy a VM the user asked to keep when SSH readiness or local state update is transient.
  - confidence: 0.86
- **[P2] Check State before returning stopped no-IP leases:** `internal/providers/tart/backend.go:595-599`
  - body: The Resolve guard checks `State`, but `prepareLease` still treats empty IP plus omitted `Running` as a stopped lease. With Tart JSON that has `State=running` and no `Running` field, non-status callers can get a successful LeaseTarget with no SSH host instead of a clear no-IP error.
  - confidence: 0.88
