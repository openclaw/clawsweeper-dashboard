## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve Tart claim state when stop or delete fails:** `internal/providers/tart/backend.go:272-276`
  - body: ReleaseLease ignores both `tart stop` and `tart delete` errors and then removes the claim and stored key. If teardown fails for a real VM, Crabbox loses the state needed to target or retry cleanup; only prune state after successful teardown or a proven-missing stale VM.
  - confidence: 0.93
- **[P1] Honor --keep on late Tart acquire failures:** `internal/providers/tart/backend.go:162-174`
  - body: The earlier IP/key-injection failures preserve kept VMs, but `prepareLease`, claim creation, and endpoint update errors still stop/delete unconditionally. That can destroy a VM the user requested to keep after SSH readiness or metadata persistence fails.
  - confidence: 0.9
- **[P2] Stop launched kept VMs when startup is canceled:** `internal/providers/tart/backend.go:433-437`
  - body: For `keep=true`, `startVM` uses `exec.Command`, so a context cancellation after `Start` does not stop Tart. Returning here lets Acquire attempt only `deleteVM`, which can leave a started VM unclaimed if delete refuses a running VM.
  - confidence: 0.84
