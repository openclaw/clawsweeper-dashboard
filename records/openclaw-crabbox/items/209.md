## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve claims when Tart deletion really fails:** `internal/providers/tart/backend.go:273`
  - body: `ReleaseLease` ignores `tart delete` errors and then removes the lease claim and stored key. That is only safe for a confirmed-missing stale VM; if delete fails because the VM is busy or Tart errors, Crabbox loses its local lease handle while the VM may still be running.
  - confidence: 0.93
- **[P2] Treat State=running as running when IP is empty:** `internal/providers/tart/backend.go:596`
  - body: `Resolve` correctly accepts instances whose JSON omits `Running` but has `State: running`, yet `prepareLease` uses only `!inst.Running` when IP is empty. A running VM with omitted `Running` and no IP can therefore return a lease without SSH instead of failing with the no-IP diagnostic.
  - confidence: 0.84
