## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Propagate real delete failures before dropping lease state:** `internal/providers/tart/backend.go:273-276`
  - body: `ReleaseLease` currently discards the result of `deleteVM` and then removes the lease claim/key. If `tart delete` fails for a real existing VM, Crabbox reports success but loses the metadata needed to manage the still-running VM. Please only tolerate the confirmed missing-VM/stale-claim case and return other delete failures before removing state.
  - confidence: 0.93
- **[P2] Use State when handling a running VM with no IP:** `internal/providers/tart/backend.go:595-601`
  - body: `Resolve` now permits `State: "running"` when Tart omits `Running`, but this branch still treats `Running=false` as stopped when the IP is empty. A running VM with no IP can therefore return a LeaseTarget with empty SSH fields to ssh/code/cache callers instead of a clear no-IP error. Use `instanceRunning(inst.State)` here too.
  - confidence: 0.9
- **[P2] Clean up kept VMs when startup is canceled:** `internal/providers/tart/backend.go:433-437`
  - body: When the context is canceled after `tart run` starts for a kept lease, this branch returns without stopping the process. `Acquire` then tries `delete` without a stop and ignores that error, which can leave an unclaimed kept VM running after cancellation. Stop/kill this startup-canceled VM or make acquisition cleanup reliably remove it before returning.
  - confidence: 0.78
