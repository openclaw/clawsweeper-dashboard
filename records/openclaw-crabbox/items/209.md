Full review comments:

- **[P1] Preserve generic Tart SSH and work-root overrides:** `internal/cli/config.go:777-785`
  - body: When `provider: tart` is selected, `baseConfig` already has Tart defaults, so this branch copies `admin` and `/Users/admin/crabbox` back over top-level `ssh.user` and `workRoot`. Users with generic Crabbox config will silently run as the wrong account/path unless they duplicate settings under `tart:`.
  - confidence: 0.93
- **[P1] Use Tart state when Running is omitted:** `internal/providers/tart/backend.go:185-186`
  - body: This rejects any non-status reuse when the optional `Running` JSON field is absent, even if Tart reports `State: "running"` and an IP is available. That blocks existing running leases from `run`, `ssh`, `cache`, and similar target commands.
  - confidence: 0.94
- **[P1] Do not delete kept Tart VMs after post-start failures:** `internal/providers/tart/backend.go:132-135`
  - body: After `startVM` succeeds, failures in IP polling, SSH-key injection, readiness, claim creation, or endpoint update still stop and delete the VM even when `Keep` is true. That removes the partially provisioned VM the operator asked to preserve for inspection.
  - confidence: 0.9
- **[P1] Honor cancellation before starting kept Tart VMs:** `internal/providers/tart/backend.go:409-414`
  - body: The keep path uses plain `exec.Command` and the startup select ignores `ctx.Done()`, so a canceled acquire can still launch a detached Tart VM and return nil after the observation timer. Check cancellation before start and during the observation window.
  - confidence: 0.9
- **[P2] Let stop prune claim-backed missing Tart VMs:** `internal/providers/tart/backend.go:538`
  - body: When a local Tart claim points at a VM that no longer exists, `Resolve(ReleaseOnly)` errors here before `ReleaseLease` can remove the stale claim and stored key. Release-only resolution should return a stale target or otherwise prune the claim without calling Tart stop/delete for a missing VM.
  - confidence: 0.88
- **[P3] Align explicit disk zero with the clone-default contract:** `internal/providers/tart/flags.go:49-51`
  - body: The flag help and provider docs describe `0` as clone-default, but this validation rejects `--tart-disk 0`. That also prevents overriding a configured disk size back to clone-default from the CLI.
  - confidence: 0.87
