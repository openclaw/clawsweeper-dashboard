Full review comments:

- **[P1] Return Tart release failures before clearing lease state:** `internal/providers/tart/backend.go:272-273`
  - body: `ReleaseLease` ignores both `stopVM` and `deleteVM` failures and then removes the lease claim and stored key. If Tart fails to stop or delete a real VM, `crabbox stop` can report success while the VM keeps running and the local retry/cleanup metadata is gone; only confirmed missing-instance stale claims should be pruned this way.
  - confidence: 0.91
- **[P2] Honor keep after late acquisition failures:** `internal/providers/tart/backend.go:160-173`
  - body: After the VM is running and the SSH key is injected, failures in `prepareLease`, claim persistence, or endpoint update always stop and delete the VM even when `req.Keep` is true. That contradicts the provider's promised kept-lease failure behavior and prevents users from inspecting late startup failures.
  - confidence: 0.82
