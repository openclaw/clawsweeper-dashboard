## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve generic SSH and workRoot settings for Tart:** `internal/providers/tart/backend.go:65-68`
  - body: `applyDefaults` always copies the Tart defaults back onto `cfg.SSHUser` and `cfg.WorkRoot`. Because `baseConfig` already sets Tart's user/work root, a user who sets generic `ssh.user` or top-level `workRoot` still gets `admin` and `/Users/admin/crabbox`, which is the override reported in the thread.
  - confidence: 0.88
- **[P1] Check cancellation before launching kept VMs:** `internal/providers/tart/backend.go:409-418`
  - body: For `--keep`, `startVM` uses `exec.Command` and starts `tart run` without checking whether the request context is already canceled. A canceled acquisition can still launch a detached local VM before later cleanup has a chance to run.
  - confidence: 0.87
- **[P2] Treat running State as running when Running is omitted:** `internal/providers/tart/backend.go:185-186`
  - body: `Resolve` rejects any instance whose `Running` bool is false, but Tart state is already available and older or variant JSON can omit that bool. A claimed VM with `State: running` and no `Running` field will be rejected for normal SSH-target commands.
  - confidence: 0.84
- **[P2] Make tart disk-zero behavior match the flag help:** `internal/providers/tart/flags.go:49-52`
  - body: The flag help says `--tart-disk` accepts `0 = use clone default`, but `applyFlags` rejects `0` as an error. Either the flag should allow zero and skip resizing or the help/docs should stop advertising that value.
  - confidence: 0.82
- **[P2] Let stop prune stale claims whose Tart VM is gone:** `internal/providers/tart/backend.go:528-538`
  - body: When an identifier resolves to a Tart claim whose instance no longer exists, `resolveInstance` returns an error before the release path can remove the stale claim. That leaves `crabbox stop --provider tart <lease>` unable to clean the exact stale-claim state reported in the thread.
  - confidence: 0.77
- **[P1] Refresh the config merge against current main:** `internal/cli/config.go:17-20`
  - body: Current main added config fields in `internal/cli/config.go` after this PR's base, and `git merge-tree` now produces conflict markers in the `Config` struct. The branch needs a refresh that preserves both current `Sync.Includes` work and Tart's new config fields before the final tests are meaningful.
  - confidence: 0.9
