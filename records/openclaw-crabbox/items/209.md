## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve the generic Tart SSH user:** `internal/providers/tart/backend.go:49-50`
  - body: When `tart.user` is unset, `applyDefaults` defaults it to `admin` and later overwrites `cfg.SSHUser` from that value. A user who already configured the generic `ssh.user` will still have Tart inject and connect as `admin`, so the provider should fall back to the existing SSH user before choosing the Tart default.
  - confidence: 0.89
- **[P1] Preserve kept Tart VMs after startup failures:** `internal/providers/tart/backend.go:133-135`
  - body: After `startVM(..., keep=true)` succeeds, failures in IP polling, key injection, lease preparation, or claim updates still stop and delete the VM unconditionally. That defeats `--keep` for exactly the debugging case it is meant to support; match the adjacent providers and only tear down these post-start failures when the lease is not kept.
  - confidence: 0.91
- **[P2] Honor cancellation before starting kept Tart VMs:** `internal/providers/tart/backend.go:409-414`
  - body: The kept path uses `exec.Command` without checking `ctx`, so a request whose context has already been canceled can still launch a detached VM and then fail later without a claim. Check the context before starting the detached process or otherwise prevent canceled acquires from creating untracked VMs.
  - confidence: 0.86
- **[P2] Treat State=running as running when Running is omitted:** `internal/providers/tart/backend.go:185-186`
  - body: Resolve gates non-status callers on `inst.Running`, but Tart JSON that has `State: running` and omits the `Running` field unmarshals this bool as false. That makes valid running VMs look stopped for run/ssh/code/cache/egress; use the normalized state as a fallback when the boolean is absent or false.
  - confidence: 0.88
- **[P2] Allow the documented Tart disk zero value:** `internal/providers/tart/flags.go:49-51`
  - body: The flag help documents `--tart-disk 0` as clone default, but the flag path rejects zero before it can mean clone default. Either allow explicit zero and skip disk resizing, or change the help/docs so users do not get a documented value that always errors.
  - confidence: 0.93
- **[P2] Let stop prune Tart claims whose VM is gone:** `internal/providers/tart/backend.go:538`
  - body: For a lease claim whose VM was deleted externally, `resolveInstance` returns an error before `ReleaseLease` can remove the stale claim and key. That leaves `crabbox stop --provider tart <lease>` unable to clean up the broken record, so the release-only path needs to return enough lease information to prune missing-instance claims.
  - confidence: 0.84
