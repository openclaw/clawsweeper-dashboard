Full review comments:

- **[P1] Preserve lease state when Tart release really fails:** `internal/providers/tart/backend.go:272-276`
  - body: ReleaseLease discards both `tart stop` and `tart delete` errors and then removes the lease claim and stored key. That is safe only for a confirmed missing stale claim; for a real Tart failure it leaves the VM running or undeleted while Crabbox loses its local tracking. Tolerate only not-found/missing-VM cases and return other errors before clearing state.
  - confidence: 0.91
- **[P2] Honor keep when SSH readiness fails:** `internal/providers/tart/backend.go:160-164`
  - body: The `--keep` path now preserves VMs for `waitForIP` and `injectSSHKey` failures, but a later `prepareLease` error from `waitForSSHReady` still stops and deletes the VM unconditionally. That removes the started VM the user asked to keep for debugging; gate this cleanup on `!req.Keep` too, or document a different policy.
  - confidence: 0.84
