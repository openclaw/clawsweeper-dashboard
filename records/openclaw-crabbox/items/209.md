## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Preserve kept VMs after final readiness failures:** `internal/providers/tart/backend.go:162-163`
  - body: For `--keep` acquisitions, failures from `waitForIP` and `injectSSHKey` preserve the VM, but a later `prepareLease(..., wait=true)` failure still unconditionally stops and deletes it. That makes an SSH readiness timeout destroy a VM the user explicitly asked Crabbox to keep.
  - confidence: 0.92
- **[P1] Preserve claims when Tart delete fails for a live VM:** `internal/providers/tart/backend.go:273-276`
  - body: `ReleaseLease` ignores `tart delete` failures and then removes the lease claim and stored key. If deletion fails for anything other than a genuinely missing stale VM, Crabbox loses the state needed to manage the still-existing Tart VM.
  - confidence: 0.9
- **[P2] Treat running State as running when Running is omitted:** `internal/providers/tart/backend.go:596-599`
  - body: `Resolve` checks `State` when `Running` is false, but `prepareLease` still treats any empty-IP instance with `Running=false` as a non-SSH state response. Tart JSON that omits `Running` but reports `State: running` can therefore return a lease with no SSH target instead of failing clearly.
  - confidence: 0.88
