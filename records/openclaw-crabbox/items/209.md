## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Preserve generic Tart SSH settings:** `internal/cli/config.go:777-785`
  - body: Because `baseConfig()` pre-populates `Tart.User` and `Tart.WorkRoot`, this branch copies the provider defaults back over a user's top-level `ssh.user` and `workRoot` unless they duplicate the values under `tart:`. Preserve generic settings unless the Tart-specific fields were explicitly configured.
  - confidence: 0.93
- **[P1] Preserve kept Tart VMs after startup failures:** `internal/providers/tart/backend.go:133-134`
  - body: After `startVM` succeeds, every later failure path still stops and deletes the VM. That defeats `--keep` exactly when the operator needs to inspect a partially started local VM; guard the post-start cleanup with `!req.Keep`.
  - confidence: 0.94
- **[P1] Honor cancellation before kept Tart launches:** `internal/providers/tart/backend.go:410-414`
  - body: The keep path uses plain `exec.Command` and then waits only for process exit or the observe timeout, so a canceled context can still launch a detached Tart VM and return success. Check `ctx.Err()` before start and handle `ctx.Done()` during the observe window.
  - confidence: 0.91
- **[P2] Treat State=running as running when Running is omitted:** `internal/providers/tart/backend.go:185-186`
  - body: This rejects non-status resolve calls when the optional JSON `Running` bool is absent, even if Tart reports `State: "running"`. Use the existing normalized state helper so valid running VMs can be reused.
  - confidence: 0.92
- **[P2] Allow the documented Tart disk zero value:** `internal/providers/tart/flags.go:50-51`
  - body: The flag help and provider docs describe disk zero as clone-default, but this validation rejects `--tart-disk 0`. Allow zero as the sentinel and continue rejecting negative values.
  - confidence: 0.95
- **[P2] Let stop prune stale Tart claims whose VM is gone:** `internal/providers/tart/backend.go:538`
  - body: When a claim exists but the referenced VM no longer appears in `tart list`, release-only resolve errors before `ReleaseLease` can remove the local claim/key. Return a stale release target for this case and skip provider stop/delete for the missing VM.
  - confidence: 0.88
