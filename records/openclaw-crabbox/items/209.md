Full review comments:

- **[P2] Preserve kept VMs after post-start failures:** `internal/providers/tart/backend.go:133-135`
  - body: After `tart run` has started, the IP readiness path unconditionally stops and deletes the VM even when the user requested `--keep`. A transient DHCP/SSH readiness failure would destroy the local macOS VM that keep mode should leave available for inspection; mirror the Multipass/local-container pattern and only remove post-start resources when `!req.Keep`.
  - confidence: 0.9
- **[P2] Honor cancellation before detached Tart launch:** `internal/providers/tart/backend.go:409-414`
  - body: The keep path builds `exec.Command("tart", ...)` without checking `ctx` before `cmd.Start`, so a timed-out or canceled acquire can still launch a detached VM. Check the context before starting the kept process, and cover that path with a regression test.
  - confidence: 0.87
- **[P3] Align tart-disk help with validation:** `internal/providers/tart/flags.go:24`
  - body: The flag help promises `0 = use clone default`, but `ApplyFlags` rejects `--tart-disk 0`. Either accept explicit zero as clone-default or update the help/docs so users are not told to pass a value that errors out.
  - confidence: 0.92
