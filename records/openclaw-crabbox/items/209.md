## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Use Tart state when Running is absent:** `internal/providers/tart/backend.go:185-186`
  - body: `Running` defaults to false when the field is omitted from `tart list` JSON, so a VM with `State:"running"` can be rejected here as stopped for run/ssh/vnc/code/cache/egress. Reconcile this with `instanceRunning(inst.State)` instead of relying on the bool alone.
  - confidence: 0.9
- **[P1] Preserve kept VMs after startup failures:** `internal/providers/tart/backend.go:133-140`
  - body: After `tart run` has started the VM, these failure paths stop and delete it even when `req.Keep` is true. That defeats the operator's keep-for-inspection request on IP wait or SSH key injection failures; mirror the adjacent local-provider pattern and only remove the VM automatically when keep is false.
  - confidence: 0.88
- **[P2] Honor context cancellation for kept starts:** `internal/providers/tart/backend.go:409-414`
  - body: The keep=true path uses `exec.Command` and the startup select has no `ctx.Done()` branch, so a canceled acquisition can still launch a detached Tart VM. Check cancellation before starting and during the observation window, and avoid leaving a VM running from a canceled request.
  - confidence: 0.86
- **[P3] Align tart disk help with validation:** `internal/providers/tart/flags.go:24`
  - body: The flag help says `0 = use clone default`, but the parser rejects `--tart-disk 0` as invalid. Either accept explicit zero as the documented clone-default request or change the help text so users are not told to pass a value that fails.
  - confidence: 0.92
