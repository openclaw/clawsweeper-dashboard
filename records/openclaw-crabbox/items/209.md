## What This Changes

The PR adds a local `tart` SSH-lease provider for Apple Silicon macOS VMs, with CLI/YAML/env configuration, provider registration, status plumbing, tests, and provider docs.

## Best Possible Solution

Preserve real `tart delete` error propagation for VMs that still exist, but treat the already-missing instance path as stale local state and prune the claim plus stored SSH key without calling `tart delete`.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: resolve a Tart lease claim whose VM is absent, then release it; `resolveInstance` returns a missing instance and `ReleaseLease` returns the `tart delete` error before pruning local state.

## Solution Assessment

No. The current release path is not quite the best solution because it treats an already-missing VM like a real deletion failure; a narrow stale-state branch would keep retry safety for real VMs while allowing local state cleanup.
