## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Preserve generic SSH and workRoot config for Tart:** `internal/cli/config.go:777-785`
  - body: `baseConfig` gives `Tart.User` and `Tart.WorkRoot` non-empty defaults, and this block copies them back into the generic SSH/work-root fields. A config that sets only generic `ssh.user` or top-level `workRoot` is reset to `admin` and `/Users/admin/crabbox`; fall back to generic values when Tart-specific fields were not explicitly set, while keeping explicit `tart.user`/`tart.workRoot` higher precedence.
  - confidence: 0.86
- **[P2] Honor cancellation before launching kept VMs:** `internal/providers/tart/backend.go:410-434`
  - body: For `keep=true`, this starts `tart run` with `exec.Command` and then waits only for process exit or the startup timer. If the context is already canceled, or cancels during the observation window, Crabbox can still launch a retained local VM; check `ctx` before `Start` and include `ctx.Done()` in the startup wait so canceled warmups cannot leave abandoned Tart instances.
  - confidence: 0.84
- **[P2] Make --tart-disk 0 match the advertised clone-default behavior:** `internal/providers/tart/flags.go:49-51`
  - body: The flag help advertises `0 = use clone default`, and the docs describe disk resize as opt-in, but this branch rejects `--tart-disk 0` before the provider runs and also rejects an explicit zero disk in config validation. Either accept zero as the documented clone-default sentinel or change the help/docs/tests so users are not given a broken reset path.
  - confidence: 0.88
- **[P2] Treat a running State as running when the boolean is omitted:** `internal/providers/tart/backend.go:185-186`
  - body: `Running` is an optional JSON field from `tart list`, so omitted values decode to false even when `State` is `running`. This makes non-status Resolve reject a claimed VM that has a running state and IP; use the existing normalized state helper for this boundary so older or varied Tart JSON still works.
  - confidence: 0.82
- **[P2] Let stop prune stale claims whose Tart VM is gone:** `internal/providers/tart/backend.go:538`
  - body: When a local claim resolves to an `instance:` scope but the VM no longer appears in `tart list`, this path errors before `ReleaseOnly` can return a target for `ReleaseLease` to remove the stale claim and stored key. For release-only resolution, return a missing-instance target or otherwise prune the local claim without calling `tart stop/delete`.
  - confidence: 0.8
