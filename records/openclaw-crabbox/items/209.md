## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Bootstrap Crabbox readiness before waiting for SSH:** `internal/providers/tart/backend.go:544`
  - body: `prepareLease` waits with the default SSH ready check, which requires `/usr/local/bin/crabbox-ready`. The tart backend only injects the public key and never creates that script or prepares the work root, so a default VM can have working SSH but still time out before Crabbox returns a lease; add a macOS bootstrap step like the existing macOS paths or set a proven provider-specific ready check.
  - confidence: 0.89
- **[P1] Do not mount the guest work root as a host directory:** `internal/providers/tart/backend.go:385-386`
  - body: `tart run --dir` takes a host path and exposes it inside macOS under `/Volumes/My Shared Files/<tag>`, but this passes `cfg.WorkRoot`, which is Crabbox's guest work root. That can fail on hosts without `/Users/admin/crabbox` and it does not make files available at the configured guest work root; remove this mount or add an explicit host-mount config with guest-side setup.
  - confidence: 0.84
