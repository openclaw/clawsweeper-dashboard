## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Preserve kept VMs after SSH-readiness failures:** `internal/providers/tart/backend.go:162-164`
  - body: `prepareLease(..., wait=true)` performs the SSH readiness wait, but this error path always stops and deletes the VM. For `crabbox run --keep --provider tart`, a VM that already booted and had its key injected can still be destroyed when SSH readiness or local lease bookkeeping fails, which violates the keep behavior the PR documents. Gate these post-start cleanups on `!req.Keep`, matching the existing local-provider pattern.
  - confidence: 0.89
- **[P2] Prune missing-VM claims before returning delete errors:** `internal/providers/tart/backend.go:273-274`
  - body: When `resolveInstance` returns a stale claim for a VM that is already gone, `ReleaseLease` still runs `tart delete` and returns that error before removing the claim/key. That means `crabbox stop --provider tart <lease>` can fail to prune exactly the stale local state this PR says it handles. Skip deletion for `State == "missing"` or treat Tart's not-found delete as success before removing local lease state.
  - confidence: 0.82
