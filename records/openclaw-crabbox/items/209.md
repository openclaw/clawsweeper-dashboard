## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve generic SSH and work root overrides:** `internal/cli/config.go:777-785`
  - body: When `provider: tart` is selected, `baseConfig()` has already filled `cfg.Tart.User` and `cfg.Tart.WorkRoot`, so this block always copies `admin` and `/Users/admin/crabbox` over top-level `ssh.user` and `workRoot`. That makes a normal Crabbox config silently run as the wrong remote user/path unless the user also knows to set the Tart-specific keys; track explicit `tart.user`/`tart.workRoot` and let generic settings flow through when provider-specific values were not set.
  - confidence: 0.96
- **[P2] Allow zero disk to keep the clone default:** `internal/providers/tart/flags.go:49-51`
  - body: The flag help and provider docs define disk `0` as clone-default, and `configureVM` already treats explicit zero by skipping `--disk-size`, but this validation rejects `--tart-disk 0` before a run can start. Users also cannot reset an env/YAML disk override back to clone-default, so disk validation should reject only negative values and add regression coverage for the explicit-zero path.
  - confidence: 0.93
