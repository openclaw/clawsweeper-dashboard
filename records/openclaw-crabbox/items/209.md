- **[P1] Preserve generic SSH and work root overrides:** `internal/cli/config.go:777-785`
  - body: This branch initializes `cfg.Tart.User` and `cfg.Tart.WorkRoot` with provider defaults in `baseConfig`, so these checks are true even when the user only configured generic `ssh.user` or `workRoot`. Selecting `provider: tart` then rewrites values like `root` and `/custom/crabbox` back to `admin` and `/Users/admin/crabbox` before the provider runs; track explicit Tart-specific fields or inherit the generic values before applying defaults.
  - confidence: 0.93
- **[P2] Allow zero disk to keep the clone default:** `internal/providers/tart/flags.go:49-51`
  - body: The flag help and docs describe `--tart-disk 0` as clone-default behavior, and `configureVM` already skips `--disk-size` when `Disk == 0`. Rejecting zero here means users cannot reset a configured disk override back to clone default; accept zero and reject only negative disk values in both flag and config validation paths.
  - confidence: 0.95
