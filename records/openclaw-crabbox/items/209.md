Full review comments:

- **[P1] Preserve top-level SSH and work root overrides:** `internal/cli/config.go:777-785`
  - body: `baseConfig()` initializes `cfg.Tart.User` and `cfg.Tart.WorkRoot`, so this Tart branch always copies those defaults over `cfg.SSHUser` and `cfg.WorkRoot`. A config that selects Tart but sets the generic `ssh.user` or `workRoot` loses those values unless it also repeats them under `tart:`, which breaks the existing generic config contract for the new provider.
  - confidence: 0.91
- **[P2] Allow zero disk to keep the clone default:** `internal/providers/tart/flags.go:49-51`
  - body: The flag help and provider docs describe `--tart-disk 0` as clone-default behavior, and `configureVM` skips `--disk-size` unless the value is greater than zero. Rejecting zero here makes the documented reset path impossible, including when a user needs to override an env/config disk size back to the base image default.
  - confidence: 0.92
