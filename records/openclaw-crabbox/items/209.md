## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve Tart claim state when release fails:** `internal/providers/tart/backend.go:272-275`
  - body: `ReleaseLease` currently discards both `stopVM` and `deleteVM` errors and then removes the lease claim and stored SSH key. If `tart stop` or `tart delete` fails for a real VM, `crabbox stop` will report success while leaving the VM behind without the claim/key state needed for a normal retry; only confirmed missing-VM stale-claim pruning should tolerate delete failure.
  - confidence: 0.9
- **[P2] Honor keep leases when SSH readiness fails:** `internal/providers/tart/backend.go:160-164`
  - body: After `startVM` succeeds, `prepareLease(..., wait=true)` performs the SSH readiness wait; if that fails, this block always stops and deletes the VM. That means `crabbox run --keep --provider tart` can still destroy a started VM on readiness failure, which defeats the preservation behavior promised for kept leases.
  - confidence: 0.88
