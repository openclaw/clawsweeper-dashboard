## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve kept Tart VMs after readiness failures:** `internal/providers/tart/backend.go:132-140`
  - body: For `--keep` acquisitions, the VM has already been cloned and started by this point, but `waitForIP` and `injectSSHKey` failures still unconditionally stop and delete it. Adjacent local providers skip post-create cleanup when `req.Keep` is true, so users who asked to keep a failing VM for inspection will lose the very VM they need to debug.
  - confidence: 0.91
- **[P2] Check cancellation before detached Tart starts:** `internal/providers/tart/backend.go:409-410`
  - body: The keep path uses `exec.Command` and never checks `ctx` before `Start`, so if the acquisition context is canceled between configuration and `startVM`, Crabbox can still launch a detached Tart VM and then fail later, leaving an unexpected local VM behind. Check `ctx.Err()` before starting the detached process while still releasing the process after a successful start.
  - confidence: 0.86
- **[P3] Align tart-disk help with validation:** `internal/providers/tart/flags.go:24`
  - body: The flag help says `0 = use clone default`, but `applyFlags` rejects `--tart-disk 0` and the tests assert that rejection. Users following `--help` will get an error; either allow zero intentionally or change the help to say the flag should be omitted to use the clone default.
  - confidence: 0.96
