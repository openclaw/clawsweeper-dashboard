Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Preserve generic SSH user and work root:** `internal/cli/config.go:777-786`
  - body: When `provider: tart` is selected with only top-level `ssh.user` and `workRoot`, the prefilled Tart defaults are copied back over those generic settings. Track whether `tart.user`/`tart.workRoot` were explicitly set and only let the provider-specific values win when they were configured, otherwise existing generic Crabbox config is ignored.
  - confidence: 0.92
- **[P2] Allow documented clone-default disk reset:** `internal/providers/tart/flags.go:49-52`
  - body: The flag help says `--tart-disk 0` means use the clone default, but this branch rejects zero before the provider can apply that sentinel. Either accept zero and skip `tart set --disk-size`, or change the help/docs so users are not given a reset value that always fails.
  - confidence: 0.94
- **[P2] Preserve kept VMs after post-start failures:** `internal/providers/tart/backend.go:131-135`
  - body: After `startVM` succeeds, every later Acquire failure still stops and deletes the VM even when `req.Keep` is true. That removes the exact VM an operator asked to keep for inspection after IP, key injection, SSH readiness, claim, or endpoint failures; guard those post-start cleanup paths with `if !req.Keep`.
  - confidence: 0.93
- **[P2] Honor cancellation before starting kept VMs:** `internal/providers/tart/backend.go:405-414`
  - body: The keep path uses `exec.Command` so the Tart process can outlive Crabbox, but it never checks `ctx.Err()` before launch or listens for `ctx.Done()` during the startup observation window. A canceled acquire can still start a kept VM and return success, so keep mode needs explicit cancellation handling around the detached process.
  - confidence: 0.9
- **[P2] Use Tart state instead of the optional Running field:** `internal/providers/tart/backend.go:185-186`
  - body: `Running` defaults to false when the JSON field is omitted, so a claimed record with `State: "running"` can be rejected as stopped. Use the existing normalized `instanceRunning(inst.State)` helper for this Resolve gate so older or shape-different Tart output does not block valid SSH-target reuse.
  - confidence: 0.94
- **[P2] Let stop prune claim-backed missing VMs:** `internal/providers/tart/backend.go:532-538`
  - body: For a local claim whose referenced Tart VM is already gone, release-only resolution errors before `ReleaseLease` can remove the stale claim and stored key. Return a release target for claim-backed missing instances and let the release path prune local Crabbox state without calling `tart stop` or `tart delete`.
  - confidence: 0.88
