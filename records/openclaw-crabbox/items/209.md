Full review comments:

- **[P1] Return release errors before dropping Tart lease state:** `internal/providers/tart/backend.go:272-276`
  - body: ReleaseLease discards both `b.stopVM` and `b.deleteVM` errors, then removes claim/key state unconditionally. If `tart stop` or `tart delete` fails for an existing VM, `crabbox stop` reports success and loses the metadata needed to retry while the VM may still be running; only stale missing-VM pruning should bypass delete errors.
  - confidence: 0.94
- **[P2] Preserve kept Tart VMs after all post-start failures:** `internal/providers/tart/backend.go:162-163`
  - body: After `startVM` succeeds, the `waitForIP` and `injectSSHKey` failures respect `req.Keep`, but `prepareLease`, claim persistence, and endpoint update failures still stop and delete the VM unconditionally here. A `--keep` Tart acquisition can therefore lose the started VM on SSH-ready or claim persistence failure; guard these cleanup calls the same way as the adjacent local VM providers.
  - confidence: 0.86
