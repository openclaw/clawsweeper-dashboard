## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve generic Tart SSH and workRoot overrides:** `internal/cli/config.go:777-785`
  - body: `baseConfig` already fills `cfg.Tart.User` and `cfg.Tart.WorkRoot`, so this branch treats those defaults as explicit and copies them back into `cfg.SSHUser`/`cfg.WorkRoot`. A repo that sets the existing top-level `ssh.user` or `workRoot` with provider=tart will be silently reset to `admin` and `/Users/admin/crabbox`, causing key injection and sync to target the wrong account/path. Track whether Tart-specific values were explicit, or leave Tart defaults empty until generic overrides can win.
  - confidence: 0.93
- **[P2] Honor cancellation before detached Tart launch:** `internal/providers/tart/backend.go:410-414`
  - body: For `keep=true`, `startVM` uses `exec.Command` and does not check `ctx` before `Start` or while waiting through `startupObserveTimeout`. If the acquire context is already canceled or cancels during that window, Crabbox can still launch a detached VM and only notice later. Check `ctx.Err()` before starting and include `ctx.Done()` in the observe select so canceled acquires do not create new kept VMs.
  - confidence: 0.88
- **[P3] Align tart-disk help with validation:** `internal/providers/tart/flags.go:24`
  - body: The flag help says `--tart-disk 0` means clone default, but `applyFlags` rejects zero as an error. That makes the CLI self-contradictory for users following `--help`; either accept explicit zero as clone default or change the help/docs to say the flag must be positive and omitted for clone default.
  - confidence: 0.96
