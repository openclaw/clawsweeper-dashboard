- **[P1] Preserve claims when Tart delete fails:** `internal/providers/tart/backend.go:272-276`
  - body: `ReleaseLease` ignores `deleteVM` errors and then removes the lease claim and stored key. If `tart delete` fails for an existing VM, Crabbox forgets ownership while the VM remains, so normal stop/status cleanup loses the state it needs; tolerate only confirmed missing-VM stale claims, otherwise return the delete error and keep the claim/key.
  - confidence: 0.93
- **[P2] Treat state=running VMs without an IP as unavailable:** `internal/providers/tart/backend.go:595-599`
  - body: `Resolve` accepts Tart JSON where `Running` is omitted but `State` says `running`; however, `prepareLease` checks only `!inst.Running` when IP is empty and can return a lease with no SSH target. Use the same `instanceRunning(inst.State)` check here so SSH-target commands fail clearly instead of receiving an unusable target.
  - confidence: 0.88
