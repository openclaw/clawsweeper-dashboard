## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Propagate real delete failures before dropping lease state:** `internal/providers/tart/backend.go:273-276`
  - body: `ReleaseLease` ignores both `stopVM` and `deleteVM` errors and then removes the lease claim and stored SSH key. If `tart delete` fails for a real existing VM, Crabbox reports success while losing the state needed to manage that VM; tolerate only the stale missing-VM case before pruning state.
  - confidence: 0.96
- **[P2] Use State when handling a running VM with no IP:** `internal/providers/tart/backend.go:595-601`
  - body: `Resolve` checks `instanceRunning(inst.State)`, but `prepareLease` still treats empty IP as a stopped lease whenever `Running` is false. Tart JSON can omit `Running`, so a `State: "running"` VM with no IP can return a LeaseTarget with no SSH/error instead of failing clearly for SSH-target callers.
  - confidence: 0.93
- **[P2] Clean up kept startup cancellations consistently:** `internal/providers/tart/backend.go:433-437`
  - body: For `keep=true`, `startVM` starts `tart run` without a context-bound command, then returns on `ctx.Done()` without stopping the launched process. `Acquire` treats that as a pre-claim failure and only calls `deleteVM`, so cancellation during startup can leave a running VM without lease state.
  - confidence: 0.84
