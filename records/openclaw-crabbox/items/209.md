## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Preserve lease state when Tart deletion really fails:** `internal/providers/tart/backend.go:273-276`
  - body: `ReleaseLease` currently discards every `tart delete` error and then removes the lease claim and stored key. That is only safe when the VM is confirmed already missing; for real stop/delete failures, Crabbox forgets the lease while the VM may still exist, so cleanup and later stop commands lose the ownership state they need.
  - confidence: 0.91
- **[P2] Return no-IP errors for running VMs when Running is omitted:** `internal/providers/tart/backend.go:595-599`
  - body: `Resolve` allows a Tart instance whose `State` is `running`, but `prepareLease` then treats `Running == false` as stopped when the IP is empty. Tart JSON can omit `Running`, so non-status callers can receive a lease with no SSH target instead of a clear `tart instance has no IP address` error.
  - confidence: 0.87
