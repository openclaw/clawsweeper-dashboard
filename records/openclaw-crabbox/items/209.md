## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve generic Tart SSH settings:** `internal/providers/tart/backend.go:49-57`
  - body: Because `baseConfig()` pre-populates `Tart.User` and `Tart.WorkRoot`, this defaulting block always rewrites `cfg.SSHUser`/`cfg.WorkRoot` to `admin` and `/Users/admin/crabbox`. A config with top-level `ssh.user` or `workRoot` therefore shows and uses the wrong Tart account/root unless the user duplicates the setting under `tart:`, so keep the Tart-specific defaults from masking explicit generic settings.
  - confidence: 0.9
- **[P1] Preserve kept Tart VMs after post-start failures:** `internal/providers/tart/backend.go:133-134`
  - body: Once `tart run` has succeeded, the later IP, key injection, SSH readiness, and claim-update failure paths still stop and delete the VM even when the user requested `--keep`. That removes the exact VM state the operator asked to preserve for debugging; guard these cleanup calls the way adjacent local VM acquisition paths do after startup.
  - confidence: 0.93
- **[P1] Honor cancellation before keep-mode launch:** `internal/providers/tart/backend.go:409-414`
  - body: The keep path uses `exec.Command` and never checks `ctx` before starting `tart run`, so a canceled acquire can still launch a detached Tart process and only then notice cancellation in later steps. Use a cancellation-aware boundary for keep mode so timeouts and canceled runs do not start unwanted VMs.
  - confidence: 0.88
- **[P2] Treat state running as active without Running:** `internal/providers/tart/backend.go:185-186`
  - body: `Running` is optional JSON state from Tart, so when it is omitted Go leaves it false even if `State` is `running`. This rejects a valid claimed running VM for every non-status resolve path; use the normalized `instanceRunning(inst.State)` helper or derive `Running` from `State` before enforcing this check.
  - confidence: 0.92
- **[P2] Let stop prune stale claimed Tart leases:** `internal/providers/tart/backend.go:538`
  - body: When a lease claim exists but the referenced VM is already gone, release-only resolve returns an error here before the stop path can remove the stale claim and stored key. Return enough claim information for `crabbox stop --provider tart <lease>` to prune local state when the provider object is missing.
  - confidence: 0.84
- **[P2] Align the disk zero sentinel:** `internal/providers/tart/flags.go:49-52`
  - body: The flag help says `0 = use clone default`, but the parser rejects `--tart-disk 0`, so users cannot clear a configured disk resize back to the clone default from the CLI. Either accept explicit zero as the documented sentinel or remove that sentinel wording everywhere.
  - confidence: 0.86
