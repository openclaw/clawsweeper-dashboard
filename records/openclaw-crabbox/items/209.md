## Summary

Keep open: current main does not include a Tart provider, and this PR is useful and proof-backed, but the latest head still leaves stale Tart lease claims stuck when the VM is already gone because `ReleaseLease` calls `tart delete` before claim/key pruning.

## What This Changes

This PR adds a Tart local macOS VM SSH-lease provider, Tart config/flags/status plumbing, provider docs, and Tart-focused Go tests.

## Best Possible Solution

Land the Tart provider after `ReleaseLease` treats already-missing VMs as local claim/key cleanup while still propagating real delete failures for VMs that exist.

## Reproduction Assessment

Yes, from source inspection: a Tart claim whose provider scope points at a missing VM resolves to `State: "missing"`, then `crabbox stop --provider tart <lease>` reaches `ReleaseLease` and returns on `tart delete` before pruning local state.

## Solution Assessment

No, not quite: the provider design fits the existing SSH-lease adapter model, but the release path needs a narrow stale-claim fix before this is the maintainable solution.
