## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Preserve ownership state when Tart delete fails:** `internal/providers/tart/backend.go:273-276`
  - body: `ReleaseLease` ignores `tart delete` errors and then removes the lease claim and stored key. That is only safe for a known stale/missing VM; for a real delete failure it leaves the VM behind while Crabbox forgets the ownership state needed to stop or clean it later.
  - confidence: 0.91
- **[P2] Treat State=running as running when IP is missing:** `internal/providers/tart/backend.go:595-601`
  - body: `Resolve` now checks `instanceRunning(State)`, but `prepareLease` still decides the missing-IP path using only `inst.Running`. If Tart omits `Running` while `State` is `running`, non-status callers can receive a lease with no SSH target instead of the intended no-IP error.
  - confidence: 0.88
- **[P2] Honor keep after SSH readiness failures:** `internal/providers/tart/backend.go:160-164`
  - body: After the VM starts and the key is injected, a `prepareLease` or claim-update failure always stops and deletes the VM. That still violates `--keep` for post-start failures; gate this cleanup the same way the waitForIP and key-injection branches do.
  - confidence: 0.82
