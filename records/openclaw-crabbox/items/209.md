Overall confidence: 0.89

Full review comments:

- **[P1] Preserve generic Tart SSH user and work root:** `internal/providers/tart/backend.go:51-54`
  - body: When config selects `provider: tart` and sets top-level `ssh.user` or `workRoot`, `baseConfig` has already filled `cfg.Tart.User`/`WorkRoot`, so `applyDefaults` copies `admin` and `/Users/admin/crabbox` back over the generic settings. Preserve generic values unless `tart.user` or `tart.workRoot` was explicitly set, otherwise users run in the wrong account/path.
  - confidence: 0.91
- **[P1] Honor --keep after a Tart VM starts:** `internal/providers/tart/backend.go:123-124`
  - body: After `startVM` succeeds, the later `waitForIP`, SSH injection, lease preparation, and claim update failure branches unconditionally stop and delete the VM. That breaks `--keep` exactly when users need to inspect a partially provisioned local VM; guard post-start cleanup with `if !req.Keep`, matching adjacent local providers.
  - confidence: 0.9
- **[P2] Honor cancellation before starting kept Tart VMs:** `internal/providers/tart/backend.go:410`
  - body: The keep path uses plain `exec.Command` and then waits only for process exit or the fixed observation timer, so a canceled context can still launch a kept VM and return success. Check `ctx.Err()` before start and handle `ctx.Done()` during the observation window while preserving detach behavior after startup is accepted.
  - confidence: 0.88
- **[P2] Resolve running Tart VMs from State when Running is absent:** `internal/providers/tart/backend.go:179`
  - body: `Running` defaults to false when the field is omitted from `tart list --format json`, so a record with `State:"running"` can be rejected as stopped. Use the normalized `instanceRunning(inst.State)` boundary here so older or variant Tart JSON still resolves active claimed leases.
  - confidence: 0.88
- **[P2] Allow the documented clone-default disk reset:** `internal/providers/tart/flags.go:45-46`
  - body: The flag help says `--tart-disk 0` means use the clone default, but this validation rejects zero before the provider can skip `--disk-size`. Either accept zero as the documented sentinel or change the help/docs and config behavior so the CLI contract is not contradictory.
  - confidence: 0.86
- **[P2] Prune stale Tart claims when the VM is gone:** `internal/providers/tart/backend.go:538`
  - body: For a claim-backed lease whose Tart VM was already deleted, `resolveInstance` errors before the release-only path can return a target, so `crabbox stop --provider tart <lease-or-slug>` cannot remove the stale local claim/key. Add a release-only stale-claim path that prunes Crabbox state without calling `tart stop/delete` for a missing VM.
  - confidence: 0.86
