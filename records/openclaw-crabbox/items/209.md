## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Honor keep mode on all post-start failures:** `internal/providers/tart/backend.go:161-174`
  - body: The new waitForIP and injectSSHKey branches skip cleanup for keep=true, but errors after prepareLease, claimLeaseForRepoProviderScopePond, or updateLeaseClaimEndpoint still stop and delete the started VM unconditionally. That means a user who asked --keep can lose the VM during the final SSH readiness wait or claim updates, so guard these cleanup blocks the same way the earlier post-start failures are guarded.
  - confidence: 0.93
- **[P1] Preserve Tart claims when VM deletion fails:** `internal/providers/tart/backend.go:272-275`
  - body: ReleaseLease ignores both stopVM and deleteVM errors and then removes the claim/key. If tart delete fails for a live VM, Crabbox loses ownership while the VM remains, so only tolerate the stale missing-VM case and otherwise return the delete error before removing tracking.
  - confidence: 0.91
