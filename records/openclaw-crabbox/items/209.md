## Summary

Keep this PR open: the Tart provider is useful and has credible host proof, but the current head still mishandles stale Tart claims when the VM was deleted outside Crabbox, leaving `crabbox stop --provider tart <lease>` unable to prune local state.

## What This Changes

Adds a Tart local SSH-lease provider for Apple Silicon macOS VMs, including CLI/YAML/env configuration, provider registration/status plumbing, docs, and provider tests.

## Best Possible Solution

Keep the macOS Tart provider path, but make missing-instance release prune the local claim/key while preserving delete failure protection for VMs that still exist.

## Reproduction Assessment

Yes, from source inspection: a stale claim resolves to a missing Tart instance, `crabbox stop` passes that through `ReleaseOnly`, and `ReleaseLease` then calls `tart delete` before pruning the claim/key. I did not run Tart locally in this review.

## Solution Assessment

No, not quite yet: the provider direction is maintainable, but the release path needs a narrow missing-VM branch plus regression coverage before it is the best version of this fix.
