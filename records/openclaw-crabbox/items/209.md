Full review comments:

- **[P2] Honor cancellation before starting kept VMs:** `internal/providers/tart/backend.go:410-414`
  - body: When `keep` is true, this starts `tart run` with `exec.Command` and the startup wait below never checks `ctx.Done`. If the acquisition context is already canceled or times out during startup, Crabbox can still spawn a kept VM and then race cleanup/claiming, so the process should remain context-bound until the lease is safely handed off or cleaned up.
  - confidence: 0.86
- **[P3] Align disk help with validation:** `internal/providers/tart/flags.go:24`
  - body: The help text advertises `--tart-disk 0` as “use clone default”, but `ApplyFlags` rejects explicit zero. Users following `--help` will get an error, so either allow explicit zero or change the help to say the flag should be omitted for clone-default disk sizing.
  - confidence: 0.93
