## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve generic Tart SSH settings:** `internal/cli/config.go:1093-1099`
  - body: Because `baseConfig` pre-populates `Tart.User` and `Tart.WorkRoot`, the Tart defaulting path cannot tell defaults from explicit provider settings and later overwrites top-level `ssh.user`/`workRoot`. Existing users who set the generic SSH account or checkout path would silently run Tart leases as `admin` under `/Users/admin/crabbox` unless they duplicate the values under `tart:`.
  - confidence: 0.92
- **[P2] Allow the documented clone-default disk value:** `internal/providers/tart/flags.go:49-54`
  - body: The new flag help says `--tart-disk 0` means use the clone default, but this branch rejects zero before provisioning starts. That prevents users from overriding a configured disk size back to clone-default and makes the CLI contract disagree with the provider docs.
  - confidence: 0.96
- **[P1] Preserve kept Tart VMs after post-start failures:** `internal/providers/tart/backend.go:131-140`
  - body: After `startVM` succeeds, every later acquisition failure still stops and deletes the VM even when `req.Keep` is true. That defeats the main debugging value of `--keep`: if IP readiness, key injection, SSH readiness, claim writing, or endpoint update fails, the started VM is removed before the operator can inspect it.
  - confidence: 0.9
- **[P1] Honor cancellation before keep-mode launch:** `internal/providers/tart/backend.go:409-435`
  - body: The keep-mode branch uses `exec.Command` and the observation select does not check `ctx.Err()` or `ctx.Done()`, so a canceled acquire context can still launch a detached Tart VM and return success after the timer. Keep mode can still detach the process, but it needs to honor cancellation before and during startup observation.
  - confidence: 0.88
- **[P2] Treat running state as active without Running:** `internal/providers/tart/backend.go:185-187`
  - body: `Running` is an optional JSON bool and defaults to false when Tart omits it, even when `State` says `running`. This check would reject claimed running VMs for non-status commands, so reuse paths should use the normalized `instanceRunning(inst.State)` helper instead.
  - confidence: 0.94
- **[P2] Let stop prune stale claimed Tart leases:** `internal/providers/tart/backend.go:538`
  - body: When a local claim exists but the referenced Tart VM is already gone, release-only resolution errors here before `ReleaseLease` can remove the stale claim and stored key. Stop by lease or slug should be able to clear Crabbox-owned local state without trying to stop/delete a missing VM.
  - confidence: 0.87
