## Best Possible Solution

Land the macOS Tart provider after ReleaseLease prunes stale missing-VM claims without hiding real delete failures for existing VMs, and track Ubuntu Tart support separately if maintainers want it.

## Reproduction Assessment

Yes for the review blocker: source inspection shows resolveInstance can return a stale claim with State="missing", and ReleaseLease then returns the tart delete error before pruning claim/key state. A focused unit test can reproduce it with an empty tart list and a lease-only ReleaseLease request.

## Solution Assessment

No, not yet; the provider design is generally aligned, but the stop/release path should special-case proven missing VMs and prune local state while preserving real delete-error propagation for live VMs.
