Full review comments:

- **[P1] Preserve generic SSH and work root overrides:** `internal/cli/config.go:777-785`
  - body: Because baseConfig pre-populates cfg.Tart.User and cfg.Tart.WorkRoot, this branch treats built-in Tart defaults as provider-specific user input and overwrites generic ssh.user/workRoot. A config with provider: tart plus top-level ssh.user/workRoot still resolves to admin and /Users/admin/crabbox unless tart.user/workRoot are also set, so existing generic config conventions are ignored. Track explicit Tart user/workRoot or let generic values flow before applying provider defaults.
  - confidence: 0.93
- **[P2] Allow zero disk to keep the clone default:** `internal/providers/tart/flags.go:50-51`
  - body: The flag help and provider docs describe zero as the clone-default disk sentinel, but --tart-disk 0 is rejected here before the explicit marker is useful, and the later cfg.Tart.Disk == 0 validation rejects it too. Users cannot clear an env/YAML disk override back to clone default even though the provider contract says they can; reject only negative disk values and cover explicit zero.
  - confidence: 0.91
