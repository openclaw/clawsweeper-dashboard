Full review comments:

- **[P1] Preserve generic Tart SSH settings:** `internal/cli/config.go:777-785`
  - body: `applyProviderConfigDefaults` now always copies the Tart defaults into `cfg.SSHUser` and `cfg.WorkRoot`, so a config that sets generic `ssh.user` or top-level `workRoot` for an SSH provider is silently reset to `admin` and `/Users/admin/crabbox` unless the user also sets `tart.user` or `tart.workRoot`. Please only override those generic fields when the Tart-specific value was explicitly configured, and add coverage for the generic config path.
  - confidence: 0.9
- **[P1] Preserve kept VMs after post-start failures:** `internal/providers/tart/backend.go:132-140`
  - body: After `tart run` succeeds, the readiness and key-injection failure paths always stop and delete the VM even when `req.Keep` is true. That differs from the adjacent local providers and defeats `--keep`/warmup inspection for Tart startup failures; guard these cleanup calls with `!req.Keep` and cover the behavior.
  - confidence: 0.88
- **[P2] Honor cancellation before detached Tart starts:** `internal/providers/tart/backend.go:410-417`
  - body: The keep=true path uses `exec.Command` and the startup observation select does not watch `ctx.Done()`, so a canceled acquisition can still launch a detached `tart run` before the later cleanup path notices. Check the context before starting the process and while observing startup, with a regression test for canceled kept starts.
  - confidence: 0.84
