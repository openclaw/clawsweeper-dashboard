## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve generic Tart SSH user and work root:** `internal/providers/tart/backend.go:49-57`
  - body: When `provider: tart` is selected with top-level `ssh.user` or `workRoot`, these defaults are already set to `admin` and `/Users/admin/crabbox`, so `applyDefaults` overwrites the user's generic settings before the VM is created. That sends key injection, SSH, and sync to the wrong account/path unless the user duplicates the values under `tart.*`; seed Tart from the generic fields unless a Tart-specific override was explicitly configured.
  - confidence: 0.93
- **[P1] Preserve kept Tart VMs after startup failures:** `internal/providers/tart/backend.go:133-135`
  - body: After `tart run` has started, failures while waiting for IP, injecting the key, or preparing the lease always stop and delete the VM. That contradicts the existing local-provider `keep=true` pattern and destroys the VM a user asked to keep for debugging; guard these cleanup calls with `!req.Keep` after the VM has been created.
  - confidence: 0.91
- **[P2] Honor cancellation before starting kept Tart VMs:** `internal/providers/tart/backend.go:410`
  - body: The `keep=true` branch uses `exec.Command` and the startup observation select has no `ctx.Done()` case, so a canceled acquire context can still launch a detached Tart VM and return success after the timer. Check `ctx.Err()` before `Start` and abort/kill during the observation window if cancellation arrives.
  - confidence: 0.9
- **[P2] Treat Tart State=running as running when Running is omitted:** `internal/providers/tart/backend.go:185-186`
  - body: `tartInstance.Running` defaults to false when Tart JSON omits that field, and `Resolve` rejects non-status callers from that bool alone. A VM with `State: "running"` but no `Running` field is therefore rejected as stopped; use the normalized State fallback when deciding whether the instance is running.
  - confidence: 0.9
- **[P2] Allow the documented Tart disk zero value:** `internal/providers/tart/flags.go:50-51`
  - body: The flag help says `--tart-disk` accepts `0` to use the clone default, but the apply path rejects `--tart-disk 0` as invalid. Either allow explicit zero consistently for CLI/env/YAML or change the docs and flag help so the documented contract matches runtime behavior.
  - confidence: 0.92
- **[P2] Let stop prune Tart claims whose VM is gone:** `internal/providers/tart/backend.go:538`
  - body: When a local Tart claim resolves to an instance that no longer exists in `tart list`, `resolveInstance` returns an error before the release-only stop path can remove the stale local claim and per-lease key. For release-only resolution, return a missing-instance lease target so `ReleaseLease` can prune local state without calling `tart stop/delete`.
  - confidence: 0.88
