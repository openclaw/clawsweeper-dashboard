Full review comments:

- **[P2] Report stopped leases without requiring an IP:** `internal/providers/tart/backend.go:567-568`
  - body: `crabbox status` uses `Resolve` for SSH-lease providers that do not implement `Status`, but Tart resolves stopped VMs with an empty IP and then `prepareLease` fails before returning the known stopped state. Stopped Tart VMs normally have no IP, so status should return a non-ready stopped view while keeping run/hydrate paths strict about needing an SSH target.
  - confidence: 0.9
- **[P2] Validate Tart resource floors before cloning:** `internal/providers/tart/flags.go:35-45`
  - body: The provider only rejects non-positive CPU and memory values, so values like `--tart-cpu 1` or very small memory are accepted by Crabbox and fail later in `tart set` after the VM has already been cloned. Official Tart performs below-minimum resource checks in `set`, so reject these before provisioning and cover CLI/env/YAML config paths.
