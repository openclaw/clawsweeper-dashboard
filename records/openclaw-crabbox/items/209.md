## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Preserve generic Tart SSH and work-root overrides:** `internal/providers/tart/backend.go:49-67`
  - body: Because `baseConfig` already fills `Tart.User` and `Tart.WorkRoot`, this defaulting path overwrites generic `ssh.user` and top-level `workRoot` with `admin` and `/Users/admin/crabbox` whenever Tart-specific fields are unset. Users who rely on the existing generic settings will silently get a different account/path for Tart leases.
  - confidence: 0.91
- **[P1] Use Tart state when Running is omitted:** `internal/providers/tart/backend.go:185-186`
  - body: `Running` unmarshals to false when Tart omits that optional field, so a JSON item like `{State:"running"}` is rejected here as stopped. Resolve/status reuse should derive running state from `State` when the boolean is absent or make the field nullable.
  - confidence: 0.9
- **[P1] Preserve kept VMs after post-start failures:** `internal/providers/tart/backend.go:132-140`
  - body: After a kept VM has been started, the IP, key injection, prepare, and claim error paths still stop and delete it. That defeats `--keep` for the exact failure cases where the user needs the VM left around for inspection.
  - confidence: 0.88
- **[P1] Honor cancellation before detached Tart starts:** `internal/providers/tart/backend.go:409-418`
  - body: The keep path uses `exec.Command` without checking `ctx`, so a canceled acquire can still launch a detached Tart VM and then return as if the caller stopped waiting. Check cancellation before starting or use context-aware startup until detach is safe.
  - confidence: 0.86
- **[P2] Let stop prune claim-backed missing Tart VMs:** `internal/providers/tart/backend.go:252-254`
  - body: When release has only a lease ID, `resolveInstance` returns an error if the VM is already gone, so this exits before removing the stale lease claim and stored key. A stopped/deleted Tart VM should still allow local claim cleanup for that lease.
  - confidence: 0.84
- **[P3] Align the Tart disk zero contract:** `internal/providers/tart/flags.go:24-51`
  - body: The flag help says `0 = use clone default`, but the parser rejects zero as invalid. Either accept an explicit zero as clone-default or remove that contract from the CLI help and docs so users do not get contradictory behavior.
  - confidence: 0.82
