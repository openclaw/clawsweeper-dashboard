## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Honor cancellation before starting kept VMs:** `internal/providers/tart/backend.go:410-414`
  - body: For kept leases this path uses exec.Command and never checks ctx before Start or while waiting out the startup observation window. A canceled acquire can still launch a detached Tart VM after the caller has given up, leaving cleanup to best-effort recovery instead of preventing the side effect.
  - confidence: 0.9
- **[P2] Use Tart state when deciding Resolve is running:** `internal/providers/tart/backend.go:185-186`
  - body: Resolve rejects non-status callers on !inst.Running, but Running is an optional JSON field and decodes to false when omitted. A Tart list entry with State:"running" but no Running field will be treated as stopped, blocking reuse of an otherwise running claimed lease.
  - confidence: 0.92
- **[P2] Let stop prune stale Tart claims:** `internal/providers/tart/backend.go:252-254`
  - body: When stop resolves by lease ID, a claim whose VM has already disappeared makes resolveInstance return an error before ReleaseLease reaches removeLeaseClaim. That leaves the stale local claim and key stuck even though stop is the command users would expect to clean it up.
  - confidence: 0.9
- **[P3] Align the tart disk flag help with validation:** `internal/providers/tart/flags.go:24`
  - body: The flag help says --tart-disk 0 uses the clone default, but applyFlags rejects zero as invalid. Either accept explicit zero as the documented no-resize value or update the help so users are not told to pass a value the CLI refuses.
  - confidence: 0.86
