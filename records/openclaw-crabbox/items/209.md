Full review comments:

- **[P1] Treat State=running as running when no Tart IP exists:** `internal/providers/tart/backend.go:604-607`
  - body: This branch returns a successful LeaseTarget with no SSH target whenever `tart ip` is empty and the JSON `Running` bool is omitted. Tart list entries can still report `State: "running"`, so SSH-target callers can resolve a claimed running VM into an empty endpoint instead of getting the intended no-IP error; use `inst.Running || instanceRunning(inst.State)` here and add that regression case.
  - confidence: 0.9
- **[P2] Fully detach kept Tart run processes:** `internal/providers/tart/backend.go:427-439`
  - body: For `--keep`, this starts `tart run` with stderr connected to an in-process buffer and keeps `cmd.Wait()` running in a goroutine, but it never releases the process or detaches stdio despite the comment and PR body saying kept VMs are fully detached. A kept VM can remain tied to the Crabbox process lifetime/log pipe; use an actual detached process setup before returning success.
  - confidence: 0.77
