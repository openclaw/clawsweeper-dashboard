## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve claims when Tart release fails:** `internal/providers/tart/backend.go:272-276`
  - body: `ReleaseLease` currently ignores both `tart stop` and `tart delete` failures and then removes the lease claim and stored key. If either command fails for a real VM, Crabbox loses the retry metadata while the VM may still be running; only the confirmed stale/missing-VM case should prune local state.
  - confidence: 0.93
- **[P2] Fail running Tart leases that have no IP:** `internal/providers/tart/backend.go:595-600`
  - body: This branch treats an empty IP as a stopped lease whenever `inst.Running` is false, but Tart JSON can omit `Running` while `State` is still `running`. That lets non-status callers receive a `LeaseTarget` with no SSH target instead of a clear no-IP error; use the same `instanceRunning(inst.State)` fallback here.
  - confidence: 0.88
- **[P2] Honor keep after SSH readiness failures:** `internal/providers/tart/backend.go:161-164`
  - body: After a kept VM starts and key injection succeeds, `prepareLease` runs the final SSH readiness check. If that check fails, this cleanup path still stops and deletes the VM even when `req.Keep` is true, contradicting the provider's kept-lease behavior.
  - confidence: 0.87
