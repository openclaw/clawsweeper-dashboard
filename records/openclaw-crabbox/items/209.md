## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Honor keep after late acquire failures:** `internal/providers/tart/backend.go:162-164`
  - body: After `startVM` succeeds, `prepareLease` can still fail while waiting for SSH readiness, but this branch stops and deletes the VM even when the user requested `--keep`; the claim and endpoint failure branches below repeat the same pattern. That defeats the keep contract exactly when an operator needs the partially provisioned VM for inspection.
  - confidence: 0.93
- **[P1] Preserve claims when Tart release cleanup fails:** `internal/providers/tart/backend.go:272-276`
  - body: `ReleaseLease` ignores both `stopVM` and `deleteVM` errors and then removes the lease claim and stored SSH key. If Tart refuses to stop/delete a real VM because it is busy or permissions fail, Crabbox loses the only local state for a still-live VM; only confirmed missing-instance stale claims should be pruned this way.
  - confidence: 0.95
- **[P2] Treat running-by-state VMs without IP as unresolved:** `internal/providers/tart/backend.go:595-600`
  - body: When Tart omits the `Running` JSON field but reports `State: running`, an empty `tart ip` result enters this `!inst.Running` branch and returns a successful lease with no SSH target. `Resolve` already treats `State` as running, so non-status callers should get a clear no-IP error until the VM has a usable address.
  - confidence: 0.78
