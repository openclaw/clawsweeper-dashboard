## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve state when Tart delete fails:** `internal/providers/tart/backend.go:273`
  - body: `ReleaseLease` ignores every `tart delete` error and then removes the lease claim and stored key. If the delete fails for a real VM, Crabbox will report release success while losing the state needed to retry cleanup, leaving an undeleted VM behind. Only tolerate a known stale/missing-VM delete failure; otherwise return the error and keep the claim/key.
  - confidence: 0.93
- **[P2] Stop kept VMs on startup cancellation:** `internal/providers/tart/backend.go:433-437`
  - body: In `keep` mode this branch returns on `ctx.Done()` after starting a plain `exec.Command`, but it neither kills/stops the VM nor records a claim before acquire reports failure. If cancellation lands during the startup observation window, a Tart VM can continue running unmanaged. Stop it on this failure path or detach only after startup observation succeeds.
  - confidence: 0.82
