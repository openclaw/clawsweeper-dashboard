## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Honor cancellation before kept Tart starts:** `internal/providers/tart/backend.go:410`
  - body: When `keep` is true, `startVM` uses `exec.Command` and the startup select does not observe `ctx.Done()`. A canceled acquisition can still launch a detached local VM and return success after the observe timeout; check context before `Start` and during the observe window without tying the kept child lifetime to the canceled context.
  - confidence: 0.91
- **[P1] Preserve kept VMs after post-start failures:** `internal/providers/tart/backend.go:133-139`
  - body: After `tart run` succeeds, IP wait and SSH-key injection failures always stop and delete the VM. Existing local providers guard post-create cleanup with `!req.Keep`, so this removes the VM a `--keep` user expected to inspect after a failed acquisition.
  - confidence: 0.88
- **[P1] Use Tart state when Running is omitted:** `internal/providers/tart/backend.go:185`
  - body: `Running` is an optional decoded bool; if `tart list` returns `State: "running"` without `Running`, Go decodes false and non-status resolve rejects a valid lease as stopped. Normalize runnability from `State` or synthesize `Running` after decode.
  - confidence: 0.9
- **[P3] Align the Tart disk zero contract:** `internal/providers/tart/flags.go:24`
  - body: The flag help advertises `--tart-disk 0` as the clone-default sentinel, but validation rejects explicit zero. Either accept zero as documented or remove that promise from the CLI help so users do not get a contradictory contract.
  - confidence: 0.86
