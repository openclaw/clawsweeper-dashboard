## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve kept VMs after post-start failures:** `internal/providers/tart/backend.go:131-140`
  - body: After `startVM` succeeds, `waitForIP` and `injectSSHKey` failures still stop and delete the VM even when the user passed `--keep`. That defeats keep-mode debugging and differs from the adjacent local providers, which only remove post-create resources on failure when `!req.Keep`.
  - confidence: 0.91
- **[P1] Honor cancellation before detached Tart starts:** `internal/providers/tart/backend.go:409-415`
  - body: The keep path uses `exec.Command` and never checks `ctx` before `cmd.Start`, so a canceled acquire can still launch a local Tart VM after the caller has already given up. Check cancellation before starting the detached process, while still letting an allowed keep-mode VM survive Crabbox exit.
  - confidence: 0.87
- **[P2] Use Tart state when Running is absent:** `internal/providers/tart/backend.go:185-186`
  - body: `Running` defaults to false when the Tart JSON field is omitted, but the state string can still say the VM is running. This branch rejects those leases for non-status resolves, so a real running claimed VM can become unusable through `run`, `ssh`, `code`, `cache`, or `egress`; use the normalized state as the fallback.
  - confidence: 0.86
- **[P2] Make disk-zero help match validation:** `internal/providers/tart/flags.go:24`
  - body: The flag help says `--tart-disk` accepts `0 = use clone default`, but `applyFlags` rejects zero as an error. Users following `--help` get a validation failure, so either accept zero as explicit clone-default or change the help text and tests to require omission for clone-default.
  - confidence: 0.94
- **[P2] Preserve generic SSH user and work root:** `internal/providers/tart/backend.go:49-67`
  - body: Tart defaults are already populated in `baseConfig`, so `applyDefaults` always overwrites `cfg.SSHUser` and `cfg.WorkRoot` with Tart's defaults even when the user set generic `ssh.user` or top-level `workRoot` and did not set `tart.user` or `tart.workRoot`. That makes provider-neutral Crabbox settings ineffective for this provider.
  - confidence: 0.78
