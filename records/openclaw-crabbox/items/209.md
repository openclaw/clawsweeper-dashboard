## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve kept VMs when SSH readiness fails:** `internal/providers/tart/backend.go:162-164`
  - body: `prepareLease` performs the SSH readiness wait. If that wait fails for a `--keep` acquisition, this block still stops and deletes the VM, unlike the earlier waitForIP/injectSSHKey cleanup paths that honor `req.Keep`. That destroys the kept VM the user asked to retain for debugging or later reuse.
  - confidence: 0.93
- **[P1] Do not drop claims after real delete failures:** `internal/providers/tart/backend.go:273-275`
  - body: `ReleaseLease` ignores every `deleteVM` error and then removes the lease claim and stored key. That is safe only for a missing stale VM; for a real Tart delete failure, Crabbox loses the state needed to retry cleanup while the VM remains on the host.
  - confidence: 0.91
- **[P2] Use State when a no-IP VM may still be running:** `internal/providers/tart/backend.go:596-601`
  - body: This no-IP branch treats any instance with `Running=false` as stopped, but Tart can omit the `Running` field while `State` is still `running`. In that case Resolve can return a LeaseTarget without SSH instead of failing with the intended no-IP error.
  - confidence: 0.86
