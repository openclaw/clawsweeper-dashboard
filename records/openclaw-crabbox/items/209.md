Full review comments:

- **[P1] Preserve generic Tart user and work-root overrides:** `internal/cli/config.go:777-785`
  - body: Because baseConfig pre-populates Tart.User and Tart.WorkRoot, this branch copies admin and /Users/admin/crabbox back over a user's generic ssh.user and top-level workRoot. That makes existing Crabbox config conventions silently ineffective for provider=tart unless users duplicate the values under tart:.
  - confidence: 0.93
- **[P1] Preserve kept VMs after post-start failures:** `internal/providers/tart/backend.go:132-135`
  - body: After tart run succeeds, every later acquisition failure still stops and deletes the VM, even when the caller used --keep. That removes the local VM exactly when the operator needs to inspect DHCP, SSH injection, readiness, or claim-writing failures; the adjacent Multipass provider guards comparable post-create cleanup with !req.Keep.
  - confidence: 0.95
- **[P1] Honor cancellation before detached Tart starts:** `internal/providers/tart/backend.go:409-434`
  - body: The keep=true path uses exec.Command and then waits only for process exit or the fixed observation timer. If the acquire context is already canceled or cancels during this window, the provider can still launch a detached Tart VM and report startup success instead of returning the context error.
  - confidence: 0.9
- **[P1] Use Tart state when Running is omitted:** `internal/providers/tart/backend.go:185-186`
  - body: Resolve rejects non-status callers based on inst.Running, but that bool is false when Tart JSON omits the field even if State is "running". Use the normalized state helper here so claimed running leases are not incorrectly rejected as stopped.
  - confidence: 0.94
- **[P2] Let release-only stop prune missing Tart claims:** `internal/providers/tart/backend.go:538`
  - body: A claim-backed stop fails in resolveInstance when the referenced VM is already absent from tart list, so ReleaseLease never gets a target and the stale local claim/key remain. Release-only resolution should return a missing-instance target for claim-owned leases and prune local state without calling tart stop/delete.
  - confidence: 0.86
- **[P3] Align tart disk zero validation with help text:** `internal/providers/tart/flags.go:24`
  - body: The flag help advertises --tart-disk 0 as clone default, but applyFlags rejects zero before run starts. Either allow zero as the documented sentinel or change the help/docs so users are not given a failing command.
  - confidence: 0.92
