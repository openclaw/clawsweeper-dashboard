## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve generic SSH and workRoot config for Tart:** `internal/cli/config.go:777-785`
  - body: `baseConfig()` pre-fills `Tart.User` and `Tart.WorkRoot`, and this branch then copies those values over `cfg.SSHUser` and `cfg.WorkRoot`. A generic config such as `ssh.user: root` plus top-level `workRoot: /custom/crabbox` would be silently replaced with `admin` and `/Users/admin/crabbox`; Tart-specific values should win only when explicitly set.
  - confidence: 0.9
- **[P2] Honor cancellation before launching kept VMs:** `internal/providers/tart/backend.go:410`
  - body: When `keep` is true this path uses `exec.Command` and the startup observation select only waits for process exit or the 2s timer. If the acquire context is already canceled, Crabbox can still launch `tart run` and return nil before later cleanup, which is the path where a local VM can be left running after the caller has abandoned the operation.
  - confidence: 0.86
- **[P3] Align tart-disk help with validation:** `internal/providers/tart/flags.go:24`
  - body: The flag help advertises `0 = use clone default`, but the validation below rejects `--tart-disk 0` as invalid. Users following `--help` will get an error, so either the help text should drop the zero claim or explicit zero should be accepted consistently.
  - confidence: 0.95
