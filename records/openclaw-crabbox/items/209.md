## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Preserve generic Tart SSH settings:** `internal/providers/tart/backend.go:65-68`
  - body: When `tart.user` or `tart.workRoot` are not explicitly configured, the provider defaults still overwrite top-level `ssh.user` and `workRoot`, so a generic Crabbox config can silently run as `admin` under `/Users/admin/crabbox`. Preserve the generic values unless the Tart-specific fields were explicitly set.
  - confidence: 0.94
- **[P1] Preserve kept Tart VMs after post-start failures:** `internal/providers/tart/backend.go:133-135`
  - body: After `startVM` succeeds, every later acquisition failure still stops and deletes the VM, even when the user requested `--keep`. That removes the local VM in exactly the failure cases where the operator needs to inspect it; guard the post-start cleanup with `!req.Keep`.
  - confidence: 0.93
- **[P1] Honor cancellation before keep-mode Tart launch:** `internal/providers/tart/backend.go:410`
  - body: The keep path uses plain `exec.Command` and the startup observation select never watches `ctx.Done()`, so a canceled acquire can still launch a detached Tart VM and return success after the timer. Check the context before starting and during the observation window, killing the process if cancellation wins.
  - confidence: 0.92
- **[P2] Treat State=running as active without Running:** `internal/providers/tart/backend.go:185-186`
  - body: `Running` is an optional JSON bool, so when Tart omits it this check treats a `State:"running"` VM as stopped and blocks reuse through non-status Resolve callers. Use the normalized state helper for the running/stopped boundary.
  - confidence: 0.95
- **[P2] Allow --tart-disk 0 to mean clone default:** `internal/providers/tart/flags.go:50-51`
  - body: The flag help and docs describe `0` as the clone-default sentinel, but this validation rejects `--tart-disk 0` before the provider can skip `--disk-size`. Reject negative values only, and cover CLI/env/YAML explicit-zero behavior consistently.
  - confidence: 0.94
- **[P2] Let release-only stop prune stale Tart claims:** `internal/providers/tart/backend.go:538`
  - body: When a claim resolves to a VM that is already gone, `resolveInstance` errors before `ReleaseLease` can remove the local claim and key. For release-only resolution, return a stale target so stop can prune Crabbox state without calling Tart stop/delete for a missing VM.
  - confidence: 0.9
