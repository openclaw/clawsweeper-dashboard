## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Propagate Tart delete failures before removing claims:** `internal/providers/tart/backend.go:272-275`
  - body: `ReleaseLease` currently discards both `stopVM` and `deleteVM` errors, then removes the lease claim and stored key. If `tart delete` fails for a live VM, Crabbox reports release success and forgets the lease while the VM remains on the host. Keep the claim and return the delete error except for an explicitly confirmed stale/missing-VM pruning path.
  - confidence: 0.91
- **[P2] Treat state=running as running when IP is missing:** `internal/providers/tart/backend.go:595-601`
  - body: This branch only checks `inst.Running`, but Tart JSON can omit `Running` while still reporting `State: "running"`. In that case a non-status resolve with an empty or `--` IP can return a `LeaseTarget` with no SSH target instead of failing with the no-IP error. Use the same `instanceRunning(inst.State)` fallback here before allowing the stopped-status path.
  - confidence: 0.87
