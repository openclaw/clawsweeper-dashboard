## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve kept VMs after SSH readiness failures:** `internal/providers/tart/backend.go:162-163`
  - body: `Acquire` now preserves `--keep` VMs for `waitForIP` and `injectSSHKey` failures, but if `prepareLease` fails while waiting for SSH readiness this unconditional stop/delete still destroys the VM. A kept Tart VM that boots but never becomes SSH-ready should be left for debugging/reuse, matching the PR's kept-lease semantics and adjacent provider patterns.
  - confidence: 0.88
- **[P1] Preserve claims when Tart deletion fails:** `internal/providers/tart/backend.go:272-275`
  - body: `ReleaseLease` discards both `tart stop` and `tart delete` errors and then removes the local lease claim/key. If `tart delete` fails for an existing VM, Crabbox loses the only claim for a still-present VM, so later list/stop/cleanup cannot manage it as the same lease; only the stale/not-found case should be tolerated.
  - confidence: 0.9
- **[P2] Treat running State as running when IP is absent:** `internal/providers/tart/backend.go:595-601`
  - body: This branch checks only the optional `Running` boolean before returning a stopped-style lease with no SSH target. Tart JSON can omit `Running` while `State` is `running`, so `Resolve` can hand `run`, `ssh`, or other SSH-target commands a LeaseTarget with an empty SSH host instead of failing with a clear no-IP error.
  - confidence: 0.86
