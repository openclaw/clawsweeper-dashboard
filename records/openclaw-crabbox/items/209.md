## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Honor cancellation before launching kept Tart VMs:** `internal/providers/tart/backend.go:410-414`
  - body: For `keep=true`, this uses plain `exec.Command` and the select below never watches `ctx.Done()`. If the acquire context is already canceled or is canceled during the startup observation window, Crabbox can still launch a detached Tart VM and return nil, leaving local VM state behind a canceled operation. Check `ctx.Err()` before `Start`, add a cancellation case, and kill/reap the process before returning the context error.
  - confidence: 0.91
- **[P3] Make disk help match the validation contract:** `internal/providers/tart/flags.go:24`
  - body: The flag help still says `0 = use clone default`, but this same function rejects `--tart-disk 0` as invalid. Users following `--help` will get an error; the help should say to omit the flag to keep the clone default, or the validation should accept zero consistently.
  - confidence: 0.97
