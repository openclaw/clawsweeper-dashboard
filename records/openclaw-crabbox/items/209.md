## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve claim state when Tart release fails:** `internal/providers/tart/backend.go:272-276`
  - body: `ReleaseLease` ignores both `tart stop` and `tart delete` errors, then removes the lease claim and stored key. If Tart fails because the VM is busy, the CLI is broken, or the delete only partially succeeds, Crabbox reports release success and loses the retry state while the VM can remain running; return real stop/delete errors before state removal, with only a narrow already-missing-VM exception for stale claim pruning.
  - confidence: 0.91
- **[P2] Treat running-state VMs as running when IP is missing:** `internal/providers/tart/backend.go:595-599`
  - body: This IP guard only checks `inst.Running`, but the branch already accounts for Tart JSON where `Running` is omitted and `State` is `running`. In that case an empty or `--` IP returns a LeaseTarget with no SSH target instead of failing non-status resolves, so SSH-target commands can proceed with unusable connection data; use the normalized running-state check here too.
  - confidence: 0.86
