Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve Tart claims when stop or delete fails:** `internal/providers/tart/backend.go:272-273`
  - body: `ReleaseLease` currently discards both `stopVM` and `deleteVM` errors and then removes the lease claim and stored SSH key. If Tart fails to stop or delete a real VM, Crabbox will report release success while leaving the VM running with no claim/key metadata for retry; only confirmed stale/missing-VM cleanup should tolerate delete failures.
  - confidence: 0.9
- **[P2] Honor keep when the final readiness check fails:** `internal/providers/tart/backend.go:161-163`
  - body: The earlier `waitForIP` and `injectSSHKey` failures respect `req.Keep`, but a later `prepareLease` failure still always stops and deletes the VM. Since `prepareLease(..., true)` includes the SSH readiness wait after the VM has started, a kept lease can still be destroyed on a post-start readiness failure.
  - confidence: 0.86
