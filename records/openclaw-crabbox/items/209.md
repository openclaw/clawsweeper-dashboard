## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve kept VMs after post-start failures:** `internal/providers/tart/backend.go:132-135`
  - body: When `--keep` is set, a VM that has already started should remain available for inspection or reuse after readiness/claim failures. This path unconditionally stops and deletes the VM on IP discovery failure, and the same pattern repeats for SSH injection, lease preparation, and claim updates; the adjacent local providers only remove started resources when `!req.Keep`.
  - confidence: 0.91
- **[P1] Honor cancellation before launching kept VMs:** `internal/providers/tart/backend.go:410`
  - body: For kept VMs this bypasses the caller's context and calls `cmd.Start()` even when the acquire context is already canceled. An abandoned run can still boot a macOS VM and consume local host resources, so gate the kept launch on `ctx.Err()` before spawning while preserving the intended detached lifetime after a valid start.
  - confidence: 0.87
- **[P3] Align the --tart-disk help text:** `internal/providers/tart/flags.go:24`
  - body: The flag help tells users that `0` means use the clone default, but `applyFlags` rejects explicit zero for `--tart-disk`. Describe the clone default as the unset behavior or allow zero consistently so the CLI does not advertise an unusable value.
  - confidence: 0.92
