## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve claims when Tart delete fails:** `internal/providers/tart/backend.go:273-276`
  - body: ReleaseLease ignores `deleteVM` errors and then removes the lease claim/key unconditionally. That is only safe for a verified stale or already-missing VM; if `tart delete` fails because a live VM is busy or stop did not complete, Crabbox loses the ownership record for a VM that still exists, so later stop/status/cleanup can no longer manage it.
  - confidence: 0.9
- **[P2] Use Tart state when deciding whether a lease needs an IP:** `internal/providers/tart/backend.go:595-599`
  - body: Resolve now treats `State` as authoritative when Tart omits the `Running` field, but this branch still checks only `inst.Running`. A running VM with omitted `Running` and empty `tart ip` output will be returned as a successful LeaseTarget with no SSH target instead of failing as no-IP, which breaks run/ssh callers; use `instanceRunning(inst.State)` here too.
  - confidence: 0.87
