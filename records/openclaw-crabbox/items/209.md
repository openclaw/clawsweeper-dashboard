## Best Possible Solution

Fix release error handling so local state is pruned only after successful cleanup or an explicitly stale missing-VM path, then rebase cleanly and make the maintainer decision on whether Tart belongs as a built-in provider.

## Reproduction Assessment

Yes. Source inspection is enough to reproduce the review finding: make `tart stop` or `tart delete` return an error during `ReleaseLease`, and the code still removes the local claim/key and returns nil.

## Solution Assessment

No. The provider shape mostly follows existing local-provider patterns, but `ReleaseLease` should distinguish stale missing-VM pruning from real cleanup failures before this is the safest implementation path.
