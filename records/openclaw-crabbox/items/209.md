## What This Changes

Adds a Tart local VM provider for macOS on Apple Silicon, including config/flags, lifecycle backend, registration, docs, and provider tests.

## Best Possible Solution

Land the Tart provider after separating stopped-VM status reporting from runnable SSH resolution, so status can report stopped while run/hydrate fail early with a Tart-specific diagnostic.

## Reproduction Assessment

Yes for the review finding: source inspection shows stopped Tart instances return a LeaseTarget with no SSH target, while run with an existing lease uses Resolve before SSH wait/sync. The PR discussion also includes a focused stopped-VM repro for the same path.

## Solution Assessment

No, not yet merge-ready: reporting stopped state is useful for status, but the same success path should not be returned to runnable callers. The narrow fix is a Tart Status method or equivalent status-only path plus strict Resolve behavior for non-running instances.
