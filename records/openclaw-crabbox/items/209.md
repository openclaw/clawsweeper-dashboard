## Repair Work Prompt

On PR https://github.com/openclaw/crabbox/pull/209, fix the Tart provider stale release path. Observable bug: when a Crabbox Tart lease claim points at an instance that no longer appears in `tart list`, `Resolve(ReleaseOnly)` can return a missing-instance target but `ReleaseLease` still calls `tart stop`/`tart delete`; the delete error prevents `removeLeaseClaim` and `removeStoredTestboxKey`, so `crabbox stop --provider tart <lease-or-slug>` cannot prune stale local state. Implement a narrow missing-instance marker/path in `internal/providers/tart/backend.go` so claim-backed missing VMs remove the local claim and stored key without calling tart stop/delete, while real delete failures for existing VMs still propagate. Add a focused regression test in `internal/providers/tart/provider_test.go` for a stale claim with empty tart list. Do not add Linux/Ubuntu guest support or broaden provider scope in this repair.
