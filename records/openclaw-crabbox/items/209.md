## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve kept VMs after readiness failures:** `internal/providers/tart/backend.go:133-140`
  - body: `Acquire` still stops and deletes the VM after `waitForIP` or SSH key injection fails, even when `req.Keep` is true. That makes `--keep` lose the started VM state users kept for reuse or debugging after a post-start failure.
  - confidence: 0.9
- **[P1] Honor cancellation before launching kept VMs:** `internal/providers/tart/backend.go:409-414`
  - body: The kept path uses `exec.Command` and starts `tart run` without checking `ctx`, so a canceled acquire can still launch and detach a VM. Check the context before start, and keep the non-kept `CommandContext` behavior for cancellable runs.
  - confidence: 0.88
- **[P1] Preserve the generic SSH user default:** `internal/providers/tart/backend.go:49-65`
  - body: When `tart.user` is unset, `applyDefaults` fills `admin` and then assigns `cfg.SSHUser = cfg.Tart.User`, overwriting top-level `ssh.user` or `CRABBOX_SSH_USER`. Existing users who set the generic SSH user can end up injecting keys and syncing as the wrong account.
  - confidence: 0.87
- **[P2] Use Tart state when Running is omitted:** `internal/providers/tart/backend.go:185-186`
  - body: `Running` is an optional JSON bool that defaults to false when omitted, but `Resolve` rejects all non-status callers solely on `!inst.Running`. Tart inventories that report `State: running` without `Running` will be treated as stopped; use the state helper as the fallback.
  - confidence: 0.86
- **[P2] Prune stale claims when the VM is gone:** `internal/providers/tart/backend.go:538`
  - body: If a claim points to an instance that no longer exists, `resolveInstance` returns before the release path can remove the claim. `crabbox stop --provider tart <lease>` should be able to clean that stale local claim instead of leaving the user stuck with broken state.
  - confidence: 0.84
- **[P3] Align the tart disk zero contract:** `internal/providers/tart/flags.go:24`
  - body: The flag help says `--tart-disk` accepts `0 = use clone default`, but the flag parser rejects zero as invalid. Either make explicit zero mean clone default or change the help/docs so users do not get a contradictory config contract.
  - confidence: 0.82
