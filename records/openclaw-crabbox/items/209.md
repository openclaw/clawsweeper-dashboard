Full review comments:

- **[P1] Return Tart release failures before dropping lease state:** `internal/providers/tart/backend.go:272-273`
  - body: ReleaseLease currently discards both `tart stop` and `tart delete` errors and then removes the lease claim/key. If Tart fails to stop or delete a real VM, `crabbox stop --provider tart` can report success while leaving the VM behind and erasing the state needed to retry cleanup; only the known stale missing-VM path should be tolerated.
  - confidence: 0.93
- **[P2] Treat running Tart state as running when IP is missing:** `internal/providers/tart/backend.go:595-599`
  - body: This stopped-VM shortcut checks only the optional `Running` bool. Tart list output may omit that field while `State` is `running`, so a running VM with an empty or transient `tart ip` result can be returned with no SSH target instead of failing with the existing no-IP error; use `inst.Running || instanceRunning(inst.State)` here too.
  - confidence: 0.82
