Overall confidence: 0.91

Full review comments:

- **[P1] Preserve generic Tart user and workRoot overrides:** `internal/cli/config.go:776-785`
  - body: Because `baseConfig()` pre-populates `Tart.User` and `Tart.WorkRoot`, this branch treats defaults as provider-specific user input and copies them back over top-level `ssh.user` and `workRoot`. A config with `provider: tart`, `ssh.user: root`, and `workRoot: /custom/crabbox` still resolves to `admin` and `/Users/admin/crabbox`; add explicit Tart user/workRoot markers so generic settings fall through unless `tart.user` or `tart.workRoot` is actually set.
  - confidence: 0.94
- **[P1] Preserve kept VMs after post-start failures:** `internal/providers/tart/backend.go:131-139`
  - body: `crabbox warmup --keep` should leave the started VM available for inspection when readiness or key injection fails, but these error paths always call `stopVM` and `deleteVM`. That destroys the exact VM the user asked to preserve for debugging.
  - confidence: 0.95
- **[P2] Use normalized state when Running is omitted:** `internal/providers/tart/backend.go:185-186`
  - body: This rejects every non-status resolve when `Running` is false, even if Tart reports `State: "running"` and omits the `Running` field. Use the normalized state helper, or treat either `Running` or running state as usable, so older/variant Tart JSON does not strand active leases.
  - confidence: 0.91
- **[P2] Honor cancellation before detached Tart launch:** `internal/providers/tart/backend.go:409-433`
  - body: The `keep` branch starts `tart run` with `exec.Command` and the startup select never watches `ctx.Done()`, so a canceled acquire can still launch and detach a VM before returning. Check cancellation before and during the observation window, and clean up when cancellation wins the startup race.
  - confidence: 0.9
- **[P2] Let stop prune stale claims for missing VMs:** `internal/providers/tart/backend.go:532-538`
  - body: When a local claim points at a Tart VM that is already gone, `resolveInstance` errors before release-only stop can remove the local claim and stored key. Return a release target for claim-backed missing instances and have `ReleaseLease` prune local state without calling `tart stop/delete`.
  - confidence: 0.88
- **[P3] Align tart-disk help with validation:** `internal/providers/tart/flags.go:24`
  - body: The flag help says `0 = use clone default`, but `--tart-disk 0` is rejected below as non-positive. Either allow explicit zero to mean clone default or change the help/docs so users are not told to pass a value that fails validation.
  - confidence: 0.89
