## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve kept Tart VMs after post-start failures:** `internal/providers/tart/backend.go:133-135`
  - body: After `startVM` succeeds, this branch still stops and deletes the VM on IP, SSH-key, readiness, claim, or endpoint-update failure regardless of `req.Keep`. That breaks the existing local-provider expectation that `--keep` preserves partially started resources for diagnosis; guard these post-start cleanup blocks with `if !req.Keep` and cover both keep and non-keep behavior.
  - confidence: 0.96
- **[P1] Honor cancellation before starting kept Tart VMs:** `internal/providers/tart/backend.go:410`
  - body: The keep path uses `exec.Command` and never checks `ctx` before `cmd.Start` or while waiting for the startup observation timeout. A canceled acquire can still launch a long-lived Tart VM and return success after two seconds; check `ctx.Err()` before starting and abort or kill the process if cancellation arrives during observation.
  - confidence: 0.94
- **[P1] Treat Tart State=running as running in Resolve:** `internal/providers/tart/backend.go:185-186`
  - body: This rejects non-status reuse based on the optional `Running` JSON bool. When Tart output has `State:"running"` but omits `Running`, Go leaves the bool false and Crabbox reports the VM as stopped even if it has an IP, blocking `run --id` and other SSH-target commands; use the normalized `instanceRunning(inst.State)` boundary instead.
  - confidence: 0.93
- **[P1] Preserve generic Tart SSH user and work root overrides:** `internal/cli/config.go:777-785`
  - body: Because `baseConfig` pre-populates `cfg.Tart.User` and `cfg.Tart.WorkRoot`, this branch copies `admin` and `/Users/admin/crabbox` back over top-level `ssh.user` and `workRoot`. Users who set Crabbox's generic SSH account or checkout path lose those settings unless they duplicate them under `tart`; add explicit Tart markers or inheritance so provider-specific values only win when explicitly set.
  - confidence: 0.92
- **[P2] Align --tart-disk zero semantics with the help text:** `internal/providers/tart/flags.go:24`
  - body: The registered flag says `0 = use clone default`, but `applyFlags` rejects `--tart-disk 0`. Users following `--help` cannot reset a configured disk size back to clone default; either accept zero consistently as the sentinel or remove that advertised contract from help/docs/tests.
  - confidence: 0.9
