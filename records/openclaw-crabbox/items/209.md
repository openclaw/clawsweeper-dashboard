## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve kept VMs after SSH readiness failures:** `internal/providers/tart/backend.go:162-163`
  - body: `Acquire` now skips cleanup for keep=true during `waitForIP` and `injectSSHKey`, but if `prepareLease` fails while waiting for SSH readiness this block still stops and deletes the VM unconditionally. That destroys the VM the user asked to keep for post-failure inspection; gate this cleanup on `!req.Keep` or intentionally preserve/claim the started VM.
  - confidence: 0.92
- **[P1] Keep the lease claim when Tart delete fails:** `internal/providers/tart/backend.go:273-275`
  - body: `ReleaseLease` ignores `deleteVM` errors and then removes the lease claim and stored key. If `tart delete` fails for anything other than a confirmed missing/stale VM, the VM remains on the host but Crabbox forgets it, so later stop/status/release paths lose ownership state.
  - confidence: 0.94
- **[P2] Reject running Tart leases that have no IP:** `internal/providers/tart/backend.go:596-599`
  - body: `Resolve` checks `State` when Tart omits `Running`, but `prepareLease` still uses only `!inst.Running` to treat an empty IP as a stopped/status-only lease. A running VM with `Running` omitted and no current IP can therefore return a LeaseTarget with no SSH target instead of a clear no-IP error for SSH-target commands.
  - confidence: 0.82
