## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Preserve generic SSH and work-root overrides:** `internal/cli/config.go:777-785`
  - body: `baseConfig` now pre-populates Tart user/work-root defaults, so this branch always copies `admin` and `/Users/admin/crabbox` back over generic `ssh.user` and `workRoot` when `provider: tart` is selected. That silently breaks existing config patterns; track Tart-specific explicitness and inherit top-level values when provider-specific values were not set.
  - confidence: 0.94
- **[P1] Preserve kept VMs after post-start failures:** `internal/providers/tart/backend.go:131-140`
  - body: After `startVM` succeeds, every later failure path still calls `stopVM` and `deleteVM` even when `req.Keep` is true. A DHCP, SSH-key, readiness, or claim-write failure can delete the VM the user explicitly asked to keep for inspection or reuse; guard post-start cleanup on `!req.Keep`.
  - confidence: 0.96
- **[P1] Honor cancellation before launching kept VMs:** `internal/providers/tart/backend.go:409-434`
  - body: The keep branch starts `tart run` with plain `exec.Command`, and the startup observation select only waits for process exit or a timer. If the acquire context is already canceled or cancels during that window, Crabbox can still launch a detached VM and return success; check `ctx.Err()` before start and handle `ctx.Done()` during observation.
  - confidence: 0.95
- **[P2] Use Tart state when Running is omitted:** `internal/providers/tart/backend.go:185-186`
  - body: `Resolve` rejects non-status callers based on the optional JSON `Running` bool, which defaults false when Tart omits that field. A record with `State: "running"` can therefore be rejected as stopped; use the existing normalized `instanceRunning(inst.State)` boundary here.
  - confidence: 0.92
- **[P3] Align the tart disk zero contract:** `internal/providers/tart/flags.go:49-54`
  - body: The flag help says `--tart-disk 0` means clone default, and `configureVM` already skips `--disk-size` for zero, but CLI/env/config validation rejects explicit zero. That prevents users from overriding a configured disk size back to clone default; either accept zero and reject only negatives or change the documented contract consistently.
  - confidence: 0.9
- **[P2] Let stop prune stale Tart claims:** `internal/providers/tart/backend.go:173-178`
  - body: `Resolve` always calls `resolveInstance` before the release-only path, so `crabbox stop --provider tart <lease>` fails when the local claim points at a VM that is already gone and never reaches `ReleaseLease` to remove the claim/key. Add a release-only claim path that returns a stale target and prunes local state without calling `tart stop/delete` for a missing VM.
  - confidence: 0.88
- **[P2] Preserve direct stop for crabbox-named local VMs:** `internal/providers/tart/backend.go:179-183`
  - body: For release-only resolution, this branch rejects a `crabbox-` Tart VM with no claim before returning a target to the stop command. Adjacent local VM behavior allows direct cleanup by instance name, and listing already treats the prefix as Crabbox-owned, so `crabbox stop --provider tart <instance>` should be able to remove an explicitly named orphaned Crabbox VM or the docs must make the stricter recovery path intentional.
  - confidence: 0.82
