## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve generic Tart SSH user defaults:** `internal/providers/tart/backend.go:49-65`
  - body: When `provider=tart` is selected and the user has only configured the existing generic `ssh.user`, `applyDefaults` replaces it with `admin` because `cfg.Tart.User` is empty and is defaulted before `cfg.SSHUser = cfg.Tart.User`. That makes Tart ignore an established provider-neutral setting and inject/connect as the wrong macOS account unless users duplicate the value under `tart.user`.
  - confidence: 0.93
- **[P1] Honor cancellation before kept Tart starts:** `internal/providers/tart/backend.go:409-418`
  - body: The `keep=true` path uses `exec.Command` and never checks `ctx` before `cmd.Start` or while observing startup, so a canceled acquire can still launch a Tart VM and only notice cancellation later. Please fail before starting when the context is already done, and observe cancellation during the startup window so canceled commands do not create new kept local VMs.
  - confidence: 0.9
- **[P1] Preserve kept VMs after post-start failures:** `internal/providers/tart/backend.go:131-140`
  - body: After `startVM` succeeds, every later acquisition failure still calls `stopVM` and `deleteVM` even when `req.Keep` is true. That contradicts the kept-lease behavior described in the PR and removes the VM a user asked to keep for debugging or reuse when IP polling, SSH injection, readiness, claim, or endpoint update fails.
  - confidence: 0.91
- **[P2] Use Tart state when Running is omitted:** `internal/providers/tart/backend.go:185-186`
  - body: `Resolve` rejects all non-status callers when `inst.Running` is false, but `Running` is a decoded bool and becomes false when older or variant Tart JSON omits it even if `State` says `running`. Use the existing state normalization as the source of truth so running claimed VMs do not get rejected as stopped.
  - confidence: 0.87
- **[P3] Align the Tart disk zero contract:** `internal/providers/tart/flags.go:49-52`
  - body: The flag help says `--tart-disk 0` means use the clone default, but the flag handler rejects zero as an error. Either accept explicit zero without marking a resize or change the user-facing contract so the CLI does not document a value it refuses.
  - confidence: 0.9
- **[P2] Prune stale Tart claims when stop targets a missing VM:** `internal/providers/tart/backend.go:538`
  - body: When a lease claim exists but the referenced VM no longer appears in `tart list`, `resolveInstance` returns an error before the release path can remove the stale local claim. `crabbox stop --provider tart <lease>` should be able to clear that claim instead of forcing users into a separate cleanup command.
  - confidence: 0.82
