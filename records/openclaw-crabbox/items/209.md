Full review comments:

- **[P1] Preserve generic Tart SSH and workRoot overrides:** `internal/cli/config.go:777-785`
  - body: `baseConfig` seeds `Tart.User` and `Tart.WorkRoot`, so this branch always rewrites generic `ssh.user` and top-level `workRoot` to the Tart defaults unless users duplicate them under `tart`. That makes accepted generic SSH config ineffective for key injection and sync; preserve non-default generic values when Tart-specific fields were not explicitly set.
  - confidence: 0.9
- **[P2] Honor cancellation before detached Tart launch:** `internal/providers/tart/backend.go:410-422`
  - body: For `--keep`, this uses `exec.Command` instead of a context-bound command, and the startup wait never observes `ctx.Done()`. If acquire is canceled before or during startup, Crabbox can still launch a detached VM and rely on best-effort cleanup; check cancellation before starting and while observing startup.
  - confidence: 0.88
- **[P3] Align tart-disk help with validation:** `internal/providers/tart/flags.go:24`
  - body: The flag help advertises `0 = use clone default`, but `applyFlags` rejects `--tart-disk 0`. Users following `--help` get an error, so either allow 0 as an explicit no-resize sentinel or remove that wording.
  - confidence: 0.93
