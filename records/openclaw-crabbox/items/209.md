Full review comments:

- **[P1] Preserve kept Tart VMs after readiness failures:** `internal/providers/tart/backend.go:162-163`
  - body: When `req.Keep` is true, the earlier waitForIP/injectSSHKey paths preserve the VM, but failures from `prepareLease` while waiting for SSH, claim creation, or endpoint update still stop and delete it here. That makes `--keep` unreliable after the VM has already started; guard these cleanup calls with `if !req.Keep` as the adjacent local providers do and add regressions for those failure paths.
  - confidence: 0.92
- **[P2] Prune confirmed-missing stale claims on stop:** `internal/providers/tart/backend.go:273-274`
  - body: `resolveInstance` can return `State: "missing"` when the lease claim exists but `tart list` no longer contains the VM, yet `ReleaseLease` still calls `tart delete` and returns before removing the local claim/key on the expected not-found error. `crabbox stop --provider tart <lease>` can therefore leave stale state behind; skip deletion and prune the claim/key when the VM was already confirmed missing, while still propagating delete failures for present VMs.
  - confidence: 0.87
