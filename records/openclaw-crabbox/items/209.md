- **[P1] Preserve kept VMs after all post-start failures:** `internal/providers/tart/backend.go:162-163`
  - body: The earlier waitForIP/injectSSHKey failures now respect `req.Keep`, but a later `prepareLease` failure still runs `tart stop` and `tart delete` unconditionally.   - confidence: 0.93
- **[P1] Keep claims when Tart delete fails for real VMs:** `internal/providers/tart/backend.go:273-275`
  - body: `ReleaseLease` ignores `deleteVM` errors and then removes the lease claim/key. That is safe only for a known missing VM; if `tart delete` fails because the VM is busy or Tart errors, Crabbox loses its claim while the VM remains, leaving an orphaned local VM.
  - confidence: 0.91
