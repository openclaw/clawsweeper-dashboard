## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Use Tart state when deciding Resolve is stopped:** `internal/providers/tart/backend.go:185-186`
  - body: `Resolve` still rejects every non-status caller when `inst.Running` is false. If `tart list --format json` omits that bool while `State` is `running`, Go leaves `Running` false and valid leases become unusable for run/ssh/code/cache paths; normalize this through `instanceRunning(inst.State)` and cover the omitted-bool case.
  - confidence: 0.9
- **[P2] Let stop prune stale Tart claims:** `internal/providers/tart/backend.go:538`
  - body: When a lease claim points at a VM that no longer exists, `resolveInstance` returns an error before `crabbox stop` reaches `ReleaseLease`, so the stale claim remains stuck. The ReleaseOnly path should be able to return enough lease state to remove the claim, or otherwise prune the claim deliberately.
  - confidence: 0.88
- **[P2] Honor cancellation before starting kept VMs:** `internal/providers/tart/backend.go:410`
  - body: The keep path builds `exec.Command` without the context and calls `Start` without first checking `ctx.Done()`. A canceled acquire can therefore still launch and detach `tart run`, leaving a local VM after Crabbox has already stopped waiting.
  - confidence: 0.86
- **[P3] Align the tart disk flag help with validation:** `internal/providers/tart/flags.go:24`
  - body: The help text advertises `--tart-disk 0` as clone-default behavior, but the parser rejects zero and the docs now describe clone-default as the unset case. Update the help text or accept zero so users do not follow a flag hint that immediately fails.
  - confidence: 0.93
