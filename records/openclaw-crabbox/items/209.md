## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve generic SSH user and work root:** `internal/providers/tart/backend.go:65-68`
  - body: Because baseConfig pre-populates Tart.User and Tart.WorkRoot, these unconditional assignments overwrite top-level `ssh.user` and `workRoot` even when the user did not set `tart.user` or `tart.workRoot`. Provider-specific values should win only when explicitly configured; otherwise the generic settings need to flow into the Tart config.
  - confidence: 0.94
- **[P1] Honor cancellation before detached Tart starts:** `internal/providers/tart/backend.go:410-414`
  - body: The keep path starts `tart run` with plain `exec.Command` and the select below never observes `ctx.Done()`, so an already-canceled or timed-out acquire can still launch a detached VM and return success after the observation timer. Check `ctx.Err()` before starting and handle cancellation during the startup window.
  - confidence: 0.93
- **[P1] Keep started VMs after keep-mode readiness failures:** `internal/providers/tart/backend.go:132-135`
  - body: After `startVM` succeeds, this failure path still stops and deletes the VM even when the user requested `--keep`. That removes the local VM exactly when DHCP, key injection, SSH readiness, or claim-writing failures need inspection; guard post-start cleanup with `!req.Keep` while preserving normal cleanup for non-keep runs.
  - confidence: 0.91
- **[P2] Use Tart state when deciding if Resolve is running:** `internal/providers/tart/backend.go:185-186`
  - body: `Running` is an optional JSON boolean and defaults to false when omitted, so a record with `State:"running"` can be rejected as stopped. Use the existing normalized `instanceRunning(inst.State)` helper here so running Tart VMs remain reusable even when the boolean field is absent.
  - confidence: 0.9
- **[P2] Align disk-zero help with validation:** `internal/providers/tart/flags.go:49-52`
  - body: The flag help advertises `0 = use clone default`, but the validation rejects `--tart-disk 0` before the run starts. Either accept zero as the documented clone-default sentinel or change the help/docs and tests so users are not told to pass a value that fails.
  - confidence: 0.88
