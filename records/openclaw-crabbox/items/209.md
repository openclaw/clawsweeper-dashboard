- **[P1] Prune missing Tart leases after release-only resolve:** `internal/providers/tart/backend.go:283-284`
  - body: `crabbox stop` first calls `Resolve(... ReleaseOnly:true)`, which returns a `LeaseTarget` whose `Server.CloudID` is already set even when `resolveInstance` produced `State:"missing"` for a stale claim. Because this method only checks `tartState(inst.State) == "missing"` inside the fallback block where `name == ""`, the real stop path falls through to `tart stop`/`tart delete`; delete then fails for an already-deleted VM and the stale claim/key remain. Please prune the missing-state claim before stop/delete and add a regression that passes the ReleaseOnly-resolved lease into `ReleaseLease`.
