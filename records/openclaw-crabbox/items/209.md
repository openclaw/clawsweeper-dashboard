## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve generic Tart user and work-root overrides:** `internal/cli/config.go:1095-1096`
  - body: Because `baseConfig()` pre-populates `Tart.User` and `Tart.WorkRoot`, the later Tart defaulting path treats provider defaults as configured values and overwrites top-level `ssh.user` / `workRoot`. A config with `provider: tart`, `ssh.user: root`, and `workRoot: /custom/crabbox` still resolves to `admin` and `/Users/admin/crabbox`, so custom Tart images run as the wrong account/path unless the user discovers the provider-specific keys. Add explicit Tart user/workRoot markers or defer these defaults so generic values flow through unless Tart-specific values are set.
  - confidence: 0.9
- **[P1] Use Tart state when Running is absent:** `internal/providers/tart/backend.go:185-186`
  - body: This rejects any non-status Resolve when `inst.Running` is false, but `Running` is an optional JSON boolean and decodes to false when Tart omits it even if `State` is `running`. That makes existing running leases fail as stopped. Use the normalized state helper for this boundary and add a regression with `State:"running"` and no `Running` field.
  - confidence: 0.94
- **[P1] Preserve kept VMs after post-start failures:** `internal/providers/tart/backend.go:133-134`
  - body: After `startVM` returns nil, the wait-for-IP, key injection, SSH readiness, claim, and endpoint-update failure branches still stop/delete the VM unconditionally. For `--keep`, those are the failure cases where users need the local VM left around for inspection or reuse; adjacent local providers guard later cleanup with `if !req.Keep`.
  - confidence: 0.92
- **[P1] Honor cancellation before detached Tart starts:** `internal/providers/tart/backend.go:409-414`
  - body: The keep path uses `exec.Command` so the VM can outlive Crabbox, but it never checks `ctx.Err()` before `Start` and the observation select has no `ctx.Done()` case. A canceled acquire can therefore launch `tart run`, wait for the timer, and return nil; check cancellation before starting and terminate the process if cancellation arrives during startup observation.
  - confidence: 0.9
- **[P3] Align the tart disk zero help text:** `internal/providers/tart/flags.go:24`
  - body: The flag help says `0 = use clone default`, but `applyFlags` rejects `--tart-disk 0` as invalid. Either allow explicit zero or update the help text so users are told to omit the flag for clone default.
  - confidence: 0.88
