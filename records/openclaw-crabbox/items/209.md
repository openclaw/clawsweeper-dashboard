## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve generic Tart user and workRoot overrides:** `internal/cli/config.go:777-785`
  - body: The Tart defaults branch copies the provider defaults back over top-level `ssh.user` and `workRoot`, so `provider: tart` ignores existing generic config unless the user repeats values under `tart:`. Track explicit `tart.user`/`tart.workRoot` values and let generic settings flow through when the provider-specific keys are unset.
  - confidence: 0.93
- **[P1] Preserve kept VMs after post-start failures:** `internal/providers/tart/backend.go:133-134`
  - body: After `startVM` succeeds, failures in IP polling, key injection, readiness, claim creation, or endpoint update still call `stopVM` and `deleteVM` even when `req.Keep` is true. That defeats `--keep` exactly when a user needs the partially provisioned VM for inspection, so guard the post-start cleanup with the keep flag and cover the failure paths.
  - confidence: 0.94
- **[P2] Honor cancellation before detached Tart launch:** `internal/providers/tart/backend.go:409-410`
  - body: The keep path uses plain `exec.Command` and the startup observation select never watches `ctx.Done()`, so an already-canceled acquire can still launch a detached Tart VM and return success after the timer. Check `ctx.Err()` before `Start` and handle cancellation during the observation window.
  - confidence: 0.91
- **[P2] Use normalized state when Running is omitted:** `internal/providers/tart/backend.go:185-186`
  - body: `Running` is an optional decoded boolean, so a Tart record with `State: "running"` but no `Running` field is treated as stopped and rejected for non-status Resolve callers. Use the existing normalized state helper for this boundary so running leases remain usable.
  - confidence: 0.9
- **[P2] Let release-only stop prune stale claims:** `internal/providers/tart/backend.go:538`
  - body: When a local Tart claim points at a VM that no longer appears in `tart list`, release-only resolution errors before `ReleaseLease` can remove the stale claim and stored key. Resolve claim-backed release-only requests to a missing-instance target and prune local state without calling Tart for the absent VM.
  - confidence: 0.88
- **[P3] Align tart-disk help with validation:** `internal/providers/tart/flags.go:24`
  - body: The flag help still says `0 = use clone default`, but `applyFlags` rejects `--tart-disk 0`. Either accept zero as the documented sentinel or remove that claim from the help text and docs so users do not get contradictory CLI guidance.
  - confidence: 0.86
