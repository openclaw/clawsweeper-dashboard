## Summary

Keep open: current main has timing JSON, proof, and artifact gates, but it does not implement deterministic profiling/fuel budgets or a WASI/wasmtime metering backend. The broad WASI provider at https://github.com/openclaw/crabbox/pull/211 was closed unmerged, so this narrower product question still needs maintainer design review rather than cleanup closure.

## What This Changes

This issue requests exploring a narrow deterministic performance evidence and budget gate for `crabbox run`, likely starting with instruction/fuel metering for wasm-compatible workloads.

## Best Possible Solution

Define a small provider-neutral perf evidence contract, capability-gate the backend that can produce deterministic metrics, and prove it on one pure deterministic workload before adding any general WASI provider surface.

## Reproduction Assessment

Not applicable. This is a feature/product exploration rather than a bug report, and source inspection confirms the requested perf-budget behavior is not present on current main.

## Solution Assessment

Unclear. The narrower deterministic evidence direction is more maintainable than reviving the broad WASI provider, but maintainers still need to choose the metric, CLI/API shape, runtime dependency, and proving workload.
