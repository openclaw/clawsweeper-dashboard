## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Reject non-Apple-silicon Macs before running the provider:** `internal/providers/applecontainer/backend.go:506-509`
  - body: The provider is described as Apple-silicon-only, but this gate only checks `runtime.GOOS`. On `darwin/amd64` it proceeds as if supported and fails later against an unsupported runtime; check `runtime.GOARCH == "arm64"` and cover that path before probing `container`.
  - confidence: 0.86
- **[P2] Keep the bootstrap on a root-capable image path:** `internal/providers/applecontainer/backend.go:327-334`
  - body: The bootstrap runs `apt-get`, `useradd`, `chown`, and writes `/etc/sudoers.d`, but the run args do not force or validate a root init user. A configured image with a non-root `USER` can exit before `sshd` starts and leave `crabbox run` waiting for port 22 until timeout; force root if supported or reject/document unsupported image users with a regression test.
  - confidence: 0.78
- **[P3] Document the macOS 26 runtime requirement:** `docs/providers/apple-container.md:42-43`
  - body: The new provider docs only say Apple-silicon macOS, while Apple's current `container` README says the runtime is supported on macOS 26 and older releases are not supported. Add that prerequisite so users on older Apple-silicon macOS do not chase Crabbox setup failures.
  - confidence: 0.9
