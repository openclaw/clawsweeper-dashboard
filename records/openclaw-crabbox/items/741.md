## Summary

Keep open: this PR is a viable candidate fix with sufficient Windows log proof, but it changes the default privileged Windows prep path to a fail-closed Chocolatey digest pin, so maintainers should explicitly accept the compatibility and digest-refresh policy before merge.

## What This Changes

The branch updates the Windows developer-tools prep script to download the Chocolatey installer to a temporary file, verify a SHA-256 pin before execution, add URL/digest environment overrides, and add a Node static regression test.

## Best Possible Solution

Land a maintainer-approved version of this Chocolatey verification flow, document or explicitly accept the digest update path, let https://github.com/openclaw/crabbox/issues/721 close on merge, and keep Node/Docker hardening scoped to https://github.com/openclaw/crabbox/issues/686.
