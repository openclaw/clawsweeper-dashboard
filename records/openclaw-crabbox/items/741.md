## Label Justifications

- `P1`: This is an urgent security hardening PR for a documented privileged image-prep workflow that can persist downloaded installer compromise into reusable Windows images.
- `merge-risk: 🚨 compatibility`: The PR changes default Windows image prep to a hash-pinned fail-closed bootstrap, which can break existing bakes when the upstream installer changes unless maintainers own the pin and override path.

## Summary

Keep open: this PR addresses a real current-main supply-chain gap and the patch is directionally correct, but it changes a privileged Windows bootstrap to fail closed and only provides static test proof, so it needs maintainer review plus real Windows prep proof before merge.

## What This Changes

The PR changes the Windows developer-tools prep script to download Chocolatey's installer to a temp file, verify a SHA-256 digest before execution, expose URL/digest env overrides, and add a Node script regression test.

## Best Possible Solution

Land a maintainer-reviewed version that verifies Chocolatey's installer before execution, keeps the override path explicit, and includes real Windows prep proof before merge.

## Reproduction Assessment

Yes. Source inspection on current main shows the default Windows image bake selects the affected prep script, and that script executes Chocolatey's downloaded installer through `Invoke-Expression` without integrity verification; no live Windows bake was run.

## Solution Assessment

Yes, mostly. The PR follows the narrow remediation by downloading, hashing, failing closed, and executing a local file, but merge readiness still depends on real Windows prep proof and maintainer acceptance of the fail-closed default.
