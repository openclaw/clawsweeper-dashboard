## Summary

Keep open: current main still sends the Freestyle API key as a bearer header and formats non-2xx Freestyle response bodies into user-visible errors without redacting the configured key.

## What This Changes

Issue reports that Freestyle non-2xx API response snippets can echo the configured bearer token into Crabbox diagnostics without redaction.

## Best Possible Solution

Add provider-local Freestyle redaction at the non-2xx response-body boundary and cover it with regression tests for at least one shared path plus the helper behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a Freestyle test server returning a non-2xx body containing the configured key would have that body included directly in the returned error.

## Solution Assessment

Yes. The narrow maintainable fix is to redact the exact configured key and Bearer form before formatting Freestyle API error snippets while preserving status and useful provider context.
