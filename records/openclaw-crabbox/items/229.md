## Summary

The prerequisite generic evidence primitive is shipped, but current main does not contain the requested `crabbox data run` command. The issue is explicitly paired with the open, clean implementation at https://github.com/openclaw/crabbox/pull/235, so it should remain open until maintainers approve and merge that public command/config surface or close the PR.

## What This Changes

Add a thin `crabbox data run` wrapper that expands repository-defined data workflows into existing lease, sync, environment, run, timing, JUnit, and bounded-artifact primitives.

## Best Possible Solution

Land a maintainer-approved thin wrapper that delegates to existing run primitives, keeps raw data and ETL semantics caller-owned, validates only bounded evidence, and preserves provider-specific retrieval or enforcement behind adapter capabilities.

## Reproduction Assessment

Not applicable; this issue requests a new CLI and repository configuration surface rather than reporting broken existing behavior. Source inspection confirms that current main has the prerequisite artifact guard but not the requested wrapper.

## Solution Assessment

Yes in architectural direction, because a thin wrapper over generic run primitives is narrower and more maintainable than an ETL subsystem. The exact `crabbox data`, `dataRuns`, manifest, and provider-capability contracts still require maintainer approval through the linked PR.
