## Label Justifications

- `P2`: This optional provider is valuable and platform-bounded, with one normal-priority correctness blocker.
- `merge-risk: 🚨 session-state`: The split claim and endpoint writes can leave persisted lease state associated with a removed VM.
- `merge-risk: 🚨 security-boundary`: The provider executes privileged host and guest PowerShell, handles guest credentials, and mutates a lease disk offline.
- `merge-risk: 🚨 availability`: Provisioning or cleanup failures can leave leases unavailable or require Windows host intervention.

## Summary

The provider direction is strong and backed by convincing live Windows proof, but acquisition splits claim creation from endpoint persistence; if the second write fails, the VM is removed while a stale lease claim can remain, unlike the atomic current-main Tart pattern.

## What This Changes

The PR adds a built-in Hyper-V provider for local Windows VM leases, including differencing-disk lifecycle management, PowerShell Direct guest bootstrap, configuration, documentation, and focused tests.

## Best Possible Solution

Use the existing atomic endpoint-aware claim helper during acquisition and add a regression test proving a persistence failure cannot leave a claim for a removed VM.

## Reproduction Assessment

Yes for the review finding: source inspection shows a successful initial claim mutation followed by a separately fallible endpoint mutation whose error path removes the VM without deleting the claim.

## Solution Assessment

Mostly yes: the provider-local architecture and lifecycle design are appropriate, but acquisition should reuse the existing atomic endpoint-aware claim helper rather than introduce a two-write state transition.
