## Summary

Keep open: current main does not have a Hyper-V provider, and this PR is the active implementation, but it is not merge-ready because live Hyper-V proof is still missing and the target defaulting path can either override or reject the intended target incorrectly.

## What This Changes

The PR adds a Hyper-V SSH lease provider for local Windows VMs, including provider registration, config and flags, unit tests, and provider documentation.

## Best Possible Solution

Land a Hyper-V provider only after target defaults preserve explicit user intent, configured-provider usage works without extra target boilerplate, and redacted live Windows Hyper-V lifecycle proof covers acquire, SSH readiness/run, status/list, release, and cleanup.

## Reproduction Assessment

Yes for the target-defaulting bug from source inspection: lease flags apply `--target` before Hyper-V `ApplyFlags`, and Hyper-V `applyDefaults` rewrites `target=linux` to Windows. Real Hyper-V lifecycle behavior itself is not reproduced because no live proof is present.

## Solution Assessment

No as submitted: the provider design is plausible, but the target defaulting needs a narrow fix and the runtime lifecycle still needs live Windows Hyper-V proof. The safer path is to default only implicit provider selection to Windows while rejecting explicit non-Windows targets.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Preserve explicit target selection before defaulting Hyper-V:** `internal/providers/hyperv/flags.go:48-49`
  - body: `applyLeaseCreateFlags` applies `--target` before provider `ApplyFlags`, so `crabbox warmup --provider hyperv --target linux` reaches this branch with `cfg.TargetOS == "linux"` and `applyDefaults` silently rewrites it to Windows. The same split defaulting also leaves config-file `provider: hyperv` paths vulnerable to reaching `Configure` with the global Linux default and being rejected. Hyper-V needs to distinguish an implicit global default from an explicit non-Windows target: default implicit provider selection to native Windows, but reject explicit non-Windows targets instead of overwriting them.
  - confidence: 0.87
