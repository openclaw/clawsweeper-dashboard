## Summary

This is a valuable, well-proven provider addition, but the opt-in password initialization path writes an unrestricted configured username into an elevated cmd.exe RunOnce command; that credential boundary must be hardened before merge.

## What This Changes

Adds a built-in Hyper-V provider for local Windows VM leases, including differencing-disk lifecycle management, PowerShell Direct guest bootstrap, configuration, documentation, and tests.

## Best Possible Solution

Replace the cmd.exe credential string with a safely encoded first-boot script or strictly validate both credential fields before persisting RunOnce, then retain the existing environment-only secret transport and exact-head Windows lifecycle proof.

## Solution Assessment

No, not yet; the adapter boundary, differencing-disk model, and proof are strong, but the first-boot credential command must safely handle every accepted configured username.
