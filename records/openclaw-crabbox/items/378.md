## Label Justifications

- `P2`: The issue reports a medium-severity local confidentiality problem with a bounded CLI output surface.
- `impact:security`: The affected behavior is local file permissions for run outputs that can contain sensitive provider, lease, stdout/stderr, proof, or failure diagnostic data.

## Summary

Keep open: current main still creates the reported local run outputs with public/default file modes while the docs say those captures and bundles can be secret-bearing.

## What This Changes

The issue asks Crabbox to make local `crabbox run` outputs private by default for downloads, stream captures, proof files, and failure bundles.

## Best Possible Solution

Make Crabbox-created sensitive local outputs use private defaults, add focused mode regression tests under a permissive umask, and leave any explicit sharing/export workflow as a separate maintainer decision.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: run with a permissive umask and create any of the named output files, which current code opens with 0666/0644 or creates parent directories with 0755. I did not run filesystem-mode tests because this cleanup review is read-only.

## Solution Assessment

Yes, the narrow maintainable fix is to keep the existing paths and behavior while changing Crabbox-created sensitive files to 0600 and created output directories to 0700, with tests covering the final modes.
