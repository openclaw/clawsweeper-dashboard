## Summary

Keep open: current main still passes the Cloudflare Sandbox bridge HTTP client through without a redirect policy while authenticated bridge calls add bearer auth and request bodies, which conflicts with the repository's documented same-origin credential-destination policy.

## What This Changes

The issue requests an exact-origin redirect guard for authenticated Cloudflare Sandbox bridge requests.

## Best Possible Solution

Add a provider-local secure bridge HTTP client that clones the caller client, rejects redirects leaving the normalized bridge scheme/host/effective port before caller CheckRedirect, redacts redirect errors, and covers cross-origin, same-origin, caller-policy, and redaction cases.

## Reproduction Assessment

Yes at source level: current main constructs the bridge client without CheckRedirect and authenticated requests go through that client. I did not run a network reproduction in this read-only pass.

## Solution Assessment

Yes: the requested exact-origin guard matches the existing coordinator and peer-provider pattern, and it can be implemented narrowly in the Cloudflare Sandbox client with regression coverage.
