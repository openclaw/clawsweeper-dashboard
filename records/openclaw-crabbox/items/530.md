## Summary

Keep open: current main still lacks a `crabbox station` command or any mode that starts a sandbox-agent-style daemon in a lease and bridges its HTTP/SSE API. The merged Station work is a deliberately disabled roadmap/config skeleton, so this remains a product and security design item.

## What This Changes

Request to add a mode that leases a box, runs a coding-agent harness daemon inside it, and exposes that daemon over HTTP/SSE for external callers.

## Best Possible Solution

Keep this issue open as the narrow harness-daemon follow-up, first settle the generic SSH-backed Station MVP, then add any agent daemon bridge only after the auth, egress, credential, and evidence boundaries are approved.

## Solution Assessment

Unclear for implementation: running a sandbox-agent over existing bridge plumbing is plausible, but the maintainable path should go through the Station lifecycle and security contract instead of a one-off daemon bridge.
