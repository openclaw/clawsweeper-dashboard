## Summary

Keep open: current main and v0.32.0 do not have a sandbox-agent/HTTP-SSE harness mode; existing run, delegated-provider, bridge, and port surfaces still cover fixed commands or provider-specific port sharing, while related Station and mitos work remains open product/security design.

## What This Changes

The issue requests a mode that leases a box, starts a coding-agent harness daemon inside it, and exposes that daemon's HTTP/SSE API through Crabbox.

## Best Possible Solution

Keep this as product/security design work, decide whether it belongs under Station or a provider-neutral bridge/session capability, then implement one narrow mode with explicit auth, egress, credential, cleanup, docs, and tests.
