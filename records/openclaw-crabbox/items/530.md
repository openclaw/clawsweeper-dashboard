## Summary

Keep open: current main still does not implement the requested sandbox-agent or harness HTTP/SSE runtime mode. The merged Station work adds a deliberately disabled roadmap/config skeleton, and the live CLI/runtime bridge remains fixed-command or code-server/provider-specific rather than a generic agent daemon surface.

## What This Changes

The issue requests a Crabbox mode that leases a box, starts a coding-agent harness daemon such as sandbox-agent inside it, and exposes that daemon over an HTTP/SSE bridge.

## Best Possible Solution

Keep this open and route it through the Station roadmap: first settle a generic supervised Station runtime and bridge contract, then decide whether a sandbox-agent-style harness daemon belongs as an agent profile, plugin, or separate provider-neutral bridge capability.

## Reproduction Assessment

## Solution Assessment

Unclear as a direct implementation path: reusing bridge plumbing is plausible, but the safer maintainable path is to define the Station/agent runtime, credential, egress, and provider-capability contract before adding a daemon mode.
