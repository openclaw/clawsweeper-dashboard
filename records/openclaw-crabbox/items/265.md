## Summary

The feature is absent from current main and fits the established SSH lease provider architecture, but it needs a maintainer product decision on lifecycle defaults and the minimum supported Coder configuration before implementation.

## What This Changes

Requests a first-class Coder-backed SSH lease provider with lifecycle, configuration, documentation, and tests.

## Best Possible Solution

Add a direct-only Linux SSHLeaseBackend adapter that uses the installed Coder CLI and its credential store, defaults release to stopping workspaces, isolates Coder naming and lifecycle behavior in the provider package, and ships only the minimum configuration proven by an authenticated end-to-end smoke test.

## Solution Assessment

## Risks / Open Questions

- Coder workspaces may contain persistent user state, so stop-versus-delete behavior and when local claims are removed require an explicit, conservative contract.
- The proposed option set is broad for a first version; template parameters, presets, owner-qualified resolution, agent selection, cleanup, and deletion policy could expand the implementation and compatibility surface substantially.
- Live validation requires an authenticated Coder deployment and representative Terraform template, so unit tests alone cannot establish reliable workspace lifecycle and ProxyCommand SSH behavior.
