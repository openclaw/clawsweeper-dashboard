## Summary

Keep this PR open: the native Screen Sharing direction is useful, but the final diff still advertises the generic desktop capability before tart is wired or guarded for the existing WebVNC/screenshot command paths, and the credential story needs maintainer sign-off.

## What This Changes

The PR adds tart `FeatureDesktop`, enables macOS Screen Sharing during desktop lease acquisition, adds `--tart-user`, and updates tart docs/tests for native VNC use.

## Best Possible Solution

Scope tart desktop support to the flows that actually work today with explicit guards for unsupported helpers, or implement the missing macOS WebVNC/screenshot credential path before advertising full desktop capability.

## Reproduction Assessment

Yes from source inspection for the command-routing issue: adding `FeatureDesktop` makes tart match `supportsDirectSSHWebVNC`, whose remote command requires Linux noVNC tooling on the guest. The credential-dependent screenshot path is also source-reproducible because it reads `/var/db/crabbox/vnc.password`, which the PR no longer provisions.

## Solution Assessment

No; enabling native Screen Sharing is useful, but using the generic `FeatureDesktop` surface without tart-specific WebVNC/screenshot guards is broader than the implemented support. The safer path is to either implement those macOS paths or explicitly fail them before merge.
