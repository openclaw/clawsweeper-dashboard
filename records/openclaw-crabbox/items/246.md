## Summary

Keep open: this is a useful tart desktop follow-up, but the patch is not merge-ready because advertising `FeatureDesktop` makes tart use a WebVNC path that expects target-side noVNC/websockify and Linux `ss`, while the PR only enables macOS Screen Sharing. ## What This Changes

The PR advertises tart desktop capability, enables guest macOS Screen Sharing during desktop acquire, adds tart provider tests, and updates tart provider docs.

## Best Possible Solution

Keep the tart desktop feature, but merge it only after WebVNC uses a macOS-capable bridge path or provisions its target-side dependencies, with focused tests and redacted live tart proof.

## Reproduction Assessment

Yes for the review finding: source inspection shows the new tart `FeatureDesktop` bit would route `crabbox webvnc` into `directSSHWebVNC`, whose remote command requires target-side noVNC/websockify and `ss`. I did not run a live tart host.

## Solution Assessment

No, not as-is. The maintainable path is to either make direct WebVNC work with macOS Screen Sharing directly or provision and validate the target-side bridge dependencies the selected path requires.
