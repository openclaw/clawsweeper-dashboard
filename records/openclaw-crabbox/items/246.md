## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Route tart WebVNC through a macOS-capable bridge:** `internal/providers/tart/provider.go:27`
  - body: Adding FeatureDesktop makes tart satisfy supportsDirectSSHWebVNC, so crabbox webvnc enters directSSHWebVNC before any coordinator path. That path still launches target-side noVNC/websockify and checks it with ss, while this patch only starts macOS Screen Sharing on 5900; a stock tart macOS guest can therefore pass the new VNC probe but fail before the browser viewer opens.
  - confidence: 0.91
- **[P1] Keep tart passwords out of process arguments:** `internal/providers/tart/backend.go:550-553`
  - body: The new password is written by interpolating it into the tart exec bash -c argument, and the PR also exposes a --tart-password flag. Repository guidance says provider secrets must not be passed as command-line arguments, so this can leak the guest account password through shell history or process listings.
  - confidence: 0.88
- **[P2] Correct the tart desktop credential docs:** `docs/providers/tart.md:67`
  - body: The docs say acquire generates a 16-character password and sets the account password to match, but the code now records the configured existing account password and deliberately does not reset secure-token accounts. Users following this page will expect a generated per-lease credential that is not actually created.
  - confidence: 0.87
