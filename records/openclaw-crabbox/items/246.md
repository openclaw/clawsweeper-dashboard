## Summary

Keep this PR open: the tart desktop feature is still missing from current main and the latest branch addresses the prior WebVNC routing/credential issues, but `enableScreenSharing` can currently report success even when Screen Sharing was not enabled.

## What This Changes

The PR adds tart `--desktop` support by advertising `FeatureDesktop`, enabling macOS Screen Sharing during acquire, guarding WebVNC for macOS leases, adding tests, and updating tart docs.

## Best Possible Solution

Land tart desktop as native macOS Screen Sharing with WebVNC guarded, but require a positive Screen Sharing start/reachability check before marking a lease ready.

## Reproduction Assessment

Yes for the patch defect from source inspection: all Screen Sharing start commands are followed by `|| true`, so the script exits successfully even when the service is not enabled. The happy path has live proof, but the failure path is not guarded.

## Solution Assessment

Mostly yes: native Screen Sharing with WebVNC guarded is a maintainable scope for tart. The implementation should prove Screen Sharing is running before returning ready, and maintainers should accept the documented host-local VNC boundary.
