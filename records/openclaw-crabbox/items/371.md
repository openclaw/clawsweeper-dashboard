## Summary

Keep open: current main still deletes bridge-ticket records before lease, role, or adapter binding validation, so the reported availability/security weakness remains present.

## What This Changes

The issue reports that WebVNC, code, egress, and runtime-adapter bridge tickets can be consumed on mismatched endpoints before endpoint binding validation runs.

## Best Possible Solution

Change Worker ticket consumption so endpoint binding validation passes before the irreversible delete, while preserving expiry behavior and single-use deletion for valid upgrades.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: mint a ticket for lease or adapter A, present it to endpoint B, and the current helper deletes it before the handler rejects the binding mismatch.

## Solution Assessment

Yes. The reported remediation is the narrow maintainable fix: validate lease identifier, egress role, or adapter ID against the stored ticket before deleting the ticket record.
