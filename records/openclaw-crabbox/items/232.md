Full review comments:

- **[P2] Preserve bootstrap_dir before release uses it:** `internal/providers/localcontainer/backend.go:202`
  - body: The PR makes `ReleaseLease` depend on `lease.Server.Labels["bootstrap_dir"]`, but local-container `Touch` rebuilds labels through `TouchDirectLeaseLabels` and only restores a fixed allowlist that does not include `bootstrap_dir`. Because label sanitization rewrites path separators, a normal direct run/status touch can hand release an untrusted cleanup path and leave the temp directory behind; preserve this label and add a touch regression test.
  - confidence: 0.9
- **[P2] Clean the bootstrap dir without re-inspecting after inspect fails:** `internal/providers/localcontainer/backend.go:105-109`
  - body: `cleanupContainer` only learns the bootstrap directory by calling `inspectContainer` again. On the first failure path it runs because inspect already failed, so that second inspect is unlikely to recover the label and the newly created `crabbox-bootstrap-*` directory can survive; carry the local bootstrap path out of `createContainer` or otherwise clean it directly.
  - confidence: 0.87
