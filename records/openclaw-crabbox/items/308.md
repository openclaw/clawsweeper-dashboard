## Summary

Keep open: current main still provisions Tailscale and records readiness metadata, but release and expiry cleanup only delegate to provider deletion and do not remove the tailnet node explicitly.

## What This Changes

The issue requests best-effort cleanup of Tailscale membership when Tailscale-enabled leases are released or expire.

## Best Possible Solution

Define a provider-neutral Tailscale cleanup contract that records only non-secret node identity and cleanup status, attempts best-effort removal during release/expiry, and documents fallback to ephemeral-node removal.

## Reproduction Assessment

Yes, from source inspection: the Tailscale-enabled lease flow can be traced through provisioning and release, and current release/expiry cleanup has no explicit Tailscale removal step. I did not run a live tailnet reproduction.

## Solution Assessment

Unclear; explicit cleanup is the right end state, but maintainers should choose between CLI-side remote logout, coordinator-side API deletion, or a hook that supports both before implementation.
