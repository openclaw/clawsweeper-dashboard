## Summary

Keep open: current main still issues and accepts isolated code viewer sessions independently of portal logout, and no duplicate issue or fix PR owns the revocation work.

## What This Changes

The issue reports that isolated Code viewer sessions remain usable after portal logout because the isolated-origin code-session cookie and Durable Object session are independent of the cleared portal session.

## Best Possible Solution

Bind isolated Code viewer sessions to an active portal session or revoke them server-side when the portal session logs out, with regression tests for stale isolated-session replay.

## Reproduction Assessment

Yes by source trace: isolated code-origin routing bypasses normal portal auth, bootstrap creates a separate Code viewer session, and validation accepts that session without a current portal-session check. I did not run a live Worker reproduction.

## Solution Assessment

Yes in direction: server-side revocation or active portal-session binding is the narrow maintainable fix because clearing only same-origin browser cookies would not cover isolated hosts or replayed stored sessions.
