## Summary

Keep open: current main still returns raw Upstash Box non-2xx response bodies from the shared API error helper, so the configured API key can be exposed if the provider or an intermediary echoes it in an error body.

## What This Changes

The issue requests redaction of the configured Upstash Box API key from provider API error bodies before Crabbox returns or prints diagnostics.

## Best Possible Solution

Make the Upstash Box client redact the trimmed configured API key from every API response-body error, and add focused socket-free regression tests for doJSON, ExecStream, and UploadFile failure paths.

## Reproduction Assessment

Yes from source inspection: a socket-free RoundTripper can return a non-2xx body containing the configured key and the current apiError path will include that body unchanged. I did not run tests because this review must keep the checkout read-only.

## Solution Assessment

Yes; passing the configured API key into the Upstash Box error formatter, for example by making apiError a client method, is the narrowest maintainable repair and matches neighboring provider patterns.
