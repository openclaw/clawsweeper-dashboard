## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Roll back OnAcquired failures even with --keep:** `internal/providers/scaleway/provider.go:376`
  - body: `OnAcquired` runs after the Scaleway server/key exist but before the local lease claim is written. If it returns an error with `--keep`, the deferred cleanup records `kept-after-failure` instead of deleting the resource, even though the `AcquireRequest` contract requires provider rollback for `OnAcquired` errors; a failed controller acknowledgment can therefore leave a live VM/key that the controller never accepted.
  - confidence: 0.95
- **[P2] Preserve ambiguous slug errors during release:** `internal/providers/scaleway/provider.go:412-419`
  - body: When live inventory returns an ambiguous slug error, `ReleaseOnly` suppresses that error and falls through to local claim lookup. `crabbox stop --provider scaleway <slug>` can then release whichever claim is found even though the live provider state already proved the slug is ambiguous, so release-only resolution should still return the alias error unless the request is an exact cloud-ID match.
  - confidence: 0.9
- **[P2] Persist Tailscale metadata before advertising it:** `internal/providers/scaleway/provider.go:38`
  - body: The provider advertises `FeatureTailscale`, but the backend does not implement `TailscaleMetadataBackend.UpdateTailscaleMetadata`. Current direct-provider Tailscale metadata is persisted only through that interface, so a Tailscale-enabled Scaleway lease can later resolve without `tailscale_ipv4` or `tailscale_fqdn` and fail under `--network tailscale`.
  - confidence: 0.9
- **[P2] Preserve SDK-selected Scaleway region and zone:** `internal/providers/scaleway/client.go:134-139`
  - body: `cfgForRun` fills `cfg.Scaleway.Region` and `Zone` with Crabbox defaults before `newClient`, so these checks always overwrite `SCW_DEFAULT_REGION`/`SCW_DEFAULT_ZONE` or the active SDK profile unless users also set Crabbox-specific overrides. Only apply Crabbox region/zone when explicitly configured, or merge SDK env/profile after provider defaults.
  - confidence: 0.87
