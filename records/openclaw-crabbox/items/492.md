## Summary

Keep open: current main still formats OpenComputer provider-controlled non-2xx error text without redacting the configured API key, and no open or merged same-repo PR was found that owns this specific leak.

## What This Changes

The issue asks Crabbox to redact the configured OpenComputer API key from provider-controlled API error text before surfacing diagnostics.

## Best Possible Solution

Redact the configured OpenComputer API key from all OpenComputer API error text, then add focused regression coverage for both JSON API errors and file-upload errors.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a non-2xx OpenComputer response whose JSON error or raw body contains the configured key will be formatted by apiError without redaction.

## Solution Assessment

Yes. The narrow maintainable fix is provider-local redaction in the OpenComputer API error path, not a core behavior change or new configuration option.
