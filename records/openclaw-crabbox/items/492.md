## Summary

Keep open: current main still formats OpenComputer provider-controlled non-2xx error text without redacting the configured API key, and no linked fix PR or canonical replacement was found.

## What This Changes

The issue asks Crabbox to redact the configured OpenComputer API key from provider-controlled non-2xx OpenComputer API error diagnostics.

## Best Possible Solution

Fix this in the OpenComputer provider client by redacting the exact configured API key and common API-key header forms before constructing ocAPIError, with focused regression coverage for JSON and upload non-2xx paths.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: return a non-2xx OpenComputer response whose JSON error contains the configured key, and apiError will format that message without redaction.

## Solution Assessment

Yes. The narrow maintainable fix is provider-local redaction in the OpenComputer API error path, not a core behavior change or new configuration option.
