## Summary

Keep open: current main does not implement this feature, and this PR needs repair before merge because it advertises Islo as tailnet-capable while the userspace-only bring-up does not satisfy Crabbox's existing peer-mesh contract and it silently drops existing Tailscale options.

## What This Changes

The PR adds exec-driven Tailscale bring-up for Islo leases, records tailnet endpoints on claims, updates pond peer fallback behavior, and documents/tests the dual-plane behavior.

## Best Possible Solution

Make Islo satisfy the existing tailnet peer contract for ordinary sandbox traffic and .cbx names, including exit-node/login-server behavior, or expose this as a clearly limited userspace/SOCKS mode instead of FeatureTailscale.

## Reproduction Assessment

Yes for the PR defects from source: the patch shows userspace Tailscale plus FeatureTailscale advertisement and omits existing exit-node/login-server flags. I did not run a live Islo tailnet setup.

## Solution Assessment

No: the feature direction is useful, but the implementation currently overstates the transport capability and leaves existing Tailscale settings unsupported. A safer solution is to meet the existing tailnet contract or use a separate limited capability.
