## Summary

Keep open: current main does not have local-container native checkpoints, and the latest PR still has a blocking daemon-scope bug where a checkpoint recorded with `DOCKER_HOST` can later verify/delete against an ambient `DOCKER_CONTEXT`.

## What This Changes

The PR adds opt-in local-container native `docker-commit` checkpoint create/verify/delete support with docs, unit tests, and a Docker-backed proof test.

## Best Possible Solution

Keep the opt-in create/verify/delete design, but make recorded `DOCKER_HOST` override or clear ambient `DOCKER_CONTEXT` during verify/delete and add regression coverage before merge.

## Reproduction Assessment

Yes. Source inspection shows `dockerCommitCmd` preserves ambient `DOCKER_CONTEXT` while appending recorded `DOCKER_HOST`, and Docker's CLI contract gives `DOCKER_CONTEXT` precedence.

## Solution Assessment

No. The feature shape is reasonable, but the daemon-scope replay must make the recorded host authoritative before this is the narrow maintainable solution.
