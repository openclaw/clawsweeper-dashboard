## Summary

Keep open: current main does not implement local-container native checkpoints, but this PR is not merge-ready because docker-commit checkpoints cannot fork through the current provider contract and docker-socket work roots need archive fallback or explicit capture. Real behavior proof is also missing.

## What This Changes

The branch adds a `docker-commit` native checkpoint kind, direct Docker commit/delete/verify handling, local-container checkpoint/fork feature declarations, and unit tests for capability selection.

## Best Possible Solution

Land a provider-owned implementation that sets the committed image as the fork base, preserves archive fallback for host-mounted work roots or captures them explicitly, and includes real create/fork/delete proof for a Docker-compatible runtime.

## Reproduction Assessment

Yes from source inspection: after a docker-commit checkpoint is recorded as native, `checkpoint fork` must call `applyNativeCheckpointForkConfig`, which errors because local-container does not implement `NativeCheckpointForkProvider`. I did not run Docker integration because this review is read-only.

## Solution Assessment

No: the direction is useful, but the current implementation advertises fork support before providing the provider fork hook and changes auto behavior for docker-socket leases without preserving workspace state.

- **[P1] Implement the fork config before advertising fork support:** `internal/providers/localcontainer/provider.go:27`
  - body: Adding `FeatureFork` exposes docker-commit checkpoints as forkable, but `checkpoint fork` calls `applyNativeCheckpointForkConfig`, which requires the selected provider to implement `NativeCheckpointForkProvider`. This provider only adds `NativeCheckpointCapability`, so every docker-commit fork exits before `docker run` can use the image.
  - confidence: 0.94

  - body: The new capability is unconditional for any local-container with a CloudID. Existing docker-socket leases mount the work root from the host, so switching auto mode from archive to `docker commit` does not capture that workspace; guard `req.Config.LocalContainer.DockerSocket` or snapshot and restore the host work root before advertising native support for that mode.
  - confidence: 0.86
