## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Use the nested local-container Docker socket flag:** `internal/providers/localcontainer/provider.go:54`
  - body: `core.Config` does not have a top-level `DockerSocket` field; the existing field is `Config.LocalContainer.DockerSocket`. As written, the provider code and added test fixture should fail to compile, so this needs to use the nested local-container config in both places.
  - confidence: 0.98
- **[P2] Keep auto mode aligned with archive fallback:** `internal/cli/checkpoint_test.go:606`
  - body: Current checkpoint routing only considers direct native providers in auto mode when a non-auto strategy is explicitly requested, and the docs say direct providers other than Parallels keep archive fallback by default. This test either fails against the current code or pressures a compatibility-breaking default change; make docker-commit opt-in through `--mode native` unless maintainers explicitly choose a new default.
  - confidence: 0.9
- **[P1] Record immutable Docker image identity:** `internal/cli/checkpoint_native.go:167`
  - body: Every create uses the mutable tag `crabbox-checkpoint-<name>`, while verify/delete prefer the recorded tag over the immutable image ID. If a user creates another checkpoint with the same name, the old record can verify or remove the newer image; record a digest-unique reference or make verify/delete prefer immutable identity.
  - confidence: 0.92
- **[P2] Persist the Docker runtime and daemon scope:** `internal/cli/checkpoint.go:1185`
  - body: Create uses the configured runtime at checkpoint time, but verify/delete reload the current config and run against whatever runtime or Docker daemon is active later. A checkpoint created with podman, nerdctl, a Docker context, or a different daemon can be reported missing or deleted from the wrong place after config changes.
  - confidence: 0.84
- **[P2] Do not advertise fork before local-container can fork:** `internal/cli/checkpoint.go:1192`
  - body: The branch intentionally leaves local-container fork support to https://github.com/openclaw/crabbox/pull/208, but successful docker-commit verify reports `fork_or_delete`. Until this PR implements the provider fork hook, the audit should only point users at the supported delete/local actions.
  - confidence: 0.88
