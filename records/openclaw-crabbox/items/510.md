## What This Changes

The PR changes signed user token parsing to require one canonical two-segment `cbxu_<payload>.<signature>` shape and adds auth, Code portal, and docs coverage for rejecting suffixed spellings.

## Best Possible Solution

Land the narrow parser hardening with the auth and Code portal regression tests after policy cleanup of the release note and new test example values.

## Reproduction Assessment

Yes. Source inspection on current main shows `verifyUserToken` ignores trailing signed-token segments while Code portal logout revocation keys are based on the raw token string; I did not run tests because the review had to keep the checkout read-only.

## Solution Assessment

Yes. Rejecting noncanonical token shapes at the signed-token parser is the narrow maintainable fix because Crabbox only mints `cbxu_<payload>.<signature>` tokens and downstream revocation keys depend on raw token strings.
