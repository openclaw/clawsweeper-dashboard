## Summary

Keep open: current main still has the reported GCP label-only inventory and workspace-recovery fallback, and the canonical-search pass did not find a merged or open GCP-specific fix that supersedes this issue.

## What This Changes

The issue reports that the GCP provider trusts forgeable `crabbox=true` and `lease` labels for admin pool inventory and workspace recovery.

## Best Possible Solution

Add provider-owned GCP authenticity checks and recovery hooks so core can recover leases by verified cloud identity instead of adopting resources from forgeable labels.

## Reproduction Assessment

Yes. Source inspection on current main shows GCP inventory is selected by `labels.crabbox = true`, and workspace recovery falls back to adopting a listed machine whose labels contain the target lease id when the provider wrapper lacks stronger lookup hooks.

## Solution Assessment

Unclear. The suggested direction is sound, but the exact durable authenticity record and compatibility behavior for existing GCP leases need maintainer security review.
