## Summary

Keep open: current main and latest release still have the existing lease/run/job lifecycle, but no Station command, station profile config, station evidence model, or scoped model-access surface. This is directionally aligned roadmap work, but it needs maintainer product and security decisions before implementation, especially because it depends on https://github.com/openclaw/crabbox/pull/190.

## What This Changes

Request to add a Station lifecycle primitive with station profiles, then an agent profile that grants explicit scoped model/tool access for long-running repo-owned workloads.

## Best Possible Solution

Keep this as an open roadmap item, settle https://github.com/openclaw/crabbox/pull/190 first, then specify and ship generic Station without model credentials before adding explicit, revocable model access.

## Solution Assessment

Unclear as an implementation path until maintainers decide the lifecycle, evidence, storage, and model-access threat model. The proposed boundary is directionally aligned, but the safest route is generic Station first and explicit model access later.
