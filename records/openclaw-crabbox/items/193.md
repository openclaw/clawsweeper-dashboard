## Summary

Keep open: current main and v0.26.0 still expose warmup/run/job primitives rather than Station, and there is no stationProfile or modelAccess config/API surface yet. This remains a broad product and security roadmap item, with the named harness dependency still open in https://github.com/openclaw/crabbox/pull/190.

## What This Changes

Request to add a durable Station lifecycle primitive, an agent station profile, and later scoped modelAccess credential delivery for supervised long-running workloads.

## Best Possible Solution

Keep this open as a maintainer-owned roadmap: settle the Station lifecycle contract first, land or reject the harness dependency, then split modelAccess into a separate security-reviewed design after Station is reliable.

## Reproduction Assessment

## Solution Assessment

Unclear as an implementation plan until maintainers decide the storage, supervisor, status, restart, credential, and egress boundaries. The phased direction of Station before modelAccess is the safer shape to preserve.
