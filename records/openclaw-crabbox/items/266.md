## Summary

This is a valid, currently unimplemented provider proposal that fits Crabbox’s delegated-run extension model, but it requires maintainer decisions on product scope, dependency ownership, credential routing, and the minimum supported v1 capability set before implementation.

## What This Changes

Add a first-class OpenSandbox delegated-run provider with lifecycle management, archive-based workspace sync, secret-safe authentication, local ownership claims, configuration, documentation, and live smoke coverage.

## Best Possible Solution

Approve a deliberately narrow Linux delegated-run v1 contract, then implement OpenSandbox entirely behind its provider adapter with trusted endpoint selection, off-argv credentials, ownership verification, focused unit coverage, provider documentation, and redacted live lifecycle proof.

## Reproduction Assessment

Not applicable; this requests a new provider rather than reporting broken existing behavior, and current-main inspection confirms the capability is absent.

## Solution Assessment

The delegated-run model is the appropriate existing extension point, but the full proposed surface is not yet a settled best implementation because the SDK strategy, trusted endpoint rules, and minimum v1 capabilities require maintainer judgment.
