## Summary

Keep this PR open: current main does not already provide a Freestyle provider, and the branch has useful live proof plus fixes for the earlier auth, argv, sync-only, and actions-runner blockers. One merge-blocking bug remains in Freestyle env forwarding: it shell-quotes environment variable names, which makes Bash treat normal assignments like `'FOO'='bar'` as a command instead of exporting `FOO`.

## What This Changes

The PR adds a built-in Freestyle delegated-run provider with REST lifecycle calls, archive sync, CLI config/env wiring, registration, tests, docs, and a changelog entry.

## Best Possible Solution

Land the Freestyle provider after env forwarding uses validated shell-safe assignments or an env profile with regression coverage, while preserving env-only auth and the delegated-provider guardrails already added.

## Reproduction Assessment

Yes for the review finding: source inspection shows the generated env assignment shape, and a local Bash check confirms quoted assignment names fail before the user command can receive the variable. The live Freestyle path itself was not independently rerun in this read-only review.

## Solution Assessment

No: the provider direction is plausible, but the env forwarding implementation is not the narrowest maintainable solution yet. It should follow adjacent provider patterns by using validated unquoted variable names with quoted values, or by uploading and sourcing an env profile.
