On https://github.com/openclaw/crabbox/pull/148, keep the Freestyle provider implementation but fix the no-sync workspace preservation bug. `Run` currently calls `prepareWorkspace` for `NoSync`, and `prepareWorkspace` uses `cfg.Sync.Delete`; because the default is true, an existing kept Freestyle lease can have `/workspace/<freestyle.workdir>` removed before the command. Change Freestyle so normal sync can still delete when configured, but `--no-sync` only ensures the workdir exists. Add a focused regression test similar to Modal's `TestRunNoSyncDoesNotDeleteExistingWorkspace` with `cfg.Sync.Delete=true`, an existing lease/id path, and `NoSync: true`; assert the prepare command does not contain `rm -rf` and still creates the directory. Remove the `CHANGELOG.md` edit because release notes are release-owned. Do not change the Freestyle API shape or auth behavior beyond this repair.
