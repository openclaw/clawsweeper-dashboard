Overall confidence: 0.89

Full review comments:

- **[P1] Remove the API key command-line flag:** `internal/providers/freestyle/backend.go:55`
  - body: This registers `--freestyle-api-key` and applies it into config, which exposes the provider token through process listings and shell history. Existing provider policy keeps API keys env-only, so please remove this flag path and add a regression test that no Freestyle key/token flag is registered.
  - confidence: 0.96
- **[P2] Preserve argv when building Freestyle commands:** `internal/providers/freestyle/backend.go:359`
  - body: `freestyleExecCommand` always joins argv with spaces and ignores the existing shell/argv helpers, so non-shell commands with spaces, metacharacters, or leading env assignments can run differently from other providers. Use the existing `shouldUseShell`/`shellScriptFromArgv`/`shellWords` pattern and cover a spaced-argument case.
  - confidence: 0.9
- **[P2] Reject unsupported Actions runner warmups:** `internal/providers/freestyle/backend.go:99`
  - body: Delegated warmup receives `ActionsRunner` directly, and this implementation creates a VM and reports success without registering a GitHub Actions runner. Add the same early unsupported error used by other delegated providers so `warmup --actions-runner` does not silently allocate an unusable sandbox.
  - confidence: 0.86
- **[P2] Honor the archive-sync sync-only contract:** `internal/providers/freestyle/provider.go:24`
  - body: The provider advertises `FeatureArchiveSync`, which core validation treats as support for `--sync-only`, but the backend later rejects `SyncOnly`. Either implement sync-only after uploading into a kept sandbox or stop advertising archive-sync and adjust the related capability behavior intentionally.
  - confidence: 0.82
