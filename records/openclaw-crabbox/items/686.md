## Summary

Keep open: current main still downloads and uses the Windows Node MSI and Docker Engine zip in the documented developer-image prep path without pinned digest verification, and no related PR currently owns the Node/Docker repair.

## What This Changes

The issue requests pinned integrity verification for Node and Docker artifacts downloaded by the Windows developer-tools image preparation script.

## Best Possible Solution

Add reviewed SHA-256 verification before Node MSI installation and Docker zip extraction/service registration, with an explicit policy for non-default versions.

## Reproduction Assessment

Yes, source-reproducible: the current prep script downloads the Node MSI and Docker zip and uses them before any digest verification. A live Windows image bake was not run in this read-only review.

## Solution Assessment

Yes in direction, but not fully specified: the narrow maintainable fix is a shared SHA-256 assertion before privileged use plus a maintainer-approved rule for non-default versions.
