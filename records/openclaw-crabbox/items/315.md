## Summary

Keep open: the PR addresses a real security-hardening gap that is still present on current main and v0.29.0, with credible direct live proof, but the release-owned CHANGELOG.md edit should be removed before merge.

## What This Changes

This PR changes the Go and Worker Tailscale bootstrap generators, matching tests and docs, so auth keys are piped to `tailscale up --auth-key=file:/dev/stdin` instead of being passed through argv.

## Best Possible Solution

Land the stdin auth-key bootstrap change with regression coverage and docs, while leaving CHANGELOG.md updates to the release process.

## Reproduction Assessment

Yes. Current main source and tests still generate `tailscale up --auth-key="$TS_AUTHKEY"` in both direct and Worker bootstrap paths, and the linked issue is source-reproducible.

## Solution Assessment

Yes, with one cleanup. Matching Islo's `file:/dev/stdin` pattern in both generators is the narrow maintainable fix, while the changelog line should be dropped from this normal PR.
