## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Honor sync.include before building SmolVM archives:** `internal/providers/smolvm/sync.go:21`
  - body: syncWorkspace builds the SmolVM archive with syncManifest(req.Repo.Root, excludes), and the local wrapper calls core.BuildSyncManifest rather than the filtered manifest path. That drops cfg.Sync.Includes, so a repo that whitelists a few paths can still upload non-whitelisted files to the hosted Smol Machines API. Thread the include list through this path and add regression coverage.
  - confidence: 0.93
- **[P1] Write SmolVM stderr to stderr:** `internal/providers/smolvm/client.go:255-257`
  - body: ExecStream appends result.Stderr to the same writer used for stdout, and backend.Run passes b.rt.Stdout there. Commands that emit diagnostics will contaminate stdout and can break callers expecting stdout/stderr separation. Pass a stderr writer through the SmolVM client/backend path and cover it with a test.
  - confidence: 0.91
