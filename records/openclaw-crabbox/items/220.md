## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Validate SmolVM base URLs before sending secrets:** `internal/providers/smolvm/client.go:151-152`
  - body: `newAPI` accepts `cfg.Smolvm.BaseURL` after trimming only, then `doJSON` builds requests from that value while `addHeaders` attaches the bearer token. That lets a configured non-HTTPS or malformed endpoint receive SmolVM credentials plus archive/exec payloads; parse the URL and require HTTPS or loopback HTTP, and reject userinfo/query/fragment like the other authenticated providers do.
  - confidence: 0.94
- **[P1] Honor sync.include when building SmolVM archives:** `internal/providers/smolvm/sync.go:20-23`
  - body: `syncWorkspace` calls the SmolVM `syncManifest` wrapper with only excludes, and that wrapper uses `BuildSyncManifest` instead of the filtered manifest API. Users who configured `sync.include` expect only whitelisted paths to upload, but this path archives the whole tree minus excludes and sends extra files to the hosted provider.
  - confidence: 0.96
- **[P2] Decode snake_case exec exit codes:** `internal/providers/smolvm/client.go:58-60`
  - body: The duplicated struct tag on `ExitCode` only exposes `exitCode` to `encoding/json`; `exit_code` is ignored. If SmolVM returns the snake_case field, a failed command decodes as zero and Crabbox can report delegated commands, archive extraction, or env writes as successful.
  - confidence: 0.86
