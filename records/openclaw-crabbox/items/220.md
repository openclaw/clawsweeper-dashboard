## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Honor sync.include before building SmolVM archives:** `internal/providers/smolvm/sync.go:21`
  - body: `syncWorkspace` builds the archive from `syncManifest(req.Repo.Root, excludes)`, which ignores `cfg.Sync.Includes`. Users who set `sync.include` to whitelist paths would still upload the whole non-excluded checkout to SmolVM, breaking the documented sync contract and potentially sending files they intentionally left out.
  - confidence: 0.96
- **[P1] Decode snake_case exec exit codes explicitly:** `internal/providers/smolvm/client.go:59`
  - body: This struct tag only lets `encoding/json` bind the camelCase `exitCode` field; the second `json:"exit_code"` tag is ignored. If the API returns `exit_code`, Crabbox will leave `ExitCode` at zero and treat a failed SmolVM command as successful.
  - confidence: 0.94
- **[P2] Send remote stderr to the stderr stream:** `internal/providers/smolvm/client.go:255-256`
  - body: `ExecStream` still receives only a stdout writer and writes `result.Stderr` into it. Since `Run` passes `b.rt.Stdout`, SmolVM command diagnostics are mixed into stdout instead of going through the runtime stderr path.
  - confidence: 0.92
