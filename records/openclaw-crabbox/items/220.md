Full review comments:

- **[P1] Decode snake_case exec exit codes:** `internal/providers/smolvm/client.go:59`
  - body: `encoding/json` will not use both tags in `json:"exitCode" json:"exit_code"`; if the SmolVM exec API returns `exit_code`, `ExitCode` stays zero and a failing remote command can be reported as success. Add a separate field or custom unmarshal path and cover a nonzero snake_case response.
  - confidence: 0.91
- **[P2] Route remote stderr to stderr:** `internal/providers/smolvm/client.go:255-256`
  - body: `ExecStream` receives only the stdout writer and writes `result.Stderr` to it, so SmolVM command diagnostics are delivered on stdout instead of stderr. Thread a stderr writer through this path and update the regression test so stdout and stderr remain separate.
  - confidence: 0.92
