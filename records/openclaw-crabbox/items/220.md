## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Validate SmolVM API origins before sending bearer tokens:** `internal/providers/smolvm/client.go:151-152`
  - body: `newAPI` accepts any configured `cfg.Smolvm.BaseURL`, while the client later adds the SmolVM bearer token to every request. Because that base URL can come from repo config, env, or a flag, a malicious or mistaken config can send `smk_` credentials to an arbitrary origin unless this is hosted-only by default or guarded behind an explicit unsafe/self-hosted opt-in with tests.
  - confidence: 0.91
- **[P2] Route SmolVM stderr to the runtime stderr writer:** `internal/providers/smolvm/client.go:255-257`
  - body: `ExecStream` appends `result.Stderr` to the same writer used for stdout, and `Run` passes `b.rt.Stdout`, so a successful command that emits diagnostics on stderr contaminates stdout. This breaks stdout parsing/capture expectations; pass a separate stderr writer or otherwise keep the streams distinct and update the regression test that currently expects `hi\nwarn\n` on stdout.
  - confidence: 0.88
