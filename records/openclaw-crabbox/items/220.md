Full review comments:

- **[P1] Decode snake_case exec exit codes:** `internal/providers/smolvm/client.go:59`
  - body: If the SmolVM API returns `exit_code`, this duplicate struct tag still leaves `ExitCode` at zero because `encoding/json` will not use the second `json` tag. A failed remote command can therefore flow through `Run` as success, including timing JSON with exit 0, so add a real snake_case field/custom unmarshal and a nonzero regression.
  - confidence: 0.92
- **[P1] Send remote stderr to Runtime.Stderr:** `internal/providers/smolvm/client.go:255-256`
  - body: `ExecStream` only receives the stdout writer and writes `result.Stderr` into it, while `Run` passes `b.rt.Stdout` as that writer. Users redirecting or parsing stdout will get diagnostics mixed into stdout; pass a separate stderr writer through the API seam and cover it with a regression.
  - confidence: 0.93
