## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Validate SmolVM API URLs before adding bearer credentials:** `internal/providers/smolvm/client.go:151`
  - body: `newAPI` accepts `cfg.Smolvm.BaseURL` verbatim and `doJSON` then attaches the bearer token to requests built from that string. A config typo or malicious config can send the SmolVM API key plus archive/env payloads over plain HTTP or to an unintended host; match the existing delegated-provider pattern by parsing the base URL up front and requiring HTTPS except trusted loopback tests.
  - confidence: 0.92
- **[P1] Decode snake_case SmolVM exec exit codes:** `internal/providers/smolvm/client.go:59`
  - body: This tag does not make Go decode both `exitCode` and `exit_code`; `reflect.StructTag.Get("json")` only returns the first json tag. If the hosted exec API returns snake_case, a failing command decodes with the zero-value exit code and Crabbox reports success, so the response needs an explicit fallback field or custom decode plus coverage for `exit_code`.
  - confidence: 0.95
