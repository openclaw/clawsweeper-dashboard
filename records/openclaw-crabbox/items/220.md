## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Honor sync.include before archiving SmolVM workspaces:** `internal/providers/smolvm/sync.go:21`
  - body: This path calls syncManifest(...), which maps to BuildSyncManifest without the configured include whitelist. A user with sync.include set expects only whitelisted paths to be uploaded, but SmolVM would still tar the full tracked/nonignored workspace and send it to the hosted API. Use the filtered manifest path and add regression coverage.
  - confidence: 0.96
- **[P1] Validate the SmolVM API URL before adding auth:** `internal/providers/smolvm/client.go:151`
  - body: newAPI accepts smolvm.baseUrl/CRABBOX_SMOLVM_BASE_URL after trimming only, and doJSON later sends Authorization plus archive/env bodies to c.base + path. Reject missing scheme/host and unsafe schemes before constructing the client so config mistakes cannot receive credentials or workspace payloads.
  - confidence: 0.9
- **[P1] Decode snake_case SmolVM exit codes:** `internal/providers/smolvm/client.go:59`
  - body: The tag `json:"exitCode" json:"exit_code"` does not make encoding/json read both names; StructTag.Get returns the first json key. If /exec returns `exit_code`, failed commands and archive/write steps decode as zero and look successful. Add a custom unmarshal or alias field with tests.
  - confidence: 0.92
