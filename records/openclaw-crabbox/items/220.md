## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Validate SmolVM base URLs before sending bearer tokens:** `internal/providers/smolvm/client.go:151-152`
  - body: `newAPI` copies `cfg.Smolvm.BaseURL` directly into the client, and `doJSON` later attaches the SmolVM bearer token before create/exec/archive/write calls. Because this value can come from config/env/flags, a checked-in or accidental value can send credentials and payloads to an unintended endpoint; validate the scheme/host or allow-list the trusted SmolVM API before any request.
  - confidence: 0.9
- **[P2] Decode snake_case exec exit codes before trusting success:** `internal/providers/smolvm/client.go:59`
  - body: `encoding/json` only uses the first `json` key in this struct tag, so a response with `exit_code` leaves `ExitCode` at zero. That can make failed archive injection, env writes, or commands look successful; add an alias/custom unmarshal path and a nonzero `exit_code` regression test.
  - confidence: 0.93
