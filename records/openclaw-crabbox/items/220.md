Full review comments:

- **[P1] Validate SmolVM base URLs before sending secrets:** `internal/providers/smolvm/client.go:151-152`
  - body: `newAPI` accepts `cfg.Smolvm.BaseURL` directly, and `doJSON` then sends `Authorization` plus exec/archive JSON bodies to `c.base`.   - confidence: 0.93
- **[P1] Honor sync.include when building SmolVM archives:** `internal/providers/smolvm/sync.go:21`
  - body: `syncWorkspace` builds the archive manifest with `syncManifest(req.Repo.Root, excludes)`, whose wrapper calls `BuildSyncManifest` and ignores `cfg.Sync.Includes`. Users who configured a whitelist can still send the full tracked/dirty manifest to hosted SmolVM; pass includes through `BuildSyncManifestFiltered` and add regression coverage.
  - confidence: 0.95
- **[P2] Decode snake_case exec exit codes:** `internal/providers/smolvm/client.go:59`
  - body: This struct tag has two `json` keys, so `encoding/json` will decode only `exitCode`; if the API returns `exit_code`, `ExitCode` stays zero and failed delegated commands can look successful. Add an explicit snake_case fallback or custom unmarshal plus a regression test.
  - confidence: 0.91
