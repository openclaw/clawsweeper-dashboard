## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Decode snake_case exec exit codes:** `internal/providers/smolvm/client.go:59`
  - body: This struct tag only gives `encoding/json` one usable `json` key, so an exec response shaped as `{"exit_code":1,...}` leaves `ExitCode` at zero. That makes `run`, archive injection, and file writes capable of reporting success after a remote command actually failed; add a real alias/custom decode path and a non-zero regression test.
  - confidence: 0.9
- **[P2] Route SmolVM stderr away from stdout:** `internal/providers/smolvm/client.go:255-256`
  - body: `Run` passes `Runtime.Stdout` into `ExecStream`, and this branch appends `result.Stderr` to that same writer. A command like `cmd 2>err >out` through Crabbox will leak remote stderr into stdout, so the provider needs a stderr writer path or equivalent backend handling before merge.
  - confidence: 0.92
