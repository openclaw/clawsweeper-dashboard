## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Decode snake_case exec exit codes:** `internal/providers/smolvm/client.go:59`
  - body: `execResult.ExitCode` currently has two `json` tags, but Go's decoder only uses one field name. If SmolVM returns `exit_code`, failed remote commands decode as exit 0 and `Run`/archive injection can report success; add an explicit snake_case field or custom unmarshal plus a non-zero regression test.
  - confidence: 0.91
- **[P2] Validate base URLs before sending bearer tokens:** `internal/providers/smolvm/client.go:151-152`
  - body: `newAPI` accepts `cfg.Smolvm.BaseURL` as-is and `doJSON` then sends the `Authorization: Bearer` header to that base. Reject invalid URLs and non-HTTPS non-loopback URLs, matching the safer RunPod/Railway/Cloudflare pattern, so a typo or config override cannot leak `smk_*` credentials over plaintext.
  - confidence: 0.86
