Full review comments:

- **[P1] Validate SmolVM base URLs before sending tokens:** `internal/providers/smolvm/client.go:151`
  - body: `newAPI` trusts `cfg.Smolvm.BaseURL` and the same client later adds the Smol Machines bearer token to every request. Because this PR wires that base URL through config/env/flags, a project config or override can redirect the user's hosted API token and archive/env payloads to an unintended endpoint; validate or allow-list the base URL before constructing authenticated requests and add rejection tests.
  - confidence: 0.92
- **[P2] Decode snake_case exec exit codes:** `internal/providers/smolvm/client.go:59`
  - body: The duplicate struct tag only gives `encoding/json` the `exitCode` name; it does not populate `ExitCode` from `exit_code`. Since `ExecStream` returns this field directly, a response shaped with `exit_code: 1` can be reported as exit 0, so add explicit decoding for both response shapes plus a regression test.
  - confidence: 0.86
