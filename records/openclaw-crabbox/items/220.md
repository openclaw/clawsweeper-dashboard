## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Decode snake_case exec exit codes:** `internal/providers/smolvm/client.go:59`
  - body: `ExitCode` has two `json` keys in the struct tag, but Go's JSON decoder uses the first `json` tag value, so an API response containing `exit_code` leaves this field at zero. That can make a failing delegated exec look successful, including cleanup and timing paths that rely on the returned exit code.
  - confidence: 0.93
- **[P2] Preserve exec stderr as stderr:** `internal/providers/smolvm/client.go:255-257`
  - body: This writes the API's stderr payload to the stdout writer, so `crabbox run` will mix command diagnostics into stdout. That breaks stream separation for users and scripts that consume stdout; pass a stderr writer through the client/backend path or otherwise emit `result.Stderr` to `b.rt.Stderr`.
  - confidence: 0.9
