Full review comments:

- **[P1] Decode snake_case exec exit codes:** `internal/providers/smolvm/client.go:59`
  - body: `encoding/json` only uses the first `json` tag here, so an API response like `{"exit_code":7}` leaves `ExitCode` at zero. That can make failed archive injection, file writes, or user commands look successful; add an explicit fallback/custom decode path and a regression test for snake_case failure responses.
  - confidence: 0.9
- **[P2] Route SmolVM stderr to stderr:** `internal/providers/smolvm/client.go:255-256`
  - body: `Run` passes only `Runtime.Stdout` into `ExecStream`, and this code appends `result.Stderr` to that same writer. Commands that write diagnostics to stderr will be emitted on stdout, so thread a stderr writer through the API or write returned stderr to `Runtime.Stderr`.
  - confidence: 0.93
