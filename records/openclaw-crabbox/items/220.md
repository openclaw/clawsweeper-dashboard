## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use the shared safe sync archive writer:** `internal/providers/smolvm/sync.go:113`
  - body: This new provider copies the old raw `tar --null -T -` archive path, but current `main` replaced that pattern with `core.CreateSyncArchive` because option-like filenames such as `--checkpoint-action=...` must be archived as data, not interpreted by local tar. Please route SmolVM through the shared archive helper and add the same option-like filename coverage so running SmolVM sync on an untrusted repo cannot execute local tar options.
  - confidence: 0.92
- **[P1] Validate SmolVM API URLs before sending bearer auth:** `internal/providers/smolvm/client.go:151`
  - body: `cfg.Smolvm.BaseURL` can come from config or env, but `newAPI` accepts it verbatim and every request later attaches `Authorization: Bearer ...` plus exec/archive payloads to that origin. Match the other credentialed HTTP providers by parsing the URL, rejecting userinfo/query/fragment, requiring HTTPS except loopback, and restricting custom hosts to the intended trusted/loopback set before sending credentials.
  - confidence: 0.9
- **[P1] Honor sync.include before archiving workspaces:** `internal/providers/smolvm/sync.go:21`
  - body: SmolVM builds its manifest with `syncManifest(req.Repo.Root, excludes)`, and the wrapper calls `BuildSyncManifest` without `cfg.Sync.Includes`. Users who set `sync.include` as a whitelist will still upload the rest of the repository to the hosted SmolVM API, unlike the current delegated providers that call `BuildSyncManifestFiltered(..., cfg.Sync.Includes)`.
  - confidence: 0.93
- **[P1] Decode snake_case exec exit codes:** `internal/providers/smolvm/client.go:59`
  - body: A struct tag cannot register two `json` names; `encoding/json` will use `exitCode` here and ignore `exit_code`. If SmolVM returns snake_case exit codes, failed commands decode as zero and Crabbox reports them as successful, so add an explicit fallback field or custom unmarshal with a regression test.
  - confidence: 0.88

## Security Review

- **[high] Raw tar path can interpret repo filenames as tar options:** `internal/providers/smolvm/sync.go:113`
  - body: SmolVM shells out to local tar with manifest-provided names instead of the shared safe archive writer added on current main, reintroducing the option-like filename class that the repository recently hardened.
