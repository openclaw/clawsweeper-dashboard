## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Validate SmolVM base URLs before sending tokens:** `internal/providers/smolvm/client.go:151`
  - body: `newAPI` accepts the configured SmolVM base URL verbatim, and `doJSON` later sends `Authorization: Bearer <smk key>` to `c.base + path`. A malicious or mistaken value such as a cleartext remote URL or URL with userinfo can exfiltrate provider credentials, so parse the URL and reject non-HTTPS non-loopback origins plus userinfo/query/fragment before constructing the client.
  - confidence: 0.96
- **[P2] Keep SmolVM stderr off stdout:** `internal/providers/smolvm/client.go:255-256`
  - body: `ExecStream` falls back to `Exec`, then writes `result.Stderr` to the same writer used for command stdout. That still corrupts stdout for commands that emit machine-readable output or use shell redirection, so pass stderr separately through the API/backend path and test that stdout remains clean.
  - confidence: 0.98
