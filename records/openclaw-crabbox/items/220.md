## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Honor sync.include when building SmolVM archives:** `internal/providers/smolvm/sync.go:21`
  - body: SmolVM builds its archive with `syncManifest(req.Repo.Root, excludes)`, and that wrapper calls `BuildSyncManifest` without the configured include whitelist. Users who rely on `sync.include` can upload more of the repo than intended; pass `cfg.Sync.Includes` through `BuildSyncManifestFiltered` and add regression coverage.
  - confidence: 0.96
- **[P1] Validate SmolVM base URLs before sending secrets:** `internal/providers/smolvm/client.go:151-152`
  - body: `--smolvm-base-url`/config/env are trimmed and then used for every request while `addHeaders` unconditionally attaches the bearer token. A mistyped or malicious `http://`, userinfo, query, or fragment URL can receive API keys and archive payloads; parse and reject unsafe URLs before constructing the client.
  - confidence: 0.94
- **[P2] Decode snake_case exec exit codes:** `internal/providers/smolvm/client.go:58-59`
  - body: Go only uses the first `json` tag here, so a response containing `exit_code` leaves `ExitCode` at zero. That can make failed SmolVM commands look successful; add an explicit fallback field or custom unmarshal path with a regression test.
  - confidence: 0.92
