## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Honor sync.include when building SmolVM archives:** `internal/providers/smolvm/sync.go:21`
  - body: This builds the SmolVM archive with `syncManifest(req.Repo.Root, excludes)`, and the PR-local wrapper calls `core.BuildSyncManifest` without `cfg.Sync.Includes`. Current delegated providers pass `b.cfg.Sync.Includes` into the filtered manifest helper, so a repo that whitelists only selected paths would still upload the whole tracked/unignored tree through SmolVM.
  - confidence: 0.95
- **[P1] Validate SmolVM base URLs before sending secrets:** `internal/providers/smolvm/client.go:151`
  - body: `newAPI` accepts the configured base URL verbatim, while repo config can set `smolvm.baseUrl` and `doJSON` later attaches the bearer token and archive/exec payloads to that origin. Please restrict this to the expected HTTPS host unless an explicit trusted override path is approved, otherwise an untrusted repo config can redirect local SmolVM credentials and synced content.
  - confidence: 0.86
- **[P2] Decode snake_case exec exit codes:** `internal/providers/smolvm/client.go:59`
  - body: A single struct field cannot decode both `exitCode` and `exit_code` with duplicate `json` tags, so an API response using `exit_code` leaves `ExitCode` at zero. That would make failed remote commands, archive extraction, or env-file writes look successful to Crabbox.
  - confidence: 0.88
- **[P1] Refresh the provider docs conflict on current main:** `docs/providers/README.md:38`
  - body: A merge-tree check against current main produces conflicts in provider docs/config registration; this line collides with current main's delegated-provider list that now includes `docker-sandbox`. Please rebase and preserve both current-main provider additions and the SmolVM entry before this can be reviewed as a mergeable head.
  - confidence: 0.91
