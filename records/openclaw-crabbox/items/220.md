## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Decode snake_case SmolVM exec exits:** `internal/providers/smolvm/client.go:59`
  - body: `encoding/json` will not populate `ExitCode` from an `exit_code` field through this duplicate struct tag, so any SmolVM /exec response that returns only snake_case keeps `ExitCode` at 0. Since archive injection, env writes, and command execution all key success off that value, a failed remote exec can be reported as successful; normalize both field names and add a failing regression.
  - confidence: 0.94
- **[P1] Route SmolVM stderr away from stdout:** `internal/providers/smolvm/client.go:255-256`
  - body: `ExecStream` only receives the stdout writer and writes `result.Stderr` into it, while the backend passes `b.rt.Stdout`. That corrupts user-visible and machine-readable stdout for commands that emit diagnostics; pass a separate stderr writer through the API/fakes and update the test to assert the streams stay separate.
  - confidence: 0.97
