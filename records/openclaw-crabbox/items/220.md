## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Honor sync.include when building SmolVM archives:** `internal/providers/smolvm/sync.go:21`
  - body: `syncWorkspace` builds the archive manifest with `syncManifest(req.Repo.Root, excludes)`, which drops the configured `sync.include` whitelist. Users who intentionally sync only selected paths with SmolVM would still upload the rest of the non-excluded repository; use the filtered manifest path with `syncIncludes` and add a regression test.
  - confidence: 0.93
- **[P1] Decode snake_case exit_code before reporting success:** `internal/providers/smolvm/client.go:59`
  - body: This struct tag does not make encoding/json accept both `exitCode` and `exit_code`; only `exitCode` will populate `ExitCode`. If the SmolVM API returns `{"exit_code":1,...}`, `ExecStream` returns 0 and the run is reported as successful, so add an explicit snake_case decode path and test it.
  - confidence: 0.94
- **[P1] Validate SmolVM API origins before sending tokens:** `internal/providers/smolvm/client.go:151`
  - body: `newAPI` accepts `--smolvm-base-url` / config / env verbatim, and the client later sends `Authorization: Bearer <smk token>` to that host. Match the existing RunPod/Railway pattern by parsing the URL and requiring HTTPS except for loopback test servers, or get explicit maintainer approval for a broader endpoint policy.
  - confidence: 0.88
