## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Validate SmolVM API origins before sending bearer auth:** `internal/providers/smolvm/client.go:151-152`
  - body: `smolvm.baseUrl` can come from repo config or env, and this client then sends the Smol Machines bearer token plus exec/archive request bodies to that origin. Please constrain the default authenticated client to the hosted Smol Machines origin, or require an explicit unsafe/custom-origin opt-in with tests before any payload or Authorization header is sent.
  - confidence: 0.9
- **[P1] Honor sync.include when building SmolVM manifests:** `internal/providers/smolvm/sync.go:20-22`
  - body: This builds the archive manifest with excludes only, unlike the existing delegated providers that pass `cfg.Sync.Includes`. A user who set a sync include whitelist would upload the whole non-excluded repo to SmolVM, so the SmolVM wrapper should expose/pass includes and add regression coverage.
  - confidence: 0.92
- **[P1] Decode snake_case exec exit codes:** `internal/providers/smolvm/client.go:58-59`
  - body: The duplicate struct tag does not make `encoding/json` accept both `exitCode` and `exit_code`; `exit_code` responses leave `ExitCode` at zero. That can make failed run, archive extract, or file-write exec calls appear successful, so add a separate field/custom unmarshal and cover the snake_case response shape.
  - confidence: 0.88
