- **[P1] Validate SmolVM base URLs before sending bearer tokens:** `internal/providers/smolvm/client.go:151-152`
  - body: CRABBOX_SMOLVM_BASE_URL, config, and --smolvm-base-url flow directly into c.base, and every request later attaches Authorization plus may carry archive or exec payloads.   - body: SmolVM builds its upload manifest with excludes only, so a user who configured sync.include as a whitelist will still upload the whole tracked/untracked repo set to the hosted SmolVM API.   - confidence: 0.9

  - body: The duplicate struct tag does not make encoding/json decode both exitCode and exit_code; snake_case responses leave ExitCode at zero. If SmolVM returns exit_code for a failed command, Crabbox can report success, so add an explicit fallback field or custom unmarshal plus a failing-command regression test.
  - confidence: 0.86
