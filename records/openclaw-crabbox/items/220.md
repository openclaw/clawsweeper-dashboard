## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Honor sync.include when building SmolVM archives:** `internal/providers/smolvm/sync.go:21`
  - body: SmolVM builds the archive manifest from `syncManifest(req.Repo.Root, excludes)`, which drops the existing `sync.include` whitelist. Users who configured that whitelist to limit what Crabbox uploads would send the rest of their tracked or dirty checkout to the hosted SmolVM API, unlike the other delegated archive providers that pass `cfg.Sync.Includes` into the filtered manifest path.
  - confidence: 0.96
- **[P1] Decode snake_case exec exit codes explicitly:** `internal/providers/smolvm/client.go:59`
  - body: The struct tag `json:"exitCode" json:"exit_code"` does not make `encoding/json` fill `ExitCode` from `exit_code`; it only uses the effective `exitCode` name. If the exec response uses the snake_case field, a non-zero remote command leaves `ExitCode` as 0 and `Run` reports success.
  - confidence: 0.9
- **[P2] Send remote stderr to the stderr stream:** `internal/providers/smolvm/client.go:255-256`
  - body: `ExecStream` writes `result.Stderr` to the same writer used for stdout, so command diagnostics are delivered as stdout. That can corrupt stdout-consuming workflows and hides stderr from the expected stream; route it through `Runtime.Stderr` or an explicit stderr writer.
  - confidence: 0.88
