## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Validate SmolVM base URLs before sending bearer tokens:** `internal/providers/smolvm/client.go:151-152`
  - body: `newAPI` accepts the configured base URL as-is, and every `doJSON` call then attaches `Authorization: Bearer ...` before sending lifecycle, exec, archive, and file-write payloads. Because this URL is configurable through config/env/flags, a bad value can redirect the SmolVM API key plus repository archive/env payloads to an unintended host; validate the scheme/host before constructing the client and cover rejection in tests.
  - confidence: 0.9
- **[P2] Decode snake_case exec exit codes before trusting success:** `internal/providers/smolvm/client.go:59`
  - body: The duplicate struct tag on `ExitCode` only lets `encoding/json` see `exitCode`, so an API response like `{ "exit_code": 7 }` leaves `ExitCode` at zero. That makes `Run`, archive injection, and env-file writes treat failed guest commands as success, so add an alias/custom decode path and a nonzero `exit_code` regression test.
  - confidence: 0.93
