## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Honor sync.include before archiving SmolVM workspaces:** `internal/providers/smolvm/sync.go:21`
  - body: This builds the SmolVM archive manifest with excludes only, so a repo configured with `sync.include` will still upload the whole working tree to the hosted SmolVM `/exec` path. Other delegated providers pass `b.cfg.Sync.Includes` into `BuildSyncManifestFiltered`; SmolVM should do the same and add a regression test so whitelisted sync scopes remain a security and bandwidth boundary.
  - confidence: 0.94
- **[P2] Decode snake_case SmolVM exit codes:** `internal/providers/smolvm/client.go:59`
  - body: This struct tag only registers the first `json` key, so responses that use `exit_code` leave `ExitCode` at zero. If the API returns a failed exec as `{"exit_code": 1, ...}`, `ExecStream` reports success and `Run` skips the failure path and keep-on-failure handling; add an explicit snake_case field or custom unmarshal with coverage.
  - confidence: 0.86
- **[P1] Validate SmolVM API URLs before sending bearer auth:** `internal/providers/smolvm/client.go:151-152`
  - body: `newAPI` stores the configured base URL as an unchecked string and `doJSON` then attaches the bearer token to the concatenated request URL. Before this hosted provider ships, reject malformed, non-HTTP(S), and unsafe base URLs before constructing authenticated requests so credentials and archive exec payloads are not sent to the wrong endpoint.
  - confidence: 0.81
