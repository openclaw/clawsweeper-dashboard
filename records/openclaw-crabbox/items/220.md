## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Decode snake_case exec exit codes:** `internal/providers/smolvm/client.go:59`
  - body: This field only registers the `exitCode` JSON key with `encoding/json`; the second `json` tag is ignored. If SmolVM returns `exit_code`, nonzero command, archive, or env-write failures can be reported as success, so add an explicit fallback/unmarshal path and a nonzero `exit_code` regression test.
  - confidence: 0.88
- **[P2] Send exec stderr to stderr:** `internal/providers/smolvm/client.go:255-256`
  - body: `ExecStream` gets stderr separately from the exec response, but writes it into the stdout writer used by `crabbox run`. That corrupts machine-readable stdout for commands that also emit diagnostics; route stderr through the backend stderr path or update the stream API to accept separate writers.
  - confidence: 0.82
