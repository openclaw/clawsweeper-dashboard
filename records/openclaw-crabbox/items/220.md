## Review Findings

Overall confidence: 0.94

Full review comments:

- **[P1] Honor sync.include before building SmolVM archives:** `internal/providers/smolvm/sync.go:21`
  - body: Current Crabbox sync supports `sync.include` as a whitelist, but this provider calls the unfiltered manifest builder before creating the tarball. A user who scopes sync to selected paths would still upload every tracked or unignored file to the hosted Smol Machines API, defeating the data-boundary and large-repo behavior; use the filtered manifest path and add a regression that non-included files are not archived.
  - confidence: 0.97
- **[P1] Write SmolVM stderr to stderr:** `internal/providers/smolvm/client.go:255-256`
  - body: `Run` passes only `b.rt.Stdout` into `ExecStream`, and `ExecStream` writes `result.Stderr` into that same writer. That mixes diagnostics into stdout and can break callers or scripts that parse stdout separately from stderr; pass a stderr writer through this path and update the test that currently expects `hi\nwarn\n` on stdout.
  - confidence: 0.95
