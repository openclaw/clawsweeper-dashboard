Full review comments:

- **[P1] Validate SmolVM base URLs before sending secrets:** `internal/providers/smolvm/client.go:151-152`
  - body: newAPI accepts any configured base URL and every request later adds the SmolVM bearer token. A user or config mistake such as an http:// non-loopback URL would send credentials and archive/exec payloads outside the expected HTTPS control plane; validate the URL before returning the client, following the existing HTTPS-or-loopback pattern.
  - confidence: 0.94
- **[P1] Honor sync.include when building SmolVM archives:** `internal/providers/smolvm/sync.go:21`
  - body: This builds the SmolVM archive manifest from excludes only, so a repository with sync.include configured as a whitelist will still upload files outside that whitelist to the hosted provider. Pass cfg.Sync.Includes into the manifest builder and add a regression test matching the existing delegated providers.
  - confidence: 0.95
- **[P2] Decode snake_case exec exit codes:** `internal/providers/smolvm/client.go:59`
  - body: The duplicate struct tag does not make encoding/json decode exit_code, so a response shaped like {"exit_code": 7} leaves ExitCode at zero and can make failed commands look successful. Add an explicit fallback field or custom unmarshal with coverage for both API spellings.
  - confidence: 0.91
