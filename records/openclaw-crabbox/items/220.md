Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Decode snake_case exec exit codes:** `internal/providers/smolvm/client.go:59`
  - body: The duplicate struct tag only registers `exitCode`, so a response shaped with `exit_code` leaves `ExitCode` at zero. That lets a failed remote command look successful; add an explicit snake_case field or custom unmarshal plus a nonzero regression test.
  - confidence: 0.88
- **[P1] Route SmolVM stderr to stderr:** `internal/providers/smolvm/client.go:255-257`
  - body: `ExecStream` appends `result.Stderr` to the stdout writer, and the test now expects `hi\nwarn\n` on stdout. That corrupts command stdout for any tool that emits diagnostics; pass a stderr writer through the backend/API and assert stdout remains clean.
  - confidence: 0.9
