## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Honor sync.include when building SmolVM archives:** `internal/providers/smolvm/sync.go:21`
  - body: This manifest path only applies excludes, so a repository configured with `sync.include` still uploads the rest of the working tree to the hosted SmolVM API. Other delegated providers pass `cfg.Sync.Includes` to the filtered manifest helper; SmolVM should do the same and cover it with a regression test.
  - confidence: 0.96
- **[P1] Decode snake_case SmolVM exit codes:** `internal/providers/smolvm/client.go:59`
  - body: A struct tag cannot map `ExitCode` to both `exitCode` and `exit_code`; `encoding/json` will use the first key here. If the API returns `exit_code`, failed commands decode as exit code 0 and Crabbox reports success.
  - confidence: 0.94
- **[P2] Keep remote stderr out of stdout:** `internal/providers/smolvm/client.go:255-257`
  - body: `ExecStream` appends `result.Stderr` to the stdout writer used for command output. Because the API response has a separate stderr field, this can corrupt machine-readable stdout; route it to `Runtime.Stderr` or extend the interface to pass both streams.
  - confidence: 0.9
