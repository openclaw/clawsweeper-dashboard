Overall confidence: 0.93

Full review comments:

- **[P1] Honor sync.include when building SmolVM archives:** `internal/providers/smolvm/sync.go:21`
  - body: `syncWorkspace` builds the manifest with `syncManifest(req.Repo.Root, excludes)`, and this package's wrapper calls `BuildSyncManifest` without the include whitelist. Existing delegated providers pass `cfg.Sync.Includes` into the filtered manifest helper; as written, a user who configured `sync.include` for a hosted SmolVM run still uploads the rest of the repo archive to the Smol Machines API.
  - confidence: 0.96
- **[P1] Keep SmolVM stderr out of stdout:** `internal/providers/smolvm/client.go:255-256`
  - body: `Run` passes `b.rt.Stdout` into `ExecStream`, and `ExecStream` writes `result.Stderr` to that same writer. That makes remote stderr part of command stdout for SmolVM, so redirects and structured stdout can be corrupted even though the API returned separate streams.
  - confidence: 0.94
