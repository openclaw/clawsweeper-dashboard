## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Decode snake_case exec exit codes:** `internal/providers/smolvm/client.go:59`
  - body: Go's JSON decoder will only use the first `json` tag here, so `exit_code` responses leave `ExitCode` at zero. A SmolVM exec response like `{"exit_code":1,"stderr":"failed"}` would make `Run` report success, so add explicit snake_case decoding and a non-zero regression test before merging.
  - confidence: 0.95
- **[P2] Keep remote stderr on stderr:** `internal/providers/smolvm/client.go:255-256`
  - body: `ExecStream` writes `result.Stderr` to the same writer used for stdout, and `Run` passes `b.rt.Stdout` there. That contaminates stdout for callers that parse command output; mirror the modal/wandb pattern by carrying a stderr writer through the SmolVM exec path and test separated streams.
  - confidence: 0.9
