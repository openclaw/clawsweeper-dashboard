## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Honor sync.include for SmolVM archives:** `internal/providers/smolvm/core.go:117`
  - body: This wrapper drops the configured include whitelist and calls the unfiltered manifest builder, so `sync.include` users will still upload the whole repo to the hosted SmolVM API. Existing delegated providers route includes through `BuildSyncManifestFiltered`; SmolVM needs the same path plus regression coverage.
  - confidence: 0.94
- **[P1] Decode snake_case exec exit codes:** `internal/providers/smolvm/client.go:59`
  - body: A struct tag cannot make `encoding/json` accept both `exitCode` and `exit_code`; this field only decodes the camelCase name. If the API returns `exit_code`, failed remote commands decode as exit 0 and Crabbox can report success incorrectly.
  - confidence: 0.93
- **[P2] Send SmolVM stderr to stderr:** `internal/providers/smolvm/client.go:255-256`
  - body: `ExecStream` writes `result.Stderr` to the stdout writer, so command diagnostics become part of stdout redirection/capture. Pass a stderr writer through the SmolVM API interface and call it from `Run` with `b.rt.Stderr`.
  - confidence: 0.9
