Full review comments:

- **[P1] Honor sync.include when building SmolVM archives:** `internal/providers/smolvm/sync.go:21`
  - body: This path calls the two-argument manifest wrapper, so configured `sync.include` whitelist patterns are dropped before the archive is built. Current core and adjacent delegated providers pass the include list into filtered manifest construction; with SmolVM, a selected-file sync can upload the whole tracked/unignored tree instead.
  - confidence: 0.97
- **[P1] Decode snake_case exec exit codes:** `internal/providers/smolvm/client.go:59`
  - body: `json:"exitCode" json:"exit_code"` is not a dual alias for `encoding/json`; it decodes only the `exitCode` name. If smolfleet returns `exit_code`, failed execs decode as zero and `Run` reports success, so add explicit support and coverage for both response shapes.
  - confidence: 0.92
- **[P1] Validate SmolVM API origins before sending tokens:** `internal/providers/smolvm/client.go:151-152`
  - body: The SmolVM base URL is trimmed and used directly while every request adds `Authorization: Bearer ...`. Unlike existing Cloudflare/Railway clients, this accepts remote `http`, userinfo, query, or fragment components, so a configured base URL can receive smk tokens and archive payloads over an unsafe origin.
  - confidence: 0.95
