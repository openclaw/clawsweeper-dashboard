## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Honor sync.include when building SmolVM archives:** `internal/providers/smolvm/sync.go:21`
  - body: This uses the unfiltered manifest helper, so a repo configured with `sync.include` will still archive and upload every non-excluded changed/untracked path to SmolVM. Existing delegated providers call `BuildSyncManifestFiltered` with `cfg.Sync.Includes`; the SmolVM path needs the same include-aware manifest and a regression test.
  - confidence: 0.95
- **[P1] Decode snake_case SmolVM exit codes before reporting success:** `internal/providers/smolvm/client.go:59`
  - body: This struct tag only registers `exitCode`; if the API returns the documented or legacy `exit_code` shape, `ExitCode` remains zero and `ExecStream` reports a failing command as successful. Add a separate decode path or custom response type that accepts both `exitCode` and `exit_code`, with a failing-command regression test.
  - confidence: 0.9
- **[P1] Guard custom API origins before sending SmolVM tokens:** `internal/providers/smolvm/client.go:151`
  - body: `CRABBOX_SMOLVM_BASE_URL`, config, and `--smolvm-base-url` can redirect bearer-authenticated lifecycle and exec requests to any URL. Because this provider sends `smk_*` tokens plus command/archive payloads, custom origins need an explicit maintainer-approved guard or clearly documented opt-in before merge.
  - confidence: 0.82
