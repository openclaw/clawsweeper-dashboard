## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Honor sync.include when building SmolVM archives:** `internal/providers/smolvm/sync.go:21`
  - body: This provider declares `FeatureArchiveSync`, but `syncWorkspace` builds its manifest without the configured include whitelist. Users who set `sync.include` to upload a small subset can still send the broader repository archive to the hosted SmolVM API; pass `cfg.Sync.Includes` into manifest construction and add a regression test.
  - confidence: 0.96
- **[P1] Decode snake_case exec exit codes:** `internal/providers/smolvm/client.go:59`
  - body: This struct tag only gives `encoding/json` one field name, so a smolfleet response containing `exit_code` leaves `ExitCode` at zero. That can turn a failed remote command into a successful Crabbox run; add an alias or custom unmarshal path and cover both response shapes.
  - confidence: 0.94
- **[P2] Route remote stderr to stderr:** `internal/providers/smolvm/client.go:255-257`
  - body: `ExecStream` appends `result.Stderr` to the same writer used for stdout, and the backend passes `b.rt.Stdout` there. Remote diagnostics therefore land on stdout instead of stderr; thread a stderr writer or split the streams at the backend boundary.
  - confidence: 0.93
