## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Validate SmolVM base URLs before sending secrets:** `internal/providers/smolvm/client.go:151-152`
  - body: `newAPI` accepts `cfg.Smolvm.BaseURL` verbatim, then `doJSON` sends the bearer token plus archive/env/command payloads to `c.base + path`. A config/env/flag value such as `http://remote.example` or a URL with userinfo/query/fragment would receive secrets without the HTTPS-or-loopback guard used by nearby authenticated providers, so this client should reject unsafe base URLs before construction.
  - confidence: 0.91
- **[P2] Reject unsupported delegated run options before executing:** `internal/providers/smolvm/backend.go:59-60`
  - body: `Run` starts provisioning and sync without calling the shared delegated-option guard, so flags such as `--capture-stdout`, `--download`, `--artifact-glob`, `--script`, `--stop-after`, and `--fresh-pr` can be accepted but ignored by SmolVM. Call `core.RejectDelegatedSyncOptionsForSpec` or equivalent at the start of `Run` and add regression coverage so users get a hard error instead of a successful run missing requested outputs.
  - confidence: 0.88
