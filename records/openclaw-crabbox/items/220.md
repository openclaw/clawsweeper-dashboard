  - confidence: 0.94
- **[P1] Honor sync.include when building SmolVM archives:** `internal/providers/smolvm/sync.go:21`
  - body: SmolVM builds its archive manifest with only excludes, so a user-configured `sync.include` whitelist is ignored and extra repository files can be uploaded to the hosted provider. Use the filtered manifest path with `cfg.Sync.Includes` and add a regression test.
  - confidence: 0.96

  - body: The duplicate struct tag on `ExitCode` does not make `encoding/json` accept `exit_code`, so a response containing only snake_case exit status leaves the exit code at zero. Add a separate field or custom unmarshal so failed commands cannot be reported as successful.
