Overall confidence: 0.89

Full review comments:

- **[P1] Decode snake_case exec exit codes:** `internal/providers/smolvm/client.go:59`
  - body: The duplicate struct tag here does not make encoding/json decode both names; if SmolVM returns `exit_code`, ExitCode stays at 0, so failed commands and failed archive/env writes can be reported as successful. Add an explicit snake_case field or custom unmarshal path and cover a nonzero `exit_code` response in tests.
  - confidence: 0.93
- **[P2] Send remote stderr to the stderr writer:** `internal/providers/smolvm/client.go:255-256`
  - body: ExecStream currently writes `result.Stderr` to the stdout writer, and Run calls it with only `b.rt.Stdout`, so diagnostics from the remote command are merged into stdout. That corrupts stdout for callers expecting command output and differs from existing delegated providers that keep stdout/stderr separate.
  - confidence: 0.88
