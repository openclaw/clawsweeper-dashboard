Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Decode snake_case exec exit codes:** `internal/providers/smolvm/client.go:59`
  - body: `encoding/json` only honors one JSON name for this field, so this tag decodes `exitCode` but not the intended `exit_code` fallback. If SmolVM returns a non-zero `exit_code`, Crabbox will leave `ExitCode` at zero and report the failed remote command as successful.
  - confidence: 0.9
- **[P2] Keep remote stderr on stderr:** `internal/providers/smolvm/client.go:255-256`
  - body: This writes remote stderr into the stdout writer, so command diagnostics contaminate stdout and callers cannot distinguish clean output from errors. Pass a stderr writer through the SmolVM exec path and cover stderr-only output in tests.
  - confidence: 0.89
