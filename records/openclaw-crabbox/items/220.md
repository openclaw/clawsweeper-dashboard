## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Honor sync.include before archiving SmolVM workspaces:** `internal/providers/smolvm/sync.go:21`
  - body: SmolVM builds its archive manifest with `syncManifest(req.Repo.Root, excludes)`, and the local wrapper calls `BuildSyncManifest` rather than the filtered helper. Existing delegated providers pass `cfg.Sync.Includes`, so a user who configured `sync.include` to whitelist files would still upload the broader tracked workspace to the hosted SmolVM API.
  - confidence: 0.96
- **[P1] Validate the SmolVM API URL before adding auth:** `internal/providers/smolvm/client.go:151-152`
  - body: `newAPI` accepts `smolvm.baseUrl` as a trimmed string and `doJSON` later adds the bearer token and archive/env payloads to requests built from that value. Please parse the URL, reject userinfo/query/fragment and missing scheme/host, and require https except for loopback http, matching the existing hosted clients.
  - confidence: 0.94
- **[P2] Decode snake_case SmolVM exit codes:** `internal/providers/smolvm/client.go:59`
  - body: The tag `json:"exitCode" json:"exit_code"` does not make Go decode both names; `encoding/json` uses only the first `json` key. If the API returns `exit_code`, failed commands decode as exit 0, so add an explicit fallback field or custom unmarshal plus a regression test.
  - confidence: 0.86
