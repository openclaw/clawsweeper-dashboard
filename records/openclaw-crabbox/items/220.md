Full review comments:

- **[P1] Honor sync.include before building SmolVM archives:** `internal/providers/smolvm/sync.go:21`
  - body: `syncWorkspace` builds the upload manifest with only excludes, so `sync.include` is ignored for SmolVM even though this provider advertises archive sync and uploads the resulting tarball to a hosted API. Please follow the current delegated archive provider pattern by passing `cfg.Sync.Includes` into filtered manifest construction and add regression coverage so whitelisted syncs cannot leak extra files.
  - confidence: 0.96
- **[P1] Route SmolVM stderr to stderr:** `internal/providers/smolvm/client.go:255-257`
  - body: `ExecStream` writes `result.Stderr` into the stdout writer, and `Run` passes `b.rt.Stdout` here, so command diagnostics become normal stdout. That breaks callers that parse stdout or rely on stream separation; please return/write stderr separately, likely by passing a stderr writer through this API, and update the test that currently expects mixed output.
  - confidence: 0.95
