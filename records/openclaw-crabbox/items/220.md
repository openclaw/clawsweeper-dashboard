## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Validate SmolVM base URLs before sending tokens:** `internal/providers/smolvm/client.go:151-152`
  - body: `newAPI` accepts `cfg.Smolvm.BaseURL` from config/env/flags and `doJSON` then sends `Authorization: Bearer ...` to `c.base + path`. A typo or malicious config can receive the `smk_*` token before any scheme/host validation runs; parse the URL and require HTTPS or an explicit loopback test endpoint before constructing the client.
  - confidence: 0.92
- **[P2] Keep remote stderr out of stdout:** `internal/providers/smolvm/client.go:255-257`
  - body: `Run` passes `b.rt.Stdout` into `ExecStream`, and this block appends the API `stderr` field to that same writer. Commands that emit machine-readable stdout plus diagnostics on stderr will produce corrupted stdout for this provider; route remote stderr to `b.rt.Stderr` or return the streams separately.
  - confidence: 0.9
