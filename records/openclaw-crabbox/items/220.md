## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Validate SmolVM API origins before sending bearer auth:** `internal/providers/smolvm/client.go:151-152`
  - body: `cfg.Smolvm.BaseURL` can come from config/env and is stored unchecked here; `doJSON` later sends `Authorization: Bearer ...` plus create/exec/archive payloads to `c.base + path`. A checked-in repo config can redirect a user's `smk_*` token and workspace archive to an arbitrary host before any warning, so constrain the origin or require an explicit unsafe override before attaching auth.
  - confidence: 0.92
- **[P1] Honor sync.include when building SmolVM manifests:** `internal/providers/smolvm/sync.go:21`
  - body: This path calls the SmolVM manifest wrapper without `cfg.Sync.Includes`, and the wrapper uses `BuildSyncManifest` instead of `BuildSyncManifestFiltered`. Users who set `sync.include` to limit uploaded files will still have the full tracked manifest archived and injected into SmolVM, so pass the includes through and add a regression test.
  - confidence: 0.94
- **[P1] Decode snake_case exec exit codes:** `internal/providers/smolvm/client.go:59`
  - body: A struct tag like `json:"exitCode" json:"exit_code"` does not make `encoding/json` read both names; only the first `json` tag is used. If the API returns `exit_code`, nonzero remote exec/archive/write failures decode as zero and Crabbox reports success, so add a separate fallback field or custom unmarshal plus a nonzero snake_case regression.
  - confidence: 0.9
