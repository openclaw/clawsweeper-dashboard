## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Decode snake_case SmolVM exec exits:** `internal/providers/smolvm/client.go:59`
  - body: The duplicate `json` tag here only gives `encoding/json` one effective field name; it will not decode `exit_code` into `ExitCode`. If the SmolVM /exec API returns a nonzero snake_case exit code, Crabbox can treat the failed command, archive injection, or env write as exit 0, so please add a failing regression and decode both shapes explicitly.
  - confidence: 0.91
- **[P1] Route remote stderr away from stdout:** `internal/providers/smolvm/client.go:255-256`
  - body: `backend.Run` passes `b.rt.Stdout` into `ExecStream`, and this block writes remote stderr to that same writer. That mixes diagnostics into command stdout and can corrupt user-visible or machine-readable output; route stderr through the runtime stderr path and cover it with a regression.
  - confidence: 0.94
