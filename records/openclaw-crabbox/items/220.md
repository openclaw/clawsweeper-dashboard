## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Validate the SmolVM base URL before building requests:** `internal/providers/smolvm/client.go:151-152`
  - body: `newAPI` accepts `cfg.Smolvm.BaseURL` after trimming only, then `doJSON` appends paths and sends the bearer token plus archive/env/command payloads to that base. Please mirror the nearby authenticated-provider validation: parse the URL, reject userinfo/query/fragment, and require HTTPS except loopback HTTP.
  - confidence: 0.94
- **[P2] Reject unsupported delegated options inside SmolVM Run:** `internal/providers/smolvm/backend.go:59`
  - body: SmolVM `Run` does not call the delegated option rejection helper at the backend boundary. The CLI path rejects these before dispatch, but the provider backend contract requires delegated backends to reject unsupported capture/download/script/fresh-pr options themselves so direct backend callers and tests cannot silently lose requested outputs.
  - confidence: 0.83
- **[P2] Decode snake_case exec exit codes explicitly:** `internal/providers/smolvm/client.go:59`
  - body: The duplicate struct tag on `ExitCode` only lets `encoding/json` use one JSON name, so an `exit_code` response will leave the exit code at zero while stdout/stderr fallbacks still decode. Add a separate fallback field or custom normalization so failed exec responses cannot be reported as success.
  - confidence: 0.82
