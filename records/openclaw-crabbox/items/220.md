- **[P1] Honor sync.include before archiving SmolVM workspaces:** `internal/providers/smolvm/sync.go:21`
  - body: This builds the SmolVM archive manifest with excludes only, so a repo configured with `sync.include` will still upload the whole working tree to the hosted `/exec` path.   - confidence: 0.94

  - body: This struct tag only registers the first `json` key, so responses using `exit_code` leave `ExitCode` at zero. A failed exec in that shape would be reported as success and skip keep-on-failure handling; add an explicit snake_case field or custom unmarshal with coverage.

  - body: `newAPI` stores the configured base URL unchecked and `doJSON` then attaches the bearer token to the concatenated request URL. Reject malformed, userinfo-bearing, and non-HTTPS non-loopback URLs before authenticated requests are constructed.
  - confidence: 0.84
