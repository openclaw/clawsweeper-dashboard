## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Honor sync.include when building SmolVM archives:** `internal/providers/smolvm/sync.go:21`
  - body: SmolVM builds its archive with the unfiltered manifest helper, so a repo with `sync.include` still sends every non-excluded tracked/untracked file to the hosted provider. Match the other delegated providers by using the filtered manifest path with `cfg.Sync.Includes` and add a regression test for a whitelisted sync.
  - confidence: 0.94
- **[P1] Decode snake_case exec exit codes:** `internal/providers/smolvm/client.go:59`
  - body: The duplicate struct tag does not make `encoding/json` populate `ExitCode` from `exit_code`. If the API returns that shape for a failed command, Crabbox will see zero and report success; add an explicit snake_case field or custom unmarshal coverage.
  - confidence: 0.86
- **[P2] Route remote stderr away from stdout:** `internal/providers/smolvm/client.go:255-256`
  - body: `ExecStream` writes the API `stderr` field into the stdout writer, so commands that emit machine-readable stdout can be corrupted by diagnostics. Preserve stdout/stderr separation in the backend or API wrapper and cover it with a test.
  - confidence: 0.88
