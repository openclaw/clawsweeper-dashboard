## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Decode snake_case exec exit codes:** `internal/providers/smolvm/client.go:59`
  - body: The SmolVM exec result uses a duplicate `json` tag on `ExitCode`, so `encoding/json` will not populate it from a snake_case `exit_code` response. When the hosted API returns `exit_code: 1`, Crabbox can leave `ExitCode` at zero and report the failed remote command as successful.
  - confidence: 0.9
- **[P2] Route SmolVM stderr to stderr:** `internal/providers/smolvm/client.go:255-257`
  - body: `ExecStream` writes `result.Stderr` into the same writer used for stdout, and `Run` passes `b.rt.Stdout` there. This pollutes command stdout and breaks callers that capture or pipe stdout expecting only the remote command's stdout.
  - confidence: 0.88
