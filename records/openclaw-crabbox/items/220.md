Full review comments:

- **[P1] Honor sync.include before building SmolVM archives:** `internal/providers/smolvm/sync.go:21`
  - body: `sync.include` is a documented whitelist, and existing delegated archive providers pass `cfg.Sync.Includes` into `BuildSyncManifestFiltered`. This path builds the SmolVM archive manifest without includes, so a run can still package and upload files outside the user's whitelist to the hosted Smol Machines API.
  - confidence: 0.94
- **[P2] Send SmolVM stderr to stderr:** `internal/providers/smolvm/client.go:255-256`
  - body: `Run` calls `ExecStream` with `b.rt.Stdout`, but this method appends `result.Stderr` to the same writer. That sends remote diagnostics to stdout and can corrupt machine-readable output or shell redirection; extend the seam or run path so stderr goes to `b.rt.Stderr`.
  - confidence: 0.88
