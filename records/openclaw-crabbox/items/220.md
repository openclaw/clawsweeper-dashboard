## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Decode snake_case SmolVM exit codes:** `internal/providers/smolvm/client.go:59`
  - body: The struct tag here cannot make `encoding/json` accept both `exitCode` and `exit_code`; it uses the first JSON name. If the exec API returns `exit_code`, a failing command decodes as exit 0, so `Run` can report success and skip failure handling. Add explicit dual-shape decoding and coverage for nonzero snake_case responses.
  - confidence: 0.89
- **[P1] Validate custom SmolVM base URLs before sending tokens:** `internal/providers/smolvm/client.go:151-152`
  - body: `BaseURL` can come from repo config, env, or flags, and every request later receives the SmolVM bearer token. A checked-in config can redirect `provider: smolvm` to an arbitrary host and receive the user's key and archive payload. Restrict the default path to the official host or require an explicit trusted override with tests.
  - confidence: 0.86
