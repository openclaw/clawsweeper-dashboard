## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.98

Full review comments:

- **[P1] Enforce repository claims for raw machine identifiers:** `internal/providers/smolvm/backend.go:401-406`
  - body: The raw ID/name branches return a remotely named Crabbox machine without applying `claimLeaseForRepoProvider` when `repoRoot` is present. A kept machine claimed by repository A can therefore be reused and synchronized from repository B without `--reclaim`, potentially overwriting A's workspace; apply the same claim check used by the local-claim path before returning.
  - confidence: 0.98
- **[P1] Honor keep-on-failure during setup errors:** `internal/providers/smolvm/backend.go:108-114`
  - body: After acquiring a new machine, sync and workspace-preparation errors return while `shouldStop` remains true, so deferred cleanup deletes the machine despite `--keep-on-failure`. Route every post-acquisition setup failure through `handleDelegatedRunFailure`, including environment-file and command-build failures, and cover retained and default cleanup behavior.
  - confidence: 0.98
- **[P2] Make custom workdirs match the advertised contract:** `internal/providers/smolvm/backend.go:64`
  - body: Every run calls `workspaceFolder`, which rejects absolute paths outside `/workspace`, even though `--smolvm-workdir` is described as a general absolute path and the new configuration test uses `/srv/app`. Either support validated arbitrary absolute workdirs throughout prepare, sync, and exec, or explicitly restrict the flag, config tests, and docs to `/workspace`.
  - confidence: 0.99
- **[P2] Bound archive upload memory consumption:** `internal/providers/smolvm/client.go:281-285`
  - body: `InjectArchive` reads the full archive, creates a second full base64 value, embeds it in a command, and then JSON-marshals the request. Repositories permitted through large-sync preflight can consume several times the compressed size and terminate the CLI; stream the request or enforce a provider-specific maximum before allocating the payload.
  - confidence: 0.94
