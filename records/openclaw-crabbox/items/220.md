Full review comments:

- **[P1] Honor sync.include before building SmolVM archives:** `internal/providers/smolvm/core.go:116-117`
  - body: `sync.include` is a whitelist in current core and existing delegated archive providers pass `cfg.Sync.Includes` into `BuildSyncManifestFiltered`. This wrapper calls `BuildSyncManifest`, so a SmolVM run with an include whitelist still packages and posts the rest of the repo to the hosted Smol Machines API; please thread includes through here and add a focused regression.
  - confidence: 0.94
- **[P2] Send SmolVM stderr to stderr:** `internal/providers/smolvm/client.go:255-256`
  - body: `Run` calls this with `b.rt.Stdout`, so appending `result.Stderr` here emits remote stderr on stdout. That breaks redirection and machine-readable stdout for commands that write diagnostics; since the API is non-streaming, write stderr through `b.rt.Stderr` from the run path or extend the seam to keep the streams separate.
  - confidence: 0.88
