## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Honor sync.include when building SmolVM archives:** `internal/providers/smolvm/sync.go:21`
  - body: This builds the SmolVM archive manifest with only excludes, so a user who configured `sync.include` will still send the whole eligible repo to the SmolVM API. Other delegated providers route through `BuildSyncManifestFiltered` with `cfg.Sync.Includes`; please do the same here and add a regression test that excluded files are not archived.
  - confidence: 0.96
- **[P1] Decode exit_code before reporting SmolVM command success:** `internal/providers/smolvm/client.go:59`
  - body: This struct tag only gives `encoding/json` an effective `exitCode` name; a snake_case `exit_code` response will leave `ExitCode` at zero. That can turn failed remote commands into successful Crabbox runs, so decode both spellings explicitly and cover the snake_case response in tests.
  - confidence: 0.94
- **[P1] Guard custom API origins before sending SmolVM tokens:** `internal/providers/smolvm/client.go:151`
  - body: `cfg.Smolvm.BaseURL` can come from config/env/flags, and the client then sends the `smk_*` bearer token to that origin for every request. Since this PR documents hosted Smol Machines keys, custom origins need to be restricted to an approved safe set or explicitly approved, documented, and tested as a trust boundary before merge.
  - confidence: 0.88
