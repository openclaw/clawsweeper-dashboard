## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Honor sync.include when building SmolVM archives:** `internal/providers/smolvm/sync.go:21`
  - body: This builds the SmolVM tarball from `core.BuildSyncManifest`, which only applies excludes. The normal run path uses the filtered manifest with `sync.include`, so a repo that whitelists a few paths can still upload files outside that whitelist to the hosted SmolVM API.
  - confidence: 0.94
- **[P1] Decode snake_case SmolVM exit codes:** `internal/providers/smolvm/client.go:59`
  - body: A struct field cannot have two active `json` tags; `encoding/json` will use `exitCode` here and ignore `exit_code`. If the API returns `exit_code`, a failed remote command can decode as exit 0 and look successful.
  - confidence: 0.93
- **[P2] Route remote stderr away from stdout:** `internal/providers/smolvm/client.go:255-256`
  - body: `Run` passes only `b.rt.Stdout` into `ExecStream`, and `ExecStream` writes `result.Stderr` to that same writer. That pollutes stdout for commands that emit diagnostics while keeping stdout machine-readable.
  - confidence: 0.9
