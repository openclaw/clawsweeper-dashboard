## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Decode snake_case exec exit codes:** `internal/providers/smolvm/client.go:59`
  - body: The tag `json:"exitCode" json:"exit_code"` does not make `encoding/json` accept both names; it will use the first `json` tag, so a response containing only `exit_code` decodes as zero. That can make a failed remote command look successful, so decode both shapes explicitly and add non-zero coverage.
  - confidence: 0.93
- **[P2] Validate base URLs before sending bearer tokens:** `internal/providers/smolvm/client.go:151-152`
  - body: `smolvm.baseUrl` and `CRABBOX_SMOLVM_BASE_URL` flow into `client.base`, while every request later receives `Authorization: Bearer ...`. Because repo config can select this provider/base URL and the key comes from the user's environment, a checked-out repo could redirect the token to an arbitrary host unless custom hosts are explicitly validated or confirmed.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The diff introduces a hosted bearer-token provider and currently lacks a guard against sending that token to an arbitrary configured base URL.

Concerns:

- **[medium] Guard bearer-token base URLs:** `internal/providers/smolvm/client.go:151`
  - body: The SmolVM client accepts the configured base URL and attaches the bearer token to requests built from it; repo-level config can therefore redirect a user's environment-supplied SmolVM key unless the custom endpoint path is constrained or explicitly trusted.
