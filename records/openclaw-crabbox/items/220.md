## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Honor sync.include when building SmolVM archives:** `internal/providers/smolvm/sync.go:21`
  - body: When `sync.include` is configured, this path calls the SmolVM wrapper that only takes excludes, so the archive is built from the full excluded manifest instead of the configured whitelist. A repo relying on `sync.include` can upload files it intended not to send to the hosted provider; pass `cfg.Sync.Includes` through the filtered manifest path and add a regression test.
  - confidence: 0.93
- **[P1] Decode snake_case SmolVM exec exit codes:** `internal/providers/smolvm/client.go:59`
  - body: This struct tag only gives `encoding/json` the first `json` key, so a response containing `exit_code` leaves `ExitCode` at 0. If the API returns a nonzero snake_case exit code, Crabbox reports the remote command as successful; add explicit decoding for both shapes and cover the nonzero case.
  - confidence: 0.9
- **[P2] Send remote stderr to stderr:** `internal/providers/smolvm/client.go:255-256`
  - body: `ExecStream` appends `result.Stderr` to the stdout writer, and the backend passes `b.rt.Stdout` here. That corrupts stdout for commands whose output is parsed while hiding the stderr stream from callers; route stderr through `b.rt.Stderr` or extend the SmolVM API seam to carry both writers.
  - confidence: 0.88
