## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Validate SmolVM base URLs before sending tokens:** `internal/providers/smolvm/client.go:151-152`
  - body: `newAPI` stores the configured base URL directly, and every request later adds the `Authorization` header. A `http://...` URL or one with unsafe components can receive the bearer token; match the existing Cloudflare/Railway pattern by parsing the URL, requiring HTTPS except loopback, and rejecting userinfo/query/fragment before constructing the client.
  - confidence: 0.96
- **[P1] Decode snake_case exec exit codes:** `internal/providers/smolvm/client.go:59`
  - body: The tag `json:"exitCode" json:"exit_code"` does not make `encoding/json` read both names, so an API response with `exit_code` leaves `ExitCode` at zero. That can make failed execs, archive extraction, or file writes look successful; add an alternate field or custom unmarshal plus a regression test for nonzero `exit_code`.
  - confidence: 0.94
- **[P2] Drop the unrelated global slug vocabulary change:** `internal/cli/slug.go:20`
  - body: Adding a word to `leaseSlugAdjectives` changes the modulus used by `newLeaseSlug`, and the updated golden fixtures show deterministic lease IDs now produce different generated slugs. That global user-visible behavior is unrelated to adding the SmolVM provider and should be removed from this PR or split into its own reviewed change.
  - confidence: 0.87

## Security Review

Status: needs_attention
