## Label Justifications

- `P1`: The issue is a high-impact auth/session revocation bypass for Code portal access, bounded by possession of a valid signed user token.
- `impact:security`: The report concerns bypassing a logout revocation security control for signed browser sessions.
- `impact:auth-provider`: The affected surface is signed `cbxu_` user-token authentication and portal cookie authentication.
- `impact:session-state`: The bypass depends on Code portal revocation state being keyed by raw session-token strings.

## Summary

Keep open: current main still accepts suffixed signed user tokens while Code portal logout revocation hashes the raw token string, so the reported logout-revocation bypass is source-reproducible and not owned by another open fix.

## What This Changes

The issue reports that non-canonical `cbxu_` token spellings can authenticate and bypass Code portal logout revocation keyed by the raw token string.

## Best Possible Solution

Reject non-canonical signed user token strings by requiring exactly two non-empty segments after `cbxu_`, while preserving valid canonical tokens and adding focused auth plus Code portal logout regression tests.

## Reproduction Assessment

Yes. Source tracing on current main shows `verifyUserToken` ignores trailing token segments and the Code portal logout path stores revocation by `sha256Hex` of the raw token string.

## Solution Assessment

Yes. The narrow maintainable fix is to reject non-canonical signed-token shapes at verification time, because Crabbox already mints only `cbxu_<payload>.<signature>` tokens.
