## Summary

This PR should stay open: it is a MEMBER-authored branch and the current diff still has a source-reproducible timing outcome bug where delegated reports can infer a nonzero exit code but classify the error from the original zero-valued `RunResult`.

## What This Changes

The branch adds a shared timing-report helper, routes delegated run timing reports through it, and extends Blacksmith artifact-failure timing JSON coverage.

## Best Possible Solution

Keep the shared helper, but have it finalize outcome fields from the effective timing report exit code while preserving provider-specified status and error kind.

## Reproduction Assessment

Yes for the patch defect from source inspection: `timingReportFromDelegatedRunResult` can infer `report.ExitCode` from `ExitError`, while `TimingReportWithRunResult` finalizes `errorKind` from the unchanged zero-valued `RunResult`. I did not run tests because this review is constrained to read-only inspection.

## Solution Assessment

No: the helper currently finalizes from the original `RunResult` before reconciling the effective report exit code. The narrow maintainable fix is to classify against a temporary result that carries the effective exit code, while preserving provider-owned outcome fields.
