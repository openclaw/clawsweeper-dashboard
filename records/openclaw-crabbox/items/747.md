## Summary

Keep open: current main and v0.33.0 still use the shared providerKey fallback, and this PR is a narrow candidate fix for the linked security issue, but it needs real behavior proof and maintainer sign-off for the intentional direct-API compatibility change before merge.

## What This Changes

The PR removes the Worker leaseConfig shared providerKey fallback, derives an omitted coordinator providerKey from the finalized lease ID before provider preparation, and adds a Worker regression test for direct AWS leases without providerKey.

## Best Possible Solution

Land the narrow omitted-providerKey security fix with redacted real behavior proof and maintainer approval for the compatibility change, while leaving explicit caller-supplied providerKey hardening to https://github.com/openclaw/crabbox/issues/706.

## Reproduction Assessment

## Solution Assessment

Yes for the omitted-key scope: deriving the provider key from the finalized lease ID before provider preparation is narrow and matches the documented CLI invariant. It does not solve the adjacent explicit providerKey override scope tracked at https://github.com/openclaw/crabbox/issues/706.
