## What This Changes

The PR removes the Worker static providerKey fallback, derives omitted direct lease provider keys from the finalized lease ID before provider preparation, and adds Worker regression coverage.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Yes for the omitted-key scope: deriving the provider key from the finalized lease ID before provider preparation is narrow and matches the documented CLI invariant. It does not solve the adjacent explicit providerKey override scope tracked at https://github.com/openclaw/crabbox/issues/706.
