## Summary

Keep open: the PR targets the right security issue, but the manage-only event guard also blocks the existing CLI path that attaches a use-shared lease after the run record is created, so it is not merge-ready.

## What This Changes

The branch changes Worker run-event lease attribution from a visibility check to a manage/admin check and adds a regression test for use-share audit re-tag attempts.

## Best Possible Solution

Land a narrower security fix that blocks unrelated post-hoc event re-tagging while preserving validated use-share runs and documented replacement-flow audit attribution.

## Reproduction Assessment

Yes. Source inspection shows `crabbox run` creates a run with an empty lease ID and later posts `lease.created`; the branch's manage-only guard rejects that event for use-share leases.

## Solution Assessment

No. The one-line guard blocks the reported attack but also blocks a documented shared-lease audit path; the fix needs validated backing-lease context rather than a blanket manage-only rule.
