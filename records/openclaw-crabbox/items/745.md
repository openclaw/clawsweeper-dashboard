## Summary

## What This Changes

The PR changes Worker run event lease re-attribution to require manage/admin access, defers explicit `crabbox run --id ...` run-history creation until the lease is resolved, and adds focused Worker and Go regression tests.

## Best Possible Solution

Land the security fix after maintainers accept the fail-closed attribution contract and the contributor adds redacted real runtime proof for both the allowed explicit/use-share path and the rejected unrelated re-tag path.

## Reproduction Assessment

Yes. Source inspection of current main shows a run owner can post an event with a different visible use-shared leaseID, after which event summary and attribution code can add that lease owner to the run audit surfaces; I did not run the Worker harness in this read-only review.

## Solution Assessment

Mostly yes. The patch takes the narrow security path by requiring manage/admin access for changed event leaseID values and preserving explicit-lease runs by creating history with the resolved leaseID, but maintainers still need to accept the compatibility contract and require real runtime proof.
