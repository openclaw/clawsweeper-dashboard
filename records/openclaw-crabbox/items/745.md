The branch tightens Worker run-event lease attribution changes to require manage/admin access, defers explicit `crabbox run --id ...` run-history creation until lease resolution, and adds focused Worker and CLI regression tests.

## Best Possible Solution

Land the security fix after maintainers accept the fail-closed attribution contract and the contributor adds redacted real runtime proof for both the allowed explicit/use-share path and the rejected unrelated re-tag path.

## Reproduction Assessment

## Solution Assessment

Mostly yes. The patch takes the narrow security path by requiring manage/admin access for changed event leaseID values and preserving explicit-lease runs by creating history with the resolved leaseID, but maintainers still need to accept the compatibility contract and require real runtime proof.
