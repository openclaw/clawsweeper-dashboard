## Summary

This PR addresses a real CLI gap, but the static auto-route implementation is not merge-ready because it derives the SSH host from a normalized lease slug, which breaks common static hosts such as DNS names and IP addresses.

## What This Changes

The branch adds `--id` support to `crabbox stop`, changes run stop hints to emit `--id`, and routes `static_` lease IDs through the SSH static provider across lease commands.

## Best Possible Solution

Keep the `stop --id` improvement, but make static auto-routing use a real host mapping or explicitly limit and test the feature so generated static lease IDs never imply an unrecoverable host.

## Reproduction Assessment

Yes. Source inspection shows current main rejects `crabbox stop --id`, and the PR-introduced static-route bug is reproducible from `staticLease` generating IDs from `normalizeLeaseSlug(cfg.Static.Host)` while the patch assigns that slug back to `cfg.Static.Host`.

## Solution Assessment

No. Adding `stop --id` is the right direction, but deriving a static SSH host from the normalized lease ID suffix is not a reliable fix for generated static leases.
