## Summary

This is substantive new provider work that is not present on current main, but it is not ready to merge because released installations would advertise Apple VZ without distributing its required helper, and the PR lacks real Apple Silicon runtime proof.

## What This Changes

The PR adds an experimental Apple Silicon Virtualization.framework provider, a separate helper binary, provider configuration and documentation, lifecycle hardening, and unit and smoke-script coverage.

## Best Possible Solution

Package and install a matching helper for supported Darwin arm64 release paths, define an intentional trusted-image policy, and provide redacted end-to-end Apple Silicon evidence covering doctor, first boot, command execution, timeout cleanup, status, and stop.

## Reproduction Assessment

Yes for the distribution defect: the proposed release configuration packages only `crabbox`, while the registered provider requires `crabbox-apple-vz-helper`. The runtime hardening claims cannot be reproduced with high confidence from the supplied evidence because no real Apple Silicon run is shown.

## Solution Assessment

No, not yet. A provider adapter and platform helper are reasonable boundaries, but an officially usable provider must include an intentional helper distribution path, trusted-image policy, and real-host lifecycle validation.
