## Summary

The PR has strong Apple Silicon live proof and follows Crabbox’s provider-adapter model, but two concrete availability defects block merge: a failed PID-metadata write can orphan a newly spawned helper, and fixed checksums attached to Ubuntu’s mutable `release/` image URLs will eventually break fresh leases after upstream image refreshes. Helper distribution remains an explicit maintainer product decision because release installs advertise the provider without shipping its required helper.

## What This Changes

Adds an experimental built-in Apple Virtualization.framework provider, native helper, configuration, documentation, dependencies, and tests for local Linux SSH leases on Apple Silicon macOS.

## Best Possible Solution

Make daemon startup transactional, pin default images to immutable dated artifacts or securely refresh their checksums, and require an explicit maintainer decision on shipping a built-in provider whose helper is absent from release artifacts.

## Reproduction Assessment

Yes for the review findings: force the PID metadata write after daemon spawn to fail to reproduce the orphan path, and replace an upstream `release/` image while retaining the pinned digest to reproduce fresh-provision checksum failure. The feature itself also has convincing real Apple Silicon success proof.

## Solution Assessment

No in its current form; the provider-owned adapter is appropriate, but transactional startup and immutable image inputs are required, while helper distribution needs explicit maintainer policy rather than an incidental runtime error.

- **[P1] Terminate the helper when PID metadata cannot be committed:** `internal/applevzhelper/cli_darwin_arm64.go:213-214`
  - body: If this metadata write fails after `cmd.Start`, the function returns while the new `serve` process keeps running and the partially prepared VM state remains behind. Treat this as a post-spawn rollback path: terminate and wait for the exact child and remove the instance directory, with a regression test that injects the write failure.
  - confidence: 0.98
- **[P1] Pin default images to immutable artifacts:** `internal/cli/os_image.go:39-40`
  - body: These `release/` URLs are moving aliases, but the code hard-codes their current SHA-256 values. Ubuntu periodically replaces those files, so after the next refresh every uncached default Apple VZ start will fail checksum verification until Crabbox ships an update. Use dated immutable release URLs, or fetch and authenticate matching upstream checksum metadata without weakening verification.
  - confidence: 0.97
