## What This Changes

The branch rejects ASCII control characters in portal OAuth `returnTo` targets and adds Worker callback-flow tests for CRLF, CR, LF, NUL, and DEL fallback to `/portal`.

## Best Possible Solution

Land the narrow OAuth hardening after redacted real broker/OAuth proof shows tainted `returnTo` values fall back to `/portal` while still issuing the portal session cookie.

## Reproduction Assessment

Yes, at source level. Current main decodes `returnTo`, stores it in pending OAuth state, and later emits it as a `Location` header; a read-only Node sanity check shows the decoded CRLF value is rejected by `Response` headers.

## Solution Assessment

Yes. Rejecting decoded ASCII control characters in `safePortalReturnTo` before persisting OAuth state is the narrow maintainable fix, and the added callback-flow regression test covers the intended fallback behavior.
