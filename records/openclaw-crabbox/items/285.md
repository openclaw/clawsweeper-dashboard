## Summary

This feature is not implemented or superseded, and it fits Crabbox’s established delegated-run provider architecture; however, accepting a new first-party provider and narrowing its authentication, API-version, synchronization, and destructive-lifecycle contract requires maintainer judgment before implementation.

## What This Changes

Add a first-party Blaxel delegated-run provider supporting sandbox lifecycle, archive synchronization, remote execution, retained leases, cleanup, diagnostics, and secret-safe configuration.

## Best Possible Solution

Approve a bounded Phase 1 provider-local adapter that reuses Crabbox’s generic delegated-run, archive-sync, lease, and normalized output contracts; pin a tested Blaxel API version and require matching local claims plus remote ownership labels for every destructive action.

## Reproduction Assessment

Not applicable: this issue requests a new provider and does not report broken behavior on current main.

## Solution Assessment

Yes at the architectural level: a provider-local delegated-run adapter is the established and narrowest maintainable direction. Maintainers still need to approve the first-party provider decision and reduce the initial API, authentication, synchronization, and lifecycle scope to a reviewable contract.
