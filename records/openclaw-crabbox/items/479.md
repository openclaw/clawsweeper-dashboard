## Summary

Keep open: current main still creates a kept prewarm lease, returns directly on Actions hydration or probe failure, and only releases on ready-pool registration failure.

## What This Changes

The issue requests cleanup or actionable stop guidance when `crabbox prewarm` fails after creating a kept lease during Actions hydration or probe execution.

## Best Possible Solution

Release the newly created SSH lease by default on post-warmup hydrate or probe failure, preserving ready-pool gating and adding focused regression coverage.

## Reproduction Assessment

Yes, by source inspection: an SSH-backed `prewarm` with configured Actions hydration or a probe command parses a lease id, then returns directly on hydrate/probe error without release. I did not run a live paid provider reproduction during this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to generalize the existing ready-pool failure release helper for all post-warmup prewarm failures and cover hydrate, timeout-like, and probe errors with focused tests.
