## Summary

Close as duplicate: the underlying security issue is still valid on current main, but the same artifact broker org-omission problem is already tracked by the older open canonical issue at https://github.com/openclaw/crabbox/issues/702.

## What This Changes

The issue reports that brokered artifact upload object keys include the authenticated owner and caller prefix but omit the authenticated organization, allowing same-owner cross-org namespace collisions.

## Best Possible Solution

Consolidate discussion and remediation in https://github.com/openclaw/crabbox/issues/702, including an org-aware key namespace and a deliberate compatibility plan for existing owner-only artifact URLs.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to request brokered uploads with the same owner, caller prefix, and file name but different org headers; current main passes only owner into the object key builder.

## Solution Assessment

Yes for the technical direction, but not as a separate tracking thread. The narrow fix should add a normalized org segment to newly signed brokered artifact keys while the canonical issue owns the compatibility decision.
