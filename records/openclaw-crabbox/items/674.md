## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Pin the Lambda MicroVM runner base images:** `runtimes/aws-lambda-microvm/Dockerfile:1-6`
  - body: This new remote execution image is built from mutable `golang:1.26-alpine` and `ubuntu:24.04` tags, and the repository's digest checker does not include this Dockerfile. Pin both base images by SHA-256 digest and add this runtime Dockerfile to the checker so the reviewed runner cannot drift after merge.
  - confidence: 0.93
- **[P2] Reject broad Lambda MicroVM workdirs:** `internal/providers/awslambdamicrovm/flags.go:91-93`
  - body: `validateConfig` currently rejects only empty, relative, unclean paths and `/`, so broad paths like `/tmp`, `/usr`, `/etc`, and `/workspace` still pass. Delegated archive sync replaces the workdir tree, so require a dedicated safe subdirectory like the other delegated providers do.
  - confidence: 0.9
- **[P2] Bound each runner health probe:** `internal/providers/awslambdamicrovm/backend.go:405`
  - body: When the MicroVM reaches `RUNNING`, `waitReady` calls `runner.Health(ctx, vm)` with the top-level CLI context. If the endpoint accepts and stalls, the five-minute readiness deadline is not rechecked and rollback can hang; wrap each health probe in a short per-request deadline or use a bounded HTTP client.
  - confidence: 0.86
- **[P2] Serialize retained MicroVM operations:** `internal/providers/awslambdamicrovm/backend.go:86-98`
  - body: The reuse path resolves and touches the claim before archive sync without a per-lease operation lock, so two concurrent `run --id` calls can interleave upload, extract, replace, and command execution against the same retained MicroVM. Add the same local lease-operation lock pattern used by reusable delegated providers around reuse and lifecycle operations.
  - confidence: 0.84
- **[P2] Reject Tailscale options for MicroVM runs:** `internal/providers/awslambdamicrovm/provider.go:32-37`
  - body: This delegated HTTP runner never joins the tailnet, but `Configure`, `Warmup`, and `Run` do not reject Tailscale settings. Shared config can therefore create a successful lease labeled as Tailscale-requested while it remains reachable only through the Lambda MicroVM endpoint.
  - confidence: 0.82
