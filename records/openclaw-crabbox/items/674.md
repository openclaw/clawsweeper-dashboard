## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Reap the shell before waiting on inherited pipes:** `runtimes/aws-lambda-microvm/main.go:179-180`
  - body: The exec handler waits for stdout/stderr readers before calling `cmd.Wait()`. If the shell backgrounds a child that inherits those pipes, the direct shell can exit but the readers never see EOF until that child exits, so `crabbox run` can hang after the requested command is done.
  - confidence: 0.87
- **[P2] Preserve raw stdout and stderr bytes:** `runtimes/aws-lambda-microvm/main.go:245`
  - body: This converts command output chunks to a JSON string, and Go JSON strings cannot preserve invalid UTF-8 bytes. Commands that emit binary or non-UTF-8 output will be corrupted before the client writes it back.
  - confidence: 0.9
- **[P3] Warn when warmup keeps despite --keep=false:** `internal/providers/awslambdamicrovm/backend.go:54`
  - body: The provider documents `warmup` as always retained, matching other delegated providers, but this path silently creates a retained MicroVM even when the user passes `--keep=false`. Add the standard warning so users do not leave billable capacity retained without an explicit notice.
  - confidence: 0.72
