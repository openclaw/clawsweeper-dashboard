The issue reports that local WSL gateway setup fails when localized `wsl --version` output uses Chinese labels instead of `WSL version:`.

## Best Possible Solution

Update `WslInstallSupport.TryParseWslVersion` to parse WSL's version number in a locale-tolerant way, add localized regression fixtures, and preserve the existing minimum-version gate.
