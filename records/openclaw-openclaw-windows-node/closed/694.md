## Summary

Current main and the v0.6.3 release already handle the reported Swedish/localized `wsl --version` output by matching the stable WSL product token and semantic version rather than English label text, with regression coverage for `sv-SE` and multiline package-version output.

## What This Changes

The issue reports installer failure during WSL preflight when localized `wsl --version` output such as Swedish `WSL-version:` is not parsed despite WSL working.

## Best Possible Solution

Keep the v0.6.3/current parser behavior and ask for a fresh setup log only if the latest installer still fails on this localized output.

## Reproduction Assessment

Yes, at source level: feed `WSL-version: 2.7.3.0` or the reported multiline localized output into `WslInstallSupport.TryParseWslVersion`. I did not run a Windows installer reproduction from this Linux read-only review checkout.

## Solution Assessment

Yes. Matching the stable WSL product token plus semantic version is the narrow maintainable fix because it avoids hard-coding localized label strings while preserving the existing preflight gate.
