Full review comments:

- **[P2] Constrain the node process exemption:** `src/OpenClaw.SetupEngine/SetupSteps.cs:970`
  - body: This now treats any listener whose `ss` output contains `node` as if it is the OpenClaw gateway. If another Node.js app is already bound to the gateway port, setup will proceed past preflight and then fail or wait in `openclaw gateway start` instead of reporting the real conflict. Please match the command line/path to the OpenClaw gateway process, not the generic `node` process name.
  - confidence: 0.88
- **[P3] Return promptly on terminal auth errors:** `src/OpenClaw.SetupEngine/SetupSteps.cs:1509`
  - body: The setup wait now ignores every `ConnectionStatus.Error`. `OpenClawGatewayClient` raises `Error` for terminal auth failures and stops auto-reconnect once `_authFailed` is set, so invalid tokens or rejected device signatures will sit until the full pairing timeout. Keep the transient-disconnect retry behavior, but resolve terminal/auth errors immediately, for operator and node waits.
  - confidence: 0.82
