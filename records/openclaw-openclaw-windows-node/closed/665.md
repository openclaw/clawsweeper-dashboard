## Summary

Current master already fixes the reported chat Markdown list-wrapping failure with a focused renderer change and regression coverage; the report matches v0.6.0, where the fix had not shipped yet.

## What This Changes

Long Markdown bullet list items in chat should wrap inside the bubble instead of rendering or clipping on a single line.

## Best Possible Solution

Keep the current master renderer fix and ship it in the next release; no additional core change is needed for this report.

## Reproduction Assessment

Yes for the affected v0.6.0 build: the release source used a horizontal HStack for list rows, which matches the reported single-line clipping. Current master source shows the failing layout was replaced by a Grid with regression coverage.

## Solution Assessment

Yes. The current fix is the narrow maintainable solution: keep Markdown block rendering intact and change only list row layout so content receives finite measure width.
