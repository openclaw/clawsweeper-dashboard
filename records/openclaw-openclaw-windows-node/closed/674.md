## Summary

Close as implemented: the failure signature matches the old 30s configure-gateway timeout, and current main extends that step to 120s with regression coverage and clearer timeout diagnostics.

## What This Changes

Fresh local WSL gateway installation fails during the configure-gateway step when `wsl.exe` times out after about 30 seconds and reports an empty exit -1 error.

## Best Possible Solution

Keep the 120s configure-gateway timeout and specific timeout message, and direct affected users to a companion build containing 3ddade7072b068b72afd228bbebd8fdd54f7ba23 or later.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No current-main failing path was reproduced; the reporter's log matches the v0.6.0 30s configure-gateway timeout exactly, and current main replaces that path with a 120s timeout covered by tests.

## Solution Assessment

Is this the best way to solve the issue? Yes; extending the WSL config step timeout and improving timeout diagnostics directly addresses the observed slow-WSL failure without changing credential handling or the setup flow.
