## Summary

## What This Changes

The issue requests that the onboarding wizard reconnect cleanly and avoid dead-ends when the gateway restarts during setup.

## Best Possible Solution

## Solution Assessment

Yes. The current implementation is the maintainable fix for the no-dead-end requirement: it reconnects the setup runner and exposes retry/restart/skip recovery in the UI while acknowledging that gateway restart clears exact wizard session position.
