## What This Changes

The issue reports that the setup `preflight-wsl` step fails to parse localized or UTF-16/NUL-interleaved `wsl.exe --version` output in released Windows installers.

## Best Possible Solution

Keep the shipped v0.6.3 parser behavior and close this release-verification tracking issue as implemented.

## Solution Assessment

Yes. Parsing a stable `WSL` product-token line plus a semantic version is the narrowest maintainable fix for localized human-readable labels while preserving the setup preflight and version-threshold behavior.
