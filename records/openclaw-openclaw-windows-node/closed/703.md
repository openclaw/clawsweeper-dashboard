## Summary

Keep open; current main still appears to ship the same x64 app-local VC runtime path while ONNX/Sherpa remains in the tray and the native dependency verifier does not cover the reported load failure.

## What This Changes

The issue reports a tray startup crash loop caused by older app-local VC runtime DLLs shadowing the newer runtime needed by ONNX/Sherpa native libraries.

## Best Possible Solution

Update packaging so app-local VC runtime DLLs are new enough for the bundled ONNX/Sherpa stack, then extend native dependency validation to catch Sherpa/ONNX loading or minimum runtime-version failures.

## Reproduction Assessment

Yes for a source-level path: publish or run the Windows tray with the current win-x64 app-local VC runtime package and exercise the Piper/Sherpa TTS path. I did not run the live Windows crash in this read-only review.

## Solution Assessment

Yes; the narrow maintainable fix is release packaging plus native dependency validation, not a TTS workaround that leaves stale app-local runtime DLLs shadowing newer system runtimes.
