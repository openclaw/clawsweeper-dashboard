The branch enables the setup device-pair plugin when reachable, adds tray-side wizard console-log tailing, unwraps nested wizard note messages, auto-launches auth-looking URLs, and adds helper/tail tests.

## Best Possible Solution

Land a narrowed version that surfaces the missing wizard auth content, enables device-pair only under tested setup conditions, and auto-opens only recognized auth/device-code URLs with real onboarding proof attached.

## Solution Assessment

No: the overall direction is maintainable, but the current auto-launch predicate is broader than the stated OAuth/device-code scope. The safer fix is to constrain launch eligibility to recognized auth hosts and auth-specific paths, then show real onboarding proof.
