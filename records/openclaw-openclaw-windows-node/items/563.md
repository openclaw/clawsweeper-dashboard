## What This Changes

Show a user-visible diagnostic when a manual AppInstaller update check cannot resolve, fetch, parse, or use its feed.

## Best Possible Solution

Apply the existing localized failed-check dialog pattern inside the final approved AppInstaller update service, logging diagnostic detail for missing feeds, HTTP failures, invalid XML, and unavailable registration while keeping normal user copy concise.

## Reproduction Assessment

No. Current main has no AppInstaller update-check path, so the reported feed failure cannot be reproduced there; the relevant implementation exists only in the proposed distribution PR.

## Solution Assessment

Yes, conditionally. Reusing the current Updatum failure-dialog pattern is the narrowest maintainable UX, but it should be added only within the AppInstaller architecture maintainers decide to land.
