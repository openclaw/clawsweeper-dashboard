## Best Possible Solution

Keep the typed `sessions.files` rail, but add an older-gateway `agents.files` fallback or an explicit, tested upgrade gate before merge; let https://github.com/openclaw/openclaw-windows-node/issues/871 continue tracking artifacts and polish.

## Reproduction Assessment

## Solution Assessment

No as currently proposed: replacing the legacy transport without fallback is not the safest upgrade path. The maintainable solution is typed `sessions.files` plus a preserved fallback or explicit maintainer-approved upgrade proof.
