## Summary

Keep open: the feature is useful and the screenshot proof is sufficient, but the patch still replaces the existing Workspace `agents.files` path with `sessions.files` only, which can make file viewing fail for older gateways after app upgrade.

## What This Changes

Adds a WinUI Workspace session file rail backed by typed `sessions.files.list/get` APIs, with browsing, search, preview UI, localization strings, a projection model, and tray tests.

## Best Possible Solution

Keep the session-scoped rail, but preserve the older-gateway `agents.files` fallback or land an explicit maintainer-approved minimum-gateway upgrade gate; leave artifact rows and polish to https://github.com/openclaw/openclaw-windows-node/issues/871.

## Reproduction Assessment

Yes for the review finding: source inspection shows current main still uses `agents.files.list/get`, while this PR loads through `sessions.files.list/get` and shows unsupported when that typed method is unavailable. I did not run a live older-gateway setup in this read-only review.

## Solution Assessment

No as currently proposed: replacing the legacy transport without fallback is not the safest upgrade path. The maintainable solution is typed `sessions.files` plus a preserved fallback or explicit maintainer-approved upgrade proof.
