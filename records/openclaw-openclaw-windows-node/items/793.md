## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Store browser-control credentials per gateway:** `src/OpenClaw.Shared/SettingsData.cs:61`
  - body: This adds `BrowserControlToken` to global `SettingsData`, but browser-control auth is a gateway credential. Because `NodeService` applies this single setting to whichever gateway is active, switching gateways can send one gateway's token to another forwarded control host and bypasses the registry-backed credential boundary. Please store the override on the active gateway/credential path or keep using `SharedGatewayToken`.
  - confidence: 0.9
- **[P2] Gate browser.proxy on the effective override token:** `src/OpenClaw.Tray.WinUI/Services/NodeService.cs:390`
  - body: The new token is only passed after `ShouldRegisterBrowserProxy` succeeds, and that predicate still requires `sharedGatewayToken`. A QR/bootstrap/device-token-only setup with `BrowserControlToken` configured will never advertise `browser.proxy`, so the auth-token override cannot fix that case. Gate on the effective browser-control token and add a NodeService/gating regression test.
  - confidence: 0.87
