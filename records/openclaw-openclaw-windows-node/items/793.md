## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Store browser-control overrides per gateway:** `src/OpenClaw.Shared/SettingsData.cs:61`
  - body: `BrowserControlToken` is persisted in global `SettingsData`, but browser-control auth is tied to the active gateway. Switching gateways can send one gateway's secret to another forwarded control host and bypasses the registry-backed credential boundary; move the override to the `GatewayRecord`/credential path or keep using `SharedGatewayToken`.
  - confidence: 0.9
- **[P2] Gate browser.proxy on the effective override token:** `src/OpenClaw.Tray.WinUI/Services/NodeService.cs:390`
  - body: The new token is only passed after `ShouldRegisterBrowserProxy` succeeds, and that predicate still requires `sharedGatewayToken`. A QR/bootstrap/device-token-only setup with `BrowserControlToken` configured will never advertise `browser.proxy`, so the override cannot fix that case; gate on the effective browser-control token and add a regression test.
  - confidence: 0.87
