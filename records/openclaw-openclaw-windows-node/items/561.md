## Summary

This PR is useful but not merge-ready: current main still uses the old thinking-row rendering, while the PR moves the UI in the intended direction but still has a source-confirmed failed-tool visibility bug and only partial screenshot proof.

## What This Changes

This PR updates the WinUI chat root/timeline so the assistant thinking placeholder renders as a chat bubble, tool progress can nest inside assistant/thinking bubbles, and Chat Exploration debug visibility/usage controls are isolated from production chat.

## Best Possible Solution

Keep the visual grouping direction, but gate thinking-bubble tool nesting with the same error-prominence rule used for assistant bubbles and add real WinUI proof for the changed paths.

## Reproduction Assessment

Yes for the review finding by source inspection: a pre-assistant error ToolCall in an active turn enters the thinkingNestedTool path without BurstIsNestable, so it is nested and consumed instead of rendered externally. I did not run WinUI because this is a read-only review.

## Solution Assessment

No, not yet: the UX direction is maintainable, but the thinking-bubble tool path should reuse the existing BurstIsNestable/error-prominence rule before merge.
