## Summary

Keep open: this member-authored release PR contains useful runtime-packaging work, but the installer change should not merge until the per-user installer versus machine-wide VC redist prerequisite path is resolved and the ARM64 portable ZIP removal is explicitly accepted.

## What This Changes

The PR adds x64 app-local VC runtime packaging, native dependency release verification, bundled VC redistributables for Inno installers, pauses the ARM64 portable ZIP, and improves startup/native dependency logging.

## Best Possible Solution

Keep the runtime dependency fix, but make clean-host install reliable by using app-local runtime files where possible or an explicit elevated prerequisite/admin handoff with x64 and ARM64 proof, and treat ARM64 portable ZIP removal as a deliberate release-policy decision.

## Solution Assessment

Full review comments:

- **[P1] Handle VC redist elevation before gating launch:** `installer.iss:119-125`
  - body: This installer still declares `PrivilegesRequired=lowest`, so Inno will not request elevation for the setup process; Microsoft documents the VC redist as a target-system prerequisite and its troubleshooting guidance says to run the redist installer as administrator. Here the per-user installer starts `vc_redist.exe /install /quiet /norestart` via `Exec`; when that fails to start or exits unsuccessfully, `VCRuntimeInstallSucceeded` remains false and the new `[Run]` check skips launching the tray. On clean/stale hosts, especially ARM64 where no app-local VCRT is shipped, the installer can complete without repairing the missing runtime this PR targets. Please add an app-local/elevated prerequisite path or prove the non-admin clean install path before merge. ([documentation.help](https://documentation.help/Inno-Setup/topic_setup_privilegesrequired.htm?utm_source=openai))
