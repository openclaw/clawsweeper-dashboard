## Summary

Keep open: this is a useful fresh draft feature PR with visible screenshots, but the current footer summary precedence can mask newly captured per-assistant usage with stale thread totals until sessions refresh.

## What This Changes

The PR adds assistant-footer usage display, shared usage formatting for chat and tray session cards, monotonic usage snapshots, rounded tray progress fill corners, and removes the Diagnostics chat explorations launcher.

## Best Possible Solution

Prefer the latest assistant entry usage metadata when it exists, fall back to thread session totals only when no entry snapshot is available, and keep the shared formatter plus focused regression coverage.

## Reproduction Assessment

Source-reproducible: the PR updates assistant entry metadata separately, while OpenClawChatRoot formats thread-level session totals first. I did not run the app or tests because this review must keep the checkout read-only.

## Solution Assessment

No: the feature direction is reasonable, but the implementation should prefer fresh latest-entry usage before falling back to session totals. That keeps the footer tied to the assistant message it annotates.
