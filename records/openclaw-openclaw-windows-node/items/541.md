## Summary

## What This Changes

The PR revamps the WinUI Config page into a schema-guided editor with tree navigation, detail editing, search, JSON diff preview, permission-aware states, save/reconnect handling, localization updates, and focused tray tests.

## Best Possible Solution

Fix the stale-base retry so pending edits are reapplied to the latest gateway config after refresh, then add real proof for save/reconnect and permission-state behavior before merge.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: save with pending edits, receive a stale baseHash response, let the page refresh config, then save again. Because _editSnapshot remains set while pending changes exist, the second save still uses the stale base.

## Solution Assessment

No. The PR direction is maintainable, but the current stale-conflict path should rebase pending edits onto the refreshed server snapshot instead of continuing to prefer the old edit snapshot.
