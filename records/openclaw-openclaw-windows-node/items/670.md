## Summary

Keep this open: current `master` and the provided release code still parse only the English `WSL version:` label, so the reported Spanish `Versión de WSL:` output remains a valid setup-blocking bug.

## What This Changes

The issue reports that onboarding WSL preflight rejects localized Spanish `wsl --version` output even though WSL is installed.

## Best Possible Solution

Teach the WSL preflight parser to extract the version from localized `wsl --version` output and add focused regression coverage using the reported Spanish sample.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: the reported Spanish output does not contain the English `WSL version:` label the current regex requires, and `PreflightWslStep` treats that parse failure as terminal. I did not run the Windows setup engine in this Linux review environment.

## Solution Assessment

Yes, a narrow parser fix is the best path: preserve the current English path, support localized labels or label-independent WSL version extraction, and add a regression test with the Spanish sample.
