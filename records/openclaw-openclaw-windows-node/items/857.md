## Summary

Keep open: current main has meaningful sanitizer and diagnostics-bundle work, but it does not prove the full issue goal that all collected diagnostics are PII-scrubbed before leaving the device; the parent release-confidence epic and adjacent trace/upload issues are still open.

## What This Changes

The issue requests that collected diagnostic output be scrubbed for PII before it leaves the device.

## Best Possible Solution

Define one conservative diagnostics-export scrub boundary, run every copied, saved, bundled, or uploaded diagnostics artifact through it, and add regression coverage for fallback support context plus representative URLs, paths, IPs, emails, process names, node/channel labels, activity text, and token-like fields.

Unclear as written. The maintainable solution is not a one-off redaction at one button, but a shared diagnostics-export scrubber or equivalent policy-backed boundary consumed by trace collection and one-click upload.
