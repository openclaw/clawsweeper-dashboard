## Summary

Keep open: current main pins the gateway version for first-run install and has WSL keepalive/control primitives, but it does not compare the installed WSL CLI version with the Tray-required LKG or offer an in-place alignment upgrade.

## What This Changes

Request a Tray-managed local WSL gateway alignment flow that detects installed OpenClaw CLI version drift from the Tray's pinned LKG, prompts the user, and performs a confirmed in-place upgrade/restart.

## Best Possible Solution

Add a Tray-owned local WSL gateway alignment service that runs after the setup-managed distro is ready, reads the installed CLI version from WSL, compares it to GatewayLkgVersion.LkgVersion, surfaces mismatch state in Settings/toast, and runs the pinned install plus restart/recheck only after user confirmation.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: install a local gateway with one pinned LKG, update to a Tray build with a different GatewayLkgVersion, and observe that startup keepalive starts the distro without comparing or upgrading the installed CLI version.

## Solution Assessment

Unclear until maintainers approve the product direction. If lock-step is accepted, the requested path is the maintainable implementation shape because it reuses the existing LKG pin, WSL command runner, install command, and WslGatewayController restart instead of re-running onboarding.
