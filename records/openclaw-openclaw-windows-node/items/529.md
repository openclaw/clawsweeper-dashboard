## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Stop stale keepalives when local setup is inactive:** `src/OpenClaw.Tray.WinUI/App.xaml.cs:1782-1783`
  - body: This branch used to stop the managed WSL keepalive when the active gateway was no longer local; now it returns immediately. A user who switches to a remote gateway or disables local setup can leave the OpenClawGateway sleep process alive indefinitely, so restore the managed stop path or an equivalent cleanup before returning.
  - confidence: 0.91
- **[P1] Kill the keepalive process tree during rollback:** `src/OpenClaw.SetupEngine/SetupSteps.cs:2355`
  - body: The keepalive is launched as `wsl.exe -d <distro> -- sleep infinity`, which can leave descendant WSL processes holding the distro alive. Killing only the parent can let uninstall continue with a live child and block unregister/cleanup; use `Kill(entireProcessTree: true)` and wait best-effort.
  - confidence: 0.9
- **[P1] Overwrite setup-owned tray settings after setup:** `src/OpenClaw.SetupEngine/SetupContext.cs:212-213`
  - body: Setup finishes by merging defaults with `TryAdd`, so an existing `EnableNodeMode=false` or disabled capability remains false even after the user completes local gateway setup. These keys are owned by the setup result and the old flow explicitly enabled node mode, so overwrite only the setup-owned fields while preserving unrelated settings.
  - confidence: 0.93
