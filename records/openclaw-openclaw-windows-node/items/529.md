## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Filter legacy credentials before rewriting settings:** `src/OpenClaw.SetupEngine/SetupContext.cs:190-194`
  - body: `MergeIntoSettingsFile` preserves every unknown key from the existing settings file, which includes legacy `Token` and `BootstrapToken` fields on upgrades. Current `SettingsManager` intentionally reads those only for migration and never writes them back, so SetupEngine can re-persist stale credentials outside `GatewayRegistry` and revive fallback auth state; filter those keys or write through the same sanitized settings model.
  - confidence: 0.88
- **[P2] Point startup registration at the parent tray exe:** `src/OpenClaw.SetupEngine.UI/Pages/CompletePage.xaml.cs:105-110`
  - body: In the release layout the workflow publishes `OpenClaw.Tray.WinUI.exe` in `publish\` and copies SetupEngine.UI under `publish\SetupEngine`, but these candidates look inside `SetupEngine\` or a nonexistent `OpenClaw.Tray.WinUI\` sibling folder. The Finish-page startup toggle will silently do nothing in installed builds unless this also checks the parent directory tray exe or receives the tray path from the launcher.
  - confidence: 0.9
