## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve the managed keepalive lifecycle:** `src/OpenClaw.Tray.WinUI/App.xaml.cs:1809`
  - body: This now starts a raw `wsl.exe -d <distro> -- sleep infinity` every tray launch without the existing marker/adoption logic, and it no longer stops the tracked keepalive when the active gateway is no longer local. Current main uses `WslDistroKeepAlive.EnsureStarted/Stop` specifically to avoid untracked orphan sleeps that future launches and uninstall cannot reliably adopt or clean up.
  - confidence: 0.96
- **[P1] Kill the full WSL keepalive process tree:** `src/OpenClaw.SetupEngine/SetupSteps.cs:2355`
  - body: The new rollback path kills only the matched `wsl.exe` parent. The current uninstall contract uses `Kill(entireProcessTree: true)` because `wsl.exe` can leave `wslhost` and in-distro `sleep` children holding distro state, which can make the following unregister/delete steps fail or hang.
  - confidence: 0.95
- **[P1] Overwrite setup-owned node settings after pairing:** `src/OpenClaw.SetupEngine/SetupContext.cs:213`
  - body: `MergeIntoSettingsFile` copies existing settings and then uses `TryAdd`, so a user with an existing default `settings.json` containing `EnableNodeMode=false` keeps node mode disabled even after SetupEngine pairs the node. The previous local setup explicitly set `EnableNodeMode = true`; setup-owned keys like node mode and capability toggles need to be assigned from the setup config while preserving unrelated preferences.
  - confidence: 0.92
