## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Package SetupEngine.UI before depending on it:** `src/OpenClaw.Tray.WinUI/App.xaml.cs:2720-2724`
  - body: `ShowOnboardingAsync` now returns when `OpenClaw.SetupEngine.UI.exe` is not found, and `CliUninstallHandler` delegates to the same executable. The release workflow still only runs `dotnet publish src/OpenClaw.Tray.WinUI ... -o publish`, and the installer copies that publish folder, so installed builds will not have the `SetupEngine` subfolder that this code searches. First-run setup and uninstall would fail at runtime unless the release/MSIX/installer artifact path is updated and tested.
  - confidence: 0.92
- **[P1] Preserve nonlocal gateway records during cleanup:** `src/OpenClaw.SetupEngine/SetupSteps.cs:166-177`
  - body: This cleanup removes whatever registry record matches the setup URL and deletes its identity directory before creating the new local gateway. A user can have a manually configured or external localhost gateway record at the same URL; deleting it drops stored device tokens and breaks reconnect. Keep the existing ownership/classification guard or require a confirmed app-owned local replacement before removing registry records or identity directories.
  - confidence: 0.88
- **[P2] Keep installer URL shell-quoted and HTTPS-restricted:** `src/OpenClaw.SetupEngine/SetupSteps.cs:525-527`
  - body: `InstallUrl` is config-driven but is interpolated directly into a `bash -lc` script as `curl -fsSL {installUrl} | bash`. Whitespace or shell metacharacters in the config can change the command, and this drops the current setup path's `ShellQuote(...)` plus `--proto '=https' --tlsv1.2` guard. Validate allowed schemes and shell-quote the URL before piping to bash.
  - confidence: 0.87
