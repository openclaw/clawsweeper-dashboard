## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Overwrite setup-owned settings after setup:** `src/OpenClaw.SetupEngine/SetupContext.cs:213`
  - body: `MergeIntoSettingsFile` copies existing settings and then uses `TryAdd`, so a user who previously had `EnableNodeMode=false` or disabled node capabilities will keep those stale values after a successful local setup. That leaves the newly paired local gateway without the expected Windows node/capabilities until the user manually fixes settings; preserve unknown preferences, but assign the keys this setup flow owns.
  - confidence: 0.93
- **[P1] Stop stale keepalives when local setup is inactive:** `src/OpenClaw.Tray.WinUI/App.xaml.cs:1782-1785`
  - body: When the active gateway URL is empty or no longer local, this method now returns without stopping the keepalive that SetupEngine starts for the local WSL distro. Current main explicitly stops the old keepalive in that branch, so after an upgrade or switch to a remote gateway the OpenClawGateway `sleep infinity` process can keep WSL and the gateway alive unexpectedly.
  - confidence: 0.86
- **[P1] Kill the keepalive process tree during rollback:** `src/OpenClaw.SetupEngine/SetupSteps.cs:2355`
  - body: Rollback matches the keepalive `wsl.exe` and calls `proc.Kill()` on only that parent process. The keepalive is launched as `wsl.exe -d <distro> -- sleep infinity`, so killing only the parent can leave the in-distro sleep or WSL host alive and block `wsl --unregister`/VHD cleanup; use process-tree kill and wait or terminate the distro before cleanup.
  - confidence: 0.88
