## Summary

Keep this PR open, but it is not merge-ready: the source-visible release packaging, credential cleanup, and installer execution blockers remain, and the PR still lacks inspectable real behavior proof for the installed setup/uninstall path.

## What This Changes

This PR replaces tray-hosted local gateway onboarding/setup with a standalone SetupEngine library/CLI plus SetupEngine.UI WinUI process, adds setup/E2E tests and CI wiring, and removes the existing in-process onboarding/local setup implementation and tests.

## Best Possible Solution

Land this only after the replacement is packaged in every installed artifact, preserves existing nonlocal gateway identities, restores safe installer input handling, and has redacted installed-layout setup and uninstall proof.

## Reproduction Assessment

Yes for the blocking findings: source inspection of the PR head shows the missing release packaging path, URL-only credential deletion, and unsafe installer command construction. I did not run Windows setup because this review is read-only and tests/builds would create artifacts.

## Solution Assessment

No; the out-of-process SetupEngine direction may be valuable, but this patch is not yet the safest implementation because it breaks installed-layout assumptions and drops current credential and installer safeguards. A safer path is to preserve the current behavior until the replacement has packaging, migration/cleanup parity, and real installed proof.
