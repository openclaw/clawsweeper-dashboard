Full review comments:

- **[P1] Sanitize setup logs before exporting them as-is:** `src/OpenClaw.Tray.WinUI/Services/DiagnosticsBundleBuilder.cs:62`
  - body: The new bundle says persisted log text is included as-is and then appends setup log tails, but `SetupLogger.Sanitize` does not use the shared log sanitizer that redacts local user paths, URLs, hosts, emails, and IPs. Setup logs include install paths, command stdout/stderr, gateway URLs, and port diagnostics, so this can leak private machine details in a shareable bundle; route setup logging through the same sanitizer or drop these tails until that is true.
  - confidence: 0.9
- **[P2] Contain setup log discovery failures:** `src/OpenClaw.Tray.WinUI/Services/DiagnosticsBundleBuilder.cs:138-142`
  - body: `Directory.EnumerateFiles` can throw after `Directory.Exists` succeeds, and that exception escapes `Build`, causing the preview flow to show only a generic failure instead of a partial diagnostics bundle. Wrap setup-log discovery in the same failure-as-section pattern used for file tails so an inaccessible setup log folder does not break the whole export.
  - confidence: 0.82
