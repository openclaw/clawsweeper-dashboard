## Summary

Keep this PR open because current main still uses the older WSL export/import setup path, so the deterministic setup change is unique; however, the patch needs maintainer review before merge because it unconditionally raises the local setup requirement to WSL 2.4.4 and can block users whose current setup path still works.

## What This Changes

The PR replaces SetupEngine's WSL export/import flow with direct clean named WSL install, moves OS/WSL preflight before cleanup, hardens install-path cleanup, updates setup UI/docs, and adds SetupEngine regression tests.

## Best Possible Solution

Land the direct clean-install path only after the compatibility policy is explicit: preserve a fallback for older supported WSL hosts, or intentionally document and approve the new minimum with upgrade proof.

## Reproduction Assessment

No high-confidence local reproduction was run in this read-only Linux review, but source inspection clearly shows current main depends on export/import and the PR replaces it with direct named install. The PR body reports Windows E2E validation for the happy path.

## Solution Assessment

No, not yet: direct clean install is a good direction for determinism, but the unconditional WSL 2.4.4 gate changes upgrade behavior without an explicit compatibility decision or fallback.
