## Summary

Keep open for maintainer review: the patch appears correct and well-proven, but it is still draft and intentionally changes unsupported `screen.snapshot` formats from tolerant fallback/mismatched MIME behavior to a structured error.

## What This Changes

The PR validates and normalizes `screen.snapshot` image formats before capture, derives the response MIME from the normalized format, and adds shared tests plus test README coverage.

## Best Possible Solution

Land the focused hardening after draft readiness if maintainers accept enforcing the documented `png`/`jpeg` snapshot contract; otherwise keep tolerant fallback but still derive the MIME type from the actual encoded format.

## Reproduction Assessment

Yes. Source inspection shows current main can be exercised by calling `screen.snapshot` with an unsupported format such as `svg+xml`, where the Windows backend encodes PNG but echoes the requested format into the response path; the PR body adds live after-fix MCP/winnode proof.

## Solution Assessment

Yes, with one maintainer policy check. Validating `png`/`jpeg`, normalizing `jpg`, and deriving MIME from the validated value is the narrow fix, but maintainers should accept the fail-closed behavior for unsupported formats.
