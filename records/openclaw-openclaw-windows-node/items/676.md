## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Wait for approval resolve responses before clearing UI:** `src/OpenClaw.Shared/OpenClawGatewayClient.cs:427`
  - body: `ResolveExecApprovalAsync` calls `SendTrackedRequestAsync`, but that helper only sends the frame and returns; an `ok:false` response is handled later by `HandleResponse`/`HandleRequestError` without completing this task. Because `RespondToPermissionAsync` clears the pending banner immediately after this await, a rejected, unauthorized, or stale `exec.approval.resolve` makes the UI show Allowed/Denied while the agent can still be blocked and the user has no retry button. Please send this RPC through a response-awaiting path, keep the banner on negative responses, and add a regression test for `ok:false`.
  - confidence: 0.9
