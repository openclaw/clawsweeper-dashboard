- **[P1] Preserve the gateway-doctor terminal API:** `src/OpenClaw.Connection/GatewayTerminalLauncher.cs:7-10`
  - body: Against the review base, the moved helper replaces the current terminal launcher contract with an interface that only exposes `Open`.   - confidence: 0.89
- **[P2] Cancel the wizard session before disconnecting:** `src/OpenClaw.SetupEngine.UI/Pages/WizardPage.xaml.cs:859`
  - body: This restart path can run while a gateway wizard session is still live. If the WSL restart command fails before the daemon is killed, `_client` has already been cleared by `DisconnectAsync()`, so the later error/retry path cannot send `wizard.cancel` and the next start can dead-end on an already-running session.
  - confidence: 0.86
- **[P2] Keep direct terminal launches visible:** `src/OpenClaw.Connection/GatewayTerminalLauncher.cs:141`
  - body: Current main uses `UseShellExecute = !command.UsesWindowsTerminal` so direct `wsl.exe`/`ssh.exe` fallbacks allocate a visible console. This moved copy forces `UseShellExecute = false`, so if Windows Terminal is unavailable or fails, Open terminal can start without giving the user a visible shell.
  - confidence: 0.8
