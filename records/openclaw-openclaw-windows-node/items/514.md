## Summary

The latest commit appears to address the earlier wrong-thread remount edge by forcing a remount on every pending SessionsPage handoff, but this PR is still not merge-ready because the SessionsPage rewrite drops existing localization hooks and the screenshot does not prove the Chat handoff or overflow actions.

## What This Changes

The PR restructures SessionsPage into a Fluent-style scrollable layout, filters cron sessions, moves actions into an overflow flyout, and adds an Open chat handoff through HubWindow and ChatPage.

## Best Possible Solution

Keep the Fluent Sessions redesign and deep-link only after preserving SessionsPage localization coverage and adding redacted real UI proof of the Sessions-to-Chat and overflow-action flows.

## Reproduction Assessment

Yes for the review defect: source inspection shows current main has SessionsPage x:Uid/resource bindings and the PR head replaces them with hard-coded English strings. The Chat handoff itself still needs observed UI proof, but the localization regression is high-confidence from the diff.

## Solution Assessment

No. The forced remount is a reasonable handoff direction, but the page redesign should preserve existing localization hooks and resource entries before it is the best maintainable solution.
