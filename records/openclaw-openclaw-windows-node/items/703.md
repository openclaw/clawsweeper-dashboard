## Summary

Keep open; current main still appears vulnerable to the reported packaging/runtime failure. The repo still pins and copies the x64 app-local VC runtime from `VCRuntime.CefSharp.140` 1.0.5, while release validation only probes `libsodium.dll` and does not verify Sherpa/ONNX native initialization or the VC runtime file version.

## What This Changes

Fix the tray startup crash loop by shipping and validating VC++ runtime DLLs compatible with the bundled Sherpa/ONNX Runtime native dependencies.

## Best Possible Solution

Update release packaging so app-local VC runtime DLLs cannot shadow with an older runtime, then extend native dependency validation to cover Sherpa/ONNX loading or minimum Microsoft runtime file versions.
