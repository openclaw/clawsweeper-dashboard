## Summary

Keep open: this PR addresses a real canvas URL handling problem, but the branch is not merge-ready because it bypasses the current high-risk URL approval path and still sources the tunnel rewrite origin from settings instead of the active gateway record.

## What This Changes

This PR adds `CanvasGatewayUrlRewriter`, changes `canvas.navigate` to navigate the WebView canvas, passes a configured gateway URL into `CanvasWindow`, and adds tray tests for external, tunnel, and relative URL rewriting.

## Best Possible Solution

Land a narrower fix that preserves external URLs, sources configured and effective origins from the active gateway connection state, and only moves `canvas.navigate` into WebView2 with equivalent high-risk approval and maintainer-accepted compatibility semantics.

## Reproduction Assessment

## Solution Assessment

No. The useful URL-rewrite fix should stay, but the branch needs an equivalent high-risk approval gate and active-record-based tunnel origin plumbing before it is the narrowest maintainable solution.
