## Summary

Keep this PR open, but not merge-ready: it usefully narrows gateway URL rewriting and adds live proof, yet approved high-risk URLs are still routed into `CanvasWindow.Navigate`, where the existing WebView safety gate rejects localhost/private destinations that current main launches through the browser after approval.

## What This Changes

The PR changes `canvas.navigate` to navigate the WebView after URL risk approval, adds denial response handling, narrows gateway URL rewriting with configured/effective origins, and adds shared/tray coverage for denial and rewrite behavior.

## Best Possible Solution

Align `canvas.navigate` to one explicit maintainer-approved policy: public/external WebView navigation if desired, denied prompts returning false, and approved high-risk destinations either deliberately supported in WebView or intentionally kept on browser fallback with tests and live proof.
