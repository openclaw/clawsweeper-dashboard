## Summary

Keep this PR open, but it is not merge-ready: the branch improves external canvas URL handling, yet the saved SSH-tunnel compatibility path still sources the configured gateway from settings instead of the active gateway record, and the tunnel rewrite path still lacks real Windows/WebView proof.

## What This Changes

The branch routes `canvas.navigate` into the WebView canvas, narrows gateway URL rewriting with a new helper, and adds tray-side rewrite and source-shape tests.

## Best Possible Solution

Keep the WebView/external URL fix, but source configured gateway origin from the active gateway record or connection-manager state and require redacted real Windows/WebView proof for a saved SSH-tunnel gateway URL before merge.

## Reproduction Assessment

Yes. Source inspection shows a manager-owned node connects tunneled gateways through the active `GatewayRecord`, while the PR's configured-origin helper reads `SettingsManager`; a saved tunnel record not mirrored into settings reproduces the missed rewrite path.

## Solution Assessment

No. The direction is useful, but the branch should use the active gateway record or connection-manager state for the configured origin and include real saved-tunnel proof before it is the safest fix.
