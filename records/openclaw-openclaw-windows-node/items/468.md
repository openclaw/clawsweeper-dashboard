## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Hard-gate MSIX assets before release creation:** `.github/workflows/ci.yml:555-559`
  - body: The release now advertises only MSIX/AppInstaller downloads, but the MSIX job and downloads are still allowed to fail. If one architecture is missing, this step can still create a release with missing or mismatched install/update assets. Make both MSIX builds, downloads, signing, and AppInstaller renders hard requirements before publishing a latest release.
  - confidence: 0.9
- **[P1] Make the recovery command available after MSIX uninstall:** `docs/uninstall-msix.md:53-56`
  - body: The docs make `openclaw-winnode --purge-wsl-orphans` the supported recovery path, but this PR explicitly defers packaging the CLI and the manifest does not expose an appExecutionAlias. After a user uninstalls the MSIX, that command will not be available unless they separately installed the CLI, so either package the CLI here or make the PowerShell fallback the primary recovery path.
  - confidence: 0.91
- **[P1] Drop keyless chat events instead of routing them to main:** `src/OpenClaw.Tray.WinUI/Chat/OpenClawChatDataProvider.cs:859`
  - body: This fallback maps chat and agent events without a canonical session key into the synthetic `main` thread. Current main deliberately drops and logs those events because they cannot be safely matched to the optimistic timeline; routing them to `main` can mix output into the wrong session and hide a gateway protocol bug. Restore the drop/log behavior and its regression coverage.
  - confidence: 0.88
- **[P1] Keep the marker-backed WSL keepalive lifecycle:** `src/OpenClaw.Tray.WinUI/Services/LocalGatewaySetup/LocalGatewaySetup.cs:1720`
  - body: This version of `WslDistroKeepAlive` only tracks processes in the current tray process and only stops them on that process exit. It drops the marker-backed adoption/stop logic and the startup, repair, and remove re-arm points from current main, so the local gateway can time out after tray restart or logon and stale keepalives are harder to clean up. Preserve the marker-backed lifecycle while layering the MSIX changes on top.
  - confidence: 0.87
