- **[P1] Read the workflow tag input from the environment:** `.github/workflows/appinstaller-feed-pr.yml:33`
  - body: `inputs.tag` is spliced directly into a PowerShell single-quoted string in a workflow that has `contents: write` and `pull-requests: write`.   - body: This branch works around MSIX SetupEngine crashes by code-building the first-run pages, but the changed navigation still routes to `PermissionsPage` and `CompletePage`, whose constructors still call `InitializeComponent()`. Once the gateway wizard finishes or an early lock/error path is hit, the setup flow can crash on the same fragile XAML path instead of completing.
  - confidence: 0.78
- **[P2] Do not auto-close unowned Terminal windows after builds:** `build.ps1:281`
  - body: Successful builds now run a global window closer that matches any visible Windows Terminal window titled `Terminal` above the size threshold. Because the script does not baseline windows or tie them to OpenClaw/msbuild-owned processes, `./build.ps1` can hide and close an unrelated fresh idle Terminal window on the developer's desktop.
  - confidence: 0.82
