## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Constrain ghost cleanup to owned windows:** `scripts/cleanup-ghost-windows.ps1:153-181`
  - body: `Find-GhostFrames` only checks for a visible WindowsTerminal window with class `CASCADIA_HOSTING_WINDOW_CLASS`, title `Terminal`, owner process `WindowsTerminal`, and a large size. Because `build.ps1` now runs this script automatically, a real user Terminal window with the default title can match and be hidden/closed after a successful build, daemon cycle, or scheduled task. Tie cleanup to windows spawned by this build/test run, record a baseline, or keep this as a manual/WhatIf recovery tool.
  - confidence: 0.89
- **[P2] Propagate StartupTask failures to settings:** `src/OpenClaw.Tray.WinUI/Services/AutoStartManager.cs:100-102`
  - body: The existing settings page still calls `AutoStartManager.SetAutoStart(...)` after saving the setting, but this wrapper now fire-and-forgets `RequestEnableAsync()` and discards a `false` result. In packaged installs, cancelling the Windows StartupTask prompt or hitting `DisabledByUser` leaves `AutoStart` persisted as enabled even though Windows did not enable launch-at-login. Make the settings path await `SetAutoStartAsync` and revert the toggle/settings on failure, like the tray-menu toggle already does.
  - confidence: 0.86
