- **[P2] Surface packaged update-check results:** `src/OpenClaw.Tray.WinUI/App.xaml.cs:3596-3597`

  - confidence: 0.9
- **[P2] Await StartupTask failures from settings:** `src/OpenClaw.Tray.WinUI/Services/AutoStartManager.cs:100-103`
  - body: The compatibility wrapper discards the boolean from `SetAutoStartAsync`, and `SettingsPage.PersistAutoStart()` still saves the toggle before calling this fire-and-forget method. In packaged installs, `RequestEnableAsync()` can return disabled states or fail, leaving settings.json saying AutoStart=true while Windows will not launch the app; make the settings page await the async API and revert/report failure like the tray-menu path.
  - confidence: 0.88
