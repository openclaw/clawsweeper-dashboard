## Summary

Current main already implements the requested VC++ runtime packaging fix: x64 publish now replaces the stale NuGet-provided 14.29 DLLs with current Visual Studio redist DLLs and the release verifier rejects under-version runtimes. ## What This Changes

The issue asks OpenClaw Companion to stop bundling the old MSVCP140.dll 14.29 runtime that crashes speech/text paths and to ship a newer VC++ runtime in the next Companion release.

## Best Possible Solution

Keep the main-branch packaging and verifier fix, then ship a post-v0.6.3 Companion release containing it.
