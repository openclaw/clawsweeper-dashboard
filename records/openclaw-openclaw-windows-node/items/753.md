Overall correctness: patch is incorrect

Full review comments:

- **[P2] Replace the old connection issue notification:** `src/OpenClaw.Tray.WinUI/App.xaml.cs:2278-2288`
  - body: `AppNotificationService` only coalesces entries with the same `DedupeKey`, but this path keeps the fixed ID `connection:issue` while changing the dedupe key for each error/pairing state. If the gateway error changes, the old notification remains current/queued; when the connection later clears, `Dismiss("connection:issue")` removes only one matching entry and can reveal a stale issue that is no longer true. Dismiss or replace the previous fixed-ID notification before showing the new key, or use a stable dedupe key for this singleton notification.
