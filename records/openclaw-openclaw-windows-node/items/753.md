- **[P2] Replace the old connection issue notification:** `src/OpenClaw.Tray.WinUI/App.xaml.cs:2321-2331`
  - body: `AppNotificationService` coalesces by `DedupeKey`, but this path keeps the fixed id `connection:issue` while changing the key for each error or pairing state. A second connection issue can be queued behind the first, and clearing the issue removes only one matching id, revealing a stale issue that is no longer true.
  - confidence: 0.88

  - body: Sandbox risk notifications use the fixed id `sandbox:risk` with a dedupe key derived from the current risk. If the risk changes, the previous risk remains active or queued instead of being replaced, so the banner can keep showing an obsolete sandbox warning until the user manually clears it.
  - confidence: 0.81
