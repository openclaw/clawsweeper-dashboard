## Summary

The request remains necessary on current main: both web Canvas and native A2UI Canvas use ordinary WinUI windows without a fullscreen presenter transition or F11/Escape fullscreen handling. It is a bounded, low-priority UX feature with no merged or open canonical replacement, but implementation should cover both Canvas surfaces consistently.

## What This Changes

Add a borderless fullscreen mode to Windows Canvas, with F11 toggling it and Escape exiting it.

## Best Possible Solution

Add one reusable, reversible fullscreen controller used by both Canvas window implementations, preserving prior window state and reliably handling F11 and Escape even when embedded content has focus.
