## Summary

This feature remains necessary: current main and v0.6.3 provide separate WebView2 and native A2UI Canvas windows, but neither implements an F11/Escape fullscreen transition, and no canonical issue or implementation PR was found.

## What This Changes

Add a borderless fullscreen mode to Windows Canvas windows, toggled with F11 and exited with F11 or Escape.

## Best Possible Solution

Add reusable fullscreen state management shared by both Canvas windows, using the WinUI AppWindow presenter API to enter borderless fullscreen and safely restore the previous presenter and bounds on F11, Escape, or close.

## Reproduction Assessment

Not applicable: this is a request for a new capability rather than broken established behavior. Current-main source inspection gives a high-confidence way to verify that no F11/Escape fullscreen path exists today.

## Solution Assessment

Yes, with one refinement: the keyboard behavior should be backed by shared fullscreen state management used by both Canvas window implementations, rather than duplicated independently.
