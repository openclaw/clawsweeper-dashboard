## Summary

## What This Changes

The PR adds an approved execution payload for exec-approvals V2, direct-argv launching in `LocalCommandRunner`, MXC fail-closed handling for direct argv, and shared tests around payload and runner behavior.

## Best Possible Solution

Keep the direct-argv payload design, but land it only after payload construction rejects modifiers at every env unwrap layer and regression tests cover nested transparent and nested modified wrappers.

## Reproduction Assessment

Yes. Source inspection gives a focused path: build an identity for `env env FOO=bar node script.js`; `HasModifiers` returns false at the outer level, then `UnwrapForResolution` can drop the inner modifier before payload construction.

## Solution Assessment

No. The local direct-argv runner and MXC fail-closed guard are the right shape, but the payload builder needs an all-level env modifier guard before this security contract is merge-ready.
