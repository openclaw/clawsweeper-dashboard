## What This Changes

## Best Possible Solution

Build and validate the ExecApprovedExecution payload before any allowlist side effects in the prompt/fallback branch, add regression coverage for no persistence on payload failure, then report the AGENTS-required validation.

## Reproduction Assessment

Yes. Source inspection of the PR head shows the prompt/fallback branch writes allowlist side effects before BuildApprovedExecution can return null for an unresolved executable or modified env wrapper.

## Solution Assessment

No. The direct-argv direction is maintainable, but the payload must be built before any approval-state side effects so fail-closed results do not mutate allowlist state.
