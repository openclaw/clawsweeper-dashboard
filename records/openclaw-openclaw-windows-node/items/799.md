## Summary

## What This Changes

## Best Possible Solution

Keep the direct-argv payload direction, but fail closed for or faithfully model modified `env` wrappers before merge, add assignment and flag regression coverage, and report the AGENTS-required validation.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a modified `env` command can be resolved through `UnwrapForResolution`, then `BuildApprovedExecution` emits only the unwrapped argv while losing the wrapper’s environment semantics.

## Solution Assessment

No. The direct-argv direction is maintainable, but the payload builder should only unwrap transparent `env` wrappers or fail closed when `env` has assignments or flags.
