## Summary

## What This Changes

The branch adds an ExecApprovedExecution payload to V2 allow results, direct-argv execution in LocalCommandRunner, and shared tests for argv fidelity, env wrappers, and no-store-write failure cases.

## Best Possible Solution

Land the direct-argv approval payload only after unrepresentable payloads are rejected before approval side effects and every active ICommandRunner honors or explicitly rejects Argv.

## Reproduction Assessment

Yes. Source inspection of PR head shows batch payloads are built before the runner rejects them, and current main shows MxcCommandRunner serializes only legacy Command/Args fields while NodeService uses it for system.run.

## Solution Assessment

No. The direct-argv design is the right shape, but this branch should fail closed before approval writes for unrepresentable batch payloads and define MxcCommandRunner's Argv behavior before merge.
