## Summary

Keep this PR open, but not merge-ready: the remaining blocker is a concrete master-detail layout bug where long workspace filenames can expand the file-list pane and starve the detail pane.

## What This Changes

The PR redesigns the per-agent Workspace page into a WinUI master-detail file list with rendered/raw markdown viewing, localized workspace status text, and a Workspace icon catalog entry.

## Best Possible Solution

Keep the master-detail direction, but bound the file-list pane so filenames trim inside a stable width and the detail pane remains usable across practical window sizes.

## Reproduction Assessment

Yes for the review finding by source inspection: the PR combines an Auto-width master column with unconstrained no-wrap filename content, so long filenames can force the left pane wider than intended. I did not run the WinUI app in this read-only review.

## Solution Assessment

No as submitted: the redesign is a maintainable direction, but it needs an explicit width constraint or sizing contract for the file-list pane before merge.

- **[P2] Bound the workspace file list column:** `src/OpenClaw.Tray.WinUI/Pages/WorkspacePage.xaml:126`
