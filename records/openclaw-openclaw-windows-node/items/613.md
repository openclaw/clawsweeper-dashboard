## Summary

## Best Possible Solution

Yes from source inspection: configure `EnableMcpServer=true`, `EnableNodeMode=false`, an active gateway record with a persisted node token, and no operator credential; the PR calls `ConnectNodeOnlyAsync` and returns before `TryStartLocalMcpOnlyNode()` can preserve MCP-only mode.

## Solution Assessment

No: the reconnect direction is useful, but the current condition uses an MCP-inclusive node-service predicate for a gateway-node reconnect. A dedicated `EnableNodeMode`/gateway-node predicate is the narrower maintainable fix.
