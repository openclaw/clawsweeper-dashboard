## Summary

Keep this PR open: it targets a real tray-menu flicker problem, but the current patch has a blocking resource-scope bug and still lacks real tray-menu behavior proof.

## What This Changes

The PR adds a tray-menu-local ToggleSwitch template and helper methods, then routes connection and permission tray toggles through them to remove open-time switch flicker.

## Best Possible Solution

Keep the tray-scoped instant toggle approach only after resolving the style from the tray menu/window resource scope and adding redacted real tray-menu proof.

## Reproduction Assessment

Yes. For the introduced blocker, PR head puts the style in `RootGrid.Resources` but looks it up through `Application.Current.Resources` before tray-menu build paths create toggles; I did not run the Windows tray UI in this read-only review.

## Solution Assessment

No. The scoped style direction is reasonable, but the current implementation uses the wrong resource dictionary; the narrow maintainable fix is to resolve the style from the tray window/root scope or make it truly app-scoped.
