## Summary

Keep open: the tray flicker fix is useful, but the patch currently resolves a Grid-scoped style from `Application.Current.Resources`, which can break tray-menu toggle construction, and it lacks real tray-menu proof.

## What This Changes

The branch adds a TrayMenuWindow-scoped ToggleSwitch control template, centralizes tray toggle construction/state updates, and routes the gateway and permissions toggles through the helper.

## Best Possible Solution

Keep the scoped tray-menu flicker fix, but resolve the style from the actual window/root resource scope or move it to an app-level resource, then attach redacted live tray-menu proof.

## Reproduction Assessment

Yes for the introduced blocker by source inspection: PR head puts the style in `RootGrid.Resources` but looks it up through `Application.Current.Resources` before using the helper from tray-menu build paths. I did not run the Windows tray UI in this read-only review.

## Solution Assessment

No. The scoped style direction is reasonable, but the current implementation uses the wrong resource dictionary; the narrow maintainable fix is to resolve the style from the tray window/root resource scope or make it truly app-scoped.

Full review comments:

- **[P1] Resolve the tray style from the right resource scope:** `src/OpenClaw.Tray.WinUI/Windows/TrayMenuWindow.xaml.cs:162`
  - body: `TrayMenuInstantToggleSwitchStyle` is added under `RootGrid.Resources` in `TrayMenuWindow.xaml`, but this helper reads it from `Application.Current.Resources`.
