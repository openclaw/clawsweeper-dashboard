Full review comments:

- **[P1] Persist profile choices into runtime settings:** `src/OpenClaw.SetupEngine.UI/Pages/CapabilitiesPage.xaml.cs:153-164`
  - body: The Read-only and Standard profiles only update `_config.Capabilities`; after setup the tray rebuilds node capabilities from persisted `SettingsManager` `Node*` flags, which remain default-on. A user who selected Read-only can therefore still expose commands, camera, location, or browser capabilities after restart unless the profile is also mapped into the persisted settings source of truth.
  - confidence: 0.92
- **[P1] Persist the startup toggle before restarting:** `src/OpenClaw.SetupEngine.UI/Pages/CompletePage.xaml.cs:98`
  - body: Defaulting the completion toggle on only passes `true` to the immediate restart handoff; setup has already written `AutoStart=false`, and normal tray settings application later calls `SetAutoStartAsync(_settings.AutoStart)`. Save the user's choice to settings, or keep the existing opt-in default, so startup state does not flip after restart.
  - confidence: 0.9
