Full review comments:

- **[P1] Persist capability profiles into tray settings:** `src/OpenClaw.SetupEngine.UI/Pages/CapabilitiesPage.xaml.cs:153-164`
  - body: The Read-only and Standard profiles only update `_config.Capabilities`, but `VerifyEndToEndStep` writes settings from `_config.Settings` and the restarted tray registers capabilities from SettingsManager `Node*` flags. With the default config those flags stay true, so a user who selected Read-only can later advertise `system.run`, camera, location, or browser after the tray restarts.
  - confidence: 0.91
- **[P1] Persist the startup toggle before restarting:** `src/OpenClaw.SetupEngine.UI/Pages/CompletePage.xaml.cs:98`
  - body: Defaulting `StartupToggle` on makes Finish pass `true` to `RequestSetupCompleted`, but setup has already written `Settings.AutoStart=false` and the restarted tray reapplies `_settings.AutoStart` on launch. That can enable the registry once and then disable it or show it as off after restart, so the user's completion choice is not durable.
  - confidence: 0.9
