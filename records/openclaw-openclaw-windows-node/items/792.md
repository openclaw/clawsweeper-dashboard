## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Persist profile choices into tray settings:** `src/OpenClaw.SetupEngine.UI/Pages/CapabilitiesPage.xaml.cs:153-163`
  - body: `WriteCapabilities()` only copies the selected profile into `_config.Capabilities`, but setup later persists `ctx.Config.Settings` to `settings.json`. On a fresh install those settings still default camera, location, and browser to enabled, so a user who chooses Standard or Read-only can finish setup and later have Settings/NodeService re-enable capabilities they declined. Update the matching `TraySettingsConfig` flags when writing the selected profile and cover that persisted state in setup tests.
  - confidence: 0.9
- **[P1] Keep launch-at-startup opt-in:** `src/OpenClaw.SetupEngine.UI/Pages/CompletePage.xaml.cs:97-98`
  - body: This changes the setup completion toggle from off to on by default. Because `Finish` passes the toggle state to `RequestSetupCompleted`, first-run users can get an OS autostart entry without explicitly opting in, which contradicts the existing setup default and the committed complete-page proof image. Keep the default off unless maintainers deliberately accept this product/default change and add matching proof.
  - confidence: 0.88
