## Summary

Keep open: current main has a meaningful safe Markdown subset and selection-enabled TextBlocks, but the requested full-fidelity Markdown viewer and select-all chat-text behavior are not definitely implemented. The existing renderer intentionally flattens links/images/HTML and renders multi-block messages as separate controls, while the parent tracking work remains open.

## What This Changes

The issue requests a chat Markdown viewer with full-fidelity rendering and text selection, including the ability to select all chat text.

## Best Possible Solution

Define a safe chat Markdown viewer contract, then implement a single selectable message surface or equivalent select-all behavior while preserving the current no-fetch/no-click policy unless maintainers explicitly change it.

## Reproduction Assessment

No. I did not run a live WinUI repro, but source inspection gives a plausible current-main path: multi-block Markdown is rendered as separate selection-enabled controls rather than one selectable document surface.

## Solution Assessment

No. Treating the current renderer as complete is not the best solution; the maintainable path is to settle the fidelity and safety contract first, then implement focused viewer and selection behavior with tests or visual proof.
