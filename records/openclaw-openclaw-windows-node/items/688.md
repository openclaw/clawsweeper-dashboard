## Summary

Keep open: current main still has no per-step Back navigation in the gateway-driven setup wizard, and the maintainer discussion says this needs a gateway protocol or declarative wizard decision rather than a Windows-only button.

## What This Changes

Request Back navigation in the Windows setup gateway wizard so users can return to earlier setup steps without cancelling and restarting.

## Best Possible Solution

Define the gateway wizard navigation contract first, preferably a declarative manifest with local draft state or an explicit back/previous RPC with side-effect boundaries, then update the Windows setup UI and regression tests around that contract.

## Reproduction Assessment

Yes for the observable gap: source inspection of current main shows the gateway setup wizard has forward, restart, skip, and cancel paths but no per-step Back action or wizard.back RPC. No live Windows run was performed in this read-only review.

## Solution Assessment

No for a Windows-only Back button on the current protocol. The safer direction is to define gateway-supported back/manifest semantics before changing the client UI.
