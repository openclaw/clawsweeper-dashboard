## Summary

Keep this PR open for review: the branch adds useful approval plumbing, but the central in-bubble approval behavior is still routed through the composer banner, and the external PR has no real behavior proof for the approval flow.

## What This Changes

The PR maps gateway approval events into chat permission state, sends Allow/Deny slash commands, preserves pending permissions across tool events, adds seam repair/proactive bootstrap changes, updates localized banner copy, and adds tray tests.

## Best Possible Solution

Render exec approvals as timeline content inside the assistant bubble, keep the existing gateway approval lifecycle, and prove Allow/Deny plus resolved clearing against a real gateway before landing the drive-by fixes.

## Reproduction Assessment

Yes for the review finding: source inspection shows approval events become PendingPermission, OpenClawChatRoot passes that state to OpenClawComposer, and the composer renders the banner above the input. No real setup proof was provided for the live gateway path.

## Solution Assessment

No. The narrow maintainable fix is to model approval prompts as timeline content rendered by OpenClawChatTimeline inside the assistant bubble, while keeping the slash-command response plumbing after lifecycle tests and proof.
