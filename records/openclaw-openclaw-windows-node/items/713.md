## Summary

Keep open: current main still sources x64 app-local VC runtime DLLs from VCRuntime.CefSharp.140, so the fix direction is useful, but this draft is missing real Windows behavior proof and has a release-validator concern because it adds the managed `sherpa-onnx.dll` wrapper to a Win32 native `LoadLibrary` probe.

## What This Changes

This draft PR changes x64 publish to replace NuGet VC runtime DLLs with Visual Studio redist DLLs, adds release-time VC runtime and TTS native stack checks, and updates release docs/tests.

## Best Possible Solution

Land a narrow release fix that sources x64 VC runtime DLLs from a current Microsoft redist source, keeps native probes scoped to native DLLs, makes the x64 publish prerequisite an explicit maintainer-approved choice, and includes redacted Windows x64 publish/tray startup proof.

## Reproduction Assessment

No high-confidence runtime reproduction was run here. Source inspection shows current main still publishes x64 VC runtime DLLs from `VCRuntime.CefSharp.140` 1.0.5 while the app references ONNX/Sherpa native stacks, so the reported failure is plausible but not locally reproduced on Windows.

## Solution Assessment

No, not as submitted. The MSBuild direction is plausible, but the release verifier should avoid native-probing the managed Sherpa wrapper and the PR needs real Windows x64 after-fix proof before merge.
