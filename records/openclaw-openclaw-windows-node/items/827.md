## Summary

Keep open: current main still violates the documented MCP-only mode contract, and this PR is a meaningful repair path, but it still has a bounded MCP error-surfacing defect and lacks visible proof for the new Connection/Permissions UI states.

## What This Changes

This PR separates gateway-node startup gating from local MCP initialization, adds MCP-only and node-starting UI states with localization, adjusts MCP-only capability controls, and adds tray tests.

## Best Possible Solution

Land a version that keeps the documented startup split, surfaces MCP startup failures anywhere Local MCP is enabled, and includes redacted visible proof for the Connection/Permissions UI plus the runtime MCP smoke.

## Reproduction Assessment

Yes, from source inspection: current main documents MCP-only mode as local-only with no gateway requirement, but the startup gate still treats EnableMcpServer as node-enabled and the UI maps MCP-only to node off. The contributor also provided a live MCP-only terminal smoke for the PR head.

## Solution Assessment

Mostly yes: separating gateway-node startup from local MCP startup matches the documented mode matrix and is the narrow maintainable direction. The patch should also surface MCP startup errors when Node Mode is on and add visible UI proof before merge.
