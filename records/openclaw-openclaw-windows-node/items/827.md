## Best Possible Solution

Preserve the node/MCP startup split, surface MCP startup failures in the MCP status UI regardless of Node Mode, and add redacted visual proof of the new Connection/Permissions states before merge.

## Reproduction Assessment

Yes from source inspection. Current main documents MCP-only as local-only but still routes MCP enablement through the gateway-node gate and renders node mode as off when MCP is enabled without node mode.

## Solution Assessment

Mostly yes: separating gateway-node startup from local MCP startup is the maintainable direction. The patch should also surface MCP startup failures when Node Mode is on, and the proof should cover the visible UI states it changes.
