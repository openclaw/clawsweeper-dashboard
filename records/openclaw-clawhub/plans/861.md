The related open report at https://github.com/openclaw/clawhub/issues/651 shows the same raw transport-failure class, and no open PR currently owns the fix.

## Plan

ApiRoutes.whoami ...)` path, preserve `/api/v1/whoami` verification and never save unverified browser tokens, but make low-level fetch/curl transport failures produce sanitized actionable output with network/proxy/TLS/DNS context when available plus login-specific guidance for proxy setup, `clawhub login --device`, and `clawhub login --token`.
