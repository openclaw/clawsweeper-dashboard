# Coding Plan for openclaw/clawhub#874: Publish silently drops extensionless files and .tsv files

Source report: [records/openclaw-clawhub/items/874.md](records/openclaw-clawhub/items/874.md)

## Summary

Keep open. Current main still matches the reported publish bug: CLI publish, archive upload, and GitHub import rely on extension allowlists before any content fallback; `.tsv` is still absent; and there is no linked implementation PR for this issue.

## Plan

Fix the skill publish file classification bug where `clawhub publish` silently omits extensionless text files and `.tsv` files. Implement a shared conservative text-file classification policy: add `tsv` to the shared schema allowlist and the CLI mirror; classify extensionless text files by content without admitting binary files; preserve path safety, `.git`/`node_modules`, ClawHub ignore files, and macOS junk handling; and warn authors about skipped non-ignored files during CLI publish. Keep CLI publish/listing, zip hashing, browser/archive upload expansion, GitHub import, and Convex publish validation consistent. Add regression tests for `bin/openclaw-kraken` or `bin/gurkerl`, `config/endpoints.tsv`, binary extensionless rejection, skipped-file warnings, and quiet expected ignores. Add a `CHANGELOG.md` entry under the unreleased fixes section for the user-facing publish fix; do not broaden support to arbitrary binary/plugin package upload or change package publish semantics beyond safe shared-helper consistency.
