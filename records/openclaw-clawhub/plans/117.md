## Summary

Keep this issue open. Current main has owner soft-delete and restore through the backend, HTTP API, CLI, docs, and tests, but the signed-in web app still lacks a discoverable owner delete/restore workflow and the dashboard cannot show owner-deleted skills for restore.

## Plan

Implement owner-facing skill soft-delete and restore in the ClawHub web app. Reuse existing soft-delete semantics; do not add normal-owner hard delete and do not weaken moderation/scanner/security restore gates. Add a destructive confirmation flow from skill settings and/or dashboard row actions, derive authorization server-side, support direct owners and publisher admins consistently, and provide a way for owners to find and restore their own owner-deleted skills without full-table scans. Preserve current CLI/API behavior. Add focused regression coverage for settings/dashboard delete and restore visibility, owner-deleted discovery, and guard behavior. Update specs/manual-testing or user docs only where the web workflow changes documented behavior, and add a CHANGELOG.md entry if this repair lane produces the implementation PR.
