# Coding Plan for openclaw/clawhub#117: Cannot delete own skills

Source report: [records/openclaw-clawhub/items/117.md](records/openclaw-clawhub/items/117.md)

## Summary

Keep this issue open. Current main has owner soft-delete/restore through backend, HTTP API, CLI, and docs, but the signed-in web app still lacks a discoverable owner skill delete/restore workflow; a prior closed PR proposed that exact web flow but was not merged.

## Plan

Implement signed-in owner skill delete and restore in the ClawHub web app. Reuse setSkillSoftDeletedInternal semantics through an appropriate Convex/web mutation boundary; do not add normal-user hard delete. Add a destructive owner delete action with confirmation in SkillOwnershipPanel and/or skill dashboard row actions, then make owner-deleted skills discoverable for restore with an include-deleted owner dashboard/settings path. Preserve security gates from convex/skills.ts: owners can restore only owner-initiated hides, while moderation/scanner/ban/merge/security hides stay staff-controlled. Align direct owner and publisher-member authorization with existing owner tools. Add focused regression coverage for the owner settings flow, dashboard row/action visibility, active-vs-deleted dashboard query behavior, and backend authorization edge cases. Include a concise CHANGELOG.md Unreleased entry for the user-facing web feature. Do not change CLI/API delete semantics except where needed to support the web flow safely.
