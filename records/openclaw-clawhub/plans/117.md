# Coding Plan for openclaw/clawhub#117: Cannot delete own skills

Source report: [records/openclaw-clawhub/items/117.md](records/openclaw-clawhub/items/117.md)

## Summary

Keep this issue open. Current main has owner soft-delete/restore primitives in the backend, HTTP API, CLI, docs, and tests, but the signed-in web app still lacks owner-facing skill delete/restore controls and the dashboard filters soft-deleted skills out before owners could restore them.

## Plan

Implement owner-facing skill delete and restore in the ClawHub web app. Reuse the existing soft-delete/undelete semantics in `convex/skills.ts`; derive the actor from authenticated identity server-side; support direct owners, linked personal publishers, and publisher members consistently; and preserve the existing rule that normal owners cannot restore moderation/security/malware-hidden skills. Add confirmation UI from owner settings and/or dashboard row actions, add a discoverable way for owners to see or restore owner-initiated soft-deleted skills, and do not add normal-owner hard delete. Update focused UI/backend tests and any specs/docs needed for changed intended behavior. Stop and ask for maintainer guidance if the implementation would require a new hard-delete policy or broad moderation-policy change.
