# Coding Plan for openclaw/clawhub#117: Cannot delete own skills

Source report: [records/openclaw-clawhub/items/117.md](records/openclaw-clawhub/items/117.md)

## Summary

Keep this issue open. Current main supports owner skill soft-delete/restore through the API and CLI, but the signed-in web app still has no discoverable skill delete/restore control and the owner dashboard still excludes soft-deleted skills before owners can restore them.

## Plan

Fix https://github.com/openclaw/clawhub/issues/117 by adding owner-facing skill soft-delete and restore in the signed-in web app. Reuse the existing soft-delete semantics in convex/skills.ts; add a public mutation or safe wrapper that derives the authenticated user server-side and does not accept a user id from the client. Expose a destructive, confirmed Delete skill action from SkillOwnershipPanel and/or the dashboard row menu, and expose a way for owners to see and restore their own soft-deleted skills. Preserve all moderation/security gates: owners must not be able to restore or re-delete over moderator, scanner, ban, or security-redaction hides, and normal owners must not hard-delete. Keep org/publisher authorization consistent with dashboard membership rules. Update focused UI and Convex tests, update public docs/spec text only if the web behavior changes documented flows, and add a concise CHANGELOG.md unreleased entry for the user-facing web capability.
