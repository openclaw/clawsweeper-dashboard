# Coding Plan for openclaw/clawhub#117: Cannot delete own skills

Source report: [records/openclaw-clawhub/items/117.md](records/openclaw-clawhub/items/117.md)

## Summary

Keep this issue open. Current main has owner soft-delete and restore through backend, HTTP API, CLI, docs, and tests, but the signed-in web app still lacks a discoverable skill delete/restore workflow and owner dashboard queries exclude soft-deleted skills before owners can restore them.

## Plan

Implement owner-facing skill soft-delete and restore in the ClawHub web app. Reuse the existing soft-delete semantics in convex/skills.ts, but expose them through a public mutation that derives the actor with requireUser server-side; do not accept a client-supplied userId and do not add hard-delete behavior. Add a Delete skill action with explicit destructive confirmation in SkillOwnershipPanel and/or dashboard row actions. Add a restore path for owner-deleted skills, likely by extending the owner dashboard with an include-deleted/hidden view or a focused owner-only query that can return owner-initiated soft-deleted skills without exposing moderation-hidden content. Preserve existing gates: owners may restore owner-initiated hides, but not moderator/system/security/merge/malware hides. Add focused UI and Convex tests, update docs/specs only if user-visible behavior wording changes, and avoid unrelated dashboard or moderation refactors.
