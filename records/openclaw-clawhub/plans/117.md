Keep this issue open. Current main has owner soft-delete/restore primitives in the API, CLI, docs, and backend, but the signed-in web app still lacks a discoverable skill delete/restore workflow and the dashboard excludes soft-deleted skills before owners can restore them.

## Plan

Add a delete action with confirmation in the owner settings panel and/or dashboard row actions. Add an owner-visible include-deleted or restore path so owners can find and restore their own owner-deleted skills. Update focused tests for SkillOwnershipPanel, dashboard pagination/visibility, undelete gate behavior, and HTTP/CLI regressions as needed.
