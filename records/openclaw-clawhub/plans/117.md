# Coding Plan for openclaw/clawhub#117: Cannot delete own skills

Source report: [records/openclaw-clawhub/items/117.md](records/openclaw-clawhub/items/117.md)

## Summary

Keep this issue open. Current main supports owner soft-delete/restore through backend, HTTP API, CLI, docs, and tests, but the signed-in web app still lacks a discoverable owner delete/restore workflow and the dashboard filters deleted skills before owners can restore them.

## Plan

Implement a signed-in ClawHub web flow that lets owners soft-delete and restore their own published skills using the existing soft-delete semantics. Add server-side mutations or helpers that derive the actor from Convex auth, do not accept userId from the client, and reuse/preserve the current setSkillSoftDeletedInternal authorization and owner-undelete gates. Add destructive confirmation UI in SkillOwnershipPanel and/or dashboard row actions, and add an owner-visible include-deleted dashboard/settings path so owner-deleted skills can be found and restored. Preserve soft-delete behavior, 30-day slug reservation messaging, and all moderation/scanner/ban/merge/security-redaction protections; do not add hard-delete for normal owners. Update focused tests for the settings/dashboard UI and any new Convex query/mutation paths, and update specs/docs only if the web behavior changes user-facing expectations.
