## Summary

Keep open. Current main supports owner soft-delete/restore through CLI and API, but the signed-in web app still exposes only rename/merge/settings-style skill owner controls and filters soft-deleted skills out of the dashboard, so the self-service website workflow remains unfinished.

## Plan

Implement owner-facing web soft-delete and restore for skills. Reuse the existing soft-delete semantics in convex/skills.ts, derive the acting user server-side, and do not add normal-owner hard delete. Add a public Convex mutation or wrapper only as needed for signed-in web users; preserve the current gates that prevent owners from restoring moderator-hidden, merged, banned, security-redacted, or malicious skills. Add delete confirmation UI in SkillOwnershipPanel and/or dashboard skill row actions, and add a way for owners to see and restore eligible soft-deleted skills since listDashboardPaginated currently filters them out. Keep publisher membership semantics consistent with dashboard ownership, update focused specs if the intended security behavior changes, and avoid unrelated dashboard redesign.
