Source report: [records/openclaw-clawhub/items/2132.md](records/openclaw-clawhub/items/2132.md)

## Summary

Keep open: current main still defaults the dashboard to the personal publisher and does not render a control that can switch to an org publisher, while the backend and spec already support owner-publisher scoped dashboard reads.

## Plan

Fix `/dashboard` so users who belong to both a personal publisher and org publishers can view org-owned dashboard content. Add a visible publisher selector when `api.publishers.listMine` returns more than one publisher; keep personal as the default, but allow selecting an org. Use the selected publisher id/handle for skills, packages, empty-state copy, and dashboard quick actions. `New Plugin` should preserve the selected owner via `ownerHandle`; `New Skill` should also prefill the selected owner if the publish route needs a small search-param extension. Do not change backend authorization semantics or package/skill ownership rules. Add or extend `src/routes/-dashboard.test.tsx` for the personal-plus-org case, including that switching to the org sends `ownerPublisherId` and surfaces org packages. Keep the repair scoped; changelog handling is maintainer-owned unless the repo has an existing ClawHub changelog entry point for this user-facing fix.
