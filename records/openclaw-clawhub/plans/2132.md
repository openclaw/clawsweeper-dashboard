Source report: [records/openclaw-clawhub/items/2132.md](records/openclaw-clawhub/items/2132.md)

## Summary

Keep open. Current main still has an actionable dashboard UI gap: organization-owned package listing is supported after a publisher is selected, but the dashboard auto-selects the personal publisher and exposes no visible way to switch to an organization publisher.

## Plan

Fix the dashboard so users who belong to multiple publishers can view organization-owned packages and skills. Add a visible publisher selector or equivalent grouped dashboard UI using `api.publishers.listMine`; preserve the personal publisher as the default but allow switching to organization publishers. Ensure package and skill queries use the selected `ownerPublisherId`, make supported `New Plugin` and `New Skill` actions carry the selected owner context, and show the selected handle clearly in empty states. Add or update regression coverage for a user with both personal and organization publishers. Do not change package search/indexing semantics, Convex membership authorization, or unrelated dashboard row actions. Leave `CHANGELOG.md` untouched unless a maintainer explicitly asks for release-note work.
