## Summary

Keep open: current main still defaults the dashboard to the personal publisher and exposes no rendered control to switch to an organization publisher, while the backend already supports membership-checked org dashboard reads.

## Plan

Fix the /dashboard org-publisher visibility bug in openclaw/clawhub. Current main defaults selectedPublisherId to the personal publisher and renders no control to switch, so users who belong to an org publisher cannot see org-owned packages/skills from the dashboard. Add a visible publisher selector using api.publishers.listMine that is reachable even in the empty dashboard state, defaults to the personal publisher, and updates selectedPublisherId so both api.skills.listDashboardPaginated and api.packages.list use the selected ownerPublisherId. Keep the UI scoped to dashboard/publish-owner behavior; do not change backend authorization rules or ownership-transfer flows. Make New Plugin use the selected ownerHandle as it does today, and extend New Skill routing/search initialization if needed so quick actions publish under the selected owner. Add/update tests for a user with both personal and org publishers where the personal dashboard is empty but the org has packages/skills, including switching the selector and verifying query args/action owner handle. Update specs only if the intended dashboard selector behavior changes; otherwise no docs change is required.
