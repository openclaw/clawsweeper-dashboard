# Coding Plan for openclaw/clawhub#2132: Dashboard does not expose org publisher packages after transfer/publish

Source report: [records/openclaw-clawhub/items/2132.md](records/openclaw-clawhub/items/2132.md)

## Summary

Keep open. Current main has backend support for listing org-owned dashboard packages when an org publisher is selected, but the dashboard UI auto-selects the personal publisher and exposes no control that can change the selection, so org packages can remain hidden from a member’s dashboard.

## Plan

Fix https://github.com/openclaw/clawhub/issues/2132 by adding a visible `/dashboard` publisher selector for users whose `api.publishers.listMine` result has more than one publisher. Keep the backend authorization and package ownership logic unchanged. The selector should default to the personal publisher, allow switching to org publishers where the actor is a member, and make the skills/packages dashboard queries use the selected publisher id. Update empty states so they clearly show the selected owner handle. Make dashboard quick actions preserve the selected owner where supported: plugin publish already accepts `ownerHandle`; if skill publish needs an `ownerHandle` search param for the dashboard action, add it narrowly and preserve existing update-mode behavior. Add focused tests for a user with personal plus org publishers proving the org package appears after selection and the personal dashboard remains the default. Do not change package transfer semantics, package name scoping, publisher membership authorization, or public search behavior.
