## Summary

Keep open: current main still has backend support for org-owned dashboard package and skill reads, but the dashboard UI defaults to the personal publisher and exposes no visible publisher switcher, so organization-owned packages can remain unreachable from the page.

## Plan

Fix the dashboard org-publisher visibility bug for users who belong to both a personal publisher and one or more organization publishers. In src/routes/dashboard.tsx, add a visible publisher selector when api.publishers.listMine returns multiple publishers, keep packages and skills queries scoped to the selected publisher id, and make New Plugin/New Skill actions use the selected publisher's handle. Preserve the existing backend membership checks in convex/packages.ts and convex/skills.ts; do not add a new config option or broaden package visibility. Add or update dashboard tests for a personal-plus-org publisher user verifying the selector, org package/skill query arguments, empty-state owner label, and publish-action owner routing. Update specs only if intended behavior changes, and do not add a changelog entry unless maintainers ask for one.
