## Summary

Keep open. Current main still supports org-scoped dashboard reads in the backend, but the Dashboard UI auto-selects the personal publisher and exposes no visible control to switch to an organization publisher, matching the reported failure mode.

## Plan

Fix the Dashboard org-publisher visibility bug reported in https://github.com/openclaw/clawhub/issues/2132. On `/dashboard`, add a visible publisher selector when `api.publishers.listMine` returns more than one publisher, default safely to the personal publisher, and use the selected publisher id for both dashboard skills and packages. Make empty states and creation actions show or carry the selected owner where the existing routes support an owner handle. Do not change package search, package ownership semantics, or org membership authorization. Add focused regression coverage for a user with both a personal publisher and an organization publisher showing that switching exposes org-owned packages and skills. Update specs only if the intended dashboard owner-selection behavior changes beyond the existing org migration intent.
