## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has a source-level path where clean installed/downloaded content can fail registry fingerprint resolution because local update hashing includes generated root metadata that publish/resolve fingerprints do not include.

## Plan

Fix this issue and the related update-fingerprint cluster (#178, #416, #937). On current main, `clawhub update` hashes extracted local text files, skill downloads inject/extract generated root `_meta.json`, and `/api/v1/resolve` compares against published version fingerprints that do not include that generated file. Define one canonical skill-content fingerprint contract shared by publish, download, install, update, sync, and resolve so a fresh install or force-update immediately reports up to date, while real local edits still block non-forced updates. Do not broadly ignore all JSON files, do not bypass registry resolve, and do not weaken malware/suspicious update checks. Add focused regression coverage for generated metadata exclusion/alignment and for real local edits still producing a no-match refusal.
