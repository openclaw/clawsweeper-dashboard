## Summary

Keep open. This pass does not auto-close ClawHub issues, and current main still has a source-confirmed false-positive path where downloaded skill archives add root `_meta.json`, local update hashing includes root JSON files, and registry resolution compares against published file fingerprints that do not include that generated metadata.

## Plan

Fix the false `clawhub update` local-change warning after clean install or force update. Define a single canonical fingerprint contract across CLI install/update/sync and backend publish/download/resolve so downloaded installed content resolves to the published version while real local edits still block non-forced update. Investigate generated root `_meta.json`, origin metadata, `listTextFiles`, `hashSkillFiles`, `buildSkillFingerprint`, `hashSkillZip`, `resolveVersionByHash`, publish fingerprinting, and download zip generation. Add regression coverage for a freshly installed or force-updated skill whose downloaded zip includes generated metadata and then updates without `--force`; also cover that a real content edit still reports local changes. Do not broadly ignore all JSON or all metadata-like files, and do not weaken tamper/local-edit detection. Preserve existing CLI output semantics except for the false positive. If the fix is user-facing, include a concise `CHANGELOG.md` fix entry and add or update a focused spec note for the fingerprint contract if the behavior is not already captured.
