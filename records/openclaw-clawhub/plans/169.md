ClawHub policy keeps issues open in this cleanup pass, and current main still has a source-confirmed clean-install mismatch path: downloaded skill archives can add generated root `_meta.json`, local update hashing includes root JSON files, and registry resolution compares against published-version fingerprints that do not include that generated file.

## Plan

Fix the false `clawhub update` local-change warning after a fresh install or force update. Current main downloads skill zips with generated root `_meta.json`, extracts it into the skill directory, hashes root JSON during update, and asks the registry to resolve a fingerprint that was built from a different file set. Define one canonical skill-content fingerprint contract across publish/download/install/update/sync/resolve. Prefer excluding only ClawHub-generated root `_meta.json` from local/update comparison or avoiding its extraction for skill installs; do not broadly ignore all JSON or user-authored metadata. Add regression coverage showing install or force-update followed immediately by normal update resolves as up to date, and that a real authored-file edit still blocks non-forced update.
