## Summary

Keep open. Current main still has a source-reproducible clean-install fingerprint mismatch path: generated root `_meta.json` from downloads is installed and hashed locally, while registry resolve compares against published-version fingerprints that do not include that generated file.

## Plan

Fix the false `local changes (no match)` warning for clean ClawHub installs and force-updates. Current issue: https://github.com/openclaw/clawhub/issues/169. Related context: https://github.com/openclaw/clawhub/issues/178, https://github.com/openclaw/clawhub/issues/416, and https://github.com/openclaw/clawhub/issues/937. Define a shared fingerprint boundary for published skill content across publish, download, install, update, sync, and resolve. Generated ClawHub root metadata such as download-created `_meta.json` must not make a clean install look locally modified, but authored JSON and other text files must still be covered by integrity checks. Prefer a small shared helper or explicit contract over ad hoc ignores. Add regression coverage for a metadata-bearing downloaded archive extracted to a skill directory, update/sync fingerprint resolution, and a real authored local edit still blocking non-forced update. Update `specs/` if the fingerprint contract is clarified.
