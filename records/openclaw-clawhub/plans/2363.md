## Summary

Keep open: current main still routes normal skill publishing through a global slug lookup before the owner-migration guard, while the schema and org spec define skill uniqueness as publisher-scoped. The related broad fix at https://github.com/openclaw/clawhub/pull/2299 closed unmerged, and the merged availability fix at https://github.com/openclaw/clawhub/pull/2179 does not fix the publish write path.

## Plan

Fix skill publish slug scoping for https://github.com/openclaw/clawhub/issues/2363. In `convex/skills.ts` and `convex/lib/skillPublish.ts`, make ordinary `insertVersion` and publish preflight target resolution use `(ownerPublisherId, slug)` via `by_owner_publisher_slug` instead of treating `slug` as globally unique. Preserve explicit `migrateOwner:true` behavior and all source/destination authority checks for real ownership migrations. Audit slug alias handling touched by this publish path so same-slug aliases from another publisher do not incorrectly block a scoped new publish unless legacy/global alias semantics require an ambiguity-safe response. Add focused regression coverage for a same local slug under two different publishers and for migration still requiring explicit intent and authorization. Do not broaden slug validation, publisher role policy, or unrelated unscoped read/management APIs.
