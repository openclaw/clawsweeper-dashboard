## Summary

Keep open: current main still has a source-reproducible CLI path where relative skill publish paths are resolved under the resolved global workdir, so a configured Clawdbot/OpenClaw workspace can make `skill publish .` scan the wrong folder and throw `SKILL.md required`. The 0.12.3 broad-ignore manifest recovery is present, but it does not address this relative-path/workdir behavior.

## Plan

Fix the remaining `clawhub skill publish .` / legacy `clawhub publish .` bug where a relative skill source path is resolved against the configured/global workdir instead of the caller's cwd. Implement the smallest CLI change so an existing relative skill folder from the process cwd, especially `.`, is used for skill publish when it contains the requested folder or root `SKILL.md`/`SKILLS.md`; preserve absolute paths and existing configured-workdir behavior for install/update/sync and package-publish flows. Add regression coverage for a cwd skill folder with a different configured Clawdbot/OpenClaw workspace that would otherwise throw `SKILL.md required`. Do not broaden ignore handling beyond the existing root-manifest recovery. Add a user-facing changelog entry if the project policy requires it for this CLI fix.
