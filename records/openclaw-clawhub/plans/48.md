# Coding Plan for openclaw/clawhub#48: publish: 'SKILL.md required' error despite file existing

Source report: [records/openclaw-clawhub/items/48.md](records/openclaw-clawhub/items/48.md)

## Summary

Keep open. Current main still has a source-reproducible CLI path where `skill publish .` can inspect the resolved global workdir instead of the caller's skill folder, and the related open report at https://github.com/openclaw/clawhub/issues/1509 describes the same user-visible failure.

## Plan

Fix the ClawHub CLI skill publish path-resolution bug reported in https://github.com/openclaw/clawhub/issues/48 and https://github.com/openclaw/clawhub/issues/1509. Current `cmdPublish` resolves `folderArg` against `opts.workdir`; when `resolveWorkdir()` returns a configured Clawdbot/OpenClaw workspace different from `process.cwd()`, running `clawhub skill publish .` inside a skill folder can inspect the wrong directory and throw `SKILL.md required`. Implement a publish-specific source resolver that preserves absolute paths, prefers the caller's cwd-relative path when a relative publish path resolves to a directory containing root `SKILL.md` or `SKILLS.md`, and avoids changing install/update/sync global-workdir semantics. Keep the existing broad-ignore root-manifest recovery intact. Add focused regression coverage for a cwd skill folder with a different configured/global workdir, plus any needed nonregression coverage for explicit `--workdir`/env workdir precedence. Add an Unreleased Fixes changelog entry for the user-facing CLI fix. Do not change backend publish validation or package/plugin publish behavior.
