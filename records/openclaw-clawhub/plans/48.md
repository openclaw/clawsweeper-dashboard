## Plan

Fix the remaining CLI skill-publish path-resolution bug tracked by https://github.com/openclaw/clawhub/issues/48 and https://github.com/openclaw/clawhub/issues/1509. Current `cmdPublish` resolves relative folder arguments with `resolve(opts.workdir, folderArg)`, so `clawhub skill publish .` can inspect a configured Clawdbot/OpenClaw workspace instead of the caller's skill directory and then throw `SKILL.md required`. Implement a conservative publish-specific resolution path: preserve absolute paths and valid explicit/configured workdir paths, but when a relative argument would miss a root `SKILL.md`/`SKILLS.md` while the cwd-resolved candidate is a valid skill folder, publish that cwd candidate. Do not broadly change install/update/sync/global workdir behavior, backend validation, plugin publish behavior, or ignore semantics; keep the existing broad-ignore manifest recovery. Add regression coverage for a cwd skill folder with a different configured/global workdir, and add an Unreleased Fixes changelog entry if this repair lane owns release notes.
