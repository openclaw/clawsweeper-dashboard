## Summary

Keep open: this is an issue, and the ClawHub cleanup policy only auto-closes PRs. Current main still has a source-reproducible path where `skill publish .` can resolve against a configured global workdir instead of the caller's skill folder, which can produce the reported `SKILL.md required` error.

## Plan

Fix the ClawHub CLI publish path resolution bug reported in https://github.com/openclaw/clawhub/issues/48 and related https://github.com/openclaw/clawhub/issues/1509. `clawhub skill publish .` and the legacy `clawhub publish .` should treat user-entered relative source paths as relative to the caller's current directory when that path resolves to a skill folder containing `SKILL.md`, while preserving absolute paths and existing configured `--workdir` / `CLAWHUB_WORKDIR` behavior for install, update, sync, and other commands. Add regression coverage for a cwd skill folder with a different configured Clawdbot/OpenClaw default workspace. Do not change backend publish validation, package publish behavior, or broad workdir semantics outside the publish command unless the tests prove it is necessary.
