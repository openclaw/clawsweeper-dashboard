# Coding Plan for openclaw/clawhub#2013: ClawScan package review prompt omits declared openclaw.environment env/config metadata

Source report: [records/openclaw-clawhub/items/2013.md](records/openclaw-clawhub/items/2013.md)

## Summary

Keep open: current main still derives package `environment:declared` capability tags from `package.json` `openclaw.environment`, but package ClawScan evaluation does not pass that env/config metadata into the prompt fields that render requirements. The related implementation in https://github.com/openclaw/clawhub/pull/2009 was closed unmerged, and the adjacent open ClawScan PR does not close this issue.

## Plan

Fix package ClawScan prompt context so package releases with declared `package.json` `openclaw.environment` metadata do not render env vars/config paths as `none`. Implement a sanitized allowlisted mapper from `release.extractedPackageJson.openclaw.environment` into the existing `metadata.openclaw`/requirements shape consumed by `assembleEvalUserMessage`; include optional env var declarations, required env vars where represented, config paths, and existing prompt-compatible credential context only when safely normalized. Do not spread the full untrusted `openclaw` object into prompt metadata, and preserve existing requirements fields instead of overwriting them. Add focused regression tests for package LLM evaluation and prompt rendering, including malformed unrelated `openclaw` fields. Update a focused spec note if the repair clarifies scanner intent around package environment metadata.
