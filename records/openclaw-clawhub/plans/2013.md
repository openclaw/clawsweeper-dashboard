# Coding Plan for openclaw/clawhub#2013: ClawScan package review prompt omits declared openclaw.environment env/config metadata

Source report: [records/openclaw-clawhub/items/2013.md](records/openclaw-clawhub/items/2013.md)

## Summary

Keep open: current main still has the reported ClawScan prompt mismatch. Package capability extraction derives `environment:declared` from `package.json` `openclaw.environment`, but package LLM evaluation does not pass that declared env/config metadata into the prompt fields that render requirements; the closed related implementation PR was not merged and remains useful context.

## Plan

Fix the ClawScan package review prompt so package releases with declared `package.json` `openclaw.environment` metadata do not render env vars/config paths as `none`. Add a sanitized mapper, likely in `convex/llmEval.ts` or a small helper, that reads `release.extractedPackageJson.openclaw.environment` and forwards only normalized prompt-relevant fields into `parsed.metadata.openclaw`: required env vars to `requires.env`, optional env declarations to `envVars`, primary credential when safely declared, and config paths to `requires.config`. Preserve any existing recognized requirement fields when valid, ignore malformed or unrelated fields, and do not spread the full untrusted `openclaw` object into prompt metadata. Add regression coverage for required and optional env vars, config paths, malformed unrelated metadata such as non-array `install`, and preservation of existing requirements. Include a `CHANGELOG.md` fix entry because this changes user-visible ClawScan package review behavior. Do not change package capability tag extraction or scanner policy wording beyond the prompt context needed for this bug. Coordinate with https://github.com/openclaw/clawhub/pull/2111 if it lands first because it touches nearby ClawScan files.
