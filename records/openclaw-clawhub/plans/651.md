## Summary

Keep open. ClawHub cleanup policy does not auto-close issues, and current main still has a source-supported path where CLI transport failures in search or login can surface only the raw `fetch failed` message despite the merged proxy and Node 22 fixes.

## Plan

Fix the ClawHub CLI `fetch failed` diagnostics gap tracked by https://github.com/openclaw/clawhub/issues/651 and related https://github.com/openclaw/clawhub/issues/861. Keep request semantics unchanged, but add centralized, redacted transport diagnostics in the shared HTTP/error-formatting path so Node Undici failures, DNS/TLS/certificate/proxy failures, timeouts/aborts, and Bun curl failures produce actionable messages instead of only raw `fetch failed` or `curl failed`. Cover `clawhub search` and token/browser login verification paths. Do not add new config flags or change auth/token verification behavior. Redact Authorization headers, API tokens, token query parameters, callback URLs, proxy credentials, and private endpoint details. Update focused tests and troubleshooting docs if needed.
