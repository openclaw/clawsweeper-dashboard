Current main has proxy support and troubleshooting docs, but `search`, token login, and browser-login token verification can still surface transport failures through the shared formatter as only the top-level `fetch failed` message; ClawHub cleanup policy also keeps issues open rather than auto-closing them.

## Plan

Fix the shared ClawHub CLI transport diagnostics for raw `fetch failed` and curl transport failures. Keep request behavior, retries, rate-limit/status handling, token verification, proxy support, and Bun curl behavior intact. Add a centralized formatter/helper around the existing HTTP/error boundary so DNS, TLS/certificate, proxy, timeout/abort, and curl failures produce actionable, redacted messages for `clawhub search`, token login, browser-login token verification, and other `apiRequest` callers. Redact Authorization headers, API tokens, token query parameters, callback URLs, proxy credentials, and private endpoint details. Update focused tests for `formatError` or the new helper, Node `apiRequest` transport errors, Bun curl stderr behavior if applicable, and search/login command output.
