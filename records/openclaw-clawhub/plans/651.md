## Summary

Keep open. Current main still sends CLI search and token-login transport failures through a generic formatter that can print only `fetch failed`, so this remains a narrow actionable CLI diagnostics bug rather than a cleanup-close candidate.

## Plan

Fix the ClawHub CLI diagnostics bug reported in https://github.com/openclaw/clawhub/issues/651 and the related open report https://github.com/openclaw/clawhub/issues/861. Add centralized, redacted transport-error formatting in the shared CLI HTTP/error boundary so Node fetch failures, timeout/abort failures, DNS/TLS/proxy failures, and Bun curl non-zero exits produce actionable guidance instead of only `fetch failed` or raw stderr. Preserve existing request semantics, HTTP status/rate-limit behavior, retry behavior, proxy support from https://github.com/openclaw/clawhub/pull/363, and successful search/login flows. Add focused regression coverage for search and token-login error output plus shared HTTP/formatter behavior. Redact Authorization headers, API tokens, callback URL tokens, query/hash token values, proxy credentials, and private endpoint details. Add a short CHANGELOG.md fix entry for user-facing CLI error output unless maintainers plan to handle release notes separately.
