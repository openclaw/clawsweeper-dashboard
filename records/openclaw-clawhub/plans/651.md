## Plan

Fix ClawHub CLI raw transport diagnostics for search/login failures like this issue. On current main, `cmdSearch` and `cmdLogin` catch `apiRequest` failures and display `formatError(error)`, while `formatError` returns only the top-level `Error.message`; `fetchWithTimeout` rethrows Undici `Error` objects unchanged and Bun curl paths throw raw stderr or `curl failed`. Add centralized, redacted diagnostics in the shared HTTP/UI boundary for DNS, TLS/certificate, proxy, timeout/abort, curl, and nested Undici cause failures. Preserve successful request behavior, HTTP status/rate-limit messages, retries, timeout durations, Bun curl behavior, and `/api/v1/whoami` token verification. Do not add config flags or broaden product behavior. Add focused regression coverage for Node and Bun transport failures plus search/login formatting, and update troubleshooting docs if the new guidance changes user-facing diagnostics.
