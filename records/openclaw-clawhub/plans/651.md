## Summary

Keep open. Current main still has a source-supported path where CLI search and login transport failures surface as the raw `fetch failed` message, and ClawHub cleanup policy keeps issues open rather than auto-closing them.

## Plan

Fix the ClawHub CLI raw `fetch failed` diagnostics path for search/login transport failures. Add centralized, redacted diagnostics at the shared HTTP/error-formatting boundary so DNS, TLS/certificate, proxy, timeout/abort, Undici, and curl failures produce actionable messages while preserving existing HTTP status errors, retries, rate-limit hints, timeout semantics, and Bun curl behavior. Cover both `clawhub search` and login token verification paths. Do not bypass `/api/v1/whoami` verification, do not add a new config option, and do not assume every failure is a proxy problem. Redact Authorization headers, API tokens, token query parameters, browser callback URLs, proxy credentials, and environment-derived secrets. Update focused troubleshooting docs and add a user-facing changelog entry if the repo’s release convention expects one for CLI fixes.
