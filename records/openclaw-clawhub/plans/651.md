## Summary

Keep open. Current main still allows shared CLI transport failures to surface as raw `fetch failed` messages in search and login paths, and ClawHub cleanup policy keeps issues open rather than auto-closing them; the related login report at https://github.com/openclaw/clawhub/issues/861 is also still open.

## Plan

Fix the ClawHub CLI raw `fetch failed` diagnostics path without changing registry semantics. Add centralized, redacted transport-error classification at the shared HTTP/error-formatting boundary so DNS, TLS/certificate, proxy, timeout/abort, Undici, and curl failures produce actionable guidance for commands such as search and login. Preserve `/api/v1/whoami` token verification, retries, HTTP status/rate-limit handling, successful requests, and Bun curl behavior. Do not add a new config option and do not assume every failure is proxy-related. Redact Authorization headers, API tokens, callback URLs, proxy credentials, and credentials embedded in URLs or environment variables. Add focused regression coverage for search/login or shared formatter behavior, and update troubleshooting docs if user-facing guidance changes.
