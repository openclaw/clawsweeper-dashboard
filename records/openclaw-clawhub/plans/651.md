# Coding Plan for openclaw/clawhub#651: why

Source report: [records/openclaw-clawhub/items/651.md](records/openclaw-clawhub/items/651.md)

## Summary

Keep open. Current main still routes CLI search and token-login transport failures through a raw `Error.message` formatter, so users can still see only `fetch failed`; the merged proxy support covers one root cause but not the remaining diagnostics gap.

## Plan

Fix the ClawHub CLI diagnostics gap where search/login transport failures surface only as `fetch failed`. Add centralized, redacted transport-error classification in the shared HTTP/error-formatting boundary used by `cmdSearch`, token login, and other network commands. Cover Undici `TypeError('fetch failed')` with causes, DNS/TLS/certificate/proxy/timeout/abort cases where detectable, and Bun curl non-zero stderr/`curl failed`; preserve existing HTTP status handling, retries, rate-limit hints, proxy env support, auth persistence, request semantics, and API routes. Redact Authorization headers, API tokens, callback URL tokens, token query/hash values, proxy credentials, and private endpoints. Update focused tests for `packages/clawhub/src/http.ts`, `packages/clawhub/src/cli/ui.ts`, and search/login command behavior; update troubleshooting docs only if the user-facing guidance changes.
