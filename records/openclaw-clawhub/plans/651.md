## Summary

Keep open: current main still has a source-supported CLI path where search and token-login transport failures can surface as only `fetch failed`, and ClawHub cleanup policy does not auto-close issues.

## Plan

Fix the ClawHub CLI transport-diagnostics bug reported at https://github.com/openclaw/clawhub/issues/651 and related https://github.com/openclaw/clawhub/issues/861. Add centralized, redacted diagnostics at the shared HTTP/error-formatting boundary so `clawhub search`, browser/token login verification, and other network commands do not surface only raw `fetch failed` for DNS, TLS/certificate, proxy, timeout/abort, Undici, or Bun curl transport failures. Preserve existing request semantics, `/api/v1/whoami` token verification, retries, rate-limit/status errors, proxy support, and Bun curl behavior. Add focused regression coverage for Node fetch errors with nested causes, non-Error throws, curl stderr/nonzero exits, CLI formatter output, and the search/login command paths. Update troubleshooting docs only for current behavior and do not add a new config option or bypass auth verification.
