## Summary

Keep open: current main still has a source-supported CLI path where search and token-login transport failures can surface as only `fetch failed`, and ClawHub cleanup policy does not auto-close issues.

## Plan

Fix the ClawHub CLI diagnostic gap where search and login-style network commands can print only raw `fetch failed`. Add centralized, redacted transport-error formatting at the shared HTTP/CLI error boundary so DNS, TLS/certificate, proxy, timeout/abort, and Bun curl failures produce actionable guidance. Preserve existing request semantics, `/api/v1/whoami` token verification before config writes, HTTP status/rate-limit messages, retry behavior, proxy support, and Bun curl behavior. Cover the related issue cluster at https://github.com/openclaw/clawhub/issues/651 and https://github.com/openclaw/clawhub/issues/861. Add focused regression tests for nested Undici-style causes, curl stderr/nonzero exits, timeout/abort cases, token/login output, and redaction of Authorization headers, API tokens, callback token/state values, proxy credentials, and private endpoints. Update troubleshooting/CLI docs only as needed for the new diagnostics; do not add a new config option or bypass auth verification.
