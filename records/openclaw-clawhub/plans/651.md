Keep open. ClawHub cleanup policy keeps issues open, and current main still has a valid CLI diagnostics gap where search and login can show raw transport failures as `fetch failed` instead of actionable network guidance.

## Plan

Add centralized, redacted classification in the shared HTTP/error-formatting boundary so Undici fetch failures, DNS/TLS/certificate errors, proxy failures, timeouts/aborts, and Bun curl stderr produce actionable messages for commands such as search and login while preserving existing request semantics, HTTP status handling, retries/rate-limit hints, timeout behavior, and proxy support from https://github.com/openclaw/clawhub/pull/363.
