# Coding Plan for openclaw/clawhub#1509: Bug: clawhub publish fails with 'SKILL.md required' for new skills

Source report: [records/openclaw-clawhub/items/1509.md](records/openclaw-clawhub/items/1509.md)

## Summary

Keep open. ClawHub issue cleanup is not allowed to auto-close open issues, and current main still has a source-reproducible publish path where `publish .` can resolve against a configured global workdir instead of the caller's skill folder.

## Plan

Fix the remaining `clawhub skill publish .` / legacy `clawhub publish .` source-path bug for https://github.com/openclaw/clawhub/issues/1509 and related https://github.com/openclaw/clawhub/issues/48. Current main resolves the publish folder with `resolve(opts.workdir, folderArg)`, so when `resolveWorkdir` falls back to a configured Clawdbot/OpenClaw workspace, a user running publish from a skill folder containing `SKILL.md` can scan the wrong folder and get `SKILL.md required`. Implement a focused CLI fix that treats user-entered relative publish source paths as relative to the caller's cwd when that resolves to an existing skill folder, while preserving absolute paths and intentional configured `--workdir` / `CLAWHUB_WORKDIR` flows for install, update, sync, and package publish. Add regression coverage for a cwd skill folder with a different configured/global workdir, keep the existing broad-ignore manifest recovery behavior, and add an Unreleased Fixes changelog entry. Stop and ask for maintainer guidance if the implementation would require a new flag, broad global workdir semantics change, or backend publish-policy change.
