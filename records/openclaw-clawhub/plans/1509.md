# Coding Plan for openclaw/clawhub#1509: Bug: clawhub publish fails with 'SKILL.md required' for new skills

## Summary

Keep open. ClawHub issue cleanup is not allowed to auto-close open issues, and current main still has a source-reproducible publish path where `publish .` can resolve against a configured global workdir instead of the caller's skill folder.

## Plan

Fix the ClawHub CLI skill publish path resolution bug tracked by https://github.com/openclaw/clawhub/issues/1509 and https://github.com/openclaw/clawhub/issues/48. On current main, `cmdPublish` resolves relative source paths with `resolve(opts.workdir, folderArg)`, so `clawhub skill publish .` can scan a configured Clawdbot/OpenClaw workspace instead of the caller's skill directory and then fail with `SKILL.md required`. Implement the narrowest publish-only resolution change so user-entered relative source paths resolve to the caller's cwd when that identifies the requested skill folder, while preserving absolute paths and intentional configured `--workdir`/`CLAWHUB_WORKDIR` behavior for install/update/sync and explicit publish paths. Do not change backend `SKILL.md` validation or the existing broad-ignore root manifest recovery. Add focused regression coverage for a cwd skill folder with `SKILL.md` and a different configured/global workdir, and update changelog/docs only if the repo policy expects it for this user-facing CLI fix.
