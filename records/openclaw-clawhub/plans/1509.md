# Coding Plan for openclaw/clawhub#1509: Bug: clawhub publish fails with 'SKILL.md required' for new skills

Source report: [records/openclaw-clawhub/items/1509.md](records/openclaw-clawhub/items/1509.md)

## Summary

Keep open. ClawHub cleanup policy does not auto-close issues, and current main still has a source-reproducible `skill publish .` path where the CLI can inspect a configured workspace instead of the folder containing `SKILL.md`.

## Plan

Fix the CLI skill publish path that can produce `SKILL.md required` even when the caller's current directory contains `SKILL.md`. On current main, `cmdPublish` resolves `folderArg` with `resolve(opts.workdir, folderArg)`; when `resolveWorkdir` selected a configured Clawdbot/OpenClaw workspace different from cwd, `clawhub skill publish .` can inspect the wrong folder. Implement publish-specific relative source resolution: keep absolute paths working, prefer the caller's cwd-relative path when that resolves to the intended skill folder/root manifest, and preserve explicit `--workdir`/env workdir behavior for non-publish install/update/sync flows. Keep `ensureRootManifestFile`, plugin-folder detection, backend publish validation, payload shape, and package publish behavior unchanged. Add focused regression coverage for a cwd skill folder with a different configured/global workdir. Add or update an Unreleased Fixes changelog entry if the current tree still has that convention.
