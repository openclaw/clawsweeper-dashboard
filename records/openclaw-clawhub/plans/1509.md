# Coding Plan for openclaw/clawhub#1509: Bug: clawhub publish fails with 'SKILL.md required' for new skills

Source report: [records/openclaw-clawhub/items/1509.md](records/openclaw-clawhub/items/1509.md)

## Summary

Keep open. The broad-ignore manifest omission is fixed on current main and noted in v0.12.3, but current main still has a source-reproducible `publish .` path where a configured global workdir can differ from the caller's skill folder, and related https://github.com/openclaw/clawhub/issues/48 remains open.

## Plan

Fix the remaining `clawhub skill publish .` / `SKILL.md required` path where `cmdPublish` resolves a user-entered relative source path under `opts.workdir` even when the caller's cwd is the skill folder. Preserve absolute path behavior and intentional configured-workdir behavior for install/update/sync and explicit publish paths. Prefer a narrow helper in the publish command path that uses the cwd-resolved relative path when it exists and looks like a skill folder with root `SKILL.md` or `SKILLS.md`; do not change backend publish validation or broad ignore semantics. Add regression coverage for a cwd skill folder with a different configured/global workdir, ensure the existing broad-ignore manifest recovery test remains valid, and add a `CHANGELOG.md` Fixes entry. Related refs: this issue, https://github.com/openclaw/clawhub/issues/48, and superseded https://github.com/openclaw/clawhub/pull/1856.
