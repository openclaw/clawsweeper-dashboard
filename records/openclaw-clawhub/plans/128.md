# Coding Plan for openclaw/clawhub#128: Feature Request: Add --dry-run flag to update command

Source report: [records/openclaw-clawhub/items/128.md](records/openclaw-clawhub/items/128.md)

## Summary

Keep open. Current main still lacks `clawhub update --dry-run`; the update command only exposes the existing mutating flags, and `cmdUpdate` still reaches directory replacement, origin metadata writes, lockfile mutation, and lockfile persistence.

## Plan

Implement `clawhub update --dry-run` for this issue. Add the Commander flag in `packages/clawhub/src/cli.ts`, plumb it into `cmdUpdate`, and add a preview-only path in `packages/clawhub/src/cli/commands/skills.ts` that reuses existing registry lookup, moderation, fingerprint resolution, semver, local-change, and pin checks. The dry-run path must report would-update/up-to-date/skipped results for single-skill and `--all` modes and must not call `rm`, `downloadZip`, `extractZipToDir`, `writeSkillOrigin`, or `writeLockfile`. Preserve existing mutating update behavior, suspicious/malware handling, pinned-skill rules, and `--version` validation. Add focused tests in `packages/clawhub/src/cli/commands/skills.test.ts`, document the flag in `docs/cli.md`, and add release/changelog notes only if the current repo convention requires one for user-facing CLI features. Do not change package publish dry-run or sync dry-run behavior.
