## Summary

Keep open. Current main still treats user-initiated account deletion as terminal and rejects the deleted GitHub OAuth identity, while the remaining request needs production support, identity verification, and account-lifecycle security/product judgment rather than cleanup closure.

## What This Changes

The issue requests restoring, purging, or unbinding a GitHub OAuth-bound ClawHub account after deletion so the user can register again and receive an active session.

## Best Possible Solution

Maintainers should choose the deleted-account lifecycle policy, then either add an audited recovery/re-registration or unlink path with support tooling and tests, or keep deletion terminal while making the support route explicit.

## Reproduction Assessment

Yes for the source-level path: current main deactivates deleted users, rejects them during OAuth completion, and excludes them from active sessions. I did not verify the reporter's production account rows.

## Solution Assessment

No for the requested one-off purge as a general repository fix. The maintainable path needs an explicit recovery or terminal-deletion policy covering identity proof, privacy, ban separation, auditability, and operator behavior.
