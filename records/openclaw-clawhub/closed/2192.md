## Summary

Keep open: current main still has the direct Blacksmith Testbox surface and no Crabbox/UI-proof commands, so this PR is not obsolete. The branch has real linked proof artifacts, but `proof:publish` introduces a token-handling issue that should be fixed before merge.

## What This Changes

## Best Possible Solution

Keep the Crabbox/UI-proof direction, but change artifact publishing to authenticate without storing or passing token-bearing remote URLs, then rerun the targeted publisher tests and static gate.

## Solution Assessment
