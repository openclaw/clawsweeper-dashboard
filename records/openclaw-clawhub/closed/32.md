## Summary

Keep open. Current main deliberately blocks deleted/deactivated users during OAuth and now surfaces a deleted-account error, but the discussion and related reports still need a maintainer product/security decision on terminal deletion versus recovery or account recreation.

## What This Changes

Users who self-deleted a ClawHub account cannot sign back in or re-register with the same GitHub OAuth identity.

## Best Possible Solution

Maintainers should choose the deleted-account lifecycle, then either keep terminal deletion with clearer OAuth/support guidance or implement scoped same-provider recovery/account recreation with ban, purge, audit, and identity safeguards.

## Reproduction Assessment

Yes, at source level: deleteAccount writes deactivatedAt/purgedAt, OAuth calls handleDeletedUserSignIn, and tests assert deleted users are rejected. I did not run a live production OAuth flow or inspect affected account rows.

## Solution Assessment

Unclear until maintainers choose the deletion lifecycle policy. Current permanent deletion is internally consistent, but recovery or re-registration would need an explicit security-reviewed product path.
