## Summary

Keep open; this new feature PR is not implemented on main and is not merge-ready because the backend admin persona provider is gated only by an env var plus a non-production deployment string, and after-fix real behavior proof is absent.

## What This Changes

The PR adds a local-dev persona switcher UI, wires it into app providers, and adds a Convex credentials provider/upsert path for seeded owner, user, and admin personas.

## Best Possible Solution

Land a revised version that keeps the UI localhost-only, constrains the credentials provider to truly local Convex backends, documents the required env flags, and includes redacted runtime proof.

## Reproduction Assessment

Yes for the review finding: source inspection of the PR diff shows the provider accepts the admin persona whenever `DEV_AUTH_ENABLED=1` and the deployment string is not production-like. I did not run a live Convex backend.

## Solution Assessment

No. The focused extraction is narrower than the earlier broad branch, but the backend gate needs to be local-only before this is a maintainable implementation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Restrict dev persona auth to local backends:** `convex/auth.ts:18-21`
  - body: The backend provider only checks `DEV_AUTH_ENABLED=1` and that `CONVEX_DEPLOYMENT` is not prod/production before accepting the admin persona.
