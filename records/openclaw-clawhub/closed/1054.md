## Summary

Keep open. Current main intentionally blocks reauthentication for permanently deleted accounts, and ClawHub policy keeps issues open unless a maintainer resolves the account-lifecycle decision or support case.

## What This Changes

The issue asks ClawHub to create a usable GitHub OAuth session after a user has deleted their account.

## Best Possible Solution

Maintainers should choose and document the post-deletion lifecycle: keep permanent blocking with clear support UX, provide a verified administrative purge path, or implement an explicit re-registration flow with tests and spec updates.

## Reproduction Assessment

Yes, by source inspection. Deleting an account writes deactivatedAt/purgedAt, OAuth reauth rejects through handleDeletedUserSignIn, and users.me returns null through the active-user helper.

## Solution Assessment

No, not as an automatic session-creation fix. The requested behavior conflicts with the current irreversible deletion policy, so the maintainable path needs maintainer account-lifecycle and security judgment first.
