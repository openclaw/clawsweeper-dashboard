## Summary

Keep open. Current main and the latest tagged release still intentionally block GitHub OAuth sign-in for deactivated/deleted accounts, the discussion has multiple same-symptom affected users, and ClawHub issue cleanup policy keeps issues open unless there is a PR-only close reason.

## What This Changes

Request that GitHub OAuth sign-in recover, relink, or recreate a ClawHub account after user-initiated account deletion.

## Best Possible Solution

Define and implement a maintainer-approved deleted-account recovery, unlinking, or re-registration policy across OAuth, deletion, auth account binding, CLI token flow, operator docs/specs, and regression tests.

## Reproduction Assessment

Yes, by source inspection: deleting an account sets deactivatedAt/purgedAt and OAuth re-entry calls handleDeletedUserSignIn, which throws before the web or CLI token flow can see an active user. I did not run a live production OAuth flow or inspect production rows.

## Solution Assessment

No, a narrow OAuth-only recreate patch is not the best path yet. The maintainable solution needs an explicit recovery, unlink, or re-registration policy that preserves identity proof, deletion privacy, token revocation, audit, and banned-account boundaries.
