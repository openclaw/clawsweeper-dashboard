## Summary

Current main still resolves bare OpenClaw plugin aliases before publisher handles, so a `tencent` publisher can be shadowed. The issue should stay open because https://github.com/openclaw/clawhub/pull/2808 is an open linked fix candidate with closing syntax, not a merged implementation.

## What This Changes

The issue reports that a publisher handle matching an OpenClaw plugin alias can route to the plugin page instead of the publisher profile.

## Best Possible Solution

Review and land the linked pull request, or replace it with an equivalent narrow fix that lets existing publisher handles win while preserving OpenClaw plugin aliases when no publisher owns the handle.

## Reproduction Assessment

Yes. Source inspection on current main shows publisher links go through `/$slug`, and the resolver returns an OpenClaw plugin alias before checking whether a publisher owns that handle.

## Solution Assessment

Yes, with maintainer confirmation of the precedence policy. Resolving existing publisher handles before bare plugin aliases is the narrow fix, and the linked pull request preserves aliases when no publisher handle exists.
