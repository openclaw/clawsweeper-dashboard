## Summary

Current main now ships an owner-authenticated scan report download path that gives package owners exact-version scanner details, including static-analysis reason codes and finding objects with code/file/line. The exact `package moderation-status` command remains an aggregate trust summary, but the central false-positive triage problem is covered by the shipped `scan download --kind plugin` and package version JSON surfaces.

## What This Changes

Request to expose scanner rule and file/line detail to package owners so suspicious package verdicts can be triaged without blind iteration.

## Best Possible Solution

Keep the shipped owner-authenticated scan report download and exact-version JSON as the diagnostic surface; leave `package moderation-status` as an aggregate trust summary unless maintainers later want extra discoverability.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms the old moderation-status path is aggregate-only, while current main now provides exact-version scanner detail through scan report download and version JSON.

## Solution Assessment

Yes. The shipped path is the narrow maintainable solution because it gates detail to authenticated owners/publishers or staff and exact versions, while returning the full stored report archive with `static-analysis.json`.
