## Summary

Keep open. Current main still lacks native per-skill pinning, and the open linked implementation PR is the right path to finish before this issue closes.

## What This Changes

Add native per-skill pin/unpin support so update, update --force, and update --all cannot overwrite operator-frozen installed skills.

## Best Possible Solution

Review and land https://github.com/openclaw/clawhub/pull/1806 or an equivalent implementation that persists pin metadata, blocks direct pinned updates, skips pinned entries for update --all, and honors pins under --force.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken documented behavior. Source inspection gives high confidence that native pin/unpin commands, pin metadata, and pin-aware update behavior are absent from current main.

## Solution Assessment

Yes; native CLI and lockfile enforcement is the right solution shape because wrapper-only enforcement cannot cover every cron, script, shell, or agent using clawhub directly. The linked implementation PR is the maintainable path to review first.
