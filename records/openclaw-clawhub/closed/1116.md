## Summary

Keep open. Current main intentionally blocks deleted/deactivated ClawHub accounts during GitHub OAuth and now surfaces a deleted-account message, but the report remains a clustered, user-blocking account recovery request that needs maintainer product and security direction rather than cleanup closure.

## What This Changes

The issue asks for a deleted ClawHub account to be hard-deleted, reset, restored, or re-registerable so the same GitHub identity can sign in again.

## Best Possible Solution

Maintainers should choose and document the deleted-account lifecycle, then implement either an audited recovery/re-registration/admin-reset path or keep deletion irreversible with a clear OAuth error and support workflow.

## Reproduction Assessment

Yes, at source level: deleteAccount writes deactivatedAt/purgedAt, OAuth re-entry calls handleDeletedUserSignIn, and tests assert deleted users are rejected. I did not run a live production OAuth flow or inspect the reporter's database row.

## Solution Assessment

No, the requested hard-delete or automatic recreate path is not clearly the best fix yet because it bypasses unresolved deletion, privacy, audit, token, and ban policy. The maintainable solution needs an explicit account-lifecycle decision plus audited implementation or support tooling.
