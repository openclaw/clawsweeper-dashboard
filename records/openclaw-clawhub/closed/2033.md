## Summary

Keep open: current main does not implement role-aware package help gating, and the latest PR head still needs changes before merge because version flags can trigger role lookup and the user-facing CLI fix lacks a changelog entry.

## What This Changes

The PR adds a ClawHub CLI command-path registry, role-aware help audiences for moderator/admin visibility, and expanded admin help tests.

## Best Possible Solution

Land the role-aware help gate after exempting version flags from role lookup, adding regression coverage, and recording the CLI fix in the Unreleased changelog.

## Reproduction Assessment

Yes from source inspection: current main exposes privileged package commands in normal help, and the PR head’s helper classifies option-only version invocations as help. I did not run the PR branch because this review is read-only.

## Solution Assessment

No as currently patched: the command-audience registry is a reasonable narrow fix, but version flags need a local/offline bypass before merge.

## Review Findings

Overall confidence: 0.88

Full review comments:
