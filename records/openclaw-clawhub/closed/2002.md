## Summary

Keep open. Current main deliberately keeps `latest` semver-managed during publish and ignores caller-provided `latest` tags on lower or prerelease publishes; changing that behavior conflicts with the merged backport guard and needs maintainer product judgment.

## What This Changes

The issue requests that publishing a prerelease or lower semver version with `--tags latest` move the visible `latest` pointer to that just-published version.

## Best Possible Solution

Define the contract explicitly: either document publish-time `latest` as semver-managed and direct rolling publishers to retagging or custom tags, or add a deliberate override path with backend, CLI, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: publish a lower or prerelease semver with `tags: ['latest']`, and `insertVersion` skips that tag while tests assert the previous latest pointer remains selected.

## Solution Assessment

Unclear. The requested behavior is maintainable only after maintainers decide whether publish-time `latest` should override the semver guard from https://github.com/openclaw/clawhub/pull/1832 or whether the existing behavior should be documented more clearly.
