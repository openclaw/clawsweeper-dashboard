## Summary

Keep open. Current main still intentionally blocks GitHub reauth after self-deletion, and changing that behavior requires a maintainer-owned product/security decision rather than cleanup closure.

## What This Changes

A user requests help signing in or re-registering with GitHub after accidentally deleting their ClawHub account.

## Best Possible Solution

Define the account-lifecycle policy, then either keep irreversible deletion with clear support guidance or ship an audited admin/support recovery or unlink flow that preserves deletion, moderation, token, ownership, and audit invariants.

## Reproduction Assessment

Yes, by source inspection: self-service deletion sets deactivatedAt, and the GitHub OAuth callback rejects deactivated users before normal session setup. I did not run a live OAuth flow, so this is source-reproducible rather than live-reproduced.

## Solution Assessment

No for an autonomous fix. The maintainable path first needs a product/security decision on whether recovery, unlinking, clean re-registration, or operator-only handling should exist and how it preserves deletion invariants.
