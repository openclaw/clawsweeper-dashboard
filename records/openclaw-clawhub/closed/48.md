## Summary

Keep open. ClawHub cleanup policy does not auto-close issues, and current main still has a source-reproducible CLI path where `skill publish .` can inspect the resolved global workdir instead of the caller's skill folder.

## What This Changes

The issue reports that `clawhub publish .` fails with `SKILL.md required` even though the target skill directory contains a valid `SKILL.md`.

## Best Possible Solution

Make skill publish resolve local relative source paths to the caller's intended skill folder when that folder contains a root manifest, while preserving absolute paths and non-publish workdir semantics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with a configured Clawdbot/OpenClaw default workspace different from an unmarked cwd, `clawhub skill publish .` can inspect the wrong folder and hit `SKILL.md required` even when the caller's cwd contains `SKILL.md`.

## Solution Assessment

Yes. The narrow maintainable fix is publish-specific source-path resolution plus regression coverage, not backend validation changes or a broad rewrite of global workdir behavior.
