## What This Changes

The PR adds full-text search indexes for skill display names and slugs, queries them from direct skill search alongside prefix indexes, and adds tests for non-first-token recall.

## Best Possible Solution

Keep the shipped current-main implementation and close this superseded branch; any future cleanup should wait until the Convex search indexes are ready and be handled in a focused follow-up.

## Reproduction Assessment

Yes. The original underrecall is source-reproducible from the old prefix-only direct search path and recency-bounded fallback, and current main now has regression tests for `yijian`/`vision` non-first-token recall.

## Solution Assessment

Yes for current main, but no for this branch. The merged implementation is the narrower maintainable path because it keeps the full-text indexes while preserving all-token matching and bounded fanout; this PR branch should not be merged.
