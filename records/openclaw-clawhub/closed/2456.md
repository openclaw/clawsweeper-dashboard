## Summary

Keep open: current main still sends the configured server `GITHUB_TOKEN` on the publish account-age lookup and has no source-level fix for a bad token, while the reported numeric GitHub user lookup works unauthenticated. This is not a cleanup-close candidate because the remaining action is production-secret verification or a deliberate code change to the security-sensitive publish gate.

## What This Changes

The issue reports that CLI and web skill publishing fail with `GitHub account lookup failed` because the Convex GitHub account-age lookup appears to use an expired or invalid server-side `GITHUB_TOKEN`.

## Best Possible Solution

Restore publishing by rotating or removing the broken production token, or land a deliberate, tested lookup-resilience path that preserves the numeric-ID age gate and documents the intended behavior in specs.

## Reproduction Assessment

Yes from source, but not as a live production reproduction in this read-only review. The publish paths call `requireGitHubAccountAge`, which sends `GITHUB_TOKEN` when set and throws `GitHub account lookup failed` for non-OK lookup responses other than 403/429.

## Solution Assessment

Unclear. Refreshing the Convex token may be the fastest operator fix, while adding fallback or alerting behavior in code is plausible but needs maintainer judgment because this is a security-sensitive publish gate.
