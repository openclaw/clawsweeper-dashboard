## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still lacks REST/CLI owner selection for skill publishing; open PR #1840 is the implementation candidate rather than a close reason before merge.

## What This Changes

Allow REST and CLI publishing of new versions for skills owned by an organization publisher.

## Best Possible Solution

Review and land #1840, or an equivalent focused change that threads ownerHandle/--owner through the shared schema, REST JSON and multipart parsing, CLI payload generation, server-side publisher-role authorization, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: REST/CLI publish cannot name an org owner, so publishVersionForUser reaches insertVersion without ownerPublisherId and falls back to the actor's personal publisher before rejecting an existing org-owned slug.

## Solution Assessment

Yes. The narrow maintainable direction is to reuse the web publish action's ownerHandle publisher-resolution path with server-side publisher-role checks, then expose that through the REST and CLI publish surfaces.
