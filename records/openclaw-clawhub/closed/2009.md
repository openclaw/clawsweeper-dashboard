## Summary

Keep open: current main still omits package `openclaw.environment` from ClawScan prompt metadata, and the PR's direction is useful, but the proposed mapper forwards arbitrary package `openclaw` fields and drops existing `requires` data.

## What This Changes

The PR parses package.json `openclaw.environment` during package LLM evaluation, forwards env/config metadata to ClawScan prompt assembly, and adds focused coverage.

## Best Possible Solution

Land a sanitized environment-metadata mapper that forwards only normalized env/config prompt fields, preserves valid existing requirements, and adds regression coverage for malformed unrelated metadata.

## Reproduction Assessment

Yes. By source inspection, current main derives package `environment:declared` tags but does not pass package `openclaw.environment` into the `metadata.openclaw` path that `assembleEvalUserMessage` reads.

## Solution Assessment

No, not as written. The narrow maintainable fix is to construct sanitized prompt metadata for env/config only and merge requirements instead of spreading the full package `openclaw` object.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Sanitize OpenClaw metadata before forwarding it:** `convex/llmEval.ts:75`
  - body: `buildOpenClawMetadataForEval` spreads the full package `openclaw` object into prompt metadata. If a package includes malformed `openclaw.install`, `assembleEvalUserMessage` can treat it as an array and call `.map`, aborting package LLM analysis before an OpenAI request or stored error. Build a sanitized object with only normalized prompt fields, or normalize unrelated fields before forwarding.
  - confidence: 0.9
- **[P2] Preserve existing requirements when adding config paths:** `convex/llmEval.ts:78`
  - body: When `environment.configPaths` is present, this replaces any existing `openclaw.requires` object with `{ config }`. Packages that also declare `requires.env`, `bins`, or `anyBins` lose those prompt signals, so merge config paths into a sanitized requirements object instead.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The diff forwards untrusted package `openclaw` metadata into prompt assembly more broadly than the stated environment fields.

  - body: Spreading the package `openclaw` object forwards unrelated untrusted fields into ClawScan prompt assembly; malformed fields such as `install` can crash evaluation or pollute scanner context. Only validated environment-derived fields should be forwarded.
