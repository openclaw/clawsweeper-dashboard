## Summary

Keep this issue open. Current main has owner soft-delete/restore primitives through the backend, HTTP API, CLI, docs, and tests, but the signed-in web app still lacks a discoverable owner delete/restore flow and the dashboard still filters deleted rows out of the owner view.

## What This Changes

The issue requests that skill authors be able to delete their own published ClawHub skills.

## Best Possible Solution

Add a signed-in web delete/restore workflow that reuses the existing soft-delete helper, derives identity server-side, authorizes owners and publisher admins consistently, and exposes owner-deleted rows without allowing owners to restore moderation/security-hidden content.

## Reproduction Assessment

Yes, as a source-level gap: owner settings renders summary, publisher note, rename, and merge only, while dashboard owner queries use active indexes that exclude soft-deleted rows. I did not run a live browser flow.

## Solution Assessment

Unclear until maintainers decide the web self-service policy. If approved, reusing setSkillSoftDeletedInternal behind a narrow authenticated web mutation is the maintainable path; hard-delete or a second deletion contract would be riskier.
