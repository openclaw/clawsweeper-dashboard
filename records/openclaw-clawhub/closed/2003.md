Keep open. ClawHub cleanup policy keeps issues open, and although current main no longer appears to match ordinary GHCR compose images for this rule, the reported hosted scan still needs operator/security verification before any source-level conclusion.

## What This Changes

The issue asks ClawHub to stop classifying docker-compose skills that pull disclosed public container images as `suspicious.install_untrusted_source`, or to classify that risk more precisely.

## Best Possible Solution

Have a security/moderation owner verify or rescan InvestorClaw under current v2.4.24 policy; if GHCR compose images still produce this rule, add a narrow regression for local compose image references while leaving scanner-detail disclosure to https://github.com/openclaw/clawhub/issues/2109.
