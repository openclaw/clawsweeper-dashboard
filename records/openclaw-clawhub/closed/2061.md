## Summary

Keep open: current main does not implement the requested neutral badge treatment, and this PR needs revision because it neutralizes a shared suspicious-status class used by non-Review security indicators and lacks after-fix real behavior proof.

## What This Changes

This PR changes suspicious security scan badge styling from warning/yellow to neutral/gray in the detail summary and shared scan status CSS while keeping the visible `Review` label.

## Best Possible Solution

Scope the neutral styling to user-facing `Review` verdict badges, preserve warning treatment for concern/finding indicators, and attach real after-fix proof before merge.

## Reproduction Assessment

Yes for the PR regression: source inspection shows the changed `.scan-status-suspicious` class is also used by non-Review concern and static-finding indicators. I did not run the app because this was a read-only review.

## Solution Assessment

No. The safer implementation is to add or reuse a neutral class/Badge variant only for `Review` verdict badges instead of changing the shared suspicious-status class globally.
