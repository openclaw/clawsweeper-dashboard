## Summary

Keep open. This is an issue, and the ClawHub cleanup policy for this pass does not auto-close issues; independently, current main still has a source-level path where an ordinary republish can hide the current skill row during pending scan/review and public search then filters it out.

## What This Changes

The issue reports that republishing an already approved skill removes it from search until the new version completes review and scanning.

## Best Possible Solution

Implement a version-aware public projection that keeps the last approved skill card, digest, file/API surface, and searchable embedding visible while a new version remains gated, then promotes the new projection only after scans and moderation clear it.

## Reproduction Assessment

Yes, at source level. An ordinary publish can schedule new scans, write hidden pending-review moderation state to the existing skill row and digest, and then public search filters the hidden row through toPublicSkill.

## Solution Assessment

No patch is under review. The earlier direct preserve-active approach is not the best endpoint because it risks conflating public visibility with pending-version safety; the safer solution is a last-approved projection with explicit promotion rules.
