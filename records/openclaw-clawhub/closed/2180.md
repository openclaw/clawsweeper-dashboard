- **[P2] Avoid immutable caching for mutable skill OG data:** `server/routes/og/skill.png.ts:80`
  - body: This route now includes avatar, download, and audit state in the rendered image, but still marks any versioned response as `public, max-age=31536000, immutable`. Those values can change without the URL changing, so a versioned skill card can keep showing stale audit or stats data after moderation or usage changes; either include the mutable values in the cache key or use a short cache for this dynamic image.
  - confidence: 0.88

  - body: The new plugin OG route renders downloads and audit status fetched from package metadata, then makes the response immutable whenever `latestVersion` is present. Package scan status and stats can change after a release, so this can preserve stale security/status information for the same image URL; use a non-immutable cache or add the mutable fields to the generated URL.
