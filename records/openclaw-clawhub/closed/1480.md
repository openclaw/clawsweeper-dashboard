## Summary

Keep open: current main still lacks dashboard skill pagination, and the latest PR branch is not merge-ready because the new public paginated query can mix personal-publisher results with other publisher rows for the same owner user.

## What This Changes

The PR adds active owner/publisher skill indexes, a `skills.listDashboardPaginated` Convex query, `/dashboard` `usePaginatedQuery` Load More UI, and focused dashboard pagination tests.

## Best Possible Solution

Land the pagination-only dashboard feature after the personal-publisher path preserves publisher scoping and legacy-row behavior with cursor-safe indexed pagination, leaving dashboard search/count work to separate follow-ups.

## Reproduction Assessment

Yes. A fixture with a personal publisher plus an org-published skill sharing the same `ownerUserId` exercises the PR branch that queries `by_owner_active_updated` and can return rows outside the requested personal publisher.

## Solution Assessment

No. The indexed pagination direction is maintainable, but the current personal-publisher branch should not replace publisher scoping with an owner-wide query unless maintainers explicitly want a cross-publisher owner inventory.
