## Summary

Keep open. Current main still sends `/upload` to `/skills/publish`, where validation errors are summarized in a later Validation panel instead of being rendered as accessible field-local messages, and ClawHub cleanup policy keeps issues open.

## What This Changes

The issue asks for upload-page validation errors to be shown closer to the relevant inputs instead of being hidden in a later reminder area.

## Best Possible Solution

Add accessible field-level validation to the current publish/upload form while preserving the aggregate Validation card as the complete preflight summary.

## Reproduction Assessment

Yes. Source inspection is a high-confidence reproduction path: current main computes validation issues and renders them in a later card while the individual controls lack inline messages or field-specific accessibility attributes.

## Solution Assessment

Yes. Reusing the existing validation source for inline field messages plus `aria-invalid` / `aria-describedby`, while keeping the summary card, is narrower and more accessible than a popup-only reminder.
