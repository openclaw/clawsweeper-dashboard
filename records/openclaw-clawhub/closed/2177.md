## Summary

Keep open under the ClawHub issue policy: this is a fresh user-reported publish-flow bug, and current main still has a source-visible owner-routing gap in the New Version path that can plausibly drive the reported `checkSlugAvailability` failure.

## What This Changes

Clicking New Version on a published skill should open the update publish flow instead of showing a Convex `skills:checkSlugAvailability` error screen.

## Best Possible Solution

Preserve the current skill owner through the New Version route and make update-mode slug availability wait for that owner before preflight, while leaving backend publish authorization unchanged.

No high-confidence runtime reproduction was established in this read-only review. Source inspection shows a credible current-main path where the settings New Version link and legacy redirect omit the owner, allowing the publish route to preflight slug availability with the wrong owner during update mode.

## Solution Assessment

Yes for a narrow repair: carry or derive the existing owner handle before slug availability runs, then add regression coverage for the settings New Version path. A broader publish or ownership redesign is not needed for this report.
