## Summary

Keep open: current main does not contain the PR's theme-toggle wiring or legacy-key policy, the related bug remains open, and the proposed patch still needs a pre-hydration theme-script fix before it is safe to merge.

## What This Changes

The PR rewires the header theme ToggleGroup for pointer and keyboard system/light/dark selection, removes the runtime `clawdhub-theme` fallback, and adds header/theme regression tests.

## Best Possible Solution

Make the current-main theme selector work consistently for pointer, keyboard, cycle-button, runtime storage, and first paint, with one explicit legacy-key policy covered by tests.

## Reproduction Assessment

Yes. A focused check is to set only `clawdhub-theme=dark` before page load after applying the PR shape; the root script would still first-paint dark while hydrated runtime code would default to system.

## Solution Assessment

No. The PR's broader input coverage is reasonable, but the storage policy must be updated in both `src/lib/theme.ts` and the inline root script or kept in both places.
