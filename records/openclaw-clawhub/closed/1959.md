## Summary

This should stay open: current main still closes rescan-guidance issues with reopen-after-rescan guidance, and the linked production example shows that path failing for a non-collaborator reporter.

## What This Changes

The issue asks ClawHub to fix the automation loop that closes scanner-dispute issues while telling reporters to reopen them after rescanning.

## Best Possible Solution

Define a scanner-appeal path usable by non-collaborators, then update the label classifier, close workflow, guidance copy, and tests so scanner disputes do not receive unreachable reopen instructions.

## Reproduction Assessment

Yes. The linked #1958 production case shows the loop, and current main source still labels matching scanner disputes, posts reopen-after-close guidance, and closes the labeled issue.

## Solution Assessment

Unclear. There is no proposed patch, and the narrow code change depends on maintainer/security policy for how scanner appeals should remain writable by non-collaborators.
