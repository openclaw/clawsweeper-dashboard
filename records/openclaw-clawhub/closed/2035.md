## What This Changes

The PR adds a tracked Codex environment, `dev:worktree` and `seed:dev` scripts, a Bun worktree helper, and README/CONTRIBUTING guidance for detached worktree development.

## Best Possible Solution

Remove the personal env fallback, make env discovery portable or explicitly configured, and add terminal proof from a detached worktree for the run and seed flows before maintainer approval of the checked-in Codex workflow.

## Reproduction Assessment

Yes, from source inspection: a detached worktree on any machine without its own `.env.local` can only fall back to a contributor-specific `/Users/...` path. That is enough to reproduce the portability failure without mutating the checkout.

## Solution Assessment

No. The PR direction is useful, but the fallback should be based on `CLAWHUB_ENV_FILE`, an explicit argument, or portable worktree discovery rather than a personal path, and the PR should include real detached-worktree output before merge.
