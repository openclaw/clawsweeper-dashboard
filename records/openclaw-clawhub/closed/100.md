The issue reports CLI token auth failing because `https://clawhub.ai` allegedly redirects to `https://www.clawhub.ai` and strips the bearer `Authorization` header.

## Best Possible Solution

Maintainers should verify the current published CLI with a fresh production token on both apex and www hosts, then either consolidate this into the broader Unauthorized cluster or land one focused canonical-domain/auth fix with regression coverage.

## Reproduction Assessment

No. A live dummy-token probe no longer reproduces the reported `clawhub.ai` to `www.clawhub.ai` redirect, and proving whether valid tokens still fail requires a fresh production token and account state.

## Solution Assessment

Unclear. Switching defaults to `www.clawhub.ai` matched the original workaround, but current source and live routing use the apex host, so maintainer production-token verification should come before changing CLI defaults or hosting configuration.
