ClawScan package reviews should include declared package `openclaw.environment` env var and config metadata instead of reporting those requirements as none.

## Best Possible Solution

Add a sanitized package-environment mapper into package LLM evaluation, preserving existing prompt requirements and adding regression coverage for optional env vars, config paths, and malformed unrelated metadata.

## Solution Assessment

Yes. The narrow maintainable fix is to normalize `openclaw.environment` into the existing prompt requirements shape, not to change scanner policy or forward arbitrary package metadata.
