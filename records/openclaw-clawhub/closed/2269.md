## Summary

Current main and the v0.20.0 release now implement the central request: direct authenticated `clawhub install` writes local metadata, then posts the installed-skill snapshot to the registry telemetry endpoint, which creates install stat events for the counters.

## What This Changes

The issue reports that direct skill installs update local metadata but do not report install events, leaving `installsAllTime` and `installsCurrent` at zero.

## Best Possible Solution

Keep the v0.20.0 implementation: authenticated, telemetry-enabled direct installs report through the existing snapshot endpoint, while unauthenticated or opted-out installs remain outside the counters by policy.

## Reproduction Assessment

No for current main: source inspection shows `cmdInstall` now posts install telemetry after updating local metadata and the lockfile. The old missing-POST behavior no longer matches the current install path.

## Solution Assessment

Yes: the current implementation reuses the authenticated, opt-out-aware telemetry snapshot path instead of adding a separate counter endpoint, and it keeps install success independent from telemetry availability.
