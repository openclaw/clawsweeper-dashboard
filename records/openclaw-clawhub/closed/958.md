## Summary

Keep open because this is an openclaw/clawhub issue and the cleanup policy leaves issue closure to maintainers, even though the reported HTTP API regression is resolved on current main and production.

## What This Changes

The issue reports that `GET /api/v1/skills` returned an empty list after the V4 migration because the HTTP handler still called the deprecated empty `skills.listPublicPage` query.

## Best Possible Solution

Keep the V4/digest-backed HTTP listing implementation and let a maintainer close this issue manually as implemented; handle deprecated-stub hardening as a separate cleanup if desired.

## Reproduction Assessment

No, not against current main or production. The original curl path now returns non-empty items, and current main routes the HTTP handler to `listPublicApiPageV1` rather than the deprecated empty `listPublicPage` query.

## Solution Assessment

Yes. The shipped fix moves the HTTP API to an indexed digest-backed listing query, which is narrower and safer than reviving the old V1 listing query.
