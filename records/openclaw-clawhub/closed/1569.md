## Summary

Keep open. ClawHub issue cleanup policy does not auto-close issues, and the reported direct `clawhub transfer` path is still user-recipient based; current main only provides publish-time owner migration as a partial path while publisher-targeted transfer work remains open.

## What This Changes

The issue asks for a way to transfer personally owned skills to an organization so other org users can publish and update them.

## Best Possible Solution

Keep the explicit publish-time `--owner` plus `--migrate-owner` path available, then land a maintainer-approved publisher-targeted transfer model if direct transfer commands/API remain required.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: direct transfer still accepts `toUserHandle`, resolves only active users or user-kind personal publishers, and stores `toUserId`, so an organization publisher handle reaches `User not found`.

## Solution Assessment

Unclear as a final product direction. Publish-time migration is a maintainable partial solution, but direct transfer-to-org should be implemented as a publisher-targeted transfer model rather than by only changing recipient lookup.
