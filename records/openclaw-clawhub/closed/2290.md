## Summary

Keep open: this PR changes a security-sensitive production scan path and is not ready to merge because the worker installs Codex from `@latest` with production secrets, terminal worker failures would leave artifacts looking pending, and no real worker run proof is included.

## What This Changes

Adds a scheduled/dispatch GitHub Actions Codex worker and Convex `securityScan` queue for skill/package ClawScan classification, makes VirusTotal telemetry-only, trusts verified `@openclaw/*` plugin packages by default, and updates API/UI status tests and specs.

## Best Possible Solution

Land a pinned, reproducible production worker that records terminal scan failures on the target artifact and includes redacted live worker proof for fresh publish, successful classification, and failure paths.

## Reproduction Assessment

Yes. Source inspection is enough for the PR defects: the workflow installs `@openai/codex@latest` in Production, and terminal failures are recorded only on the job row rather than the artifact scan result.

## Solution Assessment

No. The worker direction is plausible, but this patch should pin the production Codex install, surface terminal worker failures on the target artifact, and provide live worker proof before merge.
