## Summary

Keep open. Current main still intentionally rejects deleted/deactivated GitHub OAuth sign-in, and this unresolved account-recovery cluster needs maintainer product and security direction before restore or unlink behavior changes.

## What This Changes

The issue requests restoring a deleted ClawHub account or clearing its GitHub OAuth binding so the same GitHub identity can sign in again.

## Best Possible Solution

Define an explicit audited recovery or re-registration policy for verified self-deleted accounts, or reaffirm irreversible deletion with a clear support path while keeping banned and abuse-disabled identities blocked.

## Reproduction Assessment

Yes, at source level: deleteAccount leaves a deactivated/purged user tied to the auth identity, and the OAuth callback rejects that user on later GitHub sign-in. I did not run a live GitHub OAuth flow.

## Solution Assessment

No, not as a one-off binding deletion or silent account restore. The maintainable path needs a policy-backed audited recovery or unlink flow that distinguishes self-deleted accounts from banned or abuse-disabled identities.
