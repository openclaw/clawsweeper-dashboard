## Summary

Keep open. Current main still treats user-initiated deletion as irreversible for GitHub OAuth, active-user state, and CLI/API-token access, while this account-specific restore or purge request needs verified operator handling plus a product/security decision.

## What This Changes

The reporter asks to restore or purge a deleted ClawHub account so the same GitHub login can access the site and CLI again.

## Best Possible Solution

Verify GitHub ownership and production user/provider rows, then choose one documented recovery policy and apply it consistently across OAuth, active-user gates, CLI, UI, docs, and tests.

## Reproduction Assessment

Yes, at source level. Deletion writes deactivatedAt/purgedAt and revokes tokens, OAuth rejects that state, active-user helpers hide the user, and CLI token creation requires an active user; the exact production account still needs operator verification.

## Solution Assessment

No autonomous source-only fix is the best next step. The maintainable path is verified operator handling plus an explicit product/security decision about restore, relink, recreate, or support-only deletion semantics.
