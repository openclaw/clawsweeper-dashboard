## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Enforce byte caps before fetching blobs:** `convex/httpApiV1/skillsV1.ts:1711`
  - body: The handler builds `blobTasks` for every file and calls `ctx.storage.get` for all of them before checking `MAX_EXPORT_TOTAL_BYTES`, then only compares bytes after `blob.arrayBuffer()`. With 1000 skills and the existing 10MB publish-file limit, a valid export can force far more storage reads and memory work than the advertised 256MB cap; use recorded file sizes to reject or stop before scheduling storage reads, and test that over-cap files do not call `storage.get`.
  - confidence: 0.89
- **[P2] Prevent export metadata path collisions:** `convex/httpApiV1/skillsV1.ts:1794-1795`
  - body: A published skill can already contain a root `_export_skill_meta.json`; the export then adds the same `${publisher}/${slug}/_export_skill_meta.json` path and `buildMergedExportZip` throws, failing the entire page. Put generated metadata somewhere user files cannot collide or handle this collision without a 500.
  - confidence: 0.87
- **[P2] Throttle unauthorized export attempts:** `convex/httpApiV1/skillsV1.ts:1580-1586`
  - body: This handler checks `requireApiTokenUser` before `applyRateLimit`, so missing or invalid bearer requests return 401 without consuming the export/IP bucket. Other auth-required v1 endpoints rate-limit first; do the same here so failed-token traffic is bounded.
  - confidence: 0.84
- **[P2] Require a public owner before exporting:** `convex/skills.ts:10481`
  - body: The export list only checks `latestVersionId` and `isPublicSkillDoc`, but the current public API list also drops digest rows when `digestToOwnerInfo` cannot produce a public owner. Reuse that owner-visibility check so deleted or deactivated publisher/user rows cannot be exported under the fallback user-id path.
  - confidence: 0.78
