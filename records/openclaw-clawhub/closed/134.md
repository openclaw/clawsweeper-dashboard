## Summary

Keep open. Current main still has a source-reproducible path where an ordinary non-trusted publisher updates an already-active skill, the skill row is patched to hidden pending scan, and public search/detail/list/API paths filter it out; ClawHub cleanup policy for this pass also keeps issues open for maintainer handling.

## What This Changes

The issue reports that publishing an update to an existing skill returns success or a server error while the skill disappears from search, inspect, direct detail, and public discovery surfaces.

## Best Possible Solution

Implement a version-aware public projection: keep the last approved skill card, digest, searchable embedding, API metadata, and safe file surface visible while the new version remains pending scan/review, then promote the new version only after scanner and moderation gates clear it.

## Reproduction Assessment

Yes, at source level. An ordinary non-trusted publisher updating an already-active skill can reach `insertVersion`, have the skill-level row patched to hidden pending scan, and then disappear from public search/list/detail/API paths that require active moderation status.

## Solution Assessment

No patch is under review. The prior direct preserve-active approach was closed because it can bypass pending-scan gating; the safer solution is last-approved-version continuity while the new upload remains gated.
