## Summary

Current main no longer creates GitHub commits for skill backup artifacts: the old GitHub mirror was replaced by registry artifact backups in object storage, so future skill publish/update backups no longer produce bot-authored commits in openclaw/skills.

## What This Changes

The issue asks ClawHub to author generated openclaw/skills commits with the publishing user's GitHub identity instead of the ClawHub bot identity.

## Best Possible Solution

Keep the object-storage registry backup implementation and treat the old generated-GitHub-commit author request as resolved for future ClawHub publishes.

## Reproduction Assessment

No current-main reproduction remains. The old GitHub commit backup writer is gone, and current main routes skill publish backups to registry artifact object storage instead.

## Solution Assessment

Yes for current main. Replacing generated GitHub commits with registry artifact objects removes the misleading bot-authored commit history for future publishes without inventing risky user author metadata.
