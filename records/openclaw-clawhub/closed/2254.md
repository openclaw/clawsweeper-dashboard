## Summary

Keep open: this is a protected `security` issue, and the published skill text confirms the reported tokened Apify search-to-run workflow while the live ClawHub scan still marks the listing clean.

## What This Changes

The issue reports an indirect prompt-injection risk in a published Apify skill that can pass untrusted Apify Store metadata into agent context before credentialed Actor execution.

## Best Possible Solution

A security maintainer should review the live listing, decide whether to hide or require a publisher update/rescan, and consider hardening ClawScan guidance for untrusted marketplace-result ingestion before credentialed tool use.

## Reproduction Assessment

No high-confidence runtime exploit was reproduced, but the source path is clear: the published SKILL.md documents raw Apify Store search result ingestion followed by credentialed Actor detail fetch and execution. A full reproduction would need a controlled APIFY_TOKEN and a crafted Apify Actor/search result.

## Solution Assessment

The suggested skill-level mitigations are directionally appropriate, but this should not be handled as an automated repo patch. The narrowest maintainable path is security moderation plus publisher update or targeted ClawScan policy hardening if maintainers decide this pattern should be flagged.
