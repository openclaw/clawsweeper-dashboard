## Summary

Keeping this open under the ClawHub issue policy. Current main appears to cover the central user need through direct org-publisher transfer plus publish-time owner migration, but maintainers should decide whether the stricter immediate admin-to-admin transfer semantics fully satisfy the requested pending org-acceptance flow.

## What This Changes

Request support for moving existing skills from personal ownership to organization ownership without deleting/re-publishing and losing stats or canonical URLs.

## Best Possible Solution

Keep the current source/destination-admin transfer and explicit publish-time migration paths, then have maintainers decide whether a separate pending org-acceptance flow is still required.

## Reproduction Assessment

No for the original failure on current main. Source and handler tests show an org publisher handle posted through the existing CLI body shape is routed to the direct owner-transfer mutation instead of failing as `User not found`.

## Solution Assessment

Yes for consolidating an existing skill under an org when the same actor can administer both owners. A pending recipient-acceptance flow would be a broader product choice and should stay with the publisher-transfer track if maintainers still want it.
