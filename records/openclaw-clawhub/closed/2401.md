## Summary

Keep open: the cleanup removes a released CLI option rather than only hiding the redundant help entry, so existing `clawhub skill verify <slug> --json` scripts can fail even though JSON remains the default output.

## What This Changes

The PR removes the `clawhub skill verify --json` option and its `VerifySkillOptions` type field, then adds a help-output e2e regression and includes it in `ci:e2e-http`.

## Best Possible Solution

Hide or de-emphasize `--json` in help while preserving it as an accepted compatibility alias, with regression coverage for both help omission and `skill verify <slug> --json` still succeeding.

## Reproduction Assessment

Yes. Source inspection shows current main registers `--json`, the PR deletes that registration, and there is no `allowUnknownOption` path to keep `skill verify <slug> --json` accepted.

## Solution Assessment

No. Removing the registered flag is not the safest cleanup; the maintainable path is to hide or deprecate `--json` while keeping it accepted as a compatibility alias.
