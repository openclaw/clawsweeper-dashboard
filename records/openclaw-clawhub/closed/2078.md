## Summary

The patch still leaves a source-reproducible provenance-wipe path for owner-merged skills, so it should not pass as-is.

## What This Changes

The PR tightens Convex skill soft-delete/undelete authorization, formats HTTP authz errors, updates VT hide provenance, adds audit metadata and regression tests, and records a changelog entry.

## Best Possible Solution

Preserve protected hide provenance for every non-restorable state, including owner-initiated merges, and cover the delete-to-undelete chain with a regression test.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an owner.merged row has hiddenBy set to the owner, owner delete clears moderationReason, and a follow-up owner undelete passes.

## Solution Assessment

No. The restore-side gate is the right direction, but the delete guard must also treat denied reasons like owner.merged as protected or avoid clearing provenance on already-hidden protected rows.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P2] Block owner re-delete of merged skills:** `convex/skills.ts:8174-8175`
  - body: `mergeOwnedSkillIntoCanonicalByActor` creates hidden rows with `hiddenBy` set to the owner and `moderationReason: "owner.merged"`. This guard treats that as owner-initiated, so owner delete reaches the data-hygiene reset, clears the deny reason, and a follow-up undelete can restore the merged-away skill. Treat denied reasons such as `owner.merged` as protected here or preserve provenance for already-hidden rows.
