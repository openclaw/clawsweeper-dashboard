## Summary

Keep open: current main still treats self-deleted GitHub-linked accounts as terminal, and the requested purge or re-registration path needs maintainer product/security direction rather than cleanup closure.

## What This Changes

The issue asks for a way to sign in, purge, or re-register after a self-deleted ClawHub account remains unable to create an active GitHub OAuth session.

## Best Possible Solution

Define one canonical lifecycle policy for deleted GitHub-linked accounts, then implement the approved recovery or support path across Convex auth, account deletion/admin tooling, UI messaging, specs/docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: account deletion sets deactivatedAt/purgedAt, GitHub OAuth reauth rejects that user, and active auth helpers then return no active user; I did not run a live production deletion test.

## Solution Assessment

No. A one-off named-account purge is not the best repository-level solution because deleted-account recovery needs an approved lifecycle policy that preserves OAuth identity binding, deletion guarantees, and ban boundaries.
