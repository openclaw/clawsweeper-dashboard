## Summary

Keep open: current main still has the moderated-skill transfer race, and this branch is a narrow fix, but it still needs contributor real behavior proof and has a non-persisting cancellation write in the new error path.

## What This Changes

The branch adds a moderation-state guard to `acceptTransferInternal` that rejects pending transfer acceptance for skills marked malicious, hidden, or removed.

## Best Possible Solution

Land a guarded accept-transfer path with regression coverage and proof so malicious, hidden, or removed skills cannot change owner through a pending transfer before autoban completes.

## Reproduction Assessment

Yes, there is a high-confidence source reproduction path: create a pending transfer, have scanner code patch the skill to malicious or hidden before the scheduled autoban transaction runs, then current main accepts the transfer because it only checks `softDeletedAt`. I did not execute a live Convex reproduction in this read-only review.

## Solution Assessment

Yes for the core race: checking moderation state in the shared `acceptTransferInternal` gate is the narrow maintainable fix. The cancellation persistence should be adjusted if maintainers expect the pending transfer to be closed as `cancelled`.
