## Summary

Keep open. Current main still matches the reported publish bug: CLI publish, archive upload, and GitHub import rely on extension allowlists before any content fallback; `.tsv` is still absent; and there is no linked implementation PR for this issue.

## What This Changes

The issue asks for `clawhub publish` to include `.tsv` and extensionless text files, and to warn authors when files are skipped instead of silently publishing incomplete skills.

## Best Possible Solution

Adopt one conservative text-file classification policy across skill publish entry points: add `.tsv`, classify extensionless text by content without admitting binaries, surface skipped-file warnings to authors, and cover CLI publish, archive/web upload, GitHub import, and server validation with regression tests.

## Reproduction Assessment

Yes, source-reproducible: a skill containing `SKILL.md`, `bin/openclaw-kraken`, and `config/endpoints.tsv` will have the latter two excluded by current extension checks before upload. I did not run a live publish/install round trip in this read-only review.

## Solution Assessment

Yes. The maintainable fix is a shared conservative classifier plus author-visible warnings, not one-off path exceptions; it should preserve binary rejection and existing ignore behavior.
