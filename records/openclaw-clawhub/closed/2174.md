## Summary

Keep open: current main does not implement publisher-selectable skill icons, and this fresh feature PR still needs normal review because one update path can clear saved icons and external real behavior proof is missing.

## What This Changes

The PR adds an optional `skills.icon` protocol string, backend normalization and projections, a skill publish icon picker, card/list rendering, and tests for curated lucide icons.

## Best Possible Solution

Land a reviewed custom-icon implementation that preserves existing icons across all update responses, keeps backend validation narrow, and includes real proof of the publish and card/list rendering flow.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No live run was performed, but source inspection gives a clear path: an owner updating a hidden or moderated skill gets `getBySlug` fallback data without `icon`, then the PR submit path sends `icon: ""` and clears it.

## Solution Assessment

Is this the best way to solve the issue? Not yet; the overall feature direction is plausible, but the PR should preserve icon data in the moderated fallback or avoid treating missing update data as an explicit clear.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82
