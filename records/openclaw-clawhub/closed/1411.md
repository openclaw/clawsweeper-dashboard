- **[P1] Keep moderation state in sync with edited summaries:** `convex/skills.ts:4814-4818`
  - body: The publish path derives moderation flags and `isSuspicious` from the persisted summary, but this mutation only patches `summary` and `updatedAt`. Owners can replace public/searchable text after scanning while trust metadata still describes the old content, so route the edit through moderation recomputation, rescan, or an explicit review/override flow.
  - confidence: 0.91

  - body: The detail page exposes the editor through `canManage`, which includes members of `skill.ownerPublisherId`, but `updateSummary` only permits the direct `ownerUserId` or moderators. Publisher members can open the editor and then always fail on save, so align the backend authorization or narrow the UI gate.
  - confidence: 0.94
- **[P2] Cap editable summaries on the server:** `convex/skills.ts:4800-4803`
  - body: This public mutation accepts any Convex string and stores it in public skill metadata read by detail, list, API, and digest surfaces. Add a server-side maximum and matching textarea `maxLength` so one edit cannot bloat public payloads.
  - confidence: 0.82
- **[P3] Normalize blank summaries before saving:** `convex/skills.ts:4815-4816`
  - body: Saving an empty textarea writes an empty string, but the static display only falls back for nullish values, so a cleared summary renders as a blank subtitle instead of `No summary provided.` Trim and store a missing value, or render blank strings through the same fallback.
  - confidence: 0.78
