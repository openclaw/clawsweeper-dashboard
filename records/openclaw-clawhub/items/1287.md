## Summary

Keep open: current main still lacks the package-specific changelog preview and most of the publish-plugin UI work, while the open PR is non-mergeable and has two source-reproducible changelog defects.

## What This Changes

The PR branch redesigns the publish-plugin page, adds package-specific changelog preview backend/frontend plumbing and tests, extracts publish helpers, and tweaks skill/soul stat display.

## Best Possible Solution

Refresh this as a rebased branch or replacement PR that ports package-specific changelog preview onto the current package publishing flow, preserves current folder upload and code-plugin metadata validation, and fixes the remaining changelog edge cases.

## Reproduction Assessment

Yes. Source inspection of the PR head shows a failed active preview request leaves prior auto text in place, and the package fallback path treats an existing prior release without a README as an initial release.

## Solution Assessment

No, not yet. The package-specific changelog direction is maintainable, but the PR should clear or invalidate stale auto text on failure, track prior-release existence separately from README availability, and be refreshed against current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Clear stale auto changelogs on generation failure:** `src/routes/publish-plugin.tsx:277`
  - body: When a preview request fails after a previous successful auto-generated changelog, this catch only switches the status to error and leaves the old auto text in the textarea. A user can then publish release notes for the wrong package, so clear auto-sourced text or block submission for the failed active request.
  - confidence: 0.88
- **[P2] Track prior package releases without README files:** `convex/lib/changelog.ts:268`
  - body: `oldReadmeText` is null both when there is no prior package release and when the prior release has no readable README. Because `generateFallback` treats null as an initial release, updates to existing packages can get misleading initial-release notes; track release existence separately from README availability.
  - confidence: 0.84
