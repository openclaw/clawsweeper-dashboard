## Reproduction Assessment

Yes. Source inspection of the PR head shows a failed active preview request leaves prior auto text in place, and the package fallback path treats a prior release without a README the same as no prior release.

## Solution Assessment

No, not yet. The package-specific changelog direction is maintainable, but this branch should clear or invalidate stale auto text on failure, track prior-release existence separately from README availability, and be refreshed against current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Clear stale auto changelogs on generation failure:** `src/routes/publish-plugin.tsx:275-278`
  - body: When a preview request fails after a previous successful auto-generated changelog, this catch only switches the status to error and leaves the old auto text in the textarea. A user can then publish stale release notes for a different package, so clear auto-sourced text or block submission for the failed active request.
  - confidence: 0.88
- **[P2] Distinguish prior releases without README files:** `convex/lib/changelog.ts:268`
  - body: `oldReadmeText` becomes `null` both when there is no previous release and when the previous release has no README. Since `generateFallback` treats null as an initial release, updates to an existing package that previously shipped without a README get misleading initial-release notes; track previous-release existence separately from README availability.
  - confidence: 0.84
