- **[P2] Clear stale auto changelogs on generation failure:** `src/routes/publish-plugin.tsx:275-278`
  - body: When a preview request fails after a previous successful auto-generated changelog, this catch only switches the status to error and leaves the old auto text in the textarea. A user can then publish stale release notes for a different package, so clear auto-sourced text or block submission for the failed active request.
  - confidence: 0.88
- **[P2] Track previous releases separately from README availability:** `convex/lib/changelog.ts:268`
  - body: `oldReadmeText` becomes `null` both when there is no previous release and when the previous release has no README, but `generateFallback` treats null as an initial release. Existing packages that previously shipped without a README will get misleading initial-release notes; keep a separate `hasPreviousRelease` signal for fallback generation.
  - confidence: 0.84
