- **[P2] Clear stale generated changelogs after preview failures:** `src/routes/publish-plugin.tsx:275-278`
  - body: If a previous auto-generated changelog exists, selecting another package and then hitting a failed preview request leaves the old text in the textarea while only changing the status to error.   - confidence: 0.88

Since the fallback treats `null` as an initial release, updates to an existing package that previously shipped without a README get misleading initial-release notes; keep a separate `previous`/`hasPreviousRelease` signal for fallback wording.
  - confidence: 0.84
