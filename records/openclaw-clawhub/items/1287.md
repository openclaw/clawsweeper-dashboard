- **[P2] Clear stale auto changelogs on generation failure:** `src/routes/publish-plugin.tsx:275-278`
  - body: When a preview request fails after an earlier auto-generated changelog succeeded, this catch only marks the status as error and leaves the old auto text in the textarea. A user can then publish release notes for a different package, so clear auto-sourced text/source or block submission for the failed active request.
  - confidence: 0.88
- **[P2] Distinguish prior releases without README files:** `convex/lib/changelog.ts:268`
  - body: `oldReadmeText` becomes `null` both when there is no previous release and when the previous package release has no README. Since fallback generation treats `null` as an initial release, updates to an existing package can get misleading initial-release notes; track previous-release existence separately from README availability.
