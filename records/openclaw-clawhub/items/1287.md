Full review comments:

- **[P2] Clear stale auto changelogs after preview failures:** `src/routes/publish-plugin.tsx:277`
  - body: When a later preview request fails after a prior auto-generated changelog, the catch path only marks the status as error and leaves the old auto text in place. Switching packages after a successful preview can therefore publish release notes for the wrong package; clear auto-sourced text or mark it unusable when the active request fails.
  - confidence: 0.88
- **[P2] Track previous releases separately from README text:** `convex/lib/changelog.ts:268`
  - body: `oldReadmeText` is `null` both when no prior package release exists and when a prior release exists without a README. Since the fallback treats `null` as an initial release, updates to an existing package that previously had no README can get misleading initial-release notes; carry prior-release existence separately.
  - confidence: 0.84
