## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Clear stale auto changelogs on generation failure:** `src/routes/publish-plugin.tsx:275-278`
  - body: If a previous preview succeeds and a later preview request fails for another package, the catch path only marks the status as error while leaving the old auto-generated text in the changelog field. That stale text can then be submitted as release notes for the wrong package, so clear auto-sourced text or mark it unusable when the active request fails.
  - confidence: 0.88
- **[P2] Distinguish prior releases without README files:** `convex/lib/changelog.ts:268`
  - body: `oldReadmeText` is `null` both when there is no prior package release and when the prior release exists but has no readable README. Because the fallback treats `null` as an initial release, updates to an existing package can get misleading `Initial release` notes; track prior-release existence separately from README availability.
  - confidence: 0.84
