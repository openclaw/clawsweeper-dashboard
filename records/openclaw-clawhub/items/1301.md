Current main contains the package trust, verification-tier, trusted-publisher, and scan scheduling plumbing, but the issue asks for a maintainer/security policy answer and production scanner-health inspection for specific published code plugins; source review alone cannot resolve either, and the ClawHub cleanup policy keeps issues open.

## What This Changes

The issue asks whether active community code-plugin publishers can get verified or trusted status and why several named code-plugin releases remained in pending scan state after more than 24 hours.

## Best Possible Solution

Maintainers should state whether community verified or trusted publisher status is available, planned, or intentionally cohort-limited, then inspect production scan state for the named plugins. If that inspection identifies a queue, environment, backfill, or scan-status finalization bug, it should be tracked as a narrower source fix.
