## Summary

Keep open. Current main and v0.22.0 have Official publisher rows, release verification tiers, trusted publishing, public scan status, and scan backfill plumbing, but they still do not define a public community verified-publisher path; the named packages no longer appear publicly pending, so the scanner part needs production history rather than a source-only fix.

## What This Changes

The issue asks whether ClawHub has a community verified-publisher path for active code plugin contributors and why several code plugin scans stayed pending.

## Best Possible Solution

Maintainers should define one community publisher trust model, document whether community verification is available or planned, and inspect production scanner history for the named packages before opening any narrower source bug.

## Reproduction Assessment

Partly: source confirms the public community verification path is missing. The original 24+ hour pending scans no longer reproduce on public package detail endpoints, so explaining that symptom needs production scanner history.

## Solution Assessment

No, not as an ad hoc badge grant or code-only fix. The maintainable solution is a maintainer-owned security/product policy that defines the trust signal before any UI, API, moderation, or publisher-status change grants it.
