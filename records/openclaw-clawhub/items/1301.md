## What This Changes

The issue asks whether ClawHub has a verified-publisher path for community code plugin publishers and why several published code plugins stayed in Scan: pending after more than 24 hours.

## Best Possible Solution

Maintainers should answer the verification policy publicly, preferably align it with the open vetted-community RFC, document the process or absence of one, and inspect production scanner state for the named plugins before deciding whether a narrower source bug exists.

## Reproduction Assessment

No. The verified-publisher path is a product/security policy question, and the scan-pending claim requires production packageRelease, job, env, worker, and log state for the named plugins rather than a source-only current-main reproduction.

## Solution Assessment

No immediate source change is the best solution yet. The maintainable path is maintainer security/product policy plus production scanner inspection, then a focused code/docs issue only if that inspection finds a concrete queue, worker, env, or status bug.
