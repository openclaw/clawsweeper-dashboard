## What This Changes

The issue asks for a verified-publisher path for active community code-plugin contributors and clarification of several named code plugins staying in pending security scan status.

## Best Possible Solution

Resolve the verified or vetted community trust model in https://github.com/openclaw/clawhub/issues/2272, document it across specs/API/admin/UI, and separately inspect production scanner state for the named plugins if they still show pending.

No. The verified-publisher portion is a policy request, and reproducing the scan-pending claim requires hosted package release records, securityScanJobs, scanner configuration, and failure logs for the named packages.

## Solution Assessment

Yes for keeping this open for manual review. Current trust and scan primitives are the right substrate, but a stronger public trust signal needs maintainer-approved criteria and the scanner half needs production audit before any source fix is scoped.
