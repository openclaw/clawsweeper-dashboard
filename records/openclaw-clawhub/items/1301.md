## Summary

Keep open. ClawHub cleanup policy keeps issues open, and the remaining verified-publisher path is a maintainer/security policy decision; the original pending-scan symptom no longer reproduces from the public package API for the named plugins.

## What This Changes

The issue asks whether active community code-plugin publishers can get a verified/trusted status between community and official, and why several published plugins stayed pending for more than 24 hours.

## Best Possible Solution

Maintainers should publish the community verified/trusted publisher policy, keep owner rescan guidance available, and inspect production scan/artifact details before granting trust to any executable plugin publisher.

## Reproduction Assessment

No. The public package API now reports the seven named plugins as suspicious or clean rather than pending, and the remaining verified-publisher question is a policy path rather than a bug reproduction.

## Solution Assessment

No source-only fix is clearly the best solution. The narrow maintainable path is maintainer/security policy review plus production scanner inspection if the historical pending state still matters.
