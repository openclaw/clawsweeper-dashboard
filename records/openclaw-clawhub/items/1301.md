## Summary

Keep open. This remains a maintainer-owned marketplace trust and production scanner-health question: current main has package trust primitives, trusted-publisher gates, scan scheduling, and fallback scan handling, but no public verified-community publisher process that source review can safely apply.

## What This Changes

The issue asks whether active community code-plugin publishers can get a verified publisher status and why several published code plugins stayed in pending scan status after more than 24 hours.

## Best Possible Solution

Maintainers should settle the vetted-community publisher policy alongside the open trust-badge requests, document the outcome, and separately inspect production scanner records for the named plugins before scoping any source fix.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No. The verified-publisher request is a policy question, and the scan-pending symptom requires production package records plus scanner job and log access for the named plugins.

## Solution Assessment

Is this the best way to solve the issue? No automated source change is the best next step yet. The maintainable path is maintainer security/product review plus production scanner inspection, with any confirmed scanner defect split into a narrower follow-up.
