## Summary

Keep open. Current main has Official publisher policy, release verification tiers, trusted-publisher publishing, and package scan queueing, but it still does not implement a public verified/vetted community publisher path; the named pending scans require production scanner and queue inspection rather than a source-only close.

## What This Changes

The issue asks whether active community code-plugin publishers can get verified/trusted status and why several named code-plugin scans remained pending after more than 24 hours.

## Best Possible Solution

Use the open vetted-community RFC for the trust-semantics decision, inspect production scanner state for the named packages, and open narrower source work only if that inspection identifies a concrete queue, configuration, or worker bug.

## Reproduction Assessment

No. The verified-publisher question is a policy/feature request, and reproducing the pending scans requires production package records, queue state, scanner configuration, and failure logs for the named plugins.
