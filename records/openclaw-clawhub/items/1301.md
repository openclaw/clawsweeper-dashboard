Keep open: current main has official/community/private channels, release verification tiers, and package scan plumbing, but it still does not define a public verified or vetted community publisher path. ## What This Changes

The issue asks whether ClawHub has a verified publisher path between community and official for active community code-plugin contributors, and why the named code-plugin scans remained pending for more than 24 hours.

## Best Possible Solution

Resolve the vetted-community publisher/plugin policy in the open RFC, document the approved trust semantics, implement any schema/admin/API/UI changes consistently, and separately inspect production scan history if the original pending-state report still matters.
