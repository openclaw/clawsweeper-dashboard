## Summary

Keep open. This is an issue in the ClawHub target, and the remaining work is a maintainer trust-policy answer plus production scanner-health inspection for the named code plugins, not a source-only cleanup close.

## What This Changes

The issue asks whether active community code-plugin publishers can obtain verified status and why several source-linked plugins remained in pending scan state for more than 24 hours.

## Best Possible Solution

Maintainers should state the current community verified/trusted-publisher policy, inspect the named production plugin scan records, and file a narrower source issue only if that inspection finds a queue, configuration, or scan-finalization bug.

## Reproduction Assessment

No high-confidence reproduction is available from the checkout alone. Source shows new code-plugin releases start as pending and scanner jobs can skip without deployment secrets, but reproducing the reported 24+ hour pending state requires production release records, job state, env, and logs for the named plugins.

## Solution Assessment

No source-only fix is clearly the best solution yet. The narrow maintainable path is maintainer/security review of the trust policy plus production scanner inspection; current rescan guidance from related #1526 may help affected publishers but does not answer this policy question.
