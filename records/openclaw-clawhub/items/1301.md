## Summary

Keep open: current main has community/official/private channels, source/provenance/rebuild verification tiers, trusted-publisher plumbing, scan scheduling, and live public scan states for the named packages, but it still does not provide a documented vetted-community publisher path and the original production scanner history needs maintainer inspection.

## What This Changes

The issue asks whether active community code-plugin publishers can get a verified-publisher status between community and official, and why several named code plugins stayed in pending scan status after publication.

## Best Possible Solution

Resolve the vetted-community trust model in https://github.com/openclaw/clawhub/issues/2272, document the approved policy across specs/API/admin/UI, and inspect production scanner records for the named packages before scoping any narrow source fix.
