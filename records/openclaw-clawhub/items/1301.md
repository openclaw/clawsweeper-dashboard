## Summary

Keep open. Current main has official publisher policy, trusted publishing, release verification tiers, and package scan queues, but it still does not provide the requested public verified or vetted-community publisher path; the named pending scans also need production scanner/queue inspection rather than a source-only close.

## What This Changes

The issue asks whether active community code-plugin publishers can get verified-publisher-style trust status and why several published code plugins stayed at pending scan status after more than 24 hours.

## Best Possible Solution

Use https://github.com/openclaw/clawhub/issues/2272 for the vetted-community policy decision, then inspect production scanner state for the named packages and open a narrower source bug only if pending scans trace to queue, config, or worker failure.

No. The verified-publisher path is a policy/feature request, and the pending-scan symptom needs production package records, queued job state, scanner configuration, and failure logs for the named plugins before it can be reproduced with high confidence.

## Solution Assessment
