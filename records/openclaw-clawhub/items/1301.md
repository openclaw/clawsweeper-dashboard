## Summary

Keep open: this remains a live ClawHub security/product support question. Current main has official/community/private channels, release verification tiers, trusted-publisher controls, and package scan scheduling, but no approved community verified-publisher path and no source-only way to verify the named production scan-pending reports.

## What This Changes

The issue asks whether active community code-plugin publishers can get a verified-publisher status between community and official, and why several named code-plugin scans stayed pending for more than 24 hours.

## Best Possible Solution

Resolve the verified or vetted community trust model through https://github.com/openclaw/clawhub/issues/2272, document the approved semantics across specs/API/admin/UI, and separately inspect production scanner state for the named plugins if they still show pending.

## Reproduction Assessment

No. The verified-publisher portion is a policy request, and reproducing the scan-pending claim requires hosted packageRelease records, securityScanJobs, scanner configuration, and failure logs for the named packages.

## Solution Assessment

Yes for keeping this open for manual review. Current trust and scan primitives are the right substrate, but a stronger public trust signal needs maintainer-approved criteria and the scanner half needs production audit before any source fix is scoped.
