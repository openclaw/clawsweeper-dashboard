## Summary

Keep this issue open. ClawHub v0.21.0 supports package verification states and owner-requested rescans, but it still has no documented community publisher-verification path, and source inspection cannot determine why the named production scans remained pending.

## What This Changes

Define the verification path for active community plugin publishers and clarify or resolve code-plugin scans that remain pending for an extended period.

## Best Possible Solution

Define and document whether ClawHub will offer a non-Official community trust tier, including eligibility and revocation rules, then inspect the named production scan jobs and expose actionable timeout, status, and retry information to package owners.

## Reproduction Assessment

Unclear. Current source confirms that no generic community Official path exists and explains how packages display as pending, but reproducing the prolonged production delay requires live scan records for the named packages.

## Solution Assessment

No, a rescan alone is not the complete solution. Maintainers must first decide the community trust policy, then diagnose production scanner behavior and implement focused status or retry improvements if an operational defect is confirmed.
