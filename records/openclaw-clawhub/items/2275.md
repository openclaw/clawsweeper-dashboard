## Summary

Keep open: current main and the live API still reproduce the supported non-trending cursor bug, and an open linked fix PR already targets the issue.

## What This Changes

The issue reports that `GET /api/v1/skills` repeats the first page when callers pass back the advertised `nextCursor`.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/clawhub/pull/2286 with a scoped fix that accepts the full self-describing `getPage()` cursor for supported non-trending sorts while preserving stale/wrong-index safeguards.

## Reproduction Assessment

Yes. The live API repeated the same `sort=updated` page when replaying its own `nextCursor`, and current main source shows the emitted full helper key is rejected on decode.

## Solution Assessment

Yes for the active non-trending fix direction. The narrow maintainable solution is to accept full matching self-describing `getPage()` keys while keeping raw-array and wrong-index cursor guards intact.
