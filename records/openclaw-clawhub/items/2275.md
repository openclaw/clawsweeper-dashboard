## What This Changes

The issue requests fixing `GET /api/v1/skills` cursor pagination so replaying `nextCursor` advances instead of returning the first page again.

## Best Possible Solution

Land or replace the linked non-trending cursor decoder fix with real API proof, and track cursor support for `trending` separately if maintainers want that behavior.

## Reproduction Assessment

Yes. A live `clawhub.ai` round trip for `sort=updated&limit=10` reproduced byte-identical first and second responses, and current main source shows why the emitted full helper cursor is rejected on replay.

## Solution Assessment

Yes for the active fix direction. The narrow maintainable repair is to accept full self-describing `getPage()` keys for the matching index while preserving raw-array, wrong-index, and stale-cursor guards.
