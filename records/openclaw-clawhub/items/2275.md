## Summary

Keep open: this is a valid current API pagination bug, and an open linked fix PR already references it with closing syntax, so the issue should remain open until that PR merges or is closed.

## What This Changes

The issue requests that `GET /api/v1/skills` honor the server-returned `cursor` parameter instead of repeating the first page for non-trending sort modes.

## Best Possible Solution

Review and land or close the open linked fix PR, keeping the repair scoped to accepting the full non-trending public list cursor shape already emitted by the API.

## Reproduction Assessment

Yes. The issue provides a concrete production curl reproduction, and current source shows the same failure path: emitted full index keys can be rejected and the query restarts from the first page.

## Solution Assessment

Yes. The narrow maintainable solution is to make the cursor decoder and regression tests accept the full self-describing non-trending `getPage` index-key shape while preserving stale or wrong-index cursor handling.
