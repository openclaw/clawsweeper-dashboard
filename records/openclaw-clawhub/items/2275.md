## What This Changes

The issue reports that `GET /api/v1/skills` returns the same page and `nextCursor` when clients replay the server-provided cursor.

## Best Possible Solution

Keep the merged https://github.com/openclaw/clawhub/pull/2286 implementation as the canonical non-trending cursor fix, and track any desired `trending` pagination as a separate API enhancement.

## Reproduction Assessment

No, not against current main or production for the documented non-trending cursor contract. Replaying `nextCursor` now advances for the tested non-trending sorts, and the regression test covers the full `getPage()` cursor replay path.

## Solution Assessment

Yes. The merged decoder change is the narrow maintainable fix because it accepts full self-describing cursors only for the matching index and prefix while preserving stale-cursor guards.
