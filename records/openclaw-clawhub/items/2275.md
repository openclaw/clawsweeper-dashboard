## Summary

Keep open: current main and the live API still repeat the first page when a documented non-trending `nextCursor` is replayed, and the linked fix PR is open rather than merged.

## What This Changes

Fix `GET /api/v1/skills` cursor pagination so replaying `nextCursor` advances to the next page for documented non-trending sorts.

## Best Possible Solution

Land the scoped non-trending cursor decoder fix from https://github.com/openclaw/clawhub/pull/2286 or an equivalent replacement, then track any desired trending pagination separately.

## Reproduction Assessment

Yes. A live `clawhub.ai` round trip still returns identical first and second pages when replaying `nextCursor`, and current main source shows the emitted full helper key is rejected by the decoder.

## Solution Assessment

Yes for the scoped bug. The best path is to accept full self-describing `getPage()` keys for the matching non-trending index while preserving stale or wrong-index guards.
