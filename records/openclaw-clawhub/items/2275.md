## What This Changes

Fix `GET /api/v1/skills` so replaying a returned `nextCursor` advances to the next page for documented non-trending sorts.

## Best Possible Solution

Review and land or replace the linked cursor-decoder fix, preserving non-trending cursor semantics and treating any trending pagination request separately.

## Reproduction Assessment

Yes. A live `clawhub.ai` round trip reproduced byte-identical first and second pages, and current main source shows the returned full helper cursor key is rejected when supplied back.

## Solution Assessment

Yes for the active fix direction. The narrow maintainable solution is to accept full self-describing `getPage()` keys for the matching index while preserving stale and wrong-index guards.
