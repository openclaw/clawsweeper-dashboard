## Summary

Keep open only because the ClawHub cleanup policy for this sweep does not auto-close issues; the documented non-`trending` `/api/v1/skills` cursor replay bug is fixed on current `main`, covered by regression tests, and present in production deploy tags.

## What This Changes

The issue reports that replaying `/api/v1/skills` `nextCursor` returns the same first page instead of advancing.

## Best Possible Solution

Keep the merged non-trending cursor decoder fix as the canonical implementation and track any desired `trending` cursor support separately.

## Reproduction Assessment

No. On current source and production, replaying a non-`trending` `nextCursor` advances to a different page, and the regression test covers the same full-cursor replay path.

## Solution Assessment

Yes. Accepting full self-describing `getPage()` keys only when the index and equality prefix match is the narrow maintainable fix while preserving stale-cursor guards.
