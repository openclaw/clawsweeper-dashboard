## Best Possible Solution

Review and land the linked cursor-decoder fix, then close this issue after non-trending API pagination is proven against the real endpoint.

## Solution Assessment

Yes. The best path is the already-linked narrow decoder fix that accepts the full self-describing `getPage()` key while preserving wrong-index and legacy cursor guards.
