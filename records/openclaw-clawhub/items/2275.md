## Summary

Keep open under the ClawHub issue policy: the documented non-trending `/api/v1/skills` cursor loop is fixed on current `main` and production by https://github.com/openclaw/clawhub/pull/2286, but this sweep is configured not to auto-close ClawHub issues.

## What This Changes

The issue reports that `GET /api/v1/skills` ignores the returned `nextCursor` and repeats the first page across sort modes.

## Best Possible Solution

Keep https://github.com/openclaw/clawhub/pull/2286 as the canonical non-trending cursor fix, and track any desired `trending` cursor pagination as a separate API enhancement.

## Reproduction Assessment

No. The original cursor replay no longer reproduces for documented non-trending pagination: production `sort=updated&limit=3` advanced to different second-page slugs, and current main has regression coverage for replaying the full self-describing cursor.

## Solution Assessment

Yes for the documented non-trending bug. The merged decoder accepts full `getPage()` cursors only for the matching index and equality prefix, preserving stale-cursor safeguards while honoring the opaque cursor contract.
