## Summary

Keep open. The pagination bug is valid and source-reproducible, but this issue has an open closing PR with a targeted fix, so it should remain open until that PR lands or is closed.

## What This Changes

GET /api/v1/skills should advance when clients pass back nextCursor for documented non-trending sorts instead of repeating the first page.

## Best Possible Solution

Review and land https://github.com/openclaw/clawhub/pull/2286 after real API pagination proof, preserving the non-trending cursor contract and tracking any trending pagination request separately.

## Reproduction Assessment

Yes. Source inspection of current main shows the API emits self-describing getPage index keys as nextCursor and then rejects those full keys on reuse; a live clawhub.ai check also returned the same 10 slugs and same cursor after passing nextCursor back.

## Solution Assessment

Yes. The linked PR’s direction is the narrow maintainable fix: accept full helper keys only for matching self-describing cursors while preserving wrong-index and legacy raw-array guards.
