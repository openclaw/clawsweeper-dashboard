## Summary

Keep open only because this ClawHub sweep does not auto-close issues; the documented non-trending `/api/v1/skills` cursor replay bug is fixed on current `main`, covered by regression tests, and present in production.

## What This Changes

The issue reports that replaying the returned `/api/v1/skills` `nextCursor` returns the same first page instead of advancing pagination.

## Best Possible Solution

Keep the merged non-trending cursor decoder fix as the canonical implementation, and track any desired `trending` cursor pagination as a separate API design item.

## Reproduction Assessment

No, not for the documented non-`trending` API contract on current `main` or production. Replaying a live `sort=updated` `nextCursor` now advances to a different page, and the regression test covers the same full-cursor replay path.

## Solution Assessment

Yes. Accepting full self-describing `getPage()` keys only when the index and equality prefix match is the narrow maintainable fix for the opaque cursor contract while keeping stale-cursor guards intact.
