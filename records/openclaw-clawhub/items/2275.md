## Summary

Keep open: current main and live production still repeat the first `GET /api/v1/skills` page when replaying the advertised cursor, and the open linked fix PR should land or be replaced before this issue closes.

## What This Changes

The issue reports that `GET /api/v1/skills` ignores returned cursors and repeats the first page across sort modes.

## Best Possible Solution

Review and land https://github.com/openclaw/clawhub/pull/2286 or an equivalent replacement for non-trending cursor decoding, while tracking any trending pagination request separately.

## Reproduction Assessment

Yes. A live `clawhub.ai` request replayed the advertised `sort=updated` cursor and returned byte-identical first and second pages, and current main source explains the same failure by rejecting the emitted full helper key.

## Solution Assessment

Yes for the active fix direction. The narrow maintainable solution is to accept full self-describing `getPage()` keys for the matching index while preserving stale/wrong-index and legacy raw-array guards.
