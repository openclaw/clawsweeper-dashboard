## Summary

Keep open: current main still has a source-level cursor handling bug for non-trending `GET /api/v1/skills`, and https://github.com/openclaw/clawhub/pull/2286 is an open linked fix candidate, so this issue should remain open until that PR lands or is replaced.

## What This Changes

The issue asks for `GET /api/v1/skills` to honor the server-provided `nextCursor` instead of repeating the first page for paginated requests.

## Best Possible Solution

Land the linked cursor-decoder fix after after-fix real API proof is added, then close this issue as implemented and track any desired trending cursor support separately.

## Reproduction Assessment

Yes. A live clawhub.ai replay of the advertised `sort=updated&limit=10` cursor reproduced a byte-identical first and second page on a stable attempt, and current main source shows the emitted full `getPage()` key is rejected back to the first page.

## Solution Assessment

Yes for the active fix direction. The narrow maintainable path is to accept full self-describing `getPage()` keys for the matching index while preserving legacy raw-array and wrong-index guards; the issue should stay open until that lands.
