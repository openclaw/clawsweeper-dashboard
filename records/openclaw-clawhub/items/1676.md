## Summary

Keep open. Current main appears to cover the core automatic GitHub username rename path for derived personal handles, but this is an open ClawHub issue and the cleanup policy only permits auto-closing PRs. The remaining decisions around opt-in versus automatic rename behavior, alias/reservation semantics, public URL policy, docs/API coverage, and org ownership coordination still need maintainer judgment.

## What This Changes

The issue asks ClawHub to support GitHub username changes so stored user handles, profile/skill owner URLs, and transfer target lookup no longer remain stuck on the old GitHub login.

## Best Possible Solution

Maintainers should decide and document the handle-rename policy, then either bless the current automatic derived-handle sync and fill any missing alias/reservation/docs gaps, or specify an explicit authenticated rename UX/API with tests coordinated with publisher and org ownership.
