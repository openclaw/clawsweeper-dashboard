## Summary

Keep open. Current main has structured static-scan findings, but package moderation-status still exposes only aggregate scan state and reasons; deciding how much rule/file/evidence detail to expose is security-sensitive product work.

## What This Changes

The issue asks ClawHub to expose scanner rule and file:line details in CLI moderation inspection output and document server-side rule IDs.

## Best Possible Solution

Define an owner/staff/public scanner-finding disclosure contract, then wire redacted `staticScan.findings` into package moderation-status and CLI inspect output with docs and regression tests.

## Reproduction Assessment

Yes, source-level. The package moderation-status type, query, API schema, and CLI formatter all return aggregate scan state and reasons without static finding details, while the persisted static scan model already has code/file/line fields.

## Solution Assessment

Unclear until maintainers decide the disclosure policy. The narrow maintainable direction is owner-visible, redacted diagnostics rather than publicly exposing raw scanner evidence by default.
