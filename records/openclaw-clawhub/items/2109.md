## Summary

Keep open. Current main already stores scanner findings and exposes them through package version/static-analysis surfaces, but the requested owner `moderation-status` contract and human `package inspect` output still omit rule/file/line detail, and exposing scanner evidence needs maintainer disclosure policy.

## What This Changes

Request to expose scanner rule ID plus file:line detail in package moderation-status / inspect output and document server-side scanner rule identifiers.

## Best Possible Solution

Define a sanitized scanner-disclosure contract, expose approved rule IDs and file:line details to authorized package owners and staff in moderation-status and package inspect, and document intentionally public rule identifiers.

## Reproduction Assessment

Yes, from source inspection. Current main stores rule/file/line finding data and exposes it in version/static-analysis paths, while moderation-status and human package inspect still return only aggregate scanner and moderation data.

## Solution Assessment

Unclear until maintainers set the disclosure policy. Surfacing sanitized owner/staff rule and location details is likely the right direction, but raw evidence snippets should not be broadened without explicit access-control and redaction rules.
