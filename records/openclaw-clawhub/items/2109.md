The issue asks ClawHub to expose scanner rule IDs and file:line details in moderation-status or inspect output for publisher false-positive triage.

## Best Possible Solution

Define a sanitized owner/staff scanner-disclosure contract, expose approved rule IDs and file:line details in the relevant CLI/API surfaces, and document intentionally public rule identifiers while keeping raw evidence scoped by policy.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainers set disclosure policy. The safest direction is sanitized owner/staff-scoped rule and file:line output, not broad raw evidence exposure across public surfaces.
