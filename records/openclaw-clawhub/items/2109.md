## Summary

Keep open. Current main still does not expose scanner rule/file/line detail through package moderation-status or human package inspect, and the stored findings include evidence snippets, so this needs maintainer disclosure policy before implementation.

## What This Changes

Request scanner finding details, including rule IDs and file/line locations, in package moderation-status and inspect output for publisher false-positive triage.

## Best Possible Solution

Define a field-level scanner disclosure contract, expose sanitized rule ID plus file:line details to authorized owners and staff in moderation-status/package inspect, redact raw evidence by default, and document stable public rule identifiers.

## Reproduction Assessment

Yes. Source inspection shows package staticScan findings store code/file/line/evidence, while moderation-status schema/query/CLI and human package inspect still expose only aggregate scan and verification status.

## Solution Assessment

Unclear. The requested direction is useful, but the best implementation should be sanitized and owner/staff-scoped rather than exposing raw scanner evidence or full server-side rule internals broadly.
