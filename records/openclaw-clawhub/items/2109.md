## Summary

Keep open. Current main still lacks a sanitized package scanner finding-detail contract in `package moderation-status` and normal human `package inspect` output, and this security-sensitive disclosure needs maintainer policy judgment.

## What This Changes

Request to show scanner rule and file:line details in package moderation-status/inspect output and document server-side rule IDs for false-positive triage.

## Best Possible Solution

Define a single package scanner diagnostic contract: approved rule IDs plus file:line/message for owners and staff in moderation-status and human inspect, with raw evidence redacted by default and public docs for intentionally disclosed rule identifiers.

## Reproduction Assessment

Yes, by source inspection. Current main stores package static findings with rule and file/line data, while moderation-status and normal human package inspect output do not surface those details.

## Solution Assessment

Unclear until maintainers decide disclosure policy. The narrow maintainable path is owner/staff-scoped rule and file:line output with raw evidence redacted or separately gated, not a direct dump of stored scanner snippets.
