## Summary

Keep open. Current main still lacks a sanitized rule/file/line diagnostic contract in `package moderation-status` and normal human `package inspect` output, and this ClawHub issue needs maintainer policy judgment before implementation.

## What This Changes

Requested behavior is to expose scanner rule and file:line finding details in publisher-facing moderation-status or inspect output, and document server-side rule identifiers.

## Best Possible Solution

Define one sanitized scanner-disclosure contract, expose approved rule IDs and file:line details to authorized publishers and staff through moderation-status and inspect, and document intentionally public rule identifiers without dumping raw evidence by default.

## Reproduction Assessment

Yes, by source inspection. Current main stores package static findings with rule and file/line data, while moderation-status and normal human package inspect output do not expose sanitized finding details.

## Solution Assessment

Unclear until maintainers decide disclosure policy. The maintainable path is owner/staff-scoped rule and file:line output with raw evidence redacted or gated, not a direct dump of stored scanner snippets.
