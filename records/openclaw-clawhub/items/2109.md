## Summary

Keep open. Current main still stores scanner findings with rule/file/line data, but package moderation-status, human package inspect, and package moderation docs expose only aggregate scan/moderation state; adding this needs an explicit scanner-disclosure and redaction policy.

Expose scanner rule/file/line details in package moderation-status/package inspect and document server-side scanner rule IDs for publisher false-positive triage.

## Best Possible Solution

Define a sanitized scanner-disclosure contract, expose approved rule IDs and file:line details to authorized owners and staff in moderation-status/package inspect, and document intentionally public rule identifiers.

## Solution Assessment

Unclear until maintainers set disclosure policy. The maintainable direction is sanitized owner/staff-scoped rule and file:line output, not broad raw evidence exposure across public surfaces.
