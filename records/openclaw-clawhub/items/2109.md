## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still does not expose package scanner rule/file/line details in `package moderation-status` or human `package inspect` output; the remaining work needs a deliberate disclosure policy.

## What This Changes

Request to expose sanitized scanner rule and file:line diagnostics in package moderation-status/inspect output and document server-side rule IDs.

## Best Possible Solution

Define a sanitized owner/staff-scoped scanner diagnostic contract, document intentionally public rule IDs, then wire it through the package schema, Convex API, CLI output, docs, and tests.

## Reproduction Assessment

Yes, by source inspection. Current main stores package static findings with code/file/line data, while moderation-status and human inspect output expose only aggregate scan and verification fields.

## Solution Assessment

Unclear until maintainers decide disclosure policy. The maintainable direction is sanitized rule/file/line output for authorized publishers and staff, not exposing raw `staticScan` evidence wholesale.
