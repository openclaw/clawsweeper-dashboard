Keep open. This issue is still a valid ClawHub security/product follow-up: current main has static scanner details available on the package version API, but the owner-facing `package moderation-status` API/CLI remains aggregate-only, and the broader scanner disclosure boundary needs maintainer review.

## What This Changes

Request to expose scanner rule/file/line details in package moderation-status or package inspect output, and to document server-side scanner rules enough for false-positive triage.

## Best Possible Solution

Expose a safe authenticated owner/moderator path for existing staticScan code/file/line details, document the intentionally disclosed fields, and reserve raw evidence or broader rule catalogs for explicit security policy approval.

## Reproduction Assessment

Yes for the output limitation: current source shows moderation-status and normal package inspect human output omit scanner finding details while the live version API has code/file/line data for the cited release. ## Solution Assessment

Yes directionally for a narrow owner-facing disclosure of existing staticScan fields. Publishing broader server-side rules, regexes, raw evidence, LLM findings, or VirusTotal details needs maintainer security/product approval first.
