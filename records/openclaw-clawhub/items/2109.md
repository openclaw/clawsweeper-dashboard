## Summary

Keep open. Current main stores package static-scan findings and exposes them on the exact-version JSON endpoint, but the owner-facing moderation-status schema/query/CLI and normal human inspect output still do not surface rule/file/line diagnostics, and the remaining disclosure boundary needs maintainer security/product approval.

## What This Changes

The issue requests scanner finding detail such as rule ID plus file and line in package moderation-status or inspect output, with enough documented scanner context for false-positive triage.

## Best Possible Solution

Expose a narrow owner/staff diagnostic surface for stable static finding identifiers and file/line/message data, document that contract, and keep scanner internals or sensitive evidence behind an approved security policy.

## Reproduction Assessment
