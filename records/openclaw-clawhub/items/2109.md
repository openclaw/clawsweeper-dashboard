## Summary

Keep open. ClawHub policy keeps issues open, and current main still lacks the requested owner-facing scanner rule/file/line detail in `package moderation-status` and human `package inspect` output even though stored package static-scan data can contain that detail.

## What This Changes

Expose static-analysis rule IDs and file:line locations in package moderation-status or inspect output, and document server-side rule identifiers for false-positive triage.

## Best Possible Solution

Define a sanitized owner/staff-scoped scanner-disclosure contract, document stable public rule IDs, then update the schema, Convex resolver, API docs, CLI output, and tests consistently.

## Reproduction Assessment

Yes, by source inspection. Current main stores package static findings with code/file/line/evidence, while the package moderation-status schema/backend and human inspect formatter still expose only aggregate scan and moderation information.

## Solution Assessment

Unclear until maintainers set disclosure policy. The maintainable direction is sanitized owner/staff-scoped rule and file:line output that avoids broadly exposing raw scanner evidence.
