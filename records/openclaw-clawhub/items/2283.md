Full review comments:

- **[P1] Start package ban continuations after the ban is committed:** `convex/packages.ts:2904-2910`
  - body: The user ban and autoban paths call this batch before `users.deletedAt` is patched, but this guard only permits that pre-commit state for the first page. Because the first page schedules the next page with a cursor from this separate mutation, page 2 can run while the parent ban mutation still has not committed `deletedAt`, return `stale`, and leave packages after the first 25 plus their publish tokens active. Commit the ban before scheduling package pagination, or make the continuation guard safe for the same in-flight ban, and cover a multi-page owner in tests.
  - confidence: 0.84
