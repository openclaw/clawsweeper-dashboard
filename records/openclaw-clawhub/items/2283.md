## Summary

This PR should stay open for maintainer sign-off: the patch looks structurally sound and has sufficient live Convex proof, but it intentionally changes package sanction and publish-token recovery policy rather than landing as conservative cleanup.

## What This Changes

This PR hides personal package/plugin resources and revokes package publish tokens on owner ban or account deletion, restores matching ban-hidden packages on unban, preserves org-owned packages, and adds tests, spec, UI copy, and changelog coverage.

## Best Possible Solution

Land a maintainer-approved moderation policy that sanctions personal package resources, preserves org-owned resources, permanently revokes affected publish tokens, and keeps the matching-ban restore invariant covered by regression tests.

## Reproduction Assessment

Yes. Source inspection on current main shows user ban/delete flows do not touch packages or package publish tokens, while package reads already hide rows with softDeletedAt.

## Solution Assessment

Yes, pending maintainer policy sign-off. Reusing package soft-delete/search-digest handling plus indexed owner batches and focused token revocation is the narrowest maintainable implementation I found.
