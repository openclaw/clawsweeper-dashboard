## Summary

Keep open: the PR has strong proof for the primary ban/delete/unban path, but the latest head still lets scheduled package-ban continuations use the active-owner override after an unban, so it is not merge-ready.

## What This Changes

The PR adds package sanction handling so personal packages are hidden or restored with user bans/deletions, publish tokens are revoked, org-owned packages are preserved, and moderation specs/UI copy/tests are updated.

## Best Possible Solution

Land the package sanction behavior only after scheduled continuations are stale-safe across unban/reban boundaries and maintainers accept the personal-versus-org ownership policy.
