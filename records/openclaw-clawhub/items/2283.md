## Summary

Keep this PR open. The direction is useful, but it is not merge-ready because the patch appears to cover only future ban/delete events, the live PR is currently conflicting with main, and the PR body still has test/CI evidence rather than live behavior proof.

## What This Changes

The PR hides personal package resources when owners are banned or deactivated, revokes package publish tokens, tracks package soft-delete reasons, and restores only matching ban-hidden packages on unban.

## Best Possible Solution

Extend the fix so existing sanctioned personal owners are covered by a cursor-based admin/backfill sweep or an owner-status visibility guard, then rebase and prove the live moderation flows before merge.

## Reproduction Assessment

Yes at source level. A package row whose owner already had deletedAt or deactivatedAt before deployment never enters the new event batches, while PR-head public package surfaces still check package soft deletion rather than owner sanction state.

## Solution Assessment

No, not yet. The event-batch approach is a good core shape, but the best solution also needs pre-existing sanctioned-owner coverage, conflict resolution against current main, and redacted live proof.
