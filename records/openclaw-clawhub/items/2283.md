## Summary

Keep this PR open: the package sanction direction is useful, but the latest head still has a P1 stale-continuation path where a delayed package-ban page can mutate package visibility and publish-token state after the owner has been unbanned.

## What This Changes

The branch adds Convex package sanction batches, schema/tests/spec/UI copy so personal packages are hidden or restored with user bans/deletions, package publish tokens are revoked, and org-owned packages are preserved.

## Best Possible Solution

Land the package sanction behavior after scheduled continuations are stale-safe across unban/reban boundaries and maintainers accept the personal-versus-org ownership policy.

Yes, at source level: the latest head carries `allowActiveOwnerBeforeCommit` onto a cursor-based second page with an active owner, and the shared scheduler copies that flag into continuations. I did not run tests because this was a read-only review.

No. The proposed sanction direction is plausible, but the implementation needs a stale-safe continuation guard and explicit maintainer policy acceptance before it is the narrow maintainable solution.
