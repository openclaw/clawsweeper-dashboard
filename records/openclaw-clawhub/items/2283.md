Full review comments:

- **[P1] Stop ban batches after a matching unban:** `convex/packages.ts:2951-2954`
  - body: The scheduled continuation keeps `allowActiveOwnerBeforeCommit: true`, so if an admin unbans the user while a multi-page package ban is still queued, `owner.deletedAt` is undefined and this branch still treats the owner as an in-flight ban. Please stale-stop once the active pre-commit window is over, or move the package batch after the user ban timestamp is committed, and add a regression for unban between queued pages.
