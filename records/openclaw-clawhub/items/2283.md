## Summary

The PR is useful but still not merge-ready: the latest head keeps the active-owner package-ban override on scheduled pages, so a delayed continuation can enforce a ban after an admin has already unbanned the owner.

## What This Changes

The PR adds package/plugin sanction handling for banned or deactivated personal owners, including package soft-delete reasons, publish-token revocation, matching unban restore, org-owned preservation, tests, specs, UI copy, and changelog text.

## Best Possible Solution

Land the package sanction behavior only after scheduled continuations are stale-safe across unban/reban boundaries and maintainers accept the personal-versus-org ownership policy.

## Reproduction Assessment

Yes, at source level: the PR-head handler accepts an active owner when allowActiveOwnerBeforeCommit is true, and the scheduler preserves that flag on cursor continuations. I did not execute tests because this was a read-only review.

## Solution Assessment

No. The policy direction is plausible, but the implementation needs a stale-safe continuation guard and explicit maintainer acceptance of the personal-versus-org sanction boundary before it is the narrow maintainable solution.
