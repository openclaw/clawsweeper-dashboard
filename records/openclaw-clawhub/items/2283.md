Keep open: the PR is useful security hardening, but latest head still has a package-ban pagination race that can fail open for owners with more than one package batch, leaving later personal packages and publish tokens active.

## What This Changes

## Best Possible Solution

Repair package ban pagination so every continuation runs after or safely through the committed ban state, add multi-page regression coverage, then land after maintainer policy sign-off on personal versus org package sanctions.
