## Summary

Keep open: current main still lacks package cleanup for banned/deactivated personal owners, and this PR is a plausible fix, but it should not merge until the restore audit actor and real behavior proof blockers are addressed.

## What This Changes

The PR hides and restores personal package/plugin rows during ban, account deletion, and unban flows, revokes package publish tokens, adds `packages.softDeletedReason`, and updates tests, specs, and changelog.

## Best Possible Solution

Keep the package cleanup design, but thread the admin unban actor through package restore batches and require redacted live proof before merge.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source-level yes: current main's user ban/delete/unban paths do not clean up packages, and the submitted diff exposes the restore audit misattribution without needing a runtime setup.

## Solution Assessment

Is this the best way to solve the issue? No, not as submitted: the batch cleanup approach is narrow and maintainable, but restore audits need the admin actor instead of the restored owner and the PR still needs real behavior proof.
