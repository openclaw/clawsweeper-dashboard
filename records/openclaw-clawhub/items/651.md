## Repair Work Prompt

Fix the ClawHub CLI raw `fetch failed` diagnostics reported here and in https://github.com/openclaw/clawhub/issues/861. Add centralized, redacted transport-error classification in the shared HTTP/error-formatting boundary so Undici fetch failures, DNS/TLS/certificate/proxy errors, request timeouts/aborts, and Bun curl failures produce actionable guidance without changing successful request semantics. Preserve `/api/v1/whoami` verification before writing login config, preserve HTTP status/rate-limit/retry behavior, and do not add new config flags. Add focused tests for HTTP/UI formatting plus search/login failure paths and update troubleshooting docs if user-facing guidance changes. Do not expose tokens, Authorization headers, callback URLs, proxy credentials, or private endpoints in diagnostics.
