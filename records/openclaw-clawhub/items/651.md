## Summary

Keep open. Current main still has a clear source-level path where CLI search and token-login transport failures are rendered through the generic formatter as raw `fetch failed`, while the merged proxy and Node 22 fixes only address specific root causes.

## What This Changes

CLI search reports a generic `fetch failed` error in affected environments, with follow-up comments showing token login can fail the same way.

## Best Possible Solution

Add centralized, redacted CLI transport diagnostics in the shared HTTP/error-formatting boundary so DNS, TLS/certificate, proxy, timeout/abort, and curl failures produce actionable guidance without changing request semantics.

## Reproduction Assessment

Yes for a source-level path: make `fetchWithTimeout` receive an Undici-style `TypeError('fetch failed')` or make the Bun curl path fail, then `cmdSearch`/`cmdLogin` still render `formatError(error)`. The original reporters' exact network environments were not reproduced.

## Solution Assessment

Yes. A shared HTTP/error-formatting repair is the narrowest maintainable solution because search, login, install, publish, and related CLI commands already converge through this boundary.
