## Summary

Keep open. Current main still allows shared CLI transport failures to surface as raw `fetch failed` messages in search and login paths, and ClawHub cleanup policy keeps issues open rather than auto-closing them; the related login report at https://github.com/openclaw/clawhub/issues/861 is also still open.

## What This Changes

The issue reports `clawhub search "web"` failing with the raw CLI error `fetch failed`.

## Best Possible Solution

Add centralized, redacted CLI transport diagnostics at the shared HTTP/error-formatting boundary, with focused tests and troubleshooting docs for search and login failures.

## Reproduction Assessment

Yes for the source-level raw diagnostics path: if `apiRequest` receives a fetch/curl transport failure, search and token login still print only `formatError(error)`, which returns `Error.message`. The reporters' exact network root cause was not reproduced from the available environment data.

## Solution Assessment

Yes. A shared HTTP/error-formatting repair is the narrowest maintainable solution because search, login, install, publish, and related CLI commands already converge through the same request and formatter code.

## Repair Work Prompt

Fix the ClawHub CLI raw `fetch failed` diagnostics path without changing registry semantics. Add centralized, redacted transport-error classification at the shared HTTP/error-formatting boundary so DNS, TLS/certificate, proxy, timeout/abort, Undici, and curl failures produce actionable guidance for commands such as search and login. Preserve `/api/v1/whoami` token verification, retries, HTTP status/rate-limit handling, successful requests, and Bun curl behavior. Do not add a new config option and do not assume every failure is proxy-related. Redact Authorization headers, API tokens, callback URLs, proxy credentials, and credentials embedded in URLs or environment variables. Add focused regression coverage for search/login or shared formatter behavior, and update troubleshooting docs if user-facing guidance changes. Related reports: https://github.com/openclaw/clawhub/issues/651 and https://github.com/openclaw/clawhub/issues/861; prior partial fixes: https://github.com/openclaw/clawhub/pull/363 and https://github.com/openclaw/clawhub/pull/245.
