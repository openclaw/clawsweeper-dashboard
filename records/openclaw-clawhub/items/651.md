## What This Changes

The issue reports `clawhub search "web"` failing with a raw `fetch failed` error in a user terminal.

## Best Possible Solution

Add centralized, redacted CLI transport diagnostics at the shared HTTP/error boundary for DNS, TLS/certificate, proxy, timeout/abort, connection, and curl failures while preserving request semantics.

## Solution Assessment

Yes. A shared HTTP/error-formatting repair is the narrowest maintainable path because search, login, install, publish, and related CLI commands already converge through the same request and formatting code.
