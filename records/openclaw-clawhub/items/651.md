## Repair Work Prompt

Implement a focused ClawHub CLI diagnostics fix for this issue. The observable problem is that `clawhub search "web"` and `clawhub login --token ...` can surface low-level transport failures as raw `fetch failed` output. Add centralized, redacted diagnostics in the shared HTTP/error formatting path so DNS, TLS/certificate, proxy, timeout/abort, and Bun curl failures produce actionable guidance while preserving existing HTTP status errors, retry/rate-limit behavior, successful requests, and proxy support from #363. Do not hard-code every `fetch failed` as a proxy problem, and do not print Authorization headers, API tokens, token query parameters, or credentials embedded in proxy URLs/environment variables. Add focused tests for formatter/HTTP diagnostic behavior. Do not make unrelated CLI UX changes or update release metadata/CHANGELOG unless maintainers explicitly request it; note the user-visible diagnostics change in the PR summary.
