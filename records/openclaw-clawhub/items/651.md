## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has a valid CLI diagnostics gap where search and login can show raw transport failures as `fetch failed` instead of actionable network guidance.

## What This Changes

The issue reports `clawhub search "web"` failing with `fetch failed`, with follow-up comments showing the same symptom for search and token login.

## Best Possible Solution

Add centralized, redacted CLI transport diagnostics at the shared HTTP/error-formatting boundary so search, login, install, publish, and related commands provide actionable guidance without changing request semantics.
