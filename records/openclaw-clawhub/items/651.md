## What This Changes

## Best Possible Solution

Add centralized, redacted CLI transport diagnostics in the shared HTTP/error-formatting boundary so DNS, TLS/certificate, proxy, timeout/abort, and curl failures produce actionable guidance while preserving request and token-verification semantics.

Yes, at source level: inject an Undici-style `TypeError('fetch failed')` or a nonzero curl transport failure into the HTTP path used by `cmdSearch` or token login, and current main prints `formatError(error)`. I did not recreate the reporters' exact network environments.

## Solution Assessment

Yes. A shared, redacted HTTP/error-formatting repair is the narrowest maintainable solution because search, login, install, publish, package, and sync already converge through that boundary.
