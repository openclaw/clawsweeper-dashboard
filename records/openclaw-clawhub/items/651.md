## Summary

Keep open. Current main still routes CLI search and token-login transport failures through the generic formatter as raw `Error.message`, so proxy and Node 22 fixes are present but the actionable diagnostics gap remains.

## What This Changes

The issue reports `clawhub search "web"` failing with raw `fetch failed` output, with comments adding the same symptom for token login and search.

## Best Possible Solution

Add centralized, redacted CLI transport diagnostics in the shared HTTP/error-formatting boundary so DNS, TLS/certificate, proxy, timeout/abort, and curl failures produce actionable guidance while preserving request semantics.

## Solution Assessment

Yes. A shared HTTP/error-formatting repair is the narrowest maintainable solution because search, login, install, publish, and related CLI commands already converge through this boundary.
