## What This Changes

Investigate why ClawHub CLI search and login commands fail with a generic `fetch failed` error on affected macOS environments.

## Best Possible Solution

Consolidate the shared work under https://github.com/openclaw/clawhub/issues/861, add sanitized nested network-cause output for Node failures across both search and login, and validate it in a real affected macOS environment without weakening token verification.

No, there is no high-confidence live reproduction for the reporters' current environments. Source inspection does confirm that a Node fetch failure can surface only as the observed generic message, while the exact underlying network cause remains unknown.

## Solution Assessment

Yes, the narrow maintainable direction is to expose a sanitized nested network cause and retain existing proxy support rather than assume every failure needs the same transport workaround. A real macOS reproduction should verify the final diagnostic and any targeted transport repair.
