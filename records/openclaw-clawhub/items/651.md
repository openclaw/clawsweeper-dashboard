Add centralized, redacted CLI transport diagnostics in the shared HTTP/error-formatting boundary, with search/login-relevant guidance and regression coverage while preserving request semantics.

## Repair Work Prompt

Fix the ClawHub CLI `fetch failed` diagnostics bug reported by this issue and related fetch-failure reports. Add a centralized, redacted transport-error formatter/classifier in the shared CLI HTTP/error boundary so Undici/Node fetch failures, abort/timeouts, DNS/TLS/proxy errors, and Bun curl failures produce actionable guidance instead of raw `fetch failed` or `curl failed`. Keep existing request semantics, retry/rate-limit handling, HTTP status messages, proxy support, auth token behavior, and registry URL handling unchanged. Do not assume every failure is a proxy issue. Redact Authorization headers, API tokens, callback URL tokens, query/hash token values, proxy credentials, and private endpoint details. Add focused regression coverage for Node fetch errors, Bun curl stderr/status failures, `formatError`/CLI output, `cmdSearch`, and token-login/whoami failure handling where practical. Update public troubleshooting/docs only if the user-facing guidance changes.
