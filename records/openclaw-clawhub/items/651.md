## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still routes CLI transport failures from search/login through a raw shared formatter; the merged proxy support from #363 addresses one root cause but not the recurring diagnostics gap.

## What This Changes

The issue reports `clawhub search "web"` failing on macOS with only `fetch failed` output, with later comments showing the same symptom for search and token login.

## Best Possible Solution

Add centralized, redacted CLI transport diagnostics at the shared HTTP/error boundary for DNS, TLS/certificate, proxy, timeout/abort, connection, and curl failures while preserving existing proxy support and HTTP semantics.

## Reproduction Assessment

Yes for the observable diagnostics gap: source inspection shows transport-level `Error("fetch failed")` or curl stderr can reach `cmdSearch`/`cmdLogin` and be rendered unchanged by `formatError`. The exact reporter network root cause is not reproducible from the issue alone.

## Solution Assessment

Yes. A shared HTTP/error-formatting fix is the narrowest maintainable solution because search, login, and many other CLI commands already converge there; special-casing only search or only proxies would leave the same raw output elsewhere.
