## Summary

Current main still contains the PAT fallback, so this PR is not obsolete or implemented on main; the patch is a narrow credential-hardening change that should stay open for secops/maintainer review.

## What This Changes

The PR changes `.github/workflows/clawsweeper-dispatch.yml` so the ClawSweeper dispatcher uses only `steps.token.outputs.token` for `GH_TOKEN` instead of falling back to `secrets.OPENCLAW_GH_TOKEN`.

## Best Possible Solution

Land the narrow App-token-only dispatcher change after secops confirms the required GitHub App secret is configured and the PAT fallback is intentionally retired.

## Reproduction Assessment

Not applicable; this is a workflow credential-hardening PR rather than a runtime bug report. Source inspection confirms current main still has the fallback and the PR removes only that fallback expression.

## Solution Assessment

Yes; the PR is the narrowest maintainable way to enforce App-token-only dispatch while preserving the existing empty-token skip behavior. The remaining question is operational/secops confirmation, not a patch defect.
