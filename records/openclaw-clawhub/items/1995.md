## Summary

Keep this PR open: current main still uses the PAT fallback, so the PR is not obsolete or implemented on main. The diff is narrow and looks functionally correct, but it changes workflow credential behavior and should get secops approval plus check/status review before merge.

## What This Changes

The PR changes `.github/workflows/clawsweeper-dispatch.yml` so `GH_TOKEN` uses only `steps.token.outputs.token` instead of falling back to `secrets.OPENCLAW_GH_TOKEN`.

## Best Possible Solution

Land the one-line workflow hardening after secops verifies the GitHub App dispatch path and accepts the PR status checks.

## Reproduction Assessment

Not applicable; this is workflow credential hardening rather than a runtime bug. Source inspection confirms current main still has the fallback expression and the PR removes only that expression.

## Solution Assessment

Yes; changing the `GH_TOKEN` assignment is the narrowest maintainable way to enforce App-token-only dispatch while preserving the workflow's existing empty-token skip behavior.
