## Summary

Keep open: current main still contains the PAT fallback this PR removes, so the change is not already implemented on main. The patch is narrow and appears correct, but it changes workflow credential behavior and should be reviewed by secops before merge.

## What This Changes

The PR updates `.github/workflows/clawsweeper-dispatch.yml` so `GH_TOKEN` comes only from `steps.token.outputs.token`, removing the `secrets.OPENCLAW_GH_TOKEN` fallback.

## Best Possible Solution

Land the one-line workflow hardening after secops verifies the ClawSweeper GitHub App credential path is ready.

## Reproduction Assessment

Not applicable; this is workflow credential hardening rather than a runtime bug report. Source inspection confirms current main still has the fallback and the PR diff removes only that fallback expression.

## Solution Assessment

Yes; using only `steps.token.outputs.token` is the narrowest maintainable way to enforce App-token-only dispatch while preserving the existing no-token skip behavior. The remaining question is operational credential readiness, not a patch defect.
