## Summary

Keep open. Current main still has the PAT fallback, so ClawHub policy does not allow closing this PR as already implemented; the patch itself is narrow credential hardening and should go through secops/maintainer review.

## What This Changes

The PR changes `.github/workflows/clawsweeper-dispatch.yml` so ClawSweeper dispatch uses only the generated GitHub App token instead of falling back to `secrets.OPENCLAW_GH_TOKEN`.

## Best Possible Solution

Land the one-line workflow hardening after secops verifies the GitHub App token can dispatch to `openclaw/clawsweeper` without the PAT fallback.

## Reproduction Assessment

Not applicable; this is workflow credential hardening rather than a runtime bug. Source inspection confirms current main still has the fallback expression, and the supplied PR diff removes only that expression.

## Solution Assessment

Yes; removing the fallback at the `GH_TOKEN` assignment is the narrowest maintainable way to enforce App-token-only dispatch while preserving the workflow's existing empty-token skip behavior.
