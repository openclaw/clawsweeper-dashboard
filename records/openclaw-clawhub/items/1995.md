## Summary

Keep open; current main does not yet contain the requested hardening, and this one-line workflow credential change is secops-owned infrastructure that needs normal maintainer review rather than cleanup closure.

## What This Changes

This PR changes `.github/workflows/clawsweeper-dispatch.yml` so the dispatch step uses only `steps.token.outputs.token` for `GH_TOKEN` instead of falling back to `secrets.OPENCLAW_GH_TOKEN`.

## Best Possible Solution

Land the one-line workflow hardening after secops verifies the GitHub App token can dispatch to `openclaw/clawsweeper` without the PAT fallback.

## Reproduction Assessment

Not applicable; this is workflow credential hardening rather than a runtime bug. Source inspection confirms current main still has the fallback expression and the PR removes only that expression.

## Solution Assessment

Yes; removing the fallback at the `GH_TOKEN` assignment is the narrowest maintainable way to enforce App-token-only dispatch while keeping the script's empty-token skip behavior.
