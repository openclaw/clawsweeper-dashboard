## Summary

Keep open: current main still has the PAT fallback, and this PR is a narrow security hardening change that has not been implemented on main. The diff itself looks correct, but workflow credential changes should be validated by secops before merge.

## What This Changes

The PR changes `.github/workflows/clawsweeper-dispatch.yml` so the ClawSweeper dispatch token comes only from `steps.token.outputs.token` instead of falling back to `secrets.OPENCLAW_GH_TOKEN`.

## Best Possible Solution

Land the one-line workflow hardening after secops verifies the GitHub App dispatch path and accepts the PR status checks.

## Reproduction Assessment

Not applicable; this is workflow credential hardening rather than a runtime bug. Source inspection confirms current main still has the fallback expression while the PR removes only that expression.

## Solution Assessment

Yes; setting `GH_TOKEN` to only `steps.token.outputs.token` is the narrowest maintainable way to enforce App-token-only dispatch while preserving the existing empty-token skip behavior.
