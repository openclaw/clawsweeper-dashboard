## What This Changes

The PR changes `.github/workflows/clawsweeper-dispatch.yml` so the dispatch step uses only `steps.token.outputs.token` instead of falling back to `secrets.OPENCLAW_GH_TOKEN`.

## Best Possible Solution

The end state should be a reviewed dispatcher that uses only the GitHub App token and explicitly skips when that credential is unavailable.

## Reproduction Assessment

Not applicable; this is a credential-policy workflow cleanup rather than a product bug with a failing runtime path. Source inspection verifies current main still has the fallback and the PR removes only that expression.

## Solution Assessment

Yes, as a code change; replacing the env expression with `steps.token.outputs.token` is the narrowest implementation and preserves the existing token creation and no-token skip path. Maintainers still need to confirm the intended credential policy before merge.
