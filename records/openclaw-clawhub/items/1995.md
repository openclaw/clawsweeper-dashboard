## What This Changes

The PR changes `.github/workflows/clawsweeper-dispatch.yml` so ClawSweeper dispatch uses only `steps.token.outputs.token` for `GH_TOKEN`, removing the `secrets.OPENCLAW_GH_TOKEN` fallback.

## Best Possible Solution

Land the one-line hardening after secops confirms the ClawSweeper GitHub App token can dispatch to `openclaw/clawsweeper` and required checks are green.

## Reproduction Assessment

## Solution Assessment

Yes; changing the `GH_TOKEN` assignment is the narrowest maintainable code change for enforcing App-token-only dispatch. The remaining question is operational readiness, not a patch defect.
