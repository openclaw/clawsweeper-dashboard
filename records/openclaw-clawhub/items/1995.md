## What This Changes

The PR changes `.github/workflows/clawsweeper-dispatch.yml` so `GH_TOKEN` uses only `steps.token.outputs.token`, removing fallback to `secrets.OPENCLAW_GH_TOKEN`.

## Best Possible Solution

Land the one-line workflow hardening after secops verifies the GitHub App dispatch path and required PR checks are acceptable.

## Reproduction Assessment

Not applicable; this is workflow credential hardening rather than a runtime bug. Source inspection verifies current main still has the fallback expression and the PR removes only that expression.

## Solution Assessment

Yes; the one-line `GH_TOKEN` assignment change is the narrowest maintainable way to enforce App-token-only dispatch while preserving the existing empty-token skip behavior.
