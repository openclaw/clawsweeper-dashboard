## Summary

Keep this PR open: current main still has the PAT fallback, and the patch is a narrow credential-hardening change rather than redundant. The remaining gate is maintainer/secops confirmation that the GitHub App dispatch path is correctly configured before merge.

## What This Changes

This PR changes `.github/workflows/clawsweeper-dispatch.yml` so dispatcher `GH_TOKEN` comes only from `steps.token.outputs.token`, removing the `secrets.OPENCLAW_GH_TOKEN` fallback.

## Best Possible Solution

Land the one-line fallback removal after secops verifies the ClawSweeper GitHub App secret and installation cover the dispatch target.
