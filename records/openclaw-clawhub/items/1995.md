## Summary

Keep open because current main still contains the PAT fallback that this PR removes. The patch appears correct and security-narrowing, but it should get secops review before merge because it changes workflow credential behavior.

## What This Changes

This PR changes `.github/workflows/clawsweeper-dispatch.yml` so `GH_TOKEN` uses only `steps.token.outputs.token` instead of falling back to `secrets.OPENCLAW_GH_TOKEN`.

## Best Possible Solution

Land the one-line workflow hardening after secops verifies the GitHub App dispatch credential works for `openclaw/clawsweeper`.

## Reproduction Assessment

Not applicable; this is workflow credential hardening rather than a runtime bug. Source inspection confirms current main still has the fallback expression that the PR removes.

## Solution Assessment

Yes; the PR is the narrowest maintainable change to enforce App-token-only dispatch while preserving the existing empty-token skip behavior. The remaining question is operational credential readiness, not a patch defect.
