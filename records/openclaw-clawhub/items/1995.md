## Summary

Keep open: current main still contains the fallback this PR removes, so the PR is not obsolete. The patch is narrow and appears correct, but it changes a security-sensitive workflow credential path covered by secops ownership.

## What This Changes

The PR changes `.github/workflows/clawsweeper-dispatch.yml` so the ClawSweeper dispatch step uses only `steps.token.outputs.token` for `GH_TOKEN` instead of falling back to `secrets.OPENCLAW_GH_TOKEN`.

## Best Possible Solution

Land the one-line workflow hardening if secops confirms the GitHub App dispatch credential is configured and PAT fallback should remain unsupported.

## Reproduction Assessment

Not applicable; this is a workflow hardening PR rather than a bug report. Source inspection confirms current main still has the fallback that the PR removes.

## Solution Assessment

Yes; the patch is the narrowest maintainable way to enforce App-token-only dispatch while preserving the existing no-token skip behavior. The remaining check is operational secops review, not a code defect.
