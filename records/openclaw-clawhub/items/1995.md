Keep open: current main still contains the PAT fallback, and this narrow workflow hardening should receive secops maintainer review before merge.

## What This Changes

The PR changes the ClawSweeper dispatch workflow so `GH_TOKEN` comes only from `steps.token.outputs.token`, removing the `OPENCLAW_GH_TOKEN` PAT fallback.

## Best Possible Solution

Land the App-token-only dispatcher after secops confirms the required GitHub App secret and accepts the credential-boundary change.

## Reproduction Assessment

Not applicable; this is a workflow credential-hardening PR rather than a runtime bug report. Source inspection shows current main has the PAT fallback and the PR diff removes that expression.

## Solution Assessment

Yes; the patch is the narrowest maintainable way to enforce App-token-only dispatch while preserving the existing empty-token skip behavior. The remaining concern is operational review of secret configuration, not a code defect.
