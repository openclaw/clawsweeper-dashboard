## Summary

Keep open: current main still contains the PAT fallback, so this PR is not obsolete or implemented. The one-line workflow hardening looks correct, but it changes a security-sensitive dispatch credential policy and should get maintainer review before merge.

## What This Changes

The PR changes `.github/workflows/clawsweeper-dispatch.yml` so `GH_TOKEN` comes only from the ClawSweeper GitHub App token output instead of falling back to `secrets.OPENCLAW_GH_TOKEN`.

## Best Possible Solution

Merge a reviewed version once maintainers confirm the GitHub App dispatch credential is the intended sole path.

## Reproduction Assessment

Not applicable; this is a workflow credential-policy cleanup rather than a reproducible product bug. Source inspection verifies current main still has the fallback and the PR removes that fallback from the provided patch.

## Solution Assessment

Yes. Removing `secrets.OPENCLAW_GH_TOKEN` from the `GH_TOKEN` expression while preserving the GitHub App token step and empty-token skip path is the narrowest maintainable solution.
