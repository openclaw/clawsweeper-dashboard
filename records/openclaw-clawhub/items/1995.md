## Summary

Keep open: current main still has the PAT fallback, while this PR is a narrow workflow credential-hardening change that needs secops review rather than cleanup closure.

## What This Changes

This PR changes `.github/workflows/clawsweeper-dispatch.yml` so the ClawSweeper dispatch step uses only the GitHub App token output instead of falling back to `secrets.OPENCLAW_GH_TOKEN`.

## Best Possible Solution

Land the one-line workflow hardening after secops verifies the GitHub App token can dispatch to `openclaw/clawsweeper` without the PAT fallback.

## Reproduction Assessment

Not applicable; this is workflow credential hardening rather than a runtime bug. Source inspection and the PR files API provide a high-confidence configuration check: main uses the fallback and the PR removes only that expression.

## Solution Assessment

Yes; assigning `GH_TOKEN` only from `steps.token.outputs.token` is the narrowest maintainable way to enforce App-token-only dispatch while keeping the workflow's empty-token skip behavior.
