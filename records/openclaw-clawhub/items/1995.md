## Summary

Keep open: current main still has the PAT fallback, so this PR is not implemented on main. The patch is a narrow credential hardening change, but workflow/secops owners should confirm the GitHub App dispatch path is operational before merge.

## What This Changes

The PR changes `.github/workflows/clawsweeper-dispatch.yml` so `GH_TOKEN` uses only `steps.token.outputs.token` instead of falling back to `secrets.OPENCLAW_GH_TOKEN`.

## Best Possible Solution

Land the one-line workflow hardening after workflow/secops owners confirm the GitHub App dispatch credential can reach `openclaw/clawsweeper`.
