## Summary

Keep open for maintainer/operator handling. Current main still makes owner/API deletion a soft-delete, retained version rows block republishing the same version, and the existing hard-delete path is admin-only and destructive; ClawHub cleanup policy also keeps issues open.

## What This Changes

The requester asks maintainers to hard-delete all versions of JasonStarlight/phosor-ai so the skill can be republished from a clean v1.0.0.

## Best Possible Solution

A maintainer should verify ownership and purge scope, then use the existing audited admin hard-delete path or design a documented owner reset/version-purge workflow if ClawHub wants to support this regularly.

## Reproduction Assessment

Yes for the source-level behavior: current main routes owner/API deletion through soft-delete, keeps `skillVersions`, and rejects republishing an existing version. The production ownership check and actual destructive purge cannot be reproduced from the checkout.

## Solution Assessment

No automated source fix is the best immediate solution. The current admin-only hard-delete path may be the right tool, but a maintainer must first verify requester ownership and decide the production purge/retention scope.
