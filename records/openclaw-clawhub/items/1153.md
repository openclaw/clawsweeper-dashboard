## Summary

Keep open for maintainer/operator handling. Current main still exposes owner deletion as soft-delete, existing version rows still block republishing the same version, and the hard-delete path is admin-only and destructive.

The issue asks maintainers to hard-delete all published versions of JasonStarlight/phosor-ai so the owner can republish a clean v1.0.0.

## Best Possible Solution

A maintainer should verify requester ownership and purge scope, then run an audited admin purge or design a documented owner reset/version-purge workflow if this should be supported regularly.

## Solution Assessment

No automated source fix is the best immediate solution. This is a destructive production data operation, so maintainer/operator verification and an explicit policy/scope decision should come before any code change.
