## Summary

Keep open. Current main still lacks the requested soul-search parity, and the PR head still has two merge-blocking issues: soul fallback is bypassed when embedding generation fails, and fallback scans `souls.by_updated` with a post-query soft-delete filter even though current main has an active-soul index. The earlier Greptile comment usefully confirms the intended parity shape, but the later review findings remain valid against the current PR head.

## What This Changes

The PR branch changes `convex/search.ts` and `convex/search.test.ts` to add soul lexical fallback, merge/deduplication, shared boosted scoring, incremental soul embedding hydration, and related tests.

## Best Possible Solution

Land an updated soul-search parity change on current main that mirrors the current `searchSkills` flow: embedding failures fall through to lexical fallback, soul fallback uses `souls.by_active_updated` without `.filter()`, vector and fallback matches are deduplicated, scoring is shared, hydration is incremental, and tests cover both successful vector searches and embedding-failure fallback.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Use the active soul index for fallback:** `convex/search.ts:533-535`
  - body: `lexicalFallbackSouls` queries `souls.by_updated` and then filters `softDeletedAt`, so the fallback reads rows the existing `souls.by_active_updated` index can exclude. Query `by_active_updated` with `q.eq("softDeletedAt", undefined)` and update the test fake to expect that index.
  - confidence: 0.93
- **[P2] Run soul fallback after embedding failures:** `convex/search.ts:401-403`
  - body: The new fallback only runs after vector search succeeds. If `generateEmbedding` throws, this catch still returns `[]`, so exact slug/name searches fail during embedding outages while skill search falls back correctly. Mirror the nullable-vector flow from `searchSkills` and add a regression test.
  - confidence: 0.88
