Keep open. Current main has ordinary skill badges and suspicious-skill manual overrides, not a paid expedited human-review certificate; this is security-sensitive product/payment policy work and ClawHub issue cleanup policy keeps issues open.

## What This Changes

Request to add a paid expedited human-review pipeline and public certificate or badge for skills that pass manual security review.

## Best Possible Solution

Maintainers should first define whether paid expedited human review should exist and what the trust signal means, then implement payment state, review queues, moderator workflow, public badge/API display, audit trails, appeals, revocation, and docs as one coherent feature.
