## Summary

Current main does not implement publisher-wide installation: the public CLI installs one skill slug, while publisher profiles separately expose paginated skill and plugin catalogs. The feature remains useful, but its combined skill/plugin scope needs maintainer decisions about executable-plugin trust, selection, confirmation, and partial failures; the collaborator-authored issue must remain open.

## What This Changes

Add a publisher-scoped command or flow that installs all skills and plugins published by that publisher.

## Best Possible Solution

Define a publisher install plan that enumerates eligible items, separates skills from executable plugins, previews versions and moderation status, requires explicit plugin confirmation, and reports per-item outcomes without weakening existing install checks.

## Reproduction Assessment

Not applicable; this issue requests a new publisher-wide installation capability, and current source and docs consistently show only item-scoped installation.

## Solution Assessment

Unclear as stated; existing publisher catalogs are the right foundation, but combining skills and executable plugins into one implicit install action is unsafe without explicit selection and trust semantics.
