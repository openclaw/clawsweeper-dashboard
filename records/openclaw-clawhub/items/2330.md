## Summary

Keep open: this is a contributor-authored feature request, and current main still only supports per-skill/per-package install or download flows rather than publisher-scoped batch installation across skills and plugins. The remaining work is a product/API contract decision, so it should not be auto-closed or queued as a narrow bug fix yet.

## What This Changes

Request support for installing every skill and plugin published by a given publisher in one flow, similar to `npx skills add mattpocock/skills`.

## Best Possible Solution

Define a first-class publisher-scoped catalog and install contract, then implement CLI/OpenClaw support that preserves per-item safety checks, clear prompts, lockfile metadata, and partial-failure reporting.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection confirms current main supports single-target skill install plus package/plugin browsing or download rather than publisher-wide batch install.

## Solution Assessment

Unclear until maintainers choose the product contract. The narrowest maintainable path is an explicit publisher-scoped API and install UX, not client-side installation of arbitrary search results.
