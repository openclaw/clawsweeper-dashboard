## Summary

This should stay open: current main still installs one skill slug at a time, package/plugin support is browse/download-oriented rather than publisher-wide install, and the collaborator-authored issue already carries an explicit product-decision signal.

## What This Changes

Request support for installing every skill and plugin published by a selected publisher, similar to a publisher/repository-wide add command.

## Best Possible Solution

Design a publisher-scoped preview flow with explicit package-family filters, confirmation, and per-item safety checks, then route each selected item through the appropriate existing skill or plugin installer with clear partial-failure reporting.

## Reproduction Assessment

Not applicable: this is a new publisher-wide installation capability request, not a report of broken documented behavior. Source inspection shows the current install path accepts one skill slug at a time and package commands stop at artifact download/verification.

## Solution Assessment

No, not as an unqualified install-all shortcut. The safer solution is a family-aware, previewable flow that preserves package-specific trust, moderation, compatibility, and install checks.
