## Summary

Keep open: current main supports single-skill install, installed-skill update-all, and package browsing/download/inspect, but it does not provide a publisher-scoped batch install flow for all skills/plugins. This is a new product/API capability with trust, moderation, selection, and partial-failure semantics to settle before implementation.

## What This Changes

Request a CLI/API flow to install all skills/plugins published by a selected publisher.

## Best Possible Solution

Define the publisher-scoped catalog/install contract first, then add an indexed owner/publisher list API and CLI/OpenClaw install flow that runs every item through existing trust, moderation, compatibility, and lockfile handling with clear partial-failure reporting.

## Reproduction Assessment

Not applicable. This is a feature request, and source/docs inspection shows current main only documents and implements single-skill install, installed-skill update-all, and package browse/download/inspect flows.

## Solution Assessment

Unclear until maintainers choose the product contract. The narrow maintainable direction is a first-class publisher-scoped catalog/install design rather than installing ad hoc client-side search results.
