## Summary

Current main supports installing one skill slug at a time, not a publisher’s complete catalog. This collaborator-authored request remains a valid but explicitly product-gated feature because aggregate installation needs defined membership, trust, package-family, update, and partial-failure semantics.

## What This Changes

Add a CLI or package workflow that installs all skills or plugins published by one ClawHub publisher.

## Best Possible Solution

Define an explicit curated bundle or collection contract that records package membership and versions, then let the CLI install that aggregate while preserving package-level trust, audit, and update visibility.

## Reproduction Assessment

Not applicable; this requests a new aggregate installation capability rather than reporting a failure in existing behavior. Source inspection confirms the present CLI intentionally accepts one skill slug.

## Solution Assessment

No, not as an implicit install-everything shortcut. A curated bundle or collection is safer because it makes package membership, trust, versions, and future additions explicit.
