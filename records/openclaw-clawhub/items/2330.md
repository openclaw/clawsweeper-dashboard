## Summary

Current main still installs one skill slug at a time, while publisher profiles expose paginated skill and plugin catalogs. The requested bulk-install capability remains unimplemented, and its combined skill/plugin scope requires explicit maintainer product direction; the collaborator-authored issue should remain open.

## What This Changes

Add a command or selector that installs every skill or plugin published by one publisher.

## Best Possible Solution

Add an explicit publisher-scoped bulk command that previews the resolved package set, clearly separates or scopes skills versus plugins, preserves every existing per-item safety gate, and reports partial failures without silently undoing successful installs.

Not applicable; current main clearly accepts one skill slug per installation, and this issue requests a new publisher-wide capability rather than reporting broken established behavior.

## Solution Assessment

Unclear; publisher-wide selection is reasonable, but maintainers must decide whether skills and plugins share one command and define preview, trust, confirmation, and partial-failure semantics.
