## Summary

Keep open: current main still installs skills one slug at a time, package/plugin support is browse/download/verify oriented rather than publisher-wide install, and this collaborator-authored feature request carries an explicit product-decision signal.

## What This Changes

The issue asks for a workflow to install every skill and plugin published by a given publisher, similar to installing all skills from a GitHub skill repo.

## Best Possible Solution

Design a publisher-scoped preview flow with package-family filters, explicit confirmation, per-item safety checks, and clear partial-failure reporting through the appropriate skill or plugin installer.

## Reproduction Assessment

Not applicable: this is a new publisher-wide installation capability request, not a report of broken documented behavior. Source inspection shows current main accepts one skill slug at a time and package commands stop at artifact browse/download/verification.

## Solution Assessment

No, not as an unqualified install-all shortcut. The safer solution is a family-aware, previewable flow that preserves package-specific trust, moderation, compatibility, and install checks.
