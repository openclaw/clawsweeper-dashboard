## Summary

Current main still installs one skill slug at a time and has no publisher-wide skill/plugin installation workflow. The request fits the repository’s CLI-ergonomics direction, but the collaborator-authored, product-labeled issue needs maintainer decisions about mixed package types and bulk-install safety before implementation.

## What This Changes

Add a publisher-scoped CLI command that installs all skills and plugins from that publisher.

## Best Possible Solution

Provide an explicit publisher bulk-install workflow that previews the complete eligible catalog, allows package-type selection, preserves per-package trust and safety checks, and clearly reports skipped or failed packages.

## Reproduction Assessment

Not applicable; current main intentionally installs one skill slug at a time, and the issue requests a new publisher-wide capability rather than reporting broken behavior.

## Solution Assessment

The publisher-scoped direction is reasonable, but the best command shape remains undecided because mixed skill/plugin selection, confirmation, safety gating, and partial failures need an explicit product contract.
