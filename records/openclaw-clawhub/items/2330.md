## Summary

Keep open: current main still only supports one-at-a-time skill install plus package/plugin browse, inspect, download, and verify surfaces, not a publisher-wide installer; the collaborator-authored request also carries an explicit product-decision signal.

## What This Changes

The issue asks for a CLI capability to install all skills or plugins published by a given publisher in one operation.

## Best Possible Solution

Design a publisher-scoped preview flow with family filters, explicit confirmation, per-item safety checks, and clear partial-failure reporting, then implement it through the appropriate skill/plugin installer paths after maintainers choose the command contract.

## Reproduction Assessment

Not applicable: this is a new publisher-wide installation capability request, not a report of broken documented behavior. Source inspection shows current main accepts one skill slug at a time and exposes package browse/download/verify rather than publisher-wide install.

## Solution Assessment

No, not as an unqualified install-all shortcut. The safer solution is a family-aware, previewable flow that preserves package-specific trust, moderation, compatibility, and install checks.
