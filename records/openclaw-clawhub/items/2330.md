## Summary

Current main still installs one skill slug at a time and has no publisher-wide installation path. Publisher catalogs now include both skills and plugins, but those package families have distinct trust and compatibility requirements; because this collaborator-authored request also carries an explicit product-decision signal, it should remain open for maintainer direction.

## What This Changes

Add a publisher-scoped operation that installs all skills and plugins published by that publisher.

## Best Possible Solution

Provide a publisher-scoped preview with explicit package-family filters and confirmation, then route each selected item through its established installer and report per-package successes, skips, and failures.

## Reproduction Assessment

Not applicable; this requests a new publisher-wide installation capability rather than reporting broken documented behavior. Source inspection clearly establishes that current main accepts one skill slug per installation.

## Solution Assessment

No, not as an unqualified mixed-package install-all shortcut. The safer maintainable path is a previewable, family-aware flow that preserves each package type's existing trust and compatibility checks.
