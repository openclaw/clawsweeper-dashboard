## Summary

Keep open: current main still supports single-item skill install and catalog browsing, but it does not define a publisher-scoped batch install contract for skills and plugins; the existing labels and source shape point to a product/API decision before implementation.

## What This Changes

Add support for installing every skill and plugin published by a given publisher, similar to `npx skills add mattpocock/skills`.

## Best Possible Solution

Define a first-class publisher-scoped catalog and install contract, then add CLI/OpenClaw support that preserves per-item safety checks, confirmation, lockfile metadata, and clear partial-failure reporting.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection shows current main supports single-target install plus catalog browsing rather than publisher-wide batch install.

## Solution Assessment

Unclear until maintainers choose the product contract. The narrowest maintainable path is an explicit publisher-scoped API and install UX, not client-side installation of arbitrary search results.
