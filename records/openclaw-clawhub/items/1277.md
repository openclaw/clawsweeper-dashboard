## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing pull requests that current main definitely implements. The substantive request also remains unresolved: current main still flags `child_process` execution as `suspicious.dangerous_exec`, has no recognized inline or `HOOK.md` process-spawn allow signal, and only clears a narrower environment-credential static finding when external scanners are clean.

## What This Changes

The issue asks ClawHub to document or implement a trusted suppression mechanism for intentional `child_process` spawns declared by hook metadata or inline allow comments.

## Best Possible Solution

Define an explicit scanner policy for intentional hook process spawns: either document `suspicious.dangerous_exec` as an always-surfaced manual-review signal, or implement a narrow validated allowlist with backend, API/UI, docs, and regression-test coverage.
