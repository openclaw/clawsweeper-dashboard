## Summary

Keep open. The exact live `q=wework` search no longer returned two cards during this review, but both package identities are still live and current main still has no canonical alias or historical-package collapse for plugin catalog results; ClawHub cleanup policy also keeps issues open for maintainer handling.

## What This Changes

The issue asks the plugin catalog and API to collapse duplicate WeWork cards that represent the same underlying plugin through historical or alias slugs.

## Best Possible Solution

Define a canonical package and alias model for plugin identities, keep legacy detail/install locators compatible, collapse browse/search results on that identity, and add WeWork-style backend and frontend regression coverage.

## Reproduction Assessment

Partially. Live production no longer returned two cards for the exact `q=wework` search during this review, but both WeWork records are still live and current main would pass through separate alias-equivalent package records when both enter the same result set.

## Solution Assessment

No current implementation fully solves it. The existing exact package-ID or family/name de-dupe is useful but not sufficient; the maintainable fix needs an explicit canonical alias or lineage policy rather than a query-specific heuristic.
