## Summary

Keep open. The WeWork duplicate still reproduces in production, and current main only de-duplicates exact package family/name matches without a canonical alias layer; this ClawHub issue needs maintainer product/data judgment rather than auto-close.

## What This Changes

The issue asks for plugin catalog and search results to collapse historical or alias-equivalent package records so one plugin does not render as duplicate cards.

## Best Possible Solution

Define a canonical package and alias model, choose or backfill the canonical WeWork identity, preserve legacy detail/install locators, collapse browse and search on canonical identity, and add backend plus frontend regressions.

## Reproduction Assessment

Yes. The live production API on 2026-05-02 returns both WeWork package records for `q=wework`, and current-main source inspection shows no alias-equivalent de-duplication path.

## Solution Assessment

Unclear. The requested alias collapse direction is valid, but the best implementation should be a maintainer-approved canonical identity and data-cleanup path rather than an ad hoc frontend or search-only filter.
