## Summary

Keep open. The original exact production `q=wework` duplicate card path did not reproduce in my current check, but production still has two live WeWork package identities and current main has no canonical alias collapse for package records.

## What This Changes

The issue asks for plugin catalog browse/search to collapse historical or alias-equivalent WeWork package records into one canonical card.

## Best Possible Solution

Define a canonical package and alias policy, choose the canonical WeWork record, preserve any required legacy locators, collapse browse/search on canonical identity, and add WeWork-style backend/frontend regression coverage.

## Reproduction Assessment

No high-confidence exact UI reproduction was established: current production `q=WeWork` returned only one WeWork search row. Source and production detail endpoints still show two live package identities with no alias-collapse layer.

## Solution Assessment

Unclear as an automated fix. A display-name or runtime-name heuristic could hide legitimate packages, so the safer solution is a maintainer-approved canonical identity and data-cleanup policy.
