## Summary

Keep open. The exact production `q=wework` duplicate no longer reproduces today, but both WeWork package records still resolve and current main still has no canonical package alias/merge semantics; ClawHub cleanup policy also keeps issues open unless a PR is certainly implemented on main.

## What This Changes

The issue asks the plugin catalog to collapse historical or alias-equivalent WeWork package records so search/list views do not render duplicate cards.

## Best Possible Solution

Define a canonical package identity and legacy alias policy for plugin packages, repair the duplicate WeWork records accordingly, and add backend/frontend regression coverage that preserves valid legacy locators without listing duplicate catalog cards.

## Reproduction Assessment

No for the exact live `q=wework` production query today: it returns only one WeWork card. At source level, the alias-collapse gap is clear because both package records remain live and current search/list paths do not canonicalize alias-equivalent package identities.

## Solution Assessment

Unclear. A UI-only or search-only collapse is not the best fix; the maintainable path needs a maintainer-approved canonical identity and data repair policy before implementation.
