## Summary

Keep open. The duplicate WeWork package identity still exists in production browse results, current main lacks a canonical package/alias collapse layer for plugin catalog rows, and ClawHub cleanup policy keeps issues open for maintainer judgment.

## What This Changes

The issue reports duplicate WeWork plugin catalog cards caused by historical or alias package records appearing as separate plugin results.

## Best Possible Solution

Define canonical package alias semantics, select the canonical WeWork identity, preserve intended legacy locators, repair the production data, and add backend/frontend regression coverage for browse and search.

## Reproduction Assessment

Yes. The live production plugin browse API still returns both WeWork package records when paginated, and current main’s source shows no canonical alias collapse that would prevent such duplicate catalog cards.

## Solution Assessment

Unclear. A catalog-only heuristic is not the best maintainable fix until maintainers decide the canonical package identity and legacy locator/backfill policy.
