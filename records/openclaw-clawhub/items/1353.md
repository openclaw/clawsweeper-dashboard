## Summary

Keep open. The exact WeWork search duplicate is not showing in production today, but current main still lacks a canonical package/alias collapse for historical package records, and both WeWork detail records remain live.

## What This Changes

The issue asks the plugin catalog to collapse historical or alias-equivalent package records so one plugin release lineage appears as one card.

## Best Possible Solution

Define a canonical package/alias policy, repair or alias the duplicate WeWork records with compatibility for legacy locators, and add backend plus frontend regression coverage for alias-equivalent catalog results.

## Solution Assessment

Unclear until maintainers choose the canonical package and compatibility policy. A narrow code-only de-dupe by display name, source repo, or runtime-ish similarity would be risky without an explicit alias model or data repair plan.
