## Summary

Keep open. Production still returns duplicate WeWork plugin records, and current main still lacks a canonical package/alias collapse in plugin browse/search; the new package identity repair mutation is a useful admin/data tool but does not resolve the catalog behavior by itself.

## What This Changes

The issue asks for plugin catalog browse/search results to collapse alias or historical package records so WeWork appears as one card.

## Best Possible Solution

Define canonical package/alias semantics, choose the canonical WeWork identity, preserve intended legacy locators, apply a maintainer-approved data repair or migration, and add backend plus frontend regression coverage.

## Reproduction Assessment

Yes. The live production search API returns both WeWork records, and both detail endpoints resolve to separate code-plugin packages with matching visible metadata but different names/runtime IDs.

## Solution Assessment

Unclear. Current main’s admin identity repair is a useful building block, but the maintainable fix still needs explicit canonical alias/backfill behavior before catalog code collapses records.
