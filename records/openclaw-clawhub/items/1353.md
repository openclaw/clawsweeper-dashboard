## Summary

Keep open. Current production still returns duplicate WeWork plugin records, current main still only de-duplicates exact package family/name records, and ClawHub cleanup policy keeps issues open unless maintainers close them manually.

## What This Changes

The issue asks the plugin catalog/search API to collapse historical alias-equivalent package records such as `@openclaw/wework` and `openclaw-wework` into one canonical plugin card.

## Best Possible Solution

Define canonical package/alias semantics, choose the canonical WeWork identity, preserve unambiguous legacy locators, and collapse browse/search responses on that identity with backend and frontend regression coverage.

## Reproduction Assessment

Yes. The live production search API currently returns both WeWork records, and both detail endpoints resolve to separate code-plugin packages with matching visible metadata but different names/runtime IDs.

## Solution Assessment

Unclear. Current main’s admin identity repair is a useful building block, but the maintainable fix still needs explicit canonical alias/backfill behavior before catalog code collapses records.
