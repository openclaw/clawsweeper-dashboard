## Summary

Keep open. ClawHub cleanup policy keeps issues open, and although the exact visible `q=wework` duplicate did not reproduce in the latest production search, current main still lacks a package alias/canonical identity model while both WeWork locators remain live as separate package records.

## What This Changes

The issue requests collapsing alias-equivalent plugin package records such as `openclaw-wework` and `@openclaw/wework` into one canonical catalog card.

## Best Possible Solution

Define canonical package identity and alias semantics, repair or migrate the duplicate WeWork records, preserve unambiguous legacy locators, and add backend plus frontend regression coverage for catalog/search collapse.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Partially: the exact `q=wework` duplicate did not reproduce in current production, but both detail locators still resolve and current main’s catalog/search code has no alias-equivalent collapse path.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose canonical package and alias/backfill semantics; a narrow display-name or runtime-ID de-dupe would be too risky without that policy.
