## Summary

Keep open. Current main still only de-duplicates plugin search results by exact family/name keys, the browse path returns package candidates directly, and live production still exposes both WeWork package records. Because this is a ClawHub issue and the safe fix needs a maintainer decision on canonical package identity and legacy locator/backfill behavior, it should remain open for maintainer follow-up.

## What This Changes

The issue requests that plugin catalog and search results collapse duplicate cards for the same underlying plugin release lineage, exemplified by WeWork entries for `openclaw-wework` and `@openclaw/wework`.

## Best Possible Solution

Define a maintainer-approved canonical package and alias model, choose or backfill the canonical WeWork identity, preserve compatibility for legacy detail/install locators, collapse browse and search results on that canonical identity, and add backend plus frontend regression coverage for this duplicate pattern.
