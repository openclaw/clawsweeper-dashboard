## Summary

Keep open. The operator cleanup is useful, but the patch can delete a queued ClawScan rescan after a publisher note update because it no longer checks whether the final ClawScan result predates `clawScanNoteUpdatedAt`.

## What This Changes

The PR replaces the redundant VT queue prune helper with an internal operator mutation that cancels queued `vt-update` scan jobs older than a cutoff across skill and package targets.

## Best Possible Solution

Keep the bounded cutoff-based cancellation path, but preserve stale ClawScan-note rescan protection for both skill and package targets, add regression coverage for it, and require redacted live operator proof before merge.

## Reproduction Assessment

Yes, from source inspection. The note update mutations stamp `clawScanNoteUpdatedAt` and enqueue a rescan, while the PR branch deletes queued `vt-update` jobs after only checking for any final ClawScan status.

## Solution Assessment

No. The cutoff-based cancellation is a reasonable emergency operator tool, but the safest implementation keeps the stale-note guard and makes the VT-disagreement cancellation an explicit maintainer-accepted tradeoff.
