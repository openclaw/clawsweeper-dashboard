## What This Changes

The PR adds structured artifact-coverage metadata and floors otherwise benign prompt-backed ClawScan results to review when artifact text is truncated, omitted, or stripped before evaluation.

## Best Possible Solution

Rebase the focused coverage design onto current main, retain one auditable fail-closed rule at the LLM result/moderation boundary, and demonstrate persisted coverage plus the visible review result with real oversized skill and package scans.

## Solution Assessment

Mostly yes. Structured coverage plus a result-boundary floor is narrower and more maintainable than merely increasing prompt limits, but maintainers should explicitly confirm the policy that every stripped hidden comment is material incomplete coverage.
