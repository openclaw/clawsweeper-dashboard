## Summary

Current main still permits a benign prompt-backed ClawScan result when material artifact text was truncated or omitted, so this security fix remains necessary. The proposed trusted coverage floor is directionally sound, but the branch conflicts with newer moderation work, treats every stripped hidden comment as incomplete coverage, and lacks after-fix proof from a real ClawScan run; likely routing candidates are vincentkoc, theonejvo, and Patrick Erichsen.

## What This Changes

The PR records incomplete prompt artifact coverage, floors benign LLM results to review, persists the coverage metadata, and exposes the reason through moderation.

## Best Possible Solution

Rebase the shared coverage computation onto current moderation semantics, enforce the floor only for content maintainers consider materially unreviewed, and demonstrate the persisted and user-visible outcomes with a real scanner execution and representative artifacts.

## Reproduction Assessment

Yes. Current main slices prompt-backed artifact text before evaluation and can accept a benign model response without recording that material text was omitted; the PR's regression tests exercise that exact path.

## Solution Assessment

Partly. Computing coverage from trusted prompt assembly and enforcing it after model parsing is the narrowest sound architecture, but the universal hidden-comment rule and integration with newer moderation semantics require maintainer judgment.
