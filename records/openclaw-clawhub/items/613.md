## Summary

Keep open. Current main still lacks device-flow authentication, the repository policy for ClawHub issues is to keep issues open, and the issue is paired with open, unmerged PR #1867 using closing syntax. The remaining work is also security-sensitive token issuance infrastructure, so it needs maintainer/security review rather than automated cleanup closure.

## What This Changes

Request to add GitHub/RFC 8628-style device-flow login to the ClawHub CLI so headless agents can authenticate without a localhost browser redirect.

## Best Possible Solution

Keep this issue open and use PR #1867 as the current implementation candidate or replacement starting point. A complete end state should add CLI `--device` support plus server-side device-code issuance, polling/token exchange endpoints, an authenticated verification page, secure API-token creation/storage, rate limiting, and tests for success, pending, slow_down/backoff, expiry, denial, replay, and persistence behavior.
