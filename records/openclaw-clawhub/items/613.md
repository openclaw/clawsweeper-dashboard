## Summary

Keep open. Current main still lacks device-flow authentication, and open PR https://github.com/openclaw/clawhub/pull/1867 references this issue with closing syntax but is unmerged, so the issue should remain tracked until that end-to-end auth work or a replacement lands.

## What This Changes

Request device-flow authentication for `clawhub login` so headless agents can authenticate without a localhost browser redirect.

## Best Possible Solution

Land an end-to-end device-flow auth design with CLI `--device` support, server-side code issuance and token polling endpoints, authenticated verification UI, security controls, docs, and regression coverage.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence gap check: current main has token-paste and localhost loopback login only, with no `--device` flag or device-code endpoints.

## Solution Assessment

Yes in direction, but not as a client-only change. The maintainable solution is a complete client, backend, verification-page, and security-controls implementation rather than exposing a flag before the token exchange exists.
