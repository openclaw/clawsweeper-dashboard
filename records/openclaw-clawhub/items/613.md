## Summary

Keep open. Current main still exposes only token-paste or localhost loopback browser login, and open PR #1867 is an unmerged closing implementation candidate, so this remains active maintainer/security work under the ClawHub issue policy.

## What This Changes

Request device-flow authentication so headless agents can log in without a browser-based localhost OAuth callback.

## Best Possible Solution

Keep this issue open and land or replace PR #1867 with an end-to-end device-flow implementation covering the CLI flag, server code issuance, polling/token exchange, verification UI, secure token creation, rate limiting, and protocol-state coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows `clawhub login --no-browser` without `--token` fails, no `--device` flag exists, and the only documented headless flow is manual token paste.

## Solution Assessment

Yes in direction, but not yet as a current-main solution. The maintainable fix needs both CLI support and server-side device-code issuance, verification, polling, token exchange, and security controls.
