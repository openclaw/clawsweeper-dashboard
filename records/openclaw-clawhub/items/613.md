## Summary

Keep open. Current main still lacks device-flow authentication, this ClawHub issue should not be auto-closed, and open PR #1867 is already paired with the request but has not merged.

## What This Changes

The issue requests GitHub-style device-flow authentication so headless agents can run `clawhub login` without a localhost browser callback.

## Best Possible Solution

Keep the issue open and land or replace PR #1867 with an end-to-end device-flow implementation across CLI, server endpoints, verification UI, token issuance, rate limiting, and protocol-state tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: `clawhub login --no-browser` without `--token` fails, there is no `--device` flag, and current docs only describe manual token paste for headless use.

## Solution Assessment

Yes in direction, but only as an end-to-end implementation. A client-only device flow is not sufficient without server-side code issuance, polling/token exchange, verification UI, and security controls.
