## What This Changes

Request device-code authentication so headless agents can log in without a localhost browser callback.

## Best Possible Solution

Keep this issue open and use https://github.com/openclaw/clawhub/pull/1867 as the current implementation candidate or replacement starting point for an end-to-end device-flow auth design.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence gap check: current main has token-paste and localhost loopback login only, with no `--device` flag or device-code endpoints.

## Solution Assessment

Yes in direction, but only as a complete auth feature. A client-only flag is not sufficient because server-side device-code issuance, polling/token exchange, verification UI, and security controls are the core of the requested behavior.
