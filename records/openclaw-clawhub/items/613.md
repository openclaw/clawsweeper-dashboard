## Summary

Keep open. Current main still provides only token-paste or loopback browser auth, and open PR #1867 uses closing syntax for this request but has not merged; ClawHub cleanup policy also keeps issues open.

## What This Changes

Add device-code authentication so headless agents can complete `clawhub login` without a localhost browser OAuth callback.

## Best Possible Solution

Land or replace PR #1867 with an end-to-end device flow covering CLI support, server code issuance, token polling, a verification page, secure token creation, rate limiting, and protocol tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `clawhub login --no-browser` without `--token` fails, there is no `--device` flag, and docs only describe manual token paste for headless login.

## Solution Assessment

Yes in direction. A device-code flow is the maintainable fit for headless login, but it needs complete server-side issuance and security controls rather than only client-side polling.
