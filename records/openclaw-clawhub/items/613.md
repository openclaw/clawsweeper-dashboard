## Summary

Keep open. Current main still has token-paste and localhost loopback browser login only, the open unmerged PR #1867 is paired to this request, and ClawHub cleanup policy keeps issues open rather than auto-closing them.

## What This Changes

The issue requests GitHub-style device flow authentication so headless agents can log in without a localhost browser OAuth callback.

## Best Possible Solution

Land or replace PR #1867 with an end-to-end device-flow implementation covering CLI flags, server issuance and polling endpoints, an authenticated verification page, secure token creation, rate limiting, and protocol-state coverage.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: `clawhub login --no-browser` without `--token` rejects immediately, there is no `--device` flag, and the documented headless path is manual token paste.

## Solution Assessment

Yes in direction, but only as an end-to-end auth feature. A client-only device-flow flag would not be sufficient without server-side code issuance, polling/token exchange, verification UI, and security controls.
