## Summary

Keep open. Current main still lacks device-flow authentication, this is a new auth/token issuance feature rather than a cleanup-close candidate, and the issue is paired with open unmerged PR #1867 that uses closing syntax for the same work.

## What This Changes

Request to add GitHub/RFC 8628-style device-flow authentication for `clawhub login` so headless agents can authenticate without a localhost browser redirect.

## Best Possible Solution

Keep this issue open and use PR #1867, or a replacement, to land an end-to-end CLI plus server device-flow implementation with secure token issuance and protocol coverage.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence gap check: current main has token-paste and localhost loopback login only, with no `--device` flag or device-code endpoints.

## Solution Assessment

Yes in direction, but only as a full auth feature. A maintainable solution needs CLI support plus server-side code issuance, polling/token exchange, verification UI, rate limiting, and security tests; a client-only flag would not solve the reported problem.
