## Summary

Keep open. Current main still has only token-paste and localhost loopback browser login, no device-flow CLI flag or server endpoint surface, and the issue is already paired with open closing PR #1867; ClawHub cleanup policy also keeps issues open unless they are PRs already implemented on main.

## What This Changes

Request device-flow authentication so headless ClawHub CLI users can log in without a local browser OAuth callback.

## Best Possible Solution

Keep this issue open and use PR #1867, or a replacement, to land an end-to-end device-flow implementation covering CLI, server endpoints, verification UI, secure token creation, rate limiting, and protocol-state tests.

## Reproduction Assessment

Yes. Source inspection on current main shows `clawhub login --no-browser` rejects without `--token`, there is no `--device` flag, and the server route surface has no device-code issuance or polling endpoints.

## Solution Assessment

Yes in direction, but only as a full auth feature. The maintainable solution is an end-to-end CLI plus server implementation; a client-only device flag would leave the token exchange and verification flow missing.
