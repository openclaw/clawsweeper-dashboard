## Summary

Keep open. Current main still lacks device-flow authentication, ClawHub cleanup policy keeps issues open, and the issue is paired with open unmerged PR #1867 using closing syntax.

## What This Changes

Add device-flow authentication so headless agents can authenticate without a browser-localhost OAuth callback.

## Best Possible Solution

Keep this issue open and use PR #1867 as the current implementation candidate or replacement starting point for a complete CLI plus server device-flow design.

## Reproduction Assessment

Yes. Current main has a deterministic source-level reproduction: `clawhub login --no-browser` without `--token` fails, no `--device` flag exists, and no device-code endpoints are registered.

## Solution Assessment

Unclear. A GitHub-style device flow is a plausible direction, but the maintainable fix needs both CLI support and server-side device-code issuance, verification, polling/token exchange, rate limiting, and tests.
