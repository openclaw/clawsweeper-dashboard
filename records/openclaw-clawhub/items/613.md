## Summary

Keep open. Current main still only supports token-paste and localhost loopback CLI login, and the open closing PR https://github.com/openclaw/clawhub/pull/1867 remains unmerged, so this ClawHub issue should stay tracked for maintainer/security review.

## What This Changes

Request device-flow authentication for headless ClawHub CLI login so agents can authenticate without a localhost browser callback.

## Best Possible Solution

Land an end-to-end device-flow auth design with CLI --device support, server-side code issuance and polling/token exchange endpoints, an authenticated verification page, secure API-token creation/storage, rate limiting, docs, and regression coverage.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence gap check: current main has token-paste and localhost loopback login only, with no --device flag or device-code endpoint surface.

## Solution Assessment

Yes in direction, but not as a client-only change. The maintainable solution is a complete client, backend, verification-page, and security-controls implementation reviewed together.
