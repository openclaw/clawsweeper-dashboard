## Summary

Keep open. Current main still lacks device-flow authentication, the linked closing PR #1867 is open and unmerged, and this ClawHub issue should remain tracked until an end-to-end auth implementation lands.

## What This Changes

Request device-flow authentication so headless agents can authenticate without a localhost browser OAuth callback.

## Best Possible Solution

Keep this issue open and land or replace PR #1867 with a complete device-flow implementation covering CLI support, server endpoints, verification UI, secure token issuance, rate limiting, and protocol-state tests.

## Reproduction Assessment

Yes. Source inspection on current main shows `clawhub login --no-browser` without `--token` fails, no `--device` flag exists, and no device-code server routes are registered.

## Solution Assessment

Yes in direction, but not as a client-only change. The maintainable solution needs both CLI polling behavior and server-side device-code issuance, verification, token exchange, expiry, denial, replay, backoff, and rate-limit handling.
