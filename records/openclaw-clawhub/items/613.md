## Summary

Keep open. Current main still only supports token-paste or localhost loopback browser login, device-flow auth is a new security-sensitive auth/token issuance feature, and open PR https://github.com/openclaw/clawhub/pull/1867 already tracks an implementation candidate with closing syntax.

## What This Changes

Request GitHub-style device flow authentication so headless agents can authenticate without a local browser callback.

## Best Possible Solution

Land device flow as an end-to-end auth feature covering CLI `--device`, backend code issuance and token exchange, authenticated verification UI, rate limiting/security controls, docs, changelog, and regression coverage.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence gap check: current main has token-paste and localhost loopback login only, with no `--device` flag or device-code endpoints.

## Solution Assessment

Yes in direction, but only as a complete auth feature. A client-only flag is not sufficient because the server-side device-code issuance, polling/token exchange, verification page, and security controls are the core of the requested behavior.
