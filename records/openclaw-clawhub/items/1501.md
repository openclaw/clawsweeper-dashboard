## What This Changes

The issue requests transferring ownership of `baidu-netdisk-storage` to `@BaiduNetdiskAIBot` after `clawhub transfer request` reports `toUserHandle required` despite a handle argument.

## Best Possible Solution

Land or extract the plain-object CLI payload fix tracked by https://github.com/openclaw/clawhub/issues/1300, then verify whether `@BaiduNetdiskAIBot` is a personal user recipient or belongs on the org/publisher transfer path tracked by https://github.com/openclaw/clawhub/issues/1675.

## Reproduction Assessment

Yes, source-reproducible. Current main sends a pre-stringified transfer request body into an HTTP helper that serializes POST bodies again, so the server receives a JSON string and cannot read `payload.toUserHandle`.

## Solution Assessment

Yes for the reported CLI error: send plain object bodies from the affected command handlers and update tests to assert that boundary. If the recipient is an org/non-user publisher, that should be solved through the publisher-targeted transfer design rather than broadening this user-only path ad hoc.
