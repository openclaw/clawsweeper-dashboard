The issue requests transferring ownership of `baidu-netdisk-storage` to `@BaiduNetdiskAIBot` after `clawhub transfer request` returned `toUserHandle required`.

## Best Possible Solution

Land or extract the plain-object CLI payload fix tracked by https://github.com/openclaw/clawhub/issues/1300 and https://github.com/openclaw/clawhub/issues/1364, then verify whether the Baidu recipient is a personal user publisher or needs the publisher-targeted transfer design tracked separately.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, source inspection on current main shows the transfer command sends a JSON string, both HTTP transports stringify it again, and the server requires `payload.toUserHandle` on an object.

## Solution Assessment

Is this the best way to solve the issue? Yes for the reported CLI error: pass plain object bodies from the CLI and update tests to assert that boundary. If the recipient is not a personal user publisher, the broader publisher-targeted transfer model should handle that separately.
