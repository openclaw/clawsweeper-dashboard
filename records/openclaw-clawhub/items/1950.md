## Repair Work Prompt

Fix the capability-tag false positive reported here. `alon-fact-check@0.1.2` is instruction-only, but the current classifier tags it as `can-make-purchases` and `crypto` because `PURCHASE_PATTERNS` treats standalone `pay` as purchase authority and purchase detection propagates to crypto. Narrow `convex/lib/skillCapabilityTags.ts` so generic prose like `Pay attention to dates` does not match, while preserving intentional payment signals such as `payment`, `payments`, `paid automatically`, `pay per call`, `micro-payments`, checkout, `purchase`, and scoped `buy` wording. Add focused regression coverage in `convex/lib/skillCapabilityTags.test.ts`. Do not special-case the owner, slug, or version. Do not run production data repair in the code PR; note that maintainers should use the existing backfill/moderator path to recalculate or clear stored tags after the classifier change.
