ClawHub cleanup policy keeps issues open for maintainer handling, and current main still supports the reported mismatch: backend moderation preserves and exposes a clean staff override, while public security UI surfaces can still render raw suspicious scanner statuses from the latest version.

## What This Changes

The issue reports a regression where a manually cleared skill still shows suspicious static/security scan status on public pages after a ClawHub rescan or update.

## Best Possible Solution

Define a single public representation for staff-cleared scan findings, then apply it consistently across the skill detail sidebar, scanner detail pages, versions tab, and any API security snapshot that feeds public UI while preserving raw scanner evidence where maintainers intentionally want it available.
