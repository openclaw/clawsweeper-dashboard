## Summary

This should stay open: current main still has a thinner Plugins browse surface than Skills, and the requested category taxonomy plus sort semantics need maintainer product direction before implementation.

## What This Changes

The issue requests first-class plugin discovery controls, including category filters, clearer sort options, and preservation of existing verified and executes-code filters.

## Best Possible Solution

Keep the RFC open while maintainers decide the plugin taxonomy and sort semantics, then implement the chosen browse controls through the existing BrowseSidebar and package catalog APIs where the data is accurate.

## Reproduction Assessment

Not applicable; this is a product/UX RFC rather than a bug report. Source inspection confirms current main lacks the proposed plugin categories and richer sort controls.

## Solution Assessment

Unclear until maintainers answer the RFC questions. Reusing the Skills browse pattern looks maintainable, but accurate global sorting may need API or metadata work rather than only client-side UI changes.
