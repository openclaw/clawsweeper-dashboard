## Summary

This should stay open: it is a fresh product/UX RFC for ClawHub plugin discovery, and current main still lacks the proposed plugin categories and updated sort controls. The repo policy also keeps ClawHub issues open unless a terminal close reason is proven.

## What This Changes

The issue requests first-class plugin browse categories and clearer plugin sort controls while preserving verified and executes-code filters.

## Best Possible Solution

Keep the RFC open while maintainers choose the plugin taxonomy and sort semantics, then implement the selected controls against the existing browse/sidebar and package catalog surfaces.

## Solution Assessment
