## Summary

Keep open: this is a same-day RFC for new plugin browse controls, current main still has the thinner plugin discovery surface described, and the category/sort choices need maintainer product review.

## What This Changes

Request to add plugin browse categories and clearer sort options, reusing the Skills browse pattern while preserving verified and executes-code filters.

## Best Possible Solution

Maintainers should decide the initial plugin taxonomy and sort semantics, then implement the Plugins page controls against the existing package catalog/filter APIs or extend those APIs where global ordering is required.

## Reproduction Assessment

Not applicable; this is an RFC feature request, not a broken existing contract. Source inspection confirms the requested controls are absent on current main.

## Solution Assessment

Unclear until product review finishes; keyword-backed categories would match the Skills pattern, but maintainers still need to settle taxonomy and API-backed sort behavior before implementation.
