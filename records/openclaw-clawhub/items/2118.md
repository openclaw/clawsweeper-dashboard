## Summary

This should stay open: it is a fresh product/UX RFC for plugin discovery, and current main only has search, featured/code-plugin sorting, and verified/executes-code filtering on the Plugins page.

## What This Changes

Add first-class plugin discovery controls, including category-backed browsing and clearer sort options on the Plugins browse page.

## Best Possible Solution

Maintain the RFC while maintainers decide the plugin taxonomy and sort semantics, then implement the chosen browse controls using the existing BrowseSidebar and package catalog APIs where accurate.

## Solution Assessment
