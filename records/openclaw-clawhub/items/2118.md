## Summary

This should stay open because it is a fresh product/UX RFC and current main still lacks plugin categories plus the requested browse sort semantics.

## What This Changes

Add first-class plugin discovery controls by aligning the Plugins browse page with the Skills category and sort UX while preserving verified and executes-code filters.

## Best Possible Solution

Keep the RFC open, decide the initial plugin category taxonomy and sort semantics, then implement the chosen controls through the existing BrowseSidebar and package catalog APIs where accurate.

## Reproduction Assessment

Not applicable; this is a product/UX RFC rather than a bug report. Source inspection confirms current main does not yet provide the proposed plugin categories or sort controls.

## Solution Assessment

Unclear until maintainers answer the RFC questions. Reusing the Skills browse pattern is plausible, but accurate global sort behavior may need package API support rather than only client-side UI changes.
