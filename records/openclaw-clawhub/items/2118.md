## Summary

Keep open: this is a fresh product/API feature RFC, current main still lacks plugin category browse controls and category API support, and the related implementation PR at https://github.com/openclaw/clawhub/pull/2126 is open rather than merged.

## What This Changes

This RFC requests first-class plugin discovery controls: plugin categories, clearer plugin browse/search sorting, and preservation of existing trust filters.

## Best Possible Solution

Keep the RFC open while maintainers decide the plugin category taxonomy and API source of truth, then land or supersede https://github.com/openclaw/clawhub/pull/2126 with matching tests, docs, runtime proof, and backfill/deploy handling.

## Solution Assessment

Unclear until maintainers answer the RFC questions. Reusing the Skills browse pattern is plausible, but taxonomy, sort semantics, and keyword-backed versus metadata/API-backed category filtering need product/API agreement before this becomes a narrow implementation task.
