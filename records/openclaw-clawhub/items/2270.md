## Summary

Keep open: the attached screenshots now provide real UI proof, but the patch introduces an over-broad category keyword matcher that can misclassify ordinary skill text such as `latest` as Dev Tools via `test`.

## What This Changes

The PR adds inferred skill categories, category chips and browse filters, a category-related skills query/section, tests, styles, and a behavioural-proof PR-template section.

## Best Possible Solution

Tighten category inference in both client and Convex helpers, keep the inspected UI behavior, and land after mirrored regression tests pass.

## Reproduction Assessment

Not applicable for a user-reported bug because this PR adds a new feature. The review finding is source-reproducible from the matcher: `latest` contains the `test` keyword and would be scored as Dev Tools.

## Solution Assessment

No: the feature direction is plausible, but arbitrary substring matching is too broad for category inference. A safer fix is exact/plural/prefix-style token matching with mirrored client/server regression tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Require safer category keyword matches:** `src/lib/categories.ts:92`
  - body: The new matcher treats any token containing a 4+ character keyword as a category hit, so common unrelated words like `latest` match Dev Tools through `test`, and `metadata` can match Data through `data`. That makes skill detail chips, related skills, and browse filters classify otherwise unrelated skills into the wrong category; please tighten this to whole-token/plural/prefix rules or an allowlist and mirror the fix in the Convex helper.
  - confidence: 0.9
