## Summary

Current main does not support Mermaid diagrams: the shared sanitized Markdown pipeline renders the linked `mermaid` fence as an ordinary code block, and no related issue or PR currently owns the work. The request fits ClawHub’s web-ergonomics direction, but the dependency, sanitization boundary, accessibility, and client-cost policy need maintainer approval before implementation.

## What This Changes

Add Mermaid diagram rendering for fenced Mermaid blocks in ClawHub’s website Markdown views.

## Best Possible Solution

Extend the shared Markdown renderer with lazy client-side handling for `mermaid` fences, strict Mermaid security settings plus sanitized output, accessible labeling, theme-aware styling, and a readable source-code fallback when loading or parsing fails.

## Reproduction Assessment

Yes, at source level with high confidence: the linked published SKILL.md contains a fenced Mermaid flowchart, and current main has no Mermaid-specific rendering path, so the fence remains ordinary code. This is a missing capability rather than a regression in an existing documented contract.

## Solution Assessment

Yes in direction, but only through the shared Markdown component with lazy loading, strict sanitization, accessibility, and graceful fallback. A skill-page-specific renderer or unsanitized generated markup would not be the best maintainable solution.
