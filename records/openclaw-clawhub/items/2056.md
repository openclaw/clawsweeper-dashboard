Keep this PR open: current main has not implemented the private staff CLI split, and the PR currently drops existing skill reporting/appeal CLI workflows while also lacking observable real behavior proof.

## What This Changes

The PR adds a private `@openclaw/clawhub-staff` workspace CLI for user/package operations, removes public CLI/docs registrations for staff and skill moderation commands, and adds package CI/lint/Knip/typecheck coverage.

## Best Possible Solution
