## Summary

Keep this open: current main and the latest release still reject package publishes whenever the normalized package name resolves to any skill slug, and collaborator discussion has not settled the replacement namespace policy.

## What This Changes

The issue asks ClawHub to stop blocking an existing package version publish when the same owner also has a skill with the same slug, or to provide an actionable migration path.

## Best Possible Solution

Maintainers should decide and document the skill/package namespace policy, then update `convex/packages.ts`, focused package publish tests, and the durable slug/plugin specs while preserving reserved alias and ownership protections.

## Reproduction Assessment

Yes. Source inspection of current main shows the package publish path still looks up a skill by normalized package name and rejects any match before allowing the package version publish path to complete; I did not run live Convex in this read-only review.

## Solution Assessment

Unclear. A same-owner existing-package exception would address the reporter's concrete upgrade, but collaborator discussion points toward a broader typed-namespace policy that maintainers should settle first.
