## Summary

The central publish problem remains on current main and in v0.21.0: package publishing still rejects any same-name skill. The CLI rename argument wiring is now present, but that workaround does not settle the open product decision between the documented shared namespace and the collaborator-proposed typed namespaces.

## What This Changes

Allow upgrades to an existing package despite a matching skill slug, or provide a reliable supported migration path away from the collision.

## Best Possible Solution

Choose and document one namespace model; if typed namespaces are adopted, remove the cross-type guard with focused ownership and reserved-name coverage, clear artifact-type labeling, updated specs, and a real Convex publish validation.

## Reproduction Assessment

Yes at source level: current main and v0.21.0 unconditionally reject a package publish whenever a same-name skill exists, matching the reported failure. This read-only review did not execute a live Convex publish.

## Solution Assessment

No approved best fix exists yet: the closed same-owner exception conflicts with the current shared-namespace spec, while the proposed typed-namespace alternative requires an explicit maintainer decision.
