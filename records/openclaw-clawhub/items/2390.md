Keep this open: current main and the latest release still contain the unconditional package-vs-skill slug collision guard, and the linked fix PR was closed unmerged after collaborator feedback that the namespace policy needs maintainer review.

## What This Changes

The issue asks ClawHub to allow an existing package owner to publish a new package version when a skill with the same slug also exists, or to provide an actionable migration path.

## Best Possible Solution

Maintainers should decide and document the skill/package namespace policy, then update `convex/packages.ts`, focused publish tests, and the relevant specs while preserving reserved alias and ownership protections.

## Reproduction Assessment

Yes, from source inspection: a real package publish whose normalized name matches any skill slug or historical alias still reaches the unconditional collision guard. I did not run a live publish because this read-only review would require account/package fixture state.
