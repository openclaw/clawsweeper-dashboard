## Summary

Keep open. Current main has meaningful ClawHub-side mitigations for malicious skill distribution, but repository policy keeps ClawHub issues open and the remaining security posture still needs maintainer judgment across transparency, install prompts, file/readme access, scanner coverage, and OpenClaw runtime boundaries.

## What This Changes

The issue reports that ClawHub/molthub may distribute malicious skills, citing a legacy skill page as an example.

## Best Possible Solution

Maintainers should define the intended posture for malware-blocked skills, then enforce it consistently across detail pages, install prompts, readme/file APIs, ZIP/package downloads, and CLI/install surfaces while preserving an explicit staff or redacted transparency path.

## Reproduction Assessment

No, not for the exact example URL alone: on May 10, 2026 the legacy page resolved to a canonical page shell while the canonical skill API and file API returned 404. Source inspection gives a medium-confidence current-main policy path because malware-blocked skills can remain visible/readable for transparency and the install card still renders while ZIP downloads are blocked.

## Solution Assessment

No, not as an automated cleanup or one-shot patch. The maintainable path is a maintainer security decision first, with scanner integrations such as AgentGuard evaluated as prior art rather than treated as existing protection.
