## Summary

Keep open. ClawHub cleanup policy does not auto-close issues, and this security-sensitive report still has unresolved distribution-policy work: current main detects and blocks some malicious skill downloads, but malware-blocked detail/file surfaces remain publicly readable for transparency and the public skill page still renders install prompts. AgentGuard from #75 is useful prior art, but that PR closed unmerged and is not shipped protection.

## What This Changes

The issue reports that a ClawHub-hosted skill could distribute malicious installation code and asks for stronger marketplace protection against malicious skill distribution.

## Best Possible Solution

Maintainers should define the intended handling for malware-blocked skills and enforce it consistently across detail pages, install prompts, readme/file APIs, ZIP/package downloads, and CLI install flows, with an explicit staff or redacted transparency path if public visibility is kept.

## Reproduction Assessment

Yes, for the issue class. Current tests reproduce the obfuscated terminal install payload detection, and source/tests show that malware-blocked detail and file surfaces can remain publicly inspectable while install prompts still render.

## Solution Assessment

Unclear. Current static scanning, auto-hide, download blocking, and moderation evidence are useful mitigations, but the remaining fix depends on a maintainer security-policy decision rather than a narrow automated patch.
