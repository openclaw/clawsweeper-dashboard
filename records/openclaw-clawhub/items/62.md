## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main has meaningful malicious-skill mitigations but still leaves a security/product policy question around public transparency surfaces and install prompts for malware-blocked skills.

## What This Changes

The issue reports that ClawHub can distribute malicious skill content, using a linked hosted skill as an example.

## Best Possible Solution

Define the intended posture for malware-blocked skills, then enforce it consistently across detail pages, install prompts, readme/file APIs, ZIP/package downloads, and CLI install flows while treating AgentGuard from PR #75 as prior art unless maintainers choose a first-party integration.

## Reproduction Assessment

Yes, for the issue class: current tests reproduce the obfuscated terminal payload pattern, and current source/tests show malware-blocked detail and file surfaces can remain publicly readable while install commands still render. The specific hosted example may have changed, but it is not needed to validate the remaining policy gap.

## Solution Assessment

Unclear. Current static scanning, auto-hide, download blocking, and moderation tooling are useful mitigations, but the remaining transparency-versus-redaction posture is a maintainer security/product decision rather than a narrow automated fix.
