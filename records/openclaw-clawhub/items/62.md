## Summary

Keep open. Current main has meaningful scanner, publish-time, and ZIP-download mitigations, but malicious-skill distribution policy is still unresolved across public detail pages, file/readme APIs, install prompts, downstream mirrors, and OpenClaw runtime boundaries; the ClawHub cleanup policy also does not allow auto-closing open issues.

## What This Changes

The issue reports that ClawHub can expose malicious skill content and asks for stronger protection against malicious code distribution.

## Best Possible Solution

Maintainers should define the intended posture for malware-blocked skills, then enforce it consistently across detail pages, install prompts, readme/file APIs, ZIP/package downloads, mirrors, and CLI/install clients while preserving any explicit staff or redacted transparency path.

## Reproduction Assessment

No high-confidence live reproduction for the exact reported skill is available: the canonical API returned 404 on May 16, 2026. Yes for the remaining source-level policy gap: current main still allows public malware-blocked detail/file visibility and renders copyable install UI while ZIP downloads are blocked.

## Solution Assessment

No, this is not best handled as an automated one-shot cleanup. The narrow maintainable path is a maintainer security decision first, followed by consistent enforcement across registry, file/API, install, mirror, and client surfaces.
