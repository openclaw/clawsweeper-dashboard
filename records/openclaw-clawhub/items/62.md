## Summary

Keep open. This is an open ClawHub security issue, and current main shows meaningful registry-side mitigations but not a settled end-to-end distribution policy for malicious skills across detail pages, raw file APIs, install UI, archives, and downstream consumers.

## What This Changes

The issue reports that a malicious skill could be distributed through ClawHub/molthub-style skill pages, using a live skill URL as the example.

## Best Possible Solution

Maintainers should define the intended posture for malware-blocked skills, then enforce it consistently across skill detail pages, install prompts, readme/file APIs, ZIP/package download fallbacks, archives, downstream mirrors, and CLI/install surfaces while preserving any explicit staff or redacted transparency path.

## Reproduction Assessment

No high-confidence live reproduction for the exact example URL is available from the sparse issue body. Source inspection gives a medium-confidence current-main path for the policy gap: malware-blocked content can remain publicly readable for transparency and the install UI path still exists while ZIP downloads are blocked.

## Solution Assessment

No, this should not be solved as a one-shot automated cleanup. The maintainable path is a maintainer security decision first, with scanner integrations such as AgentGuard treated as prior art rather than assumed shipped protection.
