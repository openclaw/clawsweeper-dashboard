## Summary

Keep open. This is an open ClawHub security issue, and current main still leaves a maintainer policy question about whether malware-blocked skills should remain publicly inspectable for transparency while install affordances are visible.

## What This Changes

The issue asks ClawHub to address malicious skill distribution risk shown by a linked published skill URL.

## Best Possible Solution

Maintainers should define the intended security posture for malware-blocked skills, then enforce it consistently across detail pages, install prompts, readme/file APIs, ZIP downloads, and CLI install flows while preserving any explicit staff or redacted transparency path.

## Reproduction Assessment

Yes, for the issue class. Current tests exercise an obfuscated terminal install payload, and source inspection shows malware-blocked detail/file surfaces remain publicly readable while the install command card still renders without moderation state.

## Solution Assessment

Unclear. Current static scanning, auto-hide, download blocking, and moderation visibility are useful mitigations, but the remaining transparency-versus-redaction choice is a maintainer security/product decision rather than a narrow automated fix.
