## Summary

Keep open. This is an open ClawHub security issue, and current main has useful malware mitigations but still leaves a maintainer policy decision around public transparency versus strict distribution blocking for malware-blocked skills; AgentGuard from #75 is useful prior art, but that PR is closed unmerged and the scanner is not shipped in this tree.

## What This Changes

The report warns that ClawHub can distribute malicious skills, with an example published skill URL and later discussion of AgentGuard as scanner prior art.

## Best Possible Solution

Maintainers should define the intended posture for malware-blocked skills, then enforce it consistently across detail pages, install prompts, readme/file APIs, ZIP downloads, and CLI install flows while preserving any explicit staff or redacted transparency path.

## Reproduction Assessment

Yes, for the issue class from source inspection and tests. Current main detects and hides obfuscated install-payload publishes, while malware-blocked detail and file surfaces remain publicly readable and the detail UI still renders install commands.

## Solution Assessment

Unclear. The current scanner, auto-hide, download blocking, and moderation visibility are useful mitigations, but the remaining transparency-versus-redaction choice is a maintainer security/product decision rather than a narrow automated fix.
