## Summary

Keep open. Current main has meaningful registry-side mitigations for this malicious-skill distribution class, but the remaining malware-blocked detail/file/install exposure is a maintainer security policy decision, and this ClawHub cleanup pass keeps issues open. AgentGuard at https://github.com/openclaw/clawhub/pull/75 is useful prior art, but that PR closed unmerged and is not shipped protection.

## What This Changes

The issue reports that a published ClawHub/MoltHub skill could distribute malicious code and links a live example.

## Best Possible Solution

Maintainers should define the intended malware-blocked skill posture, then enforce it consistently across detail pages, install prompts, readme/file APIs, ZIP/package downloads, CLI install flows, and any staff or redacted transparency path.

## Reproduction Assessment

Yes. Source inspection and existing regression tests give a high-confidence path for the reported issue class: current main detects and auto-hides the obfuscated installer-payload family while intentionally leaving some malware-blocked public surfaces reachable for transparency.

## Solution Assessment

Unclear. The current scanner, auto-hide behavior, and download block are useful mitigations, but the remaining transparency-versus-redaction posture and any AgentGuard integration require maintainer security/product judgment before a narrow implementation is safe.
