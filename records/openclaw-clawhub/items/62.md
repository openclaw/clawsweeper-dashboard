## Summary

Keep open. Current main has meaningful malicious-skill mitigations, but this is an open ClawHub security issue and the repository cleanup policy keeps issues open; the remaining blocked-skill exposure and scanner-integration posture still needs maintainer security judgment.

## What This Changes

Report that a ClawHub-hosted skill example could distribute malicious code and ask for registry-side security handling.

## Best Possible Solution

Maintainers should define the intended posture for malware-blocked skills, then enforce it consistently across detail pages, install prompts, readme/file APIs, ZIP/package downloads, CLI install flows, scanner integrations, and any staff or redacted transparency path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: obfuscated terminal install payloads are detected and auto-hidden, while malware-blocked detail and file surfaces remain deliberately readable under the transparency model.

## Solution Assessment

Unclear. Current main's scanner, auto-hide, ZIP-download block, and CLI install block are useful mitigations, but stricter redaction or external scanner enforcement needs maintainer security policy before a narrow implementation is safe.
