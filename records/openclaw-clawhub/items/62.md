## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still leaves a security/product-policy gap: malicious uploads are detected and download-blocked, but malware-blocked detail/file surfaces remain publicly inspectable and the public detail page still renders install affordances. AgentGuard from #75 is useful prior art, but that PR closed unmerged and is not shipped ClawHub protection.

## What This Changes

The issue reports that a published ClawHub skill could distribute malicious code, with an example hosted skill link.

## Best Possible Solution

Maintainers should define the intended handling for malware-blocked skills, then enforce it consistently across detail pages, install prompts, readme/file APIs, ZIP/package downloads, and CLI install flows while preserving an explicit staff or redacted transparency path.

## Reproduction Assessment

Yes, source-reproducible for the issue class. Current tests cover the obfuscated terminal payload pattern, and source/tests show malware-blocked detail/file surfaces can remain public while install commands still render.

## Solution Assessment

Unclear. Current static scanning, auto-hide, download blocking, manual override limits, and redaction tooling are useful mitigations, but the remaining exposure is a security/product posture decision rather than a narrow code-only fix.
