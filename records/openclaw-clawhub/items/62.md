## Summary

Keep open. Repository policy for this cleanup pass keeps ClawHub issues open, and current main still has a security-sensitive policy gap: registry scanning and ZIP download blocking exist, but malware-blocked skill detail/file surfaces remain publicly inspectable and the install UI can still render copyable install prompts. AgentGuard from https://github.com/openclaw/clawhub/pull/75 is useful prior art, but that PR closed unmerged and is not shipped protection.

## What This Changes

The issue reports malicious-code distribution risk through ClawHub-hosted skill pages and related install surfaces.

## Best Possible Solution

Maintainers should define the intended security posture for malware-blocked skills, then enforce it consistently across skill detail pages, install prompts, readme/file APIs, ZIP/package downloads, mirrors, and CLI/install clients while preserving any explicit staff or redacted transparency path.

## Reproduction Assessment

No high-confidence live reproduction for the exact example URL is available from the sparse report. Source inspection does show the current-main policy conflict: malware-blocked skills can remain publicly visible/readable for transparency and still render install UI while ZIP downloads are blocked.

## Solution Assessment

No, not as an automated cleanup or one-shot patch. The narrow code mitigation exists for known malicious install prompts, but the remaining answer requires a maintainer security decision about transparency, redaction, install prompts, mirrors, downstream clients, and runtime boundaries.
