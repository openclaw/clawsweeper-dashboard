## Summary

Keep open. Current main has real registry-side mitigations for malicious skill distribution, but this is an open ClawHub security issue and repository policy does not auto-close issues; the remaining public exposure posture needs maintainer security judgment.

## What This Changes

Review and resolve the reported risk that ClawHub can distribute malicious skills through public skill pages and install surfaces.

## Best Possible Solution

Maintainers should define the intended malware-blocked skill posture, then enforce it consistently across public detail pages, raw/readme file APIs, install prompts, ZIP/package downloads, CLI install flows, and any staff or redacted transparency path.

## Reproduction Assessment

Yes. Source inspection and current regression tests give a high-confidence reproduction path for the issue class: obfuscated terminal install payloads are detected and hidden, while malware-blocked detail/file surfaces remain intentionally readable for transparency.

## Solution Assessment

Unclear. Current main's scanner, auto-hide, ZIP download block, and CLI install block are useful mitigations, but stricter public redaction or external scanner integration needs maintainer security policy before a narrow implementation is safe.
