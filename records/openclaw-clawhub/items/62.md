## Summary

Keep open. ClawHub cleanup policy keeps issues open, and the security question is still unresolved: current main now detects and hides several malicious-skill patterns and blocks ZIP downloads, but malware-blocked detail/file surfaces remain publicly readable for transparency and the public detail UI still renders install commands.

## What This Changes

The issue asks ClawHub to address malicious skill distribution risk, citing a published skill URL as an example.

## Best Possible Solution

Define and document the intended blocked-skill distribution posture, then enforce it consistently across detail pages, install prompts, readme/file APIs, ZIP/package downloads, CLI install flows, and any staff-only transparency path.

## Reproduction Assessment

Yes. A skill with moderationFlags containing blocked.malware can be traced through getBySlug, readme/file access, and the detail page; current tests also assert malware-blocked skill files remain readable while ZIP downloads are blocked.

## Solution Assessment

Unclear. Current static scanning and download blocking are valuable mitigations, but the maintainable fix needs an explicit maintainer decision on transparency versus redaction before narrowing the code change.
