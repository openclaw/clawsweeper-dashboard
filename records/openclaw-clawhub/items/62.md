## Summary

Keep open. ClawHub cleanup policy does not auto-close issues, and this security-sensitive report still has a real policy gap: current main blocks malicious publishes/downloads and adds a security redaction path, but malware-blocked detail/file surfaces can remain public for transparency and install prompts are still rendered.

## What This Changes

The issue reports that ClawHub/MoltHub can distribute a malicious skill and points to a suspicious hosted artifact as an example.

## Best Possible Solution

Maintainers should define the target handling for malicious skill pages and enforce it consistently across detail pages, install prompts, readme/file APIs, ZIP/package downloads, and CLI install flows, using AgentGuard from #75 only as prior art unless a first-party scanner integration is explicitly chosen.

## Reproduction Assessment

Yes, for the issue class: current tests reproduce the obfuscated terminal payload pattern, and source/tests prove malware-blocked detail/file surfaces can remain publicly inspectable while install commands still render. The specific linked hosted artifact is not needed to keep this security-policy issue open.

## Solution Assessment

Unclear. Current static scanning, auto-hide, download blocking, and security-redaction tooling are useful mitigations, but choosing transparency versus redaction is a maintainer security/product decision rather than a narrow automated fix.
