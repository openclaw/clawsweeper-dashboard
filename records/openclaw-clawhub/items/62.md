## Summary

Keep open. Current main has meaningful registry-side mitigations, but this is a security-sensitive ClawHub issue and the remaining distribution posture still needs maintainer judgment across malware-blocked detail pages, raw file/API reads, install prompts, mirrors/downstream clients, and OpenClaw runtime boundaries.

## What This Changes

The issue reports that ClawHub can distribute malicious skills and asks for stronger protection against malicious code distribution from marketplace skill pages.

## Best Possible Solution

Maintainers should define the intended security posture for malware-blocked skills, then enforce it consistently across detail pages, install prompts, readme/file APIs, ZIP/package downloads, mirrors, and CLI/install clients while preserving any explicit staff or redacted transparency path.

## Reproduction Assessment

No high-confidence live reproduction for the exact example URL is available from the sparse report. Source inspection does show the remaining current-main policy gap: malware-blocked skills can stay publicly visible/readable for transparency and still render copyable install UI while ZIP downloads are blocked.

## Solution Assessment

No, not as an automated cleanup or one-shot code fix. Current main has useful registry-side scanner and download mitigations, but the remaining answer requires a maintainer security decision about transparency, redaction, install prompts, mirrors, and runtime boundaries.
