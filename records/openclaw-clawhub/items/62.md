## Summary

Keep open. This is an open ClawHub security issue, and this repository cleanup pass only allows auto-closing ClawHub pull requests that are certainly implemented on main; current main has meaningful registry-side malware mitigations, but the remaining distribution posture is still a security/product decision.

## What This Changes

The issue reports that ClawHub can be used to distribute a malicious skill, with a specific published skill URL as the example.

## Best Possible Solution

Maintainers should define the intended malware-blocked distribution posture, then enforce it consistently across skill detail pages, install prompts, readme/file APIs, ZIP/package downloads, CLI/install surfaces, and any staff-only or redacted transparency path.

## Reproduction Assessment

No, not for the exact example URL from the sparse issue body. Source inspection gives a medium-confidence reproduction path for the current policy gap: malware-blocked skills can remain visible/readable for transparency while the public install card still renders, even though ZIP downloads are blocked.

## Solution Assessment

No, not as an automated cleanup or one-shot fix. The narrow maintainable path is a maintainer security decision first, with AgentGuard from https://github.com/openclaw/clawhub/pull/75 evaluated as prior art rather than treated as shipped protection.
