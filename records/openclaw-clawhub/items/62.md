## Summary

Keep open. Current main has meaningful ClawHub-side malware scanning, auto-hiding, moderation holds, and ZIP download blocking, but this is an open ClawHub security issue and the remaining distribution posture still needs maintainer security judgment across public detail, raw file/API, install prompt, mirror/downstream, and OpenClaw runtime boundaries; AgentGuard from https://github.com/openclaw/clawhub/pull/75 is useful prior art but closed unmerged.

## What This Changes

The issue reports that ClawHub can distribute malicious skills, with a linked example of a suspicious published skill page.

## Best Possible Solution

Maintainers should define the intended security posture for malware-blocked skills, then enforce it consistently across skill detail pages, install prompts, readme/file APIs, ZIP/package download fallbacks, mirrors, and CLI/install surfaces while preserving any explicit staff or redacted transparency path.

## Reproduction Assessment

No high-confidence live reproduction for the exact example URL from the sparse issue body. Source inspection gives a medium-confidence current-main reproduction path for the policy gap: malware-blocked skills can remain publicly visible/readable for transparency and still render install UI while ZIP downloads are blocked.

## Solution Assessment

No, not as an automated cleanup or one-shot code fix. The maintainable path is a maintainer security decision first, with scanner integrations such as AgentGuard evaluated as prior art rather than treated as existing protection.
