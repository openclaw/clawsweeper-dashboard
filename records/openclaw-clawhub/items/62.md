## Summary

Keep open. This is an open ClawHub security issue, and the repository cleanup policy for this pass does not auto-close issues; current main has meaningful registry-side malware mitigations, but the public distribution posture still needs maintainer security judgment.

## What This Changes

The issue reports that a published ClawHub skill could distribute malicious code, using a specific skill page as the example.

## Best Possible Solution

Define the intended malware-blocked distribution posture, then enforce it consistently across detail pages, install prompts, file/readme APIs, ZIP/package downloads, CLI/install surfaces, and any staff-only transparency path.

## Reproduction Assessment

No high-confidence live reproduction was established from the sparse issue body. Source inspection does show the relevant current-main paths: static malicious uploads are hidden, ZIP downloads are blocked, but malware-blocked detail/file surfaces and install prompts still have unresolved policy behavior.

## Solution Assessment

No, not as a one-shot automated fix. The safest path is a maintainer security decision first, with AgentGuard from https://github.com/openclaw/clawhub/pull/75 evaluated as prior art rather than treated as an already shipped scanner.
