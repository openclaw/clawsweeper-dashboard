## Summary

Keep open. This is an open ClawHub security issue, and the cleanup policy for this pass only permits auto-closing ClawHub pull requests that current main certainly implements; current main has meaningful malicious-skill mitigations, but the remaining exposure posture and scanner-integration direction still need maintainer security judgment. AgentGuard from https://github.com/openclaw/clawhub/pull/75 is useful prior art, but it closed unmerged, and the TrentClaw integration in https://github.com/openclaw/clawhub/pull/2050 is still open rather than shipped protection.

## What This Changes

The issue reports that ClawHub may distribute malicious skill content and links to a public skill page as an example.

## Best Possible Solution

Maintainers should define the intended posture for malware-blocked skills, then enforce it consistently across detail pages, install prompts, readme/file APIs, ZIP/package downloads, CLI install flows, and any staff or redacted transparency path.

## Reproduction Assessment

Yes. Source inspection and current regression tests give a high-confidence path for the issue class: obfuscated terminal install payloads are detected and hidden at publish time, while malware-blocked detail and file surfaces remain intentionally readable under the transparency model.

## Solution Assessment

Unclear. Current main's scanner, auto-hide, ZIP-download block, and CLI install block are useful mitigations, but stricter public redaction or external scanner integration needs maintainer security policy before a narrow implementation is safe.
