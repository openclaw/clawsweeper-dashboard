## Summary

Keep open. The cleanup policy for openclaw/clawhub keeps issues open, and current main only partially mitigates the malicious-skill distribution risk: static scanning, hidden publishes, download blocking, and owner moderation exist, but public malware-blocked detail/file transparency, visible install prompts, and any external scanner trust boundary still need maintainer security judgment. The AgentGuard work at https://github.com/openclaw/clawhub/pull/75 is useful prior art, but it closed unmerged and is not shipped protection.

## What This Changes

The issue asks ClawHub to address a malicious-skill distribution risk, with a legacy ClawdHub skill page cited as an example.

## Best Possible Solution

Maintainers should define the intended malware-blocked distribution policy, then enforce it consistently across detail pages, install prompts, readme/file APIs, ZIP/package downloads, CLI/install surfaces, and any approved scanner integration while preserving explicit owner/staff review paths.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path for the issue class: malicious install prompts are detected and hidden at publish time, while malware-blocked detail/file surfaces intentionally remain readable under the current transparency model.

## Solution Assessment

Unclear. Current registry mitigations are useful, but choosing strict redaction versus public transparency and deciding whether to integrate AgentGuard is a maintainer security/product policy decision.
