## Summary

Keep open. This is an open ClawHub issue, and the cleanup policy for this pass only allows auto-closing ClawHub pull requests that current main certainly implements; current main has meaningful malicious-skill mitigations, but the remaining public exposure and install-prompt posture is still a security policy decision.

## What This Changes

The issue reports that ClawHub can distribute malicious skill content, with a hosted example and a later AgentGuard scanner proposal as related prior art.

## Best Possible Solution

Maintainers should define the intended security posture for malware-blocked skills, then enforce it consistently across detail pages, install prompts, readme/file APIs, ZIP/package downloads, and CLI/install surfaces while preserving any deliberate staff or redacted transparency path.

## Reproduction Assessment

Yes, for the issue class: current source and tests show obfuscated terminal payloads are detected and some malicious distribution paths are blocked, while current source also shows malware-blocked detail/file visibility and install prompting remain policy-sensitive surfaces. The specific linked hosted artifact is not needed to verify the remaining class of risk.

## Solution Assessment

Unclear. Current static scanning, auto-hide, download blocking, and security documentation are useful mitigations, but the remaining transparency-versus-redaction posture requires maintainer security/product judgment before a narrow code change is clearly best.
