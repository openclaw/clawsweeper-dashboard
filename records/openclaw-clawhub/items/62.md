## Summary

Keep open. Current main has meaningful registry-side malware detection and download blocking, but this is a ClawHub security issue and the remaining distribution posture across public detail, file, install, mirror, and OpenClaw runtime boundaries still needs maintainer security judgment; AgentGuard from https://github.com/openclaw/clawhub/pull/75 is useful prior art but closed unmerged.

## What This Changes

The issue reports a malicious-skill distribution risk through ClawHub/molthub-style hosting and gives a live example URL.

## Best Possible Solution

Maintainers should define the intended malware-blocked distribution policy, then enforce it consistently across public detail pages, install prompts, readme/file APIs, ZIP and CLI install paths, mirror retention, and any staff-only or redacted transparency flow.

## Reproduction Assessment

No, not for the exact example URL from the sparse issue body. Source inspection gives a medium-confidence path for the current policy gap: malware-blocked skills can remain publicly visible/readable for transparency while the public install card still renders, even though ZIP downloads are blocked.

## Solution Assessment

No, not as an automated cleanup or one-shot fix. The maintainable path is a maintainer security decision first, with scanner integration such as AgentGuard evaluated as prior art rather than treated as existing protection.
