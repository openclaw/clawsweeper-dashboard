## Summary

Keep open. ClawHub policy keeps open issues open, and the substantive security question is still unresolved: current main adds meaningful malicious-skill mitigations, but malware-blocked detail/file surfaces remain publicly readable for transparency and public detail pages still render install commands. AgentGuard from #75 is useful prior art, but that PR closed unmerged, and the broader related security report in #662 remains open.

## What This Changes

Review whether ClawHub allows malicious skills to be distributed or installed from public skill pages and APIs.

## Best Possible Solution

Maintainers should decide the intended posture for malware-blocked skills, then enforce it consistently across skill detail pages, install prompts, readme/file APIs, ZIP/package download fallbacks, and CLI/install surfaces while preserving an explicit staff or redacted transparency path. If scanner integration is desired, AgentGuard from #75 should be evaluated alongside the broader open security report in #662.
