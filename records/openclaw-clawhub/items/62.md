## Summary

Keep open. Current main has meaningful registry-side mitigations for the reported malicious-skill distribution class, and the AgentGuard proposal at https://github.com/openclaw/clawhub/pull/75 is useful prior art but closed unmerged; the remaining malware-blocked detail/file/install exposure is still a maintainer security policy decision, and this ClawHub cleanup pass keeps issues open.

## What This Changes

The issue reports a security risk that ClawHub can distribute malicious skills, with a linked published skill as an example.

## Best Possible Solution

Maintainers should define the intended malware-blocked skill posture, then enforce it consistently across detail pages, install prompts, readme/file APIs, ZIP/package downloads, CLI install flows, and any staff or redacted transparency path.

## Reproduction Assessment

Yes, source inspection and existing regression tests give a high-confidence path for the issue class. Current main detects and auto-hides the obfuscated installer-payload family, while malware-blocked detail/file surfaces and install affordances remain reachable by design.

## Solution Assessment

Unclear. The current scanner, auto-hide behavior, and download block are useful mitigations, but the remaining transparency-versus-redaction posture and any AgentGuard integration require maintainer security/product judgment before a narrow implementation is safe.
