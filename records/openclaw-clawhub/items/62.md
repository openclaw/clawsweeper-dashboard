## Summary

Keep open. Repository policy for this cleanup pass keeps ClawHub issues open, and current main still leaves a security-sensitive distribution policy decision unresolved even though it has useful registry-side malware scanning, hiding, download blocking, mirror deletion, and third-party API guidance. AgentGuard from https://github.com/openclaw/clawhub/pull/75 is useful prior art, but that PR closed unmerged and is not shipped ClawHub protection.

## What This Changes

The issue reports malicious code distribution risk through a ClawHub-hosted skill example.

## Best Possible Solution

Maintainers should define the intended posture for malware-blocked skills, then enforce it consistently across detail pages, install prompts, readme/file APIs, ZIP/package downloads, mirrors, downstream clients, and any explicit staff or redacted transparency path; AgentGuard from https://github.com/openclaw/clawhub/pull/75 can be evaluated as prior art rather than assumed shipped.

## Reproduction Assessment

No high-confidence live reproduction is available for the exact example URL from the sparse report. Source inspection does reproduce the current policy gap: malware-blocked skills can remain publicly visible/readable while ZIP downloads are blocked and install UI still renders.

## Solution Assessment

No, not as an automated cleanup or one-shot code fix. The narrow scanner and download mitigations are useful, but the remaining solution requires a maintainer security decision about transparency, redaction, install prompts, mirrors, downstream clients, and OpenClaw runtime boundaries.
