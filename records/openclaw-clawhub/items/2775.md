## Summary

Keep open: this PR targets the right backend pagination bug and adds useful coverage, but the new community probe can still end category pagination early when its bounded `take()` sees only hidden or excluded rows. The external PR also needs after-fix real behavior proof before merge.

## What This Changes

The PR adds a non-paginated community-availability probe for official-first plugin category listing, adds regression coverage, and removes an unused prod smoke retry assignment.

## Best Possible Solution

Keep the single-paginate direction, but make the community probe conservative when the bounded read is inconclusive, cover that edge case, and add redacted after-fix API or browser proof.

## Reproduction Assessment

## Solution Assessment

No as submitted. Avoiding the second `paginate()` is the right layer, but the bounded probe must not return a terminal page when it cannot prove there are no later visible community rows.
