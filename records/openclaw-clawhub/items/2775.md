## Summary

Keep open: this is a useful fix direction for the linked public catalog bug, but the patch currently short-circuits community fill-through when the official phase has fewer results than requested, and the PR still lacks after-fix real API/Convex proof.

## What This Changes

The PR replaces the official-first category community probe with a non-paginated digest availability check, adds regression coverage for one paginate call, and removes an unused prod HTTP smoke assignment.

## Best Possible Solution

Keep the non-paginated community availability probe only for full official pages, preserve community fill-through for underfilled official pages, add regression coverage for zero/under-limit official results, and post after-fix real API proof.

## Reproduction Assessment

Yes for the underlying bug from production evidence in the linked report and PR body, with source support in the official-first category splitter. I did not run a local Convex reproduction in this read-only review.

## Solution Assessment

No: avoiding a second paginate is the right layer, but returning on every completed official phase drops the existing community fill-through behavior. The narrower fix is to use the digest probe only after a full official page and otherwise fetch community results for the remainder.
