## What This Changes

The issue reports that OpenClaw 2026.4.2 fails against Alibaba Cloud DashScope compatible mode for qwen-turbo because unsupported OpenAI-compatible request parameters are rejected.

## Best Possible Solution

Fix DashScope compatibility in OpenClaw's provider serialization layer by omitting or normalizing unsupported OpenAI-compatible fields, then optionally publish or catalog a DashScope provider plugin/workaround in ClawHub if the runtime surface supports it.

## Reproduction Assessment

Unclear for this repository: the reporter provided a concrete external OpenClaw/DashScope setup and exact error, but current ClawHub main lacks the chat provider serialization path needed to reproduce it locally.

## Solution Assessment

Not applicable as a patch in this repo. A ClawHub-only change would not stop DashScope from rejecting chat parameters; the maintainable fix belongs in the OpenClaw runtime/provider layer or a provider plugin hook with that capability.
