## Summary

Keep open. Current main still matches the reported deleted-account GitHub sign-in lockout, the recovery/unlink cluster remains open, and resolving this needs maintainer product/security policy rather than conservative issue cleanup.

## What This Changes

The issue asks ClawHub maintainers to restore access or allow re-registration after the reporter accidentally deleted a GitHub-linked ClawHub account.

## Best Possible Solution

Define a maintainer-approved account lifecycle policy, then either document irreversible deletion as the support outcome or implement an audited recovery/unlink flow across OAuth, deletion state, operator tooling, docs, and tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: delete a ClawHub account, complete GitHub OAuth again, and handleDeletedUserSignIn rejects the deactivated user before normal session or CLI token setup can complete.

## Solution Assessment

No for an autonomous patch. The maintainable path first needs a product/security decision on whether recovery, provider unlinking, clean re-registration, or operator-only cleanup should exist and how it preserves identity and deletion invariants.
