## Summary

Keep open. Current main still blocks GitHub sign-in for permanently deleted ClawHub accounts, and resolving this support cluster needs a maintainer product/security decision rather than cleanup closure.

## What This Changes

The reporter asks for GitHub sign-in or re-registration to work again after they accidentally deleted their ClawHub account.

## Best Possible Solution

Define a maintainer-owned account recovery outcome: either keep irreversible deletion as policy with a clear support response, or add an audited recovery/unlink flow with docs and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: delete a ClawHub account, then complete GitHub OAuth again; handleDeletedUserSignIn rejects the deactivated user before normal session setup completes.

## Solution Assessment

No for an autonomous patch. The maintainable solution first needs a product/security decision on whether recovery, provider unlinking, or operator-only cleanup should exist and how it preserves deletion, ban, token, ownership, and audit invariants.
