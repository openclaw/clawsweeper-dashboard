## Summary

Keep open. Current main still intentionally blocks GitHub sign-in after self-deletion, and ClawHub cleanup policy keeps issues out of conservative auto-close; resolving this needs maintainer account-recovery and security policy rather than source-only cleanup.

## What This Changes

The issue asks for help signing back in or re-registering with GitHub after accidentally deleting a ClawHub account.

## Best Possible Solution

Define an operator-approved recovery outcome: either keep irreversible deletion with clear support handling, or add an audited restore/unlink flow that preserves bans, token revocation, ownership, published content, and deletion warnings.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, then a later GitHub OAuth completion calls handleDeletedUserSignIn and rejects that deactivated user; I did not run a live OAuth flow.

## Solution Assessment

No for an autonomous source patch. Current code and docs intentionally define irreversible deletion, so reactivation or GitHub unlinking needs maintainer-approved identity, security, and operations semantics first.
