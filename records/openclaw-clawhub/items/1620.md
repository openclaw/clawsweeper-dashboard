## Summary

Keep open. Current main still intentionally blocks GitHub sign-in after self-deletion, this is an open support/security policy issue rather than an implemented cleanup target, and several related account-recovery reports remain open.

## What This Changes

The issue asks for help signing in or re-registering with GitHub after accidentally deleting a ClawHub account.

## Best Possible Solution

Define a maintainer-owned account recovery outcome: either keep irreversible deletion and document the support response, or add an audited restore/unlink flow that preserves bans, ownership, token revocation, published content, and deletion warnings.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: delete an account, then a later GitHub OAuth completion calls handleDeletedUserSignIn and rejects the deactivated user; I did not run a live OAuth flow.

## Solution Assessment

No for an autonomous source patch. The narrow maintainable path first needs a maintainer decision on whether same-GitHub recovery should exist and how it interacts with irreversible deletion, OAuth identity binding, bans, ownership, tokens, and auditability.
