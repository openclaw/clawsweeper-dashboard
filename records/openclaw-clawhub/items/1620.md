## Summary

Keep open. ClawHub cleanup policy keeps open issues in this lane, and current main still matches the reported account lockout: self-deletion creates a permanently deactivated account, OAuth rejects it, and admin unban does not recover that state.

## What This Changes

The issue asks for account support or re-registration after a user accidentally deleted a ClawHub account linked to GitHub.

## Best Possible Solution

Define the account recovery policy first; if recovery is allowed, add an audited admin/support reactivation or GitHub-unlink flow that preserves ban, deletion, token, audit, and published-content boundaries.

## Reproduction Assessment

Yes, from source inspection. Running the settings deletion path leaves the user with deactivatedAt/purgedAt, and the OAuth callback calls handleDeletedUserSignIn, which throws for that state.

## Solution Assessment

Unclear until maintainers choose the account policy. Current main is internally consistent with the irreversible-deletion docs, while recovery or unlinking would be a deliberate support/security feature rather than a narrow bug fix.
