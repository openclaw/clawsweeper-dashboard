## Summary

Keep open. Current main still intentionally blocks normal GitHub sign-in after permanent self-deletion, and changing that outcome needs maintainer product/security policy rather than cleanup closure.

## What This Changes

The issue requests help re-registering or signing in with GitHub after the reporter accidentally deleted their ClawHub account.

## Best Possible Solution

Define a maintainer-owned recovery policy, then either keep irreversible deletion documented as the supported outcome or implement an audited support/admin recovery or GitHub unlink flow with regression coverage.

## Reproduction Assessment

Yes from source inspection. Delete a ClawHub account, then complete GitHub OAuth again for the same bound identity; current auth code reaches handleDeletedUserSignIn and rejects the deactivated record before normal session setup.

## Solution Assessment

No autonomous fix path is best yet. The maintainable solution first needs a product/security decision on whether recovery, provider unlinking, or operator-only cleanup should exist and how it preserves deletion and moderation invariants.
