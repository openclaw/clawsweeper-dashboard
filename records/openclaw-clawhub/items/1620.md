## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still intentionally blocks GitHub sign-in after self-deletion; resolving this needs maintainer/operator account-recovery and security policy rather than cleanup closure.

## What This Changes

The issue asks ClawHub maintainers to restore access or allow GitHub sign-in/re-registration after the reporter accidentally deleted their ClawHub account.

## Best Possible Solution

Define and document a maintainer-owned recovery outcome: either keep permanent deletion with clearer support guidance, or add an audited admin/support restore or GitHub-unlink flow that preserves bans, ownership, tokens, and audit history.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: self-deletion writes deactivatedAt/purgedAt, then the GitHub OAuth completion path calls handleDeletedUserSignIn and rejects that user; I did not run a live OAuth flow.

## Solution Assessment

No for an autonomous patch. Current main matches the irreversible-deletion spec, so changing recovery or unlink behavior needs maintainer-approved account, security, and operations semantics first.
