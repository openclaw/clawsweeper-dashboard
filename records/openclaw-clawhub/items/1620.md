## Summary

Keep open. Current main intentionally blocks GitHub reauth for deactivated accounts after self-deletion, and changing that requires a maintainer product/security decision rather than conservative cleanup.

## What This Changes

The issue asks for access restoration or re-registration after a deleted ClawHub account can no longer complete GitHub sign-in.

## Best Possible Solution

Define the account lifecycle policy, then either keep and document irreversible deletion with a clear support response or ship an audited recovery/unlink flow that preserves auth, deletion, moderation, token, and ownership invariants.

## Reproduction Assessment

Yes, by source inspection: delete a ClawHub account, complete GitHub OAuth again, and the callback calls handleDeletedUserSignIn, which rejects deactivated users before session setup. I did not run a live OAuth flow, so this is source-reproducible rather than live-reproduced.

## Solution Assessment

No for an autonomous fix. The maintainable path first needs a maintainer product/security decision on whether recovery, unlinking, clean re-registration, or operator-only handling should exist and how it preserves deletion invariants.
