## Summary

Keep open. Current main still intentionally blocks re-authentication for permanently deleted ClawHub accounts, and resolving access after self-deletion requires maintainer product/security policy rather than conservative cleanup.

## What This Changes

The issue requests help signing in or re-registering with GitHub after the reporter accidentally deleted their ClawHub account.

## Best Possible Solution

Define the account lifecycle policy, then either document irreversible deletion as the support outcome or implement an audited recovery/unlink flow that preserves identity, ownership, token, and ban boundaries.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: delete a ClawHub account, complete GitHub OAuth again, and handleDeletedUserSignIn rejects the deactivated user before normal session or token setup can complete.

## Solution Assessment

No for an autonomous patch. The maintainable path first needs a product/security decision on whether recovery, provider unlinking, clean re-registration, or operator-only cleanup should exist and how it preserves account lifecycle invariants.
