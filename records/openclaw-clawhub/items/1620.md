## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main deliberately blocks sign-in for permanently deleted accounts; restoring access or unlinking a GitHub identity is a maintainer/operator product and security decision, not a cleanup close.

## What This Changes

The reporter asks for help signing in or re-registering with GitHub after accidentally deleting their ClawHub account.

## Best Possible Solution

Choose and document a maintainer-approved account recovery policy: either keep irreversible deletion with clear support guidance, or implement an audited admin/support restore or GitHub-unlink flow that preserves bans, ownership, tokens, and audit invariants.

## Reproduction Assessment

Yes, from source inspection. deleteAccount sets deactivatedAt/purgedAt, and the next GitHub OAuth completion path calls handleDeletedUserSignIn, which rejects deactivated users before sign-in completes; I did not run a live OAuth flow.

## Solution Assessment

No for an autonomous source patch. Current main matches the documented irreversible-deletion policy, and any restore or unlink path needs maintainer-approved account, security, and operations semantics first.
