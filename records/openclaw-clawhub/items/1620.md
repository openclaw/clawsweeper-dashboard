## Summary

Keep open. Current main intentionally blocks GitHub re-sign-in for permanently deleted ClawHub accounts, and this open issue needs maintainer product/security policy for recovery or unlinking rather than conservative cleanup.

## What This Changes

The issue asks for help restoring or re-registering GitHub sign-in after the reporter accidentally deleted their ClawHub account.

## Best Possible Solution

Define a maintainer-owned account lifecycle policy, then either document irreversible deletion as the supported outcome or implement an audited recovery/unlink flow that preserves deletion, identity, moderation, token, and ownership invariants.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: delete a ClawHub account, complete GitHub OAuth again, and handleDeletedUserSignIn rejects the deactivated user before normal session setup can complete.

## Solution Assessment

No for an autonomous cleanup fix. The maintainable path first needs a product/security decision on whether recovery, provider unlinking, clean re-registration, or operator-only cleanup should exist and how it preserves deletion invariants.
