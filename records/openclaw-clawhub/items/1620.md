## Summary

Keep open. Current main still matches the reported lockout, and the ClawHub cleanup policy keeps issues open; changing this behavior requires maintainer product/security review because the current spec intentionally treats user deletion as irreversible.

## What This Changes

The issue requests help signing in or re-registering with GitHub after the reporter deleted their ClawHub account.

## Best Possible Solution

Define the account lifecycle policy, then either keep irreversible deletion with clear support guidance or ship an audited admin/support recovery or unlink flow that preserves deletion, moderation, token, and ownership invariants.

## Reproduction Assessment

Yes, by source inspection: deleteAccount sets deactivatedAt, and the GitHub OAuth callback calls handleDeletedUserSignIn, which rejects deactivated users before normal session setup. I did not run a live OAuth flow, so this is source-reproducible rather than live-reproduced.

## Solution Assessment

No for an autonomous fix. The maintainable path first needs a maintainer product/security decision on whether recovery, unlinking, clean re-registration, or operator-only handling should exist and how it preserves deletion invariants.
