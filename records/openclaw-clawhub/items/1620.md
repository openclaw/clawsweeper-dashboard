## Summary

Keep open. Current main still deliberately blocks GitHub sign-in for permanently deleted ClawHub accounts, related reports remain open, and the repository cleanup policy for ClawHub issues does not auto-close this kind of account-recovery/security-policy item.

## What This Changes

The issue requests help signing in or re-registering with GitHub after the reporter accidentally deleted their ClawHub account.

## Best Possible Solution

Define a maintainer-approved recovery policy, then implement it consistently across OAuth, account deletion state, auth account binding, operator/audit flows, user-facing errors, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: delete a ClawHub account through settings, then complete GitHub OAuth again; handleDeletedUserSignIn rejects the deactivated user before normal session setup completes.

## Solution Assessment

No for an autonomous source patch. The maintainable solution first needs a product/security decision on whether recovery, provider unlinking, or operator-only cleanup should exist and how it preserves identity, deletion, ban, token, ownership, and audit invariants.
