## Summary

Keep open. This is an open ClawHub issue, and current main still intentionally blocks GitHub reauth after self-deletion; resolving it needs maintainer account-recovery policy and security judgment rather than cleanup closure.

## What This Changes

The issue asks for help signing in or re-registering with GitHub after the reporter accidentally deleted their ClawHub account.

## Best Possible Solution

Maintainers should choose and document the account recovery outcome: keep irreversible deletion with a clear support response, or add an audited admin/support flow for safe reactivation or GitHub unlinking.

## Reproduction Assessment

Yes, from source inspection. A self-deleted account has deactivatedAt/purgedAt set, and GitHub OAuth completion calls handleDeletedUserSignIn, which rejects that record before normal login can complete.

## Solution Assessment

Unclear until maintainers choose the account policy. A narrow code change that clears OAuth binding or reactivates the user would be risky without verification, audit, moderation, token, and published-content rules.
