## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still matches the reported lockout: self-deletion deactivates the account, OAuth sign-in rejects deactivated users, and admin unban does not restore permanently deleted accounts.

## What This Changes

The issue asks for recovery or re-registration after a user deleted their ClawHub account and can no longer sign in with the same GitHub account.

## Best Possible Solution

Define a maintainer-owned account recovery outcome: either keep irreversible deletion as policy with clear support guidance, or add an audited admin/support recovery or unlink flow that preserves ban and moderation boundaries.

## Reproduction Assessment

Yes. The source path is high-confidence: deleteAccount sets deactivatedAt, OAuth completion calls handleDeletedUserSignIn, and tests assert deactivated or legacy deleted users are rejected on reauth.

## Solution Assessment

Unclear. A direct unlink or restore change is not obviously safe until maintainers decide the recovery policy for permanent deletion, OAuth identity binding, bans, tokens, and published content ownership.
