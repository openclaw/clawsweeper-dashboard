## What This Changes

A user requests help signing in or re-registering with GitHub after deleting their ClawHub account.

## Best Possible Solution

Define a maintainer-owned recovery outcome: keep irreversible deletion with clearer support guidance, or add an audited admin/support restore or GitHub-unlink flow that preserves bans, ownership, tokens, audit logs, and deletion warnings.

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, then a later GitHub OAuth completion calls handleDeletedUserSignIn and rejects that deactivated user; I did not run a live OAuth flow.

No for an autonomous source patch. The current behavior matches the documented irreversible-deletion policy, so restoring or unlinking the GitHub identity needs maintainer-approved account, security, and operations semantics first.
