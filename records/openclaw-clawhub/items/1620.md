## Summary

Keep open. Current main still blocks GitHub sign-in for permanently deleted ClawHub accounts, this is an issue rather than an implemented PR, and the recovery path needs maintainer product/security policy before changing auth behavior.

## What This Changes

The issue asks for recovery, unlinking, or re-registration after a ClawHub account was accidentally deleted and the same GitHub identity can no longer sign in.

## Best Possible Solution

Define a maintainer-owned account recovery policy, then either document the irreversible-deletion support response or implement an audited recovery/unlink flow across OAuth, deletion state, admin tooling, docs, and tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: delete a ClawHub account through settings, then complete GitHub OAuth again; handleDeletedUserSignIn rejects the deactivated user before normal session setup completes.

## Solution Assessment

No for an autonomous patch. The maintainable solution first needs a product/security decision on whether recovery, provider unlinking, or operator-only cleanup should exist and how it preserves identity, deletion, ban, token, ownership, and audit invariants.
