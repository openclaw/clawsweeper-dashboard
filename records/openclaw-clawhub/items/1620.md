## Summary

Keep open. Current main still intentionally blocks GitHub sign-in for self-deleted accounts, and the repository cleanup policy for ClawHub keeps issues open; resolving this requires maintainer account-recovery and security policy rather than conservative cleanup closure.

## What This Changes

The issue requests help restoring access or re-registering with GitHub after accidental ClawHub account deletion.

## Best Possible Solution

Maintainers should decide and document the account recovery outcome, either preserving irreversible deletion with a clear support response or adding an audited admin/support flow for safe reactivation or GitHub unlinking.

## Reproduction Assessment

Yes, from source inspection. A self-deleted account has deactivatedAt/purgedAt set, and GitHub OAuth completion calls handleDeletedUserSignIn, which rejects that record before normal login completes.

## Solution Assessment

Unclear until maintainers choose the account policy. A narrow code change that clears OAuth binding or reactivates the user would be risky without verification, audit, moderation, token, and published-content rules.
