## Summary

Keep open. Current main intentionally blocks GitHub re-sign-in after self-deletion, and ClawHub cleanup policy keeps issues open unless a PR is already implemented on main; the recovery or unlink path needs maintainer product and security handling.

## What This Changes

The issue asks for help restoring access or re-registering after a self-deleted ClawHub account remains tied to GitHub sign-in.

## Best Possible Solution

Choose and document a maintainer-owned account recovery outcome: preserve irreversible deletion with a support response, or build an audited recovery/unlink flow that handles identity, bans, tokens, handles, and publisher ownership.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: self-deletion sets deactivatedAt/purgedAt, then GitHub OAuth completion calls handleDeletedUserSignIn and rejects deactivated users.

## Solution Assessment

Unclear until maintainers choose the account policy. A narrow code change that simply clears an OAuth binding or reactivates a user would be unsafe without audited recovery rules.
