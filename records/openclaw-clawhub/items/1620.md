## Summary

Keep open. Current main intentionally blocks sign-in after user-initiated account deletion, and changing that needs maintainer product/security review rather than cleanup auto-close.

## What This Changes

The issue asks for GitHub sign-in or re-registration to work after the reporter accidentally deleted their ClawHub account.

## Best Possible Solution

Define a maintainer-approved account lifecycle policy, then either keep irreversible deletion documented as the support outcome or add an audited recovery/unlink/admin flow that preserves deletion privacy, bans, token revocation, ownership, and audit logs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: delete a ClawHub account, complete GitHub OAuth again with the same bound identity, and handleDeletedUserSignIn rejects the deactivated user before normal session setup can proceed.

## Solution Assessment

No for an autonomous patch. The maintainable path first needs a product/security decision on whether recovery, provider unlinking, clean re-registration, or operator-only cleanup should exist and how it preserves deletion invariants.
