## Summary

Keep open. Current main still treats self-deleted ClawHub accounts as permanently deactivated, blocks GitHub OAuth reauthentication for that state, and has no admin/support recovery or unlink flow; ClawHub cleanup policy also keeps open issues in this review lane.

## What This Changes

The issue asks for help signing back into ClawHub or re-registering with GitHub after accidentally deleting a ClawHub account.

## Best Possible Solution

Define a maintainer-owned account recovery policy, then either document irreversible deletion as the supported outcome or add an audited support/admin flow that can reactivate or unlink self-deleted GitHub identities safely.

## Reproduction Assessment

Yes, from source inspection. The path is deleteAccount setting deactivatedAt/purgedAt, OAuth completion calling handleDeletedUserSignIn, and tests asserting that deactivated or legacy self-deleted users are blocked from reauth.

## Solution Assessment

Unclear until maintainers choose the account policy. The narrow maintainable path is either explicit support documentation for irreversible deletion or a deliberately designed recovery/unlink flow with audit and moderation safeguards.
