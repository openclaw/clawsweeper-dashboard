## Summary

Keep open. Current main still matches the reported account lockout, and ClawHub review policy keeps issues open; resolving this requires an account-recovery/security policy decision rather than conservative cleanup.

## What This Changes

The reporter asks for ClawHub account access to be restored or re-registration to work after deleting the account linked to GitHub.

## Best Possible Solution

Define the account recovery policy first; then either document the irreversible-deletion support response or add an audited admin/support reactivation or GitHub-unlink flow with moderation and identity safeguards.

## Reproduction Assessment

Yes, from source inspection. The deletion path writes deactivatedAt/purgedAt, and the OAuth callback calls handleDeletedUserSignIn, which rejects that state before login can complete.

## Solution Assessment

Unclear until maintainers choose the account policy. Current main is internally consistent with irreversible-deletion docs, while recovery or unlinking would be a deliberate support/security feature rather than a narrow bug fix.
