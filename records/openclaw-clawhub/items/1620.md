## Summary

Keep open. Current main still deliberately blocks GitHub sign-in for permanently deleted ClawHub accounts, the request is part of an active deleted-account recovery cluster, and resolving it requires maintainer/operator policy for identity recovery rather than conservative cleanup closure.

## What This Changes

The issue asks ClawHub maintainers to restore access or allow GitHub re-registration after an accidental user-initiated account deletion.

## Best Possible Solution

Choose and document a maintainer-approved account recovery policy: either keep irreversible deletion with clear support guidance, or implement an audited operator/admin restore or GitHub-unlink flow that preserves bans, ownership, tokens, privacy, and audit history.

## Reproduction Assessment

Yes, from source inspection. deleteAccount sets deactivatedAt/purgedAt, the GitHub OAuth completion callback calls handleDeletedUserSignIn, and the handler rejects that account state before sign-in can complete; I did not run a live OAuth flow.

## Solution Assessment

No for an autonomous source patch. Current main matches the documented irreversible-deletion policy, and the maintainable fix requires a maintainer-approved recovery or no-recovery policy before code changes.
