## Summary

Keep open. ClawHub policy keeps issues open during cleanup, and current main still intentionally blocks GitHub sign-in after user-initiated account deletion; changing that needs maintainer product and security policy rather than an autonomous fix.

## What This Changes

The issue asks for GitHub sign-in or re-registration to work again after the reporter accidentally deleted their ClawHub account.

## Best Possible Solution

Define the account lifecycle policy, then either document irreversible deletion as the supported outcome or add an audited recovery/unlink flow that preserves auth, deletion, moderation, token, and ownership invariants.

## Reproduction Assessment

Yes. By source inspection, delete a ClawHub account, complete GitHub OAuth again, and the auth callback rejects the deactivated user before session setup completes.

## Solution Assessment

No for an autonomous fix. The maintainable path first needs a maintainer product/security decision on whether recovery, unlinking, clean re-registration, or operator-only handling should exist.
