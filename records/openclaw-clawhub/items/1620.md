## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still matches the reported lockout: self-deletion creates an irreversible deactivated account, OAuth sign-in rejects that state, and existing admin unban/restore paths do not recover permanently deleted accounts.

## What This Changes

The reporter asks for help signing in or re-registering with GitHub after accidentally deleting their ClawHub account.

## Best Possible Solution

Define the account recovery policy first; if recovery is allowed, implement an audited admin/support reactivation or GitHub-unlink flow that preserves ban, purge, token, and published-content boundaries.

## Reproduction Assessment

Yes, from source inspection. The path is deleteAccount setting deactivatedAt/purgedAt, OAuth completion calling handleDeletedUserSignIn, and tests asserting that deactivated or legacy self-deleted users are blocked from reauth.

## Solution Assessment

Unclear until maintainers choose the account policy. Current main is internally consistent with documented irreversible deletion, while recovery or unlinking would be a deliberate support/security feature rather than a narrow bug fix.
