## Summary

Keep open. Current main still matches the reported post-deletion GitHub lockout, and resolving same-identity re-registration or recovery needs maintainer account-support and security policy rather than conservative cleanup.

## What This Changes

The issue asks for help signing in or re-registering with GitHub after the reporter accidentally deleted their ClawHub account.

## Best Possible Solution

Maintainers should choose and document a support policy: keep irreversible deletion with a clear support response, or add an audited admin recovery/unlink flow that preserves deletion, ban, OAuth, token, handle, publisher, and skill ownership boundaries.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: self-delete calls deleteAccount, which sets deactivatedAt/purgedAt, and OAuth completion calls handleDeletedUserSignIn, which throws for deactivated users; auth tests assert this behavior.

## Solution Assessment

Unclear. Reactivation or unlinking is not safely a narrow bug fix because current docs and tests intentionally treat deletion as irreversible, so maintainers need to define the recovery policy first.
