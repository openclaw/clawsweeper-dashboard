## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only allows auto-closing pull requests that current main certainly implements. Current main still matches the reported lockout: self-deletion creates a permanently deactivated user record, OAuth completion rejects that record on reauth, and the admin unban path excludes permanently deleted accounts.

## What This Changes

The issue asks for help recovering access or re-registering with GitHub after the reporter accidentally deleted their ClawHub account.

## Best Possible Solution

Define a maintainer-owned account recovery outcome: either keep irreversible deletion as policy and document the supported support response clearly, or add an audited admin/support flow that can reactivate or unlink self-deleted GitHub identities while preserving bans, moderation holds, published content ownership, token revocation, and deletion warnings.
