## Summary

Keep open. Current main still intentionally blocks sign-in after self-deletion, and ClawHub cleanup policy for this repository does not auto-close issues; resolution needs maintainer account-recovery and security-policy handling rather than conservative source cleanup.

## What This Changes

The issue asks ClawHub to restore access or allow re-registration with a GitHub identity after the user accidentally deleted their ClawHub account.

## Best Possible Solution

Define a maintainer-approved recovery outcome: either keep deletion irreversible with a clear support response, or add an audited admin/support flow that restores or unlinks GitHub identities without weakening bans, moderation holds, token revocation, ownership, or deletion warnings.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: self-deletion sets deactivatedAt/purgedAt, and a later GitHub OAuth completion calls handleDeletedUserSignIn and rejects that deactivated user; I did not run a live OAuth flow.

## Solution Assessment

No for an autonomous source patch. The current implementation and docs intentionally make deletion irreversible, so reactivation or unlinking needs maintainer-approved identity, security, and operations semantics first.
