## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still matches the reported lockout: self-deletion makes the user permanently deactivated, OAuth completion rejects that user on reauth, and admin unban is not a recovery path.

## What This Changes

Request help restoring or unlinking a GitHub identity after accidental ClawHub account deletion so the user can sign in or re-register.

## Best Possible Solution

Define a maintainer-owned account recovery outcome: either keep irreversible deletion and document the support response clearly, or add an audited admin/support flow for verified recovery or GitHub unlinking.

## Reproduction Assessment

Yes, this is source-reproducible on current main: `deleteAccount` writes `deactivatedAt`, and the OAuth callback calls `handleDeletedUserSignIn`, which rejects that state. I did not run a live GitHub OAuth flow.

## Solution Assessment

No autonomous code fix is the best solution yet. The narrow maintainable path requires maintainer policy on irreversible deletion, identity verification, bans/moderation, tokens, and published content before implementing recovery or unlinking.
