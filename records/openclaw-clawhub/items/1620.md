## What This Changes

The user asks for access or re-registration after deleting a ClawHub account and then being unable to sign in with GitHub.

## Best Possible Solution

Define a maintainer-owned recovery outcome: either keep irreversible deletion and document the support response, or add an audited admin/support reactivation or GitHub-unlink flow with clear abuse and ownership safeguards.

## Reproduction Assessment

Yes, from source inspection. Running the settings deletion path stores deactivatedAt/purgedAt, and the GitHub OAuth callback calls handleDeletedUserSignIn, which rejects that state.

Unclear until maintainers choose the account policy. Current main is internally consistent with the irreversible-deletion docs, while recovery or unlinking would be a deliberate support/security feature.
