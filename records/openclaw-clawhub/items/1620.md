## Summary

Keep open. Current main still implements the reported lockout: self-deletion writes irreversible deactivation state and GitHub OAuth rejects that user on reauth, while the ClawHub cleanup policy keeps issues open for maintainer/operator resolution.

## What This Changes

The issue requests account access recovery or re-registration after a self-deleted ClawHub account prevents GitHub sign-in.

## Best Possible Solution

Define a maintainer-owned recovery outcome: either keep irreversible deletion and document the support response clearly, or add an audited admin/support restore or unlink flow that preserves bans, ownership, tokens, published content, and deletion warnings.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: delete an account through settings, then complete GitHub OAuth again; handleDeletedUserSignIn rejects the deactivated user before normal session setup completes. I did not run a live OAuth flow.

## Solution Assessment

No for an autonomous source patch. The maintainable path first needs a maintainer decision on whether same-GitHub recovery should exist and how it interacts with irreversible deletion, OAuth identity binding, bans, ownership, tokens, and auditability.
