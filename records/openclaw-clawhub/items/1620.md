## Summary

Keep open. Current main and v0.16.0 still enforce irreversible self-deletion during GitHub OAuth sign-in, and the ClawHub cleanup policy keeps issues open while maintainers decide the account recovery/security policy.

## What This Changes

The user is asking for ClawHub access to be restored or their GitHub identity unbound after accidentally deleting their account.

## Best Possible Solution

Maintainers should choose an account-recovery policy, then either document irreversible deletion as the supported outcome or add an audited admin/support flow for safe restore or GitHub identity unlinking.

## Solution Assessment

No autonomous code fix is clearly best yet. The requested recovery or re-registration outcome conflicts with the current irreversible-deletion spec, so maintainers need to decide the supported policy first.
