## Summary

Keep open. Current main still intentionally blocks GitHub sign-in for permanently deactivated ClawHub accounts, related account-recovery reports remain open, and this ClawHub issue needs maintainer product/security handling rather than cleanup closure.

## What This Changes

The issue asks for GitHub sign-in or account recovery after the reporter accidentally deleted their ClawHub account.

## Best Possible Solution

Define a maintainer-approved recovery policy, then implement or document it consistently across OAuth, account deletion state, operator/audit flows, user-facing errors, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: delete a ClawHub account through settings, then complete GitHub OAuth again; handleDeletedUserSignIn rejects the deactivated user before normal session setup.

## Solution Assessment

No for an autonomous source patch. The maintainable solution first needs a product/security decision on whether recovery, provider unlinking, or operator-only cleanup should exist and how it preserves identity, deletion, ban, token, ownership, and audit invariants.
