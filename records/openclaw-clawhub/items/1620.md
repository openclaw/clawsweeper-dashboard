## Summary

Keep open. Current main still intentionally blocks GitHub sign-in after self-deletion, and the remaining work is maintainer account-recovery policy and production support handling rather than conservative source cleanup.

## What This Changes

The issue requests help signing in or re-registering with GitHub after self-deleting a ClawHub account.

## Best Possible Solution

Adopt an operator-approved account recovery policy, then either keep irreversible deletion with clear support handling or implement an audited restore/unlink flow that preserves bans, ownership, tokens, and moderation boundaries.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `deleteAccount` sets `deactivatedAt`/`purgedAt`, then a later GitHub OAuth completion calls `handleDeletedUserSignIn` and rejects that deactivated user; I did not run a live OAuth flow.

## Solution Assessment

No for an automated source patch. The requested re-registration or unlink behavior changes the documented irreversible-deletion model, so maintainers need to settle account-recovery policy first.
