## Summary

Keep open: current main still intentionally blocks GitHub sign-in for permanently deactivated accounts, and resolving this is an identity-recovery policy/operator decision rather than a safe cleanup close.

## What This Changes

The issue asks for help signing in or re-registering with GitHub after the reporter accidentally deleted their ClawHub account.

## Best Possible Solution

Define a maintainer-owned account recovery policy, then either document irreversible deletion support handling clearly or implement an audited admin/support restore-or-unlink flow that preserves bans, tokens, ownership, audit logs, and deletion warnings.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: delete an account through settings, then a later GitHub OAuth completion reaches handleDeletedUserSignIn and rejects the deactivated user before sign-in completes; I did not run a live OAuth flow.

## Solution Assessment

No for an autonomous source patch. Current behavior matches documented irreversible deletion semantics, so the narrow maintainable path first needs a maintainer decision on recovery, unlinking, identity verification, bans, ownership, and auditability.
