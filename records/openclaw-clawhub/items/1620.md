## Summary

Keep open. Current main now surfaces a clearer blocked-account message, but source, docs, and tests still intentionally block GitHub reauth for permanently deleted accounts; resolving access requires maintainer/operator account-recovery policy, and the ClawHub cleanup policy keeps issues open.

## What This Changes

The issue asks for help signing in or re-registering with the same GitHub account after accidentally deleting a ClawHub account.

## Best Possible Solution

Define a maintainer-approved account recovery outcome: either preserve irreversible deletion with clear support guidance, or add an audited admin/support flow for safe reactivation or GitHub identity unlinking while preserving bans, ownership, tokens, and audit records.

## Reproduction Assessment

Yes, from source inspection. A GitHub-linked account deleted through deleteAccount gets deactivatedAt/purgedAt, and the next OAuth completion calls handleDeletedUserSignIn, which rejects before normal login completes.

## Solution Assessment

No for an autonomous source patch. Current main’s clearer error message is useful, but account restoration or GitHub unlinking is the real solution and needs maintainer security/privacy policy before implementation.
