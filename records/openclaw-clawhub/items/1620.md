## Summary

Keep open. This is an open ClawHub issue with a source-reproducible account lockout in current main, but the repository cleanup policy only auto-closes qualifying pull requests; resolving this needs maintainer product/security policy for deleted-account recovery rather than conservative issue cleanup.

## What This Changes

The issue requests help signing in or re-registering with GitHub after the reporter accidentally deleted their ClawHub account.

## Best Possible Solution

Define a maintainer-owned account lifecycle policy, then either document irreversible deletion as the supported outcome or implement an audited recovery/unlink flow that preserves deletion, identity, moderation, token, and ownership invariants.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: delete a ClawHub account, complete GitHub OAuth again, and handleDeletedUserSignIn rejects the deactivated user before normal session setup can complete.

## Solution Assessment

No for an autonomous cleanup or repair. The maintainable path first needs a product/security decision on whether recovery, provider unlinking, clean re-registration, or operator-only cleanup should exist and how it preserves account deletion invariants.
