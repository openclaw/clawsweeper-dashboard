## Summary

Keep open. Current main still matches the reported lockout after self-deletion, and resolving it needs maintainer product/security policy rather than conservative cleanup.

## What This Changes

The issue asks for a way to sign in or re-register with GitHub after accidentally deleting a ClawHub account.

## Best Possible Solution

Define the account lifecycle policy, then either document irreversible deletion as the supported outcome or add an audited recovery/unlink flow that preserves auth, deletion, moderation, token, and ownership invariants.

## Reproduction Assessment

Yes, by source inspection: delete a ClawHub account, complete GitHub OAuth again, and the OAuth callback calls handleDeletedUserSignIn, which rejects deactivated users before session setup completes. I did not run a live OAuth flow, so this is source-reproducible rather than live-reproduced.

## Solution Assessment

No for an autonomous fix. The maintainable path first needs a maintainer product/security decision on whether recovery, unlinking, clean re-registration, or operator-only handling should exist and how it preserves deletion invariants.
