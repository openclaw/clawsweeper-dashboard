## Summary

Keep open. The ClawHub cleanup policy says issues stay open, and current main still matches the reported account lockout rather than providing a recovery or GitHub-unlink path.

## What This Changes

The issue asks for help restoring access or re-registering with GitHub after accidentally deleting a ClawHub account.

## Best Possible Solution

Maintainers should choose the account lifecycle policy: keep irreversible deletion and document the support response clearly, or add an audited admin/support recovery or GitHub-unlink flow with abuse and identity safeguards.

## Reproduction Assessment

Yes. From source inspection, deleting an account writes deactivatedAt/purgedAt, and the GitHub OAuth callback calls handleDeletedUserSignIn, which rejects that deactivated user; tests assert the same blocking behavior.

## Solution Assessment

Unclear until maintainers choose the account policy. A narrow reactivation or OAuth-unlink patch would be risky without audited recovery rules for identity, bans, deleted data, and published content.
