## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still matches the reported lockout: self-deletion creates a permanently deactivated account, OAuth sign-in rejects that account, and the available unban path is not a recovery path for deactivated users.

## What This Changes

The issue requests help restoring access or re-registering with GitHub after accidentally deleting a ClawHub account.

## Best Possible Solution

Define a maintainer-owned account recovery outcome: either keep irreversible deletion as product policy with clear support/docs wording, or add an audited admin/support flow that can reactivate or unlink self-deleted GitHub identities without weakening bans or ownership boundaries.

## Reproduction Assessment

Yes. Current main has a high-confidence code-level reproduction: call deleteAccount for a GitHub-linked user, then complete OAuth for that user and handleDeletedUserSignIn rejects the deactivated record; the inspected auth tests assert that path.

## Solution Assessment

No patch is proposed here. A one-off purge or unlink request is not the best source-level solution; the maintainable path is an explicit audited recovery policy or a documented irreversible-deletion support response.
