## Summary

Keep open. Current main still blocks GitHub sign-in after self-deletion by design, related account-recovery reports remain open, and ClawHub cleanup policy keeps issues open unless a maintainer resolves the support/policy path.

## What This Changes

The issue asks for help restoring access or re-registering with GitHub after accidentally deleting a ClawHub account.

## Best Possible Solution

Define a maintainer-owned recovery outcome: either keep irreversible deletion and document the support response, or add an audited admin/support restore or unlink flow that preserves bans, ownership, tokens, published content, and deletion warnings.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: delete an account through settings, then a later GitHub OAuth completion calls handleDeletedUserSignIn and rejects the deactivated user before login completes; I did not run a live OAuth flow.

## Solution Assessment

No for an autonomous source patch. The narrow maintainable path first needs a maintainer decision on whether same-GitHub recovery should exist and how it interacts with irreversible deletion, OAuth identity binding, bans, ownership, tokens, and auditability.
