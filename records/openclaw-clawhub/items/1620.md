## Summary

Keep open. Current main still matches the reported account lockout, and the ClawHub cleanup policy keeps issues open unless they are PRs already implemented on main; recovery or unlinking needs maintainer product/security judgment.

## What This Changes

The issue asks for help signing in or re-registering with GitHub after accidentally deleting a ClawHub account.

## Best Possible Solution

Maintainers should choose a support policy: either keep irreversible deletion and document the supported response clearly, or add an audited admin/support recovery or GitHub-unlink flow that preserves deletion, ban, token, handle, ownership, and verification boundaries.

## Reproduction Assessment

Yes. From source inspection, deleteAccount sets deactivatedAt/purgedAt, and OAuth completion invokes handleDeletedUserSignIn, which rejects deactivated users during GitHub sign-in.

## Solution Assessment

Unclear until maintainers choose the account policy. A narrow change that simply clears the OAuth binding or reactivates the user would not be the safest solution without audited recovery rules.
