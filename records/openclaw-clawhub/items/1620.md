## Summary

Keep open. Current main intentionally blocks GitHub reauth for permanently deleted ClawHub accounts and now surfaces clearer blocked-account copy, but restoring access or unlinking the GitHub identity remains a maintainer/operator policy decision for this open issue.

## What This Changes

The reporter cannot sign in or re-register with GitHub after accidentally deleting their ClawHub account and asks for access recovery.

## Best Possible Solution

Maintainers should either preserve irreversible deletion with clear support guidance, or approve an audited admin/support recovery or GitHub-unlink flow that preserves deletion, ban, token, ownership, and audit invariants.

## Reproduction Assessment

Yes, from source inspection. deleteAccount writes deactivatedAt/purgedAt, and the OAuth completion callback calls handleDeletedUserSignIn, which rejects that deactivated user before sign-in completes.

## Solution Assessment

No for an autonomous source patch. Current main matches the documented irreversible-deletion policy; the requested recovery needs maintainer-approved account, security, and operations policy before implementation.
