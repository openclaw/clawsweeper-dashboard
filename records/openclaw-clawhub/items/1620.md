## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still intentionally matches the reported lockout: self-deletion creates a permanently deactivated user, OAuth completion rejects that user on reauth, and admin unban does not restore deactivated accounts.

## What This Changes

The reporter asks for help signing in or re-registering with GitHub after accidentally deleting their ClawHub account.

## Best Possible Solution

Define a maintainer-owned account recovery outcome: either keep irreversible deletion and document the support response clearly, or add an audited admin/support flow that can reactivate or unlink self-deleted GitHub identities without weakening ban and moderation boundaries.

## Reproduction Assessment

Yes, from source inspection. deleteAccount writes deactivatedAt/purgedAt, OAuth reauth calls handleDeletedUserSignIn and rejects deactivated users, and tests assert the blocked reauthentication path.

## Solution Assessment

Unclear until maintainers choose the account policy. Current main is internally consistent with irreversible deletion, while the requested recovery would require a deliberate support/security design rather than a narrow bug fix.
