## Summary

Keep open. Current main still treats self-deleted ClawHub accounts as permanently deactivated and rejects the later GitHub OAuth completion, and the ClawHub cleanup policy for this review keeps issues open for maintainer/operator resolution rather than auto-closing them.

## What This Changes

The user asks for help signing in or re-registering with GitHub after accidentally deleting their ClawHub account.

## Best Possible Solution

Define a maintainer-owned recovery outcome: either keep irreversible deletion and document the support path, or add an audited admin/support restore or unlink flow that preserves bans, ownership, tokens, published content, and deletion warnings.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: delete an account through settings, then complete GitHub OAuth again; handleDeletedUserSignIn rejects the deactivated user before login completes. I did not run a live OAuth flow.

## Solution Assessment

No for an autonomous source patch. The narrow maintainable path first needs a maintainer decision on whether same-GitHub recovery should exist and how it interacts with irreversible deletion, OAuth identity binding, bans, ownership, tokens, and auditability.
