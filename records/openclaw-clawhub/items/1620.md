## Summary

Keep open. Current main still intentionally blocks GitHub sign-in after self-deletion, and ClawHub cleanup policy keeps issues open; resolving this needs maintainer/operator account-recovery and security-policy handling, not source-only cleanup closure.

## What This Changes

The issue asks for account recovery or re-registration after deleting a ClawHub account linked to GitHub.

## Best Possible Solution

Define a maintainer-owned recovery outcome: either keep irreversible deletion with clearer support guidance, or add an audited admin/support restore or GitHub-unlink flow that preserves bans, ownership, tokens, audit logs, and deletion warnings.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, then a later GitHub OAuth completion calls handleDeletedUserSignIn and rejects that deactivated user; I did not run a live OAuth flow.

## Solution Assessment

No for an autonomous source patch. Current main matches the documented irreversible-deletion policy, so restoring or unlinking the GitHub identity needs maintainer-approved account, security, and operations semantics first.
