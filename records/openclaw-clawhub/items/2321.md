## Summary

Keep this open: this collaborator-authored moderation/security RFC is still an active policy decision. Current main has public acceptable-usage, moderation, audit, and scanner-scope docs that cover parts of the request, but it does not fully settle the category-to-consequence matrix for crypto/blockchain boundaries, quality/spam handling, discovery, installability, upload review, and account enforcement.

## What This Changes

The issue requests an RFC discussion to define ClawHub content policy categories and map them to visibility, registry, upload-review, installability, and enforcement outcomes.

## Best Possible Solution

Maintainers should synthesize a category-to-consequence matrix, land public policy in docs/acceptable-usage.md and docs/moderation.md, and keep reviewer-only scanner/enforcement invariants in specs/security-moderation.md.
