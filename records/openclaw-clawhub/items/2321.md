## Summary

Keep this open: this collaborator-authored moderation/security RFC is still an active product and security-policy decision. Current main and v0.21.0 include broad acceptable-usage guidance, Security Audit/ClawScan context, and moderation-state mechanics, but not the requested category-to-enforcement matrix for visibility, installability, public policy, reviewer guidance, and account action.

## What This Changes

The issue asks maintainers to turn content-policy categories into clear review buckets covering visibility, registry inclusion, upload review, public policy, reviewer-only guidance, and account enforcement.

## Best Possible Solution

Maintainers should post an RFC synthesis, then land accepted category wording in `docs/acceptable-usage.md` and preserve scanner/moderation invariants in `specs/security-moderation.md`.
