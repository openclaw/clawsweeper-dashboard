## Summary

This remains an active, collaborator-authored security and product-policy RFC: current main implements moderation controls and public operational guidance, but it does not contain the referenced category policy or resolve the discussion’s substantive distinctions between enforcement categories, positive trust signals, scanner findings, financial/trading content, and on-chain identity use cases.

## What This Changes

Define content-policy categories and specify how they affect discovery, registry inclusion, installability, human review, removal, and account enforcement.

## Best Possible Solution

Reach an explicit maintainer decision that separates enforcement categories from positive trust signals, defines narrow allowed and disallowed examples plus scanner-evidence rules, records private enforcement invariants in a durable moderation spec, and publishes only the appropriate user-facing policy and appeal guidance.
