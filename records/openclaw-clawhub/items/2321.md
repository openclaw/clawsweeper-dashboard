## Summary

Keep this open: this is a collaborator-authored, security-sensitive RFC, and current main has broad acceptable-usage and scanner-context guidance but not the requested category-to-enforcement matrix. The live discussion still needs maintainer product and security synthesis, especially around blockchain/finance boundaries, positive trust signals, and what belongs in public policy versus reviewer guidance.

## What This Changes

The RFC asks ClawHub to define content-policy review buckets and map categories such as spam, crypto/finance/trading, and malicious scan findings to visibility, registry inclusion, upload review, and account-enforcement outcomes.

## Best Possible Solution

Maintainers should post an RFC synthesis, then land the accepted category matrix in public acceptable-usage docs and preserve reviewer-only scanner or moderation invariants in the security-moderation spec.
