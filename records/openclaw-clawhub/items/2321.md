## Summary

Keep this open: this collaborator-authored moderation/security RFC still needs maintainer product and security-policy synthesis. Current main and v0.21.0 contain broad acceptable-usage guidance, scanner-context rules, and moderation mechanics, but not the requested category-to-enforcement matrix for visibility, installability, public policy, reviewer guidance, and account action.

## What This Changes

This issue asks maintainers to define content-policy categories as review buckets and map them to registry visibility, upload review, installability, reporting, and account-enforcement consequences.

## Best Possible Solution

Maintainers should post an RFC synthesis, decide the category-to-consequence matrix, land public policy in docs, and preserve scanner/moderation invariants in specs.
