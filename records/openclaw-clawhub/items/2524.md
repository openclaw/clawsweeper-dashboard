## What This Changes

The issue requests replacing the production GitHub OAuth App used for ClawHub sign-in with an official ClawHub/OpenClaw-owned app and rotating the Convex production OAuth credentials.

## Best Possible Solution

Have a maintainer/operator create or transfer the production GitHub OAuth App to official OpenClaw/ClawHub ownership, rotate `AUTH_GITHUB_ID` and `AUTH_GITHUB_SECRET` together, and verify both web and CLI sign-in against the live consent screen.
