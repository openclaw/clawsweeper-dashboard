## What This Changes

The issue asks ClawHub to use an official OpenClaw/ClawHub-owned GitHub OAuth App for web and CLI sign-in consent instead of a personal ClawdHub-branded app.

## Best Possible Solution

Use an official org-owned GitHub OAuth App named for ClawHub, configure the production callback/homepage, rotate production Convex `AUTH_GITHUB_ID`/`AUTH_GITHUB_SECRET`, and validate fresh web plus CLI sign-in against the real consent screen.
