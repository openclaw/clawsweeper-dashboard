## What This Changes

The issue reports that the production GitHub OAuth consent screen is branded as a personal ClawdHub app instead of an official ClawHub/OpenClaw OAuth app.

## Best Possible Solution

An authorized maintainer should move production sign-in to an official ClawHub/OpenClaw-owned GitHub OAuth App, rotate the Convex production AUTH_GITHUB_ID and AUTH_GITHUB_SECRET together, and verify both web and CLI auth in a fresh session.

## Solution Assessment

Yes, if maintainers confirm the official owner/account. Changing or transferring the production OAuth App and rotating the existing Convex production credentials is the narrowest durable fix; a source-only PR would not change GitHub's consent branding.
