## What This Changes

The issue asks for GitHub OAuth consent during ClawHub web and CLI sign-in to show official ClawHub/OpenClaw app branding instead of a personal ClawdHub OAuth app.

## Best Possible Solution

Treat this as a production auth-ops/security task: use an official org-owned OAuth App named for ClawHub, point it at the Convex auth callback, rotate production Convex OAuth credentials, and validate web plus CLI sign-in against the real GitHub consent screen.

## Solution Assessment
