## Summary

Keep open: current main and v0.22.0 still route GitHub sign-in through the Convex Auth GitHub provider selected by AUTH_GITHUB_ID/AUTH_GITHUB_SECRET, and source review does not prove the production OAuth app branding has moved to an official OpenClaw-owned app. The issue is collaborator-authored and security/auth-provider sensitive, so it needs maintainer/operator follow-up rather than cleanup.

## What This Changes

The issue requests replacing the production GitHub OAuth app behind ClawHub sign-in with officially branded ClawHub/OpenClaw-owned credentials.

## Best Possible Solution

Have a maintainer create or transfer the production GitHub OAuth app to the official OpenClaw/ClawHub owner, rotate Convex production AUTH_GITHUB_ID and AUTH_GITHUB_SECRET together, confirm callback URLs, and verify web plus CLI sign-in against the real consent screen.
