## What This Changes

The issue requests moving production GitHub OAuth sign-in to an official ClawHub/OpenClaw-owned OAuth app so the consent screen no longer shows personal ClawdHub branding.

## Best Possible Solution

Have a maintainer create or transfer the production GitHub OAuth app to the official OpenClaw/ClawHub owner, rotate Convex production AUTH_GITHUB_ID and AUTH_GITHUB_SECRET together, and verify both web and CLI sign-in against the real consent screen.
