## Summary

Because this is collaborator-authored, security-labeled, assigned operational work affecting production credentials, it needs explicit maintainer handling rather than closure or an automated source fix.

## What This Changes

Replace the personal-looking GitHub OAuth consent identity with an official ClawHub/OpenClaw-owned OAuth App for web and CLI authentication.

## Best Possible Solution

Move the GitHub OAuth App under official organization control, preserve the deployed callback contract, rotate the Convex production credentials through the manual operations path, and retain before/after proof of successful web and CLI authorization with the official consent identity.

## Solution Assessment

Yes. Correcting ownership and branding on the existing external OAuth App boundary, then rotating credentials, is narrower and more maintainable than changing ClawHub application code, which cannot set GitHub's consent owner label.
