## Summary

Keep open: current main still omits package `openclaw.environment` from package LLM eval prompt metadata, and this PR is directionally useful but introduces two blocking issues in the new metadata helper.

## What This Changes

The PR parses package.json `openclaw.environment` into env/config prompt metadata for package LLM evaluation and adds unit coverage.

## Best Possible Solution

Land a sanitized mapper that forwards only normalized environment-derived prompt fields, preserves valid existing requirements, ignores malformed unrelated metadata, and adds regression coverage for those cases.

## Reproduction Assessment

Yes, by source inspection: current main derives `environment:declared` tags from package `openclaw.environment` but does not pass that metadata into the prompt context consumed by `assembleEvalUserMessage`. I did not execute tests because this review is read-only.

## Solution Assessment

No, not as written: the requested direction is appropriate, but the PR should sanitize the forwarded metadata and merge config paths into existing requirements instead of spreading and overwriting package `openclaw` data.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Sanitize forwarded OpenClaw metadata before prompting:** `convex/llmEval.ts:433`
  - body: `buildOpenClawMetadataForEval` can return unrelated package `openclaw` fields such as a malformed string `install`. `assembleEvalUserMessage` treats `install` as an array and calls `.map`, so this can abort package LLM evaluation before analysis is persisted; build a sanitized env/config-only object or normalize unrelated fields before forwarding.
  - confidence: 0.9
- **[P2] Preserve existing requirements when adding config paths:** `convex/llmEval.ts:79`
  - body: When `environment.configPaths` is present, the helper replaces any existing `openclaw.requires` object with `{ config }`. Packages that also declare `requires.env`, `bins`, or `anyBins` would lose those requirement signals in the ClawScan prompt, so merge config paths into a sanitized requirements object instead.
  - confidence: 0.84

## Security Review

Status: needs_attention
