## Summary

Keep open: this security-labeled RFC is still an active maintainer policy question, and current main has trust primitives but no documented vetted-community review path or status semantics.

## What This Changes

Define whether and how ClawHub should support a documented vetted-community review status for third-party plugins without making them official OpenClaw packages.

## Best Possible Solution

Decide and record the vetted-community policy first, then expose it consistently through specs, schema, admin workflow, public trust APIs, docs, and UI as distinct from both official status and release moderation.

## Reproduction Assessment

Not applicable: this is a security/product RFC, not a report of broken behavior. Source and docs inspection confirms current main lacks the requested public vetted-community path.

## Solution Assessment

Yes, keeping this open for maintainer policy review is the right path. A code PR before the criteria, authority, scope, and revocation semantics are settled would encode a trust signal prematurely.
