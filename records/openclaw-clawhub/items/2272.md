## Summary

Keep open: this is a security-labeled RFC that asks for a new public trust/review policy, and current main has related trust primitives but no vetted-community status or documented public review path.

## What This Changes

The issue requests a documented public path and semantics for vetted community plugins distinct from official packages and ordinary source-linked community packages.

## Best Possible Solution

Decide the vetted-community policy first, then persist it in specs and expose it consistently through schema, admin workflows, public trust APIs, docs, and UI without making it visually equivalent to official packages.
