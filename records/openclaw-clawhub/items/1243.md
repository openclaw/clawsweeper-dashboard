## Repair Work Prompt

Fix tag-aware skill search recall for ClawHub. Treat normalized skill version tag names (`Object.keys(skill.tags)`) and `capabilityTags` as lexical match candidates in the current `searchSkills` vector-hydrated path and `lexicalFallbackSkills` path, so queries such as `phone` and `voice` can find skills that carry those tags. Preserve existing moderation and `nonSuspiciousOnly` behavior; do not clear or reinterpret suspicious flags. Coordinate behavior with #1254 rather than adding a one-off special case for one slug, and leave #1326 as the moderation review. Avoid unbounded full-document scans; use existing lightweight digest fields or, if schema changes are necessary, add indexed normalized fields with a cursor-based backfill. Add focused regression tests in `convex/search.test.ts` for tag-only matches and for the existing fallback/vector paths. No changelog entry is expected for this targeted backend search fix unless maintainers request one.
