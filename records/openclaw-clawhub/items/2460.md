## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Rate-limit feed queries before running them:** `convex/httpApiV1/feedsV1.ts:397`
  - body: The lane handlers call `ctx.runQuery` without first using the established `applyRateLimit(ctx, request, "read")` path used by other public read APIs. Because these routes are unauthenticated and the query can read thousands of digest rows, repeated requests bypass the registry throttling boundary; apply the read limit before feed work and return rate headers with successful responses.
  - confidence: 0.94
- **[P1] Reuse the public skill catalog visibility guard:** `convex/feeds.ts:111-112`
  - body: `rootFeed` emits skill digests after local moderation checks, but current public package catalog code also requires visible owner state through `digestToOwnerInfo(digest)?.owner`. Without that guard, feed lanes can expose active skill digests whose owner/profile state makes them invisible in the existing public catalog.
  - confidence: 0.91
- **[P2] Bound feed reads to the requested page size:** `convex/feeds.ts:78`
  - body: Even `?limit=1&type=skill` reads `MAX_ROOT_FEED_ITEMS` skill digests, and the plugin path can read two 5,000-row package-family pages before filtering. This makes the public route cost scale with catalog size rather than request size; read only enough indexed rows for the requested lane/type or add cursor-backed pagination.
  - confidence: 0.89
