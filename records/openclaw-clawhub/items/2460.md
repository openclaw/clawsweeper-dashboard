  - body: The `reviewed` lane accepts every active, non-deprecated skill whose legacy `isSuspicious` flag is not true.   - confidence: 0.94
- **[P1] Do not present an unsigned content hash as an attestation:** `convex/httpApiV1/feedsV1.ts:361-364`
  - body: This value is only SHA-256 of the response body; it has no signature or trusted key, so it cannot attest that ClawHub produced or approved the feed. Because the index marks it required and the docs recommend pinned trust, consumers can mistake integrity for authenticity. Rename/document it as an unsigned content digest, or add a verifiable signature and key-discovery contract.
  - confidence: 0.93
- **[P2] Add a durable specification for feed trust policy:** `convex/httpApiV1/feedsV1.ts:25-31`
  - body: The patch creates public `official` and `reviewed` classifications from moderation, scanning, and verification data but records their rationale only in handler constants and user docs. Repository policy requires security-sensitive moderation, scanner, installability, and API trust-boundary intent in `specs/`; without that contract future changes can silently alter what clients trust. Add a focused spec and align tests/docs to it.
  - confidence: 0.97
