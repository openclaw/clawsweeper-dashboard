Full review comments:

- **[P1] Keep the root feed query internal:** `convex/feeds.ts:59`
  - body: `rootFeed` is registered with `query`, so it becomes a public Convex function callers can invoke directly with high limits, bypassing the HTTP route's rate-limit/cache/attestation contract. Since only the HTTP action should need this raw digest query, register it as an internal query and call it through `internal.feeds.rootFeed`.
  - confidence: 0.88
- **[P1] Serve root feeds from published snapshots:** `convex/httpApiV1/feedsV1.ts:405-410`
  - body: This handler builds every public feed request by running the live `rootFeed` query, which can read up to 5,000 skill digests plus 5,000 rows for each plugin family before filtering. The existing hosted-feed contract says feed responses should come from refreshed stored publications with ETag/CDN caching, so this path should move onto that model or equivalent protection before merge.
  - confidence: 0.9
- **[P1] Use a distinct ID for the root official feed:** `convex/httpApiV1/feedsV1.ts:18`
  - body: `clawhub-official` already identifies the hosted feed with schema version 2. Advertising a new schema-v1 root official feed with the same id can make clients that pin, cache, or compare feeds by id treat incompatible sources as the same feed.
  - confidence: 0.86
- **[P2] Add the feed routes to the OpenAPI contract:** `docs/http-api.md:139`
  - body: The PR documents and exposes new public v1 feed routes, but `public/api/v1/openapi.json` still omits the paths and response schemas. Clients using the published OpenAPI document will not discover or validate this API family.
  - confidence: 0.82
