## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Rate-limit feed queries before running them:** `convex/httpApiV1/feedsV1.ts:385-388`
  - body: The new feed handler calls `ctx.runQuery` without first using the established `applyRateLimit(ctx, request, "read")` path used by other public read APIs. Because these routes are unauthenticated and the query can read thousands of digest rows, repeated requests bypass the registry's throttling boundary; apply the read rate limit to the index and lane handlers and return the rate headers with successful responses.
  - confidence: 0.93
- **[P1] Reuse the public skill catalog visibility guard:** `convex/feeds.ts:106-108`
  - body: `rootFeed` filters skills with only local moderation checks and then emits them, but current public package catalog code also requires `digestToOwnerInfo(digest)?.owner` through `isVisibleSkillCatalogDigest`. Without that guard, feed lanes can expose active skill digests whose owner/profile state makes them invisible in the existing public catalog; reuse the same catalog visibility path before emitting skill feed entries.
  - confidence: 0.9
- **[P2] Bound feed reads to the requested page size:** `convex/feeds.ts:79`
  - body: Even `?limit=1&type=skill` reads `MAX_ROOT_FEED_ITEMS` skill digests, and the all-plugin path reads two 5,000-row package-family pages before filtering. This makes the new public route cost scale with catalog size rather than request size; read only enough indexed rows for the requested lane/type or add cursor-backed pagination.
  - confidence: 0.88
- **[P2] Build feed index URLs from the public API origin:** `convex/httpApiV1/feedsV1.ts:348-357`
  - body: The feed index derives advertised URLs from `new URL(request.url).origin`, which can expose Convex/Vercel preview origins instead of the canonical ClawHub API origin. Existing HTTP helpers already centralize this with `publicApiOrigin(request)`; use that path so production feed documents advertise stable public URLs.
  - confidence: 0.78
