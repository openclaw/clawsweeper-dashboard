## Summary

Keep this draft open, but it is not merge-ready: current main does not provide catalog feed lanes, and the branch has meaningful unique work, yet it duplicates security-sensitive catalog policy, performs up to 15,000 digest reads for even tiny responses, lacks the required durable trust-boundary spec and live Convex proof, and currently conflicts with main. The five related OpenClaw consumer and policy PRs are also still open drafts, so maintainers must approve the cross-repository contract before landing it.

## What This Changes

The PR adds five public catalog feed routes, a Convex root-feed query, deterministic feed serialization and SHA-256 attestation, schema route constants, HTTP documentation, and mocked handler tests.

## Best Possible Solution

Establish one maintainer-approved feed contract that derives visibility and trust labels from shared catalog policy, uses bounded indexed pagination proportional to requested output, records its invariants under `specs/`, and ships only after real Convex endpoint and consumer compatibility proof.

## Reproduction Assessment

Yes for the patch defects: source inspection shows that `limit=1` still executes three `.take(5000)` queries, and the feed lanes use locally defined trust predicates that differ from current shared skill and package policy. The feature itself is new behavior, not a reported current-main bug.

## Solution Assessment

No. A root feed can be a useful producer surface, but the maintainable implementation must share canonical visibility and safety policy, bound its database work, and settle the cross-repository contract before exposing stable trust-labeled lanes.
