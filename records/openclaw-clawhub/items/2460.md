## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.98

Full review comments:

- **[P1] Make the feed data query internal:** `convex/feeds.ts:52`
  - body: `rootFeed` uses the public `query` wrapper and the HTTP handler invokes it through `api`, so clients can call the raw digest query directly and bypass the sanitized feed shape, cache policy, and attestation contract. Export an internal query and invoke it through `internal` instead.
  - confidence: 0.99
- **[P1] Rate-limit feed reads before querying:** `convex/httpApiV1/feedsV1.ts:394-408`
  - body: The new unauthenticated handler runs the catalog query without the public-read limiter used by existing API handlers. Because one request can read thousands of digest rows, anonymous callers can repeatedly consume backend capacity without the normal rate-limit rejection or headers.
  - confidence: 0.99
- **[P1] Use canonical catalog visibility guards:** `convex/feeds.ts:13-50`
  - body: These matchers create a parallel publication policy from only a subset of digest fields. They can drift from current owner resolution, moderation, release availability, security blocking, and installability rules and expose entries that canonical catalog APIs suppress; derive feed rows through shared public conversion/visibility helpers.
  - confidence: 0.92
- **[P2] Bound feed reads to the requested result size:** `convex/feeds.ts:70-95`
  - body: The response limit does not constrain source reads: each enabled digest query still takes up to 5,000 rows before filtering and slicing. A request for one all-types item can therefore read up to 15,000 documents; use indexed pagination that stops once enough matching entries are collected.
  - confidence: 0.99
- **[P2] Document the feed trust boundary in specs:** `docs/http-api.md:139`
  - body: The patch defines moderation-derived public lanes and an attestation contract but only updates user-facing HTTP docs. Repository policy requires durable internal intent for security-sensitive API boundaries, including the exact meaning of `reviewed` and what the hash does not guarantee.
  - confidence: 0.99
