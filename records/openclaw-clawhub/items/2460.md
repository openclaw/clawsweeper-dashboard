## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Make the feed data query internal:** `convex/feeds.ts:59`
  - body: Registering `rootFeed` with `query` exposes raw digest rows as a public Convex API, bypassing the HTTP feed shape, attestation, cache headers, and route-level safeguards. Use an `internalQuery` behind the HTTP handler or return only the sanitized feed contract from any public surface.
  - confidence: 0.92
- **[P1] Rate-limit feed reads before querying:** `convex/httpApiV1/feedsV1.ts:405-408`
  - body: The feed handler calls `ctx.runQuery` without the shared public read limiter used by current catalog HTTP endpoints. Apply `applyRateLimit(ctx, request, "read")` before database work and carry its headers through success and error responses.
  - confidence: 0.9
- **[P1] Use catalog visibility guards for feed entries:** `convex/feeds.ts:111-112`
  - body: Filtering digest rows with local feed checks can drift from the canonical catalog rules for owner resolution, public conversion, hidden rows, and moderation state. Build feed entries through the existing catalog visibility helpers or equivalent shared guards before exposing them publicly.
  - confidence: 0.86
- **[P2] Bound feed reads to the requested page size:** `convex/feeds.ts:72-97`
  - body: Even `limit=1` reads up to 5,000 skills and up to 5,000 rows from each plugin family before filtering and slicing. Use paginated or page-sized catalog reads so small feed requests do not burn large read budgets.
  - confidence: 0.88
