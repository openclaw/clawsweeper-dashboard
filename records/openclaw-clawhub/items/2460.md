Full review comments:

- **[P1] Make the feed data query internal:** `convex/feeds.ts:59`
  - body: `rootFeed` uses the public `query` wrapper, so any Convex client can invoke it directly and receive raw digest rows outside the HTTP response shaping, cache policy, and attestation contract. Export it as an internal query and call it through `internal` from the HTTP action.
  - confidence: 0.98
- **[P1] Apply canonical package visibility rules:** `convex/feeds.ts:43-45`
  - body: The feed excludes only `malicious` packages, so `pending`, `not-run`, or otherwise non-installable entries can appear in the `all` and `community` feeds even though current catalog behavior hides unsafe pending packages. Reuse the canonical package eligibility helper rather than maintaining a parallel filter.
  - confidence: 0.94
- **[P1] Rate-limit feed reads before querying:** `convex/httpApiV1/feedsV1.ts:394-408`
  - body: The anonymous handler invokes the broad digest query without checking the public read limiter. A caller can repeatedly trigger large catalog scans despite the five-minute response cache, so apply the established rate boundary before `ctx.runQuery` and return its headers.
  - confidence: 0.97
- **[P2] Bound database reads to the requested result size:** `convex/feeds.ts:72-103`
  - body: A request for a small feed still reads up to 5,000 skills and 10,000 plugin rows before filtering and slicing. Use indexed lane-specific pagination or an overfetch strategy tied to `limit`, with a continuation/truncation rule, instead of always reading every maximum-sized partition.
  - confidence: 0.96
- **[P2] Add a durable feed trust-boundary specification:** `docs/http-api.md:139`
  - body: This new public API defines moderation eligibility, reviewed status, package safety semantics, and attestation behavior, but the patch only adds user-facing HTTP documentation. Add a focused `specs/` note defining these invariants and their relationship to canonical catalog visibility before making the contract public.
  - confidence: 0.93
