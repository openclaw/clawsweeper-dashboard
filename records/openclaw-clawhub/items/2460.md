## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P1] Paginate lanes instead of dropping entries beyond the scan cap:** `convex/feeds.ts:72-112`
  - body: The query takes at most 5,000 rows from each digest index before applying `official`, `community`, or `reviewed` filtering, and the HTTP API has no cursor. Once a digest exceeds that window, matching older entries can never appear in the lane even when callers request the maximum limit; `truncated` only reports the loss without a way to continue. Add cursor-complete or lane-indexed retrieval before presenting these endpoints as catalog feeds.
  - confidence: 0.98
- **[P1] Reuse canonical trust and installability policy for lane membership:** `convex/feeds.ts:12-56`
  - body: These local predicates create a second source of truth for public, official, reviewed, and malicious eligibility. ClawHub's package policy explicitly treats trust as more than an `official` boolean and distinguishes verification scope and installability; future moderation changes would let feed membership drift from browse/install behavior. Move lane eligibility behind shared canonical helpers and add compatibility tests against the existing public catalog rules.
  - confidence: 0.93
