## Summary

Current main still leaves historical ClawPack releases with incomplete review inputs, so the PR remains useful; however, the proposed write path can become permanently stuck after a transient post-mutation failure, and the contributor has not supplied the required real Convex runtime proof.

## What This Changes

The PR adds an admin-only Convex migration that reconstructs incomplete historical ClawPack release file rows from stored artifacts, resets review state, and queues fresh package scans.

## Best Possible Solution

Make repair completion idempotent across file replacement and both scan queues—either transactionally recording resumable repair state or providing a reconciliation path—then prove dry-run, narrow apply, retry, and final review state in a configured Convex deployment.

## Reproduction Assessment

## Solution Assessment

No, not yet. A separate admin-gated, dry-run-first migration is the correct direction, but the current post-mutation scan sequence lacks a reliable retry path and has not been validated in real Convex.
