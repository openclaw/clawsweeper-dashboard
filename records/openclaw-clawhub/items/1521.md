Keep open. Repository policy for this ClawHub cleanup lane keeps issues open, and current main still makes skill publication depend on a successful embedding request after retryable 429/5xx failures are exhausted. The flagged-version cleanup side also requires live production moderation, storage, ownership, and audit checks that cannot be resolved from the source checkout alone.

## What This Changes

The issue asks maintainers to fix a skill-publish failure caused by an OpenAI embedding 500 and to remove or clear flagged versions for the reporter’s skill.

## Best Possible Solution

Maintainers should inspect the OpenAI request ID and production Convex failure logs, verify the live shaymizuno skill ownership, aliases, versions, moderation flags, storage artifacts, and audit history, then resolve any hosted cleanup with staff tooling. If a source change is approved, publishing should record a durable pending-embedding state for exhausted retryable provider outages and backfill embeddings later while still failing configuration errors and non-retryable request failures.
