## Summary

Keep open. Current main and the live API still encode the reported skill license as MIT-0/no-attribution while the upstream package declares MIT, and resolving that mismatch requires maintainer/legal policy direction rather than cleanup closure.

## What This Changes

The issue asks ClawHub to show the reported skill's license as standard MIT instead of MIT-0 because its upstream package and repository declare MIT.

## Best Possible Solution

Maintainers should choose one licensing model and implement it consistently across upload consent, Convex storage, public schemas and APIs, listing UI, CLI output, docs, migrations or backfills, and tests.

## Reproduction Assessment

Yes. The live API still returns MIT-0 for the reported skill while npm and the upstream package.json declare MIT, and current-main source inspection shows the publish/storage/display path hardcodes MIT-0.

## Solution Assessment

No for a display-only label change. The maintainable solution is a coherent license-policy decision followed by matching schema, storage, API, UI, CLI, docs, and migration work.
