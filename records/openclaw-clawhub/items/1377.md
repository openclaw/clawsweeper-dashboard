## Summary

Keep open. The reported mismatch is still valid, but current main intentionally models published skills as a platform-wide MIT-0/no-attribution contract, so changing it safely needs maintainer/legal product direction rather than conservative cleanup; ClawHub issue policy also keeps issues open.

## What This Changes

The issue asks ClawHub to show the reported skill's MIT license instead of MIT-0/no-attribution metadata.

## Best Possible Solution

Maintainers should choose one license model and apply it consistently across upload acceptance, Convex storage, public API schemas, listing UI, CLI output, docs, migrations or backfills, and tests.

## Reproduction Assessment

Yes. The live API still returns MIT-0 for the reported skill while upstream package metadata and LICENSE say MIT, and current main hard-codes MIT-0 across schema, storage, publish, docs, and display paths.

## Solution Assessment

No. A display-only MIT label is not the best fix because it would create a second source of truth; maintainers need to decide between enforced fixed MIT-0 policy and full per-skill SPDX support.
