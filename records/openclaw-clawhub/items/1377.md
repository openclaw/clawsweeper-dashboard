Keep open. Current main and production still report MIT-0 for the named skill while upstream package metadata says MIT, and resolving that mismatch requires a maintainer/legal licensing-policy decision rather than conservative cleanup closure.

## What This Changes

The issue asks ClawHub to show the reported skill's actual MIT license instead of displaying the platform MIT-0/no-attribution license.

## Best Possible Solution

Maintainers should choose one licensing model and implement it consistently across upload consent, Convex storage, public schemas/API responses, listing UI, CLI output, docs, migration/backfill, and regression tests.

## Reproduction Assessment

Yes. The live ClawHub API returns MIT-0 for the reported skill while npm and the upstream package.json declare MIT, and current main hard-codes MIT-0 across schema, storage, publish, docs, CLI, and UI surfaces.

## Solution Assessment

No. A display-only MIT label would create a second source of truth; the maintainable fix needs a maintainer/legal licensing-model decision followed by a full-stack implementation.
