## Summary

Keep open under the ClawHub cleanup policy. The MIT versus MIT-0 mismatch still reproduces, but current main intentionally implements a platform-wide MIT-0 skill license contract, so changing the result requires maintainer/legal product direction rather than conservative cleanup.

## What This Changes

The issue asks ClawHub to show the reported skill's upstream standard MIT license instead of the MIT-0 license currently displayed by ClawHub.

## Best Possible Solution

Maintainers should choose one license model and implement it consistently across upload consent, storage, API schemas, listing UI, CLI output, docs, migrations/backfills, and tests.

## Reproduction Assessment

Yes. The live ClawHub API returns MIT-0 for the reported skill while upstream package and repository metadata declare MIT, and current main has a clear source path that writes, validates, documents, and renders MIT-0.

## Solution Assessment

No for a display-only change to MIT. The maintainable path is a maintainer/legal decision to either enforce the platform MIT-0 contract more clearly or add coherent per-skill SPDX license support across the full stack.
