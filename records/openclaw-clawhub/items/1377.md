Keep open: current main still hard-codes MIT-0/no-attribution for published skills while the reported upstream package declares MIT, and changing that safely needs maintainer/legal product direction rather than cleanup closure.

## What This Changes

The issue asks ClawHub to display standard MIT for `openclaw-workspace-sync` instead of MIT-0/no-attribution, matching the upstream package metadata.

## Best Possible Solution

Maintainers should choose the license model, then either reaffirm MIT-0 with clear author consent or implement per-skill SPDX support consistently across publish consent, storage, API schemas, listing UI, CLI output, docs, tests, and migration/backfill handling.

## Reproduction Assessment

Yes. The live API returns MIT-0 for `openclaw-workspace-sync` while upstream package metadata declares MIT, and current main has a direct source path that writes, validates, documents, returns, and renders the MIT-0 literal.

## Solution Assessment

No for a display-only switch to MIT. The maintainable solution is a maintainer/legal policy decision followed by either a coherent fixed MIT-0 contract or full per-skill SPDX license support.
