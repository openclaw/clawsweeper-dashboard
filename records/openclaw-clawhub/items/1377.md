## Summary

Keep open: the reported MIT versus MIT-0 mismatch is still present, but current main intentionally models ClawHub skills as a platform-wide MIT-0/no-attribution contract, and ClawHub cleanup policy does not auto-close issues.

## What This Changes

The issue requests that ClawHub display the reported skill's actual MIT license instead of the platform MIT-0/no-attribution license.

## Best Possible Solution

Maintainers should choose one license model, then implement it consistently across upload consent, storage, API schemas, listing UI, CLI output, docs, tests, and any migration or backfill path.

## Reproduction Assessment

Yes. The live API returns MIT-0 while npm and GitHub package metadata for the reported package return MIT, and current-main source has a direct hard-coded MIT-0 path through publish, storage, API, docs, and UI.

## Solution Assessment

No for a display-only change to MIT. The maintainable solution is a maintainer/legal product decision followed by either reaffirmed MIT-0 platform terms or coherent per-skill SPDX license support across all affected surfaces.
