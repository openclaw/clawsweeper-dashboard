## Summary

Keep open: the MIT versus MIT-0 mismatch is still visible, but current main implements an intentional platform-wide MIT-0 license contract, so resolving this needs maintainer/legal product direction rather than conservative cleanup.

## What This Changes

The issue asks ClawHub to display the reported skill’s standard MIT license instead of MIT-0/no-attribution text.

## Best Possible Solution

Maintainers should choose one license model and implement it consistently across upload consent, Convex storage, public schemas, API responses, listing UI, CLI output, docs, tests, and any migration/backfill path.

## Reproduction Assessment

Yes for the reported mismatch: live ClawHub still reports MIT-0 for the skill while npm and GitHub metadata report MIT, and current-main source writes, validates, documents, and renders only MIT-0. I did not run the local app, but the source path and live API are enough for high-confidence reproduction.

## Solution Assessment

No for a display-only switch to MIT. The maintainable solution is a maintainer/legal policy decision followed by either reaffirmed MIT-0 enforcement or coherent per-skill SPDX support across all affected surfaces.
