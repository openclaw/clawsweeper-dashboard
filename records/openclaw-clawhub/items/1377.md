## Summary

Keep open. Current main and the latest release still present skill licensing as a platform-wide MIT-0/no-attribution contract, while the reported upstream package still declares MIT; resolving that mismatch needs maintainer/legal product direction rather than conservative cleanup closure.

## What This Changes

The issue asks ClawHub to display the reported skill's upstream MIT license instead of the platform-wide MIT-0 license.

## Best Possible Solution

Define the license model explicitly, then update schema, storage, API, UI, CLI, docs, migrations, and tests so platform license and any upstream/source license cannot be confused.

## Reproduction Assessment

Yes for the observable mismatch: current source renders and stores the fixed MIT-0 platform license, while the reported upstream package currently declares MIT. It is not a strict bug reproduction because current ClawHub docs and schemas intentionally define MIT-0 as the platform license.

## Solution Assessment

No, directly switching this one listing to MIT is not the narrowest maintainable fix because it would bypass the current platform publishing contract. The safer solution is a maintainer-approved model that distinguishes platform licensing from upstream/source licensing if both are shown.
