## Summary

Keep open. Current main still encodes skill licensing as a platform-wide MIT-0/no-attribution contract, so the reported MIT versus MIT-0 listing mismatch remains unresolved and needs maintainer/legal product direction rather than cleanup closure.

## What This Changes

The issue asks ClawHub to display the skill's actual MIT license instead of the platform MIT-0/no-attribution label for `openclaw-workspace-sync`.

## Best Possible Solution

Maintainers should choose one license model and implement it consistently across upload consent, storage/schema, public APIs, listing UI, CLI output, docs, migrations or backfills, and tests.

## Reproduction Assessment

Yes. Source inspection on current main is enough: publish/storage still force `MIT-0`, and the detail header still renders the platform license constant while the report identifies the upstream package as MIT.

## Solution Assessment

No for a display-only label change. The maintainable path is a coherent license-policy change or per-skill SPDX support because the current MIT-0 contract is encoded across several public surfaces.
