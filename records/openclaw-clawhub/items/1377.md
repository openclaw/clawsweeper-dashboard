## What This Changes

The issue asks ClawHub to show standard MIT for `openclaw-workspace-sync` instead of MIT-0 because the upstream package and LICENSE declare MIT.

## Best Possible Solution

Maintainers should choose one license model and implement it consistently across upload acceptance, Convex storage, public API schemas, listing UI, CLI output, docs, migrations or backfills, and tests.

## Reproduction Assessment

Yes. The live ClawHub API returns MIT-0 for the reported skill while upstream package metadata and LICENSE say MIT, and current main hard-codes MIT-0 across the publishing/display path.

## Solution Assessment

No. A display-only MIT label is not the best fix because it would create a second license source of truth; maintainers need to decide between enforced fixed MIT-0 policy and full per-skill SPDX support.
