## Summary

Keep open. The MIT versus MIT-0 mismatch still reproduces, and current main intentionally enforces a platform-wide MIT-0/no-attribution contract, so resolving this needs maintainer/legal product direction rather than cleanup.

## What This Changes

The issue asks ClawHub to display the reported skill's standard MIT license instead of the platform MIT-0/no-attribution license shown on its listing.

## Best Possible Solution

Maintainers should choose one license model and then implement it consistently across publish consent, Convex storage, API schemas, listing UI, CLI output, docs, tests, and any needed migration.

## Reproduction Assessment

Yes. The live API returns MIT-0 for the reported skill while upstream npm/GitHub sources declare MIT, and current main has a clear source path that writes, validates, documents, and renders the MIT-0 literal.

## Solution Assessment

No for a display-only switch to MIT. The maintainable solution is a maintainer/legal policy decision followed by a full-stack license model change if per-skill SPDX licenses are desired.
