## Summary

Keep open. Current main still models skill licensing as a platform-wide MIT-0/no-attribution contract, so the reported MIT versus MIT-0 mismatch remains unresolved; ClawHub policy also keeps issues open for maintainer judgment rather than auto-closing them.

## What This Changes

The issue asks ClawHub to stop displaying a skill package licensed as standard MIT as MIT-0/no-attribution.

## Best Possible Solution

Maintainers should choose one license model and implement it consistently across upload acceptance, Convex storage, API schemas, listing UI, CLI output, docs, migrations or backfills, and tests.

## Reproduction Assessment

Yes. Current main has a high-confidence static reproduction path: the shared constants, public schema, Convex storage validator, publish path, listing header, and docs still hard-code MIT-0/no-attribution behavior.

## Solution Assessment

No. A label-only change to MIT would make the UI disagree with the current platform license contract; the maintainable fix needs a coherent policy decision and cross-surface implementation.
