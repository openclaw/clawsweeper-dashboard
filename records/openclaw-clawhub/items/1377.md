## Summary

Keep open. The report is still valid against current main: ClawHub models published skill licensing as a platform-wide MIT-0/no-attribution contract and renders that constant on listings, while the issue asks for standard MIT attribution-required license handling. Under the supplied ClawHub cleanup policy, issues are not auto-closed, and this one also needs maintainer/legal product direction before changing the platform license model.

## What This Changes

The issue requests that the `openclaw-workspace-sync` listing show standard MIT license terms instead of MIT-0/no-attribution text, and suggests checking whether the mismatch affects other skills.

## Best Possible Solution

Maintainers should choose one authoritative license model, then implement it consistently across upload acceptance, Convex storage, public API schemas, listing UI, CLI output, docs, migrations or backfills, and tests so published listings cannot contradict the license terms users and authors rely on.
