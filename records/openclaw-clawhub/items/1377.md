## Summary

Keep open. Current main, the latest release tag, and the live API still expose published skills under ClawHub's fixed MIT-0 platform license, so showing this skill as MIT remains a maintainer/legal product decision rather than an implemented cleanup item.

## What This Changes

The issue asks ClawHub to show the `openclaw-workspace-sync` skill as MIT instead of MIT-0 because its source package declares MIT.

## Best Possible Solution

Maintainers should either reaffirm the fixed MIT-0 contract with clearer publisher remediation guidance, or deliberately reopen per-skill licensing as a product/legal project spanning publish consent, storage, API, UI, CLI, docs, and migrations.

## Reproduction Assessment

Yes from source inspection and the live API: current main still publishes and documents skills under a fixed MIT-0 platform license, and the live API for the reported skill returns MIT-0. I did not run a local browser because the backend/API/docs path already establishes the current behavior.

## Solution Assessment

No automatic code fix is clearly best yet. Displaying MIT alone would conflict with the current platform license contract unless maintainers first decide to support package-specific licenses.
