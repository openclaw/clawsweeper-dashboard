## Summary

Keep open. Current main still models published skill licensing as a platform-wide MIT-0/no-attribution contract across schema, storage, publish, UI, CLI, tests, and docs, so the reported MIT versus MIT-0 mismatch remains unresolved but needs maintainer/legal product direction rather than a narrow cleanup close.

## What This Changes

Request to display the openclaw-workspace-sync license as standard MIT, or otherwise stop showing MIT-0/no-attribution when upstream package metadata and license text say MIT.

## Best Possible Solution

Maintainers should choose the licensing model, then align publish consent, storage/schema, public API, web UI, CLI output, docs, migrations or backfills, and tests so listings cannot disagree with the actual license contract.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: publishing and storage force MIT-0, and SkillHeader renders the shared MIT-0 constant, though I did not run the app against the live skill page.

## Solution Assessment

Unclear. A display-only MIT change is not the best fix because it would contradict the platform-wide MIT-0 contract; the maintainable solution needs an explicit license policy decision followed by a coherent product-wide implementation.
