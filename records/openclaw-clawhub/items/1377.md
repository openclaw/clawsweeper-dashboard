## Summary

Keep open. Current main and the latest release still present skill licensing as a platform-wide MIT-0/no-attribution contract, while the reported upstream package license is MIT; resolving that mismatch needs maintainer/legal product direction rather than conservative cleanup closure.

## What This Changes

The issue asks ClawHub to display the reported skill's standard MIT license instead of the platform MIT-0/no-attribution license shown on the listing.

## Best Possible Solution

Define a maintainer-approved license model, then update publish acceptance, Convex storage, public API schemas, listing UI, CLI output, docs, specs, migrations or backfills, and tests so platform license and upstream/source license cannot be confused.

## Reproduction Assessment

Yes for the observable mismatch: current source renders and stores the fixed MIT-0 platform license, while the issue reports the upstream package/license sources as MIT. This is source-reproducible rather than manually reproduced in a browser during this read-only review.

## Solution Assessment

No. Directly switching this one listing to MIT would bypass the current platform license contract; the maintainable solution is a coherent policy and data model that distinguishes platform distribution license from upstream/source license if both are shown.
