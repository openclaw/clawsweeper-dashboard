## Summary

Keep open. The reported mismatch is still observable, but current main intentionally encodes MIT-0 as a platform-wide skill license, so resolving this needs maintainer/legal product direction rather than cleanup closure; ClawHub policy also keeps issues open.

## What This Changes

The issue asks ClawHub to show the reported skill's actual MIT license instead of MIT-0/no-attribution metadata.

## Best Possible Solution

Maintainers should choose one license model and implement it consistently across publish consent, schema/storage, public APIs, listing UI, CLI output, docs, migrations or backfills, and tests.

## Reproduction Assessment

Yes. The live API still reports MIT-0 for the reported skill while upstream package/LICENSE sources declare MIT, and current main's source shows the publish/storage/UI/docs path forces MIT-0.

## Solution Assessment

No for a display-only label change. The maintainable fix needs a coherent license-policy decision or per-skill SPDX support because several public contracts currently encode MIT-0.
