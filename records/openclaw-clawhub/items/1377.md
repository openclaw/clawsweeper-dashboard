## Summary

Keep open. Current main still treats published skill licensing as a platform-wide MIT-0/no-attribution contract, so the reported MIT versus MIT-0 mismatch remains unresolved, and the ClawHub cleanup policy keeps issues open for maintainer judgment.

## What This Changes

The issue requests that a skill listing display the upstream package license as standard MIT rather than MIT-0/no-attribution when the package metadata and LICENSE file say MIT.

## Best Possible Solution

Maintainers should choose one license model and implement it consistently across upload acceptance, Convex storage, public API schemas, listing UI, CLI output, docs, migrations or backfills, and tests.

## Reproduction Assessment

Yes. Source inspection on current main shows the listing surface renders the MIT-0 platform constant, while schema, storage, publish, API, docs, and CLI paths still only model MIT-0.

## Solution Assessment

No. A display-only change is not the best fix because it would make the UI disagree with the platform license contract; the maintainable solution is a coherent licensing policy change across all public and stored surfaces.
