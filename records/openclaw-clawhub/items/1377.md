## Summary

Keep open. Current main and the live API still encode the reported skill as MIT-0 while the upstream package declares MIT, and prior maintainer discussion closed per-skill license metadata in favor of a fixed MIT-0 platform policy, so this needs maintainer/legal direction rather than cleanup.

## What This Changes

The issue asks ClawHub to display the skill's upstream MIT license instead of the platform MIT-0/no-attribution license.

## Best Possible Solution

Maintainers should decide whether ClawHub's platform MIT-0 consent overrides upstream package metadata or whether to add per-skill SPDX support, then implement that decision consistently across upload, storage, APIs, UI, CLI, docs, and migrations.

## Reproduction Assessment

Yes. The live skill API returns MIT-0 for the reported skill, and current main has a clear source path that writes, validates, and renders the platform MIT-0 literal; I did not run a local publish/UI session in this read-only review.

## Solution Assessment

No for a display-only MIT label. The current implementation intentionally encodes a platform MIT-0 contract, so the maintainable fix is a policy decision followed by a full-stack license model change if maintainers choose per-skill SPDX support.
