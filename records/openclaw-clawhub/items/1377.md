## Summary

Keep open. Current main still enforces a fixed MIT-0/no-attribution skill license across publishing, storage, API/schema, UI, docs, and tests, while the report says this published package is actually MIT; because ClawHub issues should not be auto-closed and the fix changes licensing policy, this needs maintainer/legal product direction.

## What This Changes

The issue asks ClawHub to display the published skill's actual MIT license instead of the platform MIT-0 label and no-attribution summary.

## Best Possible Solution

Maintainers should choose one license model and implement it consistently across upload consent, Convex storage, public schemas/APIs, listing UI, CLI output, docs, migrations or backfills, and tests.

## Reproduction Assessment

Yes. Source inspection on current main shows publish/storage force MIT-0 and the skill detail header renders the platform constant; the issue supplies the upstream MIT evidence for the affected package.

## Solution Assessment

No for a display-only label change. The maintainable path is a coherent licensing-policy decision or per-skill SPDX support because current main intentionally encodes MIT-0 across several public surfaces.
