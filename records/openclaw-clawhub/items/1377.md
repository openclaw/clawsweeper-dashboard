## Summary

Keep open. Current main still models ClawHub skill licensing as MIT-0/no-attribution across schema, storage, publish, API, UI, CLI, docs, and tests, so the reported MIT versus MIT-0 mismatch remains unresolved and needs maintainer/legal product direction.

## What This Changes

The issue requests that ClawHub display the skill’s actual MIT license instead of MIT-0/no-attribution text.

## Best Possible Solution

Maintainers should choose one license policy and implement it consistently across publishing consent, schema/storage, public APIs, listing UI, CLI output, docs, migrations or backfills, and tests.

## Reproduction Assessment

Yes. Current main reproduces the reported path because publishing stores MIT-0, schema and APIs only model MIT-0, and the skill detail header renders the MIT-0 platform constant.

## Solution Assessment

Unclear. A UI-only change to show MIT is not the best fix because it would contradict the current platform-wide license contract; the maintainable path needs explicit maintainer/legal product direction first.
