## Summary

Keep open. The reported MIT versus MIT-0 mismatch is still observable, but current main intentionally treats published skills as a platform-wide MIT-0/no-attribution contract, so resolving this needs maintainer/legal product direction rather than a label-only code change.

## What This Changes

The issue asks ClawHub to display the reported skill's standard MIT license instead of the platform MIT-0/no-attribution license.

## Best Possible Solution

Choose one licensing model and implement it consistently across upload acceptance, Convex storage, public API schemas, listing UI, CLI output, docs, migrations or backfills, and tests.

## Reproduction Assessment

Yes. Source inspection shows current main stores and renders the platform MIT-0 constant, and live checks show ClawHub returns MIT-0 for the skill while upstream package metadata says MIT.

## Solution Assessment

No. A display-only change is not the best way to solve this because it would create a second license source of truth against the current MIT-0 publish contract; maintainers need to decide the license model first.
