Keep open. The reported mismatch is still valid, but current main intentionally treats published skills as a platform-wide MIT-0/no-attribution contract, so changing the visible license safely needs maintainer/legal product direction; ClawHub issue policy also keeps issues open.

## What This Changes

The issue asks ClawHub to show standard MIT for `openclaw-workspace-sync` instead of MIT-0/no-attribution.

## Best Possible Solution

Maintainers should choose one license model and apply it consistently across upload acceptance, Convex storage, public API schemas, listing UI, CLI output, docs, migrations or backfills, and tests.

## Reproduction Assessment

Yes. The live API returns MIT-0 for the reported skill while npm and the upstream repository declare MIT, and current main hard-codes MIT-0 across schema, storage, publish, docs, CLI, and display paths.

## Solution Assessment

No. A display-only MIT label is not the best fix because it would create a second source of truth; maintainers need to decide between enforced fixed MIT-0 policy and full per-skill SPDX support.
