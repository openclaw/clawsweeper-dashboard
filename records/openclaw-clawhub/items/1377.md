Keep open: the reported mismatch is still observable, and current main intentionally enforces a platform-wide MIT-0 license while the upstream package declares MIT. Resolving this needs maintainer/legal product direction rather than conservative cleanup.

## What This Changes

## Best Possible Solution

Maintainers should choose one license model, then implement it consistently across upload acceptance, storage, API schemas, listing UI, CLI output, docs, tests, and any needed migration.

## Reproduction Assessment

Yes. The live ClawHub API returns MIT-0 for this skill while npm and GitHub metadata return MIT, and current main source stores and renders the platform MIT-0 constant.

## Solution Assessment

No for a display-only switch to MIT. The maintainable solution is a maintainer/legal decision followed by a coherent fixed-MIT-0 enforcement path or per-skill SPDX license support.
