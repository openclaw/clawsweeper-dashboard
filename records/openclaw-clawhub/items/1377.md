## Summary

Keep open. Current main and the latest release still implement a platform-wide MIT-0/no-attribution skill license, while the reported package and npm metadata say MIT, so the central mismatch remains unresolved and needs maintainer/legal product direction rather than cleanup closure.

## What This Changes

The issue asks ClawHub to display the reported skill's actual MIT license instead of the platform MIT-0/no-attribution license.

## Best Possible Solution

Maintainers should choose one license model and implement it consistently across upload acceptance, Convex storage, public API schemas, listing UI, CLI output, docs, migrations or backfills, and tests.

## Reproduction Assessment

Yes. Source inspection and the live ClawHub API show ClawHub still writes and displays MIT-0 for the reported skill, while npm and the upstream package.json report MIT; I did not need a local browser to establish the mismatch.

## Solution Assessment

No automatic display-only fix is clearly best. Showing MIT alone would contradict the current platform license contract unless maintainers decide to support per-skill SPDX licensing and migrate the related surfaces together.
