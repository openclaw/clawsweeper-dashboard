## Summary

Keep open. The reported MIT versus MIT-0 mismatch is confirmed, but current main intentionally models published skills as a platform-wide MIT-0/no-attribution contract, and the related per-skill licensing PR was closed with explicit product direction against license overrides.

## What This Changes

The issue asks ClawHub to display the reported skill's license as standard MIT instead of MIT-0/no-attribution and to check whether this affects other skill listings.

## Best Possible Solution

Maintainers should choose the ClawHub licensing model and implement it consistently across publish consent, storage, API schemas, listing UI, CLI output, docs, migrations or backfills, and tests.

## Reproduction Assessment

Yes. The live ClawHub API returns MIT-0 for the reported skill while npm and upstream sources declare MIT, and current main hard-codes MIT-0 across schema, storage, publish, docs, API, CLI, and display surfaces.

## Solution Assessment

No, not as a display-only fix. The maintainable solution needs a policy decision between enforcing the fixed MIT-0 contract and adding coherent per-skill SPDX support, not a one-line label change.
