## Summary

Keep open. Current main still encodes MIT-0 as a platform-wide skill license across shared schema, Convex storage, publish flow, listing UI, CLI/docs, and tests, so the reported MIT display mismatch remains unresolved and needs maintainer/legal product direction rather than conservative cleanup.

## What This Changes

The issue requests that the reported skill listing display standard MIT, with attribution required, instead of the current MIT-0/no-attribution license.

## Best Possible Solution

Maintainers should choose one license model and implement it consistently across upload consent, storage/schema, public APIs, listing UI, CLI output, docs, migrations or backfills, and tests.

## Reproduction Assessment

Yes. Current main has a high-confidence source-level path: publish writes MIT-0, storage/schema only accept MIT-0, and the skill header renders the MIT-0 platform constant.

## Solution Assessment

No for a display-only label change. The maintainable solution is a coherent license-policy decision or per-skill SPDX support because the current platform contract encodes MIT-0 across multiple public surfaces.
