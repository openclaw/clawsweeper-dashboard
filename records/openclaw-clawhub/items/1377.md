## Summary

Current main and the live listing still use ClawHub’s documented MIT-0 platform license, so this is not already implemented. The report exposes a real conflict between upstream license metadata and ClawHub’s publication contract that needs an explicit product/legal decision before code changes.

## What This Changes

Make ClawHub represent standard MIT licensing for a skill whose upstream package and license file declare MIT rather than MIT-0.

## Best Possible Solution

Adopt one coherent end-to-end policy: either clearly reject conflicting upstream license terms while enforcing MIT-0, or support per-version SPDX licenses across ingestion, storage, APIs, CLI, UI, documentation, and existing-data migration.

## Reproduction Assessment

Yes for the observable mismatch: current source and the live API consistently produce MIT-0 for the reported skill. However, current documentation defines that result as platform policy rather than an accidental classifier failure.

## Solution Assessment

No, changing only the displayed text is not the best solution. Maintainers should first choose the licensing contract, then update every publication and consumption surface consistently.
