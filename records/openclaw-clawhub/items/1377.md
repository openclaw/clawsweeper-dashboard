## Summary

Keep open. Current main still intentionally models ClawHub skill publishing as a fixed MIT-0/no-attribution contract, so this remains a real policy and product mismatch rather than a cleanup close.

## What This Changes

The issue asks ClawHub to show the upstream standard MIT license for `openclaw-workspace-sync` instead of MIT-0/no-attribution text.

## Best Possible Solution

Maintainers should choose the license model, then update publish consent, storage/schema, public APIs, listing UI, CLI output, docs, migrations/backfills, and tests consistently.

## Reproduction Assessment

Yes. Source inspection on current main is enough: publish/storage force `MIT-0`, and the detail header renders the platform constant while the report identifies the upstream package as MIT.

## Solution Assessment

No for a display-only label change. The maintainable path is a coherent license-policy change or per-skill SPDX support because the current contract encodes MIT-0 across several public surfaces.
