The issue reports that `openclaw-workspace-sync` is shown on ClawHub as MIT-0/no-attribution even though the upstream package declares standard MIT.

## Best Possible Solution

Maintainers should choose one licensing contract, then update publish consent, storage, API schemas, listing UI, CLI output, docs, migrations/backfills, and tests so ClawHub listings cannot disagree with the actual license grant.

## Reproduction Assessment

Yes. Current source writes, stores, accepts, and displays MIT-0, and the live API returns MIT-0 for the reported skill while npm and GitHub package/license metadata report MIT.

## Solution Assessment

No. A one-line listing-label change is not the best fix because the current product contract is encoded across schema, publish consent, storage, API, UI, CLI, docs, and tests; maintainers first need to choose the license model.
