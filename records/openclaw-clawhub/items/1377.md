## Summary

Keep open: current main still implements a fixed MIT-0 platform-license model, while the requested MIT display would change the ClawHub licensing contract and needs maintainer/legal product direction. Repository cleanup policy also keeps ClawHub issues open rather than auto-closing them.

## What This Changes

The issue reports that a skill listing displays MIT-0 even though the package metadata, LICENSE file, npm page, and GitHub repository identify standard MIT.

## Best Possible Solution

Maintainers should choose one license model and implement it consistently across upload consent, Convex storage, public schemas, API responses, UI, CLI output, docs, tests, and any migration/backfill needed for existing published versions.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: publishing writes MIT-0, storage validates only MIT-0, docs and upload consent advertise MIT-0, and the listing UI renders the shared MIT-0 constant.

## Solution Assessment

No for a display-only switch to MIT. The maintainable fix is a maintainer/legal policy decision followed by either reaffirmed MIT-0 platform licensing or coherent per-skill SPDX support across all affected surfaces.
