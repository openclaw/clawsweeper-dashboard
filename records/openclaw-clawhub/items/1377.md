## Summary

Keep open: the MIT versus MIT-0 mismatch still reproduces, but current main intentionally models ClawHub skill publishing as a platform-wide MIT-0/no-attribution contract, so changing it needs maintainer/legal product direction rather than cleanup auto-close.

## What This Changes

The issue asks ClawHub to display the `openclaw-workspace-sync` skill license as standard MIT instead of MIT-0, matching the upstream package metadata and LICENSE text.

## Best Possible Solution

Maintainers should choose one license model and then implement it consistently across publish consent, storage, API schemas, listing UI, CLI output, docs, tests, and any needed migration/backfill.

## Reproduction Assessment

Yes. The live API still returns MIT-0 for `openclaw-workspace-sync` while npm/GitHub metadata declare MIT, and current-main source has a direct path that writes, validates, documents, returns, and renders the MIT-0 literal.

## Solution Assessment

No for a display-only switch to MIT. The maintainable fix is a maintainer/legal policy decision followed by either reaffirmed MIT-0 publishing terms or coherent per-skill SPDX license support across all affected surfaces.
