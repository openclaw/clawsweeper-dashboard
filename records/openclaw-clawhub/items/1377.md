## Summary

Keep open. The mismatch is still reproducible, but current main intentionally models ClawHub skills as a platform-wide MIT-0/no-attribution contract; changing that safely needs maintainer/legal product direction, and this repository policy does not auto-close ClawHub issues.

## What This Changes

The issue requests that ClawHub display the reported skill's standard MIT license instead of the platform MIT-0 license shown on the listing/API.

## Best Possible Solution

Maintainers should choose one license model and implement it consistently across upload acceptance, Convex storage, public API schemas, listing UI, CLI output, docs, migrations or backfills, and tests.

## Reproduction Assessment

Yes. The live API returns MIT-0 for the reported skill while npm and upstream package metadata declare MIT, and current main hard-codes MIT-0 across schema, storage, publish, docs, CLI/display-adjacent surfaces.

## Solution Assessment

No. A display-only MIT label is not the best fix because it would create a second source of truth; maintainers need to decide between enforcing the fixed MIT-0 contract and adding full per-skill SPDX support.
