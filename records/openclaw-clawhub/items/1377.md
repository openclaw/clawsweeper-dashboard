## Summary

Keep open: current main and production still expose `MIT-0` as the skill license, and the remaining question is a maintainer product/legal decision about platform distribution license versus upstream source license metadata.

## What This Changes

The issue asks ClawHub to display the skill’s upstream MIT license instead of the platform-wide MIT-0 license shown on the listing.

## Best Possible Solution

Keep the MIT-0 platform distribution contract explicit, relabel that scope so it is not mistaken for the upstream source license, and add a separate author-declared upstream license only after maintainers define provenance and conflict handling.

## Reproduction Assessment

Yes for the observable output: production API and current source both show `MIT-0` for the reported skill listing path. No as a strict bug against current docs, because docs and publish flow intentionally describe all ClawHub-published skills as MIT-0.

## Solution Assessment

No, directly replacing MIT-0 with MIT is not the best current fix because it would collapse platform distribution terms into upstream package metadata. A maintainable solution should separate the labels or add a distinct upstream-license field after policy approval.
