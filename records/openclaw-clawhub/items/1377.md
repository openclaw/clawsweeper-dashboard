## Summary

Keep open. Current main, the latest release tag, and the live API still expose the reported skill under the fixed MIT-0 platform license, so changing this to MIT remains a maintainer/legal product decision rather than an implemented cleanup item.

## What This Changes

The issue asks ClawHub to display standard MIT licensing for `openclaw-workspace-sync` instead of MIT-0, and to check whether other MIT-licensed skills are being treated the same way.

## Best Possible Solution

Maintainers should choose one public skill-license model and apply it consistently across upload consent, Convex storage, API schemas, UI, CLI output, docs, tests, and any migration or backfill path.

## Reproduction Assessment

Yes. Source inspection shows current main hard-codes MIT-0 in storage, API/schema, UI, CLI, and docs, and the live API for the reported slug still returns `MIT-0`; I did not need to run a local browser to establish the source-level path.

## Solution Assessment

No clear code-only fix yet. Displaying MIT alone is not the best solution unless maintainers first decide whether ClawHub should preserve upstream package licenses or continue enforcing a platform-wide MIT-0 contract.
