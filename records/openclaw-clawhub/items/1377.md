## Summary

Keep open. Current main still encodes a platform-wide MIT-0 skill license across publish, storage, API, UI, CLI, and docs, so the reported MIT-versus-MIT-0 mismatch is unresolved and needs maintainer/legal product direction.

## What This Changes

The issue requests that the `openclaw-workspace-sync` listing show standard MIT with attribution-required wording instead of MIT-0/no-attribution wording.

## Best Possible Solution

Maintainers should choose one license model and implement it consistently across upload acceptance, Convex storage, public APIs, listing UI, CLI output, docs, migrations/backfills, and tests.

## Reproduction Assessment

Yes. Source inspection on current main shows the publish path stores MIT-0, the schema and Convex validators only model MIT-0, and the listing header renders that platform constant.

## Solution Assessment

No for a label-only display change. The maintainable solution is a policy decision followed by an end-to-end implementation, because the current product contract encodes MIT-0 on multiple public surfaces.
