## Summary

Keep open. Current main, the latest release tag, and the live ClawHub API still use a fixed MIT-0 platform license, while the reported package sources identify MIT, so the central mismatch remains unresolved and needs maintainer/legal product direction rather than cleanup closure.

## What This Changes

The issue asks ClawHub to display the reported skill's license as MIT instead of the current MIT-0/no-attribution license text.

## Best Possible Solution

Maintainers should choose one license model and implement it consistently across upload acceptance, Convex storage, public API schemas, listing UI, CLI output, docs, migrations or backfills, and tests.

## Reproduction Assessment

Yes. Source and live API checks show ClawHub still stores, validates, documents, and displays a fixed MIT-0 platform license, while npm and the upstream GitHub repository for the reported skill report MIT; I did not run a local browser because the API and source paths establish the mismatch.

## Solution Assessment

No automatic display-only fix is clearly best. Showing MIT alone would conflict with the current platform license contract unless maintainers decide to support per-skill SPDX licensing and migrate the related surfaces together.
