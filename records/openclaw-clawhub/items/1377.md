## Summary

Keep open. Current main, the latest release tag, and the live ClawHub API still expose published skills under the platform-wide MIT-0/no-attribution license while the reported upstream package sources say MIT, so the central mismatch remains unresolved and needs maintainer/legal product direction.

## What This Changes

The issue asks ClawHub to display the reported skill's actual MIT license instead of the platform-wide MIT-0/no-attribution license.

## Best Possible Solution

Maintainers should choose one license model and implement it consistently across upload consent, Convex storage, public API schemas, listing UI, CLI output, docs, migrations or backfills, and tests.

## Reproduction Assessment

Yes. Source inspection and a live API check show ClawHub still stores, validates, documents, and displays MIT-0 while the reported upstream package JSON and LICENSE identify MIT.

## Solution Assessment

No automatic display-only fix is clearly best. Showing MIT alone would conflict with the current platform license contract unless maintainers decide to support per-skill SPDX licensing and migrate the related surfaces together.
