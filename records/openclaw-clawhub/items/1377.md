## Summary

Current main still deliberately enforces and displays MIT-0 for every ordinary published skill, so the reported mismatch remains unresolved; however, supporting upstream MIT declarations requires a maintainer product/legal decision rather than a display-only bug fix.

## What This Changes

Change ClawHub’s license handling so a skill whose upstream package uses standard MIT is not presented as MIT-0.

## Best Possible Solution

Choose whether ClawHub remains MIT-0-only or supports publisher-selected SPDX licenses, then implement that contract coherently across publisher acceptance, persisted version metadata, migrations, API and CLI schemas, documentation, and listing UI.

## Reproduction Assessment

Yes at source level: current main always stores and renders MIT-0 for ordinary published skills, regardless of an upstream package’s MIT declaration. The behavior is established by the shared constant, publishing path, listing component, and documentation.

## Solution Assessment

No, not as a display-only correction. The narrow maintainable solution requires deciding the actual publishing contract and then updating every license-bearing surface and existing-data behavior consistently.
