## Summary

Keep open: the MIT versus MIT-0 mismatch still exists, but current main intentionally treats ClawHub skill publishing as a platform-wide MIT-0/no-attribution contract, and ClawHub cleanup policy keeps issues open rather than auto-closing them.

## What This Changes

The issue requests that a ClawHub skill listing display the package’s standard MIT license instead of the platform MIT-0/no-attribution license.

## Best Possible Solution

Maintainers should choose one license model and implement it consistently across upload consent, storage, API schemas, listing UI, CLI output, docs, tests, and any migration/backfill path.

## Reproduction Assessment

Yes. The live API returns MIT-0 for the reported skill while npm/GitHub declare MIT, and current-main source has a direct path that writes, validates, documents, returns, and renders the MIT-0 literal.

## Solution Assessment

No for a display-only switch to MIT. The maintainable fix is a maintainer/legal policy decision followed by either reaffirmed MIT-0 publishing terms or coherent per-skill SPDX license support across all affected surfaces.
