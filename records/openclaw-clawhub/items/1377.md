## Summary

Keep open. The MIT versus MIT-0 mismatch still reproduces, but current main intentionally models published skills as a platform-wide MIT-0/no-attribution contract, so changing it safely needs maintainer/legal product direction rather than conservative cleanup.

## What This Changes

The issue asks ClawHub to display the reported skill's actual MIT license instead of the platform MIT-0/no-attribution license.

## Best Possible Solution

Maintainers should choose the intended license model and implement it consistently so upload consent, Convex storage, public API/CLI output, listing UI, docs, migrations, and tests cannot make contradictory license claims.

## Reproduction Assessment

Yes. A live API request for openclaw-workspace-sync returns MIT-0 while npm and the upstream GitHub package.json declare MIT, and current main has source paths that write, validate, document, and render the MIT-0 literal.

## Solution Assessment

No for a display-only switch to MIT. The maintainable solution is a maintainer/legal policy decision followed by either a coherent per-skill SPDX implementation or reaffirmed enforcement of the platform MIT-0 contract.
