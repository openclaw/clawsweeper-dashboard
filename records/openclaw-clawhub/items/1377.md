## Summary

Keep open. The reported MIT versus MIT-0 mismatch is still source-reproducible and visible in the live API, but current main intentionally models published skills as a platform-wide MIT-0/no-attribution contract, so a safe fix needs maintainer/legal product direction rather than an isolated display-label change.

## What This Changes

The issue asks ClawHub to display the reported skill as standard MIT instead of MIT-0 because the upstream package and license file declare MIT.

## Best Possible Solution

Maintainers should choose one licensing model and implement it consistently across upload acceptance, stored version metadata, public API schemas, listing UI, CLI output, docs, migrations, and tests.

## Reproduction Assessment

Yes. Current main hard-codes MIT-0 through schema, storage, publish, UI, CLI, and docs, and the live ClawHub API returns MIT-0 for the reported skill while upstream package metadata and LICENSE text say MIT.

## Solution Assessment

No. A display-only label change is not the best fix because it would create a second license source of truth; the maintainable solution needs a licensing policy decision first.
