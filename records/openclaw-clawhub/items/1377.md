## Summary

Keep open. Current main and the live API still expose MIT-0 for the reported skill while upstream package sources declare MIT, but current code/docs intentionally model published skills as a platform-wide MIT-0 contract, so resolving this needs maintainer/legal product direction rather than conservative cleanup.

## What This Changes

The issue asks ClawHub to show the reported skill’s upstream MIT license instead of the platform-displayed MIT-0/no-attribution license.

## Best Possible Solution

Maintainers should choose one license model and implement it consistently: either reaffirm/enforce the MIT-0 publishing contract despite upstream package metadata, or add full per-skill SPDX support across upload consent, storage, APIs, UI, CLI, docs, migrations, and tests.

## Reproduction Assessment

Yes. The live ClawHub API returns MIT-0 for the reported skill while upstream npm/GitHub metadata declare MIT, and current main has a clear source path that validates, writes, documents, and renders only the MIT-0 platform license.

## Solution Assessment

No for a display-only switch to MIT. The maintainable solution is a maintainer/legal policy decision followed by a full-stack license model change if per-skill SPDX licenses are desired, or a clarified MIT-0 publishing contract if that remains policy.
