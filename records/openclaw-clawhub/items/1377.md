## Summary

Keep open. The reported mismatch is still source-reproducible and visible in production: the ClawHub API reports MIT-0 for the skill while npm/upstream metadata declares MIT, but current main intentionally models published skills as a platform-wide MIT-0/no-attribution contract, so changing this safely needs maintainer/legal product direction rather than cleanup closure.

## What This Changes

The issue requests that ClawHub display the reported skill’s actual MIT license instead of the platform MIT-0 license.

## Best Possible Solution

Maintainers should choose one license model and make upload acceptance, stored metadata, public schemas, API responses, listing UI, CLI output, docs, migrations/backfills, and tests consistent with that decision.

## Reproduction Assessment

Yes. The live API returns MIT-0 for the reported skill while npm and upstream package metadata declare MIT, and current main hard-codes MIT-0 through schema, storage, publish, docs, CLI, and listing surfaces.

## Solution Assessment

No. A display-only MIT label is not the best fix because it would create a second source of truth; the maintainable solution is a license-model decision followed by a full-stack implementation.
