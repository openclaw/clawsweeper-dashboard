## Summary

Keep open. Current main, the latest release, and production still expose MIT-0 as the platform-wide skill license, while the reported upstream package metadata says MIT; the remaining work is a maintainer/legal product decision about platform-license versus upstream-source-license semantics, not an implemented cleanup.

## What This Changes

The issue requests that the reported skill listing display standard MIT license metadata instead of MIT-0/no-attribution metadata.

## Best Possible Solution

Maintainers should define whether ClawHub listings show the platform distribution license, upstream package license, or both, then implement that model consistently across publishing, storage, public API, UI, CLI, docs, and any needed migration/backfill.

## Reproduction Assessment

Yes for the observable mismatch: the production API returns MIT-0 for the reported skill while the upstream npm package reports MIT, and current source renders/stores the platform MIT-0 value. This is source-reproducible rather than a strict broken-contract bug because current docs and schemas intentionally define MIT-0 as the platform skill license.

## Solution Assessment

No for a direct MIT replacement; that would conflate upstream package license metadata with the platform distribution license. The maintainable solution is to make the platform-license label explicit and add a separately named upstream/source-license model only if maintainers define provenance and conflict handling.
