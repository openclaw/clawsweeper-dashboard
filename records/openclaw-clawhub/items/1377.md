## Summary

Keep open. Current main, the latest v0.23.0 release tag, and the live API still expose published skill licenses under ClawHub's fixed MIT-0 platform policy, so showing this skill as MIT remains a maintainer/legal product decision rather than an implemented cleanup item.

## What This Changes

The issue asks ClawHub to display the published skill's actual MIT license instead of the platform/default MIT-0 license text.

## Best Possible Solution

Maintainers should either reaffirm the fixed MIT-0 contract with publisher remediation guidance, or deliberately reopen per-skill license support as a product/legal project spanning publish consent, storage, API, UI, CLI, docs, and migrations.

## Reproduction Assessment

Yes from source inspection and the live API: current main stores, validates, documents, and displays published skill licenses as MIT-0, and the live API for the reported skill returns MIT-0. I did not run a local browser because the backend/API/docs path establishes the current behavior.

## Solution Assessment

No automatic code fix is clearly best yet. Displaying MIT for one listing would contradict the current platform license contract unless maintainers first decide to support package-specific licenses or change publisher consent semantics.
