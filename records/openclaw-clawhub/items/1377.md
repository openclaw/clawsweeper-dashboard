## Summary

Keep open. The reported mismatch is still observable, but current main intentionally models ClawHub-published skills as a platform-wide MIT-0/no-attribution contract, so fixing this safely needs maintainer/legal product direction rather than a one-line display change.

## What This Changes

The issue asks ClawHub to display the `openclaw-workspace-sync` skill as MIT with attribution required instead of MIT-0/no-attribution.

## Best Possible Solution

Choose one licensing model and make it consistent across upload acceptance, stored version metadata, public API schemas, listing UI, CLI output, docs, migrations, and tests.

Yes. Source inspection and live hosted checks show ClawHub still returns/renders MIT-0 for this skill while upstream package metadata says MIT; I did not run a local app server because the source path is direct.

## Solution Assessment

No. Changing only the displayed label is not the best fix because it would create a second source of truth against the current MIT-0 publishing contract; maintainers need to decide the license model first.
