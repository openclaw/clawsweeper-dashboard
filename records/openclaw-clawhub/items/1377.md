## Summary

Keep open. The mismatch is still reproducible, but current main intentionally treats published skills as a platform-wide MIT-0/no-attribution contract, so a safe fix needs maintainer/legal product direction rather than an isolated label change; ClawHub cleanup policy also keeps issues open.

## What This Changes

The issue asks ClawHub to display standard MIT for `openclaw-workspace-sync` instead of MIT-0 when upstream package metadata and license text say MIT.

## Best Possible Solution

Maintainers should choose one licensing model and implement it consistently across upload acceptance, stored version metadata, public API schemas, listing UI, CLI output, docs, migrations, and tests.

## Reproduction Assessment

Yes. Current main hard-codes MIT-0 through schema, storage, publishing, UI, CLI, and docs, and the live ClawHub API still returns MIT-0 for the reported skill while upstream metadata says MIT.

## Solution Assessment

No. Changing only the visible listing label is not the best fix because it would create a second license source of truth; the maintainable solution needs a policy decision before code changes.
