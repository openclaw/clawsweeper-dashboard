## Summary

Keep open. The MIT versus MIT-0 mismatch still reproduces on the live API, but current main intentionally models skill publishing as a platform-wide MIT-0/no-attribution contract; resolving this needs maintainer/legal product direction, and ClawHub cleanup policy keeps issues open rather than auto-closing them.

## What This Changes

The issue asks ClawHub to display the reported skill's upstream MIT license instead of the platform MIT-0 license/no-attribution copy.

## Best Possible Solution

Maintainers should choose one license model and implement it consistently across upload consent, Convex storage, public API schemas, listing UI, CLI output, docs, migrations or backfills, and tests.

## Reproduction Assessment

Yes. The live API returns MIT-0 for `openclaw-workspace-sync` while upstream npm/GitHub metadata declare MIT, and current main has a direct source path that writes, validates, documents, and renders the MIT-0 literal.

## Solution Assessment

No for a display-only switch to MIT. The maintainable path is a maintainer/legal policy decision followed by either reaffirmed MIT-0 publishing terms or a full-stack per-skill SPDX license model.
