## Summary

Current main and production intentionally use MIT-0 as ClawHub’s platform distribution license, so the report is not an accidental MIT classifier failure; however, the generic `License` presentation can reasonably be read as the author’s upstream license, and no current-main change or canonical item resolves that ambiguity. Keep this open for a maintainer decision on whether to relabel the field or add separate source-license metadata.

## What This Changes

Clarify ClawHub’s license presentation when a skill’s upstream package declares MIT but the registry records the separately accepted MIT-0 platform distribution license.

## Best Possible Solution

Keep MIT-0 as the explicit ClawHub distribution license, relabel it so that scope is unmistakable, and add a separately named author-declared source license only if maintainers define its provenance and conflict behavior.

## Reproduction Assessment

Not applicable as a strict bug reproduction: the reported production output is confirmed, but it matches the repository’s intentional MIT-0 platform-license contract rather than demonstrating a classifier malfunction.

## Solution Assessment

No, directly replacing MIT-0 with MIT is not the best solution because it would discard the platform distribution-license meaning; explicit labeling and, if approved, a separate upstream-license field is safer.
