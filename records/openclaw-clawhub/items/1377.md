## Summary

Keep open. Current main and the live API still expose the reported skill as MIT-0 while upstream package metadata says MIT, and changing this safely requires a maintainer/license-policy decision because ClawHub currently treats published skills as a fixed MIT-0/no-attribution contract.

## What This Changes

The issue asks ClawHub to display standard MIT for a skill whose upstream package metadata declares MIT instead of the platform MIT-0 license.

## Best Possible Solution

Maintainers should choose one license contract, either fixed MIT-0 with conflict enforcement or first-class per-skill SPDX metadata, then update upload acceptance, storage, APIs, UI, CLI, docs, migrations, and tests consistently.

## Reproduction Assessment

Yes. Source inspection and the live API both show MIT-0 for the reported skill while upstream npm and GitHub metadata declare MIT.

## Solution Assessment

No for a display-only MIT label. The maintainable solution is a license-model decision followed by a full-stack implementation so UI, API, CLI, docs, storage, and publish consent cannot disagree.
