## Summary

Keep open. The reported MIT versus MIT-0 mismatch still reproduces, but current main intentionally models skill licensing as a platform-wide MIT-0 contract, so changing it safely requires maintainer/legal product direction rather than conservative cleanup.

## What This Changes

The issue asks ClawHub to display the reported skill's standard MIT license instead of the platform MIT-0/no-attribution label.

## Best Possible Solution

Maintainers should choose one license model and implement it consistently across upload consent, Convex storage, public API schemas, listing UI, CLI output, docs, migrations or backfills, and tests.

## Reproduction Assessment

Yes. The live API returns MIT-0 for the reported skill while upstream npm/GitHub sources declare MIT, and current main has a clear source path that writes, validates, documents, and renders the MIT-0 literal.

## Solution Assessment

No for a display-only switch to MIT. The maintainable solution is a maintainer/legal policy decision followed by a full-stack license model change if per-skill SPDX licenses are desired, or a reaffirmed MIT-0 publishing contract if that remains the product policy.
