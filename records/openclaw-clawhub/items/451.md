## Summary

Keep open. Repository policy keeps ClawHub issues open, and current main still does not expose verified external GitHub profile links beside public usernames/profile names; repository links are only partially present on metadata-driven skill/package surfaces.

## What This Changes

Request GitHub profile links next to usernames and on public user pages, plus clearer repository links for projects.

## Best Possible Solution

Public owner/profile views should offer internal ClawHub navigation plus a separate external GitHub link when verified GitHub identity data is available, while repository/source links remain tied to trusted skill or package metadata.

Not applicable as a bug reproduction; this is a UI/data enhancement request. Source inspection of current main shows the requested profile/name surfaces render text or internal ClawHub links and public data does not carry a GitHub profile URL.

## Solution Assessment

Yes. The maintainable direction is to expose verified GitHub login/profile URL data and render a separate external affordance without guessing from mutable handles or nesting anchors.
