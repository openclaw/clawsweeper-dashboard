## Summary

Keep open: the PR contains useful UI and route fixes, but the new official-publisher payload can also promote publisher official status into the skill-level title badge, which overstates the trust signal. The contributor screenshots cover the plugin listing and CLI/Prompt toggle, but not the official creator badge or hash-preserving redirect behavior.

## What This Changes

The PR removes plugin star counts from listings, flattens the skill install CLI/Prompt toggle, exposes official publisher status on skill detail owners, adjusts official badge placement, preserves plugin redirect hash fragments, and adds tests/proof images.

## Best Possible Solution

Keep publisher official status as a creator/UserBadge signal only, keep the title OfficialTag tied to skill.badges.official, add regression coverage for that separation, and post redacted real browser or terminal proof for the official badge and hash redirect paths.
