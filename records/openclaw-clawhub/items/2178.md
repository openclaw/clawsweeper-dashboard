## Repair Work Prompt

Fix the New Version publish flow crash where `/skills/publish?updateSlug=...` can call `api.skills.checkSlugAvailability` with a missing or wrong owner. On current main, `SkillOwnershipPanel` links to `/publish-skill?updateSlug=...` without `ownerHandle`, `/publish-skill` strips `ownerHandle`, and `src/routes/skills/publish.tsx` can default to the personal publisher before the existing skill owner is loaded. Preserve `ownerHandle` when rendering New Version links and legacy redirects, and make update-mode slug availability wait for or use the existing skill owner before falling back to a personal owner. Do not loosen backend publish authorization, do not redesign owner migration, and keep changes scoped to the web publish flow plus regression tests. Reference both https://github.com/openclaw/clawhub/issues/2177 and this issue in the PR body.
