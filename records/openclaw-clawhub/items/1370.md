## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still lacks an end-to-end owner-facing plugin package lifecycle workflow: package-wide delete plumbing exists, but package hide/undelete and version withdrawal are missing from the CLI, owner UI, public docs, and restore API.

## What This Changes

Request owner/admin hide, delete, undelete, and version-withdrawal controls for ClawHub plugin packages across the site, CLI, API, and docs.

## Best Possible Solution

Define the package and package-release lifecycle model, then implement owner/admin controls end to end across Convex mutations, HTTP routes, clawhub CLI commands, dashboard or plugin management UI, tests, and public docs while coordinating version-specific semantics with #905 and stale plugin cleanup needs from #1744.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main has only partial package-wide delete plumbing and lacks the requested owner-facing package undelete, hide, and version-withdrawal workflow.

## Solution Assessment

Unclear until maintainers decide lifecycle semantics. The maintainable direction is likely to reuse existing ownership, soft-delete, audit, and moderation models, but package-wide restore and release-level delisting need product decisions first.
