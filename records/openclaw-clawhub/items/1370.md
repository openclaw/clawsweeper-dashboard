## Summary

Keep open. Current main now has package-wide owner delete in the dashboard, HTTP API, CLI, docs, and tests, but the requested lifecycle is still incomplete: there is no package undelete/restore, hide/unhide alias, or version-level package de-list workflow, and ClawHub issue policy keeps issues open for maintainer handling.

## What This Changes

Requested owner-facing hide/delete/undelete controls for ClawHub plugin packages, including CLI, site UI, docs, and possible version-level withdrawal.

## Best Possible Solution

Define the package lifecycle policy, then complete owner/admin package restore and any version-level withdrawal workflow across Convex mutations, HTTP routes, CLI, dashboard UI, tests, and public docs.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence in the remaining gap because current main exposes package-wide delete but no package undelete/restore, hide/unhide, or version-level package withdrawal command/API.

## Solution Assessment

Unclear until maintainers decide lifecycle semantics. The shipped package-wide delete is a useful partial solution, but the maintainable end state needs explicit restore, slug reservation, version de-listing, ownership, and audit behavior.
