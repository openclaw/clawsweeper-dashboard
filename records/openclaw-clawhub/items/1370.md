## Summary

Keep open. Current main now covers package-wide owner delete across the API, CLI, dashboard, docs, and tests, but the broader requested lifecycle is still incomplete: package restore/undelete, hide/unhide, version-level delisting, and recovery semantics remain unresolved and security-sensitive.

## What This Changes

Add owner-facing hide, delete, undelete, and version-delisting controls for ClawHub plugin packages.

## Best Possible Solution

Define a package and package-release lifecycle policy, then implement consistent owner/admin controls across Convex mutations, HTTP routes, the `clawhub` CLI, dashboard UI, tests, and public docs.
