## Summary

Keep open. ClawHub issue cleanup policy keeps issues open, and current main still lacks an end-to-end owner-facing plugin package hide/delete/undelete or version-withdrawal workflow despite partial package-wide soft-delete plumbing.

## What This Changes

Add owner/admin lifecycle controls so plugin package publishers can hide, delete, undelete, or delist broken ClawHub package releases.

## Best Possible Solution

Define the package and package-release lifecycle model, then implement owner/admin controls end to end across Convex mutations, HTTP routes, CLI, dashboard/plugin UI, tests, and public docs while coordinating #905 and #1744.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main has only partial package-wide soft-delete support and no complete owner-facing package lifecycle workflow.

## Solution Assessment

Unclear until maintainers decide lifecycle semantics. The maintainable direction is to reuse ownership, soft-delete, audit, moderation, and package release models, but package restore and version delisting need product decisions first.
