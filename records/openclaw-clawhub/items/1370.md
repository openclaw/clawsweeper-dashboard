## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main only partially addresses the request with package-wide soft delete; package restore, hide/unhide, release-level withdrawal, and recovery semantics remain unresolved. Related open reports at https://github.com/openclaw/clawhub/issues/905, https://github.com/openclaw/clawhub/issues/1744, and https://github.com/openclaw/clawhub/issues/2080 point to the same lifecycle gap from adjacent angles.

## What This Changes

Request owner-facing website, CLI, API, and docs support to hide, delete, undelete, or delist ClawHub plugin packages and package versions.

## Best Possible Solution

Define the package and package-release lifecycle model, then implement owner/admin restore, hide/unhide, version withdrawal or delisting, UI and CLI workflows, tests, and public docs together.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection on current main gives high confidence that package-wide delete exists, while restore, hide/unhide, and owner-controlled version withdrawal remain absent.

## Solution Assessment

Unclear until maintainers settle lifecycle semantics. The narrow maintainable path is an end-to-end package lifecycle design rather than adding another isolated command around the current package-wide delete.
