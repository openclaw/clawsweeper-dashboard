## Summary

Keep open. This is a feature request, and current main still lacks an end-to-end owner-facing package lifecycle workflow even though package/release soft-delete storage and a package-wide DELETE API exist.

## What This Changes

Add owner/admin hide, delete, undelete, or version-delisting workflows for plugin packages across ClawHub site and CLI surfaces, analogous to skills.

## Best Possible Solution

Define package-wide and release-specific lifecycle semantics, then implement owner/admin controls end to end across Convex mutations, HTTP routes, CLI commands, dashboard or plugin UI, tests, and public docs while coordinating with #905 and #1744.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature gap. The gap is verifiable from current-main source by checking schema, package API, CLI, dashboard/plugin UI, docs, and tests for missing package restore and version-withdrawal workflows.

## Solution Assessment

Unclear until maintainers choose lifecycle semantics. The maintainable direction is to reuse existing ownership, soft-delete, audit, and moderation models, but package-wide restore and release-level delisting need product decisions first.
