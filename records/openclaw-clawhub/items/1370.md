Keep open. This is a valid feature request, and ClawHub cleanup policy only auto-closes PRs; current main provides package-wide delete, but restore, hide/unhide, version-level withdrawal, and recovery semantics remain unresolved.

Add owner-facing hide, delete, undelete, and version-withdrawal controls for ClawHub plugin packages across the site, CLI, API, and docs.

## Best Possible Solution

Define the package and package-release lifecycle model, then implement owner/admin restore, hide/unhide, version withdrawal or delisting, dashboard and CLI workflows, regression tests, audit behavior, and public docs together.

## Solution Assessment
