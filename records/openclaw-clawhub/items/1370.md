## Summary

Keep open. This is a feature request and ClawHub cleanup policy keeps issues open; current main still lacks an end-to-end owner-facing package hide/delete/undelete or version-withdrawal workflow despite partial package soft-delete and moderation plumbing.

## What This Changes

Request owner-facing hide/delete/undelete and version-withdrawal controls for ClawHub plugin packages across the site, CLI, API, and docs.

## Best Possible Solution

Define the package and package-release lifecycle model, then implement owner/admin controls end to end across Convex mutations, HTTP routes, CLI commands, dashboard or plugin UI, tests, and public docs while coordinating version-specific behavior with #905 and cleanup needs from #1744.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature gap. Current-main source inspection gives a high-confidence verification path by checking the package API, CLI, owner dashboard, staff management UI, docs, schema, and tests for missing package restore/version withdrawal workflows.

## Solution Assessment

Unclear until maintainers decide lifecycle semantics. The maintainable direction is likely to reuse the existing ownership, soft-delete, audit, and moderation models, but package-wide restore and release-level delisting need product decisions before a narrow implementation is safe.
