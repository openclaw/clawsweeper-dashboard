## Summary

Keep open. This is a feature request, and current main still only provides package-wide soft delete for plugin packages; restore, hide/unhide, version withdrawal, and end-to-end owner docs/workflows remain unresolved.

## What This Changes

Add owner-facing lifecycle controls for ClawHub plugin packages, including package hide/delete/undelete and ideally version-level delisting across site, CLI, API, and docs.

## Best Possible Solution

Define the package and package-release lifecycle model, then implement owner/admin workflows consistently across Convex mutations, HTTP routes, CLI commands, dashboard UI, tests, and public docs.
