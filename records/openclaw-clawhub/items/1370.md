## Summary

Keep open. The report is still valid on current main: package soft-delete storage and a package-wide DELETE API exist, but ClawHub does not yet provide an end-to-end owner-facing hide/delete/undelete or version-withdrawal workflow for plugin packages; the cleanup policy also keeps ClawHub issues open.

## What This Changes

Add owner/admin hide, delete, undelete, and release-withdrawal controls for ClawHub plugin packages across site, CLI, API, and docs.

## Best Possible Solution

Define the package and package-release lifecycle model, then implement owner/admin controls across Convex mutations, HTTP routes, clawhub CLI commands, dashboard/plugin UI, tests, and public docs while coordinating version-specific behavior with #905 and stale plugin cleanup from #1744.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature gap. Current-main inspection gives a high-confidence verification path by checking the package API, CLI, owner dashboard, staff management UI, docs, schema, and tests for missing package restore/version withdrawal workflows.

## Solution Assessment

Unclear until maintainers decide the lifecycle semantics. The maintainable direction is likely to reuse the existing ownership/audit model, but package-wide restore, release-level delisting, and installer behavior need product decisions before a narrow implementation is safe.
