## Summary

Keep open. This issue is valid and current main still lacks a complete owner-facing package lifecycle workflow: the backend has package and release soft-delete state plus a package-wide DELETE API, but package undelete/restore, hide/unhide, version-level controls, CLI commands, owner UI, and docs are still incomplete; ClawHub cleanup policy also keeps issues open rather than auto-closing them.

## What This Changes

The issue requests owner/admin hide, delete, undelete, and preferably version delist controls for ClawHub plugin packages across the website, CLI, API, and docs.

## Best Possible Solution

Define the package and package-release lifecycle model, then implement owner/admin controls end to end across Convex mutations, HTTP routes, clawhub CLI commands, dashboard or package settings UI, tests, and public docs, coordinating version-specific behavior with #905 and stale plugin cleanup needs from #1744.
