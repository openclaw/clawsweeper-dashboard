## Summary

Keep open. The ClawHub cleanup policy keeps issues open, and current main still only partially covers package lifecycle: package-wide soft delete exists in backend/API, but owner-facing package restore, hide/unhide, CLI commands, UI controls, version-level withdrawal, and docs are still missing.

## What This Changes

The issue requests owner-facing hide, delete, undelete, and optional version-delisting controls for ClawHub plugin packages across the site, CLI, API, and docs.

## Best Possible Solution

Define the package and package-release lifecycle model, then implement owner/admin controls across Convex mutations, HTTP routes, clawhub CLI, owner or management UI, tests, and public docs while coordinating version-specific behavior with #905 and stale plugin cleanup needs from #1744.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature gap. Current-main inspection gives a high-confidence verification path by checking the package API, CLI, schema, owner/staff UI, docs, and tests for missing package restore, hide/unhide, and release-withdrawal workflows.

## Solution Assessment

Unclear until maintainers decide lifecycle semantics. Reusing the existing owner/moderator authorization and audit model looks maintainable, but package-wide restore, release-level delisting, and installer behavior need product decisions before a narrow implementation is safe.
