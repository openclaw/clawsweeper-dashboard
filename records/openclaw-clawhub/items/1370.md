## Summary

Keep open. Current main still has only partial package lifecycle support, and this ClawHub cleanup policy keeps issues open even when they look closeable; owners still lack an end-to-end package hide/delete/undelete or version-withdrawal workflow.

## What This Changes

Add owner-facing hide, delete, undelete, and version-withdrawal controls for ClawHub plugin packages across the site, CLI, API, and docs.

## Best Possible Solution

Define the package and package-release lifecycle model, then implement owner/admin controls end to end across Convex mutations, HTTP routes, clawhub CLI commands, owner-facing UI, tests, and public docs while coordinating version-specific behavior with #905 and stale plugin cleanup from #1744.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature gap. Current-main source inspection gives high confidence by checking schema, package API routes, CLI commands, management UI, docs, and package lifecycle tests.

## Solution Assessment

Unclear until maintainers decide the lifecycle semantics. The maintainable direction is likely to reuse the existing owner/admin authorization and audit model, but package-wide restore, selected-release delisting, and installer behavior need product decisions first.
