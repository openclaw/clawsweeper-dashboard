## Summary

Keep open. Current main still has only partial package lifecycle support: package and release soft-delete fields plus a package-wide DELETE API exist, but owner-facing package hide/delete/undelete controls are still missing from the CLI, owner UI, restore API, version-level workflow, and docs.

## What This Changes

Request owner/admin lifecycle controls for ClawHub plugin packages, including website, CLI, API, and docs support for package hide/delete/undelete or version withdrawal.

## Best Possible Solution

Define the package and package-release lifecycle policy, then implement owner/admin controls end to end across Convex mutations, HTTP routes, clawhub CLI commands, dashboard or plugin management UI, tests, and public docs while coordinating version-specific behavior with #905 and cleanup needs from #1744.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature gap. Source inspection on current main verifies the requested flows are absent while partial package soft-delete storage and package-wide delete plumbing exist.

## Solution Assessment

Unclear until maintainers decide lifecycle semantics. The maintainable direction is likely to reuse existing ownership, soft-delete, audit, and moderation models, but package-wide restore and release-level delisting need policy decisions first.
