## Risks / Open Questions

- Package-wide delete currently reserves the package name and gives owners no self-service restore or recovery path, as tracked by https://github.com/openclaw/clawhub/issues/2080.
- Package-level restore semantics are unresolved: restoring all releases, only latest, or selected releases would have different install and audit behavior.
- Version-scoped delisting overlaps with https://github.com/openclaw/clawhub/issues/905 and https://github.com/openclaw/clawhub/issues/1947 and needs consistent latest/tag/download/search behavior.
- Owner-facing lifecycle controls for executable plugins must preserve authorization and audit evidence while reliably removing withdrawn packages from ClawHub-first install resolution.
