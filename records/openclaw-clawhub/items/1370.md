Keep open. Current main partially addresses the request with package-wide delete, but package restore/undelete, hide/unhide, version-specific withdrawal, docs, and recovery semantics remain unresolved; the ClawHub cleanup policy also keeps issues open for maintainer handling.

## What This Changes

Add owner-facing hide/delete/undelete and version-withdrawal controls for ClawHub plugin packages across site UI, CLI, API, and docs.

## Best Possible Solution

Define the package lifecycle policy first, then implement audited owner/admin package restore, hide/unhide, version withdrawal or delisting, UI, CLI, tests, and public docs together.
