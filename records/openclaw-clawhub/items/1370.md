## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has only partial package lifecycle support: package-wide soft-delete exists, but owner-facing package hide/undelete and version withdrawal are not implemented end to end.

## What This Changes

Add owner/admin hide, delete, undelete, and version-withdrawal controls for plugin packages across ClawHub UI, CLI, API, and docs.

## Best Possible Solution

Define the package and package-release lifecycle model first, then implement owner/admin controls end to end across Convex mutations, HTTP routes, CLI, owner UI, tests, and docs.

## Solution Assessment

Unclear until maintainers decide lifecycle semantics. The narrow maintainable direction is to reuse the existing ownership, soft-delete, audit, and moderation patterns, but package-wide restore and release-level withdrawal need product decisions first.
