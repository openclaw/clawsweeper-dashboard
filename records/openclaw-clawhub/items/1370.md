## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still lacks an end-to-end owner-facing package hide/delete/undelete or version-withdrawal workflow despite having partial package-wide soft-delete plumbing.

## What This Changes

Add owner/admin lifecycle controls so ClawHub plugin package owners can hide, delete, undelete, or delist broken package releases without maintainer intervention.

## Best Possible Solution

Define the package and package-release lifecycle model, then implement owner/admin controls end to end across Convex mutations, HTTP routes, CLI, dashboard/plugin UI, tests, and public docs.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main has only a package-wide DELETE API and no owner-facing package undelete/hide/version delist flow.

## Solution Assessment

Unclear until maintainers decide lifecycle semantics. The maintainable direction is likely to reuse existing ownership, soft-delete, audit, and moderation models, but package-wide restore and release-level delisting need product decisions first.
