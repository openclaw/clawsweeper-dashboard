## Summary

Keep open. Current main has useful package-wide delete support, but the requested package restore, hide/unhide, and version-level delist workflows are still missing; ClawHub cleanup policy also keeps issues open for maintainer handling.

## What This Changes

The issue requests owner-facing lifecycle controls for ClawHub plugin packages, including package hide/delete/undelete and package-version withdrawal via website, CLI, and docs.

## Best Possible Solution

Define the package and package-release lifecycle policy, then implement one consistent owner/admin workflow across Convex mutations, HTTP routes, clawhub CLI, dashboard UI, tests, and public docs while coordinating version-specific work with https://github.com/openclaw/clawhub/issues/905 and https://github.com/openclaw/clawhub/issues/1947.

## Solution Assessment
