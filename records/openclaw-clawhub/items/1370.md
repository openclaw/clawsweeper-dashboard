## Summary

Keep open. The ClawHub cleanup policy keeps issues open, and current main still lacks a complete owner-facing package hide/delete/undelete or version-withdrawal workflow; only partial package-wide soft-delete plumbing exists.

## What This Changes

Add owner/admin hide, delete, undelete, and version delisting controls for ClawHub plugin packages across API, CLI, website UI, and docs.

## Best Possible Solution

Define package and release lifecycle semantics, then implement owner/admin controls end to end across Convex mutations, HTTP routes, CLI commands, management UI, tests, and public docs.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence current main only has package-wide soft-delete storage/API, not complete owner-facing package lifecycle controls.

## Solution Assessment

Unclear until maintainers decide package-wide restore versus release-level withdrawal semantics. The maintainable direction is to extend the existing ownership, soft-delete, audit, and moderation models rather than adding an ad hoc delete path.
