## Summary

Keep open. This is an issue, and the repository cleanup policy only permits automated closure for PRs that are certainly implemented on main; current main also still blocks GitHub reauthentication for deactivated accounts and has no account-restore or GitHub-identity-unlink path.

## What This Changes

The issue requests restoration of an accidentally deleted ClawHub account or unlinking the linked GitHub identity so the same GitHub account can register again.

## Best Possible Solution

Define an audited support/admin recovery or unlink workflow for deleted GitHub-linked accounts, with ownership verification, ban/deletion invariants, production runbook coverage, and regression tests for existing deactivated records.
