## Summary

Keep open. This is an open ClawHub issue, and the cleanup policy only permits automated closure for PRs; current main also still treats deleted accounts as permanently unable to sign in, so restoration or GitHub unlinking needs maintainer/operator policy and production handling.

## What This Changes

The reporter asks for restoration of a deleted ClawHub account or unlinking the GitHub identity so the same GitHub account can re-register.

## Best Possible Solution

Define an audited account recovery or GitHub unlink process that preserves ban, token, publisher, and purged-data boundaries, or explicitly uphold irreversible deletion and handle affected users through support.

## Reproduction Assessment

Yes, source-reproducible rather than live-production reproduced: deleteAccount sets deactivatedAt/purgedAt, GitHub reauth reaches handleDeletedUserSignIn, and tests assert the permanent deleted-account rejection. This does not verify the individual production account rows.

## Solution Assessment

No for ad hoc restoration or GitHub unlinking from public issue comments. The maintainable solution is a maintainer-approved, audited recovery/unlink policy or an explicit decision to keep deletion permanent.
