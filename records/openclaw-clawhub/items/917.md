## Summary

Keep open. Current main still treats owner-initiated account deletion as irreversible, and the requested account restoration or GitHub identity unlinking needs maintainer/operator policy and production account handling rather than cleanup closure.

## What This Changes

The issue asks for restoration of an accidentally deleted ClawHub account or unlinking the GitHub identity so the same GitHub account can register again.

## Best Possible Solution

Decide the deleted-account recovery policy, then either implement an audited admin/support restore or GitHub unlink flow with tests, or document and enforce that owner-initiated deletion remains permanent.

## Reproduction Assessment

Yes, source-reproducible rather than live-production reproduced: deleteAccount writes deactivatedAt/purgedAt, the GitHub OAuth callback rejects through handleDeletedUserSignIn, and tests assert the permanent deleted-account error. This does not verify the individual production account records.

## Solution Assessment

No for ad hoc restoration or direct GitHub unlinking from an issue comment. The maintainable path is a maintainer-approved recovery policy plus an audited implementation or an explicit decision to keep deletion permanent.
