## Summary

Keep open. ClawHub cleanup policy keeps open issues open, and current main still treats user-initiated deletion as irreversible with no GitHub identity unlink or account restoration path, so this needs maintainer/operator handling.

## What This Changes

Request restoration or GitHub identity unlinking after accidental account deletion so the same GitHub account can register or sign in again.

## Best Possible Solution

Define a maintainer-approved audited recovery or GitHub unlink workflow, or document and perform an operator-only irreversible-deletion response while preserving bans, tokens, publisher ownership, and audit records.
