## Summary

Keep open. Current main still deliberately treats owner-initiated deletion as irreversible, and the requested restoration or GitHub identity unlinking is a production account/security policy matter rather than a cleanup-close candidate.

## What This Changes

Request to restore a deleted ClawHub account or unlink its GitHub identity so the same GitHub account can re-register.

## Best Possible Solution

Define a maintainer-approved audited recovery/unlink policy for deleted GitHub-linked accounts, or explicitly document and enforce permanent deletion, then handle affected accounts through that operator path.

## Reproduction Assessment

Yes, source-reproducible: deleteAccount sets deactivatedAt/purgedAt, the GitHub OAuth callback calls handleDeletedUserSignIn, and tests assert the permanent deleted-account error. This does not verify the individual production account records.

## Solution Assessment

No for ad hoc restoration or GitHub unlinking from a public issue comment. The maintainable path is an audited admin/support recovery flow or an explicit product decision that deletion remains permanent.
