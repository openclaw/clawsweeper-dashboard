## Summary

Keep open. This is an open ClawHub support/policy issue, and current main still intentionally blocks reauthentication for permanently deleted users while a draft RFC leaves account recovery policy undecided.

## What This Changes

The issue requests restoration of an accidentally deleted ClawHub account or unlinking the associated GitHub identity so the same GitHub account can register again.

## Best Possible Solution

Decide and document the account-deletion recovery policy; if recovery is accepted, implement an audited same-provider restore or unlink flow with tests that preserve bans, autobans, identity matching, and purged-data boundaries.

## Reproduction Assessment

Yes, from source inspection rather than live production records: deleteAccount writes deactivatedAt/purgedAt, the OAuth callback calls handleDeletedUserSignIn, and tests assert that deleted users are rejected instead of restored.

## Solution Assessment

No for an ad hoc restore, unlink, or direct database edit. The maintainable solution is a maintainer-approved audited recovery path, or an explicit decision to keep deletion permanent with clearer support handling.
