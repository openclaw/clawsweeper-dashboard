## Summary

Keep open. Current main still treats user-initiated deletion as irreversible and blocks GitHub reauthentication for deactivated users, while the requested restoration or GitHub unlinking is security-sensitive production account recovery that needs maintainer/operator policy rather than cleanup closure.

## What This Changes

The issue requests restoration of an accidentally deleted ClawHub account or unlinking the GitHub identity so the same GitHub account can re-register.

## Best Possible Solution

Define an audited support/admin recovery or GitHub unlink flow, or explicitly confirm permanent deletion and route affected users through a private operator process that preserves ban, purge, token, handle, publisher, and audit boundaries.

## Reproduction Assessment

Yes for the general account-lock behavior: source inspection shows deletion sets deactivatedAt/purgedAt and OAuth rejects deactivated users. This does not verify the reporter's production rows.

## Solution Assessment

No for ad hoc restoration or unlinking from a public issue. The maintainable solution is an audited operator/admin recovery policy, or an explicit product decision that deleted accounts cannot be recovered.
