## Summary

Keep open. Current main still treats user-initiated deletion as permanent, blocks GitHub reauthentication for deactivated users, and has no restore or GitHub identity unlink flow; ClawHub cleanup policy also keeps open issues open for maintainer handling.

## What This Changes

The issue requests restoring an accidentally deleted ClawHub account or unlinking the GitHub identity so the same GitHub account can register again.

## Best Possible Solution

Define an audited maintainer/operator recovery or identity-unlink process, document the policy, and add regression coverage that preserves ban, ownership, and deletion protections.

## Reproduction Assessment

Yes, for the source-level blocked reauth path: deleteAccount writes deactivatedAt/purgedAt, the GitHub OAuth callback rejects deactivated users, and tests assert that rejection. The specific production records still require maintainer/operator access to verify.

## Solution Assessment

No, an ad hoc authAccounts clear or silent reactivation is not the safest solution. The maintainable path is an audited recovery policy with explicit identity checks and regression coverage.
