## Summary

Keep open. Current main still treats owner-initiated account deletion as irreversible and has no account restore or GitHub identity unlink flow; for ClawHub issues, this is security-sensitive maintainer/operator work rather than cleanup-closeable work.

## What This Changes

The issue requests restoration of an accidentally deleted ClawHub account or unlinking the reporter's GitHub identity so they can re-register.

## Best Possible Solution

Settle the deleted-account recovery policy, likely through the related cluster led by https://github.com/openclaw/clawhub/issues/504, then implement or document an audited restore/unlink/operator process consistently across OAuth, account deletion, auth bindings, tests, and user-facing docs.

## Reproduction Assessment

Yes, source-reproducible for the product behavior: deleteAccount writes deactivatedAt/purgedAt, GitHub OAuth calls handleDeletedUserSignIn, and tests assert the permanent deleted-account error. This does not verify the reporter's production database row.

## Solution Assessment

No for ad hoc restoration or GitHub unlinking from an issue comment. The maintainable solution is a maintainer-approved, audited recovery or unlink policy that preserves deletion privacy and ban protections.
