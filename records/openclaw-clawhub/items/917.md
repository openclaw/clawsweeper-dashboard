## Summary

Keep open. Current main still treats user-initiated deletion as irreversible, rejects later GitHub OAuth for deactivated users, and has no account restore or GitHub identity unlink path; because this is an open ClawHub issue, cleanup policy also does not auto-close it.

## What This Changes

The issue requests restoration of an accidentally deleted ClawHub account or unlinking the GitHub identity so the same GitHub account can register again.

## Best Possible Solution

Define an audited support/admin recovery or unlink workflow for deleted GitHub-linked accounts, including ownership verification, ban preservation, production runbook coverage, and tests for existing deactivated records.

## Reproduction Assessment

Yes. A code-level reproduction is clear: delete a GitHub-linked account so `deactivatedAt` is set, then sign in through GitHub and the auth callback rejects it; `convex/auth.test.ts` already covers the rejection path.

## Solution Assessment

No patch is under review. The requested outcome needs a maintainer-approved recovery/unlink policy and audited admin or operator path rather than an automated source-only change.
