## Summary

Keep open. This is an open ClawHub issue, and the cleanup policy only allows automatic closure for pull requests that current main certainly implements. Current main still treats user deletion as irreversible, blocks later GitHub OAuth reauthentication for deactivated accounts, and does not expose an account restore or GitHub identity unlink path.

## What This Changes

The issue asks ClawHub maintainers to restore a accidentally deleted account or unlink the reporter's GitHub identity so the same GitHub account can register again.

## Best Possible Solution

Define an audited support/admin recovery path for deleted GitHub-linked accounts, or document and perform an operational unlink/restore process for affected users, while preserving ban protections and adding tests for existing deleted-account records.
