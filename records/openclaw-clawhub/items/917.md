## Summary

Keep open. Current main still blocks GitHub sign-in for permanently deleted users and has no account-restore or GitHub identity unlink path; the requested recovery is a security-sensitive operator/product decision, not safe automated cleanup.

## What This Changes

The issue asks ClawHub to restore a deleted account or unlink the reporter's GitHub identity so the same GitHub account can register again.

## Best Possible Solution

Define an audited support/admin recovery policy for deleted GitHub-linked accounts, or explicitly document irreversible deletion with a private support route that preserves ban and privacy boundaries.

## Reproduction Assessment

Yes for the general lockout path: source inspection shows deleted accounts get deactivatedAt/purgedAt and GitHub OAuth rejects deactivated users. This does not verify the reporter's production row or identity ownership.

## Solution Assessment

No for ad hoc restoration or unlinking from a public issue. The maintainable solution is an audited operator recovery policy or an explicit product decision that deleted accounts cannot be recovered.
