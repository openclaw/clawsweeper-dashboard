## Summary

Keep open. This is an issue rather than a PR cleanup target, current main intentionally treats user-initiated deletion as irreversible, and account restoration or GitHub identity unlinking needs maintainer/operator policy plus production account handling.

## What This Changes

A user asks ClawHub operators to restore a deleted account or unlink its GitHub identity so the same GitHub account can register again.

## Best Possible Solution

Define a maintainer-approved deleted-account recovery policy and either implement an audited admin/support restore or GitHub unlink flow, or keep irreversible deletion explicit and handle affected accounts only through approved operations.

## Reproduction Assessment

Yes, source-reproducible: deleteAccount writes deactivatedAt/purgedAt, the GitHub OAuth callback rejects deactivated users, and tests assert the permanent deleted-account message. This does not verify the individual production records for the affected users.

## Solution Assessment

No for ad hoc restoration or direct GitHub unlinking from an issue comment. The maintainable solution is an audited maintainer-approved recovery policy or an explicit decision that user-initiated deletion remains permanent.
