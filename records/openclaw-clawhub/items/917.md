## Summary

Keep this open. This is an open ClawHub issue, and the cleanup policy for this repository only permits automated closure for pull requests that current main certainly implements; current main still treats deleted GitHub-linked accounts as blocked from reauthentication rather than restored or unlinked.

## What This Changes

The issue asks maintainers to restore an accidentally deleted ClawHub account or unlink the reporter's GitHub identity so the same GitHub account can register again.

## Best Possible Solution

Define an audited maintainer/operator recovery path for deleted GitHub-linked accounts, or document and perform a deliberate operational unlink/restore process for affected users while preserving ban and deletion-policy protections.

## Reproduction Assessment

Yes for the source-level behavior: a deleted account with deactivatedAt reaches handleDeletedUserSignIn during GitHub OAuth and is rejected with the deleted-account reauth message. No production-account reproduction was attempted because the requested restoration/unlink action requires operator access.

## Solution Assessment

No automated source-only cleanup is the best solution here. The maintainable path is a maintainer-approved support/admin recovery policy or operation, with tests around deleted-account records and GitHub authAccounts linkage if product behavior changes.
