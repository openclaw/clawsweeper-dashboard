## Summary

Keep open. This is an open ClawHub issue, and current main still deliberately blocks sign-in for permanently deleted users; resolving restoration or GitHub unlinking needs maintainer policy and operator handling rather than cleanup closure.

## What This Changes

The issue asks maintainers to restore a deleted ClawHub account or unlink the associated GitHub identity so the same GitHub account can register again.

## Best Possible Solution

Define an audited support/admin recovery or re-registration policy, then update Convex auth/account code, operator tooling, user-facing copy, docs, and regression tests to preserve ban and ownership protections.

## Reproduction Assessment

Yes, from source inspection rather than live production records. deleteAccount sets deactivatedAt/purgedAt, the GitHub OAuth callback calls handleDeletedUserSignIn, and tests assert the deleted-account rejection path.

## Solution Assessment

No for a one-off source patch or silent database edit. The maintainable solution is a maintainer-approved, audited recovery or unlink process that explicitly separates self-deleted accounts from banned or abuse-disabled accounts.
