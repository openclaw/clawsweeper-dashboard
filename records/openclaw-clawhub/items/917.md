## Summary

Keep open. Repository cleanup policy for ClawHub keeps issues open, and current main still treats user-initiated account deletion as irreversible with no restore or GitHub-identity-unlink workflow for affected users.

## What This Changes

Request restoration of an accidentally deleted ClawHub account or unlinking of the GitHub identity so the same GitHub account can register again.

## Best Possible Solution

Define an audited support/admin recovery or GitHub unlink workflow for deleted accounts, with ownership verification, deletion-policy approval, ban safeguards, operator runbook coverage, and regression tests for existing deleted-account records.

## Reproduction Assessment

Yes. A source-level reproduction is clear: delete a GitHub-linked account so deactivatedAt is set, then sign in through GitHub and handleDeletedUserSignIn rejects the callback; auth.test.ts covers the rejection path.

## Solution Assessment

No patch is under review. The requested outcome should be handled through a maintainer-approved recovery/unlink policy and audited admin/operator path rather than an ad hoc source-only change.
