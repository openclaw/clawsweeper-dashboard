## Summary

Keep open. Current main still treats user-initiated deletion as permanent and blocks GitHub reauth for deactivated accounts; the requested restore/unlink action needs maintainer account-recovery policy and operator access, and ClawHub issue cleanup should not auto-close issues.

## What This Changes

The issue asks for restoring an accidentally deleted ClawHub account or unlinking the associated GitHub identity so the same GitHub account can re-register.

## Best Possible Solution

Define a maintainer-approved, audited account recovery or GitHub-unlink policy across OAuth, authAccounts, deletion, admin tooling, docs, and regression tests, or explicitly keep deletion terminal with clearer support escalation.

## Reproduction Assessment

Yes. By source inspection, settings calls users.deleteAccount, which sets deactivatedAt/purgedAt, and OAuth completion calls handleDeletedUserSignIn, which rejects deactivated users; I did not inspect live production records.

## Solution Assessment

No for an autonomous restore or binding deletion. The maintainable path needs a policy-backed, audited recovery/unlink flow, or an explicit maintainer decision to keep deletion irreversible with support guidance.
