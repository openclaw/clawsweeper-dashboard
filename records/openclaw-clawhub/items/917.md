## Summary

Keep open. The issue asks for production account restoration or GitHub identity unlinking, and current main still intentionally blocks sign-in for permanently deleted users; for this repository, open ClawHub issues are not cleanup-close candidates, and the remaining action needs maintainer/operator policy judgment.

## What This Changes

The issue requests restoration of an accidentally deleted ClawHub account or unlinking the associated GitHub identity so the user can re-register.

## Best Possible Solution

Define an audited support/admin recovery or unlink process for deleted GitHub-linked accounts, then update auth/account code, operator docs/specs, and regression tests while preserving ban and ownership protections.

## Reproduction Assessment

Yes, from source inspection rather than live production records: account deletion writes deactivatedAt/purgedAt, OAuth calls handleDeletedUserSignIn, and tests assert the deleted-account rejection path.

## Solution Assessment

No for an ad hoc restore, unlink, or direct database edit. The maintainable direction is a maintainer-approved, audited recovery path that separates self-deleted accounts from banned or abuse-disabled accounts.
