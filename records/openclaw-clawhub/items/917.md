## Summary

Keep open. Current main intentionally treats user-initiated deletion as irreversible and still blocks GitHub reauthentication for deactivated users, while restoration or GitHub unlinking is a security-sensitive production account recovery decision that needs maintainer/operator handling.

## What This Changes

The issue asks for restoration of an accidentally deleted ClawHub account or unlinking the GitHub identity so the same GitHub account can register again.

## Best Possible Solution

Define an audited support/admin recovery or GitHub unlink policy, or explicitly confirm irreversible deletion and route affected users through a private operator process.

## Reproduction Assessment

Yes for the general lockout behavior: source inspection shows deletion sets deactivatedAt/purgedAt and OAuth rejects deactivated users. This does not verify the reporter's production rows.

## Solution Assessment

No for ad hoc restoration or unlinking from a public issue. The maintainable path is an audited operator/admin recovery decision, or an explicit product decision that deleted accounts cannot be recovered.
