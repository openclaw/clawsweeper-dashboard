## Summary

Keep open. Current main still treats user-initiated deletion as irreversible and blocks GitHub reauthentication, while restoration or GitHub identity unlinking is a security-sensitive operator/product decision that should not be automated from public issue data.

## What This Changes

The issue asks for account restoration or unlinking the reporter's GitHub identity so the same GitHub account can re-register after accidental ClawHub account deletion.

## Best Possible Solution

Define an audited private admin/support recovery policy for deleted GitHub-linked accounts, or explicitly reaffirm irreversible deletion, while preserving ban protections, identity ownership, and privacy semantics in tests and operator docs.

## Reproduction Assessment

Yes for the general lockout path: source inspection shows deleted accounts get deactivatedAt/purgedAt and GitHub OAuth rejects deactivated users. This does not verify the reporter's specific production row or identity ownership.

## Solution Assessment

No for ad hoc restoration or unlinking from a public issue. The maintainable solution is a private, audited operator recovery policy or an explicit product decision that deleted accounts cannot be recovered.
