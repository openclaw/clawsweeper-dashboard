## Summary

Keep open. This is a ClawHub support and policy issue, and repository cleanup policy keeps issues open; current main still intentionally blocks deleted-account reauthentication and has no account restore or GitHub identity unlink flow.

## What This Changes

The issue requests restoration of an accidentally deleted ClawHub account or unlinking the GitHub identity so the same GitHub account can register again.

## Best Possible Solution

Decide and document the deleted-account recovery policy, then either implement an audited same-GitHub restore/unlink admin flow or explicitly keep deletion permanent with a clear support process.

## Reproduction Assessment

Yes. From source inspection, deleteAccount writes deactivatedAt and purgedAt, the OAuth callback calls handleDeletedUserSignIn, and tests assert deleted users are rejected instead of restored.

## Solution Assessment

No for an ad hoc restore, unlink, or direct database edit. The maintainable path is a maintainer-approved recovery policy plus an audited operator/admin flow that preserves ban, identity, and purge boundaries.
