## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still treats user-initiated deletion as irreversible with no account restore or GitHub identity unlink path; the related reports show this is affecting multiple users and needs maintainer/operator handling.

## What This Changes

The issue asks for restoration of an accidentally deleted ClawHub account or unlinking the reporter's GitHub identity so the same GitHub account can re-register.

## Best Possible Solution

Define an audited support/admin recovery or GitHub unlink workflow for deleted GitHub-linked accounts, with ownership verification, audit logging, ban preservation, and regression coverage for existing deactivated records.

## Reproduction Assessment

Yes. A source-level reproduction is clear: delete a GitHub-linked account so deactivatedAt is set, then sign in through GitHub and the OAuth callback rejects the existing user; auth.test.ts already covers the rejection path.

## Solution Assessment

No patch is under review. The requested outcome should be solved through a maintainer-approved recovery/unlink policy and audited admin or operational path, not an ad hoc code-only bypass.
