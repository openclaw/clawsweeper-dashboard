## Summary

Keep open. Current main still treats owner-initiated account deletion as permanent, and the requested account restoration or GitHub unlink needs maintainer/operator policy handling rather than cleanup closure.

## What This Changes

The reporter asks for a deleted ClawHub account to be restored or for the GitHub identity to be unlinked so the same GitHub account can register again.

## Best Possible Solution

Decide and document the deleted-account recovery policy; if recovery is accepted, implement an audited same-provider restore or unlink flow with tests that preserve bans, autobans, identity matching, purged-data boundaries, and ownership rules.

## Reproduction Assessment

Yes, source-reproducible rather than live production: deleteAccount writes deactivatedAt/purgedAt, the OAuth callback calls handleDeletedUserSignIn, and tests assert deleted users are rejected instead of restored.

## Solution Assessment

No for ad hoc restoration or direct GitHub unlink from an issue comment. The maintainable solution is a maintainer-approved recovery policy plus an audited implementation or an explicit decision to keep deletion permanent.
