## Summary

Keep open. This is an open ClawHub issue, and the cleanup policy only permits automated closure for pull requests; current main also still intentionally blocks deleted-account reauthentication and does not provide a restore or GitHub identity unlink path.

## What This Changes

The issue requests restoration of an accidentally deleted ClawHub account or unlinking of its GitHub identity so the same GitHub account can re-register.

## Best Possible Solution

Define an audited support/admin recovery or unlink policy for deleted GitHub-linked accounts, or explicitly keep deletion permanent and route affected users through a private operator process.

## Reproduction Assessment

Yes for the general lockout behavior: source inspection shows deletion sets deactivatedAt/purgedAt and OAuth rejects deactivated users. This does not verify the reporter's production rows.

## Solution Assessment

No for ad hoc restoration or unlinking from a public issue. The maintainable solution is an audited operator/admin recovery policy, or an explicit product decision that deleted accounts cannot be recovered.
