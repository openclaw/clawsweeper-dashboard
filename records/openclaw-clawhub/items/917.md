## Summary

Keep open. Current main intentionally blocks sign-in for permanently deleted users and has no account restore or GitHub identity unlink path, while this issue needs security-sensitive maintainer/operator handling rather than cleanup closure.

## What This Changes

The issue requests restoring an accidentally deleted ClawHub account or unlinking the associated GitHub identity so the same GitHub account can register again.

## Best Possible Solution

Adopt one explicit policy: either a private audited recovery/unlink flow that preserves ban and purge semantics, or clear operator guidance that deleted accounts are not recoverable and account-specific requests must be handled privately.

## Reproduction Assessment

Yes for the general lockout behavior: source inspection shows account deletion writes deactivatedAt/purgedAt and the OAuth callback rejects deactivated users. This does not verify the reporter's production rows.

## Solution Assessment

No for ad hoc restoration or GitHub unlinking from a public issue. The maintainable solution needs a private, audited operator/admin recovery decision or an explicit irreversible-deletion policy confirmation.
