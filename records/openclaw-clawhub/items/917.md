## Summary

Keep open. Current main still intentionally makes user-initiated deletion irreversible and has no account restore or GitHub identity unlink flow; for ClawHub issues, cleanup policy also only allows auto-closing PRs that are certainly implemented on main.

## What This Changes

The issue asks maintainers to restore a deleted ClawHub account or unlink its GitHub identity so the same GitHub account can register again.

## Best Possible Solution

Define an audited maintainer/operator recovery or identity-unlink process, document the policy, and add coverage that preserves ban and ownership protections for deleted-account records.

## Reproduction Assessment

Yes, from source inspection rather than live production records. deleteAccount sets deactivatedAt/purgedAt, the GitHub auth callback rejects that state, and tests assert the rejection path.

## Solution Assessment

No, an ad hoc authAccounts clear or silent reactivation is not the safest solution. The maintainable path is an audited admin/operator recovery policy with explicit identity checks and regression coverage.
