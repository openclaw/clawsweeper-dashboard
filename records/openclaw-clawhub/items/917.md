## Summary

Keep this open. Current main still treats user-initiated deletion as irreversible, blocks GitHub reauthentication for deactivated users, and has no account restore or GitHub identity unlink flow; ClawHub cleanup policy also only permits automated closure for PRs that are certainly implemented on main.

## What This Changes

The issue asks for restoration of an accidentally deleted ClawHub account or unlinking the GitHub identity so the same GitHub account can register again.

## Best Possible Solution

Define an audited support/admin recovery or identity-unlink process for deleted GitHub-linked accounts, update the security intent and operator docs, and add coverage for deleted-account records without weakening ban or ownership protections.

## Reproduction Assessment

Yes, from source inspection rather than live production records. deleteAccount sets deactivatedAt/purgedAt, the GitHub auth callback rejects that user on later sign-in, and tests assert the rejection path.

## Solution Assessment

No, an ad hoc binding clear or silent reactivation is not the safest solution. The maintainable path is a verified, audited operator/admin recovery flow with explicit policy for deleted identities and related ownership data.
