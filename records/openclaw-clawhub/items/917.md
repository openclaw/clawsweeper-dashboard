## Summary

Keep open. ClawHub cleanup policy does not allow auto-closing issues, and current main still treats user-initiated deletion as irreversible while blocking GitHub reauthentication for deactivated users.

## What This Changes

The issue asks maintainers to restore a deleted ClawHub account or unlink its GitHub identity so the same GitHub account can register again.

## Best Possible Solution

Define an audited maintainer/operator recovery or unlink process for self-deleted GitHub-linked accounts, then update auth/account code, operator docs, specs, and regression tests while preserving ban and ownership protections.

## Reproduction Assessment

Yes, from source inspection: deleting an account sets deactivatedAt/purgedAt, OAuth sign-in calls handleDeletedUserSignIn, and tests assert that deactivated users are rejected with the deleted-account message.

## Solution Assessment

No for an ad hoc database edit or silent GitHub unlink. The maintainable path is a maintainer-approved audited recovery flow that explicitly separates self-deleted users from banned or abuse-disabled accounts.
