## Summary

Keep open. Current main still treats user-initiated deletion as irreversible and blocks GitHub reauth for deactivated accounts; ClawHub cleanup policy also keeps issues open unless they are PRs already implemented on main.

## What This Changes

The issue requests restoration of an accidentally deleted ClawHub account or unlinking the reporter's GitHub identity so the same GitHub account can register again.

## Best Possible Solution

Define an audited support/admin recovery or GitHub unlink workflow for deleted accounts, with ownership verification, deletion-policy approval, ban safeguards, operator runbook coverage, and regression tests for existing deleted-account records.

Yes. A source-level reproduction is clear: delete a GitHub-linked account so deactivatedAt is set, then sign in through GitHub and handleDeletedUserSignIn rejects the callback; auth.test.ts covers that rejection path.
