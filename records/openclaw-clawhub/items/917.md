## Summary

Keep open. Current main intentionally treats user-initiated deletion as irreversible, and the requested production identity recovery needs maintainer/operator policy and security review rather than cleanup closure.

## What This Changes

The issue requests restoration of an accidentally deleted ClawHub account or unlinking the reporter’s GitHub identity so the same GitHub account can re-register.

## Best Possible Solution

Define a maintainer-approved audited support/admin recovery or unlink flow for deleted GitHub-linked accounts, or explicitly confirm that deletion remains permanent and route users through a private operator process.

## Reproduction Assessment

Yes for the general account-lock behavior: source inspection shows deletion sets deactivatedAt/purgedAt and OAuth rejects deactivated users. This does not verify the reporter’s production rows.

## Solution Assessment

No for ad hoc restoration or unlinking from a public issue. The maintainable solution is an audited operator/admin recovery policy, or an explicit product decision that deleted accounts cannot be recovered.
