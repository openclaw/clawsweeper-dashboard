## Summary

Keep open. This is a ClawHub issue, and the cleanup policy only permits auto-closing pull requests that current main certainly implements; current main still treats user deletion as irreversible and has no GitHub identity unlink or account restore path for affected production users.

## What This Changes

The issue requests restoration of an accidentally deleted ClawHub account or unlinking the linked GitHub identity so the same GitHub account can re-register.

## Best Possible Solution

Define an audited support/admin recovery or GitHub identity unlink process for deleted GitHub-linked accounts, with clear policy, operator controls, and tests for existing deactivated records.

## Reproduction Assessment

Yes. A high-confidence source-backed path is: link a GitHub account, call deleteAccount, then attempt GitHub OAuth again; current code sets deactivatedAt and the OAuth callback throws the deleted-account reauth error, with tests covering the helper behavior.

## Solution Assessment

Unclear for implementation because no patch is proposed. The requested direction is valid, but the safest solution needs maintainer/operator policy for account ownership verification, audit logging, and deletion invariants before code automation should change it.
