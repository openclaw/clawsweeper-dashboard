## Summary

Keep open. Current main still treats user-initiated deletion as irreversible and rejects GitHub OAuth reauthentication for deactivated users, while restoration or GitHub identity unlinking is a security-sensitive production recovery decision that needs maintainer/operator handling.

## What This Changes

The issue requests account restoration or unlinking a GitHub identity after accidental ClawHub account deletion so the same GitHub account can register or sign in again.

## Best Possible Solution

Define an audited operator/admin recovery policy for deleted GitHub-linked accounts, or explicitly confirm irreversible deletion and route affected users through a private support process that preserves ban and privacy boundaries.

## Reproduction Assessment

Yes for the general lockout path: source inspection shows deleted accounts get deactivatedAt/purgedAt and GitHub OAuth rejects deactivated users. This does not verify the reporter's production row or identity ownership.

## Solution Assessment

No for ad hoc restoration or unlinking from a public issue. The maintainable solution is an audited maintainer/operator recovery policy, or a documented product decision that deleted accounts cannot be recovered.
