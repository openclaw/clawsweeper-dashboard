## Summary

Keep open. Current main still encodes user-initiated deletion as irreversible and rejects GitHub reauth for deactivated users; because this is an open ClawHub issue and identity recovery is security-sensitive support work, it needs maintainer follow-up rather than cleanup closure.

## What This Changes

The issue asks maintainers to restore an accidentally deleted ClawHub account or unlink the GitHub identity so the user can re-register.

## Best Possible Solution

Define an audited support/admin recovery or GitHub unlink workflow, with ownership verification, deletion-policy approval, ban safeguards, operator runbook coverage, and regression tests for existing deleted-account records.

## Reproduction Assessment

Yes. A source-level reproduction is clear: delete a GitHub-linked account so deactivatedAt is set, then sign in through GitHub and handleDeletedUserSignIn rejects the callback; auth.test.ts covers that path.

## Solution Assessment

No patch is under review. The maintainable direction is a maintainer-approved recovery/unlink policy and audited admin/operator path rather than an ad hoc source-only or database-only change.
