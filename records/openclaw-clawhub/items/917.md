## Summary

Keep open. Current main intentionally blocks GitHub sign-in for permanently deleted users and has no restore or GitHub identity unlink flow; this ClawHub issue needs maintainer/operator policy and production-account handling.

## What This Changes

The issue requests restoration of an accidentally deleted ClawHub account or unlinking the GitHub identity so the same GitHub account can re-register.

## Best Possible Solution

Define an audited support/admin recovery or GitHub unlink process for self-deleted accounts, then update auth/account code, operator docs/specs, and regression tests while preserving ban and ownership protections.

## Reproduction Assessment

Yes, from source inspection and tests rather than live production records: deleteAccount records deactivatedAt/purgedAt, OAuth sign-in routes through handleDeletedUserSignIn, and tests assert deleted users are rejected.

## Solution Assessment

No for an ad hoc restore, unlink, or direct database edit. The maintainable solution is a maintainer-approved audited recovery path that separates self-deleted accounts from banned or abuse-disabled accounts.
