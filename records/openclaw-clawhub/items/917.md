## Summary

Keep open. Current main still blocks GitHub sign-in for deactivated/purged accounts and has no account-restore or GitHub identity unlink path; the live discussion and related open reports show a real account-access cluster that needs maintainer/operator security and product review, not conservative cleanup.

## What This Changes

The issue asks for accidental account deletion recovery through account restoration or unlinking a GitHub identity so the same GitHub account can register again.

## Best Possible Solution

Define an audited private support/admin recovery policy that restores or unlinks deleted GitHub identities only after identity verification and ban/privacy checks, or explicitly document deletion as irreversible with a clear support response.

## Reproduction Assessment

Yes for the general lockout path: source inspection shows deleteAccount sets deactivatedAt/purgedAt and GitHub OAuth calls handleDeletedUserSignIn, which rejects deactivated users. This does not verify the reporter's production row or identity ownership.

## Solution Assessment

No for ad hoc restoration or unlinking from public issue data. The maintainable solution is a private verified operator workflow or an explicit product decision that deletion is irreversible.
