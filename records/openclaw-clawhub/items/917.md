Keep open. This is a ClawHub issue, and cleanup policy does not allow automated closure for open issues; current main still intentionally blocks GitHub reauth for permanently deleted users, so restoration or identity unlinking needs maintainer/operator policy and production-account handling.

## What This Changes

The issue requests restoration of a deleted ClawHub account or unlinking of the associated GitHub identity so the same GitHub account can register again.

## Best Possible Solution

Define an audited support/admin recovery or unlink process for deleted GitHub-linked accounts, then update auth/account code, operator tooling, docs, specs, and regression tests while preserving ban and ownership protections.

## Reproduction Assessment

Yes, from source inspection rather than live production records: deleteAccount records deactivatedAt and purgedAt, the OAuth callback routes deleted users through handleDeletedUserSignIn, and tests assert the rejection path.

## Solution Assessment

No for an ad hoc restore, unlink, or direct database edit. The maintainable solution is a maintainer-approved audited recovery path that explicitly separates self-deleted accounts from banned or abuse-disabled accounts.
