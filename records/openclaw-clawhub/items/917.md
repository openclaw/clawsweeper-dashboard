## Summary

Keep open. This is a ClawHub issue, so cleanup policy does not allow automated closure here, and current main still intentionally blocks GitHub reauth for permanently deleted users; restoration or identity unlinking needs maintainer/operator policy and production-account handling.

## What This Changes

The issue asks maintainers to restore a deleted ClawHub account or unlink the reporter's GitHub identity so the same GitHub account can register again.

## Best Possible Solution

Define an audited support/admin recovery or unlink process for deleted GitHub-linked accounts, then update auth/account code, operator tooling, docs, and regression tests while preserving ban and ownership protections.

## Reproduction Assessment

Yes, from source inspection rather than live production records: deleteAccount sets deactivatedAt and purgedAt, the OAuth callback calls handleDeletedUserSignIn, and tests assert the deleted-account rejection path.

## Solution Assessment

No for an ad hoc unlink, restore, or silent database edit. The maintainable solution is a maintainer-approved audited recovery path that explicitly separates self-deleted accounts from banned or abuse-disabled accounts.
