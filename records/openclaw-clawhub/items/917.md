## Summary

Keep open. This is a ClawHub issue, and the cleanup policy keeps issues open; current main still treats user deletion as irreversible and exposes no account restore or GitHub identity unlink path for the reported recovery request.

## What This Changes

The issue requests restoring a deleted ClawHub account or unlinking the associated GitHub identity so the same GitHub account can register again.

## Best Possible Solution

Define an audited support/admin recovery path for deleted GitHub-linked accounts, or document and perform an operational unlink/restore process for affected users while preserving deletion, ban, token, publisher, and content invariants.

## Reproduction Assessment

Yes. The code path is reproducible from source and tests: deleteAccount sets deactivatedAt/purgedAt, then GitHub OAuth sign-in reaches handleDeletedUserSignIn and throws the deleted-account reauth error for deactivated users.

## Solution Assessment

Unclear. Account restoration or GitHub unlinking may be the right product outcome, but it should be implemented as an audited maintainer/operator flow rather than an ad hoc source-only change.
