## Summary

Keep open. Current main still treats user-initiated deletion as irreversible, blocks later GitHub OAuth reauthentication for deactivated accounts, and has no account restore or GitHub identity unlink path; related open reports show this affects multiple users and needs maintainer/operator handling.

## What This Changes

Request restoration of a deleted ClawHub account or unlinking of the GitHub OAuth identity so the same GitHub account can register again.

## Best Possible Solution

Define an audited support/admin recovery or unlink workflow for deleted GitHub-linked accounts, including ownership verification, ban preservation, production runbook coverage, and tests for existing deactivated records.

## Reproduction Assessment

Yes. A code-level reproduction is clear: delete a GitHub-linked account so `deactivatedAt` is set, then sign in again through GitHub and the auth callback rejects it; `convex/auth.test.ts` already covers this rejection path.

## Solution Assessment

No proposed patch is under review. The requested outcome should be solved through a maintainer-approved recovery/unlink policy and audited admin path rather than an ad hoc source-only change.
