## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still intentionally rejects GitHub reauth for permanently deleted users with no restore or identity-unlink path.

## What This Changes

The issue requests restoration of an accidentally deleted ClawHub account or unlinking the associated GitHub identity so the user can re-register.

## Best Possible Solution

Define a maintainer-approved, audited support/admin recovery or unlink process for deleted GitHub-linked accounts, with tests and docs that preserve ban and ownership protections.

## Reproduction Assessment

Yes, from source inspection rather than live production records: deleteAccount writes deactivatedAt/purgedAt, the OAuth callback calls handleDeletedUserSignIn, and tests assert deleted-account rejection.

## Solution Assessment

No for an ad hoc unlink, restore, or silent database edit. The maintainable path is an audited maintainer/operator recovery flow that clearly separates self-deleted accounts from banned or abuse-disabled accounts.
