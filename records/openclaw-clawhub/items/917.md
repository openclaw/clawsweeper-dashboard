## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits automated closure for PRs; current main also still treats deleted accounts as permanently deactivated and has no user/admin path to restore the account or unlink the GitHub identity for re-registration.

## What This Changes

The issue requests restoration of an accidentally deleted ClawHub account or unlinking of the reporter's GitHub identity so the same GitHub account can register again.

## Best Possible Solution

Define an audited support/admin recovery or re-registration policy, then update the Convex auth/account code, operator tooling, user-facing copy, specs/docs, and regression tests while preserving ban and ownership protections.

## Reproduction Assessment

Yes, from source inspection rather than live production records: deleted users have deactivatedAt/purgedAt set, GitHub OAuth calls handleDeletedUserSignIn, and tests assert the deleted-account rejection path.

## Solution Assessment

No for a one-off source patch or silent database edit. The maintainable solution is a maintainer-approved, audited recovery or unlink process that separates self-deleted accounts from banned or abuse-disabled accounts.
