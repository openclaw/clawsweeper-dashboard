## Summary

Keep open. Current main still intentionally blocks deleted-account GitHub reauthentication and has no account restore or GitHub identity unlink path; this issue also remains an open ClawHub issue, so cleanup policy does not allow auto-closing it.

## What This Changes

The reporter requests restoration of an accidentally deleted ClawHub account or unlinking of the associated GitHub identity so they can register again.

## Best Possible Solution

Define an audited private support/admin recovery policy that restores or unlinks deleted GitHub identities only after identity verification, ban/privacy checks, and audit logging, or explicitly document deletion as irreversible with a clear support response.

Yes for the general lockout path: source inspection shows deleteAccount sets deactivatedAt/purgedAt and GitHub OAuth calls handleDeletedUserSignIn, which rejects deactivated users. This does not verify the reporter's production row or identity ownership.

## Solution Assessment

No for ad hoc restoration or unlinking from public issue data. The maintainable solution is a private verified operator workflow or an explicit product decision that deletion is irreversible.

## Security Review

Status: needs_attention

Summary: Account restoration or GitHub unlinking is security-sensitive because it changes identity, deletion, ban, and ownership boundaries.

Concerns:

- **[medium] Treat GitHub unlinking as privileged recovery:** `convex/lib/githubIdentity.ts:13`
  - body: authAccounts.providerAccountId is the GitHub identity anchor used by ownership/security logic, so unlinking or rebinding it must require private identity verification and audit logging.
  - confidence: 0.9
- **[medium] Preserve deleted and banned account separation:** `convex/auth.ts:45`
  - body: The auth path distinguishes deactivated accounts from ban-related deleted users, so any recovery flow must not let reauthentication bypass bans or malware autobans.
  - confidence: 0.88
- **[medium] Respect purged-account deletion semantics:** `convex/users.ts:486`
  - body: deleteAccount clears profile, contact, GitHub-derived, and telemetry fields while setting purgedAt, so restoration must not silently recreate or expose data intentionally purged by deletion.
  - confidence: 0.86
