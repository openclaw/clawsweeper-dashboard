## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy keeps issues open; current main also still treats user-initiated deletion as irreversible and has no restore or GitHub identity unlink path for the requested account recovery.

## What This Changes

The issue asks for a deleted ClawHub account to be restored or for the reporter's GitHub identity to be unlinked so the same GitHub account can register again.

## Best Possible Solution

Define an audited support/admin recovery or unlink workflow for deleted GitHub-linked accounts, including ownership verification, ban preservation, production runbook coverage, and tests for existing deactivated records.

## Reproduction Assessment

Yes. A source-level path is clear: deleteAccount sets deactivatedAt, GitHub OAuth reauth calls handleDeletedUserSignIn, and that helper rejects deactivated users; convex/auth.test.ts covers this rejection path.

## Solution Assessment

No proposed patch is under review. The maintainable solution is a maintainer-approved, audited account recovery or unlink path rather than a one-off database edit or source-only bypass.
