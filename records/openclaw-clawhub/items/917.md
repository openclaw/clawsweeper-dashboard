## Summary

Keep open. This is an open ClawHub issue and current main still treats user-initiated deletion as irreversible, so restoration or GitHub identity unlinking needs maintainer/operator policy rather than cleanup closure.

## What This Changes

The issue asks for a deleted ClawHub account to be restored or for the linked GitHub identity to be unbound so the user can register again.

## Best Possible Solution

Define a maintainer-approved audited support/admin recovery path for deleted GitHub-linked accounts, or explicitly confirm that deletion remains permanent and route affected users through a private operator process.

## Reproduction Assessment

Yes. The general account-lock behavior is source-reproducible: deleteAccount sets deactivatedAt/purgedAt, GitHub OAuth calls handleDeletedUserSignIn, and tests assert the permanent deleted-account error; this does not verify the individual production rows.

## Solution Assessment

No for ad hoc restoration or GitHub unlinking from a public issue. The maintainable solution is an audited operator/admin recovery policy, or an explicit product decision that deleted accounts cannot be recovered.
