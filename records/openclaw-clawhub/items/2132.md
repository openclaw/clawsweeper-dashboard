## Summary

Keep open. Current main still supports org-scoped dashboard reads in the backend, but the Dashboard UI auto-selects the personal publisher and exposes no visible control to switch to an organization publisher, matching the reported failure mode.

## What This Changes

The issue requests a Dashboard publisher switcher or grouped view so organization-owned packages are visible to members after transfer or publish.

## Best Possible Solution

Add a Dashboard publisher selector backed by `api.publishers.listMine`, scope both skills and packages to the selected publisher, carry the selected owner into creation actions where supported, and add regression coverage for a user with personal plus org publishers.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a signed-in user with both a personal publisher and an org membership lands on `/dashboard`, the effect selects the personal publisher, and no rendered control calls `setSelectedPublisherId` to switch to the org.

## Solution Assessment

Yes. A Dashboard owner selector is the narrow maintainable fix because the backend already accepts `ownerPublisherId` and enforces membership for org-owned package and skill dashboard reads.
