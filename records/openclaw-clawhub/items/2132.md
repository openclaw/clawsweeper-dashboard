## Summary

Keep open: current main still leaves organization-owned packages unreachable from the dashboard UI because it auto-selects the personal publisher and renders no control to switch to an org publisher, while the backend already supports owner-publisher dashboard reads.

## What This Changes

The issue requests a dashboard publisher selector or equivalent grouped view so org-owned packages are visible to org owners/admins/publishers after transfer or publish.

## Best Possible Solution

Add a dashboard publisher selector backed by `api.publishers.listMine`, keep package and skill queries scoped to the selected publisher, align publish actions with the selected handle, and cover the personal-plus-org case in dashboard tests.

## Reproduction Assessment

Yes. A source-level reproduction path is clear: sign in as a user with both a personal publisher and an org publisher, load `/dashboard`, and the code selects the personal publisher while exposing no UI that can set the org publisher id.

## Solution Assessment

Yes. Reintroducing a visible dashboard publisher selector is the narrowest maintainable fix because existing backend queries already support owner-publisher scoping and membership checks; grouped sections or persistence can be follow-ups.
