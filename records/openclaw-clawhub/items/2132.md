## Summary

Keep open. Current main still has the Dashboard publisher-selection gap described in the issue, and ClawHub cleanup policy keeps issues open for maintainer follow-up rather than auto-closing them.

## What This Changes

The issue requests that the Dashboard expose organization-owned packages to signed-in members by allowing publisher/org scope selection instead of silently using the personal publisher.

## Best Possible Solution

Add a Dashboard publisher selector backed by `api.publishers.listMine`, scope package and skill lists to the selected publisher, and add regression coverage for a user with both personal and organization publishers.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a signed-in user with both a personal publisher and an org membership lands on `/dashboard`, the effect selects the personal publisher, and no rendered control changes the selected publisher to the org.

## Solution Assessment

Yes. A Dashboard owner selector is the narrow maintainable fix because the backend already supports `ownerPublisherId` and enforces membership for org-owned package and skill dashboard reads.
