## Summary

Keep open: current main still defaults the dashboard to the personal publisher and renders no control that can switch the dashboard to an org publisher, while the backend already supports membership-checked org-owned package and skill dashboard reads.

## What This Changes

The issue requests a dashboard publisher selector so signed-in org members can view and publish against organization-owned packages and skills.

## Best Possible Solution

Add a `/dashboard` publisher selector backed by `api.publishers.listMine`, scope skills and packages to the selected publisher, pass the selected owner into publish actions, and cover the personal-plus-org case in dashboard tests.

## Reproduction Assessment

Yes. A high-confidence source path exists: sign in as a user with both personal and org publishers, load `/dashboard`, and current code auto-selects the personal publisher without rendering a way to set the org publisher id.

## Solution Assessment

Yes. A visible publisher selector is the narrowest maintainable fix because the dashboard queries already accept `ownerPublisherId` and the orgs spec already requires owner selection for dashboard quick actions.
