## Summary

Keep open. Current main still matches the reported gap: the Dashboard has publisher-scoped query plumbing but still auto-selects the personal publisher and exposes no visible way to switch to an org publisher.

## What This Changes

The issue asks for `/dashboard` to show packages and skills for organization publishers the signed-in user can manage, or provide a publisher selector.

## Best Possible Solution

Add a Dashboard publisher selector backed by `api.publishers.listMine`, scope both skills and packages to the selected publisher, and cover a personal-plus-org publisher case in route tests.

## Reproduction Assessment

Yes, by source inspection. A signed-in user with both a personal publisher and an org membership lands on `/dashboard`, the effect selects the personal publisher, and no rendered control changes `selectedPublisherId` to the org.

## Solution Assessment

Yes. A Dashboard owner selector is the narrow maintainable fix because backend package and skill queries already accept `ownerPublisherId` and enforce publisher membership.
