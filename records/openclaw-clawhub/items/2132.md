## Summary

Keep open: current main still defaults the dashboard to the personal publisher and does not render a control that can switch to an org publisher, while the backend and spec already support owner-publisher scoped dashboard reads.

## What This Changes

The issue requests that `/dashboard` expose organization-owned publisher packages for users who can manage those organization publishers.

## Best Possible Solution

Add a visible `/dashboard` publisher selector for multi-publisher users, keep personal as the default, and scope dashboard content plus publish quick actions to the selected owner with regression coverage.

## Reproduction Assessment

Yes. A high-confidence source path exists: sign in as a user with both personal and org publishers, load `/dashboard`, and current code auto-selects the personal publisher without rendering a way to set the org publisher id.

## Solution Assessment

Yes. A visible publisher selector is the narrowest maintainable fix because the dashboard queries already accept `ownerPublisherId` and the orgs spec already requires owner selection for dashboard quick actions.
