## Summary

Keep open. Current main still matches the reported gap: org publisher membership is available to the dashboard and backend queries support `ownerPublisherId`, but the route auto-selects the personal publisher and exposes no visible control to switch to the org; ClawHub policy keeps issues open rather than auto-closing them.

## What This Changes

The issue asks for `/dashboard` to show organization-owned packages for publisher members, either by adding a publisher selector or grouping managed publishers.

## Best Possible Solution

Add a Dashboard publisher selector backed by `api.publishers.listMine`, scope both skills and packages to the selected publisher, make the selected handle clear in empty states and creation actions, and cover personal-plus-org publishers in route tests.

## Reproduction Assessment

Yes, by source inspection. A signed-in user with personal and org publishers lands on the personal publisher, and current main has no visible dashboard control that changes `selectedPublisherId` to the org.

## Solution Assessment

Yes. A dashboard owner selector is the narrow maintainable fix because the backend queries and publish flows already understand publisher membership and `ownerPublisherId`.
