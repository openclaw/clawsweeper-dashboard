## Summary

Keep open: current main still defaults the dashboard to the personal publisher and does not render a control that can switch to an org publisher, while the backend already supports membership-checked org-owned dashboard reads.

## What This Changes

Expose organization publisher-owned dashboard packages after transfer/publish, preferably through a dashboard publisher selector.

## Best Possible Solution

Add a focused `/dashboard` publisher selector backed by `api.publishers.listMine`, scope skills and packages to the selected publisher, pass the selected owner into publish actions, and cover the personal-plus-org case in dashboard tests.

## Reproduction Assessment

Yes. A high-confidence source path exists: sign in as a user whose `listMine` returns personal plus org publishers, load `/dashboard`, and current code auto-selects the personal publisher without rendering a way to select the org publisher.

## Solution Assessment

Yes. A visible dashboard publisher selector is the narrowest maintainable fix because the dashboard queries already accept `ownerPublisherId` and the orgs spec already calls for dashboard quick-action owner selection.
