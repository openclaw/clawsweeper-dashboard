## Summary

Keep open: current main still auto-selects the personal publisher on `/dashboard` and exposes no UI control to switch to an org publisher, while the backend already supports membership-checked org dashboard reads.

## What This Changes

The issue requests a dashboard publisher selector so signed-in members can view and publish against organization-owned packages and skills.

## Best Possible Solution

Add a `/dashboard` publisher selector backed by `api.publishers.listMine`, keep personal as the default, scope dashboard skills/packages and publish quick actions to the selected owner, and cover the personal-plus-org case in tests.

## Reproduction Assessment

Yes. A high-confidence source path exists: sign in as a user with both personal and org publishers, load `/dashboard`, and current code auto-selects the personal publisher without rendering a way to set the org publisher id.

## Solution Assessment

Yes. A visible publisher selector is the narrowest maintainable fix because the dashboard queries already accept `ownerPublisherId` and the orgs spec already requires owner selection for dashboard quick actions.
