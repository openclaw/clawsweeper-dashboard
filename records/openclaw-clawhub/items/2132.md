## Summary

Keep open. Current main still appears to have the reported Dashboard owner-selection gap: organization-owned package reads are supported by backend queries, but the Dashboard auto-selects a personal publisher and exposes no visible control to switch to an organization publisher.

## What This Changes

The issue requests a Dashboard publisher selector or grouped view so signed-in organization members can view organization-owned packages after transfer or publish.

## Best Possible Solution

Add a Dashboard publisher switcher backed by listMine, scope packages and skills to the selected publisher, and cover personal-plus-organization membership in Dashboard tests.

## Reproduction Assessment

Yes, by source inspection. A signed-in user with both a personal publisher and an organization membership lands on the personal publisher because the effect prefers kind === 'user', and no rendered control changes selectedPublisherId.

## Solution Assessment

Yes. The narrow maintainable fix is a Dashboard owner selector because backend listMine, package listing, and skill dashboard paths already accept organization publisher scope and enforce membership.
