## Summary

Keep open. Current main still has an actionable dashboard UI gap: organization-owned package listing is supported after a publisher is selected, but the dashboard auto-selects the personal publisher and exposes no visible way to switch to an organization publisher.

## What This Changes

The requested change is to expose organization publisher packages on the dashboard, either with a publisher selector or grouped publisher sections.

## Best Possible Solution

Add a dashboard publisher switcher backed by `api.publishers.listMine`, scope both skills and packages to the selected publisher, carry the selected owner into supported creation flows, and add regression coverage for a user with both personal and organization publishers.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a signed-in user with personal and organization publishers lands on `/dashboard`, the effect selects the personal publisher, and no rendered control calls `setSelectedPublisherId` to switch to the organization.

## Solution Assessment

Yes. A dashboard publisher selector is the narrow maintainable direction because backend package and skill listings already accept `ownerPublisherId` and enforce membership for organization-owned dashboard reads.
