## Summary

Keep open: current main still appears to have the reported dashboard gap. The backend dashboard queries can list org-owned skills/packages by `ownerPublisherId`, but `/dashboard` defaults to the personal publisher and exposes no control to switch to an organization publisher.

## What This Changes

The issue asks for `/dashboard` to show organization-owned packages for publishers the signed-in user can manage, or to provide a publisher/org switcher.

## Best Possible Solution

Add a visible dashboard publisher selector backed by `api.publishers.listMine`, preserve the personal default, and route the selected publisher through dashboard lists, empty states, and publish-entry links without changing backend ownership semantics.

## Reproduction Assessment

Yes. With a signed-in user who has a personal publisher plus an org publisher containing packages, current source initializes the dashboard to the personal publisher and has no rendered control that can switch to the org publisher.

## Solution Assessment

Yes. A narrow selector or equivalent grouped dashboard view is the maintainable fix because the backend already authorizes and lists by `ownerPublisherId`; changing package ownership, search, or transfer behavior would be the wrong layer for this report.
