## Summary

Keep open: current main still defaults the dashboard to the personal publisher and does not render a control that can switch to an organization publisher, while the backend and orgs spec already support org-scoped dashboard reads.

## What This Changes

The issue requests a dashboard publisher selector so organization-owned packages and skills are visible and manageable after publish or transfer.

## Best Possible Solution

Add a /dashboard publisher selector backed by api.publishers.listMine, scope skills and packages to the selected publisher, pass the selected owner into publish actions, and cover the personal-plus-org case in dashboard tests.

## Reproduction Assessment

Yes. A high-confidence source path exists: sign in as a user with both personal and org publishers, load /dashboard, and current code auto-selects the personal publisher without exposing a way to set the org publisher id.

## Solution Assessment

Yes. A visible publisher selector is the narrowest maintainable fix because the dashboard queries already accept ownerPublisherId and specs/orgs.md already requires owner selection for dashboard quick actions.
