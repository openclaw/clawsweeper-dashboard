## Summary

Keep open: current main still has backend support for org-owned dashboard package and skill reads, but the dashboard UI defaults to the personal publisher and exposes no visible publisher switcher, so organization-owned packages can remain unreachable from the page.

## What This Changes

The issue asks for the dashboard to expose packages owned by organization publishers the signed-in user can manage, ideally through a publisher selector or grouped owner sections.

## Best Possible Solution

Add a dashboard publisher selector backed by api.publishers.listMine, keep skill/package queries scoped to the selected publisher, align publish actions with the selected owner handle, and cover personal-plus-org membership in dashboard tests.

## Reproduction Assessment

Yes, as a source-level reproduction path: sign in as a user with both a personal publisher and an org publisher, load /dashboard, and the code selects the personal publisher while rendering no control that can switch selectedPublisherId to the org.

## Solution Assessment

Yes, reintroducing a visible dashboard publisher selector is the narrowest maintainable fix because the backend owner-publisher listing and membership checks already exist; grouped dashboard sections or persistence can be follow-ups.
