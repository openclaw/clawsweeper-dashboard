## Summary

Keep open. Current main has backend support for listing org-owned dashboard packages when an org publisher is selected, but the dashboard UI auto-selects the personal publisher and exposes no control that can change the selection, so org packages can remain hidden from a member’s dashboard.

## What This Changes

Expose organization publisher packages in `/dashboard` after transfer or publish by letting eligible users view or switch from their personal publisher to an organization publisher.

## Best Possible Solution

Add a dashboard publisher switcher backed by `api.publishers.listMine`, scope both skills and packages to the selected publisher, make creation actions carry the selected owner where the publish route supports it, and add regression coverage for a user with both personal and org publishers.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a signed-in user with a personal publisher plus an org membership lands on `/dashboard`, the effect selects the personal publisher, and no rendered control calls `setSelectedPublisherId` to switch to the org.

## Solution Assessment

Yes. A dashboard owner selector is the narrow maintainable fix because the backend already accepts `ownerPublisherId` and enforces membership for org-owned skill/package dashboard reads.
