## Summary

Keep open: current main still matches the reported failure path, with backend org-owner reads available but the dashboard UI defaulting to the personal publisher and rendering no control to switch to an org publisher.

## What This Changes

The issue reports that org-owned packages visible in search and CLI inspect cannot be reached from `/dashboard` because the UI defaults to the personal publisher without a publisher switcher.

## Best Possible Solution

Restore a visible dashboard publisher selector backed by `api.publishers.listMine`, keep dashboard queries scoped to the selected publisher, and align dashboard publish actions with the selected owner handle.

## Reproduction Assessment

Yes, as a source-level path: a signed-in user with both a personal publisher and an org publisher loads `/dashboard`, the code auto-selects the personal publisher, and the render has no control to change `selectedPublisherId` to the org.

## Solution Assessment

Yes, restoring a visible publisher selector is the narrowest maintainable fix because the backend membership and owner-publisher dashboard queries already exist; URL or local-storage persistence can be a follow-up.
