## Summary

Keep open: current main still lacks a dashboard publisher switcher, so a user with both personal and org publisher access is auto-scoped to the personal publisher while org-owned packages remain unreachable from the dashboard UI.

## What This Changes

Expose organization-owned packages on `/dashboard` by letting signed-in publisher members switch the dashboard from their personal publisher to an org publisher.

## Best Possible Solution

Add a dashboard publisher selector backed by `api.publishers.listMine`, keep skill/package queries scoped to the selected publisher, and make dashboard publish actions preserve that selected owner handle.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: sign in as a user with a personal publisher plus an org publisher, load `/dashboard`, and current main auto-selects the personal publisher with no UI path to choose the org publisher.

## Solution Assessment

Yes. A visible publisher selector is the narrowest maintainable fix because the membership list and owner-publisher dashboard queries already exist; grouped sections or persistence can be follow-ups.
