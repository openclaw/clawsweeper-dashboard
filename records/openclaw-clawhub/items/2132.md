## Summary

Keep open: current main still auto-selects the personal publisher on the dashboard and exposes no rendered control to switch to an org publisher, while the backend and orgs spec already support org-scoped dashboard reads.

## What This Changes

Expose organization publisher-owned packages and skills on the dashboard by letting signed-in members switch the dashboard owner context.

## Best Possible Solution

Add a focused dashboard publisher selector backed by listMine, keep the personal default, scope both lists to the selected publisher, and pass that selected owner into publish actions with regression coverage for a personal-plus-org account.

## Reproduction Assessment

Yes. A high-confidence source path exists: sign in as a user with both personal and org publishers, load /dashboard, and current code auto-selects the personal publisher without rendering a way to set the org publisher id.

## Solution Assessment

Yes. A visible publisher selector is the narrowest maintainable fix because the dashboard queries already accept ownerPublisherId and the orgs spec already requires owner selection for dashboard quick actions.
