## Summary

Keep open: current main still has backend support for org-owned dashboard reads, but the dashboard UI auto-selects the personal publisher and no longer exposes a visible publisher switcher, so org packages remain unreachable from the page.

## What This Changes

The issue requests that `/dashboard` expose organization-owned packages for signed-in org publisher members after package transfer or publish.

## Best Possible Solution

Add a dashboard publisher selector backed by `api.publishers.listMine`, keep package and skill queries scoped to the selected publisher, align publish actions with the selected handle, and cover the personal-plus-org case in dashboard tests.

## Reproduction Assessment

Yes, as a source-level reproduction path: sign in as a user with both a personal publisher and an org publisher, load `/dashboard`, and the code selects the personal publisher while rendering no control that can switch `selectedPublisherId` to the org.

## Solution Assessment

Yes, reintroducing a visible dashboard publisher selector is the narrowest maintainable fix because the backend owner-publisher listing and membership checks already exist; grouped dashboard sections or persistence can be follow-ups.

## Repair Work Prompt

Fix the dashboard org publisher visibility bug. In current main, `/dashboard` uses `api.publishers.listMine` and `selectedPublisherId`, but after auto-selecting the personal publisher there is no visible control to switch to org publishers; org-owned packages are therefore hidden even though `api.packages.list({ ownerPublisherId })` and `api.skills.listDashboardPaginated({ ownerPublisherId })` support them. Add a visible publisher selector in `src/routes/dashboard.tsx` when the viewer has multiple publishers, preserve the personal default, and make selecting an org re-run skills/packages queries with that publisher id. Keep backend ownership/search behavior unchanged. Ensure empty states and publish actions reflect the selected `@handle`; preserve plugin `ownerHandle`, and only extend `src/routes/skills/publish.tsx` with `ownerHandle` search prefill if needed to keep New Skill aligned. Add regression coverage in `src/routes/-dashboard.test.tsx` with mocked personal-plus-org publishers and org package results. Do not change unrelated dashboard row action behavior.
