## Summary

Keep open. This is a valid current-main dashboard bug, and repository policy for `openclaw/clawhub` keeps issues open during cleanup rather than auto-closing them.

## What This Changes

The issue asks for the dashboard to expose organization-owned packages for publishers the signed-in user belongs to, likely through an owner selector or grouped publisher sections.

## Best Possible Solution

Add a dashboard publisher switcher backed by `api.publishers.listMine`, scope skills and packages to the selected publisher, carry that owner into creation actions where supported, and cover a personal-plus-org publisher case in tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a signed-in user with both a personal publisher and org membership lands on `/dashboard`, the effect selects the personal publisher, and no rendered control calls `setSelectedPublisherId` to switch to the org.

## Solution Assessment

Yes. A dashboard owner selector is the narrow maintainable fix because the backend already accepts `ownerPublisherId` and enforces membership for org-owned dashboard package and skill reads.
