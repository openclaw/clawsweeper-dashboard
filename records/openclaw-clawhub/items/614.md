## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still leaves the reported public profile server-error path plausible because `/u/$handle` calls `users.getByHandle`, which delegates to duplicate-sensitive `.unique()` handle lookups on ordinary user and publisher indexes.

## What This Changes

The issue reports that the public profile URL for handle `jisuapi` fails with a Convex `users:getByHandle` server error, likely due to an account or handle conflict.

## Best Possible Solution

Verify the production request ID and `jisuapi` identity rows across users, publishers, and auth-linked data, then either repair/merge the conflicting records or add deterministic, privacy-safe duplicate-tolerant handle resolution with regression tests.

## Reproduction Assessment
