## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still leaves the reported public profile server-error path plausible because `/u/$handle` calls `users.getByHandle`, which delegates into duplicate-sensitive `.unique()` handle lookups on ordinary indexes. Related #1179 is adjacent identity/publisher work, but it tracks first-time publish/profile-save failures rather than this public profile lookup failure.

## What This Changes

The issue reports that `https://clawhub.ai/u/jisuapi` returns a Convex `users:getByHandle` server error, likely from a GitHub/account handle conflict.
