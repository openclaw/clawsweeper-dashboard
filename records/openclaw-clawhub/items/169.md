## Summary

Keep open. ClawHub policy keeps issues open in this pass, and current main still has a source-confirmed false-positive path: downloaded skill zips can add root `_meta.json`, local update hashing includes root JSON files, and registry resolution compares against published-version file fingerprints that do not include that generated metadata.

## What This Changes

The issue reports that `clawhub update` falsely reports `local changes (no match)` for freshly installed or force-updated skills instead of recognizing clean installed content.

## Best Possible Solution

Define one canonical skill-content fingerprint contract shared by publish, download, install, update, sync, and resolve so generated download metadata cannot trigger false local-change warnings while real local edits still block non-forced updates.

## Reproduction Assessment

Yes, from source inspection. Install extracts all zip entries, the download zip can contain generated root `_meta.json`, update hashes root JSON files, and resolve compares against published-version files that omit that generated metadata.

## Solution Assessment

Yes. A focused canonical-fingerprint fix is the narrow maintainable path; using `--force`, rescan guidance, or a broad JSON exclusion would preserve or weaken the integrity workflow.
