## Summary

Keep open under the ClawHub issue policy: current main and the v0.15.0 tag already cover the clean-install false positive with stored install fingerprints and regression coverage, but this cleanup pass does not auto-close issues.

## What This Changes

The issue reports that `clawhub update` treats freshly installed skills as local changes because installed skill origin metadata lacked a clean fingerprint baseline.

## Best Possible Solution

Keep the v0.15.0 stored-fingerprint behavior and ask for a new current-CLI reproduction only if users still see the warning after reinstall or force update creates origin fingerprints.

## Reproduction Assessment

No. The original v0.5.0 reproduction is clear, but current main stores install fingerprints and has regression coverage asserting the warning is not printed when registry resolve returns no match.

## Solution Assessment

Yes. Storing the installed content fingerprint in origin metadata is the narrow maintainable fix because it preserves local-edit detection while giving CLI-installed bytes a clean baseline.
