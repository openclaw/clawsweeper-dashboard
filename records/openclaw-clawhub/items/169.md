## Summary

Keep open under the ClawHub issue policy for issues, but the reported fresh-install false positive appears fixed on current main and shipped in `v0.15.0` by storing an installed fingerprint and trusting that baseline when registry hash resolution has no match.

## What This Changes

The issue reports that `clawhub update` incorrectly warns `local changes (no match)` immediately after a clean skill install.

## Best Possible Solution

Keep the shipped stored-fingerprint baseline for clean installs and updates; if maintainers want this issue closed, do that after optional live smoke confirmation and track any older-install migration separately.

## Reproduction Assessment

No high-confidence way to reproduce the reported fresh-install failure against current main was established. Source inspection and regression coverage show fresh installs now persist a fingerprint and `cmdUpdate` trusts that same-slug baseline when registry resolve returns no match, but I did not run a live registry smoke test.

## Solution Assessment

Yes. Storing and refreshing the post-install fingerprint is a narrow maintainable fix for the clean-install false positive while preserving local-edit detection when installed files drift from the stored baseline.
