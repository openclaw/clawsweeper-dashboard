## Repair Work Prompt

Fix the ClawHub update fingerprint false positive reported in https://github.com/openclaw/clawhub/issues/169 and related reports. Expected behavior: a fresh install or force-update followed immediately by `clawhub update <slug>` or `clawhub update --all` should resolve to the installed version or report up to date, while real edits to authored skill files still block non-forced update. Define a canonical fingerprint contract shared by publish, download, install/update hashing, sync, and `/api/v1/resolve`; do not broadly ignore all JSON files or weaken authored-file integrity checks. Likely implementation areas are `packages/clawhub/src/skills.ts`, `packages/clawhub/src/cli/commands/skills.ts`, `packages/clawhub/src/cli/commands/sync.ts`, `convex/downloads.ts`, `convex/lib/skillZip.ts`, `convex/lib/skillPublish.ts`, and `convex/skills.ts`. Add regression coverage for generated root `_meta.json`, fresh install/update, force-update/update, and real local edit detection. Do not change unrelated skill publishing, moderation, or pinning semantics.
