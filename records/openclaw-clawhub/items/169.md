## What This Changes

The issue reports that `clawhub update` falsely flags freshly installed skills as locally changed because clean installs cannot be matched back to a registry fingerprint.

## Best Possible Solution

## Repair Work Prompt

Fix the false `clawhub update` local-change warning after a fresh install or force update. Current main downloads skill zips with generated root `_meta.json`, extracts it into the skill directory, hashes root JSON during update, and asks the registry to resolve a fingerprint that was built from a different file set. Define one canonical skill-content fingerprint contract across publish/download/install/update/sync/resolve. Prefer excluding only ClawHub-generated root `_meta.json` from local/update comparison or avoiding its extraction for skill installs; do not broadly ignore all JSON or user-authored metadata. Add regression coverage showing install or force-update followed immediately by normal update resolves as up to date, and that a real authored-file edit still blocks non-forced update. Coordinate with the related reports and add a user-facing changelog entry if the repo’s release process expects one for CLI bug fixes.
