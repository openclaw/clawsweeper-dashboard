## Summary

Keep open under the ClawHub review policy for issues, but current main and the shipped CLI now cover the reported fresh-install path by storing an install fingerprint and trusting it when registry resolution cannot match.

## What This Changes

The issue asks for `clawhub update` to stop reporting `local changes (no match)` immediately after a clean `clawhub install`.

## Best Possible Solution

Keep the shipped fingerprint fallback for fresh installs and force-updates; only add a narrow legacy-origin migration if maintainers want old no-fingerprint installs to self-heal without a one-time force update.

## Reproduction Assessment

No current-main reproduction path for the reported fresh-install bug. Source and regression coverage show fresh installs now store a fingerprint and update trusts that stored fingerprint when registry resolution returns no match.

## Solution Assessment

Yes. The current implementation is a narrow fix for the central symptom because it preserves registry-based matching while adding a local clean-install baseline instead of broadly weakening file hashing.
