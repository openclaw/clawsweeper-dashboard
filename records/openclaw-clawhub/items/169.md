## Repair Work Prompt

Fix the `clawhub update` false `local changes (no match)` warning for freshly installed or force-updated skills. Align the fingerprint contract across publish, download, install/update local hashing, and `/api/v1/resolve`: generated ClawHub root metadata such as download `_meta.json` must not make clean installed content differ from the published-version fingerprint, while authored JSON and other text files must still be covered by integrity checks. Add focused regression coverage for install or force-update followed by normal update resolving as up to date, and for real local edits still blocking non-forced update. Do not add a config flag, do not weaken authored-file hashing broadly, and coordinate with related reports https://github.com/openclaw/clawhub/issues/178 and https://github.com/openclaw/clawhub/issues/937.
