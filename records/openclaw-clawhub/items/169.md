## Summary

Keep open under the ClawHub issue policy: current main and the v0.15.0 changelog cover the reported clean-install false positive with stored install fingerprints and regression coverage, but this cleanup pass does not auto-close issues.

## What This Changes

The issue reports that `clawhub update` falsely reports `local changes (no match)` immediately after a clean skill install.

## Best Possible Solution

Keep the v0.15.0 stored-fingerprint behavior, and ask for a new current-CLI reproduction only if users still see the warning after reinstall or force-update creates origin fingerprints.

## Reproduction Assessment

No. The original v0.5.0 reproduction is clear, but current main records an install fingerprint and trusts it when registry resolve returns no match, with a regression test asserting the warning is not printed.

## Solution Assessment

Yes. Storing the installed content fingerprint in origin metadata is a narrow maintainable fix because it preserves authored-file local-edit detection while giving CLI-installed bytes a clean baseline.
