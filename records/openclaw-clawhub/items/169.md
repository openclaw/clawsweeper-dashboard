## Summary

Keep open. Repository cleanup policy keeps ClawHub issues open, and current main still has a source-reproducible clean-install/update fingerprint mismatch path involving generated download metadata, local file hashing, and registry-side published-file fingerprints.

## What This Changes

Fix `clawhub update` so freshly installed or force-updated skills match registry fingerprints instead of reporting `local changes (no match)`.

## Best Possible Solution

Define one canonical fingerprint contract shared by publish, download, install, update, sync, and resolve so ClawHub-generated install/download metadata does not create false positives while real local edits still block non-forced updates.

## Reproduction Assessment

Yes. Source inspection shows downloaded skill zips can install generated root `_meta.json`, local update hashing includes root JSON files, and registry resolve compares against published files that do not include that generated metadata.

## Solution Assessment

Yes. The narrow maintainable fix is to align the compared fingerprint inputs, not to add a user-facing flag or broadly weaken local-change detection.
