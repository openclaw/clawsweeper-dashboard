## What This Changes

The issue reports `clawhub update` falsely reporting `local changes (no match)` immediately after a clean install or force update.

## Best Possible Solution

Define one canonical skill-content fingerprint contract shared by publish, download, install, update, sync, and resolve, with regression coverage for both clean installs and real local edits.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: update hashes installed files including root JSON, downloaded archives can add root `_meta.json`, and resolve compares against published-version file fingerprints.

## Solution Assessment

Yes. The narrow maintainable solution is to align the compared file sets; relying on `--force` or broadly ignoring JSON would weaken the local-change guard.
