## Summary

Keep open under the ClawHub issue policy, but current main and the v0.15.0 tag appear to cover the reported clean-install false positive: installs and updates now store a local content fingerprint, and update can use that stored baseline when registry hash resolution returns no match.

## What This Changes

The issue reports `clawhub update` falsely reporting `local changes (no match)` immediately after a fresh skill install.

## Best Possible Solution

Keep the current stored-fingerprint fallback and have maintainers close the issue manually once they are satisfied that the released CLI path covers fresh installs and the older-install seeding note is acceptable.

## Reproduction Assessment

No high-confidence current-main reproduction remains from source inspection: current main writes a stored origin fingerprint and `cmdUpdate` can match that baseline when registry resolve returns null. I did not execute the live CLI install/update path in this read-only review.

## Solution Assessment

Yes. Recording the post-extract installed fingerprint is a narrow fix for the reported clean-install false positive, while preserving local-edit detection for authored files instead of broadly ignoring JSON or generated-looking files.
