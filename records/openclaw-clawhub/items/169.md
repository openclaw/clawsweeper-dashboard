## Repair Work Prompt

Fix the false `local changes (no match)` result after a clean `clawhub install` or `clawhub update --force`. Define a canonical skill-content fingerprint contract shared by CLI local hashing, zip hashing, publish fingerprinting, download metadata, `resolveVersionByHash`, sync, and maintenance backfills. Generated download metadata such as root `_meta.json` must not cause a clean install to fail resolve, while real edits to `SKILL.md`, config JSON, or other published text files must still be detected. Add focused regression tests showing a downloaded/installed skill immediately resolves to its published version and that a real local edit still blocks non-forced update. Do not weaken tamper detection by ignoring all JSON or arbitrary generated-looking files. Related reports: #178 and #937.
