## Summary

Keep open. ClawHub cleanup policy does not auto-close issues, and current main still has a source-confirmed false-positive path where clean installed content can hash differently from the backend version fingerprint used by update resolution.

## What This Changes

The issue asks for `clawhub update` to recognize freshly installed skills as clean instead of reporting `local changes (no match)`.

## Best Possible Solution

Define one canonical skill-content fingerprint contract shared by publish, download, install, update, sync, and resolve so clean installs resolve while real local edits still block non-forced updates.

## Reproduction Assessment

Yes, from source inspection. A downloaded skill can include generated root `_meta.json`, local update hashing includes root JSON files, and registry resolve compares against fingerprints derived from published version files that omit that metadata.

## Solution Assessment

Yes. A focused fingerprint-contract fix is the narrow maintainable path; relying on `--force` or broadly excluding JSON would preserve or weaken the integrity workflow.
