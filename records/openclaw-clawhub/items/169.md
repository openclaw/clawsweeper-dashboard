## Summary

Keep open under the ClawHub issue policy, but current main and the shipped v0.15.0+ CLI path appear to cover the reported clean-install false positive with stored install fingerprints and regression coverage.

## What This Changes

The issue reports that `clawhub update` treats freshly installed skills as locally modified because update cannot match the installed files to a clean baseline.

## Best Possible Solution

Keep the shipped stored-fingerprint fallback, and let maintainers manually smoke-test or close this issue once they are satisfied with the legacy-install baseline behavior.

## Reproduction Assessment

No, not against current main. Source inspection shows fresh installs and updates now persist an installed fingerprint and `cmdUpdate` trusts that same-slug baseline when registry resolve returns no match.

## Solution Assessment

Yes. Recording and trusting the post-install fingerprint for the same slug is a narrow fix for the clean-install false positive while preserving local-change detection when files drift from the stored baseline.
