## Repair Work Prompt

Fix the ClawHub CLI/backend fingerprint mismatch where `clawhub update` reports `local changes (no match)` immediately after install or force-update. In current main, downloads add root `_meta.json`, extraction writes it into the installed skill folder, local hashing includes root JSON, and backend resolve compares against published `version.files` fingerprints. Define one canonical published-content fingerprint contract shared by CLI local hashing/zip hashing and backend publish/resolve/download/sync. Prefer excluding only ClawHub-generated root `_meta.json` from local/install fingerprints or avoiding writing it into installed skill content; do not ignore all JSON or user-authored metadata. Add regression coverage for a downloaded/generated `_meta.json` case that resolves as up to date, and for a real authored JSON/local edit still being detected. Update specs/docs if the fingerprint contract changes, and avoid unrelated CLI behavior changes.
