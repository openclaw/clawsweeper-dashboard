## Summary

Keep open. Repository policy keeps ClawHub issues open, and current main still has a source-confirmed false-positive path for clean installs: skill downloads add generated root `_meta.json`, installs extract it, local update hashing includes root JSON files, and registry resolve compares against fingerprints built from published version files that do not include that generated metadata.

## What This Changes

Fix `clawhub update` so freshly installed or force-updated skills are recognized as clean instead of reporting `local changes (no match)`.

## Best Possible Solution

Align the canonical fingerprint contract across publish, download, install, update, sync, and resolve so generated ClawHub metadata cannot create false local-change warnings while authored file edits still block non-forced updates.

## Reproduction Assessment

Yes, from source inspection. A downloaded skill can include generated root `_meta.json`, the installer extracts it, local update hashing includes root JSON files, and registry resolve compares against published-version fingerprints that omit that generated file.

## Solution Assessment

Yes. The narrow maintainable fix is to define and test one shared fingerprint input set; using `--force`, adding per-skill ignore files manually, or storing a separate origin hash alone would not reliably preserve the intended local-edit check.
