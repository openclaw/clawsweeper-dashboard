## Summary

Keep open. ClawHub cleanup policy does not auto-close issues, and current main still has a source-confirmed clean-install/update mismatch path: generated root `_meta.json` can be installed and hashed locally, while registry resolve compares against published-version fingerprints that omit that generated file.

## What This Changes

The issue reports that `clawhub update` falsely reports `local changes (no match)` for freshly installed skills instead of recognizing the clean installed version.

## Best Possible Solution

Land one focused CLI/backend integrity fix that aligns local install/update hashing, downloaded zip contents, and registry resolve fingerprints on the same canonical published file set while preserving detection of real authored-file edits.

## Reproduction Assessment

Yes, by source inspection rather than a live run. Downloaded skill zips can include generated root `_meta.json`, local hashing includes root JSON files, and registry resolve compares against published file fingerprints that do not include that generated metadata.

## Solution Assessment

Yes, a focused fingerprint-contract fix is the narrow maintainable path. Adding only an origin hash or ignoring all JSON would either miss the current `_meta.json` path or weaken integrity checks.
