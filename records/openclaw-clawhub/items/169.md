## Summary

Keep open under the ClawHub issue policy: current main and CLI v0.15.0 appear to address the reported fresh-install false positive, but this cleanup pass only auto-closes eligible PRs, not issues.

## What This Changes

The issue reports that `clawhub update` falsely flags freshly installed skills as locally modified because install metadata lacked a clean-content fingerprint baseline.

## Best Possible Solution

Keep the v0.15.0 stored-origin fingerprint behavior and ask for a new current-CLI reproduction only if users still see the warning after reinstall or force-update creates origin fingerprints.

## Reproduction Assessment

No. The original v0.5.0 steps were concrete, but current main writes and trusts an installed fingerprint for freshly installed or force-updated content, and regression coverage asserts the warning is not printed when registry resolution cannot match that fingerprint.

## Solution Assessment

Yes. Storing the installed content fingerprint in origin metadata is the narrow maintainable solution because it preserves real local-edit detection while giving CLI-installed bytes a clean baseline.
