## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has a credible clean-install fingerprint mismatch path between CLI-local hashing, downloaded archive contents, and server-side version resolution.

## What This Changes

The issue reports `clawhub update` rejecting freshly installed or force-updated skills as `local changes (no match)` because clean local fingerprints cannot be resolved by the registry.

## Best Possible Solution

Define one canonical skill-content fingerprint contract shared by publish, download, install, update, sync, and resolve so fresh installs resolve cleanly while real local edits still block non-forced updates.

## Reproduction Assessment

Yes. The issue body gives concrete install-then-update steps, and current source shows a source-level reproduction path where a downloaded generated `_meta.json` can be hashed locally but absent from publish-time resolve inputs. I did not run the CLI because this review is read-only.

## Solution Assessment

No proposed patch is attached. The suspected missing origin hash explains part of the symptom, but the narrow maintainable fix is to align the fingerprint contract rather than bypass resolve or broadly ignore file classes.
