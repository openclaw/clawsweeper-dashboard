## Summary

Keep open. Current main still has a source-confirmed clean-install fingerprint mismatch path, and ClawHub cleanup policy does not auto-close issues even when they look implemented or related reports were closed.

## What This Changes

The issue reports that `clawhub update` treats freshly installed skills as locally modified because clean install fingerprints are not recognized.

## Best Possible Solution

Define one canonical skill-content fingerprint contract shared by publish, download, install, update, sync, and resolve so fresh installs resolve cleanly while real local edits still block non-forced updates.

## Reproduction Assessment

Yes, from source inspection. A downloaded skill can contain generated root `_meta.json`, local update hashing includes root JSON files, and registry resolve compares against fingerprints built from published version files that omit that generated metadata.

## Solution Assessment

Yes. A focused fingerprint-contract fix is the narrow maintainable solution; relying on `--force` or excluding all JSON would either leave the warning unusable or weaken integrity checks.

Fix the ClawHub CLI/backend fingerprint mismatch that makes `clawhub update` report `local changes (no match)` immediately after a fresh install or force-update. Treat this issue as the canonical open report and preserve context from https://github.com/openclaw/clawhub/issues/178 and https://github.com/openclaw/clawhub/issues/937. Align the canonical fingerprint file set used by publish, download/install, local update hashing, sync, and registry resolve so generated ClawHub download metadata such as root `_meta.json` does not create false local-change warnings, while authored JSON and other real skill text files are still covered by integrity checks. Avoid a broad ignore of all JSON or all generated-looking files. Add focused regression coverage for a downloaded/archive-installed skill containing root `_meta.json` resolving as the installed version, plus a real local edit still blocking non-forced update.
