## Summary

Keep open. ClawHub cleanup policy does not auto-close issues, and current main still has a source-confirmed clean-install fingerprint mismatch path involving generated download metadata and registry-side published-file fingerprints.

## What This Changes

The issue reports that `clawhub update` falsely reports `local changes (no match)` immediately after a clean install or forced update.

## Best Possible Solution

Define one canonical skill-content fingerprint contract shared by publish, download, install, update, sync, and resolve so clean installs match and real local edits still block non-forced updates.

## Reproduction Assessment

Yes, from source inspection. A downloaded skill can include generated root `_meta.json`, local update hashing includes root JSON files, and registry resolve compares against fingerprints derived from published version files that omit that generated metadata.

## Solution Assessment

Yes. A focused fingerprint-contract fix is the narrow maintainable path; relying on `--force` or broadly excluding authored files would preserve or weaken the integrity workflow.
