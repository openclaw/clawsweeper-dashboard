## Summary

Keep open. ClawHub cleanup policy does not auto-close issues in this pass, and current main still has a source-confirmed clean-install/update fingerprint mismatch path involving generated download metadata and registry-side published-file fingerprints.

## What This Changes

The issue asks for `clawhub update` to treat freshly installed or force-updated skills as clean instead of reporting `local changes (no match)`.

## Best Possible Solution

Define one canonical skill-content fingerprint contract shared by publish, download, install, update, sync, and resolve so clean installs match while real local edits still block non-forced updates.

## Reproduction Assessment

Yes, from source inspection. Downloaded skills can include generated root `_meta.json`, local update hashing includes root JSON files, and registry resolve compares against published version files that do not include that generated metadata.

## Solution Assessment

Yes. The narrow maintainable fix is to align the compared fingerprint inputs, either by excluding only ClawHub-generated install/download metadata from local hashing or by avoiding that metadata in installed skill content, while keeping authored JSON covered.
