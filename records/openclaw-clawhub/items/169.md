## Summary

Keep open. This pass does not auto-close ClawHub issues, and current main still has a source-confirmed false-positive path where downloaded skill archives add root `_meta.json`, local update hashing includes root JSON files, and registry resolution compares against published file fingerprints that do not include that generated metadata.

## What This Changes

The issue reports that `clawhub update` falsely reports `local changes (no match)` immediately after a clean install or force update.

## Best Possible Solution

Define one canonical skill-content fingerprint contract shared by publish, download, install, update, sync, and resolve so fresh installs resolve cleanly while real local edits still block non-forced updates.

## Reproduction Assessment

Yes, from source inspection. A downloaded skill can include generated root `_meta.json`, local update hashing includes root JSON files, and registry resolve compares against fingerprints derived from published version files that omit that generated metadata.

## Solution Assessment

Yes. A focused fingerprint-contract fix is the narrow maintainable path; the current origin-metadata regression coverage does not address the root `_meta.json` mismatch, and relying on `--force` would preserve the broken integrity workflow.
