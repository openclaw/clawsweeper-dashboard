## What This Changes

The issue reports that `clawhub update` falsely reports `local changes (no match)` immediately after a clean skill install.

## Best Possible Solution

Keep the v0.15.0 stored-fingerprint behavior, and ask for a fresh current-CLI reproduction only if users still see the warning after reinstall or force-update creates origin fingerprints.

## Reproduction Assessment

No. The original v0.5.0 steps were concrete, but current main records an install fingerprint and trusts it when registry resolve returns no match; I did not run a live registry install/update in this read-only review.

## Solution Assessment

Yes. Storing the installed content fingerprint in origin metadata is the narrow maintainable fix because it preserves real local-edit detection while giving CLI-installed bytes a clean baseline.
