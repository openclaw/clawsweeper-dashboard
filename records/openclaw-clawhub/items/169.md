## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has a source-level clean-install mismatch path: the CLI hashes installed text files, downloaded skill zips can add root `_meta.json`, and registry resolve compares against publish-time version files.

## What This Changes

The issue asks for `clawhub update` to recognize freshly installed skills as clean instead of reporting `local changes (no match)`.

## Best Possible Solution

Make fresh installs, force-updates, publish fingerprints, download zips, sync, and resolve use the same canonical skill-content fingerprint while preserving local tamper detection.

## Reproduction Assessment

Yes, there is a high-confidence source-level path: install extracts a zip that can contain `_meta.json`, `update` hashes that JSON locally, and resolve compares against publish-time files. I did not execute the CLI because this review was read-only.

## Solution Assessment

Yes, the narrow maintainable fix is to share a canonical fingerprint contract across publish/download/install/update/resolve. Storing an origin hash alone may help, but it should not bypass server-side content matching or weaken local-change detection.
