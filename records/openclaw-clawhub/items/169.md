## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has a source-level false-positive path where clean installed/downloaded skill content can fail registry fingerprint resolution because local update hashing can include generated root metadata that publish/resolve fingerprints do not include.

## What This Changes

The issue reports that `clawhub update` falsely reports `local changes (no match)` immediately after a clean skill install or force update.

## Best Possible Solution

Define one canonical skill-content fingerprint contract shared by publish, download, install, update, sync, and resolve, so clean installs resolve while real local edits still block non-forced updates.

## Reproduction Assessment

Yes, from source inspection. The update command hashes installed files and refuses when `/api/v1/resolve` has no match, while installed downloads can contain generated root `_meta.json` outside the published-version fingerprint set.

## Solution Assessment

Yes. The narrow maintainable fix is to align the fingerprint contract across client and server while preserving detection of real local edits, not to bypass matching or ignore broad classes of files.
