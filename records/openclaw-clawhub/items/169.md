## Summary

Keep open under the repository rule for ClawHub issues, but current main no longer shows the clean-install false-positive path in source: installs and updates now persist a local content fingerprint, and update trusts that stored fingerprint when registry resolve cannot match it.

## What This Changes

The issue asks for `clawhub update` to stop reporting `local changes (no match)` immediately after a clean skill install or force-update.

## Best Possible Solution

Keep the current stored-fingerprint fallback and let a maintainer manually close this issue after confirming the released CLI behavior against a real install/update path.

## Reproduction Assessment

No high-confidence current-main reproduction remains from source inspection: fresh installs now write an origin fingerprint and update can match it even when registry resolve returns null. I did not run the live CLI path in this read-only review.

## Solution Assessment

Yes for the reported clean-install false positive: recording the installed fingerprint preserves authored JSON coverage and avoids broadly ignoring generated metadata. A stronger server-side fingerprint contract could still be designed later, but it is not required to address this report.
