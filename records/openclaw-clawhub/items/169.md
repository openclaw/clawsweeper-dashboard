## What This Changes

The issue asks for `clawhub update` to stop reporting `local changes (no match)` immediately after a fresh clean install.

## Best Possible Solution

Keep the v0.15.0 stored-fingerprint behavior; track any legacy-origin migration or separate sync-reporting leftovers in narrower follow-ups.

## Reproduction Assessment

No. Against current main, a fresh install records an origin fingerprint and update can trust that fingerprint when resolve returns no match, so the reported v0.5.0 clean-install path no longer reaches the false warning.

## Solution Assessment

Yes. The stored-origin fingerprint fallback is the narrow maintainable fix for this issue because it preserves real local-edit detection while giving bytes installed by the CLI a clean baseline.
