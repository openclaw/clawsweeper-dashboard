## Summary

Keep open under the ClawHub issue policy for issues, but the reported fresh-install false positive appears fixed on current main and shipped in v0.15.0 via stored install fingerprints plus regression coverage.

## What This Changes

The issue reports that `clawhub update` falsely treats freshly installed skills as locally modified because install metadata lacked a clean-content fingerprint baseline.

## Best Possible Solution

Keep the shipped stored-fingerprint baseline for clean installs and updates; any stricter registry-side fingerprint contract should be handled separately without weakening authored-file integrity checks.

## Reproduction Assessment

No, not against current main. Source inspection and regression coverage show fresh installs now persist a fingerprint and `cmdUpdate` trusts that same-slug baseline when registry resolve returns no match.

## Solution Assessment

Yes. Storing and refreshing the post-install fingerprint is a narrow maintainable fix for the clean-install false positive while preserving local-edit detection when installed files drift from the stored baseline.
