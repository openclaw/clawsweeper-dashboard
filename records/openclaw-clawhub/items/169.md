## Summary

Keep open. Repository policy keeps ClawHub issues open in this cleanup pass, and current main still has a source-confirmed clean-install/update fingerprint mismatch path involving generated download metadata and registry-side published-file fingerprints.

## What This Changes

The issue reports that `clawhub update` flags freshly installed skills as `local changes (no match)` instead of recognizing their installed version.

## Best Possible Solution

Define one canonical skill-content fingerprint contract shared by publish, download, install, update, sync, and resolve so clean installs and force-updates resolve to their installed version while real local edits still block non-forced updates.

## Reproduction Assessment

Yes, from source inspection. A downloaded skill can include generated root `_meta.json`, the local update hash includes root JSON files, and registry resolve compares against published version files that do not include that generated metadata.

## Solution Assessment

Yes. The narrow maintainable fix is to align the compared fingerprint inputs, either by excluding only ClawHub-generated install/download metadata from local hashing or by avoiding that metadata in installed skill content, while keeping authored JSON covered.
