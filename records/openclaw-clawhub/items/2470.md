## Best Possible Solution

Land the storage-derived metadata hardening after trusted `_storage` metadata is checked before blob downloads, with regression coverage for forged metadata and over-total early rejection.

## Reproduction Assessment

## Solution Assessment

No as written. Deriving metadata from storage is the right direction, but the fix should read `_storage` metadata through Convex before blob downloads and only fetch blobs after trusted size checks pass.
