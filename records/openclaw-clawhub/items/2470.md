## Best Possible Solution

Land the storage-derived metadata hardening after trusted `_storage` metadata is checked before blob downloads, with regression coverage for forged metadata and early per-file/total rejection.

## Reproduction Assessment

## Solution Assessment

No as written. Deriving metadata from storage is the right direction, but the maintainable fix should read `_storage` metadata first and only fetch blobs after trusted size checks pass.
