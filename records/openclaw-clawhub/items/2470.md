## Summary

This PR addresses a real publish-integrity defect on current main, but it is not merge-ready because typeless stored blobs still fall back to caller-controlled MIME metadata, the security-sensitive invariant is not documented, and no real Convex storage proof covers representative or near-limit publishes. ## What This Changes

The PR derives published skill file size, content type, and SHA-256 from stored blobs before validation and adds mocked regressions for forged caller metadata and oversized stored files.

## Best Possible Solution

Make storage-owned metadata or deterministic path classification authoritative for admission, derive hashes and sizes from stored bytes, record the invariant in a focused spec, and prove ordinary plus near-limit publishing in a real Convex deployment.

No, not yet. Deriving metadata from stored bytes is the correct narrow direction, but the caller MIME fallback leaves the trust boundary incomplete and the storage-dependent runtime behavior still needs real validation.
