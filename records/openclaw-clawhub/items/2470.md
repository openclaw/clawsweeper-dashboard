## What This Changes

The branch changes `publishVersionForUser` to load publish blobs from storage, derive size, content type, and SHA-256 from the stored data, enforce limits on stored bytes, and add regression tests for mismatched caller metadata.

## Best Possible Solution

Land the central publish-layer storage-source-of-truth fix after normal maintainer approval and required checks, preserving the existing publish payload shape for callers.

## Reproduction Assessment

Yes, from source inspection: current main enforces text and size limits from `args.files` before reading storage, so a caller can provide metadata that differs from stored bytes. I did not run a live Convex reproduction in this read-only review.

## Solution Assessment

Yes. Centralizing the derivation in `publishVersionForUser` is the narrow maintainable fix because UI, CLI, HTTP, import, and restore paths converge there.
