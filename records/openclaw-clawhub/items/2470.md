## What This Changes

The PR changes skill publishing to derive file size, content type, and sha256 from Convex storage and adds regression tests for forged caller metadata.

No as written. Deriving metadata from storage is the right direction, but size enforcement should use trusted storage metadata before blob downloads.

- **[P1] Enforce size limits before reading blobs:** `convex/lib/skillPublish.ts:519`
  - body: `publishVersion` still accepts a caller-supplied array of storage IDs. This helper fetches every blob before checking `MAX_PUBLISH_TOTAL_BYTES`, so a caller can submit many valid storage IDs with bogus small metadata and make the action pull more than the 50MB publish limit before rejection.
