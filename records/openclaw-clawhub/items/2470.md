## Best Possible Solution

Land the storage-derived metadata hardening after trusted _storage metadata is checked before blob downloads and the publish path is validated against a real Convex runtime.

## Solution Assessment

No as written. Deriving metadata from storage is the right direction, but the maintainable fix should read trusted _storage metadata before blob downloads and only fetch blobs after size limits pass.
