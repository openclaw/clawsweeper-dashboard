## What This Changes

The issue asks what the public skill-page icons mean and why the reporter's `yescan-scan-universal` skill was not found by searching `yescan`.

## Best Possible Solution

A maintainer should answer with the current guidance: download icons mean downloads, package/box icons mean versions/packages, `yescan` literal recall is now handled, and future invisible-skill reports should check moderation and metadata using the documented search guidance.

## Reproduction Assessment

## Solution Assessment

Yes. The current implementation is the narrow maintainable solution: keep icon semantics distinct in UI components and use indexed digest prefix candidates for literal slug/name recall rather than adding another search mode.
