## Repair Work Prompt

Implement deterministic slug-prefix recall for ClawHub skill search. Scope this PR to backend search recall only; do not bundle UI icon changes, which are tracked by #1534/#1854. Current main only injects exact slug matches in `convex/search.ts`, and `lexicalFallbackSkills` scans recent `skillSearchDigest` rows before prefix-filtering candidates. Add a lightweight indexed candidate path that can return public, non-soft-deleted slugs such as `yescan-scan-universal` for short queries like `yescan` without relying on vector ranking or bounded recent scans. Preserve `highlightedOnly`, `nonSuspiciousOnly`, `capabilityTag`, owner hydration, duplicate merging, and existing scoring semantics. Follow Convex bandwidth rules: avoid `.filter().collect()`, full skill-document scans, and cron-style whole-table scans. If a schema/index or lookup table is added, include the required generated Convex changes and a cursor-based backfill/deploy-safe path. Add focused tests covering slug-prefix recall outside the recent fallback window, exact slug behavior, and interaction with existing token prefix matching.
