## What This Changes

The issue reports that CLI browser auth and fresh web-created API tokens return `Unauthorized` instead of verifying and storing a ClawHub CLI token.

## Best Possible Solution

Confirm production OAuth and API-token paths with fresh browser-created and settings-created tokens, inspect `users.ensure` plus affected account/publisher/token records, then land one focused fix or consolidate the cluster under a canonical maintainer-owned tracker.

## Solution Assessment

Unclear for implementation until production auth and account state are checked. The older `www.clawhub.ai` workaround is obsolete, so the safer path is maintainer-led live verification before changing registry defaults or token verification logic.
