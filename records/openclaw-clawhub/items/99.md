## Summary

Keep open. Repository policy keeps ClawHub issues open, and current main plus live routing checks do not prove this valid-token browser login failure is fixed; the old `www.clawhub.ai` workaround is now contradicted by production redirects and related Unauthorized reports remain open.

## What This Changes

The issue reports that browser auth completes but the CLI receives `Unauthorized` instead of verifying and storing the returned token.

## Best Possible Solution

Maintainers should verify fresh browser-created and Settings-created tokens in production against the current CLI and canonical apex host, then land one focused fix at the proven failing layer with regression coverage and consolidate the related Unauthorized reports.

## Reproduction Assessment

No. Source inspection traces the reported symptom through browser callback token delivery into `/api/v1/whoami`, but reproducing the valid-token `Unauthorized` failure requires a real production account/token that was not available in this read-only review.

## Solution Assessment

Unclear. The historical `https://www.clawhub.ai` workaround is no longer the safest fix because current source, live discovery, and production redirects use the apex host; the maintainable fix should follow real-token production verification.
