## Summary

Keep open. The repository policy for this ClawHub cleanup pass keeps issues open unless the item is a PR that is certainly implemented on main, and current main does not prove the reported login/token-verification failure is fixed: browser auth still hands the CLI a token and registry, the CLI still verifies `/api/v1/whoami` before persisting credentials, auth failures still surface as plain `Unauthorized`, and the related redirect-focused PR #101 was closed unmerged while related Unauthorized reports remain open.

## What This Changes

The issue reports that `clawhub login` completes browser authentication but the CLI then fails token verification with `Unauthorized` instead of storing usable credentials.

## Best Possible Solution

Confirm the production auth path using fresh browser-created and settings-created CLI tokens, decide the canonical `clawhub.ai` versus `www.clawhub.ai` registry behavior, make the CLI verify and persist that registry consistently, add focused regression coverage for host/token behavior, and consolidate related Unauthorized reports under one maintainer-owned tracker or fix.
