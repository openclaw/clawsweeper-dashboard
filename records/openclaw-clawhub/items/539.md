## Summary

Keep open. Current main has meaningful mitigations for authenticated users and better 429 retry handling, but anonymous/shared-IP search, install, and download traffic still uses IP buckets and related reports remain open. The ClawHub review policy also says issues should stay open for maintainer handling; only PRs that are certainly implemented on main are eligible for auto-close.

## What This Changes

The issue requests changes to ClawHub registry rate limiting so VPS/shared-IP users can search and install without persistent 429s, including elevated authenticated limits, better caching/offline options, and mirror/proxy guidance.

## Best Possible Solution

Define one explicit shared-IP registry strategy: verify production client-IP attribution, keep authenticated requests on per-user buckets, decide whether anonymous search/install/download should get higher limits or be guided into login by default, and document any mirror/offline registry path separately with matching backend, CLI, and docs coverage.
