## Summary

Keep open. Repository policy for this sweep keeps ClawHub issues open, and the report still has a substantive core gap: current main parses and stores normalized `metadata.openclaw`, but the static scanner still reads env declarations from shallow inputs and the public v1 skill metadata response still omits env/install/dependency declarations.

## What This Changes

The issue requests that ClawHub publish, registry metadata, and security scanning support env vars and install specs declared in `SKILL.md` `metadata.openclaw` so legitimate skills are not flagged suspicious for registry metadata mismatches.

## Best Possible Solution

Define a supported normalized runtime metadata contract for OpenClaw skill metadata, then route that same normalized shape through static scanning, LLM prompt inputs, public API responses, UI/setup prompts, docs, and tests for `requires.env`, `envVars`, install specs, and dependency declarations.
