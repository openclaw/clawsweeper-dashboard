## Summary

Keep open: the desktop height-cap removal is useful and current main still has the capped nested scroll, but the proposed mobile preview clips focusable file buttons behind the See all overlay and needs a narrow accessibility fix before merge.

## What This Changes

The PR removes the desktop Files tab list height cap, adds a mobile See all preview for long file lists, restyles the file list and preview panel, and updates tests and changelog.

## Best Possible Solution

Keep the desktop list uncapped, but make the collapsed mobile preview expose only focusable rows that are actually visible before the See all control.

## Reproduction Assessment

Yes. Current main source shows the capped Files list, and the PR diff plus inspected mobile screenshot give a high-confidence path for the introduced clipped-focus issue on narrow screens with many files.

## Solution Assessment

No. The desktop direction is maintainable, but the mobile collapse should avoid rendering tabbable rows that are hidden by `overflow: hidden` before the See all CTA.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep clipped file rows out of the tab order:** `src/styles.css:8599-8601`
  - body: The collapsed mobile list still renders 8 file buttons, but this CSS clips the body at 360px; the attached mobile screenshot shows only about 4 rows visible under the overlay. Because `overflow: hidden` does not remove the remaining buttons from the tab order, keyboard users can focus invisible file rows before reaching See all. Render only the visible preview rows or make clipped rows inert/hidden until expansion.
