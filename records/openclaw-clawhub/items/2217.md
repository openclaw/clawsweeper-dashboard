## Summary

Keep this PR open: the desktop height-cap fix is useful and well scoped, but the mobile preview currently clips rendered file buttons, which can leave hidden rows focusable before the user expands the list.

## What This Changes

The PR removes the desktop Files tab height cap, adds a mobile “See all” affordance, and polishes the Files panel layout, empty state, metadata, tests, and changelog.

## Best Possible Solution

Keep the desktop list uncapped, but adjust the mobile preview so every rendered file row is visibly reachable until See all expands the list.

## Reproduction Assessment

Yes. Current main’s desktop cap is visible in CSS, and the PR’s mobile clipping issue is source-reproducible from the 8-row render plus `overflow: hidden` rule; the submitted mobile screenshot also shows the clipped preview.

## Solution Assessment

No. Removing the desktop cap is the right direction, but the mobile implementation should not clip rendered, focusable rows; render only the visible preview rows or remove the clipping and rely on the explicit expansion state.
