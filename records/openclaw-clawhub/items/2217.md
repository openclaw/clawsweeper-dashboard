Keep open: current PR head still has a merge-blocking breakpoint mismatch that hides files at tablet widths between 481px and 899px.

## What This Changes

The PR removes the desktop Files tab height cap, adds mobile file-list truncation with a See all affordance, polishes file viewer styling, adds focused tests, and updates the changelog.

## Best Possible Solution

Align the component and CSS breakpoints so every truncated viewport has a visible, tested expansion control while desktop remains uncapped.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: open the PR between 481px and 899px wide with more than eight files, where the component renders only eight entries but the See all overlay remains hidden by CSS.

## Solution Assessment

No. The product direction is reasonable, but the implementation should first make the JS mobile cutoff and CSS media query agree and add mid-width regression coverage.
