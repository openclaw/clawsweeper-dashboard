## Summary

Keep this PR open: current main still has the desktop file-list height cap, and the PR’s latest head still has a source-visible breakpoint bug that can hide files from 481px through 899px.

## What This Changes

The PR removes the desktop Files tab height cap, adds mobile file-list truncation with a See all affordance, polishes file viewer layout, updates a focused test, and adds a changelog entry.

## Best Possible Solution

Align the JS and CSS mobile breakpoints so every truncated viewport has a visible expansion control while desktop remains uncapped.

## Reproduction Assessment

## Solution Assessment

No. The PR direction is reasonable, but the implementation should first make the component cutoff and CSS media query agree and add mid-width regression coverage.
