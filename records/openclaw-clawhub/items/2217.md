## Summary

Keep open: the PR addresses a real Files tab usability problem that current main still has, but the current head still hides files at tablet/narrow desktop widths because the JavaScript truncation and CSS affordance breakpoints do not match.

## What This Changes

The PR removes the desktop file-list height cap, adds mobile file-list truncation with a See all overlay, polishes the file preview/list UI, updates focused component tests, and adds a changelog entry.

## Best Possible Solution

Make the truncation and expansion-affordance breakpoints identical, preferably max-width 899 to match the desktop grid cutoff, and keep a regression covering a greater-than-eight-file mid-width viewport.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: view PR head at 481px-899px with more than eight files, where React slices the list but the CSS keeps the See all overlay hidden.

## Solution Assessment

No for the current patch. The UX direction is narrow and maintainable, but the implementation must first make the JS mobile cutoff and CSS media query agree and add a mid-width regression check.
