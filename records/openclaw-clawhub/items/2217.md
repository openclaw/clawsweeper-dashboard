## Summary

Keep open: current main still has the desktop file-list cap, and the PR head still has a merge-blocking breakpoint mismatch where tablet-width viewports truncate the file list but keep the See all overlay hidden.

## What This Changes

The PR removes the desktop Files tab height cap, adds mobile file-list truncation with a See all control, polishes file viewer styling, updates a component test, and adds a changelog entry.

## Best Possible Solution

Align the truncation and overlay breakpoints so every truncated viewport has a visible expansion control while desktop remains uncapped.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: at 481px-899px the component renders only the first eight files, while the CSS that displays the See all overlay only applies at 480px and below.

## Solution Assessment

No. The direction is reasonable, but the implementation should first make the JS mobile cutoff and CSS media query agree and add a mid-width regression check.
