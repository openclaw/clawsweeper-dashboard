## Summary

Keep open: current main does not implement the coverage contract, and PR head still has a P1 security-boundary defect because accepted ranges can have no meaningful line/byte interval; real worker proof is also missing.

## What This Changes

The PR adds required `artifact_coverage` output for the Codex scan worker, prompts for submitted-file coverage, rejects benign results without coverage, adds tests, and documents the invariant.

## Best Possible Solution

Keep the worker-level coverage contract, but validate non-empty ordered line or byte intervals and land it only with redacted live worker proof plus coordinated scanner-policy review.

## Solution Assessment

No, not yet. The worker boundary is right, but clean-scan proof must validate meaningful inspected intervals and include live worker evidence before this is a safe fail-closed contract.
