## Summary

This PR is still necessary because current main does not independently gate benign Codex results on artifact coverage, but it needs real worker proof before merge. The implementation is focused, CI-backed, and aligned with the repository’s security-spec policy; the main unresolved concern is operational: this new fail-closed contract can turn otherwise benign scans into worker failures if the model’s structured output or normalized paths do not match the verifier.

## What This Changes

The PR adds a required `artifact_coverage` schema object, prompt instructions, benign-verdict enforcement, focused tests, and a security-moderation invariant.

## Best Possible Solution

Keep enforcement immediately before clean-result storage, then prove one real complete benign scan and one intentionally incomplete benign scan through the actual worker/job path, confirming successful storage, rejection diagnostics, retry behavior, and supported path normalization.

## Reproduction Assessment

Yes at the source level: current main accepts a schema-valid benign verdict without independently comparing coverage claims to submitted files, and the candidate adds a deterministic check at that boundary. The discussion does not establish a real current-main scan reproduction.

## Solution Assessment

Yes, conditionally: enforcing immediately before clean storage is the narrowest maintainable boundary, and the focused tests are appropriate. Live worker proof is still required because the stricter schema and path contract can change scan completion behavior in ways mocks and CI do not demonstrate.
