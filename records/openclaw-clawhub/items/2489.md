## Summary

Keep open: current main does not implement Codex scan artifact coverage proof, and the PR still has a security-boundary blocker because accepted coverage ranges can have all line and byte coordinates set to null, with no real worker proof yet.

## What This Changes

Adds `artifact_coverage` to the Codex scan worker schema and prompt, validates benign results against submitted files, expands worker tests, and records the invariant in `specs/security-moderation.md`.

## Best Possible Solution

Require accepted coverage ranges to include concrete inspected line or byte intervals, keep suspicious/malicious diagnostics usable, and land only after a redacted real Codex worker run shows compatible output and rejection behavior.
