- **[P1] Handle zero-byte files in coverage proof:** `scripts/security/run-codex-scan-worker.ts:998`
  - body: The new interval validator rejects any byte range where `endByte === startByte`. Submitted files are stored with `file.size` and only oversized files are rejected, so a valid zero-byte artifact file can reach the worker; honest full coverage for that file is `startByte: 0, endByte: 0`, which this gate rejects and can make benign scans retry or fail. Allow zero-byte full-file coverage and keep nonzero empty intervals rejected.
