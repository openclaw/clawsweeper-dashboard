## Summary

This PR remains useful because current main lacks benign artifact-coverage enforcement, but it should not merge yet: the schema makes coverage mandatory for every verdict despite the stated benign-only policy, and the contributor has not supplied real worker proof.

## What This Changes

The PR adds structured artifact-coverage output and rejects benign Codex scan results unless all submitted files have claimed full or head-and-tail inspection ranges.

## Best Possible Solution

Keep strict complete-coverage enforcement for benign trust elevation, but allow incomplete coverage metadata on suspicious or malicious results so security-relevant findings survive, then demonstrate both acceptance and rejection paths with redacted real worker output.

## Reproduction Assessment

Yes, from source inspection: `artifact_coverage` is required by the schema for every output, while the later assertion and documented invariant are verdict-specific to benign results. A live worker run is still needed to measure how often this becomes an operational failure.

## Solution Assessment

No, not as currently expressed in the schema; the narrow maintainable solution is to fail closed for benign trust elevation without discarding useful suspicious or malicious results solely for incomplete coverage metadata.
