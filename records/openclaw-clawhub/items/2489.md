## Summary

This PR is still useful and not implemented on current main: it adds the missing worker-time artifact coverage contract before a benign Codex scan can become a clean ClawScan result. I found no concrete line-level defect, but merge still needs maintainer acceptance because the change deliberately makes clean scans fail closed when live worker output lacks valid coverage proof.

## What This Changes

The PR requires `artifact_coverage` in Codex scan worker output, prompts Codex with the submitted artifact manifest, rejects benign results without valid full or head/tail byte-range proof, adds worker tests, and documents the invariant in the security moderation spec.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Yes, conditionally. The worker/schema boundary is the narrow maintainable place to enforce clean-result coverage, and the PR adds range validation plus tests; the remaining question is maintainer acceptance of the fail-closed scanner contract.
