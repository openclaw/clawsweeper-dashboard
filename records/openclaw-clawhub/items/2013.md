## Repair Work Prompt

Fix ClawScan package review prompts so package.json openclaw.environment metadata is included in structured review context. On current main, evaluatePackageReleaseWithLlm stores release.capabilities only under parsed.metadata and does not pass capabilityTags; assembleEvalUserMessage only reads skill-style requires.env/envVars/requires.config, so package reviews can say env vars/config paths are none despite environment:declared. Add the smallest compatible metadata path: pass package release capabilityTags into SkillEvalContext and extract/format known openclaw.environment env/config/credential fields from package.json or persisted package metadata for the Requirements/context section. Preserve the raw artifact file block and avoid broad scan-policy changes. Add regression coverage for a package-style eval context with openclaw.environment optional env vars and config paths, and add a CHANGELOG.md Fixes entry without inventing contributor credit.
