## Summary

Keep open: current main still derives package `environment:declared` capability tags from `openclaw.environment`, but package ClawScan evaluation does not pass that env/config metadata into the prompt fields that render requirements. The closed related PR is useful context, but it was not merged and the issue remains source-reproducible.

## What This Changes

ClawScan should include declared package `openclaw.environment` env/config metadata in package review prompts instead of reporting those requirements as `none`.

## Best Possible Solution

Add a sanitized package-environment-to-prompt mapper that forwards only declared env/config requirements into the existing `metadata.openclaw` prompt shape, preserves existing requirements, and adds regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows package capability extraction reads `openclaw.environment`, while package LLM evaluation never supplies that metadata to the prompt path that renders env vars and config paths.

## Solution Assessment

Yes. The narrow maintainable fix is to normalize only the declared environment fields into prompt requirements and avoid forwarding arbitrary package metadata.
