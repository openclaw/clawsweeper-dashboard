## Summary

Keep open: current main still derives package `environment:declared` capability tags from `package.json` `openclaw.environment`, but package ClawScan evaluation does not pass that env/config metadata into the prompt fields that render requirements. The related implementation in https://github.com/openclaw/clawhub/pull/2009 was closed unmerged, and the adjacent open ClawScan PR does not close this issue.

## What This Changes

The issue asks package ClawScan reviews to include declared `package.json` `openclaw.environment` env/config metadata in the LLM requirements context.

## Best Possible Solution

Add a sanitized package-environment-to-prompt mapper that forwards only declared env/config requirements into the existing `metadata.openclaw` prompt shape, preserves existing requirements, and adds regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows package capability extraction reads `openclaw.environment`, while package LLM evaluation never supplies that metadata to the prompt path that renders env vars and config paths.

## Solution Assessment

Yes. The narrow maintainable fix is to normalize only declared environment fields into prompt requirements and avoid forwarding arbitrary package metadata.
