## Summary

Keep open: current main still appears to omit declared package `openclaw.environment` metadata from the ClawScan package review prompt, while the registry extraction path does derive `environment:declared` capability tags from that same package metadata. The issue is source-reproducible and narrow enough for a focused repair, and the related open PR does not use closing syntax or prove this exact issue is already landed.

## What This Changes

The issue requests that package ClawScan reviews include declared `package.json` `openclaw.environment` env/config metadata in the LLM requirements context instead of reporting those declarations as none.

## Best Possible Solution

Add a sanitized mapper from package `openclaw.environment` into the existing prompt requirements shape, preserving existing prompt context and tests while avoiding wholesale forwarding of untrusted package metadata.

## Reproduction Assessment

Yes. Source inspection on current main shows package registry extraction reads `openclaw.environment`, while package LLM evaluation does not pass those declarations into the prompt requirements path that renders env vars and config paths.

## Solution Assessment

Yes. The narrow maintainable fix is to normalize only declared package environment fields into prompt requirements, rather than adding new scanner policy or forwarding arbitrary `openclaw` metadata.
