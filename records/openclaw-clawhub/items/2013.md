## Summary

Keep open: current main still derives package `environment:declared` tags from `package.json` `openclaw.environment`, but package ClawScan LLM evaluation does not pass that env/config metadata into the prompt fields that render requirements. The related implementation was closed unmerged, and no merged canonical follow-up fixes the package prompt path.

## What This Changes

Request that package ClawScan prompts include sanitized declared env/config metadata from `package.json` `openclaw.environment` instead of reporting those requirements as none.

## Best Possible Solution

Add a sanitized package-environment-to-prompt mapper that forwards only declared env/config requirements into the existing `metadata.openclaw` prompt shape, preserves existing requirements, and adds regression coverage.

## Reproduction Assessment

Yes, by source inspection on current main. Package capability extraction reads `openclaw.environment`, while package LLM evaluation never supplies that metadata to the prompt path that renders env vars and config paths.

## Solution Assessment

Yes. The narrow maintainable fix is to normalize only declared environment fields into prompt requirements and avoid forwarding arbitrary package metadata.
