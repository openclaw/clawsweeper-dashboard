## Summary

Keep open: current main still omits declared package `openclaw.environment` env/config metadata from the ClawScan package-review prompt, while the package registry path already derives `environment:declared` capability tags from that same metadata.

## What This Changes

ClawScan package reviews should include declared package `openclaw.environment` env vars and config paths in the LLM prompt, or avoid saying none when that metadata exists.

## Best Possible Solution

Add a sanitized mapper from package `openclaw.environment` into the existing prompt requirements shape, preserving valid existing requirements while avoiding wholesale forwarding of untrusted package metadata.

## Reproduction Assessment

Yes, source-reproducible. Current main derives `environment:declared` from package metadata but does not pass those env/config declarations into the prompt requirements fields that ClawScan renders.

## Solution Assessment

Yes. The narrow maintainable fix is to normalize declared package environment fields into prompt requirements, not to add scanner policy or forward arbitrary `openclaw` metadata.

## Repair Work Prompt

Fix the ClawScan package-review prompt so package releases that declare `package.json` `openclaw.environment` env vars/config paths have those declarations represented in the existing prompt requirements context. Add a sanitized mapper, likely in `convex/llmEval.ts`, that reads `release.extractedPackageJson.openclaw.environment` and produces only normalized fields the prompt already understands, such as `metadata.openclaw.envVars`, `metadata.openclaw.primaryEnv` when appropriate, and `metadata.openclaw.requires.config` while preserving any valid existing `requires` fields. Do not spread arbitrary package `openclaw` metadata into the prompt. Use the closed prior PR as context but avoid its reviewed defects: over-broad metadata forwarding and replacing existing requirements. Check open PR https://github.com/openclaw/clawhub/pull/2111 before finalizing to avoid conflicts in `convex/llmEval.ts` or `convex/lib/securityPrompt.ts`. Add regression coverage for required and optional env vars, config paths such as `~/.openclaw/agents/main/agent/models.json`, and malformed unrelated `openclaw` fields that must not crash or pollute prompt assembly. No public docs change is required unless the metadata mapping contract changes.
