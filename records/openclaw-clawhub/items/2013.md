## Summary

Keep open: ClawHub cleanup policy keeps issues open, and current main still shows the reported scanner mismatch. Package capability extraction recognizes `openclaw.environment`, but package ClawScan prompt construction does not forward declared env/config metadata into the requirements fields rendered for LLM review.

## What This Changes

The issue asks ClawScan package review prompts to include declared `package.json` `openclaw.environment` env vars and config paths, or avoid claiming none are declared when `environment:declared` is present.

## Best Possible Solution

Add a narrowly scoped, sanitized mapper from package `openclaw.environment` into the existing prompt requirements shape, preserve existing prompt inputs, and cover required/optional env vars plus config paths with regression tests.

## Reproduction Assessment

Yes. Source inspection on current main shows package registry capability extraction reads `openclaw.environment`, while package LLM evaluation does not pass that metadata to the prompt path that renders env vars and config paths.

## Solution Assessment

Yes. The narrow maintainable fix is to normalize only declared environment fields into the existing `metadata.openclaw` prompt requirements shape, not to pass the full untrusted package metadata object.
