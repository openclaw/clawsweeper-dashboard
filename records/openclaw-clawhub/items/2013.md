## What This Changes

The issue asks ClawScan package reviews to include declared `package.json` `openclaw.environment` env vars and config paths in the LLM requirements context.

## Best Possible Solution

Add a small package-environment-to-prompt mapper that preserves existing requirements, forwards sanitized env/config declarations, and covers package prompt assembly with regression tests.

## Reproduction Assessment

Yes. Source inspection shows a package release can derive `environment:declared` tags from `package.json` metadata while the LLM prompt path receives no `metadata.openclaw`, so the Requirements section has no env/config declarations to print.

## Solution Assessment

Yes. The narrow maintainable fix is to translate only the package environment declarations needed by the existing requirements context instead of changing broader ClawScan policy or publish validation.
