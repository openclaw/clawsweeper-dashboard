## Summary

Keep open: current main still shows the ClawScan package-review mismatch, and repository cleanup policy keeps issues open unless a maintainer resolves them. The registry derives `environment:declared` from package metadata, but package LLM evaluation does not pass the declared env/config details into the prompt fields that render requirements.

## What This Changes

ClawScan package reviews should include declared `package.json` `openclaw.environment` env/config metadata in the LLM prompt instead of reporting none.

## Best Possible Solution

Add a sanitized package-environment-to-prompt mapper that forwards declared env vars and config paths into the existing ClawScan requirements context, preserves existing requirements, and adds focused regression tests.

## Reproduction Assessment

Yes. Source inspection on current main shows package `environment:declared` tags are derived from `openclaw.environment`, while the package LLM eval context never supplies that metadata to the prompt fields that print env vars and config paths.

## Solution Assessment

Yes. The narrow maintainable solution is to map sanitized `openclaw.environment` env/config declarations into the existing `metadata.openclaw` prompt shape without broadening untrusted metadata forwarding.
