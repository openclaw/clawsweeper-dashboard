## Summary

Keep open: current main still has the reported mismatch. Package extraction derives `environment:declared` from `package.json` `openclaw.environment`, but package LLM evaluation does not map those declarations into the prompt requirements path, and the earlier implementation PR was closed unmerged.

## What This Changes

ClawScan package reviews should include declared `package.json` `openclaw.environment` env vars/config paths in the LLM prompt, or avoid claiming none are declared when `environment:declared` exists.

## Best Possible Solution

Add a sanitized package-environment mapper into the package LLM evaluation path so declared env vars and config paths populate the existing prompt requirements shape without forwarding arbitrary package metadata.

## Reproduction Assessment

Yes, by source inspection. Current main extracts `openclaw.environment` into capability tags, while `evaluatePackageReleaseWithLlm` does not feed those declarations into the `metadata.openclaw` requirements path that renders env vars and config paths.

## Solution Assessment

Yes. The narrow maintainable fix is to normalize only declared package environment fields into prompt metadata, preserving existing requirements and avoiding a broader scanner-policy change.
