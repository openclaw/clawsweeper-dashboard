## Repair Work Prompt

Fix the ClawScan package-review mismatch where `openclaw.environment` metadata produces `environment:declared` capability tags but env vars/config paths are not included in the LLM requirements prompt. Implement a sanitized mapper from package `package.json` `openclaw.environment` into the existing prompt shape used by `assembleEvalUserMessage`, forwarding only normalized env/config-related fields, preserving existing `requires.env`, `requires.bins`, `requires.anyBins`, and `requires.config`, and avoiding any spread of the full untrusted `openclaw` object. Add focused regression tests covering required and optional env vars, config paths such as `~/.openclaw/agents/main/agent/models.json`, preservation of existing requirements, and malformed unrelated `openclaw` fields that must not crash prompt assembly. Add a `CHANGELOG.md` entry for the user-facing ClawScan package review fix, and update `specs/` only if the accepted environment metadata aliases or scanner intent are clarified. Do not change package publishing policy, scanner verdict thresholds, or unrelated registry metadata behavior.
