## Summary

Keep open: ClawHub cleanup policy keeps issues open, and current main still has a source-reproducible mismatch where package environment capability tags can be derived from `openclaw.environment` while the package LLM prompt does not receive those env/config declarations.

## What This Changes

The issue requests that ClawScan include declared package `openclaw.environment` env vars and config paths in the LLM review prompt, or avoid saying none are declared when environment metadata exists.

## Best Possible Solution

Add a narrow sanitizer/mapper from package `openclaw.environment` into the existing ClawScan requirements prompt shape, preserving current metadata and adding regression coverage for env vars and config paths.

## Reproduction Assessment

Yes. Source inspection shows the package registry can emit `environment:declared` from `package.json` `openclaw.environment`, while package LLM evaluation does not map that environment block into the prompt fields that render env vars and config paths.

## Solution Assessment

Yes. The narrow maintainable fix is to normalize and forward only declared environment requirements into the existing prompt metadata, not to expose the full package metadata object.

## Repair Work Prompt

Fix ClawScan package review prompt metadata for declared package environment requirements. Current main derives `environment:declared` capability tags from package `package.json` `openclaw.environment` in `convex/lib/packageRegistry.ts`, but `convex/llmEval.ts` does not map that metadata into the requirements fields consumed by `convex/lib/securityPrompt.ts`, so package prompts can say env vars and config paths are `none`. Add a narrow sanitizer/normalizer that forwards only declared env/config requirement fields into the existing prompt metadata shape, preserve existing package requirements and capabilities, and do not pass the full untrusted `openclaw` object through. Add focused regression coverage for package env vars and config paths. Update a focused `specs/` note only if the intended ClawScan metadata boundary changes.
