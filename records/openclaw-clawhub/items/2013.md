Keep open: current main still omits declared package `openclaw.environment` metadata from the package ClawScan prompt path even though package extraction and readiness surfaces recognize declared environment metadata; the related implementation was closed unmerged and no open PR closes this issue.

## What This Changes

Include sanitized `package.json` `openclaw.environment` env/config metadata in package ClawScan prompt requirements, or avoid reporting that metadata as absent when it is declared.

## Best Possible Solution

Add a sanitized package-environment-to-prompt mapper that forwards only declared env/config requirements into the existing prompt metadata shape, preserves existing requirements, and adds regression coverage for malformed unrelated `openclaw` fields.
