## Repair Work Prompt

Fix the ClawScan package review prompt mismatch reported at https://github.com/openclaw/clawhub/issues/2013. On current main, package capability extraction reads `package.json` `openclaw.environment` and emits `environment:declared`, but `evaluatePackageReleaseWithLlm` does not pass declared env vars/config paths into the `metadata.openclaw` shape that `assembleEvalUserMessage` renders. Add a sanitized package-environment-to-prompt mapper in or near `convex/llmEval.ts` that parses the package artifact's `package.json`, forwards only normalized env/config prompt fields, preserves any existing requirements fields, and does not spread the full untrusted `openclaw` object. Add regression coverage for required/optional env vars, config paths, malformed unrelated OpenClaw metadata, and preservation of existing requirements. Include a `CHANGELOG.md` fix entry because ClawScan output is user-visible. Do not change package capability tag derivation except where tests require a shared normalizer.
