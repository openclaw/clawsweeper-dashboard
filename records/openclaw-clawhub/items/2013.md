## Repair Work Prompt

Fix the ClawScan package prompt mismatch reported in https://github.com/openclaw/clawhub/issues/2013. On current main, package capability extraction derives `environment:declared` from `package.json` `openclaw.environment`, but `evaluatePackageReleaseWithLlm` does not pass declared env/config metadata into the `metadata.openclaw` shape consumed by `assembleEvalUserMessage`. Implement a conservative allowlisted mapper that reads `release.extractedPackageJson.openclaw.environment` and forwards only normalized prompt-relevant fields such as required/optional env vars, primary credential when clearly declared, and config paths into existing requirements metadata. Preserve any existing valid `requires` fields; do not spread or forward the full untrusted `openclaw` object. Add regression coverage for required env vars, optional env vars, config paths, preservation of existing requirements, and malformed unrelated metadata that must not crash prompt assembly. Re-check open adjacent ClawScan work before editing, but do not broaden this into publisher notes or UI refactors. Do not add unrelated changelog or generated-file churn unless repo conventions require it for the final fix.
