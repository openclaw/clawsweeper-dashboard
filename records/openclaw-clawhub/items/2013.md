Keep open: current main still source-reproduces the package ClawScan prompt mismatch, and ClawHub cleanup policy keeps issues open for maintainer follow-up rather than auto-closing them.

## What This Changes

ClawScan package reviews should include declared `package.json` `openclaw.environment` env/config metadata in the LLM prompt instead of reporting those requirements as none.

## Best Possible Solution

Add a sanitized package-environment-to-prompt mapper that forwards only declared env/config requirements into the existing `metadata.openclaw` shape, preserves any existing requirements, and adds regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows package capability extraction reads `openclaw.environment`, while package LLM evaluation never supplies that metadata to the prompt path that renders env vars and config paths.

## Solution Assessment

Yes. The narrow maintainable fix is to normalize only declared environment fields into prompt requirements and avoid forwarding arbitrary package metadata.

## Repair Work Prompt

Fix the package ClawScan prompt mismatch from https://github.com/openclaw/clawhub/issues/2013. Current main derives package `environment:declared` capability tags from `package.json` `openclaw.environment`, but `evaluatePackageReleaseWithLlm` does not pass declared env/config metadata into the `metadata.openclaw` shape read by `assembleEvalUserMessage`. Implement a conservative mapper in `convex/llmEval.ts` or a small helper that reads the stored `release.extractedPackageJson.openclaw.environment` source and forwards only normalized prompt-safe fields such as declared env vars and config paths into `metadata.openclaw` (`envVars`, `primaryEnv` when appropriate, and merged `requires.config`/`requires.env` if supported). Preserve any existing prompt metadata, do not spread the full untrusted `openclaw` object, and stop for maintainer guidance if the fix requires formalizing broad new package metadata schema. Add focused regression tests covering required/optional env vars, config paths, malformed unrelated `openclaw` fields, and preserving existing requirements. Do not change unrelated ClawScan UI or publisher-note work.
