## Summary

Keep open: current main still has the package ClawScan prompt mismatch, and ClawHub cleanup policy keeps issues open until a fix lands.

## What This Changes

ClawScan package reviews should account for declared `package.json` `openclaw.environment` env/config metadata instead of reporting that none is declared.

## Best Possible Solution

Add a sanitized package-environment-to-prompt mapper that forwards declared env/config requirements into the existing ClawScan prompt shape and covers it with regression tests.

## Reproduction Assessment

Yes. Source inspection on current main shows package `environment:declared` tags are derived from `openclaw.environment`, while package LLM evaluation never supplies that metadata to the prompt fields that render env vars and config paths.

## Solution Assessment

Yes. The narrow maintainable fix is to normalize only the declared env/config fields into `metadata.openclaw` requirements and preserve existing requirements, rather than forwarding arbitrary package metadata.
