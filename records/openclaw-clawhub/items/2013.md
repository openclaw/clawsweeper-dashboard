## Repair Work Prompt

Fix the package ClawScan prompt path so package releases with `package.json` `openclaw.environment` declarations do not report env vars/config paths as none. On current main, add a helper in `convex/llmEval.ts` or a focused Convex lib that reads `release.extractedPackageJson.openclaw.environment` and maps declared env/config fields into the `parsed.metadata.openclaw` shape consumed by `assembleEvalUserMessage`. Only forward allowlisted, normalized prompt fields such as `requires.env`, `requires.config`, `envVars`, and `primaryEnv`; do not spread the full untrusted `openclaw` object, do not change skill evaluation behavior, and preserve existing requirements when adding config paths. Add regression coverage for required env vars, optional env vars, config paths, malformed unrelated `openclaw` fields, and existing requirement preservation. Re-check whether open PR https://github.com/openclaw/clawhub/pull/2111 has landed before editing because it touches adjacent ClawScan files. Update `specs/security-moderation.md` only if the implementation clarifies durable scanner intent.
