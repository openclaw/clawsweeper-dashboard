Full review comments:

- **[P1] Require ownership proof before granting Official:** `convex/lib/officialPublishers.ts:33-34`
  - body: `isOfficialPublisher` returns true for any active org publisher whose normalized handle is `openclaw`. Because self-serve org creation only blocks that handle when a reserved-handle row exists, a missing or wrong ownership reservation would turn that org and all of its members into Official publishers with official-channel rights. Fail closed unless the `openclaw` publisher is tied to an authoritative OpenClaw-owned reservation or org id, and add a regression test for the unreserved or wrong-owner case.
  - confidence: 0.88
- **[P2] Populate owner.official for skill detail badges:** `src/components/SkillHeader.tsx:151`
  - body: `SkillHeader` now treats `owner?.official` as the source for owner-derived Official skills, but the skill detail query still serializes `owner` with plain `toPublicPublisher`, so OpenClaw-owned skills without a persisted skill badge will not show the new badge. Wire `toPublicPublisherWithOfficial` or equivalent into the skill detail owner data and cover it with a route or backend test.
  - confidence: 0.91
