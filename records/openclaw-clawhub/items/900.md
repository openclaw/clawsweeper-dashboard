## Summary

Keep open. Current main still blocks deleted/deactivated GitHub identities from becoming active users again, and resolving the reported re-login loop requires a maintainer product/security decision rather than cleanup closure.

## What This Changes

The issue requests that a self-deleted ClawHub account linked to GitHub be able to sign in again or be restored/relinked so account access and CLI tokens work.

## Best Possible Solution

Maintainers should choose one deleted-account policy, then enforce it consistently across OAuth callback handling, active-user gates, token creation, settings/header/CLI UI, docs/specs, and regression tests.

## Reproduction Assessment

Yes, at source level. Deletion writes deactivatedAt/purgedAt, OAuth reauth throws for deleted/deactivated users, active-user helpers return no user, the header falls back to sign-in, and token creation requires an active user; I did not run a live production OAuth deletion flow.

## Solution Assessment

No autonomous patch is the best next step yet. Deleted-account restoration, relinking, recreation, or support-only recovery changes an auth identity boundary and needs maintainer product/security policy first.
