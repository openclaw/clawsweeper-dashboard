## Summary

Keep open. Current main still enforces permanent deletion for this account-lifecycle path, and ClawHub cleanup policy keeps issues open unless maintainers explicitly resolve them.

## What This Changes

The issue requests re-login, account recreation or relinking, and CLI token access after a user deletes their ClawHub account tied to GitHub.

## Best Possible Solution

Adopt one explicit deleted-account identity policy and enforce it consistently across OAuth callback handling, active-user gates, token creation, header/settings/CLI UX, support messaging, docs, and regression tests.

## Reproduction Assessment

Yes, at source level. Current main marks self-deleted users as deactivated, blocks OAuth reauth with the permanent-deletion error, returns no active `me`, and requires active user state before CLI token creation.

## Solution Assessment

No, not as an autonomous cleanup fix. The narrow maintainable path is a maintainer-approved policy first, followed by code and tests for either restore/relink/recreate behavior or clearer irreversible-deletion support routing.
