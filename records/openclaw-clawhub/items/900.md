## Summary

Keep open. Current main still blocks self-deleted/deactivated GitHub identities from becoming active users again, and the remaining decision is a security-sensitive account recovery/relink policy rather than conservative cleanup; repository policy also keeps ClawHub issues open unless a valid close reason applies.

## What This Changes

The issue asks ClawHub to allow or clearly handle login, account recreation, restoration, or GitHub relinking after a user deletes their ClawHub account.

## Best Possible Solution

Define one explicit deleted-account recovery policy, then implement it consistently across OAuth callback handling, provider identity binding, active-user gates, CLI token creation, settings/header UI, operator docs/specs, and regression tests.

## Reproduction Assessment

Yes, at source level. Deletion writes inactive/purged state, OAuth rejects that state, active-user helpers make `users.me` unavailable, and token creation requires an active user; I did not run a live production OAuth deletion flow.

## Solution Assessment

No autonomous code change is the best next step yet. Maintainers first need to decide whether deleted GitHub identities may be restored, unlinked, recreated, or routed through support only.
