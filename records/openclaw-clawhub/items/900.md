## Summary

Keep open. Current main still treats self-deleted/deactivated GitHub identities as inactive and blocks OAuth reactivation, so the reported sign-in loop and CLI-token lockout remain unresolved; ClawHub cleanup policy keeps issues open until maintainers choose the account-deletion policy.

## What This Changes

The issue requests that a user who deleted their ClawHub account can sign in again with the same GitHub identity and regain account access, including CLI tokens.

## Best Possible Solution

Adopt one explicit deleted-account identity policy, then enforce it consistently across OAuth callback handling, active-user gates, token creation, settings/header/CLI UI, docs, audit behavior, and regression tests.

## Reproduction Assessment

Yes, by source inspection. Delete an account so deactivatedAt/purgedAt are set, sign in again with the same GitHub identity, and current main rejects reactivation while users.me stays null and token creation remains gated on an active user.

## Solution Assessment

Unclear. Restore, relink, recreate, or support-only routing could each address the user problem, but maintainers need to choose the deletion and identity-binding policy before a narrow implementation is safe.
