## Summary

Keep open. Current main still enforces irreversible deleted-account handling and matches the reported re-login/CLI-token lockout at source level; resolving it needs maintainer product/security policy rather than cleanup close.

## What This Changes

Request deleted ClawHub accounts be able to sign in again with the same GitHub identity, or stop leaving users in a live sign-in/CLI-token loop.

## Best Possible Solution

Choose one deletion policy and enforce it consistently across OAuth, active-user gates, token creation, settings/header/CLI UI, docs, and regression tests: either audited restore/relink/recreate, or irreversible deletion with clear support routing.

## Reproduction Assessment

Yes. There is a high-confidence source-level path: deleteAccount sets deactivatedAt/purgedAt, OAuth reauth throws, users.me returns null, and CLI token creation requires requireUser; I did not run a live OAuth flow with the reporter's GitHub identity.

## Solution Assessment

No. Directly allowing deleted GitHub identities to recreate or relink accounts is not the best autonomous fix until maintainers decide the deletion/restoration policy and security model.
