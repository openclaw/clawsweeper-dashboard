## Summary

Keep open. Current main still blocks self-deleted/deactivated GitHub identities from becoming active users again, and this ClawHub issue needs maintainer product/security policy before any restore, relink, recreate, or support-only behavior changes.

## What This Changes

The issue requests that a GitHub identity from a self-deleted ClawHub account be restored, relinked, or recreated on sign-in instead of leaving the user logged out without CLI-token access.

## Best Possible Solution

Adopt one explicit deleted-account identity policy and enforce it consistently across OAuth callback handling, active-user gates, token creation, settings/header/CLI UI, support messaging, docs, and regression tests.

## Reproduction Assessment

Yes, at source level. A self-deleted account gets deactivatedAt/purgedAt, OAuth reauth throws the permanent-deletion error, users.me returns null through active-user gates, and CLI token creation requires requireUser; I did not run a live OAuth flow with the reporter's GitHub identity.

## Solution Assessment

No, not as an autonomous code change. The narrow maintainable path is a maintainer-approved deletion/restoration policy first, followed by implementation and tests for either restore/relink/recreate or clearer irreversible-deletion support routing.
