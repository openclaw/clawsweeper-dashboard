## Summary

Keep open. Current main still rejects deleted/deactivated GitHub identities from becoming active users again; recent error surfacing improves messaging, but the requested restore/relink/recreate behavior remains a product/security account-recovery decision.

## What This Changes

Request that a self-deleted ClawHub account linked to GitHub can sign in again or be recreated/relinked instead of looping with the sign-in button and CLI token access unavailable.

## Best Possible Solution

Define one deleted-account recovery policy, then implement it consistently across OAuth, active-user gates, token creation, header/CLI messaging, support/admin tools, docs/specs, and regression tests.

## Reproduction Assessment

Yes, at source level. Deletion writes `deactivatedAt`/`purgedAt`, OAuth rejects deleted/deactivated users, active-user helpers return no user, and token creation requires an active user; I did not run a live production OAuth deletion flow.

## Solution Assessment

No autonomous restore/relink patch is the best next step yet. Maintainers first need to decide whether deleted GitHub identities may be restored, unlinked, recreated, or routed through support only.
