## Summary

Keep open. Current main intentionally blocks deleted/deactivated accounts from OAuth reauth, same-symptom reports remain open, and recovery/relinking is now an explicit security/product policy question rather than conservative cleanup.

## What This Changes

The issue requests that a deleted ClawHub account tied to a GitHub identity can sign in again or be re-linked/recreated instead of returning to a live sign-in button and blocking CLI tokens.

## Best Possible Solution

Adopt a maintainer-approved deleted-account recovery policy, then implement it consistently across OAuth, auth account binding, deletion, operator tooling, UI/CLI messaging, specs/docs, and regression tests.

## Reproduction Assessment

Yes, at source level. Deletion writes deactivatedAt/purgedAt, OAuth rejects that user state, active-user helpers make users.me null, and token creation requires an active user; I did not run a live production OAuth deletion flow.

## Solution Assessment

No autonomous code change is the best next step yet. Restoring, recreating, or unlinking a deleted GitHub identity contradicts the current irreversible-deletion spec and RFC 0001 leaves the recovery policy open.

## Security Review

Status: needs_attention

Summary: Deleted-account recovery is security-sensitive because it changes identity reuse, deletion privacy, token revocation, and audit semantics.

Concerns:

- **[medium] Define recovery authority before changing sign-in:** `specs/security-moderation.md:139`
  - body: The current spec says user-initiated deletion is irreversible, while the draft RFC leaves recovery open; OAuth restore, relink, or re-registration needs explicit policy approval before implementation.
  - confidence: 0.95
- **[medium] Preserve identity and ban safeguards:** `convex/auth.ts:37`
  - body: The OAuth helper checks matching identity state and ban audit records before rejecting deleted users; any recovery path needs equivalent bypass prevention and abuse handling.
  - confidence: 0.9
- **[medium] Define token and privacy semantics:** `convex/users.ts:338`
  - body: Account deletion revokes API tokens and clears profile/contact/GitHub-derived fields, so recovery must define whether credentials, publisher links, audit history, and personal data are restored or recreated.
  - confidence: 0.9
