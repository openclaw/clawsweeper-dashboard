## Summary

Keep open. Current main still enforces irreversible self-deletion for the same GitHub identity, and the requested re-login/relink behavior conflicts with the documented account-deletion policy and token semantics; ClawHub cleanup policy also keeps issues open rather than auto-closing them.

## What This Changes

Allow self-deleted GitHub-linked accounts to re-login, recreate, or relink so users stop seeing the sign-in loop and can access CLI tokens again.

## Best Possible Solution

Adopt one explicit deleted-account policy and enforce it consistently across OAuth callback handling, active-user gates, token creation, settings/header/CLI UI, docs, audit logs, and regression tests.

## Reproduction Assessment

Yes, by source inspection. Delete an account so `deactivatedAt`/`purgedAt` are set, sign in again with the same GitHub identity, and current main rejects OAuth reactivation while `users.me` stays `null` and CLI token creation never starts.

## Solution Assessment

Unclear. Restore/relink/recreate could solve the user problem, but it is not clearly the best fix until maintainers decide the deletion, identity-binding, audit, and token policy.
