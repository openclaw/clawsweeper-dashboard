## Summary

Keep open. Current main still rejects deleted/deactivated GitHub identities during OAuth and then treats them as no active user, while the documented deletion policy says accounts cannot be restored by logging in again; changing that needs maintainer product and security judgment.

## What This Changes

Request that a user who deleted a ClawHub account can sign in with the same GitHub identity again and access CLI tokens, instead of remaining effectively logged out.

## Best Possible Solution

Define one deleted-account recovery policy, then enforce it consistently across OAuth callback handling, active-user gates, token creation, settings/header/CLI UI, docs, audit logging, and regression tests.

## Reproduction Assessment

Yes. The source-level path is high confidence: self-service deletion sets `deactivatedAt`, GitHub OAuth reauth throws the deleted-account error, `users.me` returns `null`, and CLI token creation never runs because there is no active user.

## Solution Assessment

Unclear. The requested restore/relink/recreate behavior may be the right outcome, but the maintainable solution first needs a product/security decision on whether deletion stays irreversible or becomes recoverable under audited conditions.
