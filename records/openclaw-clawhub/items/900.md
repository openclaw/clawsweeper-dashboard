## Summary

Keep open. Current main still blocks self-deleted or deactivated GitHub identities from becoming active users again, and ClawHub cleanup policy keeps issues open unless maintainers decide otherwise; the remaining choice is a product/security policy decision rather than a narrow bug fix.

## What This Changes

Allow or otherwise handle re-login after a user deletes a ClawHub account linked to GitHub, including access to CLI token creation.

## Best Possible Solution

Define one deleted-account identity policy, then enforce it consistently across OAuth callback handling, active-user gates, token creation, settings/header/CLI UI, support messaging, docs, and regression tests.

## Reproduction Assessment

Yes, by source inspection. A GitHub-linked user with `deactivatedAt` or migrated legacy `deletedAt` is rejected during OAuth, then fails active-user checks so `users.me` is null and CLI token creation is unreachable.

## Solution Assessment

Unclear. Restoring, relinking, recreating, or support-routing the GitHub identity may solve the user problem, but maintainers first need to choose the privacy and security semantics for irreversible deletion.
