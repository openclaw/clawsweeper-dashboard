## Summary

Keep open. Current main still blocks deleted or deactivated GitHub identities from becoming active users again, and the remaining recovery/relink behavior is a product and security policy decision rather than conservative cleanup.

## What This Changes

The issue requests that a user who deleted their ClawHub account can sign back in with the same GitHub account and regain access to CLI token creation instead of seeing the sign-in loop/button state.

## Best Possible Solution

Define one maintainer-approved deleted-account recovery policy, then enforce it consistently across OAuth, account deletion, active-user gates, CLI token creation, UI messaging, operator tooling, specs/docs, and regression tests.

## Reproduction Assessment

Yes, at source level. Current main shows deletion writes deactivatedAt/purgedAt, OAuth rejects that state, users.me returns null through the active-user gate, and CLI token creation requires an active user; I did not run a live production OAuth deletion flow.

## Solution Assessment

No autonomous cleanup fix is the best next step. Restoring, recreating, or unlinking a deleted GitHub identity changes account recovery, identity ownership, token, and privacy semantics, so maintainers need to choose the policy first.

## Security Review

Status: needs_attention

Summary: Deleted-account recovery is security-sensitive because it changes identity reuse, deletion privacy, token revocation, and audit semantics.

Concerns:

- **[medium] Define recovery authority before changing sign-in:** `specs/security-moderation.md:180`
  - body: The current spec says user-initiated deletion is irreversible, while RFC 0001 leaves recovery open; OAuth restore, relink, or re-registration needs explicit policy approval before implementation.
  - confidence: 0.95
- **[medium] Preserve identity and ban safeguards:** `convex/auth.ts:40`
  - body: The OAuth helper checks matching identity state and ban audit records before rejecting deleted users; any recovery path needs equivalent bypass prevention and abuse handling.
  - confidence: 0.9
- **[medium] Define token and privacy semantics:** `convex/users.ts:475`
  - body: Account deletion revokes API tokens and clears profile/contact/GitHub-derived fields, so recovery must define whether credentials, publisher links, audit history, and personal data are restored or recreated.
  - confidence: 0.9
