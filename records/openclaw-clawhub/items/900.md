## Summary

Keep open. Current main still blocks deleted/deactivated GitHub identities from becoming active users again, and solving the requested re-login/relink behavior requires an explicit product/security account-recovery policy rather than conservative cleanup.

## What This Changes

The issue asks for GitHub sign-in to recover, relink, or recreate a ClawHub account after user-initiated deletion instead of leaving web and CLI auth stuck in the sign-in state.

## Best Possible Solution

End with one documented deleted-account policy, then enforce it consistently across OAuth, active-user gates, token creation, settings/header/CLI UI, support/admin tooling, specs/docs, and regression tests.

## Reproduction Assessment

Yes, at source level: deletion writes `deactivatedAt`/`purgedAt`, OAuth rejects deleted or deactivated users, active-user helpers return no user, and token creation requires an active user. I did not run a live production OAuth deletion flow.

## Solution Assessment

No autonomous code change is the best next step yet. Maintainers first need to decide whether deleted GitHub identities may be restored, unlinked, recreated, or routed through support only.
