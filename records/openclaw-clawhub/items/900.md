## Summary

Keep open. Current main still enforces irreversible deleted-account handling: OAuth reauth is blocked, active-user lookups return no user, the header falls back to sign-in, and CLI token creation requires an active user. Because any fix changes deletion, OAuth identity binding, and token-access semantics, this needs maintainer product/security judgment before implementation.

## What This Changes

A user reports that after deleting a ClawHub account, GitHub OAuth returns to the site but the account is not reactivated and CLI tokens remain inaccessible.

## Best Possible Solution

Choose one deleted-account policy, then implement it consistently across OAuth callback handling, active-user gates, token creation, header/settings/CLI messaging, management recovery paths, audit logging, docs, and regression tests.

## Reproduction Assessment

Yes. The source-level path is clear: delete through `users.deleteAccount`, then sign in with the same GitHub identity; current main deactivates the user, blocks OAuth reauth, returns no active user, and prevents CLI token creation.

## Solution Assessment

Not applicable because there is no proposed patch. The requested restore/relink/recreate behavior may be right, but it is not safely implementable until maintainers choose the account-deletion policy.
