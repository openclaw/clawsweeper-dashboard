## Summary

Keep open. Current main still intentionally rejects deleted/deactivated accounts, the internal spec says user deletion is irreversible, and the related open cluster plus draft RFC show recovery/relinking is a product and security policy decision rather than conservative cleanup.

## What This Changes

The issue requests that a deleted GitHub-linked ClawHub account be recoverable, relinked, or recreated on OAuth sign-in so web login and CLI token creation work again.

## Best Possible Solution

Adopt a maintainer-approved deleted-account recovery policy, then implement one consistent path across OAuth, user deletion state, active-user gates, token creation, UI/CLI messaging, operator tooling, specs/docs, and regression tests.

## Reproduction Assessment

Yes, at source level. The current path is deleteAccount setting deactivatedAt/purgedAt, OAuth rejecting that state, active-user helpers returning no user, and CLI token creation requiring an active user; I did not run a live production OAuth deletion flow.

## Solution Assessment

No autonomous code change is the best next step yet. Restore, recreate, unlink, or support-only recovery each changes security and privacy semantics, so maintainers should decide the policy before implementation.

## Security Review

Status: needs_attention

Summary: Deleted-account recovery is security-sensitive because it changes identity reuse, deletion privacy, token revocation, ownership, and abuse-prevention semantics.

Concerns:

- **[medium] Define recovery authority before changing sign-in:** `specs/security-moderation.md:180`
  - body: The current spec says user-initiated deletion is irreversible while RFC 0001 leaves account recovery open, so OAuth restore, relink, or re-registration needs explicit product/security approval first.
  - confidence: 0.95
- **[medium] Preserve deleted-account and ban safeguards:** `convex/auth.ts:45`
  - body: The OAuth helper currently rejects deleted users and checks blocking ban audit actions for legacy deletedAt users; any recovery path needs equivalent bypass prevention and abuse handling.
  - confidence: 0.9
- **[medium] Define token and data-restoration semantics:** `convex/users.ts:475`
  - body: Account deletion revokes API tokens and clears profile/contact/GitHub-derived fields, so recovery must define whether credentials, publishers, audit history, and personal data are restored, recreated, or kept purged.
  - confidence: 0.9
