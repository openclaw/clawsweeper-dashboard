## Summary

Keep open. Current main still enforces irreversible deletion for GitHub-backed users, and the requested re-login/relink behavior remains a security-sensitive account-lifecycle decision with multiple related reports still open.

## What This Changes

The issue requests that a deleted ClawHub account can sign back in with the same GitHub identity and regain account features such as CLI token access.

## Best Possible Solution

Maintainers should choose one deleted-account identity policy, then implement it consistently across OAuth callback handling, active-user gates, token creation, account/settings/header/CLI UX, specs/docs, and regression tests.

## Reproduction Assessment

Yes, at source level. Current main deactivates and purges the user on self-deletion, blocks the same identity during OAuth, returns no active users.me, and gates CLI token creation on an active user; I did not run a live OAuth deletion flow.

## Solution Assessment

No autonomous code change is the best next step yet. Restoring, relinking, recreating, or support-routing deleted identities changes a security-sensitive irreversible-deletion policy, so maintainers should decide that policy first.

## Security Review

Status: needs_attention

Summary: Deleted-account recovery is security-sensitive because it changes identity reuse, deletion privacy, token revocation, and ownership semantics.

Concerns:

- **[medium] Define deleted-account recovery policy first:** `specs/security-moderation.md:136`
  - body: The internal spec currently says user-initiated deletion is irreversible and deleted accounts cannot be restored by logging in again, so restore, relink, or re-registration needs explicit maintainer policy approval.
  - confidence: 0.95
- **[medium] Preserve OAuth identity and ban safeguards:** `convex/auth.ts:37`
  - body: The OAuth callback blocks deleted/deactivated identities and uses existingUserId checks before migrating legacy deleted records, so any recovery path needs equivalent identity-matching and ban/autoban protections.
  - confidence: 0.9
- **[medium] Define token and ownership semantics:** `convex/users.ts:338`
  - body: Account deletion revokes API tokens and clears profile/contact fields while published content remains public, so recovery must define whether tokens, audit history, publishers, and ownership are restored or recreated.
  - confidence: 0.9
