## Summary

Keep open. Current main still intentionally blocks deleted/deactivated GitHub identities from becoming active users again, and the requested restore/relink/recreate behavior is an unresolved account-recovery policy decision rather than conservative cleanup.

## What This Changes

The issue requests that users who delete their ClawHub account can sign in again with GitHub and regain account access, including CLI tokens.

## Best Possible Solution

Settle the deleted-account recovery policy, then implement one consistent path across OAuth, auth bindings, active-user gates, CLI token creation, UI messaging, operator tooling, specs/docs, and regression tests.

## Reproduction Assessment

Yes, at source level. Deletion writes deactivatedAt/purgedAt, OAuth rejects that state, active-user helpers make users.me null, and token creation requires an active user; I did not run a live production OAuth deletion flow.

## Solution Assessment

No autonomous restore/relink code change is the best next step yet. The maintainable path needs a product/security decision about whether deleted GitHub identities may be restored, unlinked, recreated, or routed through support only.
