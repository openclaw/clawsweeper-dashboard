## Summary

Keep open. Current main still blocks deleted/deactivated GitHub identities from becoming active users again, while the desired restore/relink/recreate behavior conflicts with the documented irreversible-deletion policy and needs maintainer product/security judgment.

## What This Changes

The issue requests that a user who deleted a GitHub-linked ClawHub account can sign in again or otherwise regain access instead of remaining in a logged-out state.

## Best Possible Solution

End with one maintainer-approved deleted-account policy enforced across OAuth callback handling, active-user gates, token creation, settings/header/CLI UI, docs, and regression tests.

## Reproduction Assessment

Yes, by source inspection. Delete a GitHub-linked account so `deactivatedAt` and `purgedAt` are set, then sign in with the same GitHub identity; current main rejects reauth, leaves `users.me` null, and prevents CLI token creation.

## Solution Assessment

Unclear. Restoring, relinking, or recreating the account may be the desired user outcome, but it changes the documented irreversible-deletion contract and needs maintainer-approved security/privacy policy first.
