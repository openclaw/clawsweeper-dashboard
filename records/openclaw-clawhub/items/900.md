## Summary

Keep open. Current main still blocks deleted/deactivated GitHub identities from becoming active users again; the UI now has better auth-error handling, but the requested recovery/relink behavior remains a security-sensitive product decision.

## What This Changes

The issue requests a deleted-account GitHub re-login or recovery path, plus clearer handling when OAuth returns but the account remains inactive.

## Best Possible Solution

Choose one deleted-account policy, then enforce it consistently across OAuth callback handling, active-user gates, CLI token creation, settings/header messaging, specs/docs, and regression tests.

## Reproduction Assessment

Yes, at source level. Deletion writes deactivatedAt/purgedAt, OAuth reauth throws for deleted/deactivated users, users.me returns null through the active-user gate, and CLI token creation requires an active user; I did not run a live production OAuth deletion flow.

## Solution Assessment

No autonomous patch is the best next step yet. Restoring, relinking, recreating, or permanently rejecting deleted GitHub identities changes an auth identity boundary and needs maintainer product/security policy first.
