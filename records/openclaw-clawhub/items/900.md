## Summary

Keep open. Current main still intentionally blocks deleted/deactivated GitHub identities from becoming active users again, which matches the reported sign-in loop and CLI-token lockout, and this ClawHub cleanup pass keeps open issues open unless maintainers decide the policy outcome.

## What This Changes

The issue asks for GitHub OAuth re-login and CLI token access to work after a user deletes their ClawHub account.

## Best Possible Solution

Choose and document one deleted-account policy, then implement it consistently across OAuth callback handling, active-user gates, token creation, settings/header/CLI UI, management recovery paths, audit logging, and regression tests.

## Reproduction Assessment

Yes. The issue gives concrete steps, and current source shows the same path: deletion writes `deactivatedAt`, OAuth reauth throws, `users.me` becomes null, the header falls back to sign-in, and CLI token creation never runs without an active user.

## Solution Assessment

Unclear because there is no patch and the requested restore/relink behavior is a product/security decision. A maintainable fix should first define whether deleted accounts may be restored, relinked, recreated, or only routed to support.
