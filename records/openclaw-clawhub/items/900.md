## Summary

Keep open. Current main still blocks self-deleted GitHub identities from becoming active users again, and account recovery is now an explicit RFC/product-security policy question rather than an implemented behavior.

## What This Changes

The issue requests a recovery/relink path after self-deleting a ClawHub account so GitHub sign-in and CLI token creation work again.

## Best Possible Solution

Finish the account-recovery RFC or equivalent maintainer decision, then implement one consistent restore, relink, recreate, or support-only path across OAuth, active-user gates, CLI tokens, UI copy, specs, and tests.

## Reproduction Assessment

Yes, at source level: the OAuth callback rejects deleted/deactivated identities, active-user gates return no user, and the web/CLI paths stay in sign-in mode when users.me is null. I did not run a live GitHub OAuth deletion flow.

## Solution Assessment

No autonomous fix is the best next step yet. Restore, relink, recreate, or support-only handling changes security-sensitive account lifecycle semantics, so maintainers should settle the policy first.

## Security Review

Status: needs_attention

Summary: Deleted-account recovery is security-sensitive because it changes identity reuse, irreversible deletion, token revocation, and ownership semantics.

Concerns:

- **[medium] Define recovery policy before changing auth behavior:** `specs/security-moderation.md:139`
  - body: The internal spec currently says user-initiated deletion is irreversible and deleted accounts cannot be restored by logging in again, so any recovery/relink path needs an explicit maintainer decision.
  - confidence: 0.95
- **[medium] Preserve deleted and banned identity safeguards:** `convex/auth.ts:24`
  - body: The OAuth callback blocks deactivated users and checks legacy deleted users against audit-log ban records before rejecting sign-in, so recovery work must preserve equivalent safeguards.
  - confidence: 0.9
- **[medium] Specify token and ownership semantics:** `convex/users.ts:332`
  - body: Self-deletion revokes API tokens and clears profile/contact fields while published content remains public, so recovery must define whether tokens, publishers, audit history, and ownership are restored or recreated.
  - confidence: 0.9
