## Summary

Keep open. Current main still intentionally blocks self-deleted GitHub-backed accounts from becoming active users again, and ClawHub cleanup policy says issues stay open unless maintainers resolve them; the remaining work is a product/security account-lifecycle decision shared with several open related reports.

## What This Changes

The issue requests that deleted GitHub-backed ClawHub accounts can re-login, relink, or otherwise recover instead of remaining in a sign-in loop with CLI tokens inaccessible.

## Best Possible Solution

Maintainers should choose one deleted-account identity policy, then implement it consistently across OAuth callback handling, active-user gates, token creation, settings/header/CLI UX, operator tooling, docs/specs, and regression tests.

## Reproduction Assessment

Yes, at source level. Current main sets deactivatedAt/purgedAt on self-deletion, rejects the same identity during OAuth, returns no active users.me, and gates CLI token creation on an active user.

## Solution Assessment

No autonomous code change is the best next step yet. The requested restore/relink/recreate behavior changes a security-sensitive irreversible-deletion policy, so maintainers should decide the policy before implementation.

## Security Review

Status: needs_attention

Summary: Deleted-account recovery is security-sensitive because it changes identity reuse, deletion privacy, token revocation, and ownership semantics.

Concerns:

- **[medium] Define deleted-account recovery policy first:** `specs/security-moderation.md:132`
  - body: The spec currently says user-initiated deletion is irreversible and deleted accounts cannot be restored by logging in again, so restore, relink, or re-registration needs explicit maintainer approval.
  - confidence: 0.95
- **[medium] Preserve identity and ban safeguards:** `convex/auth.ts:37`
  - body: The OAuth callback blocks deleted/deactivated identities and checks existingUserId before migrating legacy deleted records, so any recovery path needs equivalent GitHub identity and ban/autoban protections.
  - confidence: 0.9
- **[medium] Define token and ownership semantics:** `convex/users.ts:338`
  - body: Account deletion revokes API tokens and purges profile/contact fields while published content remains public, so recovery must define whether tokens, audit history, publishers, and ownership are restored or recreated.
  - confidence: 0.9
