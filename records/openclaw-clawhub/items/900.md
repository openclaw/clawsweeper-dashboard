## Summary

Keep open. Current main still blocks same-GitHub sign-in after self-deletion, and resolving the requested recovery/relink behavior requires maintainer product and security policy rather than conservative cleanup closure.

## What This Changes

The issue requests a way for a deleted ClawHub account to sign in again with the same GitHub identity and regain account features such as CLI tokens.

## Best Possible Solution

Choose one deleted-account policy, then enforce it consistently across OAuth callback handling, active-user gates, auth identity binding, token creation, settings/header/CLI UI, docs, audit behavior, and regression tests.

## Reproduction Assessment

Yes, by source inspection. Delete a GitHub-linked account so `deactivatedAt` and `purgedAt` are set, then sign in with the same GitHub identity; current main rejects OAuth reactivation, leaves `users.me` null, and prevents CLI token creation.

## Solution Assessment

Unclear. Same-GitHub restore/relink/recreate may be the desired user outcome, but it changes the documented irreversible-deletion contract and should wait for a maintainer-approved security and privacy policy.
