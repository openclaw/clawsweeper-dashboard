## Summary

Keep open. Current main still rejects deleted/deactivated GitHub identities during OAuth sign-in and keeps CLI token access behind active-user checks; the desired restore, relink, or recreate behavior conflicts with the documented irreversible-deletion policy and needs maintainer product/security judgment.

## What This Changes

Request that a deleted GitHub-linked ClawHub account can sign back in or be restored/relinked instead of remaining logged out with CLI token access blocked.

## Best Possible Solution

Make one maintainer-approved deleted-account policy and enforce it consistently across OAuth callback handling, active-user gates, token creation, settings/header/CLI UI, docs, and regression tests.

## Reproduction Assessment

Yes, by source inspection. Delete a GitHub-linked account so `deactivatedAt` and `purgedAt` are set, then sign in with the same GitHub identity; current main rejects reauth, leaves `users.me` null, and blocks CLI token creation.

## Solution Assessment

Unclear. Restoring, relinking, or recreating the account may be the right user outcome, but the maintainable fix depends on the deletion/privacy policy; preserving irreversibility would instead need clearer OAuth, settings, CLI, and support messaging.
