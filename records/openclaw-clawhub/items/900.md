## Summary

Keep open. Current main still treats self-deleted/deactivated GitHub accounts as inactive and blocks the same GitHub identity during OAuth re-login, and ClawHub review policy keeps issues open unless maintainers explicitly resolve them.

## What This Changes

The issue asks ClawHub to let a user who deleted their GitHub-backed account sign in again and regain access to account features such as CLI token creation.

## Best Possible Solution

Maintainers should choose one deleted-account identity policy, then implement it consistently across OAuth callback handling, active-user gates, CLI token creation, settings/header/CLI UX, operator recovery tooling if needed, docs/specs, and regression tests.

## Reproduction Assessment

Yes, at source level. Current main sets deactivatedAt/purgedAt on self-deletion, rejects the same identity during OAuth, returns no active users.me, and gates CLI token creation on an active user.

## Solution Assessment

Unclear until maintainers choose the policy. A direct restore/relink/recreate implementation would not be the best next step without deciding how irreversible deletion, identity reuse, token revocation, and ownership history should work.

## Security Review

Status: needs_attention

Summary: Deleted-account recovery is security-sensitive because it changes identity reuse, deletion privacy, token revocation, and ownership semantics.

Concerns:

- **[medium] Define deleted-account recovery policy first:** `specs/security-moderation.md:130`
  - body: The spec currently says user-initiated deletion is irreversible and deleted accounts cannot be restored by logging in again, so restore, relink, or re-registration needs explicit maintainer approval.
  - confidence: 0.95
- **[medium] Preserve identity and ban safeguards:** `convex/auth.ts:37`
  - body: The OAuth callback blocks deleted/deactivated identities and checks existingUserId before legacy migration, so any recovery path needs equivalent GitHub identity, ban, and autoban protections.
  - confidence: 0.9
- **[medium] Define token and ownership semantics:** `convex/users.ts:338`
  - body: Account deletion revokes API tokens and purges profile/contact fields while published content remains public, so recovery must define whether tokens, audit history, publishers, and ownership are restored or recreated.
  - confidence: 0.9
