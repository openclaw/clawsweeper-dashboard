## Summary

Keep open. Current main still matches the report: self-deleted/deactivated GitHub identities are blocked during OAuth reauth, active-user lookups return no user, the header falls back to the GitHub sign-in button, and CLI token creation remains gated on an active user. ClawHub cleanup policy also keeps issues open, and the related reports are open rather than merged or superseded.

## What This Changes

Handle deleted ClawHub accounts attempting to sign in again with GitHub so users do not get stuck in a logged-out/sign-in-loop state and can regain CLI token access when policy allows it.

## Best Possible Solution

Make one explicit deleted-account policy and implement it consistently across OAuth callback handling, active-user gates, token creation, settings/header/CLI UI, management recovery paths, audit logging, and docs, with regression tests for self-deleted re-login and CLI auth behavior.
