## Summary

Keep open. Current main still deliberately blocks self-deleted GitHub identities from becoming active users again, and the remaining request is a security-sensitive account-recovery policy decision rather than a narrow bug fix; ClawHub cleanup policy also keeps issues open unless maintainers decide otherwise.

## What This Changes

User reports that after deleting a ClawHub account, GitHub OAuth returns to the app but leaves them unable to re-login or create CLI tokens.

## Best Possible Solution

Finalize the account-recovery policy, then either implement an audited restore/relink/recreate path end to end or preserve irreversible deletion with clear OAuth, CLI, settings, docs, and support messaging.

## Reproduction Assessment

Yes, at source level. Current main deactivates and purges the user on self-deletion, rejects the same identity during OAuth, returns no active users.me, and gates CLI token creation on an active user.

## Solution Assessment

No autonomous code change is the best next step. The requested restore, relink, or recreate behavior changes a security-sensitive irreversible-deletion policy, so maintainers should decide the policy before implementation.

## Security Review

Status: needs_attention

Summary: Deleted-account recovery affects identity reuse, deletion privacy, token revocation, and ownership semantics.

Concerns:

- **[medium] Define deleted-account recovery policy first:** `specs/security-moderation.md:136`
  - body: The spec currently says user-initiated deletion is irreversible and cannot be restored by logging in again, so recovery, relinking, or re-registration needs explicit policy approval before code changes.
  - confidence: 0.95
- **[medium] Preserve OAuth identity and ban safeguards:** `convex/auth.ts:37`
  - body: The OAuth callback blocks deleted/deactivated identities and checks existingUserId before migrating legacy deleted records; any recovery path needs equivalent identity-matching and ban/autoban protections.
  - confidence: 0.9
- **[medium] Define token and ownership recovery semantics:** `convex/users.ts:338`
  - body: Account deletion revokes API tokens and purges profile/contact fields while published content may remain public, so recovery must define whether tokens, audit history, publishers, and ownership are restored or recreated.
  - confidence: 0.9
