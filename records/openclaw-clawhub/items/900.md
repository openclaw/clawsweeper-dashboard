## Summary

Keep open. Current main still treats owner-deleted accounts as inactive and blocks the same GitHub identity from becoming an active user again; related reports remain open, and the remaining fix is a product/security account-recovery decision rather than conservative cleanup.

## What This Changes

Requests a deleted-account recovery/relink/recreate path after GitHub OAuth so the user can sign in and create CLI tokens again.

## Best Possible Solution

Choose and document one deleted-account policy, then enforce it consistently across OAuth, active-user gates, CLI token creation, settings/header messaging, support/admin tooling, docs/specs, and regression tests.

## Reproduction Assessment

Yes, at source level: deletion writes `deactivatedAt`/`purgedAt`, OAuth rejects deleted or deactivated users, active-user helpers return no user, and token creation requires an active user. I did not run a live production OAuth deletion flow.

## Solution Assessment

No autonomous code change is the best next step yet. Maintainers first need to decide whether deleted GitHub identities may be restored, unlinked, recreated, or routed through support only.
