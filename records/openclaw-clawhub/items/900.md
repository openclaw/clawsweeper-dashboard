## Summary

Keep open. Current main still intentionally blocks deleted/deactivated GitHub identities from becoming active users again, related same-symptom account recovery reports remain open, and the remaining decision is security/product policy rather than conservative cleanup.

## What This Changes

The issue requests that a user who deleted their ClawHub account can sign in again with GitHub, regain an active account state, and create CLI tokens.

## Best Possible Solution

Define one maintainer-approved deleted-account policy, then enforce it consistently across OAuth, auth identity binding, active-user gates, token creation, settings/header/CLI messaging, specs/docs, and regression tests.

## Reproduction Assessment

Yes, at source level. Deletion writes deactivatedAt/purgedAt, OAuth rejects that state, users.me returns null through the active-user gate, and CLI token creation requires requireUser; I did not run a live production OAuth deletion flow.

## Solution Assessment

No autonomous code change is the best next step yet. Restore, relink, recreate, or support-only recovery semantics need a maintainer security/product decision before implementation.

## Security Review

Status: needs_attention

Summary: Deleted-account recovery is security-sensitive because it changes identity reuse, deletion privacy, token revocation, and audit semantics.

Concerns:

- **[medium] Define recovery authority before changing sign-in:** `specs/security-moderation.md:139`
  - body: The current spec says owner-initiated deletion is irreversible while RFC 0001 leaves recovery open, so OAuth restoration, relinking, or re-registration needs explicit policy approval before implementation.
  - confidence: 0.95
- **[medium] Preserve identity and ban safeguards:** `convex/auth.ts:37`
  - body: The OAuth helper verifies the incoming identity state and checks ban audit records before rejecting legacy deleted users; any recovery path needs equivalent bypass prevention and abuse handling.
  - confidence: 0.9
- **[medium] Define token and privacy semantics:** `convex/users.ts:338`
  - body: Account deletion revokes API tokens and clears profile, contact, and GitHub-derived fields, so recovery must define whether credentials, publisher links, audit history, and personal data are restored or recreated.
  - confidence: 0.9
