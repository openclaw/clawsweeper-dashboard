## Summary

Keep open. Current main still blocks self-deleted/deactivated GitHub identities from becoming active users again, while the requested recovery/relink behavior conflicts with the current irreversible-deletion policy and needs maintainer product/security judgment.

## What This Changes

Requests that a GitHub identity tied to a self-deleted ClawHub account be able to sign in, recreate, or relink so the web UI and CLI token flow become usable again.

## Best Possible Solution

Maintainers should choose one deleted-account policy, then enforce it consistently across OAuth callback handling, active-user gates, token creation, settings/header/CLI UI, docs/specs, and regression tests.

## Reproduction Assessment

Yes, at source level. Deletion writes deactivatedAt/purgedAt, OAuth reauth throws for deleted/deactivated users, active-user helpers return no user, and token creation requires an active user; I did not run a live production OAuth deletion flow.

## Solution Assessment

No autonomous patch is the best next step yet. Deleted-account restoration, relinking, recreation, or support-only handling changes an auth identity boundary and needs maintainer product/security policy first.
