## Summary

Keep open. Current main still intentionally blocks self-deleted/deactivated GitHub identities from becoming active users again, and ClawHub policy keeps issues open unless maintainers resolve the product/security decision.

## What This Changes

The issue asks ClawHub to let a user sign in again with the same GitHub identity after deleting their ClawHub account, or otherwise resolve the apparent sign-in loop and CLI-token lockout.

## Best Possible Solution

Choose one explicit deleted-account policy, then implement it consistently across OAuth callback handling, active-user gates, token creation, settings/header/CLI UI, audit logging, docs, and regression tests.

## Reproduction Assessment

Yes. The source-level path is high confidence: `deleteAccount` sets `deactivatedAt`, OAuth reauth throws the deleted-account error, `users.me` returns `null`, and the CLI/header flows require an active user doc.

## Solution Assessment

Unclear. The requested recreate/relink behavior may be right, but the maintainable fix first needs a product/security decision about whether deletion remains irreversible or becomes recoverable under audited conditions.
