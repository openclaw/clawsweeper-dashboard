## Summary

Keep open. Current main still enforces irreversible self-deletion and blocks the same deleted/deactivated GitHub identity from becoming an active user again, while the requested restore/relink/recreate behavior needs maintainer product and security policy.

## What This Changes

The issue requests that a GitHub identity tied to a self-deleted ClawHub account can log in again or recover/relink the account instead of looping with the sign-in button still shown.

## Best Possible Solution

End with one explicit deleted-account policy, then enforce it consistently across OAuth callback handling, active-user gates, token creation, account/settings/header/CLI UI, docs/specs, support/admin tooling, and regression tests.

## Reproduction Assessment

Yes, at source level. Deletion writes `deactivatedAt`/`purgedAt`, OAuth rejects deleted/deactivated users, active-user helpers return no user, and token creation requires an active user; I did not run a live production OAuth deletion flow.

## Solution Assessment

No autonomous reactivation patch is the best next step yet. Maintainers first need to decide whether deleted GitHub identities may be restored, unlinked, recreated, or routed through support only.
