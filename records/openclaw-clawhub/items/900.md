## Summary

Keep open. Current main still blocks deleted/deactivated GitHub identities from becoming active users again, and the requested recreate/relink behavior conflicts with the documented irreversible-deletion policy, so this needs maintainer product and security judgment rather than cleanup closure.

## What This Changes

Allow or otherwise handle re-login/re-registration for a GitHub identity after its ClawHub account was deleted, including CLI token access and sign-in UI behavior.

## Best Possible Solution

Adopt one deleted-account policy and enforce it consistently across OAuth callback handling, active-user gates, token creation, settings/header/CLI UI, docs, auditability, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: delete an account so `deactivatedAt`/`purgedAt` are set, sign in again with the same GitHub identity, then OAuth rejects reauth, `users.me` stays `null`, and CLI token creation never starts.

## Solution Assessment

Unclear. Automatic restore/relink/recreate may solve the user problem, but it is not clearly the best fix until maintainers choose the deletion, identity-binding, audit, and token policy.
