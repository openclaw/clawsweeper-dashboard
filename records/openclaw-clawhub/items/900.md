## Summary

Keep open. Current main still blocks deleted/deactivated GitHub identities from becoming active users again, which matches the reported sign-in loop and CLI-token lockout; ClawHub policy also keeps issues open unless a maintainer resolves the product/security decision.

## What This Changes

After deleting a ClawHub account, signing in again with the same GitHub identity should restore, relink, or recreate access instead of leaving the user logged out with CLI tokens unavailable.

## Best Possible Solution

Make one explicit deleted-account policy, then implement it consistently across OAuth callback handling, active-user gates, token creation, header/settings/CLI UI, audit logging, docs, and regression tests.

## Reproduction Assessment

Yes. The source-level path is high confidence: delete an account to set `deactivatedAt`, attempt GitHub OAuth reauth, then observe `users.me` returning `null`, the header showing sign-in, and CLI token creation never starting.

## Solution Assessment

Unclear. The requested restore/relink/recreate behavior may be correct, but the maintainable fix first needs a product/security decision about whether deletion remains irreversible or becomes recoverable under audited conditions.
