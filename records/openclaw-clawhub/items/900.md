## Summary

Keep open. Current main still enforces irreversible self-deletion for GitHub-backed users, and the requested re-login/relink behavior remains a security-sensitive account-lifecycle decision with several related open reports.

## What This Changes

The issue asks ClawHub to let a GitHub-backed account become usable again after self-deletion, or avoid the live sign-in loop and CLI-token lockout.

## Best Possible Solution

Maintainers should choose one deleted-account identity policy and then implement it consistently across OAuth, active-user gates, token creation, settings/header/CLI UX, docs/specs, and regression tests.

## Reproduction Assessment

Yes, at source level. Current main deactivates and purges the user on self-deletion, rejects the same identity during OAuth, returns no active users.me, and gates CLI token creation on an active user.

## Solution Assessment

No autonomous code change is the best next step yet. The requested restore/relink/recreate behavior changes a security-sensitive irreversible-deletion policy, so maintainers should decide the policy first.

## Security Review

Status: needs_attention

Summary: Deleted-account recovery is security-sensitive because it changes identity reuse, deletion privacy, token revocation, and ownership semantics.

Concerns:

- **[medium] Define deleted-account recovery policy first:** `specs/security-moderation.md:132`
  - body: The spec currently says user-initiated deletion is irreversible and deleted accounts cannot be restored by logging in again, so restore, relink, or re-registration needs explicit maintainer approval.
  - confidence: 0.95
- **[medium] Preserve OAuth identity and ban safeguards:** `convex/auth.ts:37`
  - body: The OAuth callback blocks deleted/deactivated identities and checks existingUserId before migrating legacy deleted records, so any recovery path needs equivalent GitHub identity and ban/autoban protections.
  - confidence: 0.9
- **[medium] Define token and ownership semantics:** `convex/users.ts:338`
  - body: Account deletion revokes API tokens and purges profile/contact fields while published content remains public, so recovery must define whether tokens, audit history, publishers, and ownership are restored or recreated.
  - confidence: 0.9
