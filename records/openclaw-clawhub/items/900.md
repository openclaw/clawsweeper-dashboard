## Summary

Keep open. Current main still intentionally blocks deleted/deactivated GitHub identities from becoming active users again, while the requested restore/relink/recreate behavior is an auth identity policy decision with security implications.

## What This Changes

Allow or clearly handle sign-in after a user deletes their ClawHub account, including GitHub OAuth relink/recreate/restoration and CLI token access.

## Best Possible Solution

Make one explicit deleted-account identity policy, then enforce it consistently across OAuth callback handling, active-user gates, token creation, settings/header/CLI UI, docs/specs, support/admin tooling, and regression tests.

## Reproduction Assessment

Yes, at source level. Deletion writes deactivatedAt/purgedAt, OAuth rejects deleted/deactivated users, active-user helpers return no user, and token creation requires an active user; I did not run a live production OAuth deletion flow.

## Solution Assessment

No autonomous reactivation patch is the best next step yet. Maintainers first need to decide whether deleted GitHub identities may be restored, unlinked, recreated, or routed through support only.
