## Summary

Keep open. Current main still intentionally blocks deleted/deactivated GitHub identities from becoming active users again, while the requested restore/relink/recreate behavior conflicts with the documented irreversible-deletion policy and needs maintainer product/security judgment.

## What This Changes

The issue asks for users who deleted their ClawHub account to be able to sign back in with the same GitHub identity and regain access to account features such as CLI tokens.

## Best Possible Solution

Choose one deleted-account identity policy, then enforce it consistently across OAuth callbacks, active-user gates, token creation, settings/header/CLI UI, support messaging, docs, and regression tests.

## Reproduction Assessment

Yes, at source level. A self-deleted account gets deactivatedAt/purgedAt, OAuth reauth throws the permanent-deletion error, users.me returns null through active-user gates, and CLI token creation requires requireUser; I did not run a live OAuth flow with the reporter's GitHub identity.

## Solution Assessment

No, not as an autonomous code change. The narrow maintainable path is a maintainer-approved deletion/restoration policy first, followed by implementation and tests for either restore/relink/recreate or clearer irreversible-deletion support routing.
