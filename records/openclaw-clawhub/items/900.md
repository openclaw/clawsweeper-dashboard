## Summary

Keep open. Current main still blocks deleted/deactivated GitHub identities from becoming active users again, and the fix requires maintainer product/security judgment around deletion recovery, identity relinking, and token issuance.

## What This Changes

Handle re-login after a user deletes their ClawHub account, including the GitHub OAuth loop, sign-in button state, and CLI token access.

## Best Possible Solution

Set one explicit deleted-account recovery policy, then implement it consistently across OAuth handling, active-user gates, token creation, settings/header/CLI UI, docs, and regression tests.

## Reproduction Assessment

Yes. The source path is high confidence: delete an account to set `deactivatedAt`, sign in again with the same GitHub identity, then `users.me` returns `null`, the header shows sign-in, and CLI token creation never starts.

## Solution Assessment

Unclear. The requested restore/relink/recreate behavior may be right, but current docs promise irreversible deletion, so maintainers need to choose the account-recovery policy before implementation.
