## Summary

Keep open. Current main still blocks deleted/deactivated GitHub identities from becoming active users again, and the requested restore/relink/recreate behavior needs a maintainer product and security decision before code should change.

## What This Changes

The issue requests that a self-deleted ClawHub account can sign in again with GitHub and regain access to account features such as CLI tokens instead of remaining in a sign-in loop.

## Best Possible Solution

End with one explicit deleted-account identity policy enforced across OAuth, active-user gates, token creation, header/CLI UI, docs/specs, and regression tests.

## Reproduction Assessment

Yes, at source level. The current code path writes `deactivatedAt`/`purgedAt`, blocks deleted/deactivated users during OAuth and active-user lookup, and prevents CLI token creation without an active user; I did not run a live production OAuth deletion flow.

## Solution Assessment

No autonomous restore/relink patch is the best next step yet. Maintainers first need to decide whether deleted GitHub identities may be restored, unlinked, recreated, or routed through support only.
