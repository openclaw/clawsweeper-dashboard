## Summary

Keep open. Current main still blocks self-deleted/deactivated GitHub identities from becoming active users again, which matches the reported sign-in/CLI-token lockout, and the ClawHub cleanup policy keeps issues open unless maintainers resolve the product/security decision.

## What This Changes

The issue requests that a GitHub-linked account deleted in ClawHub can log in again, be restored or relinked, and regain CLI-token access instead of staying in a sign-in loop.

## Best Possible Solution

Maintainers should choose one deleted-account policy and enforce it consistently across OAuth callback handling, active-user gates, token creation, settings/header/CLI UI, audit logging, docs, and regression tests.

## Reproduction Assessment

Yes. The source-level path is high confidence: delete an account so `deactivatedAt` is set, sign in again with the same GitHub identity, then `users.me` remains `null`, the header still renders sign-in, and CLI token creation never starts.

## Solution Assessment

Unclear. The requested restore/relink/recreate behavior may be the right product outcome, but the narrow maintainable fix first requires a policy and security decision about irreversible deletion, GitHub identity binding, auditability, and token access.
