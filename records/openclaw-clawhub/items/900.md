## Summary

Keep open. Current main still intentionally blocks self-deleted/deactivated GitHub identities from becoming active users again, and the remaining decision is security/product policy rather than a safe cleanup close or autonomous fix.

## What This Changes

The issue asks ClawHub to allow a deleted account's GitHub OAuth identity to log in again by restoring, relinking, or recreating the account instead of leaving the user in a signed-out/sign-in-button state.

## Best Possible Solution

Make one explicit deleted-account policy decision, then enforce it consistently across OAuth callback handling, active-user gates, API token creation, settings/header/CLI UI, operator docs, specs, and regression tests.

## Reproduction Assessment

Yes, from source inspection rather than a live account: self-deletion marks the user `deactivatedAt`, OAuth rejects that state, active-user helpers return no `me` document, and token creation requires `requireUser`. That is a high-confidence source reproduction of the reported login loop and CLI-token lockout.

## Solution Assessment

No, the requested restore/relink behavior should not be implemented ad hoc until maintainers decide whether deletion remains irreversible or a deliberate restore/recreate flow is allowed. The maintainable fix is a policy-backed backend and UI change with tests, not a sign-in-button-only patch.
