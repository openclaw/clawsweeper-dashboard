## Summary

Current main intentionally limits Official status to ClawHub-managed publisher records and provides no public application endpoint, so the reporter’s central question remains unresolved. The related requests at https://github.com/openclaw/clawhub/issues/1301 and https://github.com/openclaw/clawhub/issues/184 confirm this is a broader trust-policy decision, not an implemented self-service workflow or a narrow code bug.

## What This Changes

Clarify how a community publisher can obtain a verified or official trust designation for a published ClawHub skill and plugin.

## Best Possible Solution

Adopt one documented publisher-trust policy that distinguishes automated artifact verification from Official publisher designation and states eligibility, application, review scope, expiry or revocation, appeals, and catalog presentation.

## Reproduction Assessment

Not applicable. This issue asks how to obtain a trust designation; it does not report runtime behavior that can be reproduced against current main.

## Solution Assessment

No. Granting one publisher a badge ad hoc would leave the same ambiguity for everyone else; the maintainable solution is a security-reviewed, documented policy and intake path shared with the broader related discussion.
