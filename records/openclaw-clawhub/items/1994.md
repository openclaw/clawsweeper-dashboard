## Summary

Keep open. Current main still exposes plugin trust through official/community/private channels, release verification tiers, scan/moderation state, and the Verified UI tied to `isOfficial`; it does not provide a documented public path for a third-party community plugin like Spectyra to receive a separate Verified-style status. The broader policy question is still open in https://github.com/openclaw/clawhub/issues/2272.

## What This Changes

The issue asks how the linked Spectyra skill and plugin can receive Verified-style marketplace status so users can discover and trust them.

## Best Possible Solution

Keep Spectyra on the current community/source-linked/scan/readiness signals until maintainers resolve the vetted-community policy, then evaluate the linked skill and plugin against that documented security and trust bar.
