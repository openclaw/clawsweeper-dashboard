## Summary

Current main does not provide a public community application path for an Official or human-verified badge. It implements separate artifact verification tiers and a ClawHub-managed Official designation, while the broader verification-policy question remains open at https://github.com/openclaw/clawhub/issues/1301; under the ClawHub cleanup policy this issue should stay open for maintainer consolidation rather than be auto-closed.

## What This Changes

Request a documented route to verified status for an existing ClawHub skill and code plugin so users can recognize it as trusted.

## Best Possible Solution

Use the broader publisher-verification discussion to define one public community policy covering eligibility, application and review ownership, badge wording, revocation, and the distinction between artifact scans and publisher endorsement, then document that workflow for skill and plugin authors.

## Reproduction Assessment

Not applicable. This issue requests a verification program and guidance rather than reporting broken behavior with a current-main reproduction path.

## Solution Assessment

No. Granting or documenting ad hoc verification for one publisher would create inconsistent trust semantics; the maintainable solution is a shared policy resolved with https://github.com/openclaw/clawhub/issues/1301.
