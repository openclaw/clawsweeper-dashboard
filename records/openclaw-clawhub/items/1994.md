## Summary

Keep open. Current main has official-publisher badges, source/provenance verification tiers, trusted-publisher publish plumbing, readiness checks, and scan status, but it still does not expose a documented public path for a third-party community plugin like Spectyra to obtain Verified-style status; the broader vetted community plugin policy is still open in https://github.com/openclaw/clawhub/issues/2272.

## What This Changes

The issue asks how a third-party ClawHub publisher can get a published skill and plugin marked Verified.

## Best Possible Solution

Define and document a vetted community plugin review path, or explicitly state that no such path exists, then apply that policy consistently to Spectyra and similar plugins.

## Reproduction Assessment

Not applicable; this is a marketplace verification/admin request rather than a reproducible bug. Source and docs inspection shows current main exposes official, source/provenance, trusted-publisher, readiness, and scan signals but no public third-party Verified request flow.

## Solution Assessment

No; a one-off badge or approval is not the best maintainable solution without a documented trust policy. The safer path is to resolve the vetted-community-plugin RFC and make the criteria, badge semantics, and review authority explicit.
