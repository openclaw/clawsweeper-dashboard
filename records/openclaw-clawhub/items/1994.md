## Summary

Current main provides official-publisher status, provenance tiers, automated scans, and moderation signals, but no application path for community verification; the unresolved policy is now more comprehensively framed by the open vetted-community RFC at https://github.com/openclaw/clawhub/issues/2272.

## What This Changes

Provide a documented way for published community skills and plugins to receive a verified trust designation on ClawHub.

## Best Possible Solution

Resolve the vetted-community RFC with distinct public signals for publisher identity, source provenance, automated scans, manual audit, and official endorsement, plus documented eligibility, application, expiry, revocation, and appeal rules.
