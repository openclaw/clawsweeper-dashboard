## Summary

Keep open for maintainer product and security-policy review. Current main supports automated package verification signals and a separately managed Official publisher designation, but it provides no public application path for an arbitrary community publisher; the broader community-verification policy is also being discussed in https://github.com/openclaw/clawhub/issues/1301 and https://github.com/openclaw/clawhub/issues/184.

## What This Changes

Provide a clear path for a published community skill and plugin to obtain a verified trust designation.

## Best Possible Solution

Define one documented community verification model with explicit eligibility, evidence, scope, review, appeal, expiration, and revocation rules, while clearly separating it from automated artifact verification and the existing Official publisher designation.
