## Summary

Keep open: current main does not implement the role-aware moderator/admin CLI help behavior, and the PR is useful, but the proposed patch introduces a version-command regression and is missing the required changelog entry.

## What This Changes

The PR adds role-aware CLI help audiences so logged-out/user help hides moderator/admin root, package, and trusted-publisher commands while tests cover filtered typo/error help.

## Best Possible Solution

Keep the role-aware help design, but exempt CLI version flags from role lookup and add the missing Unreleased changelog entry before merge.

## Reproduction Assessment

Yes from source inspection: the PR's option-only fallback treats `clawhub -V` and `clawhub --cli-version` as help, then `resolveHelpRole` can call `/api/v1/whoami`. I did not execute the PR branch because this review is read-only.

## Solution Assessment

No as currently patched: the role-aware audience split fits the existing admin help gate, but classifying every unknown option-only invocation as help is too broad. A narrow version-flag bypass plus changelog entry is the safer fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep version flags out of help-role lookup:** `packages/clawhub/src/cli/adminHelp.ts:135`
  - body: The new fallback treats any option-only invocation that is not `--help`/`-h` as a help request. Since `cli.ts` registers `-V, --cli-version` as local version output, a logged-in `clawhub -V` or `clawhub --cli-version` now reads config and calls whoami before printing the version, which can delay an offline/local command and unnecessarily sends the token to the registry. Exempt the version flags or classify them as known non-help requests.
  - confidence: 0.88
- **[P3] Add the required changelog entry:** `CHANGELOG.md:5`
  - body: This is a user-facing CLI fix, but the diff does not update `CHANGELOG.md`. Repo policy requires user-facing fixes to be listed under `Unreleased`, so please add a concise CLI fix entry before merge.
  - confidence: 0.9
