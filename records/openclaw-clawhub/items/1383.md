## What This Changes

The issue requests a ClawHub CLI/API registry discovery path for exact slug lookup and slug-prefix/list-style enumeration, rather than only ranked search.

## Best Possible Solution

Add a first-party, indexed registry discovery path that preserves ranked search semantics while exposing exact slug lookup and slug-prefix enumeration through the public API, CLI, docs, and focused regression tests.

## Repair Work Prompt

Implement a narrow registry discovery path for skills. Preserve existing ranked search behavior; do not make `/api/v1/search` claim complete enumeration unless a distinct mode is added. Add exact slug lookup and slug-prefix enumeration support through an indexed backend path, preferably using `skillSearchDigest` normalized slug indexes with cursor pagination and no full-table scans. Expose the capability through the ClawHub CLI in a backward-compatible way, such as `clawhub search --exact <slug>` and `clawhub search --prefix <prefix>` or an equivalent clearly documented hub-list mode. Include JSON output support where the surrounding command surface already supports it. Update public API/CLI schemas and docs, add regression tests for CLI request construction, HTTP handler parameter forwarding/validation, backend exact/prefix pagination, empty results, limit/cursor behavior, and nonSuspicious filtering. Do not change unrelated package/plugin search behavior or ranked search scoring beyond what the new discovery mode requires.
