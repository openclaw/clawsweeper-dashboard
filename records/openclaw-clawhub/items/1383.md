## Summary

Current main still lacks reliable CLI enumeration by slug prefix or exact match. Ranked `search`, local-only `list`, and single-page `explore` do not solve the requested registry-discovery workflow, and no canonical replacement PR or issue was found.

## What This Changes

Add deterministic prefix, exact-slug, or complete paginated registry discovery to the ClawHub CLI.

## Best Possible Solution

Preserve relevance-ranked `search` and add one backward-compatible registry enumeration surface with exact resolution, indexed prefix filtering, cursor traversal, and documented JSON output.

## Reproduction Assessment

Yes, at source level: current main’s `search` is relevance-ranked and limited, `explore` lacks exact or prefix filtering, and `list` only covers local installations.

## Solution Assessment

The requested capability is appropriate, but the proposed commands are alternatives rather than a single implementation plan; maintainers should select one stable CLI/API contract and indexed filtering strategy.
