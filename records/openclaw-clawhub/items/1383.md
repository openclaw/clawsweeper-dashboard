## Summary

Keep open. Current main improves literal recall within ranked search, but it still does not provide a deterministic, pageable exact/prefix registry enumeration path, and no canonical implementation item now owns that remaining work.

## What This Changes

Add a deterministic CLI/API path for exact, prefix, or pageable registry skill enumeration rather than relying only on relevance-ranked search.

## Best Possible Solution

Provide a documented cursor-pageable registry listing API with indexed exact and slug-prefix filters, then expose it through a clear CLI list or explicit search mode while preserving relevance-ranked search semantics.

## Reproduction Assessment

Yes at source level: current main exposes ranked and capped discovery paths but no complete exact/prefix enumeration contract. The command options, candidate caps, and documentation are directly verifiable in current source.

## Solution Assessment

No single syntax proposed by the issue is established as best yet, and overloading ranked search alone would conflate relevance with completeness. A dedicated pageable enumeration API, surfaced through an explicit CLI mode, is the narrowest maintainable direction.
