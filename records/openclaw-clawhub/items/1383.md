## Summary

Keep open. This is a feature request, and the ClawHub cleanup policy keeps issues open; current main still exposes skill search as ranked/bounded discovery rather than deterministic exact-slug, slug-prefix, or hub-list enumeration.

## What This Changes

Request deterministic CLI/API registry discovery for skills by exact slug, slug prefix, or hub list mode instead of relevance-only search.

## Best Possible Solution

Add a narrow indexed discovery surface that preserves ranked search while exposing exact slug lookup and slug-prefix enumeration through CLI JSON output, public v1 API support, docs, and focused regression tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection confirms the current CLI/API expose ranked search and bounded browsing without exact or prefix enumeration inputs.

## Solution Assessment

Yes at the product-direction level: a separate indexed enumeration path is safer than redefining relevance search as complete. The unresolved part is which public CLI/API shape maintainers want to support.
