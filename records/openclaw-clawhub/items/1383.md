## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still lacks deterministic exact-slug or slug-prefix registry enumeration through the skill CLI/API path; the existing behavior is relevance search plus bounded browsing.

## What This Changes

The issue requests deterministic CLI/API registry discovery for exact slugs and slug prefixes instead of relying only on ranked skill search.

## Best Possible Solution

Add a narrow indexed registry discovery surface that preserves ranked search while exposing exact slug lookup and slug-prefix enumeration through CLI JSON output, public v1 API support, docs, and focused regression tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection confirms current main exposes relevance search and bounded browsing, not exact or prefix enumeration inputs.

## Solution Assessment

Yes at the direction level: a separate indexed enumeration path is safer than redefining relevance search as complete. The unresolved part is which public CLI/API shape maintainers want to support.
