## Summary

Keep open. This is a feature request, and ClawHub cleanup policy keeps issues open; current main still lacks deterministic exact-slug or slug-prefix registry enumeration through the CLI/API, with only ranked search and bounded browse/list surfaces available.

## What This Changes

Add a registry-oriented ClawHub CLI/API discovery path for exact slug and slug-prefix skill enumeration instead of relying only on ranked search.

## Best Possible Solution

Add a narrow indexed discovery surface that preserves ranked search while exposing exact slug lookup and slug-prefix enumeration through CLI JSON output, public v1 API support, docs, and focused regression tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The gap is source-confirmed on current main because the CLI, HTTP handlers, docs, and backend search paths expose ranked or bounded browsing behavior without exact/prefix enumeration.

## Solution Assessment

Yes at the product-direction level: a separate indexed enumeration path is safer than redefining relevance search as complete. The unresolved part is the public CLI/API shape maintainers want to support.
