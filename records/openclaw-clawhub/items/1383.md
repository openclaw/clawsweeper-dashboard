## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still lacks a deterministic CLI/API exact-slug or slug-prefix enumeration path; ranked search injects literal recall but still sorts and slices results, while registry browsing only supports bounded sort pagination.

## What This Changes

The issue requests a registry-oriented CLI discovery path for exact slug lookup, slug-prefix enumeration, or hub listing instead of relying only on relevance-ranked search.

## Best Possible Solution

Add a first-party, indexed registry discovery surface that preserves ranked search while exposing exact slug lookup and slug-prefix enumeration through the public API, CLI JSON/text output, docs, and focused regression tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection on current main confirms the requested exact/prefix enumeration path is still missing from the CLI, HTTP v1 handlers, and backend list/search surfaces.

## Solution Assessment

Yes. The requested direction is a maintainable fit for core because terminal registry discovery needs first-party CLI/API support; the narrowest safe implementation is an additive indexed enumeration path rather than changing ranked search semantics.
