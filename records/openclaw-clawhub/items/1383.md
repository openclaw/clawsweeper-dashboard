## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still lacks deterministic skill exact-slug or slug-prefix enumeration; the available CLI/API paths are relevance-ranked search or bounded registry browsing, so this remains a feature request needing maintainer direction.

## What This Changes

Request deterministic ClawHub skill discovery by exact slug, slug prefix, or hub-list style CLI output instead of relying only on ranked search.

## Best Possible Solution

Add a backward-compatible indexed registry discovery surface that preserves ranked search while exposing exact slug lookup and slug-prefix enumeration through CLI JSON output, public v1 API support, docs, and focused regression tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection confirms current main exposes ranked search and bounded browse/list endpoints without deterministic exact or prefix enumeration inputs.

## Solution Assessment

Unclear at the public-interface level: an indexed prefix/exact discovery path is the right direction, but maintainers still need to choose the CLI/API shape before implementation.
