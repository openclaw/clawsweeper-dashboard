## Summary

Keep open. Current main still does not expose deterministic exact-slug or slug-prefix registry enumeration through the skill CLI/API path; the existing surfaces are relevance search or bounded registry browsing, and this new public CLI/API feature needs maintainer direction before implementation.

## What This Changes

The issue requests registry-oriented ClawHub CLI discovery for exact slug, slug prefix, or hub listing instead of relying only on ranked search.

## Best Possible Solution

Add a narrow indexed, cursor-paginated registry discovery surface that preserves ranked search while exposing exact-slug lookup and slug-prefix enumeration through CLI JSON/API support, docs, and focused tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection confirms the current CLI/API expose relevance search and bounded browsing without exact or prefix enumeration inputs.

## Solution Assessment

Yes at the direction level: a separate indexed enumeration path is the maintainable approach because ranked relevance search should not be redefined as a complete listing mechanism. The unresolved part is the exact public CLI/API shape maintainers want to support.
