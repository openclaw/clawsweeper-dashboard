## Repair Work Prompt

Implement deterministic registry skill discovery for exact slug and slug-prefix enumeration. Preserve the existing relevance-ranked `clawhub search <query>` behavior. Add a backward-compatible CLI surface, preferably `clawhub search --prefix <prefix>` and `clawhub search --exact <slug>` with JSON support where the command already has or needs machine-readable output; optionally add a hub listing mode only if it fits the existing CLI shape without changing local `clawhub list` defaults. Add public v1 API support for exact/prefix enumeration, using indexed Convex queries and cursor pagination over a lightweight skill projection; do not implement this with `.filter()` scans or by relying on vector search fallback. Update shared response schemas/types as needed, docs/cli.md and docs/http-api.md, and focused tests for CLI request construction/output, HTTP parameter forwarding/validation, and backend exact/prefix behavior. Coordinate with related PR #1256 only as context; do not depend on it as a complete solution.
