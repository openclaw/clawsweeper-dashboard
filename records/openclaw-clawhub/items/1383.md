## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main only adds prefix recall inside ranked search; it still lacks deterministic exact-slug, slug-prefix, or hub-list CLI/API enumeration.

## What This Changes

The request asks for registry-style CLI discovery by exact slug, slug prefix, or hub listing instead of relying only on ranked search results.

## Best Possible Solution

Add indexed cursor-paginated exact-slug and normalized slug-prefix registry discovery in the v1 API and CLI while preserving ranked search semantics.

## Reproduction Assessment

Yes. Static inspection of current main is enough: CLI search only sends `q` and `limit`, the public list API lacks slug filters, and the existing prefix path is capped ranked recall rather than enumeration.

## Solution Assessment

Yes, the requested direction is the right narrow fix if implemented as a separate deterministic discovery path. The safer approach is not to stretch ranked search, but to add explicit indexed exact/prefix listing controls with tests and docs.
