## Summary

Keep open. This is a feature request, and current main still exposes skill discovery as relevance search or bounded browsing rather than deterministic exact-slug, slug-prefix, or hub-list enumeration.

## What This Changes

The issue requests a CLI/API path for exact or prefix-based skill registry enumeration instead of relying on ranked search results.

## Best Possible Solution

Define one registry discovery contract, then add indexed cursor-paginated exact/prefix support through backend, public API, CLI, docs, and focused tests while preserving ranked search semantics.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that current main lacks the requested deterministic prefix/exact/list-style CLI and API discovery surface.

## Solution Assessment

Unclear until maintainers choose the public surface. The maintainable direction is a narrow indexed exact/prefix discovery path, not treating ranked relevance search as a complete listing mechanism.
