## Summary

Keep open. Current main and the latest v0.23.0 tag still expose ranked skill search plus bounded registry browsing, but not deterministic exact-slug or slug-prefix enumeration from the CLI/API.

## What This Changes

The issue requests deterministic prefix, exact, or list-style registry discovery in the ClawHub CLI instead of relying only on ranked search.

## Best Possible Solution

Add a documented, cursor-pageable registry discovery surface with indexed exact-slug and slug-prefix filters, then expose it through explicit CLI flags or a dedicated hub list mode while preserving ranked search semantics.

## Reproduction Assessment

Yes. Source inspection on current main shows CLI `search` only accepts query and limit, `list` is local-only, `explore` is bounded browsing, and the public API handlers lack exact or prefix enumeration inputs.

## Solution Assessment

Unclear until maintainers choose the public contract. The maintainable direction is an indexed, cursor-pageable exact/prefix registry API exposed through explicit CLI flags or a dedicated list mode, not stretching relevance-ranked search into an implicit total listing.
