## Summary

Keep open. Current main still improves recall inside ranked search, but it does not provide deterministic exact-slug or slug-prefix registry enumeration through the ClawHub CLI or public skills API.

## What This Changes

The issue requests a registry-oriented CLI discovery path for exact, prefix, or list-style skill enumeration instead of relying only on relevance-ranked search.

## Best Possible Solution

Add a documented, cursor-pageable registry discovery surface with indexed exact-slug and slug-prefix filters, then expose it through explicit CLI flags or a dedicated list mode while preserving ranked search semantics.

## Reproduction Assessment

Yes. Source inspection on current main shows ranked, capped search and sorted browsing paths, but no complete exact-slug or slug-prefix enumeration contract in the CLI or API.

## Solution Assessment

Unclear until maintainers choose the public contract. The narrow maintainable direction is indexed, cursor-pageable API support plus explicit CLI flags or a dedicated list mode, not stretching relevance-ranked search into an implicit total listing.
