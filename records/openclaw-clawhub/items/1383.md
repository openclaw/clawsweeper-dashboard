## Summary

Keep open. Current main still does not expose deterministic exact-slug or slug-prefix enumeration through the skill CLI/API path; recent search-prefix recall helps ranked search, but it is not a complete registry listing mode.

## What This Changes

The issue requests a registry-oriented ClawHub CLI discovery mode for exact skill lookup, slug-prefix enumeration, and hub listing rather than ranked search only.

## Best Possible Solution

Add a narrow registry discovery surface that preserves ranked search while exposing exact slug lookup and cursor-paginated slug-prefix enumeration through indexed backend queries, public v1 API support, CLI JSON output, docs, and focused tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection on current main confirms the relevant CLI/API paths lack exact or prefix enumeration inputs.

## Solution Assessment

Yes at the direction level: a separate indexed enumeration path is safer than redefining relevance search as complete. The unresolved part is which public CLI/API shape maintainers want to support.
