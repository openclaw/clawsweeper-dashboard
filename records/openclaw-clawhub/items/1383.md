## Summary

Keep open. Current main still provides ranked skill search and bounded registry browsing, but not deterministic exact-slug or slug-prefix enumeration for the CLI/API discovery workflow requested here.

## What This Changes

The issue requests deterministic registry-style skill discovery in the ClawHub CLI, such as prefix, exact, or hub listing modes.

## Best Possible Solution

Add a documented, indexed, cursor-pageable registry discovery surface for exact slug and slug-prefix enumeration, then expose it through explicit CLI flags or a dedicated hub list mode while preserving ranked search semantics.

## Reproduction Assessment

Yes. Source inspection on current main shows `search` and `/api/v1/search` are relevance-ranked and limit-sliced, while `explore` and `/api/v1/skills` are sorted browsing surfaces without exact or prefix filters.

## Solution Assessment

Unclear until maintainers choose the public contract. The narrow maintainable direction is indexed, cursor-pageable API support plus explicit CLI flags or a dedicated hub-list mode, not stretching relevance-ranked search into an implicit total listing.
