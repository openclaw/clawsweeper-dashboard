## Repair Work Prompt

Implement registry-oriented skill discovery for ClawHub. Preserve the existing ranked `search` behavior, and add a deterministic exact-slug and slug-prefix enumeration path through the CLI and public v1 API. A conservative implementation can expose `search --exact <slug>` and `search --prefix <prefix>` or an equivalent backward-compatible hub-list mode, but it must return complete paginated registry results rather than scored relevance results. Use existing `skillSearchDigest` normalized slug indexes, add cursor pagination where needed, and do not use `.filter()` or capped search recall for enumeration. Update CLI human/JSON output, API schemas, docs, tests, generated Convex API files if signatures change, and CHANGELOG.md. Do not change package/plugin catalog semantics or unrelated ranking logic.
