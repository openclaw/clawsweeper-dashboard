## Summary

Current main still provides relevance-ranked search and bounded registry browsing, but no reliable exact or prefix enumeration path, so the requested capability remains necessary and needs a maintainer product decision.

## What This Changes

Add exact, prefix-based, or complete registry skill enumeration to the ClawHub CLI instead of relying solely on relevance-ranked search.

## Best Possible Solution

Provide one documented, cursor-paginated registry listing surface with indexed exact and prefix filters, while retaining `search` for relevance-ranked discovery.

## Reproduction Assessment

Yes at source level: current CLI search is explicitly vector-ranked, while the browse path lacks exact or prefix filters and does not expose pagination, so neither guarantees complete family enumeration.

## Solution Assessment

No single proposed command is clearly best yet; the narrowest maintainable solution is a paginated listing contract with indexed exact and prefix filters rather than several overlapping discovery modes.
