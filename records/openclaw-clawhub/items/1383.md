## Summary

Keep open. Current main still lacks deterministic exact-slug or slug-prefix registry enumeration through the skill CLI/API, and this repository’s cleanup policy says ClawHub issues should stay open unless they are PRs already implemented on main.

## What This Changes

Request a registry-oriented CLI discovery path for exact or prefix-based skill enumeration instead of relying only on ranked search.

## Best Possible Solution

Add a first-party, indexed, cursor-paginated exact slug and slug-prefix registry discovery surface while preserving ranked search semantics.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The gap is source-confirmed because current main exposes ranked search and unfiltered browse/list paths, not deterministic exact or prefix enumeration.

## Solution Assessment

Yes on the requested direction, but the exact public surface still needs maintainer choice. The narrow maintainable implementation is a separate indexed enumeration/filter path rather than redefining ranked search as complete.
