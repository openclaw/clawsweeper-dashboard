## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still lacks deterministic exact-slug or slug-prefix enumeration through the CLI/API path; search has bounded prefix recall, but not a complete registry listing mode.

## What This Changes

The issue requests deterministic registry-style skill discovery in the ClawHub CLI, such as exact-slug lookup, slug-prefix enumeration, or hub listing.

## Best Possible Solution

Add a narrow registry discovery surface that preserves ranked search while exposing exact slug lookup and slug-prefix enumeration through indexed backend queries, public v1 API support, CLI JSON output, docs, and focused regression tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection confirms current-main search/explore/list are relevance search or bounded browsing without exact or prefix enumeration inputs.

## Solution Assessment

Yes at the direction level: a separate indexed enumeration path is safer than redefining relevance search as complete. The unresolved part is which public CLI/API shape maintainers want to support.
