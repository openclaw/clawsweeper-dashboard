## Summary

Current main improves literal search recall but still does not provide reliable complete prefix or exact registry enumeration: CLI search remains a bounded relevance query, and CLI explore reads only one capped browse page. The request remains useful and needs a maintainer decision on the permanent CLI/API contract.

## What This Changes

Requests a registry-oriented CLI discovery mode for exact, prefix, or complete hub skill enumeration.

## Best Possible Solution

Add an indexed, cursor-paginated slug enumeration contract and expose explicit exact/prefix modes through a clearly named CLI surface while preserving relevance search semantics.

## Reproduction Assessment

Yes, at source level. Current search is relevance-ranked and bounded, while explore consumes one capped page without cursor traversal or prefix filtering.

## Solution Assessment

Unclear until maintainers choose the public contract. A dedicated indexed enumeration path is more maintainable than redefining ordinary relevance search as an exhaustive listing operation.
