## Summary

Keep open. Current main still lacks deterministic exact-slug or slug-prefix registry enumeration for skills; the available CLI/API paths are relevance search or bounded browsing, and ClawHub cleanup policy keeps issues open.

## What This Changes

Request a registry-oriented CLI/API discovery path for exact skill slugs, slug prefixes, or hub-backed skill listing instead of relying on ranked search.

## Best Possible Solution

Add a narrow, indexed, cursor-paginated registry discovery surface for exact slug lookup and slug-prefix enumeration, with CLI JSON output, public API documentation, and focused regression tests while preserving ranked search semantics.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The gap is source-confirmed on current main because the CLI, HTTP handlers, docs, and backend search paths expose ranked or bounded browse behavior without exact/prefix enumeration.

## Solution Assessment

Yes at the product-direction level: a separate indexed enumeration path is safer than redefining relevance search as complete. The unresolved part is the public CLI/API shape maintainers want to support.
