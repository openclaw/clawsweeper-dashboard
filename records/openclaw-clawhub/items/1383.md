Request to add prefix, exact, or list-style registry discovery to the skill CLI and supporting API so users can enumerate skill families reliably.

## Best Possible Solution

Add a separate indexed registry enumeration surface with exact slug lookup and cursor-paginated slug-prefix listing through backend/API/CLI/docs/tests while leaving ranked search relevance intact.

## Solution Assessment

Yes at the direction level: a separate indexed enumeration path is safer than redefining ranked search or relying on bounded prefix recall. The exact public CLI/API shape still needs maintainer direction.
