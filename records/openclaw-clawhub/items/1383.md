## Summary

Keep open. This is a still-valid feature request: current main has ranked search and bounded browse/catalog paths, but no deterministic exact-slug or slug-prefix enumeration path in the skill CLI/API. ClawHub cleanup policy also keeps issues open.

## What This Changes

Request exact-slug and slug-prefix/list-style registry discovery in the ClawHub CLI/API instead of relying only on ranked search.

## Best Possible Solution

Add indexed, cursor-paginated exact and slug-prefix registry discovery in the backend and v1 API, then expose it through CLI human and JSON output while preserving ranked search.

## Solution Assessment

Yes. A first-party indexed exact/prefix discovery path is the narrow maintainable solution; capped ranked-search prefix recall should remain a recall improvement, not the listing API.
