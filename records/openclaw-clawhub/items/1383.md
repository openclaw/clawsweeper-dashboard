## Summary

Keep open. Current main and v0.20.0 still expose ranked skill search plus bounded registry browsing, but not a deterministic CLI/API contract for exact slug lookup, slug-prefix enumeration, or full hub listing.

## What This Changes

Request CLI support for exact, slug-prefix, or list-style registry skill discovery instead of relying only on relevance search.

## Best Possible Solution

Add a deterministic, paginated discovery path for exact slug and slug-prefix enumeration while preserving ranked search as the relevance-oriented command.

## Reproduction Assessment

Yes, from source inspection. Current main exposes `search <query...>` with only `--limit`, `list` for installed skills, and bounded `explore`, so the requested deterministic registry-discovery workflow is still missing.

## Solution Assessment

Unclear until maintainers choose the public contract. A separate deterministic, paginated mode or endpoint would be safer than changing ranked search semantics in place.
