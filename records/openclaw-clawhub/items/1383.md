## Summary

Keep open. ClawHub policy keeps issues open, and current main still has relevance-ranked skill search plus bounded browse/list paths rather than deterministic exact-slug or slug-prefix registry enumeration.

## What This Changes

The issue requests prefix, exact, or list-style registry discovery in the ClawHub CLI so users can enumerate skills instead of relying on ranked search.

## Best Possible Solution

Add a first-party, indexed registry discovery path that preserves ranked search semantics while exposing exact slug lookup and slug-prefix enumeration through the public API, CLI, docs, and focused regression tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The gap is source-confirmed on current main because search remains relevance-ranked and browse/list surfaces do not accept exact or prefix enumeration parameters.

## Solution Assessment

Yes, the requested direction is appropriate, but it should be implemented as an explicit enumeration path rather than redefining ranked search as complete. The narrowest maintainable solution is indexed exact/prefix support with CLI/API/docs/tests.
