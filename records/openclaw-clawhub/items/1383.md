## Summary

Keep open. Current main and the latest v0.23.0 release still expose relevance-ranked search and bounded registry browsing, but not deterministic exact-slug or slug-prefix enumeration through the ClawHub CLI/API.

## What This Changes

Request to add exact, prefix, or list-style registry discovery to ClawHub CLI skill search so users can enumerate matching skill slugs instead of relying on ranked search.

## Best Possible Solution

Add a documented, cursor-pageable registry discovery surface with indexed exact-slug and slug-prefix filters, then expose it through explicit CLI flags or a dedicated list mode while preserving ranked search semantics.

## Reproduction Assessment

Yes. Source inspection on current main shows CLI search only accepts query and limit, the registry list path only accepts limit/sort/json at the CLI layer, and the public list API only accepts limit/cursor/sort/nonSuspicious options.

## Solution Assessment

Unclear until maintainers choose the public contract. The narrow maintainable direction is indexed, cursor-pageable API support plus explicit CLI flags or a dedicated list mode, not stretching relevance-ranked search into an implicit total listing.
