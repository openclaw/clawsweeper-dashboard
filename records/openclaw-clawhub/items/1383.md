## Summary

Keep open. Current main improves exact and prefix recall inside relevance-ranked search, but it still does not expose deterministic, pageable exact-slug or slug-prefix registry enumeration through the CLI or public API, and no canonical replacement item owns that remaining product/API decision.

## What This Changes

The issue requests deterministic prefix, exact, or list-style registry discovery from the ClawHub CLI instead of only relevance-ranked skill search.

## Best Possible Solution

Add a documented, cursor-pageable registry discovery surface with indexed exact-slug and slug-prefix filters, then expose it through explicit CLI flags or a dedicated list mode while preserving relevance-ranked search semantics.

## Reproduction Assessment

Yes. Source inspection on current main shows ranked, capped search and sorted browsing paths, but no complete exact-slug or slug-prefix enumeration contract in the CLI or API.

## Solution Assessment

No. The current search recall improvements are useful but not the best solution for enumeration because they remain capped and ranked; the maintainable path is a deliberate indexed, pageable API/CLI contract.
