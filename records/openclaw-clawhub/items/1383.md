## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still lacks deterministic exact-slug or slug-prefix registry enumeration; the newer direct-prefix search recall only improves capped ranked search.

## What This Changes

The issue requests deterministic CLI registry discovery for exact slugs, slug prefixes, or hub listing instead of relying only on relevance-ranked skill search.

## Best Possible Solution

Add a narrow registry discovery surface that preserves ranked search while exposing exact slug lookup and slug-prefix enumeration through indexed, cursor-paginated backend queries, public v1 API support, CLI JSON/human output, docs, tests, and a changelog entry.

## Reproduction Assessment

Not applicable as a runtime bug, but the feature gap is high-confidence. Current-main inspection shows only relevance search and bounded browse/list paths, with no deterministic exact or prefix enumeration CLI/API.

## Solution Assessment

Yes, the requested direction is best solved in core CLI/API rather than as a plugin because it needs first-party registry query parameters and command semantics. The implementation should be a separate indexed enumeration path, not a change that makes ranked search pretend to be complete.
