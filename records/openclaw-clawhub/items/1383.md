## Summary

Keep open. This is a valid feature request, and current main still lacks a deterministic exact-slug, slug-prefix, or hub-list discovery contract in the ClawHub CLI/API; the existing surfaces remain ranked search plus bounded registry browsing.

## What This Changes

The issue requests deterministic CLI registry discovery for skills by exact slug, slug prefix, or hub list instead of relying only on ranked search.

## Best Possible Solution

Add a narrow registry discovery surface that preserves ranked search while exposing exact slug lookup and slug-prefix enumeration through indexed cursor-paginated backend queries, public v1 API support, CLI JSON output, docs, and focused tests.

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection gives high confidence current main lacks deterministic exact, prefix, or hub-list CLI/API discovery.

## Solution Assessment
