## Summary

Keep open. Current main still lacks deterministic exact-slug or slug-prefix registry enumeration through the ClawHub CLI/API; the available surfaces are ranked search, installed-local listing, or bounded registry browsing.

## What This Changes

The issue requests a registry-oriented ClawHub CLI discovery path for exact slug and slug-prefix skill enumeration instead of relying only on relevance-ranked search.

## Best Possible Solution

Add a narrow indexed registry discovery surface with exact slug lookup and cursor-paginated slug-prefix enumeration through backend queries, public v1 API support, CLI output, docs, and focused tests while preserving ranked search.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection of current main shows the relevant CLI search/list/explore paths and HTTP handlers still do not expose exact, prefix, or source-hub enumeration inputs.

## Solution Assessment

Yes at the direction level: a separate indexed enumeration path is safer than redefining ranked search or relying on bounded prefix recall. The exact CLI/API shape still needs maintainer choice before implementation.
