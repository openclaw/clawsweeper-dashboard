## Summary

Keep open. Current main still improves recall inside ranked search, but it does not expose deterministic exact-slug, slug-prefix, or hub-list registry enumeration through the CLI or public skill API, and no canonical duplicate or replacement PR owns that product/API decision.

## What This Changes

Request to add exact, prefix, or list-style registry discovery for ClawHub CLI search so users can enumerate skills by slug or family instead of relying on ranked search.

## Best Possible Solution

Add a documented, cursor-pageable registry discovery surface with indexed exact-slug and slug-prefix filters, then expose it through explicit CLI flags or a dedicated list mode while preserving ranked search semantics.

## Reproduction Assessment

Yes for the missing capability by source inspection: current main has q+limit CLI search and limit/cursor/sort API listing, but no exact or prefix enumeration surface. This is a feature request rather than a broken existing contract.

## Solution Assessment

No. Current prefix/exact recall inside relevance search is useful, but it is not the best solution for enumeration because the results remain capped and ranked; the maintainable path is a deliberate indexed, pageable API/CLI contract.
