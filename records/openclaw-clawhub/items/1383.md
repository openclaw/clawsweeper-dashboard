## Summary

Keep open. ClawHub issue cleanup policy keeps issues open, and current main still lacks deterministic exact-slug or slug-prefix enumeration; the current CLI/API surfaces remain relevance search or bounded registry browsing.

## What This Changes

Request adding CLI/API skill discovery paths for exact slug, slug-prefix, or full hub listing instead of relying on ranked search.

## Best Possible Solution

Add a narrow indexed registry discovery surface that preserves ranked search while exposing exact slug lookup and slug-prefix enumeration through CLI JSON output, public v1 API support, docs, and focused regression tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection confirms the current CLI/API expose ranked search and bounded browsing without exact or prefix enumeration inputs.

## Solution Assessment

Yes at the direction level: a separate indexed enumeration path is safer than redefining relevance search as complete. The unresolved part is which public CLI/API shape maintainers want to support.
