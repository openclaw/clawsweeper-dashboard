## Summary

Keep open. Current main and v0.23.0 still lack deterministic prefix or exact registry enumeration in the ClawHub CLI/API; search has prefix recall, but it is still ranked and sliced rather than a complete listing contract.

## What This Changes

The issue requests a registry-oriented ClawHub CLI discovery path for exact slug or slug-prefix enumeration instead of relying only on ranked search.

## Best Possible Solution

Add a documented, cursor-pageable registry discovery surface with indexed exact-slug and slug-prefix filters, then expose it through explicit CLI flags or a dedicated hub list mode while preserving ranked search semantics.

## Reproduction Assessment

Yes. Source inspection on current main shows the CLI search path only accepts query and limit, the installed `list` command is local-only, and the public skill list API is sorted browsing rather than exact or prefix enumeration.

## Solution Assessment

Unclear until maintainers choose the public contract. The narrow maintainable direction is indexed, cursor-pageable API support plus explicit CLI flags or a dedicated list mode, not stretching relevance-ranked search into an implicit total listing.
