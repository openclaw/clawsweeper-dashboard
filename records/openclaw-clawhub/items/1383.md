## Summary

Keep open. Current main and the latest v0.23.0 release still expose ranked/capped skill search and bounded registry browsing, but not deterministic exact-slug or slug-prefix enumeration through the ClawHub CLI/API.

## What This Changes

Request registry-oriented ClawHub CLI discovery for exact slug, slug-prefix, or hub-list enumeration instead of relying only on ranked search.

## Best Possible Solution

Add a documented, cursor-pageable registry discovery surface with indexed exact-slug and slug-prefix filters, then expose it through explicit CLI flags or a dedicated list mode while preserving ranked search semantics.

## Reproduction Assessment

Yes. Source inspection on current main shows the CLI search path only accepts query and limit, and the public skills list API only accepts limit/cursor/sort/nonSuspicious options, so there is no exact or prefix enumeration path.

## Solution Assessment

Unclear until maintainers choose the public contract. The narrow maintainable direction is indexed, cursor-pageable API support plus explicit CLI flags or a dedicated list mode, not stretching relevance-ranked search into an implicit total listing.
