## Summary

Keep open. Current main still has relevance-ranked skill search plus bounded browse/list paths, not deterministic exact-slug or slug-prefix registry enumeration, and the ClawHub cleanup policy keeps issues open.

## What This Changes

The issue requests a CLI registry-discovery mode for exact-slug and slug-prefix enumeration instead of relying on relevance-ranked skill search.

## Best Possible Solution

Add an indexed, cursor-paginated registry discovery path for exact slug lookup and slug-prefix enumeration through the public API and CLI while preserving current relevance search semantics.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The gap is source-confirmed on current main because search remains ranked and list/explore remain bounded browse surfaces without exact or prefix enumeration parameters.

## Solution Assessment

Yes at the product-direction level: a separate indexed enumeration path is safer than redefining relevance search as complete. The remaining decision is the API/CLI shape maintainers want to support first.
