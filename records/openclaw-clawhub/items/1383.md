## Summary

Keep open. Current main still provides relevance-ranked skill search plus bounded registry browsing, but not deterministic exact-slug or slug-prefix enumeration through the CLI or public API.

## What This Changes

Request to add registry-oriented exact, prefix, or list-style skill discovery to the ClawHub CLI.

## Best Possible Solution

Add a documented, cursor-pageable registry discovery surface with indexed exact-slug and slug-prefix filters, then expose it through explicit CLI flags or a dedicated list mode while preserving ranked search semantics.

## Reproduction Assessment

Yes. Source inspection on current main shows search and explore are ranked or bounded browsing paths, with no CLI/API exact-slug or slug-prefix enumeration contract.

## Solution Assessment

Unclear until maintainers choose the public contract. The narrow maintainable direction is indexed, cursor-pageable API support plus explicit CLI flags or a dedicated list mode, not stretching relevance-ranked search into an implicit total listing.
