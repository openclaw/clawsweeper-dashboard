Keep open. This is a valid feature request, and current main still lacks a deterministic exact-slug, slug-prefix, or hub-list discovery contract in the ClawHub CLI/API; the current surfaces remain ranked search or bounded browsing.

## What This Changes

The issue requests deterministic registry-style ClawHub CLI discovery for exact skill slugs, slug prefixes, or hub listing instead of relying only on ranked search.

## Best Possible Solution

Add a narrow registry discovery surface that preserves ranked search while exposing exact slug lookup and slug-prefix enumeration through indexed cursor-paginated backend queries, public v1 API support, CLI JSON output, docs, and focused tests.
