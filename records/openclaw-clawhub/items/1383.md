## Repair Work Prompt

Implement a narrow registry discovery path for skills that supports exact slug lookup and slug-prefix enumeration without changing ranked search semantics. Add indexed, cursor-paginated backend support using existing skillSearchDigest/skills indexes or a similarly lightweight projection; do not introduce full-table scans. Expose the behavior through the public v1 API and the ClawHub CLI, preferably with conservative `search --exact <slug>` and `search --prefix <prefix>` or an equivalent hub-list mode that returns machine-readable output. Update schema/API response validation, CLI help, docs, and focused tests. Preserve existing `search <query>` relevance behavior and existing `explore` behavior unless the new list mode explicitly extends it. Do not add seed-only skill content or unrelated package/plugin changes.
