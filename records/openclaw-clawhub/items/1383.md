## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still lacks deterministic exact-slug, slug-prefix, or hub-list enumeration for skill discovery; the implemented prefix work improves ranked recall but does not make search a complete listing mechanism.

## What This Changes

Request a CLI/API registry discovery mode for exact slug, slug-prefix, or hub listing instead of relying only on ranked skill search.

## Best Possible Solution

Add a narrow indexed registry discovery surface that preserves ranked search while exposing exact slug lookup and slug-prefix enumeration through CLI JSON output, public v1 API support, docs, and focused regression tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection confirms current main exposes ranked search and bounded browsing, not deterministic exact or prefix enumeration.

## Solution Assessment

Yes at the product-direction level: a separate indexed enumeration path is safer than redefining ranked search as complete. The unresolved part is which public CLI/API shape maintainers want to support.
