## Summary

Keep open. This is a valid feature request, and current main still exposes ranked skill search and bounded registry browsing rather than deterministic exact-slug or slug-prefix enumeration.

## What This Changes

Request deterministic CLI registry discovery for skills by exact slug, slug prefix, or hub listing instead of only ranked search.

## Best Possible Solution

Add a narrow registry discovery surface that preserves ranked search while adding exact slug lookup and cursor-paginated prefix listing through indexed backend queries, v1 API support, CLI JSON output, docs, and focused tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection on current main confirms the relevant CLI/API paths lack exact or prefix enumeration inputs.

## Solution Assessment

Unclear until maintainers choose the public API and CLI shape. The safest technical direction is a separate indexed enumeration path rather than redefining relevance search as complete.
