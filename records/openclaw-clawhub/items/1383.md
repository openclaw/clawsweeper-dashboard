## Summary

Keep open. Current main and v0.21.0 still provide relevance-ranked/capped skill search plus sorted registry browsing, not deterministic exact-slug or slug-prefix registry enumeration, and no canonical replacement item owns the remaining behavior.

## What This Changes

The issue requests a CLI/API path for exact or prefix-based ClawHub skill enumeration instead of relying only on ranked search.

## Best Possible Solution

Add a documented, cursor-pageable registry discovery surface with indexed exact-slug and slug-prefix filters, then expose it through a clear CLI mode while preserving relevance-ranked search semantics.

## Reproduction Assessment

Yes, at source level: current main exposes ranked/capped search and sorted browsing but no exact or prefix enumeration contract. The missing CLI flags/API filters are directly visible in command registration, CLI request construction, HTTP handlers, docs, and tests.

## Solution Assessment

Unclear: the request identifies a valid gap, but no single proposed syntax is established as the best contract. A dedicated pageable enumeration API surfaced through an explicit CLI mode is safer than overloading ranked search alone.
