## Summary

Keep open. Current main still provides ranked, capped search and bounded registry browsing, but not deterministic exact-slug or slug-prefix enumeration through the ClawHub CLI/API.

## What This Changes

Request adds registry-oriented CLI discovery for exact slug, slug prefix, or hub-wide skill listing instead of relying on ranked search.

## Best Possible Solution

Add a documented, cursor-pageable registry discovery surface with indexed exact-slug and slug-prefix filters, then expose it through explicit CLI flags or a dedicated list mode while preserving ranked search semantics.

## Reproduction Assessment

Yes. Source inspection on current main shows the CLI/API paths support ranked search and bounded browse pages, but no deterministic exact-slug or slug-prefix enumeration mode.

## Solution Assessment

Unclear until maintainers choose the public contract. The narrow maintainable direction is indexed, cursor-pageable API support plus explicit CLI flags or a dedicated list mode, not stretching relevance-ranked search into an implicit total listing.
