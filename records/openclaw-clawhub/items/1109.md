## Summary

Keep open. Current main has the right stable GitHub numeric-ID/profile-sync foundation, but this is an open ClawHub issue and the live GitHub sign-in endpoint still fails before the username-change verifier mismatch or requested relink can be proven resolved.

## What This Changes

The reporter cannot log in via GitHub after changing their GitHub username, sees a `convexAuthOAuthVerifier` token mismatch, and asks for account relinking.

## Best Possible Solution

Restore production GitHub OAuth session creation for active accounts, keep identity anchored to immutable GitHub `providerAccountId`, and only perform or add an audited recovery/relink path after maintainer verification of the affected production auth records.

## Reproduction Assessment

Yes for the current production sign-in-start failure: both canonical and direct Convex GitHub sign-in endpoints returned HTTP 500 on May 10, 2026. No for the username-change-specific verifier mismatch, because that requires live OAuth callback state and the affected production auth rows.

## Solution Assessment

Not applicable for a patch because this is an issue, not a proposed fix. The existing numeric-ID mapping and post-login profile sync are the right foundation, but the safest next step is maintainer/operator auth triage before any relink or source change.
