Thanks for the report, and sorry this took a while to get into the tracker state.

This generic username-change path is fixed now. The GitHub OAuth profile maps the stable numeric GitHub id into the auth identity, and the post-login hook schedules a GitHub profile sync so username/avatar changes are applied after sign-in instead of treating the renamed login as a different account.

Proof checked today:
- `convex/auth.ts` maps `profile.id` as the provider id and schedules `internal model.users.syncGitHubProfileAction` after OAuth sign-in.
- `convex/users.ts` updates the stored username/handle/display fields when the GitHub login changes.
- Regression test: `bunx vitest run convex/users.test.ts -t "updates handle and display name when GitHub login changes"` passed.

Closing the generic bug as fixed. Per-user account recovery reports are being handled separately where we need production-account proof.
