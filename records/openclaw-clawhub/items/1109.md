Keep open. Restore the production GitHub OAuth start/callback path, keep identity anchored to numeric GitHub `providerAccountId`, inspect the affected production account rows, and add a narrowly scoped recovery/admin path only if operator triage proves it recurs.

The current source-level numeric-ID mapping and post-login profile sync are the right foundation, but they do not prove this production report is resolved.
