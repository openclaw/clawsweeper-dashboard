## Summary

Keep open. Current main has the right stable GitHub numeric-ID and profile-sync foundation, but the live GitHub sign-in route still returns HTTP 500 and the account-specific verifier mismatch or relink request cannot be proven resolved from source alone.

## What This Changes

The issue reports failed GitHub OAuth login after a GitHub username change, with a localStorage `convexAuthOAuthVerifier` token mismatch and a request to relink the account.

## Best Possible Solution

Restore production GitHub OAuth for active accounts, keep identity anchored to numeric `providerAccountId`, and perform any relink or recovery only after audited maintainer verification of production auth records.

## Reproduction Assessment

Yes for the current production sign-in availability failure: current main is production-tagged and both canonical and direct Convex GitHub sign-in endpoints return HTTP 500. No for the username-change-specific verifier mismatch because that requires live OAuth callback state and the affected production auth rows.

Not applicable for a patch because this is an issue, not a proposed fix. The existing numeric-ID mapping and post-login profile sync are the right foundation, but the safest next step is maintainer/operator auth triage before any relink, production config change, or source repair is scoped.
