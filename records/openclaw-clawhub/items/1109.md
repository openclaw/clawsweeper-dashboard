## Summary

Keep open. Current main has the correct stable GitHub numeric-ID mapping and a post-login profile sync path for renamed accounts, but this report is a production login/relink request involving a `convexAuthOAuthVerifier` mismatch that cannot be proven fixed from the checkout. ClawHub cleanup policy also keeps issues open unless they are PRs already implemented on main, and related OAuth/username-change reports remain open.

## What This Changes

The issue asks maintainers to restore GitHub OAuth login after a GitHub username change and relink the affected ClawHub account to the new GitHub username.

## Best Possible Solution

ClawHub should keep using the immutable GitHub provider id for auth continuity across username changes, while a maintainer with production access inspects the affected auth records and either repairs the data or adds a focused recovery/admin path if the same verifier or duplicate-account failure is reproducible.
