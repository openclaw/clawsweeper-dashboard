Keep open. Current main has the right stable GitHub numeric-ID and post-login profile-sync foundation, but the live GitHub sign-in start route still returns HTTP 500 and the account-specific verifier/relink state cannot be verified from source alone; ClawHub cleanup policy also keeps issues open.

The issue requests restoring GitHub OAuth login after a GitHub username change and relinking the affected account to the new GitHub username.

## Best Possible Solution

Restore the production GitHub OAuth redirect/session path, keep account identity anchored to immutable `providerAccountId`, inspect the affected production auth rows, and add a maintainer-owned recovery or consolidation path only if triage proves one is needed.
