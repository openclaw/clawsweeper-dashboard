## Summary

Keep open. Current main has stable GitHub numeric-ID mapping and post-login profile sync for renamed accounts, but this open ClawHub issue reports a production OAuth verifier/session mismatch and an account relink request that cannot be proven resolved from source inspection.

## What This Changes

The issue reports failed GitHub OAuth login after a GitHub username change, with `convexAuthOAuthVerifier` token mismatch observed in localStorage and a request to relink the affected account.

## Best Possible Solution

ClawHub should preserve sign-in across GitHub username changes via immutable provider IDs, then maintainers should verify the affected production auth linkage and either repair the account or add a focused audited recovery path if this recurs.

## Reproduction Assessment

No. The report gives a plausible user flow and concrete localStorage symptom, but high-confidence reproduction requires live GitHub OAuth callback state plus production auth records for the affected account.

## Solution Assessment

Unclear. The current source design is the right narrow direction for username changes, but the requested relink cannot be safely solved by a blind repo-only patch without production identity verification.
