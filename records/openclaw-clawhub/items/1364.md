## Summary

Keep open: current main still contains the reported CLI double-serialization bug, and ClawHub policy keeps issues open for maintainer follow-up rather than auto-closing them.

## What This Changes

The issue requests that `clawhub transfer request <slug> <handle>` send the positional handle as the API payload field `toUserHandle`.

## Best Possible Solution

Land or split a focused CLI fix that sends plain object bodies from transfer, rename, and merge command handlers while keeping `apiRequest` as the single JSON serialization layer.

## Reproduction Assessment

Yes. Static source inspection against current main gives a high-confidence reproduction path: the CLI passes a JSON string, both HTTP transports stringify it again, and the API then sees no `toUserHandle` property.

## Solution Assessment

Yes, passing a plain object from the command layer is the narrow maintainable fix. The safer maintainer path is to land that portion directly or as part of #1635 if the broader ownership-transfer changes are ready.
