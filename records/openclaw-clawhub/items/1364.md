## Summary

Keep open: current main still has the reported CLI request-body double-serialization bug, and ClawHub cleanup policy keeps issues open for maintainer follow-up rather than auto-closing them.

## What This Changes

The issue requests that `clawhub transfer request <slug> <handle>` send the positional handle as the API payload field `toUserHandle`.

## Best Possible Solution

Land a focused CLI fix that passes plain object bodies from transfer, rename, and merge command handlers to `apiRequest`, updates command tests to assert object bodies, and leaves JSON serialization centralized in the HTTP helper.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: the command passes a JSON string, both HTTP transports stringify it again, and the API then sees no `toUserHandle` property.

## Solution Assessment

Yes. Passing a plain object from the command layer is the narrow maintainable fix; teaching the server to accept double-encoded JSON would preserve the wrong client boundary.
