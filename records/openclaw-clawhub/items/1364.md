## Repair Work Prompt

Fix the CLI double JSON serialization bug for ownership-related POST commands. On current main, `clawhub transfer request <slug> <handle>` passes `body: JSON.stringify({ toUserHandle, message })` to `apiRequest`, while `apiRequest` serializes POST bodies again; `skill rename` and `skill merge` in `packages/clawhub/src/cli/commands/ownership.ts` have the same boundary bug for `newSlug` and `targetSlug`. Keep `apiRequest` as the JSON serialization boundary, pass plain object bodies from affected command handlers, update transfer and ownership command tests to assert object bodies, and only add shared HTTP defensive handling if both Node fetch and Bun/curl paths have regression coverage. Do not change the server API contract or broaden into org/package ownership-transfer product work from the open consolidated PR.
