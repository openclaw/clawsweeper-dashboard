## What This Changes

The issue requests that `clawhub transfer request <slug> <handle>` map the positional handle to the `toUserHandle` API payload field.

## Best Possible Solution

Keep `apiRequest` as the single JSON serialization layer and update the affected CLI commands plus focused tests to pass plain object bodies.

## Reproduction Assessment

Yes. Static source inspection on current main gives a high-confidence path: the command passes a JSON string, both HTTP transports stringify it again, and the API then sees no `toUserHandle` property.

## Solution Assessment

Yes. Passing a plain object from the command layer is the narrow maintainable fix; the maintainer coordination point is whether to land it directly or split it from the broader open ownership-transfer PR.
