## Summary

Keep open: current main still contains the source-confirmed double-serialization bug, and the related ownership-transfer PR is open and conflicting rather than landed.

## What This Changes

The issue requests that `clawhub transfer request <slug> <handle>` send the positional handle as the `toUserHandle` API payload field.

## Best Possible Solution

Land or extract a focused CLI fix that passes plain object bodies to `apiRequest`, updates command tests to assert object payloads, and keeps the HTTP helper as the single JSON serialization layer.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: the command sends a JSON string, both HTTP transports stringify it again, and the server requires `payload.toUserHandle` on an object.

## Solution Assessment

Yes. Passing a plain object from the CLI is the narrow maintainable fix; reparsing double-encoded strings server-side would mask the client boundary bug.
