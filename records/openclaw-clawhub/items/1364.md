## Summary

Keep open: current main still has the reported CLI serialization bug, and ClawHub policy keeps issues open for maintainer follow-up rather than auto-closing them.

## What This Changes

The issue reports that `clawhub transfer request <slug> <handle>` fails because the positional handle is not sent as the API `toUserHandle` payload field.

## Best Possible Solution

Land a focused CLI fix that sends plain object bodies from transfer, rename, and merge command handlers while keeping `apiRequest` as the single JSON serialization layer.

## Reproduction Assessment

Yes. Static source inspection on current main gives a high-confidence reproduction path: the CLI passes a JSON string, both HTTP transports stringify it again, and the API then sees no `toUserHandle` property.

## Solution Assessment

Yes. Passing plain objects from command handlers is the narrow maintainable fix; the server should not be taught to accept double-encoded request strings.
