## Summary

Keep open: current main still has the reported CLI double-serialization bug, and ClawHub cleanup policy keeps issues open for maintainer follow-up rather than auto-closing them. An open related PR, #1635, already changes this command path to pass a plain object body, so the best next step is maintainer review or splitting of that narrow fix.

## What This Changes

The issue asks for `clawhub transfer request <slug> <handle>` to send the positional handle as `toUserHandle` in the API request body.

## Best Possible Solution

Keep `apiRequest` as the single JSON serialization layer and update the affected CLI command tests so command handlers pass plain object bodies.

## Reproduction Assessment

Yes. Static source inspection on current main gives a high-confidence reproduction path: the command passes a JSON string, both HTTP transports stringify it again, and the API then sees no `toUserHandle` property.

## Solution Assessment

Yes. Passing a plain object from the command layer is the narrow maintainable fix; the server should not grow compatibility for double-encoded request strings.
