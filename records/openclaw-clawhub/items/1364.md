## Summary

Keep open: current main still has the source-confirmed double-serialization path for transfer requests, and the consolidated ownership-transfer work that includes a repair remains open and dirty rather than landed.

## What This Changes

Fix the CLI transfer request command so the positional recipient handle is sent as `toUserHandle` in an object payload rather than a double-serialized JSON string.

## Best Possible Solution

Extract or land the plain-object request-body fix for transfer, rename, and merge while leaving the broader org/package ownership-transfer product work in the consolidated PR.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: the command sends a JSON string, both HTTP transports stringify it again, and the server requires `payload.toUserHandle` on an object.

## Solution Assessment

Yes. Passing plain objects from the affected CLI commands and updating tests is the narrow maintainable fix; parsing double-encoded strings server-side would mask the client boundary bug.
