## What This Changes

The issue reports that `clawhub transfer request <slug> <handle>` sends a pre-serialized body so the API cannot read `toUserHandle`.

## Best Possible Solution

Land a focused CLI fix that passes plain object bodies from transfer, rename, and merge commands, updates tests to assert object payloads, and keeps `apiRequest` as the single JSON serialization boundary.

## Reproduction Assessment

Yes, by source inspection: the command passes a JSON string, both HTTP transports stringify request bodies again, and the server requires `payload.toUserHandle` on an object. I did not run a live authenticated transfer in this read-only review.

## Solution Assessment

Yes. Passing plain objects from the affected CLI commands is the narrow maintainable fix; server-side tolerance for double-encoded strings would mask the client boundary bug.
