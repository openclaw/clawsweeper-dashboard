## Summary

Keep open. Current main still contains the reported CLI serialization bug: the transfer command passes a JSON string as `body`, while both HTTP transports serialize POST bodies again before the API checks `toUserHandle`. ClawHub policy also keeps issues open for maintainer follow-up rather than auto-closing them.

## What This Changes

The issue requests that `clawhub transfer request <slug> <handle>` send the positional handle as the `toUserHandle` API field instead of failing with `toUserHandle required`.

## Best Possible Solution

Land a focused CLI repair that passes plain object bodies from transfer, rename, and merge command handlers to `apiRequest`, updates command tests to assert object bodies, and preserves the HTTP helper as the only JSON serialization layer.

The command layer should pass plain object bodies to `apiRequest`; `apiRequest` should remain the single layer that JSON-serializes POST bodies.
