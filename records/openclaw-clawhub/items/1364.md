## Summary

Keep open: current main still has the reported CLI double-serialization bug, and openclaw/clawhub cleanup policy keeps issues open. Related PR #1635 is still open and contains the object-body repair, so the issue should remain open until that fix lands or is extracted.

## What This Changes

The issue requests that `clawhub transfer request <slug> <handle>` send the positional handle as the API `toUserHandle` field.

## Best Possible Solution

CLI command handlers should pass plain object bodies to `apiRequest`, with tests asserting the object shape and the HTTP helper remaining the single JSON serialization layer.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: the command sends a JSON string, both HTTP transports stringify it again, and the handler requires `payload.toUserHandle` on an object.

## Solution Assessment

Yes. Passing a plain object from the CLI and updating the command test is the narrow maintainable fix; reparsing double-encoded strings server-side would mask a client boundary bug.
