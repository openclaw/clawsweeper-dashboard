## Summary

Keep open: current main still has the reported CLI serialization bug by source inspection, and ClawHub cleanup policy keeps issues open. The related open PR #1635 already includes the same body-shape repair path, so maintainer review or a narrow split from that PR is the right next step.

## What This Changes

The issue asks that `clawhub transfer request <slug> <handle>` send the positional handle as the `toUserHandle` JSON field instead of failing with `toUserHandle required`.

## Best Possible Solution

Keep `apiRequest` as the single JSON serialization layer, pass plain object bodies from the affected CLI commands, and update command tests to assert object payloads.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: the command passes a JSON string, both HTTP transports stringify it again, and the server then sees no `toUserHandle` property.

## Solution Assessment

Yes. Passing a plain object from the command layer is the narrow maintainable fix; accepting double-encoded strings server-side would preserve the wrong API boundary.
