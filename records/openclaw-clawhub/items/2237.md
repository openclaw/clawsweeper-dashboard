## What This Changes

The issue asks ClawHub to address a published skill whose Telegram /gitsearch fallback interpolates remote user input into a shell=True subprocess.

## Best Possible Solution

Temporarily hide or otherwise moderate the vulnerable published skill, require a fixed release using argv-based subprocess execution with shell=False, and rescan before restoring public install confidence.

## Reproduction Assessment

Yes, at source level: the downloaded latest artifact shows Telegram input reaches `_web_search`, which interpolates the query into a shell=True subprocess. I did not execute the exploit payload against a live host.

## Solution Assessment

Yes, the reporter's suggested code-level remedy is the narrow fix: pass the query as an argv element with shell=False, then add validation and documentation before republishing.
