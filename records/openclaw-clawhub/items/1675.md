## Summary

Keeping this open under the ClawHub cleanup policy, and current main still cannot re-home an existing skill to an organization: publish-as-org exists for selected publishes, but the ownership-transfer workflow remains user-recipient based.

## What This Changes

The issue requests organization ownership for skills and a supported way to transfer an existing personally owned skill to an org owner without losing stats or URLs.

## Best Possible Solution

Land a publisher-targeted transfer model that stores source and target publisher ids, authorizes acceptance by target owner/admin membership, preserves current skill state and aliases, and updates the API, CLI, UI, docs, and tests compatibly.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an org handle reaches the transfer endpoint as toUserHandle, but recipient resolution only returns active users or user-kind personal publishers, and the transfer table plus accept path require user ids.

## Solution Assessment

No. A small org lookup added to the current toUserHandle flow is not the best fix; the maintainable path is the publisher-targeted transfer model already described in docs/orgs.md and partly explored in https://github.com/openclaw/clawhub/pull/1635.
