## Summary

Keeping this open under the ClawHub issue policy. Current main partially addresses the user problem with publish-time owner migration, but the direct transfer command and transfer API still model recipients as users rather than organization publishers, and the remaining work overlaps the open conflicting transfer PR at https://github.com/openclaw/clawhub/pull/1635.

## What This Changes

The issue requests a supported path to move an existing skill from a personal owner handle to an organization owner without deleting and republishing.

## Best Possible Solution

Keep publish-time owner migration as the supported immediate path, then decide whether to land a publisher-targeted transfer model that stores publisher recipients and enforces source/destination authorization explicitly.

## Reproduction Assessment

Yes. Source inspection shows the direct transfer path still accepts toUserHandle, resolves only users or user-kind personal publishers, and stores toUserId, so an organization publisher handle cannot be represented as a direct transfer destination on current main.

## Solution Assessment

Unclear. Publish-time owner migration is a maintainable partial solution, but direct transfer-to-org should be implemented as a publisher-targeted transfer design rather than a resolver-only tweak.
