## Summary

Keeping this open for maintainer review. Current main now supports the publish-as-org/re-home path with explicit owner migration, but the reported `clawhub transfer request` and transfer API path still remain user-targeted, and ClawHub issue cleanup policy keeps issues open rather than auto-closing them.

## What This Changes

Request org ownership support for skills, including transferring or otherwise re-homing existing personal-owned skills to an org without losing stats or URLs.

## Best Possible Solution

Keep the explicit `--owner` plus `--migrate-owner` publish path as the current migration route, and separately design publisher-targeted transfers if direct transfer commands remain a product requirement.

## Reproduction Assessment

Yes. Source inspection shows the transfer endpoint and CLI still send `toUserHandle`, the resolver only returns users or user-kind personal publishers, and the transfer table stores `toUserId`, so an org publisher handle is not a valid transfer recipient.

## Solution Assessment

Unclear. Current main’s explicit owner-migration publish flow is a maintainable solution for owners who can republish, but it does not make the transfer API/CLI publisher-targeted; maintainers should decide whether that remaining path is still required.
