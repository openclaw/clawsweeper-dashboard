## Summary

Keeping this open. Current main still models skill transfers as user-to-user, so org handles cannot be transfer recipients even though org publishers and web publish-as-org support now exist.

## What This Changes

Support transferring or otherwise moving existing skills from a personal owner to an org owner without deleting and republishing.

## Best Possible Solution

Implement a publisher-targeted skill transfer flow that supports personal-to-org, org-to-personal, and org-to-org transfers while preserving stats, aliases, and existing client compatibility.

## Reproduction Assessment

Yes. Source inspection on current main shows an org handle reaches a user/personal-publisher resolver and returns null, causing requestTransferInternal to throw User not found before a transfer can be created.

## Solution Assessment

Unclear. The maintainable direction is the documented publisher-targeted transfer model, not a narrow org lookup bolted onto the existing toUserHandle/toUserId schema.
