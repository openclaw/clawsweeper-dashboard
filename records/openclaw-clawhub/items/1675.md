## Summary

Keeping this open. Current main still has the reported product gap: skill transfers are user-recipient only even though org publishers exist, and the org design notes explicitly call for a publisher-targeted transfer model.

## What This Changes

Support moving existing skills from personal ownership to org publishers through transfer-to-org, publish-as-org, or an equivalent re-home flow.

## Best Possible Solution

Implement publisher-targeted skill transfers that preserve stats, aliases, existing install URLs, and user-to-user compatibility while authorizing org acceptance through publisher membership.

## Reproduction Assessment

Yes. The source path is high confidence: an org handle is normalized into toUserHandle, then getActiveUserByHandleOrPersonalPublisher returns null for org publishers and requestTransferInternal throws User not found before a transfer can be created.

## Solution Assessment

Unclear. Transfer-to-org matches the documented direction, but the maintainable fix should update the transfer model around publisher targets rather than bolt org lookup onto the existing user-only toUserHandle schema.
