## Summary

Keep open. Current main has partial organization publisher support, including web publish-as-org for new skill publishes, but the reported no-loss personal-to-org transfer path is still missing: transfers are modeled as user-to-user, org publishers are rejected by the recipient resolver, and accepting a transfer assigns the recipient’s personal publisher. ClawHub cleanup policy also keeps issues open rather than auto-closing them.

## What This Changes

The issue requests support for organization ownership and personal-to-organization transfer of existing skills without deleting and republishing.

## Best Possible Solution

Implement a core publisher-targeted skill transfer flow: resolve user and org publisher handles, authorize by publisher membership, preserve stats and aliases, keep old user-to-user clients compatible, update CLI/API schemas, and add regression coverage for personal-to-org, org-to-user, and org-to-org transfers.
