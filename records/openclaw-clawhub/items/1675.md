## Summary

Keeping this open. Current main partially supports org ownership for new web skill publishes, but existing skill transfers are still user-targeted and do not provide the requested path to move a personal skill under an org publisher while preserving stats and URLs.

## What This Changes

Request org ownership support for skills, especially transferring existing user-owned skills to org publishers without deleting and republishing.

## Best Possible Solution

Implement publisher-targeted skill transfers with source/target publisher IDs, org owner/admin acceptance, audit coverage, and backwards-compatible API/CLI behavior that preserves existing skill stats, aliases, and install/page URLs.

## Reproduction Assessment

Yes by source inspection. An org handle reaches the transfer flow as toUserHandle, but the resolver only returns active users or personal user publishers, and the transfer table can only store user IDs.

## Solution Assessment

Unclear. The requested transfer-to-org outcome is right for existing skills, but the maintainable solution is a publisher-targeted transfer model rather than bolting org lookup onto the current user-only schema.
