## Summary

Keeping this open. Current main supports org publishers and publish-as-org for new skill publishes, but the user-facing transfer flow still targets users, and existing personal-owned skills cannot be re-homed to org publishers through the normal transfer path.

## What This Changes

Request support for moving existing skill ownership from a personal publisher to an organization publisher while preserving the skill record, stats, and canonical owner URL.

## Best Possible Solution

Implement publisher-targeted skill transfers that store source and target publisher ids, preserve stats and slug aliases, authorize acceptance through org membership, and migrate the API/CLI/UI contract compatibly.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the transfer endpoint and CLI pass toUserHandle, recipient resolution only returns users or user-kind personal publishers, and org publishers return null.

## Solution Assessment

Not yet. A small org lookup inside the current toUserHandle flow would leave the schema, accept/list paths, audit metadata, and compatibility contract user-centric; the maintainable fix is the publisher-targeted transfer model already sketched in docs/orgs.md.
