## Summary

Keeping this open: current main still has a user-recipient skill transfer model, so org publisher handles cannot be transfer destinations, and ClawHub cleanup policy keeps issues open unless they are PRs already implemented on main.

## What This Changes

Request support for skill org ownership, especially moving an existing user-owned skill to an org publisher handle without deleting or republishing it.

## Best Possible Solution

Implement a publisher-targeted skill transfer flow that can move existing skills between personal and org publishers, preserves stats and aliases, and keeps user-to-user transfer compatibility.

## Reproduction Assessment

Yes. The source-level path is high confidence: an org handle reaches a resolver that returns null for publisher.kind === "org", then requestTransferInternal throws User not found.

## Solution Assessment

Unclear. Transfer-to-org is necessary for existing skills, but the maintainable implementation should follow the documented publisher-transfer model rather than patching org lookup into a user-only transfer schema.
