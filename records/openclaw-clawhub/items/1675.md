## Summary

Keeping this open. Current main still has user-recipient skill transfers only; org publishers and publish-as-org support exist, and the org design notes explicitly track publisher-targeted transfers as remaining work.

## What This Changes

Request support for moving existing skills from personal user ownership to org publisher ownership without deleting and republishing.

## Best Possible Solution

Implement publisher-targeted skill transfers that preserve stats, aliases, and existing install/page URLs while authorizing acceptance through target publisher owner/admin membership.

## Reproduction Assessment

Yes. Source inspection shows an org handle is normalized as toUserHandle, then rejected because the resolver only returns active users or personal user publishers, and the transfer record requires toUserId.

## Solution Assessment

Unclear. Transfer-to-org matches the documented direction, but the maintainable solution is a publisher-targeted transfer model rather than bolting org lookup onto the existing user-only schema.
