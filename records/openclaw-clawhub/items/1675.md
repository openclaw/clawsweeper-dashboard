## Summary

Keeping this open. Current main still cannot target org publishers in the normal skill transfer flow, and the remaining work is a publisher-transfer model change rather than a cleanup close.

## What This Changes

Request support for consolidating existing user-owned skills under org ownership through org-targeted transfer, publish-as-org, or automatic re-home.

## Best Possible Solution

Implement publisher-targeted skill transfers that preserve stats, slug aliases, install/page compatibility, and authorize acceptance through target publisher owner/admin membership.

## Reproduction Assessment

Yes. Source inspection shows org handles are sent as toUserHandle, then rejected because the transfer resolver only returns active users or personal user publishers and the transfer record requires toUserId.

## Solution Assessment

Unclear. Transfer-to-org is the best fit for preserving existing stats and URLs, but the maintainable fix is a publisher-targeted transfer model rather than bolting org lookup onto the user-only schema.
