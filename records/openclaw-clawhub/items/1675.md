## Summary

Current main still only supports user-recipient skill transfers; publish-as-org exists, but it does not provide a path to transfer an existing skill to an org publisher, and the org design notes track publisher-targeted transfers as remaining work.

## What This Changes

Request support for moving an existing skill from an individual owner to an org/publisher owner without losing stats, aliases, or canonical URLs.

## Best Possible Solution

Implement publisher-targeted skill transfers that preserve existing skill state and authorize acceptance through target publisher owner/admin membership.

## Reproduction Assessment

Yes. Source inspection on current main shows an org handle is normalized as toUserHandle, resolved only through users or personal user publishers, then stored as toUserId, so org publishers cannot satisfy the path.

## Solution Assessment

Unclear. Transfer-to-org matches the documented direction, but the maintainable solution is a publisher-targeted transfer model rather than bolting org lookup onto the existing user-only schema.
