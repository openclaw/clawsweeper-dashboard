## Summary

Keeping this open. Current main still cannot transfer an existing skill to an org publisher: the transfer path is user-targeted, the schema stores only user recipients, and repo docs describe publisher-targeted transfers as planned work.

## What This Changes

Request support for org-owned skills, especially transferring existing user-owned skills to an org handle without losing stats or URLs.

## Best Possible Solution

Implement publisher-targeted skill transfers that preserve stats, slug aliases, canonical URLs, and authorize acceptance through the maintainer-approved target-org role model.

## Reproduction Assessment

Yes. Source inspection on current main shows an org handle is still submitted as toUserHandle, then rejected because recipient resolution only returns active users or personal user publishers.

## Solution Assessment

Unclear. Transfer-to-org is the best fit for preserving existing skills, stats, and URLs, but the maintainable fix is a publisher-targeted transfer model rather than adding org lookup to a user-only schema.
