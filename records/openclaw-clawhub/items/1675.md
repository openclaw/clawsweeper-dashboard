## Summary

Keep open. Current main now supports the publish-based owner migration path, but the reported direct `clawhub transfer request <slug> <org-handle>` path is still user-targeted, and the remaining publisher-target transfer design is already active maintainer work rather than cleanup-close material.

## What This Changes

The issue requests a supported path for moving skills from an individual owner to an organization owner, either via transfer, publish-as-org, or publish-time re-homing.

## Best Possible Solution

Keep the explicit publish owner-migration path for self-service moves, and resolve any remaining direct transfer-to-org requirement through the consolidated publisher-target transfer design.

## Reproduction Assessment

Yes. Source inspection shows the transfer API/CLI still send `toUserHandle`, the resolver returns only users or user-kind personal publishers, and the transfer record stores `toUserId`, so an org publisher handle is not a valid direct transfer recipient.

## Solution Assessment

Unclear. Current main's `--owner` plus `--migrate-owner` publish flow is a maintainable solution for users who can republish, but maintainers still need to decide whether direct publisher-target transfers from https://github.com/openclaw/clawhub/pull/1635 are required.
