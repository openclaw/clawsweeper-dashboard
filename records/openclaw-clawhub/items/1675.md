## Summary

Keeping this open. Current main still has partial org publisher support, but the existing skill transfer flow remains user-recipient based and cannot move an already-published personal skill to an org publisher; the ClawHub cleanup policy also keeps issues open rather than auto-closing them.

## What This Changes

The issue requests a no-loss path to move already-published skills from a personal owner handle to an organization owner handle.

## Best Possible Solution

Implement a core publisher-targeted skill transfer flow that resolves user and org publisher handles, authorizes by publisher membership, preserves stats and existing aliases, and keeps old user-to-user clients compatible.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: an org handle passed as toUserHandle goes through a user/personal-publisher resolver, which returns null for org publishers before a transfer can be created.

## Solution Assessment

Unclear. The requested direction is valid, but the maintainable fix should target the existing transfer model rather than only adding publish-time owner selection, because users need to re-home existing skills without deleting them or losing stats.
