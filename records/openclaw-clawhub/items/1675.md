## Summary

Keeping this open. Current main supports publish-time owner migration, but the direct `clawhub transfer request` and transfer API still target users rather than organization publishers; the broader publisher-transfer implementation is still being handled in https://github.com/openclaw/clawhub/pull/1635.

## What This Changes

Request to let existing skills move from personal publisher ownership to an organization, either through direct transfer, publish-time owner selection, or automatic re-home.

## Best Possible Solution

Keep the publish-time migration path available, then land or extract a publisher-targeted transfer flow with source/destination publisher fields, owner/admin checks, acceptance/approval semantics, alias preservation, and audit logging.

## Reproduction Assessment

Yes. Source inspection on current main shows the direct transfer path accepts `toUserHandle`, resolves only users or user-kind personal publishers, and stores `toUserId`, so an organization publisher handle remains invalid as a transfer destination.

## Solution Assessment

Unclear. The current `ownerHandle` plus `migrateOwner` publish path is a maintainable partial solution for re-homing during publish, but direct transfer-to-org needs a publisher-targeted transfer model rather than a small recipient resolver change.
