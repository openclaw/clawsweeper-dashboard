## Summary

Keeping open. Current main has org publishers and a web owner selector for new skill publishes, but existing skill transfers still resolve recipients as users and cannot transfer ownership to org publishers.

## What This Changes

Request support for org-owned skill workflows, especially transferring an existing user-owned skill to an org without losing stats or breaking existing URLs/install commands.

## Best Possible Solution

Skill ownership transfers should target publishers, preserve stats/aliases/install compatibility, and authorize acceptance by owner/admin members of the target publisher.

## Reproduction Assessment

Yes. Source inspection shows the transfer handler accepts toUserHandle, calls a resolver that returns null for org publishers, and stores transfers with toUserId only.

## Solution Assessment

Unclear. The current web publish-as-org path addresses part of future publishing, but solving the reported consolidation path needs a publisher-targeted transfer model rather than another lookup inside a user-only schema.
