## Summary

Keep open. Current main now supports the publish-based owner migration path, but the direct skill transfer API and CLI remain user-targeted, and the ClawHub cleanup policy for this pass keeps open issues open rather than closing them as implemented.

## What This Changes

Request support for moving skill ownership from a personal user handle to an organization through transfer-to-org, publish-as-org, or publish-time re-homing.

## Best Possible Solution

Keep the explicit publish owner-migration path as the self-service route, and land a maintainer-approved publisher-target transfer flow through https://github.com/openclaw/clawhub/pull/1635 or a successor if direct transfer commands remain required.

## Reproduction Assessment

Yes. Source inspection shows the direct transfer API and CLI still send `toUserHandle`, resolve only users or personal user publishers, and persist `toUserId`, so an org publisher handle is not a valid transfer recipient on current main.

## Solution Assessment

Unclear. Current main implements the publish-as-org/migrate-owner option, but a direct transfer-to-org command requires a broader publisher-target transfer model rather than a resolver-only patch.
