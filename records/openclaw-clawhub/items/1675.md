## Summary

Keeping this open under the ClawHub issue policy. Current main now supports publish-time owner migration, but the direct `clawhub transfer request` and transfer API still target users/personal publishers rather than org publishers, and the remaining direct-transfer work overlaps the open, conflicting publisher-transfer PR at https://github.com/openclaw/clawhub/pull/1635.

## What This Changes

Request support for moving skills from personal publishers to organization publishers, either by direct transfer-to-org or by publish-time owner migration.

## Best Possible Solution

Keep the explicit publish-time owner migration as the supported immediate path, then land or extract a publisher-targeted transfer flow if direct transfer commands/API remain a product requirement.

## Reproduction Assessment

Yes. Source inspection shows the direct transfer path still accepts `toUserHandle`, resolves only active users or user-kind personal publishers, and stores `toUserId`, so an organization publisher handle cannot be a transfer destination on current main.

## Solution Assessment

Unclear. The merged `ownerHandle` plus `migrateOwner` publish flow is a maintainable way to re-home a skill while publishing, but a direct transfer-to-org command/API would need a publisher-targeted transfer design rather than a small resolver tweak.
