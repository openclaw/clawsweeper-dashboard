## Summary

Keeping this open. Current main still has a user-targeted skill transfer model, and the org-transfer outcome needs a publisher-targeted transfer design rather than conservative cleanup.

## What This Changes

The issue requests a supported path to move existing user-owned skills under an organization owner without deleting and republishing.

## Best Possible Solution

Implement publisher-targeted skill transfers that preserve stats and URL/install compatibility, with org acceptance authorized through target publisher owner/admin membership.

## Reproduction Assessment

Yes. Source inspection shows an org handle reaches the transfer resolver as toUserHandle, but org publishers return null because only active users or personal user publishers are accepted.

## Solution Assessment

Unclear. Transfer-to-org is the right user outcome for existing skills, but the maintainable fix is a publisher-targeted transfer model rather than bolting org lookup onto a user-only schema.
