## Summary

Keeping this open. Current main still models skill ownership transfer as user-to-user, so org handles cannot be accepted as transfer destinations; solving it needs a publisher/org transfer design rather than conservative cleanup.

## What This Changes

The issue asks for a path to move an existing user-owned skill to an organization owner without deleting and republishing.

## Best Possible Solution

Implement publisher-targeted skill transfers that preserve stats and canonical/install URL compatibility while authorizing acceptance through target publisher ownership or org publisher roles.

## Reproduction Assessment

Yes. Source inspection on current main shows org handles are sent through the user-handle transfer path, and organization publishers cannot resolve to the required toUserId transfer target.

## Solution Assessment

Unclear. Transfer-to-org is the strongest fit for preserving existing skill history, but the maintainable solution is a publisher-targeted transfer model rather than adding an org lookup to a user-only schema.
