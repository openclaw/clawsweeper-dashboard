## Summary

Keeping this open. Current main has org publishers and partial publish-as-org support, but the skill transfer path is still user-recipient based and cannot move an existing personal skill to an org publisher; ClawHub cleanup policy also keeps issues open rather than auto-closing them.

## What This Changes

Support transferring an already-published personal skill to an org publisher so owner URLs and stats can move without deleting and republishing.

## Best Possible Solution

Implement a publisher-targeted skill transfer flow that resolves user and org publisher handles, authorizes by publisher membership, preserves stats and aliases, and keeps existing user-to-user clients compatible.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: sending an org handle as toUserHandle reaches a resolver that only returns users or personal publishers, then requestTransferInternal throws User not found before a transfer can be created.

## Solution Assessment

Unclear. Transfer-to-org is the right product direction, but the maintainable fix should follow the documented publisher-transfer model rather than bolting org lookup onto the existing user-only transfer schema.
