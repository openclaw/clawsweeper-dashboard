## Summary

Keeping this open. Current main still has user-targeted skill transfers, while org ownership support exists elsewhere only as partial publisher infrastructure; the linked reports #779, #1501, #1569, and #1575 show this is an active product gap rather than conservative cleanup.

## What This Changes

Request support for moving existing skill ownership from a personal publisher to an org publisher while preserving stats and canonical owner URLs.

## Best Possible Solution

Implement publisher-targeted skill transfers that preserve existing stats, slug aliases, install compatibility, and canonical owner URLs across user and org publishers.

## Reproduction Assessment

Yes. Source inspection shows an org handle reaches the transfer flow as toUserHandle, but only active users or personal user publishers can resolve, so org publishers return null before a transfer can be created.

## Solution Assessment

Unclear. Transfer-to-org is the right user outcome for existing skills, but the maintainable path is the documented publisher-targeted transfer model rather than adding org lookup to a user-only schema.
