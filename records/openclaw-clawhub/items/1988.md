## Summary

Keep open: the live third-party `agentic-money` artifact still contains the ambiguous wallet SDK sanitization guardrail, and ClawHub policy keeps issues open unless a strong close reason applies. This is not a bundled current-main source bug, but it is a concrete security/moderation follow-up for the published skill owner or ClawHub moderators.

## What This Changes

The issue asks for concrete validation rules in the published `agentic-money` `SKILL.md` before user-derived wallet, network, amount, address, task, or capability values are passed into SDK calls.

## Best Possible Solution

Have the skill owner or ClawHub moderators publish or request a new `agentic-money` version with explicit wallet SDK validation, approval, credential metadata, and dependency rules, while keeping broader payment-skill governance in https://github.com/openclaw/clawhub/issues/2020.

Yes for the live ClawHub artifact: fetching the published `SKILL.md` reproduces the vague guardrail, and the scan endpoint confirms the wallet private-key and transaction-signing context. It is not a current-main source reproduction because the affected skill is not bundled in this repository.

## Solution Assessment

Unclear. Replacing the vague wording with testable validation rules is the right direction, but the exact Ethereum, SDK, dependency, shell-safety, and approval rules should be chosen by the skill owner or security/moderation reviewers before a replacement artifact is published.
