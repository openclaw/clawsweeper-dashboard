## Summary

Keep open: this is a valid security-sensitive report against a live third-party ClawHub skill artifact, not something current main has already fixed. The live `agentic-money` SKILL.md still uses ambiguous sanitization wording in a wallet/ETH transaction context, and ClawHub issue-cleanup policy keeps issues open for maintainer follow-up.

## What This Changes

The issue asks the published `agentic-money` skill to replace an undefined “unsanitized user input” guardrail with concrete validation rules for wallet SDK parameters.

## Best Possible Solution

Have the skill owner or ClawHub moderators publish or request a new `agentic-money` version with explicit wallet SDK validation, approval, credential, and dependency rules, while routing broader payment-skill policy through the open RFC.

## Reproduction Assessment

Yes for the live ClawHub artifact: fetching SKILL.md reproduces the vague guardrail, and the scan endpoint confirms the wallet private-key and transaction-signing context. It is not a current-main source bug because the affected skill is not bundled in this repository.

## Solution Assessment

Unclear. Replacing vague wording with testable validation rules is the right direction, but the exact Ethereum, SDK, dependency, shell-safety, and approval rules should be chosen by the skill owner or a security maintainer before a replacement artifact is published.
