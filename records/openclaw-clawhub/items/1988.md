## Summary

Keep open: repository policy keeps ClawHub issues open, and the live third-party `agentic-money` artifact still contains ambiguous sanitization guidance in a wallet-signing flow. This needs skill-owner and security/moderation follow-up, not source-repo cleanup closure.

## What This Changes

The issue asks for concrete, testable sanitization semantics in the published `agentic-money` skill guardrail.

## Best Possible Solution

Have the skill owner or ClawHub moderators publish or require a new `agentic-money` version with explicit wallet input validation, declared credential metadata, dependency pinning guidance, and per-transaction approval rules, while keeping broader payment trust policy in the related RFC.

## Reproduction Assessment

Yes for the live ClawHub artifact: fetching `SKILL.md` reproduces the vague guardrail, and the scan endpoint confirms the private-key and transaction-signing context. This is not a current-main source bug because the affected skill is a published third-party artifact rather than bundled repo content.

## Solution Assessment

Unclear. Replacing vague wording with testable validation rules is directionally right, but the exact Ethereum, SDK, shell-safety, dependency, and approval policy should be chosen by the skill owner or security/moderation reviewers before a replacement artifact is published.
