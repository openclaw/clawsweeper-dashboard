## Summary

Keep open: the live third-party `agentic-money` artifact still has ambiguous sanitization guidance in a private-key-backed wallet/payment flow, and current main exposes file/scan/moderation surfaces rather than a source patch that can rewrite the published artifact. The broader payments trust-policy discussion remains open at https://github.com/openclaw/clawhub/issues/2020.

## What This Changes

The issue asks for concrete, testable input-validation rules in the published `agentic-money` skill instead of the vague “unsanitized user input” guardrail.

## Best Possible Solution

Have the skill owner or ClawHub moderators request or publish a new `agentic-money` version with explicit wallet input validation and approval rules, then rescan it; use the open Featured Payments RFC for broader payment-skill trust policy.

## Reproduction Assessment

Yes for the live ClawHub artifact: fetching `SKILL.md` reproduces the vague guardrail, and the version/scan endpoints confirm the wallet/payment context. It is not a current-main source bug because the affected content is a third-party published artifact and no local `agentic-money` source exists in this repo.

## Solution Assessment

Unclear as a repo fix: replacing vague wording with testable validation rules is directionally right, but the exact wallet validation and approval policy should be chosen by the skill owner or security/moderation reviewers before a replacement artifact is published.
