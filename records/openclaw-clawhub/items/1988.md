## Summary

Keep open for maintainer security/product follow-up: the live published skill still contains the ambiguous sanitization guardrail, and the remaining decision is how ClawHub or the skill publisher should define and enforce wallet/SDK input validation rather than a safe repository autofix.

## What This Changes

The issue requests concrete, testable validation semantics for the published `zscole/agentic-money` skill's vague “unsanitized user input” SDK guardrail.

## Best Possible Solution

A maintainer/security reviewer should decide whether to require the publisher to update the live skill, open a formal moderation report, or add platform policy guidance for high-risk wallet skills.

## Reproduction Assessment

Yes. The live SKILL.md still contains the vague sanitization guardrail while its examples and workflow require wallet/SDK calls with user-shaped inputs; I verified this from the public ClawHub API without executing transactions.

## Solution Assessment

Unclear. Replacing the vague guardrail with testable validation rules is the right shape, but the exact wallet, network, and string validation policy should be set by a maintainer/security reviewer or the skill publisher before changing live guidance.
