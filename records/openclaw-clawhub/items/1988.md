## Summary

Keep open: ClawHub issue policy keeps issues open, and the live third-party `agentic-money` artifact still has ambiguous sanitization guidance in a wallet/payment flow. Current main exposes file and scan APIs plus artifact-based moderation docs, but a source-repo patch would not update the already published skill artifact.

## What This Changes

The issue asks for the third-party `agentic-money` skill to define concrete, enforceable sanitization rules before user-derived values are passed to wallet SDK calls.

## Best Possible Solution

Have the skill owner or ClawHub moderators request or publish a new `agentic-money` version with explicit wallet input validation and approval rules, while coordinating broader payment trust expectations with https://github.com/openclaw/clawhub/issues/2020.

## Reproduction Assessment

Yes for the live ClawHub artifact: fetching `SKILL.md` reproduces the vague guardrail, and the version/scan endpoints confirm the wallet/payment context. This is not a current-main source bug because the affected content is a third-party published artifact.

## Solution Assessment

Unclear as a repo fix: replacing vague wording with testable rules is directionally right, but the exact wallet validation and approval policy should be chosen by the skill owner or security/moderation reviewers before a replacement artifact is published.
