## Summary

Keep open: ClawHub issue policy does not auto-close issues, and the live third-party `agentic-money` artifact still has ambiguous sanitization wording in a financially sensitive wallet-signing skill. This needs skill-owner or security/moderation follow-up rather than a conservative source-repo cleanup close.

## What This Changes

Define concrete validation semantics for user-derived SDK parameters in the published `agentic-money` skill instead of relying on a vague `unsanitized user input` guardrail.

## Best Possible Solution

Have the skill owner or ClawHub moderators publish or request a new `agentic-money` version with explicit wallet input validation, credential metadata, dependency pinning guidance, and approval rules.

## Reproduction Assessment

Yes for the live ClawHub artifact: fetching `SKILL.md` reproduces the vague guardrail, and the scan endpoint confirms the wallet private-key and transaction-signing context. It is not a current-main source bug because the affected skill is not bundled in this repository.

## Solution Assessment

Unclear. Replacing vague wording with testable validation rules is the right direction, but the exact Ethereum, SDK, dependency, shell-safety, and approval rules should be chosen by the skill owner or security/moderation reviewers before a replacement artifact is published.
