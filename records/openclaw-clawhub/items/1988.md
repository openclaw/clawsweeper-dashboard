## Summary

Keep open: ClawHub policy keeps issues open, and the live third-party `agentic-money` artifact still has ambiguous sanitization wording in a financially sensitive wallet-signing skill.

## What This Changes

The issue asks for the published `agentic-money` skill to replace an undefined `unsanitized user input` guardrail with concrete, enforceable validation rules for wallet/SDK calls.

## Best Possible Solution

Have the skill owner or ClawHub moderators publish or require a new `agentic-money` version with explicit wallet input validation, credential metadata, dependency pinning guidance, and approval rules.

## Reproduction Assessment

Yes for the live ClawHub artifact: fetching `SKILL.md` reproduces the vague guardrail, and the scan endpoint confirms private-key and transaction-signing context. It is not a current-main source bug because the affected third-party skill is not bundled in this repository.

## Solution Assessment

Unclear. Replacing vague wording with testable validation rules is the right direction, but the exact Ethereum, SDK, dependency, shell-safety, and approval rules should be chosen by the skill owner or security/moderation reviewers before a replacement artifact is published.
