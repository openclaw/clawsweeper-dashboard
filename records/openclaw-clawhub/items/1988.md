## Summary

Keep open: ClawHub policy keeps issues open, and the live third-party `agentic-money` artifact still has ambiguous wallet SDK sanitization guidance that needs skill-owner or security/moderation judgment.

## What This Changes

The issue asks for `agentic-money` to replace vague “unsanitized user input” wording with concrete validation rules before wallet SDK calls.

## Best Possible Solution

Have the skill owner or ClawHub moderators publish or request a new `agentic-money` version with explicit wallet SDK validation, approval, credential, and dependency rules.

## Reproduction Assessment

Yes for the live ClawHub artifact: fetching SKILL.md reproduces the vague guardrail, and the scan endpoint confirms the wallet private-key and transaction-signing context. It is not a current-main source bug because the affected skill is not bundled in this repository.

## Solution Assessment

Unclear. Replacing vague wording with testable validation rules is the right direction, but the exact Ethereum, SDK, dependency, shell-safety, and approval rules should be chosen by the skill owner or a security maintainer before a replacement artifact is published.
