## Summary

Keep open. This is an open ClawHub issue, and current main plus live routing no longer match the original `clawhub.ai` to `www.clawhub.ai` redirect theory closely enough to close or queue a blind source fix.

## What This Changes

The issue reports CLI token authentication failing because requests to the default apex registry allegedly redirect to `www.clawhub.ai` and lose the bearer Authorization header.

## Best Possible Solution

Maintainers should verify the current published CLI with a fresh production token on the apex host, then either consolidate this report into the broader Unauthorized cluster or land one focused fix at the proven failing layer.

## Reproduction Assessment

No. A live dummy-token probe no longer reproduces the reported apex-to-www redirect, and high-confidence reproduction now requires a real production token and account state.

## Solution Assessment

Unclear. The historical `https://www.clawhub.ai` default switch is not the best proven fix because current main and live routing make `https://clawhub.ai` the canonical auth-sensitive host.
