Keep open. ClawHub cleanup policy keeps issues open for maintainer judgment, and the current evidence does not prove the production auth path is resolved: source still verifies tokens through the canonical registry, while live routing no longer matches the original non-www to www redirect theory.

The issue requests resolving CLI token auth failures caused by a reported `clawhub.ai` to `www.clawhub.ai` redirect stripping bearer Authorization headers.

## Best Possible Solution

The desired end state is a verified canonical-domain auth flow where fresh API tokens work with the default CLI registry, with any remaining Unauthorized behavior consolidated under the broader auth tracker or fixed at the proven failing layer.

## Solution Assessment
