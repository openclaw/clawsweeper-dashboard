## Summary

Keep open. Current main still uses the non-www canonical host, the reported non-www to www redirect no longer reproduces with a dummy bearer request, and ClawHub issue cleanup policy keeps issues open unless maintainers verify and close them.

## What This Changes

The issue reports that CLI token authentication fails when the default registry redirects between `clawhub.ai` and `www.clawhub.ai`, stripping the bearer header.

## Best Possible Solution

Maintainers should verify the current published CLI with a real token, then either close or consolidate this with #99 if non-www works, or make one consistent canonical-domain fix across hosting, CLI defaults, discovery, docs, and tests.

## Reproduction Assessment

Unclear. The original non-www to www redirect does not reproduce with a dummy bearer request on May 2, 2026, but a real production token is needed to verify the actual CLI auth flow.

## Solution Assessment

Unclear. The closed #101 direction of switching defaults to `www.clawhub.ai` is not clearly the best fix now that live traffic redirects `www` back to non-www and current source/docs intentionally advertise non-www.
