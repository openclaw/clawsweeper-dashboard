## Summary

Keep open. Current main has not made the requested switch to `https://www.clawhub.ai`, live traffic now redirects `www` back to non-www rather than reproducing the original non-www to `www` redirect with a dummy token, and this ClawHub issue needs maintainer verification with a real production token before it can close or consolidate with #99.

## What This Changes

The issue asks for CLI authentication to work with the default registry without losing bearer authorization across a canonical-host redirect.

## Best Possible Solution

Maintainers should verify the current published CLI against production with a real token. If the default non-www flow works, close or consolidate this with #99; if it still fails, make one consistent canonical-domain/auth fix across hosting configuration, CLI defaults, well-known discovery, docs, and focused CLI/auth tests.
