Keep open. ClawHub cleanup policy keeps issues open, and current main plus live routing no longer support the original apex-to-www redirect theory strongly enough to make a source change without maintainer production-token verification.

## What This Changes

The issue requests fixing CLI authentication failures allegedly caused by `https://clawhub.ai` redirecting to `https://www.clawhub.ai` and stripping the bearer token.

## Best Possible Solution

Maintainers should verify a fresh production token against the canonical apex host, then either consolidate this with the related Unauthorized reports or fix the proven failing layer with focused CLI/auth coverage.

## Reproduction Assessment

No. The specific `clawhub.ai` to `www.clawhub.ai` redirect no longer reproduces with a dummy bearer request, while the full valid-token CLI failure requires a real production token and account state.

## Solution Assessment

Unclear. Switching defaults to `www.clawhub.ai` matched the original workaround, but current source, docs, discovery, and live routing make the apex host canonical, so maintainer production-token verification should come before a code or hosting change.
