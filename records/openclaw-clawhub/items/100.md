## Summary

Keep open. Current main still uses the non-www canonical registry, the originally reported non-www to www redirect no longer reproduces with a dummy bearer request, and this ClawHub issue needs maintainer verification with a real production token plus a canonical-domain decision.

## What This Changes

The issue reports CLI token authentication failing when the default `https://clawhub.ai` registry redirects across hosts and strips the bearer Authorization header.

## Best Possible Solution

Maintainers should verify the current published CLI with a real production token, then either close or consolidate this with #99 if non-www works, or make one consistent canonical-domain fix across hosting, CLI defaults, discovery, docs, and tests.

## Reproduction Assessment

Unclear. The original non-www to www redirect did not reproduce with a dummy bearer request on May 5, 2026, but a real production token and current published CLI run are needed to verify the actual auth flow.

## Solution Assessment

Unclear. Switching defaults to `www.clawhub.ai` is not clearly the best fix now that current source, discovery, docs, and live redirects all point toward non-www as canonical.
