## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main plus live routing do not prove this redirect-based auth report is fixed or still failing with a real token.

## What This Changes

The issue reports that CLI token auth fails when the default registry crosses between `clawhub.ai` and `www.clawhub.ai`, causing bearer Authorization to be lost before `/api/v1/whoami`.

## Best Possible Solution

Verify fresh production tokens against the current published CLI and canonical non-www host, then either close or consolidate this with the broader Unauthorized tracker, or land one focused auth/domain fix at the proven failing layer.

## Reproduction Assessment

No. Source inspection confirms a redirect-sensitive bearer-token path, but the live dummy-token probe no longer reproduces the reported `clawhub.ai` to `www.clawhub.ai` redirect and a real production token is required for high-confidence verification.

## Solution Assessment

Unclear. Changing defaults to `https://www.clawhub.ai` is not currently proven best because current source, discovery, docs, maintainer PR comments, and live routing all point at non-www as canonical.
