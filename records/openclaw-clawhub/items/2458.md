## Summary

The branch still contains useful unique feature work, but it is not merge-ready: it conflicts with current main, reintroduces static official-publisher allowlists that current main replaced with database-backed policy, changes the public verification trust boundary without the required durable spec, and provides only mocked/test evidence rather than a real Convex publish/readback run.

## What This Changes

The PR lets official organization publishers upload a root `skill-card.md`, preserves it through CLI publishing, and serves it as trusted card and verification metadata instead of generating a replacement.

## Best Possible Solution

Keep server-side `officialPublishers` rows as the sole authorization source, let the CLI upload a requested card without embedding trust policy while the server accepts or rejects it, document eligibility/revocation and verification semantics in a focused spec, and validate the rebased flow on real Convex state.
