## Summary

Keep open: current main and v0.22.0 still do not show Downloads alongside Installs on public skill-card/list surfaces. The related open PRs cover either the skill detail sidebar only or a broader downloads-first policy, so this card-specific dual-metric request still needs maintainer product judgment.

## What This Changes

The issue requests showing Downloads alongside Installs on public skill cards so publishers can show both reach and unique install adoption.

## Best Possible Solution

Maintainers should decide the public metric hierarchy, then either add Downloads as secondary compact card context or deliberately keep cards installs-only with explanatory docs/detail-page treatment.

## Reproduction Assessment

Yes, at source level: current main renders compact skill cards/list rows with install counts and no Downloads metric beside them. I did not run a browser because source and tests already show the requested card display is absent.

## Solution Assessment

Unclear until maintainers choose the metric hierarchy. A narrow UI implementation is straightforward, but it partially revises the recent installs-primary rollout unless Downloads is explicitly accepted as secondary card context.
