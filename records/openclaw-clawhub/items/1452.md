## Summary

Keep open. Current main still intentionally blocks sign-in for self-deleted/deactivated accounts, and the ClawHub cleanup policy says issues should remain open unless maintainers resolve or supersede them.

## What This Changes

The issue asks ClawHub to remove a stale GitHub OAuth binding or restore a deleted account so the same GitHub account can sign in again.

## Best Possible Solution

Define an explicit audited recovery or re-registration policy for verified self-deleted accounts, or reaffirm irreversible deletion and document the support path while preserving banned and abuse-disabled blocks.

## Reproduction Assessment

Yes, at source level: deleteAccount sets deactivatedAt/purgedAt, the OAuth callback calls handleDeletedUserSignIn, and auth tests assert rejected reauth for deactivated and legacy deleted users. I did not run a live GitHub OAuth flow.

## Solution Assessment

No, not as a one-off binding deletion or silent account restore. The maintainable path needs a policy-backed audited recovery or unlink flow that distinguishes self-deleted accounts from banned or abuse-disabled identities.
