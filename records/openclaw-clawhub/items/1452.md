## Summary

Keep open. This is a ClawHub issue, and the cleanup policy keeps issues open; current main still intentionally blocks deleted/deactivated GitHub OAuth reauth, so resolving it needs maintainer account-recovery and security policy rather than conservative cleanup.

## What This Changes

Issue requests that ClawHub remove a stale GitHub OAuth binding or restore a deleted account so the same GitHub identity can sign in again.

## Best Possible Solution

Define an audited recovery or re-registration flow that distinguishes self-deleted accounts from banned or abuse-disabled accounts, then update Convex auth/users code, operator guidance, public docs, and regression tests around that policy.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, and the OAuth callback rejects that user on the next GitHub sign-in; I did not run a live OAuth flow.

## Solution Assessment

No. One-off OAuth binding removal or account restoration is not the best durable fix because current docs/specs make deletion irreversible and recovery must stay separate from banned-account handling.
