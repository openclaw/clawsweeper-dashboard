## Summary

Keep open. Current main still deliberately rejects GitHub reauth for deleted/deactivated users, and ClawHub cleanup policy keeps issues open; resolving this needs account-recovery/security policy rather than conservative auto-closure.

## What This Changes

The issue asks maintainers to remove a stale GitHub OAuth binding or restore a deleted account so the same GitHub login can sign in again.

## Best Possible Solution

Define an audited recovery or re-registration flow that can unlink or rekey only eligible self-deleted accounts while preserving bans, abuse blocks, and deletion semantics.

## Reproduction Assessment

Yes. Source and tests show the path: deleteAccount marks the user deactivated, and the OAuth callback throws for deactivated users; I did not run a live GitHub OAuth flow.

## Solution Assessment

No. Ad hoc per-account restoration or authAccounts deletion is not the best repository-level fix; the maintainable path is an explicit audited recovery/unlink policy with tests for self-deleted, banned, and abuse-disabled accounts.
