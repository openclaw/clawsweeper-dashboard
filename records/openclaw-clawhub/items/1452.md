## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only allows auto-closing pull requests that current main certainly implements. Current main still intentionally blocks deleted/deactivated GitHub OAuth reauth, documents self-deletion as irreversible, and has no restore or OAuth-binding unlink flow for this support case.

## What This Changes

The issue requests removal of a stale GitHub OAuth binding or restoration of a deleted ClawHub account so the same GitHub account can sign in again.

## Best Possible Solution

Define an explicit, audited account recovery or re-registration policy for user-initiated deletions, then implement the chosen flow in Convex auth/admin code with tests, UI copy, and docs while keeping banned and abuse-disabled accounts blocked.
