## Summary

Keep open. This is an issue, and the ClawHub cleanup policy only auto-closes PRs that are already implemented on main; current main still intentionally blocks deleted/deactivated GitHub OAuth reauth, so the remaining work is maintainer account-recovery and security policy handling.

## What This Changes

The issue requests restoring a deleted ClawHub account or clearing the GitHub OAuth binding so the same GitHub account can sign in again.

## Best Possible Solution

Define an explicit audited recovery or re-registration flow for user-initiated deletions that preserves banned and abuse-disabled account blocks, then update auth code, operator tooling, docs/specs, and Convex tests together.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount writes deactivatedAt/purgedAt, GitHub OAuth completion calls handleDeletedUserSignIn, and tests assert deleted/deactivated users are rejected rather than re-registered.

## Solution Assessment

No. One-off OAuth binding removal or account restoration is not the best durable solution because recovery must distinguish self-deleted accounts from banned or abuse-disabled accounts and preserve auditability.
