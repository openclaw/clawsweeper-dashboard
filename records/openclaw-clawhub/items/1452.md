## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still deliberately blocks deleted/deactivated GitHub OAuth reauth while documenting user deletion as irreversible and offering no OAuth-binding unlink or account-restore path.

## What This Changes

The issue asks ClawHub to clear a stale GitHub OAuth binding or restore a deleted account so the same GitHub identity can sign in again.

## Best Possible Solution

Define and ship an explicit audited recovery or re-registration flow that distinguishes self-deletion from bans, updates Convex auth/account code, UI/docs copy, and tests together.

## Reproduction Assessment

Yes. The code-level reproduction is covered by auth tests that construct deactivated or legacy deleted users and expect GitHub reauth to throw DELETED_ACCOUNT_REAUTH_MESSAGE; the linked reports describe the matching web sign-in loop.

## Solution Assessment

Unclear. The requested manual unlink or restore would address this reporter, but the maintainable solution needs a policy-backed recovery path that preserves deletion semantics and ban separation.
