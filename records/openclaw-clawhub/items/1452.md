## Summary

Keep open. Current main intentionally blocks deleted/deactivated GitHub reauthentication, the repository cleanup policy does not auto-close issues, and changing this requires maintainer/operator security and product direction.

## What This Changes

Reporter asks ClawHub to remove a stale GitHub OAuth binding or restore a deleted account so the same GitHub login can sign in again.

## Best Possible Solution

Adopt a maintainer-approved account recovery or re-registration policy with audited same-provider unlink/rekey handling, ban safeguards, operator tooling, docs/spec updates, and regression tests.

## Reproduction Assessment

Yes, by source inspection. deleteAccount sets deactivatedAt/purgedAt and the OAuth completion path calls handleDeletedUserSignIn, which rejects that state; I did not run a live production OAuth flow or inspect the reporter’s row.

## Solution Assessment

No, ad hoc binding removal or restoration from an issue comment is not the best solution. The maintainable path needs explicit recovery policy, same-provider verification, audit logging, and ban/deletion safeguards, or an explicit decision that deletion remains irreversible.
