## Summary

Keep open. Current main still deliberately blocks sign-in for deleted/deactivated GitHub OAuth users, and ClawHub cleanup policy keeps issues open unless a maintainer resolves them; any fix needs product and security direction around account recovery, deletion permanence, and ban separation.

## What This Changes

Request to restore a deleted ClawHub account or remove its GitHub OAuth binding so the same GitHub account can sign in again.

## Best Possible Solution

Define an explicit audited recovery or re-registration flow that handles eligible self-deleted accounts, keeps banned and abuse-disabled accounts blocked, and updates Convex auth, admin tooling, UI copy, docs, and tests together.

## Reproduction Assessment

Yes. Source and tests provide a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, then the OAuth callback calls handleDeletedUserSignIn and rejects the returning GitHub identity.

## Solution Assessment

Unclear. The requested restore or OAuth unlink could solve the individual account problem, but the maintainable solution needs maintainer policy first so privacy deletion semantics and abuse controls are not changed piecemeal.
