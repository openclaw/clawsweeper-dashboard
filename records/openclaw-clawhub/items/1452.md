## Summary

Keep open. Current main still blocks GitHub sign-in for deleted/deactivated ClawHub accounts and documents that deletion is irreversible, so this needs maintainer product/security direction rather than cleanup closure.

## What This Changes

The issue requests clearing or restoring a stale GitHub OAuth/account binding so the reporter can sign in again after deleting their ClawHub account.

## Best Possible Solution

Define an audited recovery or re-registration flow that can unlink or rekey only verified self-deleted accounts, preserves ban and abuse blocks, and updates UI, docs, specs, and Convex tests to match the chosen policy.

## Reproduction Assessment

Yes, at source level: deleteAccount marks the account deactivated, the OAuth callback rejects deactivated users, and tests assert that deleted/deactivated users cannot sign in again. I did not run a live GitHub OAuth flow.

## Solution Assessment

No, not as a one-off restore or binding deletion. The maintainable solution is a policy-backed, audited recovery/re-registration path that keeps bans and abuse-disabled accounts blocked.
