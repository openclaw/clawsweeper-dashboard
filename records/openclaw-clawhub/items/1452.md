## Summary

Keep open. Current main still blocks deleted-account GitHub OAuth reauth by design, and this issue needs maintainer product/security direction rather than conservative cleanup closure.

## What This Changes

The issue requests restoring a deleted ClawHub account or clearing the stale GitHub OAuth binding so the same GitHub identity can sign in again.

## Best Possible Solution

Define an explicit, audited account-recovery or re-registration policy for verified self-deleted accounts, or reaffirm irreversible deletion and document an operator handling path that keeps banned and abuse-disabled identities blocked.

## Reproduction Assessment

Yes, at source level: deleting an account creates deactivatedAt/purgedAt state, and the OAuth callback rejects that state through handleDeletedUserSignIn. I did not run a live GitHub OAuth flow.

## Solution Assessment

No, not as a one-off binding deletion or silent restore. The maintainable path needs a policy-backed recovery or unlink flow that distinguishes self-deleted accounts from banned or abuse-disabled identities.
