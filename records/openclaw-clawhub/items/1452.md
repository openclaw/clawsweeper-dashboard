## Summary

Keep open. Current main intentionally blocks deleted-account GitHub reauth and this ClawHub cleanup policy does not auto-close issues, so this needs maintainer product/security handling rather than cleanup closure.

## What This Changes

Request account recovery or OAuth unlinking so a user who deleted their ClawHub account can sign in again with the same GitHub identity.

## Best Possible Solution

Define an explicit audited recovery or re-registration flow for verified self-deleted accounts, or reaffirm irreversible deletion and document an operator support path that preserves banned and abuse-disabled blocks.

## Reproduction Assessment

Yes, at source level: deleteAccount creates the deactivatedAt/purgedAt state and the OAuth callback rejects that state during sign-in, with tests asserting the rejection. I did not run a live GitHub OAuth flow.

## Solution Assessment

No, not as an ad hoc binding deletion or silent restore. The maintainable solution is a policy-backed recovery or unlink path that distinguishes self-deleted users from banned or abuse-disabled identities.
