## Summary

Keep open. Current main still blocks deleted/deactivated GitHub OAuth sign-in by design, and this issue is part of an unresolved account-recovery cluster that needs maintainer product and security direction.

## What This Changes

The reporter asks for GitHub OAuth binding removal or account restoration so they can sign in again after deleting their ClawHub account.

## Best Possible Solution

Define an explicit audited recovery or re-registration policy for verified self-deleted accounts, or reaffirm irreversible deletion and document the support path while preserving banned and abuse-disabled account blocks.

## Reproduction Assessment

Yes, at source level: deleteAccount sets deactivatedAt/purgedAt, the OAuth callback calls handleDeletedUserSignIn, and auth tests assert rejected reauth for deactivated and legacy deleted users. I did not run a live GitHub OAuth flow.

## Solution Assessment

No, not as a one-off binding deletion or silent account restore. The maintainable path needs a policy-backed audited recovery or unlink flow that distinguishes self-deleted accounts from banned or abuse-disabled identities.
