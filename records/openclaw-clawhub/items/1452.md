## Summary

Keep open. Current main still intentionally blocks deleted/deactivated GitHub reauth and the ClawHub cleanup policy does not auto-close issues; resolving this needs an account-recovery or unlink policy with auth/security review.

## What This Changes

Requests account recovery or GitHub OAuth unbinding after deleting a ClawHub account and being unable to sign in again with the same GitHub identity.

## Best Possible Solution

Define an audited recovery or re-registration flow that can unlink or rekey only verified self-deleted accounts, keeps banned and abuse-disabled accounts blocked, updates UI/spec/docs, and adds Convex coverage for re-signup, operator recovery, and banned-account rejection.

## Reproduction Assessment

Yes, at source level: deleteAccount marks the user deactivated, the OAuth callback rejects deactivated users, and tests assert that deleted/deactivated users cannot sign in again. I did not run a live GitHub OAuth flow.

## Solution Assessment

No, not as a one-off unlink or restore. The maintainable fix is a policy-backed audited recovery path that distinguishes self-deleted accounts from bans and abuse states.
