## Summary

Keep open. For openclaw/clawhub, this cleanup pass should not auto-close issues, and current main still intentionally treats user-initiated deletion as irreversible, so changing the GitHub OAuth re-sign-in behavior needs maintainer product/security direction.

## What This Changes

The issue asks maintainers to remove the retained GitHub OAuth binding or restore the deleted ClawHub account so the same GitHub identity can sign in again.

## Best Possible Solution

Define an audited account recovery or re-registration policy that distinguishes self-deleted accounts from banned or abuse-disabled identities, then update Convex auth/account code, operator surfaces, specs/docs, and regression tests to enforce it.

## Reproduction Assessment

Yes, by source inspection. A user who runs deleteAccount gets deactivatedAt/purgedAt set, and the OAuth callback rejects that same deactivated user on the next GitHub sign-in; I did not run a live OAuth flow.

## Solution Assessment

No. A one-off binding removal or account restore is not the best durable solution because current policy says deletion is irreversible and the fix must preserve banned/abuse-disabled account blocking.
