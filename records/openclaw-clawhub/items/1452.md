## Summary

Keep open. Current main still deliberately blocks GitHub OAuth reauth for permanently deleted accounts, and the ClawHub cleanup policy keeps issues open unless a maintainer resolves them; fixing this needs account recovery and security policy direction.

## What This Changes

The issue asks maintainers to clear or restore a deleted account's GitHub OAuth binding so the same GitHub account can sign in again.

## Best Possible Solution

Define an explicit audited recovery or re-registration flow for self-deleted accounts, with tests and docs that preserve banned-account separation and irreversible deletion semantics.

## Reproduction Assessment

Yes, from source and existing unit tests. deleteAccount sets deactivatedAt/purgedAt, then the OAuth callback calls handleDeletedUserSignIn and throws for that deactivated user before sign-in can complete.

## Solution Assessment

Unclear. A one-off restore or OAuth unlink could solve this reporter's account, but the maintainable repository fix needs an explicit policy and audited implementation for recovery, re-registration, and banned-account separation.
