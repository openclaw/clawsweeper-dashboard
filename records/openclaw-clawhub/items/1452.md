## What This Changes

The issue asks for a deleted ClawHub account’s GitHub OAuth binding to be removed or restored so the same GitHub account can sign in again.

## Best Possible Solution

Define an explicit, audited recovery or re-registration flow for verified self-deleted accounts, or reaffirm irreversible deletion and give support a documented account-specific handling path that preserves banned and abuse-disabled blocks.

## Reproduction Assessment

Yes, at source level: deleteAccount creates deactivatedAt/purgedAt state, and handleDeletedUserSignIn rejects that state during OAuth sign-in with tests asserting the rejection. I did not run a live GitHub OAuth flow.

## Solution Assessment

No, not as a one-off binding deletion or silent restore. The maintainable solution is a policy-backed recovery/unlink path that distinguishes self-deleted users from banned or abuse-disabled identities.
