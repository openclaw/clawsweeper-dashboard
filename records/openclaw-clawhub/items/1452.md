## Summary

Keep open. Current main still treats user-initiated account deletion as a permanent GitHub reauth block, and the repository cleanup policy for openclaw/clawhub does not auto-close issues; resolving this needs account recovery and security policy direction.

## What This Changes

The issue asks ClawHub to unlink the stale GitHub OAuth binding or restore the deleted account so the same GitHub identity can sign in again.

## Best Possible Solution

Define an audited self-deletion recovery or re-registration policy, then update Convex auth/account code, operator UI or tooling, docs/specs, and regression tests to enforce it while keeping banned identities blocked.

## Reproduction Assessment

Yes. Source inspection shows deleteAccount sets deactivatedAt/purgedAt and the OAuth callback rejects that user on the next sign-in; current auth tests assert the same blocking path, though I did not run a live OAuth flow.

## Solution Assessment

No. Directly removing an OAuth binding or restoring a row is not the safest standalone fix because current specs intentionally make deletion irreversible and abuse-related account blocks must remain enforceable.
