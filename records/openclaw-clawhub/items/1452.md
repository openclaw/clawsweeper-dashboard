## Summary

Keep open. The ClawHub cleanup policy keeps issues open, and current main still intentionally blocks deleted/deactivated GitHub OAuth reauth; resolving this needs maintainer account-recovery and security policy rather than conservative cleanup.

## What This Changes

The issue asks for the deleted account's GitHub OAuth binding to be removed or the deleted account restored so the same GitHub identity can sign in again.

## Best Possible Solution

Define an audited account recovery or re-registration flow that can unlink or rekey only eligible self-deleted accounts, keeps banned and abuse-disabled accounts blocked, and updates Convex auth, operator guidance, docs/specs, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount marks the user deactivated/purged, and the next GitHub OAuth callback rejects that user in handleDeletedUserSignIn; I did not run a live OAuth flow.

## Solution Assessment

No. One-off OAuth binding removal or account restoration is not the best durable solution because it would bypass the current irreversible-deletion boundary without a policy split for self-deleted, banned, and abuse-disabled accounts.
