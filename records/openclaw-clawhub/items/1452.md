## Summary

Keep open. Current main still intentionally rejects deleted/deactivated GitHub OAuth reauth, documents user deletion as irreversible, and has no restore or OAuth-binding unlink path; ClawHub cleanup policy also keeps issues open rather than auto-closing them.

## What This Changes

The issue asks for a way to sign in again with the same GitHub account after deleting a ClawHub account, either by removing the stale OAuth binding or restoring the deleted account.

## Best Possible Solution

Define and ship an audited account recovery or re-registration policy that can safely handle user-initiated deletions while keeping banned and abuse-disabled accounts blocked, then update Convex auth code, admin/support tooling, UI copy, docs, and tests together.

## Reproduction Assessment

Yes. The source and tests give a high-confidence reproduction path: delete an account, sign in with the same GitHub identity, and afterUserCreatedOrUpdated calls handleDeletedUserSignIn, which rejects deactivated users with the deleted-account error.

## Solution Assessment

No, not as a one-off support unlink. The maintainable solution needs an explicit recovery policy and ban-separation checks before implementing restore, rekey, or OAuth-binding deletion behavior.
