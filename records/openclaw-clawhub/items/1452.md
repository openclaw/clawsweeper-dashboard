## Summary

Keep open. Current main still intentionally rejects GitHub sign-in for deleted/deactivated accounts, and this issue is a ClawHub account-recovery/security policy decision rather than cleanup-closable work.

## What This Changes

The issue requests a way to remove or restore a stale GitHub OAuth binding so a deleted ClawHub account can sign in again.

## Best Possible Solution

Define an audited recovery or re-registration flow for verified self-deleted accounts that preserves banned and abuse-disabled account blocks, then update code, tests, specs, and user-facing copy together.

## Reproduction Assessment

Yes, at source level: self-service deletion sets deactivatedAt/purgedAt, the OAuth callback rejects that state, and auth tests assert the rejection path. I did not run a live GitHub OAuth flow.

## Solution Assessment

No, a one-off OAuth binding deletion or account restore is not the best fix. The maintainable solution is a policy-backed, audited recovery path that distinguishes self-deletion from bans and abuse-disabled accounts.
