## Summary

Keep open. Current main still intentionally blocks GitHub OAuth sign-in for self-deleted/deactivated users, and the remaining work is a product/security decision about account recovery rather than conservative issue cleanup.

## What This Changes

Request to remove a stale GitHub OAuth binding or restore a deleted ClawHub account so the same GitHub account can sign in again.

## Best Possible Solution

Define an audited recovery or re-registration flow for verified self-deleted accounts that preserves banned and abuse-disabled blocks, updates policy/docs/UI, and adds Convex coverage for recovery and rejection paths.

## Reproduction Assessment

Yes, at source level: deleteAccount creates deactivatedAt/purgedAt state and the OAuth callback rejects that state, with tests asserting the rejection. I did not run a live GitHub OAuth flow.

## Solution Assessment

No, not as a one-off binding deletion or silent restore. The maintainable solution is a policy-backed recovery path that distinguishes self-deleted accounts from banned or abuse-disabled accounts.
