## Summary

Keep open. This is a ClawHub issue, and the repository cleanup policy only permits auto-closing PRs; current main still deliberately blocks reauth for permanently deleted GitHub-linked accounts, so the remaining work needs maintainer product/security direction rather than conservative cleanup.

## What This Changes

The issue asks maintainers to remove a stale GitHub OAuth binding or restore a deleted ClawHub account so the same GitHub identity can sign in again.

## Best Possible Solution

Define an audited recovery or re-registration flow that separates self-deleted accounts from banned or abuse-disabled accounts, then update Convex auth/account code, operator UI/docs, and regression tests to enforce it.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, and the next GitHub OAuth callback rejects that user through handleDeletedUserSignIn; I did not run a live OAuth flow.

## Solution Assessment

No. Removing the OAuth binding or restoring an account as a one-off is not the best durable solution because current main documents irreversible deletion and needs a policy split from banned-account recovery.
