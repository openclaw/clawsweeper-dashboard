## What This Changes

The issue requests account restore or GitHub OAuth binding cleanup after a user deletes their ClawHub account and can no longer sign in with the same GitHub identity.

## Best Possible Solution

Define an explicit audited recovery or re-registration policy for verified self-deleted accounts, or reaffirm irreversible deletion and document a support path that preserves banned and abuse-disabled blocks.

## Reproduction Assessment

Yes, at source level: deleteAccount creates the deactivatedAt/purgedAt state and the OAuth callback rejects that state during sign-in, with tests asserting the rejection. I did not run a live GitHub OAuth flow.

## Solution Assessment

No, not as a one-off binding deletion or silent restore. The maintainable solution is a policy-backed recovery or unlink flow that distinguishes self-deleted users from banned or abuse-disabled identities.
