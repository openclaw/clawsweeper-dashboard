## Summary

Keep open. Current main still intentionally blocks GitHub OAuth sign-in for deactivated/self-deleted accounts, and this account-recovery request crosses deletion policy, auth binding, and abuse-control boundaries that need maintainer product/security direction.

## What This Changes

The issue requests restoring a deleted account or clearing its GitHub OAuth binding so the same GitHub account can sign in again.

## Best Possible Solution

Define an audited recovery or re-registration flow for verified self-deleted accounts that preserves banned and abuse-disabled blocks, updates specs/UI/docs, and adds Convex coverage for re-signup, operator recovery, and rejection paths.

## Reproduction Assessment

Yes, at source level: deleteAccount creates a deactivated/purged user state, the OAuth callback rejects that state, and auth tests assert the rejection path. I did not run a live GitHub OAuth flow.

## Solution Assessment

No, not as a one-off binding deletion or silent restore. The maintainable solution is a policy-backed recovery path that distinguishes self-deleted accounts from banned or abuse-disabled accounts.
