## Summary

Keep open. Current main still deliberately blocks deleted/deactivated GitHub OAuth reauth, the account-recovery policy is still unresolved, and the openclaw/clawhub cleanup rule does not auto-close issues.

## What This Changes

The issue requests restoration or removal of a stale GitHub OAuth binding so the reporter can sign in again after deleting their account.

## Best Possible Solution

Finalize the account-recovery policy, then implement an audited restore, unlink, or explicit irreversible-support path across auth, account deletion, operator tooling, docs, and tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, OAuth calls handleDeletedUserSignIn, and tests assert deleted/deactivated users are rejected; I did not run a live OAuth flow.

## Solution Assessment

No. One-off OAuth binding removal or account restoration is not the best durable fix because recovery must be separated from banned or abuse-disabled accounts and audited as policy.
