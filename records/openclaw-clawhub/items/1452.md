## Summary

Keep open. Current main deliberately blocks GitHub sign-in after user-initiated account deletion, and the remedy crosses account recovery, deletion semantics, and abuse controls; ClawHub cleanup policy also keeps issues open rather than auto-closing them.

## What This Changes

The issue asks maintainers to clear or restore a deleted account's GitHub OAuth binding so the same GitHub identity can sign in again.

## Best Possible Solution

Define an explicit, audited recovery or re-registration policy that can unlink or rekey only eligible self-deleted accounts while keeping banned and abuse-disabled identities blocked, then update Convex auth/account code, UI/docs, and regression tests to match.

## Reproduction Assessment

Yes. Source inspection shows deleteAccount sets deactivatedAt/purgedAt and the OAuth callback rejects that user on the next sign-in; current tests assert the same blocking path, though I did not run a live OAuth flow.

## Solution Assessment

No. Directly removing the OAuth binding or restoring the account is not the safest standalone fix because current specs intentionally make deletion irreversible and bans must remain blocked.
