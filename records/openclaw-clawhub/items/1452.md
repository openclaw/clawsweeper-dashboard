## Summary

Keep open. This is a ClawHub issue, and repository cleanup policy only permits auto-closing PRs that current main certainly implements; current main still intentionally rejects GitHub OAuth sign-in for deactivated/deleted users and has no audited restore or OAuth-unlink path for this support case.

## What This Changes

The issue asks maintainers to restore a deleted ClawHub account or remove the stale GitHub OAuth binding so the same GitHub account can sign in again.

## Best Possible Solution

Define and ship an explicit, audited recovery or re-registration policy for self-deleted accounts that keeps banned/abuse-disabled identities blocked, then update Convex auth/user code, admin tooling, UI copy, docs, and regression tests together.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, then the OAuth afterUserCreatedOrUpdated callback calls handleDeletedUserSignIn and rejects that returning identity.

## Solution Assessment

Unclear. The requested restore or OAuth unlink can solve the individual support case, but it is not safely the best repo-level fix without a maintainer decision on deletion semantics, audit requirements, and banned-account separation.
