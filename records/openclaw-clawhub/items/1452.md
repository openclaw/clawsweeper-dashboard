## Summary

Keep open. Current main still intentionally blocks GitHub OAuth sign-in for deleted/deactivated accounts, the docs/specs describe deletion as non-recoverable, and this issue needs account-recovery/security policy plus possible production data handling rather than cleanup closure.

## What This Changes

The issue requests restoring a deleted ClawHub account or clearing its GitHub OAuth binding so the same GitHub account can sign in again.

## Best Possible Solution

Define an explicit audited recovery or re-registration flow for user-initiated deletions, keep banned and abuse-disabled accounts blocked, then update auth, operator tooling, docs/specs, and Convex tests together.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `deleteAccount` sets `deactivatedAt`/`purgedAt`, GitHub OAuth completion calls `handleDeletedUserSignIn`, and tests assert deleted/deactivated users are rejected rather than re-registered.

## Solution Assessment

No. One-off OAuth binding removal or account restoration is not the best durable fix because recovery must distinguish self-deleted accounts from banned or abuse-disabled accounts and preserve auditability.
