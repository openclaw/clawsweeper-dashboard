## Summary

Keep open. Current main still intentionally blocks GitHub OAuth reauth for deleted/deactivated accounts, and resolving this recurring support cluster requires maintainer account-recovery and security policy rather than conservative cleanup.

## What This Changes

The reporter requests either unlinking the stale GitHub OAuth binding or restoring the deleted ClawHub account so the same GitHub account can sign in again.

## Best Possible Solution

Define a maintainer-approved recovery or re-registration policy that distinguishes self-deleted accounts from banned or abuse-disabled accounts, then update OAuth, account deletion, operator tooling, docs, and regression tests around that policy.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, the OAuth callback calls handleDeletedUserSignIn, and that handler rejects the next sign-in for the same deactivated user; I did not run a live OAuth flow.

## Solution Assessment

No. One-off OAuth binding removal or account restoration is not the best durable fix because it changes the current irreversible-deletion contract and must preserve banned-account enforcement.
