## Summary

Keep open. This is a ClawHub issue, and the cleanup policy only permits auto-closing pull requests that current main certainly implements; current main still blocks same-GitHub sign-in after user-initiated deletion and the remaining fix needs account-lifecycle and security policy direction.

## What This Changes

Request to restore or unlink a self-deleted ClawHub account so the same GitHub OAuth identity can sign in again.

## Best Possible Solution

Define and ship an explicit, audited account recovery or re-registration policy that can handle self-deleted accounts without weakening banned-account separation, and update Convex auth code, admin/support tooling, UI copy, docs, and tests together.

## Reproduction Assessment

Yes. The source and tests provide a high-confidence reproduction path: deleteAccount sets deactivatedAt/purgedAt, then the OAuth afterUserCreatedOrUpdated callback calls handleDeletedUserSignIn and rejects the returning GitHub identity.

## Solution Assessment

Unclear. The requested restore or OAuth unlink could solve the user problem, but the maintainable solution needs maintainer policy first so privacy deletion semantics and abuse controls are not changed piecemeal.
