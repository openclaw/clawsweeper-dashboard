## Summary

Keep open. This is an issue, and the ClawHub cleanup policy only permits auto-closing implemented or mostly implemented pull requests; current main still intentionally blocks deleted/deactivated GitHub reauth, so resolving this needs maintainer product and security direction.

## What This Changes

The issue requests account restoration or GitHub OAuth identity unlinking so a user can sign in again after deleting a ClawHub account.

## Best Possible Solution

Define an audited recovery or re-registration flow that can unlink or rekey verified self-deleted accounts, keeps banned and abuse-disabled accounts blocked, updates spec/UI/docs, and adds Convex coverage for re-signup, operator recovery, and banned-account rejection.

## Reproduction Assessment

Yes, at source level: deleteAccount creates a deactivated account state, the OAuth callback rejects that state, and auth tests assert the rejection path. I did not run a live GitHub OAuth flow.

## Solution Assessment

No, not as a one-off unlink or restore. The maintainable solution is a policy-backed, audited recovery path that distinguishes user-initiated deletion from bans and abuse-disabled accounts.
