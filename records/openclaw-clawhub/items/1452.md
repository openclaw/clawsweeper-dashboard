## Summary

Keep open. Current main still treats user-initiated deletion as irreversible and rejects GitHub OAuth sign-in for deactivated or legacy deleted users, while the open report cluster shows unresolved account-recovery demand that needs maintainer product and security direction.

## What This Changes

The issue asks ClawHub to restore a deleted account or clear its GitHub OAuth binding so the same GitHub identity can sign in again.

## Best Possible Solution

Define an audited recovery or re-registration policy for self-deleted accounts that can clear or rekey only eligible GitHub bindings while preserving banned, abuse-disabled, and security-blocked identities.

## Reproduction Assessment

Yes, at source level: deleteAccount creates deactivatedAt/purgedAt state and handleDeletedUserSignIn rejects that state during OAuth callback, with tests asserting the rejection. I did not run a live GitHub OAuth flow.

## Solution Assessment

No, not as a one-off binding deletion or silent restore. The maintainable solution is a policy-backed recovery or unlink flow that distinguishes self-deleted users from banned or abuse-disabled identities.
