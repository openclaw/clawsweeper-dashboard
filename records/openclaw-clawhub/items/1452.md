## Summary

Keep open. Current main still intentionally blocks deleted/deactivated GitHub OAuth reauth, there is no restore or OAuth-unlink path for this support case, and ClawHub issue cleanup policy does not auto-close issues.

## What This Changes

The issue requests restoring a deleted ClawHub account or removing the stale GitHub OAuth binding so the same GitHub identity can sign in again.

## Best Possible Solution

Define an explicit audited recovery or re-registration flow for user-initiated deletions that keeps banned and abuse-disabled accounts blocked, then update auth, operator tooling, docs, and Convex tests together.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, GitHub OAuth completion calls handleDeletedUserSignIn, and tests assert deleted or deactivated users are rejected rather than re-registered.

## Solution Assessment

No. One-off OAuth binding removal or account restoration is not the best durable fix because recovery must distinguish self-deleted accounts from banned or abuse-disabled accounts and preserve auditability.
