## Summary

Keep open. Current main still deliberately blocks GitHub OAuth sign-in for self-deleted/deactivated users, and changing that behavior crosses account deletion policy, auth binding, and abuse-control boundaries that need maintainer product/security direction.

## What This Changes

The issue requests an account restore or GitHub OAuth unlink path after a deleted ClawHub account remains bound to the same GitHub identity.

## Best Possible Solution

Define an audited recovery or re-registration flow for verified self-deleted accounts that preserves banned and abuse-disabled blocks, updates specs/docs/UI, and adds Convex coverage for recovery and rejection paths.

## Reproduction Assessment

Yes, at source level: deleteAccount creates deactivatedAt/purgedAt state and the OAuth callback rejects that state, with tests asserting the rejection. I did not run a live GitHub OAuth flow.

## Solution Assessment

No, not as a one-off binding deletion or silent restore. The maintainable solution is a policy-backed recovery path that distinguishes self-deleted accounts from banned or abuse-disabled accounts.
