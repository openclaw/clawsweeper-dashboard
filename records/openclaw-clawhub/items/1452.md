## Summary

Keep open. Current main still intentionally rejects GitHub OAuth sign-in for user-deleted accounts, and ClawHub cleanup policy only permits auto-closing pull requests that are certainly implemented on main, not unresolved issues like this account-recovery request.

## What This Changes

The issue asks ClawHub maintainers to restore a deleted account or clear the stale GitHub OAuth binding so the same GitHub identity can sign in again.

## Best Possible Solution

Define an audited account recovery or OAuth-unlink policy for self-deleted accounts, keep banned and abuse-disabled accounts blocked, then update Convex auth/admin tooling, UI copy, docs, and tests together.

## Reproduction Assessment

Yes. Source inspection provides a high-confidence path: sign in, call `deleteAccount`, then the next GitHub OAuth callback reaches `handleDeletedUserSignIn` and throws for the deactivated user.

## Solution Assessment

Unclear. The requested restore or OAuth unlink could solve the reporter's account, but the maintainable fix needs explicit product/security policy and audit behavior before changing `authAccounts` or deactivated users.
