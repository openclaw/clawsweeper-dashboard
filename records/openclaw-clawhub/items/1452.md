## Summary

Keep open. This is an issue, and the ClawHub cleanup policy only permits auto-closing pull requests that are certainly implemented on main; current main still deliberately blocks GitHub re-sign-in after user-initiated account deletion and the remedy needs account recovery/security policy.

## What This Changes

The issue requests removing a stale GitHub OAuth binding or restoring a deleted ClawHub account so the same GitHub identity can sign in again.

## Best Possible Solution

Define an audited account-recovery or re-registration flow that can unlink or rekey only eligible self-deleted accounts, keeps banned and abuse-disabled accounts blocked, and updates Convex auth/account code, UI copy, docs/specs, and regression tests.

## Reproduction Assessment

Yes. Source inspection shows deleteAccount sets deactivatedAt/purgedAt and the OAuth callback rejects that user on the next sign-in; current tests assert that blocking path, though I did not run a live GitHub OAuth flow.

## Solution Assessment

No. Directly removing an OAuth binding or restoring the account as a one-off is not the safest maintainable fix because current specs intentionally make deletion irreversible and banned identities must remain blocked.
