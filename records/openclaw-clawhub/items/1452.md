## Summary

Keep open. This is an issue, and the ClawHub cleanup policy only permits auto-closing implemented pull requests; current main still intentionally blocks GitHub OAuth reauthentication for permanently deleted accounts, so this needs maintainer account-recovery/security policy and likely operator handling.

## What This Changes

The issue requests restoring a deleted ClawHub account or clearing the stale GitHub OAuth binding so the same GitHub identity can sign in again.

## Best Possible Solution

Define a maintainer-approved audited recovery or re-registration policy for self-deleted accounts, keep banned and abuse-disabled accounts blocked, then update auth code, operator tooling, docs/specs, and regression tests together.

## Reproduction Assessment

Yes, by source inspection. deleteAccount writes deactivatedAt/purgedAt and the OAuth completion path rejects that state; I did not run a live production OAuth flow or inspect the reporter's row.

## Solution Assessment

No for the requested ad hoc binding removal or restoration from an issue comment. The maintainable path needs explicit policy, same-provider verification, audit logging, ban safeguards, and tests, or an explicit maintainer decision that deletion remains irreversible.
