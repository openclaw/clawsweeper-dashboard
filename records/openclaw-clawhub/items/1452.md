## Summary

Keep open. Current main still blocks deleted/deactivated GitHub reauth and the spec treats user deletion as irreversible, while live search shows an active cluster of related reports, so this needs maintainer product/security direction rather than cleanup closure.

## What This Changes

The issue requests a way to sign in with the same GitHub account after deleting a ClawHub account, either by removing the OAuth binding or restoring the deleted account.

## Best Possible Solution

Define an explicit audited recovery or re-registration flow for verified self-deleted accounts, or reaffirm irreversible deletion and document an operator support path that preserves banned and abuse-disabled blocks.

## Reproduction Assessment

Yes, at source level: deleteAccount creates the deactivatedAt/purgedAt state and the OAuth callback rejects that state during sign-in, with tests asserting the rejection. I did not run a live GitHub OAuth flow.

## Solution Assessment

No, not as a one-off binding deletion or silent restore. The maintainable solution is a policy-backed recovery/unlink path that distinguishes self-deleted users from banned or abuse-disabled identities.
