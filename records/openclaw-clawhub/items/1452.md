## Summary

Keep open. This is a ClawHub issue, and current main still intentionally blocks deleted/deactivated GitHub OAuth reauth; resolving it needs maintainer account-recovery and security policy handling rather than conservative cleanup closure.

## What This Changes

The issue asks for a way to sign in again with the same GitHub account after deleting a ClawHub account, either by clearing the OAuth binding or restoring the deleted account.

## Best Possible Solution

Define an explicit audited recovery or re-registration flow for self-deleted accounts that keeps banned and abuse-disabled accounts blocked, then update auth code, operator tooling, docs/specs, and Convex tests together.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, GitHub OAuth completion calls handleDeletedUserSignIn, and auth tests assert deactivated and legacy deleted users are rejected.

## Solution Assessment

No. One-off OAuth binding removal or account restoration is not the best durable repo fix because recovery must distinguish self-deleted accounts from banned or abuse-disabled accounts and preserve auditability.
