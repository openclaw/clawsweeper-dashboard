## Summary

Keep open. Current main still intentionally blocks sign-in for deleted/deactivated GitHub-linked accounts, and for this repository the cleanup policy keeps issues open unless maintainers resolve the underlying product/security decision.

## What This Changes

The issue asks for a deleted ClawHub account to be restored or its GitHub OAuth binding removed so the same GitHub account can sign in again.

## Best Possible Solution

Define an explicit audited recovery or re-registration policy for self-deleted accounts, keep banned and abuse-disabled accounts blocked, then update auth code, operator tooling, docs/specs, and tests together if maintainers choose to allow recovery.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: `deleteAccount` deactivates and purges the user, GitHub OAuth completion calls `handleDeletedUserSignIn`, and tests assert deleted/deactivated users are rejected rather than re-registered.

## Solution Assessment

No, one-off OAuth binding removal or account restoration is not the best durable solution because recovery must distinguish self-deleted accounts from banned or abuse-disabled accounts and preserve auditability.
