## Summary

Keep open. Current main still deliberately blocks GitHub OAuth sign-in for deleted/deactivated accounts, the repo policy says ClawHub issues should not be auto-closed by this cleanup pass, and the remaining work is an account-recovery/security policy decision rather than a narrow bug fix.

## What This Changes

The issue asks for a deleted ClawHub account to be restored or its GitHub OAuth binding removed so the same GitHub account can sign in again.

## Best Possible Solution

Define an audited account-recovery or re-registration policy that distinguishes user-initiated deletion from bans/abuse disables, then update auth, operator tooling, docs/specs, and tests to match that policy.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount writes deactivatedAt/purgedAt, OAuth calls handleDeletedUserSignIn, and tests assert deleted/deactivated users are rejected; I did not run a live OAuth flow.

## Solution Assessment

No. One-off OAuth binding removal or account restoration is not the best durable fix because recovery must be audited and separated from banned, disabled, or abuse-related account states.
