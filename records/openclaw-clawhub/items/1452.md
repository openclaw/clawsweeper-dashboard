## Summary

Keep open. Current main still deliberately blocks GitHub OAuth reauthentication for deleted/deactivated accounts, the spec defines user deletion as irreversible, and related recovery/unlink reports remain open; this needs maintainer security/product and operator handling rather than conservative cleanup closure.

## What This Changes

Allow a user who deleted their ClawHub account to sign in again with the same GitHub identity by restoring the account or clearing the OAuth binding.

## Best Possible Solution

Define a maintainer-approved, audited recovery or re-registration policy for self-deleted accounts, keep banned and abuse-disabled accounts blocked, then update auth code, operator tooling, docs/specs, and regression tests together.

## Reproduction Assessment

Yes, by source inspection. deleteAccount writes deactivatedAt/purgedAt and the GitHub OAuth completion path calls handleDeletedUserSignIn, which rejects that state; I did not run a live production OAuth flow or inspect the reporter's row.

## Solution Assessment

No for the requested ad hoc binding removal or restoration from an issue comment. The maintainable path needs explicit policy, same-provider verification, audit logging, ban safeguards, and tests, or an explicit decision that deletion remains irreversible.
