## Summary

Keep open. This is an open ClawHub issue, and current main still intentionally blocks deleted/deactivated GitHub OAuth reauth; the remaining work is an account-recovery/security policy decision, not conservative cleanup closure.

## What This Changes

The requester wants the old GitHub OAuth binding removed or the deleted ClawHub account restored so the same GitHub account can sign in again.

## Best Possible Solution

Define an explicit audited recovery or re-registration policy for self-deleted accounts, keep banned and abuse-disabled accounts blocked, then update auth code, operator tooling, docs/specs, and tests together if maintainers choose to allow recovery.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `deleteAccount` marks the user deactivated/purged, GitHub OAuth completion calls `handleDeletedUserSignIn`, and tests assert deleted/deactivated users are rejected rather than re-registered.

## Solution Assessment

No. One-off OAuth binding removal or account restoration is not the best durable solution because recovery must distinguish self-deleted accounts from banned or abuse-disabled accounts and preserve auditability.
