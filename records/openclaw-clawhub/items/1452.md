## Summary

Keep open. ClawHub cleanup policy does not auto-close issues, and current main still intentionally blocks deleted/deactivated GitHub OAuth reauth, so this needs maintainer account-recovery and security policy handling rather than cleanup closure.

## What This Changes

The issue asks for GitHub OAuth account recovery or binding removal after a user deletes their ClawHub account and can no longer sign in.

## Best Possible Solution

Define an explicit audited recovery or re-registration flow for user-initiated deletions that keeps banned and abuse-disabled accounts blocked, then update auth code, operator tooling, docs/specs, and Convex tests together.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount marks the user deactivated/purged, GitHub OAuth completion calls handleDeletedUserSignIn, and tests assert deleted/deactivated users are rejected rather than re-registered.

## Solution Assessment

No. One-off OAuth binding removal or account restoration is not the best durable solution because recovery must distinguish self-deleted accounts from banned or abuse-disabled accounts and preserve auditability.
