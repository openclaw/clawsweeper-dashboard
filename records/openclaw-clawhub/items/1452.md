Keep open. This is an issue, and ClawHub cleanup policy does not auto-close open issues; current main also still intentionally blocks deleted/deactivated GitHub reauth, so the remaining work is a maintainer product/security decision rather than stale cleanup.

## What This Changes

The issue requests restoring or unlinking a deleted account's GitHub OAuth binding so the same GitHub account can sign in again.

## Best Possible Solution

Define an audited account recovery or re-registration policy for self-deleted accounts, keep banned and abuse-disabled accounts blocked, then update auth/account code, specs/docs, and Convex tests together.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: after GitHub OAuth, `handleDeletedUserSignIn` rejects deactivated accounts and migrates legacy deleted accounts before rejecting them, and tests assert that behavior.

## Solution Assessment

No for an automated source-only fix. The requested restore/unlink can be valid, but the best implementation needs a maintainer-approved, audited recovery policy rather than deleting OAuth bindings or reactivating rows ad hoc.
