## Summary

Keep open. Current main still intentionally blocks GitHub OAuth sign-in for deactivated/deleted ClawHub accounts, the public/spec docs describe deletion as non-restorable, and the repository cleanup policy keeps ClawHub issues open unless a maintainer resolves the account-recovery policy.

## What This Changes

The issue requests removal of the stale GitHub OAuth binding or restoration of a deleted ClawHub account so the same GitHub account can sign in again.

## Best Possible Solution

Define an audited account recovery or re-registration flow that distinguishes self-deleted accounts from banned or abuse-disabled accounts, then update Convex auth/users code, operator guidance, docs, and regression tests to match that policy.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, then the GitHub OAuth callback calls handleDeletedUserSignIn and throws the deleted-account message; I did not run a live OAuth flow.

## Solution Assessment

No. One-off deletion of authAccounts bindings or ad hoc account restoration is not the best durable solution because current docs/specs make deletion irreversible and recovery must remain separate from banned-account handling.
