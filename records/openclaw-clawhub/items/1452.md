## Summary

Keep open. This is an open ClawHub issue, current main still intentionally blocks deleted/deactivated GitHub reauth and tracks account recovery after owner deletion as unresolved product/security policy, so conservative cleanup should not close it.

## What This Changes

The issue asks maintainers to remove a stale GitHub OAuth binding or restore a deleted account so the same GitHub identity can sign in again.

## Best Possible Solution

Define a maintainer-approved audited recovery or unlink policy, then update OAuth, account deletion, admin/operator tooling, user-facing errors, docs/specs, and regression tests together.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, GitHub OAuth completion calls handleDeletedUserSignIn, and tests assert deleted or deactivated users are rejected rather than re-registered.

## Solution Assessment

No. One-off OAuth binding removal or account restoration is not the best durable fix because recovery must distinguish self-deleted accounts from banned or abuse-disabled accounts and preserve auditability.
