## Summary

Keep open. Current main still intentionally blocks GitHub re-sign-in for deactivated/deleted accounts, and this issue is part of an unresolved auth/account-recovery cluster that needs maintainer product and security direction rather than cleanup closure.

## What This Changes

The issue requests removing a stale GitHub OAuth binding or restoring a deleted ClawHub account so the same GitHub identity can sign in again.

## Best Possible Solution

Define and implement an audited recovery or re-registration flow for user-initiated deletions, keep banned and abuse-disabled accounts blocked, and update UI, docs/specs, and Convex tests to match the chosen policy.

## Reproduction Assessment

Yes, from source inspection: deleting an account sets `deactivatedAt`, and the OAuth callback rejects later sign-in for that linked user instead of unlinking or re-registering. I did not run a live OAuth flow in this read-only review.

## Solution Assessment

No, not as a direct binding delete or blind restore. The maintainable fix is an explicit audited recovery/re-registration policy with safeguards for banned and abuse-disabled accounts.
