## Repair Work Prompt

Fix the skill detail mobile overflow reported for iOS/Safari. Scope the change to the skill detail surface: SkillHeader, SkillInstallCard, SkillInstallSurface/SkillCommandLineCard, SkillDetailTabs, and related CSS. Harden long summaries, .stat runtime/dependency/install/link rows, bundle metadata, install commands, tabs, README markdown, code blocks, and tables so they cannot make document.documentElement.scrollWidth exceed clientWidth at mobile widths. Prefer min-width: 0, flex-wrap, overflow-wrap:anywhere, and targeted break/scroll behavior; do not rely on clipping content with overflow-x:hidden as the only fix. Add or update deterministic mobile overflow coverage that does not skip solely because /api/v1/skills/gifgrep is unavailable, using a fixture with long unbroken env/config/bin/link/command/readme values. Do not change public API behavior, unrelated layout surfaces, dependencies, or source skill data. Add a CHANGELOG.md Unreleased Fixes entry for the user-facing UI fix.
