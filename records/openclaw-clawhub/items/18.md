## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits automated implemented-on-main closure for pull requests. Current main also does not conclusively prove the iOS/Safari skill-detail mobile breakage is fixed: the page has responsive safeguards, but long runtime/install metadata still flows through weakly wrapped rows and the mobile overflow test can skip when its live fixture is missing.

## What This Changes

Make the skill detail page render correctly on mobile iOS/Safari without broken layout or horizontal overflow.

## Best Possible Solution

Harden the skill detail mobile layout so long summaries, stat/runtime metadata rows, install commands, links, tabs, README content, code blocks, and tables cannot increase document scrollWidth on iOS/Safari or mobile Chrome, then cover that behavior with deterministic mobile overflow tests that do not depend only on a live production API fixture.
