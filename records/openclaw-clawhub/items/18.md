## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still lacks high-confidence proof that the mobile Safari skill-detail overflow is fixed: the active detail path remains in place, long runtime/install/detail values still pass through weakly wrapped rows, and the only direct mobile overflow test can skip on live fixture availability.

## What This Changes

The issue reports broken iOS/Safari mobile rendering on the skill detail page, with horizontal clipping shown in an attachment.

## Best Possible Solution

Harden the skill detail layout so long summaries, runtime metadata rows, install commands, links, tabs, README content, code blocks, and tables cannot increase document scrollWidth on mobile Safari or mobile Chrome, then cover it with deterministic overflow tests that do not depend solely on live API fixtures.

## Reproduction Assessment

## Solution Assessment

Yes. A narrow CSS/layout hardening PR with deterministic mobile overflow coverage is the most maintainable path; avoid broad overflow hiding that could clip content without proving responsive fit.
