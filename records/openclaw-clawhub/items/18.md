Keep open. This is a ClawHub issue, and the cleanup policy keeps issues open; the current skill-detail path also still has plausible mobile overflow sources for the reported `withings-health` case and the related open content-overflow report.

## What This Changes

The issue reports broken iOS/Safari mobile rendering on the skill detail page, with the screenshot showing the `withings-health` detail view overflowing horizontally.

## Best Possible Solution

Harden the skill-detail mobile layout so long summaries, runtime metadata, install commands, links, tabs, README code, tables, and images cannot create document-level horizontal overflow, then cover `withings-health` and the related overflow cases with deterministic regression tests.

## Reproduction Assessment

No high-confidence current-main browser reproduction was established. The screenshot and live skill data identify the `withings-health` content shape, and source inspection maps it to long summary/runtime rows, but this pass did not run iOS/Safari or Playwright.

## Solution Assessment

Yes. The best fix is a narrow CSS/layout hardening change plus deterministic overflow coverage; a broad outer `overflow-x: hidden` alone would be weaker because it could clip content without proving responsive behavior.
