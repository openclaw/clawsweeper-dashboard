  - body: The PR replaces the current `VITE_FEATURE_SOULS` read with a hard-coded `defaultValue: false`; with no LaunchDarkly client-side ID, `FeatureFlagProvider` returns that fallback and existing deployments that enabled Souls locally lose the header/footer/home links.   - body: This identifies every signed-in browser session to LaunchDarkly with the Convex user id and role, but the PR only documents flag setup and does not define that third-party data boundary. Keep contexts anonymous/minimal or document explicit maintainer approval for role/user targeting.
