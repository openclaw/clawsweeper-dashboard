- **[P1] Preserve the existing Souls env flag path:** `src/lib/features.ts:9`
  - body: `FEATURE_SOULS` currently reads `VITE_FEATURE_SOULS`; this replacement hard-codes the no-LaunchDarkly fallback to `false`, so an existing deployment with `VITE_FEATURE_SOULS=true` would silently lose Souls links.   - confidence: 0.9
- **[P2] Minimize the LaunchDarkly user context:** `src/lib/featureFlagContext.tsx:89-93`
  - body: This identifies every signed-in browser session to LaunchDarkly with the Convex user id and role, but the PR only needs a Souls rollout and the docs do not define that data-sharing boundary. Keep contexts anonymous or send documented minimal targeting data unless maintainers explicitly approve role/user targeting.
  - confidence: 0.78

Add a `.catch(() => {})` or equivalent handling for the same reason other async side effects in this codebase swallow non-critical failures.
  - confidence: 0.72
