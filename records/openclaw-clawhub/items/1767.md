## What This Changes

Request a local plugin scanner CLI/package that validates plugin manifests, SKILL.md metadata, and static code findings before publish with severity, exit-code, and JSON output.

## Best Possible Solution

Define a maintainer-approved local scanner that reuses shared package/schema validation and deterministic static moderation logic, exposes stable human and JSON findings, and documents how local results relate to hosted scans.

Unclear. A local scanner is plausible, but the best maintainable solution depends on maintainer choices about CLI versus package placement, hosted scanner parity, severity semantics, and false-positive policy.
