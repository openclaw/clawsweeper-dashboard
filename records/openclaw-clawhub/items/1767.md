## What This Changes

Request a local plugin scanner CLI/package that validates manifest, SKILL.md, code-scan, severity, exit-code, and JSON-output findings before publishing.

## Best Possible Solution

Define a maintainer-approved local scanner that reuses shared manifest/schema validation and deterministic static moderation logic, emits stable human and JSON findings with severity codes, and documents how local results relate to hosted publish-time scans.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The verification path is source and docs inspection on current main, which shows dry-run, pack, verify, and hosted scan paths but no local pre-publish scanner command.

## Solution Assessment

Unclear. A local scanner is a plausible direction, but the maintainable shape depends on scanner scope, severity semantics, hosted parity, and whether the command belongs in ClawHub, OpenClaw, or a standalone package.
