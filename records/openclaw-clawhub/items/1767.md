## Summary

Keep open. Repository policy for this cleanup pass keeps ClawHub issues open, and current main still has package dry-run previews, hosted publish-time/static scan plumbing, package security pages, and rescan surfaces, but no local pre-publish plugin scanner CLI covering the requested manifest, SKILL.md, code, severity, exit-code, and JSON-output checks.

## What This Changes

Request a local plugin scanner package or CLI subcommand for pre-publish validation of plugin manifests, SKILL.md metadata, code scan findings, severities, exit codes, and JSON output.

## Best Possible Solution

Define a maintainer-approved local scanner that reuses shared manifest/schema validation and deterministic static moderation logic, emits stable human and JSON findings with severity codes, and documents how local results relate to hosted publish-time scans.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The verification path is source and docs inspection on current main, which shows hosted scans and dry-run previews but no local pre-publish scanner command.

## Solution Assessment

Unclear as an implementation path. A local scanner is plausible, but the maintainable solution needs maintainer decisions on scanner scope, hosted parity, false-positive policy, output contract, and CLI/package placement.
