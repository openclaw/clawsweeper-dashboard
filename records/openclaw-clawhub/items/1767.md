## Summary

Keep open. Current main still has package publish dry-run, hosted publish-time static scanning, package security rescan support, and narrow local package metadata validation, but it does not provide a local plugin lint/scanner command that reports manifest, SKILL.md, and code/security findings before publishing.

## What This Changes

Add a local plugin scanner or CLI preflight command so plugin authors can validate manifest metadata, SKILL.md declarations, code scan findings, severity output, JSON output, and publish readiness before uploading.

## Best Possible Solution

Define a maintainer-approved local ClawHub plugin scanner that reuses shared manifest/schema validation and deterministic static moderation logic, adds missing plugin/SKILL metadata checks, supports stable human and JSON output with severity/exit-code semantics, and documents how local results relate to hosted scanning.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request for a new local CLI/preflight scanner. Source inspection gives high confidence that current main still lacks the requested local scanner surface.

## Solution Assessment

Unclear. The direction is useful, but the best implementation needs maintainer decisions about package-vs-standalone CLI placement, hosted parity, severity levels, exit codes, and JSON schema stability.
