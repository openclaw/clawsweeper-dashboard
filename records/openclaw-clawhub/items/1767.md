## Summary

Keep open. Current main has a repo-local trusted-developer ClawScan dry-run script, but not the requested public plugin-author lint package, CLI subcommand, or publish dry-run integration with stable pre-publish findings.

## What This Changes

Request a local plugin scanner/lint package or CLI command for pre-publish validation of manifests, SKILL.md metadata, and code-risk findings.

## Best Possible Solution

Promote or replace the internal dry-run script with a maintainer-approved public preflight surface that shares deterministic scan logic, defines JSON/human output and exit behavior, covers manifest/SKILL metadata checks, and documents how results relate to hosted ClawScan.

## Solution Assessment

Unclear. The direction is reasonable, but the best implementation needs maintainer decisions on package-versus-CLI placement, hosted parity, severity mapping, exit codes, and which findings should block publish versus guide review.
