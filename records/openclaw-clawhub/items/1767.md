## Summary

Keep open. ClawHub issue policy keeps issues open, and the feature gap remains: current main has publish dry-run, hosted publish-time scanning, and a repo-local trusted-developer ClawScan script, but no supported public local plugin lint/pre-publish scanner CLI or package.

## What This Changes

Request a supported local plugin scanner/lint command or package that validates manifest, SKILL.md, and code findings before publishing, with JSON output, severities, and nonzero exits.

## Best Possible Solution

Define a maintainer-approved local scanner contract that reuses shared deterministic manifest/schema and moderation logic where possible, documents hosted-scan parity and differences, and exposes stable human and JSON output through the chosen public CLI/package surface.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main lacks a supported public local plugin lint/pre-publish scanner command, even though it has a repo-local developer script.

## Solution Assessment

Unclear as an implementation plan. The requested direction is reasonable, but maintainers still need to choose CLI placement, hosted parity, severity semantics, JSON/exit behavior, and documentation before a narrow implementation is safe.
