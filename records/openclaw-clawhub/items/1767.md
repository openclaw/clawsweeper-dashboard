## Summary

Keep open. This is a ClawHub issue, and repository policy for this cleanup pass keeps issues open; substantively, current main still has hosted publish-time/static scan and dry-run publish plumbing but no local pre-publish plugin scanner CLI that covers the requested manifest, SKILL.md, code, severity, exit-code, and JSON-output checks.

## What This Changes

The issue requests a local plugin scanner or CLI subcommand for pre-publish validation of plugin manifests, SKILL.md metadata, static code findings, severity levels, exit codes, and JSON output.

## Best Possible Solution

Define a maintainer-approved local scanner that reuses shared manifest/schema validation and deterministic static moderation logic, emits stable human and JSON findings with severity codes, and documents how local results relate to hosted publish-time scans.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The verification path is source and docs inspection on current main, which shows hosted scans and dry-run previews but no local pre-publish scanner command.

## Solution Assessment

Unclear. A local scanner is a plausible direction, but the best maintainable implementation depends on maintainer decisions about scanner scope, output contract, hosted parity, and whether it belongs in the CLI or a standalone package.
