## Summary

Keep open. Current main still has package publish dry-run, narrow local package validation, hosted publish-time static scanning, and owner rescan flows, but no local pre-publish plugin scanner/linter; ClawHub cleanup policy also keeps issues open.

## What This Changes

Add a local pre-publish plugin scanner/linter CLI or package that validates plugin manifests, SKILL.md metadata, static security findings, and JSON/severity output before publishing.

## Best Possible Solution

Define a maintainer-approved local ClawHub scanner that reuses shared manifest/schema validation and deterministic static moderation logic, adds missing plugin/SKILL metadata checks, supports stable human and JSON severity output, and documents how local findings relate to hosted publish-time scans.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main lacks a local pre-publish plugin scanner command and that dry-run remains a metadata/payload preview.

## Solution Assessment

Unclear. A local scanner is a plausible direction, but the best implementation depends on maintainer decisions about command placement, severity semantics, hosted parity, and scanner policy.
