## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still lacks the requested local pre-publish plugin/package lint scanner; `package publish --dry-run` only previews the publish plan while hosted scanning remains a backend publish-time path.

## What This Changes

The issue requests a local pre-publish scanner or CLI lint command for plugin authors to validate manifests, SKILL.md metadata, static security findings, JSON output, and severity/exit-code behavior before publishing.

## Best Possible Solution

Define a maintainer-approved local ClawHub scanner that reuses shared manifest/schema validation and deterministic static moderation logic where safe, adds plugin/SKILL metadata checks, emits stable human and JSON results, and documents how local results relate to hosted publish-time scans.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main lacks a local plugin/package lint scanner and that dry-run remains a publish-plan preview.

## Solution Assessment

Unclear. A local scanner is a plausible direction, but the best implementation depends on maintainer choices about CLI/package placement, severity semantics, hosted scan parity, and which security findings should be disclosed or block locally.
