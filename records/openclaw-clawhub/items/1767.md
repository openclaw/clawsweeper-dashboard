## Summary

Keep open. ClawHub policy keeps issues open, and the requested local pre-publish plugin scanner is still absent from current main; the existing dry-run, hosted ClawScan/static scanning, and rescan paths do not provide the local lint workflow described here.

## What This Changes

Request a local CLI/package command that validates plugin manifests, SKILL.md metadata, and static code-scan findings before publishing.

## Best Possible Solution

Define and document a maintainer-approved local scanner that reuses shared manifest/schema validation and deterministic static moderation logic, supports stable human and JSON output, and clearly states how local results relate to hosted publish-time scanning.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The current-main verification path is a CLI/docs/source search plus dry-run inspection, which shows no local plugin lint/scanner command.

## Solution Assessment

Unclear until maintainer scope is set. A local command is a plausible direction, but the maintainable solution depends on approved severity codes, package placement, and scanner parity boundaries.
