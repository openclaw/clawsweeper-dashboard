## Summary

Keep open. Repository cleanup policy keeps ClawHub issues open, and the current feature gap is still real: main has a repo-local ClawScan dry-run script plus package publish dry-run, but not a public plugin-author lint/scanner command with stable pre-publish findings.

## What This Changes

Request a public local plugin scanner or CLI pre-publish lint command that validates plugin manifests, SKILL.md metadata, code-risk heuristics, JSON output, exit codes, and severity levels before publishing.

## Best Possible Solution

Define a maintainer-approved public scanner surface that reuses shared manifest/schema validation and deterministic moderation logic, adds the missing plugin/SKILL metadata checks, emits stable human and JSON output with exit codes, and documents how local results relate to hosted ClawScan.
