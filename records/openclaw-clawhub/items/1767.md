## Label Justifications

- `P2`: This is a concrete publisher workflow and CI improvement, but it is not an urgent production regression.
- `impact:security`: The requested workflow exposes credential, metadata, and suspicious-code findings before a plugin is published.

## Summary

The request remains necessary because current `main` and `v0.20.2` intentionally reject local-folder scans and only allow scans of already-published artifacts; the approved scan design also explicitly deferred unpublished plugin packages. The existing `clawhub scan` work is useful progress, but it does not remove the plugin publish-and-rescan loop described here.

## What This Changes

Add a pre-publish CLI workflow that validates local plugin manifests, skill metadata, and security findings with structured output and CI-friendly exit codes.

## Best Possible Solution

Extend the canonical authenticated scan-job API and `clawhub scan` CLI to accept ephemeral code-plugin bundles, reuse the same hosted validation and scanner rules, and return stable human/JSON results and meaningful exit codes without creating public registry state.

## Reproduction Assessment

Yes, at source level. On current `main`, `clawhub scan <path>` is explicitly rejected, and the approved design excludes unpublished plugin packages, so the reported pre-publish plugin gap is directly verifiable.

## Solution Assessment

Yes in direction, but the maintainable implementation is to extend the canonical ClawHub scan engine rather than create a separate plugin-only scanner that can diverge from hosted enforcement.

## Risks / Open Questions

- A standalone plugin linter could drift from ClawHub’s hosted scanner rules, severities, and blocking policy.
- Accepting arbitrary unpublished plugin bundles requires explicit authentication, retention, size/rate limits, abuse controls, and a guarantee that uploaded code is analyzed but never executed.
- The issue mixes deterministic schema validation with heuristic security scanning; maintainers should decide which checks can run fully locally and which require the hosted scan service.
