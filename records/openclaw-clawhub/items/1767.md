## Summary

Keep open. ClawHub issue cleanup policy does not auto-close issues, and the gap is still real: current main has a repo-local ClawScan dry-run script and package publish dry-run, but no supported public plugin-author lint/scanner command with stable findings, exit codes, and pre-publish scanner semantics.

## What This Changes

Request a local plugin-author scanner or lint CLI that validates manifests, SKILL.md metadata, code security signals, JSON output, and exit codes before publishing.

## Best Possible Solution

Define a maintainer-approved public plugin preflight scanner that reuses shared schema/static moderation logic, adds the missing manifest and SKILL metadata checks, emits stable human and JSON output with exit codes, and documents how local results relate to hosted ClawScan.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main still lacks the requested public plugin-author lint/scanner command.

## Solution Assessment

Unclear. The requested direction is reasonable, but the best implementation needs maintainer decisions on package versus CLI placement, hosted parity, severity and exit semantics, and which checks can be deterministic before publish.
