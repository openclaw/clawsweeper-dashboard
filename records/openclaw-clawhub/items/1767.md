## Summary

Keep open. This is a concrete feature request, and current main still has publish dry-run, hosted static scanning, and owner rescan flows but no local pre-publish plugin scanner with manifest, SKILL.md, code, severity, exit-code, and JSON-output checks; ClawHub cleanup policy also keeps issues open.

## What This Changes

Request a local pre-publish plugin scanner or lint command that validates plugin manifests, SKILL metadata, and code before publishing with structured severity output.

## Best Possible Solution

Define a maintainer-approved local scanner that reuses shared schema validation and deterministic static moderation logic, adds missing plugin/SKILL metadata checks, emits stable human and JSON findings, and documents how local results relate to hosted publish-time scans.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The feature gap is high-confidence from source and docs inspection: current main has hosted scans and dry-run previews but no local pre-publish plugin scanner command.

## Solution Assessment

Unclear. A local scanner is plausible, but the best maintainable design depends on maintainer decisions about command placement, hosted parity, severity semantics, and scanner policy.
