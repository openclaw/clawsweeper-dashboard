## Summary

Keep open. ClawHub policy keeps issues open, and the requested local pre-publish plugin scanner is still a real gap on current main: package publish has a dry-run payload preview, hosted publish-time static scanning, and rescan requests, but no local lint/scanner command with manifest, SKILL.md, code findings, severity-coded output, JSON mode, and nonzero exits before publishing.

## What This Changes

The issue requests a local plugin scanner CLI or publish dry-run mode that validates OpenClaw plugin manifests, SKILL.md metadata, code/security heuristics, JSON output, severity levels, and exit codes before publishing.

## Best Possible Solution

Define a maintainer-approved local ClawHub scanner that reuses shared manifest/schema validation and deterministic static moderation logic, adds missing plugin/SKILL metadata checks, supports stable human and JSON output with severity-coded findings, and documents how local results relate to hosted publish-time scanning.
