## Summary

Keep open. This is a ClawHub feature request, and current main still lacks a local pre-publish plugin scanner CLI; the existing dry-run publish and hosted scan paths do not provide the requested local findings, severity, exit-code, and JSON-output behavior.

## What This Changes

Request a local plugin scanner or CLI subcommand for pre-publish validation of plugin manifests, SKILL.md metadata, code security heuristics, and CI-friendly output.

## Best Possible Solution

Define a maintainer-approved local scanner that reuses shared publish-plan validation and deterministic static moderation logic, emits stable human/JSON findings with severity and exit codes, and documents how local results relate to hosted scans.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The verification path is source and docs inspection on current main, which shows hosted scans and dry-run previews but no local pre-publish scanner command.

## Solution Assessment

Unclear. A local scanner is a plausible direction, but the best maintainable solution depends on maintainer decisions about scanner scope, hosted parity, output contract, severity policy, and whether it belongs in the existing CLI or a standalone package.
