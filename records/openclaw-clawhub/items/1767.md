## Summary

Keep open. ClawHub cleanup policy keeps issues open, and the requested local pre-publish plugin scanner is still a real feature gap on current main: the CLI has package publish dry-run, local skill sync, rescan requests, and hosted/static scan persistence, but no local plugin lint/scanner command that emits manifest, SKILL.md, and source findings before publish.

## What This Changes

The issue requests a local plugin scanner or CLI lint command for validating plugin manifests, SKILL.md metadata, and code heuristics before publishing.

## Best Possible Solution

Define a maintainer-approved local ClawHub scanner that reuses shared manifest/schema validation and deterministic static moderation logic, adds missing plugin/SKILL metadata checks, supports human and JSON output with stable severity codes, and documents how local results relate to hosted publish-time scanning.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is to inspect the current CLI command surface, dry-run publish path, docs, and hosted static scan path for the absence of a local plugin scanner.

## Solution Assessment

Unclear until maintainers choose the scanner contract. Reusing shared validation and deterministic moderation code looks like the narrowest maintainable direction, but the local security-finding surface needs explicit scope and severity decisions first.
