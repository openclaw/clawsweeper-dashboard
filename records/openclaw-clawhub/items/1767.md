## Summary

Keep open. This is an open ClawHub feature request, and current main still has package dry-run, narrow local compatibility checks, hosted publish-time static scanning, and owner rescan flows, but no local pre-publish plugin/package scanner that reports manifest, SKILL.md, and code findings before publishing.

## What This Changes

Add a local plugin scanner or CLI lint/dry-run flow that validates plugin metadata, SKILL.md declarations, code heuristics, JSON output, and severity levels before publish.

## Best Possible Solution

Define a maintainer-approved local ClawHub scanner that reuses shared manifest validation and deterministic static moderation where safe, adds missing plugin/SKILL metadata checks, emits stable human and JSON severities, and documents how local results relate to hosted publish-time scans.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main lacks a local plugin/package lint scanner and that dry-run remains a metadata/payload preview.

## Solution Assessment

Unclear. A local scanner is a plausible direction, but the maintainable implementation depends on maintainer decisions about CLI/package placement, hosted parity, severity semantics, and security disclosure policy.
