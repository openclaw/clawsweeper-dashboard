## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has only a repo-local trusted-developer ClawScan dry-run plus publish-side scanning, not the requested public pre-publish plugin lint/scanner surface.

## What This Changes

Request a public local plugin scanner package, CLI subcommand, or publish dry-run that validates plugin manifests, SKILL.md metadata, code-scan findings, JSON output, severities, and exit codes before publishing.

## Best Possible Solution

Define a maintainer-approved public scanner surface that reuses deterministic hosted scan logic where possible, adds manifest/SKILL metadata checks, supports stable human and JSON output, and documents how local results relate to hosted ClawScan.
