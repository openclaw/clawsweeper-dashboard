## Summary

Keep open. Current main has useful groundwork in a repo-local `clawscan:local` script, but the requested public pre-publish plugin scanner CLI/package with stable lint findings, JSON output, severity/exit semantics, and documented hosted-scan parity is still not implemented; ClawHub cleanup policy also keeps issues open.

## What This Changes

Request a public local plugin scanner CLI/package or dry-run publish mode for pre-publish manifest, SKILL.md, and code validation.

## Best Possible Solution

Define a maintainer-approved public scanner that reuses shared manifest/schema validation and deterministic static scan logic, adds missing plugin/SKILL metadata checks, returns stable human and JSON output with documented severity and exit-code semantics, and explains how local results relate to hosted ClawScan.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection on current main gives high confidence that the public CLI still lacks a package/plugin lint scanner, while only dry-run preview, hosted scanning, and a repo-local script exist.

## Solution Assessment

Unclear. A local scanner is a plausible solution, but the current internal script is intentionally narrower, and the durable design needs maintainer decisions on CLI/package placement, severity and exit semantics, hosted parity, and security disclosure policy.
