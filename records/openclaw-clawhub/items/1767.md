## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has publish dry-run, narrow package validation, hosted publish-time static scanning, and owner rescans, but no local pre-publish plugin scanner/lint command for manifest, SKILL.md, and code findings.

## What This Changes

The issue requests a local ClawHub/OpenClaw plugin scanner or lint command that reports manifest, SKILL.md, and code-scan findings before publishing.

## Best Possible Solution

Define a maintainer-approved local ClawHub scanner that reuses shared manifest/schema validation and deterministic static moderation logic, adds missing plugin/SKILL metadata checks, supports stable human and JSON severity output, and documents how local results relate to hosted publish-time scans.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main lacks a local pre-publish plugin scanner command and that dry-run remains a metadata/payload preview.

## Solution Assessment

Unclear. A local scanner is plausible, but the best implementation depends on maintainer decisions about CLI/package placement, severity semantics, hosted parity, and scanner disclosure policy.
