## Summary

Current main now provides the requested pre-publication local workflow through `clawhub package validate <path>`, which runs the bundled Plugin Inspector against a local plugin, produces human and machine-readable findings, and fails on hard errors; the implementation landed after the latest release, so this is fixed on main but not yet shipped in `v0.20.0`.

## What This Changes

Requests a local CLI or package that validates plugin manifests, SKILL.md metadata, and security findings before publication.

## Best Possible Solution

Keep `clawhub package validate` as the dedicated author-facing pre-publish command, while continuing to distinguish its local Plugin Inspector findings from hosted scanners that require registry-side infrastructure.

## Reproduction Assessment

Not applicable: this is a feature request, and current-main source, tests, and documentation directly confirm the requested local validation workflow now exists.

## Solution Assessment

Yes: a dedicated `clawhub package validate` command reuses the maintained Plugin Inspector and is clearer than requiring publication or overloading publish dry-run.
