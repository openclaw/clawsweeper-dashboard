## Summary

Keep open. ClawHub policy keeps issues open, and current main still lacks a local plugin lint/scanner; existing package dry-run only previews publish metadata, while static security scanning runs after upload.

## What This Changes

Add a local pre-publish plugin scanner or CLI command that validates plugin manifests, SKILL.md metadata, code-risk heuristics, JSON output, severity levels, and exit behavior before publishing.

## Best Possible Solution

Define a maintainer-approved local scanner that reuses shared manifest validation and deterministic static moderation logic, adds missing plugin/SKILL checks, emits stable human and JSON severities, and documents how local results relate to hosted scans.

Unclear. The requested direction is reasonable, but maintainers need to decide package-versus-standalone CLI placement, scanner parity, and which findings warn, block, or route to review.
