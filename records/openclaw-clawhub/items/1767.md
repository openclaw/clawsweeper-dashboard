## Summary

Keep open. Current main still has package dry-run, published-package readiness/rescan surfaces, and hosted static scanning, but no local pre-publish plugin scanner that reports manifest, SKILL.md, and code findings before upload; ClawHub cleanup policy also keeps issues open.

## What This Changes

Request a local plugin scanner package or CLI command that validates plugin manifests, SKILL.md metadata, and suspicious code patterns before publishing.

## Best Possible Solution

Define a maintainer-approved local scanner that reuses shared schema validation and deterministic static moderation logic, adds missing plugin/SKILL metadata checks, emits stable human and JSON findings, and documents how local results relate to hosted scans.
