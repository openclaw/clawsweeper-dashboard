## Summary

Keep open. Repository policy keeps ClawHub issues open, and current main still has package dry-run, narrow local preflight validation, hosted/static moderation, and rescan paths, but no local pre-publish plugin scanner with the requested manifest, SKILL.md, code-scan, severity, exit-code, and JSON-output contract.

## What This Changes

Request a local plugin scanner CLI/package for pre-publish manifest, SKILL.md, code, severity, exit-code, and JSON validation.

## Best Possible Solution

Define a maintainer-approved local scanner that reuses shared manifest/schema validation and deterministic static moderation logic where possible, emits stable human and JSON findings with severity codes, and documents how local results relate to hosted publish-time scans.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The verification path is source/docs inspection on current main, which shows dry-run previews and hosted scans but no local pre-publish scanner command.

## Solution Assessment

Unclear. A local scanner is a plausible direction, but the best maintainable implementation depends on maintainer decisions about CLI/package placement, hosted parity, severity semantics, and acceptable false positives.
