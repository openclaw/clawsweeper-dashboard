## What This Changes

The issue requests a local plugin scanner or CLI lint/dry-run command that validates plugin manifests, SKILL.md metadata, and code findings before publishing.

## Best Possible Solution

Define the public scanner boundary around `clawhub package validate` or a new scan/lint command, then expose deterministic local checks and documented hosted-parity limits with stable human and JSON output.

## Reproduction Assessment

Yes. Source inspection shows a public local Plugin Inspector validation command exists, while `clawhub scan <path>` still rejects local paths and the fuller local ClawScan dry-run is only an internal script.

## Solution Assessment

No, not completely. `clawhub package validate` is the right direction for deterministic author-facing validation, but the requested hosted-scan parity and credential/static findings still need a maintainer-approved public boundary.
