## Summary

Keep open. This is a concrete feature request and current main still lacks a local plugin/package pre-publish scanner; ClawHub review policy also keeps issues open unless there is a terminal close reason, and the related onboarding issue overlaps without superseding this narrower request.

## What This Changes

The issue requests a local plugin/package scanner CLI that validates manifest, SKILL.md, and static security findings before publishing.

## Best Possible Solution

Define a maintainer-approved local scanner that reuses shared manifest/schema validation and deterministic static moderation logic, adds missing plugin/SKILL metadata checks, emits stable human and JSON findings, and documents how local results relate to hosted ClawScan.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence current main has publish dry-run and hosted static scanning, but no local lint/scanner command that reports these findings before publish.

## Solution Assessment

Unclear. A local scanner is a plausible direction, but the best implementation needs maintainer decisions on CLI/package placement, severity and exit-code semantics, hosted parity, and which security findings should be exposed before publish.
