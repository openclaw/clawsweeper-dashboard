## Summary

Keep open. Current main still lacks a local pre-publish plugin/package scanner; `package publish --dry-run` previews metadata only, while ClawScan/static findings are still produced on the hosted publish path after upload.

## What This Changes

The issue requests a local plugin/package scanner CLI for pre-publish validation of manifests, SKILL.md metadata, static-code findings, severity levels, JSON output, and nonzero exits.

## Best Possible Solution

Define a maintainer-approved local ClawHub scanner that reuses shared manifest/schema validation and deterministic static moderation logic, emits stable human and JSON findings, and documents how local results relate to hosted publish-time ClawScan.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence current main lacks a local scanner command, with only metadata dry-run preview and hosted post-upload scanning present.

## Solution Assessment

Unclear. A local scanner is a plausible direction, but the maintainable solution needs decisions on CLI/package placement, stable severity and exit-code semantics, hosted parity, and which security findings should be exposed before publish.
