## Summary

Keep open. ClawHub policy keeps issues open, and the feature gap is still real: current main has hosted publish-time scanning, package dry-run previews, and a repo-local ClawScan script, but no supported public plugin lint/pre-publish scanner for authors.

## What This Changes

Request to add a local plugin lint/pre-publish scanner CLI or package that validates plugin manifests, SKILL metadata, and code findings before publishing.

## Best Possible Solution

Define a maintainer-approved local scanner contract that reuses shared deterministic manifest/schema and static moderation logic, adds missing plugin/SKILL metadata checks, and documents local-vs-hosted parity with stable human and JSON output.

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main lacks a supported public plugin lint/pre-publish scanner command even though it has hosted scanning and a repo-local script.

## Solution Assessment

Unclear as an implementation plan. The direction is reasonable, but maintainers still need to choose CLI placement, scanner parity, severity semantics, JSON/exit behavior, and documentation before implementation.
