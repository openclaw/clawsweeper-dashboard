## Summary

Keep this issue open. Current main now provides useful local plugin preflight through `clawhub package validate` and package publish dry-run, but the broader requested security feedback remains unavailable before upload because `clawhub scan` explicitly rejects local folders. The related onboarding issue remains open and does not supersede this narrower scanner request.

## What This Changes

Requests a local CLI or package that validates plugin manifests, skill metadata, and security scanner findings before publication.

## Best Possible Solution

Make `clawhub package validate` the single supported local preflight, extend it with shared deterministic manifest and metadata checks, and expose only versioned, security-approved scanner rules locally while clearly identifying analysis that still requires server submission.

## Reproduction Assessment

Yes at source level. Passing a local path to `clawhub scan` deterministically reaches `rejectLocalScan()`, and the documentation confirms that full scan results require an uploaded version.

## Solution Assessment

The existing `clawhub package validate` command is the best foundation, but duplicating the entire server scanner locally would be risky; shared, versioned deterministic rules with an explicit server-only boundary are the narrower maintainable direction.
