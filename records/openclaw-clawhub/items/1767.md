## Summary

Keep open. This is an open ClawHub issue, and current main still lacks the requested public pre-publish plugin scanner CLI/package even though it now has package dry-run preview, an internal local ClawScan script, and hosted publish-time scanning.

## What This Changes

Request a local plugin scanner/linter CLI or package that validates manifest, SKILL.md, and code-scan findings before publishing, with JSON output, severity levels, and finding-based exit codes.

## Best Possible Solution

Define a maintainer-approved public scanner that reuses shared manifest validation and deterministic static scan logic, documents hosted-scan parity, and provides stable human/JSON output with severity and exit-code semantics.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection on current main confirms there is no public `clawhub package lint` or plugin-lint command, only package publish dry-run preview, an internal repo script, and server-side publish scanning.

## Solution Assessment

Unclear. A local scanner is a plausible direction, but the maintainable implementation depends on maintainer decisions about CLI/package placement, scanner parity, severity/exit behavior, and disclosure policy.
