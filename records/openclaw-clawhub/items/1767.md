## Summary

Keep open. ClawHub cleanup policy keeps issues open, and the feature gap is still real: current main has package publish dry-run, hosted publish-time scanning, and a repo-local `clawscan:local` script, but no public plugin/package lint scanner CLI with stable manifest/SKILL/code findings, severity semantics, and finding-based exit behavior.

## What This Changes

The issue requests a public local plugin scanner CLI or package for pre-publish manifest, SKILL.md, static code, JSON-output, and exit-code validation.

## Best Possible Solution

Define a maintainer-approved public scanner path that reuses shared manifest/schema validation and deterministic static scan logic, adds the missing SKILL/plugin metadata checks, supports stable human and JSON output, and documents how local findings relate to hosted ClawScan.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection and live issue state give high confidence that current main lacks the requested public plugin/package lint scanner command.

## Solution Assessment

Unclear. A local scanner is a plausible direction, but the maintainable solution needs maintainer decisions on command/package placement, severity and exit-code semantics, hosted parity, and security disclosure policy.
