## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has package dry-run, narrow local validation, hosted publish-time scanning, and rescan commands, but no local pre-publish plugin scanner/linter that reports manifest, SKILL.md, and code findings before publish.

## What This Changes

Requested a local pre-publish plugin scanner or CLI lint command that validates plugin manifests, SKILL.md metadata, and static code findings before publishing.

## Best Possible Solution

Define a maintainer-approved local ClawHub plugin scanner that reuses shared manifest/schema validation and deterministic static moderation logic, adds missing SKILL.md and plugin metadata checks, emits stable human and JSON severity output, and documents how local findings relate to hosted publish-time scans.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main lacks a local pre-publish plugin scanner command and that package dry-run remains a metadata/payload preview.

## Solution Assessment

Unclear. A local scanner is a plausible direction, but the best implementation depends on maintainer decisions about command placement, severity semantics, hosted parity, and security-disclosure policy.
