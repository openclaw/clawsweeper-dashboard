## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still lacks a local plugin lint/scanner command; package dry-run only previews publish metadata, while static scanner and ClawScan behavior remain hosted publish-time paths.

## What This Changes

Request a local plugin scanner CLI or dry-run publish mode that validates plugin manifests, SKILL.md metadata, static security findings, structured JSON output, and severity before publishing.

## Best Possible Solution

Define a maintainer-approved local scanner contract that reuses shared deterministic validation where feasible, adds the missing plugin/SKILL metadata checks, provides stable human and JSON output, and documents how local results relate to hosted publish-time scanning.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection verifies current main has package publish dry-run and hosted scan paths, but no local plugin lint/scanner command that reports the requested findings before publish.

## Solution Assessment

Unclear. The requested direction is reasonable, but the maintainable solution needs maintainer decisions on command/package placement, severity mapping, JSON schema, and parity with hosted ClawScan/static analysis before implementation.
