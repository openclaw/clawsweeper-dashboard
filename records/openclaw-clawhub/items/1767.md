## Summary

Current main substantially improves local plugin validation, but it still does not provide canonical pre-publish ClawScan security results for a local plugin package, so meaningful requested behavior remains open.

## What This Changes

Add a supported pre-publish plugin scanner with manifest, SKILL.md, code-security, severity, exit-code, and JSON validation.

## Best Possible Solution

Keep Plugin Inspector as the fast deterministic local stage, then add an authenticated non-publishing plugin scan job that returns the canonical structured security report without creating a public package version.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source and tests nevertheless establish the product gap with high confidence because local package validation exists while local canonical scans are explicitly rejected.

## Solution Assessment

No, not exactly as an independent scanner implementation. Reusing the canonical scan-job and report pipeline for non-publishing plugin artifacts would avoid scanner drift, while Plugin Inspector should continue handling deterministic local checks.
