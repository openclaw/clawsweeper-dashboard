## Summary

Keep open. This is a valid ClawHub feature request, and current main still does not expose a public `clawhub` plugin lint/scanner command that gives pre-publish manifest, SKILL.md, code-risk, severity, and JSON findings before publishing.

## What This Changes

Request a local plugin scanner CLI or dry-run publish mode for pre-publish validation of plugin manifests, SKILL.md metadata, and code-risk findings.

## Best Possible Solution

Define a maintainer-approved local scanner that reuses shared deterministic validation/static scan logic, adds the missing plugin/SKILL metadata checks, emits stable human and JSON severity output, and documents how local results relate to hosted ClawScan.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main lacks the requested public local plugin lint/scanner command despite adjacent dry-run and internal ClawScan script pieces.

## Solution Assessment

Unclear. The requested direction is useful, but maintainers still need to decide package-versus-CLI placement, scanner parity, stable severity semantics, and which findings should block, warn, or route to review.
