## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has only publish dry-run, hosted static scanning, and rescan flows rather than the requested local pre-publish plugin scanner.

## What This Changes

This issue requests a local pre-publish plugin scanner or CLI lint command that validates plugin manifests, SKILL.md metadata, code security heuristics, severities, exit codes, and JSON output before publishing.

## Best Possible Solution

Define a maintainer-approved local scanner that reuses shared manifest/schema validation and deterministic static moderation logic, emits stable human and JSON findings with severity codes, and documents how local results relate to hosted publish-time scans.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The verification path is source/docs inspection, which shows dry-run, hosted scan, and rescan paths but no local pre-publish scanner command.

## Solution Assessment

Unclear. A local scanner is plausible, but the best maintainable shape depends on maintainer decisions about scanner scope, severity semantics, hosted parity, and whether it belongs in the CLI or a standalone package.
