## Summary

Keep open. Current main still lacks a supported public local plugin lint/pre-publish scanner; dry-run publish, hosted scan results, and the repo-local ClawScan dry-run script are useful adjacent pieces but do not satisfy the requested stable CLI/package contract.

## What This Changes

The issue requests a supported local plugin scanner or CLI lint/dry-run command that validates plugin manifests, SKILL.md metadata, and code/security findings before publishing.

## Best Possible Solution

Define a maintainer-approved local ClawHub scanner contract that reuses shared deterministic validation, documents hosted-scan parity and divergence, and exposes a stable public CLI/package surface with human and JSON output.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main lacks the requested supported public local lint/pre-publish scanner command, even though hosted scanning and a repo-local dry-run script exist.

## Solution Assessment

Unclear as a final implementation plan. The direction is useful, but maintainers still need to choose the public surface, hosted parity rules, severity semantics, JSON schema, exit behavior, and privacy model before implementation.
