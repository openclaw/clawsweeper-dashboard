## Summary

Keep open. Current main has useful adjacent preflight tooling, especially `clawhub package validate`, but the requested supported local plugin scanner/lint command with manifest, SKILL.md, code-finding, severity, exit-code, and JSON scanner parity is not definitely implemented.

## What This Changes

Request to add a local pre-publish plugin lint/scanner command or package with manifest, SKILL.md, code-scan, severity, exit-code, and JSON output.

## Best Possible Solution

Define a maintainer-approved local pre-publish scanner surface that reuses shared Plugin Inspector and deterministic scanner logic, emits stable human and JSON output, documents hosted-scan parity, and keeps security-sensitive behavior in specs.

## Reproduction Assessment

Yes, at source level: current main documents and implements `clawhub scan` as published-scan only, while `package validate` is a partial Plugin Inspector preflight rather than the requested local scanner. I did not run the CLI because this was a read-only review.

## Solution Assessment

No, the current partial implementation is not enough to close this issue. The maintainable solution needs a chosen command/API surface and an explicit parity contract between local validation and hosted ClawHub scanner results.
