## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still lacks the requested local pre-publish plugin scanner; existing dry-run, hosted static scanning, and rescan paths do not provide the local lint workflow described in the report.

## What This Changes

Request a local ClawHub/OpenClaw plugin lint or scanner CLI/package for pre-publish manifest, SKILL.md, and code/security validation with CI-friendly output.

## Best Possible Solution

Provide a maintainer-approved local scanner that reuses shared manifest/schema validation and deterministic moderation logic, adds missing plugin/SKILL metadata checks, emits stable human and JSON findings, and documents how local results relate to hosted scanning.

## Solution Assessment

Unclear. A local command is a plausible direction, but the best maintainable solution depends on maintainer decisions about scanner scope, severity semantics, offline parity, and CLI/package placement.
