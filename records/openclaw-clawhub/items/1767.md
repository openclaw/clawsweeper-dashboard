## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main only partially addresses this request with a repo-local `clawscan:local` script; it is not yet a supported pre-publish `clawhub package/plugin lint` workflow with stable manifest/SKILL checks, severity semantics, nonzero finding exits, and documented hosted-scan parity.

## What This Changes

Request a local pre-publish scanner/linter for OpenClaw plugins that validates manifests, SKILL.md metadata, code security heuristics, JSON output, and error/warning exit behavior.

## Best Possible Solution

Productize the local dry-run into a supported CLI/package command that reuses shared static scan and manifest validation, defines stable severity/exit/JSON semantics, and documents hosted parity plus LLM data-disclosure or static-only behavior.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection shows current main has a repo-local `clawscan:local` script but no supported package/plugin lint command matching the requested pre-publish workflow.

## Solution Assessment

Unclear. A local scanner is directionally appropriate, but the current repo-local dry-run script is not the full supported CLI/package solution, and maintainers still need to define scope, severity exits, hosted parity, and data-disclosure behavior.
