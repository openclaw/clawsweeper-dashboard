## Summary

Keep open. Current main has hosted/published scan commands, package publish dry-run, and a repo-local ClawScan helper, but the public CLI and docs still reject local scans and no supported local plugin pre-publish lint/scan contract exists.

## What This Changes

Request a supported local CLI/package that validates plugin manifests, SKILL.md metadata, and code-scanner findings before publishing, with JSON, severity, and exit-code output.

## Best Possible Solution

Define a supported public ClawHub CLI or package scanner contract for plugins that reuses hosted validation/static-scan logic where safe and documents severity, JSON schema, exit codes, and privacy boundaries.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs do give a high-confidence current-main check: the public CLI rejects local path scans and package dry-run is not a scanner.

## Solution Assessment

Unclear until maintainers choose the product/security contract. The narrow maintainable path is a supported ClawHub CLI/package scanner with explicit hosted-parity and privacy semantics, not simply exposing the repo-local helper unchanged.
