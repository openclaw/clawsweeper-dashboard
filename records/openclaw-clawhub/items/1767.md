## Summary

Keep open. Current main partially addresses the request with `clawhub package validate` for bundled Plugin Inspector findings, but the broader supported pre-publish scanner requested here still lacks local ClawScan/static/LLM parity, credential/code-finding semantics, and a maintainer-approved redaction and severity contract.

## What This Changes

Request a local plugin pre-publish scanner CLI that validates plugin manifests, SKILL.md metadata, code/security heuristics, JSON output, exit codes, and severity levels before publishing.

## Best Possible Solution

Define one maintainer-approved pre-publish plugin scanner contract, likely by extending `clawhub package validate` or documenting a sibling command with stable human/JSON output, redaction, severity, and hosted-scan parity rules.

## Reproduction Assessment

Yes, at source level: current main supports Plugin Inspector validation through `clawhub package validate`, rejects local `clawhub scan` paths, and keeps ClawScan static/LLM local scanning behind an internal script requiring `OPENAI_API_KEY`. This is a capability gap rather than a failing runtime bug.

## Solution Assessment

Unclear. A supported local pre-publish scanner is useful, but maintainers should choose whether to extend `clawhub package validate`, promote the internal ClawScan dry-run, or define a separate command before implementation.
