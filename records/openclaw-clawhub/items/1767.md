## Summary

Keep open. This is an open ClawHub feature request, and current main still has package publish dry-run plus hosted post-upload scanning, but no local plugin lint/scanner command that reports manifest, SKILL.md, and code-risk findings before publish.

## What This Changes

Request to add a local plugin scanner or lint/dry-run CLI that validates plugin manifests, SKILL metadata, code-risk heuristics, JSON output, and severity levels before publish.

## Best Possible Solution

Define a maintainer-approved local scanner that reuses shared manifest/schema validation and deterministic static moderation logic, emits stable human and JSON severity output, and documents how local results relate to hosted publish-time scans.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main lacks the requested local plugin lint/scanner command.

## Solution Assessment

Unclear. The requested direction is reasonable, but the best implementation needs maintainer decisions on package-versus-built-in CLI placement, scanner parity, and which findings warn, block, or route to review.
