## Summary

Keep open. Current main has an internal `clawscan:local` repo script, but it still does not ship the requested public pre-publish plugin scanner CLI/package with stable manifest/SKILL/code lint checks, finding-based exit behavior, and documented hosted-scan parity.

## What This Changes

Add a local pre-publish plugin scanner or lint command that reports manifest, SKILL.md, and code findings with JSON output, severity levels, and nonzero exits for publish-blocking findings.

## Best Possible Solution

Promote or replace the internal local ClawScan script with a maintainer-approved public CLI/package that reuses shared validation and static scan logic, defines stable JSON/severity/exit-code semantics, and documents how local results relate to hosted ClawScan.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection on current main gives high confidence there is no public `clawhub package/plugin lint` scanner, only publish dry-run preview, hosted scanning, and an internal repo script.

## Solution Assessment

Unclear. A local scanner is a plausible direction, and the new internal script is useful groundwork, but the maintainable solution needs maintainer decisions on CLI/package placement, severity and exit-code semantics, hosted parity, and scanner disclosure policy.
