## Summary

Current main still does not provide pre-publish scanning for a local plugin: the public scan command rejects local folders, and plugin scan archives are only available for already-published versions. The related onboarding issue now treats this issue as the canonical scanner contract, so the remaining work is neither implemented nor superseded.

## What This Changes

Add a local pre-publish plugin scanner CLI that validates plugin manifests, skill metadata, and security signals with CI-friendly structured output and meaningful exit status.

## Best Possible Solution

Extend the canonical ClawHub scan API and CLI with authenticated ephemeral unpublished-plugin scans, reuse shared package-contract and production scanner logic, and clearly separate deterministic validation from hosted heuristic analysis in stable JSON and exit-code semantics.

## Reproduction Assessment

Yes for the reported feature gap: current main explicitly rejects local-path scans, and the only documented plugin scan command downloads results for an already-published version.

## Solution Assessment

Yes in overall direction, but the best implementation is an extension of the canonical ClawHub scan contract rather than a separate scanner package that could drift from hosted policy.
