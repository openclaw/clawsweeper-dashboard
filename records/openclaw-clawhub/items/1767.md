## Summary

Keep open. Current main has useful adjacent pieces, especially `clawhub package validate` and an internal `clawscan:local` script, but the supported CLI/docs still do not provide the requested local plugin scanner/lint surface with clear hosted-scan parity before publish.

## What This Changes

Request a local pre-publish plugin scanner or lint CLI that checks plugin manifests, SKILL.md metadata, code/security patterns, severity levels, JSON output, and nonzero exits before authors publish.

## Best Possible Solution

Define one supported local pre-publish plugin scanner contract that reuses shared validation/static-scan logic where possible, documents how results relate to hosted ClawScan, and exposes stable human and JSON output for authors and CI.

## Reproduction Assessment

Yes, from source inspection: current main rejects local `clawhub scan` paths and only documents `package validate` as Plugin Inspector validation, not the full local scanner requested. I did not execute the CLI because this was a read-only checkout review.

## Solution Assessment

No, closing as solved would be premature. `package validate` and the internal local ClawScan script are useful partial paths, but maintainers still need to choose the supported scanner surface and parity contract.
