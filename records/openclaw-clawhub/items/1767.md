## Summary

Keep open. Current main has useful adjacent author-facing validation, but it still does not provide the requested supported local plugin scanner with clear hosted-scan parity before publish.

## What This Changes

Request a local CLI/package command that plugin authors can run before publishing to catch manifest, SKILL.md, credential-pattern, and scanner-style findings with human and JSON output.

## Best Possible Solution

Define one supported local pre-publish plugin scanner contract that reuses shared validation/static-scan logic, documents hosted-scan parity limits, and exposes stable human and JSON output for authors and CI.

## Reproduction Assessment

Yes. Source inspection shows current main supports `clawhub package validate` and an internal local ClawScan script, but the public scan command still rejects local paths and docs do not present a full local scanner parity surface.

## Solution Assessment

No. Closing as solved would be premature because the current implementation is a useful partial path, while maintainers still need to choose the supported scanner/lint command surface and parity contract.
