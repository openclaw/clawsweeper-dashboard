## Summary

Keep open. ClawHub cleanup policy does not auto-close issues, and current v0.16.0/main still lacks a public `clawhub package lint` or pre-publish scanner command; the existing publish dry-run and repo-local ClawScan script do not satisfy the requested stable plugin-author scanner contract.

## What This Changes

Request a public local plugin scanner CLI for manifest, SKILL.md, and code validation before publishing.

## Best Possible Solution

Define a maintainer-approved public scanner contract that reuses shared deterministic manifest/schema and moderation logic, adds the missing plugin/SKILL metadata checks, emits stable human and JSON findings with documented severities and exit codes, and clearly documents how local results relate to hosted ClawScan.

## Solution Assessment

Unclear. The requested direction is reasonable, but the best implementation needs maintainer decisions on CLI placement, hosted parity, severity semantics, exit behavior, local data handling, and documentation before code is started.
