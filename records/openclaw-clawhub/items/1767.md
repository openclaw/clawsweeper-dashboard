## Summary

Current main already ships the requested local pre-publish plugin validation path as `clawhub package validate <source>`, with bundled Plugin Inspector checks, JSON output, nonzero hard-error exits, report files, tests, remediation docs, and release provenance in v0.22.0.

## What This Changes

Request a local CLI scanner or lint command so plugin authors can validate plugin package metadata, manifests, and scanner-style findings before publishing.

## Best Possible Solution

Keep `clawhub package validate <source>` as the supported local pre-publish plugin validation path, and track future scanner-parity refinements separately if authors find new gaps.

## Reproduction Assessment

## Solution Assessment
