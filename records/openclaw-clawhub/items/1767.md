## Summary

Keep open. Current main/v0.16.0 has a repo-local `clawscan:local` developer script and hosted publish-time scanning, but still does not provide the requested supported public plugin lint/pre-publish scanner command or package; PR https://github.com/openclaw/clawhub/pull/2143 explicitly left public CLI/backend dry-run support out of scope.

## What This Changes

Request a public local plugin scanner/linter CLI or package for manifest, SKILL.md, code-scan, severity, JSON-output, and pre-publish validation before publishing plugins.

## Best Possible Solution

Define a maintainer-approved local scanner contract that reuses shared deterministic manifest/schema and moderation logic where possible, documents hosted-scan parity and differences, and exposes stable human and JSON output through the chosen public CLI/package surface.
