## Summary

Current main already covers the requested local pre-publish plugin validation loop through `clawhub package validate <source>`, and that implementation shipped in the `v0.22.0` CLI release.

## What This Changes

The issue requests a local plugin scanner or lint CLI that plugin authors can run before publishing, with manifest/SKILL.md/code checks, structured output, severity levels, and nonzero exits for blocking findings.

## Best Possible Solution

Keep `clawhub package validate <source>` as the supported local pre-publish plugin validation path, and track future scanner-parity refinements separately if authors identify concrete gaps.

## Reproduction Assessment

Not applicable as a feature request. Source, docs, tests, Git history, and release metadata show the requested local pre-publish plugin validation path now exists on current main.

## Solution Assessment

Yes. The implemented `clawhub package validate <source>` path is a narrow maintainable solution because it reuses the bundled Plugin Inspector instead of creating a parallel scanner contract.
