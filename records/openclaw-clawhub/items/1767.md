## Summary

Current main still requires authors to publish before scanning and provides no local plugin preflight equivalent, so the requested capability remains necessary; it needs maintainer product and security decisions rather than automated implementation.

## What This Changes

Requests a local pre-publish plugin scanner CLI that validates manifests, SKILL.md metadata, and security heuristics with CI-friendly output.

## Best Possible Solution

Add a versioned plugin preflight surface that shares authoritative schema and static-analysis rules, reports unavailable server-only checks, and keeps local advisory results distinct from registry enforcement.

## Reproduction Assessment

Yes for the capability gap: current main deterministically rejects local paths in `clawhub scan`, and the registered plugin path only downloads reports for versions already submitted.

## Solution Assessment

No, a separate scanner implementation would risk policy drift; the best approach is to share versioned authoritative rules and explicitly define local versus server-only checks.
