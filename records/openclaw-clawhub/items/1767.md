## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still lacks the requested public pre-publish plugin scanner CLI/package; the only local scanner is a repo script for trusted developers while package dry-run remains a metadata preview.

## What This Changes

Request a local pre-publish scanner or CLI command that validates plugin manifests, SKILL.md metadata, code-scan findings, JSON output, severities, and exit codes before publishing.

## Best Possible Solution

Define a maintainer-approved public plugin preflight surface that reuses shared manifest/schema validation and deterministic static scan logic, supports stable human and JSON output, and documents how local results relate to hosted ClawScan.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main has no public package/plugin lint scanner, only package publish dry-run, hosted publish scanning, and an internal local ClawScan script.

## Solution Assessment

Unclear. A local scanner is a plausible direction, but the maintainable solution needs maintainer decisions on CLI/package placement, severity and exit-code semantics, hosted parity, and scanner disclosure policy.
