Request a local pre-publish plugin scanner or CLI linter that validates plugin manifests, SKILL.md metadata, code findings, severities, JSON output, and exit codes before publishing.

## Best Possible Solution

Define a maintainer-approved local scanner that reuses shared deterministic validation/scanning logic, emits stable human and JSON findings, and documents how local results relate to hosted publish-time ClawScan.

## Solution Assessment

Unclear. A local scanner is plausible, but the maintainable solution needs maintainer decisions on command/package placement, severity semantics, hosted parity, and which security findings should be exposed before publish.
