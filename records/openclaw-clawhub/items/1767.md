Keep open. Current main has a repo-local ClawScan dry-run script and hosted publish-time scanning, but it still lacks the requested supported public plugin lint/pre-publish scanner command or package; the remaining work is product- and security-sensitive rather than closable cleanup.

## Best Possible Solution

Define a maintainer-approved local scanner contract that reuses shared deterministic manifest/schema and moderation logic where possible, documents hosted-scan parity and differences, and exposes stable human and JSON output through the chosen public CLI/package surface.
