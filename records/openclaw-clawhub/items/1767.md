## Summary

Keep open. ClawHub issue policy keeps issues open, and the feature gap still exists: current main has hosted publish-time scanning and a repo-local ClawScan dry-run script, but not a supported public plugin lint/pre-publish scanner CLI/package with stable findings, severity, JSON, and exit semantics.

## What This Changes

Request a public local plugin scanner or CLI lint/dry-run command that catches manifest, SKILL.md, and code security findings before plugin authors publish.

## Best Possible Solution

Define a maintainer-approved local scanner contract that reuses shared manifest/schema validation and deterministic static moderation logic, adds missing plugin/SKILL metadata checks, documents hosted parity, and exposes stable human plus JSON output through the chosen public CLI/package surface.

## Reproduction Assessment

## Solution Assessment

Unclear as a final implementation plan. The direction is reasonable, but maintainers still need to choose the public surface, hosted parity rules, severity semantics, JSON schema, exit behavior, and documentation before implementation.
