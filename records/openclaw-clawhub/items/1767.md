Keep open. ClawHub cleanup policy keeps issues open, and current main still does not provide the requested supported public local plugin scanner CLI/package; the merged repo-local ClawScan script is intentionally not the public pre-publish workflow this issue asks for.

## What This Changes

The issue requests a local plugin scanner CLI or package that plugin authors can run before publishing to validate manifest, SKILL.md metadata, and code-scan findings with JSON output and failure severities.

## Best Possible Solution

Define a maintainer-approved local scanner contract that reuses shared manifest/schema validation and deterministic static moderation logic, documents local-vs-hosted parity, and exposes stable human and JSON output through the chosen public CLI/package surface.

## Security Review

Status: needs_attention

Summary: A public local scanner is security-sensitive because its findings, severities, and exit codes must not conflict with hosted ClawScan and moderation policy.

Concerns:

- **[medium] Define scanner parity before productizing local ClawScan:** `convex/lib/staticPublishScan.ts:22`
  - body: Hosted package scanning runs storage-backed static moderation after upload, while the repo-local script runs local static plus LLM analysis with an OpenAI key; a public CLI must define which results are authoritative and how severity and exit codes map to hosted outcomes.
  - confidence: 0.9
- **[medium] Decide the local LLM scan trust boundary:** `scripts/local-clawscan-dry-run.ts:107`
  - body: The repo-local scanner requires OPENAI_API_KEY and calls the OpenAI Responses API, so exposing it as supported publisher tooling needs explicit privacy, cost, authority, and output-stability decisions.
  - confidence: 0.86
- **[low] Account for metadata-only ClawPack scanning:** `specs/security-moderation.md:144`
  - body: The security spec says ClawPack package releases keep static/LLM scan inputs metadata-only for now, so a public local scanner must avoid promising deeper parity unless maintainers intentionally expand the scan contract.
  - confidence: 0.82
