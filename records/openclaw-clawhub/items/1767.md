## Summary

Keep open. This is an open ClawHub issue, and the feature gap remains on current main: package dry-run, narrow local validation, hosted publish-time static scanning, and rescans exist, but no local plugin lint/scanner command reports manifest, SKILL.md, and code findings before publishing.

## What This Changes

Request a local plugin scanner or CLI dry-run/lint mode that validates plugin manifests, SKILL.md metadata, code security heuristics, JSON output, and severity-based exits before publish.

## Best Possible Solution

Define a maintainer-approved local scanner that reuses shared manifest/schema validation and deterministic static moderation, adds plugin/SKILL metadata checks, supports stable human and JSON severity output, and documents how local results relate to hosted scans.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main lacks a local pre-publish plugin scanner command and that package dry-run remains a publish payload preview.

## Solution Assessment

Unclear. A local scanner is a plausible direction, but the best implementation depends on maintainer decisions about CLI/package placement, severity semantics, hosted parity, and scanner disclosure policy.
