## Summary

Keep open. This is an open ClawHub issue, and current main still has package publish dry-run, narrow local package validation, hosted publish-time static scanning, and owner rescan flows, but no local plugin lint/scanner command that reports manifest, SKILL.md, and code findings before publish; the broader onboarding issue remains open and overlapping rather than superseding this narrower request.

## What This Changes

Request a local plugin scanner or CLI lint command that validates plugin manifests, SKILL.md metadata, and static security findings before publishing.

## Best Possible Solution

Define a maintainer-approved local ClawHub plugin scanner that reuses shared manifest/schema validation and deterministic static moderation logic, adds the missing plugin/SKILL metadata checks, emits stable human and JSON severity output, and documents how local results relate to hosted publish-time scans.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main lacks a local pre-publish plugin scanner and that dry-run remains a publish-plan preview.

## Solution Assessment

Unclear. A local scanner is a plausible direction, but the best implementation depends on maintainer decisions about CLI/package placement, severity semantics, hosted scan parity, and coordination with the broader onboarding work.
