## Summary

Current main implements the requested local pre-publish scanner through `clawhub package validate <source>`, which runs the bundled Plugin Inspector against a local plugin folder with offline static validation, JSON output, report artifacts, severity handling, and non-zero exits for hard errors. The related publish workflow also supports `clawhub package publish <source> --dry-run`, so the central feedback-loop problem is resolved.

## What This Changes

Add a local pre-publish plugin scanner CLI that validates plugin manifests, SKILL metadata, and suspicious code patterns with structured output and actionable exit codes.

## Best Possible Solution

Keep Plugin Inspector as the single canonical rule engine and retain `clawhub package validate <source> --json` plus publish dry-run as the documented pre-publication validation workflow.
