## Summary

Keep open. ClawHub cleanup policy keeps issues open, and the current main recheck still shows hosted/static scanning and publish dry-run plumbing but no local pre-publish plugin scanner CLI with the requested manifest, SKILL.md, code, severity, exit-code, and JSON-output checks.

## What This Changes

The issue requests a local CLI or package scanner that plugin authors can run before publishing to validate plugin manifests, SKILL.md metadata, static code findings, severities, exit codes, and JSON output.

## Best Possible Solution

Define a maintainer-approved local scanner that reuses shared manifest/schema validation and deterministic static moderation logic, emits stable human and JSON findings, and documents how local results relate to hosted publish-time scans.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The verification path is source and docs inspection on current main, which shows hosted scans and dry-run previews but no local pre-publish scanner command.

## Solution Assessment

Unclear. A local scanner is plausible, but the best maintainable path depends on maintainer choices about scanner scope, severity semantics, hosted parity, and whether it belongs in the ClawHub CLI, a standalone package, or dry-run publish behavior.
