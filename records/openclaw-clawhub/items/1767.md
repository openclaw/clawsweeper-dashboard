## Summary

Keep open. ClawHub issue cleanup policy keeps issues open, and the feature gap is still real: current main has publish dry-run preview, hosted/static publish scanning, and an internal local ClawScan script, but no public pre-publish plugin lint/scanner CLI with stable manifest/SKILL/code checks, severity semantics, JSON output, and finding-based exit behavior.

## What This Changes

Request a public local plugin scanner or CLI lint path that validates plugin manifests, SKILL metadata, code/static findings, JSON output, severities, and nonzero exits before publishing.

## Best Possible Solution

Define a maintainer-approved local ClawHub scanner that reuses shared manifest/schema validation and deterministic static scan logic, adds missing plugin/SKILL metadata checks, supports human and JSON output with stable severity and exit-code semantics, and documents how local results relate to hosted publish-time ClawScan.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main still lacks a public local plugin lint/scanner command and only has publish dry-run preview, hosted scanning, and an internal repo script.

## Solution Assessment

Unclear. A local scanner is a plausible direction, but maintainers need to decide scanner placement, severity/exit-code semantics, hosted parity, and security disclosure policy before implementation.
