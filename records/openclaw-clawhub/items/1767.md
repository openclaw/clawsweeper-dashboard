## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still lacks the requested local pre-publish plugin lint/scanner workflow; existing dry-run, package rescan, and hosted publish-time scanning paths do not satisfy the feature request.

## What This Changes

Request a local CLI/package command that validates plugin manifests, SKILL.md metadata, and code scan findings before publishing, with severity levels and JSON output.

## Best Possible Solution

Define and ship a maintainer-approved local scanner that reuses shared package/schema validation and deterministic static moderation logic, adds the missing plugin/SKILL metadata checks, and documents how local severities map to hosted publish gates.

## Reproduction Assessment

Not applicable as a runtime bug. The feature gap is high-confidence by inspecting current CLI registrations, package bins, dry-run behavior, docs, and backend scanner call sites on current main.

## Solution Assessment

Unclear until maintainer scope is set. A local scanner is a plausible direction, but the maintainable solution depends on approved severity codes, JSON schema, offline/hosted parity, and CLI/package placement.
