## Summary

Keep open. Current main shipped useful adjacent preflight pieces, including `clawhub package validate`, `skill publish --dry-run`, and `package publish --dry-run`, but a supported local plugin scanner/lint command with scanner-style manifest, SKILL.md, and code findings is not definitely implemented; the public scan command still rejects local paths and the local ClawScan dry-run is an internal script.

## What This Changes

The issue requests a local CLI/package that plugin authors can run before publishing to validate plugin manifests, SKILL.md metadata, and scanner-style security findings.

## Best Possible Solution

Define a maintainer-approved local preflight path, either by expanding `clawhub package validate` or adding a dedicated lint/scan command, that reuses shared deterministic validation and scanner logic, emits stable human and JSON output, and documents how local results relate to hosted publish-time scans.

## Reproduction Assessment

Yes, at source level: current main has package validation and publish dry-run paths, but the public `clawhub scan` path rejects local folders and no supported local plugin-lint/scanner command is documented. I did not run the CLI because this was a read-only review.

## Solution Assessment

No, not as a close decision. The existing package validation path is a useful partial solution, but treating it as the full scanner requested here would assume security-scan parity and severity semantics that current code and docs do not clearly guarantee.
