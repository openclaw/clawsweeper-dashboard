## Summary

Keep open: current main fixed new ClawPack uploads to persist parsed artifact entries, but older metadata-only npm-pack releases can still need a data repair and rescan because inspect/file/LLM surfaces continue to read `release.files`.

## What This Changes

The issue requests that ClawPack/npm-pack package review and inspect surfaces use actual artifact entries instead of only metadata files.

## Best Possible Solution

Add a cursor-based repair/backfill that parses stored ClawPack artifacts for affected npm-pack releases, repopulates missing `release.files`, and reschedules package static/LLM scans while keeping the current all-entry write path for new uploads.

## Solution Assessment
