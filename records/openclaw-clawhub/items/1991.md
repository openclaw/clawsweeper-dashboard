## Summary

Keep open: the report is still valid for existing published ClawPack rows, and ClawHub cleanup policy does not auto-close issues. Current main now stores extracted ClawPack entries for new publishes, but live historical releases still expose only metadata to file/detail and LLM review paths.

## What This Changes

The issue asks ClawHub to prevent ClawPack npm-pack LLM reviews from judging only metadata files when the artifact contains implementation files.

## Best Possible Solution

Choose the approved ClawPack review source of truth, then either backfill and rescan historical npm-pack inventories or make metadata-only APIs and prompts explicitly describe their limited review scope.

## Reproduction Assessment

Yes. Live API checks reproduce the metadata-only file inventory and false missing-code LLM summaries for npm-pack releases, and current-main source shows those historical rows are still consumed through release.files.

## Solution Assessment

Unclear. Current main's extracted-entry storage is a good fix for new uploads, but historical verdicts need a maintainer decision between backfill/rescan, artifact-on-demand review, or explicit metadata-only wording.
