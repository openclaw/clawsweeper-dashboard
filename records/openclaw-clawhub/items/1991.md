## Summary

Keep open: current main now prevents the original two-file ClawPack upload shape for new publishes, but existing live releases still have truncated persisted file metadata and stale scan verdicts with no linked fixing PR or repair path.

## What This Changes

The issue asks ClawHub to make ClawPack/npm-pack code-plugin inspection and LLM review see artifact truth rather than only `package.json` and `openclaw.plugin.json`.

## Best Possible Solution

Add a cursor-based admin repair that rebuilds affected ClawPack release file metadata from stored tarballs, requeues stale package scan results, and updates the spec/tests so inspect, raw-file, and scanner surfaces agree with artifact truth.

## Reproduction Assessment

Yes. Live API checks reproduce the mismatch, and current main source shows package detail, raw-file lookup, and package LLM review all consume the persisted `release.files` list that is truncated on affected rows.

## Solution Assessment

Yes. Since current main already stores all parsed entries for new uploads, the narrow maintainable fix is a stored-data repair plus scan requeue and spec reconciliation rather than a publishing redesign.
