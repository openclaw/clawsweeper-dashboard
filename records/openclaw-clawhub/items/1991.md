The issue asks ClawHub to make ClawPack/npm-pack package inspect and LLM review use actual tarball contents instead of only persisted metadata files.

## Best Possible Solution

Add a cursor-based ClawPack release repair/backfill that reconstructs persisted file metadata from stored tarballs, requeues affected completed scans through the tarball-aware worker, and updates specs so inspect/raw-file and scan behavior match artifact truth.
