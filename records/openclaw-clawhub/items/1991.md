The issue reports that ClawPack npm-pack releases expose only metadata files to package inspect and LLM review even though the downloadable tarball contains implementation files.

## Best Possible Solution

Add a cursor-based admin repair that parses stored ClawPack tarballs, rebuilds missing release file metadata, requeues affected package ClawScan/LLM results, and updates specs/docs so inspect, raw-file, and scanner surfaces agree with artifact truth.
