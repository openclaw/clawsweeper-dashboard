## Repair Work Prompt

Fix the ClawPack/npm-pack package review context bug reported in this issue. Current main stores the exact uploaded `.tgz` and npm artifact facts, but `release.files` contains only `package.json` and `openclaw.plugin.json`; package LLM/static review and inspect/file surfaces treat that metadata list as the implementation surface. Keep artifact storage, download, and verification behavior intact. Add bounded ClawPack-aware context so the LLM prompt distinguishes registry metadata files from actual artifact facts and, where safe, selected parsed text entries such as README/source/runtime files. Avoid unbounded deep scans or inline serving of large/binary tarball entries. Add regression coverage for the prompt/evaluator and any API/CLI surface changed; update docs only if the public behavior or scan-scope wording changes.
