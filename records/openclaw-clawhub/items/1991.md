Keep open: the live registry still reproduces the metadata-only ClawPack inventory mismatch, and current main still feeds package detail, file serving, and package LLM review from `release.files`. ## What This Changes

The issue reports that ClawPack/npm-pack code-plugin releases can be falsely marked suspicious because package review sees only metadata files instead of the full artifact contents.

## Best Possible Solution

Define the authoritative ClawPack inventory for package detail, file serving, and ClawScan review, then backfill or clearly label historical metadata-only releases and rescan affected verdicts under that policy.

Unclear. Persisting parsed ClawPack entries for new uploads is the right storage direction, but historical inventory repair and public ClawScan rescan behavior need a maintainer policy decision before implementation.
