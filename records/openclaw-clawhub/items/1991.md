## Repair Work Prompt

Fix the ClawPack/npm-pack metadata truncation reported in https://github.com/openclaw/clawhub/issues/1991. On current main, legacy packageReleases can have artifactKind/npmFileCount/clawpackStorageId showing a full npm-pack artifact while release.files contains only package.json and openclaw.plugin.json; this breaks package inspect --files, GET /api/v1/packages/{name}/file, compatibility ZIPs, and stale LLM/SkillSpector review context. Add a cursor-paginated admin/internal repair for active npm-pack releases that parses the stored ClawPack tarball with the existing parseClawPack helper, stores any missing extracted entries with the same path, contentType, file-size, sha256, and storage conventions as new uploads, patches release.files and latestVersionSummary when the repaired release is latest, and enqueues or forces a package scan re-run for stale analyses. Do not mutate artifact bytes, do not change legacy ZIP behavior, and keep security/moderation fail-closed behavior intact. Update specs/security-moderation.md to reconcile durable intent with current artifact-backed scan behavior, and add regression coverage for new uploads, legacy repair, raw-file lookup, version detail files, and scan requeue.
