## Summary

Keep open: current production still reproduces the ClawPack artifact-versus-`release.files` mismatch for affected npm-pack releases, and current main only fixes new uploads rather than repairing existing truncated rows or stale scan verdicts.

## What This Changes

The issue asks ClawHub to make ClawPack/npm-pack package inspection and LLM review use the real artifact file entries instead of only persisted metadata files.

## Best Possible Solution

Add a cursor-paginated admin repair that parses stored ClawPack tarballs, restores missing release.files entries, requeues stale package scans, and updates specs/tests so inspection, raw-file, compatibility ZIP, and review surfaces agree with artifact truth.

## Solution Assessment
