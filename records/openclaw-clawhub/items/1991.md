## What This Changes

The issue asks ClawHub to make ClawPack/npm-pack package inspection and LLM review use the actual artifact file entries rather than only package metadata, avoiding false missing-code verdicts.

## Best Possible Solution

Add a cursor-based stored-release repair that reconstructs ClawPack file metadata from existing tarballs, requeues affected package scans, and updates the durable spec so inspect, raw-file, and scanner surfaces agree with artifact truth.

## Solution Assessment

Yes. Current main already stores parsed ClawPack entries for new uploads, so the narrow maintainable fix is stored-release metadata repair plus targeted scanner reset/requeue rather than a new publish workflow.
