## Summary

Keep open: the live affected ClawPack releases still reproduce the metadata-only inventory and stale suspicious LLM verdict, while current main only fixes future uploads by storing parsed ClawPack entries. Because this is a ClawHub issue and the remaining work changes moderation/review state, it should stay open for maintainer policy and backfill planning.

## What This Changes

Make ClawPack/npm-pack package detail, file access, static scan, and LLM review use actual artifact entries instead of only metadata files so complete code plugins are not flagged as missing code.

## Best Possible Solution

Choose the authoritative ClawPack review inventory, then backfill affected npm-pack release files and rescan or explicitly label historical metadata-only verdicts.

## Reproduction Assessment

Yes. Live public API checks reproduce metadata-only version.files and a 404 README.md file lookup while the artifact download contains the missing code and documentation files.

## Solution Assessment

Unclear. Current main is a good future-upload fix, but the remaining production mismatch needs a maintainer decision on historical backfill, rescans, and whether the metadata-only moderation spec should change.
