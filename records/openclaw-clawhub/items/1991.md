## Summary

ClawPack/npm-pack package review and inspect surfaces should use actual tarball entries instead of treating persisted metadata files as the complete package.

## Best Possible Solution

Add a cursor-based repair/backfill that reconstructs stored ClawPack file metadata from existing tarballs, requeues or resets affected ClawScan results, and updates specs/docs so inspect, raw-file, and scanner surfaces agree with artifact truth.
