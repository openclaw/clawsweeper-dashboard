## Summary

Keep open: current main fixes new ClawPack uploads and artifact-backed worker scans, but existing npm-pack rows still expose truncated `release.files` through version metadata, raw-file lookup, compatibility download inputs, and legacy LLM review data; live API checks still reproduce the false missing-code suspicious verdict.

## What This Changes

The issue asks ClawHub to make ClawPack/npm-pack code-plugin review and file surfaces evaluate the actual tarball entries instead of only registry metadata files.

## Best Possible Solution

Backfill existing ClawPack package releases from stored tarballs and requeue package scans so version files, raw-file endpoints, compatibility ZIPs, and review verdicts reflect artifact contents.
