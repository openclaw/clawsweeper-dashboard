## Summary

Keep open: the live registry still reproduces the metadata-only ClawPack inventory and stale missing-code LLM verdict for historical npm-pack releases, while current main only fixes new upload ingestion and leaves the historical scan/file policy unresolved.

## What This Changes

This issue asks ClawHub to make ClawPack/npm-pack package file and LLM review surfaces evaluate the actual tarball entries instead of only metadata files.

## Best Possible Solution

Define the authoritative ClawPack review/file inventory, then either backfill stored file metadata and rescan affected releases or clearly document historical file/detail and LLM surfaces as metadata-only.

## Reproduction Assessment

Yes. The live ClawHub API shows @openclaw/codex-claw@2026.5.6 has npmFileCount 11 but only two stored files and a missing-code suspicious LLM summary, while the artifact download lists the implementation files.

## Solution Assessment

Unclear. Storing parsed files for new uploads is a useful partial fix, but historical verdicts need a maintainer choice between backfill/rescan, artifact-on-demand review, or explicit metadata-only wording.
