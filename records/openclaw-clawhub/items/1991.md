## Summary

Keep open: the live registry still reproduces the historical npm-pack mismatch, and current main still routes raw file lookup, static rescans, and package LLM review through `release.files`; new uploads now store extracted ClawPack entries, but historical review behavior still needs a maintainer moderation decision.

## What This Changes

The issue asks ClawHub package review to include or clearly distinguish actual ClawPack tarball entries so npm-pack code plugins are not falsely flagged as missing implementation files.

## Best Possible Solution

Maintainers should choose the ClawPack review source of truth, then either backfill and rescan affected npm-pack releases or make historical metadata-only review/API surfaces explicit.
