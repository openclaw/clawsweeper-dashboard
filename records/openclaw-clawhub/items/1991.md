## What This Changes

The issue reports that ClawPack/npm-pack code-plugin releases expose only metadata files to package detail/file/LLM review surfaces, producing false missing-code scanner verdicts despite complete downloadable artifacts.

## Best Possible Solution

Add a cursor-paginated admin repair that parses stored ClawPack tarballs, restores missing release.files entries, requeues stale package scan results, and updates specs/tests so inspect, raw-file, and review surfaces agree with artifact truth.

## Reproduction Assessment

Yes. Public API checks reproduce the mismatch for two npm-pack releases, and current main still drives the affected detail, raw-file, compatibility ZIP, and package LLM surfaces from persisted release.files.

## Solution Assessment

Yes. Current main already stores full ClawPack entries for new uploads and the scan worker can inspect the tarball, so the narrow maintainable fix is a backfill/repair plus scan requeue for existing rows.
