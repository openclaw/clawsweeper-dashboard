## Summary

Keep open: the live package still reproduces the metadata-only review surface and suspicious missing-code verdict, while current main only fixes newly uploaded ClawPack rows and the durable security spec still says deep tarball LLM/static scanning is not enabled.

## What This Changes

The issue asks ClawHub to prevent npm-pack/ClawPack package reviews from treating metadata-only `release.files` as the complete artifact contents.

## Best Possible Solution

Define the authoritative ClawPack inventory for package APIs and ClawScan, then either backfill and rescan historical releases or explicitly label metadata-only review scope so verdicts stop implying the artifact is hollow.

## Reproduction Assessment

Yes. The live codex-claw release shows npmFileCount 11 with only two stored files and a suspicious missing-code LLM verdict, while current main still feeds package file and LLM surfaces from release.files for existing rows.

## Solution Assessment

Unclear. Persisting parsed entries for new uploads is the right storage direction, but historical inventory repair and public ClawScan verdict changes need an explicit maintainer security policy.
