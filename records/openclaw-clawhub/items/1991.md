## Summary

Keep open: current main still has the ClawPack/npm-pack review blind spot where package detail, file lookup, and LLM evaluation use metadata-only release files as if they were the package’s implementation surface.

## What This Changes

Request that ClawPack/npm-pack package review and inspect surfaces distinguish registry metadata files from the full downloadable artifact to avoid false missing-code suspicious verdicts.

## Best Possible Solution

Keep artifact storage/download behavior intact, but add explicit ClawPack artifact review context that labels release.files as metadata and provides artifact entry facts or selected safe text so missing-code verdicts are based on the actual npm-pack artifact boundary.

## Reproduction Assessment

Yes, as a source-reproducible path. Current main stores only two ClawPack metadata files in release.files, and package inspect/file plus LLM evaluation read from that list rather than the stored tarball entries.

## Solution Assessment

Yes. The narrow maintainable fix is to correct the package review context and artifact/metadata labeling, without changing artifact digests or turning this into unbounded dependency-graph scanning.

## Repair Work Prompt

Fix the ClawPack/npm-pack package review blind spot. Current upload stores the tarball and only package.json/openclaw.plugin.json in release.files; package detail/file and LLM evaluation then treat release.files as the complete implementation surface, so complete code plugins can receive missing-code suspicious verdicts. Preserve existing artifact storage, download, digest, and documented metadata-only deep-scan policy, but add explicit artifact context: label release.files as registry metadata and provide artifact entry facts or selected safe text from the stored ClawPack where appropriate so the evaluator cannot infer absent code from metadata-only files. Add regression coverage for an npm-pack containing README.md and dist/index.js in the tarball but not in release.files, covering package detail/file or CLI inspect behavior and the LLM prompt context. Do not implement unbounded full-tarball scanning, dependency-graph scanning, or digest-changing repack behavior.
