## Summary

Keep open: live package APIs still reproduce the two-file ClawPack metadata mismatch and stale suspicious LLM verdicts, while current main only prevents the bug for new uploads and has no linked fixing PR for existing releases.

## What This Changes

Ensure ClawPack/npm-pack code-plugin releases expose and scan the actual parsed artifact entries instead of only `package.json` and `openclaw.plugin.json` metadata.

## Best Possible Solution

Add a cursor-based backend repair that parses stored ClawPack tarballs, rebuilds missing release file metadata, requeues affected package LLM/ClawScan results, and updates the spec/docs so inspect, raw-file, and scanner surfaces agree with artifact truth.

## Solution Assessment

Yes. Current main already establishes parsed-entry storage as the new-upload behavior, so the narrow maintainable fix is repairing existing release metadata and requeueing stale scanner verdicts rather than redesigning package publish.
