## Summary

Keep open. Current main has the right shape for new ClawPack uploads, but historical npm-pack release rows can still have truncated `release.files`, and the open linked PR is the active fix candidate.

## What This Changes

The issue reports that ClawPack/npm-pack package review and inspect surfaces use truncated `release.files` rows, so complete code-plugin artifacts can be flagged as missing implementation files.

## Best Possible Solution

Review and land https://github.com/openclaw/clawhub/pull/2583, or an equivalent maintainer-approved migration, so historical ClawPack releases get artifact-derived file rows and fresh scans.

## Reproduction Assessment

Yes from source inspection and reporter evidence. Current main's package LLM review and file APIs read `release.files`, so historical rows containing only `package.json` and `openclaw.plugin.json` reproduce the hollow-package review surface.

## Solution Assessment

Yes, the artifact-derived repair plus rescan path is the narrow maintainable fix. The remaining decision is maintainer approval of the migration runner and production rollout, not whether the bug is real.
