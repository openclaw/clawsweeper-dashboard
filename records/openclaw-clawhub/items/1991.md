## Summary

Keep open: live npm-pack releases still reproduce the metadata-only file inventory and stale suspicious ClawScan verdict, while current main only covers new upload storage and leaves historical backfill/rescan policy unresolved.

## What This Changes

The issue asks ClawHub package detail/file and LLM review surfaces to use actual ClawPack npm-pack contents instead of only metadata files.

## Best Possible Solution

Define the authoritative ClawPack inventory and scan scope, then backfill and rescan affected npm-pack releases or explicitly label historical views as metadata-only.

## Reproduction Assessment

Yes. The live @openclaw/codex-claw@2026.5.6 API shows an npm-pack artifact with 11 files but only two stored version.files, and the artifact download lists the missing implementation files.

## Solution Assessment

Unclear. Current main's parsed-entry storage is the right direction for new uploads, but historical releases need a maintainer decision between backfill/rescan, artifact-on-demand review, or explicit metadata-only wording.
