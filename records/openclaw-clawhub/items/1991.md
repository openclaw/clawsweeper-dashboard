## Summary

Keep open: live package surfaces still reproduce the metadata-only ClawPack inventory and stale missing-code LLM verdicts, while current main only partially mitigates new uploads and leaves historical review-scope policy unresolved.

## What This Changes

The issue asks ClawHub to stop feeding metadata-only ClawPack inventories into package file/detail and LLM review surfaces for npm-pack code-plugin releases.

## Best Possible Solution

Define the authoritative ClawPack review input, reconcile the spec with ingestion behavior, then either backfill and rescan affected releases or clearly scope historical file/detail and LLM surfaces as metadata-only.

## Reproduction Assessment

Yes. The live ClawHub API shows @openclaw/codex-claw@2026.5.6 has npmFileCount 11 but only two stored files and a missing-code suspicious LLM summary, while the artifact download lists the implementation files.

## Solution Assessment

Unclear. Storing parsed files for new uploads is a useful partial fix, but historical verdicts need a maintainer choice between backfill/rescan, artifact-on-demand review, or explicit metadata-only wording.
