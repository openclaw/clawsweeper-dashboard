Keep open: the issue is still reproducible on the live registry and current `main` still derives package file and package LLM-review surfaces from `release.files`; new uploads now store parsed ClawPack entries, but historical inventory repair and ClawScan rescans need maintainer policy handling.

## What This Changes

The issue requests that ClawPack/npm-pack package review and file surfaces evaluate actual artifact entries instead of only metadata files so complete code-plugin releases are not flagged as missing code.

## Best Possible Solution
