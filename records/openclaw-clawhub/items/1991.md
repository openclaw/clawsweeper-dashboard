## Label Justifications

- `P2`: The bug produces false suspicious verdicts and incomplete inspection for a bounded set of historical npm-pack releases, while the verified artifacts remain downloadable.
- `impact:data-loss`: Implementation file metadata was silently omitted from persisted release rows used by inspection and moderation, despite remaining present in the underlying artifact.

## Summary

Current main fixes ingestion for new ClawPack publishes by persisting every parsed tarball entry, but affected historical releases still have incomplete `release.files`, and package inspection plus LLM evaluation continue to consume those persisted rows. The linked repair PR remains open, so this issue is still necessary until that migration lands or is replaced.

## What This Changes

Repair historical npm-pack ClawPack releases whose persisted inspection and review file rows omit implementation files present in the verified artifact.

## Best Possible Solution

Land a guarded, indexed, resumable migration that reconstructs historical release files from verified stored ClawPack artifacts, refreshes fingerprints and stale review state, and validates the operator workflow against a real Convex deployment.

## Reproduction Assessment

Yes. A historical npm-pack release whose `npmFileCount` exceeds the length of `release.files` follows a current-main path where inspection and LLM evaluation expose only those incomplete persisted rows.

## Solution Assessment

Yes in direction. Reconstructing the canonical historical rows from the verified artifact avoids adding competing artifact-reading behavior to every consumer, but the migration needs real Convex runtime validation before landing.
