## Summary

Keep open: ClawHub cleanup policy does not auto-close issues, and the historical/live npm-pack package surfaces still reproduce the metadata-only inventory and stale missing-code suspicious verdict despite current-main mitigations for new uploads.

## What This Changes

The issue requests that ClawPack/npm-pack package detail, file, and LLM review surfaces evaluate actual artifact entries instead of treating metadata-only `release.files` as the full package contents.

## Best Possible Solution

Define the authoritative review input for historical npm-pack releases, then backfill/rescan affected inventories or make file/detail/LLM surfaces explicitly describe their limited stored scope.

## Reproduction Assessment

Yes. Live API checks reproduce the metadata-only file inventory and stale missing-code LLM summary, and current-main source shows historical rows are still consumed through `release.files`.

## Solution Assessment

Unclear. The new-upload storage fix is useful, but the remaining historical verdict/file-surface problem needs a maintainer choice between backfill/rescan, artifact-on-demand review, or explicit metadata-only labeling.
