## Summary

Keep open: ClawHub policy keeps issues open for maintainer follow-up, and current main still has the reported ClawPack review blind spot. The upload path stores the real npm-pack tarball separately, but package file/detail and LLM review still use the metadata-only `release.files` list, which can make a complete code plugin look source-empty.

## What This Changes

The issue asks ClawHub to make npm-pack/ClawPack package inspection and LLM review aware of actual artifact contents instead of treating metadata-only `release.files` as the complete package source.

## Best Possible Solution

Keep artifact storage and download behavior intact, but expose enough parsed ClawPack artifact metadata or selected entries to package inspect/file and package LLM review so complete npm-pack plugins are not reviewed as source-empty.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: publish stores only metadata files for ClawPack releases, and package file/detail plus LLM review consume that same `release.files` list; I did not execute the live LLM path.

## Solution Assessment

Yes. The narrowest maintainable fix is to teach package inspect/file and LLM review to distinguish metadata-only `release.files` from actual npm-pack artifact contents, rather than changing artifact verification or download semantics.

## Repair Work Prompt

Fix the ClawPack npm-pack package inspection and LLM review blind spot reported in https://github.com/openclaw/clawhub/issues/1991. Keep existing artifact storage, verification, and download semantics intact. Update package detail/file and LLM review surfaces so npm-pack/ClawPack releases no longer treat metadata-only `release.files` as the complete source bundle: either expose selected parsed artifact entries such as README/SKILL/source/runtime files from the stored tarball, or pass explicit artifact-entry facts into the prompt so complete packages are not flagged as missing implementation. Add focused regression coverage for a multipart ClawPack publish containing package.json, openclaw.plugin.json, README/SKILL/source/runtime files, asserting the API/inspect and LLM prompt context distinguish metadata files from artifact contents. Do not broaden this into a full security scanning redesign or change VirusTotal/artifact verification behavior.
