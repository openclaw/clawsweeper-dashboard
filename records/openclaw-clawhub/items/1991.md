## Summary

Keep open: the live package surfaces still reproduce the reported metadata-only inventory and stale missing-code LLM verdict, and ClawHub cleanup policy does not auto-close issues. Current main mitigates new uploads, but historical npm-pack rows and public verdicts still need maintainer moderation-policy follow-up.

## What This Changes

The issue asks ClawHub package detail, file, and LLM review surfaces to evaluate real npm-pack ClawPack artifact entries instead of treating metadata-only release files as the whole package.

## Best Possible Solution

Define the authoritative ClawPack review input for new and historical npm-pack releases, then either backfill and rescan affected release file inventories or explicitly scope public file and LLM surfaces to metadata-only data.
