## Summary

Keep open: the report still reproduces for historical ClawPack/npm-pack releases, and current main still drives package detail, file serving, and package LLM review from `release.files`. New uploads now store parsed ClawPack entries, but repairing old rows and changing public ClawScan verdict inputs requires maintainer policy direction.

## What This Changes

The issue requests that ClawPack/npm-pack package file and LLM-review surfaces use the actual artifact inventory instead of treating metadata-only `release.files` as the full package.

## Best Possible Solution

Define the authoritative ClawPack inventory for package detail, file serving, and ClawScan review, then backfill or clearly label historical metadata-only releases and rescan affected verdicts under that policy.

## Reproduction Assessment

Yes. The live registry reproduces the mismatch, and current main still routes package detail, file lookup, and package LLM review through `release.files`, which reproduces the same failure for any historical npm-pack row stored with only metadata files.

## Solution Assessment
