## Summary

Keep open: current main fixes new ClawPack uploads, but source inspection still supports the remaining bug for previously stored metadata-only npm-pack releases because inspect, file, static scan, and LLM review paths continue to consume `release.files`.

## What This Changes

The issue requests that ClawPack npm-pack package review, inspect, and file surfaces evaluate actual artifact entries rather than only metadata files so code-plugin releases are not falsely flagged as missing implementation files.

## Best Possible Solution

Add a cursor-based ClawPack repair/backfill that parses stored npm-pack artifacts for affected releases, repopulates missing `release.files`, and reschedules package static and LLM scans while keeping the current all-entry new-upload path.
