## Summary

Keep open: the live affected release still exposes only metadata files to package detail/file/LLM surfaces even though its ClawPack artifact contains implementation files. Current main now stores extracted ClawPack entries for new uploads, but historical rows still need an approved inventory backfill and moderation rescan path before this report is resolved.

## What This Changes

Make ClawPack/npm-pack package file and LLM review surfaces use the actual tarball contents, not metadata-only release files, so complete code plugins are not flagged as missing code.

## Best Possible Solution

Adopt a single ClawPack review source of truth, backfill affected npm-pack release inventories from stored artifacts, rerun static/LLM moderation for those releases, and update specs/docs to match the chosen policy.

## Reproduction Assessment

Yes. Live API checks reproduce the affected release exposing only package.json/openclaw.plugin.json while its downloaded artifact contains implementation files, and current-main source shows file lookup and LLM review still consume release.files.

## Solution Assessment

Unclear. Storing extracted entries for new uploads is the right direction, but resolving this issue also needs an approved historical inventory backfill and moderation rescan rather than only changing the publish path.
