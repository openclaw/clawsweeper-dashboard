## Summary

Keep open: the live npm-pack examples still reproduce metadata-only `version.files` and stale review inputs, while current main only covers extracted-file storage for new uploads and leaves historical backfill/rescan policy unresolved.

## What This Changes

The issue asks ClawHub to make ClawPack/npm-pack package LLM and file-review surfaces evaluate artifact contents rather than only metadata files.

## Best Possible Solution

Define the authoritative ClawPack inventory and scan scope, backfill stored file metadata from verified artifacts where approved, then rescan affected releases or explicitly label historical reviews as metadata-only.

## Reproduction Assessment

Yes. The live API for `@openclaw/codex-claw@2026.5.6` still shows an npm-pack artifact with 11 files but only two stored `version.files`, while the artifact download contains the missing implementation files.

## Solution Assessment

Unclear. Parsed-entry storage for new uploads is the right direction, but the remaining historical backfill/rescan or metadata-only labeling decision should be made explicitly before changing public scan outcomes.
