## Summary

Keep open: current main fixes new multipart ClawPack publishes, but existing npm-pack releases can still have metadata-only `release.files`, and package detail, raw-file lookup, static scan, and LLM review still consume that stored inventory. ClawHub cleanup policy also keeps issues open unless maintainers resolve them explicitly.

## What This Changes

The issue reports that npm-pack ClawPack releases can be fully downloadable while package inspect and LLM moderation see only metadata files, causing false missing-code suspicious verdicts.

## Best Possible Solution

Use one audited artifact-derived ClawPack inventory for package detail, raw file lookup, static scan, and LLM review, then repair affected historical npm-pack releases with an explicit rescan and moderation-state plan.

## Reproduction Assessment

Yes. Source inspection plus live registry checks show npm-pack releases with complete artifacts but metadata-only `release.files`, and current-main detail/file/LLM/static paths consume that stored inventory.

## Solution Assessment

Unclear. The current new-publish extraction fix is the right building block, but the remaining solution needs maintainer choice between backfilling stored files, reading selected tarball files on demand, and how to update historical moderation state.
