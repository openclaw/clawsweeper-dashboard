The branch adds a new pure Convex library scanner and unit tests for detecting raw Node filesystem API usage and OpenClaw fs-safe usage in package file contents.

## Best Possible Solution

Land the scanner only with a concrete package-scan integration path, regression coverage for common import formats, and explicit alignment with the package scan scope in `specs/security-moderation.md`.

## Reproduction Assessment

Yes for the PR findings: the CI log shows the deadcode failure, and source inspection shows module references and bindings are read one line at a time. This is not a current-main runtime bug because the scanner files are only on the PR branch.

## Solution Assessment

No. A standalone uncalled production module is not the narrowest mergeable path under the repo gates; it should either be integrated into the intended package scan/dry-run path or held until that path is ready.
