## Summary

Keep open. This draft member PR contains meaningful scanner work that is not on current main, but it is not merge-ready because the new helper is unused by production code and the raw `fs` binding logic can create false evidence.

## What This Changes

Adds a new `convex/lib/packageFilesystemEvidenceScan` helper and tests to scan package file contents for raw Node filesystem and OpenClaw fs-safe evidence.

## Best Possible Solution

Make the scanner part of a documented package scan or admin dry-run path, update the package-scan spec if package contents are now in scope, and keep the evidence rules conservative enough to avoid false raw-fs hits.

## Reproduction Assessment

Yes for the review findings. The CI log reproduces the dead-code failure, and source inspection shows a deterministic false-positive path for unbound `fs.readFile(...)` calls.

## Solution Assessment

No. The scanner is a useful foundation, but the current branch is not the best merge shape until it is wired into an intended production/admin scan path or explicitly accepted as a production entry and the raw-fs binding logic is tightened.
