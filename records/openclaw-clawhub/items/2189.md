## Summary

Keep open. This member-authored draft PR is not implemented on current main and still has merge-blocking review findings: the new production helper is dead code under the static gate, and the scanner can report raw filesystem usage for unrelated objects named `fs`.

## What This Changes

Adds `convex/lib/packageFilesystemEvidenceScan` and tests to report bounded raw Node filesystem and OpenClaw fs-safe evidence from package file contents.

## Best Possible Solution

Wire the scanner into a documented package scan or admin dry-run path, update package-scan intent if content evidence is now in scope, and require an observed raw-fs binding before warning on filesystem calls.

## Reproduction Assessment

Yes for the review findings. The CI log reproduces the static dead-code failure, and source inspection shows a deterministic false-positive path for unbound `fs.readFile(...)` calls.

## Solution Assessment

No. The scanner is a useful foundation, but the branch is not the best merge shape until maintainers choose the intended scan path or approved entry point and the raw-fs binding logic is tightened.
