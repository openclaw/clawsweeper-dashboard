Full review comments:

- **[P2] Wire the scanner into an approved entry point:** `convex/lib/packageFilesystemEvidenceScan.ts:78-85`
  - body: `scanPackageFilesystemEvidence` is exported from a production Convex lib file, but only the new test imports it and CI already fails `deadcode:files` on this path. This needs a production/admin scan caller or an explicit approved standalone entry before merge.
  - confidence: 0.93
- **[P2] Require a confirmed raw fs binding:** `convex/lib/packageFilesystemEvidenceScan.ts:215`
  - body: `rawFsNamespaces` starts with `fs`, so code that defines a package-local virtual `fs` object and calls `fs.readFile(...)` will be reported as raw Node filesystem usage without any `fs` or `node:fs` import. Start empty and add bindings only after confirmed imports/requires.
  - confidence: 0.9
- **[P2] Track multi-line raw fs imports:** `convex/lib/packageFilesystemEvidenceScan.ts:109-113`
  - body: The scanner only recognizes a raw fs import when the module literal and executable import syntax are on the same physical line. Standard formatted multi-line imports can therefore miss the helper/namespace binding and let later raw filesystem calls escape the evidence bucket.
  - confidence: 0.86
