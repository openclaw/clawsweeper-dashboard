## Summary

Keep this open. Current main still has a credible publish-update path where an ordinary non-trusted update can move an active skill row to hidden pending-scan state, and the public search/list/detail/API surfaces filter hidden rows; ClawHub policy also keeps issues open for maintainer resolution.

## What This Changes

A published skill disappeared from search/listings after a web UI update returned a Convex server error, and the reporter asks for visibility plus slug/account preservation.

## Best Possible Solution

Preserve slug ownership and public continuity by serving the last approved public version while a new upload is pending scan or review, while still hiding malware-blocked, quality-quarantined, manually moderated, removed, and soft-deleted skills with owner-visible diagnostics.

## Reproduction Assessment

Yes. A code-level reproduction path is clear: update an active skill as an ordinary non-trusted publisher, observe insertVersion writing hidden pending-scan state to the skill row, then observe public search/list/detail/API hydration filtering that row out.

## Solution Assessment

No patch is under review. The narrow maintainable direction is not simply to keep the new upload active, but to separate public last-approved-version visibility from pending moderation state for the newly uploaded version.
