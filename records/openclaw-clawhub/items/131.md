## Summary

Keep open. Current main still has a credible publish-update path where a normal non-trusted publisher updates an already-active skill, the skill row is rewritten to hidden pending-scan moderation state, and public search/list/detail/API paths filter it out. The issue is also corroborated by multiple reporters and related visibility reports, and ClawHub cleanup policy keeps issues open unless maintainers resolve them directly.

## What This Changes

A publisher reports that updating an existing published skill through the web UI produced a server error and made the skill disappear from search, listings, and public access while preserving slug ownership remains important.

## Best Possible Solution

Preserve slug ownership and public continuity by serving the last approved public version while a new version is pending scan or review, while still hiding static-malicious, quality-quarantined, user-moderated, removed, manually locked, and malware-blocked skills, with owner-visible diagnostics and regression coverage across publish, search/list/detail, API reads, and scanner promotion.
