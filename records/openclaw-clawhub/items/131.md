## Summary

Keep open. Current main still has a source-level path where an ordinary non-trusted publisher updates an existing active skill and the skill row is rewritten to hidden pending scan, after which public search/list/detail surfaces filter it out; this needs a maintainer moderation/security decision rather than cleanup closure.

## What This Changes

The issue asks ClawHub to keep a previously published skill discoverable and slug-owned after an update instead of returning a server error and disappearing from search/listings.

## Best Possible Solution

Implement version-aware public continuity: preserve slug ownership and serve the last approved public listing/API/file projection while the new upload remains gated by scan or moderation review, with owner-visible diagnostics.

## Reproduction Assessment

Yes, at source level. An ordinary non-trusted publisher updating an existing active skill can reach `insertVersion`, have the skill row patched to hidden pending scan, sync that state into search digest, and then be filtered out by public search/list/detail conversion.

## Solution Assessment

No patch is under review. The direct preserve-active approach in https://github.com/openclaw/clawhub/pull/274 was closed on moderation-policy grounds, so the safer solution is last-approved-version continuity while pending content remains gated.
