## Summary

Keep open. Current main still has a source-reproducible path where a normal non-trusted update to an existing skill writes hidden pending-scan state to the skill row, and public search/list/detail paths filter that row instead of serving the prior approved version.

## What This Changes

The issue reports that an already-published skill disappeared from search/listings after a web UI update returned a server error, with the reporter asking to preserve the slug/account binding.

## Best Possible Solution

Preserve slug ownership and public continuity by serving the last approved public version while a new upload is pending scan or review, while still hiding static-malicious, quality-quarantined, user-moderated, removed, manually locked, and malware-blocked content with clear owner diagnostics.

## Reproduction Assessment

Yes, at source level. A normal non-trusted publisher updating an existing active skill reaches insertVersion, which writes hidden pending-scan state to the skill row, and public conversion/search paths then filter it out.

## Solution Assessment

No patch is under review. The direct preserve-active-status approach in https://github.com/openclaw/clawhub/pull/274 was rejected as a moderation bypass, so the safer direction is last-approved-version continuity while the new version remains gated.
