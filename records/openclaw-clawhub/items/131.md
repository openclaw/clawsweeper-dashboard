## Summary

Keep open. Current main still has a source-reproducible path where an ordinary non-trusted update to an existing active skill writes hidden moderation state onto the skill row, and public search/list/detail hydration filters that row out; for ClawHub cleanup, issues are not auto-closed without maintainer resolution.

## What This Changes

A published skill reportedly disappeared from search and listings after a web UI update returned “Server Error Called by client,” and a later successful publish did not restore visibility.

## Best Possible Solution

Preserve slug ownership and public continuity by serving the last approved public skill card, search digest, API metadata, file surface, and searchable embedding while a new version is pending scan/review, then promote the new version only after scanner and moderation gates clear it.

## Reproduction Assessment

Yes, at source level. An ordinary non-trusted publisher updating an existing active skill reaches insertVersion, which can write hidden pending-scan state to the skill row, and public conversion/search/detail paths then filter it out; I did not run a live Convex publish in this read-only review.

## Solution Assessment

No patch is under review. The direct preserve-active-status approach in https://github.com/openclaw/clawhub/pull/274 was closed as conflicting with moderation policy, so the safer direction is last-approved-version continuity while the new upload remains gated.
