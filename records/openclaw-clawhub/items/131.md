## Summary

Keep open. This is a ClawHub issue, and current main still has a source-reproducible path where an ordinary update can hide the existing skill row during pending scan and public search/list/detail/API paths filter it out.

## What This Changes

The issue reports an existing published skill disappearing from search/listings and a web publish Server Error after an attempted update.

## Best Possible Solution

Implement a version-aware public projection that keeps the last approved skill card, digest, searchable embedding, API metadata, and safe file surface visible while a new version remains gated, then promotes the new version only after moderation/scanner gates clear it.

## Reproduction Assessment

Yes, at source level. An ordinary non-trusted publisher updating an already-active skill reaches insertVersion, which can write hidden pending-scan state to the skill row, and public conversion/search/detail/API paths then filter it out; I did not run a live Convex publish.

## Solution Assessment

No patch is under review. The direct preserve-active-status approach in https://github.com/openclaw/clawhub/pull/274 was closed as a moderation bypass, so the safer direction is last-approved-version continuity while the new upload remains gated.
