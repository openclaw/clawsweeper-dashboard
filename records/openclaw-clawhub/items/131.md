## Summary

Keep open. Current main still contains a source-reproducible update path where an existing public skill can be rewritten to a non-public moderation state during publish, and the public search/detail paths filter that state out; ClawHub cleanup policy also keeps issues open unless maintainers resolve them directly.

## What This Changes

The issue reports that updating an already published skill via the web UI produced a server error and made the skill disappear from public search/listing surfaces while preserving slug ownership mattered.

## Best Possible Solution

Serve the last approved public version for search, listing, detail, API metadata, and installable files while a new upload is pending scan or review, then promote the new version only after scanner and moderation gates clear it.

## Reproduction Assessment

Yes, at source level. An ordinary non-trusted publisher updating an existing active skill can reach insertVersion, write hidden pending-scan moderation state to the skill row, and then public search/detail conversion filters that row out; I did not run a live Convex publish in this read-only review.

## Solution Assessment

No patch is under review. The direct preserve-active-status approach from https://github.com/openclaw/clawhub/pull/274 is not the safest maintainable solution because it can expose pending content; maintainers should define last-approved-version continuity instead.
