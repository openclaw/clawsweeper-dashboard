## Summary

Keep open. Current `main` still has a source-reproducible path where an ordinary non-trusted publisher updates an already-active skill, the skill row is patched to hidden pending scan, and public search/list/detail/API paths then filter it out; ClawHub cleanup policy also does not auto-close issues.

## What This Changes

A previously public skill disappeared from search/listings after the owner published an update and saw a web publish server error.

## Best Possible Solution

Preserve slug ownership and public continuity by serving the last approved public version while a new upload is pending scan or review, then promote the new version only after scanner and moderation gates clear it.

## Reproduction Assessment

Yes, at source level. An ordinary non-trusted publisher updating an already-active skill can reach `insertVersion`, have the skill-level row patched to hidden pending scan, and then disappear from public search/list/detail/API paths that require active moderation status.

## Solution Assessment

No patch is under review. The safest direction is not the closed preserve-active-row approach, but a last-approved public projection while the new upload remains gated until scanner and moderation finalization.
