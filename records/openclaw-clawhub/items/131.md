## Summary

Keep open. Current main still has a source-reproducible path where a normal non-trusted update to an existing active skill writes hidden pending-scan state onto the skill row, and public search/detail hydration filters that row out; ClawHub cleanup policy also keeps issues open unless maintainers resolve them directly.

## What This Changes

The issue reports that updating an already published skill through the web UI produced a server error and caused the skill to disappear from public search and listings.

## Best Possible Solution

Implement a version-aware public projection that keeps the last approved skill card, digest, API metadata, file access, and searchable embedding visible while a new version is pending review, then promotes the new version only after moderation and scanner gates clear it.

## Reproduction Assessment

Yes, at source level. An ordinary non-trusted publisher updating an existing active skill reaches insertVersion, which can write hidden pending-scan state to the skill row, and public conversion/search/detail paths then filter it out; I did not run a live Convex publish in this read-only review.

## Solution Assessment

No patch is under review. The direct preserve-active-status approach in https://github.com/openclaw/clawhub/pull/274 was closed as conflicting with moderation policy, so the safer direction is last-approved-version continuity while the new upload remains gated.
