## Summary

Keep this issue open. Current main still has a source-level path where a non-trusted publisher updating an active skill can rewrite the skill row to hidden pending-scan state, and public search/list/detail/API paths filter hidden rows out instead of serving the last approved public version.

## What This Changes

The issue reports that an already published skill disappeared from search/listings after a web UI update failed with a Convex server error, with concern about preserving the existing slug and account ownership.

## Best Possible Solution

Preserve slug ownership and public continuity by serving the last approved version while a new upload is pending scan or review, while still hiding malicious, quarantined, removed, and manually moderated content with owner-visible diagnostics.

## Reproduction Assessment

Yes, at source level. Updating an active skill as an ordinary non-trusted publisher flows through insertVersion, writes hidden pending-scan moderation to the skill row, and public search/list/detail/API hydration rejects that row.

## Solution Assessment

No patch is under review. The narrow maintainable fix is a last-approved-version continuity model, not wholesale public exposure of pending uploads while moderation and malware scanning are still running.
