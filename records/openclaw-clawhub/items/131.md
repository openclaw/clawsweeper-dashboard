## Summary

Keep open. Current main still has a source-reproducible path where an ordinary non-trusted update to an existing skill writes hidden moderation state onto the skill row and public search/list/detail hydration filters that row out; ClawHub cleanup policy also keeps issues open for maintainer resolution.

## What This Changes

The issue reports that an already-published skill disappeared from public search/listings after an attempted web UI update returned a server error, with the reporter asking for slug/account preservation.

## Best Possible Solution

Preserve slug ownership and public continuity by serving a version-aware last-approved skill card, digest, embedding, API metadata, and file surface while a new upload remains gated, then promote the new projection only after moderation and scanner gates clear it.

## Reproduction Assessment

Yes, at source level. An ordinary non-trusted publisher updating an existing active skill reaches insertVersion, which can write hidden pending-scan state to the skill row, and public conversion/search/list paths then filter it out; I did not run a live Convex publish in this read-only review.

## Solution Assessment

No patch is under review. The previous preserve-active approach in https://github.com/openclaw/clawhub/pull/274 was closed as conflicting with moderation policy, so the safer direction is last-approved-version continuity while the new upload remains gated.
