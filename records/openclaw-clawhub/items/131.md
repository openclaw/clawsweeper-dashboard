## Summary

Keep open. Current main still has a source-reproducible path where an ordinary skill update can move the skill row to hidden pending-scan state, and fixing that requires moderation/security policy judgment rather than conservative issue cleanup.

## What This Changes

The issue reports that updating a previously published skill produced a server error and made the skill disappear from search/listings while preserving the slug/account association remained important.

## Best Possible Solution

Implement a version-aware public projection that preserves slug/account ownership and serves the last approved public version while a new upload remains gated by scan and moderation review.

## Reproduction Assessment

Yes, at source level: an ordinary non-trusted publisher updating an existing active skill can have the skill row patched to hidden, then search/list/detail hydration filters it out. I did not reproduce the reporter’s private production row or the UI server-error toast directly.

## Solution Assessment

No patch is under review. The direct preserve-active approach from https://github.com/openclaw/clawhub/pull/274 was rejected on moderation-policy grounds, so the safer path is last-approved-version continuity while the new version stays gated.
