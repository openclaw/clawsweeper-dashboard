## Summary

Keep this open. Current main still has a source-reproducible path where a normal non-trusted update to an active skill writes hidden pending-scan moderation state to the skill row, and public search/list/detail paths filter that row instead of serving the last approved version; ClawHub policy also keeps issues open for maintainer resolution.

## What This Changes

The issue reports an already-published skill disappearing from public search/listings after a web UI update hit a server error, with later comments emphasizing slug and account preservation for an actively used API skill.

## Best Possible Solution

Preserve slug ownership and public continuity by serving the last approved public version while a new upload remains pending scan/review, with owner-facing diagnostics and scanner promotion preserving the moderation boundary.

## Reproduction Assessment

Yes, at source level. A normal non-trusted publisher updating an existing active skill reaches insertVersion, which writes hidden pending-scan state to the skill row, and public conversion/search paths then filter it out.

## Solution Assessment

No patch is under review. The previously proposed preserve-active-status approach in https://github.com/openclaw/clawhub/pull/274 was rejected as a moderation bypass, so the safer direction is last-approved-version continuity while the new version remains gated.
