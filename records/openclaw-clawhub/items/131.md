## Summary

Keep this open. Current main still has a source-reproducible path where an ordinary publisher update can write hidden pending-scan state onto an existing skill row, and ClawHub policy keeps issues open for maintainer resolution.

## What This Changes

The issue reports that updating an already published skill through ClawHub caused a server error and removed the skill from search and public listings.

## Best Possible Solution

Serve the last approved public version for an active skill while a newly uploaded version is pending scan or review, preserving slug ownership and public continuity while still hiding malicious, quality-quarantined, removed, or manually locked content with owner-visible diagnostics.

## Reproduction Assessment

Yes at source level. Updating an active skill as an ordinary non-trusted publisher reaches insertVersion, writes hidden pending-scan moderation state onto the existing skill row, and public search/list/detail hydration then filters that row out.

## Solution Assessment

No patch is under review. The narrow maintainable direction is not to expose every pending upload, but to separate public last-approved-version reads from the pending version under moderation.
