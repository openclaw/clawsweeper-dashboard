## Summary

Keep this open. Current main still has a source-level path where an ordinary non-trusted publisher updating an active skill can rewrite the skill row to hidden pending moderation, and public search/list/detail hydration then filters it out; ClawHub policy also keeps issues open unless a maintainer resolves them.

## What This Changes

Reports that updating a published skill via the web UI can produce a server error and remove the skill from public search/listings while slug ownership remains important.

## Best Possible Solution

Preserve slug ownership and public continuity by serving the last approved version while a new upload is pending scan/review, while still hiding malicious, quality-quarantined, manually moderated, removed, and malware-blocked content with clear owner diagnostics.

## Reproduction Assessment

Yes, at source level. Updating an active skill as an ordinary non-trusted publisher flows through insertVersion, writes hidden pending-scan moderation to the skill row, and public search/list/detail hydration rejects that row.

## Solution Assessment

No patch is under review. The safest solution is a last-approved-version continuity model, not simply exposing pending updated content while moderation scans are still unresolved.
