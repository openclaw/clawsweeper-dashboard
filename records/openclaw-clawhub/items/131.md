## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has a source-reproducible path where an ordinary update to an existing skill can write hidden pending-scan state onto the skill row, after which public search/list/detail/API paths filter it out.

## What This Changes

The issue asks ClawHub to preserve visibility and slug ownership for an already-published skill after an update instead of returning a publish server error and disappearing from search/listings.

## Best Possible Solution

Implement a version-aware public projection that keeps the last approved skill card, digest, searchable embedding, API metadata, and safe file surface visible while the new version remains gated, then promotes the new projection only after scanner and moderation gates clear it.

## Reproduction Assessment

Yes, at source level. An ordinary non-trusted publisher updating an already-active skill can reach `insertVersion`, have the skill row patched to hidden pending scan, and then disappear from public search/list/detail/API paths that require active moderation status.

## Solution Assessment

No patch is under review. The direct preserve-active approach from https://github.com/openclaw/clawhub/pull/274 was rejected on moderation-policy grounds, so the safer direction is last-approved-version continuity while new content remains gated.
