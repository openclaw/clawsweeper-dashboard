## Summary

Keep open. Current main still has a source-level path where an ordinary non-trusted publisher updating an already-active skill can rewrite the skill row to hidden pending scan, and public search/list/detail paths then filter it out; ClawHub cleanup policy also keeps issues open unless maintainers resolve them directly.

## What This Changes

A published skill disappears from public search/listings after the owner attempts a version update and sees a publish server error.

## Best Possible Solution

Preserve slug ownership and public continuity by serving the last approved public version while the new upload remains gated by scan/review, with owner-visible diagnostics and regression coverage across publish, search/list/detail, API reads, file access, and scanner promotion.

## Reproduction Assessment

Yes, at source level. An ordinary non-trusted publisher updating an existing active skill can reach `insertVersion`, have the skill row patched to hidden `pending.scan`, and then be filtered out by public search/list/detail conversion; I did not run a live Convex publish in this read-only review.

## Solution Assessment

No patch is under review. The direct preserve-active approach in https://github.com/openclaw/clawhub/pull/274 was closed on moderation-policy grounds, so the safer direction is last-approved-version continuity while pending content remains gated.
