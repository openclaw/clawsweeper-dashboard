## Summary

Keep this issue open under ClawHub cleanup policy. Current main still has a credible publish-update path where an ordinary publisher can move an already-active skill to hidden pending-scan state, and public search/detail/API hydration then suppresses it instead of serving the last approved public version.

## What This Changes

A publisher reports that updating an already published skill through the web UI produced a server error and made the skill disappear from search, listings, and public API access while needing slug ownership preserved.

## Best Possible Solution

Preserve slug ownership and public continuity by serving the last approved public version while a new upload is pending scan or review, while still hiding malware-blocked, manually moderated, removed, quality-quarantined, and explicitly hidden skills with owner-visible diagnostics.

## Reproduction Assessment

Yes. A high-confidence code reproduction path exists: update an active skill as a normal non-trusted publisher, observe insertVersion patch moderationStatus to hidden, then verify public search/detail/API paths reject the row through toPublicSkill.

## Solution Assessment

Unclear. The requested outcome is correct, but the implementation should be a version-aware last-approved public projection rather than simply keeping unscanned updated content public.
