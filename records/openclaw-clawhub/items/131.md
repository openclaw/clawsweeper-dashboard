## Summary

Keep this open. Current main still has a source-level path where an ordinary non-trusted publisher updating an existing active skill can rewrite the skill row to hidden pending moderation, and public search/list/detail hydration then filters it out; the fix needs maintainer moderation/security policy rather than cleanup closure.

## What This Changes

Publishing an update to an existing skill should not make the previously public skill disappear from search/listings or lose slug continuity while the new version is pending review.

## Best Possible Solution

Preserve slug ownership and public continuity by serving the last approved version while a new upload is pending scan/review, while still hiding malicious, quality-quarantined, manually moderated, removed, and malware-blocked content with clear owner diagnostics.

## Reproduction Assessment

Yes, at source level. Updating an active skill as an ordinary non-trusted publisher flows through insertVersion, writes hidden pending-scan moderation to the skill row, and public search/list/detail hydration rejects that row.

## Solution Assessment

No patch is under review. The safest fix is not wholesale exposure of pending uploads, but a last-approved-version continuity model that keeps public reads stable while moderation evaluates the new version.
