## Summary

Keep this open. Current main still has a source-reproducible path where a normal non-trusted publisher updating an active skill writes hidden moderation state onto the skill row, and public search/list/detail conversion then filters it out; ClawHub policy also keeps issues open for maintainer resolution.

## What This Changes

The issue reports that updating a published skill via the web UI hid it from search/listings and returned a publish server error, with slug ownership preservation as the main user concern.

## Best Possible Solution

Serve the last approved public version for an active skill while a new upload remains pending scan/review, preserving slug ownership and public continuity without exposing unreviewed content.

## Reproduction Assessment

Yes, at source level. A normal non-trusted publisher updating an existing active skill reaches insertVersion, which writes hidden moderation state to the skill row, and public conversion/search paths then filter it out.

## Solution Assessment

No patch is under review. A simple preserve-active-status change was already rejected as a moderation bypass in https://github.com/openclaw/clawhub/pull/274; the safer solution is last-approved-version continuity while the new version remains gated.
