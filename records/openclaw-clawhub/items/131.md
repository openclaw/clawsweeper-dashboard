## Summary

Keep this open. Current main still has a source-reproducible path where an ordinary non-trusted publisher updating an existing active skill can rewrite the skill row to hidden moderation state, after which public search/detail/list conversion filters it out; ClawHub policy also keeps issues open unless maintainers resolve them.

## What This Changes

The issue reports that an already published skill disappeared from search/listings after a web UI update, with a publish server error and concern about preserving the slug/account association.

## Best Possible Solution

Preserve slug ownership and public continuity by serving the last approved public version while a new update is pending scan/review, while still hiding malicious, quality-quarantined, manually moderated, removed, and malware-blocked content with clear owner diagnostics.

## Reproduction Assessment

Yes, at source level. Updating an existing active skill as an ordinary non-trusted publisher reaches insertVersion, writes hidden moderation state to the skill row, and public search/detail/list conversion then filters that row out.

## Solution Assessment

No patch is under review. The narrow source path is clear, but the maintainable solution is a last-approved-version continuity model rather than exposing newly uploaded content before moderation gates complete.
