## Summary

Keep open. This is an issue, which the ClawHub cleanup policy does not auto-close, and current main still has a source-level path where an ordinary update can hide an existing active skill before public search/list/detail surfaces hydrate it.

## What This Changes

The issue asks ClawHub to stop a previously published skill from disappearing from search/listings after a web UI update while preserving the owner’s slug/account association.

## Best Possible Solution

Implement a version-aware public projection that preserves slug ownership and serves the last approved public listing, digest, API metadata, and safe file surface while a new version remains gated by scan or moderation review.

## Reproduction Assessment

Yes, at source level: an ordinary non-trusted publisher updating an already-active skill can reach `insertVersion`, have the skill row patched to hidden pending scan, sync that state into search digest, and then be filtered out by public search/list/detail conversion. I did not reproduce the reporter’s private production row or the web UI server-error toast directly.

## Solution Assessment

No patch is under review. The direct preserve-active approach in https://github.com/openclaw/clawhub/pull/274 was closed on moderation-policy grounds, so the safer direction is last-approved-version continuity while the new upload remains gated.
