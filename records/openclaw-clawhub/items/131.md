## Summary

Keep open. Current main still has a source-reproducible publish-update path where an ordinary publisher can move an existing skill into hidden pending-scan state, after which public search/list/detail/API surfaces filter it out; ClawHub issue cleanup policy also does not auto-close issues.

## What This Changes

A publisher reports an existing skill disappearing from search/listings and showing a server error after publishing an update via the web UI.

## Best Possible Solution

Implement a version-aware public projection that keeps the last approved skill card, digest, searchable embedding, API metadata, and safe file surface visible while a new version is pending scan/review, then promotes the new version only after scanner and moderation gates clear it.

## Reproduction Assessment

Yes, at source level. An ordinary non-trusted publisher updating an already-active skill can reach `insertVersion`, have the skill row patched to hidden pending scan, and then disappear from public search/list/detail/API paths that require active moderation status.

## Solution Assessment

No patch is under review. The prior direct preserve-active approach is not the safest fix because it can bypass pending-scan visibility gating; the maintainable direction is last-approved-version continuity while the new upload remains gated.
