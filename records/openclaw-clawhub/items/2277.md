## Summary

Keep open for maintainer review: current main appears vulnerable to the moderated-skill pending-transfer race and the patch targets the right gate, but the external PR still needs real behavior proof and a small cancellation persistence cleanup.

## What This Changes

The branch adds a moderation-state guard in `acceptTransferInternal` so pending transfer acceptance rejects skills marked malicious, hidden, or removed.

## Best Possible Solution

Land a guarded accept-transfer path with regression coverage and proof so malicious, hidden, or removed skills cannot change owner through a pending transfer before autoban completes.

## Reproduction Assessment

Yes, from source inspection: create a pending transfer, have a scanner patch the skill to `moderationVerdict: "malicious"` and `moderationStatus: "hidden"` before the scheduled autoban runs, then current main's `acceptTransferInternal` proceeds past only the `softDeletedAt` check and accepts ownership.

## Solution Assessment

Yes for the narrow owner-change race: checking moderation state immediately after loading the skill blocks the shared pending-transfer accept gate. The cancellation status persistence and after-fix proof should be tightened before merge.
