## Summary

Keep open. Repository policy for this ClawHub cleanup pass does not allow auto-closing issues, and the requested behavior is still only partially addressed: current main can suppress an uncorroborated VirusTotal Code Insight suspicious result in moderation when local scans are clean, but public VT/API/UI surfaces still expose the Code Insight status without a first-class AV detection ratio.

## What This Changes

The issue asks ClawHub to stop labeling a skill as suspicious from VirusTotal heuristic or Code Insight metadata when the linked VirusTotal engine report has 0/63 detections, or to show that detection ratio next to the status.

## Best Possible Solution

Define one canonical VirusTotal policy for skills: compute and expose an AV engine detection ratio from engineStats, keep Code Insight visible as contextual heuristic analysis, base public warning semantics on the agreed signal split, update API/UI/docs/tests, and provide a rescan/backfill or moderator override path for already-scanned affected skill versions.
