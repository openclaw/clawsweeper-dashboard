## Summary

Keep open. Current main still has a source-reproducible publish path where uncached GitHub account age triggers a GitHub API lookup and 403/429 responses produce the reported rate-limit error; a second user’s Publish plugin report matches the package publish path using the same gate.

## What This Changes

Publishing a new skill fails with a GitHub API rate-limit error during the upload/publish flow.

## Best Possible Solution

Verify production GITHUB_TOKEN/quota and Convex 403/429 logs, then reduce avoidable publish-time GitHub lookups or improve retry/reset guidance while preserving immutable-ID account-age enforcement.

## Reproduction Assessment

Yes, by source inspection. A normal publish by a user without cached githubCreatedAt calls requireGitHubAccountAge, and a GitHub 403 or 429 response throws the reported rate-limit error; I did not run a live production publish.

## Solution Assessment

Unclear. The narrowest maintainable fix depends on production token/quota state and logs, and any code mitigation must preserve the security-sensitive GitHub account-age verification.
