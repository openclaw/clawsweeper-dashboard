## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has the reported failure path: normal skill/API publish and manual package/plugin publish hit the GitHub account-age gate; users without cached `githubCreatedAt` trigger a GitHub API lookup; 403/429 responses throw the reported rate-limit error.

## What This Changes

The issue reports that publishing a skill fails with a GitHub API rate-limit error during upload/publish.

## Best Possible Solution

Keep the account-age gate, verify production `GITHUB_TOKEN` and 403/429 logs, then reduce avoidable publish-time GitHub lookups by persisting or backfilling `githubCreatedAt` for users missing it.

## Reproduction Assessment

Yes. For a user without cached `githubCreatedAt`, a normal publish path reaches `requireGitHubAccountAge`; when GitHub returns 403/429, existing tests assert the same rate-limit error that the publish handlers surface.

## Solution Assessment

Unclear because no patch is proposed. The narrow maintainable direction is operational verification first, then a targeted cache/backfill or error-handling mitigation rather than bypassing the account-age gate.
