## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has the reported publish failure path: normal skill publish reaches the GitHub account-age gate, uncached users trigger a GitHub API lookup, and 403/429 responses map to the exact rate-limit error; SummerSec’s Publish plugin comment is consistent with the package publish path using the same gate.

## What This Changes

The issue reports that uploading a new skill fails with a GitHub API rate-limit error, with a related comment reporting the same problem through the Publish plugin.

## Best Possible Solution

Keep the account-age gate, verify production GITHUB_TOKEN and GitHub 403/429 logs, then reduce avoidable first-publish GitHub lookups or improve retry/reset detail without weakening verification.

## Reproduction Assessment

Yes, from source inspection. Publish as a user without cached githubCreatedAt, have the GitHub account lookup return 403 or 429, and the publish path emits the reported rate-limit error.

## Solution Assessment

Unclear. The narrowest maintainable fix depends on production evidence because current main already supports GITHUB_TOKEN, while source mitigations touch the security-sensitive account-age gate.
