## Summary

Keep open. Current main still has the reported publish failure path, related publish/import rate-limit reports remain open, and the ClawHub cleanup policy does not auto-close issues.

## What This Changes

Publishing a skill fails with a GitHub API rate-limit error, and a commenter reports the same failure class through the Publish plugin.

## Best Possible Solution

Keep the account-age gate, verify production GITHUB_TOKEN and Convex 403/429 logs, then either correct deployment configuration or safely reduce avoidable first-publish GitHub lookups with caching/backfill and clearer retry detail.

## Reproduction Assessment

Yes, from source inspection. Publish as a user without cached githubCreatedAt, have the GitHub account lookup return 403 or 429, and the publish path emits the reported rate-limit error.

## Solution Assessment

Unclear. The narrowest fix depends on production evidence because current main already supports GITHUB_TOKEN, while source mitigations touch the security-sensitive account-age gate.
