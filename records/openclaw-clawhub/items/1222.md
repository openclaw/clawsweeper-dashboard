## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has a source-reproducible publish path where users without cached `githubCreatedAt` can hit the GitHub account-age lookup and receive the reported 403/429 rate-limit error; the Publish plugin comment is consistent with manual package publish sharing the same gate.

## What This Changes

Skill publishing/uploading fails with GitHub API rate-limit errors during the account-age lookup, including a reported Publish plugin/package publish path.

## Best Possible Solution

Keep the 14-day immutable-ID account-age gate, verify production 403/429 logs and `GITHUB_TOKEN` quota, then reduce avoidable publish-time GitHub lookups or safely backfill/cache `githubCreatedAt`.

## Reproduction Assessment

Yes, from source inspection and existing tests. Publish as a user without cached `githubCreatedAt`, have the GitHub account lookup return 403 or 429, and current main emits the reported rate-limit error.

## Solution Assessment

Unclear. The safest fix may be production `GITHUB_TOKEN`/quota remediation or a code change such as safer caching/backfill, and that choice needs maintainer operational and anti-abuse judgment.
