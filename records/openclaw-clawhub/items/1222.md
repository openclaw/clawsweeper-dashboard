## Summary

Keep open. Current main still contains the reported publish failure path, and ClawHub cleanup policy keeps issues open unless a PR is certainly implemented on main.

## What This Changes

Skill publish/upload fails for some users with a GitHub API rate-limit error during the GitHub account-age publish gate.

## Best Possible Solution

Keep the account-age gate, verify production GitHub token/quota state, then reduce avoidable publish-time GitHub API dependence by caching or backfilling `githubCreatedAt` for users missing it.

## Reproduction Assessment

Yes. Source inspection gives a concrete current-main path for users missing cached `githubCreatedAt`, and existing tests assert that 403/429 GitHub responses throw the same rate-limit error reached by skill and package publish.

## Solution Assessment

Unclear because no patch is proposed. The narrow maintainable direction is production config/log verification first, then a targeted cache/backfill mitigation rather than bypassing the account-age gate.
