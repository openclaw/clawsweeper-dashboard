Keep open. Current main still has a source-reproducible publish path where uncached GitHub account age triggers a GitHub API lookup and 403/429 responses surface the reported rate-limit error; ClawHub cleanup policy also keeps issues open for maintainer follow-up.

## What This Changes

The issue reports that publishing a skill fails with a GitHub API rate-limit error during upload/publish.

## Best Possible Solution

Keep immutable-ID account-age verification, verify production `GITHUB_TOKEN` quota and Convex 403/429 logs, then reduce avoidable publish-time GitHub lookups through safe caching/backfill or clearer retry/reset handling without weakening the anti-abuse gate.

Yes, by source inspection. A normal publish by a user without cached `githubCreatedAt` calls `requireGitHubAccountAge`, and GitHub 403/429 responses throw the reported rate-limit error; I did not run a live production publish.

## Solution Assessment

Unclear. The narrowest maintainable fix depends on production token/quota state and logs, and any code mitigation must preserve the security-sensitive account-age verification rather than bypassing it.
