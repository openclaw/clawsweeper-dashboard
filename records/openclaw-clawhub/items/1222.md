## Summary

Keep open. Current main still contains the reported publish-time failure path, and the repository cleanup policy for openclaw/clawhub keeps issues open rather than auto-closing them. The issue discussion and the Publish plugin comment match the same GitHub account-age lookup path returning the exact rate-limit message.

## What This Changes

Publishing a skill can fail with a GitHub API rate-limit error when ClawHub checks the publisher's GitHub account age.

## Best Possible Solution

Keep the account-age anti-abuse gate, but remove avoidable publish-time dependence on exhausted GitHub API quota by verifying production GITHUB_TOKEN/403/429 logs and then persisting or backfilling githubCreatedAt for users who are missing it, with clearer operator/user guidance when quota is exhausted.
