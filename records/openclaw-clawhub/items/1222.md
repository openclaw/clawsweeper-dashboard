## Summary

Keep open. Current main still contains the reported publish-time failure path: skill publish uses the GitHub account-age gate, missing cached account age triggers a GitHub API lookup, and 403/429 responses return the exact rate-limit message; the Publish plugin comment matches the package publish path using the same gate.

## What This Changes

Fix publish/upload so skill and plugin publishing do not fail with a GitHub API rate-limit error during GitHub account-age lookup.

## Best Possible Solution

Keep the account-age gate, verify production GITHUB_TOKEN/quota state, then reduce publish-time dependence on GitHub API quota by persisting or backfilling githubCreatedAt for users missing it.

## Reproduction Assessment

Yes. A user without cached githubCreatedAt will trigger requireGitHubAccountAge to call GitHub, and the existing tests assert that 403/429 responses throw the same rate-limit error reached by skill and package publish paths.

## Solution Assessment

Unclear because no patch is proposed. The maintainable direction is not to bypass the gate, but to verify production configuration first and then add a targeted cache/backfill or error-handling mitigation if needed.
