The issue reports that publishing or uploading a skill fails with `GitHub API rate limit exceeded` during the ClawHub publish flow.

## Best Possible Solution

Keep the account-age gate, verify production GitHub token/quota and Convex 403/429 logs, then reduce avoidable publish-time GitHub lookups through safe caching, backfill, or clearer retry/reset handling without bypassing immutable-ID verification.
