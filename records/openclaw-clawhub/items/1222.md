## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has the reported source path: ordinary skill publish can call the GitHub account-age gate, users missing cached GitHub creation time trigger a GitHub API lookup, and GitHub 403/429 responses produce the rate-limit error shown in this issue. The related Publish plugin comment is consistent with the package publish path sharing the same gate.

## What This Changes

Publishing a skill should not fail persistently with a GitHub API rate-limit error during the upload/publish flow.

## Best Possible Solution

Keep the account-age gate, verify production GITHUB_TOKEN and 403/429 logs first, then reduce avoidable publish-time GitHub lookups with safe caching/backfill or clearer retry guidance if maintainers choose a code change.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: publish as a user without cached githubCreatedAt, have the GitHub account lookup return 403 or 429, and the publish path emits the reported rate-limit error.

## Solution Assessment

Unclear. The narrowest fix depends on production evidence: the existing GITHUB_TOKEN configuration may address a quota problem, while a code mitigation needs maintainer approval because it touches the anti-abuse account-age gate.
