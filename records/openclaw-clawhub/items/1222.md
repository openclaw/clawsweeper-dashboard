## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has a source-reproducible publish failure path: normal skill publish reaches the GitHub account-age lookup, users without cached githubCreatedAt trigger a GitHub API call, and 403/429 responses throw the reported rate-limit error. SummerSec's Publish plugin comment is also consistent with manual package publish using the same gate.

## What This Changes

The issue reports skill publishing failing with a GitHub API rate-limit error during upload/publish.

## Best Possible Solution

Keep the account-age gate, verify production GITHUB_TOKEN and GitHub 403/429 logs, then reduce avoidable first-publish GitHub lookups or improve retry/reset detail without weakening verification.

## Reproduction Assessment

Yes, from source inspection. Publish as a user without cached githubCreatedAt, have the GitHub account lookup return 403 or 429, and current main emits the reported rate-limit error.

## Solution Assessment

Unclear. Existing GITHUB_TOKEN support may be the operational fix, but source mitigations need maintainer judgment because they touch the security-sensitive account-age gate.
