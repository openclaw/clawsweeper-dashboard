## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has the reported publish failure path: uncached users publishing a skill hit the GitHub account-age lookup, and GitHub 403/429 responses still produce the reported rate-limit error; the Publish plugin comment is consistent with package publish using the same gate.

## What This Changes

The issue requests a fix for skill upload/publish failures caused by GitHub API rate-limit errors during the publish account-age check.

## Best Possible Solution

Keep the account-age gate, verify production GITHUB_TOKEN and GitHub 403/429 logs, then reduce avoidable first-publish GitHub lookups or improve retry/reset detail without weakening verification.

## Reproduction Assessment

Yes, from source inspection. Publish as a user without cached githubCreatedAt, have the GitHub account lookup return 403 or 429, and the publish path emits the reported rate-limit error.

## Solution Assessment

Unclear. The existing GITHUB_TOKEN mitigation may be the correct operational fix, but a code change would touch the security-sensitive account-age gate and should be chosen with production evidence.
