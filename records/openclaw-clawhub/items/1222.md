## Summary

Keep open. Current main still has a source-reproducible path where normal skill publish reaches the GitHub account-age lookup, uncached users trigger a GitHub API request, and GitHub 403/429 responses produce the reported rate-limit error; repository cleanup policy also keeps ClawHub issues open.

## What This Changes

Skill publish fails when the GitHub account-age lookup hits GitHub API rate limiting during upload or publish.

## Best Possible Solution

Verify production Convex GITHUB_TOKEN/quota and 403/429 logs, keep the immutable-ID account-age gate, then correct deployment config or safely reduce first-publish GitHub lookups through githubCreatedAt caching/backfill.

Yes. Source inspection gives a high-confidence path: publish as a user without cached githubCreatedAt and have the GitHub account lookup return 403 or 429; current main emits the reported error.
