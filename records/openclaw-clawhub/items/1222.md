## Summary

Keep open. Current main still has a source-reproducible publish failure path where users without cached GitHub account age hit a GitHub API lookup during publish, and 403/429 responses surface the reported rate-limit error; ClawHub cleanup policy keeps issues open.

## What This Changes

The issue reports skill upload/publish failing with a GitHub API rate-limit error, with a related comment that the Publish plugin sees the same failure class.

## Best Possible Solution

Keep the GitHub account-age control, but make normal publish avoid unnecessary unauthenticated GitHub API dependence through verified production token configuration plus cached or backfilled `githubCreatedAt` data for affected users.

## Reproduction Assessment

Yes. Source inspection gives a concrete current-main path for users missing cached `githubCreatedAt`, and existing tests assert that 403/429 GitHub responses throw the reported rate-limit error.

## Solution Assessment

Unclear because no patch is proposed. The narrow maintainable direction is production `GITHUB_TOKEN` and 403/429 log verification first, then a targeted cache or backfill mitigation that preserves the anti-abuse gate.
