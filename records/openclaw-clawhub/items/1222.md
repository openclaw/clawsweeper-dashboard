## Summary

Keep open. This is an open ClawHub issue, so cleanup policy does not auto-close it, and current main still has a source-reproducible skill publish path where a user missing cached GitHub account age can hit GitHub 403/429 and receive the reported rate-limit error; the Publish plugin comment matches the package publish gate using the same account-age check.

## What This Changes

Skill publishing should not fail during upload with a GitHub API rate-limit error.

## Best Possible Solution

Verify production GITHUB_TOKEN/quota and Convex 403/429 logs, then reduce avoidable first-publish GitHub lookups or improve retry/reset guidance while preserving immutable-ID account-age enforcement.

## Reproduction Assessment

Yes, by source inspection. On current main, ordinary skill publish by a user without cached githubCreatedAt calls requireGitHubAccountAge; if the GitHub lookup returns 403 or 429, current code throws the reported rate-limit error.

## Solution Assessment

Unclear. The narrowest maintainable fix depends on production token/quota state and logs, and any code mitigation must preserve the security-sensitive GitHub account-age verification.
