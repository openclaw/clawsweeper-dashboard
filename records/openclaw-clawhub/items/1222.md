## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current `main` still has the reported publish-time failure path: normal skill publish can call GitHub for missing cached account age and returns the exact rate-limit message on 403/429; the Publish plugin comment is consistent with package publish using the same gate.

## What This Changes

Fix skill publishing failures where upload/publish returns a GitHub API rate-limit error.

## Best Possible Solution

Keep the 14-day account-age gate, verify production token/quota state, then reduce publish-time dependence on live GitHub lookups by persisting or backfilling `githubCreatedAt` for users missing it.

## Reproduction Assessment

Yes. A user without cached `githubCreatedAt` who publishes while GitHub returns 403/429 will follow the tested `requireGitHubAccountAge` path and receive the reported rate-limit error.

## Solution Assessment

Unclear because no patch is proposed. The maintainable direction is to avoid bypassing the gate: verify production `GITHUB_TOKEN`/quota first, then add a targeted cache, backfill, or sign-in-time persistence mitigation if needed.
