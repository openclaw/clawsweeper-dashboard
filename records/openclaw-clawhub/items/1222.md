## Summary

Keep open. Current main still contains the reported publish failure path: skill publish reaches the GitHub account-age gate, users missing cached `githubCreatedAt` trigger a GitHub API lookup, and 403/429 responses throw the reported rate-limit error. SummerSec's Publish plugin comment is consistent with package publish sharing the same gate, and ClawHub cleanup policy keeps issues open.

## What This Changes

Stop skill publishing from failing with GitHub API rate-limit errors during the publish account-age check.

## Best Possible Solution

Keep the account-age gate, verify production `GITHUB_TOKEN` and 403/429 logs, then reduce avoidable publish-time GitHub API dependence by caching or backfilling `githubCreatedAt` for users missing it.

## Reproduction Assessment

Yes. Source inspection gives a concrete current-main path for users missing cached `githubCreatedAt`, and existing tests assert that 403/429 GitHub responses throw the same rate-limit error reached by skill and package publish.

## Solution Assessment

Unclear because no patch is proposed. The narrow maintainable direction is production config/log verification first, then a targeted cache or backfill mitigation that preserves the anti-abuse gate.
