## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still contains the reported publish failure path: skill publish calls the GitHub account-age gate, missing cached account age triggers a GitHub API lookup, and 403/429 responses throw the exact rate-limit message; the Publish plugin comment is consistent with manual package publish using the same gate.

## What This Changes

Publishing a skill fails with a GitHub API rate-limit error, with a related commenter reporting the same failure class through the Publish plugin.

## Best Possible Solution

Keep the account-age gate, verify production `GITHUB_TOKEN` and 403/429 logs, then reduce publish-time GitHub API dependence by persisting or backfilling `githubCreatedAt` for users missing it.

## Reproduction Assessment

Yes. Source inspection gives a concrete path for users without cached `githubCreatedAt`, and existing tests assert that 403/429 GitHub responses throw the same rate-limit error reached by skill and package publish.

## Solution Assessment

Unclear because no patch is proposed. The maintainable direction is ops verification first, followed by a targeted cache/backfill or error-handling mitigation rather than bypassing the account-age gate.
