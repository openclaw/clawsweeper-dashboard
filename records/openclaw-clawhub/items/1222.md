## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has a source-reproducible publish failure path: normal skill publish reaches the GitHub account-age lookup, uncached users can receive the exact 403/429 rate-limit error, and the Publish plugin comment matches manual package publish using the same gate.

## What This Changes

The issue reports that uploading or publishing a new skill fails with a GitHub API rate-limit error, with a second user seeing the same class of failure through the Publish plugin.

## Best Possible Solution

Keep the account-age gate, verify production GITHUB_TOKEN and GitHub 403/429 logs, then reduce avoidable first-publish GitHub lookups or improve retry/reset detail without weakening verification.

## Reproduction Assessment

Yes, from source inspection. Publish as a user without cached githubCreatedAt, have the GitHub account lookup return 403 or 429, and current main emits the reported rate-limit error.

## Solution Assessment

Unclear. Existing GITHUB_TOKEN support may be the operational fix, but source mitigations need maintainer judgment because they touch the security-sensitive account-age gate.
