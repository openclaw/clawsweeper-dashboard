## Summary

Keep open. Current main already fixes the original anchor-href bug, but this draft branch still broadens README rewriting to image URLs that point at a text-oriented file endpoint, and it is missing the required changelog entry for a user-facing fix.

## What This Changes

The PR replaces href-only README link handling with a markdown AST rewrite for links, images, and definitions, plus focused regression tests and one lint cleanup.

## Best Possible Solution

Land a focused README document-link rewrite now, and only rewrite image sources after ClawHub has an explicit image/asset-serving contract with matching docs and tests.

## Reproduction Assessment

Yes. The PR diff itself reproduces the behavior: `![Logo](img/logo.png)` is rewritten to `/api/v1/skills/.../file?...`, while current main's target endpoint reads and returns text-oriented file responses.

## Solution Assessment

No. Rewriting markdown links and definitions is a reasonable direction, but routing image `src` values to the existing raw text file API is not the narrowest maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Avoid routing README images to the text file endpoint:** `src/lib/skillReadmeLinks.ts:21`
  - body: The plugin treats `image` as a URL node, so relative images become `<img src="/api/v1/skills/.../file?...">`. Current main's file API reads blobs with `blob.text()` and returns `safeTextFileResponse`; skill publishing rejects non-text files and SVG responses are locked down with attachment/CSP, so this is not a general asset-serving path. Leave image URLs alone or add a binary-safe asset endpoint before rewriting image `src` values.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `src/components/SkillDetailTabs.tsx:106`
  - body: This user-facing fix changes README navigation, but the PR diff does not update `CHANGELOG.md`'s Unreleased Fixes section. Repo policy requires a changelog entry for user-facing `fix` changes before merge.
  - confidence: 0.92
