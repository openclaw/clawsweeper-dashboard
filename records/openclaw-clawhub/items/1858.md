## Summary

Keep open. Current main has the href-only part of the README link fix, but this draft PR still proposes broader markdown rewriting and has a functional blocker around relative images before it should merge.

## What This Changes

The branch replaces the href-only README link helper with a remark plugin for link, image, and definition URLs, updates regression tests, and includes a small Set-iteration lint cleanup.

## Best Possible Solution

Land this PR or an equivalent fix after it either handles relative README images through a binary-safe response path or limits the rewrite to links and definitions, with tests and a changelog entry.

## Reproduction Assessment

Yes. Static inspection is enough here: the PR rewrites `![Logo](img/logo.png)` to the skill file API, while that endpoint reads blobs as text, so binary images cannot be served faithfully.

## Solution Assessment

No. The markdown-aware link/definition rewrite is a good direction, but routing image nodes through the current text-file API is not the narrowest maintainable solution.
