## Summary

Keep open. Current main includes a narrower href-only README link fix, but this draft PR still contains useful unmerged work for markdown images and definition URLs, and I found no discrete blocking review finding in the proposed patch.

## What This Changes

The PR replaces the current href-only README link helper with a remark plugin that rewrites skill README links, images, and definitions to safe skill file API URLs, plus focused regression tests.

## Best Possible Solution

Land this PR or an equivalent focused change after maintainer review if ClawHub should support relative README images and definitions in addition to ordinary links.

## Reproduction Assessment

Yes. A current-main README image such as `![Logo](img/logo.png)` follows the non-href path in `SkillDetailTabs`, and `rehypeProxyImages` leaves relative image paths unchanged, so the remaining behavior can be reproduced from the code path without modifying the checkout.

## Solution Assessment

Yes, with one maintainer policy check. A remark-level rewrite is the narrow maintainable approach for markdown links, images, and definitions, but the root-relative URL behavior should be intentionally accepted.
