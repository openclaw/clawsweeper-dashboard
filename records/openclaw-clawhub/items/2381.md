## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Apply search before min-skill filtering:** `convex/publisherAbuse.ts:175-179`
  - body: When `minSkillCount` is set, this early return skips the `search` predicate entirely, so admins searching within the queue get all nominations above the skill floor and the total count no longer matches the query. Apply the search check regardless of the skill filter, then do the score-based min-skill filter in the later loop.
  - confidence: 0.95
- **[P2] Refresh nominations when scores return to pass:** `convex/publisherAbuse.ts:377-384`
  - body: Because nominations are only upserted for non-`pass` labels, a publisher that was previously nominated but later scores `pass` keeps its old `latestScoreId`, label, and `lastScoredAt`. The management queue can then keep showing stale false positives instead of the latest dry-run state.
  - confidence: 0.9
- **[P2] Remove or use unused scoring exports:** `convex/lib/publisherAbuseScoring.ts:1-5`
  - body: `ci:static` is failing in `deadcode:exports` for unused exports from this new module, including `PUBLISHER_ABUSE_MODEL_VERSION`, `computePublisherAbusePressure`, `comparePublisherAbuseScores`, and exported types. Keep helpers/types internal unless another production or test entry point actually imports them.
  - confidence: 0.95
- **[P3] Stop failed score runs from continuing:** `convex/publisherAbuse.ts:512-513`
  - body: `requireRunningRun` returns non-running runs instead of rejecting them, so a `failed` run whose phase is still `collecting` or `finalizing` can be resumed and patched by the page handlers. Throw for non-running status or split the helper so callers do not treat failed runs as active.
  - confidence: 0.82
