## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep search active with min-skill filters:** `convex/publisherAbuse.ts:175-179`
  - body: When `minSkillCount > 0`, this branch returns `true` before checking `search`, so an admin combining a handle search with a minimum skill count gets every fetched nomination and an inflated `total`. Apply the search predicate regardless of the min-skill filter, then keep the score-based skill-count check in the later loop.
  - confidence: 0.94
- **[P2] Update nominations when a score returns to pass:** `convex/publisherAbuse.ts:377-383`
  - body: Because nominations are only upserted for non-pass labels, an existing nomination keeps its old `latestScoreId`, label, and `lastScoredAt` after a later run scores the publisher as `pass`. Refresh the nomination with the latest score or resolve it when the label becomes `pass`, otherwise admins will triage stale pressure data.
  - confidence: 0.9
- **[P3] Stop processing non-running score runs:** `convex/publisherAbuse.ts:512`
  - body: `requireRunningRun` returns failed or completed runs instead of rejecting them, so a failed run whose phase is still `collecting` or `finalizing` can continue to append score data. Throw for non-running runs or rename the helper and make callers explicitly handle failed/completed states.
  - confidence: 0.76
