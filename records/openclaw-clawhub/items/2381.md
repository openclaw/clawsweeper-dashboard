Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Apply search even when min-skill filtering is active:** `convex/publisherAbuse.ts:175-179`
  - body: When `minSkillCount > 0`, this callback returns before checking `search`, so the management queue ignores the search box and reports a misleading `total`. Apply the search predicate independently, then do the score-based min-skill filter after fetching the score row.
  - confidence: 0.95
- **[P2] Refresh nominations when a publisher returns to pass:** `convex/publisherAbuse.ts:377-384`
  - body: Existing nominations are updated only for non-pass labels, so a publisher that later scores `pass` keeps the old `latestScoreId`, label, and `lastScoredAt` in the queue. Update nominations from the latest score, or make the queue resolve the latest `(ownerKey, modelVersion)` score instead of relying on stale nomination state.
  - confidence: 0.9
- **[P2] Reject non-running runs before collecting or finalizing:** `convex/publisherAbuse.ts:512-513`
  - body: `requireRunningRun` returns completed or failed runs instead of enforcing `status === "running"`, so a stale scheduled run can still flow into collect/finalize when the phase matches. Throw for non-running runs or rename the helper and add explicit caller guards.
  - confidence: 0.85
- **[P2] Fix the dead-code export failure:** `convex/lib/publisherAbuseScoring.ts:1`
  - body: The static CI gate fails in `deadcode:exports` because several scoring constants/functions/types are exported but unused outside this module. Remove unnecessary exports or add the intended supported call sites so `bun run ci:static` can pass.
  - confidence: 0.95
