- **[P2] Keep search active with min-skill filters:** `convex/publisherAbuse.ts:175-179`
  - body: When minSkillCount is positive, this branch returns before applying the search predicate, so an admin combining a handle search with a minimum skill count gets every fetched nomination and an inflated total.   - confidence: 0.95
- **[P2] Refresh nominations when a score returns to pass:** `convex/publisherAbuse.ts:377-383`
  - body: Because nominations are only upserted for non-pass labels, an existing nomination keeps its old latestScoreId, label, and lastScoredAt after a later run scores that publisher as pass. Refresh or resolve the existing nomination so admins do not triage stale pressure data.
  - confidence: 0.92

  - body: A scheduled continuation with runId always starts from phase collecting instead of the persisted run phase. If collection has already advanced to finalizing and maxPages is 1, the action spends its only page on a no-op collect result and reschedules forever without finalizing.
  - confidence: 0.87
