## What This Changes

The PR adds production HTTP smoke to the deploy workflow, expands production UI smoke to publish/import coverage, adds 429 retry handling and skill-detail memoization to prod HTTP smoke, and updates the upload signed-out assertion for the publish redirect.

## Best Possible Solution

Land a corrected version that wires the existing prod HTTP smoke script into deploy CI, expands production UI smoke to publish/import workflows, aligns signed-out assertions or route copy with the rendered UI, keeps bounded 429 retry behavior and skill-detail memoization, and removes unreachable retry fallback code.
