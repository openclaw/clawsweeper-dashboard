## Summary

Keep open for maintainer review: current main does not implement the PR's intended nonsuspicious index routing for lexical fallback, and the branch also contains a broader fixed-200 client-side search pagination change that can truncate broad searches.

## What This Changes

The PR routes lexical fallback search through a nonsuspicious index when requested, adds related tests, and changes /skills search pagination to fetch a fixed 200 results for client-side paging.

## Best Possible Solution

Land a narrow update against current main's skillSearchDigest fallback that selects by_nonsuspicious_updated and by_nonsuspicious_created with the active+nonsuspicious prefix when requested, preserves default search semantics, and handles the fixed-200 pagination idea as a separate product/performance decision.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Remove the fixed search result cap:** `src/routes/skills/-useSkillsBrowseModel.ts:111`
  - body: Hard-coding search requests to `limit: 200` means broad searches can never load results beyond the first 200 server matches. The current load-more flow increases the server limit and re-runs search, so replacing it with cached client-side paging silently truncates valid results once those 200 items are exhausted.
