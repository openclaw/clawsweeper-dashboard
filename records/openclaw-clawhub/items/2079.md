## Summary

Keep open. The PR is not implemented on current main, and latest head still has a blocking correctness issue where the hidden-suspicious vector probe can report results that the unfiltered search would not render; it also still needs the required changelog entry.

## What This Changes

The PR refactors skill search into a shared helper, adds `searchSkillsWithMeta` hidden-suspicious metadata, surfaces a disclosure in unified search and header typeahead, and adds regression tests plus generated Convex API typing.

## Best Possible Solution

Keep the disclosure approach, but make the metadata probe follow the same eligibility as rendered search results and record the user-facing fix in the changelog before merge.

## Reproduction Assessment

Yes. The linked issue gives concrete live steps with `国泰海通`, and current-main source shows unified search filtering suspicious skills while `/skills` defaults to including them.

## Solution Assessment

No. The overall disclosure design is maintainable, but the vector-pool hidden probe still needs to mirror the main search token gate before this is the narrowest correct fix.
