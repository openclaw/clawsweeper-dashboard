## Summary

Keep open: this maintainer-authored PR is not implemented on current main, and the latest head still hides the disclosure when all matching skill results are filtered as suspicious.

## What This Changes

The PR adds a Convex `searchSkillsWithMeta` action and switches unified search, `/search`, and header typeahead to show a hidden-suspicious-results disclosure linking to `/skills`.

## Best Possible Solution

Keep the safer unified-search default, but render the hidden-results disclosure in every `/search` state where the hidden flag is true and add focused regression coverage for the empty-results case.

## Reproduction Assessment

Yes. Source inspection shows the current-main filter mismatch, and the PR-specific regression is reproducible from the render branches by returning `results: []` with `hasHiddenSuspicious: true`.

## Solution Assessment

No, not yet. The disclosure direction is reasonable, but the notice must be outside the non-empty results branch and the typeahead hint should keep keyboard order aligned with visual order.
