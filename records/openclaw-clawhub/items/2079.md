## Summary

Keep open: current main still has the reported search-filter mismatch, and the latest PR head still has a source-reproducible false-positive path in the vector hidden-result probe before this can merge.

## What This Changes

The branch refactors skill search into a shared helper, adds `searchSkillsWithMeta` hidden-suspicious metadata, surfaces notices in unified search/typeahead, and adds regression tests plus generated Convex API typing.

## Best Possible Solution

Keep the disclosure approach, but make the hidden-suspicious flag follow the same vector token eligibility as rendered search results and add the required changelog entry before merge.

## Reproduction Assessment

Yes. Current main source shows unified search using `nonSuspiciousOnly: true` while `/skills` defaults to including suspicious skills, and PR head source shows the vector hidden probe missing the render-path token gate.

## Solution Assessment

No. The overall disclosure design is a maintainable fix, but the vector probe can still report hidden results that would not appear in an unfiltered search, so it needs a narrower eligibility check first.
