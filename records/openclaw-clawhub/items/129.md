## Summary

Keep open. Repository policy for this cleanup pass keeps ClawHub issues open, and the security concern remains valid: current main has filtering and pruning for the GitHub mirror, but the removal path is still periodic/batch-oriented rather than deterministically tied to delete, moderation-hide, scanner-malicious, report auto-hide, or ban transitions. Prior ClawSweeper comments already reproduced that ClawHub did not serve the example skill while the public mirror still served it, so this needs maintainer/security follow-up rather than closure.

## What This Changes

The issue asks ClawHub to prevent removed or malicious skills from persisting in the public openclaw/skills GitHub mirror and downstream marketplaces after they are removed from ClawHub.

## Best Possible Solution

Keep this open until ClawHub has a deterministic mirror-removal or tombstone policy triggered by skill deletion, hidden/removed moderation, scanner-malicious blocking, report auto-hide, and user bans, with focused tests and a production cleanup plan for known stale malicious mirror entries such as bird-js.
