## Summary

Keep open. Current main intentionally rejects reauthentication for deleted or deactivated GitHub-linked accounts, and this request asks for account restoration or identity unlinking, which remains an unresolved security/privacy policy and operator action rather than a cleanup-close candidate.

## What This Changes

The issue asks maintainers to restore a deleted ClawHub account or remove its GitHub OAuth binding so the reporter can sign up or log in again.

## Best Possible Solution

Define and implement a maintainer-approved deleted-account recovery or no-recovery policy, with private identity verification, audit logging, ban-safe checks, docs, and tests for GitHub OAuth rebinding or refusal.

## Reproduction Assessment

Yes for the general lockout path by source inspection: account deletion writes deactivatedAt/purgedAt and GitHub OAuth reauthentication calls handleDeletedUserSignIn, which rejects deleted or deactivated users. I did not verify the reporter's production account row or identity ownership.

## Solution Assessment

No for ad hoc public restore or unlink. The maintainable solution is an explicit private recovery/unlink policy with ownership verification and audit logging, or a clear product decision that deleted accounts cannot be recovered.
