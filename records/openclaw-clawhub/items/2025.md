## Summary

Keep open: current main intentionally blocks GitHub reauthentication after permanent account deletion, and ClawHub cleanup policy keeps issues open. This remains an account recovery and GitHub identity-binding policy request in an active open cluster.

## What This Changes

The reporter asks ClawHub to restore a deleted GitHub-linked account or remove the GitHub OAuth binding so they can sign up again.

## Best Possible Solution

Maintain a single maintainer-owned recovery policy for this cluster: either keep irreversible deletion with clearer support guidance, or implement an audited same-provider recovery/GitHub-unlink flow with ban safeguards and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Settings calls users.deleteAccount, which writes deactivatedAt/purgedAt, and GitHub OAuth completion calls handleDeletedUserSignIn, which rejects the deactivated user; I did not inspect the reporter's production record.

## Solution Assessment

No for ad hoc restoration or GitHub unlinking from an issue comment. The maintainable path needs maintainer-approved recovery policy, identity verification, audit logging, and tests, or an explicit decision that deletion remains irreversible.
