## Summary

Keep open: this is an open ClawHub issue, and the repository cleanup policy only permits auto-closing PRs that are already implemented on main; current main also intentionally blocks deleted-account GitHub reauthentication while recovery or OAuth-binding removal remains a support, security, and product-policy decision.

## What This Changes

The issue asks to restore a deleted account or remove its GitHub OAuth binding so the same GitHub account can sign up again.

## Best Possible Solution

Finalize the account-recovery policy, then implement an audited restore, unlink, or explicit irreversible-support path that preserves bans, provider identity ownership, purged-data boundaries, docs, and tests.

## Reproduction Assessment

Yes, by source inspection: settings calls users.deleteAccount, that mutation sets deactivatedAt/purgedAt, and the OAuth callback rejects deactivated users. I did not run a live GitHub OAuth flow or inspect the reporter's production record.

## Solution Assessment

No for an ad hoc restore or binding deletion. The maintainable path needs a policy-backed, audited recovery or unlink flow, or an explicit decision to keep deletion irreversible with clearer support guidance.
