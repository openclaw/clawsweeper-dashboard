## Summary

Keep open: current main intentionally blocks GitHub sign-in after user-initiated account deletion, and resolving the reported lockout requires maintainer/operator identity, product, and security policy handling rather than conservative cleanup.

## What This Changes

The issue requests account restoration or removal of a GitHub OAuth binding so the reporter can sign up again after deleting their ClawHub account.

## Best Possible Solution

Define and document a maintainer-approved account recovery policy, then either add an audited admin/support restore or GitHub-unlink flow with tests, or keep deletion terminal with clear support guidance.

## Reproduction Assessment

Yes, by source inspection: settings calls users.deleteAccount, that mutation sets deactivatedAt/purgedAt, and the next GitHub OAuth completion calls handleDeletedUserSignIn and rejects the matched user. I did not run a live OAuth flow or inspect the reporter's production records.

## Solution Assessment

No for a direct autonomous restore or binding deletion. The maintainable fix needs a policy-backed, audited recovery or unlink path, or an explicit decision to keep deletion irreversible with clearer support handling.
