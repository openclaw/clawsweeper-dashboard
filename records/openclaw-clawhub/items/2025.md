Keep open: current main intentionally blocks sign-in after user-initiated deletion, and resolving this reporter's GitHub OAuth lockout requires maintainer/operator identity verification plus a security and product policy decision.

## What This Changes

The issue requests account restoration or removal of a GitHub OAuth binding after a deleted account can no longer log in or re-register.

## Best Possible Solution

Define an audited account recovery or provider-unlink policy, then implement a guarded support/admin flow with tests or explicitly keep deletion irreversible with clearer support guidance.

Yes, by source inspection: settings calls users.deleteAccount, that mutation sets deactivatedAt/purgedAt, and the OAuth callback rejects deactivated users. I did not run a live OAuth flow or inspect the reporter's production record.
