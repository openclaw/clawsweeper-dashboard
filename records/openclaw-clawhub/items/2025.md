## Summary

Keep open: ClawHub cleanup policy does not auto-close issues, and current main still intentionally blocks deleted-account GitHub reauthentication without a restore or GitHub-binding unlink path. This remains an account recovery and identity-binding policy item in an active open cluster.

## What This Changes

The issue asks for account restoration or GitHub OAuth binding removal after a deleted ClawHub account can no longer re-register or sign in.

## Best Possible Solution

Define a maintainer-owned deleted-account recovery policy, then either implement an audited same-provider restore or GitHub-unlink flow with tests, or document and enforce that deletion remains irreversible.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: settings calls users.deleteAccount, which writes deactivatedAt/purgedAt, and GitHub OAuth completion rejects through handleDeletedUserSignIn. I did not verify the reporter's production account record.

## Solution Assessment

No for ad hoc restoration or GitHub unlinking from an issue comment. The maintainable path needs maintainer-approved identity recovery policy, ownership verification, audit logging, and regression coverage, or an explicit decision to keep deletion permanent.
