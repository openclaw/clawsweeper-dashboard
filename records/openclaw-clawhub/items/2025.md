## Summary

Keep open: current main intentionally blocks GitHub sign-in after user-initiated deletion, and this account recovery/unlink request needs maintainer/operator identity verification plus a security/product policy decision.

## What This Changes

The issue requests recovery or removal of a GitHub OAuth binding after account deletion prevents signing back in or registering again.

## Best Possible Solution

Define an audited account recovery or GitHub identity-unlink policy, then either add a guarded support/admin flow with tests or explicitly keep deletion terminal with clearer support guidance.

## Reproduction Assessment

Yes, by source inspection: settings calls users.deleteAccount, that mutation sets deactivatedAt/purgedAt, and OAuth completion rejects users with deactivatedAt. I did not run a live OAuth flow or inspect the reporter's production record.

## Solution Assessment

No for an autonomous restore or binding deletion. The maintainable path needs policy-backed identity verification and audited recovery/unlink handling, or an explicit decision to keep deletion irreversible with support guidance.
