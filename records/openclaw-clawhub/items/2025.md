## Summary

Keep open: ClawHub cleanup policy keeps issues open, and current main intentionally blocks GitHub reauthentication for permanently deleted accounts, so this remains a live account-recovery and identity-binding policy request rather than a cleanup close.

## What This Changes

The issue asks maintainers to restore a deleted GitHub-linked account or remove its GitHub OAuth binding so the user can sign up again.

## Best Possible Solution

Decide and document the deleted-account recovery policy, then either keep deletion irreversible with clearer support guidance or implement an audited admin/support restore or GitHub unlink flow with same-provider verification, ban safeguards, audit logs, and tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: settings invokes users.deleteAccount, that mutation writes deactivatedAt/purgedAt, and GitHub OAuth completion calls handleDeletedUserSignIn, which rejects the deactivated user; I did not inspect the reporter's production record.

## Solution Assessment

No for ad hoc restoration or direct GitHub unlinking from an issue comment. The maintainable path needs maintainer-approved recovery policy, identity verification, audit logging, and tests, or an explicit decision that deletion remains irreversible.
