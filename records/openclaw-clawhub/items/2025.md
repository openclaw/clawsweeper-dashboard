## Summary

Keep open. Current main intentionally treats user-initiated account deletion as irreversible and blocks GitHub reauthentication, while this request asks for restoration or GitHub identity unlinking; that needs maintainer/operator product and security judgment, not conservative cleanup closure.

## What This Changes

The issue requests restoring a deleted ClawHub account or removing its GitHub OAuth binding so the same GitHub account can sign up again.

## Best Possible Solution

Define a maintainer-approved audited support/admin recovery path for deleted GitHub-linked accounts, or explicitly keep deletion irreversible while providing clear support guidance for affected users.

## Reproduction Assessment

Yes for the general account-lock behavior by source inspection: deleteAccount writes deactivatedAt/purgedAt and OAuth completion rejects deactivated or legacy deleted users. No for this reporter's specific production row without maintainer/operator access.

## Solution Assessment

No for ad hoc public-issue restoration or GitHub unlinking. The maintainable solution is an audited account recovery/unlink policy, or a clear product decision that deleted accounts cannot be recovered.
