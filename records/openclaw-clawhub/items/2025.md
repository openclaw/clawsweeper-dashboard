## Summary

Keep open. Current main intentionally blocks GitHub reauthentication for deactivated/purged accounts, and the requested restore or GitHub-binding removal is security-sensitive support work that needs maintainer/operator policy approval rather than automated cleanup.

## What This Changes

The issue asks ClawHub to restore a deleted account or remove its GitHub OAuth binding so the user can sign up again.

## Best Possible Solution

Define a maintainer/security-approved support policy for deleted GitHub-linked accounts, either an audited private recovery/unlink path or a documented no-recovery decision that preserves purge and ban boundaries.

## Reproduction Assessment

Yes for the general lockout path: source and tests show deletion writes deactivatedAt/purgedAt and GitHub OAuth rejects those users. This review did not verify the reporter's specific production row or identity ownership.

## Solution Assessment

No for ad hoc restoration or unlinking from a public issue. The maintainable path is an audited maintainer/operator recovery policy, or a documented product decision that deleted accounts cannot be recovered.
