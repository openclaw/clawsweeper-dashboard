## Summary

Keep open: the report matches current main's intentionally irreversible account-deletion path, but resolving the lockout requires maintainer support access, identity verification, and a security/product policy decision rather than an autonomous repo-only fix.

## What This Changes

The issue asks maintainers to restore a deleted GitHub-linked account or remove its existing GitHub OAuth binding so the reporter can sign up again.

## Best Possible Solution

Define a maintainer-approved account recovery policy, then either add an audited support/admin restore or GitHub-unlink flow, or explicitly keep deletion terminal with clearer support guidance.

## Reproduction Assessment

Yes, by source inspection: settings calls users.deleteAccount, which sets deactivatedAt/purgedAt, and OAuth completion calls handleDeletedUserSignIn, which rejects deactivated users. I did not run a live OAuth flow or inspect the reporter's production record.

## Solution Assessment

No for an autonomous direct restore or binding deletion. The maintainable path needs a policy-backed, audited recovery or unlink path, or an explicit decision to keep deletion irreversible with support guidance.
