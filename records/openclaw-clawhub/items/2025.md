## Summary

Keep open: this is an open ClawHub issue, and the repository cleanup policy only permits auto-closing PRs that are implemented on main. Current main still intentionally blocks deleted/deactivated GitHub OAuth sign-in, so resolving the reporter's request needs maintainer/operator policy and identity verification rather than cleanup closure.

## What This Changes

The issue asks maintainers to restore a deleted ClawHub account or remove its GitHub OAuth binding so the same GitHub identity can sign up again.

## Best Possible Solution

Define an audited account-recovery or GitHub identity-unlink policy, then either implement a guarded operator flow with tests or explicitly keep deletion irreversible with clearer support guidance.

## Reproduction Assessment

Yes, by source inspection: settings calls users.deleteAccount, that mutation sets deactivatedAt/purgedAt, and the OAuth callback rejects deactivated users. I did not run a live OAuth flow or inspect the reporter's production record.

## Solution Assessment

No for a direct autonomous restore or binding deletion. The maintainable path needs a policy-backed, audited recovery or unlink flow, or an explicit decision to keep deletion irreversible with support guidance.
