## Summary

Keep open: this is an open ClawHub issue, current main intentionally blocks deleted-account GitHub reauthentication, and restoration or OAuth unlinking needs maintainer/operator policy and identity handling.

## What This Changes

The reporter asks ClawHub to restore a deleted account or remove the GitHub OAuth binding so they can sign up again.

## Best Possible Solution

Decide the deleted-account recovery policy, then implement an audited admin/support restore or GitHub unlink flow with tests, or document and enforce that owner-initiated deletion remains permanent.

## Reproduction Assessment

Yes, source-reproducible: settings calls deleteAccount, the mutation writes deactivatedAt/purgedAt, and the OAuth callback rejects deactivated users. I did not verify the reporter's production account records or run a live GitHub OAuth flow.

## Solution Assessment

No for ad hoc restoration or binding deletion from an issue comment. The maintainable solution is a maintainer-approved recovery policy plus an audited operator/admin path, or an explicit permanent-deletion support response.
