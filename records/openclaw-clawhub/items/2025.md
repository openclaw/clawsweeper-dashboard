## Summary

Keep open. Current main intentionally blocks GitHub reauthentication for deactivated or purged accounts, while the requested restore or GitHub-binding removal is security-sensitive account recovery work that needs maintainer/operator policy approval.

## What This Changes

The issue asks ClawHub operators to restore a deleted GitHub-linked account or remove its GitHub OAuth binding so the same GitHub account can register again.

## Best Possible Solution

Define an audited support/admin recovery policy for deleted GitHub-linked accounts, or explicitly confirm irreversible deletion and route affected users through a private operator process that preserves privacy and ban boundaries.

## Reproduction Assessment

Yes for the general lockout path: source and tests show deletion writes deactivatedAt/purgedAt and GitHub OAuth rejects those users. This review did not verify the reporter's specific production row or identity ownership.

## Solution Assessment

No for ad hoc restoration or unlinking from a public issue. The maintainable path is an audited maintainer/operator recovery policy, or a documented product decision that deleted accounts cannot be recovered.
