## Summary

Keep open: current main intentionally blocks re-login after user-initiated account deletion, and changing that needs maintainer identity, production data, product, and security policy handling.

## What This Changes

The reporter asks maintainers to restore a deleted GitHub-linked account or remove its GitHub OAuth binding so they can sign up again.

## Best Possible Solution

Define and document a maintainer-approved account recovery policy, then either add an audited admin/support restore or GitHub-unlink flow with tests, or explicitly keep deletion terminal with clear support guidance.

## Reproduction Assessment

Yes, by source inspection: settings calls users.deleteAccount, that mutation sets deactivatedAt/purgedAt, and the next GitHub OAuth completion calls handleDeletedUserSignIn and rejects the matched user. I did not run a live OAuth flow or inspect the reporter's production records.

## Solution Assessment

No for a direct autonomous restore or binding deletion. The maintainable fix needs a policy-backed, audited recovery or unlink path, or an explicit decision to keep deletion irreversible with clearer support handling.
