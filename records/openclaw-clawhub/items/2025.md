## Summary

Keep open: current main intentionally treats user-initiated account deletion as irreversible and blocks GitHub reauth, so this needs maintainer support/product-policy handling rather than conservative cleanup.

## What This Changes

The issue asks for restoring a deleted ClawHub account or removing its GitHub OAuth binding so the reporter can sign up again.

## Best Possible Solution

Handle the specific account through maintainer support/admin tooling, then deliberately update auth/deletion policy, admin tooling, specs, and user-facing docs only if ClawHub wants deleted GitHub identities to be restorable or reusable.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: settings calls users.deleteAccount, which sets deactivatedAt and purgedAt, and the OAuth callback rejects later sign-in for that user; I did not run a live GitHub OAuth flow.

## Solution Assessment

Unclear as a code change. The requested restore/unlink path conflicts with the current irreversible-deletion policy and admin unban deliberately rejects deactivated accounts, so maintainer support review is the narrowest next step.
