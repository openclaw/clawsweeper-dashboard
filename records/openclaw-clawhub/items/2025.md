## Summary

Keep open: this is an open ClawHub issue, and cleanup policy does not auto-close issues; current main also intentionally blocks GitHub reauthentication for permanently deleted accounts, so this remains a live account recovery/security-policy request.

## What This Changes

The issue asks for restoration of a deleted ClawHub account or removal of the GitHub OAuth binding so the reporter can sign up again.

## Best Possible Solution

Finalize the deleted-account recovery policy, then either keep deletion irreversible with clear support guidance or implement an audited admin/support restore or GitHub unlink flow that preserves identity, ban, token, publisher, and purged-data boundaries.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: settings invokes users.deleteAccount, which writes deactivatedAt/purgedAt, and GitHub OAuth completion calls handleDeletedUserSignIn, which rejects the deactivated user; I did not inspect the reporter's production record.

## Solution Assessment

No for a one-off restore or binding deletion from an issue comment. The maintainable path needs maintainer-approved recovery policy, identity verification, audit logging, and tests, or an explicit decision that deletion remains irreversible.
