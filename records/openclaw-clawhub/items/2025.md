## Summary

Keep open: current main intentionally blocks deleted-account GitHub OAuth reauth, and the requested restore or GitHub unlink needs maintainer/operator policy, identity verification, and production-record review. ClawHub cleanup policy also keeps issues open rather than auto-closing them.

## What This Changes

The issue requests either restoration of a deleted ClawHub account or removal of the GitHub OAuth binding so the same GitHub identity can register again.

## Best Possible Solution

Decide the account-recovery policy, then either keep deletion irreversible with clearer support guidance or implement an audited restore/GitHub-unlink operator flow with identity verification and ban/deletion safeguards.

## Reproduction Assessment

Yes, by source inspection: account deletion sets `deactivatedAt`/`purgedAt`, and the GitHub OAuth callback rejects deactivated users. I did not run a live OAuth flow or inspect the reporter's production account record.

## Solution Assessment

No for an ad hoc restore or binding deletion. The maintainable path needs a policy-backed, audited recovery or unlink flow, or an explicit decision to keep deletion irreversible with clearer support handling.
