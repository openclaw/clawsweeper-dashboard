## Summary

Keep open: current main still intentionally blocks GitHub sign-in after permanent account deletion, and this needs maintainer/operator product and security handling rather than cleanup closure.

## What This Changes

The issue asks maintainers to restore a deleted ClawHub account or remove its GitHub OAuth binding so the same GitHub identity can sign up again.

## Best Possible Solution

Define an audited maintainer-approved recovery or unlink policy for self-deleted GitHub-linked accounts, or document the operator-only support response, while preserving bans, deletion guarantees, ownership, tokens, and audit invariants.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: settings calls users.deleteAccount, which sets deactivatedAt/purgedAt, and the next GitHub OAuth completion calls handleDeletedUserSignIn and rejects that user; I did not run a live OAuth flow.

## Solution Assessment

Unclear as a code change. Directly restoring the account or deleting the OAuth binding is not the safest standalone fix because current specs make deletion irreversible and the ban/deletion boundaries need maintainer policy first.
