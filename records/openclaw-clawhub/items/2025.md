## Summary

Keep open: this is an open ClawHub issue, and current main intentionally blocks GitHub reauthentication after permanent account deletion; the requested restore or GitHub unlink is an identity-sensitive operator/product policy decision, not a cleanup-close candidate.

## What This Changes

Restore a deleted ClawHub account or remove its GitHub OAuth binding so the same GitHub account can register again.

## Best Possible Solution

Define a maintainer-approved audited support/admin recovery or GitHub-unlink policy for deleted accounts, or explicitly preserve irreversible deletion and document the operator response for affected users.

## Reproduction Assessment

Yes for the source path: Settings calls users.deleteAccount, deleteAccount sets deactivatedAt/purgedAt, and GitHub OAuth calls handleDeletedUserSignIn, which rejects deactivated users. I did not verify the reporter's specific production record.

## Solution Assessment

No for ad hoc restoration or GitHub unlinking from a public issue. The maintainable path is an audited admin/support recovery flow or an explicit product decision that deletion remains permanent.
