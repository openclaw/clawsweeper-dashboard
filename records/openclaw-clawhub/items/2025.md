## Summary

Keep open: current main still deliberately blocks GitHub OAuth sign-in after permanent account deletion, and this is an active account-recovery/identity-binding support request that needs maintainer/operator policy rather than cleanup closure.

## What This Changes

The issue asks for account restoration or GitHub identity unlinking after a deleted account can no longer sign in or re-register with GitHub OAuth.

## Best Possible Solution

Promote the draft account-recovery RFC into an accepted policy, then either implement an audited restore/unlink support path with ban safeguards and tests or explicitly document that deletion remains irreversible.

## Reproduction Assessment

Yes, by source inspection. Deleting a GitHub-linked account calls deleteAccount, which writes deactivatedAt/purgedAt, and the OAuth completion path calls handleDeletedUserSignIn, which rejects that state; I did not inspect the reporter's production row.

## Solution Assessment

No for ad hoc restoration or GitHub binding removal from an issue comment. The maintainable solution needs an explicit recovery policy, same-provider identity verification, audit logging, ban safeguards, and regression coverage, or a confirmed decision that deletion stays permanent.
