## Summary

Keep open: current main still intentionally blocks GitHub OAuth reauthentication for permanently deleted accounts, and restoring the reporter or removing the GitHub binding is an unresolved support, security, and product-policy decision rather than a conservative cleanup close.

## What This Changes

The issue asks for account restoration or GitHub OAuth unlinking so the reporter can sign up again after deleting their ClawHub account.

## Best Possible Solution

Decide and document the account-recovery policy; if recovery is allowed, implement an audited restore or GitHub unlink operator flow with identity verification that preserves bans, purged-data boundaries, and regression tests.

## Reproduction Assessment

Yes, by source inspection: settings calls users.deleteAccount, that mutation sets deactivatedAt/purgedAt, and the OAuth callback rejects deactivated users. I did not run a live OAuth flow or inspect the reporter's production record.

## Solution Assessment

No for an ad hoc restore or binding deletion. The maintainable path needs a policy-backed, audited recovery or unlink flow, or an explicit decision to keep deletion irreversible with clearer support guidance.
