## Summary

Keep open: current main intentionally blocks GitHub sign-in after user-initiated account deletion, and resolving this reporter's lockout requires maintainer support and product-policy judgment rather than conservative cleanup.

## What This Changes

The issue asks maintainers to restore a deleted ClawHub account or remove its GitHub OAuth binding so the reporter can sign up again.

## Best Possible Solution

Handle the individual account through maintainer support/admin tooling, then deliberately update account-deletion policy, audited tooling, specs, and docs only if deleted GitHub identities should become recoverable or reusable.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: settings calls `users.deleteAccount`, the mutation sets `deactivatedAt` and `purgedAt`, and the OAuth callback rejects that deleted user on later sign-in; I did not run a live GitHub OAuth flow.

## Solution Assessment

Unclear as a code change. The requested restore/unlink path conflicts with the current irreversible-deletion spec and admin unban boundary, so maintainer support review is the narrowest next step.
