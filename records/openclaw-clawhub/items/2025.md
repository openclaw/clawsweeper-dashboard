## Summary

Keep open: current main intentionally blocks OAuth sign-in for permanently deleted accounts, and this report is part of a recurring account-recovery support cluster. Resolving it needs maintainer identity verification plus a security/product policy decision, not conservative source cleanup.

## What This Changes

The reporter asks maintainers to restore a deleted GitHub-linked ClawHub account or remove its GitHub OAuth binding so they can sign up again.

## Best Possible Solution

Define a maintainer-approved recovery policy, then either add an audited admin restore/GitHub-unlink flow or explicitly keep deletion terminal with clearer support guidance.

## Reproduction Assessment

Yes. By source inspection, settings calls users.deleteAccount, which sets deactivatedAt/purgedAt, and OAuth completion calls handleDeletedUserSignIn, which rejects deactivated users; I did not run a live OAuth flow or inspect the reporter's production record.

## Solution Assessment

No for an autonomous direct restore or binding deletion. The maintainable path needs a policy-backed, audited recovery or unlink path, or an explicit decision to keep deletion irreversible with support guidance.
