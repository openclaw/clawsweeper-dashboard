## Summary

Keep open. This is an account-access support/security issue: current main intentionally blocks reauth for permanently deleted users, and restoring or unlinking a GitHub identity needs maintainer/operator policy rather than conservative cleanup closure.

## What This Changes

The issue asks maintainers to restore a deleted ClawHub account or remove its GitHub OAuth binding so the user can sign up again.

## Best Possible Solution

Define an audited maintainer/operator recovery or unlink process for deleted GitHub-linked accounts, or explicitly preserve irreversible deletion and document the private support path while keeping ban and purge protections intact.

## Reproduction Assessment

Yes for the general lockout: source inspection shows account deletion writes deactivatedAt/purgedAt and OAuth rejects deactivated users. This does not verify the reporter's production row or whether a separate OAuth/session issue also applies.

## Solution Assessment

No for ad hoc restore or unlink from a public issue. The safer solution is a maintainer-approved, audited account recovery policy or an explicit product decision that deleted accounts remain unrecoverable.
