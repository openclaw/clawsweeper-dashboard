## Summary

Keep open. Current main intentionally treats account deletion as irreversible and blocks GitHub reauth, but this is a valid affected-user recovery request in an open issue cluster and needs maintainer/security product judgment.

## What This Changes

The issue asks maintainers to restore a deleted ClawHub account or remove the GitHub OAuth binding so the same GitHub account can sign up again.

## Best Possible Solution

Adopt a maintainer-approved deleted-account recovery policy and implement it consistently across OAuth callbacks, account deletion, GitHub identity binding, operator tooling, audit logs, user-facing errors, docs/specs, CLI token access, and regression tests.

## Reproduction Assessment

Yes, by source inspection: deleting an account writes deactivatedAt/purgedAt, and GitHub OAuth re-entry calls handleDeletedUserSignIn, which rejects the same user before a usable session exists. I did not run a live OAuth flow or inspect the reporter's production row.

## Solution Assessment

No, automatically restoring the account or purging the GitHub binding is not clearly the best fix yet. The maintainable path needs an explicit security/product policy before changing deletion, provider identity binding, audit, token, and privacy semantics.
