## Summary

Keep open. Current main intentionally blocks OAuth reauthentication for deleted/deactivated accounts, the related deleted-account recovery cluster remains open, and ClawHub cleanup policy keeps issues open; the remaining work is a security-sensitive account recovery policy decision rather than conservative cleanup.

## What This Changes

The issue requests account restoration or GitHub identity unlinking after deleting a ClawHub account prevents GitHub OAuth sign-in or re-registration.

## Best Possible Solution

Define an audited support/admin recovery or GitHub unlink policy for deleted GitHub-linked accounts, or explicitly keep deletion irreversible and route affected users through a private operator process.

## Reproduction Assessment

Yes for the general lockout path: source and tests show account deletion writes deactivatedAt/purgedAt and OAuth reauthentication rejects those users. I did not verify the reporter's specific production row or identity ownership.

## Solution Assessment

No for ad hoc restoration or unlinking from a public issue. The maintainable path is an audited maintainer/operator recovery policy, or a documented product decision that deleted accounts cannot be recovered.
