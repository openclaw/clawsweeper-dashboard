## Summary

Keep open: current main intentionally blocks GitHub OAuth sign-in after user-initiated deletion, and this support request needs maintainer identity, security, and product-policy handling rather than cleanup closure.

## What This Changes

The reporter asks maintainers to restore a deleted GitHub-linked ClawHub account or remove the GitHub OAuth binding so the same GitHub account can sign up again.

## Best Possible Solution

Define an approved account recovery or re-registration policy, then add an audited support/admin restore or GitHub-unlink path, or explicitly keep deletion terminal with clearer support guidance.

## Reproduction Assessment

Yes. Source inspection shows settings calls users.deleteAccount, that mutation sets deactivatedAt/purgedAt, and the GitHub OAuth callback rejects deactivated users; I did not run a live OAuth flow or inspect the reporter's production record.

## Solution Assessment

No for a direct restore or binding deletion. The maintainable solution needs a policy-backed, audited path that separates self-deleted accounts from banned or abuse-disabled identities.
