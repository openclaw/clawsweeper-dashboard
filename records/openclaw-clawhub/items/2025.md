## Summary

Keep open: current main intentionally blocks deleted/deactivated GitHub OAuth reauth, and resolving the requested restore or unlink needs maintainer/operator policy plus identity verification. The ClawHub cleanup policy also keeps issues open rather than auto-closing them.

## What This Changes

The issue asks maintainers to restore a deleted GitHub-linked account or remove its GitHub binding so the user can sign up again.

## Best Possible Solution

Decide and document the account-recovery policy; if recovery is allowed, implement an audited restore or GitHub unlink operator flow with identity verification that preserves bans, purged-data boundaries, and tests.

## Reproduction Assessment

Yes, by source inspection: settings calls users.deleteAccount, that mutation sets deactivatedAt/purgedAt, and the OAuth callback rejects deactivated users. I did not run a live OAuth flow or inspect the reporter's production record.

## Solution Assessment

No for an ad hoc restore or binding deletion. The maintainable path needs a policy-backed, audited recovery or unlink flow, or an explicit decision to keep deletion irreversible with clearer support guidance.
