## Summary

Keep open. Current main intentionally blocks deleted accounts from GitHub reauth and has no restore or GitHub identity unlink path, while ClawHub cleanup policy keeps issues open for maintainer judgment.

## What This Changes

The reporter asks maintainers to restore a deleted ClawHub account or remove the GitHub OAuth binding so the same GitHub identity can register again.

## Best Possible Solution

Define and document an audited support/admin recovery or unlink policy for deleted GitHub-linked accounts, then handle this account and related reports through that path while preserving ban and deletion boundaries.

## Reproduction Assessment

Yes, source-reproducible: current main's delete flow sets deactivatedAt/purgedAt and OAuth sign-in throws the permanent deleted-account error. This does not verify the reporter's production account record.

## Solution Assessment

No for ad hoc restoration or GitHub unlinking from a public issue. The maintainable path is an audited operator/admin recovery flow or an explicit product decision that deletion remains permanent.
