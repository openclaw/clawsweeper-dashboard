## Summary

Keep open: current main intentionally treats owner-initiated deletion as irreversible, and ClawHub cleanup policy does not auto-close issues; this needs maintainer/operator account-recovery and identity-binding policy.

## What This Changes

The issue requests restoration of a deleted ClawHub account or removal of its GitHub OAuth binding so the reporter can sign up again.

## Best Possible Solution

Define a maintainer-approved deleted-account recovery policy, then either add an audited admin/support restore or GitHub unlink flow with tests and docs, or explicitly document that deletion remains permanent.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: settings invokes users.deleteAccount, that writes deactivatedAt/purgedAt, and GitHub OAuth completion calls handleDeletedUserSignIn, which rejects the deactivated user; the reporter's production record was not inspected.

## Solution Assessment

No for ad hoc restoration or GitHub unlinking from an issue comment. The maintainable path needs maintainer-approved recovery policy, verified identity ownership, audit logging, and regression coverage.
