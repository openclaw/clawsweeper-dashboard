## Summary

Keep open. Current main intentionally blocks reauthentication for deleted GitHub-linked accounts, and restoring the account or unlinking the GitHub identity is a security-sensitive recovery/policy decision rather than a safe source-only cleanup.

## What This Changes

The issue requests restoring a deleted ClawHub account or removing its GitHub OAuth binding so the same GitHub account can sign up again.

## Best Possible Solution

Define an audited private support/admin recovery policy for deleted GitHub-linked accounts, or explicitly confirm irreversible deletion and document the operator response path for affected users.

## Reproduction Assessment

Yes for the general lockout path: current source and tests show deletion writes deactivatedAt/purgedAt and GitHub OAuth rejects those users. This review did not verify the reporter's specific production row or identity ownership.

## Solution Assessment

No for ad hoc restoration or unlinking from a public issue. The maintainable path is an audited maintainer/operator recovery policy, or a documented product decision that deleted accounts cannot be recovered.
