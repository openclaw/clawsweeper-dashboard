## What This Changes

The issue asks maintainers to restore a deleted GitHub-linked ClawHub account or remove its GitHub OAuth binding so the reporter can sign up again.

## Best Possible Solution

Define a maintainer-approved deleted-account recovery policy, then provide an audited support/admin restore or GitHub-unlink path with provider ownership checks and preserved audit/token safeguards.

Yes, by source inspection: settings calls users.deleteAccount, which sets deactivatedAt/purgedAt, and OAuth completion calls handleDeletedUserSignIn, which rejects deactivated users. I did not run a live OAuth flow or inspect the reporter's production record.

No for an autonomous direct restore or binding deletion. The maintainable path needs a policy-backed, audited recovery or unlink flow, or an explicit decision to keep deletion irreversible with clearer support guidance.
