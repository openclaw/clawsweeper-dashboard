## Summary

Keep open. Current main intentionally treats user-initiated deletion as irreversible and blocks deleted accounts during GitHub OAuth, but this is an account-specific recovery or unlink request with security and privacy implications, so it needs maintainer/operator decision rather than ClawSweeper closure.

## What This Changes

The reporter asks for restoration or removal of their GitHub OAuth binding after deleting their ClawHub account so they can sign in again.

## Best Possible Solution

Define an audited private support/admin recovery policy for deleted GitHub-linked accounts, or explicitly reaffirm irreversible deletion and route any unlinking through a verified operator-only process.

## Reproduction Assessment

Yes for the general lockout path: source shows deletion sets deactivatedAt/purgedAt and the GitHub auth callback rejects deactivated users, with tests covering that behavior. The reporter's exact production row and identity ownership were not verified.

## Solution Assessment

No for an ad hoc public restore or unlink. The maintainable solution is a deliberate private recovery/unlink policy with ownership verification and audit logging, or an explicit product decision that deleted accounts cannot be recovered.
