Keep open: current main intentionally blocks login after user-initiated deletion, and this is an account-support plus product-policy/security decision rather than conservative cleanup for an issue.

## What This Changes

The issue asks maintainers to restore a deleted GitHub-linked account or remove the GitHub binding so the reporter can sign up again.

## Best Possible Solution

Yes. Source inspection gives a high-confidence path: Settings deletion calls users.deleteAccount, that marks the user deactivated, and the OAuth callback rejects subsequent sign-in for that user.

## Solution Assessment

Unclear as a code change. Restoring or unlinking the account conflicts with the current documented irreversible-deletion policy, so maintainer support and product/security review should come before implementation.
