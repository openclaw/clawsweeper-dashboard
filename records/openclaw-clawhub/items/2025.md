## Summary

Keep open: current main intentionally blocks GitHub reauthentication after permanent account deletion, and ClawHub cleanup policy keeps issues open. This remains an identity recovery or GitHub binding policy request, with closely related open reports such as https://github.com/openclaw/clawhub/issues/917 and https://github.com/openclaw/clawhub/issues/1452.

## What This Changes

The issue asks for account restoration or removal of a GitHub OAuth binding after permanent account deletion prevents signing in again.

## Best Possible Solution

Define one maintainer-owned recovery policy for deleted GitHub-linked accounts, then either keep irreversible deletion with clearer support guidance or implement an audited same-provider restore/unlink flow with ban safeguards and regression tests.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: Settings calls users.deleteAccount, that writes deactivatedAt and purgedAt, and GitHub OAuth completion calls handleDeletedUserSignIn, which rejects deactivated users. This does not verify the reporter's specific production record.

## Solution Assessment

No for ad hoc restoration or direct GitHub unlinking from an issue comment. The maintainable path needs maintainer-approved recovery policy, identity verification, audit logging, and tests, or an explicit decision that deletion remains irreversible.
