## Summary

Keep open: current main intentionally blocks sign-in for deleted/deactivated GitHub-backed ClawHub accounts, and resolving this reporter’s lockout requires verified production account handling plus a maintainer product/security policy decision.

## What This Changes

The issue requests account restoration or removal of the reporter’s GitHub OAuth binding so they can sign up again after account deletion.

## Best Possible Solution

Define an audited account recovery or GitHub identity-unlink policy, then either add guarded support/admin tooling with tests or explicitly keep deletion terminal with clearer support guidance.

## Reproduction Assessment

Yes, by source inspection: account deletion sets deactivatedAt/purgedAt, the OAuth callback rejects deactivated users, and tests assert that path. I did not run a live OAuth flow or inspect the reporter’s production record.

## Solution Assessment

No for a direct autonomous restore or binding deletion. The maintainable path needs a maintainer-approved, audited recovery/unlink policy or an explicit decision to keep deletion irreversible with better support guidance.
