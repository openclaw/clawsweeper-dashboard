## Summary

Keep open: this is an open ClawHub issue and a live account-recovery request. Current main intentionally blocks GitHub reauthentication for permanently deleted accounts, and restoration or GitHub OAuth unlinking needs maintainer/operator security policy and production data handling.

## What This Changes

The issue requests account restoration or removal of the GitHub OAuth binding after account deletion prevents GitHub login or re-registration.

## Best Possible Solution

Define a maintainer-approved recovery outcome, then either keep deletion irreversible with clear support guidance or add an audited admin/support restore or GitHub unlink flow with identity verification, audit logging, docs, and tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: settings invokes users.deleteAccount, that writes deactivatedAt/purgedAt, and GitHub OAuth completion calls handleDeletedUserSignIn, which rejects the deactivated user; I did not inspect the reporter's production record.

## Solution Assessment

No for a one-off restore or binding deletion from a public issue. The maintainable path needs maintainer-approved recovery policy, identity verification, audit logging, and tests, or an explicit decision that deletion remains irreversible.
