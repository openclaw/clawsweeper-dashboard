Keep open. Current main intentionally blocks GitHub reauthentication for deactivated/purged accounts, while the requested restore or GitHub-binding removal remains security-sensitive support/operator work that needs a maintainer product decision.

## What This Changes

The issue asks for account restoration or removal of the deleted account's GitHub OAuth binding so the reporter can sign up again.

## Best Possible Solution

Define an audited support/admin recovery policy for deleted GitHub-linked accounts, or explicitly confirm irreversible deletion and route affected users through a private operator process that preserves privacy and ban boundaries.

Yes for the general lockout path: source and tests show deletion writes deactivatedAt/purgedAt and GitHub OAuth rejects those users. This review did not verify the reporter's specific production row or identity ownership.

## Solution Assessment
