## Summary

Keep open. Current main intentionally blocks permanently deleted ClawHub accounts from GitHub sign-in, the same account-lockout pattern is clustered in other open reports, and resolving it needs account-lifecycle/security policy plus production operator handling rather than cleanup closure.

## What This Changes

The user requests account restoration or removal of their GitHub OAuth binding after deleting their ClawHub account so they can sign up again.

## Best Possible Solution

Define a maintainer-approved audited recovery/unlink or explicitly terminal support policy for deleted GitHub-linked accounts, then implement it consistently in auth callbacks, admin/support tooling, UI errors, docs/specs, and regression tests.

## Reproduction Assessment

Yes, source-reproducible for the deleted-account lockout: deleteAccount sets deactivatedAt/purgedAt, the OAuth callback rejects that state, and tests assert the deleted-account error. This does not verify Onlydreams' production account record.

## Solution Assessment

No for ad hoc restore or unlink from a public issue. The safer repository-level solution is a maintainer-approved recovery, purge/recreate, or terminal-support policy with provider-account verification, audit logging, and abuse safeguards.
