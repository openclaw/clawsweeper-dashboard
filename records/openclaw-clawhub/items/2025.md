## Summary

Keep open: this is an open ClawHub issue, and cleanup policy does not auto-close issues; current main also intentionally blocks GitHub reauthentication for permanently deleted accounts, so this remains a live account recovery and identity-binding policy request.

## What This Changes

The issue asks maintainers to restore a deleted GitHub-linked account or remove its GitHub OAuth binding so the reporter can sign up again.

## Best Possible Solution

Decide and document the deleted-account recovery policy, then either keep deletion irreversible with clear support guidance or implement an audited admin/support restore or GitHub unlink flow with same-provider verification, ban safeguards, audit logs, and tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: settings invokes users.deleteAccount, which writes deactivatedAt/purgedAt, and GitHub OAuth completion calls handleDeletedUserSignIn, which rejects the deactivated user; I did not inspect the reporter's production record.

## Solution Assessment

No for ad hoc restoration or direct GitHub unlinking from an issue comment. The maintainable path needs maintainer-approved recovery policy, identity verification, audit logging, and tests, or an explicit decision that deletion remains irreversible.
