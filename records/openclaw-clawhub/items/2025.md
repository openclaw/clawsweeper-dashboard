## Summary

Keep open: current main intentionally blocks GitHub reauthentication for permanently deleted accounts, and this live ClawHub account-recovery request needs maintainer/operator security-policy handling rather than cleanup closure.

## What This Changes

The issue asks for a deleted GitHub-linked ClawHub account to be restored or for its GitHub OAuth binding to be removed so the reporter can sign up again.

## Best Possible Solution

Define a maintainer-approved recovery outcome: either keep deletion irreversible with clear support guidance, or add an audited admin/support restore or GitHub unlink flow that preserves bans, identity ownership, purged-data boundaries, tokens, publishers, docs, and tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: settings invokes users.deleteAccount, that writes deactivatedAt/purgedAt, and GitHub OAuth completion calls handleDeletedUserSignIn, which rejects the deactivated user; I did not inspect the reporter's production record.

## Solution Assessment

No for a one-off restore or binding deletion from an issue comment. The maintainable path needs maintainer-approved recovery policy, identity verification, audit logging, and tests, or an explicit decision that deletion remains irreversible.
