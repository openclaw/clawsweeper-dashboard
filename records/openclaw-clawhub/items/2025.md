## Summary

Keep open. Current main intentionally blocks GitHub OAuth reauth for deleted/deactivated accounts, and this account-specific restore/unlink request is security- and privacy-sensitive, so it needs maintainer/operator policy handling rather than cleanup closure.

## What This Changes

The issue requests account restoration or removal of a GitHub OAuth binding so the reporter can sign up again after deleting their ClawHub account.

## Best Possible Solution

Define a maintainer-approved deleted-account recovery, unlinking, or no-recovery policy that preserves GitHub identity proof, deletion privacy, token revocation, ban boundaries, audit logs, docs, and tests.

## Reproduction Assessment

Yes for the general lockout path by source inspection: deleteAccount sets deactivatedAt/purgedAt and GitHub OAuth sign-in calls handleDeletedUserSignIn, which rejects deactivated users. I did not verify the reporter's production row or identity ownership.

## Solution Assessment

No for ad hoc public restore or unlink. The maintainable solution is a deliberate private recovery/unlink policy with ownership verification and audit logging, or an explicit product decision that deleted accounts cannot be recovered.
