## Summary

Keep open: current main deliberately treats self-deleted GitHub-linked accounts as permanently deactivated, and ClawHub cleanup policy keeps issues open for maintainer handling rather than closing valid account-support cases.

## What This Changes

The reporter cannot sign in after deleting a GitHub-linked account and asks for account restoration or removal of the GitHub OAuth binding.

## Best Possible Solution

Define a maintainer-approved deleted-account recovery policy, then either add an audited support/admin restore or GitHub-unlink flow, or keep deletion terminal with clearer support guidance.

## Reproduction Assessment

Yes, by source inspection: account deletion reaches users.deleteAccount, and the GitHub OAuth completion path rejects any existing user with deactivatedAt. I did not run a live OAuth flow or inspect the reporter's production record.

## Solution Assessment

No for an autonomous repo-only restore or binding deletion. The maintainable path needs a policy-backed, audited support or admin flow, or an explicit decision that deleted accounts remain permanently unrecoverable.
