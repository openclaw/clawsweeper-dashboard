## Summary

Keep open. Current main intentionally blocks GitHub OAuth sign-in for deleted/deactivated accounts and has no restore or GitHub-binding unlink path, while this account-access request needs maintainer/operator product and security review rather than conservative cleanup closure.

## What This Changes

The issue asks maintainers to restore a deleted account or remove its GitHub OAuth binding so the reporter can sign up again.

## Best Possible Solution

Define a maintainer-approved, audited support/admin recovery or GitHub unlink policy, or explicitly keep deletion irreversible and route affected users through a private operator process.

## Reproduction Assessment

Yes for the general lockout behavior: source inspection shows deleteAccount sets deactivatedAt/purgedAt and GitHub OAuth re-entry rejects deactivated or legacy deleted users. I did not run a live OAuth flow or verify the reporter's production row.

## Solution Assessment

No for ad hoc restore or unlink from a public issue. The maintainable path is an audited operator/admin recovery policy, or an explicit product decision that deleted accounts cannot be recovered.

## Security Review

Status: needs_attention

Summary: Account restoration or GitHub unlinking is security-sensitive because it changes identity binding and permanent-deletion boundaries.

Concerns:

- **[medium] Treat GitHub unlink as privileged recovery:** `convex/lib/githubIdentity.ts:13`
  - body: authAccounts.providerAccountId is used as the GitHub identity anchor; unlinking or rebinding it needs verified ownership checks and an auditable operator path.
  - confidence: 0.9
- **[medium] Preserve deleted-versus-banned semantics:** `convex/auth.ts:45`
  - body: The auth path separately blocks deactivated users and ban-related deleted users, so recovery must not let reauthentication bypass bans or malware autobans.
  - confidence: 0.88
- **[medium] Define privacy semantics for purged accounts:** `convex/users.ts:486`
  - body: deleteAccount clears profile, contact, GitHub-derived, and telemetry fields while setting purgedAt, so restoration must not silently revive data that deletion intentionally purged.
