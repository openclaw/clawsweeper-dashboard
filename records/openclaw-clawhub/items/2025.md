## Summary

Keep open: current main intentionally blocks login after self-deletion, but the requested recovery or GitHub OAuth unlinking is an identity-sensitive support/product decision rather than a repo-only bug fix.

## What This Changes

The issue asks for account restoration or removal of the GitHub OAuth binding so the reporter can sign up again after deleting their account.

## Best Possible Solution

Define and document a maintainer-approved deletion recovery or OAuth unlink policy, then add an audited support/admin flow if recovery is approved; otherwise keep deletion terminal and improve support guidance.

## Reproduction Assessment

Yes, by source inspection: settings calls users.deleteAccount, which writes deactivatedAt/purgedAt, and OAuth completion calls handleDeletedUserSignIn, which rejects deactivated users. I did not run a live OAuth flow or inspect the reporter's production record.

## Solution Assessment

No for an autonomous direct restore or binding deletion. The maintainable path needs a policy-backed, audited recovery or unlink flow, or an explicit decision to keep deletion irreversible with clearer support guidance.
