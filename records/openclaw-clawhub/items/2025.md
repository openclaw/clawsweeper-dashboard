## Summary

Keep open: ClawHub cleanup policy does not auto-close issues, and current main intentionally blocks GitHub reauthentication for permanently deleted accounts, so this remains a live account-recovery and identity-binding policy request.

## What This Changes

The issue requests account restoration or removal of the GitHub OAuth binding after account deletion prevents re-login or re-registration.

## Best Possible Solution

Define the deleted-account recovery policy, then either implement an audited admin/support restore or GitHub unlink flow with tests, or explicitly document and enforce that deletion remains permanent.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: settings invokes users.deleteAccount, deletion writes deactivatedAt/purgedAt, and GitHub OAuth completion rejects through handleDeletedUserSignIn; I did not inspect the reporter's production record.

## Solution Assessment

No for ad hoc restoration or binding deletion from issue text. The maintainable path needs maintainer-approved recovery policy, identity verification, audit logging, and tests, or an explicit decision that deletion remains irreversible.
