## Summary

Keep open: this is an open ClawHub issue, and current main intentionally blocks GitHub reauthentication for permanently deleted accounts; resolving it needs maintainer/operator account-recovery policy rather than cleanup closure.

## What This Changes

The reporter requests restoration of a deleted ClawHub account or removal of the GitHub OAuth binding so they can register again.

## Best Possible Solution

Resolve the draft account-recovery policy, then either implement an audited admin/support restore or GitHub unlink flow with identity verification and tests, or document and keep enforcing irreversible owner-initiated deletion.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: settings invokes users.deleteAccount, that mutation writes deactivatedAt/purgedAt, and GitHub OAuth completion calls handleDeletedUserSignIn, which rejects the deactivated user with the permanent deleted-account message.

## Solution Assessment

No for a one-off restore or binding deletion from an issue comment. The maintainable path needs maintainer-approved recovery policy, verified identity ownership, audit logging, tests, and production operator handling, or an explicit decision that deletion remains irreversible.
