## Summary

Keep open. Current main intentionally blocks deleted/deactivated accounts from GitHub OAuth and has no restore or unlink path, while this account-access request needs maintainer/operator product and security review.

## What This Changes

The reporter requests restoring a deleted ClawHub account or unbinding its GitHub OAuth identity so the same GitHub account can sign up again.

## Best Possible Solution

Define a maintainer-approved, audited support/admin recovery or GitHub unlink policy, or explicitly keep deletion irreversible and route affected users through a private operator process.

## Reproduction Assessment

Yes for the general account-lock behavior: source and tests show deletion writes deactivatedAt/purgedAt and OAuth rejects those users. I did not verify the reporter's specific production row.

## Solution Assessment

No for ad hoc restore or unlink from a public issue. The maintainable path is an audited operator/admin recovery policy, or an explicit product decision that deleted accounts cannot be recovered.
