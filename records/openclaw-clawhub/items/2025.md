## Summary

Keep open. Current main and v0.16.0 intentionally treat user deletion as irreversible and block GitHub reauthentication, so the requested restore or GitHub-binding removal is security-sensitive account recovery work that needs maintainer/operator policy review.

## What This Changes

The issue requests restoring a deleted ClawHub account or removing its GitHub OAuth binding so the same GitHub identity can sign up again.

## Best Possible Solution

Define an audited private support/admin recovery policy for deleted GitHub-linked accounts, or explicitly document irreversible deletion as final while preserving privacy, identity, and ban boundaries.

## Reproduction Assessment

Yes for the general lockout path: current source and tests show deletion writes deactivatedAt/purgedAt and GitHub OAuth rejects those users. This review did not verify the reporter's specific production row or identity ownership.

## Solution Assessment

No for ad hoc restoration or unlinking from a public issue. The maintainable path is an audited maintainer/operator recovery policy, or a documented product decision that deleted accounts cannot be recovered.
