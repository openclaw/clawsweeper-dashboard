## Summary

Keep open: current main still intentionally blocks GitHub reauthentication for permanently deleted accounts, and ClawHub cleanup policy does not auto-close issues. This is a live account-recovery and identity-binding request that needs maintainer/operator security policy and production-data handling.

## What This Changes

The issue asks for account restoration or GitHub identity unlinking after account deletion prevents GitHub OAuth login.

## Best Possible Solution

Decide the deleted-account recovery policy, then either implement an audited admin/support restore or GitHub unlink flow with tests and docs, or document and enforce that owner-initiated deletion remains permanent.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: deleteAccount writes deactivatedAt/purgedAt, GitHub OAuth calls handleDeletedUserSignIn, and tests assert the permanent deleted-account rejection. I did not verify the reporter's production account record.

## Solution Assessment

No for ad hoc restoration or GitHub binding deletion from an issue comment. The maintainable path needs maintainer-approved recovery policy, identity verification, auditability, and tests, or an explicit decision that deletion remains irreversible.
