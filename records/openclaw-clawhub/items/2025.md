## Summary

Keep open: this is an open ClawHub issue, current main still intentionally blocks GitHub reauthentication for permanently deleted accounts, and the requested restore/unlink action needs maintainer/operator account-recovery and security-policy handling.

## What This Changes

The issue asks for account restoration or GitHub identity unlinking after a deleted account can no longer sign in with GitHub OAuth.

## Best Possible Solution

Finalize the deleted-account recovery policy, then either keep deletion irreversible with clear support guidance or add an audited admin/support restore or GitHub unlink flow with tests and docs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: settings invokes users.deleteAccount, that writes deactivatedAt/purgedAt, and GitHub OAuth completion calls handleDeletedUserSignIn, which rejects the deactivated user; I did not inspect the reporter's production record.

## Solution Assessment

No for a one-off restore or binding deletion from an issue comment. The maintainable path needs maintainer-approved recovery policy, identity verification, audit logging, and tests, or an explicit decision that deletion remains irreversible.
