## Summary

Keep open: ClawHub cleanup policy does not auto-close issues, and current main still intentionally blocks GitHub reauthentication after permanent account deletion; this needs account-recovery/security policy handling, not conservative cleanup closure.

## What This Changes

The issue asks ClawHub operators to restore a permanently deleted account or remove its GitHub OAuth binding so the same GitHub identity can sign up again.

## Best Possible Solution

Define a maintainer-approved audited recovery/unlink policy for deleted GitHub-linked accounts, or explicitly keep irreversible deletion and handle affected users through that documented operator path.

## Reproduction Assessment

Yes. Source inspection shows Settings calls users.deleteAccount, which writes deactivatedAt/purgedAt, and GitHub OAuth completion calls handleDeletedUserSignIn, which rejects a matching deactivated user.

## Solution Assessment

No for ad hoc restoration or GitHub unlinking from an issue comment. The maintainable solution needs verified ownership, audit logging, ban safeguards, privacy semantics for purged data, and an explicit maintainer policy decision.
