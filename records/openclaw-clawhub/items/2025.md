## Summary

Keep open. Current main intentionally blocks GitHub reauthentication for deactivated or purged accounts, while the requested restore or GitHub-binding removal is security-sensitive account recovery work that needs maintainer/operator policy approval.

## What This Changes

The issue asks for a deleted GitHub-linked ClawHub account to be restored or for its GitHub OAuth binding to be removed so the user can sign up again.

## Best Possible Solution

Define an audited private support/admin recovery policy for deleted GitHub-linked accounts, or explicitly affirm irreversible deletion and handle related reports such as https://github.com/openclaw/clawhub/issues/504 and https://github.com/openclaw/clawhub/issues/917 consistently.

## Reproduction Assessment

Yes for the general lockout path: source shows deletion sets deactivatedAt/purgedAt and the GitHub auth callback rejects deactivated users. The reporter's exact production row and identity ownership were not verified.

## Solution Assessment

No for an ad hoc public restore or unlink. The maintainable solution is an audited private recovery/unlink policy, or a deliberate product decision that deleted accounts cannot be recovered.
