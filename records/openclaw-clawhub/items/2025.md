## Summary

Keep open: this is an open ClawHub support/security issue, and the repository cleanup policy only permits auto-closing PRs that are certainly implemented or mostly implemented on main. Current main intentionally blocks GitHub OAuth for deleted/deactivated accounts, so resolving the request needs maintainer/operator identity verification and a product/security policy decision.

## What This Changes

The reporter asks for restoration of a deleted account or removal of the GitHub OAuth binding so they can sign up again.

## Best Possible Solution

Decide and document the deleted-account recovery policy, then implement an audited same-provider restore or unlink operator path with tests, or explicitly keep deletion irreversible with clearer support guidance.

## Reproduction Assessment

Yes, by source inspection: settings calls users.deleteAccount, that mutation sets deactivatedAt/purgedAt, and the OAuth callback rejects deactivated users. I did not run a live OAuth flow or inspect the reporter's production record.

## Solution Assessment

No for an ad hoc restore or binding deletion. The maintainable path needs a policy-backed, audited recovery or unlink flow, or an explicit decision to keep deletion irreversible with support guidance.
