## Summary

Keep open: ClawHub cleanup policy keeps issues open, and current main still intentionally blocks GitHub reauthentication for permanently deleted accounts without an account-restore or GitHub-binding unlink path. This remains account recovery/security policy work, not conservative cleanup.

## What This Changes

The issue requests account restoration or removal of the GitHub OAuth binding after account deletion prevents login or re-registration.

## Best Possible Solution

Define a maintainer-owned recovery policy, then either keep irreversible deletion with clearer support guidance or implement an audited same-provider restore/GitHub-unlink flow with identity proof, audit logs, ban safeguards, and regression tests.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: Settings calls users.deleteAccount, which writes deactivatedAt/purgedAt, and GitHub OAuth completion calls handleDeletedUserSignIn, which rejects the deactivated user. I did not inspect the reporter's production account row.

## Solution Assessment

No for ad hoc restoration or direct GitHub unlinking from an issue comment. The maintainable solution needs maintainer-approved recovery policy, identity verification, audit logging, and tests, or an explicit decision that deletion remains irreversible.
