## Summary

Keep open: current main intentionally makes user-initiated deletion irreversible, and resolving this GitHub OAuth lockout needs maintainer production access, identity verification, and security/product policy handling.

## What This Changes

The issue requests restoring a deleted GitHub-linked ClawHub account or removing its GitHub OAuth binding so the reporter can sign up again.

## Best Possible Solution

Define a maintainer-approved account recovery policy, then either add an audited admin/support restore or GitHub-unlink flow with tests, or keep deletion terminal with clearer support guidance.

## Reproduction Assessment

Yes, by source inspection: settings calls users.deleteAccount, which sets deactivatedAt/purgedAt, and OAuth completion calls handleDeletedUserSignIn, which rejects deactivated users. I did not run a live OAuth flow or inspect the reporter's production records.

## Solution Assessment

No for a direct autonomous restore or binding deletion. The maintainable path needs a policy-backed, audited recovery or unlink path, or an explicit decision to keep deletion irreversible with support guidance.
