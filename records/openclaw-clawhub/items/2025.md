## Summary

Keep open. Current main intentionally blocks deleted-account GitHub reauth, and this reporter’s requested restore or GitHub unlink is security-sensitive account recovery work that needs maintainer/operator policy rather than cleanup closure.

## What This Changes

The reporter asks ClawHub to restore a deleted account or remove the GitHub OAuth binding so the same GitHub account can sign up again.

## Best Possible Solution

Define a maintainer-approved audited recovery or GitHub unlink policy, or explicitly keep deletion irreversible and route affected users through private support verification.

## Reproduction Assessment

Yes for the general behavior: source and tests show deletion writes deactivatedAt/purgedAt and the OAuth callback rejects deleted users. I did not verify this reporter's specific production row.

## Solution Assessment

No for ad hoc restore or unlink from a public issue. The maintainable path is an audited maintainer/operator recovery policy, or an explicit product decision that deleted accounts cannot be recovered.
