## Summary

Keep open: current main intentionally blocks GitHub reauthentication for permanently deleted accounts, and ClawHub cleanup policy does not auto-close issues. This remains a live account recovery and identity-binding policy request with related open reports.

## What This Changes

The issue requests account restoration or GitHub identity unlinking after a deleted GitHub-linked ClawHub account can no longer sign in or re-register.

## Best Possible Solution

Consolidate the related reports, then either keep irreversible deletion with clearer support guidance or implement an audited same-provider recovery/GitHub-unlink flow with ban safeguards and tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: settings invokes users.deleteAccount, which writes deactivatedAt/purgedAt, and GitHub OAuth completion calls handleDeletedUserSignIn, which rejects the deactivated user; I did not inspect the reporter's production record.

## Solution Assessment

No for ad hoc restoration or direct GitHub unlinking from an issue comment. The maintainable path needs maintainer-approved recovery policy, identity verification, audit logging, and tests, or an explicit decision that deletion remains irreversible.
