## Summary

Keep open: current main intentionally blocks GitHub sign-in after user-initiated account deletion, and resolving this lockout requires maintainer/operator identity, product, and security policy handling rather than conservative cleanup.

## What This Changes

The issue asks maintainers to restore a deleted GitHub-linked account or remove its GitHub OAuth binding so the reporter can sign up again.

## Best Possible Solution

Define and document an audited account recovery or GitHub-unlink policy, then implement a maintainer-approved support/admin path or keep deletion terminal with clearer support guidance.

## Reproduction Assessment

Yes, by source inspection: settings deletion reaches users.deleteAccount, that mutation sets deactivatedAt/purgedAt, and later GitHub OAuth completion rejects the matched user through handleDeletedUserSignIn. I did not run a live OAuth flow or inspect the reporter's production records.

## Solution Assessment

No for an immediate autonomous restore or binding deletion. The maintainable solution needs a policy-backed, audited recovery or unlink path, or an explicit decision that account deletion remains irreversible.
