## Summary

Keep open: current main intentionally rejects GitHub reauthentication for permanently deleted accounts and has no restore or OAuth unlink path, while this repository’s cleanup policy keeps ClawHub issues open for maintainer handling. This belongs with the active account-recovery cluster rather than cleanup closure.

## What This Changes

The issue asks for account restoration or removal of a GitHub OAuth binding after a deleted account can no longer sign in.

## Best Possible Solution

Maintain one maintainer-owned recovery policy for this cluster: either keep irreversible deletion and give clear support guidance, or implement an audited same-provider restore/unlink flow with ban safeguards and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Settings calls users.deleteAccount, which writes deactivatedAt/purgedAt, and GitHub OAuth completion calls handleDeletedUserSignIn, which rejects the deactivated user; this does not verify the reporter’s production record.

## Solution Assessment

No for ad hoc restoration or unlinking from an issue comment. The maintainable path needs maintainer-approved recovery policy, identity verification, audit logging, and tests, or an explicit decision that deletion remains irreversible.
