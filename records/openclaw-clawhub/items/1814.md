## Summary

Keep open. ClawHub cleanup policy keeps issues open, current main still lacks REST/CLI owner selection for skill publishing, and #1840 is an open implementation candidate rather than a close reason before merge.

## What This Changes

Request support for publishing new versions of org-owned skills through authenticated REST/CLI, likely via an owner selector such as `ownerHandle` or `--owner`.

## Best Possible Solution

Review and land #1840, or an equivalent focused fix that threads ownerHandle/--owner through the shared schema, REST JSON and multipart parsing, CLI payload generation, server-side publisher-role authorization, and regression coverage.

## Reproduction Assessment

Yes. Source inspection shows current main has no REST/CLI owner selector, so `publishVersionForUser` reaches `insertVersion` without `ownerPublisherId`; `insertVersion` then defaults to the actor's personal publisher and rejects an existing org-owned slug.

## Solution Assessment

Yes. The narrow maintainable direction is to reuse the web publish action's `ownerHandle` publisher-resolution path with server-side `publisher` role checks, then expose that through the REST and CLI publish surfaces.
