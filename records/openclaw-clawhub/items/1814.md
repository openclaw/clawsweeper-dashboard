## Summary

Keep open. Current main still lacks the requested REST/CLI owner selector for org-owned skill publishes, and the linked open PR #1840 is the implementation candidate to review rather than a reason to close this issue before merge.

## What This Changes

The issue requests REST/CLI support for selecting an org publisher when publishing new versions of org-owned skills from CI.

## Best Possible Solution

Land #1840 or an equivalent focused fix that threads ownerHandle/--owner through schema, REST JSON and multipart parsing, CLI payload generation, server-side publisher-role resolution, and regression coverage.

## Reproduction Assessment

Yes, from source inspection: publishing an existing org-owned slug through POST /api/v1/skills or `clawhub skill publish` has no owner selector, so current main falls back to the actor's personal publisher and hits the slug-taken branch when the skill belongs to another publisher.

## Solution Assessment

Yes, the linked direction is the narrowest maintainable fix: preserve personal-publisher defaults, add an explicit owner selector, and resolve authorization server-side before passing ownerPublisherId into the core publish path.
