## Summary

Keep open. ClawHub cleanup policy keeps issues open, current main still lacks the requested REST/CLI owner selector for skill publishing, and open PR #1840 is the linked implementation candidate rather than a reason to close early.

## What This Changes

Request authenticated REST/CLI owner targeting so org publisher members can publish new versions of org-owned skills from CI.

## Best Possible Solution

Land PR #1840 or an equivalent focused fix that threads ownerHandle/--owner through the shared skill publish schema, CLI multipart payload, REST JSON and multipart parsing, server-side publisher-role resolution, and regression tests.

## Reproduction Assessment

Yes, from source inspection: REST skill publish validates a schema without ownerHandle and cmdPublish sends no owner field, so an org-owned slug publish falls back to the actor's personal publisher and reaches the slug-taken check.

## Solution Assessment

Yes. The narrow maintainable direction is the ownerHandle/--owner flow already used by the web action pattern, with server-side publisher-role authorization; current main is missing that path for REST/CLI.
