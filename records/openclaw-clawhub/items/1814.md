## Summary

Keep open. Current main still lacks REST/CLI owner selection for org-owned skill version publishing, and linked PR #1840 is still open rather than merged, so this issue remains valid under the ClawHub cleanup policy.

## What This Changes

The issue requests authenticated REST/CLI support for selecting an organization owner when publishing new versions of org-owned skills.

## Best Possible Solution

Land #1840 or an equivalent focused change that threads ownerHandle through shared schema validation, JSON and multipart REST parsing, CLI publish options, server-side publisher-role authorization, and regression coverage.

## Reproduction Assessment

Yes. Source inspection shows the REST/CLI publish path has no owner selector, so org-owned slug updates reach insertVersion without ownerPublisherId and default to the actor's personal publisher.

## Solution Assessment

Yes. Reusing the existing web publish ownerHandle resolution with publisher-role checks is the narrow maintainable direction for closing the REST/CLI gap.
