## Summary

Keep open. This is a ClawHub issue, and current main still lacks the requested REST/CLI owner selector for org-owned skill publishes; PR #1840 remains the open implementation candidate rather than a reason to close before merge.

## What This Changes

Allow REST API and CLI skill publishes to target an org publisher so members can publish new versions of org-owned skills from CI.

## Best Possible Solution

Ship a focused, auth-checked owner selector through the shared skill publish schema, REST JSON and multipart parsing, CLI payload generation, and publishVersionForUser ownerPublisherId path with regression coverage for org-owned updates.

## Reproduction Assessment

Yes. The source path is high-confidence: REST/CLI skill publishes cannot carry ownerHandle today, so publishVersionForUser reaches insertVersion without ownerPublisherId and defaults to the actor's personal publisher, which triggers the slug-taken rejection for an existing org-owned slug.

## Solution Assessment

Yes. The narrow maintainable fix is to add ownerHandle/--owner at the REST and CLI boundary, then resolve the target publisher server-side with publisher-role authorization before publishing; PR #1840 appears to be the current candidate for that direction.
