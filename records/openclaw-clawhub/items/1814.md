## Summary

Keep open. ClawHub cleanup policy keeps issues open, current main still lacks the requested REST/CLI owner selector for skill publishing, and open PR #1840 is the linked implementation candidate rather than a close reason before merge.

## What This Changes

Allow org members with publisher access to publish new versions of org-owned skills through the REST API and CLI, not only the web UI.

## Best Possible Solution

Land PR #1840, or an equivalent focused fix, so ownerHandle/--owner flows through shared publish validation, JSON and multipart REST parsing, CLI payload generation, publisher-role authorization, and regression coverage.

## Reproduction Assessment

Yes. Source inspection shows REST/CLI skill publish payloads contain no owner selector while the server publish core defaults to the actor's personal publisher and rejects an existing org-owned slug as taken.

## Solution Assessment

Yes. The narrow maintainable direction is to reuse the existing web action's ownerHandle publisher-resolution pattern and expose it through REST/CLI with server-side membership checks.
