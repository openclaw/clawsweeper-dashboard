## Summary

Keep open. ClawHub cleanup policy keeps issues open in this lane, and current main still lacks the requested REST/CLI owner selector for org-owned skill publishes; the linked open PR #1840 is the implementation candidate to review.

## What This Changes

Add REST/CLI support for publisher-role org members to publish new versions under an org-owned skill, for example with `--owner` or `ownerHandle`.

## Best Possible Solution

Land PR #1840 or an equivalent focused fix that threads `ownerHandle`/`--owner` through the shared skill publish schema, CLI multipart payload, REST JSON and multipart parsing, server-side publisher-role resolution, and regression tests.

## Reproduction Assessment

Yes. The code path is high-confidence: current REST/CLI skill publish cannot send or preserve an owner target, and `insertVersion` falls back to the actor's personal publisher before rejecting an existing org-owned slug as taken.

## Solution Assessment

Yes. The narrow maintainable fix is to mirror the web publish pattern: accept an owner selector, resolve it server-side with publisher-role authorization, and pass `ownerPublisherId` into the existing publish core.
