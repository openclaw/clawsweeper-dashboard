## Summary

Keep open. ClawHub cleanup policy keeps issues open, current main still lacks the requested REST/CLI owner selector for skill publishing, and open PR #1840 is the linked implementation candidate rather than a close reason before merge.

## What This Changes

Allow authenticated REST and CLI skill publishing to target an org publisher so publisher-role org members can publish new versions from CI.

## Best Possible Solution

Review and land PR #1840, or an equivalent focused fix that threads ownerHandle/--owner through the shared skill publish schema, REST JSON and multipart parsing, CLI payload generation, server-side publisher-role authorization, and regression coverage.

## Reproduction Assessment

Yes. From source inspection, REST/CLI skill publish payloads contain no owner selector while insertVersion defaults to the actor's personal publisher and rejects an existing skill owned by a different publisher.

## Solution Assessment

Yes. The narrow maintainable direction is to reuse the existing ownerHandle publisher-resolution pattern from the web action and expose it through the REST/CLI publish path with server-side role checks.
