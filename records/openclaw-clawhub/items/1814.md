## Summary

Keep open. ClawHub cleanup policy keeps issues open, current main still lacks the requested REST/CLI owner selection, and #1840 is an open implementation candidate rather than a merged fix.

## What This Changes

Allow authenticated REST and CLI publishing of new versions for organization-owned skills by selecting the target org publisher.

## Best Possible Solution

Review and land #1840, or an equivalent focused fix that threads ownerHandle/--owner through the shared schema, REST JSON and multipart parsing, CLI payload generation, server-side publisher-role authorization, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: REST and CLI publish requests still reach publishVersionForUser without ownerPublisherId, so an existing org-owned slug is compared against the actor's personal publisher.

## Solution Assessment

Yes. The narrow maintainable direction is to reuse the web publish action's existing ownerHandle publisher-resolution path with server-side publisher role checks, then expose it through the REST and CLI publish surfaces.
