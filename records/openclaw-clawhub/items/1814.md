## Summary

Keep open. Current main still lacks REST/CLI owner selection for skill publishing, and the open closing PR #1840 is an implementation candidate rather than a reason to close this issue before merge.

## What This Changes

The issue requests REST and CLI support for publishing new versions of org-owned skills by authenticated org members with publisher access.

## Best Possible Solution

Review and land #1840, or an equivalent focused fix that threads ownerHandle/--owner through schema parsing, JSON and multipart REST publish, CLI payload generation, server-side publisher-role authorization, and regression coverage.

## Reproduction Assessment

Yes. Source inspection provides a high-confidence path: REST/CLI publish cannot carry ownerHandle, so publishVersionForUser reaches insertVersion without ownerPublisherId and defaults to the actor's personal publisher.

## Solution Assessment

Yes. The narrow maintainable solution is to reuse the existing web action's ownerHandle publisher-resolution path with server-side publisher-role checks and expose it through the REST and CLI publish surfaces.
