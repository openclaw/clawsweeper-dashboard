## Summary

Keep open. The reporter added a screenshot and bundle file list, and current main still has a publish-blocking path where a character-capped embedding payload can exceed OpenAI's token limit, throw a non-retryable embedding error, and abort version insertion.

## What This Changes

The issue asks ClawHub skill publishing to stop failing with an 8192-token OpenAI embedding error when SKILL.md is small but the bundle contains additional text files.

## Best Possible Solution

Bound publish-time embedding input by a model-safe token budget, or retry known input-length failures with safely truncated text, while preserving scan validation and version insertion for otherwise valid bundles.

## Reproduction Assessment

Yes at source level: a bundle whose included text produces a token-dense 12,000-character embedding input can hit the OpenAI 8192-token limit, and current main converts that error into a publish-blocking ConvexError. I did not live-replay the exact reporter bundle because the bundle contents were not attached.

## Solution Assessment

Not applicable as a proposed patch. The narrow maintainable fix is token-aware truncation or a targeted over-limit retry/fallback in the embedding enrichment path, not changing upload validation or broad publish semantics.
