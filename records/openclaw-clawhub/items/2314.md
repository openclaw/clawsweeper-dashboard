## Summary

Keep open. Current main still has the source-level publish-blocking embedding path, and the linked implementation candidate at https://github.com/openclaw/clawhub/pull/2337 is open and unmerged, so this issue should remain open until that PR lands or is replaced.

## What This Changes

Skill publishing can fail with an 8192-token embedding error even when SKILL.md itself is small.

## Best Possible Solution

Land or replace https://github.com/openclaw/clawhub/pull/2337 with a token- or byte-safe embedding input cap that preserves scan validation and version insertion for valid bundles.

## Reproduction Assessment

Yes at source level: current main builds a 12,000-character embedding string from SKILL.md plus selected text files and sends it to OpenAI, where an input-limit 4xx aborts publish before insertion. I did not live-replay the exact reporter bundle because its contents were not attached.

## Solution Assessment

Yes: the narrow maintainable solution is to bound publish-time embedding input by a provider-safe budget or retry known over-limit failures with safe truncation. The current issue should stay open while the linked PR is reviewed and proven.
