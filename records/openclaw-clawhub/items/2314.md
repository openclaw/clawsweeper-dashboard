## Summary

Keep open. Current main still has a publish path where a character-capped embedding payload can exceed the OpenAI 8192-token limit, the 4xx embedding error is surfaced as a ConvexError before version insertion, and the reporter has screenshots for two affected skills.

## What This Changes

The issue reports skill publishing failing with an OpenAI 8192-token embedding error even though SKILL.md is small.

## Best Possible Solution

Add token-budgeted embedding text construction, or a targeted over-limit retry with safely truncated text, so otherwise valid skill publishes can insert after scans while preserving meaningful embeddings.

## Reproduction Assessment

Yes at source level: current main can build a 12,000-character embedding input from SKILL.md plus non-Markdown files, and OpenAI length-limit 4xx errors are converted into publish-blocking ConvexErrors. I did not live-replay the reporter's exact bundle because the bundle contents were not attached.

## Solution Assessment

Not applicable as a proposed patch. The narrow maintainable fix is to bound publish-time embedding input by token budget or retry known input-length failures with truncated text, not to change upload validation or broad publish semantics.
