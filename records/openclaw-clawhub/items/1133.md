## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Keep not_found VT results queued:** `convex/skills.ts:4147-4148`
  - body: The restore path only keeps absent, pending, or loading VT results queued. Current moderation logic treats `vtAnalysis.status === "not_found"` as pending, so this can publish a skill with an unresolved VT result and remove it from scan selection.
  - confidence: 0.92
- **[P1] Preserve pre-hold moderation locks:** `convex/skills.ts:4124-4129`
  - body: This predicate treats post-hold `user.moderation` as proof that a skill belongs to the lifted hold, but the hold batch rewrites non-malicious hidden rows, including quality or manual hides, to that reason with a fresh `hiddenAt`. Store prior state or skip ambiguous rows so lifting a user hold cannot publish independently quarantined content.
  - confidence: 0.84
- **[P2] Record cumulative restore counts:** `convex/users.ts:813`
  - body: The lift audit records only the first batch's `restoredCount`; when more pages are scheduled, later restores are not reflected in the returned count or `user.moderation.lift` metadata. Accumulate counts or write continuation audit events so large recoveries have a complete trail.
  - confidence: 0.86
