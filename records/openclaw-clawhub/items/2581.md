## Summary

This focused branch appears technically sound and addresses the telemetry durability gaps left after the merged predecessor, but it deliberately changes public discovery from downloads to all-time installs while the canonical semantics issue remains open. Because the author is a repository member and the remaining question is an explicit product/compatibility choice rather than a concrete patch defect, this PR should stay open for maintainer judgment.

## What This Changes

The PR changes recommended and skill-search popularity from downloads to all-time installs, adds daily per-user/skill/root install-event deduplication and pruning, makes telemetry clearing batch-safe around concurrent reinstalls, cleans dedupe state on hard delete, and updates regression tests, API docs, and a durable spec.

## Best Possible Solution

Explicitly approve one discovery metric policy, then land the focused dedupe, pruning, batch-clear, cleanup, cursor, documentation, and spec changes with the chosen ranking semantics and a recorded production upgrade check.
