## What This Changes

The issue reports GitHub import and publish failing after successful repo detection because owner-publisher digest sync hit Convex's multiple-paginated-query limit.

## Best Possible Solution

Keep the cursor-continuation implementation, smoke test the deployed backend with real import and CLI publish flows, then consolidate the duplicate publish-pagination reports.

## Reproduction Assessment

No high-confidence current-main failure was established. The historical report is concrete, but current source and regression tests show the named owner-publisher digest helpers now perform one paginated query per invocation and continue via scheduled cursor calls.

## Solution Assessment

Yes for the source-level fix direction. Cursor continuation is the narrow maintainable solution because it preserves indexed pagination while avoiding Convex's one-paginated-query-per-function limit and the unbounded-read risk of the closed https://github.com/openclaw/clawhub/pull/1183 collect approach.
