## Summary

Keep open under the ClawHub issue policy: current main contains the source-level cursor-continuation fix and regression coverage for the reported Convex pagination failure, but ClawHub issues are not auto-closed by this cleanup pass.

## What This Changes

GitHub import and CLI publish fail at the final publish step because owner-publisher digest sync hits Convex’s single-paginated-query limit.

## Best Possible Solution

Keep the cursor-continuation implementation, smoke test deployed GitHub import and CLI publish for affected accounts, then manually close or consolidate the duplicate publish-pagination reports.

## Reproduction Assessment

No. The historical report is concrete, but current main no longer shows the repeated-paginate source path: each owner-publisher digest helper performs one paginated query per invocation and schedules cursor continuation.

## Solution Assessment

Yes for the source-level fix direction. Cursor continuation is the narrow maintainable solution because it preserves indexed pagination while avoiding Convex’s one-paginated-query-per-function limit and the unbounded-read risk of the closed collect-based proposal.
