## Summary

The explicit install-event direction is useful, but this PR should not merge as written: removing sync reconciliation leaves normal uninstalls and disappeared roots unable to decrement current install state, while the branch also needs conflict resolution and a maintainer decision on ranking policy.

## What This Changes

The PR replaces sync-snapshot install telemetry with explicit `clawhub install` events, removes installs from recommended/search popularity, adds a high-volume install/download close-ratio abuse signal, and updates related CLI, API, schema, tests, docs, and specs.

## Best Possible Solution

Keep explicit install events for trustworthy new-install attribution, but pair them with an uninstall event or snapshot reconciliation that only deactivates missing installs; then split or explicitly resolve the ranking policy before rebasing and validating fresh plus upgrade behavior on real Convex.

## Reproduction Assessment

Yes, at source level: install events create or reactivate active state, sync no longer reports absence, and no ordinary CLI uninstall or reconciliation call remains to trigger deactivation. A real Convex run is still needed to demonstrate the stale counters end to end.

## Solution Assessment

No. Explicit install events are a good attribution source, but fully removing reconciliation makes `installsCurrent` incorrect; the narrow maintainable solution separates new-install counting from current-state removal tracking.
