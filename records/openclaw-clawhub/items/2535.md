## Summary

The PR has strong real-runtime evidence and broad passing validation, but it removes the only normal reconciliation path that marks uninstalled roots inactive, so `installsCurrent` would stop representing current installs. The branch is also conflict-marked and intentionally changes legacy telemetry and public ranking contracts, requiring maintainer direction.

## What This Changes

Moves install counting to explicit `clawhub install` events, makes legacy sync telemetry a no-op, removes installs from search/recommended ranking, and extends temporal publisher-abuse scoring with high install/download-ratio evidence.

## Best Possible Solution

Keep explicit install events for prompt attribution, but preserve a bounded reconciliation mechanism for removals and stale roots, then resolve the branch and obtain explicit maintainer approval for the legacy-client and ranking contracts.

## Reproduction Assessment

Yes for the review defect: source inspection shows that after a successful explicit install creates an active root, ordinary `sync` no longer reports absence and the no-op endpoint cannot invoke removal or stale-root expiry, leaving the current count active.

## Solution Assessment

No, not yet: explicit install events are a sound source for new installs, but replacing reconciliation entirely is not sufficient while the product still exposes a current-install metric; retain a removal signal or redefine that metric comprehensively.
