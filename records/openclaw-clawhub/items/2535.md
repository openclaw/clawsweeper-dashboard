## Summary

Keep open: the current head is clean and demonstrates the new event flow, but it removes the only fallback that deactivates legacy or missed-event installs, so existing `installsCurrent` state can remain stale indefinitely. Likely related people: Peter Steinberger introduced sync-based install telemetry, Patrick-Erichsen recently maintained telemetry and introduced the temporal abuse scan, and jesse-merhi previously introduced the recommended-ranking behavior.

## What This Changes

The PR replaces sync-snapshot install accounting with explicit install/uninstall events, makes legacy sync telemetry a compatibility no-op, removes installs from discovery ranking, and adds a high-volume install/download-ratio abuse signal.

## Best Possible Solution

Keep explicit install/uninstall events as the primary signal, add an indexed and bounded reconciliation path for legacy or missed removal events, and settle discovery ranking through one canonical maintainer-approved policy.

## Reproduction Assessment

Yes. Create an active sync-derived install on current main, upgrade to this branch, then remove the skill without a successful new uninstall telemetry post; source inspection shows no remaining snapshot or stale-root expiry path that can deactivate it.

## Solution Assessment

No. Explicit events are a better primary attribution source, but event-only deactivation is not upgrade-safe for legacy state or removal paths outside this CLI.
