## Summary

The current head is cleanly mergeable and fixes the earlier missing-uninstall concern, but it removes the only automatic expiry path for install state created by existing sync clients. The PR should remain open because legacy current-install counts can become permanently stale after upgrade, and the branch also needs a maintainer choice between its stars-plus-downloads ranking policy and the opposite installs-based policy in the dependent PR.

## What This Changes

The PR replaces sync-snapshot install telemetry with explicit install/uninstall events, removes installs from search and recommended ranking, and adds a high-volume install/download close-ratio publisher-abuse signal.

## Best Possible Solution

Preserve explicit install/uninstall telemetry for new CLI activity, add a cursor-batched migration or retained expiry job that safely deactivates stale legacy sync-derived roots, validate mixed old/new state in a real Convex deployment, and settle one ranking contract for this PR and https://github.com/openclaw/clawhub/pull/2581.

## Reproduction Assessment

Yes at source level: create an active sync-derived root older than 120 days, deploy this branch, and observe that sync is now a no-op and no remaining function expires that root or deactivates its install. The PR still needs a real Convex upgrade fixture to demonstrate the persisted-state result.

## Solution Assessment

No, not yet. Explicit install and uninstall events are a maintainable forward path, but deleting snapshot expiry without migrating legacy rows is not upgrade-safe, and the ranking choice must be reconciled with the dependent PR.
