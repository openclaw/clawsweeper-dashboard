## Summary

Keep open; this patch is not ready because it makes CLI search hide suspicious-but-not-blocked skills by default, while current main just merged the opposite UI search policy to show those results.

## What This Changes

The PR adds a `--include-suspicious` CLI search flag, sends `nonSuspiciousOnly=true` by default, and adds mocked CLI command tests for the default and opt-out request parameters.

## Best Possible Solution

Align CLI search with the current repository-wide suspicious-result policy, or make a deliberate maintainer product decision to change that policy across surfaces with matching docs and tests.

## Reproduction Assessment

Yes, from source inspection: current main web search and skills browse omit `nonSuspiciousOnly`, while this PR adds it to CLI search by default and the backend filters only when that argument is set.

## Solution Assessment

No; the implementation is mechanically small, but it conflicts with current main's freshly merged search policy, so the safer path is to first confirm the intended suspicious-result behavior across CLI and web surfaces.
