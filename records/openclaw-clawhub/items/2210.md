## Summary

Keep this PR open: the full-stack proof direction is useful, but the patch introduces a security-sensitive artifact handling issue and the PR does not include inspectable real behavior proof for the new local Convex runner.

## What This Changes

The PR adds `proof:ui --backend local-convex`, `--dev-auth`, `--env`, and `--seed-command` support, with tests plus skill/spec documentation for full-stack UI proof.

## Best Possible Solution

Keep the local Convex backend mode, but store or delete secret-bearing env files outside collected artifacts, add focused regression coverage for that behavior, and attach redacted terminal/log proof from a real full-stack run.

## Reproduction Assessment

Not applicable: this is a feature PR rather than a bug report. The blocking artifact issue is source-reproducible from the generated script path and the existing artifact copy path.

## Solution Assessment

No: the feature direction is reasonable, but the current implementation should not persist env files containing explicit proof variables inside the copied artifact directory.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Keep proof env files out of copied artifacts:** `scripts/ui-proof.mjs:337-339`
  - body: This writes every `--env` and dev-auth value to `$remote_out/.env.local`. The runner later copies `$remote_out` back wholesale, so a full-stack proof that passes tokens, OAuth credentials, or JWT keys leaves those secrets in the proof artifact directory. Put the env file in a temp path outside `remote_out` or delete it before artifact copy.
  - confidence: 0.88
