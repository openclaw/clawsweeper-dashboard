## Summary

Keep open. Current `main` still uses the shared Convex proof backend, and this PR is not obsolete; however, the proposed patch leaves ways for `proof:ui` to miss the lane-local backend invariant and still lacks inspectable real behavior proof.

## What This Changes

The PR changes `proof:ui` to run every proof lane against lane-local Convex, adds explicit dev-auth/env/seed options, updates tests, and documents the full-stack proof invariant.

## Best Possible Solution

Keep the full-stack proof direction only if maintainers accept the invariant, then block reserved env overrides, isolate stale local Convex env files, post runtime proof, and get CI green before merge.

## Reproduction Assessment

Not applicable: this is a feature PR for maintainer proof tooling, not a bug report with a current-main failing reproduction path. The important behavior is source-verifiable from the current runner and PR diff.

## Solution Assessment

No: the direction is plausible, but the current patch does not fully enforce the lane-local backend invariant and can source stale local deployment state. A safer implementation should reject or protect reserved Convex env keys and isolate `.env.local` alongside `.convex/local/default`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Prevent env overrides of lane-local Convex:** `scripts/ui-proof.mjs:269`
  - body: Because `opts.env` is spread after the fixed lane env, `--env VITE_CONVEX_URL=...`, `--env VITE_CONVEX_SITE_URL=...`, or `--env CONVEX_DEPLOYMENT=...` overrides the lane-local backend in both exports and the Convex env file. That keeps an implicit shared/external backend escape hatch, which contradicts the new always-full-stack invariant; reject reserved keys or merge user env before the fixed lane values.
  - confidence: 0.92
- **[P2] Clear stale local env before waiting:** `scripts/ui-proof.mjs:308-313`
  - body: The candidate lane reuses `$PWD`, but this setup only removes `.convex/local/default` before the wait loop accepts any existing `$app_root/.env.local`. A stale file from a previous local Convex proof can be sourced before the new deployment writes its config, so `convex run --push`, seeding, and the build can target the wrong or missing local deployment; remove or isolate `.env.local` before startup or verify it matches the fresh local config.
  - confidence: 0.86
