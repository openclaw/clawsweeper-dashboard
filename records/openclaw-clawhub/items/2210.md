## Summary

Keep open: current main still uses shared Convex URLs, and this PR is useful but not mergeable while runtime proof is missing and the diff can leak proof env values or bypass lane-local backend isolation.

## What This Changes

The PR changes `proof:ui` to run lane-local Convex for each proof lane, adds dev-auth/env/seed CLI options, expands tests, and documents the full-stack invariant.

## Best Possible Solution

Land a revised full-stack proof runner after reserved backend env keys are protected, generated env/secrets stay out of copied artifacts, checks pass, and redacted runtime proof shows the lane-local Convex path working.

## Reproduction Assessment

Not applicable: this is a feature PR for proof tooling rather than a bug report. I checked current-main behavior, PR-head code, live checks, and PR comments instead of reproducing a failing current-main path.

## Solution Assessment

No: lane-local full-stack proof may be the right direction, but this implementation should not merge while caller env can bypass backend isolation and generated env values can be retained in artifacts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep generated env files out of proof artifacts:** `scripts/ui-proof.mjs:304-305`
  - body: `lane_env_file` is created under `$remote_out`, and `copyRemoteArtifacts` rsyncs that directory back wholesale. With the new `--env KEY=VALUE` option, tokens or private endpoints can land in `.artifacts/clawhub-ui-proof/.../.env.local`; write this file outside the copied artifact tree or scrub it before artifact collection.
  - confidence: 0.9
- **[P2] Reserve the lane-local Convex env keys:** `scripts/ui-proof.mjs:269`
  - body: Because `opts.env` is spread after the lane-local Convex URL/deployment fields, a caller can pass `--env VITE_CONVEX_URL=...` or `--env CONVEX_DEPLOYMENT=...` and silently route proof to a shared or production-like backend. Reject those reserved keys or make the lane-local values win.
  - confidence: 0.87

## Security Review
