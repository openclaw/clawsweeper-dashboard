## Summary

Keep open: the central proof-runner change is not on current main, and this PR still has merge blockers around missing real behavior proof, env/artifact security, and a failing local-auth check.

## What This Changes

The PR changes `proof:ui` to run each lane against lane-local Convex, adds `--dev-auth`, repeated `--env`, and `--seed-command`, updates runner tests, and documents the full-stack proof invariant.

## Best Possible Solution

Revise the PR so reserved backend env keys cannot be overridden, generated env files are written outside copied artifacts or scrubbed, CI is green, and redacted runtime proof shows lane-local Convex startup, push, build, preview, and artifact output before merging the always-full-stack policy.

## Reproduction Assessment

Not applicable: this is a proof-tooling feature PR rather than a bug report. Source inspection confirms current main still uses shared Convex URLs and the PR head changes that behavior.

## Solution Assessment

No: the full-stack direction may be useful, but this implementation is not the best mergeable solution while caller env can bypass backend isolation and generated env values can be copied into artifacts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep generated env files out of proof artifacts:** `scripts/ui-proof.mjs:304-305`
  - body: `lane_env_file` is written under `$remote_out`, and artifact collection rsyncs that directory wholesale. With the new `--env KEY=VALUE` flag, tokens or private endpoints can be retained in `.artifacts/clawhub-ui-proof/.../.env.local`; write this file outside the copied artifact tree or scrub it before artifact collection.
  - confidence: 0.9
- **[P2] Reserve the lane-local Convex environment keys:** `scripts/ui-proof.mjs:269`
  - body: Because `opts.env` is spread after the lane-local Convex URL/deployment fields, a caller can pass `--env VITE_CONVEX_URL=...` or `--env CONVEX_DEPLOYMENT=...` and silently route proof to a shared or production-like backend. Reject those reserved keys or make the lane-local values win.
  - confidence: 0.87
