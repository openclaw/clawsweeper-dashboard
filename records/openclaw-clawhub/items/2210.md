## Summary

Keep open: current main does not implement lane-local full-stack UI proof, and the proposed PR still has blocking env-handling defects plus missing after-fix real behavior proof.

## What This Changes

The branch changes `proof:ui` to start lane-local Convex per proof lane, adds dev-auth/env/seed options, expands tests, and documents the full-stack proof invariant.

## Best Possible Solution

Keep the full-stack proof direction only after the runner reserves lane-local backend keys, keeps env files out of copied artifacts, passes CI, and includes redacted runtime proof.

## Reproduction Assessment

Not applicable: this is a feature PR for proof tooling rather than a bug report. I source-checked current main and the PR head instead of reproducing a failing current-main path.

## Solution Assessment

No in the current form: the branch targets the right runner/docs/tests surface, but it does not preserve the full-stack invariant when reserved env keys are overridden and it can retain env values in copied artifacts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep generated env files out of proof artifacts:** `scripts/ui-proof.mjs:304-305`
  - body: `lane_env_file` is written under `$remote_out`, and `copyRemoteArtifacts` rsyncs that whole directory back into the proof artifacts. With the new repeated `--env KEY=VALUE` option, tokens or private endpoints can be retained in `.artifacts/clawhub-ui-proof/.../.env.local`; write this file outside the copied artifact tree or delete/scrub it before collection.
  - confidence: 0.9
- **[P2] Reserve the lane-local Convex environment keys:** `scripts/ui-proof.mjs:269`
  - body: `opts.env` is spread after the lane-local Convex deployment and URL fields, so `--env VITE_CONVEX_URL=...` or `--env CONVEX_DEPLOYMENT=...` can silently put proof back on a shared or non-lane backend. Reject reserved proof backend keys or apply lane-local values last so the always-full-stack invariant cannot be bypassed.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The diff introduces sensitive-data handling risk by allowing arbitrary env values to be copied into proof artifacts and by allowing backend isolation to be bypassed.

Concerns:

- **[medium] Proof env values can be copied as artifacts:** `scripts/ui-proof.mjs:304`
  - body: The generated lane env file is written inside `$remote_out`, which is later copied back as the proof artifact directory; arbitrary `--env` values may include secrets or private endpoints and should not be retained there.
  - confidence: 0.9
- **[low] User env can bypass local backend isolation:** `scripts/ui-proof.mjs:269`
  - body: Because `opts.env` overrides the lane-local Convex URL/deployment fields, a proof run can be configured to talk to a shared or production-like backend despite the new isolation invariant.
  - confidence: 0.84
