## Summary

This PR should stay open: current main does not already implement the full-stack proof behavior, and the PR head still has concrete env-handling/security defects plus missing contributor runtime proof.

## What This Changes

The PR changes `proof:ui` to start lane-local Convex per lane, add `--dev-auth`, repeated `--env`, and `--seed-command`, expand tests, and document the always-full-stack proof invariant.

## Best Possible Solution

Keep the full-stack proof direction only after the runner reserves lane-local backend env keys, keeps user env values out of copied artifacts, and has redacted runtime proof of the changed path.

## Reproduction Assessment

Not applicable: this is a feature PR for proof tooling rather than a bug report. I source-checked current main and the PR head instead of reproducing a failing current-main path.

## Solution Assessment

Unclear until the env-handling defects are fixed and maintainers accept the workflow change. The branch targets the right runner/docs/tests surface, but the current implementation does not preserve the stated invariant or keep arbitrary proof env values out of artifacts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep generated env files out of proof artifacts:** `scripts/ui-proof.mjs:304-305`
  - body: `lane_env_file` is placed under `$remote_out`, and that whole remote output directory is later copied back as proof artifacts. Since the new `--env KEY=VALUE` flag can carry tokens or private endpoints, those values can be retained in `.artifacts/clawhub-ui-proof/.../.env.local`; write this file outside the copied artifact tree or delete/scrub it before artifact collection.
  - confidence: 0.9
- **[P2] Reserve the lane-local Convex environment keys:** `scripts/ui-proof.mjs:269`
  - body: The user-provided `opts.env` is spread after the lane-local Convex URL/deployment fields, so `--env VITE_CONVEX_URL=...` or `--env CONVEX_DEPLOYMENT=...` can silently put the browser back on a shared or non-lane backend. Reject reserved proof backend keys or apply them after user env so the new always-full-stack invariant cannot be bypassed.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The diff introduces concrete sensitive-data handling risk by persisting arbitrary `--env` values into copied proof artifacts.

Concerns:

- **[medium] Proof env values can be copied as artifacts:** `scripts/ui-proof.mjs:304`
  - body: The generated lane env file is written inside `$remote_out`, which is later copied back as the proof artifact directory; arbitrary `--env` values may include secrets or private endpoints and should not be retained there.
  - confidence: 0.9
- **[low] User env can bypass local backend isolation:** `scripts/ui-proof.mjs:269`
  - body: Because `opts.env` overrides the lane-local Convex URL/deployment fields, a proof run can be configured to talk to a shared or production-like backend despite the new isolation invariant.
  - confidence: 0.84
