## Summary

Keep open: current main does not implement the full-stack UI proof behavior, and this PR still needs real runtime proof plus fixes for env override and artifact hygiene before merge.

## What This Changes

The PR changes `proof:ui` to run each proof lane against lane-local Convex, adds dev-auth/env/seed CLI options, updates tests, and documents the full-stack proof invariant.

## Best Possible Solution

Keep the full-stack proof direction, but land it only after reserved env keys cannot bypass lane-local Convex, generated env files stay out of proof artifacts, and real runtime proof is posted.

## Reproduction Assessment

Not applicable: this is a feature PR for maintainer proof tooling, not a bug with a failing current-main reproduction path. The current behavior and proposed behavior are source-verifiable from the runner code and PR diff.

## Solution Assessment

No: the PR targets the right runner/docs/tests surface, but it should reject or protect reserved proof env keys and keep generated env files outside synced artifacts before the workflow change is maintainable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Prevent --env from overriding lane backend URLs:** `scripts/ui-proof.mjs:269`
  - body: `laneLocalConvexEnv` spreads `opts.env` after the lane-local values, so a caller can pass `--env VITE_CONVEX_URL=...`, `--env VITE_CONVEX_SITE_URL=...`, or `--env CONVEX_DEPLOYMENT=...` and still run against a shared backend. That contradicts the new always-full-stack invariant; reject reserved proof env keys or ensure the lane-local values win.
  - confidence: 0.88
- **[P2] Keep generated env files out of proof artifacts:** `scripts/ui-proof.mjs:304-305`
  - body: The lane env file is written under `$remote_out`, and the lane later rsyncs that directory back into `.artifacts/clawhub-ui-proof`. Any value passed through the new generic `--env` flag is therefore persisted in proof outputs; put this file outside `$remote_out` or delete it before artifact copy.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: Needs attention: the new generic proof env path can persist caller-provided env values inside synced proof artifacts.

Concerns:

- **[medium] Env values can be copied into proof artifacts:** `scripts/ui-proof.mjs:304`
  - body: `lane_env_file` is placed in `$remote_out`, which is recursively copied back as the lane proof output. Because `--env` is generic, this can preserve auth or private values in `.artifacts/clawhub-ui-proof`; the env file should live outside the synced artifact tree or be removed before copy.
  - confidence: 0.84
