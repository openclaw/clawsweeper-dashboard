## Summary

Keep open. The PR is not implemented on current main, and the proposed patch still has blocking env-handling/security defects plus missing real behavior proof for the changed proof runner path.

## What This Changes

The PR changes `proof:ui` to run lane-local Convex per lane, adds `--dev-auth`, repeated `--env`, and `--seed-command`, and updates proof tests, skill docs, and specs.

## Best Possible Solution

Land the full-stack proof direction only after reserved backend env keys are protected, generated env values are kept out of copied artifacts, CI is clean, and redacted runtime proof shows the lane-local Convex path working.

## Reproduction Assessment

Not applicable: this is a feature PR for proof tooling rather than a bug report. I source-checked current main and the PR head instead of reproducing a failing current-main path.

## Solution Assessment

Unclear: the full-stack proof direction is plausible, but this implementation is not ready because env overrides can bypass lane isolation and generated env values can be copied into artifacts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Move generated env files out of copied artifacts:** `scripts/ui-proof.mjs:304-305`
  - body: `lane_env_file` is written under `$remote_out`, and that directory is later rsynced back as proof artifacts. With the new repeated `--env` flag, tokens or private endpoints can be retained in `.artifacts/clawhub-ui-proof/.../.env.local`; write this file outside the copied tree or scrub it before artifact collection.
  - confidence: 0.9
- **[P2] Reserve lane-local Convex environment keys:** `scripts/ui-proof.mjs:269`
  - body: Spreading `opts.env` after the lane-local Convex values lets `--env VITE_CONVEX_URL=...` or `--env CONVEX_DEPLOYMENT=...` route proof away from the lane backend. Reject reserved backend keys, or ensure the lane-local values win, so the always-full-stack invariant cannot be bypassed.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The patch introduces concrete sensitive-data and backend-isolation risks in proof env handling.

Concerns:

- **[medium] Proof env values can be copied as artifacts:** `scripts/ui-proof.mjs:304`
  - body: The generated lane env file is written under `$remote_out`, which is copied back wholesale; arbitrary `--env` values may include secrets or private endpoints and should not be retained in proof artifacts.
  - confidence: 0.9
- **[low] User env can bypass lane-local backend isolation:** `scripts/ui-proof.mjs:269`
  - body: Because `opts.env` overrides the lane-local Convex URL/deployment fields, a proof run can be configured to use a shared or production-like backend despite the new isolation invariant.
  - confidence: 0.84
