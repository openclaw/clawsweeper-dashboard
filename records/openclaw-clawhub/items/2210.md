## Summary

Keep this PR open: current main does not implement the full-stack proof-runner behavior, and the branch has useful work, but the patch needs fixes for proof env handling plus contributor-supplied real behavior proof before it is merge-ready.

## What This Changes

The PR changes `proof:ui` to start lane-local Convex per proof lane, add `--dev-auth`, repeated `--env`, and `--seed-command`, expand tests, and document the full-stack proof invariant.

## Best Possible Solution

Keep the full-stack proof direction only if the runner preserves lane-local backend env, keeps user env secrets out of copied artifacts, and maintainers accept the always-full-stack invariant with runtime proof.

## Reproduction Assessment

Not applicable: this is a feature PR for proof tooling rather than a bug report. I source-checked current main and the PR head instead of reproducing a failing current-main path.

## Solution Assessment

Unclear until the env-handling defects are fixed and maintainers accept the workflow change. The branch targets the right runner/docs/tests surface, but the current implementation does not fully preserve the stated full-stack invariant or keep arbitrary proof env values out of artifacts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep generated env files out of proof artifacts:** `scripts/ui-proof.mjs:304-305`
  - body: `lane_env_file` is placed under `$remote_out`, and that whole remote output directory is later copied back as proof artifacts. Since the new `--env KEY=VALUE` flag can carry tokens or private endpoints, those values can be retained in `.artifacts/clawhub-ui-proof/.../.env.local`; write this file outside the copied artifact tree or delete/scrub it before artifact collection.
  - confidence: 0.9
- **[P2] Reserve the lane-local Convex environment keys:** `scripts/ui-proof.mjs:269`
  - body: The user-provided `opts.env` is spread after the lane-local Convex URL/deployment fields, so `--env VITE_CONVEX_URL=...` or `--env CONVEX_DEPLOYMENT=...` can silently put the browser back on a shared or non-lane backend. Reject reserved proof backend keys or apply them after user env so the new always-full-stack invariant cannot be bypassed.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The diff introduces a concrete sensitive-data handling risk by persisting arbitrary `--env` values into copied proof artifacts.
