## Summary

Keep open: current main does not implement the lane-local full-stack proof invariant, and this PR is useful but not merge-ready because the current head still has security-sensitive env handling defects, missing real behavior proof, and a failing `playwright-local-auth` check.

## What This Changes

The PR changes `proof:ui` to start lane-local Convex per lane, adds `--dev-auth`, repeated `--env`, and `--seed-command`, expands runner tests, and documents the full-stack proof invariant.

## Best Possible Solution

Land a revised full-stack proof runner only after reserved backend env keys are protected, env files or secrets are kept out of artifacts, checks pass, and redacted runtime proof shows the lane-local Convex path working.

Not applicable: this is a feature PR for proof tooling rather than a bug report. I source-checked current main, the PR head, live checks, and PR comments instead of reproducing a failing current-main path.

## Solution Assessment

No: the direction may be useful, but the current implementation is not the best mergeable solution while reserved env keys can bypass lane-local backends and generated env values can be copied into artifacts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep generated env files out of proof artifacts:** `scripts/ui-proof.mjs:304-305`
  - body: `lane_env_file` is written under `$remote_out`, and artifact collection rsyncs that directory wholesale. With the new `--env KEY=VALUE` flag, secrets or private endpoints can be retained in `.artifacts/clawhub-ui-proof/.../.env.local`; move this file outside the copied artifact tree or scrub it before collection.
  - confidence: 0.9
- **[P2] Reserve lane-local Convex environment keys:** `scripts/ui-proof.mjs:269`
  - body: Because `opts.env` is spread after the lane-local Convex URL/deployment fields, a caller can pass `--env VITE_CONVEX_URL=...` or `--env CONVEX_DEPLOYMENT=...` and silently route proof away from its lane-local backend. Reject those reserved keys or make the lane-local values win.
  - confidence: 0.87

## Security Review
