## Summary

Keep open: current main does not include the lane-local full-stack proof behavior, but this PR is not merge-ready because real runtime proof is missing and the env/artifact handling still undermines the isolation and security invariant.

## What This Changes

The PR changes `proof:ui` so each proof lane starts and uses lane-local Convex, adds explicit dev auth/env/seed options, expands runner tests, and documents the full-stack proof invariant.

## Best Possible Solution

Land a revised full-stack proof runner after reserved backend env keys are protected, generated env files are kept out of copied artifacts, CI is green, and redacted runtime proof shows the lane-local Convex path working.

## Reproduction Assessment

Not applicable: this is a proof-tooling feature PR rather than a bug report. Source inspection shows current main still uses shared Convex URLs, and the PR head changes that behavior.

## Solution Assessment

No: the direction may be useful, but this implementation is not the best mergeable solution while `--env` can override reserved backend keys and generated env values can be copied into proof artifacts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep generated env files out of proof artifacts:** `scripts/ui-proof.mjs:304-305`
  - body: `lane_env_file` is written under `$remote_out`, and artifact collection later rsyncs that directory wholesale. With the new `--env KEY=VALUE` flag, tokens or private endpoints can be retained in `.artifacts/clawhub-ui-proof/.../.env.local`; write this file outside the copied artifact tree or scrub it before artifact collection.
  - confidence: 0.9
- **[P2] Reserve the lane-local Convex environment keys:** `scripts/ui-proof.mjs:269`
  - body: Because `opts.env` is spread after the lane-local Convex URL/deployment fields, a caller can pass `--env VITE_CONVEX_URL=...` or `--env CONVEX_DEPLOYMENT=...` and silently route proof to a shared or production-like backend. Reject those reserved keys or ensure the lane-local values always win.
  - confidence: 0.87

## Security Review
