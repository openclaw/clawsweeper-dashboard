## Summary

Keep open: current main still uses the shared Convex proof backend, while this PR adds the lane-local full-stack path but is not merge-ready because real behavior proof is missing and the diff has env-isolation and artifact-retention blockers.

## What This Changes

The PR changes `proof:ui` so each proof lane starts lane-local Convex, adds `--dev-auth`, repeated `--env`, and `--seed-command`, expands runner tests, and documents the full-stack proof invariant.

## Best Possible Solution

Land a revised full-stack proof runner that protects reserved backend keys, keeps generated env files out of copied artifacts, preserves the documented lane modes, and includes redacted runtime proof.

## Reproduction Assessment

Not applicable: this is a proof-tooling feature PR rather than a bug report. Source inspection shows current main still uses shared Convex URLs and the PR head changes that behavior.

## Solution Assessment

No: the direction may be useful, but this implementation is not the best mergeable solution while `--env` can override reserved backend keys and generated env values can be copied into artifacts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep generated env files out of proof artifacts:** `scripts/ui-proof.mjs:304-305`
  - body: `lane_env_file` is written under `$remote_out`, and artifact collection later rsyncs that directory wholesale. With the new `--env KEY=VALUE` flag, tokens or private endpoints can be retained in `.artifacts/clawhub-ui-proof/.../.env.local`; write this file outside the copied artifact tree or scrub it before artifact collection.
  - confidence: 0.9
- **[P2] Reserve the lane-local Convex environment keys:** `scripts/ui-proof.mjs:269`
  - body: Because `opts.env` is spread after the lane-local Convex URL/deployment fields, a caller can pass `--env VITE_CONVEX_URL=...` or `--env CONVEX_DEPLOYMENT=...` and silently route proof to a shared or production-like backend. Reject those reserved keys or ensure the lane-local values always win.
  - confidence: 0.87
