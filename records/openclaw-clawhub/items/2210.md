## Summary

Keep open: current main still uses the shared-backend UI proof runner, and this PR remains a live feature/policy change. The patch also needs real behavior proof and has a correctness blocker where explicit env overrides can bypass the proposed lane-local Convex invariant.

## What This Changes

The branch changes `proof:ui` to run each lane against lane-local Convex, adds `--dev-auth`, `--env`, and `--seed-command`, updates UI-proof tests, and documents the full-stack proof invariant.

## Best Possible Solution

Enforce lane-local Convex and site env values as reserved, add regression coverage for reserved `--env` overrides, then land only after maintainer acceptance and posted runtime proof.

## Reproduction Assessment

Not applicable: this is a feature PR for maintainer proof tooling, not a bug report with a failing current-main reproduction path. The env override blocker is source-reproducible from the PR head because `opts.env` is spread after the generated lane-local URLs.

## Solution Assessment

No, not as currently patched. The direction targets the right runner/docs/tests surface, but the implementation should prevent `--env` from overriding lane-owned Convex/site variables before the full-stack invariant is reliable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep lane-owned URLs from being overridden:** `scripts/ui-proof.mjs:269`
  - body: The new generic `--env` map is spread after the generated lane-local Convex and site values, so a caller can pass `--env VITE_CONVEX_URL=...` or `--env VITE_CONVEX_SITE_URL=...` and build/run proof against an external backend again. That reintroduces the shared-backend path this PR is meant to remove; reject reserved keys or make the lane-owned values take precedence.
