## Summary

Keeping this open: it is a valid unresolved feature request, and ClawHub cleanup policy keeps issues open unless a PR is certainly implemented on main. Current main can model soft-deleted published versions and hides them from public readers/downloads, but the exposed CLI and HTTP write paths still only delete or restore whole skills/packages; the broader policy work is already centered on https://github.com/openclaw/clawhub/issues/905.

## What This Changes

Request an owner-facing command or API to deprecate, hide, or delist one published version without deleting the rest of its history.

## Best Possible Solution

Use https://github.com/openclaw/clawhub/issues/905 as the canonical lifecycle-policy thread, then add authenticated version-scoped owner/staff API and CLI support once permissions, latest-version, restore, audit, cache, and naming semantics are settled.
