## Summary

Keep open: this is a valid unresolved feature request, and ClawHub cleanup policy keeps issues open for maintainer handling. Current main can model and gate soft-deleted skill versions, but the exposed owner CLI/API/docs still only manage whole skills, with the broader lifecycle policy already tracked in https://github.com/openclaw/clawhub/issues/905.

## What This Changes

Requested owner-facing CLI/API support to deprecate, hide, or delist an individual published skill version without affecting the rest of the skill history.

## Best Possible Solution

Use https://github.com/openclaw/clawhub/issues/905 as the canonical lifecycle-policy thread, then add authenticated version-scoped owner/org-member/staff API and CLI support with consistent latest/tag, cache, download, restore, audit, and docs behavior.

## Solution Assessment

Unclear until maintainers settle the broader lifecycle policy. A version-scoped owner API plus CLI is directionally right, but implementing this independently would skip latest-version, tag, cache, restore, permission, and audit decisions.
