## Summary

Keep open. This is a current feature request: main has internal staff-only version soft-delete plumbing and public readers honor it, but there is no owner-facing version delist/deprecate CLI, HTTP API, or documented workflow; open #905 also tracks the broader same lifecycle gap, and ClawHub issue policy keeps issues open.

## What This Changes

Add an owner-facing command or API to delist, deprecate, hide, or remove a specific published skill version without deleting the whole skill history.

## Best Possible Solution

Triage this with open #905, define the version lifecycle policy, then add authenticated owner/staff CLI, HTTP API, docs, tests, and audit-preserving restore behavior consistently.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection on current main gives high confidence that internal version soft-delete support exists but no owner-facing lifecycle command or write API exists.

## Solution Assessment

Unclear until maintainers settle product policy. A version-scoped command/API matches the request directionally, but the maintainable solution needs explicit authorization, latest-version, restore, audit, and documentation semantics first.
