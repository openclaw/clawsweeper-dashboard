## Summary

Keep open. This ClawHub issue requests an owner-facing version lifecycle feature, and current main still does not expose a publisher CLI/API/docs workflow to delist, hide, or deprecate one published version.

## What This Changes

## Best Possible Solution

Define a version-scoped lifecycle policy, then expose authenticated owner/org-member and staff controls consistently across CLI, HTTP API, docs, public readers, audit logs, and restore behavior.
