## Summary

Keep open: this is a valid unresolved feature request, and this repository’s cleanup policy keeps ClawHub issues open for maintainer handling. Current main already has internal per-version soft-delete state and public read gates, but it still lacks an owner-facing CLI/API workflow to delist, deprecate, hide, or restore one published version.

## What This Changes

The issue requests a CLI or API command to delist, deprecate, or hide a single published version without affecting the rest of the skill history.

## Best Possible Solution

Use https://github.com/openclaw/clawhub/issues/905 as the canonical lifecycle-policy thread, then add authenticated version-scoped owner/staff API and CLI support once permissions, latest/tag behavior, restore semantics, audit logging, and cache handling are settled.

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence in the gap because current main models and gates soft-deleted versions but exposes only skill-scoped owner lifecycle controls.

## Solution Assessment

Unclear until maintainers settle the broader lifecycle policy in https://github.com/openclaw/clawhub/issues/905. A version-scoped owner API plus CLI is directionally right, but implementing it independently would skip latest-version, tag, restore, audit, cache, and permission decisions.
