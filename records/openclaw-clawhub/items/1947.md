## Summary

Keep open: this is a valid unresolved feature request, and ClawHub policy keeps issues open for maintainer handling. Current main can model and hide soft-deleted skill versions, but owner-facing CLI/API lifecycle controls are still skill-scoped, while the broader per-version lifecycle policy is already tracked at https://github.com/openclaw/clawhub/issues/905.

## What This Changes

The issue requests CLI/API support to deprecate, hide, or delist one published version without deleting the whole skill history.

## Best Possible Solution
