## Summary

Keep open: this is an unresolved feature request that overlaps the open lifecycle-policy thread at https://github.com/openclaw/clawhub/issues/905. Current main can model and hide soft-deleted versions from public reads/downloads, but it still lacks owner-facing version-scoped delist/deprecate CLI or HTTP write paths.

## What This Changes

Add a command or API workflow to deprecate, hide, or delist a specific published version without removing the rest of the skill history.

## Best Possible Solution

Use https://github.com/openclaw/clawhub/issues/905 as the canonical policy thread, then add authenticated version-scoped owner/staff API and CLI support with consistent public access, audit, restore, tag/latest, docs, and tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence in the gap because current main models hidden versions and hides them from readers, but exposes only skill-scoped owner lifecycle controls.

## Solution Assessment

Unclear until maintainers settle the broader lifecycle policy. A version-scoped owner API plus CLI is directionally right, but implementing it independently would skip latest-version, tag, restore, audit, cache, and permission decisions.
