## Summary

Keep open. ClawHub issue policy does not auto-close issues, and current main still lacks an owner-facing version delist/deprecate workflow; it only has internal staff soft-delete plumbing plus public read filters, while #905 remains open for the broader same lifecycle gap.

## What This Changes

Request an owner-facing CLI/API workflow to deprecate, hide, or delist a specific published skill version without deleting the whole skill.

## Best Possible Solution

Triage this with open #905, define the version lifecycle policy, then implement version-scoped CLI/API/docs/tests with audit-preserving restore behavior.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection of current main confirms there is no version-scoped owner CLI/API/docs workflow.

## Solution Assessment

Unclear until maintainers settle product policy. A version-scoped command/API fits the request, but authorization, latest-version, restore, and audit semantics need explicit design first.
