## Summary

Keep open. The requested owner-facing version delist/deprecate workflow is still missing on current main; current code has internal staff-only skill-version soft-delete plumbing and public readers honor that state, while open #905 tracks the broader same lifecycle gap.

## What This Changes

Request an owner-facing command or API to deprecate, hide, or delist an individual published version without deleting the whole skill.

## Best Possible Solution

Use #905 as the canonical lifecycle discussion, define owner/staff semantics, then implement version-scoped API, CLI, docs, tests, audit logging, and restore behavior consistently.

## Solution Assessment
