## Summary

Keep open: this is a valid feature request and ClawHub issue policy keeps issues open for maintainer handling. Current main can represent and publicly suppress soft-deleted skill versions, but owner-facing CLI/API/docs still only expose skill-level delete, hide, undelete, and unhide; the broader policy thread is https://github.com/openclaw/clawhub/issues/905.

## What This Changes

The issue requests an owner-facing CLI/API command to deprecate, delist, hide, or otherwise withdraw a single published skill version without removing the whole skill.

## Best Possible Solution

Use https://github.com/openclaw/clawhub/issues/905 as the canonical lifecycle policy thread, then add authenticated owner/org-member/staff version-scoped controls that update skillVersions.softDeletedAt, write version-specific audit records, and keep lists, detail/file/download endpoints, tags/latest, docs, and tests consistent.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence in the current gap because version soft-delete state and public gates exist, while owner-facing per-version lifecycle controls are absent.

## Solution Assessment

Unclear until maintainers settle the broader lifecycle policy. A version-scoped owner API plus CLI is directionally appropriate, but the maintainable solution must define latest, tag, cache, download, restore, permission, and audit semantics first.
