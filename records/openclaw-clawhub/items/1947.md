## Summary

Keep open. This remains a valid feature request: current main has data-model and internal staff/moderator support for soft-deleting skill versions, but no owner-facing CLI or HTTP workflow to deprecate, hide, or delist one published version. It also substantially overlaps open #905, and repository policy keeps ClawHub issues open.

## What This Changes

Request an owner-facing command or API to deprecate, hide, or delist a single published version without deleting the rest of the skill history.

## Best Possible Solution

Use #905 as the canonical lifecycle-policy thread, then add a version-scoped owner API and CLI for non-latest versions with clear restore and audit behavior.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection shows read-only public version endpoints and internal staff-only version soft-delete support, but no owner-facing version-scoped CLI/API workflow.

## Solution Assessment

Unclear until maintainers settle lifecycle policy. A version-scoped owner API plus CLI command matches the request directionally, but the broader #905 policy questions should be resolved first.
