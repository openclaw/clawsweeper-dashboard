## Summary

Keep open. Current main still lacks an owner-facing command or public API to delist/deprecate one published skill version; existing support is limited to skill-level hide/delete plus internal moderator/admin-only version soft-delete plumbing, and #905 remains the broader open lifecycle tracker.

## What This Changes

Request an owner-facing `clawhub version deprecate/delist` style command and supporting API to hide or deprecate a single published skill version without deleting the whole skill.

## Best Possible Solution

Use #905 as the canonical lifecycle policy discussion, then add version-scoped API, CLI, docs, tests, audit logging, and restore/deprecation semantics once maintainers settle the rules.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection on current main verifies there is no owner-facing version-scoped write route or CLI command.

## Solution Assessment

Unclear until maintainers settle lifecycle policy. A version-scoped API plus CLI command matches the request directionally, but latest-version handling, restore behavior, deprecation wording, and owner versus staff permissions should be defined first.
