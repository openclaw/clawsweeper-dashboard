## Summary

Keep open: this is a valid feature request, and ClawHub cleanup policy keeps issues open. Current main can represent and gate a withdrawn individual version, but owner-facing API, CLI, docs, and management controls still only expose skill-level delete/hide/undelete/unhide, with the broader lifecycle policy already tracked in https://github.com/openclaw/clawhub/issues/905.

## What This Changes

The issue requests a CLI or API workflow to deprecate, hide, or delist one published skill version without deleting the whole skill.

## Best Possible Solution

Consolidate product/security policy in https://github.com/openclaw/clawhub/issues/905, then implement authenticated owner/org-member/staff version-scoped controls that update `skillVersions.softDeletedAt`, write version-specific audit records, and keep list/detail/file/download/tag/latest behavior consistent.

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence in the gap because current main models and gates withdrawn versions but exposes owner lifecycle controls only at the skill level.

Unclear until maintainers settle the broader lifecycle policy in https://github.com/openclaw/clawhub/issues/905.
