## What This Changes

Request an owner-facing CLI/API workflow to deprecate, hide, or delist a specific published version without affecting the rest of the skill history.

## Best Possible Solution

Use https://github.com/openclaw/clawhub/issues/905 as the lifecycle-policy thread, then add authenticated version-scoped owner and staff controls that update skillVersions.softDeletedAt with consistent CLI, API, docs, audit, latest/tag, restore, list, detail, file, and download behavior.

## Solution Assessment
