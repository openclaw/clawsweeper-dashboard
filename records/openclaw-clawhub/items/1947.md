## Summary

Repository policy keeps ClawHub issues open, and this remains an unresolved feature request. Current main has internal moderator/admin soft-delete support for skill versions and public reads hide soft-deleted versions, but there is no owner-facing version-scoped CLI or HTTP lifecycle workflow; the request also overlaps open #905.

## What This Changes

The issue requests an owner-facing CLI/API command to deprecate, hide, or delist one published version without affecting the rest of the skill.

## Best Possible Solution

Use #905 as the canonical lifecycle-policy thread, then implement a version-scoped owner API and CLI once permission, latest-version, restore, and audit semantics are settled.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection on current main shows read-only public version endpoints and internal staff-only version soft-delete support, but no owner-facing version-scoped CLI/API workflow.

## Solution Assessment

Unclear until maintainers settle the broader lifecycle policy in #905. A version-scoped owner API plus CLI matches the request directionally, but implementing it independently would skip latest-version, restore, audit, and permission decisions.
