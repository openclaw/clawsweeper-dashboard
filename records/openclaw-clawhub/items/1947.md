## Summary

Keep open: this is a valid unresolved feature request, and ClawHub cleanup policy keeps issues open for maintainer handling. Current main can represent and gate soft-deleted skill versions, but the exposed owner CLI/API lifecycle remains whole-skill scoped, while the broader lifecycle policy is already being discussed at https://github.com/openclaw/clawhub/issues/905.

## What This Changes

The issue requests owner-facing CLI/API support to deprecate, delist, hide, or withdraw one published skill version without removing the whole skill.

## Best Possible Solution

Use https://github.com/openclaw/clawhub/issues/905 as the canonical lifecycle-policy thread, then add authenticated version-scoped owner/staff API and CLI support that updates skillVersions.softDeletedAt, records version-specific audit events, and keeps list/detail/file/download/tag/latest behavior consistent.

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence in the gap because current main models and gates soft-deleted versions but exposes only skill-scoped owner lifecycle controls.

## Solution Assessment
