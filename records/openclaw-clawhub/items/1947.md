## Summary

Keep open: this is a valid feature request, and ClawHub review policy keeps issues open for maintainer handling. Current main can represent and publicly suppress withdrawn individual versions, but owner-facing CLI, HTTP API, docs, and UI lifecycle controls are still skill-scoped and need product/security policy.

## What This Changes

The request asks for owner-facing CLI/API support to deprecate, hide, or delist an individual published skill version without affecting the rest of the skill history.

## Best Possible Solution

Use https://github.com/openclaw/clawhub/issues/905 as the canonical lifecycle policy thread, then add authenticated owner/org-member/staff version-scoped controls with audit logs and consistent public gating.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence in the current gap because version soft-delete state and public gates exist, while owner-facing per-version lifecycle controls are absent.

## Solution Assessment

Unclear until maintainers settle lifecycle policy. A version-scoped owner API plus CLI is directionally appropriate, but the maintainable solution must define latest, tag, download, cache, restore, permission, and audit semantics first.
