## Summary

Keep open. This is a feature request for an owner-facing version delist/deprecate workflow; current main only has internal staff-only version soft-delete plumbing and skill-level owner delete/undelete commands, and ClawHub cleanup policy keeps issues open.

## What This Changes

Add a CLI/API workflow to deprecate, hide, or delist one published skill version without removing the rest of the skill history.

## Best Possible Solution

Consolidate policy with #905, then implement a version-scoped lifecycle flow across Convex mutations, HTTP API, CLI commands, docs, audit logs, and tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main has internal version soft-delete support but no owner-facing version lifecycle command or write API.

## Solution Assessment

Unclear until maintainers settle the product policy. A version-scoped command/API is directionally aligned, but the maintainable solution should define authorization, latest-version constraints, restore semantics, docs, and audit logging first.
