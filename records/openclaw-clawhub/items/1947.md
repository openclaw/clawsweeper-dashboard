## Summary

Keep open: this is a valid feature request, and ClawHub policy keeps issues open. Current main has internal staff/moderator version soft-delete support, but no owner-facing API or CLI command for deprecating, hiding, or delisting one published version; the remaining work overlaps open #905.

## What This Changes

Request an owner-facing command or API to deprecate, hide, or delist a single published version without deleting the whole skill.

## Best Possible Solution

Use #905 as the canonical lifecycle-policy thread, then add an owner-scoped version API and CLI for non-latest versions once restore, audit, and permission semantics are settled.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection on current main shows read-only public version endpoints and internal staff-only version soft-delete support, but no owner-facing version-scoped CLI/API workflow.

## Solution Assessment

Unclear until maintainers settle lifecycle policy. A version-scoped owner API plus CLI command matches the request directionally, but the broader #905 policy questions should be resolved first.
