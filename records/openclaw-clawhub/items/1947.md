## Summary

## What This Changes

Request a command to deprecate, hide, or delist one published skill version without removing the rest of the package history.

## Best Possible Solution

Fold this into #905 or implement a shared owner/staff skill-version lifecycle API and CLI command with audit logging, docs, tests, and explicit latest/tag behavior.

Not applicable as a feature request. Current-main inspection gives a high-confidence gap check: skill version records can be hidden, but the public CLI/API write surface has no single-version delist/deprecate command.

## Solution Assessment
