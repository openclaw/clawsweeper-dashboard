- **[P1] Import or reuse the API-token auth helper:** `convex/httpApiV1/starsV1.ts:64`
  - body: `requireApiTokenUser` is called here but is never imported in this file, so the branch will fail TypeScript/Convex build before the new endpoint can run.   - body: This added import redeclares `internalMutation`, `mutation`, and `query` from the import on line 4, which creates duplicate identifiers. Combine the imports into one `./functions` import that also includes `internalQuery`.
  - confidence: 0.97
- **[P1] Export the shared stars-list schema:** `packages/schema/src/schemas.test.ts:15`
  - body: The new shared schema test imports `ApiV1StarsListResponseSchema`, but the PR head does not export that schema from `packages/schema/src/schemas.ts`; it only appears in the CLI package copy. This makes the shared schema tests fail to compile until the schema is added to the shared package too.
  - confidence: 0.94
- **[P2] Do not turn non-auth listing failures into Unauthorized:** `convex/httpApiV1/starsV1.ts:91-92`
  - body: This broad catch wraps query, tag resolution, and serialization failures as `401 Unauthorized`, so authenticated callers would see a misleading auth error for server-side failures such as read-limit or schema problems. Keep 401s limited to auth failures and return an appropriate non-auth error for operational failures.
  - confidence: 0.82
