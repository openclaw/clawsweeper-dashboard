## Summary

Keep open: the contributor added useful after-fix terminal proof, but the patch still makes the shared public soft-delete fallback echo arbitrary unexpected 500 exception text back to clients.

## What This Changes

The PR changes the HTTP API v1 soft-delete error mapper and one handler test so whitelisted validation messages return 400 and unknown 500 bodies append the cleaned underlying error text.

## Best Possible Solution

Return cleaned messages only for explicit 4xx validation, auth, and not-found cases; keep unexpected 500 bodies generic and cover both the invalid package delete case and generic fallback in tests.

## Reproduction Assessment

Yes. Current main source shows `normalizePackageName` throws the relevant package-name `ConvexError`, while `softDeleteErrorToResponse` only maps `slug required` to 400 and otherwise falls through to a generic 500; the contributor's before/after terminal proof also matches that path.

## Solution Assessment

No as written. The whitelisted 400 mapping is the right narrow direction, but the unknown 500 branch should remain generic instead of returning arbitrary cleaned exception text.
