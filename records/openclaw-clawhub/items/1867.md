## Summary

Keep open: current main does not implement device-flow login, and this PR still enables a user-facing CLI path whose server endpoints, verification page, and auth controls are not present in the branch. The earlier double body-read bug was fixed in the PR head, but the remaining gap is end-to-end auth design and review.

## What This Changes

Adds a `--device` login mode to the ClawHub CLI, a device-auth helper that calls `/cli/device/code` and `/cli/device/token`, and unit tests for that helper.

## Best Possible Solution

Land device flow as a complete rollout covering CLI flags, Convex HTTP endpoints, verification UI, token issuance controls, docs, changelog, and protocol tests.

## Reproduction Assessment

Yes. Source inspection shows the branch posts to `/cli/device/code` and `/cli/device/token`, while current main exposes no matching HTTP routes or verification page.

## Solution Assessment

No. Enabling the CLI flag before backend support is not the best merge path; complete the server/UI/security controls first or keep the CLI path unavailable.
