## Summary

Keep open: current main does not implement device-flow auth, and this PR remains a client-only feature without the matching device-code endpoints, verification UI, security controls, or changelog entry. The earlier double JSON-read review issue appears fixed in the latest PR head.

## What This Changes

Adds a `--device` login mode to the ClawHub CLI, a `deviceAuth` request/polling helper, and unit tests for client-side device-flow authentication.

## Best Possible Solution

Complete #613 as an end-to-end device-flow rollout covering CLI flags, Convex HTTP endpoints, the `/device` verification UI, token issuance controls, docs, changelog, and protocol/security tests.

## Reproduction Assessment

Yes. Source inspection is enough: the PR adds a `--device` flag that posts to `/cli/device/code` and `/cli/device/token`, while current main and the PR file list do not add those routes or a `/device` verification page.

## Solution Assessment

No. The client helper is useful, but enabling the flag before the server and verification surface exists is not the narrowest maintainable solution for a security-sensitive auth flow.
