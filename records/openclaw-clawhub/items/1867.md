## Summary

Keep open. Current main does not implement device flow auth, and this PR still exposes a client-side `--device` path before the required server endpoints, verification UI, token issuance controls, and changelog entry exist.

## What This Changes

Adds a ClawHub CLI `--device` login mode with device-code request/polling helpers and unit tests.

## Best Possible Solution

Land device flow as an end-to-end feature across CLI, Convex HTTP routes, authenticated verification UI, token issuance controls, docs, changelog, and regression tests.

## Reproduction Assessment

Yes for the review findings. Source inspection shows the PR calls `/cli/device/code` and `/cli/device/token`, while current main has no matching route or verification page.

## Solution Assessment

No. A client-only flag is not the best fix; the maintainable path is to ship the CLI entry point together with the backend verification and token-exchange controls, or keep it hidden until those exist.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Add server support before exposing device login:** `packages/clawhub/src/cli.ts:136`
  - body: The new `--device` option sends users into a flow that posts to `/cli/device/code` and `/cli/device/token`, but this PR does not add those routes or a verification page. Merging this as-is would expose a login mode that cannot complete against the shipped app.
  - confidence: 0.94
- **[P3] Validate device timing fields before polling:** `packages/clawhub/src/deviceAuth.ts:68-70`
  - body: `requestDeviceCode` checks the string fields but accepts missing or non-numeric `expires_in` and `interval`. Those values then drive the expiry display, deadline, and sleep interval, which can turn protocol errors into `NaN` timing behavior.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `packages/clawhub/src/cli.ts:136`
  - body: This is a user-facing CLI feature, and the branch does not update `CHANGELOG.md`. Add an Unreleased entry so the eventual CLI release notes cover the new auth path.
  - confidence: 0.87
