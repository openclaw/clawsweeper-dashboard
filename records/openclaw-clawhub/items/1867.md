## Summary

Keep open: current main does not implement device-flow auth, and this PR remains a client-side CLI feature that depends on missing server endpoints, verification UI, and token-issuance controls.

## What This Changes

The PR adds a `--device` CLI login option, a device-flow request/poll helper, and unit tests for client-side polling.

## Best Possible Solution

Complete the linked device-flow feature as an end-to-end rollout covering CLI flags, Convex HTTP endpoints, a `/device` verification UI, token controls, docs, changelog, and protocol/security tests.

## Reproduction Assessment

Yes. Source inspection is enough: the PR routes `--device` to new `/cli/device/*` calls, while current main and the PR file list do not provide the matching Convex endpoints or verification page.

## Solution Assessment

No. The client helper is useful, and the double-read bug is fixed, but enabling the flag before the server and verification surface exists is not the narrowest maintainable solution.
