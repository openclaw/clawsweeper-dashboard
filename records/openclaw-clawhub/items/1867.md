## Summary

Keep open. The PR is a useful client-side slice for the open device-flow feature, and commit c77aa40 fixed the double JSON parse, but it still exposes `--device` before current main has the required server endpoints, verification UI, and token controls.

## What This Changes

Adds a `--device` CLI login option with device-code request/polling helpers and unit tests for the ClawHub CLI.

## Best Possible Solution

Land an end-to-end device flow covering backend code issuance/token exchange, authenticated verification UI, expiry/single-use/rate-limit/scope controls, CLI docs, changelog, and regression tests.

## Reproduction Assessment

Yes for the review findings. Source inspection shows the PR posts to `/cli/device/code` and `/cli/device/token`, while current main has no matching routes or device-code verification page.

## Solution Assessment

No. The client helper is useful, but exposing the public CLI flag before the backend and verification UI land is not the narrowest maintainable solution.
