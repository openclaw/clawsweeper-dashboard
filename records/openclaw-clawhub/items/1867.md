## Summary

Keep open: current main does not implement device-flow login, and this PR is useful feature work but incomplete because it enables a user-facing CLI flag before the required server endpoints, verification page, and token controls exist.

## What This Changes

The PR adds a `--device` login mode to the ClawHub CLI with a new device-flow client helper and unit tests.

## Best Possible Solution

Complete #613 end to end across CLI flags, Convex HTTP endpoints, verification UI, token issuance controls, docs, changelog, and protocol tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the PR registers `--device` and posts to `/cli/device/code` and `/cli/device/token`, while current main and the PR diff do not define those routes or a verification page.

## Solution Assessment

No. A client-only `--device` flag is not the best way to solve headless auth; the maintainable path is to land the server-side RFC 8628 flow and keep the CLI path unavailable until that support exists.
