## Summary

Keep open: current main does not implement device-flow login, and this PR still exposes a public CLI flag before the backend endpoints, verification UI, and token issuance controls exist.

## What This Changes

Adds a `--device` option to `clawhub login` and `clawhub auth login`, a device-auth helper for requesting and polling codes, and unit tests for that helper.

## Best Possible Solution

Land device flow only as a complete auth feature, with CLI, Convex HTTP routes, authenticated verification UI, token controls, docs, changelog, and regression tests coordinated with #613.

## Reproduction Assessment

Yes for the review findings. Source inspection shows the PR posts to `/cli/device/code` and `/cli/device/token`, while current main has no matching Convex routes or device-code verification page.

## Solution Assessment

No. The client helper is a useful slice, but exposing `--device` before the server contract and security controls land is not the narrowest maintainable solution.
