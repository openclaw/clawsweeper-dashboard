## Summary

Keep this PR open. The earlier double body-read concern was addressed, but current main does not implement device-flow login and this branch still enables a client flow whose backend routes and verification page are missing.

## What This Changes

This PR adds a `--device` option to both CLI login commands, a client-side device-code request/polling helper, and unit tests for headless authentication.

## Best Possible Solution

Ship device flow end to end with CLI support, Convex HTTP endpoints, a verification page, token issuance controls, docs/changelog, and protocol/security regression tests, or keep the flag hidden until that path is ready.

## Reproduction Assessment

Yes. Source inspection is enough: the PR posts to `/cli/device/code` and `/cli/device/token`, while current main has no matching Convex routes and no `/device` verification page.

## Solution Assessment

No. The client helper is a useful slice, but enabling the CLI flag before backend and verification support exists is not the narrowest maintainable solution; the flow should land end to end or remain hidden until server support is ready.
