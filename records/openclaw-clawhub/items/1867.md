## Summary

Keep open: current main does not implement device-flow login, and this PR is still incomplete because it exposes a `--device` CLI path without the matching server endpoints, verification UI, and token-issuance controls. The earlier double JSON-read review finding appears fixed in `c77aa4028c497a610d940d3617ee780cda21557c`, so it is no longer listed as a blocker.

## What This Changes

The PR adds a `--device` login option, a client-side device-code request/polling helper, and unit tests for CLI device-flow authentication.

## Best Possible Solution

Complete #613 as an end-to-end device-flow rollout covering CLI flags, Convex HTTP endpoints, the verification page, token issuance controls, docs, changelog, and protocol/security tests.

## Reproduction Assessment

Yes. Source inspection of the PR head is enough: `--device` routes into `/cli/device/code` and `/cli/device/token`, while current main and the PR diff do not add matching Convex routes or a `/device` verification UI.

## Solution Assessment

No. The client helper is useful, but enabling the flag before backend and verification support exists is not the narrowest maintainable solution; this should land as a complete auth flow or remain hidden until server support is ready.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Add the device endpoints before enabling the flag:** `packages/clawhub/src/cli.ts:136`
  - body: The new `--device` option routes users into a flow that posts to `/cli/device/code` and `/cli/device/token`, but neither current main nor this PR adds those HTTP routes or a `/device` verification page. Merging this would expose a login mode that cannot complete against the shipped app.
  - confidence: 0.93

  - body: `requestDeviceCode` validates the string fields but the login command immediately uses `expires_in` and `interval` for display, timeout, and sleep math. A malformed response can produce `NaN` timing or invalid polling behavior instead of a clear protocol error.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `packages/clawhub/src/cli.ts:136`
  - body: This adds a user-facing CLI login mode, which is a feature under the repo policy, but the branch does not update `CHANGELOG.md`. Add an Unreleased entry so CLI release notes cover the new auth path.
  - confidence: 0.86
