## Summary

Keep open. Current main does not implement device-flow auth, and this PR is still useful but incomplete: the double JSON-read bug was fixed, while the exposed CLI flag still depends on missing backend routes and verification UI.

## What This Changes

The branch adds a `--device` option to both CLI login commands, a new deviceAuth helper/test module that calls `/cli/device/code` and `/cli/device/token`, and a device login command that stores the returned token.

## Best Possible Solution

Ship device flow end to end with the CLI flag/helper, Convex HTTP endpoints, `/device` verification page, token issuance controls, docs/changelog, and protocol/security tests.

## Reproduction Assessment

Yes. Source inspection of PR head shows `--device` posts to `/cli/device/code` and `/cli/device/token`, while current main and the PR diff do not add those HTTP routes or a `/device` verification page.

## Solution Assessment

No. The client helper is a useful slice, but enabling the user-facing flag before backend and verification support exists is not the narrowest maintainable solution; the feature should land end to end or stay hidden until server support is ready.
