## Summary

Keep open. Current main does not implement device-flow login, and this PR still exposes a client-only `--device` flow without the required server endpoints or verification UI; the earlier double-JSON-read review comment was fixed in the second PR commit.

## What This Changes

Adds a `--device` CLI login path with a client-side device-code request/polling helper and unit tests for Device Flow authentication.

## Best Possible Solution

Complete #613 as an end-to-end device-flow rollout covering CLI flags, Convex HTTP endpoints, verification UI, secure token issuance controls, docs, changelog, and protocol/security tests.

## Reproduction Assessment

Yes. Source inspection of the PR head is enough: `--device` routes into `/cli/device/code` and `/cli/device/token`, while current main and the PR diff do not add matching Convex routes or a `/device` verification UI.

## Solution Assessment

No. The client helper is useful, but enabling the flag before backend and verification support exists is not the narrowest maintainable solution; this should land as a complete auth flow or remain hidden until server support is ready.
