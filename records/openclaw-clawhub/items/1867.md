## Summary

Keep open: the PR is not implemented on current main, and the proposed client-only device flow would expose a public CLI login mode before the server routes, verification UI, and token issuance controls exist.

## What This Changes

Adds a `--device` CLI login mode with device-code request/polling helpers and unit tests for headless ClawHub authentication.

## Best Possible Solution

Implement the complete RFC 8628 flow across CLI, Convex HTTP routes, authenticated verification UI, token issuance controls, docs, changelog, and regression tests before making `--device` public.

## Reproduction Assessment

Yes for the review findings. Source inspection shows the PR calls `/cli/device/code` and `/cli/device/token`, while current main has no matching routes or device-code verification page.

## Solution Assessment

No. The client helper is a useful slice, but exposing the flag before the backend, verification UI, token controls, docs, and changelog land is not the narrowest maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Add the device endpoints before exposing the flag:** `packages/clawhub/src/cli.ts:136`
  - body: The new `--device` option sends users into a flow that posts to `/cli/device/code` and `/cli/device/token`, but this PR does not add those HTTP routes or a verification page. Merging this would expose a login mode that cannot complete against the shipped app.
  - confidence: 0.94
- **[P3] Validate device timing fields before polling:** `packages/clawhub/src/deviceAuth.ts:73`
  - body: `requestDeviceCode` validates the string fields but accepts responses where `expires_in` or `interval` are missing or non-numeric. The login command then uses those values for display, timeout, and sleep math, which can produce `NaN` timing instead of a clear protocol error.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `packages/clawhub/src/cli.ts:136`
  - body: This adds a user-facing CLI login mode, which is a feature under the repo policy, but the branch does not update `CHANGELOG.md`. Add an Unreleased entry so CLI release notes cover the new auth path.
  - confidence: 0.87
