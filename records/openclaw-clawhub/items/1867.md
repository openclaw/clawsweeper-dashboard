## Summary

Keep open. Current main does not implement device-flow auth, and this PR is still a client-side slice with blocking backend, verification UI, changelog, and auth/security design work remaining; the earlier double JSON-read review item was addressed in the PR head.

## What This Changes

Adds a client-side `--device` login mode to the ClawHub CLI with device-code request/polling helpers and unit tests.

## Best Possible Solution

Implement the complete RFC 8628 path across CLI, Convex HTTP, authenticated verification UI, token issuance controls, docs, changelog, and regression tests before making `--device` public.

## Reproduction Assessment

Yes for the review findings. Source inspection and the PR diff show the new CLI flag calls `/cli/device/code` and `/cli/device/token`, while current main has no matching routes or device-code verification page.

## Solution Assessment

No. The client helper is a useful start and the double-read bug was fixed, but exposing `--device` before the backend, verification UI, and token controls land is not the narrowest maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Add the device endpoints before exposing the flag:** `packages/clawhub/src/cli.ts:136`
  - body: The new `--device` option sends users into a flow that posts to `/cli/device/code` and `/cli/device/token`, but this PR does not add those HTTP routes or a verification page. Merging this would expose a login mode that cannot complete against the shipped app.
  - confidence: 0.94
- **[P3] Validate device timing fields before polling:** `packages/clawhub/src/deviceAuth.ts:68-70`
  - body: `requestDeviceCode` validates the string fields but accepts responses where `expires_in` or `interval` are missing or non-numeric. The login command then uses those values for display, timeout, and sleep math, which can produce `NaN` timing instead of a clear protocol error.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `packages/clawhub/src/cli.ts:136`
  - body: This adds a user-facing CLI login mode, which is a feature under the repo policy, but the branch does not update `CHANGELOG.md`. Add an Unreleased entry so CLI release notes cover the new auth path.
  - confidence: 0.87
