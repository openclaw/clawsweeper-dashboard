## Summary

Keep open: current main does not implement device-flow login, and this PR still exposes a CLI flag before the server endpoints, verification page, and auth controls exist. The earlier double JSON-read review issue was fixed in `c77aa40`, but the end-to-end feature is still incomplete.

## What This Changes

The PR adds a `--device` CLI login mode, a device-code request/poll helper, and unit tests for client-side RFC 8628-style authentication.

## Best Possible Solution

Complete device flow as an end-to-end auth feature covering CLI flags, Convex HTTP endpoints, the verification UI, token issuance controls, docs, changelog, and protocol/security tests.

## Reproduction Assessment

Yes. Source inspection of the PR head is enough: `--device` routes into `/cli/device/code` and `/cli/device/token`, while current main and the PR diff do not add matching HTTP routes or a `/device` verification UI.

## Solution Assessment

No. The client helper is useful, but enabling the flag before the server and verification surface exist is not the narrowest maintainable rollout; this should land as a complete auth flow or remain hidden until backend support is ready.
