## Summary

This PR should stay open for maintainer auth/security review. The double JSON-read problem noted earlier appears fixed in the PR head, but the branch still exposes a public `--device` login mode while the required server endpoints and verification UI are explicitly outside the PR and absent from current main.

## What This Changes

Adds a `--device` login mode to the ClawHub CLI, a client-side device-code request/polling helper, and unit tests for the helper.

## Best Possible Solution

Ship device flow end to end with the CLI flag, Convex HTTP endpoints, authenticated verification UI, token issuance controls, docs/changelog, and protocol/security regression tests, or keep the flag hidden until that path is ready.

## Reproduction Assessment

Yes. Source inspection of the PR diff shows the new flag calls `/cli/device/code` and `/cli/device/token`, while current main has no matching Convex routes or device-code verification page.

## Solution Assessment

No. The client helper is a useful slice, but enabling the public CLI flag before the backend, verification UI, and token controls land is not the narrowest maintainable solution.
