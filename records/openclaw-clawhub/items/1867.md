## Summary

Keep open. Current main does not implement device-flow auth, and this PR is still a client-side feature patch with server endpoints, verification UI, and token-issuance controls missing; the earlier double JSON-read review issue is fixed in the latest PR head.

## What This Changes

The PR adds a `--device` CLI login option, a device-code request/polling helper, and unit tests for client-side device-flow authentication.

## Best Possible Solution

Complete #613 as an end-to-end device-flow rollout covering CLI flags, Convex HTTP endpoints, an authenticated verification page, secure token issuance, docs, changelog, and protocol/security tests.

## Reproduction Assessment

Yes. Source inspection is enough: current main has only token/browser login, while the PR routes `--device` to `/cli/device/code` and `/cli/device/token` without adding those server routes or a verification UI.

## Solution Assessment

No. The client helper is useful, but enabling the flag before the backend and verification surface exists is not the narrowest maintainable solution; this should land as a complete auth flow.
