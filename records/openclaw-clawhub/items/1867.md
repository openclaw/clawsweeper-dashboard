## Summary

Keep this PR open. The earlier double JSON-read issue was fixed, but the branch still exposes a user-facing device login mode while the required server endpoints and verification UI are not present on current main or in the PR.

## What This Changes

Adds a `--device` option to ClawHub CLI login, a client-side device-code request/polling helper, and unit tests for device-flow authentication.

## Best Possible Solution

Ship device flow as an end-to-end auth feature covering the CLI flag/helper, Convex HTTP endpoints, verification page, token issuance controls, docs/changelog, and protocol/security tests.

## Reproduction Assessment

Yes. Source inspection of the PR head shows `--device` posts to `/cli/device/code` and `/cli/device/token`, while current main has no matching HTTP routes or `/device` verification route.

## Solution Assessment

No. The client helper is a useful slice, but exposing the flag before backend and verification support exists would ship a login mode that cannot complete.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Add the device endpoints before enabling the flag:** `packages/clawhub/src/cli.ts:136`
  - body: The new `--device` option sends users into a flow that posts to `/cli/device/code` and `/cli/device/token`, but this PR does not add those HTTP routes or a verification page. Merging this would expose a login mode that cannot complete against the shipped app.
  - confidence: 0.93
- **[P3] Validate device timing fields before polling:** `packages/clawhub/src/deviceAuth.ts:73-75`
  - body: `requestDeviceCode` checks the string fields but returns `expires_in` and `interval` unchecked, while `cmdDeviceLogin` uses them for display, timeout, and sleep math. A malformed response can produce `NaN` timing or invalid polling instead of a clear protocol error.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `packages/clawhub/src/cli.ts:136`
  - body: This adds a user-facing CLI login mode, which is a feature under the repo policy, but the branch does not update `CHANGELOG.md`. Add an Unreleased entry so CLI release notes cover the new auth path.
  - confidence: 0.86
