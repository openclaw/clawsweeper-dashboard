## Summary

Keep open: current main does not implement device flow, and the PR is still a useful implementation candidate for #613, but it exposes a client-only `--device` login path without the required backend endpoints or verification page.

## What This Changes

The branch adds a ClawHub CLI `--device` login mode, a `deviceAuth` polling helper with tests, and login-command wiring to `/cli/device/code` and `/cli/device/token`.

## Best Possible Solution

Complete #613 end to end with CLI flags, Convex HTTP endpoints, verification UI, token issuance controls, docs, changelog, and protocol tests.

## Reproduction Assessment

Yes. Source inspection shows the PR registers `--device` and posts to `/cli/device/code` and `/cli/device/token`, while current main and the PR patch do not provide those routes or a verification page.

## Solution Assessment

No. A client-only `--device` flag is not the best way to solve headless auth; the maintainable path is an end-to-end RFC 8628 rollout with server-side controls before exposing the CLI path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Implement the device endpoints before enabling the flag:** `packages/clawhub/src/cli.ts:136`
  - body: The new `--device` option exposes a login path that posts to `/cli/device/code` and `/cli/device/token`, but neither current main nor this PR defines those server routes or the verification page. Users would get a login mode that cannot succeed against the shipped app.
  - confidence: 0.91
- **[P3] Validate device timing fields before polling:** `packages/clawhub/src/deviceAuth.ts:70-72`
  - body: `requestDeviceCode` validates the string fields but accepts any `expires_in` and `interval` values, then `cmdDeviceLogin` uses them for display, timeout, and sleep math. A malformed response can produce invalid polling behavior instead of a clear protocol error.
  - confidence: 0.82
- **[P3] Add the required changelog entry:** `packages/clawhub/src/cli.ts:136`
  - body: This adds a user-facing CLI login mode, which is a feature under the repo policy. Add an Unreleased changelog entry so the CLI release notes cover the new auth path.
  - confidence: 0.86
