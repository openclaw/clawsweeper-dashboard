## Summary

Keep open: current main still only supports browser loopback and token-based headless login, while this PR is a client-only device-flow slice tied to open #613 and would expose a login flag before the required server routes and verification UI exist.

## What This Changes

The PR adds a `--device` option to CLI login commands, a device-code request/polling helper, and unit tests for client-side device-flow authentication.

## Best Possible Solution

Ship device flow end to end: CLI flag/helper, Convex HTTP endpoints, verification page, token issuance controls, docs/changelog, and protocol/security tests.

## Reproduction Assessment

Yes. Source inspection of the PR diff shows `--device` posts to `/cli/device/code` and `/cli/device/token`, while current main has no matching HTTP routes or `/device` verification page.

## Solution Assessment

No. The client helper is a useful slice, but enabling the flag before backend and verification support exists is not the narrowest maintainable solution; the flow should land end to end or remain hidden until server support is ready.
