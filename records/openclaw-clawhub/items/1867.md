## Summary

Keep open: the double body-read concern was fixed, but this is still a client-only device-flow feature and current main/PR head do not provide the required Convex endpoints or verification UI. Because it closes the still-open #613 and touches auth/token issuance, it needs end-to-end maintainer review rather than cleanup closure.

## What This Changes

The PR adds a `--device` CLI login mode, device-code request/polling helpers, and unit tests for headless ClawHub authentication.

## Best Possible Solution

Ship device flow as an end-to-end auth feature covering the CLI flag/helper, Convex HTTP endpoints, verification page, token issuance controls, docs/changelog, and protocol/security tests.

## Reproduction Assessment

Yes. Source inspection of the PR head shows `--device` posts to `/cli/device/code` and `/cli/device/token`, while current main and the PR diff do not add those HTTP routes or a `/device` verification page.

## Solution Assessment

No. The client helper is useful, but enabling the CLI flag before backend and verification support exists is not the narrowest maintainable solution; the flow should land end to end or stay hidden until server support is ready.
