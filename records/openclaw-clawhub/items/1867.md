## Summary

Keep open. Current main still lacks device-flow auth, and this PR is client-side only while the server endpoints, verification UI, and token-issuance controls needed for a working secure flow are still missing.

## What This Changes

The PR adds a `--device` CLI login mode, a new device-code request/polling helper, and unit tests for the client-side device-flow path.

## Best Possible Solution

Complete #613 as an end-to-end device-flow rollout covering CLI flags, Convex HTTP endpoints, the `/device` verification UI, token issuance controls, docs, changelog, and protocol/security tests.

## Reproduction Assessment

Yes. Source inspection of the PR head is enough: `--device` routes into `/cli/device/code` and `/cli/device/token`, while current main and the PR file list do not include matching Convex HTTP routes or a `/device` verification UI.

## Solution Assessment

No. The client helper is useful, but enabling the flag before the server/device verification surface exists is not the narrowest maintainable solution; this should land as a complete auth flow.
