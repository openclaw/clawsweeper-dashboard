## Summary

Keep open: current main does not implement device-flow login, and this PR is still client-only. The double JSON-read issue noted earlier is fixed in the latest PR head, but merging as-is would expose a CLI login flag that depends on missing `/cli/device/*` endpoints and a missing `/device` verification UI.

## What This Changes

The PR adds a client-side `--device` login mode to the ClawHub CLI, a new device-flow helper with tests, and documents server-side endpoints as required follow-up.

## Best Possible Solution

Complete the feature end to end for #613: CLI flag, Convex endpoints, `/device` verification UI, token lifecycle controls, docs, changelog, and protocol/security tests should land together.

## Reproduction Assessment

Yes. Source inspection is enough: the PR adds a CLI path that posts to `/cli/device/code` and `/cli/device/token`, while current main and the PR file list do not add matching HTTP routes or a verification UI.

## Solution Assessment

No. The client helper is useful, but enabling the flag before the server/device verification surface exists is not the narrowest maintainable solution; this should land as a complete auth flow.
