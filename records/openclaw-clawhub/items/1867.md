## Summary

Keep open. Current main does not implement device flow, and this PR still exposes a user-facing CLI flag before the server endpoints, verification page, and auth/security controls exist.

## What This Changes

The PR adds a `--device` CLI login option, a device-code request/polling helper, and unit tests for headless ClawHub authentication.

## Best Possible Solution

Ship device flow end to end: CLI flag/helper, Convex HTTP endpoints, a `/device` verification page, token issuance controls, docs/changelog, and protocol/security tests, or keep the flag hidden until server support is ready.

## Reproduction Assessment

Yes. Source inspection of the PR head shows `--device` posts to `/cli/device/code` and `/cli/device/token`, while current main has no matching HTTP routes or `/device` verification page.

## Solution Assessment

No. The client helper is a useful slice, but enabling the public flag before backend and verification support exists is not the narrowest maintainable solution.
