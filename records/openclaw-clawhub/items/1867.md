## Summary

Keep open: current main does not implement device-flow auth, and the PR branch still exposes a client-only login mode while the required endpoints, verification UI, and server-side controls are absent.

## What This Changes

The PR adds a `--device` CLI login option plus device-code request/polling helpers and unit tests for the ClawHub CLI.

## Best Possible Solution

Ship device flow as an end-to-end auth feature with backend device-code issuance/token exchange, verification UI, server controls, CLI docs/changelog, and regression tests.

## Reproduction Assessment

Yes for the review findings. Source inspection shows the PR posts to `/cli/device/code` and `/cli/device/token`, while current main has no matching routes or user-code verification page.

## Solution Assessment

No. The client helper is a useful slice, but exposing the public flag before the backend, verification UI, and auth controls exist is not the narrowest maintainable solution.
