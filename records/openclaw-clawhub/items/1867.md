## Summary

Keep open: current main does not implement device-flow auth, and the PR branch exposes a client-only auth path without the server endpoints or verification page needed for the command to work. The earlier double body-read review appears resolved in the PR head.

## What This Changes

Adds a `--device` login mode to the ClawHub CLI, a device-flow client module, and unit tests for code request and polling behavior.

## Best Possible Solution

Land device flow as a complete auth feature: server endpoints, verification UI, token creation controls, CLI client, docs, changelog, and tests in one reviewed rollout.

## Reproduction Assessment

Yes. The PR client posts to `/cli/device/code` and `/cli/device/token`, while current main and the PR diff contain no route or handler for those paths, so `clawhub login --device` would hit missing endpoints once exposed.

## Solution Assessment

No. A client-only flag is not the best fix for #613; either include the server/device verification implementation in the same PR or keep the flag unavailable until the backend is ready.
