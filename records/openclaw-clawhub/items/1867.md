## Summary

Keep open: current main still lacks device-flow auth, and this PR is a useful but incomplete client-side slice that would expose a login mode that cannot complete without server endpoints and verification UI.

## What This Changes

Adds a `--device` option to both CLI login commands, client-side device-code request/polling helpers, and unit tests.

## Best Possible Solution

Land device flow as an end-to-end auth feature covering CLI `--device`, backend code issuance/token exchange, authenticated verification UI, security controls, docs, changelog, and regression coverage.

## Reproduction Assessment

Yes for the review findings. Source inspection of the PR head shows the CLI posts to `/cli/device/code` and `/cli/device/token`, while current main has no matching routes or user-code verification page.

## Solution Assessment

No. A client-only flag is not the best mergeable shape because the server-side device-code issuance, verification UI, token exchange, and security controls are the core of the feature.
