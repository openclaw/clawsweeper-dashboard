## Summary

Keep open: current main does not implement device-flow auth, so this PR is not obsolete under the ClawHub cleanup policy. The latest patch addresses the earlier double-read bug, but it still exposes a new `--device` auth path before the server endpoints and verification UI exist, so it needs maintainer auth/security review before merge.

## What This Changes

The PR adds a ClawHub CLI `--device` login option, a device-code request/polling helper with tests, and a device-login branch that stores the returned API token.

## Best Possible Solution

Land or supersede this with an end-to-end device-flow implementation covering the CLI flag, Convex HTTP endpoints, verification UI, one-time device-code storage, expiry and rate limiting, token creation, docs, tests, and maintainer auth-security review; keep the CLI flag hidden until those pieces are ready.
