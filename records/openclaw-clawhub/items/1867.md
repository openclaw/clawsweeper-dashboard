## Summary

Keep this PR open: the proposed client-side device-flow pieces are useful, and the earlier double JSON-read bug was fixed, but current main still lacks the server endpoints and verification UI needed for this feature to work safely.

## What This Changes

The PR adds a `--device` login flag, a CLI device-code request/polling helper, and unit tests for ClawHub CLI device-flow authentication.

## Best Possible Solution

Complete #613 as an end-to-end device-flow rollout covering CLI flags, Convex HTTP endpoints, verification UI, token issuance controls, docs, changelog, and protocol/security tests.

## Reproduction Assessment

Yes. Source inspection of the PR head is enough: `--device` calls `/cli/device/code` and `/cli/device/token`, while current main and the PR diff do not add matching Convex routes or a `/device` verification page.

## Solution Assessment

No. The client helper is a useful start, but enabling the flag before the backend and verification surface exists is not the narrowest maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Add the device endpoints before enabling the flag:** `packages/clawhub/src/cli.ts:136`
  - body: The new `--device` option routes users into `cmdDeviceLogin`, which posts to `/cli/device/code` and `/cli/device/token`, but neither current main nor this PR adds those HTTP routes or a `/device` verification page. Merging this as-is would advertise a login mode that cannot succeed against the shipped app.
  - confidence: 0.93
- **[P3] Validate device timing fields before polling:** `packages/clawhub/src/deviceAuth.ts:75`
  - body: `requestDeviceCode` checks only the string fields, but `cmdDeviceLogin` immediately uses `expires_in` and `interval` for display, timeout, and sleep math. A malformed response can produce `NaN` timing or invalid polling behavior instead of a clear protocol error.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `packages/clawhub/src/cli.ts:136`
  - body: This adds a user-facing CLI login mode, which is a feature under the repo policy, but the branch does not update `CHANGELOG.md`. Add an Unreleased entry so the CLI release notes cover the new auth path.
  - confidence: 0.86
