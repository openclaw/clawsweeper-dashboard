## Summary

Current main still does not implement the PR's intended device-flow login, and the PR remains an incomplete client-only slice of the auth flow. The remaining blockers are not obsolete cleanup: enabling the flag before the server endpoints and verification UI exist would expose a login mode that cannot succeed.

## What This Changes

The PR adds a `--device` option to the ClawHub CLI login commands, a device-code request and polling helper, and unit tests for that helper.

## Best Possible Solution

Complete the device-flow feature as an end-to-end auth rollout covering CLI flags, Convex routes, verification UI, token issuance controls, tests, docs, and changelog.

## Reproduction Assessment

Yes. Source inspection is sufficient: the PR enables a `--device` login path that posts to new device endpoints, while current main and the PR body show the matching server routes and verification UI are absent.

## Solution Assessment

No. The client helper is useful, but enabling a public CLI flag before backend device-code issuance, verification, and token controls exist is not the best maintainable rollout.
