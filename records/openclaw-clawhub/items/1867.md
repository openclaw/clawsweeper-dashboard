## Summary

Keep open: this is a useful client-side start for device-flow login, but current main does not implement the feature and this PR still enables a user-facing auth flag without the required server endpoints, verification UI, changelog entry, and token-flow controls.

## What This Changes

The PR adds a `--device` CLI login option, a device-code request and polling helper, and unit tests for client-side device-flow authentication.

## Best Possible Solution

Complete #613 as a single end-to-end device-flow rollout covering CLI flags, Convex HTTP endpoints, the `/device` verification UI, token issuance controls, docs, changelog, and protocol/security tests.

## Reproduction Assessment

Yes. Source inspection is enough: the PR flag routes into `/cli/device/code` and `/cli/device/token`, while current main and the PR file list do not provide those server routes or the `/device` verification surface.

## Solution Assessment

No. The client helper is useful, but enabling a shipped CLI flag before the server and verification surface exist is not the narrowest maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Add the device endpoints before enabling the flag:** `packages/clawhub/src/cli.ts:136`
  - body: The new `--device` option routes users into `cmdDeviceLogin`, which posts to `/cli/device/code` and `/cli/device/token`, but neither current main nor this PR adds those HTTP routes or a `/device` verification page. Merging this as-is would advertise a login mode that cannot succeed against the shipped app.
  - confidence: 0.92
- **[P3] Validate device timing fields before polling:** `packages/clawhub/src/deviceAuth.ts:73-77`
  - body: `requestDeviceCode` checks only the string fields, but `cmdDeviceLogin` immediately uses `expires_in` and `interval` for display, timeout, and sleep math. A malformed response can produce `NaN` timing or invalid polling behavior instead of a clear protocol error.
  - confidence: 0.82
- **[P3] Add the required changelog entry:** `packages/clawhub/src/cli.ts:136`
  - body: This adds a user-facing CLI login mode, which is a feature under the repo policy, but the branch does not update `CHANGELOG.md`. Add an Unreleased entry so the CLI release notes cover the new auth path.
  - confidence: 0.86
