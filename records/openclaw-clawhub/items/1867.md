## Summary

Keep open. Current main still lacks device-flow auth, and this PR remains a client-side slice that would expose a login mode whose server endpoints and verification UI are absent.

## What This Changes

The branch adds a `--device` login mode, a device-code request/polling helper, and tests for the CLI client side of device-flow auth.

## Best Possible Solution

Land a complete device-flow rollout or replacement covering CLI, Convex HTTP endpoints, verification page, token issuance controls, docs/changelog, and protocol tests.

## Reproduction Assessment

Yes. Source inspection is enough: PR head routes `--device` to `/cli/device/code` and `/cli/device/token`, while current main and the PR diff do not add matching HTTP routes or a `/device` verification page.

## Solution Assessment

No. The client helper is a useful slice, but enabling a user-facing flag before backend, UI, and security controls exist is not the narrowest maintainable solution.

## Review Findings
