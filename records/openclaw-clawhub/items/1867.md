## Summary

Keep open. The branch is still a client-side slice of device-flow auth: it adds the CLI flag and polling helper, but the server endpoints and verification UI needed to complete the flow are not present, and the linked feature request #613 is still open.

## What This Changes

The PR adds a `--device` CLI login path, a device-flow polling helper with tests, and dispatch from both `login` and `auth login`.

## Best Possible Solution

Ship device flow end to end with CLI support, Convex HTTP endpoints, a verification page, token issuance and polling controls, docs/changelog, and protocol/security regression tests.

## Reproduction Assessment

Yes. Source inspection is enough: the PR calls `/cli/device/code` and `/cli/device/token`, while current main's Convex router and web routes do not define those paths.

## Solution Assessment

No. A client-only flag is not sufficient; the maintainable path is to land the CLI with backend, UI, security controls, docs/changelog, or keep it hidden until those pieces exist.
