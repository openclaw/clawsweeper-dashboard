## Summary

Keep open: current main does not implement device-flow auth, and this PR exposes a client-side login flag while the required server endpoints, verification UI, and auth controls are still absent.

## What This Changes

The PR adds a `--device` CLI login mode, client-side device-code request/polling helpers, and unit tests for device-flow authentication.

## Best Possible Solution

Complete #613 as an end-to-end auth rollout covering CLI flags, Convex HTTP endpoints, the verification page, token issuance controls, docs, changelog, and protocol/security tests.

## Reproduction Assessment

Yes. Source inspection of the PR diff is enough: the new flag posts to `/cli/device/code` and `/cli/device/token`, while current main has no matching HTTP routes or `/device` route.

## Solution Assessment

No. The client helper is useful, but the auth flow should not be exposed until the server verification/token exchange and security policy land with it.
