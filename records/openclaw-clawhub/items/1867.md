## Summary

Keep open. The PR is a useful client-side slice for #613, and the double JSON parse noted earlier was fixed, but current main still lacks the required device-code endpoints and verification page, so merging this would expose a login mode that cannot complete.

## What This Changes

The PR adds a `--device` login option plus client-side device-code request/polling support for the ClawHub CLI.

## Best Possible Solution

Ship device flow end to end with CLI support, Convex HTTP endpoints, a verification page, token issuance controls, docs/changelog, and protocol/security regression tests, or keep the CLI flag hidden until that path is ready.

## Reproduction Assessment

Yes. Source inspection is enough: the PR posts to `/cli/device/code` and `/cli/device/token`, while current main has no matching Convex HTTP routes and no `/device` verification page.

## Solution Assessment

No. The client helper is a useful slice, but enabling the flag before backend and verification support exists is not the narrowest maintainable solution; the flow should land end to end or remain hidden until server support is ready.
