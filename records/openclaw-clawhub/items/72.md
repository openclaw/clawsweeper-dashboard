## Summary

Keep open. This is an open ClawHub issue, and the cleanup policy only permits auto-closing PRs; substantively, current main still verifies token login through registry discovery/defaults and `/api/v1/whoami`, while the old `www` workaround no longer matches live routing, so maintainer production-token verification is still needed.

## What This Changes

Fresh Settings-created `clh_` API tokens reportedly fail `clawhub login --token` verification with `Unauthorized`.

## Best Possible Solution

Maintainers should verify the current published CLI with a fresh production Settings token; if it still fails, land one focused fix for the proven failing layer and consolidate any shared remaining work with the broader Unauthorized issue cluster.

## Reproduction Assessment

No. The source path and user reports are clear, and a dummy live probe checked the current host redirect behavior, but proving the reported valid-token failure requires a fresh Settings-created production token and account state that this review did not have.

## Solution Assessment

Unclear. Switching defaults to `https://www.clawhub.ai` matched early workaround reports, but current production redirects `www` back to the apex host and the redirect-specific issue is closed, so the next fix should follow fresh production-token evidence.
