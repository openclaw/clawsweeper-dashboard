## What This Changes

The issue reports that Settings-created `clh_` API tokens fail `clawhub login --token` with `Unauthorized` instead of being verified and stored by the CLI.

## Best Possible Solution

End state should be a verified production auth path where a fresh Settings-created `clh_` token works with the default CLI registry for `login --token`, `whoami`, and a minimal publish or sync smoke path, with any source fix targeted to the proven failing layer.

## Reproduction Assessment

No high-confidence live reproduction was established because the failure requires a real production Settings-created token. Source inspection confirms the reported path: `login --token` verifies through `/api/v1/whoami` and surfaces an auth failure if token validation fails.

## Solution Assessment

Unclear. The old `https://www.clawhub.ai` workaround was useful evidence in January, but current production routing points `www` back to the apex host, so maintainers should verify the current real-token failure before changing CLI defaults, discovery, hosting, token lookup, or account-state handling.
