## Summary

Keep open. This is an open ClawHub issue, and the cleanup policy does not auto-close issues; substantively, the old `www` workaround is now obsolete while a fresh production `clh_` token still needs maintainer verification against the current default registry.

## What This Changes

Fresh Settings-created `clh_` API tokens reportedly fail `clawhub login --token` verification with `Unauthorized`.

## Best Possible Solution

Maintain a verified production auth path where a fresh Settings-created `clh_` token works with the default registry for `login --token`, `whoami`, and a minimal publish or sync smoke; if it still fails, fix the specific failing layer with focused regression coverage.

## Reproduction Assessment

No. The issue has clear historical reproduction reports, but this review could not use a real fresh production token, and dummy-token probes show the original apex-to-www redirect no longer occurs.

## Solution Assessment

Unclear. Switching defaults to `www.clawhub.ai` matched the historical workaround but is not the best current fix because live production now redirects `www` back to the apex host; maintainers should verify a real token first and then target the exact failing layer.
