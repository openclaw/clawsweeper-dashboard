## What This Changes

Fresh ClawHub API tokens created in Settings should succeed with `clawhub login --token` instead of returning `Unauthorized`.

## Best Possible Solution

Maintainers should verify the latest published CLI against production with a fresh Settings-created token and browser-created token, then fix the exact failing layer or consolidate this into the active auth cluster once production behavior is proven.

No high-confidence live reproduction is available from this read-only review because a real production token is required. Source inspection confirms the reported verification path: `login --token` calls `/api/v1/whoami` before storing credentials, and that endpoint rejects token/account failures as Unauthorized.

## Solution Assessment

Unclear. A host-default-only change matched the original workaround, but current production redirects `www.clawhub.ai` back to `clawhub.ai`, so maintainers should reproduce with a real token before changing CLI defaults, token lookup, or hosting configuration.
