## Summary

Keep open. This is an open ClawHub issue, and the cleanup policy for this repository does not auto-close issues; substantively, current main still verifies Settings-created tokens through the default registry and `/api/v1/whoami`, while live routing now redirects `www.clawhub.ai` back to the apex host, so the old workaround is not a safe close/fix signal without a real production token check.

## What This Changes

Fix `clawhub login --token` so Settings-created `clh_` API tokens verify and store successfully with the default registry.

## Best Possible Solution

Maintainers should verify a fresh production Settings-created token against the current published CLI; if it still fails, land one targeted fix at the proven failing layer with focused coverage for registry discovery, redirect/header behavior, token lookup, and `whoami`.

## Solution Assessment
