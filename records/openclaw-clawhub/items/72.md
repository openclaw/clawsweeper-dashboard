## Summary

Keep open. This is an open ClawHub issue, and current main does not prove the valid-token failure is fixed: the CLI still verifies supplied tokens through the default registry and `/api/v1/whoami`, the older `www.clawhub.ai` workaround was closed as obsolete, and a real production Settings-created token is still needed to verify the current failure.

## What This Changes

A Settings-created `clh_` API token should work with `clawhub login --token`, but the reporter saw `Unauthorized` for fresh valid tokens.

## Best Possible Solution

Verify a fresh production Settings token with the current published CLI and default registry, then fix the proven failing layer or consolidate the related Unauthorized reports under one maintainer-owned tracker.

## Reproduction Assessment

No. Historical reports and the source path are strong, but this review only verified current host routing and invalid-token handling; a real production Settings-created token is required for high-confidence reproduction.

## Solution Assessment

Unclear. The old fix of switching defaults to `https://www.clawhub.ai` is not currently proven best because current source, live discovery, and observed production redirects all point back to the apex canonical host.
