## Summary

Keep open. ClawHub cleanup policy keeps open issues with unresolved auth behavior in maintainer hands, and current evidence does not prove the valid-token failure is fixed: source still verifies `login --token` through `/api/v1/whoami`, while the original `www` workaround is stale because live routing now redirects `www.clawhub.ai` back to `clawhub.ai`.

## What This Changes

The issue requests that Settings-created `clh_` API tokens work with `clawhub login --token` using the default CLI registry.

## Best Possible Solution

Maintain a verified production auth path where a fresh Settings-created `clh_` token works with the default registry for `login --token`, `whoami`, and a minimal publish/sync smoke path; if it still fails, fix the specific failing layer with focused tests and keep bearer-token validation intact.

## Solution Assessment
