## Summary

Keep open. ClawHub cleanup policy keeps issues open, this issue is paired with open implementation PRs, and current main still lacks the requested declared security-tool category/context mechanism; the existing mitigation is only a narrow external-scanner suppression path.

## What This Changes

The issue requests a declared security-tool skill category so moderation can distinguish coherent scanner/audit behavior from malicious or incoherent behavior, with a recovery path for affected publishers.

## Best Possible Solution

Resolve this through maintainer review of #1132 and #1133, or a replacement design that parses declared security-tool metadata, contextualizes only allowlisted suspicious reason codes, preserves malicious and other hard safety floors, passes category context and false-category guidance to the LLM evaluator, documents the behavior, tests the moderation boundaries, and provides an admin recovery path for already affected publishers.
