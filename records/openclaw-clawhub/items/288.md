## Summary

Keep open. Current main still lacks the requested behavioral/Oathe scanner in storage, publish/rescan lifecycle, cron polling, public security snapshots, skill detail UI, scanner routes, and browse/search controls; related PR #562 is closed and unmerged. This is also a ClawHub issue, so it is not an auto-close candidate under the repository policy.

## What This Changes

Add a behavioral security analysis signal, suggested as an Oathe integration, alongside VirusTotal and ClawScan with detail-page display, full report links, and browse/search sort and filter support.

## Best Possible Solution

Define a reviewed, opt-in behavioral scanner provider contract, then implement persisted scan results, publish/rescan submission, bounded indexed polling, public API exposure, skill detail/report UI, and indexed browse/search sort or filter fields once the data model and trust semantics are settled.
