Keep open: current main does not contain the Codex worktree setup, and this draft PR still has a blocking portability issue plus missing real behavior proof.

## What This Changes

This draft PR adds a checked-in Codex environment, `dev:worktree` and `seed:dev` Bun scripts backed by `scripts/dev-worktree.ts`, and README/CONTRIBUTING updates for local worktree setup and seeding.

## Best Possible Solution

Generalize the worktree helper so env discovery is portable, remove personal absolute paths from code and docs, and add after-fix terminal proof from a detached worktree for the run and seed flows.

## Reproduction Assessment

Yes, from source inspection: `findEnvFile()` falls back only to `.env.local` in the current worktree or a contributor-specific absolute path. A fresh detached worktree on another machine without its own `.env.local` cannot use the documented canonical fallback.

## Solution Assessment

No; the PR direction is useful, but the proposed fallback is not a maintainable repo-level solution. The helper should use an explicit/env-configured path or a portable discovery mechanism, and the PR should show real detached-worktree output before merge.
