## Summary

Keep this draft PR open: current main does not contain the Codex worktree setup, but the proposed branch still has a portable env-discovery bug and lacks required real detached-worktree proof.

## What This Changes

The PR adds a checked-in Codex environment, `dev:worktree` and `seed:dev` scripts, a Bun worktree helper, and README/CONTRIBUTING updates for local setup and seeding.

## Best Possible Solution

Generalize env discovery, remove personal paths from code and docs, and add terminal proof from a detached worktree for the run and seed flows before maintainer approval.

## Reproduction Assessment

Yes, from source inspection: a detached worktree on any machine without its own `.env.local` can only fall back to a contributor-specific `/Users/...` path. That is enough to reproduce the portability failure without mutating the checkout.

## Solution Assessment

No. The PR direction is useful, but the fallback should be based on `CLAWHUB_ENV_FILE`, an explicit argument, or portable worktree discovery rather than a personal path, and the PR should include real detached-worktree output before merge.
