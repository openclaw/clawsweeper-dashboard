## Summary

Keep open. Current main still supports the reported publish-to-search drop for ordinary publishers, and this ClawHub cleanup pass keeps issues open rather than auto-closing them; the fix also needs maintainer decisions around moderation-safe public projection semantics.

## What This Changes

The issue requests that publishing a new version of an already approved skill not remove the last approved skill from search while review and scans run.

## Best Possible Solution

Implement a version-aware public projection that keeps the last approved skill card, digest, and searchable embedding visible while a new version is pending review, then promotes the new projection only after scans and moderation clear it.

## Reproduction Assessment

Yes from source inspection. An ordinary republish can write `moderationStatus: hidden` onto the current skill row, and every search hydration path then calls `toPublicSkill`, which filters that row out.

## Solution Assessment

Yes at the product level: preserving a last-approved public projection is the right narrow end state. The exact promotion, rollback, and unchanged-fingerprint reuse rules need maintainer design before implementation.
