## Summary

Keep open. Current main still has a source-supported path where an ordinary republish moves the canonical skill row to hidden pending-scan state, and public search then filters that row instead of serving the last approved version.

## What This Changes

The issue asks ClawHub to keep an already approved skill searchable while a newly published version is under review or VirusTotal scanning.

## Best Possible Solution

Implement a version-aware public projection that keeps the last approved skill card, digest, and searchable embedding visible while a new version is pending review, then promotes the new projection only after moderation and scanning clear it.

## Reproduction Assessment

Yes. The source-level reproduction is publish a higher latest version as an ordinary non-trusted publisher, observe `insertVersion` set the skill to hidden pending scan, then see public search hydration filter that hidden skill/digest out.

## Solution Assessment

Unclear. Keeping the last approved version visible is the right target, but the exact data model and unchanged-content review reuse need maintainer product and security decisions before implementation.
