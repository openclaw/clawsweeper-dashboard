## Summary

Keep open. This is a ClawHub issue, so this cleanup pass should not auto-close it; independently, current main still has a source-reproducible path where an ordinary publish can hide the existing skill row during pending scan and public search then filters it out.

## What This Changes

The issue requests that already-approved skills remain searchable while a newly published version is scanned or reviewed, and that unchanged content not re-trigger review flags.

## Best Possible Solution

Implement a version-aware public projection that keeps the last approved skill card, digest, searchable embedding, API metadata, and file surface visible while the new version remains gated, then promotes the new projection only after moderation and scans clear it.

## Reproduction Assessment

Yes, at source level. An ordinary non-trusted publisher can publish a new version, `insertVersion` can write hidden pending-scan state to the existing skill row, and public search then filters that row through `toPublicSkill`; I did not run a live Convex publish in this read-only review.

## Solution Assessment

No patch is under review, and the direct preserve-active approach in https://github.com/openclaw/clawhub/pull/274 is not the safest fix because it bypasses pending-scan visibility gating. The maintainable direction is last-approved-version continuity while the new upload remains gated until scanner and moderation finalization.
