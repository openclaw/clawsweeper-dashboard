## Summary

Keep open. Current main still has the reported publish-to-search drop for ordinary publishers: publishing schedules fresh scans, writes a hidden moderation state onto the skill row, mirrors that state into the search digest, and public search drops hidden skills; ClawHub cleanup policy also keeps issues open.

## What This Changes

Ensure an already approved skill remains searchable while a newly published version is under review, and avoid re-running review for unchanged content where that is safe.

## Best Possible Solution

Implement a version-aware public projection that keeps the last approved skill card, digest, file access boundary, and searchable embedding live until the new version clears review, then promotes the new projection atomically.

## Reproduction Assessment

Yes, from source inspection. An ordinary publish can set the skill row to `hidden`, search hydrates through the digest/current skill state, and `toPublicSkill` rejects hidden skills, so the reported search drop follows without needing a live production publish.

## Solution Assessment

Unclear until maintainers choose the visibility model. The requested outcome is right, but the safest implementation is a last-approved public projection, not exposing the pending version or blindly reusing old review results.
