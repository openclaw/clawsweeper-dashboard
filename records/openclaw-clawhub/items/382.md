## Summary

Keep open. ClawHub cleanup policy does not auto-close issues in this pass, and current main still has a source-supported path where an ordinary republish hides the canonical skill during pending scan/review and public search filters it out.

## What This Changes

The issue requests that already-approved skills remain searchable while a newly published version is under review or scanning.

## Best Possible Solution

Implement a version-aware public projection that keeps the last approved skill card, digest, and searchable embedding visible during pending review, then promotes the new version only after moderation and scanning clear it.

## Reproduction Assessment

Yes. The source-level path is clear on current main: ordinary republish calls `insertVersion`, can set the skill to hidden, syncs that hidden state into search digest, demotes the prior embedding, and public search filters the hidden row.

## Solution Assessment

Unclear as an immediate automated fix. Keeping the last approved version visible is the right end state, but maintainers need to define the projection and review-reuse policy so unreviewed content is not exposed.
