## Summary

Keep open. Current main still has the source path that can hide an approved skill during an ordinary republish, and ClawHub cleanup policy keeps issues open rather than auto-closing them.

## What This Changes

The issue requests that an already approved skill remain searchable while a newly published version is scanned or reviewed.

## Best Possible Solution

Keep the last approved public projection searchable during pending review, then promote the new version's card, digest, and embedding only after moderation and scans clear it.

## Reproduction Assessment

Yes from source inspection. An ordinary republish can write `moderationStatus: hidden` onto the skill row, and public search then hydrates through `toPublicSkill`, which filters that row out.

## Solution Assessment

Unclear for an automated fix. A last-approved projection is the likely right direction, but maintainers should choose the promotion, rollback, and unchanged-fingerprint reuse rules before implementation.
