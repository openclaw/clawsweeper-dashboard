## Summary

Keep open: ClawHub cleanup policy does not auto-close issues, and current main still has a source-reproducible path where a normal non-trusted republish hides the skill row, mirrors that hidden state into search, and filters it from public results while review/scans are pending.

## What This Changes

The issue asks for an already-approved skill to remain searchable while a newly published version is under Instruction Scope and VirusTotal review.

## Best Possible Solution

Implement a version-aware public projection that keeps the last approved card, digest, embedding, API/file surface, and ownership visible while a new version remains gated, then promotes the new projection only after moderation and scans clear it.

## Reproduction Assessment

Yes, from source inspection. A normal non-trusted republish can write `moderationStatus: hidden` to the skill row, sync that hidden state into `skillSearchDigest`, demote the previous embedding, and then public search filters the row out; I did not run a live Convex publish in this read-only review.

## Solution Assessment

No patch is under review, and the direct preserve-active approach is not the best fix because it can bypass pending-scan visibility gating. The safer solution is last-approved public continuity while the new upload remains quarantined until scanner and moderation promotion rules complete.
