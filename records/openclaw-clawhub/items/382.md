## Summary

Keep open. Current main still has a source-level path where an ordinary publish can hide an existing skill during pending scan/review, and the ClawHub cleanup policy for this repository does not auto-close issues.

## What This Changes

The issue asks for already-approved skills to remain searchable while a newly published version is under Instruction Scope and VirusTotal review, with unchanged content not re-triggering review unnecessarily.

## Best Possible Solution

Implement a version-aware public projection that keeps the last approved skill card, digest, searchable embedding, API metadata, and file surface visible while the new version remains gated, then promotes the new projection only after moderation and scans clear it.

## Reproduction Assessment

Yes, at source level. An ordinary non-trusted publish can write hidden pending-scan state to the existing skill row, sync that state into the search digest, and then public search filters the row through `toPublicSkill`; I did not run a live Convex publish in this read-only review.

## Solution Assessment

No patch is under review. The prior direct preserve-active approach in https://github.com/openclaw/clawhub/pull/274 is not the safest solution because it can bypass pending-scan visibility gating; the maintainable path is last-approved-version continuity while the new upload remains gated.
