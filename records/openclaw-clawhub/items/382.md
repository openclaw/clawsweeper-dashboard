## Summary

Keep open. Current main still supports the reported publish-to-search drop for ordinary publishers, and the ClawHub cleanup policy for this pass does not auto-close issues.

## What This Changes

Already approved skills should remain searchable on the last approved version while a newly published version is under Instruction Scope and VirusTotal review.

## Best Possible Solution

Implement a version-aware public projection that keeps the last approved skill card, digest, and searchable embedding visible while a new version is pending, then promotes the new projection only after moderation and scans clear it.

## Reproduction Assessment

Yes from source inspection, though I did not run a live Convex publish. An ordinary non-trusted republish follows `publishVersionForUser` to `insertVersion`, writes `moderationStatus: hidden`, and public search then filters the skill through `toPublicSkill`/`isPublicSkillDoc`.

## Solution Assessment

Unclear for an automated fix. Keeping a last-approved public projection is the right direction, but maintainers should choose the promotion and fingerprint-reuse rules so moderation safety is preserved.
