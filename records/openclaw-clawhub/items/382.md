## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still supports the reported publish window where an ordinary republish hides the skill row while pending scans run and public search filters it out.

## What This Changes

The issue asks for already-approved skills to remain searchable while a newly published version is under review or scanning, and for unchanged content not to trigger the same review window unnecessarily.

## Best Possible Solution

Implement a version-aware public projection that keeps the last approved skill card, digest, and searchable embedding visible during review, then atomically promotes the new projection only after moderation and scanning clear it.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to republish a higher semver version of an active skill as a non-trusted ordinary publisher and inspect search before scanners clear: the publish path patches the skill to hidden pending-scan state, syncs that state to the digest, and public search converts it through `toPublicSkill`.

## Solution Assessment

Unclear for automation, but the requested product behavior is the right direction. The maintainable fix is not a one-line status exception; it needs a last-approved public projection and explicit promotion rules so moderation safety is preserved.
