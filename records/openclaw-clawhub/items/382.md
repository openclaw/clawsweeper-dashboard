## Summary

Keep open. Current main still supports the reported publish-to-search drop for ordinary publishers, and the ClawHub cleanup policy for this pass keeps issues open rather than auto-closing them.

## What This Changes

The issue asks for already approved skills to remain searchable while a newly published version is under Instruction Scope review and VirusTotal/LLM scanning.

## Best Possible Solution

Implement a version-aware public projection that keeps the last approved skill card, digest, and searchable embedding visible during new-version review, then promotes the new projection only after moderation/scanning clears it.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: an ordinary republish schedules scans, writes `moderationStatus: hidden` to the existing skill, syncs that state into the search digest, and public search drops hidden rows. I did not run a live publish/search cycle in this read-only review.

## Solution Assessment

Unclear until maintainers choose the moderation model. Keeping the last approved projection visible is the right direction, but the data model and scanner reuse rules need explicit product/security review before a focused patch is safe.
