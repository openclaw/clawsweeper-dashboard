## Summary

Keep open. The ClawHub cleanup policy for this pass keeps issues open, and current main still has a source-backed path where an ordinary republish hides the canonical skill row during pending review and search filters it out.

## What This Changes

Already-approved skills should remain searchable while a newly published version is reviewed, and unchanged content should not unnecessarily retrigger review.

## Best Possible Solution

Implement a version-aware public projection that preserves the last approved skill card, digest, and searchable embedding during pending review, then promotes the new projection only after moderation and scans clear it.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: republish a higher version as an ordinary non-trusted publisher, observe `insertVersion` set the skill row to hidden pending scan, then public search hydration filters that hidden row out.

## Solution Assessment

Unclear. Keeping the last approved projection visible is the right target, but unchanged-content review reuse should wait for a maintainer-approved fingerprint and scanner-policy model.
