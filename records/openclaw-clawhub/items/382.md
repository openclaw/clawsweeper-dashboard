## Summary

Keep open: current main still has the source path where a non-trusted republish can hide the skill row, mirror that hidden state into search, and then public search filters it out. The related search-index reports are still open, and this needs maintainer product/security design rather than cleanup closure.

## What This Changes

The issue asks for already approved skill versions to remain searchable while a newly published version is reviewed/scanned, and for unchanged content not to re-trigger review unnecessarily.

## Best Possible Solution

Implement a version-aware public projection that keeps the last approved card, digest, embedding, and public file surface visible while a new version remains gated, then promotes the new projection only after moderation and scans clear it.

## Reproduction Assessment

Yes, from source inspection. A normal republish by a non-trusted publisher can write `moderationStatus: hidden` to the current skill, sync that to `skillSearchDigest`, demote the previous embedding, and then public search filters the row out.

## Solution Assessment

Yes at the product level, but not by making hidden pending-scan rows public. The maintainable solution is a last-approved public projection with explicit promotion, rollback, file-access, and scan-reuse rules.
