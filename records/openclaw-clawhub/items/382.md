## Summary

Keep open. The ClawHub cleanup policy for this pass does not auto-close issues, and current main still has a source-supported path where an ordinary republish hides the canonical skill row during pending review and search filters it out.

## What This Changes

The issue requests that already-approved skills remain searchable while a newly published version is scanned or reviewed.

## Best Possible Solution

Implement a version-aware public projection that keeps the last approved skill card, digest, and searchable embedding visible during pending review, then atomically promotes the new projection only after scanning and moderation clear it.

## Reproduction Assessment

Yes. The source-level path is clear: publish a new latest version as an ordinary non-trusted publisher, observe `insertVersion` set the skill to hidden pending scan, then search hydration filters that hidden digest/skill out of public results.

## Solution Assessment

Unclear as written. Keeping the last approved version visible is the right target, but unchanged-content review reuse should only be added after maintainers define a safe fingerprint and scanner-policy model.
