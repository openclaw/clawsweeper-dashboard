## Summary

Keep open. Current main still supports the reported publish-to-search drop for ordinary publishers, and ClawHub cleanup policy keeps issues open for maintainer follow-up.

## What This Changes

The issue requests that an already-approved skill stay searchable while a newly published version is scanned or reviewed, with unchanged content avoiding redundant review flags.

## Best Possible Solution

Implement a version-aware public projection that keeps the last approved skill card, digest, and searchable embedding visible until the new version clears scanning and moderation, with regression coverage for approved-skill republishes.

## Reproduction Assessment

Yes from source inspection. An ordinary republish can write `moderationStatus: hidden` onto the current skill row, and the search paths then call `toPublicSkill`, which filters that row out.

## Solution Assessment

Yes at the product level: preserving a last-approved public projection is the narrow maintainable direction. The exact promotion, rollback, file-access, and unchanged-fingerprint reuse rules need maintainer design before implementation.
