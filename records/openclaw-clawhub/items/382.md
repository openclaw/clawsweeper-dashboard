## Summary

Keep open: current main still supports the reported publish-to-search disappearance, and ClawHub cleanup policy keeps issues open; the repair needs maintainer design because it crosses moderation, scanning, search, and public file-access boundaries.

## What This Changes

Requests that an already approved skill remain searchable via its last approved version while a newly published version is under Instruction Scope, VirusTotal, LLM, and dependency review.

## Best Possible Solution

Implement a version-aware public projection that keeps the last approved skill card, digest, and searchable embedding visible while a new latest version is pending review, then promotes the new projection only after moderation and scans clear it.

## Reproduction Assessment

Yes, from source inspection. An ordinary republish can write `moderationStatus: hidden` onto the current skill row, the digest mirrors that state, and public search calls `toPublicSkill`, which filters hidden rows out.

## Solution Assessment

Yes at the product level, but not by letting hidden pending-scan rows through the public filter. The safer maintainable fix is a last-approved projection with explicit promotion, rollback, file-access, and scan-reuse rules.
