## Summary

Keep open. Current main still supports the reported publish-to-search drop for ordinary publishers, and the ClawHub cleanup policy for this pass keeps issues open rather than auto-closing them.

## What This Changes

The issue asks for an already-approved skill to remain searchable while a newly published version is pending review and scanning.

## Best Possible Solution

Implement a version-aware public projection that keeps the last approved skill card, digest, file boundary, and searchable embedding visible until the new version clears review, then promotes the new projection atomically.

## Reproduction Assessment

Yes, from source inspection. A normal publish can set the skill row to `hidden`, the digest sync mirrors that state, and public search hydration rejects hidden skills, so the reported disappearance follows without a live production publish.

## Solution Assessment

Unclear until maintainers choose the moderation visibility model. The requested outcome is right, but the maintainable fix is likely a last-approved public projection rather than exposing the pending version or blindly reusing old scan results.
