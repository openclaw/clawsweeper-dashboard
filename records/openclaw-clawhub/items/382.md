## Summary

Keep open. Current main still has a source-reproducible path where an ordinary non-trusted publish can move the current skill row to hidden pending review, mirror that state into search, and then public search filters it out; ClawHub cleanup policy also does not auto-close issues.

## What This Changes

The issue asks for already-approved skills to remain searchable while a newly published version is under review or scan.

## Best Possible Solution

Implement a version-aware public projection that keeps the last approved skill card, digest, searchable embedding, API metadata, and public file surface visible while the new version remains gated, then promotes the new projection only after moderation and scans clear it.

## Reproduction Assessment

Yes, at source level. An ordinary non-trusted publisher can publish a new version, `insertVersion` can write hidden pending-review state onto the existing skill row, and public search then filters that row through `toPublicSkill`.

## Solution Assessment

Yes at the product direction level, but not by making hidden pending-scan rows public. The safer maintainable path is a last-approved public projection with explicit promotion, rollback, file-access, and scan-reuse rules.
