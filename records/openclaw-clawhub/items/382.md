## Summary

Keep open. Current main still supports the reported publish-to-search disappearance for ordinary publishers, and the ClawHub cleanup policy for this pass keeps issues open rather than auto-closing them.

## What This Changes

A previously approved skill should remain searchable while a newly published version is pending review/scanning, with unchanged content not needlessly retriggering review flags.

## Best Possible Solution

Implement a version-aware public projection that keeps the last approved skill card, digest, file boundary, and searchable embedding visible until the new version clears review, then promotes the new projection atomically.

## Reproduction Assessment

Yes, from source inspection. A normal non-trusted publish can write `hidden` onto the skill row, sync that state into the search digest, and then public search hydration filters the skill out.

## Solution Assessment

Unclear until maintainers choose the moderation visibility model. The requested outcome is right, but the maintainable fix is likely a last-approved public projection rather than exposing the pending version or blindly reusing old scan results.
