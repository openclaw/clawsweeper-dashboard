## Summary

Keep open. ClawHub cleanup policy does not auto-close issues, and current main still has a source-supported path where an ordinary republish hides the canonical skill row during pending review while public search filters that row out.

## What This Changes

The issue asks ClawHub to keep the last approved skill indexed while a newly published version is scanned and reviewed.

## Best Possible Solution

Implement a version-aware public projection that keeps the last approved skill card, digest, file access boundary, and searchable embedding visible while the new latest version is pending, then promotes the new projection only after moderation and scans clear it.

## Reproduction Assessment

Yes, at source level. Publish a new latest version as an ordinary non-trusted publisher: `insertVersion` writes the skill to hidden pending review, digest sync mirrors that state, and public search hydration filters it out.

## Solution Assessment

Unclear as an automated repair. Keeping the last approved version visible is the right target, but the exact projection, promotion, and unchanged-content reuse policy need maintainer product and security review first.
