## Summary

Keep open. The ClawHub cleanup policy does not auto-close issues, and current main still has a credible publish path where an ordinary republish rewrites the skill row to hidden while scans run, then search/list hydration filters it out instead of serving a last-approved public projection.

## What This Changes

The issue asks ClawHub to keep an already-approved skill searchable while a newly published version is rescanned and reviewed, and to avoid re-triggering review flags for unchanged content.

## Best Possible Solution

Implement a version-aware public projection for skills: keep the last approved skill card, digest, and searchable embedding visible while a new version is pending review, then atomically promote the new projection only after moderation and scanning clear it.
