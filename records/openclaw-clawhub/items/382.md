## Summary

Keep open: ClawHub cleanup policy does not auto-close issues, and current main still has the source path where a normal republish can hide the skill row, mirror that state into search, and remove the previously approved skill from public search while review/scans run.

## What This Changes

Keep an already approved skill searchable while a newly published version is pending scan/review, with safe handling for unchanged-content review reuse.

## Best Possible Solution

Implement a version-aware public projection that keeps the last approved skill card, search digest, embedding, API metadata, and public file surface visible while a new version remains gated, then promotes the new projection only after moderation and scanner gates clear it.

## Reproduction Assessment

Yes, at source level. A normal non-trusted republish can write `moderationStatus: hidden` onto the existing skill row, sync that state into search, demote the previous embedding, and then public search filters the skill out.

## Solution Assessment

No patch is under review. The safest direction is last-approved-version continuity; the direct approaches of allowing hidden pending rows through public filtering or preserving active status during updates conflict with the current moderation policy shown in closed related PRs.
