## Summary

Keep open: this is a valid feature request, and current main still only writes changelog text during publish flows while exposing version changelogs through read/list endpoints. ClawHub cleanup policy keeps issues open for maintainer follow-up.

## What This Changes

The issue requests a CLI command for editing an already-published version changelog or release note.

## Best Possible Solution

Design and add an authenticated, resource-scoped edit API plus CLI command that updates the version row and any latest summary projection atomically without changing artifacts, fingerprints, or integrity metadata.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The high-confidence check is source inspection showing publish-time changelog writes and read-only version endpoints with no edit command on current main.

## Solution Assessment

Unclear as stated: a dedicated CLI command is reasonable, but `clawhub changelog edit <version>` is under-scoped because the resource is not identified. The maintainable path is a server-backed, resource-scoped command/API with authorization, audit behavior, and projection updates.
