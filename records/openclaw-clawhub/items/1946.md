## Repair Work Prompt

Implement a focused changelog edit flow for openclaw/clawhub issue 1946. Add authenticated server-side mutations and HTTP API endpoints for editing an existing skill version changelog and package release changelog, requiring the caller to be the owning user/publisher member or staff. Add CLI commands with explicit resource scope, for example `clawhub skill changelog edit <slug> --version <version> --changelog <text>` and `clawhub package changelog edit <name> --version <version> --changelog <text>`; do not implement a version-only target. Update the version document/release changelog and, when the edited version is the latest version/release, update the parent `latestVersionSummary` in the same mutation. Preserve release files, artifacts, fingerprints, hashes, scan results, and version identity. Add focused tests for CLI request construction, API routing/auth failures, non-owner rejection, successful update, and latest-summary sync. Add a user-facing `CHANGELOG.md` entry. Do not change publish semantics except where shared helpers are needed.
