## Summary

Keep open: current main still only accepts changelog text during publish/sync and returns it through read-only version APIs; editing an existing version changelog would be a new mutable release-metadata capability that needs maintainer policy before implementation.

## What This Changes

The issue requests a `clawhub changelog edit <version>` style command for editing published version changelogs after the fact.

## Best Possible Solution

Define an authenticated owner/staff backend mutation for version/release changelog edits with audit and denormalized update semantics, then expose it through CLI/UI/docs/tests.
