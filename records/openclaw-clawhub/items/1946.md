## Summary

Keep open: current main supports changelog text during publish/sync and read-only version responses, but there is no supported API/CLI path for editing an already-published changelog. This is a new mutable release-metadata feature that needs maintainer policy before implementation.

## What This Changes

The issue requests a command such as `clawhub changelog edit <version>` to modify version changelog text after publication.

## Best Possible Solution

Define authenticated owner/staff backend mutations for supported versioned surfaces, update version rows plus any denormalized latest summaries and audit logs consistently, then expose the approved behavior through CLI/UI/docs/tests.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection shows publish-time changelog input plus read-only changelog output rather than a broken current contract.

## Solution Assessment

No. A CLI command may be a reasonable final surface, but the backend mutation, authorization, audit trail, supported object scope, and denormalized summary behavior should be designed first.
