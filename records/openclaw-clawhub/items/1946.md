## Summary

Keep open: this is a feature request, and current main still has publish-time changelog writes plus read-only version responses but no post-publication changelog edit command or API mutation.

## What This Changes

Request a `clawhub changelog edit <version>`-style command to update an existing version changelog after publishing.

## Best Possible Solution

Add an authenticated, resource-scoped changelog edit flow that updates the version/release row and any latest projections atomically without changing artifacts, file hashes, integrity metadata, or dist tags.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The high-confidence check is source inspection showing publish-time changelog writes and read-only version endpoints with no edit command or mutation on current main.

## Solution Assessment

Unclear as a concrete implementation because the proposed version-only command is under-specified. A CLI command is reasonable only if backed by a server-side mutation that includes resource identity, version, authorization, audit, and projection updates.
