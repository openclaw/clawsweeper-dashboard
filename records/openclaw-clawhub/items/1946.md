## Summary

Keep open: current main still has publish-time changelog input and read-only changelog display, but no post-publish version changelog edit flow. This is a new mutable release-metadata feature that needs maintainer decisions around scope, authorization, audit history, and latest-summary synchronization.

## What This Changes

Request a CLI command for editing an already-published version changelog, such as `clawhub changelog edit <version>`.

## Best Possible Solution

Define an authenticated owner/staff changelog edit model for versioned releases, including audit history and latest-summary propagation, then expose it through the agreed CLI/API/UI surfaces with regression coverage.

## Reproduction Assessment

Not applicable. This is a feature request, and current source shows changelogs as publish-time release metadata plus read-only output rather than a broken existing contract.

## Solution Assessment

No, not as a standalone CLI command yet. A command may be one useful interface, but the maintainable solution first needs backend mutation scope, authorization, audit behavior, affected artifact types, and denormalized latest-summary rules.
