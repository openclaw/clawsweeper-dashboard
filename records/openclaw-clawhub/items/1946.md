Add a resource-scoped CLI/API/backend flow for skills and packages: require a skill slug or package name plus `--version`, accept a non-interactive changelog string, and update the stored version/release changelog plus any latest summary/search projection when the edited version is the current latest. Do not add souls support unless existing code already provides the same version model cleanly.
