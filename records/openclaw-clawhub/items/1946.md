## Summary

Keep open: current main supports creating, displaying, and inspecting version changelogs, but I found no CLI, HTTP API, or owner UI path for editing an already-published version changelog. This is a new feature with published-metadata mutability rules to decide, so it is not a conservative cleanup close.

## What This Changes

The issue requests a way to edit changelog text for an existing published skill version, such as a `clawhub changelog edit <version>` command.

## Best Possible Solution

Define an authenticated version-changelog edit flow with explicit authorization, audit, and denormalization semantics, then add the CLI/API/UI support around that contract.

## Reproduction Assessment

Not applicable. This is a new CLI/API feature request, and source inspection shows no existing changelog edit behavior to reproduce as broken.

## Solution Assessment

No. A CLI command alone is not the best complete solution because backend mutation rules, authorization, audit behavior, and latest-version summary updates need to be designed together.
