## Summary

Keep open: current main treats changelogs as publish-time version metadata and docs describe published version records as immutable, so this is a new product/API feature rather than conservative cleanup.

## What This Changes

The issue requests a CLI command to edit an existing version changelog directly, such as `clawhub changelog edit <version>`.

## Best Possible Solution

Define the changelog mutability contract first; if approved, add an authenticated, audited owner/admin update path and wire it through the CLI and any affected skill/package display summaries consistently.

## Reproduction Assessment

Not applicable. This is a feature request, and current docs plus CLI source show changelogs are accepted during publish rather than through an existing edit workflow.

## Solution Assessment

Unclear. A command like `clawhub changelog edit <version>` may be a reasonable UX, but the maintainable solution needs a backend permission/audit contract and an explicit decision about mutating version metadata.
