## Summary

Keep open: current main supports creating, displaying, and inspecting version changelogs, but it does not expose a CLI, HTTP API, or owner settings path for editing an already-published version changelog. Because this is a ClawHub issue and the change affects the mutability model for published metadata, it needs maintainer product/API review rather than conservative cleanup closure.

## What This Changes

The issue requests a command to edit changelog text for an existing published skill version.

## Best Possible Solution

Define an authenticated changelog edit flow with explicit audit and permission semantics, then update the backend, CLI/API docs, and any latest-version summary or digest state that depends on the edited version.
