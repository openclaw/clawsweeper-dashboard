## Summary

## What This Changes

Add a CLI workflow for authorized publishers to edit the changelog associated with an already-published skill version.

## Best Possible Solution

Define an audited correction policy, then add one owner-authorized backend mutation that atomically updates all changelog mirrors and expose it through the CLI and HTTP API with focused runtime coverage.
