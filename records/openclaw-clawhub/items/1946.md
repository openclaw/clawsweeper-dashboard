## Summary

Keep open: this is a new feature request, and current main still has publish-time changelog writes plus read-only version endpoints but no CLI/API mutation for editing an existing version changelog. ClawHub cleanup policy also keeps issues open unless they are closed by maintainers or implemented through merged work.

## What This Changes

Request a CLI command and backing API path to edit an existing version changelog after publish.

## Best Possible Solution

Add an authenticated slug/name-plus-version changelog edit flow that updates the version row and any latest projections atomically while leaving published artifacts immutable.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives a high-confidence verification path: changelogs are written during publish and exposed through read-only version endpoints, with no edit route or command.

## Solution Assessment

Yes, a dedicated CLI command is a maintainable direction if it is backed by server-side mutations. The safer shape is scoped by skill/package name plus version, not a global version-only command.
