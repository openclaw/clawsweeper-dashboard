## Repair Work Prompt

Implement an authenticated changelog edit flow for existing ClawHub versions. Add resource-scoped CLI commands rather than a version-only global command, for example skill and package changelog edit forms that require a slug/name plus --version and new changelog text. Back the CLI with server-side HTTP/API mutations that authorize the owner or staff, update the stored skillVersions/packageReleases changelog, and update latestVersionSummary/latest projections when the edited version is currently latest. Preserve immutable release artifacts, files, hashes, fingerprints, and scan results. Add audit logging where the surrounding ownership/admin mutations do so. Update public CLI/API docs and focused tests. Do not change publish semantics or package artifact handling.
