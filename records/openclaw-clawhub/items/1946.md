Keep open. This is a new mutable published-version metadata feature, not a broken existing contract; current main supports changelog text at publish time and read-only version output, while existing version rows/releases are rejected rather than edited.

## Best Possible Solution

Define an authenticated backend edit contract first, then expose CLI/API/docs with owner or staff authorization, audit logging, latest-summary and search-digest synchronization, and focused regression tests.
