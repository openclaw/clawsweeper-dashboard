## Repair Work Prompt

Implement this issue as a focused changelog-edit feature. Add authenticated `clawhub` CLI support for editing an existing version changelog using a resource identity plus version, for example skill slug/package name plus `--version` and new changelog text; do not implement a global version-only command. Add v1 API route(s) and Convex mutation(s) for skill and package versions that authorize the owner/publisher or staff, update only `skillVersions.changelog`/`changelogSource` or `packageReleases.changelog`, update `latestVersionSummary` when the edited version is the current latest, and write an audit log. Do not mutate version files, fingerprints, hashes, artifacts, scan results, or package tarball metadata. Add targeted tests for CLI request construction, API auth/failure cases, latest projection updates, and non-latest edits; update CLI/API docs and add a `CHANGELOG.md` entry for the user-facing feature.
