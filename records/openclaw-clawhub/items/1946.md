## Summary

Keep open: current main still only accepts changelog text during publish/sync and exposes version changelogs read-only, with no CLI/API/UI path for editing an existing version changelog. This is a new mutable-release-metadata feature that needs maintainer product and API decisions before implementation.

## What This Changes

Request a `clawhub changelog edit <version>` command for correcting or updating published version changelogs.

## Best Possible Solution

Define an authenticated owner/staff version-changelog update API with audit and denormalized-summary handling, then add CLI/UI/docs/tests around that API.

## Reproduction Assessment

Not applicable. This is a new feature request, and source inspection shows publish/read changelog paths but no existing edit command, API route, or owner UI flow that is failing.

## Solution Assessment

No. A CLI command alone is not the best complete solution because the backend mutation, authorization/audit rules, and denormalized latest-summary semantics need to be designed together.
