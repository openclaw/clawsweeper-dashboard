## Repair Work Prompt

Implement a skill-scoped version changelog edit flow for ClawHub. Add a CLI command that requires a skill slug and version plus changelog text or an editor/stdin-friendly input path; do not implement the example as version-only because versions are not globally unique. Back it with an authenticated API route and Convex mutation that verifies the caller owns or can publish for the skill, updates `skillVersions.changelog` and `changelogSource`, records an audit log entry, and when the edited version is the current latest, updates `skills.latestVersionSummary` and the `skillSearchDigest` projection consistently. Add schema/client response validation, CLI tests, HTTP handler tests, backend mutation tests covering latest and non-latest versions, docs updates, and a `CHANGELOG.md` entry under Unreleased. Do not change publish semantics, tag promotion behavior, package publishing, or unrelated moderation flows.
