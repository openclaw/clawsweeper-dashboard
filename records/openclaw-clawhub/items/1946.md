## Summary

Keep open: current main supports creating and reading changelogs, but not editing an already-published version changelog. This is a feature request that needs maintainer product/API judgment around mutable release metadata, authorization, audit logging, and denormalized latest-summary updates.

## What This Changes

The issue requests a CLI command such as `clawhub changelog edit <version>` for editing version changelogs after publish.

## Best Possible Solution

Define an authenticated owner/staff changelog edit API with audit and denormalized summary updates, then add the CLI command, docs, and focused regression coverage around that API.

## Reproduction Assessment

Not applicable. This is a new feature request, and current source inspection shows publish/read changelog paths but no existing edit command, endpoint, or owner UI path that is failing.

## Solution Assessment

No. A CLI command alone is not the best complete solution because the backend mutation, authorization rules, audit behavior, and denormalized latest-summary semantics need to be designed together.
