## Summary

Keep open: current main still lacks a CLI command or authenticated API mutation for editing an already-published version changelog; changelog text is written during publish/sync and then exposed through read-only version metadata.

## What This Changes

The issue requests a CLI workflow for editing the changelog text of an already-published version.

## Best Possible Solution

Add an authenticated, skill-version-scoped changelog edit flow that updates `skillVersions` and any latest-version projections atomically, with package parity handled as an explicit scope decision.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The high-confidence verification path is source inspection showing publish-time writes and read-only version endpoints with no edit command or route.

## Solution Assessment

Mostly yes: a dedicated CLI command is maintainable if backed by a server-side mutation. The safer shape is slug plus version scoped, rather than a global version-only command.

## Repair Work Prompt

Implement a focused PR for editing published skill version changelogs. Add an authenticated, resource-scoped CLI flow such as `clawhub skill changelog edit <slug> --version <version> --changelog <text>` or an equivalent explicit slug+version shape; do not implement a version-only global command. Back it with a server-side HTTP route and Convex mutation that verifies the caller owns the skill's publisher or has staff privileges, updates the matching `skillVersions.changelog` and `changelogSource: "user"`, and, when the edited version is the latest version, updates `skills.latestVersionSummary` plus `skillSearchDigest` consistently. Keep publish semantics, version files, artifact hashes, moderation scan fields, and package release behavior unchanged unless maintainers explicitly decide to add package parity in this PR. Add CLI schema/types, docs/OpenAPI entries if applicable, and focused tests for auth failure, missing version, latest projection sync, non-latest edits, and CLI request construction.
