## Repair Work Prompt

Implement the changelog edit feature requested by openclaw/clawhub#1946. Add resource-scoped CLI support for editing existing changelog text for skills and packages, using a non-interactive flag such as `--changelog <text>` and a command shape that includes the resource name/slug and version; do not implement a version-only global command because versions are not globally unique. Add server-side HTTP action routes and Convex mutations that require the owner/publisher or staff, update `skillVersions.changelog` or `packageReleases.changelog`, set changelog source to user where applicable, update `skills.latestVersionSummary` or `packages.latestVersionSummary` when the edited version is currently latest, update parent `updatedAt`, and insert audit log entries. Do not mutate version files, storage IDs, fingerprints, integrity hashes, package artifacts, or scan results. Add focused CLI/API/backend tests, update CLI/API docs, and add a user-facing `CHANGELOG.md` entry.
