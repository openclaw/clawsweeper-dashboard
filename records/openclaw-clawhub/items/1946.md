## Summary

Keep open: current main still supports creating, reading, and displaying changelogs, but it does not provide a CLI command, HTTP route, or owner UI for editing an already-published version changelog. Because this is a ClawHub issue and the requested behavior changes published metadata mutability, it needs maintainer product/API judgment rather than cleanup closure.

## What This Changes

The issue requests a dedicated command for editing the changelog text of a specific already-published version.

## Best Possible Solution

Define an authenticated changelog edit flow with explicit scope, authorization, audit behavior, and denormalized-summary updates before adding CLI/API/UI entry points.
