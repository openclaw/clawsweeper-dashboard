## Summary

Keep open: current main still treats version changelogs as publish-time metadata with read-only display/API paths, and the ClawHub cleanup policy keeps issues open unless they are PRs already implemented on main.

## What This Changes

Add a way, such as `clawhub changelog edit <version>`, to modify changelog text for an already-published version.

## Best Possible Solution

Define authenticated owner/staff backend mutations for the approved versioned surfaces, audit each edit, sync denormalized summaries consistently, then expose the workflow through CLI/UI/docs/tests.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection shows publish-time changelog input plus read-only changelog output rather than a broken current contract.

## Solution Assessment

No. A CLI command is a plausible final surface, but the backend mutation, authorization, audit trail, supported object scope, and denormalized summary behavior should be designed first.
