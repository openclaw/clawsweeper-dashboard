## Summary

Keep open: this is a valid feature request, and current main still stores changelog text only through publish-time flows while exposing version changelogs through read APIs. ClawHub cleanup policy also keeps issues open for maintainer follow-up rather than closing feature requests.

## What This Changes

The issue requests a CLI command to edit the changelog for an already-published version.

## Best Possible Solution

Add an authenticated, resource-scoped changelog edit API and CLI flow that updates the version/release row and any latest projections atomically without changing artifacts, fingerprints, file metadata, or integrity data.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The high-confidence check is source inspection showing publish-time changelog writes and read-only version endpoints with no edit command or mutation on current main.

## Solution Assessment

Unclear as stated: a dedicated CLI command is reasonable, but `clawhub changelog edit <version>` is under-scoped. The maintainable path is a server-backed, resource-scoped command/API with authorization, audit semantics, and projection updates.
