## Repair Work Prompt

Implement a resource-scoped changelog edit feature for existing ClawHub versions. Add authenticated server-side mutations and HTTP routes for skills and packages, such as editing `/api/v1/skills/:slug/versions/:version/changelog` and `/api/v1/packages/:name/versions/:version/changelog`, with owner/staff authorization and audit logging. Add CLI commands under the existing resource groups rather than a global version-only command, for example `clawhub skill changelog edit <slug> --version <version> --changelog <text>` and the package equivalent. Update the version row/release row, set user-authored changelog source where applicable, and if the edited version is the latest, update the parent latest summary and any search/list projection that stores the changelog. Do not modify release files, artifacts, fingerprints, scan results, or download behavior. Add focused unit tests for API handlers and CLI requests, and add a `CHANGELOG.md` entry for the user-facing feature.
