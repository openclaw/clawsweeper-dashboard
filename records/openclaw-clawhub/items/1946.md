## Summary

Keep open: current main still supports changelog text only at publish time and through read-only version surfaces, and ClawHub policy keeps issues open rather than auto-closing them. This is a new mutable-release-metadata feature that needs maintainer product/API judgment.

## What This Changes

Add a command such as `clawhub changelog edit <version>` to modify published version changelog entries.

## Best Possible Solution

Define an authenticated owner/staff changelog update API that updates version rows and denormalized summaries consistently, then add the CLI command, docs, and focused regression coverage.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection shows publish/read changelog paths but no existing edit command, endpoint, or owner UI flow that is failing.

## Solution Assessment

No. A CLI command alone is not the best complete solution because the backend mutation, authorization rules, audit behavior, and denormalized latest-summary semantics need to be designed together.
