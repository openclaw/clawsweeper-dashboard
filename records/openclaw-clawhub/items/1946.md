## What This Changes

Request a CLI command for editing an existing version changelog after publication.

Add an authenticated, resource-scoped changelog edit flow that updates the version row and any latest/search projections atomically while preserving release file immutability.

## Repair Work Prompt

Implement a resource-scoped changelog edit feature for existing published versions. Add a CLI flow such as `clawhub changelog edit <skill-or-package> <version>` or an equivalent existing-command subcommand, backed by an authenticated server-side mutation/API route. Scope the edit to the owning user or staff/admin permissions, update the stored version/release changelog and any latest/search projections consistently, and do not mutate release files, fingerprints, manifests, or package archives. Decide explicitly whether both skills and packages are supported; if only skills are implemented, document/package-scope should remain unchanged. Add focused tests for unauthorized edits, missing version/resource, successful update, and read API output after edit. Do not add unrelated publishing changes.
