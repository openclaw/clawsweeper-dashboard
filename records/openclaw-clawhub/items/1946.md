## Summary

Keep open: this is a valid feature request, and current main still only supports changelog text during publish/read flows with no CLI or API path to edit an already-published version changelog. ClawHub cleanup policy keeps issues open for maintainer follow-up.

## What This Changes

Request adding a command or workflow to edit the changelog for an already-published version.

## Best Possible Solution

Define an authenticated, resource-scoped changelog edit flow that updates the version row and derived latest/search summaries atomically while preserving published artifacts and integrity metadata.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection on current main shows changelog text is accepted at publish time and exposed through read-only version endpoints, but there is no edit command or mutation.

## Solution Assessment

Unclear as stated: a dedicated CLI command may be appropriate, but `<version>` alone is under-scoped because versions are resource-scoped. The maintainable path should include a server-backed resource selector, authorization, audit behavior, and projection synchronization.
