## Summary

Keep open: this is a valid feature request, and current main still has publish-time changelog input plus read-only version endpoints, but no CLI or API flow for editing an already-published version changelog. ClawHub cleanup policy also keeps issues open for maintainer follow-up.

## What This Changes

Request a command or workflow to edit changelog text for an already-published version.

## Best Possible Solution

Add an authenticated, resource-scoped changelog edit flow that updates the version/release row and any latest projections atomically without changing artifacts, files, fingerprints, or integrity metadata.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The high-confidence check is source inspection on current main showing publish-time changelog writes and read-only version endpoints, with no edit command or mutation.

## Solution Assessment

Unclear as stated: a dedicated command is plausible, but `clawhub changelog edit <version>` is under-scoped. The maintainable solution should be resource-scoped and server-backed with authorization, audit, and projection updates.
