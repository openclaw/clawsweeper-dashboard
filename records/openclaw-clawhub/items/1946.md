## Summary

Keep open: this is a feature request for a new edit path, and current main still treats changelogs as publish-time data exposed by read APIs rather than mutable version metadata. ClawHub cleanup policy keeps issues open for maintainer follow-up.

## What This Changes

Request a CLI command to edit changelog text for an already-published version.

## Best Possible Solution

Define an authenticated resource-and-version-scoped edit flow that updates the version/release row and any latest projections atomically without changing artifacts, files, fingerprints, or integrity metadata.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives a high-confidence check that changelogs are written at publish time and exposed by read endpoints without a current edit command or mutation.

## Solution Assessment

Unclear as stated because `clawhub changelog edit <version>` is under-scoped. A maintainable implementation should include resource identity, version, authorization, audit/provenance behavior, and latest-summary updates.
