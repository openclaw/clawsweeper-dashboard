## Repair Work Prompt

Implement an authenticated changelog edit flow for existing skill and package versions. Add server-side mutations/routes that accept a skill slug or package name, a version, and changelog text; require owner or staff authorization; patch `skillVersions.changelog`/`packageReleases.changelog`; set an appropriate changelog source when applicable; and update `skills.latestVersionSummary` or `packages.latestVersionSummary` when the edited version is the current latest. Preserve files, fingerprints, integrity fields, artifacts, stats, and release timestamps. Add CLI commands following existing Commander patterns, preferring resource-scoped forms such as skill/package changelog edit rather than a global version-only command. Add focused tests for API auth, non-owner rejection, latest projection sync, non-latest edits, CLI request shape, and package/skill behavior. Include a `CHANGELOG.md` entry for the user-facing feature. Do not implement soul changelog editing unless the existing CLI/API surface already makes the scope unambiguous.
