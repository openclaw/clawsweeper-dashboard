## Summary

Keep open: current main still treats changelogs as publish-time release metadata and read-only output, so this is a new mutable release-metadata feature that needs maintainer scope and policy decisions.

## What This Changes

The issue requests a command for editing published version changelogs directly, such as `clawhub changelog edit <version>`.

## Best Possible Solution

Define an authenticated owner/staff version changelog edit flow with audit history and latest-summary synchronization, then expose it through CLI/API/UI/docs/tests once the product boundary is agreed.
