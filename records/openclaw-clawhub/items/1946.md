## Summary

Keep open: current main can create and display version changelogs at publish time, but it still lacks a CLI command or authenticated API path to edit an existing version changelog after publish.

## What This Changes

Request a CLI workflow for editing the changelog/release notes attached to a specific published version.

## Best Possible Solution

Add a resource-scoped authenticated changelog edit flow that updates the stored version and any latest-version search/listing summaries consistently.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The high-confidence verification path is source/API inspection showing publish-time changelog writes and read-only version endpoints with no edit command.

## Solution Assessment

Yes, a dedicated command is a reasonable direction if backed by a server-side mutation. The safer shape is an explicit skill-scoped command first, with package support added only if maintainers want the broader product scope.
