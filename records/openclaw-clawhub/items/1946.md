## Summary

Keep open: current main still has publish-time and read-only changelog surfaces, but no path for editing an existing version changelog. This is a new mutable release-metadata feature that needs maintainer API, authorization, audit, and denormalization decisions before implementation.

## What This Changes

Add a command/API flow for editing changelog text on an already-published version.

## Best Possible Solution

Define an authenticated owner/staff backend mutation for version-changelog edits that updates the version row and any latest summary or digest copies, then add CLI/UI/docs/tests around the settled scope.

## Solution Assessment
