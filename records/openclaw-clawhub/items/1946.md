## Summary

Keep open: this is a valid ClawHub feature request, and current main still only records changelog text during publish/sync while exposing existing version changelogs through read-only APIs.

## What This Changes

Request a `clawhub` command for editing an already-published version changelog.

## Best Possible Solution

Add a resource-scoped, authenticated edit flow for existing version changelogs, updating the canonical version row plus any latest-version summaries/search digests that mirror it.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The high-confidence verification path is source/API inspection showing publish-time changelog writes and read-only version endpoints, with no edit command or update route on current main.

## Solution Assessment

Yes, a dedicated command is a reasonable direction if it is backed by a server-side mutation. The maintainable shape should be explicit about the target resource, such as a skill-scoped slug plus version, rather than a global version-only command.
