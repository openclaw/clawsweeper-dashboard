## What This Changes

The issue requests a command for editing version changelogs after publication, such as `clawhub changelog edit <version>`.

## Best Possible Solution

Add an authenticated, resource-scoped changelog edit flow that patches changelog metadata and latest projections atomically while keeping published artifacts immutable.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The high-confidence check is source inspection showing publish-time changelog writes and read-only version endpoints with no edit route or CLI command.

## Solution Assessment

Yes, a dedicated command backed by server-side mutation is a maintainable direction, but it should be scoped by skill/package/soul identifier plus version rather than version alone.
