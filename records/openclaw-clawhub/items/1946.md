## Summary

Keep open: current main still lacks an edit flow, and this is a feature/product decision rather than conservative cleanup. The existing implementation creates changelogs during publish and exposes them through read surfaces, but no CLI/API mutation updates a published version's changelog.

## What This Changes

Request a CLI workflow such as `clawhub changelog edit <version>` to update release-note text for an already-published skill version.

## Best Possible Solution

Define an authenticated version-changelog edit flow across the CLI and v1 API with clear authorization, audit semantics, and synchronized canonical plus denormalized data updates.
