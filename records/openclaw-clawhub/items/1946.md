## Summary

Keep open: current main supports entering or auto-generating changelog text during publish flows and reading persisted changelogs afterward, but there is no post-publication edit path. For ClawHub issues, this should stay open for maintainer product/policy follow-up rather than cleanup-close.

## What This Changes

The issue requests a command to edit already-published version changelog text, such as `clawhub changelog edit <version>`.

## Best Possible Solution
