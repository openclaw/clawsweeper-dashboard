## Repair Work Prompt

Implement editable published version changelogs for ClawHub without mutating release files, fingerprints, package artifacts, hashes, or scan results. Add resource-scoped CLI commands and API/backend support for editing changelog metadata on an existing skill or package version, requiring an authenticated owner/staff actor. Prefer explicit resource scoping such as skill slug or package name plus `--version`; do not implement a global version-only edit command. Update latest-version summaries/search projections when the edited version is currently latest, and add audit/authorization behavior consistent with nearby owner/staff mutations.
