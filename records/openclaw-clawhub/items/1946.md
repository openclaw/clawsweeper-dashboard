## Summary

Keep open: current main supports changelog text during publish/sync and exposes persisted changelogs through read-only version APIs, but post-publish changelog editing is a new mutable release-metadata feature that needs maintainer policy and design.

## What This Changes

Request a CLI command for editing an already-published version changelog, such as `clawhub changelog edit <version>`.

## Best Possible Solution

Define authenticated owner/staff backend mutations for the supported versioned resources, update version rows plus denormalized latest summaries and audit records consistently, then expose the approved behavior through CLI/UI/docs/tests.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection shows publish-time changelog input plus read-only changelog output rather than a broken current contract.

## Solution Assessment

No. A CLI command is a reasonable eventual surface, but the backend mutation, authorization, audit trail, supported resource scope, and denormalized summary behavior should be designed first.
