## Summary

Keep open: ClawHub cleanup policy keeps issues open, and current main still only accepts changelogs during publish/sync while exposing existing version changelogs as read-only data. A mutable changelog flow needs maintainer product/API decisions.

## What This Changes

The issue requests a CLI command to edit changelogs for already-published versions.

## Best Possible Solution

Define a resource-scoped API and CLI flow for authorized changelog edits that updates the version row and any latest projections atomically without touching artifacts, file hashes, or integrity metadata.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The high-confidence check is source inspection showing publish-time changelog writes and read-only version endpoints with no edit mutation or CLI command on current main.

## Solution Assessment

Unclear as a concrete implementation because `clawhub changelog edit <version>` is under-scoped. A dedicated command could be appropriate if backed by a server-side mutation that includes resource type/name, version, authorization, audit logging, and projection updates.
