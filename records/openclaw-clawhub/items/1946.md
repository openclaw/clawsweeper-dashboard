## Summary

Keep open. This is a new post-publish changelog editing feature with product and security implications, and current main still only supports changelog text during publishing plus read-only version output.

## What This Changes

Request to add a command such as `clawhub changelog edit <version>` for editing published version changelogs.

## Best Possible Solution

Define an authenticated backend edit contract first, then expose it through CLI/API/docs with owner or staff authorization, audit logging, latest-summary synchronization, and focused regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that current main has publish-time changelog input and read-only changelog output, but no post-publish edit command or mutation.

## Solution Assessment

Unclear. A standalone CLI command is not enough by itself; the maintainable solution needs an authenticated backend mutation/API plus explicit authorization, audit, and latest-summary update semantics.
