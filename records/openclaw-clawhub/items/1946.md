## What This Changes

Request a CLI workflow for editing existing version changelog or release-note text after publish.

Add an authenticated, resource-scoped changelog edit flow that patches only version/release metadata and synchronizes latest/search projections atomically.

## Solution Assessment

## Repair Work Prompt

Implement a resource-scoped changelog edit feature for already-published ClawHub versions. Add authenticated CLI commands under the existing skill/package namespaces rather than a global version-only command; require the skill slug or package name plus an explicit semver `--version`, and accept changelog text through existing CLI conventions. Add server-side mutations/routes requiring owner or staff authorization that patch only `changelog`/`changelogSource` on the targeted `skillVersions` row or `packageReleases` row. If the edited version is the latest version/release, update `latestVersionSummary`; for skills, also keep `skillSearchDigest` synchronized. Preserve release files, fingerprints, artifacts, hashes, security scan data, and duplicate-version publish behavior. Add focused unit/API/CLI tests and docs, and include a CHANGELOG.md entry for the user-facing CLI/API feature without forcing bot-only thanks credit.
