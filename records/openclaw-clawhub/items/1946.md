## Summary

Keep open: current main still has changelog creation/read paths but no CLI, API, or UI path for editing an already-published version changelog. This is a new mutable-release-metadata feature that needs maintainer product/API decisions, and this repository policy keeps issues open.

## What This Changes

Request a command such as `clawhub changelog edit <version>` to update changelog text for an existing published version.

## Best Possible Solution

Define an authenticated owner/staff version-changelog update API with audit and denormalized-summary handling, then layer CLI/UI/docs/tests on that API.

## Reproduction Assessment

Not applicable. This is a new feature request, and source inspection shows publish/read changelog paths but no existing edit command, API route, or owner UI flow that is failing.

## Solution Assessment

No. A CLI command alone is not the best complete solution because backend mutation semantics, authorization, audit behavior, and latest-summary updates need to be designed together.
