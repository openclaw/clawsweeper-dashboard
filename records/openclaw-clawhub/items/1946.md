## Summary

Keep open: this is a valid feature request, and current main still lacks a CLI or API path to edit changelogs on already-published versions. The command/API shape needs maintainer product decisions, so this should not be auto-closed or sent straight to an automated fix lane.

## What This Changes

The issue requests a command for editing changelogs on already-published versions, such as `clawhub changelog edit <version>`.

## Best Possible Solution

Add an authenticated, resource-scoped changelog edit flow that patches the version/release row and any latest projections atomically without changing artifacts, files, fingerprints, or integrity metadata.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The high-confidence check is source inspection showing publish-time changelog writes and read-only version endpoints with no edit command or mutation on current main.

## Solution Assessment

Unclear as stated: a dedicated CLI command is reasonable, but `changelog edit <version>` is not resource-scoped enough. The maintainable path is a server-backed command/API that identifies the skill/package/soul, enforces authorization, and updates projections consistently.
