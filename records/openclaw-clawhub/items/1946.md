## Summary

Keep open: current main still treats changelogs as publish-time metadata plus read-only version history, and there is no CLI/API path to edit an existing version changelog. The related changelog-preview issue overlaps, but it tracks package preview/manual-edit behavior rather than this post-publish edit command.

## What This Changes

The issue requests a CLI command for editing a published version changelog after release.

## Best Possible Solution

Make a maintainer decision on whether version changelogs are mutable; if yes, add a scoped authenticated API and CLI flow with audit logging, docs, and focused Convex/CLI tests.
