## Repair Work Prompt

Implement authenticated version changelog editing for published ClawHub versions. Add a resource-scoped CLI/API flow, for example `clawhub skill changelog edit <slug> <version> --changelog <text>`; do not add a global version-only command. Add the server-side authenticated mutation and HTTP route for skills first, updating `skillVersions.changelog` and `changelogSource: "user"`; when the edited version is the latest, update `skills.latestVersionSummary` and `skillSearchDigest.latestVersionSummary` atomically. Deliberately decide whether package release parity belongs in the same PR; if included, add the analogous package command/route and update `packageReleases.changelog` plus `packages.latestVersionSummary` for latest releases. Preserve release file contents, fingerprints, storage records, and version numbers. Use existing API-token auth, ownership/staff authorization patterns, audit logging conventions where present, and add docs plus a `CHANGELOG.md` entry for the user-facing CLI feature.
