## Summary

Keep open. This issue requests a new post-publish changelog editing capability; current main treats version records as immutable, accepts changelogs only during publish/sync, and exposes version changelogs through read-only UI/API surfaces.

## What This Changes

Add a CLI workflow for editing an existing version changelog after publication.

## Best Possible Solution

Define an approved post-publish changelog edit contract with authenticated backend/API support, CLI UX, ownership checks, audit logging, latest-summary synchronization, docs, and focused tests, or explicitly preserve immutable version records.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs inspection gives high confidence that current main supports changelog input at publish time and read-only changelog output afterward, but no post-publish edit path.

## Solution Assessment

Unclear. A CLI command alone is not the best complete solution; the maintainable path needs an explicit backend mutation/API contract plus authorization, audit, and latest-summary semantics.
