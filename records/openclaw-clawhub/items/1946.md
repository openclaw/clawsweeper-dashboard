## Summary

Keep open: current main supports publishing and reading version changelogs, but there is still no command, HTTP API, or owner UI flow for editing an already-published version changelog. This is a ClawHub feature request that needs maintainer product/API judgment about mutable published metadata.

## What This Changes

Add a ClawHub command to edit the changelog text for a specific already-published version.

## Best Possible Solution

Define an authenticated changelog edit flow with explicit permission, audit, and latest-summary semantics, then update the backend, CLI/API docs, and focused regression coverage together.

## Solution Assessment

No. A CLI command alone is not the complete solution because the backend mutation, authorization rules, audit behavior, and denormalized latest-summary updates need to be designed together.
