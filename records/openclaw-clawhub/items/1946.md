## What This Changes

Add an authenticated, skill-version-scoped changelog edit flow that updates the version row and latest-version projections atomically, with package parity handled as a separate explicit scope decision if needed.

## Repair Work Prompt

Implement a focused ClawHub skill changelog edit flow for already-published skill versions. Add an authenticated API route and Convex mutation, using a REST shape consistent with the existing API such as `POST /api/v1/skills/{slug}/versions/{version}/changelog`, that requires the caller to be the owner/publisher member or staff, validates the changelog text, updates `skillVersions.changelog` and `changelogSource`, and if the edited version is the current latest version also updates `skills.latestVersionSummary` and `skillSearchDigest` through existing digest helpers. Add a CLI command with explicit resource scope, for example `clawhub skill changelog edit <slug> <version> --changelog <text>`; use an existing interactive pattern only if one already fits, otherwise keep it flag/stdin based. Add schema response validators, docs in `docs/cli.md`, and unit tests for CLI request shape, HTTP handler behavior, authorization failure, non-latest version edits, and latest-version denormalization. Do not change publish semantics, and do not implement broad package or soul parity unless maintainers explicitly expand the scope.
