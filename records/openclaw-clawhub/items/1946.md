## Repair Work Prompt

Implement authenticated support for editing the changelog of an already-published skill version for openclaw/clawhub#1946. Use a resource-scoped command/API shape, not a version-only global command; for example `clawhub skill changelog edit <slug> <version> --changelog <text>` or an equivalent established CLI pattern. Add a server-side authenticated mutation behind an HTTP API route that verifies the caller can manage the skill, updates `skillVersions.changelog` and sets `changelogSource` to `user`, and if the edited version is the current latest also updates `skills.latestVersionSummary` and `skillSearchDigest.latestVersionSummary` consistently. Add response schemas in the schema packages as needed, CLI tests and Convex/API tests, docs/CLI help, and a `CHANGELOG.md` entry. Do not implement package release editing unless it is deliberately included with matching tests and docs; otherwise leave package parity as an explicit follow-up note.
