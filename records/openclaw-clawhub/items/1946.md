## Repair Work Prompt

Implement version changelog editing for published ClawHub skill versions. Add a resource-scoped CLI command, for example `clawhub skill changelog edit <slug> <version> --changelog <text>` or an equivalent established CLI shape; do not implement a global version-only command. Add an authenticated HTTP API route and Convex mutation that verifies ownership/staff permissions, updates `skillVersions.changelog` and sets `changelogSource` to `user`, and if the edited version is the current latest updates `skills.latestVersionSummary` plus the `skillSearchDigest` projection atomically. Add schema/API client typings as needed, CLI tests, API handler tests, backend mutation/projection tests, docs/CLI help, and a `CHANGELOG.md` entry because this is user-facing. Keep package-release changelog editing out of scope unless the maintainer explicitly expands the task; document that scope in tests or docs rather than changing package release behavior incidentally.
