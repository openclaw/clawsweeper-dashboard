## Summary

Keep open: this is a new feature request, and current main still only supports changelog creation at publish time plus read-only version inspection. For ClawHub issues, this workflow should not auto-close useful open issues, and this one needs maintainer product/API judgment before implementation.

## What This Changes

Add a way, such as `clawhub changelog edit <version>`, to edit an already-published version changelog.

## Best Possible Solution

Define an authenticated changelog edit flow with explicit authorization, audit, and denormalized-summary semantics, then add matching CLI/API documentation and focused tests.

## Reproduction Assessment

Not applicable. This is a request for a new CLI/API editing capability, and source inspection shows changelog writes exist only during publish rather than as broken existing behavior.

## Solution Assessment

No. A CLI command alone is not the best complete solution because the backend mutation, authorization rules, audit behavior, and latest-version summary updates need to be designed together.
