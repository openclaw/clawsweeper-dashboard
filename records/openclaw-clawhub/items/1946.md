## What This Changes

The issue requests a command or workflow to edit changelog text for an already-published version.

## Best Possible Solution

Add an authenticated, resource-scoped changelog edit API and CLI wrapper that updates only release-note fields and synchronized projections atomically while preserving artifacts, files, fingerprints, and integrity metadata.

Unclear as stated: a dedicated command is reasonable, but `clawhub changelog edit <version>` is under-scoped because versions are per resource. The maintainable path is a server-backed, resource-scoped API plus CLI command with authorization, audit behavior, and projection updates.
