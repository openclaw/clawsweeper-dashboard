## Summary

Keep open: current main still only accepts changelog text during publish/sync and exposes persisted version changelogs through read-only API/CLI/UI paths. This is a new mutable release-metadata feature, and this ClawHub cleanup policy keeps issues open rather than auto-closing them.

## What This Changes

Request adds a workflow for editing an already-published version changelog, for example `clawhub changelog edit <version>`.

## Best Possible Solution

Define authenticated owner/staff backend mutations for approved versioned surfaces, audit each edit, update latest summaries consistently, then expose the behavior through CLI/UI/docs/tests.

## Reproduction Assessment

## Solution Assessment

No. A CLI command is a plausible eventual surface, but the backend mutation, authorization, audit trail, supported object scope, and denormalized summary behavior should be designed first.
