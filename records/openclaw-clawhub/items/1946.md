## Summary

Keep open: current main still has publish-time changelog input and read-only version changelog surfaces, but no edit command/API/UI. This is a new mutable-release-metadata feature that needs maintainer product/API judgment before implementation.

## What This Changes

Request a CLI command for editing a published version changelog after release.

## Best Possible Solution

Define an authenticated owner/staff changelog update API that updates version rows and denormalized summaries consistently, then add the CLI command, docs, and focused regression coverage.

## Solution Assessment

No. A CLI command alone is not the best complete solution because the backend mutation, authorization rules, audit behavior, and denormalized latest-summary semantics need to be designed together.
