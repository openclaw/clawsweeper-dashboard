Request to add a CLI workflow such as `clawhub changelog edit <version>` for editing already-published version changelogs or release notes.

## Best Possible Solution

Define an authenticated backend edit contract for skill versions and package releases, with owner/staff authorization, audit logging, summary/digest synchronization, CLI/docs coverage, and focused regression tests.
