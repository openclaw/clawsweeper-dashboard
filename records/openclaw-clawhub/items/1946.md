## Summary

Keep open: this is a feature request, and current main still records changelogs at publish time and exposes them read-only without a CLI/API path to edit an existing version changelog. ClawHub cleanup policy keeps issues open for maintainer follow-up.

## What This Changes

Request a CLI workflow, such as `clawhub changelog edit <version>`, to edit published version changelog or release-note text after publish.

## Best Possible Solution

Add an authenticated, resource-scoped changelog edit API and CLI flow only after the mutability, authorization, audit, and projection-sync rules are defined.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The high-confidence check is source inspection showing publish-time changelog writes and read-only version endpoints with no edit command or mutation on current main.

## Solution Assessment

Unclear as a concrete implementation because the proposed version-only command is under-scoped. A CLI command is reasonable only if backed by a server-side mutation with resource identity, authorization, audit behavior, and projection updates.
