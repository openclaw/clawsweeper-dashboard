## What This Changes

The PR records vector-search scores by skill ID during hydration, applies that score when a lexically retained duplicate lacks an embedding ID, and adds a regression test for the overlap case.

## Best Possible Solution

Keep the direct-first recall and deduplication behavior, restore the vector score for the same hydrated skill as proposed, and confirm the affected query with redacted before/after output from a real Convex-backed environment before landing.

## Reproduction Assessment

Yes at source level: current `main` retains the direct lexical duplicate first and then computes zero vector contribution because that entry has no `embeddingId`. The linked production observations and focused mocked test support the path, although no real after-fix runtime run is attached.

## Solution Assessment

Yes, this is the narrowest maintainable correction visible in the current design because it preserves merge and recall ordering while transferring only the already-returned vector score to the same skill. It should not be merged until the runtime hydration path is demonstrated in a real Convex-backed setup.
