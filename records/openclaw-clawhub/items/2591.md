The branch records hydrated vector scores by skill ID so a retained direct lexical match receives its vector contribution, and adds a regression test for that duplicate-recall path.

## Best Possible Solution

Keep the established lexical-first deduplication order, apply each hydrated vector score through the stable skill ID, and confirm through a real backend search that the affected result receives exactly one vector contribution and the expected ranking.
