## Summary

Current main still loses the vector contribution when direct lexical recall wins skill-ID deduplication, and the proposed stable-ID score association is a focused correction with no concrete code defect found. The PR should remain open because the contributor has supplied source analysis and mocked coverage but not the repository-required after-fix proof from a real Convex deployment or live API search.

## What This Changes

The PR records hydrated vector scores by skill ID so lexical-retained duplicate skills receive their vector contribution, and adds regression coverage for that overlap.

## Best Possible Solution

Keep the existing lexical-first recall precedence while associating vector similarity with stable skill identity, then verify through a real Convex-backed `news` search that the retained lexical result receives its vector contribution and the expected ranking improves.

## Reproduction Assessment

Yes at source level with high confidence: direct lexical recall can retain an entry without `embeddingId` while skill-ID deduplication drops its hydrated duplicate, causing current-main ranking to use zero vector contribution. The related issue also supplies production-score evidence, but the PR does not include an after-fix runtime reproduction.

## Solution Assessment

Yes provisionally: associating the hydrated vector score with the stable skill ID is narrower and safer than changing established recall precedence, and no concrete implementation defect was found. Real Convex-backed proof is still required to establish that it resolves the observed production ordering.
