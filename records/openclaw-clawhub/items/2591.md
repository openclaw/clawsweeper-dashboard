## Summary

The patch is a focused and plausible fix for a reproduced search-ranking defect, but it should remain open because the external contributor has provided only mocked coverage and no after-fix evidence from a real Convex-backed search.

## What This Changes

The PR records hydrated vector scores by canonical skill ID, applies them when lexical-first deduplication retains an entry without an embedding ID, and adds a regression test for that overlap.

## Best Possible Solution

Keep lexical-first deduplication, associate each vector-recalled skill’s computed score with its canonical skill ID, and verify through the real search endpoint that the retained result gains the expected vector contribution.

## Reproduction Assessment

Yes. Current-main source establishes the zero-vector-score path, and the linked production API breakdown identifies an affected `news` result with the missing contribution.

## Solution Assessment

Yes. Associating an already-computed vector score with the canonical skill ID fixes the loss without changing recall order or introducing a competing search implementation.
