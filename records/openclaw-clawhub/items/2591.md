## Summary

Current main still has the reported score-loss defect, and this focused patch follows the existing search architecture without introducing a blocking correctness or security problem. The PR should remain open for review, but contributor-supplied proof from a real Convex-backed search is still required before merge.

## What This Changes

The PR preserves vector relevance scores by skill ID when a direct lexical result shadows the same vector-hydrated skill and adds a regression test for that merge path.

## Best Possible Solution

Keep the skill-ID score association while preserving the established recall order, then confirm the affected query and score through a redacted real Convex-backed search run before merge.

## Reproduction Assessment

Yes. Current-main source deterministically retains a direct lexical duplicate without its embedding ID, and the linked live `news` query report is consistent with the resulting missing vector contribution.

## Solution Assessment

Yes. Associating the already-returned vector score with the hydrated skill identity is a narrow repair that preserves existing recall and merge ordering instead of creating a parallel search path.
