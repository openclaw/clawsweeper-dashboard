## Summary

Current main has a confirmed ranking defect when lexical-first deduplication retains a skill without its embedding identifier, and this focused patch addresses that root cause without changing recall order. The PR should remain open because its reported validation is mocked-only and repository policy requires a real Convex-backed validation path before merge.

## What This Changes

The PR records hydrated vector scores by canonical skill ID, applies them to lexical-retained duplicates during ranking, and adds a regression test for the overlap.

## Best Possible Solution

Keep lexical-first deduplication while associating each hydrated score with the canonical skill ID, and confirm through the real search endpoint that the retained result gains the expected vector component.

## Reproduction Assessment

Yes. Current-main source establishes the zero-vector-score path, and the linked production API breakdown shows an affected `news` result missing that contribution.

## Solution Assessment

Yes. Associating an already-computed vector score by canonical skill ID fixes the loss without changing recall order, deduplication precedence, or the public search contract.
