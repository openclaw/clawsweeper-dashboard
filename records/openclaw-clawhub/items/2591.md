- `P2`: This fixes a reproducible search-ranking defect affecting a bounded subset of lexical/vector duplicate results.

## Summary

Current main still has the lexical-first deduplication path that drops the hydrated candidate’s vector contribution, and this focused patch appears to correct it without changing recall order or eligibility. The PR should remain open because the contributor has provided only mocked test evidence, not the required after-fix behavior from a real ClawHub/Convex search setup.

## What This Changes

The PR records vector scores by skill ID so a retained direct lexical match can inherit the score of its deduplicated hydrated vector candidate, and adds a regression test for that case.

## Best Possible Solution

Retain the narrow skill-ID score propagation and merge it after a real Convex-backed search demonstrates that the affected lexical result gains its vector contribution while existing recall and ordering remain stable.

## Reproduction Assessment

Yes at source level: current main deterministically keeps the direct-prefix entry during skill-ID deduplication and then assigns zero vector score because that retained entry lacks `embeddingId`. The linked issue also supplies a concrete `news`/`the-news` case, although this review did not execute a live deployment.

## Solution Assessment

Yes, subject to runtime proof. Keying the already-returned vector score by stable skill ID preserves the existing merge winner and ordering rules without adding a parallel recall path or changing user configuration.
