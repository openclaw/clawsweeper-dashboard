## What This Changes

The PR records hydrated vector similarity by skill ID and reuses it when lexical-first deduplication retains the same skill without an embedding ID, with a regression test for that overlap.

## Best Possible Solution

Keep the narrow score-by-skill preservation and merge it after redacted live Convex-backed search output confirms that the lexical-retained skill receives its hydrated vector contribution.

## Reproduction Assessment

Yes, from current-main source: lexical entries can win skill-ID deduplication without `embeddingId`, after which the scorer supplies zero vector contribution. The PR has not demonstrated that path against a running Convex deployment.

## Solution Assessment

Yes. Keying the already-fetched vector score by canonical skill ID preserves existing recall precedence and is narrower and safer than changing merge order.
