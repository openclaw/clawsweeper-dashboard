## Summary

Keep open for normal PR review: the patch targets a real source-level search scoring defect and the implementation is narrow, but current main does not yet include it and the external PR still needs after-fix real behavior proof before merge.

## What This Changes

The PR stores vector scores by skill id and uses them when a direct lexical result without an embedding id shadows the hydrated vector result, with a regression test for that overlap.

## Best Possible Solution

Land this or an equivalent score-by-skill attribution fix after redacted real search output confirms lexical-retained skills receive their vector contribution on the final merge result.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: current main keeps direct lexical entries before hydrated duplicates and scores only by the retained entry's embeddingId, so lexical-retained entries without embeddingId can lose vector score; I did not run a live production query.

## Solution Assessment

Yes. Score attribution by skill id is the narrowest maintainable fix because it preserves recall and merge ordering while restoring the vector contribution; it still needs real behavior proof before merge.
