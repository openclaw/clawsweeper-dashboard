## Summary

Keep open: ClawHub cleanup policy keeps issues open, and current main still models skill transfers as user-to-user rather than publisher or organization targets. Related #1431 and #1675 remain open, with #1635 as the open, unmerged implementation candidate.

## What This Changes

Request to transfer existing personally owned skills to an organization so other organization members can publish and update them.

## Best Possible Solution

Land or replace #1635 with a publisher-targeted transfer model that supports personal-to-org, org-to-user, and org-to-org transfers while preserving existing user transfers, aliases, stats, audit logs, and canonical install URLs.

## Reproduction Assessment

Yes, from source inspection. The CLI and HTTP API still send and require `toUserHandle`, while the backend resolves that handle to active users and excludes organization publishers, matching the reported `User not found` path.

## Solution Assessment

Yes, a publisher-targeted transfer model is the maintainable solution. A resolver-only patch would not cover the schema, acceptance, role, audit, alias, and URL-preservation semantics needed for organization ownership transfers.
