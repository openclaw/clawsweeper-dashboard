## Summary

Keep open: current main gives owners a publish-time migration path, but the reported `clawhub transfer` command/API path still resolves only user recipients, and ClawHub cleanup policy keeps issues open for maintainer triage.

## What This Changes

Requests a way to move existing personal-account skills into an organization so organization members can publish and update them.

## Best Possible Solution

Keep the explicit `--owner` plus `--migrate-owner` publish path as the current migration route, and consolidate any direct org-transfer work under https://github.com/openclaw/clawhub/issues/1675 or https://github.com/openclaw/clawhub/pull/1635.

## Reproduction Assessment

Yes, for the direct transfer path. Source inspection shows the CLI/API still send `toUserHandle`, the backend resolves only active users or personal user publishers, and org publisher handles fall through to `User not found`.

## Solution Assessment

Unclear. The current publish `--migrate-owner` path is a maintainable answer for owners who can republish, but direct transfer-to-org support is a separate authorization-sensitive product decision.
