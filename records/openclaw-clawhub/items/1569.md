## Summary

Keep open: current main now provides a documented publish-time owner-migration path, but the reported `clawhub transfer` command/API path is still user-targeted and cannot directly target org publishers. Related org-transfer work remains open, so this issue still needs maintainer triage rather than cleanup closure.

## What This Changes

Request support or guidance for moving existing skills from a personal publisher to an organization publisher so org members can publish updates.

## Best Possible Solution

Keep the documented `--owner` plus `--migrate-owner` publish flow as the immediate migration path, and use https://github.com/openclaw/clawhub/issues/1675 or https://github.com/openclaw/clawhub/pull/1635 to decide whether direct org-targeted transfer commands/API should also land.

## Reproduction Assessment

Yes, for the direct transfer path. Source inspection shows the CLI/API still send `toUserHandle`, the backend resolves only active users or personal user publishers, and org publisher handles still fall through to `User not found`.

## Solution Assessment

Unclear. The current publish `--migrate-owner` path is a maintainable answer for owners who can republish, but direct transfer-to-org support is a separate authorization-sensitive product decision that should stay on the existing transfer track.
