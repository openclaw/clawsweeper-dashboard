## Summary

Keep open: ClawHub cleanup policy keeps issues open, and current main still models skill transfers as user-to-user rather than publisher/org targets. Related #1431 and #1675 remain open, with #1635 as the open implementation candidate.

## What This Changes

The issue asks for a way to transfer existing personally owned skills to an organization so other users can publish and update them.

## Best Possible Solution

Land or replace #1635 with a publisher-targeted transfer model that supports personal-to-org, org-to-user, and org-to-org transfers while preserving existing user transfers, aliases, stats, audit logs, and canonical URLs.

## Reproduction Assessment

Yes, from source inspection. The CLI and HTTP API still send and require `toUserHandle`, while the backend resolves that handle only to active users or personal user publishers, matching the reported `User not found` path for org handles.

## Solution Assessment

Yes, consolidating this through #1635 or a maintainer-selected successor is the best path. A resolver-only patch would not cover the schema, acceptance, role, audit, and URL-preservation semantics needed for organization ownership transfers.
