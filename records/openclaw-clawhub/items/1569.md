## Summary

Keep open under the ClawHub issue policy. Current main documents and implements a publish-time owner migration path for personal-to-org moves, but the reported direct `clawhub transfer request <skill> <org>` path still targets users only, and publisher-targeted transfer work remains open.

## What This Changes

The issue asks for a way to move personally owned skills to an organization so other org users can publish and update them.

## Best Possible Solution

Keep the documented `clawhub skill publish --owner <org> --migrate-owner` path for immediate owner moves, and consolidate standalone org transfers in the publisher-targeted transfer model tracked by https://github.com/openclaw/clawhub/pull/1635 and https://github.com/openclaw/clawhub/issues/1675.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: direct transfer still sends and requires `toUserHandle`, resolves only active users or user-kind personal publishers, and stores `toUserId`, so an organization handle reaches the `User not found` path; I did not run a live authenticated transfer.

## Solution Assessment

Unclear as a final product direction. The current publish-time migration path is a maintainable partial solution, but a direct transfer-to-org feature should be implemented as a publisher-targeted transfer model rather than by only changing the recipient lookup.
