## Summary

Keep open: ClawHub cleanup policy keeps issues open, and current main still models skill transfers as user-to-user rather than publisher/org targets. Related #1431 and #1675 remain open, with #1635 as the open, unmerged implementation candidate.

## What This Changes

The issue asks for a way to transfer skills currently owned by a personal publisher to an organization publisher.

## Best Possible Solution

Land or replace #1635 with a publisher-targeted ownership transfer model that supports personal-to-org, org-to-user, and org-to-org transfers while preserving existing user transfers, aliases, stats, audit logs, and canonical URLs where possible.

## Reproduction Assessment

Yes, from source inspection. The CLI and HTTP API still send and require `toUserHandle`, while the backend resolves that handle only to active users or personal user publishers, matching the reported `User not found` path for org handles.

## Solution Assessment

Yes, a publisher-targeted transfer model is the right direction. A resolver-only patch would not cover the schema, acceptance, role authorization, audit, alias, stats, and canonical URL semantics needed for organization ownership transfers.
