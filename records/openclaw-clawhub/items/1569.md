## Summary

Keep open: ClawHub cleanup policy keeps issues open, and current main still leaves the direct `clawhub transfer` path user-targeted even though publish-time owner migration now exists.

## What This Changes

The issue asks how to move skills from a personal publisher to an organization after `bun clawhub transfer` fails with `User not found` for an org handle.

## Best Possible Solution

Keep the supported `--owner` plus `--migrate-owner` publish route, and consolidate any direct transfer-to-org work under the open publisher-targeted transfer track with explicit authorization and audit rules.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path for the direct transfer command/API: an org handle reaches a user-only resolver and falls through to `User not found`. I did not run a live transfer because this review is read-only and the source path is deterministic.

## Solution Assessment

Unclear as a final product direction. The merged publish migration is a narrow maintainable route for owners who can republish, but making `clawhub transfer` accept orgs requires a publisher-targeted transfer model rather than a small lookup change.
