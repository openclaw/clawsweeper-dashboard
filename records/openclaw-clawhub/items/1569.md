## Summary

Keep open: this ClawHub issue remains unresolved on current main, where skill transfers are still modeled as user-to-user and the org-transfer work is open/unmerged.

## What This Changes

Request support for transferring an existing skill from a personal publisher/user to an organization so other users can publish and update it.

## Best Possible Solution

Implement publisher-targeted ownership transfers that cover personal-to-org, org-to-user, and org-to-org moves while preserving existing user-to-user behavior, aliases, stats, audit logs, and install URLs where possible.

## Reproduction Assessment

Yes from source inspection, though I did not run a live account/org reproduction. The CLI and HTTP API send `toUserHandle`, and the backend resolver excludes org publishers, so an org handle reaches the `User not found` path.

## Solution Assessment

Yes, consolidating this through #1635 or a maintainer-selected successor is the best path. A narrow one-off change would not cover the schema, acceptance, role, audit, and URL-preservation semantics called out by the current org-transfer plan.
