## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still implements skill ownership transfers as user-to-user only, so personal-to-organization transfer remains unresolved; related #1431 and #1675 are open and #1635 is an open implementation candidate.

## What This Changes

The issue asks for a way to move a personally owned skill to an organization so other org members can publish and update it.

## Best Possible Solution

Land or replace the open publisher-targeted transfer implementation so transfers address publishers rather than users and cover user-to-org, org-to-user, and org-to-org without losing aliases, stats, audit history, or existing user-to-user behavior.

## Reproduction Assessment

Yes. The static path is high-confidence: `clawhub transfer request <skill> <org>` sends `toUserHandle`, the API and mutation resolve only active users or personal publishers, and org publishers are rejected by the resolver.

## Solution Assessment

Unclear until #1635 is reviewed. The maintainable direction is a publisher-target transfer model, not a one-off org branch inside the current user-to-user transfer schema.
