## Summary

Keep open: ClawHub cleanup policy keeps issues open, and current main still models skill ownership transfers as user-to-user rather than publisher/org-targeted transfers.

## What This Changes

Request a way to move an existing personally owned skill to an organization so other organization users can publish or update it.

## Best Possible Solution

Implement a publisher-targeted ownership transfer model that covers personal-to-org, org-to-user, and org-to-org moves while preserving existing user-to-user behavior and compatibility aliases.

## Reproduction Assessment

Yes, from source inspection. The CLI and HTTP API still send toUserHandle, and the backend resolver excludes organization publishers, so an org handle reaches the described 'User not found' path.

## Solution Assessment

Yes, a publisher-targeted transfer model is the maintainable solution. A resolver-only tweak would not cover schema, acceptance, role, audit, and URL-preservation semantics for organization ownership.
