## Summary

Keep open: this is a ClawHub issue, and current main still implements skill transfers as user-to-user only rather than publisher/org-targeted transfers. Related #1431 and #1675 remain open, and #1635 is still an open, unmerged implementation candidate for the broader ownership-transfer model.

## What This Changes

The issue requests a supported way to transfer an existing personally owned skill to an organization so org members can publish and update it.

## Best Possible Solution

Implement a publisher-targeted transfer model that supports personal-to-org, org-to-user, and org-to-org ownership moves while preserving existing user transfers, aliases, stats, audit logs, and canonical URLs.

## Reproduction Assessment

Yes, from source inspection. The CLI and HTTP API still send and require `toUserHandle`, while the backend resolves that handle only to active users or personal user publishers, matching the reported `User not found` path for org handles.

## Solution Assessment

Yes, a publisher-targeted ownership transfer model is the narrow maintainable direction. A resolver-only patch would not cover schema, acceptance, org role, audit, and URL-preservation semantics needed for organization ownership transfers.
