## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still routes skill transfer requests through a user-only recipient model, so personal-to-organization transfer remains unimplemented while related open work already tracks the broader fix.

## What This Changes

The issue asks for a way to transfer an existing personally owned skill to an organization so other organization members can publish and update it.

## Best Possible Solution

Implement a publisher-targeted ownership transfer model that supports personal-to-org, org-to-user, and org-to-org transfers while preserving existing user-to-user behavior, aliases, stats, audit logs, and install URLs where possible; consolidate that work through #1635 or a maintainer-selected successor.
