## Summary

Keep open: repository cleanup policy keeps ClawHub issues open, and current main still only supports user-to-user skill transfers while org/publisher transfer support remains unresolved through related open items and PR #1635.

## What This Changes

The issue asks how to transfer skills from a personal publisher to an organization publisher so other org members can publish and update them.

## Best Possible Solution

Land or replace #1635 with a publisher-targeted ownership transfer model that covers user, org, and org-to-org transfers while preserving aliases, stats, audit logs, and canonical install URLs.

## Reproduction Assessment

Yes. Source inspection shows the CLI and HTTP API send and require `toUserHandle`, while the backend resolves that handle to active users and excludes organization publishers, matching the reported `User not found` path.

## Solution Assessment

Yes. A publisher-targeted transfer model is the maintainable fix direction; changing only the user resolver would leave role authorization, acceptance, audit, alias, and URL-preservation semantics incomplete.
