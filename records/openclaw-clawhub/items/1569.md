## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main only supports moving a personal skill to an org through publish-time owner migration; the direct `clawhub transfer` path still resolves recipients as users, while broader org-transfer work remains open at https://github.com/openclaw/clawhub/pull/1635.

## What This Changes

Requests a way to transfer personal skills to an organization so other org members can publish and update them after `bun clawhub transfer` returns `User not found`.

## Best Possible Solution

Keep `clawhub skill publish --owner <org> --migrate-owner` as the immediate supported move path, then land or extract a publisher-targeted transfer model from https://github.com/openclaw/clawhub/pull/1635 or a maintainer-selected successor if standalone org transfers remain required.

## Reproduction Assessment

Yes, by source inspection rather than a live authenticated run. The CLI and HTTP handler still send `toUserHandle`, the backend resolves only active users or user-kind personal publishers, and an org publisher handle reaches the `User not found` branch.

## Solution Assessment

Unclear as a final product direction. The current publish-time migration path is a maintainable partial solution, but direct transfer-to-org should be solved as a publisher-targeted transfer model rather than by only changing recipient lookup.
