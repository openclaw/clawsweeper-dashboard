## Summary

Keep open: ClawHub cleanup policy keeps issues open, and current main still implements skill ownership transfers as user-to-user rather than organization/publisher-targeted transfers. Related #1431 and #1675 remain open, with #1635 as the open implementation candidate for the broader org-transfer model.

## What This Changes

The issue asks for a way to transfer a personally owned skill to an organization so other organization users can publish and update it.

## Best Possible Solution

Implement a publisher-targeted ownership transfer model that supports personal-to-org, org-to-user, and org-to-org transfers while preserving existing user-to-user behavior, aliases, stats, audit logs, and install URLs where possible.

## Reproduction Assessment

Yes. Source inspection shows the CLI and HTTP API still require `toUserHandle`, while the backend resolves that handle only to active users or personal user publishers, which matches the reported `User not found` path for org handles.

## Solution Assessment

Yes. A publisher-targeted transfer model, as proposed by #1635 or a maintainer-selected successor, is the maintainable path; a resolver-only patch would not cover org acceptance, approval, schema, audit, alias, and URL semantics.
