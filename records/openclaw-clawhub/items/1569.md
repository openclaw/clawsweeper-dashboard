## Summary

Keep open: ClawHub cleanup policy keeps issues open, and current main still models skill transfers as user-to-user rather than publisher/org targets. Related #1431 and #1675 remain open, with #1635 as the open implementation candidate.

## What This Changes

The request asks for a way to transfer skills from a personal publisher to an organization so other org members can publish and update them.

## Best Possible Solution

Land or replace #1635 with a publisher-targeted transfer model that covers user/org directions while preserving existing user transfers, aliases, stats, audit logs, and canonical install URLs.

## Reproduction Assessment

Yes, from source inspection. The CLI and HTTP API still send and require `toUserHandle`, while the backend resolves that handle to active users and excludes organization publishers, matching the reported `User not found` path.

## Solution Assessment

Yes, consolidating this through #1635 or a maintainer-selected successor is the best path. A resolver-only patch would not cover the schema, acceptance, role, audit, and URL-preservation semantics needed for org ownership transfers.
