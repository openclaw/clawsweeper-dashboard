## Summary

Keep open: current main supports a related publish-time owner migration path, but the reported direct `clawhub transfer` command/API path still only targets users, and ClawHub cleanup policy keeps issues open for maintainer triage.

## What This Changes

The issue asks for a way to move personally owned skills to organization ownership so organization members can publish and update them after the transfer command returns `User not found`.

## Best Possible Solution

Use the documented `--owner` plus `--migrate-owner` publish path for immediate personal-to-org moves, then consolidate any direct publisher-targeted transfer design under https://github.com/openclaw/clawhub/issues/1675 or a maintainer-selected successor to https://github.com/openclaw/clawhub/pull/1635.

## Reproduction Assessment

Yes, at source level. Current main still has the CLI/API send `toUserHandle`, the backend resolves only users or user-kind personal publishers, and the schema stores `toUserId`, so an organization publisher handle cannot be a direct transfer recipient.

## Solution Assessment

Unclear for the direct transfer command. The publish-time `ownerHandle` plus `migrateOwner` route is a maintainable supported workaround, but a true transfer-to-org command/API should be a publisher-targeted transfer model rather than a small resolver change.
