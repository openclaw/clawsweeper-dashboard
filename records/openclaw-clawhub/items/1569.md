## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still implements skill ownership transfers as user-to-user only; the requested personal-to-organization transfer remains unresolved and overlaps with open org-transfer work.

## What This Changes

The issue asks for a way to transfer an existing skill from a personal publisher to an organization so other organization members can publish and update it.

## Best Possible Solution

Implement publisher-targeted skill ownership transfers for personal-to-org, org-to-user, and org-to-org moves while preserving existing user-to-user behavior, aliases, stats, audit logs, and install URLs; consolidate the work through #1635 or a maintainer-selected successor.

## Reproduction Assessment

Yes. The current code path reproduces the reported failure with high confidence: an org handle is sent as `toUserHandle`, the backend resolver only returns users or personal publishers, and an org publisher therefore falls through to `User not found`.

## Solution Assessment

Unclear for this issue because it contains no patch. The maintainable direction is a publisher-targeted transfer model; adding a one-off org lookup to the current user-to-user schema would leave the core ownership model inconsistent.
