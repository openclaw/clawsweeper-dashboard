## Summary

Keep open: repository policy keeps ClawHub issues open in this cleanup pass, and current main still models skill transfers as user-to-user rather than organization-publisher targets.

## What This Changes

The issue asks for a way to transfer skills from a personal publisher to an organization so other org users can publish and update them.

## Best Possible Solution

Implement publisher-targeted ownership transfers that cover personal-to-org, org-to-user, and org-to-org moves while preserving existing user-to-user behavior, aliases, stats, audit logs, and install URLs where possible.

## Reproduction Assessment

Yes, from source inspection. The CLI and HTTP API still send `toUserHandle`, and the backend resolver excludes organization publishers, so an org handle reaches the `User not found` path described in the issue.

## Solution Assessment

Yes, consolidating this through #1635 or a maintainer-selected successor is the best path. A narrow resolver-only change would not cover the schema, acceptance, role, audit, and URL-preservation semantics needed for org ownership transfers.
