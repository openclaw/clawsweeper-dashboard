## Summary

Keep open: this is a ClawHub issue, and the cleanup policy only auto-closes implemented or mostly implemented pull requests. Current main supports publish-time owner migration, but the reported direct `clawhub transfer` path is still user-to-user and the org-targeted transfer work remains open in https://github.com/openclaw/clawhub/issues/1675 and https://github.com/openclaw/clawhub/pull/1635.

## What This Changes

The issue asks how to move existing personal skills into organization ownership after `bun clawhub transfer` returns `User not found`.

## Best Possible Solution

Keep the explicit publish owner-migration route as the immediate supported path, and consolidate any direct transfer command/API work under a publisher-targeted transfer design in https://github.com/openclaw/clawhub/issues/1675 or a maintainer-selected successor.

## Reproduction Assessment

Yes, for the direct transfer path from source inspection. The CLI/API still send `toUserHandle`, the backend resolves only active users or personal user publishers, and org publisher handles fall through to `User not found`.

## Solution Assessment

Unclear. The merged publish `--owner` plus `--migrate-owner` path is a maintainable migration route, but direct transfer-to-org support should be a publisher-targeted transfer model rather than a small handle-resolver change.
