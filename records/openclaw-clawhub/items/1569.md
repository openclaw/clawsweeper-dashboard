## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main only supports the personal-to-org path through publish-time owner migration; the direct `clawhub transfer` API/CLI path still resolves recipients as users, while related org-transfer work remains open at https://github.com/openclaw/clawhub/pull/1635.

## What This Changes

The issue asks for a way to transfer personally owned skills to an organization so other org members can publish and update them.

## Best Possible Solution

Keep the documented `clawhub skill publish --owner <org> --migrate-owner` path for immediate owner moves, and land or extract a publisher-targeted transfer flow from the open ownership-transfer work if a standalone `clawhub transfer` org path remains required.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: direct transfer still sends and requires `toUserHandle`, resolves only active users or user-kind personal publishers, and stores `toUserId`, so an organization publisher handle reaches the `User not found` path; I did not run a live authenticated transfer.

## Solution Assessment

Unclear as a final product direction. The current publish-time migration path is a maintainable partial solution, but direct transfer-to-org should be solved as a publisher-targeted transfer model rather than by only changing the recipient lookup.
