## Summary

Keep open under the ClawHub cleanup policy for issues. Current main documents and supports publish-time owner migration, but the direct `clawhub transfer` / transfer API path is still user-targeted and the broader publisher/org transfer work remains open at https://github.com/openclaw/clawhub/pull/1635.

## What This Changes

The issue asks for a way to move existing skills from a personal publisher to an organization publisher after `clawhub transfer` fails for an org handle.

## Best Possible Solution

Keep the explicit publish `--owner` plus `--migrate-owner` route as the supported immediate path, and land or extract a publisher-targeted transfer flow if direct org-transfer commands remain required.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: direct transfer still accepts `toUserHandle`, resolves only users or user-kind personal publishers, and stores `toUserId`, so an organization publisher handle is not a valid destination; I did not run a live authenticated transfer.

## Solution Assessment

Unclear as a final product direction. The current publish-time owner migration is a narrow maintainable path for republishing into an org, while direct transfer-to-org needs a publisher-targeted transfer design rather than a small lookup change.
