## Summary

Keep open: ClawHub cleanup policy keeps issues open, and current main still models skill transfers as user-to-user rather than organization/publisher-targeted transfers. Related #1431 and #1675 remain open, with #1635 as the open implementation candidate.

## What This Changes

The issue asks for a way to transfer a skill from a personal publisher to an organization publisher after `bun clawhub transfer` reports `User not found`.

## Best Possible Solution

Implement publisher-targeted ownership transfers for personal-to-org, org-to-user, and org-to-org moves while preserving existing user-to-user behavior, aliases, stats, audit logs, and install URLs where possible.

## Reproduction Assessment

Yes. From source inspection, `clawhub transfer request <slug> <org-handle>` still sends `toUserHandle`, and the backend resolver excludes org publishers before reaching the `User not found` path described here.

## Solution Assessment

Yes. Consolidating this through #1635 or a maintainer-selected successor is the right direction; a resolver-only tweak would not cover schema, recipient acceptance, org roles, audit metadata, and URL-preservation semantics.
