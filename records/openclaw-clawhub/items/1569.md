## Summary

Keep open: current main still implements skill ownership transfers as user-targeted transfers, not publisher/org-targeted transfers. Related reports remain open, and https://github.com/openclaw/clawhub/pull/1635 is the open unmerged implementation candidate for the broader org-aware transfer model.

## What This Changes

The issue asks for a way to move an existing personal-owned skill to an organization publisher so organization members can publish and update it.

## Best Possible Solution

Implement a publisher-targeted ownership transfer model that supports personal-to-org, org-to-user, and org-to-org transfers while preserving existing user transfers, aliases, stats, audit logs, and install URLs; https://github.com/openclaw/clawhub/pull/1635 or a maintainer-selected successor is the right consolidation point.

## Reproduction Assessment

Yes, from source inspection. The CLI and HTTP API still send `toUserHandle`, while `requestTransferInternal` resolves that handle only to active users or personal user publishers, matching the reported `User not found` path for org handles.

## Solution Assessment

Yes, the publisher-targeted transfer model sketched in `docs/orgs.md` is the maintainable direction. A resolver-only patch would not cover schema, acceptance, listing, audit, org-role, and URL-preservation semantics.
