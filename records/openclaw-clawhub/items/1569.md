## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still only implements user-to-user skill transfers; #1635 is open and unmerged, so the requested personal-to-organization transfer path is not implemented.

## What This Changes

Allow a user to move an existing personal skill to an organization so org members can publish or update it instead of the transfer command failing with `User not found`.

## Best Possible Solution

Land a publisher-targeted ownership-transfer design through #1635 or a maintainer-selected successor that covers personal-to-org, org-to-user, and org-to-org transfers while preserving existing aliases, stats, audit logs, and install URLs.

## Reproduction Assessment

Yes from source inspection: the CLI and HTTP handler require `toUserHandle`, and the backend resolver only accepts users or personal user publishers. An organization publisher handle therefore reaches the current `User not found` path.

## Solution Assessment

Yes at the product direction level: replacing user-only recipients with publisher targets is the narrow maintainable fix. The exact org role and approval behavior should be decided in #1635 or a successor before merging.
