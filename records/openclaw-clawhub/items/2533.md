## Summary

Keep this PR open for maintainer review: it addresses the linked deleted-account recovery problem and has useful UI proof, but the current hard-delete path has merge-blocking data-retention and scalability risks in security-sensitive account/package cleanup.

## What This Changes

The PR hard-deletes account, personal publisher, org publisher, skill, plugin/package, auth, token, and related rows during account/org deletion, adds a recovery purge backfill, and updates settings deletion confirmations with resource rows and tests.

## Best Possible Solution

Land a paginated hard-delete design that preserves or tombstones security/moderation evidence according to an explicit maintainer retention decision, then update the security-moderation spec and cover both small and high-volume packages.

## Reproduction Assessment

Yes. Source inspection shows the PR deletes package moderation rows during hard deletion and collects all package child rows in one mutation; a package with enough reports, releases, tokens, or stat events would exercise the failure path.

## Solution Assessment

No, not as-is. The direction is useful for deleted-account recovery, but the maintainable solution needs paginated cleanup plus an explicit moderation/audit retention model before irreversible deletion ships.

Full review comments:

- **[P1] Preserve package moderation evidence during owner purge:** `convex/packages.ts:2981-2991`
  - body: `hardDeletePackageDoc` deletes package reports, appeals, and their moderation event logs whenever an account or publisher hard-deletes a package. That lets a self-service deletion remove the report/appeal timeline for potentially unsafe packages, while `specs/security-moderation.md` treats package reports and `packageModerationEventLogs` as moderation/audit surfaces. Keep these rows as redacted/tombstoned evidence, or get an explicit retention-policy change before wiring them into owner deletion.
  - confidence: 0.87
- **[P1] Batch package hard-delete child cleanup:** `convex/packages.ts:3019-3023`
  - body: This mutation collects and deletes every package child row in one transaction, including releases, scan jobs, reports, upload tickets, and stat events. A package with many downloads/stat events or releases can exceed Convex transaction limits and leave account deletion unable to complete. Mirror the skill hard-delete phase model or otherwise page each child table before deleting the package row.
  - confidence: 0.84
