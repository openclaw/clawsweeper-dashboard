## Summary

Keep open. The report is still valid for the legacy `clawdhub@latest` npm path: current source and the maintained `clawhub` package declare `undici`, but live registry metadata still resolves `clawdhub@latest` to `0.3.0` without that dependency, and ClawHub issue policy keeps issues open for maintainer handling.

## What This Changes

Report that `pnpm dlx`/`npx` installs of `clawdhub@latest` fail because the published legacy package imports `undici` without declaring it as a dependency.

## Best Possible Solution

A maintainer should either publish or promote a fixed stable legacy `clawdhub` release whose metadata includes `undici`, or deprecate/redirect the legacy package to `clawhub` with clear migration guidance, then close the duplicate reports after a clean install smoke test.
