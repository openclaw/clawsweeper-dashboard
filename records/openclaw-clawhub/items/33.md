## Summary

Keep open. The maintained `clawhub` package and current source declare `undici`, but the reported legacy `clawdhub@latest` npm path still resolves to `0.3.0` without that dependency, so the user-visible failure remains unresolved and needs maintainer npm/package handling.

## What This Changes

The issue reports that `pnpm dlx clawdhub@latest install sonoscli` fails because the published package imports `undici` without declaring it as a dependency.

## Best Possible Solution

End with one stable user path: publish or promote a fixed stable `clawdhub` release with `undici`, or explicitly deprecate/redirect the legacy package to `clawhub`, then consolidate the duplicate reports.

## Reproduction Assessment

Yes. I did not run `dlx` or `npx` because that would create package-manager cache artifacts, but the source-level path is high-confidence: `clawdhub@latest` resolves to `0.3.0` without `undici`, and the runtime imports `undici`.

## Solution Assessment

No. Adding `undici` to current source alone is no longer sufficient because the maintained `clawhub` package already includes it; the remaining fix is legacy npm package/dist-tag handling or an explicit deprecation path.
