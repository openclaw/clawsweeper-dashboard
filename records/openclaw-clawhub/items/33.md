## Summary

Keep open under the ClawHub issue policy. Current `main` and the maintained `clawhub` npm package declare `undici`, but live legacy `clawdhub@latest` still resolves to `0.3.0` without that dependency while its runtime imports it, so the reported install path remains user-visible and needs maintainer npm registry handling.

## What This Changes

The issue reports that `pnpm dlx clawdhub@latest install sonoscli` fails because the published legacy `clawdhub` runtime imports `undici` without declaring it as a dependency.

## Best Possible Solution

Resolve the legacy npm package state by publishing or promoting a fixed stable `clawdhub` release, or by deprecating/redirecting `clawdhub@0.3.0` with clear migration guidance to `clawhub`, then smoke test a clean Node install and consolidate the duplicate reports.

## Reproduction Assessment

Yes. Live npm metadata and tarball inspection show `clawdhub@latest` resolves to `0.3.0`, whose package metadata omits `undici` while `dist/http.js` imports it; I did not run a clean install because this was a read-only review.

## Solution Assessment

No. Adding `undici` to current source is already done for the maintained `clawhub` package; the remaining maintainable fix is npm dist-tag, publish, deprecation, or migration handling for the legacy package.
