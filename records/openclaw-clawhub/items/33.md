## Summary

Keep open. Current `main` and the maintained `clawhub` npm package include `undici`, but the exact reported `clawdhub@latest` path still resolves to `0.3.0` without that dependency, so the user-visible legacy install failure remains unresolved.

## What This Changes

The issue asks for `undici` to be declared as a runtime dependency so `pnpm dlx clawdhub@latest` and `npx clawdhub@latest` do not fail with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

End with one stable user path: publish or promote a fixed stable `clawdhub` package whose dependency metadata includes `undici`, or intentionally deprecate/redirect the legacy package to `clawhub` with clear guidance.

## Reproduction Assessment

Yes. Live registry metadata shows `clawdhub@latest` still points at `0.3.0` without `undici`, and the CLI runtime imports `undici`; I did not run `dlx`/`npx` because that would create package-manager cache artifacts.

## Solution Assessment

No. Adding `undici` to the current source package alone is no longer sufficient because `packages/clawhub` and `clawhub@latest` already include it; the remaining solution is legacy npm package/dist-tag handling or an explicit deprecation path.
