## Summary

Keep open. Current source and the maintained `clawhub` npm package include `undici`, but live `clawdhub@latest` still resolves to `0.3.0`, whose metadata omits `undici` while its published runtime imports it; under the ClawHub issue policy this needs maintainer handling rather than auto-close.

## What This Changes

The issue reports that `pnpm dlx clawdhub@latest install sonoscli` installs `clawdhub@0.3.0`, whose runtime imports `undici` without declaring it as a dependency.

## Best Possible Solution

Make the legacy npm path unambiguous by publishing or promoting a fixed stable `clawdhub`, or by deprecating and redirecting `clawdhub@0.3.0` to `clawhub`, then smoke test a clean Node package-runner install and consolidate duplicates.
