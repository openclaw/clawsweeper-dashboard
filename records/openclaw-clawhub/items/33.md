## Summary

Keep open. Current main and the maintained `clawhub` package include `undici`, but the exact reported `clawdhub@latest` path still resolves to `clawdhub@0.3.0` without that dependency, so the user-visible failure remains unresolved and needs maintainer npm/package handling.

## What This Changes

The issue requests adding `undici` as a runtime dependency so `pnpm dlx` or `npx` installs of the legacy `clawdhub` CLI do not fail with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

End with one stable user path: either publish or promote a fixed stable `clawdhub` release whose dependency metadata includes `undici`, or intentionally deprecate and redirect the legacy package with clear guidance to use `clawhub`, then consolidate the duplicate reports.
