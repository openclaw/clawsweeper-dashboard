## Summary

Keep open. Current main and the maintained `clawhub` package include `undici`, but the reported legacy `clawdhub@latest` path still resolves to `0.3.0` without that dependency; under the ClawHub cleanup policy, issues stay open and this needs maintainer npm package or deprecation handling.

## What This Changes

The issue asks for the published `clawdhub` CLI package to declare `undici` so clean `pnpm dlx` or `npx` installs do not crash with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

End with one stable user path: publish or promote a fixed stable `clawdhub` release whose metadata includes `undici`, or intentionally deprecate/redirect the legacy package to `clawhub` and consolidate the duplicate reports.

## Reproduction Assessment

Yes. Live npm metadata shows the reported `clawdhub@latest` path still selects `0.3.0` without `undici`, and the CLI runtime imports `undici`; I did not need to mutate the checkout or run an installing smoke test to establish the dependency gap.

## Solution Assessment

No. Adding `undici` to current source is already done for the maintained `clawhub` package; the narrow remaining solution is npm dist-tag/publishing/deprecation action for the legacy package.
