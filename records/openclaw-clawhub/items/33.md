## Summary

Keep open. The maintained `clawhub` package and current source include `undici`, but the exact reported legacy `clawdhub@latest` npm path still resolves to `0.3.0` without that dependency, so the user-visible failure remains unresolved and needs maintainer npm/release handling.

## What This Changes

The issue requests that the legacy `clawdhub` CLI package include `undici` so `pnpm dlx` and `npx` installs do not crash at runtime.

## Best Possible Solution

End with one stable user path: publish or promote a fixed stable `clawdhub` release whose dependency metadata includes `undici`, or intentionally deprecate/redirect the legacy package with clear guidance to use `clawhub`, then consolidate the duplicate reports.

## Reproduction Assessment

Yes. Live registry metadata and tarball inspection show `clawdhub@latest` still resolves to `0.3.0`, whose runtime imports `undici` while its package metadata omits it.

## Solution Assessment

No. Adding `undici` to current source is already done; the narrow maintainable solution is npm dist-tag, publish, deprecation, or migration handling for the legacy `clawdhub` package.
