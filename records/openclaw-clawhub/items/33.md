## Summary

Keep open. The current source and maintained `clawhub` package declare `undici`, but the exact reported legacy `clawdhub@latest` npm path still resolves to `0.3.0` without that dependency, so the user-visible failure remains unresolved and needs maintainer package/distribution handling.

## What This Changes

The issue asks for the published `clawdhub` CLI package to include `undici` so fresh npm, npx, and pnpm dlx installs do not fail with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

Resolve the legacy package distribution path by publishing or promoting a fixed stable `clawdhub`, or deprecating/redirecting it to `clawhub`, then smoke test and consolidate the duplicate reports.

## Reproduction Assessment

Yes. Registry metadata and tarball inspection show `clawdhub@latest` still resolves to `0.3.0` without `undici` while its published build imports `undici`; this is not a current-main source failure because `packages/clawhub/package.json` already declares the dependency.

## Solution Assessment

No for the suggested source-only dependency edit. Current main already has the dependency, so the narrow remaining solution is maintainer-controlled npm registry, release, dist-tag, deprecation, or migration handling for the legacy package.
