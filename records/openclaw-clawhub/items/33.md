## What This Changes

The issue requests that the published `clawdhub` CLI package include `undici` as a runtime dependency so `pnpm dlx` and `npx` installs do not crash with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

End with one stable CLI install path: publish or promote a fixed stable legacy `clawdhub` package that includes `undici`, or intentionally deprecate/redirect that package to the maintained `clawhub` CLI, then consolidate #23 and #51.

## Reproduction Assessment

Yes. I did not run `pnpm dlx` because that would install packages, but live registry and tarball inspection reproduce the failure precondition: `clawdhub@latest` is 0.3.0, its package metadata omits `undici`, and its bundled HTTP module imports `undici`.

## Solution Assessment

No for a source-only patch. Current `main` already includes `undici` for the maintained `clawhub` package, so the narrow remaining fix is npm package/dist-tag administration or an intentional legacy-package redirect/deprecation decision.
