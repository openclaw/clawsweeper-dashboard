## Summary

Keep open. The maintained `clawhub` source/package has `undici`, but live `clawdhub@latest` still resolves to `0.3.0` without that dependency, so the reported install path remains broken and needs maintainer npm registry handling.

## What This Changes

The issue asks for `undici` to be included with the legacy `clawdhub` CLI package so `pnpm dlx` and `npx` installs do not crash at runtime.

## Best Possible Solution

Make one supported install path clear: promote or publish a stable legacy `clawdhub` package with `undici`, or deprecate and redirect legacy users to the maintained `clawhub` package.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes: live npm metadata still maps `clawdhub@latest` to `0.3.0` and that version's dependency map lacks `undici`, while current CLI code imports `undici` at startup.

## Solution Assessment

Is this the best way to solve the issue? No: adding `undici` to current source is already done for `clawhub`; the best remaining fix is npm registry action or explicit legacy-package deprecation/redirect guidance.
