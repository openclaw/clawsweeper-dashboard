## Summary

Keep open. Current source and the maintained `clawhub` npm package include `undici`, but live `clawdhub@latest` still resolves to `0.3.0` without that dependency, so the reported legacy install path remains broken and needs maintainer npm/package handling.

## What This Changes

The issue asks for `undici` to be included as a runtime dependency so `pnpm dlx` or `npx clawdhub@latest` does not fail with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

End with one stable user path: publish or promote a fixed stable `clawdhub` release whose metadata includes `undici`, or intentionally deprecate and redirect the legacy package to `clawhub`.

## Reproduction Assessment

Yes. Registry metadata still proves that `clawdhub@latest` installs `0.3.0` without `undici`, while the CLI HTTP module imports `undici` at startup.

## Solution Assessment

No, the original source-only dependency addition is no longer the best remaining fix because current `clawhub` source already includes it. The narrow remaining solution is npm registry/package action or explicit legacy-package deprecation.
