## Summary

Keep open. Current `main` and the maintained `clawhub` npm package include `undici`, but the exact reported `clawdhub@latest` install path still resolves to `0.3.0` without that dependency, so the user-visible legacy failure remains unresolved and needs maintainer npm handling.

## What This Changes

The issue reports that `pnpm dlx clawdhub@latest install sonoscli` fails because the published `clawdhub` package imports `undici` without declaring it as a dependency.

## Best Possible Solution

End with one stable user path: either publish or promote a fixed stable `clawdhub` release whose dependency metadata includes `undici`, or deprecate/redirect the legacy package with clear guidance to use `clawhub`, then consolidate the duplicate reports.

## Reproduction Assessment

Yes. Fetching live registry metadata shows `clawdhub@latest` is still `0.3.0` without `undici`, while the current CLI HTTP client imports `undici` at startup.

## Solution Assessment

No. Adding `undici` to current source is already done for the maintained `clawhub` package; the narrow remaining solution is npm registry action or an intentional legacy-package deprecation/redirect.
