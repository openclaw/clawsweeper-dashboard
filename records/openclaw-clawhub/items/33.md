## Summary

Keep open under the ClawHub issue policy and because the user-visible legacy install path is still broken: current source and the maintained `clawhub` package include `undici`, but live npm metadata for `clawdhub@latest` still points at a version whose dependency metadata omits it.

## What This Changes

The issue asks for the legacy `clawdhub` npm package to include `undici` so `pnpm dlx` and `npx` installs do not fail with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

End with one stable user path: publish or promote a fixed stable `clawdhub` release whose dependency metadata includes `undici`, or intentionally deprecate/redirect the legacy package to the maintained `clawhub` package.

## Reproduction Assessment

Yes. Live npm registry metadata still shows `clawdhub@latest` resolving to `0.3.0`, whose dependencies omit `undici`, while the CLI HTTP client imports `undici`; that matches the reported clean-install module-resolution failure.

## Solution Assessment

No, not as a repo-only source change. Current source already includes `undici` for the maintained package, so the remaining durable fix is npm package publishing, dist-tag, or deprecation handling for the legacy `clawdhub` package.
