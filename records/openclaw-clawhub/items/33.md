## Summary

Keep open. Current source and the maintained `clawhub` npm package include `undici`, but the exact reported `clawdhub@latest` path still resolves to `0.3.0` without that dependency; for ClawHub issues this should remain open for maintainer npm/package handling.

## What This Changes

The issue asks for the legacy `clawdhub` npm package to declare `undici` so clean `pnpm dlx` and `npx` installs do not crash at startup.

## Best Possible Solution

End with one stable user path: publish or promote a fixed stable `clawdhub` release with `undici`, or intentionally deprecate/redirect the legacy package to `clawhub` with clear user guidance.

## Reproduction Assessment

Yes. Registry/source inspection is enough: `clawdhub@latest` is still `0.3.0` without `undici`, while the CLI runtime imports `undici`; I did not run `dlx`/`npx` because that would write package-manager cache artifacts.

## Solution Assessment

No, a repo-only dependency patch is not the remaining fix because current source already includes `undici`. The narrow maintainable resolution is npm dist-tag/publish ownership or an explicit legacy-package deprecation path.
