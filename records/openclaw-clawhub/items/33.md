## Summary

Keep open. Current main and the maintained `clawhub` package declare `undici`, but live `clawdhub@latest` still resolves to `0.3.0` without that dependency while its published runtime imports it, so the reported legacy install path remains broken and needs maintainer npm/package handling.

## What This Changes

The issue asks for `undici` to be declared as a runtime dependency so clean `pnpm dlx` or `npx clawdhub@latest` installs do not fail with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

End with one stable user path: publish or promote a fixed stable `clawdhub` release whose dependency metadata includes `undici`, or intentionally deprecate/redirect the legacy package with clear guidance to use `clawhub`, then smoke test the clean install path.

## Reproduction Assessment

Yes. Live npm metadata plus tarball inspection show `clawdhub@latest` still resolves to `0.3.0`, whose metadata omits `undici` while its runtime imports it; I did not run a clean install because this review is read-only.

## Solution Assessment

No. Adding `undici` to current source is already done for the maintained `clawhub` package; the remaining maintainable fix is npm publish, dist-tag, deprecation, or migration handling for the legacy package.
