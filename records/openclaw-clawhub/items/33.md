## Summary

Keep open. Current source and the maintained `clawhub` npm package include `undici`, but the exact reported `clawdhub@latest` path still resolves to legacy `0.3.0` without that dependency, so the user-visible failure remains unresolved and needs maintainer package handling.

## What This Changes

The issue reports that `pnpm dlx clawdhub@latest install sonoscli` fails because the published `clawdhub` package imports `undici` without listing it as a runtime dependency.

## Best Possible Solution

End with one stable user path: publish or promote a fixed stable `clawdhub` release whose dependency metadata includes `undici`, or intentionally deprecate and redirect the legacy package to `clawhub`, then consolidate the duplicate reports.

## Reproduction Assessment

Yes, at source and registry level. Live npm metadata for `clawdhub@latest` omits `undici`, while the runtime HTTP module imports it directly.

## Solution Assessment

No, not as a source-only `package.json` edit. Current source already includes `undici` for `clawhub`; the remaining fix is a maintainer npm dist-tag, legacy publish, or deprecation decision for `clawdhub`.
