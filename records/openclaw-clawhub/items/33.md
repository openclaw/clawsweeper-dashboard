## Summary

Keep open. Current source and `clawhub@latest` include `undici`, but the reported `clawdhub@latest` path still resolves to `0.3.0` without that dependency, so the user-visible legacy install failure remains unresolved and needs maintainer npm/release handling.

## What This Changes

The issue asks for the legacy `clawdhub` npm package to include `undici` so `pnpm dlx`/`npx` installs do not crash with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

End with one stable user path: publish or promote a fixed stable `clawdhub` release whose dependency metadata includes `undici`, or deprecate/redirect the legacy package with clear guidance to use `clawhub`, then consolidate the duplicate reports.

## Solution Assessment
