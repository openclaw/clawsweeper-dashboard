## Summary

Keep open under the ClawHub issue policy. Current source and the maintained `clawhub` package already include `undici`, but the exact reported legacy `clawdhub@latest` path still resolves to a broken npm package state, so this needs maintainer npm registry handling rather than source-only cleanup.

## What This Changes

The issue reports that `pnpm dlx`/`npx clawdhub@latest` installs `clawdhub@0.3.0`, whose runtime imports `undici` without declaring it as a dependency.

## Best Possible Solution

End with one stable user path: publish or promote a fixed stable `clawdhub` release whose dependency metadata includes `undici`, or clearly deprecate/redirect the legacy package to `clawhub`, then consolidate the duplicate reports.
