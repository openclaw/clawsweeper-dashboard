## Summary

Keep open. The maintained `clawhub` package and current source include `undici`, but the exact reported legacy `clawdhub@latest` path still resolves on npm to `0.3.0` without that dependency, so the user-visible failure remains for clean `npx`/`pnpm dlx` legacy installs.

## What This Changes

The issue asks for the legacy `clawdhub` npm package to include `undici` so `pnpm dlx clawdhub@latest install sonoscli` does not fail with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

End with one stable user path: either promote or publish a fixed stable `clawdhub` release whose metadata includes `undici`, or intentionally deprecate/redirect the legacy package to `clawhub` with clear guidance.
