## Summary

Keep open. Current main and the maintained `clawhub` npm package include `undici`, but the exact reported `clawdhub@latest` path still resolves to `0.3.0` without that dependency, so the user-visible legacy install failure remains for maintainer handling.

## What This Changes

The issue asks for the published `clawdhub` CLI package to include `undici` so `pnpm dlx` and `npx` installs do not crash at runtime.

## Best Possible Solution

End with one stable user path: publish or promote a fixed stable `clawdhub` release with `undici`, or clearly deprecate/redirect the legacy package to `clawhub`.

## Solution Assessment
