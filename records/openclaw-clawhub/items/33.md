## Summary

Keep open. The maintained `clawhub` source and npm package include `undici`, but the exact reported legacy `clawdhub@latest` path still resolves to `0.3.0` without that dependency, so the user-visible failure remains for legacy installs and needs maintainer npm/package handling.

## What This Changes

Add `undici` to the published CLI dependency metadata so `pnpm dlx` or `npx clawdhub@latest` can run without `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

End with one stable user path: publish or promote a fixed stable `clawdhub` release whose dependency metadata includes `undici`, or explicitly deprecate/redirect the legacy package to `clawhub` with clear guidance.

## Solution Assessment

No. The suggested source-only `package.json` addition is already present for the maintained `clawhub` package; the remaining fix is npm dist-tag/publish/deprecation handling for the legacy `clawdhub@latest` path.
