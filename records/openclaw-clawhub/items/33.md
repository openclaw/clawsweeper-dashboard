## Summary

Keep open. Current source and the maintained `clawhub` npm package include `undici`, but the exact reported `clawdhub@latest` path still resolves to a stable package without that dependency, so the remaining user-visible problem is npm package/dist-tag handling rather than a normal source-only fix.

## What This Changes

The issue requests that the missing `undici` runtime dependency for `clawdhub@latest` be included so `pnpm dlx` and `npx` installs do not crash with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

End with one stable user path: publish or promote a fixed stable `clawdhub` package whose metadata includes `undici`, or intentionally deprecate/redirect the legacy package to `clawhub` with clear user guidance.

## Reproduction Assessment

Yes. Without running an installing command, the live `clawdhub@latest` metadata and tarball show that clean `pnpm dlx` or `npx` installs receive `0.3.0`, whose dist imports `undici` while package metadata does not install it.

## Solution Assessment

No. Adding `undici` to current source alone is not the best remaining fix because current `clawhub` source and npm metadata already include it; the unresolved path needs npm registry action or an explicit legacy-package deprecation decision.
