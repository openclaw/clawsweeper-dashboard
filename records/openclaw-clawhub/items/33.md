## Summary

Keep open. Current source and the maintained `clawhub` npm package include `undici`, but the exact reported `clawdhub@latest` path still resolves to `clawdhub@0.3.0` without that dependency, so the user-visible legacy install failure remains unresolved and needs maintainer npm/package handling.

## What This Changes

The issue requests that the package installed by `clawdhub@latest` include the missing `undici` runtime dependency so `npx` and `pnpm dlx` commands do not fail with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

End with one stable user path: either publish or promote a fixed stable `clawdhub` release whose dependency metadata includes `undici`, or intentionally deprecate/redirect the legacy package with clear guidance to use `clawhub`.

## Reproduction Assessment

Yes. The live registry metadata is enough for a high-confidence reproduction path: `clawdhub@latest` resolves to `0.3.0`, that package lacks `undici`, and the CLI HTTP client imports `undici` at runtime.

## Solution Assessment

No. Adding `undici` to current source alone is already done for `clawhub`; the remaining maintainable solution is registry-level action for the legacy `clawdhub` package or an explicit deprecation/redirect decision.
