## Summary

Keep open. Current main covers the narrow source-level fix by using GitHub's immutable numeric provider ID and syncing mutable profile fields, but this is an open ClawHub issue and the supplied cleanup policy only permits auto-closing PRs; the related open #1109 also leaves production/account-state auth triage unresolved.

## What This Changes

Allow sign-in after a GitHub username change by keying account lookup on the stable GitHub user ID and treating the username as mutable profile data.

## Best Possible Solution

Production should consistently authenticate by numeric providerAccountId, refresh mutable GitHub profile fields for active accounts, and provide an identity-verified recovery or consolidation path for stale, duplicate, deleted/deactivated, or verifier/session state.

## Reproduction Assessment

Unclear. The issue gives plausible steps and #1109 adds a concrete token-mismatch clue, but this read-only source review cannot complete production GitHub OAuth with a renamed account or inspect the affected Convex rows.

## Solution Assessment

Yes for the code-level direction, but not complete for the live symptom. Stable providerAccountId lookup plus mutable profile sync is the maintainable source path; any remaining account repair should be maintainer-owned and identity-verified.
