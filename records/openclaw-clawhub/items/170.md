## Summary

Keep open. Current main has the stable GitHub numeric-ID and profile-sync foundation, but ClawHub cleanup policy keeps issues open and the related open username-change report still requires production OAuth/account-state verification.

## What This Changes

The issue reports that GitHub OAuth sign-in fails after the same GitHub account changes username, and asks ClawHub to key identity on the immutable GitHub user ID while treating username as mutable profile data.

## Best Possible Solution

Keep the numeric providerAccountId and profile-sync implementation, then have maintainers verify production auth state and add an audited recovery path only if triage shows a repeatable verifier, duplicate-account, or relink failure.

## Reproduction Assessment

No. The report gives plausible OAuth rename steps and the related report adds a concrete verifier-mismatch symptom, but a high-confidence current-main reproduction requires live GitHub OAuth callback state and production auth records.

## Solution Assessment

Unclear for the remaining live symptom. Numeric providerAccountId lookup plus post-login profile sync is the right code foundation, but any relink, consolidation, or verifier repair needs maintainer production identity verification first.
