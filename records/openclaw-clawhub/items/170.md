## Summary

Keep open. Current main has the right stable GitHub numeric-ID and login-time profile-sync foundation, and the old proposed fix is superseded by merged work, but this is an open ClawHub issue and the related production OAuth/account-state reports are still unresolved.

## What This Changes

The issue reports that GitHub OAuth sign-in stops recognizing the same account after a GitHub username rename and should rely on the immutable GitHub user ID.

## Best Possible Solution

Keep identity anchored to numeric `providerAccountId`, then have maintainers verify production OAuth/account state and add an audited recovery path only if a repeatable verifier, duplicate-account, or relink failure remains.

## Reproduction Assessment

No. The report gives plausible OAuth rename steps and related issues add concrete production symptoms, but a high-confidence current-main reproduction needs live GitHub OAuth callback state, browser storage/cookies, Convex Auth logs, and production account records.

## Solution Assessment

Unclear. Numeric provider ID lookup plus post-login profile sync is the right code foundation, but any remaining relink, consolidation, or verifier repair should be scoped only after maintainer production identity verification.
