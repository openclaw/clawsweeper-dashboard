The remaining login symptom is also represented by open related report #1109, so this needs maintainer production/account-state triage rather than closure.

## What This Changes

The issue reports that GitHub OAuth sign-in fails after a user changes their GitHub username, and asks ClawHub to use GitHub’s immutable numeric user ID for account lookup while treating the username as mutable profile data.

## Best Possible Solution

Production should authenticate GitHub users by numeric providerAccountId, refresh mutable GitHub login/avatar/profile and personal-publisher fields for active accounts, and provide a maintainer-owned recovery or consolidation path for stale, deleted/deactivated, duplicate, or verifier/session state that still blocks reauth.
