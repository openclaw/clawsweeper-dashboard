## What This Changes

The issue asks ClawHub to keep GitHub OAuth sign-in working after a username rename by using the immutable GitHub user ID for account lookup and treating the username as mutable profile data.

## Best Possible Solution

Production should authenticate GitHub users by numeric `providerAccountId`, refresh mutable GitHub profile fields for active users, and provide a maintainer-owned recovery or consolidation path for stale, duplicate, deleted/deactivated, or OAuth verifier state that still blocks reauth.
