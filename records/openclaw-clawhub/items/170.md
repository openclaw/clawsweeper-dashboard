## What This Changes

The issue asks ClawHub to keep GitHub OAuth sign-in working after a GitHub username change by using immutable GitHub user IDs and refreshing mutable username/profile fields.

## Best Possible Solution

Restore production GitHub OAuth start/callback behavior, keep account identity anchored to immutable providerAccountId, and add or perform maintainer-verified recovery only if production state shows stale, duplicate, deleted/deactivated, or verifier/session records blocking reauth.
