## Summary

Keep open. Current main has the stable GitHub numeric-ID and profile-sync foundation, but this is an open ClawHub issue, the cleanup policy keeps issues open, the current production GitHub sign-in route still returns HTTP 500, and the related open report at https://github.com/openclaw/clawhub/issues/1109 keeps the account-state/verifier side unresolved.

## What This Changes

Keep ClawHub GitHub sign-in working after a GitHub account rename by using immutable GitHub IDs and refreshing mutable profile fields.

## Best Possible Solution

Restore the production GitHub OAuth redirect/session path, keep identity anchored to numeric providerAccountId, inspect affected auth rows, and add an audited recovery or consolidation path only if production triage proves one is needed.
