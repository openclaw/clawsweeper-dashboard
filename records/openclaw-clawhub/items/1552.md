Keep open. ClawHub cleanup policy does not auto-close issues, and current production main still shows an unresolved OAuth outage: the site and public API respond, but both GitHub sign-in endpoints return HTTP 500 before the reported callback path can be replayed.

## What This Changes

The issue reports GitHub OAuth completing with a callback redirect but failing to create an authenticated session, especially after a prior account deletion.

## Best Possible Solution

Restore production GitHub OAuth so active accounts receive a browser session, keep deleted/banned-account outcomes explicit, and add focused smoke or regression coverage once the failing layer is identified.
