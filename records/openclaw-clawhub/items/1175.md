The issue reports that `clawhub publish` fails when personal-publisher creation triggers owner-publisher search digest syncs that violate Convex's single paginated-query limit.

## Best Possible Solution

Keep the cursor-continuation implementation, verify the production Convex backend with a publish smoke test/log check, then close or consolidate the duplicate publish-pagination reports.

## Reproduction Assessment

Yes for the original failure: the issue gives CLI steps and a stack trace, and related reports show the same publish/import path. Against current main, the specific repeated-pagination owner-publisher helper shape is no longer present.

## Solution Assessment

Yes. One owner-publisher digest page per internal mutation with scheduled continuation cursors is the narrow Convex-safe fix; the closed collect-based PR would be less bounded.
