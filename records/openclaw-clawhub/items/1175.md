## What This Changes

The issue reports that `clawhub publish` fails while ensuring a personal publisher because owner-publisher search-digest rebuilds trigger Convex's multiple-pagination guard.

## Best Possible Solution

Keep the cursor-based owner-publisher digest rebuild implementation, verify any affected account can publish on production if desired, then have a maintainer close or consolidate this issue with the fixed publish-pagination reports.

## Reproduction Assessment

No for current main: source inspection shows the reported owner-publisher rebuild helpers now do one `.paginate()` per invocation and pass continuation cursors through scheduled internal mutations. I did not run a live production publish, so deployed behavior is not independently reproduced here.

## Solution Assessment

Yes, the current-main continuation approach is the best code direction because it preserves indexed pagination and avoids unbounded `.collect()` reads. The remaining action is maintainer issue cleanup and optional production smoke verification, not a new code repair.
