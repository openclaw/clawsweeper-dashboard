## Summary

Keep open. Repository policy for this ClawHub cleanup pass keeps issues open, and the source still shows only partial coverage: package/CLI paths ignore `.git/`, but skill upload, multipart API, GitHub import, and backend publish scan paths still use macOS-junk-only filtering before storage and scanning.

## What This Changes

The issue requests that standard VCS and Python generated artifacts such as `.git/`, `__pycache__/`, `.pyc`, and `.pyo` be excluded or distinguished so clean skill publishes are not flagged as suspicious.

## Best Possible Solution

Add a small shared publish-artifact cleanup helper used before persistence and static/LLM moderation across skill upload expansion, multipart skill API parsing, GitHub import, and backend publish filtering. It should cover only `.git/`, `__pycache__/`, `.pyc`, and `.pyo` while preserving normal source-file scanning and existing macOS-junk handling.
