## Summary

Keep open. ClawHub cleanup policy keeps issues open, and this remains a live production moderation/search-visibility case: `cloudq` is currently flagged suspicious, while current main intentionally suppresses suspicious skills from the normal unified-search path.

## What This Changes

The reporter uploaded the `cloudq` skill and asks why searching ClawHub for `cloudq` does not surface it in the web UI.

## Best Possible Solution

A maintainer should review `cloudq` in production moderation, rescan or clear the suspicious verdict if it is wrong, and keep the suspicious-skill search filter unless a deliberate safety policy change is made.

## Reproduction Assessment

Yes for the hosted behavior: the live detail API reports `cloudq` as suspicious, live default search can return it, and `nonSuspiciousOnly=true` returns no results. Source inspection shows the web unified search passes `nonSuspiciousOnly: true`, so the UI follows that suppression path.

## Solution Assessment

Unclear as a source change. The narrow maintainable path is production moderation review, rescan, or manual override if the verdict is wrong; changing search to surface suspicious skills would conflict with current documented filtering.
