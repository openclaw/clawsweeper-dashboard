Keep open. ClawHub cleanup policy keeps issues open, and the report still maps to a live moderation/search-visibility case: the uploaded skill exists publicly, but current web search deliberately suppresses suspicious skills by default.

## What This Changes

A publisher wants the uploaded `cloudq` skill to be discoverable from ClawHub search by its slug.

## Best Possible Solution

A maintainer should inspect the production `cloudq` moderation state, rescan or clear it if the verdict is wrong, and decide whether owner-facing search suppression guidance or appeal/status UX needs to change.

## Reproduction Assessment

Yes. Current production APIs reproduce the visibility split: `cloudq` appears in unfiltered public search and detail lookup, but disappears when `nonSuspiciousOnly=true`, which is the filter used by the current web search path.

## Solution Assessment

Unclear. The current suppression is consistent with safe defaults, so the best fix may be moderation action or clearer owner-facing status rather than changing search to show suspicious skills by default.
