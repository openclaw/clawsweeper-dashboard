## Summary

Keep open. ClawHub cleanup policy does not auto-close issues, and this report still reproduces as a live moderation/search visibility boundary for `cloudq`: the skill exists, but public web search uses the non-suspicious filter that suppresses it.

## What This Changes

The issue reports that the uploaded skill `cloudq` cannot be found through ClawHub search by its slug.

## Best Possible Solution

A maintainer should inspect and, if appropriate, rescan or clear the production `cloudq` moderation state, then clarify owner-facing visibility or appeal guidance without weakening the non-suspicious public search filter.

## Reproduction Assessment

Yes. Live API checks reproduce the report: `cloudq` exists and appears in default search, but disappears when `nonSuspiciousOnly=true`, which is the flag used by the web search hook.

## Solution Assessment

Unclear. Current main deliberately suppresses suspicious skills from non-suspicious search, so the narrowest maintainable path is production moderation review first, not an automated code change.
