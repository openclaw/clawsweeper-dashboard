Keep open. ## What This Changes

The issue asks for `clawhub publish .` / `clawhub skill publish .` to recognize an existing `SKILL.md` in the caller's skill directory instead of failing with `SKILL.md required`.

## Best Possible Solution

Keep the issue open and fix `cmdPublish` so user-entered relative source paths can resolve against the caller's cwd for skill folders, with regression coverage for a different configured workdir.

## Reproduction Assessment

Yes. A high-confidence reproduction path is a cwd containing `SKILL.md` plus a configured Clawdbot/OpenClaw default workspace elsewhere; current source resolves `.` under that workspace and can hit the existing `SKILL.md required` guard.

## Solution Assessment

Yes, a focused publish-only path resolution fix is the best direction. It should avoid broad `resolveWorkdir` changes so install/update/sync continue to use configured workspaces as designed.
