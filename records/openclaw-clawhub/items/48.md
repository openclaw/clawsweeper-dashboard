## Summary

Keep open: current main still has a source-reproducible CLI path where `skill publish .` can inspect the resolved global workdir instead of the caller's skill directory, and ClawHub policy keeps issues open rather than auto-closing them. The released 0.12.3 manifest-recovery fix addressed a related broad-ignore failure, but it does not change this relative-path/workdir behavior.

## What This Changes

The issue reports that CLI skill publishing fails with `SKILL.md required` even when the target directory contains a valid `SKILL.md`.

## Best Possible Solution

Skill publish should resolve user-entered relative source paths against the caller's current directory when that path is an existing skill folder, while retaining the configured workdir behavior for installed-workdir operations and fallback cases.

## Reproduction Assessment

Yes, from source inspection. With a configured Clawdbot/OpenClaw default workspace that differs from cwd and no cwd marker, `clawhub skill publish .` can scan the configured workspace and throw `SKILL.md required` even when cwd contains `SKILL.md`.

## Solution Assessment

Yes. The narrow maintainable fix is to adjust only skill-publish source-path resolution and add regression coverage for a cwd skill folder with a different configured/global workdir, without broad workdir or sync behavior changes.
