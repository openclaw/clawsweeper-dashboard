## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has a credible path to the reported publish failure because relative publish paths are resolved under the global workdir rather than necessarily the caller directory containing `SKILL.md`; related #1509 remains open for the same user-visible error.

## What This Changes

The issue asks for `clawhub publish .` / `clawhub skill publish .` to stop failing with `SKILL.md required` when the target folder contains a valid `SKILL.md`.

## Best Possible Solution

Skill publish should resolve user-entered relative source paths against the caller's cwd when that path identifies a skill folder, while preserving absolute paths and existing configured-workdir behavior for install/update/sync flows.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence static reproduction: when the resolved global workdir differs from the shell cwd, `skill publish .` scans the global workdir for `SKILL.md` instead of the cwd skill folder.

## Solution Assessment

Yes. A narrow publish-only path-resolution fix plus regression coverage is the maintainable path; broad changes to global workdir semantics would be riskier.

## Repair Work Prompt

Fix the CLI skill publish path-resolution bug where `clawhub publish .` or `clawhub skill publish .` can throw `SKILL.md required` even though the caller's current directory contains `SKILL.md`. Keep the change scoped to skill publish: user-entered relative source paths should resolve against the process cwd when that identifies the requested skill folder, absolute paths should keep working, and install/update/sync global workdir semantics should not be changed. Define and test the precedence for `--workdir` / `CLAWHUB_WORKDIR` with publish. Add focused regression coverage for cwd containing `SKILL.md` while the resolved global workdir points elsewhere, and make sure related #1509 would be covered by the same behavior. Add a CHANGELOG.md entry if the repo treats this user-facing CLI fix as release-noted.
