## Summary

Keep open: ClawHub cleanup policy does not auto-close issues, and current main still has a source-reproducible path where `clawhub skill publish .` can resolve `.` against the configured global workdir instead of the caller's skill folder. The broad-ignore `SKILL.md` omission was fixed in `v0.12.3`, but the remaining relative-path/workdir behavior is still not covered; the related open report is https://github.com/openclaw/clawhub/issues/1509.

## What This Changes

The issue reports CLI publish failing with `SKILL.md required` even when the caller's skill folder contains a valid `SKILL.md`.

## Best Possible Solution

Skill publish should resolve user-entered relative source paths against the caller's current directory when they point to a local skill folder, while preserving absolute paths and configured-workdir flows, with focused regression coverage.

## Reproduction Assessment

Yes, by source inspection. With a configured Clawdbot/OpenClaw default workspace different from an unmarked cwd, `clawhub skill publish .` can scan the configured workspace and hit `SKILL.md required` even when the caller's cwd contains `SKILL.md`.

## Solution Assessment

Yes. The narrow maintainable repair is skill-publish source-path resolution plus regression coverage; backend validation and the already-shipped broad-ignore manifest recovery should not be broadened for this remaining path.

## Repair Work Prompt

Fix the remaining `SKILL.md required` publish failure tracked by https://github.com/openclaw/clawhub/issues/48 and https://github.com/openclaw/clawhub/issues/1509. In `packages/clawhub/src/cli/commands/publish.ts`, make `cmdPublish` resolve user-entered relative source paths against `process.cwd()` when that path exists and points to the requested local skill folder, especially `.` from inside a folder containing `SKILL.md` or `SKILLS.md`. Preserve absolute paths and existing configured-workdir behavior for install/update/sync flows and for publish paths that intentionally refer to folders under `opts.workdir`. Add focused regression coverage in `packages/clawhub/src/cli/commands/publish.test.ts` for a cwd skill folder with `opts.workdir` pointing elsewhere, plus coverage that normal workdir-relative and absolute publish paths still work. Do not change backend publish validation, auth, registry API behavior, or the broad-ignore manifest recovery. Add an Unreleased Fixes changelog bullet if the repair lane is expected to carry release-note updates.
