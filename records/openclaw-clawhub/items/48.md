## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has a source-level path to the reported failure when `skill publish .` is run from a skill folder while the resolved global workdir points elsewhere.

## What This Changes

The issue reports that `clawhub publish .` fails with `SKILL.md required` even when the current directory contains a valid `SKILL.md`.

## Best Possible Solution

Skill publish should resolve user-entered relative source paths consistently with the documented cwd-based publish workflow, while preserving absolute paths and existing configured-workdir behavior for install/update/sync flows.

## Reproduction Assessment

Yes from source inspection. With a configured default workspace that differs from cwd and no cwd marker, `resolveWorkdir` can return that workspace, then `cmdPublish` resolves `.` there and can miss the `SKILL.md` in the caller's skill directory.

## Solution Assessment

Yes. A narrow CLI fix should adjust publish source-path resolution and add regression coverage for a cwd skill folder with a different configured workdir, without changing unrelated sync/install semantics.
