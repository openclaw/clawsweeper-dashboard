## Summary

Keep open. Current main still has a source-backed path to the reported `SKILL.md required` failure: `cmdPublish` resolves relative publish paths against the resolved global workdir, which can differ from the caller's current skill directory, and the documented workflow uses `skill publish .`; related #1509 remains the same user-visible failure cluster.

## What This Changes

The issue reports that CLI skill publishing fails with `SKILL.md required` even when the target directory contains a valid `SKILL.md`.

## Best Possible Solution

Make skill publish resolve user-entered relative source paths from the caller's cwd when they point to a skill folder, while preserving absolute paths and existing configured-workdir behavior for other CLI flows.

## Reproduction Assessment

Yes. A high-confidence reproduction path is a cwd containing `SKILL.md` plus a configured or env global workdir elsewhere; source inspection shows `skill publish .` can scan that other workdir and then throw `SKILL.md required`.

## Solution Assessment

Yes. A focused publish-only path resolution fix with regression coverage is the narrowest maintainable solution; changing global `resolveWorkdir` behavior would be riskier.

## Repair Work Prompt

Fix the CLI skill publish path-resolution bug reported here and in related #1509. `clawhub skill publish .` and the legacy `clawhub publish .` should find `SKILL.md` in the caller's current skill directory even when `CLAWHUB_WORKDIR`, `--workdir`, or a configured Clawdbot/OpenClaw default workspace points elsewhere. Keep the change scoped to skill publish path handling; do not broadly change global `resolveWorkdir` semantics for install/update/sync. Preserve absolute path behavior and sync's existing absolute `skill.folder` calls. Add regression coverage in `packages/clawhub/src/cli/commands/publish.test.ts` for a cwd skill folder with a different configured/global workdir, and add a user-facing `CHANGELOG.md` fix entry. Stop and ask for maintainer guidance if the solution requires a new CLI flag or product-policy change.
