## Summary

Keep open. Repository policy for this ClawHub pass keeps issues open, and current main still has a credible source-backed path to the reported `SKILL.md required` failure because relative publish paths are resolved under the resolved global workdir instead of necessarily under the caller's skill directory.

## What This Changes

The issue reports that `clawhub publish .` fails with `SKILL.md required` even though the skill directory contains a valid `SKILL.md`.

## Best Possible Solution

Skill publish should resolve user-entered relative source paths against the caller's current directory when that path points to a skill folder, while preserving absolute paths and existing configured-workdir behavior for other CLI flows.

## Reproduction Assessment

Yes. A high-confidence reproduction path is a cwd containing `SKILL.md` plus a configured Clawdbot/OpenClaw default workspace elsewhere; current main resolves `skill publish .` under that workspace and can miss the cwd file.

## Solution Assessment

Yes. A focused publish-only path resolution fix with regression coverage is narrower and safer than changing global `resolveWorkdir` semantics across install, update, and sync.
