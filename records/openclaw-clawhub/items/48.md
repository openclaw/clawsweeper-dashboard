## Summary

Keep open. ClawHub cleanup policy for this pass keeps issues open, and current main still has a source-reproducible path where `skill publish .` can resolve the source folder under a configured global workdir instead of the caller's skill directory.

## What This Changes

The issue reports that `clawhub publish .` fails with `SKILL.md required` even when a valid `SKILL.md` exists in the directory being published.

## Best Possible Solution

Update the CLI publish command so user-entered relative source paths resolve to the caller's intended skill directory when it contains `SKILL.md`, while preserving explicit workdir behavior for workflows that rely on configured Clawdbot/OpenClaw workspaces.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: run `clawhub skill publish .` from a skill directory while the resolved global workdir points elsewhere, and publish can inspect the wrong folder before throwing `SKILL.md required`.

## Solution Assessment

Yes. The narrow maintainable fix is to adjust publish source-path resolution and add a regression test for a cwd skill folder with a different configured workdir, without broadening install/update/sync workdir behavior.
