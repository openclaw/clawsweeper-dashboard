## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has a source-reproducible CLI path where `skill publish .` resolves the source folder under the global workdir rather than the caller's skill directory.

## What This Changes

Fix CLI publish failing with `SKILL.md required` even when the target skill folder contains a valid `SKILL.md`.

## Best Possible Solution

Make `skill publish` resolve user-entered local relative source paths to the caller's intended skill folder when that folder contains a root manifest, while preserving absolute paths and existing non-publish workdir semantics.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: run `clawhub skill publish .` from a skill directory while the resolved global workdir points elsewhere, and publish can inspect the wrong folder before throwing `SKILL.md required`.

## Solution Assessment

Yes. The narrow maintainable fix is publish-specific source-path resolution plus regression coverage, not backend validation changes or a broad rewrite of global workdir behavior.
