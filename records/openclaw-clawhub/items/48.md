## Summary

Keep open: current main still has a source-level path to the reported `SKILL.md required` failure, and the ClawHub cleanup policy keeps issues open. Related #1509 is open with the same user-visible publish error, so one focused CLI repair should cover both.

## What This Changes

The issue reports that `clawhub`/legacy `clawdhub publish .` errors with `SKILL.md required` even when `SKILL.md` exists in the skill folder.

## Best Possible Solution

Make skill publish resolve user-entered relative source paths from the caller's current directory when that denotes a skill folder, while preserving absolute paths and existing configured-workdir semantics for other CLI flows, with regression coverage.

## Reproduction Assessment

Yes, from source inspection. With a configured Clawdbot/OpenClaw default workspace that differs from cwd and no cwd marker, `skill publish .` scans the configured workspace and can hit `SKILL.md required` even when cwd contains `SKILL.md`.

## Solution Assessment

Yes. The narrow maintainable fix is to adjust publish source-path resolution and add a regression test for cwd skill folders with a different configured workdir, without changing install/update/sync semantics.
