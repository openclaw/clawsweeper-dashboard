## Summary

Keep open: current main still has a credible source-level path to the reported `SKILL.md required` failure, and ClawHub policy keeps issues open. Related #1509 is open with the same user-visible publish error, so one focused CLI repair should cover both reports.

## What This Changes

The issue reports that `clawhub`/`clawdhub publish` rejects a valid skill directory with `SKILL.md required` even though `SKILL.md` exists.

## Best Possible Solution

Skill publish should resolve user-entered relative source paths from the caller's cwd when they identify a skill folder, while preserving existing workspace-driven install/update/sync behavior and adding focused regression coverage.

## Reproduction Assessment

Yes, from source inspection. With a configured Clawdbot/OpenClaw default workspace that differs from cwd and no cwd marker, `skill publish .` can scan the configured workspace and throw `SKILL.md required` even when cwd contains `SKILL.md`.

## Solution Assessment

Yes. The narrow maintainable fix is to adjust publish source-path resolution and add regression tests for cwd skill folders with a different configured/global workdir, without changing install/update/sync semantics broadly.
