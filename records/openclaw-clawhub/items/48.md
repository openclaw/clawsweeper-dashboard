## Summary

Keep open. ClawHub policy keeps issues open, and current main still has a source-backed path to the reported `SKILL.md required` failure when a relative publish path is resolved under the global workdir instead of the caller's skill directory.

## What This Changes

The issue reports that CLI skill publishing fails with `SKILL.md required` even though the requested source directory contains a valid `SKILL.md`.

## Best Possible Solution

Make `skill publish` resolve relative source paths from the caller's cwd when that candidate is a skill folder, while preserving absolute paths and existing workdir behavior for other CLI flows, with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: run `skill publish .` from a directory containing `SKILL.md` while the resolved global workdir points to a different configured Clawdbot/OpenClaw workspace, causing publish to inspect the wrong folder.

## Solution Assessment

Yes. A publish-scoped source-path resolver plus regression test is the narrowest maintainable fix; changing global workdir resolution would have a broader blast radius.

## Repair Work Prompt

Fix ClawHub CLI skill publish path resolution for relative source paths. On current main, `cmdPublish` resolves `folderArg` with `resolve(opts.workdir, folderArg)`, so `clawhub skill publish .` from a skill directory can look in a configured Clawdbot/OpenClaw workspace instead of the caller's cwd and throw `SKILL.md required`. Implement a publish-scoped resolver that keeps absolute paths unchanged and resolves relative paths against `process.cwd()` when that candidate is a skill folder/root `SKILL.md`; preserve existing `--workdir`/`CLAWHUB_WORKDIR` behavior for install/update/sync and avoid broad global `resolveWorkdir` changes. Add regression coverage for cwd containing `SKILL.md` while `opts.workdir` points elsewhere, keep existing plugin-folder rejection behavior, and add a `CHANGELOG.md` Unreleased fix entry. Do not change backend publish APIs unless tests prove the CLI fix is insufficient.
