## Repair Work Prompt

Fix the ClawHub CLI skill publish path resolution bug reported in openclaw/clawhub#48 and openclaw/clawhub#1509. Current `cmdPublish` resolves `skill publish <path>` with `resolve(opts.workdir, folderArg)`, which makes `skill publish .` scan the configured global workdir instead of the caller's cwd when Clawdbot/OpenClaw workspace config exists elsewhere. Implement a narrow fix so user-entered relative source paths for skill publish resolve against the process cwd when that resolves to a skill folder or `SKILL.md`, while preserving absolute paths and existing configured-workdir behavior for install/update/sync flows. Add focused regression coverage for publishing a cwd skill folder when `GlobalOpts.workdir` points elsewhere. Do not rewrite unrelated CLI workdir semantics, package publish behavior, registry/auth handling, or Convex code. Include a short changelog/PR note describing the CLI publish fix.
