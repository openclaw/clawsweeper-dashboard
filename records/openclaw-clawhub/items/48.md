## Summary

Keep open: current main still has a source-level path to the reported failure, and ClawHub cleanup policy keeps issues open. Related #1509 is open and reports the same user-visible publish error, so one focused CLI repair should cover both.

## What This Changes

The issue asks for `clawhub`/`clawdhub publish .` to publish a valid skill directory instead of throwing `SKILL.md required` when `SKILL.md` exists.

## Best Possible Solution

Skill publish should resolve user-entered relative source paths from the caller’s current directory when that denotes the skill folder, while preserving absolute paths and existing configured-workdir behavior for other CLI flows, with focused regression coverage.

## Reproduction Assessment

Yes, from source inspection. With a configured Clawdbot/OpenClaw default workspace that differs from cwd and no cwd marker, `skill publish .` can scan the configured workspace instead of the caller’s skill folder and then hit `SKILL.md required`.

## Solution Assessment

Yes. The narrow maintainable fix is to adjust publish source-path resolution and add a regression test for cwd skill folders with a different configured workdir, without changing install/update/sync semantics.

## Repair Work Prompt

Fix the CLI skill publish path-resolution bug reported here and in #1509. On current main, `cmdPublish` resolves `<path>` with `resolve(opts.workdir, folderArg)`, so `clawhub skill publish .` run from a skill folder can inspect a configured Clawdbot/OpenClaw workdir instead of cwd and then throw `SKILL.md required`. Implement a narrow fix in `packages/clawhub/src/cli/commands/publish.ts` and tests as needed so relative publish sources that resolve from `process.cwd()` to an existing skill folder are used from cwd when the global workdir came from implicit Clawdbot/OpenClaw fallback; preserve absolute paths and explicit `--workdir`/`CLAWHUB_WORKDIR` behavior, and do not change install, update, sync, or package publish semantics. Add regression coverage in `packages/clawhub/src/cli/commands/publish.test.ts` for a cwd skill folder with a different configured workdir and no `.clawhub` marker. Add a user-facing `fix` entry to `CHANGELOG.md` unless maintainers decide this release stream handles CLI changelog elsewhere.
