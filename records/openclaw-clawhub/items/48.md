## What This Changes

Publishing a skill from a directory containing `SKILL.md` should not fail with `SKILL.md required` when using `clawhub publish .` or `clawhub skill publish .`.

## Best Possible Solution

Skill publish should resolve user-entered relative source paths against the caller's cwd when that path is a skill folder, while preserving absolute paths and existing configured-workdir behavior for install/update/sync flows; add focused regression coverage.

## Repair Work Prompt

Fix the `clawhub publish .` / `clawhub skill publish .` bug where a valid cwd skill folder can fail with `SKILL.md required` because `cmdPublish` resolves relative source paths under the resolved global workdir. Keep the change focused to skill publish path handling: relative source paths should work from the caller's cwd when that path is a skill folder, absolute paths must remain absolute, and install/update/sync configured-workdir behavior must not be broadened accidentally. Add regression coverage for a cwd skill folder with a different configured/global workdir, cover the related #1509 scenario, and add a concise `CHANGELOG.md` entry for the user-facing fix. Do not rewrite package publish behavior or global workdir resolution unless a small helper is needed for publish only.
