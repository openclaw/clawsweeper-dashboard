## Summary

Keep open. Current main still has a source-reproducible CLI path where `skill publish .` resolves the source folder under the resolved global workdir instead of the caller's skill directory, and the related open report at https://github.com/openclaw/clawhub/issues/1509 describes the same user-visible failure.

## What This Changes

The issue reports that `clawhub publish`/`skill publish` throws `SKILL.md required` even when the target skill folder contains a valid `SKILL.md`.

## Best Possible Solution

Make skill publish resolve local relative source paths to the caller's intended skill folder when that folder contains a root manifest, while preserving absolute paths and non-publish workdir semantics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with a configured Clawdbot/OpenClaw default workspace different from an unmarked cwd, `clawhub skill publish .` can inspect the wrong folder and hit `SKILL.md required` even when the caller's cwd contains `SKILL.md`.

## Solution Assessment

Yes. The narrow maintainable fix is publish-specific source-path resolution plus regression coverage, not backend validation changes or a broad rewrite of global workdir behavior.

## Repair Work Prompt

Fix the ClawHub CLI skill publish path-resolution bug reported in https://github.com/openclaw/clawhub/issues/48 and https://github.com/openclaw/clawhub/issues/1509. Current `cmdPublish` resolves `folderArg` with `resolve(opts.workdir, folderArg)`, so when the global workdir comes from `--workdir`, `CLAWHUB_WORKDIR`/`CLAWDHUB_WORKDIR`, or `resolveClawdbotDefaultWorkspace()` and differs from `process.cwd()`, `clawhub skill publish .` can inspect the wrong folder and throw `SKILL.md required`. Add a conservative publish-specific resolver: absolute paths stay absolute; relative paths that resolve from the caller cwd to an existing skill folder with root `SKILL.md` or `SKILLS.md` should be used for publish; avoid broad changes to `resolveWorkdir` and preserve install/update/sync semantics. Add regression coverage for a cwd skill folder with a different configured/global workdir and keep the existing broad-ignore root-manifest recovery passing. Add docs or a concise changelog note only if current repository policy requires it for this user-facing CLI fix.
