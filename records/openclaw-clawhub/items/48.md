## What This Changes

The issue reports that CLI publish throws `SKILL.md required` even when the selected skill folder contains a valid `SKILL.md`.

## Repair Work Prompt

Fix the CLI skill publish path-resolution bug reported in https://github.com/openclaw/clawhub/issues/48 and https://github.com/openclaw/clawhub/issues/1509. Current `cmdPublish` resolves relative source paths with `resolve(opts.workdir, folderArg)`, which can make `clawhub skill publish .` inspect a configured Clawdbot/OpenClaw workspace instead of the caller's skill directory. Implement a conservative resolver for skill publish only: absolute paths should keep working; explicit `--workdir` / env workdir semantics should be preserved where clearly intentional; install/update/sync and package publish behavior should not be broadened. Add focused regression coverage for a cwd skill folder containing `SKILL.md` when the resolved global workdir points elsewhere, and keep the existing broad-ignore root-manifest recovery intact. Do not change backend publish validation or unrelated CLI workdir behavior. Add/update changelog or docs only if the repo's current release policy requires it for this user-facing CLI fix.
