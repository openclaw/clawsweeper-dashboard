## Repair Work Prompt

Fix the remaining ClawHub CLI skill-publish path-resolution bug reported in https://github.com/openclaw/clawhub/issues/48 and https://github.com/openclaw/clawhub/issues/1509. Current `cmdPublish` resolves relative source paths with `resolve(opts.workdir, folderArg)`, so `clawhub skill publish .` can inspect a configured Clawdbot/OpenClaw global workdir instead of the caller's skill directory containing `SKILL.md`. Implement a publish-specific resolver that handles absolute paths unchanged, resolves cwd-relative skill folders when the requested relative path points to a folder with `SKILL.md` or `SKILLS.md`, and falls back conservatively to existing workdir-relative behavior when that cwd-relative folder is not a skill. Preserve the landed `ensureRootManifestFile` broad-ignore recovery from commit `6925ec761c56`; do not change install/update/sync global workdir semantics or backend validation. Add focused regression coverage for a cwd skill folder with a different configured/global workdir. Add/update docs only if the behavior or troubleshooting guidance changes.
