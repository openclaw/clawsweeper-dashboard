## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has a source-supported path to the reported `SKILL.md required` failure because `skill publish .` resolves the source path under the global workdir instead of necessarily using the caller's current directory.

## What This Changes

The issue reports that CLI skill publishing fails with `SKILL.md required` even when the target folder contains a valid `SKILL.md`.

## Best Possible Solution

Fix `cmdPublish` so user-entered relative source paths resolve against the caller's cwd when that path is a skill folder, while preserving absolute paths and existing configured-workdir behavior for other CLI flows.

## Reproduction Assessment

Yes. A high-confidence reproduction path is a cwd containing `SKILL.md` plus a resolved Clawdbot/OpenClaw workdir elsewhere; current source then scans the workdir path for `.` and can hit the existing `SKILL.md required` guard.

## Solution Assessment

Yes. A focused publish-only path resolution fix with regression coverage is the narrowest maintainable direction; broad changes to `resolveWorkdir` would carry unnecessary risk for install/update/sync behavior.
