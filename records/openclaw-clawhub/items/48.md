Keep the change scoped to publish/source path resolution; do not broadly change install, update, sync, or global `resolveWorkdir` semantics. Preserve absolute path behavior and define/cover precedence for `--workdir` and `CLAWHUB_WORKDIR` with relative publish paths. Add regression tests for a cwd skill folder while `opts.workdir` points elsewhere, plus any minimal tests needed around existing configured-workdir behavior. Do not add unrelated refactors or changelog churn unless the repo requires a CLI bugfix changelog entry.
