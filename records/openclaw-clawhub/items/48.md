## Summary

Keep open: ClawHub issue cleanup should not auto-close open issues, and current main still has a source-reproducible path where `skill publish .` can scan the configured global workdir instead of the caller's skill folder. The broad-ignore `SKILL.md` omission was fixed in `v0.12.3`, but the remaining relative-path/workdir behavior is still not covered.

## What This Changes

The issue reports that CLI skill publish returns `SKILL.md required` even when the selected skill directory contains a valid root `SKILL.md`.

## Best Possible Solution

Skill publish should resolve user-entered relative source paths against the caller's current directory when that points to the requested skill folder, while preserving absolute paths and configured-workdir behavior for install/update/sync flows, with focused regression coverage.

## Reproduction Assessment

Yes, by source inspection. With a configured Clawdbot/OpenClaw default workspace different from an unmarked cwd, `clawhub skill publish .` can scan the configured workspace and throw `SKILL.md required` even when cwd contains `SKILL.md`.

## Solution Assessment

Yes. The narrow maintainable repair is skill-publish source-path resolution plus regression coverage; backend validation and the already-shipped broad-ignore manifest recovery should not be broadened for this remaining path.

Do not change backend publish validation or package publish behavior.
