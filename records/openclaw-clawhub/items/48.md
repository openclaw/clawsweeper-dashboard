## What This Changes

The issue reports that `clawhub publish`/`clawhub skill publish` can reject a valid skill folder with `SKILL.md` as missing.

## Best Possible Solution

Make skill publish resolve user-entered relative source paths against the caller's cwd when that identifies a skill folder, while preserving absolute paths and existing configured-workdir behavior elsewhere.

## Reproduction Assessment

Yes, from source inspection. With a configured Clawdbot/OpenClaw default workspace that differs from cwd and no cwd marker, `clawhub skill publish .` can scan the configured workspace and throw `SKILL.md required` even when cwd contains `SKILL.md`.

## Solution Assessment

Yes. The narrow maintainable fix is to adjust publish source-path resolution and add regression coverage for cwd skill folders with a different configured/global workdir, without broad install/update/sync behavior changes.
