## What This Changes

Prevent checkbox selection order in the OpenClaw multi-model picker from unintentionally determining primary and fallback model roles.

## Best Possible Solution

In OpenClaw, expose an explicit primary model plus an ordered fallback list, then make the configuration CLI and Web UI serialize and synchronize that same role-based state without deriving priority from checkbox chronology.

## Reproduction Assessment

No. The report gives concrete steps for OpenClaw 2026.3.8, but the affected configuration flow is absent from this ClawHub checkout and no failing current-main OpenClaw run or regression test was available.

## Solution Assessment

Unclear. Explicit model roles are more maintainable than hidden click-order semantics, but OpenClaw owners must decide whether and how users explicitly reorder fallback priority across both interfaces.
