Keep this issue open. The ClawHub cleanup policy does not auto-close issues, and the current production deployment for the reviewed main SHA still has a live GitHub auth failure: the site and public skills API return 200, while the GitHub sign-in endpoint returns HTTP 500 before the original callback/no-session path can complete.
