## Summary

Keep this issue open. ClawHub cleanup policy does not auto-close open issues, and current production mapped to current main still shows a real GitHub sign-in failure: the site and skills API return 200, but both the clawhub.ai and direct Convex GitHub sign-in endpoints return HTTP 500 before the callback/session step.

## What This Changes

The issue reports GitHub OAuth authorization completing on clawhub.ai but returning to the upload/login UI without an authenticated session in Chrome.

## Best Possible Solution

Restore production GitHub OAuth so active accounts receive a durable Convex Auth session, preserve explicit blocked-account rejection, and add focused auth smoke or regression coverage after the failing layer is identified.

No for the exact callback-succeeds/no-session variant, because the flow currently fails before GitHub authorization can complete.
