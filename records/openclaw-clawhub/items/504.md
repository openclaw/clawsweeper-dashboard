## Summary

Keep open. Current main still rejects deleted/deactivated account reauth by design, the issue has multiple affected-user confirmations, and the remaining change needs maintainer-approved account recovery and GitHub identity-binding policy rather than cleanup closure.

## What This Changes

The issue asks for deleted GitHub-backed ClawHub accounts to be restorable, relinkable, or cleanly re-registered after OAuth instead of remaining signed out.

## Best Possible Solution

Keep this as the canonical deleted-account GitHub OAuth recovery policy item and implement a maintainer-approved path consistently across OAuth, CLI token creation, deletion, auth account binding, audit/operator flows, user-facing errors, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `deleteAccount` sets `deactivatedAt`/`purgedAt`, the OAuth callback calls `handleDeletedUserSignIn`, and the handler throws before web or CLI token flows can observe an authenticated active user.

## Solution Assessment

No for an immediate automated fix. Restoring, relinking, or allowing clean re-registration changes the current irreversible-deletion contract and needs a maintainer-approved security and privacy policy first.
