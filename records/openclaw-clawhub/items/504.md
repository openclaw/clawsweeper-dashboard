## Summary

Keep open. Current main still makes user-initiated deletion permanent and rejects GitHub OAuth reauthentication for deactivated/deleted accounts, while this issue and related reports show a real affected-user cluster. ClawHub cleanup policy also keeps issues open rather than auto-closing them from this review lane.

## What This Changes

The issue requests a recovery, relinking, or clean re-registration path for users who deleted a ClawHub account and then cannot sign in again with the same GitHub identity.

## Best Possible Solution

Keep this as the canonical deleted-account GitHub OAuth recovery triage item and implement a maintainer-approved policy consistently across OAuth, account deletion, auth bindings, CLI token creation, operator/audit flows, user-facing errors, docs/specs, and regression tests.

## Reproduction Assessment

Yes for the core blocked path by source inspection: a deleted or deactivated user signing in through GitHub reaches handleDeletedUserSignIn and is rejected before a usable session or CLI token can be created. I did not run a live production OAuth flow against affected account state.

## Solution Assessment

Unclear. Auto-restoring or auto-recreating accounts is not obviously the safest fix because current docs and tests make deletion irreversible; maintainers should first choose the account recovery policy and then implement the narrow flow that matches it.
