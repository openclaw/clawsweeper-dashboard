## Summary

Keep open. Current main still blocks OAuth sign-in for deleted/deactivated accounts, and the remaining request is a security- and privacy-sensitive account recovery or re-registration policy rather than conservative cleanup.

## What This Changes

Requests a way for users who deleted their ClawHub account to sign back in with GitHub and regain account and CLI token access.

## Best Possible Solution

Keep this as the canonical deleted-account GitHub OAuth recovery triage item and implement a maintainer-approved policy consistently across OAuth, auth account bindings, account deletion, operator tooling, docs/specs, and regression tests.

## Reproduction Assessment

Yes, by source inspection. A deleted or deactivated user signing in through GitHub reaches `handleDeletedUserSignIn`, which rejects the session; I did not run a live production OAuth flow.

## Solution Assessment

Unclear. Automatically restoring or recreating accounts is not yet the best fix because it contradicts the current irreversible-deletion spec; maintainers should choose the recovery/security policy first.
