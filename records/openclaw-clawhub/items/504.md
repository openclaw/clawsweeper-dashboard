## Summary

Keep open. Current main still blocks deleted/deactivated accounts from signing in, the discussion and related search show many affected users asking for recovery or re-registration, and the current RFC leaves account recovery after owner-initiated deletion as an unresolved trust/product decision.

## What This Changes

Request a GitHub OAuth recovery or re-registration path after a user deletes their ClawHub account and can no longer get a session or CLI token.

## Best Possible Solution

Keep this as the canonical deleted-account GitHub OAuth recovery triage item and implement the maintainer-approved policy consistently across OAuth, account deletion, auth bindings, CLI tokens, audit/operator flows, docs/specs, and regression tests.

## Reproduction Assessment

Yes, by source inspection. A deleted or deactivated user reaches handleDeletedUserSignIn during GitHub OAuth and is rejected before a usable session or CLI token can exist; I did not run a live production OAuth flow.

## Solution Assessment

Unclear. Automatically restoring or recreating accounts during OAuth is not the best fix until maintainers decide the recovery authority, privacy, audit, and token semantics in the draft RFC.
