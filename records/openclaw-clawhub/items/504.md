## Summary

Keep open. Current main still intentionally rejects OAuth sign-in for deleted/deactivated accounts and documents user deletion as irreversible, so the remaining work is a maintainer account-recovery policy decision rather than conservative cleanup.

## What This Changes

The issue asks ClawHub to let users sign in or re-register with GitHub after deleting their ClawHub account, including access to CLI tokens.

## Best Possible Solution

Keep this as the canonical deleted-account GitHub OAuth recovery triage item and implement a maintainer-approved recovery policy consistently across web OAuth, CLI token creation, account deletion, identity binding, operator flows, user-facing errors, docs, and regression tests.

## Reproduction Assessment

Yes. The source-level path is clear: deleteAccount sets deactivatedAt, then the GitHub OAuth callback calls handleDeletedUserSignIn and rejects that user, with tests asserting the blocked behavior.

## Solution Assessment

No. Automatic re-registration, hard cleanup, or identity unlinking is not the best fix until maintainers approve the security, privacy, audit, and support semantics for deleted-account recovery.
