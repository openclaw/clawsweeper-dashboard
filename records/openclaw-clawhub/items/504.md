## Summary

Keep open. Current main still deliberately blocks OAuth sign-in for deleted/deactivated accounts, the discussion shows multiple affected users, and ClawHub cleanup policy keeps issues open; the remaining work is security-sensitive account recovery policy rather than conservative cleanup.

## What This Changes

The issue requests that GitHub OAuth sign-in recover, relink, or recreate access after a user deletes their ClawHub account.

## Best Possible Solution

Keep this as the canonical deleted-account GitHub OAuth recovery triage item and implement a maintainer-approved policy consistently across web OAuth, CLI token creation, account deletion, auth account binding, operator flows, user-facing errors, docs, and regression tests.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: deleteAccount sets deactivatedAt, then the OAuth callback calls handleDeletedUserSignIn and throws for that user; the current auth tests explicitly assert the blocked behavior.

## Solution Assessment

No. A quick purge, automatic re-registration, or cleanup close is not the best path until maintainers approve the security and privacy semantics for deletion, identity rebinding, and recovery.
