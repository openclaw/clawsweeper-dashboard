## Summary

Keep open. Current main still intentionally blocks OAuth reauth for deleted/deactivated accounts, and resolving this would require a security/privacy product decision about account recovery, identity unlinking, or operator cleanup rather than conservative issue cleanup.

## What This Changes

Request to let a GitHub-backed ClawHub account be restored, re-linked, or cleanly re-registered after user-initiated account deletion instead of staying logged out after OAuth.

## Best Possible Solution

Keep this as the canonical deleted-account GitHub OAuth recovery item and implement a maintainer-approved policy consistently across OAuth, deletion, auth account binding, operator/admin flows, user-facing errors, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, the OAuth callback calls handleDeletedUserSignIn, and that throws before web or CLI token flows can observe an authenticated user.

## Solution Assessment

No for automation. Recreating, relinking, or unlinking a deleted GitHub-backed account changes the current irreversible-deletion contract and should follow a maintainer-approved security and privacy policy.
