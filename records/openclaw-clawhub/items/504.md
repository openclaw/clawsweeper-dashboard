## Summary

Keep open. Current main still treats user-initiated account deletion as permanent and rejects GitHub OAuth reauthentication for deactivated or legacy-deleted users; the remaining work is security- and privacy-sensitive account recovery policy, and ClawHub cleanup policy keeps issues open.

## What This Changes

The issue requests that users who deleted their ClawHub account can sign back in with the same GitHub identity or otherwise recover/unlink the account instead of remaining in an OAuth sign-in loop.

## Best Possible Solution

Keep this as the canonical deleted-account GitHub OAuth recovery item and implement a maintainer-approved policy consistently across OAuth, CLI token creation, account deletion, auth account binding, operator/admin flows, user-facing errors, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, the OAuth callback calls handleDeletedUserSignIn, and that throws before web or CLI token flows can observe an authenticated user.

## Solution Assessment

No for an immediate automated code change. Recreating, relinking, or unlinking a deleted GitHub-backed account changes the current irreversible-deletion contract and needs a maintainer-approved security and privacy policy first.
