## Summary

Keep open. Current main still treats user-initiated account deletion as permanent and blocks GitHub reauth for deleted/deactivated users, while the issue discussion and related open reports show multiple affected users asking for recovery, unlinking, or clean re-registration.

## What This Changes

The issue requests that GitHub OAuth sign-in recreate, restore, or re-link a ClawHub account after user-initiated account deletion instead of leaving the user logged out.

## Best Possible Solution

Keep this as the canonical deleted-account GitHub OAuth recovery item and implement a maintainer-approved policy consistently across OAuth, deletion, identity binding, admin/operator flows, user-facing errors, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, the OAuth callback calls handleDeletedUserSignIn, and that throws before the web or CLI token flow can observe an authenticated user.

## Solution Assessment

No for an automated repair. The requested recreate/re-link behavior changes the current irreversible-deletion contract, so maintainers should first choose recovery, unlinking, or permanent-deletion semantics.
