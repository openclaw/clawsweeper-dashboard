## Summary

Keep open. Current main still deliberately blocks deleted/deactivated GitHub-backed accounts from signing in again, and this ClawHub issue should remain open for maintainer policy rather than cleanup closure.

## What This Changes

The issue requests recovery or clean re-registration for GitHub OAuth users whose ClawHub account was deleted and now cannot establish a web or CLI session.

## Best Possible Solution

Keep this as the canonical deleted-account GitHub OAuth recovery policy item and implement a maintainer-approved path consistently across OAuth, CLI token creation, deletion, auth account binding, operator tooling, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, afterUserCreatedOrUpdated calls handleDeletedUserSignIn, and the handler throws before web or CLI token flows can observe an authenticated active user.

## Solution Assessment

No for an immediate automated fix. Recreating, relinking, restoring, or unlinking a deleted GitHub-backed account changes the current irreversible-deletion contract and needs a maintainer-approved security and privacy policy first.
