## Summary

Keep open. Current main still treats deleted/deactivated accounts as unable to sign in, the issue has multiple affected-user confirmations, and current RFC text still leaves account recovery after owner-initiated deletion as an unresolved trust/product decision.

## What This Changes

The issue asks ClawHub to let a user who deleted their ClawHub account sign in again with the same GitHub identity instead of remaining logged out after OAuth.

## Best Possible Solution

Resolve the account-recovery policy through the RFC path, then implement the chosen restore, unlink/re-register, or operator-cleanup behavior consistently across OAuth, deletion, auth bindings, tokens, docs/specs, and regression tests.

## Reproduction Assessment

Yes, by source inspection. A deleted or deactivated user reaches handleDeletedUserSignIn during GitHub OAuth and is rejected before a usable session exists; I did not run a live production OAuth flow.

## Solution Assessment

Unclear. Automatically restoring or recreating accounts during OAuth is not the best fix until maintainers decide the recovery authority, privacy, audit, and token semantics documented as open in the RFC.
