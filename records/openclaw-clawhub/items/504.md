## Summary

Keep open. Current main deliberately blocks OAuth sign-in for deleted/deactivated accounts and now surfaces blocked-account errors, while the issue and related reports show a real account-recovery/support problem that needs maintainer policy and production-data handling rather than cleanup closure.

## What This Changes

The issue asks ClawHub to allow GitHub login or clean re-registration after a user deletes their ClawHub account instead of returning from OAuth still logged out.

## Best Possible Solution

Keep this as the canonical deleted-account OAuth recovery item and implement a maintainer-approved recovery or unlinking policy across web auth, deletion, CLI tokens, operator flow, docs/specs, and regression tests.

## Reproduction Assessment

Yes, from source inspection rather than a live production account. Deleting an account sets `deactivatedAt`/`purgedAt`, the OAuth callback rejects deleted/deactivated users, and the CLI token route remains blocked unless an active session exists.

## Solution Assessment

No for an automated source-only repair. Recreating or relinking an account on OAuth is not the safest maintainable fix until maintainers define the deletion recovery policy and the required audit/security guarantees.
