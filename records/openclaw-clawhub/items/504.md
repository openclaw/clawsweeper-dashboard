## Summary

Keep open. Current main still blocks deleted/deactivated accounts from GitHub OAuth reauth, and the repeated reports show a real account-recovery need; for ClawHub issues this should stay open for maintainer policy rather than cleanup closure.

## What This Changes

The issue requests that GitHub OAuth after ClawHub account deletion recreate or re-link the account so users can regain web and CLI-token access.

## Best Possible Solution

Keep this as the canonical deleted-account GitHub OAuth recovery item and implement a maintainer-approved policy consistently across OAuth, CLI token creation, deletion, identity binding, operator tooling, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, afterUserCreatedOrUpdated calls handleDeletedUserSignIn, and the handler throws before web or CLI token flows can observe an authenticated active user.

## Solution Assessment

No for an immediate automated fix. Recreating, relinking, restoring, or unlinking a deleted GitHub-backed account changes the current irreversible-deletion contract and needs a maintainer-approved security and privacy policy first.
