## Summary

Keep open. Current main still blocks deleted/deactivated GitHub-backed accounts during OAuth reauth, and the discussion plus related open reports show this is an active account recovery problem rather than cleanup-ready stale noise.

## What This Changes

Allow users who deleted their ClawHub account to sign in again with the same GitHub identity, either by restoring, relinking, or cleanly re-registering the account.

## Best Possible Solution

Keep this as the canonical deleted-account GitHub OAuth recovery item and implement a maintainer-approved policy across web OAuth, CLI token creation, account deletion, auth account binding, audit/operator flows, user-facing errors, docs, and regression tests.

## Reproduction Assessment

Yes. The source-level path is high confidence: `deleteAccount` sets `deactivatedAt`, then GitHub OAuth calls `handleDeletedUserSignIn`, which throws before the app or CLI token flow can enter an authenticated state.

## Solution Assessment

No. Automatic re-registration or ad hoc record purging is not the best fix until maintainers approve the security, privacy, and audit semantics for deletion, identity rebinding, and recovery.
