## Summary

Keep open. Current main still intentionally blocks GitHub sign-in for deleted/deactivated accounts and only improves the error surface; the remaining request is a security- and privacy-sensitive account recovery policy decision, with multiple affected-user confirmations and related open reports.

## What This Changes

Request a way for users who deleted their ClawHub account to sign in again with the same GitHub account, either by recovery, relinking, or clean re-registration.

## Best Possible Solution

Keep this as the canonical deleted-account OAuth recovery triage path, decide the policy through the RFC/security review, then implement the chosen restore, unlink/re-register, or operator-cleanup behavior with docs and regression tests.

## Reproduction Assessment

Yes, by source inspection on current main: account deletion sets deactivatedAt/purgedAt, and the OAuth callback rejects deactivated or legacy deleted users before a usable session exists. I did not run a live production OAuth flow or inspect the reporter’s account row.

## Solution Assessment

No for an automatic restore or one-off auth binding deletion as a cleanup action. The maintainable solution needs an explicit account recovery policy covering identity proof, ban separation, audit logs, purged profile data, and operator behavior.
