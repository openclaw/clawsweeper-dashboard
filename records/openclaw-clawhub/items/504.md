## Summary

Keep open. Current main still treats user-initiated deletion as permanent and blocks GitHub reauth for deleted/deactivated users, while this issue and related reports show multiple affected users asking for recovery, unlinking, or clean re-registration.

## What This Changes

The issue requests that users who deleted a ClawHub account can sign in again with GitHub and regain access to web and CLI token flows.

## Best Possible Solution

Define and implement a maintainer-approved deleted-account recovery or unlink policy across OAuth, deletion, CLI token creation, auth binding, operator workflows, docs/specs, and regression tests.

## Reproduction Assessment

Yes, by source inspection: account deletion writes deactivatedAt/purgedAt, OAuth re-entry calls handleDeletedUserSignIn, and active-user helpers reject deleted or deactivated users before web or CLI token access can succeed. I did not run a live OAuth flow or inspect production rows.

## Solution Assessment

No, a narrow OAuth-only restore patch is not the best path yet. The maintainable solution needs an explicit recovery, unlink, or re-registration policy that preserves identity proof, deletion privacy, token revocation, audit, and banned-account boundaries.
