## Summary

Keep open. Current main still deliberately blocks deleted/deactivated accounts from OAuth sign-in; source, tests, specs, and repeated affected-user comments show the remaining work is account recovery policy rather than conservative cleanup.

## What This Changes

Request a recovery or clean re-registration path after a user deletes a ClawHub account and tries to sign in again with the same GitHub OAuth identity.

## Best Possible Solution

Adopt a maintainer-approved account recovery policy and implement it consistently across OAuth callback handling, account deletion, auth binding, CLI token creation, operator tooling, docs/specs, and regression tests.

## Reproduction Assessment

Yes, by source inspection. A deleted or deactivated user reaches handleDeletedUserSignIn during GitHub OAuth and is rejected before a usable authenticated session exists; I did not run a live production OAuth flow.

## Solution Assessment

Unclear. Automatically restoring or recreating accounts during OAuth is not the right fix until maintainers decide recovery authority, privacy, audit, and token semantics.
