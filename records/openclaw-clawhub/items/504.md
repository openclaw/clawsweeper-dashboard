## Summary

Keep open. Current main still deliberately blocks OAuth sign-in for deleted/deactivated accounts, and this open ClawHub issue needs maintainer policy for account recovery rather than cleanup closure.

## What This Changes

The issue requests that GitHub OAuth restore, re-link, or recreate a ClawHub account after user-initiated account deletion instead of leaving the user logged out.

## Best Possible Solution

Keep this as the canonical deleted-account GitHub OAuth recovery item and implement one approved policy consistently across OAuth, deletion, identity binding, operator/audit flows, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, and the OAuth callback rejects that user before any logged-in session or CLI token flow can proceed.

## Solution Assessment

No for an automated fix. The requested restore/recreate behavior conflicts with current docs and tests, so maintainers should first choose the account recovery, unlinking, or operator-cleanup semantics.
