## Summary

Keep open. Current main still blocks GitHub sign-in for deleted/deactivated accounts, the live issue has multiple affected-user reports and related open issues, and changing the outcome requires an account-lifecycle/security decision rather than conservative cleanup.

## What This Changes

Request to allow GitHub sign-in or clean re-registration after a ClawHub account has been deleted so the user can access account and CLI token flows again.

## Best Possible Solution

Keep this as the canonical deleted-account GitHub OAuth recovery item and implement a maintainer-approved policy across OAuth, deletion state, auth account bindings, CLI tokens, operator/audit flows, docs, and regression tests.

## Reproduction Assessment

Yes, by source inspection: deleteAccount writes deactivatedAt/purgedAt, afterUserCreatedOrUpdated calls handleDeletedUserSignIn, and that helper rejects deactivated or legacy deleted users before a usable session exists. I did not inspect the affected production account rows or run a live OAuth flow.

## Solution Assessment

No for an automatic restore, purge, or auth binding deletion as cleanup. The maintainable solution needs explicit account recovery policy covering identity proof, deletion privacy, ban separation, auditability, and production support behavior.
