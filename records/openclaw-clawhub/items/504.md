## Summary

Keep open. This is an open ClawHub issue, and current main still treats deleted/deactivated accounts as unable to complete normal GitHub sign-in while multiple users report the same production symptom; the remaining work is an account recovery policy decision, not conservative cleanup.

## What This Changes

Users who deleted a ClawHub account want GitHub OAuth sign-in to restore, relink, or recreate access instead of returning to a logged-out loop.

## Best Possible Solution

Keep this as the canonical deleted-account GitHub OAuth recovery item and implement a maintainer-approved policy across OAuth, CLI token access, deletion state, auth account bindings, operator/audit flows, docs, and regression tests.

## Reproduction Assessment

Yes for the current blocked sign-in path by source inspection: account deletion records deactivatedAt/purgedAt, and the GitHub OAuth callback rejects deactivated or legacy deleted users before a usable session exists. I did not run a live production OAuth flow or inspect reporter account rows.

## Solution Assessment

No for an automatic one-off restore or auth binding deletion as cleanup. The maintainable solution needs an explicit account recovery policy covering identity proof, ban separation, auditability, purged profile data, and operator behavior.
