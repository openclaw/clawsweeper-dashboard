## Summary

Keep open. Current main still blocks OAuth sign-in for deleted/deactivated accounts by design, and the issue discussion shows multiple affected users still need a maintainer-approved account recovery path rather than cleanup closure.

## What This Changes

Allow or guide recovery, unlinking, or re-registration for GitHub-backed users after deleting their ClawHub account.

## Best Possible Solution

Keep this as the canonical deleted-account GitHub OAuth recovery triage item and implement one approved policy consistently across web OAuth, CLI token creation, account deletion, auth account binding, operator flows, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, the OAuth callback calls handleDeletedUserSignIn, and tests assert deactivated or legacy deleted users are rejected.

## Solution Assessment

No for an automated code fix. The requested recovery/re-registration behavior conflicts with current docs and tests, so maintainers should choose restore, unlink-and-recreate, or operator cleanup semantics before implementation.
