## Summary

Keep open. Current main still deliberately blocks deleted-account GitHub reauth, the behavior is codified in docs and tests, and ClawHub policy keeps issues open; the remaining work is a security-sensitive account recovery policy rather than cleanup.

## What This Changes

The issue requests GitHub OAuth recovery or clean re-registration after a ClawHub account has been deleted.

## Best Possible Solution

Keep this as the canonical deleted-account OAuth recovery item and implement a maintainer-approved policy consistently across OAuth, deletion, identity binding, admin/operator flows, user-facing errors, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `deleteAccount` sets `deactivatedAt`/`purgedAt`, and the OAuth callback rejects that user before a web session or CLI token flow can proceed.

## Solution Assessment

No for an automated repair. The requested recreate/re-link behavior conflicts with current irreversible-deletion docs and tests, so maintainers should first choose the recovery or unlinking semantics.
