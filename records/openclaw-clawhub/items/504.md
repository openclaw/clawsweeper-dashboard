## Summary

Keep open. Current main and v0.16.0 still intentionally block GitHub OAuth reauth for deleted/deactivated accounts, the live discussion has multiple same-symptom reports, and ClawHub cleanup policy keeps issues open unless a PR-only close reason applies.

## What This Changes

The issue requests a way for users who deleted their ClawHub account to sign in again with GitHub and regain web and CLI token access.

## Best Possible Solution

Keep this as the canonical deleted-account GitHub OAuth recovery triage item and define a maintainer-approved recovery, unlinking, or re-registration policy across OAuth, deletion, tokens, audit/operator flows, specs, docs, and tests.

## Reproduction Assessment

Yes, by source inspection: deletion sets deactivatedAt/purgedAt, OAuth re-entry calls handleDeletedUserSignIn, and active-user helpers reject deleted or deactivated users before web or CLI token access can succeed. I did not run a live OAuth flow or inspect production rows.

## Solution Assessment

No, a narrow OAuth-only recreate patch is not the best path yet. The maintainable solution needs an explicit recovery, unlink, or re-registration policy that preserves identity proof, deletion privacy, token revocation, audit, and banned-account boundaries.
