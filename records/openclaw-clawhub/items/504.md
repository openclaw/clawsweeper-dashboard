## Summary

Keep open. Current main still intentionally rejects OAuth reauthentication for deleted or deactivated ClawHub accounts, and the issue discussion shows multiple users hitting the same post-deletion login loop. Repository policy also says ClawHub issues should stay open, and the remaining work is security-sensitive account recovery policy rather than conservative cleanup.

## What This Changes

The issue asks ClawHub to allow a GitHub-backed user to sign in, recover, unlink, or re-register after deleting their ClawHub account instead of returning to the signed-out state.

## Best Possible Solution

Keep this as the canonical deleted-account GitHub OAuth recovery triage item. The desired end state is a maintainer-approved account recovery policy implemented consistently across web OAuth, CLI token creation, account deletion, auth account binding or unlinking, audit/operator flows, user-facing errors, docs, and regression tests.
