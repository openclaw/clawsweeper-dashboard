## Summary

Keep open. Current main still blocks GitHub sign-in for deleted/deactivated accounts, and resolving this would change account recovery, provider identity binding, and deletion/privacy policy rather than provide a conservative cleanup close.

## What This Changes

The issue requests that GitHub OAuth recover, relink, or recreate access after a user deletes their ClawHub account instead of leaving them signed out.

## Best Possible Solution

Keep this as the canonical deleted-account GitHub OAuth recovery item and implement a maintainer-approved policy consistently across OAuth, deletion, auth account binding, operator/admin flows, user-facing errors, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, the OAuth callback calls handleDeletedUserSignIn, and that throws before web or CLI token flows can observe an authenticated user.

## Solution Assessment

No for automation. The requested recreate/relink behavior changes the current irreversible-deletion contract and needs a maintainer-approved security and privacy policy before implementation.
