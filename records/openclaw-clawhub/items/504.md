## Summary

Keep open. Current main still deliberately rejects OAuth sign-in for deleted/deactivated accounts; the only current-main improvement I found is better user-facing error surfacing, not recovery or re-registration. ClawHub policy also keeps issues open in this cleanup workflow, and the remaining work is a security/privacy product decision rather than conservative auto-close cleanup.

## What This Changes

Allow a user who deleted their ClawHub account to sign in again with GitHub, restore or recreate the account, and regain CLI token access instead of staying logged out.

## Best Possible Solution

Keep this as the canonical deleted-account GitHub OAuth recovery item until maintainers choose and document a recovery/re-registration policy, then implement it consistently across Convex auth, account deletion, auth account binding, CLI token creation, operator flows, user-facing errors, specs/docs, and regression tests.

## Reproduction Assessment

Yes, the remaining account-recovery problem is source-reproducible: `deleteAccount` sets permanent deactivation state, OAuth sign-in rejects that state, and CLI token creation requires successful auth. I did not run a live GitHub OAuth flow in this read-only review.

## Solution Assessment

No automated repair is the best next step yet. Recreating, restoring, or unlinking deleted accounts requires an explicit maintainer security/privacy policy before code should change.
