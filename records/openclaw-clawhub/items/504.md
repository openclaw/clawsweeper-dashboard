## Summary

Keep open. Current main still treats deleted accounts as permanently deactivated, and the issue discussion plus related open reports show an unresolved account-recovery/support problem that needs maintainer security and product policy.

## What This Changes

Users who deleted a ClawHub account linked to GitHub report that OAuth completes but leaves them logged out, blocking account access, CLI tokens, and re-registration.

## Best Possible Solution

Define and implement a maintainer-approved deleted-account recovery or unlink policy with audit coverage across OAuth, account deletion, CLI auth, authAccounts, docs/specs, and operator support.

## Reproduction Assessment

Yes, from source inspection rather than a live production account. `deleteAccount` sets `deactivatedAt`/`purgedAt`, `handleDeletedUserSignIn` rejects deleted/deactivated users, and CLI token creation requires an authenticated `me`.

## Solution Assessment

No for an automated source-only repair. Recreating, restoring, or unlinking an account during OAuth is not the safest solution until maintainers define the deletion recovery policy and required audit/security guarantees.
