## Summary

Keep open. Current main still intentionally blocks OAuth reauth for deleted/deactivated accounts, while this issue and related reports show affected users asking for restore, unlinking, or clean re-registration; for ClawHub issues, this is not a conservative auto-close candidate.

## What This Changes

The issue requests that a user who deleted their ClawHub account can sign back in with the same GitHub identity and regain access to CLI token creation instead of remaining logged out.

## Best Possible Solution

Keep this as the canonical deleted-account GitHub OAuth recovery item, decide the recovery policy, then implement the chosen restore, unlink/re-register, or operator-cleanup path consistently across OAuth, CLI tokens, deletion, docs/specs, audit, and tests.

## Reproduction Assessment

Yes, by source inspection. A deleted or deactivated user reaches handleDeletedUserSignIn during GitHub OAuth and is rejected before a usable authenticated session exists; I did not run a live production OAuth flow.

## Solution Assessment

Unclear. Automatically restoring or recreating accounts during OAuth is not the best fix until maintainers decide recovery authority, privacy, audit, token revocation, and operator-cleanup semantics.
