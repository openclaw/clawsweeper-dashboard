## Summary

Keep open. Current main still intentionally blocks GitHub reauth for deleted/deactivated accounts, the discussion shows multiple affected users, and ClawHub cleanup policy does not auto-close issues; the remaining work is a security-sensitive account recovery policy decision.

## What This Changes

The issue requests a way for users who deleted their ClawHub account to sign in again with the same GitHub identity and access web or CLI token flows.

## Best Possible Solution

Keep this as the canonical deleted-account GitHub OAuth recovery policy item and implement a maintainer-approved path consistently across web OAuth, CLI token creation, account deletion, identity binding, operator flows, user-facing errors, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount writes deactivatedAt/purgedAt, afterUserCreatedOrUpdated calls handleDeletedUserSignIn, and the handler throws before web or CLI token flows can observe an authenticated active user.

## Solution Assessment

No for an immediate automated fix. Restoring, recreating, or unlinking a deleted GitHub-backed account changes the current irreversible-deletion contract and needs a maintainer-approved security and privacy policy first.
