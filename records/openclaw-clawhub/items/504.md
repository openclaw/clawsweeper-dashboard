## Summary

Keep open. Current main still rejects deleted/deactivated users during GitHub OAuth instead of recreating or relinking an account, and the related reports show this remains an active account-recovery policy problem rather than a cleanup-close candidate.

## What This Changes

The issue requests a way to sign in again with GitHub after deleting a ClawHub account, instead of returning to a logged-out/sign-in state.

## Best Possible Solution

Keep this as the canonical deleted-account GitHub OAuth recovery triage item and implement a maintainer-approved recovery/unlink/operator policy consistently across OAuth, deletion, CLI tokens, audit behavior, docs, and tests.

## Reproduction Assessment

Yes, from source inspection: a deleted or deactivated user reaches handleDeletedUserSignIn during OAuth and is rejected instead of recreated. I did not run a live OAuth flow, so the exact browser button-loop presentation was not runtime-reproduced.

## Solution Assessment

No, not as an automatic recreate/relink change. The safest solution is a maintainer-approved account recovery policy because the requested behavior changes deletion, identity binding, token, privacy, and audit semantics.
