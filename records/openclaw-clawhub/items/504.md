## Summary

Keep open. Current main still treats user-initiated account deletion as permanent and blocks GitHub OAuth reauth for deleted/deactivated users, while the thread and related reports show multiple affected users asking for recovery or clean re-registration.

## What This Changes

Allow a user who deleted a ClawHub account to sign back in with GitHub and regain or recreate account access, including CLI token access.

## Best Possible Solution

Keep this as the canonical deleted-account GitHub OAuth recovery item and implement one maintainer-approved policy consistently across OAuth, deletion, identity binding, operator/audit flows, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, and the OAuth callback rejects that user before a logged-in session or CLI token flow can proceed.

## Solution Assessment

No for an automated fix. The requested restore/recreate behavior conflicts with current docs and tests, so maintainers should first choose the account recovery, unlinking, or operator-cleanup semantics.
