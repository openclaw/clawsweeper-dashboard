## Summary

Keep open. The live ClawHub API still reproduces the stale SKILL.md mismatch, and current main appears to serve the latest version row's stored blob, so the next useful step is maintainer production publish/upload tracing rather than conservative cleanup closure.

## What This Changes

The issue reports that a skill page continues to render the original SKILL.md after later successful publishes with updated content.

## Best Possible Solution

Trace production publish payload hashes, upload logs, storage IDs, and skillVersions rows for the affected publishes; repair bad rows if submitted bytes were correct and add a regression proving changed SKILL.md bytes produce distinct stored hashes and rendered content.

## Solution Assessment
