## What This Changes

The issue asks ClawHub to show the SKILL.md and derived metadata from the latest published skill version after repeated CLI publishes and delete/re-publish attempts.

## Best Possible Solution

Trace the affected publish payloads and production storage rows, repair the bad version file records if needed, and add a regression test or publish-time guard at the boundary that allowed stale SKILL.md bytes to be stored for new versions.

Yes for the visible symptom: public API calls show latest 0.4.0 while all version rows serve the same SKILL.md sha and Gmail-only content. The underlying publish-path cause is not yet reproducible from source alone without the reporter's local payloads or production upload logs.

## Solution Assessment
