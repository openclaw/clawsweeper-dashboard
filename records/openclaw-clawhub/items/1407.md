## Summary

Keep open. This is an open ClawHub issue, and the current live data plus maintainer production follow-up still show a real stale-content problem at the publish/upload boundary rather than an implemented or cache-only issue.

## What This Changes

The issue requests that skill pages show the SKILL.md content from the latest published skill version instead of stale first-publish content.

## Best Possible Solution

Trace the CLI multipart payload and publish mutation logs for affected publishes, compare uploaded SKILL.md hashes with stored skillVersions.files rows, repair bad data if needed, and add regression coverage at the failing boundary.
