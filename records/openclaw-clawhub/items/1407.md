## What This Changes

The issue reports that publishing later versions of `deepu105/auth0-token-vault` leaves the skill page showing the first `SKILL.md` content.

## Best Possible Solution

Trace the publish payload/storage hashes for the affected versions, add validation or instrumentation that changed `SKILL.md` bytes produce new stored hashes and rendered content, then repair affected production rows if the submitted bytes were correct.
