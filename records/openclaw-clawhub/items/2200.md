## Summary

Keep open. Current main still has a source-visible deleted-slug mismatch: availability and publish handle expired owner-unpublish reservations, but rename rejects the soft-deleted `by_slug` row first and merge refuses a soft-deleted source; cross-publisher reclaim also needs security/product policy.

## What This Changes

The issue requests a way to reuse, release, merge, or reclaim a soft-deleted skill slug across owners when the caller has appropriate authority.

## Best Possible Solution

Keep the 30-day owner-delete reservation, make post-TTL rename/publish/availability behavior consistent, and require audited authority checks for any in-window or cross-publisher reclaim path.

## Reproduction Assessment

Yes, from source inspection: after an owner-unpublished slug reservation expires, availability and publish treat the slug as claimable, but rename still rejects the soft-deleted `skills.by_slug` row and merge rejects soft-deleted sources. I did not run a live Convex reproduction or inspect production data.

## Solution Assessment

Unclear for the full request: aligning rename with the existing expired-reservation release contract is the narrow fix, but in-window or cross-publisher reclaim needs maintainer security/product judgment.
