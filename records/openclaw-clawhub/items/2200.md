## Summary

Current production code provides the safe workflow requested: authorized users can restore a deleted skill, merge skills while independently authorized over both publishers, and rely on bounded reservation expiry for eligible unpublished deletions. The original cross-owner merge limitation is no longer present.

## What This Changes

The issue requests a safe way to reuse a deleted skill slug or transfer or merge it across authorized owners.

## Best Possible Solution

Keep the current restore-then-authorized-merge workflow and bounded reservation expiry, preserving staff-only recovery for banned, moderated, or disputed namespaces.

## Reproduction Assessment

No on current main: merge authorizes both owners independently, and deleted owned skills have an undelete path before merge. Eligible unpublished deletions also release their slug after a bounded reservation.

## Solution Assessment

Yes; restore-then-merge plus bounded reservation expiry solves legitimate recovery without globally freeing protected, banned, or moderated namespaces.
