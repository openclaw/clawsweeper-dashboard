## Summary

Keep open: current main still lets availability and publish reclaim expired owner-deleted slugs, but rename checks the soft-deleted `skills.by_slug` row first and merge rejects soft-deleted sources, while cross-publisher self-service reclaim remains a security/product decision.

## What This Changes

The issue asks for deleted skill slugs to become reusable or reclaimable across personal and organization publishers when the actor has permission for both.

## Best Possible Solution

Keep the 30-day owner-delete reservation, align rename with publish after expiry, and add any in-window or cross-publisher reclaim path only with explicit audited authority checks.

## Reproduction Assessment

Yes, from source inspection: after an owner-deleted skill's reservation expires, availability and publish can treat the slug as claimable, but rename still rejects the soft-deleted row first and merge rejects a soft-deleted source. I did not run a live Convex reproduction or inspect production data.

## Solution Assessment

Unclear for the full request: immediate reuse or cross-publisher reclaim before the 30-day TTL needs maintainer product/security judgment. The narrow maintainable fix is to align rename with the existing expired owner-unpublish release contract and improve guidance while a slug is still legitimately reserved.
