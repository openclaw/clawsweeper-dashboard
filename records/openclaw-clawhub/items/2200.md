## Summary

Keep open: current main intentionally reserves owner-unpublished skill slugs for 30 days, and the requested immediate reuse or cross-owner reclaim would change an ownership-sensitive policy boundary rather than clean up an already-fixed issue.

## What This Changes

The issue requests a way to reuse or release a deleted skill slug, including cross-owner reclaim or merge when the actor controls both owners.

## Best Possible Solution

Define an audited owner-authorized release or transfer path for deleted skill slugs, or document that the existing 30-day reservation and maintainer support path is the intended policy.

## Reproduction Assessment

Yes, from source inspection for the general path: owner deletion creates a reserved hidden row, rename checks the global slug index without skipping soft-deleted rows, and merge rejects soft-deleted skills. I did not exercise a live Convex database in this read-only review.

## Solution Assessment

No as a direct behavior change yet; simply bypassing deleted-slug reservations would be too broad. The maintainable solution needs a maintainer-approved release or transfer policy with ownership verification and audit logging.
