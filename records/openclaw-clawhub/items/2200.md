## Summary

Keep open: current main still has the source-level mismatch where expired owner-deleted slugs can be shown as claimable and publish can lazily release them, but rename rejects the soft-deleted row first and merge cannot use a soft-deleted source. The existing admin reclaim path and the resolved support case at https://github.com/openclaw/clawhub/issues/1770 help with staff-mediated cleanup, but they do not satisfy the requested self-service personal/org reclaim policy.

## What This Changes

The issue requests a way to reuse, release, merge, or reclaim a deleted skill slug across publishers when the actor has authority over both owners.

## Best Possible Solution

Keep the 30-day owner-delete reservation, align rename with publish after expiry, and add any in-window or cross-publisher reclaim path only with explicit audited authority checks.

## Reproduction Assessment

Yes, from source inspection: after an owner-deleted skill's reservation expires, availability and publish can treat the slug as claimable, but rename rejects the soft-deleted row first and merge rejects a soft-deleted source. I did not run a live Convex reproduction or inspect production rows.

## Solution Assessment

Unclear for the full request: immediate reuse or cross-publisher reclaim before the 30-day TTL needs maintainer product and security judgment. The narrow maintainable fix is to align rename with the existing expired owner-unpublish release contract and improve guidance while a slug is still legitimately reserved.
