## Summary

Keep open: current main still has a source-level mismatch where availability and publish can reclaim an expired owner-deleted slug, but rename rejects the soft-deleted row first and merge rejects soft-deleted sources; cross-publisher reclaim also needs product/security policy.

## What This Changes

The issue requests a path to reuse, release, merge, or reclaim a soft-deleted skill slug across personal and organization publishers when the actor has authority for both.

## Best Possible Solution

Keep the 30-day owner-delete reservation, align rename with publish after expiry, and add any in-window or cross-publisher reclaim path only with explicit audited authority checks.

## Reproduction Assessment

Yes, at source level: after an owner-deleted skill's reservation expires, availability and publish can treat the slug as claimable, but rename rejects the soft-deleted row first and merge rejects a soft-deleted source. I did not run a live Convex reproduction or inspect production data.

## Solution Assessment

Unclear for the full request: immediate reuse or cross-publisher reclaim before the 30-day TTL needs maintainer product/security judgment. The narrow maintainable fix is to align rename with the existing expired owner-unpublish release contract and improve guidance while a slug is still legitimately reserved.
