## Summary

Keep open. Current main still has a source-visible deleted-skill slug collision path, and the requested cross-owner reclaim/release behavior needs maintainer policy because it affects namespace ownership.

## What This Changes

The issue asks for deleted skill slugs to become reusable or reclaimable across owners when the actor has authority for both entities.

## Best Possible Solution

Keep the 30-day owner-delete reservation, make rename/publish/availability consistent after expiry, and require audited authority checks or a staff flow for any in-window or cross-publisher reclaim.

## Reproduction Assessment

Yes, from source inspection: a soft-deleted skill can remain in `skills.by_slug`, owner rename rejects that row before lazy release, and merge rejects deleted sources. I did not run a live Convex reproduction or inspect production data.

## Solution Assessment

Unclear for the full request because cross-owner reclaim before expiry needs maintainer policy. The narrowest maintainable fix is to align rename with the documented expired-reservation release path while keeping protected deleted slugs reserved.
