## Summary

Current main resolves the reported permanent slug lock: owner-deleted skills receive a 30-day reservation, then the hidden record is moved to a non-public internal slug so another publisher can claim the original slug; this shipped in v0.20.2.

## What This Changes

Allow a deleted skill slug to become reusable or reclaimable across owners without bypassing ownership controls.

## Best Possible Solution

Keep the shipped time-bounded reservation and internal tombstone rename, which prevents permanent slug squatting while preserving moderation and security holds.

## Reproduction Assessment

Yes. The original source behavior kept a soft-deleted row occupying the public slug, and the regression tests now directly exercise reservation expiry followed by a new claim.

## Solution Assessment

Yes. A time-bounded reservation followed by an internal tombstone slug is narrower and safer than cross-owner merging because it frees the public identifier without transferring the deleted skill’s ownership or history.
