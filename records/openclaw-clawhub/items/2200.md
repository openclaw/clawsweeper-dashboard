## Summary

Keep open. Current main still has a source-reproducible recovery gap for managed soft-deleted skill slugs, and ClawHub cleanup policy does not auto-close issues.

## What This Changes

Request a recovery path to reuse, release, or merge a soft-deleted skill slug across personal and organization owners when the actor manages both entities.

## Best Possible Solution

Define an audited owner recovery flow that lets an authorized actor restore, transfer, merge, or explicitly release a deleted slug across publishers they administer while preserving the 30-day reservation and alias-squatting protections.

## Reproduction Assessment

Yes for the source-level path: current main rejects rename against a managed soft-deleted slug, points the user to merge, and then merge requires live source and target rows that the UI does not list. I did not inspect or mutate production data for a specific slug.

## Solution Assessment

Unclear as an implementation direction: immediate deleted-slug release is not obviously the safest fix because current docs intentionally reserve owner-deleted slugs for 30 days. The maintainable path is an explicit, audited recovery flow with product-approved ownership and TTL semantics.
