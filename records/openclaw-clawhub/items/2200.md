## Summary

Keep open: current main still has the mismatch reported here. Expired owner-deleted slugs can be shown as claimable and publish can lazily release them, but rename rejects the soft-deleted row first and merge refuses a soft-deleted source; the personal-to-organization reclaim path also needs explicit ownership/security policy.

## What This Changes

The issue requests a way to reuse or release a deleted skill slug, including reclaiming or merging it across personal and organization publishers when the actor controls both.

## Best Possible Solution

Keep the 30-day owner-delete reservation, align rename with the expired lazy-release contract, and add any in-window or cross-publisher reclaim path only with explicit audited authority checks for both publishers.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: after an owner-deleted slug reservation expires, availability and publish can treat it as claimable, but rename rejects the soft-deleted row first and merge rejects a soft-deleted source. I did not run a live Convex reproduction or inspect production data.

## Solution Assessment

Unclear for the full request: immediate reuse or cross-publisher reclaim before the 30-day TTL needs maintainer product/security judgment. The narrow maintainable fix is to align rename with the existing expired owner-unpublish release contract and improve guidance while a slug is still legitimately reserved.
