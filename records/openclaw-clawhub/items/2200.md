## Summary

Keep open. Current main still has a source-visible gap: expired owner-unpublish slugs can be shown as available and released during publish, but rename rejects the soft-deleted row first and merge refuses deleted sources, so the reported workflow still needs a maintainer product/security decision.

## What This Changes

The issue requests a way to reuse, release, merge, or reclaim a deleted skill slug across owners when the actor has authority over both entities.

## Best Possible Solution

Keep the 30-day owner-delete reservation, align rename with the existing post-TTL publish/availability release contract, and require audited publisher-authority checks or a staff-only flow for in-window and cross-publisher reclaim.

## Reproduction Assessment

Yes, from source inspection: a soft-deleted owner-unpublished row remains indexed by slug, rename rejects it as an existing skill, and merge rejects it as a deleted source. I did not run a live Convex reproduction or inspect production rows.

## Solution Assessment

Unclear for the full request until maintainers choose the cross-owner reclaim policy. The narrow maintainable path is to align rename with the existing post-TTL release contract while preserving audited authority checks for any cross-publisher reclaim.
