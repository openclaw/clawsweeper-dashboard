## Summary

Keep open. Current main still has the deleted-slug lifecycle gap: availability and publish know about post-expiry owner-unpublish release, but owner rename rejects the soft-deleted slug row and merge cannot consume deleted rows; the cross-owner reclaim part needs maintainer policy/security judgment.

## What This Changes

Allow a deleted skill slug to be reused, released, merged, or reclaimed across owners when the actor has authority over both entities.

## Best Possible Solution

Define the allowed same-actor cross-owner reclaim rules, then align rename, merge, publish, availability, UI copy, docs, and tests around one deleted-slug lifecycle.

## Reproduction Assessment

Yes for the source-level mismatch: a rename to a slug held by a different soft-deleted skills.by_slug row reaches the collision branch, while merge rejects deleted source or target rows. The exact live personal/org production state from the report was not verified.

## Solution Assessment

Unclear. A narrow code fix could make post-expiry rename reuse the existing lazy-release behavior, but immediate cross-owner reclaim before the 30-day hold expires needs a maintainer product/security decision.
