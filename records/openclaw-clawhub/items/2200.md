## Summary

Keep open: current main still preserves the mismatch reported here. Availability and publish can treat an expired owner-deleted slug as claimable, but rename rejects the soft-deleted row first, and merge refuses a soft-deleted source; the cross-publisher reclaim part still needs ownership and security policy review.

## What This Changes

The issue requests a way to reuse, release, merge, or reclaim a deleted skill slug across owners when the actor has authority over both entities.

## Best Possible Solution

Keep the 30-day owner-delete reservation, align rename with the existing post-expiry publish release contract, and add any in-window or cross-publisher reclaim path only with explicit audited authority checks.
