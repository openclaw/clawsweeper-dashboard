## Summary

Close as implemented: current main provides an owner undelete path, permits merging skills across different publishers when the actor can manage both, and releases owner-deleted slugs after a 30-day reservation. These paths resolve the reported dead end while preserving namespace ownership safeguards.

## What This Changes

Provide a safe way to restore, reuse, or merge a deleted skill slug across publishers controlled by the same user.

## Best Possible Solution

Keep the shipped restore-then-merge workflow for immediate authorized moves and the 30-day lazy release as the fallback for abandoned owner-deleted slugs.

## Reproduction Assessment

No. On current main, an owner can undelete the source, merge it into a skill under another publisher they manage, or allow the deleted slug to become claimable after its reservation expires.

## Solution Assessment

Yes. Restoring before merging preserves skill history, redirects, and authorization checks, while time-bounded release prevents deleted slugs from remaining unavailable forever.
