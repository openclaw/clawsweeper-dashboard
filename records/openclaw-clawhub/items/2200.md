## Summary

Keep open under the ClawHub issue policy. Current main still has a real deleted-slug lifecycle gap: availability and publish handle expired owner-unpublished slugs, but rename rejects any different slug row and merge/listing paths exclude deleted rows; cross-owner reclaim needs maintainer policy and security judgment.

## What This Changes

The issue requests a way to reuse, release, merge, or reclaim a deleted skill slug across owners when the same actor can manage both entities.

## Best Possible Solution

Maintainers should define deleted-slug reclaim semantics across personal and organization publishers, then align rename, merge/reclaim, publish, availability, UI copy, docs, and tests to one audited lifecycle.

## Reproduction Assessment

Yes at source level: rename to a slug held by a different soft-deleted skills.by_slug row reaches the collision branch, while merge rejects deleted source or target rows. The exact live personal/org production state from the report was not verified.

## Solution Assessment

Unclear as an implementation path. A narrow fix could make rename honor the post-expiry release path, but same-actor cross-owner reclaim during the hold needs maintainer policy and security decisions first.
