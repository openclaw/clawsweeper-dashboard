## Summary

Current main only partially addresses the report: owner-deleted slugs become publicly claimable after a 30-day reservation, but there is still no supported immediate reclaim or merge flow for a user who controls both the deleted source publisher and a different destination publisher. The related owner-scoping work remains open and does not make this distinct lifecycle request obsolete.

## What This Changes

Add an authorization-safe way to immediately reclaim or merge a soft-deleted skill slug across personal and organization publishers controlled by the same user.

## Best Possible Solution

Preserve the 30-day public reservation, but add a dedicated restore-and-transfer or reclaim operation that permits immediate movement only after verifying the caller has management authority for both the deleted source publisher and the destination publisher, with personal-to-organization and organization-to-personal coverage.

## Reproduction Assessment

Yes, at source level. Current main deliberately reserves owner-deleted slugs for 30 days and only exposes a privileged reclaim transfer path, so the reported lack of immediate self-service cross-publisher reuse follows directly from the current contract and mutation surface.

## Solution Assessment

No, unconditional release on deletion is not the best solution because it weakens accidental-delete protection. The narrow maintainable solution is an explicit dual-authority restore/transfer operation that leaves public reclaim subject to the existing TTL.
