Keep open: current main still does not provide a self-serve way to reuse or merge a deleted slug across publisher boundaries, and the remaining behavior is security-sensitive ownership policy rather than safe cleanup. The related owner-scoped slug PR is open and unmerged, so it cannot close this issue yet.

## What This Changes

The issue asks for deleted skill slugs to be reusable, released, merged, or reclaimed across owners when the actor has authority over both entities.

## Best Possible Solution

Define the cross-owner deleted-slug reclaim policy first, then implement a scoped backend/UI flow that honors owner authority, 30-day reservations, moderation/security exclusions, and audit provenance.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: availability and publish can treat an expired owner-unpublished slug as claimable, but rename still rejects the existing soft-deleted row and merge rejects soft-deleted source or target rows.

## Solution Assessment

Unclear. A reclaim/release flow is plausible, but the best implementation depends on maintainer policy for cross-owner authorization, TTLs, protected moderation states, and audit records.
