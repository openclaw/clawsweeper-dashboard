## Summary

Keep open. Current main still has a source-visible deleted-slug collision path: availability and publish partially honor post-TTL release, but rename rejects the soft-deleted slug row first and merge refuses deleted sources, so the reported workflow is not fully solved.

## What This Changes

The issue asks for deleted skill slugs to become reusable or reclaimable across personal and organization owners when the actor has authority for both.

## Best Possible Solution

Keep the documented 30-day owner-delete reservation, align rename/publish/availability after expiry, and route in-window or cross-publisher reclaim through audited publisher authority checks.

## Reproduction Assessment

Yes, from source inspection: after an owner-unpublished slug reservation expires, rename still sees the soft-deleted row in skills.by_slug and rejects before any lazy release, while merge rejects a soft-deleted source. I did not run a live Convex reproduction or inspect production data.

## Solution Assessment

Unclear for the full request: aligning rename with the existing expired owner-unpublish release contract is narrow, but immediate reuse or cross-publisher reclaim before the 30-day TTL needs maintainer product/security judgment.
