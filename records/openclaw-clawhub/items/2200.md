## Summary

Keep open. Current main still has a source-visible deleted-slug collision path for owner rename and merge, and the cross-publisher reclaim part needs explicit authorization and audit policy before implementation.

## What This Changes

The issue requests a way to reuse, release, merge, or reclaim a deleted skill slug across publishers when the actor can manage both owners.

## Best Possible Solution

Keep the documented 30-day owner-delete reservation, align rename with publish and availability after expiry, and route in-window or cross-publisher reclaim through audited publisher authority checks.

## Reproduction Assessment

Yes. Source inspection shows that after a manageable skill is soft-deleted, renaming another manageable skill to that slug hits the soft-deleted skills.by_slug row and fails before expired-reservation release logic, while merge rejects a soft-deleted source.

## Solution Assessment

Unclear for the full request. The narrow maintainable fix is to align rename with the existing post-TTL release contract, while immediate or cross-publisher reclaim needs a maintainer product/security decision.
