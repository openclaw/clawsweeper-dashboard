## Summary

Keep open. Current main still has a source-visible deleted-slug collision path, and the remaining behavior crosses publisher ownership boundaries, so this needs maintainer product and security review rather than cleanup closure.

## What This Changes

Request a way to reuse, release, merge, or reclaim a soft-deleted skill slug when the actor has permission over the relevant personal or organization publishers.

## Best Possible Solution

Keep the 30-day owner-delete reservation, then add a publisher-aware audited path that releases expired owner-unpublished slug rows consistently for rename and publish while giving clear guidance for in-window or cross-publisher cases.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: soft-delete one manageable skill, then rename another manageable skill to that slug; rename sees the deleted row in skills.by_slug and rejects before any expired-release handling, while merge rejects soft-deleted sources.

## Solution Assessment

Unclear for the full request. Aligning rename with the existing expired owner-unpublish release contract is narrow, but immediate reuse or cross-publisher reclaim before the TTL expires needs maintainer product/security judgment.
