## Summary

Keep open. ClawHub issue cleanup policy keeps issues out of auto-close scope, and current main still has a real deleted-slug ownership gap that needs maintainer product and security judgment before a self-service fix is safe.

## What This Changes

The issue asks for deleted skill slugs to be reusable or reclaimable across owners when the same user has permission for both entities.

## Best Possible Solution

Define an audited publisher-aware deleted-slug reclaim or release flow that preserves the 30-day reservation policy, verifies authority over both the deleted source owner and destination owner, and updates backend, UI, API docs, and regression coverage together.

## Reproduction Assessment

Yes for the general source path: current main rejects renaming to any different skills.by_slug row, merge rejects a soft-deleted source, and the UI only lists active skills for the current owner/publisher. I did not verify the reporter's exact live production data.

## Solution Assessment

Unclear: the current merge suggestion is not sufficient for a deleted cross-owner source, but the safe fix needs a maintainer decision on reservation timing, authority checks, alias handling, and audit behavior.
