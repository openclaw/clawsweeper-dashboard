## Summary

Keep open. Current main still has a source-visible deleted-slug collision path, and this open ClawHub issue also needs maintainer product/security judgment before changing cross-owner reclaim behavior.

## What This Changes

The issue asks for deleted skill slugs to become reusable or reclaimable across personal and organization owners when the actor has authority for both.

## Best Possible Solution

Keep the 30-day deleted-slug reservation as the baseline, align rename with the expired-reservation lazy-release contract, and require audited publisher authority checks for any in-window or cross-publisher reclaim path.

## Reproduction Assessment

Yes, from source inspection: a soft-deleted row can remain in skills.by_slug, rename rejects it as a collision, and merge rejects it as a deleted source. I did not run a live Convex reproduction or inspect production data.

## Solution Assessment

Unclear for the full request because immediate or cross-publisher reclaim needs product and security policy. The narrow maintainable code direction is to make rename honor the existing expired owner-unpublish release contract and keep broader reclaim behind explicit authorization.
