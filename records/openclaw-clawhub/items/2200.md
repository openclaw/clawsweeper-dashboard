## Summary

Keep open: current main still has a source-level mismatch where expired owner-deleted slugs can be shown as claimable and published over, but rename rejects the soft-deleted row first and merge rejects a soft-deleted source. The broader personal/org reclaim path is ownership-sensitive and still needs maintainer product and security policy.

## What This Changes

The issue asks for deleted skill slugs to be reusable or releasable, and mergeable or reclaimable across personal and organization publishers when the actor has authority.

## Best Possible Solution

Keep the 30-day owner-delete reservation, align rename with the existing expired-claim behavior, and add any in-window or cross-publisher reclaim path only with explicit audited authority checks.

## Reproduction Assessment

Yes, from source inspection: after an owner-deleted skill's reservation expires, availability and publish can treat the slug as claimable, but rename rejects the soft-deleted row first and merge rejects a soft-deleted source. I did not run a live Convex reproduction or inspect production data.

## Solution Assessment

Unclear for the full request: post-expiry rename alignment is a narrow maintainable fix, but immediate or cross-publisher self-service reclaim needs maintainer product and security judgment.
