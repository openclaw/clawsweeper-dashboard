## Summary

Keep open. Current main still has a source-visible mismatch: availability and publish handle expired owner-delete slug reservations, but rename rejects the soft-deleted `skills.by_slug` row first and merge requires a live source skill; cross-publisher reclaim also needs explicit product/security policy.

## What This Changes

Requests a way to reuse a soft-deleted skill slug, including renaming another skill to that slug or reclaiming/merging it across publishers when the actor can manage both.

## Best Possible Solution

Keep the 30-day owner-delete reservation, align rename with the existing post-expiry lazy release contract, and route in-window or cross-publisher reclaim through audited authority checks for both publishers.

## Reproduction Assessment

Yes, from source inspection: an expired owner-unpublished slug can be reported available and claimed by publish, but owner rename still rejects the soft-deleted `skills.by_slug` row and merge rejects a soft-deleted source. I did not run a live Convex reproduction or inspect production data.

## Solution Assessment

Unclear for the full request: immediate reuse or cross-publisher reclaim before the 30-day TTL needs maintainer product/security judgment. The narrow maintainable fix is to align rename with the existing expired owner-unpublish release contract while keeping in-window reclaim guarded.
