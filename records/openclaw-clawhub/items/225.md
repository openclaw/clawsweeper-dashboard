## Summary

Keep open. Current main partially addresses the request with skill publishing, ignore handling, server-side static scan persistence, acceptable-use docs, and a repo-local ClawScan dry-run script, but it still lacks a supported public skill lint command, direct `skill publish --dry-run`, and shared browser/CLI pre-upload suspicious-scan feedback; the overlapping local scanner request remains open at https://github.com/openclaw/clawhub/issues/1767.

## What This Changes

The issue requests better onboarding for skill upload management, including skill linting, CLI upload/dry-run support, pre-upload suspicious-scan feedback, and clearer abuse-policy guidance.

## Best Possible Solution

Define a maintainer-approved shared skill preflight layer that browser upload, direct CLI publish, and any local scanner can reuse, with stable lint/dry-run output, scanner parity, ignore guidance, and abuse-policy wording aligned to moderation.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature and policy/onboarding request. Source inspection gives high confidence that current main lacks a supported public skill lint command, direct `skill publish --dry-run`, and shared pre-upload suspicious-scan feedback.

## Solution Assessment

Unclear as one broad implementation. The direction is useful, but the best path is to split scanner parity, CLI lint/dry-run behavior, browser preflight UX, exit-code semantics, and abuse-policy copy before implementation.
