## Summary

Keep open. Current main partially addresses onboarding with skill publish, ignore handling, stored static scans, install-time suspicious warnings, and acceptable-usage docs, but the requested skill lint command, direct skill-publish dry run, and shared pre-upload suspicious-scan feedback remain missing.

## What This Changes

Add clearer skill publishing onboarding and upload management, including CLI lint/upload tooling, pre-upload suspicious-skill feedback, ignore handling guidance, and abuse/sandbox policy copy.

## Best Possible Solution

Build a shared skill preflight layer for CLI and browser publish, then add `clawhub skill lint <path>`, direct `clawhub skill publish --dry-run`, scanner-aligned remediation output, explicit ignore behavior, and policy-linked enforcement copy.

## Reproduction Assessment

Not applicable as a feature/onboarding request. Source inspection gives a high-confidence verification path for the gap: there is no skill lint command, no direct `skill publish --dry-run`, and no browser/CLI pre-upload suspicious-scan path.

## Solution Assessment

No, not as one broad automatic repair. The maintainable path is to split CLI lint/dry-run, shared scanner preflight, browser UX, and moderation-policy decisions into smaller reviewed changes.
