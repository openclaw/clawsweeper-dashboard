## Summary

Keep open. ClawHub cleanup policy does not auto-close issues, and current main still only partially covers the request: skill CLI publishing, ignore handling, hosted scan persistence, acceptable-usage policy, and a repo-local ClawScan dry-run script exist, but there is no public `clawhub skill lint`, no direct `clawhub skill publish --dry-run`, and no shared browser/CLI pre-upload suspicious-scan feedback.

## What This Changes

The issue requests better skill-author onboarding through CLI lint/upload tooling, pre-upload suspicious-scan feedback, clearer ignore handling, and moderation-policy guidance.

## Best Possible Solution

Define a shared, maintainer-approved skill preflight layer that both CLI and browser upload can call, then expose it through `clawhub skill lint <path>` and `clawhub skill publish --dry-run` with policy copy that matches hosted ClawScan and moderation behavior.
