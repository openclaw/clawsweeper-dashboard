## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main only partially addresses the request: CLI/browser publish validation, ignore handling, stored static scans, and acceptable-usage copy exist, but a skill lint command, direct `skill publish --dry-run`, and shared pre-upload suspicious-scan feedback are still missing.

## What This Changes

The issue requests clearer skill developer onboarding, especially CLI lint/upload tooling, pre-upload suspicious-skill feedback, abuse-policy clarity, and conflict-resolution guidance.

## Best Possible Solution

Build a shared skill preflight layer used by both CLI and browser publish, with `clawhub skill lint <path>`, `clawhub skill publish --dry-run`, scanner-aligned findings, explicit ignore behavior, and policy-linked remediation copy.
