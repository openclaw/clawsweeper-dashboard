## Summary

Keep open. This is an open ClawHub feature/onboarding issue, and current main still only partially addresses it: ignore handling, publish-time static scans, rescan, and policy docs exist, but skill lint, direct skill publish dry-run, and shared pre-upload scanner feedback remain missing.

## What This Changes

The issue asks for better skill author onboarding through CLI lint/upload tooling, pre-upload suspicious-scan feedback, clearer ignore behavior, and moderation/abuse guidance.

## Best Possible Solution

Split the work into a shared skill preflight layer, `clawhub skill lint <path>`, direct `clawhub skill publish --dry-run`, browser pre-upload findings, and policy-linked remediation guidance, coordinated with https://github.com/openclaw/clawhub/issues/1767.
