## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main only partially addresses the request: skill publish, ignore handling, server-side static scans, and abuse docs exist, but skill lint, direct skill publish dry-run, and shared pre-upload suspicious-scan feedback are still missing.

## What This Changes

Request a skill lint/upload preflight workflow that clarifies ignored files, suspicious warnings, abuse policy, and contributor remediation before or during upload.

## Best Possible Solution

Define a maintainer-approved shared preflight layer for CLI and browser publishing, then expose it through skill lint, direct skill publish dry-run, remediation-oriented scan output, documented ignore behavior, and policy-aligned abuse guidance.
