## Summary

Keep open. Current main partially addresses the request with skill publishing, ignore handling, hosted static scans, acceptable-use docs, and a repo-local ClawScan dry-run script, but it still lacks a supported public skill lint/preflight command, direct `skill publish --dry-run`, and shared pre-upload suspicious-scan feedback.

## What This Changes

The issue requests better skill publisher onboarding through local lint/preflight tooling, clearer upload ignore behavior, earlier suspicious-scan feedback, abuse guidance, and a better path for resolving flagged skills.

## Best Possible Solution

Define a shared, maintainer-approved skill preflight layer that browser upload, direct CLI publish, and local scanner tooling can reuse, with stable output semantics and moderation-approved abuse guidance.
