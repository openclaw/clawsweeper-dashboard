## Summary

Keep open. Current main partially addresses onboarding with skill publishing, ignore handling, hosted static scan persistence, acceptable-use docs, and a repo-local ClawScan dry-run script, but the requested public skill lint command, direct `skill publish --dry-run`, and shared pre-upload suspicious-scan feedback are still missing; ClawHub cleanup policy also keeps issues open unless they are closed by ordinary maintainer action.

## What This Changes

The issue requests better developer onboarding around skill uploads, including CLI lint or dry-run support, pre-upload suspicious-scan feedback, clearer ignore handling, and abuse/remediation guidance.

## Best Possible Solution

Define a shared, maintainer-approved skill preflight layer that browser upload, direct CLI publish, local scanner tooling, and hosted scan reporting can reuse, with stable output semantics and moderation-approved abuse guidance.
