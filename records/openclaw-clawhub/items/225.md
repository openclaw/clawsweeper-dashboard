## Summary

Keep open. Repository cleanup policy for ClawHub issues does not allow auto-closing this open feature/onboarding request, and current main still only partially addresses it: ignore handling and publish-time static scans exist, but there is still no skill lint command, no direct skill publish dry-run, and no shared pre-upload scanner feedback path.

## What This Changes

The issue requests better skill-author onboarding through a CLI linter/upload tool, pre-upload suspicious-scan feedback, clearer ignore behavior, and clearer abuse/remediation guidance.

## Best Possible Solution

Split the work into a shared skill preflight layer, a `clawhub skill lint <path>` command, a direct `clawhub skill publish --dry-run` mode, browser pre-upload findings, and policy-linked remediation guidance.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature and onboarding request. Source inspection gives a high-confidence verification path for the remaining gaps: no skill lint command, no direct `skill publish --dry-run`, and no pre-upload static-scan feedback path.
