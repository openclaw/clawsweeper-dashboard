## What This Changes

The issue requests clearer skill onboarding, CLI lint/upload management, pre-upload suspicious-warning feedback, and clearer moderation/abuse handling for iterative skill publishing.

## Best Possible Solution

Define a shared skill preflight contract that both CLI and browser upload can call, then expose it through a public skill lint command and direct skill publish dry-run with stable human/JSON output, remediation-oriented findings, explicit ignore-file behavior, and linked acceptable-use guidance.
