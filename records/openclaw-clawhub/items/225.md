## Summary

Keep open. Current main only partially addresses the request: skill publishing, ignore handling, static scan persistence, and acceptable-usage docs exist, but skill lint, direct skill publish dry-run, and shared pre-upload suspicious feedback are still missing.

## What This Changes

Request asks for better skill publisher onboarding through a CLI linter/upload tool, earlier suspicious-scan feedback, clearer abuse guidance, and conflict-resolution or sandboxing flow.

## Best Possible Solution

Adopt a shared skill preflight layer used by both CLI and browser upload, then expose separate lint/dry-run commands and remediation-oriented moderation guidance with clear policy links.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence verification path: current main has no skill lint command, no direct skill publish --dry-run, and no pre-upload static moderation scan in the browser flow.

## Solution Assessment

No as a single umbrella change. The maintainable path is to split CLI lint/dry-run, browser preflight, static scanner reuse, and moderation-policy copy into reviewable pieces.
