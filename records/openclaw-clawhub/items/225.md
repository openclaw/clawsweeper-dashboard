## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still only partially covers this request: skill publishing, ignore handling, server-side scan persistence, and policy docs exist, but a skill lint command, direct skill publish dry-run, and shared pre-upload suspicious-scan feedback are still missing.

## What This Changes

The issue requests better skill-author onboarding with CLI lint or dry-run upload support, earlier suspicious-scan feedback, clearer ignore-file expectations, and clearer moderation/abuse guidance.

## Best Possible Solution

Define a shared skill preflight layer that both CLI and browser upload can call, then split it into maintainable pieces: local schema/path/text checks, `clawhub skill lint <path>`, direct `clawhub skill publish --dry-run`, remediation-oriented suspicious findings, and policy copy that matches hosted ClawScan and moderation behavior.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature and policy/onboarding request. Source inspection gives high confidence that current main lacks a public skill lint command, a direct skill publish dry-run flag, and shared pre-upload suspicious-scan feedback.

## Solution Assessment

Unclear as a single broad change. The requested direction is valid, but the maintainable solution should be a maintainer-approved split that keeps local/browser preflight, hosted ClawScan parity, and moderation policy consistent.
