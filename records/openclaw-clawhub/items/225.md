## Summary

Keep open. Current main only partially addresses the request: skill publishing, ignore handling, server-side static scanning, and moderation docs exist, but there is still no public skill lint command, no direct `skill publish --dry-run`, and no shared pre-upload suspicious-scan feedback for CLI or browser publishing.

## What This Changes

The issue requests better ClawHub skill author onboarding through lint/upload preflight tooling, clearer ignored-file behavior, earlier suspicious-scan feedback, and clearer abuse/sandboxing guidance.

## Best Possible Solution

Define a shared, maintainer-approved skill preflight layer that both CLI and browser publishing can call, then expose skill lint, direct skill publish dry-run, remediation-oriented scan output, documented ignore behavior, and policy-aligned abuse guidance.

## Reproduction Assessment

Not applicable as a runtime bug; this is an onboarding, publishing, and security-policy feature request. Source inspection gives high confidence current main still lacks a public skill lint command, direct `skill publish --dry-run`, and shared pre-upload suspicious-scan feedback.
