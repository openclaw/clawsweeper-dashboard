## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still only partially addresses the request: skill publish, ignore handling, server-side static scan persistence, and acceptable-usage docs exist, but there is no `clawhub skill lint`, no direct `clawhub skill publish --dry-run`, and no shared pre-upload suspicious-scan feedback for CLI or browser upload.

## What This Changes

Request improved skill onboarding with a CLI linter/upload workflow, clearer ignore and abuse guidance, and suspicious-scan feedback before upload completes.

## Best Possible Solution

Build a shared skill preflight layer for CLI and browser publish, then add `clawhub skill lint <path>`, direct `clawhub skill publish --dry-run`, scanner-aligned remediation output, explicit ignore behavior, and policy-linked enforcement guidance.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature/onboarding request. Source inspection gives a high-confidence verification path for the remaining gaps: no skill lint command, no direct `skill publish --dry-run`, and no pre-upload static-scan feedback path.

## Solution Assessment

No, not as one broad automatic repair. The maintainable path is to split CLI lint/dry-run, shared scanner preflight, browser UX, and moderation-policy copy into smaller reviewed changes.
