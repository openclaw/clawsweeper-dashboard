## Summary

Keep open. Current main partially addresses the onboarding concerns with ignore handling, sync/package dry-run flows, stored static scans, and acceptable-usage docs, but the requested direct skill lint command, `skill publish --dry-run`, and shared pre-upload suspicious-scan feedback are still missing; ClawHub cleanup policy keeps issues open.

## What This Changes

Request clearer skill onboarding with CLI lint/upload tooling, pre-upload suspicious-scan feedback, and better abuse/remediation guidance.

## Best Possible Solution

Build a shared skill preflight layer used by both CLI and browser publish, with `clawhub skill lint <path>`, direct `skill publish --dry-run`, remediation-oriented suspicious findings, explicit ignore behavior, and policy-linked enforcement copy.

## Reproduction Assessment

Not applicable as a feature/onboarding request. Source inspection on current main gives a high-confidence verification path: there is no skill lint command, no direct `skill publish --dry-run`, and no browser/CLI pre-upload suspicious-scan path.

## Solution Assessment

No, not as one umbrella repair. The maintainable path is to split CLI lint/dry-run, shared scanner preflight, browser UX, and moderation-policy decisions into reviewable pieces.
