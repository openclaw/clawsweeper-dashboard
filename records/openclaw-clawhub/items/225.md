## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still only partially covers the request: skill publish, ignore handling, publish-time static scans, and abuse docs exist, but there is still no skill lint command, no direct `clawhub skill publish --dry-run`, and no shared pre-upload scanner feedback for CLI or browser publish.

## What This Changes

Request a CLI skill linter/upload tool, clearer browser and CLI pre-upload validation, and policy guidance around suspicious uploads and abuse handling.

## Best Possible Solution

Split this into a shared skill preflight layer, `clawhub skill lint <path>`, `clawhub skill publish --dry-run`, browser pre-upload findings, and policy-linked remediation guidance.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature and onboarding request. Source inspection gives a high-confidence verification path for the remaining gaps: no skill lint command, no direct `skill publish --dry-run`, and no pre-upload static-scan feedback path.

## Solution Assessment

No, not as one broad automatic repair. The maintainable direction is to split CLI lint/dry-run, shared scanner preflight, browser UX, and moderation-policy copy into smaller reviewed changes.
