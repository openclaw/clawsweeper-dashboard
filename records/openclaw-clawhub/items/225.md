## Summary

Keep open: this is an open ClawHub feature and policy issue, and current main only partially covers the requested onboarding while leaving CLI lint/dry-run and shared pre-upload scanner feedback unresolved.

## What This Changes

The issue requests clearer skill onboarding and upload management, including CLI lint/upload tooling, suspicious-scan feedback before publish completes, and clearer abuse/remediation guidance.

## Best Possible Solution

Define a shared maintainer-approved skill preflight layer used by CLI and browser upload: deterministic local checks where possible, `clawhub skill lint <path>`, direct `clawhub skill publish --dry-run`, remediation-oriented scanner output, documented ignore behavior, and clear acceptable-usage enforcement links.
