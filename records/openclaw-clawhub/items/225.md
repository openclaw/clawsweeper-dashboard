## Summary

Keep open. Current main partially addresses the onboarding concern with ignore handling, stored static scans, sync/package dry-runs, and acceptable-usage docs, but the requested skill lint command, direct `skill publish --dry-run`, and shared pre-upload suspicious-scan feedback are still missing; ClawHub cleanup policy also keeps issues open.

## What This Changes

The issue requests better onboarding around skill linting, CLI skill upload management, pre-upload suspicious-skill feedback, and clearer abuse/remediation guidance.

## Best Possible Solution

Create a shared skill preflight layer used by both CLI and browser publish, with `clawhub skill lint <path>`, direct `clawhub skill publish --dry-run`, remediation-oriented scanner findings, documented ignore behavior, and policy-linked abuse guidance.
