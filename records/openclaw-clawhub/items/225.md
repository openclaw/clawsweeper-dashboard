## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main only partially addresses this request: CLI skill publishing, ignore handling, static scan persistence, and acceptable-usage copy exist, but there is still no skill lint command, no `skill publish --dry-run`, and no shared pre-upload suspicious-scan feedback across browser and CLI flows.

## What This Changes

Request to add skill linting and CLI upload tooling while improving pre-upload suspicious warnings, ignore behavior clarity, abuse-policy guidance, and remediation around suspicious skill flags.

## Best Possible Solution

Build a shared skill preflight layer that both CLI and browser upload can call: local schema/path/text/static checks where feasible, a `clawhub skill lint <path>` command, a direct `clawhub skill publish --dry-run` mode, remediation-oriented suspicious findings, explicit ignore-file behavior, and user-facing acceptable-usage enforcement copy or links.
