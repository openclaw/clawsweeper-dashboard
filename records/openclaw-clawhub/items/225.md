## Summary

Keep open. Current main partially addresses the onboarding request with skill publishing, ignore handling, acceptable-use docs, server-side scan persistence, and a repo-local ClawScan dry-run script, but the requested public skill lint command, direct `skill publish --dry-run`, and shared pre-upload suspicious-scan feedback are still missing; ClawHub cleanup policy also does not auto-close issues.

## What This Changes

The issue asks for clearer onboarding plus skill upload tooling: local lint/dry-run upload checks, earlier suspicious-scan feedback, abuse-policy clarity, and safer contributor remediation guidance.

## Best Possible Solution

Define a maintainer-approved shared skill preflight layer that browser upload, direct CLI publish, and local scanner tooling can reuse, with stable lint/dry-run output, hosted scanner parity, ignore guidance, and moderation-approved abuse-policy wording.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature and policy/onboarding request. Source inspection gives high confidence that current main still lacks a supported public skill lint command, direct `skill publish --dry-run`, and shared pre-upload suspicious-scan feedback.

## Solution Assessment

Unclear as one broad implementation. The direction is useful, but maintainers should split scanner parity, CLI lint/dry-run behavior, browser preflight UX, exit-code semantics, and abuse-policy copy before implementation.
