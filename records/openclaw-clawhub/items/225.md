## Summary

Current main implements CLI skill publishing, but it still requires authors to upload before obtaining scan results, and the abuse/remediation questions remain unresolved product-policy work. The local scanner portion is already tracked by the narrower open issue https://github.com/openclaw/clawhub/issues/1767, but that does not cover the entire umbrella request.

## What This Changes

The issue requests CLI lint/upload tooling, pre-upload security feedback, clearer abuse policy, and a contributor-remediation workflow for suspicious skills.

## Best Possible Solution

Keep CLI publishing as the supported upload path, continue local scanner design in https://github.com/openclaw/clawhub/issues/1767, and narrow this issue to explicit documentation and UI requirements for repository packaging, abuse consequences, appeals, and contributor remediation.

## Reproduction Assessment

Yes at source level for the central scanner gap: current v0.21.0 documentation explicitly says local path scans are unsupported and require uploading first. The abuse and remediation portions are policy requests rather than a single runtime bug.

## Solution Assessment

No as one umbrella implementation. CLI upload already exists, local scanning belongs in the narrower canonical issue, and the abuse/remediation workflow needs separately defined product and security policy.
