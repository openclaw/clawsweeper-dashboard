## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still only partially addresses the request: CLI ignore handling and policy docs exist, but there is no skill lint command, no direct `skill publish --dry-run`, and no shared pre-upload static-scan feedback path. The related open scanner request at https://github.com/openclaw/clawhub/issues/1767 overlaps on local scanner design but does not supersede this broader onboarding issue.

## What This Changes

The issue requests better skill-author onboarding through CLI lint/upload tooling, pre-upload scanner feedback, clearer ignore behavior, and abuse/remediation guidance.

## Best Possible Solution

Split this into a shared preflight/scanner design, a `clawhub skill lint <path>` command, a direct `clawhub skill publish --dry-run` mode, browser pre-upload findings, and policy-linked remediation guidance.

## Reproduction Assessment
