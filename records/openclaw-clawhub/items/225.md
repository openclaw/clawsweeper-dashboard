## Summary

Keep open. Current main partially addresses the onboarding request with skill publishing, ignore handling, hosted static scans, acceptable-use docs, and a repo-local ClawScan dry-run script, but it still lacks a supported public skill lint command, direct `skill publish --dry-run`, and shared pre-upload suspicious-scan feedback.

## What This Changes

The issue requests better publisher onboarding around skill linting, dry-run upload checks, pre-upload suspicious-scan feedback, `.git`/ignore behavior, and clearer abuse-policy guidance.

## Best Possible Solution

Define a shared, maintainer-approved skill preflight layer that browser upload, direct CLI publish, and local scanner tooling can reuse, with stable output semantics and moderation-approved abuse guidance.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature and policy/onboarding request. Source inspection gives high confidence that current main still lacks a supported public skill lint command, direct `skill publish --dry-run`, and shared pre-upload suspicious-scan feedback.

## Solution Assessment

Unclear as one broad implementation. The direction is useful, but maintainers should split scanner parity, CLI lint/dry-run behavior, browser preflight UX, exit-code semantics, and abuse-policy copy before implementation.
