## Summary

Keep open. Current main partially addresses onboarding with ignore handling, adjacent dry-run flows, stored static scans, and acceptable-usage docs, but the direct skill lint command, direct `skill publish --dry-run`, and shared pre-upload suspicious-scan feedback are still missing; ClawHub cleanup policy keeps issues open.

## What This Changes

The issue requests better onboarding and skill upload management with CLI linting/upload tooling, pre-upload suspicious warnings, clearer ignore behavior, and clearer abuse guidance.

## Best Possible Solution

Build a shared skill preflight layer used by both CLI and browser publish, with a `clawhub skill lint <path>` command, direct `clawhub skill publish --dry-run`, scanner-aligned findings, explicit ignore behavior, and policy-linked remediation copy.

## Reproduction Assessment

Not applicable as a feature/onboarding request. Source inspection on current main gives a high-confidence verification path: there is no skill lint command, no direct `skill publish --dry-run`, and no browser/CLI pre-upload suspicious-scan path.

## Solution Assessment

No, not as one broad automatic repair. The maintainable path is to split CLI lint/dry-run, shared scanner preflight, browser UX, and moderation-policy decisions into reviewable pieces.
