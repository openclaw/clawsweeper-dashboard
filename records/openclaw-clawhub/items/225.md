## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still only partially addresses the request: skill publishing, ignore handling, server-side static scans, and acceptable-usage docs exist, but there is no skill lint command, no direct `skill publish --dry-run`, and no shared pre-upload suspicious-scan feedback.

## What This Changes

Request to improve skill onboarding with CLI lint/upload preflight, clearer ignore behavior, earlier suspicious-warning feedback, abuse-policy clarity, and remediation-oriented moderation guidance.

## Best Possible Solution

Build a shared skill preflight layer used by both CLI and browser upload, then expose `clawhub skill lint`, direct `skill publish --dry-run`, local/static findings where feasible, explicit ignore behavior, and remediation-focused policy guidance.
