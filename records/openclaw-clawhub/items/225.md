Repository policy keeps ClawHub issues open, and current main only partially addresses the request: skill publish, ignore handling, server-side static scans, and acceptable-usage docs exist, but skill lint, direct `skill publish --dry-run`, and shared pre-upload suspicious feedback are still missing.

## What This Changes

Request onboarding and upload-management improvements for skill authors, including a linter, CLI upload/preflight tooling, pre-upload suspicious feedback, clearer ignore behavior, and abuse-policy guidance.

## Best Possible Solution

Build a shared skill preflight layer for CLI and browser upload, then expose focused lint/dry-run surfaces and remediation-oriented moderation guidance with clear policy links.

## Solution Assessment
