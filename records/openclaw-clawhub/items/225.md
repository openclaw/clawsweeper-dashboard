## Summary

Keep open. This is an open ClawHub feature and policy request, and current main only partially addresses it: skill publishing, CLI ignore handling, hosted static scan persistence, acceptable-use docs, and a repo-local ClawScan script exist, but the supported public skill lint command, direct `skill publish --dry-run`, and shared pre-upload suspicious-scan feedback are still missing; the narrower local scanner request remains open at https://github.com/openclaw/clawhub/issues/1767.

## What This Changes

Request a clearer skill-author onboarding flow with CLI upload/lint tooling, pre-upload suspicious-scan feedback, clearer `.git` ignore behavior, and abuse/remediation guidance.

## Best Possible Solution

Define a maintainer-approved shared skill preflight layer that browser upload, direct CLI publish, and any local scanner can reuse, with stable lint/dry-run output, hosted scanner parity, ignore guidance, and moderation-approved abuse-policy wording.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature and policy/onboarding request. Source inspection gives high confidence that current main lacks a supported public skill lint command, direct `skill publish --dry-run`, and shared pre-upload suspicious-scan feedback.

## Solution Assessment

Unclear as one broad implementation. The direction is useful, but maintainers should split scanner parity, CLI lint/dry-run behavior, browser preflight UX, exit-code semantics, and abuse-policy copy before implementation.
