## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still only partially addresses this request: direct skill publishing, sync dry-run, ignore handling, static scan persistence, and acceptable-usage docs exist, but there is no dedicated skill lint command, no direct `skill publish --dry-run`, and no shared pre-upload suspicious-scan feedback.

## What This Changes

The issue asks for better skill author onboarding with CLI lint/upload tooling, clearer ignore behavior, pre-upload suspicious warnings, and clearer abuse/moderation guidance.

## Best Possible Solution

Adopt a shared skill preflight layer used by both CLI and browser upload, then expose separate lint/dry-run commands and remediation-oriented moderation guidance with clear policy links.

## Reproduction Assessment
