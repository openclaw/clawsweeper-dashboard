## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still only partially addresses the request: skill publishing, ignore handling, server-side scan persistence, acceptable-use docs, and a repo-local ClawScan dry-run script exist, but there is no supported public skill lint command, no direct `skill publish --dry-run`, and no shared pre-upload suspicious-scan feedback for browser and CLI publishing.

## What This Changes

The issue requests better publisher onboarding around skill upload management: CLI linting/dry-run, clearer ignore behavior, earlier suspicious-scan feedback, and abuse/remediation guidance.

## Best Possible Solution

Define a maintainer-approved shared skill preflight layer that browser upload, direct CLI publish, and local scanner tooling can reuse, with stable lint/dry-run output, scanner parity, ignore guidance, and abuse-policy wording aligned to moderation behavior.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature and policy/onboarding request. Source inspection gives high confidence that current main lacks a supported public skill lint command, direct `skill publish --dry-run`, and shared browser/CLI pre-upload suspicious-scan feedback.

## Solution Assessment

Unclear as one broad implementation. The direction is useful, but the best path is to split scanner parity, CLI lint/dry-run behavior, browser preflight UX, exit-code semantics, and abuse-policy copy before implementation.
