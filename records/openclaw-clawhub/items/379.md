## Summary

Keep open. This is a ClawHub issue, and the cleanup policy only permits auto-closing PRs that current main definitely implements; the current code also still has a real display/API gap where backend moderation can suppress uncorroborated VirusTotal Code Insight suspicious results, while visible VirusTotal badges and v1 security snapshots still derive status from the raw stored VT status or verdict.

## What This Changes

The issue asks ClawHub to show VirusTotal zero-detection Code Insight-only results as clean or undetected instead of suspicious, and to avoid the related warning banner/badge false positive.

## Best Possible Solution

Add a shared VirusTotal display/status normalization helper used by skill scan badges, scanner detail pages, the detail security summary, and v1 security snapshots. It should reuse the existing backend suppression semantics for Code Insight-only suspicious results with zero malicious and zero suspicious engine counts, keep the Code Insight text visible as context, preserve stronger warnings whenever another scanner corroborates risk, and include a recompute/backfill path for stale skill-level moderation flags where needed.
