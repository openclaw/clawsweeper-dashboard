## Summary

Keep open for maintainer merge review: current main does not have publisher recovery, and this PR appears to be the active, well-evidenced implementation with remaining maintainer-owned authority-transfer risk.

## What This Changes

This PR adds an admin-only HTTP endpoint and admin CLI command to recover a personal publisher between verified immutable GitHub OAuth provider IDs, with schema, docs, spec, and regression-test updates.

## Best Possible Solution

Land the guarded recovery path with staff identity verification, dry-run before apply, audit logging, and a separate resumable migration path for publishers that exceed the per-table bound.

## Reproduction Assessment

Yes. Source inspection shows current main has no publisher-recovery route and personal publisher access is linkedUserId-based, so a replacement GitHub principal remains unable to publish under the legacy personal publisher.

## Solution Assessment

Yes. An admin-only transfer keyed by both immutable GitHub provider IDs, with dry-run, explicit staff verification, conflict rejection, bounded migration, and audit logging, is the narrowest maintainable path compared with rewriting Convex Auth rows or relinking by mutable usernames.
