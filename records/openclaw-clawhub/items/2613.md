## Summary

Keep open: current main still lacks a personal publisher recovery route, and this PR adds the guarded admin recovery path with strong local Convex proof and no discrete actionable patch defect found. The remaining concern is maintainer acceptance of the security-sensitive operator procedure and bounded migration behavior before merge.

## What This Changes

This PR adds an admin-only personal publisher recovery HTTP endpoint, admin CLI command, shared response schema, docs/spec notes, and regression tests for transferring a legacy personal publisher between verified immutable GitHub provider IDs.

## Best Possible Solution

Land the guarded recovery path with dry-run-first staff verification, audit logging, and a separate resumable migration path for publishers that exceed the per-table bound.

## Reproduction Assessment

Yes. Source inspection shows current main has no `/api/v1/users/publisher-recovery` route and personal publisher access is linkedUserId-based, so a replacement GitHub principal cannot publish under the legacy personal publisher without staff recovery.

## Solution Assessment

Yes. An admin-only transfer keyed by both immutable GitHub provider IDs, with dry-run, explicit staff verification, conflict rejection, bounded migration, and audit logging, is the narrowest maintainable path compared with rewriting Convex Auth rows or relinking by mutable usernames.
