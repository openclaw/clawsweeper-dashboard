## Summary

Keep open: current main does not yet include the recovery path, and this PR is the viable implementation candidate with no discrete blocking code finding from this review.

## What This Changes

The PR adds an admin-only personal publisher recovery HTTP API, admin CLI command, schema/docs/spec updates, and regression coverage for verified GitHub principal replacement.

## Best Possible Solution

Land the guarded recovery path with the documented staff verification procedure, using dry-run before apply and a separate resumable migration for publishers that exceed the per-table bound.

## Reproduction Assessment

Yes. Current main lacks a publisher-recovery endpoint and authorizes personal publishers by linkedUserId, so a replacement GitHub principal remains a separate user that cannot publish under the legacy personal publisher.

## Solution Assessment

Yes. An admin-only transfer keyed by both immutable GitHub provider IDs, with dry-run, explicit staff verification, conflict rejection, bounded migration, and audit logging, is narrower and safer than rewriting Convex Auth rows or relinking by mutable usernames.
