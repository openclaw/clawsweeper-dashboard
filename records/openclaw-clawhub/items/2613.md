## Summary

The guarded recovery design and live Convex proof are strong, but the apply transaction leaves the recovered handle’s active reservation assigned to the previous user. That creates conflicting persisted authorities that a routine post-login synchronization can encounter, so the automerge-opted PR needs one narrow repair before merge.

## What This Changes

Adds an admin-only, dry-run-first personal publisher recovery workflow across Convex, HTTP API, admin CLI, schema, docs, specs, and tests.

## Best Possible Solution

Atomically reassign the active reservation to the verified destination user in the recovery mutation, fail closed on ambiguous or third-party reservation state, and cover a later normal profile synchronization in tests and refreshed live proof.
