## Summary

Keep this PR open with changes required: the recovery transaction can transfer publisher control while leaving protected-handle authority assigned to the former OAuth principal, causing a later profile synchronization to reject or change the recovered handle.

## What This Changes

The PR adds an admin-only, dry-run-first personal publisher recovery API and CLI that transfers publisher ownership between verified immutable GitHub principals and migrates bounded owner-linked resources.

## Best Possible Solution

In the same recovery transaction, transfer an active reservation for the recovered handle to the replacement user or fail closed, then prove that a later GitHub profile synchronization preserves the recovered handle and publisher link.

## Reproduction Assessment

Yes. Recover a publisher whose handle has an active reservation for the former user, then run `syncGitHubProfileInternal` for the replacement user; source inspection shows `canUserClaimHandle` rejects the handle because the reservation still names the former user.

## Solution Assessment

No, not yet. The guarded recovery design is appropriately narrow, but it must transfer or reject every active authority for the handle atomically rather than updating publisher ownership alone.
