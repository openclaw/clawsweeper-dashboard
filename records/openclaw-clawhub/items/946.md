## Summary

Keep open. This is a ClawHub issue, and the supplied repository policy only allows auto-closing pull requests that current main definitely implements. Current main also still does not treat the reporter's top-level `required_credentials` or `permissions` frontmatter as ClawHub registry or scanner metadata, so the moderation appeal remains an unresolved schema and security-policy question.

## What This Changes

The reporter asks ClawHub to clear the `buff-roundup` suspicious flag and to recognize top-level SKILL.md `required_credentials` and `permissions` frontmatter as declared credential and permission metadata.

## Best Possible Solution

Maintainers should decide whether top-level `required_credentials` and `permissions` are supported ClawHub manifest aliases. If yes, implement them consistently across the shared schema, SKILL.md parser, static declaration collector, security prompt context, UI/API/docs/tests, and a rescan or backfill path; if not, ask the publisher to republish using the documented `metadata.openclaw.requires.env`, `primaryEnv`, or `envVars` shape and then make an explicit moderation decision for the skill.
