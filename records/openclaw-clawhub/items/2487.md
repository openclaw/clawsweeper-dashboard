## Summary

## What This Changes

The PR deletes the standalone ClawHub CLI GitHub Release workflow and removes its repair runbook from specs/deploy.md.

## Best Possible Solution

Preserve a no-republish GitHub Release repair path, either by keeping the standalone workflow or adding and documenting an explicit repair mode in the npm release workflow.

## Reproduction Assessment

Yes. Source inspection shows the remaining npm release workflow exits when the package version is already published, before the later GitHub Release creation step.

## Solution Assessment

No. Deleting the standalone repair workflow is only safe after maintainers accept manual repair or add an equivalent no-publish repair mode to the npm release workflow.
