## Summary

Keep open: this PR removes the documented post-publish GitHub Release repair path, but the remaining CLI npm release workflow still fails early when the package version is already on npm, before it can create or repair the GitHub Release.

## What This Changes

The PR deletes the standalone CLI GitHub Release repair workflow and updates the deploy runbook to remove that workflow’s repair instructions.

## Best Possible Solution

Preserve a no-republish GitHub Release repair path, either by keeping the standalone workflow or adding an explicit repair mode to the npm release workflow before deleting it.

## Reproduction Assessment

Yes from source inspection: after npm publish succeeds, a rerun of the remaining npm release workflow reaches the already-published package check and exits before GitHub Release creation.

## Solution Assessment

No; deleting the repair workflow is not the safest solution unless the remaining release workflow gains an explicit no-publish repair path or maintainers intentionally accept manual repair.
