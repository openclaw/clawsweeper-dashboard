## Summary

Keep open: the PR is collaborator-authored and current main still has the deleted workflow as the documented no-republish repair path for CLI GitHub Release failures after npm publish succeeds.

## What This Changes

The branch deletes the standalone ClawHub CLI GitHub Release workflow and removes its repair runbook from specs/deploy.md.

## Best Possible Solution

Preserve a no-republish GitHub Release repair path, either by keeping the standalone workflow or adding and documenting an explicit repair mode in the npm release workflow.

## Reproduction Assessment

Yes. Source inspection shows that after npm publish succeeds, rerunning the remaining npm release workflow exits on the already-published package before reaching GitHub Release creation.

## Solution Assessment

No. Deleting the repair workflow is only safe if maintainers intentionally accept manual repair or first add an equivalent no-publish repair mode to the npm release workflow.
