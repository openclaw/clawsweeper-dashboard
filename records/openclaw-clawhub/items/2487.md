## Summary

This PR should remain open because it removes the guarded recovery path for a partially completed CLI release. The retained workflow rejects an already-published npm version before reaching GitHub Release creation, so it cannot recover when npm publishing succeeds but a later release step fails.

## What This Changes

The PR deletes the standalone CLI GitHub Release repair workflow and its documented operator recovery procedure, consolidating CLI release handling into the npm release workflow.

## Best Possible Solution

Make the retained workflow verify an existing matching npm artifact, skip republishing, and continue through idempotent GitHub Release repair before deleting the dedicated recovery action.

## Reproduction Assessment

Yes at source level: let npm publishing succeed and a later GitHub Release step fail; rerunning non-preflight exits at the already-published check before release creation.

## Solution Assessment

No; consolidation is maintainable, but deleting recovery before adding equivalent resumability creates a release dead end.
