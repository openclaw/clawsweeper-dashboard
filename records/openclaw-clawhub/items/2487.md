## Summary

This PR needs explicit maintainer handling and should not merge as-is: the retained npm workflow rejects an already-published version before its GitHub Release stage, so deleting the standalone workflow removes the documented recovery path for partial CLI releases.

## What This Changes

The PR deletes the standalone CLI GitHub Release create/repair workflow and removes its recovery procedure from the deploy specification.

## Best Possible Solution

Preserve a documented, independently runnable GitHub Release repair operation that validates the tag and release proof but never attempts to republish an existing npm version.

## Reproduction Assessment

Yes. Current-main source shows npm publication precedes GitHub Release creation, while a subsequent publish run exits when that package version already exists.

## Solution Assessment

No. Removing the workflow is not the narrowest maintainable solution unless equivalent no-republish recovery is retained or release owners explicitly adopt and document a validated manual process.
