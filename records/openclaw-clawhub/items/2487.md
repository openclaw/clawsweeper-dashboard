## Summary

This PR should stay open for release-automation owner review. Current main still has a documented, tested repair-only GitHub Release workflow, and the remaining npm release workflow exits on an already-published package before it can repair the GitHub Release.

## What This Changes

The PR deletes the standalone ClawHub CLI GitHub Release workflow and removes deploy runbook instructions for using it as a repair path.

## Best Possible Solution

Keep the workflow, or replace it with an explicit tested repair-only path that can create or update the GitHub Release after npm already contains the version, then update the runbook accordingly.

## Reproduction Assessment

Yes. From source inspection, the remaining npm release workflow exits when the package version is already published before it reaches GitHub Release creation, while the deleted workflow can operate from npm dist metadata without republishing.

## Solution Assessment

No. Removing the workflow is too broad unless the PR preserves or replaces the post-publish GitHub Release repair path, and the final direction needs release-automation owner judgment.
