## Summary

Keep open: this collaborator-authored PR is not safe to merge as-is because it removes the documented no-republish GitHub Release repair path while the remaining npm release workflow still fails before release creation once the package is already published.

## What This Changes

This PR deletes the standalone ClawHub CLI GitHub Release workflow and removes its repair instructions from specs/deploy.md.

## Best Possible Solution

Preserve a no-republish GitHub Release repair path, either by keeping the standalone workflow or adding an explicit repair mode to the npm release workflow before deleting it.

## Reproduction Assessment

Yes. From source inspection, after npm publish succeeds a rerun of the remaining npm release workflow reaches the already-published package check and exits before GitHub Release creation.

## Solution Assessment

No. Deleting the standalone repair workflow is not the narrowest safe cleanup unless the remaining release workflow gains an explicit no-publish repair path or maintainers intentionally accept manual repair.
