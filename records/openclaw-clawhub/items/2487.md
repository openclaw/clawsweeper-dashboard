## Summary

Current main still contains and documents the standalone CLI GitHub Release repair workflow, and this PR would remove that operator recovery path while the remaining npm workflow exits before release creation when the package version is already published.

## What This Changes

The PR deletes `.github/workflows/clawhub-cli-github-release.yml` and removes the deploy-spec runbook section for using that workflow to repair CLI GitHub Releases.

## Best Possible Solution

Keep the repair-only workflow, replace it with an equivalent tested recovery mode, or document an owner-approved manual recovery procedure before removing it.

## Reproduction Assessment

Yes. Source inspection reproduces the recovery gap: if npm already has `clawhub@<version>`, the surviving npm workflow exits before GitHub Release creation, while the deleted workflow is the documented repair path.

## Solution Assessment

No. Deleting the workflow is too broad unless the PR preserves or replaces the post-publish GitHub Release repair path, or release owners explicitly retire that workflow.
