## Summary

Keep open: current main does not implement this PR, and the branch addresses a source-reproducible publish ownership mismatch, but it is not merge-ready because real behavior proof is still missing and maintainer review flagged runtime-id reuse after soft-delete for policy/security confirmation.

## What This Changes

The branch adds scoped package owner mismatch validation in schema, backend, and publish UI; audits package soft-deletes; blocks releases onto soft-deleted packages; changes runtime-id collision handling; moves publish routes to `/plugins/publish` and `/skills/publish`; and updates tests, CODEOWNERS, and CodeQL publish metadata.

## Best Possible Solution

Land the ownership mismatch fix after proof is added, and either explicitly confirm runtime IDs become reusable after soft-delete or keep them reserved until a separate admin release policy exists.

## Reproduction Assessment

Yes, from source inspection: current main lets a selected `ownerHandle` and scoped package name travel separately into publish target resolution without comparing them. I did not run a live publish flow in this read-only review.

## Solution Assessment

Unclear as merge-ready: the shared validation plus backend and UI enforcement is a maintainable direction for the ownership bug, but the bundled runtime-id reuse change needs maintainer policy/security confirmation before it should ship.
