## Summary

Keep open: source inspection supports a real moderation false-positive class, but the concrete path is loopback raw-IP URLs in manifest files rather than GHCR image pulls. Tuning this scanner affects abuse detection, so it needs maintainer/security approval before implementation.

## What This Changes

The issue asks for the static moderation analyzer to stop over-flagging docker-compose-based skills as `suspicious.install_untrusted_source` for trusted/local container install patterns.

## Best Possible Solution

Refine the manifest heuristic to exempt localhost/loopback install endpoints while preserving public raw-IP and URL-shortener warnings, with regression tests for both safe and unsafe manifest cases.

## Reproduction Assessment

Yes. A YAML/JSON/TOML manifest containing `http://127.0.0.1:<port>/...` reaches `scanManifestFile`, matches `RAW_IP_URL_PATTERN`, and emits `suspicious.install_untrusted_source`; the public v4.1.22 compose sample has that shape.

## Solution Assessment

Yes, but narrower than the issue’s suggested allowlists or metadata field. A loopback-aware manifest matcher is the most maintainable fix unless maintainers want a broader registry trust policy.
