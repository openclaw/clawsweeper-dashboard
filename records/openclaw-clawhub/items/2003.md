## Summary

Keep open: current main still supports a real static-scan false positive, but the strongest evidence points to loopback raw-IP URLs in bundled manifest files rather than the GHCR image line. Because this changes security moderation behavior, it should stay with maintainers for approval before implementation.

## What This Changes

The issue asks ClawHub to reduce false `suspicious.install_untrusted_source` findings for docker-compose-based skills that bundle compose files and pull public registry images.

## Best Possible Solution

Tune the moderation engine to exempt localhost and loopback manifest URLs while preserving public raw-IP and shortener warnings, with focused regression tests for both allowed local endpoints and still-flagged remote sources.

## Reproduction Assessment

Yes. The public moderation endpoint still reports `suspicious.install_untrusted_source` for InvestorClaw, and current main has a deterministic source path that flags raw IPv4 HTTP(S) URLs in YAML/JSON/TOML manifests.

## Solution Assessment

Yes, but narrower than the issue's registry allowlist suggestion. Exempting loopback and localhost manifest URLs while preserving public raw-IP and shortener warnings is more maintainable than adding a broad trusted-registry or installation-type exception first.
