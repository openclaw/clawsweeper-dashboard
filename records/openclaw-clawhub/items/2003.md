## Summary

Keep open. The issue is a valid, localized scanner false-positive/policy report, but ClawHub cleanup policy keeps issues open, and changing `install_untrusted_source` trust heuristics for compose manifests is security-sensitive maintainer work rather than conservative auto-close cleanup.

## What This Changes

The issue asks ClawHub to reduce or clarify `suspicious.install_untrusted_source` static scanner findings for docker-compose-based skills that bundle local compose files and pull public container images.

## Best Possible Solution

Define a narrow, context-aware static scanner policy for docker-compose manifests that exempts expected loopback service metadata while preserving warnings for URL shorteners, public raw-IP install sources, mutable remote compose fetches, and weak container provenance.

## Reproduction Assessment

Yes. Public ClawHub data reproduces `suspicious.install_untrusted_source` for InvestorClaw 4.1.22, and current main still applies the raw-IP manifest rule to YAML files without a loopback compose exception.

## Solution Assessment

No for a broad verified-registry allowlist as the first fix; that is too coarse for a supply-chain scanner. A maintainer-approved compose-manifest policy that distinguishes loopback metadata from risky install sources is the safer direction.
