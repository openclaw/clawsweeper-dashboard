## Summary

Keep open: current main still has the manifest raw-IP/shortener rule that can flag loopback URLs in bundled compose YAML, and changing container/compose trust treatment is a scanner security-policy decision rather than conservative cleanup.

## What This Changes

The issue asks ClawHub to reduce `suspicious.install_untrusted_source` false positives for docker-compose-based skills that use bundled compose files and public container images.

## Best Possible Solution

Define a narrow compose-manifest scanner policy that permits expected loopback service metadata while preserving warnings for URL shorteners, public raw-IP install sources, remote mutable compose fetches, mutable image tags, and weak container provenance.

## Reproduction Assessment

Yes. Current source clearly scans bundled YAML manifests with the raw-IP rule, and the live affected version still reports `suspicious.install_untrusted_source` while its `compose.yml` contains loopback URLs.

## Solution Assessment

Unclear. A broad verified-registry allowlist or new metadata knob is not clearly the safest fix; a narrower compose-manifest exception plus explicit container provenance rules is more maintainable.
