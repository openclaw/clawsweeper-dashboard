## Summary

Keep open: current main still has the manifest rule that can emit `suspicious.install_untrusted_source` for loopback/raw-IP URLs in YAML, live affected-version output still shows that static reason, and changing Docker Compose/container provenance trust is a security-policy decision rather than conservative cleanup.

## What This Changes

The issue asks ClawHub to reduce false-positive `install_untrusted_source` findings for docker-compose skills that use bundled compose files and public container images.

## Best Possible Solution

Define a narrow compose-manifest scanner policy that allows expected loopback service metadata while preserving warnings for URL shorteners, remote compose fetches, public raw-IP install sources, mutable tags, and weak container provenance.

## Reproduction Assessment

Yes. Current source plus the live scan API reproduce the reported reason code for `investorclaw` 4.1.22, and the affected compose file contains loopback URLs that match the raw-IP manifest rule.

## Solution Assessment

Unclear. A broad verified-registry allowlist is not the safest fix; a narrower compose-manifest exception plus explicit container provenance rules would be more maintainable.
