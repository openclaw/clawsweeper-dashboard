## Summary

Keep open. Current main still flags raw-IP URLs in YAML/JSON/TOML manifests as `suspicious.install_untrusted_source`, the affected public InvestorClaw version confirms that static finding, and changing scanner trust policy is not conservative cleanup for a ClawHub issue.

## What This Changes

The issue asks ClawHub to stop over-classifying docker-compose-based skills as `suspicious.install_untrusted_source` when they use local compose files and public container images.

## Best Possible Solution

Define a narrow manifest-scanner policy for docker-compose loopback service metadata while preserving warnings for URL shorteners, public raw-IP install sources, mutable remote compose fetches, and weak container provenance.

## Reproduction Assessment

Yes. Source inspection of current main shows manifest files are scanned with `RAW_IP_URL_PATTERN`, and the public InvestorClaw 4.1.22 scan/file endpoints show `suspicious.install_untrusted_source` with `http://127.0.0.1` compose metadata.

## Solution Assessment

No for a broad verified-registry allowlist as the first fix; it is too coarse for a supply-chain scanner. A context-aware compose manifest refinement with explicit scanner-policy approval is the safer path.
