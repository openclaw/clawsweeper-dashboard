## Summary

Keep open: current main still has a source-reproducible path where manifest raw-IP URLs become `suspicious.install_untrusted_source`, and the scanner has no docker-compose/GHCR-specific refinement yet. Because this is an abuse-detection heuristic, it needs maintainer/security approval before implementation.

## What This Changes

Refine static moderation so docker-compose-style skills are not overflagged as `suspicious.install_untrusted_source` for legitimate container-image workflows.

## Best Possible Solution

Refine the manifest raw-IP matcher to exempt localhost/loopback install endpoints while preserving warnings for public raw-IP URLs and shorteners; defer verified-registry allowlists or metadata fields unless maintainers choose that trust policy.

## Reproduction Assessment

Yes, for the narrowed false-positive class: source inspection shows YAML/JSON/TOML manifests containing a loopback raw-IP URL still match `RAW_IP_URL_PATTERN` and emit `suspicious.install_untrusted_source`. I do not see source evidence that a GHCR `image:` directive alone triggers this reason code.

## Solution Assessment

No, not as a broad verified-registry allowlist or new `installation_type` field yet. The narrower maintainable fix is a loopback-aware manifest matcher, with security review to keep public raw-IP and shortener warnings intact.
