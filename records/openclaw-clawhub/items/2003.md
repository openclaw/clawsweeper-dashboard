## Summary

Keep open: the false-positive class is real, but current evidence points to loopback raw-IP URLs in YAML manifests rather than GHCR image pulls, and changing a moderation heuristic needs maintainer/security approval.

## What This Changes

The issue asks for the static moderation analyzer to stop over-firing `suspicious.install_untrusted_source` on docker-compose based skills that use local compose files and trusted container images.

## Best Possible Solution

Refine the manifest heuristic to exempt local loopback install endpoints while preserving warnings for public raw-IP URLs and URL shorteners, with regression tests for both sides.

## Reproduction Assessment

Yes, by source inspection and public scan evidence. InvestorClaw 4.1.22 still shows `suspicious.install_untrusted_source`, its manifest files contain loopback HTTP raw-IP URLs, and current main routes those YAML files through `RAW_IP_URL_PATTERN`.

## Solution Assessment

Yes, but narrower than the issue's proposed GHCR allowlist or new metadata field. A loopback-aware manifest matcher is the smallest maintainable fix; any broader verified-registry policy should be an explicit maintainer decision.
