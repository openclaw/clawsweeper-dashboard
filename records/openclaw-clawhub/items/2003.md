## Summary

Keep open. Current main still has the manifest-level raw-IP/shortener rule that can produce this finding, the affected public version demonstrates the reported scanner output, and changing install-source trust heuristics is security policy work rather than conservative cleanup.

## What This Changes

The issue asks ClawHub to reduce false `suspicious.install_untrusted_source` static findings for docker-compose skill bundles that use local compose metadata and public registry images.

## Best Possible Solution

Define a narrow compose-aware static scanner policy that exempts expected loopback service metadata while preserving warnings for URL shorteners, public raw-IP install sources, mutable remote compose fetches, and weak container provenance.

## Reproduction Assessment

Yes. Source inspection shows the current manifest scanner can flag YAML loopback URLs, and the public 4.1.22 scan/file endpoints show `suspicious.install_untrusted_source` alongside those compose URLs.

## Solution Assessment

No, not as a broad verified-registry allowlist. The safer path is a maintainer-approved compose-manifest rule that distinguishes loopback metadata from actual untrusted install sources.
