## Summary

Keep open: the report identifies a plausible static-scanner false-positive class, but current evidence points to manifest raw-IP/loopback matching rather than GHCR image pulls, and changing this security heuristic needs maintainer approval.

## What This Changes

The issue asks to reduce `suspicious.install_untrusted_source` false positives for docker-compose-based skills that pull container images from trusted public registries.

## Best Possible Solution

Refine the manifest heuristic to exempt localhost/loopback install endpoints while preserving warnings for public raw-IP URLs and URL shorteners, with regression tests for both cases.

## Reproduction Assessment

Yes, by source inspection. A YAML/JSON/TOML manifest containing a loopback raw-IP URL will match `RAW_IP_URL_PATTERN` and flow through `scanManifestFile` to `suspicious.install_untrusted_source`; I did not run tests because this review is read-only.

## Solution Assessment

Unclear as proposed. A verified-registry allowlist or new `metadata.installation_type` field needs product/security policy, while the narrow source-backed repair is a loopback-aware manifest matcher.
