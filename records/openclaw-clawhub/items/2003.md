## Summary

Keep open: this looks like a real static-scanner false-positive class, but current source supports a narrower manifest raw-IP/loopback path rather than the GHCR image-pull hypothesis. Because the change would tune a security moderation heuristic, it should get maintainer/security approval before implementation.

## What This Changes

Refine the static moderation analyzer so docker-compose/local compose-based skills are not flagged as `suspicious.install_untrusted_source` for acceptable trusted or local install sources.

## Best Possible Solution

Refine the manifest heuristic to exempt loopback/local install endpoints while preserving warnings for public raw-IP URLs and URL shorteners; consider registry or metadata policy only after maintainer/security approval.

## Reproduction Assessment

Yes, by source inspection. A YAML/JSON/TOML manifest containing `http://127.0.0.1:<port>/...` matches the raw-IP URL pattern and flows through `scanManifestFile` to `suspicious.install_untrusted_source`.

## Solution Assessment

Yes for a narrow loopback-aware manifest matcher, pending maintainer/security approval. A broad verified-registry allowlist or new `installation_type` field would be a larger product/security policy change.
