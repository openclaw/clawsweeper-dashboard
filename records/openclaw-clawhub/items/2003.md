## Summary

Keep open: this is a source-reproducible moderation-scanner false-positive class, but the concrete path appears to be loopback raw-IP URLs in manifest files rather than GHCR image pulls. Tuning this reason code affects abuse detection, so it needs maintainer/security approval before implementation.

## What This Changes

The issue requests reducing false positives for docker-compose-based skills that currently receive `suspicious.install_untrusted_source`.

## Best Possible Solution

Refine the manifest install-source heuristic to treat loopback/local URLs as local endpoints while preserving warnings for public raw-IP URLs and URL shorteners, with regression tests for both cases.

## Reproduction Assessment

Yes, by source inspection and affected public files. InvestorClaw 4.1.22 contains `http://127.0.0.1:...` URLs in YAML manifests, and current main still routes any HTTP(S) IPv4 URL in manifest files to `suspicious.install_untrusted_source`.

## Solution Assessment

Yes, but narrower than the GHCR allowlist or new metadata-field suggestions. A loopback-aware manifest matcher is the smallest maintainable fix unless maintainers choose a broader trusted-registry policy.
