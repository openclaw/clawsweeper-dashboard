Keep open. ClawHub cleanup policy keeps issues open, and current main appears to have narrowed the specific `install_untrusted_source` path since the reported engine version, but the historical hosted scan still needs maintainer/operator verification rather than source-only closure.

## What This Changes

The reporter asks ClawHub static analysis to stop classifying local docker-compose skills that pull a public verified GHCR image as `suspicious.install_untrusted_source`.

## Best Possible Solution

Have a security/moderation owner verify or rescan InvestorClaw under current v2.4.24 policy; if GHCR compose images still produce this rule, add a focused regression that local compose image refs are not install-source findings while shorteners/raw IPs still are, and leave scanner-detail disclosure to https://github.com/openclaw/clawhub/issues/2109.

No. The report gives concrete historical versions and a plausible cause, but current main v2.4.24 no longer has a source path that matches ordinary GHCR compose images for `install_untrusted_source`, and public current InvestorClaw data does not show a moderation object.
