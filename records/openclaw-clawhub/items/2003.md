## What This Changes

The issue asks ClawHub to reduce `suspicious.install_untrusted_source` false positives for docker-compose skills using local compose manifests and public verified container images.

## Best Possible Solution

Refine the static manifest matcher to exempt approved loopback/local install endpoints while preserving public raw-IP and URL-shortener warnings, with regression tests for both allowed and blocked cases.

## Reproduction Assessment

Yes, by source inspection. A `.yml`, `.yaml`, `.json`, or `.toml` manifest containing `http://127.0.0.1:<port>/...` matches the generic raw-IP URL regex and emits `suspicious.install_untrusted_source`; the GHCR-image hypothesis is not reproduced from current source.

## Solution Assessment

No for a broad verified-registry allowlist or new metadata field based on the current evidence. The narrower maintainable path is a loopback-aware manifest rule approved by maintainers/security, plus regression coverage that keeps public raw-IP URLs flagged.
