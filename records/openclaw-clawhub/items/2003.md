## Summary

Keep open: ClawHub cleanup policy keeps issues open, and current source plus the public moderation endpoint still support a real static-scan false positive in docker-compose/config manifests.

## What This Changes

The issue asks for the static moderation analyzer to stop treating legitimate docker-compose-based skill installs as `suspicious.install_untrusted_source`.

## Best Possible Solution

Tighten the manifest scanner to exempt localhost and loopback service endpoints while preserving warnings for public raw-IP URLs, shorteners, and genuinely remote install sources.

## Reproduction Assessment

Yes. The public moderation endpoint shows the current flag, and current source deterministically applies the raw-IP manifest heuristic to the published loopback URLs.

## Solution Assessment

No for broad registry allowlisting or a new installation-type field as the first fix. The narrower maintainable repair is to distinguish loopback/localhost manifest endpoints from public raw-IP install sources and cover both behaviors with regression tests.
