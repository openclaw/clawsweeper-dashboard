## Summary

Keep open. Current main still contains the manifest scanner path that can classify loopback URLs in YAML compose metadata as `suspicious.install_untrusted_source`, and public scan data confirms the affected version was flagged; changing this rule is security scanner policy work.

## What This Changes

The issue asks ClawHub to reduce false-positive `install_untrusted_source` static-scan flags for docker-compose-based skills with bundled compose files and registry images.

## Best Possible Solution

Codify a narrow manifest-scanner policy that exempts expected loopback compose service metadata while preserving warnings for URL shorteners, public raw-IP install sources, mutable remote compose fetches, and weak container provenance.

## Reproduction Assessment

Yes. Source inspection shows current main applies the raw-IP URL rule to `.yml` manifests, and public API data shows InvestorClaw 4.1.22 was flagged with `suspicious.install_untrusted_source` while its compose file contained matching `127.0.0.1` URLs.

## Solution Assessment

No for a broad verified-registry allowlist as the first fix; that is too coarse for a supply-chain scanner. The safer direction is a narrow manifest-context refinement with regression coverage after maintainer/security sign-off.
