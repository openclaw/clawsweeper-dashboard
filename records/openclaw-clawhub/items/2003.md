## Summary

Keep open. ClawHub cleanup policy does not auto-close issues, and current main still contains the manifest raw-IP heuristic that can produce the reported `suspicious.install_untrusted_source` result for docker-compose YAML containing loopback URLs.

## What This Changes

Refine ClawHub's static analyzer so docker-compose-based skills do not get false-positive `suspicious.install_untrusted_source` results for expected local compose metadata or trusted container usage.

## Best Possible Solution

Codify a narrow manifest-scanner policy that exempts expected loopback compose service metadata while preserving warnings for URL shorteners, public raw-IP install sources, mutable remote compose fetches, and weak container provenance.

## Reproduction Assessment

Yes. Source inspection shows current main applies `RAW_IP_URL_PATTERN` to `.yml` manifests, and the live 4.1.22 scan reports `suspicious.install_untrusted_source` while the compose file contains matching `127.0.0.1` URLs.

## Solution Assessment

No for a broad verified-registry allowlist as the first fix; that would be too coarse for a supply-chain scanner. The safer path is a narrow manifest-context refinement with regression coverage after maintainer/security sign-off.
