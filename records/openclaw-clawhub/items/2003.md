## Summary

Keep open: the public scan data confirms a current v2.4.22 static-scan false positive for InvestorClaw, and the evidence points to loopback raw-IP URLs in compose/config manifests rather than the GHCR image line.

## What This Changes

The issue asks to refine ClawHub's static analyzer so docker-compose skills are not mislabeled as `suspicious.install_untrusted_source` when they use bundled compose files and public container images.

## Best Possible Solution

Narrow the manifest raw-IP heuristic so localhost/loopback service endpoints used by local compose/MCP configs are not treated as untrusted install sources, while preserving warnings for public raw-IP and shortener-based install sources.

## Reproduction Assessment

Yes. The public ClawHub scan endpoint shows the failing `install_untrusted_source` result for InvestorClaw 4.1.22, and current source shows the deterministic raw-IP manifest heuristic that matches its loopback URLs.

## Solution Assessment

Yes, with a narrower target than the issue hypothesis. The best first fix is to exempt localhost/loopback manifest endpoints and add regression coverage; broader verified-registry allowlisting can be considered separately if still needed.
