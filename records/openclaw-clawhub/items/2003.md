## Summary

Keep open: current main still has a source-reproducible moderation-scanner false-positive path, but tuning `install_untrusted_source` is security-sensitive and should get maintainer/security approval before implementation.

## What This Changes

The issue requests tuning ClawHub's static moderation analyzer so docker-compose based skills are not flagged as `suspicious.install_untrusted_source` for trusted or local compose install patterns.

## Best Possible Solution

Refine manifest scanning to exempt localhost and loopback install endpoints while preserving public raw-IP and URL-shortener warnings, with focused regression tests for manifest YAML and existing code-path behavior.

## Reproduction Assessment

Yes, by source inspection rather than an executed failing test. A YAML/JSON/TOML manifest containing `http://127.0.0.1:<port>/...` still matches the shared raw-IP regex and flows through `scanManifestFile` to `suspicious.install_untrusted_source`.

## Solution Assessment

Yes, but narrower than the issue's registry allowlist suggestion. A loopback-aware manifest matcher is the most maintainable repair unless maintainers explicitly want a broader verified-registry policy or new metadata field.
