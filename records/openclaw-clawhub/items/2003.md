## Summary

Keep open: current main and the public moderation endpoint still support a real static-scan false positive, with the strongest evidence pointing to loopback raw-IP URLs in bundled manifest files rather than the GHCR image line. Because this tunes a security moderation heuristic, it should get maintainer/security approval before implementation.

## What This Changes

The issue asks ClawHub to reduce false-positive `suspicious.install_untrusted_source` findings for docker-compose-based skills that use bundled compose files and public registry images.

## Best Possible Solution

Update the moderation engine to distinguish localhost/loopback manifest URLs from public raw-IP or shortener install sources, and add regression tests for both allowed local endpoints and still-flagged remote sources.

## Reproduction Assessment

Yes. The public moderation endpoint still reports `suspicious.install_untrusted_source` for InvestorClaw, and current main deterministically applies the manifest raw-IP heuristic to the loopback URLs present in the published files.

## Solution Assessment

Yes, with a narrower fix than the issue hypothesis. Exempting loopback/localhost manifest URLs while preserving public raw-IP and shortener warnings is more maintainable than adding a trusted-registry allowlist or new installation-type config first.
