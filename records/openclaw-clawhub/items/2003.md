## Summary

Keep open: this looks like a real static-scan false positive, but ClawHub policy keeps issues open and the fix changes security moderation heuristics that should get maintainer/security approval.

## What This Changes

The issue asks for the static analyzer to stop classifying docker-compose-based skills with trusted container-image pulls as `suspicious.install_untrusted_source`.

## Best Possible Solution

Update the moderation engine to exempt localhost/loopback manifest URLs while preserving public raw-IP and shortener warnings, with regression tests for JSON/YAML/TOML manifest files.

## Reproduction Assessment

Yes. The public moderation endpoint shows `suspicious.install_untrusted_source` for InvestorClaw, and current source deterministically applies the manifest raw-IP heuristic to loopback URLs in the published files.

## Solution Assessment

Yes, but narrower than the issue hypothesis. Exempting loopback/localhost manifest URLs is more maintainable than adding a verified-registry allowlist or new installation-type metadata first.
