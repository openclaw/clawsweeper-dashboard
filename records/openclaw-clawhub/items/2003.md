## Summary

Keep open: the GHCR image-pull hypothesis is not supported by current scanner source, but there is still a source-level false-positive path for manifest-local loopback raw-IP URLs, and changing that moderation heuristic needs maintainer/security approval.

## What This Changes

The issue requests static moderation scanner tuning so docker-compose based skills are not incorrectly reported as `suspicious.install_untrusted_source`.

## Best Possible Solution

Refine manifest scanning to exempt localhost/loopback install endpoints while preserving public raw-IP and shortener warnings, with regression tests and maintainer/security approval.

## Reproduction Assessment

Partly: source inspection gives a clear reproduction path for YAML/JSON/TOML manifests containing `http://127.0.0.1:<port>/...`, which still maps to `suspicious.install_untrusted_source`. The exact GHCR image-pull theory is not reproduced by current main source or the latest public InvestorClaw files.

## Solution Assessment

Yes, but narrower than the issue hypothesis. A loopback-aware manifest matcher is the maintainable fix unless maintainers explicitly choose a broader verified-registry allowlist or new installation metadata policy.
