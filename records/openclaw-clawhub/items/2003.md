## Summary

Keep open: the report is valid enough to route, but current main still has the manifest raw-IP/shortener heuristic and changing Docker Compose/container trust treatment is a security-policy decision rather than conservative cleanup.

## What This Changes

The issue asks ClawHub to reduce or clarify `suspicious.install_untrusted_source` false positives for docker-compose-based skills that pull public registry images.

## Best Possible Solution

Define a narrow compose-manifest scanner policy that permits expected loopback service metadata while preserving warnings for URL shorteners, public raw-IP install sources, remote mutable compose fetches, mutable image tags, and weak container provenance.

## Reproduction Assessment

Yes. Current source plus the live scan API provide a high-confidence path: the affected version still reports `suspicious.install_untrusted_source`, and its bundled compose manifest contains loopback URLs that match the current manifest raw-IP rule.

## Solution Assessment

Unclear. A broad verified-registry allowlist or new metadata knob is a security-policy choice; the safer maintainable direction is a narrower compose-manifest exception plus explicit container provenance rules.
