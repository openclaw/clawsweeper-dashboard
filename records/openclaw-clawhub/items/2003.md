## Summary

Keep open: this is a source-reproducible moderation false-positive class, but it touches a security scanner heuristic and should get maintainer/security approval before implementation.

## What This Changes

Request to reduce false positives from the static moderation analyzer for docker-compose skill bundles that pull public container images and expose localhost compose endpoints.

## Best Possible Solution

Refine the manifest scanner to exempt localhost and loopback install endpoints while preserving warnings for public raw-IP URLs and shorteners; consider verified-registry allowlists only as a separate policy decision.

## Reproduction Assessment

Yes, by source inspection. A `.yaml` or `.yml` manifest containing `http://127.0.0.1:<port>/...` flows through `scanManifestFile`, matches `RAW_IP_URL_PATTERN`, and produces `suspicious.install_untrusted_source`.

## Solution Assessment

No for the broad allowlist/metadata direction as the first fix. The narrower maintainable solution is a loopback-aware manifest matcher with regression tests and explicit security review.
