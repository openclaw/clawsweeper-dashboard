## Summary

Keep open: current main still contains a source-reproducible false-positive path in the static moderation manifest scanner, but the exact public InvestorClaw latest version has moved on and scanner policy changes need maintainer/security approval.

## What This Changes

The issue requests refining static moderation so docker-compose skills with local compose manifests and trusted or pinned container images are not flagged as `suspicious.install_untrusted_source`.

## Best Possible Solution

Add a narrow, test-backed moderation change that exempts loopback/localhost manifest URLs while preserving public raw-IP and URL-shortener warnings.

## Reproduction Assessment

Yes for the source path: a manifest file containing `http://127.0.0.1/...` matches `RAW_IP_URL_PATTERN` and flows into `scanManifestFile`. I did not reproduce the exact live InvestorClaw latest-version state because it now reports only `suspicious.llm_suspicious`.

## Solution Assessment

No to a broad GHCR allowlist or new installation-type config as the first fix. A narrower loopback/localhost manifest exception with regression tests is the maintainable path, pending maintainer/security approval.
