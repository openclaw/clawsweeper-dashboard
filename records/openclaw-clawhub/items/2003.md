## Summary

Keep open: the report describes a plausible moderation-scanner false-positive class, but current source points to manifest raw-IP/loopback URLs rather than GHCR image pulls, and tuning this security heuristic needs maintainer/security approval.

## What This Changes

The issue asks ClawHub to reduce false positives from `suspicious.install_untrusted_source` for docker-compose skill publishes that use bundled compose files and public registry images.

## Best Possible Solution

Refine the manifest heuristic to exempt loopback/local install endpoints while preserving warnings for public raw-IP URLs and URL shorteners, with regression coverage for manifest YAML.

## Reproduction Assessment

Yes, by source inspection. A YAML/JSON/TOML manifest containing `http://127.0.0.1:<port>/...` still matches `RAW_IP_URL_PATTERN` and flows through `scanManifestFile` to `suspicious.install_untrusted_source`, though I did not execute the scanner against a fixture.

## Solution Assessment

Yes, but narrower than the issue hypothesis. A loopback-aware manifest matcher is more maintainable than a verified-registry allowlist or new `installation_type` metadata field unless maintainers explicitly choose that product policy.
