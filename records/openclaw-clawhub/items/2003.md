## Summary

Keep open: current main still contains the manifest rule that can emit `suspicious.install_untrusted_source` for raw-IP URLs in YAML, the affected live scan still reports that static reason, and changing Docker Compose/container provenance trust needs maintainer security-policy review.

## What This Changes

The issue asks ClawHub to refine or better classify the static scanner warning for docker-compose-based skills that use bundled compose files and public registry images.

## Best Possible Solution

Define a narrow compose-manifest scanner policy that permits expected loopback service metadata while preserving warnings for URL shorteners, remote compose fetches, public raw-IP install sources, mutable tags, and weak container provenance.

## Reproduction Assessment

Yes. Source inspection shows the raw-IP manifest rule that can emit this code, and the live scan API still reports `suspicious.install_untrusted_source` for `investorclaw` 4.1.22 with loopback URLs in the fetched compose file.

## Solution Assessment

Unclear. The reporter's broad verified-registry allowlist is not clearly the safest solution; a narrower compose-manifest exception plus explicit container provenance rules would be more maintainable.
