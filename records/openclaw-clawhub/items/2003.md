## Summary

Keep open: the report has a clear source-level false-positive path in the current static moderation scanner, although the likely trigger is loopback raw-IP URLs in manifest YAML rather than the GHCR image directive itself.

## What This Changes

The issue requests refining static moderation so docker-compose-based skills with bundled compose files are not flagged as `suspicious.install_untrusted_source` solely for expected container/local-service setup.

## Best Possible Solution

Refine the manifest scanner to distinguish loopback compose/MCP service URLs from public raw-IP or shortener install sources, add regression coverage for compose manifests, and then rescan affected versions if maintainers agree.

## Reproduction Assessment

Yes. Current main shows a deterministic source path: manifest YAML containing `http://127.0.0.1...` matches `RAW_IP_URL_PATTERN` and produces `suspicious.install_untrusted_source`; the live InvestorClaw 4.1.22 scan confirms that reason on engine v2.4.22.

## Solution Assessment

Unclear as proposed. The GHCR verified-namespace allowlist is not supported as the narrowest source fix; a safer first fix is to handle loopback/local compose endpoints separately while preserving warnings for public raw-IP and shortener install sources.
