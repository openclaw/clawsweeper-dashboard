## Summary

Keep open. Current main still has a manifest-wide raw-IP URL rule that can emit `suspicious.install_untrusted_source`, and the public InvestorClaw 4.1.22 scan still shows that exact static finding; changing this rule is a scanner trust-policy decision, not conservative cleanup.

## What This Changes

The issue asks ClawHub to refine the static analyzer so docker-compose-based skills are not broadly flagged as `suspicious.install_untrusted_source` for expected compose images or benign compose metadata.

## Best Possible Solution

Define a narrow compose-aware scanner policy that permits expected loopback service metadata while preserving warnings for URL shorteners, public raw-IP install sources, remote compose fetches, mutable image tags, and weak container provenance.

## Reproduction Assessment

Yes. Source inspection shows the current manifest scanner will emit `suspicious.install_untrusted_source` for dotted-quad HTTP(S) URLs in YAML/JSON/TOML, and the public affected version still exposes that finding alongside loopback compose metadata.

## Solution Assessment

Unclear until maintainers set scanner policy. A broad verified-registry allowlist is not the narrowest safe fix; a compose-context exception for loopback metadata plus explicit container provenance checks is safer.
