## Summary

Keep open: this is a valid static-scanner false-positive class, but current main still emits the reason for loopback raw-IP URLs in manifests. The original GHCR image-pull hypothesis is not supported by the source path or live scan evidence, and relaxing a moderation heuristic needs maintainer/security approval.

## What This Changes

The issue requests refining `suspicious.install_untrusted_source` so docker-compose skills using bundle-local compose files and trusted or pinned container images are not flagged as untrusted installs.

## Best Possible Solution

Refine the manifest heuristic to exempt loopback/local install endpoints while preserving public raw-IP and URL-shortener warnings, with regression coverage for both safe docker-compose localhost manifests and unsafe remote install sources.

## Reproduction Assessment

Yes, by source inspection and live scan evidence. Current main applies `RAW_IP_URL_PATTERN` to manifest files, and the public scan for InvestorClaw 4.1.22 reports `suspicious.install_untrusted_source` while its compose/install files contain loopback `127.0.0.1` URLs.

## Solution Assessment

Yes, but narrower than the proposed allowlist or metadata field. A loopback-aware manifest matcher plus regression tests is the maintainable path unless maintainers choose a broader verified-registry policy.
