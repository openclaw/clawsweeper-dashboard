## Summary

Keep open: ClawHub cleanup policy does not auto-close issues, and current main still supports the reported static-scan false positive rather than showing an implemented fix.

## What This Changes

The issue asks for the static analyzer to stop over-classifying docker-compose-based skills with bundled compose files and public registry images as `suspicious.install_untrusted_source`.

## Best Possible Solution

Tune the moderation engine to exempt localhost and loopback manifest URLs while preserving warnings for public raw-IP and shortener install sources, with regression tests for both allowed and still-flagged cases.

## Reproduction Assessment

Yes. The public moderation endpoint still reports `suspicious.install_untrusted_source` for the affected skill, and current main has a deterministic manifest raw-IP scan path that matches the reported loopback URL evidence.

## Solution Assessment

Yes, with a narrower fix than the issue's registry allowlist suggestion. Distinguishing loopback or localhost manifest service URLs from public raw-IP install sources is more maintainable than adding a new installation-type config first.
