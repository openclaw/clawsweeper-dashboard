## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has the manifest raw-IP rule that plausibly produces this docker-compose false positive while live ClawHub still reports `suspicious.install_untrusted_source` for the affected version.

## What This Changes

The issue asks ClawHub to refine or clarify the static analyzer behavior that reports `suspicious.install_untrusted_source` for docker-compose-based skills using public container images.

## Best Possible Solution

Define a narrow compose-manifest scanner policy that permits expected loopback service metadata while preserving warnings for unsafe install sources, mutable remote compose fetches, and weak container provenance.

## Reproduction Assessment

Yes, by source inspection plus live API evidence. Current main scans manifest YAML for raw IPv4 URLs, and the affected compose manifest contains loopback URLs while the live scan endpoint still reports `suspicious.install_untrusted_source`.

## Solution Assessment

Unclear until maintainers set scanner policy. A broad verified-registry allowlist or metadata escape hatch is riskier than a targeted compose-manifest exception paired with explicit container provenance rules.
