## Summary

Keep open. Current main still contains the static-scan path that can classify loopback URLs in YAML manifests as `suspicious.install_untrusted_source`, and public scan data confirms the affected version was flagged while a later publisher-side URL change cleared the static result; changing this rule needs maintainer/security judgment.

## What This Changes

Refine ClawHub static moderation so docker-compose skills with local compose manifests are not misclassified as `suspicious.install_untrusted_source` for expected local service metadata.

## Best Possible Solution

Codify a narrow manifest-scanner policy that exempts expected loopback compose service metadata while preserving warnings for URL shorteners, public raw-IP install sources, mutable remote compose fetches, and weak container provenance.

## Reproduction Assessment

Yes. Source inspection shows current main applies `RAW_IP_URL_PATTERN` to `.yml` manifests, and public API evidence shows InvestorClaw 4.1.22 was flagged with `suspicious.install_untrusted_source` while its compose file contained matching `127.0.0.1` URLs.

## Solution Assessment

No for a broad verified-registry allowlist as the first fix; that is too coarse for a supply-chain scanner. The safer direction is a narrow manifest-context refinement with regression coverage after maintainer/security sign-off.
