## Summary

Keeping this open for maintainer/security follow-up: the report is specific, but current main does not show a docker-compose/GHCR path for `suspicious.install_untrusted_source`, and ClawHub policy keeps issues open rather than auto-closing them.

## What This Changes

The issue asks ClawHub to refine static moderation so docker-compose skills pulling public verified-registry container images are not classified as `suspicious.install_untrusted_source` without clearer provenance risk.

## Best Possible Solution

Confirm the production scan evidence for the affected versions, then codify a narrow scanner policy and regression test that treats bundle-local compose images distinctly from URL-shortener/raw-IP install sources while preserving provenance warnings for risky container pulls.
