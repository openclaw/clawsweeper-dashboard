## Summary

Keep open. Current main still has a deterministic manifest scan path that can flag YAML compose metadata containing raw IPv4 URLs as `suspicious.install_untrusted_source`, and the public affected version still shows that static finding; changing this rule is scanner trust policy, not conservative cleanup.

## What This Changes

Request to refine ClawHub's static scanner so docker-compose-based skills with local compose metadata and trusted container images are not over-flagged as `install_untrusted_source`.

## Best Possible Solution

Define a narrow, compose-aware static scanner policy for local service metadata while preserving warnings for URL shorteners, public raw-IP install sources, mutable remote compose fetches, and weak container provenance.

## Reproduction Assessment

Yes, with high confidence from source inspection and the public affected version. Current main applies the raw-IP manifest rule to YAML files, and InvestorClaw 4.1.22 still returns `suspicious.install_untrusted_source` on the public scan endpoint.

## Solution Assessment

No for a broad verified-registry allowlist as the first fix; that would be too coarse for a supply-chain scanner. A narrower compose-manifest policy, approved by maintainers and covered by scanner tests, is the safer direction.
