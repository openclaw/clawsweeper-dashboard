## Summary

Keep open: source inspection confirms a real static-scanner false-positive path for loopback raw-IP URLs in manifest files, but the GHCR image-pull hypothesis is not supported by the current scanner. Because this touches an abuse-detection heuristic, it needs maintainer/security approval before implementation.

## What This Changes

Refine ClawHub static moderation so docker-compose-based skills are not misclassified as `suspicious.install_untrusted_source` when the install surface is local and intentional.

## Best Possible Solution

Refine the manifest scanner to exempt localhost/loopback install endpoints while preserving public raw-IP and URL-shortener warnings, with regression coverage for both allowed and flagged cases.

## Reproduction Assessment

Yes, by source inspection. A YAML/JSON/TOML manifest containing `http://127.0.0.1:<port>/...` still matches `RAW_IP_URL_PATTERN` and flows through `scanManifestFile` to `suspicious.install_untrusted_source`.

## Solution Assessment

Yes, but narrower than the issue’s GHCR-image hypothesis. A loopback-aware manifest matcher is the most maintainable fix unless maintainers explicitly want a broader registry-trust policy or new metadata field.
