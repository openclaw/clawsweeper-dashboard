## Summary

Keep open: ClawHub issues are not cleanup-close candidates, and current main still has a source-visible scanner path that can flag bundled docker-compose manifests for loopback raw-IP URLs under `suspicious.install_untrusted_source`. The report remains actionable, but changing this rule affects install-source and container provenance policy.

## What This Changes

The issue asks ClawHub to refine static moderation so docker-compose-based skills are not broadly flagged as `suspicious.install_untrusted_source` when they use bundled compose files and public registry images.

## Best Possible Solution

Define a narrow compose-manifest scanner policy that permits expected local service metadata while preserving warnings for unsafe remote install sources and weak container provenance.

## Reproduction Assessment

Yes, by source inspection plus live API evidence. Current main scans manifest files for raw IPv4 HTTP URLs, and the affected published compose manifest contains loopback URLs while the live static scan still reports `suspicious.install_untrusted_source`.

## Solution Assessment

Unclear until maintainers set scanner policy. A targeted compose-manifest exception paired with explicit container provenance rules is safer than a broad verified-registry allowlist or metadata escape hatch.
