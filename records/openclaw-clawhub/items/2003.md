## Summary

Keep open: this remains a valid moderation-scanner false-positive class, but current main points to the manifest raw-IP matcher rather than GHCR image directives, and changing that heuristic needs maintainer/security approval.

## What This Changes

The issue requests a less noisy `suspicious.install_untrusted_source` heuristic for docker-compose-based skills that pull public registry images and use bundled compose/install manifests.

## Best Possible Solution

Refine the manifest scanner to exempt localhost and loopback URLs while preserving public raw-IP and URL-shortener warnings, then add regression tests for both clean and suspicious manifest cases.

## Reproduction Assessment

Yes, by source inspection. A YAML/JSON/TOML manifest containing `http://127.0.0.1:<port>/...` still flows through `scanManifestFile` and matches `RAW_IP_URL_PATTERN`, producing `suspicious.install_untrusted_source`.

## Solution Assessment

Yes for the narrow loopback-aware matcher; that is more maintainable than a verified-registry allowlist or new `installation_type` field unless maintainers explicitly choose that product policy.
