## Summary

Keep open: current source and the public moderation endpoint still support a real static-scan false positive, with the strongest evidence pointing to loopback raw-IP URLs in bundled compose/config manifests rather than the GHCR image line.

## What This Changes

The issue asks ClawHub to reduce `suspicious.install_untrusted_source` false positives for docker-compose-based skills that use bundled compose files and public container images.

## Best Possible Solution

Update the moderation engine to distinguish localhost/loopback service endpoints from public raw-IP or shortener install sources, and cover both paths with regression tests.

## Reproduction Assessment

Yes. The public moderation endpoint shows `suspicious.install_untrusted_source` for InvestorClaw 4.1.22, and current source deterministically applies the raw-IP manifest heuristic to the loopback URLs present in the published files.

## Solution Assessment

Yes, with a narrower fix than the issue hypothesis. Exempting loopback/localhost manifest URLs while preserving public raw-IP and shortener warnings is more maintainable than adding a trusted-registry allowlist or new installation-type config first.
