## Summary

Keep open: current main still has a source-reproducible false-positive path for manifest raw-IP URLs, but the reported GHCR image-pull hypothesis is not supported by the deterministic scanner rule. Because this changes a security moderation heuristic and ClawHub policy keeps issues open, it needs maintainer/security judgment.

## What This Changes

The issue requests reducing `suspicious.install_untrusted_source` false positives for docker-compose-based skills that pull container images from public registries.

## Best Possible Solution

Have a maintainer/security reviewer approve a narrow scanner policy, likely a loopback-aware manifest exception with regression tests that preserve warnings for public raw-IP URLs and URL shorteners.

## Reproduction Assessment

Partially: yes by source inspection for manifest raw-IP or loopback URLs, which still flow to `suspicious.install_untrusted_source`. I did not find a current-main deterministic rule that directly flags GHCR `image:` directives.

## Solution Assessment

Unclear without maintainer/security policy input. A narrow loopback/private-address distinction appears safer than verified-registry allowlists or a new `installation_type` field unless maintainers explicitly want that product direction.
