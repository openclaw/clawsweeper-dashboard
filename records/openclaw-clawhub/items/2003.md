## Summary

Keep open: current main still has a source-reproducible false-positive path for `suspicious.install_untrusted_source`, but the evidence points to loopback raw-IP URLs in manifest files rather than GHCR image pulls. Because this is a moderation/security heuristic, the narrow scanner change needs maintainer approval before implementation.

## What This Changes

The issue asks ClawHub's static analyzer to stop treating docker-compose skills with local compose manifests and verified/public container image pulls as `suspicious.install_untrusted_source`.

## Best Possible Solution

Refine the manifest heuristic to exempt localhost and loopback install endpoints while preserving warnings for public raw-IP URLs and URL shorteners, then add regression tests for both allowed and blocked cases.

## Reproduction Assessment

Yes, by source inspection. A `.yaml`, `.yml`, `.json`, or `.toml` file containing an HTTP loopback IPv4 URL still matches `RAW_IP_URL_PATTERN` in `scanManifestFile` and emits `suspicious.install_untrusted_source`, although the live latest InvestorClaw version is currently clean.

## Solution Assessment

Yes, but narrower than the issue hypothesis. A loopback-aware manifest matcher is the maintainable fix supported by current code evidence; verified-registry allowlists or a new `installation_type` field would be broader product policy changes.
