## Summary

Keep open: ClawHub issue cleanup policy does not auto-close issues, and current main still has a manifest raw-IP rule that can explain the docker-compose `install_untrusted_source` false-positive class.

## What This Changes

Refine the static scanner so docker-compose skills with local compose files and expected loopback/container metadata are not mislabeled as `suspicious.install_untrusted_source`.

## Best Possible Solution

Define a narrow compose-manifest scanner policy that permits expected loopback service metadata while preserving warnings for unsafe install sources, mutable remote compose fetches, and weak container provenance.

## Reproduction Assessment

Yes, by source inspection plus live API evidence. Current main still scans YAML/TOML/JSON manifests for raw IPv4 URLs, and the affected compose manifest contains loopback URLs while the live static scan reports `suspicious.install_untrusted_source`.

## Solution Assessment

Unclear until maintainers set scanner policy. A targeted compose-manifest exception plus explicit container provenance checks is safer than a broad verified-registry allowlist or metadata escape hatch.
