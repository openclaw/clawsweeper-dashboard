## Summary

Keep this open for maintainer/security follow-up. Current main and public scan data support a real over-fire path, but the evidence points to loopback raw-IP URLs in docker-compose manifests rather than the GHCR image reference, and ClawHub issue policy keeps issues open.

## What This Changes

The issue reports that ClawHub's static analyzer flags docker-compose-based skills using GHCR images as `suspicious.install_untrusted_source`.

## Best Possible Solution

Confirm the stored static evidence, then codify a narrow scanner policy and regression tests that distinguish local docker-compose service endpoints from remote install sources while preserving warnings for URL shorteners, raw public-IP downloads, mutable remote compose fetches, and unpinned container provenance.

## Reproduction Assessment

Yes by source inspection and production scan evidence. Current main flags raw-IP URLs in manifest files, and the affected public scan for InvestorClaw 4.1.22 shows `suspicious.install_untrusted_source` with available context pointing to loopback `127.0.0.1` URLs rather than the GHCR image line.

## Solution Assessment

No for the registry allowlist as written; it is too broad for a supply-chain scanner. The safer direction is a narrow manifest-context refinement, backed by tests for loopback compose metadata, GHCR image references, raw public-IP URLs, and shorteners, after maintainer/security approval.
