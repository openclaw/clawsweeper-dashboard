## Summary

Keep open. Current main still has a source-level path that can classify loopback raw-IP URLs in YAML manifests as `suspicious.install_untrusted_source`, and public scan data confirms the affected InvestorClaw version was flagged under engine `v2.4.22`; changing this scanner behavior is security policy work for maintainers.

## What This Changes

The issue asks ClawHub to reduce false positives from `suspicious.install_untrusted_source` for docker-compose-based skills that use local compose files and public container images.

## Best Possible Solution

Codify a narrow manifest-scanner policy that exempts expected loopback compose service metadata while preserving warnings for URL shorteners, public raw-IP install sources, mutable remote compose fetches, and weak container provenance.

## Reproduction Assessment

Yes. Source inspection shows current main applies `RAW_IP_URL_PATTERN` to `.yml` manifests, and public scan data shows InvestorClaw 4.1.22 was flagged with `suspicious.install_untrusted_source` while its compose file contained matching `127.0.0.1` service URLs.

## Solution Assessment

No for a broad verified-registry allowlist as the first fix; that is too coarse for a supply-chain scanner. The safer direction is a narrow manifest-context refinement with regression coverage after maintainer/security sign-off.
