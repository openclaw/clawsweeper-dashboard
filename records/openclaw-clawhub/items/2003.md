## Summary

Keep open: ClawHub cleanup policy keeps issues open, and current main still has a manifest raw-IP heuristic that can match loopback Docker Compose service metadata. Adjusting this rule affects scanner security policy and container supply-chain trust, so it needs maintainer judgment before implementation.

## What This Changes

Refine the static scanner so docker-compose-based skills with bundled compose files and registry images are not misclassified as `suspicious.install_untrusted_source`, or are classified more precisely.

## Best Possible Solution

Define a narrow compose-manifest policy that permits expected loopback service metadata while preserving warnings for URL shorteners, public raw-IP install sources, remote mutable compose fetches, and weak container provenance, then add regression tests and a spec note.

## Reproduction Assessment

Yes, by source inspection plus live API evidence. Current main scans every YAML manifest for raw IPv4 URLs, the affected published compose file contains loopback URLs, and the live scan still reports `suspicious.install_untrusted_source`.

## Solution Assessment

Unclear until maintainers set scanner policy. A targeted compose-manifest exception with explicit container provenance rules is safer than a broad verified-registry allowlist or publisher-declared escape hatch.
