## Summary

Keep open: this is a ClawHub issue, and current main plus the live affected artifact still support the reported false-positive shape. Resolving Docker Compose, loopback manifest URLs, and container provenance is a security scanner policy decision rather than conservative cleanup.

## What This Changes

The issue asks ClawHub to reduce over-broad `suspicious.install_untrusted_source` findings for docker-compose-based skills while preserving useful install-source warnings.

## Best Possible Solution

Define a narrow compose-manifest scanner policy that permits expected loopback service metadata while preserving explicit warnings for unsafe install sources and weak container provenance.

## Reproduction Assessment

Yes. Current source still applies the raw-IP manifest rule to YAML files, and the live affected 4.1.22 artifact still reports `suspicious.install_untrusted_source` while its `compose.yml` contains loopback URLs that match that rule.

## Solution Assessment

Unclear. A broad verified-registry allowlist or new metadata knob is a security-policy choice; the safer maintainable direction is a narrower compose-manifest exception plus explicit container provenance rules.
