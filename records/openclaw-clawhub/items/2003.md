## Summary

Keep open. Current main still has the manifest raw-IP path that can emit `suspicious.install_untrusted_source`, public InvestorClaw 4.1.22 still shows that static reason from engine `v2.4.22`, and ClawHub policy for this cleanup pass keeps issues open rather than auto-closing them.

## What This Changes

The issue asks ClawHub to reduce `suspicious.install_untrusted_source` false positives for docker-compose skills, especially local compose files with GHCR images and loopback service URLs.

## Best Possible Solution

Approve and codify a narrow manifest-scanner rule for expected loopback compose service metadata while preserving warnings for URL shorteners, public raw-IP install sources, mutable remote compose fetches, and weak container provenance.

## Reproduction Assessment

Yes. Current main applies `RAW_IP_URL_PATTERN` to YAML manifests, and the public InvestorClaw 4.1.22 compose file contains `127.0.0.1` URLs while its scan result reports `suspicious.install_untrusted_source` from static engine `v2.4.22`.

## Solution Assessment

Unclear until maintainers approve the scanner policy. A broad verified-registry allowlist is not the best first fix; a narrow context-aware loopback manifest exception is safer.
