## Summary

Keep open. Current main still has the manifest rule that can emit `suspicious.install_untrusted_source` for YAML files containing raw-IP URLs, the public affected version reproduces the static finding, and changing this scanner heuristic is security-policy work rather than conservative cleanup.

## What This Changes

Request to reduce false-positive `suspicious.install_untrusted_source` results for docker-compose skill bundles that use local compose files and public container images.

## Best Possible Solution

Define a narrow, context-aware docker-compose manifest policy that handles loopback service metadata safely while preserving warnings for shorteners, public raw-IP install sources, mutable remote compose fetches, and weak container provenance.

## Reproduction Assessment

Yes. Current source inspection shows the manifest scanner applies the raw-IP rule to YAML manifests, and the public 4.1.22 scan/file endpoints show `suspicious.install_untrusted_source` alongside loopback compose metadata.

## Solution Assessment

No for a broad verified-registry allowlist as the first fix; that is too coarse for a supply-chain scanner. A narrower compose-manifest policy with maintainer security approval is the safer direction.
