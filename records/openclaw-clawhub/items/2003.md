## Summary

Keep open: current main still has a source-reproducible moderation false-positive path, but the evidence points to loopback raw-IP manifest URLs rather than GHCR image pulls, and tuning this security heuristic needs maintainer/security approval.

## What This Changes

The issue asks for the static moderation analyzer to stop classifying benign docker-compose skill installs as `suspicious.install_untrusted_source`.

## Best Possible Solution

Refine manifest raw-IP detection to exempt loopback/local service endpoints used by compose installs while preserving warnings for public raw-IP and URL-shortener install sources, with regression tests for both paths.

## Reproduction Assessment

Yes. Source inspection and a regex probe show that a YAML manifest containing `http://127.0.0.1:<port>/...` still flows through `scanManifestFile` to `suspicious.install_untrusted_source`, while GHCR `image:` strings do not match that path.

## Solution Assessment

Yes, but narrower than the issue hypothesis. A loopback-aware manifest matcher is the maintainable fix boundary unless maintainers explicitly choose a verified-registry allowlist or new docker-compose metadata policy.
