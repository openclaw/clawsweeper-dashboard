## Summary

Keep open: ClawHub cleanup policy keeps issues open, and current main still contains the manifest raw-IP heuristic that can emit `suspicious.install_untrusted_source` for docker-compose YAML with loopback service URLs. The report is source-reproducible, but changing compose/container trust classification is security-policy-sensitive and needs maintainer direction.

## What This Changes

The reporter asks ClawHub to refine static moderation so docker-compose-based skills with bundled compose files and public registry images are not misclassified as `suspicious.install_untrusted_source`.

## Best Possible Solution

Define a narrow compose-manifest scanner policy that permits expected loopback service metadata while preserving warnings for unsafe install sources and weak container provenance, then add regression coverage and a security spec note.

## Reproduction Assessment

Yes, by source inspection plus live API evidence. Current main scans manifest YAML for raw IPv4 URLs, and the published affected compose file contains loopback URLs while the live scan still reports `suspicious.install_untrusted_source`.

## Solution Assessment

Unclear until maintainers set scanner policy. A targeted compose-manifest exception with explicit container provenance rules is safer than a broad verified-registry allowlist or a metadata escape hatch.
