## Summary

Keep open. Current main still contains the manifest raw-IP heuristic that can classify loopback compose metadata as `suspicious.install_untrusted_source`, and fixing that path changes scanner trust policy.

## What This Changes

The issue asks ClawHub to reduce false `suspicious.install_untrusted_source` findings for docker-compose skill bundles that use local compose files and public registry images.

## Best Possible Solution

Codify a narrow manifest-scanner policy that exempts expected loopback compose service metadata while preserving warnings for URL shorteners, public raw-IP install sources, mutable remote compose fetches, and weak container provenance.

## Reproduction Assessment

Yes. Current main applies `RAW_IP_URL_PATTERN` to YAML manifests, and the public InvestorClaw 4.1.22 compose file contains `127.0.0.1` URLs while its version security data reports `suspicious.install_untrusted_source` from static engine `v2.4.22`.

## Solution Assessment

No for a broad verified-registry allowlist as the first fix; that is too coarse for a supply-chain scanner. The safer direction is a narrow manifest-context refinement after maintainer/security approval.
