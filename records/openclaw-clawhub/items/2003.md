## Summary

Keep open: this is a valid moderation-scanner false-positive class, but current evidence points to manifest raw-IP loopback URLs rather than GHCR image pulls. Because changing this heuristic affects abuse detection, it needs maintainer/security approval before implementation.

## What This Changes

The issue asks ClawHub to reduce false positives where docker-compose-style skill manifests are flagged as `suspicious.install_untrusted_source`.

## Best Possible Solution

Refine the manifest heuristic to exempt localhost/loopback install endpoints while preserving warnings for public raw-IP URLs and URL shorteners, with regression tests for both cases.

## Reproduction Assessment

Yes, by source inspection. A YAML/JSON/TOML manifest containing `http://127.0.0.1:<port>/...` still matches `RAW_IP_URL_PATTERN` and flows through `scanManifestFile` to `suspicious.install_untrusted_source`, although the latest public InvestorClaw version no longer shows that reason code.

## Solution Assessment

Yes, but narrower than the issue hypothesis. A loopback-aware manifest matcher is more maintainable than a verified-registry allowlist or new `installation_type` metadata field unless maintainers explicitly want that product policy.
