## Summary

Keep open: this is a valid scanner false-positive class that affects security moderation heuristics, and ClawHub issue policy keeps issues open for maintainer judgment. Current source supports a loopback/raw-IP manifest path for `suspicious.install_untrusted_source`, but not the broader GHCR image-pull hypothesis.

## What This Changes

Request to reduce false-positive `suspicious.install_untrusted_source` findings for docker-compose-style skills that use local compose files and public container images.

## Best Possible Solution

Refine the manifest heuristic to exempt localhost/loopback install endpoints while preserving warnings for public raw-IP URLs and URL shorteners, with regression tests for both safe and unsafe cases.

## Reproduction Assessment

Yes, by source inspection for the loopback manifest variant: a `.yml`/`.yaml` manifest containing `http://127.0.0.1:<port>/...` still matches `RAW_IP_URL_PATTERN` and flows to `suspicious.install_untrusted_source`. The original GHCR image-pull hypothesis is not supported by the current static scanner code I found.

## Solution Assessment

Unclear for the suggested verified-registry allowlist; the source-backed narrow fix is a loopback-aware manifest matcher. Because this is a security moderation heuristic, maintainer/security approval should come before implementation.
