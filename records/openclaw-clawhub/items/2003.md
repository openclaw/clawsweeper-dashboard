## Summary

Keep open. Current main still has a manifest static-scan path that can classify loopback URLs in YAML compose metadata as `suspicious.install_untrusted_source`, and public scan data confirms the affected InvestorClaw version hit that code while later publisher-side changes only worked around it.

## What This Changes

Refine ClawHub static analyzer handling for docker-compose skills so expected compose metadata is not reported as an untrusted install source.

## Best Possible Solution

Codify a narrow manifest-context policy that exempts expected loopback compose service metadata while preserving warnings for URL shorteners, public raw-IP install sources, mutable remote compose fetches, and weak container provenance.

## Reproduction Assessment

Yes. Source inspection shows current main applies `RAW_IP_URL_PATTERN` to `.yml` manifests, and public API evidence shows InvestorClaw 4.1.22 was flagged while its compose file contained matching `127.0.0.1` URLs.

## Solution Assessment

No for the broad verified-registry allowlist suggested in the report; the safer path is a narrow compose-manifest refinement with regression coverage after maintainer/security sign-off.
