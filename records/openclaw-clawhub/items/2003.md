## Summary

Keep open. Current main still applies the install-source raw-IP/shortener manifest rule to YAML compose files, the public affected version still shows the reported static finding, and changing scanner trust heuristics is security-policy work rather than conservative cleanup.

## What This Changes

The issue asks ClawHub to reduce false-positive `suspicious.install_untrusted_source` findings for docker-compose skill bundles using local compose files, GHCR images, and loopback service metadata.

## Best Possible Solution

Define a maintainer-approved compose-manifest scanner policy that exempts safe local loopback metadata while preserving warnings for URL shorteners, public raw-IP install sources, mutable remote compose fetches, and weak container provenance.

## Reproduction Assessment

Yes. Current source inspection shows the YAML manifest scanner applies the raw-IP install-source rule, and the public 4.1.22 scan/file endpoints show the same reason code on a compose file containing loopback metadata.

## Solution Assessment

No for a broad verified-registry allowlist as the first fix; it is too coarse for a supply-chain scanner. A narrow compose-manifest rule with regression tests and maintainer security-policy approval is the safer path.
