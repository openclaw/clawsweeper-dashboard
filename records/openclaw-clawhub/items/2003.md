## Summary

Keep this open for maintainer/security follow-up. Current main and public scan data support a real over-fire path, but ClawHub issue policy keeps issues open and the fix changes scanner trust-boundary behavior.

## What This Changes

Request refinement of `suspicious.install_untrusted_source` so docker-compose-based skills with local service endpoints and container images are not over-flagged as untrusted installs.

## Best Possible Solution

Confirm the stored static evidence, then add a narrow scanner rule and regression tests that distinguish loopback docker-compose service metadata from remote install sources while preserving warnings for shorteners, raw public-IP downloads, mutable remote compose fetches, and unpinned container provenance.

## Reproduction Assessment

Yes. Source inspection shows current main flags raw-IP URLs in manifest files, and the affected public scan for InvestorClaw 4.1.22 shows `suspicious.install_untrusted_source` with evidence pointing to loopback compose URLs rather than the GHCR image string.

## Solution Assessment

No for the broad registry allowlist as written; verified publisher status does not prove immutability or signing. A narrower manifest-context refinement for loopback service URLs is the safer maintainable direction after maintainer/security approval.
