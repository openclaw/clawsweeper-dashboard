## Summary

Keep open: current main still has the manifest raw-IP/shortener rule that can explain this static scanner false-positive shape, and changing Docker Compose/container trust treatment is a security-policy decision rather than conservative cleanup.

## What This Changes

Request to refine ClawHub's static scanner so docker-compose-based skills are not over-classified as `suspicious.install_untrusted_source`.

## Best Possible Solution

Define a narrow compose-manifest scanner policy that permits expected loopback service metadata while preserving useful warnings for unsafe install sources and weak container provenance.

## Reproduction Assessment

Yes. Current source plus the live scan API provide a high-confidence path: the affected version still reports `suspicious.install_untrusted_source`, and its bundled compose manifest contains loopback URLs that match the current manifest raw-IP rule.

## Solution Assessment

Unclear. A broad verified-registry allowlist or new metadata knob is a security-policy choice; the safer maintainable direction is a narrower compose-manifest exception plus explicit container provenance rules.
