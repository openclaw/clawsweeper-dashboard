## Summary

Keep open. Current main still has the manifest raw-IP/shortener rule that can produce this static finding, the public affected version confirms the reported result, and scanner trust-policy changes are security-sensitive rather than conservative cleanup.

## What This Changes

The issue asks ClawHub to reduce or reclassify `suspicious.install_untrusted_source` static findings for docker-compose-based skills that use public container images and local compose metadata.

## Best Possible Solution

Define a narrow, context-aware static scanner policy for docker-compose manifests, especially loopback service metadata, while preserving warnings for URL shorteners, public raw-IP install sources, mutable remote compose fetches, and weak container provenance.

## Reproduction Assessment

Yes. Current source inspection shows the manifest scanner applies the raw-IP rule to YAML manifests, and the public 4.1.22 scan/file endpoints show `suspicious.install_untrusted_source` alongside loopback compose metadata.

## Solution Assessment

No for a broad verified-registry allowlist as the first fix; it is too coarse for a supply-chain scanner. A narrower compose-manifest policy, approved by maintainers, is the safer direction.
