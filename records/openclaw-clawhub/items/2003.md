## Summary

Keep open: current main still has a source-visible false-positive path for manifest raw-IP URLs, including loopback, but the GHCR image-pull hypothesis is not backed by the scanner source and any heuristic tuning is security-sensitive.

## What This Changes

The issue asks ClawHub to reduce false-positive `suspicious.install_untrusted_source` moderation findings for docker-compose-based skills that pull public container images.

## Best Possible Solution

Refine install-source scanning to distinguish localhost or loopback manifest endpoints from public raw-IP and shortener install sources, unless maintainers explicitly choose a broader Docker registry trust policy.

## Reproduction Assessment

Yes, by source inspection for the narrower loopback-manifest false-positive: YAML/JSON/TOML files containing `http://127.0.0.1:<port>/...` still flow to `suspicious.install_untrusted_source`. The GHCR image-pull hypothesis is not reproduced from current scanner source.

## Solution Assessment

Yes for a narrow loopback-aware manifest matcher, with maintainer/security approval. A verified-registry allowlist or new `installation_type` field would be a broader product policy change.
