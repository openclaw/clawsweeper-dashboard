## Summary

Keep open: current main and the public moderation endpoint still support a real `install_untrusted_source` false positive, but changing security scanner heuristics needs maintainer/security approval.

## What This Changes

The issue asks ClawHub to reduce false positives from the static analyzer's `suspicious.install_untrusted_source` finding for docker-compose-based skills using bundled compose/config files and public registry images.

## Best Possible Solution

Update the moderation engine to distinguish safe loopback/local manifest URLs from public raw-IP or shortener install sources, with regression coverage for both allowed local endpoints and still-flagged remote sources.

## Reproduction Assessment

Yes. The public moderation endpoint still reports `suspicious.install_untrusted_source` for InvestorClaw, and current main's manifest scanner deterministically flags raw IPv4 URLs in YAML/JSON/TOML files without a loopback exception.

## Solution Assessment

Yes, with a narrower fix than the issue hypothesis. A loopback-aware manifest scanner change is more maintainable than a broad trusted-registry allowlist or new installation-type metadata before the actual trigger is confirmed.
