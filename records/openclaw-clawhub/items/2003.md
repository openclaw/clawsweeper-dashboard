## Summary

Keep open. Current main still has a manifest-scanner path that can emit `suspicious.install_untrusted_source` for raw loopback HTTP URLs in YAML compose files, and changing that rule is security-sensitive scanner policy rather than conservative cleanup.

## What This Changes

The issue asks ClawHub to refine `install_untrusted_source` static analysis for docker-compose-based skills that use local compose files and public container images.

## Best Possible Solution

Define a narrow compose-manifest scanner policy that permits expected loopback service metadata while preserving warnings for URL shorteners, public raw-IP install sources, remote compose fetches, mutable tags, and weak container provenance.

## Reproduction Assessment

Yes. Source inspection on current main shows the manifest scanner still emits `install_untrusted_source` for raw-IP URLs in YAML, and the public affected version shows that finding alongside loopback compose metadata.

## Solution Assessment

Unclear. A broad verified-registry allowlist is not the safest fix; a narrower compose-manifest exception for expected loopback metadata plus explicit container provenance rules would be more maintainable.
