## Summary

Keep open: this issue is a current ClawHub scanner-policy bug report, and repository policy keeps issues open. Current main still has a manifest rule that can emit `suspicious.install_untrusted_source` for loopback/raw-IP URLs in YAML, while the proposed registry allowlist direction needs maintainer security judgment.

## What This Changes

Request that the static scanner stop over-flagging legitimate Docker Compose skill manifests as `suspicious.install_untrusted_source`.

## Best Possible Solution

Define a narrow compose-manifest policy that permits expected loopback service metadata while preserving warnings for URL shorteners, public raw-IP install sources, remote compose fetches, mutable tags, and weak container provenance.

## Reproduction Assessment

Yes. Source inspection and a regex check show current main still matches loopback compose URLs as raw-IP install sources, and live API output for the affected 4.1.22 release still carries the reported static reason code.

## Solution Assessment

Unclear. A broad verified-registry allowlist is not the safest fix; the maintainable path is a narrower compose-manifest exception plus explicit container provenance rules.
