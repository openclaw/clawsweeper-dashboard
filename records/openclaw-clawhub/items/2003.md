## Summary

Keep open: current main still has a source-reproducible scanner false-positive path for loopback URLs in YAML/JSON/TOML manifests, even though the latest InvestorClaw release worked around it by switching to `localhost`. Because this changes a security moderation heuristic, it needs maintainer/security approval before implementation.

## What This Changes

Refine ClawHub's static moderation scanner so docker-compose-based skills are not incorrectly labeled with `suspicious.install_untrusted_source` for benign local compose metadata.

## Best Possible Solution

Approve a narrow loopback/localhost exemption for manifest scanning, add regression tests that preserve public raw-IP and shortener warnings, and leave registry trust allowlists or new metadata fields for a separate product decision.

## Reproduction Assessment

Yes, by source inspection. A `.yml`, `.yaml`, `.json`, or `.toml` manifest containing `http://127.0.0.1:<port>/...` matches `RAW_IP_URL_PATTERN` and flows through `scanManifestFile` to `suspicious.install_untrusted_source`.

## Solution Assessment

Unclear until maintainer/security policy signs off. The safest apparent fix is narrower than the requested registry allowlist: exempt loopback/local manifest URLs while preserving public raw-IP and URL-shortener warnings.
