## Summary

Keep open: current source and live scan evidence support a real moderation false-positive class, but ClawHub policy keeps issues open and this scanner change needs maintainer/security judgment before implementation.

## What This Changes

The issue asks for the static moderation analyzer to stop reporting `suspicious.install_untrusted_source` for docker-compose skill installs that pull a known public container image.

## Best Possible Solution

Refine the manifest scanner to exempt localhost/loopback install endpoints while preserving public raw-IP and URL-shortener warnings, with regression tests and maintainer/security approval.

## Reproduction Assessment

Yes, by source inspection and public API evidence rather than a local test run. Current main still matches loopback raw-IP URLs in manifest files and emits `suspicious.install_untrusted_source`, and the public 4.1.22 InvestorClaw scan shows that exact reason code on engine v2.4.22.

## Solution Assessment

Partly. The verified-registry allowlist or new metadata field suggested in the issue is broader than the evidence supports; a narrower loopback-aware manifest matcher is the safer maintainable fix if maintainers accept the security policy tradeoff.
