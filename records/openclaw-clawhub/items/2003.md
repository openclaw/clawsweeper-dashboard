## Summary

Keep open: the report points to a real moderation-scanner false-positive class, but the concrete source path is raw-IP loopback URLs in manifest files rather than GHCR image pulls. Because this changes an abuse-detection heuristic, it needs maintainer/security approval before a fix is queued.

## What This Changes

The issue asks to reduce false positives from the static analyzer’s `install_untrusted_source` finding for docker-compose-based skills that pull public container images.

## Best Possible Solution

Refine the manifest scan to exempt localhost/loopback install endpoints while preserving warnings for public raw-IP URLs and URL shorteners, with regression tests for both paths.

## Reproduction Assessment

Yes, by source inspection and live historical scan data. InvestorClaw 4.1.22 shows `suspicious.install_untrusted_source`, and current main still matches raw-IP URLs in YAML manifests without excluding loopback addresses.

## Solution Assessment

Unclear until maintainer/security approves the policy. The narrowest technical direction is a loopback-aware manifest matcher, not a verified-registry allowlist or a new `installation_type` metadata field.
