## Summary

Keep open. Current main still applies `suspicious.install_untrusted_source` to raw-IP URLs in YAML/JSON/TOML manifests, the affected InvestorClaw 4.1.22 public scan still reports that exact static reason, and this ClawHub cleanup lane keeps issues open while scanner trust-policy changes get maintainer/security review.

## What This Changes

Request to refine ClawHub's static `install_untrusted_source` scanner for docker-compose-based skills so legitimate local compose manifests are not treated like untrusted install sources.

## Best Possible Solution

Define a narrow manifest-scanner policy that treats expected loopback compose service metadata differently while preserving warnings for URL shorteners, public raw-IP install sources, mutable remote compose fetches, and weak container provenance.

## Reproduction Assessment

Yes. Source inspection shows current main still applies `RAW_IP_URL_PATTERN` to manifest files, and the public InvestorClaw 4.1.22 scan/file endpoints show `suspicious.install_untrusted_source` alongside `127.0.0.1` compose metadata.

## Solution Assessment

No for a broad verified-registry allowlist as the first fix; it is too coarse for a supply-chain scanner. A narrower compose-aware loopback refinement, approved as scanner policy, is the safer path.
