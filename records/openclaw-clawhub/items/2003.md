## Summary

Keep open: current main still has a source-reproducible false-positive path for manifest files containing loopback raw-IP HTTP URLs, while the reported GHCR `image:` hypothesis is not supported by the deterministic scanner code. Because this tunes a moderation/security heuristic, it needs maintainer/security approval before implementation.

## What This Changes

Refine ClawHub's static moderation scanner so docker-compose-based skills using local compose files and trusted container images are not flagged as `suspicious.install_untrusted_source`.

## Best Possible Solution

Refine the manifest scanner to exempt localhost/loopback install endpoints used by local docker-compose services while preserving warnings for URL shorteners and public raw-IP install sources; defer verified-registry allowlists or new metadata fields unless maintainers explicitly choose that policy.

## Reproduction Assessment

Yes, by source inspection. A `.yml`, `.yaml`, `.json`, or `.toml` manifest containing `http://127.0.0.1:<port>/...` flows through `runStaticPublishScan` to `scanManifestFile`, matches `RAW_IP_URL_PATTERN`, and emits `suspicious.install_untrusted_source`.

## Solution Assessment

Yes, but narrower than the issue hypothesis. A loopback-aware manifest matcher is the most maintainable first fix; verified-registry allowlists or an `installation_type` field would require a broader product/security policy decision.
