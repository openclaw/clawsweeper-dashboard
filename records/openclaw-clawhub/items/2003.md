## Summary

Keep open. Current main still applies the `install_untrusted_source` static rule to raw-IP URLs in YAML/JSON/TOML manifests, the public InvestorClaw 4.1.22 scan confirms the reported static finding, and changing this rule is scanner trust-policy work rather than conservative issue cleanup.

## What This Changes

The issue asks ClawHub to reduce false positives where docker-compose-based skills are classified as `suspicious.install_untrusted_source`.

## Best Possible Solution

Define a narrow manifest-scanner policy that treats expected loopback compose service metadata differently while preserving warnings for URL shorteners, public raw-IP install sources, mutable remote compose fetches, and weak container provenance.

## Reproduction Assessment

Yes. Source inspection shows current main applies `RAW_IP_URL_PATTERN` to YAML manifests, and the public InvestorClaw 4.1.22 scan/file endpoints show `suspicious.install_untrusted_source` with `http://127.0.0.1` compose metadata.

## Solution Assessment

No for a broad verified-registry allowlist as the first fix; it is too coarse for a supply-chain scanner. A narrower context-aware compose manifest refinement, approved as scanner policy, is the safer path.
