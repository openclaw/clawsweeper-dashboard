## Summary

Keep open. Current main reduced the user-facing impact by making static suspicious findings advisory, but the `install_untrusted_source` static finding can still fire on compose YAML loopback URLs; ClawHub cleanup policy also keeps issues open for maintainer handling.

## What This Changes

The issue asks ClawHub to reduce or clarify `suspicious.install_untrusted_source` findings for docker-compose-based skills that pull public container images from trusted-looking registries.

## Best Possible Solution

Define a narrow compose-manifest scanner policy that permits expected loopback service metadata while preserving warnings for URL shorteners, public raw-IP install sources, remote compose fetches, mutable tags, and weak container provenance.

## Reproduction Assessment

Yes, from source inspection and public artifact checks. Current main still scans YAML manifests for raw-IP HTTP URLs and emits `suspicious.install_untrusted_source`, while the affected `4.1.22` compose file contains loopback `http://127.0.0.1` service metadata and the public scan shows that static finding.

## Solution Assessment

Unclear. A broad verified-registry allowlist is not the safest fix; a compose-aware exception for expected loopback metadata plus explicit container provenance rules would be more maintainable.
