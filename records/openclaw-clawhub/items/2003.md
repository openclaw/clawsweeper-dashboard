## Summary

Keep open: current main still has a source-reproducible false-positive path for `suspicious.install_untrusted_source` in manifest files, but the concrete path is raw-IP/loopback URLs rather than GHCR image directives. Because this is a security moderation heuristic, the next step should be maintainer/security review before implementation.

## What This Changes

The issue asks ClawHub to reduce false `suspicious.install_untrusted_source` static analyzer findings for docker-compose-based skills that use local compose files and verified public container images.

## Best Possible Solution

Refine the manifest scanner to exempt localhost/loopback install endpoints while preserving warnings for public raw-IP URLs and URL shorteners, with regression tests for both cases.

## Reproduction Assessment

Yes, by source inspection for a narrower path: a YAML/JSON/TOML manifest containing `http://127.0.0.1:<port>/...` still matches the raw-IP pattern and reaches `suspicious.install_untrusted_source`. The GHCR `image:` hypothesis itself is not reproduced from current scanner code.

## Solution Assessment

Yes, but narrower than the issue proposal. A loopback-aware manifest matcher is the safest maintainable fix unless maintainers explicitly want a broader verified-registry allowlist or new `installation_type` policy.
