## Summary

Keeping this open under ClawHub issue policy. The reported GHCR-image hypothesis is not supported by current source or live scan details, but there is a narrower, source-backed scanner false positive: manifest raw-IP URL detection can classify loopback MCP/dashboard URLs as `suspicious.install_untrusted_source`.

## What This Changes

The issue requests reducing false positives for docker-compose-based skills that pull public registry images so they are not treated like untrusted install-source execution.

## Best Possible Solution

Refine the static scanner so local loopback service URLs in compose/config manifests are not labeled as untrusted install sources, while keeping public raw-IP and shortener detections conservative.

## Reproduction Assessment

Yes. This is source-reproducible from `scanManifestFile`: any raw IPv4 URL in a manifest triggers `suspicious.install_untrusted_source`, and live 4.1.22 scan data confirms the stored finding while pointing at localhost/127.0.0.1 URLs rather than the GHCR image.

## Solution Assessment

No for the broad verified-registry allowlist as the first fix. The safer narrow repair is to exempt loopback/local service URLs from the raw-IP install-source heuristic and add regression tests proving public raw IPs and URL shorteners still flag.
