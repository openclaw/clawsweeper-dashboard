## Summary

Keep open. Current main still contains the manifest raw-IP heuristic that can flag loopback docker-compose metadata as `suspicious.install_untrusted_source`, and changing that scanner behavior is security policy work rather than conservative cleanup.

## What This Changes

The issue asks ClawHub to reduce false-positive `suspicious.install_untrusted_source` results for docker-compose-based skill packages.

## Best Possible Solution

Codify a narrow manifest-scanner policy that exempts expected loopback compose service metadata while preserving warnings for URL shorteners, public raw-IP install sources, mutable remote compose fetches, and weak container provenance.

## Reproduction Assessment

Yes. Source inspection shows YAML manifests are scanned with a raw-IP URL regex, and the public InvestorClaw 4.1.22 scan confirms `suspicious.install_untrusted_source` from engine `v2.4.22`; I did not run tests because this was a read-only review.

## Solution Assessment

No for a broad verified-registry allowlist as the first fix. The safer direction is a narrow context-aware exemption for loopback compose service metadata after maintainer/security approval.
