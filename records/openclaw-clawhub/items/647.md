## Summary

Keep open. This is an open ClawHub issue, and the cleanup policy only permits conservative auto-close for pull requests that current main certainly implements. Current main has OAuth callback handling and clearer blocked-account messaging, but it does not prove the reported production GitHub OAuth/session-loss behavior is fixed; several related OAuth/session-loop reports remain open and the likely root cause may involve production auth configuration or account state.

## What This Changes

The issue reports that GitHub OAuth authorization succeeds, but ClawHub returns the user to a logged-out state without persisting a session.

## Best Possible Solution

Treat this as part of a canonical OAuth/session-loop investigation: make production GitHub OAuth reliably create an active session for valid accounts, keep deleted/banned-account outcomes explicit, and add regression coverage around callback handling, user bootstrap, and signed-in UI state.
