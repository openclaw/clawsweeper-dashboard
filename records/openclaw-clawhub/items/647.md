## Summary

Keep open. This is an open ClawHub issue, and current main does not prove the reported GitHub OAuth/session-loss path is fixed; production on the reviewed SHA currently returns HTTP 500 from the GitHub sign-in endpoints while related OAuth/session-loop reports remain open.

## What This Changes

The issue reports that GitHub OAuth authorization succeeds but ClawHub returns the user to a logged-out state without establishing a session.

## Best Possible Solution

Restore reliable GitHub OAuth session creation for active accounts, preserve explicit deleted/banned-account rejection, and add focused callback/session smoke or regression coverage once the failing production layer is identified.
