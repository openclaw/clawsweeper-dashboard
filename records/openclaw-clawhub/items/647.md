## What This Changes

GitHub OAuth authorization succeeds on clawhub.ai, but the callback leaves the user logged out and repeated attempts can hit install limits.

## Best Possible Solution

Treat this as a canonical OAuth/session-loop investigation that separates active-account session creation from intentionally blocked deleted/deactivated accounts, then add targeted regression coverage for callback handling, user bootstrap, and signed-in UI state.
