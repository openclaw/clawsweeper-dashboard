## Summary

Keep open. Current main has OAuth callback handling, blocked-account messaging, and intentional deleted/deactivated-account rejection, but that does not prove the reported production GitHub OAuth session-loss behavior is fixed; multiple related login-loop reports are still open.

## What This Changes

The issue reports GitHub OAuth authorization returning to ClawHub without persisting a session, leaving the user logged out after the callback.

## Best Possible Solution

Reliable GitHub sign-in for active accounts, explicit terminal messaging or support handling for deleted/banned users, and regression coverage around callback handling, user bootstrap, and signed-in UI state.

## Solution Assessment
