## Summary

Keep open. This is an open ClawHub issue, and current main plus live checks do not prove the OAuth/session-loss report is fixed; production sign-in still returns HTTP 500 before callback completion while related OAuth/session-loop reports remain open.

## What This Changes

The issue reports GitHub OAuth authorization completing but ClawHub failing to persist an authenticated session, leaving users logged out after callback.

## Best Possible Solution

Run an operator-backed OAuth/session investigation, restore reliable session creation for active GitHub accounts, preserve explicit deleted/banned-account rejection, and add focused callback/session regression coverage plus redacted production smoke proof.
