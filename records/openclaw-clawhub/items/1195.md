## Repair Work Prompt

Fix the Convex owner-publisher digest sync pagination bug reported by this issue and related publish/import reports. In `convex/functions.ts`, change `syncPackageSearchDigestsForOwnerPublisherIdInternal` and `syncSkillSearchDigestsForOwnerPublisherIdInternal` so each invocation performs at most one `.paginate()` call for the relevant indexed owner-publisher query, processes that page, and if `isDone` is false schedules the same internal mutation again with `continueCursor`. Add validated optional cursor/page-size args as needed, preserve existing missing-table behavior, and keep the existing package/skill jobs separate. Do not change unrelated publish semantics, package publish retry behavior, or UI code. Add focused tests in `convex/functions.test.ts` that simulate multi-page package and skill owner-publisher digest rebuilds and assert one paginate call per invocation plus continuation scheduling. Reference related reports in the PR summary/changelog as one backend fix cluster.
