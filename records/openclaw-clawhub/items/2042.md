## Summary

Keep open: this is a ClawHub issue, and current cleanup policy keeps issues open unless there is a separate non-code close reason. Current main has native structural quality gating and security scan/moderation paths, but it does not integrate the proposed `agent-skill-infra` CLI or provide the requested semantic scoring, behavior eval, and version-drift curation workflow.

## What This Changes

The issue proposes using the external `agent-skill-infra` CLI as a complementary automated skill quality-scoring, behavior-testing, and version-drift tool for ClawHub curation.

## Best Possible Solution

Maintainers should reconcile this with the broader quality-signals tracker, then decide whether the external CLI belongs as optional reviewer documentation, staff-only tooling, CI guidance, or a formal ClawHub quality signal with approved supply-chain and data-flow boundaries.

## Reproduction Assessment

Not applicable; this is a workflow/tooling integration proposal rather than a broken behavior report. Source inspection gives a high-confidence check that current main has internal quality and scanner paths but not the named external CLI integration.
