## Summary

Keep open: current main has deterministic upload-quality gating, ClawScan/SkillSpector/VirusTotal audit surfaces, and Plugin Inspector validation for plugins, but it does not implement the requested external CLI workflow for semantic scoring, eval-based behavior testing, or version-baseline drift checks.

## What This Changes

The issue asks ClawHub to consider agent-skill-infra as an automated skill quality scoring and testing CLI for the curation workflow.

## Best Possible Solution

Define one advisory quality-signal contract through the open quality-signal and content-policy discussions, then either document this CLI as optional reviewer tooling or integrate vetted, explainable outputs while keeping security verdicts and enforcement decisions separate.

## Reproduction Assessment

Not applicable: this issue proposes a new curation and reviewer workflow rather than reporting broken behavior under an existing contract. The high-confidence check is source, live issue, related discussion, and history review.

## Solution Assessment

No, direct integration is not established as the best solution yet. Maintainers first need to decide evaluator authority, score semantics, data handling, dependency ownership, and overlap with existing scanners and quality signals.
