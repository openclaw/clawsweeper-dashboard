## Summary

Keep open: current main has deterministic publish-time quality gating plus ClawScan, SkillSpector, VirusTotal, plugin-inspector, and `clawhub scan` surfaces, but it still lacks the requested agent-skill-infra-style semantic scoring, eval-based behavior testing, and version-drift curation workflow. The request overlaps https://github.com/openclaw/clawhub/issues/1310 and https://github.com/openclaw/clawhub/issues/2321, but neither fully supersedes this tool/workflow proposal.

## What This Changes

The issue proposes using an external automated skill quality scoring, behavior testing, and version-awareness CLI as complementary ClawHub curation tooling.

## Best Possible Solution

Maintainers should define one advisory quality-signal contract and data-handling boundary, then decide whether to document this CLI as optional reviewer tooling or integrate vetted outputs separately from security verdicts and enforcement decisions.
