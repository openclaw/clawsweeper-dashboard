## Summary

Current main has a private publish-time anti-spam quality assessment and a mature security-scanning pipeline, but it does not implement the proposed semantic scoring, behavior-evaluation, and version-baseline workflow. The adjacent quality-signals and content-policy discussions remain open, so this requires a product and security-boundary decision rather than cleanup as completed or superseded work.

## What This Changes

Evaluate whether ClawHub should integrate or formally adopt the external agent-skill-infra CLI for automated skill quality scoring, behavior evaluation, and version-drift checks in curation workflows.

## Best Possible Solution

Consolidate this proposal with https://github.com/openclaw/clawhub/issues/1310 into a maintainer-owned scoring RFC that defines explainable dimensions, evidence provenance, prompt-only coverage, freshness, appeals, and strict separation from security verdicts, then evaluate external tools as interchangeable evidence providers rather than the product contract itself.

## Reproduction Assessment

Not applicable: this issue proposes a new curation and scoring capability rather than a failure of defined current behavior.

## Solution Assessment

No, adopting the external CLI wholesale is not yet the best solution; maintainers first need a canonical scoring and policy contract, after which this tool could be evaluated as an optional evidence provider.
