## Summary

Current main has substantial security-scanning and moderation infrastructure, including SkillSpector and LLM-derived security dimensions, but it does not implement the proposed semantic quality scoring, eval-driven behavior testing, or version-baseline workflow. The idea remains potentially useful, while adoption or endorsement requires a maintainer decision about signal ownership, external model processing, and keeping subjective quality separate from security verdicts; the broader discussion at https://github.com/openclaw/clawhub/issues/1310 is the natural design venue.

## What This Changes

Evaluate whether ClawHub should integrate or formally support an external CLI that scores skill quality, runs behavior evals, and compares versions for curation.

## Best Possible Solution

Use https://github.com/openclaw/clawhub/issues/1310 to define provider-neutral quality dimensions, score provenance, and reviewer-facing semantics, then evaluate this CLI as an optional pilot whose results remain clearly separate from security verdicts and automated enforcement.
