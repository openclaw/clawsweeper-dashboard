## Summary

Keep open: this is a ClawHub issue and a new curation/tooling adoption proposal, not a current-main bug. Current main has internal publish-time quality and scan/moderation paths, but no `agent-skill-infra` integration or documented reviewer workflow, and the related broader quality-signal discussion remains open at https://github.com/openclaw/clawhub/issues/1310.

## What This Changes

The issue proposes using the external `agent-skill-infra` CLI to score SKILL.md quality, run eval-style behavior tests, and compare skill versions before manual curation.

## Best Possible Solution

Decide whether `agent-skill-infra` belongs as optional reviewer documentation, staff-only tooling, CI guidance, or a backend signal, then integrate only the approved slice into the existing quality/moderation model after privacy and supply-chain review.

## Reproduction Assessment

Not applicable; this is a workflow/tooling integration proposal rather than a broken behavior report. Source inspection gives a high-confidence check that current main has quality/scanner paths but no `agent-skill-infra` reviewer workflow.

## Solution Assessment

Unclear; adopting the external CLI wholesale is not yet proven to be the best maintainable path. A safer direction is to compare it with the existing quality gate and moderation console, then add only a narrow documented or staff-only workflow if maintainers approve it.
