## Summary

Keep open: this is a ClawHub issue, and the cleanup policy keeps issues open unless a maintainer resolves them. Current main has a narrow publish-time content-quality gate and security audit surfaces, but it does not implement the proposed external, multidimensional quality-scoring curation workflow.

## What This Changes

The issue asks ClawHub to evaluate `agent-skill-infra`, an external Python CLI for semantic skill scoring, behavior tests, and version-drift checks, as a complementary curation tool.

## Best Possible Solution

Define a maintainer-approved quality-signal model and evaluator provenance, decide whether `agent-skill-infra` is an external recommended tool or a ClawHub integration, then implement schema/API/UI/docs only after security boundaries are clear.
