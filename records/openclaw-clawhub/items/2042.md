## Summary

Keep open: ClawHub cleanup policy keeps issues open, and this proposal still needs maintainer product and trust-policy decisions. Current main has a narrow publish-time content quality gate plus security scan surfaces, but no multidimensional curator scoring, behavior-eval, or version-drift workflow for agent-skill-infra-style scoring.

## What This Changes

This issue proposes using the external agent-skill-infra CLI as a complementary automated skill quality, behavior, version-drift, and security precheck tool for ClawHub curation.

## Best Possible Solution

Define a product-owned quality-signal model distinct from security/moderation, decide whether agent-skill-infra is an optional curator tool or integrated service, then add approved schema, API/search support, UI, backfill, and operator docs.

## Reproduction Assessment

Not applicable: this is a product/tooling proposal rather than a runtime bug. Source inspection gives a high-confidence baseline that current main only has a narrow publish-time content gate and no multidimensional scoring workflow.

## Solution Assessment

Unclear: the external CLI is useful context, but direct integration is not yet established as the best path. Maintainers should first decide score semantics, provider/data boundaries, rollout, and the relationship to https://github.com/openclaw/clawhub/issues/1310.
