## Summary

Keep open: this is a ClawHub workflow/tooling adoption proposal, not a current-main bug. Current main has native publish-time quality gating and scan/moderation paths, but no `agent-skill-infra` integration or semantic/behavior/version-drift scorer, and the related broader quality-signal discussion remains open at https://github.com/openclaw/clawhub/issues/1310.

## What This Changes

Evaluate adopting `agent-skill-infra` CLIs for automated skill quality scoring, behavior testing, and version drift checks in ClawHub's curation workflow.

## Best Possible Solution

Maintainers should decide whether this belongs as optional reviewer documentation, staff-only tooling, CI guidance, or a formal ClawHub signal, then define the supply-chain, privacy, schema/API, and UI boundaries before implementation.

## Reproduction Assessment

Not applicable; this is a workflow/tooling integration proposal rather than a broken behavior report. Source inspection gives a high-confidence check that current main has internal quality/scanner paths but no `agent-skill-infra` integration.
