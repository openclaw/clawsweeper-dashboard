## Summary

Keep open: current main and the latest v0.17.0 release have a narrow publish-time SKILL.md quality gate plus security-audit and SkillSpector evidence surfaces, but they do not implement the proposed semantic quality scoring, eval-based behavior testing, or version-drift curation workflow. The external agent-skill-infra tool is relevant context, and related quality/content-policy discussions remain open, so this needs maintainer product and security review.

## What This Changes

Request to consider agent-skill-infra as a complementary ClawHub curation tool for semantic skill quality scoring, behavior tests, and version-drift checks.

## Best Possible Solution

Define a maintainer-owned quality-signal model distinct from safety verdicts, coordinate it with the open quality-signals and content-policy threads, then decide whether agent-skill-infra should feed that model.
