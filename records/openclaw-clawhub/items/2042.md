## Summary

Keep this issue open: current main has deterministic upload-quality gates and multiple security scanners, but it does not implement the proposed semantic scoring, eval execution, and version-comparison workflow; adopting that workflow still requires product and security decisions.

## What This Changes

Evaluate whether the external agent-skill-infra quality-testing CLI should integrate with ClawHub’s skill-curation workflow.

## Best Possible Solution

Define a provider-neutral, advisory quality-signal contract in the canonical quality discussion, then evaluate this CLI as a pilot provider with explicit provenance, privacy, failure, appeal, and non-enforcement semantics.
