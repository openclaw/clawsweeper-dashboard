## Summary

Keep open: this is a ClawHub workflow/tooling adoption proposal, and the ClawHub cleanup policy keeps issues open for maintainer handling. Current main has structural publish-time quality gates and security scans, but no `agent-skill-infra` integration or endorsed semantic, behavior-test, and version-drift scoring workflow.

## What This Changes

The issue proposes evaluating the external `agent-skill-infra` CLI for semantic quality scoring, behavior evals, and version-drift checks in ClawHub curation.

## Best Possible Solution

Maintainers should decide whether this belongs as optional reviewer documentation, staff-only tooling, CI guidance, or a formal ClawHub quality signal, then define supply-chain, privacy, schema/API, and UI boundaries before implementation.

## Reproduction Assessment

Not applicable; this is a workflow/tooling integration proposal rather than a broken behavior report. Source inspection gives a high-confidence check that current main has internal structural quality and scanner paths but not the named external CLI integration.

## Solution Assessment

Unclear; adopting the external CLI wholesale is not yet proven to be the narrowest maintainable solution. A safer path is to compare it with the existing quality gate and https://github.com/openclaw/clawhub/issues/1310, then add only an approved documented or staff-only workflow if maintainers want it.
