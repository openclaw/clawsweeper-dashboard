## Summary

Keep open: this is a ClawHub feature/workflow proposal for adopting an external quality-scoring tool, not a current-main bug. Current main has internal publish-time quality gating and scan/moderation paths, but no `agent-skill-infra` integration or documented reviewer workflow, so this needs maintainer product and security judgment.

## What This Changes

The issue proposes using the external `agent-skill-infra` CLI as a first-pass skill quality, behavior-test, and version-drift tool for ClawHub curation.

## Best Possible Solution

Compare `agent-skill-infra` against the existing quality and moderation pipeline, then only add a narrow documented or staff-only workflow if maintainers approve the score semantics, privacy boundaries, and supply-chain model.

Not applicable; this is a workflow/tooling adoption proposal rather than a broken behavior report. Source inspection gives a high-confidence check that current main has quality/scanner paths but no `agent-skill-infra` integration.
