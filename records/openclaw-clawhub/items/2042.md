## Summary

Keep open: current main has a narrow publish-time content-quality gate and security scan surfaces, but it does not implement the proposed multidimensional curation scoring workflow. The linked agent-skill-infra project is relevant context, and adopting it would need maintainer product and security decisions first.

## What This Changes

The issue proposes using the external agent-skill-infra CLI to automate semantic skill scoring, behavior tests, and version-drift checks in ClawHub’s curation workflow.

## Best Possible Solution

Define a product-owned quality-signal model distinct from security/moderation, decide whether agent-skill-infra is an optional curator tool or integrated service, then add approved schema, API/search support, UI, backfill, and operator docs.

## Reproduction Assessment

Not applicable: this is a product/tooling proposal rather than a runtime bug. Source inspection gives a high-confidence baseline that current main only has a publish-time content gate and security scan surfaces, not multidimensional curator scoring.

## Solution Assessment

Unclear: the external CLI is useful context, but direct integration is not yet established as the best path. Maintainers should first decide score semantics, evaluator/data boundaries, rollout, and the relationship to https://github.com/openclaw/clawhub/issues/1310.

## Security Review

Status: needs_attention

Summary: Integrating external quality scoring into curation is security-adjacent because it can influence trust labels, install decisions, and artifact/provider data boundaries.

Concerns:

- **[medium] Separate quality labels from safety verdicts:**
  - body: User-visible quality scores near moderation or scanner output could imply a skill is safe to run unless the product model clearly separates quality, security, moderation, and install-blocking semantics.
  - confidence: 0.86
- **[medium] Review external evaluator data boundaries:**
  - body: A workflow using GitHub Models, Anthropic, or scanner integrations may send skill artifacts or metadata outside ClawHub, so maintainers should decide privacy, retention, provenance, and failure handling before implementation.
  - confidence: 0.82
