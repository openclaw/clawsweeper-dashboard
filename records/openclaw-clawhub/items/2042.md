## Summary

Keep open: current main has deterministic publish-time quality gating and ClawScan/SkillSpector/VirusTotal scan surfaces, but it does not implement the requested agent-skill-infra-style semantic scoring, eval-based behavior testing, or version-drift workflow. The remaining decision overlaps the broader quality-signals discussion at https://github.com/openclaw/clawhub/issues/1310 and the content-policy RFC at https://github.com/openclaw/clawhub/issues/2321, but this issue still has unique external-tool workflow context.

## What This Changes

This issue proposes adopting or documenting agent-skill-infra as a complementary curation tool for semantic skill scoring, eval-based behavior testing, and version drift checks.

## Best Possible Solution

Maintainers should define one advisory quality-signal contract and data-handling boundary, then either document this CLI as optional reviewer tooling or integrate vetted outputs separately from security verdicts and enforcement decisions.
