## Summary

Keep open: this is a ClawHub issue, and the cleanup policy does not auto-close issues. The linked agent-skill-infra toolkit is relevant, but current main only has a narrow publish-time content quality gate plus security scan surfaces; adopting multidimensional quality scoring still needs maintainer product and trust-policy decisions.

## What This Changes

The issue proposes using agent-skill-infra as a complementary CLI for SKILL.md quality scoring, behavior tests, version drift checks, and security-review prechecks in ClawHub curation.

## Best Possible Solution

Define a product-owned quality-signal model distinct from security/moderation, decide whether agent-skill-infra is an optional curator tool or integrated service, then add approved schema, API/search support, UI, backfill, and operator docs.

Not applicable: this is a product/tooling proposal rather than a runtime bug. Source inspection gives a high-confidence baseline that current main only has a narrow publish-time content gate and no multidimensional scoring workflow.
