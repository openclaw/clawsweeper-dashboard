## Summary

Keep open: this is a ClawHub issue, and current main still has only a narrow publish-time content-quality gate plus security scan surfaces, not an adopted multidimensional curation scoring workflow. The linked agent-skill-infra CLI is relevant implementation context, but using it in ClawHub needs maintainer product and security-policy decisions first.

## What This Changes

The issue proposes using the external agent-skill-infra CLI as an automated quality, behavior-test, and version-awareness aid for ClawHub skill curation.

## Best Possible Solution

Define a product-owned quality-signal model distinct from security/moderation, decide whether agent-skill-infra is an optional curator tool or integrated service, then add approved schema, indexed API/search support, UI, backfill, and operator docs.

## Reproduction Assessment

Not applicable: this is a product/tooling proposal rather than a runtime bug. Source inspection gives a high-confidence baseline that current main lacks the requested multidimensional quality-scoring workflow.

## Solution Assessment

Unclear: the external CLI is useful context, but direct integration is not yet established as the best path. Maintainers should first decide score semantics, provider/data boundaries, rollout, and the relationship to https://github.com/openclaw/clawhub/issues/1310.
