## Summary

Keep open: this is a ClawHub workflow/tooling adoption proposal, not a current-main bug or closeable issue. Current main has native publish-time quality gating and scan/moderation paths, but it does not integrate `agent-skill-infra` or provide the proposed semantic, behavior-test, and version-drift scoring workflow; the broader quality-signal discussion remains open at https://github.com/openclaw/clawhub/issues/1310.

## What This Changes

Adopt or document `agent-skill-infra` as an automated first-pass skill quality, behavior-test, and version-awareness tool for ClawHub curation.

## Best Possible Solution

Maintainers should decide whether this belongs as optional reviewer documentation, staff-only tooling, CI guidance, or a formal ClawHub signal, then define supply-chain, privacy, schema/API, and UI boundaries before implementation.

## Reproduction Assessment

Not applicable; this is a workflow/tooling integration proposal rather than a broken behavior report. Source inspection gives a high-confidence check that current main has internal quality/scanner paths but no `agent-skill-infra` integration.
