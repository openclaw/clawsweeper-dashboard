Keep open: this is a ClawHub workflow/tooling adoption proposal, not a current-main bug, and the ClawHub cleanup policy keeps issues open. Current main has native publish-time content-shape quality gating and scan/moderation paths, but no `agent-skill-infra` integration or semantic/behavior/version-drift scorer; the broader quality-signal discussion remains open at https://github.com/openclaw/clawhub/issues/1310.

## What This Changes

The issue proposes evaluating the external `agent-skill-infra` CLI as a complementary automated quality, behavior, and version-drift scorer for ClawHub skill curation.

## Best Possible Solution

Maintainers should decide whether this belongs as optional reviewer documentation, staff-only tooling, CI guidance, or a formal ClawHub signal, then define the supply-chain, privacy, schema/API, and UI boundaries before implementation.
