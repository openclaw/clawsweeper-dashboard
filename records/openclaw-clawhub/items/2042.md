Current main has internal publish-time quality gating plus security scan/moderation paths, but no `agent-skill-infra` integration or documented reviewer workflow, and ClawHub issue cleanup policy keeps issues open for maintainer product decisions.

The issue proposes using the external `agent-skill-infra` CLI modules for semantic quality scoring, behavior tests, and version-aware review in ClawHub curation workflows.

## Best Possible Solution
