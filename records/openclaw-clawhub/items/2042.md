## Summary

Keep open: current main has structural publish-time SKILL.md quality gating, security-audit evidence, and SkillSpector/ClawScan surfaces, but it does not implement the proposed semantic quality score, eval-based behavior testing, or version-drift curation workflow. The external agent-skill-infra tool is relevant context, and the related quality-signal and content-policy discussions remain open, so this needs maintainer product and security review.

## What This Changes

The issue proposes integrating or adopting the external agent-skill-infra CLI for automated skill quality scoring, behavior tests, and version-drift checks in ClawHub curation.

## Best Possible Solution

Define a maintainer-owned quality-signal model distinct from security/moderation verdicts, coordinate it with the open quality-signals and content-policy threads, then decide whether agent-skill-infra should feed that model.
