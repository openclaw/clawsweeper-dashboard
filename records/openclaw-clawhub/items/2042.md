## Summary

Keep open: this is a new ClawHub curation/tooling adoption proposal, not a bug. Current main already has quality scoring plus scan/moderation pipelines, but it does not integrate or document `agent-skill-infra`, and ClawHub issue policy leaves this kind of product/tooling decision for maintainers.

## What This Changes

The issue proposes evaluating or integrating the external `agent-skill-infra` CLI for automated skill quality, behavior testing, and version drift scoring in ClawHub curation workflows.

## Best Possible Solution

Keep the current ClawHub quality and moderation pipeline as-is until maintainers decide whether `agent-skill-infra` should be optional documentation, reviewer tooling, or a narrow staff-only signal after privacy and supply-chain review.

## Reproduction Assessment

Not applicable; this is a workflow/tooling integration proposal rather than a broken behavior report. Source inspection confirms ClawHub has current quality/scanner paths but no `agent-skill-infra` integration on current main.

## Solution Assessment

Unclear; adopting the external CLI wholesale is not yet proven to be the best maintainable path. A safer direction is to compare it against the existing quality gate and moderation console, then add only a narrow documented or staff-only workflow if maintainers approve it.
