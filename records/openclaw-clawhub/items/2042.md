## Summary

Keep open: this is a ClawHub curation/tooling adoption proposal around the linked agent-skill-infra project, not a current-main bug. Current main has internal publish-time quality gates and scan/moderation paths, but no integration or approved reviewer workflow for this external CLI, so it needs maintainer product and security judgment.

## What This Changes

The issue proposes using agent-skill-infra for semantic skill scoring, behavior tests, version-drift checks, and security-review preflight in ClawHub’s curation workflow.

## Best Possible Solution

Keep the current quality and moderation pipeline until maintainers decide whether agent-skill-infra belongs as optional documentation, CI guidance, staff-only reviewer tooling, or a narrow backend signal with clear privacy and supply-chain boundaries.

## Reproduction Assessment

Not applicable; this is a workflow/tooling integration proposal rather than a broken behavior report. Source inspection gives a high-confidence check that current main has quality/scanner paths but no agent-skill-infra integration.

## Solution Assessment

Unclear; adopting the external CLI wholesale is not yet proven to be the best maintainable path. A safer direction is to compare it against the existing quality gate and moderation console, then add only a narrow documented or staff-only workflow if maintainers approve it.
