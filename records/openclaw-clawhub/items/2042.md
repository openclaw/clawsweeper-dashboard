## Summary

This remains a product and security decision overlapping the broader quality-signals discussion and the content-policy RFC.

## What This Changes

The issue requests maintainer consideration of an external Python CLI that scores skill quality, runs eval-based behavior tests, and compares versions as a ClawHub curation aid.

## Best Possible Solution

Define one advisory quality-signal contract through the related product discussions, then either document this CLI as optional reviewer tooling or integrate vetted, explainable outputs while keeping security verdicts and enforcement decisions separate.
