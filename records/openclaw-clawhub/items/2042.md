## Summary

Keep this open: current main has deterministic anti-spam quality gates and ClawScan security review, but it does not implement or endorse the proposed semantic scoring, eval-driven behavior testing, or version-baseline workflow. The remaining work is a product and security decision about whether this external CLI becomes optional reviewer tooling, an integrated ClawHub signal, or independent ecosystem tooling.

## What This Changes

The issue proposes integrating or endorsing an external automated skill quality scoring, behavior-testing, and version-awareness CLI for ClawHub’s curation workflow.

## Best Possible Solution

Define one advisory quality-signal contract through maintainer product/security review, then either document this CLI as optional non-authoritative reviewer tooling or integrate vetted, explainable outputs while keeping deterministic enforcement and ClawScan security verdicts separate.
