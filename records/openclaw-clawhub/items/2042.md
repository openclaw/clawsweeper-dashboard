## Summary

Keep open: this is a ClawHub workflow/tooling proposal, not a current-main bug. The proposed `agent-skill-infra` CLI could be useful as a complementary curation tool, but current main only has native publish-time quality gating plus scanner/moderation paths, and the broader quality-signal discussion remains open at https://github.com/openclaw/clawhub/issues/1310.

## What This Changes

Request to evaluate or integrate the external `agent-skill-infra` CLI as a first-pass skill quality scoring, behavior testing, and version-drift aid for ClawHub curation.

## Best Possible Solution

Maintainers should decide whether this belongs as optional reviewer documentation, staff-only tooling, CI guidance, or a formal ClawHub signal, then define supply-chain, privacy, score semantics, API/schema, and UI boundaries before implementation.

## Reproduction Assessment

Not applicable; this is a workflow/tooling integration proposal rather than a broken behavior report. Source inspection gives a high-confidence check that current main has internal quality/scanner paths but no `agent-skill-infra` integration.

## Solution Assessment

Unclear; adopting the external CLI wholesale is not yet proven to be the narrowest maintainable solution. A safer path is to compare it with the existing quality gate and https://github.com/openclaw/clawhub/issues/1310, then add only a documented or staff-only workflow if maintainers approve it.
