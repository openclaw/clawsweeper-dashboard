## Summary

Keep open: current main has a narrow publish-time content quality gate, but it does not implement the external `agent-skill-infra` scoring/testing/version workflow or a public multidimensional quality model, and ClawHub cleanup policy keeps issues open for maintainer judgment.

## What This Changes

This issue proposes using `agent-skill-infra` as a complementary CLI for automated skill quality scoring, behavior testing, and version-drift checks before manual ClawHub review.

## Best Possible Solution

Define one product-owned quality-signal model, reconcile this tool proposal with https://github.com/openclaw/clawhub/issues/1310, then decide whether an external CLI integration, internal evaluator, or docs-only reviewer workflow belongs in ClawHub.
