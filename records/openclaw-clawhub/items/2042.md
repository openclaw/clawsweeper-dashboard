## Summary

Keep open: this is a ClawHub workflow/tooling adoption proposal, and the ClawHub cleanup policy keeps issues open for maintainer handling. Current main has native structural quality gating and security scan/moderation paths, but no `agent-skill-infra` integration or semantic, behavior-test, and version-drift scoring workflow; the broader quality-signals discussion remains open at https://github.com/openclaw/clawhub/issues/1310.

## What This Changes

Consider adopting or documenting `agent-skill-infra` as complementary automated quality scoring, behavior testing, and version-drift tooling for ClawHub curation.

## Best Possible Solution

Maintainers should decide whether this belongs as optional reviewer documentation, staff-only tooling, CI guidance, or a formal ClawHub quality signal, then define supply-chain, privacy, schema/API, and UI boundaries before implementation.
