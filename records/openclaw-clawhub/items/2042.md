## Summary

Keep open: this is an open ClawHub issue and the repository cleanup policy keeps ClawHub issues open for maintainer handling. Current main has publish-time structural quality gating and scanner/moderation paths, but it does not integrate or document `agent-skill-infra` or the requested semantic scoring, behavior eval, and version-drift curation workflow.

## What This Changes

The issue proposes adopting or documenting `agent-skill-infra` as an automated skill quality scoring and testing tool for ClawHub curation.

## Best Possible Solution

Maintainers should reconcile this with the open quality-signals discussion, then decide whether the tool belongs as optional reviewer documentation, staff-only tooling, CI guidance, or a formal ClawHub quality signal with approved supply-chain, privacy, schema/API, and UI boundaries.
