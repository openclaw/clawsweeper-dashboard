## Summary

Keep open: current main has deterministic upload-quality gating and mature ClawScan/SkillSpector/VirusTotal audit surfaces, but it does not implement the requested external CLI workflow for semantic scoring, eval-based behavior testing, or version-baseline drift checks. The remaining work is a product and security decision that overlaps the broader quality-signals discussion at https://github.com/openclaw/clawhub/issues/1310 and the content-policy RFC at https://github.com/openclaw/clawhub/issues/2321.

## What This Changes

The issue asks ClawHub to consider or integrate an external CLI for automated skill quality scoring, behavior evals, and version drift checks in the curation workflow.

## Best Possible Solution

Define one advisory quality-signal contract through the related product discussions, then either document this CLI as optional reviewer tooling or integrate vetted, explainable outputs while keeping security verdicts and enforcement decisions separate.

## Reproduction Assessment

Not applicable: this issue proposes a new curation and reviewer workflow rather than reporting broken behavior under an existing contract. The high-confidence check is source, live issue, related discussion, and history review.

## Solution Assessment

No, direct integration is not established as the best solution yet. Maintainers first need to decide evaluator authority, score semantics, data handling, dependency ownership, and overlap with existing scanners and quality signals.

## Risks / Open Questions
