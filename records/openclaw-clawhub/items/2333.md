## Summary

Keep open: current main still intentionally blocks deleted/deactivated accounts from GitHub sign-in, and I did not find an audited admin purge path that frees the reserved GitHub auth binding for fresh signup. Resolving this would change the irreversible-deletion security policy, so it needs maintainer product/security approval before implementation.

## What This Changes

The issue requests an admin-supported recovery path that lets eligible self-deleted users purge reserved GitHub auth state and sign up again without restoring deleted content or bypassing security holds.

## Best Possible Solution

Approve and implement an admin-only audited purge recovery for eligible self-deleted accounts, with explicit identity verification, destructive-effect logging, same-GitHub fresh-signup coverage, and unchanged fail-closed handling for bans and security holds.

## Reproduction Assessment

Yes, from source inspection: self-service deletion sets deactivatedAt/purgedAt, and the OAuth callback rejects that account on later GitHub reauth; tests assert the blocked path. I did not run a live GitHub OAuth reproduction.

## Solution Assessment

Unclear until maintainers decide the policy: automatic fresh signup conflicts with the current irreversible-deletion spec. The maintainable direction is a separately approved admin-only purge operation for eligible self-deletes, not generic reactivation.
