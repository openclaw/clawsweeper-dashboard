## Label Justifications

- `P1`: Affected users can be completely blocked from ClawHub sign-in after deleting their account, and this issue consolidates many duplicate reports.
- `impact:security`: The requested purge path would change account-standing enforcement and must not bypass bans, malware autobans, disabled accounts, or abuse flags.
- `impact:auth-provider`: The issue centers on GitHub OAuth identity reuse and Convex Auth provider binding state after account deletion.

## Summary

Keep open: current main still intentionally blocks deleted/deactivated users from signing in again, while the requested fresh-signup purge path is a security-sensitive account-recovery policy change that also relates to the open RFC at https://github.com/openclaw/clawhub/issues/2320.

## What This Changes

Request an admin-only audited purge/recovery path that frees eligible self-deleted GitHub identities for fresh signup without restoring deleted content or bypassing bans or security holds.

## Best Possible Solution

Approve and implement an admin-only audited purge recovery for eligible self-deleted accounts, aligned with the account-recovery RFC and preserving fail-closed handling for security holds.

## Reproduction Assessment

Yes, from source inspection: self-service deletion sets deactivatedAt/purgedAt, and the OAuth callback rejects that account on later GitHub reauth; auth tests assert the blocked path. I did not run a live GitHub OAuth reproduction.

## Solution Assessment

Unclear until maintainers decide the policy: automatic fresh signup conflicts with the current irreversible-deletion spec. The narrow maintainable direction is a separately approved admin-only purge operation for eligible self-deletes, not generic reactivation.
