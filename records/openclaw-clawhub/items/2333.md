## Summary

Keep open: current main still blocks self-deleted/deactivated users from signing in again with the same GitHub identity, and the requested fresh-signup purge path remains unresolved security-sensitive account-recovery work tied to https://github.com/openclaw/clawhub/issues/2320.

## What This Changes

Request an admin-only audited purge/recovery path that frees eligible self-deleted GitHub identities for fresh signup without restoring deleted content or bypassing bans.

## Best Possible Solution

Approve an RFC-backed admin-only audited recovery operation for eligible self-deleted accounts, with regression coverage proving fresh signup works while security holds remain blocked.
