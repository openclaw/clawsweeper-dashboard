## Summary

Keep open: current main and v0.17.0 still reject deleted/deactivated GitHub users, and the same-GitHub fresh-signup purge path remains unresolved security-sensitive account-recovery work tied to https://github.com/openclaw/clawhub/issues/2320.

## What This Changes

Add a supported admin/account-recovery path so eligible self-deleted users can purge old ClawHub/auth binding state and sign up again with the same GitHub identity.

## Best Possible Solution

Approve and document a narrow admin-only audited purge/recovery flow for eligible self-deletes that frees auth/provider identity for fresh signup while keeping bans and abuse holds blocked.

Yes, from source inspection: deleteAccount sets deactivatedAt/purgedAt and handleDeletedUserSignIn rejects that account on later GitHub reauth; auth tests assert the blocked path. I did not run a live GitHub OAuth reproduction.

## Solution Assessment
