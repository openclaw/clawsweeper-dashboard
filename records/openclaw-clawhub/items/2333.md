## Label Justifications

- `P1`: Users who self-deleted can remain completely blocked from ClawHub sign-in with their GitHub identity, and this issue consolidates many duplicate reports.
- `impact:security`: The requested purge path changes account-standing enforcement and must not bypass bans, malware autobans, disabled accounts, or abuse flags.
- `impact:auth-provider`: The issue centers on GitHub OAuth identity reuse and Convex Auth provider binding state after account deletion.

## Summary

Keep open: current main still deliberately rejects self-deleted/deactivated users on GitHub reauth, and ClawHub issue policy does not support auto-closing this live security-sensitive recovery request.

## What This Changes

The issue requests an admin-supported purge/recovery path so eligible self-deleted users can sign up fresh with the same GitHub identity without restoring deleted content or bypassing security holds.

## Best Possible Solution

Approve and implement a narrowly scoped admin-only audited purge recovery for eligible self-deleted accounts, with identity verification, same-GitHub fresh-signup coverage, and unchanged fail-closed handling for security holds.

## Reproduction Assessment

Yes, at source level: self-service deletion sets deactivatedAt/purgedAt, and the OAuth callback rejects that account on later GitHub reauth; tests assert the rejected path. I did not run a live GitHub OAuth reproduction.

## Solution Assessment

Unclear until maintainers decide the account-recovery policy; the maintainable direction is an admin-only audited purge for eligible self-deletes, not automatic reactivation or unrestricted identity reuse.
