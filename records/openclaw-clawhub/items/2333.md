## Label Justifications

- `P1`: Affected users remain fully blocked from ClawHub sign-in after deleting their account, and many duplicate reports have been consolidated here.

## Summary

Keep open: current main still treats user-initiated deletion as irreversible, blocks OAuth reauth for deactivated/purged users, and has no admin purge/recovery path for reusing the same GitHub identity. This remains the canonical security/auth-provider item; the recent commenter’s older PR was about missing OAuth environment variables, not the deleted-account recovery state tracked here.

## What This Changes

The issue requests an admin-supported purge/recovery path that lets eligible self-deleted ClawHub users reuse the same GitHub OAuth identity for a fresh signup without restoring purged content or bypassing account-standing blocks.

## Best Possible Solution

Define and implement an admin-only audited recovery purge for eligible self-deleted accounts, with identity verification, explicit destructive-effect logging, same-GitHub fresh-signup coverage, and unchanged fail-closed handling for bans, malware autobans, disabled accounts, and abuse holds.

## Reproduction Assessment

Yes, from source inspection: self-service deletion sets `deactivatedAt`/`purgedAt`, and OAuth reauth with the same stored user is rejected by `handleDeletedUserSignIn`. I did not run a live GitHub OAuth flow.

## Solution Assessment

No automated fix is ready yet: the requested recovery path changes the current irreversible-deletion policy. The maintainable solution is a separately approved admin-only purge operation with security negative coverage.
