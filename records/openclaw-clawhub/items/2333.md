## Summary

Keep open: current main still blocks deleted/deactivated users from signing in again, and the requested same-GitHub fresh-signup purge path remains unresolved security-sensitive account-recovery work tied to https://github.com/openclaw/clawhub/issues/2320.

## What This Changes

Request an admin-supported, audited purge/recovery path that lets eligible self-deleted users sign up again with the same GitHub identity without restoring deleted content or bypassing security holds.

## Best Possible Solution

Approve a narrow RFC-backed admin-only audited purge operation for eligible self-deleted accounts, with regression coverage proving fresh signup works and security holds remain blocked.
