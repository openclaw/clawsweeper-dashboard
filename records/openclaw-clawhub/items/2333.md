## Summary

Keep open: current main still blocks reauth for deactivated/purged users and has no admin purge/recovery path; this is a security/auth-provider policy decision for maintainers, not a cleanup close.

## What This Changes

Request an admin-supported recovery purge so eligible self-deleted users can sign up again with the same GitHub identity without restoring deleted content or bypassing account-standing blocks.

## Best Possible Solution

Define an admin-only audited recovery purge for eligible self-deleted accounts, with identity verification, destructive-effect logging, fresh-signup coverage, and unchanged fail-closed handling for banned or security-held accounts.

## Reproduction Assessment

Yes. From source inspection, deleting an account marks the user deactivated/purged and the OAuth callback rejects that same user on reauth; tests assert the blocked path, though I did not run a live GitHub OAuth flow.

## Solution Assessment

Yes as a product direction: the maintainable solution is an explicit admin-only audited purge for eligible self-deletes, not automatic restoration or a generic unban path.
