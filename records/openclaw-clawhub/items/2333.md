## Summary

Keep open: current main still intentionally blocks deleted/deactivated users from signing in again, while the requested same-GitHub fresh-signup purge path remains unresolved security-sensitive account-recovery work tied to the open RFC at https://github.com/openclaw/clawhub/issues/2320.

## What This Changes

The issue requests an admin-supported purge/recovery path so eligible self-deleted users can reuse the same GitHub identity for a fresh signup without restoring deleted content by default.

## Best Possible Solution

Approve a narrow RFC-backed admin-only audited purge operation for eligible self-deleted accounts, with regression coverage proving fresh signup works and security holds remain blocked.

## Reproduction Assessment

Yes, at source level: `deleteAccount` sets `deactivatedAt`/`purgedAt`, and `handleDeletedUserSignIn` rejects that account on later GitHub reauth. I did not run a live GitHub OAuth reproduction.

## Solution Assessment

Unclear until maintainers decide the policy: the requested admin-only audited purge shape is plausible, but generic automatic fresh signup would conflict with the current irreversible-deletion spec.
