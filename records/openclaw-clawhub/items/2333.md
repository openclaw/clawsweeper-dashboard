## Summary

Keep open: current main intentionally blocks sign-in for deactivated/purged accounts and I did not find an admin recovery purge path. This is the canonical security/auth-provider item for deciding whether self-deleted accounts can be purged for fresh GitHub signup.

## What This Changes

The issue requests an admin-audited recovery path that purges eligible self-deleted account state so the same GitHub identity can sign up fresh without restoring deleted content.

## Best Possible Solution

Define and implement an admin-only audited purge recovery for eligible self-deleted accounts only, with explicit identity verification, destructive-effect logging, tests for same-GitHub fresh signup, and unchanged fail-closed handling for bans and security holds.

## Reproduction Assessment

Yes, from source inspection: `deleteAccount` marks the user deactivated/purged, and the OAuth callback rejects that user on reauth; tests assert the blocked path. I did not run a live GitHub OAuth reproduction.

## Solution Assessment

No automated code-only solution yet: the requested recovery path changes the current irreversible-deletion policy. The maintainable direction is a separately approved admin purge operation that preserves ban/security boundaries.
