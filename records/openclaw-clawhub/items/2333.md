## Summary

Keep open: current main still intentionally blocks deleted/deactivated users from signing in again, while the requested same-GitHub fresh-signup purge path remains unresolved security-sensitive account-recovery work tied to https://github.com/openclaw/clawhub/issues/2320.

## What This Changes

Investigate a supported admin recovery path that lets eligible self-deleted users purge old auth/provider state and sign up again with the same GitHub identity.

## Best Possible Solution

Approve a narrow RFC-backed admin-only audited purge operation for eligible self-deleted accounts, with regression coverage proving fresh signup works and security holds remain blocked.
