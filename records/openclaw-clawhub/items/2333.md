## Summary

Keep open: current main still intentionally blocks self-deleted/deactivated accounts from signing in again, and the requested fresh-signup purge path changes a security-sensitive account-deletion policy. The related closed PR at https://github.com/openclaw/clawhub/pull/1720 was an OAuth environment diagnostic change, not a deleted-account recovery fix.

## What This Changes

Request an admin-only recovery/purge path that frees an eligible self-deleted user's GitHub OAuth identity for fresh signup without restoring deleted content or bypassing security holds.

## Best Possible Solution

Approve and implement an admin-only audited purge recovery for eligible self-deleted accounts, with identity verification, destructive-effect logging, same-GitHub fresh-signup coverage, and unchanged fail-closed handling for security holds.

## Reproduction Assessment

Yes, from source inspection: self-service deletion sets deactivatedAt/purgedAt, and the OAuth callback rejects that account on later GitHub reauth; auth tests assert the blocked path. I did not run a live GitHub OAuth reproduction.

## Solution Assessment

Unclear until maintainers decide the policy: automatic fresh signup conflicts with the current irreversible-deletion spec. The narrow maintainable direction is a separately approved admin-only purge operation for eligible self-deletes, not generic reactivation.
