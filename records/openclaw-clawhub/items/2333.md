## Summary

Keep open: current main still blocks self-deleted accounts from signing in again, and the requested fresh-signup purge path is a security-sensitive account-recovery policy and implementation change tied to the open RFC at https://github.com/openclaw/clawhub/issues/2320. The commenter’s closed PR at https://github.com/openclaw/clawhub/pull/1720 addressed missing OAuth env diagnostics, not deleted-account identity recovery.

## What This Changes

The issue requests an admin-supported purge/recovery path that lets eligible self-deleted users create a fresh ClawHub account with the same GitHub identity while keeping banned and abuse cases blocked.

## Best Possible Solution

Approve and implement an admin-only audited recovery purge for eligible self-deleted accounts, coordinated with https://github.com/openclaw/clawhub/issues/2320, while preserving fail-closed handling for security holds and avoiding implicit content restoration.

## Reproduction Assessment

Yes, from source inspection: self-service deletion sets deactivatedAt/purgedAt, and the OAuth callback rejects that account on later GitHub reauth; auth tests assert the blocked path. I did not run a live GitHub OAuth reproduction.

## Solution Assessment

Unclear until maintainers decide the policy: automatic fresh signup conflicts with the current irreversible-deletion spec. The narrow maintainable direction is a separately approved admin-only purge operation for eligible self-deletes, not generic reactivation.

## Risks / Open Questions

- The accepted policy is unresolved: current specs say user-initiated deletion is irreversible, while the RFC asks what recovery path should exist after intentional account deletion.
- A purge operation could become a ban, malware-autoban, disabled-account, or abuse-hold bypass unless eligibility and account standing are checked before freeing identity state.
- Implementation must enumerate user flags, authAccounts provider bindings, API/session/token state, publisher/ownership reservations, audit logs, and deleted-content semantics before allowing fresh signup.
