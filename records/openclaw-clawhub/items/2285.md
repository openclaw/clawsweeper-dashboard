## What This Changes

The branch updates Convex publisher membership, resource authorization, package transfer checks, tests, changelog, and org specs so personal publishers use `linkedUserId` instead of stale `publisherMembers` rows for authority.

## Best Possible Solution

Apply the `linkedUserId` boundary to every publish, manage, and transfer authorization entry point, especially `resolvePublishTargetForUserInternal`, with focused package/skill publish regressions and redacted live denial proof before merge.

## Reproduction Assessment

Yes. ## Solution Assessment

No, not yet. Splitting authorization by publisher kind is the right direction, but the PR still needs the shared publish resolver and publish-path regression coverage before it is the narrow maintainable fix.
