## Summary

Keep open: the PR moves in the right direction, but the active package publish target resolver still trusts stale membership rows for personal publishers, and the PR body only provides test/CI output rather than real behavior proof.

## What This Changes

The PR changes Convex publisher and package authorization, tests, changelog, and org specs so personal publishers rely on `linkedUserId` instead of stale membership rows for management and transfers.

## Best Possible Solution

Harden every active personal-publisher authorization entry point used by publish, manage, and transfer flows, especially `resolvePublishTargetForUserInternal`, then land with focused regressions and redacted live proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: a stale `publisherMembers` row for a `kind: "user"` publisher can still satisfy `resolvePublishTargetForUserInternal`, which package publishing uses as its owner target resolver. I did not execute a live Convex mutation in this read-only review.

## Solution Assessment

No, not yet: splitting authorization by publisher kind is the right direction, but the patch misses the active package publish resolver. The maintainable fix is to apply the same `kind === "user"` linked-user boundary in `resolvePublishTargetForUserInternal` and add a regression for package publishes through stale personal-publisher memberships.
