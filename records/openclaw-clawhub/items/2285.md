## Summary

Keep this PR open: the latest head fixes the earlier publish-target resolver gap, but the personal-publisher boundary is still incomplete because other membership-based authorization and owner-read paths still trust stale rows, and the external PR still lacks real after-fix behavior proof.

## What This Changes

The PR changes publisher membership, package transfer, publish-target resolution, tests, changelog, and org specs so personal publishers authorize by `linkedUserId` while org publishers continue to use memberships.

## Best Possible Solution

Centralize a publisher-kind-aware authorization helper and use it for every membership-based personal-publisher decision, including publish resolution, skill/package transfers, dashboards, private reads, moderation-status reads, and stale-row cleanup, with regressions for stale personal rows and positive org memberships.
