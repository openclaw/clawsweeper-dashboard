## Summary

This PR should stay open: current head narrows several membership and transfer paths, but the active package publish resolver still trusts stale personal-publisher membership rows, and the behavior proof is only tests/CI output.

## What This Changes

The PR updates Convex publisher membership, shared publisher authorization helpers, package-owner transfer checks, tests, changelog, and org specs so personal publishers authorize through `linkedUserId` rather than collaborative membership.

## Best Possible Solution

Apply the personal-publisher `linkedUserId` boundary in `resolvePublishTargetForUserInternal`, keep org membership behavior intact, add package publish regression coverage, and land only after redacted live proof shows stale-membership denial.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: a user with a stale `publisherMembers` role on another user's personal publisher can pass `resolvePublishTargetForUserInternal` during package publish. I did not execute a live Convex mutation in this read-only review.

## Solution Assessment

No, not yet: splitting authorization by publisher kind is the right direction, but the active package publish target resolver still needs the same personal-publisher boundary and a focused regression.
