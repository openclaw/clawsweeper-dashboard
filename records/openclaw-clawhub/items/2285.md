## What This Changes

The PR changes publisher membership helpers, package ownership transfer checks, tests, changelog, and org specs so personal publishers use `linkedUserId` rather than collaborative membership rows.

## Best Possible Solution

Update the PR to preserve or backfill legacy unlinked personal publishers while consistently ignoring extra personal-publisher memberships, add focused regressions, then require redacted real runtime proof for the auth paths.

## Reproduction Assessment

Yes, from source: current main allows stale membership rows in several personal-publisher authorization paths, and the PR head shows the intended split. The patch regressions are also source-reproducible because `linkedUserId` remains optional and some access helpers still trust membership rows.

## Solution Assessment

No, not yet. The linked-user versus org-membership split is the right direction, but the patch needs legacy-data handling, consistent access-surface coverage, and real behavior proof before it is the safest fix.
