## Summary

Keep this PR open: it is a valid security fix direction, but the current branch still leaves ownership-transfer paths that can honor stale personal-publisher membership rows, and it lacks required real behavior proof.

## What This Changes

The branch adds publisher-helper and membership-flow guards so personal publishers authorize by `linkedUserId`, lets linked users remove stale extra personal memberships, and adds tests/spec/changelog coverage.

## Best Possible Solution

Apply the same `linkedUserId`/org-only personal-publisher rule to all ownership-management paths, especially package and skill transfers, then add focused regression coverage and real runtime proof.
