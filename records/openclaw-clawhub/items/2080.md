## Summary

Keep open: current main still has one-way package soft delete for plugins, hides the deleted package, and blocks republishing the same normalized package name; resolving this needs maintainer policy plus production ownership verification, not cleanup closure.

## What This Changes

Clarify and fix recovery after a ClawHub plugin package is deleted so org owners can restore, repair ownership, hard-delete, or understand slug reservation before republishing.

## Best Possible Solution

Define package deletion semantics, then ship an audited owner/admin recovery path and update UI, CLI, API docs, and specs so package name reservation is explicit.

## Reproduction Assessment

Yes for the central blocked-republish behavior: source inspection shows dashboard/API deletion sets softDeletedAt and current publish rejects the same normalized package name. The exact production ownerPublisherId state still needs maintainer verification.

## Solution Assessment

Unclear. A restore/undelete or admin recovery flow is likely safer than freeing deleted names by default, but maintainers need to choose the reservation and recovery policy first.
