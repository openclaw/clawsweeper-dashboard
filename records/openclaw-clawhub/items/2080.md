## Summary

Keep open: current main still reserves the deleted package row by `normalizedName`, blocks publishing another release to that name, and has no package restore/undelete or soft-deleted transfer recovery path. The reporter's exact error text has changed on main, but the org-owner recovery problem remains unresolved and needs maintainer policy.

## What This Changes

Request a supported recovery policy for org-owned package names that remain reserved after UI package deletion.

## Best Possible Solution

Define package deletion semantics, then add one audited package restore, ownership-repair, or name-release flow with UI/CLI/docs copy that accurately tells owners whether names remain reserved.

## Reproduction Assessment

Yes, at source level: delete through the dashboard/API soft-deletes the package and releases, then publishing the same normalized name reaches `insertReleaseInternal` and fails because the soft-deleted row is still found. I did not run a live Convex publish against production data.

## Solution Assessment

No current solution is complete. Current main tells users to restore deleted packages, but only active packages can be transferred and no package restore/undelete path exists.
