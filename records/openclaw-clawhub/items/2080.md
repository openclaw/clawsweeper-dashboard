## Summary

Keep open under the ClawHub issue policy; current main still preserves deleted package rows and ownership keys while publish resolves by global package name, and the reporter confirmed that this matches their production case.

## What This Changes

Request a maintainer-supported recovery or policy fix for org-owned package names that remain reserved or mapped to a stale publisher after dashboard deletion.

## Best Possible Solution

Define package delete and name-reservation semantics, then implement an explicit package recovery path and align dashboard, CLI, and API wording with that policy.

## Reproduction Assessment

Yes, as a source-level path: dashboard/API delete soft-deletes the package row while publish still resolves that row by normalizedName and enforces its stored owner key. I did not run a live production Convex reproduction, but the reporter confirmed the same observable failure.

## Solution Assessment

Unclear until maintainers choose the intended package-name reservation policy. A maintainable fix should avoid silently creating a second package identity and should make restore, transfer, hard delete, or permanent reservation explicit.
