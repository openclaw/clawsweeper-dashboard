## Summary

Keep open: current main explains the reported behavior and has delete/undelete/transfer/admin repair primitives, but the live stale ownership recovery still needs maintainer/admin verification and security/product judgment; ClawHub cleanup policy also does not auto-close issues.

## What This Changes

The issue requests a supported recovery path when a UI-deleted plugin package is hidden from ClawHub but its package name or ownerPublisherId still blocks publishing.

## Best Possible Solution

Maintainers should inspect the live package and publisher rows, verify requester authority, apply an audited restore/transfer/name repair or hard-delete only when appropriate, and then document the approved deleted-package reservation and recovery semantics.

Unclear as an automatic source fix: restore, transfer, and admin repair primitives already exist, but stale/deleted owner recovery needs maintainer security and product judgment before expanding self-service behavior.
