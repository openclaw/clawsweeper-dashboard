## Summary

Keep open under the ClawHub issue policy; current main clarifies the publish error for deleted packages, but still reserves the deleted package name and has no restore, hard-delete, or deleted-row transfer recovery path.

## What This Changes

The issue asks for a recovery or clarified reservation policy when a UI-deleted org-owned plugin package remains reserved and blocks republishing.

## Best Possible Solution

Define package deletion semantics, add an audited restore/undelete or permanent-removal recovery path, and update dashboard/API/CLI wording to make name reservation explicit.

## Reproduction Assessment

Yes, as a source-level path: dashboard/API delete soft-deletes the package row, publish resolves that row by global name, and current main rejects new releases on the deleted package. I did not run a live Convex production-row reproduction.

## Solution Assessment

Unclear until maintainers choose the intended reservation policy. The current clearer error is useful but not sufficient because no supported recovery path exists for a deleted package row.
