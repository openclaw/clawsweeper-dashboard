## Summary

Keep open: current main still soft-deletes package rows and releases, hides them from public package surfaces, and blocks publishing the same normalized package name again without a package restore or recovery path. The broader lifecycle request remains open at https://github.com/openclaw/clawhub/issues/1370, but this issue has a concrete production recovery case and should not be closed as cleanup.

## What This Changes

The issue requests a maintainer-supported recovery path after UI deletion leaves a plugin package name reserved or mapped to stale ownership.

## Best Possible Solution

Define package deletion/reservation semantics, then add an audited owner/admin restore or recovery flow and use it to handle the reporter's production package after ownership verification.

## Reproduction Assessment

Yes, from source inspection. Dashboard/API deletion sets softDeletedAt on the package and releases, public surfaces hide the package, and publish rejects the existing normalized name; I did not run a live production publish against the reporter's row.

## Solution Assessment

Unclear. Releasing the package name by default is not clearly safe; an audited restore or owner/admin recovery path is the safer direction, but maintainers need to choose the lifecycle policy first.
