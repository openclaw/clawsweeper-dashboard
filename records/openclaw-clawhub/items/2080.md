## Summary

Keep open: current main still reserves package rows by global normalized name, blocks publishes against soft-deleted or mismatched-owner package records, and hides soft-deleted packages from manage/staff lookup. There are CLI/admin recovery tools, but the reported stale ownership and slug-reservation recovery path is not definitively solved for affected publishers.

## What This Changes

The issue asks for a supported recovery path when deleting a ClawHub plugin package hides it from UI/search but leaves the package name or ownerPublisherId mapping blocking future publishes.

## Best Possible Solution

Define a guarded package recovery policy that distinguishes restore, owner transfer, hard delete, and slug release, then make the UI/CLI/docs expose that outcome without weakening publisher namespace authority.

## Reproduction Assessment

No high-confidence live reproduction was run against the named production packages. Source inspection does reproduce the relevant class: soft-deleted package rows and mismatched ownerPublisherId records still block publish by normalizedName.

## Solution Assessment

No, the current behavior is not a complete solution because it offers partial CLI/admin recovery but still leaves deleted packages hidden from manage lookup and unresolved stale-owner policy. A safer fix needs maintainer-approved recovery semantics and focused regression coverage.
