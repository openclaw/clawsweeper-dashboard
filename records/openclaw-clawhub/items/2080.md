## Summary

Keep open: current main still hides soft-deleted plugin packages, preserves the package-name record, blocks publishing another release, and has no owner/admin package restore or deleted-record recovery path. The ClawHub cleanup policy also keeps issues open unless a maintainer resolves them outside this review.

## What This Changes

Confirm and provide a recovery path when a plugin package deleted from the UI remains hidden but blocks republishing under the same org-owned package name.

## Best Possible Solution

Define package deletion semantics, then implement an audited owner/admin package restore or recovery flow across Convex, API, CLI, UI, docs, and a spec note for namespace reservation behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: delete a plugin package through the dashboard or package DELETE API, then publish the same normalized name and hit the soft-deleted package rejection while the package is hidden from read and owner surfaces.

## Solution Assessment

Unclear. Restore or undelete looks like the likely maintainable direction, but maintainers first need to choose whether package deletion reserves names forever, releases names on a policy schedule, supports hard-delete, or remains irreversible with clearer warnings.
