## Summary

Keep open: current main still keeps deleted package rows as reserved package identities and blocks publishing when the existing row is deleted or mapped to a different owner. Resolving the remaining recovery cases changes package namespace authority, so this needs maintainer/security policy and likely live admin record review.

## What This Changes

The issue requests supported recovery or clear semantics when a UI-deleted package remains reserved or mapped to a stale publisher and blocks republishing.

## Best Possible Solution

Define and document package delete/recovery semantics, then use audited owner/admin restore, transfer, repair, or name-release flows only after verifying namespace authority.

## Reproduction Assessment

Yes, at source level: current main hides soft-deleted packages from public reads while retaining the normalized package row, and publish rejects deleted or different-owner rows. The exact production records still require admin inspection.

## Solution Assessment

Unclear as an automatic fix: restore, transfer, and admin repair primitives are safer than a publish-time bypass, but maintainers must choose whether deletion reserves names or permits verified release/reassignment.
