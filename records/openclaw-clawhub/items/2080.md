## Summary

Keep open. ClawHub cleanup policy keeps issues open, and although current main now has package restore/transfer primitives, this specific stale package ownership case still needs production record verification and a maintainer recovery decision.

## What This Changes

The issue asks for a recovery path when a plugin package deleted from the UI remains reserved or tied to a stale publisher and blocks republishing.

## Best Possible Solution

Have a maintainer inspect the production package, release, publisher, and audit-log rows, then use package undelete if ownership is correct or an audited admin repair/transfer/hard-delete path if the stored publisher is stale.

## Reproduction Assessment

Partly. Source inspection shows the current path where soft-deleted packages disappear from public reads but remain reserved and blocked during publish; the exact belongs-to-another-publisher outcome depends on production ownerPublisherId state that is not available locally.

## Solution Assessment

Keeping this open is the right outcome. The narrow safe path is audited recovery after production verification, not silently freeing or reassigning the package name during publish.
