## Summary

Keep open: ClawHub issue cleanup policy keeps issues open, and current main still has one-way package soft-delete behavior that hides the package while reserving the normalized name and blocking republish until maintainers define recovery semantics.

## What This Changes

The issue requests recovery semantics for plugin packages that are deleted from the UI but continue to reserve the package name and block republishing.

## Best Possible Solution

Define package delete semantics, then add an audited owner/admin restore or recovery path and update the UI/docs to state whether deleted package names remain reserved.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: delete a plugin package through the dashboard or package DELETE API, then publish the same normalized package name and hit the soft-deleted package rejection while read/UI surfaces hide the package.

## Solution Assessment

Unclear. A restore or undelete path is likely safer than releasing names immediately, but maintainers need to choose the package deletion, reservation, hard-delete, and ownership-recovery policy first.
