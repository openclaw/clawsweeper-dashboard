## Summary

Keep open under the ClawHub issue policy: current main partially improves the publish error for soft-deleted packages, but it still preserves the deleted package identity and has no package restore, undelete, or soft-deleted transfer/repair path.

## What This Changes

The issue requests clear recovery semantics for org-owned plugin package names that remain reserved after a dashboard delete blocks republishing.

## Best Possible Solution

Pick an explicit package-delete policy and implement one maintainer-supported recovery path, with dashboard/API/CLI wording that states whether names remain reserved.

## Reproduction Assessment

Yes, from source inspection: dashboard/API delete sets `softDeletedAt` on package and releases while preserving name and owner fields, and publish name lookup sees that row before rejecting deleted-package publishes. I did not run a live Convex production-row reproduction in this read-only pass.

## Solution Assessment

Unclear. The clearer error from https://github.com/openclaw/clawhub/pull/2073 is useful, but the durable solution needs maintainer choice among restore/undelete, ownership repair/transfer, hard delete, or permanent reservation messaging.
