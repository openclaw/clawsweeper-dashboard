Keep open. ## What This Changes

The issue asks for a supported recovery path when a ClawHub plugin package is hidden/deleted but its package slug or owner mapping remains reserved and blocks later publishes.

## Best Possible Solution

Maintainers should inspect the affected production package documents, apply a narrow audited restore/transfer/identity-repair or hard-delete decision, and clarify the package deletion/recovery policy in owner-facing docs or UI.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainers inspect the production records and decide the recovery policy. The safest direction is an audited restore, transfer, or admin repair path with equivalent authorization checks, not automatic slug release.
