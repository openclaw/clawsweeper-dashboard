## Summary

Keep open: current main still soft-deletes plugin packages in the UI/CLI, hides them from public surfaces, and then blocks publishing the same package name again without a package restore or owner recovery path.

## What This Changes

The issue requests a maintainer-supported recovery path when deleting a plugin package from the UI leaves the package namespace reserved or owned and republish fails.

## Best Possible Solution

Define package deletion and reservation semantics, then implement an audited package restore or owner/admin recovery flow across Convex, HTTP/CLI/UI, docs, and tests.

## Reproduction Assessment

Yes, from source inspection. Dashboard/API delete sets softDeletedAt on package and release rows, public surfaces hide those rows, and publish rejects the existing normalized package name; I did not run a live production publish for the reporter's row.

## Solution Assessment

Unclear. Releasing the package name by default is not clearly safe; an audited restore/undelete or admin recovery path looks safer, but maintainers need to choose the lifecycle policy first.
