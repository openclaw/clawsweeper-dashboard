## Summary

Keep open. ClawHub policy for this review run says issues should stay open, and current main still implements the lifecycle gap described here: dashboard deletion soft-deletes and hides package rows while publish continues to treat the retained row as a reserved package name and can report publisher ownership conflicts for stale owner data.

## What This Changes

The issue asks for a recovery path or clearer semantics when a UI-deleted plugin package remains reserved and cannot be republished by an org owner.

## Best Possible Solution

Keep soft-deleted names reserved by default, then add or use an audited maintainer recovery path for restore, ownership repair/transfer, or hard delete after inspecting the affected package records.

## Reproduction Assessment

Unclear for the exact production error: source inspection shows UI delete hides but retains the package reservation, while the specific 'belongs to another publisher' result depends on the affected production package ownerPublisherId data. The current source path is enough to reproduce the reserved-name lifecycle gap without production access.

## Solution Assessment

Yes for keeping this open. The maintainable direction is an explicit, audited recovery or ownership repair workflow rather than allowing ordinary publish to implicitly reclaim soft-deleted package names.
