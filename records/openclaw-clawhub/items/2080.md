## Summary

Keep open: current main still hides UI-deleted plugin packages while keeping the package row and normalized name reserved, and there is no owner-facing restore or stale-owner recovery path for the reporter’s case.

## What This Changes

The issue asks how org owners can recover or republish a package after UI deletion hides it while the package slug or owner mapping still blocks publish.

## Best Possible Solution

Define package deletion and reservation semantics, then add an audited owner/admin package restore or recovery flow with UI/CLI/docs coverage and regression tests, coordinating with https://github.com/openclaw/clawhub/issues/1370.

## Reproduction Assessment

Yes, from source inspection. UI delete soft-deletes and hides package rows, while publish still rejects the existing normalized package name; I did not run a production publish against the reporter’s row.

## Solution Assessment

Unclear. Releasing the name by default is not clearly safe; an audited restore/undelete or maintainer recovery flow is the safer direction until maintainers decide the lifecycle policy.
