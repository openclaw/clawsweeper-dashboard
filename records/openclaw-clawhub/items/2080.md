Keep open. Current main has package delete/undelete and transfer primitives, but the reported package still needs maintainer inspection because the observed error matches an ownership mismatch, while normal soft-deleted package publishing should now tell the owner to restore first.

## What This Changes

The issue requests a recovery path when a plugin package deleted from the UI is still blocked by package ownership or slug reservation during republish.

## Best Possible Solution

A maintainer should inspect the production package record and either restore it, repair/transfer ownership with audit, or open a focused UI/docs task that explains soft-delete reservation and exposes the approved recovery path.

## Solution Assessment
