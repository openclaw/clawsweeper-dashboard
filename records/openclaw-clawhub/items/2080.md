## Summary

Keep open. Current main confirms package deletion is a soft-delete that hides the package while retaining the normalized package row, and publish still blocks deleted or owner-mismatched rows; the production ownership mismatch needs maintainer recovery rather than cleanup closure.

## What This Changes

The issue asks for a supported recovery or clarification path when a UI-deleted org-owned package remains hidden but its name or ownership still blocks republishing.

## Best Possible Solution

Verify the production package row and provide an audited maintainer recovery path for this package, while coordinating broader owner-facing package restore and lifecycle semantics through https://github.com/openclaw/clawhub/issues/1370.

## Reproduction Assessment

Partly: source inspection shows dashboard delete soft-deletes package rows/releases, public reads hide them, and publish rejects soft-deleted or owner-mismatched existing rows. The exact production “belongs to another publisher” state still needs maintainer access to the live package record.

## Solution Assessment

Unclear until maintainers choose recovery semantics. The safest path is audited restore, owner repair, or hard-delete after production verification, not silent name reuse during publish.
