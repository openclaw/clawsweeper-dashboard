## Summary

Keep open. This is an active, recent issue and current main still preserves soft-deleted package rows by normalized name, which matches the reported stale ownership/slug reservation recovery gap; ClawHub policy also says issues are not cleanup-close candidates here.

## What This Changes

The issue asks for a maintainer-supported recovery policy when a package is deleted from the UI but its slug or ownership mapping still blocks republishing.

## Best Possible Solution

Define explicit package deletion/reservation semantics and add an audited restore, transfer, owner-repair, or hard-delete recovery path for verified package owners or admins.

## Reproduction Assessment

Yes from source inspection: dashboard deletion soft-deletes the package, public views hide the row, and publishing still resolves the preserved normalizedName and blocks it. I did not run a live production publish against the reporter's package.

## Solution Assessment

Unclear. A blind release of the slug on delete is not clearly safe; the maintainable solution is an explicit audited recovery flow with ownership verification and clear UI messaging.
