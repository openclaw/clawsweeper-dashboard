## Summary

Keep open. Current main already exposes suspicious/moderation state through the UI and JSON API, but the static skill-page head still lacks the machine-readable moderation metadata requested for HTML-only agent fetches; ClawHub cleanup policy also keeps issues open for maintainer handling.

## What This Changes

The issue requests machine-readable suspicious/moderation status in static skill-page HTML and API responses so agents can avoid flagged skills.

## Best Possible Solution

Add stable public HTML head metadata for already-public skill moderation state, source it from `loaderData.initialData.result.moderationInfo`, keep the v1 JSON moderation response as the API source of truth, add focused flagged/clean-page tests, and document how agents should prefer API data when available.
