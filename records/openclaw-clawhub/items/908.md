## Summary

Keep open. Current main still surfaces upload/publish validation through a later aggregate Validation panel without field-local accessible messages, and ClawHub cleanup policy keeps issues open unless a PR is certainly implemented on main.

## What This Changes

The issue requests clearer upload-page input validation feedback, preferably under each input or as a popup.

## Best Possible Solution

Add accessible field-level validation for publish/upload metadata inputs, upload/dropzone state, and license acceptance while preserving the aggregate Validation card as the complete preflight summary.

## Reproduction Assessment

Yes. Source inspection shows /upload redirects to /publish-skill, where invalid submit scrolls the aggregate Validation card and the individual controls lack field-local validation text or aria-describedby wiring.

## Solution Assessment

Yes. Reusing the existing validation source for inline accessible field messages while keeping the summary card is the narrowest maintainable direction; a popup-only reminder would be weaker for accessibility.
