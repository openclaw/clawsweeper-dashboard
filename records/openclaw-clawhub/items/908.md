## Summary

Keep open. Current main still routes the upload page to the publish-skill form and surfaces validation through a later aggregate Validation card instead of field-local accessible messages, and ClawHub cleanup policy keeps issues open unless they are PRs already implemented on main.

## What This Changes

The issue requests clearer upload-page input validation errors, preferably displayed under invalid inputs or surfaced more prominently.

## Best Possible Solution

Add accessible field-level validation for publish/upload metadata inputs, dropzone/file state, and license acceptance while retaining the aggregate Validation card as the complete preflight summary.

## Reproduction Assessment

Yes. Source inspection shows /upload redirects to /publish-skill, where invalid submit scrolls to a later aggregate Validation card and the individual controls lack field-local validation text or aria-describedby wiring.

## Solution Assessment

Yes. Reusing the existing validation source for inline accessible field messages while keeping the summary card is the narrowest maintainable direction; a popup-only reminder would be weaker for accessibility.
