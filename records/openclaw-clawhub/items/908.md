## Summary

Keep open. ClawHub policy keeps issues open, and current main still surfaces upload/publish validation through an aggregate Validation card instead of field-local, accessible feedback near the invalid controls.

## What This Changes

Show upload-page input validation errors closer to the affected controls or through clearer immediate feedback.

## Best Possible Solution

Keep the issue open and add accessible field-level errors to the upload/publish form while retaining the existing Validation card as the full preflight summary.

## Reproduction Assessment

Yes. Rendering the Upload component with empty required fields shows validation text and a disabled publish button, and current source places those messages in the later Validation card rather than associating them with the controls.

## Solution Assessment

Yes. The narrow maintainable fix is to derive field-specific messages from the existing validation model, wire them with aria-invalid/aria-describedby, and preserve the aggregate summary for complete preflight context.
