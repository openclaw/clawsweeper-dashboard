Keep open. Current main still renders upload/publish validation as an aggregate Validation card rather than field-local accessible messages, and ClawHub cleanup policy keeps issues open unless they are PRs already implemented on main.

## What This Changes

The issue asks for upload-page input validation errors to be displayed near the invalid inputs or surfaced more prominently.

## Best Possible Solution

Add accessible field-level validation for the publish/upload metadata inputs, upload/dropzone state, and license acceptance while preserving the aggregate Validation card as the complete preflight summary.

## Reproduction Assessment

Yes. Source inspection against current main shows /upload redirects to /publish-skill, where validation messages are rendered in a later aggregate panel rather than under or described by the invalid controls.

## Solution Assessment

Yes. Reusing the existing validation model for field-local accessible messages is the narrowest maintainable direction; a popup-only reminder would be weaker for accessibility.
