Keep open. ClawHub policy keeps issues open, and current main still presents upload/publish validation through an aggregate Validation panel rather than field-local messages or accessible invalid-state wiring near the affected controls.

## What This Changes

## Best Possible Solution

Add accessible field-level validation for metadata inputs, upload/dropzone file checks, slug collision, and license acceptance while retaining the aggregate Validation panel as a complete preflight summary.

Yes. Render the current `Upload` component with empty fields: validation messages appear in the Validation panel and the submit button is disabled, while the invalid controls lack nearby messages or `aria-invalid` / `aria-describedby` associations.

## Solution Assessment

Yes. Field-local messages backed by the existing validation source are the narrow maintainable fix; a toast-only prompt would not solve the placement or accessibility relationship problem.
