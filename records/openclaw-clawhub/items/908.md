## Summary

Keep open. Current main still routes the upload page into the publish form and surfaces validation through a separate aggregate Validation panel, while the affected inputs/dropzone/checkbox do not have field-local error text or field-specific ARIA error relationships.

## What This Changes

The issue asks for upload-page validation errors to be shown closer to the affected inputs or surfaced more prominently.

## Best Possible Solution

Add accessible field-level validation to the publish/upload form for metadata inputs, upload/dropzone state, and license acceptance while keeping the aggregate Validation panel as the full preflight summary.

## Reproduction Assessment

Yes. Source inspection on current main shows /upload redirects to /publish-skill, and the publish form renders required-field errors only in the aggregate Validation card without field-local error text or ARIA error relationships.

## Solution Assessment

Yes. The narrow maintainable direction is to add field-local accessible messages derived from the existing validation state while preserving the summary panel; a toast-only popup would be less discoverable and weaker for accessibility.
