## Summary

Keep open. ClawHub policy keeps issues open, and current main still routes the upload page to a publish form where validation errors are collected in a separate Validation panel rather than shown inline or accessibly associated with invalid controls.

## What This Changes

The issue asks for upload-page input validation errors to be shown near the relevant fields or surfaced more immediately instead of being hidden deep in the page.

## Best Possible Solution

Add accessible field-level validation to the publish/upload form for metadata inputs, upload/dropzone state, and license acceptance while preserving the aggregate Validation panel as a complete preflight summary.

## Reproduction Assessment

Yes. Rendering the upload/publish form with empty defaults or following /upload to /publish-skill shows required errors only in the Validation panel, while the invalid controls lack inline error text and aria error relationships.

## Solution Assessment

Yes. The narrow maintainable fix is field-local accessible validation plus the existing summary panel; relying only on a popup/toast would not address discoverability or accessibility as well.
