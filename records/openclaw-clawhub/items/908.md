## Summary

Keep open. Current main still routes upload to the publish form and renders validation as a separate aggregate panel rather than field-local messages or accessible error relationships, so the requested UX/accessibility improvement remains.

## What This Changes

Request upload-page input validation errors to be shown near the affected inputs or as a more visible popup.

## Best Possible Solution

Add accessible field-level validation to the publish/upload form while preserving the aggregate Validation panel as a full preflight summary.

## Reproduction Assessment

Yes. Source inspection shows rendering the upload/publish form with empty defaults produces required validation messages only in the later Validation panel, with no field-local error text or aria error relationships on the invalid controls.

## Solution Assessment

Yes. Field-local accessible validation plus the existing summary panel is the narrowest maintainable direction; a popup-only solution would be less discoverable and weaker for accessibility.
