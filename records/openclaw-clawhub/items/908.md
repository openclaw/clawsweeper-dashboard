## Summary

Keep open. Current main still routes the upload page to the publish form and shows validation as an aggregate Validation panel rather than field-local messages or accessible error relationships, so the requested UX/accessibility improvement remains.

## What This Changes

The issue asks for upload-page validation errors to appear closer to the relevant inputs or more prominently instead of being hidden in the current form flow.

## Best Possible Solution

Add accessible field-level validation for metadata inputs, upload/dropzone state, and license acceptance while preserving the existing aggregate Validation panel as the complete preflight summary.

## Reproduction Assessment

Yes. Source inspection shows /upload redirects to /publish-skill, and the publish form renders required validation messages only in the later Validation panel with no field-local error text or aria error relationships.

## Solution Assessment

Yes. Field-local accessible validation plus the existing summary panel is the narrowest maintainable direction; a popup-only solution would be less discoverable and weaker for accessibility.
