## Summary

Keep open. Current main still routes upload publishing through `/skills/publish`, where validation is rendered in a later aggregate panel rather than as field-local accessible messages, and ClawHub cleanup policy keeps issues open.

## What This Changes

The issue requests clearer upload-page validation errors near the relevant input controls instead of hidden deeper in the page.

## Best Possible Solution

Add accessible field-level validation to the current skill publish/upload form for metadata inputs, upload/dropzone state, owner migration, and license acceptance while preserving the aggregate Validation card as the complete preflight summary.

## Reproduction Assessment

Yes. Source inspection of current main shows `/upload` redirects to `/skills/publish`, where invalid state is summarized in a later Validation card and the individual controls lack inline validation messages or field-specific accessibility wiring.

## Solution Assessment

Yes. Reusing the existing validation source for inline field messages plus `aria-invalid` / `aria-describedby`, while keeping the summary card, is narrower and more accessible than a popup-only reminder.
