## Summary

Keep open. ClawHub policy keeps issues open, and current main still renders upload/publish validation as an aggregate Validation panel rather than field-local, accessible messages near the invalid controls.

## What This Changes

The issue asks for upload-page input validation errors to be shown near the relevant fields or surfaced more immediately instead of being hidden deep in the page.

## Best Possible Solution

Add accessible field-level validation to the publish/upload form for metadata inputs, upload/dropzone state, and license acceptance while preserving the aggregate Validation panel as a complete preflight summary.

## Reproduction Assessment

Yes. Current source gives a concrete path: /upload redirects to /publish-skill, and with empty/default fields the required messages are rendered only in the Validation panel while the invalid controls lack field-level error text and aria error relationships.

## Solution Assessment

Yes. The narrow maintainable direction is field-local accessible validation plus the existing summary panel; a toast-only popup would be less discoverable and would not address screen-reader association.
