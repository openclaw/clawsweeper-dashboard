## Summary

Keep this PR open for maintainer/member follow-up: current main does not include the shared metering work, and the branch still has an upgrade-sensitive blocker where missing `DOWNLOAD_METERING_HMAC_SECRET` can silently stop public download counter events.

## What This Changes

This PR adds shared Convex download metering for skill and package downloads with HMAC identity hashing, daily dedupe/rollup tables, weekly snapshots, cron jobs, tests, and an internal spec note.

## Best Possible Solution

Land shared metering only after counter continuity on missing-secret deployments is preserved or deployment is explicitly gated, with regression coverage for configured and unconfigured secret paths.

## Reproduction Assessment

Yes. Source inspection is enough: the new HMAC helper throws when the secret is absent, and the changed download routes catch that failure after removing the legacy counter write path.

## Solution Assessment

No, not as currently proposed. Shared metering is a maintainable direction, but the rollout should preserve counter continuity or fail setup/deploy before traffic reaches an unmetered backend.
