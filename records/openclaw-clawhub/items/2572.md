## Summary

Keep open for maintainer handling: current main protects package and catalog cursors from query-shape drift, while this PR removes that compatibility layer and can reintroduce stale-cursor failures or skipped catalog sources.

## What This Changes

The PR removes query-key metadata, cursor-reset propagation, and associated stale-cursor tests from package and HTTP catalog listing cursors while keeping the family-scoped downloads-index behavior.

## Best Possible Solution

Keep the family-scoped downloads-index fix while preserving safe recovery for mismatched package and merged catalog cursors, unless maintainers explicitly choose and document the cursor compatibility break.

## Reproduction Assessment

Yes from source inspection, but I did not run a live Convex reproduction. The PR removes query-key normalization, then passes decoded inner cursors to .paginate(), and the repo already treats Convex 'cursor is from a different query' errors as stale-cursor failures elsewhere.

## Solution Assessment

No. The safer maintainable path is to keep the simplification only if stale package and merged catalog cursors still reset or retry cleanly; otherwise this is an intentional API compatibility change needing maintainer approval.
