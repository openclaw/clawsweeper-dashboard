Keep open. Current main still leaves the reported CLI failure source-reproducible: local package publish sources are resolved under the resolved global workdir, which can differ from the plugin directory where the user ran `clawhub package publish .`, and no linked PR closes this issue.

## What This Changes

The issue asks for `clawhub package publish . --family code-plugin` to recognize an existing root `openclaw.plugin.json` instead of failing with a missing-manifest error.

## Best Possible Solution

Fix local package-publish source resolution so relative local sources use the caller's intended package directory when no explicit workdir override is supplied, while preserving manifest validation and GitHub source behavior.

## Solution Assessment

Yes. A focused resolver fix with regression tests is the narrow maintainable solution; weakening `openclaw.plugin.json` validation would hide that the CLI read the wrong package directory.
