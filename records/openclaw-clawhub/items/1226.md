## Summary

Keep open. Current main still has a source-reproducible `package publish .` path-resolution bug that can raise the reported manifest error, and the open closing PR https://github.com/openclaw/clawhub/pull/2235 is already the active fix path.

## What This Changes

The issue asks for `clawhub package publish .` to accept a valid local code-plugin directory with a root `openclaw.plugin.json` instead of failing with `openclaw.plugin.json required`.

## Best Possible Solution

Review and land https://github.com/openclaw/clawhub/pull/2235, or an equivalent focused resolver fix, with regression coverage for caller cwd versus configured workdir while preserving explicit workdir, GitHub-source, and compatibility-metadata validation.

## Reproduction Assessment

Yes. Source inspection on current main shows `package publish .` resolves `.` under `opts.workdir`, and `resolveWorkdir` can choose a configured workspace that differs from the plugin cwd, making the exact manifest error reachable.

## Solution Assessment

Yes. The narrow maintainable solution is to resolve local package publish/pack paths from the caller's intended package directory before falling back to the resolved OpenClaw workdir, with tests for the cwd/workdir precedence.
