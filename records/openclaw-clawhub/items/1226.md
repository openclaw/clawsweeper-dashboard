## Repair Work Prompt

Fix the CLI package-publish local source resolution bug from https://github.com/openclaw/clawhub/issues/1226. When `clawhub package publish . --family code-plugin` or another relative local folder source is run from a plugin/package directory and no explicit workdir override is supplied, resolve the source against the caller's intended package directory rather than an unrelated Clawdbot/OpenClaw default workspace. Preserve explicit `--workdir` / `CLAWHUB_WORKDIR` behavior, local-directory precedence for `owner/repo`, GitHub shorthand/URL behavior, `.tgz` handling, and the existing `openclaw.plugin.json` plus compatibility-metadata validation. Add regression coverage for a configured default workspace that differs from the plugin cwd and for precedence cases. Do not remove or weaken package manifest validation. Add a concise changelog/release-note entry only if the repo has an existing ClawHub CLI bug-fix changelog location; otherwise keep the change to code, tests, and any necessary CLI docs clarification.
