## Summary

Keep open. Current main still has the workdir-relative package source resolution that can surface the reported manifest error, and the linked open fix PR at https://github.com/openclaw/clawhub/pull/2235 is the active implementation path.

## What This Changes

The issue requests that `clawhub package publish . --family code-plugin` accept a plugin directory containing `openclaw.plugin.json` instead of failing with `openclaw.plugin.json required`.

## Best Possible Solution

Review and land https://github.com/openclaw/clawhub/pull/2235, or an equivalent focused resolver fix, with regression coverage for caller cwd versus configured workdir while preserving manifest and compatibility validation.

## Reproduction Assessment

Yes. Source inspection shows a high-confidence path: when the resolved CLI workdir differs from the plugin cwd, `package publish .` resolves `.` under `opts.workdir`, then validation checks that folder and can throw `openclaw.plugin.json required`.

## Solution Assessment

Yes. The narrow maintainable solution is to fix local package publish/pack source resolution to prefer the caller's intended package directory before falling back to the resolved workdir, without weakening manifest or compatibility checks.
