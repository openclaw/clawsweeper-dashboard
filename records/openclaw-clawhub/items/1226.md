## Summary

Keep open. The repository cleanup policy keeps ClawHub issues open, and current main still has the reported false-manifest path: local package publish sources are resolved relative to the CLI's resolved global workdir, which can differ from the plugin directory where the command was run.

## What This Changes

The issue reports that `clawhub package publish . --family code-plugin` rejects a local code plugin with `openclaw.plugin.json required` even though the manifest exists in the package root.

## Best Possible Solution

Update the CLI so local package publish sources resolve against the caller's intended package directory when no explicit workdir override is supplied, while preserving explicit workdir/env behavior and GitHub source handling. Add regression coverage for the configured-workspace versus plugin-cwd case so the manifest check reads the actual package files.

## Repair Work Prompt

Fix the local source resolution bug for `clawhub package publish` reported here. Current main resolves relative local package sources like `.` against `opts.workdir`, and `opts.workdir` can fall back to a configured Clawdbot/OpenClaw workspace instead of the directory where the user invoked the command. Implement the smallest CLI change so local package publish sources resolve against the caller's intended package directory when no explicit workdir override is supplied, while preserving explicit `--workdir`, `CLAWHUB_WORKDIR`/`CLAWDHUB_WORKDIR`, GitHub URL/shorthand handling, local-directory-over-GitHub-shorthand precedence, source attribution, and existing compatibility validation. Add regression tests for a plugin cwd containing `openclaw.plugin.json` when the configured default workspace points elsewhere, and ensure the next error for the reported GuardClaw-style package is compatibility metadata rather than missing manifest. Do not change server package APIs, package manifest requirements, trusted publisher behavior, or unrelated plugin catalog UI. Mention related publish-path reports #48 and #1509 in the PR context if the shared resolution change also affects skill publish.
