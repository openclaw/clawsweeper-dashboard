## Repair Work Prompt

Fix the ClawHub CLI package publish local-source resolution bug reported in https://github.com/openclaw/clawhub/issues/1226. `clawhub package publish . --family code-plugin` should read package files from the caller's intended local source directory when no explicit workdir override is supplied, instead of resolving `.` through a configured Clawdbot/OpenClaw workspace and then failing with `openclaw.plugin.json required`. Keep existing manifest validation, source repo/commit requirements, and GitHub source/shorthand behavior intact. Add regression coverage for a plugin cwd without `.clawhub` markers while configured Clawdbot/OpenClaw default workspace points elsewhere, plus coverage that explicit `--workdir` or env workdir semantics remain intentional. Do not relax code-plugin metadata validation as a workaround.
