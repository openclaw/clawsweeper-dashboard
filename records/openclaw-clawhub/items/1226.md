## Repair Work Prompt

Fix the local package publish resolver bug reported in https://github.com/openclaw/clawhub/issues/1226. Current behavior resolves relative local `clawhub package publish <source>` paths through the CLI's resolved workdir, so `clawhub package publish . --family code-plugin` can read a configured Clawdbot/OpenClaw workspace instead of the caller's plugin repo and fail with `openclaw.plugin.json required`. Keep the fix scoped to local source resolution for package publish: relative local folder and `.tgz` inputs should use the caller's intended package directory when no explicit workdir/env override is supplied, while preserving GitHub URL/shorthand behavior, existing local-directory-over-GitHub precedence, explicit `--workdir`/`CLAWHUB_WORKDIR`, ClawPack publish, and local git source attribution. Add regression tests for a plugin cwd with a separate configured workspace lacking `openclaw.plugin.json`; the next error for the reported GuardClaw package after path resolution may be the newer required compatibility metadata, so do not weaken manifest or compatibility validation. Update specs/docs only if the intended source/workdir semantics change.
