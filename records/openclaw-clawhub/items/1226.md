## Summary

Keep open. This is an open ClawHub issue, current main still has a source-reproducible workdir-relative `package publish .` path that can raise the reported manifest error, and open closing PR https://github.com/openclaw/clawhub/pull/2235 is the active implementation path.

## What This Changes

The issue reports that `clawhub package publish .` rejects a valid code plugin with `openclaw.plugin.json required` even though the manifest exists in the plugin repository root.

## Best Possible Solution

Review and land https://github.com/openclaw/clawhub/pull/2235, or an equivalent focused resolver fix, with regression coverage for caller cwd versus configured workdir while preserving manifest, compatibility, and GitHub-source validation.

## Reproduction Assessment

Yes. Source inspection shows that when the resolved CLI workdir differs from the plugin cwd, `package publish .` resolves `.` under `opts.workdir`, lists that folder, and can throw `openclaw.plugin.json required`.

## Solution Assessment

Yes. The narrow maintainable fix is to resolve local package publish/pack paths from the caller's intended package directory before falling back to the resolved OpenClaw workdir, with tests for cwd/workdir precedence and no relaxation of manifest or compatibility checks.
