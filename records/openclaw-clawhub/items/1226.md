## What This Changes

The issue reports that `clawhub package publish . --family code-plugin` rejects a valid plugin repository with `openclaw.plugin.json required`.

## Best Possible Solution

Review and land https://github.com/openclaw/clawhub/pull/2235, or an equivalent focused resolver fix, with regression coverage for caller cwd versus configured workdir while preserving manifest, compatibility, and GitHub-source validation.

## Reproduction Assessment

Yes. Source inspection shows `package publish .` can resolve `.` under the configured CLI workdir rather than the caller's plugin folder, then throw the exact `openclaw.plugin.json required` error when that resolved folder lacks the manifest.

## Solution Assessment

Yes. The narrow maintainable solution is to resolve local package publish/pack paths from the caller's intended package directory while preserving explicit workdir, env workdir, configured workspace, and GitHub source precedence.
