## What This Changes

The issue reports that `clawhub package publish .` rejects a code plugin with `openclaw.plugin.json required` even when the manifest exists in the local plugin folder.

## Best Possible Solution

Review and land https://github.com/openclaw/clawhub/pull/2235, or apply the same focused cwd-before-configured-workdir resolver fix with regression coverage for package publish and pack.

## Reproduction Assessment

Yes, source-reproducible. Current main resolves `package publish .` from `opts.workdir`, which can be a configured workspace instead of the caller's plugin folder, then raises `openclaw.plugin.json required` when that resolved folder lacks the manifest.

## Solution Assessment

Yes. The linked PR is the best current path because it directly changes local package source resolution to prefer the caller cwd while preserving configured-workdir fallback and GitHub source handling.
