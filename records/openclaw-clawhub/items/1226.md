## Summary

Keep open. Current main still has a source-reproducible CLI path-resolution bug for local package publish sources, and open linked PR https://github.com/openclaw/clawhub/pull/2235 already owns the fix path.

## What This Changes

The issue asks for `clawhub package publish .` to accept a valid code-plugin folder containing `openclaw.plugin.json` instead of failing with the manifest-required error.

## Best Possible Solution

Review and land https://github.com/openclaw/clawhub/pull/2235, or an equivalent focused resolver fix, with regression coverage for caller cwd versus configured workdir.

## Reproduction Assessment

Yes. Source inspection on current main shows `package publish .` resolves `.` under `opts.workdir`, while `resolveWorkdir` can choose a configured workspace different from the plugin cwd and then raise the exact manifest error from that wrong folder.

## Solution Assessment

Yes. The linked PR’s direction is the narrow maintainable fix: resolve local publish/pack paths from the caller’s intended package directory before falling back to the resolved OpenClaw workdir, with focused precedence tests.
