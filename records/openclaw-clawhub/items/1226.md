## What This Changes

The issue reports that `clawhub package publish .` rejects a valid code-plugin folder with `openclaw.plugin.json required`.

## Best Possible Solution

Review and land https://github.com/openclaw/clawhub/pull/2235, or an equivalent focused resolver fix, with regression coverage for caller cwd versus configured workdir.

## Reproduction Assessment

Yes. Source inspection on current main shows `package publish .` resolves `.` under `opts.workdir`, while `resolveWorkdir` can choose a configured workspace different from the plugin cwd and then raise the exact manifest error from that wrong folder.

## Solution Assessment

Yes. The linked PR's direction is the narrow maintainable fix: resolve local package publish/pack paths from the caller's intended package directory before falling back to the resolved OpenClaw workdir, with focused precedence tests.
