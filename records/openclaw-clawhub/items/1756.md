## Summary

Keep open. The current main branch still has a credible source-level path for the reported reorder: the UI replaces the full relevance result array when scroll-driven loading increases the search limit, while the backend builds and slices a limit-dependent ranked pool. ClawHub cleanup policy also keeps issues open unless a maintainer resolves them manually.

## What This Changes

The issue reports that `/skills` relevance search initially omits high-relevance results and silently changes the top ordering after the user scrolls enough to trigger loading more results.

## Best Possible Solution

Make `/skills` relevance pagination stable so the first displayed page for a query remains a prefix of later larger/page requests. A durable fix should decouple recall/candidate expansion from the display limit or page over a stable ranked result set, with backend and UI regression coverage for the initial 25-result request versus the 50-result load-more request.
