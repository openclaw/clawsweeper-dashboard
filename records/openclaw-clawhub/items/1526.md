## Summary

Keep open. The target policy does not auto-close ClawHub issues, and the requested verified-publisher or high-risk-audited path is still not implemented for skills: current main has clean/suspicious/malicious skill moderation, a clean-only moderator override, and admin-controlled trusted-publisher metadata, while the live pop-pay endpoints still report a suspicious verdict.

## What This Changes

The issue asks for a way to distinguish pop-pay as a high-risk but audited or verified-publisher skill rather than treating its sensitive payment-automation capability as ordinary suspicious code.

## Best Possible Solution

Maintainers should make an explicit trust-and-safety policy call: either document that sensitive-capability skills remain under the existing suspicious/manual-override model, or define a staff-managed verified or audited classification that preserves user warnings while distinguishing reviewed high-risk capability from malicious findings. Any pop-pay-specific change should follow artifact and provenance review.
