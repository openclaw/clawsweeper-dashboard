## Summary

Keep open. This is a concrete production GitHub OAuth/session bug, current frontend code requires a returned exchange code to create a session, live auth endpoints are still failing, and ClawHub cleanup policy keeps issues open for maintainer follow-up.

## What This Changes

The issue reports that GitHub OAuth returns to `https://clawhub.ai/` without `?code` or `?error`, so `AuthCodeHandler` cannot exchange a session.

## Best Possible Solution

Verify and repair production Convex Auth, GitHub OAuth, and Vercel site URL alignment so the callback returns either an exchange code or an error, then add a production smoke or diagnostic check for this failure.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not for the exact missing-code callback redirect, because this review did not complete an interactive GitHub OAuth flow; source inspection shows why the frontend cannot create a session without `code`, and live checks reproduce an auth-specific production failure.

## Solution Assessment

Is this the best way to solve the issue? Unclear. The report points at the right production auth-origin boundary, but maintainers need production logs and OAuth/Vercel/Convex configuration checks before choosing a source patch, env repair, or upstream workaround.
