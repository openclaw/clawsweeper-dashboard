Keep open. This is a concrete production GitHub OAuth/session bug, current main still requires the callback to deliver a frontend exchange `code`, live sign-in endpoints are returning HTTP 500, and ClawHub cleanup policy keeps issues open unless a narrow close condition applies.

Reported that GitHub OAuth reaches the Convex callback but redirects to `https://clawhub.ai/` without `?code=` or `?error=`, leaving the frontend unable to create a session.

## Best Possible Solution

Treat this as part of the active OAuth no-session cluster: verify production Convex/GitHub/Vercel auth settings and logs, fix the narrow source or operational cause, then prove a fresh GitHub sign-in stores Convex Auth tokens.
