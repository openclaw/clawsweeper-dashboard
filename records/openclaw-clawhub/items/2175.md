Keep open. This is still an active production GitHub OAuth/session failure: current main depends on a callback `code` or `error`, live sign-in endpoints return HTTP 500, and related OAuth/session reports remain open under the ClawHub issue cleanup policy.

## What This Changes

Restore the GitHub OAuth callback flow so successful production sign-in redirects back to ClawHub with a frontend exchange code or error and creates session tokens.

## Best Possible Solution

Run an operator-backed OAuth/session investigation, restore active GitHub account session creation, preserve explicit deleted/banned-account rejection, and add focused regression coverage plus redacted production smoke proof.
