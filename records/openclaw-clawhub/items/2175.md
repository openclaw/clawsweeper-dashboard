## Summary

Keep open. This is an open ClawHub issue, repository cleanup policy keeps issues open, current source still depends on a callback `code` or `error` to create/surface auth state, and live sign-in endpoints currently fail with HTTP 500 before the reported callback path.

## What This Changes

The issue reports that production GitHub OAuth redirects to the ClawHub app root without a frontend exchange code, leaving users unauthenticated.

## Best Possible Solution

Restore production GitHub OAuth so active users receive a frontend exchange code or explicit error, then add import-safe diagnostics or deploy smoke checks while preserving deleted/banned-account rejection.

## Reproduction Assessment

No high-confidence exact browser reproduction of the callback redirect was completed. Source inspection confirms a no-code redirect cannot create a session, and live production checks currently reproduce a related HTTP 500 before callback.

## Solution Assessment

Unclear because there is no proposed patch and the failure crosses frontend callback handling, Convex Auth routes, production env, OAuth app settings, and account state. The requested direction to check production redirect/auth-origin settings is appropriate, but source-only cleanup is not enough to close it.
