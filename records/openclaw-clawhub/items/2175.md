Keep open. This is an open ClawHub issue and repository cleanup policy keeps issues open; the current deployed auth path is still unhealthy, and current main still depends on a callback `code` or `error` to complete a frontend session.

## What This Changes

The issue reports that GitHub OAuth returns from Convex to the ClawHub root without a frontend exchange code or error, leaving the app unable to create a session.

## Best Possible Solution

Run maintainer/operator OAuth triage, choose the canonical tracker, restore production GitHub sign-in for active accounts, preserve deleted/deactivated/banned-account rejection, and add targeted regression coverage plus redacted production smoke proof.
