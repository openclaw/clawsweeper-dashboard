## Summary

Keep open. This is a concrete production GitHub OAuth/session failure, live production sign-in still returns HTTP 500 before entering OAuth, and ClawHub cleanup policy keeps issues open rather than auto-closing them.

## What This Changes

The issue reports that GitHub OAuth reaches the Convex callback but redirects to the app root without a frontend exchange code, leaving users unauthenticated.

## Best Possible Solution

Run a canonical operator-backed OAuth/session investigation, restore reliable GitHub session creation for active accounts, preserve explicit blocked-account rejection, and add redacted production smoke proof plus focused regression coverage.
