## Summary

Keep open. Current main fixes the GitHub monorepo `--source-path` failure, but org namespace inference is intentionally still not implemented; publishing under an org remains the explicit `--owner` path and needs maintainer product/security judgment.

## What This Changes

The issue asks `clawhub package publish` to publish scoped packages under an org namespace without `--owner` and to support `owner/repo@ref --source-path` for monorepo plugin subdirectories.

## Best Possible Solution

Keep the fixed GitHub subdirectory handling, then either document `--owner <org>` as the supported org-publish contract or explicitly design authorized org inference with backend checks and regression coverage.

## Reproduction Assessment

Partly. Source inspection and existing tests give a high-confidence path for current main: GitHub `owner/repo@ref --source-path <subdir>` is fixed, while org ownership is selected only when `--owner` populates `ownerHandle`.

## Solution Assessment

Unclear for automatic inference. The current explicit `--owner` flow is the narrowest maintainable contract; inferring org ownership from package scope or repo metadata needs maintainer product and security approval.
