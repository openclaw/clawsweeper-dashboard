## Summary

Keep open. Current main fixes the monorepo `--source-path` failure, but organization-owner inference remains intentionally unimplemented and needs maintainer product/security judgment rather than automated cleanup.

## What This Changes

The issue asks `package publish` to handle `owner/repo@ref --source-path` monorepo subdirectories and infer organization ownership from package scope or source repository instead of defaulting to the personal namespace.

## Best Possible Solution

Keep the fixed GitHub subdirectory handling, then either document explicit `--owner <org>` as the supported contract or design authorized org inference with backend membership checks and regression coverage.

## Reproduction Assessment

Partly. Current source and tests give high-confidence evidence that `owner/repo@ref --source-path <subdir>` now works, while the remaining org-inference gap is visible because `ownerHandle` is populated only from `--owner`.

## Solution Assessment

Unclear for inference. The explicit `--owner` path is the narrowest current contract, while inferring org ownership from package scope or repository metadata needs product and security approval before implementation.
