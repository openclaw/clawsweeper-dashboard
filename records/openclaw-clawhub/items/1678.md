## Summary

Keep open. Current main fixes the GitHub monorepo `--source-path` failure, but automatic org namespace inference remains unimplemented and needs maintainer product/security judgment.

## What This Changes

Request package publishing to support org namespace ownership for local and GitHub monorepo subdirectory sources, including inference from package scope or source repo metadata.

## Best Possible Solution

Keep `--owner <org>` as the documented contract unless maintainers approve a backend-authorized inference design with regression coverage for local and GitHub monorepo inputs.

## Reproduction Assessment

Partly. Source inspection shows the old GitHub `--source-path` failure is fixed and covered, and also shows that omitting `--owner` leaves `ownerHandle` absent so the backend defaults to the personal publisher.

## Solution Assessment

Unclear for automatic inference. The current explicit `--owner` workflow is the narrowest maintainable solution; inferring ownership from package scope or repo metadata needs product and security approval.
