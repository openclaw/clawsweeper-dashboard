## Summary

Keep open. Current main fixes the monorepo GitHub `--source-path` path, but org ownership remains explicit through `--owner` / `ownerHandle` and still needs maintainer product/security judgment; ClawHub cleanup policy keeps issues open.

## What This Changes

Request that `clawhub package publish` support publishing a monorepo subdirectory plugin under an org namespace, including GitHub `--source-path` handling and org ownership behavior.

## Best Possible Solution

Keep the fixed GitHub subpath handling, then either document explicit `--owner <org>` as the org-publish contract or implement authorized owner inference with backend membership checks and regression coverage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Partly: source inspection shows the original `--source-path` bug is fixed and covered, while the remaining org-inference gap is clear because `ownerHandle` is only populated from `--owner`.

## Solution Assessment

Is this the best way to solve the issue? No for an automated fix: automatic org inference is not clearly the best policy without maintainer approval, and the current explicit `--owner` path is the narrower authorized solution.
