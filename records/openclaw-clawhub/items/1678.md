## Summary

Keep open. Current main has fixed the GitHub monorepo `--source-path` failure, but org namespace inference from package scope or source repo remains intentionally unresolved and needs maintainer product/security judgment rather than cleanup closure.

## What This Changes

The issue asks package publishing to support monorepo subdirectories and to publish org-scoped packages under the org namespace without requiring an explicit owner flag.

## Best Possible Solution

Keep the shipped monorepo source-path fix, then either document `--owner <org>` as the supported org-publish contract or design authorized org inference with explicit backend checks and regression coverage.

## Reproduction Assessment

Partly yes. Source inspection and tests show the monorepo `owner/repo@ref --source-path <subdir>` path is fixed, while current main still only sends an org owner when `--owner` populates `ownerHandle`.

## Solution Assessment

Unclear for automatic inference. The current explicit `--owner` flow is the narrowest supported contract; inferring ownership from package scope or source repo metadata needs maintainer product and security approval.
