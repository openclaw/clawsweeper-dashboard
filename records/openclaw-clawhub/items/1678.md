## Summary

Keep open. Current main fixes the GitHub monorepo `--source-path` failure, but publishing under an org namespace is still an explicit `--owner` workflow and the requested inference from package scope or source repo remains a product/security decision.

## What This Changes

Request package publishing to support monorepo subdirectories and publish org-scoped packages under the organization without requiring an explicit owner flag.

## Best Possible Solution

Keep the fixed GitHub subdirectory handling, then either document `--owner <org>` as the supported org-publish contract or design authorized org inference with explicit backend checks and regression coverage.

## Reproduction Assessment

Partly. Source inspection and tests give a high-confidence path for the current behavior: `owner/repo@ref --source-path <subdir>` is fixed, while org publishing is only selected when `--owner` populates `ownerHandle`.

## Solution Assessment

Unclear for automatic inference. The current explicit `--owner` flow is the narrowest maintainable contract; inferring org ownership from package scope or repo metadata needs maintainer product and security approval.
