## Summary

Keep open. The report still maps to current main: GitHub shorthand sources are fetched before `--source-path` is applied, while organization publishing is currently an explicit `--owner` / `ownerHandle` contract rather than inferred from package scope or source repository. ClawHub cleanup policy also keeps issues open rather than auto-closing them.

## What This Changes

The issue requests a package-publish flow that can publish a monorepo subdirectory plugin under an organization namespace instead of the authenticated user's personal namespace.

## Best Possible Solution

Apply `--source-path` to resolved GitHub shorthand and URL sources before fetching and add a regression for monorepo subdirectory publishing. Separately choose and document the organization-owner contract, preserving backend publisher membership checks for any inferred or explicit owner.
