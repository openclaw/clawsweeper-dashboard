## Summary

Keep open. Current main fixes the GitHub monorepo `--source-path` failure, but publishing under an org namespace still depends on explicit `--owner` / `ownerHandle`, so the remaining package-scope or source-repo inference request needs maintainer product and authorization review.

## What This Changes

The issue asks package publishing to support org-owned monorepo subdirectory plugins without falling back to the authenticated user's personal namespace.

## Best Possible Solution

Keep the fixed GitHub subdirectory handling, then either document explicit `--owner <org>` as the supported org-publish contract or design authorized org inference with backend membership checks and regression coverage.

## Reproduction Assessment

Partly. Source inspection gives high confidence that `owner/repo@ref --source-path <subdir>` is fixed on current main, while org publishing still only happens when `--owner` populates `ownerHandle`.
