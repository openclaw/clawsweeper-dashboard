## Summary

Keep open. Current main fixes the reported monorepo `--source-path` failure, but org ownership is still intentionally explicit through `--owner` and the remaining request to infer org namespace needs maintainer product/security judgment; the ClawHub cleanup policy also keeps issues open.

## What This Changes

Request to make `clawhub package publish` support org namespaces for local-folder publishes and monorepo subdirectories for GitHub `owner/repo@ref --source-path` inputs.

## Best Possible Solution

Keep the fixed `--source-path` behavior, document and release the explicit `--owner <org>` workflow, and only add owner inference if maintainers define a clear authorized ownership contract.

## Reproduction Assessment

Yes. The original GitHub shorthand failure has a high-confidence regression path in `packages.test.ts`, and the remaining owner-inference behavior is directly observable because the CLI only sends `ownerHandle` when `--owner` is provided.

## Solution Assessment

Unclear for the remaining request. Applying `--source-path` before fetch is the right narrow fix, but automatic org inference is not clearly safer or more maintainable than the current explicit owner selection without a product/security decision.
