## Summary

Keep open under the ClawHub issue policy. Current main appears to fix the monorepo `--source-path` failure, but the remaining org-namespace inference request is a product/security decision because org publishing is currently explicit through `--owner` and backend publisher membership checks.

## What This Changes

The issue asks for `clawhub package publish` to support monorepo subdirectory publishing under an organization namespace, including better handling of org-scoped package names and GitHub source paths.

## Best Possible Solution

Keep the fixed GitHub subdirectory handling, then either document `--owner <org>` as the supported org-publish contract or design authorized org inference with backend membership checks and regression coverage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Partly: source inspection and tests show the original `--source-path` failure is fixed, while the remaining org-inference gap is visible because `ownerHandle` is populated only from `--owner`; I did not run a live publish.

## Solution Assessment

Is this the best way to solve the issue? Unclear for automatic inference: the current explicit `--owner` path is the narrowest maintainable contract, while inference from package scope or repo metadata needs maintainer product and security approval.
