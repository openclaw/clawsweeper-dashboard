## What This Changes

Requested enforcement for case-colliding `SKILL.md`/`skill.md` and `PROTOCOL.md`/`protocol.md` files in skill packages.

## Best Possible Solution

Maintainers should choose a compatibility policy, likely preserving lowercase reads while rejecting case-colliding duplicates, decide whether `PROTOCOL.md` belongs in the package contract, then enforce it consistently across web upload, API/server publish, GitHub import/sync, CLI publish/sync, docs, and any content cleanup plan.

## Reproduction Assessment

Yes, at source level. Current docs and publish/import paths still accept lowercase or plural skill readme aliases and do not consistently reject case-colliding duplicates, though I did not rerun the external Windows checkout reproduction.

## Solution Assessment

Unclear until maintainers choose the format policy. The reporter's May 11 suggestion to keep lowercase compatibility but reject simultaneous case variants is narrower than uppercase-only enforcement, while `PROTOCOL.md` still needs a ClawHub scope decision.
