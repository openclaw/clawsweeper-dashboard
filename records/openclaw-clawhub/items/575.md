## Summary

Keep this issue open. Current main still intentionally allows lowercase and plural skill readme variants across docs, CLI scan/publish, browser publish, GitHub import, and server-side publish validation; enforcing uppercase-only would be a breaking package-format policy decision, not an implemented cleanup.

## What This Changes

The issue requests uppercase-only enforcement for `SKILL.md` and `PROTOCOL.md` to prevent duplicate-case package files and Windows checkout failures.

## Best Possible Solution

Maintainers should define the canonical casing policy and whether `PROTOCOL.md` belongs in the package contract, then update docs, specs, validators, import/upload/publish paths, and any content migration plan together.

## Reproduction Assessment

Not applicable as a strict bug reproduction: current ClawHub docs and validators intentionally allow lowercase and plural skill readme variants. Source inspection gives high confidence that uppercase-only enforcement is absent on current main.

## Solution Assessment

No for an automated narrow fix: enforcing uppercase-only would change the public package contract and add an undefined `PROTOCOL.md` rule. The maintainable path is a maintainer policy decision followed by coordinated validation, docs, tests, and content cleanup.
