## Summary

Keep this issue open: current main still documents and implements lowercase/plural skill readme compatibility, `PROTOCOL.md` is not a defined ClawHub package contract, and this repository cleanup policy keeps ClawHub issues open unless maintainers resolve them.

## What This Changes

The issue requests enforcing uppercase `SKILL.md` and `PROTOCOL.md` package files to avoid duplicate lowercase/uppercase files that fail on case-insensitive filesystems.

## Best Possible Solution

Maintainers should define the canonical casing and `PROTOCOL.md` policy, then update docs, CLI scan/sync/publish, browser upload, GitHub import, server validation, readme resolution, tests, and any content migration plan consistently.

## Reproduction Assessment

Yes for the ClawHub source side: current main clearly accepts and documents lowercase/plural readme variants across the relevant paths. The exact Windows checkout collision is in the external openclaw/skills content repository and was not reproduced in this checkout.

## Solution Assessment

Unclear. Uppercase-only enforcement may prevent case-insensitive filesystem collisions, but it conflicts with the current public package-format contract and should follow an explicit maintainer migration decision.
