## Summary

## What This Changes

Request to enforce canonical casing for skill package metadata files, especially preventing mixed `SKILL.md`/`skill.md` and `PROTOCOL.md`/`protocol.md` collisions on case-insensitive filesystems.

## Best Possible Solution

Define a compatibility-preserving casing policy, decide whether `PROTOCOL.md` belongs in the ClawHub package contract, then apply duplicate-case rejection consistently across docs, import, browser upload, CLI publish, and server validation.

## Reproduction Assessment
