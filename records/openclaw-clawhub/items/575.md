## Label Justifications

- `P2`: The issue causes concrete Windows and case-insensitive-filesystem failures for a limited subset of skill packages.
- `impact:other`: Case-colliding package paths make affected repositories and exported packages unreliable on common filesystems.

## Summary

Current main still permits case-conflicting skill paths, so the cross-platform problem remains. The issue should stay open for a maintainer decision because strict uppercase enforcement would also remove documented lowercase manifest compatibility.

## What This Changes

Require ClawHub skill packages to avoid case-conflicting SKILL.md and PROTOCOL.md paths, with a policy decision on whether uppercase names must be canonical.

## Best Possible Solution

Preserve lowercase manifest compatibility, reject case-folded duplicate file paths at the shared publish boundary and all archive/import entry points, and document retaining `SKILL.md` or `PROTOCOL.md` when resolving a conflict.

## Reproduction Assessment

Yes. The report provides a concrete Windows Git checkout transcript, and current source preserves case-distinct paths without a shared case-folded collision check.

## Solution Assessment

No for unconditional uppercase-only enforcement. Rejecting collisions is the narrowest maintainable solution because lowercase manifests are intentionally documented and accepted today.
