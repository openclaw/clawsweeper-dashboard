## Summary

Keep this issue open: current main still documents and implements lowercase/plural skill readme compatibility, so uppercase-only enforcement is not implemented and would be a breaking skill-format policy decision. The ClawHub review policy for this run also keeps issues open unless maintainers resolve them.

## What This Changes

The issue requests enforcing uppercase-only `SKILL.md` and `PROTOCOL.md` files to prevent duplicate casing variants and case-insensitive filesystem checkout collisions.

## Best Possible Solution

Maintainers should define the canonical casing policy and whether `PROTOCOL.md` belongs in the package contract, then update docs, CLI scan/sync/publish, browser upload, GitHub import, server validation, readme resolution, tests, and any content migration plan consistently.

## Solution Assessment

Unclear. Uppercase-only enforcement may prevent future case-insensitive filesystem collisions, but it conflicts with the current public format contract and needs an explicit maintainer migration decision first.
