## Summary

Keep this issue open. Current main still documents and implements lowercase/plural skill readme compatibility, and this repository's cleanup policy only closes ClawHub PRs under the implemented-on-main rules; enforcing uppercase-only would be a breaking package-format policy decision.

## What This Changes

Request to enforce uppercase-only `SKILL.md` and `PROTOCOL.md` package files and reject lowercase or conflicting duplicates.

## Best Possible Solution

Maintainers should define the canonical casing policy and whether `PROTOCOL.md` belongs in the package contract, then update docs, CLI scan/sync/publish, browser upload, GitHub import, server validation, readme resolution, and any content cleanup plan consistently.

## Reproduction Assessment

Not applicable as a current ClawHub bug reproduction: the existing documented behavior intentionally accepts lowercase variants. Source inspection does confirm that current main still permits the variants the request wants to forbid.

## Solution Assessment

No, not as an automated cleanup change. The maintainable path is a maintainer policy decision first, followed by one consistent validation/docs/migration update if uppercase-only becomes the contract.
