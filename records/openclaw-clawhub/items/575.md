## Summary

Keep this issue open: current main still documents and implements lowercase and plural skill readme compatibility across docs, CLI discovery/publish, browser publish, GitHub import, server publish validation, and readme lookup. Uppercase-only enforcement would be a breaking skill-format policy change, and this repository's cleanup policy keeps issues open unless maintainers resolve them.

## What This Changes

Request to enforce canonical uppercase `SKILL.md` and `PROTOCOL.md` names and reject lowercase or case-conflicting skill package files.

## Best Possible Solution

Maintainers should first define the canonical casing policy and whether `PROTOCOL.md` is part of the package contract, then update docs, CLI scan/sync/publish, browser upload, GitHub import, server validation, readme resolution, tests, and any content migration plan consistently.

## Reproduction Assessment

Yes for the ClawHub side: current-main source inspection shows lowercase and plural readme variants are still accepted and documented. The exact Windows checkout collision is in the external `openclaw/skills` content repo and was not reproduced in this checkout.

## Solution Assessment

Unclear. Uppercase-only enforcement may prevent future case-insensitive filesystem collisions, but it conflicts with the current public format contract and needs an explicit migration/product decision first.
