## Summary

Keep this issue open. Current main still documents and implements lowercase and plural skill readme compatibility, while uppercase-only `SKILL.md` plus the requested `PROTOCOL.md` rule would be a breaking format-policy and migration decision for maintainers.

## What This Changes

Request ClawHub enforce uppercase-only `SKILL.md` and `PROTOCOL.md` to avoid duplicate-case content and case-insensitive filesystem conflicts.

## Best Possible Solution

Maintainers should define the canonical casing policy and `PROTOCOL.md` contract, then update docs, CLI, web upload, GitHub import, server validation, readme resolution, and content migration consistently.

## Reproduction Assessment

Yes for the current ClawHub behavior: source and docs show `skill.md` and `skills.md` are still accepted across the main publish/import/read paths. I did not reproduce the Windows checkout conflict because it belongs to the separate `openclaw/skills` content repository.

## Solution Assessment

Unclear. Uppercase-only enforcement may prevent future case-insensitive filesystem conflicts, but it is a breaking public format change and should follow an explicit maintainer migration decision.
