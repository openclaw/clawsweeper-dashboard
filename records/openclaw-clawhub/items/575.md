## Summary

Keep this open because current main still intentionally accepts lowercase and legacy skill readme aliases across several publish/import paths, and only one GitHub-backed sync path has a narrower duplicate-slug guard. The remaining `PROTOCOL.md` part is still not a defined ClawHub skill package contract, so this needs a maintainer policy decision before implementation.

## What This Changes

Request to enforce a canonical uppercase skill/protocol filename policy and reject case-colliding variants that break case-insensitive filesystems.

## Best Possible Solution

Define a compatibility-preserving filename policy, decide whether `PROTOCOL.md` belongs in the skill package contract, then update docs and all publish/import/upload validators consistently with any content cleanup plan.
