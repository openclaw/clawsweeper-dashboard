## Summary

Keep this issue open: current main still documents and implements lowercase/plural skill readme compatibility, and uppercase-only enforcement would be a breaking public skill-format policy decision. ClawHub cleanup policy for this repository also keeps issues open unless maintainers resolve them.

## What This Changes

The issue requests enforcing uppercase-only skill package files, especially `SKILL.md` and `PROTOCOL.md`, to avoid duplicate casing and case-insensitive filesystem conflicts.

## Best Possible Solution

Maintainers should define the canonical casing policy and whether `PROTOCOL.md` belongs in the skill package contract, then update docs, CLI scan/sync/publish, browser upload, GitHub import, server validation, readme resolution, tests, and any content migration plan consistently.

## Reproduction Assessment

Yes for the current ClawHub behavior: source inspection shows lowercase and plural readme variants are still accepted across docs and validation paths. The Windows checkout collision itself is in the external skills content repository and was not reproduced here.

## Solution Assessment

Unclear. Uppercase-only enforcement may prevent case-insensitive filesystem collisions, but it conflicts with the current public format contract and needs an explicit maintainer migration decision.
