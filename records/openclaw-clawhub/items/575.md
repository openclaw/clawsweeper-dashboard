## Summary

Keep this issue open. Current main still documents and implements lowercase/plural skill readme compatibility, while the requested uppercase-only `SKILL.md` / `PROTOCOL.md` policy is a breaking format and migration decision for maintainers.

## What This Changes

Request to require uppercase `SKILL.md` and `PROTOCOL.md` casing and reject lowercase or conflicting case variants.

## Best Possible Solution

Maintainers should define the canonical casing policy and whether `PROTOCOL.md` belongs in the package contract, then update docs and validation consistently across CLI, web upload, GitHub import, server publish, readme resolution, and any content cleanup plan.

## Reproduction Assessment

Yes for the current ClawHub behavior: source inspection shows lowercase and plural readme variants are still accepted in docs and validation paths. The Windows checkout conflict itself is external to this checkout and was not reproduced here.

## Solution Assessment

Unclear. Uppercase-only enforcement may address case-insensitive filesystem conflicts, but applying it now would be a breaking compatibility change and should follow an explicit maintainer migration decision.
