## Summary

Keep this issue open. Current main still documents and implements lowercase and plural skill readme compatibility, while the requested uppercase-only rule would be a breaking package-format policy decision that also adds an undefined `PROTOCOL.md` contract.

## What This Changes

Require skill packages to use uppercase `SKILL.md` and `PROTOCOL.md`, rejecting lowercase or conflicting case variants.

## Best Possible Solution

Define the canonical casing policy and `PROTOCOL.md` contract first, then update docs, specs, validators, import/upload/publish paths, and any external content migration together.

## Reproduction Assessment

Not applicable as a strict bug reproduction: current ClawHub docs and validators intentionally allow lowercase and plural skill readme variants. Source inspection gives high confidence that uppercase-only enforcement is absent on current main.

## Solution Assessment

No for an automated narrow fix: enforcing uppercase-only would change the public package contract and add an undefined `PROTOCOL.md` rule. The maintainable path is a maintainer policy decision followed by coordinated validation, docs, tests, and content cleanup.
