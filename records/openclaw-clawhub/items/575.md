## Summary

Keep this issue open: current main still documents and implements lowercase/plural skill readme compatibility, and uppercase-only enforcement would be a breaking package-format policy decision rather than conservative cleanup.

## What This Changes

The issue requests enforcing canonical uppercase `SKILL.md` and `PROTOCOL.md` filenames to prevent duplicate case variants and Windows checkout collisions.

## Best Possible Solution

Maintainers should define the canonical casing policy and whether `PROTOCOL.md` belongs in the package contract, then update docs, CLI scan/sync/publish, browser upload, GitHub import, server validation, readme resolution, and content migration consistently.

## Reproduction Assessment

Yes for the current ClawHub behavior: source inspection shows lowercase and plural readme variants are still accepted. The Windows checkout collision itself is in the external skills content repo and was not reproduced here.

## Solution Assessment

Unclear. Uppercase-only enforcement may prevent case-insensitive filesystem collisions, but it conflicts with the current public format contract and needs an explicit maintainer migration decision.
