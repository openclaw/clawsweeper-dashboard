## Summary

Keep this issue open. Current main still intentionally accepts lowercase and plural skill readme variants in docs, CLI scan/publish, upload/import, and server validation, so uppercase-only enforcement would be a breaking package-format policy decision rather than a cleanup close candidate.

## What This Changes

Request to enforce uppercase `SKILL.md` and `PROTOCOL.md` casing and reject mixed-case duplicates that break case-insensitive checkouts.

## Best Possible Solution

Maintainers should first define the canonical package casing policy and whether `PROTOCOL.md` belongs in the contract, then update docs and all scan/import/upload/publish validators together with a content migration plan.

## Reproduction Assessment

Not applicable as a strict bug reproduction because current ClawHub docs intentionally allow lowercase variants. Source inspection gives high confidence that uppercase-only enforcement is absent on current main.

## Solution Assessment

No for an automated narrow fix: enforcing uppercase-only would be a breaking format-policy change and `PROTOCOL.md` is not yet defined in the ClawHub package contract. The maintainable path is a maintainer policy decision followed by coordinated validation, docs, tests, and content cleanup.
