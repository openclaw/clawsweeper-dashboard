## Summary

Keep this issue open. Current main still documents and implements lowercase and plural skill readme compatibility across docs, CLI discovery/publish, browser upload, GitHub import, and server publish validation, so uppercase-only enforcement remains an unresolved breaking format-policy decision.

## What This Changes

Request to enforce uppercase-only `SKILL.md` and `PROTOCOL.md` naming so skill folders cannot contain case-conflicting lowercase variants.

## Best Possible Solution

Maintainers should define the canonical casing policy and whether `PROTOCOL.md` belongs in the package contract, then update docs, CLI scan/sync/publish, browser upload, GitHub import, server publish, readme resolution, and any content cleanup plan consistently.

## Reproduction Assessment

Not applicable as a strict bug reproduction because current ClawHub docs intentionally allow lowercase variants. Source inspection gives high confidence that uppercase-only enforcement is absent on current main.

## Solution Assessment

No for an automated narrow fix: enforcing uppercase-only would be a breaking package-format policy change and the `PROTOCOL.md` contract is undefined. The maintainable path is a maintainer policy decision followed by coordinated validation, docs, tests, and content migration work.
