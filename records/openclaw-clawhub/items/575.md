## Summary

Keep this open: current main still documents and implements lowercase/plural skill readme compatibility, and `PROTOCOL.md` is not a defined ClawHub skill package contract. For this repository, issues are not cleanup-close candidates; the remaining work needs an explicit breaking format-policy and migration decision.

## What This Changes

Require skill packages to use uppercase `SKILL.md` and `PROTOCOL.md` only, rejecting same-folder case variants that break case-insensitive checkouts.

## Best Possible Solution

Maintainers should decide the canonical casing policy and `PROTOCOL.md` contract first, then update docs, CLI/UI/import/server validation, readme resolution, and any existing content migration consistently.

## Solution Assessment

Unclear. Uppercase-only enforcement may prevent case-insensitive filesystem collisions, but it conflicts with the current public format contract and should follow an explicit migration/product decision.
