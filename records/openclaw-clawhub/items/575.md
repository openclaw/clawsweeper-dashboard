## Summary

Keep this issue open: current main still intentionally documents and implements lowercase/plural skill readme compatibility, while `PROTOCOL.md` is not a defined ClawHub package contract in the inspected docs or validators. ## What This Changes

Request enforcement of uppercase `SKILL.md` and `PROTOCOL.md` package files to avoid duplicate-case files and case-insensitive filesystem checkout conflicts.

## Best Possible Solution

Maintainers should first define the canonical casing and `PROTOCOL.md` policy, then update docs, CLI scan/sync/publish, browser upload, GitHub import, server validation, readme resolution, tests, and any content migration plan consistently.
