## Summary

Keep this open: current main and v0.23.0 still document and implement lowercase/legacy skill readme compatibility, while `PROTOCOL.md` is not defined as a ClawHub skill package contract.

## What This Changes

The issue asks ClawHub to enforce canonical uppercase `SKILL.md` and `PROTOCOL.md` casing, or at least reject case-colliding skill files that break case-insensitive filesystems.

## Best Possible Solution

Maintainers should define the canonical case-collision policy first, then update docs and validation consistently across CLI scan/sync/publish, browser upload, GitHub import/sync, server publish, and any external content cleanup plan.

## Reproduction Assessment

Yes, at source level. Current main still accepts `skill.md` / `skills.md` aliases and has no observed duplicate case-collision rejection, though I did not recreate the Windows checkout against the external skills repository.

## Solution Assessment

Unclear until maintainers choose the policy. The reporter's compatibility-preserving duplicate-case rejection is narrower than uppercase-only enforcement, but `PROTOCOL.md` still needs a ClawHub scope decision.
