## Summary

Keep this issue open. Current main still documents and implements lowercase/plural skill readme compatibility, and changing to uppercase-only is a breaking package-format and migration decision that maintainers need to make explicitly.

## What This Changes

The issue requests uppercase-only enforcement for `SKILL.md` and `PROTOCOL.md` to avoid duplicate-case skill files and Windows checkout conflicts.

## Best Possible Solution

Maintainers should define the canonical casing policy and whether `PROTOCOL.md` belongs in the package contract, then update docs and validation consistently across CLI scan/sync/publish, browser upload, GitHub import, server publish, readme resolution, and any content cleanup plan.

## Reproduction Assessment

Partially. Source inspection gives a high-confidence path for the current ClawHub behavior because lowercase and plural readme variants are still accepted; the Windows checkout conflict itself is in the external `openclaw/skills` content repo and was not reproduced in this checkout.

## Solution Assessment

Unclear. Uppercase-only enforcement may prevent future case-insensitive filesystem collisions, but it is a breaking public format-policy change and should follow an explicit maintainer migration decision.
