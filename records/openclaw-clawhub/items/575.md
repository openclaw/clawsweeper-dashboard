## Summary

This issue should stay open because current main still intentionally accepts lowercase and plural skill readme names, while enforcing uppercase-only would be a breaking skill package-format policy and migration decision. The request is valid product cleanup, but it is not implemented on main and the repository cleanup policy does not auto-close issues.

## What This Changes

Request enforcement of uppercase-only `SKILL.md` and `PROTOCOL.md` names to prevent case-colliding skill package files, especially on case-insensitive filesystems.

## Best Possible Solution

Maintainers should define the canonical casing policy, decide whether `PROTOCOL.md` belongs in the package contract, then update docs and every import/publish/readme validation path consistently with a migration plan.

## Reproduction Assessment

Yes for the ClawHub side: source inspection shows current main still accepts lowercase/plural skill readme names in docs, CLI scan/publish, GitHub import, and server publish validation. I did not reproduce the separate Windows checkout failure in `openclaw/skills` because this repository's source already proves uppercase-only enforcement is absent.

## Solution Assessment

Unclear until maintainers decide the package-format policy. Uppercase-only enforcement could prevent future case collisions, but making it current behavior would be a breaking compatibility change that needs docs, validation, and content migration decisions.
