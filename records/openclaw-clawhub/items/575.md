Keep this issue open. Current main still permits lowercase and plural skill readme variants and has no case-insensitive duplicate-path rejection rule, while the reporter's latest suggestion changes the request into a package-format policy decision that maintainers should settle before implementation.

## What This Changes

Request enforcement against mixed-case duplicate skill package files such as `SKILL.md`/`skill.md` and `PROTOCOL.md`/`protocol.md`, while possibly preserving lowercase readme compatibility.

## Best Possible Solution

Define a compatibility-preserving case-insensitive duplicate-path policy, decide whether `PROTOCOL.md` belongs in the package contract, then enforce it consistently across CLI scan/sync/publish, browser upload, GitHub import, server publish, docs, and content cleanup.

## Solution Assessment
