## Summary

Keep this issue open. Current main still documents and implements lowercase/plural skill readme compatibility, while `PROTOCOL.md` is not a current ClawHub package contract; changing this is a breaking format and migration policy decision rather than a cleanup close.

## What This Changes

Request to enforce uppercase-only `SKILL.md` and `PROTOCOL.md` package files and prevent case-conflicting skill submissions.

## Best Possible Solution

Maintainers should define the canonical casing policy, decide whether `PROTOCOL.md` belongs in the package contract, then update docs, validation, import/publish/read paths, tests, and any content cleanup plan consistently.

## Reproduction Assessment

Yes for the ClawHub behavior: source inspection shows current main still accepts lowercase and plural readme variants in docs, CLI, browser upload, GitHub import, server publish, and readme resolution. The external Windows checkout failure in `openclaw/skills` was not reproduced locally.

## Solution Assessment

Unclear until maintainers decide the format policy. Uppercase-only enforcement may prevent future case-insensitive checkout conflicts, but it conflicts with current compatibility behavior and needs a migration plan rather than a narrow bug fix.
