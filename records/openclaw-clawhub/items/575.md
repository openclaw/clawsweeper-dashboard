## Summary

Keep this issue open. Current main still documents and implements lowercase and plural skill readme compatibility, and the requested uppercase-only rule is a breaking package-format policy decision that ClawHub maintainers need to make explicitly.

## What This Changes

Request to enforce uppercase-only `SKILL.md` and `PROTOCOL.md` package files and reject conflicting lowercase variants.

## Best Possible Solution

Maintainers should define the canonical casing policy and `PROTOCOL.md` contract, then update docs and validation consistently across CLI scan/sync/publish, browser upload, GitHub import, server publish, readme resolution, and any content migration plan.

## Reproduction Assessment

Yes for the current ClawHub behavior: source inspection shows lowercase and plural readme variants are still accepted in docs and validation paths. The Windows checkout collision itself is external to this repo and was not reproduced here.

## Solution Assessment

Unclear. Uppercase-only enforcement may prevent case-insensitive filesystem conflicts, but it conflicts with the current public format contract and should follow an explicit maintainer migration decision.
