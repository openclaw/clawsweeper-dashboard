## Summary

Keep this issue open. Current main still documents and implements lowercase/plural skill readme compatibility across docs, CLI discovery/publish, browser publish, GitHub import, server validation, and readme resolution; uppercase-only enforcement plus a `PROTOCOL.md` rule is an unresolved package-format policy and migration decision.

## What This Changes

Request to reject lowercase or duplicate skill package manifest files and require uppercase `SKILL.md` and `PROTOCOL.md` casing.

## Best Possible Solution

Maintainers should define the canonical casing policy, decide whether `PROTOCOL.md` belongs in the package contract, migrate or clean affected published content, then update docs and validation consistently across all publish/import/read paths.

## Reproduction Assessment

Not applicable as a current ClawHub bug reproduction. Source inspection shows current main intentionally accepts lowercase/plural readme variants, while the reported Windows checkout failure is in the separate `openclaw/skills` content repository.

## Solution Assessment

Unclear. Immediate uppercase-only rejection is not clearly the best fix until maintainers decide the compatibility and migration policy; after that, enforcement should be centralized and applied across CLI, web, GitHub import, server publish, and readme resolution.
