## What This Changes

## Best Possible Solution

Adopt a compatibility-preserving case-insensitive collision policy for accepted skill files, decide whether `PROTOCOL.md` belongs in the package contract, then update docs, validation, tests, and content-cleanup guidance together.

Yes, by source inspection for the ClawHub validation gap: current main accepts lowercase/plural readme variants and has no case-insensitive duplicate-path rejection in the inspected CLI, browser upload, GitHub import, and server publish paths. ## Solution Assessment

No for strict uppercase-only enforcement because current docs and code deliberately allow lowercase and plural readme variants. The better path is the reporter's narrowed collision rule after maintainers decide accepted variants, `PROTOCOL.md` scope, and cleanup behavior.
