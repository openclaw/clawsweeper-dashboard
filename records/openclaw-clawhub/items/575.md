## Summary

Keep this issue open. Current main still deliberately supports lowercase and plural skill manifest names, and only GitHub source sync has a partial duplicate-slug guard; ClawHub does not consistently reject case-colliding `SKILL.md`/`skill.md` bundles, and `PROTOCOL.md` is not defined in the package contract.

## What This Changes

The issue requests a canonical casing policy for `SKILL.md` and `PROTOCOL.md`, or at least rejection of bundles that contain case-colliding uppercase and lowercase variants.

## Best Possible Solution

Define the canonical casing policy first, then consistently reject case-insensitive duplicate relative paths across browser upload, CLI scan/publish/sync, GitHub import, server publish, docs, and tests while handling existing corpus cleanup separately.

## Reproduction Assessment

Yes at source level. Current main lower-cases accepted manifest names in the publish/import paths without a global case-collision rejection, though I did not run a Windows checkout reproduction.

## Solution Assessment

Unclear until maintainers choose the policy. The reporter's narrowed approach, keeping lowercase compatibility while rejecting simultaneous case variants, looks safer than uppercase-only enforcement, but `PROTOCOL.md` still needs scope definition.
