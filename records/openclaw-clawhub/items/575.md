## Summary

Keep this issue open. Current main still documents and implements lowercase/plural skill readme compatibility across docs, CLI scan/publish, browser upload, GitHub import, server publish validation, and stored readme resolution; uppercase-only enforcement is not implemented. This is also a ClawHub issue, and the repository cleanup policy says issues should remain open unless maintainers resolve them.

## What This Changes

The issue requests enforcing uppercase-only `SKILL.md` and `PROTOCOL.md` names to avoid duplicate mixed-case files and Windows case-insensitive checkout problems.

## Best Possible Solution

Maintainers should define the canonical casing policy for skill package files, decide whether `PROTOCOL.md` belongs in the package contract, then update docs and validation consistently across CLI scan/sync/publish, browser upload, GitHub import, server publish, stored readme resolution, and any content cleanup plan.
