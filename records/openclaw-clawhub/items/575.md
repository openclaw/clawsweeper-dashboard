## Label Justifications

- `P2`: This is a confirmed cross-platform portability bug with meaningful but package-specific blast radius.
- `impact:other`: Case-colliding files can make package checkout, extraction, and manifest selection ambiguous or unusable on case-insensitive filesystems.

## Summary

Current main still permits case-folded duplicate package paths, and the documented lowercase manifest compatibility makes uppercase-only enforcement a product decision rather than an already-implemented cleanup.

## What This Changes

Require portable, unambiguous `SKILL.md` and `PROTOCOL.md` package paths by addressing uppercase/lowercase filename collisions.

## Best Possible Solution

Define case-folded path uniqueness as a server-enforced package invariant, mirror it in clients for early errors, preserve an explicit legacy manifest alias policy, and clean existing collisions with a deterministic keep-one-file rule.

## Reproduction Assessment

Yes. Source inspection shows that a case-sensitive tree can submit both filename variants and backend validation does not reject the folded collision; the reporter also supplied a concrete Windows checkout transcript.

## Solution Assessment

No, unconditional uppercase-only enforcement is not the best current solution because it conflicts with established lowercase compatibility. Duplicate case-fold rejection with an explicit alias and cleanup policy is narrower and safer.
