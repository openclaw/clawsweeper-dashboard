## Summary

Keep this issue open. Current main still documents and implements lowercase/plural skill readme compatibility, and uppercase-only enforcement would be a breaking skill package-format policy decision rather than a conservative cleanup close.

## What This Changes

Request enforcement of uppercase `SKILL.md` and `PROTOCOL.md`, including rejecting lowercase duplicates that collide on case-insensitive filesystems.

## Best Possible Solution

Maintainers should define the canonical casing policy and whether `PROTOCOL.md` belongs in the package contract, then update docs, CLI scan/sync/publish, browser upload, GitHub import, server validation, readme resolution, and any content migration plan consistently.

## Reproduction Assessment

Yes for the current ClawHub behavior: source inspection shows lowercase and plural readme variants are still accepted in docs and validation paths. The Windows checkout collision itself is external to this repo and was not reproduced here.

## Solution Assessment

Unclear. Uppercase-only enforcement may prevent case-insensitive filesystem collisions, but it conflicts with the current public format contract and should follow an explicit maintainer migration decision.
