## Label Justifications

- `P2`: The issue is a bounded but meaningful cross-platform package reliability and compatibility improvement.
- `impact:other`: Case-colliding files can make skill repositories ambiguous and leave Windows checkouts unusable or perpetually dirty.

## Summary

Current main and v0.20.0 still intentionally accept case-insensitive `SKILL.md` aliases without shared case-folded duplicate-path rejection, so the cross-platform ambiguity remains; maintainers must decide the alias policy and whether `PROTOCOL.md` belongs in ClawHub validation.

## What This Changes

Require skill ingestion to prevent case-colliding marker files and define canonical casing for `SKILL.md` and `PROTOCOL.md`.

## Best Possible Solution

Define the accepted marker aliases, then reject case-folded duplicate paths through shared validation across every ingestion path and handle `PROTOCOL.md` only if maintainers add it to the package contract.

## Reproduction Assessment

Yes. Current source case-folds marker names for recognition while preserving original paths, and the reporter provides a concrete Windows checkout transcript showing the filesystem collision.

## Solution Assessment

No, unconditional uppercase-only enforcement is not yet the best solution because current specs deliberately support aliases; shared collision rejection with an explicit compatibility decision is safer.
