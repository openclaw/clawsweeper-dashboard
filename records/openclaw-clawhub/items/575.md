## Summary

Keep this issue open. Current main still documents and implements lowercase/plural skill readme compatibility, and `PROTOCOL.md` is not a defined ClawHub package contract, so uppercase-only enforcement remains a maintainer policy and migration decision rather than conservative cleanup.

## What This Changes

The issue requests that ClawHub enforce uppercase `SKILL.md` and `PROTOCOL.md` names to avoid duplicate-case packages and case-insensitive filesystem checkout failures.

## Best Possible Solution

Maintainers should define the canonical casing policy and decide whether the Windows problem is best solved by case-insensitive duplicate-path rejection, uppercase-only canonical names, or both, then update docs/specs plus CLI, browser, GitHub import, server validation, and migration guidance together.

## Reproduction Assessment

No runtime reproduction was run in this read-only review, but the enforcement gap is source-reproducible: current main accepts lowercase/plural skill readme variants and has no `PROTOCOL.md` validation contract. The Windows checkout failure itself is evidenced by the issue body for the separate openclaw/skills content repo.

## Solution Assessment

No for an automated uppercase-only fix: that would change the documented package contract and add an undefined `PROTOCOL.md` rule. A maintainable fix needs maintainer policy direction, and a narrower duplicate-case rejection may be safer than immediately banning lowercase-only packages.
