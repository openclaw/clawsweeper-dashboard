The issue asks ClawHub to enforce canonical casing for `SKILL.md` and `PROTOCOL.md` so mixed-case duplicate files do not break case-insensitive filesystems.

## Best Possible Solution

Maintainers should choose the canonical casing and duplicate-case policy first, decide whether `PROTOCOL.md` belongs in the package contract, then update docs and validation consistently across CLI, browser upload, GitHub import, API, and server publish paths.

## Reproduction Assessment
