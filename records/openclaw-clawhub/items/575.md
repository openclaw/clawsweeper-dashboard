## What This Changes

Request enforcement of canonical skill/protocol filename casing, or at least rejection of packages containing case-insensitive duplicate `SKILL.md`/`skill.md` and `PROTOCOL.md`/`protocol.md` files.

## Best Possible Solution

Adopt a compatibility-preserving case-insensitive collision policy for accepted skill readme variants, decide whether `PROTOCOL.md` belongs in the skill format, then update docs, validation, import/publish/read paths, tests, and content cleanup guidance consistently.

## Solution Assessment
