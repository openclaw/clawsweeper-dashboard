## Summary

Keep this open. Current main still accepts lowercase and plural skill readme variants across docs, CLI, browser upload, GitHub import, and server publish, while `PROTOCOL.md` is not a defined ClawHub skill-package contract.

## What This Changes

Request to enforce a single canonical casing policy for skill package files such as `SKILL.md`/`skill.md` and `PROTOCOL.md`/`protocol.md` to avoid case-insensitive filesystem collisions.

## Best Possible Solution

Define the casing policy first, then consistently reject case-insensitive duplicate relative paths across publish, sync, GitHub import, browser upload, server validation, docs, and tests while preserving any chosen compatibility aliases.

## Reproduction Assessment

Yes, at source level. Current main lowercases and accepts skill readme aliases in multiple publish/import paths without a global case-collision rejection, though I did not run a Windows checkout reproduction.

## Solution Assessment

Unclear until maintainers choose the policy. The reporter's narrowed approach, keeping lowercase compatibility while rejecting simultaneous case variants, looks safer than uppercase-only enforcement, but `PROTOCOL.md` still needs scope definition.
