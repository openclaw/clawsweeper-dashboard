## Summary

Keep this issue open. Current main still documents and implements lowercase/plural skill readme compatibility across the relevant ClawHub paths, and changing that to uppercase-only is a breaking package-format policy and migration decision.

## What This Changes

The issue requests uppercase-only enforcement for `SKILL.md` and `PROTOCOL.md` to prevent duplicate-case package files and case-insensitive filesystem checkout problems.

## Best Possible Solution

Maintainers should define the canonical casing policy, decide whether `PROTOCOL.md` belongs in the package contract, then update docs, CLI scan/sync/publish, browser upload, GitHub import, server publish validation, readme resolution, and any external content cleanup plan consistently.

## Reproduction Assessment

Yes for the ClawHub side: source inspection shows current main still accepts lowercase/plural skill readme names in CLI, web upload, GitHub import, server publish, and readme resolution. I did not reproduce the separate Windows checkout of `openclaw/skills` because the current source paths are sufficient to verify non-enforcement here.

## Solution Assessment

Unclear until maintainers decide the package-format policy. Uppercase-only may prevent future case collisions, but enforcing it now would break documented compatibility and needs a migration/content cleanup plan.
