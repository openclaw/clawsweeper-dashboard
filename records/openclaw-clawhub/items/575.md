## Summary

Keep this issue open. Current main still deliberately accepts lowercase and plural skill readme variants, and the latest human comment narrows the request into a compatibility-preserving case-collision policy that still needs maintainer judgment.

## What This Changes

The issue asks ClawHub to enforce canonical casing for skill package files, now narrowed to accepting lowercase compatibility while rejecting packages that contain case-insensitive duplicates such as `SKILL.md` and `skill.md`.

## Best Possible Solution

Define a compatibility-preserving case-collision policy, decide whether `PROTOCOL.md` belongs in the package contract, then enforce it consistently across CLI scan/sync/publish, browser upload, GitHub import, server publish, stored readme resolution, and content cleanup.
