## Summary

Keep this open: current main still documents and implements lowercase/plural skill readme compatibility, and the requested uppercase-only rule would be a breaking skill package-format policy decision rather than conservative cleanup.

## What This Changes

The issue asks ClawHub to enforce uppercase `SKILL.md` and `PROTOCOL.md` casing and reject duplicate lowercase/uppercase variants.

## Best Possible Solution

Maintainers should define the canonical casing policy and whether `PROTOCOL.md` belongs in the package contract, then update docs, CLI/UI/import/server validation, readme resolution, and any content cleanup plan consistently.

## Reproduction Assessment

Yes for the ClawHub source side: current main clearly accepts and documents lowercase/plural readme variants. The exact Windows checkout collision is in the separate `openclaw/skills` content repository and was not reproduced in this checkout.

## Solution Assessment

Unclear. Uppercase-only enforcement may prevent case-insensitive filesystem collisions, but it conflicts with the current public format contract and needs an explicit migration/product decision first.
