## Summary

Keep this issue open. Current main still documents and implements lowercase and plural skill readme compatibility, and the ClawHub cleanup policy says issues should remain open unless maintainers resolve them; enforcing uppercase-only is a breaking package-format decision rather than conservative cleanup.

## What This Changes

The issue requests rejecting lowercase/case-conflicting `skill.md` and `protocol.md` variants so skill packages use only uppercase `SKILL.md` and `PROTOCOL.md`.

## Best Possible Solution

Maintainers should define the canonical casing policy and whether `PROTOCOL.md` belongs in the package contract, then update docs and validation consistently across CLI scan/sync/publish, browser upload, GitHub import, server publish, stored readme resolution, and content cleanup planning.

## Reproduction Assessment

Yes for the current ClawHub behavior: source inspection shows lowercase and plural readme variants are still accepted in docs and validation paths. The Windows checkout conflict itself is external to this checkout and was not reproduced here.

## Solution Assessment

Unclear. Uppercase-only enforcement may address case-insensitive filesystem conflicts, but applying it now would remove documented compatibility and should follow an explicit maintainer migration decision.
