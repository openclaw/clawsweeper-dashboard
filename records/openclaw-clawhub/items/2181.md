## Summary

Keep open for maintainer/operator follow-up: current main intentionally treats deleted accounts as deactivated/purged and blocks re-registration, and ClawHub cleanup policy does not auto-close open issues.

## What This Changes

The issue asks maintainers to manually clear a GitHub-linked purged/deactivated ClawHub account so the reporter can register again.

## Best Possible Solution

Define one supported deleted-account lifecycle: either keep deletion terminal with clear support messaging, or add an audited verified restore/unlink/recreate flow that preserves OAuth identity, deletion privacy, bans, tokens, and audit safeguards.

## Reproduction Assessment

Yes, at source level. deleteAccount sets deactivatedAt/purgedAt, handleDeletedUserSignIn rejects deactivated users during OAuth, and unbanUserWithActor refuses deactivated accounts; I did not verify the reporter's production row.

## Solution Assessment

No for a direct unban or flag clear from public issue data. The maintainable solution needs private identity verification plus maintainer security/product approval for either an audited recovery/unlink path or an explicit terminal-deletion policy.
