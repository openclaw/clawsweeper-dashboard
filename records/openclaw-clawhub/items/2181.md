## Summary

Keep open for maintainer/operator handling: current main intentionally treats user deletion as irreversible, and clearing a purged/deactivated GitHub account requires production data plus security/product judgment.

## What This Changes

The issue requests manual clearing of a purged/deactivated ClawHub GitHub account so the reporter can register or sign in again.

## Best Possible Solution

Maintainers should define one supported deleted-account lifecycle, then either keep deletion terminal with clear support messaging or add an audited verified recovery, unlink, or recreate flow that preserves identity, deletion, ban, token, and audit safeguards.

## Reproduction Assessment

Yes, at source level. A user who runs deleteAccount gets deactivatedAt/purgedAt, later OAuth sign-in is rejected by handleDeletedUserSignIn, and the normal unban path rejects deactivated accounts; I did not verify the reporter's production row.

## Solution Assessment

No for a direct manual clear or ordinary unban from a public issue. The maintainable path needs verified identity, production access, and a maintainer-approved account-recovery or terminal-deletion policy before changing purged/deactivated account state.
