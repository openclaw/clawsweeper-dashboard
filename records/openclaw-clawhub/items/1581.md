## Summary

Keep open. The ClawHub cleanup policy keeps issues open, live production GitHub sign-in still returns HTTP 500 from both the canonical and direct Convex endpoints, and current main routes web auth through env-driven Convex Auth rather than a source-hardcoded localhost callback.

## What This Changes

The issue reports that GitHub OAuth for clawhub.ai redirects to a localhost callback instead of the production Convex callback, blocking web login.

## Best Possible Solution

Restore production GitHub OAuth credential, callback, issuer, site URL, and cookie/session alignment; verify browser and CLI login end to end; then consolidate overlapping OAuth reports that share the same root cause.

## Reproduction Assessment

Yes for the central production auth failure: both the canonical clawhub.ai and direct Convex GitHub sign-in endpoints return HTTP 500 while the public API is healthy. The exact localhost callback stage is not currently reachable because sign-in fails before GitHub authorization.

## Solution Assessment

Yes for keeping this open and routing it to maintainer/operator auth triage. A source-only cleanup PR is not the best first fix because current main does not hardcode a web localhost callback and the likely repair requires production OAuth, Convex, and Vercel access.
