## Summary

Keep open. Production GitHub sign-in still returns HTTP 500, current main uses environment-backed Convex Auth rather than a hardcoded localhost web callback, and ClawHub cleanup policy keeps open issues in maintainer hands.

## What This Changes

The issue reports that GitHub OAuth on clawhub.ai redirects to a 127.0.0.1 callback instead of the production Convex callback after authorization.

## Best Possible Solution

Restore production GitHub OAuth configuration so clawhub.ai redirects into GitHub and completes on the production Convex callback, then verify no localhost callback appears and add a production smoke check for the flow.

## Reproduction Assessment

Partially. The current production outage is reproducible because both the canonical and direct Convex sign-in endpoints return HTTP 500, but the exact 127.0.0.1 post-authorization callback cannot be rechecked until the sign-in endpoint redirects again.

## Solution Assessment

Yes for maintainer/operator repair, not for a source-only callback edit. Current main routes web auth through Convex and environment variables, so the narrowest maintainable path is production OAuth/env repair plus an import-safe diagnostic follow-up if needed.
