## Summary

Keep open. Current main does not show a hardcoded localhost web OAuth callback, but production GitHub sign-in is still failing before it can redirect, and ClawHub cleanup policy keeps issues open unless a PR is certainly implemented on main.

## What This Changes

The issue asks for production GitHub OAuth to redirect through the Convex production callback instead of a local 127.0.0.1 callback.

## Best Possible Solution

Restore production GitHub OAuth configuration so sign-in starts with a GitHub redirect and completes on the production Convex callback, then add a production smoke guard for HTTP 500 starts and localhost callback regressions.

## Reproduction Assessment

Yes for the user-visible auth failure: both production GitHub sign-in endpoints return HTTP 500 instead of a GitHub redirect. The exact 127.0.0.1 callback still needs an interactive OAuth completion or maintainer access to the GitHub OAuth App settings.

## Solution Assessment

No source-only repair is clearly the best fix yet. The narrow maintainable path is to repair the production OAuth App and Convex/Vercel auth configuration, then add smoke coverage so this cannot recur silently.
