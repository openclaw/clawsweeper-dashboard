## Summary

Keep open. This is an open ClawHub issue, and current production auth is still failing: current main has env-driven Convex/GitHub auth with no hardcoded web callback to 127.0.0.1, but both production sign-in endpoints return HTTP 500 before a GitHub redirect can be verified.

## What This Changes

The report asks maintainers to fix production GitHub OAuth so authorization callbacks use the Convex production site instead of 127.0.0.1.

## Best Possible Solution

Restore production GitHub OAuth so clawhub.ai starts and completes through the production Convex callback without localhost targets, then add a production smoke check for sign-in HTTP 500s and localhost callback regressions.

## Reproduction Assessment

Partially. The live production auth failure is reproducible with `curl -I` against both the Vercel and Convex sign-in endpoints, but the specific 127.0.0.1 callback redirect cannot be verified while the flow fails earlier with HTTP 500.

## Solution Assessment

Yes for direction: the narrowest maintainable path is to repair the production OAuth app and Convex/Vercel auth environment first, since current source already routes web auth through Convex and does not hardcode a localhost web callback.
