## Summary

Keep open. This is an open ClawHub issue, current production GitHub sign-in still returns HTTP 500, and the current source routes OAuth through environment-backed Convex Auth rather than a hardcoded localhost callback.

Report that production GitHub OAuth redirects to 127.0.0.1 after authorization instead of completing on the production Convex callback URL.

## Best Possible Solution

Restore production GitHub OAuth configuration so clawhub.ai redirects into GitHub and completes on the production Convex callback, then add a smoke check that catches HTTP 500 responses and localhost callback URLs.

## Reproduction Assessment

Partially. The current production auth outage is reproducible because both sign-in endpoints return HTTP 500, but the exact 127.0.0.1 post-authorization callback cannot be rechecked until the sign-in endpoint redirects again.

## Solution Assessment

Yes for maintainer/operator repair, not for a source-only hardcoded callback edit. The narrowest maintainable path is production OAuth/env repair plus an import-safe diagnostic or deploy validation follow-up if needed.
