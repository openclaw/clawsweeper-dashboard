## Summary

Keep open. Production GitHub sign-in is still failing, and source inspection points to environment-driven Convex Auth/GitHub OAuth configuration rather than a localhost callback hardcode that current main already fixed.

## What This Changes

The issue requests that GitHub OAuth on clawhub.ai complete through the production Convex callback instead of redirecting to 127.0.0.1.

## Best Possible Solution

Restore production GitHub OAuth so both clawhub.ai and the Convex site start the GitHub authorization flow with a non-localhost redirect and complete a real login, then keep a production smoke check for the sign-in endpoint and interactive session path.

## Reproduction Assessment

Yes, partially. The current production sign-in endpoint failure is reproducible with GET requests returning HTTP 500 on both clawhub.ai and the Convex site, but the older 127.0.0.1 callback cannot be reached while the flow fails before authorization.

## Solution Assessment

No for a source-only callback edit. Current main already routes auth through Convex and environment variables, so the narrowest maintainable fix is production OAuth/Convex environment repair plus smoke verification unless deployment logs reveal a code regression.
