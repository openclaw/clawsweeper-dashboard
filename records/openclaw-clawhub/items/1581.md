## Summary

Keep open. This is an open ClawHub issue, cleanup policy only auto-closes PRs, and the production GitHub auth path still fails before the reported callback can be proven fixed.

## What This Changes

The issue reports that GitHub OAuth redirects to a localhost callback instead of the production Convex callback, breaking ClawHub login.

## Best Possible Solution

Restore production GitHub OAuth credential, callback, issuer, and site URL alignment, verify web and CLI login end to end, then add a request-time diagnostic or deploy smoke check for auth 500s and localhost callback targets.

## Reproduction Assessment

Partly: the central production auth outage is reproducible because both live GitHub signin endpoints return HTTP 500 while the public API is healthy. The exact 127.0.0.1 post-authorization redirect is not currently reproducible because the flow fails before GitHub authorization.

## Solution Assessment

Yes for routing: keeping this open for maintainer/operator auth triage is the narrowest maintainable path. A source-only fix is not enough until production OAuth app settings, Convex/Vercel env, cookies, and auth logs are verified; any diagnostic should follow the request-time or deploy/env validation direction from https://github.com/openclaw/clawhub/pull/1720.
