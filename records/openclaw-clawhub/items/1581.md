## Summary

Keep open. ClawHub cleanup policy keeps issues open, and the live production GitHub sign-in start still returns HTTP 500 while current source routes auth through Convex without a hardcoded web callback to localhost.

## What This Changes

The issue reports GitHub OAuth from clawhub.ai redirecting to a 127.0.0.1 callback and failing with ERR_CONNECTION_REFUSED instead of completing against the production Convex callback.

## Best Possible Solution

Restore production GitHub OAuth configuration so both production sign-in endpoints redirect to GitHub and complete on the production Convex callback, then add a smoke guard for HTTP 500 starts and localhost callback regressions.

## Reproduction Assessment

Yes for the observable production sign-in failure: both https://clawhub.ai/api/auth/signin/github and https://wry-manatee-359.convex.site/api/auth/signin/github currently return HTTP 500. The exact localhost callback still requires interactive OAuth completion or OAuth App settings access.

## Solution Assessment

No source-only repair is clearly the best fix yet. The narrow maintainable path is production OAuth App plus Convex/Vercel auth-env inspection, followed by repo smoke coverage if a guard needs to be added.
