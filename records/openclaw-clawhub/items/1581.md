## Summary

Keep open. The live GitHub sign-in endpoints still return HTTP 500, current main does not hardcode a localhost web OAuth callback, and ClawHub cleanup policy keeps open issues in maintainer hands rather than auto-closing them.

## What This Changes

The issue asks for GitHub OAuth on clawhub.ai to return through the production Convex callback instead of a 127.0.0.1 callback.

## Best Possible Solution

Restore production GitHub OAuth so clawhub.ai starts GitHub authorization, completes on the production Convex callback without localhost targets, establishes a session, and has a production smoke check for the sign-in endpoint and browser login path.

## Reproduction Assessment

Yes, partially. The current production failure is reproducible because both canonical and direct Convex sign-in endpoints return HTTP 500; the exact 127.0.0.1 callback cannot be rechecked while the flow fails before GitHub authorization.

## Solution Assessment

No for a source-only localhost callback edit. The narrow maintainable next step is maintainer/operator verification of the production GitHub OAuth App plus Convex/Vercel auth environment, followed by a code change only if logs reveal a source regression.
