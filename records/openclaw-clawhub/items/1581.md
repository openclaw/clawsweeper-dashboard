## Summary

Keep open. Current main does not hardcode a localhost web OAuth callback and routes auth through the production Convex site, but the live production GitHub sign-in endpoints still return HTTP 500, so the user-visible auth failure is not resolved and likely needs production configuration access.

## What This Changes

The issue reports production GitHub OAuth sending users to 127.0.0.1 after authorization instead of the production Convex callback.

## Best Possible Solution

Restore production GitHub OAuth so sign-in starts with a GitHub redirect and completes on the production Convex callback, then add a smoke guard for HTTP 500 starts and localhost callback regressions.

## Reproduction Assessment

Yes for the observable production sign-in failure: both production /api/auth/signin/github endpoints returned HTTP 500 on 2026-05-03. The exact localhost callback redirect still requires an interactive OAuth completion or access to the GitHub OAuth App settings.

## Solution Assessment

No source-only repair is clearly the best fix yet. The narrow path is to repair production OAuth App and Convex/Vercel auth configuration, then add repo smoke coverage so this does not recur silently.
