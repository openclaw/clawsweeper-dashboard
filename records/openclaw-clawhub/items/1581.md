Keep open. The current source still routes GitHub sign-in through env-backed Convex Auth, live production sign-in endpoints still return HTTP 500 while the public API is healthy, and the exact localhost callback path is not proven fixed because the flow now fails before GitHub authorization.

## What This Changes

The issue reports production GitHub OAuth redirecting to 127.0.0.1 instead of the production Convex callback after GitHub authorization.

## Best Possible Solution
