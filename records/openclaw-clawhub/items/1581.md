## Summary

Keep open. This is an auth-sensitive production issue, ClawHub cleanup policy does not auto-close issues, and the live GitHub sign-in endpoints still fail before the original localhost-callback redirect can be rechecked.

## What This Changes

The issue reports production GitHub OAuth redirecting to a localhost callback instead of the production Convex callback after GitHub authorization.

## Best Possible Solution

Restore production GitHub OAuth credential, callback, issuer, site URL, and cookie alignment, then verify web login and CLI token creation end to end against clawhub.ai without exposing secrets.

## Reproduction Assessment

Yes for the central production auth failure: current curl checks show both canonical and direct Convex GitHub sign-in endpoints return HTTP 500 while the public skills API is healthy. The later localhost callback path is not replayable right now because sign-in fails before GitHub authorization.

## Solution Assessment

Yes for keeping this open and routing it to maintainer/operator auth triage. A source-only PR is not the best first fix because current main does not hardcode localhost in the inspected web sign-in paths, and the likely repair requires production OAuth, Convex, and Vercel configuration access.
