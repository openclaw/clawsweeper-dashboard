## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing pull requests that current main certainly implements. Current main does not hardcode the reported 127.0.0.1 web OAuth callback, but production GitHub sign-in still returns HTTP 500 on both the Vercel and Convex endpoints, so the user-visible auth failure is not proven fixed.

## What This Changes

The issue reports production GitHub OAuth redirecting to a local 127.0.0.1 callback after GitHub authorization instead of completing against the Convex production callback.

## Best Possible Solution

Restore production GitHub OAuth so clawhub.ai starts GitHub authorization successfully, completes on the production Convex callback without any localhost or 127.0.0.1 target, creates a session, and is covered by production smoke checks for HTTP 500 and localhost callback regressions. After the root cause is confirmed, consolidate the overlapping OAuth reports such as #1347 and #1717.
