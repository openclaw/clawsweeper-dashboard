## What This Changes

The issue reports that GitHub OAuth login from clawhub.ai redirects to a localhost callback instead of the production Convex callback.

## Best Possible Solution

Restore production GitHub OAuth credential, callback, issuer, and site URL alignment, verify web and CLI login end to end, then add an import-safe request-time diagnostic or deploy smoke check for auth 500s and localhost callback targets.

## Reproduction Assessment

Partly: the broader production auth outage is reproducible because both live GitHub sign-in endpoints return HTTP 500 while the public API returns HTTP 200. The exact post-authorization 127.0.0.1 callback is not currently reproducible because sign-in fails before GitHub authorization.
