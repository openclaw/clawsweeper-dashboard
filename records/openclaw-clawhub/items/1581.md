## Summary

Keep open. This is a ClawHub issue, and the live production GitHub sign-in path still fails before OAuth can complete, so the reported auth breakage is not fixed and needs maintainer/operator handling.

## What This Changes

The issue reports production GitHub OAuth redirecting to a localhost callback instead of the Convex production callback, blocking sign-in.

## Best Possible Solution

Restore production GitHub OAuth credential, callback, issuer, and site URL alignment, then add import-safe diagnostics or deploy smoke checks after web and CLI login are verified end to end.

## Reproduction Assessment

Yes for the central production auth failure: both the canonical clawhub.ai and direct Convex GitHub sign-in endpoints currently return HTTP 500 while the public skills API returns HTTP 200. The exact 127.0.0.1 post-authorization redirect is not currently replayable because sign-in fails before GitHub authorization starts.

## Solution Assessment

Yes for the current routing decision: keeping this open for maintainer/operator auth triage is the narrowest maintainable path. The closed related PR shows eager module-load env validation is not the preferred fix; request-time diagnostics or deploy/env validation plus production config repair is safer.
