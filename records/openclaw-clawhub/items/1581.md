## Summary

Keep open. ClawHub cleanup policy keeps issues open, and the production OAuth problem is still observable: current source has no hardcoded localhost web callback, but the live GitHub sign-in endpoints return HTTP 500 instead of starting OAuth.

## What This Changes

The issue reports that production GitHub OAuth redirects to 127.0.0.1/local development callback instead of the production Convex callback after authorization.

## Best Possible Solution

Restore production GitHub OAuth configuration so clawhub.ai starts GitHub authorization and completes on the production Convex callback, then add a production smoke guard for the sign-in start endpoint and localhost callback regressions.

## Reproduction Assessment

Yes for the observable production sign-in failure: both production /api/auth/signin/github endpoints return HTTP 500 instead of a GitHub redirect. The exact localhost callback redirect still needs an interactive OAuth completion or OAuth App settings access.

## Solution Assessment

Partly. Updating the OAuth app callback may be necessary, but current evidence points to the broader production auth configuration surface, so the safest fix is maintainer repair of GitHub OAuth App plus Convex/Vercel env followed by smoke coverage.
