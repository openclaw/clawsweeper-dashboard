## Summary

Keep open. ClawHub cleanup policy keeps issues open, and the live production GitHub sign-in endpoints still return HTTP 500 while current main shows the callback behavior is environment-backed rather than a hardcoded localhost source URL.

## What This Changes

The issue reports GitHub OAuth redirecting to a localhost callback instead of the production Convex callback after authorization.

## Best Possible Solution

Production GitHub OAuth should start with a GitHub redirect and complete on the production Convex callback, with smoke checks catching HTTP 500 starts and localhost callback regressions.

## Reproduction Assessment

Partially: the observable production sign-in failure is reproducible because both production /api/auth/signin/github endpoints returned HTTP 500 on 2026-05-04. The exact localhost callback redirect is still unclear because OAuth cannot currently reach the callback stage without production app/settings access.

## Solution Assessment

No source-only repair is clearly the best fix yet. The narrow maintainable path is to repair the production GitHub OAuth App and Convex/Vercel auth environment, then add production smoke coverage for the failure mode.
