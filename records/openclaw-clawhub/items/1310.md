## Summary

## What This Changes

Define multidimensional quality signals for skill documentation, maintenance, dependency hygiene, claim accuracy, complexity, and alternatives alongside existing security evidence.

## Best Possible Solution

Keep this as the canonical design discussion, define separate explainable dimensions and provenance in a focused spec or RFC, and evaluate external tools such as https://github.com/openclaw/clawhub/issues/2042 only after that contract exists.
