## Summary

Keep open: this is a ClawHub issue and current main still has a narrow publish-time quality/moderation gate, not the requested public multidimensional skill quality-scoring model. The related external quality-scoring tool discussion at https://github.com/openclaw/clawhub/issues/2042 is useful context, but it is also open and does not supersede this broader product/trust-model request.

## What This Changes

The issue proposes public, multidimensional quality signals for ClawHub skills beyond current safety and moderation scans.

## Best Possible Solution

Define a product-owned quality-signal model distinct from security/moderation, reconcile it with https://github.com/openclaw/clawhub/issues/2042, then add approved schema, indexed API/search support, backfill, browse/detail UI, and operator documentation.

## Reproduction Assessment

Not applicable: this is a product/data proposal rather than a runtime bug. Source inspection gives a high-confidence baseline that current main lacks a public multidimensional quality-scoring surface.

## Solution Assessment

No patch is under review, and the proposal is not yet a narrow implementation plan. The maintainable path is for maintainers to decide score dimensions, semantics, provenance, rollout, and separation from security labels before code changes.
