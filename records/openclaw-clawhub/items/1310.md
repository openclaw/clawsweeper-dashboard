## Summary

Keep open: current main has a narrow publish-time quality gate and mature security-audit surfaces, but it does not implement the requested public multidimensional quality signals for documentation, maintenance, dependency hygiene, alternatives, or comparative skill quality.

## What This Changes

The issue requests a public quality-signal/scoring model for ClawHub skills, based on external top-50 analysis and gaps between existing trust layers.

## Best Possible Solution

Define an explicit product/RFC contract for advisory quality dimensions, then implement schema, API, browse/detail UI, freshness, backfill, and policy semantics without conflating them with security or moderation verdicts.

## Reproduction Assessment

Not applicable: this is a product/data proposal for new public quality signals, not a broken existing behavior report. The high-confidence check is source, docs, live issue, related-item, and history review.

## Solution Assessment

No, not yet as a generic aggregate score. The narrower maintainable path is a maintainer-approved model that defines independent dimensions, evidence sources, freshness, rollout, and separation from security and moderation decisions.

## Risks / Open Questions

- A single public quality badge could be mistaken for security approval unless each dimension has clear provenance, freshness, limitations, and enforcement meaning.
- The external top-50 scan and raw dataset were not independently validated from the local checkout, so the methodology remains discussion input.
- Automated claim accuracy, prompt quality, and alternative ranking need a defensible evaluation and appeal/update model before they affect publisher visibility or user trust.
- Broad rollout likely requires schema, API, search, UI, and backfill changes plus policy decisions about whether scores affect ranking, installability, moderation, or only advisory display.
