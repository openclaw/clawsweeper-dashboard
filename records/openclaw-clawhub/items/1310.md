## Summary

Keep open: this is a ClawHub feature/product issue, and current main still has a narrow publish-time content-quality gate plus security/moderation scan surfaces, not the requested public multidimensional skill quality-scoring model. The open related discussion at https://github.com/openclaw/clawhub/issues/2042 is useful context, but it is not a merged or shipped replacement.

## What This Changes

Request public quality signals for ClawHub skills that complement security scans with dimensions like documentation, maintenance, dependency hygiene, claim accuracy, and alternatives.

## Best Possible Solution

Define a product-owned quality-signal model distinct from security/moderation, reconcile it with https://github.com/openclaw/clawhub/issues/2042, then add approved schema, indexed API/search support, backfill, browse/detail UI, and operator documentation.

## Reproduction Assessment

Not applicable: this is a product/data proposal rather than a runtime bug. Source inspection gives a high-confidence baseline that current main lacks the requested public multidimensional quality-scoring surface.

## Solution Assessment

No patch is under review, and the proposal is not yet a narrow implementation plan. The maintainable path is for maintainers to decide score dimensions, semantics, provenance, rollout, and separation from security labels before code changes.
