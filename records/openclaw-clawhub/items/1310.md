## Summary

Current main has robust, separately presented security audits plus objective signals such as update time, versions, popularity, official status, and related skills, but it does not implement the requested multi-dimensional quality scoring. The proposal remains useful, overlaps https://github.com/openclaw/clawhub/issues/2042, and needs maintainers to define taxonomy, provenance, freshness, appeals, and discovery or enforcement effects before implementation.

## What This Changes

Evaluate adding structured quality signals for ClawHub skills alongside existing security and trust indicators.

## Best Possible Solution

Write a focused trust-and-quality signals RFC/spec that inventories existing objective metadata, defines any new dimensions independently, and specifies provenance, freshness, prompt-only treatment, publisher appeals, and whether each signal affects display, ranking, review, or enforcement.
