## Summary

Keep open: current main still has only a narrow publish-time content quality gate and public security/moderation surfaces, not the requested public multidimensional skill quality-scoring model. The related open proposal at https://github.com/openclaw/clawhub/issues/2042 overlaps with this discussion, but it does not supersede this issue's top-50 data and broader quality-signal questions.

## What This Changes

The issue proposes a public, multidimensional quality signal for ClawHub skills based on top-50 skill analysis, covering documentation, claim accuracy, complexity, maintenance, dependencies, alternatives, and prompt-only skill quality.

## Best Possible Solution

Define a product-owned quality-signal model distinct from security/moderation, reconcile it with https://github.com/openclaw/clawhub/issues/2042, then add approved schema, API/search support, backfill/recompute paths, browse/detail UI, and public/operator documentation.
