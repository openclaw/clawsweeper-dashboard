## Summary

Keep open: current main has a private publish-time anti-spam quality assessment and extensive security-audit signals, but it does not implement the requested public multidimensional model for documentation, claim accuracy, maintenance, dependency hygiene, complexity, and alternatives. The related tooling proposal at https://github.com/openclaw/clawhub/issues/2042 and policy RFC at https://github.com/openclaw/clawhub/issues/2321 remain open, so neither provides an implemented canonical replacement.

## What This Changes

Add public, multidimensional skill-quality signals alongside existing security indicators, covering documentation, claim accuracy, complexity, maintenance, dependency hygiene, and alternatives.

## Best Possible Solution

Create one maintainer-owned RFC in `specs/` that defines explainable quality dimensions, evidence provenance, prompt-only coverage, score freshness and appeals, and strict separation from security verdicts and enforcement; then implement the approved contract consistently across schema, API, UI, and representative datasets.
