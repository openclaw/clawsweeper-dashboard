## Summary

Keep open: ClawHub policy does not auto-close issues, and current main still lacks the requested public multidimensional skill quality-scoring model beyond security/moderation signals and a narrow publish-time quality hold.

## What This Changes

The issue proposes adding structured public quality signals for ClawHub skills, including documentation completeness, claim accuracy, maintenance, dependency hygiene, alternatives, and code/prompt quality dimensions.

## Best Possible Solution

Define a product-owned quality-signal model distinct from security/moderation, reconcile it with https://github.com/openclaw/clawhub/issues/2042, then add approved schema, recompute/backfill paths, indexed API/search support, browse/detail UI, and operator documentation.

## Reproduction Assessment

Not applicable: this is a product/data proposal rather than a runtime bug. Source inspection gives a high-confidence check that current main has only a publish-time quality gate and no public multidimensional quality-scoring surface.

## Solution Assessment

No patch is under review, and this is not yet a narrow implementation plan. The maintainable path is for maintainers to decide score dimensions, semantics, provenance, rollout, and relationship to the open tooling proposal before code changes.
