## Summary

Keep open: current main has upload-time content quality gating plus security audit signals, but not the requested public multidimensional skill quality-scoring model, and ClawHub cleanup policy does not auto-close open issues.

## What This Changes

The issue proposes a public multidimensional quality-signal model for ClawHub skills based on an external analysis of popular skills and trust-signal gaps.

## Best Possible Solution

Define a product-owned quality-signal model distinct from security/moderation, reconcile it with https://github.com/openclaw/clawhub/issues/2042, then add approved schema, indexed API/search support, backfill, browse/detail UI, and operator documentation.

No patch is under review, and the proposal is not yet a narrow implementation plan. The maintainable path is for maintainers to decide score dimensions, semantics, provenance, rollout, and separation from security labels before code changes.
