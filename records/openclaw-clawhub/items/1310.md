## Summary

Keep open: current main has security-audit surfaces and a narrow publish-time quality gate, but it does not implement the requested public multidimensional quality signals for documentation, maintenance, dependency hygiene, alternatives, or comparative skill quality. The related open items overlap the policy/tooling space but do not replace this broader product discussion.

## What This Changes

The issue asks ClawHub to consider public multidimensional quality signals for skills based on an external analysis of the top installed skills.

## Best Possible Solution

Define an explicit product/RFC contract for separate advisory quality dimensions, then implement schema, API, browse/detail UI, freshness, backfill, and policy semantics without conflating them with security or moderation verdicts.

## Reproduction Assessment

Not applicable: this is a product/data proposal for new public quality signals, not a broken existing behavior report. The high-confidence check is source, docs, live issue, related-item, and history review.

## Solution Assessment

No, not yet as a generic aggregate score. The narrower maintainable path is a maintainer-approved model that defines independent dimensions, evidence sources, freshness, rollout, and separation from security and moderation decisions.
