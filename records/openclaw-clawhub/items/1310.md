## Summary

Keep open: current main has only a narrow publish-time content quality gate plus security/moderation signals, not the requested public multidimensional skill quality model. The related tracker at https://github.com/openclaw/clawhub/issues/2042 is open context, not a shipped implementation or a reason to close this discussion.

## What This Changes

The issue proposes public multidimensional quality signals for ClawHub skills based on an external top-50 scan and trust-signal gaps.

## Best Possible Solution

Define a product-owned quality-signal model distinct from security/moderation, reconcile it with https://github.com/openclaw/clawhub/issues/2042, then add approved schema, indexed API/search support, backfill, browse/detail UI, and operator documentation.

No patch is under review, and the proposal is not yet a narrow implementation plan. The maintainable path is for maintainers to decide score dimensions, semantics, provenance, rollout, and security-label separation before code changes.
