## Summary

Keep open: current main has a deterministic publish-time content quality gate and a mature security audit stack, but it still does not implement the requested public, multidimensional skill quality signals. The related discussions at https://github.com/openclaw/clawhub/issues/2042 and https://github.com/openclaw/clawhub/issues/2321 overlap, but neither supersedes this broader quality-signal proposal.

## What This Changes

The issue requests structured public quality signals for ClawHub skills beyond existing security scanners, based on a top-50 skill analysis.

## Best Possible Solution

Define an advisory quality-signal contract distinct from security and moderation verdicts, then implement approved schema, API/search support, backfill, and browse/detail UI around those dimensions.

## Reproduction Assessment

Not applicable: this is a product/data proposal for new public quality signals, not a broken existing behavior report. The high-confidence check is live issue review plus source inspection showing the requested public capability is absent.

## Solution Assessment

Unclear: structured quality signals are a plausible direction, but a generic aggregate score is not established as the best solution yet. Maintainers need to define dimensions, evidence sources, freshness, rollout, and separation from security/moderation before implementation.
