## Summary

Keep open. This is a ClawHub issue, and current main still has publish-time quality gating plus security/moderation scan surfaces, not the requested public multidimensional skill quality scoring model.

## What This Changes

The issue proposes structured public quality signals for skills, covering documentation, claim accuracy, complexity, maintenance, dependency hygiene, alternatives, and prompt-only skill evaluation alongside security signals.

## Best Possible Solution

Define a product-owned quality-signal model distinct from security/moderation, then add agreed schema, API/search support, backfill/recomputation, and browse/detail UI presentation.
