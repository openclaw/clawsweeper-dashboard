## Summary

Keep open: current main has a narrow publish-time content-quality gate and security/moderation audit surfaces, but it does not implement the requested public, multidimensional skill quality-signal model. The related external-tooling proposal at https://github.com/openclaw/clawhub/issues/2042 is still open and useful context, not a merged or shipped replacement.

## What This Changes

The issue requests public multidimensional quality signals for ClawHub skills, based on a top-50 ecosystem analysis showing gaps beyond existing safety scanners.

## Best Possible Solution

Define a product-owned quality-signal model distinct from security/moderation, reconcile it with https://github.com/openclaw/clawhub/issues/2042, then add approved schema, indexed API/search support, backfill, browse/detail UI, and operator documentation.
