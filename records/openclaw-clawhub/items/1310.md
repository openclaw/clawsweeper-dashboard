## Summary

Current main has mature security-audit surfaces and an internal publish-time anti-spam quality gate, but it does not provide the requested public, multidimensional quality signals for documentation, maintenance, claim accuracy, dependency hygiene, or alternatives. The related open issues cover a proposed external testing tool and content-policy categories, respectively, but neither currently owns or implements this full product decision, so this issue remains useful for maintainer discussion.

## What This Changes

Add public, multidimensional quality signals for ClawHub skills that complement rather than replace security scan verdicts.

## Best Possible Solution

Define explainable, separately labeled quality dimensions with provenance and freshness, validate them on a representative prompt-only and code-bearing corpus, and keep security verdicts and moderation enforcement visibly distinct.

## Reproduction Assessment

Not applicable; this issue proposes a new quality and curation capability rather than reporting broken behavior with an established contract.

## Solution Assessment

No, not yet as a generic aggregate score. The narrower maintainable approach is a product RFC defining independent dimensions, evidence sources, freshness, rollout, and their strict separation from security and moderation decisions.
