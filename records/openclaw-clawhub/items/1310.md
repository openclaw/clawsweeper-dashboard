## Summary

Keep open: this is a ClawHub issue, and current main still lacks the requested public multidimensional skill quality-scoring model; the existing quality code is a narrow publish-time content gate plus security/moderation surfaces.

## What This Changes

The issue proposes adding public quality signals for ClawHub skills, covering dimensions like documentation, maintenance, dependency hygiene, claim-vs-reality, alternatives, and quality/security signal divergence.

## Best Possible Solution

Define a product-owned quality-signal model distinct from security/moderation, reconcile it with https://github.com/openclaw/clawhub/issues/2042, then add approved schema, indexed API/search support, UI, backfill, and operator docs.

## Reproduction Assessment

Not applicable: this is a product/data proposal rather than a runtime bug. Source inspection gives a high-confidence baseline that current main has only a narrow publish-time quality gate and no public multidimensional quality-scoring surface.

## Solution Assessment

No patch is under review, and the proposal is not yet a narrow implementation plan. The maintainable path is for maintainers to decide score semantics, provenance, rollout, and separation from security labels before code changes.
