## Summary

Keep open: ClawHub cleanup policy does not auto-close issues, and current main still has a narrow publish-time content quality hold rather than the requested public multidimensional skill quality-scoring model. The related tooling proposal at https://github.com/openclaw/clawhub/issues/2042 is open and should be reconciled with this product/security discussion.

## What This Changes

The issue proposes adding public, multidimensional quality signals for ClawHub skills based on an external top-50 skill analysis that found gaps beyond existing security and moderation indicators.

## Best Possible Solution

Define a product-owned quality-signal model distinct from security/moderation, reconcile it with https://github.com/openclaw/clawhub/issues/2042, then add approved schema, backfill/recompute paths, indexed API/search support, browse/detail UI, and operator documentation.
