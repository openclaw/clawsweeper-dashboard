## Summary

Keep open: current main still has a narrow publish-time SKILL.md content quality gate plus security/moderation signals, not the requested public multidimensional skill quality-scoring model. The related open proposal at https://github.com/openclaw/clawhub/issues/2042 overlaps, but it does not supersede this issue's top-50 dataset and broader trust-signal questions.

## What This Changes

Discuss adding public multidimensional quality signals for ClawHub skills, based on an external top-50 scan that found gaps between security verdicts and broader quality dimensions.

## Best Possible Solution

Define a product-owned quality-signal model distinct from security/moderation, reconcile it with https://github.com/openclaw/clawhub/issues/2042, then add approved schema, API/search support, recompute/backfill paths, browse/detail UI, and operator documentation.
