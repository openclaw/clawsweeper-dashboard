## Summary

Keep open: this is a ClawHub issue that still needs product and security review; current main has a publish-time content-quality gate and security/moderation scan surfaces, but not the requested public multidimensional skill quality-scoring model.

## What This Changes

The issue proposes public quality signals for skills, based on static analysis of popular skills and dimensions such as documentation, maintenance, dependency hygiene, claim accuracy, and alternatives.

## Best Possible Solution

Define a product-owned quality-signal model distinct from security/moderation, reconcile it with https://github.com/openclaw/clawhub/issues/2042, then add approved schema, indexed API/search support, backfill, UI, and operator docs.
