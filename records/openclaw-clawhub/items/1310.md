## Summary

Keep open: current main has a narrow publish-time content quality gate and mature security-audit surfaces, but it does not implement the requested public multidimensional quality signals for documentation, claim accuracy, maintenance, dependency hygiene, alternatives, or comparative skill quality.

## What This Changes

The issue requests public multidimensional quality signals for ClawHub skills based on a top-50 static-analysis study and observed trust-signal gaps.

## Best Possible Solution

Define an explicit advisory quality-signal contract, then implement schema, API/search support, backfill, and detail/listing UI while keeping quality separate from security and moderation verdicts.

## Reproduction Assessment

Not applicable: this is a product/data proposal for new public quality signals, not a broken existing behavior report. The high-confidence check is source, docs, live issue, related-item, and history review.

## Solution Assessment

No, not as a generic aggregate score yet. The maintainable path is a maintainer-approved model that defines independent dimensions, evidence sources, freshness, rollout, and separation from security and moderation decisions.
