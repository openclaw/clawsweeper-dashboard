## Summary

Current main does not implement the requested public multidimensional skill-quality model. It exposes detailed security-audit findings, freshness, capability tags, update dates, and private moderation or quality-hold decisions, but documentation completeness, claim accuracy, proportional complexity, dependency hygiene, maintenance scoring, and alternatives remain undefined as a public contract; the related tooling proposal and content-policy RFC are also still open.

## What This Changes

Evaluate adding public multidimensional quality signals for ClawHub skills beyond existing security scanning.

## Best Possible Solution

Define separate, explainable quality dimensions with evidence sources, freshness rules, prompt-only handling, UI semantics, and appeal or moderation boundaries, then implement them as distinct signals rather than an unexplained aggregate trust score.

## Reproduction Assessment

Not applicable; this issue proposes a new public quality and trust capability rather than reporting broken existing behavior. Source inspection verifies the capability is absent, but there is no failure scenario to reproduce.

## Solution Assessment

No, not as a direct aggregate score yet. The maintainable path is to first define independent evidence-backed dimensions and their relationship to security, moderation, freshness, and appeals, then decide whether an aggregate presentation is warranted.

## Risks / Open Questions

- A single aggregate score could conflate security evidence, subjective quality, maintenance activity, and moderation policy, causing users to treat unlike signals as equivalent.
- Prompt-only skills lack code-level evidence, so broad scoring would require new evaluation methods and defensible freshness, disclosure, and appeal rules.
- The related proposals overlap but are not identical: issue 2042 is tooling-oriented, while issue 2321 governs policy categories; closing this issue now could lose the broader public quality-dimension discussion.
