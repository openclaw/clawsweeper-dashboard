## Summary

Keep open: the feature is useful and not already fully on main, but the patch has a source-level blocker where resetting prefill state can drop an explicit empty summary during file replacement.

## What This Changes

The PR extends the skill publish form to import SKILL.md descriptions into Short summary, show a dismissible discovery banner, add tests, and raise the publish summary limit from 200 to 300 characters.

## Best Possible Solution

Keep the banner and 300-character contract, but preserve explicit summary edits unless a non-empty imported SKILL.md description is actually applied to the field.

## Reproduction Assessment

Yes, source-level: the PR clears summaryTouchedRef during file upload, while the submit path only sends an empty summary when that ref remains true. I did not run the browser flow locally.

## Solution Assessment

No, not as currently written. The maintainable shape is to reset the recommendation UI separately from the explicit user-edit state, or clear the touched state only when applying a real imported description.
