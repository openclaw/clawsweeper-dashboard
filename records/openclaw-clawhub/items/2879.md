## Best Possible Solution

Keep the new import note and 300-character contract, but separate recommendation reset from the user-edit sentinel so only an applied non-empty imported description clears manual summary state.

## Reproduction Assessment

Yes, source-level: the latest head clears summaryTouchedRef during file replacement, while submit only sends an empty summary if that ref remains true. I did not run the browser flow locally.

## Solution Assessment

No, not as currently written. The maintainable fix is to reset the recommendation UI separately from the explicit user-edit state, or clear the touched state only when applying a real imported description.
