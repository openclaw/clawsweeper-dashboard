## Summary

Current main does not provide a cron-template catalog or install contract: ClawHub’s package model is explicitly limited to skills, code plugins, and bundle plugins. The request remains a valid but security-sensitive product expansion that needs maintainer decisions on artifact shape, runtime import, consent, moderation, updates, and revocation before implementation.

## What This Changes

Add a ClawHub catalog for browsing, previewing, sharing, and installing reusable cron job templates into OpenClaw.

## Best Possible Solution

Define a constrained, versioned automation manifest with a complete pre-install preview, explicit operator consent, capability and secret declarations, provenance/moderation metadata, and safe disable/update/revocation semantics; then add ClawHub browse and publish surfaces only after OpenClaw exposes a compatible import API.

Not applicable: this issue requests a new catalog, artifact type, and installation workflow rather than reporting a defect in current behavior.

## Solution Assessment

No, not yet as the proposed skill-like one-click install flow. The maintainable solution first establishes a schedule-specific manifest and explicit OpenClaw trust/import contract, then builds the ClawHub catalog around that contract.

## Risks / Open Questions

- A shared template can cause persistent future execution, so a naive one-click flow could conceal prompts, commands, destinations, credentials references, agent selection, or destructive schedules from the operator.
- The OpenClaw runtime import/export contract, version compatibility, update behavior, disable/uninstall semantics, and ownership of installed jobs are not defined in this repository.
- ClawHub moderation and scanning currently understand existing package families; treating cron templates as ordinary skills or plugins could leave unsafe schedule-specific behavior unreviewed.
