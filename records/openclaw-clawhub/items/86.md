## Summary

Current main does not implement a shareable scheduled-job artifact or installation flow: ClawHub catalogs skills and plugin packages, while the proposed recurring-execution lifecycle remains a product and security decision. No separate canonical issue or viable PR currently owns this feature, so it remains necessary and should stay open.

## What This Changes

Add a ClawHub catalog with preview and one-click installation for reusable OpenClaw cron-job templates.

## Best Possible Solution

Define a security-reviewed scheduled-automation artifact and OpenClaw installation contract that exposes its schedule, actions, required capabilities, credential use, destination, update behavior, and explicit activation consent before building the ClawHub catalog experience.

## Solution Assessment

Unclear: ClawHub is the appropriate discovery surface, but maintainers must first decide whether scheduled automations are a new package family, a constrained bundle format, or a runtime-owned artifact with a separate install API.
