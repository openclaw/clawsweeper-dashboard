## Summary

Keep open. Current main mitigates cached legacy registry values and the canonical API is healthy, but the reported `registry.clawhub.ai` hostname still returns Vercel `DEPLOYMENT_NOT_FOUND`, so older clients, direct callers, and explicit registry overrides can still hit the failure.

## What This Changes

Restore `clawhub explore` compatibility for users whose CLI or direct calls still resolve skills through `registry.clawhub.ai`.

## Best Possible Solution

Restore or redirect `registry.clawhub.ai` to the canonical ClawHub deployment, or deliberately retire it with migration guidance while preserving intentional custom registry overrides.

## Reproduction Assessment

Yes. A live request to `https://registry.clawhub.ai/api/v1/skills?limit=3` returned HTTP 404 with `x-vercel-error: DEPLOYMENT_NOT_FOUND`, while `https://clawhub.ai/api/v1/skills?limit=3` returned HTTP 200 with skill items.

## Solution Assessment

Yes for maintainer follow-up, not for automated code-only repair. Current main already handles cached legacy registry values, so the durable remaining fix is legacy-host deployment or redirect policy, or an explicit retirement decision.
