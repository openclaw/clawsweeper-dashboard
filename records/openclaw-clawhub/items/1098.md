## Summary

Keep open. Current main and the v0.15.0 CLI now mitigate cached non-explicit `registry.clawhub.ai` values, and the canonical API is live, but the reported legacy hostname still returns Vercel `DEPLOYMENT_NOT_FOUND`, so the remaining user-visible failure needs maintainer-owned domain/deploy follow-up.

## What This Changes

Restore `clawhub explore` compatibility for clients that still reach the retired `registry.clawhub.ai` endpoint instead of the canonical ClawHub API.

## Best Possible Solution

Restore or redirect `registry.clawhub.ai` to the canonical ClawHub deployment, keep the v0.15.0 CLI fallback, and then manually close this issue with upgrade guidance if maintainers are satisfied.

## Reproduction Assessment

Yes. A live curl to `https://registry.clawhub.ai/api/v1/skills?limit=3` still returns HTTP 404 `DEPLOYMENT_NOT_FOUND`, while the canonical `https://clawhub.ai/api/v1/skills?limit=3` returns populated skills.

## Solution Assessment

Yes for the remaining work: restoring or redirecting the legacy host is the narrowest maintainable fix because current main already handles cached non-explicit legacy registry values. Broadening CLI behavior to override explicit custom registries would be a separate policy choice and is not needed to fix the hostname outage.
