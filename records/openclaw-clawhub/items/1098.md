## Summary

Keep open. Current main and the live canonical API are healthy, but the reported legacy registry hostname still returns Vercel `DEPLOYMENT_NOT_FOUND`, so the backward-compatibility failure remains for older clients, direct callers, or explicit legacy registry configuration.

## What This Changes

The issue reports that `clawhub explore` stopped working because the legacy registry host returns Vercel `DEPLOYMENT_NOT_FOUND` while search still works.

## Best Possible Solution

Restore or redirect `registry.clawhub.ai` at the hosted Vercel/DNS layer, then consider a narrow CLI migration for cached non-explicit legacy values while preserving explicit custom registries.

## Reproduction Assessment

Yes. The legacy host and its `/api/v1/skills` path still return `DEPLOYMENT_NOT_FOUND`, and source inspection shows `cmdExplore` uses the resolved registry base for that request.

## Solution Assessment

Yes for the primary direction. Hosted domain restoration is the narrow compatibility fix; a CLI fallback is secondary and needs maintainer judgment around cached versus explicit registry overrides.
