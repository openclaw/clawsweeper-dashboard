## Summary

Keep open. The ClawHub cleanup policy keeps issues open, and the reported legacy host still reproduces the Vercel `DEPLOYMENT_NOT_FOUND` failure even though current main and the live canonical API now point discovery and default registry traffic at `https://clawhub.ai`.

## What This Changes

The issue reports that `clawhub explore` stopped working because it used `registry.clawhub.ai`, which now returns Vercel `DEPLOYMENT_NOT_FOUND` while `clawhub search` still works.

## Best Possible Solution

Restore or redirect `registry.clawhub.ai` to the canonical ClawHub deployment for backward compatibility, then consider a narrow CLI compatibility patch for cached non-explicit legacy values while preserving intentional custom registry overrides.

## Reproduction Assessment

Yes. On 2026-05-04, direct requests to `registry.clawhub.ai` and its `/api/v1/skills?limit=3` endpoint returned Vercel 404 `DEPLOYMENT_NOT_FOUND`, and current source shows `explore` will use that endpoint when the resolved registry base is the legacy host.

## Solution Assessment

Yes for the main direction: hosted-domain restoration or redirect is the narrow durable fix. A CLI fallback is secondary and should be limited so explicit custom registry configuration remains respected.
