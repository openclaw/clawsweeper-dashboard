## Summary

Keep open. Current main mitigates cached legacy registry values and the canonical API is healthy, but `registry.clawhub.ai` still returns Vercel `DEPLOYMENT_NOT_FOUND`, so older clients, direct callers, and explicit registry configurations can still fail.

## What This Changes

Restore `clawhub explore` behavior for callers that still resolve or configure the retired `registry.clawhub.ai` registry host.

## Best Possible Solution

Restore or redirect `registry.clawhub.ai` to the canonical ClawHub deployment, or deliberately retire it with migration guidance while preserving intentional custom registry overrides.

## Reproduction Assessment

Yes. A live request to `https://registry.clawhub.ai/api/v1/skills?limit=3` returned HTTP/2 404 with `x-vercel-error: DEPLOYMENT_NOT_FOUND`, while the canonical `https://clawhub.ai/api/v1/skills?limit=3` returned HTTP/2 200 with skill items.

## Solution Assessment

Yes for maintainer follow-up, not for automated source-only repair. Current main already handles cached legacy registry values, so the durable remaining fix is legacy-host deployment or redirect policy, or an explicit retirement decision.
