## Summary

Keep open. The reported legacy hostname still returns Vercel `DEPLOYMENT_NOT_FOUND`, current main only defaults new registry discovery to `https://clawhub.ai`, and ClawHub cleanup policy leaves issues for maintainer follow-up rather than auto-closing them.

## What This Changes

The issue reports that `clawhub explore` stopped working because it resolves to `registry.clawhub.ai`, which returns Vercel `DEPLOYMENT_NOT_FOUND`, while `clawhub search` still works.

## Best Possible Solution

Restore or redirect `registry.clawhub.ai` to the canonical ClawHub deployment, then consider a narrow cached-legacy-registry migration that preserves explicit custom registry overrides.

## Reproduction Assessment

Yes. Curling `https://registry.clawhub.ai/api/v1/skills?limit=3` currently returns HTTP 404 with `x-vercel-error: DEPLOYMENT_NOT_FOUND`, and current source shows `cmdExplore` uses the resolved registry for that same API path.

## Solution Assessment

Yes for the primary direction. Restoring or redirecting the legacy hostname is the narrow compatibility fix; any CLI fallback is secondary and needs maintainer judgment around cached versus explicit custom registries.
