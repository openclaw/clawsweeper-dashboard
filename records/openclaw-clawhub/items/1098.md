## Summary

Keep open. The live legacy hostname still returns Vercel `DEPLOYMENT_NOT_FOUND`, and current main only defaults new discovery traffic to `https://clawhub.ai`; cached or explicit `registry.clawhub.ai` values can still break `clawhub explore`.

## What This Changes

The issue reports that `clawhub explore` stopped working because it can resolve to the legacy `registry.clawhub.ai` deployment, which returns `DEPLOYMENT_NOT_FOUND`.

## Best Possible Solution

Restore or redirect `registry.clawhub.ai` to the canonical ClawHub deployment, then consider a narrow CLI migration for cached non-explicit legacy registry values while preserving explicit custom registries.

## Reproduction Assessment

Yes. The live legacy endpoint currently returns Vercel `DEPLOYMENT_NOT_FOUND`, and source inspection shows `cmdExplore` uses the resolved registry while the resolver does not classify `registry.clawhub.ai` as legacy.

## Solution Assessment

Yes for the primary direction. Restoring or redirecting the legacy host is the narrow compatibility fix; a CLI fallback is secondary and needs maintainer judgment around explicit custom registry overrides.
