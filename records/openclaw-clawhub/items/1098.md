## Summary

Keep open. Current main and the live canonical API use `https://clawhub.ai`, but the reported legacy host still returns Vercel `DEPLOYMENT_NOT_FOUND`, and ClawHub issue cleanup policy keeps issues open rather than auto-closing them.

## What This Changes

The issue reports that `clawhub explore` stopped working for users whose CLI or direct calls resolve through `registry.clawhub.ai`.

## Best Possible Solution

Restore or redirect `registry.clawhub.ai` to the canonical ClawHub deployment, then consider a narrow cached-legacy-registry migration that preserves explicit custom registry overrides.

## Reproduction Assessment

Yes. The legacy endpoint currently returns Vercel `DEPLOYMENT_NOT_FOUND`, and current source shows `cmdExplore` uses the resolved registry base for `/api/v1/skills` when a cached or explicit registry points there.

## Solution Assessment

Yes. Restoring or redirecting the legacy host is the narrow compatibility fix; a CLI fallback is only a secondary mitigation for future clients.
