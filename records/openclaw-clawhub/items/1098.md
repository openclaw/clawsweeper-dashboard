## Summary

Keep open. ClawHub CLI v0.15.0 mitigates cached registry values by migrating `registry.clawhub.ai` back to `clawhub.ai`, but the live legacy hostname still returns Vercel `DEPLOYMENT_NOT_FOUND`; the remaining work is deployment/domain policy rather than a completed repo fix.

## What This Changes

The issue reports that `clawhub explore` no longer works because it can resolve to the retired `registry.clawhub.ai` host, which returns a Vercel deployment-not-found error.

## Best Possible Solution

Restore or redirect `registry.clawhub.ai` to the canonical ClawHub deployment, or explicitly retire the hostname and document the supported migration while preserving intentional custom registry overrides.

## Reproduction Assessment

Yes. A live request to the reported legacy API endpoint still returns HTTP 404 with Vercel `DEPLOYMENT_NOT_FOUND`, while the same skills endpoint on `https://clawhub.ai` returns HTTP 200 with catalog items.

## Solution Assessment

Yes for maintainer follow-up, not for an automated code-only repair. Current main already handles the cached CLI fallback, so the durable remaining fix is legacy-host deployment/redirect policy or a deliberate decision to retire that hostname.
