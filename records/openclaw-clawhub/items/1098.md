## Summary

Keep open. Current main and the canonical live API are healthy, but the reported legacy registry hostname still returns Vercel DEPLOYMENT_NOT_FOUND, so this remains a hosted-domain compatibility issue for older clients, direct callers, and explicit legacy registry settings.

## What This Changes

The issue asks for `clawhub explore` to work again after calls involving `registry.clawhub.ai` began failing with Vercel DEPLOYMENT_NOT_FOUND.

## Best Possible Solution

Restore or redirect `registry.clawhub.ai` to the canonical ClawHub deployment, then consider a narrow CLI migration for cached legacy registry values while preserving explicit custom registries.

## Reproduction Assessment

Yes. Curling `https://registry.clawhub.ai/` or `https://registry.clawhub.ai/api/v1/skills?limit=3` currently returns Vercel DEPLOYMENT_NOT_FOUND, and current main's `cmdExplore` uses the resolved registry base for that skills request.

## Solution Assessment

Yes for the primary direction. Hosted domain restoration is the narrow compatibility fix; a CLI-only fallback is secondary because it cannot fix direct legacy-host requests and must preserve explicit custom registries.
