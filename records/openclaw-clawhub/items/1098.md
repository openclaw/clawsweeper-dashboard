## Summary

Keep open. Current main and the live canonical API now use `https://clawhub.ai`, but the reported legacy `registry.clawhub.ai` host still returns Vercel `DEPLOYMENT_NOT_FOUND`, so older clients, direct callers, or explicit legacy registry settings remain broken and need maintainer deployment/domain follow-up.

## What This Changes

The issue reports that `clawhub explore` stopped working because it used `registry.clawhub.ai`, which returned Vercel `DEPLOYMENT_NOT_FOUND`, while `clawhub search` still worked.

## Best Possible Solution

Restore or redirect `registry.clawhub.ai` to the canonical ClawHub deployment for backward compatibility, then optionally add a narrow CLI fallback for cached non-explicit legacy registry values while preserving intentional custom registry overrides.

## Reproduction Assessment

Yes. The reported legacy host still returns HTTP 404 with `DEPLOYMENT_NOT_FOUND`, and the current explore source path shows that a resolved registry base is used for the skills-list request.

## Solution Assessment

Unclear for a code-only repair. The maintainable fix starts with restoring or redirecting the legacy hostname; any CLI fallback should be a small compatibility decision rather than overriding all explicit registry settings.
