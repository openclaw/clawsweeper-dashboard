## Summary

Keep open. The ClawHub cleanup policy keeps issues open, and the reported legacy hostname still reproduces the Vercel `DEPLOYMENT_NOT_FOUND` failure even though current CLI defaults and discovery now point to `https://clawhub.ai`.

## What This Changes

Restore `clawhub explore` and catalog access for clients that resolve or use the legacy `registry.clawhub.ai` registry host.

## Best Possible Solution

Restore or redirect `registry.clawhub.ai` to the canonical ClawHub deployment for backward compatibility, then consider a narrow CLI migration for cached non-explicit legacy values.

## Reproduction Assessment

Yes. On May 3, 2026, the live legacy host returned HTTP 404 `DEPLOYMENT_NOT_FOUND` for both the root and `/api/v1/skills?limit=3`, while the canonical API returned HTTP 200 with catalog items.

## Solution Assessment

Yes for the main direction. The narrow durable fix is hosted-domain restoration or redirect first; CLI fallback should be secondary and should preserve explicit custom registry overrides.
