## Summary

Keep open. The reported legacy hostname still returns Vercel `DEPLOYMENT_NOT_FOUND`, and current main only protects default discovery traffic; cached, direct, or explicit `registry.clawhub.ai` registry values can still break `clawhub explore`.

## What This Changes

Restore `clawhub explore` compatibility for registry traffic that resolves to `registry.clawhub.ai`.

## Best Possible Solution

Restore or redirect `registry.clawhub.ai` to the canonical ClawHub deployment, then consider a narrow cached-registry migration that preserves intentional custom registries.

## Reproduction Assessment

Yes. The live legacy host and its `/api/v1/skills` path return Vercel `DEPLOYMENT_NOT_FOUND`, and source inspection shows `cmdExplore` uses the resolved registry base for that API call.

## Solution Assessment

Yes for the primary direction. Restoring or redirecting the legacy host is the narrow compatibility fix; a CLI fallback is secondary and needs maintainer judgment around cached versus explicit registry overrides.
