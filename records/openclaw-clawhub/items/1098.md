## Summary

Keep open. Current main mitigates new default CLI discovery by using the canonical `https://clawhub.ai`, but this is a ClawHub issue and the exact legacy hostname from the report still reproduces as a live hosted-domain compatibility failure that source cleanup should not auto-close.

## What This Changes

`clawhub explore` fails for users whose resolved registry points at `registry.clawhub.ai`, which returns Vercel `DEPLOYMENT_NOT_FOUND` even though the canonical skills API works.

## Best Possible Solution

Restore or redirect `registry.clawhub.ai` to the canonical ClawHub deployment, then consider a narrow CLI compatibility migration for cached non-explicit legacy registry values.

## Reproduction Assessment

Yes. The live legacy host still returns HTTP 404 `DEPLOYMENT_NOT_FOUND`, while the canonical well-known config and skills API return HTTP 200.

## Solution Assessment

Yes. Hosted-domain restoration or redirect is the narrowest complete fix because it covers old clients and direct callers; a CLI fallback is only a secondary mitigation.
