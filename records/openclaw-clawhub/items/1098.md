## Summary

Keep open. Current main mitigates cached legacy registry values and the canonical API is healthy, but the reported legacy hostname still reproduces Vercel `DEPLOYMENT_NOT_FOUND`, so older clients, direct callers, and explicit registry configurations can still fail.

## What This Changes

The issue reports that `clawhub explore` stopped working because it used the legacy `registry.clawhub.ai` host, while `clawhub search` still works.

## Best Possible Solution

Restore or redirect `registry.clawhub.ai` to the canonical ClawHub deployment, or explicitly retire it with migration guidance while preserving intentional custom registry overrides.

## Reproduction Assessment

Yes. A live request to `https://registry.clawhub.ai/api/v1/skills?limit=3` returned HTTP 404 with `x-vercel-error: DEPLOYMENT_NOT_FOUND`, while `https://clawhub.ai/api/v1/skills?limit=3` returned HTTP 200 with skill items.

## Solution Assessment

Yes for keeping this routed to maintainer follow-up, not for a new source-only autofix. Current main mitigates cached legacy registry values, but the durable remaining fix is legacy-host deployment or redirect policy, or an explicit retirement decision.
