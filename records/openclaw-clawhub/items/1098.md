## Summary

Keep open. Current main and the shipped CLI mitigate cached `registry.clawhub.ai` values, but the legacy hostname still returns Vercel `DEPLOYMENT_NOT_FOUND` while the canonical ClawHub API is healthy, so direct callers and explicit registry overrides can still hit the reported failure.

## What This Changes

The issue reports that `clawhub explore` stopped working because it used `registry.clawhub.ai`, which returns `DEPLOYMENT_NOT_FOUND`, while search still works.

## Best Possible Solution

Restore or redirect `registry.clawhub.ai` to the canonical ClawHub deployment, or explicitly retire the hostname with migration guidance while preserving intentional custom registry overrides.

## Reproduction Assessment

Yes. A live request to `https://registry.clawhub.ai/api/v1/skills?limit=3` returned HTTP 404 with `x-vercel-error: DEPLOYMENT_NOT_FOUND`, while `https://clawhub.ai/api/v1/skills?limit=3` returned HTTP 200 with skill items.

## Solution Assessment

Yes for maintainer follow-up, not for automated code-only repair. Current main already handles cached legacy registry values, so the durable remaining fix is legacy-host deployment/redirect policy or a deliberate retirement decision.
