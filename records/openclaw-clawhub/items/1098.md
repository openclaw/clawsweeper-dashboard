## Summary

Keep open. The reported legacy hostname failure still reproduces on April 29, 2026: `registry.clawhub.ai` returns Vercel `DEPLOYMENT_NOT_FOUND`, while the canonical `https://clawhub.ai` well-known config and skills API are live. Current main mitigates new default CLI discovery by using the canonical host, but it does not restore the legacy hostname or classify `registry.clawhub.ai` as a legacy cached registry value, so the user-visible compatibility problem remains and needs maintainer follow-up.

## What This Changes

The issue requests restoring `clawhub explore` after it started resolving through `registry.clawhub.ai`, which returns Vercel `DEPLOYMENT_NOT_FOUND`.

## Best Possible Solution

Restore or redirect `registry.clawhub.ai` to the canonical ClawHub deployment for backward compatibility, then consider a narrow CLI compatibility patch that treats cached non-explicit `registry.clawhub.ai` values as legacy while preserving intentional custom registry overrides.
