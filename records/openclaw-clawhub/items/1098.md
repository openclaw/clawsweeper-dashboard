## Summary

Keep open. Current main mitigates cached `registry.clawhub.ai` values for newer CLI builds and the canonical API is healthy, but the live legacy registry hostname still returns Vercel `DEPLOYMENT_NOT_FOUND`, so direct legacy-host callers and explicit registry configurations still need maintainer follow-up.

## What This Changes

The issue reports that `clawhub explore` stopped working because the legacy `registry.clawhub.ai` deployment returns `DEPLOYMENT_NOT_FOUND` while `clawhub search` still works.

## Best Possible Solution

Restore or redirect `registry.clawhub.ai` at the hosted Vercel/DNS layer, and keep the current CLI fallback for non-explicit cached legacy values without changing explicit custom registry semantics unless maintainers choose that migration.

## Reproduction Assessment

Yes. The legacy host and `/api/v1/skills` path still reproduce the reported `DEPLOYMENT_NOT_FOUND`, and source inspection shows explicit registry use can still route `explore` through that base.

## Solution Assessment
