## Summary

Keep open. The reported legacy host still returns Vercel `DEPLOYMENT_NOT_FOUND`, current main only routes default discovery to `https://clawhub.ai`, and ClawHub issue cleanup policy does not auto-close open issues even when partially mitigated.

## What This Changes

The issue asks for `clawhub explore` to work again when older or cached registry resolution points at `registry.clawhub.ai`.

## Best Possible Solution

Restore or redirect `registry.clawhub.ai` to the canonical ClawHub deployment, then add a narrow CLI migration for cached non-explicit legacy registry values while preserving intentional custom registries.

## Reproduction Assessment

Yes. The live legacy endpoint reproduces the reported `DEPLOYMENT_NOT_FOUND`, and current source shows `cmdExplore` uses the resolved registry while the resolver does not classify `registry.clawhub.ai` as legacy.

## Solution Assessment

Yes for the primary direction. Restoring or redirecting the legacy host is the complete compatibility fix; a CLI cache migration is useful secondary hardening but cannot fix direct callers or already-published clients.
