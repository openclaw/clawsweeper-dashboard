## Summary

Keep open. Current main and live discovery default to `https://clawhub.ai`, but the reported legacy hostname still returns Vercel `DEPLOYMENT_NOT_FOUND`, so older cached or explicit registry configurations can still hit the user-visible failure.

## What This Changes

The issue reports that `clawhub explore` no longer lists skills because it reaches the legacy `registry.clawhub.ai` deployment, while `clawhub search` still works.

## Best Possible Solution

Restore or redirect `registry.clawhub.ai` to the canonical ClawHub deployment, then consider a narrow cached-legacy-registry migration that preserves explicit custom registry overrides.

## Reproduction Assessment

Yes. The reported legacy endpoint currently returns HTTP 404 with Vercel `DEPLOYMENT_NOT_FOUND`, and current source shows `clawhub explore` uses the resolved registry base for `/api/v1/skills`; I did not run the CLI itself because that can write user config.

## Solution Assessment

Yes. Restoring or redirecting the legacy host is the complete compatibility fix for older clients and direct callers; a CLI fallback is only a secondary mitigation and needs maintainer policy judgment around explicit registries.
