## Summary

Keep open. Current main defaults new CLI discovery to the canonical `https://clawhub.ai`, but the reported legacy registry hostname still reproduces `DEPLOYMENT_NOT_FOUND`, so the user-visible compatibility problem remains for older clients, direct callers, cached settings, or explicit registry values.

## What This Changes

Report that `clawhub explore` stopped working because it used `registry.clawhub.ai`, which now returns Vercel `DEPLOYMENT_NOT_FOUND`.

## Best Possible Solution

Restore or redirect `registry.clawhub.ai` to the canonical ClawHub deployment for backward compatibility, then consider a narrow CLI migration for cached non-explicit legacy values.

## Reproduction Assessment

Yes. The live legacy host still returns HTTP 404 `DEPLOYMENT_NOT_FOUND` on May 3, 2026 for both the root and `/api/v1/skills`, while the canonical `clawhub.ai` well-known config and skills API return HTTP 200.

## Solution Assessment

Yes. The narrow maintainable fix is hosted-domain restoration or redirect first; source-only fallback is secondary and should preserve explicit custom registries.
