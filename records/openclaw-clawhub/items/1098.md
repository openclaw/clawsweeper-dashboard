## Summary

Keep open. The canonical ClawHub API is healthy, but the reported legacy hostname still returns Vercel `DEPLOYMENT_NOT_FOUND`, and ClawHub cleanup policy leaves issue closure to maintainers.

## What This Changes

The issue reports that `clawhub explore` stopped working because discovery can resolve to `registry.clawhub.ai`, which returns a Vercel deployment-not-found error.

## Best Possible Solution

Restore or redirect `registry.clawhub.ai` to the canonical ClawHub deployment, then consider a narrow cached-legacy-registry migration that preserves explicit custom registry overrides.

## Reproduction Assessment

Yes. The live legacy endpoint still returns Vercel `DEPLOYMENT_NOT_FOUND`, and current main shows `cmdExplore` uses the resolved registry base for `/api/v1/skills`.

## Solution Assessment

Yes for the primary direction. Restoring or redirecting the legacy host is the narrow compatibility fix; a CLI fallback is secondary and needs maintainer judgment around cached versus explicit registry values.
