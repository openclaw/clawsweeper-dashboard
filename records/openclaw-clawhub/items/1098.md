## Summary

Keep open. The live legacy `registry.clawhub.ai` hostname still returns Vercel `DEPLOYMENT_NOT_FOUND`, and current main only defaults/discovers `https://clawhub.ai`; cached or explicit legacy registry values can still break `clawhub explore`.

## What This Changes

The issue reports that `clawhub explore` stopped working because the CLI was using `registry.clawhub.ai`, which returns a Vercel deployment-not-found error.

## Best Possible Solution

Restore or redirect `registry.clawhub.ai` to the canonical ClawHub deployment, then consider a narrow CLI migration for cached non-explicit legacy values while preserving intentional custom registries.

## Reproduction Assessment

Yes. The live legacy endpoint still returns Vercel `DEPLOYMENT_NOT_FOUND`, and source inspection shows `cmdExplore` uses the resolved registry to build `/api/v1/skills`.

## Solution Assessment

Yes for the primary direction. Restoring or redirecting the legacy host is the narrow compatibility fix; a CLI fallback is secondary and needs maintainer judgment around cached versus explicit registry overrides.
