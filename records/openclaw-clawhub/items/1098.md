## Summary

Keep open: current main defaults new CLI discovery to the canonical ClawHub host, but the reported legacy registry hostname still returns Vercel DEPLOYMENT_NOT_FOUND and can still break older, cached, or explicit registry configurations.

The issue requests restoring `clawhub explore` after its registry host began returning Vercel DEPLOYMENT_NOT_FOUND while `clawhub search` still worked.

## Best Possible Solution

Restore or redirect `registry.clawhub.ai` to the canonical ClawHub deployment, then consider a narrow CLI compatibility rule for cached non-explicit legacy values while preserving intentional custom registries.
