## Summary

Keep open. Current main defaults new CLI discovery to `https://clawhub.ai`, but the reported legacy host still returns Vercel `DEPLOYMENT_NOT_FOUND`, so older clients, direct callers, cached values, or explicit registry settings can still hit the failure.

## What This Changes

The issue asks for `clawhub explore` discovery to work again after the legacy `registry.clawhub.ai` deployment started returning `DEPLOYMENT_NOT_FOUND`.

## Best Possible Solution

Restore or redirect `registry.clawhub.ai` to the canonical ClawHub deployment for backward compatibility, then consider a narrow CLI migration for cached non-explicit legacy values.

## Reproduction Assessment

Yes. The live legacy host returns HTTP 404 `DEPLOYMENT_NOT_FOUND` on 2026-05-03, and current source shows `clawhub explore` can still use that host when it is explicit or cached.

## Solution Assessment

Yes. Hosted-domain restoration or redirect is the narrowest complete fix; source-only fallback is secondary and should preserve intentional custom registries.
