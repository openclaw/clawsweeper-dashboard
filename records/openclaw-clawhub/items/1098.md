The issue reports that `clawhub explore` stopped working because it used `registry.clawhub.ai`, which returns Vercel `DEPLOYMENT_NOT_FOUND`, while search still works.

## Best Possible Solution

Restore or redirect `registry.clawhub.ai` to the canonical ClawHub deployment for backward compatibility, or explicitly retire it with migration guidance while preserving intentional custom registry overrides.

## Reproduction Assessment
