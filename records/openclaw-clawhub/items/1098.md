## Summary

Keep open. The current CLI defaults and well-known discovery point to `https://clawhub.ai`, and the canonical API returns skills, but the reported legacy hostname still returns Vercel `DEPLOYMENT_NOT_FOUND`, so older cached or explicit registry configurations can still hit the user-visible failure.

## What This Changes

The issue reports that `clawhub explore` stopped working because it resolves through `registry.clawhub.ai`, which now returns Vercel `DEPLOYMENT_NOT_FOUND`, while `clawhub search` still works.

## Best Possible Solution

Restore or redirect `registry.clawhub.ai` to the canonical ClawHub deployment, then add a narrow cached-legacy-registry migration only if maintainers want extra CLI resilience.

## Reproduction Assessment

Yes. The reported legacy endpoint currently returns HTTP 404 with Vercel `DEPLOYMENT_NOT_FOUND`, and source inspection shows `clawhub explore` uses the resolved registry base for `/api/v1/skills`; I did not run the CLI itself because that can update user config.

## Solution Assessment

Yes. Restoring or redirecting the legacy host is the only complete fix for older clients and direct callers; a CLI-only change can be a useful mitigation but would be incomplete on its own.
