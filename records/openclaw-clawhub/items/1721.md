## Summary

Keep open: current main and v0.17.0 cover registry metadata, package trust metadata, install pinning/telemetry, moderation, and disclosure, but they do not implement or accept a SkillClaw-style lifecycle-governance model. This remains a broad product, privacy, and security RFC rather than completed work.

## What This Changes

The issue proposes evaluating SkillClaw-style lifecycle metadata and governance for long-lived local skill libraries, including deduplication, evolution, and sharing across agents, devices, or teams.

## Best Possible Solution

Treat this as RFC input and decide whether to document SkillClaw as related tooling, consolidate it with the related open discussions, or design lifecycle/governance APIs with explicit privacy, provenance, consent, moderation, reversibility, and trust-boundary requirements.
