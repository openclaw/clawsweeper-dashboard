## Summary

Keep open: current main still has registry metadata, package trust metadata, minimal install telemetry, and local install pinning, but no accepted SkillClaw-style post-task skill evolution or lifecycle-governance model; related open discussions at https://github.com/openclaw/clawhub/issues/707 and https://github.com/openclaw/clawhub/issues/1883 are useful context, not superseding fixes.

## What This Changes

The issue proposes exploring SkillClaw-style skill lifecycle metadata and governance for deduplication, merging, improvement, sharing, and long-lived skill library management.

## Best Possible Solution

Treat this as RFC input and decide whether to document SkillClaw as related tooling, consolidate it with https://github.com/openclaw/clawhub/issues/707 and https://github.com/openclaw/clawhub/issues/1883, or design lifecycle/governance APIs with explicit privacy, provenance, consent, moderation, reversibility, and trust-boundary requirements.
