Keep open. This is a broad product/security RFC about lifecycle governance for local skill libraries, not a current-main bug or closeable duplicate; current main still treats ClawHub primarily as a registry/catalog with minimal install telemetry and no SkillClaw-style evolution or sharing layer.

The issue proposes considering SkillClaw-style lifecycle metadata, deduplication, evolution, sharing, and governance for long-lived skill libraries as a possible ClawHub direction.
