Keep open under the ClawHub issue cleanup policy. Current main addresses the reported publish-flow slug collision through explicit owner migration, while direct org-targeted transfer remains separate authorization-sensitive transfer work.

## What This Changes

Allow an existing skill owner to move a skill from a personal publisher to an organization publisher through republish or a dedicated transfer action.

## Best Possible Solution

Keep publish-time `ownerHandle` plus `migrateOwner` as the supported path for this report, and consolidate any direct transfer command/API work in the publisher-target transfer track.

## Reproduction Assessment

## Solution Assessment

Yes for the reported publish-flow failure: explicit opt-in migration with source and destination authorization is the narrow maintainable fix. Direct org-transfer should be handled in the separate transfer design because it changes ownership authority beyond republish.
