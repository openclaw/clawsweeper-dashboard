## Summary

Keep open under the ClawHub cleanup policy for issues. Current main fixes the original publish-flow slug-collision path with explicit owner migration, but direct org-targeted transfer remains separate open work.

## What This Changes

The issue asks for changing an existing skill owner from a personal publisher to an organization without a slug-collision error, either during publish or through a dedicated action.

## Best Possible Solution

Keep the shipped publish-time migration as the supported path for moving a skill during republish, and handle direct org-targeted transfer through the existing transfer issue/PR with explicit authorization rules.

## Reproduction Assessment

No for the original publish-flow failure: current source shows a confirmed owner change sends `migrateOwner` and the backend migrates instead of throwing the slug-taken collision. The separate direct transfer-to-org path is source-reproducible and tracked elsewhere.

## Solution Assessment

Yes for the reported publish-flow failure: explicit opt-in migration with source and destination authorization is the narrowest maintainable fix. Direct org-transfer support is a broader ownership-transfer design and should stay in the transfer track.
