## Summary

Keep open. Current main still has the source-level failure path, and the ClawHub cleanup policy for this pass keeps open issues open unless a maintainer closes them separately.

## What This Changes

Request support for moving an existing skill from a personal publisher to an organization publisher, either during republish or through a dedicated transfer flow.

## Best Possible Solution

Land a maintainer-approved publisher-target transfer design that updates the web flow, transfer API/CLI, slug aliases, audit logs, docs, and regression tests consistently.

## Reproduction Assessment

Yes. The source path is clear: choose an organization owner in the publish UI for an existing personal skill, publishVersion resolves that org publisher, and insertVersion rejects the existing skill because ownerPublisherId differs.

## Solution Assessment

Unclear. Allowing republish to mutate ownerPublisherId would fix the immediate error, but a dedicated publisher-transfer flow with explicit role checks and audit logging may be the safer long-term solution.
