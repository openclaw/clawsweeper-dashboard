## Summary

Keep this issue open. Current main still treats user-initiated account deletion as terminal during GitHub OAuth, related account-lockout reports remain open, and changing recovery or purge behavior needs maintainer account-lifecycle and security policy.

## What This Changes

The issue asks for a deleted-account recovery or purge path so affected users can sign in or re-register with the same GitHub account instead of returning to signed-out state.

## Best Possible Solution

Define one canonical lifecycle policy for deleted GitHub-linked accounts, then implement the approved recovery behavior across Convex auth, admin/support tooling, UI messaging, docs, and regression tests.

## Reproduction Assessment

Yes, from source inspection. The path is deleteAccount setting deactivatedAt/purgedAt, the OAuth callback rejecting that user on later GitHub sign-in, and active auth helpers returning no user for deactivated accounts; I did not run a live OAuth reproduction.

## Solution Assessment

No. A one-off purge or silent re-create behavior is not the best source change until maintainers choose the account lifecycle and security policy for deleted GitHub-linked accounts.
