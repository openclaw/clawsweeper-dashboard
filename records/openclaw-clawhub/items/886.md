## Summary

Keep this issue open. Current main intentionally blocks self-deleted GitHub-linked accounts during OAuth and now surfaces clearer blocked-account messaging, but there is still no approved purge, re-registration, restore, or support recovery path for affected users.

## What This Changes

The issue requests purging or otherwise recovering a deleted GitHub-linked ClawHub account so the user can sign in or register again.

## Best Possible Solution

Define one canonical lifecycle policy for deleted GitHub-linked accounts, then implement the approved recovery or support path across Convex auth, admin tooling, UI messaging, specs/docs, and regression tests.

## Reproduction Assessment

Yes, from source inspection. Deleting an account sets deactivatedAt and purgedAt, the GitHub OAuth callback rejects that user on later sign-in, and active auth helpers then return no active user; I did not run a live OAuth flow.

## Solution Assessment

No, not as a one-off purge or silent recreate. The maintainable fix needs an approved account lifecycle and security policy before changing OAuth bindings, deletion state, recovery tooling, or support operations.
