## Summary

Keep open. Current main still treats self-deleted accounts as permanently deactivated during GitHub OAuth, and the related open report cluster shows a real recovery/support problem that needs maintainer policy and production data handling rather than cleanup closure.

## What This Changes

The issue asks for account recovery or backend purge so a user who deleted a GitHub-linked ClawHub account can sign in or re-register again.

## Best Possible Solution

Define one maintainer-approved deleted-account recovery policy, then implement it consistently across OAuth, account deletion, auth binding, operator tooling, user-facing errors, docs/specs, and regression tests.

## Reproduction Assessment

Yes, for the reported inability to sign in after account deletion: source inspection shows deletion sets deactivatedAt/purgedAt, the OAuth callback rejects those users, and tests assert the block. The older silent-loop UX may now surface a clearer deleted-account error on current main.

## Solution Assessment

No for an automated source-only purge or OAuth recreate path. The maintainable fix needs a product and security decision first so recovery or unlinking cannot bypass identity, ban, audit, or privacy guarantees.
