## Summary

Keep this issue open. Current main still blocks deleted or deactivated GitHub-linked users during OAuth and has no approved account purge or re-registration path, while the requested recovery behavior is security-sensitive and needs maintainer policy/support direction.

## What This Changes

A user asks for help purging or resetting a deleted ClawHub account so the same GitHub identity can sign in again instead of returning to the sign-in page.

## Best Possible Solution

Define a single deleted-account lifecycle policy, then implement the approved support or recovery path across OAuth callback handling, account deletion/admin tooling, UI messaging, specs/docs, and regression tests.

## Reproduction Assessment

Yes, from source inspection. deleteAccount marks the account deactivated/purged, the OAuth callback rejects that state, and active auth helpers hide the user from app sessions.

## Solution Assessment

No, a one-off purge is not the best source-level solution. The maintainable fix needs an explicit recovery or support policy that preserves deletion semantics, OAuth identity binding, and ban protections.
