## Summary

Keep open. Current main still intentionally rejects deleted/deactivated ClawHub users during GitHub OAuth and suppresses them from active app sessions, so the reported login loop is a valid account-lifecycle/auth issue rather than cleanup-closeable. For ClawHub issues, this also needs maintainer handling rather than auto-close.

## What This Changes

The issue requests recovery or backend cleanup for users who delete their ClawHub account and then cannot sign back in with the same GitHub OAuth identity.

## Best Possible Solution

Define one canonical lifecycle policy for deleted GitHub-linked accounts, then implement the approved recovery behavior across the Convex auth callback, deletion/admin tooling, UI error messaging, docs, and regression tests for deleted, deactivated, banned, and legacy deletedAt users.
