## Summary

Keep open: current main still treats self-deleted GitHub-linked accounts as terminal during OAuth, and recovery or re-registration needs maintainer product/security policy rather than cleanup closure.

## What This Changes

The issue asks for recovery or backend cleanup after a self-deleted ClawHub account can no longer sign in or re-register through GitHub OAuth.

## Best Possible Solution

Define one canonical deleted-account lifecycle policy, then implement the approved OAuth recovery, admin/support tooling, user messaging, specs/docs, and regression tests together.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, handleDeletedUserSignIn rejects that identity during OAuth, and active auth helpers return no active session; I did not run a live production deletion test.

## Solution Assessment

No. A one-off purge is not the best repo-level solution because deleted-account recovery needs an approved lifecycle policy that preserves OAuth identity binding, deletion guarantees, and ban boundaries.
