## Summary

Keep this issue open. Current main still treats self-deleted GitHub-linked accounts as terminal during OAuth, and ClawHub cleanup policy keeps issues open for maintainer handling rather than auto-closing them.

## What This Changes

The issue asks for a recovery, purge, or re-registration path after a user deletes a ClawHub account and then cannot complete GitHub sign-in.

## Best Possible Solution

Define a canonical deleted-account lifecycle policy, then implement the approved recovery or support path across Convex auth, account/admin tooling, user messaging, docs/specs, and regression tests.

## Reproduction Assessment

Yes, from source inspection. Deleting an account sets deactivatedAt and purgedAt, later GitHub OAuth rejects the linked user as deleted, and active auth helpers then present no active session.

## Solution Assessment

No, not as a one-off purge or silent recreate. The maintainable fix needs an explicit account lifecycle and security policy before changing OAuth bindings, deletion state, recovery tooling, or support operations.
