## Summary

Keep open: current main still blocks sign-in for deleted ClawHub accounts, and the remaining work is a security-sensitive account-lifecycle/support decision rather than cleanup closure.

## What This Changes

The issue requests a way to sign back in or re-register after account deletion leaves GitHub OAuth returning the user to the signed-out state.

## Best Possible Solution

Define one canonical deleted-account lifecycle and recovery policy, then implement the approved auth, admin/support tooling, user messaging, specs/docs, and regression tests together.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, handleDeletedUserSignIn rejects that user during OAuth, and active auth helpers return no session. I did not perform a live production account-deletion test.

## Solution Assessment

No, a one-off hard purge is not the best repo-level solution. The maintainable fix needs an approved recovery or support policy that preserves OAuth identity binding, irreversible deletion semantics, and ban handling.
