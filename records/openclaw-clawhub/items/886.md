## Summary

Keep open. Current main still treats user-initiated deletion as terminal during GitHub OAuth, so the reported lockout is source-reproducible, but changing recovery or purge behavior needs maintainer account-lifecycle and security policy.

## What This Changes

The issue asks ClawHub maintainers to restore or purge a deleted GitHub-linked account so the user can sign in and re-register.

## Best Possible Solution

Define one canonical policy for deleted GitHub-linked accounts, then implement the approved recovery or support-only flow across Convex auth, admin tooling, UI messaging, docs, and regression tests.

## Reproduction Assessment

Yes, from source inspection. Deleting an account sets deactivatedAt/purgedAt, the next OAuth callback rejects that user, and active-session helpers suppress the user afterward; I did not run a live production repro in this read-only review.

## Solution Assessment

Unclear. A one-off purge may unblock a user, but the maintainable solution needs a maintainer-approved lifecycle policy so recovery, recreation, bans, and OAuth identity binding stay consistent.
