## Summary

Keep this issue open. Current main still intentionally blocks GitHub sign-in for permanently deleted accounts, multiple related account-lockout reports remain open, and changing the recovery path needs maintainer account-lifecycle and security policy.

## What This Changes

The issue asks for a way to sign in or re-register after a user deletes their ClawHub account and later completes GitHub OAuth.

## Best Possible Solution

Define one canonical deleted-account recovery policy, then implement the approved support or self-service path across Convex auth, admin tooling, UI messaging, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, the OAuth callback rejects that user on later GitHub sign-in, and active auth helpers return no user for deactivated accounts.

## Solution Assessment

No. A one-off purge/recreate behavior is not the best source change until maintainers define the account-lifecycle and security policy for deleted GitHub-linked accounts.
