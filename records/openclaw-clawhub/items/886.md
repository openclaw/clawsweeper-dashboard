## Summary

Keep open: current main still treats self-deleted ClawHub accounts as terminal during GitHub OAuth, and recovery needs an explicit account-lifecycle, support, and security policy before implementation changes.

## What This Changes

The issue requests a way to recover or re-register after deleting a ClawHub account linked to GitHub and then getting returned to the signed-out state.

## Best Possible Solution

Define one canonical deleted-account lifecycle policy, then implement the approved OAuth recovery or support path with admin tooling, UI messaging, specs/docs, and regression coverage for deleted, deactivated, banned, and legacy deletedAt users.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: self-service deletion sets deactivatedAt/purgedAt, OAuth reauth rejects that user, and active auth helpers return no active session; I did not run a live production deletion test.

## Solution Assessment

No. A one-off purge is not the best repo-level solution because deleted-account recovery needs an approved lifecycle policy that preserves OAuth identity binding, deletion guarantees, and ban boundaries.
