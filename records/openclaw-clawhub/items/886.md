## Summary

Keep open: current main still blocks self-deleted ClawHub accounts during GitHub OAuth, and resolving the reporter’s re-registration/purge request needs an account-lifecycle and security policy decision rather than cleanup closure.

## What This Changes

The issue requests recovery or backend cleanup for GitHub sign-in after a user deletes their ClawHub account and then cannot re-register.

## Best Possible Solution

Define one canonical deleted-account lifecycle policy, then implement the approved OAuth behavior, support/admin tooling, user messaging, docs/spec updates, and regression tests together.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, handleDeletedUserSignIn rejects that user during OAuth, and active auth helpers return no session; I did not perform a live production deletion test.

## Solution Assessment

No. A one-off purge is not the best repo-level solution because recovery behavior needs an approved lifecycle policy that preserves OAuth identity binding, deletion guarantees, and ban boundaries.
