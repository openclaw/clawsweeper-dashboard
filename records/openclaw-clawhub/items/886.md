## Summary

Keep open. Current main still treats user-initiated account deletion as terminal for GitHub OAuth, and resolving the requested purge/re-registration path needs maintainer account-lifecycle and security policy rather than cleanup closure.

## What This Changes

The issue asks for a recovery or purge/re-registration path for GitHub OAuth accounts that remain signed out after self-deleting a ClawHub account.

## Best Possible Solution

Define one canonical policy for self-deleted GitHub-linked accounts, then implement the approved recovery, support, or explicit-rejection flow across auth, admin tooling, UI messaging, docs, and regression tests.

## Reproduction Assessment

Yes, from source inspection. deleteAccount writes deactivatedAt/purgedAt, the OAuth callback rejects deleted or deactivated users, access helpers hide them from active sessions, and tests assert that sign-in remains blocked.

## Solution Assessment

Unclear. A narrow code change that recreates or purges records during login would touch identity and moderation boundaries, so the best solution depends on a maintainer-approved lifecycle/support policy first.
