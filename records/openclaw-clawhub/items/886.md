## Summary

Keep this issue open. Current main surfaces clearer blocked-account auth errors, but it still intentionally rejects self-deleted GitHub-linked accounts and has no approved purge, restore, or re-registration path.

## What This Changes

The issue asks ClawHub maintainers to purge or otherwise recover a self-deleted GitHub-linked account so the user can sign in or register again.

## Best Possible Solution

Define one canonical deleted-account lifecycle policy, then implement the approved recovery/support path across Convex auth, admin tooling, UI messaging, docs/specs, and regression tests.

## Reproduction Assessment

Yes, from source inspection. The path is account deletion setting deactivatedAt/purgedAt, the OAuth callback rejecting that user on later GitHub sign-in, and active auth helpers returning no active user; I did not run a live OAuth flow.

## Solution Assessment

No, not as a one-off purge or silent recreate. The maintainable fix needs an approved account lifecycle and security policy before changing OAuth bindings, deletion state, or support recovery behavior.
