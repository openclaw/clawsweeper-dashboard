## Summary

Keep this issue open. Current main still treats self-deleted GitHub-linked accounts as terminal during OAuth, and ClawHub cleanup policy does not auto-close issues; choosing purge, restore, or re-registration behavior needs maintainer product, security, and production-support handling.

## What This Changes

The issue asks ClawHub to recover or purge a deleted GitHub-linked account so the user can sign in or re-register instead of returning to the sign-in page.

## Best Possible Solution

Define one canonical lifecycle policy for deleted GitHub-linked accounts, then implement the approved recovery or support behavior across Convex auth, account deletion or admin tooling, UI messaging, specs/docs, and regression tests.

## Reproduction Assessment

Yes, from source inspection. deleteAccount sets deactivatedAt/purgedAt, the OAuth callback rejects those users, and active auth helpers expose no active user, matching the reported OAuth-completes-but-still-signed-out behavior.

## Solution Assessment

No, a one-off purge request is not the best source-level solution by itself. The maintainable path needs an explicit recovery or support policy that preserves OAuth identity binding, irreversible deletion semantics, and ban protections.
