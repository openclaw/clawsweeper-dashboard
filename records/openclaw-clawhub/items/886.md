## Summary

Keep open. Current main still treats user-initiated account deletion as terminal and rejects later GitHub OAuth sign-in for deleted/deactivated users, and ClawHub issue cleanup policy keeps issues in maintainer hands rather than auto-closing them.

## What This Changes

The issue asks ClawHub to recover, purge, or re-register a GitHub-linked account after account deletion leaves sign-in stuck in a loop.

## Best Possible Solution

Define one canonical lifecycle policy for deleted GitHub-linked accounts, then implement the approved recovery behavior across the Convex auth callback, account deletion/admin tooling, UI messaging, docs, and regression tests.

## Reproduction Assessment

Yes. The source path is clear: deleteAccount sets deactivatedAt/purgedAt, the next OAuth callback calls handleDeletedUserSignIn and throws, and access helpers then suppress the user from active sessions; auth tests cover this behavior.

## Solution Assessment

Unclear. A fix should follow a maintainer-approved recovery, purge, or support-only policy; blindly recreating or purging records during login is not the safest path because it touches identity and moderation boundaries.
