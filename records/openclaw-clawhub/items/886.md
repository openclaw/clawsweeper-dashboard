## Summary

Keep this issue open. Current main still makes user-initiated account deletion terminal for GitHub OAuth, and the requested purge/re-registration path needs maintainer support and security policy rather than conservative cleanup closure.

## What This Changes

The issue asks maintainers to recover or purge a self-deleted ClawHub account so the same GitHub identity can sign in again instead of returning to the sign-in page.

## Best Possible Solution

Define one canonical deleted-account lifecycle policy, then implement the approved support or recovery path across OAuth callback handling, admin/account tooling, UI messaging, specs/docs, and regression tests.

## Reproduction Assessment

Yes, from source inspection. Deleting an account sets deactivatedAt/purgedAt, the OAuth callback rejects that state, and active auth helpers hide the user, matching the reported return-to-signed-out behavior.

## Solution Assessment

No, a one-off purge is not the best source-level solution. The maintainable path needs an explicit account lifecycle and support policy that preserves OAuth identity binding, irreversible deletion semantics, and ban protections.
