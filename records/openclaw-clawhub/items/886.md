## Summary

Keep open. Current main still blocks self-deleted GitHub-linked accounts from signing in again, and this ClawHub issue needs maintainer-owned account lifecycle and support/security policy rather than cleanup closure.

## What This Changes

The issue asks ClawHub maintainers to recover or purge a self-deleted GitHub-linked account so the reporter can sign in again.

## Best Possible Solution

Define one canonical lifecycle policy for deleted GitHub-linked accounts, then implement the approved recovery, purge, or support-only behavior across auth, admin tooling, UI messaging, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection on current main shows `deleteAccount` sets `deactivatedAt`/`purgedAt`, the OAuth callback rejects that state, and active auth helpers suppress the user from app sessions.

## Solution Assessment

Unclear. The requested purge/re-register outcome is not a narrow bug fix until maintainers choose the account lifecycle policy and safeguards for deleted, banned, and legacy users.
