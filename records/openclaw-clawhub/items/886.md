## Summary

Keep this issue open. Current main still intentionally blocks sign-in for permanently deleted ClawHub accounts, and the repeated account-lockout reports need a maintainer account-lifecycle/security decision rather than conservative cleanup closure.

## What This Changes

The issue requests a purge or recovery path so a GitHub-linked account can sign in again after user-initiated ClawHub account deletion.

## Best Possible Solution

Define one canonical deleted-account lifecycle policy, then implement the approved recovery or support path across the Convex OAuth callback, account/admin tooling, UI messaging, specs/docs, and regression tests.

## Reproduction Assessment

Yes, from source inspection. Deleting an account sets deactivatedAt/purgedAt, the OAuth callback rejects that state, and active auth helpers hide the user, matching the reported return-to-signed-out behavior.

## Solution Assessment

No, a one-off purge is not the best source-level solution. The safer path is an explicit maintainer-approved lifecycle policy that preserves OAuth identity binding, deletion semantics, and ban protections.
