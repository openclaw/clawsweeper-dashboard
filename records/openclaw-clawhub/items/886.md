## Summary

Keep this issue open. Current main still blocks GitHub sign-in for self-deleted or deactivated accounts, and ClawHub cleanup policy keeps issues open for maintainer handling rather than auto-closing them.

## What This Changes

The issue requests a recovery or purge path for users who delete a ClawHub account and then get stuck trying to sign in again with GitHub.

## Best Possible Solution

Define one canonical lifecycle policy for deleted GitHub-linked accounts, then implement the approved recovery behavior across auth, admin/support tooling, UI messaging, docs, and regression tests.

## Reproduction Assessment

Yes from source inspection: a user marked deactivatedAt or legacy deletedAt is rejected during GitHub auth and filtered out of active sessions, matching the reported loop. I did not run a live OAuth flow or mutate production-like account data in this read-only review.

## Solution Assessment

No, an ad hoc purge is not the best repository-level fix because repeated reports show a policy and tooling gap. The maintainable solution should first choose restore, purge/recreate, or support-only recovery, then cover deleted, deactivated, banned, and legacy deletedAt cases with tests.
