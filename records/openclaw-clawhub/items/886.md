## Summary

Keep this issue open. Current main still intentionally treats self-deleted accounts as terminal during GitHub OAuth, and the requested recovery path needs maintainer account-lifecycle and security policy rather than conservative cleanup.

## What This Changes

The issue requests recovery or backend cleanup for users stuck in a GitHub OAuth sign-in loop after deleting their ClawHub account.

## Best Possible Solution

Define one canonical deleted-account recovery policy, then implement the approved support/admin or self-service path across Convex auth, account lifecycle tooling, UI messaging, docs, and regression tests.

## Reproduction Assessment

Yes, from source inspection rather than a live OAuth run. Self-deletion sets deactivatedAt/purgedAt, the OAuth callback rejects that user on later GitHub sign-in, and tests assert the rejected reauth behavior.

## Solution Assessment

No, current main is not the best final solution for the requested recovery. A direct purge/recreate path needs maintainer policy first so deletion semantics, OAuth identity binding, and ban bypass protections stay consistent.
