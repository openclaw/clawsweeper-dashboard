## Summary

Keep this issue open. Current main still intentionally blocks GitHub OAuth for deleted/deactivated accounts, the related account-lockout cluster remains open, and resolving this needs maintainer account-lifecycle, security, and production-support policy rather than conservative cleanup closure.

## What This Changes

The issue reports a GitHub OAuth sign-in loop after a user deleted their ClawHub account and asks for purge or re-registration support.

## Best Possible Solution

Define one supported post-deletion lifecycle for GitHub-linked accounts, then implement the approved restore, purge/recreate, unlink/relink, or explicitly terminal support flow across OAuth, admin tooling, UI messaging, specs/docs, and regression tests.

## Reproduction Assessment

Yes, at source level: deleting an account writes deactivatedAt/purgedAt, the OAuth callback calls handleDeletedUserSignIn, and active-user helpers reject deactivated users. I did not verify the reporter’s production database record.

## Solution Assessment

No for a one-off purge as the source-level solution. The maintainable fix needs a maintainer-approved account recovery or terminal-support policy with provider-account verification, audit logging, and abuse safeguards.
