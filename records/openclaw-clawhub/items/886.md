## Summary

Keep open: current main still blocks GitHub sign-in for permanently deleted ClawHub accounts, and resolving this needs maintainer product/security policy rather than cleanup closure.

## What This Changes

The reporter asks ClawHub to purge or otherwise recover a GitHub-linked account after self-service deletion causes OAuth to return them to the signed-out state.

## Best Possible Solution

Define one canonical lifecycle policy for deleted GitHub-linked accounts, then implement the approved recovery or support path across Convex auth, admin tooling, UI messaging, specs/docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: self-service deletion sets deactivatedAt/purgedAt, GitHub OAuth reauth rejects that user, and active auth helpers return no active session; I did not run a live production deletion test.

## Solution Assessment

No. A one-off purge is not the best repo-level solution because deleted-account recovery needs an approved lifecycle policy that preserves OAuth identity binding, deletion guarantees, and ban boundaries.
