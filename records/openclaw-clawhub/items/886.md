## Summary

Keep this issue open. Current main now surfaces a clearer blocked-account auth error, but it still intentionally rejects self-deleted accounts during GitHub OAuth and does not provide the requested purge, restore, or re-registration path.

## What This Changes

The issue requests a way for a user who deleted their ClawHub account to sign in or re-register via GitHub instead of being returned to the signed-out state.

## Best Possible Solution

Define a canonical deleted-account recovery policy, then implement the approved support or product path across Convex auth, admin tooling, UI messaging, docs, and regression tests.

## Reproduction Assessment

Yes, from source inspection. The path is account deletion setting deactivatedAt/purgedAt, the OAuth callback rejecting that user on later GitHub sign-in, and active auth helpers returning no user; I did not run a live OAuth flow.

## Solution Assessment

No, not as a one-off source change. A purge or silent recreate path is not the best fix until maintainers decide the deleted-account recovery and security policy.
