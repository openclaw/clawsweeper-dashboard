## Summary

Keep open: current main still treats self-deleted ClawHub accounts as terminal during GitHub OAuth, and the remaining recovery/re-registration question needs account-lifecycle and security policy review rather than cleanup closure.

## What This Changes

The issue asks for recovery or purge support after a user deletes their ClawHub account and then gets stuck returning to the sign-in page after GitHub authorization.

## Best Possible Solution

Define one canonical deleted-account lifecycle policy, then implement the approved OAuth recovery, admin/support tooling, user messaging, docs/spec updates, and regression tests together.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, handleDeletedUserSignIn rejects that identity during OAuth, and active auth helpers return no active session; I did not run a live production deletion test.

## Solution Assessment

No. A one-off purge is not the best repo-level solution because deleted-account recovery needs an approved lifecycle policy that preserves OAuth identity binding, deletion guarantees, and ban boundaries.
