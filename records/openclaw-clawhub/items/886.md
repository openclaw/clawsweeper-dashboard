## Summary

Keep this issue open. Current main still treats self-deleted ClawHub accounts as terminal during GitHub OAuth, the source path matches the reported signed-out loop, and the remedy needs account-lifecycle/security policy plus possible production support action rather than cleanup closure.

## What This Changes

The issue requests recovery from a deleted ClawHub account state that causes GitHub OAuth to return to the sign-in page instead of creating an active session.

## Best Possible Solution

Define one canonical deleted-account lifecycle policy, then implement the approved recovery or support path across Convex auth, account/admin tooling, UI messaging, specs/docs, and regression tests.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: deleting an account sets deactivatedAt/purgedAt, the OAuth callback rejects that state, and active auth helpers hide the user afterward. I did not perform a live production account deletion test.

## Solution Assessment

No, a one-off purge is not the best source-level solution. The safer path is maintainer-approved recovery/support policy and tooling that preserves OAuth binding, deletion semantics, and ban protections.
