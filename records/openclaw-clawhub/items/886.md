## Summary

Keep this issue open. Current main still blocks GitHub sign-in for self-deleted accounts by design, although the UI now has clearer error handling, and resolving the remaining account-recovery request needs maintainer account-lifecycle and security policy rather than cleanup auto-close.

## What This Changes

The issue asks maintainers to recover or purge a self-deleted GitHub-linked ClawHub account so GitHub OAuth can create a usable session again.

## Best Possible Solution

Define a canonical deleted-account recovery policy, then implement the approved support/admin or self-service path across Convex auth, account lifecycle tooling, UI messaging, docs, and regression tests.

## Reproduction Assessment

Yes, from source inspection rather than a live OAuth run. deleteAccount writes deactivatedAt/purgedAt, the OAuth callback rejects that user on later GitHub sign-in, and tests assert the rejected reauth behavior.

## Solution Assessment

No current code path is the best final solution for the requested recovery. A direct purge or recreate change needs maintainer policy first so deleted, banned, legacy deletedAt, and GitHub identity-change cases are handled consistently.
