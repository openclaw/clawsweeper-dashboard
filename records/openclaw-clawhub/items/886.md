## Summary

Keep open. Current main still rejects deleted/deactivated ClawHub users during GitHub OAuth and active-session resolution, and the ClawHub cleanup policy keeps open issues in maintainer hands rather than auto-closing them.

## What This Changes

The reporter asks for account cleanup or re-registration after deleting a ClawHub account and then getting stuck in a GitHub OAuth sign-in loop.

## Best Possible Solution

Define one canonical deleted-account lifecycle policy, then implement the approved recovery or support behavior across the OAuth callback, deletion/admin tooling, UI messaging, docs, and regression tests.

## Reproduction Assessment

Yes. A deleted account record with deactivatedAt from deleteAccount will hit handleDeletedUserSignIn during GitHub OAuth and be suppressed by the active-user helpers afterward, which reproduces the authorize-then-still-logged-out behavior at the code level.

## Solution Assessment

Unclear. The requested purge may fix individual accounts, but the maintainable solution needs an explicit maintainer policy that separates self-deletion recovery from bans, audit history, and OAuth identity binding.
