## Summary

Keep open: current main still treats user-initiated account deletion as terminal during GitHub OAuth, and this ClawHub issue needs maintainer account-lifecycle/security policy rather than cleanup closure.

## What This Changes

The reporter asks for deleted ClawHub account cleanup or re-registration after GitHub OAuth returns to the sign-in flow.

## Best Possible Solution

Define one canonical deleted-account lifecycle and recovery policy, then implement the approved auth, support/admin tooling, user messaging, specs/docs, and regression tests together.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, handleDeletedUserSignIn rejects that user during OAuth, and active auth helpers return no session; I did not perform a live production deletion test.

## Solution Assessment

No. A one-off purge is not the best repo-level fix because recovery behavior needs an approved policy that preserves OAuth identity binding, deletion guarantees, and ban boundaries.
