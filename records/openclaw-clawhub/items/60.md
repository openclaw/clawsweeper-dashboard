## Summary

Keep open. Current main does not contain or clearly own the reported legacy `molt.bot` zsh installer or its `cleanup_npm_clawdbot_paths` function, while the maintained ClawHub install paths are separate; ClawHub cleanup policy therefore does not support closing this as implemented.

## What This Changes

The issue reports `curl -fsSL https://molt.bot/install.sh | zsh` failing during npm install because a zsh cleanup glob for legacy `.clawdbot-*` Node module paths had no matches.

## Best Possible Solution

Resolve the installer support boundary explicitly: if the legacy hosted installer is supported, make its source traceable and harden its zsh cleanup glob; if it is retired, redirect user-facing guidance to the maintained OpenClaw and ClawHub install commands with the supported Node range.
