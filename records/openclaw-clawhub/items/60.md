## What This Changes

The issue reports a curl-piped legacy `molt.bot` installer failing during npm install because zsh treats an unmatched Homebrew `.clawdbot-*` cleanup glob as an error.

## Best Possible Solution

Resolve the installer support boundary explicitly: if the legacy hosted installer remains supported, make its source traceable and harden the zsh cleanup glob for unmatched Homebrew npm paths and supported shells; if it is retired, update or redirect user-facing install guidance to maintained OpenClaw/ClawHub commands and state the supported Node range.
