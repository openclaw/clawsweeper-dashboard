The issue reports npm installation failing in a legacy `curl -fsSL https://molt.bot/install.sh | zsh` flow because zsh treats an unmatched `.clawdbot-*` Homebrew npm cleanup glob as an error.

## Best Possible Solution

Resolve the installer boundary explicitly: if the legacy hosted installer is supported, make its source traceable and harden the zsh glob cleanup; if retired, redirect users to maintained OpenClaw/ClawHub install commands and current Node guidance.

## Reproduction Assessment

## Solution Assessment
