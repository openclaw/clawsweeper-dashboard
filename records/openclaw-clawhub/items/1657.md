## Summary

Keep open. Current main does not implement dependency-registry phantom-package detection, and the PR is still an active security-sensitive implementation with blocking branch issues rather than work that has already landed elsewhere.

## What This Changes

This PR adds a Convex async dependency-registry scanner for bundled requirements/package/Cargo/pyproject files, stores scan/cache state, schedules publish-time scans, injects a DEP_NOT_FOUND static-scan finding, and tweaks line-zero rendering in the security scan UI.

## Best Possible Solution

Land this PR or a successor after regenerating committed Convex API types, treating any unresolved registry lookup as retryable/error unless confirmed missing packages already make the scan suspicious, making retry state queryable without a fixed in-memory window, and completing maintainer/security review against the linked dependency-confusion remediation plan.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Regenerate Convex API types for the new module:** `convex/lib/skillPublish.ts:336-338`
  - body: The branch adds `convex/depRegistryScan.ts` and calls `internal.depRegistryScan`, but the committed generated API types at the PR head still do not include a `depRegistryScan` module. With the generated types committed this way, TypeScript will reject these references during Convex/deploy typecheck. Run Convex codegen and commit the updated `_generated/api.d.ts` entry.
  - confidence: 0.94
- **[P1] Mark partial registry failures as retryable:** `convex/depRegistryScan.ts:508-518`
  - body: When a registry lookup exhausts retries, this code drops the dependency from `results` and only sets `abortedDueToErrors` after three consecutive failures. If one or two failures are interleaved with successful lookups, the scan can still write `status = "clean"` even though some dependencies were never checked, and the early skip then prevents a later retry. Track any unresolved lookup and return `error` unless confirmed missing packages already make the scan suspicious.
  - confidence: 0.91
- **[P2] Index retryable dependency scans:** `convex/depRegistryScan.ts:640-643`
  - body: This retry query only inspects the newest 500 `skillVersions` and filters `depRegistryAnalysis.status` in memory. Older error-state versions outside that window will never be scheduled for retry, so failed scans can remain stuck indefinitely. Store a denormalized status field and query it with an index, or use cursor-based pagination that cannot strand older errors.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff is a security-sensitive scanner and currently has concrete false-clean and stale-retry gaps that can leave dependency-confusion exposure unreported.

Concerns:

  - body: Retry-exhausted registry lookups are omitted from the result set unless three failures are consecutive, so a scan can report clean even though some dependencies were never verified against their registries.
  - confidence: 0.91
