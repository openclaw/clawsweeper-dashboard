## Summary

Keep open. Current main still lacks `clawhub update --dry-run`, and ClawHub cleanup policy keeps issues open unless they are resolved by a merged implementation or another explicit close reason applies.

## What This Changes

Add a `--dry-run` flag to `clawhub update` that previews pending skill updates without modifying installed files or lock state.

## Best Possible Solution

Add preview support inside the existing update resolution path, then document it and test that single-skill and `--all` dry-runs produce useful output without filesystem or lockfile writes.

## Reproduction Assessment

Not applicable as a bug reproduction. The feature gap is source-checkable: current main has no update `--dry-run` flag, no `dryRun` option in `cmdUpdate`, and no non-mutating preview branch.

## Solution Assessment

Yes. Extending the existing update command is the narrow maintainable direction because it can reuse the same registry, moderation, version, fingerprint, and local-change checks instead of adding a separate checker that could drift.
