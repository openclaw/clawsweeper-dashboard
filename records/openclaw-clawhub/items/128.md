## Summary

Keep open. Current main still lacks `--dry-run` support for `clawhub update`, and ClawHub cleanup policy keeps issues open for implementation rather than auto-closing them.

## What This Changes

Add a `--dry-run` option to `clawhub update` that previews available updates without changing installed skills.

## Best Possible Solution

Add a planning path inside `cmdUpdate` that reuses existing registry, moderation, fingerprint, version-resolution, and semver checks, then exits before directory, download, origin metadata, or lockfile writes.

## Reproduction Assessment

Yes. Source inspection against current main is enough: the CLI does not register `--dry-run`, `cmdUpdate` has no dry-run option, and the implementation reaches mutating update work.

## Solution Assessment

Yes. Extending the existing update command with a no-mutation planning branch, focused tests, docs, and changelog coverage is the narrow maintainable solution.

## Repair Work Prompt

Implement `clawhub update --dry-run` for this issue. Add the flag to `packages/clawhub/src/cli.ts` and thread a `dryRun` option into `cmdUpdate` in `packages/clawhub/src/cli/commands/skills.ts`. The dry-run path should reuse the existing registry lookup, moderation checks, local fingerprinting, version resolution, semver comparison, and `--all`/single-skill selection logic, but must exit before `rm`, `downloadZip`, `extractZipToDir`, `writeSkillOrigin`, or `writeLockfile` mutate disk. Print clear preview output for would-update, already up-to-date, and skipped/error states; define behavior for fingerprint-based lockfile reconciliation without writing it. Add focused tests in `packages/clawhub/src/cli/commands/skills.test.ts` proving preview output and no mutation calls. Update `docs/cli.md` and add a user-facing `CHANGELOG.md` entry. Do not change package publish or sync dry-run behavior.
