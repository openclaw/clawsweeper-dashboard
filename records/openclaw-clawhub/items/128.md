Keep open. Current main still has no `clawhub update --dry-run` flag or preview path, and the live issue is explicitly labeled as needing a product decision, so this feature request should stay with maintainers instead of cleanup closure.

## What This Changes

The issue requests a `--dry-run` flag for `clawhub update` that previews pending skill updates without mutating installed skills or lock state.

## Best Possible Solution

Define the dry-run contract, then add a preview path inside `cmdUpdate` that reuses existing registry, moderation, fingerprint, semver, pin, and local-change checks and exits before any directory, origin metadata, or lockfile write occurs.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request for a new CLI flag. Source inspection gives high confidence that current main has no update dry-run flag, option plumbing, docs, or tests.

## Solution Assessment

Unclear until maintainers choose the dry-run contract. If accepted, extending `cmdUpdate` is the narrowest maintainable path because that command already owns update eligibility, moderation checks, fingerprint handling, pin behavior, and writes.
