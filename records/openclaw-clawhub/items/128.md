## Summary

Keep open. Current `main` still lacks `clawhub update --dry-run`, the update path still performs file/origin/lockfile writes after resolving an update, and ClawHub cleanup policy keeps issues open for maintainer follow-up rather than auto-closing them.

## What This Changes

Add a `--dry-run` mode to `clawhub update` that previews available updates without changing installed skill files or lock state.

## Best Possible Solution

Add a planning path inside `cmdUpdate` that reuses existing registry, moderation, fingerprint, version, pin, and semver checks, prints would-update/up-to-date/skipped output, and exits before any directory, origin metadata, or lockfile write.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current `main` has no update dry-run flag, option plumbing, preview output, or no-write branch.

## Solution Assessment

Yes. Extending `cmdUpdate` is the narrowest maintainable solution because that command already owns the update checks that dry-run must preview without duplicating registry, moderation, fingerprint, pin, or semver logic elsewhere.
