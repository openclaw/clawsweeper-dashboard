## Summary

Keep open. Current main still lacks `clawhub update --dry-run`, and ClawHub cleanup policy keeps issues open unless a merged implementation or another explicit close reason applies.

## What This Changes

The issue requests a `--dry-run` flag for `clawhub update` that previews available updates without changing installed skills or lockfiles.

## Best Possible Solution

Add a non-mutating planning path inside the existing update resolution flow, document the new flag, and cover single-skill plus `--all` dry-runs with focused no-write tests.

## Reproduction Assessment

Not applicable as a bug reproduction. The feature gap is source-checkable: current main has no update `--dry-run` flag, no `dryRun` option in `cmdUpdate`, and no preview-only branch before filesystem and lockfile writes.

## Solution Assessment

Yes. Extending the existing update command is the narrow maintainable direction because it can reuse the same registry, moderation, fingerprint, version, and local-change checks instead of adding a separate checker that could drift.

## Repair Work Prompt

Implement `clawhub update --dry-run` as a focused CLI feature. Add the Commander option in `packages/clawhub/src/cli.ts`, extend `cmdUpdate` options in `packages/clawhub/src/cli/commands/skills.ts`, and reuse the existing registry, moderation, fingerprint, version-resolution, pinned-skill, local-change, and `--version` checks. In dry-run mode, print a clear plan for single-skill and `--all` updates, including would-update, already-up-to-date, skipped pinned, suspicious/malware-blocked, not-found, and local-change cases as applicable. Do not call `rm`, `downloadZip`, `extractZipToDir`, `writeSkillOrigin`, or `writeLockfile`, and do not mutate the in-memory lock entry for fingerprint reconciliation when `dryRun` is true. Add focused tests in `packages/clawhub/src/cli/commands/skills.test.ts` proving dry-run output and absence of filesystem/lockfile writes, then update `docs/cli.md` and any maintained CLI README examples. Use existing package publish/sync dry-run wording as a style reference, but do not change those commands.
