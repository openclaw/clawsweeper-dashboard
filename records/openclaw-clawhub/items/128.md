## Summary

Keep open. Current main still lacks `clawhub update --dry-run`, and the issue is a narrow CLI feature request with no merged or open superseding implementation found.

## What This Changes

Add a `--dry-run` flag to `clawhub update` so users can preview would-update and up-to-date results without changing installed skill files or lock state.

## Best Possible Solution

Add a no-write planning path inside `cmdUpdate` that reuses existing registry, moderation, fingerprint, semver, pin, and version checks, documents the flag, and tests both preview output and absence of writes.

## Solution Assessment

Yes. Extending `cmdUpdate` is the narrowest maintainable path because that command already performs the checks a dry-run preview must report without repeating update logic elsewhere.
