## Summary

Keep open. Current main still lacks `clawhub update --dry-run`, the existing update flow remains mutating, and ClawHub cleanup policy keeps issues open unless a specific close rule applies.

## What This Changes

Requested behavior is a `clawhub update --dry-run` preview mode that reports available updates without modifying installed skills or lock state.

## Best Possible Solution

Add a preview branch inside the existing update flow that reuses registry lookup, moderation checks, fingerprints, semver resolution, and pin handling, then exits before downloads, directory changes, origin metadata writes, or lockfile writes.

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main has no update dry-run flag, option plumbing, or no-write branch.

## Solution Assessment

Yes. Extending the existing `clawhub update` path is the narrow maintainable solution because it can reuse the current registry, fingerprint, moderation, pin, and version-resolution logic instead of duplicating a separate checker.
