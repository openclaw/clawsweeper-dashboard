## Summary

Keep open. Current main still lacks a `--dry-run` flag and preview-only path for `clawhub update`; this is a narrow CLI feature request, not an implemented or duplicate issue.

## What This Changes

Add a `--dry-run` flag to `clawhub update` that previews pending skill updates without modifying installed files or the lockfile.

## Best Possible Solution

Add a no-mutation planning path to the existing update command, document the flag, add a changelog entry, and cover single-skill plus `--all` dry-run behavior in focused CLI tests.

## Solution Assessment

## Repair Work Prompt

Implement `clawhub update --dry-run` for openclaw/clawhub. Add the flag in `packages/clawhub/src/cli.ts`, extend `cmdUpdate` options in `packages/clawhub/src/cli/commands/skills.ts`, and reuse the existing registry, moderation, local fingerprint, version resolution, and semver checks to print a preview for single-skill and `--all` updates. Dry-run must not call `rm`, download/extract zips, write origin metadata, mutate the lockfile, or prompt; it should report would-update, up-to-date/already-at, skipped local changes, suspicious/malware-blocked, and not-found states clearly. Add focused tests in `packages/clawhub/src/cli/commands/skills.test.ts`, update `docs/cli.md`, and add a user-facing `CHANGELOG.md` entry. Do not change unrelated install, uninstall, sync, or package publish behavior.
