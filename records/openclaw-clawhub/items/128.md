## Summary

Keep open. Current main still lacks a `--dry-run` mode for `clawhub update`; the command exposes only `--all`, `--version`, and `--force`, while `cmdUpdate` still proceeds into directory removal, download/extract, origin metadata writes, and lockfile writes.

## What This Changes

Add a `--dry-run` flag to `clawhub update` so users can preview single-skill or `--all` updates without changing local skill files or lock state.

## Best Possible Solution

Add a planning path to `cmdUpdate` that reuses existing registry, moderation, local fingerprint, version resolution, and semver checks, prints clear preview output, and exits before any file, origin metadata, or lockfile write occurs.

## Reproduction Assessment

Yes. Source inspection on current main is enough to reproduce the missing feature: `clawhub update --dry-run` has no registered flag or `cmdUpdate` option, and the implementation continues into mutating update work.

## Solution Assessment

Yes. The narrowest maintainable fix is to add dry-run handling inside the existing update command flow rather than creating a separate command, because it can reuse the same resolution and safety checks while short-circuiting before mutation.
