## Summary

Keep open. Current main still does not implement `clawhub update --dry-run`, and ClawHub cleanup policy says issues should remain open unless a specific close rule applies; this is a valid, bounded feature request rather than an implemented bug fix.

## What This Changes

Add a `--dry-run` option to `clawhub update` so users can preview available skill updates without changing installed files, origin metadata, or the lockfile.

## Best Possible Solution

Add a planning branch inside the existing update flow that reuses current registry, version, fingerprint, moderation, and local-change checks, then exits before downloads, directory writes, origin metadata writes, or lockfile writes.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature gap. Source inspection gives high confidence that current main has no update `--dry-run` flag, option plumbing, or no-write branch.

## Solution Assessment

Yes. Extending `clawhub update` is the narrow maintainable direction because it can reuse existing update checks rather than adding a separate checker that could drift.
