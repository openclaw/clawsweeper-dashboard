## Summary

Keep open. Current main still does not expose `clawhub update --dry-run`, and this is a valid new CLI feature request rather than a broken existing behavior; ClawHub cleanup policy keeps issues open rather than auto-closing them.

## What This Changes

Add a `--dry-run` mode to `clawhub update` that previews available skill updates without changing local files or lock state.

## Best Possible Solution

Add preview support inside the existing update resolution path so it reuses registry lookup, moderation checks, version selection, local fingerprint checks, and pinned-skill handling, then exits before directory, metadata, or lockfile writes.

## Reproduction Assessment

Yes for the feature gap: current main has no registered `--dry-run` flag for `clawhub update`, no `dryRun` option in `cmdUpdate`, and no preview-only branch before mutating update work. This is a new CLI mode request rather than a broken existing contract.

## Solution Assessment

Yes. Extending the existing update command is the narrow maintainable direction because the preview should share the real update resolver and safety checks instead of adding a separate checker with divergent behavior.
