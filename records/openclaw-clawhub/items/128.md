Keep open. Current main still lacks `clawhub update --dry-run`, the implementation still proceeds into update writes, and the latest issue state includes an explicit product-decision label, so this should stay open for maintainer direction rather than cleanup closure.

## What This Changes

Request to add a `--dry-run` flag to `clawhub update` that previews available skill updates without mutating installed files or lock state.

## Best Possible Solution

Define the dry-run contract, then add a preview path inside `cmdUpdate` that reuses existing update eligibility checks and exits before directory, origin metadata, or lockfile writes.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request for a new CLI flag. Source inspection gives high confidence that current main has no update dry-run flag, option plumbing, docs, or tests.

## Solution Assessment

Unclear until maintainers choose the dry-run contract. If accepted, extending `cmdUpdate` is the narrowest maintainable path because that command already owns version resolution, moderation checks, fingerprint handling, pin behavior, and writes.
