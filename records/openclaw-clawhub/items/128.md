## Summary

Keep open. Current main still lacks `--dry-run` on `clawhub update`, and this ClawHub cleanup policy keeps issues open rather than auto-closing them.

## What This Changes

Add a `--dry-run` flag to `clawhub update` that previews pending and up-to-date skill updates without mutating installed files or lockfiles.

## Best Possible Solution

Add a no-mutation planning path to the existing update command, document the flag, add a changelog entry, and cover single-skill plus `--all` dry-run behavior in focused CLI tests.

## Reproduction Assessment

Not applicable as a bug reproduction, but the feature gap is source-confirmed: current main has no `--dry-run` flag or preview-only branch for `clawhub update`.

## Solution Assessment

Yes. Adding a `--dry-run` option to the existing update command and returning before directory, download, origin metadata, and lockfile writes is the narrow maintainable solution.

## Repair Work Prompt

Implement `clawhub update --dry-run` for the packages/clawhub CLI. Register the flag in `packages/clawhub/src/cli.ts` and extend `cmdUpdate` in `packages/clawhub/src/cli/commands/skills.ts` to support a preview path for both single-skill and `--all` modes. Reuse the existing registry lookup, moderation checks, local fingerprint resolution, version validation, and semver comparison, then print clear would-update/up-to-date/skipped/not-found output and return before `rm`, `downloadZip`, `extractZipToDir`, `writeSkillOrigin`, or `writeLockfile`. Preserve existing behavior when dry-run is false. Add focused tests in `packages/clawhub/src/cli/commands/skills.test.ts` proving preview output and no mutation calls for update-needed and already-up-to-date cases. Update `docs/cli.md`, `packages/clawhub/README.md` if needed, and `CHANGELOG.md` for the user-facing feature. Do not change package publish or sync dry-run behavior.
