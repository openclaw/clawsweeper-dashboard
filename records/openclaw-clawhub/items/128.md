## Summary

Keep open. Current main still lacks `--dry-run` for `clawhub update`, and the command still proceeds into update mutations; this is a valid narrow CLI feature, but ClawHub policy keeps issues open rather than auto-closing them.

## What This Changes

Add a `--dry-run` flag to `clawhub update` that previews available updates and up-to-date skills without changing local skill files or lock metadata.

## Best Possible Solution

Add a planning path in `cmdUpdate` that reuses the existing registry, moderation, fingerprint, version-resolution, and semver checks, then exits before directory, download, origin metadata, or lockfile writes while documenting and testing the flag.

## Reproduction Assessment

Yes. Source inspection of current main is enough: the CLI exposes no `--dry-run` option for `update`, and `cmdUpdate` has no dry-run branch before its mutating work.

## Solution Assessment

Yes. Extending the existing update command with a preview-only branch, plus focused docs and no-mutation tests, is the narrow maintainable fix for this request.

## Repair Work Prompt

Implement `clawhub update --dry-run` for openclaw/clawhub. Add the flag wiring in `packages/clawhub/src/cli.ts`, extend `cmdUpdate` in `packages/clawhub/src/cli/commands/skills.ts`, and keep existing non-dry-run behavior unchanged. The dry-run path should reuse the current registry lookup, moderation checks, local fingerprint resolution, version selection, and semver/up-to-date checks, then print clear preview output for single-skill and `--all` modes. It must not call `rm`, `downloadZip`, `extractZipToDir`, `writeSkillOrigin`, or `writeLockfile`, and it should not prompt in preview mode. Define output for would-update, up-to-date, skipped local changes, suspicious, malware-blocked, and not-found states. Update `docs/cli.md` and add focused tests in `packages/clawhub/src/cli/commands/skills.test.ts` proving preview output and no mutation. Do not change unrelated package publish or sync dry-run behavior, and do not add repo-wide refactors.
