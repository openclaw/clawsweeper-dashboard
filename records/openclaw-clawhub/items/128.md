## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still lacks an update dry-run mode in the CLI implementation, docs, and focused tests.

## What This Changes

Add a `--dry-run` mode to `clawhub update` so users can preview planned skill updates without changing installed files or lock state.

## Best Possible Solution

Add preview support inside the existing update flow, reuse the same resolution and safety checks as real updates, report would-update/up-to-date/skipped states, and return before any directory, download, metadata, or lockfile write.

## Reproduction Assessment

Yes for the feature gap: source inspection shows no `--dry-run` option registered for `clawhub update`, no `dryRun` option accepted by `cmdUpdate`, and no preview-only branch before mutating update work. This is a requested feature rather than a broken existing contract.

## Solution Assessment

Yes. Extending the existing update command is the narrow maintainable path because it can reuse the same registry lookup, moderation, fingerprint, version, and local-change checks instead of duplicating update logic.
