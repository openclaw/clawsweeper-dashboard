## Summary

Keep open. This is a feature request issue, current main still lacks `clawhub update --dry-run`, and the only related dry-run PR was for `package publish`, not the skill update command.

## What This Changes

Add a `--dry-run` flag to `clawhub update` so users can preview available updates without changing installed skill files, origins, or lockfiles.

## Best Possible Solution

Add a planning path inside the existing update flow that reuses registry lookup, moderation checks, fingerprints, semver resolution, and pin handling, then exits before downloads, directory writes, origin metadata writes, or lockfile writes.
