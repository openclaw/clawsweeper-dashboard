## Summary

Keep open. Current main still lacks `clawhub update --dry-run`; the CLI wiring, `cmdUpdate` option shape, docs, and focused tests all show only the existing mutating update path.

## What This Changes

Add a `--dry-run` flag to `clawhub update` that previews available skill updates without changing installed files, origin metadata, or the lockfile.

## Best Possible Solution

Implement a preview branch in the existing update flow that reuses current registry, moderation, fingerprint, version, and local-change checks, prints planned states, and returns before any file, origin metadata, or lockfile writes.

## Reproduction Assessment

Yes for the feature gap: source inspection on current main shows no `--dry-run` option on `update` and no `dryRun` input or preview-only branch in `cmdUpdate`. This is a requested feature, not a broken existing CLI contract.

## Solution Assessment

Yes. Extending the existing update command with a `dryRun` option and focused no-mutation tests is the narrow maintainable path; a separate command or manual `explore` loop would duplicate update resolution behavior.
