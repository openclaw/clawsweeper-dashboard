## Summary

Keep open. Current main still lacks `clawhub update --dry-run`; the update command only exposes the existing mutating flags, and `cmdUpdate` still reaches directory replacement, origin metadata writes, lockfile mutation, and lockfile persistence.

## What This Changes

Add a `--dry-run` flag to `clawhub update` so users can preview available skill updates without changing installed files or lockfile state.

## Best Possible Solution

Add a planning path inside `cmdUpdate` that reuses the existing registry, moderation, local fingerprint, version resolution, semver, and pin checks, reports would-update/up-to-date/skipped results, and exits before any file, origin metadata, or lockfile write.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main has no update dry-run flag, option plumbing, or no-write branch.

## Solution Assessment

Yes. Extending `cmdUpdate` is the narrowest maintainable direction because that command already performs the lookup, moderation, fingerprint, version, and pin checks needed for a preview.
