## Summary

Keep open. Current main still lacks `clawhub update --dry-run`; the update command only exposes mutating flags, and `cmdUpdate` still reaches directory replacement, origin metadata writes, and lockfile persistence.

## What This Changes

Requested a `clawhub update --dry-run` mode that previews available skill updates without changing installed files or lock state.

## Best Possible Solution

Add a no-write planning path inside `cmdUpdate` that reuses the existing update checks and reports would-update, up-to-date, and skipped results for single-skill and `--all` modes.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main has no update dry-run flag, option plumbing, or no-write branch.

## Solution Assessment

Yes. Extending `cmdUpdate` is the narrowest maintainable solution because that command already performs the registry, moderation, fingerprint, semver, pin, and version checks needed for a preview.
