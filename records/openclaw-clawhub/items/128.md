## Summary

Keep open. Current main still lacks `clawhub update --dry-run`, and ClawHub cleanup policy says issues should stay open unless there is a merged implementation or another explicit close reason.

## What This Changes

Request adds a `--dry-run` flag to `clawhub update` so users can preview single-skill and `--all` updates without changing installed files or lockfiles.

## Best Possible Solution

Add a no-write planning branch inside the existing update flow that reuses registry, moderation, fingerprint, version, and semver checks, then prints would-update/up-to-date/skipped output for single and `--all` modes.

## Reproduction Assessment

Not applicable as a bug reproduction. The feature gap is source-checkable: current main has no update `--dry-run` flag, no `dryRun` option in `cmdUpdate`, and no preview-only branch before file/origin/lockfile writes.

## Solution Assessment

Yes. Extending the existing update command is the narrow maintainable direction because it can reuse current registry, moderation, fingerprint, version, and local-change checks instead of adding a separate checker that could drift.
