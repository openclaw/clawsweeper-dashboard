## Summary

Keep open. Current main still lacks `clawhub update --dry-run`, and ClawHub cleanup policy keeps issues open unless they are resolved by a merged PR or otherwise explicitly closeable.

## What This Changes

The issue requests a `--dry-run` mode for `clawhub update` that previews available skill updates without changing installed files or lockfiles.

## Best Possible Solution

Add preview support to the existing update flow so it reuses real registry, moderation, fingerprint, version, and local-change checks, then exits before any filesystem or lockfile writes.

## Reproduction Assessment

Not applicable as a bug reproduction, but the feature gap is source-checkable: current main has no update `--dry-run` flag, no `dryRun` option in `cmdUpdate`, and no non-mutating preview branch.

## Solution Assessment

Yes. Extending the existing update command is the narrow maintainable solution because it can share the same resolution and safety checks instead of adding a separate checker that could drift.
