Keep open. Current main still lacks `clawhub update --dry-run`, the update command has no dry-run option or preview-only branch, there is no linked closing PR, and the ClawHub cleanup policy keeps issues open for maintainer follow-up.

## What This Changes

Requested addition of a `--dry-run` flag to `clawhub update` that previews available skill updates without mutating installed skills or lock state.

## Best Possible Solution

Add a preview path inside `cmdUpdate` that reuses existing registry, moderation, fingerprint, version, pin, and semver checks, prints would-update/up-to-date/skipped output, and exits before any file, origin metadata, or lockfile write.

## Solution Assessment

Yes. Extending `cmdUpdate` is the narrowest maintainable solution because that command already owns update eligibility, version resolution, local fingerprint checks, pin handling, moderation checks, and writes.
