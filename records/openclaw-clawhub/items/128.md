## Summary

Keep open. Current main still lacks `--dry-run` for `clawhub update`, and ClawHub policy keeps issues open rather than auto-closing them even when the request is well scoped.

## What This Changes

Add a `--dry-run` flag to `clawhub update` so users can preview would-update and up-to-date results without changing installed skills.

## Best Possible Solution

Implement a no-mutation planning branch in the existing update command, document it, and cover both single-skill and `--all` behavior with focused CLI tests.

## Reproduction Assessment

Yes for the current gap: source inspection on current main shows no `--dry-run` flag on `clawhub update` and no preview-only branch in `cmdUpdate`. This is a feature request rather than a broken existing contract.

## Solution Assessment

Yes. Adding a `--dry-run` option to the existing update command, reusing the existing version/fingerprint/moderation checks and returning before writes, is the narrow maintainable solution.
