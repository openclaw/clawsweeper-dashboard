## Summary

Keep open. Current main still lacks `clawhub update --dry-run`: the CLI does not register the flag, `cmdUpdate` has no `dryRun` option, and the update path still proceeds to file, origin metadata, and lockfile writes. ClawHub cleanup policy keeps issues open unless a specific close rule applies.

## What This Changes

Request adding a `--dry-run` flag to `clawhub update` that reports pending skill updates without changing installed files or lock state.

## Best Possible Solution

Add a no-write planning branch to the existing update flow that reuses registry, moderation, fingerprint, version, and semver checks, then document and test single-skill and `--all` dry-run behavior.

## Reproduction Assessment

Not applicable as a bug reproduction. The feature gap is source-checkable: current main has no update `--dry-run` flag, no `dryRun` option in `cmdUpdate`, and no preview-only branch before writes.

## Solution Assessment

Yes. Extending the existing update command is the narrow maintainable direction because it can reuse the current registry, moderation, fingerprint, version, and local-change checks instead of adding a separate checker that could drift.
