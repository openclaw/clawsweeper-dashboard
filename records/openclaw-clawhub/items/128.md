## Summary

Keep open. Current main still lacks a `--dry-run` mode for `clawhub update`; the command only exposes `--all`, `--version`, and `--force`, and the implementation continues into directory removal, download/extract, origin metadata writes, and lockfile writes. The request is narrow and valid, but repository policy keeps ClawHub issues open rather than auto-closing them.

## What This Changes

The issue requests a `--dry-run` flag for `clawhub update` that reports available updates without modifying installed skills or lock state.

## Best Possible Solution

Add a planning path for `clawhub update` that reuses the existing registry, moderation, local fingerprint, version resolution, and semver checks, then prints clear would-update/up-to-date/skipped output and exits before any directory, download, origin metadata, or lockfile write occurs. Document the flag and cover no-mutation behavior in focused CLI tests.
