## Summary

Keep open. Current main still lacks `clawhub update --dry-run`; the CLI has no flag, `cmdUpdate` has no dry-run option, and the update path still performs directory, origin metadata, and lockfile writes.

## What This Changes

The issue requests a `--dry-run` flag for `clawhub update` that previews available updates without modifying installed skills or lockfiles.

## Best Possible Solution

Add a planning path to the existing update command that reports would-update/up-to-date/skipped states and exits before any directory, download, origin metadata, or lockfile write occurs.

## Reproduction Assessment

Yes for the feature gap: source inspection on current main shows no `--dry-run` flag on `clawhub update` and no preview-only branch in `cmdUpdate`. This is a requested feature, not a broken existing CLI contract.

## Solution Assessment

Yes. Extending the existing update command with a dry-run option that reuses current resolution and safety checks, then returns before mutating operations, is the narrow maintainable solution.
