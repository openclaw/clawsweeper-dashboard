## Summary

Keep open. Current main still lacks `--dry-run` for `clawhub update`, and the command still reaches mutating update work; for ClawHub issues, this should remain open for implementation rather than auto-close.

## What This Changes

Add a preview-only `--dry-run` flag to `clawhub update` so users can see pending skill updates without changing local state.

## Best Possible Solution

Add a preview branch in `cmdUpdate` that reuses existing registry, moderation, fingerprint, version-resolution, and semver checks, then exits before directory, download, origin metadata, or lockfile writes.

## Reproduction Assessment

Yes. Source inspection against current main is enough: `clawhub update` exposes no `--dry-run` flag, and `cmdUpdate` has no preview-only branch before mutating work.

## Solution Assessment

Yes. Extending the existing update command with a no-mutation planning path, focused tests, docs, and changelog coverage is the narrow maintainable solution.

Update the Commander wiring in `packages/clawhub/src/cli.ts`, extend `cmdUpdate` options in `packages/clawhub/src/cli/commands/skills.ts`, and reuse the existing registry, moderation, local fingerprint, version resolution, and semver logic.
