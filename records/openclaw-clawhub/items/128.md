## Summary

Keep open. Current main still lacks `--dry-run` on `clawhub update`, and the update path still performs file and lockfile mutations; ClawHub policy also keeps issues open rather than auto-closing implemented or closeable issue work.

## What This Changes

The issue requests a `--dry-run` flag for `clawhub update` that previews available skill updates without changing installed files or lockfiles.

## Best Possible Solution

Add a no-mutation planning path to the existing update command, document it, add regression tests for single-skill and `--all`, and keep package publish/sync dry-run behavior unchanged.

## Reproduction Assessment

Yes for the current feature gap: source inspection on current main shows no `--dry-run` flag on `clawhub update` and no preview-only branch in `cmdUpdate`. This is a feature request rather than a broken existing contract.

## Solution Assessment

Yes. Adding a `--dry-run` option to the existing update command, reusing the existing metadata, moderation, fingerprint, and version-resolution logic, then returning before writes is the narrow maintainable solution.
