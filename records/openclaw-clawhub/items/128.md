## Summary

Keep open. Current main still exposes `clawhub update` without a preview flag and the implementation continues into file and lockfile writes, so the requested behavior remains absent.

## What This Changes

Add a `--dry-run` flag to `clawhub update` so users can preview available skill updates without changing installed files or the lockfile.

## Best Possible Solution

Add a no-mutation planning path to the existing update command, document the flag, add an Unreleased changelog entry, and cover single-skill plus `--all` dry-run behavior in focused CLI tests.

## Reproduction Assessment

Yes for the feature gap: source inspection on current main shows no `--dry-run` flag on `clawhub update` and no preview-only branch in `cmdUpdate`. This is a requested feature, not a broken existing CLI contract.

## Solution Assessment

Yes. Extending the existing update command with a dry-run option that returns before directory, download, origin metadata, and lockfile writes is the narrow maintainable path.
