## Summary

Keep open. Current main still lacks `clawhub update --dry-run`, and ClawHub cleanup policy keeps issues open unless they are resolved by maintainer action.

## What This Changes

The issue requests a `--dry-run` flag for `clawhub update` so users can preview skill updates without changing installed files, origin metadata, or lock state.

## Best Possible Solution

Add a preview branch to the existing update flow that reuses current registry, moderation, fingerprint, version, and local-change checks, reports planned outcomes, and returns before any filesystem or lockfile writes.

## Reproduction Assessment

Yes for the feature gap: current source shows no `--dry-run` flag on `update`, no `dryRun` field in `cmdUpdate`, and no preview-only branch before the mutating update path. This is a requested feature rather than a broken existing contract.

## Solution Assessment

Yes. Extending the existing update command is the narrow maintainable path because it can reuse the same resolution and safety checks; a separate command would duplicate update logic.
