Full review comments:

- **[P1] Reuse the package download helper for version rows:** `src/components/PluginVersionsPanel.tsx:29-33`
  - body: This new helper rebuilds package download URLs and falls back to `https://clawhub.ai`, while the rest of the plugin route uses `getPackageDownloadPath()` for same-origin encoded API paths. In local, preview, or non-production deployments, the version-row download buttons can leave the current deployment and hit production, so this should reuse the existing helper instead of adding a parallel source of truth.
  - confidence: 0.88
- **[P1] Remove the invalid owner id from the test fixture:** `src/__tests__/package-detail-route.test.tsx:1013`
  - body: `ci:types-build` fails because this fixture adds `_id` to the plugin detail owner shape, but that type only accepts the public owner fields used by the route. The branch cannot pass the required type/build gate until this fixture matches the typed loader data.
  - confidence: 0.95
- **[P2] Render loaded empty files:** `src/components/SkillFilesPanel.tsx:270-274`
  - body: A successful zero-byte file load stores `fileContent` as an empty string, but this branch renders the code block only when `fileContent` is truthy. Selecting an empty file therefore clears loading/error state and leaves a blank viewer body instead of showing an empty loaded file state.
  - confidence: 0.9
- **[P2] Update the activity label tooltip test contract:** `src/components/ActivityMetricLabel.tsx:14-20`
  - body: The component now always renders the `About activity counts` button, but the existing unit test still asserts that button is absent, so `ci:unit` fails. Either make the tooltip conditional if that was intended or update the test to assert the new warning behavior.
  - confidence: 0.92
- **[P2] Avoid the brittle raw CSS contract assertion:** `src/__tests__/ui-design-contract.test.ts:282-284`
  - body: The updated design contract now searches for an exact raw stylesheet substring, and the current stylesheet does not contain that formatting, so `ci:unit` fails. Keep the contract tied to the parsed selector or another stable assertion rather than raw CSS whitespace.
  - confidence: 0.86
