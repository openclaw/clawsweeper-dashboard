## Summary

Keep open for maintainer product scoping. Current main has adjacent skill metadata, capability tags, engagement-based recommended sorting, and native CLI pinning, but it still does not implement inter-skill conflict metadata, task-context recommendations, or a runtime skill-overload policy.

## What This Changes

The issue asks how ClawHub/OpenClaw should handle skill overload, skill collisions, task-context recommendations, conflict metadata, and production version pinning.

## Best Possible Solution

Split this into maintainer-approved narrow work: publish measured loading guidance, keep CLI pinning as the version-lock path, and design any conflict metadata or task-context recommendation surface with explicit schema, API, UI, and runtime ownership.
