## Summary

Keep open for maintainer product scoping. Current main covers adjacent discovery, capability filtering, versioned installs, and native pinning, but it still does not provide task-context skill recommendations, inter-skill dependency/conflict metadata, or measured guidance for runtime skill overload.

## What This Changes

The issue asks ClawHub to address skill overload through guidance, task-context recommendations, dependency/conflict metadata, and production version-pinning practices.

## Best Possible Solution

Keep the existing search, capability-tag, versioning, and pinning behavior, then write a scoped product/spec decision for measured overload guidance, optional conflict metadata, and task-context recommendations before implementation.

## Reproduction Assessment

Not applicable: this is a product/design request based on operator experience, not a concrete failing current-main bug. Source review can verify which requested capabilities exist, but there is no stable reproduction threshold for skill overload.

## Solution Assessment

No single proposal is ready as the best implementation path. Guidance, dependency/conflict metadata, task-context recommendation, and update policy have separate ownership and compatibility contracts that maintainers should scope independently.
