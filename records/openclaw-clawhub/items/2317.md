## Summary

The requested consolidation remains necessary: current main and v0.20.2 still mix top-level skill operations with a singular `skill` subgroup. This collaborator-authored feature needs maintainer judgment on naming and backward compatibility, and no canonical replacement issue or PR currently owns the work.

## What This Changes

Move most skill-specific top-level CLI commands beneath a consolidated `clawhub skills` command group now that ClawHub also supports packages and plugins.

## Best Possible Solution

Agree on a consistent skill/package namespace, introduce grouped skill commands while preserving top-level aliases with clear deprecation messaging, and migrate help text, documentation, and CLI tests before removing established commands.

## Reproduction Assessment

Yes. Current-main command registrations and public documentation directly reproduce the mixed top-level and grouped skill-command hierarchy described by the issue.

## Solution Assessment

No, not as an unconditional command move. The maintainable approach requires a maintainer-approved namespace plus a backward-compatible alias and deprecation plan.
