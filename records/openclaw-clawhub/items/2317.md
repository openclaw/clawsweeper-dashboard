## Summary

Current release `v0.21.0` still uses the mixed top-level and grouped command model, so the requested consolidation remains necessary. This collaborator-authored CLI design proposal should stay open for an explicit command-taxonomy and backward-compatibility decision.

## What This Changes

Consolidate skill-oriented top-level CLI commands under a `clawhub skills` command group now that the CLI also manages plugins/packages.

## Best Possible Solution

Define one canonical skill/package command taxonomy, preserve existing top-level commands as compatibility aliases through a documented deprecation window, and update help output, tests, and user documentation together.

## Reproduction Assessment

Yes for the reported layout: the current released source directly registers many skill commands at top level beside `skill` and `package` groups. This is a source-verifiable feature request rather than a runtime bug.

## Solution Assessment

Unclear. Consolidation is a reasonable direction, but singular/plural naming, overlap with package families, and the backward-compatible deprecation contract require maintainer approval.
