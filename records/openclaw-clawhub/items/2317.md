## Summary

Current `main` and v0.20.2 still use a mixed hierarchy of top-level skill commands plus `skill` and `package` groups, so the requested consolidation remains unresolved. This collaborator-authored feature needs an explicit maintainer decision on naming and backward compatibility rather than automated cleanup.

## What This Changes

Consolidate most skill-oriented top-level CLI commands under a dedicated `clawhub skills` command group now that the CLI also supports plugins and packages.

## Best Possible Solution

Choose one canonical skill namespace, add the grouped commands while preserving existing top-level paths as compatibility aliases for a documented transition, and update help output, CLI artifact coverage, public docs, and the relevant design spec together.

## Reproduction Assessment

Yes. Current-main command registration and README examples directly demonstrate the mixed top-level and grouped hierarchy; this is a product-interface request rather than a runtime failure.

## Solution Assessment

Unclear. Grouping skill commands is directionally maintainable, but maintainers must first decide singular versus plural naming, alias coverage, and the deprecation contract.
