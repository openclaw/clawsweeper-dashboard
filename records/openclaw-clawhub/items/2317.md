## Summary

The requested CLI consolidation is not implemented on current main: skill lifecycle and discovery commands remain top-level while publishing and package operations use separate groups. The issue should remain open because it requires an explicit compatibility and command-taxonomy decision, and its collaborator author also requires maintainer handling rather than automated closure.

## What This Changes

Consolidate most skill-specific top-level ClawHub CLI commands beneath a skill namespace now that the CLI also supports plugin/package commands.

## Best Possible Solution

Choose one stable skill namespace, add grouped forms for the selected commands, preserve existing top-level forms as documented compatibility aliases for a deliberate deprecation window, and update help output, CLI tests, and public docs together.

## Reproduction Assessment

Yes. Current-main command registration and published CLI documentation directly reproduce the mixed top-level and grouped skill command structure described by the issue.

## Solution Assessment

The consolidation direction is maintainable, but the issue alone does not resolve the singular/plural namespace, command boundaries, alias lifetime, or deprecation behavior required for a safe implementation.
