## Summary

Keep open: current main and latest v0.21.0 still expose the skill command family mostly at the root `clawhub` level, and this collaborator-authored ergonomics change needs maintainer naming and compatibility decisions before implementation.

## What This Changes

The issue requests consolidating most skill-related top-level `clawhub` CLI commands under a skills namespace now that plugin/package commands also exist.

## Best Possible Solution

Define the stable skill namespace and compatibility policy, then update CLI registration, docs, and CLI tests so grouped skill commands exist while intentional root aliases remain or deprecate on a maintainer-approved schedule.

## Reproduction Assessment

Yes for the current CLI shape: source and docs inspection show skill commands still registered and documented at the root level while package/plugin commands are grouped. This is not a runtime bug reproduction, but the reported command-surface mismatch is source-reproducible.

## Solution Assessment

Unclear until maintainers choose the command taxonomy and deprecation policy. The narrowest maintainable path is likely to add the grouped skill namespace while preserving root aliases during a documented compatibility window.
