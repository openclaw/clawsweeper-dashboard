## Summary

Keep open: current main and release v0.20.0 still use the mixed command layout, while this collaborator-authored public CLI contract change requires explicit maintainer direction on naming and backward compatibility.

## What This Changes

Consolidate most root-level skill commands under a skill namespace now that the CLI also manages plugins and packages.

## Best Possible Solution

Introduce a maintainer-approved grouped skill namespace while retaining root aliases for a defined compatibility period, with synchronized help output, tests, public documentation, and durable CLI intent.

## Reproduction Assessment

Yes. Current-main source and the v0.20.0 release directly reproduce the mixed layout of root-level skill workflows and a partial singular `skill` namespace.

## Solution Assessment

Unclear until maintainers choose the permanent namespace and compatibility policy. Grouped commands with retained aliases and an explicit deprecation plan are safer than an unconditional command move.
