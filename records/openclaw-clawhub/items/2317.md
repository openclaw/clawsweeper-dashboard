## Summary

Current main still exposes the mixed top-level skill command surface, so the requested consolidation remains necessary. Because this is a collaborator-authored CLI product and compatibility decision with no canonical replacement item, it should stay open for explicit maintainer direction.

## What This Changes

Consolidate most top-level skill operations beneath a `clawhub skills` command group now that the CLI also manages plugins/packages.

## Best Possible Solution

Define one canonical skill namespace, add grouped equivalents for the selected commands, retain compatible top-level aliases through a documented deprecation window, and cover root help plus both invocation forms with CLI tests.
