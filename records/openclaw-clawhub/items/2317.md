## Summary

Keep open: current main and the latest CLI release still expose most skill operations as top-level `clawhub` commands, and this collaborator-authored CLI namespace cleanup needs maintainer compatibility and naming decisions before implementation.

## What This Changes

The issue requests moving most skill-related top-level `clawhub` CLI commands under a `clawhub skills` namespace now that skills and plugins both exist.

## Best Possible Solution

Maintainers should choose the stable CLI namespace and compatibility policy, then consolidate skill commands with focused help, docs, and CLI tests while preserving intentional legacy aliases.

## Reproduction Assessment

Yes. Source inspection of `packages/clawhub/src/cli.ts` on current main shows the main skill commands still registered at the root while only a narrow `skill` group exists.

## Solution Assessment

Unclear. The consolidation direction is aligned, but the best implementation depends on maintainer choices about `skill` versus `skills`, which top-level aliases remain, and deprecation timing.
