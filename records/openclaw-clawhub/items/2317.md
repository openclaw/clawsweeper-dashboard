## Summary

Current `v0.21.0` still exposes most skill consumer operations at the CLI root while grouping publishing under `skill` and plugin bundles under `package`, so the requested consolidation remains unimplemented product work. This collaborator-authored issue should stay open for an explicit compatibility and naming decision rather than conservative cleanup.

## What This Changes

Consolidate most skill-specific top-level CLI commands under a dedicated `clawhub skills` command group now that the CLI also manages plugin packages.

## Best Possible Solution

Make one skill namespace canonical, retain all established root spellings as compatibility aliases during a documented deprecation window, and update help, docs, and CLI artifact tests together.

## Reproduction Assessment

Yes. Current `v0.21.0` source and documentation directly reproduce the mixed hierarchy: most skill operations are root commands while skill publishing and package/plugin operations are grouped.

## Solution Assessment

Unclear as currently phrased. Consolidation is directionally sound, but maintainers must first choose singular versus plural naming, command membership, alias behavior, and deprecation timing so existing workflows remain compatible.
