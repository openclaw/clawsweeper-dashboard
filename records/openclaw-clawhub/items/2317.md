Keep open: current main and release v0.20.0 still expose most skill workflows as root-level CLI commands, and this collaborator-authored public CLI contract change requires explicit maintainer direction before implementation.

## What This Changes

Consolidate skill-oriented root commands under a dedicated `clawhub skills` namespace now that the CLI also manages plugins/packages.

## Best Possible Solution

Add a maintainer-approved grouped skill namespace while preserving documented root aliases through a defined compatibility period, updating help output, command tests, public CLI docs, and durable CLI intent together.

## Reproduction Assessment

Yes. Current-main source and public CLI documentation directly reproduce the reported mixed layout: most skill workflows remain at the root while only two commands use the partial singular `skill` namespace.

## Solution Assessment

Unclear until maintainers choose the permanent namespace and compatibility policy. The safest implementation is grouped skill commands with retained root aliases and an explicit deprecation plan, rather than an unconditional command move.
