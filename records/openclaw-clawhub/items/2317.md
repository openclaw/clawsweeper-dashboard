## Summary

Current main still has the mixed command hierarchy described in this issue: most skill workflows remain root commands, while `skill` contains only a narrower set of publishing and verification operations. This collaborator-authored proposal remains necessary and should stay open for a maintainer decision on naming and backward compatibility.

## What This Changes

Consolidate skill-oriented top-level CLI commands beneath a `clawhub skills` command group now that the CLI also manages plugins and packages.

## Best Possible Solution

Adopt one consistent, maintainer-approved namespace for skill operations, retain existing root commands as compatibility aliases with deprecation messaging, and update help snapshots, command tests, and public CLI documentation together.

## Reproduction Assessment

Yes. Current main’s command registration directly reproduces the mixed root-level and grouped skill command hierarchy described in the issue.

## Solution Assessment

No implementation has been selected yet; the best approach is compatibility-preserving consolidation rather than immediately removing or renaming established root commands.
