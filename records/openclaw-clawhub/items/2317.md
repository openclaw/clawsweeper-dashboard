## Summary

Keep open: this is a valid ClawHub CLI ergonomics/product cleanup request, and current main still exposes most skill workflows as top-level commands while only publish/rename/merge live under `skill`. It is not a bug-fix cleanup close candidate because the command naming, singular/plural namespace, and backwards-compatible alias/deprecation policy need maintainer judgment.

## What This Changes

The issue asks to consolidate most skill-related top-level `clawhub` CLI commands under a `clawhub skills` command group now that skills and plugins/packages coexist.

## Best Possible Solution

Add a maintainer-approved skills namespace with tests and docs while preserving existing top-level aliases through an explicit deprecation or compatibility plan.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs inspection confirm the reported CLI shape on current main, but the issue is asking for a command-namespace redesign rather than reporting a failing behavior.

## Solution Assessment

Directionally yes, but not as a breaking move. The maintainable path is to add or choose the skill namespace and keep current top-level commands as aliases until maintainers decide a deprecation policy.
