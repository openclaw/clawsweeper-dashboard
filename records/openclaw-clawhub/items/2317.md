## Summary

Current main still has the inconsistent command hierarchy described in the report, and no merged or viable canonical replacement was found. This collaborator-authored issue should remain open because reorganizing the public CLI requires an explicit maintainer decision about naming and backward compatibility.

## What This Changes

Consolidate most top-level skill-oriented CLI commands beneath a `clawhub skills` command group now that the CLI also manages plugins/packages.

## Best Possible Solution

Establish one canonical skills namespace, preserve existing root spellings as tested compatibility aliases during a documented deprecation window, and update help output, invocation tests, and CLI documentation together.

## Reproduction Assessment

Yes. Current main source directly shows the reported mixture of root-level skill commands, a singular `skill` group, and a separate `package` group.

## Solution Assessment

Unclear as currently specified. Consolidation is a maintainable direction, but maintainers must choose the canonical group name, alias policy, deprecation period, and treatment of cross-cutting commands before implementation.
