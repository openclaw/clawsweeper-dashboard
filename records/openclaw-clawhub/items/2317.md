## Summary

Keep open: this is a valid CLI ergonomics/product cleanup request, and current main still exposes most skill workflows as root-level commands while only publish/rename/merge live under `skill`. The remaining decision is the supported namespace plus backward-compatible alias and deprecation policy.

## What This Changes

The issue asks to move most skill-related `clawhub` CLI root commands under a skills command namespace now that both skills and plugins exist.

## Best Possible Solution

Define a maintainer-approved `skills` namespace, preserve existing root commands as aliases for a documented compatibility window, and update CLI tests plus docs around the chosen command names.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs inspection confirm the reported CLI shape on current main, but the issue is asking for a command-namespace redesign rather than reporting failing behavior.

## Solution Assessment

Directionally yes, but not as a breaking move. The maintainable path is to choose the skills namespace and keep current top-level commands as aliases until maintainers decide a deprecation policy.
