## Summary

Current main and the latest v0.20.2 release still expose a mixed hierarchy: most skill lifecycle commands remain top-level while only published-skill operations use the singular `skill` group. The request is therefore still necessary, but its collaborator authorship and unresolved namespace and compatibility choices require explicit maintainer product judgment.

## What This Changes

Consolidate skill-oriented top-level CLI commands beneath a consistent skill namespace now that ClawHub also manages plugin packages.

## Best Possible Solution

Make one skill namespace canonical, preserve the existing root forms as compatibility aliases for a defined transition, and update help output, documentation, and tests for both canonical and legacy invocations.

## Reproduction Assessment

Yes. Current command registration and CLI documentation directly reproduce the mixed root-level and grouped skill command hierarchy described by the issue.

## Solution Assessment

Unclear. Consolidation is directionally maintainable, but maintainers must decide singular versus plural naming, which operations belong in the group, and how long legacy aliases remain supported.
