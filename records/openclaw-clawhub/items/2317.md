## Summary

Current main still has the command split described by the issue, and VISION.md supports CLI ergonomics work; however, this collaborator-authored request requires a maintainer decision on namespace scope and backward compatibility before implementation.

## What This Changes

Consolidate most top-level skill operations into a `clawhub skills` command group now that the CLI also exposes plugin/package operations.

## Best Possible Solution

Define a coherent skill/package/plugin command taxonomy, add `clawhub skills` as the canonical skill namespace, and preserve existing top-level invocations as tested compatibility aliases through an explicit deprecation period.

## Reproduction Assessment

Yes. Current-main command registration provides a high-confidence source reproduction: skill operations remain split between root commands and a limited singular `skill` group.

## Solution Assessment

Unclear. Consolidation is directionally maintainable, but moving commands without first deciding the canonical singular/plural taxonomy, command membership, aliases, and deprecation contract would be unsafe.
