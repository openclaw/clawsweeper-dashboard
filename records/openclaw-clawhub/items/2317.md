## Summary

The requested consolidation is still necessary: current main retains most skill operations as top-level commands while package/plugin operations are grouped, and the collaborator-authored issue requires an explicit maintainer decision on the public CLI migration contract.

## What This Changes

Move most skill-oriented top-level CLI commands into a consolidated skill command group alongside the existing grouped package/plugin commands.

## Best Possible Solution

Define `clawhub skill` as the canonical grouped surface, add the applicable install/search/update and publication lifecycle subcommands, retain hidden or deprecated root aliases for a documented transition, and test both canonical and compatibility invocations plus help output.

## Reproduction Assessment

Yes. Current main directly reproduces the reported mixed hierarchy in `packages/clawhub/src/cli.ts`: most skill operations remain at the root while only publish and verify are grouped under `skill`.

## Solution Assessment

Not yet fully. Consolidation is a maintainable direction, but the implementation should wait for a maintainer decision on canonical naming, command scope, compatibility aliases, and deprecation timing.
