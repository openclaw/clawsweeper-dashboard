## What This Changes

The issue reports that `clawhub sync` and `clawhub publish` fail because CLI v0.7.0 did not send `acceptLicenseTerms` in the publish payload.

## Best Possible Solution

Keep the fixed license-consent behavior, leave this issue open for maintainer cleanup policy, and route any remaining publish failure to the open GitHub account-age rate-limit reports rather than changing the consent contract again.

## Reproduction Assessment

No for the original current-main bug: source inspection and the current npm tarball show `publish` now sends `acceptLicenseTerms: true`, `sync` delegates to that path, and tests assert the payload field. The later GitHub rate-limit behavior is a separate source-reproducible path already tracked by open rate-limit reports.

## Solution Assessment

Yes for the license-payload portion: explicit CLI consent plus API enforcement and regression coverage is the narrow maintainable fix. The downstream GitHub account-age/rate-limit message should be handled in the existing rate-limit cluster after production log and configuration review.
