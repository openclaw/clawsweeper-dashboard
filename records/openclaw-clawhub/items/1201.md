## What This Changes

## Best Possible Solution

Keep the cursor-scheduled digest implementation, smoke-test production CLI publish and web upload for affected or first-time publishers, then close or consolidate the related publish-pagination reports.

## Reproduction Assessment

No. The original March 23, 2026 reports give a credible CLI/web reproduction, but current main no longer runs multiple owner-publisher paginated queries inside one mutation; I did not run a live production publish smoke test.

## Solution Assessment

Yes. Current main's self-scheduled cursor approach is the maintainable fix because it preserves indexed pagination and avoids the document-limit risks in the closed collect/take proposals such as https://github.com/openclaw/clawhub/pull/1202.
