## What This Changes

The issue reports a ClawHub skill that reads arbitrary local document paths, base64-encodes the bytes, and sends them to a third-party PaddleOCR API with bearer-token auth.

## Best Possible Solution

Keep the issue open until publisher remediation and ClawHub scanner/moderation policy for Python local-file uploads are both reviewed and reflected in the live skill state.
