## What This Changes

The issue reports that the hosted bobholamovic/paddleocr-doc-parsing skill can read arbitrary local files, base64-encode them, and upload them to a third-party PaddleOCR API under bearer-token auth without sufficient safeguards or disclosure.

## Best Possible Solution

Complete a maintainer security/moderation disposition for the hosted listing: require publisher safeguards and disclosure, or keep it hidden/removed until the upload behavior is acceptable.

## Reproduction Assessment

Yes. The current public hosted source shows the local-file base64 upload path, and current main has regression tests covering the same shell and Python patterns; I did not invoke the external PaddleOCR service.

## Solution Assessment

No. Scanner coverage is a useful mitigation, but it is not the final solution because the remaining question is whether the hosted listing should be accepted, remediated by the publisher, hidden, or removed.
