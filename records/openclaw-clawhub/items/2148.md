## What This Changes

Adds a shared `SignInPrompt` component and replaces several signed-out route and comment UIs with it, with new prompt and stars route tests.

## Best Possible Solution

Preserve the shared prompt direction only after restoring explicit auth-loading handling on routes like `/import` and updating the branch against current `main`.

## Reproduction Assessment

Yes, at source level: the PR branch drops `/import`'s `isLoading` handling and renders `SignInPrompt` while `isAuthenticated` is initially false. A browser repro would be loading `/import` as an already signed-in user while Convex auth is still initializing.

## Solution Assessment

No. The shared component is a maintainable direction, but `/import` should keep a loading or disabled state until auth resolution instead of showing an actionable sign-in prompt immediately.
