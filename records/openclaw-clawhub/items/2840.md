## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Validate resolved image hosts before fetching:** `server/og/fetchImageDataUrl.ts:54`
  - body: This helper now treats any non-IP HTTPS hostname as safe, but it never checks what address the hostname resolves to before `fetch()` runs. Because `/og/profile` accepts an `avatar` query value and calls this path with `allowPublicHttps`, an attacker-controlled domain can resolve to link-local/private/metadata space and still be fetched server-side; validate resolved IPs for the initial URL and redirects or keep this on a trusted image boundary.
  - confidence: 0.92
- **[P2] Keep publisher download counts compact:** `src/lib/og.ts:146`
  - body: This serializes the raw integer download count into the OG URL, and `profile.png.ts` prefers the query value as-is, so publisher cards bypass `formatOgStat` and render values like `43456` instead of the compact value promised in the PR body. Format the value before serializing it or parse and format numeric query values in the route.
  - confidence: 0.95
