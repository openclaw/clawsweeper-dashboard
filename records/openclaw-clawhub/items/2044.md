## Summary

Keep open: current main still has different suspicious-skill defaults between unified search and the skills directory, and https://github.com/openclaw/clawhub/pull/2079 is an open, unmerged fix PR with closing syntax for this issue.

## What This Changes

The issue asks for homepage/unified search and `/skills` to return consistent skill results for the same query, or to visibly explain the different suspicious-skill filters.

## Best Possible Solution

Review and land or close https://github.com/openclaw/clawhub/pull/2079; the durable end state should document the suspicious-skill visibility contract and enforce it with shared UI/query behavior plus regression coverage.

## Reproduction Assessment

Yes. Current main shows unified search passing `nonSuspiciousOnly: true` while `/skills` defaults it to false, and the previous ClawSweeper review recorded a live 1-versus-6 check for `国泰海通` on May 7, 2026.

## Solution Assessment

Mostly yes. The linked PR's disclosure approach is narrower than changing moderation defaults, but maintainers should finish that PR rather than close this issue before merge.
