## Summary

This should stay open under the ClawHub issue policy, and current main still shows the reported split: unified search filters suspicious skills by default while /skills does not unless the visible directory filter is enabled.

## What This Changes

The issue requests consistent skill search results, or visible filter disclosure, between homepage/unified search and /skills for the same query.

## Best Possible Solution

Define a shared suspicious-skill visibility rule for unified and directory search, then enforce it with visible UI state and regression coverage.

## Reproduction Assessment

Yes, for the root cause: current main shows two search entry points calling the same backend action with different nonSuspiciousOnly defaults. The exact 1-versus-6 count still depends on production data for the reported query.

## Solution Assessment

Unclear until maintainers choose the intended moderation policy. The narrow maintainable fix is either to align the defaults through a shared search contract or keep the filtering split but make the active filter visible in unified search.
