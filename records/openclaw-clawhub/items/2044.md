## Summary

Keep this open: current main still has different suspicious-skill defaults between unified search and the skills directory, and choosing whether to align results or disclose the filter needs maintainer product/security judgment.

## What This Changes

Requests consistent skill search results, or visible filter disclosure, between homepage/unified search and `/skills` for the same query.

## Best Possible Solution

Define one public-search moderation visibility contract, then enforce it consistently across unified search and `/skills` with visible UI state when a surface intentionally hides suspicious skills.

## Reproduction Assessment

Yes for the source-level mismatch: current main passes `nonSuspiciousOnly: true` from unified search and `false` by default from `/skills` to the same Convex action. The exact production result-count mismatch still depends on live data.

## Solution Assessment

Unclear until maintainers choose the intended moderation policy. The narrow maintainable fix is either a shared default across both surfaces or visible disclosure/filter state in unified search.
