## Summary

Keep this open: current main still has different suspicious-skill defaults between unified search and the skills directory, and the open closing PR at https://github.com/openclaw/clawhub/pull/2079 has not merged.

## What This Changes

The issue reports inconsistent skill search counts because homepage/unified search hides suspicious skills by default while `/skills` includes them unless the `Hide suspicious` filter is enabled.

## Best Possible Solution

Review and land or close https://github.com/openclaw/clawhub/pull/2079; after a fix merges, close this issue as implemented.

## Reproduction Assessment

Yes. Current main source shows unified search hard-coding `nonSuspiciousOnly: true` while `/skills` defaults it to false, and the live public search API still returns different counts for the reported query when that flag changes.

## Solution Assessment

Yes, the linked disclosure approach is the narrowest maintainable direction if its review blockers are resolved, because it preserves the safer unified-search default while explaining why counts differ.
