## Summary

Keep this open: current main still has different suspicious-skill defaults between unified search and the skills directory, and the linked closing PR is open rather than merged.

## What This Changes

The issue asks ClawHub to make homepage/unified search and `/skills` return consistent skill results for the same query or visibly disclose different suspicious-skill filters.

## Best Possible Solution

Land a vetted fix such as the linked PR after resolving review findings, keeping the safer unified-search default while clearly disclosing hidden suspicious matches or documenting one shared search policy.

## Reproduction Assessment

Yes. Current main hardcodes `nonSuspiciousOnly: true` in unified search while `/skills` defaults it to false, and the public API returned the reported 1-versus-6 split for `国泰海通` on May 7, 2026.

## Solution Assessment

Yes in direction, but not resolved on main yet. Keeping suspicious skills hidden in unified search while adding a visible hidden-results disclosure is a maintainable path, provided the linked PR’s remaining review findings are fixed before merge.
