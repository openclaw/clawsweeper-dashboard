## Summary

This should stay open under the ClawHub issue policy, and the reported split is real: current main sends unified search through the suspicious-filtered path while `/skills` defaults that filter off.

## What This Changes

Request to make homepage/unified skill search and `/skills` return consistent skill results for the same query, or clearly expose any different suspicious-skill filter.

## Best Possible Solution

Define one product rule for suspicious-skill visibility across global and directory search, then enforce it with shared UI state and regression tests.

## Reproduction Assessment

Yes. Live API checks on May 6, 2026 reproduced the 6-versus-1 split for `国泰海通`, and current main shows the two entry points passing different `nonSuspiciousOnly` defaults.

## Solution Assessment

Unclear until maintainers choose the moderation policy. The narrow maintainable solution is either to align the defaults through a shared search contract or keep the split but make the active filter visible in unified search.
