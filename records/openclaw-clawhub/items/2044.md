## Summary

This should stay open under the ClawHub issue policy, and current main still has the reported source-level split: unified search hides suspicious skills by default while /skills does not unless the visible directory filter is enabled.

## What This Changes

Request to make homepage/unified search and /skills return consistent skill results for the same query, or visibly explain differing suspicious-skill filters.

## Best Possible Solution

Define one product rule for suspicious-skill visibility across global and directory search, then enforce that shared rule with visible UI state and regression tests.

## Reproduction Assessment

Yes for the root cause: current main shows the two search entry points calling the same backend action with different `nonSuspiciousOnly` defaults. The exact reported result count still depends on production data for `国泰海通`.

## Solution Assessment

Unclear until maintainers choose the intended moderation policy. The narrow maintainable path is either to align defaults through a shared search contract or keep the filtering split but make the active filter visible in unified search.
