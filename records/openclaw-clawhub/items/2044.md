## Summary

Keep this open: current main still has different suspicious-skill defaults between unified search and the skills directory, and the closing PR at https://github.com/openclaw/clawhub/pull/2079 is still open rather than merged.

## What This Changes

Request that homepage/unified search and `/skills` either return consistent skill results for the same query or visibly disclose default suspicious-skill filtering differences.

## Best Possible Solution

Review and land or close https://github.com/openclaw/clawhub/pull/2079; after it merges, close this issue as implemented by that PR.

## Reproduction Assessment

Yes. Current main shows homepage search routing to `/search`, unified search hard-coding `nonSuspiciousOnly: true`, and `/skills` defaulting the same flag to false; the live API also still returns different counts for the reported query when the flag changes.

## Solution Assessment

Yes. Preserving the safer unified-search default while adding a clear disclosure and `/skills` path is the narrowest maintainable direction, but the linked PR still needs review/merge handling before this issue is resolved.
