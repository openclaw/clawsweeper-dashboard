## Summary

This should stay open under the ClawHub issue policy, and the source still supports the reported inconsistency: unified search hides suspicious skills by default while the /skills directory does not unless the visible filter is enabled.

## What This Changes

The issue requests consistent skill-search results, or a visible explanation of different suspicious-skill filtering, between homepage/unified search and /skills.

## Best Possible Solution

Define a single product rule for suspicious-skill visibility across global and directory search, then enforce it through shared search state, visible UI copy where needed, and focused regression tests.

## Reproduction Assessment

Yes, for the source-level root cause: current main shows two search entry points calling the same backend action with different nonSuspiciousOnly defaults. The exact 1-versus-6 result count still depends on production data for the reported query.

## Solution Assessment

Unclear until maintainers choose the intended moderation policy. The narrow maintainable fix is either to align the defaults through a shared search contract or keep the filtering split but make the active filter visible in unified search.
