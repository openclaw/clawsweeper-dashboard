## Summary

Keep this open: current main still has different suspicious-skill defaults between unified search and `/skills`, and https://github.com/openclaw/clawhub/pull/2079 is an open closing PR rather than a merged fix.

## What This Changes

The issue requests aligning suspicious-skill filtering between homepage/unified search and `/skills`, or visibly explaining the filter difference.

## Best Possible Solution

Land a reviewed fix that keeps one documented moderation-visibility contract across search surfaces, either by aligning defaults or by clearly disclosing hidden suspicious matches wherever users see filtered results.

Yes. Source inspection gives a high-confidence path: unified search hardcodes `nonSuspiciousOnly: true`, while `/skills` defaults it to false and the backend applies that flag when hydrating results.
