## Summary

Keep open: current main still applies different suspicious-skill defaults between unified search and `/skills`, `/search` has no visible disclosure, production search returns different counts for the same query when the flag changes, and the open member PR has not merged.

## What This Changes

The report asks for homepage/unified search and `/skills` to return consistent skill results for the same query or visibly disclose differing suspicious-skill filters.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/clawhub/pull/2079 with a fix that preserves moderation semantics and either aligns the defaults or clearly discloses the hidden-suspicious filter on `/search`.

## Reproduction Assessment

Yes. Current main shows `/search` hard-coding `nonSuspiciousOnly: true` while `/skills` defaults it to false, and the production Convex action returns 3 versus 6 results for `国泰海通` when that flag is toggled.

## Solution Assessment

Yes. The safest narrow direction is to keep unified search’s hide-suspicious default and add visible disclosure or an explicit opt-out; changing the default globally should be a maintainer moderation decision.
