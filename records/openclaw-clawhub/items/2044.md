## Summary

Keep this open: ClawHub cleanup policy keeps issues open, and current main still has different suspicious-skill filtering defaults between unified search and the skills directory. The fix needs maintainer product/security judgment because it changes whether suspicious skills are hidden or shown with clearer context.

## What This Changes

The issue asks for homepage/unified search and `/skills` to return consistent skill results for the same query, or to visibly explain different default filters.

## Best Possible Solution

Define one shared moderation-visibility rule for public search surfaces, then enforce it consistently with visible UI state and focused regression coverage.

## Reproduction Assessment

Yes for the source-level root cause: current main has two search entry points calling the same backend action with different `nonSuspiciousOnly` defaults. The exact production 1-versus-6 browser count is not fully reproduced from source or the public API check.

## Solution Assessment

Unclear until maintainers choose the intended moderation policy. The narrow maintainable solution is either to align defaults through a shared search contract or keep the split and make the active moderation filter visible in unified search.
