## Summary

Keep this open: current main still has the reported split between unified search and the skills directory, and the fix changes how suspicious skills are hidden or surfaced in public search, so it needs maintainer product/security judgment.

## What This Changes

Request to make homepage/unified search and `/skills` return consistent skill results for the same query or clearly show their different suspicious-skill filters.

## Best Possible Solution

Define one shared public-search moderation visibility contract, then enforce it consistently across homepage `/search`, header typeahead, and `/skills` with visible filter state and focused regression coverage.

## Reproduction Assessment

Yes for the root cause: current main has two search entry points passing different `nonSuspiciousOnly` defaults into the same backend action. The public API also reproduces the reported 6-versus-1 result split for `国泰海通` when the flag is toggled.

## Solution Assessment

Unclear until maintainers choose the intended moderation policy. The narrow maintainable fix is either to align the defaults through a shared search contract or keep the split and make the active suspicious-skill filter visible in unified search.
