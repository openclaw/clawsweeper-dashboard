## Summary

Keep this open: current main still has the reported source-level split between unified search and the skills directory, and the repair needs maintainer policy for suspicious-skill visibility.

## What This Changes

The issue asks for homepage/unified skill search and `/skills` search to return consistent results for the same query, or to visibly explain differing default filters.

## Best Possible Solution

Define one shared moderation-visibility rule for public search surfaces, enforce it consistently, and add visible UI state plus focused regression coverage.

## Reproduction Assessment

Yes for the source-level root cause: current main has two search entry points calling the same backend action with different `nonSuspiciousOnly` defaults. The exact production result-count mismatch still depends on live data for `国泰海通`.

## Solution Assessment

Unclear until maintainers choose the intended moderation policy. The narrow maintainable path is either to align defaults through a shared search contract or keep the split and make the active filter visible in unified search.
