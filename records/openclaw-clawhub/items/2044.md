## Summary

Keep this open: current main still has a source-level split between unified search and the skills directory, and resolving it needs maintainer policy judgment around suspicious-skill visibility.

## What This Changes

Make homepage/unified search and `/skills` return consistent skill results for the same query, or visibly disclose that different suspicious-skill filtering is active.

## Best Possible Solution

Define one shared public-search moderation rule, then enforce it consistently across `/search` and `/skills` with visible filter state and focused regression coverage.

## Reproduction Assessment

Yes for the source-level path: current main routes unified search through `useUnifiedSearch` with `nonSuspiciousOnly: true`, while `/skills` derives `search.nonSuspicious ?? false`. The exact reported result counts still depend on production data for `国泰海通`.

## Solution Assessment

Unclear until maintainers choose the intended moderation policy. The narrow maintainable fix is either a shared search visibility contract or an explicit UI indication when unified search is hiding suspicious skills.
