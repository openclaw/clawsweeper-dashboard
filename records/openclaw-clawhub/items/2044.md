## Summary

Keep open: current main still has source-level inconsistent suspicious-skill filtering between unified search and `/skills`, and the intended public moderation default needs maintainer product/security judgment.

## What This Changes

The issue asks for homepage/unified search and `/skills` to return consistent skill results for the same query or clearly expose their different suspicious-skill filtering defaults.

## Best Possible Solution

Define one shared moderation-visibility rule for public search surfaces, then enforce it consistently with visible UI state and focused regression coverage.

## Reproduction Assessment

Yes for the root cause: current main calls `searchSkills` with `nonSuspiciousOnly: true` in unified search and defaults the same flag to false in `/skills`. The exact production result-count mismatch is data-dependent and was not reproduced locally.

## Solution Assessment

Unclear until maintainers choose the intended moderation policy. The narrow maintainable fix is either to centralize the default across search surfaces or keep the split and make the active filter visible in unified search.
