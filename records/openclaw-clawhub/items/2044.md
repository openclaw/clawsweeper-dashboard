## Summary

This should stay open under the ClawHub issue policy, and current main still shows the reported split: unified search hides suspicious skills by default while /skills does not unless the user enables the visible filter.

## What This Changes

The issue reports inconsistent skill-search results between homepage/unified search and the /skills directory for the same query.

## Best Possible Solution

Define one suspicious-skill visibility rule for global and directory search, then enforce it with shared search UI state and regression tests.

## Reproduction Assessment

Yes, for the root cause: current main has two search entry points calling the same backend action with different nonSuspiciousOnly defaults. The exact 1-versus-6 result count still depends on production data for the reported query.

## Solution Assessment

Unclear until maintainers choose the intended moderation policy. The narrow maintainable fix is either to align the defaults through a shared contract or keep the filtering split but make the active filter visible in unified search.
