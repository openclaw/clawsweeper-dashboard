## Summary

Keep this open: current main still has different suspicious-skill defaults between unified search and the `/skills` directory, and https://github.com/openclaw/clawhub/pull/2079 is an open, unmerged closing PR for the same work.

## What This Changes

The issue asks for homepage/unified search and the `/skills` directory to either return consistent skill results for the same query or visibly disclose any different suspicious-skill filtering.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/clawhub/pull/2079 so unified search keeps a deliberate moderation default and clearly tells users when suspicious skill matches are hidden.

## Reproduction Assessment

Yes. Current main source shows the homepage routes searches to `/search`, unified search passes `nonSuspiciousOnly: true`, and `/skills` defaults the same flag to false unless the user enables `Hide suspicious`.

## Solution Assessment

Yes, the linked disclosure approach is a narrow maintainable direction because it preserves the safer unified-search default while explaining missing results; changing visibility defaults across surfaces should remain a maintainer moderation-policy decision.
