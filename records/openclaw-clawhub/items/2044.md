## Summary

Keep this open: current main still has the unified-search versus skills-directory suspicious-filter mismatch, and an open linked PR is already attempting the fix so the issue should remain open until that PR lands or is closed.

## What This Changes

The issue asks for homepage/unified search and `/skills` skill results to be consistent for the same query, or for the UI to disclose that different suspicious-skill filters are applied.

## Best Possible Solution

Review and land https://github.com/openclaw/clawhub/pull/2079 or an equivalent patch that preserves the safer unified-search default while clearly disclosing hidden suspicious results and linking to the existing `/skills` filter path.

## Reproduction Assessment

Yes. Current main has a clear source reproduction: unified search passes `nonSuspiciousOnly: true`, `/skills` defaults it to false, and the live API confirmed the flag changes `国泰海通` from 2 to 6 results on May 8, 2026.

## Solution Assessment

Yes, the linked PR direction is the best current path if it passes review: disclose the hidden suspicious-results state while keeping suspicious skills out of inline unified search by default.
