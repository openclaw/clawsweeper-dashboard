## Summary

Keep this open: current main still has different suspicious-skill defaults between unified search and the skills directory, and ClawHub policy says issues are not auto-closed by this cleanup pass.

## What This Changes

Request to make homepage/unified search and `/skills` return consistent skill results for the same query, or clearly show the different suspicious-skill filter.

## Best Possible Solution

Adopt one documented moderation-visibility contract for public search surfaces, then encode it in shared query/UI state with focused regression coverage.

## Reproduction Assessment

Yes. Current main shows unified search hard-coding `nonSuspiciousOnly: true` while `/skills` defaults it to false, and the live API confirmed the reported 1-versus-6 split for `国泰海通` on May 7, 2026.

## Solution Assessment

Unclear until maintainers choose the intended moderation policy. The narrow maintainable path is either one shared default across public search surfaces or a visible unified-search filter/explanation that matches the existing `/skills` state.
