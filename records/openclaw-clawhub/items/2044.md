## Summary

Keep open: current main still has the reported source-level split between unified search and /skills, and ClawHub issue policy keeps issues open rather than auto-closing them. Resolving it needs a maintainer decision about suspicious-skill visibility, not a cleanup close.

## What This Changes

The issue asks for homepage/unified skill search and /skills search to return consistent results for the same query or visibly disclose differing suspicious-skill filters.

## Best Possible Solution

Define the intended suspicious-skill visibility rule for each search surface, implement it through shared search/UI state, and add regression coverage for both entry points.

## Reproduction Assessment

Yes, for the root cause: current main shows the two entry points call the same backend action with different nonSuspiciousOnly defaults. The exact reported result count still needs production data to verify.

## Solution Assessment

Unclear until maintainers choose the intended moderation policy. The narrow maintainable path is either a shared default across search surfaces or a visible unified-search filter/notice that explains the difference.
