## Summary

Keep open: current main still has different suspicious-skill defaults between unified search and the skills directory, and an open linked PR is the active fix path.

## What This Changes

The issue asks for consistent skill search results across homepage/unified search and `/skills`, or a visible explanation when different default filters apply.

## Best Possible Solution

Review and land or close https://github.com/openclaw/clawhub/pull/2079; the end state should preserve the intended moderation-visibility policy and make any filter difference explicit to users.

## Reproduction Assessment

Yes. Source inspection on current main shows homepage search routes into unified search with `nonSuspiciousOnly: true`, while `/skills` defaults that flag to false and exposes it as an optional filter.

## Solution Assessment

Mostly yes for the linked direction: disclosing hidden suspicious results preserves the safer unified-search default while addressing the unexplained result-count mismatch. Changing defaults instead would need an explicit moderation policy decision.
