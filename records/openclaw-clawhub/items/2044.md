## Summary

Keep open under the ClawHub issue policy. Current main still has the reported source-level split: unified search hides suspicious skills by default while `/skills` does not unless the visible filter is enabled, and the right fix needs maintainer moderation/product judgment.

## What This Changes

The issue requests consistent skill-search results, or a visible explanation of different default filters, between homepage/unified search and `/skills`.

## Best Possible Solution

Define one moderation visibility rule for suspicious skills across global search and directory search, then implement it with shared caller defaults or clearly visible filter state plus regression coverage.

## Reproduction Assessment

Yes, for the root cause: current main shows the two search entry points calling the same backend action with different `nonSuspiciousOnly` defaults. The exact production result count for `国泰海通` still depends on live data.

## Solution Assessment

Unclear until maintainers choose the intended moderation policy. The narrow maintainable fix is either to align the defaults through a shared search contract or keep the split but make the active filter visible in unified search.
