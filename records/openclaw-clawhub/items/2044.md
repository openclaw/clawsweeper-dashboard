## Summary

Keep this open under the ClawHub issue policy; current main still has the reported split, and the remaining fix needs a maintainer decision on suspicious-skill visibility across search surfaces.

## What This Changes

The issue asks for homepage/unified search and `/skills` to return consistent skill results for the same query, or to make the different suspicious-skill filter state visible.

## Best Possible Solution

Define one suspicious-skill visibility rule for global and directory search, then enforce it through shared search defaults or visible filter state with regression coverage.

## Reproduction Assessment

Yes, for the source-level root cause: current main sends `nonSuspiciousOnly: true` from unified search while `/skills` defaults the same flag to false. I did not verify the exact 1-versus-6 production count because that depends on live data.

## Solution Assessment

Unclear until maintainers choose the intended moderation policy. The narrow maintainable path is to centralize the search contract or expose the active suspicious-skill filter, not to remove the safety filter without UI context.
