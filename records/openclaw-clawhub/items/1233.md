## Summary

Keep open. Current main only partially overlaps the PR: the skill detail header already uses a download icon but still keeps stars first, keeps current and all-time installs as separate visible metrics, shared SkillStats/SoulStats still use the package icon and old ordering, and the new installs tooltip helper is absent. The PR also still has a blocking accessibility issue because one installs metric is moved into a native title tooltip, and the narrower download-icon work overlaps open draft #1854.

## What This Changes

The PR updates skill detail, shared skill stats, and soul stats UI semantics, adds an installs tooltip formatter with tests, and adds skill-detail coverage for install-count edge cases.

## Best Possible Solution

Land a clean, rebased stats-semantics update that consistently applies the chosen download/install icon semantics and metric ordering across the public skill and soul stat surfaces, preserves visible or ARIA-accessible current and all-time install counts, keeps current main layout behavior, and coordinates with or subsumes #1854.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Expose current installs without relying on title:** `src/components/SkillHeader.tsx:224`
  - body: When `installsAllTime > 0`, the rendered text shows only the all-time install count while `installsCurrent` is available only through the native `title` tooltip. Native title tooltips are not reliably reachable on touch devices or keyboard-only paths, so the current install metric regresses from visible text on main to inaccessible hover-only detail. Keep both values visible or use an accessible tooltip pattern with proper ARIA wiring.
  - confidence: 0.88
