## Summary

Keep open: the feature is useful, but the patch currently emails raw manual ban reasons to users and lacks after-fix real behavior proof for the email path.

## What This Changes

The PR adds a Resend-backed internal action and schedules ban notification emails from manual and malware bans, with tests plus moderation/deploy documentation updates.

## Best Possible Solution

Keep the email feature, but send only allowlisted public reason copy or a dedicated public-notice field, then add redacted runtime proof for the Resend dispatch path.

## Reproduction Assessment

Yes for the PR defect: source inspection shows a manual ban with any unrecognized reason reaches the email formatter and is returned verbatim into the outgoing payload. I did not execute the path because this review is read-only.

## Solution Assessment

No as-is. The maintainable path is to keep internal ban reasons for audit/admin use and send only allowlisted, public-safe copy or an explicit public notice field.
