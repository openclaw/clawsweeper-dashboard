## Summary

Keep this PR open: the email-notification feature is useful, but the current patch still has a security-sensitive blocker because manual ban reasons can be sent verbatim to users and Resend, and the contributor has not supplied real runtime email proof.

## What This Changes

The branch adds Resend-backed ban and account-restoration emails to Convex moderation flows, with tests, docs/spec updates, env examples, and a new `resend` dependency.

## Best Possible Solution

Ship the email feature only after outgoing ban copy is allowlisted or separated from internal moderator notes, restored-listing docs match the implemented summary behavior, and redacted Resend runtime proof is attached.
