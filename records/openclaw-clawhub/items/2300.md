## Summary

Keep open: the PR adds a useful moderation email feature, but the latest head still has a P1 security-boundary blocker because unrecognized manual ban reasons can be copied into external Resend email payloads, and the PR still lacks real behavior proof.

## What This Changes

The branch adds Resend-backed ban and account-restored email notifications for manual bans, malware autobans, unbans, and autoban remediation, plus docs, env examples, and unit coverage.

## Best Possible Solution

Keep the notification feature, but derive email copy only from allowlisted public reason codes or a dedicated public-notice field, then add tests and redacted runtime proof for the Resend dispatch path.

## Reproduction Assessment

Yes for the PR defect by source inspection: a manual ban with any unrecognized reason reaches the email formatter and is returned verbatim into the outgoing Resend payload. I did not execute it because this review is read-only.

## Solution Assessment

No, not as-is. The maintainable path is to separate private moderation notes from user-facing email copy, then prove the real Resend path with redacted output before merge.
