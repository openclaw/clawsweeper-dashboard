## Summary

Keep open: this PR is not implemented on main and still has a security-sensitive blocker because free-form manual ban reasons can be copied into external ban notification emails, plus the contributor has not provided real behavior proof for the Resend path.

## What This Changes

The PR adds Resend-backed ban and account-restoration email notifications for manual bans, malware autobans, unbans, and autoban remediation, with env docs, moderation docs/specs, and tests.

## Best Possible Solution

Keep the email notification feature, but send only allowlisted public reason copy or a dedicated public-notice field, then add redacted runtime proof for the Resend dispatch path before merge.

## Reproduction Assessment

Yes for the PR defect by source inspection: a manual ban with any unrecognized reason reaches the email formatter and is returned verbatim into the outgoing payload. I did not execute the path because this review is read-only.

## Solution Assessment

No as-is. The narrow maintainable path is to keep internal ban reasons for audit/admin use and send only allowlisted public copy or an explicitly user-facing notice field.
