## What This Changes

The issue reports that GitHub OAuth authorization returns to clawhub.ai but the site remains logged out instead of refreshing user session state.

## Best Possible Solution

Restore production GitHub sign-in end to end for active accounts, preserve deleted/deactivated/banned account rejection, pick a canonical OAuth/session tracker, and add browser-level OAuth/session smoke proof plus targeted regression coverage for the chosen fix.

Yes. The central production auth failure is reproducible today because both the canonical clawhub.ai and direct Convex GitHub sign-in endpoints return HTTP 500 instead of entering OAuth, although the later callback-without-session symptom cannot be replayed while sign-in fails earlier.

## Solution Assessment

Yes for keeping this open and routing it to maintainer/operator triage. The safest fix depends on production OAuth configuration, Convex logs, cookies/JWT behavior, and account-state policy rather than a source-only cleanup PR.
