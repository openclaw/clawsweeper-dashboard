## Summary

Keep open: the production GitHub sign-in endpoint is still failing before OAuth redirect, the report is corroborated by users and a broad open OAuth/session cluster, and ClawHub cleanup policy keeps issues open unless a merged implementation clearly resolves them.

## What This Changes

The issue reports that GitHub OAuth authorization returns to clawhub.ai but the frontend remains signed out because session state does not refresh.

## Best Possible Solution

Restore production GitHub sign-in for active accounts, preserve deleted/banned account rejection, choose a canonical OAuth/session tracker, and add browser-level smoke or regression proof for callback and session state.

## Reproduction Assessment

Yes for the central production auth failure: on 2026-05-17, both the canonical clawhub.ai and direct Convex GitHub sign-in endpoints returned HTTP/2 500 instead of entering OAuth. The exact callback-succeeds-but-no-session browser path was not replayed because production currently fails earlier.

## Solution Assessment

Yes for keeping this open and routing it to maintainer/operator triage; a source-only automated fix is not the safest path until production OAuth config, Convex logs, cookie/JWT behavior, and account-state data identify the failing branch.
