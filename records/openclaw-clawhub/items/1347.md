## Summary

Keep this open. The issue is a concrete, corroborated production GitHub OAuth/session failure, and current main plus the supplied discussion do not prove the live ClawHub OAuth callback, session cookie, active-user lookup, and account-state paths are fixed.

## What This Changes

Users report that GitHub OAuth on clawhub.ai authorizes successfully but returns to ClawHub without refreshing into an authenticated session.

## Best Possible Solution

Keep a canonical OAuth/session tracker open until maintainers verify production GitHub sign-in for fresh and existing accounts, inspect production OAuth/env/logs, fix the specific defect found, and add browser-level regression coverage with clearer user-facing errors.

## Reproduction Assessment

Yes for the reported production symptom: the issue gives concrete clawhub.ai OAuth steps and multiple corroborating reports; the exact current-main root cause is not reproducible from source alone because live OAuth config, cookies, deployment env, and account state are external.

## Solution Assessment

Is this the best way to solve the issue? Unclear. A blind source patch is not the best next step; maintainers need production auth logs and configuration/account-state inspection before choosing between an env fix, callback/session fix, or account recovery/messaging change.
