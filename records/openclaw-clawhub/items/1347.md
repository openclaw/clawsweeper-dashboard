## Summary

Keep open. This is an open ClawHub issue, which repository cleanup policy keeps in maintainer hands, and the live GitHub sign-in path is still not healthy: both the canonical and direct Convex sign-in endpoints returned HTTP 500 during this review.

## What This Changes

The issue reports that GitHub OAuth returns to ClawHub but the site remains in a signed-out state instead of refreshing user information.

## Best Possible Solution

## Solution Assessment

Unclear as a source-only fix. The narrowest maintainable next step is maintainer/operator triage of production OAuth credentials, Convex/Vercel env, logs, cookies, and account state before deciding whether the repair is config, dependency, or app code.
