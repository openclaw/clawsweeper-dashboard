Keep open: the OAuth/session failure is corroborated, still reproducible at the production sign-in endpoint, and ClawHub cleanup policy does not auto-close issues. The remaining fix needs maintainer/operator auth triage rather than a source-only cleanup decision.

## What This Changes

Users report that GitHub OAuth authorization returns to clawhub.ai without refreshing authenticated session state, leaving the site signed out.

## Best Possible Solution

Restore production GitHub sign-in end to end for active accounts, preserve deleted/deactivated/banned account rejection, pick a canonical OAuth/session tracker, and add browser-level OAuth/session smoke proof plus targeted regression coverage for the chosen fix.

Yes. The central production auth failure is reproducible because both the canonical clawhub.ai and direct Convex GitHub sign-in endpoints return HTTP 500 instead of entering OAuth, although the later callback-without-session symptom cannot be replayed while sign-in fails earlier.
