## Summary

Keep open. This is an open ClawHub issue, cleanup policy only allows auto-closing implemented PRs, and current production checks still show the GitHub OAuth sign-in route failing before a session can be established.

## What This Changes

The issue reports that GitHub OAuth authorization on clawhub.ai redirects back to ClawHub but the UI remains signed out instead of refreshing the user's login state.

## Best Possible Solution

Keep a canonical OAuth/session tracker open, verify production GitHub sign-in for active and affected accounts, inspect private OAuth/env/log/cookie state, then land the smallest config, dependency, or source fix with focused regression and smoke coverage.

## Reproduction Assessment

Yes for the observable production login failure: live root and public API requests succeed, but both GitHub OAuth sign-in endpoints return HTTP 500. The exact callback-succeeds/no-session variant cannot be replayed until the sign-in endpoint redirects again.

## Solution Assessment

Unclear as a source-only fix. The narrowest maintainable path is maintainer/operator triage of OAuth credentials, Convex/Vercel env, callback cookies, logs, and account state before choosing a config, dependency, or code repair.
