## What This Changes

The issue reports that GitHub OAuth on clawhub.ai completes authorization but returns to ClawHub without refreshing session/user state, leaving the user signed out.

## Best Possible Solution

Restore production GitHub sign-in end to end for active accounts, preserve deleted/deactivated/banned account rejection, choose a canonical OAuth/session tracker for the cluster, and add browser-level OAuth/session smoke proof plus targeted regression coverage for the chosen fix.
