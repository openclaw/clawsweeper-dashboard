## Summary

Keep open: this is a concrete, corroborated production GitHub OAuth/session failure, and current live endpoints still fail before a healthy GitHub redirect. ClawHub cleanup policy also keeps issues open unless a maintainer resolves them explicitly.

## What This Changes

The issue reports that GitHub OAuth authorization returns to clawhub.ai but the UI remains logged out because session/auth state does not refresh.

## Best Possible Solution

Restore production GitHub OAuth end to end for active accounts, preserve blocked-account rejection and user-facing errors, then add browser-level OAuth/session smoke proof plus targeted regression coverage for the confirmed root cause.

## Reproduction Assessment

Yes. The central production sign-in failure reproduces because both the canonical clawhub.ai sign-in endpoint and direct Convex GitHub sign-in endpoint return HTTP 500 on the currently tagged production main; the later callback no-session symptom cannot be replayed until sign-in redirects correctly.

## Solution Assessment

Yes for keeping this open and routing it to maintainer/operator auth triage. The safest fix depends on production OAuth configuration, Convex/Vercel logs, cookies/JWT behavior, and blocked-account policy rather than a source-only cleanup PR.
