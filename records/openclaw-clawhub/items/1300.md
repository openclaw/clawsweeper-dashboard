## Summary

Keep open under the ClawHub issue cleanup policy. Current main and the v0.15.0 CLI tag already contain the object-body fixes for transfer, rename, and merge, so there is no remaining ClawSweeper repair lane for this issue.

## What This Changes

The issue reports that CLI transfer, rename, and merge commands double-serialize JSON request bodies, causing required API fields to be missing.

## Best Possible Solution

Keep the v0.15.0 object-body implementation and ask for a fresh current-CLI reproduction only if transfer, rename, or merge still fail.

## Reproduction Assessment

No, not against current main. The original v0.9.0 report and follow-up reproduction were credible, but the current command handlers and focused tests no longer contain the pre-stringified payload path; this read-only review did not run live authenticated production commands.

## Solution Assessment

Yes. Passing plain objects from the affected command handlers is the narrow maintainable fix for the required-field failures, and it matches the server's parsed object-field contract; defensive string-body handling in the shared HTTP client would be optional hardening.
