## Summary

Keep open under the ClawHub cleanup policy for issues, but the reported CLI double-serialization bug is fixed on current main and in the v0.15.0+ CLI line. The related closed PRs and issue comments correctly identified the root cause; no repair lane is needed now.

## What This Changes

The issue reports CLI transfer request, skill rename, and skill merge commands failing because command handlers pre-serialized JSON bodies before the shared HTTP helper serialized them again.

## Best Possible Solution

Keep the current object-body implementation, direct affected users to CLI v0.15.0 or later, and leave final issue closure to maintainers under the ClawHub issue cleanup policy.

## Reproduction Assessment

No, not against current main. The original v0.9.0 failure is credible, but current source sends object payloads and focused tests assert object bodies for transfer, rename, and merge.

## Solution Assessment

Yes. Passing plain objects from the affected command handlers while keeping `apiRequest` as the JSON serialization boundary is the narrow maintainable fix, and current main implements it.
