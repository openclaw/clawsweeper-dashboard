## Summary

Keep open under the ClawHub issue cleanup policy for issues, but the reported CLI double-serialization bug is fixed on current main and shipped in v0.15.0. No ClawSweeper repair lane is needed unless someone provides a fresh reproduction on v0.15.0 or newer.

## What This Changes

The issue asks to stop double JSON serialization for `transfer request`, `skill rename`, and `skill merge` CLI POST bodies.

## Best Possible Solution

Keep the v0.15.0 object-body implementation and ask for a fresh live reproduction only if transfer, rename, or merge still fail on v0.15.0 or newer.

## Reproduction Assessment

No. The original v0.9.0 failure path is clear, but current main no longer sends pre-stringified command payloads and focused tests assert object bodies for the affected commands.

## Solution Assessment

Yes. Passing plain objects from the affected command handlers is the narrow maintainable fix for the required-field failures; shared HTTP string-body tolerance would be optional hardening rather than required repair.
