## Summary

The reported CLI double-serialization bug is fixed on current main and shipped in the CLI release line starting with v0.15.0; transfer, rename, and merge now pass object bodies while the HTTP helper remains the serialization boundary.

## What This Changes

The issue asks for `transfer request`, `skill rename`, and `skill merge` to stop passing pre-serialized JSON strings into `apiRequest`.

## Best Possible Solution

Keep the current object-body command implementation and direct affected users on CLI v0.9.0 or stale caches to upgrade to v0.15.0 or later.

## Reproduction Assessment

No, not against current main. The old v0.9.0 failure is credible from the report and follow-up comments, but current source and focused tests now use object payloads for all three affected commands.

## Solution Assessment

Yes. Passing plain objects from the command handlers while keeping `apiRequest` as the JSON serialization boundary is the narrow maintainable fix, and current main plus v0.15.0 implement it.
