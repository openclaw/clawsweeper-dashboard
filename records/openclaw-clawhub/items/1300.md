## What This Changes

The issue reports that transfer request, skill rename, and skill merge sent pre-stringified JSON bodies into an HTTP helper that stringifies request bodies itself.

## Best Possible Solution

## Reproduction Assessment

No. Current main no longer has the reported double-serialization path: the affected command handlers pass object bodies, and the focused tests assert those object payloads.

## Solution Assessment

Yes. Passing plain objects from the affected command handlers and leaving apiRequest as the JSON serialization boundary is the narrow maintainable fix; shared string-body tolerance would be optional hardening.
