## Summary

Keep open under the ClawHub issue policy: this is an issue, not a PR, even though current main and the v0.15.0 CLI tag already fix the reported transfer, rename, and merge double-serialization bug.

## What This Changes

The issue reports that `transfer request`, `skill rename`, and `skill merge` pass pre-stringified JSON bodies that the shared HTTP helper serializes a second time.

## Best Possible Solution

Keep the current object-body command boundary, treat the reported bug as fixed in v0.15.0, and only reopen engineering work if someone reproduces the same failures on v0.15.0 or newer with live output.

## Reproduction Assessment

No current-main reproduction path remains. The historical v0.9.0 reports are credible, but current main sends object bodies and tests assert that boundary for all three affected commands.

## Solution Assessment

Yes. Passing plain objects from the affected command handlers is the narrow maintainable fix for this bug; defensive string-body passthrough in the HTTP helper would be optional hardening rather than required for the reported commands.
