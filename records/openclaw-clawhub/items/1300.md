## Summary

Keep open. Current main still has the reported double-serialization path for transfer request, skill rename, and skill merge; the related fix PRs were closed without merging, and ClawHub policy keeps issues open for maintainer follow-up.

## What This Changes

The issue reports that transfer request, skill rename, and skill merge pre-serialize JSON request bodies before the shared HTTP client serializes them again.

## Best Possible Solution

Ship a focused CLI bug fix that restores the plain-object `apiRequest` contract for these commands, adds regression coverage, and preserves the existing API/server behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows the commands pass JSON strings, both HTTP runtime paths stringify again, and the server handlers read object fields that will be missing after double encoding.

## Solution Assessment

Yes. The narrow maintainable fix is to pass plain objects from the affected commands, update tests to assert object payloads, and optionally harden the shared HTTP serializer with coverage for both Node and Bun/curl paths.
