Keep open under the ClawHub issue-cleanup policy: the reported v0.9.0 double-serialization bug is fixed on current main and in the v0.15.0+ CLI line, but this workflow does not auto-close ClawHub issues.

## What This Changes

The issue reports that transfer request, skill rename, and skill merge pre-serialized JSON bodies before passing them to apiRequest, causing required-field errors after a second serialization pass.

## Best Possible Solution

Keep the current command-side object-body implementation, and let maintainers manually close or mark the issue resolved once they are satisfied that v0.15.0 or later is the supported retry path.

## Reproduction Assessment

No, not against current main. The original v0.9.0 reports remain credible, but current source sends object payloads and focused tests assert object shape for all three reported commands.

## Solution Assessment

Yes. Passing plain objects from the affected command handlers while keeping apiRequest as the serialization boundary is the narrow maintainable fix for the reported failures.
