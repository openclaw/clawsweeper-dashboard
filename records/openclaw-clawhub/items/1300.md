## Summary

Keep open under the ClawHub issue policy even though current main and the v0.15.0 changelog show the reported CLI payload bug is fixed; ClawHub cleanup only auto-closes implemented pull requests, not issues.

## What This Changes

The issue reports that transfer request, skill rename, and skill merge pre-stringify POST bodies before the shared HTTP helper stringifies them again, causing required-field API errors.

## Best Possible Solution

Treat the reported command bug as fixed in v0.15.0 and leave any issue closure to maintainer-controlled cleanup unless someone reproduces the same failures on v0.15.0 or newer.

## Reproduction Assessment

No. On current main, the three affected command handlers pass object bodies and their tests assert object payloads, so the reported double-serialization path is no longer present in source.

## Solution Assessment

Yes. Passing plain objects from the affected command call sites is the narrow maintainable fix for the reported failures; shared HTTP string-body tolerance would be broader hardening, not required for this issue.
