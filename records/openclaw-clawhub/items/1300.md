## Summary

Keep open under the ClawHub issue policy, and current main still source-reproduces the reported CLI bug: the affected commands pass JSON strings into an HTTP helper that serializes request bodies again, while the server expects object fields.

## What This Changes

The issue asks to fix double JSON serialization in `transfer request`, `skill rename`, and `skill merge` CLI requests.

## Best Possible Solution

Land a focused CLI fix that sends plain object bodies from the affected commands, updates tests to assert that contract, and keeps `apiRequest` as the JSON serialization boundary.

## Reproduction Assessment

Yes, by source inspection on current main. The commands pass strings, both HTTP transports stringify again, and the server reads object fields, which explains the reported required-field errors without needing a live authenticated run.

## Solution Assessment

Yes. Passing plain objects from the command handlers is the narrow maintainable fix; optional HTTP tolerance is acceptable only with coverage for both runtime paths, while server-side parsing of double-encoded strings would mask the client contract bug.
