## Summary

Keep open. Current main still has the reported double-serialization path for transfer request, skill rename, and skill merge; the closed fix PRs were not merged, and the open ownership-transfer PR only overlaps the transfer path while remaining broad and conflicting.

## What This Changes

Fix the CLI transfer request, skill rename, and skill merge commands so they send plain object request bodies instead of pre-stringified JSON.

## Best Possible Solution

Land a focused CLI bug fix that restores the plain-object apiRequest contract for transfer, rename, and merge, with regression tests for command payload shape and both HTTP serialization paths if the helper is hardened.

## Reproduction Assessment

Yes, by source inspection. The affected commands pass JSON strings, both HTTP transports stringify again, and the API handlers require object fields that will be absent after double encoding.

## Solution Assessment

Yes. The narrow maintainable fix is to pass plain objects from the command handlers and update tests to assert that boundary; server-side reparsing would mask the client contract bug.
