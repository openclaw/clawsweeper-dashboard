## Summary

Keep open under the ClawHub issue cleanup policy, but the reported CLI double-serialization bug is fixed on current main and in the v0.15.0+ CLI line. The comments and unmerged PRs correctly identified the root cause; no repair lane is needed now.

## What This Changes

The issue reports that `transfer request`, `skill rename`, and `skill merge` sent pre-stringified JSON bodies to `apiRequest`, causing required-field errors after a second serialization.

## Best Possible Solution

Keep the current object-body implementation, direct affected users to retry with v0.15.0 or later, and let maintainers close the issue manually when satisfied.

## Reproduction Assessment

No, not against current main. The original v0.9.0 reports are credible, but current source sends object payloads and focused tests assert the object shape for all three reported commands.

## Solution Assessment

Yes. Passing plain objects from the affected command handlers while keeping `apiRequest` as the JSON serialization boundary is the narrow maintainable fix, and current main implements it.
