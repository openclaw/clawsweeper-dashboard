## Summary

Keep open under the ClawHub issue cleanup policy: the reported v0.9.0 bug is fixed on current main and in the v0.15.0 CLI tag, but ClawHub issue cleanup does not auto-close issues.

## What This Changes

The issue reports that CLI transfer request, skill rename, and skill merge commands sent pre-stringified JSON bodies that the shared HTTP helper serialized again.

## Best Possible Solution

Keep the v0.15.0 object-body implementation and request a fresh current-CLI reproduction only if these commands still fail.

## Reproduction Assessment

No, not against current main. The original v0.9.0 and follow-up reports were credible, but current command handlers and focused tests no longer contain the pre-stringified payload path.

## Solution Assessment

Yes. Passing plain objects from the affected command handlers is the narrow maintainable fix for the required-field failures; defensive string-body handling in the shared HTTP client would be optional hardening.
