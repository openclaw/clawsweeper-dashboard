## Summary

Keep open. Current main still double-serializes the POST bodies for transfer request, skill rename, and skill merge, the server still expects parsed object fields, and the related fix PRs were closed without merging.

## What This Changes

The issue requests fixing CLI transfer request, skill rename, and skill merge POST bodies that are JSON-stringified before being passed to the shared HTTP client.

## Best Possible Solution

Land a focused CLI fix that sends plain object bodies from the affected commands, updates command tests to assert object payloads, and covers any shared HTTP hardening in both runtime paths.

## Reproduction Assessment

Yes. The source-level path is high confidence: the affected commands pass string bodies, both HTTP runtime paths stringify again, and the server handlers require object fields; I did not invoke live authenticated registry mutations.

## Solution Assessment

Yes. The narrow maintainable fix is to remove command-level pre-serialization and update regression tests; shared HTTP string-body tolerance is reasonable only if covered for both Node and Bun/curl paths.
