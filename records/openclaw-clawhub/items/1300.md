## Summary

Keep open under the ClawHub issue policy, but the reported CLI double-serialization bug appears fixed on current main and tagged for CLI v0.15.0; no repair-lane work is needed unless the same transfer, rename, or merge failures reproduce on v0.15.0 or newer.

## What This Changes

The issue reports that `transfer request`, `skill rename`, and `skill merge` pre-stringified JSON bodies before passing them to the shared CLI HTTP helper, causing required-field API errors.

## Best Possible Solution

Keep the v0.15.0 object-body implementation and only reopen engineering work if a fresh report reproduces the same commands failing on v0.15.0 or newer with live output.

## Reproduction Assessment

No. On current main, the three affected command handlers pass object bodies and the focused command tests assert those object payloads, so the reported double-serialization path is no longer present in source.

## Solution Assessment

Yes. Passing plain objects from the affected command handlers is the narrow maintainable fix for the reported failures; shared HTTP tolerance for pre-serialized bodies would be optional hardening rather than required for these commands.
