## Summary

Keep open. Current main still contains the reported double-serialization path for transfer request, skill rename, and skill merge; the related repair PRs #973, #1251, and #1365 were closed unmerged, while open #1635 only overlaps the transfer side of the bug.

## What This Changes

The issue requests fixing CLI transfer request, skill rename, and skill merge so they pass plain object bodies to `apiRequest` instead of pre-stringified JSON.

## Best Possible Solution

Land or split a focused fix that sends object bodies for all three affected commands, updates tests to assert object payloads, and coordinates with #1635 so rename and merge do not remain broken.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the affected commands pass string bodies, both HTTP runtime paths stringify again, and the API handlers read object fields from the parsed payload.

## Solution Assessment

Yes. The narrow maintainable solution is to remove command-level pre-serialization and update tests; any shared HTTP tolerance should be covered in both transport paths.
