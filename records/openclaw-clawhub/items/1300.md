## Summary

Keep open. Current main still contains the reported CLI double-serialization path for transfer request, skill rename, and skill merge, and the related repair PRs in the discussion were closed without merging.

## What This Changes

The issue asks to stop double JSON-serializing transfer request, skill rename, and skill merge POST bodies in the ClawHub CLI.

## Best Possible Solution

Send plain object bodies from the affected CLI commands, update command tests to assert object payloads, optionally harden both HTTP runtime paths, and add an Unreleased changelog fix.

## Reproduction Assessment

Yes. Source inspection on current main shows transfer, rename, and merge pass string bodies to apiRequest, both HTTP paths JSON.stringify args.body, and server handlers read object fields; I did not run live authenticated registry mutations.

## Solution Assessment

Yes. Removing command-level pre-serialization is the narrowest maintainable fix; shared HTTP string-body tolerance is reasonable only if covered for both Node and Bun/curl paths.
