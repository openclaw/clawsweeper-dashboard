## Summary

Keep open under the ClawHub issue policy for issues, but the reported CLI double-serialization path is fixed on current main and included in the v0.15.0 CLI tag; no repair-lane work is needed unless the failure reproduces on v0.15.0 or newer.

## What This Changes

The issue reports that transfer request, skill rename, and skill merge pre-serialize JSON bodies before passing them to the CLI HTTP helper, causing required fields to be missing after a second serialization.

## Best Possible Solution

Keep the v0.15.0 object-body implementation and ask for a fresh live repro only if transfer, rename, or merge still fail on v0.15.0 or newer.

## Reproduction Assessment

No. Current main no longer has the reported pre-stringified command payloads, and focused command tests assert object bodies for transfer, rename, and merge.

## Solution Assessment

Yes. Passing plain objects from the affected command handlers is the narrow maintainable fix for the reported failures; HTTP tolerance for pre-serialized bodies would be optional hardening.
