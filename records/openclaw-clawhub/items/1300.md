## Summary

Keep open under the ClawHub issue cleanup policy: the reported v0.9.0 double-serialization paths are fixed on current main and in the v0.15.0 CLI tag, but this workflow does not auto-close ClawHub issues.

## What This Changes

The issue reports that CLI transfer request, skill rename, and skill merge pre-serialized JSON request bodies before calling the shared HTTP helper, causing required-field API errors.

## Best Possible Solution

Keep the current object-body implementation and ask for a fresh current-CLI reproduction only if these commands still fail in v0.15.0 or later.

## Reproduction Assessment

No, not against current main. The original v0.9.0 reports were credible, but current command handlers and focused tests no longer contain the pre-stringified payload path.

## Solution Assessment

Yes. Passing plain objects from the affected command handlers is the narrow maintainable fix for the required-field failures; defensive string-body handling in the shared HTTP helper would be optional hardening.
