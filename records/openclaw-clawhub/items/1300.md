## What This Changes

The issue reports that CLI transfer request, skill rename, and skill merge pre-stringify JSON bodies before `apiRequest` serializes them again, producing required-field server errors.

## Best Possible Solution

Keep the object-body command implementations and ask for a fresh report only if these commands still fail on CLI v0.15.0 or later.

## Reproduction Assessment

No, not against current main. The current command handlers pass plain objects to `apiRequest`, focused tests assert object payloads, and v0.15.0 contains the full transfer/rename/merge fix.

## Solution Assessment

Yes. Passing plain objects from the affected command handlers and leaving `apiRequest` as the JSON serialization boundary is the narrow maintainable fix; defensive string-body handling would be optional hardening.
