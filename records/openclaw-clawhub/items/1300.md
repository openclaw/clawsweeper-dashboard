## Summary

Keep open under the ClawHub issue cleanup policy. Current main and the v0.15.0 tag already contain the object-body fixes for transfer, rename, and merge, so there is no repair lane left for this issue.

## What This Changes

The issue reports CLI transfer, skill rename, and skill merge failing because command handlers pre-serialized JSON bodies before the shared HTTP client serialized them again.

## Best Possible Solution

Keep the v0.15.0 object-body implementation and ask for a fresh current-CLI reproduction only if transfer, rename, or merge still fail.

## Reproduction Assessment

No. Current main no longer contains the reported pre-stringified payloads, and focused command tests assert object bodies for transfer, rename, and merge; this review did not run live authenticated production commands.

## Solution Assessment

Yes. Passing plain objects from the affected command handlers is the narrow maintainable fix for the required-field failures; defensive string-body handling in the shared HTTP client would be optional hardening.
