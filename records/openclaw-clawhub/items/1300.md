## What This Changes

The issue reports that transfer request, skill rename, and skill merge passed pre-stringified JSON bodies to an HTTP helper that serializes bodies again.

## Best Possible Solution

Keep the v0.15.0 object-body implementation and ask for a fresh live reproduction only if transfer, rename, or merge still fail on v0.15.0 or newer.

## Reproduction Assessment

No. Current main no longer has the reported pre-stringified command payloads, and focused tests assert object bodies for transfer, rename, and merge; I did not run a live authenticated ownership command.

## Solution Assessment

Yes. Passing plain objects from the affected command handlers is the narrow maintainable fix for the required-field failures; HTTP string-body tolerance would be optional hardening rather than necessary repair.
