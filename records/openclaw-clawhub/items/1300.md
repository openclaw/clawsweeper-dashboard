## Repair Work Prompt

Fix the CLI double-JSON serialization bug reported in https://github.com/openclaw/clawhub/issues/1300. On current main, `cmdTransferRequest`, `cmdRenameSkill`, and `cmdMergeSkill` pass `JSON.stringify(...)` strings to `apiRequest`, while both HTTP transports serialize `args.body` again; the server then cannot read `toUserHandle`, `newSlug`, or `targetSlug`. Change the affected command handlers to pass plain object bodies, update `transfer.test.ts` and `ownership.test.ts` to assert object equality rather than string substrings, and add/update HTTP tests only if you also introduce defensive string-body handling. Keep the fix focused: do not implement package/org transfer behavior from https://github.com/openclaw/clawhub/pull/1635, do not change transfer authorization, recipient acceptance, ownership checks, or audit semantics, and do not touch unrelated raw `fetch` callers that intentionally call `JSON.stringify`. Include a concise bug-fix changelog entry only if the repository’s current release workflow expects one for CLI user-facing fixes.
