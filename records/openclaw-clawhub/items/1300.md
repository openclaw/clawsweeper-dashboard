Keep open under the ClawHub cleanup policy for issues. The reported serialization bug appears fixed on current main and in the v0.15.0 CLI release, so there is no code repair lane left from this item.

## What This Changes

The issue reports that transfer request, skill rename, and skill merge commands double-serialized JSON request bodies before sending them to the API.

## Best Possible Solution

Keep the released object-body implementation and have a maintainer close this issue manually as fixed, or ask for a fresh current-version report if any transfer, rename, or merge path still fails.

## Reproduction Assessment

No. The original v0.9.0 reproduction was valid, but current main no longer has the double-serialization path: the affected commands pass object bodies and tests assert that boundary.

## Solution Assessment

Yes. Passing plain objects from the affected command handlers keeps `apiRequest` as the single JSON serialization boundary and is narrower than teaching the server to accept double-encoded request strings.
