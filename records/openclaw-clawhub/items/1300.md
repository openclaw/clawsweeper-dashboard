## Summary

Keep open under the ClawHub issue policy even though current main and v0.15.0 now fix the reported CLI payload shape. The earlier linked fix PRs were closed unmerged, but equivalent transfer, rename, and merge changes landed directly afterward.

## What This Changes

The issue reports that CLI transfer request, skill rename, and skill merge pre-stringify JSON request bodies before `apiRequest`, causing required fields to be lost after double serialization.

## Best Possible Solution

Treat the command bug as fixed in v0.15.0 and only reopen engineering work if someone can reproduce the same failures on v0.15.0 or newer with live CLI output.

## Reproduction Assessment

No. Against current main, the three affected command handlers pass plain object bodies and the command tests assert object payloads, so the reported double-serialization path is no longer present in source.

## Solution Assessment

Yes. Passing plain objects from the affected commands is the narrow maintainable fix for the reported failures; HTTP string-body tolerance is optional broader hardening, not required for these commands.
