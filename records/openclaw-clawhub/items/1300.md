Keep open under the ClawHub issue cleanup policy: the reported CLI serialization bug is fixed on current main and shipped in v0.15.0, but this workflow does not auto-close ClawHub issues.

## What This Changes

The issue reports that transfer request, skill rename, and skill merge passed pre-stringified JSON bodies into `apiRequest`, causing double serialization and required-field API errors.

## Best Possible Solution

Keep the current object-body implementation and let maintainers manually close this issue once they are satisfied that v0.15.0 or newer is the supported retry path.

## Reproduction Assessment

No, not against current main from source inspection. The original v0.9.0 report and later reproduction comments were credible, but the current command handlers and tests no longer contain the pre-stringified payload path for the three reported commands.

## Solution Assessment

Yes. Passing plain objects from the affected command handlers is the narrow maintainable fix for the required-field failures; defensive string-body handling in the shared HTTP helper remains optional hardening rather than necessary for the reported current call sites.
