## What This Changes

The issue reports that transfer request, skill rename, and skill merge commands double-serialize JSON request bodies and fail with required-field errors.

## Best Possible Solution

Keep the current object-body command implementation and direct affected CLI users to upgrade to v0.15.0 or later.

## Reproduction Assessment

No, not against current main. The original CLI v0.9.0 failure is credible, but current source and focused tests show object payloads for transfer, rename, and merge.

## Solution Assessment

Yes. Passing plain objects from the affected command handlers while keeping apiRequest as the JSON serialization boundary is the narrow maintainable fix, and current main implements it.
