## Summary

Keep open. Current main still source-reproduces the reported CLI serialization bug: transfer request, skill rename, and skill merge pass JSON strings into an HTTP helper that serializes request bodies again, while the server handlers require object fields. The closest complete fix, https://github.com/openclaw/clawhub/pull/1251, was closed without merging, and the broader open ownership-transfer PR is conflicting and does not replace a focused repair.

## What This Changes

Fix CLI transfer request, skill rename, and skill merge so request bodies are serialized once and arrive at the API as JSON objects.

## Best Possible Solution

Land a focused CLI repair that sends plain object bodies from the affected commands, updates tests to assert object payloads, and keeps broader org/package ownership-transfer work separate.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: the affected commands pass strings, both HTTP transports stringify again, and the server handlers require object fields.

## Solution Assessment

Yes. Passing plain objects from the affected command handlers is the narrow maintainable fix; optional HTTP string-body tolerance is acceptable only with coverage for both runtime serialization paths.

Fix the current-main CLI double-JSON-serialization bug for transfer request, skill rename, and skill merge. In `packages/clawhub/src/cli/commands/transfer.ts` and `packages/clawhub/src/cli/commands/ownership.ts`, send plain object bodies to `apiRequest` instead of pre-stringifying. Update `packages/clawhub/src/cli/commands/transfer.test.ts` and `packages/clawhub/src/cli/commands/ownership.test.ts` so they assert object payloads, not string substrings. If you also harden `packages/clawhub/src/http.ts` to tolerate already-serialized strings, add focused coverage in both `packages/clawhub/src/http.test.ts` and `packages/clawhub/src/http.bun.test.ts`. Do not change ownership authorization, transfer acceptance semantics, org/package transfer modeling, or take on the broader work from https://github.com/openclaw/clawhub/pull/1635. Preserve `apiRequest` as the normal JSON serialization boundary and keep the fix scoped to this regression.
