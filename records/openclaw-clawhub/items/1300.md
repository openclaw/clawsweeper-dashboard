## Summary

Keep open. Current main still has the reported double JSON serialization path in transfer request, skill rename, and skill merge, and the related fix PRs were closed without merging; for ClawHub cleanup, issues stay open and this is a narrow repair candidate.

## What This Changes

The issue reports that three CLI commands pass pre-stringified JSON bodies to an HTTP helper that serializes request bodies again.

## Best Possible Solution

Land a focused CLI fix that sends plain object bodies from the affected commands, updates the command tests to assert object payloads, and keeps HTTP serialization behavior covered across both runtimes if the helper is hardened.

## Reproduction Assessment

Yes. The code-level path is deterministic on current main: the commands pass JSON strings, the HTTP helper stringifies again, and the server handlers read object fields; the issue comments also include CLI reproductions for rename and merge.

## Solution Assessment

Yes. The narrow maintainable solution is to remove pre-stringification at the three call sites and update tests; defensive string-body handling in apiRequest is acceptable only with coverage for both runtime paths.
