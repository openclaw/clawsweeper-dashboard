## Summary

Keep open under the ClawHub issue cleanup policy for issues, but the reported CLI double-serialization bug is fixed on current main and included in the v0.15.0 CLI tag. No repair-lane work is needed unless transfer, rename, or merge still fail on v0.15.0 or newer.

## What This Changes

The issue reports that CLI transfer request, skill rename, and skill merge passed pre-stringified JSON bodies to apiRequest, causing required-field errors after double serialization.

## Best Possible Solution

Keep the v0.15.0 object-body implementation and ask for a fresh live reproduction only if transfer, rename, or merge still fail on v0.15.0 or newer.

## Reproduction Assessment

No. Current main no longer has the reported pre-stringified command payloads, and focused command tests assert object bodies for transfer, rename, and merge.

## Solution Assessment

Yes. Passing plain objects from the affected command handlers is the narrow maintainable fix for the reported required-field failures; HTTP string-body tolerance would be optional hardening rather than necessary repair.
