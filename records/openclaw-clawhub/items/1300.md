## Summary

Keep open under the ClawHub issue policy for issues: current main and the v0.15.0 CLI tag appear to have fixed the reported double-serialization path, but this cleanup workflow only auto-closes pull requests in this repository.

## What This Changes

The issue reports that transfer request, skill rename, and skill merge commands pre-serialized JSON bodies before passing them to an HTTP helper that serializes bodies again, causing required API fields to be missing.

## Best Possible Solution

Keep the shipped v0.15.0 object-body implementation and ask for a fresh live reproduction only if transfer, rename, or merge still fail on v0.15.0 or newer.

## Reproduction Assessment

No. Current main no longer has the reported pre-stringified command payloads, and focused tests assert object bodies for transfer, rename, and merge.

## Solution Assessment

Yes. Passing plain objects from the affected command handlers is the narrow maintainable fix for the reported failures; HTTP tolerance for pre-serialized bodies would be optional hardening.
