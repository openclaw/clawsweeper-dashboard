Keep open under the ClawHub issue policy even though current main and the v0.15.0 tag now fix the reported CLI payload shape for transfer request, skill rename, and skill merge. No repair lane is needed unless someone can reproduce the same failures on v0.15.0 or newer.

## What This Changes

The issue reports that transfer request, skill rename, and skill merge send pre-stringified JSON bodies that the shared HTTP helper serializes again.

## Best Possible Solution

Treat the command bug as fixed in v0.15.0 and only reopen engineering work if someone reproduces the same failures on v0.15.0 or newer with live CLI output.

## Reproduction Assessment

No. Against current main, the three affected command handlers pass plain object bodies and the command tests assert object payloads, so the reported double-serialization path is no longer present in source.

## Solution Assessment

Yes. Passing plain objects from the affected commands is the narrow maintainable fix for the reported failures; HTTP string-body tolerance is optional broader hardening, not required for these commands.
