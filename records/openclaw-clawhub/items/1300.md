## Summary

Keep open. Current main still has the reported CLI serialization bug: transfer request, skill rename, and skill merge pass JSON strings into an HTTP helper that serializes request bodies again, while the server handlers require object fields. The closest complete fix, https://github.com/openclaw/clawhub/pull/1251, was closed without merging, and the open ownership-transfer PR only overlaps part of the transfer path while remaining broad and conflicting.

## What This Changes

The issue reports that transfer request, skill rename, and skill merge fail because their CLI handlers double-encode JSON request bodies before sending them to the API.

## Best Possible Solution

Land a focused CLI fix that restores apiRequest as the single JSON serialization boundary, updates transfer and ownership command tests to assert object payloads, and avoids folding in the broader ownership-transfer redesign.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: the affected commands pass strings, both HTTP transports stringify again, and the server handlers require object fields.

## Solution Assessment

Yes. Passing plain objects from the affected command handlers is the narrow maintainable fix; optional HTTP tolerance is acceptable only with coverage for both runtime serialization paths.
