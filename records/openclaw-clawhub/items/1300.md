## What This Changes

The issue reports that transfer request, skill rename, and skill merge fail because their CLI handlers pre-serialize JSON bodies before the shared HTTP helper serializes them again.

## Best Possible Solution

Keep the current object-body implementation and have maintainers manually close only after they are satisfied that v0.15.0 or later is the supported retry path.

## Reproduction Assessment

No, not against current main. The original v0.9.0 report and later reproduction comments were credible, but the current command handlers and tests no longer contain the pre-stringified payload path for the three reported commands.

## Solution Assessment

Yes. Passing plain objects from the affected command handlers is the narrow maintainable fix for the required-field failures; defensive string-body handling in the shared HTTP helper remains optional hardening rather than necessary for the reported current call sites.
