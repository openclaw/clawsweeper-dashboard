## Summary

Keep open under the ClawHub cleanup policy for issues, but the reported CLI double-serialization bug is fixed on current main and in the v0.15.0 CLI tag; no repair lane is needed unless transfer, rename, or merge still fail on v0.15.0 or newer.

## What This Changes

The issue reports that CLI transfer request, skill rename, and skill merge sent pre-stringified JSON bodies that were serialized again by the HTTP helper, causing required-field API errors.

## Best Possible Solution

Keep the v0.15.0 object-body implementation and ask for a fresh live reproduction only if the affected commands still fail on v0.15.0 or newer.

## Reproduction Assessment

No. I did not run a live production transfer, rename, or merge, but current main no longer has the reported pre-stringified command payloads and focused tests assert object bodies for all three commands.

## Solution Assessment

Yes. Passing plain objects from the affected command handlers is the narrow maintainable fix for the required-field failures; HTTP string-body tolerance would be optional future hardening rather than necessary repair.
