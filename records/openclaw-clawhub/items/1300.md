## Summary

Keep open under the ClawHub issue policy even though current main fixes the reported transfer, rename, and merge payload serialization bug and the complete fix is tagged in v0.15.0.

## What This Changes

The issue reports that transfer request, skill rename, and skill merge passed pre-stringified JSON bodies into `apiRequest`, causing required-field API errors after a second serialization pass.

## Best Possible Solution

Treat the reported CLI serialization bug as fixed in v0.15.0 and only reopen engineering work if someone reproduces the same failures on v0.15.0 or newer with live output.

## Reproduction Assessment

No. Against current main, the three affected command handlers pass plain object bodies and tests assert those object payloads, so the reported double-serialization path is no longer present in source.

## Solution Assessment

Yes. Passing plain objects from the affected command handlers is the narrow maintainable fix for the reported failures; HTTP string-body tolerance would be broader hardening, not required for these commands.
