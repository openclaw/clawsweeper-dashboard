## Summary

Keep open under the ClawHub issue cleanup policy, but the reported CLI double-serialization bug is fixed on current main and in the v0.15.0+ CLI line. The original report, follow-up reproduction, and closed unmerged PRs correctly identified the historical root cause; current source and focused tests now use object payloads for all three commands.

## What This Changes

The issue reports that `transfer request`, `skill rename`, and `skill merge` pre-stringified JSON bodies before calling `apiRequest`, causing server required-field errors.

## Best Possible Solution

Keep the current object-body implementation, point affected users to v0.15.0 or later, and leave manual closure to maintainers under the ClawHub issue cleanup policy.

## Reproduction Assessment

No, not against current main. The original v0.9.0 reports are credible, but current source sends object payloads and focused tests assert object bodies for transfer, rename, and merge.

## Solution Assessment

Yes. Passing plain objects from the affected command handlers while keeping `apiRequest` as the JSON serialization boundary is the narrow maintainable fix, and current main implements it.
