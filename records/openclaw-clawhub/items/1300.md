## Summary

Current main and the v0.15.0 CLI tag appear to have fixed the reported double-serialization bug, but this is an issue and the ClawHub cleanup policy keeps issues open rather than auto-closing them.

## What This Changes

The issue reports that `transfer request`, `skill rename`, and `skill merge` pass pre-stringified JSON bodies to `apiRequest`, causing required-field API failures from double serialization.

## Best Possible Solution

Keep the v0.15.0 object-body implementation and ask for a fresh current-CLI reproduction only if transfer, rename, or merge still fail.

## Reproduction Assessment

No. Current main no longer contains the reported pre-stringified payloads, and focused command tests assert object bodies for transfer, rename, and merge; I did not run live authenticated production commands.

## Solution Assessment

Yes. Passing plain objects from the affected command handlers is the narrow maintainable fix for the required-field failures; shared HTTP string-body tolerance remains optional hardening, not necessary repair.
