## What This Changes

The issue reports a CLI bug where transfer request, skill rename, and skill merge pre-stringified JSON request bodies before passing them to `apiRequest`, causing required-field API errors.

## Best Possible Solution

Keep the current object-body command implementation and direct affected CLI users to upgrade to v0.15.0 or later.

## Reproduction Assessment

No, not against current main. The original v0.9.0 failure is credible, but current source sends object payloads and focused tests assert object bodies for transfer, rename, and merge.

## Solution Assessment

Yes. Passing plain objects from the affected command handlers while keeping `apiRequest` as the JSON serialization boundary is the narrow maintainable fix, and current main implements it.
