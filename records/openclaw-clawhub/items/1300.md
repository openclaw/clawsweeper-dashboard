## Summary

Close as implemented on main: the affected CLI commands now pass plain object payloads to `apiRequest`, focused tests assert that contract, and the complete transfer/rename/merge fix is included in the v0.15.0 CLI line and later releases.

## What This Changes

The issue reports that CLI transfer request, skill rename, and skill merge pre-serialized JSON bodies before `apiRequest`, causing required-field API errors after double serialization.

## Best Possible Solution

Keep the current object-body implementation and direct affected users to CLI v0.15.0 or later.

## Reproduction Assessment

No, not against current main. The original v0.9.0 failure is credible, but current source sends object payloads and focused tests assert object bodies for transfer, rename, and merge.

## Solution Assessment

Yes. Passing plain objects from the affected command handlers while keeping `apiRequest` as the JSON serialization boundary is the narrow maintainable fix, and current main implements it.
