## Summary

Keep open: current main and v0.21.0 still fold the API-key classifier output down to a boolean badge, while the live discussion treats an auditable rationale as future work that needs product/security review.

## What This Changes

The issue requests an auditable explanation for why a skill is flagged as requiring an API key or equivalent secret.

## Best Possible Solution

Define a security-reviewed explanation model with stable reason codes and bounded sanitized evidence, then persist and expose it where publishers can diagnose the badge.

## Reproduction Assessment

Yes, at source level. Current main computes or receives rationale during evaluation, but persists and renders only the boolean classification, matching the missing auditability described in the report.

## Solution Assessment

Yes in intent, but not as raw model text. The safer maintainable solution is structured reason codes plus sanitized bounded evidence with an explicit disclosure policy.
