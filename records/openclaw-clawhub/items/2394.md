## Label Justifications

## Summary

Keep open: current main and v0.21.0 still collapse the API-key classifier output to `apiKeyRequired` and render a generic badge, while the live discussion treats exposing rationale as useful future work that needs a product/security decision.

## What This Changes

The issue requests an auditable rationale explaining why the “API key required” classifier flagged a skill.

## Best Possible Solution

Define a security-reviewed, versioned explanation model with stable reason codes and bounded sanitized evidence, persist it with the verdict, and render the safe explanation on the relevant skill/audit surfaces.

## Reproduction Assessment

Yes, at source level. Current main computes or returns rationale during evaluation but persists and renders only the boolean classification, directly matching the missing auditability described in the report.

## Solution Assessment

Yes in intent, but not as raw model text. The safer maintainable solution is structured reason codes plus sanitized bounded evidence with an explicit public-versus-staff disclosure policy.
