## What This Changes

Request to make the reason behind an `API key required` skill classification auditable to skill publishers or reviewers.

## Best Possible Solution

Define a security-reviewed, versioned explanation model with stable reason codes and bounded sanitized evidence, persist it with the verdict, and render it only on the approved audit surfaces.

## Reproduction Assessment

Yes, at source level. Current main computes or receives rationale during API-key evaluation but persists and renders only the boolean classification, which directly reproduces the missing auditability.

## Solution Assessment

Yes in intent, but not as raw model text. The safer maintainable path is structured reason codes plus sanitized bounded evidence with an explicit public-versus-staff disclosure policy.
