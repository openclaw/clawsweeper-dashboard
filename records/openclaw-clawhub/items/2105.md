## Summary

Keep this open because it is an active security-moderation support question, not a stale source-level bug. Current code and docs still make ClawScan artifact-based, and they do not yet answer whether an external checkout confirmation page is sufficient evidence for high-impact commerce actions.

## What This Changes

The issue asks what artifact evidence the Shop skill should provide to clear a ClawScan Tool Misuse finding for checkout, returns, and reorder flows.

## Best Possible Solution

Define the artifact bar for commerce skills in the security docs/spec, then ask the publisher to update SKILL.md with the required confirmation, scope, OAuth, and transaction-limit evidence before requesting a rescan or appeal.
