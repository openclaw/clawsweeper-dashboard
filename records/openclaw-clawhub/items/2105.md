## Summary

Keep open: this is a current live moderation support question about whether Shop's checkout-confirmation flow is enough evidence to clear a Review flag, and current main plus the public scan show that a moderator decision is still needed rather than an automated close or repo repair.

## What This Changes

The issue asks what artifacts are sufficient to show user confirmation for the Shop skill's Buy Now, returns, and reorder flows.

## Best Possible Solution

Provide a moderator answer that acceptable evidence should explicitly show Buy Now only opens a merchant checkout confirmation page, cannot complete payment inside the agent, and requires per-transaction human confirmation with product, variant, quantity, price, shipping, payment context, return/reorder effects, OAuth scope, token handling, and any limits clearly documented.

## Solution Assessment

Yes, moderator guidance is the best current path because the repository already defines an artifact-based review standard and the remaining question is whether specific commerce evidence satisfies that standard.
