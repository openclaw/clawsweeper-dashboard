## Summary

Keep this open: this is a recent security-moderation support question about real-money commerce artifacts, and the reporter’s follow-up still needs a maintainer policy answer on whether external checkout confirmation is sufficient evidence.

## What This Changes

The issue asks what evidence is required to clear a ClawScan review finding for a Shopify Shop skill that links users to checkout/confirmation pages for buying, returns, and reordering.

## Best Possible Solution

A maintainer should answer or document a commerce-specific evidence checklist covering per-transaction confirmation, payment and shipping scope, OAuth scopes, limits, reversibility, and when to request a rescan or appeal.

## Reproduction Assessment

Not applicable as a bug reproduction. The live scan endpoint and published SKILL.md reproduce the flagged Review state and support question, but there is no failing current-main source behavior to execute.

## Solution Assessment

Unclear. Accepting checkout URLs alone is a security-policy decision; the safer maintainable path is to define artifact-visible commerce evidence before clearing this class of finding.
