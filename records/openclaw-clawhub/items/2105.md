## Summary

Keep open: this is a current security-moderation support question about whether checkout-page handoff is enough evidence for commerce authority, and current scanner/docs policy does not define that commerce-specific threshold.

## What This Changes

The issue asks what artifacts are needed to clear a Shop skill Review finding for buying, returns, and reorder behavior when Buy Now links hand off to merchant checkout pages.

## Best Possible Solution

A maintainer should answer or document a commerce-specific evidence checklist covering per-transaction confirmation, payment and shipping scope, OAuth scopes, limits, reversibility, and when to request a rescan or appeal.

## Reproduction Assessment

Not applicable as a bug reproduction. The issue discussion, live Shop artifact, and current scanner/docs source establish a policy/support question rather than a failing current-main code path.

## Solution Assessment

Unclear. Treating checkout URLs alone as sufficient may be reasonable only after maintainers define the artifact-visible commerce evidence threshold; the safer path is to document that threshold before clearing or changing scanner behavior.
