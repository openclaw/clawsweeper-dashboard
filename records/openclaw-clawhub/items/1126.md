## Summary

Keep open. ClawHub cleanup policy keeps issues open, and the reported behavior is still only partially resolved: production now returns `openclaw-checkpoint` for the exact slug, but the `checkpoint` keyword query still misses it. Current `main` has exact-slug injection plus bounded recent-digest fallback scans, while the remaining indexed direct lexical prefix path is tracked by open #1254 and proposed in open PR #1256.

## What This Changes

Issue requests that the published `openclaw-checkpoint` skill appear in ClawHub search results for its exact slug and the `checkpoint` keyword.

## Best Possible Solution

Land or supersede #1256/#1254 with an indexed direct lexical candidate path for normalized skill slug and display-name exact, token, and prefix matches, merged ahead of vector/recent fallback ranking while preserving existing filters.
