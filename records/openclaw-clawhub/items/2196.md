## Summary

Keep this PR open: the UI direction is useful, but the patch is not merge-ready because it breaks the local-auth publish lifecycle test and uses settings access rather than publish access for the new skill action.

## What This Changes

The PR adds manager-facing New version actions to skill and plugin detail sidebars, removes the old skill-settings publish row, hides skill Report for non-staff managers, and enables dev auth for UI proof runs.

## Best Possible Solution

Keep the sidebar action, but base skill New version visibility on publish access, keep Report hiding consistent with that action, update the local-auth lifecycle scenario to the new flow, and require visible proof of the skill and plugin manager states.

## Reproduction Assessment

Not applicable as a feature PR, but the PR blockers have high-confidence reproduction evidence: the live playwright-local-auth log times out on the removed settings link, and source inspection shows publisher-role publish access is broader than settings access.

## Solution Assessment

No. The proposed UI direction is reasonable, but the implementation needs the access predicate and e2e coverage fixed before it is the narrow maintainable solution.
