## Summary

Keep open: current main still intentionally renders the dashboard deployment-drift alert whenever frontend and Convex build SHAs are both present and unequal. The live backend metadata path is healthy now, but this remains a deploy-stamping or banner-policy decision rather than something definitely implemented away on main.

## What This Changes

The issue reports a persistent dashboard deployment version drift warning when the frontend expected SHA and Convex-reported SHA differ while the dashboard otherwise works.

## Best Possible Solution

Define the production deploy SHA contract, then either ensure coordinated frontend/backend stamping for normal deploys or adjust the banner severity and copy for expected partial deploys with matching tests and deploy docs.

## Reproduction Assessment

Yes at source level: current main deterministically renders the alert when the frontend and Convex SHAs are both present and unequal, and a component test asserts that behavior. I did not run a live browser reproduction of the historical production mismatch.

## Solution Assessment

No automatic code change is proven best yet. Maintainers should first decide whether this is a deploy-stamping violation or whether the banner policy should change for intentional partial deploys.
