## What This Changes

The issue reports a persistent dashboard warning when the frontend build SHA differs from the Convex deployment SHA.

## Best Possible Solution

Keep the warning for real skew, but define whether coordinated deploy stamping must use one SHA across Convex and Vercel or whether the banner severity/copy should change for intentional partial deploys.
