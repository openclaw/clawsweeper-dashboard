## Summary

Keep open. Current main still intentionally renders the dashboard drift alert when frontend and Convex build SHAs are both present and unequal, while a live check now shows Convex stamped to current main and the frontend bundle missing `VITE_APP_BUILD_SHA`; this needs maintainer/operator deploy-stamping and banner-policy confirmation rather than cleanup closure.

## What This Changes

The issue reports a persistent dashboard deployment-drift warning caused by different frontend and Convex build SHAs.

## Best Possible Solution

Keep the deploy contract explicit: coordinated production deploys should stamp the same main SHA into Vercel and Convex, and any decision to normalize partial deploys should update banner severity/copy, tests, and the deploy spec together.

## Reproduction Assessment

Yes for the source path: provide different non-empty `VITE_APP_BUILD_SHA` and `appMeta.getDeploymentInfo().appBuildSha` values and current main is expected to render the alert. The original live production mismatch was not reproduced in this check because Convex now reports current main and the frontend bundle appears to omit `VITE_APP_BUILD_SHA`.

## Solution Assessment

Unclear. The current source behavior matches the drift detector and deploy contract, so the narrow fix is likely production SHA stamping or an explicit partial-deploy policy change rather than suppressing the warning in code.
