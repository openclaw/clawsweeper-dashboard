## Summary

Keep open. Current main and the latest release still intentionally render the dashboard drift alert whenever the frontend and Convex build SHAs are both present and different, so the reported user-visible warning has not been implemented away; the remaining question is maintainer/operator policy for expected deploy drift.

## What This Changes

The issue reports a persistent dashboard warning when the frontend build SHA and Convex deployment SHA differ even though dashboard functionality continues to work.

## Best Possible Solution

Define the deploy-drift policy first, then update the workflow, banner copy/severity, tests, and specs so matching full deploys stay quiet while intentional partial drift has a deliberate operator-facing status.

## Reproduction Assessment

Yes for the source-level behavior: set VITE_APP_BUILD_SHA and Convex APP_BUILD_SHA to different non-empty SHAs and the helper plus component tests show the alert will render. No live production reproduction of the reporter's exact drift was established.

## Solution Assessment

Unclear. Silencing the alert unconditionally is not the best path until maintainers decide whether SHA drift should be treated as a deploy health problem, an expected partial-deploy state, or a lower-severity operator notice.
