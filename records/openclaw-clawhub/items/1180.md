## Summary

Keep open: current main and latest v0.23.0 still render the dashboard deployment-drift alert whenever frontend and Convex build SHAs are both present and unequal, so the reported warning behavior is not implemented away and still needs a deploy-stamping or banner-policy decision.

## What This Changes

The issue asks ClawHub to stop showing a persistent dashboard deployment version drift warning when the dashboard otherwise works.

## Best Possible Solution

Keep actionable drift detection, but define the production deploy SHA contract; if partial deploys are normal, adjust the banner severity or copy and update the drift tests and deploy runbook together.

## Reproduction Assessment

Yes at source level: current main deterministically sets `hasMismatch` and renders the alert when non-empty frontend and Convex SHAs differ. I did not recreate the historical production deployment state in a browser session.

## Solution Assessment

No automatic code change is proven best yet. Maintainers should first decide whether production deploy stamping violated the same-SHA contract or whether the banner policy should change for intentional partial deploys.
