## Summary

Keep open. Current main still intentionally renders the deployment drift banner when both frontend and Convex build SHAs are present and different, while the remaining question is whether production should enforce matching SHA stamps or soften the alert for expected partial deploy drift.

## What This Changes

The issue reports a persistent dashboard deployment-version drift warning when the frontend build SHA and Convex-reported build SHA differ while the dashboard otherwise works.

## Best Possible Solution

Keep the drift detector, verify production SHA stamping for both deployables, and adjust the banner severity or copy only if maintainers decide partial-deploy drift is expected and non-actionable.

## Reproduction Assessment

Yes for source-level behavior: current main renders the dashboard alert when non-empty, different VITE_APP_BUILD_SHA and Convex appBuildSha values are present. The original live production mismatch was not reproduced because today’s production bundle did not expose VITE_APP_BUILD_SHA while Convex returned an appBuildSha.

## Solution Assessment

Unclear. The current implementation matches an intentional deploy drift contract, so the best fix depends on a maintainer/operator decision about production SHA stamping versus expected partial deploy drift.
