## Summary

Keep open. Current main still intentionally renders the deployment drift alert when frontend and Convex build SHAs are both present and unequal, and ClawHub cleanup policy keeps issues open for maintainer/operator handling rather than auto-closing them.

## What This Changes

The issue reports a dashboard warning where the frontend build SHA differs from the Convex-reported backend build SHA even though the dashboard otherwise works.

## Best Possible Solution

Verify production SHA stamping first; coordinated deploys should carry the same main SHA, and any expected partial-deploy drift should be reflected consistently in banner severity/copy, tests, and deploy docs.

## Reproduction Assessment

Yes for the source path: set VITE_APP_BUILD_SHA and appMeta.getDeploymentInfo().appBuildSha to different non-empty values, and current helper/component tests assert the mismatch alert. The exact live production SHA pair still needs operator verification.

## Solution Assessment

Unclear. A direct code suppression is not proven to be the best fix until maintainers verify production stamping and decide whether partial deploy drift should alert, warn differently, or be treated as expected.
