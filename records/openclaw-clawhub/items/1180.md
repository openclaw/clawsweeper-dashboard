## Summary

Keep open. Current main still intentionally renders the deployment-drift alert when frontend and Convex build SHAs are both present and unequal, while live checks now suggest the exact original production warning needs operator verification rather than source cleanup closure.

## What This Changes

The issue reports a persistent dashboard deployment-drift warning for mismatched frontend and Convex build SHAs.

## Best Possible Solution

Keep the deploy contract explicit: coordinated production deploys should stamp the same main SHA on frontend and Convex, and any change to partial-deploy warning policy should update the banner copy, tests, and deploy spec together.

## Reproduction Assessment

Yes for the source path: set `VITE_APP_BUILD_SHA` and `appMeta.getDeploymentInfo().appBuildSha` to different non-empty values and current main renders the alert. ## Solution Assessment

Unclear. The current implementation matches the drift detector, tests, and deploy contract, so the right fix depends on whether production SHA stamping is wrong or partial deploys should change warning policy.
