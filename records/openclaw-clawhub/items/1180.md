## Summary

Keep open: current main and the latest release still intentionally render the dashboard deployment-drift alert when frontend and Convex build SHAs are both present and unequal, so the remaining work is a deploy-stamping or banner-policy decision rather than an implemented issue.

## What This Changes

The issue reports a persistent red dashboard warning because the frontend build SHA and Convex APP_BUILD_SHA differ even though the dashboard otherwise works.

## Best Possible Solution

Define the production deploy SHA contract first, then either enforce matching frontend and Convex deploys or adjust the banner severity/copy for intentional partial deploys with matching tests and deploy docs.

## Reproduction Assessment

Yes at source level: current main deterministically sets hasMismatch and renders the alert when non-empty frontend and Convex SHAs differ. I did not establish a fresh live browser reproduction of the historical production warning.

## Solution Assessment

No automatic code change is proven best yet. Maintainers should first decide whether this is a deploy-stamping violation or whether the banner policy should change for intentional partial deploys.
