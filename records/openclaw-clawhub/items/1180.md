## What This Changes

The issue reports a persistent dashboard deployment-version drift warning where the frontend expected one build SHA and Convex reported another while the dashboard otherwise worked.

## Best Possible Solution

Define the production deploy-status policy explicitly, then either guarantee matching frontend/backend stamps for full deploys or make expected partial-deploy drift a lower-severity/status-specific message with focused tests and runbook updates.
