## Summary

Keep open: current main and the latest release still intentionally render the dashboard deployment-drift alert when frontend and Convex build SHAs are both present and unequal, so the reported warning remains a deploy-stamping or banner-policy decision rather than an implemented-away issue.

## What This Changes

The issue reports a persistent dashboard deployment-drift warning when the frontend build SHA and Convex app metadata SHA differ even though dashboard functionality works.

## Best Possible Solution

Define the production deploy SHA contract first, then either stamp the frontend and Convex backend consistently or adjust the banner severity/copy for intentional partial deploys with matching tests and deploy docs.

## Reproduction Assessment

Yes at source level: current main deterministically sets hasMismatch and renders the alert when non-empty frontend and Convex SHAs differ. I did not establish a fresh browser reproduction of the historical production warning.

## Solution Assessment
