## Summary

Keep open. Current main still intentionally renders the dashboard deployment-drift warning when the frontend and Convex build SHAs are both present and unequal, and ClawHub cleanup policy keeps issues open for maintainer handling rather than auto-closing them.

## What This Changes

The issue requests resolving a persistent dashboard warning caused by a mismatch between the frontend build SHA and the Convex deployment build SHA.

## Best Possible Solution

Maintain a clear deployment contract: coordinated production deploys should stamp the same main SHA into Convex and the frontend, and any expected partial-deploy drift should have explicit banner severity/copy, tests, and deploy docs.

## Reproduction Assessment

Yes for the code path: current tests reproduce the warning by setting different non-empty frontend and backend SHAs, and the helper/banner still assert mismatch behavior. The exact live production state still needs operator verification.

## Solution Assessment

Unclear. A direct code suppression is not proven to be the best fix; maintainers should first verify production stamping and decide whether partial deploy drift is expected or actionable.
