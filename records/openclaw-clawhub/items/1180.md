## Summary

Keep open. Current main still intentionally renders the deployment-drift alert when frontend and Convex build SHAs are both present and unequal, and this issue needs maintainer/operator confirmation rather than cleanup closure.

## What This Changes

The issue asks for the dashboard to stop showing a persistent red deployment-version drift warning for mismatched frontend and Convex build SHAs.

## Best Possible Solution

Keep the deploy contract explicit: coordinated production deploys should stamp the same main SHA on frontend and Convex, and any change to partial-deploy warning policy should update the banner copy/severity, tests, and deploy spec together.

## Reproduction Assessment

Yes for the source path: set `VITE_APP_BUILD_SHA` and `appMeta.getDeploymentInfo().appBuildSha` to different non-empty values, and current main is expected to render the alert. The exact live production SHA pair still needs operator verification.

## Solution Assessment

Unclear. The current implementation matches the drift detector, tests, and deploy contract, so the right solution depends on whether production stamping is wrong or partial deploys should change warning policy.
