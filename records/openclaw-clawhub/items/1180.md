## Summary

Keep open. Current main still contains and tests the dashboard deployment-drift warning when frontend and Convex build SHAs are both present and unequal, and this ClawHub cleanup policy keeps issues open for maintainer handling rather than auto-closing them. The remaining decision is whether production stamping is wrong or whether partial deploy drift should change severity/copy.

## What This Changes

The issue reports a persistent dashboard deployment-version drift warning where the frontend expected SHA differs from the Convex-reported SHA.

## Best Possible Solution

Maintain a clear production deployment contract: coordinated production deploys should stamp the same main SHA into both Convex and the frontend, and the dashboard should alert only when that drift is actionable. If partial deploys are expected, update the banner severity/copy, tests, and deploy docs together; keep #618 separate unless maintainers choose a broader version-display policy.
