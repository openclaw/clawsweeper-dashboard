## Summary

Keep open. Current main still intentionally shows the deployment-drift alert when frontend and Convex build SHAs are both present and unequal, and ClawHub cleanup policy keeps issues open for maintainer/operator handling rather than auto-closing them.

## What This Changes

The issue reports a persistent dashboard deployment-version drift warning when the frontend SHA and Convex-reported build SHA differ while the dashboard otherwise works.

## Best Possible Solution

Verify production SHA stamping first; coordinated deploys should stamp the same main SHA, and expected partial-deploy drift should be reflected consistently in banner severity, copy, tests, and deploy docs.

## Reproduction Assessment

Yes for the source path: set VITE_APP_BUILD_SHA and appMeta.getDeploymentInfo().appBuildSha to different non-empty values, and current helper/component tests assert the mismatch alert. The exact live production SHA pair still needs operator verification.

## Solution Assessment

Unclear. A direct suppression is not proven to be the best fix; maintainers should first verify production stamping and decide whether partial deploy drift should alert, warn differently, or be treated as expected.
