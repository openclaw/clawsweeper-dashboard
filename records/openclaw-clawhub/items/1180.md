## What This Changes

The issue reports a persistent dashboard warning because the frontend build SHA and Convex-reported deployment SHA differ while the dashboard otherwise works.

## Best Possible Solution

Keep a clear deployment contract: coordinated production deploys should stamp matching SHAs, and the dashboard should alert only when frontend/backend drift is actionable; if partial deploy drift is expected, adjust severity/copy, tests, and deploy docs together.

## Reproduction Assessment

## Solution Assessment
