## Summary

Keep open: current main still intentionally renders the dashboard deployment-drift alert whenever the frontend and Convex build SHAs are both populated and unequal, and no merged fix or canonical open item now owns the remaining deployment-stamping policy decision.

## What This Changes

The issue reports a persistent dashboard deployment-drift warning when the frontend and Convex backend report different build SHAs.

## Best Possible Solution

Keep the warning for real skew, but make coordinated production deploys stamp one selected main SHA across Convex and Vercel or revise the banner severity, copy, tests, and deploy docs for intentional partial deploys.

## Reproduction Assessment

Yes at source level: passing the reported non-empty unequal frontend and Convex SHAs deterministically enters the tested mismatch path and renders the alert. This review did not recreate the historical production deployment state in a browser session.

## Solution Assessment

No automatic code change is proven best yet. Maintainers should first decide whether production stamping violated the documented same-SHA contract or whether the banner policy should change for intentional partial deploys.
