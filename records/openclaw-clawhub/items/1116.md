## Summary

Keep open. Current main still intentionally blocks deleted-account GitHub OAuth reauthorization instead of recreating or relinking the account; the report is clustered with related open user reports, but changing this path requires maintainer account-lifecycle and security policy judgment.

## What This Changes

The issue asks for deleted ClawHub accounts to be reset, hard-deleted, or recreated so the same GitHub identity can sign in again after OAuth.

## Best Possible Solution

Maintainers should choose and document the deleted-account lifecycle, then implement either a controlled reset/re-registration path with ban and privacy safeguards or an explicit irreversible-deletion support workflow with clear user-facing errors.

## Reproduction Assessment

Yes, by source inspection rather than live OAuth: deleteAccount deactivates and purges the user row, OAuth completion calls handleDeletedUserSignIn, and current tests assert deleted/deactivated users are rejected. I did not run an end-to-end GitHub OAuth flow against a real deployment.

## Solution Assessment

Unclear. Recreating or hard-resetting accounts would conflict with current docs and tests that make deletion irreversible, so the maintainable fix depends on maintainer policy for deletion, privacy, abuse prevention, and support operations.
