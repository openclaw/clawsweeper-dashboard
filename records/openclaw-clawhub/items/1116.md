## Summary

Keep open. Current main still deliberately treats user-initiated deletion as irreversible and rejects later GitHub OAuth sign-in for the same deleted identity; the remaining request is a product/security decision around re-registration, admin reset, and support workflow, and ClawHub cleanup policy keeps issues open unless maintainers resolve them.

## What This Changes

Request that ClawHub allow or administratively reset GitHub OAuth sign-in after a user deletes their ClawHub account, instead of leaving that identity unable to complete login.

## Best Possible Solution

Maintainers should choose and document the deleted-account lifecycle, then implement either a controlled re-registration/admin reset path or an explicit irreversible-deletion error with a support workflow.

## Reproduction Assessment

Yes. Source inspection is high confidence: deleteAccount sets deactivatedAt/purgedAt, OAuth completion calls handleDeletedUserSignIn, and tests assert deleted/deactivated users are rejected rather than recreated.

## Solution Assessment

Unclear. The current implementation is coherent with the irreversible-deletion docs, but it does not satisfy the requested account recreation/reset behavior; the safest solution depends on maintainer policy for deletion, privacy, and abuse prevention.
