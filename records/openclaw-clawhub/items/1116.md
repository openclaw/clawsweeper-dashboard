## Summary

Keep open. Current main still blocks OAuth sign-in for deleted ClawHub accounts by design, while the requested re-registration/admin-reset path remains unresolved and needs maintainer account-lifecycle, security, and production-support decisions.

## What This Changes

The issue asks for a deleted ClawHub account to be able to re-authorize or re-register with the same GitHub identity instead of looping or remaining unauthorized.

## Best Possible Solution

Maintainers should choose and document the deleted-account lifecycle, then implement either a controlled re-registration/admin reset flow or a clear irreversible-deletion support path with coverage for active, deleted, legacy-deleted, banned, and autobanned identities.

## Reproduction Assessment

Yes, from source inspection rather than a live production account. A deleted account is marked deactivated/purged, OAuth reauth throws the deleted-account error, and active-user helpers deny an active ClawHub user session.

## Solution Assessment

No for an automated fresh-account fix. The requested recovery behavior conflicts with the current irreversible-deletion contract, so maintainers need to choose the account lifecycle and support model before code changes are safe.
