## Summary

Keep open. Current main still blocks reauthorization for deleted/deactivated ClawHub accounts instead of recreating or relinking them, and the discussion shows multiple affected users plus related open reports. Under this repository’s cleanup policy, ClawHub issues should remain open for maintainer resolution rather than being auto-closed.

## What This Changes

The issue requests a way for a user who deleted their ClawHub account to sign in or re-register with the same GitHub identity instead of getting stuck in a loading/sign-in loop.

## Best Possible Solution

Maintainers should choose and document the intended deleted-account lifecycle, then implement that policy consistently: either a controlled re-registration/admin reset flow, or an explicit irreversible-deletion error with a support workflow. The chosen implementation should keep token revocation, profile/contact purging, published-skill behavior, and ban/autoban safeguards covered by regression tests.
