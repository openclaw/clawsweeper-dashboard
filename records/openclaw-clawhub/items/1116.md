## Summary

Keep open. Current main still intentionally blocks GitHub OAuth reauthorization for deleted/deactivated accounts, and changing that requires maintainer policy and security judgment around account deletion, identity reuse, bans, and support/admin reset behavior.

## What This Changes

The issue requests that a deleted ClawHub account be restorable or recreatable after GitHub OAuth reauthorization instead of leaving the user unable to sign in.

## Best Possible Solution

Maintainers should choose and document the deleted-account lifecycle, then implement either a controlled re-registration/admin reset path or a clear irreversible-deletion support workflow with regression coverage.

## Reproduction Assessment

Yes. Source inspection is high confidence: deleteAccount deactivates and purges the user row, OAuth completion calls handleDeletedUserSignIn, and tests assert deleted/deactivated users are rejected rather than recreated.

## Solution Assessment

Unclear. The current implementation is coherent with the irreversible-deletion docs, but it does not satisfy the requested account recreation/reset behavior; the maintainable fix depends on maintainer policy for deletion, privacy, and abuse prevention.
