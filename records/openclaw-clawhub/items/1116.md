## Summary

Keep open. Current main still treats user-initiated account deletion as permanent and blocks GitHub OAuth reauthorization; it now surfaces clearer blocked-account messaging, but the requested reset, recovery, relink, or re-registration path remains unresolved and is clustered with multiple open reports.

## What This Changes

The issue requests that a user who deleted a ClawHub account can authorize with the same GitHub account again and receive a usable session instead of a loading or logged-out loop.

## Best Possible Solution

Use https://github.com/openclaw/clawhub/issues/504 as the canonical cluster and implement a maintainer-approved deleted-account lifecycle across OAuth, CLI token creation, account deletion, auth binding, operator tooling, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, the OAuth callback calls handleDeletedUserSignIn, and active auth helpers then expose no signed-in user for that identity.

## Solution Assessment

No for an automated source-only fix. Recreate, restore, purge, or relink behavior conflicts with the current irreversible-deletion contract and needs maintainer product and security approval first.
