## Summary

Keep open. Current main intentionally blocks sign-in for deleted ClawHub accounts instead of recreating or relinking them, and changing that lifecycle is a security-sensitive account policy/support decision rather than a narrow cleanup close.

## What This Changes

The issue requests that a GitHub account tied to a deleted ClawHub account be reset, purged, or allowed to re-register instead of looping after OAuth reauthorization.

## Best Possible Solution

Maintainers should choose and document the deleted-account lifecycle, then either provide a guarded admin/reset or re-registration flow, or keep deletion irreversible with clearer user-facing support guidance and tests for active, deleted, legacy-deleted, banned, and mismatched identities.

## Reproduction Assessment

Yes for the current blocked reauthorization path by source inspection: deletion writes deactivatedAt/purgedAt, OAuth rejects that account, and tests assert the rejection. The original silent infinite-loading symptom is not fully reproduced on current main because the auth UI now maps callback failures to user-facing errors.

## Solution Assessment

No for an automated fresh-account recreation or one-off purge as the whole solution. The maintainable fix needs a deliberate account-lifecycle policy that preserves OAuth ownership checks, deletion privacy semantics, token revocation, and ban/autoban boundaries.
