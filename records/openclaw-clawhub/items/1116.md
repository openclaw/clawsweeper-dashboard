## Summary

Keep open. Current main still blocks GitHub OAuth sign-in for deleted/deactivated ClawHub users, while the requested recovery/re-registration path conflicts with the documented irreversible-deletion policy and needs maintainer product, privacy, support, and security decisions.

## What This Changes

The issue asks for a deleted ClawHub account to be reset, hard-deleted, or recreated so the same GitHub identity can sign in again after OAuth.

## Best Possible Solution

Maintainers should define the deleted-account lifecycle, then implement either a controlled re-registration/admin reset flow with ban/privacy safeguards or keep deletion irreversible while making the post-OAuth error and support path explicit.

## Reproduction Assessment

Yes, from source inspection rather than a live production account. Account deletion writes `deactivatedAt`, the OAuth callback rejects that identity on the next sign-in, and active-user helpers keep the frontend from resolving an active ClawHub user.

## Solution Assessment

No for an automated fresh-account fix. The requested reset/recreate behavior conflicts with current irreversible-deletion specs and tests, so maintainers need to choose the recovery or support policy before implementation.

## Risks / Open Questions
