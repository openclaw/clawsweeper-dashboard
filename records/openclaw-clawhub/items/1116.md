## Summary

Keep open. Current main still deliberately blocks sign-in for deleted/deactivated ClawHub accounts, and the remaining restore, unlink, purge, or re-registration behavior is clustered with an open canonical report and needs maintainer product/security policy rather than cleanup closure.

## What This Changes

The issue requests a way for a GitHub OAuth user to sign back into ClawHub after deleting their ClawHub account, instead of remaining blocked by the deleted account state.

## Best Possible Solution

Use https://github.com/openclaw/clawhub/issues/504 as the canonical policy cluster and implement a maintainer-approved deleted-account lifecycle across OAuth, CLI token creation, account deletion, auth binding, operator tooling, user-facing errors, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: deleteAccount sets deactivatedAt/purgedAt, the OAuth callback rejects that user through handleDeletedUserSignIn, and active auth helpers then expose no active signed-in user.

## Solution Assessment

No for an automated fix. Restoring, recreating, purging, or relinking accounts conflicts with the current irreversible-deletion contract and needs maintainer product/security approval first.
