## Summary

Keep open. Current main still treats deleted ClawHub accounts as permanently deactivated and blocks GitHub OAuth reauthorization; the remaining re-registration/reset behavior needs maintainer product and security direction, especially because this report is clustered with multiple same-symptom users.

## What This Changes

The issue requests reauthorization, re-registration, or backend reset for a GitHub account after deleting a ClawHub account leaves OAuth unable to establish an active session.

## Best Possible Solution

Choose and document the deleted-account lifecycle, then implement either a controlled audited recovery/re-registration/admin-reset path or keep deletion terminal with clear OAuth/support messaging and regression coverage.

## Reproduction Assessment

Yes, at the source level: deleteAccount sets deactivatedAt/purgedAt, OAuth re-entry calls handleDeletedUserSignIn and rejects deleted/deactivated records, and active-user helpers return no user for those records. I did not run a live production OAuth flow or inspect the reporter's database row.

## Solution Assessment

No, a one-off hard delete or automatic recreate is not the best immediate source fix because it bypasses unresolved deletion, privacy, audit, token, and ban policy. The maintainable path is an explicit account-lifecycle decision plus audited implementation or support tooling.
