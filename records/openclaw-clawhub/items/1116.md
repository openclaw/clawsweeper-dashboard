## Summary

Keep open. Current main still blocks deleted-account GitHub reauthorization by design, and the requested account recreation or reset path remains a maintainer policy and security decision; ClawHub cleanup policy also keeps issues open unless maintainers resolve them.

## What This Changes

The issue requests that a deleted ClawHub account be reset or recreated so the same GitHub identity can authorize and sign in again instead of looping or being rejected.

## Best Possible Solution

Maintainers should choose and document the deleted-account lifecycle, then implement either a controlled re-registration/admin reset flow or an explicit irreversible-deletion support path with regression coverage.

## Reproduction Assessment

Yes for the blocked reauthorization path. Source inspection shows deletion sets deactivatedAt/purgedAt, OAuth completion rejects that row, and tests assert deleted users are not restored; I did not run a live OAuth flow.

## Solution Assessment

Unclear. The current implementation is coherent with the irreversible-deletion docs and now surfaces a user-facing error, but it does not satisfy the requested recreation/reset behavior; the safest solution depends on maintainer policy.
