## Summary

Keep open. Current main still treats self-deleted ClawHub accounts as an irreversible auth state, while this issue and the related open cluster show real users blocked from signing back in; the remaining choice is product/security policy, not conservative cleanup closure.

## What This Changes

The issue asks ClawHub to let a previously deleted GitHub OAuth account be reset or recreated so reauthorization no longer loops or leaves the user signed out.

## Best Possible Solution

Maintainers should choose and document the deleted-account lifecycle, then implement either a controlled recovery/admin reset path or an explicitly terminal support/error path with tests and specs updated.

## Reproduction Assessment

Yes at the source level: current main writes deactivatedAt/purgedAt on deletion, rejects deleted/deactivated users during OAuth handling, and hides those users from active-auth helpers. I did not run a live OAuth flow for the reporter's production account.

## Solution Assessment

No for a one-off hard-delete or automatic recreate change as the immediate fix. The maintainable path needs an explicit account lifecycle, privacy, security, ban, and support decision before code changes alter deleted-account recovery semantics.

## Security Review

Status: needs_attention

Summary: Deleted-account recovery is security-sensitive because it changes OAuth identity reuse, deletion guarantees, token/audit semantics, and ban-bypass boundaries.

Concerns:

- **[medium] Define recovery authority before changing sign-in:** `convex/auth.ts:40`
  - body: handleDeletedUserSignIn verifies the incoming identity relationship before handling deleted users; any restore, purge, recreate, or relink path needs equivalent provider-account verification and audit logging.
  - confidence: 0.9
- **[medium] Preserve deletion privacy and token revocation semantics:** `convex/users.ts:469`
  - body: deleteAccount revokes API tokens and clears profile, contact, and GitHub-derived fields, so recovery must not unexpectedly restore private data or revoked credentials.
  - confidence: 0.88
- **[medium] Keep recovery separate from bans:** `convex/users.ts:899`
  - body: Ban and unban paths distinguish deactivated accounts from banned deletedAt accounts; recovery tooling must not let banned or autobanned identities regain access through deleted-account handling.
  - confidence: 0.86
