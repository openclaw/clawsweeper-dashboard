## Summary

Keep open. Current main still treats deleted ClawHub accounts as permanently deactivated and blocks GitHub OAuth reauthorization; the remaining recovery, relink, or re-registration behavior is clustered with https://github.com/openclaw/clawhub/issues/504 and needs maintainer product/security policy rather than cleanup closure.

## What This Changes

Request a recovery, reset, or clean re-registration path for GitHub OAuth identities whose ClawHub account was deleted and can no longer establish an active session.

## Best Possible Solution

Use https://github.com/openclaw/clawhub/issues/504 as the canonical policy cluster and implement a maintainer-approved deleted-account lifecycle across OAuth, CLI token creation, account deletion, auth binding, operator tooling, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `deleteAccount` sets `deactivatedAt`/`purgedAt`, the OAuth callback calls `handleDeletedUserSignIn`, and active auth helpers then expose no signed-in active user for that identity.

## Solution Assessment

No for an automated fix. Restoring, recreating, purging, or relinking accounts conflicts with the current irreversible-deletion contract and needs maintainer product/security approval first.
