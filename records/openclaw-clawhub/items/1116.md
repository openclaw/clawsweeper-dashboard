## Summary

Keep open. Current main intentionally blocks deleted-account GitHub reauthorization instead of recreating the account, and the remaining problem is a clustered account-lifecycle/support decision rather than an auto-close candidate.

## What This Changes

The issue requests that a GitHub account previously tied to a deleted ClawHub account be reset, hard-deleted, or allowed to re-register instead of looping after OAuth.

## Best Possible Solution

Maintainers should choose and document one deleted-account lifecycle: keep deletion irreversible with a clear support/admin reset path, or add a guarded same-provider re-registration flow that preserves token revocation, purged fields, audit history, and ban/autoban blocks.

## Reproduction Assessment

Yes for the current blocked deleted-account reauth path by source inspection: deletion writes deactivatedAt/purgedAt, OAuth rejects that user, and active-auth helpers hide it. The exact old infinite-loading presentation is less certain on current main because frontend tests now cover blocked/no-session error surfacing.

## Solution Assessment

No for an automatic reset or recreate change as the next step. The requested behavior conflicts with the current irreversible-deletion contract, so the maintainable fix needs an explicit account-lifecycle and security decision first.
