## Summary

Keep open. Current main still treats deleted ClawHub accounts as terminal for GitHub OAuth reauthorization, and the remaining choice between recovery, admin reset, or a terminal support path is a product/security decision rather than cleanup closure.

## What This Changes

The issue asks for a deleted ClawHub account tied to a GitHub identity to reauthorize or re-register instead of staying signed out after OAuth.

## Best Possible Solution

Maintainers should choose and document the deleted-account lifecycle, then implement either a controlled recovery/admin reset flow or an explicitly terminal support path with specs, docs, security review, and regression coverage updated together.

## Reproduction Assessment

Yes at the source level: current main writes deactivatedAt/purgedAt on deletion, rejects deleted/deactivated users during OAuth handling, and hides those records from active-auth helpers. I did not run a live OAuth flow against the reporter's production account.

## Solution Assessment

No for the requested one-off hard delete or automatic recreate as the immediate product-wide fix. The maintainable path needs an explicit lifecycle, privacy, security, ban, and support decision before code changes alter deleted-account recovery semantics.

## Security Review

Status: needs_attention

Summary: Deleted-account recovery is security-sensitive because it changes OAuth identity reuse, deletion guarantees, token/audit semantics, and ban-bypass boundaries.

Concerns:

- **[medium] Define recovery authority before changing sign-in:** `convex/auth.ts:40`
  - body: handleDeletedUserSignIn currently verifies the incoming identity relationship before handling deleted users; any restore, purge, recreate, or relink path needs equivalent provider-account verification and audit logging.
  - confidence: 0.9
- **[medium] Preserve deletion privacy and token revocation semantics:** `convex/users.ts:469`
  - body: deleteAccount revokes API tokens and clears profile, contact, and GitHub-derived fields, so recovery must not unexpectedly restore private data or revoked credentials.
  - confidence: 0.88
- **[medium] Keep recovery separate from bans:** `convex/users.ts:822`
  - body: Ban and unban paths distinguish deactivated accounts from banned deletedAt accounts; recovery tooling must not let banned or autobanned identities regain access through deleted-account handling.
  - confidence: 0.86
