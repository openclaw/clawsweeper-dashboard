## Summary

Keep open. Current main still blocks reauthorization for deleted/deactivated ClawHub accounts, the report is clustered with multiple similar user reports, and changing this path needs maintainer policy/security judgment rather than cleanup closure.

## What This Changes

The issue asks for deleted ClawHub accounts to be reset, recreated, or relinked so the same GitHub identity can sign in again after OAuth.

## Best Possible Solution

Maintainers should choose and document the deleted-account lifecycle, then implement either a guarded re-registration/admin-reset path or a clearer irreversible-deletion support workflow with regression coverage.

## Reproduction Assessment

Yes. Source inspection is high confidence for the blocked reauthorization path: account deletion deactivates the user, OAuth completion calls `handleDeletedUserSignIn`, and tests assert deleted/deactivated users are rejected rather than recreated.

## Solution Assessment

Unclear. The current implementation matches the irreversible-deletion docs, but the requested reset/re-registration behavior is a product and security policy decision rather than a narrow source-only fix.
