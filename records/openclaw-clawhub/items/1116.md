## Summary

Keep open. Current main still treats deleted ClawHub accounts as permanently deactivated and rejects GitHub reauthorization; the requested re-registration/reset behavior is clustered with related reports and needs maintainer policy and security judgment.

## What This Changes

Allow a user who deleted their ClawHub account to sign in or re-register with the same GitHub identity, or reset that account state.

## Best Possible Solution

Maintainers should choose and document the deleted-account lifecycle, then implement either a guarded re-registration/admin-reset path or a clearer irreversible-deletion support workflow with regression coverage.

## Reproduction Assessment

Yes, from source inspection. The account-block path is clear: deletion sets `deactivatedAt`/`purgedAt`, OAuth sign-in calls `handleDeletedUserSignIn`, and tests assert deleted/deactivated users are rejected; the exact infinite-loading UI may now surface as a deleted-account error.

## Solution Assessment

Unclear. Re-registration would address the user reports, but it conflicts with the current irreversible-deletion docs and should be designed as a policy-backed recovery or support flow rather than a narrow automatic recreate.
