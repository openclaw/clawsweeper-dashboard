## Summary

Keep open. Current main still treats deleted ClawHub accounts as permanently deactivated and rejects GitHub OAuth reauthorization for that identity, while this clustered account recovery problem needs maintainer policy and production-support handling rather than cleanup closure.

## What This Changes

The issue asks ClawHub to allow or administratively reset GitHub OAuth sign-in after a previously linked account was deleted and now loops or remains unauthorized.

## Best Possible Solution

Maintainers should define and document the deleted-account lifecycle, then implement either a controlled recovery/re-registration/admin reset flow or an explicit irreversible-deletion support path with coverage for active, deleted, legacy-deleted, banned, and autobanned identities.

## Reproduction Assessment

Yes, from source inspection rather than a live production account. Deleting an account records inactive state, the next OAuth callback rejects that linked identity, and active-user helpers then keep the session from becoming an active ClawHub user.

## Solution Assessment

No for an automated fix. The requested fresh-account behavior conflicts with the current irreversible-deletion contract, so restore, purge, recreate, relink, or support-reset behavior needs maintainer security and product approval first.
