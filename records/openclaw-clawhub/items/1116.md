## Summary

Keep open. Current main still treats deleted ClawHub accounts as terminal deactivated records, so re-registering with the same GitHub identity is not supported; the remaining decision is a security-sensitive account lifecycle policy, and ClawHub cleanup policy keeps issues open.

## What This Changes

Allow or operationally support GitHub OAuth sign-in/re-registration after a ClawHub account has been deleted instead of leaving affected users blocked.

## Best Possible Solution

Maintainers should choose and document the deleted-account lifecycle, then implement either a controlled re-registration/admin reset flow or a clearly terminal support/error path with audit, privacy, token, and ban safeguards.

## Reproduction Assessment

Yes, at the source level: deletion writes deactivatedAt/purgedAt, OAuth reauth rejects deleted users, and active-auth helpers hide those records. I did not run a live OAuth flow against the reporter's production account.

## Solution Assessment

No for a one-off hard-delete or automatic recreate as an immediate cleanup fix. The maintainable path needs an explicit product/security decision for recovery authority, privacy, audit, token revocation, and ban boundaries.
