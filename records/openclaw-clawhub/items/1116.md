## Summary

Keep open. Current main still treats deleted ClawHub accounts as permanently deactivated and rejects GitHub OAuth completion for that identity; the related reports remain open, and changing this path needs maintainer policy/security judgment rather than cleanup closure.

## What This Changes

The issue asks for a deleted ClawHub account linked to GitHub to be reset or recreated so the same GitHub identity can sign in again.

## Best Possible Solution

Maintainers should choose and document the deleted-account lifecycle, then implement either a guarded re-registration/admin reset flow or a clear irreversible-deletion support workflow with regression coverage.

## Reproduction Assessment

Yes for the blocked reauthorization path: source inspection shows deletion sets deactivatedAt/purgedAt and OAuth completion rejects that user, with tests asserting the rejection. The original infinite-loading UI was not manually reproduced against a live deployment.

## Solution Assessment

Unclear. There is no proposed patch, and the requested reset/re-registration behavior conflicts with current irreversible-deletion docs, so the narrowest maintainable fix depends on maintainer policy and security decisions.
