## Summary

Keep open. Current main still treats self-deleted ClawHub accounts as permanently deactivated and blocks GitHub OAuth reauthorization; the issue is valid, clustered with other affected-user reports, and the remaining path needs account-lifecycle product/security review rather than cleanup closure.

## What This Changes

The issue asks ClawHub to let a previously deleted account re-register or be reset after GitHub OAuth reauthorization leaves the user logged out.

## Best Possible Solution

Choose and document the deleted-account lifecycle, then implement the selected audited recovery/re-registration/admin-reset path or keep deletion terminal with clear OAuth/support messaging and regression coverage.

## Reproduction Assessment

Yes, at the source level: deletion sets deactivatedAt/purgedAt, OAuth re-entry calls handleDeletedUserSignIn, and active-user helpers return no user for deleted/deactivated records. I did not run a live production OAuth flow or inspect the reporter's database row.

## Solution Assessment

No for a one-off hard-delete or automatic recreate as an immediate fix. The maintainable path needs an explicit product/security decision about recovery authority, privacy, audit, token revocation, and ban boundaries.

## Security Review

Status: needs_attention

Summary: Deleted-account recovery is security-sensitive because it changes OAuth identity reuse, deletion guarantees, token revocation, audit, and ban-bypass boundaries.

Concerns:

- **[medium] Define recovery authority before changing sign-in:** `convex/auth.ts:40`
  - body: handleDeletedUserSignIn already guards identity mismatch before touching deleted records; any restore, purge, recreate, or relink path needs equivalent provider-account verification and audit logging.
  - confidence: 0.92
- **[medium] Preserve deletion privacy and token semantics:** `convex/users.ts:475`
  - body: deleteAccount revokes API tokens and clears profile, contact, GitHub-derived, and telemetry state, so recovery must not unexpectedly restore private data or revoked credentials.
  - confidence: 0.9
- **[medium] Keep recovery separate from bans:** `convex/users.ts:910`
  - body: Legacy deleted-user sign-in checks ban audit records and admin unban rejects deactivated accounts, so recovery tooling must not let banned or autobanned identities regain access through deleted-account handling.
  - confidence: 0.87
