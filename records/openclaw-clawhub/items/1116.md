## Summary

Keep open. Current main still deliberately treats user-initiated deletion as irreversible and blocks later GitHub OAuth sign-in for the same ClawHub identity; the reported account recovery path needs maintainer account-lifecycle, privacy, security, and production-support decisions rather than cleanup closure.

## What This Changes

The issue requests that a GitHub-linked ClawHub account can sign in or be reset again after user-initiated account deletion instead of staying unauthorized after OAuth.

## Best Possible Solution

Maintainers should define the deleted-account lifecycle, then implement either a controlled re-registration/admin reset flow or an explicit irreversible-deletion support path with coverage for active, deleted, legacy-deleted, banned, and autobanned identities.

## Reproduction Assessment

Yes, from source inspection rather than a live production account. Deletion records inactive state, the next OAuth callback rejects that linked identity, and active-user helpers keep the session from becoming an active ClawHub user.

## Solution Assessment

No for an automated fresh-account fix. The requested recovery/reset behavior conflicts with the current irreversible-deletion contract, so restore, purge, recreate, relink, or support-reset behavior needs maintainer security and product approval first.
