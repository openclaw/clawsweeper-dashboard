## Summary

Keep open. The current source still intentionally treats deleted accounts as deactivated terminal records and rejects GitHub OAuth reauthorization for those identities, while the issue is externally authored, currently labeled for maintainer/product/security review, and ClawHub cleanup policy keeps issues open rather than auto-closing them.

## What This Changes

The issue asks for deleted ClawHub accounts to be reset, purged, recreated, or relinked so the same GitHub identity can sign in again after account deletion.

## Best Possible Solution

Maintainers should choose and document the deleted-account lifecycle, then implement either a controlled re-registration/admin reset path or a clear irreversible-deletion error/support path with audit, privacy, token, and ban safeguards.

## Reproduction Assessment

Yes, at the source level: deletion writes deactivatedAt/purgedAt, OAuth reauth rejects deleted users, and active-auth helpers hide those records. I did not run a live OAuth flow against the reporter's production account.

## Solution Assessment

No for an immediate one-off hard-delete or unconditional automatic recreate. The maintainable fix needs an explicit product/security decision for recovery authority, privacy, audit, token revocation, and ban boundaries.

## Security Review

Status: needs_attention

Summary: Deleted-account recovery is security-sensitive because it changes OAuth identity reuse, deletion guarantees, token revocation, audit, and ban-bypass boundaries.

Concerns:

- **[medium] Define recovery authority before changing sign-in:** `convex/auth.ts:40`
  - body: handleDeletedUserSignIn currently checks the incoming identity relationship before handling deleted users; any restore, purge, recreate, or relink path needs equivalent provider-account verification and audit logging.
  - confidence: 0.9
- **[medium] Preserve deletion privacy and token semantics:** `convex/users.ts:457`
  - body: deleteAccount revokes API tokens and clears profile, contact, and GitHub-derived fields, so recovery must not unexpectedly restore private data or revoked credentials.
  - confidence: 0.88
- **[medium] Keep recovery separate from bans:** `convex/auth.ts:50`
  - body: The deleted-account callback distinguishes legacy deleted users from ban audit records, and admin unban explicitly rejects deactivated accounts; recovery tooling must not let banned or autobanned identities regain access through deleted-account handling.
  - confidence: 0.86
