## Summary

Keep open. Current main still intentionally treats deleted ClawHub accounts as permanently deactivated and blocks later GitHub OAuth sign-in instead of recreating, relinking, or resetting the account; the report is also clustered with multiple open user reports, and ClawHub policy keeps issues open for maintainer resolution.

## What This Changes

The issue requests that a deleted ClawHub account be able to reauthorize or re-register with the same GitHub identity instead of entering a login loop.

## Best Possible Solution

Maintainers should choose and document the deleted-account lifecycle, then implement either a controlled re-registration/admin reset flow or a clear irreversible-deletion error with a support path and regression coverage.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to pass a deactivatedAt or legacy deletedAt user into handleDeletedUserSignIn; current tests assert rejection and migration rather than account recreation. I did not run a live OAuth flow in this read-only review.

## Solution Assessment

Unclear as product policy. The current implementation is internally consistent with irreversible deletion, but it does not solve the requested re-registration/reset behavior; the safest fix needs maintainer policy, privacy, and abuse-prevention decisions first.
