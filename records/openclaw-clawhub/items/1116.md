## Summary

Keep open. Current main still blocks GitHub OAuth re-registration for deleted accounts by design; while current UI has code to surface a permanent-deletion error instead of a silent loop, the requested restore/recreate/reset path remains a security-sensitive account lifecycle decision and is clustered with other open reports.

## What This Changes

The issue requests that a GitHub account be able to sign in or be reset after the linked ClawHub account was deleted, instead of looping or remaining unauthorized after OAuth.

## Best Possible Solution

Maintainers should define and document the deleted-account lifecycle, then implement either a controlled recovery/admin reset path or a clearly terminal post-OAuth error/support path with audit behavior and regression coverage.

## Reproduction Assessment

Yes, at the source level: deletion writes deactivatedAt/purgedAt, OAuth completion rejects deleted users, and active-auth helpers treat those records as signed out. I did not run a live OAuth flow against the reporter’s production account.

## Solution Assessment

No for an immediate hard-delete or automatic recreate as the fix. The safer path is a policy-backed recovery/reset or terminal support flow that preserves provider identity checks, deletion privacy, token revocation, audit logs, and ban separation.

## Security Review

Status: needs_attention

Summary: Deleted-account recovery is security-sensitive because it changes OAuth identity reuse, deletion guarantees, token revocation, audit, and ban-bypass boundaries.

Concerns:

- **[medium] Define recovery authority before changing sign-in:** `convex/auth.ts:40`
  - body: handleDeletedUserSignIn currently verifies the incoming identity relationship before handling deleted users; any restore, purge, recreate, or relink path needs equivalent provider-account verification and audit logging.
  - confidence: 0.9
- **[medium] Preserve deletion privacy and token revocation:** `convex/users.ts:475`
  - body: deleteAccount revokes API tokens and clears profile, contact, and GitHub-derived fields, so recovery must not unexpectedly restore private data or revoked credentials.
  - confidence: 0.88
- **[medium] Keep deleted recovery separate from bans:** `convex/auth.ts:51`
  - body: The deleted-account callback distinguishes legacy deleted users from ban audit records, and unban logic refuses deactivated accounts; recovery tooling must not let banned or autobanned identities regain access through deleted-account handling.
  - confidence: 0.86
