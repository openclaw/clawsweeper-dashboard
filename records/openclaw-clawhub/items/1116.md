## Summary

Keep open. Current main intentionally treats user deletion as irreversible and blocks GitHub OAuth reauthorization, while this issue is part of a real blocked-account cluster that needs maintainer product, security, and production-support scoping rather than auto-close or a narrow bot fix.

## What This Changes

Requests that a deleted GitHub-linked ClawHub account be reset, hard-deleted, or allowed to re-register instead of remaining unauthorized or stuck after OAuth.

## Best Possible Solution

Maintainers should choose and document one deleted-account lifecycle: keep deletion irreversible with a clear support/admin reset path and UI error, or add a guarded same-GitHub re-registration flow that preserves token revocation, purged data, audit history, and ban/autoban blocks.

## Reproduction Assessment

Yes for the current blocked deleted-account reauth path by source inspection: deletion writes deactivatedAt/purgedAt, OAuth rejects that user, and active-auth helpers hide it. The exact historical infinite-loading UI is less certain on current main because AppProviders tests now cover deleted-account and no-session error surfacing.

## Solution Assessment

No for an automatic hard-delete, reset, or recreate change as the next step. The requested behavior conflicts with the current irreversible-deletion contract, so the maintainable path needs an explicit account-lifecycle and security decision first.
