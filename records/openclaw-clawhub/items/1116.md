## Summary

Keep open. Current main still preserves deleted accounts as inactive records and blocks GitHub OAuth reauthorization; changing that behavior is a clustered, security-sensitive account lifecycle decision, not conservative cleanup.

## What This Changes

The issue requests that a previously deleted ClawHub account be hard-deleted, reset, recreated, or relinked so the same GitHub identity can sign in again.

## Best Possible Solution

Maintainers should define the deleted-account lifecycle and implement either a controlled recovery/admin reset path or a clearly terminal support/error path, with specs, docs, audit behavior, and regression coverage updated together.

## Reproduction Assessment

Yes, at the source level: deletion writes deactivatedAt/purgedAt, OAuth reauth rejects deleted users, and active-auth helpers hide those records. I did not run a live OAuth flow against the reporter's production account.

## Solution Assessment

No for the requested one-off hard-delete or automatic recreate behavior as an immediate code fix. The maintainable solution needs explicit product/security policy for deletion, recovery authority, privacy, and ban boundaries.
