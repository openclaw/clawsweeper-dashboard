## Summary

Keep open. Current main still treats user-initiated deletion as irreversible and blocks later GitHub OAuth sign-in for the deleted/deactivated account, and ClawHub cleanup policy keeps issues open unless they are PRs already implemented on main.

## What This Changes

The issue asks ClawHub to allow or administratively reset GitHub OAuth sign-in after a user deletes their ClawHub account and then re-authorizes the same GitHub identity.

## Best Possible Solution

Maintainers should choose and document the deleted-account lifecycle, then implement either a controlled re-registration/admin reset flow or a clear irreversible-deletion error with a support workflow and regression coverage for active, deleted, legacy-deleted, banned, autobanned, and reauthorized identities.

## Reproduction Assessment

Yes. The code-level reproduction is high confidence: deleteAccount marks the user deactivated/purged, OAuth sign-in calls handleDeletedUserSignIn, and tests assert that deactivated users are rejected; I did not run a live OAuth flow in this read-only review.

## Solution Assessment

Unclear. The current implementation is internally consistent with the irreversible-deletion docs, but it does not solve the requested account recreation/reset behavior; the safest fix depends on maintainer policy for privacy, abuse prevention, and support operations.
