## Summary

Keep open. Current main still intentionally blocks deleted/deactivated GitHub identities from signing in again, while this report and related open reports show a real account-recovery problem that needs maintainer policy and production-ops handling rather than cleanup closure.

## What This Changes

Allow a user who deleted a ClawHub account to reauthorize or re-register with the same GitHub identity instead of looping or remaining unauthorized.

## Best Possible Solution

Maintainers should choose and document the deleted-account lifecycle, then implement either a controlled recovery/re-registration/admin reset flow or an explicit irreversible-deletion support path with regression coverage for active, deleted, legacy-deleted, banned, and autobanned identities.

## Reproduction Assessment

Yes, from source inspection rather than a live production account. `deleteAccount` marks the user deactivated and purged, the next OAuth callback rejects that identity, and active-user helpers keep the app in a signed-out state.

## Solution Assessment

No, not as an automated source-only change. The requested fresh-account behavior conflicts with the current irreversible-deletion contract, so recovery, purge/recreate, relink, or support-reset behavior needs maintainer security and product approval first.
