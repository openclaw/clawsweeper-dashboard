## Summary

Keep open. Current main intentionally preserves the semver-managed latest pointer during publish, including when callers pass `latest`, so changing the reported behavior would require maintainer policy judgment rather than conservative cleanup.

## What This Changes

The issue requests that `clawhub skill publish --tags latest` move the visible latest pointer to the newly published prerelease version instead of preserving an older stable release.

## Best Possible Solution

Maintainers should choose and document one contract: keep publish-time latest semver-managed and use retagging for overrides, or add a deliberate override flow with backend, CLI, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection and existing tests give a high-confidence path: publish a lower-version or prerelease version with `tags: ['latest']`, and `insertVersion` drops that tag while preserving the previous latest pointer.

## Solution Assessment

Unclear. The requested behavior conflicts with the intentional PR #1832 backport guard, so the right solution depends on whether maintainers want publish-time overrides or clearer docs plus the existing retag path.
