## Summary

Keep open. Current main intentionally preserves the semver-managed latest pointer on lower/prerelease publishes even when `latest` is supplied, and changing that contract conflicts with the merged backport-clobbering guard, so maintainer product judgment is needed.

## What This Changes

Request that `clawhub skill publish --tags latest` move the visible `latest` pointer to the newly published prerelease instead of leaving an older stable version selected.

## Best Possible Solution

Define the publish-time `latest` contract explicitly: either document semver-managed publishing and direct users to retagging, or add a deliberate override path with backend, CLI, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection shows a publish with a lower/prerelease semver and `tags: ['latest']` reaches `insertVersion`, which skips caller-provided `latest` and leaves `latestVersionId` on the previous version; the backport test asserts that exact state.

## Solution Assessment

Unclear. The requested behavior would be a deliberate semantic change or override path, while the current implementation is the narrow fix for preventing backports from clobbering `latest`; maintainers should choose between documenting the guard or adding an explicit override.
