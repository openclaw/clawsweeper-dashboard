## Summary

Keep open. Current main intentionally preserves the semver-managed latest pointer on lower/prerelease publishes even when `latest` is supplied, so the requested behavior conflicts with the backport guard from #1832 and needs maintainer product judgment rather than cleanup closure.

## What This Changes

The issue asks for `clawhub skill publish --tags latest` to repoint `latest` to a prerelease or lower-version publish instead of preserving the semver-selected latest version.

## Best Possible Solution

Define the contract explicitly: either document publish-time `latest` as semver-managed and point users to retagging, or add a deliberate override path with backend, CLI/API, docs, and regression coverage.

## Reproduction Assessment

Yes, by source inspection: publish an existing skill with a lower or prerelease semver and `tags: ['latest']`, and `insertVersion` skips that tag while tests assert the previous latest remains selected. I did not perform a live publish against a deployment.

## Solution Assessment

Unclear. The requested behavior is coherent, but it conflicts with the intentional #1832 semver guard, so maintainers need to choose between clearer docs, changed `--tags latest` semantics, or an explicit force-retag flow.
