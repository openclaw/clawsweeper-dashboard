## Summary

Keep open. Current main intentionally treats publish-time `latest` as semver-managed for lower/prerelease publishes, so the requested override conflicts with the backport protection introduced by #1832 and needs maintainer product judgment rather than cleanup closure.

## What This Changes

The issue requests that `clawhub skill publish --tags latest` move the displayed latest version to the newly published prerelease or lower semver version.

## Best Possible Solution

Define the contract explicitly: either document publish-time `latest` as semver-managed and point users to retagging, or add a deliberate override path with backend, CLI, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: publish a lower or prerelease semver with `tags: ['latest']`, and `insertVersion` skips that tag while tests assert the previous latest remains selected.

## Solution Assessment

Unclear. The requested behavior conflicts with the intentional #1832 semver guard, so the best solution depends on whether maintainers want publish-time overrides or clearer docs plus the existing retag flow.
