## Summary

Keep open. Current main has a high-confidence source-level reproduction of the reported behavior, but it is enforced by the intentional semver-aware latest guard from #1832, so changing it needs maintainer product judgment rather than conservative cleanup.

## What This Changes

Request that `clawhub skill publish --tags latest` move `latest` to the newly published prerelease or lower version instead of preserving the semver-managed latest pointer.

## Best Possible Solution

Decide the product contract, then either document publish-time `latest` as semver-managed with retagging as the override path, or add an explicit publish override with backend, CLI, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: publish a lower or prerelease semver with `tags: ['latest']`, and `insertVersion` skips that tag while tests assert the previous latest pointer remains selected.

## Solution Assessment

Unclear. The requested behavior conflicts with the deliberate #1832 backport guard, so the best solution depends on whether maintainers want publish-time overrides or clearer docs around the existing retag flow.
