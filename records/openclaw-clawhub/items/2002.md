## Summary

Keep open. Current main intentionally preserves the higher semver latest version and drops caller-supplied `latest` during lower-version publishes, while the docs leave room for the reporter's interpretation, so this needs maintainer product/security judgment rather than cleanup closure.

## What This Changes

Request that `clawhub skill publish --tags latest` move the displayed latest skill version to the just-published prerelease instead of keeping the higher stable semver release.

## Best Possible Solution

Maintainers should define the contract explicitly: either add a deliberate latest-override path with full backend/CLI/docs coverage, or document publish-time `latest` as semver-managed and require a separate retag/rollback operation.

## Reproduction Assessment

Yes, source-reproducible. A lower semver or prerelease publish with `tags: ['latest']` computes `isNewLatest` as false, skips the caller-provided `latest` tag, and preserves the previous latest pointer; existing tests assert that behavior.

## Solution Assessment

Unclear. The requested behavior conflicts with the deliberate #1832 backport guard, so the best path is maintainer decision before changing code.
