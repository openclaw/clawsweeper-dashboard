## Summary

Keep open. Current main intentionally preserves the semver-managed latest pointer on lower or prerelease publishes and tests explicitly ignore publish-time `latest`, so changing this conflicts with #1832 and needs maintainer product/API judgment.

## What This Changes

The issue asks for `clawhub skill publish --tags latest` to repoint the visible `latest` version even when the published semver is a prerelease or lower than the current stable release.

## Best Possible Solution

Define the contract explicitly: either document publish-time `latest` as semver-managed with retagging as the override, or add a deliberate override path with backend, CLI, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: publish a lower or prerelease semver with `tags: ['latest']`, and `insertVersion` skips that tag while tests assert the previous latest pointer remains selected.

## Solution Assessment

Unclear. The requested override may be useful, but it conflicts with the intentional #1832 semver guard, so maintainers need to choose between explicit publish-time override semantics and clearer documentation of the existing retag flow.
