## Summary

Keep open. Current main deliberately treats publish-time `latest` as semver-managed and ignores caller-provided `latest` on lower/prerelease publishes, so changing this behavior needs maintainer product judgment because it conflicts with #1832.

## What This Changes

The issue requests that `clawhub skill publish --tags latest` repoint the displayed `latest` version to a newly published prerelease or lower semver version.

## Best Possible Solution

Decide the contract explicitly: either document publish-time `latest` as semver-managed and point users to retagging, or add a deliberate override path with backend, CLI, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: publish an existing skill with a lower/prerelease semver and `tags: ['latest']`, and `insertVersion` skips that tag while tests assert the previous latest remains selected.

## Solution Assessment

Unclear. The requested behavior is coherent, but it conflicts with the intentional #1832 semver guard, so maintainers need to choose between an explicit override feature and clearer docs plus the existing retag flow.
