## Summary

Keep open. Current main intentionally preserves the semver-managed latest pointer on lower-version or prerelease publishes and ignores caller-provided `latest` in the publish tag list; changing that would revise the #1832 backport-protection policy, so this needs maintainer product judgment rather than cleanup closure.

## What This Changes

The issue requests that `clawhub skill publish --tags latest` move the visible latest/tag pointer to the newly published prerelease instead of preserving an older stable semver latest.

## Best Possible Solution

Maintainers should define the contract explicitly, either documenting publish-time `latest` as semver-managed with retagging as the override path, or adding a deliberate override flow with backend, CLI, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: publish a lower-version or prerelease version with `tags: ['latest']`, and `insertVersion` drops that tag while preserving the previous latest; existing tests assert the same behavior.

## Solution Assessment

Unclear. The requested behavior conflicts with the deliberate #1832 guard, so the best fix depends on whether maintainers want publish-time overrides or clearer docs plus a separate retag flow.
