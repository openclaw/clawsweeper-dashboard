## Summary

Keep open. The reported behavior is source-reproducible on current main, but current code and tests intentionally make publish-time `latest` semver-managed for lower/prerelease publishes, so changing it needs maintainer product judgment rather than cleanup closure.

## What This Changes

The issue asks for `clawhub skill publish --tags latest` to repoint `latest` to the newly published prerelease or lower semver version instead of preserving the existing stable/latest pointer.

## Best Possible Solution

Codify the contract explicitly: either document publish-time `latest` as semver-managed with retagging as the override path, or add a deliberate override flow with backend, CLI, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: publish a lower or prerelease semver with `tags: ['latest']`, and `insertVersion` skips that tag while preserving the previous latest pointer that `inspect` prints.

## Solution Assessment

Unclear. The requested behavior conflicts with the intentional backport guard from #1832, so maintainers need to decide whether publish-time `latest` should be semver-managed, documented, or explicitly overrideable.
