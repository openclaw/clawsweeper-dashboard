## Summary

Keep open. Current main deliberately keeps publish-time `latest` semver-managed for lower or prerelease publishes, and that behavior is covered by regression tests, so the requested behavior needs a maintainer product decision rather than cleanup closure.

## What This Changes

Request that `clawhub skill publish --tags latest` move the visible `latest` pointer to the newly published prerelease version.

## Best Possible Solution

Set an explicit contract: either document publish-time `latest` as semver-managed and point users to retagging, or add a deliberate override path with backend, CLI, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: publish a lower or prerelease semver with `tags: ['latest']`, and `insertVersion` filters that tag while tests assert the previous latest remains selected.

## Solution Assessment

Unclear. The requested behavior conflicts with the intentional backport guard from https://github.com/openclaw/clawhub/pull/1832, so maintainers need to choose between clearer docs/retag guidance and a new explicit override behavior.
