## Summary

Keep open. Current main intentionally ignores publish-time `latest` on lower/prerelease skill publishes, and regression coverage asserts the reported behavior; changing that conflicts with the backport guard from #1832 and needs maintainer product judgment.

## What This Changes

The issue asks for `clawhub skill publish --tags latest` to move the displayed latest tag to a prerelease or lower-semver publish instead of preserving the older stable latest.

## Best Possible Solution

Define the contract explicitly: either document publish-time `latest` as semver-managed and point users to retagging, or add a deliberate override path with backend, CLI, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection shows `clawhub skill publish --tags latest` passes tags through, then `insertVersion` skips `latest` when the incoming semver is not greater; the backport test asserts the same path with `tags: ['latest', 'lts']`.

## Solution Assessment

Unclear. The requested behavior is not a narrow bug fix because it conflicts with the intentional #1832 guard; maintainers should choose between clearer docs/retag guidance and an explicit override behavior.
