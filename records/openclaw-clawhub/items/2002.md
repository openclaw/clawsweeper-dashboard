## Summary

Keep open: current main source-reproduces the reported behavior, but existing code comments and regression tests intentionally protect semver-selected `latest` from explicit publish-time `latest` tags, so changing it needs maintainer policy judgment.

## What This Changes

The issue requests that `clawhub skill publish --tags latest` move the visible `latest` pointer to the newly published prerelease version instead of preserving the older stable semver latest.

## Best Possible Solution

Maintainers should codify one contract: either allow explicit publish-time `latest` overrides with updated backend tests/docs, or keep semver-protected publish behavior and document a clear retag workflow, adding CLI support if web-only retagging is not enough.

## Reproduction Assessment

Yes. Source inspection on current main shows the CLI sends `tags: ["latest"]`, `insertVersion` drops `latest` unless the new version wins the semver comparison, and API/CLI display follows the unchanged `latestVersionId`.

## Solution Assessment

Unclear. The requested behavior matches mutable tag-pointer expectations, but the current implementation and tests intentionally reserve publish-time `latest` for semver promotion, so the narrow fix depends on a maintainer policy decision.
