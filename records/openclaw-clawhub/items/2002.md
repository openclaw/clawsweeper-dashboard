## Summary

Keep open. Current main deliberately preserves the semver-managed latest pointer on lower or prerelease publishes, and changing that behavior needs maintainer product judgment rather than cleanup automation.

## What This Changes

Request that `clawhub skill publish --tags latest` make a prerelease or lower-semver publish become the displayed latest version.

## Best Possible Solution

Define publish-time `latest` semantics explicitly: either document semver-managed publish behavior and use retagging for rollbacks, or implement a deliberate override path that updates all latest-derived state consistently.

## Reproduction Assessment

Yes by source inspection, though I did not run a live publish. On current main, a lower or prerelease semver publish with `tags: ['latest']` flows through `insertVersion`, drops the reserved tag, and has regression coverage asserting the previous latest remains selected.

## Solution Assessment

Unclear. The requested behavior conflicts with the intentional #1832 semver guard, so the maintainable fix depends on whether maintainers want clearer docs and retag guidance or an explicit override mechanism.
