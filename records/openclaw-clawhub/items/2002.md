## Summary

Keep open. Current main still preserves the semver-managed latest pointer for lower or prerelease publishes even when `--tags latest` is supplied, and changing that conflicts with the deliberate #1832 backport guard rather than being a narrow implementation defect.

## What This Changes

The issue asks for `clawhub skill publish --tags latest` to repoint the displayed latest version to the newly published prerelease instead of preserving the older stable latest.

## Best Possible Solution

Maintainers should choose and document the contract: keep publish-time `latest` semver-managed with a deliberate retag workflow, or add an explicit override path that updates backend, CLI, docs, and tests consistently.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: publish an existing skill with a higher current latest, then publish a lower or prerelease version with `tags: ['latest']`; `insertVersion` computes `isNewLatest` as false and drops `latest` from caller tags, and tests assert that outcome.

## Solution Assessment

Unclear. The requested publish behavior is not clearly the best fix because it conflicts with the intentional #1832 guard; the safer direction depends on maintainer policy for publish-time overrides versus a documented retag flow.
