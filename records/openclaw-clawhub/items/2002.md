## Summary

Keep open. Current main intentionally ignores publish-time `latest` for lower/prerelease publishes, and the requested behavior conflicts with the backport-protection contract from #1832, so this needs maintainer product judgment rather than cleanup closure.

## What This Changes

The issue requests that `clawhub skill publish --tags latest` move the displayed `latest` pointer to the newly published prerelease/lower version instead of preserving the higher stable version.

## Best Possible Solution

Define one contract: either document publish-time `latest` as semver-managed and guide users to retag after publish, or add an intentional override path covering backend, CLI, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: publish an existing skill at a lower/prerelease semver with `tags: ['latest']`; `insertVersion` skips the `latest` tag and tests assert the previous latest pointer remains selected.

## Solution Assessment

Unclear. The requested behavior conflicts with the semver backport guard introduced by #1832, so maintainers should first choose between documenting the current contract and adding an explicit override or retag flow.
