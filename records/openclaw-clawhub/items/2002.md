## Summary

Keep open. Current main intentionally ignores publish-time `latest` for lower-semver or prerelease publishes, and changing that would reverse the #1832 backport safety policy rather than fix a narrow implementation defect.

## What This Changes

Publishing a prerelease skill version with `--tags latest` should move the displayed latest version and `tags.latest` to that newly published version.

## Best Possible Solution

Maintainers should choose and document the contract: keep publish-time `latest` semver-managed with retagging for overrides, or add an explicit override publish path with backend, CLI, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: publish a lower-semver or prerelease version with `tags: ['latest']`, and `insertVersion` drops that tag while preserving the previous latest.

## Solution Assessment

Unclear. The requested behavior conflicts with the deliberate #1832 guard, so the best fix depends on whether maintainers want publish-time overrides or clearer docs plus a separate retag flow.
