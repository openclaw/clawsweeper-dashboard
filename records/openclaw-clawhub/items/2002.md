## Summary

Keep open. Current main intentionally preserves the semver-managed latest pointer on lower or prerelease publishes even when `latest` is supplied, and changing that contract needs maintainer product judgment.

## What This Changes

The issue requests that `clawhub skill publish --tags latest` move the visible `latest` tag to the just-published prerelease or lower-semver version.

## Best Possible Solution

Define an explicit contract: either document publish-time `latest` as semver-managed with a separate retag flow, or add a deliberate override path with backend, CLI, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection and regression tests provide a high-confidence path: publish a lower or prerelease semver with `tags: ['latest']`, and `insertVersion` preserves the previous latest pointer.

## Solution Assessment

Unclear. The requested behavior conflicts with the intentional #1832 backport guard, so maintainers need to choose between clearer semver-managed docs and an explicit override design.
