## Summary

Keeping this open: current main reproduces the reported behavior and existing tests intentionally protect it, so changing it needs a maintainer decision about the publish-time `latest` contract rather than conservative cleanup closure.

## What This Changes

The issue asks for `clawhub skill publish --tags latest` to move `latest` to a newly published prerelease version instead of preserving an older stable latest.

## Best Possible Solution

Maintainers should choose and document one contract: either publish-time `latest` is an explicit tag override, or publish keeps semver/backport protection and users must use a retag/rollback path.

## Reproduction Assessment

Yes. Source inspection shows the CLI sends `tags: ["latest"]`, but `insertVersion` drops publish-time `latest` unless the new version wins the semver comparison, and tests assert that behavior.

## Solution Assessment

Unclear. The requested behavior is coherent with tag-pointer semantics, but current code and tests intentionally reserve publish-time `latest` for semver promotion, so the narrow fix depends on a maintainer policy choice.
