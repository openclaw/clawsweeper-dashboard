## Summary

Keep open. Current main intentionally ignores caller-supplied `latest` on lower-version or prerelease publishes because of the #1832 backport guard, while the docs and CLI surface still leave the intended contract ambiguous enough to require maintainer judgment.

## What This Changes

Request that `clawhub skill publish --tags latest` move the visible `latest` pointer to the newly published prerelease or lower-semver version.

## Best Possible Solution

Maintainers should define the contract explicitly, either documenting publish-time `latest` as semver-managed and using retagging for rollbacks, or adding a deliberate override path with backend, CLI, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: publish a lower-semver or prerelease version with `tags: ['latest']`, and `insertVersion` drops that tag while preserving the previous latest; existing tests assert the same behavior.

## Solution Assessment

Unclear. The requested behavior conflicts with the deliberate #1832 guard, so the best fix depends on whether maintainers want publish-time overrides or clearer docs plus a separate retag flow.
