## Summary

Keep open. Current main source-reproduces the reported behavior, but it is protected by an intentional semver/latest guard from #1832 and needs maintainer product/security judgment rather than cleanup closure.

## What This Changes

The issue requests that `clawhub skill publish --tags latest` move the displayed latest version to the newly published prerelease even when semver ordering would otherwise keep a newer stable version as latest.

## Best Possible Solution

Maintainers should define the contract explicitly: either add a deliberate latest-override path with backend/CLI/docs/tests, or document publish-time `latest` as semver-managed and require a separate retag/rollback operation.

## Reproduction Assessment

Yes. A lower-version or prerelease publish with `tags: ['latest']` follows a clear current-main path: the publish succeeds, `latest` is filtered from caller tags, and the previous semver-latest pointer remains selected; existing tests assert that behavior.

## Solution Assessment

Unclear. The requested behavior conflicts with the deliberate #1832 guard, so the best solution is a maintainer decision on the `latest` contract before changing code.
