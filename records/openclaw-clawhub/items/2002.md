## Summary

Keep open. Current main intentionally keeps publish-time `latest` semver-managed for lower or prerelease publishes, so the requested behavior needs maintainer product judgment rather than conservative cleanup.

## What This Changes

The issue requests that `clawhub skill publish --tags latest` move the visible latest tag to the newly published prerelease even when a higher stable version already exists.

## Best Possible Solution

Define the contract explicitly: either document publish-time `latest` as semver-managed with retagging as the override path, or add a deliberate override workflow with backend, CLI, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: publish a lower or prerelease semver with `tags: ['latest']`, and `insertVersion` skips that tag while tests assert the previous latest pointer remains selected.

## Solution Assessment

Unclear. The requested behavior conflicts with the intentional #1832 semver guard, so the maintainable solution depends on whether maintainers want publish-time latest overrides or clearer docs plus the existing retag flow.
