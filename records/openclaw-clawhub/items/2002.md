## Summary

Keep open. Current main still intentionally preserves the semver-greater latest version and ignores caller-supplied publish-time `latest` tags for lower/prerelease publishes, so this is a maintainer policy decision rather than cleanup closure.

## What This Changes

Request to make `clawhub skill publish --tags latest` move displayed latest to the newly published prerelease or lower semver version.

## Best Possible Solution

Maintainers should define the contract explicitly: either add a deliberate latest-override or retag workflow with backend/CLI/docs/tests, or document publish-time `latest` as semver-managed and require separate retag/rollback for manual moves.

## Reproduction Assessment

Yes, source-reproducible. A lower semver or prerelease publish with `tags: ['latest']` makes `isNewLatest` false, drops `latest` from caller tags, and preserves the previous latest pointer; existing tests assert that path.

## Solution Assessment

Unclear. The requested behavior conflicts with the intentional #1832 backport guard, so the narrow maintainable solution depends on a maintainer decision about publish-time tag semantics.
