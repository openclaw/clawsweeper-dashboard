## Summary

Keep open. Current main intentionally ignores publish-time `latest` on lower/prerelease publishes, and existing tests assert that behavior, so this needs maintainer product/API judgment rather than conservative cleanup.

## What This Changes

Allow `clawhub skill publish --tags latest` to move the displayed `latest` pointer to a newly published prerelease version instead of preserving an older stable version.

## Best Possible Solution

Define the contract explicitly: either document publish-time `latest` as semver-managed and direct users to retagging, or add a deliberate override path with backend, CLI/docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: publish a lower or prerelease semver with `tags: ['latest']`, and `insertVersion` skips that tag while tests assert the previous latest pointer remains selected.

## Solution Assessment

Unclear. The requested publish-time override conflicts with the intentional #1832 backport guard, so the best solution depends on whether maintainers want override semantics or clearer docs plus the existing retag flow.
