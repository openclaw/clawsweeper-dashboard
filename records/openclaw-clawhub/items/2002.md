## Summary

Keep open. Current main intentionally preserves the semver-managed latest pointer on lower/prerelease publishes even when publish tags include `latest`, so resolving this needs maintainer product judgment because it conflicts with #1832.

## What This Changes

The issue asks for `clawhub skill publish --tags latest` to repoint the displayed latest version to the newly published prerelease/lower version instead of preserving the semver-managed latest pointer.

## Best Possible Solution

Define the contract explicitly: either document publish-time `latest` as semver-managed with a retag flow, or add a deliberate override path with backend, CLI, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: publish a lower/prerelease semver with `tags: ['latest']`, and `insertVersion` drops that tag while tests assert the previous latest pointer remains selected.

## Solution Assessment

Unclear. The requested publish-time override conflicts with the intentional #1832 semver guard, so maintainers should choose between an explicit override contract and clearer docs for the existing retag flow.
