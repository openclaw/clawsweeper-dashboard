## Summary

Keep open. Current main intentionally ignores publish-time `latest` tags for lower/prerelease publishes, but the CLI/docs contract is ambiguous and changing it would require a maintainer decision against the backport guard from #1832.

## What This Changes

Allow `clawhub skill publish --tags latest` to move the visible latest tag to a newly published prerelease or lower-semver version, or document that publish-time latest is semver-managed.

## Best Possible Solution

Make the `latest` contract explicit: either document publish-time `latest` as semver-managed and point users to a retag flow, or add an intentional override path with backend, CLI, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: publish a lower or prerelease semver with `tags: ['latest']`, and `insertVersion` drops that tag while tests assert the previous latest remains selected.

## Solution Assessment

Unclear. The requested behavior is coherent, but it conflicts with the intentional semver guard from #1832, so maintainers need to choose between explicit publish-time overrides and clearer docs plus retagging.
