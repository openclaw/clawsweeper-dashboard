## Summary

Keep open. Current main preserves `latest` during lower or prerelease publishes by design, including when publish tags include `latest`, so the requested override is a real product-contract decision rather than a cleanup-close candidate.

## What This Changes

The issue asks for `clawhub skill publish --tags latest` to move the displayed latest version to the newly published prerelease instead of keeping the prior stable latest.

## Best Possible Solution

Maintainers should choose and document the contract: either keep publish-time `latest` semver-managed and point users to the retag flow, or add an explicit override path with backend, CLI, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: publish a lower or prerelease semver with `tags: ['latest']`, and `insertVersion` filters out that tag while tests assert the previous latest remains selected.

## Solution Assessment

Unclear. The requested behavior is a maintainable direction only if maintainers want publish-time `latest` to override the semver backport guard; otherwise the narrower solution is docs/CLI copy that explains the existing retag flow.
