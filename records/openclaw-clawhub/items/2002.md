## Summary

Keep open. Current main deliberately keeps publish-time `latest` semver-managed for lower/prerelease publishes, and the requested override conflicts with the backport protection from #1832, so this needs maintainer product judgment rather than cleanup closure.

## What This Changes

Allow `clawhub skill publish --tags latest` to move the visible latest pointer to a prerelease/lower version, or document the intended semver-managed behavior.

## Best Possible Solution

Make the registry contract explicit: either document publish-time `latest` as semver-managed and steer users to retagging, or add a deliberate override path with backend, CLI, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the CLI sends `latest`, `publishVersionForUser` passes it through, and `insertVersion` drops it when the version is not semver-greater than the current latest.

## Solution Assessment

Unclear. The requested behavior is coherent, but it conflicts with the intentional #1832 guard, so maintainers need to choose between clearer docs/retagging and an explicit publish-time override.
