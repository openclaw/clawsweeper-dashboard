## Summary

Keep open. Current main intentionally makes publish-time `latest` semver-managed, even when `latest` is supplied in publish tags, so the report is valid as a product/docs ambiguity rather than a cleanup-close candidate.

## What This Changes

Allow `clawhub skill publish --tags latest` to move `latest` to a prerelease or lower-semver publish, or document that publish-time `latest` is semver-managed.

## Best Possible Solution

Define the product contract explicitly: either document semver-managed publish-time `latest` and point users to retagging, or add a deliberate override path with backend, CLI, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: publish a lower or prerelease semver with `tags: ['latest']`, and `insertVersion` skips that tag while tests assert the previous latest pointer remains selected.

## Solution Assessment

Unclear. The requested behavior conflicts with the intentional #1832 backport guard, so the best fix depends on whether maintainers want publish-time overrides or clearer docs plus the existing retag flow.
