## Summary

Keep open. Current main intentionally preserves the semver-selected latest version on lower-version or prerelease publishes, so the requested behavior is a product/API contract decision rather than a conservative cleanup close.

## What This Changes

The issue requests that `clawhub skill publish --tags latest` move the visible `latest` pointer to the newly published prerelease version instead of preserving an older stable version.

## Best Possible Solution

Maintainers should choose the contract explicitly: document publish-time `latest` as semver-managed with retagging for rollbacks, or add a deliberate override path with backend, CLI, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: publish a lower-semver or prerelease version with `tags: ['latest']`, and `insertVersion` drops that tag while preserving the prior latest pointer.

## Solution Assessment

Unclear. The requested behavior conflicts with the deliberate #1832 guard, so the best solution depends on maintainer policy rather than a narrow implementation fix.
