## Summary

Keep open. Current main source-reproduces the reported publish-path behavior, but it is also covered by intentional backport/security tests, so this needs a maintainer policy decision rather than cleanup closure.

## What This Changes

The issue requests that `clawhub skill publish --tags latest` repoint the displayed latest version to the newly published prerelease when explicitly supplied.

## Best Possible Solution

Maintainers should decide the intended `latest` policy, then align the publish path, docs, and regression tests to either honor explicit publish tags or clearly document semver-protected publish behavior with a retag workflow.

## Reproduction Assessment

Yes, by source inspection: with current main, an existing latest of `2.7.1` and an incoming `2.7.0-2-g0d320d1` with `args.tags` containing `latest` will evaluate `isNewLatest` false and skip the explicit latest tag. I did not run a live publish because this review is read-only.

## Solution Assessment

Unclear. Honoring explicit `latest` matches the tag-pointer model and docs, but it conflicts with current backport/security tests, so the narrow maintainable path starts with a maintainer policy decision.
