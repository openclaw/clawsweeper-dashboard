## Summary

Keep open. Current main intentionally keeps publish-time `latest` semver-managed for lower/prerelease publishes, so changing `--tags latest` into an override needs maintainer product judgment rather than cleanup closure.

## What This Changes

The issue asks for `clawhub skill publish --tags latest` to move the visible `latest` pointer to the newly published prerelease or lower version.

## Best Possible Solution

Define the contract explicitly: either document publish-time `latest` as semver-managed with retagging as the override, or add a deliberate override path with backend, CLI, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: publish a lower/prerelease semver with `tags: ['latest']`, and `insertVersion` skips that tag while tests assert the previous latest pointer remains selected.

## Solution Assessment

Unclear. The requested behavior conflicts with the intentional #1832 semver guard, so the maintainable fix depends on whether maintainers want publish-time overrides or clearer docs plus the existing retag flow.
