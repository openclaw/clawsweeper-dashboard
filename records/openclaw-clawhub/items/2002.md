## Summary

Keep open. Current main still intentionally preserves the semver-managed latest pointer for lower/prerelease publishes even when publish tags include `latest`; changing that behavior would conflict with the #1832 backport guard and needs maintainer product/docs judgment.

## What This Changes

The issue asks for `clawhub skill publish --tags latest` to move `latest` to a prerelease/lower semver version instead of leaving the displayed latest stable release selected.

## Best Possible Solution

Define the contract explicitly: either document publish-time `latest` as semver-managed with retagging as the override path, or add a deliberate override path with backend, CLI, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: publish a lower/prerelease semver with `tags: ['latest']`, and `insertVersion` skips that tag while tests assert the previous latest pointer remains selected.

## Solution Assessment

Unclear. The requested behavior conflicts with the intentional #1832 semver guard, so the best solution depends on whether maintainers want publish-time overrides or clearer docs plus the existing retag flow.
