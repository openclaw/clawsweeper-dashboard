## Summary

Keep open. Current main matches the reported behavior, but it is intentional/tested behavior from #1832, so resolving it needs maintainer product or docs judgment rather than conservative cleanup.

## What This Changes

The issue asks for `clawhub skill publish --tags latest` to move the displayed latest tag to a newly published prerelease or lower semver version.

## Best Possible Solution

Define the contract explicitly: either document publish-time `latest` as semver-managed and point users to retagging, or add a deliberate override path with backend, CLI, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: publish a lower or prerelease semver with `tags: ['latest']`, and `insertVersion` skips that tag while tests assert the previous latest pointer remains selected.

## Solution Assessment

Unclear. The requested behavior conflicts with the intentional #1832 semver guard, so maintainers need to choose between an explicit publish-time override and clearer semver-managed documentation.
