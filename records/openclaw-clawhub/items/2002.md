## Summary

Keep open. Current main confirms the reported behavior and has regression coverage that intentionally preserves the semver-managed latest pointer on lower or prerelease publishes, so changing it needs maintainer product judgment rather than conservative cleanup.

## What This Changes

Request that `clawhub skill publish --tags latest` move the visible `latest` tag to the newly published prerelease or lower version.

## Best Possible Solution

Decide and codify the contract: document publish-time `latest` as semver-managed with a retag flow, or add a deliberate override path with backend, CLI/docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: publish a lower or prerelease semver with `tags: ['latest']`, and `insertVersion` drops that tag while tests assert the previous latest pointer remains selected.

## Solution Assessment

Unclear. The requested publish-time override conflicts with the intentional #1832 semver guard, so the narrow maintainable solution depends on a maintainer decision between clearer docs and an explicit override mechanism.
