## Summary

Keep open. Current main intentionally keeps publish-time `latest` semver-managed for lower or prerelease publishes, even when `latest` is supplied, so resolving this needs a maintainer choice between override semantics and clearer docs.

## What This Changes

Request that `clawhub skill publish --tags latest` move `latest` to a prerelease/lower version, or document the current semver-managed behavior.

## Best Possible Solution

Maintainers should choose an explicit contract: either document publish-time `latest` as semver-managed and point rolling prerelease users to retagging, or add a deliberate override path with backend, CLI, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: publish a lower or prerelease semver with `tags: ['latest']`, and `insertVersion` skips that tag while tests assert the previous latest pointer remains selected.

## Solution Assessment

Unclear. The requested override conflicts with the intentional #1832 backport guard, so the narrowest maintainable solution depends on a maintainer product decision before code changes.
