## Summary

Keep open. Current main intentionally preserves the semver-managed latest pointer on lower or prerelease publishes even when `latest` is supplied during publish, so the requested behavior conflicts with the merged backport guard and needs maintainer product judgment.

## What This Changes

The issue asks for `clawhub skill publish --tags latest` to move the visible `latest` tag to the newly published prerelease version instead of preserving an older stable release.

## Best Possible Solution

The durable end state is an explicit contract: either document publish-time `latest` as semver-managed and direct users to retagging, or add a deliberate override path with backend, CLI, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: publish a lower or prerelease semver with `tags: ['latest']`, and `insertVersion` skips that tag while tests assert the previous latest pointer remains selected.

## Solution Assessment

Unclear. The requested behavior conflicts with the intentional semver guard from PR #1832, so the best solution depends on whether maintainers want publish-time overrides or clearer docs plus the existing retag flow.
