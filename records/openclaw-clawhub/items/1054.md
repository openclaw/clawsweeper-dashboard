## Summary

Keep open. Current main intentionally blocks sign-in for permanently deleted accounts and hides deactivated users from active-session queries, but ClawHub cleanup policy keeps issues open and the remaining recovery/re-registration behavior needs maintainer account-lifecycle judgment.

## What This Changes

The issue reports that after deleting a ClawHub account, GitHub OAuth completes but no active ClawHub user/session is loaded.

## Best Possible Solution

Maintainers should define and document the post-deletion recovery path: keep permanent blocking with clear UI/support, or add a verified purge/re-registration flow that preserves deletion privacy invariants.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount writes deactivatedAt/purgedAt, afterUserCreatedOrUpdated calls handleDeletedUserSignIn and throws for deactivated users, and users.me returns null through getOptionalActiveAuthUserId; I did not run a live OAuth flow.

## Solution Assessment

Unclear. Automatically creating a session after deletion is not clearly the best fix because it conflicts with the documented irreversible deletion contract; the maintainable path depends on maintainer policy.
