## Summary

Keep open. Current main intentionally blocks reauth after user-initiated deletion and hides deactivated users from active-session helpers, while docs and tests define deletion as irreversible; resolving the account access problem needs maintainer account-lifecycle policy rather than cleanup closure.

## What This Changes

The issue asks for a usable GitHub OAuth session and profile after a deleted ClawHub account signs in again.

## Best Possible Solution

Maintainers should choose and document the post-deletion lifecycle, then align OAuth callback behavior, active-user helpers, UI messaging, tests, and any administrative purge or re-registration support around that policy.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, the OAuth callback rejects deactivated users in handleDeletedUserSignIn, and users.me returns null through getOptionalActiveAuthUserId.

## Solution Assessment

Unclear. Automatically creating a session is not the best established fix because current docs and tests intentionally make deletion irreversible; a maintainer-approved re-registration or administrative purge flow would be safer if policy changes.
