## Summary

Keep open. Current main intentionally blocks reauthentication for permanently deleted accounts and now surfaces blocked-account messaging, but ClawHub cleanup policy keeps open issues open and the remaining path needs maintainer account-lifecycle/security judgment rather than cleanup closure.

## What This Changes

The issue requests that GitHub OAuth create a usable ClawHub session after a previously deleted account signs in again.

## Best Possible Solution

Maintainers should choose and document the post-deletion lifecycle: keep permanent blocking with clear support UX, provide a verified administrative purge path, or implement an explicit re-registration flow with tests and spec updates.

## Reproduction Assessment

Yes, by source inspection. Deleting an account writes deactivatedAt/purgedAt, OAuth reauth hits handleDeletedUserSignIn and rejects, and users.me returns null through the active-user helper.

## Solution Assessment

Unclear. Directly creating a session after deletion is not the best established fix because current specs and tests define deletion as irreversible; a maintainer-approved block, purge, or re-registration policy is safer.
