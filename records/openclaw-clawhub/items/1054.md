## Summary

Keep open. Current main intentionally treats post-deletion GitHub reauth as a blocked account-lifecycle state rather than a missing session write, and ClawHub issue cleanup policy keeps issues open for maintainer handling.

## What This Changes

The issue asks ClawHub to create an authenticated session and load a profile after a previously deleted account signs in again with GitHub OAuth.

## Best Possible Solution

Maintainers should choose and document the post-deletion lifecycle: permanent blocking with clear support UX, verified administrative purge, or a deliberate secure re-registration flow with tests.

## Reproduction Assessment

Yes. The source-backed path is deleteAccount setting deactivatedAt/purgedAt, OAuth callback rejecting in handleDeletedUserSignIn, and users.me returning null through getOptionalActiveAuthUserId; I did not execute the live OAuth flow.

## Solution Assessment

Unclear. Automatically creating a session after deletion conflicts with current docs and tests, so the maintainable solution depends on maintainer policy around irreversible deletion, purge, or re-registration.
