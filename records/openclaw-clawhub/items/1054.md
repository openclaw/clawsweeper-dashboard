## Summary

Keep open. Current main intentionally blocks sign-in for permanently deleted accounts and now surfaces blocked-account messaging, but this is an open ClawHub issue and the remaining path needs maintainer account-lifecycle/security policy rather than cleanup closure.

## What This Changes

The issue asks ClawHub to create a usable session and load the profile after GitHub OAuth when the account was previously deleted.

## Best Possible Solution

Maintainers should choose and document the post-deletion lifecycle, then keep permanent blocking with clear support UX, provide a verified administrative purge path, or implement an explicit re-registration flow with tests and spec updates.

## Reproduction Assessment

Yes, by source inspection. Deleting an account writes deactivatedAt/purgedAt, OAuth reauth hits handleDeletedUserSignIn and rejects, and users.me returns null through the active-user helper.

## Solution Assessment

Unclear. Directly creating a session after deletion is not the best established fix because current specs and tests define deletion as irreversible; a maintainer-approved block, purge, or re-registration policy is safer.
