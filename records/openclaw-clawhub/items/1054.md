## Summary

Keep open. Current main still matches the reported symptom through the intentional deleted-account lifecycle, and ClawHub cleanup policy keeps issues open; the remaining decision is account recovery/re-registration policy, not a narrow session-write bug.

## What This Changes

The issue asks for GitHub OAuth login after account deletion to create a recognized ClawHub session and load the user profile.

## Best Possible Solution

Maintainers should define the post-deletion account lifecycle, then align code, support tooling, docs, and tests around either permanent blocking with clear messaging, verified purge, or an explicit re-registration flow.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: account deletion marks the user deactivated, OAuth reauth is rejected in `handleDeletedUserSignIn`, and `users.me` returns null through the active-auth helper.

## Solution Assessment

No for an automated code change. Creating a session after deletion would conflict with the documented irreversible deletion contract; the maintainable solution depends on maintainer policy.
