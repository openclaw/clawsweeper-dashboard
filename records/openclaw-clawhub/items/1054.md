## Summary

Keep open. Current main intentionally blocks deleted-account reauth and hides deactivated users from active-session lookups, while ClawHub cleanup policy keeps issues open unless they are resolved PRs; changing this needs maintainer account-lifecycle and security policy.

## What This Changes

The issue asks for GitHub OAuth login after ClawHub account deletion to create a usable session and load a profile instead of returning to a logged-out UI.

## Best Possible Solution

Maintainers should choose and document the post-deletion lifecycle, then keep permanent blocking with clear support UX, add a verified admin purge path, or implement an explicit re-registration flow with tests and spec updates.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount marks the user deactivated/purged, the OAuth callback rejects that account, and users.me returns null through the active-user helper.

## Solution Assessment

Unclear. Creating a session after deletion is not the narrowest established fix because current specs and tests define deletion as irreversible; a maintainer-approved block, purge, or re-registration policy is safer.
