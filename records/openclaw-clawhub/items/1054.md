## Summary

Keep open. Current main intentionally blocks OAuth reauthentication for deleted accounts and now surfaces blocked-account messaging, but this is a ClawHub issue and the remaining account-lifecycle/support decision is not eligible for cleanup closure.

## What This Changes

The issue asks for GitHub OAuth login after account deletion to create a usable ClawHub session and load the profile instead of leaving the UI logged out.

## Best Possible Solution

Maintainers should decide and document the post-deletion lifecycle, then either preserve permanent blocking with clearer support UX, provide a verified administrative purge process, or implement an explicit re-registration flow with tests and spec updates.

## Reproduction Assessment

Yes, by source inspection for the post-deletion logged-out state. Deleting an account writes inactive tombstone fields, OAuth reauthentication hits the deleted-user rejection path, and active-session helpers then return no user.

## Solution Assessment

Unclear. Directly creating a session after deletion is not the established narrow fix because current specs and tests define deletion as irreversible; the safer path is a maintainer-approved block, purge, or re-registration policy.
