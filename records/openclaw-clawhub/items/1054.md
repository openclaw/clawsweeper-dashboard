## Summary

Keep open. Current main shows the reported login-loop symptom is an intentional deleted-account lifecycle boundary rather than a simple missed session write, and ClawHub cleanup policy keeps issues open unless maintainers resolve them.

## What This Changes

The issue asks ClawHub to create an authenticated session/profile after a user deletes their account and signs in again with GitHub OAuth.

## Best Possible Solution

Maintainers should define the intended post-deletion lifecycle and align code, docs, support tooling, and UI around either permanent blocking, a verified administrative purge path, or a deliberate tombstone/re-registration flow.

## Reproduction Assessment

Yes. The source-backed path is deleteAccount setting deactivatedAt, a later OAuth callback rejecting in handleDeletedUserSignIn, and users.me returning null through getOptionalActiveAuthUserId.

## Solution Assessment

Unclear. Creating a session after deletion is not clearly the best fix because it conflicts with the documented irreversible deletion contract; the narrow maintainable solution depends on maintainer policy.

  - confidence: 0.9
- **[medium] Avoid automatic account resurrection:** `convex/auth.ts:99`
