## Summary

Keep open. Current main maps the symptom to the deliberate deleted-account lifecycle, and ClawHub issue cleanup policy does not auto-close issues; changing the behavior needs maintainer account-recovery policy rather than a narrow source fix.

## What This Changes

The issue requests that a user who deleted their ClawHub account be able to complete GitHub OAuth and get an active session/profile instead of remaining logged out.

## Best Possible Solution

Maintainers should choose the post-deletion lifecycle and align code, docs, UI messaging, tests, and support tooling around permanent blocking, verified admin purge, or an explicit re-registration flow.

## Reproduction Assessment

Yes. The source-backed path is deleteAccount setting deactivatedAt/purgedAt, the OAuth callback rejecting in handleDeletedUserSignIn, and users.me returning null through getOptionalActiveAuthUserId; I did not run a live GitHub OAuth flow.

## Solution Assessment

Unclear. Automatically creating a session after deletion is not the narrowest maintainable fix while the repository documents deletion as irreversible; the safer direction is a maintainer-defined lifecycle and support path.
