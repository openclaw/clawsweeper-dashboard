## Summary

Keep open. Current main intentionally blocks reauth for permanently deleted accounts and now has UI paths for auth errors, but the requested post-deletion session behavior conflicts with the documented irreversible deletion policy and needs maintainer account-lifecycle judgment.

## What This Changes

The issue requests that GitHub OAuth after account deletion create a usable ClawHub session and load the profile instead of leaving the user logged out.

## Best Possible Solution

Maintainers should define and document the intended post-deletion lifecycle, then align auth, support tooling, UI messaging, and tests around either permanent blocking, verified administrative purge, or an explicit re-registration flow.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, OAuth rejects through handleDeletedUserSignIn, and users.me returns null for deactivated users.

## Solution Assessment

Unclear. Creating a session after deletion is not clearly the best fix because it conflicts with the current irreversible-deletion contract; maintainers need to choose the lifecycle policy first.
