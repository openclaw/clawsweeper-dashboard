## Summary

Keep open. ClawHub cleanup policy does not auto-close issues, and current main confirms this is an account-lifecycle/support decision: deleted accounts are intentionally deactivated, rejected during OAuth reauth, and hidden from active-session lookup.

## What This Changes

The issue reports that after deleting a ClawHub account, a later GitHub OAuth login completes but no authenticated user session is created.

## Best Possible Solution

Maintainers should choose and document the post-deletion lifecycle, then align OAuth callback behavior, active-user helpers, UI messaging, tests, docs, and any administrative purge or re-registration path around that policy.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, the OAuth callback rejects that user in handleDeletedUserSignIn, and users.me returns null through getOptionalActiveAuthUserId.

## Solution Assessment

Unclear. Automatically creating a session is not the best established fix because current docs and tests intentionally make deletion irreversible; a maintainer-approved re-registration, purge, or clearer permanent-block UX would be safer.
