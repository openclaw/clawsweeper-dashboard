## Summary

Keep open. Current main still maps the reported symptom to the deliberate deleted-account lifecycle: deletion deactivates the user, later OAuth reauth is rejected, and active-session helpers return no user. The remaining work needs maintainer account-recovery policy, especially because ClawHub issue cleanup is not allowed to auto-close issues.

## What This Changes

The issue requests that a user who deleted their ClawHub account can complete GitHub OAuth and receive an active ClawHub session again.

## Best Possible Solution

Maintainers should define the intended post-deletion lifecycle and then align code, docs, tests, support tooling, and UI around permanent blocking, verified administrative purge, or a deliberate re-registration flow.

## Reproduction Assessment

Yes. The source-backed path is deleteAccount setting deactivatedAt, OAuth callback rejecting in handleDeletedUserSignIn, and users.me returning null through getOptionalActiveAuthUserId.

## Solution Assessment

Unclear. Creating a session after deletion is not clearly the best fix because it conflicts with the documented irreversible deletion contract; the narrow maintainable answer depends on maintainer policy.
