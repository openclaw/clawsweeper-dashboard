## What This Changes

The PR adds a shared moderation-state guard for direct and accepted skill ownership transfers, with regression tests, a changelog entry, and org ownership spec guidance.

## Best Possible Solution

Adopt the shared fail-closed moderation guard for all ownership-transfer writes unless maintainers explicitly require a narrow, tested staff exception for quarantined artifacts.

## Reproduction Assessment

Yes. Source inspection on current main shows direct owner transfers can reach ownership writes without a moderation guard, while accepted transfers only block malicious, hidden, or removed states before patching ownership.

## Solution Assessment

Yes, with a policy caveat. Centralizing the moderation-transfer predicate and checking it after authorization but before owner writes is the narrow maintainable fix; maintainers still need to decide whether a staff-only emergency exception is required.
