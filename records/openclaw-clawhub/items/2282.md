## Summary

Keep this PR open. Current main does not have the direct-transfer moderation guard, so the patch is not obsolete, but the proposed guard misses existing malicious-state signals and the PR only provides tests/CI rather than real behavior proof for this security-sensitive path.

## What This Changes

The PR adds a moderation guard to direct skill owner transfers, adds ownership-transfer regression tests, updates the org ownership spec, and records a changelog fix.

## Best Possible Solution

Use the central moderation verdict semantics, keep the org ownership spec aligned with the actual transfer paths, and merge only after redacted real runtime proof shows clean transfers still work and moderated transfers are rejected.

## Reproduction Assessment

Yes, from source inspection: current main reaches transferSkillOwnershipAndEmbeddings after authorization without a moderation-state check. I did not run a live Convex mutation, but the path is clear in the current source.

## Solution Assessment

No as written. The guard is in the right ordering for direct transfers, but it should use the same malicious/suspicious semantics as existing moderation code and the spec should not claim broader coverage than the implementation provides.
