## Summary

Keep this PR open: the transfer-boundary direction is valid, but the new guard still misses ClawHub's legacy `blocked.malware` malicious state, and contributor proof is test/CI-only.

## What This Changes

This PR adds moderation-state guards to direct and accepted skill ownership transfers, plus regression tests, a changelog entry, and an ownership-transfer spec note.

## Best Possible Solution

Complete a shared ownership-transfer eligibility guard for all non-public, suspicious, and malicious skill states in both transfer paths, then land it with regression coverage and redacted live proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main reaches direct ownership writes without a moderation guard, and the accepted-transfer path only blocks malicious verdicts or hidden/removed status before writing ownership.

## Solution Assessment

No, not yet. The guard is placed at the right boundary, but it needs to include legacy malware-blocked state such as `moderationFlags: ["blocked.malware"]` in both helpers before it is the best fix.
