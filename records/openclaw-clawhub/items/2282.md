## Summary

Keep open for maintainer review: the patch is a focused moderation/ownership hardening change with source-backed root cause, live Convex proof, and no blocking correctness or security findings, but it intentionally changes transfer behavior for quarantined skills.

## What This Changes

The PR adds a shared moderation-state guard for direct and accepted skill ownership transfers, regression tests, a changelog entry, and org ownership spec guidance.

## Best Possible Solution

Adopt the shared fail-closed moderation guard for all ownership-transfer writes, unless maintainers explicitly need a narrow, tested staff exception for quarantined artifacts.

## Reproduction Assessment

Yes. Source inspection on current main shows the direct transfer path can reach the ownership write without a moderation guard, while accepted transfers only block malicious, hidden, or removed states before patching the owner.

## Solution Assessment

Yes, with a policy caveat. Centralizing the moderation transfer predicate and checking it after authorization but before owner writes is the narrow maintainable fix; the only open question is whether maintainers want a staff-only emergency exception.
