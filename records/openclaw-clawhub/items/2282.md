## What This Changes

Adds a shared moderation-transfer guard for direct and accepted skill ownership transfers, with regression tests, a changelog entry, and org ownership spec guidance.

## Best Possible Solution

Adopt the shared fail-closed guard for all ownership-transfer writes, unless maintainers explicitly choose and test a narrow staff-only exception for quarantined artifacts.

## Reproduction Assessment

Yes, source-reproducible. Current main lets direct owner transfers reach the ownership write without checking moderation state, while accepted pending transfers only block a subset of moderated states.

## Solution Assessment

Yes, with a policy caveat. Centralizing the moderation predicate and checking after authorization but before ownership writes is the narrow maintainable fix; maintainers still need to decide whether any staff-only exception belongs here.
