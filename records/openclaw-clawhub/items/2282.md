## Summary

The PR is still useful and not implemented on current main: current direct/admin/publish-time ownership paths can still write ownership without the shared moderation-transfer guard, while the branch adds that guard, focused tests, and live Convex proof. There are no blocking code findings, but the fail-closed behavior is compatibility-sensitive and needs maintainer policy acceptance before merge.

## What This Changes

The PR adds a shared moderation-transfer guard for skill ownership writes, applies it to direct, accepted, admin, reclaim, and publish-time migration paths, and adds regression coverage plus org ownership spec guidance.

## Best Possible Solution

Adopt the shared fail-closed ownership-transfer guard unless maintainers explicitly choose and test a narrow staff-only exception for quarantined artifacts.

## Reproduction Assessment

Yes, source-reproducible. Current main has ownership write paths in transferSkillOwnershipAndEmbeddings/changeOwner/reclaim and a narrower accepted-transfer guard that do not cover the full moderated-skill state set the PR blocks.

## Solution Assessment

Yes, with a policy caveat. Centralizing the moderation predicate and checking after authorization but before ownership writes is the narrow maintainable fix; maintainers still need to decide whether any staff-only exception belongs here.
