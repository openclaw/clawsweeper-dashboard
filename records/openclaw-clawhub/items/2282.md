## Summary

Keep open: current main still lacks the direct-transfer moderation guard, so this PR contains useful security hardening, but it needs a rebase after the merged transfer-acceptance work, contributor real behavior proof, and an invariant alignment fix before merge.

## What This Changes

Adds a direct-transfer moderation eligibility guard, regression coverage, a changelog entry, and org lifecycle spec wording for ownership transfers.

## Best Possible Solution

Rebase on current main, use one shared moderation-transfer eligibility rule for direct and accepted ownership writes or narrow the spec to direct transfers, keep focused regression coverage, and add redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection on current main shows an authorized direct transfer reaches the ownership patch without any moderation eligibility check, and the PR's tests target that exact handler path.

## Solution Assessment

No. The direct guard is the right direction, but the patch's broad all-transfer invariant is not yet matched by accepted-transfer handling for suspicious states; sharing the guard or narrowing the spec would be safer.
