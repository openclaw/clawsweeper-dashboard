## Summary

Keep open: current main still leaves the direct owner-transfer path without a moderation guard and only partially guards accepted transfers; the patch looks correct, but this external PR still lacks real setup proof beyond tests and CI output.

## What This Changes

This PR adds moderation-state guards to direct and accepted skill ownership transfers, plus regression tests, changelog text, and an ownership-transfer spec note.

## Best Possible Solution

Land this PR or an equivalent guard after redacted real setup proof shows both direct and accepted transfer attempts are rejected for moderated or suspicious skills.

## Reproduction Assessment

Yes, by source inspection. Current main's direct transfer path writes new ownership without a moderation guard, and accepted transfers only cancel malicious/hidden/removed skills, leaving active suspicious states uncovered.

## Solution Assessment

Yes. The PR places the guard after source authorization and before ownership writes in both transfer paths; a shared helper could reduce duplication later, but the current approach is narrow and maintainable.
