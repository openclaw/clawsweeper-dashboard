## What This Changes

The branch adds a shared moderation-transfer guard for direct and accepted skill ownership transfers, regression tests, a changelog entry, and org ownership spec guidance.

## Best Possible Solution

Adopt the shared fail-closed moderation guard for all ownership-transfer writes, unless maintainers explicitly choose and test a narrow audited staff-only exception for quarantined artifacts.

## Reproduction Assessment

Yes. Source inspection on current main shows direct transfers can reach the ownership write without a moderation guard, while accepted transfers only block malicious, hidden, and removed states.

## Solution Assessment

Yes, with a policy caveat. Centralizing the moderation transfer predicate and checking it after authorization but before owner writes is the narrow maintainable fix; maintainers still need to decide whether any staff-only emergency exception belongs here.
