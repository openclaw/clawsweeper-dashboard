Keep open. ClawHub cleanup policy keeps issues open, and current main still has a source-reproducible import-to-publish path that can emit the reported GitHub API rate-limit failure when the GitHub account-age lookup receives a 403 or 429 from GitHub.

## What This Changes

Publishing a skill through GitHub import fails with a GitHub API rate-limit message that does not resolve after waiting.

## Best Possible Solution

Keep this in the publish/import rate-limit cluster, verify production GITHUB_TOKEN and Convex 403/429 failures first, then reduce avoidable publish-time GitHub lookups without weakening account-age enforcement.

Yes, from source inspection. Publish through GitHub import as a user without cached githubCreatedAt and have the GitHub account lookup return 403 or 429; current main wraps that into the reported import publish message.
