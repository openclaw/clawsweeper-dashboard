## What This Changes

Publishing an imported GitHub skill is reported to fail repeatedly with the wrapped GitHub API rate-limit message instead of recovering after waiting.

## Best Possible Solution

Verify production Convex GITHUB_TOKEN/quota and failure logs first, then either correct deployment configuration or make the account-age lookup more resilient and actionable without weakening the publish abuse gate.

## Reproduction Assessment

Yes, source-level. On current main, GitHub import calls publishVersionForUser without a bypass; for a user missing githubCreatedAt, a GitHub 403 or 429 from the account lookup produces the reported wrapped error.

## Solution Assessment

Unclear. The narrowest maintainable fix depends on whether production lacks a valid GITHUB_TOKEN/quota or the code should reduce first-publish lookups and improve retry/reset detail while preserving the anti-abuse gate.
