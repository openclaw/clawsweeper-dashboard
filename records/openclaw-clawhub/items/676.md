## Summary

Keep open. The ClawHub cleanup policy keeps issues open, and current main still has a source-reachable GitHub import publish path that can emit the reported rate-limit message when the GitHub account-age lookup receives a GitHub 403 or 429.

## What This Changes

The issue reports GitHub-import skill publishing failing with a GitHub API rate-limit message that does not resolve by waiting.

## Best Possible Solution

Keep this in the publish/import rate-limit cluster, verify production GITHUB_TOKEN and GitHub 403/429 failures first, then reduce avoidable first-publish GitHub lookups or improve retry/reset guidance without weakening account-age verification.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: GitHub import calls publishVersionForUser, publish runs requireGitHubAccountAge for normal users, and an uncached GitHub account lookup returning 403 or 429 emits the reported message.

## Solution Assessment

Unclear. Existing GITHUB_TOKEN support may be the operational fix, but choosing between deployment configuration, safer caching/backfill, retries, or clearer error details requires production evidence and anti-abuse gate judgment.
