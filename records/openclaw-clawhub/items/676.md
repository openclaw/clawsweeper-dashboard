## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has a reachable GitHub-import-to-publish path that can emit the reported wrapped rate-limit error when the GitHub account-age lookup receives a 403 or 429 from GitHub.

## What This Changes

Publishing a skill through GitHub import should not keep failing with the GitHub API rate-limit message after waiting.

## Best Possible Solution

Keep this in the publish/import rate-limit cluster, verify production Convex failures and GITHUB_TOKEN configuration first, then refine the lookup or caching only if the evidence shows code is the right layer.

## Reproduction Assessment

Yes, source-level. A user without cached githubCreatedAt who imports a skill runs publishVersionForUser, then requireGitHubAccountAge, and a GitHub 403 or 429 produces the reported wrapped publish error.

## Solution Assessment

Unclear. The narrowest maintainable fix depends on production GITHUB_TOKEN coverage, quota behavior, and abuse-gate policy; bypassing the account-age gate would not be an appropriate fix.
