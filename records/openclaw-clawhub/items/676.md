## Summary

Keep open. Current main still has a source-reachable GitHub import publish path that can surface the reported rate-limit message, and ClawHub cleanup policy does not auto-close issues in this pass.

## What This Changes

Publishing a skill via GitHub import can fail with a GitHub API rate-limit message that does not resolve after waiting.

## Best Possible Solution

Keep this in the publish/import rate-limit cluster, verify production GITHUB_TOKEN and Convex failures first, then improve retry/reset guidance or reduce uncached first-publish lookups without weakening account-age verification.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: GitHub import calls publishVersionForUser, publishing runs requireGitHubAccountAge for normal users, and an uncached GitHub account lookup returning 403 or 429 emits the reported message.

## Solution Assessment

Unclear. The narrowest fix may be production GITHUB_TOKEN configuration, safer caching or backfill, retry/reset details, or a code change, and choosing among those requires production evidence plus anti-abuse gate judgment.
