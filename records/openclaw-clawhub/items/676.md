## Summary

Keep open. Current main still has the reported import-to-publish failure path: GitHub import wraps publish errors, publish still runs the GitHub account-age gate, and that gate throws the reported rate-limit text for GitHub 403/429 responses when `githubCreatedAt` is uncached.

## What This Changes

Publishing a skill through GitHub import can fail with a GitHub API rate-limit error that waiting does not resolve.

## Best Possible Solution

Verify production `GITHUB_TOKEN` and Convex failures first, then reduce avoidable GitHub account-age lookups or improve retry/reset handling without weakening the publish abuse gate.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: GitHub import reaches `publishVersionForUser`, the account-age gate runs for uncached users, and GitHub 403/429 responses produce the reported wrapped message.

## Solution Assessment

Unclear. The narrowest maintainable fix depends on production evidence: it may be Convex `GITHUB_TOKEN` configuration, safer caching/backfill of `githubCreatedAt`, or clearer rate-limit retry details while preserving the account-age gate.
