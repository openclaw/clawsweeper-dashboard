## What This Changes

Publishing a skill via GitHub import fails with a wrapped GitHub API rate-limit error even after waiting.

## Best Possible Solution

Keep this in the publish/import rate-limit cluster, verify production GITHUB_TOKEN and Convex failures first, then refine the GitHub lookup, caching, or operator guidance without weakening the account-age gate.

## Reproduction Assessment

Yes, at source level. A user without cached githubCreatedAt can import a skill, publishVersionForUser runs requireGitHubAccountAge, and a GitHub 403 or 429 produces the wrapped error from the report.

## Solution Assessment

Unclear. The narrowest maintainable fix depends on whether production is missing GITHUB_TOKEN, exhausting authenticated quota, or doing too many first-publish lookups; bypassing the gate would not be appropriate.
