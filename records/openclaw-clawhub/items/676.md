## Summary

Keep open. This is a ClawHub issue, and current main still has a source-reproducible GitHub import-to-publish path that can emit the reported rate-limit message; related publish rate-limit reports also remain open.

## What This Changes

Publishing a skill through GitHub import should not fail indefinitely with the GitHub API rate-limit publish error.

## Best Possible Solution

Keep this in the publish/import rate-limit cluster, verify production Convex failures and GITHUB_TOKEN quota first, then correct deployment config or reduce avoidable first-publish GitHub lookups without weakening the account-age gate.

## Reproduction Assessment

Yes, from source inspection. GitHub import calls publishVersionForUser, publishVersionForUser runs requireGitHubAccountAge by default, and a missing githubCreatedAt plus a GitHub 403/429 produces the reported inner and outer error text.

## Solution Assessment

Unclear. Existing GITHUB_TOKEN support may make this an operational fix, while code mitigations need maintainer judgment because they touch the security-sensitive account-age gate.
