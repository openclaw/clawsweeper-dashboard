## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has a source-reproducible GitHub import-to-publish path that can emit the reported wrapped rate-limit message when GitHub returns 403 or 429 during the account-age lookup.

## What This Changes

Publishing a skill through GitHub import fails with a wrapped GitHub API rate-limit message that does not clear after waiting.

## Best Possible Solution

Keep this in the publish/import rate-limit cluster, verify production Convex failures and GITHUB_TOKEN quota first, then either correct deployment configuration or reduce uncached GitHub lookups with actionable retry detail while preserving the abuse gate.

## Reproduction Assessment

Yes, source-level. On current main, GitHub import calls publishVersionForUser; for a user missing cached githubCreatedAt, a GitHub 403 or 429 in requireGitHubAccountAge produces the reported wrapped error.

## Solution Assessment

Unclear. The narrowest safe solution depends on production logs and quota state: it may be a Convex GITHUB_TOKEN deployment fix, or a code change to reduce uncached lookups and improve retry detail without bypassing the account-age gate.
