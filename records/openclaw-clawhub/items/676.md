## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has a source-reproducible GitHub-import publish path that can emit the reported rate-limit message when the GitHub account-age lookup receives a 403 or 429.

## What This Changes

Publishing a skill through GitHub import fails with a persistent GitHub API rate-limit error during the publish step.

## Best Possible Solution

Keep the account-age gate, verify production Convex 403/429 logs and GITHUB_TOKEN quota first, then either correct deployment configuration or reduce avoidable first-publish GitHub lookups without weakening immutable-ID verification.

## Reproduction Assessment

Yes, by source inspection. An uncached user publishing through GitHub import reaches requireGitHubAccountAge, and existing tests show GitHub 403 or 429 responses throw the reported rate-limit text; I did not run a live production publish.

## Solution Assessment

Unclear. The narrowest fix depends on production token/quota state and logs, and any code mitigation must preserve the security-sensitive GitHub account-age verification.
