Keep open. ClawHub cleanup policy keeps issues open, and current main still has a source-reproducible GitHub-import publish path that can emit the reported rate-limit message when the GitHub account-age lookup receives a 403 or 429.

## Best Possible Solution

Keep the account-age gate, verify production GITHUB_TOKEN/quota and Convex 403/429 logs, then reduce avoidable publish-time GitHub lookups or add actionable retry/reset detail without weakening immutable-ID verification.
