## Summary

Keep open. ClawHub cleanup policy keeps open issues open, and current main still has a reachable GitHub-import-to-publish path that emits the reported rate-limit message when the GitHub account-age lookup receives a 403 or 429 from GitHub.

## What This Changes

Publishing a skill through GitHub import fails with a GitHub API rate-limit error that does not clear after waiting.

## Best Possible Solution

Keep this in the publish/import rate-limit cluster, verify production Convex failures and GITHUB_TOKEN configuration first, then refine the GitHub account lookup or caching only if that evidence shows code is the right layer.

## Reproduction Assessment

Yes. A source-level reproduction is clear: a user missing cached githubCreatedAt imports a skill, publishVersionForUser runs requireGitHubAccountAge, and a GitHub 403 or 429 produces the reported wrapped import error; production frequency still needs logs.

## Solution Assessment

Unclear. The narrowest maintainable fix depends on whether production is missing a valid GITHUB_TOKEN, exhausting authenticated quota, or doing too many first-publish lookups; bypassing the account-age gate would not be an appropriate solution.
