## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has a reachable GitHub-import-to-publish path that can emit the reported GitHub API rate-limit message when the account-age lookup receives a 403 or 429.

## What This Changes

The issue reports skill publishing failing during GitHub import with a GitHub API rate-limit error that does not resolve after waiting.

## Best Possible Solution

Keep this in the publish/import rate-limit cluster, verify production GITHUB_TOKEN and Convex failures first, then refine the GitHub account lookup only if needed so true rate limits are actionable and uncached first-publish lookups are reduced without weakening the abuse gate.

## Reproduction Assessment

Yes, at source level: a user without cached githubCreatedAt can import a skill, publishVersionForUser runs requireGitHubAccountAge, and a GitHub 403 or 429 produces the reported wrapped import error. I did not run a live production reproduction because the decisive data is Convex logs and deployment env.

## Solution Assessment

Unclear. The narrowest maintainable fix depends on whether production is missing a valid GITHUB_TOKEN, exhausting authenticated quota, or doing too many first-publish lookups; bypassing the account-age gate would not be the right solution.
