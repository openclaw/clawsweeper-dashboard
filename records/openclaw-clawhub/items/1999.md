## Summary

Keep open: this is an open ClawHub issue, and current production main still has a reproducible GitHub sign-in availability failure before the reported callback/no-JWT path can be replayed.

## What This Changes

Report that GitHub OAuth returns to clawhub.ai without persisting Convex Auth JWT/session tokens, leaving users signed out.

## Best Possible Solution

Restore production GitHub OAuth for active accounts, preserve explicit deleted/banned-account rejection, and add targeted callback/session smoke or regression coverage after the failing layer is identified.

Yes for the current production sign-in availability failure: current main is tagged prod/v2026.05.10.4 and both canonical and direct Convex GitHub sign-in endpoints return HTTP 500 while the site and skills API return HTTP 200. No for the exact callback-succeeds/no-JWT variant, because the flow currently fails one step earlier.
