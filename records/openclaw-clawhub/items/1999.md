## Summary

Keep open: this is an open ClawHub issue and the current production deployment still shows a credible GitHub OAuth availability failure, with the sign-in route returning HTTP 500 while the site and public API respond normally.

## What This Changes

The issue reports a GitHub OAuth sign-in loop where the Convex Auth callback appears to complete but no JWT or refresh token is persisted for clawhub.ai.

## Best Possible Solution

Restore GitHub OAuth so active accounts receive a durable Convex Auth session, preserve deleted and banned account rejection, and add focused smoke or regression coverage after the failing layer is identified.

## Reproduction Assessment

Yes for the current production sign-in availability failure: current main is deployed as prod/v2026.05.10.5, the site and skills API return HTTP 200, but both canonical and direct Convex GitHub sign-in endpoints return HTTP 500. The exact callback-succeeds/no-JWT symptom cannot be replayed while the flow fails one step earlier.
