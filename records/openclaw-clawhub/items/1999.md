## Summary

Keep open. This is a fresh core auth bug report, and current main still relies on the same Convex Auth GitHub flow and custom OAuth code handler; I did not find an implemented fix, duplicate, or evidence that the report is obsolete.

## What This Changes

The issue reports that GitHub OAuth returns to clawhub.ai without persisting Convex Auth JWT or refresh-token localStorage entries.

## Best Possible Solution

Restore the GitHub OAuth token handoff by verifying the production Convex Auth callback emits the frontend `code` handoff expected by AppProviders, then fix the narrow code or deployment configuration mismatch and add regression coverage for the callback path.

## Reproduction Assessment

No high-confidence end-to-end reproduction yet. The source path is clear and the report's persistent verifier/no-token symptom is consistent with the frontend never receiving or consuming the Convex Auth `code`, but a real OAuth run and production backend env inspection are still needed.

## Solution Assessment

Unclear. Checking production `SITE_URL`, `CONVEX_SITE_URL`, OAuth callback URL, and the callback redirect payload is the right first step before deciding whether this is a code fix or deployment configuration fix.
