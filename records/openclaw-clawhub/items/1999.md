## Summary

Keep open. Current production-deployed main still returns HTTP 500 from the GitHub sign-in route on both clawhub.ai and the direct Convex site, so the OAuth/session problem remains unresolved and ClawHub cleanup policy keeps issues open.

## What This Changes

Investigate a GitHub OAuth sign-in loop where Convex Auth appears to complete but no ClawHub browser session tokens are persisted.

## Best Possible Solution

Restore valid production OAuth env/callback/site URL behavior or add import-safe request-time/deploy validation if source changes are needed, then verify web and CLI login end to end while preserving blocked-account semantics.

## Reproduction Assessment

Yes for the current production sign-in failure: both `https://clawhub.ai/api/auth/signin/github` and the direct Convex route return HTTP 500 while `https://clawhub.ai/api/v1/skills` returns 200. The original callback-succeeds/no-JWT symptom still needs a clean browser OAuth trace because the live flow currently fails before callback completion.
