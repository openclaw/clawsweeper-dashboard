## Summary

Keep open: ClawHub cleanup policy keeps issues open, and this still maps to a credible production GitHub OAuth failure rather than an already-shipped source-only fix.

## What This Changes

The issue reports that GitHub OAuth returns to clawhub.ai without persisting Convex Auth JWT or refresh-token state, leaving users logged out after callback.

## Best Possible Solution

Restore production GitHub OAuth so authorized active users receive a durable Convex Auth session while preserving PKCE/state validation and deleted or banned account rejection.

## Reproduction Assessment

Yes. A current production smoke check reproduces an OAuth availability failure: the site and public skills API return HTTP 200, while both canonical and direct Convex GitHub sign-in endpoints return HTTP 500; the exact no-JWT-after-callback variant still needs browser and Convex log correlation.
