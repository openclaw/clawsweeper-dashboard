## Summary

Keep open: this is a concrete production OAuth availability bug report, and ClawHub cleanup policy keeps issues open for maintainer handling. Current main contains the expected Convex Auth callback wiring and tests, but source inspection plus the existing ClawSweeper comment do not isolate whether the live loop is caused by code, production environment, OAuth app settings, Convex secrets, or edge routing.

## What This Changes

The issue reports that GitHub OAuth returns from the Convex Auth callback but never persists `__convexAuthJWT` or `__convexAuthRefreshToken` in `clawhub.ai` localStorage, leaving users logged out.

## Best Possible Solution

Make the production GitHub OAuth callback reliably create a Convex Auth browser session, then add a fresh-login regression or smoke check that catches verifier-only sign-in loops.
