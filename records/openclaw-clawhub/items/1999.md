## Summary

Keep open. This is a current production OAuth/session failure cluster: the documented GitHub sign-in entry point should create a web session, but the live sign-in route still returns HTTP 500 while the rest of the site is reachable, and ClawHub issue cleanup policy keeps issues open.

## What This Changes

The issue reports that GitHub OAuth returns to clawhub.ai without persisting Convex Auth JWT or refresh-token state, leaving the user logged out.

## Best Possible Solution

Restore reliable GitHub OAuth session creation for clawhub.ai and CLI token generation, while preserving Convex Auth state/PKCE validation and blocked-account rejection semantics.

## Reproduction Assessment

Yes for the central production auth failure: both the canonical clawhub.ai GitHub sign-in route and the direct Convex sign-in route returned HTTP/2 500 during review. The exact callback-succeeds/no-JWT localStorage state still needs a browser or HAR confirmation after the route redirects normally again.
