## Summary

Keep open. This is a valid auth bug cluster: current production still returns HTTP 500 for the GitHub sign-in route while the site and public API are reachable, and ClawHub cleanup policy keeps issues open rather than auto-closing them.

## What This Changes

The issue reports that GitHub OAuth completes but does not persist Convex Auth JWT/refresh tokens in clawhub.ai storage, leaving users logged out.

## Best Possible Solution

Restore reliable GitHub OAuth session creation and CLI token generation through an operator-backed fix that verifies production OAuth credentials, Convex auth env, callback behavior, and preserves existing account-state checks.

## Reproduction Assessment

Yes for the central production auth failure: both the canonical clawhub.ai sign-in route and the direct Convex sign-in route returned HTTP/2 500 during review. The exact callback-succeeds/no-JWT localStorage state still needs a browser or HAR confirmation after the sign-in route redirects normally again.
