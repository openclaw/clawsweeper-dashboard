## Summary

Keep open. This is a concrete production OAuth/session failure with auth/security impact labels, related OAuth-loop reports remain open, and current main still requires the browser callback to provide a `code` before Convex Auth JWT and refresh-token state can be persisted.

## What This Changes

The issue reports that GitHub OAuth returns to clawhub.ai without persisting Convex Auth JWT or refresh-token state, leaving users logged out in a retry loop.

## Best Possible Solution

Restore reliable production GitHub OAuth session creation for active accounts, preserve verifier/state validation and blocked-account policy, then add focused callback/session regression coverage plus redacted live smoke proof.

No high-confidence reproduction of the exact callback-succeeds/no-JWT path was completed. Source inspection confirms the frontend cannot establish a session without the callback `code`, and read-only production checks reproduce a related auth endpoint failure before that callback path.
