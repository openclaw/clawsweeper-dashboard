## Summary

Keep open. ClawHub cleanup policy does not auto-close issues, and the production GitHub sign-in route still fails while several related OAuth/session reports remain open.

## What This Changes

The issue reports that GitHub OAuth returns to clawhub.ai without persisting Convex Auth JWT or refresh-token state, leaving users logged out.

## Best Possible Solution

Restore reliable GitHub OAuth session creation for active accounts, preserve blocked-account rejection, and verify the callback with redacted browser storage/cookie traces plus focused auth regression tests.

## Reproduction Assessment

Yes for the central production sign-in failure: both https://clawhub.ai/api/auth/signin/github and the direct Convex sign-in route returned HTTP 500 while non-auth surfaces returned HTTP 200. The exact missing-localStorage callback state still needs browser/HAR verification after the route redirects again.

Not applicable because there is no proposed patch. The narrowest maintainable path is operator-backed OAuth triage across Convex Auth config, callback routing, session persistence, account-state checks, and redacted browser traces.
