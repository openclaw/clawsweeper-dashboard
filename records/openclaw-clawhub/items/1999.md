## Summary

Keep open: ClawHub policy keeps issues open, and the report remains credible. On current production deployed from current main, the site and skills API serve normally but GitHub OAuth sign-in returns HTTP 500 before the exact callback-succeeds/no-JWT stage can be replayed.

## What This Changes

The issue reports that GitHub OAuth completes but ClawHub stays logged out because Convex Auth session tokens are not persisted to clawhub.ai storage.

## Best Possible Solution

Restore production GitHub OAuth so active accounts receive a durable Convex Auth session, preserve deleted/banned/disabled account rejection, and add focused smoke or regression coverage after the failing layer is identified.

## Reproduction Assessment

Yes for the current production sign-in availability failure: current main is tagged prod/v2026.05.10.4, the site and skills API return HTTP 200, but both canonical and direct Convex GitHub sign-in endpoints return HTTP 500. No for the exact callback-succeeds/no-JWT stage, because the flow currently fails one step earlier.
