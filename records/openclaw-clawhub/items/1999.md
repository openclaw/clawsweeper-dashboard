Keep open: this is an open ClawHub issue, and the repository cleanup policy keeps issues open rather than auto-closing them. The report is credible and current production still fails the GitHub OAuth route before the original no-JWT callback path can be replayed.

Reported that GitHub OAuth returns to clawhub.ai without persisting Convex Auth JWT or refresh-token state, leaving users logged out.

## Best Possible Solution

Restore production GitHub OAuth so active accounts receive a browser session, keep blocked/deleted account outcomes explicit, and add focused smoke or regression coverage once the failing layer is identified.
