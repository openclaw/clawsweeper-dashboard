## Summary

Keep open: this is an open ClawHub issue, and current production still has a reproducible GitHub OAuth sign-in failure on the production-tagged main SHA. The exact callback-succeeds/no-JWT symptom cannot currently be replayed because the flow now returns HTTP 500 at the sign-in endpoint.

## What This Changes

The issue reports that GitHub OAuth returns to ClawHub without persisting Convex Auth JWT or refresh tokens, leaving the user logged out.

## Best Possible Solution

Restore production GitHub OAuth for active accounts, keep deleted and banned account rejection explicit, and add focused callback/session smoke or regression coverage once logs prove the failing layer.

## Reproduction Assessment

Yes for the current production login failure: current main is production-tagged and both canonical and direct Convex GitHub sign-in endpoints return HTTP 500 while the site and skills API return HTTP 200. No for the exact callback-succeeds/no-JWT variant, because the flow now fails one step earlier.
