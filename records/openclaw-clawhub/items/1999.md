## Summary

Keep open. The report still matches a real production auth failure on the reviewed main SHA: the GitHub sign-in endpoint returns HTTP 500 through both clawhub.ai and the direct Convex site, while the surrounding site/API routes are reachable.

## What This Changes

The issue reports that GitHub OAuth returns to ClawHub without persisting Convex Auth JWT or refresh tokens, leaving the user logged out and stale PKCE verifier keys behind.

## Best Possible Solution

Restore reliable GitHub OAuth session creation for active accounts, verify production auth configuration and logs, preserve blocked-account handling, and add focused callback/session regression coverage.

## Reproduction Assessment

Yes for the current production sign-in failure: the production-tagged HEAD returns HTTP 500 for both the clawhub.ai rewrite and the direct Convex GitHub sign-in endpoint. The exact callback-succeeds/no-JWT variant still needs a clean-browser OAuth trace because the flow currently fails before callback completion.
