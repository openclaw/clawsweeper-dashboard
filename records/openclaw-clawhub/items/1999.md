## Summary

Keep open: ClawHub cleanup policy keeps issues open, and this remains a credible production OAuth availability bug. Live checks still show the GitHub sign-in route returning HTTP 500 on both the canonical site and direct Convex site while current main delegates that route to Convex Auth, with related no-session reports still open.

## What This Changes

The issue reports GitHub OAuth completing server-side but failing to persist Convex Auth JWT and refresh tokens on clawhub.ai, leaving the user logged out.

## Best Possible Solution

Restore production GitHub OAuth so active accounts receive a browser session, keep blocked-account outcomes explicit, and add focused regression or smoke coverage once maintainers identify the failing layer.

Yes for the current production OAuth availability failure: both canonical and direct Convex GitHub sign-in endpoints return HTTP 500. The exact callback-succeeds/no-JWT variant is not currently replayable because the flow fails one step earlier.
