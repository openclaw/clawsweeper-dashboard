## Summary

Keep open. This is an open ClawHub issue, and current production on the reviewed main SHA still fails the GitHub sign-in start route with HTTP 500 on both clawhub.ai and the direct Convex site, so the OAuth/session problem is not resolved.

## What This Changes

The issue reports that GitHub OAuth returns to clawhub.ai without persisting Convex Auth JWT or refresh-token state, leaving users logged out.

## Best Possible Solution

Restore reliable GitHub OAuth session creation for active accounts, preserve explicit deleted/banned-account rejection, and add focused callback/session smoke or regression coverage after the failing layer is identified.

## Reproduction Assessment

Yes for the current production sign-in failure: on May 10, 2026 the production-tagged reviewed SHA returned HTTP/2 500 from both the clawhub.ai and direct Convex GitHub sign-in endpoints while a public API endpoint returned HTTP/2 200. The exact callback-succeeds/no-JWT variant still needs a clean-browser OAuth trace because the flow currently fails before callback completion.

Not applicable because there is no proposed patch. The safest path is maintainer/operator triage of production auth logs, env, OAuth callback settings, cookie/storage behavior, and account state before choosing a narrow config, dependency, or source fix.
