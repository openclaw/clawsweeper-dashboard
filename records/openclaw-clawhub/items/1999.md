Keep open. The central GitHub OAuth/session problem is still reproducible on the current production-tagged main SHA: both the canonical clawhub.ai sign-in route and the direct Convex sign-in route return HTTP 500, and ClawHub cleanup policy keeps issues open for maintainer follow-up.

The issue asks for GitHub OAuth to persist the Convex Auth JWT/refresh session on clawhub.ai after callback instead of returning the user to a logged-out state.
