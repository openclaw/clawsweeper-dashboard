## Summary

Keep open: this is still a live production GitHub OAuth/session outage. Current deployed main has the expected Convex Auth source plumbing, but both canonical and direct production sign-in endpoints return HTTP 500, and this ClawHub cleanup policy keeps issues open rather than auto-closing them.

## What This Changes

Report that GitHub OAuth returns to clawhub.ai without persisting Convex Auth JWT or refresh-token state, leaving users logged out.

## Best Possible Solution

Restore production GitHub OAuth end to end for active accounts, add import-safe diagnostics or deploy validation for auth configuration, and cover the flow with focused regression plus production smoke checks.

## Reproduction Assessment

Yes for the current production auth failure: both https://clawhub.ai/api/auth/signin/github and https://wry-manatee-359.convex.site/api/auth/signin/github returned HTTP 500 on May 9, 2026 while the public API returned 200. The exact post-authorize callback/no-JWT loop was not replayable because sign-in currently fails before redirecting to GitHub.

## Solution Assessment

Unclear as a source-only fix. Current main has the expected Convex Auth route, provider, callback-handler, and deploy docs; the narrow next step is maintainer/operator triage of production OAuth env, logs, callback settings, and an import-safe diagnostic path.
