## Summary

Keep open. This is an open ClawHub issue, and current production still fails the GitHub sign-in start route with HTTP 500 on both the clawhub.ai rewrite and the direct Convex site, so the OAuth/session-loss report is not resolved.

## What This Changes

GitHub OAuth sign-in should persist the Convex Auth session tokens on clawhub.ai after the callback instead of returning the user to a logged-out state.

## Best Possible Solution

Keep the OAuth/session cluster open until maintainers verify production GitHub sign-in end to end, inspect Convex/Vercel/GitHub OAuth settings and logs, fix the specific code or configuration fault, and add focused regression coverage.
