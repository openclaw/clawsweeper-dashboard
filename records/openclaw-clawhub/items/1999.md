## Summary

Keep open: this is an open ClawHub issue, and the repository cleanup policy keeps issues open rather than auto-closing them. The report is still credible because current production, deployed from current main, returns HTTP 500 for the GitHub OAuth sign-in route while the site and other API routes serve normally.

## What This Changes

The issue reports GitHub OAuth completing but ClawHub failing to persist Convex Auth JWT and refresh tokens, leaving the user logged out.

## Best Possible Solution

Restore production GitHub OAuth so active accounts receive a durable Convex Auth session, preserve deleted and banned account rejection, and add focused smoke or regression coverage after the failing layer is identified.
