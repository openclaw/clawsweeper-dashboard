## Summary

Keep open. This is an open ClawHub issue and current production-tagged main still has a reproducible GitHub auth failure: both the canonical and direct Convex GitHub sign-in endpoints return HTTP/2 500 while the site and public API are reachable.

## What This Changes

The issue reports that GitHub OAuth completes but clawhub.ai never persists a Convex Auth JWT or refresh token, leaving the user logged out.

## Best Possible Solution

Restore production GitHub OAuth session creation for active accounts, preserve PKCE/session and blocked-account validation, then verify with redacted browser cookie/storage traces plus focused auth smoke coverage.

## Reproduction Assessment

Yes for the central production sign-in failure: current production-tagged main returns HTTP/2 500 from both GitHub sign-in endpoints while the site and public API return 200. The exact callback-succeeds/no-JWT storage variant still needs a fresh browser or HAR trace after sign-in redirects work again.
