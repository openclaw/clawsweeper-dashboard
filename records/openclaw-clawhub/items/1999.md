## Summary

Keep open: this is a current ClawHub production auth bug, and production on the reviewed main SHA still fails GitHub OAuth before the callback can persist a session. The related OAuth/session reports remain open, so there is no safe cleanup close.

## What This Changes

GitHub OAuth sign-in on clawhub.ai should persist the Convex Auth session after callback instead of returning the user to a logged-out state.

## Best Possible Solution

Restore production GitHub OAuth end-to-end for active accounts, preserve deleted/banned-account rejection, and add focused callback/session regression or smoke coverage after the failing layer is identified.

## Reproduction Assessment

Yes for the current production sign-in availability failure: the reviewed main SHA is production-tagged, and both the canonical and direct Convex GitHub sign-in endpoints return HTTP 500. No for the exact callback-succeeds/no-JWT variant, because the flow currently fails before the callback step.

Not applicable because there is no proposed patch. The narrowest maintainable path is maintainer/operator auth triage using production logs, OAuth app settings, cookie/storage traces, and affected account records before choosing a config, dependency, or source fix.
