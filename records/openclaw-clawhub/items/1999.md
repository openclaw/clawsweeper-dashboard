Keep open: this is a credible production OAuth availability bug, and current production sign-in is still failing before the reported callback/JWT state can even be fully retested. ClawHub cleanup policy also keeps open issues in maintainer hands rather than auto-closing them as implemented.

## What This Changes

The issue reports that GitHub OAuth authorization succeeds but the ClawHub frontend remains logged out because Convex Auth session tokens are not persisted in browser storage.

## Best Possible Solution
