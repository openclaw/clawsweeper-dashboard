## Summary

Keep open: this is an open ClawHub issue, and current production still has a reproducible GitHub OAuth failure. The exact reported no-JWT-after-callback path cannot be replayed right now because the signin endpoint fails one step earlier with HTTP 500.

## What This Changes

The issue reports that GitHub OAuth returns to clawhub.ai without persisting `__convexAuthJWT` or `__convexAuthRefreshToken`, leaving users logged out.

## Best Possible Solution

Restore production GitHub OAuth so active accounts receive a durable browser session, preserve explicit blocked/deleted account outcomes, and add focused smoke or regression coverage after the failing layer is identified.

## Reproduction Assessment

Yes for the current production OAuth availability failure: both canonical and direct Convex GitHub signin endpoints return HTTP 500 while the site root serves normally. No for the exact callback-succeeds/no-JWT variant, because the flow currently fails one step earlier.

## Solution Assessment

Unclear because there is no proposed patch and the failure likely spans production Convex Auth configuration, OAuth callback cookies/PKCE, and source route delegation. The narrowest maintainable path is maintainer/operator triage followed by a verified config, dependency, or app-code fix.
