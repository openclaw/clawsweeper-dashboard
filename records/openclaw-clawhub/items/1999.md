## Summary

Keep open: current production still fails the GitHub OAuth entrypoint, the related OAuth/session reports remain open, and ClawHub review policy keeps issues open unless a maintainer resolves them.

## What This Changes

The issue reports that GitHub OAuth returns to ClawHub without persisting Convex Auth session tokens, leaving the user logged out.

## Best Possible Solution

Restore GitHub OAuth end-to-end for legitimate users, keep deleted/banned/deactivated account rejection intact, and add focused callback/session smoke coverage after identifying whether the failing layer is production config, Convex Auth routing, or frontend code handling.

## Reproduction Assessment

Yes for the current production sign-in failure: the reviewed main SHA is production-tagged, and both the canonical and direct Convex GitHub sign-in endpoints return HTTP 500. The exact callback-succeeds/no-JWT variant is not independently reproduced because the flow now fails before callback completion.
