## Summary

Keep open. Current main makes the reported ranking behavior plausible rather than resolved: slug-token lexical boosts can outweigh the small download prior, and the available docs only describe the broad ranking ingredients and rename alias behavior, not the publisher-facing guidance requested.

## What This Changes

The issue asks for clarification or documentation about why a high-download map-related skill ranks low, whether standalone slug tokens affect search, and what slug-renaming side effects publishers should expect.

## Best Possible Solution

A maintainer should answer the publisher directly, decide whether the current map versus amap ordering is intended, and then either add discoverability guidance covering slug-token effects, downloads, moderation filters, and rename aliases, or adjust ranking weights and tests if the current ordering is not desired.
