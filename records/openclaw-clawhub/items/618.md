## What This Changes

The issue reports that `clawhub sync` shows semver-bumped update targets as if they were actual registry latest versions.

## Best Possible Solution

Update the sync planning output and interactive upload-selection label so unmatched local content is described relative to the registry latest, while any bumped version is clearly labeled as a proposed upload version, with regression coverage for `latestVersion` present and `match: null`.

## Reproduction Assessment

Yes, by source inspection: a resolve response with `latestVersion: "1.0.0"` and `match: null` becomes `status: "update"`, then the formatter renders `UPDATE 1.0.0 → 1.0.1`. I did not run a live registry sync for the named skills because this review was read-only.

## Solution Assessment

Yes. The narrow maintainable fix is to clarify the sync report and picker labels while leaving the resolve API, candidate classification, semver bump calculation, and publish workflow intact.
