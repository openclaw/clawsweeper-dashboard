## What This Changes

The issue requests that `clawhub sync` report actual registry latest versions instead of semver-bumped proposed upload versions.

## Best Possible Solution

Update the sync planning output and interactive upload-selection label so `latestVersion` plus `match: null` is described as local changes relative to the registry latest, with any bumped version clearly labeled as a proposed upload version, and add regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a resolve response with `latestVersion: "1.0.0"` and `match: null` becomes an update candidate, then sync formats it as `UPDATE 1.0.0 → 1.0.1`; I did not run the CLI because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable fix is to separate registry latest from proposed publish target in the sync report and picker label while preserving the existing semver bump used for publishing.
