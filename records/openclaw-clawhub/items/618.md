## What This Changes

`clawhub sync` should report the actual registry latest version instead of displaying a semver-bumped proposed publish version as an available update.

## Best Possible Solution

Review and land or close https://github.com/openclaw/clawhub/pull/2235; if that PR is not used, make the same narrow CLI output and regression-test fix on main.

## Reproduction Assessment

Yes. Source inspection shows a resolve response with `latestVersion: "1.0.0"` and `match: null` is classified as an update, then formatted as `UPDATE 1.0.0 → 1.0.1` before publish metadata is resolved.

## Solution Assessment

Yes. The narrow maintainable fix is to keep registry latest and proposed publish version distinct in the CLI sync report and picker, with regression coverage for the `latestVersion` plus `match: null` case.
