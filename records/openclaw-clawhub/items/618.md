## What This Changes

`clawhub sync` should report actual registry latest versions instead of showing semver-bumped proposed publish versions as available updates.

## Best Possible Solution

Review and land or close https://github.com/openclaw/clawhub/pull/2235; after it merges, resolve this issue as implemented by that merged PR.

## Reproduction Assessment

Yes, source-reproducible: a resolve response with `latestVersion: "1.0.0"` and `match: null` returns an update candidate, then current formatting renders `UPDATE 1.0.0 → 1.0.1` before publish metadata is resolved.

## Solution Assessment

Yes. The narrow maintainable fix is to keep registry latest and proposed publish version distinct in the CLI sync report and picker, with regression coverage for the `latestVersion` plus `match: null` case.
