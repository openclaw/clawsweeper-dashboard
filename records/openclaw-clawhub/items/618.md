## What This Changes

## Best Possible Solution

Update the sync planning output and interactive upload-selection label to distinguish registry latest from the proposed upload version, then add regression coverage for `latestVersion` present with `match: null`.

## Solution Assessment

Yes. The narrow maintainable fix is to separate registry latest from the proposed publish target in the sync report and picker label while preserving the existing semver bump used for publishing.
