## What This Changes

This issue requests that `clawhub sync` report the actual registry latest version instead of showing nonexistent semver-bumped versions as available updates.

## Best Possible Solution

Update the sync planning output and interactive picker label so registry latest and proposed upload version are distinct, add regression coverage for `latestVersion` present with `match: null`, and keep the broader fingerprint false-positive work in https://github.com/openclaw/clawhub/issues/169.
