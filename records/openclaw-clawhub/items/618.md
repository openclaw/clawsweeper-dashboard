## Summary

Keep open. Current main still has the reported CLI presentation bug: unmatched local skill content is classified as an update and the sync report displays a semver-bumped publish target as though it were an available registry version; the related dashboard drift warning is a separate open frontend issue.

## What This Changes

`clawhub sync` should report the actual latest registry version for installed skills instead of showing a bumped publish target as an available update.

## Best Possible Solution

Update the sync planning output and interactive upload-selection label so local changes are described relative to the registry latest, with the bumped version clearly labeled only as the proposed upload version, and add regression coverage for `latestVersion` present with `match: null`.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a resolve response with `latestVersion: 1.0.0` and `match: null` becomes an update candidate, then sync formats it as `UPDATE 1.0.0 -> 1.0.1`.

## Solution Assessment

Yes. The narrow maintainable fix is to change the sync report and picker label semantics while preserving the existing semver bump used later for publishing.
