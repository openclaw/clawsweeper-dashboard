## Summary

Keep open: current main still has the reported CLI sync presentation bug, and ClawHub cleanup policy keeps issues open rather than auto-closing them. The related dashboard drift report is a separate frontend/deploy warning path and no PR was found for this exact sync version-display issue.

## What This Changes

The issue reports that `clawhub sync` displays semver-bumped upload targets as if they were available registry versions, while `clawhub inspect` shows the actual latest version.

## Best Possible Solution

Update the sync planning output and interactive upload-selection label so unmatched local content is described relative to the registry latest, while any bumped version is clearly labeled as a proposed upload version.

## Reproduction Assessment

Yes, by source inspection: a resolve response with `latestVersion: "1.0.0"` and `match: null` becomes `status: "update"`, then the formatter renders `UPDATE 1.0.0 → 1.0.1`. I did not run a live registry sync for the named skills because this review was read-only.

## Solution Assessment

Yes. The narrow maintainable fix is to clarify the sync report and picker labels while preserving the resolve API, candidate classification, semver bump calculation, and publish workflow.
