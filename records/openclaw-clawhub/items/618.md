## Summary

Keep open. Current main still has the reported CLI presentation bug: when local skill content does not match any registry version but a registry latest exists, `clawhub sync` formats the proposed semver bump as an `UPDATE` target, which reads like a non-existent registry version. Under the repository policy, ClawHub issues stay open unless maintainers handle them, and this one remains a narrow valid fix candidate.

## What This Changes

The issue asks for `clawhub sync` to display the actual registry latest version instead of showing non-existent higher versions for installed skills.

## Best Possible Solution

Update the CLI sync report and interactive upload-selection label so unmatched local content is described as local changes relative to the registry latest, with any bumped version clearly labeled as the proposed upload version. Add regression coverage for `latestVersion` present with `match: null`, and keep the dashboard drift warning tracked separately in #1180.
