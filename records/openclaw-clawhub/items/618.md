## Summary

Keep open. Current main still has the reported CLI presentation bug: when a registry latest exists but the local fingerprint matches no published version, `clawhub sync` presents the semver-bumped publish target as an `UPDATE` version, while `clawhub inspect` reports the actual registry latest.

## What This Changes

The issue reports that `clawhub sync` displays semver-bumped versions as available registry updates while `clawhub inspect` shows the actual latest registry versions.

## Best Possible Solution

Clarify the sync planning output so local unmatched content is shown as local changes relative to the registry latest, with any bumped version labeled as the proposed upload version.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the existing mocked `match: null` plus `latestVersion: 1.0.0` case becomes an update candidate, and the formatter would display `1.0.0 -> 1.0.1`; I did not run tests because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable fix is to adjust the sync report and interactive selection label wording while leaving registry resolution and publish-version calculation intact.
