## Summary

Keep open. Current main still has the source-level CLI presentation bug: local content with no matching registry version is classified as an update, and the sync report displays the semver-bumped publish target as if it were an available registry version. The linked fix at https://github.com/openclaw/clawhub/pull/2235 is still open with closing syntax, so this issue should remain open until that PR lands or is replaced.

## What This Changes

The issue reports that `clawhub sync` shows higher, nonexistent version numbers for already-current installed skills.

## Best Possible Solution

Review and land https://github.com/openclaw/clawhub/pull/2235 if maintainers agree with the CLI wording, or apply the same narrow sync-output fix with regression coverage.

## Reproduction Assessment

Yes. Source inspection shows current main maps `latestVersion` plus `match: null` to `update`, then prints a semver-bumped publish target as `UPDATE`; I did not execute the CLI because this cleanup review is read-only.

## Solution Assessment

Yes. The linked PR's direction is narrow: clarify sync report and selection labels while preserving publish mechanics, with regression coverage for the `latestVersion` present and `match: null` case.
