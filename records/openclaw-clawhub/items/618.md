Keep open: current main still has the source-reproducible CLI sync reporting bug, and ClawHub cleanup policy keeps issues open for maintainer follow-up rather than auto-closing them.

## What This Changes

The issue asks for `clawhub sync` to report actual registry latest versions instead of displaying semver-bumped publish targets as available updates.

## Best Possible Solution

Update the sync report and interactive picker to distinguish registry latest from the proposed upload version, then add regression coverage for `latestVersion` present with `match: null`.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a resolve response with `latestVersion: "1.0.0"` and `match: null` becomes an update candidate, then sync formats it as `UPDATE 1.0.0 → 1.0.1`; the CLI was not executed because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable fix is to separate registry latest from the proposed publish target in the sync report and picker label while preserving the existing semver bump used for publishing.
