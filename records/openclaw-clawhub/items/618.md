## Summary

Keep open. Current main still has the reported CLI presentation bug: unmatched local skill content is classified as an update, and the sync plan renders the semver-bumped publish target as though it were an available registry version.

## What This Changes

The issue reports that `clawhub sync` shows semver-bumped versions as available registry updates even though `clawhub inspect` reports lower actual latest versions.

## Best Possible Solution

Update the sync planning output and interactive upload-selection label so unmatched local content is described as local changes relative to the registry latest, with any bumped version clearly labeled as the proposed upload version.

## Reproduction Assessment

Yes. Source inspection on current main shows the exact path: a registry latest with `match: null` becomes an update candidate and is formatted as `UPDATE <latest> → <semver.inc(latest)>` before publish metadata is resolved.

## Solution Assessment

Yes. The narrowest maintainable fix is to change the CLI report and prompt wording, then add regression assertions for the `latestVersion` present plus `match: null` case without changing registry resolution or publish behavior.
