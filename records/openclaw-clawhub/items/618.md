## Summary

Keep open. Current main still has the reported CLI presentation bug: unmatched local skill content is treated as an update candidate and the sync report displays the semver-bumped publish target as though it were an available registry version.

## What This Changes

Fix `clawhub sync` so it reports actual registry latest versions instead of misleading bumped target versions for installed skills with unmatched local content.

## Best Possible Solution

Update the sync report and upload-selection label to describe unmatched local content as local changes relative to the registry latest, with any bumped version clearly labeled as the proposed upload version.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: mock registry resolution with `latestVersion` present and `match: null`, then observe `formatActionableStatus` render `UPDATE <latest> → <semver.inc(latest)>`.

## Solution Assessment

Yes. The narrowest maintainable fix is to change the CLI reporting and prompt wording plus add regression assertions for this state, without changing publish metadata resolution.

## Repair Work Prompt

Fix the CLI display bug where `clawhub sync` reports a semver-bumped publish target as if it were the registry latest version when local skill content does not match any registry version but `latestVersion` exists. Update `packages/clawhub/src/cli/commands/syncHelpers.ts` so the `To sync` line and interactive upload-selection label describe this state as local changes relative to the actual registry latest, and clearly label any bumped version as the proposed upload version. Preserve the existing publish flow and `resolvePublishMeta` semver bump behavior; do not change upload target calculation or registry APIs. Add regression assertions in `packages/clawhub/src/cli/commands/sync.test.ts` and/or `syncHelpers.test.ts` for `latestVersion` present with `match: null`, including noninteractive report output and interactive prompt labels. Keep #1180 out of scope except to avoid conflating dashboard drift copy with this CLI sync output.
