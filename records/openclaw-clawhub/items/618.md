## Summary

Keep open. Current main still has the CLI presentation bug: sync treats unmatched local content as actionable and displays the semver-bumped publish target as an `UPDATE` version, while `inspect` reports the actual registry latest directly.

## What This Changes

The issue asks for `clawhub sync` to report the actual registry latest version instead of showing a higher non-existent version for installed skills.

## Best Possible Solution

Make sync output distinguish the registry latest/current local content state from the proposed upload version, and cover the `latestVersion` present plus `match: null` case with regression assertions.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: a resolve response with `latestVersion: 1.0.0` and `match: null` becomes an update candidate, then sync formats it as `UPDATE 1.0.0 -> 1.0.1`. I did not execute the CLI because the review was explicitly read-only.

## Solution Assessment

Yes. The narrow maintainable fix is to adjust the sync report and interactive selection label while preserving the existing semver bump used for publishing.
