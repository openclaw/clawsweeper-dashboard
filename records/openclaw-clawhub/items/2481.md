## Summary

Current main has replaced mutable-handle-derived Official status with explicit admin-managed `officialPublishers` records, so this branch’s central authorization approach is obsolete. The PR remains open because it is member-authored and needs explicit maintainer disposition, but it should not merge as submitted: it conflicts with main, adds a parallel trust model, includes unrelated handle restrictions, and breaks Vercel’s Bun setup.

## What This Changes

The branch adds reservation-backed ownership checks for allowlisted Official handles, blocks unreserved org creation, expands reserved public-route handles, adds mocked regression tests and policy text, and introduces an npm lockfile.

## Best Possible Solution

Keep explicit admin-managed `officialPublishers` records as the sole trust boundary, then either close this obsolete branch or replace it with a narrow current-main follow-up only if a sensitive-handle creation guard remains necessary.

## Reproduction Assessment

No on current main: claiming an allowlisted organization handle does not create the explicit `officialPublishers` record now required for Official status. The old handle-squatting privilege path is therefore no longer reproducible through the inspected authorization function.

## Solution Assessment

No; current main’s explicit admin-managed record is a narrower single source of truth than reservation-, handle-, and membership-derived status. Any remaining org-creation restriction should be reviewed and implemented separately against the current model.
