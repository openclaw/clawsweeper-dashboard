## What This Changes

The issue requests correcting the VirusTotal security tag when a skill shows Suspicious even though the linked VirusTotal report says no security vendors flagged the file.

## Best Possible Solution

Expose VT scanner source and AV engine counts through the relevant frontend/API shapes, label VirusTotal Code Insight separately from AV engine detections, and render zero-engine Code Insight suspicious results without implying that VirusTotal vendors flagged the file.

## Repair Work Prompt

Fix the VirusTotal presentation mismatch reported for the skill security UI. Backend skillVersions.vtAnalysis already stores scanner and engineStats; update the frontend/API shapes so scanner and AV engine counts are available where security status is rendered. In the skill page scan panel, detail summary, and scanner detail page, distinguish VirusTotal Code Insight from AV engine/vendor detections. Do not rely on vtAnalysis.source === "code_insight" because source may be a human string from VirusTotal; prefer scanner or normalized metadata. For Code Insight suspicious results with zero malicious/suspicious AV engineStats and clean corroborating scans, do not imply that VirusTotal vendors flagged the file. Preserve existing behavior for true malicious/suspicious AV engine detections, nonzero suspicious/malicious engineStats, and corroborating LLM/static findings. Keep API/schema package response shapes aligned if new fields are exposed. Add focused tests for the zero-engine Code Insight case plus a true-engine-detection case. Do not change unrelated moderation policy, rescan behavior, or release workflow.
