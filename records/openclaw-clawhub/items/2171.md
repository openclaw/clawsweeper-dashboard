## Summary

Keep open: ClawHub cleanup policy keeps issues open, and current main still supports the report. The browser publish form can block an org publisher/admin member at slug-availability validation before the backend publish path gets to apply the selected org owner authorization.

## What This Changes

The issue reports that publisher/admin members of an organization cannot publish a new version of an org-owned transferred skill from the browser publish flow.

## Best Possible Solution

Make skill publish preflight/update validation publisher-aware for the selected current org owner while keeping server-side publish authorization authoritative.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: for an org-owned skill where the actor is a publisher/admin member but not skill.ownerUserId, the publish page gets a taken slug result and disables submit before the backend publish action can authorize org membership.

## Solution Assessment

Yes. The narrow maintainable fix is to align slug availability or update-mode validation with the selected/current publisher owner, without weakening the existing backend publish and migration checks.
