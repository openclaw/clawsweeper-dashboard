## Summary

Keep open: ClawHub issue policy keeps issues open, and current main still supports the report. The browser publish form can block an authorized org publisher/admin at slug-availability preflight even though the backend publish path resolves the selected org owner with publisher-role authorization.

## What This Changes

The issue reports that an org user with publisher/admin membership cannot publish a new version of a skill after it has been transferred into the org from another user.

## Best Possible Solution

Make the publish preflight publisher-aware for the selected/current org owner while leaving publishVersion and insertVersion as the final server-side authorization checks.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an org-owned skill where the actor is a publisher/admin member but not skill.ownerUserId receives a taken slug result in the browser before the backend publish action can authorize the selected org owner.

## Solution Assessment

Yes. The narrow maintainable fix is to update slug availability or publish-page update validation for authorized same-owner org republishes, without weakening owner-migration admin checks or public slug collision handling.
