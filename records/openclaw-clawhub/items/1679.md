## Summary

Keep open: current production still fails part of the reported default-skill install set, and current main intentionally routes slug-only duplicate skills to an ambiguity response or 404 rather than guaranteeing one unscoped default target.

## What This Changes

The issue asks ClawHub/OpenClaw to make default skill installs such as `coding-agent`, `himayala`/`himalaya`, `summarize`, and `wacli` resolve from the CLI instead of failing with `Skill not found`.

## Best Possible Solution

Define the intended default-skill install contract, then either publish or alias the official default targets in ClawHub or update OpenClaw/docs to use owner-qualified or local bundled skill resolution.

## Solution Assessment

Unclear. The narrowest durable fix depends on maintainer policy: ClawHub can add canonical public rows or aliases, or OpenClaw can stop treating these bundled/default names as ClawHub-only unscoped lookups.
