## Summary

Current ClawHub correctly returns 404 for unpublished names, while current OpenClaw already ships `coding-agent`, `himalaya`, `summarize`, and `wacli` as bundled skills; this remains open because the user-facing command or setup guidance should distinguish bundled skills from remotely installable ClawHub packages.

## What This Changes

Clarify or correct the skill-install workflow so bundled OpenClaw skills are not requested as nonexistent ClawHub packages.

## Best Possible Solution

OpenClaw should detect bundled skills and explain that they already ship locally, guiding users to satisfy each skill's binary/config requirements; ClawHub should continue resolving only published packages and genuine rename/merge aliases.

## Reproduction Assessment

Yes. Production HTTP requests reproduce the reported 404s, and current OpenClaw source confirms that the intended names are bundled local skills; `himayala` is additionally misspelled.

## Solution Assessment

No, adding speculative ClawHub records is not the best solution. The maintainable fix is to distinguish bundled local skills from community packages at the OpenClaw command or setup surface.
