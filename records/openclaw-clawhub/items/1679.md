## Summary

Current ClawHub correctly returns 404 for unpublished registry slugs, but the reported workflow still gives users misleading results when they try to install OpenClaw-bundled skills. The remaining work belongs primarily in OpenClaw’s command/setup UX and requires a maintainer decision on how bundled names and likely misspellings should be handled.

## What This Changes

The issue reports that OpenClaw’s skill-install command sends expected bundled skill names to ClawHub and receives registry 404 errors.

## Best Possible Solution

Have OpenClaw recognize bundled skill names and likely misspellings before registry lookup, returning clear already-bundled, enablement, or spelling guidance while ClawHub preserves accurate 404 responses for unpublished packages.

## Reproduction Assessment

Yes. Production endpoint checks on June 11, 2026 reproduce the reported 404 path for three unpublished names and the misspelled `himayala`, while source inspection establishes the missing-registry-record behavior.

## Solution Assessment

No, publishing aliases or duplicate packages in ClawHub is not the narrowest maintainable solution. OpenClaw should distinguish bundled skills, remote registry packages, and likely misspellings before issuing the request.
