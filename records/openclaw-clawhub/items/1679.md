Keep open. This is an open ClawHub issue, and the current source plus live API checks still support the reported 404s for `coding-agent`, `himayala`, `summarize`, and `wacli`; the likely intended `himalaya` slug now resolves.

The issue reports that installing several claimed default OpenClaw skills by slug through ClawHub returns `404 Skill not found`.

## Best Possible Solution

Maintainers should decide the canonical default-skill slugs, then publish or alias intended defaults in ClawHub and update any setup guidance that points users at invalid names.
