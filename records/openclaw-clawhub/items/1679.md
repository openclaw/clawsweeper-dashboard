## Summary

The 404s are real, but current evidence does not show a ClawHub API defect: all four names are bundled OpenClaw skills, `coding-agent` is explicitly enabled through bundled-skill configuration, and `himayala` is a misspelling of the published `himalaya` slug. This should remain open for maintainer routing because ClawHub policy only permits cleanup closes when current main definitely implements the requested change.

## What This Changes

The issue reports ClawHub 404 responses when attempting to install several OpenClaw-bundled skills through the ClawHub-backed `skills install` command.

## Best Possible Solution

Have OpenClaw recognize bundled skills and enable them or install their declared binary dependencies locally, while sending only canonical external skill slugs to ClawHub and offering a spelling suggestion for `himayala`.

## Reproduction Assessment

Yes. Direct production requests reproduce every exact reported 404, and current OpenClaw source establishes that the names refer to bundled skills; the corrected `himalaya` slug succeeds.

## Solution Assessment

No. Publishing duplicate ClawHub entries or teaching the API to treat bundled names specially would blur ownership; correcting the OpenClaw setup/CLI path is narrower and more maintainable.
