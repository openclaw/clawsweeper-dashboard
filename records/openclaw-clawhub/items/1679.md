## Summary

Keep open. The live ClawHub API still returns 404 for the reported default-skill slugs, and current main correctly returns `Skill not found` when no skill or alias exists; the unresolved part is whether these OpenClaw bundled/default names should become official registry listings, aliases, or clearer setup documentation.

## What This Changes

The issue asks for help installing OpenClaw default-skill names such as `coding-agent`, `himayala`, `summarize`, and `wacli` after ClawHub returns 404.

## Best Possible Solution

Define an authoritative default-skill contract: either publish or alias official ClawHub entries for these names, or update setup/install guidance so bundled-only skills are not resolved as missing registry slugs.

## Reproduction Assessment

Yes for the observable symptom: live `curl` checks against the canonical ClawHub API return `Skill not found` with HTTP 404 for the reported slugs. The source-only checkout cannot prove the intended registry content behind those names.

## Solution Assessment
