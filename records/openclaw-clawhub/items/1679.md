This issue should stay open because current main still has the same public registry lookup boundary: absent public skill records resolve to `404 Skill not found`, and the reported slugs still return 404 from production. ## What This Changes

The issue requests help for OpenClaw bundled/default skill names failing through ClawHub skill installation with `404 Skill not found`.

## Best Possible Solution

Define and document a single install contract: bundled OpenClaw skills should resolve locally or produce explicit CLI guidance, while ClawHub remains the registry for published skills unless maintainers intentionally publish owner-backed records.
