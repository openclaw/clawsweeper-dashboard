## Summary

The report is reproducible, but current evidence shows a product-boundary mismatch rather than missing ClawHub records alone: `coding-agent`, `wacli`, and `himalaya` are bundled by OpenClaw, `himayala` is misspelled, and ClawHub exact-slug lookup correctly returns 404 for unpublished names. The remaining work is to make the bundled-versus-registry distinction actionable in the CLI or API instead of creating ambiguous generic registry packages.

## What This Changes

Clarify or correct installation behavior when OpenClaw users try to fetch bundled or misspelled skill names through ClawHub’s exact-slug API.

## Best Possible Solution

Keep bundled skills supplied by OpenClaw, correct the `himayala` spelling, and make the CLI or ClawHub 404 explain when a requested name is bundled, misspelled, renamed, or genuinely absent without assigning ambiguous registry ownership.

## Reproduction Assessment

Yes. Direct production requests reproduce the exact 404 for the reported unpublished or misspelled slugs, while current OpenClaw source confirms that several are bundled skills rather than ClawHub packages.

## Solution Assessment

No, publishing generic ClawHub records is not the narrowest safe solution. Maintainers should preserve the bundled-versus-registry boundary and provide targeted installation or not-found guidance across the owning CLI/API surface.
