## What This Changes

Issue reports that installing expected default OpenClaw skills through ClawHub returns `Skill not found` for slugs such as `coding-agent`, `himalaya`, `summarize`, and `wacli`.

## Best Possible Solution

Define the intended default-skill install contract, then either publish or alias the missing official/default skill targets in ClawHub or update OpenClaw/docs to use owner-qualified or local bundled skill resolution.

## Reproduction Assessment

## Solution Assessment

Unclear. The narrowest durable fix depends on maintainer policy: ClawHub can add canonical public rows or aliases, or OpenClaw can stop treating these bundled/default names as ClawHub-only unscoped lookups.
