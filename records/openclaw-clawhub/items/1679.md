Keep open. The reported slugs still fail in production, current main routes CLI installs through the exact skill-detail API, and ClawHub cleanup policy keeps unresolved issues open for maintainer follow-up rather than auto-closing them.

## What This Changes

The issue reports that installing expected/default skill slugs such as `coding-agent`, `summarize`, `wacli`, and `himayala` fails with a ClawHub skill-detail 404.

## Best Possible Solution

Align the default/setup skill names with ClawHub registry state by publishing or aliasing intended official skills and correcting stale or misspelled guidance through the existing registry mechanisms.

## Reproduction Assessment

Yes for the observable API symptom: production returns 404 for the reported missing slugs, and current main installs resolve through `/api/v1/skills/<slug>`. I did not run the full install command because a successful path would write skill files and lock metadata.

## Solution Assessment

Unclear as a source-code fix. Current main already supports direct slug lookup and historical aliases, so the narrowest maintainable solution depends on a maintainer decision to publish aliases, publish official skills, or correct setup guidance.
