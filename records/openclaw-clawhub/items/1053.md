## Summary

Keep open. ClawHub cleanup policy does not auto-close open issues, and current main still leaves the reported scanner-visible mismatch possible: publish normalizes SKILL.md frontmatter into parsed registry metadata but does not ingest a separate root metadata.json env object, while the LLM registry summary still renders required env vars from requires.env rather than one normalized required-env set.

## What This Changes

The issue asks ClawHub to stop false suspicious scanner flags by making registry/scanner metadata reflect environment variable declarations from metadata.json, SKILL.md frontmatter, or scanner-side file inspection.

## Best Possible Solution

Define one canonical scanner-visible env metadata contract, normalize required environment variable names consistently during publish/backfill across supported sources, and have the security prompt, static moderation, UI/install surfaces, and rescan/manual-review tooling consume the same normalized field.
