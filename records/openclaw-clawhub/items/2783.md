## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Avoid double-consuming route limits:** `convex/http.ts:81-83`
  - body: This wrapper now runs before every route handler, but many of those handlers still call `applyRateLimit` themselves. For example, `/api/v1/packages` publish now hits the outer anonymous/default write bucket before `publishPackageV1Handler` can apply its package publish-token bucket, and successful calls then consume a second token inside the handler. That can produce early 429s for valid trusted-publisher flows and halves effective quotas on legacy routes; keep each route on one rate-limit layer or opt already-limited handlers out of the wrapper.
  - confidence: 0.9
