## Summary

## What This Changes

The PR wraps Convex HTTP route registration with a central rate-limit policy, preserves handler-level single-consumption behavior, hardens IP/token/shard handling, and adds API/docs/spec coverage.

## Best Possible Solution

Land the central single-pass rate-limit wrapper only after maintainer security and rollout approval, preserving package publish-token buckets and the documented edge-header requirements.
