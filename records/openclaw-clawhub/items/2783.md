## Summary

Keep open: this member-authored draft still has a P1 auth-header defect after the latest fix. The route wrapper now covers auth routes and rebuilds successful responses through a plain header record, which can drop distinct `Set-Cookie` values.

## What This Changes

The PR wraps Convex HTTP route registration with a central rate-limit policy, changes anonymous IP/fallback and sharded counter behavior, and adds HTTP rate-limit docs, specs, and tests.

## Best Possible Solution

Preserve multi-value headers in the wrapper and add regression coverage, then land only after maintainer security/ops acceptance of the central route policy, IP-trust posture, and fail-closed behavior.

## Reproduction Assessment

Yes, at source level for the review finding. The wrapper reconstructs successful auth-covered responses through a plain header record, and a direct Node Fetch check shows duplicate `Set-Cookie` entries collapse to one value through that conversion path.

## Solution Assessment

No, not yet. A registration wrapper is a plausible layer for Convex's HTTP boundary, but it needs lossless header handling and explicit maintainer acceptance of the security and availability tradeoffs.
