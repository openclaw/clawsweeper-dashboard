## What This Changes

The PR adds skill report and appeal schemas, staff API routes, CLI commands, docs, tests, and moderation event timelines for skills and packages.

## Best Possible Solution

Land the feature only after the new skill API and existing web report path preserve the documented report lifecycle and the contributor adds real behavior proof for the CLI/API moderation flows.

## Reproduction Assessment

Yes. Source inspection shows the new API-token report path increments reportCount without the current skill auto-hide branch, and the web UI still calls the public api.skills.report mutation whose existing-report branch ignores the new triaged/dismissed status lifecycle.

## Solution Assessment

No. The overall direction mirrors package moderation, but the skill implementation should share or preserve the existing report lifecycle semantics across web, API, and CLI entry points before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve auto-hide for API skill reports:** `convex/skills.ts:2895-2901`
  - body: The new API-token skill report path increments `reportCount` but never runs the existing auto-hide logic used by `api.skills.report`. A skill reported through `POST /api/v1/skills/{slug}/report` or `clawhub skill report` can pass the documented fourth-report threshold without setting `softDeletedAt`, `moderationStatus = "hidden"`, embeddings visibility, or the `skill.auto_hide` audit entry, so the new public reporting surface does not enforce the current moderation contract.
  - confidence: 0.91
- **[P2] Reopen triaged reports from the web report path:** `convex/skills.ts:2703-2707`
  - body: With the new report statuses, `api.skills.report` still returns `alreadyReported` for any existing report before checking whether it was triaged or dismissed. Since the skill detail page still uses this mutation, a user whose previous report was resolved cannot submit a new report from the web UI, while the new API-token path can reopen it and update `reportCount`.
  - confidence: 0.86
