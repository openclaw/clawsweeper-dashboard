Full review comments:

- **[P2] Preserve auto-hide for API skill reports:** `convex/skills.ts:2903-2907`
  - body: The API/CLI report path increments `reportCount` but never applies the existing auto-hide patch, embedding visibility update, or `skill.auto_hide` audit log. When the fourth unique report arrives through `POST /api/v1/skills/{slug}/report` or `clawhub skill report`, the skill can remain public despite the documented auto-hide contract.
  - confidence: 0.92
- **[P2] Reopen resolved reports in the web report flow:** `convex/skills.ts:2714`
  - body: After adding `confirmed` and `dismissed` statuses, this branch still returns `alreadyReported` for any existing report. The skill detail page still uses this mutation, so a user whose prior report was resolved cannot file a new web report even though the new API path can reopen it.
  - confidence: 0.88
- **[P2] Page open skill reports before filtering resolved rows:** `convex/skills.ts:3273-3287`
  - body: For `status=open`, the query paginates all reports by `createdAt` and only then drops non-open rows. Recent confirmed or dismissed reports can fill a page and hide older open reports from the moderator API/CLI.
  - confidence: 0.86
- **[P2] Include legacy triaged reports in confirmed filters:** `convex/packages.ts:2623-2625`
  - body: The schema still allows stored `triaged` package reports and `readArtifactReportStatus` maps them to `confirmed`, but `status=confirmed` queries only exact `confirmed` rows. Existing triaged package cases disappear from the confirmed report list unless staff use `all`.
  - confidence: 0.82
