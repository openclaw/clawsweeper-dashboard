## Summary

Keep this PR open for maintainer handling: the shared-builder direction is useful, but the exact head retains a credential-forwarding development proxy, executes an unpinned external repository during production builds, has a concrete TypeScript defect, lacks the required trust-boundary specification, and does not yet show adequate browser proof. The companion builder dependency at https://github.com/openclaw/docs/pull/39 is also still open.

## What This Changes

The PR replaces ClawHub’s copied docs renderer with an adapter to the shared openclaw/docs builder, serves generated output under `/docs`, and changes related navigation, docs authentication, proxying, CSP, and tests.

## Best Possible Solution

Land an approved immutable shared-builder contract, route all assistant traffic through one credential-stripping proxy, preserve or explicitly migrate existing auth and navigation behavior, document the trust boundary in `specs/`, and verify both successful and failed builds plus the complete browser flow before merge.

## Reproduction Assessment

Yes for the review defects: source inspection gives a high-confidence path for the Vite credential forwarding, and the exact-head review identifies a deterministic TypeScript error. The requested shared-docs capability itself is a feature, not a bug reproduction.

## Solution Assessment

No in its current form; sharing the renderer is maintainable, but the narrow solution is an immutable builder dependency and one safe proxy path without bundling unrelated navigation changes or an unproven route migration.

  - body: The Vite proxy sends `/ask-molty` directly to `ASK_MOLTY_PROXY_ORIGIN`, allowing browser `Cookie` and `Authorization` headers to cross origins even though the Nitro route strips them. Remove this parallel path or apply equivalent filtering before forwarding.
  - confidence: 0.99
- **[P1] Pin the shared builder before executing it:** `scripts/docs-builder.mjs:80`
  - body: `bun run build` now obtains and executes another repository, but a revision is checked out only when an optional environment variable is supplied. Make the normal build use an approved immutable commit or release so upstream changes and branch movement cannot alter or break ClawHub builds.
  - confidence: 0.97
- **[P2] Type the fetch mock so the build gate passes:** `server/routes/ask-molty/[...].test.ts:9`
  - body: The zero-argument mock causes `mock.calls[0]` to be typed as an empty tuple, so casting it to `[URL, RequestInit]` triggers TS2352. Give the mock a fetch-compatible input and init signature before inspecting its recorded arguments.
  - confidence: 0.99
- **[P2] Specify the new assistant proxy trust boundary:** `server/routes/ask-molty/[...].ts:20-31`
  - body: This adds a configurable cross-origin assistant proxy and explicit credential filtering, but no focused `specs/` note records the permitted origin, forwarded headers, credential rules, redirects, or failure behavior. Repository policy requires durable intent for security-sensitive API trust boundaries.
  - confidence: 0.98
