## What This Changes

The PR stages ClawHub documentation into the shared openclaw/docs builder, serves generated output under `/docs`, and changes docs auth/proxy behavior, header navigation, CSP, and related tests.

## Best Possible Solution

Land a rebased and reproducible docs integration that consumes an approved pinned builder revision or package, routes every assistant request through credential filtering, records the authentication/proxy invariants in `specs/`, preserves intentionally supported upgrade behavior, and includes redacted browser/network proof.

## Solution Assessment
