## Summary

This PR still needs changes before merge: the exact head exposes ClawHub credentials through the development proxy, depends on an unmerged and mutable external builder contract, removes auth-route compatibility, and lacks real browser proof for the integrated docs flow.

## What This Changes

The PR replaces ClawHub’s copied docs renderer with an adapter to the shared openclaw/docs builder, publishes generated documentation under `/docs`, and changes related auth, proxy, navigation, CSP, build, and test behavior.

## Best Possible Solution

Use one credential-stripping assistant proxy in development and production, merge and immutably pin the approved shared-builder contract, preserve or redirect `/docs/auth`, separate or explicitly approve the header redesign, document the trust boundary in `specs/`, and prove the complete browser flow.

## Reproduction Assessment

Yes for the proxy defect: exact-head source shows that Vite directly forwards browser requests to `ASK_MOLTY_PROXY_ORIGIN`, while the Nitro route strips credentials. The complete docs and auth UI flow has not been demonstrated in a real browser.

## Solution Assessment

No, not yet; centralizing docs rendering is maintainable, but the implementation must not create a weaker parallel proxy, remove route compatibility, or couple production builds to an unsettled mutable dependency.
