## Summary

The useful Mintlify source preparation is not ready to merge because the final branch removes the live documentation publishing path before its replacement exists, bypasses credential stripping in development, drops the existing docs-auth route without compatibility handling, and bundles an unrelated navigation redesign. The collaborator-authored PR also requires explicit maintainer handling, and its claimed browser result has no inspectable proof artifact.

## What This Changes

The PR converts `docs/` into a Mintlify source tree, removes the OpenClaw docs-sync bridge, and changes documentation publishing, docs auth and assistant proxying, app navigation, CSP, and build safeguards.

## Best Possible Solution

Land a narrow Mintlify-source preparation first, retain current publication and `/docs/auth` compatibility until an atomic hosted cutover, route every environment through the credential-stripping proxy, and move the header redesign into a separately reviewed UI change.
