## Label Justifications

- `P1`: The current patch can expose development credentials, break existing docs-auth consumers, or stop production builds for real users and operators.
- `merge-risk: 🚨 compatibility`: The patch removes `/docs/auth` and established navigation/theme behavior without an upgrade-compatible path.

## Summary

This PR should remain open for maintainer handling: the shared-builder direction is useful, but the current head still has concrete credential-boundary, build reproducibility, route compatibility, specification, scope, and proof blockers. The author is a repository collaborator, so cleanup closure is not permitted, and the companion builder PR is still open.

## What This Changes

The branch replaces ClawHub’s copied docs renderer with an adapter to the shared openclaw/docs builder, serves generated docs under `/docs`, and also changes docs authentication, assistant proxying, CSP, navigation, header styling, dependencies, and tests.

## Best Possible Solution

Land a narrowly scoped migration after the shared builder contract is approved: require an immutable builder revision, preserve `/docs/auth` compatibility, route development and production through the same credential-stripping boundary, document its invariants in `specs/`, and separate the header redesign unless maintainers explicitly approve it.

## Reproduction Assessment

Yes for the review defects: current-head source provides a high-confidence path showing direct development proxying, optional-only revision pinning, and removal of the old auth route. The full integrated browser behavior has not been demonstrated.

## Solution Assessment

No. Sharing the renderer is a maintainable direction, but the current implementation is not the safest solution because it weakens reproducibility, removes compatibility, creates inconsistent proxy behavior, and bundles a separate UI redesign.

Full review comments:

- **[P1] Route development through the credential-stripping proxy:** `vite.config.ts:199-207`
  - body: The Vite proxy forwards `/ask-molty` directly to `ASK_MOLTY_PROXY_ORIGIN`, so browser cookies and authorization headers bypass the stripping performed by the Nitro route. Proxy development traffic through the local server handler or explicitly remove those headers before forwarding.
  - confidence: 0.99
- **[P1] Pin the builder revision used by production builds:** `scripts/docs-builder.mjs:66-80`
  - body: When no checkout exists, the production build clones the current default branch and only honors a revision when an optional environment variable is supplied. Require an approved immutable SHA or packaged builder so identical ClawHub commits cannot later execute different code or fail solely because the external repository changed.
  - confidence: 0.98
- **[P1] Preserve the existing docs-auth route during migration:** `src/routes/auth/docs.tsx:15-17`
  - body: Renaming the route from `/docs/auth` to `/auth/docs` removes an existing callback/bookmark target without a redirect or compatibility handler. Keep the old route as an alias or add a tested migration path before changing generated docs clients.
  - confidence: 0.98
- **[P2] Specify the assistant proxy trust boundary:** `server/routes/ask-molty/[...].ts:11-25`
  - body: This new cross-origin proxy defines credential stripping, origin selection, redirect, and failure behavior, but no `specs/` note records those security invariants as required by repository policy. Add a focused specification so future changes cannot silently weaken the boundary.
  - confidence: 0.95
- **[P1] Separate or explicitly approve the header redesign:** `src/lib/nav-items.ts:50-62`
  - body: The docs-builder migration also removes Publishers navigation, tab rows, nav icons, and direct theme controls.
