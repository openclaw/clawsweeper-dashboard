## Summary

Keep open. Current source and the modern `clawhub` npm package declare `undici`, but the reported legacy `clawdhub@latest` path still resolves to `0.3.0` without that dependency, and ClawHub policy keeps issues open for maintainer handling rather than auto-closing them as source-fixed.

## What This Changes

The issue reports that the legacy `clawdhub` npm package imports `undici` at runtime without declaring it as a dependency, causing Node/npm installs to fail with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

A maintainer should make the legacy package path unambiguous by publishing or promoting a fixed stable `clawdhub`, deprecating or migrating the broken legacy package toward `clawhub`, or otherwise ensuring legacy install commands no longer resolve to a package missing `undici`.

## Reproduction Assessment

Yes. The issue gives concrete Node/npm failure steps, and current registry metadata still shows `clawdhub@latest` resolving to `0.3.0` without `undici`, which matches the reported module-resolution failure.

## Solution Assessment

No source-only change is sufficient now. Current source and the modern `clawhub` package already declare the dependency, so the narrow maintainable fix is maintainer-controlled npm distribution, dist-tag, deprecation, or migration handling for legacy `clawdhub`.
