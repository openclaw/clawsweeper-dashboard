## Summary

Keep open. Current main and the modern `clawhub` npm package declare `undici`, but the reported legacy `clawdhub@latest` install path still resolves to `0.3.0` without that dependency, so the user-visible Node failure remains a maintainer-owned npm distribution issue.

## What This Changes

The issue reports that `clawdhub@0.3.0` and `clawdhub@latest` crash under Node because the built HTTP runtime imports `undici` while the published package does not list it as a dependency.

## Best Possible Solution

A maintainer should make the legacy package path unambiguous by publishing or promoting a fixed stable `clawdhub`, deprecating or migrating the broken legacy package toward `clawhub`, or otherwise ensuring legacy install commands no longer resolve to a package missing `undici`; then close this and related reports after a clean Node >=20 smoke test.
