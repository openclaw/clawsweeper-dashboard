## Summary

Keep open: current source and the modern `clawhub` package include `undici`, but the reported legacy `clawdhub@latest` registry path still resolves to `0.3.0` without that dependency, so the user-visible failure remains outside a normal source-only fix.

## What This Changes

The issue reports that the published legacy `clawdhub` CLI crashes on Node.js because it imports `undici` without declaring it as a runtime dependency.

## Best Possible Solution

Make the legacy package path unambiguous by publishing or promoting a fixed stable `clawdhub`, deprecating or migrating it to `clawhub`, and then closing this plus the related reports after a clean Node smoke test.

## Reproduction Assessment

Yes. Live npm metadata still gives a high-confidence reproduction path: `clawdhub@latest` resolves to `0.3.0` without `undici`, while the runtime imports `undici`; I did not execute a clean install because this review is read-only.

## Solution Assessment

No source-only change is the best remaining fix. The manifest on current `main` already declares `undici`; the unresolved work is maintainer-controlled registry, release, dist-tag, deprecation, or migration handling for the legacy package.
