## Summary

Keep open. Current source and the modern `clawhub` npm package include `undici`, but the reported legacy `clawdhub@latest` install path still resolves to `0.3.0` without that dependency, so the user-visible failure remains unresolved and needs maintainer distribution action.

## What This Changes

The issue requests adding the missing `undici` runtime dependency and publishing a fixed package so Node-based `clawdhub` installs no longer crash with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

A maintainer should make the legacy package path unambiguous by publishing or promoting a fixed stable `clawdhub`, deprecating or migrating the broken legacy package toward `clawhub`, or otherwise ensuring legacy install commands no longer resolve to a package missing `undici`.

## Reproduction Assessment

Yes. The issue gives concrete Node/npm reproduction paths, and current npm registry metadata still shows `clawdhub@latest` resolving to `0.3.0` without `undici`, which matches the reported module-resolution failure.

## Solution Assessment

No source-only change is sufficient now. The dependency is already present in current source and the modern package, so the narrow maintainable fix is a maintainer-controlled npm distribution, dist-tag, deprecation, or migration action for legacy `clawdhub`.
