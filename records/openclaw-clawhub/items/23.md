## Summary

Keep open: current source and the maintained `clawhub` npm package declare `undici`, but live legacy `clawdhub@latest` still resolves to `0.3.0` without that dependency, so the user-visible Node crash remains a maintainer distribution task.

## What This Changes

Report that `clawdhub@0.3.0`/`@latest` crashes under Node because the published build imports `undici` without declaring it.

## Best Possible Solution

A maintainer should publish or promote a fixed stable `clawdhub`, deprecate or migrate the broken legacy package toward `clawhub`, or otherwise ensure legacy install commands no longer resolve to a package missing `undici`.

## Reproduction Assessment

Yes for the live legacy distribution path: registry metadata and the `clawdhub@0.3.0` tarball show a package that imports `undici` without declaring it. Current main itself does not reproduce the source dependency omission because `packages/clawhub/package.json` already includes `undici`.

## Solution Assessment

No source-only dependency change is the best remaining fix. The manifest is already fixed in current source and the maintained package, so the narrow maintainable solution is npm release, dist-tag, deprecation, or migration handling for legacy `clawdhub`.
