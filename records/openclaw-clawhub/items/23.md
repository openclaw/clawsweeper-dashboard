## Summary

Keep open: current source and the maintained `clawhub` package declare `undici`, but live `clawdhub@latest` still resolves to `0.3.0`, whose tarball imports `undici` without declaring it. The remaining fix is npm distribution handling for the legacy package, not a repo-only source patch.

## What This Changes

The issue reports that `clawdhub@0.3.0` and `clawdhub@latest` fail under Node because the published package imports `undici` without listing it as a dependency.

## Best Possible Solution
