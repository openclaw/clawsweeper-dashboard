## Summary

Keep open. Current `main` and the modern `clawhub` npm package declare `undici`, but the live legacy `clawdhub@latest` path still resolves to `0.3.0` without that dependency, so the reported Node/npm failure remains a maintainer distribution issue.

## What This Changes

The issue asks for the legacy `clawdhub` npm package to include `undici` so Node/npm installs do not fail with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

A maintainer should make the legacy package path unambiguous by publishing or promoting a fixed stable `clawdhub`, deprecating or migrating it toward `clawhub`, then verify a clean Node smoke test and close the related reports together.

## Reproduction Assessment

Yes. Live registry metadata shows `clawdhub@latest` resolves to `0.3.0` without `undici`, while the runtime imports `undici`; the issue and comments provide concrete Node/npm failure paths.

## Solution Assessment

No source-only change is the best remaining fix. Current source and the modern package already declare `undici`, so the narrow maintainable solution is maintainer-controlled npm release, dist-tag, deprecation, or migration handling for the legacy package.
