## Summary

Keep open. Current source and the modern `clawhub` package declare `undici`, but the reported legacy `clawdhub@latest` install path still resolves to `0.3.0` without that dependency, so the user-visible Node/npm failure remains a maintainer distribution issue.

## What This Changes

Add the missing `undici` runtime dependency so `clawdhub` installed via Node/npm does not fail with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

A maintainer should make the legacy package path unambiguous by publishing or promoting a fixed stable `clawdhub`, deprecating or migrating it toward `clawhub`, then verifying clean Node smoke tests and closing #33 and #51 with the same decision.

## Reproduction Assessment

Yes. Live registry metadata shows `clawdhub@latest` still resolves to `0.3.0` without `undici`, while the runtime imports `undici`, and the issue discussion provides concrete Node/npm/pnpm failure paths.

## Solution Assessment

No source-only change is the best remaining fix. Current source and the modern package already declare `undici`, so the narrow maintainable solution is maintainer-controlled npm release, dist-tag, deprecation, or migration handling for the legacy package.
