## Summary

Keep open: current source and the maintained `clawhub` package declare `undici`, but live `clawdhub@latest` still resolves to `0.3.0` without that dependency, so the reported legacy install path remains broken and needs maintainer npm distribution handling.

## What This Changes

The issue requests adding `undici` to the CLI package dependencies so Node package-runner installs of `clawdhub` do not fail with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

A maintainer should publish or promote a fixed stable `clawdhub` release, or deprecate and redirect the legacy package to `clawhub`, then verify a clean Node package-runner smoke test and consolidate duplicates.

## Reproduction Assessment

Yes. Live npm metadata and tarball inspection show `clawdhub@latest` resolves to `0.3.0`, whose package metadata omits `undici` while `dist/http.js` imports it; I did not run a clean `npx` or `pnpm dlx` smoke test in this read-only review.

## Solution Assessment

No. Adding `undici` to current source is already done for the maintained package; the narrow remaining solution is npm publish, dist-tag, deprecation, or migration handling for legacy `clawdhub`.
