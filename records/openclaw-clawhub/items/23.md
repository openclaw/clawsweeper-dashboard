## Summary

Keep open: current source and the maintained `clawhub` package include `undici`, but the reported legacy `clawdhub@latest` distribution still resolves to a package artifact that omits it, so the remaining user-visible failure needs maintainer npm distribution handling.

## What This Changes

The issue asks for the CLI package to declare `undici` so Node.js installs of `clawdhub` do not crash with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

A maintainer should resolve the legacy npm package state by publishing or promoting a fixed stable `clawdhub`, or by deprecating/migrating it to `clawhub` with clear guidance, then verify a clean Node install path.

## Reproduction Assessment

## Solution Assessment
