## Summary

Keep open. Current source and the modern `clawhub` npm package declare `undici`, but the reported legacy `clawdhub@latest` path still resolves to `0.3.0` without that dependency, so the user-visible failure is still plausible and needs maintainer distribution handling.

## What This Changes

The issue reports that `clawdhub@0.3.0` fails under Node.js because its published package imports `undici` without declaring it as a dependency.

## Best Possible Solution

A maintainer should make the legacy package path unambiguous by publishing or promoting a fixed `clawdhub`, deprecating or redirecting users toward `clawhub`, and then closing the clustered reports after a clean Node smoke test.
