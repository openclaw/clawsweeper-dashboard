## Summary

Keep open. Current `main` and the modern `clawhub` package declare `undici`, but the live legacy `clawdhub@latest` path still resolves to `0.3.0` without that dependency, so the reported Node/npm failure remains a maintainer distribution issue.

## What This Changes

The issue reports that the legacy `clawdhub` npm package imports `undici` at runtime without declaring it as a dependency.

## Best Possible Solution

A maintainer should make the legacy package path unambiguous by publishing or promoting a fixed stable `clawdhub`, deprecating or migrating users to `clawhub`, and then verifying a clean Node smoke test before closing the related reports.

## Reproduction Assessment

Yes for the live distribution path: `clawdhub@latest` still resolves to `0.3.0`, its metadata omits `undici`, and its tarball imports `undici`. Current `main` itself does not reproduce the missing dependency because the modern manifest already declares it.

## Solution Assessment

No source-only change is the best remaining fix. The maintainable path is maintainer-controlled npm release, dist-tag, deprecation, or migration handling for the legacy package.
