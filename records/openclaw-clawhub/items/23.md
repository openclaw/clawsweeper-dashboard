## Summary

Keep open. Current `main` and the modern `clawhub` package declare `undici`, but the live legacy `clawdhub@latest` path still resolves to `0.3.0` without that dependency, so the reported Node/npm failure remains a maintainer distribution issue.

## What This Changes

The issue requests that the legacy `clawdhub` package include `undici` so Node/npm installs do not crash with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

A maintainer should make the legacy package path unambiguous by publishing or promoting a fixed stable `clawdhub`, deprecating or migrating the broken legacy package toward `clawhub`, or otherwise ensuring legacy install commands no longer resolve to a package missing `undici`.

## Reproduction Assessment

Yes. The issue provides concrete Node/npm steps, and live registry metadata still shows `clawdhub@latest` resolving to `0.3.0` without `undici`, which matches the reported module-resolution failure.

## Solution Assessment

No source-only change is sufficient now. The dependency is already present in current source and in the modern `clawhub` package, so the narrow maintainable fix is a maintainer-controlled npm distribution, dist-tag, deprecation, or migration action for legacy `clawdhub`.
