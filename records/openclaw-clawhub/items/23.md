## Summary

Keep open: current source and the modern `clawhub` package declare `undici`, but the reported legacy `clawdhub@latest` distribution still points at `0.3.0` without that dependency, so the user-visible Node crash remains a maintainer npm distribution task.

## What This Changes

The issue asks for the legacy `clawdhub` CLI package to declare `undici` so Node.js installs do not fail with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

A maintainer should make the legacy package path unambiguous by publishing or promoting a fixed stable `clawdhub`, deprecating or migrating it toward `clawhub`, or otherwise ensuring legacy install commands no longer resolve to a package missing `undici`.

## Reproduction Assessment

Yes, for the live legacy npm path: registry metadata and the `clawdhub@0.3.0` tarball show `dist/http.js` importing `undici` while package metadata omits it. Current main does not reproduce the source omission because `packages/clawhub/package.json` already declares `undici`.

## Solution Assessment

No, a source-only dependency add is no longer the best remaining fix. The maintainable path is npm release, dist-tag, deprecation, or migration handling for legacy `clawdhub`, followed by a clean Node smoke test.
