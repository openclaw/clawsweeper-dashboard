## Summary

Keep open. Current `main` and the modern `clawhub` package declare `undici`, but live npm metadata still has legacy `clawdhub@latest` pointing at `0.3.0` without that dependency, so the reported Node/npm failure remains a maintainer-controlled distribution issue.

## What This Changes

The issue reports that `clawdhub@latest`/`0.3.0` crashes on Node because `dist/http.js` imports `undici` but that published package version does not declare it as a dependency.

## Best Possible Solution

A maintainer should make the legacy package path unambiguous by publishing or promoting a fixed stable `clawdhub`, deprecating or migrating it toward `clawhub`, or otherwise ensuring legacy install commands no longer resolve to a package missing `undici`, then verify #33 and #51 by the same decision.

## Reproduction Assessment

Yes. The live registry still resolves `clawdhub@latest` to `0.3.0` without `undici`, and the reported runtime imports `undici`, so a clean Node/npm install path has a high-confidence failure mode even though I did not perform an installing smoke test.

## Solution Assessment

No source-only change is the best remaining fix now. Current source and the modern `clawhub` package already declare the dependency, so the narrow maintainable resolution is a maintainer-controlled npm distribution, dist-tag, deprecation, or migration action for legacy `clawdhub`.
