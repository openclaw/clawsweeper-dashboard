## Summary

Keep open. Current source and the modern `clawhub` npm package declare `undici`, but live npm metadata still has legacy `clawdhub@latest` resolving to `0.3.0` without that dependency, so the reported Node install path remains broken and needs maintainer distribution action.

## What This Changes

The issue asks for `undici` to be declared and published so legacy `clawdhub` installs on Node do not crash with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

A maintainer should make the legacy package path unambiguous by publishing or promoting a fixed stable `clawdhub`, deprecating or migrating it toward `clawhub`, or otherwise ensuring legacy install commands no longer resolve to a package missing `undici`.

## Reproduction Assessment

Yes. The high-confidence reproduction path is a clean Node install that resolves `clawdhub@latest` to `0.3.0`; live registry metadata shows that package lacks `undici` while the runtime imports it, matching the logs in this issue, #33, and #51.

## Solution Assessment

No, the source-level fix alone is not the complete user-visible solution because it is already present on current main and in modern `clawhub`; the narrow remaining solution is maintainer-controlled npm dist-tag, release, or deprecation handling for legacy `clawdhub`.
