## Summary

Keep open: current source and the modern `clawhub` package declare `undici`, but live `clawdhub@latest` still resolves to the broken `0.3.0` artifact, and this repository’s cleanup policy keeps issues open for maintainer handling.

## What This Changes

The issue reports that legacy `clawdhub` installs fail on Node because the published package imports `undici` without declaring it as a runtime dependency.

## Best Possible Solution

A maintainer should make the legacy package path unambiguous by publishing or promoting a fixed stable `clawdhub`, deprecating/migrating it to `clawhub`, or otherwise ensuring legacy install commands no longer resolve to an artifact missing `undici`.

## Reproduction Assessment

Yes. Live registry metadata and tarball inspection show `clawdhub@latest` still resolves to `0.3.0`, whose runtime imports `undici` while its package metadata omits it; I did not execute `npx` in the read-only checkout.

## Solution Assessment

No. The requested source dependency add is already present for the maintained package, so the remaining maintainable fix is npm release, dist-tag, deprecation, or migration handling for legacy `clawdhub`.
