Keep open: current main and the modern `clawhub` package declare `undici`, but the live legacy `clawdhub@latest` distribution still resolves to the broken `0.3.0` package, and ClawHub policy keeps issues open for maintainer handling.

## What This Changes

The issue asks for the CLI package to include `undici` so Node-based `npx` or global installs do not fail with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

A maintainer should make the legacy package path unambiguous by publishing or promoting a fixed stable `clawdhub`, deprecating or migrating the broken legacy package toward `clawhub`, and then verifying a clean Node smoke test.

## Solution Assessment
