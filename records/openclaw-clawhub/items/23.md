## Summary

Keep open. Current source already declares `undici` for the modern `clawhub` CLI, but the reported user-visible failure still exists on the legacy `clawdhub@latest` npm path, so this needs maintainer distribution handling rather than cleanup closure.

## What This Changes

The issue asks for the Node/npm CLI package path to include the runtime `undici` dependency so fresh `clawdhub` installs do not fail with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

A maintainer should make the legacy package path unambiguous by publishing or promoting a fixed stable `clawdhub`, deprecating or migrating it toward `clawhub`, or otherwise ensuring legacy install commands no longer resolve to a package missing `undici`.

## Reproduction Assessment

Yes. The issue and linked reports provide concrete Node/npm or pnpm fresh-install commands that fail when the legacy package resolves to `clawdhub@0.3.0`, and the current registry metadata still matches that failure mode.

## Solution Assessment

No source-only change is the best remaining fix. Current source already declares `undici`, so the maintainable next step is maintainer-controlled npm distribution, dist-tag, deprecation, or migration handling for the legacy package path.
