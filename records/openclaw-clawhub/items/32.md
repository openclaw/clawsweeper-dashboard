## Summary

Keep open: current main intentionally blocks deleted/deactivated account sign-in and documents permanent deletion, but the report and follow-up comments request same-account re-authentication after self-deletion, which remains a maintainer product/security decision.

## What This Changes

The issue asks ClawHub to let users who soft-delete their account sign back in with the same GitHub/OAuth identity instead of returning to a signed-out state.

## Best Possible Solution

Maintainers should explicitly choose the deletion policy, then either keep irreversible deletion with clearer OAuth/settings/support messaging or implement a scoped same-identity recovery/recreation flow with regression coverage.

## Reproduction Assessment

Yes. A code-level reproduction is clear: after deleteAccount sets deactivatedAt/purgedAt, the OAuth afterUserCreatedOrUpdated callback calls handleDeletedUserSignIn and the current tests assert deactivated and legacy deleted users are rejected.

## Solution Assessment

Unclear. Current main is a coherent irreversible-deletion implementation, but it does not satisfy the requested same-account re-authentication behavior; recovery needs maintainer policy and strict identity checks first.
