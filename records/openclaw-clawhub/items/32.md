## Summary

Keep open. Current main intentionally blocks deleted/deactivated account sign-in and now surfaces clearer blocked-account messaging, but this is an open ClawHub issue in an active account-recovery support cluster, and the remaining question is product/security policy rather than conservative cleanup.

## What This Changes

The issue asks for users who soft-deleted a ClawHub account to be able to sign in again with the same OAuth identity instead of returning to the signed-out homepage.

## Best Possible Solution

Define and document the account-deletion end state: either permanent non-restorable deletion with clearer support handling, or audited same-provider recovery/recreation that preserves bans and purged data boundaries.

## Reproduction Assessment

Yes, from source inspection for current main: deleteAccount sets deactivatedAt/purgedAt and handleDeletedUserSignIn rejects deleted/deactivated OAuth sign-in. I did not run a live browser OAuth flow, so the exact original silent homepage loop is not proven on main.

## Solution Assessment

No for an automated fix. The current implementation intentionally makes account deletion irreversible, while adding recovery would change identity, privacy, ownership, and ban-boundary policy.
