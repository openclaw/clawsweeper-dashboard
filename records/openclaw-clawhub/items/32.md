## Summary

Keep open. This is an open ClawHub issue, and current main still intentionally blocks deleted/deactivated GitHub OAuth reauth while repeated user reports show unresolved account-recovery demand that needs maintainer product/security direction.

## What This Changes

The reporter asks for GitHub OAuth sign-in to work again after soft-deleting a ClawHub account and revoking OAuth access.

## Best Possible Solution

Maintainers should choose a canonical deleted-account lifecycle policy, then implement the approved recovery, re-registration, or support-only path across Convex auth, deletion/admin tooling, UI messaging, docs/specs, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, the GitHub OAuth callback calls handleDeletedUserSignIn, and that handler rejects the sign-in; I did not run a live OAuth flow.

## Solution Assessment

No for an automated one-off restore or unlink. The maintainable solution needs a maintainer-approved account lifecycle policy that separates self-deleted accounts from banned or abuse-disabled identities.
