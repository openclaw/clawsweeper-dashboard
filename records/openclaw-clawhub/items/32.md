## Summary

Keep open. Current main intentionally rejects deleted/deactivated users during OAuth and now shows a deleted-account message, but the issue asks for recovery or re-registration after self-deletion, which is a product/security decision rather than a cleanup close.

## What This Changes

The issue reports that users who deleted their ClawHub account cannot sign back in or re-register with the same GitHub OAuth identity.

## Best Possible Solution

Maintainers should choose the deleted-account lifecycle, then either keep terminal deletion with clearer OAuth/support guidance or implement scoped same-provider recovery or account recreation with ban, purge, audit, and identity safeguards.

## Reproduction Assessment

Yes, at source level: deleteAccount writes deactivatedAt/purgedAt, OAuth calls handleDeletedUserSignIn, and tests assert deleted users are rejected. I did not run a live production OAuth flow or inspect individual account rows.

## Solution Assessment

Unclear until maintainers choose the deletion lifecycle policy. The current terminal-deletion implementation is internally consistent, but recovery or re-registration would need an explicit security-reviewed product path.
