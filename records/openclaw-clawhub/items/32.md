## Summary

Keep open. Current main intentionally blocks deleted/deactivated GitHub-backed accounts and now surfaces clearer blocked-account auth errors, but the issue and clustered reports still ask for recovery or re-registration after self-deletion; that remaining decision needs maintainer product/security judgment under the ClawHub issue-cleanup policy.

## What This Changes

The issue requests that users who deleted their ClawHub account be able to sign back in or recover using the same GitHub account instead of returning to the signed-out homepage.

## Best Possible Solution

Maintainers should decide and document the deleted-account lifecycle, then either keep irreversible deletion with explicit support/sign-in guidance or implement a secure same-provider recovery/re-registration/admin-reset flow with regression coverage for deleted, deactivated, legacy deletedAt, banned, autobanned, and mismatched-identity cases.

## Reproduction Assessment

Yes, from source inspection: deleteAccount writes deactivatedAt/purgedAt, and the OAuth afterUserCreatedOrUpdated callback rejects that user with the permanent deleted-account message. I did not run a live GitHub OAuth browser flow, so the exact visual redirect loop was not runtime-reproduced here.

## Solution Assessment

No for an automatic bug-fix PR. Restoring, purging, or recreating accounts after deletion changes the current irreversible-deletion contract and needs maintainer product/security approval before code should change.
