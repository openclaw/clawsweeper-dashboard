## Summary

Keep this open. Current main now blocks deleted/deactivated-account sign-in with a clear permanent-deletion error, but the user request and follow-up comments are about whether self-deleted accounts should be recoverable, which remains a maintainer product/security decision for this ClawHub issue.

## What This Changes

The issue requests that users who soft-deleted their ClawHub account be able to authenticate again instead of returning to the homepage as signed out.

## Best Possible Solution

Maintainers should explicitly choose permanent deletion versus scoped recovery; then either keep the current irreversible policy with stronger support guidance or implement same-provider reactivation/recreation with regression coverage for deleted, deactivated, banned, autobanned, legacy deletedAt, fresh-login, and mismatched-identity cases.

## Reproduction Assessment

Yes for the current source path: a deleted or deactivated user reaching the OAuth afterUserCreatedOrUpdated callback is rejected, and tests cover deactivated, legacy deletedAt, fresh-login, banned, and autobanned cases. I did not run an end-to-end OAuth flow in this read-only review.

## Solution Assessment

Unclear. Current main is internally consistent if permanent deletion is intended, but allowing recovery or recreation under the same GitHub identity requires explicit product and security approval before code changes.
