## Summary

Keep open. Current main intentionally blocks deleted/deactivated GitHub sign-in and now surfaces a deleted-account error, but this issue and the larger deleted-account login cluster still need a maintainer account-lifecycle decision rather than cleanup closure.

## What This Changes

The issue asks for users who deleted their ClawHub account to be able to authenticate again with the same GitHub account instead of returning to the signed-out state.

## Best Possible Solution

Maintainers should choose one deleted-account lifecycle, then either keep terminal deletion with clearer support/UX or implement a guarded re-registration, restore, or GitHub-binding reset path with audit and ban safeguards.

## Reproduction Assessment

Yes, by source inspection: account deletion writes deactivatedAt/purgedAt, and OAuth completion rejects deactivated or legacy deleted users with a permanent-deletion message. I have not run a live production OAuth flow or inspected affected account rows.

## Solution Assessment

No for an automated one-off restore or unlink. The maintainable solution needs an explicit account-lifecycle and security policy before code changes alter deleted-account recovery semantics.
