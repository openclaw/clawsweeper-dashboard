## Summary

Keep open. This is an issue, and this repository cleanup pass only permits auto-closing pull requests; current main also intentionally blocks deleted-account OAuth re-auth while the requested recovery/reset behavior still needs product and security ownership.

## What This Changes

Users who delete their ClawHub account cannot sign back in or re-register with the same GitHub OAuth identity.

## Best Possible Solution

Maintainers should decide the deleted-account lifecycle, then either keep irreversible deletion with clear OAuth/support messaging or implement an audited same-provider recovery, re-registration, or operator reset path with ban and privacy safeguards.

## Reproduction Assessment

Yes, at the source level: deleteAccount persists deactivatedAt/purgedAt and handleDeletedUserSignIn rejects deleted or deactivated users during OAuth. I did not run a live production OAuth flow or inspect affected production account rows.

## Solution Assessment

No for an automated one-off restore or unlink. The maintainable solution needs an explicit account-lifecycle, privacy, ban, and security policy before code or operator changes alter deleted-account recovery semantics.
