## Summary

Keep open. Current main intentionally blocks deleted/deactivated users and now surfaces clearer deleted-account auth errors, but the issue asks for recovery or re-registration after self-deletion; for ClawHub issues, this needs maintainer product/security handling rather than cleanup closure.

## What This Changes

The issue requests that users who delete their ClawHub account can sign in or re-register again with the same GitHub account instead of completing OAuth and returning signed out.

## Best Possible Solution

Maintainers should decide and document the deleted-account lifecycle, then either keep irreversible deletion with explicit support/sign-in guidance or implement a secure same-provider recovery or re-registration flow with regression coverage.

## Reproduction Assessment

Yes, from source inspection: deleteAccount writes deactivatedAt/purgedAt, and the OAuth afterUserCreatedOrUpdated callback rejects that user with the permanent deleted-account message. I did not run a live GitHub OAuth browser flow, so the exact visual redirect-loop variant was not runtime-reproduced here.

## Solution Assessment

No for an automated fix. Restoring, purging, unlinking, or recreating accounts after deletion changes the current irreversible-deletion contract and needs maintainer product/security approval before code changes.
