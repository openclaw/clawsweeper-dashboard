## Summary

Keep open: current main intentionally blocks deleted-account sign-in and documents permanent deletion, but the report asks for same-account re-authentication after self-deletion and the ClawHub cleanup policy keeps issues open for maintainer product/security decisions.

## What This Changes

Users who soft-delete their ClawHub account expect to re-authenticate with the same GitHub account instead of returning to the home page with a sign-in button.

## Best Possible Solution

Maintainers should decide the deletion policy explicitly, then either keep irreversible deletion with clearer support/UX messaging or implement a scoped same-provider reactivation/recreation flow with regression coverage.

## Reproduction Assessment

Yes. A code-level reproduction is clear: after deleteAccount writes deactivatedAt/purgedAt, the OAuth callback reaches handleDeletedUserSignIn and throws the deleted-account error; convex/auth.test.ts covers both deactivated and legacy deletedAt cases.

## Solution Assessment

Unclear. The current implementation is internally consistent for irreversible deletion, but it does not satisfy the requested same-account re-authentication; changing that requires a maintainer security/product decision first.
