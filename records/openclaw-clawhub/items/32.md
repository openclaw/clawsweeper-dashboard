## Summary

Keep open. Current main intentionally blocks deleted/deactivated users during OAuth and now surfaces a deleted-account message, but the remaining request is a product/security decision about account recovery or re-registration, and ClawHub cleanup policy keeps issues open rather than auto-closing them.

## What This Changes

Users who self-deleted a ClawHub account report that GitHub OAuth completes but they cannot sign back in or re-register with the same GitHub identity.

## Best Possible Solution

Maintainers should choose the deleted-account lifecycle, then either keep terminal deletion with clearer support/OAuth guidance or implement a scoped same-provider recovery or account-recreation path with ban, purge, audit, and identity safeguards.

## Reproduction Assessment

Yes, at source level: deleteAccount writes deactivatedAt/purgedAt, OAuth calls handleDeletedUserSignIn, and tests assert deleted users are rejected. I did not run a live production OAuth flow or inspect individual account rows.

## Solution Assessment

Unclear until maintainers choose the deletion lifecycle policy. A one-off restore or hard-delete from a public report is not the best source fix because recovery needs identity verification, audit, privacy, and ban safeguards.
