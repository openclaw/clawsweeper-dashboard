## Summary

Keep this open. Current main intentionally treats user deletion as permanent and now surfaces a deleted-account sign-in error, but the original report and follow-up comments are asking for same-account re-authentication after self-deletion. Under the ClawHub cleanup policy, issues should remain open unless maintainers explicitly resolve the product/security policy question.

## What This Changes

The issue asks for users who soft-deleted their ClawHub account to be able to sign back in with the same GitHub account instead of returning to the homepage still signed out.

## Best Possible Solution

Maintainers should decide whether user-initiated deletion is intentionally non-restorable. If yes, keep the permanent-deletion implementation and improve OAuth/settings/support messaging; if no, implement a scoped same-provider reactivation or account-recreation flow with regression coverage for deleted, deactivated, banned/autobanned, legacy deletedAt, fresh-login existingUserId null, and mismatched identity cases.
