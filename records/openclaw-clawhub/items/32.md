## Summary

Keep this open: current main intentionally blocks deleted/deactivated users and now surfaces a permanent-deletion sign-in error, but the reported recoverability after self-deletion remains a maintainer product/security decision and ClawHub issue cleanup policy keeps issues open.

## What This Changes

Requests that users who deleted their ClawHub account be able to sign back in with the same GitHub account instead of returning to the home page/sign-in state.

## Best Possible Solution

Maintainers should explicitly choose permanent deletion versus scoped same-provider recovery; then either keep the irreversible policy with stronger support guidance or implement recovery/recreation with focused auth and deletion regression coverage.

## Reproduction Assessment

Yes for the current source path: a deleted or deactivated user reaching the OAuth afterUserCreatedOrUpdated callback is rejected, and tests cover deactivated, legacy deletedAt, fresh-login, banned, and autobanned cases. I did not run an end-to-end OAuth flow in this read-only review.

## Solution Assessment

Unclear. Current main is internally consistent if permanent deletion is intended, but allowing recovery or recreation under the same GitHub identity needs explicit product and security approval before code changes.
