## Summary

Keep open: current main deliberately blocks deleted/deactivated account sign-in and now documents that deletion is irreversible, but the report asks for same-account re-authentication after self-deletion and ClawHub cleanup policy keeps issues open for maintainer product/security decisions.

## What This Changes

Users who deleted their ClawHub account cannot sign back in with the same GitHub account and are returned to the signed-out flow.

## Best Possible Solution

Maintainers should explicitly choose the deletion policy, then either keep irreversible deletion with clearer UX/support guidance or implement a scoped same-provider recovery/recreation flow with regression coverage for self-deleted, banned, autobanned, legacy, and mismatched-identity cases.

## Reproduction Assessment

Yes. A code-level reproduction is clear: deleteAccount writes deactivatedAt/purgedAt, then OAuth afterUserCreatedOrUpdated calls handleDeletedUserSignIn, which throws the deleted-account error; convex/auth.test.ts covers this path.

## Solution Assessment

Unclear. The current permanent-deletion implementation is internally consistent, but it intentionally does not satisfy the requested same-account re-authentication, so a maintainer product/security decision should come before any repair.
