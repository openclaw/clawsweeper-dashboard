## Summary

Keep this open. Current main deliberately treats deleted accounts as permanent and now surfaces a deleted-account sign-in error, but the report and repeated follow-up comments ask for same-account re-authentication after self-deletion, so this remains a maintainer product/security decision rather than a cleanup close.

## What This Changes

Users report that after deleting a ClawHub account, GitHub OAuth completes but the same account cannot sign back in and returns to the signed-out state.

## Best Possible Solution

Maintainers should explicitly choose the account deletion policy: keep irreversible deletion with clear support/UX messaging, or implement a scoped same-provider reactivation/recreation flow with regression coverage for banned, autobanned, legacy, fresh-login, and mismatched-identity cases.

## Reproduction Assessment

Yes. The current-main path is clear from source and tests: deleteAccount marks a user deactivated/purged, then OAuth completion calls handleDeletedUserSignIn and rejects with the deleted-account message rather than restoring access.

## Solution Assessment

Unclear. The current permanent-deletion implementation is internally consistent with docs and settings copy, but it is not the requested re-auth behavior; automatic restoration is not safe to choose without maintainer policy on identity, bans, and purged data.
