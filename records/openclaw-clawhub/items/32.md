## Summary

Keep open: current main intentionally treats deleted accounts as non-restorable and now surfaces a deleted-account sign-in error, but the issue asks for same-account re-authentication after self-deletion and has multiple affected-user comments. For ClawHub issue cleanup, this needs maintainer product/security policy rather than auto-closure.

## What This Changes

Users who self-deleted their ClawHub account report that signing in again with the same GitHub account returns them to the signed-out homepage instead of restoring access.

## Best Possible Solution

Maintainers should explicitly choose the deletion policy, then either keep irreversible deletion with clearer OAuth/settings/support messaging or implement a scoped same-provider recovery/recreation flow with regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: after deleteAccount sets deactivatedAt/purgedAt, the OAuth callback calls handleDeletedUserSignIn and current tests assert deactivated and legacy deleted users are rejected.

## Solution Assessment

Unclear. Current main is internally consistent with irreversible deletion, but it does not satisfy the requested same-account re-authentication behavior; recovery needs maintainer policy and strict identity boundaries first.
