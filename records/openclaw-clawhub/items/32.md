## Summary

Keep open. Current main intentionally blocks deleted/deactivated users during OAuth and now surfaces deleted-account errors, while this issue asks for reauthentication or re-registration after self-deletion; the remaining choice is product/security policy, not conservative cleanup.

## What This Changes

The issue reports that users who self-deleted a ClawHub account cannot sign in or re-register with the same GitHub OAuth identity.

## Best Possible Solution

Maintainers should choose and document the deleted-account lifecycle, then either keep deletion terminal with clear OAuth/support messaging or implement an audited same-provider recovery, re-registration, or unlink path with ban and privacy safeguards.

## Reproduction Assessment

Yes, at the source level: deleteAccount writes deactivatedAt/purgedAt and handleDeletedUserSignIn rejects deactivated or legacy deleted users during OAuth. I did not run a live production OAuth flow or inspect affected account rows.

## Solution Assessment

No for an automated one-off restore, purge, or unlink. The maintainable solution needs explicit account-lifecycle, privacy, ban, audit, and same-provider identity policy before code or operator changes alter deleted-account recovery semantics.
