## Summary

Keep this open. Current main intentionally treats user-initiated deletion as irreversible and now surfaces deleted-account sign-in errors, but this issue and its follow-up comments ask for re-authentication/account recovery after self-deletion, so the remaining decision is product/security policy rather than cleanup closure.

## What This Changes

Users who deleted their account report that GitHub OAuth completes but ClawHub returns them to a signed-out state instead of allowing sign-in again.

## Best Possible Solution

Maintainers should decide whether self-deleted accounts are permanently non-restorable; if yes, keep the current policy and improve UX/support messaging, and if no, implement scoped same-provider restoration or account recreation with ban, autoban, legacy deletedAt, fresh-login, and mismatched-identity coverage.

## Reproduction Assessment

No high-confidence end-to-end reproduction was established because I did not run a live GitHub OAuth flow. Source and tests do show the relevant current-main path: deleted/deactivated users are rejected and should now receive a deleted-account message rather than the original silent homepage loop.

## Solution Assessment

Unclear. Automatic restoration is not clearly the best fix because current docs and tests intentionally define deletion as permanent; the maintainable path starts with a maintainer policy decision, then either messaging/support cleanup or a narrowly scoped restoration flow.
