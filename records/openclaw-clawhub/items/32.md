## Summary

Keep open. Current main deliberately blocks deleted/deactivated account reauth and now surfaces blocked-account messaging, but this ClawHub issue is an active support and policy cluster about account recovery after self-deletion, which should not be closed by conservative cleanup.

## What This Changes

The issue reports that after self-deleting a ClawHub account, GitHub OAuth returns to a signed-out homepage and the user cannot re-authenticate.

## Best Possible Solution

Maintainers should resolve the account-recovery policy/RFC, then either keep permanent deletion with clearer support/docs handling or implement a scoped, audited same-provider recovery/recreation path with regression coverage.

## Reproduction Assessment

Yes, from source inspection: deleteAccount writes deactivatedAt/purgedAt, afterUserCreatedOrUpdated calls handleDeletedUserSignIn, and that helper rejects deleted/deactivated users. I did not run a live GitHub OAuth browser flow.

## Solution Assessment

No for an automated fix. Restoring, recreating, or unlinking accounts changes deletion, ownership, identity, and ban-bypass semantics, so the narrow maintainable path starts with maintainer policy approval.
