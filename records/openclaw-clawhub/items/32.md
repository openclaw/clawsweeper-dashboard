## Summary

Keep open. Current main intentionally blocks deleted/deactivated accounts during OAuth and now warns that permanent deletion cannot sign in again, while this issue and its related cluster ask for re-authentication or reset after self-deletion; that remaining choice is product/security policy, not conservative cleanup.

## What This Changes

Users report that after deleting a ClawHub account, GitHub OAuth completes but they cannot sign back in or create a new account with the same GitHub identity.

## Best Possible Solution

Maintainers should choose and document the deleted-account lifecycle, then either keep deletion terminal with clear OAuth/support messaging or implement an audited same-provider recovery, re-registration, or operator reset path with ban and privacy safeguards.

## Reproduction Assessment

Yes, at the source level: deleteAccount writes deactivatedAt/purgedAt and handleDeletedUserSignIn rejects deactivated or legacy deleted users during OAuth. I did not run a live production OAuth flow or inspect affected account rows.

## Solution Assessment

No for an automated one-off restore, purge, or unlink. The maintainable solution needs explicit account-lifecycle, privacy, ban, audit, and same-provider identity policy before code or operator changes alter deleted-account recovery semantics.
