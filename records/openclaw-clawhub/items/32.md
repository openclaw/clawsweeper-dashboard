## Summary

Keep open. Current main intentionally blocks deleted/deactivated account reauth and now surfaces blocked-account messaging, but this open ClawHub issue has repeated affected-user reports and the remaining choice is maintainer product/security policy, not conservative cleanup closure.

## What This Changes

Users who deleted a ClawHub account are asking to re-authenticate, recover, or re-register with the same GitHub account instead of returning to a signed-out state after OAuth.

## Best Possible Solution

Maintainers should choose and document one deleted-account lifecycle policy, then implement the approved path consistently across Convex auth, account deletion, support/admin tooling, user-facing errors, docs/specs, and regression tests.

## Reproduction Assessment

Yes, from source inspection. deleteAccount writes deactivatedAt/purgedAt, afterUserCreatedOrUpdated calls handleDeletedUserSignIn, and that handler rejects deleted/deactivated users; I did not run a live GitHub OAuth browser flow.

## Solution Assessment

No for an automated fix. Restoring, unlinking, purging, or recreating accounts changes the current irreversible-deletion contract and needs maintainer product/security approval before code changes.

Status: needs_attention
