## Summary

Keep open. Current main intentionally blocks deleted/deactivated accounts from signing in again, and this issue is part of an active deleted-account OAuth cluster that needs maintainer product/security handling rather than cleanup closure.

## What This Changes

The issue asks for ClawHub GitHub login to work again after a user soft-deletes their account and reauthorizes OAuth.

## Best Possible Solution

Maintainers should choose and document the deleted-account lifecycle, then either keep deletion terminal with clear OAuth/support messaging or implement an audited same-provider recovery, re-registration, or operator reset path with ban and privacy safeguards.

## Reproduction Assessment

Yes, at the source level: deleteAccount writes deactivatedAt/purgedAt and handleDeletedUserSignIn rejects deactivated or legacy deleted users during OAuth. I did not run a live production OAuth flow or inspect affected account rows.

## Solution Assessment

No for an automated one-off restore or unlink. The maintainable solution needs an explicit account-lifecycle, privacy, ban, and security policy before code or operator changes alter deleted-account recovery semantics.
