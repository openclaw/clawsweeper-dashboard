## Summary

Keep open. Current main intentionally treats deleted accounts as permanently unable to sign in again, while this issue and the related login cluster still need a maintainer product/security decision about recovery, re-registration, or operator reset.

## What This Changes

Users who delete a ClawHub account want to sign in or re-register again with the same GitHub account instead of returning to the unauthenticated sign-in state.

## Best Possible Solution

Maintainers should choose the deleted-account lifecycle, then either keep terminal deletion with clearer UX/support messaging or implement a guarded same-provider restore, re-registration, or GitHub-binding reset path with audit and ban safeguards.

## Reproduction Assessment

Yes, by source inspection: deleteAccount writes deactivatedAt/purgedAt and handleDeletedUserSignIn rejects deactivated or legacy deleted users during OAuth. I did not run a live production OAuth flow or inspect affected account rows.

## Solution Assessment

No for an automated one-off restore or unlink. The maintainable path needs an explicit account-lifecycle and security policy before code or operator changes alter deleted-account recovery semantics.
