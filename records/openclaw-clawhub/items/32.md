## Summary

Keep open. This is a ClawHub issue, and the repository cleanup policy only allows auto-closing ClawHub pull requests; beyond that, current main treats deleted-account reauth as intentionally blocked, while the issue discussion shows repeated affected-user reports and leaves a product/security decision about account recovery versus permanent deletion.

## What This Changes

The issue reports that users who delete their account cannot sign back in with the same GitHub identity and are returned to the signed-out homepage.

## Best Possible Solution

Maintainers should decide and document the deleted-account lifecycle policy, then align OAuth handling, support/admin tooling, user-facing errors, docs/specs, and regression tests with that policy.

## Reproduction Assessment

Yes, from source inspection. The current OAuth callback path calls deleted-account handling, and the tests assert deleted/deactivated users are rejected instead of restored; I did not run a live GitHub OAuth browser flow.

## Solution Assessment

No for an automated cleanup close or fix. Account restoration versus irreversible deletion is a product and security policy decision, and current main has tests and docs backing the permanent-deletion behavior.
