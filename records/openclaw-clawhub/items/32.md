## Summary

Keep open: current main now treats self-deleted accounts as permanently deactivated and surfaces deleted-account errors, but the issue discussion asks for same-account re-authentication after self-deletion and has repeated affected-user reports. For ClawHub issue cleanup, this needs maintainer product/security policy rather than automatic closure.

## What This Changes

The issue requests fixing GitHub sign-in after a user deletes a ClawHub account and then tries to re-authenticate with the same GitHub identity.

## Best Possible Solution

Maintainers should explicitly choose whether self-deletion is restorable; then either keep irreversible deletion with clearer support/OAuth messaging or implement a scoped same-GitHub recovery or account-recreation flow with regression coverage.

## Reproduction Assessment

Yes. Source inspection shows deleteAccount writes deactivatedAt/purgedAt, OAuth completion calls handleDeletedUserSignIn, and current tests assert deactivated and legacy deleted users are rejected.

## Solution Assessment

Unclear. Current main is a coherent irreversible-deletion implementation, but it does not satisfy the requested same-account re-authentication behavior; recovery needs maintainer policy and strict identity/ban handling first.
