## Summary

Keep open: current main intentionally treats user deletion as permanent, but the issue discussion asks for same-account re-authentication after self-deletion and includes multiple affected users. For ClawHub issues, cleanup closure is not appropriate here; the remaining path is maintainer product/security policy.

## What This Changes

Users who delete their ClawHub account want to sign in again with the same GitHub account instead of returning to the sign-in state.

## Best Possible Solution

Maintainers should choose the deletion policy, then either keep irreversible deletion with clearer OAuth/settings/support messaging or implement a scoped same-identity recovery or recreation flow with regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, the OAuth callback calls handleDeletedUserSignIn, and tests assert deactivated and legacy deleted users are rejected.

## Solution Assessment

Unclear. Current main is a coherent permanent-deletion implementation, but it does not satisfy the requested same-account recovery behavior; the maintainable solution depends on maintainer product and security policy.
