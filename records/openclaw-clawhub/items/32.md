## Summary

Keep open. Current main still treats self-deleted ClawHub accounts as permanently deactivated during GitHub OAuth; the remaining restore, re-register, or reset behavior is security-sensitive account lifecycle policy rather than conservative cleanup closure.

## What This Changes

The issue asks for a deleted GitHub-linked ClawHub account to be able to sign in again, be restored, or be reset instead of returning to the signed-out homepage after OAuth.

## Best Possible Solution

Choose and document the deleted-account lifecycle, then implement either an audited recovery/re-registration/operator-reset path or confirm terminal deletion with clear post-OAuth messaging and support workflow.

## Reproduction Assessment

Yes, at source level: deleteAccount writes deactivatedAt/purgedAt, OAuth calls handleDeletedUserSignIn, and that helper rejects deleted/deactivated records. I did not run a live production OAuth flow or inspect individual account rows.

## Solution Assessment

Unclear until maintainers choose the deletion lifecycle policy. A one-off restore or hard-delete from a public report is not the best source fix because recovery needs identity verification, audit, privacy, and ban safeguards.
