## Summary

Keep open. Current main intentionally rejects deleted/deactivated users during GitHub OAuth and now surfaces a deleted-account message, but the report and follow-up comments ask for recovery or re-registration after self-deletion, which needs maintainer product/security judgment rather than cleanup closure.

## What This Changes

Report that GitHub OAuth sign-in loops or fails after a user soft-deletes their ClawHub account and tries to re-authenticate or re-register.

## Best Possible Solution

Maintainers should choose the deleted-account lifecycle, then either keep terminal deletion with clear OAuth/support guidance or implement scoped same-provider recovery or account recreation with ban, purge, audit, and identity safeguards.

## Reproduction Assessment

Yes, at source level: deleteAccount writes deactivatedAt/purgedAt, OAuth calls handleDeletedUserSignIn, and tests assert deleted users are rejected. I did not run a live production OAuth flow or inspect individual account rows.

## Solution Assessment

Unclear until maintainers choose the deletion lifecycle policy. The current terminal-deletion implementation is internally consistent, but recovery or re-registration would need an explicit security-reviewed product path.
