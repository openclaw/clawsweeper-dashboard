## Summary

Keep open. Current main intentionally blocks OAuth sign-in for deleted/deactivated accounts and now exposes deleted-account errors, but this issue and the related cluster still need maintainer policy and production-support decisions around recovery, re-registration, or irreversible deletion.

## What This Changes

The issue requests that GitHub OAuth sign-in work again after a user deletes their ClawHub account, instead of returning to the signed-out state.

## Best Possible Solution

Maintainers should decide and document the deleted-account lifecycle, then implement either a controlled recovery/re-registration/admin reset flow or an explicit irreversible-deletion support path with tests for active, deleted, legacy-deleted, banned, and autobanned identities.

## Reproduction Assessment

Yes, from source inspection rather than a live production account: deleting an account writes deactivatedAt/purgedAt, and the OAuth callback rejects that linked identity through handleDeletedUserSignIn.

## Solution Assessment

No automated fix is clearly best yet. Auto-restoring, unlinking, or recreating accounts conflicts with the current irreversible-deletion contract, so maintainers need to choose the security/privacy policy first.
