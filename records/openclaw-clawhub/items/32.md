## Summary

Keep this open. Current main now intentionally treats user-initiated deletion as irreversible and surfaces a deleted-account sign-in error, but this is an open ClawHub issue with several affected users and the remaining question is account recovery versus permanent deletion policy.

## What This Changes

Users who deleted a ClawHub account want to sign in again with the same GitHub account instead of being returned to a signed-out state.

## Best Possible Solution

Maintainers should explicitly choose irreversible deletion versus scoped reactivation, then either keep permanent deletion with clear support guidance or implement same-provider restoration/recreation with regression coverage.

## Reproduction Assessment

Yes for the current source path: a deleted or deactivated user reaching the OAuth afterUserCreatedOrUpdated callback is rejected, and helper tests cover deactivated, legacy deletedAt, fresh-login, and banned cases. I did not run an end-to-end OAuth flow in this read-only review.

## Solution Assessment

Unclear. Current main's irreversible deletion behavior is internally consistent if that is the intended policy, but restoring accounts or recreating them under the same GitHub identity needs maintainer product and security approval first.
