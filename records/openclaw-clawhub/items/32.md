## Summary

Keep open. Current main still intentionally blocks OAuth reauth for self-deleted/deactivated accounts, related affected-user reports remain open, and ClawHub cleanup policy does not auto-close issues; the remaining work is maintainer product/security triage for account recovery or support behavior.

## What This Changes

The issue reports that users who self-deleted their ClawHub account can complete GitHub OAuth but return to ClawHub still signed out and cannot re-authenticate with the same GitHub account.

## Best Possible Solution

Maintainers should choose and document the deleted-account lifecycle, then either keep irreversible deletion with clear support/sign-in guidance or implement an audited same-provider recovery, unlink, purge, or re-registration flow with regression coverage.

## Reproduction Assessment

Yes, from source inspection: `deleteAccount` writes `deactivatedAt` and `purgedAt`, then OAuth completion calls `handleDeletedUserSignIn` and rejects that user. I did not run a live GitHub OAuth browser flow.

## Solution Assessment

No for an automated source patch. Restoring, purging, unlinking, or recreating accounts after deletion changes the current irreversible-deletion contract and needs maintainer product/security approval before code changes.
