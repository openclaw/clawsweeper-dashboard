## Summary

Keep open. This is an open ClawHub issue, and current main still has a source-reproducible path where ignore-filtered file discovery can omit an existing root manifest before both CLI and backend publish validation; #1856 is an open draft fix candidate, not a merged implementation.

## What This Changes

The issue asks for `clawhub publish` to accept a valid new skill folder when `SKILL.md` exists instead of failing with `SKILL.md required`.

## Best Possible Solution

Review, land, or supersede #1856 with a CLI fix that always includes an existing root `SKILL.md` or `SKILLS.md` manifest during publish while preserving ignore behavior for non-required files, then close this issue after merge and release.

## Reproduction Assessment

Yes. Source inspection gives a deterministic path: a broad ignore rule such as `*.md` can filter out a present root `SKILL.md`, after which `cmdPublish` throws `SKILL.md required`; I did not execute it because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable fix is CLI-side recovery of an existing required root manifest after normal ignore-filtered discovery, while keeping backend manifest validation intact.
