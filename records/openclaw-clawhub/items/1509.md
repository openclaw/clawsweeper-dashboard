## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has a source-reproducible CLI path where `publish .` can resolve under a configured global workdir instead of the caller's skill folder; the broad-ignore manifest omission has been fixed, but that does not cover this remaining path.

## What This Changes

The issue reports `clawhub publish` failing with `SKILL.md required` for a new skill even though the local skill folder contains a valid `SKILL.md`.

## Best Possible Solution

Skill publish should resolve user-entered local relative source paths to the caller's intended skill folder when that folder contains `SKILL.md` or `SKILLS.md`, while preserving absolute paths, explicit configured-workdir flows, and the existing broad-ignore manifest recovery.

## Reproduction Assessment

Yes, from source inspection. With a configured Clawdbot/OpenClaw default workspace different from an unmarked cwd, `clawhub skill publish .` can inspect the wrong folder and hit `SKILL.md required` even when the caller's cwd contains `SKILL.md`.

## Solution Assessment

Yes. The narrow maintainable repair is publish source-path resolution plus regression coverage, not backend validation changes or a broader rewrite of global workdir behavior.
