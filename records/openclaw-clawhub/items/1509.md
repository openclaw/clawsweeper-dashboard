## Summary

Keep open. The broad-ignore manifest omission is fixed on current main and noted in v0.12.3, but current main still has a source-reproducible `publish .` path where a configured global workdir can differ from the caller's skill folder, and related https://github.com/openclaw/clawhub/issues/48 remains open.

## What This Changes

The issue reports that `clawhub publish .` fails with `SKILL.md required` for new skills even when the skill folder contains a valid root manifest.

## Best Possible Solution

Skill publish should resolve a user-entered relative source path against the caller's current directory when that resolves to an existing skill folder, while preserving absolute paths and intentional configured-workdir flows, with regression coverage and a changelog entry.

## Reproduction Assessment

Yes, from source inspection. With a configured Clawdbot/OpenClaw default workspace different from cwd and no cwd marker, `clawhub skill publish .` can scan the configured workspace and hit `SKILL.md required` even when cwd contains `SKILL.md`.

## Solution Assessment

Yes. The broad-ignore manifest bug is already fixed in v0.12.3; the remaining narrow fix is source-path resolution for skill publish rather than changing backend validation or broad ignore semantics.
