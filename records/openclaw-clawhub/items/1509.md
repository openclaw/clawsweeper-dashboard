## Summary

Keep open. Current main still validates the publish payload after ignore-filtered file discovery, so a present root manifest can be omitted before both CLI and backend checks; #1856 is an open draft candidate rather than a merged fix.

## What This Changes

Fix `clawhub publish` so new skill publishing does not fail with `SKILL.md required` when a valid root manifest exists.

## Best Possible Solution

Review, land, or supersede #1856 with CLI recovery of an existing root `SKILL.md` or `SKILLS.md` while preserving normal ignore behavior, then close after the fix is merged and available.

## Reproduction Assessment

Yes. Source inspection shows a broad ignore rule such as `*.md` is loaded before the publish manifest check, so a present root `SKILL.md` can be filtered out; I did not execute the reporter's Windows/npm setup.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable fix because it restores only an existing required root manifest instead of bypassing ignore rules for arbitrary files.
