## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has a source-reproducible path where `publish .` can inspect the resolved global workdir instead of the caller's skill folder; the broad-ignore manifest case was superseded by https://github.com/openclaw/clawhub/pull/1856 and landed in `clawhub@0.12.3`, but the remaining relative-path behavior is still actionable.

## What This Changes

The issue reports that `clawhub publish .` fails with `SKILL.md required` for new skills even when the root manifest exists.

## Best Possible Solution

Make skill publish resolve user-entered relative source paths to the caller's intended skill folder when that folder contains a root manifest, while preserving absolute paths and existing configured-workdir behavior outside publish.

## Reproduction Assessment

Yes, from source inspection rather than a live CLI run. With a configured Clawdbot/OpenClaw default workspace different from an unmarked cwd, `clawhub skill publish .` can inspect the wrong folder and hit `SKILL.md required` even when cwd contains `SKILL.md`.

## Solution Assessment

Yes. The narrow maintainable repair is publish-specific source-path resolution plus regression coverage, not backend validation changes or a broad rewrite of global workdir behavior.

## Repair Work Prompt

Fix the remaining `clawhub publish .` / `clawhub skill publish .` path-resolution bug for https://github.com/openclaw/clawhub/issues/1509 and https://github.com/openclaw/clawhub/issues/48. In `packages/clawhub/src/cli/commands/publish.ts`, resolve skill publish source paths so an absolute `folderArg` is preserved, and a relative `folderArg` prefers the caller's current directory when `process.cwd()` plus that relative path exists and looks like a skill folder with root `SKILL.md` or `SKILLS.md`; otherwise keep the existing `opts.workdir` behavior for configured workspace flows. Do not change backend publish validation, `listTextFiles` ignore semantics, package publish, install/update/sync semantics, or the already-landed root-manifest recovery. Add regression coverage in `packages/clawhub/src/cli/commands/publish.test.ts` for `cmdPublish(makeOpts(otherWorkdir), '.', ...)` while cwd is a skill folder and the configured workdir points elsewhere, verifying the uploaded form includes the cwd `SKILL.md`. Include a similar relative subfolder case if the helper supports it. Add/update focused docs or specs only if the path precedence rule becomes new durable intent beyond the CLI implementation.
