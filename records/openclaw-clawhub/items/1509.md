## Summary

Keep open. The broad-ignore manifest omission is fixed on current main, but source inspection still shows a credible `publish .` path where the CLI can resolve against a configured global workspace instead of the caller’s skill folder and then emit `SKILL.md required`.

## What This Changes

The issue reports that `clawhub publish` fails with `SKILL.md required` for new skill publishes even when the local folder contains a valid root `SKILL.md`.

## Best Possible Solution

The CLI should publish an explicitly supplied local skill folder from the caller’s cwd, including `.`, even when the global workdir falls back to a configured workspace, while keeping the shipped root-manifest recovery and backend validation unchanged.

## Reproduction Assessment

Yes, from source inspection. With a configured Clawdbot/OpenClaw default workspace different from an unmarked cwd, `clawhub skill publish .` can scan the configured workspace and hit `SKILL.md required` even when the cwd contains `SKILL.md`.

## Solution Assessment

Yes. The narrow maintainable fix is publish source-path resolution plus regression coverage for a mismatched configured workspace, not backend validation changes or a rollback of the broad-ignore manifest recovery.

## Repair Work Prompt

Fix the remaining `clawhub publish .` / `clawhub skill publish .` source-path bug for skill publishing. Current main already re-adds an ignored root `SKILL.md` / `SKILLS.md`; keep that behavior. The remaining problem is that `cmdPublish` resolves relative source paths with `resolve(opts.workdir, folderArg)`, while `resolveGlobalOpts` may set `opts.workdir` to a configured Clawdbot/OpenClaw workspace when the caller’s cwd has no `.clawhub` marker. Make explicitly supplied local publish paths, especially `.`, resolve against the caller’s cwd when they point at a local skill folder, while preserving absolute paths and explicit/env `--workdir` flows. Add regression coverage for a mismatched configured workspace where cwd contains `SKILL.md` but the configured workspace does not. Do not change backend publish validation, package publish behavior, sync semantics, or the shipped broad-ignore root-manifest recovery. Include a concise changelog/spec note only if repo policy requires it for this user-facing CLI fix.
