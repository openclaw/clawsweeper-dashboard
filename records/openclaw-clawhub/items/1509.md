## Summary

Keep open. Current main still has a source-reproducible publish path where a present root manifest can be omitted by ignore-filtered file discovery before CLI/backend validation, and the linked draft fix candidate is open and unmerged.

## What This Changes

Investigate `clawhub publish` failing with `SKILL.md required` for a new skill even though the root manifest exists.

## Best Possible Solution

Review, land, or supersede #1856 with a narrow CLI recovery for an existing root `SKILL.md` or `SKILLS.md` while preserving normal ignore behavior for non-required files, then close after the fix is merged and available.

## Reproduction Assessment

Yes, from source inspection. A skill folder with a root `SKILL.md` plus a broad ignore rule such as `*.md` can omit the manifest from `listTextFiles`, after which `cmdPublish` fails before upload.

## Solution Assessment

Yes. Recovering only an existing required root manifest is the narrow maintainable fix because it avoids bypassing ignore rules for arbitrary supporting files, and #1856 already targets that path.
