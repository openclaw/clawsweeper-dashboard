## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has a source-reproducible CLI path where `publish .` can resolve the source under a configured global workdir instead of the caller's skill folder; the earlier broad-ignore manifest omission has been fixed, but this remaining path is not covered by the closed PR.

## What This Changes

The issue reports that `clawhub publish` fails with `SKILL.md required` for new skills even when the root `SKILL.md` exists.

## Best Possible Solution

Skill publish should resolve user-entered relative source paths to the caller's intended skill folder when that folder contains a root manifest, while preserving absolute paths and existing configured-workdir behavior outside publish.

## Reproduction Assessment

Yes, from source inspection. With a configured Clawdbot/OpenClaw default workspace different from an unmarked cwd, `clawhub skill publish .` can inspect the wrong folder and hit `SKILL.md required` even when the caller's cwd contains `SKILL.md`.

## Solution Assessment

Yes. The narrow maintainable repair is publish-specific source-path resolution plus regression coverage, not backend validation changes or a broad rewrite of global workdir behavior.
