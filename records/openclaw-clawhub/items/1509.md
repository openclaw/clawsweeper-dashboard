## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has a source-reproducible publish path where an existing root manifest can be omitted before validation; the linked fix candidate is open and unmerged.

## What This Changes

The issue requests that `clawhub publish` successfully publish a new skill when a valid root `SKILL.md` exists instead of failing with `SKILL.md required`.

## Best Possible Solution

Review, land, or supersede #1856 with CLI recovery of an existing root `SKILL.md` or `SKILLS.md` while preserving normal ignore behavior, then close after the fix is merged and available.

## Reproduction Assessment

Yes. Source inspection shows a skill folder with a root `SKILL.md` plus a broad ignore rule such as `*.md` can cause `listTextFiles` to omit the manifest before `cmdPublish` fails; I did not run a filesystem reproduction because this review is read-only.

## Solution Assessment

Yes. Recovering only an existing required root manifest is the narrow maintainable fix because it avoids bypassing ignore rules for arbitrary files, and #1856 already targets that path.
