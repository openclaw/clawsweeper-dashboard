Keep open. This is an open ClawHub issue, and the repository cleanup policy for this repo only permits automated closure for PRs that current main definitely implements. Current main still has a credible failure path where ignore-aware file discovery can omit a present root manifest before publish validation, and the linked draft fix PR is open rather than merged.

## What This Changes

The issue reports that `clawhub publish` fails with `SKILL.md required` for new skills even when a valid root `SKILL.md` exists.

## Best Possible Solution

Review, land, or supersede PR #1856 with a CLI fix that always includes an existing root `SKILL.md` or `SKILLS.md` manifest during skill publish while preserving ignore behavior for non-required files, then close this issue after the fix is merged and available.
