## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has a source-reproducible CLI path where `skill publish .` can resolve against a configured global workdir instead of the caller's skill folder.

## What This Changes

The issue reports that `clawhub publish` fails with `SKILL.md required` for new skill folders even when `SKILL.md` exists.

## Best Possible Solution

Skill publish should treat user-supplied relative source paths as cwd-relative when they point at a skill folder, keep absolute and configured-workdir flows stable, and add regression coverage for a mismatched configured workspace.

## Reproduction Assessment

Yes, from source inspection. With a configured Clawdbot/OpenClaw default workspace different from an unmarked cwd, `clawhub skill publish .` can scan the configured workspace and hit `SKILL.md required` even when the cwd contains `SKILL.md`.

## Solution Assessment

Yes. The narrow maintainable repair is publish source-path resolution plus regression coverage; backend validation and the already-shipped broad-ignore manifest recovery should not be broadened for this remaining path.
