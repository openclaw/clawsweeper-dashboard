## Summary

Keep open. Current main still has a source-reproducible publish path where ignore-filtered file discovery can omit an existing root manifest before CLI/backend validation, and #1856 is an open draft fix candidate linked to this issue rather than a merged implementation.

## What This Changes

The issue reports that `clawhub publish` fails with `SKILL.md required` for new skill folders even when a valid root manifest exists.

## Best Possible Solution

Review, land, or supersede #1856 with narrow CLI recovery of an existing root `SKILL.md` or `SKILLS.md` while preserving ignore behavior for non-required files, then close after the fix merges and ships.

## Reproduction Assessment

Yes. Source inspection shows a skill folder with root `SKILL.md` plus a broad ignore rule such as `*.md` can omit the manifest from `listTextFiles`, after which `cmdPublish` fails before upload; I did not execute the reporter's Windows/npm setup because this review is read-only.

## Solution Assessment

Yes. Recovering only an existing required root manifest is the narrow maintainable fix because it avoids bypassing ignore rules for arbitrary files, and #1856 already targets that path.
