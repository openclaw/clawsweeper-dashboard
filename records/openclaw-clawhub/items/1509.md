## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has a source-reproducible publish path where a present root manifest can be filtered out before the CLI and backend manifest checks; #1856 is an open draft fix candidate, not merged main behavior.

## What This Changes

The issue asks for `clawhub publish` to stop failing with `SKILL.md required` when a valid new skill folder contains a root manifest.

## Best Possible Solution

Review, land, or supersede #1856 with CLI recovery of an existing root `SKILL.md` or `SKILLS.md` while preserving normal ignore behavior, then close after the fix is merged and available.

## Reproduction Assessment

Yes. Source inspection shows a skill folder with a root `SKILL.md` plus a broad ignore rule such as `*.md` can cause `listTextFiles` to omit the manifest before `cmdPublish` fails; I did not execute the Windows/npm setup.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable fix because it restores only an existing required root manifest instead of bypassing ignore rules for arbitrary files.
