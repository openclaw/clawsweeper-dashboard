## Summary

Keep open. Current main still validates only the files returned by ignore-filtered discovery, so a present root manifest can plausibly be omitted before publish validation, and the linked draft PR remains unmerged.

## What This Changes

The issue reports that `clawhub publish` rejects new skill folders with `SKILL.md required` even when a valid root `SKILL.md` exists.

## Best Possible Solution

Land or supersede #1856 with a CLI fix that preserves an existing root `SKILL.md` or `SKILLS.md` during publish while keeping ignore behavior for other files, then close this issue after merge and release.

## Reproduction Assessment

Partly yes: current-main source gives a deterministic path where a broad ignore rule such as `*.md` can remove `SKILL.md` from `listTextFiles` before `cmdPublish` validates files. The exact Windows setup remains unclear because the report does not include ignore-file contents.

## Solution Assessment

Yes, the linked PR direction appears to be the narrow maintainable fix: recover only the required root manifest after normal discovery instead of weakening ignore handling globally.
