## Summary

Keep open. Current main still has a source-reproducible publish path where an existing root manifest can be omitted by ignore-filtered file discovery before CLI/backend validation, and the linked fix candidate #1856 is open, draft, and unmerged.

## What This Changes

The issue reports that `clawhub publish` rejects new skills with `SKILL.md required` even when a root `SKILL.md` exists.

## Best Possible Solution

Review, land, or supersede #1856 with CLI recovery of an existing root `SKILL.md` or `SKILLS.md` while preserving normal ignore behavior, then close after the fix is merged and available.

## Reproduction Assessment

Yes. Source inspection shows a skill folder with root `SKILL.md` plus a broad ignore rule such as `*.md` can omit the manifest from `listTextFiles`, after which `cmdPublish` fails before upload; I did not execute the reporter's Windows/npm setup.

## Solution Assessment

Yes. Recovering only an existing required root manifest is the narrow maintainable fix because it avoids bypassing ignore rules for arbitrary files, and #1856 already targets that path.
