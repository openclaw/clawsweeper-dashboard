## Summary

Keep open. The broad-ignore `SKILL.md` omission has been fixed on current main and released, but the issue still has a source-reproducible CLI path where `publish .` can resolve against a configured global workdir instead of the caller's skill folder.

## What This Changes

The issue reports that `clawhub publish .` fails with `SKILL.md required` for a new skill even when the local `SKILL.md` exists.

## Best Possible Solution

Skill publish should resolve user-supplied relative source paths against the caller's cwd when they point at a local skill folder, preserve absolute and configured-workdir flows, and add regression coverage for a mismatched configured workspace.

## Reproduction Assessment

Yes, from source inspection. With a configured Clawdbot/OpenClaw default workspace different from an unmarked cwd, `clawhub skill publish .` can scan the configured workspace and hit `SKILL.md required` even when the cwd contains `SKILL.md`.

## Solution Assessment

Yes. The broad-ignore manifest recovery already on main should stay; the remaining narrow repair is publish source-path resolution plus regression coverage, not backend validation changes.
