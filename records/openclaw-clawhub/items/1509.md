## Summary

Keep open. Current main still has a source-reproducible `clawhub publish` path where a present root manifest can be filtered out before validation, and #1856 is an open draft fix candidate rather than merged main behavior.

## What This Changes

The issue asks for a fix or workaround when `clawhub publish` reports `SKILL.md required` for a new skill even though the root manifest exists.

## Best Possible Solution

Review, land, or supersede #1856 with CLI recovery of an existing root `SKILL.md` or `SKILLS.md` while preserving normal ignore behavior, then close after the fix is merged and available.

## Reproduction Assessment

Yes for a current-main source path: a skill folder with a root `SKILL.md` plus a broad ignore rule such as `*.md` can cause `listTextFiles` to omit the manifest before `cmdPublish` fails. I did not execute the reporter's exact Windows/npm setup, so the precise environment-specific trigger remains unconfirmed.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable fix because it restores only an existing required root manifest instead of bypassing ignore rules for arbitrary files.
