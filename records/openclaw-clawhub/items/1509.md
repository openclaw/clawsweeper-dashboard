## Summary

Keep open. Current main still validates `clawhub publish` against the ignore-filtered file list, so a present root manifest can be dropped before the CLI and backend `SKILL.md` checks; the linked fix candidate #1856 is open and unmerged, and ClawHub cleanup policy keeps issues open.

## What This Changes

Publishing a new skill can fail with `SKILL.md required` even when the root manifest exists because ignored file discovery may omit it before publish validation.

## Best Possible Solution

Review, land, or supersede #1856 with a focused CLI fallback that includes an existing root `SKILL.md` or `SKILLS.md` during publish while preserving normal ignore behavior for other files.

## Reproduction Assessment

Yes. Source inspection shows a broad `.gitignore` or `.clawhubignore` rule such as `*.md` can remove a present root `SKILL.md` before `cmdPublish` validates the manifest; I did not run the reporter's Windows/npm setup.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable fix because it restores only an existing required root manifest instead of bypassing ignore rules for arbitrary files.
