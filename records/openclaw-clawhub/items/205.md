## Summary

Keep open. ClawHub cleanup policy does not auto-close open issues, and the reported condition is still observable: the public skill detail has no skill-level moderation warning, but the public scan endpoint still reports the latest arbitrum-dapp-skill version as suspicious from the GoatCounter install-count analysis. Current main has manual override and limited VT Code Insight calibration paths, but not a maintainer-approved policy that downgrades transparent opt-out install telemetry generally.

## What This Changes

The issue requests that documented, opt-out GoatCounter install telemetry be treated as an informational scan note rather than a suspicious finding.

## Best Possible Solution

Define a maintainer-approved scanner policy for transparent, opt-out install telemetry, then update the scan API/UI and any rescan, backfill, or staff override workflow so corroborated benign telemetry appears as informational while unknown, obfuscated, or data-transmitting install calls remain suspicious.
