## Summary

Current main only contains the earlier hosted-skill backup foundation, so this PR remains useful, but it is not merge-ready: the current head has an unresolved package-owner identity defect and lacks the repository-required real Convex runtime proof.

## What This Changes

The PR extends GitHub backups to package registry artifacts with durable retry and health state, changes the default backup repository, and adds a mandatory manual backup checkpoint to production migration guidance.

## Best Possible Solution

Centralize package backup identity in one helper used by publish, retry, and reconciliation, then prove successful backup and forced failure recovery against a real Convex deployment and the intended private GitHub repository.

## Reproduction Assessment

Yes at source level: process a handle-less user-owned package through publishing and reconciliation and compare the owner key each path emits. The resulting duplicate GitHub artifacts have not yet been demonstrated in a real deployment.

## Solution Assessment

No, not in its current form; durable asynchronous retries are a maintainable direction, but all paths need one canonical artifact identity and real success/failure recovery proof.
