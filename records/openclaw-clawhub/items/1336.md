## Summary

Keep open: ClawHub cleanup policy keeps issues open, and the reported GitHub sign-in failure is still reproducible on the production-tagged current main because the GitHub sign-in start route returns HTTP 500 instead of redirecting to GitHub.

## What This Changes

The issue reports that the “Sign in with GitHub” button appears to do nothing, possibly because GitHub OAuth Client ID or related production auth configuration is missing or wrong.

## Best Possible Solution

Restore production GitHub OAuth so the sign-in start route redirects to GitHub, active accounts receive a durable session, deleted or banned accounts remain blocked, and failed starts surface a clear error.

Yes. On 2026-05-10T12:57:55Z, curl reproduced the current production failure: the site root returned HTTP 200, but both canonical and direct Convex /api/auth/signin/github endpoints returned HTTP 500 instead of redirecting to GitHub.
