Keep open: current production still returns HTTP 500 for the GitHub sign-in start endpoint, and ClawHub cleanup policy keeps issues open rather than auto-closing them as implemented. The remaining work needs maintainer/operator auth triage, not a source-only cleanup decision.

The issue reports that clicking “Sign in with GitHub” appears to do nothing, possibly because the GitHub OAuth client ID is missing or misconfigured.

## Best Possible Solution

Restore production GitHub OAuth so the sign-in start route redirects to GitHub, the callback establishes a durable session for active accounts, blocked/deleted accounts remain blocked, and failed starts or callbacks show a clear user-facing error.
