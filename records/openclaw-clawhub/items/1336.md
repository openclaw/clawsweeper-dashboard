## Summary

Keep open: this is an open ClawHub issue and the live GitHub sign-in start route still returns HTTP 500, while current main keeps the route environment-backed through Convex Auth.

## What This Changes

The issue reports that clicking Sign in with GitHub has no visible effect, likely due to missing or incorrect GitHub OAuth client configuration.

## Best Possible Solution

Restore production GitHub OAuth end to end so the sign-in endpoint redirects to GitHub, callbacks establish sessions for active accounts, blocked accounts receive explicit errors, and targeted auth smoke coverage guards the route.

## Reproduction Assessment

Yes. A 2026-05-09 UTC smoke check reproduced the production failure: /api/auth/signin/github returned HTTP 500 on both clawhub.ai and the direct Convex site endpoint while the site root was healthy.
