## Summary

Keep open. The live GitHub auth-start endpoints still return HTTP 500 while current main routes sign-in through env-backed Convex Auth; current main now handles a resolved failed start in the header UI, but that does not prove the production OAuth/configuration failure is fixed.

## What This Changes

The issue reports that the ClawHub "Sign in with GitHub" button appears unresponsive, likely due to missing or incorrect GitHub OAuth client configuration.

## Best Possible Solution

Restore production GitHub OAuth so auth start redirects to GitHub and active-account callbacks establish web and CLI sessions; then add a narrow import-safe diagnostic or deploy/env validation if maintainers want source guardrails.
