## What This Changes

The PR adds sessionStorage-backed OAuth redirect attempt tracking, a no-code/no-error GitHub callback diagnostic, CLI device user_code preservation, and focused tests.

## Best Possible Solution

Centralize GitHub redirect marker handling across all sign-in entry points, keep CLI device OAuth exchange tolerant of missing sessionStorage when user_code and code are present, and merge only with redacted live CLI/browser proof of the diagnostic.

## Reproduction Assessment

Yes for the review findings by source inspection: the PR head fallback requires a stored marker while SkillDetailPage still signs in directly, and /cli/device OAuth code consumption depends on that best-effort marker. I did not replay the original live Convex/GitHub empty-return path in a browser.

## Solution Assessment

No as written. The diagnostic direction is reasonable, but it should cover every GitHub sign-in entry point, avoid relying solely on sessionStorage for CLI device OAuth callbacks, and include after-fix real auth proof.
