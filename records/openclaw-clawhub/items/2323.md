The PR records GitHub OAuth redirect attempts, surfaces a diagnostic when the browser returns without a code, error, or session, and preserves CLI device user codes across sign-in.

## Best Possible Solution

Centralize OAuth-attempt recording so every GitHub sign-in entry point participates, port the no-code fallback onto current main's retry/error handlers, preserve CLI device authorization, and validate normal, cancelled, failed, and device-login flows against real Convex OAuth.
