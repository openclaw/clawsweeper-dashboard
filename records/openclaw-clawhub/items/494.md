## Summary

Current main has no account-gated Hubs area or Telnyx integration, so this is a new hosted-integration request rather than a missing entitlement or UI migration. It should remain open for an explicit product and security decision because the repository provides an external plugin distribution path, while a ClawHub-hosted telephony/SMS webhook would create substantial authentication, privacy, tenant-isolation, and abuse-control responsibilities.

## What This Changes

Provide a Telnyx Hubs interface and webhook URL for authenticated real-time call and SMS routing.

## Best Possible Solution

Clarify publicly that Hubs is not a current account entitlement, then support Telnyx through an independently maintained plugin and setup guide unless maintainers explicitly approve and design a secure hosted ClawHub integration.

## Reproduction Assessment

Yes, at source level. Current main contains neither a Hubs/Telnyx route nor a Telnyx backend, so the missing navigation is expected and is not reproducible as an account-specific entitlement failure.

## Solution Assessment

No. Enabling an account or migrating its UI cannot expose a capability absent from current main; the narrow maintainable direction is an external plugin path, or an explicitly approved hosted product with a reviewed security design.
