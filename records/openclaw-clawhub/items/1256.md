## Summary

Keep open. Current main still lacks the direct normalized prefix lookup this PR proposes, and the PR remains active but stale/dirty with correctness issues that should be fixed before merge.

## What This Changes

The PR adds normalized slug/display-name digest fields and indexes, a direct lexical prefix lookup merged into skill search, a digest backfill update, and regression tests for prefix recall misses.

## Best Possible Solution

Land this PR or an equivalent patch after rebasing against current main, passing existing search filters through the direct prefix path, keeping fallback candidates in the ranked pool, handling pre-backfill slug/name/acronym cases safely, and covering Midscene, kling/klingai, ScienceClaw casing, and tagged-search behavior before closing the linked recall reports.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve capability-tag filtering in direct lookup:** `convex/search.ts:236-241`
  - body: The new direct prefix query is merged into skill search results, but this call only forwards highlightedOnly and nonSuspiciousOnly. Current main filters exact slug, vector, and fallback results by capabilityTag, so direct slug/name hits from the wrong tag can leak into tagged searches and occupy result slots. Pass capabilityTag through and apply matchesCapabilityTag inside directLexicalSkills.
  - confidence: 0.91
- **[P2] Keep fallback candidates after direct prefix matches:** `convex/search.ts:250-253`
  - body: primaryMatches now includes direct prefix candidates, and fallback is skipped once that combined list reaches limit. directLexicalSkills reads capped prefix slices from indexes before final scoring, so broad prefixes can suppress recent-digest fallback candidates that should still compete in the final ranked set. Run fallback based on exact/vector sufficiency or merge fallback before the final ranking decision.
  - confidence: 0.78
- **[P2] Cover legacy slug rows before normalized backfill:** `convex/search.ts:419-426`
  - body: normalizedSlug is optional and filled by a separate backfill, but slug-prefix recall only reads the normalized-slug indexes. Existing digest rows without normalizedSlug can still miss slug-prefix queries during rollout unless they happen to be in the bounded fallback window, so add a legacy slug-prefix source or make rollout/backfill atomic for this behavior.
  - confidence: 0.82
- **[P3] Handle acronym casing in legacy name fallback:** `convex/search.ts:75-80`
  - body: During the normalizedDisplayName backfill window, legacy display-name scans are case-sensitive and only try raw, lowercase-normalized, and title-cased prefixes. A lowercase query like gpt still will not match legacy rows whose display name starts with GPT, leaving acronym-prefixed skills dependent on the bounded fallback window.
  - confidence: 0.74
