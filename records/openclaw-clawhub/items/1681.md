## Summary

Keep open. Current main still matches the reported behavior: skill README Markdown is rendered without a relative-link resolver, while skill detail URLs and canonical URLs remain non-directory-shaped. Draft PR #1858 is open, unmerged, and explicitly linked as the implementation candidate, so this should stay open for maintainer review rather than be auto-closed.

## What This Changes

Relative links in rendered skill documentation should resolve within the viewed skill's own file namespace instead of navigating to owner-level or unrelated same-origin routes.

## Best Possible Solution

Land PR #1858 or an equivalent focused fix that resolves safe relative README links to the skill file API, rejects traversal or malformed paths, preserves external URLs and same-page anchors, and adds regression coverage for both the resolver and SkillDetailTabs rendering.
