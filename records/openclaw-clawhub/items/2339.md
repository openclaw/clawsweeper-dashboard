## Summary

Keep open: current main still has CLI/API/backend skill delete and restore support, but no owner-facing web delete/restore control or owner-deleted restore discovery surface. Because restore crosses moderation and security provenance boundaries, this needs maintainer product/security review before implementation.

## What This Changes

Request to add signed-in owner and publisher-admin web controls for whole-skill soft-delete and restore while preserving existing backend gates.

## Best Possible Solution

Add the owner web flow on top of setSkillSoftDeletedInternal, with a dedicated owner-deleted restore surface and regression coverage for owner, publisher-admin, non-owner, and moderation/security denial cases.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence in the gap because owner settings expose rename/merge but not delete/restore, and owner dashboards list only active skills.

## Solution Assessment

Yes, if maintainers accept the product behavior: reusing the existing backend lifecycle gate is the narrow maintainable direction. A separate web-only authorization path would risk drifting from the current audit, reservation, visibility, and restore-denial semantics.
