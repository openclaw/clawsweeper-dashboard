## Best Possible Solution

Complete the remaining restore UX by surfacing only eligible owner-deleted rows during the 30-day reservation window and invoking the existing backend restore path with regression coverage for protected provenance denials.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainer product/security acceptance. The delete-only implementation is a useful partial direction, but the maintainable completion is a restore discovery/control flow over the existing backend gates, not separate web-only restore semantics.
