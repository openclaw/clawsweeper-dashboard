## Summary

Current main implements the owner delete half through merged PR https://github.com/openclaw/clawhub/pull/2548, but that change was explicitly delete-only and no discoverable owner restore surface exists; meaningful requested work therefore remains, and the collaborator-authored issue requires explicit maintainer handling.

## What This Changes

Add discoverable web controls for owners and publisher admins to soft-delete and restore manageable skills while preserving existing provenance-based restore restrictions.

## Best Possible Solution

Add an eligible-deleted-skills section to the signed-in dashboard or settings flow that calls the existing guarded undelete path, shows the reservation deadline, and never offers owner restore for protected provenance.

## Reproduction Assessment

Yes, at source level: current main has a tested delete-only web action, while repository-wide inspection shows no equivalent owner-facing restore control or deleted-skill discovery path.

## Solution Assessment

Yes in architecture, but the work is incomplete: the web should reuse the existing guarded undelete operation rather than create parallel lifecycle logic, with maintainers choosing the safest discovery UX.
