## Summary

Current main implements the owner-facing delete half through merged pull request https://github.com/openclaw/clawhub/pull/2548, but the signed-in web app still has no owner-facing list or action for restoring deleted skills. The remaining restore requirement is meaningful and is not tracked by another viable issue or PR, so this collaborator-authored issue should stay open for explicit maintainer follow-up.

## What This Changes

Add discoverable web controls for owners and publisher admins to soft-delete manageable skills and restore owner-deleted skills during the 30-day slug reservation window.

## Best Possible Solution

Keep the shipped settings delete action, then add a server-authoritative owner-deleted-skills view and restore action that is shown only during the reservation window and returns clear denial copy for non-owner or protected removal provenance.

## Reproduction Assessment

Yes. Source inspection shows that owner web deletion is implemented, while application UI has no owner-facing deleted-skill listing or restore mutation call; only staff management routes expose restore controls.

## Solution Assessment

Partially. Reusing the existing server-side lifecycle is the correct approach, but the merged delete-only settings action does not satisfy the issue’s explicit owner restore and discoverability acceptance criteria.
