## Summary

Current main partially addresses the request through merged owner-facing deletion, but the distinct dashboard discovery and web restore flow for owner-deleted skills remains absent. Because the remaining work is collaborator-authored and crosses moderation and security provenance boundaries, it needs explicit maintainer product and security review.

## What This Changes

Add owner-facing web controls to soft-delete and restore manageable skills while preserving existing authorization and moderation provenance gates.

## Best Possible Solution

Add an authenticated dashboard section for restorable owner-deleted skills and a server-authorized restore operation that returns eligibility from deletion provenance, with 30-day reservation copy and denial coverage for every non-owner hide source.

## Reproduction Assessment

Yes. Current-main source shows owner-facing deletion and server-side undelete support, while a focused search of the web application finds no owner restore action or deleted-skill discovery path.

## Solution Assessment

Yes, with a security constraint. Extending the existing dashboard and settings surfaces while keeping the backend authoritative for provenance and authorization is the narrowest maintainable solution; client-side inference of restore eligibility would not be safe.
