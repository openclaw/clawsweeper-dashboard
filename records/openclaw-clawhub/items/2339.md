## Summary

Keep open: current main still lacks owner-facing web delete/restore controls, while the CLI/API and backend lifecycle gates already exist. This is a new security-sensitive owner lifecycle UI, not a cleanup-close candidate.

## What This Changes

Add owner and publisher-admin web controls to soft-delete whole skills and restore owner-deleted skills during the 30-day slug reservation window while preserving moderation/security restore gates.

## Best Possible Solution

Implement the web flow on top of the existing soft-delete mutation semantics, with a discoverable owner-deleted restore surface and regression coverage for owner, publisher-admin, non-owner, and moderation/security-denial cases.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection verifies the product gap because current web settings and dashboard paths do not expose delete or restore while the CLI/API do.

## Solution Assessment

Yes, if maintainers accept the product behavior, reusing the existing server-side soft-delete path is the narrow maintainable solution. A separate web-only delete implementation would risk drifting from the authorization, audit, slug reservation, count, and embedding logic already present in backend code and tests.
