## Summary

Current main implements owner-facing web deletion but not the requested restoration flow: deleted skills are excluded from owner dashboards, no owner-facing web restore mutation or control exists, and no open replacement PR owns this security-sensitive work.

## What This Changes

Add discoverable web controls for owners and publisher admins to soft-delete and restore manageable skills while preserving protected moderation and security restore denials.

## Best Possible Solution

Add a dedicated owner-deleted dashboard state and restore action that call the shared server authorization and audit path, while keeping every protected deletion provenance hidden from and unrestorable by owners.

## Reproduction Assessment

Yes. Current source deterministically excludes soft-deleted rows from owner dashboard pagination and exposes only a deletion mutation to the web ownership panel.

## Solution Assessment

Yes at the architectural level: the narrow maintainable solution is to reuse the shared server-side soft-delete and provenance logic, but maintainers must approve the exact deleted-skill discovery UX.
