## Summary

Keep open: current main still has CLI/API and backend soft-delete support, but owner settings and dashboard surfaces expose only rename/merge/staff management paths; this is a product/security UI feature, not cleanup-closeable.

## What This Changes

The request is to add web UI controls for owners and publisher admins to soft-delete and restore whole skill listings while preserving existing CLI/API semantics and moderation gates.

## Best Possible Solution

Implement the web flow on top of the shared soft-delete semantics, with an owner-deleted restore surface and regression coverage for owner, publisher-admin, non-owner, and moderation/security-denial cases.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence in the current product gap because the owner settings and dashboard surfaces lack delete/restore while CLI/API/backend paths exist.

## Solution Assessment

Yes, if maintainers accept the product behavior, reusing the existing server-side soft-delete semantics is the narrow maintainable solution. A separate web-only lifecycle implementation would risk drifting from authorization, audit, slug reservation, count, and embedding behavior.
