## Summary

Keep open: current main still has CLI/API and shared backend lifecycle support, but the signed-in web settings/dashboard surfaces do not expose owner delete or restore controls, and the restore flow crosses security-sensitive moderation provenance gates.

## What This Changes

Add signed-in web controls for owners and publisher admins to soft-delete whole skills and restore owner-deleted skills during the 30-day slug reservation window.

## Best Possible Solution

Implement the web flow on top of setSkillSoftDeletedInternal, with a discoverable owner-deleted restore surface and regression coverage for owner, publisher-admin, non-owner, and moderation/security-denial cases.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection provides high confidence that current web settings/dashboard lack delete and restore controls while CLI/API paths exist.

## Solution Assessment

Yes, if maintainers accept the product behavior: reusing setSkillSoftDeletedInternal is the narrow maintainable path. A separate web-only lifecycle implementation would risk drifting from authorization, audit, slug reservation, count, and embedding behavior.
