## What This Changes

Add signed-in web controls for owners and publisher admins to soft-delete and restore managed skills while preserving existing authorization, audit, slug-reservation, and moderation/security restore gates.

## Best Possible Solution

Add a dashboard/settings restore section for eligible owner-deleted skills that delegates to the existing provenance-aware backend restore path, shows reservation expiry, and tests owner restore plus protected denial cases.

## Reproduction Assessment

Yes. Source inspection shows owner delete exists in settings, while dashboard pagination only returns active skills and no owner-facing web restore control or mutation exists.

## Solution Assessment

Yes in architecture, but the exact UX policy needs maintainer approval. The narrow maintainable path is to reuse the existing backend restore gate rather than create UI-only authorization rules.
