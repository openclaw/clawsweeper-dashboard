## What This Changes

Requested owner-facing web controls for soft-deleting and restoring owned skills from the dashboard or skill settings UI.

## Best Possible Solution

Add a dashboard-visible restore surface for eligible owner-deleted skills that delegates to the existing provenance-aware backend path and clearly separates protected moderation/security removals.

## Reproduction Assessment

Yes. Source inspection shows an owner delete control and mutation, while the dashboard query filters out soft-deleted skills and the ownership panel has no restore control.

## Solution Assessment

Yes in architecture, but the exact UX needs maintainer approval. Reusing the existing provenance-aware backend lifecycle is safer and more maintainable than adding separate web authorization rules.
