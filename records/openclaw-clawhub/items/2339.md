## Summary

Keep open: this is a current, well-scoped feature request, and source inspection confirms the web app still lacks owner-facing skill delete/restore controls even though the CLI/API and backend mutation exist.

## What This Changes

Add signed-in web UI controls for owners and publisher admins to soft-delete owned skills and restore owner-deleted skills during the 30-day slug reservation window.

## Best Possible Solution

Implement the web flow on top of the existing setSkillSoftDeletedInternal path, adding a discoverable deleted-skill restore surface and regression tests for owner, publisher-admin, non-owner, and moderation/security-denial cases.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection does verify the product gap: current web settings expose rename/merge only, while the CLI/API already expose delete/undelete.

## Solution Assessment

Yes, the requested direction is the maintainable path if accepted: reuse the existing server-side authorization, audit, slug-reservation, public-count, and embedding behavior instead of creating separate web-only delete logic.
