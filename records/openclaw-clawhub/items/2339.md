## What This Changes

The issue requests owner-facing web controls to soft-delete and restore managed skills while preserving existing authorization, audit, slug-reservation, and moderation/security restore gates.

## Best Possible Solution

Implement the web flow on top of setSkillSoftDeletedInternal, with a discoverable deleted-skill restore surface and regression tests for owner, publisher-admin, non-owner, and moderation/security-denial cases.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection verifies the product gap: current web settings expose rename/merge only, while CLI/API delete and undelete already exist.

## Solution Assessment

Yes, if accepted, the requested direction is the maintainable path: reuse the existing server-side authorization, audit, slug-reservation, public-count, embedding, and restore-gate behavior instead of creating separate web-only delete logic.
