## What This Changes

The issue requests web UI controls for owners and publisher admins to soft-delete and restore managed skills while preserving authorization, audit logging, 30-day slug reservation, and moderation/security restore gates.

## Best Possible Solution

Add a dashboard or settings restore section for eligible owner-deleted skills that delegates to the existing provenance-aware backend restore path, shows reservation expiry, and keeps protected hides staff-only.

## Reproduction Assessment

Yes, at source level. Current main has a settings delete action, but dashboard queries filter out soft-deleted skills and search found no owner-facing web restore control.

## Solution Assessment

Yes in architecture, but the exact UX needs maintainer product review. The narrow maintainable path is to reuse the existing provenance-aware restore gate instead of duplicating authorization in the client.
