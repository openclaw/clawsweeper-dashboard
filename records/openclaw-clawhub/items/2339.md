## Summary

Keep open: current main already has CLI/API/backend soft-delete and restore, but the signed-in owner settings and dashboard UI still do not expose owner-facing delete or restore controls. This remains a product/security-sensitive lifecycle UI request, not conservative cleanup closure.

## What This Changes

The issue requests owner-facing web controls to soft-delete and restore managed skills through the signed-in ClawHub UI.

## Best Possible Solution

Implement the web flow on top of the existing soft-delete semantics, with a discoverable owner-deleted restore surface and regression coverage for owner, publisher-admin, non-owner, and moderation/security denial cases.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence in the product gap because owner settings expose summary, publisher note, rename, and merge but not delete/restore, while the dashboard lists only active skills.

## Solution Assessment

Yes, if maintainers accept the product behavior: reusing the existing backend soft-delete path is the narrow maintainable solution. A separate web-only lifecycle implementation would risk drifting from the established authorization, audit, slug reservation, count, and embedding behavior.
