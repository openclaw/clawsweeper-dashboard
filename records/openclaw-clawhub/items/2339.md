## Summary

Keep open: current main has the CLI/API/backend lifecycle path, but the signed-in owner web UI still has no discoverable whole-skill delete control and no owner-deleted restore surface.

## What This Changes

Request to add signed-in web UI controls that let skill owners and publisher admins soft-delete whole skills and restore owner-deleted skills during the 30-day slug reservation window.

## Best Possible Solution

Implement the web flow on top of the existing backend soft-delete mutation, with a deliberate owner-deleted restore listing and regression coverage for owner, publisher-admin, non-owner, and moderation/security denial cases.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence in the product gap because owner settings expose rename/merge but not delete/restore, and the dashboard only lists active skills.

## Solution Assessment

Yes, if maintainers accept the product behavior: reusing setSkillSoftDeletedInternal is the narrow maintainable path. A separate web-only implementation would risk drifting from authorization, audit, slug reservation, count, and embedding behavior already covered in backend code and tests.
