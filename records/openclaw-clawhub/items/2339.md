## Summary

Keep open: current main has CLI/API/backend skill delete and restore behavior, but the signed-in owner web UI still lacks discoverable whole-skill delete and restore controls, and the requested lifecycle UI needs product/security acceptance before implementation.

## What This Changes

The issue requests owner-facing web UI controls for soft-deleting and restoring manageable skills while preserving existing backend authorization and moderation/security restore gates.

## Best Possible Solution

Implement the owner web flow on top of the existing backend soft-delete semantics, with a discoverable owner-deleted restore surface and regression coverage for owner, publisher-admin, non-owner, and moderation/security denial cases.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence in the product gap because owner settings expose summary, publisher note, rename, and merge only, while dashboard queries filter out soft-deleted skills.

## Solution Assessment

Yes if maintainers accept the product behavior: reusing the existing backend lifecycle path is the narrow maintainable solution. A separate web-only implementation would risk drifting from the authorization, audit, slug reservation, count, and embedding behavior already present in backend code and tests.
