## Summary

Keep this issue open. The merged work in https://github.com/openclaw/clawhub/pull/2548 added owner-facing web deletion and the 30-day reservation messaging, but current main still excludes soft-deleted skills from the dashboard and provides no owner-facing web restore path. The remaining restoration work also needs explicit maintainer product and security review because this collaborator-authored request crosses moderation provenance boundaries.

## What This Changes

Add discoverable web controls for owners and publisher admins to soft-delete manageable skills and restore eligible owner-deleted skills during the slug-reservation window.

## Best Possible Solution

Retain the shipped deletion flow and add an authenticated deleted-skills section that exposes only restorable owner deletions, shows the reservation deadline, and invokes the existing server-side undelete policy rather than reproducing eligibility logic in the client.

## Reproduction Assessment

Yes at source level: current main visibly implements owner deletion, while its dashboard queries exclude soft-deleted skills and no owner-facing web restoration call exists.

## Solution Assessment

Yes in architectural direction: the narrowest maintainable solution is to extend the authenticated management UI over the existing provenance-aware undelete policy, with maintainers first deciding which deleted records may be disclosed in the dashboard.
