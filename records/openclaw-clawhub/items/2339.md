## Summary

Current main now provides owner-facing web deletion, but it does not provide the requested discoverable restore workflow. Because the remaining restore capability crosses moderation and security provenance boundaries—and this is a collaborator-authored product item—it needs explicit maintainer handling.

## What This Changes

Add web controls for authorized skill owners and publisher admins to soft-delete and restore listings while preserving moderation and security restrictions.

## Best Possible Solution

Add a dashboard section for eligible owner-deleted skills and a narrow restore mutation that delegates to the existing provenance-aware actor path, showing reservation expiry and clear denial states without exposing protected listings.

## Reproduction Assessment

Yes. Current main exposes and tests owner deletion, while the dashboard and ownership UI contain no restore control and the web-facing owner mutation only performs deletion.

## Solution Assessment

Yes in architecture, but the exact UX needs maintainer approval. Reusing the existing provenance-aware backend lifecycle is safer and more maintainable than adding separate web authorization rules.
