## Summary

Keep this open: current main implements the owner delete UI, but the owner-visible restore path is still missing, and this collaborator-authored, security-sensitive restore boundary needs maintainer direction.

## What This Changes

Add owner-facing web controls so owners and publisher admins can soft-delete whole skills and restore owner-deleted skills during the 30-day reservation window.

## Best Possible Solution

Ship an owner restore surface for eligible owner-deleted skills that delegates to the provenance-aware backend restore gate, shows reservation expiry, and keeps protected hides staff-only.

## Reproduction Assessment

Yes, at source level. Current main renders owner delete controls and a delete-only mutation, while dashboard owner queries filter to active skills and source search found no owner-facing web restore control.

## Solution Assessment

Yes in architecture, but current main is incomplete. The narrow maintainable solution is to add a restorable-skills web surface that reuses the existing server-side restore gate instead of duplicating authorization in the client.
