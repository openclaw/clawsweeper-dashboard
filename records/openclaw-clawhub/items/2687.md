## What This Changes

The issue asks why ClawHub changed the visible skill metric from downloads to installs and why downloads are no longer shown first.

## Best Possible Solution

Maintainers should decide the public metric hierarchy, then either land a narrow dashboard/card/profile clarification or explicitly defer to the broader downloads-first rollout in https://github.com/openclaw/clawhub/pull/2730.

## Reproduction Assessment

Yes, at source level: current main renders the dashboard first skill stat as Installs and has tests asserting profile cards show installs instead of downloads. I did not run a browser because this read-only review found the behavior directly in the current UI source and tests.

## Solution Assessment

Unclear until maintainers choose the metric hierarchy. A narrow UI change is feasible, but it would revise the recent installs-primary policy unless downloads are accepted as secondary or replacement public context.
