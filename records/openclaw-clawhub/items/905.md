## Summary

Keep open. Current main can store and publicly suppress a soft-deleted individual skill version, but the owner-facing CLI, HTTP API, docs, and management workflow still operate at whole-skill scope, and ClawHub issue policy keeps issues open for maintainer handling.

## What This Changes

The issue requests owner-facing controls to hide, delete, unhide, or undelete one published skill version while leaving the skill itself live.

## Best Possible Solution

Define the version-scoped lifecycle policy, then add authenticated owner/org-member/staff controls that update skillVersions.softDeletedAt with version-specific audit logs and consistent public routing, tag/latest, download/cache, restore, docs, and tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence in the gap because version soft-delete state and public gates exist while exposed owner lifecycle controls remain skill-scoped.

## Solution Assessment

Unclear until maintainers settle policy. A version-scoped API plus CLI is directionally appropriate, but the maintainable solution must define latest, tag, download, cache, restore, permission, and audit semantics first.
