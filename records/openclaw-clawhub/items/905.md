## Summary

Keep open. Current main still has the internal state and public gates for soft-deleted skill versions, but it does not expose an owner-facing per-version hide/delete/restore workflow, and ClawHub review policy keeps issues open for maintainer handling.

## What This Changes

Request owner-facing CLI/API or UI support to hide, delete, unhide, or undelete one published skill version while keeping the skill live.

## Best Possible Solution

Define a version-scoped lifecycle policy, then add authenticated owner/org-member/staff controls that update skillVersions.softDeletedAt, write version-specific audit logs, and keep public list/detail/file/download/tag/latest behavior consistent.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence in the gap because current main models and gates soft-deleted versions but exposes only skill-scoped owner lifecycle controls.

## Solution Assessment

Unclear until maintainers settle lifecycle policy. A version-scoped owner API plus CLI is directionally right, but latest-version, tag, cache, download, restore, permission, and audit semantics should be decided first.
