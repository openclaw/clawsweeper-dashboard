## Summary

Keep open. Current main still lacks an owner-facing per-version hide/delete/restore workflow, and the ClawHub cleanup policy keeps issues open for maintainer handling even though related reports describe the same lifecycle gap.

## What This Changes

Request owner-facing CLI or API controls to hide, delete, unhide, or undelete one published skill version while leaving the skill live.

## Best Possible Solution

Define a version-scoped lifecycle policy, then add authenticated owner/org-member/staff controls that patch skillVersions.softDeletedAt, write version-specific audit logs, and keep public list, detail, file, download, latest/tag, cache, docs, and restore behavior consistent.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence in the gap because current main models hidden versions and hides them from readers, but exposes only skill-scoped owner lifecycle controls.

## Solution Assessment

Unclear until maintainers define lifecycle policy. A version-scoped owner API plus CLI is directionally right, but the maintainable fix needs explicit latest-version, tag, cache, download, permission, restore, and audit semantics first.
