## Summary

Keep this issue open. The request is valid and still unresolved on current main: ClawHub can store and hide soft-deleted individual versions internally, but owner-facing CLI, HTTP API, docs, and lifecycle controls remain whole-skill scoped.

## What This Changes

Request owner-facing controls to hide, delete, unhide, or undelete one published skill version without deleting the whole skill.

## Best Possible Solution

Define the version-scoped lifecycle policy first, then add authenticated owner/org-member/staff controls that update skillVersions.softDeletedAt with version-specific audit logs and consistent public routing, downloads, tags, docs, and tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence in the gap because version soft-delete state and public gates exist while owner-facing per-version lifecycle controls are absent.

## Solution Assessment

Unclear until maintainers settle policy. A version-scoped owner API plus CLI is directionally appropriate, but the maintainable solution must define latest, tag, download, cache, restore, permission, and audit semantics first.
