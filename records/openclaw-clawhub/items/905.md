## Summary

Keep open. Current main has moderation-only per-version soft deletion and consistently hides those versions from public history and file access, but it still lacks the requested owner-facing API, CLI, or UI workflow; collaborator guidance also designates this as the master issue for the unresolved lifecycle policy.

## What This Changes

Add owner-facing controls to withdraw and restore one published skill version without taking the skill or its other versions offline.

## Best Possible Solution

Generalize the existing moderation quarantine behavior into an audited, reversible owner withdrawal flow with explicit latest/tag rules and consistent enforcement across every public artifact path.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and CLI documentation nevertheless establish the gap with high confidence: internal moderation tombstones exist, but owner lifecycle commands remain whole-skill only.

## Solution Assessment

The requested capability is directionally correct, but reversible audited withdrawal is safer than owner hard deletion and must cover latest pointers, tags, direct access, caches, restoration, and permissions.
