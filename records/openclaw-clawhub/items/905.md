## Summary

Keep open. Current main can model and publicly suppress soft-deleted individual skill versions, but the owner-facing lifecycle surface is still skill-scoped; the reusable policy request remains valid and ClawHub issue cleanup policy keeps issues open for maintainer handling.

## What This Changes

Requests owner-facing CLI or API controls to hide, delete, unhide, or undelete one published skill version without deleting the whole skill.

## Best Possible Solution

Define version-scoped lifecycle policy, then add authenticated owner/org-member/staff controls that update skillVersions.softDeletedAt, write version-specific audit logs, and keep public routes, tags/latest behavior, downloads/caches, docs, and tests consistent.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence in the gap because version soft-delete state and public gates exist while owner-facing per-version lifecycle controls are absent.

## Solution Assessment

Unclear until maintainers settle lifecycle policy. A version-scoped owner API plus CLI is directionally appropriate, but the maintainable solution must define latest, tag, download, cache, restore, permission, and audit semantics first.
