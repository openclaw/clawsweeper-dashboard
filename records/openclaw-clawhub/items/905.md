## Summary

Keep open. Current main can store and publicly suppress soft-deleted skill versions, but the requested owner-facing per-version CLI/API/UI workflow is still missing and needs product/security policy before implementation.

## What This Changes

Allow ClawHub owners to hide, delete, restore, or undelete one published skill version without deleting the whole skill.

## Best Possible Solution

Define a version-scoped lifecycle policy, then add authenticated owner/org-member/staff controls that update skillVersions.softDeletedAt, write version-specific audit logs, and keep public listings, detail/file/download routes, tags/latest, cache behavior, docs, and tests consistent.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence in the current gap because version soft-delete state and public gates exist, while owner-facing per-version lifecycle controls are absent.

## Solution Assessment

Unclear until maintainers settle lifecycle policy. The requested owner-facing version workflow is directionally right, but the maintainable solution needs explicit semantics for latest versions, tags, downloads, caches, restore permissions, and audit evidence.
