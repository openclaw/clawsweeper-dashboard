## Summary

Keep open. Current main still lacks an owner-facing way to hide, delete, unhide, or restore one published skill version while leaving the skill live, and the ClawHub review policy keeps issues open for maintainer handling.

## What This Changes

Request an owner-facing CLI or HTTP API workflow to hide, delete, unhide, or restore an individual published skill version without deleting the whole skill.

## Best Possible Solution

Define a version-scoped lifecycle policy, then add authenticated owner/org-member/staff controls that update skillVersions.softDeletedAt with audit logs and consistent list/detail/file/download/tag/cache/restore behavior.

## Reproduction Assessment

Not applicable as a runtime bug. Source inspection gives a high-confidence gap: current CLI/API/docs expose read-only version endpoints and skill-scoped delete/undelete, not owner-facing version lifecycle controls.

## Solution Assessment

Unclear until maintainer policy is set. The requested direction is reasonable, but exposing the existing staff-only mutation directly would be too thin without explicit latest-version, tag, cache, restore, and audit semantics.
