## Summary

Keep open. Current main can internally soft-delete an individual skill version and public readers honor that state, but the requested owner-facing per-version CLI/API/UI/docs workflow is still missing, and ClawHub issue cleanup policy keeps issues open for maintainer handling.

## What This Changes

Add an owner-facing way to hide, delete, unhide, or restore one published skill version without deleting the whole skill.

## Best Possible Solution

Define a version-scoped lifecycle policy, then add authenticated owner/org-member/staff controls that update `skillVersions.softDeletedAt`, preserve audit history, and keep version lists, detail/file/download routes, tags/latest behavior, docs, and UI consistent.

## Reproduction Assessment

Not applicable as a runtime bug. Source verification is high-confidence: current main exposes skill-level lifecycle commands/routes and read-only version endpoints, while the only version deletion mutation is internal and staff-gated.

## Solution Assessment

Unclear until maintainer policy is set. The requested owner-facing workflow is directionally right, but the maintainable path should be a policy-backed lifecycle feature rather than a thin public wrapper around the internal staff mutation.
