## Summary

Keep open. This is the canonical open feature request for owner-facing single-version hide/delete/restore on skills; current main can model and publicly gate soft-deleted versions, but exposed owner workflows remain whole-skill scoped, and ClawHub issue cleanup policy does not auto-close issues.

## What This Changes

Add owner-facing CLI/API/UI support to hide, delete, unhide, or restore one published skill version while keeping the skill live.

## Best Possible Solution

Define the version-scoped lifecycle policy, then implement authenticated owner/org-member/staff controls that update `skillVersions.softDeletedAt`, write version-specific audit logs, and keep version lists, detail/file/download endpoints, tags/latest, caches, docs, and tests consistent.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence in the gap because current main models and gates soft-deleted versions but exposes owner lifecycle controls only at the whole-skill level.

## Solution Assessment

Unclear until maintainers settle lifecycle policy. The requested owner-facing per-version workflow is directionally right, but the maintainable implementation must decide latest, tags, cache, download, restore, permission, and audit semantics first.
