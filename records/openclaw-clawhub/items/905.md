## Summary

Keep open. Current main still lacks an owner-facing per-version hide/delete/restore workflow, and the ClawHub review policy keeps issues open for maintainer handling rather than auto-closing them.

## What This Changes

Request owner-facing CLI/API controls to hide, delete, unhide, or restore a single published skill version without deleting the whole skill.

## Best Possible Solution

Define a version-scoped lifecycle policy, then add authenticated owner/org-member/staff controls that update skillVersions.softDeletedAt with audit logs and consistent public access behavior.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence in the gap because current main models hidden versions and filters them from public access, but exposes only skill-scoped owner lifecycle controls.

## Solution Assessment

Unclear until maintainers define lifecycle policy. A version-scoped owner API plus CLI is directionally right, but latest-version, tag, cache, download, restore, permission, and audit semantics should be settled first.
