## Summary

Keep open. Current main can store and publicly suppress a soft-deleted individual skill version, but the exposed owner CLI/API/docs still only hide, delete, restore, or undelete an entire skill, and ClawHub issue policy keeps this feature request open for maintainer handling.

## What This Changes

Add owner-facing controls to hide, delete, unhide, or undelete one published skill version without deleting the whole skill.

## Best Possible Solution

Define the version-scoped lifecycle policy first, then add authenticated owner/org-member/staff controls that update skillVersions.softDeletedAt with version-specific audit logs and consistent public routing, tags, downloads, caches, docs, and tests.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence in the current gap because version soft-delete state and public gates exist while owner-facing per-version lifecycle controls are absent.

## Solution Assessment

Unclear until maintainers settle policy. A version-scoped owner API plus CLI is directionally appropriate, but the maintainable solution must define latest, tag, download, cache, restore, permission, and audit semantics first.
