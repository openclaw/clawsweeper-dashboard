## Summary

Keep open. Current main still lacks an owner-facing per-version hide/delete/restore workflow, and ClawHub cleanup policy keeps issues open for maintainer handling; the request also needs product and security policy before implementation.

## What This Changes

The issue requests an owner-facing way to hide, delete, restore, or undelete a single published skill version while leaving the skill live.

## Best Possible Solution

Define a version-scoped lifecycle policy, then add authenticated owner/org-member/staff controls that update skillVersions.softDeletedAt, write version-specific audit logs, and keep public lists, detail/file/scan/download, tag/latest, docs, and restore behavior consistent.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence in the gap because current main models and gates soft-deleted versions but exposes only skill-scoped owner lifecycle controls.

## Solution Assessment

Unclear until maintainers settle lifecycle policy. A version-scoped owner API plus CLI is directionally right, but latest-version, tag, restore, cache, download, permission, and audit semantics should be decided first.
