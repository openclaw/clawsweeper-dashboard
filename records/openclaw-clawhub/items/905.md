## Summary

Keep open. Current main still lacks the requested owner-facing per-version lifecycle workflow: the data model and public readers understand soft-deleted versions, but the exposed API, CLI, docs, and management UI remain skill-scoped, and this ClawHub cleanup policy keeps issues open for maintainer handling.

## What This Changes

Request owner-facing controls to hide, delete, unhide, or restore a single published skill version while keeping the skill live.

## Best Possible Solution

Settle the version lifecycle policy, then add authenticated owner/org-member/staff controls that update skillVersions.softDeletedAt with version-specific audit logs and consistent API, CLI, docs, tests, and UI behavior.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence in the gap because current main models and gates soft-deleted versions but exposes only skill-scoped owner lifecycle controls.

## Solution Assessment

Unclear until maintainers settle lifecycle policy. A version-scoped owner API plus CLI is directionally right, but latest-version, tag, cache, download, restore, permission, and audit semantics should be decided first.
