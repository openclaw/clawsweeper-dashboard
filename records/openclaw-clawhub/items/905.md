## Summary

Keep open. Current main has the data model and public gates for soft-deleted skill versions, but it still does not expose an owner-facing way to hide/delete/restore one published skill version; ClawHub issue cleanup policy also keeps issues open for maintainer handling.

## What This Changes

Add owner-facing CLI, API, or management controls to hide/delete/unhide/undelete a single published skill version without deleting the whole skill.

## Best Possible Solution

Define a version-scoped lifecycle policy, then add authenticated owner/org-member/staff controls that update skillVersions.softDeletedAt, write version-specific audit logs, and keep public lists, detail/file/download endpoints, tags/latest, caches, docs, and tests consistent.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence in the gap because version soft-delete state and public gates exist, while owner-facing per-version lifecycle controls are absent.

## Solution Assessment

Unclear until maintainers settle lifecycle policy. The requested owner-facing workflow is directionally right, but the maintainable solution needs explicit latest, tag, download, cache, restore, permission, and audit semantics.
