## Summary

Keep open. Current main still lacks the category-based moderation behavior, and this security-sensitive PR is not ready to merge because it carries an unrelated npm lockfile and needs a rebase/policy review against newer moderation safeguards.

## What This Changes

This PR adds a publisher-declared `category: security` moderation context that rewrites selected static scan reason codes to `info.security_context.*`, passes category context to the LLM evaluator, updates docs/tests, and adds `package-lock.json`.

## Best Possible Solution

If maintainers want this core moderation feature, rebase it onto current main, remove or explicitly approve the npm lockfile, and land the scanner, prompt, docs, and focused tests together after security-policy review.

## Reproduction Assessment

Yes for the PR review findings: source inspection of the supplied PR diff and current main shows the added lockfile, missing main implementation, and stale moderation-engine surface. I did not run tests because this was a read-only review.

## Solution Assessment

No. The feature direction may be useful, but this branch is not the best merge path until it is rebased onto current main, the unrelated lockfile is removed, and the moderation policy is explicitly approved.
