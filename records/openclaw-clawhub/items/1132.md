## Summary

Keep open. Current main does not implement publisher-declared security categories for moderation, and this open PR still needs maintainer/security review because it changes moderation trust policy, is dirty against current main, and carries an unrelated npm lockfile.

## What This Changes

This PR adds a `category: security` moderation context that rewrites selected static scan reason codes to `info.security_context.*`, passes the category into LLM evaluation, updates docs/tests, and adds `package-lock.json`.

## Best Possible Solution

If maintainers want this core moderation feature, rebase it onto current main, preserve existing VT/static/LLM false-positive safeguards, remove or explicitly approve the npm lockfile, and land scanner, prompt, docs, and focused tests together after security-policy review.
