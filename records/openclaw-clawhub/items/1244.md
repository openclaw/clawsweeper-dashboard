## Summary

Keep open. Current main intentionally treats user-initiated account deletion as terminal and source inspection explains the reported OAuth loop, but this is an open ClawHub issue requiring production account support plus account-lifecycle security/product judgment rather than conservative cleanup.

## What This Changes

The issue asks ClawHub maintainers to purge or recover a GitHub-linked account so the reporter can sign in again after deleting a previous account.

## Best Possible Solution

Maintainers should decide the deleted-account lifecycle policy, then either add an audited recovery or re-registration/unlink path with support tooling and tests, or keep deletion terminal while making the support route explicit in web and CLI login flows.

## Reproduction Assessment

Yes for the source-level path: current main deactivates deleted users, rejects those users during OAuth completion, and excludes them from active sessions. I did not verify the reporter's production account rows.

## Solution Assessment

No for the requested one-off purge as a general repository fix. The maintainable path needs an explicit recovery or terminal-deletion policy covering identity proof, privacy, ban separation, auditability, and operator behavior.
