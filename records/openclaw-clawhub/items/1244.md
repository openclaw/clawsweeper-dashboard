## Summary

Keep open. Current main intentionally treats user-initiated account deletion as terminal, while the reporter is asking for production account recovery or purge; that needs support access plus security/product judgment, not conservative cleanup.

## What This Changes

The issue asks maintainers to purge or unlink the reporter's deleted ClawHub account so GitHub OAuth can create an active session again.

## Best Possible Solution

Maintainers should decide the deleted-account lifecycle policy, then either add audited recovery or re-registration tooling with tests, or keep deletion terminal while making the support and error path explicit.

## Reproduction Assessment

Yes for the source-level path: current main deactivates deleted users, rejects those users during OAuth completion, and excludes them from active sessions. I did not verify the reporter's production account rows.

## Solution Assessment

No for the requested one-off purge as a general repository fix. The maintainable path needs an explicit recovery or terminal-deletion policy covering identity proof, privacy, ban separation, auditability, and operator behavior.
