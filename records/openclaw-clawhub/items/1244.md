## Summary

Keep open. Current main still treats user-initiated account deletion as terminal, and the requested recovery or purge path affects production account support, OAuth identity binding, and deletion policy rather than a narrow source-only cleanup.

## What This Changes

The issue requests restoring or purging a deleted ClawHub account record so the same GitHub user can sign in and re-register after account deletion.

## Best Possible Solution

Maintainers should decide the deleted-account lifecycle policy, then either add an audited recovery or re-registration path with support tooling and tests, or keep deletion terminal while making the support route explicit in web and CLI login flows.

## Reproduction Assessment

Yes for the source-level path: current main deactivates deleted users, rejects those users during OAuth completion, and excludes them from active sessions. I did not verify the reporter's production account rows.

## Solution Assessment

No for the requested one-off purge as a general repository fix. The maintainable path needs an explicit recovery or terminal-deletion policy covering identity proof, privacy, ban separation, auditability, and operator behavior.
