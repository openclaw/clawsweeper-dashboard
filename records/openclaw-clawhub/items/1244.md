## Summary

Keep this open. Current main still intentionally treats user-initiated ClawHub account deletion as terminal during GitHub OAuth, and this issue needs production account-support plus account-lifecycle/security judgment rather than conservative cleanup closure.

## What This Changes

Request to purge or reset a deleted ClawHub account's GitHub OAuth binding so the same GitHub user can register and sign in again.

## Best Possible Solution

Maintainers should decide the deleted-account lifecycle policy, then either add an audited recovery/unlink/re-registration path with tests and support tooling, or keep deletion irreversible while making the terminal state and support route clear in web and CLI login flows.

## Reproduction Assessment

Yes. The report gives concrete delete-account then GitHub OAuth steps, and current source shows that a deleted/deactivated user is rejected during OAuth completion and excluded from active session helpers.

## Solution Assessment

No. Manually purging one production record may unblock this reporter, but the maintainable solution is a policy-backed restore, unlink, or re-registration flow with auditability and banned-account separation.
