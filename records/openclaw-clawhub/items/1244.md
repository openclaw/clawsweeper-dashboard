## Summary

Keep this open. Current main still intentionally blocks sign-in for deleted or deactivated users after GitHub OAuth, and this open ClawHub account-lifecycle/support case needs maintainer policy and production support handling rather than cleanup closure.

## What This Changes

The issue asks maintainers to purge or reset the reporter's deleted ClawHub account record so GitHub OAuth can create an active session again.

## Best Possible Solution

Maintainers should decide the account lifecycle policy, then either add an audited recovery or re-registration path with identity verification and tests, or keep deletion irreversible while surfacing a clear terminal-state/support message in web login flows.

## Reproduction Assessment

Yes. The reporter gives concrete delete-account then GitHub OAuth steps, and current source/tests show deleted or deactivated users are rejected before an active session can be established.

## Solution Assessment

No. Manually purging one production record may unblock this reporter, but the maintainable solution is a policy-backed restore, unlink, or re-registration flow with auditability and user-facing messaging.
