## Summary

Keep open. Current main still treats user-initiated deletion as terminal and blocks normal OAuth reauth, while this remains an open ClawHub issue with related affected-user reports; the next action is account support plus security/product policy, not conservative cleanup.

## What This Changes

The issue asks maintainers to purge or recover a GitHub-linked ClawHub account after user-initiated deletion because OAuth succeeds but no ClawHub session is established.

## Best Possible Solution

Maintainers should define the deleted-account recovery policy, then either implement an audited restore/re-registration or operator cleanup path with tests and docs, or keep deletion terminal while making the support/error path explicit.

## Reproduction Assessment

Yes for the source-level account-lifecycle path: current main marks deleted accounts as deactivated, rejects those users in the OAuth callback, and excludes them from active sessions. I did not verify the reporter's production Convex/Auth records.

## Solution Assessment

No for an automatic one-off purge as a source cleanup. The maintainable solution needs an explicit account recovery policy covering identity proof, ban separation, auditability, purged profile data, and operator behavior.
