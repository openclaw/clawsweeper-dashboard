## Summary

Keep open. Current main intentionally treats user-initiated account deletion as terminal, so the reported login loop maps to a real account-lifecycle support and security-policy question rather than a cleanup-close candidate; the repository policy also says ClawHub issues should stay open.

## What This Changes

The issue asks maintainers to purge or reset the reporter's GitHub-linked ClawHub account record so they can register again after deleting a previous account.

## Best Possible Solution

Maintainers should decide the deleted-account lifecycle policy, then either add audited recovery or identity-unlink tooling with tests and operator guidance, or keep deletion terminal with clearer blocked-login support messaging.

## Reproduction Assessment

Yes for a source-level path: a deleted GitHub-backed account reaches handleDeletedUserSignIn during OAuth and is rejected when deactivatedAt is present or migrated from legacy deletedAt. I did not verify the reporter's production row.

## Solution Assessment

No for a one-off purge as the general solution. Manual cleanup may unblock one user, but the maintainable path needs a policy-backed, audited recovery or identity-unlink flow that preserves ban and ownership invariants.
