## Summary

Keep this open. Current main intentionally treats user-initiated account deletion as terminal, so the remaining request is production account recovery plus account-lifecycle policy rather than cleanup closure.

## What This Changes

The reporter asks ClawHub maintainers to purge or recover a deleted account whose GitHub OAuth flow completes but does not create an active session.

## Best Possible Solution

Maintainers should decide the deleted-account policy, then either add audited recovery/re-registration tooling with identity verification and tests, or keep deletion terminal while ensuring every login surface clearly explains that state.

## Reproduction Assessment

Yes for the blocked sign-in path from source inspection: deleting an account writes `deactivatedAt`, OAuth completion rejects that record, and active-user helpers exclude it. I did not live-test the reporter's production account record.

## Solution Assessment

No. A one-off purge may unblock this reporter, but the maintainable path is a policy-backed recovery, unlink, or terminal-state support flow with auditability.
