## Summary

Keep open. Current main still treats user-initiated ClawHub account deletion as terminal, and this issue remains a production account-support plus account-lifecycle/security policy matter rather than conservative cleanup.

## What This Changes

The issue reports a ClawHub OAuth login loop after account deletion and asks maintainers to purge or reset the GitHub-linked production account record.

## Best Possible Solution

Maintainers should decide the deleted-account lifecycle policy, then either add an audited recovery/re-registration path with identity verification and tests, or keep deletion terminal with clearer support and operator guidance.

## Reproduction Assessment

Yes for the source-level path: current main marks deleted accounts with deactivatedAt/purgedAt, rejects OAuth completion for that user, and filters deleted/deactivated users from active auth helpers. I did not verify the reporter's specific production database row.

## Solution Assessment

No for a one-off purge as the general solution. The maintainable path is a policy-backed, audited recovery or identity-unlink flow, or an explicitly terminal deletion policy with consistent messaging and support handling.
