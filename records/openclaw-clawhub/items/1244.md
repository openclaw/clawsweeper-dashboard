## Summary

Keep this open. Current main still makes user-initiated ClawHub deletion terminal during GitHub OAuth, and this open ClawHub issue needs maintainer production support plus account-lifecycle/security judgment rather than conservative cleanup closure.

## What This Changes

The issue requests a way to re-register or restore login for the same GitHub identity after deleting a ClawHub account, with a manual purge for the reporter if needed.

## Best Possible Solution

Maintainers should choose the account lifecycle policy, then add an audited restore/re-registration or GitHub identity unlink path with tests and support tooling, or keep deletion irreversible while making the terminal state and support route explicit.

## Reproduction Assessment

Yes. The issue gives clear steps, and current code provides a high-confidence reproduction path: delete an account, complete GitHub OAuth for the same deleted identity, then handleDeletedUserSignIn rejects the deactivated user before an active session can be established.

## Solution Assessment

No. Manually purging a single record may unblock this reporter, but the maintainable solution is an explicit audited recovery or re-registration flow that preserves identity verification and keeps deleted and banned account paths separate.
