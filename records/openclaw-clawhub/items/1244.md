## Summary

Keep open. Current main still intentionally rejects OAuth sign-in for deleted/deactivated ClawHub users, and this issue is also protected by maintainer-review, product-decision, security-review, and auth/security/session impact labels, so conservative cleanup should not close it.

## What This Changes

The issue asks maintainers to purge or reset a deleted ClawHub account record so GitHub OAuth can establish a session again for the same GitHub identity.

## Best Possible Solution

Maintainers should define the deleted-account lifecycle policy, then implement an audited restore, re-registration, or operator cleanup path with tests and support documentation, or keep deletion terminal while surfacing that state clearly in login/support flows.

## Reproduction Assessment

Yes for the source-level path: current main deletes by deactivating the user, OAuth completion rejects deactivated users, and active-session helpers exclude them. I did not verify the reporter's production database row or run the live OAuth flow.

## Solution Assessment

No for the requested one-off purge as a repository-level fix. The durable solution needs an explicit recovery or re-registration policy with identity proof, deletion privacy, ban separation, auditability, and operator behavior defined.
