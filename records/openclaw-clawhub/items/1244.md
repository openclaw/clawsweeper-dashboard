## Summary

Keep open. Current main intentionally treats user-initiated account deletion as terminal, and the remaining request is production account support plus an account-lifecycle/security policy decision; ClawHub issue cleanup policy also keeps issues open rather than auto-closing them.

## What This Changes

The issue asks maintainers to purge or unlink a deleted ClawHub account's GitHub identity record so the same GitHub user can register and sign in again after OAuth.

## Best Possible Solution

Maintainers should decide the deleted-account lifecycle policy, then either add an audited recovery or identity-unlink/re-registration path with tests and operator guidance, or keep deletion terminal while ensuring login clearly surfaces the support route.

## Reproduction Assessment

Yes for the generic deleted-account path from source inspection: account deletion writes deactivatedAt/purgedAt, OAuth completion rejects that user, and active-user helpers hide the record. I did not verify the reporter's production row.

## Solution Assessment

No for a one-off purge as the general solution. Manual cleanup may unblock one user, but a maintainable fix needs a policy-backed, audited recovery or identity-unlink flow that preserves ban, deletion, and ownership invariants.
