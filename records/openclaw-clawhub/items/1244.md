## Summary

This issue should stay open. Current main still treats user-initiated account deletion as terminal during GitHub OAuth, and the remaining action is production account support plus account-lifecycle policy rather than conservative cleanup.

## What This Changes

The issue requests recovery or purge of a deleted ClawHub account record so the same GitHub identity can complete OAuth and create an active session again.

## Best Possible Solution

Maintainers should decide the deleted-account lifecycle policy, then provide an audited recovery, unlink, or re-registration path with clear user-facing messaging and tests.

## Reproduction Assessment

Yes. The issue gives concrete delete-account then GitHub OAuth steps, and current source/tests show that a deactivated or legacy self-deleted user is rejected before an active session can be established.

## Solution Assessment

No. A one-off production purge may unblock this reporter, but the maintainable solution is a policy-backed recovery or re-registration flow that preserves identity verification, audit history, and separation from banned-account handling.
