## Summary

Keep this open. Current main intentionally treats user-initiated account deletion as irreversible and rejects OAuth reauth for deleted/deactivated users, so the remaining action is maintainer account support plus account-lifecycle/security policy rather than cleanup closure.

## What This Changes

The issue requests account cleanup or a recovery path after a deleted ClawHub account cannot establish a GitHub OAuth session.

## Best Possible Solution

Maintainers should decide the account lifecycle policy, then either add an audited restore/unlink/re-registration support path with tests or keep deletion irreversible while making the terminal state and support route clear in the OAuth UI.

## Reproduction Assessment

Yes, at source level. The issue gives concrete delete-then-OAuth steps, and current source/tests show deleted or deactivated users are rejected before an active session can be established.

## Solution Assessment

No, not as a one-off source change. Manually purging one production record may unblock this reporter, but the maintainable solution needs policy-backed recovery or explicit irreversible-account messaging with auditability.
