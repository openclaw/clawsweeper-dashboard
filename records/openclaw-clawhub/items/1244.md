## Summary

Keep this open. Current main still treats user-initiated deletion as irreversible and blocks deleted-account reauth, while this specific report asks for production account recovery/purge that needs maintainer support access, identity verification, and an account-lifecycle policy decision.

## What This Changes

The issue reports a ClawHub OAuth login loop after deleting an account and asks maintainers to purge or recover the GitHub-linked account record.

## Best Possible Solution

Maintainers should choose the account lifecycle policy, then either add an audited recovery or re-registration/unlink path with support tooling and tests, or keep deletion irreversible while making the terminal state and support route reliable for affected users.

## Reproduction Assessment

Yes for the general deleted-account reauth path: current source and tests show deleted or deactivated users are rejected after OAuth and are excluded from active sessions. No for the reporter's exact production record, which requires maintainer support access.

## Solution Assessment

No automated repo-only purge is not the best solution. The maintainable path is a deliberate account recovery/re-registration policy plus verified production support tooling or clearer irreversible-deletion handling.
