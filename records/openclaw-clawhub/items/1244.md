## Summary

Keep this open. Current main still treats self-deleted ClawHub accounts as terminal during GitHub OAuth, and this open ClawHub issue needs maintainer account-support and account-lifecycle/security judgment rather than cleanup closure.

## What This Changes

The issue asks maintainers to purge or reset the reporter's deleted ClawHub/GitHub OAuth record so the same GitHub account can register again.

## Best Possible Solution

Maintainers should either provide an audited recovery or re-registration path with identity verification and tests, or keep deletion irreversible while surfacing a clear terminal-state/support message in the OAuth UI.

## Reproduction Assessment

Yes. The issue gives concrete delete-account then GitHub OAuth steps, and current source/tests show deleted or deactivated users are rejected before an active session can be established.

## Solution Assessment

No. Manually purging one production record may unblock this reporter, but the maintainable solution is a policy-backed restore, unlink, or re-registration flow with auditability and user-facing messaging.
