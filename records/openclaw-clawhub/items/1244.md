## Summary

Keep open. Current main still intentionally treats deleted ClawHub accounts as terminal, while this report needs production account support plus an account-lifecycle security/product decision rather than conservative cleanup.

## What This Changes

A user reports that GitHub OAuth succeeds after deleting a ClawHub account, but ClawHub does not create a new active session and asks for the old GitHub-linked record to be purged.

## Best Possible Solution

Maintainers should decide the deleted-account recovery policy, then add an audited restore, re-registration, or operator cleanup path with tests and docs, or keep deletion terminal while making the support/error path explicit.

## Reproduction Assessment

Yes for the source-level account lifecycle path: current main marks deleted accounts as deactivated, rejects those users during OAuth completion, and excludes them from active sessions. I did not verify the reporter's production account rows.

## Solution Assessment

No for an automatic one-off purge as a repository fix. The maintainable path needs a defined recovery/support policy covering identity proof, deletion privacy, ban separation, auditability, and operator behavior.
