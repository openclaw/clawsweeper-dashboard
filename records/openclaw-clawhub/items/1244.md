## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing pull requests that current main certainly implements. Current main still intentionally treats user-initiated deletion as terminal during GitHub OAuth, so the reported account recovery problem remains unresolved.

## What This Changes

The issue requests purging or unbinding the reporter's deleted ClawHub/GitHub OAuth account record so they can re-register and establish a session after GitHub OAuth.

## Best Possible Solution

Maintainers should decide the account lifecycle policy, then either add an audited recovery or re-registration path that safely restores or unlinks the GitHub identity binding with tests and support tooling, or keep deletion irreversible while making the terminal state and support route clear in web and CLI login flows.
