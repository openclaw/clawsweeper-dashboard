## Summary

Keep open. Current `main` intentionally treats user-initiated account deletion as terminal and now surfaces blocked-account auth errors, but this issue still asks for production account recovery or purge behavior that requires maintainer support access and account-lifecycle policy.

## What This Changes

The issue requests purging or restoring a deleted ClawHub account binding so the same GitHub user can complete OAuth sign-in again.

## Best Possible Solution

Maintainers should choose and document the deleted-account lifecycle, then either provide a verified administrative purge/re-registration path or keep deletion irreversible with clear OAuth and support messaging.

## Reproduction Assessment

Yes, by source inspection for the deleted-account path: account deletion writes inactive state, OAuth completion rejects that linked user, and active-user helpers keep the session from becoming an active ClawHub user. The reporter-specific production purge cannot be reproduced without maintainer access.

## Solution Assessment

No, not as an automatic purge or session-creation fix. The requested behavior conflicts with the current irreversible-deletion policy, so the maintainable path needs an explicit account-lifecycle and security decision first.
