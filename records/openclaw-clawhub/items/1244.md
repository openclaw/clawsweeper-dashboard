## Summary

Keep open. Current main still treats user-initiated account deletion as terminal during GitHub OAuth, related deleted-account OAuth reports remain open, and this needs maintainer support/security policy rather than cleanup closure.

## What This Changes

The issue requests account recovery after deleting a ClawHub account because GitHub OAuth completes but no active ClawHub session is established.

## Best Possible Solution

Maintainers should choose one deleted-account lifecycle, then implement either audited recovery/re-registration or a clearly terminal support path across OAuth, account records, operator tooling, docs, and tests.

## Reproduction Assessment

Yes at the source level for the underlying blocked deleted-account sign-in path: deletion writes deactivatedAt/purgedAt, OAuth completion rejects that state, and active-user helpers hide the linked user. I did not live-reproduce the reporter's exact production browser loop or inspect their account rows.

## Solution Assessment

No for an automatic one-off purge or code change as cleanup. The maintainable solution needs explicit maintainer policy for recovery authority, privacy, auditability, and banned-versus-deleted account separation before implementation.
