## Summary

Keep this open. Current main still deliberately treats user-initiated account deletion as terminal during GitHub OAuth, and this open ClawHub issue needs maintainer account-support and account-lifecycle/security policy judgment rather than conservative cleanup.

## What This Changes

The issue requests account recovery or record purging so a GitHub user can re-register after deleting a ClawHub account and completing OAuth.

## Best Possible Solution

Maintainers should decide whether deletion remains irreversible or add an audited recovery/re-registration path that safely unlinks or restores the GitHub identity binding with tests, support tooling, and clear user-facing messaging.

## Reproduction Assessment

Yes. The issue gives a concrete delete-account then GitHub OAuth reauth path, and current main has source and tests that intentionally reject deleted/deactivated users during OAuth.

## Solution Assessment

Unclear. A manual purge might restore this reporter quickly, but the maintainable fix needs a policy-backed recovery or re-registration path that preserves identity verification, audit history, and banned-account separation.
