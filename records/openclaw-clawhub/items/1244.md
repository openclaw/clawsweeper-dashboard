## Summary

Keep this open. Current main still treats user-initiated account deletion as terminal during GitHub OAuth, and this issue needs maintainer production support plus account-lifecycle/security policy rather than conservative cleanup closure.

## What This Changes

The issue requests production cleanup or recovery for a GitHub OAuth account that cannot establish a ClawHub session after the prior ClawHub account was deleted.

## Best Possible Solution

Maintainers should decide the deleted-account lifecycle policy, then add an audited restore/re-registration or GitHub identity unlink path with tests and support tooling, or keep deletion irreversible while making the terminal state and support route explicit.

## Reproduction Assessment

Yes, for the account-deletion reauth block: deleteAccount marks the user deactivated, handleDeletedUserSignIn rejects the same identity during OAuth, and tests assert that behavior. I did not verify the reporter's production record or run a live OAuth flow.

## Solution Assessment

No. Manually purging one production record may unblock this reporter, but the maintainable fix is an audited recovery or re-registration path that preserves identity proof and keeps deletion and ban states separate.
