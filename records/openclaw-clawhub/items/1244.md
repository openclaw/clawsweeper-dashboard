## Summary

Keep open. Current main still treats user-initiated account deletion as terminal during GitHub OAuth, and this ClawHub issue needs maintainer support/policy handling rather than cleanup closure.

## What This Changes

The issue asks maintainers to purge or recover a deleted ClawHub account so the same GitHub OAuth identity can sign in again.

## Best Possible Solution

Maintainers should choose the account lifecycle policy, then either implement an audited restore/re-registration path or keep deletion irreversible while making the terminal state and support route explicit.

## Reproduction Assessment

Yes. The source path is reproducible by deleting an account and then completing GitHub OAuth for the same identity: current main routes the callback through handleDeletedUserSignIn and rejects the deleted/deactivated user.

## Solution Assessment

No. A one-off purge may unblock this reporter, but the durable fix should be an audited recovery/unlink path with identity verification, or an explicit product decision that deletion remains irreversible.
