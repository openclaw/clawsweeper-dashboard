## Summary

Keep open. Current main still deliberately treats user-initiated account deletion as terminal, while this report needs production account support plus a maintainer account-lifecycle/security policy decision; ClawHub cleanup policy also keeps issues open.

## What This Changes

The issue asks maintainers to purge or unlink a deleted ClawHub/GitHub account record so the reporter can re-register after OAuth loops.

## Best Possible Solution

Maintainers should decide the deleted-account policy, then either add an audited recovery/re-registration path with identity verification and tests or keep deletion terminal with clear support messaging and operator handling.

## Reproduction Assessment

Yes for the source-level blocked path: deletion writes deactivatedAt/purgedAt, OAuth completion rejects that record, and active-user helpers refuse to create an active session. I did not live-test the reporter's production account row.

## Solution Assessment

No for a one-off purge as the general solution. The maintainable path is a policy-backed, audited recovery or identity-unlink flow, or a consciously terminal deletion policy with consistent messaging and support process.
