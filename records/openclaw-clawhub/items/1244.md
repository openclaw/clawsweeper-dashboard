## Summary

Keep open. Current main intentionally treats user-initiated account deletion as terminal, and resolving this issue requires production support access plus account-recovery security/product policy rather than conservative cleanup.

## What This Changes

The issue asks ClawHub maintainers to recover or purge a deleted account so the same GitHub identity can sign in again after OAuth completes.

## Best Possible Solution

Maintainers should decide the deleted-account lifecycle policy, then add an audited restore, re-registration, or operator cleanup path with tests/docs, or keep deletion terminal with explicit support/UI messaging.

## Reproduction Assessment

Yes for the source-level path: current main deactivates deleted users, rejects them during OAuth completion, and excludes them from active sessions. I did not verify the reporter's production account rows.

## Solution Assessment

No for the requested one-off purge as a general repository fix. The maintainable solution needs a defined recovery/support policy covering identity proof, deletion privacy, ban separation, auditability, and operator behavior.
