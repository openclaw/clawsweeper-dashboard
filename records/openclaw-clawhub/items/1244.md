## Summary

Keep this open. Current main still blocks deleted ClawHub accounts during GitHub OAuth, and the remaining request is production account recovery plus an account-lifecycle policy decision rather than conservative cleanup.

## What This Changes

The issue requests recovery or purging for a GitHub OAuth identity that cannot re-register after deleting its ClawHub account.

## Best Possible Solution

Maintainers should decide the deleted-account recovery policy, then either add an audited recovery or re-registration path with tests and support tooling, or keep deletion irreversible while continuing to surface a clear blocked-account message and support route.

## Reproduction Assessment

Yes from source: a user with deactivatedAt is rejected during OAuth completion and excluded by active-user helpers, matching the reported no-session outcome after account deletion. I did not verify the reporter's production record because that requires private Convex/Auth access.

## Solution Assessment

No for automated cleanup: a one-off production purge may unblock this reporter, but the maintainable solution needs an explicit account-lifecycle decision and audited recovery or clear terminal-state UX.
