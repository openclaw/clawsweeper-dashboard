## Summary

Keep open. Current main still treats user-initiated account deletion as terminal, and this open ClawHub issue is now a production account-recovery and account-lifecycle policy question rather than conservative cleanup.

## What This Changes

The request asks maintainers to purge or unblock a GitHub OAuth identity so the reporter can re-register after deleting a ClawHub account.

## Best Possible Solution

Decide the deleted-account policy, then implement or operate an audited recovery/re-registration path with identity verification, or keep deletion terminal with consistently clear web and CLI messaging.

## Reproduction Assessment

Yes for the current-main blocked sign-in path from source inspection: deletion writes `deactivatedAt`, OAuth completion rejects that record, and active-user helpers exclude it. I did not live-test the reporter's production account record.

## Solution Assessment

No. A one-off purge may unblock the reporter, but the maintainable solution is policy-backed recovery or re-registration tooling that preserves deletion, ban, audit, and ownership boundaries.
