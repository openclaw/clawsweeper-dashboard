Keep open. Current main still treats user-initiated account deletion as terminal, and resolving the reporter's blocked GitHub OAuth path requires production support plus account-lifecycle security/product judgment.

## What This Changes

The issue requests a clean re-registration or production purge for a GitHub OAuth account that is stuck logged out after deleting its ClawHub account.

## Best Possible Solution

Maintainers should decide the deleted-account lifecycle policy, then add an audited restore, re-registration, or operator cleanup path with tests and docs, or keep deletion terminal while making the support/error path explicit.

Yes for the source-level account-lifecycle path: current main marks deleted accounts as deactivated, rejects matching OAuth callbacks, and excludes those users from active sessions. I did not verify the reporter's production Convex/Auth rows.
