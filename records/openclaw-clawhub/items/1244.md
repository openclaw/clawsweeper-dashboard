## Summary

Keep open. Current main still intentionally treats user-initiated deletion as terminal, and ClawHub cleanup policy keeps issues open; this needs maintainer production support plus account-lifecycle/security judgment.

## What This Changes

The issue asks maintainers to purge or unblock a deleted ClawHub account so the same GitHub OAuth identity can register and sign in again.

## Best Possible Solution

Maintainers should decide the account lifecycle policy, then either add an audited recovery/unlink/re-registration path with tests and support tooling or keep deletion terminal with clear web/CLI messaging and an operator playbook.

## Reproduction Assessment

Yes for the source-level path: deleteAccount sets deactivatedAt/purgedAt, OAuth completion rejects that user, and active-user helpers refuse to establish an active session. I did not live-test the reporter's production account row.

## Solution Assessment

No for a one-off purge as the general solution. The maintainable path is a policy-backed audited recovery or identity-unlink flow, or a consciously terminal deletion policy with consistent support messaging.
