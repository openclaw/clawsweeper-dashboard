## Summary

Keep open. Current main intentionally blocks sign-in for deleted ClawHub accounts, and resolving this report requires production support plus an account-recovery security/product decision rather than conservative cleanup.

## What This Changes

The issue asks maintainers to purge or recover a GitHub OAuth account record so a previously deleted ClawHub account can sign in or register again.

## Best Possible Solution

Maintainers should define the deleted-account lifecycle policy, then add an audited restore, re-registration, or operator cleanup path with tests/docs, or keep deletion terminal while making the support route explicit.

## Reproduction Assessment

Yes for the source-level path: current main deactivates deleted users, rejects those users during OAuth completion, and excludes them from active sessions. I did not verify the reporter's production account rows.

## Solution Assessment

No for the requested one-off purge as a general repository fix. The maintainable solution needs a defined recovery/support policy covering identity proof, deletion privacy, ban separation, auditability, and operator behavior.
