## Summary

Keep this open. Current main intentionally treats user-initiated account deletion as terminal, and this report still needs maintainer support/policy handling rather than conservative cleanup or an automated source-only fix.

## What This Changes

The issue asks maintainers to purge or reset a production ClawHub account record so the reporter can re-register with the same GitHub identity after account deletion.

## Best Possible Solution

Maintainers should choose the deletion recovery policy, then either add an audited restore/re-registration/unlink path with support tooling or keep deletion irreversible while making the support route and terminal state explicit.

## Reproduction Assessment

Yes, there is a high-confidence source reproduction for the same-account deletion path: deleteAccount sets deactivatedAt/purgedAt, OAuth completion rejects that user, and active-user helpers refuse deactivated records. I did not verify the reporter's production account record.

## Solution Assessment

No, a one-off production purge is not the best repository-level solution by itself. The maintainable path needs an explicit account-lifecycle policy plus audited recovery or re-registration tooling, or clearer irreversible-deletion support messaging.
