## Summary

Keep this open. Current main still treats user-initiated ClawHub account deletion as terminal, and this ClawHub issue should not be auto-closed by cleanup; the remaining action is production account support plus account-lifecycle/security policy.

## What This Changes

The requester wants the deleted ClawHub account or GitHub OAuth binding for `lnastri25` purged/reset so GitHub sign-in can create an active session again.

## Best Possible Solution

Maintainers should decide the account lifecycle policy, then provide an audited restore/re-registration or GitHub identity unlink path with tests and support tooling, or keep deletion irreversible with a clear error and support route.

## Reproduction Assessment

Yes. The issue gives clear steps, and source inspection shows the path: delete an account, complete GitHub OAuth for the same deleted identity, then `handleDeletedUserSignIn` rejects the deactivated user before an active session can be established.

## Solution Assessment

No. Manually purging one production record may unblock this reporter, but the maintainable solution is an explicit audited recovery or re-registration policy that keeps deleted and banned account paths separate.
