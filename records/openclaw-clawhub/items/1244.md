## Summary

Keep open. This is a ClawHub issue, and current main still intentionally blocks sign-in for deleted/deactivated accounts, so the remaining work is production account support plus an account-lifecycle/security policy decision rather than conservative cleanup.

## What This Changes

The issue requests help re-registering through GitHub OAuth after deleting a ClawHub account, including a possible production purge for the reporter's GitHub identity.

## Best Possible Solution

Maintainers should decide the account lifecycle policy, then either add an audited recovery/re-registration path with identity verification and tests or keep deletion terminal with clearer support messaging and operator tooling.

## Reproduction Assessment

Yes for the source-level path: deletion records deactivatedAt/purgedAt, OAuth completion rejects that user, and active-user helpers refuse to establish a session. I did not live-test the reporter's production account row.

## Solution Assessment

No for a one-off purge as the general solution. The maintainable solution is a policy-backed, audited recovery or identity-unlink flow, or an explicitly terminal deletion policy with consistent messaging and support handling.
