## Summary

Keep open. Repository policy for openclaw/clawhub does not auto-close open issues, and current main only partially addresses the report with ClawHub registry-side scanning, hiding, and install blocking. The broader security report still needs maintainer triage across OpenClaw runtime sandboxing, install approval UX, credential storage, outbound network controls, and production moderation policy.

## What This Changes

The issue reports that malicious third-party skills can use install/prerequisite instructions to run shell commands and exfiltrate local OpenClaw API keys, and asks for sandboxing, explicit approval, keychain-backed credentials, registry scanning, and network controls.

## Best Possible Solution

Keep the registry hardening already on main, then split the remaining work explicitly between ClawHub scanning/moderation/distribution policy and OpenClaw core security work for sandboxed prerequisite execution, explicit install approval, keychain-backed credential storage, and outbound network controls.
