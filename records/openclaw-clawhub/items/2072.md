## Summary

## What This Changes

The PR enforces scoped plugin publishes to match publisher ownership, adds package owner transfer APIs/CLI/UI, and refactors admin/user transfer logic while preserving package identity/history.

## Best Possible Solution

Merge only after owner transfers update packageSearchDigest/packageCapabilitySearchDigest in the same mutation, regression coverage and a changelog entry are added, and the contributor posts redacted after-fix proof for the UI/API/CLI path.

## Reproduction Assessment

Yes, from source inspection: the new transfer path reaches a helper that patches the packages row, while catalog/search owner fields are served from denormalized digest rows. A live check would transfer a package and compare detail owner versus catalog/search owner.

## Solution Assessment

No, not yet. The transfer implementation should synchronize the package search and capability digests in the same mutation, then document the user-facing change and provide real behavior proof.
