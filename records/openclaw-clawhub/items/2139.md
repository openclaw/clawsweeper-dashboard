## What This Changes

This PR adds Convex full-text search indexes for skill display names and slugs, queries them alongside existing prefix paths, and adds search regression tests.

## Best Possible Solution

Keep the implementation merged through https://github.com/openclaw/clawhub/pull/2140 as the canonical fix and close this PR without merging its older branch.

## Reproduction Assessment

Yes. Source inspection shows the pre-fix direct path was limited to full-string and first-token prefix indexes while the fallback was bounded by recent digest windows; current main now has regression tests for the failing middle-token cases.

## Solution Assessment

No for this branch, yes for current main. Full-text indexes are the right primitive, but this branch merged raw any-word full-text hits; the merged current-main implementation preserves all-token filtering and has broader regression coverage.
