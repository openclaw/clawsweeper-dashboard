## Summary

Keep open. Current main still rejects code-plugin runtimeId retargeting for existing packages, and the linked open PR is the active implementation candidate; the remaining decision is security-sensitive maintainer/admin approval of the runtimeId repair boundary.

## What This Changes

This issue requests a guarded way to correct a placeholder code-plugin package runtimeId so a package reserved with the wrong runtime id can later publish the real plugin.

## Best Possible Solution

Keep a single audited, collision-checked path for placeholder runtimeId repair, either by landing https://github.com/openclaw/clawhub/pull/2213 after approval and proof or by using the existing admin repair primitive for OpenViking after production row verification.

## Solution Assessment
