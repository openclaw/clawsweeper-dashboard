## Summary

Keep open. The OpenViking package was unblocked by the merged admin repair flow in https://github.com/openclaw/clawhub/pull/2289 and confirmed in the discussion, but the broader reusable settings/runtimeId correction flow remains a maintainer product and security decision.

## What This Changes

The issue requests a guarded way to correct or migrate a code-plugin package runtimeId/name after a placeholder package reserved the wrong runtime id.

## Best Possible Solution

Keep the admin/operator repair path for exceptional production fixes, and design any reusable settings-page repair flow with explicit authorization, collision checks, audit logs, and maintainer-approved security boundaries.

## Reproduction Assessment

Yes, at source level: current main rejects code-plugin publishes when the stored runtimeId differs from the incoming runtimeId, and the issue gives the exact package/runtimeId pair that hit this path. The specific OpenViking production row has since been repaired, so this review did not rerun a live failing publish.

## Solution Assessment

Partially. The merged admin/operator repair was the safest immediate unblock, while the broader self-service or settings-page correction flow still needs explicit maintainer product and security design.
