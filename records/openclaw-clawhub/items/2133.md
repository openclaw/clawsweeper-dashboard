Keep open. Current main still rejects code-plugin runtimeId retargeting, and this issue is paired with an open closing PR that needs maintainer/security review before the issue can be resolved.

## What This Changes

Allow a guarded way to correct or migrate a code-plugin package runtimeId when a placeholder reservation bound the package to the wrong id.

## Best Possible Solution

Review and either land or close https://github.com/openclaw/clawhub/pull/2213, or have a ClawHub admin use the existing audited repair primitive after verifying production runtimeId ownership.

## Reproduction Assessment

Yes for the generic current-main behavior: source and tests show runtimeId retargeting and active runtimeId collisions are rejected. The exact OpenViking migration still needs admin verification of production package rows.

## Solution Assessment

Unclear until maintainers approve the repair authority boundary. The safe direction is an audited, collision-checked migration path, either via the existing admin mutation or the reviewed trusted-publish exception in https://github.com/openclaw/clawhub/pull/2213.
