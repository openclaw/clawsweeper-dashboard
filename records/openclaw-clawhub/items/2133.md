Keep open. The immediate OpenViking unblock was handled by merged admin repair-name tooling and a one-off admin repair, but current main still preserves publish-time runtimeId immutability and the broader correction surface needs maintainer/security product approval.

## What This Changes

Request to provide a guarded way to correct or migrate a code-plugin package runtimeId when a placeholder reservation bound the package name to the wrong id.

## Best Possible Solution

Keep the admin repair tooling for one-off migrations, and design any broader runtimeId correction as an explicit plugin settings or admin workflow with collision checks, audit logs, and clear owner/moderator/admin authority rules.

Yes, at source level: current main rejects existing code-plugin publishes when the stored runtimeId differs from the incoming runtimeId, and tests assert that guard. I did not reproduce the exact production OpenViking row because the discussion confirms it has already been repaired.

## Solution Assessment
