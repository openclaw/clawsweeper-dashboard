## Summary

Keep open: the OpenViking package was unblocked by a one-off admin repair, but current main still rejects publish-time runtimeId retargeting and does not expose a general guarded runtimeId correction workflow.

## What This Changes

Request a guarded ClawHub flow to correct or migrate code-plugin package runtimeId values when placeholder reservations bound the wrong plugin id.

## Best Possible Solution

Keep the shipped admin repair-name path for one-off migrations, and design any broader runtimeId correction as an explicit plugin settings or admin workflow with authority checks, collision rejection, audit logging, dry-run behavior, and focused tests.
