## Summary

Keep open: the OpenViking package was unblocked by a one-off admin repair, but current main still preserves publish-time runtimeId immutability and the broader guarded correction workflow remains a security-sensitive maintainer product decision.

## What This Changes

The issue requests a guarded way to correct or migrate code-plugin package runtimeId/package identity when a placeholder package was reserved with the wrong runtime id.

## Best Possible Solution

Keep the shipped admin repair-name path for one-off migrations, and design any broader runtimeId correction as an explicit plugin settings or admin workflow with authority checks, collision rejection, audit logging, dry-run behavior, and focused tests.

## Reproduction Assessment

Yes, at source level: current main rejects direct code-plugin runtimeId retargeting during publish, and tests assert that guard. The exact OpenViking production conflict was resolved through a one-off admin repair rather than reproduced locally.

## Solution Assessment
