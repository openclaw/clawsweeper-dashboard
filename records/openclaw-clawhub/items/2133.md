Keep open. The OpenViking one-off unblock was completed through merged admin repair tooling and a production admin repair, but current main still does not expose the broader guarded runtimeId correction/settings workflow that maintainers selected over implicit publish-time retargeting.

## What This Changes

The issue requests a guarded way to correct or migrate a code-plugin package runtimeId after a placeholder package reserved the wrong id.

## Best Possible Solution

Keep the shipped admin repair-name path for one-off migrations, and design any broader runtimeId correction as an explicit plugin settings or admin workflow with collision checks, audit logging, and clear owner, moderator, and admin authority rules.
