The issue requests a safe way to correct or migrate a code-plugin package runtimeId after a placeholder package reserved the wrong plugin identity.

## Best Possible Solution

Keep the shipped admin repair-name tooling for one-off migrations, and design any broader runtimeId correction as an explicit settings or admin workflow with collision checks, audit logging, and clear authority boundaries.
