## What This Changes

The branch exposes long-form skill descriptions and minimal `{ key, required }` setup metadata through v1 list/detail responses, adds the fields to public schema packages, and denormalizes descriptions into latest-version summary storage.

## Best Possible Solution

Keep the minimal additive API contract and moderation gates, but make one resumable production migration update and verify both denormalized summary stores before merging.
