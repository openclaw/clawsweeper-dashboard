## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Backfill publisher stats before ordering on them:** `convex/schema.ts:127-131`
  - body: Existing publisher documents will not have these aggregate fields when this deploys, but the new directory paginates by `totalDownloads` before it computes fallback stats for the returned rows. That means the production Publishers page and Popular publishers cards will be ordered by missing values or recent document order until each publisher happens to change; add a cursor-based backfill or keep the query off the denormalized index until the fields are populated.
  - confidence: 0.86
- **[P2] Build package links with the scoped plugin route helper:** `convex/publishers.ts:229`
  - body: For a scoped package such as `@openclaw/codex`, this produces `/plugins/%40openclaw%2Fcodex`, while the app's plugin routes and helper expect `/plugins/@openclaw/codex`. Publisher profile package rows will therefore navigate to the wrong route for scoped plugin packages; use the existing scoped-route logic or return enough data for the frontend to build it.
  - confidence: 0.9
