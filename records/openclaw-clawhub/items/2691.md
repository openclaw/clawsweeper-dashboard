## Summary

Keep open: current main and the latest tagged release still render the public skill detail stat as all-time Installs without also showing Downloads or an in-page explanation, while the reported skill's public API data still has downloads far above install counters.

## What This Changes

The issue asks for the public skill page to show downloads alongside installs or clearly explain why the visible Installs count differs from downloads.

## Best Possible Solution

Keep install-based adoption primary if that remains the product direction, then add a maintainer-approved accessible explanation or secondary downloads metric on skill detail pages and update docs/tests together.

## Reproduction Assessment

Yes, at source and live-data level: the reported skill returns downloads far above install counters, and current main renders only formattedStats.installsAllTime as Installs on the detail page. I did not run a local browser instance.

## Solution Assessment

Unclear until maintainers choose the metric presentation. A narrow explanation or secondary downloads metric is safer than reverting the intentional installs-first public adoption signal.
