## Summary

Keep open. Current main intentionally renders telemetry-derived all-time installs as the large public skill-page metric and still exposes downloads separately through API/CLI data, but the page does not show downloads or explain the semantic gap the reporter hit.

## What This Changes

Request that the public skill detail page distinguish archive downloads from current/all-time telemetry-confirmed installs.

## Best Possible Solution

Settle the public metric contract, then update the skill detail page, docs, and focused UI tests so publishers can see why downloads and telemetry installs differ.

## Reproduction Assessment

Yes, by source inspection with the reporter's stats shape. Any skill where downloads differ from `installsAllTime` will show the large detail-page metric from `installsAllTime` while CLI/API JSON can show downloads separately.

## Solution Assessment

Unclear until maintainers choose the public metric semantics. The narrow implementation is likely small, but the durable fix should avoid silently replacing one confusing metric with another.
