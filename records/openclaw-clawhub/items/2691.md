## Summary

Keep open: current main and the latest tagged release still show all-time installs as the public skill detail stat without showing downloads or an in-page explanation, so the central clarification request remains unresolved and needs maintainer UX direction.

## What This Changes

The issue asks the public skill detail page to distinguish archive downloads from telemetry install counts instead of showing only a low `Installs` value.

## Best Possible Solution

Keep install-based adoption as the primary metric, then add a maintainer-approved accessible explanation or secondary downloads metric on skill detail pages and update docs/tests together.

## Reproduction Assessment

Yes, at source and live-data level: the reported skill returns downloads far above install counters, and current main renders only `formattedStats.installsAllTime` as `Installs` on the detail page. I did not run a local browser instance.

## Solution Assessment

Unclear until maintainers choose the metric presentation. A narrow explanation or secondary downloads metric is safer than reverting the installs-first public adoption signal.
