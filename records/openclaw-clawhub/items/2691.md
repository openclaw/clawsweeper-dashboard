## Summary

Keep open: current main still renders the skill detail page's primary public stat as all-time Installs, while downloads remain available in the data but are not shown or explained on that page. This is a product/UX metric-clarity decision, not an already-implemented fix.

## What This Changes

The issue asks the public skill page to distinguish downloads from telemetry-derived installs or explain why the two counts differ.

## Best Possible Solution

Keep installs as the primary public adoption signal, then add maintainer-approved detail-page copy or a secondary Downloads metric that defines all-time/current installs versus archive downloads.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainers decide the public metric hierarchy. A narrow accessible explanation or secondary metric is safer than reverting downloads to the primary adoption count.
