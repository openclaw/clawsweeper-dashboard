## Summary

Keep open. ClawHub cleanup policy keeps issues open in this lane, and current main still lacks durable test/spec/fixture semantics in the LLM and static scanner paths even though the two named live examples currently return no moderation block from the public API.

## What This Changes

The issue asks ClawHub to remove or refine security-audit false positives that flag defensive security skills containing malicious-looking test strings and can block automated installs or updates.

## Best Possible Solution

Define and document scanner semantics for test/spec/fixture/mock/example content, apply them consistently to LLM and deterministic static scans, keep runtime and install-executed files in scope, and ensure staff-reviewed false-positive clearance is reflected in the API and CLI install/update behavior.
