## Risks / Open Questions

- The current package file endpoint is deliberately text-only and returns 415 for PNG assets, so rewriting relative images to that endpoint without a new binary asset route would not fix the broken images.
- A binary package asset endpoint needs explicit content-type, scan-gating, cache, CSP, SVG, and private-package authorization behavior before inline README rendering is safe.
- A source/raw URL rewrite needs stable release metadata, ideally source commit and package subdirectory, so published package pages do not drift when the source branch changes.
