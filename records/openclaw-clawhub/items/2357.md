## What This Changes

The issue asks ClawHub to rewrite relative image/resource URLs in plugin README Markdown so package detail page images resolve to package or source assets instead of route-relative 404 URLs.

## Best Possible Solution

Resolve package README relative image/resource URLs to version-stable, sanitized asset URLs through a maintainer-approved binary package asset endpoint or stable source/raw URL strategy.

## Risks / Open Questions

- The current package file endpoint is deliberately text-only and returns 415 for PNG assets, so a naive rewrite to /api/v1/packages/{name}/file would leave README images broken.
- A binary package asset endpoint needs explicit content-type, scan-gating, cache, CSP, SVG, and private-package authorization behavior before it is safe to expose inline.
- A source/raw URL rewrite needs stable release metadata, preferably source commit and package subdirectory, so published package pages do not drift when the source branch changes.
