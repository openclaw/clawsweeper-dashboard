## Summary

This should stay open: the report matches current main and live production, and ClawHub issue policy does not auto-close issues. The bug is real, but the repair needs a maintainer choice because the existing package file endpoint is text-only while README images need a binary-safe or stable source/raw asset strategy.

## What This Changes

Plugin README relative images are reported to render unchanged on plugin detail pages, causing browser-resolved image routes to 404.

## Best Possible Solution

Resolve package README relative image/resource URLs to version-stable, sanitized asset URLs, either through a binary-safe package asset endpoint or a clearly supported stable source/raw URL strategy.

## Reproduction Assessment

Yes. Live production reproduces the 404, and current main source shows package README text reaches MarkdownPreview while relative image URLs are intentionally preserved; I did not run a local browser server in this read-only review.

## Solution Assessment

Unclear. The requested outcome is correct, but maintainers should choose the asset strategy before implementation because the existing package file API is text-only and source/raw rewriting has metadata stability tradeoffs.
