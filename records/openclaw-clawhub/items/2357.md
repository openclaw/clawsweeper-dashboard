## Summary

This issue should stay open: current main still passes plugin README Markdown through the generic renderer without package asset context, and the image proxy intentionally leaves relative paths unchanged, so normal README-relative images still resolve under the ClawHub route and 404.

## What This Changes

The issue asks ClawHub to rewrite plugin README relative image/resource URLs to stable package or source asset URLs instead of emitting route-relative links.

## Best Possible Solution

Resolve README-relative image/resource URLs through a version-stable, sanitized asset strategy chosen by maintainers, then add regression coverage for plugin README rendering.

## Reproduction Assessment

Yes. Source inspection on current main shows README Markdown is rendered without package asset context while relative image URLs are explicitly preserved by the image proxy, matching the reported route-relative 404 behavior.

## Solution Assessment

Unclear. The requested behavior is correct, but the implementation strategy should be selected by maintainers because serving package assets and rewriting to source/raw URLs have different stability and trust tradeoffs.
