## Summary

This is a concrete current rendering bug: the plugin detail page renders package README image URLs as relative paths, and the affected production page currently emits those paths and returns 404 for the resolved image route. The issue should stay open under the ClawHub policy for issues, with maintainer review needed for the safest asset-serving strategy.

## What This Changes

The issue requests that plugin README relative image/resource URLs be rewritten to stable package or source asset URLs when rendered on plugin detail pages.

## Best Possible Solution

Package README rendering should resolve relative image/resource URLs to version-stable, sanitized asset URLs without requiring authors to rewrite normal Markdown or weakening untrusted content handling.

## Reproduction Assessment

Yes, there is a high-confidence source and production reproduction path: the live affected page emits relative README image URLs, the resolved route 404s, and current main still routes plugin README text through MarkdownPreview where relative images are intentionally left unchanged. I did not run a local current-main browser server in this read-only review, so this is source-reproducible rather than marked fully reproduced.

## Solution Assessment

Unclear which implementation is best before maintainer input: the requested outcome is correct, but the current package file API is text-only and rejects binary assets, so maintainers should choose between a safe package asset endpoint, generated-branch raw URLs, or source-commit raw URLs before implementation.
