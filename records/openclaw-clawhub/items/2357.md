## Summary

Keep open: the issue is valid on the current plugin README rendering path, and ClawHub cleanup policy does not auto-close open issues without a merged fix. Source inspection and live checks show relative README image paths still pass through unchanged, while the existing package file endpoint is text-only and returns 415 for the referenced PNG assets.

## What This Changes

Plugin README rendering should rewrite relative image/resource URLs to a stable package or source asset URL so images included in published packages do not 404 on plugin detail pages.

## Best Possible Solution

Choose a version-stable, sanitized package README asset strategy, then resolve plugin README relative image/resource URLs through that path with focused regression coverage.

## Reproduction Assessment

Yes. The affected live plugin page still emits route-relative README image URLs, that route returns 404, the package asset exists at the generated raw URL, and current source shows relative img src values are not rewritten.

## Solution Assessment

Unclear. The requested behavior is right, but maintainers need to choose the durable asset model because the current package file endpoint is text-only and returns 415 for PNGs.
