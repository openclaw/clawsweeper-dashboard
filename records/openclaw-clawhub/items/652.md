## Summary

Keep open. Current main still creates skill-backup Git commits without explicit author or committer attribution, and ClawHub cleanup policy keeps issues open unless a close reason is strongly satisfied; this one still needs maintainer attribution and privacy policy decisions.

## What This Changes

The issue asks for skill addition/update commits mirrored to the skills repository to use the owner’s GitHub account details as the Git commit author instead of the bot identity.

## Best Possible Solution

Define a maintainer-approved attribution policy, then thread a privacy-safe GitHub author object through publish-time and bulk backup paths into both skill and metadata commit payloads with tested bot fallback behavior.

## Reproduction Assessment

Yes, at source level: current main’s backup params and both /git/commits payloads lack author or committer fields, while publish and bulk paths pass only ownerHandle. This is a feature/policy request rather than a failing documented bug reproduction.

## Solution Assessment

Unclear until maintainers choose attribution and privacy semantics. The maintainable path should use a durable GitHub identity source plus an approved public or noreply-style email value, not ownerHandle or stored private email alone.
