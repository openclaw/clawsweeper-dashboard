## Summary

Keep open. Current main still creates skills backup commits without explicit Git author or committer attribution, and this is a feature/privacy policy request rather than a cleanup-close candidate for an open ClawHub issue.

## What This Changes

Use the skill owner’s GitHub account details as the author for skill addition/update commits in the skills backup repository instead of the ClawHub bot identity.

## Best Possible Solution

Define a maintainer-approved attribution policy, then thread a privacy-safe Git author object through publish-time and bulk backup paths into both skill and metadata commit payloads with tested bot fallback behavior.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature and policy request. Source inspection on current main confirms the relevant backup paths still have no explicit Git author or committer data.

## Solution Assessment

Unclear until maintainers choose the attribution and privacy semantics. The maintainable direction is a stable GitHub identity source plus an approved public or noreply-style email value, not ownerHandle or stored email alone.
