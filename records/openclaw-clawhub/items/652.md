- `P2`: This is a normal-priority provenance improvement with limited runtime impact but a meaningful product-policy boundary.
- `impact:security`: The change affects public identity disclosure and the audit meaning of repository commit history.

## Summary

Current main still creates skill-content, metadata, and deletion commits without explicit human author metadata, so the requested capability remains necessary; however, implementation depends on a maintainer decision about whether history represents the publishing actor, the owned publisher, or automated system work, and how to avoid disclosing private email addresses.

## What This Changes

Attribute exported skill repository commits to an appropriate authenticated GitHub user instead of always showing the ClawHub GitHub App bot.

## Best Possible Solution

Define and document an actor-versus-owner attribution hierarchy, use a verified public or GitHub noreply address for eligible human actions, retain bot attribution for system operations, and preserve stable internal actor identity separately for auditing.

## Reproduction Assessment

Yes, at source level: all three current-main GitHub commit payloads omit author and committer metadata, causing the GitHub App identity to remain visible.

## Solution Assessment

No, using the authenticated GitHub user directly is not a complete solution until maintainers define actor-versus-owner semantics, privacy-safe email selection, and bot fallback behavior.
