## Summary

Keep open. Current main still does not thread Git author or committer metadata into the skills backup commit path, and as an open ClawHub issue this should remain for maintainer policy follow-up rather than cleanup closure.

## What This Changes

The issue requests that skill addition/update commits in the skills backup repository use the owner’s GitHub account details as the Git author instead of the ClawHub bot.

## Best Possible Solution

Define the attribution policy, then pass a privacy-safe Git author object through publish-time and bulk backup paths into both backup commits with explicit bot fallback behavior and focused tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms the current backup path omits author/committer fields, but there is no broken existing contract to reproduce as a bug.

## Solution Assessment

Unclear until maintainer policy is set. The requested direction is reasonable, but the maintainable implementation needs approved attribution semantics and a privacy-safe author email source.
