## Summary

Keep open. Current main still creates GitHub skill backup commits without author or committer metadata, and the remaining change needs a maintainer policy decision for attribution semantics and privacy-safe public email handling.

## What This Changes

The request is to author mirrored skill add/update commits with the skill owner’s GitHub identity rather than the backup GitHub App bot.

## Best Possible Solution

Define a maintainer-approved attribution policy, then thread a privacy-safe Git author object through publish-time and bulk backup paths into both skill and metadata commit payloads with explicit bot fallback behavior and tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is an attribution feature and policy request. Source inspection gives high confidence that current main still does not pass author or committer data into the GitHub commit payloads.

## Solution Assessment

Unclear until maintainers choose attribution and privacy semantics. The narrow maintainable direction is to derive a stable GitHub identity plus an approved public or noreply-style email value, with bot fallback behavior.
