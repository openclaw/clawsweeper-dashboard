## Summary

Keep open. Current main still creates skill backup commits without explicit Git author or committer fields, and this ClawHub issue needs maintainer attribution and privacy policy before implementation.

## What This Changes

The issue requests that skill addition/update commits mirrored to the skills repository use the owner’s GitHub account details as the Git author instead of the ClawHub bot identity.

## Best Possible Solution

Define a maintainer-approved attribution policy, then pass a privacy-safe Git author object through publish-time and bulk backup paths into both skill and metadata commit payloads with explicit fallback behavior and tests.

## Reproduction Assessment

Yes for source-level verification: current main’s backup commit payloads do not include author or committer fields for either the skill commit or metadata commit. This is a feature and policy request rather than a failing documented bug reproduction.

## Solution Assessment

Unclear until maintainers choose attribution and privacy semantics. The code path is narrow, but the correct identity source, email address, and fallback behavior need product/security policy first.
