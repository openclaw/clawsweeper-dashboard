## Summary

Keep open. Current main still omits author/committer identity from skill backup commits, and implementation needs maintainer attribution/privacy policy because Git author metadata and emails become public history.

## What This Changes

The issue asks ClawHub to attribute skill backup commits in the skills mirror to the owner’s GitHub identity instead of the backup bot.

## Best Possible Solution

Define a maintainer-approved attribution policy, then thread a privacy-safe Git author object through publish-time and bulk backup paths into both skill and metadata commit payloads with explicit bot fallback coverage.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature and policy request. Source inspection confirms current publish and bulk backup paths do not pass author or committer data into GitHub commit payloads.

## Solution Assessment

Unclear until maintainers choose attribution and privacy semantics. The narrow maintainable direction is a stable GitHub identity source plus an approved public or noreply-style email value, with bot fallback behavior.
