## Summary

Keep open. Current main and v0.17.0 still create skill backup commits without explicit Git author or committer identity, and the remaining work needs maintainer policy for attribution semantics and privacy-safe email handling.

## What This Changes

Request skill addition/update backup commits in the skills repository use the owner’s GitHub identity instead of the GitHub App bot identity.

## Best Possible Solution

Define a maintainer-approved attribution policy, then thread a privacy-safe Git author object through publish-time and bulk backup paths into both skill and metadata commit payloads with explicit bot fallback behavior and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature and policy request. Source inspection gives high confidence that current main does not pass author or committer data into GitHub backup commit payloads.

## Solution Assessment

Unclear until maintainers choose attribution and privacy semantics. The narrow maintainable direction is to derive a stable GitHub identity plus an approved public or noreply-style email value, with bot fallback behavior.
