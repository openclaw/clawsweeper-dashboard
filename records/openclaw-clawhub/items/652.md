## Summary

Keep open. Current main still creates skill backup commits without Git author or committer metadata, and the remaining change needs maintainer policy for attribution semantics and privacy-safe public email handling.

## What This Changes

The issue asks for skill addition/update commits in the skills backup repository to use the skill owner’s GitHub identity as the Git author instead of the bot identity.

## Best Possible Solution

Define the attribution policy first, then thread a privacy-safe Git author object through publish-time and bulk backup paths into both skill and metadata commits with explicit bot fallback behavior and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature and policy request. Source inspection still gives high confidence that current main does not pass author or committer data into the GitHub commit payloads.

## Solution Assessment

Unclear until maintainers choose attribution and privacy semantics. The narrow maintainable direction is to derive a stable GitHub identity plus approved public or noreply-style email value, with bot fallback behavior.
