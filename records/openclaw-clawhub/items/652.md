## Summary

Keep open. Current main still creates GitHub backup commits without explicit author or committer data, and changing this needs maintainer policy on attribution semantics and privacy-safe Git author email handling.

## What This Changes

The issue requests that skill addition/update commits in the skills backup repository use the owner’s GitHub author identity instead of the ClawHub bot identity.

## Best Possible Solution

Define a maintainer-approved attribution policy, then thread a privacy-safe Git author object through publish-time and bulk backup flows into both skill and metadata commit payloads with explicit bot fallback behavior and focused tests.

## Reproduction Assessment

Yes for the current behavior: source inspection on current main shows the backup paths carry only ownerHandle and both GitHub /git/commits payloads omit author and committer fields. This verifies the requested capability is absent rather than reproducing a broken documented contract.

## Solution Assessment

Unclear until maintainers decide the policy. The requested direction is reasonable, but the maintainable implementation depends on approved attribution semantics and a privacy-safe Git author email source.
