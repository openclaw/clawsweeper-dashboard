## Summary

Keep open. Current main still creates skill backup commits without passing Git author or committer metadata, and the remaining work needs maintainer policy for attribution semantics and privacy-safe email handling.

## What This Changes

Request that skill backup commits use the relevant GitHub user identity as the Git author instead of the GitHub App/bot identity.

## Best Possible Solution

Define an attribution policy first, then pass a privacy-safe Git author object through publish-time and bulk backup paths with explicit bot fallback behavior and tests for personal owners, org publishers, missing identity, and bulk sync.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature and policy request. Source inspection gives high confidence that current publish and bulk backup paths do not pass author or committer data into GitHub commit payloads.

## Solution Assessment

Unclear until maintainers choose attribution and privacy semantics. The narrow maintainable direction is to derive a stable GitHub identity plus approved public or noreply-style email value, with bot fallback behavior.

Status: needs_attention
