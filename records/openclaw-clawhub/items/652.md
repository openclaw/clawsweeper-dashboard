## Summary

Keep open. Current main still creates skill backup commits without explicit Git author or committer data, and the remaining work is a product/privacy attribution decision rather than a cleanup-close case for a ClawHub issue.

## What This Changes

Request skill addition/update backup commits in the skills repository to use the owner’s GitHub identity as the Git author instead of the ClawHub bot.

## Best Possible Solution

Define an attribution policy, then thread a privacy-safe Git author object through publish-time and bulk backup paths into both skill and metadata commit payloads with explicit bot fallback behavior and regression coverage.

## Reproduction Assessment

Yes for source-level verification. Current main passes only ownerHandle into the publish and bulk backup paths, and both GitHub create-commit payloads omit author and committer fields.

## Solution Assessment

Unclear until maintainers set attribution and email privacy policy. The code path is narrow, but the maintainable solution depends on the approved identity source, org-owner semantics, and fallback behavior.
