## Summary

Keep open. Current main still threads only `ownerHandle` into the skill backup pipeline and both GitHub create-commit payloads omit explicit `author` or `committer` metadata, while the remaining work needs maintainer policy for attribution semantics and privacy-safe email handling.

## What This Changes

Request that ClawHub skill backup commits in the skills repository use the skill owner's GitHub identity instead of the GitHub App or bot author.

## Best Possible Solution

Define a maintainer-approved attribution model and privacy-safe author email source, then thread an explicit Git author object through publish-time and bulk backup paths with bot fallbacks and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature and policy request. Source inspection gives high confidence that current main still lacks author and committer fields in the GitHub backup commit payloads.

## Solution Assessment

Unclear until maintainers choose attribution and privacy semantics. The narrow maintainable direction is explicit privacy-safe author metadata, not deriving Git identity from mutable `ownerHandle` or raw stored email.
