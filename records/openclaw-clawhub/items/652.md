## Summary

Keep open. Current main still creates skill backup commits without a Git author or committer identity, and the remaining work needs maintainer policy on attribution semantics and privacy-safe public Git author email handling.

## What This Changes

Request that skill addition/update commits in the skills backup repository use the skill owner’s GitHub identity instead of the ClawHub bot identity.

## Best Possible Solution

Define the attribution and public-email policy first, then thread a privacy-safe Git author object through publish-time and bulk backup paths with explicit bot fallback behavior and regression tests.

## Reproduction Assessment

Yes for source-level verification: current main’s backup commit payloads do not include author or committer fields for either the skill commit or metadata commit. This is a feature/policy request rather than a failing documented bug reproduction.

## Solution Assessment

Unclear until maintainers choose attribution and privacy semantics. After that decision, the narrow maintainable fix is to add a sanitized author object at the backup boundary and cover personal owners, publisher/org owners, missing identity, and bulk sync.
