## Summary

Keep open. Current main still backs up skill commits with bot/app-authored GitHub commit payloads, and ClawHub issue cleanup policy does not support closing this open issue; the remaining work needs maintainer policy on attribution and public author email handling.

## What This Changes

Request that skill addition and update commits in the public skills backup repository use the owner’s GitHub identity as the Git author instead of the bot account.

## Best Possible Solution

Define a maintainer-approved attribution policy, then thread a privacy-safe Git author object through publish-time and bulk backup paths with explicit fallback behavior and regression coverage.

## Reproduction Assessment

Yes for source-level verification: current main’s backup commit payloads do not include author or committer fields on either the skill or metadata commit. This is a feature/policy request rather than a broken documented behavior reproduction.

## Solution Assessment

Unclear until maintainers decide the attribution and privacy semantics. The requested direction is reasonable, but the narrow maintainable solution needs an approved identity source, email policy, and fallback model first.
