## Summary

Keep open. Current main still creates skill backup commits without a supplied Git author or committer, and this issue needs maintainer policy for attribution semantics and privacy-safe public email handling before implementation.

## What This Changes

Request that skill backup commits in the skills repository use the skill owner's GitHub identity as the Git author instead of the ClawHub bot.

## Best Possible Solution

Define an approved attribution policy, then thread a privacy-safe GitHub author object through publish-time and bulk backup paths into both skill and metadata commits with explicit bot fallback behavior.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature and policy request. Source inspection confirms current publish and bulk backup paths do not pass author or committer data into GitHub commit payloads.

## Solution Assessment

Unclear until maintainers choose attribution and privacy semantics. The narrow maintainable direction is a stable GitHub identity source plus an approved public or noreply-style email value, with explicit fallback behavior.
