## Summary

Keep open. Current main still creates GitHub skill-backup commits without explicit author or committer identity, and ClawHub cleanup policy keeps issues open unless they meet a close case; this remaining change needs maintainer attribution and privacy policy before implementation.

## What This Changes

Request skill addition/update commits mirrored to the skills repository use the owner’s GitHub author identity instead of the ClawHub bot identity.

## Best Possible Solution

Define a maintainer-approved attribution policy, then pass a privacy-safe Git author object through publish-time and bulk backup paths into both skill and metadata commit payloads with explicit bot fallback tests.

## Reproduction Assessment

Yes, at source level: current main’s backup params and both GitHub /git/commits payloads lack author or committer fields, while publish and bulk sync only pass ownerHandle. This is a feature and policy request rather than a failing documented bug reproduction.

## Solution Assessment

Unclear until maintainers choose attribution and privacy semantics. The maintainable implementation should use a durable GitHub identity source plus a safe public email value, not ownerHandle or stored private email alone.
