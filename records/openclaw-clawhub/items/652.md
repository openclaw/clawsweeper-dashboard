## Summary

Keep open. Current main still creates GitHub skill backup commits without author or committer data, and the request needs maintainer attribution and privacy policy before implementation because Git author identity becomes public repository history.

## What This Changes

The issue asks ClawHub to write skill backup commits using the relevant GitHub user identity instead of the bot identity.

## Best Possible Solution

Define an approved attribution policy, then thread a privacy-safe Git author object through publish-time and bulk backup paths into both skill and metadata commit payloads with explicit bot fallback behavior and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature and policy request. Source inspection confirms the current backup paths still do not pass author or committer data.

## Solution Assessment

Unclear until maintainers choose attribution and privacy semantics. The narrow maintainable direction is a stable GitHub identity source plus an approved public or noreply-style email value, with bot fallback.
