## Summary

Keep open. Current main still creates skill backup commits without Git author or committer metadata, and the remaining change needs maintainer policy for attribution semantics and privacy-safe public email handling.

## What This Changes

The issue requests using the skill owner’s GitHub identity as the Git author for skill backup commits instead of the ClawHub bot identity.

## Best Possible Solution

Define a maintainer-approved attribution policy, then thread a privacy-safe Git author object through publish-time and bulk backup paths into both skill and metadata commit payloads with explicit bot fallback behavior.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature and policy request. Source inspection gives high confidence that current publish and bulk backup paths do not pass author or committer data into GitHub commit payloads.

## Solution Assessment

Unclear until maintainers choose attribution and privacy semantics. The narrow maintainable direction is to derive a stable GitHub identity plus approved public or noreply-style email value, with bot fallback behavior.
