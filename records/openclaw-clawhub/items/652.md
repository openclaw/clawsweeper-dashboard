## Summary

Keep open. Current main still creates skill backup commits without explicit Git author or committer data, and the remaining decision is a feature/privacy policy choice rather than an issue-cleanup close case.

## What This Changes

Request that ClawHub use the skill owner’s GitHub identity as the author for mirrored skill addition/update commits instead of the backup bot identity.

## Best Possible Solution

Define an approved attribution policy, then thread a privacy-safe GitHub author object through publish-time and bulk backup paths into both skill and metadata commit payloads with tested bot fallback behavior.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature and privacy-policy request. Source inspection on current main confirms the relevant backup paths still have no explicit Git author or committer data.

## Solution Assessment

Unclear until maintainers choose the attribution and privacy semantics. The maintainable direction is a stable GitHub identity source plus an approved public or noreply-style email value, not ownerHandle or stored email alone.
