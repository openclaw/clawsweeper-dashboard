## Summary

Keep open. Current main still omits Git author and committer metadata from skill backup commits, and the remaining work needs maintainer policy for attribution semantics and privacy-safe public email handling.

## What This Changes

The issue requests that commits mirrored to the skills backup repository use the publishing GitHub user's author identity instead of the ClawHub bot identity.

## Best Possible Solution

Define an attribution policy, then thread a privacy-safe Git author object through publish-time and bulk backup paths into both skill and metadata commits with explicit bot fallback behavior and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature and policy request. Source inspection gives high confidence that current main does not pass author or committer data into the GitHub commit payloads.

## Solution Assessment

Unclear until maintainers choose attribution and privacy semantics. The narrow maintainable direction is to derive a stable GitHub identity plus approved public or noreply-style email value, with bot fallback behavior.
