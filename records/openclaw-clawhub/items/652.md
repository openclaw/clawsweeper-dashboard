## Summary

Current main still creates GitHub backup commits without publisher author metadata, and no merged or viable open item supersedes this request. The issue remains necessary, but implementation should wait for a maintainer decision covering immutable identity, private email handling, app committer attribution, and actorless maintenance commits.

## What This Changes

Use the authenticated publisher’s GitHub identity as the author of generated skill backup commits instead of attributing every commit solely to the GitHub App bot.

## Best Possible Solution

Define the attribution contract in a durable spec, snapshot immutable publisher identity at publication, derive a GitHub-recognized privacy-safe author address, retain the GitHub App as committer and fallback, and specify separate behavior for organization, scheduled, deletion, and legacy records.

## Reproduction Assessment

Yes, at source level. All three current Git commit creation paths omit author and committer objects, while the publish backup parameters contain no immutable GitHub identity from which publisher attribution could be constructed.

## Solution Assessment

No, the request as phrased is underspecified. The maintainable solution is a privacy-safe author-versus-committer contract with immutable identity and deterministic fallback rules, rather than copying unspecified GitHub profile details.
