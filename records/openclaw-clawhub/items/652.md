## Summary

Keep open. Current main still omits explicit Git author and committer data from skill backup commits, and this remains a product/privacy attribution feature rather than a cleanup-close candidate for a ClawHub issue.

## What This Changes

The issue requests that commits mirrored into the skills backup repository use the skill owner’s GitHub identity as the Git commit author instead of the ClawHub bot identity.

## Best Possible Solution

Define a maintainer-approved attribution policy, then thread a privacy-safe Git author object through publish-time and bulk backup paths into both skill and metadata commit payloads with explicit bot fallback behavior and tests.

## Reproduction Assessment

Yes for source-level verification: current main passes only ownerHandle into backup paths, and both GitHub create-commit payloads omit author and committer fields. This verifies the requested capability is still absent, although it is a feature/policy request rather than a broken documented contract.

## Solution Assessment

Unclear until maintainers set attribution and email privacy policy. The code path is narrow, but the maintainable solution depends on the approved identity source, org-owner semantics, and fallback behavior.
