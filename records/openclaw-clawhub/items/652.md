## Summary

Keep open. Current main and v0.20.0 still create skill backup commits with GitHub create-commit payloads that include only message, tree, and parents, so GitHub-user author attribution is not implemented; the remaining blocker is maintainer policy for whose identity and which privacy-safe email may be written to public Git history.

## What This Changes

Request that skill addition/update backup commits in the skills repository use the owning GitHub user's author details instead of the ClawHub bot/App identity.

## Best Possible Solution

Define the attribution and email privacy policy, then thread an approved privacy-safe Git author object through publish-time and bulk backup commits with explicit bot fallback tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a security-sensitive attribution feature/policy request. Source inspection gives high confidence that current main still omits author and committer metadata from the affected commits.

## Solution Assessment

Unclear until maintainers choose the attribution semantics and email source. The maintainable path is to derive an approved public or noreply-style GitHub identity and preserve bot fallback behavior for bulk or missing-identity paths.
