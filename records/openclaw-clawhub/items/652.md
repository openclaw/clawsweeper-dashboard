## Summary

Keep open. Current main and the latest release still create public skills-backup commits without explicit Git author or committer metadata, and the remaining work needs maintainer policy for attribution semantics and privacy-safe email sourcing.

## What This Changes

Request that skill addition/update commits in the public skills backup repository use the owning GitHub user's author identity instead of the ClawHub bot.

## Best Possible Solution

Define the attribution and privacy policy first, then thread an explicit privacy-safe Git author object through publish-time and bulk backup paths into both skill and metadata commits with tested bot fallback behavior.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature and policy request. Source inspection gives high confidence that current main does not pass author or committer data into the GitHub backup commit payloads.

## Solution Assessment

Unclear until maintainers choose attribution and privacy semantics. The narrow maintainable direction is to derive a stable GitHub identity plus an approved public or noreply-style email value, with explicit bot fallback behavior.
