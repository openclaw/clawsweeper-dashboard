## Summary

Keep open. This is a valid ClawHub product request, current main still creates skill-backup commits without explicit Git author or committer metadata, and ClawHub cleanup policy keeps issues open unless a terminal close case applies.

## What This Changes

The issue asks ClawHub to attribute skill mirror commits in the skills repository to the relevant GitHub user instead of the ClawHub bot.

## Best Possible Solution

Define the attribution policy, then thread a privacy-safe Git author object through publish-time and bulk backup paths into both GitHub commit payloads with explicit bot fallback behavior and focused tests.

## Reproduction Assessment

Yes. Source inspection on current main shows the publish and bulk backup paths pass only ownerHandle, and both GitHub create-commit payloads omit author and committer fields.

## Solution Assessment

Unclear until maintainers decide the policy. The requested direction is reasonable, but the narrow maintainable fix depends on approved attribution semantics and a privacy-safe author email source.

## Risks / Open Questions

- Maintainers still need to choose whether the Git author should represent the publishing actor, personal owner, organization/publisher owner, or bot fallback.
- Using stored email directly as a Git author email could expose private addresses; a safe implementation likely needs a GitHub noreply-style address or another approved source.
- Bulk sync/backfill paths may not have a clear human publishing actor, so their fallback attribution needs explicit policy.
