## Summary

Keep open. Current main still creates skill backup commits without explicit Git author or committer metadata, and this ClawHub issue needs a maintainer attribution/privacy policy before code changes choose public identity semantics.

## What This Changes

The issue asks ClawHub to author skill backup commits in the public skills repository with the skill owner’s GitHub identity instead of the ClawHub bot.

## Best Possible Solution

Define a maintainer-approved attribution policy, then thread a privacy-safe Git author object through publish-time and bulk backup paths into both skill and metadata commit payloads with explicit bot fallback behavior and tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature and policy request. Source inspection confirms current publish and bulk backup paths do not pass author or committer data into GitHub commit payloads.

## Solution Assessment

Unclear until maintainers choose attribution and privacy semantics. The narrow maintainable direction is to use a stable GitHub identity source plus an approved public or noreply-style email value, with bot fallback behavior.
