## Best Possible Solution

Define the attribution policy first, then thread a privacy-safe Git author object through publish-time and bulk backup paths into both skill and metadata commit payloads with explicit fallback behavior and focused tests.

## Risks / Open Questions

- Maintainers still need to decide whether Git author attribution should represent the publishing actor, the personal owner, an organization/publisher owner, or a bot fallback for bulk sync and restore paths.
- Using stored user email directly as a Git author email could expose private addresses; a safe implementation likely needs GitHub noreply-style attribution or another approved public identity source.
- ownerHandle alone is not enough for stable attribution because publisher/org ownership and renamed handles can diverge from the Git identity GitHub needs for linked commits.
