## Summary

Keep open. Current main still creates skill backup commits without explicit Git author or committer metadata, and this needs maintainer policy on attribution semantics plus privacy-safe public email handling before implementation.

## What This Changes

The issue requests that ClawHub skill mirror commits use the skill owner's GitHub identity as the Git author instead of the GitHub App or bot identity.

## Best Possible Solution

Define an approved attribution and email-privacy policy, then thread a safe Git author object through publish-time and bulk backup paths with explicit bot fallback behavior and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature and policy request. Source inspection gives high confidence that current publish and bulk backup paths do not pass author or committer data into GitHub commit payloads.

## Solution Assessment

Unclear until maintainers choose the attribution and privacy semantics. The narrow maintainable direction is to use a stable GitHub identity source plus an approved public or noreply-style email value, with bot fallback behavior.
