## Summary

Keep open. Current main still creates skill backup commits without author or committer identity, and the remaining fix needs maintainer policy for attribution semantics and privacy-safe public email handling.

## What This Changes

The issue asks ClawHub to use the skill owner’s GitHub identity as the author for skill backup commits instead of the GitHub App or bot identity.

## Best Possible Solution

Define a maintainer-approved attribution policy, then pass a privacy-safe Git author object through publish-time and bulk backup paths into both skill and metadata commits with explicit bot fallback behavior.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature and policy request. Source inspection confirms current publish and bulk backup paths do not pass author or committer data into GitHub commit payloads.

## Solution Assessment

Unclear until maintainers choose attribution and privacy semantics. The narrow maintainable direction is a stable GitHub identity source plus an approved public or noreply-style email value, with bot fallback behavior.
