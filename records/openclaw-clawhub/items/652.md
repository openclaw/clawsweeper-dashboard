## Summary

Keep open. Current main still creates skill backup commits without threading a user Git author or committer identity into the GitHub commit payloads, and the remaining work needs maintainer policy on attribution semantics and privacy-safe public email handling.

## What This Changes

The issue asks ClawHub to attribute mirrored skill repository commits to the publishing GitHub user instead of the ClawHub bot.

## Best Possible Solution

Define a maintainer-approved attribution policy, then pass a privacy-safe Git author object through publish-time and bulk backup paths into both skill and metadata commit payloads with explicit bot fallback behavior.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature and policy request. Source inspection gives high confidence that current main does not pass author or committer data into the GitHub backup commit payloads.

## Solution Assessment

Unclear until maintainers choose attribution and privacy semantics. The narrow maintainable direction is to derive a stable GitHub identity plus an approved public or noreply-style email value, with bot fallback behavior.
