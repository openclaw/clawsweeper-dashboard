## Summary

Keep open. Current main still creates the skills backup commits without explicit Git author or committer identity, and ClawHub cleanup policy keeps issues open unless a close reason is clearly satisfied.

## What This Changes

The issue asks for skill addition/update commits in the skills backup repository to use the owner's GitHub account details as the Git author instead of the ClawHub bot.

## Best Possible Solution

Define an approved attribution policy, then thread a privacy-safe Git author object through publish-time and bulk backup paths into both skill and metadata commit payloads with explicit bot fallback behavior.

## Reproduction Assessment

Yes for source-level verification: current main passes only ownerHandle into backup paths, and both GitHub create-commit payloads omit author and committer fields. This confirms the requested capability is still absent rather than a broken documented contract.

## Solution Assessment

Unclear until maintainers set the attribution and email privacy policy. The narrow code path is identifiable, but the maintainable solution depends on the approved identity source and fallback semantics.
