## Summary

This request remains necessary because current main does not implement publisher-attributed Git commits. The desired credit is reasonable, but maintainers must first define trusted identity fields, author-versus-committer semantics, and ownership of the external repository-writing automation.

## What This Changes

Use the authenticated publisher’s verified GitHub identity as the author of generated skill repository commits instead of attributing every commit solely to the ClawHub bot.

## Best Possible Solution

Define a documented provenance contract in which automation remains the committer and the authenticated publisher is credited as author only through stable, verified GitHub-derived identity data, then implement it at the actual repository-write boundary with integration coverage.

## Reproduction Assessment

Yes. The linked openclaw/skills history provides a high-confidence observation that generated commits use the bot identity; no interactive runtime setup is needed to reproduce that metadata behavior.

## Solution Assessment

Unclear. Crediting the publisher is reasonable, but directly substituting GitHub account details is not the safest maintainable solution until maintainers define verified identity fields and author-versus-committer semantics.
