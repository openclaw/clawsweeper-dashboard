## Summary

Keep open. Current main still creates skill backup commits without explicit Git author or committer metadata, and the remaining work needs maintainer policy on attribution semantics and privacy-safe public email handling.

## What This Changes

The issue requests that commits mirrored to the openclaw/skills repository use the skill owner's GitHub identity instead of the ClawHub bot identity.

## Best Possible Solution

Define a maintainer-approved attribution policy, then thread a privacy-safe Git author object through publish-time and bulk backup paths into both skill and metadata commit payloads with explicit bot fallback behavior and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature and policy request. Source inspection gives high confidence that current main does not pass author or committer data into the GitHub backup commit payloads.

## Solution Assessment

Unclear until maintainers choose attribution and privacy semantics. The narrow maintainable direction is to derive a stable GitHub identity plus an approved public or noreply-style email value, with bot fallback behavior.
