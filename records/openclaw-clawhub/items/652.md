## Summary

Keep open. Current main still creates skill backup commits without an explicit Git author or committer identity, and the remaining work needs maintainer policy on attribution and privacy-safe author email handling.

## What This Changes

Request that ClawHub write skill backup commits in the skills mirror with the owner or publishing GitHub user's author identity instead of the ClawHub bot identity.

## Best Possible Solution

Define the attribution policy first, then thread a privacy-safe Git author object through publish-time and bulk backup paths with explicit bot fallback behavior and regression coverage.

## Reproduction Assessment

Yes, for source-level verification: current main's backup params and both GitHub create-commit payloads lack author or committer fields. This is a feature/policy request rather than a failing documented bug path.

## Solution Assessment

Unclear until maintainers choose attribution and privacy semantics. The narrow implementation path is clear, but the correct identity source, email address, and fallback behavior need product/security policy first.
