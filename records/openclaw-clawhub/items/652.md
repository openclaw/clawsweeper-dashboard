## Summary

Keep open. Current main still creates skill backup commits without a Git author or committer object, and the remaining work is a product/security decision about public attribution and privacy-safe author email handling.

## What This Changes

The issue asks for ClawHub skill backup commits in the skills mirror repository to use the skill owner’s GitHub identity instead of the GitHub App/bot identity.

## Best Possible Solution

Define an approved attribution policy, then pass a privacy-safe Git author object through publish-time and bulk backup paths into both skill and metadata commit payloads with explicit bot fallback behavior and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature and policy request. Source inspection confirms the current backup paths still do not pass author or committer data.

## Solution Assessment

Unclear until maintainers choose attribution and privacy semantics. The narrow maintainable path is a stable GitHub identity source plus an approved public or noreply-style email value, with bot fallback.
