## Summary

Keep open. Current main still has the authorless GitHub backup path, and this ClawHub issue needs maintainer product/security judgment because Git author identity and email become permanent public repository history.

## What This Changes

The issue requests that ClawHub skill mirror commits in the GitHub backup repository use the relevant GitHub user's author identity instead of the bot account.

## Best Possible Solution

Define an approved attribution and privacy policy, then pass a safe Git author object through publish-time and bulk backup paths into both skill and metadata commits with explicit bot fallback coverage.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature and policy request. Source inspection confirms the current publish and bulk backup paths do not pass author or committer data into GitHub commit payloads.

## Solution Assessment

Unclear until maintainers choose attribution and privacy semantics. The narrow maintainable direction is a stable GitHub identity source plus an approved public or noreply-style email value, with bot fallback behavior.
