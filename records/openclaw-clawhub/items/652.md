## Summary

Keep open. Current main still creates skill backup commits without explicit Git author or committer metadata, and the remaining work needs maintainer policy for attribution semantics and privacy-safe public email handling.

## What This Changes

Use the skill owner’s GitHub identity as the author for skill backup commits instead of the GitHub App bot identity.

## Best Possible Solution

Define an attribution policy first, then pass a privacy-safe GitHub author object through publish-time and bulk backup paths into both skill and metadata commit payloads with explicit bot fallback behavior.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature and policy request. Source inspection confirms current publish and bulk backup paths do not pass author or committer data into GitHub commit payloads.

## Solution Assessment

Unclear until maintainers choose attribution and privacy semantics. The narrow maintainable direction is a stable GitHub identity source plus an approved public or noreply-style email value, with bot fallback behavior.
