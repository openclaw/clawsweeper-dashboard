## Summary

The requested attribution remains absent from current main and v0.20.0. The issue is still necessary, but implementation depends on a maintainer decision about whose identity represents a publication and which privacy-safe email may enter permanent public Git history.

## What This Changes

Use the authenticated GitHub user’s identity as the author of generated skill repository commits instead of attributing every commit to the ClawHub bot.

## Best Possible Solution

Define authorship as an explicit publish-time identity using a verified public or GitHub noreply address, document organization and automation semantics in a focused spec, and preserve bot attribution whenever no human publishing actor exists.

## Reproduction Assessment

Yes from current-main source inspection: both generated skill commit payloads omit author and committer metadata, which deterministically leaves GitHub App bot attribution. A live repository mutation is unnecessary to establish the missing fields.

## Solution Assessment

Unclear until maintainers choose whether authorship represents the authenticated publishing actor or publisher owner and approve a privacy-safe email source; copying stored user identity directly is not a safe default.
