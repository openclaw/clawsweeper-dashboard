## Summary

Keep open. Current main still creates skill backup commits without explicit Git author or committer metadata, and this ClawHub issue needs maintainer policy on attribution semantics and privacy-safe public email handling before implementation.

## What This Changes

Skill addition/update backup commits in the skills repository should use the owner’s GitHub identity as the Git author instead of the ClawHub bot.

## Best Possible Solution

Define a maintainer-approved attribution policy, then thread a privacy-safe Git author object through publish-time and bulk backup paths into both skill and metadata commits with explicit bot fallback behavior and coverage for personal owners, org publishers, missing identity, and bulk sync.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature and policy request. Source inspection gives high confidence that the current publish and bulk backup paths do not pass author or committer data into GitHub commit payloads.

## Solution Assessment

Unclear until maintainers choose attribution and privacy semantics. The narrow maintainable direction is to use a stable GitHub identity source plus an approved public or noreply-style email value, with bot fallback behavior.
