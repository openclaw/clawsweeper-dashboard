## Summary

Keep open. Current main still creates skill backup Git commits without explicit author or committer attribution, and the remaining work needs maintainer policy on owner versus publisher attribution plus privacy-safe public Git email handling.

## What This Changes

Use the skill owner’s GitHub identity as the author for skill addition/update commits in the skills backup repository instead of the GitHub App/bot identity.

## Best Possible Solution

Define the attribution and public-email policy, then thread a privacy-safe Git author object through publish-time and bulk backup paths into both skill and metadata commit payloads with explicit bot fallback tests.

## Reproduction Assessment

Yes, at source level: current main’s backup params and both GitHub /git/commits payloads lack author or committer fields, and publish/bulk paths pass only ownerHandle. This is a feature and policy request rather than a failing documented bug reproduction.

## Solution Assessment

Unclear until maintainers choose attribution and privacy semantics. The maintainable implementation should use a durable GitHub identity source plus a safe public email value, not ownerHandle or stored private email alone.
