## Summary

Keep open. This is a ClawHub issue, current main still lacks explicit Git author/committer attribution in the skill backup commit path, and the remaining work needs maintainer policy on attribution semantics and privacy-safe email handling.

## What This Changes

Use the GitHub owner's account details as the author for skill addition/update backup commits in the skills mirror instead of the bot identity.

## Best Possible Solution

Define an attribution policy, then pass a privacy-safe Git author object through publish-time and bulk backup paths into both skill and metadata commit payloads with tested bot fallback behavior.

## Reproduction Assessment

Yes for the current source behavior: the backup paths carry only ownerHandle, and both GitHub create-commit payloads omit author and committer fields. This verifies the requested capability is absent rather than a broken documented contract.

## Solution Assessment

Unclear until maintainers decide the policy. The requested direction is reasonable, but the maintainable solution depends on approved attribution semantics and a privacy-safe Git author email source.
