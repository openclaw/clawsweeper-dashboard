## Summary

The requested behavior remains missing on current main: generated skill backup commits do not include the authenticated user’s Git author identity. The issue should remain open because implementing it safely requires an explicit maintainer decision about authorship, consent, privacy-safe email handling, organization publishers, and bot fallback semantics.

## What This Changes

Use the authenticated GitHub user’s identity as the author of generated skill backup commits rather than attributing them solely to the ClawHub bot.

## Best Possible Solution

Define publisher, Git author, and bot committer as distinct provenance roles, then thread a consented privacy-safe GitHub actor identity through every generated backup commit path with an explicit bot fallback and durable security-spec documentation.

## Reproduction Assessment

Yes, at high confidence from source inspection: current main constructs both generated Git commits without a user author or committer object, matching the reported bot-only attribution. This is a requested provenance capability rather than a regression against an established contract.

## Solution Assessment

Unclear pending a product decision. A consented user author with the ClawHub bot retained as committer and fallback is safer than replacing bot attribution without explicit privacy and ownership semantics.
