## Summary

Keep open. Current main still creates GitHub skill-backup commits without explicit Git author or committer attribution, and the remaining work requires maintainer policy for attribution semantics and privacy-safe public email handling.

## What This Changes

Requests that skill addition/update commits in the skills backup repository be authored with the skill owner's GitHub identity instead of the app/bot identity.

## Best Possible Solution

Define the attribution policy, then thread a privacy-safe Git author object through publish-time and bulk backup paths into both commit payloads with tested bot fallback behavior.

## Reproduction Assessment

Yes, at source level: current main’s backup params and both /git/commits payloads lack author or committer fields, while publish and bulk paths pass only ownerHandle. This is a feature/policy request rather than a failing documented bug reproduction.

## Solution Assessment

Unclear until maintainers choose attribution and privacy semantics. The maintainable path should use a durable GitHub identity source plus an approved public or noreply-style email value, not ownerHandle or stored private email alone.
