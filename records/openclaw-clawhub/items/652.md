## Summary

Keep open. Current main still creates skill backup commits without explicit Git author or committer attribution, and this feature needs maintainer policy for attribution semantics and privacy-safe email handling before implementation.

## What This Changes

Request skill addition/update commits in the public skills backup repository to use the owner’s GitHub identity instead of the bot identity.

## Best Possible Solution

Define an attribution policy, then thread a privacy-safe GitHub author object through publish-time and bulk backup paths into both skill and metadata commits with tested bot fallback behavior.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature and policy request. Source inspection confirms current main only passes ownerHandle and both GitHub create-commit payloads omit author and committer fields.

## Solution Assessment

Unclear until maintainers choose attribution and privacy semantics. The maintainable implementation should use a stable GitHub identity source plus an approved public or noreply-style email value, not ownerHandle or stored private email alone.
