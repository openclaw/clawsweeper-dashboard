## Summary

Keep open. This is a valid ClawHub feature/security-policy request, current main still creates skill backup commits without explicit Git author or committer metadata, and the target-repo cleanup policy keeps issues open rather than auto-closing them.

## What This Changes

The issue asks for skill addition and update commits in the skills backup repository to use the owner’s GitHub identity as the Git commit author instead of the bot identity.

## Best Possible Solution

Define a maintainer-approved attribution policy, then pass a privacy-safe Git author object through publish-time and bulk backup paths into both skill and metadata commit payloads with explicit bot fallback behavior and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature and policy request. Source inspection gives high confidence that the current publish and bulk backup paths do not pass author or committer data into GitHub commit payloads.

## Solution Assessment

Unclear until maintainers choose attribution and privacy semantics. The narrow maintainable direction is to use a stable GitHub identity source plus an approved public or noreply-style email value, with documented bot fallback behavior.
