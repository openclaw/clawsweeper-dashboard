## Summary

Keep open. Current main still creates skill backup commits without Git author or committer identity, and the remaining work needs maintainer attribution and privacy policy before implementation.

## What This Changes

Request skill addition/update commits in the external skills backup repository to use the owner’s GitHub identity instead of the backup bot identity.

## Best Possible Solution

Define the attribution and privacy policy, then pass a privacy-safe Git author object through publish-time and bulk backup paths into both skill and metadata commit payloads with explicit bot fallback behavior and regression tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this is a requested attribution behavior, not a documented broken contract. Source inspection does confirm current main does not pass author or committer data into the relevant GitHub commit payloads.

## Solution Assessment

No automated code-only fix is the best next step yet. The implementation surface is narrow, but maintainers should first choose the attribution source and privacy-safe author email policy.
