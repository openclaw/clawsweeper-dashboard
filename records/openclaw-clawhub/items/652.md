## Summary

Keep open. Current main still sends skill backup Git commit payloads without an explicit author or committer identity, and the remaining work needs maintainer policy on attribution semantics and privacy-safe public Git author email handling.

## What This Changes

Use the skill owner’s GitHub identity as the author for skill backup commits instead of the ClawHub bot identity.

## Best Possible Solution

Define the attribution policy, then thread a privacy-safe Git author object through publish-time and bulk backup paths into both skill and metadata commit payloads with tests for owner, publisher, missing-identity, and bulk-sync cases.

## Reproduction Assessment

Yes, for source-level verification: current main’s backup params and both GitHub /git/commits payloads lack author or committer fields. This is a feature and policy request rather than a failing documented bug reproduction.

## Solution Assessment

Unclear until maintainers choose attribution and privacy semantics. The code path is narrow, but the correct identity source, public email value, and fallback behavior need product/security policy first.
