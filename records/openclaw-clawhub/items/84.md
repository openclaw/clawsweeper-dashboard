## Summary

Keep open under the ClawHub cleanup policy because this is an issue, not a PR. Current main appears to have fixed the core CLI/API publish and GitHub import summary persistence path, and a later merged change added web summary editing, but production/backfill verification and any CLI or REST override scope remain maintainer decisions.

## What This Changes

The issue requests persisting SKILL.md frontmatter descriptions as skill summaries for CLI publish and GitHub import, with optional manual override/update surfaces.

## Best Possible Solution

Keep the shared server-side summary derivation, verify or run the existing backfill for old null rows, and split CLI --summary or REST summary-update support into narrower follow-up issues if those controls are desired.

## Solution Assessment
