## What This Changes

The issue reports that skill stars and downloads from the CLI or website do not update visible skill metrics.

## Best Possible Solution

Keep the shipped synchronous star-count path, use https://github.com/openclaw/clawhub/issues/2343 for focused download-freeze diagnostics, and have an operator inspect production event backlog, scheduler failures, and replay needs before changing counter cadence or semantics.

## Reproduction Assessment

No high-confidence failing current-main reproduction is available. Source inspection shows public download counters can lag behind successful downloads through hourly dedupe plus the 6-hour skill-doc-stat-sync path, but production logs/backlog are needed to prove stuck or dropped events.

## Solution Assessment

Unclear as a source fix. The synchronous star implementation is the narrow fix for stars, while downloads should first be diagnosed through production event backlog and scheduler logs rather than reviving the closed cadence-only approach from https://github.com/openclaw/clawhub/pull/176.
