## Summary

Keep open. Current main records download and star events and has a shipped async stat-sync pipeline, but the visible counters still depend on delayed Convex processing and the report cannot be closed under the ClawHub issue-cleanup policy without production backlog/log evidence.

## What This Changes

The issue reports that skill star and download counters do not update after website or CLI interactions.

## Best Possible Solution

Define the expected freshness for public counters, verify production stat-event processing health, then adjust the event processing or materialization path without reintroducing hot skill-document contention.

## Reproduction Assessment

No high-confidence live reproduction. Source inspection shows that short-term stale counters are expected because events are inserted immediately but visible skill stats are materialized by a delayed 6-hour sync; proving counters are stuck requires production backlog or failure-log data.

## Solution Assessment

Unclear. A simple cron-cadence change is not clearly the best fix because daily stats and visible skill document counters are separate paths, and the right freshness/performance tradeoff needs maintainer judgment.
