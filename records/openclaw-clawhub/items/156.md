## Summary

Keep open. Current main records download and star events, but public skill counters still depend on the delayed skill-doc stat sync path, and no production evidence shows the reported counters now update reliably; the related cadence-only pull request at https://github.com/openclaw/clawhub/pull/176 was closed unmerged after the author concluded it did not address the root cause.

## What This Changes

The issue reports that starring skills and downloading skills via the CLI or ClawHub website do not update the public skill metrics.

## Best Possible Solution

Inspect production event backlog and failures first, then either repair stuck processing or deliberately define a counter freshness model that preserves dedupe and avoids hot skill-document contention.

## Reproduction Assessment

No, not for the full production symptom that counters never update. Source inspection does show a delayed-counter path: events are inserted immediately, but public skill stats are not materialized until the 6-hour skill-doc-stat-sync path runs.

## Solution Assessment

No for a blind cadence or automatic code change. The maintainable path is to inspect production health and backlog first, then repair processing or choose an explicit freshness policy instead of repeating the closed interval-only approach from https://github.com/openclaw/clawhub/pull/176.
