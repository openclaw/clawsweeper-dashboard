Keep open. ClawHub cleanup policy keeps issues open unless a PR is certainly implemented on main, and the active discussion plus related open reports still point to production Unauthorized failures despite current main containing the expected auth and publish plumbing.

## What This Changes

Publishing a skill via the CLI should not return Unauthorized immediately after a successful browser login and whoami check.

## Best Possible Solution

Make browser-created tokens and Settings-created tokens work reliably for non-maintainer accounts against production https://clawhub.ai across login, whoami, and publish; then add focused regression coverage around token storage, registry normalization, redirect/header preservation, token hash/user lookup, and publish authorization, and consolidate the related Unauthorized reports once production is verified.
