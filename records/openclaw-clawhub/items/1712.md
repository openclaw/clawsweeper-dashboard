## Summary

Keep open. Current main has the package archive download counter path, but package installs still have no event/source model, and skill install counts are scoped to logged-in sync telemetry rather than immediate CLI install reporting.

## What This Changes

The issue requests working package download/install counters and skill install counters instead of package stats and skill installs staying at zero.

## Best Possible Solution

Keep the package download counter pipeline, then define explicit package install and skill immediate-install telemetry semantics before adding new event kinds, docs, and regression coverage.

## Reproduction Assessment

Yes for the remaining source-level behavior. Current main shows package download events, but no package install event kind, and CLI skill install does not report telemetry outside the documented sync flow.

## Solution Assessment

No for an automated fix. The download counter fix is already present, while the remaining install-count behavior needs maintainer agreement on counting semantics and telemetry/privacy policy.
