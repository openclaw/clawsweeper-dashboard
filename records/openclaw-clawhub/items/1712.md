## Summary

Keep open. Current main only resolves the package download part of the report; package installs still have no ClawHub-owned event model, and skill install counters still depend on sync telemetry rather than immediate `openclaw skill add`/install behavior. A maintainer explicitly confirmed the remaining semantics are intentionally unresolved, and ClawHub cleanup policy keeps issues open.

## What This Changes

The issue asks for package download/install counters and skill install counters to increment when packages or community skills are fetched or installed.

## Best Possible Solution

Define the intended install semantics and privacy behavior, then implement package install and skill install stat event pipelines with targeted regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows package downloads now enqueue events, while package installs lack an event kind or CLI signal and skill install counters are derived from sync telemetry rather than the immediate install command.

## Solution Assessment

No for an automated repair. The remaining work needs maintainer-defined package install semantics and a privacy decision for immediate skill-install reporting before code changes are narrow enough.
