## Summary

Keep open. Current main partially fixed package archive download counting, but package install counting still has no event pipeline, skill install counts remain sync-telemetry based, and ClawHub cleanup policy keeps issues open unless the item is a PR that is certainly implemented on main.

## What This Changes

Request package download/install stats to increment from package fetch/install flows and skill install stats to increment when community skills are added via the CLI.

## Best Possible Solution

Keep the package download event pipeline, then define install-count semantics and add explicit package/skill install reporting only with matching privacy docs and regression coverage.

## Reproduction Assessment

Yes. Source inspection shows packageStatEvents accepts only download events and cmdInstall has no telemetry call, while the documented skill install counter source is authenticated clawhub sync telemetry.

## Solution Assessment

No for an automated repair. The remaining counters need a maintainer-defined install signal and privacy model before a narrow code change is safe.
