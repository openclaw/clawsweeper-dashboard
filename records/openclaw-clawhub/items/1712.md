## Summary

Keep open. Current main has the partial package download counter fix, but package installs still have no defined event/source model, and skill installs are documented as logged-in sync telemetry rather than immediate install-command telemetry.

## What This Changes

Count package downloads/installs and skill current/all-time installs from the relevant ClawHub API and CLI usage paths instead of leaving exposed counters at zero.

## Best Possible Solution

Keep the package download event pipeline, then define explicit package install and skill telemetry semantics before adding any new event kinds, docs, and regression tests.

## Reproduction Assessment

Yes for the remaining source-level behavior. Current main shows package downloads now enqueue events, while package installs have no event kind and immediate skill installs do not report telemetry outside the documented sync flow.

## Solution Assessment

No for an automated fix. The download counter fix is narrow and already present, but the remaining package install and immediate skill install behavior needs a maintainer decision on counting semantics and privacy/opt-out expectations.
