## Summary

Keep open. Current main has partially fixed package archive downloads, but the remaining package install and immediate skill install counter behavior is still undefined; the ClawHub cleanup policy also keeps issues open unless they are PRs definitely implemented on main.

## What This Changes

The issue requests package download/install counters and skill install counters to increment from real ClawHub fetch/install usage instead of remaining at zero.

## Best Possible Solution

Define the remaining stat semantics, then add an explicit package install stats path and either document sync telemetry as the skill install source or add immediate skill install reporting with preserved privacy controls.

## Reproduction Assessment

Yes. Static current-main inspection reproduces the remaining gap: package install stat writers are absent, and skill install counters are still driven by sync telemetry rather than the immediate install command.

## Solution Assessment

Unclear. Package download counting is already implemented, but the remaining install behavior should not be patched blindly until maintainers define the owned install signal and telemetry privacy semantics.
