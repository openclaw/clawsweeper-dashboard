## Summary

Keep open. Current main now counts package archive downloads, but package install counts still have no defined event/signal, and skill install counts remain tied to the documented sync telemetry flow rather than immediate CLI install reporting.

## What This Changes

Track package download/install counters and skill install counters so public stats reflect API/package downloads and CLI-added community skills.

## Best Possible Solution

Keep the package download event pipeline, then define explicit package install and immediate skill install telemetry semantics before adding event kinds, docs, and regression tests.

## Reproduction Assessment

Yes, for the remaining source-level behavior. Current main shows package downloads enqueue events, but package installs have no event kind or writer, and CLI skill install does not report telemetry outside the documented sync flow.

## Solution Assessment

No, not as an automated bug fix. The package download portion is already fixed, while the remaining install counting needs maintainer decisions on package install signals and skill telemetry privacy before implementation.
