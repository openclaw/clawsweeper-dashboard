## Summary

Keep open. Current main fixed package archive download counting, but package install counts still have no ClawHub-owned install signal and immediate skill-install reporting would change the documented sync-only telemetry model.

## What This Changes

The issue asks for package download/install counters and skill install counters to reflect real package and skill usage instead of remaining at zero.

## Best Possible Solution

Keep the package download event pipeline, then define explicit package install and immediate skill telemetry semantics before adding event kinds, API/CLI behavior, docs, and regression tests.

## Reproduction Assessment

Yes, for source-level verification of the remaining behavior. Current main records package downloads, but package installs have no event kind or writer and direct skill installs do not send telemetry outside the documented sync flow.

## Solution Assessment

No, not as an automated bug fix. The remaining work needs maintainer decisions on package install signals and skill telemetry privacy before implementation.
