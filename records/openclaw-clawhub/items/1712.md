## Summary

Keep open. Current main has the partial package download counter fix, but package installs still have no defined event/source model and skill installs are intentionally counted through logged-in sync telemetry rather than immediate install commands.

## What This Changes

Requests package download/install counters and skill install counters to increment from package fetch/install and skill install flows.

## Best Possible Solution

Define package install and skill telemetry/privacy semantics first, then add the matching event pipeline, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection on current main shows package downloads now record events, while package installs have no event kind and skill install counters are produced from sync telemetry rather than the install command.

## Solution Assessment

No for an automated fix. The remaining work needs a maintainer decision on what counts as a package install and whether immediate skill-install telemetry should exist beyond the documented sync flow.
