## Summary

Keep open under ClawHub cleanup policy: package archive download counting is fixed on current main, but package install counts and immediate skill install counts still lack defined install-signal and telemetry/privacy semantics.

## What This Changes

The issue requests package download/install counters and skill install counters to increment from API and CLI usage instead of remaining zero.

## Best Possible Solution

Keep the shipped package download event pipeline, then add explicit package and skill install event semantics with docs and regression tests after maintainers approve the telemetry behavior.

## Reproduction Assessment

Yes, for the remaining source-level behavior: current main has only package download stat events, and `clawhub skill install` only downloads and writes local files. Install counters are driven by documented `clawhub sync` telemetry instead of immediate install reporting.

## Solution Assessment

No, this is not a narrow automated bug fix as-is. Package downloads are already fixed, while package installs and immediate skill install reporting need maintainer decisions before adding event kinds, writers, docs, and tests.
