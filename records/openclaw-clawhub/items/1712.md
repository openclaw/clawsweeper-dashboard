## Summary

Keep open. Current main has the package download counter path, but package install counts still lack a defined ClawHub-owned install signal and immediate skill install counting would change the documented sync-based telemetry model.

## What This Changes

Request package download/install counters for packages and skill install counters to reflect real API and CLI usage instead of remaining zero.

## Best Possible Solution

Keep the package download event pipeline, then define explicit package install and immediate skill telemetry semantics before adding event kinds, docs, and regression tests.

## Reproduction Assessment

Yes, for the remaining source-level behavior. Current main shows package downloads enqueue events, while package installs have no event kind and immediate skill installs do not report telemetry outside the documented sync flow.

## Solution Assessment

No, not as an automated bug fix. The remaining work needs maintainer decisions on package install signals and skill telemetry privacy before implementation.
