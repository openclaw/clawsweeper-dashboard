## Summary

Keep open. Current main still exposes package stats without a package download/install writer, and skill install counters are still driven by authenticated sync telemetry rather than immediate skill install activity. The ClawHub cleanup policy also keeps issues open unless a PR is definitely implemented on main.

## What This Changes

The issue asks for ClawHub package download/install stats and skill install stats to increment for real API/CLI usage instead of remaining at zero.

## Best Possible Solution

Define the intended stat semantics, then add a package stats pipeline for ClawHub-controlled archive downloads and any explicit install signal the platform owns. For skills, either document sync telemetry as the install counter source or add deliberate immediate-install reporting that preserves opt-out and privacy behavior.
