## Summary

Keep open. Current main fixed package archive download counting, but package install counts still have no event model or writer, and skill install counts remain tied to authenticated sync telemetry rather than immediate install/uninstall commands.

## What This Changes

The issue requests package download/install counters and skill install counters to increment from package and skill usage instead of remaining at zero.

## Best Possible Solution

Keep the package download event pipeline, then define package install semantics and immediate skill install telemetry/privacy behavior before adding event kinds, docs, and regression tests.

## Reproduction Assessment

Yes, for the remaining source-level behavior. Current main shows package downloads are recorded, but package installs have no event kind or writer, and direct skill install/uninstall commands do not report telemetry outside the documented sync flow.

## Solution Assessment

No, not as an automated bug fix. The remaining work needs a maintainer decision on what counts as a package install and whether immediate skill install telemetry should be added to the current sync-based privacy model.
