## Summary

Keep open. Current main has the package download-counting partial fix, but package installs still have no event/source model and skill install counters are documented as sync telemetry rather than immediate install-command reporting.

## What This Changes

The issue requests live package download/install counters and skill install counters instead of zero-valued stats after real package or skill use.

## Best Possible Solution

Keep the package download event path, then document and implement the agreed package install and skill telemetry semantics with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows package downloads now record events, while package installs have no event kind/writer and skill install counters are produced from logged-in sync telemetry rather than the install command.

## Solution Assessment

No for an automated fix. The partial package download implementation is narrow, but the remaining install counters need maintainer-defined install and privacy semantics before code changes are safe.
