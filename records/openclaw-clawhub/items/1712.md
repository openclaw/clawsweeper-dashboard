## Summary

Under ClawHub cleanup policy this issue should stay open; current main has the package archive download fix, but package install counts and immediate skill-install counts still need maintainer-defined signals and telemetry/privacy semantics.

## What This Changes

Request to make package downloads/installs and skill installsAllTime/installsCurrent increment from package fetches and CLI skill installs.

## Best Possible Solution

Keep the package download event pipeline, then define and document package install and immediate skill-install telemetry semantics before adding event writers and regression coverage.

## Reproduction Assessment

Yes, by source inspection: packageStatEvents only supports downloads, the processor preserves package installs, and CLI skill install exits without a telemetry write. Package archive download counting itself is already fixed on current main.

## Solution Assessment

No; direct install increments are not yet the narrowest maintainable fix because the install signal and privacy model are unsettled. The safer path is a maintainer decision followed by explicit event writers, docs, and tests.
