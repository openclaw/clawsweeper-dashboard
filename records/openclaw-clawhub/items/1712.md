## Summary

Keep open. Current main has fixed package archive download counting, but package install/star counters still have no defined writer semantics and direct skill installs are still counted only through documented sync telemetry, so the remaining work needs maintainer product/privacy direction.

## What This Changes

The issue requests package download/install counters and direct community skill install counters to increment instead of staying at zero.

## Best Possible Solution

Keep the shipped package download counter, then document the intended package install/star and skill install telemetry semantics in specs/docs before adding explicit signals that preserve privacy controls.

## Reproduction Assessment

Yes. Source inspection shows package install/star events have no schema path and direct `clawhub skill install` does not post telemetry; I did not run a live Convex/CLI setup in this read-only review.

## Solution Assessment

No for an automatic code-only fix. The package download counter is already implemented, but the remaining install/star counters need agreement on what should count and how privacy/opt-out behavior should work.
