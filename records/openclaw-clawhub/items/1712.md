## Summary

Keep open. Current main has the package download counter pipeline, but package install stats still have no event/source model, and immediate skill install counting is intentionally outside the documented sync-telemetry flow.

## What This Changes

The issue requests package download/install counters and skill install counters that reflect real package fetches and skill installs.

## Best Possible Solution

Keep the package download pipeline, then define package install and skill telemetry semantics before adding event kinds, docs, and regression tests.

## Reproduction Assessment

Yes, for the remaining source-level behavior. Current main has package download events, while package installs have no event kind and immediate skill installs do not report telemetry outside the documented sync flow.

## Solution Assessment

No, not as an automated fix. The download counter is already fixed narrowly; the remaining install counters need maintainer agreement on counting semantics and telemetry privacy behavior.
