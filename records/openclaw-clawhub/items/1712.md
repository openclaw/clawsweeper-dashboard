## Summary

Keep open. Current main partially addressed the package download counter, but package install counts still have no defined event model, skill install counts remain tied to logged-in sync telemetry rather than immediate install, and the ClawHub cleanup policy keeps issues open unless the item is a PR that is certainly implemented on main.

## What This Changes

Requests package download/install counters and skill install counters to increment from ClawHub fetch/install activity instead of remaining zero.

## Best Possible Solution

Define owned install semantics first, then add the matching package and skill install event pipelines with privacy documentation and regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows package downloads now enqueue events, but package installs have no event kind or CLI signal, and skill install counters are produced from sync telemetry rather than the immediate install command.

## Solution Assessment

No for an automated repair. The remaining work is not a narrow counter bug because maintainers first need to define what counts as a package install and whether immediate skill-install telemetry should exist.
