## Summary

Keep open. Current main has fixed package archive download counting, but package install counts still have no event model, skill install counts remain tied to authenticated sync telemetry rather than immediate `skill install`, and ClawHub cleanup policy keeps issues open.

## What This Changes

Request that package downloads/installs and skill installs increment their public stats instead of remaining at zero.

## Best Possible Solution

Keep the package download event pipeline, then define install semantics and telemetry policy before adding package install and immediate skill-install reporting with docs and regression coverage.

## Solution Assessment
