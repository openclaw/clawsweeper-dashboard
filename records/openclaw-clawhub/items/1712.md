## Summary

Keep open under ClawHub cleanup policy: package archive download counting is fixed on current main, but package install counts and immediate skill-install counts still need a defined install signal plus telemetry/privacy semantics.

## What This Changes

Request package download/install counters and skill install counters to increment from ClawHub package and skill install/download flows.

## Best Possible Solution

Keep the shipped package download event pipeline, then define and document package install and immediate skill-install event semantics before adding writers and regression coverage.

## Reproduction Assessment

Yes, for the remaining source-level behavior: packageStatEvents only supports downloads, package processing preserves installs, and clawhub install does not send telemetry. Package archive download counting itself is already present on current main.

## Solution Assessment

No, the requested direct install increments are not yet the narrowest maintainable fix because package installs lack a ClawHub-owned signal and skill installs are documented as sync-based telemetry. The safer path is a maintainer decision followed by explicit event writers, docs, and tests.
