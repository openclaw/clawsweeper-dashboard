## Summary

Keep open. Current main fixed package archive download counting, but package installs still have no defined ClawHub-owned install signal and skill install counters remain tied to documented `clawhub sync` telemetry rather than immediate `skill install` activity.

## What This Changes

Request that package downloads/installs and skill install counters increment from real ClawHub API and CLI usage instead of staying at zero.

## Best Possible Solution

Define the intended install semantics, then add only the owned package install signal and either document sync-time skill install counting or deliberately add immediate skill install reporting with preserved privacy controls.

## Reproduction Assessment

Yes. Source inspection shows package downloads are now counted, while package install events do not exist and immediate skill install commands do not post telemetry; install counters are currently reproducible as sync-driven behavior.

## Solution Assessment

Unclear until maintainers decide semantics. The current package download event pipeline is a maintainable fix for downloads, but directly incrementing install counters from every install command would be a product/privacy change rather than a narrow bug fix.
