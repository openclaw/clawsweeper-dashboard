## Summary

This PR should stay open because current main does not contain the audit-preview tooltip and the proposed implementation still has a concrete trust-boundary bug: scanner overview text can appear even when public scan results are suppressed for staff-cleared items.

## What This Changes

The PR adds hover/focus audit-card tooltips with capped scanner overview text and a Read full audit CTA to skill and plugin detail pages, plus CSS, tests, and a changelog entry.

## Best Possible Solution

Keep the tooltip UX, but make preview overview copy honor suppressScanResults and add a focused regression test for staff-cleared public audit previews.

Yes. Source inspection gives a high-confidence path: render the PR head DetailSecuritySummary with suppressScanResults and suspicious scanner summaries, then hover or focus an audit card; the visible card is cleared while the tooltip overview is derived from raw scanner fields.
