## Summary

Keep open: the tooltip UX is useful and the provided screenshots show the intended behavior, but the latest PR head still exposes raw scanner summary text in previews when current main treats staff-cleared scan results as suppressed for non-staff viewers.

## What This Changes

The PR adds hover/focus audit-card preview tooltips with capped scanner overview text, a Read full audit CTA, tests, styles, and a changelog entry.

## Best Possible Solution

Keep the tooltip UX, but merge it only after preview text respects staff-cleared suppression and the suppressed scanner state has focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path on the PR head: render DetailSecuritySummary with suppressScanResults and suspicious scanner summaries, then focus an audit card; the card status is Cleared while the tooltip overview comes from raw scanner data.

## Solution Assessment

No, not as currently written. The narrow maintainable fix is to gate tooltip overview text on suppressScanResults and use the cleared/suppressed public copy instead of raw scanner summaries.
