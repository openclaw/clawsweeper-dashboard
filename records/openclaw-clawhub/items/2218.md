## Best Possible Solution

Keep the tooltip UX, but derive preview copy from suppression-aware values and cover staff-cleared suspicious scanner data with a focused regression test.

Yes. Source inspection gives a high-confidence path: render DetailSecuritySummary with suppressScanResults plus suspicious summaries, then focus an audit preview; the PR head derives preview text from raw summaries while the card status is Cleared.

## Solution Assessment

No, not as written. The narrow maintainable fix is to keep the tooltip UX but gate overview copy on suppressScanResults, using suppressed or generic cleared copy instead of raw scanner summaries.
