## Summary

This PR should stay open: the tooltip UX is useful, but the current branch can reveal suppressed scanner summaries in audit previews even though current main treats staff-cleared suspicious scans as Cleared for public viewers.

## What This Changes

The PR adds hover/focus preview tooltips with capped scanner overview text and a Read full audit CTA to compact audit cards on skill and plugin detail pages.

## Best Possible Solution

Keep the tooltip UX, but gate preview overview copy through the same suppressScanResults path as the compact audit row and add focused regression coverage for the staff-cleared state.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: render DetailSecuritySummary on the PR head with suppressScanResults and suspicious scanner summaries, then focus an audit card; the card status is Cleared while the preview text is sourced from raw scanner summaries.

## Solution Assessment

No, not as currently written. The narrow maintainable fix is to reuse the existing suppression boundary for tooltip overview text instead of displaying raw scanner summaries when scans are staff-cleared.
