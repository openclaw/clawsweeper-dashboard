## Summary

Keep open; the tooltip UI is close and the screenshots show the intended behavior, but the diff needs a suppression-policy fix before merge because staff-cleared audit cards can still preview raw scanner summaries.

## What This Changes

The PR adds hover/focus audit preview tooltips with capped scanner summaries and full-audit CTAs on skill and plugin detail pages, plus styles, tests, and a changelog entry.

## Best Possible Solution

Keep the tooltip UX, but make preview content honor scan suppression and cover that staff-cleared state with a focused DetailSecuritySummary test.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: render DetailSecuritySummary with suppressScanResults and suspicious summaries, then focus an audit card on the PR branch. The new overview constants feed the raw summaries into the preview while the card status is Cleared.

## Solution Assessment

No, not as currently written. The narrow maintainable fix is to gate tooltip overview copy on suppressScanResults, using the cleared/suppressed message or generic cleared copy instead of raw scanner summaries.
