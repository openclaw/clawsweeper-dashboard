## Summary

Keep open; current main does not include the preview, and the PR has a narrow CSS bug that can make the mouse CTA unreachable despite screenshots showing the basic tooltip rendering.

## What This Changes

Adds hover/focus audit preview panels with capped scanner summaries and a `Read full audit` CTA on skill and plugin detail audit cards.

## Best Possible Solution

Land the audit preview after fixing the hover-to-CTA gap and preserving the current link/navigation behavior with a focused test or visual check.

## Reproduction Assessment

Yes for the review finding from source inspection: the PR places the tooltip 10px above the trigger and clips the pseudo-element intended to bridge that gap. I did not run a browser reproduction, but the CSS path is direct.

## Solution Assessment

No: the user-facing preview direction is reasonable, but the patch should keep the hover bridge outside the clipped tooltip or otherwise make the mouse CTA reachable before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep the tooltip hover bridge unclipped:** `src/styles.css:4282`
  - body: The tooltip sits `10px` above the audit card, and the `::before` rule immediately below is meant to bridge that gap so users can move the mouse to the `Read full audit` CTA. Because the tooltip itself has `overflow: hidden`, that pseudo-element is clipped at the tooltip border; once the cursor leaves the card, `:hover` drops and the preview switches back to `pointer-events: none`, making the CTA unreliable or unreachable by mouse. Move the bridge to an unclipped wrapper or otherwise keep the hover path active.
  - confidence: 0.86
