## Summary

Keep open under the ClawHub issue policy: current main now fixes the original ownerless CLI/API search ambiguity, but owner-qualified install syntax remains a broader scoped-locator product decision.

## What This Changes

The issue asks ClawHub to make skills easier to identify by owner in CLI search, install guidance, or a clearer owner/name slug structure.

## Best Possible Solution

Keep the shipped owner-aware search/API/docs behavior; handle `@owner/name` install support through the planned orgs/scoped-identity migration with compatibility tests for bare slugs, lockfiles, filesystem paths, and API locators.

## Reproduction Assessment

No. The original ownerless `clawhub search` ambiguity no longer reproduces by source inspection and regression tests; the remaining owner-qualified install syntax is unsupported future scoped-locator work rather than a current broken contract.

## Solution Assessment

Yes for the shipped search fix: a separate owner column resolves the reported discovery ambiguity without teaching an unsupported install locator. Full scoped install support should wait for the broader `@owner/name` migration rather than landing as a piecemeal CLI-only change.
