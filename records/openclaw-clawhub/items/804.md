## Summary

Keep open under the ClawHub issue policy: current main has fixed the original ownerless CLI search ambiguity, but owner-scoped install/download identity remains a product/API compatibility decision and is partly tracked by https://github.com/openclaw/clawhub/issues/2348.

## What This Changes

The issue requests clearer owner-scoped skill identity in ClawHub search, download, and install flows so users can distinguish same-named skills such as `steipete/weather`.

## Best Possible Solution

Keep the shipped owner-aware search/API behavior, then resolve scoped install and download identity through a single compatible locator contract across UI, CLI, API, docs, and tests.

## Reproduction Assessment

Yes, from source for the remaining gap: current main still rejects slash-containing install locators, copies a bare-slug OpenClaw command, and names downloads from slug plus version. The original ownerless search ambiguity is no longer reproducible from source because CLI/API/schema/docs/tests now carry owner metadata.

## Solution Assessment

Partially. Showing the owner in search is the narrow maintainable fix for the reporter's immediate discovery ambiguity, but full owner-scoped install/download identity should be handled as the product/API contract tracked by https://github.com/openclaw/clawhub/issues/2348 rather than patched only in the UI.
