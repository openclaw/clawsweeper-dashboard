## Summary

Keep open under the ClawHub issue cleanup policy: current main and a production tag solve the ownerless search ambiguity, but any remaining `owner/slug` install locator is a separate product contract decision.

## What This Changes

The issue requests clearer skill identity in ClawHub search/install flows, such as owner-qualified locators, web install guidance, or an owner column in CLI search results.

## Best Possible Solution

Keep the shipped owner-aware search behavior, and treat owner-qualified install syntax as a separate explicit CLI/API/product design item if maintainers want that capability.

## Reproduction Assessment

No. The original ownerless `clawhub search` and `/api/v1/search` ambiguity no longer reproduces on current main because the CLI renders owner handles and the API returns owner metadata with regression coverage.

## Solution Assessment

Yes for the narrow ambiguity: the shipped owner column keeps the installable bare slug first and avoids teaching unsupported `owner/slug` install syntax. Full owner-qualified install would need a separate maintainer decision and contract design.
