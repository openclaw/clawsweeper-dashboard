Keep open under the ClawHub issue policy: current main has addressed the original CLI search ambiguity by showing owner metadata, but owner-scoped or versioned install locators remain a product/API compatibility decision already tracked by https://github.com/openclaw/clawhub/issues/2348.

## What This Changes

The issue asks for clearer skill identity in ClawHub search and install flows, including owner/version-qualified locators, skill-page install guidance, or an owner column in `clawhub search`.

## Best Possible Solution

Keep the shipped owner-aware search/API behavior, then resolve scoped install identity through https://github.com/openclaw/clawhub/issues/2348 by defining one locator contract with safe bare-slug compatibility and matching UI, CLI, API, docs, and tests.

## Reproduction Assessment

Yes from source for the remaining gap: current main still rejects slash-containing install locators and the command card still copies a bare slug command. The original ownerless search ambiguity is no longer reproducible from source because current CLI/API/schema/docs/tests carry owner metadata.

## Solution Assessment

Yes for the shipped discovery fix: showing owner metadata solves the reporter's immediate search ambiguity without teaching unsupported syntax. Full owner-scoped install support should be handled as a product/API contract in the linked follow-up, not as a small UI-only patch.
