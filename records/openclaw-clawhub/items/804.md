## Summary

Keep open. Current main still leaves the reported owner ambiguity in CLI search and the public v1 search contract, and ClawHub cleanup policy keeps issues open rather than auto-closing them.

## What This Changes

The issue requests clearer owner-qualified skill identity in ClawHub search/install flows so users can distinguish same-slug or similar-slug skills by publisher.

## Best Possible Solution

Expose owner identity additively in `/api/v1/search`, update both schema copies, render owner information in `clawhub search`, and document the output while leaving owner-qualified install locators for a separate decision.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: internal search entries include `ownerHandle` and `owner`, but the v1 handler, schemas, and CLI renderer drop that identity before `clawhub search` output.

## Solution Assessment

Yes for the narrow owner-disambiguated search/API display path. Full owner-qualified install syntax should remain separate because it changes deliberate slash rejection, filesystem extraction targets, and lockfile semantics.
