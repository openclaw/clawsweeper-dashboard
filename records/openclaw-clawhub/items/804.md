## Summary

Keep open. Repository policy keeps issues open, and current main still leaves the reported CLI discovery ambiguity unresolved: public search and CLI search output omit owner identity, while install remains bare-slug only.

## What This Changes

The issue asks ClawHub to make skill identity clearer by showing owner-qualified names or owner context in CLI/web install flows.

## Best Possible Solution

Expose owner identity additively in the public v1 skill search response, update both schema copies, render owner context in `clawhub search`, and document the output/API change while keeping owner-qualified install syntax separate.

## Reproduction Assessment

Yes, by source inspection: current main still prints ownerless `clawhub search` rows, omits owner fields from the v1 search response/schema, and rejects `owner/slug` install targets before resolution.

## Solution Assessment

Yes for the narrow owner-disambiguated search/API change because internal search already carries owner identity. Full owner-qualified install locators are not the safest first fix and should remain a separate maintainer decision.
