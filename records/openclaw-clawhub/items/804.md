## Summary

Keep open. Current main still leaves the reported CLI discovery ambiguity unresolved: `clawhub search` and `/api/v1/search` do not expose owner identity, while the web prompt path only partially mitigates owner-qualified context.

## What This Changes

The issue asks ClawHub to make skills less ambiguous by supporting clearer owner-qualified identity, showing install guidance on skill pages, or adding owner context to CLI search results.

## Best Possible Solution

Expose owner identity additively in `/api/v1/search`, update both schema copies, render owner context in `clawhub search`, and keep owner-qualified install syntax as a separately scoped decision.

## Reproduction Assessment

Yes. Source inspection shows `cmdSearch` prints ownerless rows, the v1 search handler omits owner fields, and both search schemas lack owner identity even though internal search entries already carry it.

## Solution Assessment

Yes for the narrow owner-disambiguated search/API display path. Full owner-qualified install syntax is broader because current install intentionally rejects slash-containing inputs before filesystem and lockfile side effects.
