## Summary

Keep open under the ClawHub issue policy and because the only meaningful remaining work is a maintainer product decision. Current main already shows owner metadata in search results/API/docs, while `owner/slug` or versioned install locators remain unsupported and affect broader install identity semantics.

## What This Changes

Request to make ClawHub skill identity less ambiguous by using owner-qualified skill locators, adding web install guidance, or showing owner information in `clawhub search`.

## Best Possible Solution

Keep the shipped owner-aware search/API/docs behavior, and route scoped install locators through an explicit install identity design such as https://github.com/openclaw/clawhub/issues/2348 with compatibility tests for bare slugs, lockfiles, downloads, extraction paths, and docs.

## Reproduction Assessment

No for the original ownerless-search ambiguity: current source and tests show owner metadata is now returned and printed. Yes from source for the remaining gap: slash-containing install locators are still rejected by `normalizeSkillSlugOrFail`.

## Solution Assessment

Yes for the shipped discovery fix: owner metadata in search resolves the reporter's immediate ambiguity without advertising an unsupported install argument. Full scoped install support should be handled as a broader owner-scoped identity contract, not a narrow CLI-only patch.
