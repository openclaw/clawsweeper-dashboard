## Summary

Keep open. Current main still leaves the reported discovery ambiguity in place: CLI search and the public search API omit owner identity, while owner-qualified install syntax remains outside the current bare-slug install path.

## What This Changes

Request clearer owner-qualified skill identity so users can distinguish similarly named skills in ClawHub search and install guidance.

## Best Possible Solution

Expose owner identity additively in `/api/v1/search`, update both schema copies and CLI search output/docs, and defer owner-qualified install locators to a separate maintainer decision.

## Reproduction Assessment

Yes. Source inspection shows `cmdSearch` prints ownerless rows, `searchSkillsV1Handler` omits owner fields, and both search schemas lack owner identity even though internal search entries already carry it.

## Solution Assessment

Yes for the narrow owner-disambiguated search/API display path. Adding `clawhub install owner/slug` in the same change is not the best first fix because current slug normalization intentionally blocks slash-containing inputs before filesystem and lockfile side effects.
