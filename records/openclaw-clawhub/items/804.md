## Summary

Keep open. Current main still leaves the reported owner ambiguity in CLI search and the public v1 search response, while ClawHub cleanup policy does not auto-close issues and there is no merged duplicate that owns this exact owner-disambiguation request.

## What This Changes

Request to make ClawHub skill discovery and install references distinguish owners, such as owner/slug prompts or an owner column in `clawhub search`.

## Best Possible Solution

Expose owner identity additively in `/api/v1/search`, update both schema copies, render owner information in `clawhub search`, and document the output while leaving owner-qualified install locators for a separate decision.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: internal search carries owner identity, but the v1 handler, schemas, and CLI renderer drop it before `clawhub search` output.

## Solution Assessment

Yes for the narrow owner-disambiguated search/API display path. Full owner-qualified install syntax should not be bundled into the first repair because it changes deliberate slash rejection, install target paths, and lockfile semantics.
