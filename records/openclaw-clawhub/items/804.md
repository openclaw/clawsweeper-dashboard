Keep open under the ClawHub issue policy: owner-aware search/API/docs have shipped, but issues are not auto-closed here and the meaningful remainder is a scoped owner-qualified install locator decision.

## What This Changes

The issue asks for clearer skill identity in ClawHub search/install flows, such as owner-qualified locators or owner information in the UI/CLI output.

## Best Possible Solution

Keep the shipped owner-aware search behavior; if maintainers want owner-qualified installs, design them as part of the scoped @owner/name locator migration with safe API, filesystem, and lockfile semantics.

## Reproduction Assessment

No. The original ownerless `clawhub search` ambiguity no longer reproduces by source inspection because current main prints owner handles and returns owner metadata; only unsupported owner-qualified install syntax remains source-confirmed.

## Solution Assessment

Yes for the narrow discovery ambiguity: owner-aware search fixes the confusing result list without teaching an unsupported locator. Full owner-qualified install support should be handled through the scoped identity migration rather than a small CLI-only patch.
