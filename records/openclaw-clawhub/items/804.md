## Summary

Keep open. Current main still leaves the reported CLI discovery ambiguity unresolved: internal search entries have owner identity, but `/api/v1/search`, its schemas, and `clawhub search` output omit it; a prior closed PR attempted part of this but was not merged.

## What This Changes

The issue requests clearer owner/slug or owner information so users can distinguish same-named skills in `clawhub search` and install the intended ClawHub skill.

## Best Possible Solution

Expose owner identity additively in `/api/v1/search`, update both schema copies, render owner information in `clawhub search`, and document the output while keeping full owner-qualified install syntax separate.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: internal search entries include `ownerHandle` and `owner`, but the v1 handler, schemas, and CLI renderer drop that identity before `clawhub search` output.

## Solution Assessment

Yes for the narrow owner-disambiguated search/API display path. Full owner-qualified install syntax should remain separate unless maintainers explicitly widen scope because it changes deliberate slash rejection, extraction targets, and lockfile semantics.
