## Summary

Keep open. Current main still leaves the reported owner ambiguity in `clawhub search` and the public v1 search response, and no canonical duplicate or linked fix PR now owns this exact owner-disambiguation work.

## What This Changes

Request clearer owner-qualified skill identity in ClawHub discovery/install flows so similarly named skill slugs can be distinguished by publisher.

## Best Possible Solution

Expose owner identity additively in public skill search, render it in CLI search output, and document the response/output while treating full owner-qualified install syntax as a separate follow-up.

## Reproduction Assessment

Yes. Source inspection on current main shows internal search entries already include owner identity, but the v1 handler, schemas, and CLI renderer drop it before `clawhub search` output.

## Solution Assessment

Yes for the narrow repair: add owner identity to the API/schema/CLI display path. Full `owner/slug` install locator support should remain separate because it changes filesystem, lockfile, and slash-validation behavior.
