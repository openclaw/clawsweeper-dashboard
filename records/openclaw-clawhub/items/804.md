## Summary

Keep open under the ClawHub issue policy: current main has fixed the original ownerless CLI search ambiguity, but the remaining owner-scoped install locator/UI command behavior is a product/API compatibility decision already tracked by https://github.com/openclaw/clawhub/issues/2348.

## What This Changes

The issue asks ClawHub to disambiguate skills by owner in search and install flows, either through owner/slug locators, clearer web install guidance, or an owner column in CLI search.

## Best Possible Solution

Keep the shipped owner-aware search/API behavior, then resolve scoped install identity through https://github.com/openclaw/clawhub/issues/2348 with one compatible locator contract across UI, CLI, API, filesystem/lockfile behavior, docs, and tests.

## Reproduction Assessment

Yes for the remaining source-level behavior: current main rejects slash-containing install locators and the web command card still copies a bare slug when owner data is supplied. The original ownerless CLI search output is no longer reproducible because current search API/schema/CLI code carries owner metadata.

## Solution Assessment

Yes for the shipped discovery fix: showing owner metadata solves the reporter's immediate search ambiguity without teaching unsupported syntax. Full owner-scoped install support should stay with the product/API contract tracked by https://github.com/openclaw/clawhub/issues/2348.
