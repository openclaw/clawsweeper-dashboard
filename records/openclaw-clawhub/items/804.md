## Summary

Keep open. Current main still leaves the reported CLI discovery ambiguity unresolved: public v1 search and `clawhub search` do not expose owner identity, while `clawhub install` rejects owner-qualified locators before lookup. The issue is a focused repair candidate, but this repository cleanup policy keeps ClawHub issues open rather than auto-closing them.

## What This Changes

The issue requests clearer skill identity in ClawHub search/install flows, such as owner-qualified naming, web install guidance, or an owner column in `clawhub search`.

## Best Possible Solution

Expose owner identity additively in public v1 search, update shared and CLI-local schemas plus docs, and render owner information in `clawhub search`; treat full owner-qualified install syntax as a separate design decision.

## Reproduction Assessment

Yes. Source inspection on current main is enough: the v1 search handler and schemas omit owner fields, `cmdSearch` prints only bare slugs, and install rejects slash-qualified locators before resolving metadata.

## Solution Assessment

Yes for the narrowed owner-display repair. It is the smallest maintainable change for the reporter’s discovery problem, while owner-qualified install locator semantics should be handled separately.
