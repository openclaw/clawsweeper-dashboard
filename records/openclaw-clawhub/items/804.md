## Summary

Keep open. Current main still leaves the reported CLI discovery ambiguity unresolved: public search and CLI search output do not expose owner identity, while install still only accepts bare slugs.

## What This Changes

Clarify skill ownership in search/install UX so users can distinguish owner-qualified skills such as steipete/weather from similarly named results.

## Best Possible Solution

Expose owner identity additively in the public v1 search response, update shared and CLI schemas/docs/tests, and render owner information in `clawhub search`; keep full scoped install locators as separate maintainer-scoped work.

## Reproduction Assessment

Yes. Code inspection gives a high-confidence reproduction path: search results from `/api/v1/search` cannot include owner identity today, `cmdSearch` prints only ownerless fields, and `cmdInstall` rejects owner-qualified locators.

## Solution Assessment

Yes for a narrow owner-disambiguated search fix. Full scoped install syntax is not the best first repair because it touches path safety, locator semantics, and the broader orgs/scoped identity plan.
