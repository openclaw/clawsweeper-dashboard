## Summary

Keep open. Current main still leaves the reported owner ambiguity in `clawhub search` and `/api/v1/search`; the issue is also outside ClawHub cleanup auto-close policy because it is an issue, not an obsolete PR.

## What This Changes

The issue asks ClawHub to make similarly named skills distinguishable by owner, especially in `clawhub search`, and suggests owner-qualified install syntax or clearer web/CLI install guidance.

## Best Possible Solution

Expose owner identity additively in public v1 search, both schema copies, CLI search output, and docs; leave owner-qualified install locators for a separately scoped decision.

## Reproduction Assessment

Yes. Source inspection shows internal search carries owner identity, but the v1 handler, schemas, and CLI renderer drop it before `clawhub search` output.

## Solution Assessment

Yes for the narrow owner-disambiguated search/API display path. Full owner-qualified install syntax should not be bundled into this repair because it changes slash validation, extraction paths, and lockfile semantics.
