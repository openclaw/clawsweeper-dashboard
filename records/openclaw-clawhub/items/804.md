## Summary

Keep open. Current main still leaves the reported CLI discovery ambiguity unresolved: public v1 search and `clawhub search` do not expose owner identity, while `clawhub install` still accepts only bare slugs.

## What This Changes

The issue asks for ClawHub search/install surfaces to disambiguate skills by owner, such as showing `owner/slug` or an owner column for duplicate-looking skill slugs.

## Best Possible Solution

Expose owner identity additively in public v1 search, update shared and CLI-local schemas/docs, and render owner information in `clawhub search`; leave owner-qualified install syntax for a separate locator design.

## Reproduction Assessment

Yes. Source inspection on current main shows the CLI/API path still cannot display owner identity for search results, and owner-qualified install strings are rejected before resolution.

## Solution Assessment

Yes for the narrowed owner-display repair. It is the smallest maintainable fix for the reported discovery problem, while full owner-qualified install support should be handled separately.
