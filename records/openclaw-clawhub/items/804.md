## Summary

Keep open under the ClawHub cleanup policy: current main has fixed the original ownerless `clawhub search` ambiguity, but owner-scoped install locators and the install-command UI remain unsupported and are already being shaped in https://github.com/openclaw/clawhub/issues/2348.

## What This Changes

The issue requests clearer owner-aware skill identity in ClawHub search/install flows so users can distinguish same-slug skills and avoid installing the wrong package.

## Best Possible Solution

Keep the shipped owner-aware search/API behavior, then resolve scoped install identity through https://github.com/openclaw/clawhub/issues/2348 with one compatible locator contract across UI, CLI, API, docs, and tests.

## Reproduction Assessment

Yes for the remaining gap: current source rejects slash-containing install locators and the command card still copies a bare slug command. The original ownerless search output is no longer reproducible from current source because CLI/API/schema/docs/tests now carry owner metadata.

## Solution Assessment

Yes for the shipped discovery fix: showing owner metadata solves the reporter's immediate search ambiguity without teaching unsupported syntax. Full owner-scoped install support should be handled as the product/API contract tracked by https://github.com/openclaw/clawhub/issues/2348.
