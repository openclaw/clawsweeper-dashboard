## Summary

Keep open. ClawHub issue policy does not auto-close issues, and current main still leaves the reported owner ambiguity in `clawhub search`: the CLI output and public v1 search contract omit owner identity even though internal search results already carry it.

## What This Changes

The issue asks for clearer owner-qualified skill identity so users can distinguish skills like `steipete/weather` from other `weather` results in ClawHub search and install flows.

## Best Possible Solution

Expose owner identity additively in the public v1 skill search response, update both schema copies, render owner context in `clawhub search`, and keep owner-qualified install syntax as a separately scoped maintainer decision.

## Reproduction Assessment

Yes, by source inspection: current main still prints ownerless search rows and the v1 search response/schema cannot carry owner identity. I did not need to run the live CLI because the missing fields are visible in the current source path.

## Solution Assessment

Yes for the narrow owner-disambiguated search/API change because internal search already has owner data. Full owner-qualified install locators are not the smallest maintainable fix and should remain a separate decision.
