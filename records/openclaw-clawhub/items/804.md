## Summary

Keep open under the ClawHub issue policy: current main has already fixed the original ownerless CLI search ambiguity, but the broader owner-qualified install locator remains a maintainer product decision.

## What This Changes

The issue asks ClawHub to make skill identity and install discovery owner-aware through clearer locators, page guidance, or an owner column in `clawhub search`.

## Best Possible Solution

Keep the shipped owner-aware search/API behavior; if maintainers want scoped installs, implement `@owner/name` as a deliberate CLI/API/filesystem/lockfile migration aligned with `specs/orgs.md`.

## Reproduction Assessment

No. Current source shows `clawhub search` now prints owner handles and the public v1 search contract returns owner metadata; the remaining `owner/slug` install syntax is an unsupported feature/product gap, not a current broken contract.

## Solution Assessment

Yes for the shipped discovery fix: a separate owner column resolves the reported wrong-result ambiguity without teaching an unsupported locator. Full scoped install support should be handled through the broader `@owner/name` migration rather than a narrow CLI-only patch.
