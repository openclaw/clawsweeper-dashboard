## Summary

Keep open. Current main still reproduces the reported owner ambiguity in CLI search and the public v1 search contract, and ClawHub cleanup policy keeps issues open rather than auto-closing them.

## What This Changes

The issue requests clearer owner-qualified identity in ClawHub skill search/install discovery so users can distinguish skills with the same or similar slugs.

## Best Possible Solution

Expose owner identity additively in `/api/v1/search`, update both schema copies, render owner information in `clawhub search`, and document the output while leaving owner-qualified install locators for a separate maintainer decision.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: internal search entries include `ownerHandle` and `owner`, but the v1 handler, schemas, and CLI renderer drop that identity before `clawhub search` output.

## Solution Assessment

Yes for the narrow owner-disambiguated search/API display path. Adding full owner-qualified install syntax is not the best first fix because current install validation deliberately rejects slash-containing locators before filesystem and lockfile operations.
