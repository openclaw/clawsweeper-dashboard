## Summary

Keep open. Current main still leaves the reported CLI discovery ambiguity unresolved: CLI/public v1 search do not expose owner identity, while `clawhub install` still accepts only bare slugs; the web install prompt only partially mitigates this outside the CLI path.

## What This Changes

The issue requests clearer owner-qualified skill identity for search and install, such as owner/slug syntax, skill-page install guidance, or an owner column in CLI search output.

## Best Possible Solution

Expose owner identity additively in public v1 search, update shared and CLI-local schemas plus API docs, and render owner information in `clawhub search`; leave full owner-qualified install syntax for a separate maintainer-reviewed locator design.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: current main’s v1 search response and CLI output omit owner fields, while install rejects slash-qualified locators before resolving or downloading a skill.

## Solution Assessment

Yes for the narrowed owner-display repair; it is the smallest maintainable fix for the reporter’s discovery problem. Full owner-qualified install support should not be bundled into an automated repair without explicit path and locator design review.
