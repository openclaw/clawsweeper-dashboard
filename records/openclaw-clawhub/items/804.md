- `impact:other`: Missing publisher identity could cause users to select and install a different public skill than intended.

## Summary

Current main resolves the reported ambiguity by displaying each search result’s publisher handle beside its installable slug. The broader owner/slug locator is unnecessary to solve the central mis-selection problem.

## What This Changes

Expose publisher identity in skill search or otherwise clarify the correct install target for similarly named skills.

## Best Possible Solution

Retain the compatibility-preserving bare-slug install contract and keep publisher handles visible in search output so users can reliably select the intended skill.

## Reproduction Assessment

No on current main. Search output now includes the publisher handle beside every result’s installable slug, and regression coverage verifies the rendered output.

## Solution Assessment

Yes. Displaying publisher identity separately is the narrowest maintainable solution because it fixes the ambiguity without expanding install, lockfile, and filesystem contracts to accept owner/slug locators.
