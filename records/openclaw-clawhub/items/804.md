## Summary

Keep open. Repository policy for this review lane keeps ClawHub issues open, and current main still leaves the core CLI search/API discovery ambiguity unresolved even though the web skill-detail prompt now shows owner-qualified identity.

## What This Changes

The issue asks for clearer owner-qualified skill identity in ClawHub search/install flows so users can distinguish similarly named skills such as `steipete/weather` from other `weather` entries.

## Best Possible Solution

Expose owner identity additively in `/api/v1/search`, update the shared schema and CLI output to show an owner-qualified display target, and leave owner-qualified install syntax as a separate maintainer decision unless explicitly scoped in.

## Reproduction Assessment

Yes for the remaining behavior: source inspection shows current main still prints ownerless `clawhub search` rows, omits owner fields from the public search response/schema, and rejects `owner/slug` install targets before resolution.

## Solution Assessment

Unclear for the full request. The narrow owner-disambiguated search/API change is the safest maintainable repair, while owner-qualified install locators should be split into a product/security decision because install paths currently reject separators.
