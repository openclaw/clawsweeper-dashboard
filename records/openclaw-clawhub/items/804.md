## Summary

Keep open. Current main still leaves the reported CLI discovery ambiguity unresolved: search/API output omits owner identity, install accepts only bare slugs, and the web prompt surface only partially mitigates the confusion.

## What This Changes

Request clearer owner-qualified skill identity for ClawHub search/install discovery, such as owner/slug install targets, a detail-page install hint, or owner columns in CLI search.

## Best Possible Solution

Expose owner identity additively in the public v1 search response and CLI search output, document the output/API change, and keep owner-qualified install locator semantics as a separate scoped decision.

## Reproduction Assessment

Yes for the remaining ambiguity by source inspection: current main prints ownerless search rows, omits owner fields from the public search response/schema, and rejects `owner/slug` install targets before resolution.

## Solution Assessment

Unclear for the full owner-qualified install syntax, which needs separate safety/product scoping. The narrow owner-disambiguated search/API change is the most maintainable next step because internal search already carries owner identity.
