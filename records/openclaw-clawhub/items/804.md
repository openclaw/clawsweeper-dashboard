## Summary

Keep open. Current main still leaves the reported CLI discovery ambiguity unresolved: the public search API and `clawhub search` output omit owner identity even though internal search entries already carry it, and ClawHub cleanup policy keeps issues open rather than auto-closing them.

## What This Changes

Request clearer owner-qualified skill identity or CLI/web cues so users can distinguish skills like `steipete/weather` from other weather-related results.

## Best Possible Solution

Expose owner identity additively in `/api/v1/search`, update both schema copies, show owner context in `clawhub search`, and keep slash-containing install locators as a separately scoped decision.

## Reproduction Assessment

Yes, by source inspection: current main's `cmdSearch` prints ownerless rows, the v1 search response omits owner fields, and both search schemas lack owner identity. I did not run the live CLI in this read-only review.

## Solution Assessment

Yes for the narrow owner-disambiguated search/API change because internal search already carries owner identity. Full owner-qualified install locators are not the narrowest first fix and should remain separately scoped.
