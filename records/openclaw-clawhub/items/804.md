## Summary

Keep open under the ClawHub issue cleanup policy: current main has shipped owner-aware search/API/docs, but any remaining `owner/slug` install locator is a separate product and contract decision rather than an auto-closeable issue.

## What This Changes

Request to make ClawHub skill discovery and install targeting less ambiguous by showing owner identity or supporting owner-qualified skill locators.

## Best Possible Solution

Keep the shipped owner-aware search behavior; if maintainers want owner-qualified installs, design a separate CLI/API/lockfile-safe locator contract.

## Reproduction Assessment

No. The original ownerless `clawhub search` ambiguity no longer reproduces on current main because the CLI prints owner handles and the API returns owner metadata; only unsupported `owner/slug` install syntax remains source-confirmed.

## Solution Assessment

Yes for the narrow discovery ambiguity: owner-aware search keeps the installable bare slug first without teaching an unsupported locator. Full owner-qualified installs need explicit maintainer product and compatibility design.
