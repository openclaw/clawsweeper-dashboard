## Summary

Keep open. Current main still leaves the reported CLI discovery ambiguity unresolved: public search and `clawhub search` do not expose owner identity, while `clawhub install` still rejects owner-qualified locators.

## What This Changes

Request clearer owner-qualified skill identity in ClawHub discovery so users can distinguish similarly named skills and avoid installing the wrong bare slug.

## Best Possible Solution

Expose owner identity additively in public v1 search, update shared and CLI-local schemas/docs, and render owner context in `clawhub search`; leave owner-qualified install syntax for a separate locator design.

## Reproduction Assessment

Yes. Source inspection on current main shows `/api/v1/search` maps away owner fields, the CLI schema/output has no owner field, and install rejects slashes before resolution.

## Solution Assessment

Yes for the narrowed owner-display repair. It is the smallest maintainable fix for the reported discovery problem, while full owner-qualified install support should be handled separately.
