## Summary

Keep open. ClawHub policy keeps issues open, and current main still leaves the reported owner-disambiguation gap in CLI/public search while the web install prompt only partially mitigates it.

## What This Changes

The issue asks for clearer owner-qualified skill identity in ClawHub search/install flows so users can distinguish similarly named skills such as `steipete/weather`.

## Best Possible Solution

Expose owner identity additively in public v1 search, update schemas/docs/tests, and render owner information in `clawhub search`; leave owner-qualified install syntax for separate design review.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current CLI search output and the public v1 search contract omit owner fields, while CLI install rejects slash-qualified locators before resolving a skill.

## Solution Assessment

Yes for the narrowed owner-display repair. It is the smallest maintainable path for the reporter’s discovery problem because internal search already has owner identity, while changing install locator semantics is broader.
