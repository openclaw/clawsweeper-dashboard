## Summary

Keep open. Current main still leaves the reported CLI discovery ambiguity unresolved: public search and CLI search results do not expose owner identity, and install still accepts only bare slugs. This is a narrow actionable issue, and ClawHub cleanup policy keeps issues open rather than auto-closing them.

## What This Changes

The issue asks for clearer owner-qualified skill identity in ClawHub search/install flows so users can distinguish entries like `steipete/weather` from other skills with similar bare slugs.

## Best Possible Solution

Expose owner identity additively in the public v1 search response, update shared/CLI schemas plus API docs, and render owner information in `clawhub search` so users can distinguish same-slug or similar-slug results. Keep owner-qualified install syntax as a separate product/security decision unless maintainers widen the work.
