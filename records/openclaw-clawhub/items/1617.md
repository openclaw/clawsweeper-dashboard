## Summary

Keep open. ClawHub cleanup policy keeps issues open, and the underlying report still has a real consistency gap: current main partially supports optional environment metadata through `envVars` with `required: false`, but the skill-format docs, CLI schema copy, and security prompt summary are not fully aligned. Open draft #1859 is already the active implementation candidate and references this issue with closing syntax.

## What This Changes

Clarify how skill authors should declare optional environment variables without incorrectly marking them as required.

## Best Possible Solution

Align on one supported optional-env declaration, likely `metadata.openclaw.envVars` entries with `required: false` unless maintainers choose `requires.optionalEnv`, then document it, update the CLI schema copy, keep parser/moderation/security-prompt behavior consistent, and land regression coverage through #1859 or an equivalent maintainer-approved change.
