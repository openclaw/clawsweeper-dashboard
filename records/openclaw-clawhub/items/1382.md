    22	## API tokens (CLI)
    23	
    24	The CLI uses a long-lived API token (Bearer token) for publish/sync/delete.
    25	
    26	### Browser flow (default)
    27	
    28	`clawhub login` does:
    29	
    30	1. Starts a loopback HTTP server on `127.0.0.1` (random port).
    31	2. Opens `<site>/cli/auth?redirect_uri=http://127.0.0.1:<port>/callback&state=...`.
    32	3. Web UI requires GitHub login, then creates a token and redirects back to the loopback server.
    33	4. CLI stores the token in the global config file.
    34	
    35	### Headless flow
    36	
    37	Create a token in the web UI (Settings → API tokens) and paste it:
    38	
    39	```bash
    40	clawhub login --token clh_...
    41	```
    42	
    43	### Token storage
    44	
    45	Default global config path:
