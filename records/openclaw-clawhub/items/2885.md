Full review comments:

- **[P2] Query active invites instead of the first pending row:** `convex/publishers.ts:3324-3329`
  - body: This duplicate check only inspects the first pending invite for the target. Because expired pending invites remain until the retention cron deletes them, an older expired row can be returned first while a later active invite exists, letting the mutation create a second active invitation for the same org and handle. Add/use an index that includes `expiresAt` for the publisher+target+status lookup, or otherwise check the bounded active set before inserting.
  - confidence: 0.91
- **[P2] Align the repair invariant with admin tooling:** `specs/orgs.md:292-294`
  - body: The new spec says direct staff repair paths require a reason and audit event, but the existing admin publisher ensure path still accepts `memberHandle`/`memberRole` and forwards them to `ensureOrgPublisherHandleInternal` without any reason. Either require and persist a reason for that admin member-add/update path or narrow the spec so it does not claim a guarantee the current API lacks.
  - confidence: 0.86
