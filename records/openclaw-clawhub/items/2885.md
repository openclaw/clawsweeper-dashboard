- **[P2] Check every active pending invite before inserting:** `convex/publishers.ts:3324-3330`
  - body: This duplicate check only takes the first `pending` row for the publisher/target/status bucket. After an older pending invite expires and a new active invite is inserted, the next create can keep seeing the older expired row first and insert another active invite for the same target. Query by an expiry-aware index or inspect enough matching rows so one unexpired pending invite blocks duplicates.
  - confidence: 0.88
- **[P2] Align the repair invariant with the admin path:** `specs/orgs.md:292-294`
  - body: The added invariant says direct staff repair paths must require a reason, but the existing admin org create/update endpoint still forwards `memberHandle`/`memberRole` to the internal member upsert without accepting or auditing a reason. Either require and persist a reason on that admin path, or narrow the spec so it does not promise a guarantee the code does not enforce.
  - confidence: 0.86
