## Summary

Keep open: current main still does not implement weekly/monthly period popularity across publishers, skills, and plugins. Recent main added install-focused ranking, but the shared temporal stats, publisher/package rollups, snapshot-pinned leaderboards, and UI period controls remain product/data-contract work.

## What This Changes

The issue requests weekly/monthly/all-time popularity tabs for publishers, skills, and plugins backed by a shared temporal popularity data contract.

## Best Possible Solution

Keep this as the canonical RFC, decide the period semantics and attribution rules, then land daily package/publisher stats and snapshot-pinned leaderboards before adding UI tabs.

## Reproduction Assessment

Not applicable: this is an RFC for a new discovery capability, not a broken existing behavior report. Source inspection confirms the requested Weekly/Monthly surfaces and backend contract are still absent on current main.

## Solution Assessment
