## Summary

Current main still provides only skill-specific seven-day trending plus all-time or download-based publisher and plugin ranking; it does not implement the RFC’s aligned weekly/monthly popularity contract for publishers, skills, and plugins, and no canonical successor was found.

## What This Changes

Add weekly, monthly, and all-time popularity across publisher, skill, and plugin browse surfaces, backed by consistent temporal statistics and stable pagination.

## Best Possible Solution

Keep this RFC as the decision record, settle the temporal semantics, then deliver separate foundation, leaderboard-contract, and UI PRs with indexed rollups and snapshot-pinned pagination before exposing the tabs.

## Reproduction Assessment

Not applicable; this RFC requests a new weekly/monthly popularity capability rather than reporting broken behavior under an existing contract. Current-main source inspection confirms the capability is absent.

## Solution Assessment

Yes at the architectural level: indexed daily rollups and snapshot-pinned leaderboards are the maintainable alternative to request-time scans or misleading skill-only publisher rankings, but maintainers must first select the scoring and attribution rules.
