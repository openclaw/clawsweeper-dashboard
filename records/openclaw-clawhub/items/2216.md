The issue requests period-based popularity tabs and supporting temporal leaderboard infrastructure across publisher, skill, and plugin browse surfaces.

## Best Possible Solution

Settle the shared period semantics first, then implement bounded package and publisher daily rollups plus snapshot-pinned Weekly/Monthly leaderboards before adding browse tabs.
