## Summary

Keep open. Current main still lacks the shared weekly/monthly popularity contract for publishers, skills, and plugins, and the RFC remains a product/data-model decision rather than a narrow bug fix.

## What This Changes

RFC proposes weekly, monthly, and all-time popularity tabs across ClawHub publisher, skill, and plugin discovery surfaces.

## Best Possible Solution

Settle shared period semantics first, then add bounded package and publisher daily rollups plus snapshot-pinned weekly/monthly leaderboards before adding the UI tabs.

## Reproduction Assessment

Not applicable. This is an RFC for a new discovery capability; source inspection confirms the requested cross-surface period contract is not implemented on current main.

## Solution Assessment

Yes. The RFC-first direction is the narrow maintainable path because a UI-only implementation would risk misleading publisher and plugin popularity rankings before the backend data contract exists.
