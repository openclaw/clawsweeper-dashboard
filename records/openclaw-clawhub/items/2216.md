## Summary

Keep open. This is a current RFC for a new temporal discovery feature, and current main still lacks a shared Weekly/Monthly popularity data contract for publishers, skills, and plugins.

## What This Changes

The issue requests period tabs for publisher, skill, and plugin browse pages so discovery can rank by Weekly, Monthly, and All time popularity.

## Best Possible Solution

Settle shared period semantics first, then implement bounded package and publisher daily rollups plus snapshot-pinned period leaderboards before adding UI tabs.

## Reproduction Assessment

Not applicable. This is an RFC for new discovery behavior; source inspection confirms the shared period contract is absent, but no existing documented behavior is broken.

## Solution Assessment

Yes. The RFC-first path is the maintainable solution because a UI-only implementation would risk misleading publisher and plugin popularity rankings.
