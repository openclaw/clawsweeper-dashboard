## Summary

Keep open. This is a current RFC for a new temporal discovery feature; current main still has all-time/update-based browse paths and no shared Weekly/Monthly popularity contract across publishers, skills, and plugins.

## What This Changes

Request to add Weekly, Monthly, and All time discovery periods for publishers, skills, and plugins backed by correct temporal popularity data.

## Best Possible Solution

Settle shared period semantics, then implement bounded package and publisher daily rollups plus snapshot-pinned period leaderboards before adding tabs to the three browse surfaces.

## Reproduction Assessment

Not applicable. This is an RFC for new discovery behavior; source inspection confirms the shared period contract is absent, but no existing documented behavior is broken.

## Solution Assessment

Yes. The RFC-first path is appropriate because UI-only tabs would risk misleading publisher and plugin popularity rankings before the data contract is correct.
