## Summary

Keep open. Current main still has all-time or updated-date browse paths plus a limited skill trending endpoint, but not the shared weekly/monthly popularity contract for publishers, skills, and plugins; the remaining work also needs product decisions before implementation.

## What This Changes

The issue requests weekly, monthly, and all-time popularity tabs for publishers, skills, and plugins backed by a shared temporal popularity data contract.

## Best Possible Solution

Settle shared period semantics first, then add bounded package and publisher daily rollups plus snapshot-pinned weekly/monthly leaderboards before adding the UI tabs.

## Reproduction Assessment

Not applicable. This is an RFC for a new discovery capability; source inspection confirms the requested shared weekly/monthly contract is not implemented on current main.

## Solution Assessment

Yes. The RFC-first direction is the maintainable path because a UI-only implementation would risk misleading publisher and plugin popularity rankings before the backend data contract exists.
