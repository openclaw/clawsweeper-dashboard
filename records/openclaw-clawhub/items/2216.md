## Summary

Current main has only skill-specific temporal popularity, while publisher and plugin period ranking plus the shared UX remain unimplemented; this RFC should stay open for an explicit product/data-contract decision rather than be treated as a narrow fix.

## What This Changes

Add weekly, monthly, and all-time popularity tabs and a consistent temporal ranking foundation for publishers, skills, and plugins.

## Best Possible Solution

First record the agreed V1 period, score, ownership-attribution, backfill/warm-up, and snapshot-pagination contract in a focused spec, then land staged indexed daily aggregates and leaderboards before adding synchronized tabs to the three browse routes.

## Reproduction Assessment

Not applicable; this is a request for a new temporal ranking capability, and source inspection confirms the requested publisher/plugin/combined behavior does not currently exist.

## Solution Assessment

Yes as an RFC direction, but not as one immediate UI patch; the narrowest maintainable path is to decide and specify the shared data contract, then implement the backend foundations and UI in staged changes.
