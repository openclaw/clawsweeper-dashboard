## Summary

Keep open. This is a current RFC for a new temporal discovery feature, and current main still lacks the shared Weekly/Monthly popularity data contract and UI parameters across publishers, skills, and plugins.

## What This Changes

Add Weekly, Monthly, and All time popularity tabs to publishers, skills, and plugins, backed by a shared temporal popularity contract.

## Best Possible Solution

Settle shared period semantics first, then implement bounded package and publisher daily rollups plus snapshot-pinned period leaderboards before adding UI tabs.

## Reproduction Assessment

Not applicable. This is an RFC for new discovery behavior; source inspection confirms the shared period contract is absent, but no existing documented behavior is broken.

## Solution Assessment

Yes. The RFC-first path is the maintainable direction because a UI-only implementation would risk misleading publisher and plugin popularity rankings.
