## Summary

Keep open. Current main still lacks the shared period-popularity contract and UI state for publishers, skills, and plugins, and the issue is an active product RFC with unresolved metric, attribution, backfill, and pagination decisions.

## What This Changes

The issue proposes Weekly, Monthly, and All time discovery tabs for publishers, skills, and plugins backed by a shared temporal popularity contract.

## Best Possible Solution

Settle shared period semantics and attribution/backfill policy first, then add bounded package and publisher daily rollups plus snapshot-pinned weekly/monthly leaderboards before wiring the UI tabs.

## Reproduction Assessment

Not applicable. This is an RFC for a new discovery capability, and source inspection confirms the requested shared period contract is still missing on current main.

## Solution Assessment

Yes, the RFC-first direction is the right shape. A UI-only change would be misleading until maintainers choose the metric semantics and add package/publisher temporal rollups.
