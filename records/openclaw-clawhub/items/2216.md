## Summary

Keep open. This is a current ClawHub feature/RFC issue that needs product and data-contract decisions, and current main still lacks shared Weekly/Monthly popularity support across publishers, skills, and plugins.

## What This Changes

RFC requests Weekly, Monthly, and All time popularity tabs for publishers, skills, and plugins backed by a shared temporal popularity data contract.

## Best Possible Solution

Settle shared period semantics first, then implement bounded package and publisher daily rollups, snapshot-pinned weekly/monthly leaderboards, and UI tabs backed by that contract.

## Reproduction Assessment

Not applicable. This is an RFC for new discovery behavior; source inspection confirms the requested shared period contract is absent, but there is no broken existing contract to reproduce.

## Solution Assessment

Yes. The RFC-first direction is the maintainable path because UI-only period tabs would risk misleading rankings while package and publisher temporal rollups do not exist.
