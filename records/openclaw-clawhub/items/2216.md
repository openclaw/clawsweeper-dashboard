## Summary

Keep open. Current main still has all-time or updated-date browse paths plus a limited skill trending endpoint, not the shared weekly/monthly popularity contract for publishers, skills, and plugins.

## What This Changes

The issue proposes weekly/monthly/all-time popularity tabs for publishers, skills, and plugins, backed by shared temporal leaderboard data instead of UI-only sorting.

## Best Possible Solution

Settle shared period semantics first, then add bounded package and publisher daily rollups plus snapshot-pinned weekly/monthly leaderboards before adding the UI tabs.

## Reproduction Assessment

Not applicable. This is an RFC for a new discovery capability; source inspection confirms the requested shared weekly/monthly contract is not implemented on current main.

## Solution Assessment

Yes at the direction level. The RFC-first path is the maintainable solution because UI-only tabs would risk misleading publisher and plugin rankings before the backend period-popularity contract exists.
