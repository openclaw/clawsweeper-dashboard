## Summary

Keep open: current main still has only skill-scoped temporal stats and lacks the shared publisher/plugin period-popularity contract and UI requested here; this remains a substantive product and backend design RFC with no canonical replacement.

## What This Changes

Add aligned weekly, monthly, and all-time popularity ranking and browse controls for publishers, skills, and plugins.

## Best Possible Solution

Approve a documented V1 score, completed-day windows, ownership attribution, and warm-up/backfill policy, then land staged indexed daily rollups, snapshot-stable leaderboards, and aligned period controls across all three browse surfaces.

## Reproduction Assessment

Not applicable: this is a well-specified request for new discovery behavior, and source inspection directly confirms that the capability is absent rather than intermittently broken.

## Solution Assessment

Yes at the architectural level: a shared temporal data foundation is safer and more maintainable than exposing skill-only or raw-event-backed tabs, but maintainers must first settle scoring, attribution, and rollout policy.
