## Summary

Keep open: current main still lacks the shared Weekly/Monthly/All time popularity contract across publishers, skills, and plugins. The existing code has all-time publisher/plugin/skill browse ranking and a skill-only trending helper, but not package/publisher daily stats, period leaderboards, snapshot-pinned pagination, or the requested UI tabs.

## What This Changes

The RFC requests period-based popularity tabs for publishers, skills, and plugins backed by shared temporal stats, rollups, leaderboards, and stable pagination.

## Best Possible Solution

Keep this as the canonical RFC, decide period semantics and attribution rules, then land daily package/publisher stats and snapshot-pinned leaderboards before adding UI tabs.

## Reproduction Assessment

Not applicable: this is an RFC for a new discovery capability, not a broken existing behavior report. Source inspection confirms the requested Weekly/Monthly surfaces and backend contract are still absent on current main.

## Solution Assessment

Yes: treating this as an RFC before implementation remains the best path. The current source shows visible tabs would be misleading until package and publisher daily stats plus a period leaderboard contract exist.
