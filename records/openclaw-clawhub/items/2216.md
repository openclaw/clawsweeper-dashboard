## Summary

Keep open. This is a current product RFC for new temporal discovery behavior, and current main still lacks the shared period popularity contract for publishers, skills, and plugins.

## What This Changes

The issue requests Weekly, Monthly, and All time popularity tabs for publishers, skills, and plugins backed by a consistent temporal popularity model.

## Best Possible Solution

Settle shared period semantics first, then add bounded package and publisher daily rollups plus snapshot-pinned weekly/monthly leaderboards before adding the UI tabs.

## Reproduction Assessment

Not applicable. This is an RFC for a new discovery capability; source inspection confirms the requested shared period contract is not implemented on current main.

## Solution Assessment

Yes. The RFC-first direction is the narrow maintainable path because a UI-only implementation would risk misleading publisher and plugin popularity rankings before the backend data contract exists.
