## Summary

Keep open: current main still lacks the shared Weekly/Monthly/All time popularity contract and UI tabs for publishers, skills, and plugins, and the remaining work needs product choices plus Convex data-foundation work.

## What This Changes

The RFC requests period-based popularity tabs for publisher, skill, and plugin browse surfaces backed by semantically aligned weekly, monthly, and all-time ranking data.

## Best Possible Solution

Keep the RFC open until maintainers choose period semantics, then implement staged daily package/publisher stats, snapshot-pinned leaderboards, and UI tabs against that shared contract.

## Reproduction Assessment

## Solution Assessment

Yes, the RFC-first direction is appropriate. The safest path is to settle scoring and attribution rules, add daily package/publisher rollups and snapshot-pinned leaderboards, then ship UI tabs against that contract.
