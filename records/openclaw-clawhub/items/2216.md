## Summary

Current main still lacks a semantically aligned weekly/monthly popularity contract for publishers, skills, and plugins; the issue remains necessary because its scoring, ownership-attribution, rollout, and pagination choices require an explicit product decision.

## What This Changes

Add weekly, monthly, and all-time popularity modes across publisher, skill, and plugin browse pages using correct temporal data contracts.

## Best Possible Solution

Retain this RFC as the canonical design item, record the accepted scoring, attribution, period, snapshot-retention, and rollout semantics in a focused spec, then implement the foundation and UI as several bounded PRs.

## Reproduction Assessment

Not applicable; this is a feature RFC rather than a broken-behavior report. Current-main source inspection nevertheless provides a high-confidence check that the requested cross-entity period contract and UI controls are absent.

## Solution Assessment

Yes at the architectural level: indexed daily rollups and snapshot-stable browse contracts should precede UI tabs. The exact scoring, ownership attribution, and rollout behavior cannot be selected safely without maintainer product judgment.
