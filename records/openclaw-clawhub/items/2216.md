## Summary

Current main still provides only a bounded skill-trending foundation; publishers and plugins lack equivalent temporal rollups, paginated leaderboards, and period selectors, and no separate issue or viable PR supersedes this RFC. The item should remain open for explicit product and data-contract decisions before implementation.

## What This Changes

Add weekly, monthly, and all-time popularity browsing across publisher, skill, and plugin discovery surfaces after establishing a consistent temporal ranking data contract.

## Best Possible Solution

Keep this issue as the canonical RFC, approve simple shared period semantics and attribution rules, then implement indexed package and publisher daily rollups plus snapshot-pinned paginated leaderboards before exposing any period tabs.

## Reproduction Assessment

Not applicable: this RFC requests new weekly and monthly discovery modes rather than reporting broken behavior in an existing documented period-ranking contract.

## Solution Assessment

Yes at the architectural boundary: a shared, indexed temporal foundation is safer and more maintainable than partial or misleading UI tabs, but maintainers must resolve the scoring, attribution, backfill, and snapshot semantics first.
