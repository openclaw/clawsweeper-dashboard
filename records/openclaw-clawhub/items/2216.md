## Summary

Keep open. This is an active RFC for a new cross-surface temporal popularity feature, and current main still lacks a shared Weekly/Monthly contract for publishers, skills, and plugins.

## What This Changes

Request to define and later add Weekly/Monthly/All time popularity tabs plus backend temporal leaderboard support for publisher, skill, and plugin browse pages.

## Best Possible Solution

Settle shared period semantics first, then implement bounded daily package and publisher rollups, snapshot-pinned weekly/monthly leaderboards, and UI tabs backed by that contract.
