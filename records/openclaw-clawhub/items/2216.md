## Summary

Current main does not implement aligned weekly/monthly popularity for publishers, skills, and plugins: only skills have a capped seven-day trending snapshot, while publisher ranking remains all-time and plugin browsing remains recency-based. No canonical replacement issue or implementation PR was found, so this RFC remains necessary but needs explicit maintainer product decisions before a large staged backend and UI program begins.

## What This Changes

Add consistent Weekly, Monthly, and All time popularity modes to publisher, skill, and plugin browse surfaces, backed by bounded temporal statistics and stable leaderboard pagination.

## Best Possible Solution

Approve and record period/scoring/attribution semantics in a focused spec, then deliver separate migration-safe PRs for package and publisher daily rollups, snapshot-paginated entity leaderboards, and finally the three aligned browse controls while retaining existing indexed all-time paths.

## Reproduction Assessment

Not applicable: this RFC proposes a new browse and ranking capability, and current-source inspection directly confirms that the cross-entity weekly/monthly contract is absent.

## Solution Assessment

Yes in broad direction: establishing indexed temporal rollups and stable leaderboard pagination before adding tabs is the narrowest honest architecture, but maintainers must first decide scoring, attribution, threshold, and backfill semantics.
