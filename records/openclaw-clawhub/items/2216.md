## Summary

Keep open. This is an active RFC for a new temporal popularity contract, and current main still only has partial skill trending plus all-time or recency browse paths for the requested publishers, skills, and plugins surfaces.

## What This Changes

Request period-based Weekly, Monthly, and All time popularity tabs plus shared backing stats and leaderboards for publishers, skills, and plugins.

## Best Possible Solution

Settle the product semantics first, then add bounded daily package and publisher stats, snapshot-pinned weekly/monthly leaderboards, and finally wire the three browse UIs to that shared contract.

## Reproduction Assessment

Not applicable. This is an RFC for new discovery behavior; source inspection confirms the requested period contract is not present, but there is no broken existing contract to reproduce.

## Solution Assessment

No, not as an immediate UI-only change. The maintainable path is to decide the shared temporal popularity semantics and backend rollups before exposing Weekly or Monthly tabs.
