## Summary

Keep open. This is an active cross-surface product RFC, and current main still lacks the shared Weekly/Monthly popularity data contract for publishers, skills, and plugins; no superseding issue or PR was found.

## What This Changes

Add Weekly, Monthly, and All time popularity periods to publishers, skills, and plugins discovery using a shared temporal stats and leaderboard contract.

## Best Possible Solution

Settle shared period semantics first, then add bounded package and publisher daily rollups, snapshot-pinned weekly/monthly leaderboards, and UI tabs backed by that contract.

## Reproduction Assessment

Not applicable. This is an RFC for new discovery behavior; source inspection confirms the requested period contract is absent, but there is no broken existing contract to reproduce.

## Solution Assessment

No, not as an immediate UI-only change. The maintainable path is to decide and build the shared temporal stats and leaderboard contract before exposing Weekly or Monthly tabs.
