## Summary

Keep open. This is a current RFC for a new temporal discovery feature, and current main still lacks a shared Weekly/Monthly popularity contract for publishers, skills, and plugins.

## What This Changes

Request to add Weekly, Monthly, and All time popularity tabs for publishers, skills, and plugins backed by correct temporal ranking data.

## Best Possible Solution

Settle the shared period semantics first, then implement bounded package and publisher daily rollups plus snapshot-pinned period leaderboards before adding UI tabs.

## Reproduction Assessment

Not applicable. This is an RFC for new discovery behavior; source inspection confirms the shared period contract is absent, but no existing documented behavior is broken.

## Solution Assessment

Yes. The RFC-first path is the maintainable solution because a UI-only implementation would risk misleading publisher and plugin popularity rankings.
