## Summary

Keep open. Current main still has a source-reproducible path where an ordinary non-trusted publisher updates an existing active skill, the skill row is patched to hidden pending scan, and public detail/search/API surfaces filter the row out; ClawHub cleanup policy also keeps issues open for maintainer resolution.

## What This Changes

A publisher reports that updating an already-published skill can return success or a server error while making the skill disappear from search, inspect, direct page, and API visibility.

## Best Possible Solution

Define and implement explicit update continuity: keep the last approved public version visible while the new upload remains gated by scan/moderation, or intentionally quarantine the whole listing with consistent owner/public UX and API messages.

## Reproduction Assessment

Yes, at source level. An ordinary non-trusted publisher updating an existing active skill reaches insertVersion, which can write hidden pending-scan state to the skill row, and public conversion/search/detail/API paths then drop it; I did not run a live Convex publish.

## Solution Assessment

Unclear because no patch is under review. The direct preserve-active-status approach in https://github.com/openclaw/clawhub/pull/274 was closed as conflicting with moderation policy, so the safer direction is previous-approved-version continuity while the new upload remains gated.
