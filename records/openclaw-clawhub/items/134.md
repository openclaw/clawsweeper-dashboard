## Summary

Keep open. ClawHub cleanup policy only allows auto-closing PRs, and current main still has a credible source-level path where an ordinary publisher’s update to an active skill is patched to hidden pending scan and then filtered out of public detail, search, catalog, and API surfaces.

## What This Changes

Publishing an update to an already-visible skill can report success or a server error while the skill disappears from public search and inspect because it is hidden pending scan.

## Best Possible Solution

Define the active-skill update moderation policy and implement it consistently across publish, dashboard/detail, search/catalog, CLI inspect, and the public API, preferably keeping the last approved public listing visible while a new version scans unless a hard block applies.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an ordinary non-trusted publisher updating an existing active skill reaches insertVersion, gets hidden pending scan, and public conversion/detail/search/API paths then drop it; I did not run the live Convex/web UI path.

## Solution Assessment

Unclear. The closed preserve-active-status PR matches the user-visible outcome, but maintainers first need to choose the security policy for already-active listings while a new upload is scanning.
