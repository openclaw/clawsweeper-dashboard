## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has a credible source-level path where an ordinary publisher's update can hide an existing active skill while public search, detail, inspect/API, and catalog paths suppress hidden skills.

## What This Changes

Publishing an updated skill can report success or a server error while the existing skill disappears from public search, inspect, direct detail, and API surfaces.

## Best Possible Solution

Define and implement explicit update moderation: keep the previous approved listing/version public while the new upload scans, or intentionally quarantine updates with clear owner, CLI, API, search, and recovery behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction path exists: publish a higher version for an existing active skill as a non-trusted ordinary publisher, observe insertVersion patch the skill to hidden pending scan, then verify public detail/search/API filters suppress it.

## Solution Assessment

Unclear. A naive preserve-active-status patch is directionally close but not necessarily the safest fix unless it prevents the new unscanned latest version from becoming public before scan resolution.
