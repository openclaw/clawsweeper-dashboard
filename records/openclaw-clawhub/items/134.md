## Summary

Keep open. ClawHub policy keeps issues open, and current main still has a credible source-reproducible update-disappearance path for ordinary non-trusted publishers: publish updates can patch an existing skill to hidden, then public detail, search, and API paths filter hidden skills out.

## What This Changes

The issue reports that publishing an update to an existing visible skill returns success or a server error, then the skill disappears from inspect, search, and public listings.

## Best Possible Solution

Define and implement the update moderation policy for already-active skills so legitimate listings do not vanish without a clear owner/public state, while unscanned or suspicious new version content remains gated appropriately.

## Reproduction Assessment

Yes from source inspection, not from a live deployment run. An update by an ordinary non-trusted publisher can compute hidden pending scan, patch the existing skill to hidden, and then public detail/search/API paths drop that hidden row.

## Solution Assessment

Unclear. Preserving active status for updates, as attempted in #274, is plausible but needs maintainer policy because a naive fix could expose new version content before scanning or lose moderation metadata.
