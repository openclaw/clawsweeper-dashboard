## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has a credible source-level path where an ordinary non-trusted publisher updating an active skill can move it to hidden pending scan while public detail, search, catalog, and API surfaces suppress hidden skills.

## What This Changes

Publishing an update to an existing skill can make the listing disappear from search, inspect, and public skill pages after `skills:publishVersion` reports a server error.

## Best Possible Solution

Define the update moderation policy explicitly, then either keep the previous approved listing/version public while the new upload scans or make quarantine status consistent across CLI, web detail/dashboard, API, search, and recovery flows.

## Reproduction Assessment

Yes. A high-confidence source reproduction path exists: publish a higher version for an existing active skill as an ordinary non-trusted publisher with a non-malicious static scan, observe `insertVersion` patch the skill to hidden pending scan, then verify public reads/search/API filter it out.

## Solution Assessment

Unclear. Preserving `active` exactly as the closed PR #274 proposed would stop disappearance, but the safer maintainable solution must avoid exposing newly uploaded unreviewed content before scanner results settle.
