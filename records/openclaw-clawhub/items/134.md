## Summary

Keep open. ClawHub cleanup policy does not auto-close issues, and current main still has a source-level path where an ordinary non-trusted publisher updating an active skill can move it to hidden pending scan while public detail, catalog, search, and API surfaces suppress hidden skills.

## What This Changes

The issue reports that publishing an updated skill version can return success or a server error while the existing skill disappears from search, inspect, and public pages.

## Best Possible Solution

Define update moderation semantics for already-active skills, preferably keeping the previous approved public listing/version visible while a newly uploaded version scans, or documenting and surfacing quarantine consistently across CLI, web, API, search, and recovery flows.

## Reproduction Assessment

Yes. A high-confidence source reproduction path exists: publish a higher version for an existing active skill as an ordinary non-trusted publisher, observe insertVersion patch the skill to hidden pending scan, then verify public detail/search/API filters suppress hidden skills.

## Solution Assessment

Unclear. A narrow preserve-active-status patch is directionally close, but the maintainable fix must choose a moderation policy that keeps prior approved content visible without serving the unreviewed new version before scanner results resolve.
