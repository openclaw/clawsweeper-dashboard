## Summary

Keep open. ClawHub cleanup policy does not auto-close issues, and current main still has a source-level path where an ordinary publisher updating an active skill can move the skill to hidden pending scan while public detail, catalog, search, and API surfaces suppress hidden skills.

## What This Changes

Publishing an update to an already-visible skill should not make the skill disappear from inspect, search, public listings, or the web UI without a clear recoverable moderation state.

## Best Possible Solution

Define and implement explicit update moderation: keep the previous approved public listing/version visible while the new upload scans, or intentionally quarantine updates with clear owner, CLI, API, search, and recovery behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction path exists: publish a higher version for an existing active skill as a non-trusted ordinary publisher, observe insertVersion patch the skill to hidden pending scan, then verify public detail/search/API filters suppress it.

## Solution Assessment

Unclear. A direct preserve-active-status fix is directionally close, but the maintainable solution must avoid exposing the newly uploaded unreviewed latest version before scanner results are resolved.
