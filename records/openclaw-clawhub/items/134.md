## Summary

Keep open. Current main still has a source-reproducible path where an ordinary non-trusted update can hide the existing skill row, and ClawHub cleanup policy does not auto-close issues.

## What This Changes

The issue reports that publishing an update to an existing visible skill can make it disappear from inspect, search, and public detail/API surfaces.

## Best Possible Solution

Implement version-aware public continuity: keep the last approved skill card, digest, API metadata, public file surface, and searchable embedding visible while the new version remains scanner/moderation-gated, then promote only after the gates clear.

## Reproduction Assessment

Yes, at source level. An ordinary non-trusted publisher updating an existing active skill reaches insertVersion, which can write hidden pending-scan state to the skill row, and public conversion/search/detail/API paths then filter it out; I did not run a live Convex publish in this read-only review.

## Solution Assessment

No patch is under review, and the direct preserve-active-row approach is not the best fix because it was closed as a moderation bypass. The safer direction is a last-approved public projection while the new upload remains gated until scanner/moderation finalization.
