## Summary

Keep open. Current main still has a source-level path where an ordinary non-trusted publisher updating an existing active skill is patched to hidden pending scan, then public detail, search, catalog, and API surfaces filter it out.

## What This Changes

The issue reports that updating a published skill via CLI or web publish can make the skill disappear from inspect/search while `skills:publishVersion` surfaces a server error.

## Best Possible Solution

Define and implement one consistent active-skill update moderation policy across publish, dashboard/detail, search/catalog, CLI inspect, and the public API, preferably keeping the last approved public listing visible while the new version scans unless a hard block applies.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an ordinary non-trusted publisher updating an existing active skill reaches `insertVersion`, gets hidden pending-scan state, and public conversion/detail/search/API paths drop it; I did not run the live Convex/web UI path.

## Solution Assessment

Unclear. The closed preserve-active-status direction matches the user-visible outcome, but maintainers first need to choose the security policy for already-active listings while a new upload is scanning.
