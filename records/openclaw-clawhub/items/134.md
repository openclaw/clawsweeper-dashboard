## Summary

Keep open. ClawHub cleanup policy does not auto-close issues, and current main still has a source-reproducible path where publishing an update can hide the existing skill row during pending scan and public search/detail/API paths then filter it out.

## What This Changes

The issue reports that publishing a new version of an existing skill can return success or a server error while making the skill disappear from search, inspect, and public detail pages.

## Best Possible Solution

Implement a version-aware public projection that keeps the last approved skill card, digest, searchable embedding, API metadata, and safe file surface visible while the new version remains gated, then promotes the new projection only after scanner and moderation checks clear it.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, at source level: an ordinary non-trusted publisher can publish a new latest version, insertVersion can patch the skill row to hidden pending scan, and public detail/search/API paths then drop that row.

## Solution Assessment

Is this the best way to solve the issue? No patch is under review; the direct preserve-active-row approach is not the safest path because it can bypass pending-scan visibility gating, so the better solution is last-approved-version continuity while the new upload remains gated.
