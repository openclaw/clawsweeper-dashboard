## Summary

Keep open. ClawHub cleanup policy does not auto-close issues, and current main still has a credible source-level path where an ordinary publisher updating an active skill can move the skill to hidden pending scan while public detail, catalog, search, and API surfaces suppress hidden skills.

## What This Changes

Issue requests a fix for published skills disappearing from inspect, search, and web publishing after a version update triggers a `skills:publishVersion` server error.

## Best Possible Solution

Define and implement explicit update moderation: either keep the previous approved public listing/version visible while the new upload scans, or intentionally quarantine updates with clear owner, CLI, API, and search/status behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction path exists: publish a higher version for an existing active skill as a non-trusted ordinary publisher, observe insertVersion patch the skill to hidden pending scan, then verify public detail/search/API filters suppress it.

## Solution Assessment

Unclear. A naive preserve-active-status patch like #274 is directionally close, but the safer maintainable solution must avoid exposing the newly uploaded unreviewed latest version before scanner results are resolved.
