## Summary

Keep open. ClawHub cleanup policy does not auto-close issues, and current main still has a source-reproducible path where an ordinary update can hide an existing skill row during pending scan and public search/detail paths then filter it out.

## What This Changes

The issue reports an already-published skill disappearing from inspect/search after publishing a new version, with the web UI surfacing a `skills:publishVersion` server error.

## Best Possible Solution

Implement a version-aware public projection: keep the last approved skill card, digest, searchable embedding, API metadata, and safe file surface visible while the new version remains pending scan/review, then promote the new version only after scanner and moderation gates clear it.

## Reproduction Assessment

Yes, at source level. An ordinary non-trusted publisher updating an already-active skill can reach `insertVersion`, have the skill row patched to hidden pending scan, and then disappear from public search/detail/list paths that require active moderation status; I did not run a live Convex publish in this read-only review.

## Solution Assessment

No patch is under review. The direct preserve-active or allow-pending-through-public-filter approaches are not the safest solution because maintainers have already flagged pending-scan visibility as security-sensitive; the safer direction is last-approved-version continuity while new content remains gated.
