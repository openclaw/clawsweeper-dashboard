## Summary

Keep open. This is a ClawHub issue, and the supplied cleanup policy only permits auto-closing pull requests; current main also still has a credible update-disappearance path because an existing skill update by an ordinary non-trusted publisher can patch the skill hidden while public detail, search, and API paths suppress hidden skills.

## What This Changes

An existing published skill reportedly disappears from inspect, search, and the web UI after publishing an updated version that hits a `skills:publishVersion` server error.

## Best Possible Solution

Define and implement the update moderation model for already-active skills, preferably keeping the previous approved public listing/version visible while a new upload scans or making quarantine-on-update consistent and transparent across CLI, owner dashboard, public detail, search, API, and recovery flows.

## Reproduction Assessment

Yes. The source-level path is clear: publish an update to an existing active skill as an ordinary non-trusted publisher, let the pending-scan/quarantine path set `moderationStatus` to `hidden`, and then public detail/search/API conversion filters the skill out.

## Solution Assessment

Unclear. The narrow code path is identifiable, but the best fix needs maintainer security/product direction on whether to retain the previous approved public version during scanning or quarantine the whole listing with clearer owner and public status.
