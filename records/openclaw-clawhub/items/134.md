## Summary

Keep open. ClawHub cleanup policy does not auto-close issues, and current main still has a credible source-level path where an ordinary publisher updating an active skill can move the listing to hidden pending scan while public detail, search, catalog, and API surfaces suppress hidden skills.

## What This Changes

The issue reports that publishing an updated version of an existing visible skill can return success or a server error while making the skill disappear from inspect, search, and the web UI.

## Best Possible Solution

Define the update moderation contract and implement it consistently: keep the previous approved public listing visible while a new version scans, or intentionally quarantine updates with clear owner, CLI, API, search, and recovery behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction path exists: publish a higher version for an existing active skill as an ordinary non-trusted publisher, observe insertVersion patch the skill hidden pending scan, then verify public detail/search/API paths suppress it.

## Solution Assessment

Unclear. A small preserve-active-status patch is directionally close, but the safest maintainable fix must preserve scanner quarantine semantics and avoid exposing newly uploaded unreviewed content by accident.
