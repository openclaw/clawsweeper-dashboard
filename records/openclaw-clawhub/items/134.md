## Summary

Keep open. ClawHub cleanup policy does not auto-close issues, and current main still has a source-level path where an ordinary non-trusted publisher updating an active skill can move the skill to hidden pending scan while public detail, catalog, search, and API surfaces suppress hidden skills.

## What This Changes

The issue reports that publishing an update to an existing skill can return success or a server error while the skill immediately disappears from public search, inspect, and detail surfaces.

## Best Possible Solution

Define update moderation explicitly: keep the previous approved public listing/version visible while the new upload scans, or intentionally quarantine updates with clear owner, CLI, API, search, and recovery behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction path exists: update an existing active skill as an ordinary non-trusted publisher, observe insertVersion patch the skill to hidden pending scan, then verify public detail/search/API filters suppress that hidden state.

## Solution Assessment

Unclear. Preserving active status is directionally close, but the safest maintainable fix should avoid making the new unreviewed latest version public before scanner results are resolved.
