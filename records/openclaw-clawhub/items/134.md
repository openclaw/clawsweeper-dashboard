## Summary

Keep open. ClawHub cleanup policy does not auto-close issues, and current main still has a credible update-disappearance path: ordinary non-trusted publishers can have an existing skill update patched to hidden while public detail, search, catalog, and API surfaces suppress hidden skills.

## What This Changes

The issue reports that publishing a new version of an existing skill can return success or a server error while the previously visible skill disappears from inspect, search, listings, API, and public pages.

## Best Possible Solution

Define and implement an explicit update-moderation model. If clean active skills should stay public, keep the previous approved listing/version visible while the new upload scans and expose clear owner/API/dashboard status; if quarantine-on-update is intentional, make that state consistent across CLI, web detail/dashboard, search, API, and recovery flows.
