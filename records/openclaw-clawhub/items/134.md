## Summary

Keep open. ClawHub cleanup policy says issues should not be auto-closed, and current main still has a source-reproducible update path where publishing a new version of an existing active skill can patch the skill row to hidden pending scan while public detail/search/API surfaces filter hidden rows out.

## What This Changes

Publishing an update to an existing visible skill can make the listing disappear from search, inspect, and the web/API surfaces after `skills:publishVersion` reports success or a server error.

## Best Possible Solution

Define update continuity so the last approved public version remains visible while the new upload stays gated by scan/moderation, or intentionally quarantine the whole listing with consistent owner, public, CLI, API, and recovery messaging.

## Reproduction Assessment

Yes, at source level. An ordinary non-trusted publisher updating an existing active skill can reach `insertVersion`, have the skill row patched to hidden pending scan, and then be dropped by public detail/search/API conversion; I did not run a live Convex publish.

## Solution Assessment

Unclear because no patch is under review. The direct preserve-active-status approach in https://github.com/openclaw/clawhub/pull/274 was closed unmerged, so the safer solution is a maintainer-approved previous-version continuity policy that does not expose unreviewed updated content by accident.
