## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has a credible source-level path where an ordinary active-skill update is patched to hidden pending scan and then filtered from public search/detail/API surfaces.

## What This Changes

Publishing an update to an existing visible skill should not make the skill disappear from CLI inspect, search, web UI, or API surfaces.

## Best Possible Solution

Define and implement the active-skill update moderation policy, ideally keeping the last approved public listing visible while a new version scans or making quarantine behavior explicit and consistent across CLI, dashboard, detail, search, and API surfaces.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: an ordinary non-trusted publisher updating an existing active skill reaches insertVersion, gets hidden pending scan, and public conversion/search/detail/API paths then drop it. I did not reproduce the exact web Server Error against a live Convex deployment.

## Solution Assessment

Unclear until maintainers choose the moderation policy. Preserving the previous approved public listing while scanning the new version is the narrowest user-protecting direction, but it must be reconciled with ClawHub's security quarantine model.
