## Summary

Keep open: current main has not implemented this redesign, and the branch has a concrete static CI blocker from deleting the separator component while leaving its Radix dependency installed.

## What This Changes

The PR rewrites Settings into query-driven focused account, organization, API token, and deletion views, moves Stars to the signed-in avatar menu, adjusts Select styling, removes the Separator primitive, and updates tests/changelog.

## Best Possible Solution

Land the redesign after the branch removes the now-unused separator dependency/lock entry, keeps static CI green, and gates org-member reads to the Organizations view.

## Reproduction Assessment

Yes for the PR blocker: GitHub CI on the PR merge ref failed `deadcode:dependencies` with the unused separator dependency, and the source diff explains why. This is a PR-head regression, not a current-main bug reproduction.

## Solution Assessment

No: the redesign direction appears reasonable and has member approval, but deleting the separator component without removing the package dependency leaves the branch unable to pass static CI. Gating the org members query is the narrower maintainability fix for the focused-view model.
