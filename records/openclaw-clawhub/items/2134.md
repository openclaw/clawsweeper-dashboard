## Summary

This PR is useful but should stay open because the redesigned org member list regresses an existing owner-management capability: org owners can no longer remove owner-role members from the UI even though the backend supports that with permission and last-owner guards.

## What This Changes

This PR redesigns Settings into query-driven focused views, moves Stars into the avatar menu, updates related styles/tests, and adds a changelog entry.

## Best Possible Solution

Land the redesign after preserving org-owner membership management and adding focused Settings coverage for the organizations view.

## Reproduction Assessment

Yes for the review finding from source inspection: the PR head hides Remove for owner-role members while current backend code allows org owners to remove another owner unless it would remove the last owner. I did not run the PR branch live.

## Solution Assessment

No: hiding all owner removals is broader than the existing permission model. The narrower fix is to keep the action available to org owners and let the existing mutation enforce admin and last-owner guards.
