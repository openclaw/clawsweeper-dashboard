## Summary

Keep open for normal PR review: this fresh settings redesign is not already implemented on current main, and the branch has screenshot proof plus maintainer approval, but source review found a narrow org member-management regression to fix before merge.

## What This Changes

The PR redesigns Settings into query-driven account, organization, API token, and account deletion views, moves Stars into the signed-in header menu, adjusts select styling, removes an unused separator component, and updates tests and the changelog.

## Best Possible Solution

Land the redesign after preserving existing org-owner member removal capability and keeping the server-side last-owner and permission checks as the source of truth.

## Reproduction Assessment

Yes, by source comparison: current main renders member removal for all member rows when the viewer can manage the org, while the PR hides the action for owner rows. That makes the owner-removal regression reproducible from the branch code without needing a live backend setup.

## Solution Assessment

No, not as-is: hiding every owner-row Remove action is broader than the backend policy and removes an existing owner workflow. The narrower fix is to show owner-row removal to org owners and let the existing mutation enforce non-owner and last-owner restrictions.
