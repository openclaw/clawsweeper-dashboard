The PR limits skill/package moderation appeals to malicious outcomes, adds review metadata and cleanup/export tooling, adds a local dev persona switcher, and updates several select controls.

## Best Possible Solution

Land a revised PR that keeps the approved malicious-only appeal policy coherent across API, CLI, cleanup, specs, and tests while hardening or removing shared-deployment dev auth and preserving repo-local skill exceptions.

## Reproduction Assessment

Yes, for the PR-introduced blockers: source inspection of the latest PR head shows the admin-capable dev auth provider and the malicious appeal auto-rejection paths. No live Convex deployment was needed to identify those code paths.

## Solution Assessment

No. The malicious-only appeal direction may be acceptable after maintainer approval, but this patch needs security hardening, reconciliation fixes, durable spec updates, restored ignore exceptions, and real behavior proof before it is the maintainable path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Gate dev persona auth to local-only deployments:** `convex/auth.ts:95-104`
  - body: This provider runs on the Convex backend whenever `DEV_AUTH_ENABLED=1` and the deployment string is not production-like. Because the allowed personas include `admin`, any shared preview/staging/dev backend with that env var can mint an admin session; the frontend localhost-only FAB check does not protect this credentials endpoint.
  - confidence: 0.9
- **[P2] Leave malicious skill appeals open during reconciliation:** `convex/skills.ts:3358-3361`
  - body: The PR goal says malicious appeals stay open, but this helper rejects matching open appeals whenever the effective status is malicious. Rescans and moderator state changes call this path, so a malicious appeal can be auto-closed instead of remaining for moderator review.
  - confidence: 0.9
- **[P2] Leave malicious package appeals open during reconciliation:** `convex/packages.ts:2533-2536`
  - body: Package reconciliation has the same malicious-to-rejected mapping. Bulk cleanup skips malicious appeals, but rescan and moderator-state paths still call this helper and can close package appeals contrary to the malicious-only appeal policy.
  - confidence: 0.88
- **[P2] Record the appeal policy in the moderation spec:** `convex/skills.ts:3484-3489`
  - body: This changes the security-sensitive appeal contract from broader suspicious/scanner-flagged outcomes to malicious-only, while the durable moderation spec on current main still records the old behavior. Update the spec in the same PR so future moderation work preserves the intended policy.
  - confidence: 0.86
- **[P3] Preserve repo-local skill ignore exceptions:** `.gitignore:38`
  - body: Removing these exceptions leaves `*/skills/*` in place, so new repo-local developer skills under `.agents/skills/**` will be ignored by default. That conflicts with the repository policy allowing Convex and blacksmith developer skills there while keeping top-level `skills/` reserved.
  - confidence: 0.82
