## Summary

This PR is not implemented on current main and should stay open, but the latest head still has security and moderation-policy blockers plus insufficient real behavior proof.

## What This Changes

The PR changes skill/package moderation so only malicious outcomes can be appealed, adds cleanup/export tooling and review metadata, adds a local dev persona auth switcher, and refactors related artifact/security UI.

## Best Possible Solution

Land a revised PR that consistently enforces malicious-only appeals across API, CLI, cleanup, tests, and specs, keeps malicious appeals open for review, and gates dev persona auth to a truly local backend.

## Reproduction Assessment

Yes, for the PR-introduced blockers: source inspection of the latest PR head shows the admin-capable dev auth provider and the malicious appeal auto-rejection paths. No live Convex deployment was needed to identify those code paths.

## Solution Assessment

No. The malicious-only appeal direction may be acceptable after maintainer approval, but this patch still needs auth hardening, appeal reconciliation fixes, durable spec updates, restored repo-local skill ignore exceptions, and real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Gate dev persona auth to local-only deployments:** `convex/auth.ts:18-22`
  - body: This provider runs whenever DEV_AUTH_ENABLED=1 and the deployment string is not production-like. Because the allowed personas include admin, any shared preview/staging/dev backend with that env var can mint an admin session; the frontend localhost-only FAB check does not protect this credentials endpoint.
  - confidence: 0.9
- **[P2] Leave malicious skill appeals open during reconciliation:** `convex/skills.ts:3374-3377`
  - body: The PR goal says malicious appeals stay open, but this helper rejects matching open appeals whenever the effective status is malicious. Rescans and moderator state changes call this path, so a malicious appeal can be auto-closed instead of remaining for moderator review.
  - confidence: 0.9
- **[P2] Leave malicious package appeals open during reconciliation:** `convex/packages.ts:2562-2565`
  - body: Package reconciliation has the same malicious-to-rejected mapping. Bulk cleanup skips malicious appeals, but rescan and moderator-state paths still call this helper and can close package appeals contrary to the malicious-only appeal policy.
  - confidence: 0.88
- **[P2] Record the appeal policy in the moderation spec:** `convex/skills.ts:3500-3504`
  - body: This changes the security-sensitive appeal contract to malicious-only, while the durable moderation spec on the PR head still records the old broader skill appeal behavior. Update the spec in the same PR so future moderation work preserves the intended policy.
  - confidence: 0.86
- **[P3] Preserve repo-local skill ignore exceptions:** `.gitignore:38-39`
  - body: Removing these exceptions leaves */skills/* in place, so new repo-local developer skills under .agents/skills are ignored by default. That conflicts with the repository policy allowing Convex and blacksmith developer skills there while keeping top-level skills/ reserved.
  - confidence: 0.82
