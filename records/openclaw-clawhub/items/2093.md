## What This Changes

The PR changes ClawHub moderation so suspicious/Review artifacts are not appealable, adds cleanup/export tooling and review verdict fields, updates moderator CLI/API/schema paths, and adjusts related tests/copy.

## Best Possible Solution

Revise the PR so automatic cleanup leaves malicious appeals open, restores repo-local skill ignore rules, and includes redacted real behavior proof for the moderation, cleanup, and export paths.

Full review comments:

- **[P2] Leave malicious skill appeals open:** `convex/skills.ts:3359-3361`
  - body: `closeOpenSkillCasesForVersion` is used by rescan and moderation reconciliation, but this branch rejects every open appeal when the effective status is malicious. That contradicts the PR's malicious-only appeal policy: a stale Review appeal that rescans to malicious would be closed automatically instead of staying open for moderator review.
  - confidence: 0.88
- **[P2] Leave malicious package appeals open:** `convex/packages.ts:2533-2536`
  - body: The package helper has the same issue: any open appeal for a release that reconciles to `malicious` is patched to `rejected`. Since malicious releases are the only appealable state after this change, these appeals should remain open rather than being auto-closed by cleanup/reconciliation.
  - confidence: 0.88
- **[P3] Restore repo-local skill exceptions:** `.gitignore:38`
  - body: Removing the `.agents/skills` unignore rules means new Convex or blacksmith developer skill files will be hidden by the existing `*/skills/*` ignore pattern. The repository policy allows these repo-local skills, so future updates would be easy to miss unless forced.
  - confidence: 0.9
