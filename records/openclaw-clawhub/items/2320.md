## Summary

This RFC is still necessary: current main implements secure publisher ownership and self-deleted-account cleanup, but it does not establish the requested public rules for namespace claims, disputes, transfers, deleted-owner handling, or confidential evidence review. The two linked namespace disputes remain open, and the item is collaborator-authored with protected security and maintainer-review labels, so it requires an explicit product and security decision.

## What This Changes

Define a transparent policy for organization or owner namespace claims and disputes, plus recovery after owner-initiated account deletion.

## Best Possible Solution

Adopt a public claim, dispute, transfer, and recovery policy; preserve sensitive verification and escalation invariants in internal specs or reviewer runbooks; then create bounded implementation follow-ups only for admin operations the accepted policy requires.
