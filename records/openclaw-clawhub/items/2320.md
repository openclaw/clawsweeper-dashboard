## Summary

Keep open: current main provides account reclaim, publisher management, handle reservation, and ownership-transfer mechanics, but it does not settle the RFC’s policy questions about claim evidence, dispute adjudication, deleted-owner namespaces, appeals, or public-versus-confidential evidence. The collaborator authorship and protected security/product-review signals also require explicit maintainer judgment.

## What This Changes

Define transparent policy for organization namespace claims, disputes, deleted-owner namespaces, and recovery after owner-initiated account deletion.

## Best Possible Solution

Adopt a policy separating public eligibility, notice, outcomes, and appeal rules from confidential verification procedures; preserve security invariants in `specs/`, publish stable user-facing behavior in `docs/`, and create bounded implementation follow-ups only after those decisions.
