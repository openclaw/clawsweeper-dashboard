## Summary

Keep open. Current main exposes partial structured scanner evidence, but it still lacks the requested line ranges, remediation guidance, explicit blocking/informational semantics, and local CLI scan contract; for ClawHub cleanup, issues also stay open unless maintainer action closes them.

## What This Changes

The issue asks ClawHub to expose actionable scanner findings for flagged skills, including rule/category, severity, file and line range, redacted snippet, remediation guidance, blocking semantics, and optionally `clawhub scan <path> --format json`.

## Best Possible Solution

Define a stable scanner-finding contract with rule/category, severity, file plus line range, redacted evidence policy, remediation guidance, and blocking semantics, then wire it consistently through persistence, API responses, skill UI, CLI publish output or a local scan command, docs, and tests.
