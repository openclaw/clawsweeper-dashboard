## What This Changes

Publishing failed on first publish because the owner-publisher skill digest sync path could run multiple Convex paginated queries in one mutation.

## Best Possible Solution

Keep the shipped cursor-continuation implementation, verify a production or staging first-publish smoke test, then consolidate this report with the related publish-pagination issues.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: publish enters insertVersion, first-publish setup can write a publisher, and that publisher trigger schedules owner-publisher digest sync. Current tests cover the regression shape by forcing an unfinished page and asserting only one paginate call per mutation.

## Solution Assessment

Yes. The current main implementation is the narrow maintainable fix because it keeps indexed pagination but schedules each continuation cursor as a separate internal mutation, avoiding Convex's single-pagination limit without switching to unbounded collection.
