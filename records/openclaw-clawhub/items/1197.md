## Repair Work Prompt

Fix the Convex publish/import pagination failure reported in this issue cluster. The observable bug is that first publish or import+publish can create/update a personal publisher, which schedules owner-publisher digest rebuilds; the package and skill rebuild helpers then loop over `.paginate()` inside one internal mutation and can trip Convex's single-paginated-query limit. Implement cursor-rescheduled owner-publisher digest rebuilds in `convex/functions.ts`: each scheduled/internal mutation should process at most one indexed page, accept/pass a continuation cursor when needed, and schedule the next page with `ctx.scheduler.runAfter(0, ...)`. Cover both `syncPackageSearchDigestsForOwnerPublisherId` and `syncSkillSearchDigestsForOwnerPublisherId`; do not replace the loops with unbounded `.collect()` or remove indexed `by_owner_publisher` queries. Add focused regression tests in `convex/functions.test.ts` for multi-page package and skill owners, including scheduler continuation args. Related refs: #1195, #1171, #1184, #1175, #1176, #1182, #1201, #1205. Do not change the CLI publish interface or unrelated search/digest behavior. Changelog: include a concise bugfix note only if the repo has an unreleased changelog path for backend fixes.
