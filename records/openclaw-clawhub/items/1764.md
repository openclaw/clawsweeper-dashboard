## Summary

Keep open. This is an open ClawHub issue, and current production GitHub sign-in is still unhealthy: the site and public API respond, but the canonical and direct Convex GitHub sign-in routes return HTTP 500. Current main routes that path through Convex Auth with GitHub credentials and issuer values supplied by deployment env, so the next repair needs maintainer/operator access rather than cleanup closure.

## What This Changes

The issue asks why an earlier GitHub auth login issue was closed and reports that some users still cannot sign in to ClawHub normally with GitHub.

## Best Possible Solution

Restore production GitHub OAuth end to end, verify active-account web and CLI login, keep deleted/banned-account outcomes explicit, and add focused regression or smoke coverage once the failing layer is identified.

## Reproduction Assessment

Yes. On 2026-05-09 around 14:25 UTC, the site root and public API responded, but both the canonical ClawHub GitHub sign-in route and direct Convex GitHub sign-in route returned HTTP 500.

## Solution Assessment

Unclear as a source-only fix. The narrowest maintainable path is maintainer/operator triage of production OAuth credentials, Convex/Vercel env, callback URLs, cookies, and logs, with any code diagnostic following the import-safe direction from https://github.com/openclaw/clawhub/pull/1720.
