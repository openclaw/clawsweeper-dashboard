Full review comments:

- **[P1] Keep synthetic inspector config out of publish source:** `.github/workflows/package-publish.yml:483-486`
  - body: This writes `.plugin-inspector.json` into `inspect_root`. For callers using local `source` values like `.` or `./plugin`, the next step publishes that same directory: `cmdPublishPackage` reads `listPackageFiles(folder)`, and code plugins run `npm pack` from `folder`. The generated inspector config can therefore be included in the uploaded bundle or ClawPack and change the published artifact. Generate it in a temporary copy, or remove/exclude it before `Run package publish`.
