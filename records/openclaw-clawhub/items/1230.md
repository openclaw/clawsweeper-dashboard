## Summary

Keep open. Current main does not implement the PR’s intended change: the header still uses only the root ToggleGroup value-change path, the runtime legacy `clawdhub-theme` fallback remains, and the pre-hydration root script still reads that same legacy key. The paired bug is still open, and the alternative PR is closed unmerged, so this branch is not superseded by a landed fix.

## What This Changes

The PR updates the header theme ToggleGroup wiring, runtime theme storage fallback, and header/theme regression tests to restore system/light/dark selection by pointer and keyboard while ignoring the legacy `clawdhub-theme` key.

## Best Possible Solution

Rebase or replace the branch so the header theme selector works consistently for segmented click, Radix keyboard/value-change, the cycle button, runtime storage, and pre-hydration first paint, with an explicit legacy-key policy and regression coverage for each supported path.
