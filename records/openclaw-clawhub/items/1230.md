## Summary

Keep open: current main does not definitely implement this PR's intended change, and the linked bug #1228 is still open. The PR also still needs maintainer handling because its legacy-storage removal is incomplete unless the pre-hydration script is updated with the same policy.

## What This Changes

The PR changes the header theme ToggleGroup, runtime theme fallback, and regression tests to restore system/light/dark selection by pointer and keyboard while ignoring the legacy `clawdhub-theme` key.

## Best Possible Solution

The final implementation should restore header theme selection for pointer, keyboard, and cycle-button paths while keeping runtime loading, pre-hydration first paint, and tests aligned with the chosen legacy-key policy.

## Reproduction Assessment

Yes. With the PR's runtime-only legacy-key removal, setting only `localStorage.clawdhub-theme = 'dark'` exercises the mismatch because the root script still reads that key before hydration while the patched runtime would default to `system`.

## Solution Assessment

No as currently proposed. Keeping both pointer and value-change paths can be a maintainable accessibility-preserving shape, but the storage policy must be applied to both `src/lib/theme.ts` and `src/routes/__root.tsx` before this is the best fix.
