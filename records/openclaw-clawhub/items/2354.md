## Summary

Keep this PR open: the canonical ClawScan field migration is useful, but the current patch is not merge-ready because it can unblock legacy malicious/static findings before canonical backfill or ClawScan completion, and the contributor has not provided real behavior proof for the security/upgrade path.

## What This Changes

The branch adds canonical ClawScan verdict/state fields for skill versions and package releases, rewires scanner, moderation, UI, API, and CLI paths to use clean/review/warn/malicious, and adds backfill plus legacy cleanup helpers.

## Best Possible Solution

Land the canonical verdict/state widening only after legacy malicious/static blocks remain fail-closed through backfill and maintainers explicitly approve any later policy narrowing with upgrade proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: a package release with staticScan.status or verification.scanStatus set to malicious and no canonical ClawScan verdict is blocked on current main but falls through to pending/not-run in the PR. I did not run tests because this review is read-only.

## Solution Assessment

No, not as currently written: canonical fields are a maintainable direction, but the widening PR should preserve fail-closed legacy malicious behavior until backfills and cleanup have been verified. A safer path is schema widening plus dual-read/dual-write first, then a separately approved policy removal.
