## Summary

Keep open. Current main has VT pending-scan recovery, stale fallback, scan detail output, and owner rescan plumbing, but this is a ClawHub issue and the cleanup policy only permits auto-closing PRs; the requested owner-facing queue position, worker heartbeat, retry timing, and per-item transition visibility are still not present.

## What This Changes

The issue reports a ClawHub skill remaining stuck in pending VirusTotal state even after the linked VirusTotal result appears to have passed.

## Best Possible Solution

Provide an authenticated owner/admin scan status surface for skills and packages that exposes safe per-item scanner state, last checked time, check count, stale reason, latest or in-progress rescan request, and a conservative next retry estimate while keeping raw worker and global queue internals private.
