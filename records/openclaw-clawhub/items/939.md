## Summary

Keep open. This is an open ClawHub issue, and the repository policy for this review lane only permits auto-closing pull requests that current main definitely implements. Current main has meaningful VirusTotal pending-scan recovery, stale fallback, public scan-detail data, and owner rescan plumbing, but it still does not expose the requested owner-facing queue position, worker heartbeat, retry timing, or per-item transition visibility, and source review cannot verify the reporter's specific production row.

## What This Changes

The issue reports a skill remaining in pending security-scan state even after VirusTotal shows a passing result, with follow-up interest in queue/status visibility for stalled scans.

## Best Possible Solution

Provide an authenticated owner/admin scan status surface for skills and packages that exposes safe per-item scanner state, last checked time, check count, stale reason, latest or in-progress rescan request, and a conservative next retry estimate, while keeping raw worker health and global queue internals private unless maintainers explicitly approve them.
