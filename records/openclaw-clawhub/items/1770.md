Keep open. This is still a live ClawHub production slug-release request: current main has the source path that blocks a rename when any different `skills.by_slug` row owns the target slug, and the remaining work requires production data inspection plus account/org ownership verification.

## What This Changes

The requester asks maintainers to purge or release the soft-deleted `creator-scraper-cv` slug so the org-owned `cv-creator-scraper` skill can be renamed.

## Best Possible Solution

A maintainer should verify the requester controls both accounts, inspect the production slug/alias/reservation state, then use the least destructive existing admin path to release or reclaim the slug and confirm the org rename succeeds.

## Reproduction Assessment

Yes for the general source path: current main rejects an owner rename when a different `skills.by_slug` row already has the requested slug, including soft-deleted rows. The exact production `creator-scraper-cv` record was not verified from read-only repository inspection.
