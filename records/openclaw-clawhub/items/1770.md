## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still blocks owner rename on any different `skills.by_slug` collision, including soft-deleted rows; resolving this requires ownership-sensitive production admin work rather than a source-only cleanup.

## What This Changes

The requester asks maintainers to purge or release the soft-deleted personal-account slug `creator-scraper-cv` so an org-owned skill can be renamed to that slug.

## Best Possible Solution

A maintainer should verify both identities, inspect production slug/alias/reservation state, use the least destructive admin reclaim or hard-delete path that actually frees the slug, then confirm the org-owned rename succeeds.

## Reproduction Assessment

Yes for the source-level blocker: a different row returned by `skills.by_slug(newSlug)` causes owner rename to throw before alias or reservation handling, and tests already treat soft-deleted slug matches as taken. The user-specific production row still needs admin inspection to confirm the exact stored state.

## Solution Assessment

The requested hard purge may be a valid support action, but it should not be done blindly. The safest solution is a verified maintainer/admin operation that accounts for soft-deleted rows, aliases, reservations, and org ownership before confirming the rename.
