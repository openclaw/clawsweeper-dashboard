Keep open. This is an issue, and the ClawHub cleanup policy for this pass keeps issues open; current evidence also still points to an unresolved production login/auth problem, with GitHub sign-in endpoints returning HTTP 500 while the CLI auth route can disable sign-in while Convex auth is loading.

## What This Changes

The issue reports that `npx clawhub login` opens `/cli/auth` with a disabled GitHub sign-in button because the production Convex WebSocket connection fails.

## Best Possible Solution

Restore and verify production GitHub OAuth and Convex auth configuration, then smoke-test `npx clawhub login` through `clawhub whoami`; add a focused `/cli/auth` loading/error fallback only if the route can still strand users after production config is healthy.

No, not for the exact WebSocket-307 disabled-button report: the reported sync endpoint upgraded with HTTP 101 in this review. There is a high-confidence related production auth failure, because both Vercel and direct Convex GitHub sign-in endpoints returned HTTP 500.

## Solution Assessment

Unclear. The narrowest immediate path is operator-side repair of production OAuth/Convex settings, with a source fallback and regression coverage only if `/cli/auth` remains able to strand users once the backend configuration is healthy.
