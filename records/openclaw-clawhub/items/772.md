## Summary

Keep open. The ClawHub cleanup policy for this pass keeps issues open, and current main still contains the reported disabled-button path tied to Convex auth loading, so there is no implemented-on-main basis to close this report.

## What This Changes

The issue reports that `npx clawhub login` opens `/cli/auth` with a disabled GitHub sign-in button because the production Convex WebSocket fails to connect with HTTP 307.

## Best Possible Solution

Production CLI login should reliably enable GitHub sign-in after auth resolution and complete token handoff through the loopback callback; if Convex auth/connectivity fails, `/cli/auth` should present a recoverable error or fallback instead of leaving the sign-in action permanently disabled, with focused regression coverage.
