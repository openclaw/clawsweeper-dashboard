## Summary

Keep this PR open for review: the temporal abuse scan is a useful backend moderation feature, but the current persistence path can write an unbounded candidate batch in one Convex mutation and should be batched before merge.

## What This Changes

Adds a nightly and manual temporal publisher abuse scan over recent skill daily stats, stores temporal evidence on publisher abuse scores, and reuses the publisher abuse review nomination queue.

## Best Possible Solution

Keep the temporal scan, but persist scores and nominations in bounded pages using the existing action/query/mutation continuation pattern and add a focused spec note for the new moderation signal.

## Reproduction Assessment

Yes, by source inspection: collect many flagged temporal candidates and the action passes the whole candidate array into one persistence mutation, which then writes per aggregate. I did not execute a Convex load reproduction in this read-only review.

## Solution Assessment

No, not yet: the scoring direction fits the existing review queue, but the persistence path should use bounded batches like the existing publisher abuse score runner before this is the maintainable implementation.
