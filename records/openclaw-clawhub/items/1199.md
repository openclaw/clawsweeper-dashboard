Keep this issue open: ClawHub cleanup policy does not auto-close issues, and current main still has a credible source-backed failure path where Settings save awaits `users.updateProfile`, which can roll back when personal-publisher synchronization throws a handle-conflict error.

## What This Changes

Fix the Settings profile save flow so display name and bio updates persist instead of failing with a Convex server error.

## Best Possible Solution

Persist personal display name and bio updates even when personal-publisher synchronization hits a recoverable handle conflict, while preserving hard failures for authorization and publisher ownership invariants.

## Reproduction Assessment

No live current-main reproduction was run. Source inspection gives a clear targeted path: a user whose derived personal-publisher handle conflicts with another publisher can hit the throw after `users.updateProfile` patches the user in the same mutation.

## Solution Assessment

Yes. The narrow maintainable direction is to decouple profile field persistence from recoverable personal-publisher sync conflicts, not to swallow every publisher-sync error or bypass ownership checks.
