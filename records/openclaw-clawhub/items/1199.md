Fix the Settings personal profile save server error reported at https://github.com/openclaw/clawhub/issues/1199. Current main awaits api.users.updateProfile from src/routes/settings.tsx; convex/users.ts patches displayName/bio and then calls ensurePersonalPublisherForUser; convex/lib/publishers.ts can throw a publisher-handle conflict, rolling back the profile save. Preserve auth, publisher ownership, and handle uniqueness invariants; do not broadly swallow all publisher sync errors; do not change org publisher update semantics.
