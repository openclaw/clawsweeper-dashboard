## Summary

Keep this issue open. ClawHub cleanup policy keeps issues open, and current main still has a source-backed Settings save failure path where `users.updateProfile` can roll back display name and bio changes when personal-publisher synchronization throws a handle-conflict error.

## What This Changes

The issue reports that saving Settings display name or bio fails from the Settings page with a Convex `users:updateProfile` server error.

## Best Possible Solution

Persist personal `displayName` and `bio` updates even when personal-publisher synchronization hits a recoverable handle conflict, while preserving hard failures for auth, ownership, and invariant violations.

## Reproduction Assessment

No, not as a live current-main reproduction. Source inspection gives a clear path where the Settings form awaits `users.updateProfile`, and that mutation can throw from `ensurePersonalPublisherForUser` after patching the user, rolling back the save.

## Solution Assessment

Yes. The narrow maintainable direction is to decouple profile-field persistence from recoverable personal-publisher handle conflicts for this save path, not to swallow all publisher-sync errors or weaken ownership checks.
