## Summary

Keep open. ClawHub cleanup policy does not auto-close issues, and current main still has a source-backed failure path where Settings save awaits `users.updateProfile`, which can throw from personal-publisher synchronization and roll back the profile edit.

## What This Changes

The issue requests that changing display name or bio on the Settings page saves successfully instead of failing with a Convex server error.

## Best Possible Solution

Persist personal display name and bio updates even when personal-publisher synchronization hits a recoverable handle conflict, while preserving hard failures for auth, ownership, and publisher integrity invariants.

## Reproduction Assessment

Yes, source-level. A fixture where the user's derived personal publisher handle is claimed by another active publisher will make `users.updateProfile` reach `ensurePersonalPublisherForUser`, throw, and roll back the Settings save; no live production mutation was run in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to decouple personal profile field persistence from recoverable personal-publisher sync conflicts, not to hide the error in the UI or swallow every publisher-sync exception.
