## Summary

Keep open. ClawHub cleanup policy keeps issues open, and the latest maintainer production recheck plus current source show this is a moderation-state follow-up rather than a stale search-index bug: global search intentionally filters suspicious skills, and both named records still remained suspicious after fresh rescans.

## What This Changes

The issue asks maintainers to explain or fix why two published skills remain absent from global web search after the reporter believed moderation had cleared.

## Best Possible Solution

Keep the global-search suspicious filter in place and complete staff moderation review; if the records are false positives, clear or override them through the existing moderation/rescan path.

## Reproduction Assessment

Yes for the observed visibility split: current source shows `/search` uses `nonSuspiciousOnly: true`, `/skills` defaults it to false, and a maintainer production recheck says both slugs remain suspicious after fresh rescans. I did not establish a source-level bug because the behavior matches the configured moderation filter.

## Solution Assessment

No for a search-index refresh as the primary fix; the narrow maintainable path is staff moderation review or an explicit visibility-policy change. Bypassing the global search filter in source would be unsafe while the production records remain suspicious.
