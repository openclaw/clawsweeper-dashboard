## Summary

Keep open. ClawHub policy keeps issues open, and the evidence points to production moderation follow-up rather than a source search-index bug: global web search deliberately requests non-suspicious skill results, while browse and CLI/API paths can query without that page-level filter; a maintainer production recheck says both named skills were still suspicious after fresh rescans.

## What This Changes

The issue asks maintainers to restore or explain web search discoverability for `skillpick` and `threeperson` after the reporter believed moderation had cleared.

## Best Possible Solution

Preserve global suspicious filtering, have staff complete moderation review or an audited manual override if these are false positives, and open a narrower diagnostics issue only if owner-facing moderation output disagrees with staff state.

## Reproduction Assessment

Yes for the visibility split from source inspection: global `/search` passes `nonSuspiciousOnly: true`, while `/skills` defaults that filter off. The cleared-moderation premise is not reproducible from the checkout, and the maintainer production recheck says the records were still suspicious.

## Solution Assessment

No to a search-index refresh or loosening global search filtering as the primary fix. The maintainable path is staff moderation review or manual override while keeping suspicious-skill filtering intact.
