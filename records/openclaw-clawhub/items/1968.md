## Summary

Keep open. Current main shows the reported web-search split is intentional suspicious-skill filtering, and the maintainer production recheck says the named skills were still marked suspicious, so this needs moderation follow-up rather than an index-refresh code change.

## What This Changes

The issue asks why `skillpick` and `threeperson` remain absent from global web search after the reporter believed moderation had cleared.

## Best Possible Solution

Preserve global-search suspicious filtering, have staff review or override the production moderation records if they are false positives, and open a narrower diagnostics issue only if owner-facing moderation output disagrees with staff state.

## Reproduction Assessment

No for the cleared-moderation premise from source alone; the provided maintainer production recheck says the records were still suspicious.

## Solution Assessment

No to a search-index refresh or removing the global-search filter as the primary fix. The maintainable path is staff moderation review or diagnostics reconciliation while keeping suspicious-skill filtering intact.
