## Summary

Keep open. ClawHub cleanup policy keeps issues open, and the evidence points to production moderation follow-up rather than a source search-index bug: global web search intentionally hides suspicious skills, while the skills browse and CLI/API paths can query without that filter; a maintainer production recheck reported the named skills were still suspicious.

## What This Changes

The issue asks why the published `skillpick` and `threeperson` skills remain absent from web search after the reporter believed moderation had cleared.

## Best Possible Solution

Preserve global-search suspicious filtering, have staff review or manually override the production records if they are false positives, and open a narrower source issue only if owner diagnostics disagree with the actual moderation state.

## Reproduction Assessment

Yes for the visibility split: current source shows global `/search` passes `nonSuspiciousOnly: true`, backend search enforces it, and `/skills` plus CLI search can query without it. The reporter’s cleared-moderation premise is not reproducible locally, but the maintainer production recheck says the records were still suspicious.

## Solution Assessment

No to a search-index refresh or removing the global-search filter as the primary fix. The narrow maintainable path is staff moderation review or an audited manual override while preserving suspicious-skill filtering.
