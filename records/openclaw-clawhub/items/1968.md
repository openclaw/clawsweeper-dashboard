## Summary

Keep open. ClawHub cleanup policy keeps issues open, and the current evidence points to maintainer moderation follow-up rather than a source search-index bug: global web search intentionally hides suspicious skills, while skills browse and CLI/API search can query without that global-page filter; a maintainer production recheck says both named skills were still flagged after fresh rescans.

## What This Changes

The issue asks maintainers to investigate why two published skills remain hidden from global web search despite the reporter believing moderation had cleared.

## Best Possible Solution

Preserve global-search suspicious filtering, have staff review or override the production moderation records if they are false positives, and open a narrower diagnostics issue only if owner-facing API/CLI output disagrees with the actual moderation state.

## Reproduction Assessment

Yes for the visibility split: current source shows global `/search` passes `nonSuspiciousOnly: true`, backend search enforces it, and `/skills` plus CLI/API search can query without that filter. The cleared-moderation premise is not reproducible from the checkout and is contradicted by the maintainer production recheck.

## Solution Assessment

No to a search-index refresh or removing the global-search filter as the primary fix. The narrow maintainable path is staff moderation review or audited manual override while keeping suspicious-skill filtering intact.
