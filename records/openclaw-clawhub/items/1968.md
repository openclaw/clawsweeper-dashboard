## Summary

Keep open. Current main and production still explain the reported split: global web search forces the non-suspicious filter, while the skills browse path leaves it optional, and both named production skills currently report suspicious moderation rather than cleared moderation.

## What This Changes

The reporter requests investigation of why `skillpick` and `threeperson` remain absent from global web search despite expected moderation clearance.

## Best Possible Solution

Maintainers should decide whether the named skills are false positives to clear with the existing moderation override/rescan tools, or whether global `/search` should expose the same suspicious-filter behavior as `/skills` with matching tests and docs.

## Reproduction Assessment

Yes. On 2026-05-02, the unfiltered public search API returned both named slugs, adding `nonSuspiciousOnly=true` returned no results, and their detail/moderation endpoints reported `isSuspicious: true`.

## Solution Assessment

Unclear. The current code path is internally consistent with hiding suspicious skills in global search, but choosing between moderation override, rescan, or UI/API behavior change is a maintainer product and trust decision.
