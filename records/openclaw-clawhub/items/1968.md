## Summary

Keep open. ClawHub cleanup policy keeps issues open, and the current evidence points to production moderation follow-up rather than a source search-index bug: global web search intentionally hides suspicious skills, while skills browse and CLI/API search can query without that global-page filter; a maintainer production recheck reported both named skills were still flagged after fresh rescans.

## What This Changes

The issue asks staff to investigate why the published skills `skillpick` and `threeperson` are hidden from global web search despite owner-visible moderation appearing cleared.

## Best Possible Solution

Preserve global-search suspicious filtering, have staff review or override the production moderation records if they are false positives, and open a narrower diagnostics issue only if owner-facing API/CLI output disagrees with the actual moderation state.

## Reproduction Assessment

Yes for the visibility split: current source shows global `/search` passes `nonSuspiciousOnly: true`, and the backend excludes suspicious results. No for the cleared-moderation premise in the local checkout; the maintainer production recheck reported both named skills still suspicious.

## Solution Assessment

No, a search-index refresh or removing the global-search filter is not the best primary fix while production still marks the skills suspicious. The narrow maintainable path is staff moderation review or audited manual override while keeping suspicious-skill filtering intact.
