Keep open. Current main intentionally lets global search hide suspicious skills while the skills browse path can include them, and production currently marks both named skills as `suspicious.llm_suspicious`; this needs maintainer moderation/product review rather than cleanup closure.

The issue asks ClawHub to investigate why `skillpick` and `threeperson` do not appear in global web search after the reporter believed moderation had cleared.

## Best Possible Solution

Maintainers should either clear confirmed false positives through the existing rescan/manual-override path, or deliberately change global search filtering policy with matching tests and documentation.

## Reproduction Assessment

Yes. Production reproduces the split: both named skills appear in public search without `nonSuspiciousOnly`, disappear with `nonSuspiciousOnly=true`, and their moderation endpoints report `suspicious.llm_suspicious`.

## Solution Assessment

Unclear. The current implementation is internally consistent with hiding suspicious skills in global search, so the right fix depends on maintainer moderation and product-policy judgment rather than a source-only search-index refresh.
