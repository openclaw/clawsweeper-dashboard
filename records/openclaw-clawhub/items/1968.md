## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main plus the maintainer production recheck point to a moderation-state follow-up rather than a stale search-index bug: global search intentionally filters suspicious skills, while skills browse does not by default.

## What This Changes

The issue asks maintainers to investigate why two published skills remain absent from global web search after the reporter believed moderation had cleared.

## Best Possible Solution

Keep the global-search suspicious filter in place and have staff complete moderation review or an explicit override for the production records; clarify owner-facing diagnostics only if they still make suspicious records look cleared.

## Reproduction Assessment

Yes for the observed visibility split: current source shows `/search` uses `nonSuspiciousOnly: true`, `/skills` defaults that filter off, and a maintainer production recheck reported both named skills still suspicious after fresh rescans. I did not reproduce a source-level bug because the behavior matches the configured moderation boundary.

## Solution Assessment

No for a search-index refresh as the primary fix; the narrow maintainable path is staff moderation review or override of the production records. Changing source search filters would be the wrong fix while the records remain suspicious.
