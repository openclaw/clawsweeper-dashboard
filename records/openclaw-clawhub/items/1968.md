## Summary

Keep open. ClawHub cleanup policy keeps issues open, and this report still has a live, reproducible moderation/search split: global search uses the suspicious-content filter while the skills directory can show those results when the optional filter is off.

## What This Changes

The reporter asks maintainers to investigate why `skillpick` and `threeperson` do not appear in global web search after moderation appeared cleared, even though CLI/API lookup still works.

## Best Possible Solution

Maintainers should review the production moderation records, then either clear the suspicious verdicts or intentionally align global search with the skills-directory filter behavior and update tests/docs for that chosen policy.

## Reproduction Assessment

Yes. Comparing production `/api/v1/search` with and without `nonSuspiciousOnly=true` for both slugs reproduces the absence from filtered/global search, and current main shows global `/search` uses that filtered mode.

## Solution Assessment

Unclear. A code-only change to remove the global filter is not clearly the best fix while the records still carry suspicious verdicts and the visibility policy has security implications.
