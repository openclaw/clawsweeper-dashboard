## Summary

Keep open. Current main and live production still show a real discoverability split: global web search applies the non-suspicious filter, `/skills` browse leaves it optional, and the named production records currently report suspicious moderation rather than cleared moderation.

## What This Changes

The issue reports that two published skills remain hidden from the global web search after the reporter expected moderation to be cleared.

## Best Possible Solution

Maintainers should decide whether these two records are false positives to clear with existing moderation override or rescan tooling, or whether global `/search` should expose the same suspicious-filter behavior as `/skills` with matching tests and docs.

## Reproduction Assessment

Yes. Live API checks on 2026-05-02 returned both slugs from unfiltered `/api/v1/search`, returned zero results with `nonSuspiciousOnly=true`, and both detail/moderation endpoints reported `isSuspicious: true` with `suspicious.llm_suspicious`.

## Solution Assessment

Unclear. The current implementation is internally consistent with hiding suspicious skills in global search, but choosing moderation override, rescan, or a UI/API behavior change is a maintainer product and trust decision.
