## Summary

Keep open. Current main and production both show this is not an index-refresh miss: global `/search` hides active suspicious skills, while `/skills` can show them unless the optional suspicious filter is enabled.

## What This Changes

The issue asks why the published `skillpick` and `threeperson` skills are absent from web search after the reporter expected moderation to be cleared.

## Best Possible Solution

Staff should review the false-positive moderation state and then either clear `isSuspicious` for these skills or make global search visibility match `/skills` with an explicit suspicious-filter UX.

## Reproduction Assessment

Yes. Current main reproduces the route difference in source, and production API calls confirm the reported slugs disappear only when `nonSuspiciousOnly=true` is applied.

## Solution Assessment

No, an index refresh is not the best explanation. The safer path is moderation review plus a product decision on whether global search should silently exclude active suspicious skills.
