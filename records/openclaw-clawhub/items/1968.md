## Summary

Keep open. Current main intentionally hides suspicious skills from global search, and the maintainer production recheck says the named skills were still suspicious, so this is staff moderation follow-up rather than a source search-index bug; ClawHub policy also keeps issues open.

## What This Changes

Reporter asks why `skillpick` and `threeperson` remain absent from global web search despite owner-visible moderation appearing cleared, and whether a search-index refresh or another moderation flag is involved.

## Best Possible Solution

Keep global-search suspicious filtering intact, resolve any false positives through staff moderation review or manual override, and open a narrower diagnostics bug only if owner-facing CLI/API output disagrees with production moderation state.

## Reproduction Assessment

## Solution Assessment

No to a search-index refresh or weakening the global-search filter as the primary fix. The maintainable path is staff moderation review or manual override if the LLM findings are false positives, with diagnostics adjusted only if owner-visible state diverges from staff state.
