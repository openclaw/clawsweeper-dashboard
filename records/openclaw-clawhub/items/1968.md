## Summary

Keep open. ClawHub cleanup policy keeps issues open, and the current evidence points to production moderation follow-up rather than a source search-index bug: global web search deliberately hides suspicious skills, while CLI search and the skills browse page can query without that filter; a maintainer production recheck found the named skills still flagged suspicious.

## What This Changes

The issue asks why the published `skillpick` and `threeperson` skills are visible through some lookup paths but missing from global web search after the reporter believed moderation had cleared.

## Best Possible Solution

Preserve the global-search suspicious filter, have staff review or manually override the named production records if they are false positives, and align owner-facing diagnostics if they disagree with search-visible moderation state.

## Reproduction Assessment

Yes for the visibility split: current source shows global `/search` passes `nonSuspiciousOnly: true`, backend search enforces that filter, while CLI search and `/skills` can query without it. No for the reporter’s cleared-moderation premise from source alone; the production recheck says the records were still suspicious.

## Solution Assessment

No to a search-index refresh or removing the source filter as the primary fix. The narrow maintainable path is staff moderation review or an audited manual override while preserving suspicious-skill filtering.
