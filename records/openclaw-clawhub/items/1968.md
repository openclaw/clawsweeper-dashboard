## Summary

Keep open. ClawHub cleanup policy keeps issues open, and the current evidence points to staff moderation follow-up rather than a source-code search-index bug: global search intentionally excludes suspicious skills, `/skills` can show them unless the hide-suspicious filter is enabled, and the maintainer production recheck says the named records are still suspicious.

## What This Changes

The reporter asks why `skillpick` and `threeperson` remain absent from ClawHub web search despite believing moderation was cleared, and whether the search index or another flag needs refresh.

## Best Possible Solution

Keep the global-search suspicious filter, resolve the named records through staff moderation or an audited manual override if appropriate, and only adjust owner-facing diagnostics if they still report a cleared state while search treats the records as suspicious.

## Reproduction Assessment

Yes for the visibility split: `/search` calls `searchSkills` with `nonSuspiciousOnly: true`, and backend search paths enforce that filter, while `/skills` defaults the filter off. No for the reporter’s cleared-moderation premise from source alone, and the maintainer production recheck says the records are still suspicious.

## Solution Assessment

No for a search-index refresh or source filter removal as the primary fix. The narrow maintainable path is staff moderation review or an audited manual override while preserving suspicious-skill filtering.
