## Summary

Keep open. Current main and production still reproduce the split between global search and skills browse, and the named production records currently report suspicious moderation rather than a cleared searchable state.

## What This Changes

The issue asks why `skillpick` and `threeperson` are absent from global web search after reportedly cleared moderation, including whether a search-index refresh or another visibility flag is involved.

## Best Possible Solution

Maintainers should decide whether these records are false positives to clear with the existing rescan/manual-override flow, or whether global `/search` should expose the same suspicious-filter choice as `/skills` with matching tests and docs.

## Reproduction Assessment

Yes. On 2026-05-02, the public API returned both named slugs without `nonSuspiciousOnly`, returned no results with `nonSuspiciousOnly=true`, and the moderation endpoints reported both as suspicious.

## Solution Assessment

Unclear. The current source behavior is internally consistent with hiding suspicious skills in global search, but the right resolution is a maintainer moderation and product-policy decision rather than a narrow automatic code fix.
