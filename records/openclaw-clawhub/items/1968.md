## Summary

Keep open. ClawHub policy keeps issues open, and the evidence points to a production moderation review rather than a search-index or source-code bug: global search intentionally hides suspicious skills, while the skills browse page can show them unless the hide-suspicious filter is enabled.

## What This Changes

The issue asks why the named published skills are absent from global web search after the reporter believed moderation had cleared.

## Best Possible Solution

Preserve the global-search suspicious filter, resolve the named records through staff moderation or an audited manual override if appropriate, and improve owner-facing diagnostics only if cleared and flagged states are displayed inconsistently.
