Keep open. Current main and production both support the report as a moderation/search-policy follow-up: global search hides suspicious skills, the skills browse page leaves that filter optional, and the two named production records still report suspicious moderation.

## Best Possible Solution

Maintainers should review the two production moderation records and decide whether to clear the suspicious verdicts or keep global search filtered; any product change should align `/search`, `/skills`, tests, and docs.

## Solution Assessment

Unclear. The current implementation is internally consistent with hiding suspicious skills from global search, but deciding whether these records should be cleared or whether global search should match `/skills` requires maintainer policy judgment.
