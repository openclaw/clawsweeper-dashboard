## Summary

Keep open. ClawHub cleanup policy keeps issues open, and the evidence points to a moderation-state follow-up rather than a stale search-index bug: global search intentionally filters suspicious skills, while the skills browse path can show them unless the user opts into hiding suspicious entries.

## What This Changes

The issue asks whether a stale web search index or separate moderation flag is hiding the published `skillpick` and `threeperson` skills after the reporter believed moderation had cleared.

## Best Possible Solution

Keep the global-search suspicious filter in place and resolve the named records through staff moderation review, manual override, or clearer owner-facing moderation diagnostics if the API still makes them look cleared.

## Reproduction Assessment

Yes for the observed visibility split: current source shows `/search` passes `nonSuspiciousOnly: true`, `/skills` does not by default, and the maintainer production recheck reported both named skills still suspicious. I did not establish a source-level bug because the behavior matches the configured moderation boundary.

## Solution Assessment

No for a search-index refresh as the primary fix; the narrow maintainable path is staff moderation review or override of the production records. Changing source search filters would be the wrong fix while the records remain suspicious.
