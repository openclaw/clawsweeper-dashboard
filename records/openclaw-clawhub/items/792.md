## Summary

Keep open. Current main still does not inject a post-install/update event into an active OpenClaw session, and this is a new feature with product and security implications rather than a cleanup-close candidate for a ClawHub issue.

## What This Changes

Request to notify an active OpenClaw session after `clawhub install` or `clawhub update` so the agent reads the installed skill's `SKILL.md` and completes setup guidance.

## Best Possible Solution

Maintainers should define the supported install/update boundary explicitly: either add a user-consented OpenClaw session handoff contract with focused CLI/session tests and docs, or keep the inspectable copy-prompt flow as the supported setup path.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection verifies the current behavior: install/update write files and metadata, while the supported setup path is an inspectable copied prompt rather than active-session injection.

## Solution Assessment

Unclear. A system event could solve the requested trigger, but it is not clearly the best maintainable solution until maintainers decide the consent and security model for active-session influence.
