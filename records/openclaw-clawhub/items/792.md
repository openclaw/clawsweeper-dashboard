## What This Changes

The issue requests injecting an active-session system event after `clawhub install` or `clawhub update` so the agent reads SKILL.md onboarding and setup instructions.

## Best Possible Solution

Define a user-consented install/update handoff contract covering event shape, OpenClaw session IPC, disclosure, moderation and hook policy, tests, and docs before implementing the active-session trigger.

## Reproduction Assessment

No as a bug reproduction; this is a feature request rather than broken documented CLI behavior. Source inspection gives high confidence that current install/update code paths end after file/metadata writes and status output with no session event or handoff.

## Solution Assessment

Unclear as proposed. A system event may solve the workflow gap, but the maintainable path must first define a safe, user-visible handoff contract instead of auto-injecting newly installed SKILL.md content into an active session.
