## Summary

Keep open. Current main still does not inject a post-install/update event into an active OpenClaw session, and this ClawHub cleanup pass keeps issues open unless they are resolved PRs; the request also needs maintainer product/security judgment before implementation.

## What This Changes

Request adding a post-install/update handoff that tells an active OpenClaw session to read the installed or updated skill's SKILL.md and complete setup guidance.

## Best Possible Solution

Keep the inspectable copy-prompt flow unless maintainers approve a user-consented OpenClaw handoff contract with install/update tests, docs, and explicit security boundaries.

## Reproduction Assessment

Yes. Source inspection on current main shows install/update write skill files and metadata but have no active-session notification, SKILL.md onboarding read, or handoff call.

## Solution Assessment

Unclear. Event injection would address the onboarding gap, but the current approved UX deliberately uses inspectable copyable prompts, so maintainers need to define consent and trust before adding direct handoff.
