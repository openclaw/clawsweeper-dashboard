## Summary

Keep open. Current main still has no post-install/update active-session handoff, GitHub shows no closing PR, and a recent Patrick-Erichsen comment says this feature should be prioritized; the remaining work needs product and security policy before implementation.

## What This Changes

Request a post-install/update system event that tells the active OpenClaw session to read the installed skill's SKILL.md and complete setup or onboarding steps.

## Best Possible Solution

Define a user-consented install/update handoff contract covering event shape, OpenClaw session IPC, disclosure, moderation/hook policy, tests, and docs before implementing the active-session trigger.

## Reproduction Assessment

Yes, as source-level verification rather than a runtime bug harness. Current main's install/update paths have no session handoff, OpenClaw IPC, or system-event injection after writing skill files and metadata.

## Solution Assessment

Unclear as proposed. A system event may be the right UX, but the maintainable solution depends on maintainer-approved consent, security, and OpenClaw session-handoff policy.

## Risks / Open Questions

- Automatic active-session injection would allow newly installed or updated skill content to influence an already-running agent, so consent, moderation state, prompt-injection handling, and disclosure rules need maintainer approval.
- The approved install-surface spec currently excludes direct OpenClaw handoff, so implementing this would intentionally revise an existing product boundary.
- Related hook/security work remains open at https://github.com/openclaw/clawhub/issues/707 and https://github.com/openclaw/clawhub/issues/2295; this feature should not define conflicting lifecycle automation rules.
- The ClawHub CLI does not currently expose an active OpenClaw session IPC or handoff surface, so the fix likely needs an explicit cross-project contract rather than a narrow CLI-only patch.
