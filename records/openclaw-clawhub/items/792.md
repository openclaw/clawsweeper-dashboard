## Summary

Keep open. Current main still does not inject a post-install/update event into an active OpenClaw session, and the request needs product and security judgment before any implementation because it would let newly installed or updated skill content influence an active agent session.

## What This Changes

Request to notify an active OpenClaw session after `clawhub install` or `clawhub update` so the agent reads SKILL.md onboarding instructions.

## Best Possible Solution

Maintainers should define the supported install/update boundary explicitly: either add a user-consented OpenClaw session handoff contract with tests and docs, or document the current inspectable copy-prompt flow as the intended setup path.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a requested new handoff behavior, not a broken existing contract. Source inspection gives high confidence that current main lacks post-install/update active-session event injection.

## Solution Assessment

Unclear as proposed. A system event may be a viable UX, but the safer next step is a maintainer-approved handoff and consent contract because the current approved design uses inspectable copyable prompts only.
