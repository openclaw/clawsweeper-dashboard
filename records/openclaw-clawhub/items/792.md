## Summary

Keep open. Current main still installs or updates skill files and local metadata without any active-session system event, and the remaining change needs maintainer product/security decisions because it would let newly installed or updated skill instructions influence an already-running OpenClaw session.

## What This Changes

The issue requests that `clawhub install` and `clawhub update` notify the active OpenClaw session after installing or updating a skill so the agent reads `SKILL.md` and performs onboarding/setup instructions.

## Best Possible Solution

Define a user-consented OpenClaw session handoff contract, including event shape, moderation/trust checks, prompt-injection handling, tests, and docs, before wiring install/update to emit it.

## Reproduction Assessment

Yes for source-level reproduction: current `cmdInstall` and `cmdUpdate` update files/metadata and emit success messages without active-session IPC or system-event injection. This is not a conventional bug reproduction because the report asks for a new handoff capability beyond the current documented CLI contract.

## Solution Assessment

Unclear as proposed. A system event may be the right UX, but the maintainable path depends on maintainer-approved consent, security, and OpenClaw session-handoff policy.
