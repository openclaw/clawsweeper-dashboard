Current main still has no post-install/update active-session handoff, the item is an issue that this ClawHub cleanup lane should not auto-close, and a recent Patrick-Erichsen comment says the feature should be prioritized; the remaining work needs product and security policy before implementation.

## What This Changes

The issue requests that `clawhub install` and `clawhub update` notify the active OpenClaw session to read the installed or updated skill's `SKILL.md` and complete onboarding/setup actions.

## Best Possible Solution

Define a user-consented install/update handoff contract covering event shape, OpenClaw session IPC, disclosure, moderation and hook policy, tests, and docs before implementing the active-session trigger.

## Reproduction Assessment

No as a bug reproduction; this is a new active-session handoff capability rather than a broken documented CLI contract. Yes as source-level verification: current main's install/update paths have no session handoff or system-event injection.

## Solution Assessment

Unclear as proposed. A system event may be the right UX, but the maintainable solution depends on maintainer-approved consent, security, and OpenClaw session-handoff policy.
