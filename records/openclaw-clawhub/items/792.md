## Summary

Keep open. Current main still installs or updates skill files and local metadata without injecting a post-install/update system event into an active OpenClaw session, and the maintainer discussion treats this as useful work that needs product and security policy before implementation.

## What This Changes

Request to notify the active agent session after `clawhub install` or `clawhub update` so it reads the installed skill's `SKILL.md` and handles onboarding or setup instructions.

## Best Possible Solution

Define a user-consented OpenClaw session handoff contract with event shape, trust and moderation rules, prompt-injection handling, tests, and docs before wiring install/update to emit it.

## Reproduction Assessment

No conventional bug reproduction applies because this is a new active-session handoff capability. Source inspection gives a high-confidence verification path: current `cmdInstall` and `cmdUpdate` have no session handoff or system-event injection.

## Solution Assessment

Unclear as proposed. A system event may be the right UX, but the maintainable path depends on maintainer-approved consent, security, and OpenClaw session-handoff policy.
