## Summary

Keep open. Current main still does not inject a post-install/update event into an active OpenClaw session, and the request is a new product/security capability rather than a cleanup-close candidate for a ClawHub issue.

## What This Changes

Request to notify an active OpenClaw session after `clawhub install` or `clawhub update` so the agent reads the installed skill's `SKILL.md` and performs setup/onboarding steps.

## Best Possible Solution

Define the install/update handoff contract explicitly: either add a user-consented active-session handoff with tests and docs, or document the inspectable copy-prompt flow as the supported setup path.

## Reproduction Assessment

Yes for the current behavior. Source inspection shows `install` and `update` dispatch directly to file/metadata-writing CLI functions without notifying an active OpenClaw session or reading `SKILL.md` onboarding instructions.

## Solution Assessment

Unclear. A system event could address the requested trigger, but it is not clearly the best maintainable solution until maintainers decide the consent and security model for active-session influence.
