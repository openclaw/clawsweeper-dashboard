## Summary

## What This Changes

The issue requests a post-install/update system event that tells the active OpenClaw session to read the installed skill's SKILL.md and complete setup or onboarding steps.

## Best Possible Solution

Define a user-consented OpenClaw session handoff contract that covers trust boundaries, hook/lifecycle policy, tests, and docs, or explicitly keep the inspectable copy-prompt flow as the supported setup path.

## Reproduction Assessment

No as a bug reproduction; this is a new active-session handoff capability rather than a broken documented contract. Source inspection gives high confidence that current main does not perform the requested install/update event injection.

Unclear as proposed. A system event could address the onboarding gap, but the maintainable path depends on maintainer-approved consent, trust-boundary, and OpenClaw/ClawHub handoff design.
