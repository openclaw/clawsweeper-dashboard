## Summary

Keep open. Current main still does not inject a post-install/update event into an active OpenClaw session, and this ClawHub cleanup policy keeps issues open unless they are PRs already implemented on main; the request also needs maintainer product/security judgment before implementation.

## What This Changes

The issue requests that ClawHub inject a post-install or post-update event into an active OpenClaw session so the agent reads the installed skill's SKILL.md and completes onboarding work.

## Best Possible Solution

Maintainers should define the install/update boundary explicitly: either add a user-consented OpenClaw session handoff contract with focused CLI/session tests and docs, or document the inspectable copy-prompt flow as the supported setup path.

## Reproduction Assessment

Yes. Source inspection on current main is enough to reproduce the missing behavior: install/update paths write skill files and metadata but have no session event, SKILL.md onboarding read, or handoff call.

## Solution Assessment

Unclear. The proposed event injection addresses the onboarding gap, but it is not clearly the best solution until maintainers define consent, trust, and the OpenClaw/ClawHub handoff boundary; the current copyable prompt flow is the safer accepted path today.
