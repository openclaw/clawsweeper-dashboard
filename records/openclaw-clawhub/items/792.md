## Label Justifications

- `P2`: The missing handoff leaves some complex skills incompletely configured, but basic registry and installation workflows remain functional.
- `impact:security`: Installed skill instructions may request arbitrary commands, configuration changes, package installation, or process restarts.
- `impact:session-state`: The requested feature injects new context and follow-up actions into an already active agent session.

## Summary

Current main still lacks any post-install or post-update handoff to an active agent session, and a repository collaborator has explicitly endorsed prioritizing the capability. The user problem is valid, but the exact execution and consent model requires maintainer product and security judgment.

## What This Changes

Request to notify the active agent session after a skill install or update so it reads and performs SKILL.md onboarding instructions.

## Best Possible Solution

Define a structured, security-reviewed onboarding result shared with native OpenClaw, notify the active session that guidance is available, and require explicit operator confirmation before privileged skill-authored actions execute.

## Reproduction Assessment

Yes, at source level. Current main’s install path ends after local file and metadata writes, optional telemetry, and success output, with no active-session event, callback, or onboarding result.

## Solution Assessment

No, not exactly as proposed. A post-install handoff is appropriate, but unconditional execution of SKILL.md instructions should be replaced by a typed host-controlled notification and explicit consent for privileged steps.
