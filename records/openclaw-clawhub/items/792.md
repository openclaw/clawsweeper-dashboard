## Summary

Keep open. Current main still lacks post-install/update active-session event injection, and this ClawHub issue is a feature request that needs maintainer product/security judgment before implementation.

## What This Changes

Requests a post-install/update active-session system event so OpenClaw reads a newly installed or updated skill's SKILL.md and runs setup or onboarding steps.

## Best Possible Solution

Maintainers should define the supported install/update boundary explicitly: either a user-consented OpenClaw session handoff contract with tests/docs, or the current inspectable copy-prompt flow as the intended setup path.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection shows install/update only write local files and metadata, while the supported setup guidance is an inspectable copy-prompt flow.

## Solution Assessment

Unclear. A system event would address the requested trigger, but it is not clearly the best maintainable solution until maintainers decide the consent, security, and OpenClaw/ClawHub handoff model.
