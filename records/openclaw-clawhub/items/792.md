Keep open. Current main still has no post-install/update active-session event injection, and this is a feature plus security/session-boundary decision rather than a conservative cleanup close.

## What This Changes

Request to notify an active OpenClaw session after `clawhub install` or `clawhub update` so the agent reads SKILL.md setup and onboarding instructions.

## Best Possible Solution

Define the supported install/update boundary explicitly: either add a user-consented OpenClaw session handoff contract with security policy, tests, and docs, or keep the inspectable copy-prompt flow as the intended boundary.

## Reproduction Assessment

Not applicable as a bug reproduction: this requests a new active-session handoff behavior, not a broken existing contract. Source inspection gives high confidence that current main lacks the requested install/update event injection.

## Solution Assessment

Unclear as proposed. A system event could address the onboarding gap, but the maintainable path needs maintainer-approved consent and trust-boundary design aligned with the existing inspectable prompt surface and the related hooks security RFC.
