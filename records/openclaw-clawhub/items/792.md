## Summary

Keep open. Current main still lacks an install/update-to-active-session handoff, and ClawHub cleanup policy keeps issues open unless a narrow close reason is proven; this request also needs maintainer product and security judgment before implementation because it would let newly installed or updated skill content influence active agent context.

## What This Changes

Request a ClawHub/OpenClaw handoff that injects a post-install or post-update system event into the active session so the agent reads the installed skill's SKILL.md and performs setup guidance.

## Best Possible Solution

Define the supported install/update boundary: either add a user-consented OpenClaw session handoff contract with security policy, tests, and docs, or keep the inspectable copy-prompt flow as the intended setup path.

## Reproduction Assessment

No as a bug reproduction; this is a new active-session handoff capability rather than a broken existing contract. Source inspection gives high confidence that current main does not perform the requested install/update event injection.

## Solution Assessment

Unclear as proposed. A system event could address the onboarding gap, but the best path depends on maintainer-approved consent, trust-boundary, and OpenClaw/ClawHub handoff design instead of a CLI-only change.
