## Summary

Keep open. Current main still does not inject a post-install/update event into an active OpenClaw session, and ClawHub cleanup policy keeps issues open unless a strong close reason applies; this request also needs product/security judgment before implementation.

## What This Changes

After `clawhub install` or `clawhub update`, notify the active agent session to read the installed skill's `SKILL.md` and perform setup or onboarding instructions.

## Best Possible Solution

Define the supported install/update boundary explicitly: either add a user-consented OpenClaw session handoff contract with focused CLI/session tests and docs, or document the inspectable copy-prompt flow as the intended setup path.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Current-main source inspection shows install/update write files and metadata, while setup guidance is only available through an inspectable OpenClaw prompt flow.

## Solution Assessment

Unclear. A system event could address the trigger, but it is not clearly the best maintainable solution until maintainers decide the consent, security, and OpenClaw/ClawHub handoff model.
