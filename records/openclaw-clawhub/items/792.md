## Summary

Keep open. Current main still does not inject a post-install/update event into an active OpenClaw session, and this is a feature/product-security decision rather than a cleanup-close candidate for a ClawHub issue.

## What This Changes

Request to make `clawhub install` and `clawhub update` notify an active OpenClaw session so the agent reads the installed or updated skill's `SKILL.md` onboarding instructions.

## Best Possible Solution

Define the supported install/update boundary explicitly: either add a user-consented OpenClaw session handoff contract with focused CLI/session tests and docs, or document the inspectable copy-prompt flow as the intended setup path.

## Reproduction Assessment

Yes for the current behavior. Source inspection shows `install` and `update` dispatch directly to file and metadata-writing CLI functions without notifying an active OpenClaw session or reading `SKILL.md` onboarding instructions.

## Solution Assessment

Unclear. A system event could address the requested trigger, but it is not clearly the best maintainable solution until maintainers decide the consent and security model for active-session influence.
