## Summary

Keep open. Current main still lacks post-install/update active-session event injection, and this ClawHub cleanup policy keeps issues open for maintainer handling rather than auto-closing them.

## What This Changes

Request to notify an active OpenClaw session after `clawhub install` or `clawhub update` so the agent reads SKILL.md and completes setup/onboarding steps.

## Best Possible Solution

Define the supported install/update boundary explicitly: either add a user-consented OpenClaw handoff contract with tests and docs, or keep the inspectable copy-prompt flow as the supported setup path.

## Reproduction Assessment

Yes. Source inspection on current main shows install/update write files and metadata but do not emit a session event, read SKILL.md for onboarding, or call a handoff surface.

## Solution Assessment

Unclear. Event injection addresses the onboarding gap, but it is not clearly the best solution until maintainers define consent, trust, and the OpenClaw/ClawHub handoff boundary.
