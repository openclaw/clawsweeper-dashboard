## Summary

Keep open. Current main still does not inject a post-install/update event into an active OpenClaw session, and this ClawHub issue is a feature/product-security decision rather than a conservative cleanup close candidate.

## What This Changes

The issue requests injecting a post-install/update system event into an active OpenClaw session after ClawHub CLI skill install or update.

## Best Possible Solution

Maintainers should define the install/update boundary explicitly: either add a user-consented OpenClaw session handoff contract with focused tests and docs, or keep the inspectable copy-prompt flow as the supported setup path.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection verifies current main writes skill files and metadata and supports inspectable copy-prompt setup, but not active-session event injection.

## Solution Assessment

Unclear. A system event could address the requested trigger, but it is not clearly the best maintainable solution until maintainers decide the consent, security, and OpenClaw/ClawHub handoff model.
