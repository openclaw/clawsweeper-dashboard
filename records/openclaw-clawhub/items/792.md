## Summary

Keep open. Current main still has no post-install/update active-session event injection, and the request needs maintainer product/security judgment because it would let newly installed or updated skill content influence an active agent session.

## What This Changes

Request to inject a system event after `clawhub install` or `clawhub update` so the active agent reads the installed skill's `SKILL.md` and completes onboarding or update steps.

## Best Possible Solution

Define the supported install/update handoff contract explicitly: either add a user-consented active-session event with security policy, tests, and docs, or keep the inspectable copy-prompt flow as the intended boundary.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a request for a new active-session handoff behavior, not a broken existing contract. Source inspection gives high confidence that current main lacks the requested install/update event injection.

## Solution Assessment

Unclear as proposed. A system event could address the onboarding gap, but the maintainable path needs a maintainer-approved consent and trust-boundary design aligned with the current inspectable prompt surface and the related hooks security RFC.
