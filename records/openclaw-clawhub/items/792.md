## Summary

Keep open. Current main and the latest CLI release still lack a post-install/update active-session handoff, and the discussion includes collaborator support; the remaining work needs product and security design rather than cleanup closure.

## What This Changes

The issue requests that ClawHub install/update notify the active OpenClaw session so the agent reads the installed skill instructions and completes onboarding after install or update.

## Best Possible Solution

Define a durable, consent-aware ClawHub/OpenClaw install/update handoff that safely selects or declines a session target, presents requested setup actions, and requires explicit approval before consequential execution.

## Reproduction Assessment

Yes, from source inspection: the current install/update commands complete without any active-session notification or system-event layer. This verifies the capability is absent, but it is a feature gap rather than a broken documented behavior.

## Solution Assessment

No, not exactly as proposed. A structured notification may be appropriate, but unconditional instructions to execute SKILL.md onboarding need explicit consent, session targeting, failure handling, and trust-boundary rules first.

## Security Review

Status: needs_attention

Summary: The request is security-sensitive because it could turn newly installed skill instructions into active agent behavior without a settled trust and consent model.

Concerns:

- **[high] Prevent implicit execution of installed instructions:**
  - body: A newly installed or updated skill controls its SKILL.md content, so automatically directing an active agent to execute onboarding can trigger commands, configuration edits, plugin activation, or restarts without a distinct operator decision.
  - confidence: 0.93
- **[medium] Authenticate and scope session delivery:**
  - body: Any system-event channel must define which local gateway and session receive the event and prevent unrelated processes or sessions from injecting trusted-looking system context.
