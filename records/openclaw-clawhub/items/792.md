## Summary

Keep open. Current main still does not inject a post-install/update event into an active OpenClaw session, and this ClawHub issue needs maintainer product/security judgment rather than cleanup closure.

## What This Changes

The issue requests a post-install/update event that tells an active OpenClaw session to read the installed skill's SKILL.md and complete setup or onboarding work.

## Best Possible Solution

Maintainers should define the supported install/update boundary explicitly: either add a user-consented OpenClaw session handoff contract with focused tests and docs, or document the inspectable copy-prompt flow as the intended setup path.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that current main installs or updates files and metadata but does not inject an active-session event.

## Solution Assessment

Unclear. A system event could address the requested trigger, but the maintainable path depends on an explicit consented OpenClaw/ClawHub handoff policy and the related lifecycle hook security contract.
