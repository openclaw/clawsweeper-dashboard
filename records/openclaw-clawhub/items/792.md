## Summary

Keep open. Current main still only installs or updates skill files and local metadata; it does not inject a post-install/update event into an active OpenClaw session, and a collaborator has said this should be prioritized. The remaining work needs maintainer product and security policy because the current install-surface spec explicitly avoids direct OpenClaw handoff.

## What This Changes

The issue requests a post-install/update active-session event so OpenClaw reads the installed or updated skill's SKILL.md and performs setup or onboarding work.

## Best Possible Solution

Define a user-consented OpenClaw session handoff contract, including event shape, trust and moderation rules, prompt-injection handling, tests, and docs, before wiring install/update to emit it.

## Reproduction Assessment

No for a conventional bug reproduction; this is a new active-session handoff capability rather than broken documented CLI behavior. Yes for source-level verification: current main's install/update paths have no session handoff or system-event injection.

## Solution Assessment

Unclear as proposed. A system event may be the right UX, but the maintainable path depends on maintainer-approved consent, security, and OpenClaw session-handoff policy.
