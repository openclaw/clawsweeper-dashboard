## Summary

Keep open. Current main and v0.21.0 still do not inject a post-install/update event into an active OpenClaw session, and the issue has explicit collaborator support; the remaining work is a product and security decision around session targeting, consent, and publisher-controlled setup instructions.

## What This Changes

Request to notify an active OpenClaw session after `clawhub install` or `clawhub update` so the agent reads the installed skill metadata and completes setup/onboarding.

## Best Possible Solution

Define a consent-aware ClawHub/OpenClaw install-update handoff contract that safely selects or declines a session target, presents requested setup actions, and requires approval before consequential execution.

## Reproduction Assessment

Yes, at source level: current main's install and update flows complete by writing files and metadata, with no active-session notification or gateway handoff. This confirms the requested capability is absent, though it is a feature gap rather than a broken documented contract.

## Solution Assessment

No, not exactly as proposed. A structured notification is plausible, but unconditional instructions to execute setup from SKILL.md need maintainer-approved consent, session targeting, trust, and failure-handling rules first.
