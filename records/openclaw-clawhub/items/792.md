## Summary

Keep open. Current main still does not inject a post-install/update event into an active OpenClaw session, and this issue needs maintainer product/security judgment because it would let newly installed or updated skill content influence active agent context.

## What This Changes

Request adding a post-install/update active-session event so OpenClaw reads the installed skill's SKILL.md and runs setup or onboarding instructions.

## Best Possible Solution

Define the install/update handoff contract explicitly: either add a user-consented active-session event with security policy, tests, and docs, or keep the inspectable copy-prompt flow as the supported boundary.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a request for a new active-session handoff behavior, not a broken existing contract. Source inspection gives high confidence that current main lacks the requested install/update event injection.

## Solution Assessment

Unclear as proposed. A system event could address the onboarding gap, but the maintainable path needs a maintainer-approved consent and trust-boundary design aligned with the current inspectable prompt surface and the open hooks security RFC.
