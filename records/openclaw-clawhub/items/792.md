## Summary

Keep open. Current main and v0.22.0 still complete `clawhub install` / `clawhub update` by writing skill files and metadata only, while the requested active-session event remains a product and security decision with explicit collaborator interest.

## What This Changes

Request to inject a post-install/update system event into an active OpenClaw session so the agent reads the installed skill's `SKILL.md` and completes setup guidance.

## Best Possible Solution

Define a consent-aware ClawHub/OpenClaw install-update handoff contract that safely selects or declines a session target, presents requested setup actions, and requires approval before consequential execution.

## Reproduction Assessment

Yes, at source level. Current `cmdInstall` and `cmdUpdate` write files/origin/lock metadata and print success without a session event or `SKILL.md` onboarding read; I did not run a live OpenClaw gateway reproduction.

## Solution Assessment

No, not exactly as proposed. A structured notification may be the right mechanism, but unconditional instructions to execute setup from `SKILL.md` need maintainer-approved consent, session targeting, trust, and failure-handling rules first.

## Risks / Open Questions

- A package-manager command would need an authenticated and deterministic way to target the right OpenClaw session when multiple or no sessions are active.
