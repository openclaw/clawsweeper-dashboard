## Summary

Keep open. Current main still does not inject a post-install/update event into an active OpenClaw session, and ClawHub cleanup policy keeps issues open; this is a concrete feature request but needs product/security review before implementation.

## What This Changes

The issue requests a post-install/post-update active-session system event that tells the agent to read the installed skill's SKILL.md and perform onboarding or setup instructions.

## Best Possible Solution

Define an explicit user-consented OpenClaw install/update handoff contract with tests and docs, or codify the inspectable copy-prompt flow as the supported setup path.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that current main writes skill files/metadata and offers inspectable copy prompts, but does not inject an active-session event.

## Solution Assessment

Unclear. A system event could solve the requested trigger, but the current approved design intentionally uses copyable prompts, so the maintainable solution depends on maintainer product/security decisions about consent and active-session influence.
