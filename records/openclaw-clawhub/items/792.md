## Summary

Keep open. Current main still does not inject a post-install/update event into an active OpenClaw session, and this ClawHub issue needs maintainer product/security judgment rather than cleanup closure.

## What This Changes

Request to inject a system event after `clawhub install` or `clawhub update` so the active agent reads the installed skill's `SKILL.md` and handles setup/onboarding steps.

## Best Possible Solution

Maintainers should define the supported install/update boundary explicitly: either a user-consented OpenClaw session handoff contract with tests and docs, or the current inspectable copy-prompt flow as the intended setup path.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that current main installs or updates files and metadata but does not inject an active-session event.

## Solution Assessment

Unclear. A system event matches the requested trigger, but the maintainable approach depends on an explicit consented OpenClaw/ClawHub handoff policy and the related lifecycle hook security contract.
