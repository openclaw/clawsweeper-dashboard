## Summary

Keep open. Current main still has no post-install/update active-session handoff, and the remaining work needs maintainer product and security policy because it would let newly installed or updated skill content influence an active agent session.

## What This Changes

Request to inject a post-install/update system event into the active OpenClaw session so the agent reads the installed skill's SKILL.md and performs onboarding/setup steps.

## Best Possible Solution

Define a user-consented install/update handoff contract covering event shape, OpenClaw session IPC, disclosure, moderation, hook policy, tests, and docs before implementing the active-session trigger.
