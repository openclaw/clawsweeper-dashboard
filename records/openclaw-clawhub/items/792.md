## Summary

Keep open. Current main still lacks active-session post-install/update event injection, and the existing product surface intentionally uses inspectable copyable prompts; changing that boundary needs maintainer product and security review.

## What This Changes

The issue requests an active-session system event after `clawhub install` or `clawhub update` so the agent reads the installed skill’s `SKILL.md` and performs setup or onboarding steps.

## Best Possible Solution

Maintainers should define the install/update boundary explicitly: either add a user-consented OpenClaw session handoff contract with focused CLI/session tests and docs, or document the current inspectable copy-prompt flow as the supported setup path.

## Reproduction Assessment

Yes. A source-level reproduction path is to follow `clawhub install` and `clawhub update` from `cli.ts` into `cmdInstall` / `cmdUpdate`; current main stops at extraction, metadata writes, and status output without reading `SKILL.md` or emitting a session event.

## Solution Assessment

Unclear. The requested system-event injection could solve the workflow gap, but it is not clearly the best fix until maintainers decide the consent and trust boundary; the safer current path is the inspectable `Install & Setup` prompt.
