## Summary

Keep open. Current main still does not inject a post-install/update event into an active OpenClaw session, and the issue has explicit collaborator support; the remaining work is a product and security decision around cross-process session handoff, consent, and publisher-controlled setup instructions.

## What This Changes

Request a ClawHub install/update flow that notifies an active agent session to read the installed skill's SKILL.md and continue setup or onboarding.

## Best Possible Solution

Define a consent-aware ClawHub/OpenClaw install-update handoff contract that safely selects or declines a session target, presents requested setup actions, and requires approval before consequential execution.

## Reproduction Assessment

Yes, at source level: current main's install and update flows complete by writing files and metadata, with no active-session notification or gateway handoff. This confirms the requested capability is absent, though it is a feature gap rather than a broken documented contract.

## Solution Assessment

No, not exactly as proposed. A structured notification is plausible, but unconditional instructions to execute setup from SKILL.md need maintainer-approved consent, session targeting, trust, and failure-handling rules first.

## Risks / Open Questions

- A package-manager command would need authenticated session discovery and deterministic behavior when multiple or no OpenClaw sessions are active.
- Automatically telling an agent to execute SKILL.md onboarding could turn publisher-controlled text into commands, config edits, plugin activation, or gateway restarts without a clear operator decision.
- The ClawHub CLI currently lacks an explicit OpenClaw gateway/session IPC contract, so a narrow CLI-only patch could create unreliable install or partial-setup behavior.
- The narrower hook-registration work at https://github.com/openclaw/clawhub/issues/2295 should align with this handoff boundary so declarative hooks, install metadata, and agent onboarding do not become competing setup sources.
