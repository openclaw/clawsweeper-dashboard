## Summary

Keep open. Current main still does not inject a post-install/update event into an active OpenClaw session, and ClawHub cleanup policy keeps issues open; the request is concrete but crosses a product/security boundary around newly installed or updated skill content influencing an active agent.

## What This Changes

Request that `clawhub install` and `clawhub update` notify the active OpenClaw session to read the installed skill's `SKILL.md` and perform setup or update onboarding.

## Best Possible Solution

Maintainers should define the supported install/update boundary: keep the inspectable copy-prompt flow as intentional, or approve a user-consented OpenClaw session handoff contract with focused CLI/session tests, docs, and hook/security-policy alignment.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that current main writes skill files and metadata and offers inspectable copy prompts, but does not inject an active-session event.

## Solution Assessment

Unclear. A system event could satisfy the requested trigger, but the narrow maintainable path depends on maintainer product/security decisions because the current approved design intentionally avoids direct handoff and automatic execution.
