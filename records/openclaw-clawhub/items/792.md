## Summary

Keep open. Current main still lacks the requested post-install/update active-session handoff, and implementing it would revise the approved inspectable prompt boundary while raising product and security questions about newly installed skill content influencing agent context.

## What This Changes

Request active-session system events after `clawhub install` or `clawhub update` so the agent reads the installed skill's `SKILL.md` and performs setup/onboarding work.

## Best Possible Solution

Define the install/update handoff contract explicitly: either add a user-consented active-session event with security policy, tests, and docs, or keep the inspectable copy-prompt flow as the supported boundary.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a request for new active-session handoff behavior, not a broken existing contract. Source inspection gives high confidence that current main lacks the requested install/update event injection.

## Solution Assessment

Unclear as proposed. A system event could address the onboarding gap, but the maintainable path needs maintainer-approved consent and trust-boundary design aligned with the current inspectable prompt surface and the open hooks security RFC.
