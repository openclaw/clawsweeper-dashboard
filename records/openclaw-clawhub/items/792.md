## Summary

Keep open. Current main still installs or updates skills as local files plus ClawHub metadata without any active-session system event, and a recent maintainer comment says this should be prioritized rather than cleaned up.

## What This Changes

The issue requests a new ClawHub/OpenClaw handoff that injects a post-install or post-update system event into the active agent session so the agent reads the installed skill's setup instructions.

## Best Possible Solution

Define a user-consented install/update handoff contract covering event shape, OpenClaw session IPC, disclosure, moderation and hook policy, tests, and docs before implementing the active-session trigger.

## Reproduction Assessment

No as a bug reproduction; this is a new active-session handoff capability rather than a broken documented CLI contract. Source inspection gives high confidence that current main does not perform the requested install/update event injection.

## Solution Assessment

Unclear as proposed. A system event may be the right UX, but the maintainable solution depends on maintainer-approved consent, security, and OpenClaw session-handoff policy.
