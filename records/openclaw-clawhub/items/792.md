## Summary

Keep open. Current main still lacks an install/update-to-active-session handoff, and a collaborator has explicitly supported the feature; implementation needs maintainer product and security judgment because it would cross the existing explicit-consent boundary for skill setup.

## What This Changes

Notify an active OpenClaw session after a ClawHub skill install or update so the agent can inspect the skill’s setup instructions and help complete onboarding.

## Best Possible Solution

Define an authenticated, user-consented notification-only handoff that makes setup instructions inspectable and requires confirmation before executing broader environment changes.

## Reproduction Assessment

Yes. Current-main source inspection shows that `cmdInstall` and `cmdUpdate` update files and metadata and report completion without emitting an event to an active OpenClaw session.

## Solution Assessment

Unclear as proposed. A session notification could address the onboarding gap, but automatically directing the agent to execute fresh skill instructions is not the safest design; maintainers first need to approve consent, trust, and IPC semantics.
