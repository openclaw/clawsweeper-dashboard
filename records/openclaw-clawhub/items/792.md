## Summary

Keep open. Current main still does not inject a post-install/update event into an active OpenClaw session, and ClawHub cleanup policy keeps issues open unless a PR is certainly implemented on main; this request also needs product/security judgment around active-session influence.

## What This Changes

The issue requests an install/update lifecycle handoff that tells the active OpenClaw session to read the newly installed or updated skill instructions and perform setup steps.

## Best Possible Solution

Maintainers should define the supported install/update boundary explicitly: either add a user-consented OpenClaw session handoff contract with CLI/session tests and docs, or document the inspectable copy-prompt flow as the intended setup path.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that current main installs or updates files and metadata but does not inject an active-session event.

## Solution Assessment
