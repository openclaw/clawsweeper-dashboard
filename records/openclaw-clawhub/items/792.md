## What This Changes

Requested behavior is to inject an active-session system event after `clawhub install` or `clawhub update` so the agent reads the installed skill's `SKILL.md` and performs approved onboarding or update steps.

## Best Possible Solution

Define a user-consented install/update handoff contract covering event shape, OpenClaw session IPC, disclosure, moderation state, lifecycle hook policy, tests, and docs before implementing the active-session trigger.

## Reproduction Assessment

No as a bug reproduction, because this is a new active-session handoff capability rather than a broken documented CLI contract. Source inspection gives high confidence that current main does not perform the requested install/update event injection.

## Solution Assessment

Unclear as proposed. A system event may be the right UX, but the maintainable solution depends on maintainer-approved consent, security, and OpenClaw session-handoff policy.
