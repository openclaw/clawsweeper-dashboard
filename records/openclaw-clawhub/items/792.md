## Summary

Keep open. Current main still does not inject a post-install/update event into an active OpenClaw session, and ClawHub cleanup policy keeps issues open unless a strong close reason applies; the related hook-policy RFC at https://github.com/openclaw/clawhub/issues/707 remains open but does not supersede this narrower handoff request.

## What This Changes

Request a post-install/update active-session event so OpenClaw reads the installed or updated skill's `SKILL.md` and performs setup guidance.

## Best Possible Solution

Maintainers should define the supported install/update handoff contract: either add a user-consented OpenClaw-native session event with focused CLI/session tests and docs, or keep the current inspectable copy-prompt flow as the intended setup path.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that current main installs and updates local files/metadata but does not inject an active-session event.

## Solution Assessment

Unclear. A system event could address the trigger, but it is not clearly the best maintainable solution until maintainers decide the consent, security, and OpenClaw/ClawHub handoff model.
