## Summary

Keep open. Current main still does not inject a post-install/update event into an active OpenClaw session, and ClawHub issue cleanup policy does not auto-close ordinary issues. The request is concrete, but it needs maintainer product/security judgment because it would let newly installed or updated skill content influence an active agent session.

## What This Changes

Request a post-install/post-update system event that tells the active agent session to read the installed skill's SKILL.md and perform setup or onboarding steps.

## Best Possible Solution

Maintainers should define the supported install/update boundary explicitly: either add a user-consented OpenClaw session handoff contract with focused tests and docs, or make the inspectable copy-prompt flow the documented setup path.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection confirms current install/update paths write files and metadata only, while setup guidance exists through an inspectable prompt flow rather than an active-session event.

## Solution Assessment

Unclear. A system event could address the requested trigger, but the current approved design intentionally uses copyable prompts, so the best solution depends on maintainer product and security decisions about active-session influence.
