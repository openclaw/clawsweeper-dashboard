## Summary

Keep open. Current main still does not inject a post-install/update event into an active OpenClaw session after ClawHub CLI install/update, and this ClawHub cleanup pass should keep issues open unless they are already resolved PRs; the request also crosses product and security boundaries around active-session influence from installed skill content.

## What This Changes

Request post-install/update active-session notification so OpenClaw reads the installed skill's `SKILL.md` and performs onboarding or update setup work.

## Best Possible Solution

Define the supported install/update boundary explicitly: either add a user-consented OpenClaw session handoff contract with focused CLI/session tests and docs, or keep and document the inspectable copy-prompt flow as the supported setup path.

## Reproduction Assessment

Yes for the current behavior. Source inspection shows install/update dispatch directly to `cmdInstall`/`cmdUpdate`, which write local files and metadata without a session notification or `SKILL.md` onboarding read.

## Solution Assessment

Unclear. A system event would address the onboarding trigger, but it changes the current explicit copy-prompt boundary and should not be treated as the narrowest maintainable solution until maintainers decide the consent and trust model.
