## Summary

Keep open: current main still lacks an install/update active-session handoff, and a recent Patrick-Erichsen comment explicitly prioritizes the feature; the remaining work needs product and security policy rather than cleanup closure.

## What This Changes

Requested behavior: after `clawhub install` or `clawhub update`, notify the active agent session to read the installed skill's `SKILL.md` and perform onboarding or setup instructions.

## Best Possible Solution

Define a user-consented install/update handoff contract covering event shape, active-session IPC, disclosure, moderation and hook policy, tests, and docs before implementing any automatic trigger.

## Reproduction Assessment

No as a bug reproduction; this is a new active-session handoff capability rather than a broken documented CLI contract. Yes as source-level verification: current main's install/update paths have no session handoff or system-event injection.

## Solution Assessment

Unclear as proposed. A system event may be the right UX, but the maintainable solution depends on maintainer-approved consent, trust, moderation, and OpenClaw session-handoff policy.
