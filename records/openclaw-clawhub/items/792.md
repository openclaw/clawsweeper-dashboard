## Summary

Keep open. Current main still does not inject a post-install/update event into an active OpenClaw session, and this feature needs maintainer product/security judgment because newly installed or updated skill content would influence an active agent session.

## What This Changes

Request automatic active-session notification after `clawhub install` or `clawhub update` so the agent reads the installed skill's `SKILL.md` and completes onboarding or setup instructions.

## Best Possible Solution

Define the supported install/update boundary explicitly: either add a user-consented OpenClaw session handoff contract with focused tests and docs, or document the inspectable copy-prompt flow as the intended setup path.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection of current main shows install/update write files and metadata, while setup guidance is only available through an inspectable copy prompt.

## Solution Assessment

Unclear. A system event could address the trigger, but it is not clearly the best maintainable solution until maintainers decide the consent, security, and OpenClaw/ClawHub handoff model.
