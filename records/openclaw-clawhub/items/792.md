## Summary

Keep open. Current main still lacks the requested post-install/update active-session event, and ClawHub cleanup policy keeps issues open unless they are PRs already implemented on main; this also needs product and security judgment before implementation.

## What This Changes

Request to inject a post-install/post-update system event into an active OpenClaw session so the agent reads the installed skill's SKILL.md and completes onboarding/setup steps.

## Best Possible Solution

Maintainers should define the install/update handoff contract: add an explicit user-consented OpenClaw session event with focused tests and docs, or keep the inspectable copy-prompt flow as the supported setup path.

## Reproduction Assessment

Yes for the observable gap. Source inspection on current main shows install/update write skill files and metadata but do not read SKILL.md for onboarding or hand off an event to an active OpenClaw session.

## Solution Assessment

Unclear. The proposed event injection addresses the onboarding gap, but it is not clearly the best solution until maintainers define consent, trust, and the OpenClaw/ClawHub handoff boundary; the current copyable prompt flow is the safer accepted path today.
