## Summary

Keep open. Current main still does not inject a post-install/update event into an active OpenClaw session, and the remaining work needs maintainer product/security judgment because it would let newly installed or updated skill content influence a running agent session.

## What This Changes

Request to notify an active OpenClaw session after `clawhub install` or `clawhub update` so the agent reads the installed skill's `SKILL.md` and completes setup or onboarding.

## Best Possible Solution

Define the user-consented OpenClaw session handoff contract, trust checks, tests, docs, and spec intent before wiring install/update to emit any active-session event.

## Reproduction Assessment

Yes. Source inspection of current `cmdInstall`, `cmdUpdate`, and CLI dispatch shows the commands update files and metadata and print status without reading `SKILL.md` into an active session or emitting a session event.

## Solution Assessment

Unclear as proposed. A system event could solve the onboarding gap, but the maintainable path depends on maintainer-approved consent, security, and OpenClaw session-handoff policy.
