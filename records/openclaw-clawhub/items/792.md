## Summary

Keep open. Current main still does not implement post-install/update active-session event injection, and repository policy keeps ClawHub issues open unless they are resolved PRs; the remaining choice also changes the product and security boundary for how installed skill content can influence an active agent session.

## What This Changes

Request that `clawhub install` and `clawhub update` notify an active OpenClaw session to read the installed skill's `SKILL.md` and run setup or onboarding instructions.

## Best Possible Solution

Define the supported install/update handoff contract explicitly: either add a user-consented session handoff with tests and docs, or document the current inspectable prompt-copy flow as the intended setup path.

## Reproduction Assessment

Yes for the current behavior. Source inspection shows install/update write files and metadata without notifying an active OpenClaw session or reading `SKILL.md` onboarding instructions.

## Solution Assessment

Unclear. A system event could address the requested onboarding trigger, but it is not clearly the best maintainable solution until maintainers decide the consent and security model for active-session influence.
