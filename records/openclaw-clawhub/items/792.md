## Summary

Keep open. Current main still installs or updates skill files and local metadata without any active-session system event, and a collaborator has already signaled this should be prioritized, so the remaining work needs maintainer product/security review rather than cleanup closure.

## What This Changes

The issue requests that `clawhub install` and `clawhub update` notify the active agent session to read the installed or updated skill instructions and run onboarding/setup steps.

## Best Possible Solution

Define a user-consented OpenClaw session handoff contract, including event shape, trust checks, tests, and docs, before wiring install/update to emit it.

## Reproduction Assessment

Yes for source-level reproduction: current `cmdInstall` and `cmdUpdate` update files/metadata and emit success messages without active-session IPC or system-event injection. I did not run the CLI because this review is read-only and the report asks for a new handoff capability beyond the current documented CLI contract.

## Solution Assessment

Unclear as proposed. A system event may be the right UX, but the maintainable path depends on maintainer-approved consent, security, and OpenClaw session-handoff policy.
