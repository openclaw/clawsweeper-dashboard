## Summary

Keep open. Current main still has no post-install/update active-session event, and the request is a new security-sensitive product boundary rather than conservative cleanup closure.

## What This Changes

Request that `clawhub install` and `clawhub update` inject a system event into the active OpenClaw session so the agent reads the installed or updated skill instructions.

## Best Possible Solution

Maintainers should define the install/update handoff contract explicitly: either a user-consented active-session event with security policy, tests, and docs, or a documented decision that the inspectable copy-prompt flow remains the supported setup path.

## Reproduction Assessment

Not applicable as a bug reproduction: this requests a new active-session handoff behavior rather than reporting a broken existing contract. Source inspection gives high confidence that current main lacks the requested install/update event injection.

## Solution Assessment

Unclear as proposed. A system event could solve the onboarding gap, but the maintainable path first needs a maintainer-approved consent and trust-boundary design aligned with the current copyable prompt surface and the related hooks security RFC.
