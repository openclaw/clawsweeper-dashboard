## Summary

Keep open. Current main still does not implement post-install/update active-session event injection, and for this repository the cleanup policy keeps issues open for maintainer handling. The request is concrete, but it crosses a product and security boundary because fetched skill content would influence an active OpenClaw session.

## What This Changes

Request post-install/update active-session event injection so OpenClaw reads a newly installed or updated skill's SKILL.md and completes setup guidance.

## Best Possible Solution

Maintainers should define the install/update boundary explicitly: either add a user-consented OpenClaw session handoff contract with focused CLI/session tests and docs, or document the inspectable copy-prompt flow as the supported setup path.

## Reproduction Assessment

Yes. Current-main source inspection reproduces the missing behavior: install/update dispatch directly to cmdInstall/cmdUpdate, and those functions only download, extract, and write metadata before printing status.

## Solution Assessment

Unclear. The requested event injection is a plausible UX, but it is not obviously the best solution until maintainers define consent, trust, and prompt-injection handling for active-session influence.
