## Summary

Keep open. Current main still lacks any post-install/update active-session system event, and a collaborator has explicitly signaled this should be prioritized, but the change needs maintainer product and security judgment before implementation.

## What This Changes

The issue requests that `clawhub install` and `clawhub update` notify the active OpenClaw session to read the installed skill’s `SKILL.md` and run onboarding or setup instructions.

## Best Possible Solution

Define a user-consented OpenClaw session handoff contract, including event shape, trust checks, tests, and docs, before wiring install/update to emit it.

## Reproduction Assessment

Yes. Source inspection shows `cmdInstall` and `cmdUpdate` update files and metadata and emit success/up-to-date messages without active-session IPC or system-event injection.

## Solution Assessment

Unclear as proposed. A system event may be the right UX, but the maintainable path depends on maintainer-approved consent, security, and OpenClaw session-handoff policy.
