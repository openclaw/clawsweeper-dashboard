## Summary

Keep open. Current main still does not inject a post-install/update event into an active OpenClaw session, and this is a feature request that needs product and security review rather than conservative cleanup closure.

## What This Changes

Request to inject a system event into an active OpenClaw session after `clawhub install` or `clawhub update` so the agent reads the installed skill's setup instructions.

## Best Possible Solution

Maintainers should define the install/update handoff contract explicitly: either a user-consented session event with security review, tests, and docs, or a documented decision that inspectable copy-prompt setup remains the supported path.

## Reproduction Assessment

Not applicable as a bug reproduction: this requests a new active-session handoff behavior rather than reporting a broken existing contract. Source inspection gives high confidence that current main lacks the requested install/update event injection.

## Solution Assessment

Unclear as proposed. A system event may be viable UX, but the maintainable path first needs a maintainer-approved consent and trust-boundary design aligned with the existing copyable-prompt install surface and the related hooks security RFC.
