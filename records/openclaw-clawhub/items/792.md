## Summary

Keep open. Current main and the latest v0.23.0 CLI still install or update skill files plus local metadata without injecting an active-session event, and the remaining work needs product and security direction before implementation.

## What This Changes

The issue requests that `clawhub install` and `clawhub update` inject an active-session system event that tells the agent to read the installed skill's `SKILL.md` and run setup or onboarding instructions.

## Best Possible Solution

Define a consent-aware ClawHub/OpenClaw install-update handoff contract with explicit session targeting, inspectable setup guidance, approval before consequential changes, docs, and CLI/session tests.

## Reproduction Assessment

Yes, at source level: current `cmdInstall` and `cmdUpdate` write skill files and metadata and print success without an active-session system event or SKILL.md onboarding read. I did not run the CLI because this review is read-only and the source path is clear.

## Solution Assessment

No, not exactly as proposed. A notification may be appropriate, but unconditional instructions to execute setup from SKILL.md should be replaced by a maintainer-approved consent, session-targeting, trust, and failure-handling contract.
