## Summary

Keep open. Current main still lacks active-session injection after ClawHub CLI install/update, and the request crosses a product/security boundary rather than being a conservative cleanup-close candidate for a ClawHub issue.

## What This Changes

Request a post-install/update system event so an active OpenClaw session reads the installed skill's SKILL.md and completes setup/onboarding work.

## Best Possible Solution

Maintainers should define the install/update boundary explicitly: either add a user-consented OpenClaw session handoff contract with focused tests and docs, or document the inspectable copy-prompt flow as the intended setup path.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection verifies current main writes files/metadata and supports inspectable copy-prompt setup, not active-session injection.

## Solution Assessment

Unclear. A system event could address the requested trigger, but it is not clearly the best maintainable solution until maintainers decide the consent, security, and OpenClaw/ClawHub handoff model.
