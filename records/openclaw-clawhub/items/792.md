## Summary

Keep open. Current main still has no post-install/update active-session event injection, and the ClawHub cleanup policy keeps issues open unless a strong close reason applies; the request is concrete but needs product and security review before implementation.

## What This Changes

The issue requests that `clawhub install` and `clawhub update` notify an active OpenClaw session to read the installed skill's `SKILL.md` and complete setup or update guidance.

## Best Possible Solution

Define the supported install/update boundary explicitly: either add a user-consented OpenClaw session handoff contract with tests and docs, or document the inspectable copy-prompt flow as the intended setup path.

## Reproduction Assessment

No, not as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main installs and updates files/metadata and offers copyable setup prompts, but does not inject an active-session event.

## Solution Assessment

Unclear. A system event could address the requested trigger, but the maintainable solution depends on an explicit consented OpenClaw/ClawHub handoff policy and the related lifecycle hook security contract.
