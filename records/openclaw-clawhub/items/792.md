## Summary

Current main still completes skill installs and updates without notifying an active OpenClaw session. The onboarding goal is valid and has positive collaborator support, but implementation requires product and security decisions because newly installed publisher content could influence privileged agent behavior.

## What This Changes

Emit a post-install/update notification to an active OpenClaw session so the agent can discover and complete skill onboarding.

## Best Possible Solution

Define a typed, authenticated install/update event carrying the skill identity, version, and local path, then let OpenClaw surface or process onboarding under explicit operator-controlled policy rather than treating SKILL.md text as automatically authoritative.

## Reproduction Assessment

Yes. Current main's install and update handlers can be followed through persistence and success output, and neither contains an active-session notification path.

## Solution Assessment

No, not as the proposed imperative free-form system message. A structured event with OpenClaw-owned authentication, session targeting, consent, and execution policy is the safer maintainable solution.
