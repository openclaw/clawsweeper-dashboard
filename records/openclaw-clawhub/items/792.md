## Summary

Keep open. Current main still lacks post-install/update active-session injection, and for ClawHub issues the conservative policy is to leave non-PR feature requests open; this one also needs maintainer product/security judgment because it would let newly installed or updated skill content influence an active OpenClaw session.

## What This Changes

Request that install/update inject a system event into the active agent session so the agent reads SKILL.md setup and onboarding instructions after a skill changes.

## Best Possible Solution

Define and document a user-consented OpenClaw session handoff contract, or explicitly keep the inspectable copy-prompt flow as the supported setup path.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection of current main shows install/update write files and metadata, while setup guidance is only available through an inspectable copy prompt.

## Solution Assessment

Unclear. A system event could address the requested trigger, but it is not clearly the best maintainable solution until maintainers decide the consent, security, and OpenClaw/ClawHub handoff model.
