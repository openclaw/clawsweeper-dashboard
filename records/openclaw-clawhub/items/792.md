## Best Possible Solution

Define the supported install/update boundary first: either add a user-consented active-session handoff contract with security policy, tests, and docs, or keep the inspectable copy-prompt flow as the intended setup path.

## Reproduction Assessment

No as a bug reproduction; this is a new active-session handoff capability rather than broken documented behavior. Source inspection gives high confidence current main does not perform the requested install/update event injection.

## Solution Assessment

Unclear as proposed. A system event could address the onboarding gap, but the best path depends on maintainer-approved consent, trust-boundary, and OpenClaw/ClawHub handoff design.
