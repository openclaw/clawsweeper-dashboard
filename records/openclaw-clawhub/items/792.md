## Summary

Keep open. Current main still does not inject a post-install/update event into an active OpenClaw session, and ClawHub policy keeps issues open for maintainer handling; this request is a concrete feature but crosses a consent and prompt-influence boundary.

## What This Changes

Request that `clawhub install` and `clawhub update` notify an active OpenClaw session to read the installed skill’s `SKILL.md` and perform setup or post-update onboarding.

## Best Possible Solution

Define the supported install/update boundary explicitly: either add a user-consented OpenClaw session handoff contract with focused tests and docs, or document the inspectable copy-prompt flow as the intended setup path.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection shows install/update write files and metadata, while setup guidance is currently available through an inspectable copy prompt rather than active-session injection.

## Solution Assessment

Unclear. A system event could address the trigger, but it is not clearly the best maintainable solution until maintainers decide the consent, security, and OpenClaw/ClawHub handoff model.
