## Summary

Keep open. Current main still performs ClawHub CLI install/update as file and lockfile operations with no active OpenClaw session handoff, and the requested automatic session influence needs product and security judgment rather than cleanup closure.

## What This Changes

Request a post-install/update active-session system event that makes OpenClaw read the installed skill's SKILL.md and run setup or onboarding instructions.

## Best Possible Solution

Define a consented OpenClaw/ClawHub handoff contract with focused tests and docs, or make the current inspectable copy-prompt flow the explicit supported path.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that current main installs or updates files and metadata but does not inject an active-session event.

## Solution Assessment

Unclear. A system event matches the requested trigger, but the maintainable approach depends on an explicit consented OpenClaw/ClawHub handoff policy and the related lifecycle hook security contract.
