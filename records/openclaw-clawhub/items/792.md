## Summary

Keep open. Current main still has no active-session handoff after ClawHub CLI install/update, and this feature needs maintainer product and security judgment rather than conservative cleanup closure.

## What This Changes

Requests a post-install/update event that tells the active OpenClaw session to read the installed skill's SKILL.md and complete onboarding or setup work.

## Best Possible Solution

Define the supported install/update boundary first, then either add a user-consented OpenClaw session handoff with focused CLI/session tests and docs or make the inspectable copy-prompt flow the explicit supported path.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that current main installs or updates files and metadata but does not inject an active-session event.

## Solution Assessment

Unclear. A system event matches the requested trigger, but the maintainable solution depends on an explicit user-consent and OpenClaw/ClawHub handoff policy; the current prompt flow is intentionally inspectable.
