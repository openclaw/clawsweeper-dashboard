## Summary

Keep open. Current main still does not inject a post-install/update event into an active OpenClaw session after ClawHub CLI install/update, and this is a feature/security boundary decision rather than conservative cleanup-closeable work.

## What This Changes

The issue requests a post-install/update system event that makes the active agent session read the installed skill's SKILL.md and perform onboarding or setup steps.

## Best Possible Solution

Define the supported install/update boundary: either add an explicit user-consented OpenClaw session handoff contract with security policy, tests, and docs, or keep the inspectable copy-prompt flow as the intended setup path.

## Reproduction Assessment

Not applicable as a bug reproduction: this asks for a new active-session handoff behavior, not a broken existing contract. Source inspection gives high confidence that current main lacks the requested install/update event injection.

## Solution Assessment

Unclear as proposed. A system event could address the onboarding gap, but the maintainable path needs maintainer-approved consent and trust-boundary design aligned with the existing inspectable prompt surface and the open hooks security RFC.
