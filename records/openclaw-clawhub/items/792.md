## Summary

Keep open. Current main still does not inject a post-install/update event into an active OpenClaw session, and this ClawHub issue needs maintainer product and security judgment rather than cleanup closure.

## What This Changes

Request that `clawhub install` and `clawhub update` notify an active OpenClaw session to read the installed skill's `SKILL.md` and complete setup/onboarding after a skill changes.

## Best Possible Solution

Define the supported install/update boundary explicitly: either add a user-consented OpenClaw session handoff contract with tests and docs, or document the current inspectable copy-prompt flow as the intended setup path.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection shows current install/update paths write files and metadata only, while setup guidance exists through the inspectable OpenClaw prompt flow.

## Solution Assessment

Unclear. A system event could address the requested trigger, but the current approved design intentionally avoids direct handoff, so the best solution depends on maintainer product and security decisions about active-session influence.
