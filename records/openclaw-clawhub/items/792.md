## Summary

Keep open. Current main and the latest v0.20.0 release still install/update skill files and metadata without active-session event injection; a collaborator comment says this should be prioritized, and the remaining work needs product/security policy because it would let newly installed or updated skill content influence an active agent session.

## What This Changes

Request to notify an active OpenClaw session after `clawhub install` or `clawhub update` so the agent reads the installed `SKILL.md` and completes setup/onboarding.

## Best Possible Solution

Define a user-consented install/update handoff contract covering session IPC, event shape, moderation/security gating, and docs, then implement CLI/OpenClaw tests if approved.

## Reproduction Assessment

Yes as source-level verification: current `cmdInstall` and `cmdUpdate` have no active-session handoff or system-event injection. This is not a broken documented CLI contract reproduction; it is a missing new capability.

## Solution Assessment

Unclear as proposed. A system event may be the right UX, but the maintainable solution depends on maintainer-approved consent, security, and OpenClaw session-handoff policy.
