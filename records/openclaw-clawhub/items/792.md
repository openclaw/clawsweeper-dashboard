## Summary

Keep open. Current main and the latest v0.23.0 CLI release still install or update skill files and local metadata without injecting an active-session event, and the remaining decision is product/security policy rather than conservative cleanup.

## What This Changes

The issue requests a post-install/update active-session system event after `clawhub install` or `clawhub update` so the agent reads the installed skill's SKILL.md setup guidance.

## Best Possible Solution

Define a consent-aware ClawHub/OpenClaw install-update handoff contract with explicit session targeting, inspectable setup guidance, approval before consequential changes, docs, and CLI/session tests.

## Reproduction Assessment

Yes, at source level. Current `cmdInstall` and `cmdUpdate` update skill files and metadata and print success without an active-session system event or SKILL.md onboarding read; I did not run a live OpenClaw gateway reproduction.

## Solution Assessment

No, not exactly as proposed. A notification may be appropriate, but unconditional instructions to execute setup from SKILL.md should be replaced by a maintainer-approved consent, session-targeting, trust, and failure-handling contract.
