## Summary

Keep open. Current main still does not inject a post-install/update event into an active OpenClaw session, and this ClawHub issue needs maintainer product/security judgment rather than conservative cleanup.

## What This Changes

After `clawhub install` or `clawhub update`, inject a system event into the active session so OpenClaw reads the installed skill's `SKILL.md` and handles onboarding/setup instructions.

## Best Possible Solution

Define the supported install/update handoff contract explicitly: either add a user-consented OpenClaw session handoff with tests and docs, or document the inspectable copy-prompt flow as the intended setup path.

## Reproduction Assessment

Yes for the current behavior. Source inspection shows install/update write files and metadata without notifying an active OpenClaw session or reading `SKILL.md` onboarding instructions.

## Solution Assessment

Unclear. A system event could address the requested onboarding trigger, but it is not clearly the best maintainable solution until maintainers decide the consent, trust, and OpenClaw/ClawHub handoff model.
