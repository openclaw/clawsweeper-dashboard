## Summary

Keep open. Current main and the latest CLI release still do not inject a post-install/update event into an active OpenClaw session; the request is concrete, but it needs maintainer product and security decisions before implementation.

## What This Changes

Request to notify an active agent session after `clawhub install` or `clawhub update` so the agent reads the installed skill's `SKILL.md` and performs setup guidance.

## Best Possible Solution

Define a consent-aware install/update handoff contract that emits structured metadata, safely selects or declines a target session, and requires approval before consequential setup actions run.

## Reproduction Assessment

Yes, at source level. Current `cmdInstall` and `cmdUpdate` complete by writing local files and metadata plus terminal status, with no gateway/session notification path; I did not run a live OpenClaw gateway reproduction.

## Solution Assessment

No, not exactly as proposed. A structured notification may be useful, but instructing the agent to execute any setup from `SKILL.md` should be replaced with a consent-aware, scoped handoff design.

## Security Review

Status: needs_attention

Summary: The requested active-session handoff is security-sensitive because installed skill text could influence agent actions after install/update.

Concerns:

- **[high] Prevent installed instructions from becoming implicit execution:**
  - body: A newly installed or updated skill controls its `SKILL.md`, so automatically telling an active agent to execute setup instructions could trigger commands, configuration edits, plugin activation, or restarts without a distinct user decision.
  - confidence: 0.93
- **[medium] Authenticate and scope session delivery:**
  - body: Any event mechanism must define which OpenClaw gateway and session can receive it and prevent unrelated local processes or sessions from injecting trusted-looking system context.
  - confidence: 0.86
