## What This Changes

Add an active-session system event after skill installation or update so the agent reads and follows the installed skill’s onboarding instructions.

## Best Possible Solution

Define a typed, host-controlled install/update notification that identifies the installed skill and available onboarding guidance, while requiring explicit operator approval before privileged skill-authored actions execute.

## Reproduction Assessment

Yes, at source level. Current main’s install and update paths end with local file and metadata changes plus CLI output, with no active-session event, callback, or onboarding result.

## Solution Assessment

No, not exactly as proposed. A post-install handoff is appropriate, but an unconditional instruction to execute onboarding steps should be replaced by a typed host-controlled notification with explicit consent for privileged actions.
