## Summary

Keep open. Current main still does not inject a post-install/update event into an active OpenClaw session, and ClawHub cleanup policy keeps ordinary issues open; the requested feature is concrete but needs product/security approval because it changes the consent and trust boundary for newly installed or updated skill content.

## What This Changes

Request to inject a post-install/update system event into the active OpenClaw session so the agent reads the installed or updated skill's SKILL.md and performs setup/onboarding work.

## Best Possible Solution

Define the supported install/update boundary explicitly: either add a user-consented OpenClaw session handoff contract with tests and docs, or document the current inspectable copy-prompt flow as the intended setup path.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that current main writes skill files/metadata and offers inspectable copy prompts, but does not inject an active-session event.

## Solution Assessment

Unclear. A system event could solve the requested trigger, but the current approved design intentionally uses copyable prompts, so the best solution depends on maintainer product/security decisions about consent and active-session influence.
