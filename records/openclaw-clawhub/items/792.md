## Summary

Keep open. Current main still installs and updates skill files plus metadata without injecting an active-session system event, and a collaborator explicitly said this feature should be prioritized; the remaining work needs maintainer product and security decisions.

## What This Changes

The issue requests an install/update handoff that notifies the active agent session to read the newly installed skill's SKILL.md and run setup/onboarding steps.

## Best Possible Solution

Define a consent-aware ClawHub/OpenClaw install-update handoff contract with explicit session targeting, inspectable setup guidance, approval before consequential changes, docs, and CLI/session tests.

## Reproduction Assessment

Yes, at source level. Current cmdInstall and cmdUpdate write skill files and metadata and print success without an active-session system event or SKILL.md onboarding read; I did not run a live OpenClaw gateway/session reproduction.

## Solution Assessment

No, not exactly as proposed. A notification may be appropriate, but unconditional instructions to execute setup from SKILL.md should be replaced by a maintainer-approved consent, session-targeting, trust, and failure-handling contract.
