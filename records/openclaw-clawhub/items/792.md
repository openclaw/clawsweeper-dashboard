## Summary

Keep open. Current main still does not inject a post-install/update event into an active OpenClaw session, and ClawHub cleanup policy keeps issues open; the requested feature is concrete but crosses a product and security boundary around newly installed or updated skill content influencing an agent.

## What This Changes

Request to notify an active OpenClaw session after ClawHub skill install/update so the agent reads SKILL.md and performs onboarding or setup instructions.

## Best Possible Solution

Maintainers should define the supported install/update boundary explicitly: add a user-consented OpenClaw session handoff contract with focused tests and docs, or reaffirm the inspectable copy-prompt flow as the intended setup path.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that current main writes skill files/metadata and offers copyable setup prompts, but does not inject an active-session event.

## Solution Assessment

Unclear. A system event could solve the requested trigger, but the current approved design intentionally avoids direct handoff, so the best path depends on maintainer product/security decisions about consent and active-session influence.
