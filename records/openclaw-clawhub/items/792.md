## Risks / Open Questions

- Automatic active-session injection would let newly installed or updated skill content influence an already-running agent, so consent, moderation state, prompt-injection handling, and disclosure rules need maintainer approval.
- The approved install-surface spec currently excludes direct OpenClaw handoff, so implementing this would intentionally revise an existing product boundary.
- Related hook/security work remains open at https://github.com/openclaw/clawhub/issues/707 and https://github.com/openclaw/clawhub/issues/2295; this feature should not define conflicting lifecycle automation rules.
- The ClawHub CLI does not currently expose an active OpenClaw session IPC or handoff surface, so the fix likely needs an explicit cross-project contract rather than a narrow CLI-only patch.
