## Summary

Keep open. Current main still does not inject a post-install/update event into an active OpenClaw session after ClawHub CLI install/update, and ClawHub cleanup policy keeps issues open unless they are resolved PRs; the requested handoff also needs product/security approval.

## What This Changes

After `clawhub install` or `clawhub update`, notify the active OpenClaw session so it reads the installed skill’s `SKILL.md` and performs onboarding/setup.

## Best Possible Solution

Define an explicit, user-consented OpenClaw handoff contract, or keep the existing inspectable copy-prompt flow as the supported setup path and document that boundary.

## Reproduction Assessment

Yes for the current behavior. Source inspection shows install/update dispatch directly to `cmdInstall`/`cmdUpdate`, which write files and lock metadata without a session notification or `SKILL.md` onboarding read.

## Solution Assessment

Unclear. A session event would address the onboarding trigger, but the current approved design explicitly excludes direct handoff and automatic execution, so maintainers should decide consent and trust boundaries first.
