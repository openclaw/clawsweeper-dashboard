## Summary

Keep open. Current main still does not implement post-install/update active-session event injection, and the ClawHub cleanup policy keeps issues open for maintainer handling. The requested behavior is concrete, but it crosses a product and security boundary because newly installed or updated skill content would influence an active agent session.

## What This Changes

The issue requests a post-install/update handoff so an active OpenClaw session reads a newly installed or updated skill's SKILL.md and performs setup or onboarding instructions.

## Best Possible Solution

Maintainers should define the install/update boundary explicitly: either add a user-consented OpenClaw session handoff contract with focused CLI/session tests and docs, or document the current inspectable copy-prompt flow as the supported setup path.
