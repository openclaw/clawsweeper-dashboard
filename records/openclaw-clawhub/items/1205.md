## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has the reported Convex failure shape in the owner-publisher digest rebuild path: publisher writes schedule package and skill digest internal mutations, but each helper can still call `.paginate()` repeatedly within one invocation and neither scheduled mutation accepts continuation state.

## What This Changes

The issue reports that `clawhub publish`/sync can fail with Convex's single-paginated-query limit when publisher-triggered skill search digest sync runs during publishing.

## Best Possible Solution

Implement one-page-per-invocation owner-publisher package and skill digest rebuilds with cursor continuation scheduling, and cover multi-page rebuild behavior in focused Convex unit tests.
