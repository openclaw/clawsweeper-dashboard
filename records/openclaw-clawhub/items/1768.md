## Summary

Keep open. Current main still lacks native per-skill pinning, and open PR #1806 is the active unmerged implementation candidate with closing syntax for this issue, so the maintainer path is to review or land that PR rather than close this issue or queue duplicate automation.

## What This Changes

Request native `clawhub pin`/`unpin` and lockfile metadata so `update`, `update --force`, and `update --all` respect operator-pinned skills.

## Best Possible Solution

Land #1806 or an equivalent maintainer-approved patch that adds pin/unpin commands, persists optional pin metadata, shows pinned state in list output, blocks direct pinned updates, skips and reports pinned entries for update --all, honors pins under --force, and updates docs/tests.

## Reproduction Assessment

Not applicable as a feature request rather than a broken documented behavior. Source inspection does confirm the requested capability is absent on current main: no pin/unpin commands, no lockfile pin fields, and no update-time pin guard.

## Solution Assessment

Yes, the active PR or an equivalent native CLI/lockfile implementation is the right solution shape because wrapper enforcement cannot reliably cover crons and multiple agents. The issue should remain open until that implementation is reviewed and merged.
