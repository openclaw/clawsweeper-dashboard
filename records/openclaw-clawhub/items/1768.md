## Summary

Keep open. Current main still lacks native per-skill pinning, and PR #1806 is the open unmerged implementation candidate with closing syntax, so the maintainer path is to review or land that PR rather than close this issue or queue duplicate work.

## What This Changes

The issue requests native CLI pin/unpin support and lockfile metadata so pinned installed skills cannot be overwritten by normal, forced, or all-skill updates.

## Best Possible Solution

Land PR #1806 or an equivalent maintainer-approved native CLI implementation that stores pin metadata, shows pinned state, blocks direct pinned updates, skips pinned entries for update --all, and honors pins under --force.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken documented behavior. Source inspection confirms the requested native pin/unpin commands, lockfile pin fields, and update-time pin guard are absent on current main.

## Solution Assessment

Yes, native CLI and lockfile enforcement is the right solution shape because wrapper-only enforcement cannot reliably cover crons, scripts, and multiple agents. The issue should stay open until PR #1806 or an equivalent implementation is merged.
