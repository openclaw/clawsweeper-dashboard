## Summary

Keep open. Current main still lacks native per-skill pinning, and the provided GitHub context shows open PR #1806 is already the active unmerged implementation candidate with closing syntax for this issue, so this should wait for maintainer review of that PR rather than be closed or duplicated.

## What This Changes

The issue requests native CLI per-skill pin/unpin support that stores pin metadata, shows pinned state in list output, blocks direct pinned-skill updates, and skips pinned skills during update --all and update --force flows.

## Best Possible Solution

Land #1806 or an equivalent maintainer-approved patch that adds pin/unpin commands, persists optional pin metadata, shows pinned state in list output, rejects direct updates for pinned skills, skips and reports pinned entries for update --all, honors pins under --force, and updates the focused CLI/schema tests and docs.
