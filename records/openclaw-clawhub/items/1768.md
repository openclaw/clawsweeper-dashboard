## Summary

Keep open. Current main still lacks native per-skill pinning, and the linked implementation PR #1806 is open and unmerged, so this issue should stay open until that PR or an equivalent implementation lands.

## What This Changes

Request native CLI pin/unpin support and lockfile pin metadata so pinned skills are shown in list, blocked for direct update, and skipped by update --all/--force.

## Best Possible Solution

Land PR #1806 or an equivalent maintainer-approved CLI implementation that persists pin metadata, surfaces pinned state, and honors pins for direct, forced, and all-skill updates.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken documented behavior. Source inspection confirms current main lacks native pin/unpin commands, lockfile pin fields, and update-time pin enforcement.

## Solution Assessment

Yes; native CLI and lockfile enforcement is the right solution shape because wrapper-only enforcement cannot reliably cover crons, scripts, and multiple agents. The issue should stay open while PR #1806 or an equivalent implementation is reviewed.
