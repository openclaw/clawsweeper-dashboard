## Summary

Keep open. Current main still lacks native per-skill pinning, and PR #1806 is an open, unmerged implementation candidate with closing syntax, so this issue should stay open until that PR or an equivalent implementation lands.

## What This Changes

Add native CLI support for per-skill pinning so pin, unpin, list, and update honor lockfile pin state, including under --force and --all.

## Best Possible Solution

Review and land PR #1806, or an equivalent maintainer-approved implementation that stores pin metadata, exposes pin/unpin, shows pinned state, blocks direct pinned updates, and skips pinned skills during bulk updates.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken documented behavior. Source inspection confirms the requested native pin/unpin commands, lockfile pin fields, and update-time pin guard are absent on current main.

## Solution Assessment

Yes; native CLI and lockfile enforcement is the maintainable solution shape because wrapper-only enforcement cannot reliably cover crons, scripts, and multiple agents. The narrow path is to review and land PR #1806 or an equivalent implementation, not queue duplicate work.
