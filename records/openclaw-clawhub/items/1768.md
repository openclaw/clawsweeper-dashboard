## Summary

Keep open. Current main still lacks native per-skill pinning, and open PR #1806 is the unmerged implementation candidate with closing syntax, so this issue should remain open until that PR or an equivalent implementation lands.

## What This Changes

Request native per-skill CLI pinning so `clawhub update`, `--force`, and `--all` cannot overwrite operator-frozen installed skills.

## Best Possible Solution

Land PR #1806 or an equivalent maintainer-approved native CLI implementation that stores pin metadata, shows pinned state, blocks direct pinned updates, skips pinned entries for `update --all`, and honors pins under `--force`.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken documented behavior. Source inspection shows current main has no native pin/unpin command, no lockfile pin metadata, and no update-time pin guard.

## Solution Assessment

Yes; native CLI and lockfile enforcement is the right solution shape because wrapper-only enforcement cannot reliably cover crons, scripts, and multiple agents. The issue should stay open until PR #1806 or an equivalent implementation is merged.
