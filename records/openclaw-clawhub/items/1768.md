## Summary

Keep open: current main still lacks native per-skill pinning, and #1806 is already the open unmerged implementation candidate linked to this issue, so cleanup should wait for maintainer review or merge rather than closing or queueing duplicate automation.

## What This Changes

Requests native per-skill pin/unpin support in the ClawHub CLI so update, update --force, and update --all honor pinned skills and list/lockfile state exposes pins.

## Best Possible Solution

Land #1806 or an equivalent maintainer-approved change that persists pin metadata, adds pin/unpin, shows pins in list output, blocks direct and forced pinned updates, and skips/reports pinned entries for update --all.

## Reproduction Assessment

Yes. Static inspection gives a high-confidence reproduction path: current main has no pin metadata schema or commands, and cmdUpdate can overwrite requested or all lockfile entries without any pin check.

## Solution Assessment

Yes. Native lockfile-backed CLI pinning is the maintainable solution shape because it protects every caller using clawhub directly; the exact implementation in #1806 still needs maintainer review.
