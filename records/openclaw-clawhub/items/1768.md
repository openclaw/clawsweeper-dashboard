## Summary

Keep open. Current main still lacks native per-skill pinning, and PR #1806 is an open, unmerged implementation candidate that explicitly references this issue, so the maintainer path is to review or land that PR rather than close this issue or queue duplicate work.

## What This Changes

The issue requests native CLI support to pin installed skills so update, update --force, and update --all cannot overwrite explicitly frozen skill bytes.

## Best Possible Solution

Land PR #1806 or an equivalent maintainer-approved native CLI implementation that persists pin metadata, shows pinned state, blocks direct pinned updates, skips pinned entries for update --all, and honors pins under --force.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken documented behavior. Source inspection of current main confirms the requested native pin/unpin commands, lockfile pin fields, list indicator, and update-time pin guard are absent.

## Solution Assessment

Yes, native CLI and lockfile enforcement is the right solution shape because wrapper-only enforcement cannot reliably cover crons, scripts, and multiple agents. The issue should stay open until PR #1806 or an equivalent implementation is merged.
