## Summary

Keep open. Current main still lacks native per-skill pinning, and the linked implementation PR remains the right place to finish this rather than closing the issue or queueing duplicate work.

## What This Changes

Request native CLI pin/unpin support so pinned installed skills cannot be overwritten by direct, forced, or all-skill updates.

## Best Possible Solution

Review and land https://github.com/openclaw/clawhub/pull/1806 or an equivalent maintainer-approved implementation that stores pin metadata, blocks direct pinned updates, skips pinned entries for update --all, and honors pins under --force.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken documented behavior. Source inspection gives high confidence that the requested native pinning surface is absent on current main.

## Solution Assessment

Yes, native CLI and lockfile enforcement is the right solution shape because wrapper-only enforcement cannot cover every cron, script, shell, or agent using clawhub directly. The existing linked PR is the natural implementation path to review first.
