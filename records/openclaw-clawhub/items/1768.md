## Summary

Keep open. Current main still lacks native per-skill pinning, and #1806 is already the open unmerged implementation candidate with closing syntax, so the right path is maintainer review of that PR rather than closing this issue or queueing duplicate automation.

## What This Changes

Request native CLI pin/unpin support that persists per-skill pin metadata, surfaces it in list output, and prevents forced/all updates from touching pinned skills.

## Best Possible Solution

Land #1806 or an equivalent maintainer-approved patch that adds pin/unpin commands, persisted pin metadata, list indicators, update enforcement for direct/all/force paths, and focused CLI/schema/docs coverage.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms current main has no pin/unpin commands or lockfile pin metadata, and cmdUpdate can still remove/reinstall targets without a pin check.

## Solution Assessment

Yes, native CLI and lockfile enforcement is the right product direction because an external wrapper cannot reliably cover all callers. The concrete implementation details should be resolved through maintainer review of #1806 or an equivalent patch.
