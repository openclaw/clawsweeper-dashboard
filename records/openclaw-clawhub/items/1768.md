## Summary

Keep open. Current main still lacks native pin/unpin support, lockfile pin metadata, and update-time pin enforcement, and PR #1806 is the open unmerged implementation candidate with closing syntax.

## What This Changes

Request native per-skill CLI pinning so update, --force, and --all cannot overwrite operator-frozen installed skills.

## Best Possible Solution

Land PR #1806 or an equivalent maintainer-approved CLI/schema/docs/test implementation that stores pin metadata, shows pinned state, blocks direct pinned updates, and skips pins during --all/--force.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken documented behavior. Source inspection on current main confirms the native pin/unpin commands, pin metadata, and update guard are absent.

## Solution Assessment

Yes, native CLI and lockfile enforcement is the right solution shape because wrapper-only enforcement cannot reliably cover all callers. The implementation should proceed through PR #1806 or equivalent maintainer review.
