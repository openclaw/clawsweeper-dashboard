## Summary

Keep this PR open for maintainer handling: it is a member-authored security hardening PR, but the branch is conflicted with current main and still uses a broad dev-runtime gate that can allow loopback token delivery in public non-production runtimes.

## What This Changes

The branch gates docs-auth loopback return origins behind `isDevRuntime()`, removes localhost origins from the Vercel `form-action` CSP, and adds Vitest coverage for dev versus production return URLs.

## Best Possible Solution

Rebase or replace this with a focused security PR that preserves current main's origin-coupled allowlist, removes production CSP loopback allowance deliberately, records the invariant in `specs/`, and includes redacted runtime/network proof.
