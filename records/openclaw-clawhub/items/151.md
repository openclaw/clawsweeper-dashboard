## Summary

Keep open. The report is concrete and plausibly valid, but current ClawHub `main` is a registry/catalog app and does not contain the Anthropic/OpenClaw runtime transcript serializer needed to verify or fix this bug here; the target policy also keeps ClawHub issues open.

## What This Changes

Report Anthropic 400s when one assistant turn emits multiple tool calls but persisted tool results are split into separate messages instead of the immediately following combined message.

## Best Possible Solution

Route the report to the OpenClaw runtime/provider tracker and fix Anthropic transcript conversion there so all tool results for one assistant tool-use turn are grouped into the immediately following message, with regression coverage for the reported JSONL shape.

## Reproduction Assessment

No. The issue provides a plausible minimal repro and transcript, but the current ClawHub tree does not contain the OpenClaw runtime/provider serializer needed for a high-confidence current-main reproduction.

## Solution Assessment

Yes for the reported behavior: grouping all tool results into one immediately following message is the narrow protocol-compatible fix, while forcing one tool call is only a fallback. In this repository, the maintainable path is routing rather than changing ClawHub catalog code.
