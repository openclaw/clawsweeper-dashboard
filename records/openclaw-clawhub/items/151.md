The issue asks OpenClaw to keep Anthropic tool_result blocks for multiple tool_use calls in the immediately following message instead of persisting each result as separate messages.

## Best Possible Solution

Fix this in the OpenClaw runtime/provider serialization path so all tool results for one assistant tool-use turn are grouped into the immediately following message, with a regression test based on the reported JSONL shape.
