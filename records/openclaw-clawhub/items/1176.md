## What This Changes

The issue reports that `clawhub publish` and `clawhub sync --all` fail during preparation because the publish path triggers Convex's single-paginated-query limit in owner-publisher digest sync.

## Best Possible Solution

Keep the cursor-continuation digest implementation, verify production publish behavior if maintainers want account-specific confirmation, then consolidate the related publish-pagination issues together.

## Reproduction Assessment

No. The original report has concrete commands and stack evidence, but current main no longer has the repeated owner-publisher pagination loop; I did not run a live publish as `hubtiger123`.

## Solution Assessment

Yes. The current one-page-per-internal-mutation continuation approach preserves indexed pagination and avoids both Convex's multiple-paginated-query limit and the unbounded `collect()` approach from https://github.com/openclaw/clawhub/pull/1183.
