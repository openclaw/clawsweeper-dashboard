## Summary

Keep open. ClawHub cleanup policy keeps issues open in this lane, and current main still contains the reported Convex failure shape: publish/sync reaches the personal-publisher path, and the owner-publisher package and skill digest rebuild helpers can still call `.paginate()` repeatedly inside a single internal mutation. The linked fix PR #1183 is closed and unmerged, and its `collect()` approach would avoid this specific error by taking on unbounded read risk.

## What This Changes

The issue reports that `clawhub publish` and `clawhub sync --all` fail for a user with Convex's multiple-paginated-queries error during owner-publisher skill digest sync.

## Best Possible Solution

Implement bounded, cursor-rescheduled owner-publisher package and skill digest rebuilds so each internal mutation processes at most one indexed page, preserves digest semantics, and has regression coverage for publishers with more than one page of packages or skills.
