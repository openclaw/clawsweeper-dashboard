## Summary

The reported publish failure is real on current main, and this focused patch safely restores the affected workflow by reconciling only bounded duplicate rows that encode the same GitHub identity. Conflicting, malformed, or excessive bindings still fail closed, the authorization invariant is documented, the integrated local Convex/CLI proof is convincing, and the linked issue should remain open until this PR merges.

## What This Changes

The PR changes GitHub auth identity lookup to accept a bounded set of duplicate rows only when every row has the same string `providerAccountId`, while rejecting conflicts or overflow and updating tests and the auth-identity specification.

## Best Possible Solution

Merge the bounded identical-identity reconciliation, then separately investigate duplicate creation in device authorization and provide an operator-safe cleanup path for existing ambiguous or excessive rows.

## Reproduction Assessment

Yes, with high confidence from source and supplied runtime evidence: current main's `.unique()` query throws whenever the affected user has duplicate indexed GitHub auth rows, matching the linked production error. The contributor also exercised the corrected behavior through real local Convex and CLI publishing.

## Solution Assessment

Yes. Comparing every row in a small bounded window against the immutable GitHub provider account ID is narrower and safer than selecting a newest or arbitrary row, while malformed, conflicting, and excessive bindings remain fail-closed.
