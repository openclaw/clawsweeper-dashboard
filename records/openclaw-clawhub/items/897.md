## Summary

Keep open: current main still excludes `.ps1` from the skill file allowlist across upload, publish, archive/import, and CLI paths, and the request needs maintainer/security approval before adding a new executable script format.

## What This Changes

Allow PowerShell `.ps1` files to be uploaded and published as part of ClawHub skills.

## Best Possible Solution

Maintainers should decide the PowerShell file policy; if approved, add `ps1` consistently to the shared schema and CLI allowlists with content-type normalization, docs, and focused tests, and if rejected, document the intentional exclusion and workaround.

## Reproduction Assessment

Yes. Current main can be checked directly: `.ps1` is absent from `TEXT_FILE_EXTENSION_SET`, so a PowerShell file with an empty or non-text MIME type fails browser-side `isTextFile` and server-side publish validation.

## Solution Assessment

Unclear until maintainer/security approval. The mechanical implementation would be narrow, but the policy decision to distribute PowerShell scripts through skill bundles should come first.
