## Summary

Keep open. Current main still excludes `.ps1` from the shared and CLI text-file allowlists, and allowing PowerShell files changes the executable script formats accepted in published skill bundles, so this needs maintainer/security approval before implementation.

## What This Changes

The issue requests allowing `.ps1` PowerShell files as supporting files in ClawHub skill uploads and publishes.

## Best Possible Solution

Maintainers should decide the bundle policy first; if approved, add `ps1` consistently to shared schema and CLI allowlists, normalize its content type, update docs and focused tests, and verify browser upload, archive/drop, GitHub import, Convex publish, raw serving, and CLI publish/discovery paths together.

## Reproduction Assessment

Yes. Static inspection gives a high-confidence reproduction path: a `.ps1` file with an empty or non-text browser MIME type fails the browser allowlist, archive/import/CLI extension filters skip it, and server publish validation rejects it unless it is treated as text.

## Solution Assessment

Unclear until maintainer/security approval. The technical implementation is likely a narrow allowlist and test/docs update, but accepting another executable script format is a product/security policy decision.
