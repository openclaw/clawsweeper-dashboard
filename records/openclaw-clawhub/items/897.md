## Summary

Keep open: current main still does not intentionally accept `.ps1` skill files, the related open report at https://github.com/openclaw/clawhub/issues/290 covers the same PowerShell support request, and this ClawHub issue needs maintainer product/security review before expanding the publishable script surface.

## What This Changes

Request to allow PowerShell `.ps1` files in ClawHub skill uploads instead of forcing authors to embed scripts in `SKILL.md`.

## Best Possible Solution

Maintainers should decide the PowerShell file policy; if approved, add `.ps1` consistently across shared and CLI allowlists, MIME normalization, browser/archive/GitHub/API/Convex validation, raw file serving, moderation expectations, tests, docs, and security-sensitive specs.

## Reproduction Assessment

Yes. Source inspection on current main shows `.ps1` is absent from `TEXT_FILE_EXTENSION_SET`, so a `.ps1` file with empty or generic MIME metadata fails the browser and server text-file gates.

## Solution Assessment

Unclear until maintainers make the product/security policy call. If approved, the maintainable solution is a coordinated allowlist, MIME, upload/import/API/CLI, moderation, docs, and test update rather than a publish-form-only change.
