## Summary

Keep open: current main still does not intentionally support `.ps1` skill files, and accepting PowerShell would expand the executable script formats allowed in published skill bundles, so it needs maintainer/security policy review.

## What This Changes

The issue asks ClawHub to allow `.ps1` PowerShell files in skill uploads and published bundles instead of requiring script content to be embedded in `SKILL.md`.

## Best Possible Solution

Maintainers should decide the PowerShell file policy; if approved, add `ps1` consistently to shared and CLI allowlists with content-type normalization, focused tests, and docs, and if rejected, document the intentional exclusion and workaround.

## Reproduction Assessment

Yes. Source inspection on current main shows `.ps1` is absent from the extension allowlists, so a PowerShell file with empty or non-text MIME type fails browser-side and server-side text-file checks.

## Solution Assessment

Unclear until maintainer/security policy is decided. The mechanical patch is narrow, but the durable solution depends on whether ClawHub wants PowerShell scripts in accepted published skill bundles.
