## Summary

Keep open. Current main still does not intentionally support `.ps1` skill files by extension, the report has concrete user impact with additional confirmations, and related #290 remains open. Because accepting PowerShell scripts would expand the executable file surface for published skill bundles, this needs maintainer product/security approval before an automated allowlist change.

## What This Changes

Request support for uploading PowerShell `.ps1` files as part of ClawHub skill bundles instead of embedding the script text in `SKILL.md`.

## Best Possible Solution

Maintainers should first decide whether PowerShell belongs in the accepted skill-file surface. If approved, add `ps1` consistently to the shared schema allowlist and CLI mirror, normalize it to an appropriate text content type, update focused tests and docs, and verify browser upload, archive/drop handling, GitHub import, Convex publish, raw file serving, and CLI publish/discovery together; if rejected, document the intentional exclusion and recommended workaround.
