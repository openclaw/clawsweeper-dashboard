## Summary

Keep open: current main still excludes `.ps1` from the shared and CLI text-file allowlists, and accepting PowerShell would expand the executable script formats allowed in published skill bundles, so it needs maintainer/security policy review alongside related #290.

## What This Changes

Allow `.ps1` PowerShell scripts to be uploaded and published as files in ClawHub skill bundles.

## Best Possible Solution

Maintainers should decide the PowerShell file policy; if approved, add `ps1` consistently to shared and CLI allowlists with MIME normalization, docs, and focused tests, otherwise document the intentional exclusion and workaround.

## Reproduction Assessment

Yes. Source inspection on current main shows `.ps1` is absent from the shared and CLI extension allowlists, so a `.ps1` file with empty or non-text MIME is rejected by browser and server text-file checks.

## Solution Assessment

Unclear until maintainer/security policy is decided. The mechanical patch is narrow, but the maintainable solution depends on whether ClawHub wants PowerShell scripts in accepted published skill bundles.
