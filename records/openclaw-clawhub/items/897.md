## Summary

Keep open: current main still does not intentionally accept `.ps1` skill files, the related report remains open, and ClawHub cleanup policy keeps issues open. The request is concrete, but adding PowerShell expands the publishable executable script surface and needs maintainer product/security approval before an allowlist change.

## What This Changes

Allow `.ps1` PowerShell script files to be uploaded and published as supporting files in ClawHub skill bundles.

## Best Possible Solution

Maintainers should decide the PowerShell file policy; if approved, add `ps1` consistently across shared and CLI allowlists, MIME normalization, browser/archive/GitHub/API/Convex validation, moderation expectations, tests, and docs.

## Reproduction Assessment

Yes. Source inspection on current main shows `.ps1` is not in `TEXT_FILE_EXTENSION_SET`; a `.ps1` file with empty or generic MIME metadata fails the browser `isTextFile` gate, publish-route validation, and Convex publish validation.

## Solution Assessment

Unclear until maintainers make the product/security policy call. If approved, the maintainable fix is a coordinated allowlist, MIME, upload/import/API/CLI, moderation, docs, and test update rather than a publish-form-only change.
