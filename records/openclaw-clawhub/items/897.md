## Summary

Keep open: current main still does not intentionally support `.ps1` skill files, and for ClawHub issues this cleanup flow should not auto-close even when related #290 tracks the same PowerShell request. The remaining decision is product/security policy for adding another executable script format to published skill bundles.

## What This Changes

The issue asks ClawHub to accept PowerShell `.ps1` files as skill bundle files instead of requiring the script to be embedded in `SKILL.md`.

## Best Possible Solution

Maintainers should decide the PowerShell file policy; if approved, add `ps1` consistently to shared and CLI allowlists with content-type normalization, docs, and focused coverage, and if rejected, document the intentional exclusion and workaround.

## Reproduction Assessment

Yes. Current main source shows `.ps1` absent from `TEXT_FILE_EXTENSION_SET`, so a PowerShell file with an empty or non-text MIME type fails browser-side and server-side text-file validation, and the CLI discovery path excludes it by extension.

## Solution Assessment

Unclear until maintainer/security approval. The code change would be narrow if approved, but the policy decision to distribute PowerShell scripts through skill bundles should come first.
