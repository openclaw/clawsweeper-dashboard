## Summary

Keep open: current main still excludes `.ps1` from the shared skill-file allowlists and publish/upload validation, and accepting PowerShell would expand the executable script surface for published skill bundles.

## What This Changes

Request to allow `.ps1` PowerShell scripts as files in ClawHub skill uploads instead of requiring the script to be embedded in `SKILL.md`.

## Best Possible Solution

If maintainers approve PowerShell support, add `ps1` consistently to the shared schema and CLI allowlists, normalize it to a text content type, update docs and focused tests, and otherwise document the intentional exclusion and workaround.

## Reproduction Assessment

Yes. Source inspection on current main shows `.ps1` is absent from `TEXT_FILE_EXTENSION_SET`, so a `script.ps1` file with empty or non-text MIME type fails browser-side `isTextFile` and server-side publish validation.

## Solution Assessment

Unclear until maintainer/security approval. The implementation would likely be narrow, but the policy decision to distribute PowerShell scripts through skill bundles should come first.
