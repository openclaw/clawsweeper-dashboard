## Summary

Keep open. Current main still does not intentionally support `.ps1` skill files, and the request is concrete with reporter context, additional confirmations, and related #290 still open; accepting PowerShell would expand the executable skill-bundle surface and needs maintainer/security approval before an allowlist change.

## What This Changes

Allow `.ps1` PowerShell scripts to be uploaded and published as part of ClawHub skill bundles.

## Best Possible Solution

Maintainers should first decide the PowerShell policy; if approved, add `ps1` consistently to the shared schema and CLI-mirrored allowlists, normalize it as text, update docs and focused tests, and verify browser upload, archive/drop handling, GitHub import, Convex publish, raw file serving, and CLI publish/discovery together.

## Reproduction Assessment

Yes. The current allowlists omit `ps1`, and the browser, archive/import, CLI, and server publish paths all gate acceptance through text MIME checks or `TEXT_FILE_EXTENSION_SET`, so a `.ps1` file with empty or non-text MIME is rejected or skipped.

## Solution Assessment

Unclear until maintainers approve the content policy. If approved, a narrow allowlist/mirror/docs/tests update is the maintainable fix; if rejected, the docs should explicitly call out the exclusion and workaround.
