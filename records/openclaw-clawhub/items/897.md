Keep open: current main still does not intentionally accept `.ps1` skill files, the related report at https://github.com/openclaw/clawhub/issues/290 remains open, and ClawHub cleanup policy keeps issues open. The request is concrete, but accepting PowerShell expands the publishable executable script surface and needs maintainer product/security approval before an allowlist change.

## What This Changes

Request support for uploading and publishing `.ps1` PowerShell files as part of ClawHub skill bundles.

## Best Possible Solution

Maintainers should decide the PowerShell file policy; if approved, add `ps1` consistently across shared and CLI allowlists, MIME normalization, browser/archive/GitHub import paths, Convex publish/API validation, moderation expectations, tests, and docs.

## Reproduction Assessment

Yes. Source inspection shows `.ps1` is absent from the shared and CLI extension allowlists, so a `.ps1` file with empty or generic MIME metadata fails the current browser and Convex publish text-file gates.

## Solution Assessment

Unclear until maintainers make the product/security policy call. A narrow allowlist-only patch would not be the best fix unless import, CLI, moderation, tests, and docs are updated together.
