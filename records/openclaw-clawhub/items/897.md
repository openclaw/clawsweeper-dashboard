Keep open: current main still does not intentionally accept `.ps1` skill files, and because this is an issue in `openclaw/clawhub` plus a product/security-sensitive file-type expansion, it should remain open for maintainer review.

## What This Changes

Allow PowerShell `.ps1` files to be uploaded and published as part of ClawHub skill bundles.

## Best Possible Solution

If maintainers approve PowerShell support, add `ps1` consistently across the shared schema and CLI allowlists, MIME normalization, browser/archive/GitHub/API/Convex validation, moderation expectations, tests, docs, and specs.

## Reproduction Assessment

Yes. Source inspection on current main shows `.ps1` is absent from the shared text extension allowlist, so a `.ps1` file with empty or generic MIME metadata fails the browser and server text-file gates.

## Solution Assessment

Unclear until maintainers make the product/security policy call. If approved, the maintainable solution is a coordinated allowlist, MIME, validation, moderation, docs, and test update rather than a publish-form-only change.
