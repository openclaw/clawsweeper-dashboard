## Summary

Keep open: current main still excludes `.ps1` from the shared and CLI text-file allowlists, so the rejection remains source-reproducible. Because this ClawHub issue asks to expand the accepted skill bundle surface to another executable script format, and related #290 is still open, it needs maintainer/security policy review rather than cleanup closure.

## What This Changes

Allow `.ps1` PowerShell scripts as valid text-based files in ClawHub skill uploads and publishing instead of requiring them to be embedded in `SKILL.md`.

## Best Possible Solution

Maintainers should make the PowerShell file policy explicit; if approved, add `ps1` consistently to shared and CLI allowlists with content-type handling, docs, and focused tests, and if rejected, document the intentional exclusion and workaround.

## Reproduction Assessment

Yes. Source inspection on current main shows `.ps1` is absent from the extension allowlists used by browser upload, server publish validation, archive handling, and CLI discovery, so a PowerShell file with an empty or non-text MIME type follows the rejection path.

## Solution Assessment

Unclear until maintainer/security approval. Adding `ps1` to the allowlists is mechanically narrow, but it is only the best solution if ClawHub intends to distribute PowerShell scripts in skill bundles.
