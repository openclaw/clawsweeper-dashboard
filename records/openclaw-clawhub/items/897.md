## Summary

Keep open: current main still excludes `.ps1` from the upload, publish, import, and CLI text-file allowlists, and accepting PowerShell would expand the executable script formats allowed in skill bundles. Related #290 remains open, so this needs maintainer/security policy review before an allowlist change.

## What This Changes

Allow `.ps1` PowerShell scripts as supporting files in ClawHub skill uploads and publishes instead of requiring authors to embed them in `SKILL.md`.

## Best Possible Solution

Maintainers should decide the PowerShell file policy; if approved, add `ps1` consistently to shared and CLI allowlists with MIME normalization, docs, and focused tests, otherwise document the intentional exclusion and workaround.

## Reproduction Assessment

Yes. Source inspection on current main shows `.ps1` is absent from the shared and CLI extension allowlists, so `.ps1` files with empty or non-text MIME are rejected by browser and server text-file checks.

## Solution Assessment

Unclear until maintainer/security policy is decided. The mechanical patch is narrow, but the maintainable solution depends on whether ClawHub wants PowerShell scripts in accepted published skill bundles.
