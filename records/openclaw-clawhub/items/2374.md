## Summary

Keep open: current main appears to regenerate and store a latest embedding during publish, but the live public symptom still reproduces for body-only SKILL.md terms and the stored production vector for the reported skill is not externally inspectable.

## What This Changes

The issue reports that republishing `the-news` with substantially changed SKILL.md content does not appear to refresh the skill's search embedding or ranking behavior.

## Best Possible Solution

Have a maintainer/operator inspect the stored latest embedding, version, digest, and visibility state for `the-news`; if the vector is stale, repair the publish/storage path, and if it is current, document or design diagnostics that preserve the search relevance contract.

## Reproduction Assessment

Partly: live public checks show body-only SKILL.md terms in `the-news` do not return the skill, and source inspection explains that body-only terms are not current eligibility fields. No high-confidence reproduction proves the stored latest embedding itself is stale.

## Solution Assessment

Unclear: a publish-path code fix is best only if production state confirms a stale vector or stale projection. If the vector is current, the better solution is a maintainer-approved docs or diagnostics change rather than weakening relevance gates by accident.
