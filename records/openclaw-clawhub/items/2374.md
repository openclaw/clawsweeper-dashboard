## Summary

Keep open: current main appears to regenerate and store embeddings on publish, but the reporter’s live symptom is real and still actionable because search eligibility is metadata-gated and the stored production vector is not externally inspectable.

## What This Changes

The issue reports that republishing `the-news` with substantially changed SKILL.md content does not appear to refresh or affect the skill embedding used by live search.

## Best Possible Solution

Keep this open for an operator check of the stored latest embedding and a maintainer decision on whether to fix a publish-path stale-vector bug, add private diagnostics, or clarify the metadata-gated search contract.

## Reproduction Assessment

Partly: live API checks reproduce the body-only term search symptom, and source inspection explains that current search filters vector candidates through public metadata. There is no high-confidence reproduction proving the stored latest production vector itself is stale.

## Solution Assessment

Unclear: the narrowest fix depends on production data showing stale embedding storage versus expected metadata-gated search behavior. The safer path is to verify the stored latest embedding first, then choose a publish-path fix, private diagnostic, or documentation/search-contract change.
