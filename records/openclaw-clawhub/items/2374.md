## Summary

Keep open: current main does not prove the reported production embedding is current, and live search still shows body-only SKILL.md terms for `the-news` failing to return the skill. Source inspection also shows that body-only terms are not enough to make a public search result eligible, so the next step is operator inspection plus a search-contract decision rather than cleanup closure.

## What This Changes

The issue reports that republishing `the-news` with substantially changed SKILL.md content does not appear to refresh search relevance or return distinctive body-only terms.

## Best Possible Solution

Have a maintainer/operator inspect the production `skills`, latest `skillVersions`, and visible `skillEmbeddings` rows for `the-news`; if the vector is stale or zeroed, repair the publish/storage/config path, and if it is current, clarify or adjust the metadata-gated search contract without exposing gameable ranking internals by default.

## Reproduction Assessment

No for the stale-vector claim: live API checks reproduce that body-only SKILL.md terms do not return `the-news`, but current source also explains that behavior through metadata-gated search eligibility even if the vector is fresh. Confirming the actual bug requires inspecting the stored production embedding row and environment state.

## Solution Assessment

Unclear until production state is checked. A stale or zero embedding needs a publish/config/storage repair, while a current embedding points to a search-contract or diagnostics decision; the closed score-breakdown PR is not an accepted solution path.
