## Summary

Keep this open: current main and the latest release still have the direct-lexical-before-vector merge path that can drop vector contribution, and the focused repair at https://github.com/openclaw/clawhub/pull/2591 is open rather than merged.

## What This Changes

The issue reports that republished skill content does not appear to affect search ranking because direct lexical matches can lose their vector score.

## Best Possible Solution

Land or replace the focused score-by-skill-id preservation fix with a regression test and real Convex/API output showing lexical-retained skills still receive vector contribution.

## Reproduction Assessment

Yes, at source level with high confidence: current main can retain a direct lexical entry without embeddingId, drop the vector-hydrated duplicate, and then score the retained entry with zero vector contribution. A live public API check for news also shows the reported missing-vector score pattern.

## Solution Assessment

Yes: preserving vector score by skill id, or enriching duplicate entries with embeddingId, is the narrow fix because it repairs score attribution without changing recall precedence or public response shape.
