## Summary

Keep open: current main does not prove the production embedding for `the-news` is current. Source shows publish should create a new embedding row, but live public searches for body-only SKILL.md terms still miss the skill and the current search path intentionally gates vector candidates through visible metadata before returning them.

## What This Changes

The issue reports that publishing new skill versions may not refresh the stored search embedding for changed SKILL.md content.

## Best Possible Solution

Have a maintainer/operator inspect the production `skills`, latest `skillVersions`, and visible `skillEmbeddings` rows for `the-news`; repair publish/config/storage if the vector is stale or zeroed, or clarify/adjust the metadata-gated search contract if the vector is current.

## Reproduction Assessment

No high-confidence reproduction for the stale-vector claim. Live public checks reproduce body-only SKILL.md terms not returning `the-news`, but current source also explains that behavior through metadata-gated search eligibility even if the vector is fresh.

## Solution Assessment

Unclear until production state is inspected. A stale or zero embedding needs a publish/config/storage repair, while a current embedding points to a search-contract or diagnostics decision; the closed score-breakdown PR is not an accepted solution path.
