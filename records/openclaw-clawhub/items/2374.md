## What This Changes

The issue reports that skill search rankings can lose refreshed vector signal after publish, later narrowed to direct lexical recall shadowing vector-hydrated matches.

## Best Possible Solution

Land or replace the focused score-by-skill-id preservation fix with a regression test and real Convex/API output showing lexical-retained skills still receive vector contribution.

## Reproduction Assessment

Yes at source level: current main can keep a direct lexical entry without `embeddingId`, drop the later vector-hydrated duplicate, and then score the retained entry with zero vector contribution. The live public API also shows the reported missing-vector ranking pattern for `news`.

## Solution Assessment

Yes: preserving vector score by skill id, or enriching duplicates with `embeddingId`, is the narrow fix because it repairs score attribution without changing recall precedence or public response shape.
