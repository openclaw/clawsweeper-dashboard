The issue reports that republishing `the-news` with substantially changed SKILL.md content does not seem to refresh the skill's search embedding or affect live search ranking.

## Best Possible Solution

Keep this open for a maintainer/operator check of the production latest embedding for `the-news`, then choose a private diagnostic, publish-path fix, or documented search eligibility change without unintentionally exposing ranking internals.

## Reproduction Assessment

Partly: live API checks and source inspection reproduce the body-only-term symptom through the current metadata-gated search path. No public, high-confidence reproduction proves that the stored latest production embedding itself is stale.

## Solution Assessment

Unclear: the narrow fix depends on whether production data shows a stale vector, an intentional search eligibility mismatch, or only missing diagnostics. The safest path is to inspect the stored latest embedding first, then decide between a publish-path repair, private diagnostics, or a search-contract change.
