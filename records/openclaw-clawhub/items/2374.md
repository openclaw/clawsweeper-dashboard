## Summary

Keep open: current main does generate and store a new embedding on publish, but the report is still actionable because the public symptom is consistent with the current metadata-gated search path and the stored production vector for the reported skill is not externally inspectable.

## What This Changes

The issue reports that republishing a skill with changed SKILL.md content does not appear to refresh the skill's search embedding.

## Best Possible Solution

Keep the issue open for a maintainer/operator check of the stored latest embedding and decide whether to land diagnostics like https://github.com/openclaw/clawhub/pull/2293, adjust search eligibility, or document the metadata gate more explicitly.

## Reproduction Assessment

Partly: source and live API checks reproduce the body-only-term symptom because current search requires matches in public metadata after vector recall. No high-confidence reproduction proves the stored latest embedding itself is stale.

## Solution Assessment

Unclear: the narrowest fix depends on whether production data shows a stale vector, a ranking/eligibility mismatch, or only missing diagnostics. A safe path is to verify stored embedding freshness first, then choose between a publish-path fix, score breakdown, or search contract change.
