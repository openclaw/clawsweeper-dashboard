## Summary

Current main still has the reported search-scoring defect: lexical recall can shadow the vector-hydrated copy of the same skill and omit its vector contribution. The issue should remain open while the paired implementation PR is reviewed.

## What This Changes

Correct skill search scoring so candidates recalled through both lexical and vector search retain their vector contribution.

## Best Possible Solution

Preserve vector similarity by skill ID through lexical/vector deduplication without changing recall order, then verify that overlapping candidates receive both lexical and vector score components through a real search API path.

## Reproduction Assessment

Yes at source level. Current main can retain a direct lexical entry without an embedding ID and consequently assign it zero vector similarity even though vector search returned the same skill.

## Solution Assessment

Yes. Keying the already-produced vector score by skill ID is the narrowest maintainable repair because it preserves existing recall and deduplication order.
