## What This Changes

The issue reports that published skill content or vector updates are not reflected in public skill search ranking because vector contribution appears missing for lexical matches.

## Best Possible Solution

Land or replace the focused score-by-skill-id preservation fix with regression coverage and real Convex/API output showing lexical-retained skills still receive vector contribution.

## Reproduction Assessment

Yes at source level: current main can keep a direct lexical entry without embeddingId, drop the later vector-hydrated duplicate, and then score the retained entry with zero vector contribution. A live public API check for news also matches the reported missing-vector ranking pattern.

## Solution Assessment

Yes: preserving vector score by skill id, or enriching duplicate entries with embeddingId, is the narrow maintainable repair because it fixes score attribution without changing recall precedence or public response shape.
