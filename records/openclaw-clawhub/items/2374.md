## Summary

Keep this open: current main still has the direct-lexical-before-vector merge path that can drop a vector contribution, and the narrow fix path is the open same-author PR at https://github.com/openclaw/clawhub/pull/2591 rather than a merged implementation.

## What This Changes

The issue reports that published skill content changes appear not to affect search ranking because lexical recall can retain a duplicate skill result without its vector score.

## Best Possible Solution

Land or replace the narrow score-by-skill-id preservation fix from https://github.com/openclaw/clawhub/pull/2591 after real Convex/API proof shows lexical-retained skills still receive vector contribution.

## Reproduction Assessment

Yes, at source level with high confidence: current main can retain a direct lexical entry without embeddingId, drop the vector-hydrated duplicate, and then score the retained entry with zero vector contribution. A live public API query for news also still shows the reported score/rank pattern, but I did not inspect the stored production vector directly.

## Solution Assessment

Yes: preserving vector scores keyed by skill id, or enriching retained duplicates with embeddingId, is the narrow maintainable fix because it repairs score attribution without changing recall precedence or public response shape.
