## Summary

Keep open: current main does regenerate and store an embedding during publish, but the reported public symptom is still actionable because search currently gates vector candidates through public metadata and the production vector for the reported skill is not externally inspectable.

## What This Changes

The issue reports that republishing `the-news` with substantially different SKILL.md content does not appear to refresh the skill's embedding-driven search behavior.

## Best Possible Solution

Have a maintainer or operator inspect the stored latest embedding/version state for `the-news`; if it is stale, repair the publish path, and if it is current, either document the metadata gate more clearly or choose a less gameable diagnostics path.

## Reproduction Assessment

Partly: source and live API checks reproduce that body-only terms in SKILL.md do not make `the-news` appear in search. No high-confidence reproduction proves the stored latest embedding itself is stale.

## Solution Assessment

Unclear: the narrowest fix depends on whether production data shows a stale vector, a ranking eligibility mismatch, or only missing diagnostics. A safe path is to verify stored embedding freshness before choosing between a publish-path fix, docs clarification, or search diagnostics.
