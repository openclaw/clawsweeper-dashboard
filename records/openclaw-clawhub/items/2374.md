## What This Changes

Fix skill search ranking so a skill recalled through both lexical and vector search retains its vector-score contribution after deduplication.

## Best Possible Solution

Land a narrow implementation that attributes vector results by skill ID across lexical/vector deduplication, then verify the corrected contribution or ranking through a real Convex deployment or live search API before closing the issue.

## Reproduction Assessment

Yes at source level with high confidence: current main retains the lexical duplicate first, discards the vector-hydrated duplicate, and then assigns zero vector score when the retained entry lacks embeddingId. The reporter also supplied a concrete live `news` query and score comparison, although after-fix runtime proof is not yet available.

## Solution Assessment

Yes in direction: retaining vector scores by skill ID fixes the information loss without changing established recall precedence. The linked PR should be accepted only after its focused test is supplemented with real Convex or live API proof.
