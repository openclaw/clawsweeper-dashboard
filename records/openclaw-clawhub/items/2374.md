## Summary

Keep this open: current main still has the direct-lexical-before-vector merge path that can drop vector contribution, and the paired open fix at https://github.com/openclaw/clawhub/pull/2591 has not merged.

## What This Changes

The issue reports that skill search can retain a lexical match without its vector score, making published skill content changes appear not to affect ranking.

## Best Possible Solution

Land or replace the narrow score-attribution fix with regression coverage and real Convex/API proof that lexical-retained skills still receive vector contribution.

## Reproduction Assessment

Yes, at source level: current main can retain a direct lexical entry without `embeddingId`, drop the vector-hydrated duplicate, and then score the retained entry with zero vector contribution. I did not run a live Convex reproduction in this read-only review.

## Solution Assessment

Yes: preserving vector scores keyed by skill id, or enriching duplicate entries with `embeddingId`, is the narrow maintainable repair because it fixes score attribution without changing recall precedence or response shape.
