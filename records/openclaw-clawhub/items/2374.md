## Summary

Keep open: current main still has the source-level search-scoring path that can drop vector contribution for lexical-retained skill matches, and the related repair is still an open PR rather than a merged fix.

## What This Changes

The issue reports that skills recalled through direct lexical search can lose their vector-score contribution, making search ranking look like embeddings did not refresh after publish.

## Best Possible Solution

Review and land https://github.com/openclaw/clawhub/pull/2591 or an equivalent narrow fix that attributes vector scores by skill id when lexical-retained entries shadow hydrated vector candidates.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: direct lexical matches can be retained without `embeddingId`, duplicates from vector hydration are dropped, and scoring falls back to zero vector score for that retained entry. I did not run the live `the-news` query against production.

## Solution Assessment

Yes, preserving vector attribution by skill id is the narrowest maintainable repair because it keeps existing recall ordering while restoring the vector contribution. The open PR still needs normal review and runtime proof before it resolves the issue.
