## Summary

Keep this open: current main still has the direct-lexical-before-vector merge path that can drop a skill's vector contribution, and the focused linked fix at https://github.com/openclaw/clawhub/pull/2591 is open rather than merged.

## What This Changes

Reports that refreshed skill embedding content is not reflected in public search ranking, narrowed to vector score loss when a direct lexical match shadows a vector-hydrated result.

## Best Possible Solution

Land or replace the focused score-attribution repair in https://github.com/openclaw/clawhub/pull/2591 after redacted real Convex or live API output confirms lexical-retained skills receive vector contribution.

## Reproduction Assessment

Yes, at source level: current main can retain a direct lexical entry without `embeddingId`, drop the vector-hydrated duplicate, and then score the retained entry with zero vector contribution. The issue discussion and existing ClawSweeper comment also document the public `news` query symptom.

## Solution Assessment

Yes, provisionally: preserving vector score by skill id or enriching duplicate entries is the narrow repair because it keeps recall ordering intact while fixing score attribution. The linked PR still needs runtime proof before merge.
