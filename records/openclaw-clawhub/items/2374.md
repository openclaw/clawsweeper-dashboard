## Label Justifications

- `P2`: The defect removes a valid ranking component for overlapping lexical/vector matches, degrading skill discoverability with a bounded search-only blast radius.
- `impact:other`: The issue affects search relevance and publisher discoverability but does not fit the specialized data, security, messaging, session, crash, or auth impact categories.

## Summary

Current main still drops vector similarity when a direct lexical match shadows the same vector-hydrated skill, and the paired open PR remains the active narrow fix, so this issue is still necessary until that PR lands or is replaced.

## What This Changes

Correct hybrid skill-search scoring so direct lexical matches retain their vector similarity contribution when the same skill is also returned by vector recall.

## Best Possible Solution

Preserve returned vector scores by skill ID across lexical/vector deduplication without changing recall order, retain focused regression coverage, and verify the corrected score through a real Convex deployment or live `/api/v1/search` request.

## Reproduction Assessment

Yes at source level: current main deterministically retains the direct lexical duplicate without its embedding ID and then assigns zero vector score. The reporter also supplied a concrete production API score comparison, although this review did not independently inspect the stored production vector.

## Solution Assessment

Yes. Keying the already-returned vector scores by skill ID is the narrowest maintainable repair because it preserves recall ordering and ranking policy while restoring the omitted score component.
