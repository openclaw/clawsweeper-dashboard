## Summary

Keep open. Current `main` still has the source-level path described by the reporter: direct lexical skill matches can shadow vector-hydrated duplicates, so the retained result can lose its vector contribution; the open fix is https://github.com/openclaw/clawhub/pull/2591 and should be reviewed or superseded rather than closing this issue now.

## What This Changes

The issue reports that ClawHub skill search may not apply refreshed embedding/vector relevance when a published skill is also recalled through direct lexical matching.

## Best Possible Solution

Review and land https://github.com/openclaw/clawhub/pull/2591 or an equivalent narrow fix, then close this issue once current `main` preserves vector scores for lexical-retained skill matches.

## Reproduction Assessment

Yes from source inspection: current `main` can keep a direct lexical entry without `embeddingId` ahead of the vector-hydrated duplicate, then score that retained entry with a zero vector contribution. I did not run a live production query in this read-only review.

## Solution Assessment

Yes, preserving vector scores by skill id or enriching direct lexical entries when their vector candidate was also recalled is the narrow maintainable fix. The existing open PR follows that direction and adds regression coverage.
