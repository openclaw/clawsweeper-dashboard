## What This Changes

The issue reports that published skill search ranking appears not to apply content-based vector relevance, later narrowed to direct lexical matches losing their vector score.

## Best Possible Solution

Land or replace a narrow score-by-skill-id preservation fix with regression coverage and real Convex/API proof that direct lexical matches still receive vector contribution.

## Reproduction Assessment

Yes, at source level with high confidence: current main can retain a direct lexical entry without `embeddingId`, drop the vector-hydrated duplicate, and then score the retained entry with zero vector contribution. I did not inspect the production stored vector directly.

## Solution Assessment

Yes: preserving vector scores keyed by skill id, or enriching duplicate entries with `embeddingId`, is the narrow maintainable fix because it repairs score attribution without changing recall precedence or the public response shape.
