## What This Changes

The issue reports that skill search can lose embedding/vector contribution when a direct lexical skill result shadows the vector-hydrated result after publish.

## Best Possible Solution

Land a focused score-by-skill-id or duplicate-enrichment fix that preserves vector contribution for lexical-retained matches, with regression coverage and redacted live Convex/API proof.

## Reproduction Assessment

Yes. Current main has the source path that drops vector score for lexical-retained duplicates, and the live `news` API result still shows `the-news` scoring like a lexical-only match.

## Solution Assessment

Yes. Preserving vector score by skill id, or enriching duplicate entries with `embeddingId`, is the narrow maintainable fix because it repairs score attribution without changing recall precedence or the public response shape.
