- `P2`: A possible stale or zero search vector degrades skill discoverability but does not block publishing or make the service unavailable.
- `impact:other`: The reported maintainer-visible impact is reduced search relevance and discoverability for newly published skill content.

## Summary

Current main intends to regenerate and promote an embedding on every semver-newer publish, but the report cannot be closed because neither public APIs nor the supplied reproduction establish whether production `the-news` has a stale or zero vector. The body-only keyword symptom is also inconclusive: current search deliberately requires query terms to appear in the slug, display name, summary, or capability tags before a vector candidate can rank.

## What This Changes

Investigate whether publishing a newer skill version refreshes the stored embedding used by live search.

## Best Possible Solution

Have an operator inspect the production `the-news` latest version, visible `skillEmbeddings` row, `embeddingSkillMap` row, vector norm, and Convex embedding configuration; repair configuration or version linkage if stale, otherwise clarify or deliberately revise the metadata-gated semantic-search contract and add safe diagnostics.

## Reproduction Assessment

No. The stale-vector claim cannot be reproduced with high confidence because the exact terms, queries, and publish versions are missing, and current search explains the body-only keyword symptom even with a fresh vector.

## Solution Assessment

Unclear. Embedding regeneration code should not change until production state is checked; a stale or zero vector needs an operational/write-path repair, while a current vector makes search eligibility and diagnostics a product decision.
