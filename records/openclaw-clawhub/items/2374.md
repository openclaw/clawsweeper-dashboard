## Summary

The reported symptom is a confirmed hybrid-search scoring bug on current main, not stale embedding publication: direct lexical recall can shadow the hydrated vector entry and omit its cosine contribution. The linked repair remains open, so this issue should stay open until that PR or an equivalent fix merges.

## What This Changes

Preserve a skill's vector contribution when its direct lexical match is retained ahead of a duplicate vector-hydrated search candidate.

## Best Possible Solution

Keep existing recall ordering and public API shape, preserve vector similarity by skill ID through lexical/vector deduplication, and validate the corrected combined score through a real Convex deployment or live search API fixture before merging.

## Reproduction Assessment

Yes. Current-main source deterministically exposes the shadowing path, and the reporter supplied live score arithmetic consistent with the affected skill receiving a zero vector contribution.

## Solution Assessment

Yes. Mapping vector scores by skill identity is the narrowest maintainable fix because it preserves the established recall order while preventing deduplication from discarding scoring metadata.
