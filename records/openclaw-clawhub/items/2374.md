## Summary

Keep open: current source shows publish does build and insert a fresh latest embedding, but live public checks still reproduce the reported search symptom for body-only SKILL.md terms, and public APIs cannot prove whether the stored production vector is stale versus a metadata-gated search contract issue.

## What This Changes

Investigate whether republishing `the-news` refreshes its stored search embedding and why substantial SKILL.md body edits do not affect live search ranking or recall.

## Best Possible Solution

Have a maintainer/operator inspect production latest-version and embedding state for `the-news`; if the vector is stale, repair the publish/index path, and if it is current, clarify the metadata-gated search contract or design safer diagnostics.

## Reproduction Assessment

Partly: live public checks reproduce that body-only terms in the current `the-news` SKILL.md do not return the skill, and source inspection explains the metadata gate. No high-confidence reproduction proves the stored latest embedding vector itself is stale.

## Solution Assessment

Unclear: the narrow fix depends on production data showing stale embedding state, stale projections, or expected metadata-gated eligibility. The safest path is production inspection before choosing between a publish-path fix, docs clarification, or diagnostics design.
