## Summary

Current main still has the reported search-scoring defect: lexical recall can shadow the vector-hydrated copy of the same skill and discard its vector contribution. The issue should remain open while the paired implementation PR is reviewed.

## What This Changes

Ensure a skill retained from direct lexical recall still receives its vector similarity score when vector search found the same skill.

## Best Possible Solution

Preserve vector similarity by skill ID through lexical/vector deduplication without changing recall order, then verify through focused tests and a real Convex or deployed search API smoke that overlapping candidates receive both lexical and vector score components.

## Reproduction Assessment

Yes at source level. Current main can retain a direct lexical entry without an embedding ID and consequently assign it zero vector similarity even though vector search returned the same skill.

## Solution Assessment

Yes. Keying the already-produced vector score by skill ID is the narrowest maintainable repair because it preserves existing recall and deduplication order while restoring the missing score component.
