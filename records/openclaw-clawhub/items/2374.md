## What This Changes

The issue reports that skill search can lose a refreshed embedding contribution when a republished skill is recalled first through lexical search.

## Best Possible Solution

Review and land https://github.com/openclaw/clawhub/pull/2591 or an equivalent narrow score-attribution fix, then close this issue once main preserves vector scores for lexical-retained skill matches.

## Reproduction Assessment

Yes at source level: current main can retain a direct lexical match without `embeddingId` before a vector-hydrated duplicate, leaving scoring with zero vector contribution for that skill. I did not run a live production or Convex runtime reproduction in this read-only review.

## Solution Assessment

Yes; preserving vector score by skill id or enriching duplicate entries during merge is the narrowest maintainable repair because it fixes scoring without changing the existing recall order.
