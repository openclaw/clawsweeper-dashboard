## What This Changes

Preserve embedding similarity when a skill recalled lexically is also present among vector-search results.

## Best Possible Solution

Keep the existing recall and deduplication precedence, retain vector similarity by stable skill ID, and validate the merged result through both the focused regression and redacted real Convex/API output for the affected search.

## Reproduction Assessment

Yes at source level. Current main deterministically retains a lexical entry without an embedding ID over its vector-hydrated duplicate and consequently applies a zero vector score; a live current-main execution was not performed during this read-only review.

## Solution Assessment

Yes. Keying already-observed vector scores by skill ID is the narrowest maintainable repair because it preserves recall ordering, ranking weights, and the public API.
