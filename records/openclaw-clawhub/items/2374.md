## What This Changes

The issue reports that refreshed skill embeddings appear not to affect search ranking, later narrowed to direct lexical skill matches losing their vector score when merged with vector results.

## Best Possible Solution

Land a focused score-by-skill-id preservation fix that keeps current recall ordering stable, backed by a regression test and real Convex or live API output showing the restored vector contribution.

## Reproduction Assessment

Yes. A public read API check for `news` still reproduces the reported ranking pattern, and current `main` has the source path that drops vector score for lexical-retained duplicates.

## Solution Assessment

Yes. Preserving vector score by skill id, or equivalently enriching duplicate entries with `embeddingId`, is the narrow maintainable fix because it repairs score attribution without changing recall precedence or public response shape.
