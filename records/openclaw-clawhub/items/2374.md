## Summary

Current main still has the reported search-ranking failure shape: lexical matches are merged ahead of vector-hydrated matches and can be scored with a zero vector contribution. The issue should stay open while the focused linked fix candidate is reviewed.

## What This Changes

The issue reports that skill search can ignore a refreshed embedding's vector score when a skill is also recalled through direct lexical search.

## Best Possible Solution

Land https://github.com/openclaw/clawhub/pull/2591 or an equivalent narrow fix that preserves vector scores by skill id when lexical and vector recall return the same skill, with regression coverage and runtime/API proof.

## Reproduction Assessment

Yes at source level: current main keeps a direct lexical duplicate before the vector-hydrated duplicate and then scores only from the retained entry's `embeddingId`. I did not run a live Convex/API reproduction in this read-only review.

## Solution Assessment

Yes: preserving vector scores by skill id, or enriching merged duplicates with the hydrated embedding id, is the narrow fix because it keeps the existing recall order while restoring the missing vector contribution.
