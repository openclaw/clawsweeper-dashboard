## Summary

Keep open: current main still intends to generate and store a fresh latest embedding during publish, but the reported live symptom can also be explained by the current metadata-gated search path, and the stored production vector for `the-news` is not publicly inspectable. The closed diagnostics PR at https://github.com/openclaw/clawhub/pull/2293 is useful context, but it was not merged and maintainers raised an API/product concern about exposing ranking internals.

## What This Changes

The issue reports that republishing `the-news` with substantially changed SKILL.md content does not appear to refresh the skill's live search embedding or ranking behavior.

## Best Possible Solution

Have a maintainer/operator inspect the production latest embedding state for `the-news`, then choose a narrow publish-path fix, a private/admin diagnostic, or a documented search-contract change based on that evidence.

## Reproduction Assessment

Partly: the public body-only-term symptom is reproducible against the live API, and source inspection explains why current search can omit that result. We do not have a high-confidence reproduction proving the stored latest embedding itself is stale.

## Solution Assessment

Unclear: the best fix depends on production vector state. Verifying the stored latest embedding should come before changing publish logic, exposing score breakdowns, or widening search eligibility.
