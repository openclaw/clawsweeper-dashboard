## Summary

Current main still has the reported ranking defect: a direct lexical result can shadow the same vector-hydrated skill and receive a zero vector contribution. The reporter’s detailed live search example supports the same root cause, and the paired open PR https://github.com/openclaw/clawhub/pull/2591 now owns the narrow fix, so this issue remains necessary until that PR lands or is replaced.

## What This Changes

Correct skill search scoring so a direct lexical match does not lose the vector similarity score for the same skill.

## Best Possible Solution

Preserve vector similarity by skill ID across lexical/vector deduplication while retaining the existing recall order, add focused regression coverage, and verify the corrected score through a real Convex or live `/api/v1/search` request.

## Reproduction Assessment

Yes at source level: current main deterministically retains a lexical duplicate without its embedding ID and therefore assigns zero vector score. The reporter also supplied a concrete production API example, although this review did not independently inspect the stored production vector.

## Solution Assessment

Yes. Keying the already-returned vector scores by skill ID is the narrowest maintainable fix because it preserves recall ordering and ranking policy while restoring the missing score component.
