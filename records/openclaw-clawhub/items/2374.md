Keep open: current main has a plausible publish-time embedding refresh path, but it does not prove the production `the-news` vector is current, and public search still shows the reported body-term discoverability symptom. ## What This Changes

The issue reports that republishing `the-news` with substantially changed SKILL.md content appears not to refresh the stored embedding or live search ranking.

## Best Possible Solution

Have a maintainer/operator inspect the production `skills`, latest `skillVersions`, visible `skillEmbeddings`, and `embeddingSkillMap` rows for `the-news`; repair publish/config/storage if the vector is stale or zero, otherwise clarify or adjust the search contract/diagnostics path.

Confirming the actual bug requires inspecting the stored production embedding row and environment state.
