## Summary

Keep this open: current main still has the direct-lexical-before-vector merge path that can drop a skill's vector contribution, and the focused linked fix at https://github.com/openclaw/clawhub/pull/2591 is open rather than merged.

## What This Changes

The issue asks ClawHub search to apply vector scores from refreshed skill embeddings even when the same skill is also recalled by direct lexical search.

## Best Possible Solution

Land or replace the focused score-by-skill-id preservation fix, with a regression test and real Convex/API output showing that lexical-retained skills still receive vector contribution.

## Reproduction Assessment

Yes at source level: current main can keep a direct lexical entry without `embeddingId`, drop the later vector-hydrated duplicate, and then score the retained entry with zero vector contribution. A live public API check also returned `cctv-news-fetcher` at rank 4 and `the-news` at rank 58 for `news`.

## Solution Assessment

Yes, provisionally: preserving vector score by skill id or enriching duplicate entries is the narrow repair because it keeps recall ordering intact while fixing score attribution. The linked PR still needs runtime proof before merge.
