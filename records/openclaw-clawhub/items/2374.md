## Summary

Keep this issue open: current main still has the source-level scoring path that can drop a vector score when a direct lexical result shadows a vector-hydrated result, and the linked fix is still an open PR rather than merged implementation.

## What This Changes

The issue reports that skill search ranking can lose the semantic/vector contribution for a published skill, later narrowed to direct lexical matches shadowing vector-hydrated search entries for the same skill.

## Best Possible Solution

Keep this issue open until a focused fix such as https://github.com/openclaw/clawhub/pull/2591 lands with regression coverage and redacted real search-output proof for the lexical/vector overlap path.

## Reproduction Assessment

Yes from source inspection: when a skill appears first in direct lexical results without embeddingId and later in vector exactMatches with embeddingId, mergeUniqueBySkillId retains the lexical entry and scoring falls back to vectorScore 0. I did not run a live Convex search in this read-only cleanup pass.

## Solution Assessment

Yes, the narrow maintainable fix is to preserve or enrich vector scoring by skill id across deduplication rather than changing embedding generation, search product policy, or public score observability.
