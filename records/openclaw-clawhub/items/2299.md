Overall correctness: patch is incorrect

Full review comments:

- **[P1] Clear publisher scope when reserving for a user:** `convex/lib/reservedSlugs.ts:181-187`
  - body: This path can reuse the latest active reservation and change `originalOwnerUserId`, but it leaves any previous `originalOwnerPublisherId` intact. After this PR, reservation matching prefers publisher scope, so reusing an org-scoped row for a rightful personal owner can keep the reservation effectively attached to the old publisher instead of the intended user.
  - confidence: 0.9
- **[P1] Preserve unauthenticated publish dry-runs:** `packages/clawhub/src/cli/commands/publish.ts:100-102`
  - body: `cmdPublish` now resolves a default owner before reaching the dry-run branch, so a user who previously ran `clawhub publish --dry-run` without auth can be forced through token/whoami resolution. Keep dry-run preview compatible by deferring required owner auth until an actual publish or by using slug-only preview behavior when no owner is available.
  - confidence: 0.82
- **[P1] Resolve the package namespace policy before codifying it:** `convex/packages.ts:6304-6312`
  - body: Switching package publish to `hasAnySkillWithSlugInternal` means any skill owner can globally block a package name, even though the current slug-routing spec treats skills and packages as separate registries. Either narrow this guard to the intended same-owner/route-conflict case or update the spec and compatibility plan before merging the policy change.
  - confidence: 0.8
