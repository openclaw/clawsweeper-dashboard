## Summary

This PR remains necessary because current main still relies on globally keyed slug paths, but it is not merge-ready: owner identity is lost in auto-generated changelog context and local installation keys, real upgrade proof is absent, and the branch conflicts with current main.

## What This Changes

The PR makes skill slugs publisher-scoped across Convex publishing and lookup, public APIs, ClawHub CLI operations, UI links, schemas, generated artifacts, and tests while retaining legacy slug-only compatibility.

## Best Possible Solution

Use `(owner, slug)` as the canonical identity through publishing, derived metadata, APIs, and local state; either encode the owner in local install keys and directories or consistently reject same-slug local collisions, preserve a documented legacy fallback, add a focused ownership/API-boundary spec, and prove fresh and upgrade flows on a real Convex setup.
