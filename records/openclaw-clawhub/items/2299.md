Full review comments:

- **[P1] Retarget reclaimed reservations to the new owner scope:** `convex/lib/reservedSlugs.ts:181-187`
  - body: With this PR, reservation checks prefer `originalOwnerPublisherId` when it is present. `upsertReservedSlugForRightfulOwner` retargets an existing reservation by changing only `originalOwnerUserId`, so reclaiming a slug from an org/publisher can leave the old publisher id attached; the rightful publisher will not match or release that reservation on publish, while the old publisher can still look like the scoped owner.
  - confidence: 0.91
- **[P2] Do not hard-code the global package-skill collision:** `convex/packages.ts:5248-5256`
  - body: This hunk preserves a global skill-vs-package bare-name block by asking whether any skill has the package name, and the added test codifies that behavior. The related discussion at https://github.com/openclaw/clawhub/issues/2390 says skills and plugins should be typed namespaces where the same bare name can be valid, so this should be removed, narrowed, or explicitly settled in specs before merge.
  - confidence: 0.82
