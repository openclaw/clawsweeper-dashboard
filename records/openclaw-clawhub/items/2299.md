## Summary

Keep this PR open because owner-scoped skill identity matches the repository’s documented direction, but the proposed legacy resolver introduces a blocking identity-boundary regression: when duplicate live slugs include an `openclaw` publisher row, it silently selects that row instead of returning ambiguity as the current durable spec requires. The PR is also conflict-blocked and its validation is test/CI-only rather than real Convex-backed behavior proof.

## What This Changes

The PR makes skill slug resolution owner-aware across backend publishing and reads, public APIs, security and moderation-adjacent operations, the ClawHub CLI, persisted metadata, and UI install/routing surfaces while retaining legacy slug-only compatibility.

## Best Possible Solution

Use `(ownerPublisherId, slug)` as the primary identity everywhere, make every multi-publisher legacy slug-only lookup return controlled ambiguity without publisher precedence, preserve explicit owner-qualified and authorized migration paths, and validate the rebased result on real Convex state with two publishers sharing a slug.
