## Summary

This PR is still necessary because current main continues to resolve publish preflight globally by slug, but it cannot be approved yet: the branch conflicts with current main, changes a security-sensitive identity contract without a durable spec, and provides no real Convex/CLI/browser proof for fresh-install and upgrade behavior.

## What This Changes

The PR makes skill slugs publisher-scoped across Convex publish/read/manage paths, public APIs, ClawHub CLI references and persisted metadata, and web install/update/download links while retaining compatibility behavior for slug-only clients.

## Best Possible Solution

Preserve the publisher-scoped identity model, but first rebase the branch, document namespace ownership, alias/reservation scope, legacy ambiguity precedence, migration authorization, and upgrade invariants in `specs/`, then validate duplicate-slug publish/read/install/update and existing-install upgrades against a real Convex backend and signed-in ClawHub flow.
