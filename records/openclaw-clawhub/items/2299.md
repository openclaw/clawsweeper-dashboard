## Summary

This owner-scoping work remains necessary, but the PR is not merge-ready because its CLI still collapses distinct `@owner/slug` references into one bare-slug local install identity. The branch also conflicts with current main, lacks required real-runtime proof, and needs a durable ownership and upgrade spec before its broad compatibility and authorization changes can land safely.

## What This Changes

The PR adds publisher-scoped skill slug resolution across Convex, public APIs, CLI commands and persisted metadata, and UI install/download links while retaining legacy slug-only compatibility responses.

## Best Possible Solution

Use owner-qualified identity as the canonical registry and local-storage key, with a backward-compatible migration for legacy slug-only installs; if local coexistence is intentionally unsupported, reject different-owner collisions before touching files or lock state. Rebase the branch, document the ownership and ambiguity invariants in `specs/`, and prove fresh publish/install plus upgrade behavior against real Convex state.
