## Summary

This PR remains necessary and follows ClawHub’s existing owner/slug routing direction, but it should not merge as written: distinct `@owner/slug` identities still collapse into one slug-named local directory and lockfile entry, enabling destructive replacement and wrong-owner updates. The conflicted head, collaborator authorship, and missing real Convex-backed proof also require explicit maintainer review.

## What This Changes

The PR makes skill slugs owner-scoped across Convex publishing and lookup, legacy HTTP compatibility, ClawHub CLI references and persisted metadata, and owner-aware UI install and download links.

## Best Possible Solution

Adopt owner-qualified registry identity with a collision-safe local policy: either key directories and lock entries by stable owner-qualified identity with a tested migration, or reject a second same-slug publisher before any removal or write; then document and prove legacy ambiguity, handle rename, and old-install upgrades.
