## Summary

Keep open. ClawHub policy keeps issues open in this cleanup pass, and current main still contains the reported Convex failure shape: owner-publisher digest sync can run repeated `.paginate()` calls inside one internal mutation, while the scheduled wrapper has no cursor continuation state. The related open reports describe the same publish-time error cluster, so this should be handled as a focused backend repair.

## What This Changes

Fix CLI publish failures caused by `syncSkillSearchDigestsForOwnerPublisherId` running multiple paginated Convex queries in one function.

## Best Possible Solution

Refactor owner-publisher package and skill digest rebuilds so each scheduled internal mutation executes at most one indexed `.paginate()` call, processes that page, and reschedules itself with `continueCursor` until complete while preserving missing-table fallbacks and digest semantics.

## Repair Work Prompt

Fix the Convex publish failure caused by owner-publisher digest rebuilds running multiple paginated queries in one function. In `convex/functions.ts`, refactor both `syncPackageSearchDigestsForOwnerPublisherId` and `syncSkillSearchDigestsForOwnerPublisherId` / their internal scheduled mutations so one invocation performs at most one indexed `.paginate()` call, processes that page, and schedules the next invocation with `continueCursor` when `isDone` is false. Preserve the existing `by_owner_publisher` indexes, missing-table fallback behavior, digest upsert semantics, and publisher-trigger scheduling. Add focused tests in `convex/functions.test.ts` that fail if a single internal invocation consumes multiple pages and verify continuation scheduling for both package and skill digest rebuilds. Regenerate `convex/_generated` if Convex function validators or references require it. Do not replace pagination with unbounded `.collect()`, and do not broaden this PR into unrelated publish, search, or schema changes. Related reports: openclaw/clawhub#1171, openclaw/clawhub#1176, openclaw/clawhub#1182, openclaw/clawhub#1195, openclaw/clawhub#1197, openclaw/clawhub#1205.
