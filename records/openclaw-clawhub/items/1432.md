## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still has the reported source-level mismatch: the docs and parser understand nested `metadata.openclaw.requires.env`, but the static moderation env collector only reads direct fields from raw frontmatter/metadata while publish and static rescan pass that raw shape through.

## What This Changes

The issue asks ClawHub to extract `openclaw.requires.env` and `primaryEnv` from published skill files so legitimate env usage is not falsely flagged as suspicious.

## Best Possible Solution

Make static publish and static rescan moderation consume the same normalized runtime metadata used by parsing/display/LLM context, add regression coverage for inline JSON `metadata.openclaw.requires.env` and `primaryEnv`, then rescan or manually review the affected DeepRead versions while leaving broader metadata-file ingestion to #1053 unless maintainers intentionally include it.
