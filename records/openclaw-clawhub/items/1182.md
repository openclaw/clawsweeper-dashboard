## Summary

Keep open. This is an issue, and the ClawHub cleanup policy for this pass does not auto-close issues; source inspection also shows current main only mitigates the original trigger path by scheduling the package and skill digest updates separately. The named helper still contains a paginated loop that can make more than one paginated query in a single internal mutation, so maintainers should verify production logs and smoke-test publish/import before resolving the cluster.

The issue reports CLI, upload, and GitHub import publish failures caused by Convex's multiple-paginated-query limit in `syncSkillSearchDigestsForOwnerPublisherId`.

## Best Possible Solution

Keep the issue open until maintainers verify whether the scheduler mitigation is deployed and resolves real CLI, upload, and GitHub import publishes for affected accounts. If failures remain, refactor the owner-publisher digest syncs into a one-paginated-query-per-mutation continuation pattern and close the related publish/import reports together.
