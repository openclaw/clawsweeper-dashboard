## Summary

Keep open. Current main still excludes `.ps1` from the shared text-file allowlists and every upload/publish/import gate checked here, and ClawHub cleanup policy keeps issues open unless they are already resolved by a merged PR.

## What This Changes

The issue asks ClawHub to accept PowerShell `.ps1` files during new skill creation.

## Best Possible Solution

Maintainers should decide whether PowerShell scripts are allowed as publishable skill content; if approved, add support consistently across shared and CLI allowlists, MIME normalization, browser/archive/GitHub import paths, Convex publish validation, moderation coverage, tests, and docs.

## Reproduction Assessment

Yes, there is a high-confidence source reproduction for the current rejection path: a `.ps1` file with empty or generic MIME metadata fails the extension allowlist in the browser helper and then fails the same text gate on publish/import paths.

## Solution Assessment

Unclear until maintainers make the policy decision. A narrow allowlist-only change would be incomplete because upload, import, server validation, docs, and moderation expectations need to move together.
