## Summary

Keep open. Current main still excludes `.ps1` from the text-file allowlists and the browser, archive, CLI, GitHub import, API, and Convex publish paths remain extension/MIME gated; the related open report at https://github.com/openclaw/clawhub/issues/897 adds concrete Windows-skill context, and ClawHub cleanup policy keeps issues open unless they are PRs already implemented on main.

## What This Changes

Support uploading `.ps1` PowerShell files as supporting files during new skill creation.

## Best Possible Solution

Maintainers should decide whether PowerShell scripts are allowed as publishable skill content; if approved, add `.ps1` consistently across shared and CLI allowlists, MIME normalization, browser/archive/GitHub/API/Convex validation, raw file serving, moderation expectations, tests, docs, and security-sensitive specs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `.ps1` is absent from the shared and CLI extension allowlists, so a `.ps1` file with empty or generic MIME metadata fails the browser and Convex publish text-file gates.

## Solution Assessment

Unclear until maintainers make the product/security policy call. If approved, the maintainable fix is a coordinated allowlist, MIME, import/API/CLI, moderation, docs, and test update rather than a publish-form-only change.
