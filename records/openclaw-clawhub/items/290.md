## Summary

Keep open. Current main still does not intentionally support `.ps1` skill files, and for `openclaw/clawhub` issues are not cleanup-close candidates unless they are resolved by maintainer action; related user context at https://github.com/openclaw/clawhub/issues/897 reinforces that this needs product/security triage before changing the accepted executable file surface.

## What This Changes

Allow ClawHub skill creation and publishing to accept PowerShell `.ps1` files.

## Best Possible Solution

If maintainers approve PowerShell support, add `ps1` consistently across shared and CLI allowlists, MIME normalization, browser/archive/GitHub import paths, Convex publish/API validation, moderation expectations, focused tests, and user-facing docs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `.ps1` is absent from the shared and CLI extension allowlists, so a `.ps1` file with empty or generic MIME metadata fails the browser and Convex publish text-file gates.

## Solution Assessment

Unclear until maintainers make the product/security policy call. A narrow allowlist-only change would not be the best fix unless import, CLI, moderation, tests, and docs are updated together.
