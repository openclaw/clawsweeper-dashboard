## Summary

Keep open. Current main still omits `.ps1` from the shared text-file allowlists and all upload, archive, CLI, GitHub-import, and Convex publish paths remain extension/MIME gated; since this is a ClawHub issue rather than a PR already implemented on main, cleanup policy also keeps it open. Related open report #897 adds useful user context and a workaround of embedding the script in `SKILL.md`, so this should be handled as one maintainer policy/security decision.

## What This Changes

Allow `.ps1` PowerShell scripts to be uploaded as skill files during skill creation and publishing.

## Best Possible Solution

Maintainers should decide whether PowerShell scripts are allowed publishable skill content; if approved, add support consistently across shared and CLI allowlists, MIME normalization, web validation, archive expansion, Convex publish/API validation, GitHub import, moderation checks, tests, and docs.

## Reproduction Assessment

Yes, from source inspection. A `.ps1` file with empty or generic MIME metadata fails the current extension allowlist in the browser helper and is rejected again by Convex publish validation.

## Solution Assessment

Unclear until maintainers make the policy decision. A narrow allowlist-only change would be incomplete because upload/import/API behavior and moderation expectations need to be aligned.
