## Summary

Keep open. Current main still omits `.ps1` from the text-file allowlists and the browser upload, archive expansion, CLI packaging, Convex publish, API raw-file, and GitHub import paths remain text-extension/MIME gated; ClawHub policy also says ordinary issues should stay open, and related open report #897 adds user context and a workaround.

## What This Changes

The issue asks ClawHub to allow `.ps1` PowerShell files during new skill creation and upload.

## Best Possible Solution

Maintainers should decide whether PowerShell scripts are allowed as publishable skill content; if approved, support should be added consistently across shared and CLI allowlists, MIME normalization, browser validation, archive expansion, Convex publish/API validation, GitHub import, moderation expectations, tests, and docs.

## Reproduction Assessment

Yes. A `.ps1` file with empty or generic MIME metadata reaches `isTextFile`, where the extension is not in `TEXT_FILE_EXTENSION_SET`, and the Convex publish path has the same text gate.

## Solution Assessment

Unclear until maintainers make the policy/security call. A narrow extension-only change would be incomplete because upload/import/API behavior, CLI packaging, docs, and static moderation expectations need to stay aligned.
