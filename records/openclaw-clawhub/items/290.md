## Summary

Keep open. Current main still rejects `.ps1` through the shared text-file allowlists and publish/import validation paths, and this ClawHub issue needs maintainer policy/security review rather than cleanup closure.

## What This Changes

Allow PowerShell `.ps1` files to be uploaded and published during skill creation.

## Best Possible Solution

Maintainers should decide whether PowerShell scripts are allowed as publishable skill content; if approved, add support consistently across shared and CLI allowlists, browser/archive upload, Convex/API/GitHub import, moderation expectations, tests, and docs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `.ps1` is not in `TEXT_FILE_EXTENSION_SET`, and browser upload, archive expansion, GitHub import, and Convex publish all gate on that text-file check.

## Solution Assessment

Unclear until maintainers make the policy call. A narrow extension allowlist change would be incomplete because upload/import/API behavior and moderation coverage need to stay aligned.
