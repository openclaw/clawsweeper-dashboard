## Summary

Keep open. Current main still rejects `.ps1` through the shared text-file allowlist and upload, archive, Convex publish, and GitHub import gates; because this is a ClawHub issue and the requested change expands executable script content, it needs maintainer policy and security review rather than cleanup closure.

## What This Changes

The issue requests allowing `.ps1` PowerShell script files during new skill creation and publish.

## Best Possible Solution

Maintainers should decide whether PowerShell scripts are allowed as publishable text skill content; if approved, support should be added consistently across shared and CLI allowlists, upload/import/publish validation, moderation expectations, tests, and docs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a `.ps1` file with empty or generic MIME metadata fails `isTextFile`, and the same extension gate is used by browser upload, archive expansion, Convex publish, and GitHub import.

## Solution Assessment

Unclear until maintainers make the policy call. A narrow allowlist-only patch would be incomplete because the upload/import/API behavior and static moderation expectations need to move together.
