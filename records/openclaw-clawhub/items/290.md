## Summary

Keep open. Current main still omits `.ps1` from the shared text-file allowlists and the browser, archive, Convex publish, and GitHub import paths remain extension/MIME gated; related open report #897 adds useful user context, and accepting PowerShell scripts needs maintainer policy/security review.

## What This Changes

The issue requests support for uploading PowerShell `.ps1` files during ClawHub skill creation.

## Best Possible Solution

Maintainers should decide whether PowerShell scripts are allowed as publishable skill content; if approved, support should be added consistently across allowlists, MIME normalization, browser/archive/Convex/GitHub import paths, static moderation, tests, and docs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a `.ps1` file with empty or generic MIME metadata reaches `isTextFile`, where `.ps1` is not in `TEXT_FILE_EXTENSION_SET`, and server publish validation uses the same gate.

## Solution Assessment

Unclear until maintainers make the policy call. Adding only the extension would be incomplete because upload/import/API behavior and PowerShell-specific moderation coverage need to move together.
