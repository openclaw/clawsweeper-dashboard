## Summary

Keep open. Current main still omits `.ps1` from the shared text-file allowlists and the browser, archive, Convex publish, API, and GitHub import paths remain extension/MIME gated; related open issue #897 adds user impact and a workaround, so this needs maintainer policy/security review rather than cleanup closure.

## What This Changes

Allow `.ps1` PowerShell scripts to be uploaded and published as skill files during new skill creation.

## Best Possible Solution

Maintainers should decide whether PowerShell scripts are allowed as publishable text skill content; if approved, add support consistently across shared and CLI allowlists, MIME normalization, browser validation, archive expansion, Convex publish/API validation, GitHub import, moderation expectations, tests, and docs.

## Reproduction Assessment

Yes. A `.ps1` file with empty or generic MIME metadata follows the current publish path into `isTextFile`, where the extension is absent from `TEXT_FILE_EXTENSION_SET`; the Convex publish path applies the same text gate.

## Solution Assessment

Unclear until maintainers make the policy call. A narrow allowlist-only change would be incomplete because upload/import/API behavior and static moderation expectations need to be aligned.
