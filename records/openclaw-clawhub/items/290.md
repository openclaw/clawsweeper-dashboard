## Summary

Keep open. Current main still excludes `.ps1` from the text-file allowlists and the web, archive, Convex publish, and GitHub import paths remain extension/MIME gated; this is also an issue in ClawHub, where cleanup policy keeps issues open unless they are already resolved by a merged PR.

## What This Changes

The issue requests support for uploading `.ps1` PowerShell scripts when creating or publishing a ClawHub skill.

## Best Possible Solution

Maintainers should decide whether PowerShell scripts are allowed as publishable text skill content; if approved, support should be added consistently across shared and CLI allowlists, MIME normalization, web validation, archive expansion, Convex publish/API validation, GitHub import, moderation expectations, tests, and docs.

## Reproduction Assessment

Yes, from source inspection. A `.ps1` file with empty or generic MIME metadata fails the current extension allowlist in the browser helper and the same text gate is enforced again by Convex publish/import paths.

## Solution Assessment

Unclear until maintainers make the policy decision. A narrow allowlist-only change would be incomplete because upload/import/API behavior and moderation expectations need to be aligned.
