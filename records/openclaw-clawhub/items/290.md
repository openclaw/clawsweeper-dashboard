## Summary

Keep open. Current main still does not intentionally support `.ps1` by extension across the upload, archive, CLI, API, and import paths, and ClawHub cleanup policy keeps issues open unless they are PRs already implemented on main; related open report #897 adds current Windows/PowerShell user context and a workaround.

## What This Changes

The issue asks ClawHub to accept PowerShell `.ps1` files during new skill creation/upload.

## Best Possible Solution

Maintainers should decide whether PowerShell scripts are allowed as publishable text skill content; if approved, add support consistently across shared and CLI allowlists, MIME normalization, upload/archive/API/import paths, moderation/static-scan expectations, tests, and docs.

## Reproduction Assessment

Yes. The deterministic reproduction path is a `.ps1` file with empty or generic MIME metadata: current helpers reject it because `TEXT_FILE_EXTENSION_SET` lacks `ps1`, and archive/import/CLI paths filter by extension.

## Solution Assessment

Unclear until maintainers make the policy/security decision. A narrow implementation would be maintainable only if it updates validation, import, moderation, tests, and docs together rather than adding `ps1` to one allowlist.
