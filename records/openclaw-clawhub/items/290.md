## Summary

Keep open. Current main still does not intentionally accept `.ps1` skill files, the related open report at https://github.com/openclaw/clawhub/issues/897 adds concrete Windows skill context and a SKILL.md workaround, and this ClawHub issue needs maintainer product/security approval before expanding the publishable executable script surface.

## What This Changes

The issue requests accepting PowerShell `.ps1` files during ClawHub skill creation and publishing.

## Best Possible Solution

Maintainers should decide the PowerShell file policy; if approved, add `ps1` consistently across shared and CLI allowlists, MIME normalization, browser/archive/GitHub import paths, Convex publish/API validation, raw file serving, moderation expectations, tests, and docs.

## Reproduction Assessment

Yes. Source inspection against current main gives a high-confidence path: `.ps1` is absent from the shared and CLI extension allowlists, so a `.ps1` file with empty or generic MIME metadata fails the current browser and Convex publish text-file gates.

## Solution Assessment

Unclear until maintainers make the product/security policy call. A narrow allowlist-only patch would not be the best fix unless import, CLI, moderation, tests, and docs are updated together.
