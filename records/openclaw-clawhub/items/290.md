## Summary

Keep open. Current main still omits `.ps1` from the shared text-file allowlists, and the browser upload, archive expansion, Convex publish, GitHub import, and CLI collection paths remain extension/MIME gated; ClawHub cleanup policy also keeps issues open unless a PR is already implemented on main.

## What This Changes

The issue requests support for uploading PowerShell `.ps1` scripts as skill files.

## Best Possible Solution

Maintainers should decide whether PowerShell scripts are allowed as publishable skill content; if approved, add support consistently across shared and CLI allowlists, MIME normalization, browser validation, archive expansion, Convex publish/API validation, GitHub import, moderation expectations, tests, and docs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a `.ps1` file with empty or generic MIME metadata fails the current text-file gates, and both the publish form and Convex publish validation reject files that fail those gates.

## Solution Assessment

Unclear until maintainers make the policy call. A narrow allowlist-only change would be incomplete because upload/import/API behavior, CLI packaging, docs, and moderation expectations need to move together.
