## Summary

Keep open. Current main still does not intentionally support `.ps1` skill files, and ClawHub cleanup policy keeps issues open unless a PR is already implemented on main. The related open report #897 adds current user context and a workaround, so this should be handled as a coordinated maintainer policy/security decision rather than closed by cleanup.

## What This Changes

The issue asks ClawHub to allow `.ps1` PowerShell scripts to be uploaded as part of skill creation.

## Best Possible Solution

Maintainers should decide whether PowerShell scripts are allowed as publishable text skill content. If approved, support should be added consistently across shared and CLI schema allowlists, MIME normalization, browser validation, archive expansion, Convex publish/API validation, GitHub import, moderation expectations, tests, and user-facing docs.
