## Summary

Keep open. Current main still does not intentionally accept `.ps1` skill files, the related open report at https://github.com/openclaw/clawhub/issues/897 adds concrete Windows-skill context and a workaround, and ClawHub cleanup policy keeps issues open unless they are PRs already implemented on main.

## What This Changes

The issue requests support for uploading `.ps1` PowerShell files during ClawHub skill creation.

## Best Possible Solution

Maintainers should decide the PowerShell file policy; if approved, add `.ps1` consistently across shared and CLI allowlists, MIME/content-type normalization, browser/archive/GitHub/API/Convex validation, raw file serving, moderation expectations, tests, docs, and any security-sensitive spec notes.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `.ps1` is absent from the shared and CLI extension allowlists, so a `.ps1` file with empty or generic MIME metadata fails the browser and Convex publish text-file gates.

## Solution Assessment

Unclear until maintainers make the product/security policy call. If approved, the maintainable fix is a coordinated allowlist, MIME, import/API/CLI, moderation, docs, and test update rather than a publish-form-only change.
