## Summary

Keep open. Current main still does not intentionally accept `.ps1` skill files, the related PowerShell upload report at https://github.com/openclaw/clawhub/issues/897 remains open with additional user context, and ClawHub cleanup policy keeps issues open unless they are PRs already implemented on main.

## What This Changes

Request support for uploading PowerShell `.ps1` files as supporting files during ClawHub skill creation.

## Best Possible Solution

If maintainers approve PowerShell support, add `ps1` consistently across shared and CLI allowlists, MIME normalization, browser/archive/GitHub/API/Convex validation, raw file serving, moderation expectations, tests, docs, and specs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `.ps1` is absent from the shared and CLI extension allowlists, so a `.ps1` file with empty or generic MIME metadata fails the browser and Convex publish text-file gates.

## Solution Assessment

Unclear until maintainers make the product/security policy call. If approved, the maintainable fix is a coordinated allowlist, MIME, import/API/CLI, moderation, docs, and test update rather than a publish-form-only change.
