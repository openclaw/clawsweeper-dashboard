## Summary

Keep open. Current main still rejects `.ps1` through the shared text-file gates, and accepting PowerShell would expand the executable script surface, so this needs maintainer policy/security review rather than cleanup closure.

## What This Changes

The issue requests support for uploading and publishing `.ps1` PowerShell files as skill content.

## Best Possible Solution

Maintainers should decide whether PowerShell scripts are allowed as publishable skill content; if approved, support should be added consistently across schema allowlists, MIME normalization, browser/archive/GitHub import paths, Convex publish/API validation, moderation checks, tests, and docs.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: a `.ps1` file with empty or generic MIME metadata misses `TEXT_FILE_EXTENSION_SET` and then fails the browser and Convex publish text-file gates.

## Solution Assessment

Unclear until maintainers make the policy call. A narrow extension allowlist change would be incomplete without coordinated import, server validation, moderation, test, and docs updates.
