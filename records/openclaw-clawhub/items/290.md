## Summary

Keep open. Current main still does not intentionally accept `.ps1` skill files, the related report at https://github.com/openclaw/clawhub/issues/897 adds concrete user context and a workaround, and this ClawHub issue needs maintainer product/security approval before expanding the publishable executable script surface.

## What This Changes

The issue asks ClawHub to allow `.ps1` PowerShell files during skill creation.

## Best Possible Solution

Maintainers should decide the PowerShell file policy; if approved, add `ps1` consistently across shared and CLI allowlists, MIME normalization, browser/archive/GitHub import paths, Convex publish/API validation, moderation expectations, tests, and docs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `.ps1` is absent from the shared and CLI extension allowlists, so a `.ps1` file with empty or generic MIME metadata fails the browser and Convex publish text-file gates.

## Solution Assessment

Unclear until maintainers make the product/security policy call. A narrow allowlist-only patch would not be the best fix unless import, CLI, moderation, tests, and docs are updated together.
