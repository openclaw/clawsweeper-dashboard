## Summary

Keep open. Current main still does not intentionally accept `.ps1` skill files, the related open report at https://github.com/openclaw/clawhub/issues/897 adds concrete Windows-skill context, and this needs maintainer product/security approval before widening the executable script surface.

## What This Changes

Request to allow PowerShell `.ps1` files during new skill creation and publishing.

## Best Possible Solution

Maintainers should decide the PowerShell file policy; if approved, add `ps1` consistently across shared and CLI allowlists, MIME normalization, browser/archive/GitHub import paths, Convex publish/API validation, moderation expectations, tests, and docs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `.ps1` is absent from the shared and CLI extension allowlists, so a `.ps1` file with empty or generic MIME metadata fails the browser and Convex publish text-file gates.

## Solution Assessment

Unclear until maintainers make the product/security policy call. If approved, the maintainable fix is a coordinated allowlist, MIME, import, CLI, moderation, docs, and test update rather than a publish-form-only change.
