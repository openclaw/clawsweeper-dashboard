## Summary

Keep open. Current main still omits `.ps1` from the text-file allowlists and the browser, archive, Convex publish, and GitHub import paths remain extension/MIME gated; the related open report #897 adds useful user context, so this needs a maintainer policy/security decision rather than cleanup closure.

## What This Changes

Support uploading PowerShell `.ps1` scripts as files during new skill creation.

## Best Possible Solution

Maintainers should decide whether PowerShell scripts are allowed as publishable skill content; if approved, add support consistently across shared/CLI allowlists, MIME normalization, browser validation, archive expansion, Convex publish/API validation, GitHub import, moderation expectations, tests, and docs.

## Reproduction Assessment

Yes. A `.ps1` file with empty or generic MIME metadata follows the current publish path into `isTextFile`, where the extension is not in `TEXT_FILE_EXTENSION_SET`; the Convex publish path has the same text gate.

## Solution Assessment

Unclear until maintainers make the policy call. A narrow allowlist-only change would be incomplete because upload/import/API behavior and static moderation expectations need to be aligned.
