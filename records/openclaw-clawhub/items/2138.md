## Best Possible Solution

## Review Findings

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve the public export boundary:** `convex/skills.ts:10413`
  - body: The export query pages `skillSearchDigest` through `by_active_created`, whose key only constrains `softDeletedAt`, then the handler exports storage blobs directly. That bypasses the public `toPublicSkill`/`isPublicSkillDoc` and download moderation checks, so hidden, removed, or malware-blocked rows can be bulk-exported if their digest is present; build this from the same public visibility predicate, or land an explicit staff-only contract with docs and audit expectations.
  - confidence: 0.9
- **[P1] Page mirror exports by update time:** `convex/skills.ts:10413`
  - body: The PR describes hourly incremental mirror sync, but this query ranges over `createdAt` using `by_active_created`. New versions, metadata edits, moderation changes, and stat/owner changes to existing skills will not reappear in later created-date windows, leaving mirrors stale; the export cursor needs an `updatedAt`-based contract.
  - confidence: 0.88
- **[P1] Rate-limit failed export authentication:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: `requireExportAuth` runs before `applyRateLimit`, so missing or invalid tokens return 401 without consuming the export bucket. That differs from the documented auth-aware rate-limit model where missing/invalid tokens fall back to IP enforcement, and it leaves the expensive auth path open to unthrottled failed attempts.
  - confidence: 0.86
- **[P2] Add aggregate bounds before building the ZIP:** `convex/httpApiV1/skillsV1.ts:1693-1740`
  - body: The handler can select up to 1000 skills, fetch every file blob, convert each blob to memory, and then build one ZIP without checking total bytes, file count, or compression output size. A valid export request can exceed Convex action memory/time limits; enforce a batch byte/file ceiling before reading and zipping blobs.
  - confidence: 0.84
