Full review comments:

- **[P1] Keep export on the public visibility path:** `convex/skills.ts:10411-10420`
  - body: `listByDateRange` pages raw `skillSearchDigest` rows using only the soft-delete prefix and the handler then reads latest-version blobs directly. That skips the existing public visibility checks, so hidden, moderation-blocked, or suspicious skills can be included in the bulk archive.
  - confidence: 0.94
- **[P1] Use an update cursor for mirror sync:** `convex/skills.ts:10408-10413`
  - body: The endpoint is intended for hourly mirror sync, but the cursor is built on `createdAt`. Republished, renamed, restored, or otherwise updated older skills keep their original creation time, so an incremental date window can miss changed content and leave mirrors stale.
  - confidence: 0.9
- **[P1] Throttle failed export auth attempts:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: `requireExportAuth` runs before `applyRateLimit`, so missing or invalid bearer tokens return before consuming the export/IP bucket. Current rate-limit docs say failed tokens fall back to IP enforcement, and this ordering leaves invalid-token attempts outside that protection.
  - confidence: 0.9
- **[P1] Bound the archive before buffering blobs:** `convex/httpApiV1/skillsV1.ts:1693-1741`
  - body: The handler allows up to 1000 skills, fetches all storage blobs, converts each blob to an ArrayBuffer, and ZIPs the result without an aggregate byte or file-count budget. One authorized request can exceed Convex action memory or CPU limits.
  - confidence: 0.86
- **[P1] Reserve a non-slug export namespace:** `convex/http.ts:76-80`
  - body: Registering `GET /api/v1/skills/export` as an exact route preempts the documented `GET /api/v1/skills/{slug}` route for a valid or grandfathered skill slug named `export`. Use a reserved namespace such as `/-/export`, or add an explicit slug reservation and migration plan.
  - confidence: 0.84
