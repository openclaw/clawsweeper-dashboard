Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Preserve the public and download visibility boundary:** `convex/skills.ts:10411-10413`
  - body: `listByDateRange` pages `skillSearchDigest.by_active_created` directly and the handler then reads internal version docs and storage blobs. That bypasses the current `toPublicSkill`/download checks that hide non-active moderation states and block `blocked.malware`, so the export can include content public listings and ZIP downloads deliberately withhold.
  - confidence: 0.95
- **[P1] Use update/tombstone pagination for mirror sync:** `convex/skills.ts:10407-10413`
  - body: The export cursor is based on `createdAt`, so an hourly mirror only sees newly created skills. Republished old skills, renamed skills, moderated or removed skills, and other updates to existing rows will not be exported again, leaving downstream mirrors stale without any error.
  - confidence: 0.9
- **[P1] Reserve or move the export route before taking the slug:** `convex/http.ts:76-80`
  - body: `/api/v1/skills/export` currently belongs to the documented `/api/v1/skills/{slug}` shape, and `export` is not in the skill slug reserved list. Registering this exact route can make an existing or future skill named `export` unreachable through the API unless the slug is reserved with a migration/compatibility plan or the endpoint uses a non-slug namespace.
  - confidence: 0.88
- **[P2] Throttle failed export auth before token lookup work:** `convex/httpApiV1/skillsV1.ts:1570-1575`
  - body: The handler calls `requireExportAuth` before `applyRateLimit`, so invalid tokens and valid non-admin tokens return before consuming the export/IP bucket. Existing rate-limit docs say missing or invalid tokens fall back to IP enforcement; keep that behavior here so this privileged route cannot be used for unthrottled token lookup pressure.
  - confidence: 0.88
- **[P1] Cap aggregate export bytes before buffering blobs:** `convex/httpApiV1/skillsV1.ts:1693-1741`
  - body: An authorized request can select up to 1000 skills, fetch every file blob, convert each to an ArrayBuffer, and then build a copied ZIP without an aggregate byte or file-count cap. Large valid skill versions can push the action over memory or response limits, so enforce a total export budget before reading and zipping blobs.
  - confidence: 0.86
