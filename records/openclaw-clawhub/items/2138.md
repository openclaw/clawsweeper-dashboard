Full review comments:

- **[P1] Filter exports through the public visibility boundary:** `convex/skills.ts:10413-10421`
  - body: This query only constrains `softDeletedAt` through `by_active_created`, then the handler reads each `latestVersionId` blob directly for the ZIP.   - confidence: 0.92
- **[P1] Avoid shadowing skill slugs with the export route:** `convex/http.ts:77-79`
  - body: `/api/v1/skills/export` sits inside the documented `/api/v1/skills/{slug}` namespace, and `export` is not reserved by the current slug validator. After this route lands, a skill with slug `export` would return the export endpoint or auth error instead of its skill detail response.
  - confidence: 0.9
- **[P2] Throttle export attempts before token lookup:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: The handler runs `requireExportAuth` before `applyRateLimit`, so missing, invalid, and non-admin tokens return before the export bucket is consumed. Invalid-token traffic can still force token hashing and token/user lookups outside the intended throttling model for this expensive endpoint.
  - confidence: 0.88
- **[P2] Enforce an aggregate export size budget:** `convex/httpApiV1/skillsV1.ts:1693-1741`
  - body: The endpoint may select 1000 skills, fetch every file blob, convert every blob to an `ArrayBuffer`, and then build one ZIP, but it never caps total files or bytes. A set of large but valid skills can push the action over memory/runtime limits and fail the export path.
  - confidence: 0.86
