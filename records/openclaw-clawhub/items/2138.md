Full review comments:

- **[P1] Preserve the public export boundary:** `convex/skills.ts:10413`
  - body: The export query returns raw `skillSearchDigest` rows using only the `softDeletedAt` prefix, so hidden, removed, or moderation-blocked skills can be included in a mirror ZIP. Existing docs say third-party mirrors must stay within public API visibility; either reuse the public filtering/download boundary or make this an explicitly approved staff-only export with audit and docs.
  - confidence: 0.92
- **[P1] Page mirror exports by update time:** `convex/skills.ts:10413`
  - body: The PR body describes hourly incremental mirror sync, but `by_active_created` only catches newly created skills. Republishing an existing skill changes its latest version and `updatedAt`, not `createdAt`, so the mirror would miss updates after the first full crawl.
  - confidence: 0.94
- **[P1] Rate-limit failed export authentication:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: `requireExportAuth` runs before `applyRateLimit`, so missing or invalid tokens return 401 without consuming the export IP bucket. Current API behavior falls missing/invalid tokens back to IP enforcement, and this privileged endpoint should keep that failed-auth throttling.
  - confidence: 0.9
- **[P1] Bound total export bytes before zipping:** `convex/httpApiV1/skillsV1.ts:1740-1741`
  - body: The handler fetches every blob for up to 1000 skills, converts each blob to a `Uint8Array`, stores all entries, and then builds the final ZIP without a total-size cap. A single valid export can exhaust action memory on large skills; enforce aggregate limits or page by byte budget before materializing the archive.
  - confidence: 0.86
