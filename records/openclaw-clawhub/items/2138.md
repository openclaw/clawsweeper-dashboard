The branch adds an admin-only `GET /api/v1/skills/export` endpoint that pages skill digests by date, reads latest version blobs, returns a merged ZIP, and exposes a new schema route constant.

## Best Possible Solution

Define a maintainer-approved mirror export contract first; if approved, implement it with the existing public visibility/download gates, pre-auth rate limiting for failures, reserved archive metadata paths, byte budgets or streaming, docs, tests, and real runtime proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Filter exports through public visibility gates:** `convex/skills.ts:10411-10413`
  - body: `listByDateRange` pages raw `skillSearchDigest` rows through `by_active_created` and the export handler then reads blobs for every returned row. Existing public catalog paths map through `toPublicSkill`/`isPublicSkillDoc`, which blocks soft-deleted, hidden/removed, and malware-blocked skills, and the docs explicitly tell third-party directories not to mirror hidden/private/moderation-blocked content outside the public API surface. Reuse that visibility gate before exporting files.
  - confidence: 0.94
- **[P2] Rate-limit export attempts before strict auth:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: The handler calls `requireExportAuth` before `applyRateLimit`, so missing or invalid tokens return 401 without consuming the export/IP bucket, and invalid bearer tokens still drive token-hash and token lookup work. The documented model says missing/invalid tokens fall back to IP enforcement; apply the export limiter before the strict admin check or otherwise consume a failure bucket.
  - confidence: 0.88
- **[P2] Reserve archive metadata paths before adding files:** `convex/httpApiV1/skillsV1.ts:1757-1759`
  - body: A published skill can have a valid `_meta.json` file because the publish sanitizer only rejects empty, traversal, and backslash paths. This handler then appends its own `${slug}/_meta.json`, and `buildMergedExportZip` throws on the duplicate path, turning a valid export page into a 500 after the blob reads. Reserve or namespace injected metadata paths and add a regression test.
  - confidence: 0.86
- **[P2] Bound total export bytes before assembling the ZIP:** `convex/httpApiV1/skillsV1.ts:1740-1741`
  - body: The endpoint can select 1000 skills, fetch every file blob, convert each blob to a `Uint8Array`, and then zip the whole page in memory without a total input or output byte budget. A page of individually valid skill bundles can exceed action memory or response-size limits; cap total bytes or use a smaller/streamed export contract before returning the archive.
  - confidence: 0.82
