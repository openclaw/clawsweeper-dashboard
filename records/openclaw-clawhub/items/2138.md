Full review comments:

- **[P1] Avoid shadowing skill detail for the export slug:** `convex/http.ts:77`
  - body: `export` is not reserved by the current skill slug validator, but this exact route is registered before the `/api/v1/skills/` path-prefix router. A valid skill at `/api/v1/skills/export` would stop resolving through the documented detail endpoint, so put this under a reserved namespace such as `/api/v1/skills/-/export` or reserve/migrate the slug explicitly.
  - confidence: 0.9
- **[P1] Keep export behind the public visibility gates:** `convex/skills.ts:10411-10419`
  - body: `listByDateRange` pages active digest rows and the handler reads raw latest-version blobs, but it never applies the same public visibility and moderation/download checks used by list, detail, and download APIs. This can archive hidden, removed, pending-scan, malware-blocked, or otherwise non-public files for a bulk mirror.
  - confidence: 0.93
- **[P1] Rate-limit export attempts before auth rejects:** `convex/httpApiV1/skillsV1.ts:1571-1575`
  - body: The handler returns from `requireExportAuth` before calling `applyRateLimit`, so missing, invalid, and non-admin tokens do not consume the export bucket. That leaves the new sensitive endpoint easier to brute-force or hammer than the documented auth-aware API behavior.
  - confidence: 0.88
- **[P1] Page mirror sync by updated time:** `convex/skills.ts:10408-10413`
  - body: The PR describes hourly incremental mirror sync, but the query bounds `by_active_created`; existing skills updated during the requested window will be missed because their `createdAt` stays outside the window. Use the updated-time index and cursor semantics for the incremental export contract.
  - confidence: 0.86
- **[P1] Enforce an aggregate export size budget:** `convex/httpApiV1/skillsV1.ts:1693`
  - body: For each page the handler loads all selected blobs into memory and then builds the final ZIP without a total byte or file-count cap. A few large or many-file skills can exceed Convex action memory/runtime and make the endpoint unreliable, even for authorized callers.
  - confidence: 0.82
