Full review comments:

- **[P1] Filter exports through the public visibility boundary:** `convex/skills.ts:10411-10419`
  - body: The export query reads active digest rows by `softDeletedAt` only and then exports the latest blobs. That bypasses the current public/download checks that hide or block moderation-hidden, removed, pending-scan, suspicious, and malware-blocked skills, so a mirror token could bulk copy content normal public routes intentionally withhold.
  - confidence: 0.93
- **[P1] Use update and tombstone semantics for mirror sync:** `convex/skills.ts:10408-10413`
  - body: Paginating by `by_active_created` only catches newly-created skills. Existing skills that are republished, renamed, moderated, removed, or restored after their original creation date will not appear in hourly incremental exports, leaving mirrors silently stale unless the API uses an updated cursor plus deletion/moderation tombstones or documents a full-snapshot-only contract.
  - confidence: 0.9
- **[P1] Move the export route out of the skill slug namespace:** `packages/schema/src/routes.ts:30`
  - body: `/api/v1/skills/{slug}` is an existing documented API shape and `export` is not reserved as a skill slug. Adding `/api/v1/skills/export` as an exact route means a valid current or future `export` skill cannot be fetched through its canonical API URL, so the endpoint needs a non-conflicting namespace or an explicit reservation/migration plan.
  - confidence: 0.87
- **[P2] Throttle failed export auth attempts:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: `requireExportAuth` runs before the export rate limit, so invalid bearer tokens and non-admin valid tokens can return before consuming the export bucket. Put a pre-auth/IP throttle in front of the token lookup or otherwise ensure failed attempts on this privileged public route are rate-limited.
  - confidence: 0.84
- **[P2] Cap the total export size before zipping:** `convex/httpApiV1/skillsV1.ts:1693`
  - body: The handler fetches blobs for up to 1000 skills, materializes every file into memory, stores all ZIP entries, and then builds the ZIP without any aggregate byte, file-count, or response-size cap. A large authorized page can exceed action memory/response limits, so enforce a total budget before reading and compressing the full batch.
  - confidence: 0.86
