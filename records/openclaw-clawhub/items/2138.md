Full review comments:

- **[P1] Filter exports through public visibility gates:** `convex/skills.ts:10411-10413`
  - body: `listByDateRange` returns raw digest rows and the handler packages their latest-version blobs, but current public surfaces use `toPublicSkill`/`isPublicSkillDoc` to hide non-active and malware-blocked skills. Filter through the public gate before exporting files so this does not create a new path around moderation.
  - confidence: 0.92
- **[P2] Rate-limit invalid export auth before token lookup:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: `requireExportAuth` runs before `applyRateLimit`, so missing or invalid tokens return before consuming the export IP bucket even though the documented model says missing/invalid tokens fall back to IP enforcement. Apply the anonymous export limit before or alongside token validation.
  - confidence: 0.9
- **[P2] Add a total byte budget to export pages:** `convex/httpApiV1/skillsV1.ts:1740-1741`
  - body: The export has a 1000-item cap but no byte cap, then reads every blob into memory and zips the full page. Since one skill bundle may be up to 50 MB, a valid page can exceed action memory or response limits; cap total bytes and paginate or fail cleanly.
  - confidence: 0.88
- **[P2] Reserve generated export metadata paths:** `convex/httpApiV1/skillsV1.ts:1757-1759`
  - body: The export always adds `${slug}/_export_skill_meta.json`, while current publish validation allows a skill file with that name. That duplicate makes `buildMergedExportZip` throw and abort the whole page; reserve, rename, or namespace generated metadata and cover it with a regression test.
  - confidence: 0.84
