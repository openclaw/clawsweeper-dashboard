## Best Possible Solution

- **[P1] Preserve the public export boundary:** `convex/skills.ts:10413`
  - body: This export query pages raw `skillSearchDigest` rows through the active-created index and the handler then reads version files directly, bypassing the existing `isPublicSkillDoc`/download moderation gates.   - confidence: 0.91
- **[P1] Page mirror exports by update time:** `convex/skills.ts:10413`
  - body: The PR's stated hourly mirror sync needs to pick up new versions of existing skills, but this query only ranges over `createdAt`. Existing skills keep their creation time when republished, so incremental exports can miss updates; use the updated-time digest index and matching cursor contract for mirror sync pages.
  - confidence: 0.94
- **[P1] Rate-limit failed export authentication:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: `requireExportAuth` runs before `applyRateLimit`, so missing or invalid bearer tokens return 401 before consuming the export IP bucket. That contradicts the current rate-limit model for missing/invalid tokens and leaves the new auth path open to unthrottled token-hash and lookup attempts.
  - confidence: 0.9
- **[P1] Bound the merged export size before reading blobs:** `convex/httpApiV1/skillsV1.ts:1693`
  - body: A request can select up to 1000 skills, then fetch every file blob, convert each to an `ArrayBuffer`, store all entries, and build the final ZIP in memory. Since skill publishes can be tens of MB per bundle, add an aggregate byte/file guard before blob reads or split pages by byte budget to avoid action memory failures.
  - confidence: 0.88
