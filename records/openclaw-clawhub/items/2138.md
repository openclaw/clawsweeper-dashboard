## Summary

## What This Changes

The PR adds an admin-only `GET /api/v1/skills/export` API that bulk-zips skill files by date range with pagination, a new rate-limit tier, route/schema updates, and ZIP path validation.

## Best Possible Solution

Maintainers should first decide whether bulk mirror export is a supported product and security boundary; if approved, land a narrower endpoint with public/moderation filtering, a non-conflicting route, updated-time pagination, pre-auth throttling, aggregate budgets, tests, specs/docs, and live proof.

- **[P1] Filter export rows through the public visibility boundary:** `convex/skills.ts:10411-10420`
  - body: The export query scans `skillSearchDigest.by_active_created` with only the soft-delete prefix and then exports latest-version blobs for every returned digest. That bypasses the public API's `toPublicSkill`/`isPublicSkillDoc` boundary and the documented warning not to mirror hidden or moderation-blocked content, so a third-party mirror could receive content current public endpoints would not serve.
  - confidence: 0.91
- **[P1] Throttle failed export auth attempts:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: `exportSkillsV1Handler` returns missing, invalid, and non-admin auth failures before calling `applyRateLimit`. That leaves the new expensive/admin-only endpoint without per-IP throttling on the exact traffic most likely to be noisy or abusive; apply a cheap export limiter before token lookup or add a dedicated failed-auth limiter.
  - confidence: 0.88
- **[P1] Use update-time pagination for mirror sync:** `convex/skills.ts:10407-10413`
  - body: The PR's stated hourly mirror-sync workflow needs changed skills, but `listByDateRange` pages on `createdAt` via `by_active_created`. Any skill created before the requested window and later republished, edited, moderated, or otherwise updated will be missed by incremental sync.
  - confidence: 0.9
- **[P1] Cap aggregate export bytes and file count:** `convex/httpApiV1/skillsV1.ts:1693-1741`
  - body: The handler fetches all selected storage blobs, converts each blob to an ArrayBuffer, accumulates `zipEntries`, then builds one ZIP, with only a skill-count limit. Large skills or many files can exceed Convex action memory/runtime limits even for a single request, and the generous admin rate limit amplifies that failure mode.
  - confidence: 0.84
- **[P1] Avoid shadowing the `export` skill slug:** `convex/http.ts:76-80`
  - body: Registering an exact `/api/v1/skills/export` route inside the documented `/api/v1/skills/{slug}` namespace makes a valid existing or legacy slug named `export` unreachable through the skill-detail API. Use a reserved `-/...` collection route or reserve/migrate the slug with an explicit compatibility plan.
  - confidence: 0.78
