Full review comments:

- **[P1] Preserve the public export boundary:** `convex/skills.ts:10413`
  - body: The export query reads active `skillSearchDigest` rows and the handler fetches latest-version blobs directly, while current public list APIs still pass entries through `toPublicSkill`/`isPublicSkillDoc` and the existing review thread already flagged suspicious-skill exposure. Please either reuse the public-safe boundary or make the privileged export contract explicit and reviewed.
  - confidence: 0.92
- **[P1] Page mirror exports by update time:** `convex/skills.ts:10398-10409`
  - body: The PR describes hourly incremental mirror sync, but this query ranges on `by_active_created`. New versions or metadata changes to existing skills update `updatedAt`, not `createdAt`, so an incremental sync by date window would miss already-created skills after they change.
  - confidence: 0.9
- **[P1] Rate-limit failed export auth before validation:** `convex/httpApiV1/skillsV1.ts:1571-1575`
  - body: `requireExportAuth` runs before `applyRateLimit`, so missing or invalid tokens return 401/403 without consuming the `export` IP bucket. Failed export auth needs to be throttled before token validation or by an equivalent failed-auth bucket.
  - confidence: 0.89
- **[P1] Bound aggregate export bytes before reading blobs:** `convex/httpApiV1/skillsV1.ts:1740-1741`
  - body: The handler queues every file for up to 1000 skills, reads each blob with `arrayBuffer()`, and then builds one merged ZIP without a total byte/file guard. Check `version.files[].size` and cap aggregate export size before storage reads so one valid request cannot exhaust action memory.
  - confidence: 0.86
