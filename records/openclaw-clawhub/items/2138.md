Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Filter exports through public visibility gates:** `convex/skills.ts:10411-10413`
  - body: The export page comes straight from `skillSearchDigest` with only `softDeletedAt` in the cursor prefix, then the handler reads latest-version blobs directly. That skips the existing `toPublicSkill`/download moderation gates, so hidden, suspicious, or malware-blocked rows can be copied into the ZIP; keep the export on the same public visibility boundary unless maintainers explicitly approve a staff-only archive boundary.
  - confidence: 0.93
- **[P1] Page mirror sync by updates, not creation time:** `convex/skills.ts:10408-10413`
  - body: The PR describes hourly incremental mirror sync, but this query bounds the catalog by `createdAt`. Any existing skill republished, restored, renamed, or otherwise updated after the mirror's last run keeps its old creation date and will be omitted, so the mirror can go stale while `X-Has-More` says the window is complete.
  - confidence: 0.91
- **[P1] Move export outside the slug route namespace:** `convex/http.ts:76-80`
  - body: Registering `/api/v1/skills/export` reserves a path that currently belongs to `GET /api/v1/skills/{slug}`. A skill whose slug is `export` can no longer be fetched through the documented detail route, so use a reserved namespace such as `/api/v1/skills/-/export` or a separate exports route.
  - confidence: 0.88
- **[P2] Throttle export requests before token auth:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: Failed export auth returns before `applyRateLimit`, so missing or invalid bearer tokens never consume the IP fallback bucket documented for bad tokens and can drive token-hash/user lookups without the export limiter. Apply the export limiter before the auth lookup or explicitly rate-limit failed-auth paths.
  - confidence: 0.9
- **[P1] Add an aggregate export size budget:** `convex/httpApiV1/skillsV1.ts:1740-1780`
  - body: The handler allows up to 1000 skills, reads every file blob into memory, and then builds another full ZIP buffer. A page with large skills or many files can exceed Convex action memory/CPU, so enforce total byte/file limits or switch to a chunked/streamed contract before exposing this endpoint.
  - confidence: 0.84
