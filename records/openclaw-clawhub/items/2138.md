## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Filter exports through the public visibility boundary:** `convex/skills.ts:10411-10420`
  - body: `listByDateRange` returns raw digest rows from `by_active_created` and the handler then exports latest-version blobs for each row. Hidden, removed, malware-blocked, or suspicious rows can still be non-soft-deleted, while current public list/detail paths go through `toPublicSkill`/`isPublicSkillDoc`; bulk export needs the same visibility gate before returning archive content.
  - confidence: 0.92
- **[P1] Use update-based pagination for mirror sync:** `convex/skills.ts:10408-10414`
  - body: The PR advertises hourly incremental mirror sync, but the query ranges over `createdAt`. Any older skill that is republished, renamed, rescanned into a different public state, or otherwise updated after its original creation window will be missed by a mirror polling recent date ranges.
  - confidence: 0.9
- **[P2] Throttle failed export auth attempts:** `convex/httpApiV1/skillsV1.ts:1571-1575`
  - body: The handler calls `requireExportAuth` before `applyRateLimit`, so requests with no token or an invalid token return 401 without consuming the new export bucket. That leaves brute-force and noisy failed-token traffic outside the endpoint's own throttling model.
  - confidence: 0.86
- **[P1] Add aggregate limits before buffering archive files:** `convex/httpApiV1/skillsV1.ts:1693-1741`
  - body: A single allowed request can collect all blob tasks, read each blob into memory, and build one ZIP for up to 1000 skills without a total byte or file-count cap. Large or unusually many files can exceed Convex action memory/time budgets and make the endpoint an availability risk.
  - confidence: 0.88
- **[P1] Avoid stealing the `export` skill slug route:** `convex/http.ts:77-79`
  - body: `/api/v1/skills/export` is currently shaped like `GET /api/v1/skills/{slug}`, and read paths intentionally keep grandfathered reserved slugs readable. Because `export` is not reserved today, adding this exact route can break detail/file access for an existing or legacy skill with that slug; use a reserved collection route such as `/-/export` or add an explicit migration/compatibility plan.
  - confidence: 0.84
