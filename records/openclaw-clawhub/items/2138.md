  - body: `listByDateRange` returns raw `skillSearchDigest` rows from `by_active_created`, while current public surfaces run rows through `toPublicSkill`/`isPublicSkillDoc` and owner visibility checks.   - body: The stated hourly mirror sync needs changes to existing skills, but this range scan is over `createdAt`. Republishing an existing skill updates `updatedAt` and `latestVersionId` without changing `createdAt`, so mirrors will miss new versions outside the skill's original creation window.
  - confidence: 0.9
- **[P1] Rate-limit failed export authentication attempts:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: Export auth runs before `applyRateLimit`, so missing or invalid bearer tokens return 401 without consuming the export IP bucket. That leaves the auth path outside the documented export throttle and makes unauthenticated traffic cheaper than intended.
  - confidence: 0.88
- **[P1] Bound export size before materializing blobs:** `convex/httpApiV1/skillsV1.ts:1740-1741`
  - body: The handler fetches all blobs, converts each to a `Uint8Array`, stores every entry, and then builds one merged ZIP for up to 1000 skills without an aggregate byte cap. A valid admin request can exceed backend memory or stall the function.
  - confidence: 0.87
