## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Keep exports inside the public visibility boundary:** `convex/skills.ts:10411-10420`
  - body: The export page reads `skillSearchDigest` by only `softDeletedAt` and `createdAt`, then fetches raw latest-version blobs. That bypasses the `isPublicSkillDoc`/non-suspicious/moderation checks used by public APIs, so an admin export can include hidden, suspicious, or malware-blocked skill files. Filter through the same public visibility rules before reading blobs.
  - confidence: 0.93
- **[P1] Move the export route out of the slug namespace:** `convex/http.ts:76-80`
  - body: `GET /api/v1/skills/{slug}` is a documented detail route, so `/api/v1/skills/export` is already a valid skill slug path. Registering the exact export route before the skills prefix makes a skill with slug `export` unreachable through the API; use a reserved namespace such as `/api/v1/exports/skills` or `/api/v1/skills/-/export`.
  - confidence: 0.9
- **[P2] Throttle failed export auth attempts:** `convex/httpApiV1/skillsV1.ts:1571-1575`
  - body: `requireExportAuth` runs before `applyRateLimit`, so missing, invalid, and non-admin tokens return 401/403 without consuming the documented export limit. Put the request through rate limiting first, or explicitly consume a bucket for failed auth attempts, so auth probing and valid non-admin tokens cannot bypass export throttling.
  - confidence: 0.88
- **[P2] Page incremental exports by update time:** `convex/skills.ts:10395-10413`
  - body: The PR says mirror clients will do hourly incremental syncs, but the range cursor is built on `by_active_created`. Republished or edited skills keep their old `createdAt`, so a mirror that has already passed that creation window will never receive the updated files.
  - confidence: 0.86
- **[P2] Add an aggregate export byte budget:** `convex/httpApiV1/skillsV1.ts:1740-1741`
  - body: The handler reads every selected blob into memory and then builds a second ZIP buffer, but the only bound is 1000 skills. A few large skills can exceed Convex action memory or stall the endpoint; track total bytes and fail or paginate before buffering the archive.
  - confidence: 0.84
- **[P2] Reserve metadata paths before writing the archive:** `convex/httpApiV1/skillsV1.ts:1757-1759`
  - body: A skill can publish a file named `_export_skill_meta.json`; the handler will then add the same `slug/_export_skill_meta.json` path for metadata and `buildMergedExportZip` throws, failing the whole page. Reject or namespace reserved export metadata paths before accepting user file entries.
  - confidence: 0.8
