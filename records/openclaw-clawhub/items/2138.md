The PR adds an admin-only `/api/v1/skills/export` ZIP export endpoint with route/schema wiring, export auth, rate limits, date-range pagination, and archive helpers.

## Best Possible Solution

Define a maintainer-approved export contract first, then implement it with public visibility/download gates, auth-failure rate limiting, reserved archive metadata paths, byte-budgeted pagination, docs/OpenAPI coverage, tests, and redacted runtime proof.

## Reproduction Assessment

Yes. Source inspection of the PR head is enough to reproduce the review findings: the export query lacks public moderation gates, auth runs before rate limiting, injected `_meta.json` can collide with valid skill files, and blob reads happen before any total byte budget.

## Solution Assessment

No. The proposed endpoint creates a new bulk export trust boundary before the mirror entitlement, visibility, rate-limit, archive-layout, byte-budget, and documentation contracts are settled; the safer path is to define those contracts before implementation.

## Review Findings

Overall confidence: 0.91

Full review comments:

- **[P1] Filter exports through public visibility gates:** `convex/skills.ts:9638-9639`
  - body: `listByDateRange` pages `skillSearchDigest` with only the active-created index, so hidden, pending-scan, removed, or malware-blocked rows can be returned and later exported by direct blob reads. Reuse the public skill/download visibility gates before including files in the archive.
  - confidence: 0.94
- **[P2] Rate-limit export auth failures before token lookup:** `convex/httpApiV1/skillsV1.ts:1503-1507`
  - body: The handler calls `requireExportAuth` before `applyRateLimit`, so missing or invalid bearer attempts return `401` before consuming the export/IP bucket and without rate headers. Move the limiter first and pass those headers through auth failures.
  - confidence: 0.84
- **[P2] Reserve injected metadata paths:** `convex/httpApiV1/skillsV1.ts:1689-1691`
  - body: The export always adds `${slug}/_meta.json`, but uploaded skill files can already use that valid path. When such a skill appears, duplicate detection throws and turns a valid export page into a server error.
  - confidence: 0.82
- **[P2] Bound archive bytes before reading blobs:** `convex/httpApiV1/skillsV1.ts:1625`
  - body: The endpoint can select up to 1000 skills and then read every selected blob into memory before zipping. Add byte-budgeted pagination or a hard cap before fetching blobs so valid catalog pages cannot exhaust runtime memory or response limits.
  - confidence: 0.78
