## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Filter exports through public visibility gates:** `convex/skills.ts:10407-10417`
  - body: `listByDateRange` returns digest rows from `by_active_created` with only `softDeletedAt` constrained, then the handler fetches each latest version's files. Current public APIs run digest rows through `toPublicSkill` / `isPublicSkillDoc`, which also excludes hidden, removed, and malware-blocked skills; the export endpoint needs the same gate before packaging files for a mirror.
  - confidence: 0.91
- **[P2] Rate-limit export attempts before auth lookup:** `convex/httpApiV1/skillsV1.ts:1569-1572`
  - body: The new handler calls `requireExportAuth` before `applyRateLimit`, so missing or invalid Bearer tokens return `401` without consuming the export IP bucket. That bypasses the documented fallback where invalid auth is still IP-enforced and leaves the token lookup path unthrottled.
  - confidence: 0.88
- **[P2] Cap export bytes before reading blobs:** `convex/httpApiV1/skillsV1.ts:1717-1718`
  - body: The handler reads every selected blob into memory and then builds a ZIP with no total byte budget. Because published skills can be much larger than the PR body's average-size estimate, a valid request can exceed action memory or runtime; sum version file sizes first and reject or page by bytes before `blob.arrayBuffer()`.
  - confidence: 0.86
