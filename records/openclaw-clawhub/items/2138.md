## Summary

Keep open. Current main does not implement this export endpoint, and the PR still has blocking review issues: the bulk archive path can include moderated skill files outside the documented public API boundary, auth runs before export rate limiting, and external real-behavior proof is missing.

## What This Changes

The PR adds an admin-authenticated paginated `GET /api/v1/skills/export` endpoint that returns bulk skill files as a merged ZIP, plus route constants, auth/rate-limit helpers, ZIP utilities, and generated schema output.

## Best Possible Solution

Define a maintainer-approved mirror/export contract that reuses existing public visibility and download moderation gates, rate-limits before expensive auth work, and ships with focused regression coverage plus real runtime proof.

## Reproduction Assessment

Yes for the review findings: source inspection of PR head shows the export query uses `by_active_created` and the handler authenticates before rate limiting, while current main defines stricter public/download gates. I did not run the PR branch.

## Solution Assessment

No; the current patch is not the best way to solve it because it creates a new bulk file path without reusing the existing moderation/download contract and performs auth before rate limiting. A safer path is to establish the mirror policy first, filter eligibility before blob reads, and add tests/proof around those guarantees.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Filter export rows through public moderation gates:** `convex/skills.ts:9639`
  - body: The export query pages `skillSearchDigest` through `by_active_created`, which only excludes soft-deleted rows. Hidden, removed, pending-scan, or `blocked.malware` skills can still have a latest version, so this new bulk ZIP path can read and archive files that the public skill and download paths currently block. Filter through the same public/download eligibility rules before fetching version blobs.
  - confidence: 0.93
- **[P2] Rate-limit export requests before token auth:** `convex/httpApiV1/skillsV1.ts:1503-1506`
  - body: `requireExportAuth` runs before `applyRateLimit`, so invalid bearer-token requests perform hashing and token lookups, then return 401 without consuming the export/IP rate-limit bucket. Existing V1 handlers rate-limit first; move export rate limiting ahead of auth and pass the rate headers into auth failure responses.
  - confidence: 0.84
