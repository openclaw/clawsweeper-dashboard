Full review comments:

- **[P1] Filter exports through the public visibility contract:** `convex/skills.ts:10411-10421`
  - body: This scan returns raw `skillSearchDigest` rows and the handler later reads version blobs directly. Current public paths run digests through `toPublicSkill`/owner checks and the docs tell third-party mirrors not to mirror hidden, private, or moderation-blocked content, so export needs to reuse that boundary or an explicitly approved moderation boundary before exposing files in bulk.
  - confidence: 0.94
- **[P1] Use an update cursor for mirror sync:** `convex/skills.ts:10413`
  - body: The endpoint paginates by `createdAt`, but the stated hourly mirror use case needs changes to existing skills too. A republish, rename, soft-hide, or metadata update on an older skill will not appear in the next created-date window, so mirrors can silently miss updates or removals; use `updatedAt` or a durable change feed instead.
  - confidence: 0.9
- **[P2] Apply export throttling before token auth:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: Missing or invalid Bearer tokens return from `requireExportAuth` before `applyRateLimit`, so failed-token attempts do not consume the new export throttle or get rate-limit headers. Put the export rate limit before auth, matching the existing HTTP handlers' fail-closed throttling pattern.
  - confidence: 0.88
- **[P1] Add aggregate limits before buffering export blobs:** `convex/httpApiV1/skillsV1.ts:1693-1741`
  - body: One allowed request can select 1000 skills, fetch every blob, convert them to `arrayBuffer`, and zip the result without a total byte or file-count cap. Since a single skill bundle can be up to 50MB, this can exhaust action memory; enforce a strict aggregate budget before storage reads and ZIP assembly.
  - confidence: 0.89
- **[P2] Avoid shadowing a valid skill slug:** `convex/http.ts:77-79`
  - body: `/api/v1/skills/{slug}` currently routes any first segment to skill detail, and `export` is not reserved as a skill slug. Adding an exact `/api/v1/skills/export` route can steal that URL from an existing or grandfathered skill, so use a reserved namespace such as `/api/v1/skills/-/export` or add an explicit migration/reservation plan.
  - confidence: 0.83
