Keep open and do not merge as-is: the bulk export idea is useful, but this PR adds a privileged full-content export boundary with missing real behavior proof and source-visible security, sync-correctness, compatibility, and availability blockers.

## What This Changes

The branch adds an admin-gated `GET /api/v1/skills/export` HTTP endpoint that builds paginated ZIP exports from skill digests and blobs, plus auth, rate-limit, route, and ZIP helper changes.

## Best Possible Solution

Maintain the existing public API boundary until maintainers approve a mirror export contract, then implement it on a reserved route with public visibility filtering, updated-time sync semantics, failed-auth throttling, aggregate budgets, tests, and redacted live proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Preserve public visibility filtering before exporting blobs:** `convex/skills.ts:10413`
  - body: `listByDateRange` returns raw `skillSearchDigest` rows from `by_active_created`, and the handler then reads internal versions/blobs for every row. That bypasses the public list/detail filtering used by current APIs, so hidden or moderation-blocked skills can be exported in bulk.
  - confidence: 0.94
- **[P1] Page mirror sync by updates, not creation time:** `convex/skills.ts:10413`
  - body: The PR's stated hourly mirror-sync workflow cannot be correct when the range scan is on `createdAt`: updates to older skills inside the sync window will not be exported. Use an update-aware cursor contract or separate explicit modes before relying on this for incremental mirrors.
  - confidence: 0.9
- **[P2] Consume export rate limits before auth rejection:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: `requireExportAuth` runs before `applyRateLimit`, so missing or invalid tokens return before the export bucket is consumed. That leaves failed export auth attempts outside the intended throttling model and inconsistent with the documented fallback-to-IP rate-limit behavior.
  - confidence: 0.9
- **[P1] Bound aggregate export bytes before buffering the ZIP:** `convex/httpApiV1/skillsV1.ts:1740-1741`
  - body: The handler fetches all selected blobs and calls `arrayBuffer()` before zipping, with no total byte or file-count guard for up to 1000 skills. Since a published skill can contain far more than the assumed average, one allowed request can exceed Convex action memory or stall the function.
  - confidence: 0.87
- **[P2] Avoid claiming a valid skill slug as the export route:** `packages/schema/src/routes.ts:30`
  - body: `/api/v1/skills/export` is currently the documented detail path for a skill whose slug is `export`, and that slug is not reserved in main. Adding this exact route can break an existing or grandfathered skill; use a reserved namespace such as `/-/export` or add a deliberate migration/reservation plan.
  - confidence: 0.83
