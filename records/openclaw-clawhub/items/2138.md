Overall confidence: 0.9

Full review comments:

- **[P1] Avoid shadowing valid skill slugs:** `convex/http.ts:77-79`
  - body: `/api/v1/skills/export` sits inside the existing `/api/v1/skills/{slug}` namespace, and `export` is not a reserved skill slug. That means an existing or future skill named `export` would no longer resolve through the documented detail/file endpoints; put bulk export under a reserved non-slug namespace such as `/api/v1/skills/-/export` and add a route/reservation test.
  - confidence: 0.89
- **[P1] Keep export inside public visibility boundaries:** `convex/skills.ts:10411-10420`
  - body: The export query pages raw `skillSearchDigest` rows by `by_active_created` and the handler then reads internal versions/blobs directly. This bypasses the `isPublicSkillDoc`/download checks that hide non-active moderation states and `blocked.malware`, so the archive can mirror content the public APIs and docs say must stay outside third-party mirrors.
  - confidence: 0.92
- **[P1] Count failed export auth against a throttle:** `convex/httpApiV1/skillsV1.ts:1571-1575`
  - body: `requireExportAuth` runs before `applyRateLimit`, so missing, invalid, and non-admin token attempts return without consuming the export bucket. Invalid token probes still exercise auth/token lookup work; either rate-limit first with the existing invalid-token-as-IP behavior or add an explicit auth-failure throttle.
  - confidence: 0.84
- **[P1] Use updated windows for incremental mirror sync:** `convex/skills.ts:10407-10413`
  - body: The PR says mirrors can do hourly incremental syncs, but `listByDateRange` pages `by_active_created` and compares the range to `createdAt`. Republishing an existing skill updates `updatedAt` while leaving `createdAt` old, so those updates are skipped by the export window.
  - confidence: 0.87
- **[P1] Add an aggregate export budget before reading blobs:** `convex/httpApiV1/skillsV1.ts:1693`
  - body: The `limit` caps skill count, but the handler queues every file, fetches all blobs, converts each to an ArrayBuffer, and then creates the final ZIP without a total byte/file cap. A single authorized request for large skills can exhaust Convex action memory; enforce an aggregate budget and return a clear error before loading the archive.
  - confidence: 0.83
