- **[P1] Filter exports through public visibility gates:** `convex/skills.ts:9637-9639`
  - body: `listByDateRange` pages `skillSearchDigest` by active-created index without applying the existing public skill/download visibility gates. Hidden, pending-scan, removed, or malware-blocked rows can be selected and then exported through direct storage blob reads.
  - confidence: 0.94
- **[P2] Rate-limit export auth failures before token lookup:** `convex/httpApiV1/skillsV1.ts:1503-1506`
  - body: The handler calls `requireExportAuth` before `applyRateLimit`, so missing or invalid bearer attempts return before consuming the export/IP bucket.   - confidence: 0.86
- **[P2] Reserve metadata paths instead of colliding with skill files:** `convex/httpApiV1/skillsV1.ts:1689-1691`
  - body: The export always injects `${slug}/_meta.json`, but published skill files can already include that path. When such a skill appears in a page, `buildMergedExportZip` throws on the duplicate path and turns a valid export request into a 500.
  - confidence: 0.83
- **[P2] Bound export bytes before reading blobs:** `convex/httpApiV1/skillsV1.ts:1625`
  - body: The endpoint can select up to 1000 skills and then reads every selected blob into memory before zipping. Since valid skill bundles may be up to 50MB, a normal page can exceed action memory or response limits; paginate by a byte budget or lower the limit before fetching blobs.
  - confidence: 0.8
