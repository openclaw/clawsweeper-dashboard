## Summary

Keep open: this is a new security-sensitive bulk mirror/export endpoint, not something already present on current main, and the current diff still bypasses public visibility/download gates while lacking inspectable real behavior proof.

## What This Changes

The branch adds an admin-only `GET /api/v1/skills/export` ZIP endpoint, export auth and rate limiting, date-range pagination over `skillSearchDigest`, merged archive helpers, and a route constant.

## Best Possible Solution

Define a maintainer-approved mirror/export contract that reuses public visibility and download gates, rate-limits invalid auth at the IP bucket, avoids archive metadata collisions, caps export bytes, documents/tests the endpoint, and includes redacted runtime proof.

## Reproduction Assessment

Yes for the review findings by source inspection: the PR query can select moderated digest rows, auth failures return before rate limiting, and valid skill-owned `_meta.json` or large pages can break the archive path. I did not run the PR branch.

## Solution Assessment

No; this is not yet the best way to solve mirror sync because it introduces a new bulk export trust boundary before public-filter, entitlement, rate-limit, archive-layout, and size contracts are settled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Filter export rows through public moderation gates:** `convex/skills.ts:9639`
  - body: `listByDateRange` pages `skillSearchDigest` by `softDeletedAt` and `createdAt` only. Hidden, removed, pending-scan, or malware-blocked rows can still be returned and then have their blobs read directly, bypassing the public list/download moderation gates.
  - confidence: 0.94
- **[P2] Rate-limit export auth failures before token lookup:** `convex/httpApiV1/skillsV1.ts:1503-1507`
  - body: The handler authenticates first and returns missing/invalid bearer responses before `applyRateLimit`, so those attempts never consume the export/IP bucket. Existing API behavior falls invalid auth back to IP enforcement; move the limiter ahead of export auth and preserve rate headers on auth failures.
  - confidence: 0.84
- **[P2] Avoid colliding with skill-owned metadata files:** `convex/httpApiV1/skillsV1.ts:1689-1690`
  - body: The export always writes `${slug}/_meta.json`, but published skill files can already use that path. When such a skill appears in a page, `buildMergedExportZip` detects the duplicate and throws, turning a valid export request into a 500.
  - confidence: 0.82
- **[P2] Bound export size before reading blobs:** `convex/httpApiV1/skillsV1.ts:1625`
  - body: The endpoint builds a page of up to 1000 skills and reads all selected blobs into memory before zipping. Since valid published skill bundles can be up to 50MB, a normal page can exhaust memory or response limits; add a byte budget or paginate by total bytes before fetching blobs.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The diff introduces a security-sensitive bulk archive endpoint that can expose moderated skill blobs and lets unauthorized attempts bypass endpoint rate limiting.

Concerns:

- **[high] Bulk export can include moderation-blocked skills:** `convex/skills.ts:9639`
  - body: The export query only constrains `softDeletedAt` via `by_active_created`; it does not enforce current public visibility rules or the single-skill download blocks for malware, pending-scan, removed, or hidden skills before reading storage blobs.
  - confidence: 0.94
- **[medium] Unauthorized export attempts bypass the limiter:** `convex/httpApiV1/skillsV1.ts:1503`
  - body: `requireExportAuth` runs before `applyRateLimit`, so missing or invalid bearer-token attempts can trigger token parsing/hash/lookup work without consuming the endpoint's export/IP rate-limit bucket.
  - confidence: 0.84
