Full review comments:

- **[P1] Filter exports through the public visibility contract:** `convex/skills.ts:10411-10421`
  - body: `listByDateRange` returns raw `skillSearchDigest` rows after only constraining `softDeletedAt`, then the handler reads latest-version blobs directly. Current public skill responses pass through `toPublicSkill`/`isPublicSkillDoc`, which also excludes hidden/removed and malware-blocked skills; this export path can leak content the public catalog is not supposed to mirror.
  - confidence: 0.93
- **[P1] Paginate mirror exports by updates, not creation time:** `convex/skills.ts:10408-10413`
  - body: The PR's stated hourly mirror sync cannot be correct if the export range walks `by_active_created`: an existing skill republished today keeps its old `createdAt`, so it will be missed by an hourly `startDate`/`endDate` crawl. Use `updatedAt` or an explicit change feed for incremental sync.
  - confidence: 0.95
- **[P1] Throttle export requests before validating bearer tokens:** `convex/httpApiV1/skillsV1.ts:1571-1575`
  - body: The export handler calls `requireExportAuth()` before `applyRateLimit()`, so missing or invalid tokens return 401 without consuming the export IP bucket. Existing API docs say missing/invalid tokens fall back to IP enforcement, and this ordering leaves token-guessing and auth lookup load outside the rate-limit model.
  - confidence: 0.9
- **[P2] Bound total archive work before reading blobs:** `convex/httpApiV1/skillsV1.ts:1693-1741`
  - body: A single allowed request can fetch all blob handles, call `ctx.storage.get()` for every file, materialize each blob with `arrayBuffer()`, and then compress the whole archive with no total byte or file-count budget. Large skills or many files can exhaust Convex action memory/time before rate limiting helps.
  - confidence: 0.88
- **[P2] Avoid shadowing the skill slug route:** `convex/http.ts:77-79`
  - body: Registering `/api/v1/skills/export` as an exact route before the `/api/v1/skills/` prefix router means a skill whose slug is `export` can no longer be fetched through the documented `GET /api/v1/skills/{slug}` endpoint. Put bulk export under a reserved namespace such as `/-/export`, or prove and enforce that this slug is impossible for existing and future skills.
  - confidence: 0.86
