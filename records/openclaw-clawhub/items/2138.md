## What This Changes

The PR adds an admin-authenticated `GET /api/v1/skills/export` endpoint for paginated merged ZIP exports, plus route, schema, auth, ZIP, and rate-limit helper changes.

## Best Possible Solution

Have maintainers decide whether mirror bulk export is desired; if yes, land a scoped API that exports only public/downloadable content, applies rate limits before auth failures, documents access, and includes tests plus redacted live proof.

## Reproduction Assessment

## Solution Assessment

No. The current patch is not the best path unless maintainers first renew the mirror/export product direction; if accepted, the implementation should reuse public visibility/download gates and the existing rate-before-auth handler pattern.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Filter exports through public visibility gates:** `convex/skills.ts:9639`
  - body: The export query pages `skillSearchDigest` via `by_active_created`, which only constrains `softDeletedAt`. Hidden, removed, non-active, or `blocked.malware` skills can still have a latest version and would be bundled even though current public list and download paths suppress them; filter through the same public/download moderation rules before reading blobs.
  - confidence: 0.93
- **[P2] Apply export rate limits before auth checks:** `convex/httpApiV1/skillsV1.ts:1503-1506`
  - body: `exportSkillsV1Handler` returns from `requireExportAuth` before calling `applyRateLimit`, so missing/invalid tokens and valid non-admin attempts bypass the new export/IP/user buckets and omit rate-limit headers. Keep the ordering consistent with existing v1 handlers by applying the export rate limit first, then auth.
  - confidence: 0.86
