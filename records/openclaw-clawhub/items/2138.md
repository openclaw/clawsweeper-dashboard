- **[P1] Throttle export attempts before checking credentials:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: `exportSkillsV1Handler` returns 401/403 from `requireExportAuth` before calling `applyRateLimit`, so repeated missing or invalid bearer tokens never consume the export/IP bucket. Move throttling ahead of auth or explicitly consume a failed-auth bucket before returning.
  - confidence: 0.91
- **[P1] Keep export out of the skill slug namespace:** `convex/http.ts:77-79`
  - body: Registering `/api/v1/skills/export` before the existing `/api/v1/skills/{slug}` prefix can shadow a valid current or legacy skill slug named `export`, and `export` is not currently reserved. Put this under a reserved namespace such as `/api/v1/skills/-/export` and reserve the path.
  - confidence: 0.86
- **[P1] Filter exported skills through public visibility rules:** `convex/skills.ts:10411-10419`
  - body: `listByDateRange` pages raw `skillSearchDigest.by_active_created` rows and the handler then exports blobs for each row, bypassing the public `toPublicSkill`/nonsuspicious visibility path. Restrict exports to the approved public surface or require a clearly approved staff-only moderation boundary.
  - confidence: 0.9
- **[P1] Page mirror exports by update time:** `convex/skills.ts:10408-10413`
  - body: The export cursor scans `createdAt`, so an hourly mirror will miss skills that were edited or republished after the last sync but created before the requested window. Use `updatedAt` or a version/update watermark for incremental sync.
  - confidence: 0.88
- **[P1] Cap aggregate export bytes before buffering blobs:** `convex/httpApiV1/skillsV1.ts:1693-1741`
  - body: The handler fetches every blob for up to 1000 skills and converts each to an `arrayBuffer` before building one ZIP, with no total byte or file-count limit. Add an aggregate budget and fail predictably before reading blobs that can exceed action memory or time limits.
  - confidence: 0.87
