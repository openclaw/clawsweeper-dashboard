## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Keep exports inside approved visibility rules:** `convex/skills.ts:10411-10420`
  - body: The new export query pages raw `skillSearchDigest` rows and the handler then reads version blobs directly, instead of reusing the public skill/version visibility checks or an approved privileged export contract. That can let the mirror endpoint drift from the documented public catalog boundary for hidden, blocked, or otherwise moderated content.
  - confidence: 0.88
- **[P1] Rate-limit failed export auth attempts:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: `requireExportAuth` runs before `applyRateLimit`, so requests with missing or invalid bearer tokens return 401/403 without consuming the export IP bucket. The existing API contract says missing or invalid tokens fall back to IP enforcement, and this new endpoint should preserve that behavior for brute-force and noisy traffic.
  - confidence: 0.94
- **[P1] Use an update-aware cursor for mirror sync:** `convex/skills.ts:10408-10413`
  - body: The export feed is intended for hourly mirror sync, but it pages by `createdAt` via `by_active_created`. Existing skills that are republished, restored, hidden, deleted, or re-moderated after creation will not be returned in the next hourly creation-date window, so mirrors will go stale.
  - confidence: 0.91
- **[P1] Bound export size before loading blobs:** `convex/httpApiV1/skillsV1.ts:1740-1741`
  - body: A single valid request can fetch up to 1000 skills, load every file blob into memory, and then build another merged ZIP buffer without any aggregate byte or file-count guard. Large skills or many files can exhaust the action before rate limiting has any chance to help.
  - confidence: 0.86
