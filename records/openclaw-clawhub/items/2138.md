- **[P1] Filter exports through the public visibility boundary:** `convex/skills.ts:10411-10421`
  - body: `listByDateRange` pages `skillSearchDigest.by_active_created` and returns digest rows directly, so the export path can include rows the public list/detail paths would suppress through `toPublicSkill`, owner visibility, hidden, or moderation-blocked checks.   - confidence: 0.93

  - body: Registering an exact `GET /api/v1/skills/export` endpoint takes over the current slug detail URL for a skill whose slug is `export`.   - confidence: 0.9
- **[P1] Rate-limit failed export auth before token lookups:** `convex/httpApiV1/skillsV1.ts:1571-1575`
  - body: `exportSkillsV1Handler` calls `requireExportAuth` before `applyRateLimit`, so invalid bearer tokens and non-admin tokens return without consuming the export rate-limit bucket. Add an IP-scoped throttle before expensive auth work, then apply key/admin limits after successful auth.
  - confidence: 0.88
- **[P1] Add an aggregate export size and file-count budget:** `convex/httpApiV1/skillsV1.ts:1693-1694`
  - body: The handler can select up to 1000 skills, fetch every file blob, materialize each `arrayBuffer`, store every ZIP entry, and then build the final archive without any aggregate byte or file-count ceiling. Large or many-file skills can exhaust the Convex action memory/runtime budget even when each individual file is valid.
  - confidence: 0.86
