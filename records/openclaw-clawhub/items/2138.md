## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep exports inside public visibility rules:** `convex/skills.ts:10411`
  - body: `listByDateRange` pages active-created digest rows directly and only constrains `softDeletedAt`, so hidden or moderation-blocked skills that current public APIs filter through `toPublicSkill`/`isPublicSkillDoc` can still be exported as full files. Reuse the public visibility helper or an equivalent indexed gate before any blob reads.
  - confidence: 0.9
- **[P1] Rate-limit failed export auth attempts:** `convex/httpApiV1/skillsV1.ts:1567-1571`
  - body: The export handler returns from `requireExportAuth` before `applyRateLimit`, so missing/invalid token attempts never consume the export/IP bucket and also miss the normal rate-limit/CORS headers. Apply the limiter before auth, or explicitly consume an anonymous export bucket for failed auth paths.
  - confidence: 0.86
- **[P1] Bound export bytes before reading blobs:** `convex/httpApiV1/skillsV1.ts:1713`
  - body: A `limit=1000` request can queue every latest-version file and call `arrayBuffer()` for all blobs before zipping, while published files can be large and there is no aggregate byte or file-count budget. Use the stored file sizes to reject, split, or paginate by total bytes before storage reads.
  - confidence: 0.85
- **[P2] Page mirror sync by changes, not creation time:** `convex/skills.ts:10411`
  - body: The stated hourly mirror sync scans `by_active_created`, so republished existing skills, restores, hides, deletes, and moderation changes after initial creation will not appear in the next window. Use `updatedAt` or a change-feed/tombstone contract if this endpoint is meant to keep mirrors current.
  - confidence: 0.82
