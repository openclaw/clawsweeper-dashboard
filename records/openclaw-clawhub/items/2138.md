## Summary

Keep open: current main does not implement this endpoint, and the PR is not merge-ready because the new bulk export trust boundary still needs product/security approval, real behavior proof, and fixes for security and availability issues.

## What This Changes

The PR adds an admin-gated `GET /api/v1/skills/export` route that pages skills by date, reads latest-version blobs, and returns a merged ZIP with manifest metadata and export-specific rate limiting.

## Best Possible Solution

Define a maintainer-approved mirror/export contract that reuses public visibility and download gates, rate-limits invalid auth, reserves archive metadata paths, caps total bytes, documents the endpoint, and includes focused tests plus redacted runtime proof.

## Reproduction Assessment

Yes for the review findings by source inspection: the PR query can select non-public digests, auth failures return before endpoint rate limiting, valid `_meta.json` files can collide, and large pages read all blobs before zipping. I did not run the PR branch.

## Solution Assessment

No. The branch introduces a new bulk archive endpoint before the public-filter, entitlement, rate-limit, archive-layout, documentation, and size contracts are settled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Filter export rows through public moderation gates:** `convex/skills.ts:9637-9646`
  - body: `listByDateRange` pages `skillSearchDigest` by `by_active_created` without applying the existing public skill/download visibility gates. Hidden, removed, pending-scan, or malware-blocked rows can still be selected and exported through direct blob reads.
  - confidence: 0.94
- **[P2] Rate-limit export auth failures before token lookup:** `convex/httpApiV1/skillsV1.ts:1503-1507`
  - body: The handler calls `requireExportAuth` before `applyRateLimit`, so missing or invalid bearer attempts return before consuming the export/IP bucket. Existing API docs say missing or invalid tokens fall back to IP enforcement; apply the limiter before export auth and preserve rate headers on auth failures.
  - confidence: 0.84
- **[P2] Avoid colliding with skill-owned metadata files:** `convex/httpApiV1/skillsV1.ts:1689-1691`
  - body: The export always adds `${slug}/_meta.json`, but published skill files can already include that path. When such a skill appears in a page, `buildMergedExportZip` throws on the duplicate path and turns a valid export request into a 500.
  - confidence: 0.82
- **[P2] Bound export size before reading blobs:** `convex/httpApiV1/skillsV1.ts:1625`
  - body: The endpoint can select up to 1000 skills and then read every selected blob into memory before zipping. Since valid skill bundles may be up to 50 MB, a normal page can exceed action memory or response limits; paginate by a byte budget or lower the limit before fetching blobs.
  - confidence: 0.78
