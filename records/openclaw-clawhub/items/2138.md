## Summary

Keep open. Current main does not contain this export endpoint, and the PR still introduces a new security-sensitive bulk-export surface with source-reproducible moderation, rate-limit, archive-collision, and byte-budget defects, plus missing real behavior proof and unresolved mirror entitlement scope.

## What This Changes

The PR adds an admin-only paginated `GET /api/v1/skills/export` bulk ZIP export endpoint with route wiring, export auth, rate limits, date-range querying, and merged archive helpers.

## Best Possible Solution

Define a maintainer-approved mirror/export contract first; then update the PR to reuse public visibility and download gates, rate-limit invalid auth by IP before token lookup, reserve archive metadata paths, enforce byte-budget pagination, document the endpoint, and add focused tests plus redacted runtime proof.

## Reproduction Assessment

Yes for the review findings by source inspection: the PR head selects export rows without public visibility gates, authenticates before endpoint rate limiting, adds `_meta.json` in the skill-owned namespace, and reads all blobs for up to 1000 skills. I did not run the PR branch.

## Solution Assessment

No. A bulk export endpoint may be viable, but this implementation opens a new trust boundary before the visibility, entitlement, rate-limit, archive-layout, byte-budget, docs, and proof contracts are settled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Filter export rows through public moderation gates:** `convex/skills.ts:9637-9645`
  - body: `listByDateRange` only pages `skillSearchDigest` by the active-created index, then the handler exports version blobs directly. Hidden, removed, pending-scan, or malware-blocked skills can therefore enter the bulk archive even though current public reads/downloads block them.
  - confidence: 0.94
- **[P2] Rate-limit export auth failures before token lookup:** `convex/httpApiV1/skillsV1.ts:1503-1507`
  - body: The handler calls `requireExportAuth` before `applyRateLimit`, so missing or invalid bearer attempts return before consuming the export/IP bucket. This conflicts with the documented missing/invalid-token IP fallback and leaves token lookup work outside the endpoint limiter.
  - confidence: 0.85
- **[P2] Reserve metadata paths outside the skill file namespace:** `convex/httpApiV1/skillsV1.ts:1689-1691`
  - body: The export always adds `${slug}/_meta.json`, but a valid published skill can already contain `_meta.json`. When that happens, `buildMergedExportZip` throws on the duplicate path and turns an otherwise valid export page into a 500.
  - confidence: 0.82
- **[P2] Bound export bytes before fetching blobs:** `convex/httpApiV1/skillsV1.ts:1625`
  - body: The endpoint can select 1000 skills and then fetch every selected blob before zipping. Since valid skill versions may be up to 50 MB, a normal page can exceed action memory or response limits; page by a byte budget before reading blobs.
  - confidence: 0.78
