## Summary

Keep open. This external PR adds a new security-sensitive bulk export endpoint, but current main has not implemented it and the proposed branch still has blocking visibility, rate-limit, archive-layout, size, proof, and product/security contract gaps.

## What This Changes

The PR adds an admin-only bulk skills export ZIP endpoint with route/schema wiring, export auth, rate limits, date-range querying, and merged archive helpers.

## Best Possible Solution

Define a maintainer-approved mirror/export contract that reuses public visibility and download gates, rate-limits invalid auth at the IP bucket, avoids archive metadata collisions, caps total export bytes, documents the endpoint, and includes focused tests plus redacted runtime proof.

## Reproduction Assessment

Yes for the review findings by source inspection: PR head selects digests from by_active_created without public gates, returns auth failures before applyRateLimit, and can fail on valid _meta.json files or large valid pages. I did not run the PR branch.

## Solution Assessment

No. The branch introduces a new admin bulk export trust boundary before the entitlement, public-filter, rate-limit, archive-layout, documentation, and size contracts are settled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Filter export rows through public moderation gates:** `convex/skills.ts:9639`
  - body: `listByDateRange` pages `skillSearchDigest` by active-created index without applying the existing public skill/download visibility gates. Hidden, removed, pending-scan, or malware-blocked rows can be selected and exported through direct blob reads.
  - confidence: 0.94
- **[P2] Rate-limit export auth failures before token lookup:** `convex/httpApiV1/skillsV1.ts:1503-1507`
  - body: The handler calls `requireExportAuth` before `applyRateLimit`, so missing or invalid bearer attempts return before consuming the export/IP bucket. Existing API docs say missing or invalid tokens fall back to IP enforcement; apply the limiter before export auth and preserve rate headers on auth failures.
  - confidence: 0.86
- **[P2] Avoid colliding with skill-owned metadata files:** `convex/httpApiV1/skillsV1.ts:1689-1691`
  - body: The export always adds `${slug}/_meta.json`, but published skill files can already include that path. When such a skill appears in a page, `buildMergedExportZip` throws on the duplicate path and turns a valid export request into a 500.
  - confidence: 0.82
- **[P2] Bound export size before reading blobs:** `convex/httpApiV1/skillsV1.ts:1625`
  - body: The endpoint can select up to 1000 skills and then read every selected blob into memory before zipping. Since valid skill bundles may be up to 50 MB, a normal page can exceed action memory or response limits; paginate by byte budget or lower the limit before fetching blobs.
  - confidence: 0.78
