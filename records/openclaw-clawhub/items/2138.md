## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Filter export rows through public visibility rules:** `convex/skills.ts:10413`
  - body: The export page is built from `skillSearchDigest.by_active_created` and then reads internal version/storage blobs directly, so hidden or moderation-blocked rows with `softDeletedAt` still unset can be packaged even though the public API policy keeps third-party mirrors inside the public surface. Build the export from the same public/download eligibility checks or an explicitly approved mirror boundary.
  - confidence: 0.9
- **[P1] Move the export endpoint out of the slug namespace:** `packages/schema/src/routes.ts:30`
  - body: `/api/v1/skills/export` is registered as an exact route before `/api/v1/skills/{slug}`, but `export` is still a valid skill slug on main. That can make an existing or future `export` skill unreachable through the documented detail route; use a reserved action namespace such as `/api/v1/skills/-/export` or reserve/migrate the slug first.
  - confidence: 0.88
- **[P2] Rate-limit failed export authentication attempts:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: The handler calls `requireExportAuth` before `applyRateLimit`, so invalid bearer tokens can hash and query token state without consuming the new export bucket. Put an IP-scoped rate limit in front of token lookup or use the shared optional-auth limiter path for rejected attempts.
  - confidence: 0.83
- **[P2] Use update and tombstone cursors for mirror sync:** `convex/skills.ts:10413`
  - body: The query pages by `createdAt`, but the PR use case is hourly mirror sync. Updates, renames, removals, moderation hides, and republished versions on older skills will not appear in a later created-date window, leaving mirrors stale or overexposed.
  - confidence: 0.86
- **[P2] Cap export size before loading storage blobs:** `convex/httpApiV1/skillsV1.ts:1693`
  - body: A single authorized request can queue all files for up to 1000 skills, read each blob into memory, and then copy the ZIP response without any aggregate byte or file-count limit. Add hard caps and fail before reading blobs so the endpoint cannot exhaust Convex action memory.
  - confidence: 0.8
