## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Filter exports through the public visibility boundary:** `convex/skills.ts:10411-10421`
  - body: `listByDateRange` returns `skillSearchDigest` rows from `by_active_created` with only `softDeletedAt` constrained, then the handler reads internal versions/blobs for every row. That can include hidden, removed, or malware-blocked skills that current public APIs exclude through `isPublicSkillDoc()`/`toPublicSkill()`, so the export can bypass the moderation boundary.
  - confidence: 0.93
- **[P1] Avoid shadowing a valid skill slug with the export route:** `convex/http.ts:77`
  - body: Adding an exact `/api/v1/skills/export` route makes that path stop behaving like the documented `/api/v1/skills/{slug}` detail endpoint. `export` is not in the reserved skill slug list and read paths intentionally keep legacy reserved-looking slugs lookup-able, so this can break an existing or future skill detail URL.
  - confidence: 0.88
- **[P1] Throttle failed export auth before token validation:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: The handler calls `requireExportAuth()` before `applyRateLimit()`, so missing, invalid, or non-admin tokens return before consuming the export rate-limit bucket. Current API docs say missing/invalid tokens fall back to IP enforcement, and this new order leaves an expensive auth path easier to hammer.
  - confidence: 0.9
- **[P2] Use updated windows for incremental mirror sync:** `convex/skills.ts:10413`
  - body: The export window is backed by `by_active_created`, so an existing skill republished or otherwise updated after the mirror's last sync will still have an old `createdAt` and can be missed by an hourly incremental export. For the stated mirror-sync use case, the cursor window needs to track updated content, or expose clearly separate full-catalog and incremental semantics.
  - confidence: 0.87
- **[P1] Enforce an aggregate archive budget before reading blobs:** `convex/httpApiV1/skillsV1.ts:1693`
  - body: The handler can select up to 1000 skills, fetch every file blob, materialize each `arrayBuffer()`, and then build a second full ZIP buffer without any total byte or file-count cap. A large allowed export can exceed Convex action memory/runtime limits and turn an admin request into an availability problem.
  - confidence: 0.86
