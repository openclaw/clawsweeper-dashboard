Full review comments:

- **[P1] Keep exports inside approved visibility rules:** `convex/skills.ts:10411-10413`
  - body: This new export query reads raw active-created digest rows and the handler then fetches raw version blobs, while the documented third-party catalog path is the public read/file API and explicitly says not to mirror hidden, private, or moderation-blocked content outside those filters. Please either reuse the public visibility/file gates for mirror sync or get an explicitly audited privileged staff-export contract before exposing this route.
  - confidence: 0.87
- **[P1] Rate-limit failed export auth attempts:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: `requireExportAuth` runs before `applyRateLimit`, so requests with no bearer token or a random invalid token return 401 without consuming the export IP bucket. That leaves the token/auth path unthrottled for failed-auth traffic and contradicts the documented missing/invalid-token fallback to IP enforcement.
  - confidence: 0.91
- **[P1] Use update-aware pagination for mirror sync:** `convex/skills.ts:10408-10413`
  - body: The PR says mirrors will run hourly incremental syncs, but this query pages by `createdAt`. Existing skills that are republished, restored, hidden, removed, or otherwise moderated after their original creation date will not appear in the next hourly export, so a mirror can drift permanently.
  - confidence: 0.9
- **[P1] Add aggregate export byte and file limits:** `convex/httpApiV1/skillsV1.ts:1693-1741`
  - body: The handler fetches every blob for up to 1000 skills, converts each to memory, stores all zip entries, and then builds one merged ZIP without a total byte or file-count guard. Since a single skill bundle may be large, one valid request can exceed Convex action memory or stall the endpoint.
  - confidence: 0.84
