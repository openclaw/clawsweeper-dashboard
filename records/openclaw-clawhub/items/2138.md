## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Filter exports through the public visibility boundary:** `convex/skills.ts:10410`
  - body: `listByDateRange` scans active digests only by `softDeletedAt`, so hidden moderation states and `blocked.malware` skills can be exported even though current public paths filter them through `isPublicSkillDoc`. The export should either use the same public boundary or require an explicitly approved staff-only moderation export contract.
  - confidence: 0.95
- **[P1] Use an update-based mirror cursor:** `convex/skills.ts:10410`
  - body: The endpoint is advertised for hourly incremental mirror sync, but a `createdAt` range will miss old skills that are republished, renamed, moderated, deleted, or otherwise updated. Mirror export needs an `updatedAt` or change-log style cursor, otherwise mirrors silently go stale.
  - confidence: 0.91
- **[P1] Throttle failed bearer-token attempts before token lookup:** `convex/httpApiV1/skillsV1.ts:1568-1572`
  - body: `exportSkillsV1Handler` calls `requireExportAuth` before `applyRateLimit`, so invalid bearer tokens still hash and query token state without consuming the export rate limit. Put an IP-based pre-auth throttle in front of token lookup, or add a rate-limit helper that charges failed auth attempts.
  - confidence: 0.93
- **[P1] Add aggregate caps before reading blobs into memory:** `convex/httpApiV1/skillsV1.ts:1693-1741`
  - body: The handler can fetch up to 1000 skills and then reads every file blob into memory before creating a ZIP copy, but it has no total byte, file-count, or response-size guard. A single authorized request can exceed Convex action memory or stall the function as catalog file sizes grow.
  - confidence: 0.89
- **[P1] Move export out of the skill slug namespace:** `convex/http.ts:76-80`
  - body: `/api/v1/skills/export` is an exact route inside the existing `/api/v1/skills/<slug>` namespace, and `export` is not shown as a reserved skill slug. This can make a valid `export` skill unreachable through the detail API, so use a reserved control segment such as `/api/v1/skills/-/export` or reserve and migrate the slug explicitly.
  - confidence: 0.86
