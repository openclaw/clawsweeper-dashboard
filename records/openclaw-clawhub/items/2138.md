Full review comments:

- **[P1] Filter export rows through the public visibility policy:** `convex/skills.ts:10411-10420`
  - body: `listByDateRange` returns raw `skillSearchDigest` rows and the handler fetches files for every returned digest. Public list code converts digests through `toPublicSkill`/`isPublicSkillDoc`, which drops hidden, non-active, and malware-blocked rows; this mirror export needs the same boundary or an explicit privileged export contract before it can ship.
  - confidence: 0.92
- **[P1] Page export by updated activity, not creation time:** `convex/skills.ts:10413-10418`
  - body: The proposed hourly mirror sync filters on `createdAt` via `by_active_created`, so a skill republished after its creation window will not appear in later incremental exports. The existing HTTP API docs distinguish `createdAt` as stable for new-skill crawls and `updated` as changing on republish, so the export cursor needs updated/latest-version semantics for incremental sync.
  - confidence: 0.9
- **[P1] Throttle failed export auth attempts:** `convex/httpApiV1/skillsV1.ts:1571-1575`
  - body: The handler returns from `requireExportAuth` before calling `applyRateLimit`, so missing or invalid bearer tokens never consume the IP fallback bucket documented for invalid tokens. Move an IP/anonymous rate check ahead of auth or otherwise account for failed export auth traffic.
  - confidence: 0.86
- **[P1] Bound the total bytes before materializing the export ZIP:** `convex/httpApiV1/skillsV1.ts:1693-1741`
  - body: With `limit` up to 1000, the handler queues all files, loads every blob, converts each to a `Uint8Array`, and then builds one ZIP. Current publish limits allow large skill bundles, so a valid request can exhaust action memory unless the export rejects or paginates by an aggregate byte budget before blob reads.
  - confidence: 0.85
