## Summary

Keep open: current main does not have this export endpoint, and the PR still introduces a security-sensitive bulk ZIP path with blocking review findings plus missing real behavior proof.

## What This Changes

Adds an admin-authenticated `GET /api/v1/skills/export` endpoint for paginated bulk skill ZIP exports, with route wiring, auth/rate-limit helpers, ZIP merging utilities, and schema route output.

## Best Possible Solution

Define and document an authorized mirror export contract that reuses the public visibility/download moderation gates, applies the established rate-limit model, has focused regression tests, and includes maintainer-approved role/throughput policy.

## Reproduction Assessment

Yes for the review findings: source inspection at PR head shows the export query can select rows current public and download code would block, and auth failures return before rate limiting. I did not run the PR branch.

## Solution Assessment

No; this is not the safest implementation yet because it creates a new bulk archive path instead of reusing the existing public/download gates and rate-limit ordering. A narrower solution should enforce the same public registry contract before reading version blobs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Filter export rows through public moderation rules:** `convex/skills.ts:9639`
  - body: The export query pages `skillSearchDigest` by `by_active_created`, which only constrains `softDeletedAt`. Hidden or `blocked.malware` skills can still have a latest version and would then be read into the merged ZIP, bypassing the public visibility and download gates; filter through the same public moderation rules before fetching blobs.
  - confidence: 0.94
- **[P2] Apply export rate limits before auth returns:** `convex/httpApiV1/skillsV1.ts:1503-1506`
  - body: `requireExportAuth` runs and returns 401/403 before `applyRateLimit`, so missing, invalid, and unauthorized export attempts never consume the export/IP bucket. Current API docs say missing or invalid tokens fall back to IP enforcement, so move the export rate-limit check before auth and include those headers in auth failures.
  - confidence: 0.86
