Overall confidence: 0.9

Full review comments:

- **[P1] Throttle export requests before checking credentials:** `convex/httpApiV1/skillsV1.ts:1571-1575`
  - body: `requireExportAuth` returns 401/403 before `applyRateLimit` runs, so missing or invalid tokens never hit the new export bucket and can still drive auth lookups. Move an IP-safe rate-limit check ahead of auth or otherwise consume a failed-auth bucket before returning auth failures.
  - confidence: 0.93
- **[P1] Filter exports through the public visibility boundary:** `convex/skills.ts:10411-10420`
  - body: `listByDateRange` returns raw digest rows from `by_active_created` and the handler zips their blobs directly, bypassing the same `toPublicSkill`/`isPublicSkillDoc` checks used by public list/detail/download paths. Apply the public visibility and owner/version filters before any blob reads so hidden, removed, or moderation-blocked content cannot enter the archive.
  - confidence: 0.91
- **[P1] Page mirror exports by updates, not creation time:** `convex/skills.ts:10413`
  - body: The export window is based on `createdAt`, but the PR's stated hourly mirror sync needs to catch republished or edited skills whose `createdAt` is old and `updatedAt` changes. Use an updated-time cursor/window or separate created/updated traversal so mirrors do not silently miss changes.
  - confidence: 0.9
- **[P1] Move export out of the skill slug namespace:** `convex/http.ts:77-79`
  - body: Adding `/api/v1/skills/export` as an exact route reserves `export` inside the documented `/api/v1/skills/{slug}` namespace without a reserved-slug migration. Put collection operations under a non-slug namespace such as `/api/v1/skills/-/export`, or add an explicit compatibility plan for existing and future slugs.
  - confidence: 0.86
- **[P1] Enforce a total export size budget before buffering blobs:** `convex/httpApiV1/skillsV1.ts:1693-1741`
  - body: The handler can collect up to 1000 skills, fetch all file blobs, convert each blob to an ArrayBuffer, and then build one ZIP with no aggregate byte or file-count ceiling. Add a hard export budget or much smaller paging/streaming boundary before reading blobs so one request cannot exhaust action memory or time.
  - confidence: 0.88
