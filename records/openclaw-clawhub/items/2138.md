## Best Possible Solution

Define the mirror-export contract first, then implement either a public-safe sync endpoint that reuses existing public visibility/download rules or an explicitly privileged staff API with auditability, update-based cursors, throttling, size caps, docs, and tests.

Full review comments:

- **[P1] Preserve the public catalog boundary:** `convex/skills.ts:10413`
  - body: The export query pages `skillSearchDigest.by_active_created` and the handler then exports latest-version blobs directly. Current public docs and helpers filter hidden and moderation-blocked skills, so this can bulk-export content that normal public list/detail/file routes would hide unless maintainers explicitly approve a privileged export contract.
  - confidence: 0.92
- **[P1] Page exports by update or version activity:** `convex/skills.ts:10408-10413`
  - body: The mirror use case in the PR body depends on incremental sync, but this query only ranges over `createdAt`. Republished files, moderation changes, restores, and hides for existing skills will not appear in later created-date windows, leaving mirrors stale.
  - confidence: 0.9
- **[P1] Throttle failed export authentication:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: The handler returns missing or invalid token failures before calling `applyRateLimit`, so bad-token traffic never consumes the export/IP bucket. That contradicts the documented rate-limit model where missing or invalid tokens fall back to IP enforcement.
  - confidence: 0.88
- **[P1] Add an aggregate export size guard:** `convex/httpApiV1/skillsV1.ts:1740-1741`
  - body: A request can select up to 1000 skills, fetch all file blobs, convert every blob to an ArrayBuffer, and then build one ZIP with another copy. Without a total file/ZIP byte cap, a valid request can exceed Convex action memory or stall the endpoint.
  - confidence: 0.86
