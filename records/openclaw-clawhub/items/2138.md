## Summary

Keep this PR open for maintainer review, but it is not merge-ready: current main does not already provide the requested bulk export API, and the proposed implementation introduces blocking compatibility, security-boundary, sync-correctness, availability, and proof gaps.

## What This Changes

The PR adds an admin-only `GET /api/v1/skills/export` endpoint that returns paginated ZIP archives of skill files for mirror synchronization.

## Best Possible Solution

Define the mirror export contract first, then implement it on a reserved route that applies approved visibility/download rules, syncs by updates plus tombstones, caps aggregate bytes/files, throttles failed auth, and includes focused tests plus inspectable real behavior proof.

## Reproduction Assessment

Not applicable as a current-main user bug. The blocking PR issues are source-reproducible from the branch diff against current main’s route, visibility, download, pagination, and rate-limit paths.

## Solution Assessment

No. A core bulk export endpoint may be useful, but this implementation should wait for an explicit mirror/security contract and then reuse or codify the same visibility, sync, throttling, and resource boundaries as the rest of the API.

- **[P1] Reuse public visibility before exporting files:** `convex/skills.ts:10411-10419`
  - body: `listByDateRange` pages `skillSearchDigest` by `by_active_created` with only the soft-delete prefix, and the handler later reads latest-version blobs internally.   - confidence: 0.93

  - body: Registering `/api/v1/skills/export` as an exact route before the `/api/v1/skills/` slug router means an existing or future skill with slug `export` loses its documented detail URL.   - confidence: 0.9
- **[P1] Page mirror exports by changes, not creation time:** `convex/skills.ts:10408-10413`
  - body: The export cursor is built from `createdAt`, so hourly mirror sync will miss new versions, metadata edits, moderation changes, deletions, restores, and other updates to existing skills. A mirror contract needs an update/tombstone stream or an updated-at cursor, not a created-at-only range.
  - confidence: 0.88
- **[P2] Throttle failed export authentication attempts:** `convex/httpApiV1/skillsV1.ts:1571-1575`
  - body: The handler authenticates before calling `applyRateLimit`, so missing or invalid bearer tokens return without consuming the export bucket. Move an IP/failed-auth rate limit ahead of token validation, or add a dedicated failed-auth bucket for this privileged endpoint.
  - confidence: 0.86

  - body: The handler can read all blobs for up to 1000 skills and then build a copied merged ZIP without a total file-count or byte cap. A few large skills can push the Convex action over memory/time limits, especially with the proposed admin export rate.
