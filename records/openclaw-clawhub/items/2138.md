## Summary

Keep open: this PR adds a new bulk mirror export feature that is not implemented on main, and the latest head still has blocking security/availability issues plus an unresolved maintainer product decision. ## What This Changes

Adds an admin-gated `GET /api/v1/skills/export` endpoint that returns date-ranged skill files and metadata as paginated merged ZIP archives, plus route, schema, auth, and rate-limit support.

## Best Possible Solution

Either keep mirrors on the documented public APIs, or approve a public-only audited export endpoint with explicit entitlement, filtering, pre-auth throttling, byte/file budgets, focused tests, docs, and real behavior proof.

## Reproduction Assessment

Yes for the review findings by source inspection: the latest head queries raw digest rows, authenticates before applying the export limiter, and buffers selected blobs before zipping. I did not execute the PR branch because this was a read-only review.

## Solution Assessment

No. Bulk mirror export needs maintainer-approved product and security boundaries first; if approved, the narrow implementation should reuse public visibility gates and add pre-auth throttling plus byte/file budgets.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Filter exports through public visibility gates:** `convex/skills.ts:10411-10413`
  - body: The export query pages `skillSearchDigest.by_active_created` with only `softDeletedAt` in the key and returns raw rows to the ZIP builder. That bypasses the existing public `toPublicSkill`/`isPublicSkillDoc` gates, so a mirror export can include moderation-hidden, removed, or malware-blocked rows that the documented public API must not mirror.
  - confidence: 0.9
- **[P2] Throttle export attempts before auth lookup:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: This calls `requireExportAuth` before consuming the new export limiter, so invalid bearer-token attempts still force token hashing and Convex lookups outside the endpoint's IP bucket. For a high-value bulk endpoint, run an IP/pre-auth limit before auth or otherwise charge invalid credentials to the export limit.
  - confidence: 0.87
- **[P2] Cap export bytes before reading blobs:** `convex/httpApiV1/skillsV1.ts:1693`
  - body: The handler builds blob tasks for the whole page and fetches up to 1000 skills' files before checking aggregate bytes or file count, then copies each blob into memory and builds a ZIP. A large page can exceed action memory or time limits before the handler can fail cleanly.
  - confidence: 0.84
