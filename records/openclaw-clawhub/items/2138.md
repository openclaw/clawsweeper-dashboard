## Review Findings

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve the public export boundary:** `convex/skills.ts:10411-10419`
  - body: For the stated mirror use case, this query cannot page `by_active_created` without applying the same public visibility and nonsuspicious gates used by catalog/download surfaces. Otherwise a mirror export can include suspicious or moderator-hidden content that normal public consumers cannot receive.
  - confidence: 0.9
- **[P1] Page mirror exports by update time:** `convex/skills.ts:10407-10413`
  - body: Hourly mirror syncs need to pick up new versions of existing skills, but this range only uses `createdAt`. A skill created last month and updated this hour will not be exported for this hour's window, so mirrors can silently miss updates.
  - confidence: 0.92
- **[P1] Rate limit failed export auth first:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: `requireExportAuth` returns 401/403 before `applyRateLimit` runs, so missing or invalid tokens never consume the export IP bucket. Put a failed-auth-safe rate check before token validation or otherwise throttle rejected export attempts.
  - confidence: 0.88
- **[P1] Bound export size before zipping:** `convex/httpApiV1/skillsV1.ts:1740-1780`
  - body: The handler reads every selected file blob into memory and then builds a second merged ZIP buffer without a total-size cap. A valid admin export over large skills can exhaust action memory or stall the backend; enforce an aggregate byte limit or stream/chunk the export.
  - confidence: 0.86
