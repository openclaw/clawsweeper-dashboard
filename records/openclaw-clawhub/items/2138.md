- **[P1] Preserve public visibility filtering before exporting blobs:** `convex/skills.ts:10410`
  - body: `listByDateRange` pages raw `skillSearchDigest` rows from `by_active_created` with only the soft-delete prefix, then the handler reads latest-version blobs directly. That bypasses the current public path through `toPublicSkill`/`isPublicSkillDoc`, so hidden or moderation-blocked skills can be included in a mirror export despite the public API docs telling third-party catalogs not to mirror that content outside public filters.
  - confidence: 0.92
- **[P1] Throttle failed export auth before token lookup:** `convex/httpApiV1/skillsV1.ts:1569-1572`
  - body: The handler returns from `requireExportAuth` before calling `applyRateLimit`, so missing or invalid bearer-token attempts do not consume the anonymous/IP quota documented for failed auth. Invalid tokens still hash and query the token table, leaving this new endpoint with an unthrottled auth-failure path.
  - confidence: 0.88
- **[P1] Page mirror exports by updates, not creation time:** `convex/skills.ts:10408-10418`
  - body: The PR describes hourly incremental mirror sync, but the new query ranges over `createdAt`. Republished or updated older skills will not appear in later hourly windows, so a mirror can permanently miss changed content unless it re-crawls old creation ranges.
  - confidence: 0.9
- **[P1] Add an aggregate export budget before reading blobs:** `convex/httpApiV1/skillsV1.ts:1645-1683`
  - body: One request can select up to 1000 skills, fetch every file blob, convert each Blob to an ArrayBuffer, and ZIP the whole batch without a total byte or file-count limit. A valid admin request for unusually large or numerous skill files can exhaust Convex action memory/CPU instead of failing early with a bounded response.
  - confidence: 0.86
- **[P2] Avoid stealing the `export` skill detail route:** `packages/schema/src/routes.ts:28`
  - body: `/api/v1/skills/{slug}` is a documented public detail route and `export` is not currently reserved as a skill slug. Registering an exact `/api/v1/skills/export` endpoint can make a current or grandfathered skill with that slug unreachable through its API detail URL; use a reserved collection-action namespace such as `/api/v1/skills/-/export` and reserve it deliberately.
