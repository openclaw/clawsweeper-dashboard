  - body: The export range is keyed on `createdAt`, but the PR's stated hourly mirror use case needs changes after creation too. A skill created last month and republished or otherwise updated during the requested window will not be returned, so an incremental mirror can silently miss changed content.
  - confidence: 0.94
- **[P2] Throttle failed export authentication attempts:** `convex/httpApiV1/skillsV1.ts:1570-1575`
  - body: The handler checks `requireExportAuth` before applying the new `export` rate limit. Invalid bearer tokens therefore hit token auth without spending the documented IP fallback bucket, unlike the surrounding HTTP handlers that rate-limit before authenticated work.
  - confidence: 0.9
- **[P2] Cap archive bytes before reading blobs:** `convex/httpApiV1/skillsV1.ts:1693-1741`
  - body: The code limits only the number of skills, then fetches all referenced blobs, materializes each blob into memory, and creates the merged ZIP. A page with many or large files can exceed Convex action memory even though `limit <= 1000`; add aggregate byte/file limits or a smaller partitioned export contract before building the archive.
  - confidence: 0.9
- **[P1] Reserve or move the export route:** `convex/http.ts:77-80`
  - body: Adding an exact `/api/v1/skills/export` route before the existing `/api/v1/skills/<slug>` router changes the response for a skill whose slug is `export`. Since current slug validation does not reserve that word and read paths preserve legacy reserved-like slugs, use a non-conflicting namespace or add a tested reservation/migration story.
