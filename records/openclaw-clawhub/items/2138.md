- **[P1] Filter export rows through the approved visibility gate:** `convex/skills.ts:10413`
  - body: `listByDateRange` reads every non-soft-deleted digest row from `by_active_created`, but current public/download paths also exclude hidden, removed, pending, and malware-blocked states before serving files.   - confidence: 0.92

  - body: The stated hourly sync use case will miss an existing skill that publishes a new version after its original `createdAt` window, because this query is bounded on `createdAt`. Use the updated index/cursor for incremental syncs, or split full new-skill crawls from update exports.
  - confidence: 0.9
- **[P1] Apply export rate limits before auth:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: The handler rejects missing or invalid tokens before `applyRateLimit`, so failed-auth traffic never consumes the export IP bucket and can repeatedly hit token hashing and lookup work. Match the existing API pattern by applying the endpoint rate limit before auth-specific checks.
  - confidence: 0.86
- **[P1] Cap exported bytes before materializing blobs:** `convex/httpApiV1/skillsV1.ts:1693`
  - body: A single request can select 1000 skills, fetch every blob, convert each file into memory, and then build one merged ZIP with no aggregate byte guard. Bound total exported bytes or page by byte budget before reading blobs so one request cannot exhaust backend memory.
  - confidence: 0.84
