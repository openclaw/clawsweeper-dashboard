## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Filter exports through the public visibility boundary:** `convex/skills.ts:10411-10419`
  - body: The export query walks `skillSearchDigest.by_active_created` with only the soft-delete prefix and returns raw digest rows that the handler turns into ZIP contents. Existing public APIs run digest rows through `toPublicSkill`/owner visibility checks, so this can archive hidden, blocked, or otherwise non-public skill content for a mirror export.
  - confidence: 0.92
- **[P1] Page incremental exports by update time:** `convex/skills.ts:10413`
  - body: The requested mirror workflow is hourly incremental sync, but this query uses `by_active_created` and bounds on `createdAt`. A republished or moderated older skill has a new `updatedAt` without a new `createdAt`, so the mirror will miss changes after its initial crawl.
  - confidence: 0.9
- **[P1] Move rate limiting before export auth:** `convex/httpApiV1/skillsV1.ts:1571-1575`
  - body: The handler returns missing, invalid, and forbidden export auth responses before `applyRateLimit` runs. Invalid bearer attempts still do token hashing and database lookups, so failed auth traffic is outside the export throttle that the PR is adding.
  - confidence: 0.88
- **[P2] Add aggregate export size limits before buffering blobs:** `convex/httpApiV1/skillsV1.ts:1740-1741`
  - body: For each page the handler fetches all selected storage blobs, converts every blob to a `Uint8Array`, and then builds one ZIP in memory, with no total byte or file-count guard. A single admin-key request can therefore push the Convex action into memory or time limits.
  - confidence: 0.86
- **[P1] Avoid taking over the export slug detail route:** `convex/http.ts:77-79`
  - body: Registering `/api/v1/skills/export` changes the documented `GET /api/v1/skills/{slug}` behavior for the slug `export`, which is not currently reserved for legacy reads. Use a reserved namespace such as `/api/v1/skills/-/export` or include an explicit migration/compatibility plan.
  - confidence: 0.84
