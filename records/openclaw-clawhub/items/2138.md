## Summary

Keep open: current main does not implement this export API, and the PR still needs maintainer product/security review, real behavior proof, and code fixes for a build-breaking cursor implementation plus bulk-export security gaps.

## What This Changes

This PR adds an admin-only `/api/v1/skills/export` ZIP endpoint with route constants, export auth/rate-limit helpers, ZIP merge helpers, and a date-range query over `skillSearchDigest`.

## Best Possible Solution

Define a maintainer-approved mirror/export contract that reuses public visibility and download gates, rate-limits invalid auth at the IP bucket, avoids archive metadata collisions, documents the endpoint, and includes focused tests plus redacted runtime proof.

## Reproduction Assessment

Yes for the review findings by source inspection: PR head calls missing/misused cursor helpers, can select moderated digests for export, and returns auth failures before rate limiting. I did not run the PR branch.

## Solution Assessment

No; this is not yet the best way to solve mirror sync because it adds a security-sensitive bulk archive surface before the trust model and public-filter contract are settled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Use the existing cursor encoder and decoder correctly:** `convex/skills.ts:9628-9651`
  - body: `decodeIndexKey` is not defined in current `convex/skills.ts`, and `encodeIndexKey` requires both an index name and key. As written, this new query will fail TypeScript before the endpoint can ship.
  - confidence: 0.96
- **[P1] Filter export rows through public moderation gates:** `convex/skills.ts:9637-9646`
  - body: `listByDateRange` pages `skillSearchDigest` by `softDeletedAt` and `createdAt` only. Hidden, removed, pending-scan, or malware-blocked rows can still be returned and then have their blobs read directly, bypassing the public list/download moderation gates.
  - confidence: 0.94
- **[P2] Rate-limit export auth failures before token lookup:** `convex/httpApiV1/skillsV1.ts:1503-1506`
  - body: The handler authenticates first and returns missing/invalid bearer responses before `applyRateLimit`, so those attempts never consume the export/IP bucket. Existing API behavior falls invalid auth back to IP enforcement; move the limiter ahead of export auth and preserve rate headers on auth failures.
  - confidence: 0.84
- **[P2] Avoid colliding with skill-owned `_meta.json` files:** `convex/httpApiV1/skillsV1.ts:1689-1690`
  - body: The export always writes `${slug}/_meta.json`, but published skill files can already use that path. When such a skill appears in a page, `buildMergedExportZip` detects the duplicate and throws, turning a valid export request into a 500.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff introduces a security-sensitive bulk archive endpoint that can expose moderated skill blobs and lets auth failures bypass endpoint rate limiting.

Concerns:

- **[high] Bulk export can include moderated blocked skills:** `convex/skills.ts:9639`
  - body: The export query only constrains `softDeletedAt` via `by_active_created`; it does not enforce current public visibility rules or the single-skill download blocks for malware, pending-scan, removed, or hidden skills before reading storage blobs.
  - confidence: 0.94
- **[medium] Unauthorized export attempts bypass the limiter:** `convex/httpApiV1/skillsV1.ts:1503`
  - body: `requireExportAuth` runs before `applyRateLimit`, so invalid bearer-token attempts can trigger token parsing/hash/lookup work without consuming the endpoint's export/IP rate-limit bucket.
  - confidence: 0.84
