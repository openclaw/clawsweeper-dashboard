Full review comments:

- **[P1] Move export off the skill slug route:** `convex/http.ts:77-79`
  - body: Adding `/api/v1/skills/export` as an exact child of `/api/v1/skills/` makes a valid or legacy skill slug named `export` unreachable through the documented detail API. `export` is not currently reserved, and read paths intentionally keep legacy/reserved-shaped slugs readable, so use a non-slug namespace such as `/api/v1/skills/-/export` or add an explicit reservation/migration plan before routing it here.
  - confidence: 0.88
- **[P1] Apply public visibility gates before exporting blobs:** `convex/skills.ts:10413-10420`
  - body: `listByDateRange` returns raw `skillSearchDigest` rows from `by_active_created` and the handler then reads latest-version blobs for those rows. That bypasses the existing `toPublicSkill`/moderation filters and can export hidden, malware-blocked, or suspicious content despite the public mirror docs saying not to bypass those boundaries.
  - confidence: 0.93
- **[P1] Rate-limit failed export auth before token lookup:** `convex/httpApiV1/skillsV1.ts:1571-1575`
  - body: The handler calls `requireExportAuth` before `applyRateLimit`, so missing, invalid, and non-admin export requests return without consuming the export bucket. Invalid bearer floods can still force token hashing and database auth lookups, contrary to the documented missing/invalid-token IP fallback.
  - confidence: 0.9
- **[P1] Bound the total export archive size:** `convex/httpApiV1/skillsV1.ts:1693`
  - body: Chunking concurrency does not bound memory: the handler queues blob reads for the full page, materializes each blob with `arrayBuffer()`, and then builds one ZIP in memory. A page with many or large skill files can exceed Convex action limits, so add an aggregate byte/file budget and return a controlled error or smaller page.
  - confidence: 0.86
- **[P2] Use updatedAt for incremental mirror windows:** `convex/skills.ts:10413-10418`
  - body: The PR describes hourly mirror sync, but the date range pages the `by_active_created` index. Existing skills republished after their original creation date will not appear in later windows, so mirrors can miss updates unless this uses `updatedAt` or an explicit created-vs-updated mode.
  - confidence: 0.87
