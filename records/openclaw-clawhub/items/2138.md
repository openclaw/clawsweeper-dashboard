## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Filter exports through the public visibility gate:** `convex/skills.ts:10413`
  - body: The export query pages `skillSearchDigest` with `by_active_created`, which only constrains `softDeletedAt`; hidden, removed, and `blocked.malware` digests can still be returned and then have their latest blobs written into the ZIP. Please reuse the public visibility predicate or a dedicated export eligibility gate and add regression coverage for hidden, removed, and malware-blocked skills.
  - confidence: 0.93
- **[P2] Rate-limit invalid export auth before token lookup:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: `requireExportAuth` runs before `applyRateLimit`, so missing, malformed, or revoked bearer tokens return 401 without consuming the export IP bucket, while invalid bearer attempts still trigger token hashing and lookup work. Put an IP/pre-auth export limiter in front of auth failures, then apply the authenticated bucket for valid export callers.
  - confidence: 0.86
- **[P2] Add a total byte budget to export pages:** `convex/httpApiV1/skillsV1.ts:1740-1741`
  - body: The endpoint caps the page at 1000 skills but then reads every file blob into memory and builds one ZIP, so a valid page with larger artifacts can exceed action memory or response limits. Track cumulative uncompressed and compressed bytes, stop before the budget, and return a clear pagination or 413-style response with tests.
  - confidence: 0.82
- **[P2] Reserve metadata paths before adding skill files:** `convex/httpApiV1/skillsV1.ts:1757-1759`
  - body: A legitimate skill file named `_meta.json` passes `validateFilePath`, then collides with the injected `${slug}/_meta.json` entry and makes the whole export throw instead of reporting a per-skill error. Treat `_meta.json` and other export-reserved names as reserved before adding files, or put generated metadata under a namespace that cannot collide.
  - confidence: 0.79
