## What This Changes

## Best Possible Solution

First settle the mirror-export product and security contract, then implement a namespaced, capped, audit-friendly export path that reuses the public visibility boundary and update-based sync semantics.

Full review comments:

- **[P1] Filter exports through the public visibility boundary:** `convex/skills.ts:10411-10413`
  - body: `listByDateRange` only constrains `softDeletedAt` via `by_active_created`, and the handler then reads and emits latest-version files directly. Current public APIs run rows through `toPublicSkill`/`isPublicSkillDoc`, which rejects hidden moderation states and `blocked.malware`, so this can export content the public mirror contract says not to mirror.
  - confidence: 0.93
- **[P1] Cursor mirror exports by updates, not creation time:** `convex/skills.ts:10407-10413`
  - body: The PR describes hourly incremental mirror sync, but this cursor scans `createdAt`. Any old skill that is republished, renamed, restored, moderated, deleted, or otherwise updated after its creation window will not appear in the next incremental export, leaving mirrors silently stale.
  - confidence: 0.9
- **[P1] Avoid stealing the export slug from skill detail reads:** `convex/http.ts:77-79`
  - body: This exact `/api/v1/skills/export` route occupies a URL that the current `/api/v1/skills/{slug}` router can serve as a normal one-segment skill slug, and `export` is not reserved by the slug validator. Move the operation under a reserved namespace such as `/-/export` or add a tested reservation/migration path.
  - confidence: 0.86
- **[P1] Cap total bytes and file count before building the ZIP:** `convex/httpApiV1/skillsV1.ts:1740-1780`
  - body: The handler reads every selected blob into memory and then builds a second full ZIP byte array without any aggregate byte, file-count, or response-size limit. A valid admin request over large skills can exceed Convex action memory or response limits and make the endpoint fail under real catalog data.
  - confidence: 0.87
- **[P2] Rate-limit failed bearer attempts before token lookup:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: `requireExportAuth` runs before `applyRateLimit`, so malformed or invalid bearer tokens hash and query token/user state without consuming an IP export bucket. That diverges from the current API rate-limit contract for missing/invalid tokens and leaves an unauthenticated load path.
  - confidence: 0.88
