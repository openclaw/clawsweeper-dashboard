## Summary

This PR should stay open for maintainer product/security review: it adds a new full-catalog bulk export trust boundary, and the current diff still has blocking security, availability, and proof gaps.

## What This Changes

The branch adds an admin-only `GET /api/v1/skills/export` HTTP endpoint that returns paginated merged ZIP archives of skill files with route/schema wiring, export auth, rate limiting, and ZIP helper changes.

## Best Possible Solution

Define a maintainer-approved mirror/export contract first, then implement the endpoint with public visibility/download gates, IP-rate-limited auth failures, reserved archive metadata paths, a total byte budget, public API docs, focused tests, and redacted runtime proof.

## Reproduction Assessment

Yes for the review findings by source inspection: the PR selects active-created digests without public gates, runs auth before rate limiting, reads all selected blobs before zipping, and injects per-skill `_meta.json` paths that can collide. I did not run the PR branch.

## Solution Assessment

No. The PR implements the mechanism before the mirror entitlement, moderation boundary, byte budget, archive layout, and public API contract are settled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Filter export rows through public moderation gates:** `convex/skills.ts:9639`
  - body: `listByDateRange` pages `skillSearchDigest` directly by `by_active_created` and the action then reads version blobs from those rows. This bypasses the current public `toPublicSkill`/`isPublicSkillDoc` and download moderation gates, so hidden, removed, malware-blocked, or otherwise non-public skills can be included in a bulk export.
  - confidence: 0.94
- **[P2] Rate-limit export auth failures before token lookup:** `convex/httpApiV1/skillsV1.ts:1503-1507`
  - body: The handler calls `requireExportAuth` before `applyRateLimit`, so missing or invalid bearer attempts return without consuming the export/IP bucket. Apply the limiter first and preserve rate headers on auth failures so unauthorized probing is still throttled.
  - confidence: 0.86
- **[P2] Bound export size before reading blobs:** `convex/httpApiV1/skillsV1.ts:1625`
  - body: The endpoint can select up to 1000 skills and then read every selected storage blob into memory before ZIP assembly. Since current publish limits allow large valid skill bundles, paginate by a total byte budget or reduce the page before fetching blobs.
  - confidence: 0.82
- **[P2] Avoid colliding with skill-owned metadata files:** `convex/httpApiV1/skillsV1.ts:1689-1691`
  - body: The export always appends `${slug}/_meta.json`, but a published skill can already contain `_meta.json`. When that happens, `buildMergedExportZip` throws on the duplicate path and turns a valid export request into a 500.
  - confidence: 0.82
