The PR adds an admin-only `GET /api/v1/skills/export` endpoint that returns cursor-paginated merged ZIP archives of skill files and metadata for mirror synchronization.

## Best Possible Solution

Define the export contract first, then implement a visibility-safe, update-time paginated export with failed-auth throttling, aggregate size limits, focused tests, docs, and redacted real behavior proof.

Full review comments:

- **[P1] Preserve the public export boundary:** `convex/skills.ts:10411-10413`
  - body: `listByDateRange` returns raw digest rows from the active-created index, and the handler later fetches version blobs directly. That bypasses the current `toPublicSkill`/`isPublicSkillDoc` visibility gate, so hidden, removed, or malware-blocked skills can enter mirror ZIPs; use the public visibility path or an explicitly approved privileged export path before exposing files.
  - confidence: 0.92
- **[P1] Page mirror exports by update time:** `convex/skills.ts:10408-10413`
  - body: The mirror workflow described in the PR is hourly incremental sync, but this query ranges over `createdAt`. Publishing a new version or editing metadata on an old skill will not appear in later date windows, leaving the mirror stale; page by the appropriate update/version timestamp instead.
  - confidence: 0.9
- **[P1] Rate-limit failed export auth before token validation:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: The handler calls `requireExportAuth` before `applyRateLimit`, so missing or invalid tokens return 401 without consuming the export IP bucket. That contradicts the documented missing/invalid-token fallback and leaves the new endpoint open to unlimited failed-auth probes.
  - confidence: 0.88
- **[P1] Add an aggregate export size guard:** `convex/httpApiV1/skillsV1.ts:1740-1741`
  - body: The endpoint reads every exported blob into memory and then builds a full merged ZIP. With `limit=1000` and existing publish limits allowing large skill bundles, a valid request can exceed Convex action memory; track total bytes/files and cap the export before reading blobs.
  - confidence: 0.86
