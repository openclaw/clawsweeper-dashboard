## Best Possible Solution

- **[P1] Filter exports through public visibility gates:** `convex/skills.ts:9637-9639`
  - body: `listByDateRange` selects `skillSearchDigest` rows only by the active-created index, then the handler reads storage blobs directly.   - body: `requireExportAuth` runs before `applyRateLimit`, so missing or invalid bearer-token attempts return without consuming the export/IP bucket or returning rate headers. The documented API behavior says missing or invalid tokens fall back to IP enforcement.
  - confidence: 0.84
- **[P2] Reserve generated metadata paths:** `convex/httpApiV1/skillsV1.ts:1689-1691`
  - body: The handler always appends `${slug}/_meta.json`, but a published skill can already contain `_meta.json` as a valid file path. In that case `buildMergedExportZip` throws on the duplicate path and turns a valid export page into a 500.
  - confidence: 0.82
- **[P2] Bound total export bytes before reading blobs:** `convex/httpApiV1/skillsV1.ts:1625`
  - body: The endpoint can select up to 1000 skills and then read every selected blob into memory before zipping; current skill versions may be up to 50 MB each. Paginate by a total byte budget or otherwise cap the selected blobs before storage reads.
  - confidence: 0.78
