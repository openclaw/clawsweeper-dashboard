## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Filter exports through the public visibility boundary:** `convex/skills.ts:10411-10414`
  - body: `listByDateRange` reads active digests by created date and the handler then exports latest-version blobs directly, bypassing the public/download checks that block hidden, removed, pending-scan, suspicious, or malware-blocked skills. A bulk archive endpoint needs to apply the same approved visibility boundary before returning files.
  - confidence: 0.92
- **[P1] Track updates and tombstones for mirror sync:** `convex/skills.ts:10413`
  - body: The stated mirror use case relies on hourly incremental sync, but this cursor only scans `createdAt` for non-soft-deleted rows. Republished versions, renames, moderation changes, and deletes of old skills will not be exported, so mirrors can silently stay stale.
  - confidence: 0.9
- **[P2] Rate-limit before authenticating export tokens:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: This handler calls `requireExportAuth` before `applyRateLimit`, so invalid bearer-token traffic still reaches token hashing and token lookup before the export IP bucket is consumed. Put rate limiting ahead of auth, or add a failed-auth throttle, so bad tokens cannot bypass the intended export limiter.
  - confidence: 0.88
- **[P2] Cap aggregate export size before buffering blobs:** `convex/httpApiV1/skillsV1.ts:1693-1741`
  - body: The only per-request cap is skill count; the handler fetches all blobs, converts them to byte arrays, then creates the ZIP. A few large skills can exceed action memory or response limits, so the endpoint needs aggregate byte/file caps before loading the archive.
  - confidence: 0.86
- **[P2] Move export out of the skill slug namespace:** `convex/http.ts:76-80`
  - body: Registering `/api/v1/skills/export` as an exact route shadows the documented `/api/v1/skills/{slug}` detail route, and current slug validation does not reserve `export`. Use a non-conflicting namespace or reserve/migrate the slug with compatibility coverage.
  - confidence: 0.84
