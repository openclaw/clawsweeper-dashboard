## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Apply export rate limiting before auth returns:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: The handler returns from `requireExportAuth` before `applyRateLimit`, so missing or invalid bearer tokens never consume the documented IP fallback bucket. For a privileged export endpoint, token-spray traffic should be throttled before returning 401/403 or through an explicit failed-auth bucket.
  - confidence: 0.91
- **[P1] Filter export rows through the public visibility policy:** `convex/skills.ts:10411-10413`
  - body: `listByDateRange` walks `skillSearchDigest.by_active_created` with only `softDeletedAt` in the key. That can include suspicious or moderation-blocked rows in a mirror export even though the public catalog docs require mirrors to stay inside public filters; reuse the public/nonsuspicious policy or document a deliberate privileged export.
  - confidence: 0.92
- **[P1] Page incremental exports by update time:** `convex/skills.ts:10408-10413`
  - body: The PR says this supports hourly incremental mirror sync, but `startDate`/`endDate` are applied to `createdAt`. Republishing an existing skill changes `updatedAt`, not `createdAt`, so a mirror that follows this API after its initial crawl will miss updated skills.
  - confidence: 0.9
- **[P1] Bound archive size before fetching blobs:** `convex/httpApiV1/skillsV1.ts:1693-1741`
  - body: A valid page can enqueue every file for up to 1000 skills and then materialize each Blob and ArrayBuffer before building the ZIP. Add aggregate file-count and byte limits before storage reads, or split the export so a large catalog page cannot exhaust the action runtime.
  - confidence: 0.87
