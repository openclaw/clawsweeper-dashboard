## Summary

Keep open. Current main does not implement this export endpoint, and the PR still introduces a security-sensitive bulk archive API with unresolved public-visibility, auth-rate-limit, metadata-collision, sizing, product-contract, and proof blockers.

## What This Changes

The branch adds an admin-only `GET /api/v1/skills/export` ZIP endpoint with route constants, export auth/rate-limit helpers, ZIP merge utilities, and a date-range query over `skillSearchDigest`.

## Best Possible Solution

Define a maintainer-approved mirror/export contract that reuses public visibility and download gates, rate-limits invalid auth at the IP bucket, avoids archive metadata collisions, bounds archive size, documents the endpoint, and includes focused tests plus redacted runtime proof.

## Reproduction Assessment

Yes for the review findings by source inspection: a non-soft-deleted hidden, removed, pending-scan, suspicious, or malware-blocked digest in range can be selected, auth failures return before rate limiting, `_meta.json` can collide, and blob export is byte-unbounded. I did not run the PR branch.

## Solution Assessment

No; this is not yet the best way to solve mirror sync because it adds a security-sensitive bulk archive surface before the trust model, public-filter contract, archive sizing, and runtime proof are settled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Filter export rows through public moderation gates:** `convex/skills.ts:9639`
  - body: `listByDateRange` pages `skillSearchDigest` by `softDeletedAt` and `createdAt` only. Hidden, removed, pending-scan, suspicious, or malware-blocked rows can still be returned and then have their blobs read directly, bypassing the public list/download moderation gates.
  - confidence: 0.94
- **[P2] Rate-limit export auth failures before token lookup:** `convex/httpApiV1/skillsV1.ts:1503-1507`
  - body: The handler authenticates first and returns missing/invalid bearer responses before `applyRateLimit`, so those attempts never consume the export/IP bucket. Existing API behavior falls invalid auth back to IP enforcement; move the limiter ahead of export auth and preserve rate headers on auth failures.
  - confidence: 0.84
- **[P2] Avoid colliding with skill-owned _meta.json files:** `convex/httpApiV1/skillsV1.ts:1689-1690`
  - body: The export always writes `${slug}/_meta.json`, but published skill files can already use that path. When such a skill appears in a page, `buildMergedExportZip` detects the duplicate and throws, turning a valid export request into a 500.
  - confidence: 0.82
- **[P2] Cap export size before reading blobs:** `convex/httpApiV1/skillsV1.ts:1625`
  - body: The endpoint caps the number of skills but not total bytes, then reads every blob and builds the full ZIP in memory. Current publish limits allow 50MB per skill bundle, so a valid page can exceed action memory/response limits instead of returning a controlled error or smaller page.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The diff introduces a security-sensitive bulk archive endpoint that can expose moderated skill blobs and lets auth failures bypass endpoint rate limiting.

Concerns:

- **[high] Bulk export can include moderated blocked skills:** `convex/skills.ts:9639`
  - body: The export query only constrains `softDeletedAt` via `by_active_created`; it does not enforce current public visibility rules or the single-skill download blocks for malware, pending-scan, removed, hidden, or suspicious skills before reading storage blobs.
  - confidence: 0.94
- **[medium] Unauthorized export attempts bypass the limiter:** `convex/httpApiV1/skillsV1.ts:1503`
  - body: `requireExportAuth` runs before `applyRateLimit`, so invalid bearer-token attempts can trigger token parsing/hash/lookup work without consuming the endpoint's export/IP rate-limit bucket.
