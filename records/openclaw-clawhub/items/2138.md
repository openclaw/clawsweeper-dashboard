## Review Findings

Overall confidence: 0.88

Full review comments:

- **[P1] Move export out of the slug route namespace:** `convex/http.ts:77-79`
  - body: `/api/v1/skills/export` is currently a slug-shaped skill detail URL under the existing `/api/v1/skills/` prefix router, and `export` is not reserved by the slug validator. Registering this exact endpoint makes a skill with slug `export` unreachable through the normal detail API, so use a non-conflicting route such as `/api/v1/skills/-/export` or add an explicit migration/reservation plan.
  - confidence: 0.9
- **[P1] Reuse the public skill visibility gate:** `convex/skills.ts:10411-10413`
  - body: `listByDateRange` pages `skillSearchDigest.by_active_created` and the handler then reads internal version blobs directly, bypassing the current `isPublicSkillDoc`/download checks that suppress non-active and malware-blocked skills. Build the export from the same approved public boundary, or make the moderator-only boundary explicit before exposing files in bulk.
  - confidence: 0.88
- **[P2] Throttle export authentication before token work:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: The handler calls `requireExportAuth` before `applyRateLimit`, so invalid bearer tokens and non-admin credentials can still trigger token hashing and auth lookups outside the export bucket. Move the endpoint rate limit ahead of auth, or add a dedicated auth-failure throttle that runs before token lookup.
  - confidence: 0.86
- **[P2] Enforce an archive size budget:** `convex/httpApiV1/skillsV1.ts:1693-1741`
  - body: One authorized request can select up to 1000 skills, fetch every file blob, convert each to an `ArrayBuffer`, and zip the full set without any total byte or file-count cap. Add an aggregate budget and fail or paginate before reading/compressing enough data to exhaust the action's memory or time limits.
  - confidence: 0.84
