Full review comments:

- **[P1] Keep exports inside public visibility rules:** `convex/skills.ts:10411-10421`
  - body: `listByDateRange` returns raw `skillSearchDigest` rows from `by_active_created` with only `softDeletedAt` constrained, and the handler later reads raw version/storage data. Existing public list/file paths run through `toPublicSkill()`/read checks, so this can bulk-export hidden or moderation-blocked content that public mirror clients must not receive.
  - confidence: 0.92
- **[P1] Rate-limit failed export auth attempts:** `convex/httpApiV1/skillsV1.ts:1571-1575`
  - body: The handler calls `requireExportAuth()` before `applyRateLimit()`, so missing or invalid Bearer tokens return 401 without consuming the export/IP bucket. That breaks the documented fallback where invalid-token traffic is still IP-throttled and leaves this high-value endpoint exposed to unmetered token guessing.
  - confidence: 0.95
- **[P1] Use update-aware pagination for mirror sync:** `convex/skills.ts:10408-10413`
  - body: The export cursor is based on `createdAt`, but the PR's stated hourly mirror sync needs to see republishes, restores, hides, deletes, and moderation changes on existing skills. Those changes update `updatedAt` or visibility state without changing creation time, so an incremental mirror can miss removals or changed files.
  - confidence: 0.9
- **[P1] Cap export bytes before zipping:** `convex/httpApiV1/skillsV1.ts:1693-1741`
  - body: The request limit caps skills, not total bytes or file count; the handler fetches every storage blob, calls `arrayBuffer()` for each, stores all ZIP entries, then builds the merged ZIP in memory. A valid admin export over large skills can exceed Convex action memory/time and affect availability.
  - confidence: 0.88
