## Summary

Keep open; this is a useful mirror-sync feature request, but the current PR is not merge-ready because it creates a privileged bulk export boundary without maintainer approval and has blocking security, compatibility, sync-correctness, availability, and proof gaps.

## What This Changes

The PR adds an admin-gated `GET /api/v1/skills/export` endpoint with ZIP aggregation, cursor pagination, export auth, an export rate-limit tier, and route/schema wiring for mirror synchronization.

## Best Possible Solution

First decide the bulk mirror-export product and security contract; if approved, implement it under a reserved namespace with public-visibility filtering, update-based pagination or a change feed, failed-auth throttling, aggregate size budgets, compatibility tests, docs/specs, and redacted live proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Filter exports through the public visibility contract:** `convex/skills.ts:10411-10420`
  - body: The export query returns raw active digest rows and the handler reads latest-version blobs directly, so hidden/non-active or malware-blocked rows can bypass the `toPublicSkill`/`isPublicSkillDoc` filtering used by public APIs. For a mirror endpoint, build the export from the same public visibility contract or require an explicit maintainer-approved staff-only content boundary.
  - confidence: 0.93
- **[P1] Throttle failed export auth attempts:** `convex/httpApiV1/skillsV1.ts:1571-1575`
  - body: `requireExportAuth` runs before `applyRateLimit`, so missing, malformed, revoked, or non-admin tokens return 401/403 without consuming any IP/user rate-limit bucket. That contradicts the documented auth-aware model and leaves export-token probing unmetered; apply the export rate limit before returning auth failures, while preserving useful error text.
  - confidence: 0.91
- **[P1] Page mirror sync by updates, not creation time:** `convex/skills.ts:10407-10413`
  - body: The PR describes hourly mirror sync, but `listByDateRange` windows on `createdAt`. Any skill created before the mirror's current window and later republished, renamed, moderated, or otherwise updated will not appear in incremental exports, leaving mirrors stale; use `updatedAt` or a durable change feed cursor for sync.
  - confidence: 0.88
- **[P1] Add aggregate export size limits before buffering blobs:** `convex/httpApiV1/skillsV1.ts:1693-1741`
  - body: The handler schedules all file blobs for the page, reads every blob into memory, and then builds one ZIP without a total byte or file-count ceiling. A single allowed request can exceed Convex action memory/time limits if a page contains large or many-file skills; enforce an aggregate budget and fail with a clear partial-page response before loading all bytes.
  - confidence: 0.86
- **[P2] Avoid stealing the `export` skill detail route:** `packages/schema/src/routes.ts:30`
  - body: `/api/v1/skills/{slug}` is a documented detail route and `export` is not currently a reserved skill slug, but this adds `/api/v1/skills/export` as an exact route. Use a reserved collection namespace such as `/api/v1/skills/-/export` or reserve/migrate the slug explicitly so existing and future slug reads do not become ambiguous.
  - confidence: 0.84
