Full review comments:

- **[P1] Keep exports inside approved visibility rules:** `convex/skills.ts:10411-10413`
  - body: `listByDateRange` returns non-deleted digest rows from `by_active_created` without passing them through the current public visibility checks or non-suspicious filtering. That lets a mirror export include content that the public API/docs say third parties must not mirror, so the export feed needs an approved visibility policy and matching filters before merge.
  - confidence: 0.93
- **[P1] Rate-limit failed export auth attempts:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: The handler calls `requireExportAuth` and returns missing/invalid-token responses before `applyRateLimit` runs. Current API docs say missing or invalid tokens fall back to IP enforcement, but this ordering leaves failed export auth traffic outside that throttle and without 429 headers.
  - confidence: 0.91
- **[P1] Use update-aware pagination for mirror sync:** `convex/skills.ts:10411-10413`
  - body: The export query pages by `createdAt`, but the stated hourly mirror use case needs to observe changes to existing skills: new versions, hides, restores, deletes, and moderation transitions. A created-date cursor will miss those changes and make downstream mirrors drift from ClawHub.
  - confidence: 0.9
- **[P1] Cap aggregate export size before reading blobs:** `convex/httpApiV1/skillsV1.ts:1740-1741`
  - body: A single valid request can fetch blobs for up to 1000 skills, convert each to `Uint8Array`, and then build one ZIP without any aggregate byte or file-count cap. Add a total-size budget before materializing blobs so export requests cannot exhaust action memory or CPU.
  - confidence: 0.87
