## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Filter exports through public visibility gates:** `convex/skills.ts:10411-10413`
  - body: The export query pages raw `skillSearchDigest` rows by `softDeletedAt` only and then packages their latest-version blobs directly. That bypasses the current public visibility policy and documented mirror boundary, so active-but-hidden, malware-blocked, or suspicious skills can be included in a bulk mirror ZIP. Gate the query or packaging through the same public visibility rules before exposing this endpoint.
  - confidence: 0.92
- **[P2] Rate-limit export attempts before auth lookup:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: This handler authenticates with `requireExportAuth` before calling `applyRateLimit`, so missing or invalid export tokens return 401 without consuming the export IP bucket. Current API docs say invalid tokens fall back to IP enforcement, and this leaves token hashing/lookup work unthrottled on a high-value endpoint. Apply a conservative IP/export limiter before token lookup, then refine to the authenticated bucket after auth succeeds.
  - confidence: 0.9
- **[P2] Cap export bytes before reading blobs:** `convex/httpApiV1/skillsV1.ts:1693`
  - body: The endpoint can fetch blobs for up to 1000 skills and then materialize every blob plus the final ZIP in memory, with no aggregate byte or file-count budget. A few large or file-heavy skills can exhaust the Convex action even though each stored file is valid. Sum version file sizes/counts and fail early before `ctx.storage.get` or `arrayBuffer()` once a documented export budget is exceeded.
  - confidence: 0.88
