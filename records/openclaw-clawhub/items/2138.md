## Best Possible Solution

Define a maintainer-approved public-mirror or privileged-staff export contract, then implement that boundary with updated-time pagination, failed-auth throttling, aggregate size protection, focused tests, docs, and redacted runtime proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Preserve the public visibility boundary:** `convex/skills.ts:10413`
  - body: The export query scans `skillSearchDigest.by_active_created` with only the active soft-delete prefix, and the handler zips those versions without running the same public visibility gate used by catalog reads. For a mirror API, this can export hidden or moderation-blocked content; filter through the approved public surface or make this an explicitly privileged staff export with tests and docs.
  - confidence: 0.91
- **[P1] Page mirror exports by update time:** `convex/skills.ts:10413`
  - body: The stated use case is hourly mirror sync, but the export window is keyed on `createdAt`. Existing skills that publish a new version after their original creation window will be missed, so the export cursor needs to track `updatedAt` or another version/update event stream.
  - confidence: 0.9
- **[P1] Rate limit failed export auth before token checks:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: `requireExportAuth` runs before `applyRateLimit`, so missing or invalid Bearer tokens return 401 before consuming the export IP bucket. That bypasses the documented invalid-token fallback to IP enforcement and leaves the new expensive endpoint exposed to unthrottled auth failures.
  - confidence: 0.89
- **[P1] Cap aggregate export bytes before zipping:** `convex/httpApiV1/skillsV1.ts:1740`
  - body: The handler reads every selected file blob into memory and then builds one ZIP for up to 1000 skills without checking total file bytes. Since a single published skill bundle can be much larger than the PR's assumed average, a valid request can exhaust backend memory; enforce an aggregate byte cap or use a narrower paging/streaming design before reading blobs.
  - confidence: 0.86
