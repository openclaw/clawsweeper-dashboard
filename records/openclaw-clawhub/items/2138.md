- **[P1] Filter export rows through the visibility boundary:** `convex/skills.ts:10411-10420`
  - body: `listByDateRange` pages raw `skillSearchDigest` rows by `softDeletedAt` only, so hidden, malware-blocked, or suspicious rows can be exported as full blobs. Apply an explicit maintainer-approved public/moderation export policy before returning file contents.
  - confidence: 0.93
- **[P1] Throttle failed export auth attempts:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: The handler returns missing, invalid, and non-admin auth failures before `applyRateLimit` runs, so denied export attempts never consume the new export/IP bucket. Apply rate limiting before auth or add an explicit denied-auth throttle path.
  - confidence: 0.91
- **[P1] Add an aggregate export budget before reading blobs:** `convex/httpApiV1/skillsV1.ts:1693`
  - body: This path can fetch up to 1000 skills, read every file into memory, keep all ZIP entries, and then compress them without a total byte or file-count cap. Sum version file sizes and reject or page smaller before any storage reads.
  - confidence: 0.9
- **[P1] Use updated-time pagination for mirror sync:** `convex/skills.ts:10413`
  - body: The stated hourly mirror sync will miss republished or edited skills because this query scans `by_active_created`; existing skill updates keep their original creation time. Page by `updatedAt` or a dedicated change cursor for incremental exports.
  - confidence: 0.9
- **[P1] Move export out of the skill slug namespace:** `convex/http.ts:77-79`
  - body: `/api/v1/skills/export` conflicts with the documented `/api/v1/skills/{slug}` route because `export` is not a reserved skill slug. Use a collection action path such as `/api/v1/skills/-/export` and add a regression test.
  - confidence: 0.85
