Full review comments:

- **[P1] Preserve public visibility filters in export:** `convex/skills.ts:10411-10413`
  - body: `listByDateRange` returns raw `skillSearchDigest` rows and the export handler then fetches blobs by internal version id, so this endpoint can export files for skills that the current public list/download paths would reject as hidden, removed, or malware-blocked. Reuse the public eligibility/download path or make this a documented audited staff export before exposing bulk files.
  - confidence: 0.9
- **[P1] Page mirror sync by update activity:** `convex/skills.ts:10408-10413`
  - body: The export cursor and date range are based on `createdAt`, but the stated mirror use case is hourly incremental sync. Republished, hidden, restored, or metadata-updated skills created outside the requested window will never be returned, so a mirror can silently miss changed content; page by update/change activity or add an explicit change feed.
  - confidence: 0.87
- **[P1] Throttle failed export auth before returning:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: `requireExportAuth` runs before `applyRateLimit`, so missing tokens, invalid tokens, and valid non-admin tokens return 401/403 without consuming the export bucket. Put a cheap export/IP rate check before auth failures, or otherwise account for failed auth attempts, so noisy callers cannot bypass throttling on this new privileged surface.
  - confidence: 0.84
- **[P1] Bound aggregate export bytes before zipping:** `convex/httpApiV1/skillsV1.ts:1740-1780`
  - body: The handler collects blob tasks for up to 1000 skills, fetches all blobs, stores each `arrayBuffer()` in `zipEntries`, and then builds one ZIP without a total byte/file cap. A valid export can exceed action memory or time limits, so enforce aggregate limits before reading/zipping and return a bounded pagination error.
  - confidence: 0.86
- **[P2] Avoid reserving an uploadable path as metadata:** `convex/httpApiV1/skillsV1.ts:1757-1759`
  - body: The export injects `${slug}/_export_skill_meta.json`; current publish sanitization rejects traversal but does not reserve that filename, so any existing skill file with the same path makes `buildMergedExportZip` throw and fail the whole page. Put metadata at a non-colliding top-level manifest path or handle conflicts deterministically.
  - confidence: 0.76
