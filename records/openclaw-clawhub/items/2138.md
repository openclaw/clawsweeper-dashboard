## Best Possible Solution

Define the export contract first, then implement either a public-safe incremental mirror export that preserves the existing visibility/download gates or a documented privileged staff export with explicit audit, auth, rate, and size limits.

## Solution Assessment

No. The useful mirror-sync goal needs a maintainer-approved export boundary plus update-based pagination, public/download visibility preservation or explicit privileged-audit semantics, failed-auth throttling, and aggregate size limits before this is the best implementation path.

- **[P1] Preserve the public download boundary:** `convex/skills.ts:10413`
  - body: This query pages `skillSearchDigest` rows with only `softDeletedAt` constrained, and the handler then reads latest-version storage blobs directly. That bypasses the existing `getBySlug`/download checks that reject pending, hidden, removed, and malware-blocked skills, so a bulk export can include files current public download APIs would block. Reuse the same visibility/download gate or make the privileged staff-only contract explicit and audited.
  - confidence: 0.91
- **[P1] Page mirror exports by update time:** `convex/skills.ts:10408-10413`
  - body: The export is intended for incremental mirror sync, but this range scan uses `createdAt`. Existing skills whose latest tag, files, metadata, owner, or moderation state changes after creation will not appear in later windows, leaving mirrors stale. Use `updatedAt`, a version/event cursor, or another tested change stream that covers updates as well as new skills.
  - confidence: 0.9
- **[P1] Rate-limit failed export authentication:** `convex/httpApiV1/skillsV1.ts:1571-1575`
  - body: `requireExportAuth` runs before the export rate limit is applied, so requests with no token or invalid tokens return without consuming the IP export bucket. That leaves a new auth-checking endpoint open to unthrottled token guessing or denial traffic; apply the IP rate limit before auth, then upgrade to the authenticated/admin bucket after successful auth if needed.
  - confidence: 0.88
- **[P1] Bound the ZIP export size:** `convex/httpApiV1/skillsV1.ts:1740-1741`
  - body: A single request can fetch up to 1000 skills, read every file blob into memory, and then build a merged ZIP without any aggregate byte, file-count, or output-size cap. A few large or many-file skills can exceed Convex action memory/time limits and make this endpoint unreliable; enforce a total budget and return a paginated/413-style response before materializing the archive.
  - confidence: 0.86
