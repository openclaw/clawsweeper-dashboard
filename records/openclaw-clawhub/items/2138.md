Full review comments:

- **[P1] Preserve the existing skill detail route for export slugs:** `convex/http.ts:77-79`
  - body: Registering `/api/v1/skills/export` under the existing slug namespace hijacks `GET /api/v1/skills/export` from the skill-detail router, while `export` is not in the reserved slug list. Any existing or future skill with that slug becomes unreachable through the documented detail URL; use a non-slug namespace such as `/api/v1/exports/skills` or add a compatibility migration/reservation with focused tests.
  - confidence: 0.9
- **[P1] Filter exports through the public visibility boundary:** `convex/skills.ts:10411-10420`
  - body: This range query returns active `skillSearchDigest` rows directly and never applies `toPublicSkill`, `nonSuspiciousOnly`, or a reviewed export-specific moderation policy before the handler reads files. A mirror export can include flagged or moderation-blocked content that the public API/docs keep out of third-party mirrors; reuse the public filtering path or define an explicit maintainer-approved export boundary.
  - confidence: 0.93
- **[P2] Use update-aware pagination for mirror sync:** `convex/skills.ts:10407-10409`
  - body: The cursor window is based on `createdAt`, so republished old skills, renames, moderation changes, deletions, and other updates do not reappear in hourly incremental windows. Mirror sync needs `updatedAt`/change-log pagination plus deletion and moderation tombstones, or downstream mirrors will silently go stale.
  - confidence: 0.9
- **[P1] Rate-limit failed export auth before token lookup:** `convex/httpApiV1/skillsV1.ts:1571-1575`
  - body: `requireExportAuth` runs before `applyRateLimit`, and invalid bearer tokens still go through token hashing and token lookup before rejection. That leaves an unauthenticated load path on token state; apply an IP/failure bucket before token lookup or add a dedicated failed-auth limiter for this endpoint.
  - confidence: 0.88
- **[P2] Bound the export archive before buffering blobs:** `convex/httpApiV1/skillsV1.ts:1740-1741`
  - body: The handler reads every selected file into memory and then builds one ZIP, with only a 1000-skill count limit and no aggregate byte/file/response cap. One authorized request can exceed Convex action memory or response limits; enforce total bytes/files or split/stream pages before calling `arrayBuffer()` for all blobs.
  - confidence: 0.86
