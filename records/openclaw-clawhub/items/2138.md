## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Filter exports through public visibility gates:** `convex/skills.ts:10411-10413`
  - body: This query walks `skillSearchDigest.by_active_created` with only the `softDeletedAt` prefix, then the handler packages the returned versions directly. Current public list/download paths filter through the public visibility and moderation gates, and docs tell mirrors not to copy hidden or moderation-blocked content, so this can expose rows that normal public APIs intentionally hide.
  - confidence: 0.92
- **[P2] Rate-limit export attempts before auth lookup:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: The handler returns from `requireExportAuth` before the export limiter runs, so invalid bearer tokens can repeatedly trigger token hashing and database lookup without consuming the export IP bucket. Apply the export limiter before or inside auth so invalid-token traffic falls back to IP enforcement like the documented model.
  - confidence: 0.86
- **[P2] Cap export bytes before reading blobs:** `convex/httpApiV1/skillsV1.ts:1693-1740`
  - body: A single allowed request can select up to 1000 skills, read every file blob, copy each `arrayBuffer`, and then build a ZIP with no aggregate size or file-count cap. Large skills can push the Convex action into memory pressure, so enforce a total export budget before and while adding blobs.
  - confidence: 0.84
