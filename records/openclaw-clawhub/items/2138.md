- **[P1] Filter exports through public visibility gates:** `convex/skills.ts:10412-10413`
  - body: The export query reads raw `skillSearchDigest` rows on `by_active_created` and returns them directly, so an admin mirror export can include hidden or moderation-blocked skills that current public APIs hide through `toPublicSkill()`/`isPublicSkillDoc()`.   - confidence: 0.9
- **[P2] Rate-limit export attempts before auth lookup:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: `requireExportAuth()` runs before `applyRateLimit()`, so missing or invalid tokens return 401 without consuming the export IP bucket. Existing HTTP API docs say invalid auth falls back to IP enforcement; this new path should preserve that abuse throttle while still returning useful auth errors.
  - confidence: 0.86
- **[P2] Cap export bytes before reading blobs:** `convex/httpApiV1/skillsV1.ts:1740`
  - body: The handler collects the page's storage blobs and converts each blob with `arrayBuffer()` before ZIP creation, but there is no total byte budget. Since a single skill can be much larger than the PR's average-size estimate, a 1000-skill page can exhaust memory or runtime; enforce a cumulative byte cap before reading/adding blobs.
  - confidence: 0.84
