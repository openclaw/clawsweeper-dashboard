## What This Changes

The PR adds a new admin-authenticated `GET /api/v1/skills/export` endpoint that returns paginated ZIP archives of skill files for mirror synchronization.

## Best Possible Solution

If maintainers approve bulk mirror exports, land a reserved, explicitly documented endpoint that uses an approved visibility boundary, update-time pagination, failed-auth throttling, aggregate export budgets, focused tests/spec notes, and redacted live proof.

  - confidence: 0.88
- **[P2] Throttle failed export authentication attempts:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: The handler calls `requireExportAuth` before `applyRateLimit`, so missing, invalid, and non-admin tokens return 401/403 before consuming the export bucket. Existing API docs say missing/invalid tokens fall back to IP enforcement; apply rate limiting first or make failed export auth consume an IP bucket.
  - confidence: 0.9
- **[P2] Page mirror sync by updates, not creation time:** `convex/skills.ts:10413`
  - body: The PR describes hourly incremental mirror sync, but `listByDateRange` pages on `by_active_created`. Republished or edited skills keep their original creation time, so a mirror using hourly date windows can miss changed files; page by `updatedAt` or make the API contract explicitly creation-only.
  - confidence: 0.9
- **[P2] Bound archive size before reading every blob:** `convex/httpApiV1/skillsV1.ts:1740`
  - body: For each page the handler reads every selected blob into memory and then builds a single ZIP, with no aggregate byte, file-count, or compressed-size budget. One admin export can therefore consume much more than the assumed average; add a hard export budget or streaming/chunking guard before materializing blob contents.
  - confidence: 0.84
