Full review comments:

- **[P1] Move export off the skill-slug route:** `convex/http.ts:77-79`
  - body: `/api/v1/skills/export` lives inside the documented `/api/v1/skills/{slug}` namespace, and `export` is not reserved by the slug validator. This can make a real or future `export` skill unreachable through the normal detail API, so use a non-slug collection route or add an explicit migration/reservation plan.
  - confidence: 0.9
- **[P1] Filter exports through the public skill boundary:** `convex/skills.ts:10413`
  - body: `listByDateRange` pages `skillSearchDigest.by_active_created` with only the soft-delete prefix, then the handler reads internal versions and blobs for those rows. That skips the current public `toPublicSkill`/moderation gates, so hidden or malware-blocked files can be included in the archive.
  - confidence: 0.94
- **[P1] Throttle export auth failures before token lookup:** `convex/httpApiV1/skillsV1.ts:1571`
  - body: The handler returns from `requireExportAuth` before calling `applyRateLimit`, so missing, invalid, and non-admin export attempts do not consume the new export bucket. Put an IP/export throttle before the privileged token lookup or make the auth helper consume the appropriate failure bucket.
  - confidence: 0.88
- **[P2] Add an aggregate export size budget:** `convex/httpApiV1/skillsV1.ts:1693`
  - body: The handler queues the page's file blobs, fetches them, materializes each as a `Uint8Array`, and only then compresses the archive. Without a total file/byte cap, a large page can exceed Convex action memory even when the item count limit is respected.
  - confidence: 0.86
