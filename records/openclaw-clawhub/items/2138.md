## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Avoid shadowing skill detail for the export slug:** `convex/http.ts:77-79`
  - body: This exact route claims `/api/v1/skills/export`, but the current API also documents `/api/v1/skills/{slug}` and the slug validator does not reserve `export`. A skill with that slug would no longer resolve through the detail endpoint, so place the archive endpoint outside the slug namespace or add an explicit reservation/migration plan.
  - confidence: 0.92
- **[P1] Keep export behind the public visibility gates:** `convex/skills.ts:10411-10420`
  - body: `listByDateRange` returns raw active digest rows and the handler reads internal version blobs directly, bypassing the current `toPublicSkill` / `isPublicSkillDoc` filtering used by list/detail APIs. The archive can therefore include hidden, removed, pending, or malware-blocked content unless the export applies the same public filter or has an explicit staff-only content policy.
  - confidence: 0.94
- **[P1] Rate-limit export attempts before auth rejects:** `convex/httpApiV1/skillsV1.ts:1571-1575`
  - body: The handler authenticates before calling `applyRateLimit`, so missing, invalid, and non-admin tokens return without consuming the export bucket. Move rate limiting ahead of auth or explicitly charge auth failures to an IP bucket so token probing and expensive auth lookups are throttled.
  - confidence: 0.9
- **[P2] Use updated pagination for incremental mirror sync:** `convex/skills.ts:10408-10413`
  - body: The PR says mirrors can run hourly incremental exports, but this query pages `by_active_created` and compares `createdAt`, so updates to existing skills will not be exported. Use an `updatedAt` cursor/index for incremental sync or document this as create-only export semantics.
  - confidence: 0.88
- **[P2] Bound archive size before buffering blobs:** `convex/httpApiV1/skillsV1.ts:1693-1741`
  - body: The handler fetches all selected blobs, converts each to an ArrayBuffer, and then builds the final ZIP with no aggregate file or byte limit. A valid admin request for large skills can exhaust action memory or response limits, so enforce a total budget before reading and zipping content.
  - confidence: 0.87
