## What This Changes

The PR adds an admin-only `GET /api/v1/skills/export` endpoint that pages skill digests by created date and returns latest skill files plus metadata in a merged ZIP archive.

## Best Possible Solution

Define an approved mirror/export contract first, then implement it with public/download filtering parity, update/tombstone sync semantics, pre-auth throttling, route-namespace safety, audit logging, and hard size caps.

## Reproduction Assessment

Not applicable as a user bug. For PR review, yes: the blockers are source-reproducible from the proposed export query, auth/rate-limit order, route registration, and archive-building code.

## Solution Assessment

No. A bulk mirror API may be useful, but this implementation is not the narrowest maintainable solution until the mirror contract and security boundary are approved and encoded in code, specs, and tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve public moderation filters in export:** `convex/skills.ts:10413`
  - body: The export query scans `by_active_created` and the handler archives raw latest-version files, so hidden, pending-scan, removed, suspicious, or malware-blocked skills that public list/download paths suppress can be included. Use the same public/download visibility checks, or land an explicitly approved staff-only export boundary with tests.
  - confidence: 0.95
- **[P1] Use an update-safe mirror cursor:** `convex/skills.ts:10408-10415`
  - body: The mirror use case depends on hourly incremental sync, but this cursor ranges over `createdAt`. Republishes, renames, moderation hides/removals, and other updates to old skills will not appear in a later `startDate` window, leaving mirrors stale unless the contract uses updated/tombstone pagination.
  - confidence: 0.91
- **[P1] Throttle failed auth before token lookup:** `convex/httpApiV1/skillsV1.ts:1568-1571`
  - body: `requireExportAuth` runs before `applyRateLimit`, so invalid Bearer tokens perform token hash/user lookups and return 401 without consuming an IP-scoped export bucket. Run a pre-auth/IP rate check before token validation or make invalid-token failures fall back to IP enforcement.
  - confidence: 0.9
- **[P1] Cap the aggregate export size before zipping:** `convex/httpApiV1/skillsV1.ts:1697-1698`
  - body: Each request can read blobs for up to 1000 skills and then builds a copied ZIP in memory, but there is no total byte, file, or entry cap. A valid admin token can force large memory and response work that can exceed Convex limits.
  - confidence: 0.88
- **[P2] Avoid occupying a valid skill slug route:** `packages/schema/src/routes.ts:28`
  - body: Adding `/api/v1/skills/export` under the skill-slug prefix makes a skill named `export` unreachable on the documented `/api/v1/skills/{slug}` detail route, and `export` is not currently reserved. Use a non-slug namespace or reserve/migrate the slug with route-collision tests.
  - confidence: 0.84
