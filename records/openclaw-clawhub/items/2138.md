## Best Possible Solution

Define an approved mirror-sync contract first, then implement a namespaced, audited, size-capped export that matches public/download visibility and uses update/tombstone pagination with tests and real proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Preserve public moderation filters in export:** `convex/skills.ts:10412-10413`
  - body: The export query reads raw `skillSearchDigest` rows from `by_active_created` and the handler exports their latest blobs without applying the public skill/download visibility gates. That can bulk-export suspicious, pending-scan, malware-blocked, hidden, or otherwise non-public skills even though current docs direct mirrors to public APIs and downloads block those states. Reuse public/download filtering or require an explicitly approved mirror entitlement before exposing files.
  - confidence: 0.94
- **[P1] Use an update-aware sync cursor:** `convex/skills.ts:10412-10413`
  - body: The endpoint is described as hourly incremental mirror sync, but this query pages by `createdAt`. Any old skill that is republished, renamed, moderated, removed, or otherwise updated outside the requested creation window will not be returned, so mirrors can silently stay stale. Base the contract on `updatedAt` plus tombstones/moderation events, or implement a full-snapshot-only export.
  - confidence: 0.91
- **[P2] Throttle failed auth before token lookup:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: `exportSkillsV1Handler` calls `requireExportAuth` before `applyRateLimit`, so missing and invalid bearer-token attempts return before consuming the export rate bucket, while invalid tokens still exercise token hashing and lookup paths. Put a pre-auth IP bucket in front of token parsing or otherwise consume the unauthenticated bucket for failures.
  - confidence: 0.88
- **[P1] Avoid stealing the export slug route:** `convex/http.ts:76-79`
  - body: Adding an exact `/api/v1/skills/export` route changes the existing `/api/v1/skills/{slug}` namespace, and the current write validator does not reserve `export` as a skill slug. Existing or future skills with that slug lose their detail URL; use a reserved sub-namespace such as `/api/v1/skills/-/export` or include an explicit migration/reservation path.
  - confidence: 0.89
- **[P2] Cap aggregate export size before buffering:** `convex/httpApiV1/skillsV1.ts:1693-1741`
  - body: The handler fetches up to 1000 skills, reads every file blob into memory, then builds and copies one ZIP without a total byte, file-count, or response-size cap. A legitimate admin request over large skills can exceed Convex action memory or response limits; enforce aggregate caps before reading blobs and return a paginated/capped error.
  - confidence: 0.86
