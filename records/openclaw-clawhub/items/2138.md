Overall confidence: 0.88

Full review comments:

- **[P1] Preserve the public download boundary:** `convex/httpApiV1/skillsV1.ts:1602-1607`
  - body: The export page is populated from a raw digest query and then loads `latestVersionId` and storage blobs directly. That skips the current public visibility/download checks for hidden, removed, pending-scan, and malware-blocked skills, so please build each export item through the existing public gate or make the privileged boundary an explicit maintainer-approved contract.
  - confidence: 0.9
- **[P1] Page exports by update or version activity:** `convex/skills.ts:10425-10430`
  - body: The PR describes hourly incremental mirror sync, but this query bounds `by_active_created` with `startDate` and `endDate`. Republished versions, metadata edits, owner changes, and moderation changes on older skills will not reappear in later created-date windows, leaving mirrors stale; use an update/version-event cursor or require documented full rescans.
  - confidence: 0.88
- **[P1] Rate-limit export auth failures:** `convex/httpApiV1/skillsV1.ts:1561-1564`
  - body: The handler returns from `requireExportAuth` before consuming the export rate bucket. Missing or invalid token traffic therefore bypasses the documented IP fallback rate limit, so abusive requests can hammer token checks without the new `export` limits; apply the rate limit first or add a failed-auth bucket.
  - confidence: 0.86
- **[P1] Bound aggregate ZIP work before reading blobs:** `convex/httpApiV1/skillsV1.ts:1730-1731`
  - body: A single request can select up to 1000 skills, enqueue every version file, read each blob into memory, retain every file buffer in `zipEntries`, and then allocate the merged ZIP. Add aggregate file-count/byte limits before fetching blobs and fail or paginate instead of risking action memory and time exhaustion.
  - confidence: 0.84
