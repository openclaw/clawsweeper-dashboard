Full review comments:

- **[P1] Filter exports through public visibility rules:** `convex/skills.ts:10411-10413`
  - body: This query pages raw `skillSearchDigest` rows and the handler then fetches blobs directly, but current public list/detail/download paths filter moderation-hidden, removed, pending-scan, and malware-blocked skills. Build exports from the same public/download-safe visibility rules before packaging files.
  - confidence: 0.94
- **[P1] Use an update and tombstone cursor for mirrors:** `convex/skills.ts:10407-10409`
  - body: A mirror that pages only by `createdAt` will miss old skills that are updated, renamed, moderated, removed, or republished after their original create window. The export contract needs update-based pagination plus deletion/moderation tombstones, or downstream mirrors can silently serve stale records.
  - confidence: 0.9
- **[P2] Rate-limit failed export auth attempts:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: The handler authenticates before calling `applyRateLimit`, so missing or invalid bearer tokens return before any export bucket is consumed. Current API docs say missing/invalid tokens fall back to IP enforcement; keep that behavior here to avoid unbounded token lookup pressure on a privileged route.
  - confidence: 0.88
- **[P1] Cap export response size before zipping:** `convex/httpApiV1/skillsV1.ts:1693-1741`
  - body: One authorized request can schedule blob reads for up to 1000 skills, convert every blob to memory, and then build one copied ZIP response without aggregate byte or file-count limits. Add conservative caps and an explicit 413 or partial-error path before reading and zipping the full page.
  - confidence: 0.86
- **[P1] Avoid occupying the skill slug route:** `packages/schema/src/routes.ts:30`
  - body: `/api/v1/skills/{slug}` is the documented skill detail URL and `export` is not currently a reserved skill slug, so this exact route can steal an existing or future skill detail path. Put bulk operations under a reserved namespace such as `/api/v1/skills/-/export`, or add an explicit reservation and migration plan.
  - confidence: 0.89
