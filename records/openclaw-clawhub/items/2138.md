## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve public visibility filters in export:** `convex/skills.ts:10411-10413`
  - body: `listByDateRange` reads active digest rows by `softDeletedAt` only, then the handler loads internal versions and blobs for everything returned. Current public APIs filter out non-public moderation states and docs explicitly say mirrors must not bypass those filters, so this can bulk-export hidden, removed, suspicious, or malware-blocked skills unless maintainers deliberately approve a privileged staff export contract.
  - confidence: 0.94
- **[P1] Page mirror sync by updates, not creation time:** `convex/skills.ts:10408-10413`
  - body: The endpoint is described as hourly mirror sync, but the cursor walks `createdAt`. Any existing skill republished or restored after its original creation date will not appear in an incremental window, so mirrors can permanently miss updates; use an update-based cursor contract or another explicit delta source.
  - confidence: 0.9
- **[P1] Throttle failed export auth attempts:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: The handler runs `requireExportAuth` before `applyRateLimit`, so missing, invalid, and non-admin tokens return without consuming the new export bucket. Invalid tokens still cause token hashing and lookup work, which leaves this privileged endpoint without the same failed-auth throttling pattern as existing v1 handlers.
  - confidence: 0.87
- **[P1] Bound export size before materializing blobs:** `convex/httpApiV1/skillsV1.ts:1740-1741`
  - body: A single request can collect up to 1000 skills, fetch every file blob, convert each to an in-memory `Uint8Array`, and then build one ZIP copy with no total byte limit. A valid admin request for large skills can exhaust Convex action memory or time; enforce per-export size limits or stream/chunk the export before merging.
  - confidence: 0.88
- **[P2] Avoid metadata filename collisions:** `convex/httpApiV1/skillsV1.ts:1757-1759`
  - body: The export always writes `${slug}/_export_skill_meta.json`, but uploaded skill files can already use that valid path. In that case `buildMergedExportZip` throws a duplicate-path error and the whole page fails instead of exporting or reporting the one conflicting skill.
  - confidence: 0.79
