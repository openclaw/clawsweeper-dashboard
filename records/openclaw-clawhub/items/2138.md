Full review comments:

- **[P1] Filter export rows through the public visibility boundary:** `convex/skills.ts:10411-10413`
  - body: The export query pages `skillSearchDigest` by `by_active_created` with only the soft-delete prefix, then the handler reads raw latest-version files. Current public list/detail paths run rows through `toPublicSkill`/`isPublicSkillDoc`, which excludes hidden, non-active, and `blocked.malware` skills, so this can package content the documented public mirror surface says not to mirror.
  - confidence: 0.96
- **[P1] Cursor mirror sync by updates, not creation time:** `convex/skills.ts:10411-10413`
  - body: The PR promises hourly incremental mirror sync, but `listByDateRange` walks the `by_active_created` index. A skill created before the requested window but republished, renamed, moderated, or otherwise updated inside it keeps the old `createdAt`, so the mirror can miss changes and stay stale.
  - confidence: 0.93
- **[P2] Apply the export rate limit before token validation:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: This handler performs `requireExportAuth` before `applyRateLimit`. Invalid bearer tokens still force token hash/user lookups and return 401 without consuming the export IP bucket, unlike existing handlers that rate-limit first, so the privileged endpoint can be used for unauthenticated query load.
  - confidence: 0.9
- **[P1] Move export off the slug route namespace:** `convex/http.ts:77`
  - body: Adding exact `/api/v1/skills/export` takes over the documented `/api/v1/skills/{slug}` path for the valid slug `export`. Existing detail/file clients for that slug would receive export auth errors instead of skill data; use a reserved admin/system namespace or reserve/migrate the slug with compatibility tests.
  - confidence: 0.88
- **[P2] Cap aggregate export bytes before buffering blobs:** `convex/httpApiV1/skillsV1.ts:1740`
  - body: The handler reads every selected file blob into `Uint8Array` buffers and then builds another ZIP buffer, but the only cap is 1000 skills. A few large versions can exceed action memory or runtime before the response is built; enforce total bytes and file count before `arrayBuffer()` and return a controlled 413.
  - confidence: 0.86
