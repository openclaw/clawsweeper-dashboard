  - body: `requireExportAuth` runs before `applyRateLimit`, so missing, invalid, and non-admin tokens hit token lookup/role checks without consuming the export/IP bucket or returning rate headers. Apply rate limiting before token validation, or explicitly consume an IP bucket for failed auth paths.

Since a single published skill bundle can be much larger than the PR's average-size assumption, add aggregate file-count/byte limits before `ctx.storage.get` and `blob.arrayBuffer()` to avoid action memory/time failures.
  - confidence: 0.86

  - body: The mirror use case claims hourly incremental sync, but this query windows and cursors on `createdAt`, so new versions or metadata changes to existing skills are skipped after the initial creation window. Use an `updatedAt` index/cursor for incremental exports, or make this a clearly separate bootstrap-only endpoint.
