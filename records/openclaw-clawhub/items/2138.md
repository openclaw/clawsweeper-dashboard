## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Filter exports through the public visibility contract:** `convex/skills.ts:10411-10420`
  - body: The export query pages raw `skillSearchDigest.by_active_created` rows and the handler reads latest-version blobs directly, so hidden, pending, removed, or suspicious content can be bundled even though normal public list/download paths pass through `toPublicSkill` and moderation checks. Build the export from the same public visibility contract, or make any broader admin-only content boundary explicit and approved.
  - confidence: 0.92
- **[P1] Reserve a route that cannot shadow skill slugs:** `convex/http.ts:76-80`
  - body: Registering `/api/v1/skills/export` under the slug namespace conflicts with the documented `/api/v1/skills/{slug}` shape because `export` is not currently reserved as a skill slug. An existing or future `export` skill would return the bulk export auth/ZIP behavior instead of its skill detail; use a reserved namespace like `/api/v1/skills/-/export` or add a deliberate migration/reservation plan.
  - confidence: 0.88
- **[P1] Page mirror exports by updates, not creation time:** `convex/skills.ts:10408-10418`
  - body: The PR is meant to support hourly mirror sync, but the cursor range is anchored on `createdAt`, so republishing an older skill updates its files and `updatedAt` without moving it into a new created-date window. Use an `updatedAt` cursor or durable change feed so mirrors do not silently miss updates.
  - confidence: 0.9
- **[P2] Rate-limit failed export authentication attempts:** `convex/httpApiV1/skillsV1.ts:1571-1575`
  - body: `requireExportAuth` runs before `applyRateLimit`, so missing or invalid bearer tokens return without consuming the export IP bucket while still forcing auth parsing/hash lookups. Apply the export rate limit before the privileged auth check, or otherwise consume a failed-auth bucket, so invalid-token traffic is throttled too.
  - confidence: 0.86
- **[P1] Bound aggregate export size before loading blobs:** `convex/httpApiV1/skillsV1.ts:1693-1741`
  - body: The handler gathers all blob tasks, fetches every blob, converts each to an `ArrayBuffer`, and then `zipSync`s the whole archive without an aggregate byte or file-count budget. Since a single skill bundle may be up to 50 MB, a 1000-skill request can exceed Convex action memory or stall the function; enforce a total budget or stream/page at a smaller bounded unit.
  - confidence: 0.89
