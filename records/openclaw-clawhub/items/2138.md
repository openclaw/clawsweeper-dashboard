## What This Changes

The PR adds an admin-only `GET /api/v1/skills/export` endpoint that returns paginated merged ZIP archives of skill files and manifest data, plus a new export auth helper, rate-limit tier, ZIP helpers, and schema route constant.

Maintainers should first decide whether ClawHub wants a private bulk export/mirror entitlement; if approved, implement it behind a reserved route with explicit visibility scope, updated-time sync semantics, failed-auth throttling, aggregate budgets, focused tests, and redacted real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Throttle export attempts before auth returns:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: The handler calls `requireExportAuth` before `applyRateLimit`, so missing or invalid Bearer requests return 401 without consuming the new export/IP quota. Put throttling in front of the auth failure path, or explicitly consume a failed-auth bucket before returning.
  - confidence: 0.9
- **[P1] Keep export out of the skill slug namespace:** `convex/http.ts:77-79`
  - body: Registering exact `/api/v1/skills/export` under the skill detail prefix takes over the documented detail URL for a valid `export` slug; the slug validator does not reserve that word. Use a reserved control segment such as `/api/v1/skills/-/export` or a top-level export route and update slug reservations.
  - confidence: 0.88
- **[P1] Filter exports through public visibility rules:** `convex/skills.ts:10411-10419`
  - body: `listByDateRange` scans only the active-created digest index and returns raw rows, so hidden, blocked, or inactive-owner skills can be included in the bulk archive. Reuse the public visibility predicate or add an explicitly approved moderation-aware export scope before fetching blobs.
  - confidence: 0.92
- **[P1] Page incremental mirror sync by updated time:** `convex/skills.ts:10413`
  - body: The cursor walks `by_active_created`, but hourly mirror sync needs updates to existing skills as well as newly created skills. Edits, republishes, moderation changes, and metadata updates to older skills will be missed unless the incremental export uses an updated-time cursor or a separate full-snapshot mode.
  - confidence: 0.86
- **[P2] Add an aggregate export size budget:** `convex/httpApiV1/skillsV1.ts:1740-1741`
  - body: A request can select 1000 skills, read every file into memory, and build one ZIP without a total byte or file-count cap. Large skills can exhaust the Convex action memory or stall the endpoint, so enforce an aggregate budget before buffering the whole archive.
  - confidence: 0.84
