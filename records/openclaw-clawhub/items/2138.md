## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Throttle export requests before checking credentials:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: `requireExportAuth` returns 401/403 before `applyRateLimit` runs, so repeated missing or invalid bearer tokens never consume the new `export` IP bucket. Move the export rate-limit check ahead of auth, matching the existing v1 handler pattern, and pass those headers into auth failures.
  - confidence: 0.96
- **[P1] Filter exports through public skill visibility:** `convex/skills.ts:10411-10413`
  - body: The export query reads raw `skillSearchDigest` rows from `by_active_created` and then fetches full version blobs for every returned digest. This skips the public list/detail filters for moderation state, suspicious rows, blocked malware, and active owners, so the archive can expose content that normal public APIs would hide.
  - confidence: 0.94
- **[P1] Page mirror exports by update time:** `convex/skills.ts:10408-10413`
  - body: The endpoint is meant to support hourly mirror sync, but this range scan uses `createdAt`; a skill created last week and edited or republished during the requested hour will not be exported. Use an updated/latest-version cursor or make the endpoint explicitly snapshot-only instead of incremental sync.
  - confidence: 0.9
- **[P1] Move the export route out of the skill slug path:** `convex/http.ts:77-79`
  - body: Adding an exact `/api/v1/skills/export` route in front of the existing `/api/v1/skills/` prefix route takes over the skill-detail URL for slug `export`, and `export` is not reserved by the current skill slug validator. Put the bulk export under a reserved admin namespace or add a compatibility-safe reservation/migration path.
  - confidence: 0.87
- **[P1] Add aggregate limits before reading all export blobs:** `convex/httpApiV1/skillsV1.ts:1740-1741`
  - body: This reads every selected blob into memory and then builds one ZIP, while a page can include up to 1000 skills and current publish limits allow large bundles per skill. Add total byte/file limits or chunking before `arrayBuffer()` so a single admin request cannot exhaust the action runtime.
  - confidence: 0.88
