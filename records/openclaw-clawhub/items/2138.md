## Summary

Keep this PR open for maintainer review: the mirror-sync goal is useful, but current main does not already provide this endpoint and the proposed implementation has blocking proof, security, sync-correctness, and capacity issues.

## What This Changes

The PR adds an admin-authenticated `GET /api/v1/skills/export` endpoint that pages skills by date and returns a merged ZIP with metadata, route/schema wiring, ZIP path validation, and a new export rate-limit tier.

## Best Possible Solution

Ship a bounded bulk export only after maintainers define the public-versus-privileged contract, then enforce visibility, updated-time pagination, auth throttling, aggregate size limits, docs, and regression tests together.

## Reproduction Assessment

Yes for the review blockers: source inspection of the PR patch shows auth-before-rate-limit ordering, raw digest export without public visibility filtering, createdAt-only pagination, and eager blob materialization. I did not execute the branch in this read-only review.

## Solution Assessment

No, not as proposed. The endpoint direction may be useful, but the maintainable path needs a maintainer-approved export contract and focused fixes before this becomes safe to merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Apply export rate limiting before auth returns:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: `requireExportAuth` returns 401/403 before `applyRateLimit` runs, so missing or invalid bearer tokens never consume the documented IP fallback bucket. Run the export rate limit first and pass its headers into auth failures so failed auth traffic is still throttled.
  - confidence: 0.9
- **[P1] Filter export rows through the public visibility policy:** `convex/skills.ts:10411-10420`
  - body: The export page is built directly from `skillSearchDigest` rows constrained only by `softDeletedAt`, then the handler exports each latest version's files. For a mirror API, this can include hidden, removed, or malware-blocked skills that normal public surfaces suppress; use the existing public visibility path or make a privileged staff export an explicit maintainer-approved contract.
  - confidence: 0.9
- **[P1] Page incremental exports by update time:** `convex/skills.ts:10413`
  - body: The date window uses `by_active_created`, so an hourly mirror will miss existing skills that publish a new version or change metadata after their original creation window. The requested incremental sync needs updated-time or version-change semantics, with cursor tests covering republished existing skills.
  - confidence: 0.87
- **[P1] Bound exports before fetching every blob:** `convex/httpApiV1/skillsV1.ts:1684-1693`
  - body: The handler queues all files for up to 1000 skills and fetches them into memory before zipping, but published files can be large. Add an aggregate byte/file-count guard or much smaller streaming/page limits before reading blobs so a valid request cannot exhaust backend resources.
  - confidence: 0.82
