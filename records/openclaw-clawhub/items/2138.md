## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve public moderation filters in export:** `convex/skills.ts:10413`
  - body: The export page is pulled from `skillSearchDigest.by_active_created` with only `softDeletedAt` constrained, then the handler reads latest version blobs directly. That bypasses the current public/download gates for hidden, removed, pending-scan, suspicious-filtered, and malware-blocked skills, so a mirror export can leak content normal clients cannot list or download.
  - confidence: 0.94
- **[P1] Use an update and tombstone sync cursor:** `convex/skills.ts:10408-10413`
  - body: Paging by `createdAt` cannot keep a mirror correct after an existing skill is republished, renamed, retagged, moderated, removed, or otherwise updated outside the original creation window. The export contract needs update-based pagination plus deletion/moderation tombstones, or mirrors will silently serve stale data.
  - confidence: 0.9
- **[P2] Rate-limit failed export auth:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: The handler authenticates before applying the export rate limit, so missing or invalid Bearer tokens return without consuming an IP bucket. That conflicts with the documented missing/invalid-token IP fallback behavior and leaves the auth lookup path unthrottled for this new endpoint.
  - confidence: 0.88
- **[P1] Cap export bytes and file count before buffering:** `convex/httpApiV1/skillsV1.ts:1693-1740`
  - body: The endpoint collects all blob tasks, reads each blob into memory, and then builds a ZIP with no aggregate byte, file-count, or response-size limit. An authorized request for 1000 large or many-file skills can exceed Convex action memory/response limits and turn this endpoint into an availability hazard.
  - confidence: 0.86
- **[P2] Avoid occupying the skill slug route namespace:** `packages/schema/src/routes.ts:30`
  - body: `/api/v1/skills/{slug}` is a documented public route and `export` is not reserved by the current slug validator. Adding `/api/v1/skills/export` as a fixed route can shadow any existing or future skill with that slug; use a reserved namespace such as `/api/v1/skills/-/export` or add an explicit reservation/migration with route-collision tests.
  - confidence: 0.84
