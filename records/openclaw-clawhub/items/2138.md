## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep exports inside the public visibility boundary:** `convex/skills.ts:10411-10420`
  - body: The export page reads `skillSearchDigest` by only `softDeletedAt` and `createdAt`, then fetches raw latest-version blobs. That bypasses the `isPublicSkillDoc`/non-suspicious/moderation checks used by public APIs, so an admin export can include hidden, removed, suspicious, or malware-blocked skill files. Filter through the same public visibility rules before reading blobs.
  - confidence: 0.93
- **[P1] Move the endpoint out of the skill-slug namespace:** `convex/http.ts:76-80`
  - body: `GET /api/v1/skills/{slug}` is a documented detail route, so `/api/v1/skills/export` is already a valid skill slug path. Registering the exact export route before the skills prefix makes a skill with slug `export` unreachable through the API; use a reserved namespace such as `/api/v1/exports/skills` or `/api/v1/skills/-/export`.
  - confidence: 0.9
- **[P2] Consume a rate-limit bucket before rejecting export auth:** `convex/httpApiV1/skillsV1.ts:1571-1575`
  - body: `requireExportAuth` runs before `applyRateLimit`, so missing, invalid, and non-admin tokens return 401/403 without consuming the documented IP/user export limit. Put the request through rate limiting first, or explicitly consume a bucket for failed auth attempts, so auth probing and valid non-admin tokens cannot bypass export throttling.
  - confidence: 0.88
- **[P2] Page incremental exports by update time:** `convex/skills.ts:10408-10413`
  - body: The PR says mirror clients will do hourly incremental syncs, but the range cursor is built on `by_active_created`. Republished or edited skills keep their old `createdAt`, so a mirror that has already passed that creation window will never receive the updated files.
  - confidence: 0.86
- **[P2] Enforce an aggregate export size budget:** `convex/httpApiV1/skillsV1.ts:1693-1740`
  - body: The handler accepts up to 1000 skills, fetches all blobs, converts every `Blob` to an `ArrayBuffer`, and then builds one ZIP in memory with no byte budget. A few large skills can exhaust Convex action memory or make the endpoint unavailable; cap total uncompressed/compressed bytes and fail a page before buffering everything.
  - confidence: 0.84
- **[P2] Reserve metadata filenames before adding them to the ZIP:** `convex/httpApiV1/skillsV1.ts:1757-1759`
  - body: A skill that already contains `_export_skill_meta.json` will produce the same `slug/_export_skill_meta.json` path as the injected metadata file, causing `buildMergedExportZip` to throw and fail the whole export. Reject or rename reserved per-skill metadata paths before queuing blob reads.
  - confidence: 0.81
