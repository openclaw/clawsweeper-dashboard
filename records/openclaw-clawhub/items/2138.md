## What This Changes

Adds an admin-only `GET /api/v1/skills/export` endpoint that returns paginated merged ZIP exports of skill files and metadata for mirror synchronization.

## Best Possible Solution

Define the mirror/export contract first, then implement a narrower API with public/download visibility parity, update-and-tombstone sync semantics, a reserved route namespace, failed-auth throttling, aggregate caps, focused tests, and real endpoint proof.

## Reproduction Assessment

Not applicable as a user bug. For the PR blockers, yes: source inspection of the patch and current main shows the export query, auth/rate-limit order, route registration, and ZIP-building paths that create the issues.

## Solution Assessment

No. The safer path is to settle the export policy and then reuse existing public/download visibility gates with update/tombstone pagination and operational caps instead of landing this bulk endpoint as written.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve moderation gates in the export query:** `convex/skills.ts:10409`
  - body: The export page reads `skillSearchDigest.by_active_created` and then zips latest-version blobs without checking the same moderation/download gates that currently block pending, malware, removed, and moderator-hidden skills. An admin token would be able to mirror content that public listing and download paths intentionally hide.
  - confidence: 0.94
- **[P1] Use update-based sync with tombstones:** `convex/skills.ts:10404-10409`
  - body: The mirror API is advertised for hourly incremental sync, but this query windows on `createdAt`; republishing, moderation changes, deletes, renames, or version updates on older skills will never appear in a later date range, leaving mirrors stale or serving content that should be gone.
  - confidence: 0.9
- **[P1] Avoid occupying a valid skill slug route:** `packages/schema/src/routes.ts:30`
  - body: Adding exact `/api/v1/skills/export` before the `/api/v1/skills/{slug}` prefix makes a skill whose slug is `export` unreachable through the documented detail URL, while the slug validator does not reserve `export` today. Put the bulk route under a reserved namespace or reserve/migrate the slug first.
  - confidence: 0.88
- **[P2] Throttle invalid export auth before token lookup:** `convex/httpApiV1/skillsV1.ts:1567-1571`
  - body: The handler calls `requireExportAuth` before `applyRateLimit`, so invalid bearer tokens still hash and query the token table without consuming an IP bucket. That conflicts with the documented missing/invalid-token fallback and exposes a privileged route to auth lookup pressure.
  - confidence: 0.86
- **[P1] Cap export size before reading blobs into memory:** `convex/httpApiV1/skillsV1.ts:1576-1579`
  - body: A single allowed request can read up to 1000 skills worth of blobs, materialize each file, then build a copied ZIP without a byte, file-count, or response-size cap. Large skills can exhaust Convex action memory or time out the endpoint, so enforce aggregate limits before fetching and building the archive.
  - confidence: 0.84
