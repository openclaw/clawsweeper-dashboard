The branch adds an admin-only `GET /api/v1/skills/export` API that pages skill digests by date and returns a merged ZIP with manifest metadata, auth, rate-limit, route, and schema wiring.

## Best Possible Solution

Define an explicit mirror export contract first, then implement it with the chosen visibility boundary, updated-at pagination, failed-auth throttling, aggregate size limits, focused tests, docs, and redacted runtime proof.

## Reproduction Assessment

## Solution Assessment

No. A bulk mirror API may be useful, but the maintainable path is to decide the public-vs-privileged export contract first and then implement that boundary with updated-at pagination, throttling, size limits, and proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Filter export rows through the public visibility policy:** `convex/skills.ts:10411-10420`
  - body: `listByDateRange` returns raw `skillSearchDigest` rows from the active-created index with only `softDeletedAt` constrained. Hidden or moderation-blocked rows can still have `softDeletedAt` unset, while current public responses go through `isPublicSkillDoc`/`toPublicSkill`, so this export can leak content that normal public skill APIs suppress.
  - confidence: 0.93
- **[P1] Page mirror exports by updated activity:** `convex/skills.ts:10413-10418`
  - body: The stated mirror sync use case is hourly incremental export, but this query windows on `createdAt`. An existing skill published months ago and updated inside the requested hour keeps its old `createdAt`, so the mirror will miss updates unless the export contract pages by `updatedAt` or another change feed.
  - confidence: 0.9
- **[P1] Throttle failed export auth attempts:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: The handler returns from `requireExportAuth` before calling `applyRateLimit`, so missing or invalid bearer tokens never consume the export IP bucket. Move a safe rate-limit check ahead of auth or otherwise charge failed auth attempts so this new expensive endpoint cannot be probed for free.
  - confidence: 0.91
- **[P1] Bound aggregate export size before reading blobs:** `convex/httpApiV1/skillsV1.ts:1693`
  - body: The code fetches every blob for the page, converts each to an `ArrayBuffer`, stores all buffers in `zipEntries`, and then creates the merged ZIP. A small count limit is not a byte limit; large skill files can still create excessive memory pressure, especially with a 600/min admin bucket.
  - confidence: 0.88
