Full review comments:

- **[P1] Keep the export route from shadowing skill slugs:** `convex/http.ts:77-79`
  - body: Registering exact `/api/v1/skills/export` means an existing or future skill with slug `export` can no longer be fetched through the documented `/api/v1/skills/{slug}` route; current slug rules do not reserve that word. Put the admin export under a reserved namespace such as `/api/v1/skills/-/export` or otherwise preserve slug lookup compatibility.
  - confidence: 0.93
- **[P1] Filter exports through the public skill boundary:** `convex/skills.ts:10413-10420`
  - body: This query reads raw `skillSearchDigest` rows via `by_active_created` and returns them without the existing public mapping/moderation checks. The handler then fetches latest version blobs, so hidden or `blocked.malware` skills can be packaged for export despite the documented mirror boundary.
  - confidence: 0.96
- **[P2] Use updated cursors for incremental mirror sync:** `convex/skills.ts:10408-10413`
  - body: The endpoint's advertised hourly sync cannot see republished or otherwise changed skills because pagination is over `createdAt`. Existing skills whose `updatedAt` changes outside their original creation window will be missed by incremental mirror pulls.
  - confidence: 0.9
- **[P2] Throttle failed export auth before token lookup:** `convex/httpApiV1/skillsV1.ts:1571-1575`
  - body: Missing and invalid Bearer tokens return from `requireExportAuth` before `applyRateLimit`, so export auth probing and token hash lookups do not consume the IP bucket that the API docs promise for missing or invalid tokens.
  - confidence: 0.9
- **[P2] Bound archive bytes before buffering blobs and ZIPs:** `convex/httpApiV1/skillsV1.ts:1693-1741`
  - body: A request can select up to 1000 skills, fetch every blob, materialize every file as a `Uint8Array`, and then create a second full ZIP buffer without an aggregate byte cap. Large skill files can exhaust Convex action memory or stall the endpoint.
  - confidence: 0.87
