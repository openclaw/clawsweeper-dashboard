The PR adds an admin-only `GET /api/v1/skills/export` endpoint that returns paginated merged ZIP archives of skill files and metadata for mirror synchronization, plus auth, rate-limit, route, schema-route, and ZIP helper changes.

## Best Possible Solution

If maintainers want mirror export, land a scoped contract with explicit mirror entitlement, public/moderation visibility rules, audit/rate/size limits, an update/tombstone cursor model, a non-colliding route namespace, focused tests, docs/specs, and redacted live proof.

- **[P1] Filter export through public visibility gates:** `convex/skills.ts:10411-10420`
  - body: `listByDateRange` pages raw `skillSearchDigest` rows from `by_active_created` and the handler zips their latest files without applying the current public/download visibility gates. That can export hidden, pending, removed, suspicious, or malware-blocked artifacts to a mirror even though the docs require third-party mirrors to stay within the public API surface.
  - confidence: 0.9
- **[P1] Use an update-aware cursor for mirror sync:** `convex/skills.ts:10407-10418`
  - body: The endpoint is described as hourly incremental mirror sync, but this query ranges over `createdAt`. Republishes, renames, moderation changes, deletes, and restored old skills will not be returned by future created-date windows, leaving mirrors silently stale.
  - confidence: 0.88
- **[P2] Throttle invalid tokens before auth lookup:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: The handler calls `requireExportAuth` before `applyRateLimit`, so malformed or invalid bearer tokens still get hashed and looked up before any export/IP limiter is charged. Put a cheap IP/anonymous throttle before token lookup or make failed auth consume the export limiter.
  - confidence: 0.83
- **[P1] Add aggregate export caps before buffering blobs:** `convex/httpApiV1/skillsV1.ts:1693-1780`
  - body: This path fetches blobs for up to 1000 skills, converts them all to memory buffers, and then builds a ZIP copy without total byte, file-count, or response-size limits. One authorized request can exhaust the Convex action memory budget on large skills.
  - confidence: 0.86
- **[P1] Avoid stealing the export skill detail route:** `convex/http.ts:73-77`
  - body: Current API docs expose `/api/v1/skills/{slug}` and current slug validation does not reserve `export`, so this exact route makes `/api/v1/skills/export` a privileged bulk export instead of the detail response for a valid skill slug. Move it to a non-colliding namespace or add an explicit reservation/migration with compatibility tests.
