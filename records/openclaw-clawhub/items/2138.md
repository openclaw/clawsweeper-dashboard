## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Filter exports through public visibility gates:** `convex/skills.ts:10411-10420`
  - body: The export query pages `skillSearchDigest.by_active_created` and then archives each latest version without applying the public visibility rules used elsewhere. That can include hidden, removed, malware-blocked, or suspicious skills in a mirror export, which conflicts with the documented public API boundary; export should go through `isPublicSkillDoc`/non-suspicious or an explicitly approved export policy before reading blobs.
  - confidence: 0.93
- **[P2] Rate-limit export attempts before auth lookup:** `convex/httpApiV1/skillsV1.ts:1571-1575`
  - body: `requireExportAuth` runs before `applyRateLimit`, so missing or invalid bearer tokens return `401` without consuming the export IP bucket. The docs and current limiter semantics make invalid auth fall back to IP throttling; keep that ordering here so token-guessing or bad-client traffic cannot bypass the new export limiter.
  - confidence: 0.89
- **[P2] Cap export bytes before reading blobs:** `convex/httpApiV1/skillsV1.ts:1740-1741`
  - body: The handler can select up to 1000 skills, fetch all their storage blobs, convert each to an `ArrayBuffer`, and then build the ZIP without a cumulative byte limit. A large but valid export can exceed action memory, so enforce a total byte/file budget before reading or adding blobs to the archive.
  - confidence: 0.86
