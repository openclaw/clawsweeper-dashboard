## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Filter export rows through the public visibility policy:** `convex/skills.ts:10411-10413`
  - body: `listByDateRange` pages `skillSearchDigest.by_active_created` with only the soft-delete prefix, then the handler exports latest-version files directly. That bypasses the existing public `toPublicSkill`/`isPublicSkillDoc` path and nonsuspicious indexes, so hidden/moderated or suspicious skills can be included in mirror archives. Constrain the export to the intended visibility set, or document and gate an explicit staff-only full-content export mode.
  - confidence: 0.92
- **[P1] Bound archive size before reading every blob:** `convex/httpApiV1/skillsV1.ts:1740-1741`
  - body: The route accepts up to 1000 skills, but each published skill bundle can be much larger than the average case and the handler reads every blob into memory before calling `zipSync`. A date range containing large but valid skills can force the action to allocate far beyond Convex/runtime limits. Add a total byte/read-work cap or stream/shard the export before merging.
  - confidence: 0.9
- **[P2] Apply export rate limiting before token auth:** `convex/httpApiV1/skillsV1.ts:1571-1574`
  - body: This handler calls `requireExportAuth` before `applyRateLimit`, unlike the existing HTTP handlers that rate-limit first. Missing or invalid bearer tokens therefore never consume the new export rate-limit bucket while still forcing token parsing/hashing and auth lookups, weakening brute-force and availability protection. Move the rate limit ahead of auth and include its headers on auth failures.
  - confidence: 0.88
- **[P2] Avoid per-skill metadata filename collisions:** `convex/httpApiV1/skillsV1.ts:1757-1759`
  - body: A skill can publish a text file named `_export_skill_meta.json`; this loop then adds another entry at the same `${slug}/_export_skill_meta.json` path, and `buildMergedExportZip` throws a duplicate-path error. One valid skill with that path would make the whole export page fail with a 500. Use a reserved root manifest structure, skip or rename the generated file, or reject the reserved path at publish time with a migration plan.
  - confidence: 0.86
- **[P2] Use update time for incremental mirror windows:** `convex/skills.ts:10408-10413`
  - body: The endpoint is described as supporting hourly mirror sync, but the date range is applied to `createdAt` via `by_active_created`. Existing skills that publish a new version after their original creation date will not appear in later incremental windows, so a mirror can miss updates after its initial crawl. Use an updated/version timestamp contract and tests for an existing-skill update.
  - confidence: 0.82
