## Summary

If ClawHub should support mirrors, define a maintainer-approved export contract under a non-slug namespace with public/download visibility enforcement, update/tombstone pagination, aggregate size caps, throttled auth failures, auditability, docs, tests, and real proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Reuse public/download visibility for export rows:** `convex/skills.ts:10411-10413`
  - body: `listByDateRange` pages `skillSearchDigest.by_active_created` with only the soft-delete prefix, and the handler then reads internal version blobs. That bypasses the current public/download checks that hide non-active, pending, malware-blocked, removed, or otherwise unavailable skills, so the export can leak files that normal users cannot fetch.
  - confidence: 0.95
- **[P1] Move export out of the skill-slug namespace:** `packages/schema/src/routes.ts:30`
  - body: `/api/v1/skills/export` sits under the documented `/api/v1/skills/{slug}` detail namespace, so an existing or future skill whose slug is `export` would lose its detail route to this admin export handler. Use a non-slug namespace such as `/api/v1/exports/skills`, or reserve and migrate the slug deliberately with tests.
  - confidence: 0.9
- **[P1] Use update or tombstone pagination for mirror sync:** `convex/skills.ts:10408-10413`
  - body: Paging by `createdAt` only works for initial bootstrap. Hourly mirror sync will miss existing skills that are republished, renamed, moderated, removed, restored, or otherwise updated after their creation window, so the export contract needs update-based pagination and explicit deletion/moderation tombstones, or it must be documented as bootstrap-only.
  - confidence: 0.92
- **[P2] Cap aggregate export work before loading blobs:** `convex/httpApiV1/skillsV1.ts:1693`
  - body: The row limit does not cap total files or bytes: a page can enqueue every file for up to 1000 skills, read every blob into memory, then build another ZIP buffer. Add aggregate byte/file limits and fail predictably before storage reads or ZIP creation so one authorized request cannot exhaust the action.
  - confidence: 0.88
- **[P2] Throttle failed export auth attempts:** `convex/httpApiV1/skillsV1.ts:1571-1575`
  - body: The handler returns from `requireExportAuth` before `applyRateLimit`, so missing, invalid, or non-admin bearer attempts never consume the new export bucket. Apply a cheap IP/token-failure limiter before auth, or otherwise spend a failed-auth bucket, to keep this privileged route from becoming an unauthenticated CPU/token-lookup flood point.
  - confidence: 0.86
