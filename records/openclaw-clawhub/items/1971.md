## Summary

Keep open: current main still supports ZIP downloads, but the regular skill detail page no longer exposes an obvious top-level download button, so the reported manual-copy and author-update workflows remain a valid UX regression.

## What This Changes

Restore an obvious skill ZIP download affordance so users can manually copy skills and authors can download their latest published version before updating.

## Best Possible Solution

Expose a visible latest-version ZIP download action for normal skills, while keeping Nix plugin bundles on their bundle-specific install path and retaining per-version archive links in the Versions tab.

## Reproduction Assessment

Yes. Code inspection on current main shows the detail hero has OpenClaw CLI/prompt copy controls, while the only regular-skill ZIP affordance is a `Zip` link inside the Versions tab and the backend download route still exists.

## Solution Assessment

Yes. A narrow frontend restoration that reuses the existing `/api/v1/download` endpoint is the most maintainable fix; no backend or archive-generation change is needed.
