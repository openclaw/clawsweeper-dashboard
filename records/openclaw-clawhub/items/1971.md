## Summary

Keep open: current main still lacks the clear latest ZIP download action reported by the user, while the backend endpoint, docs, and per-version ZIP links show the download path still exists. This is a narrow UI regression candidate, not a cleanup close.

## What This Changes

Restore a clear manual download button for latest skill ZIPs on ordinary skill detail pages.

## Best Possible Solution

Add a non-Nix latest `Download zip` action on skill detail pages backed by `/api/v1/download?slug=<slug>`, with regression coverage and no ZIP action for Nix plugin bundles.

## Solution Assessment

Yes. Restoring a latest-version `Download zip` affordance for non-Nix skill pages using the existing endpoint is the narrowest maintainable fix, while preserving the Nix bundle exception from PR #1.
