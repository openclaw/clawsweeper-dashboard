## Summary

Keep open: current main still has backend ZIP download support and per-version ZIP links, but ordinary skill detail pages do not expose a clear latest-version manual download action; ClawHub cleanup policy also keeps issues open rather than auto-closing them.

## What This Changes

The issue asks to restore an obvious manual download button for skills so users can copy ZIPs into offline workflows and authors can download their latest published version before updating.

## Best Possible Solution

Restore a visible latest-version `Download zip` affordance on normal skill detail pages using the existing `/api/v1/download?slug=<slug>` endpoint, while keeping Nix plugin bundles on their bundle-specific path.

## Solution Assessment
