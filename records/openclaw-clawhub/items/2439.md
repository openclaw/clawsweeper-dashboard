## Summary

Current main still has only the publish toast, so this focused and visually proven feature remains useful; however, the branch’s slug-wide session flash can show the success dialog for a different owner’s same-slug skill and should be repaired before merge.

## What This Changes

Adds a post-publish success dialog with Discord/X sharing, copy feedback, session-storage handoff, clean canonical URLs, legacy query handling, responsive styling, and focused tests.

## Best Possible Solution

Keep the clean-URL, one-time success dialog but carry or translate the exact intended owner/slug identity through canonical redirects, without a global slug-only marker.

## Reproduction Assessment

Yes for the introduced defect: publish `owner-a/weather`, then visit `owner-b/weather` in the same browser session before the flash is consumed; the slug-only key makes the second page show the dialog.

## Solution Assessment

No, not exactly as implemented: the dialog and clean-URL handoff are appropriate, but the fallback must preserve exact skill identity rather than treating equal slugs across owners as interchangeable.
