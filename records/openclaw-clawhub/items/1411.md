## Summary

Keep this PR open: current main still renders skill summaries read-only, but the latest PR head is not safe to merge because the new public summary-edit path leaves authorization, moderation, and search/catalog state inconsistent.

## What This Changes

The PR adds a public Convex `updateSummary` mutation, skill-detail state wiring, inline summary edit controls, and CSS for editing a skill summary from the detail page.

## Best Possible Solution

Land a rebased implementation that treats summary editing as a validated metadata update with aligned owner/publisher/moderator authorization, moderation/security recomputation, digest/API/search sync, length limits, and clear publish-override semantics.

## Reproduction Assessment

Yes for the blocking review findings from source inspection: publisher members get the edit UI through `canManage` but are rejected by `updateSummary`, and the mutation updates only the skill document while moderation and digest state remain unchanged. I did not execute the branch because this was a read-only review.

## Solution Assessment

No. The latest patch fixes the earlier invalid paragraph nesting, but the remaining backend and data-consistency gaps mean this is not yet the narrowest maintainable way to add editable summaries.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Recompute moderation when changing summaries:** `convex/skills.ts:4813-4818`
  - body: The publish path derives moderation flags and suspicious state from the summary that gets persisted, but this new mutation patches only `summary` and `updatedAt`. Owners can therefore change public searchable text after the last scan while moderation flags and the public security summary still describe the old text.
  - confidence: 0.91
- **[P1] Authorize publisher-owned skill managers:** `convex/skills.ts:4808-4810`
  - body: The detail page makes `canManage` true for members of `skill.ownerPublisherId`, and the header shows the edit button from that flag. This mutation rejects those users unless they are also the direct `ownerUserId` or a moderator, so publisher-owned skill managers can open the editor but fail on save.
  - confidence: 0.94
- **[P2] Update the search digest with summary edits:** `convex/skills.ts:4813-4818`
  - body: Skill lists, catalog, and search hydrate from `skillSearchDigest.summary`. Updating only `skills.summary` means the detail query can show the new summary while list/search/API catalog rows continue serving the old one until some unrelated digest sync runs.
  - confidence: 0.88
- **[P2] Cap editable summaries on the server:** `convex/skills.ts:4800-4803`
  - body: This public mutation accepts any Convex string and writes it into skill metadata that feeds detail, list, and catalog payloads. Add a server-side maximum length and mirror it in the textarea so clients cannot store oversized summaries.
  - confidence: 0.78
