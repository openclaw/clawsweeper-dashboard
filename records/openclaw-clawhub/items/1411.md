## Summary

Keep open: current main still lacks summary editing, and this PR has blocking authorization, moderation, validation, and DOM-structure issues before it can merge.

## What This Changes

The branch adds a public Convex `updateSummary` mutation, detail-page edit state, inline skill-header summary editing controls, and scoped CSS.

## Best Possible Solution

Land a revised metadata-update design that aligns UI and backend authorization, normalizes and caps summaries consistently with publishing, defines publish-vs-manual override semantics, and keeps moderation/search/detail state consistent.

## Reproduction Assessment

Yes. The PR diff is enough to reproduce the defects: a publisher member can see the edit button but fail backend authorization, edit mode renders a form inside a paragraph, and a direct mutation call can store an unbounded summary without recomputing moderation state.

## Solution Assessment

No. Inline editing is a reasonable direction, but the current patch is not the safest implementation because it adds a public write path without matching existing authorization, validation, and moderation workflows.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep moderation state in sync with summary edits:** `convex/skills.ts:4814-4819`
  - body: The mutation changes public summary text but only patches `summary` and `updatedAt`. Publishing derives moderation flags and `isSuspicious` from the persisted summary, so edited summaries can leave safety state describing the old text until another publish or scan updates it.
  - confidence: 0.9
- **[P1] Authorize publisher-owned skill managers:** `convex/skills.ts:4808-4810`
  - body: The edit button is shown through `canManage`, which includes members of `skill.ownerPublisherId`, but `updateSummary` only allows the direct `ownerUserId` or moderators. Publisher members can open the editor and will always fail on save.
  - confidence: 0.93
- **[P2] Move summary edit form out of the paragraph:** `src/components/SkillHeader.tsx:211-214`
  - body: Edit mode renders a `<form>` and block children inside `<p className="section-subtitle">`. Browsers implicitly close paragraphs around flow content, which can cause hydration or layout issues; use a non-paragraph wrapper for both states.
  - confidence: 0.95
- **[P2] Cap editable summary length on the server:** `convex/skills.ts:4800`
  - body: The new public mutation accepts an unrestricted string and stores it on public skill metadata. Publish-time summaries are normalized to a bounded length, so this path should enforce a server-side limit and matching textarea `maxLength`.
  - confidence: 0.84
