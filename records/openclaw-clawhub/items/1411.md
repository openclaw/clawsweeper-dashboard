## Summary

Keep open. Current main does not implement owner-editable summaries, and the latest PR diff still has blocking backend consistency and security issues even though the earlier invalid HTML nesting concern appears to be fixed.

## What This Changes

The PR adds a public Convex mutation plus skill-detail/header UI and CSS for editing a skill summary inline without republishing.

## Best Possible Solution

Land a revised implementation that treats summary edits as validated metadata updates with publisher-aware authorization, normalization, search digest sync, moderation recomputation or explicit review state, and clear behavior on future publishes.

## Reproduction Assessment

Yes. Static review gives a high-confidence reproduction path: publisher-owned skill managers satisfy current canManage but the PR backend rejects them, and accepted edits patch only the skills row while moderation and digest-backed surfaces remain stale.

## Solution Assessment

No. The latest diff fixes the invalid HTML wrapper, but directly patching raw summary text is not the best maintainable path because it bypasses established publisher authorization, publish-time normalization, search digest sync, and moderation derivation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Authorize publisher-owned managers consistently:** `convex/skills.ts:4808-4811`
  - body: The edit UI is exposed through canManage, which current main sets for ownerPublisherId members, but this mutation only allows the direct ownerUserId or a moderator. Publisher-owned skill managers can open the editor and then fail on save, so use the publisher-aware ownership helper or narrow the UI gate.
  - confidence: 0.93
- **[P1] Keep moderation state aligned with summary edits:** `convex/skills.ts:4814-4818`
  - body: The publish path derives moderation flags from the summary that gets persisted, but this mutation changes public summary text without updating moderationFlags, moderation summary fields, or isSuspicious. That lets the visible/searchable description diverge from the recorded safety state until some later publish or scan repairs it.
  - confidence: 0.9
- **[P2] Sync the search digest after summary writes:** `convex/skills.ts:4814-4818`
  - body: Catalog and search surfaces read summaries from skillSearchDigest, while this mutation only patches the skills row. After a successful edit, detail pages can show the new summary while digest-backed search, catalog, and API surfaces keep returning or scoring the old one.
  - confidence: 0.88
- **[P2] Validate and cap editable summaries:** `convex/skills.ts:4800-4803`
  - body: summary: v.string() accepts raw client text up to Convex document limits, unlike publish-time summaries that are normalized and capped. Add server-side normalization and a maximum length, with matching textarea constraints, before exposing this as a public write path.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The PR adds a public metadata write path that can desynchronize safety state and accepts unbounded public text.

Concerns:

- **[medium] Edited summaries bypass moderation recomputation:** `convex/skills.ts:4814`
  - body: updateSummary changes public summary text but does not recompute moderation flags, isSuspicious, moderation summary fields, or related review state even though publish-time code derives those fields from persisted summary text.
  - confidence: 0.9
