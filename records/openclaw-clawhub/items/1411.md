## Summary

Keep open: current main does not implement editable skill summaries, and the PR head still has concrete authorization, moderation, digest-sync, and validation blockers. The latest head does address the earlier invalid form wrapper comment by using a div instead of a paragraph.

## What This Changes

The PR adds a public Convex summary update mutation, skill-detail state wiring, inline SkillHeader edit controls, and CSS so a skill summary can be edited from the detail page.

## Best Possible Solution

Land a revised editable-summary flow that authorizes owner, publisher, and moderator actors consistently, normalizes and caps text, and keeps detail, digest/search/API, and moderation state in sync.

## Reproduction Assessment

Yes. Source inspection of PR head gives a high-confidence path: publisher managers satisfy the UI manage gate, but updateSummary only allows ownerUserId or moderators, and successful edits patch public summary text without moderation or digest sync.

## Solution Assessment

No. The current patch is not the best way to solve this because it writes skills.summary directly; the safer path is a validated, publisher-aware metadata update that also updates moderation and catalog/search projections.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Authorize publisher-owned skill managers:** `convex/skills.ts:4808-4809`
  - body: The edit button is shown through canManage, which includes members of skill.ownerPublisherId, but this mutation only permits the direct ownerUserId or moderators. Publisher managers can open the editor and will always fail on save, so align the backend authorization with the same publisher-aware ownership model or narrow the UI gate.
  - confidence: 0.93
- **[P1] Keep moderation state in sync with edited summaries:** `convex/skills.ts:4813-4818`
  - body: The publish path derives moderation flags from the summary that is persisted, but this mutation changes summary and updatedAt only. Owners or moderators can change public searchable text after the latest scan while moderationFlags, isSuspicious, and moderation summary fields still describe the old text.
  - confidence: 0.9
- **[P2] Update the search digest when summaries change:** `convex/skills.ts:4818`
  - body: skillSearchDigest stores its own summary copy for catalog/search hydration, but this mutation patches only the skills row. After saving, detail reads can show the new summary while list, search, and API digest surfaces stay stale.
  - confidence: 0.86
- **[P2] Normalize and cap edited summaries:** `convex/skills.ts:4802`
  - body: The new public mutation accepts raw v.string input and writes it into public metadata, bypassing the existing normalized 160-character summary pipeline. Add server-side normalization and a matching textarea limit before saving.
  - confidence: 0.84
