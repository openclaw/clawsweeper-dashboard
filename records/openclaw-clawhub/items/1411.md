## Summary

Keep this contributor PR open: current main does not implement editable skill summaries, and the proposed patch still has blocking authorization, moderation, digest-sync, and validation defects.

## What This Changes

The PR adds a public Convex `skills.updateSummary` mutation plus skill-detail/header UI and CSS for inline editing of a skill summary.

## Best Possible Solution

Land a revised editable-summary flow that uses publisher-aware authorization, server-side normalization and length limits, and consistent updates to moderation plus search/catalog digest state.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: publisher members satisfy the current `canManage` UI gate, while the PR mutation only allows direct owners or moderators, and successful writes patch the public skill row without updating moderation or digest state.

## Solution Assessment

No. The proposed implementation writes public summary text directly; the safer approach is a validated metadata update path that reuses publisher-aware ownership checks and keeps moderation/search-derived state synchronized.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep moderation state in sync with summary edits:** `convex/skills.ts:4814-4818`
  - body: The publish path derives moderation flags and `isSuspicious` from the summary that gets persisted, but this mutation patches only `summary` and `updatedAt`. Owners could change public searchable text after review while moderation fields still describe the old text, so route edits through moderation recomputation or an explicit manual-override workflow.
  - confidence: 0.9
- **[P1] Authorize publisher-owned skill managers:** `convex/skills.ts:4808-4812`
  - body: The edit control is shown through `canManage`, which includes members of `skill.ownerPublisherId`, but `updateSummary` only permits the direct `ownerUserId` or moderators. Publisher managers can open the editor and then always fail on save, so align the backend with the publisher-aware ownership helper or narrow the UI gate.
  - confidence: 0.93
- **[P2] Sync the search digest when saving summaries:** `convex/skills.ts:4814-4818`
  - body: Catalog and search hydration read the copied `summary` field from `skillSearchDigest`, while this mutation updates only the `skills` document. After saving, detail views may show the new summary while list/search/API digest surfaces remain stale.
  - confidence: 0.86
- **[P2] Normalize and cap editable summaries:** `convex/skills.ts:4800-4803`
  - body: The new public mutation accepts raw `v.string()` and writes it into public metadata. A client can save malformed or near-document-limit text, unlike the existing summary pipeline that compacts and caps summaries, so add server-side normalization and a matching UI limit.
  - confidence: 0.84
