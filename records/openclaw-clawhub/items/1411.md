## Summary

Keep open. Current main still has read-only skill summaries, and this PR is not obsolete; the latest PR head fixes the earlier invalid paragraph/form nesting but still has authorization, moderation, and validation defects before it can merge.

## What This Changes

The PR adds a public `skills.updateSummary` mutation, skill detail/header inline editing state and controls, and CSS for editing a skill summary from the detail page.

## Best Possible Solution

Land a revised editable-summary workflow that defines override semantics, authorizes direct owners, publisher managers, and moderators consistently, validates and normalizes text, recomputes moderation/review state, and relies on existing digest triggers for catalog/search consistency.

## Reproduction Assessment

Yes. Static review gives a high-confidence path: a publisher-member manager sees edit controls through `canManage` but `updateSummary` rejects them, and accepted direct-owner/moderator edits patch public summary text without updating moderation state.

## Solution Assessment

No. Directly patching raw `summary` is not the best fix because current main already has publisher-aware authorization helpers, normalized publish-time summaries, and moderation derivation tied to public summary text.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Authorize publisher managers consistently:** `convex/skills.ts:4805-4807`
  - body: The edit button is shown through `canManage`, which includes members of `skill.ownerPublisherId`, but this mutation only allows the direct `ownerUserId` or a moderator. Publisher-owned skill managers can open the editor and will always fail on save, so align the mutation with the publisher-aware ownership helper or narrow the UI gate.
  - confidence: 0.93
- **[P1] Keep moderation state in sync with summary edits:** `convex/skills.ts:4811-4815`
  - body: This patches public summary text without recomputing moderation flags, `isSuspicious`, moderation summary fields, or review state derived during publish. Because skill updates trigger digest sync, the new text can become searchable while the safety state still describes the old summary.
  - confidence: 0.9
- **[P2] Validate and normalize edited summaries:** `convex/skills.ts:4799`
  - body: `summary: v.string()` accepts raw client text up to Convex document limits, while the publish path normalizes summaries to a compact 160-character value. Add server-side normalization/length enforcement and a matching textarea limit so malformed clients cannot bloat public metadata responses.
  - confidence: 0.86
