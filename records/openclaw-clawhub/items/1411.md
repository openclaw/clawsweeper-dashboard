## Summary

Keep open: current main does not implement editable skill summaries, and the proposed patch has blocking correctness and security issues around authorization, moderation/search consistency, HTML structure, and input validation.

## What This Changes

This PR adds a Convex `updateSummary` mutation, detail-page state, inline header editing controls, and CSS so owners can edit a skill summary from the skill detail page.

## Best Possible Solution

Land a revised implementation that treats summary editing as a validated metadata update: align UI and backend authorization, normalize and cap the value, keep moderation and `skillSearchDigest` state consistent, define future-publish override semantics, and cover the mutation plus detail UI with focused tests.

## Reproduction Assessment

Yes. The defects are reproducible by static inspection of the PR diff against current main: publisher members can reach the UI gate but fail backend authorization, edit mode renders a form inside a paragraph, and the mutation changes public summary text without updating moderation or search digest state.

## Solution Assessment

No. The feature direction is useful, especially for the related summary-correction request, but this patch is not the best mergeable shape until the metadata write path follows the existing publisher, moderation, normalization, and digest patterns.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Sync moderation state when summaries change:** `convex/skills.ts:4814-4818`
  - body: The mutation patches `summary` and `updatedAt` only, but the publish path derives `moderationFlags`, `moderationSummary`, and `isSuspicious` from the persisted summary. Editing public text after review can leave safety state describing the old text until another publish or scan runs.
  - confidence: 0.91
- **[P2] Keep the catalog search digest in sync:** `convex/skills.ts:4814-4818`
  - body: Catalog and search results read summary from `skillSearchDigest`, but this mutation updates only the `skills` document. After a save, detail data and catalog/search surfaces can disagree, and summary text search can keep matching the old value.
  - confidence: 0.88
- **[P1] Authorize publisher-owned skill managers:** `convex/skills.ts:4808-4812`
  - body: The edit button is shown through `canManage`, which includes members of `skill.ownerPublisherId`, but the mutation only allows `ownerUserId` or moderators. Publisher members for org-owned skills can open the editor and then always fail on save.
  - confidence: 0.93
- **[P2] Move the edit form out of the paragraph:** `src/components/SkillHeader.tsx:211-214`
  - body: Edit mode renders a `<form>` and block children inside `<p className="section-subtitle">`. Browsers implicitly close paragraphs around flow content, which can cause hydration or layout issues; use a non-paragraph wrapper for both states.
  - confidence: 0.95
- **[P2] Cap editable summaries on the server:** `convex/skills.ts:4800-4803`
  - body: The public mutation accepts an unrestricted `v.string()` and writes it into public metadata. Publish-time summaries are normalized and bounded, so this path should enforce a server-side limit and matching textarea `maxLength`.
  - confidence: 0.82

## Security Review
