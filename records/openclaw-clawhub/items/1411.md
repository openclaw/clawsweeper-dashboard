## Summary

Keep open: current main still has read-only skill summaries, and this external feature PR remains blocked by source-reproducible authorization, moderation, validation, and real-behavior-proof gaps.

## What This Changes

The PR adds a public Convex `updateSummary` mutation, skill-detail/header state, inline edit controls, and CSS for editing a skill summary from the detail page.

## Best Possible Solution

Land a revised summary-editor design that defines override semantics, aligns UI and backend authorization, normalizes and caps input, and keeps moderation, audit, search, API, and detail-page state coherent.

## Reproduction Assessment

Yes for the PR review findings: source inspection shows publisher members can reach the editor through `canManage` while `updateSummary` rejects them, and edited public text bypasses the publish-time moderation derivation. I did not execute the branch in a browser or Convex deployment because this was a read-only review.

## Solution Assessment

No. The current patch fixes the earlier invalid paragraph nesting, but a maintainable solution still needs aligned authorization, server-side validation, explicit summary override semantics, and moderation/audit handling before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Authorize publisher-managed summaries consistently:** `convex/skills.ts:4808-4812`
  - body: `SkillDetailPage` passes `canManage` into the header, and current `canManage` includes members of `skill.ownerPublisherId`. This mutation only permits the direct `ownerUserId` or a moderator, so publisher members can open the editor and always hit the generic save failure.
  - confidence: 0.94
- **[P1] Keep moderation state in sync with edited summaries:** `convex/skills.ts:4814-4818`
  - body: The publish path derives `moderationFlags` and `isSuspicious` from the summary that gets persisted, but this mutation only patches `summary` and `updatedAt`. Owners can change public searchable text after scanning while moderation state still describes the old text, so route edits through the same moderation or manual-review flow.
  - confidence: 0.91
- **[P2] Cap editable summaries on the server:** `convex/skills.ts:4800-4803`
  - body: This public mutation accepts any Convex string and stores it in frequently read public metadata. Reuse the existing summary normalization/limit or add an explicit server-side maximum plus matching textarea `maxLength` so one edit cannot bloat list, detail, API, and digest payloads.
  - confidence: 0.82
- **[P3] Normalize blank summaries before saving:** `convex/skills.ts:4814-4818`
  - body: Saving an empty textarea writes an empty string, but the static display only falls back for nullish values, so clearing the field renders a blank subtitle instead of `No summary provided.` Trim input and store a missing value, or render blank strings through the same fallback.
  - confidence: 0.78
