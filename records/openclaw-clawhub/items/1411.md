## Summary

Keep open. Current main still lacks editable summaries, and this PR adds the feature but remains incorrect around publisher authorization, moderation/search consistency, and summary validation.

## What This Changes

The PR adds a public `skills.updateSummary` mutation, skill-detail summary edit state, inline header controls, and CSS for editing a skill summary from the detail page.

## Best Possible Solution

Land a revised editable-summary flow that uses publisher-aware authorization, normalizes and caps text, and keeps moderation plus catalog/search digest state consistent with the new public summary.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: publisher members satisfy current main `canManage`, while the PR mutation only allows direct owners or moderators, and accepted edits patch public summary text without updating moderation or digest state.

## Solution Assessment

No. The current patch is not the best way to solve this because it writes `skills.summary` directly instead of using the existing publisher-aware ownership helper and a normalized moderation/digest update path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Authorize publisher-owned skill managers:** `convex/skills.ts:4808-4812`
  - body: The edit button is shown through `canManage`, which includes members of `skill.ownerPublisherId`, but this mutation only permits the direct `ownerUserId` or moderators. Publisher managers can open the editor and then always fail on save, so align the backend with the publisher-aware ownership helper or narrow the UI gate.
  - confidence: 0.93
- **[P1] Keep moderation state in sync with edited summaries:** `convex/skills.ts:4814-4818`
  - body: The publish path derives moderation flags and `isSuspicious` from the summary that gets persisted, but this mutation changes `summary` and `updatedAt` only. Owners or moderators can change public searchable text after scan while moderation fields still describe the old text, so route edits through moderation/rescan or define explicit override semantics.
  - confidence: 0.9
- **[P2] Sync the catalog digest after summary edits:** `convex/skills.ts:4814-4818`
  - body: `skillSearchDigest` stores its own `summary` copy for catalog and search hydration, but this mutation patches only the `skills` row. After saving, detail reads may show the new summary while list/search/API digest surfaces stay stale; update the digest with the same normalized value.
  - confidence: 0.86
- **[P2] Normalize and cap editable summaries:** `convex/skills.ts:4800-4803`
  - body: `summary: v.string()` accepts raw client text up to Convex document limits, while the existing summary pipeline compacts text and caps it at 160 characters. Add server-side normalization/length enforcement and a matching textarea limit before writing public metadata.
  - confidence: 0.86
