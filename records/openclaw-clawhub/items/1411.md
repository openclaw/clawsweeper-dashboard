## Summary

Keep open. Current main still has no editable-summary path, and this contributor PR addresses a real user need but remains not merge-ready because the backend write path does not match existing ownership, moderation, and indexed-summary behavior.

## What This Changes

The PR adds a public `skills.updateSummary` mutation plus skill-detail state, inline header controls, and CSS for editing a skill summary from the detail page.

## Best Possible Solution

Land a revised editable-summary flow that treats manual edits as validated metadata overrides, with publisher-aware authorization, summary normalization, moderation/rescan behavior, digest sync, and focused tests.

## Reproduction Assessment

Yes. Static review of the latest PR diff gives a high-confidence path: publisher members satisfy current main `canManage` but fail the new mutation auth, and accepted owner edits patch only `skills.summary`/`updatedAt` while derived moderation and digest state remain unchanged.

## Solution Assessment

No. Directly patching `skills.summary` is not the best maintainable implementation because it bypasses the existing publisher authorization helper and the publish/rescan logic that derives public safety and indexed summary state.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Recompute moderation for edited summaries:** `convex/skills.ts:4814-4818`
  - body: The publish path derives moderation flags and scan-derived public state from the persisted summary, but this mutation changes `summary` and `updatedAt` only. An owner or moderator can therefore change public text after the latest version was scanned while `moderationFlags`, `isSuspicious`, and moderation summary fields still describe the old text.
  - confidence: 0.9
- **[P1] Authorize publisher managers consistently:** `convex/skills.ts:4808-4812`
  - body: The edit button is shown through `canManage`, which includes members of the owning publisher, but `updateSummary` only allows the direct `ownerUserId` or moderators. Publisher managers can open the editor and then always fail on save, so align the backend with publisher-aware ownership or narrow the UI gate.
  - confidence: 0.93
- **[P2] Sync the search digest after summary edits:** `convex/skills.ts:4814-4818`
  - body: Search, list, and lean API paths read summary from `skillSearchDigest`, but the new mutation only patches the `skills` row. After saving, the detail page can show the new summary while browse/search/API list surfaces continue serving the old digest value.
  - confidence: 0.86
- **[P2] Cap and normalize editable summaries:** `convex/skills.ts:4800-4803`
  - body: `summary: v.string()` accepts raw client text up to Convex document limits, unlike the publish summary pipeline that compacts and caps summary text. Add server-side normalization/length enforcement with a matching textarea limit before writing public metadata.
  - confidence: 0.86
