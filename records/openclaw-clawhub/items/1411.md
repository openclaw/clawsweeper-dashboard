## Summary

Keep open. Current main still has a read-only skill summary and no summary-edit mutation or UI symbols, while the PR diff introduces blocking functional and security issues around publisher authorization, moderation state, invalid HTML nesting, and unbounded public metadata input.

## What This Changes

The PR adds a Convex `updateSummary` mutation, detail-page state wiring, inline `SkillHeader` summary edit controls, and CSS for editing a skill summary from the detail page.

## Best Possible Solution

Land a revised summary-edit flow that treats the summary as validated public metadata: define override semantics against publish-derived summaries, authorize direct owners, publisher managers, and moderators consistently, normalize and cap the value server-side, keep moderation/search state coherent, and cover the Convex mutation plus detail UI with focused tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep moderation state in sync with edited summaries:** `convex/skills.ts:4814-4818`
  - body: The mutation changes public summary text but only patches `summary` and `updatedAt`. Publish-time moderation derives flags and `isSuspicious` from the summary being persisted, so edited summaries can leave moderation state describing the previous text until another publish or scan updates it.
  - confidence: 0.9
- **[P1] Authorize publisher-owned skill managers:** `convex/skills.ts:4808-4812`
  - body: The edit button is shown through `canManage`, which includes members of `skill.ownerPublisherId`, but `updateSummary` only permits the direct `ownerUserId` or moderators. Publisher members can open the editor and will always fail when saving publisher-owned skills.
  - confidence: 0.93
- **[P2] Move the summary edit form out of the paragraph:** `src/components/SkillHeader.tsx:211-214`
  - body: When edit mode is active, this renders a `<form>` and block children inside `<p className="section-subtitle">`. Browsers implicitly close paragraphs around flow content, which can cause hydration or layout issues; use a non-paragraph wrapper for both states.
  - confidence: 0.95
- **[P2] Cap editable summary length on the server:** `convex/skills.ts:4800-4803`
  - body: The public mutation accepts any Convex string and stores it on public skill metadata. Publish-time summaries are normalized to a bounded length, so this path should enforce a server-side limit and matching textarea `maxLength`.
  - confidence: 0.84
