## Summary

Keep open. Current main still renders skill summaries read-only, and the latest PR head fixes the earlier invalid form wrapper but still introduces backend defects around publisher authorization, moderation/search consistency, and unbounded public metadata writes.

## What This Changes

Adds a public Convex mutation plus detail-page/header UI and CSS so a skill summary can be edited inline without republishing.

## Best Possible Solution

Land a revised implementation that treats editable summaries as a validated metadata update: align owner/publisher/moderator authorization, normalize and cap the value, update search/catalog/API denormalizations, recompute or reset moderation state, audit the change, and define interaction with future publishes.

## Reproduction Assessment

Yes. Static review gives a high-confidence reproduction path: a publisher member sees owner tooling through `canManage` but the PR backend rejects them, and any accepted edit deterministically patches only `skills.summary`/`updatedAt` while moderation and search digest fields remain stale.

## Solution Assessment

No. Directly patching raw `summary` is not the best fix because current main already has publish-time normalization, moderation derivation, publisher-aware authorization helpers, and denormalized search digest patterns that this PR bypasses.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Authorize publisher managers consistently:** `convex/skills.ts:4808-4811`
  - body: The UI exposes summary editing through `canManage`, which includes members of `skill.ownerPublisherId`, but this mutation only accepts the direct `ownerUserId` or a moderator. Publisher-owned skill managers can open the editor and will always fail on save, so use the publisher-aware ownership helper or narrow the UI gate.
  - confidence: 0.93
- **[P1] Recompute moderation when summaries change:** `convex/skills.ts:4814-4818`
  - body: This writes new public summary text without updating moderation flags, `isSuspicious`, or the moderation summary fields that publish derives from the persisted summary. A user can make the public/searchable description diverge from the safety state until a later publish or scan happens.
  - confidence: 0.9
- **[P2] Sync the search digest after summary writes:** `convex/skills.ts:4818-4819`
  - body: Catalog and package search read summaries from `skillSearchDigest`, but this mutation only patches the `skills` row. After a successful edit, detail pages may show the new summary while catalog/search/API surfaces backed by the digest keep returning and scoring the old text.
  - confidence: 0.88
- **[P2] Validate and normalize edited summaries:** `convex/skills.ts:4800-4803`
  - body: `summary: v.string()` accepts any string up to Convex document limits and stores it directly. Existing publish-time summaries are normalized and capped, so this public write path should enforce the same server-side shape and add a matching textarea limit.
  - confidence: 0.84

## Security Review

Status: needs_attention
