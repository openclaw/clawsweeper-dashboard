## Summary

Keep open: current main does not implement editable skill summaries, and the PR still has blocking authorization, moderation, validation, conflict, and real-behavior-proof gaps.

## What This Changes

The PR adds a public Convex summary-update mutation, inline skill detail/header editing UI, and scoped CSS for editing a skill summary from the detail page.

## Best Possible Solution

Land a revised implementation that defines summary override semantics, aligns owner/publisher/moderator authorization, validates and normalizes input, and keeps public summary, search digest, and moderation state coherent.

## Reproduction Assessment

Yes for the PR defects: source inspection shows publisher members can reach the editor through canManage while updateSummary rejects them, and updateSummary changes public summary text without recomputing moderation state. I did not run the branch because this was a read-only review.

## Solution Assessment

No. The PR adds the expected UI entry point, but the current approach is not maintainable until authorization, validation, moderation, and summary-override policy are handled together.

  - body: The publish path derives moderation flags from the summary that gets persisted, but this mutation patches only summary and updatedAt.   - confidence: 0.94
- **[P2] Cap editable summaries on the server:** `convex/skills.ts:4800-4803`

  - confidence: 0.82

  - body: Saving a cleared textarea sends an empty string, but the read-only display only falls back for nullish values, leaving a blank subtitle. Trim input and store a missing value, or render blank strings through the same fallback.
  - confidence: 0.78
