## Summary

Keep open. Current main still blocks same-GitHub sign-in after user-initiated account deletion; it only improves the error surface. This is also an open ClawHub issue, and the cleanup policy for this repository keeps issues open unless maintainers explicitly decide otherwise.

## What This Changes

The reporter wants to sign up or sign in with GitHub again after deleting a previous ClawHub account instead of getting stuck in an OAuth refresh/sign-in loop.

## Best Possible Solution

Maintainers should choose and document the account deletion policy. If same-GitHub re-signup is allowed, implement a secure recreate or relink path that preserves abuse and ownership invariants; if deletion remains irreversible, make the OAuth result and support path explicit enough that affected users are not left in a silent loop.
