## Summary

Keep open. Current main still intentionally blocks same-GitHub sign-in after user-initiated deletion; the error surface is better than the original silent loop, but re-signup/relinking remains an unresolved product and security-policy decision.

## What This Changes

The issue asks for deleted-account users to be able to sign up or sign in again with the same GitHub account instead of getting stuck after OAuth.

## Best Possible Solution

Maintainers should choose and document the deletion policy, then either build a secure same-GitHub recreate/relink flow or keep deletion irreversible with an explicit support/error path that never looks like a loop.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: delete an account so the user has `deactivatedAt`, then complete GitHub OAuth for that identity; `handleDeletedUserSignIn` rejects it, and current tests assert that behavior.

## Solution Assessment

Unclear. Re-signup may be the right user outcome, but the current implementation intentionally blocks it, so the maintainable fix requires a deletion/relinking policy decision before code changes.
