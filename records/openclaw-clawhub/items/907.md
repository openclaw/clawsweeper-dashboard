## Summary

Keep open. Current main deliberately treats user-initiated deletion as irreversible, blocks same-GitHub OAuth after deletion, and only surfaces an error; resolving re-signup needs maintainer product and security policy, and ClawHub issue cleanup keeps issues open.

## What This Changes

Allow a user who deleted their ClawHub account to sign up or sign in again with the same GitHub identity instead of looping or refreshing after OAuth.

## Best Possible Solution

Maintainers should choose and document deletion semantics, then either build secure same-GitHub re-registration/relinking that preserves abuse protections and ownership invariants, or keep deletion irreversible with an explicit support/error path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: delete an account so deleteAccount sets deactivatedAt, then complete GitHub OAuth with the same identity; handleDeletedUserSignIn rejects it and tests assert that rejection.

## Solution Assessment

Unclear. Re-signup may be the right user outcome, but current docs and tests intentionally make deletion irreversible, so the maintainable fix requires a policy and security decision before code changes.
