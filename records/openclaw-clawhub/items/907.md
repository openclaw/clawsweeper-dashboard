## Summary

Keep open. Current main still rejects same-GitHub sign-in after user-initiated deletion, and the remaining change needs maintainer product and security judgment rather than conservative cleanup.

## What This Changes

The issue requests a way to re-sign up or recover sign-in with the same GitHub account after deleting a previous ClawHub account.

## Best Possible Solution

Maintainers should choose and document the deletion policy, then implement either secure same-GitHub recreate/relink or an explicit irreversible-deletion support flow across auth, account state, docs, and tests.

## Reproduction Assessment

Yes, by source inspection: deleteAccount writes deactivatedAt/purgedAt, afterUserCreatedOrUpdated calls handleDeletedUserSignIn, and that helper rejects deleted or deactivated users before a usable session can exist. I did not run a live GitHub OAuth flow or inspect production account rows.

## Solution Assessment

No for an automatic re-signup fix. The maintainable path needs an explicit product and security policy before changing identity recovery, deletion privacy, ban separation, publisher ownership, and operator support behavior.
