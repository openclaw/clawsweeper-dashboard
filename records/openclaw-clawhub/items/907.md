## Summary

Keep open. Current main still blocks same-GitHub sign-in after user-initiated deletion, tests and docs treat deletion as irreversible, and the ClawHub cleanup policy keeps issues open unless maintainers explicitly decide otherwise.

## What This Changes

The issue requests that users who deleted their ClawHub account be able to sign up or sign in again with the same GitHub account instead of landing in an OAuth/login loop.

## Best Possible Solution

Maintainers should decide and document the deletion policy; if re-signup is allowed, implement a secure same-GitHub recreate or relink flow that preserves abuse protections and ownership invariants, otherwise make the OAuth result and support path explicit enough that users are not left in a loop.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: delete an account so the user has deactivatedAt/purgedAt, then complete GitHub OAuth for the same identity; the auth callback rejects it and tests assert that rejection.

## Solution Assessment

Unclear. Same-GitHub re-signup may be the desired user outcome, but it changes the documented deletion contract, so the maintainable solution needs product and security policy approval first.
