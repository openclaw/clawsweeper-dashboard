## Summary

Keep open. Current main still blocks deleted/deactivated users during GitHub OAuth completion, and allowing same-GitHub re-signup would change ClawHub's documented irreversible-deletion policy and needs maintainer product/security judgment.

## What This Changes

The issue asks ClawHub to allow a user to sign up or sign in again with the same GitHub account after deleting the previous ClawHub account.

## Best Possible Solution

Maintainers should choose and document the deletion policy, then either implement a secure same-GitHub recreate/relink flow or make the blocked-account/support path explicit enough that users are not left in a loop.

## Reproduction Assessment

Yes, source-reproducible: current main rejects deleted/deactivated users during OAuth completion, and tests assert rejection for deactivated and legacy deleted users. I did not run the live GitHub OAuth browser flow, so the exact page-refresh loop was not runtime-reproduced here.

## Solution Assessment

No, not as an automatic bug fix. The requested re-signup path changes deletion and OAuth binding semantics, so the maintainable solution starts with maintainer product and security judgment.
