## Summary

Keep open. Current main still treats user-initiated deletion as irreversible and blocks same-GitHub re-signup; for ClawHub issues, this cleanup pass should not auto-close while the remaining account-lifecycle policy needs maintainer judgment.

## What This Changes

The issue requests same-GitHub sign-up again after deleting a previous ClawHub account instead of getting stuck in the OAuth/sign-in loop.

## Best Possible Solution

Maintainers should decide and document the account deletion policy, then either implement a secure same-GitHub recreate/relink flow or keep deletion irreversible with a clear OAuth/support path that prevents a confusing loop.

## Reproduction Assessment

Yes for the blocked re-signup path from source inspection: current auth code throws for deleted/deactivated users and tests assert rejection. I did not run a live browser OAuth flow, so the exact original silent refresh loop was not reproduced end to end.

## Solution Assessment

No as an automated source repair: same-GitHub re-signup changes the current irreversible-deletion policy. The maintainable path is product/security approval first, then a focused auth and account-lifecycle implementation if policy changes.
