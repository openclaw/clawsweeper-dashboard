## Summary

Keep open. Current main still blocks deleted/deactivated GitHub reauth and only surfaces an error, while same-GitHub re-signup or relinking would require a maintainer product and security decision; ClawHub cleanup policy does not auto-close issues.

## What This Changes

The issue requests being able to sign up or sign in again with the same GitHub account after deleting a prior ClawHub account.

## Best Possible Solution

Maintainers should define the account deletion/recovery policy first, then implement the chosen behavior consistently across OAuth, auth account binding, CLI token creation, deletion state, audit/operator flows, docs, and regression tests.

## Reproduction Assessment

Yes, by source inspection: deleteAccount writes deactivatedAt/purgedAt, afterUserCreatedOrUpdated calls handleDeletedUserSignIn, and that helper rejects deleted/deactivated users before a usable session can exist. I did not run a live OAuth flow or inspect production account rows.

## Solution Assessment

No for an automatic restore, purge, or auth-binding deletion as cleanup. The maintainable solution needs explicit policy covering identity proof, deletion privacy, ban separation, auditability, and operator support behavior.

## Security Review

Status: needs_attention

Summary: Deleted-account recovery is security-sensitive because it changes identity binding, irreversible deletion, and abuse-prevention semantics.

- **[high] Define recovery authority before reactivation:** `convex/auth.ts:45`

  - confidence: 0.92
- **[high] Keep banned and deleted states distinct:** `convex/auth.ts:51`
  - body: The current auth gate checks ban audit records for legacy deleted users and admin unban rejects deactivated accounts, so a recovery path must not let banned or autobanned identities regain access through deletion recovery.
  - confidence: 0.9

  - body: deleteAccount revokes API tokens and clears profile/contact/GitHub-derived fields; recovery must avoid restoring private data or previously revoked credentials unexpectedly.
