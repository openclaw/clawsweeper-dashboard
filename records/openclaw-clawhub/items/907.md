## Summary

Keep open. Current main still treats user-initiated account deletion as irreversible and blocks deleted/deactivated users during GitHub OAuth completion, and ClawHub cleanup policy keeps issues open rather than auto-closing them without maintainer judgment.

## What This Changes

The issue requests the ability to sign up or log in again with the same GitHub account after deleting a ClawHub account.

## Best Possible Solution

Maintainers should decide and document the deletion policy; if re-signup is allowed, implement a secure same-GitHub recreate or relink flow, otherwise keep irreversible deletion and make the blocked-account/support path explicit.

## Reproduction Assessment

Yes, source-reproducible: current main rejects deleted/deactivated users during OAuth completion, and tests assert that behavior for deactivated and legacy deleted users. I did not run a live GitHub OAuth browser flow, so the exact page-refresh loop is not runtime-reproduced here.

## Solution Assessment

No, not as an automatic bug fix. The requested re-signup path changes deletion and OAuth binding semantics, so the maintainable solution starts with maintainer product/security judgment before code changes.
