## Summary

Keep open. Current main still intentionally blocks sign-in for users whose account was deleted/deactivated, and repository cleanup policy for openclaw/clawhub keeps issues open unless maintainers explicitly resolve them; this remains a product and security decision, not a cleanup close.

## What This Changes

The issue requests that a user who deleted a ClawHub account can sign up again with the same GitHub OAuth identity instead of landing in a silent refresh/sign-in loop.

## Best Possible Solution

Maintainers should decide and document the deletion policy; if re-signup is allowed, implement a secure same-GitHub recreate or relink flow that preserves abuse and ownership invariants, otherwise make the OAuth/support result explicit enough that users are not left in a loop.

## Reproduction Assessment

Yes. The code-level reproduction is to delete an account and attempt GitHub OAuth with the same identity; current auth tests and handleDeletedUserSignIn show the deactivated/deleted user is rejected rather than restored or recreated.

## Solution Assessment

Unclear. Re-registration may be the right user-facing outcome, but the best implementation depends on maintainer decisions about irreversible deletion, OAuth account binding cleanup, bans, and publisher ownership.
