## Summary

Keep open. Current main still rejects same-GitHub sign-in after user-initiated account deletion and only improves the error surface; the remaining decision is product/security policy, not a completed fix.

## What This Changes

The issue requests that a user who deleted a ClawHub account be able to sign up or sign in again with the same GitHub account instead of landing in a refresh/login loop.

## Best Possible Solution

Maintainers should decide and document the deletion policy, then either implement a secure same-GitHub recreate/relink path or make irreversible deletion produce an explicit support/error flow that does not look like a silent loop.

## Reproduction Assessment

Yes. Current main has a code-level reproduction: delete an account so the user has `deactivatedAt`, then complete GitHub OAuth for that identity; `handleDeletedUserSignIn` throws, and the tests assert that behavior for deactivated and legacy-deleted users.

## Solution Assessment

Unclear. The requested re-signup behavior may be right for users, but it is not clearly the safest implementation until maintainers choose the deletion and relinking policy; a narrower maintainable fix must preserve ban, ownership, and token-revocation invariants.
