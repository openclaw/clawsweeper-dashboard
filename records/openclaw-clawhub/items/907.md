## Summary

Keep open. Current main still blocks same-GitHub sign-in after account deletion by design, while the remaining requested re-signup/relink behavior needs maintainer product and security judgment rather than cleanup auto-close.

## What This Changes

The issue requests that a user who deleted a ClawHub account can sign up or sign in again with the same GitHub account instead of being stuck after OAuth.

## Best Possible Solution

Maintainers should decide and document deletion semantics, then either build a secure same-GitHub re-registration/relinking path or keep deletion irreversible with a clear support/error path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: delete an account so the user has deactivatedAt/purgedAt, then complete GitHub OAuth for that identity; the auth callback rejects it, and tests assert that rejection.

## Solution Assessment

Unclear. Same-GitHub re-signup may be the desired user outcome, but current docs and tests intentionally block restoration, so a narrow maintainable fix needs a product and security policy decision first.
