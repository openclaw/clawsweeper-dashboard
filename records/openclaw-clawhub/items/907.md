## Summary

Keep open. Current main still blocks same-GitHub sign-in after user-initiated account deletion, and changing that behavior would require an explicit account-lifecycle product/security decision rather than conservative cleanup.

## What This Changes

User asks ClawHub to allow re-signup with the same GitHub account after deleting a previous account instead of returning to a sign-in loop.

## Best Possible Solution

Maintainers should choose one deleted-account policy, then either implement a secure restore/relink/recreate flow with regression coverage or keep deletion irreversible and make the OAuth, CLI, and support path explicit.

## Reproduction Assessment

Yes, source-reproducible: current main writes deactivatedAt/purgedAt on deletion, rejects deleted/deactivated users during OAuth completion, and keeps active-user/token gates closed. I did not run a live GitHub OAuth browser flow, so the exact page-refresh loop is not runtime-reproduced here.

## Solution Assessment

No, not as an automatic bug fix. Same-GitHub re-signup changes a documented irreversible-deletion policy, so the narrow maintainable path starts with maintainer product/security approval before code changes.

## Security Review

Status: needs_attention

Summary: Deleted-account recovery is security-sensitive because it changes identity reuse, deletion privacy, token revocation, and ownership semantics.

Concerns:

- **[medium] Define deleted-account recovery policy first:** `specs/security-moderation.md:132`
  - body: The spec currently says user-initiated deletion is irreversible and cannot be restored by logging in again, so restore, relink, or re-registration needs explicit maintainer approval.
  - confidence: 0.95
- **[medium] Preserve identity and ban safeguards:** `convex/auth.ts:37`
  - body: The OAuth callback blocks deleted/deactivated identities and checks existingUserId before migrating legacy deleted records, so any recovery path needs equivalent GitHub identity and ban/autoban protections.
  - confidence: 0.9
- **[medium] Define token and ownership semantics:** `convex/users.ts:338`
  - body: Account deletion revokes API tokens and purges profile/contact fields while published content remains public, so recovery must define whether tokens, audit history, publishers, and ownership are restored or recreated.
  - confidence: 0.9
