## Summary

Keep open. Current main still treats user-initiated deletion as irreversible and blocks same-GitHub OAuth sign-in after deletion; allowing re-signup/relinking is a product and security decision, and ClawHub cleanup policy does not auto-close issues.

## What This Changes

Allow or otherwise resolve attempts to sign up again with the same GitHub identity after a user deletes their ClawHub account.

## Best Possible Solution

Maintainers should define the account deletion recovery policy, then either implement a secure same-GitHub recreate/relink flow with tests and docs, or keep irreversible deletion and make the OAuth/support outcome explicit enough that users are not left in a loop.

## Reproduction Assessment

Yes, by source inspection. deleteAccount sets deactivatedAt/purgedAt, OAuth completion calls handleDeletedUserSignIn, and tests assert deleted/deactivated users are rejected; I did not run a live production OAuth flow.

## Solution Assessment

Unclear. Same-GitHub restoration, unlinking, or fresh re-registration could solve the user problem, but choosing one before defining recovery authority, deletion guarantees, and abuse controls would be premature.
