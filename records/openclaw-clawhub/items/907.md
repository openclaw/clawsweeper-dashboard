## Summary

Keep open. Current main now surfaces a blocked-account message, but it still intentionally rejects same-GitHub sign-in after user-initiated deletion and does not implement re-signup or relinking; for ClawHub issues, cleanup policy keeps this open for maintainer product and security judgment.

## What This Changes

Allow a user who deleted a ClawHub account to sign up or sign in again with the same GitHub identity instead of refreshing or looping after OAuth.

## Best Possible Solution

Maintainers should decide and document whether deletion remains irreversible; if re-signup is allowed, implement a secure same-GitHub recreate or relink flow that preserves ban protections, ownership invariants, and token revocation semantics.

## Reproduction Assessment

Yes, for the reported inability to re-signup: current main rejects deleted/deactivated users during OAuth completion, and tests assert that behavior. I did not run a live GitHub OAuth browser flow, so the visual page-refresh loop is source-reproducible rather than runtime-reproduced.

## Solution Assessment

No, not as a narrow bug fix yet. The requested path changes deletion and OAuth binding semantics, so the maintainable solution starts with a maintainer product/security decision before implementation.
