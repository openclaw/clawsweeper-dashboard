## Summary

Keep open. Current main still rejects same-GitHub sign-in after user-initiated deletion, while the UI now surfaces a deleted-account error instead of silently looping; resolving the requested re-signup path requires an account deletion and OAuth-binding policy decision, and ClawHub cleanup policy keeps issues open.

## What This Changes

The issue requests that a deleted ClawHub account be able to sign up or sign in again with the same GitHub account instead of refreshing or looping during GitHub OAuth.

## Best Possible Solution

Maintainers should decide and document the account deletion policy, then either implement a secure same-GitHub recreate/relink flow or keep irreversible deletion with an explicit OAuth/support path.

## Reproduction Assessment

Yes for the blocked same-GitHub re-signup path: current source rejects deleted/deactivated users during OAuth completion and tests assert that rejection. The older silent refresh loop itself was not rerun in a live browser OAuth flow.

## Solution Assessment

No as an automatic source fix: the requested re-signup path conflicts with the current irreversible-deletion policy. The safer route is maintainer product/security approval followed by a focused auth and account-lifecycle implementation.
