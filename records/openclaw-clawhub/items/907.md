## Summary

Keep open. Current main still blocks same-GitHub sign-in after user-initiated account deletion, while the requested re-signup/relink behavior conflicts with the documented irreversible-deletion policy and needs maintainer product and security judgment.

## What This Changes

Allow a user who deleted their ClawHub account to sign up or sign in again with the same GitHub account instead of getting stuck after OAuth.

## Best Possible Solution

Maintainers should decide and document the deletion policy, then either build a secure same-GitHub recreate/relink path or keep deletion irreversible with an explicit OAuth/support outcome.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: delete an account so the user gets deactivatedAt/purgedAt, then complete GitHub OAuth for the same identity; the auth callback rejects it and tests assert that rejection.

## Solution Assessment

Unclear. Same-GitHub re-signup may be the desired user outcome, but current docs and tests intentionally block restoration, so the maintainable solution needs a product and security policy decision first.
