## Summary

Keep open. Current main still blocks same-GitHub sign-in after user-initiated deletion, and resolving the request would change ClawHub account-deletion and recovery policy rather than apply a narrow bug fix.

## What This Changes

The issue requests that a user who deleted a ClawHub account can sign up or sign in again with the same GitHub account instead of being stuck in the OAuth/sign-in loop.

## Best Possible Solution

Maintainers should decide and document the deletion policy, then either implement a secure same-GitHub recreate/relink flow or keep irreversible deletion with clear OAuth and support messaging.

## Reproduction Assessment

Yes for the account lockout: current-main source and tests show deleted/deactivated users are rejected on OAuth completion. I did not run a live GitHub OAuth browser flow, so the historical silent refresh loop itself is source-reproduced rather than runtime-reproduced.

## Solution Assessment

No as an automatic fix. Same-GitHub re-signup changes deletion, identity-binding, and abuse-prevention semantics, so the maintainable path starts with maintainer product and security approval.
