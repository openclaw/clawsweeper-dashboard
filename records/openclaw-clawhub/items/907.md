## Summary

Keep open. Current main still blocks same-GitHub sign-in after user-initiated deletion, and this repository’s cleanup policy does not auto-close issues; the remaining work is an account lifecycle policy/security decision rather than a narrow automated bug fix.

## What This Changes

The issue requests that a user who deleted their ClawHub account be able to sign up or sign in again with the same GitHub account instead of looping or refreshing.

## Best Possible Solution

Maintainers should decide and document the intended deletion policy, then either implement a secure same-GitHub recreate/relink flow or keep deletion irreversible with an explicit OAuth/support path.

## Reproduction Assessment

Yes for the blocked re-signup path from source: current auth code rejects deactivated and legacy deleted users, and tests assert that behavior. The old silent refresh loop was not reproduced in a live browser OAuth flow, and current main now maps the backend failure to a visible error.

## Solution Assessment

No as an automatic fix: allowing re-signup would change the documented irreversible-deletion policy. The maintainable path is a product/security decision followed by a focused auth and account-lifecycle implementation.
