## Summary

Keep open. Current main still blocks same-GitHub sign-in after user-initiated deletion, and the remaining re-signup/relink behavior is a ClawHub account-lifecycle product and security decision rather than a closeable source cleanup item.

## What This Changes

The issue requests the ability to re-sign up or recover after deleting a ClawHub account linked to the same GitHub identity.

## Best Possible Solution

Maintainers should choose and document the deletion policy, then either implement a secure audited same-GitHub recreate/relink flow or keep deletion irreversible with an explicit OAuth/support path.

## Reproduction Assessment

Yes, by source inspection for the current blocked re-signup path: deleteAccount deactivates and purges the user, handleDeletedUserSignIn rejects deactivated or legacy deleted users, and tests assert rejection. I did not reproduce the original silent browser loop in a live production OAuth setup.

## Solution Assessment

No as an automatic source fix: allowing same-GitHub re-signup would change the current irreversible-deletion contract. The maintainable route is a maintainer-approved product/security policy followed by focused auth, user lifecycle, docs/spec, and regression-test changes.

## Security Review

Status: needs_attention

Summary: Deleted-account recovery is security-sensitive because it changes identity reuse, deletion guarantees, OAuth bindings, ownership, and ban-bypass protections.

Concerns:

- **[medium] Define recovery authority before changing sign-in:** `specs/security-moderation.md:180`
  - body: The spec currently says user-initiated deletion is irreversible and deleted accounts cannot be restored by logging in again, so restore, relink, or recreate behavior needs explicit policy approval before implementation.
  - confidence: 0.95
- **[medium] Preserve ban and identity safeguards:** `convex/auth.ts:40`
  - body: The OAuth callback checks deleted/deactivated state, mismatched existingUserId, and ban audit records before allowing sign-in; any recovery path needs equivalent identity matching and abuse-prevention checks.
  - confidence: 0.9
- **[medium] Preserve purge and token semantics:** `convex/users.ts:475`
  - body: Account deletion revokes API tokens and clears profile/contact/GitHub-derived fields, so recovery must define whether credentials, private profile data, publishers, packages, skills, and audit history are restored or recreated.
  - confidence: 0.9
