## Summary

Keep open. Current main intentionally blocks deleted/deactivated accounts during GitHub OAuth and documents user-initiated deletion as irreversible, so same-GitHub re-signup needs maintainer product and security review rather than conservative cleanup.

## What This Changes

The issue requests the ability to sign up again with the same GitHub account after deleting a previous ClawHub account instead of getting an OAuth refresh/sign-in loop.

## Best Possible Solution

Maintainers should decide and document the deletion policy, then either implement a secure same-GitHub recreate/relink flow that preserves abuse protections and ownership invariants or keep irreversible deletion while making the OAuth failure and support path explicit.

## Reproduction Assessment

Yes, by source inspection: deleteAccount sets deactivatedAt/purgedAt and handleDeletedUserSignIn rejects deactivated or legacy deleted users during OAuth completion. I did not run a live GitHub OAuth flow or inspect production rows.

## Solution Assessment

No for an automatic cleanup fix. The maintainable path needs a maintainer-approved deletion policy: either a secure relink/recreate flow or an explicit irreversible-deletion UX and support path.
