## Summary

Keep open. Current main still blocks same-GitHub sign-in after account deletion, and changing that behavior requires product and security decisions about irreversible deletion, OAuth binding, and abuse protections.

## What This Changes

The issue requests re-signup or relinking after a user deletes their ClawHub account and then tries GitHub OAuth again.

## Best Possible Solution

Maintainers should decide the deletion/recovery policy, document it, then implement either secure same-GitHub recreate/relink or clearer irreversible-deletion support behavior across auth, account state, CLI tokens, audit logs, and tests.

## Reproduction Assessment

Yes, by source inspection: deleteAccount writes deactivatedAt/purgedAt, afterUserCreatedOrUpdated calls handleDeletedUserSignIn, and that helper rejects deleted/deactivated users before a usable session can exist. I did not run a live OAuth flow or inspect production account rows.

## Solution Assessment

No for an automatic cleanup or restore. The maintainable solution needs an explicit product and security policy before changing identity recovery, deletion privacy, ban separation, and operator support behavior.
