## Summary

Keep open. Current main still blocks same-GitHub re-signup after account deletion by design, and this ClawHub issue needs maintainer product/security judgment rather than cleanup closure.

## What This Changes

The issue requests the ability to sign up or sign in again with the same GitHub account after deleting a prior ClawHub account.

## Best Possible Solution

Maintainers should decide and document the account deletion policy, then either implement a secure same-GitHub recreate/relink flow or keep deletion irreversible with an explicit OAuth/support path.

## Reproduction Assessment

Yes, by source inspection for the blocked same-GitHub reauth path: deletion marks the user inactive, the OAuth callback rejects deleted/deactivated users, and tests assert that rejection. I did not reproduce the older silent browser refresh loop in a live deployment, and current main now includes a visible deleted-account error path.

## Solution Assessment

No for an automated repair: allowing same-GitHub re-signup would change the documented irreversible-deletion contract. The maintainable path is a maintainer-approved product/security policy followed by focused auth, user lifecycle, docs/spec, and regression-test changes.
