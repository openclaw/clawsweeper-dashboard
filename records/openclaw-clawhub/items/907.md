## Summary

Keep open. Current main still treats user-initiated deletion as irreversible and rejects same-GitHub OAuth completion for deleted/deactivated users, while the requested re-signup or relink behavior needs maintainer product and security judgment.

## What This Changes

The issue asks for users who deleted their ClawHub account to be able to sign up or sign in again with the same GitHub account.

## Best Possible Solution

Maintainers should decide and document deletion semantics, then either build a secure same-GitHub recreate/relink path or keep deletion irreversible with an explicit OAuth/support outcome.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: delete an account so the user has `deactivatedAt`/`purgedAt`, then complete GitHub OAuth for that identity; the auth callback rejects it and tests assert that rejection.

## Solution Assessment

Unclear. Same-GitHub re-signup may be the desired user outcome, but current docs and tests intentionally block restoration, so the maintainable fix needs a product and security policy decision first.
