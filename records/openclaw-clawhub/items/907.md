## Summary

Keep open. Current main still blocks same-GitHub sign-in after self-service account deletion, and solving re-signup/relinking requires maintainer product and security decisions rather than conservative cleanup.

## What This Changes

The issue requests a way to sign up again with the same GitHub account after deleting a previous ClawHub account instead of returning to the login loop.

## Best Possible Solution

Maintainers should document the chosen deletion policy, then either implement a secure same-GitHub recreate/relink flow or keep irreversible deletion while making the OAuth failure and support path explicit.

## Reproduction Assessment

Yes, by source inspection: deleteAccount sets deactivatedAt/purgedAt and handleDeletedUserSignIn rejects deactivated or legacy deleted users during OAuth completion. I did not run a live GitHub OAuth flow or inspect production rows.

## Solution Assessment

No for an automatic cleanup fix. The maintainable path needs a maintainer-approved deletion policy: either a secure relink/recreate flow or an explicit irreversible-deletion UX and support path.
