## Summary

Keep open. Current main intentionally blocks same-GitHub sign-in after user-initiated deletion, and the remaining request would require a maintainer account-lifecycle and security decision rather than conservative cleanup.

## What This Changes

The issue requests that a user who deleted a ClawHub account can sign up or sign in again with the same GitHub account instead of looping or refreshing.

## Best Possible Solution

Maintainers should decide and document the deletion policy; if re-signup is allowed, implement a secure same-GitHub restore/relink/recreate flow that preserves abuse protections and ownership invariants, otherwise keep deletion irreversible and make the OAuth/CLI/support result explicit.

## Reproduction Assessment

Yes for the account-lockout behavior: current main source and tests show deletion sets deactivatedAt/purgedAt and later GitHub OAuth completion rejects the same user instead of restoring or recreating the account. I did not reproduce the exact browser refresh loop in a live OAuth flow.

## Solution Assessment

No, not as an automatic bug fix. Same-GitHub re-signup would change the documented irreversible-deletion policy, so the narrow maintainable path starts with maintainer product and security approval before code changes.

## Security Review

Status: needs_attention

Summary: Deleted-account recovery is security-sensitive because it changes identity reuse, deletion privacy, token revocation, and ownership semantics.

Concerns:

- **[medium] Define deleted-account recovery policy first:** `specs/security-moderation.md:132`
  - body: The spec currently says user-initiated deletion is irreversible and deleted accounts cannot be restored by logging in again, so restore, relink, or re-registration needs explicit maintainer approval.
  - confidence: 0.95
- **[medium] Preserve identity and ban safeguards:** `convex/auth.ts:37`
  - body: The OAuth callback blocks deleted/deactivated identities and checks existingUserId before migrating legacy deleted records, so any recovery path needs equivalent GitHub identity and ban/autoban protections.
  - confidence: 0.9
- **[medium] Define token and ownership semantics:** `convex/users.ts:338`
  - body: Account deletion revokes API tokens and purges profile/contact fields while published content remains public, so recovery must define whether tokens, audit history, publishers, and ownership are restored or recreated.
  - confidence: 0.9
