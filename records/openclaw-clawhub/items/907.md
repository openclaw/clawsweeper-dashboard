## Summary

Keep open. Current main still blocks same-GitHub sign-in after account deletion, and changing that requires an explicit account-deletion, OAuth relinking, and abuse-prevention policy rather than conservative cleanup.

## What This Changes

Allow users who deleted a ClawHub account to sign up or sign in again with the same GitHub identity instead of returning to the logged-out flow.

## Best Possible Solution

Maintainers should choose and document the deletion policy, then implement either a secure same-GitHub recreate/relink flow or an explicit irreversible-deletion support path across auth, account state, docs, and tests.

## Reproduction Assessment

Yes, by source inspection: deleteAccount writes deactivatedAt/purgedAt, afterUserCreatedOrUpdated calls handleDeletedUserSignIn, and that helper rejects deleted or deactivated users before a usable session can exist. I did not run a live GitHub OAuth flow or inspect production account rows.

## Solution Assessment

No for an automatic re-signup fix. The maintainable path needs an explicit product and security policy before changing identity recovery, deletion privacy, ban separation, publisher ownership, and operator support behavior.
