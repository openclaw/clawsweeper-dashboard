## Summary

Keep open. Current main still intentionally blocks same-GitHub re-signup after user-initiated deletion; the original silent-loop surface is better, but allowing re-registration or relinking remains an unresolved account-lifecycle and security-policy decision.

## What This Changes

The issue requests that a user who deleted a ClawHub account can sign up again with the same GitHub OAuth identity instead of landing in a refresh/sign-in loop.

## Best Possible Solution

Maintainers should choose and document deletion semantics, then either build secure same-GitHub re-registration/relinking that preserves abuse protections and ownership invariants, or keep deletion irreversible with an explicit support/error path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: delete an account so the user has deactivatedAt, then complete GitHub OAuth for that identity; handleDeletedUserSignIn rejects it, and current tests assert that behavior.

## Solution Assessment

Unclear. The requested re-signup may be the right user outcome, but current docs and tests intentionally make deletion irreversible, so the narrow maintainable solution needs a maintainer policy decision before code changes.
