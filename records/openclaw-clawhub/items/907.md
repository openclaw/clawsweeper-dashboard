## Summary

Keep open. Current main still blocks same-GitHub re-signup after user-initiated deletion, and allowing it would change the documented deletion contract plus OAuth, ban, token, and ownership semantics.

## What This Changes

Request to let users sign up or sign in again with the same GitHub account after deleting their previous ClawHub account instead of looping or staying logged out.

## Best Possible Solution

Maintainers should decide and document the deletion policy, then either implement a secure same-GitHub recreate/relink flow or make the irreversible-deletion error and support path explicit enough that users are not left in a loop.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: delete an account so the user has deactivatedAt/purgedAt, then complete GitHub OAuth for the same identity; the auth callback rejects it and tests assert that rejection.

## Solution Assessment

Unclear. Same-GitHub re-signup may be the desired user outcome, but it conflicts with the current irreversible-deletion contract, so the maintainable solution needs product and security approval first.
