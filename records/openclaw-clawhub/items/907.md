## Summary

Keep open. Current main still intentionally blocks same-GitHub sign-in after account deletion and only improves the error surface; the requested re-signup/relink behavior needs maintainer product and security judgment, and ClawHub issue cleanup policy does not auto-close issues here.

## What This Changes

The issue asks ClawHub to let a user sign up or sign in again with the same GitHub account after deleting a previous account instead of getting stuck after OAuth.

## Best Possible Solution

Maintainers should choose the account-deletion policy explicitly; if re-signup is allowed, add a secure recreate/relink flow that preserves abuse protections, otherwise keep irreversible deletion and make the OAuth result and support path unmistakable.

## Reproduction Assessment

Yes, for the core same-GitHub re-signup failure: source inspection shows deletion sets `deactivatedAt`/`purgedAt`, and the next OAuth completion rejects that user in `handleDeletedUserSignIn`. I did not run a live GitHub OAuth flow, so the old page-refresh loop is source-reproducible rather than runtime-reproduced.

## Solution Assessment

Unclear as stated. A one-off record purge would help individual reporters but is not the best product fix; the maintainable path is an explicit deletion policy plus either secure relinking/recreation or clearer irreversible-deletion messaging.
