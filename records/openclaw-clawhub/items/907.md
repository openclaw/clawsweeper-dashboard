## Summary

Keep open. Current main still treats user-initiated deletion as irreversible and blocks deleted/deactivated users during GitHub OAuth completion; allowing same-GitHub re-signup would need maintainer product and security policy, and ClawHub cleanup policy does not auto-close issues.

## What This Changes

The issue requests the ability to sign up or sign in again with the same GitHub account after deleting the previous ClawHub account.

## Best Possible Solution

Maintainers should decide and document the account deletion policy, then either implement a secure same-GitHub recreate/relink flow or keep deletion irreversible with an explicit OAuth/support path.

## Reproduction Assessment

Yes for the blocked re-signup path: current source throws for deleted/deactivated users and tests assert rejection. The exact old silent refresh loop was not exercised in a live browser OAuth flow, and current UI now maps blocked-auth failures to a visible error.

## Solution Assessment

No as an automatic fix: the requested same-GitHub re-signup changes the current irreversible-deletion policy. The maintainable path starts with product/security approval, then a focused auth and account-lifecycle implementation.
