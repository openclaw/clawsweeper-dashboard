## Summary

Keep open. Current main still rejects sign-in for deleted/deactivated GitHub-linked users, and for this repository cleanup pass ClawHub issues are not auto-closed; the remaining path needs maintainer product and security direction around deletion recovery and OAuth binding semantics.

## What This Changes

The issue asks ClawHub to allow a user who deleted their account to sign up or sign in again with the same GitHub OAuth identity instead of looping at sign-in.

## Best Possible Solution

Maintainers should define and document the deletion recovery policy, then either implement a secure same-GitHub recreate/relink flow or make irreversible deletion produce a clear support outcome instead of a loop.

## Reproduction Assessment

Yes, by source inspection. The deletion mutation leaves the user deactivated/purged, OAuth completion rejects that state, and regression tests assert rejected deleted-user sign-in rather than recreation.

## Solution Assessment

Unclear. Recreate, relink, restore, or support-assisted purge could solve the user problem, but the best path depends on maintainer policy for deletion guarantees and abuse controls.
