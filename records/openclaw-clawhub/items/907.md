## Summary

Keep open. Current main still blocks same-GitHub sign-in after account deletion and only surfaces a user-facing error; the requested re-signup or relink behavior is still a product/security decision, and ClawHub issue cleanup does not auto-close issues.

## What This Changes

Allow a user who deleted a ClawHub account to sign up or sign in again with the same GitHub account instead of looping at OAuth completion.

## Best Possible Solution

Maintainers should decide and document the deletion policy, then either implement a secure same-GitHub recreate/relink flow or make the irreversible-deletion OAuth/support outcome explicit enough that users are not left in a loop.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is: delete an account so its user has `deactivatedAt`, then complete GitHub OAuth for that identity; `afterUserCreatedOrUpdated` calls `handleDeletedUserSignIn`, and current tests assert the rejection path.

## Solution Assessment

Unclear. Same-GitHub re-signup may be the right user outcome, but it is not the narrowest safe change until maintainers choose the account-deletion and OAuth relinking policy.
