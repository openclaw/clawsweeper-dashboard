## Summary

Keep open. Current main still intentionally blocks same-GitHub sign-in after account deletion, and changing that behavior conflicts with the documented irreversible-deletion policy and needs maintainer product/security judgment.

## What This Changes

The issue requests a same-GitHub re-signup path after a user deletes their previous ClawHub account.

## Best Possible Solution

Maintainers should choose and document the deletion policy, then either implement a secure same-GitHub recreate/relink flow or keep deletion irreversible with a clear OAuth/support outcome.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: delete an account so the user has deactivatedAt/purgedAt, then complete GitHub OAuth for the same identity; the auth callback rejects it and tests assert that rejection.

## Solution Assessment

Unclear. Same-GitHub re-signup may be the desired user outcome, but it changes the documented deletion contract, so the maintainable solution needs product and security policy approval first.
