## Summary

Keep open. Current main still blocks same-GitHub sign-in after user-initiated deletion; the remaining re-signup/relink behavior needs maintainer product and security judgment, and ClawHub cleanup policy keeps issues open rather than auto-closing them.

## What This Changes

The issue asks for users who deleted their ClawHub account to sign up or sign in again with the same GitHub identity instead of returning to a logged-out loop.

## Best Possible Solution

Maintainers should choose and document the account-deletion policy, then implement either a secure same-GitHub recreate/relink flow or an explicit irreversible-deletion support/admin path across auth, docs, and tests.

## Reproduction Assessment

Yes, by source inspection: deleteAccount writes deactivatedAt/purgedAt, afterUserCreatedOrUpdated calls handleDeletedUserSignIn, and that helper rejects deleted or deactivated users before a usable session can exist. I did not run a live GitHub OAuth flow or inspect production account rows.

## Solution Assessment

No for an automatic re-signup fix. The maintainable path needs an explicit product and security policy before changing identity recovery, deletion privacy, ban separation, publisher ownership, and operator support behavior.
