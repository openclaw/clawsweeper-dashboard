## Summary

Keep open. Current main still intentionally rejects GitHub re-authentication for deleted/deactivated accounts, and the remaining work is a product/security decision about irreversible deletion, OAuth relinking, and abuse protections rather than a conservative cleanup close.

## What This Changes

The issue requests that a user who deleted a ClawHub account be able to sign up or sign in again with the same GitHub account instead of looping after OAuth.

## Best Possible Solution

Maintainers should choose and document the deletion policy, then either implement secure same-GitHub recreate/relink with abuse protections or keep irreversible deletion while making the OAuth/support outcome explicit.

## Reproduction Assessment

Yes, by source inspection: the deletion mutation writes deactivatedAt/purgedAt, the auth callback calls handleDeletedUserSignIn, and that helper rejects deleted/deactivated users before a usable session can exist. I did not run a live GitHub OAuth flow or inspect production rows.

## Solution Assessment

No for an automated cleanup fix. The durable solution needs an explicit product and security decision before changing deletion guarantees, OAuth identity binding, and ban separation.
