## Summary

Keep open. Current main still blocks same-GitHub sign-in after user-initiated deletion, and allowing re-signup or relinking would change account deletion and OAuth identity policy, so this needs maintainer product and security direction rather than cleanup closure.

## What This Changes

The issue requests that a user who deleted a ClawHub account can sign up or log in again with the same GitHub account instead of getting stuck after OAuth.

## Best Possible Solution

Maintainers should decide and document the deletion recovery policy, then either implement a secure same-GitHub recreate/relink flow or keep irreversible deletion with an explicit OAuth error and support path.

## Reproduction Assessment

Yes, for the blocked re-signup behavior by source inspection: deleteAccount sets deactivatedAt/purgedAt, OAuth completion calls handleDeletedUserSignIn, and tests assert deleted/deactivated users are rejected. I did not run a live production OAuth flow, and current main may now show an error instead of a silent refresh loop.

## Solution Assessment

Unclear. Same-GitHub restoration, identity unlinking, or fresh re-registration could solve the user problem, but choosing one requires maintainer decisions about deletion guarantees, recovery authority, and abuse controls.
