## Summary

Keep open. Current main still blocks same-GitHub re-signup after account deletion and now surfaces a permanent-deletion error, while the requested recovery/recreate behavior requires explicit product and security policy for account lifecycle handling.

## What This Changes

Allow a user who deleted their ClawHub account to sign up or sign in again with the same GitHub account instead of getting stuck after OAuth.

## Best Possible Solution

Maintainers should choose and document the account deletion policy, then either implement a secure same-GitHub recreate/relink flow or keep deletion irreversible with an explicit OAuth/support path.

## Reproduction Assessment

Yes for the blocked re-signup path: current source throws for deleted/deactivated users and tests assert rejection. The exact silent refresh loop was not exercised in a live browser OAuth flow, and current UI code now attempts to show an auth error.

## Solution Assessment

No as an automatic fix: the requested re-signup behavior changes the current irreversible-deletion policy. The narrow maintainable path starts with product/security approval, then a focused auth and account-lifecycle implementation.

## Security Review

Status: needs_attention

Summary: Deleted-account recovery is security-sensitive because it affects identity reuse, deletion guarantees, OAuth bindings, ban bypass protections, and token/ownership state.

Concerns:

- **[medium] Define recovery policy before changing auth:** `specs/security-moderation.md:130`
  - body: The spec currently states user-initiated deletion is irreversible and deleted accounts cannot be restored by logging in again, so restore/relink/recreate behavior needs explicit policy approval before implementation.
  - confidence: 0.95
- **[medium] Preserve ban and identity safeguards:** `convex/auth.ts:37`
  - body: The OAuth callback currently checks deleted/deactivated state, mismatched existingUserId, and ban audit records before blocking sign-in; any recovery path needs equivalent identity matching and abuse-prevention checks.
  - confidence: 0.9
- **[medium] Define token and ownership semantics:** `convex/users.ts:338`
  - body: Account deletion revokes API tokens and clears profile/contact fields while published content remains public, so recovery must define whether tokens, publishers, skills, packages, and audit history are restored or recreated.
