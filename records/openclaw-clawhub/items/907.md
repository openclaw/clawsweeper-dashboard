## Summary

Keep open. Current main still rejects same-GitHub sign-in after user-initiated deletion, while the requested re-signup/relink behavior conflicts with the documented irreversible-deletion policy and needs product/security judgment.

## What This Changes

The issue requests a way to sign in or re-register with the same GitHub account after deleting a ClawHub account.

## Best Possible Solution

Maintainers should decide and document whether deletion stays irreversible or supports same-GitHub re-signup, then implement either explicit blocking/support messaging or a secure recreate/relink flow with regression coverage.

## Reproduction Assessment

Yes for the blocked deleted-account path: current source rejects deactivated and legacy deleted users during auth completion, and tests assert that rejection. I did not run a live browser OAuth callback in this read-only review.

## Solution Assessment

No for an automatic repair: a re-signup/relink patch would change documented account-deletion semantics. The safer path is a maintainer-approved policy decision followed by a focused auth/account-lifecycle implementation.

## Security Review

Status: needs_attention

Summary: Account recovery after deletion is security-sensitive because it affects OAuth identity reuse, deletion guarantees, ban protections, token revocation, and ownership state.

Concerns:

- **[medium] Define recovery policy before changing auth:** `specs/security-moderation.md:136`
  - body: The spec currently states user-initiated deletion is irreversible and deleted accounts cannot be restored by logging in again, so restore, relink, or recreate behavior needs explicit policy approval before implementation.
  - confidence: 0.95
- **[medium] Preserve ban and identity safeguards:** `convex/auth.ts:37`
  - body: The OAuth callback checks deleted/deactivated state, mismatched existingUserId, and ban audit records before blocking sign-in; any recovery path needs equivalent identity matching and abuse-prevention checks.
  - confidence: 0.9
- **[medium] Define token and ownership semantics:** `convex/users.ts:338`
  - body: Account deletion revokes API tokens and clears user profile/contact fields while published content remains public, so recovery must define whether tokens, publishers, skills, packages, and audit history are restored or recreated.
  - confidence: 0.9
