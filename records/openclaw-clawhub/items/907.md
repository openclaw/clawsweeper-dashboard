## Summary

Keep open. Current main still treats user-initiated deletion as irreversible and rejects the same GitHub identity during OAuth completion, while the requested re-signup behavior requires maintainer product and security decisions rather than conservative cleanup.

## What This Changes

The issue requests a way for a user who deleted their ClawHub account to sign up or sign in again with the same GitHub account instead of being stuck in an OAuth loop.

## Best Possible Solution

Maintainers should decide and document the deletion policy, then either implement a secure same-GitHub recreate/relink flow that preserves abuse protections and ownership invariants or keep irreversible deletion while making the OAuth failure and support path explicit.

## Reproduction Assessment

Yes, by source inspection: account deletion sets deactivatedAt/purgedAt and the auth hook rejects deleted or deactivated users during GitHub OAuth completion. I did not run a live GitHub OAuth flow or inspect production rows.

## Solution Assessment

No for an automatic cleanup fix. The maintainable solution needs explicit product and security policy for deletion recovery, OAuth relinking, ban separation, publisher ownership, and privacy before implementation.
