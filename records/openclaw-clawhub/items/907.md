## Summary

Keep open. Current main still blocks same-GitHub sign-in after account deletion, and the remaining recovery/relinking behavior is a security-sensitive product decision rather than conservative cleanup.

## What This Changes

The reporter requests being able to sign up or sign in again with the same GitHub account after deleting a previous ClawHub account instead of getting stuck in a refresh/login loop.

## Best Possible Solution

Define a maintainer-approved deleted-account recovery policy, then update OAuth, auth account binding, deletion/admin flows, user-facing errors, docs/specs, and regression tests consistently.

## Reproduction Assessment

Yes, by source inspection. deleteAccount marks the user deactivated/purged, GitHub OAuth completion calls handleDeletedUserSignIn, and tests assert deleted/deactivated users are rejected rather than re-registered; I did not run a live production OAuth flow.

## Solution Assessment

Unclear. Same-GitHub restore, unlink, or re-registration could solve the user problem, but choosing one before defining recovery authority, deletion privacy, and abuse controls would be the wrong level of fix.
