## What This Changes

The PR threads optional ownerHandle/--owner support through the skill publish schema, v1 JSON and multipart parsing, server-side publisher resolution, CLI publish commands, generated schema artifacts, and regression tests.

## Best Possible Solution

Review and merge this PR, or an equivalent focused patch, after resolving mergeability and adding the changelog entry while preserving server-side publisher authorization and JSON, multipart, schema, and CLI coverage.

## Reproduction Assessment

Yes. Current main reproduces by inspection: schema parsing, multipart parsing, the REST handler, and the CLI payload path all omit an owner target, so org-owned publishes fall back to the API-token actor's personal publisher.

## Solution Assessment

Yes, with a small process cleanup. The PR follows the existing web publish pattern by normalizing ownerHandle, resolving publisher access server-side with minimumRole "publisher", and passing only the resolved ownerPublisherId into the shared publish helper.
