Yes. Current main reproduces by inspection: the REST schema/parsers and CLI payload omit an owner target, so API-token publishes fall back to the actor publisher and cannot update an existing org-owned slug.

## Solution Assessment

Yes, with a small process fix. Reusing the existing publisher resolver and passing only the resolved `ownerPublisherId` is the narrow maintainable approach; add the required changelog entry before merge.
