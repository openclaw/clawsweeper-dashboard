## Summary

Keep open. Current main still lacks REST/CLI owner-target support, and this PR is the focused implementation for the org-owned skill publish gap; the only concrete review issue found is the missing required changelog entry before merge.

## What This Changes

The PR threads optional `ownerHandle`/`--owner` through skill publish schemas, v1 JSON and multipart parsing, server-side publisher resolution, CLI payloads, generated schema artifacts, and regression tests.

## Best Possible Solution

Merge this focused implementation, or an equivalent patch, after adding the changelog entry and preserving server-side publisher-role resolution across JSON, multipart, and CLI paths.

## Reproduction Assessment

Yes. Current main reproduces by inspection: schema parsing, multipart parsing, REST handling, and CLI payload generation all omit an owner target, so API-token publishes fall back to the actor's personal publisher for org-owned slugs.

## Solution Assessment

Yes, with a small process cleanup. Reusing the existing web publish resolver with `minimumRole: "publisher"` and passing only `ownerPublisherId` into publish storage is the narrowest maintainable fix; add the required changelog entry before merge.
