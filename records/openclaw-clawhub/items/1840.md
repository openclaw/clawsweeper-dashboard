## Summary

Keep open. Current main still lacks REST/CLI owner-target support for skill publishes, and this PR is the focused implementation for the open #1814 gap; it needs changelog cleanup and normal maintainer review rather than closure.

## What This Changes

The PR threads optional `ownerHandle`/`--owner` support through skill publish schemas, JSON and multipart API parsing, server-side publisher resolution, CLI payload generation, generated schema artifacts, and regression tests.

## Best Possible Solution

Review and merge this PR, or an equivalent focused patch, after adding the changelog entry and rebasing as needed while preserving server-side publisher authorization and JSON, multipart, schema, and CLI coverage.

## Reproduction Assessment

Yes. Current main reproduces by inspection: the schema, multipart parser, REST handler, and CLI payload path all omit an owner target, so org-owned publishes fall back to the API-token actor's personal publisher.

## Solution Assessment

Yes, with a small process cleanup. The PR follows the existing web publish pattern by normalizing `ownerHandle`, resolving publisher access server-side with `minimumRole: "publisher"`, and passing only the resolved publisher id into the shared publish helper.
