## Summary

Keep open. Current main still lacks an owner target for REST/CLI skill publishing, and this PR is the focused implementation candidate, but it still needs the required changelog entry plus normal maintainer review before merge.

## What This Changes

The PR threads optional `ownerHandle`/`--owner` through skill publish schemas, JSON and multipart API parsing, server-side publisher resolution, CLI payload generation, generated schema artifacts, and regression tests.

## Best Possible Solution

Land this PR or an equivalent focused patch after adding the changelog entry, keeping publisher selection server-resolved and preserving JSON, multipart, and CLI coverage.

## Reproduction Assessment

Yes. Current main is source-reproducible: schema parsing, multipart rebuilding, REST handler calls, and CLI payload generation all omit an owner target, so API-token publishes fall back to the actor's personal publisher for org-owned slugs.

## Solution Assessment

Yes, with cleanup. Reusing `resolvePublishTargetForUserInternal` with `minimumRole: "publisher"` and passing only `ownerPublisherId` into publishing is the narrow maintainable solution; the PR should add the required changelog entry before merge.
