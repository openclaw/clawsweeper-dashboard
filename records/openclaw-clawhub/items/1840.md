## Summary

Keep open. Current main still lacks the REST/CLI owner target needed for org-owned skill publishes, and this PR is the active focused implementation rather than obsolete work; the only concrete review finding is a required changelog entry before merge.

## What This Changes

The PR adds optional `ownerHandle`/`--owner` support across skill publish schemas, v1 JSON and multipart publish handling, server-side publisher resolution, CLI payload generation, generated schema artifacts, and regression tests, plus a small maintenance lint cleanup.

## Best Possible Solution

Merge this PR or an equivalent focused patch after adding the changelog entry, preserving server-side publisher resolution with `minimumRole: "publisher"` and coverage for schema, JSON, multipart, and CLI payload flows.

## Reproduction Assessment

Yes. Current main reproduces by inspection: schema parsing, multipart parsing, the REST handler, and CLI payload generation all omit an owner target, so org-owned publishes through REST/CLI fall back to the actor's personal publisher.

## Solution Assessment

Yes, with small cleanup. The PR follows the existing web publish pattern by normalizing `ownerHandle`, resolving publisher access server-side, and passing only the resolved publisher id into the shared publish helper.
