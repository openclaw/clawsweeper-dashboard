## What This Changes

The PR threads optional ownerHandle/--owner support through skill publish schemas, v1 JSON and multipart publish parsing, server-side publisher resolution, CLI payload generation, generated schema artifacts, and regression tests.

## Best Possible Solution

Land this PR, or an equivalent focused patch, after adding a changelog entry while preserving server-side publisher authorization and JSON, multipart, schema, and CLI coverage.

## Reproduction Assessment

Yes. Current main reproduces by inspection: the schema lacks ownerHandle, multipart parsing rebuilds the payload without it, the CLI cannot send --owner, and the REST handler publishes directly through the API-token actor’s default publisher path.

## Solution Assessment

Yes, with a small process cleanup. The PR follows the existing web publish pattern by normalizing ownerHandle, resolving publisher access server-side with minimumRole publisher, and passing ownerPublisherId into the shared publish helper; the changelog entry should be added before merge.
