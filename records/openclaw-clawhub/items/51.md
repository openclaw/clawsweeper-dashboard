## Summary

Keep open. The report is still valid for the legacy `clawdhub` npm package: current source has moved to `clawhub` and declares `undici`, but live `clawdhub@latest` still resolves to `0.3.0`, whose metadata omits `undici` while its published build imports it.

## What This Changes

The issue reports that `clawdhub@0.3.0` imports `undici` from its published build without declaring it as a dependency, causing fresh npm/npx installs to fail at runtime.

## Best Possible Solution

A maintainer should resolve the legacy npm package state by publishing or retagging a stable patched `clawdhub` release, or by deprecating `clawdhub@0.3.0` with clear migration guidance to the current `clawhub` package, then close duplicate reports against the chosen canonical item.
