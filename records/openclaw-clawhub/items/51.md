## Summary

Keep open. Current source and `clawhub@latest` include `undici`, but live legacy `clawdhub@latest` still resolves to `0.3.0`, whose package metadata omits `undici` while its published build imports it.

## What This Changes

The report asks for the legacy `clawdhub` npm package to include `undici` or ship a patched release so clean installs do not crash with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

A maintainer should publish or retag a stable patched `clawdhub` release, or deprecate `clawdhub@0.3.0` with migration guidance to the current `clawhub` package, then close duplicates against the chosen canonical report.

Yes. The live npm registry metadata plus the published `clawdhub@0.3.0` tarball reproduce the root cause: the installed package would not receive `undici`, but its `dist/http.js` imports it.
