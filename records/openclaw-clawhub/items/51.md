## Summary

Keep open. The report still matches the live legacy npm package: current source declares `undici` for the renamed `clawhub` CLI, but `clawdhub@latest` still resolves to `0.3.0`, whose package metadata omits `undici` while the published build imports it; ClawHub issue policy also calls for issues to remain open for maintainer handling.

## What This Changes

The issue reports that legacy `clawdhub@0.3.0` crashes after a clean npm/npx install because its published build imports `undici` without declaring it as a dependency.

## Best Possible Solution

A maintainer should publish or retag a stable patched `clawdhub` release, or deprecate `clawdhub@0.3.0` with migration guidance to `clawhub`, then consolidate the duplicate reports.

## Reproduction Assessment

Yes. Live npm metadata plus the published `clawdhub@0.3.0` tarball reproduce the root cause: the installed package does not receive `undici`, but its `dist/http.js` imports it.

## Solution Assessment

Yes for the remaining state. Registry or deprecation action is the narrow maintainable fix because source already declares `undici` for the renamed `clawhub` package, and another source-only PR would not repair existing `clawdhub@latest` installs.
