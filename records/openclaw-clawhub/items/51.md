## Summary

Keep open. The current source package includes `undici` for the renamed `clawhub` CLI, but live `clawdhub@latest` still resolves to `0.3.0`, whose metadata omits `undici` while its published build imports it; fixing this needs npm registry/release action, and ClawHub policy keeps issues open for maintainer handling.

## What This Changes

The issue asks maintainers to add `undici` to the legacy `clawdhub` package dependencies and publish a patched release so clean installs do not crash.

## Best Possible Solution

A maintainer should publish or retag a stable patched `clawdhub` release, or deprecate `clawdhub@0.3.0` with clear migration guidance to `clawhub`, then close duplicates against the chosen canonical report.

## Reproduction Assessment

Yes. The live npm registry metadata plus the published `clawdhub@0.3.0` tarball reproduce the root cause: the installed package would not receive `undici`, but its `dist/http.js` imports it.

## Solution Assessment

Yes for the remaining state: registry or deprecation action is the narrow maintainable fix. Adding `undici` in source is already done for the renamed `clawhub` package, so another source-only PR would not repair existing `clawdhub@latest` installs.
