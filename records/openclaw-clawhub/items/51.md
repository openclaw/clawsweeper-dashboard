## Summary

Keep open. The live legacy npm package still matches the reported failure, while current main only fixes the renamed `clawhub` package/release path; ClawHub issue policy also calls for maintainer handling rather than auto-close.

## What This Changes

The issue reports that `clawdhub@0.3.0` imports `undici` from its published build without declaring it as a dependency, causing clean npm/npx installs to fail at runtime.

## Best Possible Solution

Resolve the legacy npm package state by publishing or retagging a stable patched `clawdhub` release, or by deprecating `clawdhub@0.3.0` with migration guidance to `clawhub`, then close duplicates against the chosen canonical report.

## Reproduction Assessment

Yes. The live npm registry metadata plus the published `clawdhub@0.3.0` tarball reproduce the root cause: installed metadata omits `undici`, but `dist/http.js` imports it.

## Solution Assessment

Yes for the remaining state. Registry or deprecation action is the narrow maintainable fix because current source already declares `undici` for the renamed `clawhub` package and a source-only PR would not repair existing `clawdhub@latest` installs.
