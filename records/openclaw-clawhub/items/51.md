## Summary

Keep open. This remains a valid live-package bug for the legacy `clawdhub` npm name: current source has moved to `clawhub` and declares `undici`, but `clawdhub@latest` still resolves to `0.3.0`, whose published metadata omits `undici` while its build imports it.

## What This Changes

The issue asks for the legacy `clawdhub@0.3.0` package to declare `undici` and receive a patched npm release so clean global installs do not crash.

## Best Possible Solution

A maintainer should publish or retag a stable patched `clawdhub` release, or deprecate `clawdhub@0.3.0` with clear migration guidance to `clawhub`, then close duplicate reports against the chosen canonical item.

## Reproduction Assessment

Yes. The live npm registry metadata and published `clawdhub@0.3.0` tarball reproduce the root cause: installs do not receive `undici`, while `dist/http.js` imports it.
