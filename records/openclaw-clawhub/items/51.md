## Summary

Keep open. The current `clawhub` source and npm package include `undici`, but the reported legacy `clawdhub@latest` path still resolves to `0.3.0`, whose registry metadata omits `undici` while the published build imports it; resolving that needs maintainer npm registry/release action, and ClawHub policy keeps issues open for maintainer handling.

## What This Changes

The issue asks maintainers to add `undici` to the legacy `clawdhub` npm package dependencies and publish a patched release so fresh installs do not crash with `ERR_MODULE_NOT_FOUND`.

## Best Possible Solution

A maintainer should publish or retag a stable patched `clawdhub` release, or deprecate `clawdhub@0.3.0` with clear migration guidance to the current `clawhub` package, then close duplicates against the chosen canonical report.
