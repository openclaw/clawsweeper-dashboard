## Summary

Keep open: current main still has a source-level path where one selected `clawhub sync` publish can abort the rest of the batch on the backend slug-taken error. ClawHub cleanup policy also keeps issues open rather than auto-closing them.

## What This Changes

The issue asks for `clawhub sync` to continue updating non-conflicting skills when one selected skill hits a slug ownership conflict.

## Best Possible Solution

Add CLI-level per-skill conflict handling around sync publishing, preserve backend slug ownership rules, continue remaining selected skills, and report accurate partial-sync results.

## Reproduction Assessment

Yes. Source inspection shows `/api/v1/resolve` can classify a slug-mismatched skill as `update`, `cmdSync` then directly awaits `cmdPublish`, and `cmdPublish` rethrows the backend slug-taken error.

## Solution Assessment

Yes. The narrow maintainable fix is CLI sync-loop handling for the known slug-taken publish failure; relaxing backend slug ownership or changing resolve ownership semantics is unnecessary for the reported batch-abort bug.
