## Summary

Keep open: current main still exposes the reported source-level failure path where one selected `sync` publish can throw the backend slug-conflict error and abort the rest of the batch. ClawHub cleanup policy also keeps issues open rather than auto-closing them.

## What This Changes

The issue reports that `clawhub sync` aborts all selected updates when one publish fails with a slug-already-taken conflict.

## Best Possible Solution

Keep backend slug ownership enforcement intact and make CLI sync handle known slug-taken publish failures per skill, continue later selected publishes, and report an accurate partial-failure result.

## Reproduction Assessment

Yes, there is a high-confidence source-level reproduction path, though I did not run a live registry reproduction. On current main, a selected skill classified as `update` can reach `cmdPublish`, the backend can throw the reported slug-taken error, and `cmdSync` has no per-skill catch before continuing the loop.

## Solution Assessment

Yes. The narrow maintainable fix is CLI-level per-skill handling for the known slug-conflict publish failure while preserving backend uniqueness and owner checks.

## Repair Work Prompt

Fix the `clawhub sync` batch behavior when one selected skill fails to publish because the backend reports `Slug is already taken. Choose a different slug.` with an optional `Existing skill: /<owner>/<slug>` suffix. In `packages/clawhub/src/cli/commands/sync.ts`, catch only this known other-owner slug conflict around the per-skill `cmdPublish` call, record that skill as failed or skipped with the existing skill URL when present, continue publishing the remaining selected skills, and print a clear partial-success/partial-failure summary. Preserve normal failure behavior for unrelated publish errors, keep backend uniqueness/ownership checks unchanged, and add focused regression coverage in `packages/clawhub/src/cli/commands/sync.test.ts`. Update changelog only if maintainers consider the CLI behavior user-facing for the next release.
