## Summary

Keep open: current main still has a clear source-level path where one selected `clawhub sync` publish can abort the whole batch on the backend slug-taken error, and ClawHub cleanup policy keeps issues open rather than auto-closing them.

## What This Changes

The issue requests that `clawhub sync` not fail all skill updates when one selected skill hits a slug ownership conflict.

## Best Possible Solution

Add CLI-level per-skill handling for the known slug-taken publish error, continue the remaining selected skills, and report an accurate partial-sync result while preserving backend slug ownership rules.

## Reproduction Assessment

Yes, via source inspection. `/api/v1/resolve` can classify a slug-mismatched skill as `update`, `cmdSync` then awaits `cmdPublish`, and `cmdPublish` rethrows the backend slug-taken error.

## Solution Assessment

Yes. The narrow maintainable fix is in the CLI sync publish loop; relaxing backend slug uniqueness or ownership checks is not needed for the reported batch-abort bug.
