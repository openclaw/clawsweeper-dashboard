## Summary

Keep open: ClawHub cleanup policy keeps issues open, and current main still has a source-level path where one slug-taken publish failure can abort the rest of `clawhub sync`.

## What This Changes

The issue reports that `clawhub sync` aborts when one selected skill hits a backend slug conflict.

## Best Possible Solution

Add CLI-level per-skill conflict handling around sync publishing: detect the known slug-taken publish error, record the affected skill as failed or skipped with the existing owner URL when available, continue with later selected skills, and report a clear partial-success result while preserving backend uniqueness.

## Reproduction Assessment

Yes, via source inspection. Current main can classify a slug-mismatched remote skill as `update`, then `cmdSync` directly awaits `cmdPublish`, and `cmdPublish` rethrows the backend slug-taken error.

## Solution Assessment

Yes. The narrow maintainable fix is in the CLI sync publish loop; backend slug ownership rules and the resolve endpoint contract do not need to be relaxed to fix batch continuation.
