## Summary

Keep open: current main still has a source-level path where one slug conflict during `clawhub sync` aborts the remaining selected publishes, and ClawHub cleanup policy keeps issues open.

## What This Changes

Fix `clawhub sync` so a slug conflict for one selected skill does not prevent remaining selected skills from publishing.

## Best Possible Solution

Add CLI-level per-skill handling for known slug-taken publish failures, continue with the remaining selected skills, and report an accurate partial-success result while preserving backend slug uniqueness.

## Reproduction Assessment

Yes, via source inspection. `/api/v1/resolve` can classify a slug-mismatched skill as `update`, then `cmdSync` awaits `cmdPublish`, and `cmdPublish` rethrows the backend slug-taken error.

## Solution Assessment

Yes. The narrow maintainable fix is in the CLI sync publish loop; backend slug uniqueness and owner-scoped publishing should remain unchanged.
