## Repair Work Prompt

Fix the `clawhub sync` batch-abort bug reported in https://github.com/openclaw/clawhub/issues/868. In `packages/clawhub/src/cli/commands/sync.ts`, catch publish failures per selected skill only around the `cmdPublish` call. Detect the known backend slug conflict message (`Slug is already taken. Choose a different slug.` and optional `Existing skill: ...`), record that skill as failed or skipped with the conflicting URL/message when present, continue publishing later selected skills, and print an accurate summary of uploaded versus failed/skipped skills. Do not relax `convex/skills.ts` slug uniqueness, owner-publisher checks, or explicit migration requirements, and do not silently swallow unrelated publish errors. Add focused regression coverage in `packages/clawhub/src/cli/commands/sync.test.ts` where one mocked `cmdPublish` rejects with the slug-taken message and a later selected skill is still attempted; assert the final output does not claim every selected skill uploaded successfully. Preserve existing dry-run, selection, changelog, telemetry, and duplicate-slug behavior.
