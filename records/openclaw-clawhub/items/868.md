## Repair Work Prompt

Fix the `clawhub sync` slug-conflict batch abort from https://github.com/openclaw/clawhub/issues/868. In `packages/clawhub/src/cli/commands/sync.ts`, wrap each selected skill publish so the known backend message `Slug is already taken. Choose a different slug. Existing skill: /owner/slug` is recorded as a per-skill failed or skipped result and sync continues to later selected skills. Print an accurate partial-success summary that does not claim all selected skills uploaded, include the existing skill URL when the error exposes it, and keep unrelated publish errors visible instead of swallowing them. Do not change backend slug uniqueness, owner migration, or publish authorization semantics. Add focused regression coverage in `packages/clawhub/src/cli/commands/sync.test.ts` where mocked `cmdPublish` rejects for one selected skill and a later selected skill still publishes.
