## Repair Work Prompt

Implement a focused fix for the open `clawhub sync` slug-conflict batch abort. In `packages/clawhub/src/cli/commands/sync.ts`, when a selected skill publish fails with the known backend message `Slug is already taken. Choose a different slug.` optionally followed by `Existing skill: /owner/slug`, record that skill as failed or skipped, continue publishing the remaining selected skills, and print an accurate final summary. Preserve backend uniqueness and ownership behavior; do not relax `convex/skills.ts` owner checks. Do not swallow unrelated publish failures. If any selected skill failed or was skipped, the command must not report a fully successful upload; prefer nonzero/failing exit after attempting the remaining selected skills. Add regression coverage in `packages/clawhub/src/cli/commands/sync.test.ts` where mocked `cmdPublish` rejects for one selected skill with the slug-taken message, a later selected skill is still published, and the final output/error reflects partial failure. Update docs or specs only if the user-visible sync failure semantics change.
