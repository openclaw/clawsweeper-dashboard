## Summary

Keep open: current main still has a source-level path where one slug-conflict publish during `clawhub sync` aborts the remaining selected skills, and ClawHub cleanup policy keeps issues open rather than auto-closing them.

## What This Changes

Handle `clawhub sync` so a slug conflict for one selected skill does not prevent later selected skills from updating.

## Best Possible Solution

Add CLI-level per-skill conflict handling around sync publishing while preserving backend slug uniqueness and owner-scoped publishing rules.

## Reproduction Assessment

Yes, via source inspection. `/api/v1/resolve` can classify a slug-mismatched skill as `update`, `cmdSync` then awaits `cmdPublish`, and `cmdPublish` rethrows the backend slug-taken error.

## Solution Assessment

Yes. The narrow maintainable fix is in the CLI sync publish loop; relaxing backend slug uniqueness or ownership checks is not needed for the reported batch-abort bug.
