## What This Changes

The issue requests making skill installation scan only the newly installed skill instead of rescanning all installed skills.

## Best Possible Solution

Have a maintainer first confirm the slow workflow and version; if it is `sync`, design incremental caching or fingerprints that preserve full-root telemetry/removal detection and keep an explicit full-rescan path.
