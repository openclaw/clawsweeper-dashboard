## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main does not match the reported install-time full scan: `clawhub install` installs one slug without invoking the local scanner, while the full-scan behavior is in `clawhub sync` and is coupled to documented telemetry/removal semantics.

## What This Changes

The issue asks ClawHub to avoid rescanning all installed skills during a new skill installation and scan only the newly installed skill instead.

## Best Possible Solution

Keep the issue open for maintainer triage: reproduce the exact slow workflow first, then consider cached fingerprints or incremental sync parsing only if full-root telemetry, removal detection, and an explicit full-rescan path are preserved.

## Reproduction Assessment

Unclear: source inspection verifies that `clawhub sync` scans and hashes all discovered skills, but current `clawhub install` does not call that scanner. The issue lacks the exact command, CLI version, skill count, host path, and timing/profile data needed for a high-confidence install-time reproduction.

## Solution Assessment

Unclear: scanning only a newly installed skill would fit a true install-time rescan, but that is not the current install path. If the real target is `sync`, the maintainable solution needs to preserve documented full-snapshot telemetry and removal detection rather than narrowing the scan blindly.
