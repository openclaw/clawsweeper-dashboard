## Summary

Keep open. The ClawHub cleanup policy keeps issues open, and current main does not expose a narrow install-path fix: `clawhub install` already installs a single slug without invoking the local scanner, while the full-scan behavior is in `clawhub sync` and is tied to documented full-snapshot telemetry and removal detection.

## What This Changes

The issue asks to optimize skill installation so adding one new skill scans only that new skill instead of rescanning every installed skill, while retaining a manual full scan if needed.

## Best Possible Solution

Keep the issue open for maintainer triage. Reproduce the exact slow workflow first; if the target is `sync`, design cached fingerprints or incremental parsing that preserves full-root telemetry/removal semantics and provides an explicit full-rescan path.
