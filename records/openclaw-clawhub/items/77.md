## Summary

Keep open. Current main and the latest release still only require skill display names to be non-empty across the schema, web publish flow, GitHub import, and CLI/API publish paths, so the requested title limit is not implemented and still needs a maintainer policy decision.

## What This Changes

The issue asks ClawHub to keep skill titles/display names short by adding a length limit.

## Best Possible Solution

Define one shared public display-name maximum, decide how unchanged legacy long names are handled, then enforce that policy consistently in backend publish validation, web import/publish UI, CLI/API schemas, docs, and tests.

## Reproduction Assessment

Yes at source level: current main has a clear publish path where any non-empty displayName passes through schema, web publish, GitHub import, and CLI/API request handling without a length check. I did not run a browser flow because the source path already shows the missing policy.

## Solution Assessment

Unclear until maintainers choose the maximum length and legacy handling. The maintainable path is a shared backend rule with matching UI/CLI/docs/tests, not a presentation-only clamp.
