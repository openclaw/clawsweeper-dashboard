## Summary

Keep open. Current main still accepts any non-empty skill display name across the shared backend publish path, GitHub import, web publish UI, and CLI payload path, so the requested title/display-name limit is not implemented and still needs a maintainer decision on the maximum length and legacy handling.

## What This Changes

The issue asks ClawHub to limit skill titles so published skill display names stay short and concise.

## Best Possible Solution

Define the public skill display-name policy, then enforce the chosen limit consistently in shared backend validation, web publish/import UI, CLI/API schemas, docs/specs, and regression tests with explicit treatment for existing long names.

## Reproduction Assessment

Not applicable: this is a product-policy request rather than a broken documented behavior report. Source inspection shows current publish paths accept any non-empty displayName.

## Solution Assessment

Unclear until maintainers choose the maximum length and legacy behavior. The maintainable implementation is shared backend/API validation plus UI feedback, not a UI-only cap.
