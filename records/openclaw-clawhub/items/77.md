## Summary

Keep open. Current main and the latest v0.21.0 release still only require skill display names to be non-empty across storage, publish, import, web form, and CLI/API payload paths, so the requested title limit is not implemented and still needs a maintainer policy decision.

## What This Changes

The issue requests a maximum-length policy for skill titles/display names so catalog titles stay short and concise.

## Best Possible Solution

Define one public skill display-name maximum and legacy policy, then enforce it consistently in shared backend validation, web publish/import UI, CLI/API schemas, docs, and regression tests.

## Reproduction Assessment

Yes, at source level: any non-empty displayName passes through the current schema, shared publish helper, web publish form, GitHub import, and CLI/API request schema without a length check. I did not run a browser flow because the write-path source already shows the missing policy.

## Solution Assessment

Unclear until maintainers choose the maximum length and legacy handling. The maintainable solution is a shared backend rule with matching UI/CLI/docs/tests, not only a presentation clamp.
