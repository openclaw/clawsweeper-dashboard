## Summary

Keep open. Current main still has no skill title/display-name length limit, and this ClawHub issue needs maintainer product policy rather than auto-close cleanup.

## What This Changes

Request a maximum length policy and enforcement for skill titles/display names so published listings stay concise.

## Best Possible Solution

Define the public display-name policy first, then enforce the chosen limit in shared backend publish validation, web publish/import UI, GitHub import, CLI/API schemas, docs, and focused tests.

## Reproduction Assessment

Yes. Source inspection on current main shows displayName is accepted as a plain string and only checked for non-empty values across publish, API, UI, and GitHub import paths.

## Solution Assessment

Unclear until maintainers choose the maximum length and legacy behavior. The narrow maintainable solution is a shared backend validation rule mirrored by UI/API/CLI/import affordances, not a UI-only cap.
