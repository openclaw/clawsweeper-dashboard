## Label Justifications

- `P3`: This is low-urgency marketplace readability and metadata ergonomics work rather than broken core behavior.
- `impact:other`: Unbounded skill titles can materially degrade catalog readability, but the impact does not match a more specific owned category.

## Summary

Current main only mitigates long titles in cards with a two-line clamp; it still accepts any non-empty trimmed display name, so the requested catalog policy is not implemented and needs a maintainer-selected limit.

## What This Changes

Add a short, concise maximum length policy for skill display titles.

## Best Possible Solution

Choose and document one display-name maximum, enforce it in shared server-side publish/import validation with matching UI feedback, and preserve overflow-safe rendering for legacy records.

## Reproduction Assessment

Yes from current source: submit a non-empty display name of arbitrary length through a publish or import path; backend validation trims it but does not cap its length, while cards only hide overflow visually.

## Solution Assessment

No existing implementation fully solves it; the narrow maintainable direction is shared validation after maintainers decide the limit and legacy-update behavior, not a card-only CSS change.
