## Summary

Keep open. Current main still accepts skill display names as unconstrained strings across the schema, shared publish validation, web publish, GitHub import, and CLI/API surfaces; the remaining decision is the product policy for the exact limit and legacy handling.

## What This Changes

Request to enforce a short, concise maximum length for skill titles/display names.

## Best Possible Solution

Define a documented skill display-name policy, then enforce the chosen limit in shared backend validation and mirror it in web, CLI/API, GitHub import, docs, and tests.

## Reproduction Assessment

Yes for the current behavior. Source inspection shows displayName is accepted as an unconstrained string and only checked for non-empty values across publish paths; no runtime publish was run because this review is read-only.

## Solution Assessment

Unclear until maintainers choose the maximum length and legacy behavior. The maintainable implementation should put the canonical check in shared backend publish validation, with UI/API/CLI/import paths matching that policy.
