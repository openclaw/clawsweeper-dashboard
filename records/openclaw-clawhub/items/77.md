## Summary

Current main still permits arbitrarily long non-empty skill display names across the publish and GitHub-import paths, so the request remains necessary; implementation should wait for maintainers to choose the numeric limit and legacy-record policy.

## What This Changes

Define and enforce a maximum length for skill titles so catalog names remain concise.

## Best Possible Solution

Adopt a documented display-name limit and explicit grandfathering policy, then enforce it through shared backend validation used by every skill-ingest path, with UI guidance and focused unit plus real Convex runtime coverage.

## Reproduction Assessment

Yes, at source level: current main trims skill display names and rejects only empty values in both the shared publish and GitHub-import paths, with no maximum-length check.

## Solution Assessment

Unclear until maintainers select the maximum and existing-record policy; once decided, centralized backend validation with matching UI feedback is the narrowest maintainable solution.
