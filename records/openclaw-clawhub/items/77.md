## Summary

Keep open. Current main still has no skill title/display-name length limit across the shared publish helper, GitHub import, web publish form, and CLI/API request schema, so the requested policy is not implemented and still needs maintainer judgment on the exact limit and legacy handling.

## What This Changes

The issue requests a maximum length or concision policy for skill titles/display names.

## Best Possible Solution

Define the public skill display-name policy first, then enforce the chosen limit consistently in shared backend validation, web publish/import UI, CLI/API schemas, docs or specs, and regression tests with explicit treatment for already-published long names.

## Reproduction Assessment

Not applicable: this is a product-policy request rather than a broken documented behavior report. Source inspection gives high confidence that current publish paths accept any non-empty displayName.

## Solution Assessment

Unclear until maintainers choose the maximum length and legacy behavior. The maintainable implementation is shared backend/API validation plus UI feedback across publish/import paths, not a UI-only cap.
