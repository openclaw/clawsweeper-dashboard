## Summary

Keep open. Current main and v0.23.0 still lack a skill display-name/title length policy, so the requested behavior remains unimplemented and needs a maintainer decision on the limit and legacy handling.

## What This Changes

The issue requests a maximum length or concision policy for public skill titles/display names.

## Best Possible Solution

Define one shared skill display-name policy and enforce it across backend publish validation, web publish/import UI, GitHub import, CLI/API schemas, docs, and tests with explicit handling for existing long names.

## Reproduction Assessment

Yes, at source level: current main accepts any non-empty displayName in the schema, shared publish helper, web publish form, GitHub import path, and CLI/API schema. I did not run a live publish because the validators and entry points already show the missing policy.

## Solution Assessment

Unclear until maintainers choose the maximum length and legacy handling. The maintainable path is a shared backend rule with UI/API/CLI/docs/test alignment, not a presentation-only clamp.
