## Summary

Keep open. Current main and the latest v0.21.0 release still only require skill display names to be non-empty across the backend, web publish form, GitHub import/sync, CLI publish, and API schema surfaces, so the requested title-length policy is not implemented and still needs maintainer product direction.

## What This Changes

The issue requests a product-enforced short title/display-name limit for published skills.

## Best Possible Solution

Maintainers should choose the public display-name maximum and legacy behavior, then enforce it consistently across shared backend validation, web publish, GitHub import/sync, CLI/API schemas, docs, and tests.

## Reproduction Assessment

Yes, at source level: current main and v0.21.0 store and publish skill display names as non-empty strings without a maximum-length rule. No live publish was run because the backend, import, CLI, API schema, and form validators are enough to show the missing policy.

## Solution Assessment

Unclear until maintainers choose the actual limit and legacy handling. A shared backend rule with matching UI/API/CLI/docs coverage is the maintainable shape; a presentation-only clamp would not solve publishing through other paths.
