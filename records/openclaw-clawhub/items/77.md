## Summary

Keep open. Current main and the latest v0.23.0 release still accept any non-empty skill display name, so the requested skill-title length policy remains missing and needs maintainer product direction.

## What This Changes

The issue requests a maximum length policy for skill titles so public skill display names stay short and concise.

## Best Possible Solution

Maintainers should define the public skill display-name policy first, then enforce one shared backend rule with matching web publish/import UI, CLI/API schema/docs, tests, and explicit handling for existing long names.

## Reproduction Assessment

Yes, at source level: current main accepts any non-empty displayName in the shared schema and publish paths. I did not run a live publish because the validators and entry points already show the missing policy.

## Solution Assessment

Unclear until maintainers choose the maximum length and legacy behavior. The maintainable solution is a shared backend rule with UI/API/CLI/docs/test alignment, not a presentation-only clamp.
