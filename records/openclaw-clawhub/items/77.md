## Summary

Keep open. Current main still does not enforce a skill title/display-name length limit, and the ClawHub cleanup policy for this repository keeps issues open unless maintainers handle them directly.

## What This Changes

The issue asks ClawHub to limit skill titles/display names so they stay short and concise.

## Best Possible Solution

Define a public skill display-name policy, then enforce the chosen limit consistently in shared backend validation, web publish/import UI, GitHub import, CLI/API schemas, docs, and focused tests.

## Reproduction Assessment

Yes. Source inspection shows any non-empty displayName can flow through the web form, Convex action, GitHub import, and CLI/API schema because current validation only trims and checks presence.

## Solution Assessment

Unclear. A length limit is the right direction, but the best implementation depends on maintainer decisions for the maximum length and existing long-name handling.
