## Summary

Keep open. Current main still has no skill title/display-name length limit across the main publish/import surfaces, and the missing piece is a maintainer product decision on the maximum length and legacy handling.

## What This Changes

The issue requests a maximum length policy for skill titles/display names so published skill listings stay concise.

## Best Possible Solution

Define the public skill display-name policy, then enforce the chosen limit consistently in shared publish validation, web publish/import UI, CLI/API schemas, docs/specs, and tests with explicit treatment for existing long names.

## Reproduction Assessment

Not applicable: this is a product-policy request rather than a broken documented behavior report. Source inspection shows current publish paths accept any non-empty displayName.

## Solution Assessment

Unclear until maintainers choose the public maximum and legacy behavior. The maintainable path is shared backend/API validation plus UI feedback, not a UI-only cap.
