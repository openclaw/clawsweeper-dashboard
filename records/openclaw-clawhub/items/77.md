## Summary

Keep open. Current main and the latest v0.23.0 release still accept any non-empty skill display name, so the requested title/display-name length limit remains missing and needs a maintainer decision on the exact limit and legacy handling.

## What This Changes

The issue requests a maximum length limit for skill titles/display names so published skill titles stay short and concise.

## Best Possible Solution

Define the public skill display-name policy first, then enforce the chosen limit consistently in backend publish validation, web publish/import UI, GitHub import, CLI/API schemas, docs, and tests with explicit handling for existing long names.

## Reproduction Assessment

Yes, at source level: current main accepts any non-empty displayName in the shared publish helper, schema, web publish form, GitHub import path, and CLI/API schema. I did not run a live publish because the validators and entry points already show the missing policy.

## Solution Assessment

Unclear until maintainers choose the maximum length and legacy handling. The maintainable solution is a shared backend rule with UI/API/CLI/docs/test alignment, not a presentation-only clamp.
