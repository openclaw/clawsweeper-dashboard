## Summary

Keep open. The request is still valid on current main, but ClawHub cleanup policy keeps issues open unless maintainers handle them directly, and this one needs a product decision on the exact title length and legacy handling.

## What This Changes

The issue requests a maximum length policy for skill titles/display names so catalog entries stay short and concise.

## Best Possible Solution

Define the public skill display-name policy first, then enforce the chosen limit in shared backend validation with matching UI, GitHub import, CLI/API schema, docs, and tests.

## Reproduction Assessment

Yes. Static current-main inspection shows every publish path accepts displayName as a plain string and only rejects empty values; no runtime write is needed to confirm the missing cap.

## Solution Assessment

Unclear until maintainers choose the maximum length and legacy behavior. The narrow maintainable fix is a shared backend check mirrored by clients and docs, not a web-only input limit.
