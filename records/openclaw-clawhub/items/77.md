## Summary

Keep open. Current main and the latest v0.17.0 release still accept any non-empty skill display name through the shared publish path, GitHub import, web UI, and CLI/API schema, so the requested title-length policy is not implemented and still needs a maintainer decision.

## What This Changes

The issue requests a maximum title/display-name length for skills so published skill titles stay short and concise.

## Best Possible Solution

Define the public skill display-name policy, then enforce the chosen limit consistently in shared backend/API validation, web publish/import UI, CLI schemas, docs/specs, and regression tests with explicit treatment for existing long names.

## Reproduction Assessment

Not applicable: this is a product-policy request rather than a broken documented behavior report. Source inspection gives high confidence that current publish paths accept any non-empty displayName.

## Solution Assessment

Unclear until maintainers choose the maximum length and legacy behavior. The maintainable implementation is shared backend/API validation plus UI feedback across publish/import paths, not a UI-only cap.
