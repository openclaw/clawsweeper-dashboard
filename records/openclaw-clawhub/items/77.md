## What This Changes

## Best Possible Solution

Define the public skill display-name policy, then enforce it in shared backend publish validation and mirror it across web publish/import UI, CLI/API schemas, docs, and tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this requests a new product constraint. Source inspection shows current main still accepts any trimmed non-empty displayName across the central publish paths.

## Solution Assessment
