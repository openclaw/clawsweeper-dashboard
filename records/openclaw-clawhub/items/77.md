## What This Changes

Add a maximum-length policy for skill titles/display names so ClawHub listings stay short and concise.

## Best Possible Solution

Define the public skill display-name policy first, then enforce the chosen limit in shared backend publish validation with mirrored web, import, CLI/API, docs, and regression tests.

## Solution Assessment

Unclear until maintainers choose the maximum length and legacy behavior. The maintainable implementation would be a canonical backend rule with matching UI/API/CLI/import validation, not a UI-only maxlength.
