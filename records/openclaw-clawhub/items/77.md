## Summary

Keep open. Current main and the latest v0.21.0 release still only require skill display names to be non-empty across the schema, backend publish helper, web publish form, CLI/API payloads, and GitHub import/sync paths, so the requested title-length policy is not implemented yet.

## What This Changes

The issue requests a short, concise title/display-name limit for published skills.

## Best Possible Solution

Maintainers should choose the public display-name maximum and legacy behavior, then enforce one shared backend rule with matching UI, CLI/API schemas, docs, and tests.

## Reproduction Assessment

Yes, at source level: current main accepts any non-empty displayName in the central publish paths and no shared max-length rule exists. I did not run a live publish because the validators, schemas, and request paths already show the missing policy.

## Solution Assessment

Unclear until maintainers choose the maximum length and legacy handling. The maintainable path is a shared backend rule with UI/API/CLI/docs/test alignment, not a presentation-only clamp.
