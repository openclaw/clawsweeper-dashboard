## Summary

Keep open. Current main still does not enforce a skill title/display-name length limit across the publish paths, and the remaining decision is product policy: the maximum length and handling for existing or imported long names are not defined.

## What This Changes

The issue asks ClawHub to keep skill titles/display names short and concise by adding a title limit.

## Best Possible Solution

Define one public skill display-name policy, then enforce it through a shared validator used by backend publish, web publish, CLI/API payloads, GitHub import/sync, docs, and regression tests, with explicit legacy handling.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a product/UX policy request rather than a failing existing contract. Source inspection does show current main accepts any non-empty display name through the central publish paths.

## Solution Assessment

No, current main is not the best solution because it relies on presence checks and partial UI clamping instead of a shared title policy. The narrow maintainable path is to choose the limit first, then enforce it consistently across all publish/import surfaces.
