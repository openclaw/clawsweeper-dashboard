## Summary

Keep open. Current main still does not enforce a skill title/display-name length limit, and this ClawHub cleanup pass only auto-closes PRs that are certainly implemented on main. The issue is valid, but maintainers still need to choose the maximum title length and how to handle existing long display names before a focused implementation is safe.

## What This Changes

The issue requests a policy and enforcement limit so skill titles/display names stay short and concise.

## Best Possible Solution

Define a public skill display-name policy, then enforce the chosen limit consistently in shared backend publish validation, the web publish/import UI, GitHub import, CLI/API schemas, docs, and focused tests, with explicit handling for already-published long names.
