## Summary

Keep open. Current main and v0.23.0 still accept any non-empty skill display name across backend, web publish/import, GitHub import, and CLI/API paths, so the requested length policy remains missing and needs a maintainer choice.

## What This Changes

Request a maximum length policy for public skill titles/display names so catalog entries stay short and concise.

## Best Possible Solution

Define the public skill display-name limit and legacy policy, then enforce it consistently in shared backend validation, web publish/import UI, GitHub import, CLI/API schemas, docs, and focused tests.

## Reproduction Assessment

Yes, at source level: current main accepts any non-empty skill displayName through the shared publish helper and related publish entry points. I did not run a live publish because the validators and entry points already show the missing maximum-length policy.

## Solution Assessment

Unclear until maintainers choose the maximum length and legacy handling. A backend-only or UI-only clamp would be too narrow because CLI/API and GitHub import paths also accept displayName.
