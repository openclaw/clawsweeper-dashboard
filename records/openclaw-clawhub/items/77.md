Keep open. Current main and the v0.22.0 release still accept any non-empty skill display name, so the requested title-length policy is not implemented and still needs maintainer product direction on the exact limit and legacy behavior.

## What This Changes

The issue requests a policy or validation limit so skill titles stay short and concise.

## Best Possible Solution

Maintainers should define the public skill display-name policy first, then enforce the chosen limit consistently across backend publish validation, web publish/import UI, CLI/API schemas, docs, and tests with explicit handling for existing long names.

## Solution Assessment
