## Summary

Keep open: current main and the latest release still have the static `script-src 'unsafe-inline'` CSP and inline root theme bootstrap, while this member-authored draft is the canonical open CSP hardening split that still needs maintainer ownership before merge.

## What This Changes

Adds a TanStack Start server entry for per-request nonce CSP headers, moves the root theme bootstrap to `ScriptOnce`, removes the static Vercel CSP, adds CSP/theme-bootstrap tests, and updates the auth identity spec.

## Best Possible Solution

Land the dynamic nonce CSP after maintainer review confirms production-like HTML route coverage and accepts the temporary `unsafe-eval` allowance, leaving eval removal as a separate hardening follow-up.

## Reproduction Assessment

Yes for the hardening gap: current main and v0.22.0 both show a static `script-src 'unsafe-inline'` CSP and an inline root theme bootstrap. I did not run a browser repro in this read-only review, but the source path is direct.

## Solution Assessment

Yes, this is the narrow maintainable direction: nonce the TanStack SSR scripts, move the bootstrap under nonce-managed rendering, and keep the form-action invariant in the security spec. The safer merge path is maintainer verification of production-like route coverage plus explicit acceptance of the temporary `unsafe-eval` allowance.
