## Summary

Keep open: this ClawHub issue has a protected security label, and current main still has a source-visible path where the publish UI can select an org owner but the backend rejects an existing skill whose stored owner publisher differs from the selected target.

## What This Changes

The issue requests support for moving an existing skill from a personal publisher to an organization publisher through the web publish flow.

## Best Possible Solution

Add an explicit org-aware ownership transfer path, or a deliberate publish-time transfer path, that validates publisher membership, preserves the existing skill identity and related data, records audit history, updates digests and aliases, and defines canonical redirect behavior.

## Reproduction Assessment

Yes. Source inspection shows the selected org owner is submitted to the publish action, then `insertVersion` rejects the existing skill when its stored `ownerPublisherId` differs from the selected target.

## Solution Assessment

Unclear. Publish-time transfer could solve the reported workflow, but an org-aware transfer endpoint/workflow may be safer because ownership moves require authorization, audit, alias, and URL policy decisions.
