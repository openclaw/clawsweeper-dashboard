## Summary

Keep open: this is a protected security-labeled policy issue, and current main plus the live affected skill specs still show only partial coverage for destructive or irreversible no-confirm command patterns.

## What This Changes

The issue asks ClawHub to require chat-level user confirmation before agents run destructive or irreversible skill commands with `--yes`.

## Best Possible Solution

Define the replacement approval policy in ClawHub security/authoring guidance, decide how ClawScan and deterministic moderation should enforce destructive no-confirm commands beyond delete-only cases, and coordinate updates or moderation for affected published skills.

## Reproduction Assessment

Yes, as a source and policy reproduction: current public SKILL.md files still document destructive or irreversible `--yes` examples, and current main has only partial scanner/prompt guidance rather than a general replacement-confirmation rule. This is not a local runtime failure path.

## Solution Assessment

Unclear: the requested guardrail is directionally sound, but maintainers should decide whether the durable fix is docs-only, scanner enforcement, moderation workflow, per-skill updates, or a coordinated combination.
