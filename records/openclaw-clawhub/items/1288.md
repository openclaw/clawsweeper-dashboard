## Label Justifications

- `P2`: The original defects blocked or corrupted a bounded plugin publishing workflow but did not affect the entire service.

## Summary

Current main resolves the reported user problems through the streamlined plugin publishing flow: folder selection uses a dedicated directory input with regression coverage, while package changelogs are manual on update flows and the unsafe package auto-preview path no longer exists.

## What This Changes

Fix plugin folder selection, prevent asynchronous changelog previews from overwriting manual edits, and avoid generating package changelogs from skill history.

## Best Possible Solution

Keep the current dedicated folder chooser and explicit manual changelog field for package updates, without restoring the obsolete auto-preview API.

## Reproduction Assessment

Yes. The original directory-attribute and stale-preview mechanisms are concrete in historical source, and current main provides direct source and regression-test checks for their replacement.

## Solution Assessment

Yes. The merged redesign fixes folder selection and removes both changelog failure modes at their root by using an explicit manual update changelog instead of a package auto-preview backed by the wrong history.
