## What This Changes

The issue asks to keep plugin folder selection working, prevent async auto-changelog previews from overwriting manual edits, and generate plugin/package changelog previews from package release history.

Keep the current folder-picker implementation, then have maintainers choose either documented manual-only plugin update changelogs or a scoped packageReleases-based preview with stale-result and manual-edit guards.
