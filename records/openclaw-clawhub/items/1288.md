## What This Changes

The issue asks to fix plugin publishing so folder uploads work, generated package changelogs do not overwrite manual edits, and package changelog previews use package release history instead of skill history.

## Best Possible Solution

Keep the current folder-picker implementation, then either document plugin package changelogs as manual-only or add a scoped packageReleases-based preview with stale-result and manual-edit guards on the active /plugins/publish route.

## Reproduction Assessment

Yes, at source level. Current main shows folder selection fixed, but the active plugin publish route has no package changelog preview action or async guard, and the only changelog preview helper still reads skill history.

## Solution Assessment

Unclear. The narrow maintainable path is not to revive the closed stale branch wholesale, but to make a maintainer choice between manual-only package changelogs and a scoped package-history preview on the current route.
