## Summary

Current main and the v0.20.2 release resolve the reported publishing problems: folder selection uses a directory-enabled input, plugin changelogs remain manual form state, and the active package publish backend has no skill-history changelog-preview path. The closed proposed PR is therefore no longer needed as the implementation vehicle.

## What This Changes

Fix plugin folder selection, prevent generated changelog responses from overwriting manual edits, and use package release history for plugin changelog previews.

## Best Possible Solution

Keep the released shared directory picker and manual plugin changelog workflow; track any future request for editable published-version changelogs separately in https://github.com/openclaw/clawhub/issues/1946.

## Reproduction Assessment

No. On current main and v0.20.2, folder selection is directory-enabled and plugin publishing has no asynchronous changelog-preview request capable of overwriting edits or consulting skill history.

## Solution Assessment

Yes. The current design uses a tested shared folder chooser and a manual changelog field, removing both the stale-response race and the incorrect cross-domain history dependency rather than maintaining a second preview implementation.
