## Summary

Current main fixes folder selection and avoids the overwrite race by making plugin changelogs manual, but it still does not generate previews from package history. The linked implementation was closed unmerged, so this issue remains necessary until maintainers explicitly accept manual-only behavior or restore a narrower package-specific preview.

## What This Changes

Ensure plugin folder uploads work, prevent asynchronous changelog generation from overwriting user edits, and generate package changelogs from previous package releases rather than skill history.

## Best Possible Solution

Define manual-only versus generated package changelogs as an explicit product contract; if generation is desired, add an authenticated package-specific preview using prior package releases while ensuring stale or failed requests can never replace user-entered text.

## Reproduction Assessment

Yes, at source level: current main clearly shows that the active plugin page has no package-history preview action, while the historical skill helper is package-inappropriate. The folder-picker and stale-overwrite failures are not current-main reproductions because those paths have been fixed or removed.

## Solution Assessment

Unclear. The broad closed PR is not the best landing path because it is outdated and includes unrelated UI work, but a narrow package-specific preview may be appropriate once maintainers decide whether manual-only changelogs are intentional.
