The linked broad fix PR was closed unmerged, but later mainline publishing work independently removed all three observable failure modes.

## What This Changes

Fix plugin folder selection, prevent asynchronous changelog previews from overwriting manual edits, and derive package changelogs from package release history.

## Best Possible Solution

Keep the current tested folder picker and manual package changelog workflow; treat package-aware automatic changelog generation as a separate product feature if it is desired later.

## Reproduction Assessment

No. On current main, folder selection uses an explicitly configured directory input with regression coverage, and package changelogs have no asynchronous preview or skill-history call path.

## Solution Assessment

Yes. The replacement workflow removes both changelog failure modes while retaining direct user control, and it fixes folder selection at the shared package source component rather than reviving the obsolete page.
