## What This Changes

The issue requests reliable plugin-folder selection, protection of manual changelog edits, and package-aware changelog generation.

## Best Possible Solution

Keep the consolidated plugin publisher as the single supported flow; any future request for automatic package changelog suggestions should be evaluated separately as a new capability with package-history semantics and overwrite protection.

## Solution Assessment

Yes; consolidating plugin publishing into one shared flow removes the competing implementation and eliminates both stale preview overwrites and the use of skill history for package previews.
