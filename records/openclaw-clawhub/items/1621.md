## Summary

Keep open. This is a corroborated ClawHub GitHub OAuth sign-in failure, current production still returns HTTP 500 from the GitHub auth-start route, and the ClawHub cleanup policy does not auto-close issues.

## What This Changes

The issue reports first-time GitHub sign-in on ClawHub failing after GitHub authorization, leaving the user unauthenticated and unable to publish.

## Best Possible Solution

Restore production GitHub OAuth for active accounts, keep deleted/banned-account rejection intact, and add focused diagnostics or regression coverage plus redacted production smoke proof.

## Reproduction Assessment

Yes for the central current failure: the site root returns HTTP 200 while both canonical and direct Convex GitHub auth-start endpoints return HTTP 500 instead of redirecting to GitHub OAuth. The exact post-GitHub callback/no-session path was not replayed because production currently fails earlier.

## Solution Assessment

Yes for keeping this open and routing it to maintainer/operator auth triage. A speculative source-only repair is not the narrowest maintainable solution until production logs, OAuth app settings, deployed env values, cookies, and affected account state identify the failing branch.
