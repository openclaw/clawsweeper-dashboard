## Summary

Keep open. This is a corroborated GitHub OAuth/sign-in failure for first-time publishing, the live auth-start route still returns HTTP 500, and ClawHub cleanup policy does not auto-close issues.

## What This Changes

Users report first-time GitHub sign-in returning to ClawHub without a usable session or showing a generic sign-in failure while trying to publish.

## Best Possible Solution

Restore GitHub OAuth for active accounts in production, preserve deleted/banned-account rejection, and add focused auth diagnostics or regression coverage plus redacted smoke proof.

## Reproduction Assessment

Yes for the central sign-in-start failure: live checks returned HTTP 500 from both ClawHub and direct Convex GitHub auth-start endpoints while the site root returned HTTP 200. The exact post-GitHub no-session callback was not replayed because production currently fails before the OAuth redirect.

## Solution Assessment

Yes for keeping this open and routing it to maintainer/operator auth triage. A speculative source-only repair is not the narrowest maintainable solution until production logs, OAuth app settings, env values, cookies, and affected account state identify the failing branch.
