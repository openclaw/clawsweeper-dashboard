## Summary

Keep open. ClawHub cleanup policy keeps issues open, and this still appears to be a live GitHub OAuth/sign-in failure: the current production-tagged main serves the site root but returns HTTP 500 for both canonical and direct Convex GitHub sign-in endpoints, with related OAuth/session reports still open.

## What This Changes

The issue asks ClawHub to let a first-time publisher complete GitHub sign-in after GitHub authorization instead of returning to a logged-out or failed state.

## Best Possible Solution

Keep this in the OAuth/session investigation, restore production GitHub OAuth for active accounts, preserve explicit blocked-account outcomes, and add focused auth smoke or regression coverage once the failing layer is identified.
