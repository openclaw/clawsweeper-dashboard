## Summary

Keep open. Current main still leaves the reported behavior unresolved: the skill README renderer passes Markdown links through without a relative-link resolver, the owner/slug route has no packaged-file child route, and the existing raw file API is not wired into README link rendering. Related #1681 and draft PR #1858 remain the active path for this bug.

## What This Changes

The issue requests that relative links in a skill's SKILL.md content, such as references/system-design.md, open packaged files from the skill detail page instead of navigating to a 404 owner/slug URL.

## Best Possible Solution

Land or revise the focused relative-link fix so safe SKILL.md links to packaged files resolve inside the same skill namespace, preferably through the existing file API or an approved viewer, with regression coverage for references, anchors, external schemes, and traversal rejection.
