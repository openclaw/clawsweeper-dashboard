## Best Possible Solution

Land the owned-repo picker with one explicit supported contract: either preserve third-party public URL imports or have maintainers approve and document the owned-only boundary, then validate that auth boundary in a real Convex runtime.

## Reproduction Assessment

## Solution Assessment

Not yet in merge-ready form. The owned-public picker is a maintainable direction, but the compatibility contract needs maintainer approval or a preserved public URL mode plus real Convex validation of the auth boundary.
