## Summary

This PR remains useful because current main has only the existing URL-based importer, but the branch is not merge-ready: its three direct URL actions accept third-party public repositories despite the new owned-only security promise, and the head currently conflicts with main.

The PR adds discovery, review, and batch publishing of skills from public GitHub repositories owned by the signed-in account, with backend validation, redesigned import UI, navigation, tests, docs, and specs.

## Best Possible Solution

Choose one permanent URL ownership contract, then enforce it consistently across every server action, the picker, docs, specs, and tests on a conflict-free head; if owned-only is selected, use immutable GitHub owner ids and add repo/tree/blob rejection coverage.

## Solution Assessment
