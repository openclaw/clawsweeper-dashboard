## Summary

Keep open: current production still reproduces the reported artifact digest split, and current main still lets an extra `/versions/{version}/tarball` subpath fall through to version metadata JSON instead of exact artifact bytes or a clear unsupported-route error.

## What This Changes

The issue asks ClawHub to stop serving mismatched package artifact bytes through HTTP install paths and to clarify or fix source commit provenance on plugin listings.

## Best Possible Solution

Make package artifact resolution unambiguous and digest-safe for install clients, then enforce or document source provenance rules for published plugin metadata.

## Reproduction Assessment

Yes. Current production at the current main deployment returns different byte streams and SHA-256 digests for the tarball-looking API path versus the supported artifact download and npm mirror paths, and source inspection explains the fallthrough.

## Solution Assessment

Unclear. The narrow code surface is identifiable, but maintainers should first decide whether the unsupported tarball-looking route should alias to exact ClawPack bytes or fail clearly.
