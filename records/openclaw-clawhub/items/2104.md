## Summary

Keep open: the hosted endpoint still reproduces the digest/byte mismatch, and current main still has the version-subpath fallthrough that lets an artifact-looking URL return version metadata JSON.

## What This Changes

The issue asks for package install/download HTTP routes to serve the same verified ClawPack artifact bytes and correct provenance metadata as the CLI artifact resolver path.

## Best Possible Solution

Make package artifact resolution unambiguous, keep digest-checked ClawPack tarball bytes as the install source of truth, add regression coverage for unsupported version subpaths, and correct or clarify source commit provenance.

## Reproduction Assessment

Yes. Hosted clawhub.ai endpoint checks reproduce different byte streams and digests for the tarball-looking path, explicit ClawPack artifact route, npm mirror tarball, and legacy ZIP route, and current-main source inspection explains the fallthrough.

## Solution Assessment

Unclear until maintainers choose the compatibility contract. The narrow maintainable fix is either to make unsupported version subpaths fail clearly or intentionally route compatible tarball requests to the exact verified ClawPack artifact, with regression tests for the chosen behavior.
