## Summary

Keep open: the mismatch still reproduces on production and current main still has the permissive version-subpath route, so this needs maintainer follow-up rather than cleanup.

## What This Changes

The issue asks ClawHub to stop serving digest-mismatched bytes for an artifact-looking package version tarball URL and to address incorrect source-commit provenance on the affected listing.

## Best Possible Solution

Make package artifact routes digest-safe by rejecting, redirecting, or explicitly supporting unknown version artifact subpaths, then separately correct or document the affected release's source-commit provenance.

## Reproduction Assessment

Yes. The reported production URL still returns JSON with a different digest from the supported artifact download, and current main contains the same permissive version-subpath branch that explains the behavior.

## Solution Assessment

Unclear. Rejecting unknown subpaths is the narrowest defensive fix, but maintainers should choose the public route contract before deciding between 404, redirect, or a supported /tarball alias.
