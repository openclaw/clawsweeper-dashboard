## Summary

Keep open: the install integrity mismatch still reproduces live, and current main still has the broad package-version route that can return JSON from an artifact-looking subpath. ClawHub cleanup policy keeps issues open for maintainer handling, and this one is supply-chain sensitive.

## What This Changes

The issue asks ClawHub to stop returning non-artifact bytes from package artifact-looking `/versions/{version}/tarball` URLs and to address the reported source-commit provenance mismatch.

## Best Possible Solution

Make package artifact routes digest-safe by rejecting unknown version subpaths or serving exact bytes for a documented alias, then separately correct or clarify source-commit provenance for affected releases.

## Reproduction Assessment

Yes. Live production returns JSON bytes from the reported artifact-looking URL, and current main routes the same extra segment into the broad version JSON branch.

## Solution Assessment

Unclear. Constraining the route is the narrow maintainable fix, but the public contract decision between 404, redirect, or a supported tarball alias needs maintainer judgment.
