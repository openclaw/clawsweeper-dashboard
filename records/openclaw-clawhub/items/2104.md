## Summary

Keep open: the hosted package endpoints still reproduce mismatched artifact bytes and digests, and current main still has a version-subpath fallthrough that can return metadata JSON for an artifact-looking URL. This is install-path and supply-chain relevant, and ClawHub policy keeps issues open.

## What This Changes

Report that a plugin install path fetches package HTTP API bytes that do not match the CLI/manifest ClawPack artifact digest, plus a source-commit provenance mismatch.

## Best Possible Solution

Make package artifact resolution unambiguous and digest-checked, then correct or clearly document source provenance handling for partial publish metadata.

## Reproduction Assessment

Yes. Hosted curl checks on May 9, 2026 reproduce the tarball-looking URL, explicit ClawPack artifact URL, npm mirror URL, and legacy ZIP URL returning different byte streams and digests, and current-main source explains the fallthrough.

## Solution Assessment

Unclear until maintainers choose the compatibility contract. The narrow maintainable fix is either strict unsupported-subpath rejection or an intentional compatibility route to verified ClawPack bytes, with regression tests for the selected behavior.
