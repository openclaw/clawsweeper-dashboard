## Summary

Keep open: the reported digest split still reproduces on current production, and current main still allows an unsupported tarball-looking version subpath to return metadata JSON instead of artifact bytes or a clear route error.

## What This Changes

The issue asks ClawHub to make package install artifact URLs return digest-safe bytes and to avoid misleading source commit provenance for published plugins.

## Best Possible Solution

Make package artifact resolution unambiguous and digest-safe for install clients, and tighten or document source provenance so displayed repository and commit always correspond.

## Reproduction Assessment

Yes. Current production, tagged from the reviewed main SHA, still returns different byte streams and digests for the tarball-looking path versus the supported artifact download and npm mirror paths.

## Solution Assessment

Unclear. The code-local repair is narrow, but maintainers should first decide whether `/versions/{version}/tarball` is a compatibility alias or an unsupported route that must fail clearly.
