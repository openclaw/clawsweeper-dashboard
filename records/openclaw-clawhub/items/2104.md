## Summary

Keep open: current main and live production still reproduce the install-facing digest mismatch, and ClawHub policy keeps issues open unless a maintainer resolves them.

## What This Changes

The issue reports that an install-facing package tarball URL returns bytes that do not match the manifest digest, while the CLI download path returns the expected artifact.

## Best Possible Solution

Define digest-safe package artifact routing with regression coverage for unsupported or aliased artifact subpaths, then separately document or fix the trust semantics for displayed source commits.

## Reproduction Assessment

Yes. Production currently returns the mismatched JSON digest on the tarball-looking path while the supported artifact routes return the expected tarball bytes, and current main explains the same route fallthrough.

## Solution Assessment

Unclear. The code fix is probably narrow, but maintainers should first decide whether the tarball-looking path is invalid or a supported compatibility alias, and whether the sourceCommit report belongs in the same fix.
