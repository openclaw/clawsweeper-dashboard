## Summary

Current main still exposes two artifact contracts for npm-pack releases: the primary `.tgz` route and a generated legacy ZIP route, while the overlapping issue remains open and does not fully cover this report’s unsupported `/tarball` behavior or source-commit concern. This issue remains necessary for maintainer/security judgment rather than conservative cleanup.

## What This Changes

Ensure ClawHub plugin HTTP download routes, integrity metadata, and source metadata consistently identify the artifact actually delivered to installers.

## Best Possible Solution

Define a durable per-artifact contract: npm-pack metadata hashes the exact stored `.tgz`, legacy ZIP responses expose a digest for those exact ZIP bytes or fail clearly, unsupported artifact suffixes return a non-success response, and source commits are omitted or validated rather than populated from an unrelated fallback.

## Reproduction Assessment

Yes, at source level with high confidence. Current main explicitly retains separate npm-pack and legacy ZIP delivery paths, and the June 10, 2026 discussion records the corresponding divergent live responses.

## Solution Assessment

Unclear. The integrity invariant is correct, but maintainers must choose and document the backward-compatible relationship among `sha256hash`, artifact metadata, legacy ZIP downloads, and unsupported routes before implementation.
