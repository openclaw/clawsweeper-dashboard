## Summary

Keep open: the hosted package endpoints still reproduce the reported digest mismatch, and current main still has a version-subpath fallthrough that can return metadata JSON for an artifact-looking URL. ClawHub cleanup policy keeps issues open, and this one is install-path and supply-chain relevant.

## What This Changes

The issue reports that an install-facing package API tarball path serves bytes whose digest differs from the published ClawPack artifact and manifest, plus misleading source commit metadata.

## Best Possible Solution

Make package artifact resolution unambiguous, keep digest-checked ClawPack tarball bytes as the install source of truth, add regression coverage for unsupported version subpaths, and correct or clarify source commit provenance.

## Reproduction Assessment

Yes. Hosted endpoint checks against clawhub.ai reproduce different byte streams and digests for the tarball-looking path, the explicit ClawPack artifact download, the npm mirror tarball, and the legacy ZIP route, and current-main source inspection explains the fallthrough.

## Solution Assessment

Unclear until maintainers choose the compatibility contract. The narrow maintainable fix is either to make unsupported version subpaths fail clearly or to intentionally route compatible tarball requests to the exact verified ClawPack artifact, with regression tests for the chosen behavior.
