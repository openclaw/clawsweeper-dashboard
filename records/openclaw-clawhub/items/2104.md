## Summary

Keep open: the install integrity mismatch still reproduces on the live package API, and current main still has an ambiguous package version subpath that can return metadata JSON where an install client expects artifact bytes.

## What This Changes

The issue requests digest-safe ClawHub package artifact HTTP behavior and correct source provenance for zerogpu-router installs.

## Best Possible Solution

Define one digest-safe artifact contract, implement it consistently across the HTTP API and install clients, and add regression coverage for both route semantics and source provenance inference.

## Reproduction Assessment

Yes. Live requests to the reported package route still return 5,332 bytes of JSON with the mismatched digest, while the explicit artifact download and npm mirror return the 4,678-byte artifact with the expected digest.

## Solution Assessment

Unclear. The code change is likely narrow, but maintainers should first decide whether /versions/{version}/tarball should alias exact artifact bytes or return an explicit unsupported-route error, and how sourceCommit fallback should work.
