## Summary

Keep open: ClawHub cleanup policy keeps issues open, and the reported install-facing artifact mismatch still reproduces against the hosted package endpoints while current main still has the version-subpath fallthrough that explains the archive-looking URL returning metadata JSON.

## What This Changes

The issue reports that the HTTP package tarball path for `zerogpu-router@0.1.0` returns bytes that do not match the CLI-downloaded artifact or manifest integrity.

## Best Possible Solution

Make artifact resolution unambiguous: install clients should either receive digest-checked ClawPack bytes from a documented artifact route or a clear unsupported-route error, with regression coverage and a source-commit provenance clarification if maintainers want that behavior changed.

## Reproduction Assessment

Yes. Hosted package URL checks reproduce the different byte streams and digests, and current-main source inspection shows the tarball-looking version subpath falling through to version metadata JSON.

## Solution Assessment

Unclear until maintainers choose the compatibility contract. The narrow maintainable fix is either to alias tarball-looking version paths to the exact ClawPack artifact or to make unsupported version subpaths fail clearly, then cover that behavior in endpoint tests.
