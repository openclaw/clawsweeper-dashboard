## Summary

Keep open: current main and the live API still reproduce the install-facing digest mismatch on the tarball-looking package route, and no linked PR or canonical replacement was found.

## What This Changes

Fix package artifact URL handling so plugin installs receive digest-matching artifact bytes or a clear failure, and clarify source commit provenance.

## Best Possible Solution

Define digest-safe package artifact routing with regression coverage for unsupported or aliased artifact subpaths, then separately document or fix the trust semantics for displayed source commits.

## Reproduction Assessment

Yes. The live tarball-looking URL returns JSON with the reported mismatched digest while the explicit artifact download returns the expected tarball bytes, and current main's route structure explains why.

## Solution Assessment

Unclear. The code fix is probably narrow, but maintainers should first decide whether the tarball-looking path is an invalid route or a supported alias, and whether the sourceCommit report belongs in the same fix.
