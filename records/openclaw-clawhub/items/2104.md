## Summary

Keep open: current main still has split package artifact behavior, and live ClawHub still returns different successful bodies for the explicit npm-pack artifact, legacy ZIP download, and unsupported `/tarball` suffix.

## What This Changes

The issue requests that ClawHub package install/download paths serve bytes and integrity metadata consistently for npm-pack-backed plugin versions.

## Best Possible Solution

Define one package artifact contract: explicit artifact and npm mirror paths serve exact ClawPack bytes, legacy ZIP downloads expose matching legacy integrity or fail clearly, and unsupported suffixes such as `/tarball` do not return successful metadata bytes.

## Reproduction Assessment

Yes. Source inspection on current main shows separate tarball metadata and legacy ZIP download paths, and live production on 2026-06-10 still returns different successful bodies for the affected package routes.

## Solution Assessment

Unclear. The reported invariant is correct, but the permanent fix needs maintainer/security choice between remapping legacy `sha256hash`, adding explicit legacy download metadata, redirecting, or failing unsupported routes clearly.
