## Summary

Keep open: the live API still reproduces the artifact-looking path returning JSON bytes with a different digest, and current main still has the broad version-detail route that explains it. ClawHub cleanup policy keeps issues open for maintainer resolution, and the remaining choice is API-contract sensitive.

## What This Changes

The issue reports that `/api/v1/packages/zerogpu-router/versions/0.1.0/tarball` returns different bytes than the CLI/artifact download path, causing install integrity checks to fail.

## Best Possible Solution

Make package artifact routes digest-safe: unknown version subpaths should not return metadata bytes, explicit downloads should stream the exact uploaded artifact, and source provenance should be corrected or documented as a separate rule.

## Reproduction Assessment

Yes. The live reported path returns JSON bytes with a different digest from the explicit artifact download, and current main still has a version-detail branch that accepts extra subpaths.

## Solution Assessment

Unclear. Tightening unknown subpaths to 404 is the narrowest route-safety fix, but supporting or redirecting `/tarball` would change the public API contract for clients that guessed that path.
