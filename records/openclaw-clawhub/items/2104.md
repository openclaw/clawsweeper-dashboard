## Summary

Keep open: the issue still reproduces on live production and current main, and ClawHub cleanup policy keeps issues open unless a maintainer resolves them or a fixing PR lands.

## What This Changes

The issue requests that package install artifact routes stop serving bytes that fail the package manifest digest, specifically for `zerogpu-router@0.1.0` via `/versions/{version}/tarball`.

## Best Possible Solution

Make package artifact routes digest-safe by preventing unknown version subpaths from returning metadata bytes, then separately correct or document the bad source-commit provenance for the affected release.

## Reproduction Assessment

Yes. The live `/versions/0.1.0/tarball` URL returns JSON bytes with a different SHA-256 digest from the supported `/artifact/download` route, and current main still contains the route fallthrough that explains it.

## Solution Assessment

Unclear. Rejecting unknown subpaths is the narrowest defensive fix, but maintainers should choose the public API contract before deciding between 404, redirect, or a supported `/tarball` alias.
