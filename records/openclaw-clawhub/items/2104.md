## Summary

Keep this open: current main still leaves the reported artifact-contract split in place, and live ClawHub endpoints still return different successful bodies for the unsupported tarball suffix, explicit npm-pack artifact download, and legacy ZIP download.

## What This Changes

The issue requests consistent ClawHub package artifact bytes and digest metadata across HTTP install paths for npm-pack-backed plugin releases.

## Best Possible Solution

Define one artifact contract: exact ClawPack bytes come from the explicit artifact or npm mirror path, legacy ZIP downloads expose matching legacy integrity or fail clearly, and unsupported suffixes such as /tarball do not return successful metadata bytes.

## Reproduction Assessment

Yes. Live production on June 10, 2026 still returns three successful but different bodies for the affected tarball, explicit artifact, and legacy download paths, and current main source explains the permissive route fallthrough.

## Solution Assessment

Unclear. The route strictness piece is narrow, but the permanent digest behavior needs maintainer/security choice between fail-closed, redirect, compatibility alias, ZIP-specific legacy metadata, and existing-row provenance repair.
