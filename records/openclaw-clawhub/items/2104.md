## Summary

Keep open: the hosted package endpoints still reproduce the artifact digest mismatch, and current main still has a version-subpath fallthrough that makes an archive-looking URL return version metadata JSON instead of artifact bytes. ClawHub cleanup policy keeps issues open, and this one is install-path and supply-chain relevant.

## What This Changes

The issue reports that an install-facing package tarball URL returns bytes that do not match the manifest-backed ClawPack artifact downloaded by the CLI resolver.

## Best Possible Solution

Make package artifact resolution unambiguous: install clients should receive digest-checked ClawPack bytes from the resolver/npm artifact route or a clear unsupported-route error, with regression coverage and a provenance correction or clarification for source commits.

## Reproduction Assessment

Yes. Hosted endpoint checks reproduce the different byte streams and digests, and current-main source inspection shows how an extra version subpath falls through to the version metadata JSON response.

## Solution Assessment

Unclear until maintainers choose the compatibility contract. The narrow maintainable fix is either to intentionally alias tarball-looking version paths to the exact ClawPack bytes or to make unsupported version subpaths fail clearly, then cover the chosen behavior in tests.
