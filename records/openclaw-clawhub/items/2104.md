## Summary

Keep open: the reported integrity mismatch is still supported by the live service and by current main's package route behavior, and ClawHub policy keeps issues open for maintainer handling. The remaining decision is supply-chain-sensitive: choose whether the tarball-looking subpath should alias exact artifact bytes or fail clearly, and settle the source provenance policy.

## What This Changes

Report that a package HTTP version subpath returns metadata bytes instead of the tarball expected by a plugin installer, causing an integrity mismatch, with a related source-commit provenance concern.

## Best Possible Solution

Define one digest-safe package artifact contract, implement that route behavior consistently for HTTP API and installer clients, and document the source provenance rule with regression coverage.

## Reproduction Assessment

Yes for the live reported behavior, and source inspection makes the current-main path clear. The live /tarball-looking URL returns the 5,332-byte JSON digest while /artifact/download returns the 4,678-byte ClawPack artifact digest.

## Solution Assessment

Unclear until maintainers choose the route contract. The narrow code surface is clear, but the safest fix depends on whether the unsupported tarball-looking path should alias exact bytes or fail explicitly, plus how strict publish provenance should be.
