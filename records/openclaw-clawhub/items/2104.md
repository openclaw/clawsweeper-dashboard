## Summary

Keep open: current hosted endpoints still reproduce the byte and digest mismatch, and current main still lets an archive-looking version subpath return package version JSON instead of artifact bytes. This is install-path and supply-chain relevant, and ClawHub cleanup policy keeps issues open.

## What This Changes

The issue reports that a tarball-looking package HTTP API URL returns different bytes than the CLI artifact resolver for `zerogpu-router`, causing install-time integrity failure.

## Best Possible Solution

Make package artifact resolution unambiguous: install clients should receive digest-checked ClawPack bytes from a supported resolver/npm route or a clear unsupported-route error, with regression coverage and provenance clarification.

## Reproduction Assessment

Yes. Hosted endpoint checks on May 10, 2026 reproduce the different byte streams and digests, and current-main source inspection shows the extra version subpath reaches the version metadata JSON response.

## Solution Assessment

Unclear until maintainers choose the compatibility contract. The narrow maintainable fix is either to alias the tarball-looking path to the exact ClawPack artifact or to reject unsupported version subpaths clearly, then cover the chosen behavior in tests.
