## What This Changes

The issue asks ClawHub to stop serving non-artifact bytes from an artifact-looking package tarball URL that causes install integrity failures.

## Best Possible Solution

Make package artifact routes digest-safe by rejecting unknown version subpaths or intentionally supporting a tarball alias that serves exact artifact bytes, then handle affected release provenance separately.

## Solution Assessment

Unclear. The narrowest defensive fix is to stop unknown version subpaths from returning version JSON, but maintainers should choose whether the public contract should be 404, redirect, or a supported tarball alias.
