## What This Changes

The issue reports that a tarball-looking package API URL serves version metadata instead of artifact bytes, causing plugin install integrity validation to fail and raising source commit provenance concerns.

## Best Possible Solution

Define digest-safe package artifact routing with regression coverage, then clarify how source commit provenance should be displayed and trusted.

## Reproduction Assessment

Yes. The live tarball-looking URL still returns JSON with the mismatched digest while the explicit artifact download returns the expected artifact bytes, and current main explains the route fallthrough.

## Solution Assessment

Unclear. A narrow code fix is likely, but maintainers should first choose whether the tarball-looking path should 404, redirect, or stream bytes, and whether the source commit report is handled in the same change.
