## Summary

Keep open: current main and the live service still support the reported install integrity mismatch, and this open ClawHub issue needs maintainer follow-up because package artifact routing and provenance are supply-chain-sensitive API contract decisions.

## What This Changes

The issue reports that an install-facing HTTP package artifact URL returns bytes whose SHA-256 digest differs from the manifest-backed artifact downloaded by the ClawHub CLI.

## Best Possible Solution

Define one digest-safe package artifact contract, implement it consistently across HTTP API and CLI expectations, and add regression coverage plus docs for source provenance handling.

## Reproduction Assessment

Yes. The live /versions/0.1.0/tarball-looking URL still returns 5,332 bytes of JSON with the reporter's digest, while the explicit artifact download route returns the 4,678-byte ClawPack artifact with the expected digest.

## Solution Assessment

Unclear. The code surface is narrow, but maintainers should first decide whether the unsupported tarball-looking path should alias exact artifact bytes or fail clearly, and how strict source metadata inference should be.
