## Summary

Keep open: the reported install integrity problem is still present in the current package HTTP route shape, and this ClawHub issue should not be auto-closed before a maintainer chooses the artifact route contract.

## What This Changes

The issue asks ClawHub to stop serving mismatched bytes or metadata for plugin package installs when a tarball-looking HTTP API URL diverges from the CLI-resolved artifact.

## Best Possible Solution

Define and implement one digest-safe package artifact contract, then add regression coverage and docs for the chosen route behavior plus the intended source provenance rule.

## Reproduction Assessment

Yes. A live request to the reported tarball-looking URL still returns metadata JSON with the reporter’s 5,332-byte digest, and current main source explains that behavior through the broad /versions/{version} route fallback.

## Solution Assessment

Unclear. The code surface is narrow, but maintainers should first decide whether the unsupported tarball-looking route should alias exact ClawPack bytes or fail clearly, and how strict non-trusted source provenance should be.
