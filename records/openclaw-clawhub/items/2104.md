## Summary

Keep this open: current main still has the artifact contract split, live production still returns different successful bodies for the affected package paths, and the related open issue only covers the narrower legacy sha256hash invariant while this one also includes the unsupported /tarball suffix and provenance concern.

## What This Changes

The issue reports that ClawHub package install paths expose mismatched artifact bytes, hashes, and source provenance for a plugin package.

## Best Possible Solution

Define one public artifact contract: exact ClawPack bytes come from the explicit artifact resolver or npm mirror, legacy /download remains internally consistent ZIP compatibility, unsupported version suffixes do not return successful metadata bodies, and bad existing provenance is repaired through an approved moderation or backfill path.

Yes. Live production reproduces the mismatched successful bodies, and current main source explains the /tarball fallthrough plus ZIP-only legacy route; I did not run a local Convex fixture, so this is source-reproducible rather than a local current-main fixture reproduction.

## Solution Assessment

Unclear. The bug is concrete, but the durable fix needs maintainer choice between fail-closed, redirect, compatibility alias, legacy digest repair, and existing-row provenance repair.
