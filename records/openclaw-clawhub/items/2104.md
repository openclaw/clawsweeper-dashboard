## Summary

Keep open: ClawHub cleanup policy keeps issues open, and this one still reproduces as a live install-integrity problem where a tarball-looking version subpath returns metadata JSON instead of artifact bytes.

## What This Changes

The issue reports that a package install path fetches bytes whose digest differs from the published ClawHub artifact, plus a questionable source commit value on the listing.

## Best Possible Solution

Define one digest-safe artifact contract for installs, make ambiguous version subpaths either stream the advertised artifact or fail clearly, and separately define how inferred source commits should be displayed and trusted.

## Reproduction Assessment

Yes. Live curl checks against the reported ClawHub URLs show the tarball-looking route returning 5,332 bytes of JSON while the explicit artifact route returns the expected 4,678-byte package artifact.

## Solution Assessment

Unclear. A narrow fix is likely, but maintainers should first choose whether `/versions/{version}/tarball` is a supported artifact alias or an invalid route, and whether the sourceCommit fallback should be treated as a separate provenance bug.
