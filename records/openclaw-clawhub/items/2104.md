## Summary

Keep open: the live API still reproduces the integrity mismatch, current main still has the broad version-detail fallthrough that explains it, and ClawHub cleanup policy keeps issues open unless a maintainer resolves them.

## What This Changes

The issue reports that an artifact-looking package HTTP URL returns bytes that do not match the published artifact manifest while the CLI artifact download returns the correct tarball.

## Best Possible Solution

Make artifact-serving endpoints digest-safe: unknown version subpaths should not return metadata bytes, explicit downloads should stream exact uploaded artifacts, and source-commit provenance should be fixed or documented separately.

## Reproduction Assessment

Yes. Curling the reported live `/tarball` path returns JSON bytes with a different digest from the explicit artifact download, and current main still routes extra version subpaths to version-detail JSON.

## Solution Assessment

Unclear. The bug is real, but maintainers should choose the API behavior for `/tarball` before implementation decides between rejecting, redirecting, or supporting it as an alias.
