## Summary

Keep open: the hosted endpoints still reproduce the reported byte/digest split, and current main still has route behavior that explains how an artifact-like version subpath can return metadata instead of artifact bytes. The install-facing artifact route contract is supply-chain sensitive, so this needs maintainer direction rather than cleanup closure.

## What This Changes

The issue reports that an install-facing package tarball URL returns bytes whose digest differs from the manifest-verified artifact downloaded by the ClawHub CLI.

## Best Possible Solution

Make package artifact resolution unambiguous, keep digest-checked ClawPack artifact bytes as the install source of truth, add regression coverage for unsupported artifact-like subpaths, and record the chosen install/API trust-boundary intent in specs plus public docs.

## Reproduction Assessment

Yes. The hosted endpoints currently reproduce the byte and digest mismatch, and current-main source inspection shows the broad /versions/{version} branch that can return metadata for an extra tarball segment.

## Solution Assessment

Unclear until maintainers choose the compatibility contract. The narrow maintainable direction is to make unsupported artifact-like subpaths fail clearly or intentionally route them to the exact verified ClawPack artifact bytes, with regression tests for the chosen behavior.

## Security Review

Status: needs_attention

Summary: The issue is supply-chain relevant because install-facing package routes can return bytes that differ from the manifest-verified artifact, although digest verification appears to fail closed.

Concerns:

- **[medium] Artifact-like version subpaths can return non-artifact bytes:** `convex/httpApiV1/packagesV1.ts:2408`
  - body: A tarball-looking version subpath can fall through to the version metadata JSON branch instead of failing or serving the verified ClawPack artifact, which creates ambiguous install-facing behavior around digest verification.
  - confidence: 0.92
- **[medium] Legacy ZIP compatibility can confuse artifact identity:** `convex/httpApiV1/packagesV1.ts:2553`
  - body: The legacy download route intentionally rebuilds ZIP bytes for the same version while ClawPack routes stream the uploaded .tgz, so clients that choose the wrong route can receive a different digest-bearing artifact.
  - confidence: 0.86
- **[low] Source provenance can pair unrelated repo and commit:** `packages/clawhub/src/cli/commands/packages.ts:1927`
  - body: buildSource can pair an explicit source repository with an inferred local commit when --source-commit is omitted, which can mislead package provenance display.
  - confidence: 0.73
