## Summary

Keep open: the reported artifact-integrity mismatch still reproduces on live production and current main still has the broad version-detail route that explains it. ClawHub cleanup policy keeps issues open for maintainer resolution rather than auto-closing them.

## What This Changes

The issue reports that an artifact-looking package tarball URL returns bytes that differ from the ClawHub CLI artifact download, causing install-time integrity mismatches.

## Best Possible Solution

Make unknown package version subpaths digest-safe with a documented route contract and regression coverage, then handle any bad source-commit provenance for the affected release separately.

## Reproduction Assessment

Yes. Live production on May 15, 2026 returns mismatched bytes for the reported `/tarball` path, and current main still contains the broad `/versions/{version}` branch that explains the same behavior from source inspection.

## Solution Assessment

Unclear. Rejecting unknown subpaths is the narrowest defensive fix, but maintainers should first decide the public API contract between 404, redirect, or a supported `/tarball` alias.
