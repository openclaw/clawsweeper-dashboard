## Summary

Keep open: repository policy keeps ClawHub issues open, and the report still describes a real install-facing package artifact contract problem that needs maintainer resolution rather than cleanup closure.

## What This Changes

The issue reports that a package tarball-looking HTTP API URL serves bytes or metadata with a digest different from the CLI-resolved artifact, causing OpenClaw plugin install integrity verification to fail.

## Best Possible Solution

Define one digest-safe package artifact contract for unsupported tarball-looking routes, implement it consistently in the package HTTP API, and add regression coverage plus docs for artifact resolution and source provenance.

## Reproduction Assessment

Yes. The live reported URL still returns a 5,332-byte JSON response with the reporter's digest instead of the 4,678-byte artifact, and current main source explains that through the broad /versions/{version} route fallback.

## Solution Assessment

Unclear. The code fix surface is narrow, but maintainers should first decide whether /versions/{version}/tarball should alias the exact artifact bytes or fail clearly, and whether the source-commit fallback needs the same change or a separate policy fix.
