## Summary

Keep open: current production still reproduces the reported artifact digest split, and current main still lets `/versions/{version}/tarball` fall through to version metadata JSON instead of exact artifact bytes or a clear unsupported-route response.

## What This Changes

The issue asks ClawHub to resolve an install integrity mismatch for `zerogpu-router` and review source commit provenance shown for the published plugin.

## Best Possible Solution

Make package artifact resolution unambiguous and digest-safe for install clients, with tests/docs for the chosen route behavior and a clear source provenance rule for publishes.

## Reproduction Assessment

Yes. Current production on the current-main prod tag returns different byte streams and SHA-256 digests for the tarball-looking path, explicit artifact download, npm mirror, and legacy ZIP route, and source inspection explains the fallthrough.

## Solution Assessment

Unclear. The narrow code surface is identifiable, but maintainers should first decide whether the unsupported tarball-looking route should alias to exact ClawPack bytes or fail clearly, and how strict source provenance should be.

## Security Review

Status: needs_attention
