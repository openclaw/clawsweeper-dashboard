## Summary

Keep open: the hosted endpoints still reproduce the reported byte and digest split, and current main still routes an unsupported version subpath to version metadata JSON instead of an artifact or a clear miss.

## What This Changes

The issue reports that a package install path fetches mismatched artifact bytes from ClawHub HTTP package routes and that displayed source commit provenance can point at the wrong commit.

## Best Possible Solution

Make package artifact resolution unambiguous, keep digest-checked ClawPack tarball bytes as the install source of truth, add regression coverage for unsupported version subpaths, and tighten publish source provenance rules.

## Reproduction Assessment

Yes. Hosted clawhub.ai endpoint checks reproduce the reported digest mismatch, and current-main source inspection shows why an extra version subpath returns metadata JSON rather than artifact bytes.

## Solution Assessment

Unclear until maintainers choose the compatibility contract. The narrow maintainable fix is either to make unsupported version subpaths fail clearly or to intentionally route compatible tarball requests to the exact verified ClawPack artifact, with regression tests for the chosen behavior.
