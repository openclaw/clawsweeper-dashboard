## Summary

Keep open: the hosted API still reproduces the artifact digest mismatch, and current main still has package-route behavior that lets a tarball-looking version subpath return metadata instead of unambiguous artifact bytes or a clear miss. Because this is an open ClawHub issue and the route contract is supply-chain relevant, it needs maintainer follow-up rather than cleanup closure.

## What This Changes

The issue asks ClawHub to stop serving plugin install artifact bytes whose digest differs from the CLI-resolved ClawPack tarball and to correct misleading source commit metadata.

## Best Possible Solution

Make package artifact resolution unambiguous, keep the explicit ClawPack resolver and npm mirror as the source of truth for `.tgz` bytes, add regression coverage for unsupported version subpaths, and prevent explicit source repos from pairing with unrelated inferred commits.

## Reproduction Assessment

Yes. The hosted ClawHub endpoints currently return different byte streams and digests for the tarball-looking path, explicit ClawPack artifact download, and legacy ZIP route, and current-main source inspection explains the version-subpath fallthrough.

## Solution Assessment

Unclear until maintainers choose the compatibility contract. The narrow maintainable direction is to make unsupported tarball-looking subpaths fail clearly or deliberately stream/redirect to the exact ClawPack artifact, with tests covering the chosen behavior.
