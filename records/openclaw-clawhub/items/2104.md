## Summary

Keep open: the reported artifact mismatch is real enough to warrant maintainer handling. The documented resolver and npm mirror return the expected `.tgz`, but `/versions/0.1.0/tarball` currently falls through to version JSON and the legacy `/download` route intentionally returns a ZIP with a different digest.

## What This Changes

The issue asks ClawHub to make package HTTP artifact URLs and source metadata align with the manifest and CLI for `zerogpu-router@0.1.0`.

## Best Possible Solution

Keep the legacy ZIP route ZIP-only, make the version tarball behavior explicit instead of falling through to JSON, and separately verify the package source-attribution path before changing published metadata semantics.

## Reproduction Assessment

Yes for the observable failure: live curl checks reproduced the reported mismatched bytes, and current main's router shows why `/versions/{version}/tarball` returns version JSON. I did not run a local server, so the structured status is source-reproducible rather than a full local current-main reproduction.

## Solution Assessment

Unclear: the existing `/artifact/download` and `/api/npm/...tgz` paths already solve exact artifact delivery, but maintainers should decide whether the undocumented `/tarball` path becomes a compatibility alias or a hard 404. The source-commit metadata concern should be checked separately from the download route fix.
