## What This Changes

Reports that an artifact-looking package URL returns bytes with a different digest than the CLI/artifact download path, causing plugin install integrity failures and raising a source provenance question.

## Best Possible Solution

Define a digest-safe package artifact route contract, add regression coverage for unsupported or aliased artifact subpaths, and keep provenance policy separate unless maintainers confirm it is part of the same bug.

## Reproduction Assessment

## Solution Assessment

Unclear. The problem is real, but maintainers should choose whether the route should 404, redirect, or become an intentional alias before a narrow implementation lands.
