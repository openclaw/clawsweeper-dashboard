## Summary

Current main still preserves distinct npm-pack and legacy ZIP download contracts, and the live latest release reproduces different successful bodies and digests for those routes. The issue remains necessary because maintainers must define the backward-compatible integrity contract before changing legacy metadata or route behavior.

## What This Changes

Fix plugin installation integrity mismatches caused by package metadata and HTTP download routes describing different archive bytes.

## Best Possible Solution

Specify separate authoritative integrity fields for npm-pack and legacy ZIP bytes, make every route and compatibility alias match the bytes it serves, and cover both old-client and artifact-aware installs with real HTTP validation.

## Reproduction Assessment

Yes. Live requests on June 10, 2026 reproduce distinct `.tgz` and ZIP bodies and hashes, and current-main source explicitly retains both download contracts.

## Solution Assessment

No single route or field substitution is proven safest yet. Maintainers should first choose the legacy compatibility contract, then update metadata, route behavior, tests, and the durable package-installability spec together.
