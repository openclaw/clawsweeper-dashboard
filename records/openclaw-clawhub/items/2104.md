## Summary

Keep open: the package artifact digest split still reproduces on current production, and current main still has a broad version route that lets a tarball-looking subpath return version JSON instead of exact artifact bytes or a clear unsupported-route response.

## What This Changes

The issue asks ClawHub to stop serving different bytes and digests for install-facing package artifact URLs and to clarify/correct source provenance for the affected plugin release.

## Best Possible Solution

Make package artifact resolution unambiguous and digest-safe for install clients, with regression coverage for the chosen route behavior and a clear source provenance rule for publishes.

## Reproduction Assessment

Yes. Current production on the deployed current-main tag returns different byte streams and SHA-256 digests for the tarball-looking path, explicit artifact download, npm mirror, and legacy ZIP route, and source inspection explains the fallthrough.

## Solution Assessment

Unclear. The code surface is narrow, but maintainers should first decide whether the unsupported tarball-looking route should alias to exact ClawPack bytes or fail clearly, and how strict source provenance should be.
