## Summary

Keep open: ClawHub issue cleanup does not auto-close issues, and current main still has a source-level route shape where `/versions/{version}/tarball` can fall through to version metadata instead of exact artifact bytes.

## What This Changes

The issue reports that the package HTTP API can return bytes that do not match the manifest digest for a plugin install path that expects the published artifact.

## Best Possible Solution

Constrain package version subpaths so artifact-looking URLs either serve or redirect to the exact stored artifact bytes through a documented compatibility path, or fail clearly without returning digest-mismatched metadata.

## Reproduction Assessment

Yes, at source level. Current main prefix-routes package subpaths, preserves the extra `/tarball` segment, and then accepts unknown `/versions/{version}/...` paths in the broad version-detail branch.

## Solution Assessment

Unclear. Rejecting unknown subpaths is the narrow defensive fix, but serving or redirecting `/tarball` to exact artifact bytes may be the safer compatibility path if existing installers depend on that URL shape.
