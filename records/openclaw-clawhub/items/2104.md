## Summary

Keep open: current main/latest release still reproduces the package install integrity failure, with `/versions/{version}/tarball` returning JSON where an installer expects artifact bytes.

## What This Changes

Ensure package artifact/tarball HTTP endpoints serve the exact stored plugin artifact bytes or fail clearly so install clients do not hit digest mismatches.

## Best Possible Solution

Constrain version subpaths so artifact-looking URLs either serve or redirect to exact stored ClawPack bytes through a documented compatibility contract, or fail with a clear non-artifact response; add regression coverage for the chosen API behavior.

## Reproduction Assessment

Yes. The live latest release returns JSON for the reported tarball-looking URL while the explicit artifact download returns the expected 4,678-byte artifact, and source inspection explains the routing path.

## Solution Assessment

Unclear. Rejecting unknown version subpaths is the narrow defensive fix, but maintainers should decide whether compatibility requires `/tarball` to redirect to or serve the documented npm mirror artifact URL.
