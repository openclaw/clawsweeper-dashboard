## What This Changes

The issue reports that an artifact-looking package version tarball URL returns bytes that do not match the published package manifest, causing plugin install integrity failures.

## Best Possible Solution

Define a digest-safe package artifact route contract, add regression coverage for unsupported or aliased artifact subpaths, and keep source provenance policy as separate follow-up unless maintainers confirm it shares the same root cause.

## Reproduction Assessment

Yes. The live `/versions/0.1.0/tarball` path returns JSON bytes with a different digest while `/versions/0.1.0/artifact/download` and the npm tarball URL return the expected tarball bytes.

## Solution Assessment

Unclear. The narrow code issue is real, but the best repair depends on whether maintainers want the tarball subpath to 404, redirect, or become a supported alias.
