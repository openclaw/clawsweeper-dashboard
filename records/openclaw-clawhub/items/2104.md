Keep open: the reported install integrity mismatch still reproduces, and ClawHub policy keeps issues open for maintainer follow-up rather than cleanup closure.

## What This Changes

The issue reports that the package HTTP tarball-looking URL returns bytes that do not match the artifact advertised by the CLI/manifest for `zerogpu-router@0.1.0`.

## Best Possible Solution

Define one digest-safe package artifact contract, implement it in the HTTP API and CLI expectations, and add regression coverage and docs for the chosen route and source provenance rule.

## Reproduction Assessment

Yes. Live requests to the reported URL reproduce the 5,332-byte JSON response and digest mismatch, while current source explains why the extra `/tarball` segment falls through to version metadata.

## Solution Assessment

Unclear. The code surface is narrow, but maintainers should first decide whether the tarball-looking v1 subpath should alias exact artifact bytes or return a clear unsupported-route error, and how strict source metadata inference should be.
