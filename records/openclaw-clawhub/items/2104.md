## Summary

Keep open: this issue is still valid and ClawHub cleanup policy keeps issues open; the live API still returns JSON at a tarball-looking install path while the resolver and npm mirror return the digest-matching artifact bytes.

## What This Changes

The issue reports that the package HTTP API serves bytes whose SHA-256 does not match the manifest-backed CLI artifact for `zerogpu-router@0.1.0`, with a related source commit provenance concern.

## Best Possible Solution

Define one digest-safe artifact contract, make artifact-looking subpaths either stream the exact advertised bytes or fail loudly, and add regression coverage for route semantics plus source provenance inference.

## Reproduction Assessment

Yes. Live requests reproduce the mismatched byte counts and digests, and current source inspection shows the same broad version-route fallback that makes the reported path return metadata JSON.

## Solution Assessment

Unclear. The code change is likely narrow, but maintainers should first choose the public contract for `/versions/{version}/tarball` and the trusted fallback policy for source commit metadata.
