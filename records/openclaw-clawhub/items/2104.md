## Summary

Keep open: the live service and current source still support the reported install integrity mismatch, and the fix needs maintainer judgment on the package artifact route contract and provenance policy.

## What This Changes

The issue asks ClawHub to fix a package install integrity mismatch where a tarball-looking HTTP API path returns different bytes than the resolver-advertised ClawPack artifact, plus clarify source commit provenance.

## Best Possible Solution

Define one digest-safe package artifact contract, implement it in the HTTP API and CLI expectations, then add regression coverage and docs for the chosen route behavior and source provenance rule.

## Reproduction Assessment

Yes. A live request to the reported tarball-looking URL still returns 5,332 bytes of JSON with the reporter's digest, while the supported artifact download route returns the 4,678-byte ClawPack artifact.

## Solution Assessment

Unclear. The code surface is narrow, but maintainers should first decide whether the unsupported tarball-looking path should alias exact artifact bytes or fail clearly, and how strict source metadata inference should be.
