## Summary

Keep open: the hosted package endpoints still reproduce the reported digest mismatch, and current main still has a broad version-subpath branch that lets an archive-looking URL return version metadata JSON instead of artifact bytes. ClawHub cleanup policy keeps issues open, and this one is install-path and supply-chain relevant.

## What This Changes

Reports that a ClawHub package install path for `zerogpu-router` can fetch bytes whose SHA-256 differs from the manifest-verified CLI artifact.

## Best Possible Solution

Make package artifact resolution unambiguous: install clients should receive digest-checked ClawPack bytes from a supported resolver/npm route or a clear unsupported-route error, with regression coverage and a provenance clarification or fix for source commits.

## Reproduction Assessment

Yes. Hosted endpoint checks reproduce the different byte streams and digests, and current-main source inspection shows the extra version subpath reaching the version metadata JSON response.

## Solution Assessment

Unclear until maintainers choose the route compatibility contract. The narrow maintainable fix is either to alias the tarball-looking path to the exact ClawPack artifact or to make unsupported version subpaths fail clearly, then cover the chosen behavior in tests.
