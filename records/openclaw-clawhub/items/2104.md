## Label Justifications

- `P1`: The digest mismatch blocks real plugin install workflows on legacy OpenClaw paths for packages that advertise compatibility with those clients.
- `impact:security`: The issue concerns public digest metadata used to verify executable plugin download bytes.

## Summary

Keep this open: current main still has a split contract where npm-pack artifact metadata and explicit artifact downloads use the uploaded `.tgz`, while the legacy package download route serves rebuilt ZIP bytes. Live ClawHub endpoints for the reported package still return different hashes across those paths, so this is not implemented on main.

## What This Changes

The issue asks ClawHub to stop serving plugin install paths where the artifact bytes and integrity metadata disagree between the HTTP API and CLI/artifact resolver.

## Best Possible Solution

Define one package artifact integrity contract: explicit artifact and npm mirror paths should keep exact ClawPack tarball verification, while legacy ZIP downloads should expose matching legacy integrity or fail/redirect clearly for clients that cannot verify npm-pack releases.

## Reproduction Assessment

Yes. Source inspection on current main shows separate npm-pack artifact metadata/download and legacy ZIP download paths, and live production on June 10, 2026 still returns different hashes for the reported package version.

## Solution Assessment

Unclear. The target invariant is correct, but the durable fix needs a maintainer/security choice between remapping legacy `sha256hash`, omitting it for legacy clients, adding explicit legacy download metadata, or failing unsupported artifact suffixes clearly.
