## Summary

Keep open: the hosted behavior still reproduces the digest mismatch, and current main still routes the archive-looking version subpath to version metadata JSON instead of artifact bytes or a clear unsupported-route error. This is install-path and supply-chain relevant, and ClawHub cleanup policy keeps issues open for maintainer follow-up.

## What This Changes

The issue asks ClawHub to make install-facing package artifact URLs return digest-matching ClawPack bytes or fail clearly, and to address confusing source-commit provenance.

## Best Possible Solution

Make package artifact resolution unambiguous and digest-safe, then add regression coverage for the chosen route behavior and clarify or fix source provenance handling.

## Reproduction Assessment

Yes. Hosted endpoint checks reproduce the different byte streams and digests, and current-main source inspection shows why `/versions/{version}/tarball` reaches the version metadata JSON branch.

## Solution Assessment

Unclear until maintainers choose the route compatibility contract. The narrow maintainable fix is either to alias the tarball-looking path to exact ClawPack artifact bytes or to make unsupported version subpaths fail clearly, with tests for the chosen behavior.
