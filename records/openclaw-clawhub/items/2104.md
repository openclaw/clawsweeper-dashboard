## Summary

Keep open: ClawHub issue auto-close is not allowed here, and the report still reproduces on the live API with current source showing the broad version route can return metadata JSON for a tarball-looking subpath.

## What This Changes

The issue reports that a package-version tarball URL returns content whose digest differs from the CLI-resolved published package artifact.

## Best Possible Solution

Define one digest-safe artifact contract, make unsupported artifact-looking subpaths fail or alias to exact bytes consistently, and clarify sourceCommit inference before treating provenance as trusted.

## Reproduction Assessment

Yes. Curling the reported live URL returns 5,332 bytes of JSON with the mismatched digest, while the explicit artifact download and npm mirror return the expected 4,678-byte tarball; current source explains the route fallthrough.

## Solution Assessment

Unclear. The code repair is probably narrow, but maintainers should first decide whether /versions/{version}/tarball is an alias to support or an invalid route to reject, and how sourceCommit fallback should behave.
