## Summary

Keep open: this issue is still source-reproducible on current main and the hosted endpoints still expose mismatched artifact byte streams for install-facing package routes. The report is supply-chain relevant, so it should remain with maintainers for an explicit route-compatibility decision rather than cleanup closure.

## What This Changes

The issue reports that a package install path receives artifact bytes whose SHA-256 digest differs from the manifest-verified ClawPack tarball for `zerogpu-router@0.1.0`, and that source commit metadata appears to be inferred from the publishing CLI rather than the source repository.

## Best Possible Solution

Make package artifact resolution unambiguous, keep the ClawPack tarball bytes as the install integrity source of truth, add regression coverage for unsupported version subpaths, and prevent explicit source repositories from being paired with unrelated inferred commits.

## Reproduction Assessment

Yes. The issue discussion provides concrete hosted endpoint digests, and current-main source inspection explains why the tarball-looking version subpath can return metadata while the explicit artifact and legacy download routes return different byte streams.

## Solution Assessment

Unclear until maintainers choose the route compatibility contract. The narrow maintainable fix is to make unsupported tarball-looking subpaths fail clearly or deliberately resolve to the same ClawPack artifact bytes, with tests for the chosen behavior.
