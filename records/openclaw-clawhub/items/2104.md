## Summary

Keep open: live ClawHub endpoints still reproduce the reported install-facing digest split, and current main's package version router still lets extra version subpaths fall through to JSON metadata. Because this is an open ClawHub issue and supply-chain relevant, it needs maintainer routing rather than cleanup closure.

## What This Changes

Resolve a plugin install integrity mismatch where an artifact-looking HTTP path, the verified ClawPack tarball, and the legacy ZIP route return different bytes and digests.

## Best Possible Solution

Make package artifact resolution unambiguous, keep digest-checked ClawPack tarball bytes as the install source of truth, add regression coverage for unsupported version subpaths, and correct or clarify source commit provenance.

## Reproduction Assessment

Yes. Hosted endpoint checks reproduce the byte and digest split, and current-main source inspection shows why the tarball-looking version subpath returns metadata JSON rather than artifact bytes.

## Solution Assessment

Unclear until maintainers choose the compatibility contract. The narrow maintainable fix is either to fail unsupported version subpaths clearly or intentionally route compatible tarball requests to the verified ClawPack artifact, with regression tests for the chosen behavior.
