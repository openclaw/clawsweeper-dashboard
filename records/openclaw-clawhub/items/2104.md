## Summary

Keep open: the hosted package endpoints still reproduce the reported digest mismatch, current main still lets an archive-looking version subpath return version metadata JSON, and ClawHub cleanup policy does not auto-close issues.

## What This Changes

The issue reports that an install-facing package tarball URL returns bytes with a different digest than the ClawPack artifact and that displayed source-commit provenance may be wrong.

## Best Possible Solution

Make package artifact resolution unambiguous: install clients should receive digest-checked ClawPack bytes from a supported resolver/npm route or a clear unsupported-route error, with regression coverage and source-provenance clarification or a fix.

## Reproduction Assessment

Yes. Hosted endpoint checks reproduce the different byte streams and digests, and current-main source inspection shows the extra version subpath reaches the version metadata JSON response.

## Solution Assessment

Unclear until maintainers choose the route compatibility contract. The narrow maintainable fix is either to alias the tarball-looking path to the exact ClawPack artifact or make unsupported version subpaths fail clearly, then cover the chosen behavior in tests.
