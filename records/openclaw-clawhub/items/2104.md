## Summary

Keep open: the reported integrity mismatch still reproduces against the live route and is explained by current main’s package route fallthrough; this install-facing package API behavior needs maintainer contract/security review, not cleanup closure.

## What This Changes

The issue reports that a tarball-looking package HTTP API URL serves different bytes than the CLI/artifact resolver, causing plugin install integrity mismatches and exposing questionable source-commit provenance.

## Best Possible Solution

Define one digest-safe package artifact contract, implement the chosen route behavior with regression coverage, and document the intended source provenance rule for explicit, inferred, and trusted publishes.

## Reproduction Assessment

Yes. The reported live tarball-looking URL still returns JSON bytes with the reporter’s 5,332-byte digest, while the advertised npm-pack tarball returns the expected 4,678-byte artifact; current main’s route fallthrough explains the mismatch.

## Solution Assessment

Unclear. The narrow code surface is identifiable, but the best fix depends on a maintainer decision between an exact artifact alias and a hard unsupported-route failure, plus a policy decision for source provenance display.
