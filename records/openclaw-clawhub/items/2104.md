## Summary

Keep open: the hosted package endpoints still reproduce the reported artifact mismatch, and current main still has a version-subpath fallthrough that can return version metadata JSON for an artifact-looking URL. No canonical duplicate or merged fix currently owns the remaining work.

## What This Changes

Ensure install-facing package tarball routes serve the same verified ClawPack artifact bytes as the CLI and do not publish misleading source commit provenance.

## Best Possible Solution

Make package artifact resolution unambiguous, keep digest-checked ClawPack tarball bytes as the install source of truth, add regression coverage for unsupported or compatibility tarball subpaths, and prevent mixed explicit/inferred source provenance during publish.

## Reproduction Assessment

Yes. Hosted endpoint checks reproduce the different byte streams and digests, and current-main source inspection explains the `/versions/{version}/tarball` fallthrough to metadata JSON.

## Solution Assessment

Unclear until maintainers choose the compatibility contract. The narrow maintainable fix is either to reject unsupported version subpaths clearly or intentionally route compatible tarball requests to the exact verified ClawPack artifact, with regression tests for the chosen behavior.
