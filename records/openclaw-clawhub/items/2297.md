## Summary

Keep this PR open: the Worktrunk direction is plausible, but the patch makes the documented `bun run dev:worktree` path depend on an undeclared `wt` executable and does not include real setup/start/stop proof.

## What This Changes

The PR adds Worktrunk config and `.worktreeinclude`, routes `dev:worktree` through `wt --yes dev`, adds an env-only setup mode, and terminates managed dev children by process group.

## Best Possible Solution

Keep the Worktrunk path only with an explicit compatibility story: either preserve the old setup/dev-worktree fallback when `wt` is unavailable, or pin/install-check and document Worktrunk before making it mandatory.
