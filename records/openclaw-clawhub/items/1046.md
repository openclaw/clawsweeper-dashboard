## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still matches both reported surfaces: self-deleted GitHub-linked accounts are blocked on reauth, while publish still surfaces GitHub 403/429 from the account-age gate as the reported rate-limit error.

## What This Changes

The issue requests recovery from a publish failure caused by GitHub API rate limiting and a clean sign-in or recreation path after deleting a GitHub-linked ClawHub account.

## Best Possible Solution

Maintainers should decide the deleted-account recovery policy, then either keep irreversible deletion with clearer UI/admin messaging or add a controlled recreation, relink, or admin recovery path; separately, publish should improve 403/429 recovery guidance while preserving the GitHub account-age gate.

## Reproduction Assessment

Yes. Current main has a high-confidence source/test reproduction path: self-deletion writes deactivatedAt and purgedAt, OAuth reauth rejects deactivated users, and mocked GitHub 403/429 responses still throw the reported publish error.

## Solution Assessment

Not applicable as a patch review because no implementation is proposed here. The maintainable direction is to split the account-recreation product/security decision from narrower publish-rate-limit recovery improvements.
