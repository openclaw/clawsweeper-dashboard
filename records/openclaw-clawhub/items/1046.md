## Summary

Keep open. Repository policy keeps ClawHub issues open, and current main still has source-backed paths for both reported surfaces: deleted accounts remain blocked on GitHub reauth, and publish can still surface the GitHub 403/429 rate-limit error from the account-age gate.

## What This Changes

The issue reports a skill publish failure on a GitHub API rate-limit response plus a GitHub sign-in loop after deleting a ClawHub account.

## Best Possible Solution

Maintainers should decide the deleted-account recovery policy, then either preserve irreversible deletion with clearer user/admin guidance or implement a controlled relink/recreation/admin recovery path; publish should separately improve 403/429 recovery guidance without bypassing the GitHub account-age gate.

## Reproduction Assessment

Yes. Source and tests provide a high-confidence path: account deletion writes `deactivatedAt`, OAuth reauth rejects that state, and mocked GitHub 403/429 responses still throw the reported publish rate-limit error.

## Solution Assessment

Not applicable as a patch review because no fix is proposed here. The maintainable direction is to split the deleted-account recovery policy decision from a narrower publish-rate-limit retry or guidance improvement.
