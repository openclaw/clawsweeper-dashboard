## Summary

Keep open. This is an open ClawHub issue, and current main still has a source-reproducible publish path that can emit the reported GitHub API rate-limit error; the deleted-account half now matches the documented permanent-deletion policy, but publish recovery and account lifecycle decisions still need maintainer handling.

## What This Changes

The issue asks ClawHub to recover more clearly from GitHub API rate limits during skill publish and to handle GitHub sign-in after account deletion.

Keep the permanent-deletion policy unless maintainers approve a controlled recovery path, and triage the publish-rate-limit cluster with production token/quota evidence before changing the GitHub account-age gate.
