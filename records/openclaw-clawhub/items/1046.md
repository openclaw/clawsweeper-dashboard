## Summary

Keep open. Current main still matches the report: deleted/deactivated GitHub-linked accounts are blocked on reauth, and publishing still surfaces GitHub 403/429 from the account-age lookup as the reported rate-limit error. The target-repository cleanup policy keeps ClawHub issues open, and the remaining work is a product/security decision plus a narrower publish recovery improvement.

## What This Changes

The issue requests clearer recovery for skill publishing when GitHub rate limits are hit and a clean sign-in or recreation path after deleting a GitHub-linked ClawHub account.

## Best Possible Solution

Maintainers should first decide the deleted-account lifecycle policy, then implement either clearer irreversible-deletion guidance or a controlled recreation/relink/admin recovery path, while separately improving GitHub 403/429 publish recovery guidance without bypassing the account-age gate.

Yes. Source inspection gives a high-confidence path: self-deletion sets deactivatedAt, OAuth reauth rejects that state, active auth helpers hide deleted users from sessions, and mocked GitHub 403/429 responses still produce the reported publish rate-limit error.
