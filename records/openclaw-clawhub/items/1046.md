## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still matches both reported surfaces: deleted/deactivated GitHub-linked accounts are rejected on reauth, and publish still surfaces GitHub 403/429 from the account-age gate as the reported rate-limit error.

## What This Changes

The issue asks ClawHub to recover cleanly from GitHub API rate limits during publish and to allow or clearly handle GitHub sign-in after self-deleting an account.

## Best Possible Solution

Maintainers should decide the deleted-account recovery policy, then either keep irreversible deletion with clearer UI/admin messaging or add a controlled recreation, relink, or admin recovery path; separately, publish should improve 403/429 recovery guidance while preserving the GitHub account-age gate.

## Reproduction Assessment

Yes. Current main provides a high-confidence source/test reproduction path: self-deletion writes `deactivatedAt`, OAuth reauth rejects deactivated users, and mocked GitHub 403/429 responses still throw the reported publish error.

## Solution Assessment

Not applicable as a patch review because no fix is proposed here. The maintainable direction is to split product/security policy for account recreation from the narrower publish-rate-limit recovery behavior.
