Keep open. Current main still has a source-reproducible publish path that can emit the reported GitHub 403/429 rate-limit error, while the deleted-account sign-in half now matches documented permanent-deletion behavior and a maintainer comment left the publish-flow recovery/wording unresolved.

## What This Changes

The issue requests recovery guidance for skill publishing failures caused by GitHub API rate limits and for GitHub sign-in after deleting a ClawHub account.

## Best Possible Solution

Keep the current deleted-account policy unless maintainers approve a recovery path, and triage the publish-rate-limit cluster with production logs/config before adding retry, cache, backfill, or wording changes that preserve the account-age gate.

Yes, by source inspection. Normal publish calls requireGitHubAccountAge, which throws the exact 403/429 rate-limit message, and deleted-account OAuth reaches handleDeletedUserSignIn, which rejects permanent deletion; I did not run a live production OAuth or publish flow.
