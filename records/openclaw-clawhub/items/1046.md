## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still matches the reported surfaces: deleted/deactivated accounts are blocked on GitHub reauth, while publish can still return the GitHub 403/429 rate-limit error from the account-age gate.

## What This Changes

The issue reports failed skill publishing after a GitHub API rate-limit error and failed GitHub sign-in after deleting a ClawHub account.

## Best Possible Solution

Maintainers should decide the deleted-account recovery policy, then split any implementation into a controlled account recreation/relink/admin recovery path and a separate publish 403/429 recovery improvement that preserves the GitHub account-age gate.

## Reproduction Assessment

Yes. Current-main source and tests provide a high-confidence path: self-deletion writes `deactivatedAt`, OAuth reauth rejects deactivated users, and mocked GitHub 403/429 responses still throw the reported publish rate-limit error.

## Solution Assessment

Not applicable as a patch review because no fix is proposed here. The narrow maintainable direction is to separate the account recovery policy decision from the publish-rate-limit recovery improvement.
