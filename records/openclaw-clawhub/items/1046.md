## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still matches the reported surfaces: self-deleted accounts are blocked from reauth by design, while publish can still return the reported GitHub 403/429 rate-limit error from the account-age gate.

## What This Changes

The issue asks ClawHub to recover more clearly from GitHub publish rate limits and allow sign-in or account recreation after a user deletes their account.

## Best Possible Solution

Maintainers should decide the deleted-account recovery policy first, then either keep irreversible deletion with clearer user/admin guidance or add a controlled recreation, relink, or admin recovery path; publish recovery should be handled separately with clearer 403/429 guidance or retry/backoff while preserving the GitHub age gate.

## Reproduction Assessment

Yes. Current-main source and tests provide a high-confidence path: self-deletion writes `deactivatedAt`/`purgedAt`, OAuth reauth rejects deactivated users, and mocked GitHub 403/429 responses still throw the reported publish rate-limit error.

## Solution Assessment

Not applicable as a patch review because no fix is proposed here. The maintainable direction is to split the account recovery policy decision from a narrower publish-rate-limit recovery improvement that preserves the existing anti-abuse gate.
