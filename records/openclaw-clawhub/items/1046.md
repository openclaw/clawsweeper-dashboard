## Summary

Keep open. This is an open ClawHub issue, and current main still contains both reported surfaces: self-deleted accounts are blocked on GitHub reauth, while publish can still return the reported GitHub 403/429 account-age rate-limit error.

## What This Changes

The issue asks ClawHub to recover more gracefully from GitHub publish rate limits and to allow a GitHub-linked account to sign in again after self-deletion.

## Best Possible Solution

Maintainers should decide the deleted-account recovery policy first, then either improve irreversible-deletion guidance or add a controlled recreation/relink/admin recovery path; publish should separately improve 403/429 recovery guidance around the GitHub account-age lookup.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: self-deletion sets `deactivatedAt`, OAuth reauth rejects that state, and mocked GitHub 403/429 responses still produce the reported publish rate-limit error.

## Solution Assessment

Not applicable as a patch review because no fix is proposed. The maintainable direction is to split the security/product decision for deleted-account recovery from a narrower publish-rate-limit UX or retry improvement.
