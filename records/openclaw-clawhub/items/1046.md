## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still matches both reported surfaces: deleted accounts cannot reauthenticate, and publish can still surface the GitHub 403/429 rate-limit error from the account-age gate.

## What This Changes

The issue requests clearer recovery for skill publish failures caused by GitHub rate limits and a way to sign back in or recover after deleting a GitHub-linked ClawHub account.

## Best Possible Solution

Maintainers should decide the deleted-account recovery policy, then either keep irreversible deletion with clearer user/admin guidance or implement a controlled recreation, relink, or admin recovery path; publish should separately improve 403/429 recovery guidance while preserving the GitHub account-age gate.

## Reproduction Assessment

Yes. Current-main source and tests provide a high-confidence path: self-deletion writes `deactivatedAt`, OAuth reauth rejects deactivated users, and mocked GitHub 403/429 responses still throw the reported publish rate-limit error.

## Solution Assessment

Not applicable as a patch review because no fix is proposed here. The maintainable direction is to split the account recovery policy decision from a narrower publish-rate-limit recovery improvement.
