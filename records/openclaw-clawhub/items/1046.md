## Summary

Keep open. Repository policy keeps ClawHub issues open, and current main still matches the report: self-deleted accounts are intentionally blocked from reauth, while publish still surfaces the documented GitHub 403/429 rate-limit error from the GitHub account-age gate.

## What This Changes

The issue asks ClawHub to recover or recreate a GitHub-linked account after self-deletion and improve publish recovery when the GitHub account-age lookup hits 403/429.

## Best Possible Solution

Maintainers should decide the deleted-account recovery policy first, then implement either clearer irreversible-deletion/admin-cleanup guidance or a controlled recreation/relink path; publish should separately improve 403/429 recovery guidance while preserving the account-age gate.

## Reproduction Assessment

Yes. Source and tests provide a high-confidence path: self-deletion sets `deactivatedAt`, OAuth reauth rejects deactivated users, and mocked GitHub 403/429 responses still throw the reported publish rate-limit error.

## Solution Assessment

Not applicable as a patch review because no fix is proposed. The narrow maintainable direction is to split the account recovery product/security decision from a publish-rate-limit recovery improvement.
