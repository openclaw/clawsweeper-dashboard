## Summary

Keep open. The repository cleanup policy keeps ClawHub issues open, and current main still matches both reported surfaces: self-deleted or deactivated GitHub-linked accounts are rejected during OAuth reauth, while publishing still routes through the GitHub account-age lookup that emits the reported 403/429 rate-limit error. Related open issues #504 and #886 track the deleted-account login loop, and #1674 overlaps broader account-blocking recovery.

## What This Changes

The reporter asks for recovery from a failed skill publish caused by GitHub API rate limiting and a way to sign in or recreate a GitHub-linked ClawHub account after self-deletion.

## Best Possible Solution

Maintainers should make an explicit product/security decision for self-deleted GitHub-linked accounts, then either keep the irreversible policy with clearer user-facing recovery/admin messaging or add a controlled recreation, relink, or admin recovery path with audit and abuse protections. Separately, publish should provide clearer 403/429 recovery guidance around the GitHub account-age lookup, including Retry-After/backoff behavior or deployment checks for GITHUB_TOKEN.
