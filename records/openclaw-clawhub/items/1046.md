## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main still matches the report: self-deleted accounts are intentionally blocked from reauth, while publish can still surface the documented GitHub 403/429 rate-limit error from the account-age gate.

## What This Changes

The issue reports a web skill publish failure on GitHub API rate limiting plus inability to sign back in after deleting a GitHub-linked ClawHub account.

## Best Possible Solution

Maintainers should decide the deleted-account recovery policy first, then implement either clearer irreversible-deletion guidance/admin cleanup or a controlled recreation/relink path; publish should separately improve 403/429 recovery guidance while preserving the GitHub account-age gate.

## Reproduction Assessment

Yes. Source and tests provide a high-confidence reproduction path: `deleteAccount` sets `deactivatedAt`, OAuth reauth rejects deactivated users, and mocked GitHub 403/429 responses still throw the reported publish rate-limit error.

## Solution Assessment

Not applicable as a patch review because no fix is proposed here. The narrow maintainable direction is to split the account recovery policy decision from a publish-rate-limit recovery improvement.
