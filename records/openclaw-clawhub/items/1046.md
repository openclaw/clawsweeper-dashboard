## Summary

Keep open. Repository policy keeps ClawHub issues open, and current main still has both reported surfaces: self-deleted accounts remain blocked on GitHub reauth, while publish can still return the GitHub 403/429 account-age rate-limit error.

## What This Changes

The issue asks ClawHub to recover more clearly from a GitHub publish rate-limit error and to handle GitHub sign-in after a self-deleted account.

## Best Possible Solution

Maintainers should define the deleted-account recovery policy, then implement either clearer irreversible-deletion guidance or a controlled recreation/relink/admin recovery path, while separately improving GitHub 403/429 publish recovery guidance without bypassing the age gate.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: self-deletion sets deactivatedAt, OAuth reauth rejects that state, and mocked GitHub 403/429 responses still produce the reported publish rate-limit error.

## Solution Assessment

No patch is proposed. The maintainable direction is to split the account-recovery policy decision from the narrower publish-rate-limit UX/retry improvement.
