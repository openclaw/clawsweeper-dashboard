## Summary

Keep open. Repository policy keeps ClawHub issues open, and current main still matches both reported surfaces: deleted accounts are treated as permanently deactivated on OAuth reauth, while publish can still surface GitHub 403/429 account-age gate failures as the reported rate-limit error.

## What This Changes

The issue reports a combined publishing failure from GitHub API rate limiting and inability to sign in again after deleting a ClawHub account.

## Best Possible Solution

Maintainers should decide the deleted-account recovery policy first, then either improve irreversible-deletion guidance or add a controlled recreation/relink/admin recovery path; publish should separately improve 403/429 recovery guidance around the GitHub account-age lookup.

## Reproduction Assessment

Yes. Source inspection and existing tests give a high-confidence path: self-deletion sets `deactivatedAt`, OAuth reauth rejects that state, and mocked GitHub 403/429 responses still produce the reported publish rate-limit error.

## Solution Assessment

Not applicable as a patch review because no fix is proposed. The maintainable direction is to split the security/product decision for deleted-account recovery from a narrower publish-rate-limit UX or retry improvement.
