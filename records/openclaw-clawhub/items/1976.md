## Summary

Keep open because the report matches a security-sensitive runtime approval-boundary problem. Current ClawHub main can flag and warn on nova, and the live skill remains suspicious, but this repository does not enforce the required confirmation sequence before `nova send` or `nova withdraw` executes.

## What This Changes

The issue reports that the nova skill ran `nova send 95 -t` to create a claim URL without first confirming network, amount, destination, and explicit user approval.

## Best Possible Solution

Keep nova flagged as suspicious in ClawHub and implement a runtime-enforced final-action approval gate for financial commands before allowing `send`, `withdraw`, or claim-link creation.
