## Summary

Keep open: this is still a security-sensitive runtime approval-boundary report, and a member explicitly left it open for runtime/policy follow-up. Current ClawHub main can publish, scan, warn, and moderate Nova artifacts, but it does not execute `nova send` or enforce the skipped financial confirmation sequence.

## What This Changes

The issue reports that the published Nova skill guardrail was bypassed when an agent ran `nova send 95 -t` before checking network and obtaining explicit user confirmation.

## Best Possible Solution

Keep this open for an enforceable OpenClaw/runtime approval gate around irreversible financial final actions while preserving ClawHub's artifact warning and moderation surfaces for Nova.

## Reproduction Assessment

No high-confidence current-main reproduction path exists in this repository because ClawHub does not execute Nova commands. The issue trace and live SKILL.md make the reported approval-boundary failure plausible, but the runtime executor is outside this checkout.

## Solution Assessment

No, not as a ClawHub-only wording change. The reported trace already bypassed written guardrails, so the maintainable fix is runtime-enforced approval before `send`, `withdraw`, or claim-link creation, with ClawHub continuing to surface risk metadata.
