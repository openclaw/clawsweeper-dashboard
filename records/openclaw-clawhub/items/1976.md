## Summary

Keep open: maintainer triage already says Nova should remain suspicious/review-required and this is a runtime approval-boundary problem. Current ClawHub main supports registry moderation and warnings, but not the executor gate needed to prevent financial `send`, `withdraw`, or claim-link final actions.

The issue reports that the Nova skill executed `nova send 95 -t` to create a claim link without the documented network, amount, destination, and explicit user confirmations.

## Best Possible Solution

Keep Nova visible with Review warnings in ClawHub, and enforce a runtime approval gate before financial final actions such as `send`, `withdraw`, and claim-link creation.

## Reproduction Assessment
