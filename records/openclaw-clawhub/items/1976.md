## Summary

Keeping this open: current ClawHub behavior intentionally leaves nova in the suspicious/review-required bucket, while the reported failure is an enforceable runtime approval-boundary problem for irreversible financial commands.

## What This Changes

The issue reports that nova `send` ran without the required network, amount, destination, and explicit user confirmations despite the skill guardrails.

## Best Possible Solution

Keep nova review-flagged in ClawHub while runtime maintainers design and enforce a pre-invocation approval gate for final financial actions.

## Reproduction Assessment

No, not against current ClawHub main. The trace and published SKILL.md are concrete, but reproducing the failure requires an OpenClaw runtime session and a nova financial action outside this repository.

## Solution Assessment

No for a SKILL.md-only rewrite. The guardrails already exist, so the maintainable fix is runtime enforcement before `send`, `withdraw`, or claim-link creation can be invoked.
