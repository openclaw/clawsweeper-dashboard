## Summary

Keeping this open because current maintainer triage and current source still point to an unresolved security-sensitive runtime approval-boundary issue. ClawHub intentionally leaves nova visible but suspicious/review-required; it does not enforce the confirmations before live financial commands run.

## What This Changes

The issue reports that nova’s financial-action guardrails were skipped when an agent executed `nova send 95 -t` without network, amount, destination, and explicit user confirmations.

## Best Possible Solution

Keep nova flagged in ClawHub and implement an enforceable OpenClaw/runtime approval gate for financial final actions before any live `send`, `withdraw`, or claim-link creation.

## Reproduction Assessment

No. The report includes a concrete trace and the live SKILL.md confirms the required guardrails, but this checkout does not contain a safe OpenClaw runtime harness to execute nova financial commands without moving funds.

## Solution Assessment

No for a SKILL.md-only wording change. The trace already skipped existing guardrails, so the maintainable fix is runtime-enforced approval while ClawHub continues to show review-required warnings.
