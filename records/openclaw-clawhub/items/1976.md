## Summary

Keeping this open because the maintainer discussion and current source evidence still point to an unresolved security-sensitive runtime approval-boundary issue. ClawHub currently surfaces nova as suspicious/review-required, but this repository does not enforce the required confirmation sequence before a live financial command runs.

## What This Changes

The issue asks for nova financial actions such as `send` and `withdraw` to require network, amount, destination, and explicit user confirmation before execution.

## Best Possible Solution

Keep nova flagged as review-required in ClawHub and route the durable fix to the OpenClaw runtime so financial FINAL ACTION commands require an enforceable approval gate before execution.
