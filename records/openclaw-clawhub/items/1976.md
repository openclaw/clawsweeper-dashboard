## Summary

Keeping this open: ClawHub currently leaves nova flagged as suspicious/review-required, while the remaining problem is an enforceable runtime approval boundary for irreversible financial commands outside the ClawHub marketplace code path.

## What This Changes

The issue reports that the nova skill executed `nova send 95 -t` to create a claim link without the documented network, amount, destination, and explicit user confirmations.

## Best Possible Solution

Keep nova flagged with ClawHub review warnings and implement an enforceable OpenClaw runtime approval gate before `send`, `withdraw`, or claim-link creation can run.

## Reproduction Assessment

No, not against current ClawHub main. The trace and published SKILL.md semantics are concrete, but reproducing the failure requires an OpenClaw runtime session and a nova financial action outside this repo.

## Solution Assessment

No for a SKILL.md-only rewrite. Current published docs already require confirmation, so the maintainable fix is runtime enforcement before the CLI command is invoked.
