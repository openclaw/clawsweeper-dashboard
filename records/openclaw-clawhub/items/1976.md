## Summary

Keeping this open because the current discussion and source review point to an unresolved security-sensitive runtime approval-boundary issue. ClawHub is correctly keeping nova visible but suspicious/review-required; that warning state does not enforce the confirmation sequence before a live financial command runs.

## What This Changes

The issue reports that `nova send` executed immediately despite SKILL.md requiring network, amount, destination, and explicit user confirmations before `send` or `withdraw`.

## Best Possible Solution

Keep nova flagged as review-required in ClawHub and implement an enforceable runtime approval gate for financial final actions before `send`, `withdraw`, or claim-link creation can execute.

## Reproduction Assessment

No. The reported trace and published nova SKILL.md are concrete, but current ClawHub main has no nova runtime execution path or safe financial-action harness to reproduce the bypass directly.

## Solution Assessment

No for a SKILL.md-only wording change. The maintainable fix is runtime enforcement of explicit approval before financial final commands, with ClawHub continuing to surface the suspicious/review-required state.
