## Summary

Keep open: maintainer discussion and current source both point to a security-sensitive runtime approval-boundary problem, while ClawHub main only provides registry, scan, moderation, and install-warning surfaces for Nova.

## What This Changes

The issue reports that the Nova skill let an agent run a financial claim-link send command without checking the network or getting explicit user confirmation.

## Best Possible Solution

Keep ClawHub surfacing Nova with Review warnings, and enforce explicit runtime approval for financial final actions such as `send`, `withdraw`, and claim-link creation before treating this class of skill as safe.

## Reproduction Assessment

No. The report gives a concrete trace and the live Nova SKILL.md still defines the required confirmations, but this ClawHub checkout does not contain the runtime executor that ran `nova send`.

## Solution Assessment

No. Stronger SKILL.md wording may help, but the reported trace already ignored written guardrails; runtime-enforced approval is the narrower maintainable fix for irreversible financial actions.
