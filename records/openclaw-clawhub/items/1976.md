## What This Changes

The issue reports that the Nova skill executed `nova send 95 -t` to create a claim URL without the network and amount/destination confirmations required by its SKILL.md.

## Best Possible Solution

Enforce a runtime approval gate before financial final actions such as `send`, `withdraw`, and claim-link creation, while keeping ClawHub scanner/listing signals aligned with the remaining risk.

## Reproduction Assessment

No. The report includes a concrete trace and the current Nova SKILL.md still documents the required confirmations, but current ClawHub main has no Nova executor path and ClawScan is explicitly artifact-based.

## Solution Assessment

No. Stronger SKILL wording and `--dry-run` documentation are useful guardrails, but the reported failure already ignored written instructions; runtime-enforced approval is the narrower durable fix.
