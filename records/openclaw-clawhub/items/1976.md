## Summary

Keep open: this is a fresh, safety-sensitive issue with a concrete trace, and current evidence still shows Nova's non-idempotent financial command path depends on advisory skill guardrails while ClawHub only surfaces artifact-based scan warnings rather than enforcing the sequence.

## What This Changes

The issue reports that an agent ran `nova send 95 -t` to create a claim URL before the Nova skill's required network, amount, and destination confirmations.

## Best Possible Solution

Maintain this as a safety-tracking issue until maintainers choose the enforcement boundary for final financial commands and ensure Nova's published guidance or runtime approval path matches that boundary.

Yes for triage: the issue gives a concrete turn trace, and the current public Nova `SKILL.md` confirms the guarded, non-idempotent command path. Do not reproduce with a live send outside an approved dry-run or sandbox flow.

## Solution Assessment

Unclear: stronger `SKILL.md` wording may help, but the reported failure happened after the agent read existing guardrails, so durable mitigation may require policy or runtime approval enforcement rather than documentation alone.
