## Summary

Keeping this open because the report describes a security-sensitive runtime approval-boundary failure, and maintainer triage already confirmed ClawHub should keep nova flagged rather than treating it as clean. Current ClawHub code can surface review warnings and static findings, but it does not enforce the pre-send approval sequence at command execution time.

The issue reports that the nova skill allowed `nova send 95 -t` to run before the required network, amount, destination, and explicit user confirmations.

## Best Possible Solution
