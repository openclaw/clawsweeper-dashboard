## Summary

Keep open. Current main still advertises and configures onlycrabs.ai as the SoulHub/SOUL.md host, while live checks show the domain timing out on HTTPS or serving Namecheap/Parking responses; ClawHub policy also keeps issues open for maintainer follow-up unless a PR is certainly implemented on main.

## What This Changes

The issue reports that the README advertises onlycrabs.ai as the SOUL.md registry even though the domain resolves to a parking/unreachable page, and suggests removing that section.

## Best Possible Solution

The end state should be a consistent SoulHub public contract: onlycrabs.ai either resolves to the production SoulHub experience with smoke coverage, or repository docs, defaults, tests, and user-facing copy stop presenting it as a live registry until the domain is connected.
