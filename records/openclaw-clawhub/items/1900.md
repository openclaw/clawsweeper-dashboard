## Summary

Keep open. Current main still treats user-initiated deletion as irreversible and blocks same-GitHub sign-in, while the linked #907 remains open for the broader re-signup loop; this specific request also needs maintainer-verified production account handling.

## What This Changes

The issue asks maintainers to restore or purge a deleted ClawHub account so the same GitHub identity can sign in again.

## Best Possible Solution

Handle the immediate request as verified account recovery/support, then use #907 for the broader product and security decision on whether deleted GitHub identities can ever be restored or re-created.

## Reproduction Assessment

Yes. Current-main source and tests give a high-confidence static reproduction: a deactivated or legacy deleted user passed through handleDeletedUserSignIn is rejected instead of restored or recreated.

## Solution Assessment

No patch is proposed here. Manual purge or restore may solve this user’s support case, but the maintainable product solution needs an explicit deletion/relink policy and security review before code changes.
