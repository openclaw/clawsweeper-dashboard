## Summary

Keep open. The report matches current-main deleted-account behavior: self-deletion writes irreversible deactivation state, later GitHub OAuth is rejected, and the broader same-GitHub re-registration policy is still tracked in #907.

## What This Changes

The issue requests restoring or purging residual data for a deleted GitHub-linked ClawHub account so the same GitHub identity can log in or register again.

## Best Possible Solution

Maintainers should verify control of the GitHub identity and choose an explicit production restore or purge path if policy allows, while leaving any general re-registration behavior to the #907 policy track.

## Reproduction Assessment

Yes. Current main reproduces this at the source level: deleteAccount marks the user deactivated/purged and clears identity fields, then later GitHub OAuth calls handleDeletedUserSignIn, which rejects deactivated users.

## Solution Assessment

Not applicable as a patch decision. The immediate solution is verified maintainer support; any code change should be a deliberate deletion-policy/security change with focused tests.
