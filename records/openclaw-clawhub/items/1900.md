## Summary

Keep open. Current main still implements account deletion as irreversible and rejects same-GitHub sign-in after deletion, the related re-signup report #907 is still open, and this ClawHub issue needs maintainer support/security judgment rather than automatic cleanup.

## What This Changes

The issue asks maintainers to restore or purge residual account data so a deleted ClawHub account can log in or re-register with the same GitHub account.

## Best Possible Solution

Treat this as verified account-recovery support now, and use #907 for any broader product/code change to deleted-account re-registration policy.

## Reproduction Assessment

Yes. A high-confidence reproduction path is visible in source and tests: delete an account to set `deactivatedAt`/`purgedAt`, then attempt GitHub sign-in or CLI auth with the same identity and `handleDeletedUserSignIn` rejects it.

## Solution Assessment

Unclear for automation. A maintainer-verified restore or purge is safer than a source-only automated fix because the correct outcome depends on account ownership, published asset ownership, and deletion-policy decisions.
