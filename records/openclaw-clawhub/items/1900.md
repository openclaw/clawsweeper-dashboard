## Summary

Keep open for maintainer account-recovery handling. Current main intentionally blocks reauth for self-deleted GitHub accounts and documents deletion as irreversible, while the broader same-GitHub re-signup policy remains open in #907.

## What This Changes

The reporter asks maintainers to restore or purge a self-deleted ClawHub account so the same GitHub identity can log in again and manage previously published skill/plugin content.

## Best Possible Solution

Maintainers should verify the reporter controls the GitHub identity, then choose an approved production restore or purge path; any durable same-GitHub re-registration behavior should be handled through #907 with explicit deletion-policy and security review.

## Reproduction Assessment

Yes for the source behavior: a users document with deactivatedAt reaches handleDeletedUserSignIn during GitHub OAuth and throws the deleted-account message, with regression coverage. The reporter-specific production account state and ownership verification cannot be reproduced from checkout alone.

## Solution Assessment

Not applicable as a patch decision. Verified maintainer support is the immediate path; allowing self-service same-GitHub re-registration or relinking would be a deliberate account deletion policy change tracked by #907.
