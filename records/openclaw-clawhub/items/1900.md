## Summary

Keep open. Current main intentionally blocks deleted or deactivated GitHub-linked accounts from normal sign-in, and this account-specific restore/purge request needs verified operator handling plus an explicit product/security decision rather than cleanup closure.

## What This Changes

A user asks for their deleted ClawHub account to be restored or for residual GitHub-linked data to be purged so the same GitHub account can sign in again on the website and CLI.

## Best Possible Solution

Verify GitHub ownership and the production account row, then choose and document whether deleted GitHub identities are support-only, restorable, unlinkable, or allowed to re-register, with audited operator tooling or code changes to match that policy.

## Reproduction Assessment

Yes, at source level. Deletion writes deactivatedAt/purgedAt, OAuth rejects that state, active-user helpers hide the user, and CLI token creation requires an active user; the reporter's exact production row still needs operator verification.

## Solution Assessment

No autonomous source-only fix is the best next step. The maintainable path is verified operator handling plus an explicit product/security decision about whether deleted GitHub identities may be restored, unlinked, recreated, or support-only.
