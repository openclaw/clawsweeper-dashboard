## Summary

Keep open. Current main still blocks deleted or deactivated GitHub identities from becoming active users again, this is part of an active account-recovery cluster, and the ClawHub cleanup policy keeps issues open for maintainer product/security handling rather than auto-closing them.

## What This Changes

The issue requests restoration or residual-data purge so the same GitHub identity can use ClawHub again after deleting an account.

## Best Possible Solution

Maintainers should choose and document one deletion-recovery policy, then enforce it consistently across OAuth, active-user checks, CLI token creation, operator tooling, specs, docs, and regression tests.

## Reproduction Assessment

Yes, at source level. Deletion writes deactivatedAt/purgedAt, OAuth rejects deleted or deactivated users, active-user helpers hide them, and the CLI token path requires an active user; the reporter's exact production row still needs operator verification.

## Solution Assessment

No, not as a one-off purge or silent restore. The maintainable solution needs verified operator handling plus an explicit product/security decision about whether deleted GitHub identities may be restored, unlinked, recreated, or support-only.
