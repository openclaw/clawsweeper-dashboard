## Summary

Kept open: current main intentionally blocks re-auth after user-initiated deletion, and this account-specific recovery request needs maintainer support and policy handling rather than conservative cleanup.

## What This Changes

Reporter asks maintainers to purge residual GitHub-linked data or restore their deleted ClawHub account so the same GitHub login can work again.

## Best Possible Solution

Maintainers should handle this through a verified account-support path, then either perform a production restore/purge or document that deleted accounts cannot be recovered while keeping web and CLI errors explicit.

Unclear. Restore or purge would change or bypass the current irreversible-deletion policy, so the narrowest maintainable path needs maintainer verification and an explicit account policy decision.
