## Summary

Keep open. Current main still blocks deleted/deactivated GitHub identities from becoming active users again, and this reporter-specific restore-or-purge request is security-sensitive production account recovery rather than conservative cleanup.

## What This Changes

The issue asks to restore or purge a self-deleted ClawHub account so the same GitHub identity can sign up or log in again.

## Best Possible Solution

Maintainers should verify the requester and production state, handle any approved restore or purge through an audited operator process, and keep the durable same-GitHub deletion recovery policy tracked with https://github.com/openclaw/clawhub/issues/907 and the draft RFC.

## Reproduction Assessment

Yes, at source level: deletion writes `deactivatedAt`/`purgedAt`, OAuth rejects that state, active-user helpers hide the account, and CLI token creation requires an active user. Reporter-specific production state was not inspected.

## Solution Assessment

No automatic source-only fix is the best next step. A verified one-off restore or purge may be appropriate, but durable same-GitHub recovery needs maintainer policy approval, auditability, and safeguards for banned or abuse-disabled accounts.
