## Summary

Keep open. Current main still blocks deleted GitHub-linked accounts from becoming active users again, and the requested restore, purge, or re-registration path needs account-specific operator verification plus a product/security policy decision.

## What This Changes

The user requests restoring or purging a deleted ClawHub account so the same GitHub identity can log in or register again and recover access to published skill/plugin ownership.

## Best Possible Solution

Verify GitHub ownership and production user/provider rows, then choose one documented policy for support-only irreversible deletion, audited restore, safe relink, or clean re-registration, and apply it consistently across OAuth, active-user gates, CLI, UI, docs, and tests.

Yes, at source level. Deletion writes deactivatedAt/purgedAt and revokes tokens, OAuth rejects that state, active-user helpers hide the user, and CLI token creation requires an active user; the exact production account still needs operator verification.

## Solution Assessment

No autonomous source-only fix is the best next step. The maintainable path is verified operator handling plus an explicit product/security decision about restore, relink, recreate, or support-only deletion semantics.
