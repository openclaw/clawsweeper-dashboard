Keep open. Current main still blocks same-GitHub login after user-initiated deletion, and this reporter-specific restore or purge request needs identity-sensitive maintainer/operator handling plus a product and security decision.

## What This Changes

The issue asks maintainers to restore or purge a deleted GitHub-linked ClawHub account so the reporter can log in or register again.

## Best Possible Solution

Maintainers should verify GitHub ownership and production account state, then choose and document whether deleted GitHub identities are support-only, restorable, unlinkable, or allowed to re-register, with audited operator tooling or code changes to match that policy.

## Reproduction Assessment

Yes, at source level. Deletion writes deactivatedAt/purgedAt, OAuth rejects deleted/deactivated users, active-user helpers return no user, and CLI token creation requires an active user; the reporter's exact production row still needs operator verification.

## Solution Assessment

No autonomous source-only restore or purge is the best next step. The maintainable path is verified operator handling plus an explicit product/security decision about whether deleted GitHub identities may be restored, unlinked, recreated, or support-only.
