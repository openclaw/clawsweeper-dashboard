## Summary

Keep open. This is a reporter-specific production account recovery or purge request, and current main still intentionally blocks deleted/deactivated GitHub-backed users from normal sign-in while the broader same-GitHub re-signup policy remains unresolved in open related issues.

## What This Changes

The issue asks for production account recovery or purging residual GitHub-linked data after a deleted ClawHub account can no longer log in or re-register via GitHub.

## Best Possible Solution

Verify the reporter’s production identity and account ownership through an approved operator process, handle this account case accordingly, and settle the broader deletion/re-registration policy in the open account-recovery cluster before changing source behavior.

## Reproduction Assessment

Yes, at source level: deletion writes deactivatedAt/purgedAt, OAuth rejects that state, active-user helpers return no user, and CLI token creation requires an active user. The reporter-specific production account state was not inspected.

## Solution Assessment

Unclear as an implementation path: a verified restore or purge may be the right support outcome for this user, but allowing same-GitHub recovery or re-registration changes the current irreversible-deletion contract and needs maintainer security/product approval.
