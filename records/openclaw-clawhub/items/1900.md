## Summary

Keep open. Current main still blocks deleted/deactivated GitHub identities from becoming active users again, and this account-specific restore or purge request needs verified operator handling plus product/security policy rather than cleanup closure.

## What This Changes

The issue asks for a deleted ClawHub account linked to a GitHub identity to be restored or purged so the same GitHub account can sign in again.

## Best Possible Solution

Maintainers should verify GitHub ownership and production account state, then choose and document whether deleted GitHub identities are support-only, restorable, unlinkable, or allowed to re-register, with audited operator tooling or code changes to match that policy.

## Reproduction Assessment

Yes, at source level. Deletion writes deactivatedAt/purgedAt, OAuth rejects that state, active-user helpers hide the user, and CLI token creation requires an active user; the reporter's exact production row still needs operator verification.

## Solution Assessment

No autonomous source-only fix is the best next step. The maintainable path is verified operator handling plus an explicit product/security decision about whether deleted GitHub identities may be restored, unlinked, recreated, or support-only.
