## What This Changes

The issue requests account restoration or purging residual GitHub identity data after deleting a ClawHub account and becoming unable to sign in again via the website or CLI.

## Best Possible Solution

Verify GitHub ownership and the production account row, then choose and document whether deleted GitHub identities are support-only, restorable, unlinkable, or allowed to re-register, with audited operator tooling or code changes to match that policy.

## Reproduction Assessment

Yes, at source level. Deletion writes deactivatedAt/purgedAt and revokes tokens, OAuth rejects that state, active-user helpers hide the user, and CLI token creation requires an active user; the reporter's exact production row still needs operator verification.

## Solution Assessment

No autonomous source-only fix is the best next step. The maintainable path is verified operator handling plus an explicit product/security decision about whether deleted GitHub identities may be restored, unlinked, recreated, or support-only.
