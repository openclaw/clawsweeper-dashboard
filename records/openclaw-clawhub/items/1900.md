## What This Changes

## Best Possible Solution

Maintainers should verify GitHub ownership and production account state, then choose a documented recovery policy and handle any approved restore, unlink, recreate, or refusal through an audited operator path.

No autonomous source-only restore or purge is the best next step. The maintainable path is verified operator handling plus an explicit product/security decision about whether deleted GitHub identities may be restored, unlinked, recreated, or support-only.
