Adds a static moderation reason code, scanner logic, regression tests, and a security-moderation spec note for extreme blank-line or same-line whitespace padding before later artifact content.

Land a version that emits context-padding evidence while preserving the documented non-hard-block behavior for static findings, with focused coverage for both archive publish and GitHub-backed verification/install paths.

## Reproduction Assessment

## Solution Assessment

No. The detection itself is useful, but the implementation should either keep this as ClawScan context on every path or explicitly document and test a maintainer-approved hard-block policy.
