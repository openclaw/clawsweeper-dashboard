## Summary

The detector is sound for uploaded packages, but the GitHub compatibility adapter turns the only persisted result back to `clean`; the new finding is then not retained for ClawScan or reviewers. This PR should stay open for a narrow evidence-retention repair.

## What This Changes

Adds a static moderation finding for extreme whitespace padding and maps padding-only GitHub verification results to clean so existing installs remain available.

## Best Possible Solution

Persist GitHub verification reason codes/findings separately from the installability-driving status, keeping padding-only status compatible while exposing the evidence to ClawScan and moderation review.

## Reproduction Assessment

Yes at source level: a padding-only GitHub scan enters the new conversion helper, becomes `clean`, and only that converted status reaches the persistence mutation. The PR test confirms installability but does not assert retained evidence because no such persistence exists.

## Solution Assessment

No, not yet; detection in the shared scanner is appropriate, but the GitHub path needs a separate non-blocking evidence channel instead of erasing the result.
