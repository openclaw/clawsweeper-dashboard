## Summary

Current main lacks this useful context-padding detection. The detector is focused and demonstrated, but GitHub-backed verification converts its only suspicious result to clean without persisting the finding, preventing that ingestion path from providing the review evidence described by the PR.

## What This Changes

The PR adds an extreme context-padding static moderation reason and detector, tests and documents it, and maps padding-only GitHub-backed verification results to clean to preserve installability.

## Best Possible Solution

Keep padding non-blocking for GitHub installs, but persist or forward its reason code and finding independently from the clean installability status, with regression coverage proving both outcomes.

## Reproduction Assessment

Yes. Focused fixtures reproduce both extreme blank-line and same-line whitespace padding, and the PR body shows redacted live scanner output with the new finding.

## Solution Assessment

No, not fully. Deterministic non-blocking evidence is the right mechanism, but installability status and security evidence should be represented separately instead of discarding the finding during GitHub-backed verification.
