## Summary

Keep this PR open: the detector is well-scoped and demonstrated, but the GitHub compatibility path converts the new suspicious result to the only persisted `clean` status, so GitHub-backed context-padding evidence is discarded instead of reaching later moderation review.

## What This Changes

Adds a deterministic context-padding moderation reason with regression coverage and maps context-padding-only results to clean for GitHub-backed skill verification.

## Best Possible Solution

Persist the original static status, reason codes, and finding detail for GitHub-backed skills, while deriving install promotion separately so context-padding evidence remains auditable without becoming an unintended standalone hard block.
