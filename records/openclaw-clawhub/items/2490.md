## Summary

Current main does not detect context-padding evasion, and this focused PR adds useful deterministic coverage with convincing runtime proof. The implementation appears correct, but maintainers should explicitly approve the compatibility and trust-boundary choice that padding-only findings remain non-blocking for GitHub-backed installs.

## What This Changes

Adds a static moderation reason for extreme blank-line or whitespace padding, regression coverage, durable moderation intent, and a GitHub-source exception that keeps padding-only findings installable.

## Best Possible Solution

Land the deterministic warning after maintainers confirm the source-specific policy: uploaded artifacts retain the suspicious evidence, while padding alone does not disable GitHub-backed installs and any additional suspicious reason still blocks them.

## Reproduction Assessment

Yes. The source path is deterministic, focused tests exercise the thresholds, and the supplied redacted runtime output shows the new reason on a 2,500-line padded artifact.

## Solution Assessment

Yes, subject to explicit policy approval. A high-threshold static finding is the narrowest maintainable detector, and the all-reasons check preserves blocking for mixed suspicious results while avoiding an unintended GitHub install regression.
