## Summary

Keep this PR open: the detector is useful and the submitted proof is sufficient, but the branch still needs maintainer policy alignment because the new static `suspicious.*` code can flow into GitHub-backed install blocking while the PR and moderation spec frame it as ClawScan-only context.

## What This Changes

Adds a static moderation reason for extreme blank-line or same-line whitespace padding, wires it into the static scan, adds regression tests, and records the moderation intent in `specs/security-moderation.md`.

## Best Possible Solution

Land the detector only after the GitHub-backed verification/install behavior and thresholds are explicit, with regression coverage for either non-blocking ClawScan context or intentional install blocking.

## Solution Assessment
