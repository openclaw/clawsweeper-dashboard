## Summary

Keep open: this is a protected `security` issue with source-backed evidence against a published ClawHub skill, and it needs explicit security/moderation handling rather than cleanup closure or an automated repo fix.

## What This Changes

The issue reports that the published `0xs4m1337/openclaw-whatsapp` skill's auto-reply worker places untrusted WhatsApp messages and chat history into an OpenClaw agent prompt with authenticated WhatsApp send capability.

## Best Possible Solution

Keep this open for security triage: confirm the published skill report, hide or otherwise moderate the listing if warranted, and coordinate a fixed release with prompt isolation, restricted tool authority, recipient gating, and updated threat-model docs before closing.

## Reproduction Assessment

Yes, source-reproducible: the published worker source and SKILL.md show the reported DM-to-agent-to-send path on version `0.3.0`. I did not run a live WhatsApp account probe, so this is artifact-level reproduction rather than an end-to-end exploit demonstration.

## Solution Assessment

Unclear for the exact code fix: the suggested mitigations are directionally sound, but the maintainable solution needs security/moderation judgment and likely publisher-side changes rather than a ClawHub source cleanup PR.
