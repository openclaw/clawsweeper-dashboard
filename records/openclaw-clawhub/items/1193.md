## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only allows auto-closing pull requests that are certainly implemented on current main. The current source still permits a valid API-token user to show up as `unknown` when the user handle is null, and publish still has a `Personal publisher not found` guard if personal-publisher resolution fails; the scheduled digest-sync change improves one reported backend path but does not prove the reporter's live account and publisher rows are repaired.

## What This Changes

The issue reports that `clawhub login` completes, but `whoami` resolves to unknown/null and publishing fails because the user's personal publisher cannot be found.

## Best Possible Solution

ClawHub should guarantee that every authenticated API-token user has a canonical GitHub handle and an active personal publisher before login, `whoami`, or publish can appear successful, then repair existing bad production rows and add regression coverage for null/wrong-handle users across CLI and web publish flows.
