## Summary

Keep open. Current main still allows the reported API-token identity path to return a missing handle as `unknown`, and publish still depends on personal-publisher resolution; the digest pagination path looks improved, but the reporter's live handle/publisher state is not proven repaired.

## What This Changes

The issue reports that ClawHub login can succeed while `whoami` has no GitHub handle and publishing fails because no personal publisher is resolved.

## Best Possible Solution

Guarantee that authenticated API-token users have a canonical GitHub handle and active personal publisher before token creation, `whoami`, or publish can appear successful, then repair existing bad rows and add regression coverage.

## Reproduction Assessment

Yes for the source-level failure mode: a valid API token tied to a user with no handle can produce a null-handle `whoami` response that the CLI displays as `unknown`, and publish still has the personal-publisher guard.
