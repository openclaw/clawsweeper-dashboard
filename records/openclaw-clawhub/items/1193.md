Keep open. The digest pagination sub-error has a merged fix, but current main still has a source-level path where a valid API-token user with no canonical handle is accepted by whoami and displayed as `unknown`, and the reporter's live identity/publisher rows still need maintainer or ops triage.

## What This Changes

The issue reports successful ClawHub login followed by `whoami` showing `unknown` and publishing failing because the personal publisher cannot be found.

## Best Possible Solution

Enforce the invariant that every successful login, whoami, and publish path has a canonical GitHub handle plus active personal publisher, then repair affected production rows and add regression coverage for null or wrong-handle users.

## Solution Assessment

No. The merged digest fix helps one reported sub-error, but the durable fix is to enforce and repair the identity plus personal-publisher invariant at the auth/token/whoami/publish boundaries.
