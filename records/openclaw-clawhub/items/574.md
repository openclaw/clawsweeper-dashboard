## Summary

Current ClawHub no longer emits the reported legacy `openclawmp install experience/@user/name` command, but the replacement install identifier is not yet settled: main copies a slug-only OpenClaw command while the open owner-scoping work documents ambiguity for duplicate publisher slugs. Under ClawHub’s conservative cleanup policy, this should remain open and be tracked with the canonical owner-scoping work rather than declared fully fixed.

## What This Changes

Make the package-page copy button produce a valid install command for the displayed package instead of a nonexistent author-based target that returns 404.

## Best Possible Solution

Use the owner-scoping work to define one backward-compatible install target contract, then make every skill page display and copy that same unambiguous command with tests for personal, organization-owned, legacy slug-only, and duplicate-slug cases.

## Reproduction Assessment

Yes, at source level: current main’s copy button directly copies the formatter output, and tests establish the current slug-only command shape. The original legacy `openclawmp.cc` hosted page itself is no longer reproducible from this checkout.

## Solution Assessment

Unclear. Removing the nonexistent author-ID target is correct, but the durable fix depends on the active product decision over slug-only versus owner-scoped install identifiers and matching CLI compatibility.
