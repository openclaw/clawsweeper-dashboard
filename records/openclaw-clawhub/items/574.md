## Summary

Keep open: current main no longer shows the originally reported legacy author-ID command, but the install UI still copies bare-slug CLI commands while the owner-scoped install contract remains unresolved in https://github.com/openclaw/clawhub/issues/2348 and https://github.com/openclaw/clawhub/pull/2299.

## What This Changes

The issue asks the experience/skill page copy button to copy an installable command for the current asset instead of a stale or nonexistent author-scoped command.

## Best Possible Solution

Land one backward-compatible owner-scoped install target across the skill page, CLI/API resolver, docs, and tests while preserving unambiguous bare-slug compatibility.

## Reproduction Assessment

Yes, at source level. Current main builds and copies `openclaw skills install <slug>` even when owner data exists, while the CLI still rejects slash-qualified locators; I did not live-reproduce the legacy `openclawmp.cc` page.

## Solution Assessment

No, a UI-only command replacement is not the best complete fix. The durable solution needs a compatible owner-scoped locator contract across UI, CLI/API resolution, ambiguity handling, docs, and tests.
