## Summary

## What This Changes

The issue reports that the experience/skill page copy button produced an invalid install command with a nonexistent author ID, causing install attempts to return 404.

## Best Possible Solution

Define and land one backward-compatible scoped install target across the skill page, ClawHub/OpenClaw CLI resolution, API ambiguity handling, docs, and tests, while preserving unambiguous legacy bare-slug compatibility.

## Reproduction Assessment

Yes, at source level. Current main constructs and tests a copied `openclaw skills install <slug>` command even when owner data is available; the original legacy `openclawmp.cc` page was not live-reproduced from this checkout.

## Solution Assessment

No, a UI-only command replacement is not the best complete solution. The durable fix needs compatible owner-scoped lookup, CLI help, docs, ambiguity handling, and regression coverage before the copied command changes.
