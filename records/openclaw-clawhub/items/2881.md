## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Keep pending GitHub-backed entries visible:** `convex/lib/publicBrowse.ts:56`
  - body: This excludes every skill whose `githubScanStatus` is pending. The GitHub-backed skill spec says new or changed source-backed skills keep their catalog entry visible with `moderationReason: "pending.scan"` while install/update is blocked, so this would make existing source-backed skills disappear from browse/search during verification.
  - confidence: 0.91
- **[P1] Require a real approved version before listing pending skills:** `convex/lib/publicBrowse.ts:49`
  - body: `stats.versions > 1` is not proof that a skill has an approved public release; publish increments this counter for every inserted version while the skill can still be `pending.scan`. A first-time hosted skill with two pending or unapproved versions would pass this gate and appear in public browse/search even though first versions should promote nothing until approval.
  - confidence: 0.9
- **[P1] Restore the digest and fixture types before review:** `convex/skills.ts:6010`
  - body: `skillSearchDigest` does not store `moderationSourceVersionId`, so adding it to this `Pick` makes the branch fail TypeScript, and the new helper tests also use plain string IDs/schema-invalid fixture values. Load the full skill before needing that field or add a deliberate schema/digest path, then fix the fixtures so `bun run ci:types-build` can run.
  - confidence: 0.95
