## Summary

Keep open: current main does not implement the plugin/publisher OG refresh, and the branch still has blocking security and canonical-route issues despite sufficient visual proof.

## What This Changes

The PR refreshes dynamic OG images for skills, packages/plugins, and publisher profiles with shared SVG renderers, new OG routes, metadata helpers, asset copying, and tests.

## Best Possible Solution

Rebase onto current main, target publisher metadata to /user/$handle, restrict avatar loading to trusted public images or a safe proxy boundary, then finish maintainer visual-direction approval.

## Reproduction Assessment

Yes for the blocking security finding by source inspection: query or stored avatar URLs reach fetchImageDataUrl, which only checks http/https before fetching. The visual refresh itself is a feature, so bug reproduction is otherwise not applicable.

## Solution Assessment

No. The renderer direction may be useful, but the implementation should not merge until avatar fetching is constrained and publisher metadata targets the canonical route.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Restrict server-side avatar fetches:** `server/og/fetchImageDataUrl.ts:8-11`
  - body: The new public OG routes pass requester-controlled or stored avatar URLs into this helper, but the helper accepts any http or https URL before fetching it. That creates a server-side fetch path to private or metadata endpoints if they return an image-like response, so avatar loading needs a trusted-host/public-IP guard or safe proxy boundary before merge.
  - confidence: 0.89
- **[P2] Use the canonical publisher profile route:** `src/lib/og.ts:208`
  - body: Current main redirects /p/$handle to /user/$handle, and the spec names /user/$handle as canonical. Returning /p from the metadata helper and adding head metadata to the legacy route means the visible publisher page would miss the new OG metadata and the canonical URL would point at a redirect.
  - confidence: 0.84

## Security Review

Status: needs_attention
