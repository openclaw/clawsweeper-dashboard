## Summary

This PR should stay open: current main does not already contain the plugin/profile OG routes or refreshed shared renderer, but the proposed branch has a blocking server-side image-fetch security issue and is still awaiting visual direction/conflict resolution.

## What This Changes

The branch refreshes dynamic OG images for skills, plugins/packages, and publisher profiles with shared SVG rendering, new OG routes, metadata URLs, assets, and tests.

## Best Possible Solution

Land the OG refresh after the avatar image pipeline is restricted to trusted validated sources or a hardened image proxy, the branch is rebased cleanly, and the maintainer visual direction is settled.

## Reproduction Assessment

Not applicable; this is a feature/visual refresh PR rather than a bug report. The candidate behavior is observable through the PR screenshots and new route code, while current main lacks the plugin/profile OG route files.

## Solution Assessment

No, not as-is; the visual direction may be useful, but the implementation should not merge while public OG routes can fetch arbitrary avatar URLs server-side. A safer solution is to use persisted, validated image hosts or a hardened image proxy with DNS/IP and redirect checks.
