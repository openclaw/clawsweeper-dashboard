## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Constrain OG avatar fetch targets:** `server/og/fetchImageDataUrl.ts:8-18`
  - body: The new public OG routes pass caller-supplied `avatar` values and stored owner images into this helper, which accepts any http(s) URL and fetches it server-side before validation. That creates a public SSRF/private-network probing surface, so render avatars only through a trusted allow-listed proxy/host set or reject private/untrusted targets before fetching.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The diff introduces a concrete SSRF-sensitive path by fetching arbitrary OG avatar URLs from public routes.

Concerns:

- **[high] Untrusted server-side avatar fetch:** `server/og/fetchImageDataUrl.ts:8`
  - body: `fetchImageDataUrl` allows any http(s) URL and is reached from public OG route query parameters, so a request can cause the server to contact internal or otherwise untrusted network targets. Add trusted-host/private-IP protections or route avatar rendering through the existing image proxy pattern.
  - confidence: 0.9
