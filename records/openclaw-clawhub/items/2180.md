## What This Changes

The branch refreshes dynamic OG image generation for skills, plugins, and publisher profiles, including new renderers, routes, metadata helpers, asset copying, screenshots, and tests.

## Best Possible Solution

Land the visual refresh only after rebasing onto current main, moving publisher metadata to the /user profile route, and constraining avatar loading to trusted sources or an approved image proxy boundary.

## Reproduction Assessment

Yes for the blocking review finding by source inspection: public OG routes pass query or stored avatar URLs into a helper that accepts any http(s) URL. The OG refresh itself is a feature, so bug-style reproduction is otherwise not applicable.

## Solution Assessment

No as submitted. The renderer direction is useful, but the merge path should first settle the avatar-fetch security boundary, target the canonical /user publisher route, rebase cleanly, and complete the requested visual-direction review.
