## Summary

Keep open: the PR is fresh and current main does not already include the plugin/profile OG refresh, but the branch is not merge-ready because it is merge-conflicting and introduces a public server-side avatar fetch path that needs hardening.

## What This Changes

The PR adds refreshed dynamic OG image renderers and routes for skills, plugins, and publisher profiles, updates metadata helpers to use them, and adds related assets/tests.

## Best Possible Solution

Keep the refreshed OG direction, but harden avatar fetching and rebase the publisher metadata onto the canonical current-main profile route before merge.

## Reproduction Assessment

Yes for the review blockers by source inspection. The PR routes pass public avatar values into a helper that only checks http(s) before server-side fetch, and current main renders publisher profiles at /user/$handle while the PR adds the new head to /p/$handle.

## Solution Assessment

No. The visual renderer is directionally useful, but the maintainable fix needs a trusted-avatar policy and current-main route reconciliation before this implementation is safe to merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Constrain avatar URLs before server fetches:** `server/og/fetchImageDataUrl.ts:8-11`
  - body: The public OG routes pass `avatar` query values into this helper, and this helper allows any `http`/`https` URL before calling `fetch`. Content-type and size checks happen after the server has connected, so `/og/skill`, `/og/plugin`, and `/og/profile` become SSRF primitives. Restrict this to trusted avatar/CDN hosts or remove query-supplied avatar fetching.
  - confidence: 0.88
- **[P2] Move publisher OG head to the canonical route:** `src/routes/p/$handle.tsx:33-34`
  - body: Current main renders publisher profiles from `/user/$handle` and redirects `/p/$handle` there, but this PR adds the publisher OG head only to `/p/$handle`. After rebasing, the canonical profile page would still miss these tags unless the head moves to the route that actually renders `PublisherProfile`.
  - confidence: 0.78
