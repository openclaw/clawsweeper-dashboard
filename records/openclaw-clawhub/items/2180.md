The branch refreshes dynamic OG images for skills, plugins/packages, and publisher profiles by adding shared SVG renderers, plugin/profile OG routes, extensionless wrappers, metadata helpers, asset copying, and tests.

## Best Possible Solution

Rebase onto current main, constrain OG avatar loading to trusted public image sources or another safe proxy boundary, move publisher metadata and canonical URLs to /user/$handle, then complete the requested visual-direction review.

## Reproduction Assessment

The visual refresh itself is a feature, so bug reproduction is otherwise not applicable.

## Solution Assessment

Is this the best way to solve the issue? No. The renderer direction and proof are useful, but the implementation should not merge until avatar fetching is constrained, publisher metadata targets the canonical route, conflicts are resolved, and the requested visual review is complete.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

  - body: The new public OG routes pass requester-controlled or stored avatar URLs into this helper, but it accepts any http: or https: URL before fetching it.
