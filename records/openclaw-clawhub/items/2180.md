## Summary

This PR should stay open: current main does not already contain the new OG refresh, and the branch introduces a public server-side avatar fetch that accepts arbitrary http(s) URLs, so it needs a security fix plus the already-requested visual direction review before merge.

## What This Changes

The branch refreshes dynamic OG image rendering for skills, plugins/packages, and publisher profiles by adding shared SVG rendering, plugin/profile OG routes, metadata wiring, assets, and tests.

## Best Possible Solution

Land the OG refresh only after hardening remote avatar loading and resolving the maintainer visual direction questions.

## Reproduction Assessment

Yes for the review finding: the PR source shows public OG routes passing `avatar` or stored owner images into a helper that fetches any http(s) URL. I did not run the branch, but the vulnerable path is directly traceable from the diff.

## Solution Assessment

No. The visual refresh may be a good direction, but the remote avatar fetch should reuse a trusted-host allow-list or otherwise block private/internal network targets before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Restrict OG avatar fetches to trusted image hosts:** `server/og/fetchImageDataUrl.ts:7-11`
