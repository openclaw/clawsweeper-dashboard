## Summary

Keep open: current main does not include the plugin/publisher OG refresh, and the PR still has a merge-blocking server-side avatar fetch security issue plus a canonical publisher-route mismatch.

## What This Changes

The PR refreshes dynamic OG image rendering for skills, packages/plugins, and publisher profiles, with new routes, metadata helpers, asset copying, and tests.

## Best Possible Solution

Rebase the branch, constrain OG avatar loading to a safe public-image path, attach publisher metadata to the canonical user profile route, and then finish visual-direction approval.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the blocking security finding by source inspection: public OG routes pass query or stored avatar URLs into an unrestricted HTTP(S) server-side fetch helper. The visual refresh itself is a feature, so bug reproduction is otherwise not applicable.

## Solution Assessment

Is this the best way to solve the issue? No. The renderer direction may be useful, but this implementation should not merge until avatar fetching is constrained and publisher metadata targets the canonical route.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Restrict server-side avatar fetches:** `server/og/fetchImageDataUrl.ts:8-11`
  - body: The new public OG routes pass requester-controlled or stored avatar URLs into this helper, but the helper accepts any http or https URL before fetching it. That creates a server-side fetch path to private or metadata endpoints if they return an image-like response, so avatar loading needs a trusted-host/public-IP guard or safe proxy boundary before merge.
  - confidence: 0.89
- **[P2] Use the canonical publisher profile route:** `src/lib/og.ts:208`
  - body: Current main redirects /p/$handle to /user/$handle, but this helper still returns /p as the publisher canonical URL and the PR adds publisher head metadata to the legacy route. After rebasing, the visible publisher page would miss the new OG metadata and the canonical URL would point at a redirect.
  - confidence: 0.83

## Security Review

Status: needs_attention
