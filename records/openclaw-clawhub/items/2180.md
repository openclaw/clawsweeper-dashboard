## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Restrict server-side avatar fetches:** `server/og/fetchImageDataUrl.ts:8-11`
  - body: The new public OG routes pass query or stored avatar URLs into this helper, but it accepts any http or https URL before fetching it server-side. That creates an SSRF path to private or metadata endpoints when they return an image-like response, so avatar loading needs a trusted-host/public-IP guard or safe proxy boundary before merge.
  - confidence: 0.89
- **[P2] Move publisher metadata to the canonical profile route:** `src/lib/og.ts:208`
  - body: Current main redirects /p/$handle to /user/$handle, but this helper returns /p as the publisher canonical URL and the branch adds head metadata to the legacy route. After rebasing, visible publisher pages would miss the new OG metadata and canonical URLs would point at a redirect.
  - confidence: 0.84

## Security Review
