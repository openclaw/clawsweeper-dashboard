## Summary

Keep this PR open: the OG refresh is not already on current main, the contributor provided useful visual proof, but the patch introduces a public server-side fetch of arbitrary avatar URLs and the PR body still asks for maintainer visual-direction review.

## What This Changes

Refreshes dynamic OG image rendering and metadata for skills, plugins, and publisher profiles, adding shared SVG renderers, extensionless OG routes, assets, and tests.

## Best Possible Solution

Keep the OG refresh open, harden avatar fetching to trusted sources, and merge only after the requested visual-direction decision is settled.

## Reproduction Assessment

Not applicable; this is a feature PR rather than a bug report. I verified the proposed behavior through the PR-linked rendered PNGs and source diff instead of reproducing a failing current-main path.

## Solution Assessment

No; the general OG refresh direction is plausible, but the current implementation should not fetch arbitrary avatar URLs server-side. A safer solution would use stored/trusted avatar sources or reject loopback, private, link-local, and otherwise untrusted hosts before fetching.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Restrict server-side avatar fetches to trusted image hosts:** `server/og/fetchImageDataUrl.ts:8`
  - body: The new public OG routes pass `avatar` query params and owner images into this helper, but the helper fetches any http(s) URL server-side. That lets callers make the renderer request loopback, private, or link-local targets; please allowlist trusted avatar hosts or reject internal/private addresses before fetching.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff introduces an SSRF-style risk by allowing public OG requests to make server-side fetches to arbitrary avatar URLs.

Concerns:

- **[high] Arbitrary server-side avatar fetch:** `server/og/fetchImageDataUrl.ts:8`
  - body: `fetchImageDataUrl` accepts any http(s) URL and is reached from public OG routes via query or profile metadata. Without host validation and private-network blocking, callers can cause server-side requests to internal or otherwise unintended destinations.
  - confidence: 0.86
