## Summary

The reported publisher-identity dead end remains on current main, with no related PR or canonical replacement; because this collaborator-authored issue requires an explicit security-sensitive product decision, it should remain open.

## What This Changes

Allow npm package scopes containing dots or underscores to map to authorized ClawHub publishers without renaming already-published packages.

## Best Possible Solution

Define one canonical npm-scope identity model with normalization, collision, routing, migration, and authorization rules, then implement it consistently across publisher creation, publishing, transfers, installs, moderation, repair tooling, tests, docs, and real Convex validation.

## Reproduction Assessment

Yes at source level: current main requires exact npm scope-to-owner equality while publisher identity remains a single normalized handle, establishing the reported incompatibility. No runtime mutation was attempted because this review is strictly read-only.

## Solution Assessment
