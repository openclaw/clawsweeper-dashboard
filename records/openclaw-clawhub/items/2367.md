## Summary

Keep open: current main still has npm-compatible package names, slug-only publisher handles, and exact scope-to-owner enforcement, so the affected packages remain blocked until maintainers choose an ownership model.

## What This Changes

The issue asks ClawHub to resolve npm-valid scoped package names containing dots or underscores that cannot match current slug-only publisher handles.

## Best Possible Solution

Maintainers should choose one namespace model, then implement it consistently across publisher creation, publish and transfer validation, routing, docs, and regression tests for dot/underscore npm scopes.

## Reproduction Assessment

Yes at source level: package names allow dots/underscores, publisher handles reject them, and the publish path enforces exact scope-owner matching. I did not run a live publish because this is a read-only review.

## Solution Assessment

Unclear until maintainers choose the identity model. Handle expansion, first-class aliases, and forced package renames each have different URL, migration, and authorization tradeoffs.
