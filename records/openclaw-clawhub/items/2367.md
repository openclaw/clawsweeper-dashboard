## Summary

Keep open: current main still rejects publisher handles with dots or underscores while scoped package publishing requires exact scope-to-owner matching, and this collaborator-authored namespace/security decision should stay with maintainers.

## What This Changes

The issue asks ClawHub to support or intentionally map npm-valid package scopes containing dots or underscores so existing scoped plugin packages can publish or transfer under matching publishers.

## Best Possible Solution

Maintainers should choose one namespace model, then update publisher validation or first-class scope aliases with matching publish, transfer, route, docs, tests, and security review coverage.

## Reproduction Assessment

Yes, at source level: current main accepts npm-style package names but rejects matching publisher handles containing dots or underscores, and the existing test suite asserts that failure shape for @foo.bar/demo-plugin.

## Solution Assessment

Unclear until maintainers choose the namespace model; widening handle validation, requiring package renames, and adding aliases are all product/security choices with different compatibility costs.
