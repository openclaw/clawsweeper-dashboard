## Summary

Keep open: current main still preserves the reported mismatch between npm-valid dotted/underscored package scopes and slug-only ClawHub publisher handles, and there is no closing PR or narrower canonical issue that owns the full remaining decision.

## What This Changes

The issue asks ClawHub to decide how npm-valid scopes containing dots or underscores should map to publisher identity, either by broadening handles, requiring package renames, or adding scope aliases.

## Best Possible Solution

Maintainers should choose a single publisher identity model, then implement it consistently across publisher creation, publish and transfer validation, routing, docs, and regression tests for dotted and underscored npm scopes.

## Reproduction Assessment

Yes, at source level: current main accepts package scopes with dots or underscores, rejects matching publisher handles, and enforces exact scope-to-owner matching. I did not run a live publish because this review is read-only.

## Solution Assessment

Unclear until maintainers choose the identity model. Expanding publisher handles and adding first-class scope aliases are both plausible, but either path needs URL, migration, and authorization design before implementation.
