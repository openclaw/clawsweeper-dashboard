Keep open: current main still has the reported mismatch between npm-valid dotted/underscored package scopes, slug-only publisher handles, and exact scope-to-owner publish enforcement, and ClawHub cleanup policy keeps issues open unless a terminal close reason clearly applies.

## What This Changes

The issue asks ClawHub to decide how npm-valid package scopes containing dots or underscores should map to publisher ownership.

## Best Possible Solution

Maintainers should choose one publisher identity model, then implement it consistently across publisher creation, publish and transfer validation, routing, docs, and regression tests for dotted and underscored npm scopes.

## Reproduction Assessment

Yes, at source level: current main accepts package scopes with dots or underscores, rejects matching publisher handles, and enforces exact scope-to-owner matching. I did not run a live publish because this was a read-only review.

## Solution Assessment

Unclear until maintainers choose the identity model. Expanding publisher handles and adding first-class aliases are both plausible, but each needs URL, migration, and authorization design before implementation.
