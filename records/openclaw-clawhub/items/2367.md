## Summary

Keep open: current main still has the reported mismatch between npm-valid package scopes and slug-only publisher handles, and resolving it changes the publisher identity and authorization model.

## What This Changes

The issue requests support or an intentional migration path for scoped npm package names containing dots or underscores that cannot currently map to ClawHub publisher handles.

## Best Possible Solution

Maintainers should choose one publisher identity model, then implement it consistently across publisher creation, publish and transfer validation, routing, docs, and regression tests for dotted and underscored npm scopes.

## Reproduction Assessment

Yes, at source level: current main accepts package scopes with dots/underscores, rejects matching publisher handles, and enforces exact scope-to-owner matching. I did not run a live publish because this was a read-only review.

## Solution Assessment

Unclear until maintainers choose the identity model. Expanding publisher handles and adding first-class aliases are both plausible, but each needs URL, migration, and authorization design before implementation.
