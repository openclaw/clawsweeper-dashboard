## Summary

Keep open: current main still has the reported mismatch between npm-valid dotted/underscored package scopes, slug-only publisher handles, and exact scope-to-owner enforcement. Resolving it changes ClawHub publisher identity and package namespace authorization, so it needs maintainer product and security review rather than an automated cleanup close.

## What This Changes

The issue asks maintainers to support or intentionally reject npm-valid package scopes containing dots or underscores when those scopes must map to ClawHub publishers.

## Best Possible Solution

Maintainers should choose one publisher identity model, then implement it consistently across publisher creation, publish and transfer validation, package routes, docs, and regression tests for dotted and underscored npm scopes.

## Reproduction Assessment

Yes, at source level: current main accepts package scopes with dots/underscores, rejects matching publisher handles, and enforces exact scope-to-owner matching during publish and transfer. I did not run a live publish because this was a read-only review.

## Solution Assessment

Unclear until maintainers choose the identity model. Expanding publisher handles and adding first-class aliases are both plausible, but each needs URL, migration, and authorization design before implementation.
