## Summary

Keep open: current main still rejects dots and underscores in publisher handles while scoped package publishing requires exact scope-to-owner matching, and this collaborator-authored ownership/security contract requires explicit maintainer judgment.

## What This Changes

Decide how npm scopes containing dots or underscores should map to ClawHub publisher identities without weakening package ownership guarantees.

## Best Possible Solution

Define one durable publisher identity contract covering validation, collisions, URLs, package publishing, transfers, moderation, and the four existing packages, then document it in specs and implement it with focused tests plus real Convex runtime validation.

## Reproduction Assessment

Yes, at source level: current main accepts npm-style package names but rejects their matching publisher handles when the scope contains a dot or underscore, and the test suite asserts that failure for @foo.bar/demo-plugin. This read-only review did not run a live Convex mutation.

## Solution Assessment

Unclear until maintainers choose the namespace contract; widening handles, requiring package renames, and introducing first-class aliases have materially different security, routing, and compatibility costs.
