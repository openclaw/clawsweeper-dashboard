## Summary

Keep open: current main still accepts npm package scopes with dots/underscores, but ClawHub publisher handles remain slug-only and publish/transfer paths require exact scope-owner matching. Fixing this changes publisher identity and namespace authorization, so it needs maintainer product/security review rather than cleanup closure.

## What This Changes

Allow npm-valid scoped package names with dot or underscore scopes to map to ClawHub publishers, or choose an explicit alias/rename policy.

## Best Possible Solution

Maintainers should choose one namespace model, then implement it consistently across publisher creation, publish and transfer validation, routing, docs, and regression tests for dot/underscore npm scopes.

## Reproduction Assessment

Yes at source level: package validation accepts npm scope characters like dots and underscores, while publisher handle validation rejects them and publish/transfer paths require exact scope-owner matching. I did not run a live publish because this review was read-only.

## Solution Assessment

Unclear until maintainers choose the identity model. Expanding publisher handles and adding first-class scope aliases are both plausible, but either path needs URL, migration, and authorization design before implementation.
