## Summary

Keep open: current main still has the reported mismatch between npm-valid dotted/underscored package scopes and slug-only publisher handles, and resolving it changes the publisher identity and authorization model.

## What This Changes

The issue asks maintainers to decide how npm-valid scoped package names containing dots or underscores should map to ClawHub publisher identities.

## Best Possible Solution

Maintainers should choose one publisher identity model, then implement it consistently across publisher creation, publish and transfer validation, routing, docs, specs, and regression tests for dotted and underscored npm scopes.

## Reproduction Assessment

Yes, at source level: current main accepts npm-style scopes with dots/underscores, rejects matching publisher handles, and enforces exact package-scope-to-owner matching. I did not run a live publish because this was a read-only review.

## Solution Assessment

Unclear until maintainers choose the identity model. Expanding publisher handles and adding first-class aliases are both plausible, but each needs URL, migration, and authorization design before implementation.
