## Summary

Keep open: the hardening is directionally right, but the PR head still leaves public latest-version metadata validation incomplete and lacks required real behavior proof.

## What This Changes

The PR adds shared moderation and version-ownership guards for public skill artifact reads, tags, digest-backed list metadata, and related tests/spec notes.

## Best Possible Solution

Apply the same availability check to every public `latestVersionId` resolution, add focused regressions for soft-deleted and cross-skill stale pointers, then require redacted live API or CLI proof before merge.

## Reproduction Assessment

Yes. From source inspection on the PR head, a stale `latestVersionId` can still be returned by public resolvers when it is soft-deleted, and `/api/v1/skills/resolve` still bypasses the new ownership/soft-delete helper entirely.

## Solution Assessment

No. The shared helper approach is maintainable, but the PR is not the best complete solution until the helper is applied consistently to public latest-version resolvers and covered by targeted regressions.
