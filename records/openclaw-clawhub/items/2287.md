## Summary

Keep open: current main does not implement this hardening, but this PR still leaves public latest-version paths outside the stated non-deleted/same-skill invariant and only provides test/CI evidence, not live behavior proof. The live PR is also conflicting with current main.

## What This Changes

The PR adds shared guards so moderated skill artifacts, public file routes, package compatibility reads, tags, and digest-backed latest-version summaries reject blocked or cross-skill/deleted versions.

## Best Possible Solution

Keep the hardening direction, rebase onto current main, apply the availability helper to every public `latestVersionId` resolver, preserve current ClawScan wording, add regressions, and require redacted live API or CLI proof.

## Reproduction Assessment

Yes. Source inspection on the PR head shows `getBySlug` only checks ownership before serializing `latestVersion`, and `resolveVersionByHash` still returns a direct `skill.latestVersionId` result.

## Solution Assessment

No. The shared helper approach is maintainable, but it is not complete until public latest-version resolvers all require same-skill and non-deleted versions.
