## What This Changes

Adds shared Convex guards for moderated skill artifact access, same-skill skill-version tag/latest ownership, digest latest-version ownership metadata, regression tests, changelog text, and a moderation spec invariant.

## Best Possible Solution

Apply the same same-skill latest-version guard to the public skill audit feed, add a stale digest regression for that route, and require redacted live API or Convex output showing the blocked paths.

## Reproduction Assessment

Yes, source-reproducible: seed a skillSearchDigest row whose latestVersionId/latestVersionSummary point at another skill and whose latestVersionSkillId differs from skillId, then read api.skills.listAuditPage via the /audits skills feed. The PR head still loads that latestVersionId directly and serializes the other version's audit metadata.

## Solution Assessment

No. The shared helper and digest marker are the right maintainable direction, but the implementation needs to route listAuditPage through the same ownership check and include real after-fix proof before merge.
