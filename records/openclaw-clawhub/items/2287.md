## Summary

Keep open: the PR is useful security hardening, but it is not merge-ready because the latest-version availability guard remains incomplete, proof is only tests/CI output, and the live PR is currently conflicting with main.

## What This Changes

The PR adds shared public skill-file moderation gates, same-skill/non-deleted version validation for tags and latest metadata, digest ownership markers, regression tests, a changelog entry, and a security spec note.

## Best Possible Solution

Use one shared availability guard for every public latestVersion/tag/artifact resolution, keep the digest ownership marker, add regressions for soft-deleted and cross-skill stale pointers, and land with redacted live API/CLI proof.

## Reproduction Assessment

Yes. Source inspection on PR head shows same-skill soft-deleted latestVersionId documents can still pass through getBySlug, and /api/v1/skills/resolve still exposes latestVersion from a direct resolver read.

## Solution Assessment

No. The shared helper direction is maintainable, but the PR is incomplete until the helper is applied consistently to public latest-version resolvers and covered by targeted regressions.
