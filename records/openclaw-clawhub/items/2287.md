Keep open: this is useful security hardening for public skill artifact and tag resolution paths, but the current diff still misses one documented latest-version invariant and needs after-fix real behavior proof before merge.

## What This Changes

This PR adds shared Convex guards for moderated skill-file access and same-skill version resolution, updates API/download routes and tag writes, adds regression tests, and documents the moderation/tag invariant.

## Best Possible Solution

Merge a version that enforces the artifact and version-pointer invariants across public routes, including soft-deleted latestVersionId handling, with focused regression coverage and redacted live API proof.

## Solution Assessment

No, not yet: the overall approach is the right narrow fix, but the branch still needs to reject soft-deleted `latestVersionId` rows in `getBySlug` and add real behavior proof for the changed API paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.85

Full review comments:

- **[P2] Reject soft-deleted latest versions in getBySlug:** `convex/skills.ts:1999`
  - body: The new spec says `latestVersionId` should only resolve to a non-deleted version, but this check only verifies same-skill ownership before passing the row to `toPublicSkillVersion`. If a stale `latestVersionId` points at a soft-deleted version, public skill detail/package compatibility metadata can still expose it as the latest version instead of treating it as missing. Use the availability helper or an explicit `!softDeletedAt` check here and cover that stale-pointer case.
  - confidence: 0.85
