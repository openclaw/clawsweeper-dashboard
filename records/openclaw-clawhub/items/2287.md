## Summary

Keep open: the PR improves the right invariant, but the current PR head still leaves the public skills audit feed dereferencing a digest latest-version ID without proving it belongs to the same skill, and contributor real behavior proof is still absent.

## What This Changes

Adds shared Convex guards for moderated skill artifact routes, same-skill version tags/latest metadata, digest ownership markers, tests, changelog text, and moderation spec text.

## Best Possible Solution

Apply the same same-skill latest-version guard to listAuditPage before loading audit metadata, add a stale digest regression for that route, and require redacted live API or Convex output showing the blocked paths.

## Reproduction Assessment

Yes, source-reproducible: seed a skillSearchDigest row whose latestVersionId/latestVersionSummary point at another skill and latestVersionSkillId differs from skillId, then read api.skills.listAuditPage through the skills audit feed. The PR head still loads that version ID directly and serializes its scan metadata.

## Solution Assessment

No. The shared helper and digest marker are the right maintainable direction, but the PR needs to route listAuditPage through the same ownership check and include real after-fix proof before merge.
