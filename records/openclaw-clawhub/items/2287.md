## Summary

Keep open: the PR hardens direct artifact and tag paths, but the current head still lets public list responses serialize latest-version metadata from digest summaries without same-skill ownership proof, and the contributor proof is tests/CI output only.

## What This Changes

The PR adds shared skill-file access helpers, applies moderation and same-skill version checks to public API/download/package routes, updates regression tests, adds changelog text, and documents the moderation/tag invariant.

## Best Possible Solution

Finish the invariant across public list/digest latest-version serialization, add a regression for stale cross-skill digest rows, and include redacted live API or Convex output showing the blocked paths.

## Reproduction Assessment

Yes, source-reproducible rather than live-run: a stale skillSearchDigest row with cross-skill latestVersionId/latestVersionSummary can still feed item.latestVersion into public list responses on the PR head.

## Solution Assessment

No. The shared helper and direct route guards are the right direction, but the PR should also verify or clear list/digest latestVersion output and provide real behavior proof before merge.
