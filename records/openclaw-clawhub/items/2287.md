## Summary

Keep this PR open: the hardening is useful, but the live head still leaves public latest-version validation gaps, can blank valid legacy digest-backed list metadata until backfilled, lacks real behavior proof, and currently conflicts with main.

## What This Changes

The PR adds shared moderation and version-ownership guards for public skill artifact reads, tag resolution, digest latest-version metadata, tests, changelog text, and a moderation spec note.

## Best Possible Solution

Use a widen-migrate-narrow rollout for latestVersionSkillId, apply the same ownership plus soft-delete helper to every public latest-version resolver, rebase over current main without reverting ClawScan changes, then add redacted live API or CLI proof.

## Reproduction Assessment

Yes. Source inspection on the live PR head shows getBySlug only checks ownership, /api/v1/skills/resolve still trusts skill.latestVersionId, and digest readers fail closed for legacy rows missing latestVersionSkillId.

## Solution Assessment

No. The shared guard direction is maintainable, but the current patch is not complete or upgrade-safe until validation covers every public resolver and the digest marker has a production-safe migration path.
