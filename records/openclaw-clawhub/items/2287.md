## Summary

Keep open: the PR addresses a real public API hardening gap, but the current head still leaves public latest-version metadata outside the new non-deleted same-skill invariant and lacks contributor-owned live behavior proof.

## What This Changes

The PR adds shared moderation and version-ownership guards for public skill artifact reads, tags, digest-backed latest-version metadata, tests, changelog, and a security spec note.

## Best Possible Solution

Apply the shared availability check to every public latestVersionId resolver, add regressions for soft-deleted and cross-skill stale pointers, then require redacted live API or CLI proof before merge.

## Reproduction Assessment

Yes. Source inspection on the PR head shows getBySlug can still serialize a same-skill soft-deleted latestVersionId, and resolveVersionByHash returns latestVersion from a raw pointer that the public resolve route exposes.

## Solution Assessment

No. The shared helper direction is maintainable, but this patch is incomplete until ownership and soft-delete validation are applied consistently across public latest-version resolvers.
