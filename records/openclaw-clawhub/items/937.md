## Repair Work Prompt

Fix the ClawHub CLI/backend skill fingerprint mismatch reported here and in related reports #169 and #178. Current main can download/extract generated root `_meta.json`, then `clawhub update` hashes that local JSON while `/api/v1/resolve` compares against published `version.files` fingerprints that exclude generated metadata, causing false `local changes (no match)` after install or force-update. Implement one focused fix so local install/update hashing, downloaded zip contents, and registry resolve use the same canonical published file set. Prefer excluding only generated install/download metadata such as root `_meta.json` from installed-skill fingerprints or avoiding extraction for skill installs; do not broadly ignore all root JSON because published JSON files must still be integrity-checked. Add regression coverage for force-update followed immediately by normal update and for preserving detection of a real local edit. Keep changes scoped to the CLI skill hashing/update path and Convex skill download/resolve/publish helpers/tests; do not change unrelated package/plugin download behavior. Add a short changelog/release-note entry only if the repository has an established place for CLI bug-fix notes.
