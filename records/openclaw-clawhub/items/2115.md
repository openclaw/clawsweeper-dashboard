## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Audit the actor who triggers lazy slug release:** `convex/skills.ts:775`
  - body: When a different publisher claims an expired unpublished slug, this audit row records the previous owner as `actorUserId`, even though that owner did not initiate the release. Pass the caller or an explicit system actor into this helper so moderation/history views do not misattribute the slug move.
  - confidence: 0.89
- **[P2] Prevent collisions for released unpublished slugs:** `convex/skills.ts:766-768`
  - body: The release path moves the hidden row to `unpublished-${skill._id}` without checking `skills.by_slug` or reserving the `unpublished-*` namespace. Because users can still publish that prefix, a preexisting row with the computed slug would leave duplicate slugs and make later `.unique()` lookups unsafe; use a non-user-claimable namespace or collision-check/retry before patching.
  - confidence: 0.78
