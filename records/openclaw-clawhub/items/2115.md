## Summary

This PR should stay open: the previous lazy-release audit/collision findings appear fixed in the new head, but the updated reservation helper can still release slugs from rows whose current hide was taken over by moderation/security state, and the external contributor proof is still test-only.

## What This Changes

The branch changes Convex skill merge authorization and slug reservation/release behavior, API/CLI/schema/docs/spec coverage, dev seed fixtures, and dashboard metrics/dropdown cursor UI.

## Best Possible Solution

Land a revised PR that ties unpublished slug release to current owner-unpublish provenance, keeps the docs/specs/API/CLI surfaces aligned, and includes redacted real behavior proof for the merge/delete/publish flows.

## Reproduction Assessment

Yes, at source level: the PR helper returns an explicit reservation timestamp without checking current hide provenance, and nearby moderation paths can update hidden state without clearing that new field. I did not run tests because this review is read-only.

## Solution Assessment

No: the direction is plausible and the earlier lazy-release findings are addressed, but the reservation expiry needs to be constrained to current owner-initiated hides before this is the narrow maintainable fix.

## Review Findings

- **[P2] Require owner-hide provenance before releasing slugs:** `convex/skills.ts:609-610`
