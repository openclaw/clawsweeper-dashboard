## Summary

Keep open only because the ClawHub cleanup policy does not auto-close issues; the reported HTTP API regression is resolved on current main and production.

## What This Changes

The issue reports that `GET /api/v1/skills` returned an empty list after the V4 listing migration because the HTTP handler still called the deprecated empty `skills.listPublicPage` query.

## Best Possible Solution

Keep the V4/digest-backed HTTP listing path, leave this issue for maintainer closure under ClawHub policy, and consider a separate cleanup to remove or further guard the deprecated empty stub.

## Reproduction Assessment

No, not against current main or production. The original curl path is concrete, but the live endpoint now returns non-empty items and current main routes the HTTP handler to `listPublicApiPageV1` or `listPublicTrendingPage`, not the deprecated empty query.

## Solution Assessment

Yes. The shipped fix moves the HTTP API to indexed V4/digest-backed listing queries while leaving the deprecated stub inert, which is narrower and safer than reviving the old listing query.
