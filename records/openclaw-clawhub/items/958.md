## Summary

Keep open under the ClawHub cleanup policy because this item is an issue, not a PR. The reported HTTP API regression no longer reproduces on current main or production: the route uses an active digest-backed list query and the live endpoint returns items.

## What This Changes

The issue reports that `GET /api/v1/skills` returned an empty page after the V4 migration because the HTTP handler called a deprecated empty skills listing query.

## Best Possible Solution

Keep the current digest-backed HTTP listing implementation, then track any deprecated-stub removal as a separate cleanup if maintainers want to reduce future footguns.

## Reproduction Assessment

Yes. The original curl request is still a high-confidence verification path; I ran it against production and it returned non-empty `items`, while current main routes the endpoint through `listPublicApiPageV1`.

## Solution Assessment

Yes. Routing the HTTP handler through an active digest-backed listing query is the narrow maintainable fix for the reported empty response, and the remaining deprecated stub is a separate cleanup concern.
