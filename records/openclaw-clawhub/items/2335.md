## Summary

Keep open: current main and the live site still show the central problem. The public skill API is clear and the scan API is clean, but the skill detail page still renders a top-level `Malicious` security-audit label from a one-engine VirusTotal signal even though repository policy says VirusTotal is telemetry, not public verdict authority.

## What This Changes

The issue asks ClawHub staff to remove the incorrect `Malicious` label from `markdown2doc` v1.0.4 after prior appeal handling did not clear the visible label.

## Best Possible Solution

Make the public audit outcome follow canonical ClawScan/manual moderation while keeping VirusTotal engine counts as visible telemetry, then verify the live `markdown2doc` page no longer shows a top-level `Malicious` label if staff clearance is valid.

## Reproduction Assessment

Yes. The live deployed page still renders `Malicious` in the security-audit sidebar, while the public APIs return clear moderation and clean scan state for `markdown2doc` v1.0.4.

## Solution Assessment

Unclear as a source-only change. The safest path is staff/security verification plus a canonical audit-outcome fix or the already-open ClawScan rollout, not a skill-specific suppression.
