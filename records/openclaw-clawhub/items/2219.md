## Repair Work Prompt

Fix the silent `clawhub package publish` hang reported in https://github.com/openclaw/clawhub/issues/2219. On current main, `cmdPublishPackage` builds `FormData` and awaits `apiRequestForm` for `POST /api/v1/packages`; `createSpinner` is disabled when stdout/stdin are not TTYs, so non-interactive publishes can show no progress while the 120s upload timeout waits. Implement a narrow reliability fix so non-interactive package publishes emit useful progress or diagnostics before the live publish request and stalled multipart publishes fail with a clear bounded error. Add regression coverage that simulates a stalled or timed-out package publish request and asserts the user-visible behavior. Preserve `--json` machine-readable output, preserve skill publish behavior, and do not change package schema semantics or add a config flag. Treat https://github.com/openclaw/clawhub/pull/2235 as adjacent cwd/source-path work only; do not conflate it with this no-output API wait unless it has already landed and affects the same tests.
