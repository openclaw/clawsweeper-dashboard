## What This Changes

The issue reports that a published ClawHub skill routes requester-controlled IPFS query text through LLM-generated API resolution before signing bonded Monad `approve` and `resolveRequest` transactions.

## Best Possible Solution

Have a security/moderation reviewer triage the live listing, decide whether to hide or hold it, and require a publisher update with untrusted-query isolation, static endpoint allowlists, spend limits, and operator confirmation before autonomous bonded writes.
