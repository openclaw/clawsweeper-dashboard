## Summary

Current main and release v0.20.0 implement the central artifact-first enforcement model and exact retry thresholds requested here. This remains open only because the collaborator-authored, security-sensitive policy needs explicit maintainer confirmation rather than automated closure.

## What This Changes

Change malicious scanner enforcement so submitted skill or plugin versions are blocked first, while account bans are reserved for repeated malicious artifacts or abuse.

## Best Possible Solution

A maintainer should confirm that v0.20.0's documented artifact-first thresholds are the permanent policy and either close this issue as implemented or narrow it to any still-missing evasion and abuse escalation rules.

## Reproduction Assessment

Yes. Current source, focused unit tests, and a real local-auth Playwright flow establish candidate blocking, clean-latest preservation, publisher notification, and thresholded account banning with high confidence.

## Solution Assessment

Yes for the central problem. A shared artifact-first policy with explicit, tested thresholds is the narrowest maintainable design; maintainers only need to confirm whether additional abuse and evasion criteria belong in this issue.
