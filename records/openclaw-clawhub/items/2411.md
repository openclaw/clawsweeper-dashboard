## Summary

Adds `GET /api/v1/plugins/{name}/verify` with version/tag/latest release resolution, aggregated plugin trust evidence, an exported response schema, handler tests, HTTP documentation, and a trust-semantics specification.

## Best Possible Solution

Rebase onto current main, preserve the canonical exact-release trust and artifact helpers, define reusable schemas for every public evidence section, and prove clean and failing responses against real seeded Convex package releases before approving the evidence-only semantics.

## Solution Assessment

Unclear. Aggregating canonical release evidence is a reasonable design, but a versioned trust API should not ship with arbitrary nested shapes or unresolved `ok` semantics; rebasing, explicit schemas, and maintainer approval are safer.
