## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Use the public-trust release lookup:** `convex/httpApiV1/packagesV1.ts:3336-3338`
  - body: The verify route starts from `getByNameForViewerInternal`, but current source and tests show that normal package reads return null for blocked public packages while the exact security path still returns the release with `publicDownloadBlocked`. That means a malicious or blocked plugin would return 404 instead of the failed evidence envelope this endpoint is supposed to provide.
  - confidence: 0.9
- **[P1] Keep verification evidence version-scoped:** `convex/httpApiV1/packagesV1.ts:653`
  - body: For a requested `version` or `tag`, `params.pkg.verification` is package-level latest-release metadata, because the publish path only updates package fields when a release becomes latest. Falling back to it here can attach latest provenance to an older selected release that lacks its own verification data, which violates the version-scoped trust contract.
  - confidence: 0.88
