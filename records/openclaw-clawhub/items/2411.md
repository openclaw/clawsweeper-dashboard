Full review comments:

- **[P1] Keep trusted-publisher evidence version-scoped:** `convex/httpApiV1/packagesV1.ts:698`
  - body: `trustedPublisher` is populated from the package's current trusted-publisher row, but the endpoint is version-scoped. If the publisher config is added, changed, or revoked after an older release, `/verify?version=...` will still report the new config under `provenance`, which can make clients treat an artifact as covered by trusted publishing evidence it did not have. Use release-time publish/provenance data here, or clearly move/label this as current package configuration outside release provenance.
  - confidence: 0.86
