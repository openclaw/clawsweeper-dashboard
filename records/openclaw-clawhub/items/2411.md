## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Preserve blocked-package evidence on verify:** `convex/httpApiV1/packagesV1.ts:3336-3339`
  - body: The verify route loads packages through `getByNameForViewerInternal`, but that helper hides packages whose effective scan status is blocked from non-owner viewers. For those public trust cases this route returns `Plugin not found` before it can emit `ok: false`, `security.blockedFromDownload`, and the existing `package:malicious` reason that the package security endpoint deliberately exposes.
  - confidence: 0.91
- **[P2] Keep selected-release provenance version-scoped:** `convex/httpApiV1/packagesV1.ts:653`
  - body: `params.pkg.verification` is the package/latest-release summary, so falling back to it when the selected release lacks verification can show latest-release source/provenance on an older `version` or `tag` response. For a version-scoped trust endpoint, leave release evidence unavailable or resolve exact release metadata instead of reusing the package-level latest summary.
  - confidence: 0.82
