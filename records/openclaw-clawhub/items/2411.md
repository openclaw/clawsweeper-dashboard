- **[P1] Use selected-release verification only:** `convex/httpApiV1/packagesV1.ts:653`
  - body: The endpoint is version-scoped, but `params.release.verification ?? params.pkg.verification` lets an old selected release inherit current/latest package verification when that release has no verification of its own. That can overstate source linkage, provenance, or scan status for `/verify?version=...`; return selected-release verification only here, or move package-current state into a clearly named field.
  - confidence: 0.88
- **[P1] Keep trusted-publisher config out of release provenance:** `convex/httpApiV1/packagesV1.ts:698`
  - body: This nests current package-level `packageTrustedPublishers` data inside the selected release's `provenance` object.   - confidence: 0.88
