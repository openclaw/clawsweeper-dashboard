## Summary

The patch is not merge-ready because it creates a parallel automation-facing trust decision, publishes a materially under-specified versioned schema, lacks required real Convex behavior proof, and needs maintainer approval while the related trust-model RFC remains open.

## What This Changes

Adds `GET /api/v1/plugins/{name}/verify` with version/tag resolution, aggregated trust and provenance evidence, schema exports, mocked handler tests, HTTP API docs, and a trust-boundary specification.

## Best Possible Solution

Expose one canonical, fully structural plugin evidence contract that reuses the existing package install/security decision instead of defining a parallel verdict, clearly separates evidence from endorsement, and is proven through real Convex-backed HTTP requests for clean, blocked, stale, version, and tag cases.
