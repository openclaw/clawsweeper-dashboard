## Summary

This draft contains unique, potentially useful plugin trust work that is not on current main, but it is not merge-ready: the version-scoped response can fall back to package-level verification evidence, the public v1 schema leaves most evidence objects untyped, the branch conflicts with recent package-security work, and the supplied proof only exercises mocked handlers. The unresolved trust-model RFC also makes the permanent public contract a maintainer product and security decision.

## What This Changes

Adds a version-scoped plugin verification evidence endpoint with release selection, trust and provenance aggregation, an exported response schema, handler tests, HTTP API documentation, and a security-boundary specification.

## Best Possible Solution

Rebase onto current main, make every evidence claim strictly release-scoped, reuse the latest canonical install-block and moderation helpers, export explicit nested v1 schemas, and approve the non-endorsement contract before publishing it with real Convex HTTP proof for clean, pending, stale, moderated, and blocked releases.
