## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.96

Full review comments:

- **[P1] Do not label historical releases with current official status:** `convex/httpApiV1/packagesV1.ts:675-679`
  - body: The response is documented as version-scoped, but `review.status` comes from mutable package-level `isOfficial` and `channel`. Promoting a community package later would make every older release return as official without release-scoped review evidence. Name this as current package status or persist and use review state for the selected release.
  - confidence: 0.97
- **[P1] Do not attach current publisher trust to old releases:** `convex/httpApiV1/packagesV1.ts:698`
  - body: The handler fetches the package's current trusted-publisher configuration and places it in the selected release's provenance. Configuration added after an older artifact was uploaded would therefore appear as evidence for that release. Separate current configuration from provenance or emit only publisher evidence captured for the selected release.
  - confidence: 0.94
- **[P2] Define the nested verification schema instead of `unknown`:** `packages/schema/src/packages.ts:446-453`
  - body: The exported `clawhub.plugin.verify.v1` schema leaves every substantive evidence group as `unknown`, so consumers cannot validate or type the documented contract and incompatible nested changes would still pass validation. Add concrete schemas for the nested groups before exposing a versioned API type.
  - confidence: 0.99
