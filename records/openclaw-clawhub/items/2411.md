## Summary

This draft adds a useful evidence aggregation surface that is not present on current main, but it is not merge-ready: the exported v1 schema leaves every substantive evidence section unvalidated, the branch conflicts with current main, and the supplied proof only exercises mocked data. The trust decision also needs explicit maintainer approval because it becomes a public security contract adjacent to the unresolved vetted-community RFC.

## What This Changes

The PR adds `GET /api/v1/plugins/{name}/verify` with version/tag/latest selection, a plugin trust-evidence response, schema exports, handler tests, HTTP documentation, and a durable trust-semantics specification.

## Best Possible Solution

Keep one versioned endpoint that composes the established exact-release trust primitives, but define concrete nested schemas for every evidence section, explicitly approve the pass/fail semantics, rebase onto current main, and demonstrate clean, pending/stale, and blocked releases through a running local ClawHub plus Convex instance.
