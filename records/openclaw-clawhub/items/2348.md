## Summary

Keep open: current main still renders and copies a bare-slug install command, while slash-containing owner-scoped locators are rejected by the CLI path. The requested UI change needs a coordinated locator contract rather than a UI-only patch.

## What This Changes

The issue asks for the skill detail install code block to copy an owner-scoped OpenClaw command such as `openclaw skills install <owner>/<slug>`.

## Best Possible Solution

Define one canonical scoped skill locator contract, preserve bare-slug compatibility only where unambiguous, then update the install UI, CLI/API parser and resolver, filesystem and lockfile behavior, docs, and regression tests together.

## Reproduction Assessment

Yes. Source and tests show the command card renders and copies `openclaw skills install weather` even when `ownerHandle` is supplied, and the CLI normalizer rejects slash-containing locators.

## Solution Assessment

No as a UI-only change. Owner-scoped install commands may be the right direction, but the maintainable fix must coordinate locator parsing, ambiguity handling, API resolution, filesystem and lockfile behavior, docs, and compatibility tests.
