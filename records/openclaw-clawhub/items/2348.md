## Summary

Keep open: current main still renders and copies the compact skill install command as a bare slug, while the intended scoped `@owner/slug` behavior is only in discussion and a broad open PR. The issue is also collaborator-authored, so it should stay on the maintainer path rather than be cleanup-closed.

## What This Changes

Request the skill detail install codeblock and copy command use an owner-scoped skill locator instead of a bare skill slug.

## Best Possible Solution

Align the skill page command, copy behavior, docs, tests, and CLI/resolver contract on the maintainer-selected `@owner/slug` locator while preserving legacy bare-slug compatibility for unambiguous installs.

## Reproduction Assessment

Yes, at source level: current main still routes the CLI codeblock through `formatOpenClawInstallCommand(slug)`, and tests assert `openclaw skills install weather`. I did not run a browser because this review is read-only, but the source/test path is high-confidence.

## Solution Assessment

Unclear as a final implementation until maintainers settle the locator contract. A narrow UI-only edit is not enough because the previous `owner/slug` command was rejected by OpenClaw CLI; the maintainable fix needs CLI/docs/tests/resolver alignment.
