## Summary

Keep open: current main still renders the direct install code block as a bare slug, but changing it safely needs a maintainer decision on the canonical scoped locator and compatibility behavior because today slash locators are rejected by the CLI path.

## What This Changes

Request that the skill detail install code block use an owner-scoped command such as `openclaw skills install <owner-slug>/<skill-slug>`.

## Best Possible Solution

Define one scoped skill locator contract, preserve safe bare-slug compatibility where unambiguous, then update the install UI, CLI/API parser/resolver, docs, and regression tests together.

## Reproduction Assessment

Yes. Source and tests show the command card renders and copies `openclaw skills install weather` even when `ownerHandle` is supplied, so the requested scoped-codeblock behavior is absent on current main.

## Solution Assessment

No as a UI-only change. Owner-scoped install commands are a reasonable direction, but the maintainable fix must coordinate locator parsing, safe filesystem/lockfile behavior, API resolution, docs, and compatibility tests.
