## Summary

Keep open: current main still renders the skill detail OpenClaw CLI command from the bare slug, while the issue asks for the install codeblock to use the owner-scoped locator. The issue is collaborator-authored and is paired with the open, conflicting broader scoped-slug PR at https://github.com/openclaw/clawhub/pull/2299, so it needs maintainer handling rather than cleanup closure.

## What This Changes

The issue asks for skill detail install code blocks to copy an owner-scoped skill reference instead of a bare slug.

## Best Possible Solution

Align the skill page codeblock, copy action, docs, CLI help, and resolver contract on the maintainer-selected `@owner/slug` locator while preserving a clear legacy bare-slug compatibility path.

## Reproduction Assessment

Yes. Source inspection shows the command-line card uses `formatOpenClawInstallCommand(slug)`, and the helper returns the bare-slug install command while owner context is available for the prompt path.

## Solution Assessment

Unclear. Owner-scoped install commands appear to be the right direction, but the durable fix should match the CLI/API compatibility decision rather than changing only the visible codeblock.
