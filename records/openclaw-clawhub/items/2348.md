## Summary

Current main still copies a bare-slug install command. The collaborator author has an active paired PR for owner-scoped identities, but matching OpenClaw CLI support remains unresolved, so this issue is still necessary.

## What This Changes

Change the skill-page install code block to use the canonical owner-scoped skill reference, proposed as `@owner/slug`, with matching CLI help and docs.

## Best Possible Solution

Land a compatible owner-scoped identity contract in ClawHub and the OpenClaw CLI, preserving legacy slug-only behavior where required, then make the skill page, tests, CLI help, and public docs consistently teach `@owner/slug`.

## Reproduction Assessment

Yes. Current main directly shows the command card passing only the bare slug to a formatter that emits the unscoped install command despite owner metadata being available.

## Solution Assessment

Yes in direction, but only as a coordinated ClawHub and OpenClaw CLI contract; changing the code-block text before parser support would recreate the invalid-command problem documented by prior history.
