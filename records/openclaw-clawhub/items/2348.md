## Label Justifications

## Summary

Keep open: current main still renders and copies bare-slug OpenClaw install commands, while owner-scoped install locators remain broader than a UI-only change because the current CLI/API/docs still use slug-only install semantics.

## What This Changes

The issue asks for the skill install code block to copy an owner-scoped command such as `openclaw skills install <owner-slug>/<skill-slug>`.

## Best Possible Solution

Define one canonical scoped skill locator contract, preserve bare-slug compatibility where safe, then update the install UI, CLI/API resolver, docs, and regression tests together.

## Reproduction Assessment

Yes. Source and tests show the command card renders and copies `openclaw skills install weather` even when `ownerHandle` is supplied, while the current CLI rejects slash-containing install locators.

## Solution Assessment

No as a UI-only change. Owner-scoped install commands may be the right direction, but the maintainable fix must coordinate locator parsing, ambiguity handling, filesystem/lockfile behavior, API resolution, docs, and compatibility tests.
