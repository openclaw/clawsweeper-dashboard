## Summary

Keep open. Current main still renders and copies the slug-only OpenClaw CLI command on skill detail pages, while the requested owner-scoped locator remains tied to a maintainer/product decision and the related open implementation PR is still conflicting.

## What This Changes

Request the skill detail install codeblock and copy UI to show an owner-scoped OpenClaw install command, clarified in discussion as `openclaw skills install @owner/slug`.

## Best Possible Solution

Align the skill page codeblock, copy action, docs, CLI help, and resolver contract on the maintainer-selected `@owner/slug` locator while preserving a clear legacy bare-slug compatibility path.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: render `SkillCommandLineCard` with an owner handle and the CLI tab still shows and copies `openclaw skills install weather`. I did not run a browser because this was a read-only cleanup review.

## Solution Assessment

No, a codeblock-only change is too narrow unless the install locator contract moves with it. The safer path is a coordinated `@owner/slug` parser/formatter/docs/UI update or a narrowed version of the related owner-scoped PR.
