## Summary

Keep open: current main still renders and copies a bare-slug install command even when owner data is present, but the current CLI and public docs are also slug-only. The earlier compatibility report at https://github.com/openclaw/clawhub/issues/1807 intentionally moved this UI away from `owner/slug`, so this needs a maintainer product/API decision rather than a UI-only repair.

## What This Changes

The issue requests that the skill install code block show an owner-scoped command such as `openclaw skills install <owner-slug>/<skill-slug>`.

## Best Possible Solution

Define one scoped skill locator contract, preserve safe bare-slug compatibility, then update the install UI, CLI/API parsing and resolution, docs, and regression tests together.

## Reproduction Assessment

Yes. Source and tests show the current command card renders and copies `openclaw skills install weather` even when owner data is supplied; I did not execute tests because this review is read-only.

## Solution Assessment

Unclear as a UI-only fix. Owner-scoped install commands are a reasonable direction, but the maintainable solution is a coordinated locator contract across UI, CLI/API parsing, docs, and compatibility behavior.
