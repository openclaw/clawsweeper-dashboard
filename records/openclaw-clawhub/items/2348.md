- `impact:other`: The concrete impact is misleading or ambiguous copied installation guidance, which does not fit a more specific owned impact category.

## Summary

Current main and v0.20.0 still render and copy a bare-slug OpenClaw install command, so the requested owner-scoped behavior remains necessary. The companion PR https://github.com/openclaw/clawhub/pull/2299 changes this surface but is conflicting and unmerged; because this issue is collaborator-authored and the exact `owner/slug` versus `@owner/slug` contract remains a product and compatibility decision, it should stay open for maintainer review.

## What This Changes

Update the skill-page install code block to copy an owner-scoped `openclaw skills install @owner/slug` command instead of a bare-slug command.

## Best Possible Solution

Adopt one documented owner-qualified skill reference—preferably the planned `@owner/slug` form—across OpenClaw CLI parsing, ClawHub resolution, the install UI, tests, and public docs, while retaining an explicit legacy bare-slug compatibility path.

## Reproduction Assessment

Yes, at high confidence from current source: `src/components/SkillInstallSurface.test.tsx` explicitly asserts that the bare-slug command is displayed and copied. This read-only review did not run a browser, but the deterministic formatter path and regression assertions establish the current behavior.

## Solution Assessment

The owner-scoped direction is appropriate, but changing only the code block is not the best complete fix. Maintainers should first settle the canonical `owner/slug` versus `@owner/slug` syntax and land matching CLI parsing, resolution compatibility, UI, tests, and docs.
