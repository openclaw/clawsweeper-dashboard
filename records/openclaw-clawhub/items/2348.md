## Summary

Keep open. Current main still renders and copies a bare skill slug in the install command card, while the planned scoped `@owner/slug` behavior is only represented in open, conflicting follow-up work at https://github.com/openclaw/clawhub/pull/2299 and this collaborator-authored issue needs maintainer handling.

## What This Changes

Requests that the skill detail install code block and copy action use an owner-scoped skill locator instead of a bare skill slug.

## Best Possible Solution

Land a maintainer-approved `@owner/slug` locator contract across the web install code block/copy action, docs, and compatible resolver behavior, likely by repairing or replacing https://github.com/openclaw/clawhub/pull/2299.

## Reproduction Assessment

Yes. Source inspection shows the current command card calls `formatOpenClawInstallCommand(slug)` and tests assert `openclaw skills install weather`, so the requested scoped command is absent on current main.

## Solution Assessment

Yes, if solved as an aligned locator contract rather than a UI-only string swap. The safer path is to update UI, docs, and resolver/CLI compatibility together so ClawHub does not display unsupported install syntax.
