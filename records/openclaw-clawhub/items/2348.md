## Summary

Keep open: current main still renders and copies the compact skill install command as a bare slug, while the planned scoped `@owner/slug` behavior is only in discussion and the broad related PR is open and conflicting. ## What This Changes

The issue requests that skill install code blocks use owner-scoped skill refs instead of bare skill slugs.

## Best Possible Solution

Align the skill page command, copy behavior, docs, tests, and CLI/resolver contract on the maintainer-selected `@owner/slug` locator while preserving legacy bare-slug compatibility for unambiguous installs.

Yes, at source level: current main renders and copies the compact CLI command via `formatOpenClawInstallCommand(slug)`, and component tests still assert the bare command. I did not run a live browser because this review is read-only.

## Solution Assessment
