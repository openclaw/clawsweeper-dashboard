## What This Changes

The issue asks the skill install code block and copy UI to use an owner-scoped skill reference instead of a bare skill slug.

## Best Possible Solution

Land one scoped locator contract for `@owner/slug` across UI copy, CLI/API resolution, docs, and regression tests while preserving unambiguous bare-slug compatibility.
