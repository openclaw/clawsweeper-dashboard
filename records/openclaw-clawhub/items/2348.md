## Summary

Keep open: current main still renders and copies the skill install command as a bare slug, but switching the code block to an owner-scoped locator safely requires a maintainer decision because the current install path rejects slash locators.

## What This Changes

The issue requests that the skill detail install code block use `openclaw skills install <owner-slug>/<skill-slug>` instead of a bare skill slug.

## Best Possible Solution

Define one scoped skill locator contract, preserve bare-slug compatibility where it is unambiguous, then update the install UI, CLI/API resolver, docs, specs, and regression tests together.

## Reproduction Assessment

Yes. Source and tests show the command card renders and copies `openclaw skills install weather` even when an owner handle is supplied, and the CLI normalizer rejects slash-containing locators.

## Solution Assessment

No as a UI-only change. Owner-scoped install commands are a reasonable direction, but the maintainable fix must coordinate locator parsing, safe filesystem and lockfile behavior, API resolution, docs, and compatibility tests.
