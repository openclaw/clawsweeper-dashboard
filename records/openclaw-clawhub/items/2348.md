## Summary

Keep open. Current main and v0.22.0 still render/copy a bare skill slug for the skill-page CLI command, the issue is collaborator-authored, and the owner-scoped install contract is already under maintainer review in an open conflicting PR.

## What This Changes

The issue asks the skill detail install code block and copy action to use an owner-scoped skill locator instead of a bare slug.

## Best Possible Solution

Land a maintainer-approved owner-scoped install contract that updates the skill-page command, clipboard behavior, tests, docs, and compatible CLI parsing together.

## Reproduction Assessment

Yes. Source inspection reproduces the current behavior: render `SkillCommandLineCard` with an owner handle and slug, and the CLI tab still displays and copies `openclaw skills install weather`.

## Solution Assessment

Unclear as a UI-only change. Owner-scoped copy is the likely direction, but the exact `@owner/slug` contract needs maintainer approval and matching CLI/API resolver support before it is the narrowest maintainable fix.
