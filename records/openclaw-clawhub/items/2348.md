## Summary

Keep open: current main and the latest v0.20.0 release still render and copy bare-slug install commands, while the requested scoped `@owner/slug` contract is not yet implemented and is being handled in an open compatibility-sensitive PR. This issue is also collaborator-authored, so it should not be cleanup-closed without maintainer judgment.

## What This Changes

Request that the skill detail install command UI use owner-scoped skill refs such as `openclaw skills install @owner/slug` instead of bare slugs.

## Best Possible Solution

Land one scoped locator contract for `@owner/slug` across UI copy, CLI/API resolution, docs, and regression tests while preserving unambiguous bare-slug compatibility.

## Reproduction Assessment

Yes. Source inspection shows current main renders and copies `openclaw skills install weather` even when owner props are present; I did not run a live browser flow in this read-only review.

## Solution Assessment

No, not as a codeblock-only copy change. The maintainable path is to align the scoped locator parser, ambiguity rules, docs, and UI tests together.
