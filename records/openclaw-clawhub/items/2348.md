## Summary

Keep open: current main still renders and copies the bare `openclaw skills install <slug>` command, while owner-scoped installs require a coordinated CLI/API/product decision rather than a safe issue-cleanup close.

## What This Changes

The issue requests changing skill install commands to use an owner-scoped locator such as `openclaw skills install <owner-slug>/<skill-slug>`.

## Best Possible Solution

Settle one canonical scoped install locator across OpenClaw, ClawHub API/CLI, docs, and compatibility behavior, then update the skill detail command, copy behavior, and regression tests together.

## Reproduction Assessment

Yes. Current main passes owner data into the command surface but the CLI tab formats and copies `openclaw skills install ${slug}`; this is source- and test-reproducible without running the browser UI.

## Solution Assessment

Unclear. Owner-scoped installation matches the orgs direction, but maintainers need to choose the supported locator syntax and make the CLI/API accept it before the web page advertises it.
