## Summary

Keep open: current main and the latest release still display/copy a bare-slug OpenClaw install command, and the collaborator-authored issue is paired with active owner-scoped slug work that needs maintainer compatibility direction.

## What This Changes

The issue requests that the skill detail install code block use an owner-scoped skill reference instead of a bare skill slug.

## Best Possible Solution

Land a compatible owner-scoped identity contract in ClawHub and the OpenClaw CLI, then make the skill page command block, tests, CLI help, and public docs consistently teach the supported scoped reference.

## Reproduction Assessment

Yes. Current main directly shows the command card passing only the bare slug to a formatter that emits the unscoped install command despite owner metadata being available.

## Solution Assessment

Yes in direction, but not as a UI-only string swap. The maintainable fix needs the ClawHub UI/docs and OpenClaw CLI parser/resolver to agree on one backward-compatible scoped reference.
