## Summary

Keep open: current main and v0.20.0 still render/copy bare-slug install commands, while the clarified target is `openclaw skills install @owner/slug`. The issue is collaborator-authored and paired with the open owner-scoped slug PR, so it needs maintainer review rather than cleanup closure.

## What This Changes

The issue requests that the skill detail install codeblock and copy UI use an owner-scoped skill install command instead of a bare skill slug.

## Best Possible Solution

Land one scoped locator contract for `@owner/slug` across UI copy, CLI/API resolution, docs, and regression tests while preserving unambiguous bare-slug compatibility.

## Reproduction Assessment

Yes, by source inspection: current main still renders and copies `openclaw skills install weather` even when owner props are present. I did not run a live browser flow in this read-only review.

## Solution Assessment

No, not as a UI-only copy change. The maintainable path is to align the scoped locator parser, ambiguity rules, docs, and UI tests together.
