## Summary

Keep open: current main still copies the direct install command as a bare slug, and making it owner-scoped safely needs a maintainer decision on the canonical locator plus compatible parser/resolver behavior.

## What This Changes

Request the skill detail install code block to copy an owner-scoped `openclaw skills install <owner>/<slug>` command instead of a bare slug.

## Best Possible Solution

Define one scoped skill locator contract, preserve bare-slug compatibility where unambiguous, then update the install UI, parser/resolver, docs, and regression tests together.

## Reproduction Assessment

Yes. Source and tests show the command card renders and copies `openclaw skills install weather` even when an owner handle is supplied, so the requested scoped codeblock behavior is absent on current main.

## Solution Assessment

No as a UI-only change. Owner-scoped install commands are a reasonable direction, but the maintainable fix must coordinate locator parsing, safe filesystem and lockfile behavior, API resolution, docs, and compatibility tests.
