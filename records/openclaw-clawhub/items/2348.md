## Summary

Keep open. Current main still renders the skill install code block from a bare slug, while the spec and issue discussion point toward an owner-scoped locator, and this collaborator-authored/product-decision item should stay on the maintainer path.

## What This Changes

The issue requests that the skill page install code block show an owner-scoped install command instead of a bare skill slug.

## Best Possible Solution

Land a maintainer-approved scoped install locator contract across the skill install code block, copy tests, docs, and resolver support while preserving legacy bare-slug compatibility where required.

## Reproduction Assessment

Yes, as a source-level reproduction. Current main formats and tests the visible install command as `openclaw skills install weather` even when owner metadata is available; I did not run browser proof because this review is read-only.

## Solution Assessment

Unclear as a final implementation path. Owner-scoped install UI is the right direction, but the branch discussion changed the planned format to `@owner/slug`, and a UI-only patch could get ahead of resolver compatibility.
