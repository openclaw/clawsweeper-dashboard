- `impact:other`: The issue affects correctness and trust in install guidance but does not fit the narrower data, security, session, auth, crash, or message impact categories.

## Summary

Current main still renders and copies a bare-slug install command, while owner qualification exists only in prompt text and the collaborator-authored discussion now targets `@owner/slug`; the issue remains necessary and requires a cross-surface compatibility decision.

## What This Changes

Change the skill-page install code block and clipboard command from a bare slug to a canonical owner-scoped skill reference, currently proposed as `@owner/slug`.

## Best Possible Solution

Define `@owner/slug` as the canonical user-facing reference only after the installer, resolver, UI prompts, copied commands, CLI help, docs, and compatibility tests support it consistently while preserving an intentional bare-slug fallback policy.

## Reproduction Assessment

Yes at source level: current main passes only the bare slug to the displayed command formatter, and regression tests explicitly expect that unscoped command.

## Solution Assessment

No, a code-block-only replacement is not the best fix; the canonical syntax and backward-compatible resolution behavior must be implemented and validated end to end.
