## Summary

Keep open: current main and latest release still render/copy the compact skill install command as a bare slug, while the planned scoped `@owner/slug` behavior is only in discussion and an open, dirty broad PR. The issue is collaborator-authored, so cleanup should leave it on the maintainer path.

## What This Changes

The issue requests that the skill page install codeblock/copy UI use an owner-scoped skill reference instead of a bare skill slug.

## Best Possible Solution

Align the skill page command, copy behavior, docs, tests, and CLI/resolver contract on the maintainer-selected `@owner/slug` locator while preserving legacy bare-slug compatibility where required.

## Reproduction Assessment

Yes, at source level: current main still routes the compact CLI codeblock through a slug-only formatter, and tests assert the bare-slug command. I did not run a browser because this review is read-only, but the source/test path is high-confidence.

## Solution Assessment

Unclear as a final implementation until maintainers settle the locator contract. The direction is plausible, but a narrow UI-only edit is not the best solution unless CLI/resolver/docs compatibility for `@owner/slug` is aligned at the same time.
