Keep open: current main and the latest release still render/copy bare-slug install commands, while the clarified target behavior is `openclaw skills install @owner/slug`. ## What This Changes

The issue asks for skill detail install code blocks to use owner-scoped skill refs, clarified as `openclaw skills install @owner/slug`.

## Best Possible Solution

Define and land one owner-scoped locator contract, preserving unambiguous bare-slug compatibility, then update the install UI, CLI/API resolver, docs, and regression tests together.
