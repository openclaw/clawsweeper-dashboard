## Summary

Keep open: current main still renders and copies the bare `openclaw skills install <slug>` command, and the requested owner-scoped install locator is a product/API/CLI contract change rather than a safe UI-only cleanup.

## What This Changes

Request that skill install commands use an owner-scoped locator such as `openclaw skills install <owner-slug>/<skill-slug>` instead of a bare skill slug.

## Best Possible Solution

Settle one canonical scoped install locator across OpenClaw, ClawHub API/CLI, docs, and tests, then preserve compatible slug-only behavior where it is still intentionally supported.

## Reproduction Assessment

Yes. Source and tests show current main still renders and copies `openclaw skills install ${slug}` even when owner data is available.

## Solution Assessment

Unclear as requested. Owner-scoped installation is likely the right direction, but the safest solution is a coordinated locator contract rather than changing the page command before CLI/API support exists.
