## What This Changes

The issue asks for the skill page install command UI to include the owner scope rather than showing a bare skill slug.

## Best Possible Solution

Land one compatible scoped-skill reference contract across ClawHub UI/docs and the OpenClaw install parser, then update the skill page command block to teach that exact supported syntax.

## Reproduction Assessment

Yes. Current main directly shows the command card passing only the bare slug to a formatter that emits the unscoped install command despite owner metadata being available.

## Solution Assessment

Unclear as a standalone UI-only change. The maintainable path needs the UI, docs, and CLI resolver to agree on one backward-compatible scoped reference first.
