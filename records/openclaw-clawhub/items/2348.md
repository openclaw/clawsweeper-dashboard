## Summary

Keep open under the ClawHub issue policy: current main still renders the direct install code block as a bare slug, but making it owner-scoped is broader than a UI copy change because the current CLI rejects slash locators and prior scoped-slug work showed compatibility and ownership-routing questions.

## What This Changes

Request the skill-page install code block to show `openclaw skills install <owner-slug>/<skill-slug>` instead of a bare skill slug.

## Best Possible Solution

Define one scoped skill locator contract, preserve safe bare-slug compatibility where unambiguous, then update the install UI, CLI/API resolver, filesystem and lockfile semantics, docs, specs, and regression tests together.

## Reproduction Assessment

Yes from source inspection. The current command card and tests still render and copy `openclaw skills install weather` even when `ownerHandle` is present, while the CLI rejects slash-containing locators.

## Solution Assessment

No as a UI-only change. Owner-scoped install commands are a reasonable direction, but the maintainable fix must coordinate locator parsing, safe filesystem and lockfile behavior, API resolution, docs, and compatibility tests.
