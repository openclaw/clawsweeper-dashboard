## What This Changes

The issue asks the skill detail install codeblock/copy UI to show an owner-scoped OpenClaw install command such as `openclaw skills install @owner/slug` instead of a bare skill slug.

## Best Possible Solution

Complete one canonical owner-scoped skill-ref implementation that updates the ClawHub UI copy command, docs, tests, and compatible CLI/API handling together.
