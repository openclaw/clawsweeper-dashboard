Keep open. ## What This Changes

The issue asks for the skill install command shown in the skill page code block/copy UI to include owner scope instead of a bare skill slug.

## Best Possible Solution

Settle the backward-compatible `@owner/slug` identity contract across ClawHub and OpenClaw CLI, then update the skill page, tests, CLI help, specs, and public docs to teach the approved scoped install reference consistently.

## Reproduction Assessment

Yes from source inspection: current main directly passes only the bare `slug` into the OpenClaw install-command formatter, and tests still expect `openclaw skills install weather`. I did not run a browser because this is a read-only cleanup review and the source path is direct.

## Solution Assessment

No for an isolated UI-only string change. The maintainable solution is coordinated owner-scoped resolution and CLI compatibility, otherwise ClawHub could again show a command OpenClaw rejects.
