## Summary

Keep open under the ClawHub issue policy: current main still shows/copies the bare slug command, and the prior owner/slug UI was intentionally changed after users reported that OpenClaw rejected scoped CLI arguments. The remaining work is a scoped-locator product/API decision, not a UI-only cleanup.

## What This Changes

The issue requests that the skill detail install code block copy an owner-scoped `openclaw skills install <owner>/<slug>` command instead of a bare skill slug.

## Best Possible Solution

Define one canonical scoped skill locator, preserve safe bare-slug compatibility, then update the install UI, CLI/API resolution, docs, and regression tests together.

## Reproduction Assessment

Yes. Source and tests show the current skill command card renders and copies `openclaw skills install weather` even when `ownerHandle` is supplied.

## Solution Assessment

No, not as a UI-only change. Owner-scoped commands are the right direction if maintainers choose that contract, but the maintainable fix needs CLI/API parsing, compatibility, docs, and tests alongside the UI copy.
