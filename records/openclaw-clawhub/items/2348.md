- `impact:other`: The issue affects reliable skill identity and installation, a meaningful maintainer-visible impact outside the narrower owned categories.

## Summary

Current main still displays and copies a bare-slug OpenClaw install command even though the skill page has owner metadata. This collaborator-authored issue remains necessary, and the active paired implementation at https://github.com/openclaw/clawhub/pull/2299 is not merged and explicitly depends on matching OpenClaw CLI support.

## What This Changes

Show and copy an owner-scoped skill reference in the OpenClaw install command on ClawHub skill pages.

## Best Possible Solution

Adopt one compatible owner-scoped identity contract across ClawHub and the OpenClaw CLI, preserve legacy slug-only resolution where required, then align the skill-page command, regression tests, CLI help, and public docs on `@owner/slug`.

## Reproduction Assessment

Yes. Current-main source and regression expectations directly establish that the skill page emits a bare-slug command despite having owner metadata; a live browser run is not needed to establish this source-level behavior.

## Solution Assessment

Yes in direction, but only as a coordinated ClawHub and OpenClaw CLI contract. A UI-only wording change is not the best solution because it could advertise syntax that the installer cannot yet resolve compatibly.
