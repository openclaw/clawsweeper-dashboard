## Summary

Keep open: current main still renders and copies the bare slug install command even when owner data is available, but changing the code block safely is not a UI-only bug fix because current docs and CLI install paths still use and validate bare slugs.

## What This Changes

Request that skill detail install code blocks display an owner-scoped OpenClaw install command such as `openclaw skills install <owner>/<slug>` instead of a bare slug.

## Best Possible Solution

Define one scoped skill locator contract, keep safe bare-slug compatibility, then update the install UI, CLI/API resolution, docs, and tests together.

## Reproduction Assessment

Yes by source inspection. Rendering `SkillCommandLineCard` with an owner handle still reaches `formatOpenClawInstallCommand(slug)`, and tests assert the copied command remains `openclaw skills install weather`.

## Solution Assessment

Unclear as a UI-only request. Owner-scoped commands are a plausible direction, but the maintainable fix must align UI copy with CLI/API parsing, filesystem safety, lockfile behavior, and the chosen compatibility policy.
