## Summary

Keep open. The ClawHub cleanup policy for this pass keeps issues open, and current main still has the reported mismatch: update hashes local extracted files, downloads add generated root `_meta.json`, and the server resolve path compares against published version file fingerprints that do not include that generated metadata.

## What This Changes

The issue reports that `clawhub update` warns about `local changes (no match)` after a fresh install or force update because the local fingerprint is not recognized by the registry resolve API.

## Best Possible Solution

Align downloaded/local and published/resolve fingerprint inputs so ClawHub-generated root `_meta.json` does not create false local-change warnings, while preserving integrity checks for authored JSON and other text files. Add regression coverage for install or force-update followed by update reporting up to date.
