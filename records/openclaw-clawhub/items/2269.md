## Summary

Keep open. Current main still treats skill install counters as authenticated `clawhub sync` telemetry rather than immediate `clawhub install` reporting, and the requested direct install event needs maintainer product/privacy direction.

## What This Changes

The issue requests that `clawhub install` report install events so `installsAllTime` and `installsCurrent` no longer remain zero for installed skills.

## Best Possible Solution

Maintainers should decide whether install stats remain sync-only telemetry or gain immediate install event reporting, then update the CLI, backend endpoint semantics, docs/specs, and regression coverage for dedupe, opt-out, uninstall, and current-install counting.

## Reproduction Assessment

Yes. Source inspection shows direct `cmdInstall` does not send install telemetry and only authenticated `clawhub sync` posts the telemetry endpoint; the live example still shows non-zero downloads with zero install counters.

## Solution Assessment

No for an automatic direct-POST change. The maintainable solution needs an explicit decision on privacy, opt-out, dedupe, uninstall, and `installsCurrent` semantics before code changes.
