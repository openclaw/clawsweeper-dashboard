## Summary

Keep open. Current main still treats direct `clawhub install` as a download/local-metadata path while install counters are produced from authenticated `clawhub sync` telemetry; the live example still has non-zero downloads with zero install counters, so this needs maintainer product/privacy direction rather than cleanup closure.

## What This Changes

The issue asks ClawHub to report skill installs from `clawhub install` so `installsAllTime` and `installsCurrent` no longer remain zero when downloads increase.

## Best Possible Solution

Maintainers should decide whether install stats remain sync-only or gain immediate install event reporting, then update the CLI, backend endpoint semantics, docs/specs, and tests for dedupe, opt-out, uninstall, stale roots, and current-install counting.

## Reproduction Assessment

Yes. Source inspection shows direct `cmdInstall` does not send install telemetry while `clawhub sync` does, and the live `video-to-notes` API response still shows non-zero downloads with zero install counters.

## Solution Assessment

No for an automatic direct-POST change. The maintainable path needs an explicit decision on privacy, opt-out, dedupe, uninstall, and `installsCurrent` semantics before code changes.
