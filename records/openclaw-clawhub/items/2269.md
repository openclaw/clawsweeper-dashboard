## Summary

Keep open: current main confirms the direct-install reporting gap, but the documented contract says install counters come from logged-in `clawhub sync` telemetry, so changing `clawhub install` to report installs needs maintainer telemetry/privacy judgment rather than cleanup closure.

## What This Changes

The issue requests that `clawhub install` report install events so `installsAllTime` and `installsCurrent` increment for directly installed skills.

## Best Possible Solution

Define whether install counters should remain sync-only or extend install/update/uninstall to report telemetry, then document and test the chosen privacy, auth, idempotency, and opt-out contract.

## Reproduction Assessment

Yes from source inspection: `cmdInstall` does not call `/api/cli/telemetry/sync`, while `cmdSync` does. The distinct-user count for the example skill is not reproducible from repository source alone.

## Solution Assessment

Unclear: a direct install POST is plausible, but it changes telemetry timing and semantics from the documented sync model. Maintainers should choose the product/privacy contract before implementation.
