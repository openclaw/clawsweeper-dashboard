## Summary

Keep open. Current main still uses authenticated `clawhub sync` as the skill install telemetry source, while direct `clawhub install` downloads and records local metadata without posting an install event; the live example still shows non-zero downloads with zero install counters, so this needs maintainer product/privacy direction rather than cleanup closure.

## What This Changes

Request direct `clawhub install` registry reporting so `installsAllTime` and `installsCurrent` increment instead of relying only on sync telemetry.

## Best Possible Solution

Maintainers should choose sync-only versus immediate install reporting, then update CLI behavior, backend endpoint semantics, docs/specs, and tests around the chosen privacy and current-install model.

## Reproduction Assessment

Yes. Source inspection shows direct `cmdInstall` does not send install telemetry, and the live example API still returns non-zero downloads with zero install counters; I did not run an install because this review is read-only.

## Solution Assessment

No for an automatic direct POST. Adding it is mechanically narrow, but it changes the documented sync-time telemetry boundary and `installsCurrent` semantics, so maintainers should decide the product/privacy model first.
