## Summary

Keep open. Current main still only updates skill install counters from authenticated `clawhub sync` telemetry snapshots, while direct `clawhub install` downloads and records local metadata without reporting an install event; the live example still shows non-zero downloads with zero install counters, so this needs maintainer product/privacy direction rather than cleanup closure.

## What This Changes

Request direct install reporting so `installsAllTime` and `installsCurrent` reflect `clawhub install` activity instead of only sync snapshots.

## Best Possible Solution

Maintainers should choose the telemetry model, then update CLI behavior, backend endpoint semantics, docs/specs, and tests around the chosen install-count and privacy contract.

## Reproduction Assessment

Yes. Source inspection on current main shows `cmdInstall` has no install telemetry POST while `cmdSync` does, and the live example returns non-zero downloads with both install counters at 0; I did not run a write-producing install in this read-only review.

## Solution Assessment

No for an automatic code change. Adding a direct POST is mechanically narrow, but the maintainable fix depends on a product/privacy decision about sync-only versus immediate install reporting and how to dedupe current installs.
