## Summary

Keep open. Current main still counts skill installs through authenticated `clawhub sync` telemetry, while direct `clawhub install` only resolves, downloads, writes local origin metadata, and updates the lockfile; changing that reporting boundary needs a maintainer product/privacy decision.

## What This Changes

The issue asks for direct install reporting so `installsAllTime` and `installsCurrent` stop remaining at zero when users run `clawhub install`.

## Best Possible Solution

Maintainers should decide whether install stats remain sync-only telemetry or gain immediate install-event reporting, then update CLI/backend semantics, docs/specs, and regression coverage for privacy, opt-out, dedupe, uninstall, and current-install counting.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: direct `cmdInstall` has no install telemetry POST, while the implemented install counter path is authenticated `clawhub sync` telemetry. I did not run a full local CLI plus Convex write flow in this read-only review.

## Solution Assessment

No for an automatic direct-POST change. The maintainable path needs an explicit decision on privacy, opt-out, dedupe, uninstall, and `installsCurrent` semantics before code changes.
