## Summary

Keep open. Current main still implements install counters through authenticated `clawhub sync` telemetry, while direct `clawhub install` only downloads and records local metadata; changing that behavior needs maintainer product/privacy direction, not conservative cleanup.

## What This Changes

The issue requests direct install-event reporting so `installsAllTime` and `installsCurrent` update when users run `clawhub install`.

## Best Possible Solution

Maintainers should decide whether install stats remain sync-only telemetry or gain immediate install-event reporting, then update CLI/backend semantics, docs/specs, and regression coverage for privacy, opt-out, dedupe, uninstall, and current-install counting.

## Reproduction Assessment

Yes. Source inspection shows direct `cmdInstall` does not send install telemetry, while authenticated `clawhub sync` is the current path that posts install snapshots to the registry.

## Solution Assessment

No for an automatic direct-POST change. The maintainable path needs an explicit decision on privacy, opt-out, dedupe, uninstall, and `installsCurrent` semantics before code changes.
