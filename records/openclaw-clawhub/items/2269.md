## Summary

Keep open. Current main still counts skill installs through authenticated `clawhub sync` telemetry, while direct `clawhub install` only downloads and writes local metadata; for ClawHub issues, cleanup policy also keeps this open for maintainer direction rather than auto-closing.

## What This Changes

The issue asks for `clawhub install` to report install events so skill `installsAllTime` and `installsCurrent` counters reflect direct installs.

## Best Possible Solution

Maintainers should choose sync-only versus immediate install reporting, then update CLI behavior, backend endpoint semantics, docs/specs, and tests around dedupe, opt-out, uninstall, and current-install semantics.

## Reproduction Assessment

Yes. Source inspection shows direct `cmdInstall` does not send install telemetry and only `clawhub sync` posts the telemetry endpoint; the live example still has non-zero downloads with zero install counters.

## Solution Assessment

No for an automatic direct-POST change. The maintainable solution needs an explicit decision on privacy, opt-out, dedupe, and `installsCurrent` semantics before code changes.
