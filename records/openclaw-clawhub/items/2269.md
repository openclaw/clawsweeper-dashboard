## Summary

Keep open. Current main still counts skill installs from authenticated `clawhub sync` telemetry, while direct `clawhub install` downloads and records local metadata without reporting an install event; changing that boundary needs maintainer product/privacy direction, not cleanup closure.

## What This Changes

Report direct `clawhub install` events to the registry so skill `installsAllTime` and `installsCurrent` increment for direct installs.

## Best Possible Solution

Maintainers should choose sync-only versus immediate install reporting, then update CLI behavior, backend endpoint semantics, docs/specs, and tests around the chosen privacy, dedupe, and active-install model.

## Reproduction Assessment

Yes. Source inspection shows direct `cmdInstall` does not call the telemetry endpoint, while the documented reporting path is logged-in `clawhub sync`; I did not run an install in this read-only review.

## Solution Assessment

No for an automatic code change. Adding a direct POST is mechanically narrow, but the maintainable fix needs a maintainer decision on privacy, opt-out, dedupe, and `installsCurrent` semantics first.
