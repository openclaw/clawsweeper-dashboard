## Summary

Keep open. Current main still treats authenticated `clawhub sync` as the install-count telemetry source, direct installs do not post an install event, and the live example still shows non-zero downloads with zero install counters; changing that boundary needs maintainer product/privacy direction rather than cleanup closure.

## What This Changes

The issue requests direct CLI install reporting so `installsAllTime` and `installsCurrent` update when users install a skill without running authenticated sync.

## Best Possible Solution

Maintainers should choose sync-only versus immediate install reporting, then update CLI behavior, backend endpoint semantics, docs/specs, and tests around privacy, dedupe, uninstall, and stale-root behavior.

## Reproduction Assessment

Yes. Source inspection shows `cmdInstall` does not send install telemetry while `cmdSync` does, and the live example API still returns non-zero downloads with zero install counters; I did not run a local install in this read-only review.

## Solution Assessment

No for an automatic direct-POST fix. Adding a POST is mechanically narrow, but it would change the documented sync-time telemetry boundary and needs a maintainer decision on privacy, opt-out, dedupe, and current-install semantics.
