## Summary

Keep open: current main confirms the direct-install reporting gap, but the documented contract makes install counters sync-telemetry based, so changing `clawhub install` to report installs needs maintainer privacy/product direction rather than cleanup closure.

## What This Changes

Request `clawhub install` to report registry install events so `installsAllTime` and `installsCurrent` increment for directly installed skills.

## Best Possible Solution

Decide whether install counters should remain sync-only or extend install/update/uninstall to report telemetry with the same privacy, auth, idempotency, and opt-out guarantees as `clawhub sync`.

## Reproduction Assessment

Yes for the narrow source claim: `cmdInstall` does not call `/api/cli/telemetry/sync`, while the live example currently returns zero install counters. The assertion that multiple distinct users installed the example skill is not independently reproducible from the repository.

## Solution Assessment

Unclear: direct install reporting is plausible, but it changes the documented sync-only telemetry model. A maintainable fix needs a product decision on privacy, auth, opt-out, idempotency, update, and uninstall semantics first.
