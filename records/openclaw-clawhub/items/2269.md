## Summary

Keep open. Current main still matches the report: direct `clawhub install` downloads and writes local metadata without posting install telemetry, while install counters are fed by authenticated `clawhub sync`; the live API example still shows nonzero downloads with zero install counters, so this needs maintainer product/privacy direction rather than cleanup closure.

## What This Changes

The issue requests that `clawhub install` report install events so public skill `installsAllTime` and `installsCurrent` counters reflect direct installs.

## Best Possible Solution

Maintainers should choose sync-only versus immediate install telemetry, then update CLI behavior, backend endpoint semantics, docs/specs, and tests around dedupe, opt-out, uninstall, staleness, and current-install semantics.

## Reproduction Assessment

Yes. Source inspection shows direct `cmdInstall` does not send install telemetry and only `clawhub sync` posts the telemetry endpoint; a live API check on 2026-05-21 still returned nonzero downloads with zero install counters for the reported skill.

## Solution Assessment

No for an automatic direct POST. That would address the visible counter symptom, but the maintainable solution needs an explicit decision on privacy, opt-out, dedupe, uninstall, and `installsCurrent` semantics.
