## Summary

Keep open. Current main still uses authenticated `clawhub sync` telemetry as the install-counter source, while direct `clawhub install` only resolves, downloads, and writes local metadata; adding immediate install reporting would change documented telemetry/privacy semantics and needs maintainer direction.

## What This Changes

Report direct `clawhub install` events to the registry so `installsAllTime` and `installsCurrent` reflect installs without waiting for sync telemetry.

## Best Possible Solution

Maintainers should decide the install-stat contract first, then implement either clearer sync-only semantics or an immediate install-reporting path that shares dedupe, opt-out, uninstall, stale-root, docs, specs, and tests.

## Reproduction Assessment

Yes. Source inspection on current main shows direct `cmdInstall` does not report install telemetry, and the live `video-to-notes` API example still has non-zero downloads with both install counters at zero.

## Solution Assessment

No for an automatic direct-POST change. The maintainable fix needs maintainer agreement on privacy, opt-out, dedupe, uninstall, stale-root, and `installsCurrent` semantics before code changes.
