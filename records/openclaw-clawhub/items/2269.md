## Summary

Keep open. Current main still records skill installs through authenticated `clawhub sync` telemetry, while direct `cmdInstall` downloads/extracts the skill and writes local metadata without reporting an install event; changing that behavior needs maintainer product/privacy direction rather than conservative cleanup.

## What This Changes

The issue asks for direct CLI installs to report registry install events so `installsAllTime` and `installsCurrent` reflect installed skills instead of only sync telemetry.

## Best Possible Solution

Maintainers should define the telemetry contract first, then update the CLI, backend endpoint semantics, specs/docs, and tests around dedupe, opt-out, uninstall, and current-install counting.

## Reproduction Assessment

Yes. Source inspection shows direct `cmdInstall` does not send install telemetry, while `cmdSync` posts root snapshots to the telemetry endpoint that drives install counters.

## Solution Assessment

No for an automatic direct-POST change. The maintainable path needs an explicit decision on privacy, opt-out, dedupe, and `installsCurrent` semantics before implementation.
