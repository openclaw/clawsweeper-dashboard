## Summary

Keep open. Current main still makes direct `clawhub install` download and write local install metadata without reporting an install event, while install counters are wired through authenticated `clawhub sync` telemetry; changing that behavior needs maintainer product/privacy direction.

## What This Changes

The issue asks for `clawhub install` to report install events so `installsAllTime` and `installsCurrent` reflect direct installs instead of staying zero when downloads increase.

## Best Possible Solution

Maintainers should decide whether install stats remain sync-only or gain immediate install reporting, then update the CLI, backend endpoint semantics, docs/specs, and tests for dedupe, opt-out, uninstall, and current-install counting.

## Reproduction Assessment

Yes. Source inspection shows direct `cmdInstall` does not send install telemetry and only `clawhub sync` posts the telemetry endpoint; the live example still has non-zero downloads with zero install counters.

## Solution Assessment

No for an automatic direct-POST change. The maintainable solution needs an explicit decision on privacy, opt-out, dedupe, uninstall, and `installsCurrent` semantics before code changes.
