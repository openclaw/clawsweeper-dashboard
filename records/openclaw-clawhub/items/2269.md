## Summary

Keep open. Current main still counts skill installs only through authenticated `clawhub sync` telemetry, while direct `clawhub install` downloads and writes local metadata without reporting an install event; changing that boundary needs maintainer product/privacy direction.

## What This Changes

The issue requests immediate registry-side install reporting from `clawhub install` so `installsAllTime` and `installsCurrent` no longer remain zero for directly installed skills.

## Best Possible Solution

Maintainers should decide whether skill install stats remain sync-only telemetry or gain immediate install reporting, then update the CLI, backend endpoint semantics, docs/specs, and tests for dedupe, opt-out, uninstall, and current-install counting.

## Reproduction Assessment

Yes. Source inspection shows direct `cmdInstall` does not send install telemetry, only `clawhub sync` posts the telemetry endpoint, and the live example still has non-zero downloads with zero install counters.

## Solution Assessment

No for an automatic direct-POST change. The maintainable solution needs an explicit decision on privacy, opt-out, dedupe, uninstall, and `installsCurrent` semantics before code changes.
