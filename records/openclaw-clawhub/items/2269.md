## Summary

Keep open. Current main still counts skill installs through authenticated `clawhub sync` telemetry, while `clawhub install` only downloads and writes local metadata, so immediate install reporting would change the documented telemetry and privacy model rather than being conservative cleanup.

## What This Changes

Request that `clawhub install` report an install event so skill `installsAllTime` and `installsCurrent` counters increment without a later sync.

## Best Possible Solution

Maintainers should choose the install telemetry source of truth, then update the CLI, backend endpoint surface, docs, and tests so current-install, uninstall, stale-root, and opt-out semantics stay consistent.

## Reproduction Assessment

Yes. Source inspection shows `cmdInstall` does not send install telemetry, and the live example API still returns non-zero downloads with zero install counters; I did not run a real install in this read-only review.

## Solution Assessment

No for an automatic fix. Adding a POST from `cmdInstall` is mechanically narrow, but it changes the documented sync-time telemetry model and needs a maintainer decision on privacy and `installsCurrent` semantics.
