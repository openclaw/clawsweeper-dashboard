## Summary

Keep open. Current main still does not report an install event from `clawhub install`; the implemented counter path is authenticated `clawhub sync` telemetry, so changing install-time reporting needs maintainer product/privacy direction rather than conservative cleanup.

## What This Changes

The issue asks for `clawhub install` to report install events so `installsAllTime` and `installsCurrent` reflect real local installs instead of staying at zero.

## Best Possible Solution

Maintainers should choose either to keep install counts sync-based and document that more clearly, or add an authenticated install/update/uninstall reporting model that preserves opt-out, deletion, and current-install semantics.

## Reproduction Assessment

Yes. By source inspection, `cmdInstall` writes local install metadata and the lockfile without submitting the telemetry sync request, while install counter events are produced from authenticated `clawhub sync` snapshots.

## Solution Assessment

No for an automatic code change. A direct POST from install is easy to imagine, but it would change the documented telemetry and privacy model, so maintainers should decide the counting semantics first.
