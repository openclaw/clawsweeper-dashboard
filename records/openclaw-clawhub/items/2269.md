## Summary

Keep open. Current main still implements skill install counters through authenticated `clawhub sync` telemetry, while direct `clawhub install` only downloads files and records local metadata; the live example still has non-zero downloads with zero install counters, so this needs maintainer product/privacy direction rather than cleanup closure.

## What This Changes

Request direct `clawhub install` reporting so `installsAllTime` and `installsCurrent` update without requiring a later authenticated sync.

## Best Possible Solution

Maintainers should choose sync-only versus immediate install reporting, then update CLI behavior, backend endpoint semantics, docs/specs, and tests around the chosen privacy and current-install model.

## Reproduction Assessment

Yes, for source-level reproduction: current `cmdInstall` has no install-report POST, while the live example API still returns non-zero downloads with zero install counters. I did not run a real install because this review is read-only.

## Solution Assessment

No for an automatic direct POST. The mechanical change is narrow, but it would alter documented telemetry collection and `installsCurrent` semantics, so maintainers should decide the product/privacy model first.
