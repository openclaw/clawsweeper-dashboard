## Summary

Keep open. Current main still treats authenticated `clawhub sync` as the install telemetry source, while direct `clawhub install` downloads and records local metadata without posting an install event; changing that boundary needs maintainer product/privacy direction, and ClawHub cleanup policy keeps issues open.

## What This Changes

Request that `clawhub install` report an install event so public skill stats populate `installsAllTime` and `installsCurrent` immediately.

## Best Possible Solution

Maintainers should choose sync-only versus immediate install reporting, then update CLI behavior, backend endpoint semantics, docs/specs, and tests around the chosen privacy and current-install model.

## Reproduction Assessment

Yes at source level. Current main shows direct `cmdInstall` does not send telemetry, while the live example API still has non-zero downloads and zero install counters; I did not run a real install in this read-only review.

## Solution Assessment

No for an automatic direct POST. The narrow code shape is clear, but the best solution needs maintainer agreement on privacy, opt-out, dedupe, and `installsCurrent` semantics before implementation.
