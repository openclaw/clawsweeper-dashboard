## Summary

Keep open. Current main still leaves direct `clawhub install` as a download-and-local-metadata flow while install counters are reported through authenticated `clawhub sync` telemetry, and ClawHub review policy does not auto-close issues.

## What This Changes

The issue requests that `clawhub install` report install events so `installsAllTime` and `installsCurrent` no longer remain zero for installed skills.

## Best Possible Solution

Maintainers should decide whether install stats remain sync-only or gain immediate install event reporting, then update the CLI, backend endpoint semantics, docs/specs, and tests for dedupe, opt-out, uninstall, and current-install counting.

## Reproduction Assessment

Yes. Source inspection shows direct `cmdInstall` does not send install telemetry and only `clawhub sync` posts the telemetry endpoint; the live example still returns non-zero downloads with zero install counters.

## Solution Assessment

No for an automatic direct-POST change. The maintainable solution needs an explicit decision on privacy, opt-out, dedupe, uninstall, and `installsCurrent` semantics before code changes.
