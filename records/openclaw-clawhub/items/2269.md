## Summary

Keep open. Current main still counts skill installs through authenticated `clawhub sync` telemetry snapshots, while direct `clawhub install` resolves, downloads, writes local metadata, and updates the lockfile without reporting an install event; changing that boundary needs maintainer product/privacy direction.

## What This Changes

Request direct `clawhub install` reporting so `installsAllTime` and `installsCurrent` increment for installed skills instead of relying only on later sync telemetry.

## Best Possible Solution

Maintainers should decide whether install stats remain sync-only telemetry or gain immediate install-event reporting, then update CLI/backend semantics, specs/docs, and regression coverage for privacy, opt-out, dedupe, uninstall, and current-install counting.

## Reproduction Assessment

Yes. Source inspection shows direct `cmdInstall` has no install telemetry POST, while the existing counter path is authenticated `clawhub sync` telemetry through `/api/cli/telemetry/sync`.

## Solution Assessment

No for an automatic direct-POST change. The maintainable path needs an explicit decision on privacy, opt-out, dedupe, uninstall, and `installsCurrent` semantics before code changes.
