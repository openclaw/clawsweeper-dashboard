## Summary

Keep open. Current main still implements skill install counters as authenticated `clawhub sync` telemetry, while direct `clawhub install` only downloads and records local metadata; the live example still has non-zero downloads with zero install counters, so this needs maintainer product/privacy direction rather than cleanup closure.

## What This Changes

The issue requests direct registry install reporting so `installsAllTime` and `installsCurrent` update when users run `clawhub install`.

## Best Possible Solution

Maintainers should choose sync-only versus immediate install reporting, then update CLI behavior, backend endpoint semantics, docs/specs, and tests around the chosen privacy and current-install model.

## Reproduction Assessment

Yes. Source inspection shows direct `cmdInstall` does not send install telemetry, and the live example API returns non-zero downloads with zero install counters; I did not run an install because this review is read-only.

## Solution Assessment

No, not as an automatic direct-POST fix. A direct install event is mechanically narrow, but the durable solution needs explicit dedupe, opt-out, uninstall/current-install, and docs semantics.
