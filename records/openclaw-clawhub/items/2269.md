## Summary

Keep open. Current main still matches the report: direct `clawhub install` fetches metadata, downloads the skill, and writes local origin/lockfile data without reporting install telemetry, while install counters are populated through authenticated `clawhub sync` snapshots; the live `video-to-notes` API example still shows non-zero downloads with zero install counters.

## What This Changes

The issue requests that direct `clawhub install` operations report install events so `installsAllTime` and `installsCurrent` reflect direct installs.

## Best Possible Solution

Maintainers should choose sync-only versus immediate install reporting, then update the CLI, backend endpoint semantics, docs/specs, and tests around dedupe, opt-out, uninstall, stale-root, and current-install semantics.

## Reproduction Assessment

Yes. Source inspection shows direct `cmdInstall` does not send install telemetry and only `cmdSync` posts the telemetry endpoint; the live API example still shows non-zero downloads with zero install counters, though I did not run a full CLI plus Convex install flow in this read-only review.

## Solution Assessment

No for an automatic code change. A direct POST would address the visible counter symptom, but the maintainable solution needs an explicit decision on privacy, opt-out, dedupe, and `installsCurrent` semantics.
