## Summary

Keep open. Current main still routes skill install counters through authenticated `clawhub sync` telemetry, while `clawhub install` performs local installation without an immediate install-event POST, so this needs maintainer product/privacy direction rather than conservative cleanup.

## What This Changes

The issue requests that `clawhub install` report skill install events so `installsAllTime` and `installsCurrent` increment instead of remaining zero.

## Best Possible Solution

Maintainers should choose sync-only versus immediate install reporting, then update the CLI, backend event model, docs, and tests around the chosen privacy and current-install semantics.

## Reproduction Assessment

Yes. Source inspection shows `cmdInstall` does not send install telemetry, and the live example API still returns non-zero downloads with zero install counters; this review did not execute a local install because it is read-only.

## Solution Assessment

No for an automatic code change. Adding an immediate POST is mechanically narrow, but it changes the documented sync-time telemetry and `installsCurrent` model, so maintainers should decide the product/privacy boundary first.
