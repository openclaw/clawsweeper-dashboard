## Summary

Keep open. Current main confirms the source-level observation: direct `clawhub install` resolves and downloads skills without sending install telemetry, while install counters are currently driven by authenticated `clawhub sync`; changing that boundary needs maintainer product/privacy direction, and ClawHub issue cleanup policy does not auto-close issues.

## What This Changes

Report that direct `clawhub install` does not report install telemetry, leaving skill `installsAllTime` and `installsCurrent` at zero for installs that never run authenticated sync.

## Best Possible Solution

Maintainers should choose sync-only versus immediate install reporting, then update CLI behavior, backend event semantics, docs/specs, and regression tests around the chosen privacy and current-install model.

## Reproduction Assessment

Yes. Source inspection shows direct `cmdInstall` does not send install telemetry, and the live example API still returns non-zero downloads with zero install counters; I did not run an install because this review is read-only.

## Solution Assessment

No for an automatic code change. Adding a direct POST is mechanically narrow, but it changes the documented sync-time telemetry boundary and `installsCurrent` semantics, so maintainers should decide the product/privacy model first.
