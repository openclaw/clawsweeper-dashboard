## Label Justifications

- `P2`: The report affects public install statistics and ranking semantics, but the blast radius is limited to telemetry/stat reporting rather than core availability.
- `impact:data-loss`: Direct install events are not captured as install-stat data, leaving public install counters at zero for users who do not report via sync.

## Summary

Keep open. Current main still treats authenticated `clawhub sync` as the install telemetry source, while direct `clawhub install` downloads and records local metadata without reporting an install event; changing that boundary needs maintainer product and privacy direction.

## What This Changes

Request direct registry install reporting from `clawhub install` so `installsAllTime` and `installsCurrent` reflect direct skill installs instead of only sync snapshots.

## Best Possible Solution

Maintainers should choose the telemetry boundary, then update CLI install/sync behavior, backend stat semantics, docs/specs, and tests around the selected privacy and current-install model.

## Reproduction Assessment

Yes for the source-level behavior. `cmdInstall` does not post install telemetry, and the live `video-to-notes` API currently shows non-zero downloads with both install counters at zero; I did not run a local install because this review is read-only.

## Solution Assessment

No for an automatic direct POST. The mechanical fix is clear, but it changes documented sync-only telemetry and `installsCurrent` semantics, so maintainers should decide the privacy, opt-out, and dedupe model first.
