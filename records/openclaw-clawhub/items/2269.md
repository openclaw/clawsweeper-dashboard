## Summary

Keep open. Current main still does not report an immediate install event from `clawhub install`, while the implemented and documented counter model records skill installs through authenticated `clawhub sync` telemetry, so changing this needs maintainer product/privacy direction rather than conservative cleanup.

## What This Changes

Request that `clawhub install` report skill install events so public `installsAllTime` and `installsCurrent` no longer remain zero for installed skills.

## Best Possible Solution

Maintainers should choose sync-only versus immediate install reporting, then update the CLI, backend endpoint surface, docs, and tests around that chosen privacy and current-install model.

## Reproduction Assessment

Yes. Source inspection shows `cmdInstall` does not send telemetry, and the live example API still returns non-zero downloads with zero install counters; I did not run an install because this review is read-only.

## Solution Assessment

No for an automatic code change. Adding an immediate POST is mechanically narrow, but it changes the documented sync-time telemetry and `installsCurrent` semantics, so maintainers should decide the product/privacy boundary first.

## Risks / Open Questions

- Immediate install reporting could double-count users who later run `clawhub sync` unless the direct and sync telemetry models are reconciled.
- A one-shot install POST cannot keep `installsCurrent` accurate without also defining update, uninstall, stale-root, and telemetry opt-out semantics.
- The live zero counters may reflect users who installed but never ran authenticated sync, which is expected under the current documented model but surprising in the public API.
