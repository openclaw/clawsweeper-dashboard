## Summary

Keep open: current main confirms the reported direct-install gap, but the existing product contract says install counters are populated by logged-in `clawhub sync` telemetry, so changing `clawhub install` to report installs needs maintainer telemetry/privacy judgment rather than cleanup closure.

## What This Changes

The issue requests that CLI installs report install events to the registry so `installsAllTime` and `installsCurrent` no longer remain zero while downloads increase.

## Best Possible Solution

Decide whether install counters should remain sync-only or extend install/update/uninstall to report telemetry with the same privacy, auth, idempotency, and opt-out guarantees as `clawhub sync`.

## Reproduction Assessment

Yes from source inspection: `cmdInstall` does not call the telemetry endpoint after downloading and writing local install metadata, while `cmdSync` has the telemetry report path. The exact claim of multiple independent installs for the example skill is not directly verified from repository source.

## Solution Assessment

Unclear: direct install reporting is plausible, but current docs intentionally frame install telemetry around `clawhub sync`, so maintainers should choose the telemetry contract before implementation.
