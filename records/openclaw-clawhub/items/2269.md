## Summary

Keep open. Current main still does not report an install event from `clawhub install`, but the documented implementation counts skill installs through authenticated `clawhub sync` telemetry, so this needs maintainer product/privacy direction rather than conservative cleanup; the related stats discussion at https://github.com/openclaw/clawhub/issues/1712 explicitly left immediate skill-install reporting as a semantics decision.

## What This Changes

The issue requests registry-side install reporting from the CLI install flow so `installsAllTime` and `installsCurrent` update without requiring a separate sync telemetry run.

## Best Possible Solution

Maintainers should choose whether skill install counts remain sync-based or whether install/update/uninstall should report telemetry immediately, then update the CLI, backend endpoint surface, docs, and tests around that chosen privacy model.

## Reproduction Assessment

Yes by source inspection: `cmdInstall` performs metadata GETs, downloads the zip, writes local metadata, and updates the lockfile without sending the telemetry sync request. I did not run the install command because this review must keep the checkout read-only.

## Solution Assessment

No for an automatic code change. The narrow code change is easy to imagine, but it would change the documented sync-only telemetry and current-install semantics, so maintainers should decide the product/privacy boundary first.
