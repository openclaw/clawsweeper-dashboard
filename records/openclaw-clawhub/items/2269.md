## Summary

Keep open. Current main still documents and implements skill install counts through authenticated `clawhub sync` telemetry, while direct `clawhub install` resolves, downloads, writes local origin metadata, and updates the lockfile without reporting an install event; changing that boundary needs maintainer product/privacy direction.

## What This Changes

The issue requests direct install reporting so skill `installsAllTime` and `installsCurrent` stats no longer remain zero for skills installed via `clawhub install`.

## Best Possible Solution

Maintainers should decide whether install stats remain sync-only telemetry or gain immediate install-event reporting; if direct reporting is chosen, the implementation should preserve opt-out/privacy behavior, dedupe with sync, define current-install lifecycle semantics, and update docs/specs/tests together.

## Reproduction Assessment

Yes. Source inspection shows direct `cmdInstall` has no install telemetry POST, while the implemented install counter path is authenticated `clawhub sync` telemetry.

## Solution Assessment

No for an automatic direct-POST change. The maintainable path needs an explicit decision on privacy, opt-out, dedupe, uninstall, and `installsCurrent` semantics before code changes.
