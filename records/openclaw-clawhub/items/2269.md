## Summary

Keep open. Current main still matches the report: direct `clawhub install` downloads and records local metadata without reporting an install event, while skill install counters are fed by authenticated `clawhub sync` telemetry; live API examples still show non-zero downloads with zero install counters, so changing this needs maintainer product/privacy direction.

## What This Changes

Request that `clawhub install` report skill install events so `installsAllTime` and `installsCurrent` reflect installed skills instead of staying at zero.

## Best Possible Solution

Maintainers should choose sync-only versus immediate install reporting, then update CLI behavior, backend endpoint semantics, docs/specs, and tests around dedupe, opt-out, uninstall, and current-install semantics.

## Reproduction Assessment

Yes. Source inspection shows direct `cmdInstall` does not send install telemetry and only `clawhub sync` posts the telemetry endpoint; live API checks on 2026-05-20 still showed non-zero downloads with zero install counters.

## Solution Assessment

No for an automatic code change. A direct POST would address the visible counter symptom, but the maintainable solution needs an explicit decision on privacy, opt-out, dedupe, and `installsCurrent` semantics.
