## Summary

Keep open. Current main still records install counters from authenticated sync telemetry, while direct `clawhub install` downloads and writes local metadata without reporting an install event; changing that boundary needs maintainer product/privacy direction rather than cleanup closure.

## What This Changes

The issue requests that direct `clawhub install` report install events so `installsAllTime` and `installsCurrent` no longer remain zero when downloads increase.

## Best Possible Solution

Maintainers should choose sync-only versus immediate install reporting, then update CLI behavior, backend endpoint semantics, docs/specs, and tests around the chosen privacy and active-install model.

## Reproduction Assessment

Yes. Source inspection shows direct `cmdInstall` does not send install telemetry, and the live example API still returns non-zero downloads with zero install counters; I did not run an install because this review is read-only.

## Solution Assessment

No for an automatic code change. Adding a direct POST is mechanically narrow, but it changes documented sync-time telemetry and current-install semantics, so maintainers should decide the product/privacy model first.
