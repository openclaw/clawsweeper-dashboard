## Summary

Keep open: current main still treats skill install counters as authenticated `clawhub sync` telemetry, while the requested direct `clawhub install` reporting would change telemetry timing and privacy semantics. ClawHub cleanup policy keeps issues open, and this needs maintainer product direction rather than auto-closure.

## What This Changes

Request that `clawhub install` report install events so skill `installsAllTime` and `installsCurrent` increment for direct installs.

## Best Possible Solution

Choose and document one install-counter contract: keep sync-derived reported installs and clarify that contract, or add direct install/update/uninstall reporting with the same privacy, auth, idempotency, and opt-out guarantees.

## Reproduction Assessment

Yes, from source inspection: `cmdInstall` does not call the telemetry endpoint after download and metadata writes, while `cmdSync` does. The exact claim that multiple users installed the example skill without syncing is not independently reproducible from source.

## Solution Assessment

Unclear. A direct install POST is plausible, but the current implementation relies on sync snapshots for privacy controls, uninstall detection, and current-vs-all-time semantics, so maintainers should pick the contract before implementation.
