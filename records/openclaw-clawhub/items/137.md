## Summary

Keep open. The supplied ClawHub cleanup policy says issues should remain open unless a maintainer handles them, and the current source still has a credible publish-discovery failure path: ordinary publishers can publish into hidden pending-scan state, while public search, explore, and unauthenticated inspect surfaces filter hidden skills until scan/moderation finalizes.

## What This Changes

A published skill should become discoverable via CLI search, explore, inspect, and the web dashboard after a successful publish confirmation.

## Best Possible Solution

Separate skill-level public availability from version review: keep the last approved version discoverable during new-version scans, show first-time pending skills to their owner with actionable scan status, and ensure scan finalization consistently updates moderation state, skillSearchDigest, search/list/detail hydration, CLI inspect/search/explore, and dashboard state.
