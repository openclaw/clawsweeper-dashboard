## Summary

Keep open: current main still contains a rescan-guidance path that closes labeled scan reports and tells reporters to reopen them, while the linked #1958 text reproduces that classifier path. This is a real triage workflow problem, and ClawHub policy keeps issues open for maintainer handling.

## What This Changes

The issue asks ClawHub to fix the GitHub automation loop that closes scan false-positive or rescan reports and instructs non-collaborators to reopen issues they cannot reopen.

## Best Possible Solution

Adjust Barnacle/rescan guidance so routine rescan requests get self-service instructions, while false-positive disposition appeals and no-reopen loop reports stay open for secops review with a path that works for non-collaborators.

## Reproduction Assessment

Yes. The linked #1958 text matches the rescan classifier and leads to a label that the dedicated workflow converts into comment plus close; the current meta-report text itself does not self-match, but that does not fix the linked appeal path.

## Solution Assessment

No. The current close-and-reopen guidance is not the best fit for non-collaborator reporters or nuanced static-analysis appeals; a safer solution is a secops-reviewed classifier/comment/workflow change.
