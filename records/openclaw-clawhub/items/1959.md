## Summary

This should stay open: current main still contains the scanner-dispute close loop, and linked #1958 is a concrete production example where a non-collaborator opened a duplicate because they could not reopen the automation-closed issue. The remaining fix needs maintainer/security policy direction, not conservative cleanup closure.

## What This Changes

The issue asks ClawHub to stop closing static-analyzer dispute reports with guidance that non-collaborator reporters cannot follow.

## Best Possible Solution

Define a durable analyzer-appeal path usable by non-collaborators, then update the guidance classifier, workflow, and tests so scanner disputes are not closed with unreachable reopen instructions.

## Reproduction Assessment

Yes. Source inspection shows matching scanner-dispute issues are labeled for rescan guidance, labeled issues get a comment saying to reopen after rescanning, and the same plan closes the issue; linked #1958 is a production example of the loop.

## Solution Assessment

Unclear. There is no proposed patch, and the narrow code change depends on maintainer/security policy for how analyzer appeals should work for non-collaborators.
