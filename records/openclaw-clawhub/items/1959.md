## Summary

This issue should stay open: the linked discussion gives a concrete example of the loop, and current main still has automation that comments with rescan/reopen guidance and then closes labeled rescan-guidance issues. For this repository, issue cleanup is not eligible for auto-close, and the remaining decision touches security/analyzer triage policy.

## What This Changes

Fix a ClawHub analyzer-reporting loop where automated rescan guidance closes reports and tells non-collaborators to reopen issues they cannot reopen.

## Best Possible Solution

Keep this issue open and update the rescan-guidance workflow so analyzer-dispute reports have a supported follow-up path that does not require non-collaborators to reopen closed issues.

## Reproduction Assessment

Yes. The linked #1958 report is a concrete production example, and current main deterministically comments with reopen guidance and closes any open issue carrying the r: rescan-guidance label.

## Solution Assessment

Unclear. The narrow code surface is identifiable, but maintainers need to choose the policy before implementation: stop closing these reports, change the follow-up path, or add a reviewed suppression/acknowledgement workflow.
