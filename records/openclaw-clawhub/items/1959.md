## Summary

Keep open: repository policy does not auto-close ClawHub issues, and current main still encodes the reported scanner-dispute close/reopen loop with a concrete linked production example.

## What This Changes

The issue reports that automated scanner-dispute guidance closes reports while telling non-collaborators to reopen issues they cannot reopen.

## Best Possible Solution

Define a writable follow-up path for non-collaborator skill scanner appeals, then update the rescan-guidance automation and tests to use that path while preserving secops review for suspicious findings.

Yes. The linked #1958 example plus current main provide a high-confidence reproduction: a scanner dispute is labeled r: rescan-guidance, the workflow posts reopen-after-close guidance, and the issue is closed.
