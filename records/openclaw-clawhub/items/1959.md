## Summary

This should stay open: the linked production example shows the reported loop, and current main still labels scanner-dispute reports, posts reopen-after-close guidance, and closes the labeled issue. ClawHub cleanup policy also keeps issues open in this pass.

## What This Changes

The issue reports that static-analyzer appeal issues are auto-closed with instructions to re-scan and reopen even though non-collaborators cannot reopen closed issues.

## Best Possible Solution

Update the rescan-guidance flow so scanner-dispute reports either stay open for maintainer/secops triage or route non-collaborators to a writable follow-up path, while keeping ordinary rescan-how-to guidance separate.

## Reproduction Assessment

Yes. The linked #1958 report is a concrete production example, and current main deterministically labels matching scanner appeal issues, posts reopen-after-close guidance, and closes `r: rescan-guidance` issues.

## Solution Assessment

Unclear. The code surface is narrow, but the maintainable fix depends on secops policy: keep these reports open, redirect follow-up somewhere writable, or add a reviewed analyzer acknowledgement/suppression path.
