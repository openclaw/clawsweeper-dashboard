## Summary

This issue should stay open: the linked production example shows a concrete rescan/analyzer-dispute loop, and current main still labels rescan requests for guidance, posts reopen-after-close wording, and closes labeled issues. ClawHub issue cleanup is not eligible for auto-close here, and the remaining choice touches security/analyzer triage policy.

## What This Changes

The issue asks ClawHub to fix a loop where analyzer/rescan reports are auto-closed with reopen guidance that non-collaborators cannot use.

## Best Possible Solution

Update the rescan-guidance workflow so analyzer-dispute reports have a supported follow-up path for non-collaborators, and document how reviewed credential-access patterns should be handled.

## Reproduction Assessment

Yes. Related #1958 is a concrete production example, and current main deterministically comments with reopen guidance and closes any open issue carrying r: rescan-guidance.

## Solution Assessment

Unclear. The code surface is narrow, but maintainers should choose the security triage policy before implementation: stop closing these reports, change the follow-up path, or add a reviewed suppression flow.
