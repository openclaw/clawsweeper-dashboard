## Summary

This issue should stay open: the linked production example describes a real loop, and current main still has automation that labels rescan-review requests, posts guidance telling reporters to reopen after closure, and then closes the labeled issue. ClawHub cleanup policy also says issues should remain open rather than being auto-closed by this review pass.

## What This Changes

The issue requests fixing the rescan/static-analysis reporting workflow so non-collaborator skill authors are not trapped in a close-and-reopen loop.

## Best Possible Solution

Update the rescan-guidance flow so analyzer-dispute reports have a usable follow-up path for non-collaborators without requiring them to reopen a closed issue.

## Reproduction Assessment

Yes. The linked #1958 report is a concrete production example, and current main deterministically labels matching rescan requests, posts reopen-after-close guidance, and closes r: rescan-guidance issues.

## Solution Assessment

Unclear. The code surface is narrow, but the best fix depends on maintainer/security policy: stop closing these reports, change the follow-up destination, or add a reviewed analyzer acknowledgement path.
