## Summary

Keep open: current main still contains the scanner-dispute close/reopen loop, and linked #1958 is a concrete production example from the same non-collaborator reporter. This needs maintainer/secops direction rather than cleanup closure.

## What This Changes

The issue reports that ClawHub scanner-dispute issues are auto-closed with instructions to rescan and reopen, but non-collaborator reporters cannot reopen closed issues.

## Best Possible Solution

Define a non-permission-gated scanner appeal path for skill owners, then update Barnacle, the rescan guidance workflow, and tests so disputes are not closed with unreachable reopen instructions.

## Reproduction Assessment

Yes. The linked #1958 report is a production example, and current main still labels matching scanner-dispute issues, posts reopen-after-close guidance, and closes the labeled issue.

## Solution Assessment

Unclear. There is no proposed patch, and the narrow implementation depends on maintainer/secops policy; the current close-and-reopen guidance is not sufficient for non-collaborator reporters.
