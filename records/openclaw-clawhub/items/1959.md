## Summary

This issue should stay open: it reports a real workflow failure in the ClawSweeper/triage loop, but repository policy for openclaw/clawhub only permits auto-closing PRs that are implemented on main, not issues. The linked #1958 discussion gives a concrete example where rescan guidance told a non-collaborator to reopen a closed issue they could not reopen.

## What This Changes

The issue asks for the automated issue-reporting and rescan guidance loop to stop closing repeated analyzer-dispute reports in a way that non-collaborators cannot recover from.

## Best Possible Solution

Keep this issue open and update the bot guidance/closure workflow so analyzer-dispute reports either remain reopenable/actionable for non-collaborators or direct reporters to a supported follow-up path that does not create duplicate-closure loops.

## Reproduction Assessment

Yes. The provided #1958 context is a concrete reproduction of the loop: a non-collaborator followed the rescan/reopen guidance, could not reopen the closed report, and opened a duplicate that was also closed.
