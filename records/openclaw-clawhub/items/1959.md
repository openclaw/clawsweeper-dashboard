## Summary

This issue should stay open: current main still has the reported scanner-dispute loop, and the linked #1958 report is a concrete production example from the same non-collaborator. Because this is a ClawHub issue and touches security/analyzer triage policy, it is not eligible for cleanup closure.

## What This Changes

The issue reports that analyzer-dispute issues are auto-closed with guidance to reopen, but non-collaborator authors cannot reopen them and must file duplicates.

## Best Possible Solution

Define a non-collaborator follow-up path for scanner disputes, then update the rescan-guidance automation and tests to use that path while preserving secops review for suspicious findings.

Yes. The linked #1958 report is a concrete production example, and current main deterministically labels matching scanner-dispute issues, posts reopen-after-close guidance, and closes the labeled issue.
