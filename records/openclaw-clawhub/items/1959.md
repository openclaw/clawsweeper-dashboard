## Summary

This should stay open: current main still contains the rescan-guidance close/reopen loop, and linked #1958 is a concrete production example from the same non-collaborator reporter. The fix needs maintainer/secops policy for scanner appeals rather than cleanup closure.

## What This Changes

Report that ClawHub's automated rescan-guidance closure tells non-collaborator skill authors to reopen issues they may not be allowed to reopen.

## Best Possible Solution

Define a durable scanner-appeal path usable by non-collaborators, then update the rescan guidance classifier, workflow, and tests so disputes are not closed with an unreachable reopen instruction.

## Reproduction Assessment

Yes. The linked #1958 report is a production example, and current main still labels matching scanner-dispute issues, posts reopen-after-close guidance, and closes the labeled issue.

## Solution Assessment

Unclear. There is no proposed patch, and the narrow implementation depends on maintainer/secops policy; the current close-and-reopen guidance is not sufficient for non-collaborator reporters.
