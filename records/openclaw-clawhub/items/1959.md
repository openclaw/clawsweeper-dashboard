## Summary

This should stay open: the current main branch still closes rescan-guidance issues while telling reporters to reopen them, and linked #1958 is a concrete production example from the same non-collaborator reporter. ClawHub policy also says issues should stay open unless a maintainer handles them.

## What This Changes

The reporter asks ClawHub to fix the static-analysis issue-reporting loop where automation closes scanner-dispute issues and directs non-collaborators to a reopen path they cannot use.

## Best Possible Solution

Define a scanner-appeal path that non-collaborators can use after rescans, then update the guidance classifier, close behavior, docs, and tests so scanner-dispute issues are not closed with permission-gated reopen instructions.

## Reproduction Assessment

Yes. The linked #1958 report is a production example, and current main still labels matching scanner-dispute issues, posts reopen-after-close guidance, closes the labeled issue, and tests that behavior.

## Solution Assessment

Unclear. There is no proposed patch, and the narrow implementation depends on maintainer/secops policy; the current close-and-reopen guidance is not sufficient for non-collaborator reporters.
