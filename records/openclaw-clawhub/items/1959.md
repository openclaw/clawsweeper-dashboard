This issue should stay open: the linked scanner-dispute example still matches current automation that closes rescan-guidance issues while telling reporters to reopen, which non-collaborators may not be able to do. ## What This Changes

The issue reports that ClawHub's scanner-dispute automation closes analyzer appeal issues with reopen-after-rescan guidance even when non-collaborator reporters cannot reopen them.

## Best Possible Solution
