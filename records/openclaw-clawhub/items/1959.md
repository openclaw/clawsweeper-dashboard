## Summary

This should stay open: the issue reports a current automation loop where scanner-dispute issues are closed with reopen-after-rescan guidance that non-collaborator reporters cannot use, and linked #1958 is a concrete production example. The remaining decision is a maintainer/security policy choice for ClawHub issue automation rather than a cleanup close.

## What This Changes

The issue asks ClawHub to stop closing scanner-dispute reports with instructions that require non-collaborators to reopen a closed issue.

## Best Possible Solution

Define a durable scanner-appeal path usable by non-collaborators, then update the rescan guidance classifier, workflow, and tests so disputes are not closed with an unreachable reopen instruction.

## Reproduction Assessment

Yes. The linked #1958 report is a production example, and current main still labels matching scanner-dispute issues, posts reopen-after-close guidance, and closes the labeled issue.

## Solution Assessment

Unclear. There is no proposed patch, and the narrow implementation depends on maintainer/security policy; the current close-and-reopen guidance is not sufficient for non-collaborator reporters.
