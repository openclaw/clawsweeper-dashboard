## Summary

This should stay open: current main still routes scanner-dispute issues into guidance that closes the issue while telling non-collaborators to reopen it, and linked #1958 is a concrete production example of that loop.

## What This Changes

The issue asks ClawHub to fix the scanner issue-reporting loop where authors are told to rescan and reopen automation-closed issues they cannot reopen.

## Best Possible Solution

Define a durable scanner-appeal path usable by non-collaborators, then update the flagged-skill UI, rescan classifier/workflow, and tests so disputes are not closed with an unreachable reopen instruction.

## Reproduction Assessment

Yes. The linked #1958 report is a production example, and current main still labels matching scanner-dispute issues, posts reopen-after-close guidance, and closes the labeled issue.

## Solution Assessment

Unclear. There is no proposed patch, and the narrow fix depends on maintainer/security policy; the current close-and-reopen guidance is not a maintainable solution for non-collaborator reporters.
