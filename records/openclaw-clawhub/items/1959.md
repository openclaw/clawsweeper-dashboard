## Summary

This should stay open: current main still contains the reported loop where rescan-review issues are labeled, receive guidance telling non-collaborators to reopen the same issue, and are then closed. The linked #1958 discussion provides a concrete example of the failure mode, and ClawHub cleanup policy does not auto-close issues that are not implemented fixes.

## What This Changes

The issue reports that analyzer-dispute issues are closed with rescan/reopen guidance that non-collaborators cannot act on, causing duplicate reports to be closed in a loop.

## Best Possible Solution

Keep this open and update the rescan-guidance automation so non-collaborators get a supported follow-up path after rescanning, especially for analyzer disputes over intentional credential/API-client patterns.

## Reproduction Assessment

Yes. Current main reproduces the loop statically: a matched issue receives r: rescan-guidance, the dedicated workflow posts guidance that says to reopen the issue, and the same plan closes it; #1958 shows a non-collaborator hitting that exact path.

## Solution Assessment

Unclear. A text-only change would remove the impossible reopen instruction, but the more maintainable fix likely needs a maintainer/secops choice between not closing these reports, routing them to comments/new issues, or adding a reviewed analyzer-dispute workflow.
