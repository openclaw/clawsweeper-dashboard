## Summary

Keep open: current main still intentionally blocks GitHub OAuth sign-in for permanently deleted ClawHub accounts, so the reporter's restore/unbind request remains unresolved and needs maintainer product/security handling rather than conservative cleanup.

## What This Changes

The issue requests manual unbinding, hard deletion, or restoration of a GitHub-linked ClawHub account after self-deletion blocks sign-in.

## Best Possible Solution

Define one supported deleted-account lifecycle: either keep deletion terminal with a documented support response, or add a narrow audited restore/unbind/recreate flow with GitHub ownership verification, moderation safeguards, docs, and tests.

## Reproduction Assessment

Yes, at source level. A GitHub-linked user who runs `deleteAccount` gets `deactivatedAt`/`purgedAt`, and the later OAuth callback calls `handleDeletedUserSignIn`, whose tests assert deleted users are rejected; I did not verify the reporter's production row.

## Solution Assessment

No for ad hoc hard deletion or unbinding as the durable solution. The safer path is a maintainer-approved account lifecycle policy with audited recovery authority, identity verification, and ban/deletion separation.

## Security Review

Status: needs_attention

Summary: Deleted-account recovery is security-sensitive because it changes OAuth identity reuse, deletion guarantees, audit history, privacy semantics, and ban-bypass boundaries.

Concerns:

- **[medium] Define recovery authority before unbinding OAuth identity:** `convex/auth.ts:45`
  - body: Restoring, recreating, or unlinking a deleted GitHub-backed account would override the current irreversible-deletion contract unless identity verification and audit requirements are chosen first.
  - confidence: 0.93
- **[medium] Preserve ban and deletion separation:** `convex/auth.ts:51`
  - body: The legacy deleted-user reauth path checks ban audit records before migration, so any recovery path must keep banned and malware-autobanned provider identities from regaining access through deleted-account handling.
  - confidence: 0.9
- **[medium] Preserve deletion privacy and token semantics:** `convex/users.ts:475`
  - body: `deleteAccount` clears profile/contact/GitHub-derived fields and revokes tokens, so recovery must define what can be restored and avoid unexpectedly reviving private data or credentials.
  - confidence: 0.87
