## Summary

Keep open: current main still intentionally blocks GitHub OAuth reauth for permanently deleted ClawHub accounts, so the reporter's restore/unbind request remains unresolved. Because this is a ClawHub issue and the fix changes account lifecycle, identity binding, deletion guarantees, and support authority, it needs maintainer handling rather than conservative cleanup.

## What This Changes

The issue requests restoring, hard-deleting, or unbinding a deleted ClawHub account's GitHub OAuth identity so the same GitHub user can sign in again.

## Best Possible Solution

Define one supported post-deletion lifecycle for GitHub-linked ClawHub users: keep deletion terminal with a documented support response, or add an audited restore/unbind/recreate flow with provider ownership verification, privacy rules, moderation safeguards, docs, and tests.

## Reproduction Assessment

Yes at source level. A GitHub-linked user who runs deleteAccount gets deactivatedAt, and the later OAuth callback calls handleDeletedUserSignIn, which rejects deactivated users; I did not verify the reporter's production row.

## Solution Assessment

No for ad hoc hard deletion or unbinding as the durable repository solution. The safer path is a maintainer-approved account recovery or terminal-support policy with audit logging, identity verification, and ban/deletion separation.
