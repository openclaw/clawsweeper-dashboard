## Summary

Keep open. Current main still intentionally treats user-initiated account deletion as irreversible and blocks later GitHub OAuth for that deactivated account, so the requested recovery/unbind path remains unresolved and needs maintainer account-policy handling.

## What This Changes

The issue asks maintainers to manually unbind or hard-delete a previously deleted ClawHub account's GitHub binding so the same GitHub user can log in again.

## Best Possible Solution

Define and document the account lifecycle: either an audited support-only recovery/unbind process, or a narrow admin/product flow that verifies GitHub ownership, preserves moderation protections, and has regression coverage for web and CLI login.

## Reproduction Assessment

Yes, by source inspection on current main. deleteAccount sets deactivatedAt, the GitHub OAuth callback calls handleDeletedUserSignIn, and tests assert that deactivated users are rejected.

## Solution Assessment

Unclear. The requested hard delete or unbind may unblock this account, but the maintainable solution needs an explicit maintainer policy for identity verification, restore semantics, and moderation-bypass protections.
