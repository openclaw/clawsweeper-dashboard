## Summary

Keep open. Current main still deliberately blocks GitHub re-authentication for deleted/deactivated ClawHub accounts, the same user problem is still represented by the open canonical cluster at https://github.com/openclaw/clawhub/issues/504, and ClawHub cleanup policy keeps issues open unless maintainers resolve them.

## What This Changes

The issue asks maintainers to restore, unbind, or hard-delete a GitHub-linked deleted ClawHub account so the same GitHub user can sign in again.

## Best Possible Solution

Keep this under the canonical account-lifecycle triage and implement a maintainer-approved support-only or audited restore/unbind flow consistently across OAuth, CLI token creation, deletion, auth binding, audit logs, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt and purgedAt, OAuth calls handleDeletedUserSignIn, and active-user helpers reject the linked inactive user before web or CLI token flows can proceed.

## Solution Assessment

No for an ad hoc unbind or hard delete. The maintainable solution needs an explicit security/privacy policy and audited recovery semantics that preserve GitHub provider identity ownership and ban safeguards.
