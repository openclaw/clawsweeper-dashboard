## Summary

Keep open. Current main still intentionally blocks deleted/deactivated GitHub OAuth reauth, the related canonical cluster at https://github.com/openclaw/clawhub/issues/504 remains open, and ClawHub cleanup policy keeps issues open rather than auto-closing them as implemented.

## What This Changes

The issue requests restoring, unbinding, or hard-deleting a deleted GitHub-backed ClawHub account so the same GitHub identity can log in again.

## Best Possible Solution

Keep this open under the deleted-account OAuth recovery cluster and define either a documented support-only process or an audited restore/unbind flow covering OAuth, CLI tokens, auth account binding, operator audit, docs, and tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, the OAuth callback calls handleDeletedUserSignIn, and active-user helpers reject the linked record before web or CLI token flows can proceed.

## Solution Assessment

No for an ad hoc hard delete or OAuth unbind. The maintainable solution needs an explicit maintainer security/privacy policy and audited recovery semantics that preserve providerAccountId ownership and ban safeguards.
