## Summary

Keep open. Current main still treats deleted GitHub-backed ClawHub accounts as permanently deactivated and blocks re-authentication, while related open reports such as https://github.com/openclaw/clawhub/issues/504 show this is an unresolved account-lifecycle policy issue rather than conservative cleanup.

## What This Changes

The issue requests manual restoration, hard deletion, or GitHub identity unbinding so a deleted ClawHub account can sign in or re-register with the same GitHub identity.

## Best Possible Solution

Adopt one maintainer-approved account-lifecycle policy, then implement it consistently across OAuth, CLI token creation, deletion, auth account binding, audit/operator flows, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, the OAuth callback calls handleDeletedUserSignIn, and that helper throws before web or CLI token flows can treat the user as active.

## Solution Assessment

No for an ad hoc unbind or hard-delete change. The maintainable path needs an explicit security/privacy policy and an audited restore or re-registration flow that preserves ban and providerAccountId safeguards.
