## Summary

Keep open. Current main still treats user-initiated account deletion as irreversible and rejects later GitHub OAuth for the same deactivated account, so the requested restore/unbind path remains unresolved and needs maintainer-owned account policy.

## What This Changes

The issue requests manual unbinding, hard deletion, or restoration of a deleted ClawHub account so the same GitHub identity can sign in again.

## Best Possible Solution

Define a supported account lifecycle: either document deletion as irreversible with a support-only recovery process, or add an audited restore/unbind flow that verifies GitHub ownership and preserves moderation protections.

## Reproduction Assessment

Yes. A high-confidence source path is clear: deleteAccount marks the user deactivated, then the GitHub OAuth callback calls handleDeletedUserSignIn, which rejects deactivated users; auth tests assert that behavior.

## Solution Assessment

Unclear. The requested hard delete or unbind could unblock the reporter, but the maintainable solution needs an explicit verification and recovery policy before changing code or production data.
