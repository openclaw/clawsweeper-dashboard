## Summary

Keep open: current main intentionally blocks GitHub OAuth reauth for permanently deleted accounts, and resolving this requires a security-sensitive account recovery policy plus production support rather than conservative cleanup.

## What This Changes

The issue asks maintainers to unbind, hard-delete, or restore a deleted ClawHub account so the same GitHub identity can sign in again.

## Best Possible Solution

Define the account lifecycle policy explicitly: keep deletion irreversible with documented support-only handling, or add an audited restore/unbind flow that verifies GitHub ownership and preserves ban boundaries.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: delete a GitHub-linked account, then the same GitHub OAuth sign-in reaches a deactivated user and is rejected; I did not verify this reporter's production row.

## Solution Assessment

No for ad hoc hard deletion or unbinding as the durable fix. The safer path is a maintainer-approved, auditable recovery policy that separates self-deletion from bans and moderation actions.
