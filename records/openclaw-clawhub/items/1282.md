## Summary

Keep open. This is a valid ClawHub account-recovery issue, but current repository cleanup policy keeps issues open; current main still codifies irreversible deletion and rejects re-authentication for a deleted GitHub-backed account, while related reports such as https://github.com/openclaw/clawhub/issues/504 remain open for the broader account-lifecycle decision.

## What This Changes

The reporter asks maintainers to restore, unbind, or hard-delete a deleted GitHub-linked ClawHub account so the same GitHub identity can sign in again.

## Best Possible Solution

Define the account lifecycle explicitly, then implement either a documented support-only recovery process or a narrow audited restore/unbind/re-registration flow that preserves GitHub identity ownership and moderation safeguards.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt and purgedAt, the OAuth callback rejects deleted users, and active-user helpers exclude the linked record from web or CLI access.

## Solution Assessment

No for an ad hoc unbind or hard delete. The maintainable fix needs an explicit maintainer security and privacy decision for restore, unlink, or support-only recovery semantics.
