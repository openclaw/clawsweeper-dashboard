## Summary

Keep open. Current main still implements irreversible account deletion and rejects GitHub OAuth reauth for the linked deleted account, the same account-deletion login cluster remains open, and ClawHub cleanup policy does not auto-close issues; this needs maintainer account-lifecycle and production support handling.

## What This Changes

The reporter asks ClawHub staff to restore, hard-delete, or unlink a deleted GitHub-backed account so the same GitHub identity can sign in again.

## Best Possible Solution

Define the account lifecycle deliberately: either keep deletion irreversible with a documented support-only process, or add a narrow audited restore/unlink flow that verifies GitHub identity ownership, preserves moderation protections, and covers web plus CLI login behavior with tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt/purgedAt, GitHub OAuth calls handleDeletedUserSignIn, and tests assert deleted/deactivated users are rejected; I did not run a live production OAuth flow for this reporter’s account.

## Solution Assessment

No for ad hoc unbinding or hard deletion as the durable repo fix. A maintainable solution needs a maintainer-approved recovery policy that distinguishes self-deletion from bans or abuse-disablement and records the action audibly.
