## Summary

Keep open. Current main still intentionally treats user-initiated account deletion as irreversible and blocks later GitHub OAuth for that deactivated user, and ClawHub cleanup policy keeps issues open unless a PR is already implemented on main.

## What This Changes

The issue asks maintainers to unbind, hard-delete, or restore a deleted ClawHub account so the same GitHub identity can register or log in again.

## Best Possible Solution

Define the supported account lifecycle: keep deletion irreversible with a documented support-only exception path, or add an audited restore/unbind flow that verifies GitHub ownership and preserves moderation protections.

## Reproduction Assessment

Yes. The source path is clear: deleteAccount sets deactivatedAt and purgedAt, then afterUserCreatedOrUpdated calls handleDeletedUserSignIn, which rejects deactivated users; auth tests assert that behavior.

## Solution Assessment

Unclear. A manual hard delete or unbind could unblock one account, but the maintainable solution needs an explicit recovery policy, audit trail, identity verification, and moderation-bypass protection.
