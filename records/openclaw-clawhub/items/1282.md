## Summary

Keep open. Current main still treats user-initiated account deletion as irreversible and rejects later GitHub OAuth sign-in for the same linked record, and ClawHub cleanup policy keeps issues in maintainer hands.

## What This Changes

The issue requests restoring, hard-deleting, or unbinding a deleted ClawHub account so the same GitHub identity can sign in again.

## Best Possible Solution

Define the account lifecycle explicitly: either keep deletion irreversible with a documented support-only process, or add a narrow audited restore/unbind flow that verifies GitHub ownership and preserves moderation protections.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to delete an account, then attempt GitHub OAuth for the same auth-linked user; current main reaches handleDeletedUserSignIn and throws the deleted-account error, with tests asserting that behavior.

## Solution Assessment

No. The requested manual unbind or hard delete may resolve one user, but the safer maintainable path is a maintainer-approved recovery policy and audited implementation covering web and CLI login behavior.
