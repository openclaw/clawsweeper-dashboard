## Summary

Keep open. Current main still deliberately treats user-initiated deletion as irreversible and rejects later GitHub OAuth sign-in for the same deactivated account, and ClawHub cleanup policy keeps issues in maintainer hands.

## What This Changes

The issue requests that a deleted ClawHub account's GitHub OAuth binding be unbound, hard-deleted, or restored so the same GitHub user can sign in again.

## Best Possible Solution

Define the account lifecycle explicitly: keep deletion irreversible with a documented support-only recovery path, or add a narrow audited restore/unbind flow with GitHub ownership verification, moderation protections, and web plus CLI tests.

## Reproduction Assessment

Yes. Current main provides a high-confidence code-level reproduction: delete an account so deactivatedAt is set, then GitHub OAuth reaches handleDeletedUserSignIn and throws the permanently-deleted account error.

## Solution Assessment

Unclear. The requested manual unbind or hard delete may solve this reporter's account, but the maintainable repo solution needs maintainer-approved recovery semantics and security checks first.

- Multiple related reports are open, so maintainers may want one canonical account-lifecycle decision rather than ad hoc per-user fixes.
