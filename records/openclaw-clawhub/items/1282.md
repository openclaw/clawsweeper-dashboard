## Summary

Keep open. Current main still blocks GitHub sign-in for deleted/deactivated accounts, and this open ClawHub issue needs a maintainer account-recovery policy rather than conservative cleanup; https://github.com/openclaw/clawhub/issues/504 appears to be the canonical related cluster.

## What This Changes

The reporter asks maintainers to unbind, hard-delete, or restore a deleted GitHub-linked ClawHub account so the same GitHub profile can sign in again.

## Best Possible Solution

Keep this open or consolidate it under https://github.com/openclaw/clawhub/issues/504, then implement a maintainer-approved account lifecycle policy across OAuth, CLI token creation, deletion, auth account binding, audit/admin flows, docs, and regression tests.

## Reproduction Assessment

Yes, from source inspection: deleteAccount sets deactivatedAt/purgedAt, the OAuth callback invokes handleDeletedUserSignIn, and that helper throws for deactivated users. I did not exercise a live production GitHub account.

## Solution Assessment

No for an ad hoc automated unbind or hard-delete fix. The maintainable solution needs a policy-backed, audited recovery or unlink flow that preserves ban, privacy, and deletion semantics.
