## Summary

Keep open. Current main still enforces irreversible account deletion and rejects later GitHub OAuth for the same deactivated account, and ClawHub cleanup policy keeps issues open unless maintainers make the account-lifecycle decision.

## What This Changes

The reporter asks maintainers to unbind, hard-delete, or restore a deleted GitHub-linked ClawHub account so the same GitHub user can log in again.

## Best Possible Solution

Define a supported account lifecycle: either keep deletion irreversible with a documented support-only recovery path, or add an audited restore/unbind flow that verifies GitHub ownership and preserves moderation protections.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount marks the user deactivated, then the GitHub OAuth callback rejects that deactivated user; auth tests assert the rejection.

## Solution Assessment

Unclear. The requested manual hard-delete or unbind could unblock the reporter, but the maintainable solution needs an explicit identity-verification and moderation-preservation policy before changing code or production data.
