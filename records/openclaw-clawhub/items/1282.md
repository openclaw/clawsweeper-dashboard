## Summary

Keep open. Current main intentionally treats self-deletion as irreversible and blocks the same auth-linked user from signing in again, and ClawHub cleanup policy keeps issues open unless a PR is already implemented on main.

## What This Changes

The issue requests account recovery, GitHub OAuth unbinding, or hard deletion so a deleted ClawHub account can sign in again with GitHub.

## Best Possible Solution

Define the account lifecycle policy explicitly, then either keep deletion irreversible with a documented support response or add an audited restore/unbind flow that verifies GitHub ownership and preserves moderation protections.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `deleteAccount` sets `deactivatedAt`, OAuth completion calls `handleDeletedUserSignIn`, and tests assert deleted/deactivated users are rejected; I did not run a live OAuth flow in this read-only review.

## Solution Assessment

No proposed patch is under review, and ad hoc hard deletion or unbinding is not the safest maintainable fix. The narrow path is a maintainer-approved recovery policy/tool or an explicit decision to keep deletion irreversible.
