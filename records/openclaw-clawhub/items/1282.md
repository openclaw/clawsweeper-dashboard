## Summary

Keep open. Current main still treats user-initiated account deletion as permanent and blocks later GitHub OAuth sign-in for the same deactivated user, with no restore, hard-delete, or OAuth-unbind path in the app or admin tooling. This is also an open ClawHub issue, and the repository cleanup policy only permits auto-closing PRs that are certainly implemented on main.

## What This Changes

The issue requests manual unbinding, hard deletion, or restoration of a GitHub-linked ClawHub account after user-initiated deletion blocks re-registration/login.

## Best Possible Solution

Define the intended account lifecycle explicitly: either keep deletion irreversible with a documented support-only recovery process, or add a narrow audited restore/unbind flow that verifies GitHub identity ownership, preserves moderation protections, and covers web plus CLI login behavior with tests.
