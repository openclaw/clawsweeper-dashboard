## Summary

Keep open. Current main still intentionally blocks GitHub sign-in for permanently deleted ClawHub accounts, the related recovery cluster remains open, and this ClawHub issue is account-lifecycle policy/operator work rather than a conservative cleanup close.

## What This Changes

The reporter asks ClawHub to restore, hard-delete, or unbind their deleted GitHub-linked account so the same GitHub identity can log in again.

## Best Possible Solution

Define the account lifecycle explicitly: either keep deletion irreversible with documented support handling, or implement an audited restore/unlink/re-register flow that verifies GitHub ownership, preserves ban safeguards, and covers web plus CLI login with tests.

## Reproduction Assessment

Yes by source inspection. A user can delete their account through settings, which writes deactivatedAt/purgedAt, and the next GitHub OAuth completion calls handleDeletedUserSignIn and rejects the deactivated record; I did not inspect this reporter's production row.

## Solution Assessment

No for ad hoc unbinding or hard deletion as the durable repo fix. A maintainable solution needs a maintainer-approved recovery policy, identity verification, audit logging, and safeguards that distinguish self-deletion from bans or abuse actions.
