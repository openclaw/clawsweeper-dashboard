## Summary

Keep open. Current main still treats user-initiated deletion as irreversible and rejects later GitHub OAuth for the same deleted account, and ClawHub cleanup policy keeps issues in maintainer hands rather than auto-closing them.

## What This Changes

Request to restore, unbind, or hard-delete a deleted ClawHub account record so the same GitHub identity can log in again.

## Best Possible Solution

Define a supported account lifecycle: either document deletion as irreversible with a support-only recovery process, or add an audited restore/unbind flow that verifies GitHub identity ownership and preserves moderation protections.
