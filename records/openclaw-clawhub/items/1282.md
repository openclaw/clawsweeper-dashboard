## Summary

Keep open. Current main still intentionally blocks GitHub sign-in after user-initiated account deletion, and ClawHub cleanup policy keeps issues open unless maintainers choose an account-recovery policy or consolidate this under the open canonical cluster.

## What This Changes

The issue asks maintainers to restore, hard-delete, or unlink a deleted ClawHub account so the same GitHub identity can sign in again.

## Best Possible Solution

Define a maintainer-approved deleted-account recovery policy, then implement it consistently across OAuth, CLI token creation, account deletion, auth account binding, audit/operator tooling, docs, and regression tests.

## Reproduction Assessment

Yes, from source inspection: deleteAccount sets deactivatedAt/purgedAt, the OAuth callback invokes handleDeletedUserSignIn, and that helper throws for deactivated users. I did not exercise a live production GitHub account.

## Solution Assessment

No for an ad hoc hard-delete or unbind. The maintainable path needs an audited, policy-backed restore or unlink flow that preserves provider identity ownership, privacy expectations, and ban/moderation protections.
