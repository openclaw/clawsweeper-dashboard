## Summary

Keep open. Current main still treats user-initiated deletion as irreversible and rejects later GitHub OAuth sign-in for the same auth-linked record, and this cleanup policy keeps ClawHub issues in maintainer hands rather than auto-closing them.

## What This Changes

The issue requests a way to restore, hard-delete, or unbind a deleted ClawHub account so the same GitHub identity can sign in again.

## Best Possible Solution

Define the account lifecycle explicitly: either keep deletion irreversible with a documented support-only process, or add a narrow audited restore/unbind flow that verifies GitHub ownership and preserves moderation protections.

## Reproduction Assessment

Yes. A high-confidence static reproduction is: delete an account, then attempt GitHub OAuth for the same auth-linked user; current main reaches handleDeletedUserSignIn and throws the deleted-account error while tests assert that behavior.

## Solution Assessment

No. The requested manual unbind or hard delete is not the best repo-level solution by itself; the safer path is a maintainer-approved recovery policy and audited implementation covering web and CLI login behavior.
