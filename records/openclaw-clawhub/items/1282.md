## Summary

Keep open. Current main still intentionally blocks GitHub OAuth reauth for deleted/deactivated ClawHub accounts, related account-deletion sign-in reports such as https://github.com/openclaw/clawhub/issues/504 remain open, and ClawHub cleanup policy does not auto-close issues; the remaining work is account-lifecycle policy and production support, not conservative cleanup.

## What This Changes

The issue requests recovery, unbinding, or hard deletion so a deleted ClawHub account can sign in again with the same GitHub identity.

## Best Possible Solution

Define the intended account lifecycle explicitly: either keep deletion irreversible with documented support-only handling, or add a narrow audited restore/unlink flow that verifies GitHub identity ownership, preserves moderation protections, and covers web plus CLI login behavior with tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `deleteAccount` sets `deactivatedAt`/`purgedAt`, GitHub OAuth calls `handleDeletedUserSignIn`, and tests assert deleted/deactivated users are rejected; I did not run a live production OAuth flow for this reporter’s account.

## Solution Assessment

No for ad hoc unbinding or hard deletion as the durable repo fix. A maintainable solution needs a maintainer-approved recovery policy that distinguishes self-deletion from bans or abuse-disablement and records the action audibly.

Status: needs_attention
