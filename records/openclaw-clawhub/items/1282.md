## Summary

Keep open. Current main intentionally rejects GitHub re-authentication for permanently deleted ClawHub accounts, related account-deletion login reports remain open, and this issue needs maintainer account-lifecycle/security policy rather than conservative cleanup.

## What This Changes

The issue requests a way to regain GitHub login after deleting a ClawHub account whose GitHub identity remains bound to the inactive record.

## Best Possible Solution

Define the account lifecycle explicitly, then implement either a documented support-only recovery process or a narrow audited restore/unbind/re-registration flow covering OAuth, CLI login, auth binding ownership, moderation protections, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt and purgedAt, the OAuth callback rejects that user, and active-user helpers refuse sessions tied to deleted records.

## Solution Assessment

No for ad hoc unbinding or hard deletion. The maintainable solution needs a maintainer-approved account recovery/privacy/security policy before code chooses restore, rebind, hard delete, or support-only behavior.
