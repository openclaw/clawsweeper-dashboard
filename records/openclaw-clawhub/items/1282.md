## Summary

Keep open. Current main still intentionally blocks GitHub sign-in for permanently deleted ClawHub accounts, several related reports remain open, and this ClawHub issue is outside the cleanup policy’s auto-close path; the remaining work is a maintainer account-lifecycle and security decision.

## What This Changes

The reporter asks for their deleted ClawHub account’s GitHub binding to be unbound, hard-deleted, or restored so the same GitHub account can sign in again.

## Best Possible Solution

Define a maintainer-approved account recovery outcome, then either document irreversible deletion with a support path or implement an audited restore/unlink flow that preserves provider identity, bans, token revocation, ownership, and tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: settings calls deleteAccount, that mutation sets deactivatedAt/purgedAt, and GitHub OAuth completion calls handleDeletedUserSignIn, which rejects the deactivated user; I did not inspect the reporter’s live production record.

## Solution Assessment

No for ad hoc unbinding or hard deletion as the durable repo fix. The maintainable solution needs a policy-backed, audited recovery or explicit irreversible-support process that protects identity binding and moderation boundaries.
