## Summary

Keep open. Current main still blocks GitHub sign-in for deactivated/deleted accounts, and this ClawHub issue is part of an open account-recovery cluster that needs maintainer policy rather than cleanup closure.

## What This Changes

The reporter asks for a deleted ClawHub account to be restored, hard-deleted, or unbound from their GitHub identity so they can sign in again.

## Best Possible Solution

Define the account lifecycle policy explicitly, then implement an audited restore/unbind or support-only recovery path across OAuth, CLI token creation, deletion, auth binding, operator flows, docs, and tests.

## Reproduction Assessment

Yes, from source inspection: deleteAccount sets deactivatedAt/purgedAt, the OAuth callback invokes handleDeletedUserSignIn, and that helper throws for deactivated users. I did not exercise a live production GitHub account.

## Solution Assessment

No for an ad hoc hard-delete or unbind. The maintainable solution needs a maintainer-approved, audited recovery policy that preserves deletion, privacy, and ban semantics.
