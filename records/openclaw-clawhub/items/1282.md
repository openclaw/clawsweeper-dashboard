## Summary

Keep open. Current main still rejects OAuth for deactivated self-deleted users, and the durable resolution requires account-recovery policy, identity verification, and moderation safeguards before code or production data changes.

## What This Changes

The issue asks maintainers to unbind, hard-delete, or restore a deleted ClawHub account so the same GitHub identity can sign in again.

## Best Possible Solution

Define the supported account lifecycle: either keep deletion irreversible with a documented verified-support process, or add an audited restore/unbind flow that preserves moderation protections and covers web plus CLI login behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: delete an authenticated account, then attempt GitHub OAuth with the same provider account; current auth code sees deactivatedAt and rejects sign-in, with tests covering that behavior.

## Solution Assessment

Unclear. The requested hard-delete or unbind would unblock one account operationally, but the safest maintainable solution needs an explicit recovery policy, identity verification, and ban-bypass guardrails.
