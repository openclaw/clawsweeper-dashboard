## Summary

Keep open. Current main still intentionally blocks GitHub OAuth for deactivated accounts, and deciding whether to restore or unbind a deleted account is a production account-lifecycle and security policy decision.

## What This Changes

The reporter requests manual unbinding, hard deletion, or restoration of a deleted ClawHub account so the same GitHub identity can sign in again.

## Best Possible Solution

Define a supported audited account-recovery path, or explicitly keep deletion irreversible and document the production support process for affected users.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: deleteAccount marks the user deactivated, then the GitHub OAuth callback rejects that same deactivated user; auth tests assert this behavior.

## Solution Assessment

Unclear. A one-off hard delete or unbind may unblock the reporter, but the maintainable solution needs approved identity verification, restore/unbind semantics, and moderation-preservation rules.
