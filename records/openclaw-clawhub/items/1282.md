## Summary

Keep open. Current main still treats deleted GitHub-backed ClawHub accounts as permanently deactivated and rejects re-authentication, the related canonical cluster at https://github.com/openclaw/clawhub/issues/504 remains open, and ClawHub cleanup policy reserves auto-close for PRs rather than issues.

## What This Changes

The reporter asks for a manual unbind, hard delete, or restoration of a deleted GitHub-linked ClawHub account so the same GitHub user can sign in again.

## Best Possible Solution

Define the account lifecycle explicitly, then implement either a documented support-only path or a narrow audited restore/unbind/re-registration flow across OAuth, CLI token creation, deletion, auth account binding, operator audit, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: deleteAccount sets deactivatedAt and purgedAt, the OAuth callback calls handleDeletedUserSignIn, and active-user helpers reject the linked deactivated user before web or CLI token flows can proceed.

## Solution Assessment

No for an ad hoc unbind or hard delete. The maintainable solution needs an explicit maintainer security/privacy policy and audited recovery semantics that preserve providerAccountId ownership and ban safeguards.
