## Summary

Keep open: current main still intentionally rejects GitHub OAuth reauth for permanently deleted accounts, and the ClawHub cleanup policy does not auto-close issues. Resolving this needs a maintainer account-lifecycle decision plus production support handling, not conservative cleanup.

## What This Changes

The reporter asks ClawHub staff to restore, hard-delete, or unbind a deleted GitHub-linked account so the same GitHub identity can sign in again.

## Best Possible Solution

Define the account lifecycle policy explicitly: either keep deletion irreversible with a documented support-only path, or add an audited restore/unbind flow that verifies GitHub ownership and preserves ban and moderation boundaries.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: delete a GitHub-linked account, then the same GitHub OAuth sign-in reaches a deactivated user and is rejected; I did not verify this reporter's production row.

## Solution Assessment

No for ad hoc hard deletion or unbinding as the durable fix. The safer solution is a maintainer-approved, auditable recovery policy that covers identity verification, deletion privacy, and moderation bypass protections.
