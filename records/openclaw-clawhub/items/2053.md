## Summary

Keep open: the latest live Free Ride scan still returns the reported suspicious LLM classification, and changing repeated-finding suppression or severity is a protected security-policy decision rather than conservative cleanup.

## What This Changes

The issue asks ClawScan to stop repeating the same Free Ride findings across releases, honor declared skill scope, distinguish purpose-aligned behavior from actual risk, and reduce severity for documented OpenRouter/config behavior.

## Best Possible Solution

Define a maintainer-approved ClawScan policy for acknowledged, unchanged high-impact behavior, ideally using versioned behavioral fingerprints or owner acknowledgements while keeping new drift and higher-risk evidence visible.

## Reproduction Assessment

Yes for the live behavior: the public scan endpoint for Free Ride v1.0.11 returns the reported five LLM findings while static and VirusTotal are clean. I did not run the LLM scan locally against current main, so this is live/source reproducible rather than a local current-main test reproduction.

## Solution Assessment

Unclear: the requested direction is plausible but changes security labeling policy, not just a narrow code defect. Current main already has purpose-alignment guidance, so the remaining decision is how much repeated high-impact behavior should be suppressed or downgraded.
