## Summary

Keep open: current main still resolves skill publishes through the global skill slug index before the owner-migration guard, the related owner-scoped slug PR is open and conflicting rather than merged, and this issue is collaborator-authored so it is not a cleanup-close candidate.

## What This Changes

The issue asks for normal skill publishing to resolve existing rows by owner publisher plus slug so different publishers can publish the same slug without triggering migration-only reclaim behavior.

## Best Possible Solution

Use owner-publisher-scoped resolution for normal skill publish/update while keeping cross-owner reclaim behind explicit authorized migration, with focused Convex regression coverage and spec updates for the namespace contract.

## Reproduction Assessment

Yes, at source level: current main still resolves `insertVersion` through global `skills.by_slug` before the owner mismatch guard. I did not run a live Convex publish in this read-only review.

## Solution Assessment

Unclear as an implementation choice until maintainers resolve the namespace and compatibility policy. The narrow behavior should be owner-scoped publish lookup plus explicit migration-only reclaim semantics.
