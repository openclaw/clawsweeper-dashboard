## Summary

Current main still violates the documented publisher-scoped uniqueness invariant: `insertVersion` globally resolves the slug and routes a different publisher into owner migration or collision handling. The collaborator-authored issue should remain open alongside the comprehensive implementation at https://github.com/openclaw/clawhub/pull/2299, which is still open and conflicting rather than merged implementation evidence.

## What This Changes

Make normal skill publishing resolve existing rows within the target publisher namespace instead of rejecting same-slug skills owned by other publishers.

## Best Possible Solution

Land a coherent publisher-scoped identity design that resolves normal publishes by `(ownerPublisherId, slug)` and gives every legacy slug-only surface deterministic compatibility behavior or an explicit controlled ambiguity response.
