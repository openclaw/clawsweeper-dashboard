## Summary

Keep open: current main still resolves skill publishes through the global `skills.by_slug` lookup before owner checks, and the closest owner-scoped fix is an open conflicting PR that needs maintainer review.

## What This Changes

The issue requests publisher-scoped skill publish lookup so two publishers can publish the same slug without the global `skills.by_slug` check blocking the second publisher.

## Best Possible Solution

Land a reviewed owner-scoped skill slug implementation that uses `(ownerPublisherId, slug)` for normal publish/update targets, keeps migration and reclaim authority explicit, and documents/tests legacy slug-only ambiguity behavior.

## Reproduction Assessment

Yes from source inspection: current main still enters the global `skills.by_slug` lookup in `insertVersion` before comparing publisher ownership. I did not run a live Convex publish reproduction in this read-only review.
