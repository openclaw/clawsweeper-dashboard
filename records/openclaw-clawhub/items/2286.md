## Summary

Keep open: the code direction appears correct for the real public API pagination bug, but this external PR still lacks after-fix real API behavior proof before merge.

## What This Changes

This PR widens self-describing public skills list cursor decoding to accept full `getPage()` index keys, updates cursor reuse tests, and adds a changelog note.

## Best Possible Solution

Merge the scoped decoder and regression-test change after contributor-provided real API pagination proof is added; keep trending pagination as a separate API decision if maintainers want it.

## Reproduction Assessment

Yes. Current-main source shows `GET /api/v1/skills` can emit `getPage()` index keys as `nextCursor` and then reject those full keys when supplied back; the linked issue also gives a production curl reproduction at https://github.com/openclaw/clawhub/issues/2275.

## Solution Assessment

Yes for the code direction: accepting the full key only for matching self-describing cursors preserves wrong-index and legacy raw-array guards. It is not merge-ready until after-fix real API pagination proof is added.
