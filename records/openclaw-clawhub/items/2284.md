## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

No, not quite as written. Re-reading the owner before restore is the right narrow fix, but the retimestamp predicate should mirror the legacy-aware restore predicate so older rows without `moderationStatus` are not stranded.

Full review comments:

- **[P2] Retimestamp legacy ban-hidden rows:** `convex/skills.ts:6405`
  - body: The PR now restores legacy ban-hidden rows where `moderationStatus` is missing, but this later-ban retimestamp branch only matches `moderationStatus === "hidden"`. Mirror the restore predicate here by excluding only `removed`, and add that legacy retimestamp case to the regression test.
