## Summary

Keep open. The PR fixes a real stale unban-restore race, but the current head still misses legacy ban-hidden rows without `moderationStatus` in the re-ban retimestamp path, so those rows can keep an old ban marker and miss the final legitimate unban.

## What This Changes

The branch updates Convex user ban/unban skill batches to abort stale unban restore pages, retimestamp ban-hidden skills for newer bans, skip removed rows, add regression tests, document the invariant, and add a changelog entry.

## Best Possible Solution

Land the fail-closed owner recheck after the retimestamp guard covers legacy `user.banned` soft-deleted rows without restoring removed rows, with regression coverage.

## Reproduction Assessment

Yes. Current main's restore batch lacks an owner-state recheck, and the PR body includes live Convex output for the after-fix stale-page abort path; the remaining legacy retimestamp gap is source-reproducible from the PR diff and tests.

## Solution Assessment

Not yet. Re-reading the owner and keeping ban markers monotonic is the right narrow fix, but the retimestamp guard should also cover legacy `user.banned` rows without `moderationStatus`.
