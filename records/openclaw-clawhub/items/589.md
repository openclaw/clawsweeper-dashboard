## Summary

Keep open. The ClawHub cleanup policy keeps issues open, and current main still leaves the reported install 429 path reachable: anonymous downloads use an IP or `unknown` bucket, while CLI install only sends authentication when a token is already saved. The discussion workarounds are useful, but related #1410 and #349 remain open for the underlying attribution, rate-limit policy, and UX decisions.

## What This Changes

Users report `npx clawhub@latest install <skill>` failing with `Rate limit exceeded` across multiple networks and ask for a fix or clearer install/login guidance.

## Best Possible Solution

Keep this issue open while maintainers consolidate it with #1410 for anonymous-install attribution and #349 for rate-limit policy and visibility. The desired end state is reliable anonymous bucket attribution in production, a clear decision on whether install should prompt for or require login before rate-limited downloads, and install-specific 429 guidance that points users to `clawhub login` or `clawhub login --token ...` when authentication would move them to a user bucket.
