Request an admin runtimeId repair or hard-delete for `@apify/apify-openclaw-plugin` so new publishes can use runtimeId `apify-openclaw-plugin`.

## Best Possible Solution

Verify Apify publisher authority, then use an audited admin runtimeId repair from `apify` to `apify-openclaw-plugin` if collision checks pass; avoid hard-delete unless repair cannot preserve the package safely.
