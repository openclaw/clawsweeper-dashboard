## Best Possible Solution

Verify Apify ownership and run an audited admin runtimeId repair from `apify` to `apify-openclaw-plugin`, or add a narrow admin runtimeId repair route/CLI if maintainers want this repeatable outside direct internal ops.

## Solution Assessment

Unclear until maintainer/security review. The safest likely path is an audited admin runtimeId repair, not hard delete, but a maintainer must verify authority and choose direct internal repair versus exposing a runtime-only operator command.
