## Summary

## What This Changes

The issue requests an operator repair for `@apify/apify-openclaw-plugin` because its ClawHub package runtimeId is stuck at `apify` while the upstream plugin id is `apify-openclaw-plugin`.

## Best Possible Solution

Verify Apify authority, then perform an audited admin runtimeId correction for the existing package or add a narrow dry-run operator command for runtime-only repair; reserve hard-delete for a case where correction is unsafe.

## Reproduction Assessment

Yes. The live package API still reports `runtimeId: "apify"`, the upstream manifest declares `apify-openclaw-plugin`, and current main still rejects ordinary code-plugin publishes that try to change an existing runtimeId.

## Solution Assessment
