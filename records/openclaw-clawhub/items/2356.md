## What This Changes

The issue requests an Apify package identity repair by changing the stored runtimeId or hard-deleting the existing package row so the canonical npm package can publish again.

## Best Possible Solution

Verify Apify ownership, then prefer an audited admin runtimeId repair from `apify` to `apify-openclaw-plugin` if there is no active collision; add a runtimeId-specific admin command only if maintainers want this as a repeatable operator workflow.
