## What This Changes

The issue requests an admin-safe runtimeId correction or hard-delete for `@apify/apify-openclaw-plugin`, whose ClawHub row is bound to `apify` while the upstream plugin id is `apify-openclaw-plugin`.

## Best Possible Solution

Verify Apify publisher authority, confirm no active `apify-openclaw-plugin` runtimeId collision, then perform an audited admin runtimeId repair from `apify` to `apify-openclaw-plugin`; use hard-delete only if the safer repair path cannot preserve the package correctly.

## Solution Assessment

Yes for the likely repair shape. An audited admin runtimeId repair is narrower and safer than hard-delete because it preserves package history while enforcing admin authority, collision checks, and audit logging.
