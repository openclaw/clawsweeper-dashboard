Overall confidence: 0.88

Full review comments:

- **[P2] Restore the skill history and compare tabs:** `src/components/SkillDetailTabs.tsx:18`
  - body: This narrows `DetailTab` to README/files/install metadata while deleting the compare/version rendering path. Current skill pages let users compare releases, inspect older changelogs, and download older ZIPs; after this change those established flows have no replacement.
  - confidence: 0.89
- **[P2] Validate hash tabs against rendered install tabs:** `src/components/SkillDetailPage.tsx:44-53`
  - body: `tabFromHash()` accepts `runtime`, `dependencies`, `install`, and `links` unconditionally, but `SkillDetailTabs` only renders those panels when `buildSkillInstallTabs()` returns them. A stale `#runtime` or `#dependencies` URL on a skill without that metadata sets an active tab with no panel, leaving the content area blank.
  - confidence: 0.86
- **[P2] Sync the merge target after owned skills load:** `src/components/SkillOwnershipPanel.tsx:71`
  - body: `mergeTargetSlug` is initialized from `ownedSkills[0]` only on the first render. When the owned-skill query resolves from empty to non-empty, the Select can have no matching value and the merge action remains disabled.
  - confidence: 0.82
- **[P2] Use a built-in Vitest call-order check:** `src/components/DevPersonaFab.test.tsx:124`
  - body: `toHaveBeenCalledBefore` is not a Vitest matcher installed by this repo's setup, so this new test will fail at runtime or during type checking. Compare `mock.invocationCallOrder` or add an explicit local matcher.
  - confidence: 0.9
- **[P3] Collapse the duplicate security breadcrumb segment:** `src/components/SecurityScannerPage.tsx:153`
  - body: `SecurityScannerHero` renders `skills/plugins / owner / skills/plugins / artifact / scanner`, and the second collection link points back to the same collection route. That duplicate crumb makes the security report hierarchy confusing.
  - confidence: 0.78
