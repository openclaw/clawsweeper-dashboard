Full review comments:

- **[P2] Restore the skill history and compare tabs:** `src/components/SkillDetailTabs.tsx:18`
  - body: This narrows `DetailTab` to README/files/install metadata and removes the compare/version rendering path. Current skill pages let users compare releases, inspect older changelogs, and download older ZIPs; after this change those established flows have no replacement.
  - confidence: 0.9
- **[P2] Validate hash tabs against rendered install tabs:** `src/components/SkillDetailPage.tsx:44-53`
  - body: `tabFromHash()` accepts `runtime`, `dependencies`, `install`, and `links` unconditionally, but `SkillDetailTabs` only renders panels returned by `buildSkillInstallTabs()`. A stale hash on a skill without that metadata can select a tab with no panel, leaving the content area blank.
  - confidence: 0.88
- **[P2] Use a built-in Vitest call-order check:** `src/components/DevPersonaFab.test.tsx:124`
  - body: `toHaveBeenCalledBefore` is not a Vitest matcher installed by this repo's setup, so this new test will fail at runtime or type checking. Compare `mock.invocationCallOrder` or add an explicit local matcher.
  - confidence: 0.91
- **[P2] Sync the merge target after owned skills load:** `src/components/SkillOwnershipPanel.tsx:71`
  - body: `mergeTargetSlug` is initialized from `ownedSkills[0]` only on the first render. When the list loads later, the Select can hold a value that matches no item and keep the merge action disabled despite available targets.
  - confidence: 0.86
- **[P3] Collapse the duplicate skill breadcrumb segment:** `src/components/SkillHeader.tsx:227-230`
  - body: The skill hero breadcrumb renders `skills / owner / skills / slug`, so the same collection segment appears twice and the second link repeats the listing hierarchy. Render the collection segment once.
  - confidence: 0.8
- **[P3] Collapse the duplicate security breadcrumb segment:** `src/components/SecurityScannerPage.tsx:153`
  - body: The security report breadcrumb renders `skills/plugins / owner / skills/plugins / artifact / scanner`, which duplicates the collection segment and adds a confusing dead-end navigation step. Render a single collection crumb before the artifact.
  - confidence: 0.8
