## Summary

Keep open. ClawHub cleanup policy does not auto-close issues, and current main still has a source-supported CLI path where browser login reaches `/api/v1/whoami` token verification and can surface only the raw top-level `fetch failed` transport message.

## What This Changes

The issue asks ClawHub to address `npx clawhub login` opening browser auth and then failing with a generic `fetch failed` error.

## Best Possible Solution

Keep browser-token verification intact, but add centralized redacted transport diagnostics plus login-specific proxy and `clawhub login --token` fallback guidance, with regression coverage and troubleshooting docs.

Yes. A shared HTTP/error-formatting diagnostics repair is the narrowest maintainable solution because login, search, install, publish, and related CLI commands already converge through this boundary.

Fix the ClawHub CLI raw `fetch failed` diagnostics reported in https://github.com/openclaw/clawhub/issues/861 and related https://github.com/openclaw/clawhub/issues/651. Preserve browser and token login semantics: `cmdLogin` must still verify tokens through `/api/v1/whoami` before writing config. Add centralized, redacted transport diagnostics at the shared HTTP/error-formatting boundary so Undici `TypeError('fetch failed')`, DNS/TLS/proxy/timeout/abort causes, and Bun curl nonzero exits produce actionable messages instead of only `fetch failed`. Include login-specific guidance for proxy/firewall checks and the `clawhub login --token` fallback where appropriate, but do not assume every transport failure is a proxy problem. Redact Authorization headers, API tokens, callback URLs/token query parameters, proxy credentials, and private endpoints. Add regression coverage for Node fetch errors, Bun curl errors, `formatError`, login verification failure output, and search/shared command behavior as needed; update troubleshooting docs if the user-facing guidance changes. Do not change request routing, retry/rate-limit behavior, successful response parsing, or token storage semantics.
