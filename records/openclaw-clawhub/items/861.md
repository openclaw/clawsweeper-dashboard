## Summary

Keep open. Current main still has a source-supported CLI path where browser login receives a token, verifies it with `/api/v1/whoami`, and can surface an Undici `fetch failed` as only the generic top-level message; the related shared diagnostics issue at https://github.com/openclaw/clawhub/issues/651 is also still open.

## What This Changes

The issue asks maintainers to investigate `npx clawhub login` opening the browser and then failing with a generic `fetch failed` error.

## Best Possible Solution

Keep `/api/v1/whoami` verification intact, but add centralized redacted transport diagnostics plus login-specific troubleshooting and fallback guidance.

## Reproduction Assessment

Yes for a source-level path: after browser auth returns a token, make the `/api/v1/whoami` verification fetch throw `TypeError('fetch failed')`, and current code reports only `formatError(error)`. I did not reproduce the reporter's exact macOS/network environment.

## Solution Assessment

Yes. The narrow maintainable fix is sanitized diagnostics at the shared HTTP/error boundary plus login-specific docs, not bypassing token verification or saving unverified tokens.

## Repair Work Prompt

Fix the ClawHub CLI transport diagnostics gap for the login/search `fetch failed` reports in https://github.com/openclaw/clawhub/issues/861 and https://github.com/openclaw/clawhub/issues/651. Preserve auth semantics: browser, device, and token login must verify via `/api/v1/whoami` before `writeGlobalConfig`; do not save unverified tokens, skip verification, or add a new config flag. Add centralized, redacted diagnostics for fetch/Undici/curl transport failures such as DNS, TLS/certificate, proxy, timeout, abort, and generic `TypeError('fetch failed')`; include login-specific guidance for proxy env vars and `clawhub login --token` or device login fallback where appropriate. Sanitize Authorization headers, `clh_` tokens, callback URLs, token query params, proxy credentials, and environment-derived secrets. Add focused regression tests for HTTP/error formatting and login verification failure output, and update troubleshooting docs without changing unrelated CLI behavior.
