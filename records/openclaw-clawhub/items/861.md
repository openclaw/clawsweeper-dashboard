## What This Changes

The issue reports `npx clawhub login` opening browser auth and then failing with only `fetch failed`.

## Best Possible Solution

Add centralized, redacted CLI transport diagnostics for shared HTTP failures and login verification, with focused troubleshooting docs and tests while preserving `/api/v1/whoami` token verification.

## Solution Assessment

Yes. A shared, redacted HTTP/error-formatting repair with login-specific diagnostics is the narrowest maintainable solution; it should not skip `/api/v1/whoami` verification or write unverified browser tokens to config.
