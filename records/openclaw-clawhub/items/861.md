Investigate why the ClawHub CLI browser-login flow receives a token and then fails during network verification with a generic fetch error.

## Best Possible Solution

Retest login with ClawHub CLI v0.20.2 in the affected environment; if it still fails, keep token verification intact while adding sanitized endpoint and network-cause diagnostics plus actionable proxy and token-login guidance.

## Solution Assessment
