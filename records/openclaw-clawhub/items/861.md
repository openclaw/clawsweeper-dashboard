## Summary

Keep open: ClawHub cleanup policy does not auto-close issues, and current main still has a source-supported CLI login path where browser auth can succeed but `/api/v1/whoami` token verification can surface only a generic `fetch failed` message.

## What This Changes

The issue reports `npx clawhub login` opening browser auth and then failing with an unhelpful `fetch failed` error.

## Best Possible Solution

Keep browser-token verification intact, but add centralized redacted transport diagnostics plus login-specific proxy, device-login, and `clawhub login --token` guidance with regression coverage and troubleshooting docs.

## Solution Assessment

Yes. A shared, redacted HTTP/error-formatting repair with login-specific guidance is the narrowest maintainable fix because login, search, install, publish, package, and sync already converge through the same boundary.
