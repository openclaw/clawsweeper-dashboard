In the shared ClawHub CLI HTTP boundary, normalize rejected Node/Undici fetch errors into a concise message that names the target host, preserves a useful safe underlying cause or code when available, and suggests relevant DNS, TLS, proxy, or connectivity checks without exposing tokens, authorization headers, callback parameters, or sensitive request data. Ensure `clawhub login` token verification and ordinary registry commands use the same behavior. Add focused tests for representative rejected fetches, including a nested TLS/network cause, and verify existing HTTP retry and status-error behavior remains unchanged. Do not add a new authentication mode or configuration option, and do not edit `CHANGELOG.md`. Include PR-body release-note context for the user-visible diagnostic improvement.
