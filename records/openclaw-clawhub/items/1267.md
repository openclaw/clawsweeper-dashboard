## Summary

Keep open. Current main intentionally blocks deleted/deactivated ClawHub users and now shows clearer auth errors, but it still lacks a supported purge, recreate, unlink, relink, or restore path for this clustered account lifecycle problem; ClawHub issue cleanup policy also keeps issues open for maintainer handling.

## What This Changes

Request support for recovering from a deleted ClawHub account and GitHub identity mismatch that leaves OAuth returning to a logged-out loop.

## Best Possible Solution

Define one maintainer-approved post-deletion lifecycle for GitHub-linked ClawHub users, then implement it consistently across OAuth callbacks, user/auth records, admin/support tooling, user-facing errors, docs, audit logs, and regression tests.

## Reproduction Assessment

No for the exact wakeband/zocase production records because the live database was not inspected. The broader deleted-account path is source-reproducible: current auth code and tests reject deleted or deactivated users during sign-in and active-user lookup.

## Solution Assessment

Unclear. A one-off purge could unblock one user, but the maintainable solution is a deliberate account lifecycle and identity recovery policy rather than ad hoc database cleanup.
