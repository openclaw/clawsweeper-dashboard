## Summary

Keep this open. Current main still treats user-initiated deletion as terminal, rejects later auth for deleted identities, and the clustered open reports show this needs maintainer product and security direction rather than cleanup closure.

## What This Changes

Request a ClawHub account lifecycle or support path so a user who deleted an account and changed GitHub accounts can sign in again or have stale identity records purged.

## Best Possible Solution

Define one supported post-deletion lifecycle for GitHub-linked ClawHub users, then implement it consistently across OAuth callbacks, user records, admin/support tooling, user-facing errors, docs/specs, and tests.

## Reproduction Assessment

Yes, at source level: deleteAccount sets deactivatedAt/purgedAt, the GitHub auth callback rejects deactivated or legacy deleted users, and active-user helpers refuse to load those records. I did not run a live OAuth flow or inspect the named production records.

## Solution Assessment

Unclear until maintainers choose the account lifecycle policy. A one-off purge may unblock this reporter, but the maintainable solution is an explicit restore, purge/recreate, unlink/relink, or terminal support route with security review.
