## Summary

Keep open: current main intentionally blocks sign-in for deactivated/deleted users and there is still no supported recovery, relink, purge/recreate, or support-admin restore path. The related open reports make this an account-lifecycle product and security decision, not a cleanup close.

## What This Changes

Request account cleanup or recovery after deleting a ClawHub account and switching GitHub accounts because GitHub OAuth returns to the site without an active session.

## Best Possible Solution

Define one supported post-deletion lifecycle for GitHub-linked ClawHub users, then implement it consistently across OAuth callbacks, user records, admin/support tooling, user-facing errors, docs/specs, and tests.

## Reproduction Assessment

Yes, at source level for the deleted-account OAuth path: account deletion writes terminal fields, OAuth rejects deactivated users, and active-user helpers refuse those records. I did not run a live OAuth flow or inspect the named production records.

## Solution Assessment

No for the one-off purge request: it may unblock one user, but the maintainable fix is a maintainer-approved restore, purge/recreate, unlink/relink, or explicitly terminal support route with security review.
