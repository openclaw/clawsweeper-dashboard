## Summary

Keep this open. Current main still intentionally blocks deleted ClawHub users from signing in again, while the related deleted-account OAuth reports remain open and the remaining recovery/relink/purge behavior needs maintainer product and security review.

## What This Changes

Request a way to sign in, recreate, relink, or purge a ClawHub account after account deletion and GitHub account changes.

## Best Possible Solution

Define one supported post-deletion lifecycle for GitHub-linked ClawHub users, then implement it consistently across OAuth callbacks, user records, admin/support tooling, user-facing errors, docs/specs, and tests.

## Reproduction Assessment

Yes, from source inspection for the current deleted-account path: users with deactivatedAt or legacy deletedAt are rejected during GitHub auth, and active-user helpers refuse to load them. I did not verify the named production records or run a live OAuth reproduction.

## Solution Assessment

Unclear until maintainers choose the deletion policy. A one-off purge might unblock this reporter, but the maintainable fix needs a deliberate restore, purge/recreate, unlink/relink, or explicitly terminal support route.
