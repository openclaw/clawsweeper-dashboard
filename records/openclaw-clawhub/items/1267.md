## Summary

Keep open. Current main surfaces clearer auth errors, but it still intentionally treats user-initiated account deletion as terminal and lacks a supported recovery, purge/recreate, or GitHub unlink/relink path for this clustered account-lifecycle problem.

## What This Changes

The issue requests account cleanup or recovery after a deleted ClawHub account and a later GitHub OAuth login attempt leave the reporter stuck in a sign-in loop.

## Best Possible Solution

Define one supported post-deletion lifecycle for GitHub-linked users, then implement it consistently across OAuth callbacks, user/auth-account records, admin or support tooling, user-facing errors, docs/specs, audit logs, and regression tests.

## Reproduction Assessment

No for the exact production account pair because the live database was not inspected. The broader deleted-account path is source-reproducible: current main rejects deactivated users during OAuth and excludes deleted/deactivated users from active-user lookup.

## Solution Assessment

Unclear. A one-off purge could unblock one account, but the maintainable solution is a maintainer-approved account lifecycle and identity recovery policy rather than ad hoc database cleanup.
