## Summary

Keep open. Current main still intentionally blocks deleted/deactivated ClawHub accounts from completing GitHub OAuth sign-in, while this clustered account-recovery problem needs maintainer security, privacy, and production-support policy rather than cleanup closure.

## What This Changes

The issue requests a way to recover, purge, relink, or otherwise unblock GitHub OAuth login after deleting a ClawHub account and trying to sign in with another GitHub account.

## Best Possible Solution

Choose and document one deleted-account lifecycle, then implement it coherently across OAuth, auth account bindings, account deletion, admin/support tooling, user-facing errors, docs/specs, and regression tests.

## Reproduction Assessment

No for the exact wakeband/zocase production records because this review did not inspect live data. The broader deleted-account OAuth lockout is source-reproducible: deletion records deactivatedAt/purgedAt, OAuth calls handleDeletedUserSignIn, and active-user helpers reject the user.

## Solution Assessment

No for a one-off purge as the repository-level solution. The maintainable fix is a maintainer-approved restore, purge/recreate, unlink/relink, or terminal support path with explicit identity, privacy, audit, and abuse safeguards.

## Security Review

Status: needs_attention

Summary: Deleted-account recovery is security-sensitive because it changes OAuth identity reuse, deletion privacy guarantees, and ban-bypass boundaries.

Concerns:

- **[medium] Define recovery authority before changing sign-in:** `convex/auth.ts:37`
  - body: handleDeletedUserSignIn currently checks the incoming identity relationship and blocks deleted users; any restore, purge, recreate, or relink path needs equivalent provider-account verification and audit logging.
  - confidence: 0.9
- **[medium] Preserve ban and autoban separation:** `convex/auth.ts:48`
  - body: The reauth path checks ban audit records before migrating legacy deleted users, so recovery must not let banned or malware-blocked identities regain access through deleted-account handling.
  - confidence: 0.88

  - body: deleteAccount revokes tokens and purges profile/contact/GitHub-derived fields, so recovery or re-registration needs clear rules for restored data, published content ownership, audit records, and token state.
