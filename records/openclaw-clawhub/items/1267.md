## Summary

Keep open. This is an open ClawHub issue, and the repository cleanup policy only permits auto-closing pull requests that current main certainly implements. Current main still matches the broader account-deletion OAuth failure class: deleted or deactivated users are intentionally rejected during auth, active-user helpers hide them from the app session, and there is no self-service recovery, purge/recreate, or unlink/relink path. The exact wakeband/zocase production records were not inspected, so the remaining action needs maintainer account-lifecycle and support review rather than conservative closure.

## What This Changes

The issue requests a way to recover or purge stale GitHub-linked ClawHub auth state after account deletion so the reporter can sign in with an established GitHub account instead of being stuck in a login refresh loop.

## Best Possible Solution

Define one supported post-deletion lifecycle for GitHub-linked ClawHub users, then implement it consistently across OAuth callbacks, user/auth records, support/admin tooling, user-facing errors, documentation, and tests: restore, purge/recreate, unlink/relink, or an explicitly terminal state with a clear support route.
