## Summary

Keep open. Current main intentionally blocks deleted/deactivated ClawHub users from completing GitHub sign-in, while this issue asks for recovery, purge/recreate, or relink behavior that needs maintainer account-lifecycle and security policy.

## What This Changes

Issue reports a ClawHub GitHub OAuth login loop after account deletion and asks for server-side cleanup or recovery of stale auth/user records.

## Best Possible Solution

Define one supported post-deletion lifecycle for GitHub-linked ClawHub users, then implement it consistently across OAuth callbacks, user records, admin/support tooling, user-facing errors, docs/specs, and regression tests.

## Reproduction Assessment

No for the exact wakeband/zocase production records because live data was not inspected. Yes at the source level for the broader deleted-account OAuth lockout: current auth, deletion, access, specs, and tests intentionally reject deleted or deactivated users after GitHub OAuth.

## Solution Assessment

No for a one-off purge as the repository-level solution. The maintainable path is a maintainer-approved recovery, purge/recreate, unlink/relink, or explicitly terminal support flow with provider-account verification, privacy rules, audit logging, and abuse safeguards.

## Security Review

Status: needs_attention

Summary: Deleted-account recovery is security-sensitive because it changes OAuth identity reuse, deletion guarantees, token/audit semantics, and ban-bypass boundaries.

Concerns:

- **[medium] Define recovery authority before changing sign-in:** `convex/auth.ts:40`
  - body: handleDeletedUserSignIn currently verifies the incoming identity relationship and blocks deactivated users; restore, purge, recreate, or relink behavior needs equivalent provider-account verification and audit logging.
  - confidence: 0.9
- **[medium] Preserve ban and autoban separation:** `convex/auth.ts:51`
  - body: The legacy deleted-user reauth path checks ban audit records before migration, so recovery must not let banned or malware-blocked identities regain access through deleted-account handling.
  - confidence: 0.88
- **[medium] Keep deletion privacy semantics explicit:** `convex/users.ts:486`
  - body: deleteAccount purges profile/contact/GitHub-derived fields and revokes tokens, so recovery or re-registration needs clear rules for restored data, published content ownership, audit records, and token state.
  - confidence: 0.86
