## Summary

Keep open. Current main still treats user-initiated account deletion as terminal and lacks a self-service or support-admin recovery/relink path; for ClawHub issues, cleanup policy also keeps unresolved issues in maintainer hands.

## What This Changes

The issue requests a way to recover, purge, or relink GitHub-backed ClawHub account records after account deletion causes sign-in to return to the logged-out state.

## Best Possible Solution

Define one supported post-deletion lifecycle for GitHub-linked users, then implement it consistently across OAuth callbacks, user records, support/admin tooling, user-facing errors, docs, and tests.

## Reproduction Assessment

Yes for the source-level deleted-account flow: delete a ClawHub account, then reauthenticate with the same GitHub provider identity and current main rejects the deactivated user. The exact two-GitHub-account state in the reporter's production records was not reproduced.

## Solution Assessment

No, a one-off manual purge is not the best long-term fix because it leaves deletion, identity, and abuse semantics undefined. The safer solution is an explicit restore, purge/recreate, unlink/relink, or terminal-state policy with audited implementation.
