## Summary

Keep open. Current main still treats user-initiated account deletion as terminal and blocks OAuth sign-in for deleted/deactivated users, while this clustered report asks for a recovery, purge, recreate, or relink path that requires maintainer product and security policy.

## What This Changes

The issue requests a fix for ClawHub users who cannot sign in with GitHub after deleting an account and then trying to use the same or another GitHub identity.

## Best Possible Solution

Define a supported post-deletion lifecycle for GitHub-linked ClawHub accounts, then implement that policy consistently across OAuth callbacks, user records, admin/support tooling, user-facing errors, docs, specs, audit logging, and regression tests.

## Reproduction Assessment

No, not for the exact named production accounts because their live database records were not inspected. The deleted-account failure path is source-reproducible: current auth, access, and tests reject deleted/deactivated users instead of recreating or relinking them.

## Solution Assessment

Unclear. A one-off purge could unblock this reporter, but the maintainable fix is a deliberate account lifecycle and identity recovery policy rather than ad hoc database cleanup.
