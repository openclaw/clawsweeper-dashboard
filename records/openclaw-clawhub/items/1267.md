## Summary

Keep open. Current main still deliberately treats user-initiated account deletion as terminal and has no supported recovery, purge/recreate, or GitHub relink path; ClawHub issue cleanup policy also keeps issues open for maintainer handling.

## What This Changes

Allow a GitHub-linked ClawHub account that was deleted, or followed by a GitHub account switch, to recover, purge, recreate, or relink instead of ending in a login loop.

## Best Possible Solution

Maintain a single supported post-deletion lifecycle for GitHub-linked accounts, with audited recovery, purge/recreate, or relink semantics and clear user-facing messaging.

## Reproduction Assessment

Yes for the core source path: a deleted or deactivated ClawHub user who reauthenticates through GitHub is rejected by handleDeletedUserSignIn and excluded by active-user helpers. The exact wakeband/zocase production records were not inspected.

## Solution Assessment

No. A one-off manual purge may unblock one account, but it is not the best maintainable fix because deletion, identity relink, and abuse boundaries remain undefined.
