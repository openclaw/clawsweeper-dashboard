## Summary

Keep open. Current main still intentionally blocks sign-in for deleted/deactivated ClawHub accounts, and the requested recovery or purge path needs maintainer account-lifecycle, security, and production-support judgment rather than cleanup closure.

## What This Changes

The issue requests a way to recover, purge, or relink a GitHub-backed ClawHub account after deleting a prior account and then being unable to complete GitHub sign-in.

## Best Possible Solution

Define one supported post-deletion lifecycle for GitHub-linked ClawHub users, then implement it consistently across OAuth callbacks, user records, admin/support tooling, user-facing errors, docs/specs, and regression tests.

## Reproduction Assessment

No for the exact wakeband/zocase production records because live data was not inspected. Yes at the source level for the broader deleted-account OAuth lockout: current auth, deletion, access, specs, docs, and tests intentionally reject deleted or deactivated users after GitHub OAuth.

## Solution Assessment

No for a one-off purge as the repository-level solution. The maintainable path is a maintainer-approved recovery, purge/recreate, unlink/relink, or explicitly terminal support flow with provider-account verification, privacy rules, audit logging, and abuse safeguards.
