## Summary

Keep open. Current main still treats user-initiated account deletion as terminal and has no supported recovery, purge/recreate, or GitHub unlink/relink path; the exact two-account production state was not inspected, but the broader post-deletion OAuth problem remains unresolved.

## What This Changes

Requests server-side cleanup or a supported recovery/relink path for a ClawHub account stuck in GitHub OAuth login after account deletion and a GitHub account change.

## Best Possible Solution

Define one supported post-deletion lifecycle for GitHub-linked users, then implement it consistently across OAuth callbacks, auth records, admin/support tooling, user-facing errors, docs, and tests.

## Reproduction Assessment

Partly. Current source and tests give a high-confidence reproduction for a deleted or deactivated ClawHub user reauthenticating through GitHub, but not for the exact zocase-to-wakeband production record state.

## Solution Assessment

No. A one-off manual purge may unblock one account, but the maintainable solution needs an explicit deletion, recovery, and identity-relink policy keyed to immutable GitHub identity.
