## Summary

Keep open. Current main still treats user-initiated ClawHub account deletion as irreversible and blocks deleted/deactivated users during GitHub OAuth; the remaining work is identity-sensitive account recovery/support policy, not cleanup closure.

## What This Changes

The issue requests recovery or cleanup for a ClawHub login loop after deleting a GitHub-linked account and then trying to sign in with another GitHub account.

## Best Possible Solution

Define one supported post-deletion lifecycle for GitHub-linked ClawHub users, then implement it consistently across OAuth callbacks, user/auth records, admin or support tooling, user-facing errors, docs, audit logs, and regression tests.

## Reproduction Assessment

No for the exact wakeband/zocase production records because live database state was not inspected. The broader deleted-account lockout is source-reproducible: current main records permanent deactivation, rejects deleted/deactivated users during OAuth, and excludes them from active-user lookup.

## Solution Assessment

Unclear. A one-off purge could unblock one account, but the maintainable solution is a maintainer-approved account lifecycle and identity recovery policy rather than ad hoc database cleanup.
