## Summary

Keep open. Current main still treats user-initiated account deletion as terminal and blocks later OAuth sign-in for deleted/deactivated user records, while the requested recovery/purge/relink path needs maintainer product and security policy rather than conservative cleanup.

## What This Changes

The issue requests help resolving a GitHub OAuth login loop after ClawHub account deletion and a later attempt to sign in or recover using GitHub-linked account state.

## Best Possible Solution

Define one supported post-deletion lifecycle for GitHub-linked users, then implement it consistently across OAuth callbacks, user records, admin/support tooling, user-facing errors, docs, and tests.

## Reproduction Assessment

Yes for the source-level deleted-account path: current code and tests show a deactivated or legacy self-deleted user is rejected on GitHub OAuth reauth. The exact named account pair was not reproduced because production records were not inspected.

## Solution Assessment

No, a one-off manual purge is not the best durable fix. The maintainable solution is an explicit restore, purge/recreate, unlink/relink, or terminal-state support policy with audited implementation.
