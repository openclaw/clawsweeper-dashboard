## Summary

Keep open. This ClawHub issue reports a real account lifecycle gap that current main still intentionally treats as terminal deletion rather than recovery, and the repository cleanup policy does not auto-close ClawHub issues.

## What This Changes

The issue asks for recovery or server-side cleanup after deleting a ClawHub account and then being unable to sign in with a different GitHub account.

## Best Possible Solution

Define a supported post-deletion lifecycle for GitHub-linked users, then implement it consistently across OAuth callbacks, user records, support/admin tooling, user-facing errors, docs, and tests.

## Reproduction Assessment

Yes for source-level behavior: a deleted/deactivated ClawHub user who reauthenticates through GitHub is rejected by the auth callback and excluded by active-user helpers. The exact reporter account records were not inspected.

## Solution Assessment

No, a one-off manual purge is not the best long-term solution because it leaves deletion, identity relink, and abuse semantics undefined. The maintainable path is an explicit recovery, purge/recreate, unlink/relink, or terminal-state policy with audited implementation.
