## Summary

Keep this open. Current main intentionally treats user-initiated account deletion as terminal and still lacks a recovery, purge/recreate, or unlink/relink flow, so this is not a conservative cleanup close.

## What This Changes

Request to recover or purge stale ClawHub/GitHub auth state after account deletion so GitHub login can complete again.

## Best Possible Solution

Define one supported post-deletion lifecycle for GitHub-linked users, then implement it consistently in OAuth callbacks, user records, admin/support tooling, user-facing errors, docs, and regression tests.

## Reproduction Assessment

No for the exact two-account wakeband/zocase loop; that would require production auth records or an OAuth replay. Source inspection does confirm the current deleted/deactivated-account path rejects sign-in instead of recovering or recreating the account.

## Solution Assessment

No. A one-off database purge may unblock one account, but the maintainable fix is a deliberate recovery, recreate, or relink lifecycle with security policy and tests.
