## Summary

Keep this open. Current main still treats user-initiated deletion as terminal and has no supported recovery, purge/recreate, or unlink/relink path; related reports remain open and ClawHub issue policy keeps issues open for maintainer handling.

## What This Changes

The issue requests account recovery or server-side cleanup after a deleted ClawHub account leaves GitHub sign-in stuck in a logged-out refresh loop.

## Best Possible Solution

Define one supported post-deletion lifecycle for GitHub-linked users, then align OAuth callbacks, user records, admin/support tooling, UI errors, docs/specs, and tests around that policy.

## Reproduction Assessment

Yes, at source level for the deleted-account OAuth path: account deletion writes terminal fields, OAuth rejects deactivated users, and active-user helpers refuse those records. I did not run a live OAuth flow or inspect the named production records.

## Solution Assessment

No for the one-off purge request: it may unblock one user, but the maintainable fix is a maintainer-approved restore, purge/recreate, unlink/relink, or explicitly terminal support route with security review.
