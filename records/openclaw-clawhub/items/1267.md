## Summary

Keep open. Current main intentionally blocks deleted/deactivated ClawHub users from signing in and now surfaces clearer auth errors, but it still has no supported purge, recreate, unlink, relink, or restore path for this clustered account lifecycle problem.

## What This Changes

The issue requests server-side cleanup or recovery for a ClawHub GitHub OAuth login loop after deleting one ClawHub account and trying to sign in with another GitHub account.

## Best Possible Solution

Define a maintainer-approved post-deletion lifecycle for GitHub-linked ClawHub users, then implement it consistently across OAuth callbacks, user/auth records, admin or support tooling, user-facing errors, docs, audit logs, and regression tests.

## Reproduction Assessment

No for the exact wakeband/zocase production records because the live database was not inspected. The broader deleted-account path is source-reproducible: current auth code and tests reject deleted or deactivated users during sign-in and active-user lookup.

## Solution Assessment

Unclear. A one-off purge could unblock one user, but the maintainable solution is a deliberate account lifecycle and identity recovery policy rather than ad hoc database cleanup.
