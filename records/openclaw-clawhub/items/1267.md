## Summary

Keep open. Current main intentionally treats user-initiated ClawHub deletion as terminal and still lacks a supported recovery, purge/recreate, or GitHub relink path; the ClawHub cleanup policy also says issues should not be auto-closed.

## What This Changes

The issue requests a way to recover, purge, or re-register a GitHub-linked ClawHub account after account deletion leaves login blocked.

## Best Possible Solution

Define one supported post-deletion lifecycle for GitHub-linked users, then implement it across OAuth callbacks, user records, support/admin tooling, user-facing errors, docs, and regression tests.

## Reproduction Assessment

Yes for source-level behavior: a deleted/deactivated user reauthenticates through GitHub, handleDeletedUserSignIn rejects the user, and active-user helpers exclude that record. The exact reporter account records were not inspected.

## Solution Assessment

No for a one-off manual purge as the general fix; it would leave deletion, identity relink, and abuse semantics undefined. The maintainable solution is a policy-backed recovery, purge/recreate, relink, or explicitly terminal support path keyed on immutable GitHub provider IDs.
