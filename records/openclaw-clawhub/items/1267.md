## Summary

Keep open. Current main still intentionally treats deleted ClawHub accounts as terminal during GitHub OAuth, and this issue asks for account recovery or purge behavior that needs maintainer product/security/support policy rather than cleanup closure.

## What This Changes

The issue requests a way to recover, purge, recreate, or relink a ClawHub account after deletion so GitHub OAuth login stops looping and creates an active session.

## Best Possible Solution

Define one supported post-deletion lifecycle for GitHub-linked ClawHub users, then implement it consistently across OAuth callbacks, user/auth records, admin or support tooling, user-facing errors, docs, audit logs, and regression tests.

## Reproduction Assessment

No for the exact named production accounts because the live database was not inspected. The broader deleted-account login path is source-reproducible: current main marks deletion as permanent and rejects deleted/deactivated users during OAuth and active-user lookup.

## Solution Assessment

Unclear. A one-off purge could unblock one account, but the maintainable fix is a maintainer-approved account lifecycle and identity recovery policy rather than ad hoc database cleanup.
