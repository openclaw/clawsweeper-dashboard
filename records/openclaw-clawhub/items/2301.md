## Summary

Keep open: the latest patch appears directionally correct and the earlier test-harness concern is addressed, but this external PR still lacks real behavior proof and carries rollout risk around reconciliation plus hot-path star writes.

## What This Changes

The branch changes skill star/unstar mutations to patch skill and owner star counters synchronously, ignores queued legacy star events, and adds unit plus local-auth Playwright coverage.

## Best Possible Solution

Land the synchronous counter update only after redacted real-behavior proof is posted and maintainers explicitly own the reconciliation plus post-rollout Convex monitoring plan.

## Reproduction Assessment

Yes, from source inspection: on current main, starring writes a relationship row plus queued stat event, while the detail page displays denormalized `skill.stats.stars`, so refreshing before the 6-hour doc sync can show the stale count. I did not run the local-auth flow in this read-only review.

## Solution Assessment

Mostly yes: synchronously applying canonical stat deltas in the star mutations is the narrowest direct fix for refreshed detail-page counts. The remaining question is rollout safety, not a different product direction.
