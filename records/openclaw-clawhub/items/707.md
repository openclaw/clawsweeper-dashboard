## Summary

Keep open. Current main still lacks a SKILL.md lifecycle hook contract across docs, schema/parser, moderation, API, and install/detail UI, and the requested surface is security-sensitive with related open context in #129.

## What This Changes

Request to document SKILL.md lifecycle hooks, model and scan hook metadata, and surface hook permissions before install.

## Best Possible Solution

Define a maintainer-approved hook metadata and permission model, then implement typed parsing, moderation, public serialization, UI/install warnings, and docs together.

## Reproduction Assessment

Not applicable as a feature/security RFC. The current gap is high-confidence from source inspection: docs, schema, parser, moderation, serializers, and UI all lack SKILL.md lifecycle hook support.

## Solution Assessment

Unclear until maintainers decide the policy. A docs-only update is not the best path because it would document automatic code execution before validation, warnings, and moderation exist.
