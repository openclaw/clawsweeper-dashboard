## Summary

Keep open. Current main still lacks the requested SKILL.md lifecycle hook documentation, typed metadata, parser normalization, hook-specific security analysis, public API/UI surfacing, and install transparency; this is also a ClawHub issue, so the cleanup policy does not support closing it unless a strong close reason applies.

## What This Changes

Document SKILL.md lifecycle hooks and add ClawHub metadata, security analysis, API/UI, and install transparency for skills that declare them.

## Best Possible Solution

Define a maintainer-approved hook metadata and permission policy, then implement it consistently across the skill format docs, shared schema, parser/backfill path, public API, detail/install UI, static and LLM moderation context, and install warnings.

## Reproduction Assessment

Not applicable as a runtime bug. There is a high-confidence source inspection path: current docs, schema, parser, moderation, serialization, and UI surfaces do not contain a SKILL.md lifecycle hook contract.

## Solution Assessment

Yes as a product direction, but not as a docs-only change. The maintainable solution needs a policy-backed contract that models, validates, scans, serializes, and surfaces hooks before encouraging authors to publish them.
