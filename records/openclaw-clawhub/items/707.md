## Summary

Keep open. Current main still lacks SKILL.md lifecycle hook documentation, typed skill metadata, parser normalization, hook-specific moderation, and skill-page/install transparency; ClawHub cleanup policy also keeps issues open unless a strong close reason applies, and this is not implemented.

## What This Changes

Request to document SKILL.md lifecycle hooks and add registry security analysis plus UI/API transparency before users install hook-enabled skills.

## Best Possible Solution

Define a maintainer-approved hook metadata and permission policy, then implement it consistently across docs, shared schema, parser/backfill, API serialization, static and LLM moderation context, skill detail/install UI, and user-facing warnings.

## Reproduction Assessment

Not applicable as a runtime bug. There is a high-confidence source verification path: inspect current docs, schema, parser, moderation, serializers, and UI surfaces for a SKILL.md lifecycle hook contract, and current main still lacks it.

## Solution Assessment

No, a docs-only update is not the best way to solve this issue. The maintainable path is a policy-backed implementation that models, validates, scans, serializes, and surfaces hooks before encouraging authors to publish them.
