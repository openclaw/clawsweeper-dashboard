## Summary

Keep open. Current main still lacks a first-class SKILL.md lifecycle hook contract across docs, schema/parser normalization, static/LLM moderation context, public serialization, and skill install/UI disclosure, and the related hook issues are complementary rather than superseding this broader security RFC.

## What This Changes

The issue requests documenting SKILL.md hooks and adding registry parsing, security analysis, API/UI disclosure, and install consent or warnings for lifecycle hook declarations.

## Best Possible Solution

Define a maintainer-approved hook metadata and permission policy first, then implement docs, schema/parser/backfill, public API/UI disclosure, static and LLM moderation context, and install warnings or consent together.

## Reproduction Assessment

Yes, as a source-level verification rather than a runtime bug: current main can be inspected to confirm hooks are absent from docs, schema/parser normalization, static and LLM moderation context, public serialization, and install metadata UI.

## Solution Assessment

Yes in broad direction, but not as a docs-only change. The maintainable path is policy first, followed by coordinated metadata, moderation, API, UI, and consent support.
