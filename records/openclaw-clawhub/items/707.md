## Summary

Keep open. Current main still does not document or normalize SKILL.md lifecycle hooks, does not run hook-specific security checks, and does not expose hook metadata on public API, UI, or install surfaces; the related hook issues are adjacent but do not supersede this broader security RFC.

## What This Changes

The issue requests SKILL.md hook documentation plus registry parsing, security analysis, and user-visible disclosure for lifecycle hook declarations.

## Best Possible Solution

Define a maintainer-approved hook metadata and permission policy first, then implement docs, schema/parser/backfill, public API/UI disclosure, static and LLM moderation context, and install warnings together.

## Reproduction Assessment

Yes. This is source-reproducible rather than a runtime bug: current main can be inspected to confirm hooks are absent from docs, schema/parser normalization, static checks, public serialization, and install metadata UI.

## Solution Assessment

Yes in broad direction, but not as a docs-only change. The maintainable path is policy first, followed by coordinated metadata, moderation, API, UI, and consent support.
