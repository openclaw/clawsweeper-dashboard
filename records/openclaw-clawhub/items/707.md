## Summary

Keep open. Current main still lacks first-class SKILL.md lifecycle hook documentation, schema/parser normalization, hook-specific moderation, public API serialization, and skill-page/install transparency; the visible `hooks` support is for package/plugin capability metadata, which is a separate surface. This is also a ClawHub issue, and the cleanup policy for this repo keeps issues open unless a strong close reason applies.

## What This Changes

Document SKILL.md lifecycle hooks, analyze them as security-sensitive metadata, and surface hook declarations in ClawHub skill/API/UI surfaces.

## Best Possible Solution

Define a maintainer-approved hook metadata and permission policy first, then implement it consistently across docs, schema/parser/backfill, public API, skill detail/install UI, static and LLM moderation context, and install warnings.

## Reproduction Assessment

Not applicable as a bug reproduction because this is an RFC/security feature request. Source inspection gives high confidence that the requested hook metadata contract is absent from the relevant current-main surfaces.

## Solution Assessment

Yes in broad direction, but not as a docs-only change. The maintainable path is to settle the hook trust and permission contract first, then add docs, schema/parser support, moderation checks, API/UI surfacing, and install warnings together.

## Security Review

Status: needs_attention

Summary: Lifecycle hooks are a security-sensitive skill execution surface that current main does not parse, disclose, or evaluate as first-class skill metadata.

- **[high] Define hook execution trust boundary:**
