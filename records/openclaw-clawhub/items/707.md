## Summary

Keep open. Current main still does not document, parse, validate, scan, serialize, or surface SKILL.md lifecycle hooks, and the ClawHub cleanup policy only auto-closes PRs that are certainly implemented on main.

## What This Changes

The issue asks ClawHub to document SKILL.md lifecycle hooks and add metadata parsing, security analysis, API/UI surfacing, and install-time transparency for hook declarations.

## Best Possible Solution

Define a maintainer-approved hook metadata and permission policy, then implement it consistently across docs, shared schema, parser/backfill paths, public API, skill detail/install UI, static and LLM moderation context, and user-facing warnings for high-privilege events or hook types.

## Reproduction Assessment

Not applicable as a runtime bug. There is a high-confidence source verification path: inspect current docs, schema, parser, moderation, serialization, and UI surfaces for a SKILL.md lifecycle hook contract, and those checks show it is absent.

## Solution Assessment

Yes in broad direction, but not as a docs-only patch. The maintainable solution needs policy decisions first, then a typed, scanned, serialized, and disclosed hook model rather than merely documenting an execution surface.
