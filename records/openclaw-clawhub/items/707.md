## Summary

Keep open. Current main still lacks a documented and typed SKILL.md lifecycle hook contract, hook-specific security analysis, API/UI surfacing, and install transparency; this is a security-sensitive feature request, and ClawHub policy keeps issues open unless a strong close reason applies.

## What This Changes

The issue requests documentation, validation, security analysis, and UI/API disclosure for SKILL.md lifecycle hook declarations.

## Best Possible Solution

Define a maintainer-approved hook metadata and permission policy, then implement it across docs, schema, parser/backfill, public API, skill detail/install UI, static and LLM moderation, and install warnings.

## Reproduction Assessment

Not applicable as a runtime bug; this is a security/metadata feature request. Source inspection gives a high-confidence verification path: the docs, schema, parser, scanner, API, and UI surfaces lack SKILL.md lifecycle hook support.

## Solution Assessment

No for a docs-only slice; that would be too narrow for an automatic-execution surface. The maintainable solution is a policy-backed implementation that models, validates, scans, serializes, and surfaces hooks before authors are encouraged to publish them.
