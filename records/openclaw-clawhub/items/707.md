## Summary

Keep open. Current main still lacks first-class SKILL.md lifecycle hook documentation, metadata parsing/schema support, API/UI disclosure, and hook-specific security treatment; this is also a ClawHub issue, and the cleanup policy only auto-closes PRs that are certainly implemented on main.

## What This Changes

The issue requests documenting, parsing, scanning, and displaying SKILL.md lifecycle hook declarations so users can assess hook behavior before installing a skill.

## Best Possible Solution

Define a maintainer-approved hook metadata and permission policy first, then implement docs, schema/parser/backfill, API serialization, skill detail/install UI, static and LLM moderation context, and high-privilege warnings together.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a security-sensitive RFC, not a broken existing contract. Source inspection gives high confidence that current main lacks first-class SKILL.md hook metadata support.

## Solution Assessment

Yes in broad direction, but not as a docs-only change. The safer path is to settle hook trust, event/type, blocking, and consent policy before adding documentation, parsing, moderation, API, and UI disclosure.
