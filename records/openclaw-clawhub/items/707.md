## Summary

Current main still lacks SKILL.md hook support across the docs, schema, parser, moderation, public serialization, and skill install/detail surfaces. The existing `hooks` code is plugin/package capability metadata, not the SKILL.md lifecycle hook contract requested here, so this should stay open for maintainer security/product decisions rather than conservative cleanup.

## What This Changes

The issue requests first-class documentation, parsing, security analysis, API/UI disclosure, and install-time transparency for SKILL.md lifecycle hook declarations.

## Best Possible Solution

Define a maintainer-approved hook metadata and permission policy first, then implement docs, schema/parser/backfill, public API/UI disclosure, static and LLM moderation context, and install warnings together.

## Reproduction Assessment

Yes for source verification, not as a runtime bug: current main can be inspected to confirm hooks are absent from docs, schema/parser normalization, static checks, public serialization, and install metadata UI.

## Solution Assessment

Yes in broad direction, but not as a docs-only change. The maintainable path is policy first, followed by coordinated metadata, moderation, API, UI, and consent support.
