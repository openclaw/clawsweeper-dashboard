## Summary

Keep open. Current main still lacks a SKILL.md lifecycle hook contract across docs, schema/parser, moderation, API serialization, and install/detail UI, and this security-sensitive RFC needs maintainer policy before implementation.

## What This Changes

This issue asks ClawHub to document SKILL.md lifecycle hooks and add parsing, security analysis, API/UI disclosure, and install-time transparency for them.

## Best Possible Solution

Define a maintainer-approved hook metadata and permission policy, then implement it consistently across skill format docs, shared schema/parser, API serialization, moderation, detail UI, and install warnings.

## Reproduction Assessment

Not applicable as a runtime bug; yes as a source verification path. Current main can be inspected across docs, schema, parser, moderation, serialization, and UI surfaces, and the SKILL.md hook contract is still absent.

## Solution Assessment

Unclear until maintainers decide the hook policy. A docs-only update would be incomplete because lifecycle hooks need typed modeling, validation, security analysis, API/UI disclosure, and install-consent behavior.
