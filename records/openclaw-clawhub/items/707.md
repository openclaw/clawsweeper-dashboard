## Summary

Keep open. Current main still lacks a SKILL.md lifecycle hook contract across the format docs, shared metadata schema/parser, moderation scanner, public API, and skill detail/install disclosure, so this security and product-policy RFC remains necessary.

## What This Changes

The issue requests documented, parsed, scanned, and user-visible support for SKILL.md lifecycle `hooks` metadata, including security analysis and registry/install transparency.

## Best Possible Solution

Define a maintainer-approved hook metadata and permission policy first, then implement docs, schema/parser/backfill, public API/UI disclosure, static and LLM moderation context, and install warnings or consent together.

## Reproduction Assessment

Yes, as a source-level verification: current main can be inspected to confirm lifecycle hooks are absent from the skill format docs, metadata schema/parser, moderation scanner, public serialization path, and install/detail UI surfaces.

## Solution Assessment

Yes in direction, but not as a docs-only fix. The maintainable path is a policy decision first, followed by coordinated metadata, moderation, API, UI, and consent support.
