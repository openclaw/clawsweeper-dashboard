## Summary

Keep open. Current main still lacks a SKILL.md lifecycle hook contract across docs, metadata parsing/schema, hook-specific moderation, API/UI disclosure, and install transparency; the existing `hooks` hits are package/plugin capability metadata, which is a separate surface.

## What This Changes

Document and model SKILL.md lifecycle hooks, add hook-specific security analysis, and surface hook declarations in ClawHub API/UI/install flows.

## Best Possible Solution

Define a maintainer-approved hook metadata and permission policy first, then implement it consistently across the format docs, shared schema, parser/backfill path, API, skill detail/install UI, static moderation, and LLM moderation context.

## Reproduction Assessment

Not applicable as a runtime bug. There is a high-confidence source verification path: current main can be inspected across docs, schema, parser, moderation, API, and UI, and the requested SKILL.md hook contract is still absent.

## Solution Assessment

No, a docs-only change is not the best way to solve this issue. The maintainable fix needs a policy-backed model that validates, scans, serializes, and surfaces hooks before encouraging authors to publish them.
