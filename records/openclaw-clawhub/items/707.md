## Summary

Keep open. Current main still does not document, parse, moderate, serialize, or display SKILL.md lifecycle hook metadata, and this security-sensitive RFC needs maintainer policy decisions before implementation.

## What This Changes

The issue requests documentation, security analysis, and registry UI disclosure for Claude Code `hooks` declarations in SKILL.md frontmatter.

## Best Possible Solution

Define a maintainer-approved hook metadata and permission policy, then implement it consistently across docs, schema/parser/backfill, public API, moderation, install warnings, and skill detail UI.

Not applicable as a bug reproduction because this is an RFC/security feature request. Source inspection gives high confidence that the requested hook metadata contract is absent from the relevant current-main surfaces.

## Solution Assessment

Yes in broad direction, but not as a docs-only change. The maintainable path is to settle the hook trust and permission contract first, then add docs, schema/parser support, moderation checks, API/UI surfacing, and install warnings together.

## Security Review

Status: needs_attention

Summary: The issue is security-sensitive because lifecycle hooks can run automatic command/http/prompt/agent behavior while current main does not parse or disclose them as first-class skill metadata.

Concerns:

- **[high] Define hook execution trust boundary:**
