## Summary

Keep open. Current main still lacks SKILL.md lifecycle hook documentation, typed metadata, parser normalization, hook-specific security analysis, API/UI surfacing, and install transparency; ClawHub cleanup policy also keeps issues open unless a strong close reason applies.

## What This Changes

The issue requests a documented, typed, moderated, and user-visible SKILL.md lifecycle hooks contract for ClawHub.

## Best Possible Solution

Define a maintainer-approved hook metadata and permission policy first, then implement it across docs, shared schema, parser/backfill, public API, static and LLM moderation, install/detail UI warnings, and tests.

## Reproduction Assessment

Not applicable as a runtime bug, but there is a high-confidence verification path: inspect current main docs, schema, parser, moderation, serializers, and UI surfaces for any SKILL.md lifecycle hook contract. Those checks show the requested support is still absent.

## Solution Assessment

No, a docs-only update is not the best way to solve this. The safer solution is a policy-backed implementation that models, validates, scans, serializes, and surfaces hooks before encouraging authors to publish them.
