## Summary

Keep open. Current main still lacks the requested SKILL.md lifecycle hook contract in the format docs, shared schema/parser normalization, hook-specific moderation, public serialization, and install/detail UI disclosure; repository `hooks` hits are package/plugin capability metadata or unrelated web/test hooks. This is also a ClawHub issue, and the cleanup policy keeps issues open for maintainer handling.

## What This Changes

The issue requests documentation, security analysis, and registry disclosure for Claude Code `hooks` declarations in SKILL.md frontmatter.

## Best Possible Solution

Define a maintainer-approved hook metadata and permission policy first, then implement it consistently across the skill format docs, shared schema, parser/backfill path, public API, skill detail/install UI, static and LLM moderation context, and install warnings.

## Reproduction Assessment

Not applicable as a runtime bug. There is a high-confidence source verification path: current main can be inspected across docs, schema, parser, moderation, API, and UI, and the requested SKILL.md hook contract is still absent.

## Solution Assessment

No, a docs-only update is not the best way to solve this issue. The maintainable path needs a policy-backed model that validates, scans, serializes, and surfaces hooks before encouraging authors to publish them.
