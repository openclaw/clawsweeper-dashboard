## Summary

Keep open. Current main still lacks a documented and modeled SKILL.md lifecycle hook contract: hooks are not in the skill format docs, shared Clawdis schema, parser-normalized metadata, public skill serializers, static hook-specific moderation checks, or skill install/detail UI. The only current `hooks` metadata I found is for package/plugin capabilities, which is a separate surface, and this issue is security-sensitive enough to need maintainer policy decisions before implementation.

## What This Changes

The issue requests documenting SKILL.md lifecycle hooks and adding schema, security-analysis, API, and UI transparency for hook declarations.

## Best Possible Solution

Define a maintainer-approved hook metadata and permission policy first, then implement it consistently across the skill format docs, shared schema, parser/backfill path, public API, skill detail/install UI, static moderation, LLM moderation context, and warnings for high-privilege events or hook types.
