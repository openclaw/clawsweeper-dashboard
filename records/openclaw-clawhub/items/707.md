## Summary

Keep open. Current main still does not document SKILL.md lifecycle hooks, normalize hook metadata, run hook-specific scanner checks, serialize hook data on skill APIs, or disclose hooks in the skill install/detail UI; the existing `hooks` surface is package/plugin capability metadata, which is separate from the requested SKILL.md frontmatter contract.

## What This Changes

Request to document SKILL.md lifecycle hooks and add registry security analysis, API/UI disclosure, and install-time transparency for hook declarations.

## Best Possible Solution

Define a maintainer-approved hook metadata and permission policy first, then implement docs/specs, schema/parser/backfill, static and LLM moderation context, public API/UI disclosure, and install consent or warnings together.

## Reproduction Assessment

Yes, as a source-level verification rather than a runtime bug: current main can be inspected to confirm hooks are absent from docs, schema/parser normalization, static checks, public serialization, and install metadata UI.

## Solution Assessment

Yes in broad direction, but not as a docs-only change. The maintainable path is policy first, followed by coordinated metadata, moderation, API, UI, and consent support.
