## Summary

Keep open. Current main still does not document, parse, validate, scan, serialize, or visibly disclose SKILL.md lifecycle hooks, and this security-sensitive automatic-code-execution surface needs maintainer policy before implementation.

## What This Changes

The issue requests a documented and security-reviewed SKILL.md `hooks` metadata contract, including parser/schema support, moderation checks, public disclosure, and registry UI/install visibility.

## Best Possible Solution

Define a maintainer-approved hook metadata and permission policy first, then implement it consistently across the skill format docs, shared schema, parser/backfill path, public API, skill detail/install UI, static and LLM moderation, and install warnings for high-privilege events and hook types.

Not applicable as a runtime bug; this is a security/product RFC. The source verification path is high-confidence because the current docs, schema, parser, moderation, serialization, and UI surfaces can be inspected and the SKILL.md hook contract is still absent.

## Solution Assessment

Unclear until maintainers decide the supported hook policy. A docs-only update would not be the best solution because lifecycle hooks need typed modeling, validation, security analysis, API/UI disclosure, and install-consent behavior.
