## Summary

Keep open: current main still lacks a documented, typed, parsed, moderated, serialized, or user-visible SKILL.md lifecycle hook contract, and the related security policy questions are still unresolved.

## What This Changes

The issue requests documenting SKILL.md lifecycle hooks, modeling them in ClawHub metadata, adding hook-specific security analysis, and surfacing hook declarations in the registry UI.

## Best Possible Solution

Define a maintainer-approved hook metadata and permission policy first, then implement it consistently across docs, shared schema, parser/backfill, public API, moderation, and skill detail/install surfaces.

## Reproduction Assessment

Not applicable as a runtime bug report; this is a feature/security RFC. Source inspection gives a high-confidence verification path: current main has no documented or normalized SKILL.md hook contract and no hook-specific scanner/UI handling.

## Solution Assessment

No, a docs-only update would not be the best solution. The safer path is policy approval for supported events, hook types, blocking behavior, and consent before implementing the full typed and visible lifecycle-hook surface.
