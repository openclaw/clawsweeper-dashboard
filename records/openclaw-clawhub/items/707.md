## Summary

Current main and v0.21.0 still lack a supported SKILL.md lifecycle-hook contract, hook-specific validation, install consent, and pre-install disclosure. Existing generic artifact scanning and plugin-manifest hook capabilities do not solve those gaps, and https://github.com/openclaw/clawhub/issues/2295 only covers automatic registration of OpenClaw hook bundles. Likely related people: Patrick Erichsen is the highest-confidence routing candidate for recent format, scanning, and registry work; Peter Steinberger is also relevant to the package/plugin registry model.

## What This Changes

Define secure lifecycle-hook metadata for skills and surface it throughout publishing, scanning, installation, APIs, and the registry UI.

## Best Possible Solution

Approve a durable, runtime-aware hook threat model and metadata contract, then implement structured extraction, validation, scan context, API/UI disclosure, and explicit installation consent without silently enabling executable hooks.

## Reproduction Assessment

Yes. Current-main source inspection shows no structured SKILL.md lifecycle-hook contract or hook-specific disclosure, while existing hook support belongs to distinct plugin and OpenClaw bundle mechanisms.

## Solution Assessment

No, a documentation-only acknowledgement of Claude Code syntax is not the best complete solution. The maintainable path starts with a supported runtime contract and explicit validation, consent, and disclosure semantics.

## Risks / Open Questions

- Claude Code frontmatter hooks, OpenClaw hook bundles, and plugin-manifest hooks have different lifecycle and trust semantics; treating them as one field could create a misleading or incompatible registry contract.
- Documenting executable hooks before defining allowed events, hook types, blocking behavior, consent, and installation semantics could increase use of an attack surface that ClawHub cannot yet explain or govern.
- Generic file scanning may inspect hook scripts but cannot provide declaration matching, event privilege classification, or informed pre-install disclosure without structured metadata.
