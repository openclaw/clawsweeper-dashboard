## Summary

Keep open. Current main still does not document, normalize, moderate, serialize, or display SKILL.md lifecycle hook metadata, and this security-sensitive ClawHub issue needs maintainer policy decisions before implementation.

## What This Changes

Request to document and secure SKILL.md lifecycle hook metadata, including schema/parser support, hook-specific scanning, API/UI transparency, and install-time disclosure.

## Best Possible Solution

Define a maintainer-approved hook metadata and permission policy, then implement it consistently across docs, schema, parser/backfill, public API, skill detail/install UI, static and LLM moderation context, and warnings for high-privilege hook behavior.

## Reproduction Assessment

Not applicable as a bug reproduction because this is an RFC/security feature request. Source inspection against current main shows the requested hooks contract is absent from docs, schema, parser, scanner, API serialization, and skill UI.

## Solution Assessment

No for a docs-only implementation. The maintainable path is to settle the hook trust and permission contract first, then add docs, schema/parser support, moderation checks, API/UI surfacing, and install warnings together.

Status: needs_attention
