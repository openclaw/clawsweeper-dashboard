## Summary

Keep open. Current main still does not document SKILL.md lifecycle hooks, normalize them into skill metadata, run hook-specific moderation checks, or expose hook declarations on public skill API/UI surfaces; this ClawHub issue also needs maintainer security/product judgment rather than cleanup closure.

## What This Changes

This issue requests documenting SKILL.md lifecycle hooks and adding registry security analysis plus UI/API transparency for hook declarations.

## Best Possible Solution

Define a maintainer-approved hook metadata and permission policy, then implement it consistently across docs, schema/parser, public API, skill detail/install UI, static and LLM moderation context, and install warnings.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a security/product feature request. Source inspection on current main confirms the requested docs, metadata normalization, hook-specific scanning, and hook UI/API surfaces are still absent.

## Solution Assessment

Unclear until maintainers decide the supported hook contract and consent model. A docs-only update is not the best solution because it would advertise a powerful lifecycle execution surface before ClawHub can parse, review, disclose, and warn about it consistently.
