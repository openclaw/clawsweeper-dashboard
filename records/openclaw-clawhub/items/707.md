## Summary

Keep open. Current main still does not define, parse, scan, serialize, or display SKILL.md lifecycle hook metadata, and this is a security/product RFC where ClawHub cleanup policy keeps issues open for maintainer handling.

## What This Changes

Request to document SKILL.md lifecycle hooks, add hook-specific security analysis, and surface hook declarations in the registry UI before install.

## Best Possible Solution

Define a maintainer-approved hook metadata and permission policy first, then implement it across docs, shared schema, parser/backfill, public API, skill detail/install UI, static and LLM moderation, and warnings for high-privilege hook behavior.

## Reproduction Assessment

Not applicable as a bug reproduction because this is an RFC/security feature request. Source inspection shows current main still omits the requested hooks contract from docs, schema, parser, scanner, public serialization, and skill UI.

## Solution Assessment

Unclear until maintainers settle the trust boundary; a docs-only implementation is not the best fix. The maintainable path is to decide the hook policy first, then add docs, parsing, scanning, API/UI disclosure, and install warnings together.
