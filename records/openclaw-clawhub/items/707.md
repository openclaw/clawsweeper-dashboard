## Summary

Keep open. This is a ClawHub issue, which this cleanup policy does not auto-close, and current main still lacks SKILL.md lifecycle hook documentation, normalized schema/parser support, hook-specific moderation, public API serialization, and skill-page/install transparency; the existing `hooks` metadata is for package/plugin capabilities, not SKILL.md frontmatter hooks.

## What This Changes

The issue asks ClawHub to document SKILL.md lifecycle hooks, parse and expose hook metadata, and add hook-specific security and UI transparency.

## Best Possible Solution

Define a maintainer-approved hook metadata and permission policy first, then implement it consistently across skill format docs, shared schema, parser/backfill path, public API, skill detail/install UI, static and LLM moderation context, and warnings for high-privilege events or hook types.

## Reproduction Assessment

Not applicable as a bug reproduction because this is an RFC/security feature request. Source inspection gives high confidence that current main does not document, normalize, moderate, serialize, or display SKILL.md lifecycle hooks.

## Solution Assessment

No for a docs-only implementation. The maintainable path is to settle the hook trust and permission contract first, then add docs, schema/parser support, moderation checks, API/UI surfacing, and install warnings together.

## Security Review

Status: needs_attention

Summary: The issue is security-sensitive because lifecycle hooks can run arbitrary code or agent behavior, while current ClawHub does not parse, disclose, or specifically moderate SKILL.md hook declarations.

Concerns:

- **[high] Define hook execution trust boundary:**
  - body: SKILL.md lifecycle hooks can attach command/http/prompt/agent behavior to high-privilege events. Before ClawHub documents or surfaces them as supported, maintainers need a clear policy for allowed event types, blocking hooks, agent hooks, consent, and moderation treatment.
  - confidence: 0.9
- **[medium] Avoid docs-only endorsement:** `docs/skill-format.md:73`
  - body: Adding only format documentation would tell authors to use hooks before ClawHub can validate command targets, expose hook metadata to users, or warn during install, which would widen the attack surface without matching transparency.
  - confidence: 0.86
