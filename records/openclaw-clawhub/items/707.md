## Summary

Keep open. Current main still lacks SKILL.md lifecycle hook documentation, first-class parsed metadata, hook-specific scanner checks, public API serialization, and install/UI disclosure; the package/plugin `hooks` surface is a separate capability metadata path, and the related open hook issues do not supersede this broader security RFC.

## Best Possible Solution

Define a maintainer-approved hook metadata and permission policy first, then implement docs, schema/parser/backfill, public API/UI disclosure, static and LLM moderation context, and install warnings or consent together.

Yes, as a source-level verification rather than a runtime bug: current main can be inspected to confirm hooks are absent from docs, schema/parser normalization, static checks, public serialization, and install metadata UI.
