Keep open. Current main and the latest release still lack a first-class SKILL.md hook contract across the docs, shared skill metadata schema/parser, security analysis, public API, and skill detail disclosure, so the RFC remains necessary.

## What This Changes

The issue requests documenting SKILL.md lifecycle hooks, adding hook-aware security analysis, and surfacing hook declarations in ClawHub’s registry UI/API.

## Best Possible Solution

Define a maintainer-approved hook metadata and permission policy first, then implement docs, schema/parser/backfill, public API/UI disclosure, static and LLM moderation context, and install warnings or consent together.

## Reproduction Assessment

Yes. Source inspection on current main verifies that lifecycle hooks are absent from the skill format docs, normalized metadata schema/parser, hook-specific scan policy, public skill API metadata, and skill detail disclosure surfaces.

## Solution Assessment

Yes in direction, but not as a docs-only fix. The maintainable path is a policy decision first, followed by coordinated metadata, moderation, API, UI, and consent support.
