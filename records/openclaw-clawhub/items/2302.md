## Summary

## What This Changes

This PR adds an admin autoban remediation HTTP endpoint and `clawhub-mod users remediate-autobans` command for dry-running or applying malware autoban unban/restoration decisions from current scanner verdicts.

## Best Possible Solution

Land this after `--apply` has an explicit confirmation/`--yes` gate, regression coverage and spec intent are updated, and the contributor adds redacted live CLI/API proof for dry-run and targeted apply.

## Reproduction Assessment

Yes. Source inspection shows `cmdRemediateAutobans` discards `inputAllowed` and posts the write request whenever `--apply` is set, unlike existing ban/unban commands that require confirmation or `--yes`.

## Solution Assessment

No. The backend approach may be workable, but the CLI write path needs the existing confirmation pattern and live proof before this is the safest maintainable solution.
