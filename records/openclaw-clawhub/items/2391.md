## Summary

## What This Changes

The branch changes package publish collision handling to allow same-owner skill-slug matches and adds regression tests for same-owner and different-owner package publish paths.

## Best Possible Solution

Limit the collision exception to existing package version publishes where the requested package owner and resolved skill or alias owner match, preserve first-publish and cross-owner collision blocks, and add live publish proof.

## Reproduction Assessment

Yes. Source inspection shows current main rejects a publish when the normalized package name resolves to a skill or alias before existing-package owner checks can run; I did not run a live publish because it depends on account/package state.

## Solution Assessment

No. The fix direction is right, but this implementation should narrow the exception to same-owner existing package updates instead of allowing same-owner first publishes over skills or aliases.
