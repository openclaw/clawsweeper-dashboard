## Summary

Keep open. Current main still installs skill files and ClawHub metadata without discovering or registering `hooks/openclaw`, and automatic OpenClaw hook registration would cross a security-sensitive gateway/runtime boundary that needs maintainer policy first.

## What This Changes

Request that `clawhub install` detect skill-bundled OpenClaw hook files and automatically register them with the OpenClaw gateway.

## Best Possible Solution

Define a maintainer-approved skill hook registration contract first, then implement explicit detection, disclosure, consent, security scanning, gateway registration/update behavior, docs, and tests together.

## Reproduction Assessment

Yes as a source-level verification, not a documented bug reproduction. Current main's `cmdInstall` and `cmdUpdate` paths have no hook directory discovery or OpenClaw plugin registration call.

## Solution Assessment

Unclear as proposed. Automatic registration would solve the symptom, but the maintainable path needs an explicit consent, scanning, disclosure, and gateway handoff policy first.
