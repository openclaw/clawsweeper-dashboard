## Summary

Keep open. Current main and v0.21.0 still install skill files and metadata only; automatic OpenClaw hook/plugin registration is not implemented and the remaining work needs product and security decisions before a fix PR is safe.

## What This Changes

The issue requests that `clawhub install` detect skill hook bundles under `hooks/openclaw` and automatically register them with OpenClaw.

## Best Possible Solution

Define a safe hook-install contract first, then implement explicit hook discovery, user consent, scanner/disclosure coverage, install/update tests, and docs for the selected ClawHub/OpenClaw handoff behavior.

## Reproduction Assessment

Yes, at source level: the current `clawhub install` path extracts skill files and records metadata but has no hook detection or gateway/plugin registration step. I did not run a live OpenClaw gateway reproduction.

## Solution Assessment

Unclear as requested. Automatic registration may solve the usability gap, but the maintainable solution needs an explicit consent, scanning, and disclosure policy before install/update wires executable hooks into OpenClaw.
