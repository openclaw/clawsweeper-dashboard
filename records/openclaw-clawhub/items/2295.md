## Summary

Keep open: current main still only installs skill files and ClawHub metadata, while automatic `hooks/openclaw` registration would cross an OpenClaw gateway/security boundary that needs maintainer policy first.

## What This Changes

Request to have `clawhub install` detect skill-bundled `hooks/openclaw` hook/plugin files and register them with the OpenClaw gateway automatically.

## Best Possible Solution

Define a maintainer-approved skill hook registration contract, then implement explicit detection, disclosure, consent, security scanning, gateway registration/update behavior, docs, and tests together.

## Reproduction Assessment

Yes, as a source-level verification rather than a runtime bug reproduction. Current main's `cmdInstall` and `cmdUpdate` paths have no hook directory discovery or OpenClaw plugin registration call.

## Solution Assessment

Unclear as proposed. Automatic registration would solve the symptom, but unconditional registration is not the best maintainable path until maintainers define consent, scanning, disclosure, and gateway handoff policy.
