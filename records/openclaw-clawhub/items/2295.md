## Label Justifications

- `P2`: The issue affects a meaningful setup workflow with silent hook non-activation, but the remedy requires bounded product and security design rather than urgent break/fix handling.
- `impact:security`: The requested automatic registration would make installed skill hook code part of the OpenClaw execution boundary and needs explicit trust and consent handling.

## Summary

Keep open. Current main and v0.22.0 still install or update skill files and metadata only; automatic OpenClaw hook discovery, disclosure, consent, and gateway registration remain absent and security/product-sensitive.

## What This Changes

The issue requests that `clawhub install` detect OpenClaw hook bundles in installed skills and register them with the OpenClaw gateway automatically.

## Best Possible Solution

Define a consent-aware ClawHub/OpenClaw hook install contract, including hook discovery, scanner disclosure, explicit registration approval, gateway handoff semantics, rollback behavior, and docs/tests for install and update flows.

## Reproduction Assessment

Yes, at source level. Current main's install path downloads/extracts the skill and writes origin plus lockfile metadata, with no hook discovery or gateway registration step.

## Solution Assessment

No, not exactly as proposed. Automatically running the equivalent of `openclaw plugins install` is too broad until maintainers define consent, disclosure, scanner, and gateway handoff behavior.
