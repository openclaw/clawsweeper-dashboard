## Label Justifications

- `impact:security`: The requested behavior would automatically register third-party executable code to run on OpenClaw lifecycle events.

## Summary

Current main still installs and records skill files without registering bundled OpenClaw hooks, so the reported activation gap remains. Keep this open because changing installation into automatic executable-plugin activation requires an explicit product and security decision, coordinated with the related hook-disclosure and post-install workflow issues.

## What This Changes

Have `clawhub install` automatically discover and register OpenClaw hook plugins bundled inside installed skills.

## Best Possible Solution

Define one explicit and reversible hook lifecycle that scans and displays requested events before installation, obtains operator consent, registers hooks idempotently, handles gateway restart, and removes or updates registrations with the skill.

## Reproduction Assessment

Yes at source level. Current main completes skill extraction and metadata recording without any plugin-registration call, which directly explains why bundled hooks are absent from `openclaw hooks list`.

## Solution Assessment

No, unconditional automatic registration is not yet the best solution. The maintainable implementation must add explicit consent, hook disclosure and scanning, plus complete install, update, restart, and uninstall semantics.
