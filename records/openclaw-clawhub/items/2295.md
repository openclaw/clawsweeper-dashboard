## Summary

Keep open. Current main and v0.21.0 still install or update skill files plus ClawHub metadata only; automatic OpenClaw hook registration is still missing and needs a product/security boundary decision before implementation.

## What This Changes

The issue asks for `clawhub install` to detect OpenClaw hook files in installed skills and register them with the OpenClaw gateway automatically.

## Best Possible Solution

Define an explicit hook activation contract, then implement discovery, disclosure, consent, trust checks, install/update registration, docs, and real OpenClaw gateway validation together.

## Reproduction Assessment

Yes, at source level. Current `cmdInstall` and `cmdUpdate` extract skill content and write metadata without hook detection, gateway handoff, or plugin registration; I did not run a live OpenClaw gateway reproduction.

## Solution Assessment

No, not exactly as requested. Automatic registration addresses the activation gap, but the maintainable solution needs explicit consent, scanner/disclosure coverage, and a ClawHub/OpenClaw handoff policy before executable hooks are wired in.
