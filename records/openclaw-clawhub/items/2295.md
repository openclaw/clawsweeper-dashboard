## Summary

Keep open. Current main still implements `clawhub install` as a skill file extraction and metadata workflow with no hook registration, and automatic hook registration would change code-execution and session behavior that needs maintainer policy first.

## What This Changes

The issue requests that `clawhub install` detect skill-bundled OpenClaw hooks and register them with the gateway automatically.

## Best Possible Solution

Define a maintainer-approved hook registration and consent contract first, then add explicit detection, disclosure, opt-in registration, docs, and tests in the appropriate OpenClaw or ClawHub install path.

## Reproduction Assessment

Yes, source-level: `cmdInstall` copies the skill and records origin/lock metadata without any hook detection or `openclaw plugins install` call. I did not run the install command because this review is read-only.

## Solution Assessment

Unclear as proposed. Auto-registration would address the symptom, but the maintainable path needs an explicit consent, scanning, disclosure, and gateway handoff policy before code changes.
