## Summary

Keep open. Current main still treats `clawhub install` as a file-and-metadata install path with no `hooks/openclaw` discovery or OpenClaw plugin registration, and the requested automatic registration crosses code-execution, consent, and session-state boundaries that need maintainer policy first.

## What This Changes

The issue requests automatic detection and registration of skill-bundled OpenClaw hooks during `clawhub install`.

## Best Possible Solution

Define a maintainer-approved hook registration contract first, then add explicit consent, hook detection, registration, disclosure, docs, and tests in the selected OpenClaw or ClawHub install path.

## Reproduction Assessment

Yes for the reported current behavior: source inspection shows `cmdInstall` downloads, extracts, fingerprints, and records metadata without detecting `hooks/openclaw` or invoking `openclaw plugins install`. I did not run the install command because this review is read-only.

## Solution Assessment

Unclear as proposed. Auto-registration would address the symptom, but the maintainable solution needs maintainer-approved consent, disclosure, scanning, and gateway handoff policy before implementation.
