## Summary

Keep open. Current main still installs skills as local files plus ClawHub metadata without detecting or registering `hooks/openclaw`, and automatic hook registration crosses an OpenClaw plugin code-execution boundary that needs maintainer policy first.

## What This Changes

Requests that `clawhub install` detect skill-bundled OpenClaw hooks and register them with the OpenClaw gateway automatically.

## Best Possible Solution

Define a maintainer-approved hook registration contract first, then implement explicit detection, disclosure, consent, registration, docs, and tests in the chosen ClawHub/OpenClaw install path.

## Reproduction Assessment

Yes, from source inspection: `cmdInstall` extracts the skill archive and writes origin/lock metadata without hook discovery or an OpenClaw plugin registration call. I did not run the command because this review is read-only.

## Solution Assessment

Unclear as proposed. Automatic registration would solve the symptom, but the maintainable path needs explicit maintainer policy for consent, scanning, disclosure, and OpenClaw gateway handoff.
