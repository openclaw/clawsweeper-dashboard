## Summary

Keep open. Current main and the latest tagged CLI release still install/update skill files and ClawHub metadata only; automatic OpenClaw hook discovery and registration remain missing and need a security/product decision before implementation.

## What This Changes

The issue requests that `clawhub install` detect skill-bundled OpenClaw hooks and register them with the OpenClaw gateway automatically.

## Best Possible Solution

Define a consent-aware hook capability contract, then implement discovery, disclosure, install/update registration, rollback behavior, docs, focused CLI tests, scanner coverage, and real OpenClaw gateway validation together.

## Reproduction Assessment

Yes, at source level. Current `cmdInstall` and `cmdUpdate` copy/update skill files and ClawHub metadata without hook detection, gateway handoff, or plugin registration; I did not run a live OpenClaw gateway reproduction.

## Solution Assessment

Unclear as requested. Automatic registration addresses the activation gap, but the maintainable fix needs explicit consent, scanner/disclosure coverage, and a ClawHub/OpenClaw handoff policy before executable hooks are wired in.
