## Summary

Keep open. Current main still installs and updates skill files without discovering `hooks/openclaw` or registering an OpenClaw plugin/hook, and the requested automatic registration needs maintainer product and security policy before implementation.

## What This Changes

The issue requests that `clawhub install` and update flows detect skill-bundled OpenClaw hooks and register them with the OpenClaw gateway/plugin system.

## Best Possible Solution

Define a maintainer-approved hook registration contract first, then implement explicit detection, disclosure, consent, security scanning, gateway registration/update behavior, docs, and focused tests together.

## Reproduction Assessment

Yes, as source-level verification rather than an executed runtime harness. Current main's `cmdInstall` and `cmdUpdate` paths have no hook directory discovery or OpenClaw plugin registration call.

## Solution Assessment

Unclear as proposed. Automatically running the equivalent of `openclaw plugins install` would address the symptom, but the maintainable path needs accepted consent, scanning, disclosure, and gateway handoff policy first.
