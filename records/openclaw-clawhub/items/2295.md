## Summary

Keep open. Current main and v0.21.0 still install/update skill files and ClawHub metadata only; automatic hook registration remains a real product and security boundary decision.

## What This Changes

The issue asks for `clawhub install` to detect `hooks/openclaw` content in installed skills and register it with the OpenClaw gateway automatically.

## Best Possible Solution

Define an explicit hook capability contract, then implement discovery, disclosure, consent, trust checks, install/update registration, docs, and real OpenClaw gateway validation together.

## Reproduction Assessment

Yes, at source level. Current `cmdInstall` and `cmdUpdate` extract skill content and write metadata without hook detection, gateway handoff, or plugin registration; I did not run a live OpenClaw gateway reproduction.

## Solution Assessment

Unclear as requested. Automatic registration addresses the activation gap, but the maintainable solution needs explicit consent, scanner/disclosure coverage, and a ClawHub/OpenClaw handoff policy before executable hooks are wired in.
