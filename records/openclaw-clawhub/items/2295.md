## Summary

Keep open. Current main and the latest tagged CLI release still install skill files and local metadata only; automatic OpenClaw hook/plugin registration remains missing and needs maintainer security and product decisions before implementation.

## What This Changes

The issue requests that `clawhub install` detect skill-bundled OpenClaw hooks and register them with the gateway automatically.

## Best Possible Solution

Define a safe hook-install contract first, then implement explicit hook discovery, user consent, scanner and disclosure coverage, install/update tests, and docs for the selected ClawHub/OpenClaw handoff.

## Reproduction Assessment

Yes, at source level: current `cmdInstall` extracts files and writes install metadata but has no `hooks/openclaw` discovery or OpenClaw plugin registration step. I did not run a live OpenClaw gateway reproduction.

## Solution Assessment

Unclear as requested. Automatic registration addresses the symptom, but the maintainable fix needs an explicit consent, scanning, disclosure, and install/update policy before wiring executable hooks into OpenClaw.
