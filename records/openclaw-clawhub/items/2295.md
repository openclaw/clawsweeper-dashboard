## Summary

Current main and the latest v0.20.2 release still install skill files without registering embedded OpenClaw hook plugins, so the reported incomplete setup remains real. The issue should stay open, but the requested unconditional auto-registration needs maintainer product and security judgment because current policy keeps skill installation separate from executable plugin activation and requires risky code paths to remain explicit and operator-controlled.

## What This Changes

Detect hook-bearing skill bundles during `clawhub install` and activate or clearly guide registration of their embedded OpenClaw hook plugin.

## Best Possible Solution

Keep skill file installation distinct from executable activation, but detect declared hook/plugin content and hand it to an OpenClaw-owned consent flow that shows the capability, performs compatibility and security checks, reports activation/restart status, and never labels partial setup as fully usable.

## Reproduction Assessment

Yes at source level: current main and v0.20.2 both extract and record skill files without any embedded-hook discovery or plugin-registration path. The supplied CLI steps are consistent with that implementation, although this read-only review did not run an installed OpenClaw gateway.

## Solution Assessment

No, unconditional automatic registration is not the best solution because it collapses the documented skill/plugin trust boundary. Explicit detection plus an operator-approved OpenClaw activation flow is safer and avoids silent partial installation.
