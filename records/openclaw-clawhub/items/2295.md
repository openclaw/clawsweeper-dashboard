## Summary

Keep open. Current main and v0.20.0 still treat `clawhub install` as a skill file install only, and the safe fix needs maintainer security/UX decisions because automatic registration would activate executable gateway hooks.

## What This Changes

The issue requests that `clawhub install` detect skill-bundled OpenClaw hooks and register them with the gateway instead of only copying the skill files.

## Best Possible Solution

Define a policy-backed hook install flow with capability disclosure, security scanning, and explicit user/operator consent, then wire the accepted path through the OpenClaw/ClawHub install/update surfaces instead of silently registering executable hooks.

## Reproduction Assessment

Yes. Source inspection is high confidence: current `cmdInstall` resolves, downloads or extracts, and writes origin/lock metadata without discovering `hooks/openclaw` or invoking OpenClaw plugin registration; I did not run the CLI because this was a read-only review.
