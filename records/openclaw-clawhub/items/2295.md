## Summary

Keep open. Current main still implements `clawhub install` as skill archive extraction plus origin/lockfile metadata, with no hook discovery or gateway registration, and the requested automatic registration crosses a plugin/hook execution boundary that needs maintainer policy first.

## What This Changes

The issue requests that `clawhub install` detect skill-bundled OpenClaw hooks and register them automatically with the gateway.

## Best Possible Solution

Define a maintainer-approved hook registration contract first, then implement explicit user-consented detection, disclosure, registration, docs, and tests in the selected OpenClaw or ClawHub install path.

## Reproduction Assessment

Yes, from source inspection: `cmdInstall` downloads and extracts the skill, writes origin/lock metadata, and exits without hook detection or `openclaw plugins install` registration. I did not run the install because this review is read-only.

## Solution Assessment

Unclear as proposed. Automatic registration would address the symptom, but the maintainable path needs a maintainer-approved consent, scanning, disclosure, and OpenClaw gateway handoff policy first.
