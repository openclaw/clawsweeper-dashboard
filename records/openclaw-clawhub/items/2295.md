## Summary

Keep open. Current main and the v0.23.0 CLI still install skill files and local metadata only; automatic hook discovery, security disclosure, consent, and OpenClaw gateway registration remain missing and product/security-sensitive.

## What This Changes

The issue requests that `clawhub install` detect skill-provided OpenClaw hook files and automatically register them with the OpenClaw gateway after installation.

## Best Possible Solution

Define a consent-aware hook registration contract, then implement detection, hook permission disclosure, hook-specific scanner checks, an approved OpenClaw registration handoff, docs, and focused install/update tests.

## Reproduction Assessment

Yes, at source level. Current `cmdInstall` installs files and metadata and the schema/parser/scanner surfaces do not model or register skill hook bundles; I did not run a live OpenClaw gateway reproduction.

## Solution Assessment

No, not as unconditional automatic registration. The maintainable path is a maintainer-approved, consent-aware hook registration and scanner/disclosure contract rather than blindly running `openclaw plugins install` after every skill install.
