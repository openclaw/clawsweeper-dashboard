## Summary

Keep open. Current main still installs and updates skill files plus ClawHub metadata without discovering or registering `hooks/openclaw`, and automatic activation of skill-shipped hook/plugin code needs maintainer product and security policy first.

## What This Changes

Request that `clawhub install` detect skill-bundled OpenClaw hooks and register them with the gateway so they fire after installation.

## Best Possible Solution

Define a maintainer-approved hook registration contract first, then implement explicit detection, disclosure, consent, security scanning, gateway registration/update behavior, docs, and tests together.

## Reproduction Assessment

Yes, as source-level verification rather than a runtime harness. Current main's `cmdInstall` and `cmdUpdate` paths have no hook directory discovery or OpenClaw plugin registration call.

## Solution Assessment

Unclear as proposed. Running the equivalent of `openclaw plugins install` would address the symptom, but the maintainable path needs accepted consent, scanning, disclosure, and gateway handoff policy first.
