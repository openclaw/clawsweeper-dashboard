## Summary

Keep open. Current main and v0.21.0 still install skill files and metadata only; automatic hook discovery, OpenClaw plugin registration, and gateway restart/handoff are absent, and the related open hook-security and session-handoff work does not supersede this request.

## What This Changes

The issue requests that `clawhub install` detect skill-shipped `hooks/openclaw` content and register it with OpenClaw automatically.

## Best Possible Solution

Define a safe hook-install contract first, then implement explicit hook discovery, consent, scanner/disclosure coverage, install/update tests, and docs for the chosen ClawHub/OpenClaw handoff.

## Reproduction Assessment

Yes, at source level. Current `cmdInstall` and `cmdUpdate` extract skill content and write metadata without hook detection, gateway handoff, or plugin-registration code; I did not run a live OpenClaw gateway reproduction.

## Solution Assessment

Unclear as requested. Automatic registration addresses the activation gap, but the maintainable path needs explicit consent, scanner/disclosure coverage, and a ClawHub/OpenClaw handoff policy before executable hooks are wired in.
