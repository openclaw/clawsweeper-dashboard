## Summary

Keep open. Current main and latest v0.23.0 still install skill files and metadata without detecting `hooks/openclaw` or registering OpenClaw hooks, and the remaining work needs a consent/security/product contract before implementation.

## What This Changes

The issue asks `clawhub install` to detect OpenClaw hook bundles inside installed skills and register them with the OpenClaw gateway automatically.

## Best Possible Solution

Define a consent-aware hook registration contract, then implement detection, hook-specific scanner/disclosure checks, approved OpenClaw registration handoff, docs, and focused install/update tests.

## Reproduction Assessment

Yes, at source level. Current `cmdInstall` and `cmdUpdate` install or update files and metadata without hook discovery, gateway registration, or hook-specific consent handling; I did not run a live OpenClaw gateway reproduction.

## Solution Assessment

No, not as unconditional automatic registration. The maintainable path is a maintainer-approved consent, scanner/disclosure, and OpenClaw registration contract rather than blindly running `openclaw plugins install` after every skill install.
