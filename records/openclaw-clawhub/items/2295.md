## Summary

Current `main` still installs skill files without discovering or registering bundled OpenClaw hooks. The reported gap is real, but automatic activation would cross an executable-code trust boundary that requires an explicit product and security decision; the related open issues cover adjacent hook security and post-install behavior without superseding this request.

## What This Changes

Add a secure, user-visible activation path for OpenClaw hook plugins bundled inside installed skills.

## Best Possible Solution

Define one declared hook-package contract with scanner coverage, compatibility metadata, clear permission disclosure, explicit user approval, and complete install/update/uninstall lifecycle behavior; ClawHub can then call a supported OpenClaw activation API or provide the exact safe next step.

## Reproduction Assessment

Yes at source level: the current install function has no hook discovery or registration step, and the reported `openclaw hooks list` command provides a concrete runtime confirmation path. This read-only review did not execute OpenClaw itself.

## Solution Assessment

No, silently running `openclaw plugins install` is not the best solution on its own. Activation should use the repository’s executable-package trust model with scanning, compatibility checks, disclosure, explicit consent, and lifecycle handling.
