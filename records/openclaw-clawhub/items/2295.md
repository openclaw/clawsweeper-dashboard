## Summary

Keep open. Current main still installs skills by extracting files and writing ClawHub metadata only; automatic OpenClaw hook registration is absent, and the safe fix needs maintainer product/security decisions tied to the open hook contract work.

## What This Changes

The issue requests that `clawhub install` detect skill-bundled OpenClaw hooks and register them with the OpenClaw gateway automatically or otherwise prevent silent inactive hooks.

## Best Possible Solution

Define a maintainer-approved hook metadata and consent contract first, then make install either explicitly offer safe hook registration or clearly warn/delegate users to the supported native OpenClaw install path.

## Reproduction Assessment

Yes, at source level. Current main’s `cmdInstall` has no hook discovery or gateway/plugin registration step after extraction, and the docs/schema do not define a hooks contract; I did not run a live install because this read-only review was enough to establish the missing path.

## Solution Assessment

No, not exactly as proposed. A silent automatic `openclaw plugins install` call is not the safest fix; maintainers should first define the hook contract, consent UX, security checks, and native OpenClaw handoff behavior.
