## Summary

Keep open. Current main still treats `clawhub install` as skill archive extraction plus ClawHub metadata writes, while automatic OpenClaw hook registration would add a security-sensitive runtime handoff that needs maintainer policy first.

## What This Changes

The issue asks `clawhub install` to detect skill-bundled OpenClaw hooks and register them with the OpenClaw gateway automatically.

## Best Possible Solution

Define a maintainer-approved hook registration contract first, then implement explicit detection, disclosure, consent, registration, docs, and tests in the chosen ClawHub/OpenClaw install path.

Yes, at source level: `cmdInstall` extracts the archive and writes origin/lock metadata without hook discovery or an OpenClaw plugin registration call. I did not run the command because this review is read-only.

Unclear as proposed. Automatic registration would solve the symptom, but the maintainable path needs explicit maintainer policy for consent, scanning, disclosure, and OpenClaw gateway handoff.
