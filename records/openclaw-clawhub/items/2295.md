## Summary

Keep open. Current main still treats `clawhub install` as a skill file extraction plus origin/lockfile workflow with no OpenClaw hook discovery or gateway registration, and automatically registering skill-bundled hooks needs maintainer policy because it crosses a code-execution and active-session trust boundary.

## What This Changes

Request that `clawhub install` detect skill-bundled OpenClaw hooks and register them with the gateway automatically.

## Best Possible Solution

Define a maintainer-approved hook registration contract first, then implement explicit consent, hook detection, warnings, registration, docs, and tests, or document the manual plugin install step as the supported boundary.

## Reproduction Assessment

Yes from source inspection: `cmdInstall` extracts the archive and records origin/lock metadata without hook detection or an `openclaw plugins install` call. I did not run the command because this review is read-only.

## Solution Assessment

Unclear as proposed. Auto-registration would solve the symptom, but the safer maintainable path is a maintainer-approved consent, scan, disclosure, and gateway handoff policy before implementation.
