## Summary

Keep open. Current main and v0.21.0 still copy skill files and record ClawHub install metadata without discovering or registering `hooks/openclaw`, and the remaining work needs a maintainer security/product decision before implementation.

## What This Changes

The issue requests that `clawhub install` detect OpenClaw hook content inside installed skills and automatically register it with the OpenClaw gateway/plugin system.

## Best Possible Solution

Define an explicit hook capability and consent contract first, then implement discovery, disclosure, install/update registration, docs, tests, and real OpenClaw gateway validation together.

## Reproduction Assessment

Yes, at source level. Current `cmdInstall` and `cmdUpdate` extract skill content and write metadata without hook detection, gateway handoff, or plugin registration; I did not run a live OpenClaw gateway reproduction.

## Solution Assessment

Unclear as requested. Automatic registration addresses the activation gap, but the maintainable solution needs explicit consent, scanner/disclosure coverage, and a ClawHub/OpenClaw handoff policy before executable hooks are wired in.
