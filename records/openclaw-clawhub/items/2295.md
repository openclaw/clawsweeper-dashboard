## Summary

Keep open. Current main still treats `clawhub install` as file extraction plus ClawHub metadata writes, while automatic OpenClaw hook registration would add a new security-sensitive runtime handoff that needs maintainer policy first.

## What This Changes

The issue requests that `clawhub install` detect skill-bundled OpenClaw hooks and register them automatically with the gateway.

## Best Possible Solution

Define a maintainer-approved hook registration contract first, then implement explicit detection, disclosure, consent, registration, docs, and tests in the chosen ClawHub/OpenClaw install path.
