## Summary

Keep open. Current main and latest v0.23.0 still install skill files and local metadata only; hook discovery, consent/security disclosure, and OpenClaw gateway registration remain unresolved product and security work.

## What This Changes

The issue requests automatic detection and registration of OpenClaw hooks when installing a ClawHub skill.

## Best Possible Solution

Define a consent-aware hook registration contract, then implement detection, hook permission disclosure, hook-specific scanner checks, approved OpenClaw registration handoff, docs, and focused install/update tests.

## Reproduction Assessment

Yes, at source level. Current cmdInstall installs skill files and metadata without hook discovery or gateway registration; I did not run a live OpenClaw gateway reproduction.

## Solution Assessment

No, not as unconditional automatic registration. The maintainable path is a maintainer-approved consent, scanner/disclosure, and OpenClaw registration contract rather than blindly running openclaw plugins install after every skill install.
