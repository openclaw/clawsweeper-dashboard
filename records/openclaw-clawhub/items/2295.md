## Summary

Keep open. Current main still installs skills as files plus ClawHub metadata without detecting or registering `hooks/openclaw`, and the requested automatic hook registration crosses a plugin/code-execution trust boundary that needs maintainer policy first.

## What This Changes

The issue requests automatic registration of skill-bundled OpenClaw hooks during `clawhub install` so hook-dependent skills work without a separate plugin install step.

## Best Possible Solution

Define a maintainer-approved hook registration contract first, then add explicit consent, disclosure, detection, registration, docs, and tests in the chosen OpenClaw or ClawHub install path.

## Reproduction Assessment

Yes, from source inspection: `cmdInstall` extracts the archive and writes origin/lock metadata without hook discovery or an `openclaw plugins install` call. I did not run the command because this review is read-only.

## Solution Assessment

Unclear as proposed. Auto-registration would address the symptom, but the maintainable solution needs explicit maintainer policy for consent, scanning, disclosure, and the OpenClaw gateway handoff before implementation.
