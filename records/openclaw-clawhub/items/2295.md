## Summary

Keep open. Current main and v0.21.0 still install or update skill files and ClawHub metadata only; automatic OpenClaw hook registration remains missing and needs a security/product decision before implementation.

## What This Changes

The issue asks for `clawhub install` to detect skill-provided OpenClaw hooks and register them automatically with the OpenClaw gateway.

## Best Possible Solution

Define an explicit hook capability contract, then implement discovery, disclosure, consent, trust checks, install/update registration, docs, and real OpenClaw gateway validation together.

## Reproduction Assessment

Yes, at source level: current main's install and update flows extract skill content and write metadata without hook discovery, gateway handoff, or plugin registration. I did not run a live OpenClaw gateway reproduction.

## Solution Assessment

Unclear as requested. Automatic registration addresses the activation gap, but the maintainable solution needs explicit consent, scanner/disclosure coverage, and a ClawHub/OpenClaw handoff policy before executable hooks are wired in.
