## Label Justifications

- `impact:security`: Automatic registration of skill-bundled hooks affects code execution, gateway registration, user consent, and plugin trust boundaries.

## Summary

Keep open. Current main still treats `clawhub install` as a file install plus metadata/lockfile update with no OpenClaw hook registration, and the requested automatic registration crosses a code-execution and session-state trust boundary that needs maintainer policy first.

## What This Changes

The issue asks `clawhub install` to detect skill-bundled OpenClaw hooks and register them with the OpenClaw gateway automatically.

## Best Possible Solution

Define a maintainer-approved hook registration contract first, then implement explicit user consent, detection, warnings, registration handoff, docs, and tests aligned with https://github.com/openclaw/clawhub/issues/707 and https://github.com/openclaw/clawhub/issues/792.

## Solution Assessment

Unclear as proposed. Automatic registration would address the symptom, but the maintainable solution needs explicit maintainer policy for consent, disclosure, scanning, and OpenClaw gateway handoff before implementation.
