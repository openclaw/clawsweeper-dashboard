## Label Justifications

- `impact:security`: Registering bundled hook or plugin code activates downloaded executable content across a trust boundary.

## Summary

Current main still installs skill files without detecting or registering bundled OpenClaw hooks, so the silent incomplete-install problem remains. The issue should stay open, but unconditional automatic activation needs an explicit security and product decision because it would execute downloaded skill code across the current inspectable-setup boundary.

## What This Changes

Add a safe install-time path for detecting and activating bundled OpenClaw hook or plugin content instead of leaving hook-dependent skills silently incomplete.

## Best Possible Solution

Define a permission-aware capability contract that detects bundled OpenClaw extensions, scans and displays their executable surface before installation, obtains explicit approval, and consistently handles install, update, rollback, and uninstall registration.

## Reproduction Assessment

Yes at source level: current main's install command ends after extracting files and recording install metadata, with no hook detection or registration step. This read-only review did not execute a live OpenClaw gateway reproduction.

## Solution Assessment

No, unconditional automatic registration is not the best solution because it would hide executable-code activation. Detection, disclosure, scanning, explicit consent, and lifecycle cleanup should be part of the supported design.
