## Summary

Keep open. Current main still installs and updates skill files without discovering `hooks/openclaw`, but turning skill install into automatic gateway/plugin registration crosses an executable hook trust boundary that needs maintainer product and security policy first.

## What This Changes

The issue requests that `clawhub install` detect skill-provided OpenClaw hooks and register them with the gateway automatically.

## Best Possible Solution

Define a maintainer-approved hook registration contract first, then implement explicit detection, disclosure, consent, security scanning, gateway registration/update behavior, docs, and tests together.

## Reproduction Assessment

Yes, as source-level verification rather than a runtime harness. Current main's `cmdInstall` and `cmdUpdate` paths have no hook directory discovery or OpenClaw plugin registration call.

## Solution Assessment

Unclear as proposed. Automatically running the equivalent of `openclaw plugins install` would address the symptom, but the maintainable path needs accepted consent, scanning, disclosure, and gateway handoff policy first.
