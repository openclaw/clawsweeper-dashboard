## Label Justifications

- `P2`: Hook-dependent skills can misleadingly appear installed while remaining inactive, but resolving this requires a bounded security and product decision rather than emergency response.
- `impact:security`: Registering bundled hooks enables third-party executable code on privileged OpenClaw lifecycle events.
- `impact:session-state`: The reported bootstrap hook can alter agent initialization and subsequent session context.

## Summary

Current main still completes skill installation without registering bundled OpenClaw hooks, so the reported gap remains. It is distinct from the related hook-format and session-handoff issues, but implementation requires a maintainer decision because installation would begin enabling persistent executable lifecycle behavior.

## What This Changes

Request a ClawHub install flow that detects bundled OpenClaw hooks and registers them with the gateway.

## Best Possible Solution

Define hook metadata and scanner coverage, then make installation disclose requested events and executable targets and offer explicit opt-in registration with deterministic restart, update, uninstall, and rollback behavior.

## Reproduction Assessment

Yes at source level: current main and v0.20.2 install skill files and metadata without any plugin-registration call. A live gateway run was not necessary to establish the missing code path.

## Solution Assessment

No—the proposed silent automatic installation is not the safest maintainable solution; registration should require hook-specific disclosure, explicit consent, security analysis, and lifecycle cleanup semantics.
