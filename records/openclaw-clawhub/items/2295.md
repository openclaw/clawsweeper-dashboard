## Summary

Keep open. Current main and v0.21.0 still install skill files and metadata only; automatic `hooks/openclaw` discovery and OpenClaw gateway/plugin registration are absent, and the remaining work needs maintainer product and security decisions around consent, scanning, disclosure, and the ClawHub/OpenClaw handoff.

## What This Changes

Request `clawhub install` to detect skill-provided OpenClaw hook bundles and register them with the OpenClaw gateway automatically.

## Best Possible Solution

Define a safe hook-install contract first, then implement explicit hook discovery, consent, scanner/disclosure coverage, install/update tests, and docs for the chosen ClawHub/OpenClaw handoff.

## Reproduction Assessment

Yes, at source level. Current `cmdInstall` and `cmdUpdate` extract skill content and write metadata without any hook detection, gateway handoff, or plugin-registration step; I did not run a live OpenClaw gateway reproduction.

## Solution Assessment

Unclear as requested. Automatic registration addresses the activation gap, but the maintainable path needs explicit consent, scanner/disclosure coverage, and a ClawHub/OpenClaw handoff policy before executable hooks are wired in.
