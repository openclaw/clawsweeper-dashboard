## Summary

Keep open. Current main and v0.21.0 still install skill files and ClawHub metadata without registering `hooks/openclaw` with OpenClaw, and the remaining work crosses a real hook execution consent/security boundary.

## What This Changes

The issue requests that `clawhub install` detect hook directories in installed skills and automatically register them with the OpenClaw gateway.

## Best Possible Solution

Define a maintainer-approved hook capability contract, then implement explicit discovery, disclosure, consent, registration, update handling, docs, and real OpenClaw gateway validation together.

## Reproduction Assessment

Yes, at source level. Current `cmdInstall` and `cmdUpdate` copy or replace skill files and write metadata without hook discovery, gateway handoff, or plugin registration; I did not run a live OpenClaw gateway reproduction.

## Solution Assessment

Unclear as proposed. Automatic registration addresses the activation gap, but the maintainable solution needs explicit consent, scanner/disclosure coverage, and a ClawHub/OpenClaw handoff policy before executable hooks are wired in.
