## Summary

Keep open. Current main and v0.23.0 still install skill files and ClawHub metadata only; hook discovery, consent/security disclosure, and OpenClaw gateway registration remain unresolved product and security work.

## What This Changes

The issue requests that `clawhub install` detect OpenClaw hook bundles inside installed skills and automatically register them with the OpenClaw gateway.

## Best Possible Solution

Define a consent-aware hook registration contract, then implement hook detection, hook-specific scanner/disclosure checks, approved OpenClaw registration handoff, docs, and install/update tests.

## Reproduction Assessment

Yes, at source level. Current `cmdInstall` and `cmdUpdate` install files and metadata without hook discovery or gateway registration; I did not run a live OpenClaw gateway reproduction.

## Solution Assessment

No, not as unconditional automatic registration. The maintainable path is a maintainer-approved consent, scanner/disclosure, and OpenClaw registration contract rather than blindly running `openclaw plugins install` after every skill install.
