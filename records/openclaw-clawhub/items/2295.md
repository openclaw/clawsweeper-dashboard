## Summary

Keep open. Current main and the latest CLI release still install or update skill files and metadata without discovering `hooks/openclaw`, registering an OpenClaw hook bundle, or enforcing hook-specific consent/security disclosure, so this remains a real but product/security-sensitive gap.

## What This Changes

The issue requests that `clawhub install` detect skill-provided OpenClaw hooks and register them with the OpenClaw gateway after installation.

## Best Possible Solution

Define a maintainer-approved, consent-aware hook registration contract, then implement detection, hook-specific scanner/disclosure checks, OpenClaw gateway handoff, docs, and focused install/update tests.

## Reproduction Assessment

Yes, at source level. Current install/update code paths install or refresh skill files and metadata without hook discovery, gateway registration, or hook-specific consent handling; I did not run a live OpenClaw gateway reproduction.

## Solution Assessment

No, not as unconditional automatic registration. The maintainable path is a maintainer-approved consent, scanner/disclosure, and OpenClaw registration contract rather than blindly running `openclaw plugins install` after every skill install.
