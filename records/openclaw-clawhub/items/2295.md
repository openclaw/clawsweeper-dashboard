## Summary

Keep open. Current main and v0.21.0 still copy or update skill files and ClawHub metadata only; automatic OpenClaw hook discovery and gateway registration remain absent and need a maintainer security/product decision before implementation.

## What This Changes

Request that `clawhub install` detect skill-shipped OpenClaw hooks and register them with the gateway during installation.

## Best Possible Solution

Define an explicit hook capability and consent contract, then implement discovery, disclosure, install/update registration, docs, tests, and real OpenClaw gateway validation together.

## Reproduction Assessment

Yes, at source level. Current `cmdInstall` and `cmdUpdate` extract skill content and write metadata without hook detection, gateway handoff, or plugin registration; I did not run a live OpenClaw gateway reproduction.

## Solution Assessment

Unclear as requested. Automatic registration addresses the activation gap, but the maintainable solution needs explicit consent, scanner/disclosure coverage, and a ClawHub/OpenClaw handoff policy before executable hooks are wired in.
