## Label Justifications

- `P2`: This is a meaningful installer/setup reliability improvement with security implications, but no emergency production outage is shown.
- `impact:security`: The requested automatic registration would activate hook/plugin code shipped inside installed skill content.
- `impact:session-state`: Missing registration leaves hook-dependent bootstrap or session behavior inactive after the skill appears installed.

## Summary

Keep open. Current main still installs skill files and local metadata only; it does not discover `hooks/openclaw` or register those hooks with OpenClaw, and a safe fix needs maintainer decisions around consent, scanner coverage, disclosure, and the ClawHub/OpenClaw handoff.

## What This Changes

Request automatic registration of `hooks/openclaw` bundles when `clawhub install` installs a hook-bearing skill.

## Best Possible Solution

Define a safe hook-install contract first, then implement explicit hook discovery, consent, scanner/disclosure coverage, install/update registration, tests, and docs for the chosen ClawHub/OpenClaw handoff.

## Reproduction Assessment

Yes, at source level. Current `cmdInstall` and `cmdUpdate` extract skill content and write metadata without hook detection, gateway handoff, or plugin registration; I did not run a live OpenClaw gateway reproduction.

## Solution Assessment

Unclear as requested. Automatic registration addresses the activation gap, but the maintainable path needs explicit consent, scanner/disclosure coverage, and a ClawHub/OpenClaw handoff policy before executable hooks are wired in.
