## Summary

Keep open. Current main still shows `clawhub install` as a file-copy plus metadata/lockfile workflow with no hook detection or gateway registration, but automatically registering skill-bundled OpenClaw hooks crosses a plugin/code-execution trust boundary that needs maintainer policy first.

## What This Changes

Request that `clawhub install` detect skill-bundled OpenClaw hook files and automatically register them with the gateway.

## Best Possible Solution

Define a maintainer-approved hook registration contract first, then implement explicit user-consented detection, warning, registration, docs, and tests in the native OpenClaw install path or a clearly opt-in ClawHub CLI path.
