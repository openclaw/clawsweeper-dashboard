## Summary

Keep open. Current main and v0.21.0 still install skill files and metadata only; automatic OpenClaw hook/plugin registration is absent, and the remaining work needs maintainer product and security decisions around consent, scanning, disclosure, and ClawHub/OpenClaw handoff.

## What This Changes

The issue requests automatic detection and registration of `hooks/openclaw` content when `clawhub install` installs a skill.

## Best Possible Solution

Define a safe hook-install contract first, then implement explicit hook discovery, user consent, scanner/disclosure coverage, install/update tests, and docs for the chosen ClawHub/OpenClaw handoff.

## Reproduction Assessment

Yes. Source inspection shows the current install path extracts skill content and records metadata without any `hooks/openclaw`, `HOOK.md`, handler, gateway, or plugin-registration step; I did not run a live OpenClaw gateway reproduction.

## Solution Assessment

Unclear as requested. Automatic registration may solve the usability gap, but the maintainable path needs explicit consent, scanning, disclosure, and ClawHub/OpenClaw handoff policy before executable hooks are wired into OpenClaw.
