## Summary

Keep open. Current main still treats `clawhub install` as a file-and-metadata install path, while automatic hook registration would cross OpenClaw plugin execution, consent, disclosure, and active-session policy boundaries that need maintainer judgment.

## What This Changes

The request is for `clawhub install` to detect bundled `hooks/openclaw` content and automatically register it with the OpenClaw gateway.

## Best Possible Solution

Define a maintainer-approved hook registration contract first, then add explicit detection, consent, disclosure, registration, docs, and tests in the chosen OpenClaw or ClawHub install path.

## Reproduction Assessment

Yes as source verification: current `cmdInstall` installs files and metadata without hook discovery, gateway registration, or a plugin-install call. I did not run the command because this review is read-only, and the missing behavior is not currently a documented CLI guarantee.

## Solution Assessment

Unclear as proposed. Automatically running `openclaw plugins install` would address the symptom, but the maintainable solution needs explicit maintainer policy for consent, scanning, disclosure, and OpenClaw gateway handoff.
