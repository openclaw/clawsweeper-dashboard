## Summary

Keep open; the latest PR head still has a security-sensitive owner-restore predicate gap and a normal owner self-restore regression, and maintainer-authored PRs are not cleanup-close candidates.

## What This Changes

The branch tightens `setSkillSoftDeletedInternal` owner undelete authorization, surfaces owner-denial errors as HTTP 403, adds audit `actorRole` metadata, adds undelete gate tests, and makes a slug-validator type internal.

## Best Possible Solution

Gate owner restores on authoritative self-delete provenance while separately honoring protected current moderation verdicts, flags, merge/security states, and benign clean scanner/manual-review metadata.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: owner self-delete can leave `hiddenBy` as the owner, later scanner escalation can mark the row malicious without changing that provenance, and the PR predicate still permits owner restore.

## Solution Assessment

No. The PR fixes moderator hides without a reason, but `moderationReason === undefined` is too permissive for stale scanner-blocked owner rows and too strict for benign clean metadata preserved across owner self-delete.
