## Summary

Keep open: this member-authored security PR is not eligible for cleanup closure, and the latest head still introduces a legitimate owner self-restore regression while missing the required changelog entry.

## What This Changes

The PR tightens `setSkillSoftDeletedInternal` owner undelete authorization, resets VT malicious hide provenance, surfaces forbidden undelete errors through the v1 HTTP API, adds audit `actorRole` metadata, and adds undelete gate tests.

## Best Possible Solution

Keep the stricter moderation/security gate, but allow owner self-restores when the owner created the hide and current verdict/flags are benign, then add focused regression coverage and a changelog entry.

## Reproduction Assessment

Yes. Source inspection shows current main's owner path only checks ownership, and the PR regression path is an owner self-delete on an active skill carrying benign `pending.scan`, `scanner.*.clean`, or `manual.override.clean` metadata that the delete path preserves.

## Solution Assessment

No. The latest PR closes the malware-provenance bypass, but `moderationReason === undefined` is too broad as a self-restore requirement because benign active metadata is not moderation hide provenance.
