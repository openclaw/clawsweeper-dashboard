## What This Changes

The PR tightens skill owner undelete authorization, adds `actorRole` audit metadata, adds undelete gate tests, and removes an unused slug-validator type export.

## Best Possible Solution

Gate owner restores on authoritative self-delete provenance while also honoring protected current moderation verdicts/flags/reasons, then cover the edge cases and record the user-visible fix in the changelog.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the PR predicate can misclassify owner-hidden rows after later scanner escalation, and it blocks owner self-restores for active skills that retained benign scanner-clean moderation metadata.

## Solution Assessment

No. The updated `hiddenBy` gate fixes the earlier staff-hide-without-reason gap, but `moderationReason === undefined` is both too permissive for stale scanner-blocked owner rows and too strict for benign scanner-clean self-deletes.
