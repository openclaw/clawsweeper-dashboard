## Summary

Keep open. Current main does not contain this change, and the proposed patch still misses a common staff-hide path because it gates owner restore only on `moderationReason`, while existing moderator hide flows can set `softDeletedAt`, `moderationStatus`, `moderationNotes`, and `hiddenBy` without any `moderationReason`.

## What This Changes

The PR tightens owner undelete authorization in `setSkillSoftDeletedInternal`, adds `actorRole` to soft-delete audit metadata, and adds Vitest coverage for moderation-reason-based undelete gates.

## Best Possible Solution

Revise the authorization rule so owners can restore only states they created themselves, and cover all moderator/scanner/report/security/ban/merge hide paths, including staff hides that currently carry `hiddenBy` or moderation notes without `moderationReason`.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a moderator can hide through `api.skills.setSoftDeleted`, which leaves `moderationReason` undefined, and the PR’s owner undelete gate would still allow the owner to restore that skill.

## Solution Assessment

No. Checking only `moderationReason` is too narrow; the fix should either stamp protected provenance on every non-owner hide path or gate owner restore on the actual current hidden provenance such as staff/scanner/report/security state.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Block staff-hidden skills without moderationReason:** `convex/skills.ts:8144-8145`
  - body: This still lets owners restore a skill hidden through the existing staff path. `api.skills.setSoftDeleted` requires a moderator and records `softDeletedAt`, `moderationStatus: "hidden"`, `moderationNotes`, and `hiddenBy`, but it does not set `moderationReason`; since the new gate only rejects when `skill.moderationReason` is defined, an owner can still undelete that moderator-hidden state. Please gate on authoritative hidden provenance or stamp a protected reason on staff hides, and add a regression test for this path.
