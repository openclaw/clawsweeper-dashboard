## Summary

Keep open: current main still has the broad owner restore path, and this PR does not fully close the trust gap because staff-hidden skills can exist without `moderationReason`.

## What This Changes

The PR narrows owner undelete authorization in `setSkillSoftDeletedInternal`, adds `actorRole` audit metadata, adds undelete gate tests, and removes an unused exported slug-validator type.

## Best Possible Solution

Gate owner restores on authoritative hidden provenance so owners can restore only their own soft-delete state, then cover staff-hide, scanner, report, security, ban, merge, and owner self-delete cases in tests and user-facing notes.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a moderator can hide through `api.skills.setSoftDeleted`, which sets `hiddenBy` and notes but no `moderationReason`, and the PR’s owner gate would still allow the owner undelete path.

## Solution Assessment

No. Checking only `moderationReason` is too narrow; the maintainable fix should require owner-created hidden provenance, such as `hiddenBy === args.userId`, while preserving moderator/admin override behavior.
