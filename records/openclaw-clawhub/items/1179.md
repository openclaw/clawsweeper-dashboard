## Summary

Keep open. ClawHub cleanup policy keeps issues open, and current main only partially mitigates the reported publish/profile failure: package and skill digest syncs are now scheduled as separate mutations, but each owner-publisher digest helper still loops over `.paginate()` inside one mutation and can still hit Convex's single-pagination limit for larger owner catalogs.

## What This Changes

First-time publisher publish and settings/profile saves should not fail with Convex multiple-pagination errors while ensuring a personal publisher.

## Best Possible Solution

Refactor owner-publisher digest rebuilds, and the analogous owner-user package digest trigger if still reachable, into cursor-carrying one-page scheduled mutations with regression coverage for multi-page package and skill owners; after deployment, consolidate the duplicate publish/profile reports.

## Repair Work Prompt

Fix the Convex pagination failure behind first-time publisher/profile publish flows. Refactor `syncPackageSearchDigestsForOwnerPublisherId` and `syncSkillSearchDigestsForOwnerPublisherId` so each internal mutation performs at most one indexed `.paginate()` call, carries a continuation cursor through validated args, processes one page, and self-schedules follow-up work with `ctx.scheduler.runAfter` until complete. Audit the sibling `syncPackageSearchDigestsForOwnerUserId` users trigger path for the same single-pagination constraint and handle it consistently if still reachable. Preserve existing digest contents, `by_owner_publisher`/`by_owner` index usage, and missing-table fallback behavior. Do not replace the loops with unbounded `collect()` unless maintainers explicitly accept the document-limit tradeoff. Add regression tests that simulate multi-page package and skill owner catalogs and assert only one paginate call per mutation invocation plus continuation scheduling. Regenerate committed Convex generated files if function signatures change. No changelog is expected for this internal backend bug unless the release process requires it.
