## Summary

Keep open under the ClawHub issue-cleanup policy. Current main still matches the reported skill publish/import classification bug: extensionless text scripts and `.tsv` files are omitted by extension-only discovery before upload, and there is no merged PR that owns this exact remaining work.

## What This Changes

The issue asks `clawhub publish` to preserve extensionless text files and `.tsv` files, and to warn authors when publish skips files.

## Best Possible Solution

Adopt one conservative text-file classification policy across skill publish entry points: add `.tsv`, admit extensionless text only through binary-safe byte/content checks, surface meaningful skipped-file warnings, and cover CLI, archive/web upload, GitHub import, and server validation with regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a skill folder with `SKILL.md`, an extensionless text script, and `config/endpoints.tsv` will have the latter two excluded by current discovery/filtering before publish completes.

## Solution Assessment

Yes. A shared conservative classifier plus publisher-visible skipped-file reporting is the narrowest maintainable fix; adding only `.tsv` would leave extensionless wrappers and entry-point divergence unresolved.

## Repair Work Prompt

Fix https://github.com/openclaw/clawhub/issues/874 on current main. Skill publish currently drops extensionless text scripts and `.tsv` files because CLI discovery, archive expansion, GitHub import, and validation paths classify text mostly by extension. Implement a shared conservative classifier for skill text files: add `tsv` to the shared and bundled CLI allowlists, admit extensionless text files only after byte/content sniffing that rejects NUL/binary data, preserve path safety and existing ignored paths, and surface author-visible skipped-file warnings in CLI publish and relevant web upload flows without warning for `.git`, `node_modules`, `.clawhub`, `.clawdhub`, or macOS junk. Keep package binary publish behavior unchanged. Add focused regression tests for CLI file discovery/hashing, schema text helpers, web archive/upload helpers, Convex skill publish validation, and GitHub import. Add a `CHANGELOG.md` fix entry under `0.13.0 - Unreleased`. Stop and ask a maintainer if the implementation would require accepting arbitrary binary executables instead of conservative text files.
