## Summary

Keep open. Current main still matches the reported publish/import classification bug, and ClawHub cleanup policy keeps issues open rather than auto-closing them; this is concrete enough to route to a focused repair PR.

## What This Changes

The issue requests that skill publishing include extensionless text files and `.tsv` files instead of silently dropping them.

## Best Possible Solution

Adopt one shared conservative text-file classifier across publish entry points: add `.tsv`, admit extensionless text files by byte/content checks, preserve binary rejection, warn authors about skipped files, and cover CLI publish, archive upload, GitHub import, and server validation with regression tests.

Yes. Source inspection on current main shows that a skill containing `SKILL.md`, an extensionless text script, and `config/endpoints.tsv` follows publish/import paths that omit the script and `.tsv` file before upload or validation completes.

## Solution Assessment

Yes. A shared conservative classifier plus publisher-visible skipped-file reporting is the narrowest maintainable fix; adding only `.tsv` would leave extensionless scripts and entry-point divergence unresolved.
