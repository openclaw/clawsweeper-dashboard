## Summary

Keep open. Current main still has the reported publish-file classification bug: extensionless text files and `.tsv` files are filtered out before upload/import, and ClawHub policy keeps issues open while routing concrete repairs.

## What This Changes

`clawhub publish` should preserve extensionless text executables and `.tsv` files instead of silently omitting them.

## Best Possible Solution

Adopt one conservative text-file classification policy across skill publish entry points: add `.tsv`, classify extensionless text files by content without admitting binaries, surface skipped-file warnings, and cover CLI, web/archive upload, GitHub import, and server validation with regression tests.

## Reproduction Assessment

Yes, from source inspection. A skill folder or archive containing `SKILL.md`, an extensionless text script such as `bin/gurkerl`, and `config/endpoints.tsv` follows current-main paths that omit those files before publish or import completes.

## Solution Assessment

Yes. A shared conservative classifier plus skipped-file reporting is the narrowest maintainable fix; adding `.tsv` alone would leave extensionless scripts and entry-point divergence unresolved.
