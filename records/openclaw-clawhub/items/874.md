## Repair Work Prompt

Fix the skill publish file classification bug reported in this issue. Current main skips extensionless text files before reading bytes and omits `.tsv` because the shared and bundled text extension allowlists do not include it. Implement a conservative shared policy that adds `.tsv`, accepts extensionless text files only after byte/content inspection that rejects binaries, and surfaces skipped-file warnings to authors without warning on intentional ignores such as `.git`, `node_modules`, `.clawhub`, macOS junk, or configured ignore-file entries. Keep path sanitization and binary rejection intact. Update CLI publish, skill zip hashing/sync behavior where applicable, web/archive upload expansion, GitHub import preview/import, and server-side publish validation consistently. Add focused regression tests for extensionless text scripts, `.tsv` files, binary extensionless files, and skipped-file reporting. Do not broaden skill publishing to arbitrary binary payloads.
