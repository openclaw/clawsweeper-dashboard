## Summary

Keep open. Current main still matches the reported bug: skill publish/import paths filter primarily by extension, `.tsv` is still absent from the text allowlists, and extensionless supporting scripts are omitted before publish/import can include them.

## What This Changes

Skill publishing should include extensionless text executables and `.tsv` supporting files instead of silently dropping them.

## Best Possible Solution

Adopt one conservative text-file classification policy across skill publish entry points, including `.tsv`, extensionless text detection, binary rejection, and author-visible skipped-file reporting.

## Reproduction Assessment

Yes, from source inspection. A skill folder or archive containing `SKILL.md`, an extensionless text script such as `bin/gurkerl`, and `config/endpoints.tsv` follows current-main paths that omit those files before publish/import completes.

## Solution Assessment

Yes. A focused shared classifier plus skipped-file reporting is the narrowest maintainable fix, as long as it preserves the existing text-only security boundary.
