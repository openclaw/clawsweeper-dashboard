## Summary

Keep open. Current main still has extension-only skill file discovery in the CLI, archive upload, and GitHub import paths, `.tsv` is still absent from the text allowlists, and this ClawHub issue should remain open until a fix lands.

## What This Changes

The issue asks for skill publishing to include `.tsv` files, preserve extensionless text scripts, and warn when files are skipped instead of silently publishing a broken skill.

## Best Possible Solution

Use one conservative text-file classification policy across skill publish ingress paths: add `.tsv`, classify extensionless text by bytes/content, preserve binary rejection, and report skipped files to authors.

## Reproduction Assessment

Yes, from source inspection. A skill folder or archive containing `SKILL.md`, an extensionless text script such as `bin/gurkerl`, and `config/endpoints.tsv` deterministically follows current-main paths that omit those supporting files before publish/import completes.

## Solution Assessment

Yes. A focused shared classifier plus skipped-file warnings is the narrowest maintainable repair, as long as it keeps the current text-only security boundary intact.
