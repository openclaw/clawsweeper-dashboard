## Summary

Keep open. Current main still matches the reported publish/import bug, and ClawHub cleanup policy keeps issues open while routing concrete repairs.

## What This Changes

The issue reports that ClawHub publish silently omits extensionless text executables and `.tsv` files, leaving published skills missing required runtime files.

## Best Possible Solution

Adopt one conservative text-file classification policy across skill publish entry points: add `.tsv`, classify extensionless text files by byte/content checks without admitting binaries, surface skipped-file warnings, and add regression coverage across CLI publish, archive/web upload, GitHub import, and server validation.

## Reproduction Assessment

Yes. Source inspection on current main shows an extensionless text script and `config/endpoints.tsv` are filtered out before publish/import can upload them, matching the reported broken installed-skill behavior.

## Solution Assessment

Yes. A shared conservative classifier plus skipped-file reporting is the narrowest maintainable fix; adding `.tsv` alone would leave extensionless scripts and path divergence unresolved.
