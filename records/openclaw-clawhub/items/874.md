## Summary

Keep open. Current main still matches the reported bug: publish/import paths filter files by extension before a byte-based fallback, `.tsv` is absent from the text allowlists, and ClawHub issue cleanup policy keeps issues open while routing focused repairs.

## What This Changes

The issue reports that `clawhub publish` silently omits extensionless text files and `.tsv` files, breaking published skills that depend on those files.

## Best Possible Solution

Centralize a conservative text-file classification policy across skill publish entry points: add `.tsv`, classify extensionless text files by content without admitting binaries, surface skipped-file warnings, and cover CLI, archive/web upload, GitHub import, and server validation with regressions.

## Reproduction Assessment

Yes. Source inspection on current main shows a skill containing `SKILL.md`, an extensionless text script such as `bin/gurkerl`, and `config/endpoints.tsv` follows paths that omit or reject those files before publish/import completes.

## Solution Assessment

Yes. A shared conservative classifier plus skipped-file reporting is the narrowest maintainable fix; adding `.tsv` alone would leave extensionless scripts and entry-point divergence unresolved.
