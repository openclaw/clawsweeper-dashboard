## Summary

Keep open. Current `main` still matches the reported publish bug, and ClawHub cleanup policy keeps issues open rather than auto-closing them; this remains a narrow, source-reproducible repair candidate.

## What This Changes

The issue reports that `clawhub publish` omits extensionless text files and `.tsv` support files without warning.

## Best Possible Solution

Adopt one conservative text-file classification policy across skill publish entry points: add `.tsv`, classify extensionless text files by content without admitting binaries, surface skipped-file warnings, and cover the CLI, archive/web upload, GitHub import, and server validation paths with regression tests.

## Reproduction Assessment

Yes, from source inspection. A skill folder or archive containing `SKILL.md`, an extensionless text script such as `bin/gurkerl`, and `config/endpoints.tsv` follows current-main paths that omit the extensionless and `.tsv` files before publish/import completes.

## Solution Assessment

Yes. A shared conservative classifier plus skipped-file reporting is the narrowest maintainable fix; adding `.tsv` alone would leave extensionless scripts and entry-point divergence unresolved.
