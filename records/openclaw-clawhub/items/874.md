## Summary

Keep open. Current main still matches the reported publish bug: skill publishing and related import paths rely on extension allowlists before reading bytes, `.tsv` is absent from the allowlists, and ClawHub policy keeps issues open while routing concrete repairs.

## What This Changes

The issue reports that `clawhub publish` silently omits extensionless text executables and `.tsv` support files from published skills.

## Best Possible Solution

Adopt one conservative text-file classification policy across publish entry points: add `.tsv`, classify extensionless text files by content without admitting binaries, surface skipped-file warnings, and cover CLI, archive/web upload, GitHub import, and server validation with regression tests.

## Reproduction Assessment

Yes, from source inspection. A skill folder or archive containing `SKILL.md`, an extensionless text script such as `bin/gurkerl`, and `config/endpoints.tsv` follows current-main paths that omit those files before publish or import completes.

## Solution Assessment

Yes. A shared conservative classifier plus skipped-file reporting is the narrowest maintainable fix; adding `.tsv` alone would leave extensionless scripts and entry-point divergence unresolved.
