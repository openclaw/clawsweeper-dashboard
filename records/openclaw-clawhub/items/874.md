## Summary

Keep open. Current main still matches the reported publish-file classification bug, the discussion includes two concrete user impacts, and the ClawHub cleanup policy keeps issues open rather than auto-closing them.

## What This Changes

Skill publishing should preserve extensionless text executables and `.tsv` files, or at least warn when files are skipped.

## Best Possible Solution

Adopt one conservative text-file classification policy across skill publish entry points: add `.tsv`, classify extensionless text files by content without admitting binaries, warn authors about skipped files, and cover CLI publish, archive/web upload, GitHub import, and server validation with regression tests.

## Reproduction Assessment

Yes. A skill folder containing `SKILL.md`, an extensionless text script such as `bin/gurkerl`, and `config/endpoints.tsv` follows deterministic current-main paths that skip the latter two before upload or selection.

## Solution Assessment

Yes. A focused shared classification helper plus targeted warnings is the narrowest maintainable fix, provided it preserves binary rejection and updates both schema copies.
