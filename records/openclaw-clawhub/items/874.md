## Summary

Keep open. Current main still has the reported file-classification bug in source, the discussion has two concrete user impacts, and the ClawHub cleanup policy keeps issues open rather than auto-closing them.

## What This Changes

The issue requests that skill publishing include `.tsv` and extensionless text files instead of silently dropping them.

## Best Possible Solution

Adopt one shared conservative text-file classification policy across CLI publish, web/archive upload, GitHub import, and Convex validation: allow `.tsv`, classify extensionless text by content, preserve binary rejection, and warn authors about skipped files.

## Reproduction Assessment

Yes from source inspection. A skill folder or archive containing `SKILL.md`, an extensionless text script such as `bin/gurkerl`, and `config/endpoints.tsv` deterministically follows current-main paths that omit or reject the latter two before a complete publish.

## Solution Assessment

Yes. A focused shared classifier plus targeted skipped-file warnings is the narrowest maintainable fix, provided it preserves binary rejection and updates both schema copies.
