## Summary

Keep open. Current main still uses extension-based filtering that excludes extensionless files and `.tsv` files before publish/upload, and ClawHub policy keeps issues open rather than auto-closing them.

## What This Changes

The issue requests that skill publish preserve extensionless text executables and `.tsv` config files instead of silently omitting them.

## Best Possible Solution

Adopt one conservative text-file classification policy across CLI publish, web/archive upload, GitHub import, and server validation, with `.tsv` support, extensionless text detection, skipped-file warnings, and regression tests.

## Reproduction Assessment

Yes. A skill folder or archive containing `SKILL.md`, an extensionless text script such as `bin/gurkerl`, and `config/endpoints.tsv` follows deterministic current-main paths that omit the latter two before publish selection or upload.

## Solution Assessment

Yes. A shared conservative classifier plus targeted warnings is the narrowest maintainable fix; adding only `.tsv` or only changing CLI publish would leave extensionless and alternate entry-point failures behind.
