## Summary

Keep open. Current main still matches the report: skill publish paths filter by extension before any conservative content fallback, `.tsv` is still absent from the allowlists, and ClawHub cleanup policy keeps issues open rather than auto-closing them.

## What This Changes

The issue reports that skill publish omits extensionless text files and `.tsv` files without warning.

## Best Possible Solution

Adopt one conservative text-file classification policy across CLI publish, archive/web upload, GitHub import, and server validation: add `.tsv`, classify extensionless text by content without admitting binaries, warn authors about skipped publish files, and cover the behavior with regression tests.

## Reproduction Assessment

Yes, source-reproducible: a skill with `SKILL.md`, `bin/openclaw-kraken`, and `config/endpoints.tsv` will have the latter two excluded by the current extension checks before upload. I did not run a live publish/install round trip in this read-only review.

## Solution Assessment

Yes, the suggested direction is the right boundary if it is implemented as a shared conservative classifier rather than one-off path exceptions. The fix should preserve binary rejection and add author-visible warnings for real omissions.
