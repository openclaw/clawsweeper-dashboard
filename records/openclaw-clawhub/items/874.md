## Summary

Keep open. Current main still has the reported publish-file classification bug: extensionless text files are skipped before their bytes are read, `.tsv` is absent from the allowlists, and ClawHub cleanup policy keeps issues open rather than auto-closing them.

## What This Changes

The issue requests that skill publish preserve extensionless text executables and `.tsv` files, and warn authors when files are skipped.

## Best Possible Solution

Adopt one conservative text-file classification policy across CLI publish, web/archive upload, GitHub import, and Convex validation: allow `.tsv`, sniff extensionless text without admitting binaries, and surface skipped-file warnings.

## Reproduction Assessment

Yes, from source inspection. A skill containing `SKILL.md`, an extensionless text script such as `bin/gurkerl`, and `config/endpoints.tsv` deterministically follows current-main paths that omit or reject the latter two before a complete publish.

## Solution Assessment

Yes. A focused shared classifier plus skipped-file warnings is the narrowest maintainable repair, as long as it keeps the current text-only security boundary intact.
