## Summary

Keep open. Current main still matches the reported publish bug, and ClawHub cleanup policy keeps issues open rather than auto-closing them; this is a narrow repair candidate.

## What This Changes

Issue reports that `clawhub publish` omits extensionless text executables and `.tsv` support files without warning, producing broken installed skills.

## Best Possible Solution

Adopt one conservative text-file classification policy across publish entry points: add `.tsv`, classify extensionless text files by content without admitting binaries, warn on skipped files, and cover CLI, archive/web upload, GitHub import, and server validation with regression tests.

## Reproduction Assessment

Yes, from source inspection. A skill folder or archive containing `SKILL.md`, an extensionless text script such as `bin/gurkerl`, and `config/endpoints.tsv` follows current-main paths that omit those files before publish/import completes.

## Solution Assessment

Yes. A shared conservative classifier plus skipped-file reporting is the narrowest maintainable fix; adding `.tsv` alone would leave extensionless scripts and entry-point divergence unresolved.
