## Summary

Keep open. Current main still matches the reported bug: `.tsv` is absent from the text allowlists, extensionless files are skipped before byte inspection, and the main CLI/upload/import paths still filter by extension only; ClawHub issue cleanup policy keeps issues open while routing focused repairs.

## What This Changes

The issue reports that skill publish silently omits extensionless text executables and `.tsv` files, causing installed skills to miss required runtime files.

## Best Possible Solution

Adopt one conservative text-file classification policy across skill publish entry points: add `.tsv`, classify extensionless text files by content without admitting binaries, surface skipped-file warnings, and cover CLI publish, archive/web upload, GitHub import, and server validation with regression tests.

## Reproduction Assessment

Yes. Source inspection on current main shows a skill folder with `SKILL.md`, an extensionless text script, and `config/endpoints.tsv` follows code paths that omit the script and `.tsv` file before publish/import completes.

## Solution Assessment

Yes. A shared conservative classifier plus publisher-visible skipped-file reporting is the narrowest maintainable fix; adding `.tsv` alone would leave extensionless scripts and entry-point divergence unresolved.
