## Summary

Keep open. Current main still filters skill publish/import files by extension before reading bytes, `.tsv` is absent from the allowlists, and ClawHub policy keeps issues open while routing focused repairs.

## What This Changes

Issue requests that `clawhub publish` stop silently omitting extensionless text files and `.tsv` files from published skills.

## Best Possible Solution

Centralize a conservative text-file classification policy across publish entry points, add `.tsv`, content-check extensionless text files, preserve binary rejection, warn on skipped files, and cover CLI, archive/web upload, GitHub import, and server validation with regressions.

## Reproduction Assessment

Yes, from source inspection. A skill containing `SKILL.md`, an extensionless text script such as `bin/gurkerl`, and `config/endpoints.tsv` follows current-main paths that omit the script and `.tsv` before publish/import completes.

## Solution Assessment

Yes. A shared conservative classifier plus skipped-file reporting is the narrowest maintainable fix; adding `.tsv` alone would leave extensionless scripts and entry-point divergence unresolved.
