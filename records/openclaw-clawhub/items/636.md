## Security Review

Status: needs_attention

Summary: Global mirrors and load balancing are security-sensitive because they change trusted proxy, origin, cache, API consistency, and rate-limit boundaries.

Concerns:

- **[medium] Scope trusted proxy headers before load balancing:** `convex/lib/httpRateLimit.ts:245`
  - body: Current rate-limit code treats forwarded IP headers as spoofable unless TRUST_FORWARDED_IPS is explicitly enabled, so any load balancer must define the trusted proxy layer before client identity is relied on.
  - confidence: 0.86
- **[medium] Define trusted mirror origins before generating API URLs:** `convex/httpApiV1/packagesV1.ts:545`
  - body: Package API URL generation currently trusts only canonical clawhub.ai hosts and local development hosts, so mirror domains need explicit origin, cache, and consistency rules before source changes are safe.
  - confidence: 0.82
