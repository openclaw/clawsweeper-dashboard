## Security Review

Status: needs_attention

Summary: Global mirrors and load balancing would alter trusted proxy, origin, rate-limit, cache, auth/session, and moderation boundaries, so explicit security design is needed before implementation.

Concerns:

- **[medium] Define trusted proxy headers before load balancing:** `convex/lib/httpRateLimit.ts:245`
  - body: The rate-limit code trusts forwarded IP headers only when explicitly enabled because they are spoofable without a controlled proxy layer; a load-balancer design must define that boundary before changing routing.
  - confidence: 0.88
- **[medium] Define trusted mirror origins before serving generated URLs:** `convex/httpApiV1/packagesV1.ts:545`
  - body: The package API currently trusts only clawhub.ai, www.clawhub.ai, and local hosts for forwarded-origin URL generation, so mirror domains need explicit origin, cache, and consistency rules.
  - confidence: 0.84
- **[medium] Preserve moderation gates for mirror surfaces:** `docs/http-api.md:18`
  - body: Public docs explicitly warn third-party directories not to mirror hidden, private, or moderation-blocked content; any official mirror/export path must reuse those visibility and download safety rules.
  - confidence: 0.83
