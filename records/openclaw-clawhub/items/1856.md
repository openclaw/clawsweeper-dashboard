## Summary

Keep open. Current main still has the ignore-aware publish path that can omit a root manifest, and this draft PR is a narrow, non-obsolete implementation for the still-open linked publish bug with no blocking review findings from the read-only diff pass.

## What This Changes

The PR updates `clawhub publish` to recover a root `SKILL.md` or `SKILLS.md` after ignore-aware file discovery and adds a regression test for `.gitignore` `*.md`.

## Best Possible Solution

Land a focused CLI publish fix that preserves the required root manifest despite broad ignore patterns while continuing to respect ignores for non-manifest files and plugin-folder publishing boundaries.

## Reproduction Assessment

Yes. On current main, a skill folder with `.gitignore` containing `*.md` makes `listTextFiles` omit `SKILL.md`, after which `cmdPublish` reaches the `SKILL.md required` failure path; the PR's new test is the focused reproduction scenario.

## Solution Assessment

Yes. Recovering only a root `SKILL.md` or `SKILLS.md` after normal ignore-aware discovery is the narrow maintainable fix, because it preserves existing ignore behavior for arbitrary markdown and leaves backend validation unchanged.
