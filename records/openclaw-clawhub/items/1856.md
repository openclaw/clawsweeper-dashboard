## Summary

Keep open. Current main still has the linked publish failure path: `clawhub publish` validates only the ignore-filtered file list, so a broad ignore pattern can hide the required root manifest. The PR is a narrow draft implementation for that bug, with no blocking correctness or security finding from this read-only review, so it needs maintainer review and validation rather than cleanup closure.

## What This Changes

The PR changes `clawhub publish` to add a root `SKILL.md` or `SKILLS.md` back into the upload when ignore-aware file discovery omitted it, and adds a regression test for `.gitignore` containing `*.md`.

## Best Possible Solution

Land a focused CLI publish fix that preserves the required root manifest despite broad ignore patterns while keeping plugin-folder detection and ignored non-manifest files unchanged, with the regression test guarding the linked publish flow.
