## What This Changes

The PR changes `clawhub publish` to re-add an ignored root `SKILL.md` or `SKILLS.md` manifest after ignore-aware file discovery and adds a regression test for `.gitignore` `*.md`.

## Best Possible Solution

Land a focused CLI publish fix with a regression test and an Unreleased changelog entry, preserving only the required root manifest while leaving other ignored files ignored.

## Reproduction Assessment

Yes. On current main, a skill folder with `.gitignore` containing `*.md` can cause `listTextFiles` to skip the root `SKILL.md`, after which `cmdPublish` fails with `SKILL.md required` before upload.

## Solution Assessment

Yes, with the changelog addition. The PR's publish-level manifest recovery is narrow and avoids changing shared ignore semantics for arbitrary files.
