## What This Changes

The PR changes `clawhub publish` to re-add a present root `SKILL.md` or `SKILLS.md` omitted by ignore-based text-file discovery and adds a regression test for ignored markdown.

## Best Possible Solution

Keep the focused root-manifest recovery so required manifests publish even under broad ignore rules, while non-manifest ignored files stay excluded.

## Reproduction Assessment

Yes from source inspection. On current main, a skill folder with `.gitignore` containing `*.md` can have `SKILL.md` skipped by `listTextFiles`, then `cmdPublish` fails its manifest check before upload.

## Solution Assessment

Yes for the code path. Re-adding only the required root manifest after normal ignored-file discovery is the narrow maintainable fix; the branch still needs the repository-required changelog entry before merge.
