## What This Changes

The PR changes `clawhub publish` to recover an existing root `SKILL.md` or `SKILLS.md` when normal text-file discovery omits it because of broad ignore rules.

## Best Possible Solution

Land this narrow manifest-recovery fix or an equivalent patch with focused regression coverage, then let the linked bug report close after merge.

## Reproduction Assessment

Yes. Source inspection shows a broad ignore rule such as `*.md` can cause `listTextFiles` to skip root `SKILL.md`, leaving current-main `cmdPublish` to fail the manifest check.

## Solution Assessment

Yes. Re-adding only an existing required root manifest after normal discovery is the narrow maintainable fix because arbitrary ignored files remain ignored and backend validation stays unchanged.
