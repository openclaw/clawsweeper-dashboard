## Summary

Keep open: current main still does not preserve an ignored root manifest during `clawhub publish`, and this draft PR is a focused implementation for the linked publish bug with no blocking review finding from the provided diff.

## What This Changes

The PR changes `clawhub publish` to append a required root `SKILL.md` or `SKILLS.md` after normal ignored-file discovery and adds a regression test for `.gitignore` `*.md`.

## Best Possible Solution

Land or supersede this focused CLI fix so publish always includes the required root manifest despite broad Markdown ignore rules while still excluding other ignored files.

## Reproduction Assessment

Yes. On current main, a skill folder with root `SKILL.md` and `.gitignore` containing `*.md` follows a clear source path where `listTextFiles` omits the manifest and `cmdPublish` fails before upload.

## Solution Assessment

Yes. The proposed root-only recovery after normal discovery is narrow, preserves existing ignore behavior for non-manifest files, and avoids changing backend validation.
