## Summary

Keep open. Current main still uses ignore-aware `listTextFiles(folder)` before checking for a root manifest, so the linked publish failure remains; this draft PR is a narrow implementation with no blocking review findings from the read-only diff pass.

## What This Changes

The PR updates `clawhub publish` to re-add a root `SKILL.md` or `SKILLS.md` when normal file discovery misses it due to broad ignore rules, plus a regression test for `.gitignore` `*.md`.

## Best Possible Solution

Land a focused CLI publish fix that restores only the required root manifest after ignore-aware discovery, keeps plugin-folder rejection intact, and preserves ignored supporting files unless explicitly included by existing rules.

## Reproduction Assessment

Yes. A skill folder containing `.gitignore` with `*.md` and a root `SKILL.md` follows a source-supported path where current main filters the manifest out before `cmdPublish` checks for it and raises `SKILL.md required`.

## Solution Assessment

Yes. The proposed approach is narrow: it recovers only a root manifest after normal discovery and leaves backend validation, plugin detection, and arbitrary ignored files unchanged.
