## Summary

Keep open. Current main still lacks the manifest recovery this PR adds: CLI publish validates only the ignore-filtered `listTextFiles` result, while broad ignore rules can hide a present root manifest. The PR is an open draft linked to #1509, and the read-only diff review found no blocking correctness or security issue.

## What This Changes

The PR changes `clawhub publish` to add back a missing root `SKILL.md` or `SKILLS.md` when broad ignore rules exclude Markdown files, with a regression test for `.gitignore` `*.md`.

## Best Possible Solution

Land or supersede this PR with a focused CLI publish fix that preserves the required root manifest despite broad ignore patterns while keeping ignored non-manifest files excluded and plugin-folder rejection intact.

## Reproduction Assessment

Yes. A skill folder with a root `SKILL.md` and `.gitignore` containing `*.md` follows a clear current-main source path where `listTextFiles` omits the manifest and `cmdPublish` fails with `SKILL.md required`.

## Solution Assessment

Yes. The proposed root-only recovery after normal discovery is narrow and avoids changing global ignore behavior that other CLI flows may rely on.
