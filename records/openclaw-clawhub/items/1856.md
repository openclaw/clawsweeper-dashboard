The PR changes `clawhub publish` to re-add an existing root `SKILL.md` or `SKILLS.md` to the multipart upload after ignored text-file discovery and adds regression coverage for a broad `*.md` ignore rule.

## Best Possible Solution

## Solution Assessment

Yes. Re-adding only an existing required root manifest after normal discovery is the narrow maintainable fix because arbitrary ignored supporting files stay ignored and backend validation remains unchanged.
