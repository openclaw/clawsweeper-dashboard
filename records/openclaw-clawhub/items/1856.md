## What This Changes

The PR updates `clawhub publish` to re-add a root `SKILL.md` or `SKILLS.md` when ignore-aware discovery skipped it, with a regression test for `.gitignore` containing `*.md`.

## Best Possible Solution

Land a focused CLI publish fix that preserves the required root manifest despite broad ignore patterns while continuing to respect ignore rules for non-manifest files and plugin-folder rejection.

Yes. Code inspection gives a high-confidence reproduction path on current main: create a skill folder with `.gitignore` containing `*.md`, then `listTextFiles` can omit `SKILL.md` and `cmdPublish` reaches `SKILL.md required`.

## Solution Assessment

Yes. The proposed approach is narrow and maintainable because it keeps normal ignore-aware discovery, recovers only the required root manifest, and leaves backend publish validation unchanged.
