## Summary

Keep open. Current main still lacks the root-manifest recovery path, and the PR appears to be a narrow correct fix, but it is draft and unmerged so it should not be closed as implemented.

## What This Changes

This PR updates the ClawHub CLI publish command to re-add an existing root `SKILL.md` or `SKILLS.md` when ignore-filtered file discovery misses it, with a regression test for `*.md` in `.gitignore`.

## Best Possible Solution

Mark the draft ready after maintainer review, land or supersede the narrow manifest recovery with regression coverage, then close the linked bug once the fix is merged and available.

## Reproduction Assessment

Yes. Source inspection shows a folder with root `SKILL.md` plus a broad ignore rule such as `*.md` can omit the manifest from `listTextFiles` before `cmdPublish` requires it; I did not execute the test because the review is read-only.

## Solution Assessment

Yes. Recovering only an existing required root manifest is the narrow maintainable fix because it preserves ignore behavior for arbitrary files and leaves backend validation unchanged.
