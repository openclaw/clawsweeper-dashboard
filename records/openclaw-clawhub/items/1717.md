## Summary

Keep open. ClawHub cleanup policy keeps open issues open, and the reported outage is still reproducible on current main: both live sign-in endpoints return HTTP 500 while the GitHub provider config still allows empty credential fallbacks.

## What This Changes

## Best Possible Solution

Validate and repair production Convex OAuth env plus the GitHub OAuth callback configuration, add or land a narrow fail-fast diagnostic for missing GitHub OAuth env, deploy, and smoke check both sign-in URLs plus `clawhub login`.

## Reproduction Assessment

Yes. The high-confidence reproduction is the report’s own read-only curl path: both the canonical ClawHub URL and direct Convex sign-in URL returned HTTP 500 against current production on 2026-05-02.

## Solution Assessment

Yes in direction, but incomplete as a source-only change. Production credential and callback validation is necessary for the live outage, while a small `convex/auth.ts` diagnostic would make future misconfiguration clearer without replacing the operator fix.
