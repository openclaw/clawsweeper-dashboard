## Summary

Keep open. This is a reproduced production auth outage: current main still routes GitHub sign-in through environment-backed Convex Auth, both live sign-in endpoints return HTTP 500, and the remaining repair needs maintainer/operator access rather than a source-only cleanup PR.

## What This Changes

The issue reports that production GitHub OAuth sign-in returns HTTP 500 on both clawhub.ai and the direct Convex site, blocking new CLI token generation.

## Best Possible Solution

Restore valid production Convex OAuth env and GitHub OAuth callback/site URL settings, then smoke check both sign-in endpoints, interactive web login, and the CLI token flow; add import-safe diagnostics only as a follow-up.

## Solution Assessment
