## Summary

## What This Changes

The PR adds SkillSpector execution to the Codex security-scan workflow, stores normalized SkillSpector reports on skill/package scan records, feeds them into ClawScan prompts and API/dataset schemas, and renders SkillSpector findings in the security audit UI.

## Best Possible Solution

Integrate SkillSpector through a pinned or otherwise verified source with secrets scoped only to the worker steps that need them, update the security scanner specs/docs, and land with green CI plus redacted worker/UI proof.

## Reproduction Assessment

Yes for the blocking review finding: the workflow diff shows an unpinned `pip install` from GitHub in the same job that has Production secrets at job scope. Runtime feature proof is not available from the PR body.

## Solution Assessment

No; using SkillSpector as evidence may be a good direction, but the implementation should first pin or verify the scanner source, minimize secret scope, and document the new scanner contract.
