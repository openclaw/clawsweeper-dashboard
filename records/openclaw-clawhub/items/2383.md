## Summary

Keep this PR open: the SkillSpector direction is coherent, but the current patch has a production supply-chain blocker and failing static/type checks before it can be considered merge-ready.

## What This Changes

This PR integrates SkillSpector into the Codex security scan worker, stores and exposes normalized SkillSpector analysis, updates dataset/API schemas, and renders SkillSpector findings in the security audit UI.

## Best Possible Solution

Adopt SkillSpector only through a pinned and reviewable dependency path, keep Codex as final policy adjudicator, preserve legacy ClawScan fallback, and require build-clean CI plus redacted real behavior proof for the worker and UI paths.

## Reproduction Assessment

Yes for the PR blockers: CI logs reproduce the unused `container` failure, and source inspection shows the unpinned SkillSpector install in the Production workflow. The feature behavior itself was not reproduced with real runtime proof in the PR discussion.

## Solution Assessment

No: using SkillSpector as evidence for Codex may be the right product direction, but this implementation should pin the scanner source, clear CI, and include real worker/UI proof before merge.
