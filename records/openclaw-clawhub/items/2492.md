## Summary

This PR remains necessary because current main still lets the bounded-prompt legacy evaluator write stored analysis and moderation outcomes. The advisory-only direction is well-scoped and unit-tested, but the branch is currently conflicting with main and lacks the real Convex/worker proof required for this security-sensitive authority and persisted-job change.

## What This Changes

The PR makes legacy hosted artifact evaluation advisory-only, redirects authoritative skill and package scans to the Codex-backed job worker, carries preserve-mode through queued skill scans, warns local dry-run users, and documents the moderation invariant.

## Best Possible Solution

Refresh the branch onto current main, retain the Codex-backed materialized-artifact worker as the sole authoritative verdict writer, keep legacy entry points explicitly advisory for compatibility, and prove normal plus preserve-mode state transitions in a real Convex worker setup before merge.

Yes, from current-main source with high confidence: `evaluateWithLlm` can write stored analysis and pass moderation mode after evaluating bounded artifact excerpts. This review did not run a live deployment, so the runtime symptom remains source-reproducible rather than directly reproduced.

## Solution Assessment

Yes in architectural direction: using the existing materialized-artifact Codex worker as the sole authoritative source is narrower and safer than maintaining two authoritative evaluators. Merge readiness still depends on explicit compatibility approval, conflict resolution, and real runtime proof.
