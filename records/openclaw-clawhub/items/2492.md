## Summary

Current main still lets the bounded hosted evaluator write authoritative artifact analysis, while this focused PR removes that authority and routes legacy entry points through the existing Codex worker. The direction is coherent and automated coverage is broad, but this external security-sensitive Convex change still lacks after-fix proof from a real deployment, so it should remain open rather than close or merge yet.

## What This Changes

The PR makes legacy hosted skill and package artifact evaluations advisory-only, redirects legacy manual and backfill scans to Codex-backed jobs, carries preserve mode through skill jobs, and records the authority invariant in the moderation spec.

## Best Possible Solution

Keep the Codex worker as the sole authoritative artifact scanner, retain hosted evaluation only as clearly labeled diagnostics, and verify normal, preserve-mode, and active-job compatibility paths in a real Convex deployment before merge.

## Reproduction Assessment

Yes at source level: current main’s bounded hosted evaluator writes artifact analysis that feeds authoritative moderation state. A live deployment reproduction was not performed during this read-only review.

## Solution Assessment

Yes in design: making bounded evaluation advisory and using the existing materialized-workspace worker as the single authority is narrower and safer than maintaining two trust-state writers. Merge readiness still requires real Convex proof of the compatibility paths.
