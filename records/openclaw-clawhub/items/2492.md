## Summary

Keep this PR open for maintainer review: current main still permits bounded-prompt legacy artifact evaluations to write stored security results, and the patch coherently moves that authority to the full-artifact worker, but the contributor has supplied only mocked tests and CI rather than the required real Convex enqueue-to-completion proof.

## What This Changes

The PR makes legacy hosted artifact evaluation advisory-only, redirects manual and backfill rescans to Codex-backed security-scan jobs, and persists preserve-mode behavior through queued skill scans.

## Best Possible Solution

Retain the single full-artifact Codex worker as the authoritative writer, keep bounded legacy evaluation explicitly diagnostic-only, and demonstrate normal and preserve-mode legacy rescans end to end in a real Convex deployment before merging.

## Reproduction Assessment

Yes, from source inspection: current main's legacy artifact actions can persist bounded-prompt LLM results, while the PR removes those writes and routes callers to the full-artifact worker. The asynchronous replacement has not been demonstrated in a real Convex setup.

## Solution Assessment

Yes in direction: consolidating authoritative decisions on the materialized-artifact worker is narrower and safer than maintaining two trust-writing evaluators. Merge readiness still depends on proving normal and preserve-mode compatibility end to end.
