## Summary

Current main still allows the bounded hosted artifact evaluator to write stored analysis and potentially public moderation state, so this PR remains necessary and its move to the existing Codex-backed worker is coherent. Keep it open, but require real Convex proof and explicit maintainer acceptance because legacy manual/backfill rescans become asynchronous and the authoritative trust-state writer changes.

## What This Changes

The PR makes legacy hosted artifact evaluation advisory-only, routes legacy manual and backfill rescans through `securityScanJobs`, propagates preserve mode to worker completion, and documents the moderation invariant.

## Best Possible Solution

Use the Codex-backed worker as the sole authoritative artifact-verdict path while retaining the legacy evaluator only for clearly labeled diagnostics, with real Convex evidence that normal and preserve rescans enqueue, claim, and complete with the intended moderation behavior.

## Reproduction Assessment

## Solution Assessment

Yes in architectural direction: consolidating authoritative artifact verdicts on the existing full-workspace Codex worker is narrower and safer than maintaining two trust writers. Merge readiness remains unclear until maintainers accept the asynchronous compatibility change and the contributor supplies real runtime proof.
