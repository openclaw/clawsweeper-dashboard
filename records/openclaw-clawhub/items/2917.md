## Best Possible Solution

Align `/search` and `/skills?q=...` web rendering with the existing `/api/v1/search` results for public exact-slug matches, while preserving moderation filters and owner diagnostics.

## Reproduction Assessment

## Solution Assessment

Yes, the likely best fix is to make web route initial rendering/hydration use the same backend search results that CLI/API already use. Republish/backfill guidance alone is not enough because the API side now works while web-rendered search still does not.
