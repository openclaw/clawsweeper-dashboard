## Summary

Keep open. Current main still has only prefix-index direct skill search, while this PR targets the linked underrecall bug and looks directionally sound, but it needs after-fix real behavior proof and maintainer coordination with parallel open fixes for the same issue.

## What This Changes

The PR adds full-text search indexes on `skillSearchDigest` slug/displayName, queries them from `directPrefixSkillMatches` alongside existing prefix indexes, and adds regression tests for middle-token recall and suspicious filtering.

## Best Possible Solution

Land one reviewed full-text digest search implementation that preserves moderation filters and includes real behavior proof, then close or supersede the duplicate parallel PRs.

## Reproduction Assessment

Yes, source-reproducible from current main. `directPrefixSkillMatches` only ranges over normalized full strings and first-token fields, so a digest whose slug/displayName is `baidu-yijian-vision` misses `yijian` or `vision` unless vector search or the recency fallback happens to catch it.

## Solution Assessment

Yes, the Phase 1 approach of adding full-text digest indexes alongside existing prefix paths is a narrow maintainable fix direction. The merge path should be one canonical PR with live behavior proof and the suspicious/soft-delete filters preserved.
