Keep open under the ClawHub issue cleanup policy. Current main appears to have fixed the source-level Convex pagination bug with one-page continuation mutations, so the remaining maintainer work is deployment verification and duplicate-cluster consolidation rather than a new code fix.

## What This Changes

The issue reports `clawhub publish` failing after successful authentication with Convex's multiple-paginated-query error in the owner-publisher digest sync path.

## Best Possible Solution

Verify production includes the continuation-cursor digest sync fix, then close or cross-link the related publish-pagination reports against the deployed resolution.

## Reproduction Assessment

Yes. The reported stack maps directly from CLI publish to the v1 skills API, `publishVersionForUser`, `insertVersion`, and `ensurePersonalPublisherForUser`; current-main source and tests verify the formerly failing pagination shape is now split into one-page continuations.

## Solution Assessment

Yes. The continuation-cursor implementation is the maintainable fix because it preserves indexed pagination while avoiding multiple paginated queries in one Convex mutation, unlike the closed PR's unbounded `.collect()` approach.
