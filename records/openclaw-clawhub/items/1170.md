## Summary

Keep open. The ClawHub cleanup policy keeps issues open, and current main still has a concrete Convex pagination hardening gap on the reported publish path: owner-publisher digest sync is scheduled into separate internal mutations, but each scheduled mutation can still loop through multiple `.paginate()` calls for an owner with more than one page of packages or skills.

## What This Changes

The issue reports authenticated CLI publish failures caused by Convex's "multiple paginated queries" error in `syncSkillSearchDigestsForOwnerPublisherId` via `ensurePersonalPublisherForUser`.

## Best Possible Solution

Implement cursor-based continuation for owner-publisher package and skill digest rebuilds: each scheduled internal mutation should process one indexed page, update digests for that page, and reschedule itself with `continueCursor` until complete. Add focused tests for multi-page package and skill rebuilds, then consolidate the related publish-pagination reports after deployment verification.
