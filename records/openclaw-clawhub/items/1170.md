## What This Changes

The issue reports authenticated `clawhub publish` failing because backend publisher digest sync hit Convex's multiple paginated query limit.

## Best Possible Solution

Keep the cursor-based continuation implementation, verify the Convex backend deployment includes it, then manually close or consolidate the related publish-pagination issues once affected users can publish.

## Solution Assessment

Yes. Current main's self-scheduling cursor continuation is the narrow maintainable fix because it keeps one paginated query per Convex mutation invocation without replacing pagination with unbounded `.collect()` reads.
