## Summary

Current main resolves the original ClawHub-side telemetry ambiguity by recording explicit install events, defining the counters, and ranking by installs, but the issue is still necessary because human-readable `explore` output omits the selected metric and OpenClaw CLI parity is not established in this repository. The collaborator authorship and open conflicting follow-up also require explicit maintainer handling.

## What This Changes

Clarify and align download, install, and trending metric semantics across ClawHub, its CLI, OpenClaw CLI, API, UI, documentation, and tests.

## Best Possible Solution

Keep explicit per-install telemetry as the canonical adoption signal, display the relevant value and label for every human-readable `explore` sort, verify or implement equivalent OpenClaw install reporting, and record the final cross-repository contract in public docs and the existing durable spec.

## Reproduction Assessment

Yes. Current-main source directly shows the explicit install-event model and that sorted non-JSON `explore` lines omit metric values; OpenClaw parity can only be completed with cross-repository verification.

## Solution Assessment

Partly. The merged explicit-event model is a maintainable solution to the original telemetry ambiguity, but it does not by itself satisfy the visible CLI metric and two-CLI consistency acceptance criteria.
