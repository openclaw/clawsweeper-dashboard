## Summary

Current main resolves the original sync-derived install-count ambiguity for the ClawHub CLI by recording explicit install events, but this collaborator-authored issue remains necessary because human-readable explore output still hides its selected metric, OpenClaw CLI parity is not established in this repository, and an active follow-up PR still owns ranking and telemetry-hardening decisions.

## What This Changes

Clarify and align download, current-install, all-time-install, and trending semantics across the ClawHub site/API/CLI and the OpenClaw skills CLI.

## Best Possible Solution

Adopt one maintainer-approved telemetry and ranking model, publish canonical definitions for all four metrics, show the selected metric in non-JSON explore results, and coordinate matching OpenClaw install behavior or explicit user-facing wording before closing the issue.

## Reproduction Assessment

Yes, at source level: current main directly shows the explicit ClawHub install-event path and the metric-free human explore renderer. This is primarily a product-semantics and UX consistency issue, not a failing runtime bug requiring an execution reproduction.

## Solution Assessment

Partly. Explicit install events are a clearer source of install truth than sync snapshots, but the best solution still requires a maintainer decision on ranking plus consistent definitions, output, and OpenClaw CLI behavior.
