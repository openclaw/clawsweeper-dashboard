## Summary

Current main resolves the original sync-derived install-count confusion for the ClawHub CLI, but the full request remains necessary: human-readable `explore` output still hides the selected metric, OpenClaw CLI alignment is not established here, and https://github.com/openclaw/clawhub/pull/2581 still owns ranking and telemetry-hardening decisions. This collaborator-authored issue therefore needs explicit maintainer handling.

Likely related people: `jesse-merhi` is the high-confidence owner of the current telemetry implementation; `Patrick-Erichsen` is the high-confidence owner of the metric-semantics request and current telemetry documentation; `steipete` is a medium-confidence adjacent owner from extensive history in the affected CLI/backend files.

## What This Changes

Clarify and align download, current-install, all-time-install, and trending semantics across ClawHub, OpenClaw, the API, UI, documentation, and CLI output.

## Best Possible Solution

Adopt one durable metric specification, resolve ranking and deduplication in the linked follow-up, then coordinate ClawHub and OpenClaw so both install commands, API/UI labels, documentation, and sorted human output expose the same predictable contract with focused runtime tests.

## Reproduction Assessment

Yes at the source level. The old sync-derived behavior, current explicit ClawHub install-event path, documented counters, and metric-free human `explore` output are directly traceable, although this is primarily a product-consistency request rather than a single failing runtime bug.

## Solution Assessment

Not fully. Explicit install events are a better foundation than sync-derived snapshots, but the narrow maintainable outcome also requires a settled ranking contract, visible sorted metrics, durable specifications, and coordinated OpenClaw behavior.
