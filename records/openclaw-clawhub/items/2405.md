## Summary

Keep open: current main only partially addresses install reporting, while the requested cross-surface metric contract remains unsettled and two active implementation PRs propose incompatible ranking policies. Because this is a collaborator-authored product decision, it requires explicit maintainer handling rather than cleanup closure.

## What This Changes

Define and consistently expose download, current-install, all-time-install, and trending semantics across ClawHub, both install CLIs, API/UI labels, documentation, and explore output.

## Best Possible Solution

Approve a focused durable spec for downloads, current installs, all-time installs, trending, deduplication, removal/staleness, and ranking use; then consolidate one canonical PR path and coordinate OpenClaw so telemetry, explore output, API/UI copy, docs, compatibility behavior, and real runtime tests all implement that contract.

## Reproduction Assessment

Yes, at source level. Current main exposes separate archive-download, root-snapshot install, current/all-time install, and trending paths, making the ambiguity directly traceable even though this is a semantics problem rather than one failing runtime action.

## Solution Assessment

No single proposed direction is yet the best solution. The two open PRs disagree on the permanent ranking contract, so maintainers should approve the metric semantics and compatibility behavior before consolidating implementation.
