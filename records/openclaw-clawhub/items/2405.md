## Summary

Keep open: current main still uses sync-derived install snapshots, records archive fetches separately as downloads, mixes adoption signals in ranking, and does not fully explain sorted CLI output. Two open implementation PRs propose different ranking policies, and this collaborator-authored issue requires an explicit product decision.

## What This Changes

Clarify and align download, current-install, all-time-install, and trending semantics across ClawHub, its CLI and APIs, and the OpenClaw CLI.

## Best Possible Solution

Record one durable metric contract distinguishing archive fetches, current inventory, lifetime adoption, and trending, then select a canonical branch and align telemetry, ranking, UI/API labels, both CLIs, documentation, and real Convex regression coverage to it.

## Reproduction Assessment

Yes, at source level. Current main directly exposes archive-download events, sync-derived current and all-time installs, mixed ranking signals, and CLI sort modes whose selected metric is absent from human-readable output.

## Solution Assessment

Unclear. The consistency goal is appropriate, but maintainers must choose whether explicit install events or synchronized inventory define installs, how deduplication and current-install semantics work, and which adoption signal influences ranking.
