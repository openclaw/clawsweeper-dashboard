## Summary

Current main resolves the old sync-only ClawHub telemetry model, but the requested cross-client metric contract is still incomplete: human-readable explore output omits the selected metric, OpenClaw installer parity is not established here, and a clean open PR still owns ranking and telemetry follow-up. The item is also collaborator-authored, so it requires explicit maintainer handling.

## What This Changes

Define and consistently expose downloads, current installs, all-time installs, and trending semantics across ClawHub, OpenClaw, API/UI documentation, install commands, and ranked CLI output.

## Best Possible Solution

Adopt one canonical definitions section for downloads, current installs, all-time installs, and trending; make both install clients emit or explicitly explain the same adoption signal; and show the selected metric and value in human-readable ranked output.

## Reproduction Assessment

Yes at source level. Current main deterministically exposes the explicit ClawHub install event, distinct download and ranking signals, and a human-readable explore formatter that omits metric values; OpenClaw parity requires cross-repository validation.

## Solution Assessment

No, the merged ClawHub install-event change alone is not the complete solution. The narrow maintainable solution is a maintainer-approved shared contract implemented across both clients, documentation, API/UI labels, and ranking output.
