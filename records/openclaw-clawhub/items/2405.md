## Summary

Keep open: current main confirms the reported split between archive download counters and telemetry-derived install counters, and the remaining work is a real cross-CLI wording/product decision rather than an already-implemented cleanup.

## What This Changes

Request to define and consistently present downloads, installsCurrent, installsAllTime, and trending semantics across docs, site/API labels, the ClawHub CLI, and OpenClaw install flows.

## Best Possible Solution

Choose one install-count source of truth, document the four stat terms in one public place, mirror that wording in CLI/API/UI labels, and add focused tests around ClawHub install, OpenClaw install expectations, telemetry sync, and explore metric output.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a product/docs/CLI semantics request. Source inspection does confirm the current behavior the issue describes: archive fetches count as downloads, while install counters come from later authenticated telemetry sync snapshots.

## Solution Assessment

Unclear until maintainers pick the intended product model. The narrowest maintainable path is probably to keep telemetry as the install source of truth and make that explicit everywhere, unless maintainers intentionally want install commands to emit a new adoption signal.
