## Summary

Keep open: current main has partially addressed the issue by moving ClawHub install counts to explicit `clawhub install` telemetry, but the broader cross-surface clarification is still not complete and the issue is collaborator-authored with product-decision context. Source inspection still shows human `clawhub explore` output does not show the selected sort metric, API docs and recommendation-score code disagree about whether installs affect recommended ranking, and an open related PR still owns install telemetry/ranking cleanup.

## What This Changes

The issue asks maintainers to define and align downloads, install counters, trending, and ranking language across ClawHub docs, ClawHub CLI, API, UI, and OpenClaw CLI behavior.

## Best Possible Solution

Keep this issue open until maintainers explicitly settle the remaining install/download semantics across CLI, API, UI, OpenClaw integration, and the open telemetry/ranking PR.

## Solution Assessment

Unclear: explicit install events are a maintainable answer for ClawHub CLI counters, but deciding whether to add metric columns, align OpenClaw CLI behavior, and use installs or downloads in ranking remains a maintainer product decision.
