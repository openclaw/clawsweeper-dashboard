## Summary

Current main still separates archive-fetch downloads from sync-derived install counts and does not explain the selected metric in human-readable explore output. This remains necessary, but the issue is collaborator-authored and the authoritative install signal is an active product decision also being explored by two open PRs.

## What This Changes

Clarify and align download, install, and trending metric semantics across ClawHub and OpenClaw user surfaces.

## Best Possible Solution

Settle the authoritative install-event and ranking contract with the active telemetry work, record the decision in a focused spec, then align ClawHub and OpenClaw CLI help, public docs, API/UI labels, and human-readable explore metrics with regression coverage.

## Reproduction Assessment

Yes, at source level. Current main separately records archive downloads and sync-derived installs, while the human-readable explore output does not show the metric used for ranking; no runtime execution was needed to establish those code paths.

## Solution Assessment

Unclear until maintainers choose the metric contract. The requested cross-surface audit is appropriate, but changing install events should be coordinated with https://github.com/openclaw/clawhub/pull/2535 and https://github.com/openclaw/clawhub/pull/2581 rather than implemented as isolated copy changes.
