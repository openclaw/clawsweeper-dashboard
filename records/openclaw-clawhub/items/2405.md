## Summary

Current main fixes the original sync-derived install mismatch for `clawhub install`, but the issue is still necessary: non-JSON `clawhub explore` output does not display the selected metric, no single document defines downloads, both install counters, and trending, and parity with `openclaw skills install` is not established. This collaborator-authored issue also requires explicit maintainer judgment, while https://github.com/openclaw/clawhub/pull/2581 actively owns part of the remaining ranking and telemetry contract.

## What This Changes

Clarify and align download, install, and trending metric semantics across ClawHub, OpenClaw, the site, APIs, docs, and CLI output.

## Best Possible Solution

Resolve the canonical telemetry/ranking follow-up, then add one durable spec and coordinated public documentation defining all four metrics, ensure OpenClaw emits the chosen install signal, and show the selected metric in non-JSON explore output.

## Reproduction Assessment

Yes. Current main provides a high-confidence source path by comparing the explore sort mapping with `formatExploreLine` and the separate telemetry/download documentation with the requested four-term definition.

## Solution Assessment

Yes in direction, but not complete. Explicit install events are more predictable than sync snapshots; the maintainable finish is coordinated cross-CLI emission, canonical definitions, and metric-aware output after maintainers settle ranking semantics.
