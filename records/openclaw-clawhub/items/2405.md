## Summary

Current main resolves the original sync-derived ClawHub install semantics, but it does not prove that OpenClaw installs report the same signal, and the ranking contract remains under active revision. This collaborator-authored issue should remain the product-contract tracker while the open related PR is reviewed.

## What This Changes

Clarify and align the meaning and presentation of downloads, current installs, all-time installs, and trending across ClawHub and OpenClaw surfaces.

## Best Possible Solution

Choose the canonical adoption and ranking definitions, preserve them in a focused spec and user docs, make both install clients emit or explicitly distinguish the same signal, and show the active sort metric in human-readable explore output.

## Reproduction Assessment

Yes at source level: current docs and code expose distinct download, explicit-install, and trending paths, and current documentation limits install reporting to logged-in `clawhub install`. The unresolved cross-CLI and display consistency can be checked directly against those contracts.

## Solution Assessment

Yes in direction, but incomplete: explicit install events are clearer than sync-derived snapshots, while maintainers still need to choose the ranking signal and ensure OpenClaw and human CLI output follow the same contract.
