## Summary

Current main resolves the original sync-lag model for the ClawHub CLI by recording explicit install events, but this collaborator-authored issue still owns unresolved ranking semantics, explainable human-readable output, consistent metric definitions, and parity with the separate OpenClaw CLI. A clean open related PR is an active implementation candidate, so the issue should remain open until that work lands or is rejected.

## What This Changes

Clarify and align download, install, and trending metric semantics across the ClawHub/OpenClaw CLIs, API, site, documentation, and tests.

## Best Possible Solution

Choose and land the canonical ranking and telemetry model, show the selected metric in human-readable explore output, and document and test consistent behavior for both ClawHub and OpenClaw install commands.

## Reproduction Assessment

Yes at source level: current main has concrete download, install-telemetry, ranking, and documentation paths, and the merged related PR confirms the previous sync-derived behavior. This is primarily a product-semantics request rather than a failing runtime command.

## Solution Assessment

No, not completely. Explicit install events are a sound replacement for sync-derived snapshots, but the narrowest complete solution still requires a decided ranking signal, visible sorted metrics, and verified OpenClaw CLI parity.
