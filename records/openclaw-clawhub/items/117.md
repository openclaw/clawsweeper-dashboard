## Summary

Keep this issue open. Current main has owner soft-delete/restore through backend, HTTP API, CLI, docs, and tests, but the signed-in web app still lacks a discoverable owner delete/restore workflow and the dashboard only lists active skills.

## What This Changes

Skill authors want a self-service way to delete their own published skills from ClawHub.

## Best Possible Solution

Add a signed-in web owner delete/restore flow over the existing soft-delete semantics, with destructive confirmation and a safe way to find owner-deleted skills.

## Reproduction Assessment

Yes. Source inspection shows an owner can reach dashboard/settings, but current main exposes settings/rename/merge only while dashboard skill queries filter soft-deleted rows before restore is possible.

## Solution Assessment

Yes. The narrow maintainable path is a web flow over the existing soft-delete/restore primitives, not hard-delete behavior or a new deletion policy.
