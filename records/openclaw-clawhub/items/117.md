## Summary

Keep this issue open under the ClawHub cleanup policy. Current main has backend, HTTP API, CLI, docs, and tests for owner soft-delete/undelete, but the signed-in web app still does not expose a skill delete/restore workflow and the dashboard owner query excludes soft-deleted skills before owners could restore them.

## What This Changes

The issue requests a self-service way for skill authors to delete or remove their own published skills.

## Best Possible Solution

Add a signed-in web delete/restore workflow that reuses existing soft-delete semantics, derives identity server-side, handles direct owners and publisher membership consistently, and never adds owner hard-delete behavior.

## Reproduction Assessment

Yes, from source inspection. The owner settings panel has rename/merge only, dashboard skill rows lack a delete action, and owner dashboard queries filter soft-deleted skills before rendering.

## Solution Assessment

Yes. The narrow maintainable path is to add owner-facing web controls around the existing soft-delete helper and tests, not to change CLI/API semantics or introduce hard delete for normal owners.

Implement the self-service owner skill delete/restore web workflow for https://github.com/openclaw/clawhub/issues/117. Reuse the current soft-delete semantics in convex/skills.ts; derive identity server-side; preserve the existing rule that owners may restore owner-initiated hides but not moderation, security, or malware-hidden skills; support direct owners and publisher members consistently with the existing owner settings/dashboard access model. Add discoverable web controls from SkillOwnershipPanel and/or dashboard row actions with an explicit destructive confirmation. Add a way for owners to see and restore owner-deleted skills, such as an include-deleted dashboard view or a direct restore path. Do not add hard-delete behavior for normal owners, do not weaken moderation gates, and do not change CLI/API behavior except as needed to share implementation safely. Use https://github.com/openclaw/clawhub/pull/1029 as prior art only; rebase the design on current main.
