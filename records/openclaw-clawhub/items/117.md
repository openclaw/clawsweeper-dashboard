## Summary

Keep this issue open. Current main supports owner skill soft-delete and restore through the HTTP API and CLI, but the signed-in web owner surfaces still lack a discoverable skill delete/restore workflow, and related user deletion requests remain open.

## What This Changes

The issue asks for skill authors to be able to delete their own published skills from ClawHub.

## Best Possible Solution

Add a signed-in web delete/restore workflow that reuses existing soft-delete semantics, confirms destructive actions, derives identity server-side, supports valid owner/publisher membership, and gives owners a way to find and restore their own soft-deleted skills.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: sign in as a skill owner, open the skill settings or dashboard, and current main exposes rename/merge/settings/rescan but no skill delete/restore action while dashboard queries filter soft-deleted rows out.

## Solution Assessment

Yes. The narrow maintainable solution is to add web UI and a server-side owner mutation or wrapper around the existing soft-delete primitive, not to add hard delete or change the CLI/API contract.
