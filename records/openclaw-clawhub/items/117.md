## Summary

Keep open. Current main still has owner soft-delete/restore in the backend, HTTP API, CLI, docs, and tests, but the signed-in web owner surfaces do not expose a discoverable delete/restore workflow, and ClawHub cleanup policy keeps issues open rather than auto-closing them.

## What This Changes

Skill authors want to delete or restore their own published ClawHub skills without maintainer intervention.

## Best Possible Solution

Add a signed-in web delete/restore workflow that reuses existing soft-delete semantics, derives identity server-side, supports direct owners and publisher admins consistently, and lets owners find and restore their own owner-deleted skills.

## Reproduction Assessment

Yes, from source inspection. The settings and dashboard web paths lack delete/restore controls while current backend/API/CLI/docs already define owner soft-delete and restore behavior.

## Solution Assessment

Yes. A focused web UI/query repair around the existing soft-delete helper is narrower and safer than adding hard-delete behavior or changing the CLI/API contract.
