## Summary

Keep this issue open. Current main supports owner soft-delete and restore through the HTTP API and CLI, but the signed-in web app still has no owner-visible delete/restore workflow, and the dashboard list query filters soft-deleted skills before an owner could restore them there.

## What This Changes

The issue requests that ClawHub authors be able to delete their own published skills.

## Best Possible Solution

Add a signed-in owner web workflow that reuses the existing soft-delete semantics, derives identity server-side, authorizes direct owners and publisher members consistently, confirms deletion, and lets owners find and restore soft-deleted skills from settings and/or dashboard inventory.
