## Summary

Keep this issue open. Current main has owner soft-delete and restore through the backend, HTTP API, CLI, docs, and tests, but the signed-in web app still lacks a discoverable owner delete/restore workflow and the dashboard cannot show owner-deleted skills for restore.

## What This Changes

Request a self-service way for skill authors to delete and restore their own published skills.

## Best Possible Solution

Add a signed-in web delete/restore workflow over the existing soft-delete primitive, with destructive confirmation, server-derived identity, indexed owner-deleted discovery, and no normal-owner hard delete.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a signed-in owner can open dashboard or skill settings, but dashboard rows only link to settings, settings only shows summary/publisher note/rename/merge, and owner dashboard queries exclude soft-deleted skills.

## Solution Assessment

Yes. The narrow maintainable solution is to reuse the existing soft-delete/restore helper in the web app, not add hard-delete behavior or change the deletion policy.
