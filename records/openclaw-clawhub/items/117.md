## Summary

Keep this issue open. Current main supports owner soft-delete and restore through backend, API, CLI, docs, and tests, but the signed-in web app still lacks a discoverable owner delete/restore workflow and dashboard queries hide soft-deleted owner skills before they can be restored.

## What This Changes

The issue asks for skill authors to be able to delete their own published ClawHub skills.

## Best Possible Solution

Add a signed-in web delete/restore workflow that reuses the existing soft-delete helper, derives identity server-side, supports direct owners and publisher admins consistently, and gives owners a discoverable restore path.

## Reproduction Assessment

Yes, from source inspection. The backend/API/CLI/docs support owner soft-delete and restore, while the owner settings panel and dashboard do not expose delete/restore controls or a restoration view.

## Solution Assessment

Yes, assuming maintainers want a web owner workflow. A focused UI/query change around the existing soft-delete helper is the narrowest maintainable path; hard-delete behavior or a second deletion contract would be riskier.
