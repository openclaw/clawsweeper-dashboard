## Summary

Keep this issue open. Current main has owner soft-delete/restore through backend, HTTP API, CLI, and docs, but the signed-in web app still does not expose a delete/restore workflow and the dashboard only paginates active owner skills, so the user-facing ClawHub gap remains.

## What This Changes

The issue requests that skill authors be able to delete their own published skills from ClawHub.

## Best Possible Solution

Add a signed-in owner web delete/restore workflow that reuses existing soft-delete semantics, derives identity server-side, confirms destructive actions, supports publisher membership consistently, and lets owners find and restore soft-deleted skills.

## Reproduction Assessment

Yes. Source inspection on current main shows a signed-in owner can reach settings and dashboard, but the settings panel exposes only rename/merge, row actions expose settings/rescan, and owner dashboard queries exclude soft-deleted rows.

## Solution Assessment

Yes. The narrow maintainable solution is to reuse the existing soft-delete primitive behind authenticated web mutations and owner-visible UI/query support; hard delete or client-supplied identity would be the wrong direction.
