## Summary

Keep open. Current main supports owner soft-delete/restore through API, CLI, and docs, but the signed-in web settings/dashboard still lacks owner skill delete/restore controls and the owner dashboard only queries active skills, so the web self-service deletion gap remains.

## What This Changes

Issue requests that ClawHub skill authors be able to delete their own published skills.

## Best Possible Solution

Add an authenticated web owner delete/restore workflow that reuses existing soft-delete semantics, confirms destructive actions, and lets owners find and restore their soft-deleted skills.

## Reproduction Assessment

Yes. Source inspection on current main shows owner settings render rename/merge only, dashboard skill row actions omit delete/restore, and owner dashboard queries filter out soft-deleted skills before rendering.

## Solution Assessment

Yes. The narrow maintainable fix is web UI/query/mutation support on top of the existing soft-delete primitive; hard delete or client-supplied actor identity would be the wrong boundary.
