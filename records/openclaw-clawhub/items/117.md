## Summary

Keep this issue open. Current main supports owner soft-delete and restore through backend, HTTP API, CLI, docs, and tests, but the signed-in web app still lacks a discoverable skill delete/restore workflow and owner dashboard queries exclude soft-deleted skills before owners can restore them.

## What This Changes

The issue requests that ClawHub skill authors be able to delete their own published skills.

## Best Possible Solution

Add a signed-in web delete/restore workflow that reuses the existing soft-delete semantics, confirms destructive actions, and lets owners find and restore owner-deleted skills.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: sign in as a skill owner, open the dashboard row menu or skill settings, and current main exposes settings/rescan/rename/merge but no skill delete or restore control while owner dashboard queries filter out soft-deleted skills.

## Solution Assessment

Yes. The narrow maintainable solution is a web flow over the existing soft-delete/restore primitive, not hard-delete behavior or a new deletion policy.
