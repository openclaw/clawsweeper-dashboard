## Summary

Keep this issue open. Current main supports owner soft-delete/restore through the backend, HTTP API, CLI, docs, and tests, but the signed-in web app still lacks a discoverable owner delete/restore workflow and dashboard restore path.

## What This Changes

The issue requests a way for skill authors to delete their own published skills.

## Best Possible Solution

Add a signed-in web delete/restore workflow that reuses the existing soft-delete helper, exposes owner-deleted rows for restore, and keeps moderation/security-hidden content non-restorable by owners.

## Reproduction Assessment

Yes, as a source-level gap: owner settings renders summary, publisher note, rename, and merge only, while dashboard owner queries use active indexes that exclude soft-deleted rows. I did not run a live browser flow.

## Solution Assessment

Unclear until maintainers decide the web self-service policy. If approved, reusing setSkillSoftDeletedInternal behind a narrow authenticated web mutation is the maintainable path; hard-delete or a second deletion contract would be riskier.
