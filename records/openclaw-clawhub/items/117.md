## Summary

Keep this issue open. Current main supports owner soft-delete/restore through the backend, HTTP API, CLI, docs, and e2e coverage, but the signed-in web owner settings/dashboard still lack delete/restore controls and the owner dashboard query filters deleted skills before owners could restore them.

## What This Changes

Authors want a way to delete their own published ClawHub skills.

## Best Possible Solution

Add a signed-in owner web delete/restore workflow that reuses the existing soft-delete contract, derives identity server-side, confirms destructive actions, and gives owners a way to find and restore soft-deleted skills.

## Reproduction Assessment

Yes, by source inspection: owner settings render rename/merge only, dashboard row actions render Settings/rescan only, and owner dashboard queries filter softDeletedAt rows out. I did not run the app because this review was read-only.

## Solution Assessment

Yes, the maintainable direction is to add authenticated web UI/query/mutation support on top of the existing soft-delete primitive. Hard-delete or client-supplied actor identity would be the wrong fix boundary.
