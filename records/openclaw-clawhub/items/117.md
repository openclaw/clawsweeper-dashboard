## Summary

Keep this issue open. Current main supports owner soft-delete and restore through the HTTP API, CLI, docs, and backend tests, but the signed-in web app still lacks a discoverable skill delete/restore workflow and currently hides soft-deleted skills from the owner dashboard/settings path.

## What This Changes

Requested that ClawHub skill authors be able to delete their own published skills.

## Best Possible Solution

Add signed-in web delete and restore controls that reuse existing soft-delete/undelete semantics, expose owner-deleted skills for restoration, and cover dashboard/settings behavior with regression tests.

## Reproduction Assessment

Yes. Source inspection shows the signed-in dashboard row menu only deletes plugins, owner settings only offers rename/merge, and owner skill queries filter out soft-deleted rows before a web restore flow could find them.

## Solution Assessment

Yes. The narrow maintainable path is to reuse the existing backend/API soft-delete and undelete behavior from the web UI, rather than adding a separate deletion model or normal-owner hard delete.
