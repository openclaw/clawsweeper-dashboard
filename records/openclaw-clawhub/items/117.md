## Summary

Keep open. Current main has owner soft-delete/restore through backend, HTTP API, CLI, docs, and tests, but the signed-in web owner surfaces still do not expose a skill delete/restore workflow, and ClawHub issue policy keeps issues open for maintainer handling.

## What This Changes

Allow skill authors to delete or restore their own published ClawHub skills from the product surface.

## Best Possible Solution

Add a signed-in web owner delete/restore workflow that reuses the existing soft-delete rules, confirms destructive actions, derives identity server-side, and makes owner-hidden skills discoverable enough to restore safely.

## Reproduction Assessment

Yes. Source inspection on current main shows owner settings render rename/merge only, dashboard row actions only delete plugins, and dashboard owner queries filter soft-deleted skills before any restore UI could display them.

## Solution Assessment

Yes. The narrow maintainable solution is a web owner delete/restore flow over existing soft-delete semantics; hard delete, client-supplied actor identity, or bypassing moderation provenance would be the wrong boundary.
