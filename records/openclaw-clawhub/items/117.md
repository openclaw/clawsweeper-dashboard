## Summary

Keep this issue open. Current main supports owner soft-delete/restore through backend, HTTP API, CLI, docs, and tests, but the signed-in web app still lacks a discoverable owner delete/restore flow and the dashboard omits soft-deleted owner skills before a restore action could be shown.

## What This Changes

Owners want a self-service way to delete or restore their own published ClawHub skills.

## Best Possible Solution

Add a signed-in web owner delete/restore workflow that reuses existing soft-delete internals, exposes a safe restore surface for owner-deleted rows, and keeps non-owner moderation or security hides non-restorable by owners.

## Reproduction Assessment

Yes, as a source-level gap: owner settings renders summary, publisher note, rename, and merge only, while the dashboard query uses active-only owner indexes. I did not run a live browser flow.

## Solution Assessment

Yes, if maintainers want web self-service. Reusing setSkillSoftDeletedInternal and adding a narrow owner-visible restore surface is safer than adding hard-delete behavior or a second deletion contract.
