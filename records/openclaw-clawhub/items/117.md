## Summary

Keep this open. Current main supports owner soft-delete/restore through the backend, HTTP API, CLI, and docs, but the signed-in web app still does not expose a delete/restore workflow and owner dashboard queries still filter soft-deleted skills before a user could restore them.

## What This Changes

Skill authors should be able to delete or remove their own published ClawHub skills.

## Best Possible Solution

Add a signed-in owner web delete/restore workflow that reuses existing soft-delete semantics, derives identity server-side, confirms destructive actions, and lets owners find and restore soft-deleted skills.

## Reproduction Assessment

Yes. On current main, the owner settings panel renders rename/merge only, dashboard row actions render settings/rescan only, and api.skills.list filters softDeletedAt rows for owner scopes, so a signed-in owner has no visible web path to delete or restore a skill.

## Solution Assessment

Yes, the narrow maintainable fix is to reuse the existing soft-delete primitive behind authenticated web mutations and add owner-visible UI/query support; hard-delete or client-supplied user identity would be the wrong direction.
