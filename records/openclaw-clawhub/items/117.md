## Summary

Keep open. Current main has owner soft-delete/restore in backend, HTTP API, CLI, docs, and tests, but the signed-in web owner surfaces still do not provide a discoverable skill delete/restore workflow.

## What This Changes

The issue asks for skill authors to be able to delete their own published ClawHub skills.

## Best Possible Solution

Add a signed-in web owner delete/restore workflow that reuses the existing soft-delete semantics, confirms destructive actions, and gives owners a way to find and restore their own owner-hidden skills.

## Reproduction Assessment

Yes. Source inspection on current main shows owner settings render rename/merge only, dashboard row actions omit skill delete/restore, and owner dashboard queries filter out soft-deleted skills.

## Solution Assessment

Yes. The narrow maintainable path is to add web UI plus a server-side owner mutation over the existing soft-delete primitive; hard delete or client-supplied actor identity would be the wrong boundary.
