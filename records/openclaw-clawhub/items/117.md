## Summary

Keep this issue open. Current main has owner soft-delete/restore primitives in the API, CLI, docs, and backend, but the signed-in web app still lacks a discoverable skill delete/restore workflow and the dashboard excludes soft-deleted skills before owners can restore them.

## What This Changes

Add self-service deletion and restoration for published ClawHub skills owned by the signed-in author.

## Best Possible Solution

Add a signed-in web delete/restore flow that reuses existing soft-delete semantics, confirms destructive actions, and lets owners find their own soft-deleted skills for restore.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: sign in as a skill owner, open owner tools or dashboard row actions, and current main exposes rename/merge/settings/rescan but no skill delete/restore control while the owner dashboard filters out soft-deleted skills.

## Solution Assessment

Yes. The narrow maintainable solution is to expose the existing soft-delete/restore primitive in the web app, not to invent hard delete behavior or change the CLI/API contract.
