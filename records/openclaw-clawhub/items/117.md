## Summary

Keep this open. Current main supports owner soft-delete and restore through backend, HTTP API, CLI, and docs, but the signed-in web app still lacks a delete/restore workflow and does not surface soft-deleted owner skills for restoration.

## What This Changes

Allow skill authors to delete or restore their own published ClawHub skills.

## Best Possible Solution

Add a signed-in web delete/restore workflow that reuses existing soft-delete semantics, derives identity server-side, confirms destructive actions, and lets owners find soft-deleted skills for restore.

## Reproduction Assessment

Yes, from source inspection on current main. The owner settings panel renders only rename/merge, dashboard row actions render settings/rescan only, and owner dashboard queries exclude softDeletedAt rows before any restore UI could use them.

## Solution Assessment

Yes. The narrow maintainable path is to reuse the existing soft-delete/restore primitive behind authenticated web mutations plus owner-visible UI/query support; hard-delete or client-supplied user identity would be the wrong direction.
