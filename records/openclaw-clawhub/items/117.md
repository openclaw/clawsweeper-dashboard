## Summary

Keep this open. Current main has owner soft-delete/restore through the backend, HTTP API, CLI, and docs, but the signed-in web app still exposes only rename/merge/settings/rescan surfaces and owner dashboard queries exclude soft-deleted skills before they could be restored.

## What This Changes

Skill authors need a self-service way to delete or restore their own published ClawHub skills.

## Best Possible Solution

Add a signed-in owner web delete/restore workflow that reuses soft-delete semantics, derives identity server-side, confirms destructive actions, and gives owners a way to find and restore soft-deleted skills.

## Reproduction Assessment

Yes. Source inspection on current main shows owner settings render only rename/merge, dashboard row actions render Settings/rescan only, and owner dashboard queries filter softDeletedAt rows out via active indexes.

## Solution Assessment

Yes. The narrow maintainable solution is to add authenticated web mutations/UI/query support on top of the existing soft-delete primitive; hard-delete or client-supplied identity would be the wrong direction.
