## Summary

Keep this issue open. Current main supports owner soft-delete/restore through backend, HTTP API, CLI, docs, and tests, but the signed-in web app still lacks a discoverable owner delete/restore workflow and the dashboard filters deleted skills before owners can restore them.

## What This Changes

Owners want a signed-in self-service way to delete and restore their published ClawHub skills.

## Best Possible Solution

Add a signed-in web owner delete/restore workflow that reuses existing soft-delete semantics, includes destructive confirmation, and lets owners find owner-deleted skills for restoration.

## Reproduction Assessment

Yes. Source inspection shows an authenticated owner can reach dashboard/settings, but current main exposes only settings/rename/merge controls while owner dashboard queries filter out soft-deleted skills before restore is possible.

## Solution Assessment

Yes. The narrow maintainable fix is a web flow over the existing soft-delete/restore primitives, not hard-delete behavior or a new deletion policy.
