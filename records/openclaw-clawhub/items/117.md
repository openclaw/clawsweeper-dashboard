## Summary

Keep this issue open. Current main supports owner soft-delete/restore through the backend, HTTP API, CLI, docs, and tests, but the signed-in web app still lacks an owner delete/restore workflow and ClawHub policy does not auto-close issues during cleanup review.

## What This Changes

Request to let ClawHub skill authors delete and restore their own published skills from the product surface.

## Best Possible Solution

Add a signed-in web owner delete/restore workflow that reuses the existing soft-delete internals, derives identity server-side, exposes deleted owner skills for restore, and keeps moderation/security hides non-restorable by owners.

## Reproduction Assessment

Yes, as a source-level gap: owner settings renders summary, publisher note, rename, and merge only, while dashboard owner queries use active indexes that exclude soft-deleted rows. I did not run a live browser flow.

## Solution Assessment

Unclear until maintainers decide the web self-service policy. If they want it, reusing setSkillSoftDeletedInternal behind a narrow authenticated web mutation is the safest direction; hard-delete or a second deletion contract would be riskier.
