## Summary

Keep this issue open. Current main has owner soft-delete and restore through the HTTP API, CLI, and docs, but the signed-in web app still does not expose a discoverable owner delete/restore workflow and the dashboard cannot surface soft-deleted skills for restoration.

## What This Changes

Users want to delete or restore their own published ClawHub skills without maintainer intervention.

## Best Possible Solution

Add a signed-in owner web workflow that reuses existing soft-delete/restore semantics, confirms destructive actions, authorizes owner and publisher-member access server-side, and provides a path to restore hidden skills.

## Reproduction Assessment

Yes. Source inspection against current main reproduces the gap: owner settings render rename/merge only, dashboard row actions omit delete/restore, and owner list queries filter soft-deleted skills before owners can restore them.

## Solution Assessment

Yes, a narrow web-layer repair is the best path. The backend/API/CLI primitives already exist, so the maintainable fix is UI/query coverage and authorization alignment rather than new hard-delete behavior.
