## Summary

Keep open. Current main supports owner skill soft-delete/restore through backend, HTTP API, CLI, docs, and tests, but the signed-in web owner surfaces still lack skill delete/restore controls and the dashboard filters soft-deleted skills before an owner could restore them.

## What This Changes

Skill authors want a self-service way to delete or restore their published ClawHub skills.

## Best Possible Solution

Add a signed-in web owner delete/restore workflow that reuses existing soft-delete semantics, confirms destructive actions, and lets owners find and restore their own soft-deleted skills.

## Reproduction Assessment

Yes. Source inspection on current main shows owner settings render rename/merge only, dashboard skill row actions omit delete/restore, and owner dashboard queries exclude soft-deleted skills.

## Solution Assessment

Yes. The narrow maintainable solution is web UI/query/mutation support on top of the existing soft-delete primitive; hard delete or client-supplied actor identity would be the wrong boundary.
