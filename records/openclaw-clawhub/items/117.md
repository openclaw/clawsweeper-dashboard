## Summary

Keep this issue open. Current main supports owner soft-delete and restore through the backend, HTTP API, CLI, docs, and tests, but the signed-in web app still does not expose a discoverable owner delete/restore workflow and related user requests show the need is still active.

## What This Changes

Skill owners want a self-service way to delete their own published skills from ClawHub.

## Best Possible Solution

Add a signed-in web delete/restore workflow that reuses existing soft-delete semantics, derives identity server-side, authorizes direct owners and publisher admins consistently, exposes owner-deleted skills for restore, and keeps moderation/security hides non-restorable by owners.

## Reproduction Assessment

Yes, as a source-level gap: owner settings renders summary, publisher note, rename, and merge only, while dashboard owner queries use active indexes that exclude soft-deleted rows. I did not run a live browser flow.

## Solution Assessment

Unclear until maintainers decide the web self-service policy. If approved, reusing setSkillSoftDeletedInternal behind a narrow authenticated web mutation is the safest direction; hard-delete or a second deletion contract would be riskier.
