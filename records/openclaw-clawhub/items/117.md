## Summary

Keep this issue open. Current main supports owner skill soft-delete/restore through the API and CLI, but the signed-in web app still has no discoverable skill delete/restore control and the owner dashboard still excludes soft-deleted skills before owners can restore them.

## What This Changes

Skill authors want a self-service way to delete and restore their published skills.

## Best Possible Solution

Add a signed-in web delete/restore workflow that reuses existing soft-delete semantics, derives identity server-side, authorizes direct owners and publisher members consistently, confirms destructive actions, and gives owners a way to find and restore their own soft-deleted skills.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: sign in as a skill owner, open the skill settings or dashboard, and current main exposes rename/merge/settings/rescan but no skill delete/restore action while the dashboard query filters soft-deleted rows out.

## Solution Assessment

Yes. The narrow maintainable fix is to add web UI and a public owner mutation or wrapper around the existing soft-delete primitive, rather than inventing hard delete or changing CLI/API semantics.
