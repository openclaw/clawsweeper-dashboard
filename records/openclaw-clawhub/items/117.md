## Summary

Keep open. Current main supports owner soft-delete/restore through CLI and API paths, but the signed-in web dashboard/settings flow still does not expose skill delete/restore, and owner dashboard queries still exclude soft-deleted skills before owners can restore them.

## What This Changes

Skill authors want a self-service way to delete or remove their own published ClawHub skills.

## Best Possible Solution

Add a signed-in owner web delete/restore workflow that reuses existing soft-delete semantics, derives identity server-side, confirms destructive actions, and gives owners a way to find and restore soft-deleted skills.

## Reproduction Assessment

Yes, by current-main source inspection: owner settings render rename/merge only, dashboard skill rows do not offer deletion, and dashboard queries filter out soft-deleted skills. I did not run the app because this review is read-only.

## Solution Assessment

Yes. The maintainable path is to add authenticated web UI/query/mutation support on top of the existing soft-delete primitive; hard-delete or client-supplied actor identity would be the wrong boundary.
