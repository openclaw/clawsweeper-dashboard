## Summary

Keep this issue open. Current main has backend, HTTP API, CLI, docs, and tests for owner soft-delete/restore, but the signed-in web app still lacks a discoverable owner delete/restore workflow and dashboard queries hide soft-deleted owner skills before a restore path can be shown.

## What This Changes

The issue asks for skill authors to be able to delete their own published ClawHub skills.

## Best Possible Solution

Add a signed-in web delete/restore workflow that reuses existing soft-delete semantics, derives identity server-side, authorizes direct owners and publisher admins consistently, and gives owners a discoverable restore path.

## Reproduction Assessment

Yes, as a source-level gap: owner settings expose rename and merge but no delete/restore action, and the dashboard query excludes soft-deleted rows. I did not run a live browser flow, so the reproduction is source-based rather than runtime-executed.

## Solution Assessment

Yes, if maintainers want web self-service. Reusing setSkillSoftDeletedInternal and adding a narrow UI/query restore path is safer than adding hard-delete behavior or a second deletion contract.
