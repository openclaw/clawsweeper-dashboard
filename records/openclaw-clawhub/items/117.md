## Summary

Keep this issue open. Current main supports owner soft-delete and restore through backend, HTTP API, CLI, docs, and tests, but the signed-in web app still lacks a discoverable owner delete/restore workflow and the dashboard only queries active skills.

## What This Changes

The issue asks for skill authors to be able to delete their own published ClawHub skills.

## Best Possible Solution

Add a signed-in web owner delete/restore workflow over the existing soft-delete primitive, with destructive confirmation and a safe way for owners to find their own owner-deleted skills.

## Reproduction Assessment

Yes. Source inspection shows an owner can reach dashboard/settings, but current main exposes settings/rename/merge only while dashboard skill queries filter soft-deleted rows before restore is possible.

## Solution Assessment

Yes. The narrow maintainable solution is a web flow over the existing soft-delete/restore semantics, not hard delete behavior or a new deletion policy.
