## Summary

Keep this issue open. Current main has backend/API/CLI owner soft-delete and restore support, but the signed-in web app still lacks a discoverable owner delete/restore workflow and the dashboard filters owner soft-deleted skills out before they can be restored.

## What This Changes

The issue requests that skill authors be able to delete their own published skills from ClawHub.

## Best Possible Solution

Add a signed-in web delete/restore workflow that reuses existing soft-delete semantics, derives identity server-side, covers direct owners and publisher admins consistently, and lets owners find and restore their own owner-deleted skills.

## Reproduction Assessment

Yes, from source inspection. The backend/API/CLI/docs support owner soft-delete and restore, while the web owner settings and dashboard paths do not expose delete/restore controls or a restoration view.

## Solution Assessment

Yes. A focused web UI/query change around the existing soft-delete helper is the narrowest maintainable path; adding hard-delete behavior or changing the API/CLI contract would be riskier.
