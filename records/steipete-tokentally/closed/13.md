Keep open: the PR fixes the live OpenRouter string-pricing bug and includes real behavior proof, but it still changes the exported numeric-catalog behavior from USD per 1M tokens to USD per token.

This PR changes the OpenRouter catalog converter to accept string or numeric per-token prices, switch to `pricingFromUsdPerToken`, and expand OpenRouter tests for string, numeric, zero, and malformed pricing.

## Best Possible Solution

Accept the string/per-token OpenRouter fix while preserving the released numeric/per-million behavior for hand-built numeric catalogs, with tests covering both paths.

## Reproduction Assessment

Yes. Source inspection plus a live endpoint check show current main only accepts numeric prices while OpenRouter currently returns string price fields, so `fetchOpenRouterPricingMap` will skip real priced entries.

## Solution Assessment

No as-is. The string parsing direction is right, but the patch should preserve numeric per-million compatibility unless maintainers explicitly choose and document a breaking API change.
