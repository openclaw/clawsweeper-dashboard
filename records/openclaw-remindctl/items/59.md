## Summary

Keep this open: current main and v0.3.1 contain the intended `--url` write path, but the issue is a fresh report against v0.3.1 with screenshots showing the Reminders app URL field empty, and a separate post-merge comment on the related PR reports the same behavior on macOS 15.7.5. The remaining problem is not whether the flag exists; it is whether the EventKit URL write actually reaches the app-visible URL field across supported macOS releases.

## What This Changes

The issue reports that `remindctl add --url` accepts a URL flag but the created macOS Reminder does not show that URL in the Reminders app.

## Best Possible Solution

Run a live macOS reproduction that checks both Reminders.app and `remindctl info --json`; if JSON has the URL but the app field stays blank, maintainers should decide whether `--url` must target the app-visible field or be documented as EventKit-only, and if JSON also lacks it, fix the write path.

## Reproduction Assessment

Yes for triage, but not self-reproduced here: the issue gives exact v0.3.1/macOS 26.4.1 steps with screenshots, and a separate post-merge PR comment reports the same result on macOS 15.7.5. I could not execute the macOS EventKit path in this environment, and source inspection alone does not explain the failure.

## Solution Assessment

No, closing as implemented is not the best outcome because the reported release already contains the current implementation. The narrow next step is a live runtime comparison of Reminders.app versus `info --json`, followed by either a write-path fix or a maintainer decision about the app-visible URL contract.
