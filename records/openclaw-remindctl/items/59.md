## Summary

Keep open: current main and v0.3.1 contain the intended CLI-to-EventKit `--url` write/read path, but the issue reports that released behavior still leaves the Reminders.app URL field empty, and a separate post-merge comment on the URL-support PR reports the same result on macOS 15.7.5. There is no post-release code fix on current main, so the remaining question needs live macOS/EventKit verification rather than cleanup closure.

## What This Changes

The issue reports that `remindctl add ... --url ...` accepts a URL but the created reminder does not show that URL in the macOS Reminders app.

## Best Possible Solution

Run a live macOS reproduction that compares the created reminder in Reminders.app with `remindctl add --json` and `remindctl info --json`; then either fix the write path if EventKit readback loses the URL, or make a maintainer decision about whether `--url` must target the app-visible field rather than only `EKReminder.url`.

## Reproduction Assessment

Yes for triage, but not self-reproduced here: the issue gives exact v0.3.1/macOS 26.4.1 steps with screenshots, and a separate post-merge PR comment reports the same result on macOS 15.7.5. I could not execute the macOS EventKit path in this environment.
