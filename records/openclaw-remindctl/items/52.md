## Summary

Keep open, but the patch should not merge as-is: it repeats the title-hashtag approach that a collaborator closed as misleading for native Reminders tags, while current main still documents native tags and smart lists as outside EventKit-backed support.

## What This Changes

The PR adds title-hashtag tag parsing, tag-related add/edit/show commands, a tags subcommand, JSON tag fields, and helper tests.

## Best Possible Solution

Keep native tags and smart lists unsupported under the current EventKit backend; if maintainers want tag-like workflows, implement a clearly named notes-prefix text parser/filter with docs and live CLI proof.

## Reproduction Assessment

Yes. Source inspection provides a high-confidence path: `add --tag` composes tags into the reminder title, and the EventKit store writes only `EKReminder.title`, so the PR cannot create native Reminders tags.

## Solution Assessment

No. The title-hashtag implementation is not the best solution because the same approach was rejected as misleading; a narrower notes-prefix parser/filter or explicit non-EventKit backend decision would be safer.
