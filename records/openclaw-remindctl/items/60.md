## Summary

Keep this PR open as a candidate fix for the linked app-visible URL issue, but the patch is not merge-ready because the mirror helper can delete or replace pre-existing authored `URL: ...` note lines that were never generated by remindctl.

## What This Changes

The branch mirrors a reminder's EventKit URL into notes on add/edit, updates URL help/docs, moves `isAllDay` into a new helper file, and adds unit tests for the note-mirror helper.

## Best Possible Solution

Preserve authored notes during URL replace/clear, document the approved mirror contract, and keep the EventKit URL property as the source of truth.

## Reproduction Assessment

Yes for the blocking patch defect from source: `apply(notes: "URL: old", showing: nil, replacing: old)` removes the matching note line without knowing who authored it. The linked app-visible URL issue has contributor live proof in the PR body, but this review environment did not run EventKit.

## Solution Assessment

No. Mirroring may be viable, but this implementation removes exact `URL: <previousURL>` note lines without proof they were generated by remindctl, so the safer solution must preserve authored notes or make the mirror contract explicit and migration-safe.
