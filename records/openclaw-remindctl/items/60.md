## What This Changes

The branch mirrors EventKit reminder URLs into a managed `remindctl URL: ...` note line on add/edit/clear, updates CLI/docs text, and adds mirror helper tests.

## Best Possible Solution

Implement the notes mirror with exact preservation of every non-managed note character, then have maintainers explicitly accept default `remindctl URL: ...` note mutation as the Reminders.app visibility contract.

## Reproduction Assessment

Yes for the PR defect by source inspection: `ReminderURLNoteMirror.apply` trims the remaining note text after removing candidate mirror lines, so URL replace/clear can rewrite non-managed whitespace. I did not run macOS EventKit locally.

## Solution Assessment

No: a managed note line is plausible, but this implementation still normalizes authored note text during URL operations. The safer path is exact preservation of non-managed text plus maintainer approval for the default notes-line contract.
