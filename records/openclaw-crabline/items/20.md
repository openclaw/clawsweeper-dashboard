## Summary

Keep open: this maintainer-authored PR adds a useful WhatsApp runtime factory, but the recorder replay path can emit a different inbound message ID than the fake server generated for the same admin event, so it should not merge as-is.

## What This Changes

The PR adds an env-driven WhatsApp Baileys runtime socket factory package subpath, exports its helpers and types, passes the recorder path through manifest env, and documents/tests runtime inbound replay.

## Best Possible Solution

Keep the runtime factory, but replay the fake server's canonical inbound message data from the recorder and add a regression asserting runtime and server message IDs match.
