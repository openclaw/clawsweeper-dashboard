## What This Changes

## Best Possible Solution

## Reproduction Assessment

Yes, at source level. On current main, subscribing to the WhatsApp Baileys mock socket and POSTing admin inbound would only return JSON while socket events are emitted from outbound sendMessage, and unauthenticated inbound is accepted before auth.

## Solution Assessment

Yes, with maintainer acceptance of the contract. The patch authenticates admin ingress before parsing/recording, keeps rejected inbound out of the recorder, and emits accepted inbound to subscribed sockets; the remaining question is whether that token/listener boundary is the intended OpenClaw integration shape.
