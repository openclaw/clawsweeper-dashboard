## Best Possible Solution

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with current Bun-style `argv[1]` such as `/$bunfs/root/polter-arm64`, the new regex rejects the already-present script path and prepends `/polter`, leaving Commander to parse the virtual slice path as the target.

## Solution Assessment

No. The helper is the right place to solve this, but the current predicate is too narrow; it should cover or normalize the suffixed Homebrew slice names and test them directly.
