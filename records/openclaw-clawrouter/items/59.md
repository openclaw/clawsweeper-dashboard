## Summary

Keep open: current main has scoped upstream grants, policy budget ledgers, provider readiness, and one upstream attempt per request, but it does not implement quota-aware grant pools, live per-grant quota state, cooldowns, selection policies, or same-provider failover. The linked proxy is useful prior art, but this needs maintainer product/design direction before implementation.

## What This Changes

The issue requests quota-aware upstream grant pools with live provider quota tracking, selection policies, cooldowns, same-provider failover, and admin/API visibility.

## Best Possible Solution

A maintainer-sponsored design should define the provider-neutral grant-pool/quota-state model, storage, selection semantics, stale-state behavior, and admin/API surfaces while keeping budget preflight authoritative and provider-specific collection isolated to manifests or focused adapters.
