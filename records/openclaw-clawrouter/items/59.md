## Summary

Keep open: current main still has static scoped upstream grants, policy budget ledgers, and one upstream attempt per request rather than quota-aware grant pools, live per-grant provider quota state, or same-provider failover.

## What This Changes

The issue requests quota-aware upstream grant pools, live provider quota tracking, configurable same-provider grant selection, failover, cooldowns, and admin/API visibility.

## Best Possible Solution

A maintainer-sponsored design should add a provider-neutral grant-pool and quota-state layer, keep hard budget preflight authoritative, and isolate provider-specific quota collection to manifests or focused adapters.
