## Summary

Keep open: current main has scoped upstream grants, static policy budget ledgers, and provider readiness/health, but it still resolves one fixed policy-or-tenant grant and performs one upstream attempt per request. The linked proxy is useful prior art, but quota-aware grant pools, live per-grant quota state, and same-provider failover need maintainer product/design direction before implementation.

## What This Changes

Requested support for quota-aware upstream grant pools, live provider quota tracking, and failover across same-provider grants when one grant is exhausted or unhealthy.

## Best Possible Solution

A maintainer-sponsored design should define the provider-neutral grant-pool/quota-state model, storage, selection semantics, stale-state behavior, and admin/API surfaces while preserving authoritative budget preflight and isolating provider-specific collection to manifests or focused adapters.
