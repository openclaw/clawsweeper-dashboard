## Summary

Keep open: current main has scoped upstream grants, provider readiness, and policy budget ledgers, but not live per-grant quota state, grant-pool selection, cooldowns, or same-provider failover. The linked LLM API Key Proxy is useful prior art, but this needs maintainer product/design direction before implementation.

## What This Changes

Request quota-aware upstream grant pools with live per-grant provider quota tracking, selection policies, cooldowns, admin visibility, and same-provider failover.

## Best Possible Solution

A maintainer-sponsored design should define the provider-neutral grant-pool and quota-state model, storage, selection semantics, stale-state behavior, and admin/API surfaces while preserving authoritative budget preflight and isolating provider-specific collection to manifests or focused adapters.

## Reproduction Assessment

No; reproduction is not applicable because this is a new quota-aware grant-pool capability request, not broken current behavior. Source inspection does confirm current main lacks pool selection and same-provider failover.

## Solution Assessment

Unclear; the linked proxy is useful prior art, but copying that behavior directly is not yet the narrowest maintainable ClawRouter solution. The best path needs provider-neutral quota freshness, selection, storage, and adapter boundaries decided first.
