## Summary

Keep open: current main has scoped upstream grants, provider readiness, provider health, and policy budget ledgers, but not live per-grant quota state, grant-pool selection, cooldowns, or same-provider failover. The linked LLM API Key Proxy is useful prior art, but this needs maintainer product/design direction before implementation.

## What This Changes

The issue requests quota-aware upstream grant pools, live per-grant provider quota tracking, same-provider grant failover, policy controls, and admin/API visibility.

## Best Possible Solution

A maintainer-sponsored design should define a provider-neutral grant-pool and quota-state contract with bounded provider-specific adapters, explicit stale-state semantics, and tests that keep hard policy-budget enforcement authoritative.

## Reproduction Assessment

No; reproduction is not applicable because this is a new quota-aware grant-pool capability request, not broken current behavior. Source inspection confirms current main lacks pool selection and same-provider failover.

## Solution Assessment

Unclear; the linked proxy is useful prior art, but copying that behavior directly is not yet the narrowest maintainable ClawRouter solution. The best path needs provider-neutral quota freshness, selection, storage, and adapter boundaries decided first.
