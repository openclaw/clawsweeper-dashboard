## What This Changes

This PR widens GraphQL numeric literal parsing for signed, decimal, and exponent values and adds a parser unit test for numeric `where` values.

## Best Possible Solution

Keep signed, decimal, and exponent numeric parsing for `where` values while preserving integer normalization for top-level `limit`, with regression tests for both surfaces.

## Solution Assessment

No, not as submitted. The narrow fix should preserve the existing top-level `limit` integer behavior while widening numeric parsing for `where` values.
