## Summary

## What This Changes

This PR changes `.github/workflows/upload_component.yml` to request a GitHub OIDC token for ESP Component Registry upload and removes the `IDF_COMPONENT_API_TOKEN` input.

## Best Possible Solution

Keep the OIDC direction, add `contents: read` beside `id-token: write`, and require redacted successful upload or dry-run evidence before merge.

## Reproduction Assessment

Yes for the merge blocker by source inspection: the PR adds job-level permissions with only `id-token: write`, while the job still runs checkout. The actual registry upload success is not reproduced because it requires the main-branch publish workflow and registry trust configuration.

## Solution Assessment

No as currently patched: moving to OIDC is reasonable, but the workflow should preserve checkout access and include real publish or dry-run evidence before becoming the maintained release path.
