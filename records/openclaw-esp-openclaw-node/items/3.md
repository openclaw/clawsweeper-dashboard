## Summary

Keep open: current main still sends an empty scope set, so the central scope-support feature is not already implemented, but this PR is not merge-ready because it can sign a different scope set than it transmits and it has no real ESP behavior proof.

## What This Changes

The PR adds a scope registration API and Kconfig limit, stores registered scopes, serializes them into connect requests and signed auth payloads, and updates README registration docs.

## Best Possible Solution

Define explicit scope limits and ordering, reject oversized scope sets before transport, derive the signed CSV and JSON array from one validated representation, and add boundary tests plus redacted real-device proof.

## Reproduction Assessment

Yes for the PR-introduced defect from source inspection: register scopes whose comma-separated text exceeds 127 bytes, and the PR signs only a prefix while transmitting every registered scope. The intended successful scoped-authentication behavior has not been proven in a real ESP setup.

## Solution Assessment

No in its current form; explicit scope registration may be a reasonable direction, but independent silent truncation and undefined public limits are not the narrowest maintainable solution.
