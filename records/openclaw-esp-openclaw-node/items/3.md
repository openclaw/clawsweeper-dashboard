## What This Changes

## Best Possible Solution

Define the scope contract, build signed CSV and JSON from one validated ordered representation or fail before sending, cover registry and overflow boundaries, and require redacted real-device authentication proof.

## Reproduction Assessment

Yes for the introduced defect from source inspection: register valid scopes whose comma-separated total exceeds 127 bytes, and the PR signs only a prefix while transmitting the full array. Successful intended behavior has not been reproduced in a real setup.

## Solution Assessment

No in its current form; explicit scope registration may be reasonable, but the signed and transmitted scope representations must stay identical and the public limits need maintainer confirmation.
