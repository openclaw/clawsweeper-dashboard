## Summary

Keep open: current main still signs an empty scope string and sends an empty `scopes` array, while this PR adds the missing feature but is not merge-ready because signed scopes can diverge from transmitted scopes and no real ESP/gateway proof is present.

## What This Changes

The PR adds a public scope registration API, a Kconfig scope limit, registry storage, README updates, and connect/auth payload serialization for registered scopes.

## Best Possible Solution

Define the scope limits and ordering contract, derive signature input and JSON from one validated ordered list, add boundary tests, and require redacted real-device scoped-auth proof before merge.

## Reproduction Assessment

Yes for the introduced defect by source inspection: register valid scopes whose comma-separated total exceeds 127 bytes and the PR signs only a prefix while transmitting the full array. The intended successful scoped-auth behavior has not been reproduced in a real setup.

## Solution Assessment

No in its current form; explicit scope registration may be the right direction, but signing and transmission must share one validated ordered representation and the public scope limits need maintainer confirmation.
