## Summary

This PR is still a real feature proposal rather than cleanup: current main always advertises and signs an empty scope set, while the branch adds a new public scope registry. It is not merge-ready because the proposed signed scope CSV can silently diverge from the advertised JSON scopes, and the PR includes no real behavior proof.

## What This Changes

The PR adds a configurable scope registry, `esp_openclaw_node_register_scope()`, documentation, and connect-request serialization/signing for registered scopes.

## Best Possible Solution

Land scope support only after the public API, config limit, and signed/advertised scope serialization are tested against the protocol with no silent truncation.

## Reproduction Assessment

Yes for the blocking PR defect by source inspection: on the PR branch, registering scopes whose comma-joined value exceeds the 128-byte buffer makes the signed payload omit data while the JSON still advertises all registered scopes. I did not run an ESP-IDF target test.

## Solution Assessment

No as submitted; the maintainable solution is a scope registry whose advertised JSON and signed payload are derived from the same complete representation, with explicit failure for any unsupported size or syntax.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Build the signed scope list without silent truncation:** `components/esp-openclaw-node/src/esp_openclaw_node_protocol.c:50-66`
  - body: `esp_openclaw_node_register_scope()` accepts arbitrary non-empty strings up to `ESP_OPENCLAW_NODE_MAX_SCOPES`, but `send_connect_request()` signs only the first 127 bytes and then still sends the full `params.scopes` array. A node that registers several real scopes or one long scope can produce a signature over a different scope set than it advertises, so the connect can be rejected or the authorization boundary becomes ambiguous. Allocate the CSV from the registered scopes or fail before sending when it cannot be represented.
  - confidence: 0.9
