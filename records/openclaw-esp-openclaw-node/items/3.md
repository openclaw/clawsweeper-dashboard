## Summary

Keep this PR open, but it is not merge-ready: current main still lacks scope registration, while the branch has a concrete authorization-boundary bug and no real scoped-connection proof.

## What This Changes

The branch adds a scope registry API and Kconfig limit, stores registered scopes, sends them in connect params, and uses a comma-separated scope string in the signed auth payload.

## Best Possible Solution

Define the scope API and protocol limits, then serialize one validated ordered scope set for both signature input and connect JSON, with boundary tests and redacted live authentication proof.

## Reproduction Assessment

Yes for the introduced defect: register enough non-empty scopes so the comma-separated list exceeds 127 bytes; source shows the loop truncates the signed CSV while later JSON sends the full registry. The intended success path has not been proven live.

## Solution Assessment

No in its current form; explicit scope registration may be reasonable, but the implementation must use one checked representation for signing and transmission and needs maintainer-confirmed limits.
