## Summary

Current main does not support registered scopes, so this PR remains useful, but it should not merge yet because scope overflow makes the signed authorization set differ from the transmitted JSON and no real-device scoped-authentication proof is provided.

## What This Changes

The PR adds configurable scope registration and advertises registered scopes in both connect JSON and the signed authentication payload.

## Best Possible Solution

Define the permanent scope limits and ordering contract, then use one checked ordered representation for both signing and transmission, with boundary tests and redacted real-device authentication proof.

## Reproduction Assessment

Yes for the introduced defect: register valid scopes whose comma-separated total exceeds 127 bytes, and the source path signs only a prefix while transmitting the complete array. Successful intended behavior has not been reproduced in a real setup.

## Solution Assessment

No in its current form; explicit scope registration is reasonable, but silent independent truncation is unsafe and the public protocol limits require maintainer approval.
