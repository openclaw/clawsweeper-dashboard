## Summary

Keep open: current main still lacks scope registration and always sends empty scopes, but this PR is not merge-ready because the proposed auth signing can diverge from the transmitted scopes and no real ESP/gateway behavior proof is present.

## What This Changes

The branch adds `esp_openclaw_node_register_scope()`, a scope registry/config limit, and sends registered scopes in the connect signature material and `params.scopes`.

## Best Possible Solution

Keep the scope feature behind one consistent representation: the registered scope list should be fully represented in both the signed auth payload and `params.scopes`, with explicit failure on allocation/validation errors and focused tests around long scope lists.

## Reproduction Assessment

Yes for the PR-introduced defect from source inspection: register scopes whose comma-separated text exceeds 127 bytes and the PR signs only a prefix while transmitting every registered scope. The intended successful scoped-auth behavior has not been proven in a real ESP setup.

## Solution Assessment

No in its current form; explicit scope registration may be a reasonable direction, but silent fixed-buffer truncation and an unproven public API/config shape are not the narrowest maintainable solution.
