## Summary

The requested scope capability is still absent from current main and this focused PR is the active implementation path, but it should not merge yet because valid registered scopes can exceed the fixed signing buffer and produce an authentication scope mismatch; required real-device or live-gateway proof is also absent.

## What This Changes

The PR adds a configurable scope registry and public registration API, then advertises registered scopes in connect JSON and includes them in the signed authentication payload.

## Best Possible Solution

Use complete deterministic serialization of the ordered registered scopes for signing, reject the connection before transmission if that representation cannot be created, cover boundary cases, and demonstrate a successful scoped connection against a live gateway.

## Reproduction Assessment

Yes for the introduced defect: register valid scope strings totaling more than 127 CSV bytes, and the signing loop omits later entries while the JSON array retains them. Successful intended behavior has not been reproduced in a live setup.

## Solution Assessment

No in its current form; explicit scope registration is a reasonable component direction, but the signed representation must be complete and deterministic rather than independently truncated.
