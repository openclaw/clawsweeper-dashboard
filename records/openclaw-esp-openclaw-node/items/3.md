## Summary

Keep open: current main still sends an empty scope set, so this PR addresses a real missing feature, but it cannot merge as-is because the patch can sign a truncated scope set while transmitting all registered scopes and it lacks real behavior proof.

## What This Changes

Adds a public ESP OpenClaw Node scope registration API, Kconfig limit, registry storage/cleanup, connect-request scope serialization/signing, and README docs.

## Best Possible Solution

Land scope support only after the PR uses a single validated scope representation for both signing and JSON, adds focused regression coverage, and includes redacted real ESP/gateway proof.

## Reproduction Assessment

Yes for the PR-introduced defect from source inspection: register scopes whose comma-separated text exceeds 127 bytes, then connect; the PR signs only the prefix it fit into `scopes_csv` while still sending every registered scope in JSON. The intended successful scoped-authentication behavior has not been proven in a real ESP setup.

## Solution Assessment

No in its current form; explicit scope registration may be a reasonable direction, but independent truncation and undefined public scope limits are not a maintainable authorization contract.
