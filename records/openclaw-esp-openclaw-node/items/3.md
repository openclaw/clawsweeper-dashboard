## Summary

Keep open. Current main does not implement public scope registration, and this PR adds the central feature, but the branch has a blocking auth-scope serialization bug and no real runtime proof for a scoped node connection.

## What This Changes

The PR adds configurable scope storage, `esp_openclaw_node_register_scope()`, README/Kconfig docs, and connect-request scope serialization/signing.

## Best Possible Solution

Land scope support only after signed and advertised scopes come from one complete validated representation, with explicit limits/tests and redacted runtime proof against a real gateway connect.

## Reproduction Assessment

Yes for the blocking PR defect by source inspection: the branch accepts registered scopes whose comma-joined value can exceed the fixed 128-byte signing buffer, while the JSON still advertises every registered scope. I did not run an ESP-IDF hardware or gateway integration test.

## Solution Assessment

No as submitted; the maintainable solution is to derive both the advertised JSON and signed payload from the same complete scope representation, or reject unsupported scope data before connect is sent.
