## Summary

Keep open for product judgment: current main provides encrypted Git snapshot backup and restore, but imports replace the archive from the current WhatsApp Desktop snapshot, so it does not promise independent append-only retention when upstream history disappears.

## What This Changes

Requests a durable WhatsApp backup and retention mode that preserves historical chats independently of WhatsApp Desktop and can serve as an alternative to iCloud or Google Drive backups.

## Best Possible Solution

Define a maintainer-approved archival mode with explicit retention and deletion policy, durable media handling, encrypted incremental backup, restore/export guarantees, storage controls, and upgrade tests while preserving current snapshot-mirror behavior by default.

## Reproduction Assessment

Not applicable: this requests new backup and retention guarantees rather than reporting failure of an existing documented contract. Source inspection does clearly show the current replacement-snapshot boundary motivating the request.

## Solution Assessment

No, not as a direct extension of the existing snapshot backup without a design decision. The maintainable path is an explicit opt-in archival mode with defined deletion, media, restore, privacy, and migration semantics.

## Risks / Open Questions

- An append-only archive needs explicit rules for intentional upstream deletions, privacy requests, data expiration, deduplication, and storage growth.
- Independent retention of media requires durable copying and backup semantics because default media paths can still reference WhatsApp Desktop's app container.
- Git history may retain older encrypted snapshots, but reconstructing deleted messages from repository history is not a documented or tested restore guarantee.
