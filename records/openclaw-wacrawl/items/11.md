## Label Justifications

## Summary

Keep open for product direction: current main and v0.2.7 provide encrypted backup of the current archive snapshot, but every sync fully replaces archived rows, so neither preserves messages that have disappeared from WhatsApp Desktop. No other issue or PR found in the canonical search owns the remaining durable-retention work.

## What This Changes

Requests an independent, durable WhatsApp chat backup and retention mode that does not lose history when WhatsApp Desktop removes or stops storing messages.

## Best Possible Solution

Add an explicit durable-history mode that merges newly observed records into a retained archive, with documented deletion policy, copied-media handling, integrity verification, snapshot recovery, and an opt-in default that preserves current replacement behavior.

## Reproduction Assessment

Not applicable: source, documentation, and release inspection establish that this requests a new durable-retention guarantee rather than reporting broken existing behavior.

## Solution Assessment

Yes in direction, but not as an unconditional sync change; an explicit retention mode is safer because append-only storage changes deletion, privacy, and upgrade expectations.
