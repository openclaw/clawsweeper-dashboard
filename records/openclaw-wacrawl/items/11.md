## Summary

Keep open for maintainer product direction: current main and v0.2.7 can encrypt and restore the current wacrawl archive snapshot, but sync/import still replace archive rows from WhatsApp Desktop, so disappeared upstream messages are not durably retained. I found no canonical issue or viable PR owning the remaining retention work.

## What This Changes

Request to make wacrawl a durable WhatsApp backup alternative that preserves chat history beyond WhatsApp Desktop, iCloud, or Google Drive retention.

## Best Possible Solution

Add an explicit durable-history mode that merges newly observed records into a retained archive, with documented deletion policy, copied-media behavior, integrity checks, and backup/restore semantics while preserving today’s snapshot replacement default unless maintainers choose otherwise.

## Reproduction Assessment

Not applicable: this is a feature request, and source/docs inspection shows the missing durable-retention guarantee rather than broken existing documented behavior.

## Solution Assessment

Yes in direction, but not as an unconditional sync change; an explicit retention mode is safer because preserving deleted messages changes privacy, deletion, and upgrade expectations.
