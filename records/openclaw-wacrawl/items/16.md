## Best Possible Solution

Keep the narrow import-time normalization so invalid WhatsApp sentinel timestamps never reach stored CLI JSON output.

## Solution Assessment

Yes; clamping at `appleNullTime` is the narrowest fix because it is the shared conversion path for imported nullable timestamps and leaves valid Apple-epoch timestamps unchanged.
