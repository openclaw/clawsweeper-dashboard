## Summary

Keep open: current main and the latest release still save the backup config with a direct final-path write, and the open linked fix PR has not merged yet.

## What This Changes

The issue reports that backup config persistence rewrites backup.json non-atomically, risking a truncated config and lost age recipients after an interrupted save.

## Best Possible Solution

Review and land https://github.com/openclaw/wacrawl/pull/25 or an equivalent focused atomic SaveConfig implementation, then let the merge close this issue.

## Reproduction Assessment

No live high-confidence torn-write reproduction was established. The issue is source-reproducible: backup init reaches SaveConfig, current main writes backup.json directly with os.WriteFile, and LoadConfig hard-fails invalid JSON.

## Solution Assessment

Yes. The linked PR's same-directory temp write, chmod, sync, rename, parent directory sync, and failure-path tests are the narrow maintainable fix without adding product surface.
