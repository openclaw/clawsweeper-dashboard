## Summary

Keep open: current main and v0.3.0 still rewrite backup.json directly, and the linked open pull request is a candidate fix rather than a merged implementation.

## What This Changes

The issue reports that backup config saves rewrite backup.json in place, so an interrupted backup init can truncate the authoritative config and lose age recipients.

## Best Possible Solution

Land https://github.com/openclaw/wacrawl/pull/25 or an equivalent focused atomic SaveConfig fix, then let this issue close from the merge.

## Reproduction Assessment

Yes for source-level reproduction: current main writes backup.json through os.WriteFile from backup init, and LoadConfig hard-fails on truncated JSON. No live torn-write harness was run because the interruption window is timing dependent.

## Solution Assessment

Yes. A same-directory temporary file plus chmod, sync, rename, and focused failure-path tests is the narrow maintainable fix, and the linked pull request already implements that shape.
