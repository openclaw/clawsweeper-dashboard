## Summary

Keep open: current main and v0.3.0 still save backup.json through a direct os.WriteFile path, and the linked open pull request is a candidate fix rather than a merged implementation.

## What This Changes

The issue requests making backup config saves atomic so interrupted backup initialization cannot truncate backup.json or lose age recipients.

## Best Possible Solution

Land https://github.com/openclaw/wacrawl/pull/25 or an equivalent focused atomic SaveConfig fix, then let this issue close through the merged closing reference.

## Reproduction Assessment

Yes at source level: backup init reaches SaveConfig, SaveConfig writes backup.json directly with os.WriteFile, and LoadConfig hard-fails on invalid JSON. I did not establish a live SIGKILL torn-write reproduction.

## Solution Assessment

Yes. The narrow maintainable fix is atomic same-directory temp-file replacement with permissions, sync, cleanup, and failure-path coverage, as proposed by the linked PR.
