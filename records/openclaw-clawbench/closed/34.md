## What This Changes

## Best Possible Solution

Preserve independent Testbox sessions unless maintainers explicitly approve one active session per ref; otherwise key cancellation on a real supersession/session identity and add a focused workflow guard.

## Reproduction Assessment

Yes from source inspection: the workflow is manually dispatched with a per-session `testbox_id`, while the PR groups cancellation only by `github.ref`. A live Blacksmith run was not needed to see that two documented `main` dispatches would share one concurrency group.

## Solution Assessment

No, not as-is: a ref-only key is too broad for a session-oriented `workflow_dispatch` workflow. The narrow maintainable fix is an explicit supersession/session key, or a maintainer-approved policy that one active Testbox per ref is intentional.
