## Summary

Keep open: current main still renders the execution-check command before calling `shlex.split`, so a whitespace-bearing template value is split into multiple argv entries in the `shell=False` path.

## What This Changes

The issue asks for `run_execution_check` to preserve each template-rendered runtime value as one subprocess argument when `shell=False`.

## Best Possible Solution

Tokenize the unrendered command into argv parts for `shell=False`, render each token independently, keep `shell=True` unchanged, and add whitespace-placeholder regressions for both verifier entry points or a shared helper.

## Reproduction Assessment

Yes. Current main has the exact render-then-split sequence, and a read-only check of `render_template` plus `shlex.split` reproduced the reported four-token argv for `report 2026.json`.

## Solution Assessment

Yes. Splitting the raw template before rendering each argv token is the narrowest maintainable fix for the `shell=False` contract, with the duplicate legacy verifier kept in sync or deduplicated.
