## Summary

Keep this PR open but do not merge its current head: the proposed estimator misattributes general forbidden violations and mishandles right-censored clean runs, so it can generate plausible-looking but invalid hazard, survival, and mutual-information reports. The PR also lacks the required output from a real archive run.

## What This Changes

The PR adds violation-time decomposition to the posterior dynamics pipeline, includes model and task metadata in regime outputs, and configures pytest’s repository import path.

## Best Possible Solution

Model event occurrence and right-censoring explicitly using authoritative per-turn grader evidence, validate the estimator against synthetic known results, and enable it in the default pipeline only after a real archive run produces reviewed JSON, report, and plot artifacts.

## Reproduction Assessment

Yes at source level. A non-shell forbidden violation is assigned to the final assistant turn, while a short clean run stops contributing to survival later in the default reporting horizon; focused inputs can demonstrate both without a full benchmark execution.

## Solution Assessment

No. The current implementation conflates the general forbidden-violation result with dangerous-shell detection and does not represent right-censoring correctly; explicit event and censor data is the maintainable foundation.
