## Summary

The proposed decomposition remains unique work, but it should not merge yet because it invents event times for non-shell violations, treats censoring as declining survival, and has no real archive proof; current `main` only contains the broader dynamics foundation.

## What This Changes

The PR adds a mandatory violation-time decomposition stage with JSON, plot, and Markdown outputs, carries explicit model/task metadata through regime reports, adjusts downstream parsing, and configures pytest imports.

## Best Possible Solution

Use violation-type-aware event localization and a standard right-censored survival estimator, cover both with focused fixtures, then demonstrate the full mandatory stage on representative real archives.

## Reproduction Assessment

Yes at source level: a forbidden-tool-only run follows the final-turn fallback, and a short censored run lowers survival after observation ends. No real pipeline execution was performed because the review is read-only and the contributor supplied no runtime artifact.

## Solution Assessment

No; time-to-event decomposition is a reasonable direction, but the current estimator and unconditional integration are not yet a reliable implementation of it.
