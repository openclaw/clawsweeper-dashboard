## Summary

Keep open: current main still renders shell-mode runtime placeholders with plain substitution, and this collaborator-authored draft PR is a focused fix for the linked shell quoting bug. I found no discrete blocking code finding, but the shell placeholder compatibility choice needs maintainer acceptance before merge.

## What This Changes

The PR adds quote-aware shell placeholder rendering, uses it for shell-mode execution checks and background service commands, and adds regression coverage for whitespace, metacharacters, quoted placeholders, and services.

## Best Possible Solution

Land a quote-aware literal-data renderer for shell placeholders with regression tests once maintainers confirm that runtime values are data, not supported shell fragments.

## Reproduction Assessment

Yes. Source inspection shows current main substitutes runtime values with render_template and then passes shell=True commands to the shell, so a command like cat {output_path} with a whitespace path reproduces the reported tokenization failure.

## Solution Assessment

Yes, with one maintainer decision. The PR takes the narrowest maintainable implementation path for data placeholders, but maintainers should explicitly accept the compatibility change for templates that previously used placeholders as shell fragments.
