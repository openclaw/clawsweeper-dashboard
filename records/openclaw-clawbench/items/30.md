## Best Possible Solution

## Reproduction Assessment

Yes, from source inspection rather than an executed repro: current main renders shell=True commands with render_template and then passes them to /bin/sh, so whitespace and metacharacters in placeholder values are interpreted by the shell.

## Solution Assessment

Unclear pending maintainer decision: quote-aware rendering is the narrow fix for paths and metacharacters as data, but making that the unconditional default can break templates that intentionally used shell expansion.
