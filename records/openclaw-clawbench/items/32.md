## What This Changes

The PR moves generated ClawBench run workspaces from `$OPENCLAW_STATE_DIR/workspace/clawbench/<task>` to `$OPENCLAW_STATE_DIR/workspace-clawbench/<task>`.

## Best Possible Solution

Keep benchmark agents under the dedicated sibling `workspace-clawbench` root, with a focused path-invariant test if maintainers want this isolation boundary protected against regression.

## Reproduction Assessment

Yes, from current-main source: `_create_run_workspace` places benchmark directories under the default `workspace` tree, and `_create_run_agent` registers that nested path as the run agent workspace. The review did not mutate the checkout to execute a fresh run.

## Solution Assessment

Yes. Moving only the generated workspace root to a sibling directory directly removes the nesting while preserving agent creation, task setup, and cleanup semantics.
