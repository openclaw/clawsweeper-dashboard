## Summary

This PR should stay open: it is a repository-member workflow change that is not already on main, and the current diff has a blocking automation concern because same-ref cancellation would also cancel independent Testbox sessions.

## What This Changes

The PR adds a top-level concurrency group with `cancel-in-progress: true` to `.github/workflows/ci-check-testbox.yml`.

## Best Possible Solution

Preserve independent Testbox sessions unless maintainers explicitly approve a one-active-session-per-ref policy; if cancellation is still desired, key it on an explicit supersession/session identity rather than only `github.ref`.

## Reproduction Assessment

Yes from source inspection: the workflow is manually dispatched with a per-session `testbox_id`, while the PR groups cancellation only by `github.ref`. A live Blacksmith run was not needed to see that two documented `main` dispatches would share one concurrency group.

## Solution Assessment

No, not as-is: a ref-only key is too broad for a session-oriented `workflow_dispatch` workflow. The narrow maintainable fix is an explicit supersession key, or a maintainer-approved policy that one active Testbox per ref is intentional.
