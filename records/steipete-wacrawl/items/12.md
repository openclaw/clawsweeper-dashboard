## Summary

Keep open. The PR adds a useful contact-export feature and the body now includes sufficient redacted real-behavior proof, but the default command path still relies on the existing message-only auto-sync freshness check, so contact-only WhatsApp changes can be skipped and stale contacts exported.

## What This Changes

The PR adds a `contacts export` CLI path, exposes it as crawlkit `contact-export` metadata, adds a public store contacts accessor, and adds CLI tests for the JSON contact export contract.

## Best Possible Solution

Land the feature after auto-sync freshness accounts for contact-only source changes while keeping the metadata command archive-only and preserving the minimal `display_name`/`phone_numbers` export contract.

## Reproduction Assessment

Yes from source inspection, not a live run: the new command calls `withArchiveStore`, while current `sourceAheadOfArchive` ignores contact counts and contact update state. A stale archive with unchanged messages but changed contacts can therefore skip import under default `--sync auto`.

## Solution Assessment

No, not yet. Reusing the archive sync wrapper is the right direction, but the freshness predicate needs contact-aware logic and regression coverage before this is the narrowest maintainable solution.
