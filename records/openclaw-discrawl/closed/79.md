## What This Changes

## Best Possible Solution

Land the sqlite-only publish path with help/docs that state it refreshes only the R2 SQLite bundle and assumes D1 is already hydrated.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Partly from source: current main always sends D1 ingest rows before uploading the SQLite bundle, and the PR adds a focused server test for the sqlite-only path. I did not reproduce the live D1 `SQLITE_NOMEM` condition.

## Solution Assessment

Is this the best way to solve the issue? Yes, with one small follow-up: an opt-in sqlite-only mode is a narrow way to refresh R2 without changing existing publish defaults, but the flag should be documented in help/docs before merge.
