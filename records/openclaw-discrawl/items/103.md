## Summary

Keep open: current main has source-specific sync and public-only snapshot filtering, but it does not provide the requested source-provenance diagnostics, publish-scope preflight, or `publish --public-only --check` dry run. The related umbrella at https://github.com/openclaw/discrawl/issues/100 is adjacent context, not a canonical replacement for this narrower work.

## What This Changes

Add diagnostics that report archive source/provenance and validate whether public-only publishing has enough metadata before writing a snapshot.

## Best Possible Solution

Factor one read-only publish-scope analyzer and expose it through the maintainer-approved CLI surface so users can see source provenance, metadata completeness, expected public export counts, and a concrete `discrawl sync --source discord` repair recommendation before writing snapshots.
