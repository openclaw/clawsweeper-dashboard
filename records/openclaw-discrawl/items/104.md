## Summary

Keep open: current main has channel list/show and id/name/partial query filters, but it does not provide a channel resolver command, ambiguity diagnostics with candidate ids, or the requested repeatable-workflow docs, and no canonical PR owns the work.

## What This Changes

The issue requests better Discrawl channel name-to-id resolution, ambiguity diagnostics, JSON-selected channel metadata, and docs recommending numeric ids for repeatable workflows.

## Best Possible Solution

Add a shared channel resolver used by `channels`, `search`, and `messages` that preserves numeric id behavior, reports ambiguous candidates with ids, exposes selected channel metadata in JSON where practical, and documents numeric ids for repeatable workflows.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this is a feature request for new resolver and ambiguity behavior. Source inspection confirms the requested resolver command and candidate-id ambiguity diagnostics are missing on current main.

## Solution Assessment

Unclear until maintainers settle the CLI contract. A shared resolver is the maintainable direction, but the final command, flag, JSON shape, and ambiguity policy are product/API choices.
