## Summary

Keep open: current main has per-channel retry/unavailable markers and per-attachment media fetch errors, but it does not have a durable, queryable sync/import/write failure ledger or row-context wrapping for write failures.

## What This Changes

Request durable JSON-visible sync/import/write failure records with row-level Discord identifiers and resolution state.

## Best Possible Solution

Design a narrow failure ledger plus JSON read surface, then wire sync, wiretap, snapshot/media/embedding, and store write paths to record and resolve row-scoped failures before changing ingestion behavior.

## Reproduction Assessment

Yes for the missing requested surface: current-main source inspection shows no failure ledger, failures command, or doctor recent-failures output. No for the specific duplicate attachment insert incident, because the issue does not include a runnable fixture or log.

## Solution Assessment

Unclear until maintainers choose the persistent schema and lifecycle semantics. The likely best path is a small durable ledger and JSON query surface that wraps existing row-level errors without broad behavior changes first.
