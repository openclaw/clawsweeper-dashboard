## Summary

Keep open: current main still lacks a public process, lock-owner, and read-only integrity diagnostic surface. The code has status JSON with WAL bytes, internal sync-lock metadata, and read-only SQL primitives, but `status`/`doctor` do not expose the requested flags or active-process checks, and no open PR currently implements them.

## What This Changes

Request adds JSON-capable process, SQLite lock, WAL, and read-only integrity diagnostics to `doctor` or `status` so maintainers can tell whether the archive is still being written.

## Best Possible Solution

Add a maintainer-approved diagnostic surface that reports DB path, WAL bytes, sync-lock metadata, best-effort writer/importer process state, read-only `quick_check`, and the safety distinction between archive freshness and active writers.

## Reproduction Assessment

Not applicable: this is a new diagnostic feature request, not a broken existing contract. Source inspection confirms the current `status`, `doctor`, and `sql` surfaces do not expose the requested public diagnostics.

## Solution Assessment
