## Summary

Keep open. Current main and v0.37.2 already have docs, prompt suppression, and a global Disable Keychain access toggle, but the in-app pre-alert is still OK-only and does not provide a learn-more or opt-out path before the macOS Keychain prompt.

## What This Changes

New users want clearer in-app context, learn-more/opt-out affordances, and less surprising timing before CodexBar triggers macOS Keychain/password prompts.

## Best Possible Solution

Add a narrow, maintainer-approved trust affordance to the existing Keychain pre-alert, such as Learn More plus Open Advanced Settings or Disable Keychain access, without changing provider auth, cookie import, OAuth, or Keychain storage behavior.

## Reproduction Assessment

Yes, for the current source shape: the pre-alert is an OK-only NSAlert and the opt-out exists only in Advanced settings. I did not run live Keychain-prompt validation because repository policy forbids prompt-triggering checks without explicit request.

## Solution Assessment

Yes, if scoped narrowly to copy and UI affordances around the existing pre-alert and disable toggle. A broader onboarding flow or prompt-timing change needs maintainer product/privacy sign-off first.
