## Summary

Keep open: current main now lets users edit one cost-history window, but it still does not support showing multiple configurable cost periods such as 7d, 30d, and 90d at the same time.

## What This Changes

The issue requests a UI option to show multiple cost-history periods at once, such as 7d, 30d, and 90d.

## Best Possible Solution

Decide a compact multi-period UX and data model, preferably deriving shorter periods from one sufficiently wide daily history where possible, then add focused settings/menu/dashboard tests around the selected behavior.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that current main supports one configurable cost-history window and does not expose multiple simultaneous 7d/30d/90d cost periods.

## Solution Assessment

Unclear until maintainers choose the UX and storage shape. The existing single-window editor is useful, but it is not the full multi-period feature requested here.
