## Summary

Keep open: current main still supports multiple accounts only inside provider menus, while status bar items remain merged-or-provider scoped. The later comments add useful demand signal and a concrete opt-in proposal, but the requested per-account status item mode is a new product/UX direction that needs maintainer sign-off.

## What This Changes

Allow users to show multiple accounts from the same provider as separate status bar icons so each account's usage can be tracked without opening the menu.

## Best Possible Solution

Keep this open for maintainer sign-off on an opt-in, provider-agnostic account-keyed status item design that leaves existing provider icons and in-menu account displays unchanged by default.

## Reproduction Assessment

Not applicable; this is a request for a new status bar display mode, not a bug. Source inspection gives high confidence that current main only offers same-provider multi-account visibility inside menus.

## Solution Assessment

Unclear; the per-account opt-in proposal is plausible, but maintainers should choose the UX, cap, and persistence model before implementation because current main already has segmented and stacked in-menu account displays.
