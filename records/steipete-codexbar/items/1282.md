## Summary

Keep open: current main still keeps general app/UI preferences in UserDefaults while the JSON config remains provider-focused, and moving portable UI preferences into JSON is a new data-storage behavior that needs maintainer product judgment.

## What This Changes

Request to store general UI/application preferences in a dotfile-friendly `config.json` preferences block or separate `preferences.json`.

## Best Possible Solution

Design a narrow documented schema for portable non-secret preferences, with explicit precedence over UserDefaults, upgrade behavior, and exclusions for device-local or sensitive state.

## Reproduction Assessment

Not applicable: this is a feature request rather than a bug. Source inspection confirms the current behavior the reporter described: provider config is JSON-backed, while the named UI preferences still use UserDefaults.

## Solution Assessment

Unclear until maintainers choose the product boundary. A separate portable preferences schema may be safer than putting every UI setting into the secret-bearing provider config file.
