This should stay open: current main and the latest release still do not render reserve/deficit pace for MiMo, and the related implementation was closed unmerged after maintainer review because monthly-only inference can mislead annual-plan users.

## What This Changes

The issue asks for Xiaomi MiMo token-plan usage to show the same reserve/pace indicator available for other providers.

## Best Possible Solution

Keep this issue open until MiMo cadence can be classified safely; then reuse the shared pace UI for MiMo primary token-plan windows and fail closed with windowMinutes nil when cadence is ambiguous.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection confirms current main leaves MiMo token windows without windowMinutes and the menu pace path ignores MiMo primary windows.

## Solution Assessment

Unclear until the API/product direction is settled. The narrow maintainable path is shared pace rendering only when monthly or annual cadence is authoritative, with ambiguous plans suppressed rather than inferred.
