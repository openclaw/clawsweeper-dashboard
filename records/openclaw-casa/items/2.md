## What This Changes

The PR maps non-finite HomeKit numeric values to JSON null before encoding and logs JSON encoding failures instead of silently swallowing them.

## Best Possible Solution

Land the narrow JSON mapping fix after adding inspectable redacted HomeKit or CLI proof, ideally with a focused XCTest that asserts NaN and Infinity metadata encode as JSON null without collapsing the envelope.

## Reproduction Assessment

Yes from source inspection, though not from a live HomeKit run here. Current main can place a non-finite `NSNumber` into `JSONValue.number`, and `encodedData()` falls back to literal `null` when `JSONEncoder` rejects that value.

## Solution Assessment

Yes, mapping non-finite numeric HomeKit values to JSON null at the `jsonValue(_:)` boundary is the narrowest maintainable fix. The logging change is non-essential but not a correctness blocker; a focused regression test would make the path safer.
