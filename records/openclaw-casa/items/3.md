## Summary

The proposed tunnel/node/heartbeat fields do not map to any current Casa tunnel subsystem, so they should be tracked separately if that product surface is added later.

## What This Changes

The issue requests a `GET /health` or `/status` endpoint that returns JSON for Casa proxy/tunnel health, connected nodes, and last heartbeat.

## Best Possible Solution

Keep the existing `/health` endpoint as Casa's health-check surface; open a separate feature only if Casa gains a tunnel subsystem with real node and heartbeat state to expose.
