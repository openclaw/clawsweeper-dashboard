## Summary

Keep open. Current main still lacks a built-in guard against this Python-version-specific false positive: the default review prompt only gives general low-evidence guidance, Python source slices do not automatically include project metadata, and the project model does not carry a target Python version.

## What This Changes

Prevent Clawpatch review providers from reporting Python 3.14 PEP 758 exception syntax as invalid Python.

## Best Possible Solution

Add default prompt and/or mapper support so Python syntax validity is checked against the project target version, explicitly covering PEP 758 for Python 3.14+, with focused prompt or mapper regression tests.

## Reproduction Assessment

No, not with a live provider run in this read-only review. The issue includes a concrete provider output, and source inspection shows the default prompt and Python feature context do not currently provide the Python 3.14 guard needed to avoid that class of false positive.

## Solution Assessment

No patch is proposed. The narrow maintainable path is to teach the default review prompt and, if needed, Python feature context to respect configured target Python syntax rather than asking every user to pass custom reviewer guidance.

## Repair Work Prompt

Fix the Python PEP 758 false-positive review path. Current main's default review prompt gives only general low-evidence guidance, Python source-group features do not include project metadata as context, and ProjectRecord.detected has no Python target-version field. Add a narrow default safeguard so review providers do not report Python syntax as invalid unless prompt context proves it is invalid for the configured target runtime. Cover PEP 758 specifically: unparenthesized except/except* multi-exception handlers without an as clause are valid for Python 3.14+. Prefer prompt guidance plus, if needed, adding Python metadata context to Python source/route/CLI features so pyproject requires-python can be seen. Add focused tests in src/prompt.test.ts and/or src/mapper.test.ts. Do not edit dist or change provider schemas unless necessary.
