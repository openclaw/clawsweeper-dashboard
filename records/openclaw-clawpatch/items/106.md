Add version-aware Python context and review guidance so Clawpatch treats PEP 758 syntax as valid for Python 3.14+ while still flagging it for older target versions, with focused regression coverage.

## Repair Work Prompt

Fix the Python 3.14 PEP 758 false-positive review path. Clawpatch should not report `except TypeError, ValueError:` or equivalent unparenthesized multi-exception handlers as invalid Python when repository metadata targets Python 3.14 or newer, while preserving findings for projects targeting older Python versions. Keep the change narrow: surface `.python-version` and/or `pyproject.toml` `requires-python` evidence to Python feature context or the default review prompt, add focused regression coverage, and avoid broad provider, report schema, or unrelated mapper rewrites. Validate with focused prompt/mapper tests first, then the standard checks if feasible.
