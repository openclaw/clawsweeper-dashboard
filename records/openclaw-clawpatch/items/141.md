## Summary

Keep open: current main has provider-level sandboxing and adjacent Crabbox setup, but it does not run patch agents in a forked per-task box or cloned isolated checkout.

## What This Changes

Request an isolated, forkable execution substrate for patch-agent repair runs, potentially via Crabbox or mitos, so each patch attempt is isolated from siblings and the host.

## Best Possible Solution

Add a documented execution-substrate layer that can run each repair attempt in an isolated per-task checkout or box with recorded lease/proof metadata, while keeping provider security claims honest.

## Reproduction Assessment

Not applicable: this is a feature/security architecture request, not a bug with runtime reproduction steps. Source inspection still verifies the current gap because `clawpatch fix` invokes the provider in the active checkout and docs tell users to supply isolated checkouts for untrusted write-mode runs.

Unclear: per-task isolation is directionally aligned with the safety vision, but selecting Crabbox, mitos, defaults, and state/proof behavior needs maintainer product and security direction first.
