## Summary

Keep open: current main has provider sandbox knobs and Crabbox setup, but the repair path still runs the selected provider against the current checkout and the docs still place isolated-checkout responsibility on the caller.

## What This Changes

The issue requests a warm, isolated execution substrate that can fork a fresh per-task box for patch-agent runs.

## Best Possible Solution

Add a documented execution-substrate layer that can run each patch attempt in an isolated per-task checkout or box with recorded lease/proof metadata, while keeping provider security claims honest.

Not applicable: this is a feature/security architecture request, not a bug with runtime reproduction steps.
