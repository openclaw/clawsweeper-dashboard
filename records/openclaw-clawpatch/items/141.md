## Summary

Keep open: current main has provider-level sandboxing and a merged Crabbox setup, but `clawpatch fix` still runs the provider and validation commands in the caller's checkout rather than a forked per-task box.

## What This Changes

Request for Clawpatch to run patch/repair agents in isolated, forkable per-task boxes instead of directly in the caller's checkout.

## Best Possible Solution

Add a maintainer-approved execution-substrate layer for repair attempts with per-task isolated checkouts or boxes, explicit default policy, and recorded lease/proof metadata while keeping provider security claims honest.
