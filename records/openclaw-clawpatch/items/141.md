## Summary

Keep open: current main has provider-level sandboxing and a merged Crabbox setup, but patch fixes and validation still run in the caller's checkout rather than a forked per-task box.

## What This Changes

The issue asks clawpatch to run patch agents in an isolated, forkable execution box such as Crabbox or mitos.

## Best Possible Solution

Add a documented execution-substrate layer that can run each repair attempt in an isolated per-task checkout or box with recorded lease and proof metadata, while keeping provider security claims honest.
