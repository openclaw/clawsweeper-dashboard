Full review comments:

- **[P1] Keep limiter waits referenced until review resumes:** `src/rpm-limiter.ts:13`
  - body: The RPM limiter sleeps by scheduling the default clock's timeout, but that timeout is immediately `unref()`ed. When all workers are between provider calls and the only pending work is waiting for the next RPM slot, Node can exit even though `reviewCommand` is still awaiting `limiter.acquire()`, leaving remaining features unreviewed and the run unfinished. Use a referenced timer, or another mechanism that keeps the CLI alive while throttled.
