## Label Justifications

- `P1`: The issue reports command injection through validation commands executed by the shell in normal mapper-generated workflows.

## Summary

Keep open: current main still assembles mapper validation commands with unquoted repo-derived paths, scripts, filters, and test paths that later flow to shell execution, and the open linked fix PR is the right active path rather than a reason to close early.

## What This Changes

Quote mapper-generated validation command arguments so repo-derived paths, package names, task names, filters, and test paths are treated as literal shell arguments.

## Best Possible Solution

Review and land the linked fix PR if it preserves existing simple command output while quoting all repo-derived mapper arguments before shell execution.
