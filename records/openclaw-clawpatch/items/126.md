## Summary

Keep open: the provider idea is not on current main, but the patch currently exposes a direct API provider as a write-capable fix provider even though it cannot edit the worktree, and it also misclassifies MiniMax HTTP failures.

## What This Changes

The branch adds a MiniMax fetch-based provider with env-based configuration, provider dispatch/tests, and a README provider-list entry.

## Best Possible Solution

Accept the MiniMax provider only after the supported operation set is made honest, HTTP exit codes match the provider contract, docs cover the final surface, and real behavior proof covers the final supported operations.

## Reproduction Assessment

Yes from source inspection: the new direct HTTP `fix` method can only return JSON, while current `fixCommand` records a non-failing provider result as an applied patch based on snapshots. I did not run it because this review is read-only.

## Solution Assessment

No: adding a MiniMax provider is plausible, but this implementation should either be review/map/revalidate-only or provide a real edit-capable fix path, and HTTP failure exit codes should preserve the existing provider contract.
