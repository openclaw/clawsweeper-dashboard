## Summary

Keep open: the mount-prefix mapper work is not on main and the patch is useful, but it still has a dynamic-mount correctness gap and lacks required real behavior proof before merge.

## What This Changes

The PR updates the Node route mapper, tests, and changelog to prefix Express router/app and Hono sub-app routes with literal mount paths while suppressing dynamic-only mounted child routes.

## Best Possible Solution

Land a mapper change that preserves literal Express/Hono mount prefixes and suppresses any dynamic-only child route unless the first argument is proven to be middleware, with focused regression coverage and real CLI proof.

## Reproduction Assessment

Yes: source inspection is enough to reproduce the remaining defect because `app.use(version, router)` with `router.get('/users')` falls through the added dynamic-prefix heuristic and still emits raw `/users`. I did not run tests because this review must keep the checkout read-only.

## Solution Assessment

No: the prefix-resolver direction fits the existing mapper, but the dynamic-prefix heuristic is too narrow for non-middleware identifiers such as `version`. After that gap and real behavior proof are addressed, the approach looks maintainable.
