## Summary

Keep open. Current main reads cell hyperlinks, including rich-text link runs, and can batch-update plain values, but it does not expose `gog sheets links set` or any write path for multi-link `textFormatRuns`; the request fits the repo vision as a bounded Sheets primitive.

## What This Changes

Add `gog sheets links set` to write single-link and multi-link Sheets cells, including batch JSON input backed by `updateCells` and `textFormatRuns`.

## Best Possible Solution

Add a backwards-compatible `links set` command that builds one `spreadsheets.batchUpdate` request for single-cell and batch hyperlink writes, with focused unit tests, generated docs, and safety-profile handling for the new write surface.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken existing contract. Source inspection has high confidence that current main only reads hyperlinks and that the existing batch update path writes plain value ranges.

## Solution Assessment

Yes, the requested direction is a maintainable fit if it preserves the existing read command and uses `updateCells`/`textFormatRuns` for rich-text cells. A formula-only implementation would not satisfy the multi-link cell requirement.
