Keep open: current main still preserves all-whitespace markdown table header rows in the local Docs markdown renderer, and whole-document Drive markdown replacement does not normalize the source before upload. Changing that visible table behavior needs a maintainer decision on default suppression versus an opt-in flag.

## What This Changes

Request that Docs markdown conversion drop an all-whitespace leading table header row, or expose an opt-in flag for that behavior.

## Best Possible Solution

Have maintainers choose the conversion policy, then implement shared all-whitespace header-row normalization with focused coverage for `docs write` replace/append, tab-local rendering, and `docs find-replace --format markdown`.
