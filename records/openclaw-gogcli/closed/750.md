## Summary

Keep open. This collaborator-authored PR adds useful capability discovery, but the MCP path currently lets the MCP caller opt into sensitive auth/account disclosure, which conflicts with the repo's untrusted-agent safety model and needs maintainer/security handling before merge.

## What This Changes

The PR adds `gog agent capabilities`, generated/docs coverage, and a read-only `gog_capabilities` MCP tool for reporting runtime safety, auth metadata, and filtered MCP tools.

## Best Possible Solution

Keep the CLI capability snapshot, but make MCP auth/account disclosure server-operator controlled or redacted by default, with docs and tests covering untrusted MCP deployments.

## Reproduction Assessment

Yes. Source inspection shows the default MCP tool accepts `include_auth` and `include_account` from the request and passes them directly into the capability builder, which can emit account/auth metadata.

## Solution Assessment

No. The CLI opt-ins are reasonable, but MCP disclosure should be authorized by the server operator rather than by the caller/model.
