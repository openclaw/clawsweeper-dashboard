## Summary

Keep open: current main still leaves `read-slide` structured output limited to shape text and image `contentUrl`, and `info` remains Drive metadata only. `slides raw` is a useful v0.28.0 workaround for lossless data, but it does not satisfy the requested structured `read-slide`/`info`/locator surface.

## What This Changes

Request to enrich Slides read/info output with element geometry, text styling, table-cell content, stable image source URLs, and a read-only text-to-element locator.

## Best Possible Solution

Design a stable Slides introspection contract that keeps `slides raw` as the escape hatch while adding table-aware structured detail output and a read-only locator with focused tests and generated command docs.

## Reproduction Assessment

Yes, from source inspection: `read-slide` only projects shape text and image `contentUrl`, and `info` delegates to Drive metadata. A live Google deck with a table-only slide would verify the exact user repro, but the missing code paths are clear on current main.

## Solution Assessment

Unclear until maintainers choose the API shape; the likely maintainable path is a stable, table-aware read/detail and locator API rather than relying solely on raw output. The `info` command also needs a product choice on whether it should stay Drive-backed or add Slides-native summary fields.
