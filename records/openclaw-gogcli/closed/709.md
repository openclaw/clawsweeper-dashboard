## Summary

Keep open: v0.22.0 and current main still route table deletion through the same `DeleteTableRequest`-only implementation, while the docs describe delete as removing the table object and the reporter supplied a detailed live data-loss reproduction.

## What This Changes

The issue requests that `gog sheets table delete` remove the Sheets table definition without clearing the cell values in the table range.

## Best Possible Solution

Implement a non-destructive table-delete path that preserves range values and formulas by default; keep intentional value removal in `sheets table clear` or a separately reviewed purge flag if maintainers want one.

## Reproduction Assessment

Yes: the reporter provided a concrete v0.22.0 live CLI script, and current main still has the same delete implementation from that release. I did not rerun the live Google Sheets flow in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to preserve values for the existing delete command and rely on `sheets table clear` for intentional value removal. A broader purge flag can be discussed later but is not needed to fix the data-loss bug.
