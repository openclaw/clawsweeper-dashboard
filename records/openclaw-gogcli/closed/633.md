## Summary

Keep open: current main still sends whole-document `--replace --markdown` through Drive's converter and has no post-conversion pass to replace raw `#slug` links with Docs heading fragments, matching the reporter's v0.18.0 repro.

## What This Changes

The issue asks for `gog docs write --replace --markdown` to convert Markdown in-document heading links like `[text](#slug)` into Google Docs `#heading=<headingId>` link targets.

## Best Possible Solution

Preserve the Drive converter for whole-document markdown import, then run a narrow Docs API post-process that maps generated heading IDs and rewrites only known internal fragment links.

## Reproduction Assessment

Yes. The reporter provided a concrete v0.18.0 live repro, and current main still follows the same Drive upload path without any post-conversion heading-link rewrite.

## Solution Assessment

Yes. The narrow maintainable fix is a post-process for matching in-doc fragment links after Drive assigns heading IDs, while leaving external URLs and unresolved fragments untouched.
