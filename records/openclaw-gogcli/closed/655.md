## Summary

Keep open: current main still derives `inheritFromBefore` only from `--after`, so users cannot independently request an after-position insert that inherits formatting from the following/default dimension.

## What This Changes

Request to add a `sheets insert` flag that controls the Sheets API `InsertDimensionRequest.inheritFromBefore` value and documents the command's default behavior.

## Best Possible Solution

Add a nullable `--inherit-from-before` boolean override to `sheets insert` while preserving today's omitted-flag default derived from `--after`.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: `--after` always sends `inheritFromBefore=true`, and there is no separate flag to send `false` for that insertion position. I did not run a live Google Sheets reproduction in this read-only review.

## Solution Assessment

Yes, the narrowest maintainable solution is an explicit nullable boolean flag that overrides the current derived value only when provided. Changing the default or adding a follow-up formatting clear would be less compatible and more surprising.
