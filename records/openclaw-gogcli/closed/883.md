## Summary

Current main already fixes the central quoted-draft bug: HTML-only reply text is converted into the plain alternative before the original message is quoted, and that fix shipped after the reported v0.21.0 version.

## What This Changes

The issue reports that `gmail drafts create --quote --body-html` without `--body` omits the reply text from the generated `text/plain` alternative.

## Best Possible Solution

Keep the shipped quoted-reply implementation; if maintainers want plain alternatives for all fresh HTML-only messages, track that as a separate enhancement.

## Reproduction Assessment

No on current main. Source inspection shows the quoted HTML-only draft path now derives a plain reply body before appending the quote, and regression coverage asserts that exact behavior.

## Solution Assessment

Yes for the central bug. The shared compose fix is narrower than draft-only special handling; the broader fresh HTML-only alternative behavior remains a separate product enhancement.
