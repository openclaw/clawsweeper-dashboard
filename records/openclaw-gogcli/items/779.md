## Summary

Keep open for normal maintainer handling: the PR is collaborator-authored, current main does not yet have the visible-readback fix, and the patch looks correct, but it changes the default success contract for Classroom state mutations.

## What This Changes

The PR changes Classroom course archive, unarchive, and update-state commands to wait until `Courses.Get` returns the requested course state, with docs and unit coverage.

## Best Possible Solution

Land the bounded Classroom state readback behavior if maintainers accept visible readback as the success contract for state mutations; otherwise add an explicit compatibility escape hatch before merge.

## Reproduction Assessment

Source inspection shows current main can report success from `Patch` before a later `Courses.Get` observes the new state. The PR body also reports a live Classroom archive/delete E2E, but I did not run a live reproduction in this read-only review.

## Solution Assessment

Yes, provided maintainers accept the compatibility tradeoff: the proposed path is a narrow read-after-write check for course state changes and uses the existing retryable exit-code convention.
