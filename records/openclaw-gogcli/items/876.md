## Summary

Keep open: current main and v0.31.0 still lack a user-facing Docs suggestions surface, and the existing Docs read paths do not request `SUGGESTIONS_INLINE`. The related version-history issue is closed but covers a different revisions surface, so this issue remains the canonical tracked-suggestions request.

## What This Changes

The issue requests read-only CLI access to pending Google Docs suggested edits via a `docs suggestions list` command or `--suggestions` flags on existing read paths.

## Best Possible Solution

Define a narrow read-only Docs suggestions surface that requests `SUGGESTIONS_INLINE`, walks insertion/deletion/style metadata, supports tab selection where feasible, and preserves parseable JSON/plain output.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection on current main gives a high-confidence gap check because the command structs, generated docs, and `Documents.Get` call sites lack a suggestions surface and view-mode setter.

## Solution Assessment

Unclear until maintainers choose the CLI contract. A narrow read-only `docs suggestions list` plus possibly `docs raw --suggestions` looks maintainable, while accept/reject mutation should stay out of scope.
