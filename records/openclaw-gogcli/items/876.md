## Summary

Keep open: current main and the latest release still lack a user-facing Docs suggestions surface, and the existing Docs read paths do not request `SUGGESTIONS_INLINE`. The request fits as a bounded Docs capability, but the new command/flag contract needs maintainer product review before implementation.

## What This Changes

Request to add a read-only Google Docs suggested-edits surface, likely via `docs suggestions list` and/or `--suggestions` flags on existing read commands.

## Best Possible Solution

Define a narrow read-only Docs suggestions surface that requests `SUGGESTIONS_INLINE`, walks suggested insertion/deletion/style metadata, and keeps stdout parseable for JSON and plain output.

## Reproduction Assessment

Not applicable as a bug reproduction, but source inspection gives a medium-confidence gap check: the current Docs command structs/docs and `Documents.Get` call sites lack a suggestions surface and `SuggestionsViewMode` setter. I did not run a live Docs API request against a document with pending suggestions.

## Solution Assessment

Unclear until maintainers choose the CLI shape. A narrow `docs suggestions list` plus optional `docs raw --suggestions` would fit the read-only boundary without promising accept/reject mutation.
