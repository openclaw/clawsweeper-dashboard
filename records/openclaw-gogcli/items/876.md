## Summary

Keep open: current main still lacks a Docs suggestions command or `--suggestions` read mode, and the existing Docs read paths do not request `SUGGESTIONS_INLINE`. The closed revisions work at https://github.com/openclaw/gogcli/issues/672 and https://github.com/openclaw/gogcli/pull/748 is adjacent, but it exposes Drive revision metadata rather than pending Docs suggestions.

## What This Changes

The issue requests a read-only Google Docs suggestions surface that fetches with `SUGGESTIONS_INLINE` and reports pending insertion/deletion suggestion data through parseable CLI output.

## Best Possible Solution

Define a narrow read-only Docs suggestions surface that requests `SUGGESTIONS_INLINE`, walks insertion/deletion metadata, supports tab selection where feasible, and preserves stable JSON/plain output without adding accept/reject mutation.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives a high-confidence gap check because current command structs, generated docs, and `Documents.Get` call sites lack a suggestions surface and view-mode setter.

## Solution Assessment

Unclear until maintainers choose the CLI contract. A dedicated read-only `docs suggestions list` plus optional raw support looks maintainable, while accept/reject mutation should remain out of scope.
