## Summary

Keep open: current main and the latest release still lack the requested Docs suggestions surface. Source inspection shows no `docs suggestions` command, no `--suggestions` read mode, and no Docs `Documents.Get` call that requests `SuggestionsViewMode("SUGGESTIONS_INLINE")`; the related revisions work covers Drive revision metadata, not pending Docs tracked changes.

## What This Changes

The issue requests a read-only Google Docs tracked-suggestions surface that can list or render pending suggested insertions and deletions from the CLI.

## Best Possible Solution

Define a narrow read-only Docs suggestions surface that requests `SUGGESTIONS_INLINE`, emits stable JSON/plain output, supports tab selection where feasible, and explicitly leaves accept/reject mutation out of scope.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high confidence that current main lacks the requested command or flag and never sets the Docs suggestions view mode.

## Solution Assessment

Unclear until maintainers choose the CLI contract. A dedicated read-only list command plus optional raw support looks maintainable, while accept/reject mutation should remain out of scope.
