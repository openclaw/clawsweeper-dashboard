## Summary

Keep open: the pull consumer is useful and well proven, but the branch still changes existing Pub/Sub push hook-failure behavior from HTTP 200 acknowledgement to HTTP 500 retry, which is a compatibility and delivery-risk blocker for existing deployments.

## What This Changes

This PR adds a Gmail Pub/Sub pull consumer command, shared Gmail watch processing, generated command docs, pull-focused tests, and the Google Pub/Sub v2 dependency.

## Best Possible Solution

Keep the new pull retry behavior, but preserve the released push acknowledgement policy unless maintainers explicitly accept a push-mode compatibility break with upgrade guidance.

## Reproduction Assessment

Yes for the patch defect: current main returns HTTP 200 after `sendHook` fails in `ServeHTTP`, while the PR returns through the HTTP 500 error path after `processGmailWatchPayload` reports the hook delivery error. This was established by source inspection rather than executing tests in the read-only checkout.

## Solution Assessment

No as currently implemented: sharing the processor is useful, but applying pull retry semantics to the existing push handler changes shipped compatibility behavior. The narrower fix is to keep push acknowledgement behavior separate while preserving pull redelivery semantics.
