## Summary

Keep open: the pull consumer is useful and has strong live proof, but the patch still changes shipped push hook-failure behavior from acknowledge-and-advance to HTTP 500 with cursor rollback, which can make existing Pub/Sub push deployments redeliver and backlog on downstream hook outages.

## What This Changes

This PR adds a Gmail Pub/Sub pull consumer command and generated docs, refactors push and pull through shared Gmail watch processing, adds pull-focused tests, and adds the Google Pub/Sub v2 dependency.

## Best Possible Solution

Land the pull consumer after preserving existing push hook-failure acknowledgement/advance behavior by default and documenting the new Pub/Sub subscriber credential requirement.

## Reproduction Assessment

Yes for the blocking compatibility finding: current main has code, docs, and a test proving push hook failures return HTTP 200 while recording `http_error`, and the PR changes that path to HTTP 500 with cursor rollback. The broader pull feature itself is not a bug reproduction case.

## Solution Assessment

No: the pull consumer direction is maintainable, but sharing retry/rollback semantics into the existing push handler is not the safest default. Preserve the shipped push behavior and keep the new pull retry behavior isolated or explicitly opted in.
