## Best Possible Solution

Keep the pull consumer, but split hook-failure policy so pull can nack and preserve the cursor while push preserves the current ack-and-advance behavior unless maintainers explicitly choose a breaking push retry mode.

## Solution Assessment

The narrower maintainable fix is to keep push ack-and-advance behavior separate from pull nack-and-retry behavior unless maintainers explicitly choose the breaking policy.
