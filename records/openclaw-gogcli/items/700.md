## Risks / Open Questions

- Existing Pub/Sub push setups that rely on hook failures returning success and advancing the cursor would now see HTTP 500 with cursor rollback, which can create redelivery or backlog during downstream outages.
- The pull runtime adds Google Pub/Sub subscriber credentials in addition to the existing Gmail OAuth path, so operators need an accepted credential and deployment model.
- The ack/nack policy is message-delivery-sensitive: invalid, wrong-account, stale, and no-new-message notifications are terminal, while hook, Gmail, and rate-limit failures are retryable.
- The work is coordinated with https://github.com/openclaw/rfcs/pull/8 and https://github.com/openclaw/openclaw/pull/90723, so maintainers should decide RFC acceptance and merge order before landing the stack.
