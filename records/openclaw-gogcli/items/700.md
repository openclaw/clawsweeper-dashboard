## Best Possible Solution

Land the pull consumer after preserving released push acknowledgement/cursor behavior as the default, keeping pull retry/rollback behavior, and coordinating the RFC plus downstream OpenClaw merge order.

## Solution Assessment

No as written: adding pull delivery is a good direction, but sharing the retry/rollback helper with push changes a released push contract. The narrower maintainable fix is to keep push compatibility and apply nack/rollback only to pull delivery unless maintainers explicitly choose a breaking push retry mode.
