## Risks / Open Questions

- Existing Pub/Sub push setups now receive HTTP 500 with cursor rollback on hook failure instead of HTTP 200 with cursor advance, so downstream outages can create redelivery and backlog where operators may not expect it.
- The pull runtime adds Google Pub/Sub subscriber credentials through ADC plus a direct `cloud.google.com/go/pubsub/v2` dependency; maintainers need to accept that credential and supply-chain surface.
- Ack/nack policy is message-delivery-sensitive: the PR intentionally acks invalid, wrong-account, and stale/no-message notifications while nacking hook/Gmail/rate-limit failures.
- The work is tied to the open RFC at https://github.com/openclaw/rfcs/pull/8 and companion OpenClaw integration at https://github.com/openclaw/openclaw/pull/90723, so merge order matters.
