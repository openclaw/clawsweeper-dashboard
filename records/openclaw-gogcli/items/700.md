## Risks / Open Questions

- Existing Pub/Sub push setups would now receive HTTP 500 with cursor rollback on hook failure instead of HTTP 200 with cursor advance, so downstream hook outages can create redelivery/backlog where operators may not expect it.
- The pull runtime adds a direct Google Pub/Sub dependency and requires ADC or equivalent subscriber credentials in addition to the existing Gmail account auth path.
- The ack/nack policy is message-delivery-sensitive: invalid, wrong-account, stale, and no-new-message notifications are terminal, while hook, Gmail, and rate-limit failures are retryable.
- The work is coordinated with https://github.com/openclaw/rfcs/pull/8 and https://github.com/openclaw/openclaw/pull/90723, so maintainers should decide the RFC and merge order before landing the stack.
