## Risks / Open Questions

- Existing Pub/Sub push setups that rely on hook failures returning success and advancing the cursor would now receive HTTP 500 with cursor rollback, which can create redelivery or backlog during downstream outages.
- The pull runtime adds Google Pub/Sub subscriber credentials and a new Google Cloud client dependency alongside the existing Gmail OAuth path.
- The ack/nack policy is delivery-sensitive: invalid, wrong-account, stale, and no-new-message notifications are terminal, while hook, Gmail, and rate-limit failures are retryable.
- The stack is coordinated with https://github.com/openclaw/rfcs/pull/8 and https://github.com/openclaw/openclaw/pull/90723, so maintainers need to own merge order before landing.
