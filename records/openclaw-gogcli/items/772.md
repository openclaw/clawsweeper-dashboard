## What This Changes

The PR changes auth token export/listing so export can use a non-migrating token read and auth list stops fallback keyring traversal after keyring timeouts.

## Best Possible Solution

Land the focused auth/keyring hardening once maintainers accept the remaining cross-keyring compatibility risk and normal required checks finish cleanly.

## Reproduction Assessment

## Solution Assessment

Yes: using a non-migrating token read only for export and short-circuiting auth-list fallback on keyring timeout is the narrowest maintainable fix while preserving normal legacy migration.
