## Summary

Keep open: current main has internal text-to-range helpers for existing mutation commands, but it does not expose a standalone `docs find-range` command or the requested range-reporting, whitespace-normalized, entity-decoding behavior.

## What This Changes

Request adds a new `gog docs find-range <docId> <text>` primitive that returns Google Docs API start/end index ranges for literal text matches.

## Best Possible Solution

Add a shared Docs text-range resolver and expose `docs find-range` with stable JSON/TSV output, then let the related `--at <text>` wrappers build on that resolver.

## Solution Assessment
