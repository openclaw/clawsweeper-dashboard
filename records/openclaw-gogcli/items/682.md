## Summary

Keep open: current main has private Docs text-range helpers for existing mutation commands, but no standalone `docs find-range` command or the requested normalized, entity-decoding, range-reporting surface.

## What This Changes

Add `gog docs find-range <docId> <text>` to map literal document text to Google Docs start/end index ranges with occurrence, normalization, and tab support.

## Best Possible Solution

Add a focused `docs find-range` command backed by a shared range resolver, with tests for split text runs, table traversal as needed, normalized whitespace, entity-decoded needles, occurrence selection, tab IDs, and parseable JSON/TSV output.

## Reproduction Assessment

Yes. Source inspection shows current main does not register or document `gog docs find-range`, while the existing text locator is private to mutation paths.

## Solution Assessment

Yes. A shared `find-range` primitive is the narrow maintainable path because it avoids duplicating textRun walking and index math across each by-text Docs command.
