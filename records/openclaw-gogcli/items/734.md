## Summary

Current main partially addresses the request with `gog docs paragraphs list`, but the issue remains necessary because that output flattens paragraph text and does not expose the requested per-run formatting/link state or an explicit emptiness field. The existing enumerator is the right foundation, so this should stay open as a narrow additive feature rather than create another inspection path.

Likely related people
- Peter Steinberger — high confidence; introduced the current Docs content enumerators and has the recent adjacent Docs command history.

## What This Changes

Extend Docs inspection with structured paragraph and table metadata, including ranges, named styles, emptiness, and per-text-run formatting and links.

## Best Possible Solution

Add run-level style/link data and an explicit emptiness value to the existing paragraph enumerator, reuse its table and tab traversal, preserve existing fields, and document how non-text elements and inherited style values are represented.

## Reproduction Assessment

Not applicable; this is a new-capability request, and source inspection gives a high-confidence field-by-field comparison showing that current main lacks per-run formatting and link output.

## Solution Assessment

Yes in direction, but the narrowest maintainable solution is to extend the existing `docs paragraphs list` enumerator additively rather than add a parallel `outline` parser or duplicate the raw-document traversal.
