## Summary

Keep open: current main still preserves all-blank markdown table header rows, and the whole-document Drive conversion path uploads the markdown without any table-header normalization. The requested behavior is useful but changes visible table semantics, so maintainers should first choose default suppression versus an opt-in flag.

## What This Changes

Drop or optionally suppress an all-whitespace markdown pipe-table header row so label-value tables do not render with an empty first row.

## Best Possible Solution

Have maintainers choose the policy, then implement one shared markdown-table normalization path used before Drive upload and local Docs markdown rendering, with focused tests proving only all-whitespace header rows are removed and data rows are preserved.

## Reproduction Assessment

Yes from source inspection for the local markdown renderer, and the issue supplies a v0.18.0 live Drive reproduction for whole-document replace. I did not run a live Docs API write during this read-only review.

## Solution Assessment

Unclear: default suppression is likely the cleanest UX for label-value tables, but current tests preserve the empty row, so maintainers should confirm default behavior versus an opt-in flag before implementation.
