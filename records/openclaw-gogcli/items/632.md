## Summary

Keep open: current main still preserves all-whitespace markdown table header rows, and neither the Drive upload path nor the local Docs markdown paths have a shared drop-empty-header policy or flag. Dropping that row would change visible table semantics, so maintainers should choose default suppression versus opt-in behavior before implementation.

## What This Changes

Request to make Docs markdown conversion drop an all-whitespace pipe-table header row, or add an opt-in flag, so label-value tables do not render a blank top strip.

## Best Possible Solution

Have maintainers choose the conversion policy, then add shared all-whitespace header-row normalization for markdown tables with focused tests proving only the leading blank header row is removed and data rows remain intact.

## Reproduction Assessment

Yes. The issue supplies a v0.18.0 live Drive reproduction, and current source plus tests still show the empty header row is preserved; I did not perform a live Docs API write during this read-only review.

## Solution Assessment

Unclear. Default suppression may be the better label-value table UX, but current tests preserve the row, so maintainers should choose default behavior versus an opt-in flag before code changes.
