## Label Justifications

- `P2`: Current main deterministically introduces unwanted document structure, but the defect is bounded to the markdown range-replacement workflow.
- `impact:other`: Repeated markdown replacements can accumulate unintended empty paragraphs in user documents.

## Summary

Current main still reproduces the reported defect at source level: a one-line markdown replacement is rendered with a synthetic terminal newline and inserted inside the existing paragraph. No merged or open PR currently owns this specific fix, so the issue remains necessary and is suitable for a focused repair PR.

## What This Changes

Prevent `docs find-replace --format markdown` from adding an empty paragraph when replacing inline text inside an existing paragraph.

## Best Possible Solution

Make markdown range replacement distinguish an inline paragraph fragment from block markdown, omitting only the synthetic terminal paragraph delimiter for the inline case while preserving explicit source newlines and structural boundaries.

## Reproduction Assessment

Yes at source level. Current main deterministically parses the replacement as one `MDParagraph`, appends `\n`, and inserts that result inside the existing paragraph; a credentialed live Docs reproduction was not performed during this read-only review.

## Solution Assessment

Yes, provided the fix is scoped to range-replacement context. Changing the global formatter would be unsafe because other markdown operations require terminal paragraph delimiters for valid block structure.
