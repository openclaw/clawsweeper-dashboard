## What This Changes

The PR changes `scripts/check-docs-coverage.mjs` to validate same-page and cross-page Markdown fragment anchors in addition to linked Markdown file paths.

## Best Possible Solution

Fix the source checker to mirror rendered docs headings, especially fenced code handling, then add redacted terminal output showing the anchor check catching a real broken anchor and passing valid docs.

## Reproduction Assessment

Yes, from source inspection: the PR head scans every raw `#` line as a heading, and current docs contain heading-looking lines inside fenced examples that the renderer does not expose as anchors.

## Solution Assessment

No; the implementation should track fenced code blocks or reuse rendered heading data before this becomes the standalone Markdown anchor validator.
