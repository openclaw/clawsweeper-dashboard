Keep open: this is the active candidate for the linked auth-provider bug, but the current patch should not merge until `ForceRefresh` preserves the old token source when a forced refresh fails.

## What This Changes

The PR adds a stored-OAuth-only one-time retry for replayable Google API 403 insufficient-scope responses, refreshes and persists scoped OAuth tokens, and adds auth retry/concurrency tests plus a changelog entry.

## Best Possible Solution

Keep the narrow one-time retry, but make `ForceRefresh` commit the new source only after a successful token mint and cover the failed-refresh path before merge.

## Reproduction Assessment

No high-confidence live reproduction of Google's intermittent first-response 403 is available here. Source inspection does reproduce the patch defect: `ForceRefresh` assigns the new source before returning an error from `Token`.

## Solution Assessment

No, the proposed direction is right but the implementation should preserve the previous token source until forced refresh succeeds. That is the narrowest maintainable repair before merge.
