Full review comments:

- **[P2] Preserve scope checks on cache hits:** `internal/googleapi/client_auth.go:229-231`
  - body: The new cache returns a token source before opening stored auth and before `requireStoredGrant` validation. Since cache entries are written with the requested scopes rather than scopes observed from the token response, a stale or over-broad cache entry can bypass the insufficient-grant error path and send callers to the API with a token that may not actually cover the requested service. Please cache/validate observed granted scopes, or do not use the cache before the stored-grant check can run.
  - confidence: 0.86
