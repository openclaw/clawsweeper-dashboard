## Summary

Keep open: current main does not add `--tab` or `--all-tabs` to the dedicated `gog docs raw` command. There is adjacent tab-aware behavior in `gog docs cat --raw --all-tabs`, but it is not the requested command surface and does not settle the raw output-shape decision.

## What This Changes

Requested feature: add tab-aware output modes to `gog docs raw`, including `--tab` for one tab and `--all-tabs` for multi-tab document bodies.

## Best Possible Solution

Expose tab-aware `docs raw` behavior using the existing tab traversal and title-or-ID resolution helpers, with tests and help text that preserve the no-flag raw API shape and clearly define the flagged output shape.

## Reproduction Assessment

Yes from source inspection: `DocsRawCmd` has only `docId` and `--pretty`, and it calls `Documents.Get` without `IncludeTabsContent`. I did not run the live Google Docs mutation repro because current source already establishes the missing CLI surface.

## Solution Assessment

Unclear on the exact output shape: adding tab flags is the right narrow direction, but maintainers should decide whether `--all-tabs` returns a custom envelope or the canonical `Documents.Get` response with `tabs` populated.
