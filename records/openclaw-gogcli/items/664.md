## Summary

This should stay open: current main still ignores `--account` for the existing YouTube videos and comments list commands, so users without a YouTube API key still hit the API-key-required path even when they have OAuth credentials.

## What This Changes

Request that `gog yt videos list` and `gog yt comments list` honor `--account` by using OAuth when supplied and falling back to API-key reads otherwise.

## Best Possible Solution

Honor an explicitly supplied account for videos/comments through the existing OAuth service helper, keep the no-account API-key fallback unchanged, and add focused tests plus live Google proof for both command paths.

## Reproduction Assessment

Yes, at source level rather than through a live Google run. Both affected `Run` methods unconditionally choose the API-key service, so the reported `--account` path cannot work on current main.

## Solution Assessment

Yes. The narrow maintainable fix is to branch on `flags.Account` in those two commands, preserve the existing API-key fallback when no account is supplied, and cover both selectors with unit tests.
