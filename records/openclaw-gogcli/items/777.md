## Summary

Current main and released builds since v0.12.0 already embed Go's IANA timezone database for Windows, which addresses the reported v0.11.0 `unknown time zone` Calendar failure path.

## What This Changes

Calendar commands on Windows should resolve IANA timezones such as `Africa/Johannesburg` instead of failing before listing events.

## Best Possible Solution

Keep the existing tzdata embedding and direct affected users on v0.11.0 to upgrade to v0.12.0 or newer.

## Reproduction Assessment

No current-main failing reproduction was run. The report gives clear v0.11.0 Windows steps, and current source plus release history show the missing timezone database is now embedded.

## Solution Assessment

Yes. Embedding `time/tzdata` through the CLI entrypoint is the maintainable fix for Go Windows binaries that need IANA timezone names, and the current helper path continues to use `time.LoadLocation`.
