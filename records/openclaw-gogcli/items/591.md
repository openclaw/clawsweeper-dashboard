## Repair Work Prompt

Fix the Calendar Zoom description clear bug reported in https://github.com/openclaw/gogcli/issues/591. In `gog calendar update --remove-zoom`, when removing the gog-managed Zoom block leaves `patch.Description == ""`, make the Calendar PATCH explicitly clear the description, likely by appending `"Description"` to `patch.ForceSendFields`. Preserve mixed-content behavior: user text around the managed Zoom block must remain, and unrelated update paths must not clear descriptions accidentally. Add focused regression coverage in `internal/cmd/calendar_zoom_test.go` that inspects the PATCH JSON body for a Zoom-only description (`description` sent as `""`) and a mixed-content description (managed block removed, surrounding text preserved). Do not change Zoom API deletion semantics, cross-provider mutex behavior, auth storage, or add new flags. Update changelog only if repository landing policy requires it.
