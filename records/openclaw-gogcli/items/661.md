## Summary

Keep open: current main still lacks the requested user-facing Sheets range copy/fill primitive. The existing `sheets copy` is spreadsheet-level, `sheets raw` is `Spreadsheets.Get`, and `sheets batch-update` is `values.batchUpdate`; the only current `CopyPasteRequest` use is the data-validation helper, not formula/format fill.

## What This Changes

Add a Sheets range copy/fill command that maps to `spreadsheets.batchUpdate` `CopyPasteRequest` and/or `AutoFillRequest` for formula/format propagation with relative reference adjustment.

## Best Possible Solution

Add a focused Sheets `copy-paste` and/or `fill` primitive that reuses existing A1/named-range resolution, emits dry-run/JSON output, documents the command, and proves the real Google API behavior without turning this into a broad raw mutation wrapper.
