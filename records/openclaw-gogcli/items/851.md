## Summary

Keep open: current main still exposes `docs insert` as a plain-text-only command, while the existing `docs update --markdown` path proves the position-aware markdown machinery already exists and can be reused.

## What This Changes

Add a `--markdown` flag to `gog docs insert` so positional inserts can use the existing Markdown-to-Google-Docs formatting path.

## Best Possible Solution

Add markdown mode to `docs insert` by reusing the existing position-aware markdown insertion helper, with regression coverage for explicit index, tab threading, parseable output, and unchanged plain-text behavior.

## Reproduction Assessment

Yes for the current missing capability: source inspection shows `DocsInsertCmd` has no `Markdown` field and always builds a plain `InsertTextRequest`; generated command docs also list no `--markdown` flag. I did not run the CLI because this is a read-only review and tests/builds may create artifacts.

## Solution Assessment

Yes, if maintainers accept the new flag: the narrowest maintainable solution is to reuse `prepareMarkdown`, `insertPreparedDocsMarkdownAt`, and the existing heading-link rewrite logic from `docs update --markdown`, rather than adding another renderer.
