## Summary

Keep this PR open: it fixes a real Gmail signature escaping bug and has live Gmail proof, but the new `LooksLikeHTML` regex still has a blocking false-positive path for angle-bracketed plain text such as one-letter email addresses.

## What This Changes

The PR broadens Gmail HTML detection to fragments, removes duplicate backup fragment detection, carries HTML-looking text/plain replies into quoted HTML, and adds `LooksLikeHTML` regression tests.

## Best Possible Solution

Keep the fragment-detection fix, but require an explicit tag terminator or attribute boundary plus regression tests for one-letter angle-bracketed addresses before merging.

## Reproduction Assessment

Yes. Source inspection shows current main accepts HTML signature files but only preserves raw HTML when `LooksLikeHTML` recognizes the content, and that detector misses common fragment starts like `<style>` and `<table>`; the PR body also includes live raw RFC822 before/after proof.

## Solution Assessment

No, not quite as written. The shared-helper approach is maintainable, but the regex boundary must be tightened so angle-bracketed plain text with short tag-like prefixes is still escaped or displayed literally.
