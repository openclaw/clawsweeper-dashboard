## Summary

This should stay open as a feature/product-design request: current main has agent helpers, MCP tooling, auth listing, service scope metadata, and command safety controls, but no dedicated auth.md-style capability discovery surface for agents or MCP clients.

## What This Changes

The issue requests optional machine-readable auth and capability discovery for agent and MCP clients, such as a `gog agent auth-info --json` command, MCP capability tool, or well-known capabilities endpoint.

## Best Possible Solution

Keep this open for a small design pass that defines a redacted capability snapshot schema first, then add the CLI and MCP surfaces once maintainers settle the disclosure defaults and supported transports.
