Add a validated action path to the existing Drive reply flow, reuse shared reply helpers, preserve current content-only behavior, and require unit plus live Drive API proof before merge.

## Repair Work Prompt

Implement a narrow feature for openclaw/gogcli: add Drive Reply action support to `gog drive comments reply`, accepting only `resolve` or `reopen`, sending the selected action in `drive.Reply.Action` to `Replies.Create`, and preserving existing content-only reply behavior when no action is passed. Prefer `--action` on `reply`; if sibling `resolve`/`reopen` commands are added for discoverability, keep them thin aliases over the same helper. Allow content to remain required unless maintainers choose to support action-only replies; if action-only replies are supported, validate empty content only when neither content nor action is present. Include action in dry-run output, update generated command docs/schema and `CHANGELOG.md`, and add focused tests that assert the request JSON contains `action` for both values and rejects invalid action. Do not change the Google Docs anchoring/browser-automation work tracked at https://github.com/openclaw/gogcli/issues/585.
