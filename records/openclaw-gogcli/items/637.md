## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve runtime safety flags for tool executions:** `internal/cmd/mcp.go:53`
  - body: When `gog mcp` is started with runtime guards such as `--enable-commands`, `--disable-commands`, or `--gmail-no-send`, those checks only apply to the parent `mcp` command. The tool then runs `exec.CommandContext(ctx, self, args...)`, so the child process receives none of the parent flags and an agent can invoke commands outside the intended runtime policy unless the operator used env/config/baked profiles.
  - confidence: 0.91
- **[P2] Keep the module Go version aligned with main:** `go.mod:3`
  - body: This downgrades the module directive from `go 1.26.2` to `go 1.25.9`; CI uses `go-version-file: go.mod`, while docs/spec expects Go 1.26, so the PR changes build and language semantics unrelated to MCP. Leave `go 1.26.2` unless the toolchain change is intentional and documented.
  - confidence: 0.86
- **[P2] Split the Gmail default behavior out of this PR:** `internal/cmd/gmail_search.go:36`
  - body: This MCP PR also changes a no-query Gmail search from a validation error into a live `in:inbox` API query, with the same pattern in `gmail messages list`. That is a separate user-visible CLI behavior change and can surprise scripts that relied on the current validation failure, so keep it in a separate reviewed change with docs/tests or revert it here.
  - confidence: 0.79
