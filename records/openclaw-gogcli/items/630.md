## Summary

Keep open: current main still lacks CLI margin controls for the Docs write and page-layout paths, so this is a valid narrow feature request rather than something already implemented.

## What This Changes

Requests `--margin-left`, `--margin-right`, `--margin-top`, and `--margin-bottom` flags for `gog docs write` and `gog docs page-layout`, accepting point or inch values.

## Best Possible Solution

Add a shared Docs document-style margin helper and expose the four explicit margin flags on `docs write` and `docs page-layout`, with tests and regenerated command docs.

## Reproduction Assessment

Yes, from source and generated docs: current command structs and references list `--pageless`/`--layout` but no margin flags, and the shared style helper only updates `documentFormat`. This is a capability gap rather than a failing existing contract.

## Solution Assessment

Yes, the proposed four explicit flags map cleanly to the existing Docs `UpdateDocumentStyle` surface. The safest implementation is a shared parser/helper that updates only supplied margin fields and leaves page width to the companion issue.
