## What This Changes

Add text-anchor flags to five Google Docs mutation commands so callers can target literal text instead of Docs API character indices.

## Best Possible Solution

Build or land a shared Docs text-range resolver first, then wire these five commands through it with focused unit coverage, generated command docs, and live Google Docs proof for the mutating paths.

## Reproduction Assessment

Not applicable as a feature request; source and generated docs confirm the requested flags are currently absent from the five commands.

## Solution Assessment

Yes on direction, but the maintainable path is to settle the shared `find-range` primitive first and reuse it rather than adding five separate text walkers.
