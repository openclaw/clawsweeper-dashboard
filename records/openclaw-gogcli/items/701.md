## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Include inserted cell text in later table offsets:** `internal/cmd/docs_table_inserter.go:363-364`
  - body: `BuildNativeTableRequests` returns `tableEndIndex` using only the empty-table span, but `appendTableRequests` feeds that into `nextTableInsertOffset` for the next table. The old live path advanced `tableEndIndex` after every cell insert, so later tables were shifted by prior cell text too; this new path under-shifts any second table after a populated first table, matching the reported index-out-of-bounds failure.
  - confidence: 0.9
- **[P2] Preserve request indices when splitting a batch:** `internal/cmd/docs_mutation.go:317-320`
  - body: When the request list exceeds 500, the second chunk is sent with the original pre-chunk indices even though the first chunk has already mutated the document. If table-cell inserts or later table requests fall after the split point, those indices can be stale; keep dependent request groups together, recompute indices between chunks, or fall back before splitting them.
  - confidence: 0.78
