## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Include cell text when advancing later tables:** `internal/cmd/docs_table_inserter.go:355`
  - body: `appendTableRequests` advances the next table using `tableEnd`, but `BuildNativeTableRequests` returns only the empty-table structural length here. Because the same request stream also inserts text into earlier table cells, the next table's `Location.Index` is short by the UTF-16 length of that cell text, so multi-table markdown can insert the second table at the wrong placeholder. Accumulate the inserted lengths returned by `buildTableCellRequests` into the returned end and add a populated multi-table assertion.
  - confidence: 0.86
- **[P2] Preserve indices when splitting batches:** `internal/cmd/docs_mutation.go:319`
  - body: The splitter sends later chunks with the original precomputed request indices. Once the first chunk inserts body/table/cell text, later chunks are separate `batchUpdate` calls and no longer get same-batch index adjustment, so table-cell requests split after the 500th request can target stale positions. Either avoid splitting index-dependent table sequences or recompute/read back indices between chunks, with coverage for a >500-request table body.
  - confidence: 0.8
