## Summary

Keep open: current main has comment-listing primitives that expose quoted content, but `docs write --replace --markdown` does not run an orphan-comment pre-flight or expose the requested override flags. The default warn-vs-block behavior needs maintainer policy before implementation.

## What This Changes

The issue requests a pre-flight orphan-comment check for `gog docs write --replace --markdown`, with flags to check or allow orphaning anchored comments.

## Best Possible Solution

Add a focused docs write safety primitive that reuses the shared Drive comments helper, skips resolved comments, supports tab-scoped replace, keeps stdout parseable, and makes warn-vs-block semantics an explicit maintainer choice.

## Reproduction Assessment

Yes, at source level: current `DocsWriteCmd` has no orphan-check flags, and both markdown replace paths overwrite content without listing comments first. I did not run a live Google Docs reproduction because this depends on real Drive comments and credentials.

## Solution Assessment

Yes on the pre-flight direction, but unclear on the default behavior. A compatibility-safe warning default or explicit strict mode should be chosen before implementing a blocking exit path.
