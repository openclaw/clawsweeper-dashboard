## Summary

Current main and releases since v0.17.0 already enforce early dry-run exits for both Gmail label create and modify before account, service, or Gmail API work, with regression coverage for the invalid-token dry-run path.

## What This Changes

The issue asks for `gog gmail labels create` and `gog gmail labels modify --dry-run` to exit after local validation and before Gmail auth/service/write paths.

## Best Possible Solution

Keep the existing early `dryRunExit` calls and dry-run e2e coverage; users affected on v0.15.x should use v0.17.0 or newer.

## Reproduction Assessment

No against current main. The commands now call `dryRunExit` before `requireAccount` and `newGmailService`, while the original v0.15.x failure is source-confirmed by the fix diff.

## Solution Assessment

Yes. The current implementation puts the no-side-effect boundary after cheap local validation and before account, service, label resolution, or Gmail write APIs, which matches the requested fix.
